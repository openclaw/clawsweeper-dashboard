## Summary

Keep open: current main and v2026.6.8 still intentionally preserve explicit and legacy session model pins across `/new`, `/reset`, and gateway restart, while current main has improved visibility and `/model default` clearing. The remaining behavior is compatibility-sensitive product policy, not conservative cleanup.

## What This Changes

The issue asks for session model overrides set via `/model` to stop silently persisting across restarts and `/new`, or to become clearly visible and resettable.

## Best Possible Solution

Keep the source-aware contract unless maintainers choose a new policy: explicit pins stay visible and clearable, auto fallback state clears on reset, and any bulk reset or default-precedence change is designed with docs, status, doctor, and upgrade tests together.

## Reproduction Assessment

Yes, source-reproducible: current main intentionally preserves user/legacy model overrides through `/new` and Gateway reset, and model resolution prefers those persisted selections over configured defaults. I did not run a live gateway restart reproduction in this read-only cleanup pass.

## Solution Assessment

Unclear without maintainer product direction: current main already adds visibility and explicit per-session clearing, but changing reset/restart precedence for explicit `/model` pins would alter a documented session-selection contract. A source-aware bulk reset or resolver/reconcile path is safer than silently changing precedence.
