## Summary

Keep open. Current main intentionally preserves explicit and legacy session model overrides while clearing only auto-sourced fallback pins; the remaining request for `/new`/restart semantics, bulk clearing, and source-visible status still needs a maintainer model-selection contract decision.

## What This Changes

The issue asks for session-level `/model` overrides to stop silently masking changed config defaults across gateway restarts and new sessions.

## Best Possible Solution

Define the public model-selection source contract first, then add explicit source labeling and a guarded bulk/session reset path without silently erasing intentional user pins.

## Reproduction Assessment

Yes. Source and tests give a high-confidence current-main path: seed a session with a user or legacy `modelOverride`, change the configured default, then `/new` or `sessions.reset` preserves the override while auto-sourced fallback pins are the ones cleared.

## Solution Assessment

Unclear. Automatically clearing explicit pins conflicts with current docs and tests, so the safer solution is maintainer-approved source labeling plus an explicit clear/reconcile UX.
