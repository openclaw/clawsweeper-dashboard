## Summary

Keep open: current main has useful partial fixes and visibility, but explicit and legacy session model selections still intentionally survive restart, `/new`, and `sessions.reset`, so the remaining behavior needs a maintainer model-selection policy decision.

## What This Changes

The issue asks OpenClaw to stop stale session-level model overrides from silently masking changed configured default models after restart or reset.

## Best Possible Solution

Define one source-aware model-selection contract for explicit, legacy, auto, and runtime session model state, then align reset, status, doctor, `models set`, and any clear or reconcile command with that contract.

## Reproduction Assessment

Yes, at source level. Seed a session with a user or legacy `modelOverride`, change the configured default, then `/new` or `sessions.reset` still preserves that selection while `resolveSessionModelRef` continues to prefer it over the new default.

## Solution Assessment

Unclear. Unconditionally clearing explicit pins conflicts with current docs and tests, so the maintainable path is a maintainer-approved source contract plus explicit clear, migrate, reconcile, or status/doctor UX.
