## Label Justifications

## Summary

Keep open: current main and the latest release still preserve explicit and legacy session model pins across reset/restart, while only auto fallback pins are cleared. Status and `/model default` are partial mitigations, but the remaining `/new`/reset/default-refresh behavior needs maintainer policy approval.

## What This Changes

The issue asks OpenClaw to stop stale session model overrides from silently masking updated config defaults, especially across gateway restarts and `/new`, and to provide clearer/resettable override UX.

## Best Possible Solution

Define one source-aware model-selection contract for explicit, legacy, auto, and default-derived session model state, then align `/new`, `/reset`, `/model default` or clear/inherit, `session_status`, doctor/docs, and any bulk reset/reconcile workflow with that contract.

## Reproduction Assessment

Yes, at source level. Seed a session with a user or legacy `providerOverride`/`modelOverride`, change the configured default, then `sessions.reset` reapplies the preserved selection and `resolveSessionModelRef` returns it before the configured default.

## Solution Assessment

Unclear. Unconditionally clearing explicit pins is not the best automatic fix because current docs/tests preserve exact user selections, so maintainers need to choose the permanent reset/default/clear contract first.
