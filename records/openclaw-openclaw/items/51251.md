## Summary

Keep open: current main and the latest release make explicit session pins visible and clearable per session, but they deliberately preserve user and legacy model overrides across reset/restart; bulk reset or default-precedence changes remain maintainer-owned compatibility work.

## What This Changes

The issue asks OpenClaw to stop stale session-level model overrides from silently masking changed config defaults, or to provide clearer visibility and reset controls.

## Best Possible Solution

Keep explicit session pins source-aware and visible, then add any bulk reset, reconcile, or changed default-precedence behavior through a maintainer-approved resolver/doctor/CLI design with upgrade proof.

## Reproduction Assessment

Yes, source-reproducible: current main preserves user and legacy model overrides across reset and documents that explicit session route pins are left untouched by doctor. I did not run a live gateway restart.

## Solution Assessment

No, not as a silent precedence change: current source and docs treat explicit session model choices as user-owned state, so the safer path is a maintainer-approved clear/reconcile workflow or source-aware resolver policy.
