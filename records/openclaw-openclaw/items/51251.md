## Summary

Keep open: current main and v2026.5.22 still preserve user and legacy session model pins across reset/restart, while only auto fallback pins are cleared. Status/docs now add partial visibility, but the remaining reset/default-precedence behavior needs a maintainer policy decision.

## What This Changes

The issue asks OpenClaw to make persisted session model pins visible and resettable when config defaults change, especially across gateway restarts and `/new`.

## Best Possible Solution

Define one source-aware model-selection contract for explicit, legacy, auto, and default-derived session model state, then align `/new`, `/reset`, `/status`, `session_status(model=default)`, docs, doctor, and any clear/inherit command with that contract.

## Reproduction Assessment

Yes, at source level. Seed a session with a user or legacy `modelOverride`, change the configured default, then `/new` or `sessions.reset` preserves that selection while `resolveSessionModelRef` prefers it over the new default.

## Solution Assessment

Unclear. Unconditionally clearing explicit pins conflicts with current docs, release behavior, and tests, so the maintainable path is a maintainer-approved source contract plus explicit clear, migrate, reconcile, or corrected status/doctor UX.
