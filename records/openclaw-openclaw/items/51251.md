## Summary

Keep open: current main and v2026.6.10 intentionally preserve explicit and legacy session model selections across `/new`, `/reset`, and gateway restart. Current code/docs now provide visibility and `/model default` clearing for the current session, but bulk reset or default-precedence changes remain compatibility-sensitive product work.

## What This Changes

Request to stop session-level model overrides from silently masking updated config defaults across gateway restarts and new sessions, with clearer status and reset/clear paths.

## Best Possible Solution

Keep explicit user pins source-aware and visible, then add any bulk reset or default-precedence change only through a maintainer-approved resolver, doctor, CLI, docs, and upgrade-test design.

## Reproduction Assessment

Yes, source-reproducible: current main intentionally preserves explicit and legacy model overrides through `/new`, `/reset`, and gateway reset, while status exposes the pin. I did not run a live gateway restart in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. Current main handles visibility and per-session clearing, while automatic reset or bulk clearing needs source-aware compatibility policy rather than a hidden precedence change.
