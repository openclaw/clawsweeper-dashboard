## Summary

Keep open; current main has partial source-aware cleanup and status visibility, but it still intentionally preserves explicit and legacy session model selections, and the remaining reset/bulk-clear semantics need a maintainer model-selection contract decision.

## What This Changes

The issue asks OpenClaw to prevent persisted session model overrides from silently out-prioritizing changed config defaults across gateway restarts and `/new`, with clearer status and reset/clear UX.

## Best Possible Solution

Define the public source-aware model-selection contract first, then add guarded status/source labeling and explicit reset or reconcile commands without silently erasing intentional user pins.

## Reproduction Assessment

Yes, at source level. Seed a session with a user or legacy `modelOverride`, change the configured default, then `/new` or `sessions.reset` still preserves that override while only auto-sourced fallback pins are cleared.

## Solution Assessment

Unclear. Unconditionally clearing explicit pins conflicts with current docs and tests, so the maintainable path is a maintainer-approved source contract plus explicit clear/reconcile UX.
