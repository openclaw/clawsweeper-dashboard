## Summary

Keep open: current main and v2026.6.10 intentionally preserve explicit and legacy session model pins across reset/restart, while status/docs now make the pin visible and `/model default` clears only the current session. The remaining bulk reset or precedence change is compatibility-sensitive product work, not a safe cleanup close.

## What This Changes

This issue asks OpenClaw to stop hidden persisted session model overrides from silently overriding changed config defaults, especially across gateway restarts and `/new` or `/reset` sessions.

## Best Possible Solution

Use a maintainer-approved source-aware resolver/reset design: keep exact user pins visible, preserve shipped compatibility, and add an explicit scoped reset or reconcile path only after tests, docs, and doctor semantics are aligned.

## Reproduction Assessment

Yes, source-reproducible: current main intentionally preserves explicit and legacy model overrides through reset/restart paths, and status exposes the pin. I did not run a live gateway restart in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. Current main handles visibility and per-session clearing; automatic reset, bulk reset, or default-precedence changes need source-aware compatibility policy rather than a hidden precedence flip.
