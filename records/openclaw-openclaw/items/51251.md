## Summary

Keep open: current main and the latest release improved visibility and auto-fallback cleanup, but explicit and legacy session model pins still intentionally survive reset/restart and outrank config defaults, so the remaining behavior needs a maintainer policy decision.

## What This Changes

The issue asks OpenClaw to stop or clearly surface stale session-level model overrides after default model changes and to provide a reset/inherit path.

## Best Possible Solution

Define one source-aware model-selection contract for explicit, legacy, auto, and runtime session model state, then align reset, status, doctor, models set, and any clear/inherit command with it.

## Reproduction Assessment

Yes, at source level. Seed a session with a user or legacy modelOverride, change the configured default, then /new or sessions.reset still preserves that selection while resolveSessionModelRef prefers it over the new default.

## Solution Assessment

Unclear. Unconditionally clearing explicit pins conflicts with current docs, release behavior, and tests, so the maintainable path is a maintainer-approved source contract plus explicit clear, migrate, reconcile, or status/doctor UX.
