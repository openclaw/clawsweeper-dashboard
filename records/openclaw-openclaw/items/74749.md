## Summary

Keep open. Current main still hardcodes `--skip-trust` for the bundled Google Gemini CLI backend and the runner forwards that configured argv to `gemini`; reported Gemini CLI 0.38.x and current Homebrew 0.39.0 do not define that option. PR #74775 is an open closing implementation candidate, so this issue should remain open until that PR or a maintainer replacement lands.

## What This Changes

Remove or adjust the Google Gemini CLI backend's hardcoded `--skip-trust` defaults so OpenClaw works with Gemini CLI versions that reject the flag.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that fixes both fresh and resume Gemini CLI invocations for unsupported-flag releases, updates the backend default tests and changelog, and preserves trust semantics without substituting broader auto-approval flags such as `--yolo`.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main forwards `--skip-trust` to `gemini`, and Gemini CLI v0.38.2/v0.39.0 do not define that option, matching the reporter's yargs failure path.

## Solution Assessment

Unclear as a final code path. Removing the flag is the narrowest compatibility fix for 0.38.x/0.39.0, but because upstream v0.40.1 adds `--skip-trust`, maintainers should confirm whether unconditional removal or version/help-gated use is the better long-term behavior.
