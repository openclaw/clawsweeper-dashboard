## Summary

Keep open. Current main still passes `--skip-trust` to the bundled `google-gemini-cli` backend, upstream Gemini CLI 0.38.2 does not expose that option under its strict parser, and PR #74775 is an open unmerged closing implementation candidate.

## What This Changes

Remove the unsupported `--skip-trust` flag from the Google Gemini CLI backend's fresh and resumed command arguments.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the flag from both arg lists, updates stale tests and changelog, and leaves trust or approval semantics otherwise unchanged.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust` in the Gemini CLI backend defaults and forwards configured backend args to the spawned process; upstream 0.38.2's strict option set does not define that flag.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; substituting broader approval flags would change runtime security behavior.
