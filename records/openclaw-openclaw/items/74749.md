## Summary

Keep open. Current main still passes `--skip-trust` through the `google-gemini-cli` backend, the runner forwards that argv to `gemini`, Homebrew stable `gemini-cli` 0.38.2 lacks that option, and open PR #74775 is an unmerged implementation candidate rather than a reason to close this issue early.

## What This Changes

Remove the unsupported `--skip-trust` argument from the Google Gemini CLI backend defaults so `google-gemini-cli` works with `gemini-cli` 0.38.x stable installs.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes `--skip-trust` from both fresh and resume Gemini CLI defaults, updates the backend-default test and changelog, and does not substitute broader behavior such as `--yolo` unless that security posture is explicitly chosen.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures and forwards `--skip-trust`, and Homebrew-stable `gemini-cli` 0.38.2 does not define that yargs option.

## Solution Assessment

Yes. The narrow maintainable fix is to remove the incompatible flag from `extensions/google/cli-backend.ts` and align tests/changelog; version-gating is possible but heavier, and replacing it with broader approval behavior would solve a different problem.
