## What This Changes

The issue reports that the Google Gemini CLI backend hardcodes `--skip-trust`, which newer Gemini CLI versions reject before model execution.

## Best Possible Solution

Land a narrow fix that removes `--skip-trust` from the Google plugin's CLI backend `args` and `resumeArgs`, updates the adjacent CLI backend test expectations, and proves a current Gemini CLI run after the change.

## Reproduction Assessment

Yes for source-level reproduction: current `main` and `v2026.6.6` pass `--skip-trust`, while Gemini CLI v0.38.2 uses strict yargs parsing without that option. I did not run a full live provider turn during this read-only review.

## Solution Assessment

Yes, the requested fix direction is the narrow owner-boundary fix: remove the unsupported flag from the Google plugin defaults and update the tests. The existing linked PR is directionally right but still needs test alignment and real behavior proof before it should resolve the issue.
