## Summary

Keep open. Current main still passes `--skip-trust` to `gemini` for fresh and resumed `google-gemini-cli` turns, upstream `@google/gemini-cli` 0.38.2 rejects unknown strict options, and PR #74775 is an open unmerged closing implementation candidate.

## What This Changes

Remove the unsupported `--skip-trust` flag from the bundled `google-gemini-cli` backend defaults.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the flag from both arg lists, updates the stale expectations and changelog, and leaves trust or approval semantics otherwise unchanged.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust` in the Gemini CLI backend defaults and forwards it to the spawned `gemini` process, while upstream 0.38.2's strict option set does not define that flag.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; substituting broader approval flags would change runtime security behavior.
