## Summary

Keep open. Current main still configures and forwards the unsupported Gemini CLI `--skip-trust` flag, and PR #74775 is an open unmerged closing implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue reports that the Google Gemini CLI backend passes an unsupported `--skip-trust` argument, causing gemini-cli 0.38.x invocations to fail.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the flag from both fresh and resume arg lists, updates the stale test expectation and changelog, and leaves approval/trust behavior otherwise unchanged.

## Reproduction Assessment

Yes. Current main builds `--skip-trust` into the Gemini CLI backend, the CLI runner forwards those args into the spawned `gemini` argv, and the checked `@google/gemini-cli@0.38.2` bundle is strict without that option.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; version-gating or swapping to a broader auto-approval flag would add avoidable behavior and security risk.
