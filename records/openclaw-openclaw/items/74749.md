## Summary

Keep this open: current main and `v2026.5.7` still pass `--skip-trust` to the bundled Google Gemini CLI backend, and the issue is paired with the still-open closing PR at https://github.com/openclaw/openclaw/pull/74775.

## What This Changes

Remove the unsupported `--skip-trust` argument from the Google Gemini CLI backend defaults so gemini-cli 0.38.x calls can run.

## Best Possible Solution

Land the focused fix PR after it removes the flag from both fresh and resume defaults, updates the backend-default test, and includes redacted real Gemini CLI-backed proof; then close this issue as fixed.

## Reproduction Assessment

Yes. Current main builds and forwards `--skip-trust`, and the inspected `@google/gemini-cli@0.38.2` bundle uses strict yargs parsing without defining that option; I did not run a live Gemini CLI-backed turn.

## Solution Assessment

Yes. Removing `--skip-trust` from both fresh and resume argument lists is the narrow maintainable fix; switching to `--yolo` would change approval semantics and should stay out of this bug fix.
