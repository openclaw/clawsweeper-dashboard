## Summary

Keep open. Current main still passes `--skip-trust` to `google-gemini-cli`, the runner forwards that argv into `gemini`, Homebrew stable Gemini CLI 0.38.2 still lacks the option, and open PR #74775 is the active unmerged implementation path rather than a reason to close early.

## What This Changes

Remove the unsupported `--skip-trust` flag from the `google-gemini-cli` backend default fresh and resume arguments so Gemini CLI 0.38.x invocations do not fail before model execution.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes the incompatible flag from both fresh and resume Gemini CLI defaults, updates the backend-default test and changelog, and avoids substituting broader approval behavior such as `--yolo` unless maintainers explicitly choose that security posture.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures and forwards `--skip-trust`, and Gemini CLI 0.38.2's yargs config does not define that option.

## Solution Assessment

Yes. Removing the incompatible flag is the narrow maintainable fix; version-gating is possible, but replacing it with a broader approval flag would solve a different and more security-sensitive problem.
