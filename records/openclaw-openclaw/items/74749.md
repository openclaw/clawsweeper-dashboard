## Summary

Keep open. Current main and latest release `v2026.5.2` still hardcode `--skip-trust` for the bundled `google-gemini-cli` backend, the runner forwards backend args to `gemini`, upstream Gemini CLI `v0.38.2` does not define that option under strict parsing, and the linked closing PR is still open/unmerged in the provided GitHub context.

## What This Changes

The issue asks OpenClaw to remove the unsupported `--skip-trust` argument from the Google Gemini CLI backend fresh and resume argument lists.

## Best Possible Solution

Review and land PR #74775, or an equivalent narrow patch, removing the flag from both Gemini CLI arg lists while updating the stale test expectation and changelog without broadening trust or approval behavior.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust`, the CLI runner forwards it to `gemini`, and upstream Gemini CLI `v0.38.2` uses strict yargs parsing without defining that option; I did not run a live turn.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; replacing it with `--yolo` or another approval-mode flag would change trust semantics and should be a separate product decision.
