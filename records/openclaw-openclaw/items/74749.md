## Summary

Keep this open: current main and the latest release still pass `--skip-trust` to the bundled Google Gemini CLI backend, and the issue is already paired with an open closing PR that has not merged.

## What This Changes

The issue requests removing the unsupported `--skip-trust` argument from the `google-gemini-cli` backend so Gemini CLI calls work on gemini-cli 0.38.x and newer.

## Best Possible Solution

Repair and land the linked PR so production defaults and tests both drop `--skip-trust`, with redacted real behavior proof, then close this issue after merge.

## Reproduction Assessment

Yes. Current main constructs Gemini CLI argv with `--skip-trust`, the runner forwards that argv, and upstream Gemini CLI 0.38.2 uses strict option parsing without that flag; I did not run a live Gemini CLI turn.

## Solution Assessment

Yes. Removing `--skip-trust` from both fresh and resume args is the narrow maintainable fix; switching to `--yolo` would change approval semantics and should not be bundled into this bug fix.
