## Summary

Keep this open: current main and the latest release still pass the unsupported Gemini CLI `--skip-trust` argument, and this issue is already paired with the still-open closing PR at https://github.com/openclaw/openclaw/pull/74775.

## What This Changes

The issue reports that the bundled Google Gemini CLI backend hardcodes an unsupported `--skip-trust` flag, causing gemini-cli 0.38.x calls to fail.

## Best Possible Solution

Land the focused fix at https://github.com/openclaw/openclaw/pull/74775 after it removes the flag from both default arg lists, updates the backend-default test, and includes redacted real Gemini CLI-backed proof.

## Reproduction Assessment

Yes. Current main builds and forwards `--skip-trust`, and upstream Gemini CLI v0.38.2 defines strict options without that flag; I did not run a live Gemini CLI-backed turn.

## Solution Assessment

Yes. Removing `--skip-trust` from both fresh and resume argument lists is the narrow maintainable fix; switching to `--yolo` would change approval semantics and should stay out of this bug fix.
