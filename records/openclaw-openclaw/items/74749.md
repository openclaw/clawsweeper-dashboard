## Summary

Keep open. Current main and the latest tagged release still hardcode `--skip-trust` for the bundled `google-gemini-cli` backend, the runner forwards that argv into `gemini`, upstream Gemini CLI 0.38.x does not define the flag under strict yargs parsing, and the linked closing PR is still open and unmerged.

## What This Changes

The issue reports that the bundled Google Gemini CLI backend passes an unsupported `--skip-trust` flag, causing `gemini-cli` 0.38.x invocations to fail.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the unsupported flag from both Gemini CLI arg lists, updates the stale test expectation, and leaves trust/approval behavior otherwise unchanged.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust`, forwards it into the spawned `gemini` argv, and upstream Gemini CLI 0.38.1/0.38.2 does not define that option under strict yargs parsing; no live Gemini turn was run in this read-only review.

## Solution Assessment

Yes. Removing the unsupported flag from `args` and `resumeArgs` is the narrowest maintainable fix; this issue should stay open until the linked closing PR merges or an equivalent patch lands.
