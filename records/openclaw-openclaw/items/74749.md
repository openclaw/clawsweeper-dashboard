## What This Changes

The issue requests removing hardcoded `--skip-trust` from the Google Gemini CLI backend's fresh and resume arg defaults.

## Best Possible Solution

Land or replace the focused fixing PR with the production arg removal, matching default-contract test update, and redacted real Gemini CLI-backed proof; then close this issue as implemented.

Yes, from source inspection with high confidence: current main forwards `--skip-trust` to `gemini`, while upstream Gemini CLI v0.38.2 strict option parsing does not define that flag. I did not run a live Gemini CLI-backed OpenClaw turn.

## Solution Assessment

Yes, the narrow maintainable fix is to remove `--skip-trust` from both Gemini CLI arg lists and update the matching default-contract test. Switching to `--yolo` would broaden approval semantics and should not be bundled into this bug fix.
