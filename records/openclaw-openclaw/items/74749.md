## Summary

Keep open. Current main and the latest release still hardcode `--skip-trust` for the bundled `google-gemini-cli` backend, upstream `@google/gemini-cli@0.38.2` has strict argument parsing without that option, and the linked closing PR is still open and unmerged.

## What This Changes

The issue requests removing unsupported `--skip-trust` from the bundled `google-gemini-cli` backend `args` and `resumeArgs` so Gemini CLI 0.38.x calls do not fail before model execution.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74775 or an equivalent narrow patch that removes the flag from both arg lists, updates the backend-default test, and includes redacted after-fix behavior proof.

## Reproduction Assessment

Yes. This is source-reproducible: current main emits the bad Gemini CLI arg lists, the runner forwards them, and upstream `@google/gemini-cli@0.38.2` uses strict yargs parsing without `--skip-trust`.

## Solution Assessment

Yes. Removing `--skip-trust` from fresh and resume args, with matching test updates, is the narrow maintainable fix; replacing it with `--yolo` or `--approval-mode=yolo` would change approval behavior beyond the reported compatibility bug.
