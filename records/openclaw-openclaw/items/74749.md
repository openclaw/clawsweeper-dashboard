## Summary

Keep open. Current main and the latest release still pass the unsupported `--skip-trust` flag to the bundled `google-gemini-cli` backend, upstream `@google/gemini-cli@0.38.2` rejects that argv, and the linked closing PR is still open and unmerged.

## What This Changes

The issue reports that the bundled Google Gemini CLI backend passes unsupported `--skip-trust` to `gemini-cli` >= 0.38.x, causing every `google-gemini-cli` invocation to fail.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74775 or an equivalent narrow patch that removes the flag from both Gemini CLI paths, updates the backend-default test, and includes redacted after-fix behavior proof.

## Reproduction Assessment

Yes. Current main configures and forwards the exact `--skip-trust --output-format json --prompt ...` argv, and `@google/gemini-cli@0.38.2` rejects that flag before a model call.

## Solution Assessment

Yes. Removing `--skip-trust` from both fresh and resume arg lists is the narrow maintainable fix; replacing it with `--yolo` or `--approval-mode=yolo` would broaden approval behavior beyond the reported compatibility bug.
