## Summary

Keep open. Current main still passes `--skip-trust` to Gemini CLI for fresh and resumed `google-gemini-cli` turns, upstream `@google/gemini-cli` 0.38.2 uses a strict option parser without that flag, and PR #74775 is an open unmerged closing implementation candidate.

## What This Changes

The issue asks OpenClaw to stop passing the unsupported `--skip-trust` flag to the bundled Google Gemini CLI backend.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the flag from both arg lists, updates stale expectations and the changelog, and leaves trust/approval semantics otherwise unchanged.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust` in the Gemini CLI backend defaults and forwards it to the spawned `gemini` process, while upstream 0.38.2 defines strict options without that flag.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; substituting a broader auto-approval flag would change runtime security behavior.
