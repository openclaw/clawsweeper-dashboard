## Summary

Keep open. Current main still configures the unsupported Gemini CLI `--skip-trust` flag and forwards it to the spawned `gemini` process, while PR #74775 remains open as the closing implementation candidate.

## What This Changes

The issue requests removing `--skip-trust` from the bundled `google-gemini-cli` backend fresh and resume argument lists.

## Best Possible Solution

Land PR #74775 or an equivalent maintainer patch that removes the flag from both backend arg lists, updates stale tests and changelog, and avoids broader trust or auto-approval changes.

## Reproduction Assessment

Yes. Current main builds and forwards `--skip-trust` into `gemini`, and the upstream 0.38.2 strict option surface does not define that flag.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; version-gating or replacing it with broader approval flags would add unnecessary behavior and security risk.
