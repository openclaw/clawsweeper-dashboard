## Summary

Keep open. Current main still configures `google-gemini-cli` with `--skip-trust` in both fresh and resume defaults, the CLI runner forwards those configured args to `gemini`, and PR #74775 is an open unmerged closing implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue asks OpenClaw to remove the unsupported `--skip-trust` argument from the bundled Google Gemini CLI backend because gemini-cli 0.38.x rejects it and fails every provider call.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes `--skip-trust` from both fresh and resume Gemini CLI defaults, updates the backend defaults tests and required changelog entry, leaves the already-correct docs aligned, and lets this issue close through the merged fix.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures `--skip-trust`, the runner forwards it to the child process, and upstream Gemini CLI 0.38.2 has no matching yargs option while the report provides the concrete failure text.

## Solution Assessment

Yes. Removing the unsupported flag is the narrow maintainable fix; version-gating is unnecessary here, and substituting broader auto-approval flags would solve a different and riskier problem.
