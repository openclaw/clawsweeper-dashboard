## Summary

Keep open. Current main and the latest tagged release still pass `--skip-trust` through the bundled `google-gemini-cli` backend, upstream Gemini CLI 0.38.x does not define that strict yargs option, and the closing PR is still open and unmerged.

## What This Changes

Remove the unsupported `--skip-trust` flag from the bundled `google-gemini-cli` backend so Gemini CLI 0.38.x calls do not fail during argument parsing.

## Best Possible Solution

Review and land PR #74775 or an equivalent narrow patch that removes the unsupported flag from both Gemini CLI arg lists and updates tests/changelog without broadening approval behavior.

## Reproduction Assessment

Yes. Current main builds both fresh and resume Gemini CLI args with `--skip-trust`, the runner forwards those args into `argv`, and the Gemini CLI 0.38.x bundle has strict yargs options without that flag; I did not run a live Gemini turn.

## Solution Assessment

Yes. Removing only `--skip-trust` from both arg arrays is the narrowest maintainable fix; replacing it with `--yolo` or `--approval-mode=yolo` would change approval semantics.
