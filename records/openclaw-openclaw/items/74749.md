## Summary

Keep open. Current main still passes `--skip-trust` to the bundled `google-gemini-cli` backend, upstream Gemini CLI 0.38.2 rejects unknown strict yargs options, and PR #74775 is an open unmerged closing implementation candidate.

## What This Changes

The issue asks OpenClaw to stop passing the unsupported `--skip-trust` flag to the Gemini CLI backend for fresh and resumed runs.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the flag from both arg lists, updates the stale expectations and changelog, and leaves trust/approval semantics otherwise unchanged.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust` in the Gemini CLI backend defaults and forwards configured backend args to the spawned process; upstream 0.38.2's strict option set does not define that flag.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable implementation direction, while the issue should remain open until the linked closing PR merges.
