## What This Changes

The issue requests removing the unsupported `--skip-trust` flag from the Google Gemini CLI backend `args` and `resumeArgs`.

## Best Possible Solution

Land or replace the focused fixing PR with the production arg removal, matching test updates, and redacted real Gemini CLI-backed after-fix proof; then close this issue as implemented.

## Reproduction Assessment

Yes, from source inspection with high confidence: current main forwards `--skip-trust` to `gemini`, while upstream gemini-cli v0.38.2 strict option parsing does not define that flag. I did not run a live Gemini CLI-backed OpenClaw turn in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to remove `--skip-trust` from both Gemini CLI arg lists and update the matching default-contract test. Switching to `--yolo` would broaden approval semantics and should not be bundled into this bug fix.
