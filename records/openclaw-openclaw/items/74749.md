## Summary

Keep open. Current main still configures the unsupported Gemini CLI `--skip-trust` flag, the CLI runner forwards configured backend args into the spawned `gemini` argv, and upstream `@google/gemini-cli` 0.38.2 exposes related flags but not `--skip-trust`. PR #74775 is already open with closing syntax, so this issue should stay open until that PR or a maintainer replacement lands.

## What This Changes

The issue requests removing the unsupported `--skip-trust` flag from the Google Gemini CLI backend's fresh and resume default arguments.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes `--skip-trust` from both Gemini CLI default arg lists, updates `src/agents/cli-backends.test.ts` and `CHANGELOG.md`, and leaves the already-correct public docs aligned with the source.

## Reproduction Assessment

Yes. Current main configures the reported flag and forwards it to `gemini`; the upstream 0.38.2 CLI option contract has no matching `skip-trust` option, so the source-level reproduction is high confidence.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; version-gating is heavier, and substituting broader approval flags would solve a different security/approval problem.
