## Summary

Keep open. Current main and latest v2026.4.29 still pass `--skip-trust` for `google-gemini-cli`, upstream `@google/gemini-cli` 0.38.x has a strict option parser with no such option, and the open closing PR #74775 has not merged.

## What This Changes

The issue asks OpenClaw to stop passing the unsupported `--skip-trust` flag to the bundled Google Gemini CLI backend for fresh and resumed turns.

## Best Possible Solution

Review and land #74775 or an equivalent narrow patch that removes the flag from source and stale expectations while preserving existing trust and approval semantics.

## Reproduction Assessment

Yes. Current main constructs and forwards `--skip-trust` for fresh and resumed Gemini CLI runs, and upstream `@google/gemini-cli` 0.38.2 has a strict parser without that option.

## Solution Assessment

Yes. Removing the unsupported flag from both arg lists is the narrowest maintainable fix; replacing it with `--yolo` or `--approval-mode=yolo` would broaden runtime approval behavior.
