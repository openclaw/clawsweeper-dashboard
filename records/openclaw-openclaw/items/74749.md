## Summary

Keep open. Current main and the latest release still pass `--skip-trust` to the bundled `google-gemini-cli` backend, upstream `@google/gemini-cli@0.38.2` rejects that argv under strict yargs parsing, and the linked closing implementation at https://github.com/openclaw/openclaw/pull/74775 is still open and unmerged.

## What This Changes

The issue asks OpenClaw to remove the unsupported `--skip-trust` argument from the bundled Gemini CLI backend fresh and resume arg lists.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74775 or an equivalent narrow patch that removes the flag from both Gemini CLI paths, updates the backend-default test, and includes redacted after-fix behavior proof.

## Reproduction Assessment

Yes. Current main builds and forwards the exact `--skip-trust --output-format json --prompt ...` argv, and `@google/gemini-cli@0.38.2` rejects that argv with `Unknown arguments: skip-trust, skipTrust`.

## Solution Assessment

Yes. Removing `--skip-trust` from both fresh and resume arg lists is the narrow maintainable fix; replacing it with `--yolo` or `--approval-mode=yolo` would broaden approval behavior beyond the reported compatibility bug.
