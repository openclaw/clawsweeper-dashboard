## Summary

Keep open: current main still configures the unsupported Gemini CLI `--skip-trust` argument, the CLI runner forwards configured backend args into the spawned `gemini` argv, and the backend tests still encode the stale default. PR #74775 is an open closing implementation candidate, so this issue should remain open until that PR or a maintainer replacement lands.

## What This Changes

The issue requests removing the unsupported `--skip-trust` argument from the bundled `google-gemini-cli` backend defaults so Gemini CLI 0.38.x invocations succeed.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes `--skip-trust` from both Gemini CLI default arg lists, updates the backend default tests and changelog as needed, and preserves the current docs direction that trust is handled by Gemini CLI folder trust rather than an unsupported flag.

## Reproduction Assessment

Yes. Current main still builds and spawns Gemini CLI argv containing `--skip-trust`, while the upstream 0.38.2 CLI option contract does not include that option, so the source-level reproduction is high confidence.

## Solution Assessment

Yes. Removing the unsupported flag from both fresh and resume defaults is the narrowest maintainable fix; substituting a broader auto-approval flag would solve a different and riskier behavior.
