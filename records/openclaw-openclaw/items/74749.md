## Summary

Keep open. Current main still hardcodes `--skip-trust` for the bundled `google-gemini-cli` backend, the runner forwards that argument to `gemini`, upstream Gemini CLI 0.38.x does not define the flag under strict yargs parsing, and the linked closing PR remains open and unmerged.

## What This Changes

The issue asks OpenClaw to stop passing the unsupported `--skip-trust` flag to `google-gemini-cli` so Gemini CLI 0.38.x invocations succeed.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the flag from both Gemini CLI arg lists, updates the stale test expectations and changelog, and leaves trust/approval semantics otherwise unchanged.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust` for the Gemini CLI backend and forwards it to the spawned `gemini` process, while the upstream 0.38.x strict option set does not define that flag; I did not run a live Gemini turn in this read-only review.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable fix; this issue should stay open until the linked closing PR merges or an equivalent patch lands.
