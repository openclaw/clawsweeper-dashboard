## Summary

Keep open. Current main still unconditionally passes `--skip-trust` through the bundled `google-gemini-cli` backend while the reported Gemini CLI 0.38.2 parser lacks that option, and #74775 remains an open unmerged closing PR for the same bug.

## What This Changes

The issue requests removing the unsupported `--skip-trust` flag from the bundled Google Gemini CLI backend so Gemini CLI 0.38.x calls do not fail before model execution.

## Best Possible Solution

Review and land #74775 or an equivalent compatibility patch that removes or gates `--skip-trust`, updates the stale backend tests and changelog, and avoids broader auto-approval flags.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust` in the Google Gemini CLI backend defaults and forwards it to the spawned process, while the tagged Gemini CLI 0.38.2 parser does not define that option.

## Solution Assessment

Yes for the reported 0.38.x failure: removing the unconditional flag is the narrowest fix. If maintainers want the newer Gemini CLI trust override, the safer alternative is version or capability gating rather than replacing it with `--yolo` or `--approval-mode=yolo`.
