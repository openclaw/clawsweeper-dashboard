## Summary

Keep this open: current main and the latest release still pass the unsupported Gemini CLI `--skip-trust` argument, and the issue is paired with the still-open closing PR at https://github.com/openclaw/openclaw/pull/74775.

## What This Changes

Remove the unsupported `--skip-trust` argument from the Google Gemini CLI backend `args` and `resumeArgs` defaults.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74775 after it updates the stale backend-default test and includes redacted real Gemini CLI-backed proof.

## Reproduction Assessment

Yes. Current main hardcodes and forwards `--skip-trust`, and upstream Gemini CLI v0.38.2 uses strict option parsing without that flag; I did not run a live Gemini CLI-backed turn.

## Solution Assessment

Yes. Removing `--skip-trust` from both fresh and resume defaults is the narrow maintainable fix; replacing it with `--yolo` would change approval semantics and should stay out of this bug fix.
