## Summary

Keep open. Current main and latest v2026.5.2 still pass `--skip-trust` in the bundled `google-gemini-cli` backend, upstream Gemini CLI v0.38.2 rejects unknown strict options, and PR #74775 is an open unmerged closing implementation candidate.

## What This Changes

Remove the unsupported `--skip-trust` flag from the `google-gemini-cli` backend fresh and resume argument lists so Gemini CLI 0.38.x calls do not fail.

## Best Possible Solution

Land PR #74775 or an equivalent narrow patch that removes the flag from both default arg lists, updates the stale expectations and changelog, and keeps trust/approval behavior otherwise unchanged.

## Reproduction Assessment

Yes. Current main constructs `--skip-trust` for `google-gemini-cli` and forwards it to `gemini`; upstream v0.38.2 uses strict yargs parsing without that option, so an agent turn on that CLI version reaches an unknown-argument failure.

## Solution Assessment

Yes. Removing the unsupported flag is the narrowest maintainable direction; changing to `--yolo` or a yolo approval mode would alter safety semantics beyond the reported compatibility bug.
