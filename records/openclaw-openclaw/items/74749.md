## Summary

Keep open. Current main still configures the unsupported Gemini CLI `--skip-trust` flag, the shared CLI runner forwards configured backend args into the spawned `gemini` command, and upstream `@google/gemini-cli` 0.38.2 has no matching option while yargs rejects unknown arguments. PR #74775 is an open closing implementation candidate, so this issue should remain open until that PR or a maintainer replacement lands.

## What This Changes

The issue reports that the bundled Google Gemini CLI backend passes an unsupported `--skip-trust` argument to `gemini-cli`, causing `google-gemini-cli` calls to fail on gemini-cli 0.38.x.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes `--skip-trust` from both Gemini CLI default arg lists, updates `src/agents/cli-backends.test.ts`, adds the required user-facing changelog entry, and leaves the already-correct public CLI backend docs aligned with source.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures the flag, the runner forwards it to `gemini`, and upstream gemini-cli 0.38.2 rejects unknown yargs options while exposing no `--skip-trust` option.

## Solution Assessment

Yes. Removing the unsupported flag from both fresh and resume arg lists is the narrowest maintainable fix; version-gating is heavier, and swapping to broader approval flags would solve a different trust/permission problem.
