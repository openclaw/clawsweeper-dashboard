## Summary

Keep open. Current main still configures `--skip-trust` for the bundled `google-gemini-cli` backend and forwards that backend argv directly to the spawned `gemini` process; upstream/Homebrew evidence still supports the reported 0.38.2 failure path. Open PR #74775 is an unmerged closing-syntax implementation candidate, so this issue should stay open until that PR or a maintainer replacement lands.

## What This Changes

Remove the unsupported `--skip-trust` flag from the `google-gemini-cli` fresh and resume backend defaults so gemini-cli 0.38.x invocations do not fail before model execution.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes `--skip-trust` from both `args` and `resumeArgs`, updates the backend-default test and changelog entry, and leaves trust/approval semantics otherwise unchanged.

## Reproduction Assessment

Yes. The high-confidence source-level reproduction is current main configuring and forwarding `--skip-trust` to `gemini`, while upstream gemini-cli v0.38.2 and current Homebrew stable 0.38.2 do not define that option.

## Solution Assessment

Yes. The narrow maintainable fix is to remove the incompatible flag and align tests/changelog; version-gating is heavier, and replacing it with a broader approval flag would solve a different security problem.
