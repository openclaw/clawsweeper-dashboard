## Summary

Keep open: current main has useful partial fixes for empty or narrow tool allowlists and cached plugin descriptors, but a normal first embedded run with tools enabled still enters the broad synchronous `createOpenClawCodingTools()` path before model submission. The open related PR at https://github.com/openclaw/openclaw/pull/73039 is still unmerged and does not prove the cold first-run event-loop stall is fixed.

## What This Changes

The issue reports that the first cold embedded agent run synchronously initializes the full tool set, blocking the event loop for about 21 seconds before model submission.

## Best Possible Solution

Measure current cold default embedded-run stage timings, then amortize or defer heavyweight tool/plugin initialization while preserving existing tool-policy, runtime config, and plugin laziness contracts.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only sweep. The source path is clear enough to keep open: a normal tools-enabled embedded attempt still calls the broad synchronous tool builder before model submission.

## Solution Assessment

Unclear. The issue lists plausible remedies, but the narrowest maintainable fix should be chosen from fresh cold-run stage timings and coordinated with the open memoization and tool-latency work.
