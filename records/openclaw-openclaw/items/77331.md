## Summary

Keep open: current main still reaches the full tool-construction path synchronously before model submission, and the merged related fixes only reduce selected factory costs rather than proving the cold first-run stall is gone.

## What This Changes

The issue asks OpenClaw to avoid the roughly 21-second cold synchronous `createOpenClawCodingTools()` setup before the first embedded agent run.

## Best Possible Solution

Measure current cold tool-prep stages, then move heavy imports and plugin/tool factory work off the first live embedded turn without changing tool-policy semantics.

## Reproduction Assessment

Yes for a source-level path: restart a cold gateway and trigger an embedded run that reaches `runEmbeddedAttempt()`, which synchronously calls `createOpenClawCodingTools()` before model submission. The live 21-second timing still needs a fresh cold trace on current main.

## Solution Assessment

Unclear. Prewarming, descriptor/factory memoization, finer lazy loading, or event-loop yielding could each help; the maintainable fix should be chosen after measuring the cold import and factory stages and reconciling #73039 and #76606.
