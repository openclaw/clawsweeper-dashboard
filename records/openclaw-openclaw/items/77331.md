## Summary

Keep open: current main has useful mitigations for explicit no-tools, narrow allowlists, denied optional media factories, and cached plugin descriptors, but the default tools-enabled embedded run still enters broad synchronous tool construction before model submission. The related memoization PR is still open and conflicting, and the latest source does not prove the reported cold first-run event-loop stall is fixed.

## What This Changes

The issue reports that the first default embedded agent run after gateway startup blocks around 21 seconds in synchronous `createOpenClawCodingTools()` initialization before the model call starts.

## Best Possible Solution

Measure current cold default embedded-run stage timings, then defer, amortize, or safely prewarm heavyweight tool and plugin initialization while preserving tool policy, runtime config, and plugin laziness contracts.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. The reporter supplied concrete timing and liveness logs, and current source still shows the default tools-enabled embedded attempt entering the broad synchronous tool-builder path before model submission.

## Solution Assessment

Unclear. Prewarming, memoization, lazy segmentation, and yielding are all plausible, but the narrowest maintainable fix needs fresh timing data and maintainer choice around startup readiness and plugin runtime laziness.
