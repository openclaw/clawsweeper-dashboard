## Summary

Keep open: current main still builds the embedded agent tool set synchronously on the first live run path, and the related merged work only partially reduces specific factory costs rather than proving the cold-start event-loop stall is fixed.

## What This Changes

The issue asks OpenClaw to remove the cold first-run `createOpenClawCodingTools()` event-loop stall that delays embedded agent prompts and can break latency-sensitive channels.

## Best Possible Solution

Measure cold tool-prep stages on current main, then move heavy imports and plugin/tool factory work off the first live embedded turn without changing tool policy semantics.

## Reproduction Assessment

Yes, there is a clear source-level reproduction path: restart a cold gateway and trigger an embedded run that reaches `runEmbeddedAttempt()`. This review did not execute the live timing trace, but the synchronous code path matches the reported stall window.

## Solution Assessment

Unclear. Prewarming, deeper descriptor memoization, lazy segment loading, or yielding could all help; the best fix should be chosen after measuring the cold import and factory stages and coordinating with #73039 and #76606.
