## Summary

Keep open: current main still constructs the embedded agent tool set synchronously on the first live run path, and the merged related fixes only reduce specific repeated/factory costs rather than proving the cold-start event-loop stall is gone.

## What This Changes

The issue asks OpenClaw to remove the first embedded-agent cold-start stall caused by synchronous `createOpenClawCodingTools()` work before the model call.

## Best Possible Solution

Add a measured cold-start mitigation that keeps plugin/tool policy behavior unchanged while moving expensive imports and factory work off the first latency-sensitive embedded turn.

## Reproduction Assessment

Yes for source-level reproduction: a cold gateway run that reaches `runEmbeddedAttempt()` still enters synchronous tool construction before the model call. I did not live-reproduce the 21s timing in this read-only sweep.

## Solution Assessment

Unclear. Prewarming, deeper descriptor memoization, lazy segment loading, or yielding could each address part of the issue; the narrowest maintainable fix should be chosen after current-main cold-stage timing.
