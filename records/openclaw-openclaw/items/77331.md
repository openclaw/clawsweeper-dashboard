## Summary

Keep open: current main still builds the embedded agent tool set synchronously on the first live run path, and the known related fixes only reduce narrower tool-prep costs rather than proving the cold-start event-loop stall is gone.

## What This Changes

The issue asks to remove the cold first-run synchronous tool-construction stall in embedded agent runs so latency-sensitive channel connections do not time out before the LLM call starts.

## Best Possible Solution

Add a measured cold-start mitigation in the embedded tool-prep path that moves heavyweight tool/plugin/module initialization off the first live turn while preserving current tool policy and plugin runtime semantics.

## Reproduction Assessment

Yes for source-level reproduction: restart a cold gateway and trigger an embedded run that reaches `runEmbeddedAttempt()`, which still calls `createOpenClawCodingTools()` synchronously. The exact 21s timing was not live-reproduced in this read-only sweep.

## Solution Assessment

Unclear. The report lists plausible fixes, but the narrowest maintainable path should be chosen after measuring cold import/factory stages and coordinating with #73039 and #76606.
