## Summary

Current `main` still routes the first embedded attempt through synchronous tool construction, and the recent tool-latency fixes only reduce selected media/plugin work rather than proving the cold-start event-loop stall is gone.

## What This Changes

The issue requests removing the cold synchronous `createOpenClawCodingTools()` startup cost that blocks first embedded agent runs and latency-sensitive channel WebSockets.

## Best Possible Solution

Keep this open and fix the first-turn cold path by prewarming or deferring heavy tool/module initialization away from the first user interaction, coordinated with #73039 and #76606.

## Reproduction Assessment

No high-confidence live reproduction was established in this review; the source path is clear, but the exact cold-start duration and WebSocket timeout impact remain reporter-supplied until a cold gateway benchmark is rerun.

## Solution Assessment

No, current `main` is not the complete solution because it still performs synchronous tool construction on the first embedded run; the maintainable fix should move or break up the cold work, not only filter selected factories afterward.
