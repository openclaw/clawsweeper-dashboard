## Summary

Keep open: current main still constructs the embedded agent tool set synchronously before the first model call, and the merged/current mitigations only reduce selected repeated or denied-tool costs rather than proving the cold first-run event-loop stall is gone.

## What This Changes

The issue reports that the first embedded agent run after gateway startup blocks the event loop for about 21 seconds while `createOpenClawCodingTools()` cold-loads tool modules.

## Best Possible Solution

Measure current cold tool-prep stages, then move heavy module/plugin factory work off the first live embedded turn while preserving existing tool policy and plugin runtime contracts.

## Reproduction Assessment

Yes at source level: restart a cold gateway and trigger an embedded run that reaches `runEmbeddedAttempt()`, which still synchronously calls `createOpenClawCodingTools()`. This review did not execute the live timing trace, so the exact 21s current-main wall-clock value remains unmeasured.

## Solution Assessment

Unclear. Prewarming, deeper descriptor memoization, lazy segment loading, or event-loop yielding could each help, but the maintainable fix should be chosen after measuring cold stage costs and coordinating with #73039 and #76606.
