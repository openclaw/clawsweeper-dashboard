## Summary

Keep open. Current main still uses a flat, budgeted available-skills catalog and does not implement lazy matching, names-only injection, or resolver-backed skill loading; the related closed duplicates point to this issue as the canonical skill-specific tracker, while the active hook-based PR still needs revision.

## What This Changes

The issue requests reducing per-turn skill prompt overhead by lazy-loading or otherwise minimizing the skill metadata injected into the system prompt.

## Best Possible Solution

Keep this as the canonical skill-specific overhead thread, settle the resolver, hook, or explicit prompt-mode contract, then land a focused implementation with tests and docs.

## Reproduction Assessment

Yes. Source and tests show buildWorkspaceSkillsPrompt still renders name, description, and location under budget, then falls back to name plus location only after the full block exceeds maxSkillsPromptChars; there is no lazy or names-only branch on current main.

## Solution Assessment

Unclear as a code solution. The optimization is valid, but names-only injection alone is not the safest fix while the prompt contract depends on exact SKILL.md locations; a resolver-backed flow, hook-based narrowing, or explicit documented prompt mode needs maintainer direction.
