## Summary

Keep open. Current main still uses a flat, budget-limited available-skills catalog rather than lazy skill matching or names-only/resolver-backed injection, and the active related PRs propose still-unmerged product/API directions rather than a shipped fix.

## What This Changes

Request to reduce per-turn skills prompt overhead by lazily or compactly injecting skill metadata instead of listing every installed skill with paths every turn.

## Best Possible Solution

Keep this as the skill-specific tracker, then land one documented resolver, hook-narrowing, or prompt-mode contract with tests and docs once maintainers choose the API and rollout semantics.

## Reproduction Assessment

Yes as a source-level gap check, not as a bug reproduction. Current main still renders the full skills catalog under budget, falls back to compact only after budget pressure, and has no lazy or names-only skills prompt mode.

## Solution Assessment

Unclear as an implementation path. The optimization is valid, but the safe solution needs maintainer choice between a resolver-backed contract, prompt-hook narrowing, or an explicit prompt mode before code should land.
