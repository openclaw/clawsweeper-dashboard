## What This Changes

This PR changes task cancellation/runtime aliasing, Discord and Telegram liveness handling, outbound/message-tool sanitization and schema scoping, failover classification, security trust-model config/audit behavior, `.gitignore`, tests, and coding-agent guidance.

## Best Possible Solution

Keep this PR as source context while a narrow stale-subagent cancellation fix lands with regression coverage, and review the runtime, liveness, message-tool, Telegram, security/config, and docs changes separately.

## Reproduction Assessment

Yes, by source inspection. On current main, a running subagent task whose `killSubagentRunAdmin` returns `{ found: false, killed: false }` exits before marking the task cancelled.

## Solution Assessment

No. The guard inversion plus a focused regression test is the narrow maintainable fix, while this PR bundles unrelated runtime, channel, message-tool, config, docs, and security-audit changes.

## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the real subagent runtime initializer:** `scripts/runtime-postbuild.mjs:18-23`
  - body: The synthetic `subagent-registry.runtime.js` wrapper replaces the real `ensureContextEnginesInitialized()` export with a local boolean no-op. Packaged subagent paths need the real initializer so context engines are registered before `resolveContextEngine()` is used.
  - confidence: 0.86
- **[P1] Keep trusted proxy warnings visible:** `src/security/audit.ts:411-416`
  - body: Demoting `gateway.trusted_proxies_missing` to `info` for `security.trustModel="personal-assistant"` can hide an actionable reverse-proxy spoofing warning. A single-operator deployment still needs a warning when Control UI is exposed through a proxy without `gateway.trustedProxies`.
  - confidence: 0.78
- **[P2] Keep cross-channel actions in scoped schemas:** `src/agents/tools/message-tool.ts:456-459`
  - body: Returning only the current channel's action enum rejects explicit message calls to other configured channels from scoped isolated or cron runs. Current main intentionally includes configured-channel actions so those calls validate.
  - confidence: 0.82
- **[P2] Update generated config docs if trustModel stays:** `src/config/zod-schema.ts:455-462`
  - body: Adding top-level `security.trustModel` changes the public config surface, but the generated schema/help/reference artifacts are not updated in this PR. The config docs check is expected to drift unless this config change is split out or generated artifacts are refreshed.
  - confidence: 0.9
