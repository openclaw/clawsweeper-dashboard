## Summary

Keep open, but this PR should not merge as-is: current main still has the stale subagent cancellation bug, while the branch mixes that narrow fix with unrelated runtime, channel, config, and security changes that introduce blocking regressions.

## What This Changes

The branch changes task cancellation runtime aliasing and stale subagent cancellation, Discord and Telegram liveness handling, outbound/message-tool sanitization and schema scoping, failover classification, security trust-model config/audit behavior, `.gitignore`, tests, and coding-agent guidance.

## Best Possible Solution

Land or review a split stale-task cancellation fix with a regression test, and keep runtime packaging, Discord liveness, message-tool schema, Telegram heartbeat, and security trust-model changes in separate focused reviews.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to create a running subagent task whose `killSubagentRunAdmin` returns `{ found: false, killed: false }`; current main returns `cancelled: false` before marking the task terminal.

## Solution Assessment

No. The guard inversion plus focused test is the narrow maintainable fix, but this PR also changes unrelated runtime packaging, channel liveness, message-tool schema, docs/config, and security audit behavior.

  - body: The synthetic `subagent-registry.runtime.js` wrapper replaces the real `ensureContextEnginesInitialized()` export with a local boolean no-op. Packaged subagent paths need the real initializer so context engines are registered before `resolveContextEngine()` is used.
  - confidence: 0.86

  - body: Demoting `gateway.trusted_proxies_missing` to `info` for `security.trustModel="personal-assistant"` can hide an actionable reverse-proxy spoofing warning. A single-operator deployment still needs a warning when Control UI is exposed through a proxy without `gateway.trustedProxies`.
  - confidence: 0.78
- **[P2] Keep cross-channel actions in the scoped schema:** `src/agents/tools/message-tool.ts:456-459`
  - body: Returning only the current channel's action enum rejects explicit message calls to other configured channels from scoped isolated or cron runs. Current main intentionally includes configured-channel actions so those calls validate.
  - confidence: 0.82

  - body: Adding top-level `security.trustModel` changes the public config surface, but the generated schema/help/reference artifacts are not updated in this PR. The config docs check is expected to drift unless this config change is split out or the generated artifacts are refreshed.
