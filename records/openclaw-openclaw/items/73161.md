## What This Changes

The PR changes task cancellation/runtime alias handling, Discord and Telegram liveness paths, outbound text sanitization, message-tool schema scoping, failover classification, security trust-model config/audit behavior, tests, `.gitignore`, and coding-agent skill guidance.

## Best Possible Solution

Split and land the stale subagent cancellation guard with focused regression coverage, then review the runtime packaging, Discord liveness, Telegram heartbeat/sanitization, message schema, failover, and trust-model changes as separate PRs.

Yes for the stale subagent cancellation path by source inspection: current main returns `cancelled: false` before marking the task terminal when `killSubagentRunAdmin` reports `{ found: false, killed: false }`. No high-confidence current-main repro was established for the other bundled behavior changes.

## Solution Assessment

No. The guard inversion plus a regression test is the narrow maintainable fix for the proven task bug, while the unrelated runtime, channel, schema, failover, and security-policy changes need separate review and proof.

Full review comments:

- **[P1] Preserve the real subagent runtime initializer:** `scripts/runtime-postbuild.mjs:18-23`
  - body: The synthetic `subagent-registry.runtime.js` wrapper can overwrite the stable runtime entry and replaces the real `ensureContextEnginesInitialized()` export with a local no-op.   - body: Demoting `gateway.trusted_proxies_missing` to `info` for `security.trustModel="personal-assistant"` hides an actionable reverse-proxy spoofing warning. A single-operator deployment still needs a warning when Control UI is exposed through a proxy without `gateway.trustedProxies`.
  - confidence: 0.82
- **[P2] Keep cross-channel actions in the scoped schema:** `src/agents/tools/message-tool.ts:456-459`
  - body: Returning only the current channel's action enum rejects explicit message calls to other configured channels from scoped isolated or cron runs. Current main intentionally includes configured-channel actions so those calls validate.
  - confidence: 0.84

  - body: Adding top-level `security.trustModel` changes the public config contract, but this branch does not update the generated help/docs/reference surfaces that expose config keys. Split the config change out or refresh those generated surfaces with it.
  - confidence: 0.86
