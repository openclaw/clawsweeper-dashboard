## What This Changes

The PR changes task cancellation and runtime aliasing, Discord and Telegram liveness handling, outbound/message-tool sanitization, failover classification, security trust-model config/audit behavior, `.gitignore`, and coding-agent guidance.

## Best Possible Solution

Land a narrow replacement that only changes the subagent cancellation guard and adds the missing regression test, while leaving runtime packaging, Discord transport liveness, security audit severity, and cross-channel message schemas unchanged unless reviewed separately.
