## Summary

Keep open: the stale subagent cancellation bug is real, but this PR is broad, currently dirty/conflicting, and still has blocking runtime, security, and message-tool correctness issues. The author has already acknowledged the review and said the useful task-registry guard change should be split into a narrow branch.

## What This Changes

The PR changes task cancellation/runtime packaging, Discord and Telegram liveness, outbound/message-tool behavior, failover classification, security config/audit handling, tests, `.gitignore`, and coding-agent guidance.

## Best Possible Solution

Review or land a narrow stale-subagent cancellation guard with focused regression coverage, and handle runtime packaging, Discord liveness, Telegram heartbeat, message-tool schema, failover, and security trust-model changes in separate owner-reviewed PRs.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a running subagent-backed task whose `killSubagentRunAdmin` returns `{ found: false, killed: false }`; current main returns `cancelled: false` before marking the task terminal.

## Solution Assessment

No. The stale-task guard inversion plus regression test is the narrow maintainable fix, but this PR bundles unrelated runtime packaging, channel liveness, outbound/schema, failover, config, and security policy changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the real subagent runtime initializer:** `scripts/runtime-postbuild.mjs:18-23`
  - body: The synthetic `subagent-registry.runtime.js` wrapper replaces the real `ensureContextEnginesInitialized()` export with a local boolean no-op. Packaged subagent paths need the real initializer so context engines are registered before `resolveContextEngine()` is used.
  - confidence: 0.86
- **[P1] Keep trusted proxy warnings visible:** `src/security/audit.ts:411-416`
  - body: Demoting `gateway.trusted_proxies_missing` to `info` for `security.trustModel="personal-assistant"` can hide an actionable reverse-proxy spoofing warning. A single-operator deployment still needs a warning when Control UI is exposed through a proxy without `gateway.trustedProxies`.
  - confidence: 0.78
- **[P2] Keep cross-channel actions in the scoped schema:** `src/agents/tools/message-tool.ts:456-459`
  - body: Returning only the current channel's action enum rejects explicit message calls to other configured channels from scoped isolated or cron runs. Current main intentionally includes configured-channel actions so those calls validate.
  - confidence: 0.82
- **[P2] Update generated config docs if trustModel remains:** `src/config/zod-schema.ts:455-462`
  - body: Adding top-level `security.trustModel` changes the public config surface, but the generated schema/help/reference artifacts are not updated here. Split the config change out or refresh the generated artifacts with it.
  - confidence: 0.9
