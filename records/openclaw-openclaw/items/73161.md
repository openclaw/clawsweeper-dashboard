## What This Changes

The PR changes task cancellation/runtime aliasing, Discord and Telegram liveness, outbound/message-tool sanitization and schema scoping, failover classification, security trust-model config/audit behavior, tests, `.gitignore`, and coding-agent guidance.

## Best Possible Solution

## Solution Assessment

No. The narrow guard inversion plus a regression test is the maintainable fix for the stale-task bug, while this PR bundles unrelated runtime, channel, outbound, config, and security audit behavior.

## Review Findings

- **[P1] Preserve the real subagent runtime initializer:** `scripts/runtime-postbuild.mjs:18-23`
  - body: The synthetic `subagent-registry.runtime.js` wrapper exports a local no-op `ensureContextEnginesInitialized()` instead of the real context-engine initializer. Packaged subagent paths need the real initializer before `resolveContextEngine()` is used.
  - confidence: 0.86

  - body: Demoting `gateway.trusted_proxies_missing` to `info` for `security.trustModel="personal-assistant"` can hide an actionable reverse-proxy spoofing warning. Single-operator deployments still need the warning when Control UI is exposed through a proxy without `gateway.trustedProxies`.
  - confidence: 0.78
- **[P2] Keep cross-channel actions in the scoped schema:** `src/agents/tools/message-tool.ts:456-459`
  - body: Returning only the current channel's action enum rejects explicit message calls to other configured channels from scoped isolated or cron runs. Current main intentionally includes configured-channel actions that fit the shared schema.
  - confidence: 0.82

  - body: Adding top-level `security.trustModel` changes the public config surface, but the generated schema/help/reference artifacts are not updated here. The config docs check is expected to drift unless this config change is split out or the generated artifacts are refreshed.
  - confidence: 0.9
