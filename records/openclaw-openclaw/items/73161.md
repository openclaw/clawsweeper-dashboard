## What This Changes

The PR changes task runtime alias generation and stale subagent cancellation, Discord and Telegram liveness handling, outbound sanitization, message-tool action scoping, failover classification, security trust-model config, ignore rules, and coding-agent guidance.

## Best Possible Solution

Keep current main's explicit runtime entries and Discord transport-activity liveness model, then replace this PR with a focused stale subagent cancellation fix that changes only the task-registry guard and regression test. Any trust-model config, Telegram heartbeat, message sanitization, or message-tool schema work should move to separately reviewed PRs with generated config/docs artifacts and security ownership where applicable.

Yes. A focused unit path can create a running subagent task, mock killSubagentRunAdmin to return { found: false, killed: false }, and observe current main returning cancelled=false instead of marking the task cancelled.

Full review comments:

- **[P1] Preserve the real subagent runtime initializer:** `scripts/runtime-postbuild.mjs:18-23`
  - body: The synthetic subagent-registry.runtime.js wrapper replaces the real ensureContextEnginesInitialized export with a local boolean.   - confidence: 0.86
- **[P1] Keep trusted proxy misconfiguration visible:** `src/security/audit.ts:411-416`
  - body: Demoting gateway.trusted_proxies_missing to info for security.trustModel="personal-assistant" hides an actionable reverse-proxy spoofing warning. A single-operator deployment still needs this warning when Control UI is exposed through a proxy without gateway.trustedProxies.
  - confidence: 0.8
- **[P2] Keep cross-channel actions in the scoped schema:** `src/agents/tools/message-tool.ts:456-459`
  - body: Returning only the current channel's action enum prevents a scoped Telegram or Discord run from validating a message call that explicitly targets another configured channel with that channel's actions. Current main intentionally unions safe cross-channel actions for isolated and cron agents.
  - confidence: 0.82
- **[P2] Update generated config documentation artifacts:** `src/config/zod-schema.ts:455-462`
  - body: Adding top-level security.trustModel changes the public config surface, but the PR does not update the generated schema/help/reference artifacts. The config docs check is expected to report drift until those artifacts are regenerated or the config change is split out.
  - confidence: 0.9
