## Summary

Keep open: current main still reproduces the noisy picker behavior, and this PR is useful, but it should not merge with broad substring-based internal-session classification and no changelog entry.

## What This Changes

The PR adds UI-side filtering so inactive subagent, ACP, node-backed, and spawnDepth-marked sessions are hidden from the chat picker while the selected internal session remains visible.

## Best Possible Solution

Add an explicit gateway-produced internal visibility/classification for runtime sessions and have only the normal chat picker hide inactive internal rows, while admin/debug session views can still inspect them.

## Reproduction Assessment

Yes. A focused unit case that passes a main row plus inactive subagent, ACP, and node-backed rows to resolveSessionOptionGroups on current main will include those rows because only global/unknown and cron rows are filtered.

## Solution Assessment

No. The PR targets the right UI boundary as a mitigation, but the internal-key predicate should use parsed rest-prefix semantics or explicit visibility metadata instead of whole-key substring checks.
