## Summary

Keep open. Current main still surfaces internal runtime sessions in the chat picker, and this PR addresses a real UI clutter problem, but the proposed helper can hide legitimate non-internal sessions because it matches `:subagent:` and `:acp:` anywhere in the key instead of using parsed session-key metadata or a gateway visibility contract.

## What This Changes

This PR adds UI-side filtering for inactive subagent, ACP, and node-backed session rows in the chat session picker, keeps the selected internal row visible, adds `spawnDepth` to the UI session row type, and adds focused picker tests.

## Best Possible Solution

Add an explicit gateway-produced internal classification for subagent, ACP, and node-backed runtime sessions, propagate it through the typed session row contract and UI event merge path, and have only the normal chat picker hide inactive internal rows while the Sessions/admin/debug surface can still inspect them.

## Reproduction Assessment

Yes. A source-level reproduction is to pass `resolveSessionOptionGroups` a selected `agent:main:main` row plus an `agent:main:subagent:child` row on current main; the code only skips inactive global/unknown and cron rows, so the subagent row is added to the picker.

## Solution Assessment

No. The PR is useful as a bounded mitigation, but matching `:subagent:` or `:acp:` anywhere in a flexible session key is not the best fix; a gateway visibility field or the existing parsed session-key helpers are safer and easier to maintain.
