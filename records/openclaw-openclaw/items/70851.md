## Summary

Keep open. Current main still surfaces internal runtime rows in the chat picker, and this PR targets the right user-visible problem, but the branch should not merge as-is because its whole-string internal-key matching can hide legitimate non-internal sessions.

## What This Changes

The PR adds UI-side filtering for inactive subagent, ACP, node-backed, and spawnDepth-marked sessions in the chat picker, keeps the selected internal session visible, adds a UI row type field, and adds focused picker tests.

## Best Possible Solution

Add an explicit gateway-produced internal visibility/classification for runtime sessions, propagate it through the typed session row contract, and have only the normal chat picker hide inactive internal rows while admin/debug session surfaces can still inspect them.

## Reproduction Assessment

Yes. On current main, passing resolveSessionOptionGroups an active main row plus inactive subagent/ACP/node-backed rows will include those internal rows because only global/unknown and cron rows are skipped.

## Solution Assessment

No. The PR is a useful bounded mitigation, but whole-string matching for ':subagent:' or ':acp:' is too broad; explicit visibility metadata or parsed rest-prefix helpers are the narrower maintainable path.
