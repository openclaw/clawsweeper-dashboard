## Summary

Keep open: the PR targets a real Control UI picker problem that is not fully solved on current main, but the current diff is not merge-ready because it uses heuristic internal-key matching, misses documented bare ACP session keys, and needs reconciliation with the subagent nesting behavior that has since landed.

## What This Changes

The PR adds UI-side filtering for inactive internal sessions in `resolveSessionOptionGroups`, adds `spawnDepth` to the UI row type, and adds picker tests for hidden internal rows plus the active-session exception.

## Best Possible Solution

Add explicit gateway-produced internal visibility/classification for runtime sessions, propagate it through the typed session row contract, and have the normal chat picker hide or group inactive internals according to maintainer policy while admin/debug surfaces can still request them.

## Reproduction Assessment

Yes, from source inspection. Current main still has no internal visibility/default-hide check in `resolveSessionOptionGroups`; it filters active-agent rows, global/unknown rows, and cron rows, then shows or nests the remaining runtime rows.

## Solution Assessment

No. The picker surface is a reasonable interim place to mitigate the issue, but the PR should use explicit visibility metadata or parsed/shared key helpers and handle documented bare ACP keys instead of whole-string heuristics.
