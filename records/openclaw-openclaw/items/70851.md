## Summary

Keep open: the PR addresses a real source-reproducible Control UI picker pollution problem, but the submitted diff still has correctness blockers, lacks required real behavior proof, and now conflicts with newer session-picker work on current main.

## What This Changes

The branch adds UI-side internal-session filtering for the chat picker, adds `spawnDepth` to the UI session row type, and adds focused option-group tests for hidden and active internal sessions.

## Best Possible Solution

Add or reuse explicit parsed internal session classification from the session/gateway layer, have the normal picker hide inactive internals while preserving active-session fallback and admin/debug visibility, and reconcile the subagent portion with https://github.com/openclaw/openclaw/pull/81858.

## Reproduction Assessment

Yes, from source inspection: current main still feeds subagent, ACP, and runtime rows through `resolveSessionOptionGroups` without an explicit internal visibility filter. I did not run a live UI scenario because this review is read-only.

## Solution Assessment

No, not as submitted. The narrow UI mitigation should use shared parsed session-key helpers or explicit visibility metadata instead of whole-key substring matching, and it needs reconciliation with the newer subagent picker work.

  - confidence: 0.88

  - confidence: 0.84
