## Summary

Keep open. Current main still lists internal runtime rows in the chat picker, while this PR is useful but should not merge as written because broad substring matching can hide legitimate non-internal sessions and the required changelog entry is missing.

## What This Changes

The PR adds UI-side filtering that hides inactive subagent, ACP, node-backed, and spawnDepth-marked sessions from the chat picker while keeping the selected internal session visible and adding focused picker tests.

## Best Possible Solution

Add an explicit gateway-produced internal visibility/classification field for runtime sessions, propagate it through the typed session row contract, and have the normal chat picker hide inactive internal rows while admin/debug session surfaces can still show them.

## Reproduction Assessment

Yes. On current main, passing resolveSessionOptionGroups an active main row plus inactive subagent/ACP/node-backed rows will include those internal rows because only global/unknown and cron rows are skipped.

## Solution Assessment

No, not as written. The direction is useful as a bounded mitigation, but internal detection should use explicit visibility metadata or parsed/shared internal-key helpers, and the user-facing fix needs a changelog entry before merge.
