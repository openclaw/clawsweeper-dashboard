## Summary

Keep open: current main still exposes internal runtime rows in the chat picker, and this PR is useful but needs changes before merge because the internal-key matcher is heuristic and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR adds Control UI filtering for inactive subagent, ACP, node-backed, and spawnDepth-marked sessions in the chat picker, keeps the selected internal session visible, extends the UI row type, and adds focused picker tests.

## Best Possible Solution

Add gateway-produced internal visibility or equivalent classification to session rows, have the normal chat picker hide inactive internal rows by default, keep the selected internal session visible, and preserve admin/debug session surfaces for inspection.

## Reproduction Assessment

Yes. On current main, a sessionsResult containing a normal main row plus inactive subagent, ACP, or node-backed rows will pass those internal rows through resolveSessionOptionGroups because only global/unknown and hidden cron rows are skipped.

## Solution Assessment

No. The direction is right, but whole-string ':subagent:' and ':acp:' matching is not the narrowest maintainable solution; use explicit visibility metadata or parsed session-key rest-prefix semantics instead.
