## Summary

Keep open: this PR targets a real Control UI picker problem, but it is not ready to merge because current main has overlapping subagent picker behavior, the branch still has concrete internal-key heuristic bugs, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds UI-side filtering in `resolveSessionOptionGroups` to hide inactive subagent, ACP, and node-backed sessions while preserving the selected internal session, adds `spawnDepth` to UI session rows, and adds focused picker tests.

## Best Possible Solution

Add or reuse explicit parsed session classification from the gateway/session layer, propagate it through the typed session row contract, and have the normal chat picker hide inactive internals while preserving admin/debug visibility and the active-session fallback.

## Reproduction Assessment

Yes, from source inspection. Current main still processes subagent and ACP/runtime rows through `resolveSessionOptionGroups` without an explicit internal visibility filter, so seeded `sessions.list` rows can appear unless excluded by the existing agent/kind/cron rules.

## Solution Assessment

No, not as submitted. A UI-side mitigation can be useful, but the current branch uses substring heuristics that miss bare ACP keys and can hide legitimate flexible keys; explicit visibility metadata or shared parsed key helpers are the safer direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Hide bare ACP session keys:** `ui/src/ui/chat/session-controls.ts:445-446`
  - body: `acp:<uuid>` is the documented default ACP session key shape, but this helper only marks non-agent keys internal when they start with `node-`. Those ACP rows would still appear in the picker even though the PR says ACP/internal sessions should be hidden; use `isAcpSessionKey` or explicit visibility metadata.
  - confidence: 0.88
- **[P2] Parse internal prefixes instead of matching substrings:** `ui/src/ui/chat/session-controls.ts:439-443`
  - body: The `includes(":subagent:")` and `includes(":acp:")` checks run against the whole key, so flexible explicit keys containing those path segments are hidden even when they are not internal runtime sessions. Base the check on parsed `agent:<id>` rest prefixes or explicit row metadata.
  - confidence: 0.84
