## Summary

Keep open: current main still has the reported picker pollution, and the PR is useful but should not merge as-is because the new internal-session matcher can hide non-internal flexible keys and the user-facing UI fix is missing the required changelog entry.

## What This Changes

The branch adds UI-side filtering to omit inactive subagent, ACP, node-backed, and spawnDepth-marked sessions from the chat picker while keeping the active internal session visible, plus focused picker tests and a UI row type field.

## Best Possible Solution

Add an explicit gateway-produced internal visibility/classification for runtime sessions, propagate it through the typed session row contract, and have only the normal chat picker hide inactive internal rows while admin/debug session surfaces can still inspect them.

## Reproduction Assessment

Yes. On current main, resolveSessionOptionGroups with a normal active row plus inactive subagent/ACP/node-backed rows will include those internal rows because the loop only filters global/unknown and hidden cron rows.

## Solution Assessment

No. The PR is a useful bounded mitigation, but whole-key substring matching is not the narrowest maintainable fix; parsed rest-prefix helpers or explicit visibility metadata would avoid hiding unrelated flexible session keys.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

Base internal detection on explicit visibility metadata or the parsed agent rest prefix instead.
