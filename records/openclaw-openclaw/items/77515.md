## Summary

Keep open: the PR targets a real active-memory allowlist failure, but the catch predicate is too broad and would also hide unsupported-model tool failures that should remain visible.

## What This Changes

The PR adds active-memory catch handling for empty memory-tool allowlist errors and a changelog fix entry.

## Best Possible Solution

Keep the graceful no-op for missing registered memory tools, but narrow it to the `no registered tools matched` reason and add regression coverage for both missing-tools and unsupported-model cases.

## Reproduction Assessment

Yes, by source inspection. Active-memory always passes the explicit memory tool allowlist, and the embedded-runner guard throws the reported empty-allowlist error when that allowlist resolves to no callable registered tools.

## Solution Assessment

No. Returning `NONE` is appropriate for the missing-memory-plugin case, but this implementation should not swallow the same guard prefix when the selected model cannot use tools.
