## Summary

Keep open: the PR’s direction is right, but the repaired head still misses the documented configuration variant where inherited allowlist sources are present in the guard error message.

## What This Changes

The PR changes active-memory to no-op the embedded recall sub-agent when the explicit memory tool allowlist resolves to no registered memory tools, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Match the missing registered memory-tools condition by semantic parts of the guard error, or a typed guard signal if available, while preserving unsupported-model and tools-disabled diagnostics.

## Reproduction Assessment

Yes. Source inspection shows active-memory can reach the embedded runner with the runtime memory allowlist and no registered memory tools, and the linked report provides a concrete multi-source allowlist error shape that the PR does not catch.

## Solution Assessment

No. Returning `NONE` for missing memory plugins is the narrow maintainable behavior, but exact full-message equality is too narrow because the guard message legitimately includes other explicit allowlist sources.
