## Summary

Keep open: the reported allowlist noise is valid, but this patch catches the guard by broad prefix, which also masks unsupported-model or disabled-tool allowlist failures that current main surfaces as unavailable.

## What This Changes

The PR adds a CHANGELOG entry and catches an active-memory sub-agent allowlist error so missing memory backend tools are treated as an empty memory result.

## Best Possible Solution

Handle only the no-registered-memory-tools case as empty recall, while preserving the existing unavailable warning/status for unsupported models or disabled tools.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: active-memory supplies an explicit memory runtime allowlist, and the guard throws when no callable tools remain. The PR finding is also source-reproducible because the guard uses the same prefix for other allowlist failures.

## Solution Assessment

No. The intended direction is narrow and maintainable, but the current predicate should distinguish `no registered tools matched` for the active-memory memory allowlist from unsupported-model or disabled-tool cases.

## Review Findings
