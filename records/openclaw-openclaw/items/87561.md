## What This Changes

The issue requests a shared core/channel contract requiring sanitized final fallback messages to be delivered or explicitly reported as failed, suppressed, partial, or unknown across durable text-capable channels.

## Best Possible Solution

Keep this as the canonical maintainer-owned delivery contract, review the narrow WhatsApp and receipt PRs as proving slices, then land an explicit SDK/core contract with cross-channel conformance and recovery diagnostics.
