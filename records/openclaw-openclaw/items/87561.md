## Summary

Keep open. This is a protected MEMBER-authored maintainer issue, and current main plus the latest release still contain a source-backed final-fallback loss path while related open PRs cover only narrower slices.

## What This Changes

The issue requests a core, cross-channel durable final fallback contract so sanitized final fallback text is delivered or reported as an explicit delivery failure instead of silently disappearing.

## Best Possible Solution

Keep this as the canonical maintainer-owned delivery contract, land or replace the open partial PRs as slices, then add SDK/core conformance and diagnostics coverage across durable text-capable channels.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: WhatsApp currently advertises durable final text, routes isError payloads through sendPayload, and returns an empty delivery identity without calling the sender.

## Solution Assessment

Yes. Keeping this as a broad core/channel contract is the best current path; channel-specific fixes can prove slices, but they do not by themselves settle SDK semantics, explicit outcomes, recovery diagnostics, and conformance coverage.
