Keep this open: the report is fresh, concrete, and current main still has matching direct OpenAI Responses transport, model-list timeout, fetch-timeout, idle-watchdog, and event-loop diagnostic paths without a clear shipped fix for the direct OpenAI timeout regression.

## What This Changes

The issue reports a regression where OpenAI-backed agent replies hang after updating to 2026.5.2, with slow `models.list`, high event-loop delay, idle watchdog timeouts, and `/v1/responses` fetch timeouts.

## Best Possible Solution

Reproduce the direct OpenAI API-key path on current main or the latest shipped release, capture Gateway health/stability and OpenAI transport diagnostics, then land a focused fix in the proven transport, timeout, or event-loop path.

## Reproduction Assessment

No. The report has concrete steps and current-main source still contains the matching transport, timeout, and diagnostic paths, but this review did not establish a live failing OpenAI call on current main.

## Solution Assessment

Unclear. There is no proposed patch, and forcing `api=openai-completions` is not a reliable solution because official OpenAI routes can normalize back to Responses; the maintainable fix needs root-cause proof first.
