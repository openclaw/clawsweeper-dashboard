## Summary

Keep open. Current main still leaves the reported maintenance cleanup partially unresolved: terminal `lost` cron rows cannot reach durable cron recovery, and stale CLI rows with subagent-style child-session keys can still be treated as live when only a historical session row remains. The open #75036 PR addresses the cron subset but has not merged and does not cover the CLI/subagent liveness path.

## What This Changes

Request `openclaw tasks maintenance --apply` to reconcile or prune `lost` tasks with missing backing sessions and `stale_running` rows, with optional timestamp warning cleanup.

## Best Possible Solution

Land or rework #75036 for terminal lost cron rows, then add a focused CLI/subagent liveness predicate fix with regression coverage; treat timestamp clamping as separate policy-backed cleanup.

## Reproduction Assessment

Yes. A source-level reproduction is a terminal `runtime: "cron"`, `status: "lost"` task with durable cron run evidence, plus a stale `runtime: "cli"` task whose `childSessionKey` is `agent:...:subagent:...`, whose live run context is gone, and whose session row still exists.

## Solution Assessment

No. Current main is only a partial solution; the narrow maintainable path is to handle terminal lost cron recovery and stale CLI/subagent liveness as tested predicate fixes, without bundling an unscoped timestamp rewrite.
