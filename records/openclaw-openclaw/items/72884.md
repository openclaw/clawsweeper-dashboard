## Summary

Keep open. Current `main` has fixed the narrow CLI run-identity false-liveness subset, but terminal `lost` cron rows still cannot reach durable cron recovery, and stale-running reconciliation remains covered by open follow-up work rather than merged code.

## What This Changes

The issue requests that `openclaw tasks maintenance --apply` clean `lost` backing-session-missing records and stale `running` task records instead of reporting no maintenance action.

## Best Possible Solution

Land or replace the scoped terminal-lost cron recovery and stale-running reconciliation fixes, then decide timestamp normalization separately instead of using this broad issue for another repair branch.

## Reproduction Assessment

Yes, by source inspection for the remaining terminal `lost` cron path: non-active `lost` rows cannot pass the durable cron recovery guard on current `main`. The stale-running family is also source-backed and separately tracked by open related work.

## Solution Assessment

Partly. `tasks maintenance` is the right surface, but the best path is to finish the existing narrow PRs for terminal cron recovery and stale-running reconciliation, with timestamp clamping kept out of that bug fix.
