## Summary

Keep open. Current main partially improves task cleanup, but source inspection still shows terminal `lost` cron rows cannot be recovered from durable cron evidence and subagent-owned CLI rows can still look live solely because an old session-store row exists.

## What This Changes

Request that `openclaw tasks maintenance --apply` clean up or recover `lost` backing-session-missing tasks and `stale_running` records, with optional timestamp correction.

## Best Possible Solution

Keep this as the canonical tracker until terminal lost cron recovery and subagent-owned CLI liveness are both handled with regression coverage; treat timestamp rewriting as a separate cleanup policy.

## Reproduction Assessment

Yes. A source-level reproduction is a terminal `runtime: "cron"`, `status: "lost"` task with durable cron run evidence, plus a stale `runtime: "cli"` task whose `childSessionKey` is `agent:...:subagent:...`, whose live run context is gone, and whose session-store row still exists.

## Solution Assessment

Partly. Task maintenance is the right surface, but the maintainable fix should split durable cron recovery, subagent-owned CLI liveness, and timestamp rewrite policy rather than generically pruning every audit finding.
