## Summary

Keep open. Current main still leaves the reported maintenance cleanup partially unresolved: stale CLI rows with subagent-style child-session keys can still be treated as live, and terminal lost cron rows are not recovered from durable cron history unless they are still active.

## What This Changes

Request that `openclaw tasks maintenance --apply` clean up or recover `lost` backing-session-missing records and stale running task rows, with a secondary timestamp-skew cleanup suggestion.

## Best Possible Solution

Keep this as the canonical tracker until terminal lost cron recovery and stale CLI/subagent child-session liveness are both handled, with timestamp rewriting treated as separate policy-backed cleanup if needed.

## Reproduction Assessment

Yes. A source-level reproduction is a terminal `runtime: "cron"` task with durable cron history, plus a stale `runtime: "cli"` task whose `childSessionKey` is `agent:...:subagent:...` and whose live run context is gone while the session row remains.

## Solution Assessment

No. Current main is only a partial solution; the narrow maintainable direction is to land or rework #75036 for terminal lost cron rows and add a separate regression-tested CLI/subagent liveness predicate fix.
