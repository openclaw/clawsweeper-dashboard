## Summary

Keep open. Current main still leaves the reported maintenance gap unresolved for terminal `lost` cron rows and for stale CLI rows backed only by old subagent session entries; open PR #75036 covers only the cron subset and has not merged.

## What This Changes

The issue asks for `openclaw tasks maintenance --apply` to clean or reconcile `lost` backing-session-missing task rows and stale running task records instead of reporting no work.

## Best Possible Solution

Keep this as the canonical tracker until terminal lost cron recovery and stale CLI/subagent liveness are both fixed with regression coverage; handle timestamp clamping as a separate policy decision.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: terminal `runtime: "cron"`, `status: "lost"` rows cannot reach durable recovery, and stale `runtime: "cli"` rows with `agent:...:subagent:...` child sessions can be treated as live by an old session row.

## Solution Assessment

Partly. `tasks maintenance` is the right surface, but the maintainable solution should split terminal cron recovery, CLI/subagent liveness, and timestamp-rewrite policy into narrow reviewed changes.
