## Summary

Keep open. Current main still leaves the reported maintenance cleanup partially unresolved: terminal `lost` cron rows are not recovered from durable cron history, and stale CLI tasks owned by subagent session keys can still be treated as live if the historical session row remains.

## What This Changes

The issue asks for `openclaw tasks maintenance --apply` to clean or recover `lost` backing-session-missing rows and `stale_running` task records, with optional benign timestamp correction.

## Best Possible Solution

Keep this as the canonical tracker until terminal lost cron recovery and stale CLI/subagent child-session liveness are both handled, with timestamp rewriting split into a policy-backed follow-up if maintainers want it.

## Reproduction Assessment

Yes. A source-level reproduction is a terminal `runtime: "cron"` task with durable cron run history plus a stale `runtime: "cli"` task whose `childSessionKey` is `agent:...:subagent:...` and whose live run context is gone while the session row remains.

## Solution Assessment

No. Current main is only a partial solution; the narrow maintainable direction is to land or rework #75036 for terminal lost cron rows and add a separate regression-tested CLI/subagent liveness predicate fix.
