## Summary

Keep open: current main and the latest shipped release still use output-driven CLI no-output watchdogs with no pending-Monitor or expected-idle signal. The related open PRs mitigate timeout caps and orphan output, but neither implements this issue's Monitor wait contract.

## What This Changes

The issue asks OpenClaw to stop killing CLI sessions that are intentionally silent while waiting for a long-running Monitor task.

## Best Possible Solution

Model verified pending Monitor/tool waits as bounded expected idle that pauses or heartbeats the no-output watchdog only while tool work is actually pending, while preserving overall timeouts and normal no-output protection.

## Reproduction Assessment

Yes, at source level. The issue gives concrete Monitor steps and matching watchdog logs, and current main still times silent CLI turns from stdout/stderr activity rather than pending Monitor state; I did not run the live 180-second wait.

## Solution Assessment

Unclear. Treating verified pending Monitor waits as expected idle is the right direction, but raising constants alone is only a mitigation and the maintainable liveness seam needs owner choice.
