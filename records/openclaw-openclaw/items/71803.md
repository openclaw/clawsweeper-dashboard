## Summary

Keep open. Current main still has a hard no-output watchdog for CLI turns, and the relevant supervisor and Claude live-session paths only treat process output as liveness; I did not find a pending Monitor/tool-wait signal that would make the reported idle period expected instead of a timeout.

## What This Changes

The issue requests that the CLI no-output watchdog stop terminating healthy agent sessions that are idle because they are waiting for a pending Monitor task.

## Best Possible Solution

Add an explicit expected-idle/proof-of-life contract for pending Monitor, MCP, or client-tool waits in the CLI runner and Claude live-session path, while keeping the overall request timeout and preserving the no-output watchdog for CLIs with no verified pending work.

## Reproduction Assessment

Yes. The report provides a concrete Monitor wait command and matching watchdog logs, and current main source/tests still show silent CLI processes becoming no-output watchdog failures without a pending Monitor exemption.

## Solution Assessment

Unclear for the suggested implementation details. Suspending the watchdog while a verified Monitor/tool wait is pending is directionally right, but the maintainable fix should expose a bounded pending-tool state instead of only raising the timeout cap or disabling no-output checks globally.
