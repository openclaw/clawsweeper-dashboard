## Summary

Keep open. Current main still treats silence from the CLI process as a watchdog failure, including the Claude live-session path, and there is no source-visible pending Monitor or expected-idle signal that would protect a healthy long-running tool wait.

## What This Changes

The issue asks for CLI no-output watchdog behavior to treat pending Monitor waits as expected idle instead of killing the Claude CLI session.

## Best Possible Solution

Add an explicit expected-idle or proof-of-life contract for pending Monitor/MCP/client-tool waits in the CLI runner and Claude live-session path, while preserving the overall request timeout and the no-output watchdog for truly stalled CLIs.

## Reproduction Assessment

Yes. The issue includes a concrete Monitor wait command and matching watchdog logs, and current main source/tests show silent CLI processes still hit the no-output watchdog without a pending Monitor exemption.

## Solution Assessment

No current fix is sufficient. Raising timeout constants, as in #41022, reduces false kills but does not distinguish verified pending tool work from a stalled CLI process.
