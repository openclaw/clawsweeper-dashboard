## Summary

Keep open. Current main still treats silent CLI processes as no-output watchdog failures, including the Claude live-session path, and I did not find a verified pending Monitor/tool-wait signal that would make this expected idle instead of a timeout.

## What This Changes

The issue asks OpenClaw to stop killing Claude CLI sessions that are intentionally idle while waiting for a long-running Monitor task to complete.

## Best Possible Solution

Add a bounded expected-idle/liveness contract for verified pending Monitor or tool waits in the Claude live-session and generic CLI runner paths, while preserving the overall request timeout and the no-output watchdog when no pending work is proven.

## Reproduction Assessment

Yes. The issue provides concrete Monitor steps and matching watchdog logs, and current main still has source and test coverage showing silent CLI processes become no-output watchdog failures without a pending Monitor exemption.

## Solution Assessment

Unclear. Suspending the watchdog only while a verified Monitor/tool wait is pending is directionally right, but the maintainable fix should surface explicit pending-tool state or bounded heartbeats rather than globally disabling or only raising the timeout.
