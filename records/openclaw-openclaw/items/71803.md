## Summary

Keep open: current main still has output-driven CLI no-output watchdogs with no pending-Monitor or expected-idle input. The related open PRs cover timeout constants and orphan-output idle-close deferral, but neither implements this issue's Monitor wait contract.

## What This Changes

The issue asks OpenClaw to stop terminating Claude CLI sessions that are intentionally silent while a Monitor task is pending.

## Best Possible Solution

Add a bounded expected-idle signal for verified pending Monitor/tool waits in the Claude live-session or supervisor path, while preserving overall request timeouts and normal no-output protection when no pending work is known.

## Reproduction Assessment

Yes. The issue gives concrete Monitor steps and matching watchdog logs, and current main still shows a source-level path where silent CLI turns are timed from stdout/stderr activity rather than pending Monitor state.

## Solution Assessment

Unclear. Treating verified pending Monitor/tool waits as expected idle is the right direction, but the maintainable seam needs owner choice; raising constants alone via https://github.com/openclaw/openclaw/pull/41022 would only mitigate some cases.
