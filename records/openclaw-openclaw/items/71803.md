## Summary

Keep open. Current main still enforces stdout/stderr-based no-output watchdogs in the generic CLI runner and Claude live-session path, with no verified pending Monitor/tool-wait state that would make this expected idle instead of a timeout; related #40982 and #41022 only raise timeout caps and do not solve the Monitor-specific liveness contract.

## What This Changes

The issue reports that the CLI no-output watchdog terminates healthy Claude CLI sessions while the agent is idle waiting for a long-running Monitor task.

## Best Possible Solution

Add a bounded expected-idle/liveness contract for verified pending Monitor or tool waits in the Claude live-session and generic CLI runner paths, while preserving the overall request timeout and keeping the no-output watchdog active when no pending work is proven.

## Reproduction Assessment

Yes. The issue includes concrete Monitor steps and matching watchdog logs, and current main still has source and tests around silent CLI processes becoming no-output watchdog failures without a pending Monitor exemption.

## Solution Assessment

Unclear. Exempting only verified pending Monitor/tool waits is the right direction, but the maintainable seam still needs maintainer choice between Claude stream-json state tracking, a generic supervisor hold, or a bounded heartbeat mechanism.
