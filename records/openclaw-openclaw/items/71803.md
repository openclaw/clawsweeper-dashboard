## Summary

Keep open: current main still treats CLI liveness as stdout/stderr activity, with no source-backed exemption for a turn that is deliberately idle while a Monitor/MCP/tool wait is pending. The related open timeout-cap PR may reduce symptoms, but it does not implement the expected-idle contract requested here.

## What This Changes

The issue asks OpenClaw to stop killing CLI-backed sessions that are silently waiting on a long-running Monitor task.

## Best Possible Solution

Add a bounded expected-idle mechanism for verified pending Monitor/tool waits in the Claude live-session and/or generic CLI supervisor path, while preserving overall request timeouts and normal no-output protection when no pending work is proven.

## Reproduction Assessment

Yes. The issue provides concrete Monitor steps and matching watchdog logs, and current main still has source-level evidence that silent CLI turns are timed from stdout/stderr output rather than pending Monitor state.

## Solution Assessment

Unclear. Treating verified pending Monitor/tool waits as expected idle is the maintainable direction, but only raising timeout constants or disabling the watchdog globally is not the best targeted fix.
