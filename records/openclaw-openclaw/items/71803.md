## Summary

Current main still has the reported failure shape: CLI liveness is driven by stdout/stderr, the timeout resolver has no pending Monitor/tool-wait input, and the Claude live-session path does not turn stream-json tool state into expected idle. The related timeout-cap issue and PR may reduce false kills, but they do not close the Monitor-specific liveness contract.

## What This Changes

The issue asks OpenClaw to keep CLI sessions alive when they are deliberately idle while a long-running Monitor task is pending.

## Best Possible Solution

Add a bounded expected-idle contract for verified pending Monitor or MCP tool waits, while preserving the overall request timeout and keeping the no-output watchdog active when no pending work is proven.

## Reproduction Assessment

Yes. The issue includes concrete Monitor steps and matching watchdog logs, and current main still has output-only watchdog paths that would kill a silent CLI turn without a pending Monitor exemption.

## Solution Assessment

Unclear. The requested behavior is valid, but the maintainable implementation needs a maintainer call on where to represent pending tool liveness across Claude live sessions and the generic supervisor.
