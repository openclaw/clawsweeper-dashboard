## Summary

Keep open. Current main still treats CLI silence as a watchdog failure and has no visible pending Monitor or expected-idle signal that would protect a healthy long-running tool wait; the related timeout-cap work only mitigates duration, not the state distinction this issue reports.

## What This Changes

The issue requests that CLI sessions waiting on a long-running Monitor task avoid being killed by the no-output watchdog while the task is healthy and intentionally idle.

## Best Possible Solution

Add an explicit expected-idle or proof-of-life contract for pending Monitor, MCP, or native client-tool waits in the CLI runner and Claude live-session path, while preserving the overall request timeout and no-output watchdog for genuinely stalled CLIs.

## Reproduction Assessment

Yes. The issue includes a concrete Monitor wait command and matching watchdog logs, and current main source still shows silent CLI processes hitting the no-output watchdog without a pending Monitor exemption.

## Solution Assessment

No current solution is sufficient. Raising watchdog constants as in #41022 is a useful mitigation, but the maintainable fix is to represent verified pending tool work or emit bounded heartbeats tied to that state.
