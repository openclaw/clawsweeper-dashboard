## Summary

Keep open: current main still uses output-driven CLI watchdogs, and the related open timeout PR only raises constants rather than preserving expected-idle Monitor waits.

## What This Changes

The issue requests that CLI no-output watchdogs stop killing a provider process while an agent is deliberately idle waiting for a pending Monitor task.

## Best Possible Solution

Add a bounded expected-idle mechanism for verified pending Monitor/tool waits while preserving the overall request timeout and normal no-output protection when no pending work is proven.

## Reproduction Assessment

Yes. The issue gives concrete Monitor steps and matching watchdog logs, and current main still has source-level evidence that silent CLI turns are timed by stdout/stderr output rather than pending Monitor state.

## Solution Assessment

Unclear. Treating verified pending Monitor/tool waits as expected idle is likely the right direction, but raising timeout constants alone is incomplete and the owner should choose the durable liveness seam.
