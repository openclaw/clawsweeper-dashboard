## Summary

Keep open: current main and v2026.5.7 still enforce output-driven CLI no-output watchdogs with no Monitor or pending-tool expected-idle signal. The related open work covers timeout caps, cron overrides, orphan-output logging, and diagnostics, but none implements the Monitor wait contract reported here.

## What This Changes

The issue asks OpenClaw to stop killing CLI-backed agent sessions that are intentionally silent while waiting for a long-running Monitor task to complete.

## Best Possible Solution

Add a bounded expected-idle signal for verified pending Monitor or tool waits in the Claude live-session or supervisor path, while preserving the overall timeout and normal no-output protection when no pending work is known.

## Reproduction Assessment

Yes. The issue gives concrete Monitor wait steps and matching watchdog logs, and current main still has a source-reproducible path where silent CLI turns are timed from stdout/stderr activity rather than pending Monitor state.

## Solution Assessment

Unclear. Treating verified pending Monitor or tool waits as expected idle is the right direction, but the maintainable seam needs maintainer choice; raising constants or deferring orphan-output idle close only mitigates adjacent cases.
