## Summary

Current main still enforces CLI no-output watchdogs from stdout/stderr activity with no verified pending Monitor/tool-wait exemption, so this remains a valid reliability bug rather than a cleanup-close item.

## What This Changes

The issue asks for CLI watchdog logic to avoid killing healthy sessions that are intentionally idle while waiting for a long-running Monitor task.

## Best Possible Solution

Add a bounded expected-idle contract for verified pending Monitor or tool waits in the Claude live-session and generic CLI supervisor paths, while preserving the overall request timeout and normal no-output protection when no pending work is proven.

## Reproduction Assessment

Yes. The report gives concrete Monitor steps and matching watchdog logs, and current main still has source-level evidence that idle CLI turns are timed by stdout/stderr rather than pending Monitor state.

## Solution Assessment

Unclear. Treating verified Monitor/tool waits as expected idle is the right direction, but the maintainable seam should be chosen deliberately rather than only raising timeout constants or disabling the watchdog globally.
