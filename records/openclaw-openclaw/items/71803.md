## Summary

Keep open. Current main still treats CLI silence as a watchdog failure in both the supervised CLI path and the Claude live-session path, with no source-visible pending Monitor/tool-wait signal that would exempt healthy expected idle time.

## What This Changes

The issue requests that CLI no-output watchdogs distinguish healthy idle Monitor/tool waits from stalled CLI processes so long-running Monitor jobs do not terminate active sessions.

## Best Possible Solution

Add an explicit expected-idle or proof-of-life contract for pending Monitor/MCP/client-tool waits in the CLI runner and Claude live-session path, while preserving the overall request timeout and retaining no-output watchdog protection for genuinely stalled CLIs.

## Reproduction Assessment

Yes. The issue provides concrete Monitor wait steps and matching watchdog logs, and current main source/tests show silent CLI processes still hit the no-output watchdog without a pending Monitor exemption.

## Solution Assessment

No current fix is sufficient. The timeout-cap work in #41022 is related, but the maintainable fix should identify verified pending tool work or emit a bounded heartbeat instead of treating all silence the same.
