## Summary

Keep open. Current main still forces hook-agent runs through fresh isolated sessions, and the issue has an open unmerged closing PR for the persistent hook-session mode.

## What This Changes

The issue requests multi-turn webhook hook sessions when a stable sessionKey is supplied, preferably through an explicit persistent session mode.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit sessionMode implementation that defaults to isolation and documents the trust boundary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a stable-key /hooks/agent request becomes an isolated CronJob, then resolveCronSession receives forceNew: true and cannot reuse the existing sessionId.

## Solution Assessment

Yes. The explicit sessionMode direction is the narrowest maintainable solution because it enables trusted webhook continuity without changing default isolated hook behavior.
