## Summary

Keep open. Current main still forces stable-key hook-agent requests through isolated cron sessions, and the linked open closing PR at https://github.com/openclaw/openclaw/pull/75918 has not merged.

## What This Changes

Request stable-key webhook hook sessions to reuse an existing conversation session id for multi-turn webhook interactions.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit `sessionMode` implementation that defaults to isolation and documents the webhook trust boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows a stable-key `/hooks/agent` request becomes an isolated CronJob, then `resolveCronSession` receives `forceNew: true` and cannot reuse the existing session id.

## Solution Assessment

Yes at the direction level. An explicit `sessionMode: "isolated" | "persistent"` option is the narrow maintainable path because it enables trusted webhook continuity without changing default isolated behavior.
