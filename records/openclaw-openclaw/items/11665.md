## Summary

Keep open: current main still forces hook-agent runs through isolated cron sessions, so a stable hook `sessionKey` cannot reuse the existing transcript, and the open linked implementation candidate has not merged.

## What This Changes

The issue requests multi-turn webhook hook-agent sessions by reusing an existing session when a stable hook `sessionKey` is provided.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/75918; the target end state is an explicit `sessionMode: "isolated" | "persistent"` hook option with isolated default, security docs, schema/docs generation, and regression coverage.

## Reproduction Assessment

Yes, at source level: the hook path hard-codes isolated dispatch and the isolated runner passes `forceNew: true`, so stable hook keys cannot reuse an existing session on current main.

## Solution Assessment

Unclear until the linked PR review completes; the best known direction is an explicit persistent `sessionMode` opt-in rather than automatic reuse by default, because session reuse crosses a security boundary.
