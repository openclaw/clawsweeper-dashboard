## Summary

Keep open. Current main still forces hook-agent requests through isolated execution, and https://github.com/openclaw/openclaw/pull/75918 is an open unmerged closing PR for the persistent hook-session mode, so this issue should remain open until that PR or an equivalent implementation lands.

## What This Changes

Request webhook hook-agent invocations with a stable `sessionKey` to opt into reusing the same session for multi-turn history.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit `sessionMode: "isolated" | "persistent"` implementation that defaults to isolation and documents the trust boundary.

## Reproduction Assessment

Yes, from source inspection. A direct or mapped hook with a stable `sessionKey` still reaches `resolveCronSession` with `forceNew: true`, which prevents reuse of an existing session id.

## Solution Assessment

Yes. An explicit persistent-session opt-in is the narrowest maintainable direction because it enables trusted multi-turn webhook threads without changing default isolated hook behavior; the current open PR still needs to land or be replaced.
