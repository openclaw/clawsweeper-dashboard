## Summary

Keep open. Current main still forces stable-key hook-agent requests through isolated runs, and the active closing implementation is the open PR at https://github.com/openclaw/openclaw/pull/75918.

## What This Changes

Requests persistent multi-turn webhook hook sessions when a stable hook sessionKey is provided, preferably through an explicit sessionMode option for direct hooks, mappings, and transforms.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit sessionMode implementation that defaults to isolation and documents the trust boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows a stable-key /hooks/agent request becomes an isolated CronJob, then resolveCronSession receives forceNew: true and cannot reuse the existing sessionId.

## Solution Assessment

Yes. The explicit sessionMode direction is the narrowest maintainable solution because it enables trusted webhook continuity without changing default isolated hook behavior.
