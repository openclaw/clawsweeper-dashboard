## Summary

Keep open. Current main still forces hook-agent requests through isolated execution, and open PR #75918 already uses closing syntax for the persistent hook-session mode, so this issue should remain open until that PR or an equivalent implementation lands.

## What This Changes

Request multi-turn webhook hook sessions for stable `sessionKey` values, preferably through an explicit persistent session mode.

## Best Possible Solution

Review and land #75918, or an equivalent explicit `sessionMode: "isolated" | "persistent"` implementation that defaults to isolation and documents the trust boundary.

## Reproduction Assessment

Yes, from source inspection. A direct or mapped hook with a stable `sessionKey` still reaches `resolveCronSession` with `forceNew: true`, which creates a fresh session id.

## Solution Assessment

Yes. An explicit persistent-session opt-in is the narrowest maintainable direction because it enables trusted multi-turn webhook threads without changing the default isolated hook behavior.
