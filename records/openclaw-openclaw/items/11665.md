## Summary

Keep open. Current main still hard-codes hook-agent runs as isolated, which makes the runner force a fresh session id; the linked closing PR is open rather than merged.

## What This Changes

Request multi-turn webhook hook sessions by reusing an existing session when a stable hook session key is explicitly opted into persistence.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit `sessionMode` implementation that defaults to isolation, preserves session-key policy gates, and updates schema, docs, and focused tests.

## Reproduction Assessment

Yes, by source inspection on current main. Stable hook session keys can resolve through policy, but dispatch still sets `sessionTarget: "isolated"`, causing `forceNew: true` and blocking `resolveCronSession` reuse.

## Solution Assessment

Yes at the direction level. An explicit `sessionMode: "isolated" | "persistent"` opt-in is safer than automatic reuse because it preserves default isolation and makes the security choice visible.
