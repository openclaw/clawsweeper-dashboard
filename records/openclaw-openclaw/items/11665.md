## Summary

Keep open: current main still routes stable-key hook-agent calls through isolated cron execution, which forces a fresh session id, and an open closing PR already references this issue.

## What This Changes

Allow webhook hook-agent calls with a stable `sessionKey` to opt into multi-turn session reuse instead of creating a fresh isolated session each invocation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit persistent hook-session mode, with default isolation, policy gates, docs, schema, and focused runtime proof intact.

## Reproduction Assessment

Yes, by source inspection on current main. Stable hook session keys can pass policy, but hook dispatch still sets `sessionTarget: "isolated"`, causing `forceNew: true` and blocking stored session-id reuse.

## Solution Assessment

Yes at the direction level. An explicit `sessionMode: "isolated" | "persistent"` opt-in is safer than automatic reuse because it preserves default isolation and makes the security choice visible.
