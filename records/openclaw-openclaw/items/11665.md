## Summary

Keep open. Current main still routes stable-key hook-agent requests through forced-new isolated sessions, and the open closing PR at https://github.com/openclaw/openclaw/pull/75918 has not merged.

## What This Changes

The issue requests multi-turn webhook hook-agent sessions by reusing an existing session when callers provide a stable session key.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit `sessionMode: "isolated" | "persistent"` implementation that defaults to isolation and documents the webhook trust boundary.

## Reproduction Assessment

Yes, by source inspection on current main. A stable-key hook-agent request becomes an isolated cron job, `resolveCronSession` receives `forceNew: true`, and the existing session id cannot be reused.

## Solution Assessment

Yes at the direction level. An explicit persistent-session mode is the narrow maintainable fix because it enables trusted webhook continuity without changing default isolation; automatic deterministic reuse should not be adopted without maintainer policy approval.
