## Label Justifications

- `P2`: Webhook session continuity has real user impact but remains bounded to hook, config, and session-runtime behavior rather than an urgent outage.
- `impact:session-state`: The issue is about stable hook sessionKey values failing to preserve the intended session transcript and context across turns.
- `impact:data-loss`: Each webhook invocation can create a new transcript identity, effectively losing conversational history for the active workflow and leaving old transcript files orphaned.
- `impact:security`: Any persistent hook-session fix must preserve external session-key authorization and prefix controls to avoid cross-user or cross-integration context joins.

## Summary

Keep open: current main and the latest release still force hook-agent dispatch through isolated cron runs, so stable hook session keys cannot preserve transcript continuity. The linked open implementation candidate is https://github.com/openclaw/openclaw/pull/75918, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks hook-agent webhook sessions with a stable sessionKey to support multi-turn reuse instead of always creating fresh isolated session transcripts.

## Best Possible Solution

Land or narrow https://github.com/openclaw/openclaw/pull/75918, or an equivalent PR, so direct /hooks/agent, hook mappings, and transform results get an explicit sessionMode: "isolated" | "persistent" surface with isolated as the default, schema/docs updates, and session-key safety tests.

## Reproduction Assessment

Yes, at source level. Direct and mapped hook-agent requests resolve a stable sessionKey, but current main dispatches them as isolated cron runs and passes forceNew, so repeated requests cannot reach the existing-session reuse path.

## Solution Assessment

Yes for the direction, but it is not completed on main. An explicit persistent session mode is safer than automatic reuse because it keeps the current isolated default and forces operators to opt into the session-key trust model.
