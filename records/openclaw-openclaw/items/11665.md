## Summary

Keep open. Current main still routes stable-key hook-agent requests through forced isolated sessions, and the active implementation path is the open closing PR at https://github.com/openclaw/openclaw/pull/75918.

## What This Changes

The issue asks for hook-agent requests and mappings to opt into reusing a stable `sessionKey` across webhook turns instead of always starting an isolated session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit `sessionMode` implementation that defaults to isolation and documents the webhook trust boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows a stable-key `/hooks/agent` request becomes an isolated CronJob, then `resolveCronSession` receives `forceNew: true` and cannot reuse the existing session id.

## Solution Assessment

Yes at the direction level. An explicit `sessionMode: "isolated" | "persistent"` option is the narrow maintainable solution because it enables trusted webhook continuity without changing default isolated behavior.
