## Summary

Keep open. Current main still routes hook-agent requests through isolated cron-style execution, and the open closing PR #75918 already targets the requested persistent hook-session mode.

## What This Changes

The issue requests a persistent webhook hook-session mode so stable hook sessionKey values can reuse transcript history across turns.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit sessionMode implementation that defaults to isolated and documents the trust boundary.

## Reproduction Assessment

Yes, from source inspection. A direct or mapped hook with a stable sessionKey still becomes an isolated job and reaches resolveCronSession with forceNew: true on current main.

## Solution Assessment

Yes. An explicit persistent-session opt-in is the narrowest maintainable direction because it enables trusted multi-turn webhook threads without changing default isolated hook behavior.
