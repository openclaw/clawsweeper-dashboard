## Summary

Keep open. Current main still forces stable-key hook-agent requests through fresh isolated sessions, and the open closing PR at https://github.com/openclaw/openclaw/pull/75918 has not merged.

## What This Changes

Request a persistent hook-session mode so `/hooks/agent` and hook mappings with stable `sessionKey` values can reuse transcript history across webhook invocations.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit `sessionMode: "isolated" | "persistent"` implementation that defaults to isolation and documents the webhook trust boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows a stable-key hook-agent request becomes an isolated `CronJob`, then `resolveCronSession` receives `forceNew: true` and cannot reuse the existing session id.

## Solution Assessment

Yes at the design level. An explicit persistent hook-session mode is the narrow maintainable path because it enables trusted webhook continuity without changing default isolated behavior; the open PR still needs review and merge before this issue should close.
