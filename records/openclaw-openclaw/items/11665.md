## Summary

Keep open. Current main still routes hook-agent requests through the isolated cron runner with forced-new session behavior, and the live linked PR https://github.com/openclaw/openclaw/pull/75918 is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests multi-turn webhook hook sessions by reusing an existing session when a stable hook sessionKey is supplied.

## Best Possible Solution

Review and either land or close the active explicit `sessionMode: "isolated" | "persistent"` PR, keeping isolated as the default and documenting the security model for stable external hook session keys.

## Reproduction Assessment

Yes. Source inspection shows stable hook session keys enter an isolated cron-style run that passes `forceNew: true`, while the direct payload, mapping, transform, and strict config schema surfaces have no persistent-session option.

## Solution Assessment

Yes, the best known path is the explicit persistent hook-session mode with `isolated` as the default. Automatic reuse for every stable session key would be a broader product and security decision.
