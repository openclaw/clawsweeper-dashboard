## Summary

Keep open. Current main still cannot reuse stable-key hook sessions, and the open closing PR is the active implementation path rather than a reason to close before merge.

## What This Changes

Webhook hook-agent calls should support stable `sessionKey` multi-turn reuse without changing the default isolated hook behavior.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit persistent hook-session implementation that defaults to isolation, preserves session-key policy gates, updates config/docs, and closes this issue after merge.

## Reproduction Assessment

Yes, by source inspection on current main. Stable-key direct and mapped hook-agent calls can resolve a `sessionKey`, but dispatch sets `sessionTarget: "isolated"`, the runner passes `forceNew: true`, and `resolveCronSession` cannot reuse the stored session id.

## Solution Assessment

Yes at the direction level. An explicit `sessionMode: "isolated" | "persistent"` opt-in is the narrow maintainable solution because it enables trusted webhook continuity while preserving default isolation and existing session-key gates.
