## Summary

Keep open. Current main still routes stable-key hook-agent requests through forced-new isolated sessions, and the open closing PR at https://github.com/openclaw/openclaw/pull/75918 has not merged.

## What This Changes

The issue requests multi-turn webhook hook sessions by reusing an existing session when a stable hook `sessionKey` is supplied.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75918, or an equivalent explicit `sessionMode: "isolated" | "persistent"` implementation that defaults to isolation and documents the webhook trust boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows a stable-key `/hooks/agent` request becomes an isolated CronJob, then `resolveCronSession` receives `forceNew: true` and cannot reuse the existing session id.

## Solution Assessment

Yes at the direction level. An explicit persistent hook-session mode is the narrow maintainable path because it enables trusted webhook continuity without changing default isolated behavior; the open PR still needs normal review and merge before this issue should close.
