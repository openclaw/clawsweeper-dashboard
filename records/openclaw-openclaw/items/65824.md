## Summary

Keep open. Current main has meaningful adjacent coverage, including fallback lifecycle events, verbose fallback notices, config last-known-good recovery, plugin/tool hooks, and loop-detection controls, but the central bundle is not implemented: non-verbose fallback visibility and context re-injection remain unresolved, model failover hooks are only partially split into the focused #70976/#70990 work, and there is no first-class pre-call budget gate, scheduled model health/preflight, path-level file deny list, or subagent respawn circuit breaker.

## What This Changes

The issue asks OpenClaw to address a consolidated set of production gaps around model fallback visibility/context, config and crash recovery, tool-output safety, subagent guardrails, hooks, file/path trust, per-binary trust, cost budgets, and model health checks.

## Best Possible Solution

Keep this as a maintainer-triage umbrella or split it into canonical focused issues, review the narrow #70990 model-failover hook work separately, and implement the remaining high-priority slices with explicit config contracts, tests, and docs for fallback visibility/context, safe config transitions, bounded subagent/cost loops, model health/preflight, and uniform tool policy.

Partly yes. The fallback visibility gap is source- and test-backed on current main, and the latest reporter update provides a concrete model-compatibility failure; several other entries are feature requests with production workarounds and need focused reproduction or design checks per slice.
