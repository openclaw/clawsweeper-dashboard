Keep open. Current origin/main still resolves assistant emoji in the gateway, but the bootstrap contract, UI identity state, and grouped chat rendering path do not carry it to the assistant sender label; the closed #45898 was selector-dropdown-only and did not land.

Restore the agent identity emoji that the v2 Control UI dropped from the assistant chat sender label.

## Best Possible Solution

## Solution Assessment
