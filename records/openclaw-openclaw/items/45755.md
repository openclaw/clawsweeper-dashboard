## Summary

Keep open. Current main still resolves the assistant identity emoji in the gateway, but the bootstrap contract, UI identity state, app render props, and grouped chat footer only carry/render name and avatar; related #45898 was closed unmerged and targeted the selector dropdown rather than the chat sender label.

## What This Changes

Restore the configured agent identity emoji inline with the assistant name in the Control UI chat sender label.

## Best Possible Solution

Make assistant identity emoji a first-class optional value through gateway bootstrap/RPC, UI state, chat props, and assistant footer rendering, with regression coverage for streaming and persisted chat groups.

## Reproduction Assessment

Yes, at source level. Configure an agent identity emoji and render assistant chat groups on current main: the gateway resolves/returns the emoji, but the UI state and render props lose it before `.chat-sender-name` is emitted.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate the existing identity emoji through the Control UI identity contract and render it beside the assistant sender name; selector-dropdown-only work like #45898 does not address the transcript label regression.
