## Summary

Keep open. Current main still computes the assistant emoji in the gateway, but the bootstrap contract, UI identity state, and grouped chat footer rendering only carry/render the assistant name and avatar; the related #45898 was closed unmerged and targeted the selector dropdown rather than the chat sender label.

## What This Changes

The issue reports that the v2 Control UI dropped the agent identity emoji from the assistant message header where v1 rendered it inline with the agent name.

## Best Possible Solution

Carry the resolved assistant identity emoji through the bootstrap/RPC identity contract, UI state, render props, and grouped chat footer, then show it inline with the assistant sender name for streaming and persisted assistant groups while preserving avatar override behavior.

## Reproduction Assessment

Yes. A source-level reproduction is to provide an assistant identity with emoji, render assistant chat groups on current main, and observe that the value is lost before .chat-sender-name is produced.

## Solution Assessment

Yes. The narrow maintainable fix is identity propagation plus chat footer rendering; selector-dropdown-only work like #45898 does not address the reported transcript label regression.
