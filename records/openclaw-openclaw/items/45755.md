## Summary

Keep open. Current main still computes and can return the assistant identity emoji, but the Control UI bootstrap contract, assistant identity state, app render props, and grouped chat footer only carry/render name and avatar, so the emoji is dropped before the chat sender label. The linked selector-dropdown attempt at https://github.com/openclaw/openclaw/pull/45898 was closed unmerged and did not address the chat sender label path.

## What This Changes

The issue asks to restore the configured agent identity emoji next to the assistant name in the v2 Control UI chat message header/footer.

## Best Possible Solution

Make assistant identity emoji a first-class optional value through gateway bootstrap/RPC, UI assistant identity state, chat render props, and assistant footer rendering, with regression coverage for streaming and persisted chat groups.

## Reproduction Assessment

Yes, at source level. Configure an agent identity emoji: current main resolves and can return it from the gateway, but the Control UI normalizer, app props, and grouped chat footer omit it before `.chat-sender-name` is rendered.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate the existing identity emoji through the Control UI identity contract and render it beside the assistant sender name; selector-dropdown-only work does not solve the transcript label regression.
