## Summary

Keep this open: current main and the latest release still register only `google` for the Google plugin image/video generation contracts, and the open closing PR has not landed. The ClawHub `vertex-image-adc` and `vertex-video-adc` skills are useful temporary workarounds, but they bypass rather than fix the bundled Google provider routing bug.

## What This Changes

The issue requests registering `google-vertex` as an image and video generation provider in the Google plugin manifest.

## Best Possible Solution

Land or replace the linked Google plugin fix with a clean implementation that registers `google-vertex` through both manifest contracts and runtime-resolvable media provider ids or aliases, preserves Vertex auth/base URL behavior, and includes focused tests plus redacted Vertex CLI proof.

## Reproduction Assessment

Yes from source: current main and `v2026.6.8` exclude `google-vertex` from the image/video generation contract arrays, and the runtime lookup paths depend on those provider registrations. I did not run a live Vertex CLI generation because this read-only review lacks credentialed external API proof.

## Solution Assessment

No close yet. The manifest is the right owner boundary for discovery, but the final fix should also ensure `google-vertex` is runtime-resolvable for media generation and should land through the linked PR or a clean replacement.
