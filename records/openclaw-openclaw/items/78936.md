## Summary

Keep open: the PR addresses a real source-visible gap, but the current patch still forwards the wrong attachment shape through the gateway and expands the ACP attachment boundary without a maintainer-approved opt-in decision or live Codex vision proof.

## What This Changes

Adds ACP image attachment validation and forwarding for `sessions_spawn(runtime="acp")`, threads attachments through ACP spawn and agent-command paths, updates docs/changelog, and adds focused tests plus two OpenAI test cast fixes.

## Best Possible Solution

Forward ACP images using the gateway RPC attachment contract or extend the gateway normalizer with coverage, then land ACP image forwarding only behind an explicit maintainer-approved attachment boundary and live Codex proof.

## Reproduction Assessment

Yes. Source inspection of current main shows ACP `sessions_spawn` rejects attachments before dispatch, and source inspection of the PR shows its new gateway hop would drop the proposed attachment object shape; I did not run a live ACP/Codex repro in this read-only pass.

## Solution Assessment

No. The direction is useful, but the current implementation should use the gateway attachment contract or extend the normalizer, and maintainers need to approve the ACP attachment opt-in boundary before this is the maintainable fix.
