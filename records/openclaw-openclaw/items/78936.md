## Summary

Keep open: the linked issue is valid and the PR is useful, but the current diff should not merge as-is because it bypasses the existing `sessions_spawn` attachment policy, leaves the public docs contradictory, and still only provides test/typecheck proof instead of a real ACP/Codex image run.

## What This Changes

The PR removes the ACP `sessions_spawn` attachment rejection, forwards attachments through `spawnAcpDirect` and the Gateway agent call into ACP turns, adds focused tests, adjusts two OpenAI test casts, and adds a changelog note.

## Best Possible Solution

Route only validated image attachments through the existing ACP turn attachment contract, preserve the sessions_spawn opt-in and size/count limits, update the docs, and require redacted live Codex/acpx proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main rejects any `sessions_spawn` call with `runtime: "acp"` and attachments before ACP dispatch, while the ACP turn contract can already carry image attachments.

## Solution Assessment

No, not yet. Forwarding images through ACP is the right direction, but the PR must preserve attachment policy, update docs, and provide live after-fix proof before it is the narrow maintainable fix.
