## Summary

Keep open: the PR addresses a real ACP image-routing gap, but the current patch bypasses the existing sessions_spawn attachment opt-in/limits for ACP and still has only typecheck/unit-test proof, so it is not merge-ready.

## What This Changes

The PR removes the ACP attachment rejection, forwards sessions_spawn attachments through the Gateway agent call into acpManager.runTurn, adds focused tests and OpenAI test cast adjustments, and adds a changelog entry.

## Best Possible Solution

Land ACP image attachment plumbing only after it enforces the existing sessions_spawn attachment policy, documents the ACP image contract, and includes redacted live Codex/acpx image proof.

## Reproduction Assessment

Yes from source inspection: current main accepts inline sessions_spawn attachments at the schema but the ACP branch rejects them before dispatch, and the ACP runTurn call has no image attachments. I did not execute a live ACP/Codex session in this read-only review.

## Solution Assessment

No as currently proposed: the routing direction is maintainable, but the patch should enforce the existing sessions_spawn attachment opt-in and limits before forwarding to ACP. The supported contract should also be documented before merge.
