## Summary

Keep open: the PR is a plausible direction, but it is not merge-ready because the image path can still be blocked by Gateway model-capability parsing before ACP receives the attachment, the proof is still unit-test-only, and public docs would contradict the new behavior.

## What This Changes

The PR removes the ACP attachment rejection, forwards `sessions_spawn` attachments into the Gateway `agent` call, maps parsed agent images into ACP turn attachments, adds focused tests, tweaks two OpenAI test casts, and adds a changelog entry.

## Best Possible Solution

Route ACP `sessions_spawn` images through Gateway with ACP-aware image support parity, document the image-only validation boundary, and require redacted live Codex/acpx image proof before merge.

## Reproduction Assessment

Yes from source inspection: current main rejects ACP `sessions_spawn` attachments before `spawnAcpDirect`, and the ACP Gateway call omits attachments. I did not run a live ACP/Codex vision session in this read-only review.

## Solution Assessment

No, not yet. Forwarding through the existing ACP run-turn contract is the right direction, but the Gateway `agent` parser needs ACP-aware image support before `runTurn`, and docs plus real behavior proof are still missing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve ACP images through Gateway parsing:** `src/agents/acp-spawn.ts:1413-1415`
  - body: Forwarding attachments here still sends them through the Gateway `agent` parser before `agent-command.ts` can convert images to `AcpTurnAttachment`. That parser gates images on the OpenClaw session/default model, while ACP harnesses own their recipient model; unlike `chat.send`, this path does not treat ACP sessions as image-capable. A text-only-default setup can still reject the requested Codex/acpx vision workflow before it reaches `acpManager.runTurn`.
  - confidence: 0.86
- **[P3] Document ACP image attachment support:** `CHANGELOG.md:402`
  - body: The changelog announces user-facing ACP `sessions_spawn` image support, but `docs/gateway/config-tools.md` still says ACP rejects attachments and the ACP agents docs omit the new image-only boundary. Please update the docs so users can discover this path and its validation limits.
  - confidence: 0.8
