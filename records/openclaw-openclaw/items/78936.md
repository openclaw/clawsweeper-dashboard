## Summary

Keep open: the reported behavior is source-reproducible and the code direction is plausible, but the branch is not merge-ready because it bypasses the existing sessions_spawn attachment policy, leaves user docs saying ACP rejects attachments, and still has only mocked/unit-test proof.

## What This Changes

The PR removes the ACP attachment rejection, forwards `sessions_spawn(runtime="acp")` attachments through `spawnAcpDirect` into `acpManager.runTurn`, adds focused tests and a changelog entry, and adjusts two OpenAI test casts.

## Best Possible Solution

Land a narrow ACP image-attachment fix that reuses the Gateway image parser while honoring the existing sessions_spawn attachment opt-in and limits, updates the public docs, and includes redacted live Codex/acpx proof.

## Reproduction Assessment

Yes from source inspection: current main rejects ACP `sessions_spawn` attachments before dispatch and does not pass attachments through `spawnAcpDirect`. I did not run a live ACP/Codex vision session in this read-only review.

## Solution Assessment

No, not yet: forwarding through the existing Gateway image parser is the right direction, but the PR currently bypasses the documented `tools.sessions_spawn.attachments` opt-in and limits for ACP requests.
