## Summary

Keep open: the PR targets a real source-reproducible ACP image routing gap, but the latest diff still sends ACP attachments in a shape the gateway normalizer drops and broadens the shipped subagent attachment flag into ACP inline forwarding without a clear ACP-specific opt-in; the supplied proof is unit/typecheck output only.

## What This Changes

The PR enables image attachments for `sessions_spawn(runtime="acp")`, adds ACP attachment validation and forwarding through `spawnAcpDirect`/`agent-command`, and updates tests and docs.

## Best Possible Solution

Forward validated images through the gateway-supported attachment contract into `opts.images`, keep ACP image forwarding behind a clear maintainer-approved opt-in or policy, and require live Codex vision proof before merge.

## Reproduction Assessment

Yes at source level. Current main rejects `sessions_spawn(runtime="acp")` attachments before dispatch, and the linked open issue remains paired with this PR as the implementation candidate.

## Solution Assessment

No, not as currently patched. The direction is reasonable, but the PR uses a gateway-incompatible attachment shape and needs an ACP attachment policy decision before merge.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Send ACP images through the gateway attachment contract:** `src/agents/acp-spawn.ts:1465-1467`
  - body: `spawnAcpDirect` forwards `{ mediaType, data }`, but the gateway `agent` handler only normalizes attachment `content` or nested `source` data into `images`; these objects are filtered before `agent-command` calls `acpManager.runTurn`, so the image still never reaches ACP. Use the existing `{ type: "image", mimeType, content }` shape or another supported boundary and cover that path.
  - confidence: 0.92
- **[P1] Require an ACP-specific attachment opt-in:** `src/agents/tools/sessions-spawn-tool.ts:367-371`
  - body: This reuses `tools.sessions_spawn.attachments.enabled`, which was shipped and documented for subagent workspace materialization, to start forwarding inline image bytes to ACP runtimes. Existing configs that enabled subagent attachments should not silently expand their data-sharing scope; add an ACP-specific setting or get explicit maintainer approval for the shared flag with upgrade coverage.
  - confidence: 0.86
