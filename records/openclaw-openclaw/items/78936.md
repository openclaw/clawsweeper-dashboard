## Summary

Keep open: the reported ACP image-routing gap is valid, but this branch still needs attachment-policy enforcement, docs alignment, and live behavior proof before it is merge-ready.

## What This Changes

The PR removes the ACP `sessions_spawn` attachment rejection, forwards attachments through `spawnAcpDirect` and the Gateway agent call, passes inline images into ACP agent turns, adds focused tests, adjusts two OpenAI test casts, and adds a changelog entry.

## Best Possible Solution

Land a narrow implementation that routes only validated image attachments through Gateway/ACP while honoring the existing sessions_spawn attachment opt-in and limits, updates the public docs, and includes redacted live Codex/acpx proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main rejects `sessions_spawn` calls with `runtime: "acp"` and any attachment before ACP dispatch, and `spawnAcpDirect` does not pass attachments to the Gateway agent call.

## Solution Assessment

No, not yet. Forwarding through the existing Gateway image parser is the right direction, but this PR needs to preserve the documented sessions_spawn attachment opt-in/limits and update the docs before it is the best maintainable fix.
