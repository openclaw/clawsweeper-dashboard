## What This Changes

The PR removes the ACP `sessions_spawn` attachment rejection, forwards attachments through the Gateway agent call into `acpManager.runTurn`, adds focused tests, adjusts two OpenAI test casts, and adds a changelog note.

## Best Possible Solution

Route only validated image attachments through ACP/Gateway after applying the existing sessions_spawn attachment opt-in and limits, document the ACP image boundary, and include redacted live Codex/acpx proof.

Yes by source inspection. Current main rejects ACP `sessions_spawn` attachments before dispatch and `spawnAcpDirect` does not send attachments to the Gateway agent call, although I did not run a live Codex vision session in this read-only review.

## Solution Assessment

No, not yet. Forwarding through the Gateway image parser is the right direction, but the PR needs to preserve the existing sessions_spawn attachment policy, update docs, and provide live after-fix proof before it is the best maintainable fix.
