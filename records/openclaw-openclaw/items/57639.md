## Summary

Keep open: current main and the latest release still only enforce global subagent concurrency, spawn depth, and active-child limits; a lifetime per-session spawn budget remains a new public config feature that needs maintainer semantics.

## What This Changes

The issue requests a `subagents.maxPerSession` or `maxTotal` setting to cap total subagents spawned over a session lifetime.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, enforced consistently for native and ACP `sessions_spawn` with clear reset/archive semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection of config types/schema, docs, and both spawn enforcement paths gives high-confidence proof the requested lifetime budget is absent on current main.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, and reset semantics. The requested optional budget is likely the narrowest maintainable direction if unset preserves current behavior.
