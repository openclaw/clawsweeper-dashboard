## Summary

Keep open: current main and the latest release still lack a lifetime per-session subagent spawn budget. Existing controls cap global concurrency and active children only, so this remains a valid feature/config request that needs maintainer product semantics.

## What This Changes

Request a `subagents.maxPerSession` or `maxTotal` config setting that rejects further subagent spawns once a session reaches a lifetime spawn budget.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, and enforce it consistently for native and ACP `sessions_spawn` with explicit counting and reset semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and docs inspection gives high-confidence proof that current main and `v2026.5.20` lack the requested lifetime budget.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, reset semantics, and coordination with adjacent concurrency/session-budget requests. The requested optional budget is likely maintainable if unset preserves current behavior and both native and ACP spawn paths enforce it consistently.
