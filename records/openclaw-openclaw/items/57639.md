## Summary

Keep open: current main still lacks a typed, validated, documented, or enforced lifetime per-session cap for sequential subagent spawns; existing controls cover global concurrency, spawn depth, active children, timeout, and archiving only. The related open issues cover adjacent per-target concurrency and circuit-breaker behavior, but neither supersedes this lifetime-budget request.

## What This Changes

Request to add a configurable lifetime per-session cap on the total number of subagents spawned, beyond the existing concurrent subagent limits.

## Best Possible Solution

Add an optional documented lifetime spawn budget under the subagent config, preserve current behavior when unset, and enforce it consistently for native and ACP `sessions_spawn` with clear reset/archive semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current config types, strict schema, docs, and both spawn enforcement paths lack the requested lifetime cap.

## Solution Assessment

Unclear until maintainers choose naming, defaults, and reset semantics. The requested direction is valid, and an optional maxPerSession-style budget enforced at spawn time is the narrowest maintainable shape if unset preserves existing behavior.
