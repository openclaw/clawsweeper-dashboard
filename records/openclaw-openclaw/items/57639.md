## Summary

Keep open: current main still has global subagent concurrency, spawn-depth, and active-child guards, but no typed, validated, documented, or enforced lifetime per-session cap for sequential subagent spawns. The request is concrete, but it adds a public config option and lifetime semantics that need maintainer judgment.

## What This Changes

Request adds a `subagents.maxPerSession` or `maxTotal` setting to cap the total number of subagents spawned over one session lifetime.

## Best Possible Solution

Add an optional documented lifetime spawn budget that defaults to current behavior when unset and is enforced consistently for native and ACP `sessions_spawn` with clear reset/archive/restart semantics.

## Reproduction Assessment

Yes. Source inspection shows sequential spawns can continue after prior children end because native and ACP spawn guards only count active children, and the registry test confirms ended children no longer count.

## Solution Assessment

Yes on direction, pending maintainer naming and reset-semantics review. A positive optional `maxPerSession`-style config enforced at spawn time is the narrowest maintainable shape if unset preserves existing behavior.
