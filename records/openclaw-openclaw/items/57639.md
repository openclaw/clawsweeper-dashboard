## Summary

Keep open: current main and the latest release still only expose global concurrency, depth, active-child, timeout, archive, and context-pressure controls for subagents, not a lifetime per-session spawn budget. The request remains a new public config feature with product semantics to settle before implementation.

## What This Changes

Request an optional per-session lifetime cap for total subagent spawns, separate from concurrent subagent limits.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, enforced consistently for native and ACP sessions_spawn with clear count, reset, archive, and upgrade semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new config capability. Source inspection of current main and v2026.5.19 gives high-confidence proof that the lifetime budget is absent.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, reset semantics, and how this composes with adjacent concurrency and circuit-breaker requests. The requested optional budget is likely the narrowest maintainable direction if unset preserves current behavior and both spawn paths enforce it consistently.
