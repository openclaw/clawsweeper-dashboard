## Summary

Keep open: current main and v2026.5.19 still provide concurrency, depth, active-child, timeout, archive, and context-pressure controls, but no lifetime per-session subagent spawn budget. Related open issues cover different resource-control semantics rather than replacing this request.

## What This Changes

Request an optional `subagents.maxPerSession` or `maxTotal` config limit that caps the total number of subagents spawned during one session.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, enforced consistently for native and ACP `sessions_spawn` with clear counting, reset, archive, and upgrade semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new config capability. Source and docs inspection give high-confidence proof that current main and v2026.5.19 lack the requested lifetime budget.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, reset semantics, and composition with adjacent concurrency and circuit-breaker requests. The requested optional budget is likely the narrowest maintainable direction if unset preserves current behavior and both spawn paths enforce it consistently.
