## Summary

Keep open: current main and the latest release still provide active/concurrent subagent controls, but no lifetime per-session spawn budget. This remains a real feature/config request that needs maintainer product judgment rather than conservative cleanup.

## What This Changes

Request an optional `subagents.maxPerSession` or equivalent cap on total subagents spawned over a session lifetime.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, enforced consistently for native and ACP `sessions_spawn` with explicit counting and reset semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new config capability. Source and docs inspection give high-confidence proof that current main and `v2026.5.19` lack the requested lifetime budget.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, reset semantics, and interaction with adjacent concurrency and circuit-breaker requests. The requested optional budget is likely maintainable if unset preserves current behavior and both native and ACP spawn paths enforce it consistently.
