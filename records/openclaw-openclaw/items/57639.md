## Summary

Keep open: current main still lacks a typed, validated, documented, or enforced lifetime per-session cap for sequential subagent spawns; existing guards cover global concurrency, spawn depth, and active children only.

## What This Changes

Request to add an optional `subagents.maxPerSession` or `maxTotal` lifetime cap that rejects further subagent spawns after a session reaches its budget.

## Best Possible Solution

Add an optional documented lifetime spawn budget that defaults to current behavior when unset and is enforced consistently for native and ACP `sessions_spawn` with clear reset/archive semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that the requested lifetime cap is missing from config types, strict schema, docs, and both spawn enforcement paths.

## Solution Assessment

Unclear as a final design until maintainers choose naming and reset semantics. The requested direction is valid, and an optional `maxPerSession`-style budget enforced at spawn time is the narrowest maintainable shape if unset preserves existing behavior.
