- `impact:session-state`: The request is about preventing unbounded subagent results from accumulating in a session transcript and exhausting context.

## Summary

Keep open: current main and the latest release still expose global concurrency, spawn depth, active-child, timeout, and archive controls, but no lifetime per-session subagent spawn budget. This remains a new public config feature with reset/counting semantics for maintainers to define.

## What This Changes

Request an optional `subagents.maxPerSession` or `maxTotal` setting that caps total subagents spawned over a session lifetime.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, enforced consistently for native and ACP `sessions_spawn` with clear count, reset, archive, and upgrade semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new config capability. Source inspection of config types/schema, docs, and native plus ACP spawn enforcement gives high-confidence proof the lifetime budget is absent on current main.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, and reset semantics. The requested optional budget is likely the narrowest maintainable direction if unset preserves current behavior and both spawn paths enforce it consistently.
