## What This Changes

The issue requests a `subagents.maxPerSession` or equivalent lifetime cap on total subagent spawns within one session.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, enforced consistently for native and ACP `sessions_spawn` with explicit counting and reset semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new config capability. Source and docs inspection give high-confidence proof that current main and `v2026.5.20` lack the requested lifetime budget.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, reset semantics, and interaction with adjacent concurrency and circuit-breaker requests. The requested optional budget is likely maintainable if unset preserves current behavior and both native and ACP spawn paths enforce it consistently.
