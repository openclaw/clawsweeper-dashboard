## Summary

Keep open: current main still has active/concurrent subagent controls but no lifetime per-session spawn budget, and the remaining work is a public config/product decision rather than an already-defined bug fix.

## What This Changes

The issue requests an optional `subagents.maxPerSession` or equivalent cap on the total number of subagents spawned during one session.

## Best Possible Solution

Add an optional documented lifetime spawn budget, disabled by default, enforced consistently for native and ACP `sessions_spawn` with explicit counting and reset semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this requests a new config capability. Source and docs inspection give high-confidence proof that current main and `v2026.5.20` lack the requested lifetime subagent budget.

## Solution Assessment

Unclear as a final design until maintainers choose naming, defaults, reset semantics, and interaction with adjacent concurrency and circuit-breaker work. The requested optional budget is likely maintainable if unset preserves current behavior and both native and ACP spawn paths enforce it consistently.
