## Summary

Keep open. Current main has addressed several audit bullets, but the dashboard-wide Control UI UX request is still only partially implemented: Instances remains refresh/presence-list oriented, Sessions still has dense per-row advanced controls and no explicit key-copy affordance, Config still exposes Save/Apply/Update without explanatory labels, and global shortcuts/action feedback remain limited. The related #39780 and #45711 are useful narrower follow-ups, not replacements for this umbrella audit.

## What This Changes

Requests a comprehensive Control UI UX audit and remediation across navigation, Overview, Instances, Sessions, Config, Usage, Agents, and design-system feedback for OpenClaw 2026.3.2.

## Best Possible Solution

Keep the issue open unless a maintainer intentionally splits it. The desired end state is focused Control UI work that preserves the already-fixed bullets while addressing the remaining gaps: per-instance actions or clearer presence affordances, session key copy/truncation and less repetitive advanced controls, clearer Config action semantics, broader shortcut/help UX, and durable action feedback or activity history. Treat #39780 and #45711 as narrower related trackers, not replacements.

## Reproduction Assessment

Not applicable as a single reproduction because this is a broad UX audit rather than one failing path. We do have high-confidence source-level checks for individual remaining gaps: Instances only refreshes system-presence, Sessions still renders per-row override selects without a copy action, and Config still shows Save/Apply/Update as terse action labels.

## Solution Assessment

No single automated fix is the best way to solve this issue. The maintainable path is to split or sequence the remaining UI work by surface, reuse the existing Control UI components/contracts, and let the narrower related reports guide Sessions and Config-specific changes.
