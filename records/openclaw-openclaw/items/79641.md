## Summary

Keep open: current main appears to have the duplicate context-engine `afterTurn` path and the checkpoint handoff is a plausible fix direction, but this PR needs real behavior proof and includes unrelated gateway/model-pricing health changes with a raw-error exposure concern.

## What This Changes

The branch wires a context-engine loop-hook checkpoint into final `afterTurn` finalization and also adds model-pricing bootstrap errors to health output.

## Best Possible Solution

Keep the fix scoped to the context-engine checkpoint handoff, add focused regression coverage plus redacted real-run proof, and split the model-pricing health work into its own reviewed PR.

## Reproduction Assessment

Yes, at source level: current main advances a loop-local `lastSeenLength` but finalizes with the original `prePromptMessageCount`, so an `ownsCompaction` engine that persists turn deltas can receive overlapping ranges. I did not run a live multi-tool harness in this read-only review.

## Solution Assessment

No, not as submitted: the checkpoint handoff is a narrow and plausible repair, but the PR is dirty with unrelated health changes and lacks regression/live proof. The safer path is a scoped context-engine patch with tests and redacted runtime evidence.
