## Summary

Keep open: current main still has a plausible user-abort path where the session fence can raise a raw takeover error, and the requested context-cleanup behavior is not implemented as a dedicated user-abort policy.

## What This Changes

The issue requests suppressing, downgrading, or replacing `EmbeddedAttemptSessionTakeoverError` in model-visible conversation context after a deliberate Stop/user abort.

## Best Possible Solution

Keep the session fence and logging intact, but add an explicit user-abort handling path that prevents raw takeover errors from becoming model-visible context after Stop, preferably without adding config unless maintainers want a visibility policy.

## Reproduction Assessment

No high-confidence live reproduction was established. The issue provides concrete steps and the source shows the user-abort and session-fence paths, but this review did not run a current-main Stop-button scenario.

## Solution Assessment

No for the config toggle as the default answer: the repo has a high bar for new config surface. The narrower maintainable direction is user-abort-specific context suppression with diagnostics preserved, if maintainers approve that policy.
