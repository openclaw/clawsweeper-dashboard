## Summary

Keep open: the localization direction is valid, but the PR is not mergeable against current main because Chat copy now lives across split helper modules, current locale generation covers many more bundles than the six files in the branch, the proposed fallback parameters are unsupported, and the composer placeholder would lose the active assistant name.

## What This Changes

The branch replaces selected hard-coded Chat view strings with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Repair or replace the branch so all current Chat UI copy uses the existing Control UI i18n source of truth, dynamic values use interpolation, generated locale outputs and metadata are synced for every supported locale, and focused tests cover the localized Chat surfaces without unrelated layout changes.

Yes. Static reproduction is high-confidence: current main still renders literal Chat copy in `ui/src/ui/views/chat.ts`, `ui/src/ui/chat/run-controls.ts`, `ui/src/ui/chat/status-indicators.ts`, and `ui/src/ui/chat/chat-queue.ts`, so changing the locale cannot affect those labels today.
