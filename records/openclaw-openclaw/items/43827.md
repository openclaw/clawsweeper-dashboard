## Summary

Keep open. The PR targets a real Control UI localization gap, but current main has Chat copy split across helper modules and a larger generated locale set, so this patch is incomplete and would also drop current assistant-name placeholder context.

## What This Changes

The PR replaces selected hard-coded Chat view strings in `ui/src/ui/views/chat.ts` with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Update the current Chat view and helper modules to use the i18n source of truth, preserve dynamic interpolation, regenerate all supported locale outputs and metadata, and cover the user-facing localization change with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Static source inspection on current main shows visible Chat copy still hard-coded in `chat.ts`, `run-controls.ts`, `status-indicators.ts`, `chat-queue.ts`, `context-notice.ts`, and `side-result-render.ts`, so a locale switch cannot localize those labels today.

## Solution Assessment

No. The PR points in the right direction, but the best fix must target current split helpers, preserve assistant-name interpolation, regenerate all supported locale outputs and metadata, and avoid unsupported fallback params.
