## Summary

Keep open: the PR addresses a real Chat localization gap, but current main has moved visible Chat copy into split helpers and supports more locales than the branch updates, so the patch is incomplete and would drop the assistant-aware composer placeholder after rebase.

## What This Changes

The branch replaces selected Chat view literals with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Repair or replace the branch so all current Chat UI copy uses the existing i18n source of truth, dynamic values use interpolation, all supported locales and `.i18n` metadata are synced, and focused tests plus a changelog entry cover the user-facing localization change.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Static inspection of current main shows visible Chat copy still hard-coded in `chat.ts` and split helper modules, so switching locale cannot translate those labels today.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is right, but the patch should be rebuilt against current main so helper modules, all supported locales/generated metadata, interpolation, and changelog are handled together.
