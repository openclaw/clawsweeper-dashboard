## What This Changes

The PR changes the Codex app-server turn-start path so OpenClaw runtime context is no longer prepended to `turn/start.input` and is instead passed through turn-scoped collaboration instructions, with related tests and prompt snapshots updated.

## Best Possible Solution

Use a truly turn-scoped reference-context channel if Codex app-server supports one; otherwise sanitize or reset historical decorated user items while keeping only the real user request in persisted native history, with multi-turn and compact regression proof.

## Reproduction Assessment

Yes. Current main clearly prepends the OpenClaw runtime-context marker into `turn/start.input`, and the linked issue comments provide structured live logs showing those decorated `role=user` items replayed in `response.create` and compact inputs.

## Solution Assessment

No, not yet. The direction addresses future user-input pollution, but the PR should either use a confirmed transient reference channel or sanitize existing decorated history, and the `developer_instructions` authority tradeoff needs maintainer approval.
