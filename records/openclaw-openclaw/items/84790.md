## Summary

Keep open: this targets a real Codex app-server session-state bug, but the latest head is not merge-ready because it moves raw user/workspace context onto the developer-instructions path, sanitizes historical user messages too broadly, and still only supplies harness/protocol proof rather than a real after-fix Codex turn.

## What This Changes

The PR stops prepending Codex runtime context to `turn/start.input`, passes current-turn OpenClaw context through collaboration-mode developer instructions, sanitizes stale mirrored history wrappers, clears affected native thread bindings, and updates Codex tests and prompt snapshots.

## Best Possible Solution

Keep current-turn context out of persisted user input while preserving prompt authority: use a Codex-supported transient/reference channel if one exists, or strictly quote/fence user-editable context on the supported path and sanitize only exact legacy wrappers with fresh-session and upgrade proof.

## Solution Assessment

Require the marker at the wrapper start, or parse the exact old envelope, and add a non-wrapper regression test.
