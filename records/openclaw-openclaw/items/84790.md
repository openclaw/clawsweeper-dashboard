## Summary

Keep this PR open: it targets a real Codex session-state bug and adds useful stale-history sanitization, but the replacement path sends OpenClaw skills/MEMORY context through `collaborationMode.settings.reference_context`, which is not in the current Codex app-server collaboration settings contract, so real Codex turns would not reliably receive that context. The submitted proof is still harness and snapshot validation rather than a real Codex app-server run.

## What This Changes

The PR changes Codex app-server turns to keep OpenClaw runtime context out of `turn/start.input`, adds stale mirrored-history sanitization and thread reset behavior, and updates Codex prompt snapshots and tests.

## Best Possible Solution

Use a Codex-supported, generated protocol path for lower-authority turn-scoped reference context; if that path does not exist yet, keep current-turn context model-visible through a supported interim path while sanitizing historical replay, then prove the live Codex app-server behavior.
