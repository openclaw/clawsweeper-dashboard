## Summary

Keep this PR open, but it is not merge-ready: the stale-history sanitization is useful, while the replacement context path depends on a locally added `reference_context` field that is not in the Codex collaboration settings contract I found.

## What This Changes

The PR moves Codex current-turn OpenClaw runtime context out of `turn/start.input` into `collaborationMode.settings.reference_context`, sanitizes stale mirrored history wrappers, resets affected native thread bindings, and updates Codex prompt snapshots and tests.

## Best Possible Solution

Use a generated, Codex-supported transient reference-context channel, or keep the context model-visible through an existing supported path while sanitizing historical replay and proving fresh-session plus upgrade behavior.

## Reproduction Assessment

Yes. Current main visibly builds `OpenClaw runtime context for this turn:` and prepends it into the Codex turn input path, while the linked issue provides concrete replay/growth evidence; I did not run a live Codex repro in this read-only review.

## Solution Assessment

No. The history sanitization direction is sound, but moving current-turn context into `reference_context` is not the best fix until Codex's generated protocol proves that field is consumed as model-bound reference data.
