## Summary

The PR removes OpenClaw runtime context from Codex `turn/start.input`, routes it through collaboration developer instructions, sanitizes stale mirrored history wrappers, clears affected native thread bindings, and updates tests and prompt snapshots.

## Best Possible Solution

Use a Codex-supported, non-persisted path for transient reference context, or split trusted OpenClaw-owned instructions from user-editable reference data and merge only after maintainers explicitly accept the prompt-boundary and upgrade tradeoff with real fresh-session plus stale-bound-session proof.
