## Summary

Keep open: current main has the Telegram rich-message opt-in/runtime fix, but the latest stable release is still v2026.6.8 and that tag lacks the schema key and still routes normal Telegram text through sendRichMessage. The new live comments strengthen the stable-release impact rather than making this safe to close.

## What This Changes

The issue asks for the stable release/docs state to match the Telegram richMessages opt-in behavior already present on main.

## Best Possible Solution

Publish or backport a stable release that includes the merged Telegram richMessages opt-in fix, or clearly caveat v2026.6.8 stable docs/release notes until the stable package matches the documented behavior.

## Reproduction Assessment

Yes. Source inspection of v2026.6.8 shows the missing richMessages schema key and unconditional sendRichMessage path, and multiple users supplied live stable-release confirmations; I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes for the code direction, but not as a close. Current main’s opt-in gate is the right runtime/schema fix; the remaining work is a maintainer release/backport or stable-docs decision.
