  - body: `Type.Union` emits `anyOf` for `action`, `runtime`, and `notifyPolicy`. Existing provider-facing tools use `stringEnum`/`optionalStringEnum` because some providers reject `anyOf`, so this task tool can fail registration.
  - confidence: 0.93
- **[P1] Replace team action union with a flat enum:** `src/agents/tools/team-tool.ts:22`
  - body: The team tool uses `Type.Union` for `action`, producing the same provider-incompatible `anyOf` schema. Use the shared flat string enum helper here as comparable action-dispatched tools do.
  - confidence: 0.93
- **[P2] Preserve ENOENT for missing sandbox reads:** `src/agents/sandbox/fs-bridge-path-safety.ts:80-84`
  - body: Wrapping every failed `openBoundaryWithinRequiredMount` result as `E_FS_PERMISSION_DENIED` changes missing-file reads from ENOENT into permission errors. Current callers and tests distinguish missing files from permission failures, so preserve ENOENT for path-not-found cases.
  - confidence: 0.86
- **[P2] Consolidate the sandbox readiness note:** `src/commands/doctor-sandbox.ts:241-253`
  - body: `runSandboxHealth` emits the new readiness note before image repair, and the non-Docker no-local-support branch emits overlapping capability details again. That makes doctor print duplicate sandbox readiness output for one condition.
  - confidence: 0.84
