- **[P1] Import Type from the current package in task:** `src/agents/tools/task-tool.ts:1`

  - confidence: 0.94
- **[P2] Preserve missing-read errors before wrapping denials:** `src/agents/sandbox/fs-bridge-path-safety.ts:80-84`
  - body: `openReadableFile` now wraps every failed boundary open as `permission_denied`, including ordinary missing-file opens. Current sandbox read behavior preserves the original missing-file error, so callers that branch on ENOENT/no-such-file semantics can regress.
  - confidence: 0.9
- **[P2] Register task and team in the core catalog:** `src/agents/openclaw-tools.ts:238-242`
  - body: The PR constructs `task` and `team` as core tools but does not add them to the catalog contract. Profiles, `group:openclaw`, tool sections, and known-core-tool checks are catalog-driven, so these tools will not behave consistently as first-class core tools.
  - confidence: 0.9
- **[P2] Use flat enum schemas for task:** `src/agents/tools/task-tool.ts:31-39`
  - body: `TaskToolSchema` uses `Type.Union(...Type.Literal...)` for `action`, `runtime`, and `notifyPolicy`, which emits `anyOf`. Existing tool schemas use flat enum helpers because some providers reject `anyOf` in tool schemas.
  - confidence: 0.94
- **[P2] Use a flat enum schema for team:** `src/agents/tools/team-tool.ts:22`
  - body: `TeamToolSchema.action` uses `Type.Union(...Type.Literal...)`, producing the same provider-incompatible `anyOf` shape. Use the shared flat enum helper here as well.
  - confidence: 0.93
- **[P2] Classify allowed-mount escapes as workspace boundaries:** `src/agents/fs-permission-denied.ts:22-25`
  - body: The classifier does not match `Sandbox path escapes allowed mounts`, while the PR wraps local and remote sandbox boundary errors using that text. Those denials become `reason=unknown` instead of `workspace_boundary`.
  - confidence: 0.9
- **[P2] Reject unknown task actions before output lookup:** `src/agents/tools/task-tool.ts:232-241`
  - body: After the explicit task actions, the handler falls through to the output lookup path. Direct invocation can bypass TypeBox validation, so a typo can become a misleading `task not found` response instead of an unsupported-action error.
  - confidence: 0.86
- **[P2] Translate missing team updates to ToolInputError:** `src/agents/tools/team-tool.ts:87-93`
  - body: `updateTeam()` throws a plain `Error` for an unknown team id, and this branch does not translate it. Direct tool invocation maps input errors differently from execution failures, so a user-correctable missing team update can surface as a tool failure.
  - confidence: 0.86
- **[P2] Consolidate the sandbox readiness note:** `src/commands/doctor-sandbox.ts:240-253`
  - body: `runSandboxHealth` calls the new readiness note before `maybeRepairSandboxImages`, but the non-Docker no-local-support branch also emits capability-probe details. Doctor can print overlapping sandbox readiness output for the same condition.
  - confidence: 0.84
