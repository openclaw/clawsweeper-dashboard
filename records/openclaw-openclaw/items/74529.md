## Summary

Keep open. This is a focused Matrix E2EE bug fix, but the current patch still matches the synthetic success against the original request metadata after redirects and the external PR lacks inspectable real behavior proof.

## What This Changes

The PR adds a Matrix guarded-fetch workaround that rewrites narrowly matched POST `/keys/upload` one-time-key collision 400 responses to a synthetic 200, with focused tests and a Matrix changelog entry.

## Best Possible Solution

Constrain the workaround to the effective final POST `/keys/upload` response, add redirect regression coverage, attach redacted after-fix proof, and rerun focused Matrix checks plus the failed CI jobs before maintainer sign-off.

## Reproduction Assessment

Yes, at source level. The related issue has exact duplicate OTK `/keys/upload` logs, current main passes the non-OK guarded-fetch response through unchanged, and matrix-js-sdk treats the 400 as fatal.

## Solution Assessment

No, not as currently written. The Matrix guarded-fetch boundary is a narrow OpenClaw-side mitigation point, but the matcher must use the effective final request after redirects before the synthetic success is safe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Match collisions against the effective final request:** `extensions/matrix/src/matrix/sdk/transport.ts:254-255`
  - body: `fetchWithMatrixGuardedRedirects` follows redirects and can change POST to GET for 301/302/303 responses, but the rewrite still checks the original `url` and initial method. A POST `/keys/upload` that redirects elsewhere can therefore have a collision-shaped final 400 rewritten to 200 outside the intended endpoint; please match the final URL/effective method and cover this with a redirect pass-through test.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff touches Matrix E2EE key-upload handling and currently allows the synthetic success decision to cross redirect boundaries.

Concerns:

- **[medium] Synthetic key-upload success can cross redirects:** `extensions/matrix/src/matrix/sdk/transport.ts:254`

For E2EE key upload handling, the success rewrite should be constrained to the effective final request.
