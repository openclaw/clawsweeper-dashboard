## What This Changes

The PR adds a Matrix guarded-fetch helper, tests, and changelog entry to rewrite duplicate `/keys/upload` one-time-key collision 400s into synthetic 200 responses.

## Best Possible Solution

Constrain the workaround to the effective final POST `/keys/upload` response, preserve every other Matrix error, add redirect regression coverage, and attach redacted live proof from an affected Matrix account.

## Reproduction Assessment

Yes, at source level. The related OpenClaw issue at https://github.com/openclaw/openclaw/issues/73480 includes an exact duplicate OTK `/keys/upload` 400, current main passes that non-OK response through, and matrix-js-sdk treats non-429 4xx errors as fatal.

## Solution Assessment

No, not as currently written. The guarded-fetch boundary is the right narrow mitigation point, but the synthetic success check must use the effective final request after redirects before the patch is safe.

## Review Findings
