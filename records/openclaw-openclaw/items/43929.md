## Summary

Keep open: current main and v2026.5.20 still expose active-session `chat.history` only, while the remaining work is an additive Gateway logical-history contract already being explored by related open PRs rather than something implemented on main.

## What This Changes

Request adds an optional logical-history mode to `chat.history` so one stable session key can return current and rolled transcript segments with lightweight metadata.

## Best Possible Solution

Choose one backward-compatible Gateway contract that keeps current history as the default, adds explicit logical/archive reads with bounded ordered segment metadata, and updates WebChat, `sessions_history`, docs, generated clients, and focused tests together.

## Reproduction Assessment

Yes, at source level. Current main has no logical/archive selector in `chat.history`, WebChat and `sessions_history` call the active-session path, and reset archive files are still created on disk.

## Solution Assessment

Unclear until maintainers choose the protocol and visibility contract. A Gateway-first additive mode is plausible, but it should be coordinated with the related archive-history PRs instead of handled as a competing standalone repair.
