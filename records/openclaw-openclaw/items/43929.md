- `impact:session-state`: The issue is about transcript and session continuity across rolled `sessionId` segments for one logical chat.

## Summary

Keep open: current main and the latest shipped release still expose active-session `chat.history` only, while reset archives exist on disk and related archive-history PRs remain open rather than merged.

## What This Changes

Request to add an optional logical-history mode to `chat.history` so one logical chat can include prior rolled transcript segments plus segment metadata.

## Best Possible Solution

Choose one backward-compatible Gateway contract that keeps current history as the default, adds an explicit logical/archive mode with bounded reads and ordered segment metadata, and updates WebChat, `sessions_history`, docs, generated clients, and tests together.

## Reproduction Assessment

Yes, at source level. Current main rejects any logical/archive selector, `chat.history` and `sessions_history` read only the active transcript, and reset archive files are still created on disk.

## Solution Assessment

Unclear until maintainers choose the protocol and visibility contract. A Gateway-first additive mode is plausible, but it should be coordinated with the related archive-history PRs instead of handled as a competing standalone repair.
