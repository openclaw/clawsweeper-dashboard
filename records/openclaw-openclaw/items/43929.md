## Summary

Keep open: current main still lacks a logical `chat.history` mode across reset segments, and the remaining work is an API/product decision with active related PRs rather than a conservative cleanup close.

## What This Changes

Request to add a logical-history mode to `chat.history` that returns the current segment plus prior reset segments for the same logical session key with lightweight segment metadata.

## Best Possible Solution

Define and land a coordinated, bounded read-only archive-chain API for `chat.history` and agent history tools, including segment metadata and docs, after maintainer approval of reset-archive visibility semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection confirms the current gap: reset archives are created, but `chat.history` has no logical-history parameter and reads only the active session entry.

## Solution Assessment

Unclear until maintainers settle the API and privacy contract. A read-only opt-in like the open related PR is plausible, but it must preserve bounded reads, segment ordering, metadata, docs, and reset-archive visibility expectations.
