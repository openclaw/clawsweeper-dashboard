## Summary

Keep open: the current-main bug is valid and the PR targets the right ACP projector surface, but the patch still allows untagged thought-stream text through, lacks the required changelog entry, and still needs after-fix real behavior proof.

## What This Changes

The PR reorders ACP `text_delta` filtering so tagged `stream: "thought"` events can honor `tagVisibility.agent_thought_chunk`, and adds a dispatch-level regression test file.

## Best Possible Solution

Gate thought-stream delivery on `agent_thought_chunk` visibility, or normalize that tag before projection, while preserving the unknown-stream drop and adding changelog, regression coverage, and redacted after-fix live proof.

## Reproduction Assessment

Yes. Current-main source shows `stream: "thought"` returns before tag visibility is checked, and the PR body/comment add a matching unit repro plus live pre-fix container evidence; I did not run tests because this review is read-only.

## Solution Assessment

No. Reordering the checks is the right direction, but the proposed guard is too broad because thought streams should be hidden unless they carry or are normalized to the `agent_thought_chunk` tag and that tag is visible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep thought streams gated by the thought tag:** `src/auto-reply/reply/acp-projector.ts:427`
