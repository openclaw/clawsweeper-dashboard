## Summary

Keep open: the PR targets a real current-main ACP projection bug, but the patch is not merge-ready because it can expose untagged thought-stream text, lacks after-fix real behavior proof, and needs a changelog entry.

## What This Changes

The branch reorders ACP projector filtering so visible thought-stream `text_delta` events can be delivered and adds a dispatch-level regression test suite for thought chunk delivery.

## Best Possible Solution

Tighten the projector so thought streams are hidden unless they carry or are normalized to `agent_thought_chunk` and that tag is visible, then add regression coverage, a changelog entry, and redacted live ACP delivery proof.

## Reproduction Assessment

Yes. Source inspection of current main shows a high-confidence path: `stream: "thought"` returns before `tagVisibility` is checked, and the PR’s added red test describes the same failing unit path; I did not run it because this review is read-only.

## Solution Assessment

No. Reordering the visibility check is the right direction, but the proposed guard is too broad because it allows untagged thought streams; the safer solution is to gate thought delivery specifically on `agent_thought_chunk` visibility or normalize that tag first.

## Review Findings

Overall correctness: patch is incorrect

- **[P1] Keep thought streams gated by the thought tag:** `src/auto-reply/reply/acp-projector.ts:427`
