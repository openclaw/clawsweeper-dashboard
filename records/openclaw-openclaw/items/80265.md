## Summary

Keep open: the linked bug is valid and source-reproducible, but this PR is not mergeable as written because it suppresses local credential fingerprints for all profile-backed runs, breaking the documented combined-mode auth-epoch boundary, and its proof is only unit/mock output.

## What This Changes

The PR bumps the CLI auth epoch version, skips local CLI credential hashing whenever an auth profile credential exists, updates auth-epoch unit tests, and adds changelog entries.

## Best Possible Solution

Stabilize the Claude CLI storage-presence flip through the existing authEpochMode/skipLocalCredential seam or another narrow identity fallback while preserving combined-mode invalidation, focused regression coverage, and redacted real-session proof.

## Reproduction Assessment

Yes. Source inspection shows current main hashes a local CLI part plus a profile part, so mocking the local credential reader from null to a credential while the profile stays unchanged changes the epoch and maps to invalidatedReason "auth-epoch".

## Solution Assessment

No. The proposed resolver-level shortcut is broader than the existing backend policy contract; the safer fix should preserve combined semantics and only skip or stabilize the local part for the intended Claude/profile-owned case.
