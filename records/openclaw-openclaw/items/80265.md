## Summary

Keep open: the fix targets a real auth-epoch bug, but this branch broadens the identity-less OAuth skip beyond the reported Claude path and still lacks live session-reuse proof.

## What This Changes

The PR changes CLI auth-epoch skip policy to omit local credential fingerprints for selected identity-less OAuth profiles, bumps the epoch version, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land a scoped policy-seam fix that stabilizes the identity-less Claude CLI file-presence flip without dropping identity-bearing local fingerprints, backed by focused tests and redacted live session-reuse proof.

## Reproduction Assessment

Yes. The linked issue provides deterministic before/after auth-epoch hashes and live reset logs, and current main source shows local/profile epoch parts combine before session reuse invalidates on an epoch mismatch.

## Solution Assessment

No as written. The prepare seam is the right place, but the identity-less branch should be scoped so it does not drop identity-bearing local fingerprints for other CLI backends, and it still needs live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Scope identity-less skips to the affected CLI:** `src/agents/cli-runner/prepare.ts:101-108`
  - body: This branch returns true for any selected OAuth profile with no email/accountId, regardless of which CLI backend is running. For backends such as `google-gemini-cli`, the local credential can expose accountId/email and current combined mode uses that local fingerprint to invalidate on real account changes; this change drops it whenever the selected profile is identity-less. Limit the skip to the Claude identity-less path or pass enough local/backend identity information to avoid hiding identity-bearing local credential changes.
