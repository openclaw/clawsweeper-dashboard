## Summary

Keep this PR open, but it should not merge as-is: the new identity-less OAuth skip is still gated on `preparedExecution`, while the bundled `claude-cli` backend does not provide a `prepareExecution` hook, so the reported runtime path keeps hashing local credential file presence.

## What This Changes

The PR bumps `CLI_AUTH_EPOCH_VERSION`, adds identity-less OAuth skip logic in `shouldSkipLocalCliCredentialEpoch`, adds auth-epoch and prepare tests, and updates the changelog.

## Best Possible Solution

Route the identity-less Claude CLI storage-presence exception through the real auth-epoch decision used by the bundled backend, preserve identity-bearing invalidation, and cover the production prepare/session path with focused regression tests.

## Reproduction Assessment

Yes. Current main source shows local and profile epoch parts are both hashed when `skipLocalCredential` is false, and session reuse invalidates on a versioned epoch mismatch; the linked issue also includes live `reason=auth-epoch` reset logs.

## Solution Assessment

No. The PR uses a plausible policy seam, but its new condition depends on `preparedExecution`, which the bundled `claude-cli` backend does not produce, so the reported runtime path still hashes local file presence.
