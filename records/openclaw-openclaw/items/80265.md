## Summary

Keep open: the reported current-main bug is real, but this PR's resolver-level shortcut is broader than the CLI backend auth-epoch policy and can suppress combined-mode host credential invalidation.

## What This Changes

The PR bumps the CLI auth epoch version, skips local CLI credential fingerprints whenever an auth profile credential exists, updates auth-epoch unit tests, and edits the changelog.

## Best Possible Solution

Stabilize the Claude CLI storage-presence flip through the existing `authEpochMode`/`skipLocalCredential` seam or a narrow Claude identity fallback, while preserving real account/profile invalidation and adding redacted real-session proof.

## Reproduction Assessment

Yes. Source inspection shows current main hashes a local CLI part plus a profile part, so mocking the local credential reader from null to a credential while the profile stays unchanged changes the epoch and then maps to `invalidatedReason: "auth-epoch"`.

## Solution Assessment

No. The proposed resolver-level shortcut is broader than the backend policy contract; the safer fix should preserve `combined` semantics and only skip or stabilize the local part for the intended Claude/profile-owned case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve combined auth-epoch mode:** `src/agents/cli-auth-epoch.ts:191`
