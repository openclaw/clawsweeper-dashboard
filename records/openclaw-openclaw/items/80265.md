Keep this PR open but do not merge it as-is: the current head intentionally leaves the identity-less Claude CLI file-presence epoch flip in place while still bumping the epoch version and adding a fix changelog entry.

## What This Changes

The PR bumps `CLI_AUTH_EPOCH_VERSION`, adds a regression test that documents the identity-less Claude CLI file-presence epoch flip, and adds a changelog entry claiming the fix.

## Best Possible Solution

Land a maintainer-approved auth-epoch ownership change at the prepare/backend policy seam, or another explicit narrow seam, with a regression that expects equality for identity-less Claude file-presence flips and still invalidates real account changes.

## Reproduction Assessment

Yes. Source inspection shows the current resolver hashes different part lists when the local Claude credential reader changes from null to present while the auth profile is unchanged, and session reuse maps that mismatch to `invalidatedReason: "auth-epoch"`.

## Solution Assessment

No. The current PR is not the best fix because it keeps the buggy resolver behavior while bumping the epoch version and publishing a fix note; the fix needs an explicit auth-epoch ownership policy plus a stable-epoch assertion.
