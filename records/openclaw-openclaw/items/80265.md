## Summary

Keep open: this targets a real session-continuity bug, but the current patch does not affect the bundled Claude CLI path because the new skip still depends on `preparedExecution`, which that backend does not provide. The PR also needs real Claude CLI prepare/session reuse proof before merge.

## What This Changes

The PR changes CLI auth-epoch policy, tests, and changelog so identity-less OAuth auth profiles can skip local CLI credential fingerprints when execution is prepared, and bumps the auth-epoch version.

## Best Possible Solution

Fix the auth-epoch decision at the path actually used by `claude-cli`, preserving identity-bearing invalidation and bootstrap behavior, with focused regression tests and redacted live session-reuse proof.

## Reproduction Assessment

Yes. Current-main source shows local and profile epoch parts are both hashed when `skipLocalCredential` is false, and the linked issue includes live `reason=auth-epoch` reset logs for the Claude CLI path.

## Solution Assessment

No. The proposed policy seam is plausible, but this implementation is still gated on `preparedExecution`, so it does not affect the bundled `claude-cli` backend path that reported the resets.
