## Summary

Keep this PR open: the reported bug is valid, but this patch does not affect the bundled `claude-cli` path because the new identity-less OAuth skip only runs when `preparedExecution` is present, and the current Anthropic CLI backend does not provide `prepareExecution`.

## What This Changes

The PR adds an identity-less OAuth branch to `shouldSkipLocalCliCredentialEpoch`, bumps the CLI auth epoch version, and adds auth-epoch/prepare tests plus a changelog entry.

## Best Possible Solution

Route the identity-less Claude CLI storage-presence exception through the real `claude-cli` prepare/session path, preserve invalidation for identity-bearing changes, and cover that production path with a focused regression test.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main hashes the local `claude-cli` fingerprint when `skipLocalCredential` is false and then invalidates reusable sessions on an auth-epoch mismatch; I did not run tests because this review is read-only.

## Solution Assessment

No. The PR uses the right policy seam in shape, but the new condition is gated on `preparedExecution`, which the bundled `claude-cli` backend does not produce, so the reported session reset path still hashes local file presence.
