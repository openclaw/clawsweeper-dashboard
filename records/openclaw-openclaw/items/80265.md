## Summary

Keep open: the reported bug is source-reproducible, but this PR broadens profile-credential precedence beyond the documented combined auth-epoch contract and still lacks real after-fix OpenClaw CLI session proof.

## What This Changes

The PR makes auth-profile credentials suppress local CLI credential hashing, bumps `CLI_AUTH_EPOCH_VERSION`, updates auth-epoch tests, and adds a changelog entry.

## Best Possible Solution

Stabilize the Claude CLI local-storage presence flip through a narrow backend-owned or profile-owned auth-epoch path while preserving combined host-plus-profile invalidation semantics.

## Reproduction Assessment

Yes. Current main hashes both local and profile epoch parts, so mocking the local Claude credential reader from `null` to an OAuth credential while the profile stays stable changes the epoch, and `resolveCliSessionReuse` maps that mismatch to `auth-epoch`; I did not run a live gateway flow.

## Solution Assessment

No. The resolver-level `!profileCredential` shortcut is broader than the backend contract; the safer fix should preserve `combined` semantics and only drop local credentials through the existing `authEpochMode`/`skipLocalCredential` seam or an equally narrow Claude-owned policy.
