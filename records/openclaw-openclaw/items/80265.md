## Summary

Keep this PR open for revision: the patch still gates the new identity-less OAuth skip on `preparedExecution`, but the bundled `claude-cli` backend does not provide that hook, so the reported production `auth-epoch` reset path remains unfixed.

## What This Changes

This PR adds identity-less OAuth skip logic to `shouldSkipLocalCliCredentialEpoch`, bumps `CLI_AUTH_EPOCH_VERSION`, adds auth-epoch and prepare tests, and updates the changelog.

## Best Possible Solution

Route the identity-less Claude CLI storage-presence exception through the real auth-epoch decision used by the bundled backend, preserve identity-bearing invalidation, and cover the production prepare/session path with focused regression tests.

## Reproduction Assessment

Yes. Current main source shows local and profile epoch parts are both hashed when `skipLocalCredential` is false, and session reuse invalidates on a versioned epoch mismatch; the linked issue also provides live `reason=auth-epoch` reset logs.

## Solution Assessment

No. The PR uses a plausible policy seam, but the new condition is gated on `preparedExecution`, which the bundled `claude-cli` backend does not produce, so the reported runtime path still hashes local file presence.
