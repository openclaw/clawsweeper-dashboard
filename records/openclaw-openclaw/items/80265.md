## Summary

Keep this PR open, but do not merge it as-is: the proposed skip logic is still gated on a prepared execution hook that the bundled Claude CLI backend does not provide, so the reported runtime path remains unfixed.

## What This Changes

The PR changes CLI auth-epoch policy/tests/changelog so identity-less OAuth auth profiles can skip local CLI credential fingerprints when execution is prepared and bumps the auth-epoch version.

## Best Possible Solution

Route the identity-less Claude CLI storage-presence exception through the auth-epoch decision actually used by the bundled backend, preserving real identity-bearing invalidation and bootstrap behavior with focused regression coverage and redacted live runtime proof.

## Reproduction Assessment

Yes. Current-main source shows local and profile epoch parts are both hashed when `skipLocalCredential` is false, and the linked issue provides live `reason=auth-epoch` reset logs for the same path.

## Solution Assessment

No. The proposed policy seam is plausible, but this implementation is still gated on `preparedExecution`, so it does not affect the bundled `claude-cli` backend path that reported the resets.
