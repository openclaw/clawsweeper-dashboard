  - body: This branch emits the replacement block reply and suppresses chunks, but it does not mark `lastBlockReplyText` or `lastDeliveredBlockReplyText`. A later `message_end` safety send can see no prior delivery and emit the same guarded replacement again, so record the replacement as handled like normal block chunks.
  - confidence: 0.82

  - body: The prefilter recognizes `accepted/queued`, but the start regex only finds `Sent to` or SMS/text-message `sent`, `queued`, or `delivered`. Text such as `SMS was accepted/queued. Message ID: ...` can therefore produce no claims and bypass the guard, leaving a common provider receipt form unprotected.
  - confidence: 0.85
