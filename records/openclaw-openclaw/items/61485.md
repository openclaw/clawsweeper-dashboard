## Summary

Keep open: the feature direction is useful and the PR has real runtime proof, but the latest head still regresses current main hook contracts and needs maintainer API/security review before merge.

## What This Changes

The PR changes plugin `llm_input` and `llm_output` from observe-only hooks into modifying hooks that can block or rewrite model input and replace assistant output, with embedded-runner, registry, auto-reply, type, and test updates.

## Best Possible Solution

Land a rebased, maintainer-approved writable LLM hook contract that preserves current hook timeouts, prompt-injection and conversation-access gates, the `tools` payload, LLM-boundary cloning, Google prompt-cache ordering, public docs, and context-hook coordination.
