## Summary

Keep this PR open: current main still has observe-only `llm_input`/`llm_output` hooks, but this branch is merge-conflicting and still needs rebasing plus security hardening before it is a safe public plugin contract.

## What This Changes

The PR changes `llm_input` and `llm_output` from fire-and-forget plugin hooks into awaited modifying hooks for blocking LLM calls, rewriting prompts/system prompts, and replacing assistant text.

## Best Possible Solution

Land a rebased, maintainer-approved hook contract that runs before prompt assembly, prompt-cache wrapping, image detection, and provider submission; preserves current guarded hooks; deep-clones hook payloads; and updates SDK docs/tests.

## Reproduction Assessment

Yes for the PR defects: source inspection shows current main prepares `promptForModel` and Google prompt cache before the existing hook location, while the PR head mutates the older `effectivePrompt` path and shallow-clones history. This is a feature PR, not a user bug reproduction.

## Solution Assessment

No. The feature direction is useful and the logs are meaningful, but the current branch is not the best merge shape until it is rebased onto current prompt assembly/cache behavior and hardened at the plugin trust boundary.
