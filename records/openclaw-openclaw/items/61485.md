  - body: This direct await path bypasses current main's `getModifyingHookTimeoutMs` and `withHookTimeout` handling for modifying hooks. A slow or hung `llm_input` or `llm_output` handler can stall the model turn beyond configured hook budgets.

  - body: The branch builds the Google prompt-cache wrapper before applying `llm_input` prompt and systemPrompt overrides. For cache-eligible Google runs, a plugin can believe it rewrote model input while the cached provider-visible system prompt path still uses pre-hook state.
  - confidence: 0.9
- **[P1] Keep prompt-injection guards additive:** `src/plugins/hook-types.ts:130-134`
  - body: This guarded-hook list is stale against current main and drops existing prompt-mutating hooks such as `agent_turn_prepare` and `heartbeat_prompt_contribution`. Add `llm_input` without removing those entries so `allowPromptInjection=false` keeps its current coverage.
  - confidence: 0.92
- **[P1] Use the LLM-boundary history clone:** `src/agents/pi-embedded-runner/run/attempt.ts:2472-2475`
  - body: This shallow message copy bypasses current main's `normalizeMessagesForLlmBoundary` plus structured clone helper. A hook can still mutate nested content or observe runtime-only details outside the declared result contract.
  - confidence: 0.89

  - body: When a higher-priority hook rewrites the prompt, later `llm_input` handlers see the evolved prompt but keep the original `imagesCount`. Policy hooks that gate on image presence can make an allow/block decision using stale image metadata.
  - confidence: 0.8

  - body: These result types change the public plugin API, but the docs still classify `llm_input` and `llm_output` as observation-only. Document return shapes, ordering, failure behavior, and prompt-injection policy before exposing the contract.
