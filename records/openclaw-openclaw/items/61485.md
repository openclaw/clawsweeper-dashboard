Full review comments:

- **[P1] Apply input rewrites before cache and image setup:** `src/agents/embedded-agent-runner/run/attempt.ts:4145-4148`
  - body: This applies the `llm_input` prompt/systemPrompt result after the Google prompt-cache wrapper and image detection have already been prepared from the old values. A system-prompt policy rewrite can be ignored by cachedContent, and a prompt rewrite can leave stale prompt-local images attached; run the hook before those dependent steps or rebuild them from the effective prompt/system prompt.
  - confidence: 0.9
- **[P2] Use output rewrites before terminal decisions:** `src/agents/embedded-agent-runner/run/attempt.ts:5141`
  - body: The override is only substituted in the returned attempt object, after `emptyAssistantReplyIsSilent`, `resolveTerminalAssistantTexts`, trajectory events, and artifact metadata have already used the original assistant text. A plugin that suppresses or redacts output can therefore leave stale terminal state or be misclassified; compute an effective assistant text list immediately after the hook and use it for all downstream decisions.
  - confidence: 0.87
- **[P1] Keep modifying hook semantics consistent across runners:** `src/plugins/hook-types.ts:1042-1050`
  - body: The public handler types now advertise `llm_input`/`llm_output` return values, but the CLI/Codex harness path still dispatches those hooks best-effort and discards results in `src/agents/harness/lifecycle-hook-helpers.ts`. Plugin authors would get block/rewrite/redact behavior in embedded runs but observe-only behavior in CLI/Codex runs, so either update the harness path or explicitly keep the public contract scoped with docs and tests.
  - confidence: 0.88
- **[P1] Budget the newly awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: `runLlmInput` and `runLlmOutput` now use `runModifyingHook`, and the embedded runner awaits them on the request path, but the default modifying-hook timeout table still does not include these hook names. A plugin handler that never resolves can now stall model submission or response delivery unless every registration provides its own timeout; add sane default budgets for these awaited hooks.
  - confidence: 0.86
