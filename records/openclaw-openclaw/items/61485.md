## Summary

Keep open: current main still leaves `llm_input` and `llm_output` observe-only, and the contributor supplied useful live logs, but this branch is not mergeable as-is because it is stale against current prompt assembly and would weaken existing prompt-injection hook policy.

## What This Changes

The branch changes plugin hook types, hook-runner behavior, embedded-runner and auto-reply handling, and tests so `llm_input`/`llm_output` can block, rewrite prompts/system prompts, and replace assistant text.

## Best Possible Solution

Land a maintainer-approved, rebased hook contract that applies modifications before prompt assembly, prompt-cache wrapping, image detection, and provider submission; preserves all existing prompt-injection guards; documents SDK semantics; and proves the submitted provider payload uses the modified values.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register an `llm_input` hook that rewrites prompt/system prompt, then verify the embedded runner submits the rewritten values under current prompt-submission and Google prompt-cache paths.

## Solution Assessment

No. The feature direction is plausible, and the contributor supplied useful live logs, but this branch is not the best merge shape until it is rebased around current prompt assembly/cache state and preserves all existing prompt-injection guarded hooks.
