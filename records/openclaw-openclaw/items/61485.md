## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep modifying hook semantics consistent across runners:** `src/plugins/hook-types.ts:1048-1052`
  - body: The handler map now advertises modifying results for `llm_input` and `llm_output`, but CLI and Codex/harness paths still invoke these hooks as best-effort observations and discard returned values. Plugin authors would see block/rewrite/redact work in embedded runs but silently do nothing in other supported runtimes, creating a split public API contract.
  - confidence: 0.92
- **[P1] Apply systemPrompt overrides to the model context:** `src/agents/embedded-agent-runner/run/attempt.ts:4148`
  - body: Assigning `llmInputResult.systemPrompt` only updates `systemPromptForHook`, which is used for diagnostics/precheck. The actual model call goes through `promptActiveSession(...)`, and Agent Core reads the system prompt from the active session state, so a plugin's system-prompt rewrite is not submitted to the provider.
  - confidence: 0.94
- **[P1] Recompute prompt-local images after prompt overrides:** `src/agents/embedded-agent-runner/run/attempt.ts:4145`
  - body: The image set is already loaded from the pre-hook prompt before this prompt override is applied. If a policy plugin removes an image reference, the old `imageResult.images` can still be sent; if it adds a reference, the image is never loaded, so prompt rewrites can leak or omit prompt-local media.
  - confidence: 0.91
- **[P1] Redact transcript and downstream state when output changes:** `src/agents/embedded-agent-runner/run/attempt.ts:5141`
  - body: This only swaps the returned `assistantTexts`; the original assistant remains in `messagesSnapshot`, `lastAssistant`, `currentAttemptAssistant`, and the earlier `agent_end` side effect. A redaction plugin can therefore change the delivered text while the sensitive original still persists into transcript/state and later context.
  - confidence: 0.93
- **[P1] Budget the newly awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: `runLlmInput` and `runLlmOutput` now use the modifying-hook path, but the default modifying-hook timeout map still has no entries for these two names. Without an explicit plugin timeout, a hung handler can block model submission or response delivery indefinitely, which is a regression from the previous fire-and-forget behavior.
  - confidence: 0.86
