## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Add timeouts before awaiting LLM hooks:** `src/agents/embedded-agent-runner/run/attempt.ts:4111`
  - body: This changes `llm_input`/`llm_output` from fire-and-forget observers into awaited modifying hooks, but the default modifying-hook timeout map still has no entries for either hook name. A plugin handler that hangs will now stall the agent turn indefinitely instead of merely running in the background, so these awaited hooks need bounded defaults and coverage before merge.
  - confidence: 0.91
- **[P1] Apply system prompts to the active model call:** `src/agents/embedded-agent-runner/run/attempt.ts:4147`
  - body: Assigning `llmInputResult.systemPrompt` only updates the local `systemPromptForHook` variable used by diagnostics/prechecks; it does not call `setActiveSessionSystemPrompt` or rebuild the prompt-cache wrapper that was prepared with `systemPromptText`. A plugin can appear to enforce a compliance system-prompt rewrite while the provider still receives the original system prompt.
  - confidence: 0.88
- **[P1] Rewrite the canonical assistant state before redaction:** `src/agents/embedded-agent-runner/run/attempt.ts:5138`
  - body: Returning `llmOutputAssistantTextsOverride ?? assistantTexts` changes only the attempt text payload, while `messagesSnapshot`, `lastAssistant`, `currentAttemptAssistant`, `agent_end`, and future context continue to carry the original assistant message. That defeats the advertised redaction use case because sensitive model output can still leak through transcript-backed consumers after the visible reply is replaced.
  - confidence: 0.87
- **[P2] Redetect prompt images after prompt rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4144`
  - body: When `llm_input` returns a replacement prompt, this only assigns `promptForModel`; the image loader still uses the pre-hook `promptSubmission.prompt`. A plugin that removes image references will still send the old images, and a plugin that adds image references will not load them, so prompt overrides are not applied consistently to the actual model payload.
  - confidence: 0.8
