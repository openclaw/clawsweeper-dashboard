## Summary

## What This Changes

The PR changes `llm_input` and `llm_output` from fire-and-forget observation hooks into awaited modifying hooks that can block LLM calls, rewrite prompt/system prompt input, and replace assistant output text.

## Best Possible Solution

Land only a maintainer-approved modifying-hook contract that updates provider-visible prompt/system-prompt paths, image/precheck/terminal behavior, public docs, and focused tests before plugin authors depend on it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Apply system prompt overrides to the provider path:** `src/agents/embedded-agent-runner/run/attempt.ts:4593-4595`
  - body: Assigning `llmInputResult.systemPrompt` only to `systemPromptForHook` updates diagnostics/prechecks, but the active session still uses `systemPromptText` and the Google prompt-cache wrapper is prepared with the old prompt. A plugin that returns `systemPrompt` will appear accepted while the provider can still receive the original system prompt.
  - confidence: 0.9
- **[P1] Rebuild prompt images after prompt rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4590-4592`
  - body: The PR applies `llmInputResult.prompt` after `detectAndLoadPromptImages` has already loaded images from `promptSubmission.prompt`. If a plugin adds or removes image references, the model receives stale image payloads for the rewritten prompt, which breaks the advertised prompt-rewrite contract.
  - confidence: 0.88
- **[P2] Preserve plugin-suppressed output through live-switch checks:** `src/agents/embedded-agent-runner/run/attempt.ts:5138`
  - body: Returning an `llm_output` override through `assistantTexts` alone makes an intentional `assistantTexts: []` indistinguishable from a model that produced no output. The live model switch guard restarts attempts when `attempt.assistantTexts` is empty, so a plugin trying to suppress output can be bypassed by a pending model switch unless the attempt records that output was plugin-modified.
  - confidence: 0.82
- **[P2] Add default timeout budgets for awaited LLM hooks:** `src/plugins/hooks.ts:957-1008`
  - body: This changes `llm_input` and `llm_output` from fire-and-forget to awaited modifying hooks, but the default modifying-hook timeout table does not include either hook. A plugin handler that hangs can now stall the agent turn by default, which is a regression from the previous observation-only path.
  - confidence: 0.84
- **[P2] Document the new modifying hook contract:** `src/plugins/hook-types.ts:277-365`
  - body: The PR expands the public plugin API and changes `allowPromptInjection=false` semantics for `llm_input`, but the public hook and configuration docs still describe `llm_input`/`llm_output` as observation hooks. Plugin authors would not have a stable documented contract for return fields, ordering, prompt-injection filtering, or failure behavior.
  - confidence: 0.86
