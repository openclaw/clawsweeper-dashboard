## Summary

Keep open. Current main still has observe-only LLM hooks, while this PR adds a real plugin API/security surface and still has P1 source-level blockers around provider cache ordering, prompt-injection policy preservation, and payload isolation.

## What This Changes

The PR changes `llm_input` and `llm_output` plugin hooks from observe-only callbacks to modifying hooks that can block or rewrite model input and replace assistant output.

## Best Possible Solution

Land a maintainer-approved modifying LLM hook contract that runs before provider-visible prompt/cache setup, preserves current prompt-injection and payload-isolation contracts, and documents SDK semantics.

## Reproduction Assessment

Yes. Source inspection against PR head `d06cd5f688b207a3ae40a9a084c5ac83b5392c8d` and current main `d68d8e6b2d23c0ec90ae6005f39dfa2b8200a374` gives high-confidence reproduction paths for the review findings.

## Solution Assessment

No. The feature direction is plausible, but this branch should be rebased and reshaped around current prompt assembly, guard lists, cloning helpers, and docs before merge.
