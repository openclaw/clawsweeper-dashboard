## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve modifying-hook timeouts:** `src/plugins/hooks.ts:374-376`
  - body: This path awaits each modifying handler directly, while current main wraps modifying hooks with `getModifyingHookTimeoutMs` and `withHookTimeout`. A slow or hung `llm_input` handler would bypass configured hook budgets and stall model submission.
  - confidence: 0.9
- **[P1] Apply llm_input changes before prompt-cache setup:** `src/agents/pi-embedded-runner/run/attempt.ts:2507-2514`
  - body: Google prompt-cache wrapping is prepared earlier with the pre-hook system prompt, then this block applies `prompt` and `systemPrompt` overrides. In cache-eligible Google runs, plugin rewrites can miss provider-visible cached input or silently not affect the call.
  - confidence: 0.9
- **[P1] Keep prompt-injection guards additive:** `src/plugins/hook-types.ts:130-134`
  - body: This guarded hook list adds `llm_input` but drops current-main prompt-mutating hooks such as `agent_turn_prepare` and `heartbeat_prompt_contribution`. Merging it would weaken `allowPromptInjection=false` for existing hooks.
  - confidence: 0.92
- **[P1] Use the LLM-boundary clone for hook history:** `src/agents/pi-embedded-runner/run/attempt.ts:2472-2475`
  - body: The new `llm_input` payload shallow-copies `activeSession.messages` and skips current main's LLM-boundary normalization and structured clone. Hook code can mutate nested message content or observe runtime-only details outside the declared result contract.
  - confidence: 0.88
- **[P2] Keep image counts aligned for evolved input events:** `src/plugins/hooks.ts:638-646`
  - body: After a higher-priority hook rewrites `prompt`, lower-priority hooks see the evolved prompt/system prompt but the original `imagesCount`. Policy hooks that gate on image presence can make the wrong decision before image re-detection runs.
  - confidence: 0.8
- **[P2] Document the modifying LLM hook contract:** `src/plugins/hook-types.ts:191-199`
  - body: These result types expose a new public plugin API, but the public docs still categorize `llm_input` and `llm_output` as observation-only. Plugin authors need the return shapes, ordering, failure behavior, and `allowPromptInjection=false` semantics before this ships.
  - confidence: 0.82
