## Summary

Keep open: current main still treats `llm_input` and `llm_output` as observe-only hooks, but this PR is a public, security-sensitive plugin API change with concrete correctness gaps against the current runner prompt/cache flow and prompt-injection guard list.

## What This Changes

The branch changes plugin hook types, hook-runner behavior, embedded-runner and auto-reply handling, and tests so `llm_input`/`llm_output` can block, rewrite prompts/system prompts, and replace assistant text.

## Best Possible Solution

Land a maintainer-approved, rebased hook contract that applies modifications before prompt assembly, prompt-cache wrapping, image detection, and provider submission; preserves prompt-injection policy; documents SDK semantics; and proves the embedded-runner call uses the modified values.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register an `llm_input` hook that rewrites prompt/system prompt, then verify the embedded runner submits the rewritten values under current prompt-submission and Google prompt-cache paths.

## Solution Assessment

No. The feature direction is plausible, and the contributor supplied useful live logs, but this branch is not the best merge shape until the hook runs before current prompt assembly/cache state and preserves all existing prompt-injection guarded hooks.
