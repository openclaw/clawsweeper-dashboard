## Summary

Keep open: the feature direction is useful and the PR includes real runtime proof, but the current branch is stale against main and would drop recent hook timeout, policy, payload-isolation, cache-ordering, and context-metadata contracts.

## What This Changes

The PR changes `llm_input` and `llm_output` plugin hooks from observe-only hooks into modifying hooks that can block or rewrite model input and replace assistant output.

## Best Possible Solution

Land a rebased, maintainer-approved writable LLM hook contract that preserves current timeouts, access-policy gates, payload isolation, prompt-cache ordering, context metadata, docs, and regression tests.

## Reproduction Assessment

Yes for the review findings: comparing PR head `d06cd5f688b207a3ae40a9a084c5ac83b5392c8d` with current main shows deterministic contract regressions. This is not a current-main bug reproduction because the central request is a new plugin API feature.

## Solution Assessment

No. The direction is plausible, but this branch is not the best merge shape until it is rebased and repaired to preserve current hook safety, metadata, cache, and payload contracts.
