- **[P1] Apply system-prompt overrides to the active model call:** `src/agents/embedded-agent-runner/run/attempt.ts:4148`
  - body: `llm_input` stores a returned `systemPrompt` in `systemPromptForHook`, but the session system prompt was already installed earlier and `promptActiveSession` later submits `promptForSession`. This makes the override affect diagnostics/prechecks rather than the actual model call.
  - confidence: 0.9
- **[P2] Redetect prompt images after prompt rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4145`
  - body: Images are loaded from the original `promptSubmission.prompt` before `llm_input`. If a hook rewrites `promptForModel`, the old `imageResult.images` are still sent, so removed image references can leak and newly added image references are ignored.
  - confidence: 0.88
- **[P1] Update canonical assistant state for output overrides:** `src/agents/embedded-agent-runner/run/attempt.ts:5141`
  - body: Returning replacement `assistantTexts` is not enough because delivery and follow-up logic can prefer `currentAssistant`/`lastAssistant` canonical text. Successful turns can still expose or retain the original unredacted output.
  - confidence: 0.92
