## Summary

Keep this PR open, but it is not merge-ready: the idea is useful, yet the current branch changes a public plugin hook contract without carrying the new modifying semantics through sibling runners, and the embedded runner still has prompt/image/cache/output-state security and availability gaps.

## What This Changes

This PR changes `llm_input` and `llm_output` plugin hooks from observation-only callbacks into awaited modifying hooks that can block LLM calls, rewrite prompts/system prompts, and replace assistant output text.

## Best Possible Solution

Hold this PR for maintainer API direction, then either implement one consistent modifying-hook contract across embedded, CLI, and Codex-supported surfaces with security/timeout tests and current-head proof, or keep these hooks observation-only and use existing blocking/finalization hooks for policy work.

- **[P1] Keep modifying hook semantics consistent across runners:** `src/plugins/hook-types.ts:1045-1052`
  - body: The handler map now advertises result-bearing `llm_input`/`llm_output` hooks, but current CLI and Codex harness paths still dispatch those hooks as fire-and-forget observations and ignore returned block/rewrite/redaction results. That creates a public plugin API that works in embedded runs but silently does nothing in sibling runtimes; either keep the hooks observe-only or implement/document a consistent supported contract across those callers.
  - confidence: 0.89
- **[P1] Apply prompt rewrites before cache and image setup:** `src/agents/embedded-agent-runner/run/attempt.ts:4144-4148`
  - body: `llm_input` applies `prompt` and `systemPrompt` overrides here after `prepareGooglePromptCacheStreamFn` and `detectAndLoadPromptImages` already ran with the original values. In Google prompt-cache eligible runs the system-prompt rewrite can be ignored by the wrapper, and prompt rewrites that add/remove image references leave the model with stale prompt-local images; move the hook before those steps or rebuild them from the effective prompt/system prompt.
  - confidence: 0.91
- **[P1] Redact downstream assistant state when output changes:** `src/agents/embedded-agent-runner/run/attempt.ts:5141-5144`
  - body: The override is only substituted in the returned `assistantTexts`, after the method has already classified terminal state and recorded trajectory artifacts from the original assistant text, and the returned `lastAssistant` is still the original object. A redaction plugin can therefore change the visible payload while original content/reasoning/error metadata remains in downstream state; derive final assistant text and lastAssistant before terminal classification and traces.
  - confidence: 0.88
- **[P1] Budget newly awaited LLM hooks:** `src/plugins/hooks.ts:967-1003`
  - body: This turns `llm_input`/`llm_output` into awaited modifying hooks, but those hook names still have no default `modifyingHookTimeoutMsByHook` budget. An installed plugin with no per-hook timeout can now stall model submission or response delivery where the previous fire-and-forget observer would not block the turn; add bounded defaults or keep these hooks non-blocking.
  - confidence: 0.86
