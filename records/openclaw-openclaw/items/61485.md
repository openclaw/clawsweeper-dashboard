## Summary

Keep open: this is a useful plugin API feature, but the current PR is not merge-ready because the modifying hooks are applied after cache/image setup, output overrides are not made canonical early enough, docs/tests are missing, and the branch is currently dirty against main.

## What This Changes

This PR changes the plugin `llm_input` and `llm_output` hooks from observe-only void hooks into awaited modifying hooks that can block model calls, rewrite prompts/system prompts, and replace assistant texts.

## Best Possible Solution

Land this only after maintainers confirm the plugin API direction and the rebased branch makes prompt, system prompt, image, and output overrides the single effective model-call contract with docs, focused tests, and current-head real behavior proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review blockers: source inspection shows cache setup and image detection happen before the `llm_input` mutation is applied, and output replacement is applied only at the returned attempt result.

## Solution Assessment

Is this the best way to solve the issue? No; the API idea is plausible, but the implementation needs to compute one effective prompt/system/images/output state before downstream cache, delivery, terminal, and docs contracts observe it.
