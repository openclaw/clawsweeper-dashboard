## Summary

Keep open. Current main still has observer-only LLM hooks, and this PR has useful real-run proof, but the branch still needs maintainer review and rebase work before merge because it expands plugin authority over model IO and has blocking prompt/cache and hook-policy issues against current main.

## What This Changes

The PR changes plugin `llm_input` and `llm_output` hooks from observer-only callbacks into modifying hooks that can block calls, rewrite prompts/system prompts, and replace assistant text.

## Best Possible Solution

Land a maintainer-approved, rebased hook contract that runs `llm_input` modifications before provider-visible prompt/cache construction, preserves all current prompt-injection guards, deep-clones hook payloads, and documents SDK semantics.

## Reproduction Assessment

Yes. Source inspection shows current main builds `promptForModel` and prepares Google prompt-cache before `llm_input`, while the PR head applies prompt/system overrides at an older hook site after cache setup.

## Solution Assessment

No. The feature direction is plausible and the supplied logs are useful, but this branch is not the best merge shape until it is rebased onto current prompt assembly/cache behavior and preserves current prompt-injection policy coverage.
