- **[P1] Apply systemPrompt overrides to the provider path:** `src/agents/embedded-agent-runner/run/attempt.ts:4148`
  - body: `llm_input` can now return `systemPrompt`, but this only assigns `systemPromptForHook`.   - confidence: 0.9

  - body: The replacement is stored in `llmOutputAssistantTextsOverride`, but silence handling, terminal trajectory events, and trace artifacts below still read the original `assistantTexts` until the final return. That can leak content a plugin intended to redact and make terminal decisions inconsistent with the delivered text.
  - confidence: 0.88
- **[P1] Recompute image inputs after prompt rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4145`
  - body: A prompt override is applied after `imageResult` was built from the original prompt, and the later provider call still sends that stale image set. A plugin that adds or removes image references can therefore send the wrong images to the model or leave downstream hook decisions based on stale `imagesCount`.
  - confidence: 0.86
- **[P2] Add default budgets for awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: This switches `llm_input` and `llm_output` from fire-and-forget observation to awaited modifying hooks, but they are not added to the default modifying-hook timeout map. A hung plugin can now block the agent turn unless every plugin/operator supplies a timeout manually.
  - confidence: 0.84
- **[P2] Document the new modifying hook contract:** `src/plugins/hook-types.ts:280-288`
  - body: The PR changes a public plugin API, but current docs still say `llm_input` and `llm_output` only observe provider input/output and the PR changes no docs. Plugin authors need the supported return fields, ordering, timeout, and `allowPromptInjection` behavior documented with the contract change.
  - confidence: 0.82
