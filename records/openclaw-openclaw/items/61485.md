- **[P1] Rebuild model inputs after llm_input rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4141-4144`
  - body: The returned `prompt`/`systemPrompt` are assigned after the runner has already prepared prompt-cache state, loaded prompt-local images, recorded compiled context, and computed the normalized boundary prompt used for prechecks. A system-prompt override is also not written back to the active session before `promptActiveSession`, so a policy plugin can appear to rewrite the call while the provider path still uses stale inputs.
  - confidence: 0.9
- **[P1] Apply llm_output replacements before downstream consumers:** `src/agents/embedded-agent-runner/run/attempt.ts:5138`
  - body: This substitution only changes the returned attempt payload after context maintenance, `agent_end`, terminal classification, and trajectory artifacts have already consumed `assistantTexts`, `messagesSnapshot`, and `lastAssistant`. A redaction plugin can therefore leave the original assistant content in context/diagnostics even though the user-facing payload is replaced.
  - confidence: 0.88
- **[P2] Budget awaited LLM hooks by default:** `src/plugins/hooks.ts:965`
  - body: Converting these hooks to `runModifyingHook` puts every handler on the agent-turn critical path, but the default modifying-hook timeout map does not include `llm_input` or `llm_output`. Without built-in budgets, a slow or hung plugin can stall every model call unless each plugin author remembers to configure a timeout.
  - confidence: 0.82
