## Summary

Keep open: the feature is still not implemented on current main, but this PR changes public LLM hook contracts without making the new block/rewrite/redact semantics consistent across embedded, CLI, and Codex harness paths, and the embedded output/system-prompt handling still has security-relevant correctness gaps.

## What This Changes

The PR changes `llm_input` and `llm_output` from observation-only plugin hooks into awaited modifying hooks that can block LLM calls, rewrite prompt/system prompt, and replace assistant text in the embedded runner.

## Best Possible Solution

Before merge, either make the modifying contract consistently work across embedded, CLI, and Codex harness runtimes or introduce a separately named embedded-only API, then apply prompt/output overrides canonically, add default hook budgets, docs, contract tests, and current-head real behavior proof.

  - body: This public type change tells plugin authors that `llm_input` and `llm_output` results can block, rewrite, or redact LLM traffic, but the CLI and Codex harness emitters still call the shared helper as fire-and-forget and ignore the returned result.   - confidence: 0.92
- **[P1] Apply output replacements before terminal state:** `src/agents/embedded-agent-runner/run/attempt.ts:5141`
  - body: `llm_output` replacements are only swapped into the returned `assistantTexts` here, after empty/silent detection, terminal trajectory, model.completed artifacts, and trace artifacts have already used the raw model text. Redaction or suppression plugins can therefore still leave the original output in state/telemetry decisions, and empty suppression can be misclassified as a model-empty result.
  - confidence: 0.9
- **[P1] Submit the overridden system prompt to the model:** `src/agents/embedded-agent-runner/run/attempt.ts:4147-4149`
  - body: The `systemPrompt` hook result is assigned only to `systemPromptForHook`, which is later used for prechecks and trajectory fields, while the actual session prompt submission still uses `promptForSession` plus a model-prompt transform that has no replacement system prompt. A plugin policy can appear to rewrite the system prompt without changing what the model receives.
  - confidence: 0.84
- **[P2] Budget the awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: Moving these hooks from parallel observation to awaited `runModifyingHook` means a slow or hung plugin now blocks model submission or response delivery. The default modifying-hook timeout table still has no entries for `llm_input` or `llm_output`, so a single unbudgeted handler can stall the agent turn.
  - confidence: 0.89
