Overall confidence: 0.9

Full review comments:

- **[P1] Preserve modifying-hook timeouts:** `src/plugins/hooks.ts:374-376`
  - body: This `runModifyingHook` path now awaits each handler directly, while current main wraps modifying hooks with `getModifyingHookTimeoutMs` and `withHookTimeout`. A slow or hung modifying hook, including the new `llm_input` path, can bypass configured hook budgets and stall the caller.
  - confidence: 0.88
- **[P1] Move llm_input mutations before Google cache setup:** `src/agents/pi-embedded-runner/run/attempt.ts:2507-2514`
  - body: The PR prepares the Google prompt-cache stream before awaiting `llm_input`, then applies `prompt` and `systemPrompt` overrides afterward. On prompt-cache eligible Google runs, a plugin can believe it rewrote model input while the provider-visible cached prompt path still uses pre-hook state.
  - confidence: 0.9
- **[P1] Keep prompt-injection guards additive:** `src/plugins/hook-types.ts:130-134`
  - body: This guarded-hook list is stale against current main and drops existing prompt-mutating hooks such as `agent_turn_prepare` and `heartbeat_prompt_contribution`. Add `llm_input` without removing those protections so `allowPromptInjection=false` keeps the current policy coverage.
  - confidence: 0.92
- **[P1] Use the LLM-boundary clone for hook history:** `src/agents/pi-embedded-runner/run/attempt.ts:2472-2475`
  - body: `activeSession.messages.map((m) => ({ ...m }))` only shallow-copies each message and bypasses current main's LLM-boundary normalization. A hook can mutate nested content or observe runtime-only details outside the explicit result contract; keep `cloneHookMessages(normalizeMessagesForLlmBoundary(...))` for this payload.
  - confidence: 0.89
- **[P2] Keep imagesCount aligned for evolved llm_input events:** `src/plugins/hooks.ts:638-646`
  - body: When a higher-priority hook rewrites `prompt`, the evolved event updates only `prompt` and `systemPrompt`; later hooks still see the original `imagesCount`. Policy hooks that depend on image presence can make the wrong allow/block decision before image re-detection runs.
  - confidence: 0.8
- **[P2] Document the modifying LLM hook contract:** `src/plugins/hook-types.ts:191-199`
  - body: These new result types change the public plugin API, but the public docs still describe `llm_input` and `llm_output` as observation-only hooks. Document the return shapes, ordering, failure behavior, and `allowPromptInjection=false` semantics before exposing the contract.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch expands plugin control over model input/output and leaves concrete timeout, prompt-cache, policy-guard, and payload-isolation gaps.

Concerns:

- **[medium] Hook timeout budgets can be bypassed:** `src/plugins/hooks.ts:374`
  - body: PR head awaits modifying hook handlers directly instead of preserving current main's `withHookTimeout` wrapper, so a hung plugin can stall a model turn beyond configured hook budgets.
  - confidence: 0.88
- **[high] Prompt rewrites can miss provider-visible cache state:** `src/agents/pi-embedded-runner/run/attempt.ts:2507`
  - body: `llm_input` prompt and system-prompt mutations happen after Google prompt-cache setup, so cached provider input can be built from pre-hook state.
  - confidence: 0.9
- **[high] Prompt-injection guard list can regress:** `src/plugins/hook-types.ts:130`
  - body: The PR head's `PROMPT_INJECTION_HOOK_NAMES` omits current-main prompt-mutating hooks, weakening `allowPromptInjection=false` if the branch is merged or rebased without preserving current entries.
  - confidence: 0.92
- **[high] History payload aliasing bypasses the hook contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2475`
  - body: The `llm_input` history payload shallow-clones message objects instead of using current main's structured clone and LLM-boundary normalization, leaving nested message content mutable by plugin code outside the declared return contract.
  - confidence: 0.89
- **[medium] Stale image counts can mislead policy hooks:** `src/plugins/hooks.ts:638`
  - body: Evolved `llm_input` events can show a rewritten prompt with the original `imagesCount`, so lower-priority policy plugins may fail to block newly introduced image references.
