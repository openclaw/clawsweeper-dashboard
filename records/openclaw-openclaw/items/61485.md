## Summary

Keep open. Current main still has observe-only LLM hooks, and related attempts are closed, but this branch is not merge-ready because it drops recent hook timeout, policy-gate, payload, context-metadata, and prompt-cache contracts while expanding plugin control over model input/output.

## What This Changes

The PR makes `llm_input` and `llm_output` plugin hooks awaitable/modifying so plugins can block or rewrite model input and replace assistant output.

## Best Possible Solution

Land a rebased, maintainer-approved writable LLM hook contract that preserves current timeout, policy, payload isolation, prompt-cache, context metadata, documentation, and regression tests.

## Reproduction Assessment

Yes for the review findings: source comparison of PR head d06cd5f688b207a3ae40a9a084c5ac83b5392c8d against current main shows deterministic contract regressions. This is not a current-main bug reproduction because the central request is a new plugin API feature.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best merge shape until it is rebased and repaired to preserve current timeout, permission, payload, cache, context-metadata, documentation, and test contracts.
