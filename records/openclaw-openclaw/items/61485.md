  - body: The handler map now advertises `llm_input` and `llm_output` results, but sibling harness helpers still dispatch both hooks as best-effort void calls and ignore any returned block/rewrite/redaction. A plugin using the new API would work in the embedded runner but silently fail in Codex, Copilot, or CLI harness paths, so the public contract is inconsistent.
  - confidence: 0.88
- **[P1] Rebuild model inputs after llm_input rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4144-4148`
  - body: `llm_input` rewrites are applied after the Google prompt-cache wrapper and prompt-local image detection have already been built from the old prompt/system prompt. A security plugin that changes the system prompt or image references can therefore leave stale cached content or image payloads on the actual model call.
  - confidence: 0.86
- **[P1] Apply output overrides before terminal diagnostics:** `src/agents/embedded-agent-runner/run/attempt.ts:5140-5141`
  - body: The override is only substituted in the returned `assistantTexts`, while the terminal classification and trajectory records just above still consume the original `assistantTexts`. That means an `llm_output` redaction can still leak the original model text into diagnostics and can classify the attempt using text the plugin intended to suppress.
  - confidence: 0.84
- **[P2] Add default budgets for awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: This makes `llm_input` and `llm_output` awaited modifying hooks, but the default modifying-hook timeout map still has no budget for either hook. A plugin that hangs in one of these hooks can stall every model call unless each installation manually configures a timeout.
  - confidence: 0.82
- **[P2] Update docs for the new hook contract:** `src/plugins/hook-types.ts:280-289`
  - body: The diff changes the public plugin API, but the public hook catalog still describes `llm_input` and `llm_output` as observation-only and this branch does not update docs. Plugin authors would see conflicting guidance about whether these hooks can modify or block model traffic.
  - confidence: 0.79
