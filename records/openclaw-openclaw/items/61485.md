## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Make result-bearing LLM hooks runner-consistent:** `src/plugins/hook-types.ts:1045-1052`
  - body: The public handler map now says `llm_input` and `llm_output` can return modification results, but the shared harness helpers still invoke these hooks as best-effort side effects and discard returned values. That means CLI and Codex-style runs would ignore plugin blocks, prompt rewrites, and output redactions even though the Plugin SDK type contract says those results are supported.
  - confidence: 0.94
- **[P1] Rebuild prompt-local provider inputs after rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4144-4148`
  - body: `llm_input` prompt overrides are applied after prompt-local image detection has already loaded `imageResult`, and the provider call later still sends that old image set. A plugin that removes or adds image references can therefore approve one effective prompt while the model receives stale or missing prompt-local images.
  - confidence: 0.9
- **[P1] Replace downstream assistant state on output overrides:** `src/agents/embedded-agent-runner/run/attempt.ts:5141-5145`
  - body: When `llm_output` returns replacement text, only the returned `assistantTexts` field is swapped; `lastAssistant`, `currentAttemptAssistant`, and `messagesSnapshot` still carry the original assistant message. Redaction plugins can therefore leave original response content or error metadata visible to later hooks, context maintenance, diagnostics, and failover classification.
  - confidence: 0.91
- **[P2] Add default budgets for awaited LLM hooks:** `src/plugins/hooks.ts:967-1003`
  - body: This changes LLM hooks from fire-and-forget observations into awaited sequential hooks, but `llm_input` and `llm_output` are not added to the default modifying-hook timeout table. A hung plugin handler can now block the model critical path until a broader run timeout, unlike other awaited hooks such as `before_agent_start` that have defensive defaults.
  - confidence: 0.86
