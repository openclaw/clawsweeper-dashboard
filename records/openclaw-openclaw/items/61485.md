## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep modifying hook semantics consistent across runners:** `src/plugins/hook-types.ts:1045-1052`
  - body: This public type change makes `llm_input`/`llm_output` result-returning hooks, but the CLI and harness/Codex helper paths still fire these hooks as best-effort void observations and ignore block/rewrite/redaction returns. A plugin relying on the new API would enforce policy in embedded runs but silently observe only in those other runtimes, so either keep the hooks observation-only or implement and test the same contract across all emitters.
  - confidence: 0.91
- **[P1] Re-detect images after prompt rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4144-4146`
  - body: `imageResult` is computed before `llm_input`, but this new assignment lets a hook replace the prompt after that point while the later model call still submits the old `imageResult.images`. If a redaction plugin removes an image reference, the original image can still be sent to the provider; prompt overrides need to re-run image detection or roll back the whole derived prompt payload before submission.
  - confidence: 0.9
- **[P1] Submit the overridden system prompt to the model:** `src/agents/embedded-agent-runner/run/attempt.ts:4147-4149`
  - body: This only updates `systemPromptForHook`, while the active session system prompt and Google prompt-cache wrapper are still based on the original `systemPromptText`, and the later `promptActiveSession` call does not pass this override. A plugin returning `systemPrompt` can therefore pass telemetry/prechecks without actually changing the system prompt sent to the model.
  - confidence: 0.88
- **[P2] Apply output replacements before terminal decisions:** `src/agents/embedded-agent-runner/run/attempt.ts:5141`
  - body: The override is only returned at the end of the attempt, after silent-reply, terminal-output, liveness, and metadata decisions have already consumed the original `assistantTexts` and `lastAssistant`. A redaction or suppression plugin can change the final returned text while downstream terminal state and diagnostics still reflect the unredacted model output, so apply the replacement before those decisions or carry a closed modified-output state through them.
  - confidence: 0.85
- **[P2] Budget the newly awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: Moving these hooks onto `runModifyingHook` makes the embedded runner await policy/redaction plugins, but the default modifying-hook timeout map still has no entries for `llm_input` or `llm_output`. A hung handler can now stall model submission or response delivery unless every plugin opts into its own timeout; add default budgets and focused timeout coverage for both hooks.
  - confidence: 0.86

- A prompt-redaction plugin can rewrite prompt text while the original prompt-local images remain loaded and submitted, which can leak data the plugin intended to remove.
- Awaiting previously fire-and-forget LLM hooks without default budgets can stall model submission or response delivery when a plugin handler hangs.
- The current branch is live-conflicting, so maintainers need a rebase/merge refresh before any final merge review.
