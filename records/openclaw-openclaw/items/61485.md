- **[P1] Recompute images after prompt overrides:** `src/agents/embedded-agent-runner/run/attempt.ts:4144`
  - body: This accepts a rewritten prompt after `detectAndLoadPromptImages` has already built `imageResult`, so a plugin can remove an image reference while the original image payload is still sent, or add a new reference that is never loaded. Apply the override before image detection, or rerun detection and roll back the whole effective input on failure.
  - confidence: 0.89
- **[P1] Apply system prompt overrides to the provider path:** `src/agents/embedded-agent-runner/run/attempt.ts:4147`
  - body: Assigning only `systemPromptForHook` updates hook/diagnostic state, but the model path and Google prompt-cache wrapper still use the active session/system prompt built from `systemPromptText`. A plugin returning `systemPrompt` will appear accepted while the provider can still receive the old system prompt.
  - confidence: 0.88
- **[P1] Propagate output replacements before terminal delivery:** `src/agents/embedded-agent-runner/run/attempt.ts:5138`
  - body: The `llm_output` override is only substituted into the returned attempt result, after terminal/finalization paths have already consumed the original `assistantTexts` and `lastAssistant`. A redaction plugin can therefore leave the original response visible in delivered output or diagnostics; run the modifying hook before terminal consumers and carry the replacement through every downstream path.
  - confidence: 0.87
- **[P2] Add timeout budgets for awaited LLM hooks:** `src/plugins/hooks.ts:962`
  - body: Changing `llm_input` and `llm_output` to `runModifyingHook` makes them awaited hot-path hooks, but the default modifying-hook timeout map does not add budgets for these hook names. A plugin handler that never settles can now stall the agent turn; add bounded defaults and tests for timeout fail-open behavior.
  - confidence: 0.83
- **[P2] Document the new modifying hook contract:** `src/plugins/hook-types.ts:1042-1048`
  - body: The public docs still describe `llm_input` and `llm_output` as observation hooks, while this change adds blocking, prompt rewrite, and response replacement semantics. Update the plugin docs with ordering, timeout, merge, `allowPromptInjection`, and compatibility behavior before exposing this API to plugin authors.
  - confidence: 0.82
