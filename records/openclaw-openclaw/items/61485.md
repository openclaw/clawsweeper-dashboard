## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Make result-bearing LLM hooks runner-consistent:** `src/plugins/hook-types.ts:1045-1052`
  - body: Changing these handler signatures makes `llm_input` and `llm_output` a public result-bearing plugin contract, but the harness helpers used by CLI and Codex-style runtimes still call these hooks as best-effort side effects and discard returned block/rewrite/redact results. That means a security plugin can appear to be supported by the types while only being enforced on the embedded path; either keep the contract observe-only here or wire every affected runner/adapter and docs to the same semantics.
  - confidence: 0.94
- **[P1] Rebuild provider prompt inputs after LLM rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4144-4148`
  - body: `llm_input` prompt and system-prompt overrides are applied after Google prompt-cache wrapping and prompt-local image detection have already been prepared from the original prompt/system prompt. A policy plugin can therefore approve a rewritten prompt while the provider path still uses stale cache state or image payloads; move the hook before provider-local prep or rebuild that state after the override.
  - confidence: 0.9
- **[P1] Replace downstream assistant state on output overrides:** `src/agents/embedded-agent-runner/run/attempt.ts:5141-5144`
  - body: When `llm_output` returns `assistantTexts`, the returned text list is swapped only in the final attempt result while `lastAssistant`, `currentAttemptAssistant`, terminal classification, trajectory, and telemetry still see the original model output. A redaction plugin can hide text from one surface while the unredacted assistant state remains available downstream, so the output override needs to replace or sanitize the whole accepted assistant state before those consumers run.
  - confidence: 0.91
- **[P2] Add default budgets for awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: This changes `llm_input` and `llm_output` from parallel fire-and-forget hooks into awaited modifying hooks, but the default modifying-hook timeout table still has no entries for either hook. A hung plugin can now stall the model path; add explicit default budgets and tests matching the existing awaited hook timeout pattern.
  - confidence: 0.86
