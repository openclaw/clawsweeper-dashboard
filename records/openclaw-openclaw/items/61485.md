## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Add timeouts before awaiting LLM hooks:** `src/plugins/hooks.ts:967`
  - body: `llm_input` and `llm_output` used to be fire-and-forget, so a stuck handler could not block the model turn. This PR makes both hooks awaited through `runModifyingHook`, but the default modifying timeout map still has no entries for either hook, so one hung plugin can stall every affected agent run until external cancellation.
  - confidence: 0.91
- **[P1] Update canonical assistant state for output overrides:** `src/agents/embedded-agent-runner/run/attempt.ts:5141-5144`
  - body: Returning `llmOutputAssistantTextsOverride` is not enough because the attempt still returns the original `lastAssistant`, and `buildEmbeddedRunPayloads` prefers canonical assistant text when it exists. A normal `llm_output` redaction can therefore be replaced by the original assistant content during final payload construction.
  - confidence: 0.89
- **[P1] Rebuild prompt cache with the hook system prompt:** `src/agents/embedded-agent-runner/run/attempt.ts:4148`
  - body: The Google prompt-cache wrapper is created with `systemPromptText` before `llm_input` can assign `systemPromptForHook`. On prompt-cache eligible Google runs, a plugin system-prompt override can be ignored by the cached wrapper, which breaks the new compliance/policy use case.
  - confidence: 0.86
- **[P2] Redetect prompt images after prompt rewrites:** `src/agents/embedded-agent-runner/run/attempt.ts:4145`
  - body: `imageResult` is loaded from the original `promptSubmission.prompt` before `llm_input` can replace `promptForModel`, and the later model call sends the old `imageResult.images`. Prompt rewrites that add, remove, or change image references will send stale image payloads or omit newly referenced images.
  - confidence: 0.88
