- **[P1] Make output overrides beat currentAssistant:** `src/agents/pi-embedded-runner/run/attempt.ts:3172`
  - body: When `llm_output` returns replacement `assistantTexts`, this return path only swaps `assistantTexts` and strips `lastAssistant`; it leaves `currentAttemptAssistant` as the original model message. Current payload construction prefers `currentAssistant` over `assistantTexts`, so a redaction/suppression plugin can still leak the original assistant text or reasoning through the normal reply payload. Clear or replace the current-attempt assistant when an output override is accepted, or make payload construction explicitly honor the plugin override first.
  - confidence: 0.9
- **[P1] Add default budgets for awaited LLM hooks:** `src/plugins/hooks.ts:626`
  - body: This changes `llm_input` and `llm_output` from fire-and-forget observation to awaited modifying hooks, but the modifying-hook timeout defaults on main do not include either hook. A stuck plugin handler can now block every model submission or response delivery indefinitely; add bounded defaults and regression coverage for timeout/fail-open behavior before putting these hooks on the critical path.
  - confidence: 0.88
- **[P1] Rebuild prompt cache after systemPrompt overrides:** `src/agents/pi-embedded-runner/run/attempt.ts:2506-2511`
  - body: The attempt path prepares the Google prompt-cache wrapper before `llm_input` runs, and `prepareGooglePromptCacheStreamFn` captures the pre-hook system prompt for cached content. Applying a later `systemPrompt` override only to the session can be silently ignored on prompt-cache eligible Google runs, which breaks the policy/compliance rewrite use case this PR adds. Move the hook before cache preparation or rebuild the cache wrapper when the system prompt changes.
  - confidence: 0.84
- **[P2] Keep evolved imagesCount truthful:** `src/plugins/hooks.ts:644`
  - body: Resetting evolved `imagesCount` to `0` after a prompt rewrite makes later `llm_input` policy hooks see an explicit no-image state even if the rewritten prompt adds image references. Because image detection is rerun only after all hooks finish, a lower-priority image-blocking plugin can make the wrong allow/block decision. Re-detect before evolving, carry an unknown state, or avoid exposing a false count to downstream hooks.
  - confidence: 0.8
