## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive plugin authority over prompts and responses, and current wiring can leave original model-bound or user-facing data reachable after a plugin claims to rewrite or redact it.

Concerns:

- **[high] Prompt rewrites can leave stale image payloads:** `src/agents/embedded-agent-runner/run/attempt.ts:4145`
  - body: A plugin-owned prompt rewrite does not recompute prompt-local images, so sensitive images from the original prompt can still be sent or newly referenced images can be omitted after policy evaluation.
  - confidence: 0.91
- **[high] Response redaction can leak original assistant content:** `src/agents/embedded-agent-runner/run/attempt.ts:5141`
  - body: `llm_output` replacements do not update `lastAssistant` or the payload fallback signals, so downstream delivery, metadata, or failover paths can still use the original assistant content after a redaction plugin returns replacement text.
  - confidence: 0.9

  - body: Making formerly fire-and-forget LLM hooks awaited without default timeouts expands the availability impact of a misbehaving plugin on every agent turn that reaches the LLM boundary.
