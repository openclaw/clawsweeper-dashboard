## Summary

Keep this PR open, but it is not mergeable as-is: current main still has observe-only LLM hooks, and the branch’s useful feature is blocked by current-main prompt assembly, prompt-injection policy, and payload-aliasing issues.

## What This Changes

The branch changes `llm_input` and `llm_output` from fire-and-forget plugin observers into modifying hooks that can block/rewrite model input and replace assistant output.

## Best Possible Solution

Land a maintainer-approved, rebased hook contract that mutates before provider-visible prompt/cache construction, preserves prompt-injection policy, deep-clones hook payloads, and documents SDK semantics.

## Reproduction Assessment

Yes. Source inspection against current main and the PR head gives a high-confidence path for the blocking findings: prompt/cache construction now precedes the old hook site, the PR guard list is stale, and the history payload clone is shallow.

## Solution Assessment

No. The feature direction is plausible, and the logs are useful, but this branch is not the best merge shape until it is rebased onto current prompt assembly and closes the policy and payload-isolation gaps.
