## Summary

Keep open: the feature direction is plausible and the contributor supplied live proof, but the current PR head is stale against main and introduces blocking hook timeout, policy-gate, payload-isolation, and prompt-cache regressions.

## What This Changes

The PR changes `llm_input` and `llm_output` plugin hooks into awaitable modifying hooks that can block or rewrite model input and replace assistant output, with runner handling and tests.

## Best Possible Solution

Land a rebased, maintainer-approved writable LLM hook contract that preserves current hook timeouts, prompt and conversation policy gates, LLM-boundary payload isolation, provider cache ordering, docs, and regression tests.

## Reproduction Assessment

Yes for the review findings: source comparison of PR head `d06cd5f688b207a3ae40a9a084c5ac83b5392c8d` against current main shows deterministic contract regressions. This is not a current-main bug reproduction because the central request is a new plugin API feature.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best merge shape until it is rebased and repaired to preserve current timeout, permission, payload, cache, context-metadata, documentation, and test contracts.
