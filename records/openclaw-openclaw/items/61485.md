Full review comments:

- **[P1] Bound the newly awaited hook handlers:** `src/plugins/hooks.ts:949-957`
  - body: This changes `llm_input` and `llm_output` from fire-and-forget observers into awaited modifying hooks, but no default modifying-hook timeout is added for either hook. A plugin that never resolves can now stall the model turn, whereas the existing observe-only path could not block the request in the same way. Add bounded defaults and tests, or keep the old nonblocking contract for these hook names.
  - confidence: 0.87
- **[P1] Redact before anything is delivered:** `src/agents/embedded-agent-runner/run/attempt.ts:5390-5420`
  - body: The PR applies `llm_output` replacement after the assistant result has already been assembled in the attempt tail, while terminal delivery is gated today only by `before_agent_finalize`. A redaction plugin can therefore return replacement text too late for streamed/block replies or other delivery/log paths that have already seen the original assistant text. Move the mutation to a pre-delivery boundary and prove all delivery paths use the replacement.
  - confidence: 0.82
- **[P2] Update prompt-cache state when overriding the system prompt:** `src/agents/embedded-agent-runner/run/attempt.ts:4560-4576`
  - body: The embedded runner prepares the Google prompt-cache stream function using the current `systemPromptText` before the new `llm_input` override runs. If a plugin later returns `systemPrompt`, the provider/cache wrapper can still be keyed or populated with the old system prompt. Rebuild or defer that cache setup when the hook changes prompt inputs, and cover the provider-cache path in tests.
  - confidence: 0.78
- **[P2] Distinguish intentional empty output from no model output:** `src/agents/embedded-agent-runner/run/attempt.ts:5420-5428`
  - body: Accepting `assistantTexts: []` as an output override makes a plugin-suppressed response indistinguishable from a model that returned no assistant text. The runner has existing fallback logic that checks `attempt.assistantTexts.length === 0`, so an intentional redaction-to-empty can trigger retry/model-switch behavior instead of ending cleanly. Carry an explicit plugin-output-overridden state or preserve a terminal marker for this case.
  - confidence: 0.74
