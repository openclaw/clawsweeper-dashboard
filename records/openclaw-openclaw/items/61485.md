Full review comments:

- **[P1] Keep modifying semantics consistent across runtimes:** `src/plugins/hook-types.ts:1042-1051`
  - body: The handler map now advertises `llm_input`/`llm_output` results, but current CLI and harness/Codex paths still invoke those hooks as fire-and-forget observations and ignore returned values. A plugin author would see block/rewrite/redact work in one runtime and silently do nothing in another, so either implement the contract across all advertised paths or narrow the public API/docs to the supported runtime.
  - confidence: 0.9
- **[P1] Apply prompt rewrites before derived model state:** `src/agents/embedded-agent-runner/run/attempt.ts:4131-4136`
  - body: `llm_input` prompt/systemPrompt overrides are applied after image detection and Google prompt-cache wrapping have already used the original prompt/system prompt, and the systemPrompt assignment only updates the hook/trace variable rather than the active model prompt state. Policy plugins can therefore believe they rewrote model input while the actual model call still uses stale images/cache/system prompt state.
  - confidence: 0.86
- **[P1] Preserve output suppression through payloads:** `src/agents/embedded-agent-runner/run/attempt.ts:5138`
  - body: When an `llm_output` hook returns `assistantTexts: []` or multiple replacement texts, this only swaps the returned `assistantTexts`; `lastAssistant` remains the original raw assistant. The payload builder falls back to `lastAssistant` in those cases, and run metadata also derives final assistant text from session state, so suppression or redaction can still leak the original model output.
  - confidence: 0.88
- **[P2] Budget the awaited LLM hooks:** `src/plugins/hooks.ts:965`
  - body: `runLlmInput` and `runLlmOutput` now use `runModifyingHook`, and the embedded runner awaits them before model submission/response delivery, but the default modifying-hook timeout map does not include these hook names. A plugin handler that never resolves can now hang an agent turn that used to proceed because these hooks were fire-and-forget observations.
  - confidence: 0.9
- **[P2] Update the public docs and contract tests:** `src/plugins/hook-types.ts:1042-1051`
  - body: The PR changes a public plugin API but leaves the docs saying `llm_input` and `llm_output` are observation-only and the diff does not include focused contract/runtime tests for the new block, rewrite, redaction, timeout, and sibling-runtime behavior. That makes the new contract ambiguous for plugin authors and operators.
  - confidence: 0.82
