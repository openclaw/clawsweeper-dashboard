## Summary

Keep open: current main still has observe-only `llm_input`/`llm_output` hooks, and this PR contains useful feature work plus real logs, but the branch is conflicting and has blocking API/security issues against current main’s prompt assembly and hook-policy paths.

## What This Changes

This PR changes `llm_input` and `llm_output` plugin hooks into modifying hooks so plugins can block or rewrite LLM input and replace assistant output.

## Best Possible Solution

Land a maintainer-approved, rebased plugin hook contract that runs before provider-visible prompt/cache construction, preserves existing prompt-injection guard coverage, deep-clones hook payloads, and documents SDK semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main prepares `promptForModel` and Google prompt-cache before `llm_input`, while the PR head applies prompt/system overrides at the old hook site.

## Solution Assessment

No. The feature direction is plausible and the supplied logs are useful, but this branch is not the best merge shape until it is rebased onto current prompt assembly/cache behavior and preserves current prompt-injection policy coverage.
