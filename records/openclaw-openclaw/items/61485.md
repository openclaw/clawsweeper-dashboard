## Summary

Keep open: current main still treats these hooks as observe-only, so the feature is not implemented, but this PR head is not mergeable as-is because it regresses current hook timeout, permission-gate, payload-isolation, and cache-ordering contracts.

## What This Changes

The PR changes `llm_input` and `llm_output` hooks to return modifying results that can block or rewrite model input and replace assistant output, with runner handling and focused tests.

## Best Possible Solution

Land a rebased, maintainer-approved writable LLM hook contract that preserves hook timeouts, plugin policy gates, payload isolation, `tools` context, Google cache ordering, docs, and focused tests.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best merge shape until it is rebased and repaired to preserve current timeout, permission, payload, cache, cloning, documentation, and test contracts.
