## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep modifying hook semantics consistent across runners:** `src/plugins/hook-types.ts:1045-1052`
  - body: Changing the public handler map to return `PluginHookLlmInputResult`/`PluginHookLlmOutputResult` makes this an API-wide contract, but current CLI and Codex harness helper paths still call these hooks as fire-and-forget and discard returned block/rewrite/redaction results. The same plugin would therefore protect embedded runs while silently doing nothing in sibling runners unless the PR either implements the semantics there or explicitly narrows the public contract.
  - confidence: 0.91
- **[P1] Apply input rewrites before cache and image setup:** `src/agents/embedded-agent-runner/run/attempt.ts:4144-4148`
  - body: `prompt` and `systemPrompt` overrides are applied here after Google prompt-cache wrapping and prompt-local image loading already used the old prompt/system prompt. A policy plugin can appear to rewrite or block model input while cache setup or image attachments still reflect the pre-hook input, which breaks the proposed compliance/security contract.
  - confidence: 0.9
- **[P1] Propagate output redaction into downstream state:** `src/agents/embedded-agent-runner/run/attempt.ts:5141`
  - body: The override is only used in the returned `assistantTexts`, but terminal classification, silent-reply handling, trajectory events, and `lastAssistant` are computed from the original assistant text before this return. A redaction plugin can therefore hide the visible reply while original content remains in metadata/artifacts or state used by later logic.
  - confidence: 0.88
- **[P2] Add default budgets for awaited LLM hooks:** `src/plugins/hooks.ts:967`
  - body: This turns `llm_input` and `llm_output` into sequential awaited modifying hooks, but they are not added to the default modifying-hook timeout map. Without an operator-configured timeout, a hung plugin can stall model submission or response delivery on the hot path.
  - confidence: 0.86
