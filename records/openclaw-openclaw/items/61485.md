## Summary

This PR should stay open: it proposes a useful but security-sensitive plugin API expansion, and the current branch still has blocking regressions against current main's hook timeout, policy-gate, payload-isolation, and cache-ordering contracts.

## What This Changes

The PR changes plugin hook contracts and embedded runner handling so `llm_input` can block or rewrite model input and `llm_output` can replace assistant response text, with block/error handling and focused tests.

## Best Possible Solution

Land a rebased, maintainer-approved writable LLM hook contract that preserves current hook timeouts, prompt/conversation policy gates, LLM-boundary payload isolation, tools context, provider cache ordering, docs, and regression tests.
