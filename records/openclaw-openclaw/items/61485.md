## Summary

Keep open: current main still treats `llm_input` and `llm_output` as observe-only hooks, and this branch needs a rebase plus maintainer API/security review before it can safely change prompt and response authority.

## What This Changes

The PR changes `llm_input`/`llm_output` plugin hooks to return modification results and wires embedded-runner, auto-reply, types, and tests for blocking, prompt/system rewrites, response replacement, and plugin-blocked errors.

## Best Possible Solution

Land a rebased, maintainer-approved hook contract that runs before prompt assembly, prompt-cache wrapping, image detection, and provider submission; preserves existing guarded hooks; deep-clones hook payloads; documents SDK semantics; and keeps the supplied real behavior proof current.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register an `llm_input` hook that rewrites prompt/system prompt, then verify the embedded runner submits those rewritten values through the current prompt-submission and Google prompt-cache paths.

## Solution Assessment

No. The feature direction is plausible and the supplied logs are useful, but this branch is not the best merge shape until it is rebased onto current prompt assembly/cache behavior and preserves current prompt-injection policy coverage.
