## Summary

Keep this PR open for maintainer review. Current main still treats `llm_input` and `llm_output` as observe-only hooks, so the requested capability is not implemented, but the proposed patch has blocking correctness and security-policy issues before it can land.

## What This Changes

The PR changes plugin `llm_input` and `llm_output` hook types and runner handling so hooks can block, rewrite prompts/system prompts, or replace assistant response text, with tests for the new hook-runner behavior.

## Best Possible Solution

Land a rebased, documented plugin-hook contract that applies `llm_input` modifications before prompt submission, image detection, and provider cache wrapping; preserves all existing prompt-injection policy hooks; and includes an embedded-runner integration test proving block, prompt rewrite, system prompt rewrite, and output replacement affect the actual delivered run.

## Reproduction Assessment

Yes. A focused reproduction is to register an `llm_input` hook returning `{ prompt: "rewritten" }` and assert the embedded runner submits the rewritten prompt; code inspection shows this PR sets `effectivePrompt` after `promptSubmission` is already resolved, so that test should fail as written.

## Solution Assessment

No. The direction is plausible, but the patch is not the best implementation until the hook is moved earlier or the prompt submission/cache state is recomputed, and until current prompt-injection policy entries are preserved.
