## Summary

Keep open. Current main has forward-write redaction for new config-audit records, but I found no current source path that rewrites, rotates, or doctors existing `config-audit.jsonl` lines written before https://github.com/openclaw/openclaw/pull/75095, so the historical at-rest credential residue described here remains actionable.

## What This Changes

The issue asks OpenClaw to scrub or rotate historical config-audit JSONL entries written before argv redaction so plaintext credentials no longer remain at rest after upgrade.

## Best Possible Solution

Add an idempotent `openclaw doctor --fix` repair that atomically rewrites existing `config-audit.jsonl` records, redacts `argv` and `execArgv` with the config-audit argv redactor plus structured secret redaction, and never leaves a plaintext backup behind.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a pre-redactor `config-audit.jsonl` line containing raw token-shaped argv: current main only redacts records before append and has no doctor, startup, rotation, or rewrite path for existing lines.

## Solution Assessment

No. The current forward-only redaction is a good boundary for new writes, but it does not solve historical at-rest credentials; the narrow maintainable fix is a doctor repair or equivalent one-shot scrub of existing audit JSONL.
