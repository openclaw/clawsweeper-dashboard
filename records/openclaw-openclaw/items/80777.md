## Summary

Keep this open: current main redacts new config-audit argv/execArgv records, but I found no source, doctor, or release evidence that existing pre-redactor `config-audit.jsonl` entries are scrubbed or rotated. The related merged fix only covers future writes, and a second reporter confirmed the historical residue on `v2026.5.4`.

## What This Changes

The issue requests a retroactive scrub or rotation path for pre-redactor `config-audit.jsonl` entries that may still contain plaintext Slack, Telegram, gateway, or similar credentials after upgrade.

## Best Possible Solution

Add an explicit doctor/security repair that atomically redacts or rotates existing `config-audit.jsonl` records, preserves permissions and useful forensic metadata, and tells affected users to rotate any credentials that may have been logged before the forward redactor shipped.

## Reproduction Assessment

Yes, at source level: seed a state dir with a pre-redactor JSONL record containing a token-shaped `argv` value, then run current main or `openclaw doctor --fix`; the inspected code only redacts newly built records and has no existing-log rewrite path. ## Solution Assessment

Unclear pending maintainer/security choice: the requested scrub and rotate options are both plausible, but the repo’s migration policy points away from startup-time core migration and toward an explicit repair path such as `openclaw doctor --fix`.
