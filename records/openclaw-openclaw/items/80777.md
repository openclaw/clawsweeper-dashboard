## Summary

Keep this open: current main fixes new config-audit writes, but I found no current source path that rewrites or rotates existing `config-audit.jsonl` entries, so the reported historical credential residue remains plausible and security-sensitive.

## What This Changes

The issue asks OpenClaw to scrub or rotate pre-redactor `config-audit.jsonl` records whose `argv` or `execArgv` fields may still contain plaintext credentials.

## Best Possible Solution

Add a conservative doctor/security repair that atomically redacts or rotates existing `config-audit.jsonl` entries, preserves forensic metadata and file permissions, and tells affected users to rotate any credentials that may have been written before the forward redactor shipped.

## Reproduction Assessment

Yes, at source level: seed a state dir with a pre-redactor JSONL record containing a token-shaped `argv` value, then run current main or `openclaw doctor --fix`; the inspected code only redacts new records and has no existing-log rewrite path. I did not execute this because the checkout review was read-only.

## Solution Assessment

Unclear pending maintainer choice: the requested scrub and rotate options are both plausible, but repo policy points away from startup migration and toward an explicit repair path such as `openclaw doctor --fix`.
