The PR changes Telegram command-menu sync so non-empty menus call `setMyCommands()` without first calling `deleteMyCommands()`, while empty menus still delete explicitly and tests cover both paths.

## Best Possible Solution

Keep the change Telegram-plugin-local: non-empty command-menu updates should use one `setMyCommands` pass for each supported scope, empty-menu clearing should continue to delete explicitly, and the rebased branch should include real Telegram proof plus focused command-menu tests.
