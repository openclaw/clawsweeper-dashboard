## Summary

Keep open: current main still deletes Telegram command menus before non-empty sync, and this PR is a narrow functional fix, but it still needs contributor-provided real Telegram proof and maintainer judgment on the changed failure mode before merge.

## What This Changes

The branch moves `deleteMyCommands()` into the empty-menu path so non-empty Telegram command-menu sync uses `setMyCommands()` without a preceding delete, and updates colocated tests for both paths.

## Best Possible Solution

Keep the change Telegram-plugin-local: non-empty command-menu updates should use scoped `setMyCommands`, empty-menu clearing should still delete commands, and maintainers should either accept the stale-menu failure-mode tradeoff or add a focused fallback in the same helper.

## Reproduction Assessment

Yes. Source inspection on current main shows the non-empty sync path still calls `deleteMyCommands` for the configured scopes before `setMyCommands`, and the focused mock test currently asserts that old order.

## Solution Assessment

Yes functionally. Moving deletion into the empty-menu branch is the narrowest maintainable fix for the unnecessary non-empty delete, but merge should wait for real Telegram proof and maintainer acceptance of the changed failure mode.
