## Summary

Keep open. Current main still has the behavior this PR changes, the branch is narrowly scoped and consistent with Telegram command-menu API behavior, and I did not find a blocking functional or security issue; maintainer review should decide sequencing with #73947 before merge.

## What This Changes

The branch updates Telegram command-menu sync and its colocated tests so non-empty menus call `setMyCommands()` without a preceding `deleteMyCommands()`, while empty-menu sync still deletes commands explicitly.

## Best Possible Solution

Keep the fix local to the Telegram plugin: use one `setMyCommands()` call for non-empty replacements, preserve explicit `deleteMyCommands()` for empty menus, and keep focused tests for both paths.

## Reproduction Assessment

Yes. Current main and the colocated unit test provide a high-confidence reproduction: a non-empty command sync calls `deleteMyCommands()` before `setMyCommands()` without needing live Telegram credentials.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix for the stated behavior because it changes only the Telegram command-menu helper and tests; a fallback for failed `setMyCommands()` would be a separate maintainer tradeoff, likely in the same helper.
