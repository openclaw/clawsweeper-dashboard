## Summary

Keep this PR open: the patch is directionally correct and current main still has the delete-before-non-empty-set behavior, but the branch is currently conflicting and still lacks required real Telegram behavior proof before merge.

## What This Changes

The PR changes Telegram command-menu sync so non-empty command menus call `setMyCommands()` without a preceding `deleteMyCommands()`, while empty command menus still delete explicitly.

## Best Possible Solution

Refresh the branch on current main, keep non-empty updates as set-only replacements and empty-menu clearing as explicit deletes, then land after focused tests and redacted live Telegram command-menu proof.

## Reproduction Assessment

Yes from source inspection: current main still deletes Telegram command scopes before the non-empty `setMyCommands()` path, and the focused test still asserts that delete-to-set order. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes in direction, but not merge-ready. Moving deletion into the empty-menu branch is the narrow maintainable fix, while maintainers still need to accept the set-failure tradeoff and require a branch refresh plus live Telegram proof.
