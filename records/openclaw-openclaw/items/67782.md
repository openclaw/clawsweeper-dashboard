## What This Changes

The branch changes Telegram command-menu sync and its colocated tests so non-empty menus call `setMyCommands()` for the default and `all_group_chats` scopes without first calling `deleteMyCommands()`, while empty menus still delete both scopes explicitly.

## Best Possible Solution

Keep the fix scoped to the Telegram plugin: use set-only replacement for non-empty command-menu updates, keep explicit deletes for empty-menu clearing, and land only after redacted real Telegram proof plus focused validation on a refreshed branch.

## Reproduction Assessment

Yes, from source inspection rather than live execution: current main still calls `deleteMyCommands` for both scopes before setting a non-empty command menu, and the focused current test asserts that old order. I did not run live Telegram in this read-only review.

## Solution Assessment

Yes, the proposed shape is the narrow maintainable fix for the redundant non-empty delete while preserving explicit empty-menu clearing. It is not merge-ready until the branch is refreshed and the contributor adds real Telegram proof.
