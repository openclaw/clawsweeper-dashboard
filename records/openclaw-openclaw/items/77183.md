## Summary

This belongs outside OpenClaw core: current `main` treats Lossless Claw as an external/community plugin, already has generic plugin command/allowlist surfaces for this kind of CLI, and does not protect `plugins.allow` from supported config patch paths.

## What This Changes

The issue asks for `openclaw lcm` diagnostics to be available despite a restrictive `plugins.allow`, or for `plugins.allow` to be configurable enough to enable that surface.

## Best Possible Solution

Keep LCM diagnostics in the Lossless Claw plugin/ClawHub package while OpenClaw core preserves the generic plugin install, allowlist, `config.patch`, command alias, and CLI backend surfaces.

## Reproduction Assessment

No high-confidence core reproduction path remains from source inspection: `plugins.allow` is not protected from supported patch paths, and Lossless Claw is not bundled in this repo. Reproducing the `lcm` CLI behavior would require the third-party plugin package.

## Solution Assessment

Yes, keeping this out of core is the best current direction: the repository already exposes generic plugin command and allowlist mechanisms, and the optional LCM surface belongs to the existing Lossless Claw plugin/ClawHub path.
