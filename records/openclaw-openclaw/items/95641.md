## Summary

Keep open: this PR targets a real Bedrock session-loss bug in the right provider boundary, but it is not merge-ready because the retry loop can duplicate assistant turns after visible stream events and the transient classifier is not proven against the PR's own named-error fixtures.

## What This Changes

The PR adds a one-retry wrapper around Amazon Bedrock Converse streaming, creates a fresh Bedrock client for retry attempts, and adds retry/error-classification tests.

## Best Possible Solution

Keep the Bedrock-local retry direction, but make retry event-safe, classify actual transient Bedrock error shapes, add stream-consumer regression coverage, and include redacted live Bedrock proof when credentials are available.

## Reproduction Assessment

Yes, at source level. The linked issue gives production evidence, and current main still has a Bedrock stream path where provider stream aborts become terminal error/aborted state before provider-local retry can recover.

## Solution Assessment

No, not yet. Retrying inside the Bedrock adapter is the right boundary, but this implementation must avoid emitting unretractable partial stream events before retrying and must classify the actual transient error shapes it claims to cover.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid retrying after a visible stream start:** `extensions/amazon-bedrock/stream.runtime.ts:280`
  - body: The retry loop can continue after `messageStart` or deltas have already been pushed. `agentLoop` appends a new assistant message on every `start`, so a mid-stream Bedrock drop followed by retry leaves the first attempt's assistant turn in context and then appends the retried turn. Only retry before emitting visible events, or buffer/rollback the failed attempt with regression coverage.
  - confidence: 0.91
- **[P2] Classify named Bedrock errors without instanceof only:** `extensions/amazon-bedrock/stream.runtime.ts:368-380`
  - body: The PR's own retry fixtures throw `Error` objects with Bedrock `name` values, but this branch only checks those names when the value is a `BedrockRuntimeServiceException`. Named `InternalServerException`, `ServiceUnavailableException`, and `ModelStreamErrorException` values otherwise fall through as non-transient, so the advertised retry path is not covered by those fixtures.
  - confidence: 0.87
