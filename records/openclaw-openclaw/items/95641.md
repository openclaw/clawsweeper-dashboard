## Summary

Keep open: the linked Bedrock stream-abort bug is real and the Bedrock-local retry boundary is plausible, but this PR is not merge-ready because a mid-stream retry can leave already-emitted assistant events in session state and the transient-error classifier does not match the plain named Error shapes used by its own tests.

## What This Changes

The PR adds a one-retry Bedrock Converse streaming loop with a fresh Bedrock client on retry, a transient stream-error classifier, and focused retry/classification tests.

## Best Possible Solution

Keep the Bedrock-local retry direction, but make retries event-safe, classify real named Bedrock error shapes, add stream-consumer regression coverage, and include redacted live Bedrock proof when credentials are available.

## Reproduction Assessment

Yes, at source level. The linked issue gives production evidence, and current main still has a one-shot Bedrock stream path where provider stream aborts become terminal error/aborted output before provider-local retry can recover.

## Solution Assessment

No, not yet. Retrying inside the Bedrock adapter is an acceptable boundary, but this implementation must avoid unretractable partial stream events and must classify the actual transient error shapes it claims to cover.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid retrying after visible stream output:** `extensions/amazon-bedrock/stream.runtime.ts:336`
  - body: The retry loop can continue after `messageStart` or deltas have already been pushed. `agentLoop` appends a new assistant message on every `start`, so a mid-stream Bedrock drop followed by retry can leave the first attempt's assistant turn in context and then append the retried turn. Only retry before emitting visible events, or buffer/rollback the failed attempt with regression coverage.
  - confidence: 0.91
- **[P2] Classify named Bedrock errors without instanceof only:** `extensions/amazon-bedrock/stream.runtime.ts:368-380`
  - body: The PR's own retry fixtures throw plain `Error` objects with Bedrock `name` values, but this branch only checks those names when the value is a `BedrockRuntimeServiceException`. Named `InternalServerException`, `ServiceUnavailableException`, and `ModelStreamErrorException` values otherwise fall through as non-transient, so the advertised retry path is not covered by those fixtures.
  - confidence: 0.88
