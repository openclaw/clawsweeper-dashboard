## Summary

Keep open. Current main has hook-aware durable/outbound helpers, but WebChat/chat.send and sampled Slack, Mattermost, and Signal reply paths still can send through direct broadcast or channel send code without a guaranteed `message_sending` pre-send boundary, so the requested invariant is not implemented.

## What This Changes

Request a guaranteed outbound pre-send policy boundary so all agent replies, tool sends, auto-replies, and channel sends can be rewritten or cancelled before delivery.

## Best Possible Solution

Keep this umbrella open until maintainers define one outbound lifecycle contract and migrate WebChat, gateway `chat.send`, legacy channel reply paths, and message-tool sends through a shared hook-aware durable send context with regression coverage.

## Reproduction Assessment

Yes, at source level. Register a `message_sending` hook and follow the WebChat final or gateway `chat.send` collector path, or a sampled Slack reply path; current main has direct broadcast/send code there without the shared hook-aware lifecycle.

## Solution Assessment

Yes, directionally: a single outbound lifecycle boundary is the maintainable solution. The current implementation is only partial, so maintainers still need to decide the universal contract and migrate the remaining direct delivery paths.
