## Summary

Keep open. Current main has a hook-aware durable outbound path, but the requested unbypassable pre-send guarantee is still missing because WebChat final/delta delivery and legacy plugin action fallbacks can still send user-visible output outside the `message_sending` boundary.

## What This Changes

The issue requests a single enforceable outbound pre-send policy boundary that can validate, modify, or block every user-visible outbound message path before channel delivery.

## Best Possible Solution

Define and implement a generic message lifecycle boundary that every user-visible send path must use, with documented cancel/rewrite outcomes and regression coverage for WebChat, message-tool sends, final replies, intermediate replies, and channel fallback/error paths.

## Reproduction Assessment

Yes. Source inspection shows `src/gateway/server-chat.ts` still broadcasts WebChat assistant final/delta payloads directly, and the message action service can still fall back to plugin-owned `handleAction` sends when no durable `prepareSendPayload` exists.

## Solution Assessment

Unclear as an implementation plan, but the requested invariant is the right end state. Maintainers still need to decide the permanent lifecycle contract for WebChat streaming/final messages and legacy plugin action compatibility before a safe implementation PR.
