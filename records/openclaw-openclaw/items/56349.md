## Summary

Keep open. Current main has hook-aware durable outbound helpers and dispatcher seams, but the requested unbypassable pre-send invariant is still only partial: WebChat/gateway chat final delivery can broadcast user-visible assistant output without the `message_sending` cancel/modify boundary.

## What This Changes

Request a single enforceable outbound policy boundary that validates, modifies, or blocks every user-visible outbound message before channel delivery.

## Best Possible Solution

Define one maintainer-approved outbound lifecycle contract, then migrate WebChat/gateway chat sends, legacy channel reply paths, and message-tool sends through a shared hook-aware dispatcher with regression coverage for cancel, rewrite, and successful delivery.

## Reproduction Assessment

Yes, at source level. Register a `message_sending` hook and follow the WebChat final delivery or gateway `chat.send` collector path; current main still has direct broadcast/session-send code there without the shared cancel/modify lifecycle.

## Solution Assessment

Yes, directionally: a single outbound lifecycle boundary is the maintainable solution. The current implementation is only partial, so maintainers still need to choose the universal contract and migrate remaining direct delivery paths safely.
