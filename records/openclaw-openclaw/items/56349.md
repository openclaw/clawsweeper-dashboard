## Summary

Keep open. Current main and v2026.6.8 still have partial hook-aware outbound delivery, but they do not provide the requested unbypassable pre-send guarantee because configured reply-delivery hooks can omit `message_sending`, and related issues or closed PRs do not supersede the central contract work.

## What This Changes

The issue requests a single enforceable pre-send outbound boundary where all user-visible sends can be validated, modified, or blocked before reaching channel adapters.

## Best Possible Solution

Define a versioned core outbound lifecycle contract that every user-visible send path uses exactly once, with documented ordering for channel transforms, `reply_payload_sending`, `message_sending`, cancellation, and post-send observation.

## Reproduction Assessment

Yes. Current main and v2026.6.8 show a source-level reproduction: a custom `beforeDeliver` path omits `message_sending`, and the adjacent test asserts `runMessageSending` is not called.

## Solution Assessment

Yes as an end state, but not as another one-off channel patch. The maintainable solution is a core-owned outbound lifecycle contract with exactly-once hook coverage and documented ordering across all send paths.
