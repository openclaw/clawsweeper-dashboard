## Summary

Keep open. Current main has hook-aware durable delivery in some paths, but it still lacks the requested unbypassable pre-send boundary because custom dispatcher hooks and direct channel reply paths can bypass `message_sending`.

## What This Changes

The issue requests a guaranteed outbound policy boundary that can validate, rewrite, or block every outbound message before any channel adapter sends it.

## Best Possible Solution

Define one versioned core outbound lifecycle contract that every user-visible send uses exactly once, with explicit ordering for channel transforms, `reply_payload_sending`, `message_sending`, cancellation, and post-send observation.

## Reproduction Assessment

Yes, at source level. Current main deterministically skips `message_sending` when custom `beforeDeliver` is supplied, and sibling direct-send paths such as Signal still bypass the durable hook-aware boundary.

## Solution Assessment

Yes as an end state, but not as a one-off channel patch. The maintainable solution is a core-owned outbound lifecycle contract with explicit hook ordering and exactly-once coverage.
