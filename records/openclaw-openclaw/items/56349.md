## Summary

Keep open. Current main has a durable outbound path that runs `message_sending`, but the requested unbypassable pre-send guarantee is still missing because some visible reply paths use dispatcher/direct-delivery shapes that do not include the global message-sending hook.

## What This Changes

The issue requests a single outbound policy boundary where every agent reply, message-tool send, and auto-reply can be validated, modified, or blocked before reaching a channel adapter.

## Best Possible Solution

Define one core-owned outbound delivery contract that every user-visible send must use, then add regression coverage proving `message_sending` can rewrite or cancel WebChat, channel reply, message-tool, and routed auto-reply sends consistently.

## Reproduction Assessment

Yes. Source inspection shows current-main paths where `message_sending` is not part of the final pre-delivery chain, including custom `beforeDeliver` composition and WebChat/manual dispatcher use.

## Solution Assessment

Unclear as an implementation plan, but the requested invariant is the right end state. The safest solution is a versioned core outbound lifecycle contract rather than more channel-by-channel hook patches.
