## Summary

Keep open. Current main has hook-aware durable delivery and partial dispatcher coverage, but Gateway/WebChat final delivery and the `chat.send` collector still broadcast user-visible assistant output without the guaranteed `message_sending` cancel/modify boundary.

## What This Changes

The issue requests a single unbypassable outbound pre-send policy boundary for all user-visible outbound paths, with plugin or workspace hooks able to validate, rewrite, or block messages before delivery.

## Best Possible Solution

Define one maintainer-approved outbound lifecycle contract, then migrate Gateway/WebChat final and delta paths, `chat.send` collector delivery, legacy channel replies, and message-tool sends through a shared hook-aware dispatcher with regression coverage for cancel, rewrite, and successful delivery.

## Reproduction Assessment

Yes, at source level. Register a `message_sending` hook and follow a WebChat `chat.send` assistant run: current Gateway chat final paths directly broadcast final assistant content while the shared outbound path is the hook-aware one.

## Solution Assessment

Yes directionally, but not as a one-off patch. A single outbound lifecycle boundary is the maintainable solution; maintainers still need to choose the universal contract, including WebChat streaming and fail-closed behavior.
