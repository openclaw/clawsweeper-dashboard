## Summary

Keep open. Current main has useful partial hook-aware paths, but it still does not provide the requested unbypassable pre-send boundary: WebChat/Gateway final delivery, chat.send final broadcast, and sampled direct channel dispatch paths can still avoid `message_sending`.

## What This Changes

The issue requests a single guaranteed pre-send outbound boundary so every outbound path can validate, modify, or block messages before channel delivery.

## Best Possible Solution

Define one documented outbound lifecycle seam, then route WebChat, gateway chat.send, direct channel dispatchers, message-tool sends, auto-replies, and streaming finalization through it exactly once with focused regression tests.

## Reproduction Assessment

Yes, at source level. Register a `message_sending` hook and drive a WebChat final assistant reply or a direct Slack-style dispatcher path: current main has direct final broadcast/dispatch code paths that do not call the hook runner.

## Solution Assessment

No, current main is only a partial solution. The existing shared outbound and dispatcher-wrapper seams are useful, but the requested guarantee needs a maintainer-approved lifecycle contract and broader routing coverage.
