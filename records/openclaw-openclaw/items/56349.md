## Summary

Keep open. Current main has a hook-aware durable outbound path, but the requested unbypassable pre-send guarantee is still missing because source-level reply paths can omit `message_sending`, so this remains a security-sensitive product/API design item rather than a cleanup close.

## What This Changes

The issue requests a single outbound pre-send policy boundary that can validate, rewrite, or block every user-visible outbound message before it reaches channel adapters.

## Best Possible Solution

Define one versioned core outbound lifecycle contract that every user-visible send uses exactly once, with explicit ordering for channel transforms, `reply_payload_sending`, `message_sending`, cancellation, and post-send observation.

## Reproduction Assessment

Yes, at source level. Current main deterministically skips `message_sending` for configured `beforeDeliver` composition, and Signal still has a direct reply delivery path outside the durable hook-aware boundary.

## Solution Assessment

Yes as an end state, but not as another one-off channel patch. The maintainable fix is a core-owned outbound lifecycle contract with exactly-once hook coverage and documented ordering across all send paths.
