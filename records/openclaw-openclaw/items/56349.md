## Summary

Keep open. Current main has a shared outbound path that runs `message_sending`, but the requested unbypassable pre-send guarantee is still missing because configured dispatcher paths can omit that hook, Discord preview-final edits can commit directly, Signal replies still send directly, and hook failures remain fail-open.

## What This Changes

Requested a guaranteed pre-send outbound policy boundary that can validate, modify, or block every user-visible outbound message before it reaches a channel adapter.

## Best Possible Solution

Define one versioned core outbound lifecycle for user-visible sends, migrate remaining direct and streaming/edit paths onto it, and document intentional non-user-content exceptions.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main still has configured-beforeDeliver, Discord preview-edit, Signal direct-send, and fail-open hook paths that do not satisfy an unbypassable pre-send boundary.

## Solution Assessment

Unclear pending maintainer design. A unified pre-send boundary is the right end state, but maintainers need to decide whether `message_sending`, a new versioned outbound policy API, or both own fail-closed semantics and streaming/edit behavior.
