## Summary

Keep open. Current main has a hook-aware shared outbound substrate, but WebChat final broadcasts, gateway chat.send, Slack replies, and sampled Matrix/Mattermost plugin send helpers still do not all route through one guaranteed pre-send boundary; related open issues and PRs cover slices, not the full invariant.

## What This Changes

The issue requests a guaranteed outbound pre-send policy boundary that can validate, rewrite, or cancel every user-visible outbound message before channel delivery.

## Best Possible Solution

Keep this issue as the umbrella until maintainers define one outbound lifecycle contract, then split or land focused fixes that route WebChat, gateway chat.send, message-tool sends, auto-replies, and channel-plugin replies through a shared pre-send helper with regression tests.

## Reproduction Assessment

Yes, at source level. Register a `message_sending` hook and deliver a WebChat final assistant reply: current `emitChatFinal` broadcasts the final payload directly without a hook-runner call, and Slack/Mattermost/Matrix sampled reply paths show similar direct send helpers.

## Solution Assessment

No, current main is only a partial solution. The existing shared outbound and dispatcher hook seams are useful, but the requested unbypassable boundary needs a maintainer-approved lifecycle contract and broader routing than a single narrow patch.
