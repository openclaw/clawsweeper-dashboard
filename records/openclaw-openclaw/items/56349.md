## Summary

Keep open. Current main and v2026.6.10 still have outbound paths that bypass a guaranteed pre-send policy boundary, so this issue remains valid and needs maintainer product/security design rather than cleanup closure.

## What This Changes

Request to guarantee every outbound user-visible message passes through a non-bypassable pre-send boundary that can validate, rewrite, or cancel delivery.

## Best Possible Solution

Define one versioned core outbound lifecycle for user-visible sends, migrate remaining direct and streaming paths onto it, and document any intentional non-user-content exceptions.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main and v2026.6.10 still have dispatcher, Signal, and Slack paths that can send without a guaranteed `message_sending` pre-send boundary. I did not run live channel traffic in this read-only review.

## Solution Assessment

Unclear pending maintainer design. A unified pre-send boundary is the right end state, but maintainers need to decide whether the existing `message_sending` hook or a new versioned outbound policy API owns fail-closed semantics and audit behavior.
