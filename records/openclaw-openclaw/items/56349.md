## Summary

Keep open. Current main has hook-aware shared outbound delivery and partial inbound-dispatch coverage, but Gateway/WebChat final delivery still has direct broadcast paths that bypass the documented `message_sending` rewrite/cancel boundary, so the requested unbypassable pre-send guarantee is not implemented.

## What This Changes

The issue requests a single outbound policy boundary that every user-visible outbound message must pass through before any channel or WebChat delivery.

## Best Possible Solution

Define a maintainer-approved outbound lifecycle contract, then migrate Gateway/WebChat final and delta paths, `chat.send` collector delivery, legacy channel replies, and message-tool sends through one hook-aware dispatcher with regression coverage for cancel, rewrite, and successful delivery.

## Reproduction Assessment

Yes, at source level. Register a `message_sending` hook and follow a WebChat `chat.send` assistant run: current Gateway chat final paths directly broadcast final assistant content while the shared outbound path is the hook-aware one.

## Solution Assessment

Yes directionally, but not as a one-off patch. A single outbound lifecycle boundary is the maintainable solution; maintainers still need to choose the universal contract, including WebChat streaming and fail-closed behavior.
