## What This Changes

The PR adds `scripts/docker/setup.sh --offline`, Docker install docs, and Docker setup e2e coverage so preloaded images can be reused without build or pull during setup reruns.

## Best Possible Solution

Land the additive flag after normal maintainer and CI gates, keeping the default online build/pull behavior unchanged and using this PR as the canonical implementation for the linked offline-rerun request.

## Reproduction Assessment

Yes. By source inspection, current main has no `--offline` path and still reaches build or pull before the rerun steps an airgapped host needs.

## Solution Assessment

Yes. The explicit opt-in flag is narrower than changing default image behavior or asking operators to fork the script, and it preserves the existing online setup flow.
