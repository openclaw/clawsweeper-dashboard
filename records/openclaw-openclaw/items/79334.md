## Summary

Keep open: this is a useful feature PR, but the new Codex structured extraction path misses the request agent/config auth context on the real app-server startup path, and the supplied proof is mock-backed rather than real runtime proof.

## What This Changes

The PR adds a plugin SDK media-understanding structured extraction API, exposes it through the plugin runtime facade, implements a Codex app-server provider path, and updates tests, docs, API baselines, and the changelog.

## Best Possible Solution

Keep the narrow image-first seam only if it preserves agent-scoped Codex auth/profile context, has redacted real runtime proof, and leaves the broader structured inference API to https://github.com/openclaw/openclaw/issues/80188.

## Reproduction Assessment

Yes for the review finding: source inspection shows the new structured extraction real-client branch omits `req.agentDir` and `req.cfg` even though the shared client uses those to resolve auth profile context. The feature request itself is additive, so there is no failing current-main runtime behavior to reproduce.

## Solution Assessment

No as implemented: the narrow media-understanding direction is plausible, but the Codex startup path should thread the request agent/config auth context before merge. The broader text/image structured inference direction is better tracked separately in the open follow-up.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Thread the agent auth context into Codex startup:** `extensions/codex/media-understanding-provider.ts:201-205`
  - body: The new real app-server branch starts `createIsolatedCodexAppServerClient` with only `startOptions`, `timeoutMs`, and `authProfileId`. That helper falls back to the default agent dir/config when `agentDir` and `config` are omitted, so a plugin call with a non-default agent dir, configured profile order, or agent-local Codex profile can fail or run under the wrong auth context; the supplied proof uses `clientFactory`, so it does not cover this path. Pass the request `agentDir`/`cfg` into startup and cover the no-factory path before exposing this SDK seam.
  - confidence: 0.88
