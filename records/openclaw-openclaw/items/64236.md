## Summary

Keep open. Current main still supports per-agent and global `reasoningDefault`, plus catalog-derived `on`/`off` fallback, but it does not support the requested per-model `reasoningDefault` under `agents.defaults.models[...]`.

## What This Changes

Add a per-model `reasoningDefault` configuration so selected model entries can default reasoning visibility independently.

## Best Possible Solution

Add a documented top-level `reasoningDefault: "off" | "on" | "stream"` to model entries, resolve it for the selected model before global/catalog fallback, and cover schema, docs, status/current-level helpers, and runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive config feature request. Current-main source inspection gives a high-confidence verification path showing the model-entry key is absent and the runtime never reads it.

## Solution Assessment

Yes. The narrow maintainable path is to extend the existing config/schema/runtime precedence rather than treating the merged Telegram/Feishu preview work or global-default work as a substitute.
