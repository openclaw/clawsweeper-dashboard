## Summary

Keep open. Current main still lacks the requested per-model `reasoningDefault` surface under `agents.defaults.models[...]`; it only supports per-agent/global configured defaults plus a catalog-derived model fallback that returns `on` or `off`, and related PR #71817 is still open channel-specific work rather than the core config feature.

## What This Changes

The issue requests a per-model `reasoningDefault` setting under `agents.defaults.models[...]` so different configured models can default to `off`, `on`, or `stream` without per-session commands.

## Best Possible Solution

Add a documented `reasoningDefault: "off" | "on" | "stream"` field to configured model entries, resolve it for the selected model before global/catalog fallback, and preserve the existing auth gating plus explicit `off` behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The verification path is high-confidence because the model-entry type, strict schema, generated schema, docs, and runtime precedence all omit configured per-model `reasoningDefault`.

## Solution Assessment

Yes. A narrow additive config/schema/runtime/docs/test change is the maintainable path; related PR #71817 is useful channel context but does not replace core model-entry config support.
