## Summary

Keep open. Current main still lacks the requested per-model `reasoningDefault` config surface; it supports per-agent/global configured defaults plus a catalog-derived `on`/`off` fallback, but not configured `stream` or model-by-model visibility overrides.

## What This Changes

Request adding per-model `reasoningDefault` entries under `agents.defaults.models[...]` so different models can default to `off`, `on`, or `stream`.

## Best Possible Solution

Add a documented `reasoningDefault: "off" | "on" | "stream"` field to configured model entries and resolve it for the selected model before falling back to global/catalog defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence verification because the model-entry type, strict schemas, generated schema, docs, and runtime precedence all omit configured per-model `reasoningDefault`.

## Solution Assessment

Yes. A narrow additive config/schema/runtime/docs/test change is the maintainable path; the open channel-specific PR remains useful context but does not replace the core model-entry config support.
