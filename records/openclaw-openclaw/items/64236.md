## Summary

Keep open. Current main still lacks the requested per-model `reasoningDefault` config surface, schema support, generated schema metadata, docs, and runtime resolution; the related open PR is useful channel-specific context but does not implement this core config request.

## What This Changes

Allow `agents.defaults.models["<provider>/<model>"].reasoningDefault` so selected models can default reasoning visibility to `off`, `on`, or `stream` without per-session commands.

## Best Possible Solution

Add an additive, documented `reasoningDefault: "off" | "on" | "stream"` field to configured model entries, resolve it for the selected model after explicit session/per-agent defaults and before global/catalog fallback, and cover schema, generated docs, runtime, tests, and changelog.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request with a high-confidence verification path. The proposed key is absent from the model-entry type and strict schemas, and runtime resolution never reads model-entry `reasoningDefault`.

## Solution Assessment

Yes. A narrow additive config/schema/runtime/docs/test change is the maintainable solution; the channel-specific PR #71817 should remain adjacent context rather than a replacement for the core per-model config surface.
