## Summary

Keep open. Current main supports global and per-agent reasoning defaults, but the requested per-model `reasoningDefault` under `agents.defaults.models[...]` is still absent from the config type, strict schema, generated schema, docs, and runtime resolution path; the related open PR only covers Telegram/Feishu per-agent fallback behavior and does not replace the core per-model config surface.

## What This Changes

The issue requests per-model `reasoningDefault` configuration under `agents.defaults.models` so different selected models can default to `off`, `on`, or `stream` independently.

## Best Possible Solution

Add an additive, documented `reasoningDefault: "off" | "on" | "stream"` field to configured model entries, validate and generate schema metadata for it, and resolve it for the selected model after inline/session/per-agent overrides but before global and catalog fallback while preserving the current authorization guard for configured reasoning defaults.

## Reproduction Assessment

Not applicable as a bug reproduction, but there is a high-confidence verification path: the proposed key is absent from the model-entry type and strict schema, generated schema rejects it, and runtime never reads it from the selected model entry.

## Solution Assessment

Yes. A narrow core config/schema/runtime/docs/test change is the best way to solve this; the channel-specific PR #71817 is useful context but does not implement the requested per-model configuration surface.
