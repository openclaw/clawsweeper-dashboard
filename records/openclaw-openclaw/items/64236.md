## Summary

Keep open. Current main has global and per-agent reasoning defaults, plus the merged Telegram/Feishu preview fallback, but it still lacks the requested per-model `reasoningDefault` field under `agents.defaults.models[...]` and the runtime precedence needed to apply it.

## What This Changes

Request support for configuring `reasoningDefault` per model in `agents.defaults.models` so reasoning-capable and non-reasoning models can have different default visibility.

## Best Possible Solution

Add an additive, documented `reasoningDefault: "off" | "on" | "stream"` field to configured model entries and resolve it for the selected model with explicit precedence, schema/help/docs updates, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The source-verification path is high-confidence because the model-entry type, strict schema, docs/help labels, runtime precedence, and tests all omit per-model `reasoningDefault`.

## Solution Assessment

Yes, with maintainer approval for the config contract. The narrow maintainable path is an additive config/schema/runtime/docs/test change; the merged Telegram/Feishu PR is useful related context but does not replace core per-model config support.
