## Summary

Keep open. Current main and the latest stable release still lack a `reasoningDefault` field on configured model entries, and the merged Telegram/Feishu preview fix only covers per-agent/global channel fallback behavior, not per-model config.

## What This Changes

The issue requests model-entry `reasoningDefault` support so different configured models can default to different reasoning visibility levels.

## Best Possible Solution

Add a documented `reasoningDefault: "off" | "on" | "stream"` field to model entries, resolve the selected model entry after directive/session/per-agent defaults and before global/catalog fallback, and keep catalog `reasoning: true` as compatibility fallback.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive config feature. Source inspection gives high confidence because the strict schemas lack the proposed key and runtime resolution never reads a model-entry `reasoningDefault`.

## Solution Assessment

Yes. Extending the existing model-entry config/schema/runtime path is the narrow maintainable solution; the existing global default and merged Telegram/Feishu fallback do not provide per-model `stream`/`off` selection.
