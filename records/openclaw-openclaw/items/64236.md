## Summary

Keep open. Current main and the latest release still support global and per-agent `reasoningDefault`, but not the requested per-model `reasoningDefault` on configured model entries; the merged Telegram/Feishu work is useful adjacent context but only covers channel preview fallback behavior.

## What This Changes

Request to add per-model `reasoningDefault` configuration under configured model entries so different model refs can default to `off`, `on`, or `stream`.

## Best Possible Solution

Add a documented `reasoningDefault: "off" | "on" | "stream"` field to model-entry config, resolve it for the selected model after directive/session/per-agent defaults and before global/catalog fallback, and keep catalog `reasoning: true` as the compatibility fallback.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for source-level verification: current strict schemas lack the model-entry key and runtime resolution never reads it. The exact requested config shape would be rejected or ignored on current main.

## Solution Assessment

Yes. Extending the existing model-entry config/schema/runtime path is the narrow maintainable solution; existing global defaults and the merged Telegram/Feishu preview fix do not provide per-model `stream` or `off` selection.
