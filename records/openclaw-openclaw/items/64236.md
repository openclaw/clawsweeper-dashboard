## Summary

Keep open. Current main and the latest release still support per-agent and global `reasoningDefault`, plus catalog-derived model fallback, but they do not support the requested `reasoningDefault` field inside `agents.defaults.models[...]`.

## What This Changes

Request to add per-model `reasoningDefault` configuration so different configured models can default reasoning visibility to `off`, `on`, or `stream`.

## Best Possible Solution

Add a documented `reasoningDefault: "off" | "on" | "stream"` field to model entries, resolve it after directive/session/per-agent defaults and before global/catalog fallback, and keep the catalog `reasoning: true` fallback as compatibility behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive config feature request. Source inspection gives a high-confidence verification path: the model-entry type and strict schema lack the key, and runtime resolution never reads a model-entry `reasoningDefault`.

## Solution Assessment

Yes. The narrow maintainable path is to extend the existing config/schema/runtime precedence for selected model entries, rather than treating the merged channel-preview fallback or `thinkingDefault` provider params as a substitute for reasoning visibility.
