## Summary

Keep open. Current main still lacks the requested per-model `reasoningDefault` surface under `agents.defaults.models[...]`; it supports per-agent/global defaults and a catalog-derived `on`/`off` fallback, while https://github.com/openclaw/openclaw/pull/71817 only fixed adjacent Telegram/Feishu per-agent/global preview behavior.

## What This Changes

Requests a per-model `reasoningDefault` config under `agents.defaults.models[...]` so different models can default reasoning visibility differently.

## Best Possible Solution

Add a documented `reasoningDefault: "off" | "on" | "stream"` field to configured model entries, resolve it for the selected model before global/catalog fallback, and cover the schema, docs, and runtime tests.
