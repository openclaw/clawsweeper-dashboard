## Summary

Keep open: current main and v2026.5.18 support per-agent and global `reasoningDefault`, and the merged Telegram/Feishu PR fixed channel preview fallback, but the requested per-model `agents.defaults.models[model].reasoningDefault` config field is still absent from schema, docs, and runtime precedence.

## What This Changes

The issue requests an additive config option so each configured model entry can define its own default reasoning visibility level.

## Best Possible Solution

Add a documented `reasoningDefault: "off" | "on" | "stream"` field to model-pool entries, resolve it for the selected model after directive/session/per-agent defaults and before global/catalog fallback, and keep catalog-derived reasoning as the compatibility fallback.

## Reproduction Assessment

Not applicable as a failing bug reproduction; this is an additive config feature request. Source inspection gives a high-confidence verification path: the model-entry type and strict schema omit `reasoningDefault`, and runtime resolution never reads it from `agents.defaults.models[...]`.

## Solution Assessment

Yes. Extending the existing model-entry config, schema, docs, and runtime precedence is the narrow maintainable path; global defaults, per-agent defaults, channel-preview fallback, and catalog `reasoning` metadata cannot express per-model `stream` versus `off` choices.
