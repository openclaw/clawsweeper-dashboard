## Summary

Keep open: current main still has a source-visible Matrix direct-room discovery path where an older strict `m.direct` mapping suppresses joined-room discovery, and two open PRs explicitly reference this issue with closing syntax. The issue should stay open until maintainers choose and land a fix PR.

## What This Changes

Matrix should recognize and reply to newer strict 2-member DM rooms under `dm.sessionScope: "per-room"` even when `m.direct` still maps the same user to an older DM room.

## Best Possible Solution

Review and land one Matrix plugin fix, preferably the path that keeps joined-room discovery alive and adds per-room-aware safe promotion while preserving the existing anti-false-positive guards; then close this issue after the fix merges.

## Reproduction Assessment

Yes. The issue is source-reproducible on current main by giving `inspectMatrixDirectRooms` an older strict mapped room plus a newer joined strict room: `getJoinedRooms()` is skipped because `mappedStrict` exists, so the newer room is never surfaced.

## Solution Assessment

Yes, the best direction is a Matrix-plugin fix rather than a core routing change. The maintainable solution is to evaluate valid joined strict rooms under per-room semantics while retaining local `is_direct: false`, configured-room, and seeded-cache safeguards from prior Matrix fixes.
