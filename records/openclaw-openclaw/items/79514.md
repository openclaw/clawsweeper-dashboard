Keep open: current main still has a source-visible Matrix direct-room path where an older strict `m.direct` mapping suppresses joined-room discovery, and this issue is already paired with open fixing PRs.

## What This Changes

Matrix per-room DM handling should recognize and reply in newer strict direct rooms even when `m.direct` still maps the same user to an older direct room.

## Best Possible Solution

Land one focused Matrix plugin fix that keeps valid per-room strict DMs discoverable without weakening the existing anti-false-positive DM gates.
