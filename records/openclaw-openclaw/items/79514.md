## Summary

Keep open: current main still has a source-visible Matrix direct-room path where an older strict `m.direct` mapping suppresses joined-room discovery, which matches the report's core stale-mapping/new-DM-room failure mode.

## What This Changes

Matrix per-room DM handling should keep new strict 1:1 DM rooms usable even when older `m.direct` mappings exist, and direct repair should recover stale direct-room state.

## Best Possible Solution

Matrix should make per-room DM routing and direct repair evaluate the triggering or otherwise valid strict room without letting stale `m.direct` suppress it, while preserving the explicit-DM and group-room protections from prior fixes.

## Reproduction Assessment

Yes, source-reproducible: current main skips joined-room discovery whenever a mapped strict room exists, which directly matches the old-mapping/new-strict-room scenario. I did not run a live Matrix homeserver repro during this read-only review.

## Solution Assessment

Yes, the right direction is a Matrix-plugin fix, but not a broad return to treating every 2-member room as a DM. The safer solution is per-room/repair-aware discovery and promotion with targeted regression tests that preserve the earlier anti-false-positive behavior.
