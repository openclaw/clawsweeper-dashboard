## Summary

Keep open: current main still has the Matrix direct-room discovery short-circuit that matches the report, and an open PR already references this issue with closing syntax, so the issue should stay open until that PR lands or is replaced.

## What This Changes

Matrix per-room DM handling should recognize newer strict DM rooms even when older m.direct mappings for the same user still exist.

## Best Possible Solution

Land or revise the linked Matrix plugin fix so direct inspection and repair keep joined-room discovery alive without restoring a broad “any two-member room is a DM” fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure m.direct with an older strict mapped room and join a newer strict 1:1 room for the same user; current main will not call getJoinedRooms, so the newer room is not surfaced.

## Solution Assessment

Yes, the narrow fix direction is to change Matrix direct inspection/repair and add stale-mapped-plus-newer-strict regression coverage. The issue should remain open while https://github.com/openclaw/openclaw/pull/79526 is reviewed because that PR is the current implementation candidate.
