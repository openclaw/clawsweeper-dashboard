## Summary

Keep open: current main still has a source-visible Matrix direct-room path where an older strict `m.direct` mapping suppresses joined-room discovery, matching the report's stale-mapping/new-DM-room failure mode.

## What This Changes

Make Matrix per-room DM routing and direct repair handle multiple strict DM rooms with the same user without stale `m.direct` mappings hiding newer rooms.

## Best Possible Solution

Matrix should make per-room DM routing and direct repair evaluate the triggering or otherwise valid strict room without letting stale `m.direct` suppress it, while preserving the explicit-DM and group-room protections from prior fixes.

## Reproduction Assessment

Yes, source-reproducible: current main skips joined-room discovery whenever a mapped strict room exists, which directly matches the old-mapping/new-strict-room scenario. I did not run a live Matrix homeserver repro during this read-only review.

## Solution Assessment

Yes, the right direction is a Matrix-plugin fix, but not a broad return to treating every two-member room as a DM. The safer solution is per-room and repair-aware discovery/promotion with targeted regression tests that preserve the earlier anti-false-positive behavior.
