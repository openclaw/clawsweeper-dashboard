Adds compact plugin health to `/status`, detailed `/status plugins`, typed channel setup diagnostics, and SQLite-backed cross-process context-engine and tool-schema quarantine health.

## Best Possible Solution

Key persisted health to a verifiable process incarnation, reconcile tool quarantine records against each current tool projection so successful schemas clear immediately, preserve or explicitly approve `/status` argument compatibility, and prove the recovery paths after rebasing.

Yes for the review defects from source: record a context failure under a PID later reused by another live process, or record a tool quarantine and correct the schema while keeping the plugin loaded; the current listing paths continue reporting those records.

## Solution Assessment

No; centralizing health formatting is sound, but PID-only liveness and a 24-hour recovery TTL do not represent current health accurately, and broad `/status` interception is not required to deliver plugin diagnostics.
