## Summary

Keep this PR open for repair: the status feature is useful and its live output demonstrates the intended behavior, but the context-engine health mirror has a cross-process lost-update race and introduces a file-backed runtime-state path where the repository requires canonical SQLite storage.

## What This Changes

The PR adds compact and detailed plugin-health reporting to status commands, including plugin diagnostics, channel failures, compatibility notices, and persisted runtime quarantines.

## Best Possible Solution

Persist both context-engine and tool-schema quarantine health through the shared SQLite keyed store, preserve bounded per-process cleanup, and render the status summaries from that single canonical state owner.

## Reproduction Assessment

Yes for the patch defect: two processes can read the same JSON snapshot, append different records, and each rename its result, leaving only the last writer's record. This follows directly from the changed read-modify-write path, although the read-only review did not execute a race harness.

## Solution Assessment

No. Surfacing plugin health is a reasonable direction, but a parallel JSON state file is less safe and maintainable than extending the existing SQLite-backed keyed-store pattern already used by this PR.
