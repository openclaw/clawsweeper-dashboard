## Summary

Keep open. The PR targets a real current-main crash gap, but the latest head still broadens file-watcher ENOSPC handling enough to suppress unrelated failures before merge.

## What This Changes

This PR adds a file-watch ENOSPC transient classifier, a builtin memory-core chokidar error listener, focused tests, and a changelog entry.

## Best Possible Solution

Merge a narrowed version that handles confirmed inotify/file-watch exhaustion in memory-core and the global fallback, with negative tests for generic ENOSPC watcher labels and non-transient chokidar errors.

## Reproduction Assessment

Yes. Current main lacks both the memory watcher error listener and file-watcher ENOSPC transient classification, and the PR body gives a concrete Linux inotify-limit reproduction path.

## Solution Assessment

No. Handling ENOSPC near the memory watcher plus a narrow global fallback is the right direction, but this patch should require explicit file-watch exhaustion signals and preserve visibility for unrelated watcher failures.

- **[P2] Tighten ENOSPC watcher matching:** `src/infra/unhandled-rejections.ts:394-397`
  - body: At the process-wide unhandled-rejection gate, `code === "ENOSPC"` plus a message containing the generic word `watcher` returns true, and the new test locks `watcher error: ENOSPC` as transient. A disk-full write failure wrapped by any watcher helper would now be treated as non-fatal, so require explicit inotify/file-watcher limit wording and add a negative test for generic watcher labels.
  - confidence: 0.86
- **[P2] Gate watcher degradation to exhaustion errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:449-454`
  - body: The new listener logs and swallows every chokidar `error` event. Chokidar emits `error` for EPERM/EACCES and other failures by default, so permission or config problems can now disable memory auto-sync without a visible failure; gate degraded mode to confirmed file-watch exhaustion and keep other watcher errors surfaced.
  - confidence: 0.76
