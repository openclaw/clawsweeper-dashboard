## Summary

Keep open: the PR addresses a real current-main gap, but the latest head still broadens fatal-error suppression beyond confirmed inotify/file-watcher exhaustion.

## What This Changes

The PR adds a file-watcher ENOSPC transient classifier, a memory-core chokidar error listener, watcher/unhandled-rejection tests, and a changelog entry.

## Best Possible Solution

Keep the PR open and narrow both the global classifier and memory-core listener so only confirmed file-watch exhaustion degrades, while unrelated watcher or storage failures remain fatal.

## Reproduction Assessment

Yes. Current main lacks both the memory watcher error listener and file-watcher ENOSPC transient classification, and the PR body gives a concrete Linux inotify-limit reproduction path; the PR regression can be checked with the watcher mock by emitting non-transient errors.

## Solution Assessment

No. Handling ENOSPC near the memory-core watcher is the right direction, but the patch should gate degraded mode on a narrow transient predicate and keep non-transient chokidar errors visible.

- **[P2] Preserve fatal handling for non-transient watcher errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:449-453`
  - body: The new listener logs and returns for every chokidar `error` event.   - confidence: 0.9

Since this classifier is used process-wide, require an explicit inotify/file-watcher exhaustion phrase and add a negative test for code ENOSPC plus an unrelated watcher label.
  - confidence: 0.78
