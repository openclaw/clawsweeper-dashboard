## Summary

Keep open. The crash scenario is real on current main and the PR targets the right memory-core and unhandled-rejection surfaces, but the latest diff still downgrades every memory watcher error and keeps a process-wide ENOSPC classifier broad enough to suppress unrelated failures.

## What This Changes

Adds file-watch ENOSPC transient classification, memory-core watcher error logging/degraded mode, regression tests, and a changelog entry.

## Best Possible Solution

Revise the PR so only confirmed inotify/file-watch exhaustion degrades, while unrelated watcher and storage failures remain fatal or explicitly visible with negative regression tests.

## Reproduction Assessment

Yes. The Linux inotify repro in the PR body is concrete, and current main visibly lacks both the memory watcher error listener and the file-watch transient classifier; the PR regressions can be reproduced with focused watcher mock and classifier unit tests.

## Solution Assessment

No. Handling ENOSPC near the memory-core watcher plus a narrow fallback classifier is maintainable, but this patch should gate both paths on explicit file-watch exhaustion evidence before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve fatal handling for non-transient watcher errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:449-454`
  - body: The new listener logs and returns for every chokidar `error` event. Chokidar 5 emits `error` for EPERM/EACCES and other non-ENOENT/ENOTDIR failures when `ignorePermissionErrors` is false, so permission or invalid-path failures now silently degrade memory auto-sync instead of surfacing. Gate this listener to confirmed file-watch exhaustion and keep other watcher errors fatal.
  - confidence: 0.9
- **[P2] Require explicit watcher exhaustion evidence:** `src/infra/unhandled-rejections.ts:368-396`
  - body: `hasFileWatchSignal` accepts the bare word `watcher`, and the test locks in `code: "ENOSPC"` plus `watcher error: ENOSPC` as transient. Because this classifier is process-wide, a generic wrapper around a disk-full write can be suppressed as transient. Require an inotify/file-watcher-limit phrase or source-specific handling, and add a negative test for the generic watcher label.
  - confidence: 0.84
