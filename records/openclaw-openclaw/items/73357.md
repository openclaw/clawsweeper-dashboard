## Summary

Keep open: the reported crash path is valid, but the latest PR still broadens non-fatal handling too far by downgrading all memory-core watcher errors and by accepting a generic watcher signal in the process-wide ENOSPC classifier.

## What This Changes

The PR adds file-watcher ENOSPC transient classification, a memory-core chokidar error listener, regression tests, and a changelog entry.

## Best Possible Solution

Narrow the memory-core listener and global classifier to confirmed file-watch resource exhaustion, while preserving fatal behavior for permission, invalid-path, and unrelated storage failures.

## Reproduction Assessment

Yes. I did not run the sysctl scenario in this read-only pass, but current main lacks both the watcher error listener and file-watch ENOSPC transient classification, and chokidar's source shows watcher errors can escape through the `error` event path.

## Solution Assessment

No. Handling ENOSPC near the memory-core watcher is the right direction, but the PR should gate degraded mode on a narrow file-watch exhaustion predicate and keep unrelated watcher/storage errors fatal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve fatal handling for non-transient watcher errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:449-454`
  - body: The new listener logs and returns for every chokidar `error` event. Chokidar emits `error` for EPERM/EACCES and other non-ENOENT/ENOTDIR failures when `ignorePermissionErrors` is false, so permission or invalid-path failures now silently degrade memory auto-sync instead of surfacing as fatal. Gate this listener to confirmed inotify/file-watch exhaustion and keep other watcher errors fatal.
  - confidence: 0.9
- **[P2] Require explicit file-watch exhaustion signals:** `src/infra/unhandled-rejections.ts:370`
  - body: `hasFileWatchSignal` still accepts the bare word `watcher`, so a process-wide unhandled rejection with `code: "ENOSPC"` and a generic message like `watcher error: ENOSPC` is treated as transient. Since ENOSPC also means ordinary disk-full failures, require an explicit inotify/file-watcher exhaustion phrase and make bare watcher-plus-ENOSPC a negative test.
  - confidence: 0.78
