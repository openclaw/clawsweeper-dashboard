## Summary

Keep open: the gateway crash bug is real, but the current patch is not merge-ready because it still downgrades unrelated chokidar errors and broad ENOSPC-plus-watcher messages to non-fatal behavior.

## What This Changes

The PR adds file-watch ENOSPC transient classification, a memory-core chokidar error listener, watcher/classifier tests, and a changelog entry.

## Best Possible Solution

Narrow both the global classifier and memory-core watcher listener to confirmed file-watch resource exhaustion, while preserving fatal handling for permission, invalid-path, and unrelated storage failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main has no memory watcher error listener and only network/SQLite transient unhandled-rejection classification, so a chokidar file-watch ENOSPC can still reach the fatal unhandled-rejection path; I did not run the sysctl scenario.

## Solution Assessment

No. Handling ENOSPC near the memory watcher and global fallback is the right direction, but this patch should only degrade confirmed inotify/file-watch exhaustion and keep unrelated chokidar errors fatal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve fatal handling for non-transient watcher errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:449-454`
  - body: The new listener logs and returns for every chokidar `error` event. Chokidar 5 emits `error` for EPERM/EACCES and other non-ENOENT/ENOTDIR failures when `ignorePermissionErrors` is false, so permission or invalid-path failures now silently degrade memory auto-sync instead of surfacing. Gate this listener with a boundary-safe file-watch exhaustion predicate and preserve fatal behavior for everything else.
  - confidence: 0.9
- **[P2] Require explicit file-watch exhaustion signals:** `src/infra/unhandled-rejections.ts:370`
  - body: `hasFileWatchSignal` still accepts the bare word `watcher`, so a process-wide unhandled rejection with `code: "ENOSPC"` and a generic message like `watcher error: ENOSPC` is treated as transient. Since ENOSPC also covers ordinary disk-full failures, require an explicit inotify/file-watcher limit phrase and make bare watcher-plus-ENOSPC a negative test.
  - confidence: 0.82
