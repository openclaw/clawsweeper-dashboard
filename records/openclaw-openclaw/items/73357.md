## Summary

Keep open: the crash report is valid and the local watcher handler is the right seam, but the PR still globally suppresses no-code disk-full messages and misses the required changelog entry.

## What This Changes

The PR adds a memory-core chokidar error listener, a transient file-watch error classifier for ENOSPC/inotify failures, and unit coverage for the classifier.

## Best Possible Solution

Land a narrowed version that keeps the builtin watcher error handler, gates global storage-exhaustion classification on watcher/inotify context, adds negative tests, records the user-facing fix in the changelog, and leaves gateway watch-default policy to #71335.

## Reproduction Assessment

Yes. Current main opens a builtin memory chokidar watcher without an error listener and exits for non-network/non-SQLite unhandled rejections; the lowered-inotify-limit plus memory_search flow in the PR body is a coherent reproduction path.

## Solution Assessment

No, not as-is. The local chokidar error handler is the narrow maintainable seam, but the global fallback classifier still needs watcher context for message-only storage exhaustion text and the changelog requirement is unmet.
