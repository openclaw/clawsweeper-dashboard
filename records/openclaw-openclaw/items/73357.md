## Summary

Keep open. Current main still lacks graceful handling for the reported builtin memory watcher ENOSPC path, and this PR has the right local handler and changelog entry but still over-broadly suppresses generic message-only watcher errors in the global unhandled-rejection path.

## What This Changes

This PR adds builtin memory-core chokidar error logging, extends transient unhandled-rejection classification for file-watcher resource errors, adds unit coverage, and records the fix in the changelog.

## Best Possible Solution

Land the local builtin memory watcher error handler and changelog entry with a narrower global classifier that only suppresses file-watcher resource-exhaustion/watch-limit failures, leaving gateway-mode watch defaults to #71335.

## Reproduction Assessment

Yes. On current main, lowering Linux inotify watches and triggering `memory_search` reaches `MemoryManagerSyncOps.ensureWatcher()` with no watcher `error` listener, and non-network/non-SQLite ENOSPC-style failures are not treated as transient.

## Solution Assessment

No, not as-is. The local chokidar handler is the right maintainable seam, but the global fallback should require resource-exhaustion/watch-limit evidence instead of treating generic file-watcher wording as transient.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate generic watcher messages on resource exhaustion:** `src/infra/unhandled-rejections.ts:403-405`
  - body: The message-only branch returns `true` for any string containing `file watcher` or `watcher error`, even when it lacks ENOSPC, `no space left on device`, `inotify`, or watch-limit wording. Since this feeds the global unhandled-rejection suppression path, an unrelated `file watcher failed` or `watcher error: EACCES` rejection would now be logged and ignored instead of surfacing. Keep the positive `System limit for number of file watchers reached` case, but gate generic watcher text on resource-exhaustion terms and flip the broad positive test to negative coverage.
  - confidence: 0.89
