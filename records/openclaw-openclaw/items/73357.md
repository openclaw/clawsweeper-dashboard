## Summary

Keep this PR open. The branch addresses a real current-main memory watcher crash path, but it is not merge-ready because the new global transient classifier still suppresses message-only disk-full errors without watcher context, and the user-facing fix needs a changelog entry.

## What This Changes

The PR adds a builtin memory chokidar error listener, a global transient file-watch error classifier for ENOSPC/inotify watcher failures, and unit tests for the classifier.

## Best Possible Solution

Land a narrowed version that keeps the local builtin watcher error handler, requires watcher/inotify context before any global ENOSPC or disk-full suppression, adds a user-facing changelog bullet, and either explicitly defers QMD parity to a linked follow-up or handles it in the same reviewed scope.

## Reproduction Assessment

Yes. Current main defaults memory search watching to true, `memory_search` opens a non-transient `MemoryIndexManager`, the constructor arms a chokidar watcher without an `error` listener, and the global unhandled-rejection handler exits for non-network/non-SQLite ENOSPC failures.

## Solution Assessment

No, not as-is. The local chokidar `error` handler is the right narrow seam, but the global fallback classifier must not treat plain disk-full text as a watcher failure, and the user-facing fix needs the required changelog entry before merge.
