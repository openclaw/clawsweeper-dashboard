## Summary

Keep open: the underlying ENOSPC/inotify crash is valid, but the current patch is not merge-ready because it downgrades every memory-core chokidar error to a warning, not only transient watcher-limit exhaustion.

## What This Changes

The PR adds a memory-core chokidar error listener, a global transient file-watcher error classifier, regression tests, and a changelog entry for ENOSPC/inotify watcher exhaustion.

## Best Possible Solution

Restrict memory watcher degradation to confirmed inotify/file-watch exhaustion and preserve fatal surfacing for non-transient watcher errors, with positive and negative tests around both paths.

## Reproduction Assessment

Yes. Current main has no memory watcher error listener and no file-watcher ENOSPC transient classification, and the PR body provides a concrete Linux inotify-limit reproduction path; I did not run the sysctl reproduction in this read-only review.

## Solution Assessment

No. The local handler is the right owner-module seam, but it should branch on a narrow transient watcher predicate and keep non-transient chokidar errors fatal rather than logging every error.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve fatal handling for non-transient watcher errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:449-454`
  - body: The new `error` listener logs and returns for every chokidar error. Chokidar emits `error` for EPERM/EACCES and other non-ENOENT/ENOTDIR failures when `ignorePermissionErrors` is false, so a permission/config/storage failure in the memory watcher now only logs a warning and leaves auto-sync degraded instead of surfacing a fatal error. Gate this handler with a narrow transient watcher predicate, without crossing the plugin/core boundary, and keep non-transient errors fatal.
  - confidence: 0.88
