## Summary

Keep open. The PR is aimed at a real current-main crash path, but the latest head still globally suppresses message-only disk-full text as a transient watcher error and is missing the required user-facing changelog entry.

## What This Changes

The PR adds a memory-core chokidar error listener, a new transient file-watch error classifier for unhandled rejections, and unit coverage for ENOSPC/inotify watcher cases.

## Best Possible Solution

Land a narrowed version that keeps the local memory watcher error handler, requires watcher or inotify context before global ENOSPC/no-space suppression, adds the missing negative tests and changelog entry, and leaves gateway-mode sync.watch defaults or broader QMD parity to separate linked work.

## Reproduction Assessment

Yes. The code-level reproduction is high-confidence: current main defaults memory search watching to true, arms the memory-core chokidar watcher without an error listener, and any escaped non-network/non-SQLite ENOSPC watcher rejection reaches the fatal unhandled-rejection path.

## Solution Assessment

No, not as-is. The local chokidar error handler is the right narrow seam, but the global fallback classifier must be tightened so plain disk-full text is not swallowed, and the user-facing behavior change needs changelog coverage.
