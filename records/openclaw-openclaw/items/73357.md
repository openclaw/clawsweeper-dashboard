## Summary

Keep open: the crash report is valid and current main still lacks the watcher hardening, but this PR would still broaden global unhandled-rejection suppression to plain disk-full message strings and is missing the required changelog entry.

## What This Changes

The PR adds builtin memory-core chokidar error logging plus a global transient file-watch error classifier and unit tests for ENOSPC/inotify watcher failures.

## Best Possible Solution

Land a narrowed version that keeps the local chokidar error handler, requires watcher/inotify context for global file-watch transient suppression, adds the no-code disk-full negative test, adds the required `CHANGELOG.md` fix entry, and either includes QMD parity or records it as an explicit follow-up.

## Reproduction Assessment

Yes. The source path is high-confidence: default memory sync watch is true, non-transient builtin managers call `ensureWatcher()`, current main has no watcher `error` listener or file-watch transient classifier, and chokidar emits `error` events; live sysctl exhaustion was not run in this read-only review.

## Solution Assessment

No, not as-is. The local watcher handler is the right narrow seam, but the global classifier must not suppress message-only disk-full strings without watcher/inotify context, and the user-facing fix needs a changelog entry.
