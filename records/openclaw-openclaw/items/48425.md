## Summary

Keep open: the sessions.json bloat bug is valid and current main still writes persisted `skillsSnapshot` metadata, but this PR needs a scoped serialization fix because the current global replacer can drop unrelated nested data named `skillsSnapshot`; it also still needs real behavior proof before merge.

## What This Changes

The PR adds a session-store JSON.stringify replacer used by session-store writes and disk-budget sizing to omit `skillsSnapshot` while preserving `systemPromptReport`, plus serialization tests.

## Best Possible Solution

Keep the write-path size reduction, but scope omission to the intended `SessionEntry.skillsSnapshot` field, keep disk-budget sizing aligned with the final serializer, preserve `systemPromptReport`, add coverage for nested `backendExtras.skillsSnapshot`, and attach redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on current main, `saveSessionStore` normalizes only `skillsSnapshot.resolvedSkills` and then serializes the full store with `JSON.stringify(store, null, 2)`, so entries containing `skillsSnapshot.prompt` and `skillsSnapshot.skills` remain on disk.

## Solution Assessment

No. The write-path omission is the right direction, but the current global key-name replacer is too broad; it should be constrained to the session-entry field so arbitrary nested persisted data named `skillsSnapshot` is not lost.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope skillsSnapshot stripping to session entries:** `src/config/sessions/store-serialization.ts:10`
  - body: This replacer runs for every property under the session store, so any nested persisted key named `skillsSnapshot` is dropped too. For example, ACP runtime `backendExtras` accepts arbitrary validated keys and is stored under `entry.acp.runtimeOptions`; a backend option named `skillsSnapshot` would vanish after save/restart. Please constrain the omission to the intended `SessionEntry.skillsSnapshot` field/path and add a regression case for a nested key with the same name.
