## Summary

Keep open: current main still persists `skillsSnapshot` in session entries, so the PR targets a real user-facing bloat bug, but the proposed global key-name replacer can delete legitimate nested persisted data and needs a scoped repair before merge.

## What This Changes

The PR adds a shared session-store serialization replacer and tests so session saves and disk-budget measurements omit `skillsSnapshot` while preserving `systemPromptReport`.

## Best Possible Solution

Use a session-entry-aware serialization helper shared by save and disk-budget code that omits only top-level `SessionEntry.skillsSnapshot`, preserves `systemPromptReport` and nested ACP/plugin data, and adds regression coverage for nested `skillsSnapshot` keys.

## Reproduction Assessment

Yes. Source inspection shows current main can persist `SessionEntry.skillsSnapshot` and stringifies full stores; the PR regression is source-reproducible by serializing an entry with nested `acp.runtimeOptions.backendExtras.skillsSnapshot` through the global replacer.

## Solution Assessment

No. Omitting the top-level session snapshot is the right direction, but a global `key === "skillsSnapshot"` replacer is broader than the persistence contract; scope the omission by session entry/path or serialize a sanitized store copy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Scope the snapshot filter to session entries:** `src/config/sessions/store-serialization.ts:10`
