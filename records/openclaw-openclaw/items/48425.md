## Summary

Keep open. Current main still serializes `SessionEntry.skillsSnapshot` into `sessions.json`, and this PR is aimed at a real performance problem, but the proposed global key-name replacer can drop unrelated persisted data and the user-facing fix still needs a changelog entry.

## What This Changes

This PR adds a shared session-store JSON serialization replacer, applies it to store saves and disk-budget sizing, and adds tests so `skillsSnapshot` is omitted while `systemPromptReport` remains persisted.

## Best Possible Solution

Use a session-entry-aware serialization helper shared by save and disk-budget code that omits only top-level `SessionEntry.skillsSnapshot`, preserves `systemPromptReport` and nested plugin/ACP data, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main bloat path because session saves stringify the full store after agent and auto-reply paths persist `skillsSnapshot`; the PR regression is reproducible by serializing an entry containing `acp.runtimeOptions.backendExtras.skillsSnapshot` through the proposed replacer.

## Solution Assessment

No. Excluding `skillsSnapshot` from disk is the right direction, but a global key-name replacer is broader than the persistence contract; an entry-aware serializer is the narrower maintainable fix.
