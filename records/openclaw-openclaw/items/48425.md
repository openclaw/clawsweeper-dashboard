## Summary

Keep open. Current main still has the session-store bloat path, and this PR is directionally useful, but the proposed global replacer can drop unrelated persisted nested fields named `skillsSnapshot`, so it needs a narrow repair before merge.

## What This Changes

The PR adds a shared session-store JSON serialization replacer, applies it to session saves and disk-budget sizing, and adds serialization tests so persisted stores omit `skillsSnapshot` while preserving `systemPromptReport`.

## Best Possible Solution

Use a shared session-store serialization helper that removes only top-level `SessionEntry.skillsSnapshot` during saves and disk-budget measurement, preserves `systemPromptReport`, and leaves nested ACP/plugin-owned data intact.

## Reproduction Assessment

Yes. Source inspection is enough: current main persists `SessionEntry.skillsSnapshot` and serializes the full store, while the PR regression is reproducible by serializing a store entry with nested `acp.runtimeOptions.backendExtras.skillsSnapshot`.

## Solution Assessment

No, not as written. Omitting the top-level session snapshot is the right fix direction, but the global `key === "skillsSnapshot"` replacer is broader than the persistence contract and should be scoped to session entries or implemented by serializing a sanitized store copy.
