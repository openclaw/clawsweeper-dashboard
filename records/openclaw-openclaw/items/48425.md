## Summary

Keep open: current main still has a real session-store bloat path, and this PR is useful, but the current patch is broader than the persistence contract because its global key-name replacer can drop unrelated nested data named `skillsSnapshot`.

## What This Changes

The PR adds a shared `JSON.stringify` replacer for session-store saves and disk-budget sizing, plus tests, so `skillsSnapshot` is omitted while `systemPromptReport` remains persisted.

## Best Possible Solution

Use a session-entry-aware serialization helper shared by save and disk-budget code that omits only top-level `SessionEntry.skillsSnapshot`, preserves `systemPromptReport` and nested plugin/ACP data, and records the fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows agent runs can persist `skillsSnapshot` and the store/disk-budget paths stringify full entries; the PR regression is source-reproducible by serializing a stored nested `acp.runtimeOptions.backendExtras.skillsSnapshot` key.

## Solution Assessment

No. Excluding the top-level session-entry snapshot is the right direction, but a global key-name replacer is broader than the intended persistence contract; the fix should be scoped to actual session entries.
