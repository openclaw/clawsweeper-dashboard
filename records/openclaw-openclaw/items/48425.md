## Summary

Keep open: the PR targets a real current-main session-store bloat path, but the proposed global key-name filter would silently drop valid nested persisted data named `skillsSnapshot`, so it needs a scoped repair before merge.

## What This Changes

The branch adds a session-store JSON serialization replacer, applies it to store writes and disk-budget sizing, and adds tests around omitting `skillsSnapshot` while preserving `systemPromptReport`.

## Best Possible Solution

Replace the global replacer with an explicit scoped serializer that omits only top-level `SessionEntry.skillsSnapshot`, preserves `systemPromptReport` and nested plugin/ACP data, and reuses the same serialization shape for disk-budget estimates.

## Reproduction Assessment

Yes. Source inspection shows current main still writes full session entries to `sessions.json`, and the PR regression is source-reproducible because the global replacer removes nested persisted keys named `skillsSnapshot`.

## Solution Assessment

No, not as written. The persistence boundary is the right narrow fix, but the serializer must be path-aware or entry-aware instead of filtering every matching key name in the JSON object graph.
