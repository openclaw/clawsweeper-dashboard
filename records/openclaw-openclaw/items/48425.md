## Summary

Keep this PR open because it targets a real session-store bloat path, but the current branch should not merge as-is: current main now treats the lightweight skill snapshot as durable metadata, and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds a shared JSON.stringify replacer to omit `skillsSnapshot` from session-store writes and disk-budget estimates, with serialization tests that preserve `systemPromptReport`.

## Best Possible Solution

Preserve current lightweight `skillsSnapshot` metadata, omit or dedupe only truly transient/bulky data with path-aware serialization, keep disk-budget estimates aligned, and attach redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an entry with `skillsSnapshot.prompt` and `skillsSnapshot.skills` survives current normalization and is written by `JSON.stringify(store, null, 2)`; I did not run a live file-size repro in this read-only review.

## Solution Assessment

No. Dropping the whole key by name is not the narrowest maintainable fix because current main preserves lightweight snapshot metadata, and arbitrary nested persisted data can also legally use the same key name.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve persisted skill snapshot metadata:** `src/config/sessions/store-serialization.ts:10`
  - body: This replacer drops the whole `skillsSnapshot` field during session-store serialization. Current main only treats `resolvedSkills` as runtime-only: tests preserve `prompt`, `skills`, `skillFilter`, and `version`, and `agent-command.ts` uses those fields to skip rebuilding the snapshot when version/filter still match. After save/restart, resumed sessions would lose that metadata and rebuild on every run. Serialize only truly transient data or add a dedupe contract that keeps those invariants.
  - confidence: 0.92
- **[P2] Constrain stripping away from nested persisted options:** `src/config/sessions/store-serialization.ts:10`
  - body: `JSON.stringify` invokes this replacer for every property in the store, not only the top-level `SessionEntry.skillsSnapshot`. ACP `backendExtras` accepts safe arbitrary keys and is persisted under `entry.acp.runtimeOptions`, so a backend option named `skillsSnapshot` would silently disappear after save/restart. If a replacer remains, constrain it to the intended session-entry path and cover a nested same-name key.
  - confidence: 0.9
