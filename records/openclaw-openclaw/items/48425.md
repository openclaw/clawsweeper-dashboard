## Summary

Keep open. The PR targets a real session-store bloat problem, but the patch is not safe to merge as written against current main because it drops the whole `skillsSnapshot` object globally, while current main intentionally preserves lightweight snapshot metadata and only strips the heavy runtime cache; the external PR also still lacks inspectable real behavior proof.

## What This Changes

The PR adds a session-store JSON serialization replacer and tests so saves and disk-budget estimates omit `skillsSnapshot` while preserving `systemPromptReport`.

## Best Possible Solution

Rebase onto current main and use a structured session-store serializer or normalizer that preserves durable `skillsSnapshot` prompt/catalog/version metadata and `systemPromptReport`, drops only transient heavy cache data, and covers plugin/ACP nested-key preservation.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main serializes the normalized store after stripping only `resolvedSkills`, and the PR's `JSON.stringify` replacer is source-reproducible as a global key filter that can drop unrelated nested `skillsSnapshot` keys.

## Solution Assessment

No. A global key-name replacer is not the best fix because it conflicts with current-main durable snapshot metadata and can remove unrelated persisted data; the safer path is a path-aware serializer or the existing normalizer-style approach.
