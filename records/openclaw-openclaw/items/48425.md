## Summary

Keep this PR open. Current main still has the reported sessions.json bloat path, and the PR is a useful focused fix, but the proposed global JSON replacer would drop any persisted nested property named skillsSnapshot rather than only SessionEntry.skillsSnapshot.

## What This Changes

The PR adds a session-store JSON serialization replacer, applies it to store saves and disk-budget sizing, and adds serialization tests so skillsSnapshot is omitted from sessions.json while systemPromptReport is preserved.

## Best Possible Solution

Implement a session-store serialization helper that constructs the on-disk store shape and omits only top-level SessionEntry.skillsSnapshot, while preserving systemPromptReport and all nested plugin/runtime fields. Use that same helper for save and disk-budget measurement, add nested-key regression coverage, and include an Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the main bug path: current runtime writes SessionEntry.skillsSnapshot, and saveSessionStoreUnlocked serializes the full store. The PR regression is also reproducible by serializing an entry with a nested acp.runtimeOptions.backendExtras.skillsSnapshot key through the proposed replacer.

## Solution Assessment

No. Excluding skillsSnapshot from disk is the right direction, but a global key-name replacer is not the safest implementation. The narrower maintainable fix is a session-entry-aware serializer shared by the save and disk-budget paths.
