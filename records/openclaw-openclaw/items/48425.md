## Summary

Keep open: the PR targets a real session-store bloat path that still exists on current main, but the patch is not safe as written because the global `skillsSnapshot` JSON replacer would remove unrelated nested persisted data with the same key.

## What This Changes

The PR adds a shared session-store JSON serialization replacer, applies it to store writes and disk-budget sizing, and adds tests for omitting `skillsSnapshot` while preserving `systemPromptReport`.

## Best Possible Solution

Land a scoped session-store serialization helper that omits only top-level `SessionEntry.skillsSnapshot`, preserves `systemPromptReport` and nested plugin/ACP data, and keeps disk-budget estimates aligned with actual writes.

## Reproduction Assessment

Yes. Source inspection shows current main writes full normalized session entries to `sessions.json`, and the PR regression is source-reproducible by serializing a session entry with nested `acp.runtimeOptions.backendExtras.skillsSnapshot`.

## Solution Assessment

No, not as written. The persistence boundary is the right narrow fix, but the global key-name replacer should be replaced with a path-aware sanitizer or explicit store-copy serializer that only removes the intended session-entry field.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

- **[P2] Scope the snapshot filter to session entries:** `src/config/sessions/store-serialization.ts:10`
