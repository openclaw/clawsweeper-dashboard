## Summary

Keep open: current main and v2026.6.8 have shipped backend and CLI pending-approval diagnostics, but the Control UI nodes view still ignores those fields and can still show a connected node with empty effective Caps/Commands and no approval context.

## What This Changes

Gateway 5.18+ reports connected Windows/Docker nodes with empty Caps/Commands in the Web UI, preventing node command execution where 5.12 worked.

## Best Possible Solution

Keep the shipped backend approval boundary, and add a narrow Control UI nodes-view diagnostic for approvalState, pendingRequestId, pending declared caps/commands, and the approve/reapprove remediation.

## Reproduction Assessment

Yes for the remaining Web UI gap from source: node.list can expose pending approval fields while effective caps/commands stay empty, and the Control UI render path ignores those diagnostic fields. No live Windows/Docker repro was run for post-approval command failures.

## Solution Assessment

No, closing this as implemented would be premature because the shipped backend/CLI fix is only partial for the reported Web UI symptom. The maintainable next step is a UI-only diagnostic display that uses the existing gateway fields without weakening node approval policy.
