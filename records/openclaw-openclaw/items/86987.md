## Summary

Keep open: current main and v2026.6.9 include the backend/CLI pending-approval diagnostics from the merged node diagnostics work, but the Control UI nodes view still renders only effective caps/commands and can still show a paired, connected node with no approval context.

## What This Changes

The issue asks OpenClaw to fix a Gateway 5.18+ regression where Windows/Docker nodes appear connected in the Web UI with empty Caps/Commands and node commands cannot run.

## Best Possible Solution

Keep the shipped approval boundary and add a narrow Control UI nodes diagnostic for approvalState, pendingRequestId, pending declared caps/commands, and approve/reapprove remediation.

## Reproduction Assessment

Yes for the remaining Web UI gap by source inspection: node.list can expose pending approval fields while effective caps/commands stay empty, and the Control UI render path ignores those fields. I did not run the exact Synology Docker plus Windows node repro against current main.

## Solution Assessment

No, closing this as implemented would be premature because the shipped backend/CLI fix is only partial for the reported Web UI symptom. The maintainable fix is to render the existing diagnostics in Control UI without making pending commands effective.
