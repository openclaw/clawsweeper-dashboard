## Summary

Keep open, but do not merge as-is: current main still does not accept `heartbeat: false`, while this PR regresses existing heartbeat defaults, drops the shipped `skipWhenBusy` schema field, conflates explicit `false` with absent heartbeat in prompt/ACP paths, lacks real behavior proof, and is currently conflicting with main.

## What This Changes

The PR adds `heartbeat: false` support to agent config schemas/types and threads that disabled state through heartbeat runtime helpers, prompts, ACP, delivery targets, WhatsApp heartbeat handling, generated schema output, and tests.

## Best Possible Solution

Keep the shipped `0m` disable contract unless maintainers want a boolean shorthand; if accepted, implement `heartbeat: false` through one centralized resolver that distinguishes disabled from absent and preserves current heartbeat fields and precedence.

## Reproduction Assessment

Yes for the review blockers by source inspection: the PR head deterministically changes existing default-inheritance behavior, omits `skipWhenBusy`, and treats literal `false` as absent in prompt/ACP paths. I did not run tests because this was a read-only review.

## Solution Assessment

No. Accepting `heartbeat: false` may be reasonable, but this patch needs a central disabled-state contract, current field preservation, a rebase, and maintainer approval for the new public config syntax.
