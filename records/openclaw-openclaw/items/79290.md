## Summary

Keep open. Current main has active-run data and Sessions-table live badges, but I did not find a persistent per-agent avatar or agent selector indicator across agent/chat switches; this remains a concrete Control UI enhancement that needs UI/product scoping.

## What This Changes

The issue requests a persistent processing indicator on every active agent avatar, independent of the currently selected tab.

## Best Possible Solution

Reuse the existing session active-run fields to derive per-agent processing state, then render it in a deliberate Control UI agent switcher/avatar/status surface coordinated with https://github.com/openclaw/openclaw/issues/68248 and https://github.com/openclaw/openclaw/issues/22774.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence capability gap: live status exists at the session level, but the agent selector/avatar surfaces I checked do not render persistent per-agent processing state.

## Solution Assessment

Unclear. The proposed global map should probably be replaced or backed by the existing `sessionsResult.sessions[].hasActiveRun` data, with maintainers deciding the custom picker/avatar presentation instead of adding another status source.
