## Summary

Keep open: current main has active-run state in session rows, but the Control UI chat agent/session selectors do not persistently expose per-agent processing state, and the active chat stream state is cleared on session switches. This is a real UX enhancement, not an already-shipped fix or duplicate.

## What This Changes

The issue requests a persistent processing indicator on agent avatars for all active agents, independent of the currently selected chat tab/session.

## Best Possible Solution

Derive a per-agent active state from existing sessionsResult/hasActiveRun data and expose it in the chat agent/session UI with a small, accessible indicator, after maintainer UX direction on the exact placement.

## Reproduction Assessment

Yes from source inspection: current main has hasActiveRun on session rows but does not project that state into the chat agent selector or an agent avatar list. I did not run a live multi-agent browser workflow in this read-only review.

## Solution Assessment

Unclear. The proposed global status map is directionally right, but the maintainable implementation should probably reuse existing sessionsResult active-run data first and only add a new state map if the UI cannot subscribe to session changes cleanly.
