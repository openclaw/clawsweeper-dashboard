## Summary

Keep open. Current main already tracks active session runs and shows live status in the Sessions view, but the chat agent/session controls still render plain native select labels with no persistent avatar or processing indicator for background agents; the prior ClawSweeper comment was only a timeout and no canonical duplicate or fix PR surfaced.

## What This Changes

Request a persistent processing indicator on agent avatars for all active agents, including background agents, across chat tab/session switches.

## Best Possible Solution

Add an accessible per-agent or per-session activity affordance in Control UI using the existing `sessions.hasActiveRun` data and session-change events, keeping backend task semantics unchanged unless that data proves insufficient.

## Reproduction Assessment

No high-confidence live reproduction was run. This is an enhancement rather than a broken documented behavior; source inspection confirms current chat controls expose only plain agent/session selectors while active-run state is available elsewhere.

## Solution Assessment

Unclear until maintainers choose the UI surface. The existing `hasActiveRun` session contract is the right source of truth, but the best UI may require replacing or augmenting the native selects rather than only adding status text.
