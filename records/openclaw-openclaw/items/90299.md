## What This Changes

Ensure Agent Teams subagent completion events do not report plain lost-context failure when a child result was captured or delivered.

## Best Possible Solution

Fix the existing lifecycle/announcement path so the lost-context fallback reconciles child session output/status first and reports either a successful completion or a degraded warning with the preserved result, backed by focused regression coverage.

## Reproduction Assessment

## Solution Assessment

Yes: a focused lifecycle/announce repair is the best path, not a broad shared-memory or orchestration layer. The fix should preserve true failures while avoiding plain failed status when a useful child result is the available terminal evidence.
