## Summary

Keep open: current main and the latest release still contain the stale active subagent lost-context fallback, while the remaining fix is already owned by open linked pull requests that need maintainer boundary selection.

## What This Changes

The issue reports that Agent Teams can deliver child output while marking the same subagent completion as `subagent run lost active execution context`.

## Best Possible Solution

Land or adapt the narrow transcript-backed recovery so the sweeper reports success only for a genuine current-run terminal child result and preserves lost-context failure for non-terminal, stale, or unreadable children.

## Reproduction Assessment

Yes, at source level: current main can reach the stale active-run lost-context fallback after session reconciliation returns `null`, while the announce path can still read child output for that failed outcome. I did not run a live Agent Teams race in this read-only review.

## Solution Assessment

Unclear which open branch should land, but broad display-output recovery is not the best boundary. The safest fix is current-run terminal transcript recovery or an equivalent explicit terminal-success signal.
