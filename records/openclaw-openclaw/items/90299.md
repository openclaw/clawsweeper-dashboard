## Summary

Keep open: current main and the latest release still retain the stale subagent lost-context fallback, and two linked open pull requests are already competing candidate fixes for the remaining lifecycle boundary.

## What This Changes

The issue reports Agent Teams delivering child output from a subagent while marking that same completion as lost active execution context.

## Best Possible Solution

Land or adapt the narrow current-run terminal transcript recovery so the sweeper reports success only for a genuine terminal child result and preserves lost-context failure for non-terminal, stale, or unreadable children.

## Reproduction Assessment

Yes, source-reproducible: current main can reach the stale active-run lost-context fallback after session reconciliation returns null, while the announce path can still read child output for that failed outcome. I did not run a live Agent Teams race in this read-only review.

## Solution Assessment

Unclear which open PR should land, but broad display-output recovery is not the best boundary. The safer direction is current-run terminal transcript recovery or an equivalent explicit terminal-success signal.
