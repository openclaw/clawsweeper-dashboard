## Summary

Keep open for maintainer review: the latest PR head avoids the earlier same-content collision by keeping `turnId` in the scope, but that also means re-mirrored prior content under a later turn still gets a new idempotency key.

## What This Changes

This PR changes the Codex app-server transcript mirror to key idempotency by a SHA-256 role/content fingerprint and updates mirror tests for snapshot reordering and repeated same-content turns.

## Best Possible Solution

Use a stable logical-message identity that survives re-mirroring and snapshot reordering while still distinguishing separate same-content turns, then cover the rotated-scope and repeated-content cases together.

## Reproduction Assessment

Yes for the PR's remaining gap: the PR head still prefixes the fingerprint with a turn-scoped `idempotencyScope`, so the same logical message re-emitted under a later turn produces a different key. I did not live-run WebChat in this read-only review.

## Solution Assessment

No. Role/content fingerprints help snapshot positional shifts, but keeping `turnId` in the scope cannot satisfy the rotated-scope part of the linked report; the fix needs a stable discriminator that does not collapse repeated same-content turns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
