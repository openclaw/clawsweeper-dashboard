Existing channel-bound installations can throw here after upgrade until a new inbound activity refreshes metadata, so add a fallback/backfill or migration coverage before requiring it.
  - confidence: 0.91

  - body: This always includes `graphChannelId`, even when `activity.channelData?.channel?.id` is absent, and the branch merge helper spreads incoming fields over the existing record. A sparse activity can therefore overwrite previously captured channel metadata with `undefined`; only include the field when present and preserve it across merges.
  - confidence: 0.86
- **[P1] Keep target parsing behind plugin contracts:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: Core command/binding parsing should resolve provider-specific target grammar through loaded or registered channel plugin contracts, but this imports private bundled extension modules directly into a core channel path. That bypasses the plugin boundary and can pull channel-owned code into hot paths; use the current loaded-plugin contract or an approved public seam instead.
  - confidence: 0.88
