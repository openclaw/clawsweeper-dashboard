Full review comments:

- **[P1] Keep Teams Graph operations on the state store:** `extensions/msteams/src/graph-messages.ts:2`
  - body: This branch imports the legacy file-backed conversation store for Graph message operations, but current main has moved Teams conversations to the plugin-state store and monitor/send-context already use that state store. If this lands as-is, upgraded users can have sends and monitor writes in SQLite/plugin state while these Graph operations look in the old JSON store and fail to find conversations.
  - confidence: 0.88
- **[P1] Migrate channel metadata before requiring it:** `extensions/msteams/src/graph-messages.ts:139-142`
  - body: The new channel Graph path throws whenever a stored channel conversation lacks `graphChannelId`, but existing persisted Teams channel records predate this field. Add a migration/backfill or compatibility fallback before making that metadata mandatory, otherwise existing bound channel conversations can start failing after upgrade.
  - confidence: 0.86
- **[P1] Preserve graphChannelId on sparse activities:** `extensions/msteams/src/monitor-handler/message-handler.ts:288`
  - body: This upsert writes `graphChannelId` even when `activity.channelData.channel` is absent, and the merge helper does not preserve that field. A sparse activity can erase a previously captured Graph channel id and later cause the new required-metadata check to fail.
  - confidence: 0.84
- **[P1] Rebase onto the loaded channel target contract:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: The PR adds a core target parser with direct imports from bundled extension internals, but current main has moved this compatibility path behind loaded channel plugin contracts and command binding now resolves through the active plugin channel registry. Reintroducing this stale parser bypasses the current owner boundary and risks conflicting target behavior.
  - confidence: 0.82
- **[P1] Do not publish Teams internals through the SDK facade:** `src/plugin-sdk/msteams.ts:111-118`
  - body: These exports promote Teams send/edit/delete/channel-edit implementation helpers into a public SDK facade so plugins can drive host-credential Teams operations directly. That API and security boundary needs a maintainer-approved contract or should stay behind a generic lane/runtime capability instead.
  - confidence: 0.8
