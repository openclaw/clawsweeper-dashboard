Full review comments:

- **[P1] Keep target parsing on the plugin registry seam:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: These static imports pull private bundled extension modules into a core channel parsing path. Current main routes explicit target parsing through loaded plugin contracts, and the package excludes extension dist trees, so this can reintroduce startup/package breakage and bypass external plugin ownership.
  - confidence: 0.88
- **[P1] Do not export private Teams runtime files from the SDK:** `src/plugin-sdk/msteams.ts:112-119`
  - body: The public SDK facade now re-exports implementation functions from `extensions/msteams/src/**`, but those extension dist files are excluded from the core package. Please expose a package-safe runtime seam or keep this plugin-local; otherwise `openclaw/plugin-sdk/msteams` can point at modules that are not shipped with core.
  - confidence: 0.86
- **[P1] Add an upgrade path before requiring graphChannelId:** `extensions/msteams/src/graph-messages.ts:143-147`
  - body: Existing Teams channel conversation records will not have the newly added `graphChannelId`, but this path now throws for channel Graph operations when it is missing. Add a migration/backfill or safe fallback so existing bound Teams channels do not lose get/pin/react/search behavior after upgrade.
  - confidence: 0.9
- **[P2] Preserve stored graphChannelId on sparse upserts:** `extensions/msteams/src/monitor-handler/message-handler.ts:289`
  - body: This upsert writes `graphChannelId` even when `activity.channelData?.channel?.id` is absent, and the store merge helper does not preserve that field. A sparse activity can erase previously captured channel metadata and make later Graph operations fail the new required-metadata check.
  - confidence: 0.89
