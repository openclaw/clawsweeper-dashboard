- **[P1] Preserve stored Teams channel metadata on upgrade:** `extensions/msteams/src/graph-messages.ts:139-143`
  - body: This path now throws when an existing channel conversation record lacks `graphChannelId`, but that field is introduced by this PR and older stored records will not have it.   - confidence: 0.9

  - body: `activity.channelData?.channel?.id` can be undefined, and the store merge path does not preserve this field. A sparse activity can therefore erase a previously known Graph channel id and make later channel Graph operations fail.
  - confidence: 0.86

  - body: Core command/binding parsing should not statically import private bundled extension parser modules. Current main routes this through loaded channel/plugin contracts, while these imports can pull plugin-owned code into core paths and drift from the active channel contract.
  - confidence: 0.88
- **[P1] Do not publish Teams SDK helpers backed by plugin internals:** `src/plugin-sdk/msteams.ts:112-118`
  - body: These exports make `openclaw/plugin-sdk/msteams` depend on private `extensions/msteams/src/**` implementations while the core package excludes Teams extension files. Keep the surface plugin-owned or add a maintainer-approved public contract with packaging proof.
