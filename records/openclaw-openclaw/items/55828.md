## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Add an upgrade fallback before requiring graphChannelId:** `extensions/msteams/src/graph-messages.ts:139-142`
  - body: Existing `msteams-conversations.json` channel records were created before `graphChannelId` existed, but this path now throws for any stored channel conversation missing it. After upgrade, plugin Graph actions using a normal `conversation:<id>` target can fail until another inbound channel message refreshes the store; fall back to the stored channel conversation id when usable or add a backfill before failing.
  - confidence: 0.9
- **[P1] Keep target parsing behind the plugin contract boundary:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: This core parser now statically imports private `extensions/*/src/**` modules for Discord, Teams, and Telegram. That bypasses the plugin boundary and can pull bundled channel internals into command/binding paths; expose a light public parser artifact or keep using the registered plugin contract instead.
  - confidence: 0.86
- **[P2] Preserve graphChannelId during sparse conversation upserts:** `extensions/msteams/src/monitor-handler/message-handler.ts:288`
  - body: The upsert payload always includes `graphChannelId`, even when `activity.channelData?.channel?.id` is absent. Sparse Teams activities can therefore overwrite a previously captured channel id with `undefined`, causing later Graph actions to hit the new missing-metadata error; only include the field when present and preserve it in the store merge helper.
  - confidence: 0.82
