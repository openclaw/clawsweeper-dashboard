## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve existing channel conversation metadata:** `extensions/msteams/src/graph-messages.ts:140-143`
  - body: This branch makes channel Graph operations throw when a stored channel conversation lacks `graphChannelId`, but that field is newly introduced here. Existing `msteams-conversations.json` channel entries will not have it, so bound channel operations can start failing after upgrade until a new inbound message refreshes the store; add a migration/backfill or a safe fallback path before requiring the field.
  - confidence: 0.87
- **[P1] Keep target parsing behind the plugin contract:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: Core target parsing now statically imports private `extensions/*/src` parser modules for Discord, Teams, and Telegram. Current main uses loaded plugin contracts for this path, and the scoped guides keep core out of extension internals; this can eagerly load bundled plugin code, bypass plugin disablement, and create parser drift. Move these parsers behind a public/light plugin contract or loaded-plugin lookup.
  - confidence: 0.85
- **[P2] Avoid clearing graphChannelId on sparse activities:** `extensions/msteams/src/monitor-handler/message-handler.ts:288`
  - body: The upsert payload always includes `graphChannelId`, even when `activity.channelData.channel` is absent, and this branch's merge helper does not preserve an existing value. A later sparse activity can overwrite the known Graph channel id with `undefined`, causing the new required-metadata check to fail for future channel Graph operations.
  - confidence: 0.84
