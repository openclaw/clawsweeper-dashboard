Full review comments:

- **[P1] Preserve old channel records before requiring graphChannelId:** `extensions/msteams/src/graph-messages.ts:140-143`
  - body: Existing Teams conversation-store records from current main do not have graphChannelId, but this branch throws for stored channel conversations that lack it. Upgraded channel-bound conversations can fail Graph message operations until a new inbound activity refreshes the record, so add a fallback/backfill or migration coverage before making the field required.
  - confidence: 0.91
- **[P2] Avoid erasing graphChannelId on sparse updates:** `extensions/msteams/src/monitor-handler/message-handler.ts:288`
  - body: This upsert includes graphChannelId even when activity.channelData.channel is absent, and the branch merge helper does not preserve that field. A sparse activity can drop known channel metadata and break later Graph operations, so only write the field when present and preserve it across merges.
  - confidence: 0.86
- **[P1] Keep target parsing behind plugin contracts:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: This core helper now imports private bundled channel parsers directly. Current main resolves explicit targets through loaded/registered plugin messaging contracts, so keep Teams normalization behind a public lightweight plugin seam or the existing registered contract instead of pulling extension internals into core.
  - confidence: 0.88
