  - body: This makes existing stored Teams channel conversations fail Graph operations after upgrade because older `msteams-conversations.json` records will have `teamId` but not the newly added `graphChannelId`.   - confidence: 0.9

  - body: This always includes `graphChannelId` in the upsert payload, even when Teams did not send `channelData.channel`; the store merge then spreads that undefined value over the previous record. A later Graph channel operation can fail because a sparse activity erased the only known channel id.
  - confidence: 0.86

  - body: These top-level imports make core command/binding parsing depend on extension-private parser modules instead of the loaded channel plugin contract.
