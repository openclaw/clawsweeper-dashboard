Full review comments:

- **[P1] Preserve channel metadata for existing Teams records:** `extensions/msteams/src/graph-messages.ts:140-143`
  - body: This branch treats every stored channel conversation as requiring `graphChannelId`, but that field is newly added. Existing channel bindings imported from older Teams conversation state will hit this throw for Graph message, pin, react, and search operations until a new inbound message refreshes the store. Add a doctor/backfill path or a compatible lookup before requiring the field.
  - confidence: 0.9
- **[P2] Avoid clearing graphChannelId on sparse activities:** `extensions/msteams/src/monitor-handler/message-handler.ts:288`
  - body: `graphChannelId` is always present in the upsert payload, even as `undefined`. The merge helper spreads incoming fields after preserving only timezone, so a sparse Teams activity without `channelData.channel` can erase a previously stored `graphChannelId` and later make Graph channel actions fail. Only include the field when present or preserve it in the merge helper.
  - confidence: 0.87
- **[P2] Move parser fallbacks behind public channel seams:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: Core target parsing now statically imports private extension `src` modules for Discord, Teams, and Telegram. That violates the plugin boundary and couples a core command path to bundled plugin internals; expose narrow plugin-owned parser artifacts or rely on registered messaging parsers instead.
  - confidence: 0.82
