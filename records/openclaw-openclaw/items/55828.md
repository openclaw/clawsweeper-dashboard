Full review comments:

- **[P1] Preserve channel metadata for stored Teams conversations:** `extensions/msteams/src/graph-messages.ts:139-143`
  - body: Existing msteams conversation-store records were written before graphChannelId existed. Requiring it here makes stored channel Graph operations throw after upgrade until a fresh inbound message rewrites the store, so this needs a migration/backfill or compatible lookup path before merge.
  - confidence: 0.88
- **[P2] Do not erase stored graphChannelId on sparse upserts:** `extensions/msteams/src/monitor-handler/message-handler.ts:288`
  - body: This always passes graphChannelId from optional channelData, and the merge helper only preserves timezone from the existing entry. A sparse activity can therefore clear the previously captured channel id and break later channel Graph calls.
  - confidence: 0.84
- **[P2] Keep target parsing behind the loaded channel contract:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: These top-level core imports reach into private extension source for Discord, Teams, and Telegram parsers. That bypasses the loaded/disabled plugin contract and eagerly pulls channel internals into command/binding paths; current main's loaded target parser contract is the safer shape to rebase onto.
  - confidence: 0.82
- **[P1] Do not publish a Teams SDK facade backed by plugin internals:** `src/plugin-sdk/msteams.ts:111-118`
  - body: The new public openclaw/plugin-sdk/msteams subpath exports helpers directly from extensions/msteams/src. Current main excludes dist/extensions/msteams from the root package and ships Teams as @openclaw/msteams, so the exported subpath either forces root to ship plugin internals again or points at files absent from the installed package.
  - confidence: 0.8
