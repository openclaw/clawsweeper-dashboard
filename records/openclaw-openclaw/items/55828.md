Full review comments:

- **[P1] Preserve old channel records before requiring graphChannelId:** `extensions/msteams/src/graph-messages.ts:140-143`
  - body: This field is introduced by the PR, but older stored Teams channel conversations only have the Bot Framework conversation metadata and `teamId`.   - confidence: 0.9
- **[P2] Avoid erasing graphChannelId on sparse activity upserts:** `extensions/msteams/src/monitor-handler/message-handler.ts:287-289`
  - body: This upsert always includes `graphChannelId`, even when `activity.channelData.channel.id` is absent. Because the PR's merge helper does not preserve `graphChannelId`, a sparse activity can overwrite a previously captured value with `undefined` and make later channel Graph calls fail; omit the field when absent or preserve it during merge.
  - confidence: 0.88
- **[P1] Keep target parsing behind channel plugin contracts:** `src/channels/plugins/target-parsing.ts:5-7`
  - body: Core target parsing now statically imports private extension `src` parsers for Discord, Teams, and Telegram. That bypasses the loaded channel plugin contract and hot-path laziness, so disabled or externalized channels can still be pulled into core command/binding paths and parser behavior can drift from `messaging.parseExplicitTarget`; route through a lightweight public/loaded plugin seam instead.
