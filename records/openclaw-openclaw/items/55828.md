  - body: These top-level imports pull core target parsing directly into bundled extension `src/**` modules.   - confidence: 0.91
- **[P1] Migrate channel metadata before requiring it:** `extensions/msteams/src/graph-messages.ts:140-143`
  - body: This makes every stored Teams channel conversation require `graphChannelId`, but existing `msteams-conversations.json` entries created before this PR do not have that field. After upgrade, bound channel Graph operations can throw until a new inbound channel message refreshes the store; add a doctor/backfill migration or other explicit upgrade path before runtime requires it.
  - confidence: 0.9
- **[P1] Preserve graphChannelId on sparse activities:** `extensions/msteams/src/monitor-handler/message-handler.ts:288`
  - body: The upsert payload always includes `graphChannelId`, even when `activity.channelData?.channel?.id` is absent. Because the store merge spreads incoming fields, sparse activities can overwrite a previously captured channel id with `undefined`, causing later Graph operations to hit the new missing-metadata error.
  - confidence: 0.89
- **[P1] Do not publish Teams internals through the SDK facade:** `src/plugin-sdk/msteams.ts:111-118`
  - body: These new exports turn bundled Teams send/edit/delete/channel-edit implementation helpers into plugin SDK surface. That creates a public contract and a security boundary around host-backed Teams operations without the broader API decision, docs, or compatibility plan; keep this plugin-local or expose a maintainer-approved generic channel seam.
  - confidence: 0.84
