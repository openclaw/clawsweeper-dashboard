## Summary

Keep open. Current main still lacks a native `openclaw doctor --dry-run` contract, and the related global dry-run issue plus the open `--check-config` PR cover different scopes rather than superseding this doctor repair-preview request.

## What This Changes

The issue requests a doctor-specific `--dry-run` mode that previews validation and repairs without writing config or performing other doctor side effects.

## Best Possible Solution

Add a native doctor dry-run contract that is registered, typed, documented, and honored by every doctor mutation path while keeping #41418 and #47302 separate.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a missing CLI safety feature. Source inspection gives a high-confidence verification path because the flag is not registered, not typed, not documented, and cannot guard current doctor mutation paths.

## Solution Assessment

Yes. A doctor-owned dry-run is the narrowest maintainable solution because doctor owns config repair, plugin install repair, cron normalization, and gateway service repair behavior that a global agent dry-run or deep config check would not cover.

## Repair Work Prompt

Implement `openclaw doctor --dry-run` as a doctor-specific no-mutation preview mode. Register the CLI flag and thread a typed `dryRun` option through the doctor prompter/repair-mode/config/health contribution flows. In dry-run, collect and render the same validation and repair recommendations without prompting, writing config, saving cron stores, installing/updating plugins, repairing sandbox registries/images, staging/installing/restarting gateway services, or mutating other doctor-owned state; return a defined non-zero result when repairable issues are found. Do not implement the global agent tool dry-run from #41418 or the deep `--check-config` feature from #47302. Add focused regression coverage for CLI option mapping, config finalization/write suppression, cron, plugin install repair, gateway service/daemon repair, and docs; add a user-facing changelog entry.
