## Summary

Keep open. Current main still lacks a native doctor dry-run mode, and the existing doctor repair flow can still prompt, write config, save cron state, install plugins, or repair services without a dry-run guard. A new related PR now attempts the feature, but it is open and only closes a newer duplicate, so this issue should remain available as the broader acceptance target.

## What This Changes

The issue requests a native `openclaw doctor --dry-run` mode that previews doctor validation and repair actions without writing config or performing other doctor side effects.

## Best Possible Solution

Land a doctor-scoped dry-run contract that registers and types the flag, threads it through prompter/config/health flows, suppresses prompts and all write/repair side effects, returns a defined status when repairable issues are found, and updates docs plus focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives a high-confidence absence proof: there is no doctor dry-run flag, option type, docs entry, or side-effect guard.

## Solution Assessment

Yes. A native doctor-scoped dry-run is the narrow maintainable direction here; the global agent dry-run issue and the open `--check-config` PR cover different layers, while https://github.com/openclaw/openclaw/pull/79734 should be reviewed against this issue's broader side-effect requirements.
