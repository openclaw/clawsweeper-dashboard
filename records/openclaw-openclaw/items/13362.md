## Summary

Keep open. Current main still lacks a native `openclaw doctor --dry-run` mode, and the related implementation PR at https://github.com/openclaw/openclaw/pull/79734 is still open with review blockers rather than merged or shipped.

## What This Changes

Add a `openclaw doctor --dry-run` CLI mode that previews doctor repair changes without prompts, config writes, plugin installs, state rewrites, or service repairs.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79734 with a doctor-scoped read-only planning mode that previews the same repairs as `--fix`, suppresses every doctor side effect, defines exit behavior, and updates docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high-confidence absence proof: no doctor flag, option type, docs entry, or dry-run guard exists.

## Solution Assessment

Yes. A native doctor-scoped dry-run is the narrow maintainable direction here; https://github.com/openclaw/openclaw/issues/41418 is a different global agent dry-run layer and https://github.com/openclaw/openclaw/pull/47302 is a separate `--check-config` proposal.
