## Summary

Keep open. Current main still lacks a doctor-scoped dry-run flag and the doctor repair flow still has config-write, plugin-install, cron-save, and service-repair paths that would need an explicit no-write preview contract.

## What This Changes

The issue requests an `openclaw doctor --dry-run` mode that validates and previews doctor repairs without writing config or making runtime repair changes.

## Best Possible Solution

Add a native doctor dry-run contract that registers the flag, threads it through doctor options/prompter/config and health contributions, renders planned repairs, suppresses writes/runtime repairs, returns a defined non-zero result when issues are found, and updates docs/changelog/tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested `doctor --dry-run` flag and propagation path.

## Solution Assessment

Yes. A doctor-scoped dry-run is the narrowest maintainable solution because the related global tool dry-run issue and open `--check-config` PR cover different layers and do not prevent doctor repair side effects.
