## Summary

Keep open. Current main still has no `doctor --dry-run` flag or dry-run state threaded through doctor repair/write paths, and the related global dry-run issue plus open `--check-config` PR cover different scopes.

## What This Changes

The issue requests a native `openclaw doctor --dry-run` mode that previews doctor validation and repair changes without prompting or writing.

## Best Possible Solution

Add a doctor-scoped dry-run contract that registers the flag, threads it through doctor options/prompter/config and health contributions, renders the planned repairs, suppresses all doctor writes and runtime repairs, returns a defined non-zero status when issues are found, and updates docs/changelog.

## Reproduction Assessment

Not applicable as a feature request. Source inspection still gives high confidence that current main lacks the requested flag and propagation path.

## Solution Assessment

Yes. A native doctor-scoped dry-run is the narrowest maintainable fix because the existing related work targets either global agent tool execution or deep config validation, not doctor repair side-effect preview.
