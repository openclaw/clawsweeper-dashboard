## Summary

Keep open. Current main still has no native `openclaw doctor --dry-run` flag, and the doctor flow still has config-write, cron-store, plugin-install, and service-repair paths that would need an explicit read-only contract.

## What This Changes

Add a native `openclaw doctor --dry-run` mode that previews doctor validations and recommended repairs without writing config or performing repair side effects.

## Best Possible Solution

Implement a focused doctor dry-run contract that registers and types the flag, threads it through doctor repair/config/health flows, renders the same repair plan without prompts or writes, returns a defined non-zero status when issues are found, and documents the new CLI behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence absence proof: current main has no doctor dry-run flag, type, docs entry, or side-effect guard.

## Solution Assessment

Yes, the requested native doctor dry-run is the right scoped solution. The global agent dry-run issue and the `--check-config` PR cover different layers, so this should stay a dedicated doctor CLI feature.
