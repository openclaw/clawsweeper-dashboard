## Summary

Keep open. Current main still does not implement or document `openclaw doctor --dry-run`, and the related implementation PR at https://github.com/openclaw/openclaw/pull/79734 is still open with review blockers rather than merged or shipped.

## What This Changes

Add a doctor-scoped `--dry-run` flag that previews validation and repair changes without writing config, state, plugin installs, or service changes.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79734 with a whole-doctor read-only planning mode that previews the same repairs as `--fix`, blocks writes and service/plugin side effects, defines exit behavior, and updates docs/tests.

## Reproduction Assessment

## Solution Assessment

Yes, a native doctor-scoped dry-run is the narrow maintainable direction. The active PR needs correction or replacement because dry-run must model the whole doctor repair plan without side effects, not only suppress one config write path.
