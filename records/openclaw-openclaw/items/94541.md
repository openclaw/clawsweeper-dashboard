## Summary

## What This Changes

The PR changes fetch-guard so trusted-env-proxy NO_PROXY matches get direct pinned fetches with relaxed private-network SSRF policy, plus SSRF guard tests for private, localhost, and non-NO_PROXY targets.

## Best Possible Solution

If maintainers approve NO_PROXY local access, implement exact-host direct trust that preserves metadata/link-local blocking, update web_fetch docs, add focused regression coverage, and require real private/local web_fetch proof; otherwise keep the strict fallback and supersede the candidate branches.
