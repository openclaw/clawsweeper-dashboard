## What This Changes

The PR changes `src/infra/net/fetch-guard.ts` so trusted-env-proxy NO_PROXY matches get an effective policy with `allowPrivateNetwork: true` and direct dispatch, plus SSRF tests for private, localhost, and non-NO_PROXY targets.

## Best Possible Solution

If maintainers approve NO_PROXY local access, implement scoped exact-host direct trust in `fetch-guard` that preserves metadata/link-local blocking, update docs, add focused regression coverage, and require real private/local `web_fetch` proof.

## Reproduction Assessment

## Solution Assessment

No. `fetch-guard` is the right layer, but full private-network allow is too broad for the stated security rationale; scoped exact-host trust that preserves metadata/link-local checks is the safer maintainable fix.
