## Summary

This can close because current main and the latest listed release both have `commands.list` restored as a registered Gateway RPC with operator-read scope, protocol schemas, validators, UI callers, and regression coverage; the 2026.4.25 package described in the report may have been affected, but the current supported implementation no longer matches that failure.

## What This Changes

The issue asks OpenClaw either to restore the documented Gateway RPC `commands.list` or to remove/update the protocol documentation and changelog with a migration path.

## Best Possible Solution

Keep the restored `commands.list` RPC, schemas, docs, and tests aligned in the Gateway protocol surface; no migration note is needed for current supported releases unless maintainers want to document the historical 2026.4.25 gap.

## Reproduction Assessment

No high-confidence current-main reproduction remains: source inspection shows the method is listed, scoped, wired into `coreGatewayHandlers`, implemented, typed, and tested. I did not install the historical 2026.4.25 package in this read-only review, so the proof here is that current main and `v2026.5.7` no longer reproduce the reported absence.

## Solution Assessment

Yes. Restoring and keeping the documented RPC live is the narrowest compatibility-preserving outcome, and the current implementation matches the documented operator helper contract.
