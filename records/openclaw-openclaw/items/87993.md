## Summary

Keep open: current main and v2026.6.8 still use `/End` without a package-update autostart suspension for Windows Scheduled Tasks, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to prevent legacy Windows Scheduled Task `PT3M` logon repetition from relaunching the gateway during `openclaw update` package replacement.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/88292 with an update-scoped fix that suspends only enabled Windows Scheduled Task autostart during package replacement and reliably resumes it afterward.

## Reproduction Assessment

Yes, source-reproducible. Current main stops the managed service before package install, and the Windows Scheduled Task stop implementation uses `/End` without disabling legacy repeated triggers; I did not run a native Windows update in this read-only review.

## Solution Assessment

Yes in direction. The update-scoped suspend/resume approach in https://github.com/openclaw/openclaw/pull/88292 is narrower than changing generic gateway stop semantics, but this issue should stay open until that PR merges or is explicitly replaced.
