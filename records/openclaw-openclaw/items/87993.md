## Summary

Keep open: current main and v2026.6.8 still stop the Windows Scheduled Task with `/End` only during package updates, while the open closing PR is the active implementation path rather than a merged fix.

## What This Changes

The issue reports that Windows `openclaw update` can leave a legacy PT3M Scheduled Task trigger enabled, allowing the gateway to relaunch during package replacement.

## Best Possible Solution

Land or replace the open update-scoped fix that suspends only enabled Windows Scheduled Task autostart during mutable package updates and restores it afterward.

## Reproduction Assessment

Yes. Current main stops the managed service before package install and the Windows stop implementation ends the Scheduled Task without disabling a legacy repeated trigger; I did not run a native Windows update in this read-only pass.

## Solution Assessment

Yes in direction. The update-scoped disable/re-enable shape in https://github.com/openclaw/openclaw/pull/88292 is narrower than changing generic gateway stop semantics, but this issue should stay open until that PR is merged or explicitly abandoned.
