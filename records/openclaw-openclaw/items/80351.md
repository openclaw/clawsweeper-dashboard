## What This Changes

The issue requests fixing Windows `claude-cli` launches so the supervised CLI backend can resolve npm-installed Claude `.cmd` shims or the underlying executable instead of failing with `spawn claude ENOENT`.

## Best Possible Solution

Keep the remaining engineering and review on https://github.com/openclaw/openclaw/pull/68819 or an equivalent core process patch that resolves npm-style Windows `.cmd` shims without unsafe shelling and adds focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main passes bare `claude` through the supervisor child path and the Windows shim helper does not resolve that command. A real Windows gateway run would be final runtime proof, but the reported logs match the current source path and Node's documented Windows `.cmd` behavior.
