## Summary

Keep open: this is a plausible agents auth-order improvement, but the branch is not merge-ready because it lacks real behavior proof and currently regresses against newer current-main auth candidate semantics.

## What This Changes

The PR adds a Fisher-Yates shuffle for spawned embedded-runner auth profile candidates and a new shuffle-focused Vitest file.

## Best Possible Solution

Move spawned-run candidate ordering into a production-owned helper that disperses only eligible automatic candidates while preserving locked, provider-preferred, plugin-harness, explicit-order, and cooldown contracts, then cover that path with focused regression tests and real concurrent-run proof.

## Reproduction Assessment

Yes for the source-level ordering path: current main builds ordered `profileCandidates` and the auth controller starts from index 0. No live provider-side 429 cascade was reproduced or proven by the PR materials.

## Solution Assessment

No. Randomizing spawned candidates is a plausible direction, but this patch needs to preserve current-main auth-order contracts and test the production selection path rather than a copied helper.
