## Best Possible Solution

Implement spawned-run candidate dispersion in a production-owned helper that preserves locked, provider-preferred, plugin-harness, explicit-order, and cooldown contracts, then cover that production path and provide real concurrent-run proof.

## Reproduction Assessment

Yes for the source-level ordering path: current main builds a deterministic candidate array and the auth controller starts from index 0. No high-confidence live provider-side 429 cascade reproduction is present in the PR materials.

## Solution Assessment

No. Randomizing spawned candidates is plausible, but this patch shuffles the whole candidate list and tests a copied helper instead of preserving current auth-order anchors and exercising production ordering.
