## Summary

Keep open because the concurrent subagent auth-profile herd problem still appears unsolved on current main, but this branch is not merge-ready: it conflicts with the current runner layout, lacks real behavior proof, tests a copied helper, and randomizes a compatibility-sensitive auth order.

## What This Changes

This PR adds a Fisher-Yates shuffle for non-locked subagent auth profile candidates in the old PI embedded runner and adds a small shuffle-focused test file.

## Best Possible Solution

Reimplement the spread behavior on the current embedded-runner auth candidate path while preserving user-locked profiles and explicit auth.order semantics, then add current-path tests and redacted real concurrency proof.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows current main still builds deterministic profile candidates for concurrent runs, but the PR does not include a real concurrent subagent run or logs showing the reported 429 herd.

## Solution Assessment

No. Shuffling the whole resolved profile order is too broad for current auth semantics; the safer fix is a current-path helper that only spreads eligible automatic subagent candidates while preserving explicit and user-locked choices.
