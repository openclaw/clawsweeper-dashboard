## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` config surface and still has source paths that can widen a restrictive `plugins.allow`; the linked fix PR #76085 remains open with closing syntax, so this issue should stay open until that or an equivalent fix lands.

## What This Changes

The issue requests an opt-in `plugins.bundledMode: "respect-allow"` mode so bundled runtime provider paths honor `plugins.allow` and per-plugin enablement instead of force-loading bundled provider plugins.

## Best Possible Solution

Land #76085 or an equivalent default-preserving change that adds the config/schema/help/tests and gates bundled runtime provider discovery plus activation by allowlist in the new opt-in mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a restrictive `plugins.allow` can still be widened by setup provider discovery plus `withActivatedPluginIds`, and the requested opt-in config key is absent from the strict schema.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in mode is narrower than globally changing bundled allowlist semantics, while broader manifest-first runtime-loader cleanup can remain follow-up work.
