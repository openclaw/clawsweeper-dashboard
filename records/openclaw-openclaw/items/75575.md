## Summary

Keep open. Current main still has the runtime bundled-provider compatibility paths and no `plugins.bundledMode` config key, and open PR #75644 references this issue as the implementation candidate, so the issue should remain open until that PR is resolved.

## What This Changes

Request a default-preserving `plugins.bundledMode: "respect-allow"` opt-in so runtime web-search, web-fetch, and LLM provider paths honor `plugins.allow` and per-plugin enabled flags instead of force-loading bundled provider plugins.

## Best Possible Solution

Land a reviewed fix through PR #75644 or an equivalent narrow PR that adds the config/schema/docs contract, wires `respect-allow` through the runtime provider paths, enables the intended runtime loader cache behavior, and covers the opt-in with focused plugin runtime/config tests while preserving the default compatibility mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and trace the current runtime provider paths, which still inject bundled compatibility IDs while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in is narrower than changing all bundled allowlist semantics globally, while the larger manifest-only runtime-loader cleanup can remain separate follow-up work.
