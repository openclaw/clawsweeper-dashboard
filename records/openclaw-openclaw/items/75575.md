## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` config surface and still has runtime provider/web-provider paths that can expand bundled plugins outside a restrictive `plugins.allow`; the linked open fix PR https://github.com/openclaw/openclaw/pull/76085 has closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request an opt-in `plugins.bundledMode: "respect-allow"` so runtime provider and web provider paths honor `plugins.allow` plus per-plugin disablement for bundled plugins while preserving default compat behavior.

## Best Possible Solution

Land a reviewed default-compatible config/schema/docs/test change that gates bundled runtime discovery and activation in the new mode while leaving compat behavior unchanged by default.

## Reproduction Assessment

Yes, at source level. With a restrictive `plugins.allow`, setup discovery currently admits bundled provider plugins, then `withActivatedPluginIds` adds those IDs to `plugins.allow`; current strict schema has no `plugins.bundledMode` opt-in to change that behavior.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in mode is narrower than globally changing bundled allowlist semantics, while the broader manifest-first runtime-loader cleanup can remain a follow-up architecture effort.
