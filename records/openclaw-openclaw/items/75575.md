## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` surface and still has source-level paths that can expand bundled provider plugins outside a restrictive `plugins.allow`; #76085 is an open linked implementation candidate, so this should stay open until that or an equivalent fix merges.

## What This Changes

Add an opt-in `plugins.bundledMode: "respect-allow"` mode so runtime provider, web-search, and web-fetch plugin paths honor `plugins.allow` and per-plugin disablement instead of compat-loading bundled providers.

## Best Possible Solution

Review and land #76085 or an equivalent default-preserving config/schema/docs/test change, then close after the merged code proves bundled runtime provider discovery respects the opt-in allowlist mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: restrictive `plugins.allow` can still be widened by provider setup discovery plus `withActivatedPluginIds`, and the requested strict config key is absent from the schema.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in mode is narrower than globally changing bundled allowlist semantics, while the larger manifest-first runtime-loader cleanup can remain follow-up work.
