## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` config surface and retains the bundled-provider compatibility paths, while open PR #75644 is already the active implementation candidate.

## What This Changes

Add an opt-in `plugins.bundledMode: "respect-allow"` mode so bundled runtime provider paths honor `plugins.allow` and per-plugin enablement.

## Best Possible Solution

Land a reviewed fix through PR #75644 or an equivalent narrow PR that adds schema/docs support, preserves the default compatibility mode, wires `respect-allow` through the runtime paths, and covers the behavior with focused plugin runtime tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and trace the current runtime provider paths, which still inject bundled compatibility IDs while the strict schema has no `plugins.bundledMode` key.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in is narrower than making `plugins.allow` globally authoritative immediately, while broader manifest-first runtime cleanup can remain a separate architecture follow-up.
