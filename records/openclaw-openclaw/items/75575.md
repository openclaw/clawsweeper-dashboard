## Summary

Keep open. Current main still has the bundled-provider compatibility paths described in the report, there is no existing open duplicate for the proposed `plugins.bundledMode` surface, and the issue is concrete enough for a focused fix PR rather than cleanup closure.

## What This Changes

The issue requests an opt-in `plugins.bundledMode: "respect-allow"` setting so bundled runtime web-search, web-fetch, and LLM provider paths honor `plugins.allow` and per-plugin enablement instead of force-loading bundled provider contracts.

## Best Possible Solution

Implement a backwards-compatible `plugins.bundledMode` config contract with default `compat`, make `respect-allow` disable bundled compat overrides for web-search, web-fetch, and LLM provider runtime paths, engage the existing loader cache safely, and document/test the opt-in while leaving the larger manifest-only runtime architecture as separate follow-up work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by using a restrictive `plugins.allow` and inspecting the runtime resolver options: current main still augments/forces bundled provider compat in the cited paths, even though the control-plane docs describe `plugins.allow` as exclusive.

## Solution Assessment

Yes for a near-term fix. A default-preserving opt-in is narrower and safer than changing all `plugins.allow` semantics globally; the longer-term manifest-only enumeration cleanup should not block this smaller user-facing switch.
