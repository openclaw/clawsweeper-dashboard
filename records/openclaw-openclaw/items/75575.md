## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` config surface and still has bundled-provider compatibility paths that can bypass the visible plugin allowlist; open PR #75644 is the active implementation candidate, so this issue should remain open until that PR lands or is closed/replaced.

## What This Changes

The issue requests a default-preserving `plugins.bundledMode: "respect-allow"` opt-in so runtime web-search, web-fetch, and LLM provider paths honor `plugins.allow` and `plugins.entries.<id>.enabled`, with web-provider runtime cache reuse enabled.

## Best Possible Solution

Land PR #75644 or an equivalent narrow fix that adds the config/schema/docs contract, threads the opt-in through all runtime provider paths, enables the intended web-provider cache behavior, and adds focused regression coverage while preserving the default compatibility mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and inspect the current runtime provider paths, which still apply bundled compatibility overrides while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term fix. A default-preserving opt-in is narrower than changing all bundled allowlist semantics globally, while a larger manifest-only runtime-loader cleanup can remain separate follow-up work.
