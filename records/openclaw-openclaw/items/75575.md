## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` config surface and still has runtime provider paths that can expand bundled plugins beyond a restrictive `plugins.allow`; the linked fix PR #76085 is open with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request an opt-in `plugins.bundledMode` mode so runtime provider, web search, and web fetch paths respect plugin allowlists and per-plugin disablement instead of force-loading bundled provider plugins.

## Best Possible Solution

Land a reviewed default-preserving fix, likely #76085 or equivalent, that adds config/schema/docs/tests and gates bundled runtime discovery and activation without changing compatibility mode for existing installs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: use a restrictive `plugins.allow`, then trace setup provider discovery into `withActivatedPluginIds`, which adds the discovered bundled provider ids to the allow set before runtime loading.

## Solution Assessment

Yes for the near-term direction. A default-preserving `respect-allow` opt-in is narrower than globally changing bundled plugin semantics, while leaving the deeper manifest-first runtime-loader cleanup as follow-up work.
