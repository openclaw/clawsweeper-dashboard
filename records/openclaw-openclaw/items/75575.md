## Summary

Keep open. Current main still lacks the requested config surface and keeps the bundled-provider compatibility paths, and open PR #75644 is already the active implementation candidate linked with closing syntax.

## What This Changes

Request an opt-in `plugins.bundledMode: "respect-allow"` so runtime provider, web-search, and web-fetch plugin loading honors `plugins.allow` and disabled entries while preserving compatibility by default.

## Best Possible Solution

Land PR #75644 or an equivalent narrow fix that adds the schema/help contract, gates the runtime compat paths behind `respect-allow`, preserves `compat` as the default, and includes focused runtime/config coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and inspect the current runtime provider paths, which still inject bundled compatibility behavior while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in is narrower than changing bundled allowlist semantics globally, while the larger manifest-only runtime-loader cleanup can remain separate follow-up work.
