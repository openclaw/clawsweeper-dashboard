## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` config surface and still has runtime provider paths that expand bundled plugins outside a restrictive `plugins.allow`; the linked fix PR #76085 is open with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Request an opt-in `plugins.bundledMode: "respect-allow"` mode so runtime provider, web-search, and web-fetch plugin loading honors bundled plugin allowlist and entry enablement without changing the default compat behavior.

## Best Possible Solution

Land a reviewed fix, likely #76085 or an equivalent patch, that adds the config/schema/docs/tests and gates bundled runtime discovery/activation while preserving default compatibility mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow`, observe that current setup provider discovery admits bundled plugins and `withActivatedPluginIds` activates the discovered set, while the strict schema has no `plugins.bundledMode` opt-in.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in mode is narrower than changing bundled allowlist semantics globally, with deeper manifest-first runtime-loader cleanup left as follow-up work.
