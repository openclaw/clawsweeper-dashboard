## Summary

Keep open: current main still lacks `plugins.bundledMode` and the runtime paths still widen bundled provider/web-provider activation through compatibility code; open PR #75644 is the active implementation candidate.

## What This Changes

Request an opt-in `plugins.bundledMode` setting that makes bundled provider, web-search, and web-fetch runtime loading honor `plugins.allow` and per-plugin enablement while preserving default compatibility behavior.

## Best Possible Solution

Land a reviewed fix through PR #75644 or an equivalent narrow PR that adds the schema/help contract, wires `respect-allow` through the runtime provider/web-fetch/web-search paths, preserves default compatibility mode, and covers the behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and trace the current runtime provider paths, which still inject bundled compatibility IDs while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in is narrower than changing all bundled allowlist semantics globally, while the larger manifest-only runtime-loader cleanup can remain separate follow-up work.
