## Summary

Keep open. Current main still has the bundled-provider compatibility paths and no `plugins.bundledMode` config key, and open PR #75644 is the active implementation path, so the issue should remain open until that PR is resolved.

## What This Changes

Request an opt-in `plugins.bundledMode: "respect-allow"` mode so bundled web-search, web-fetch, and LLM provider runtime paths honor `plugins.allow` and per-plugin disabled entries while preserving default compatibility behavior.

## Best Possible Solution

Ship a reviewed, default-preserving implementation through PR #75644 or an equivalent narrow PR that adds the schema/help contract, gates the affected runtime provider paths, enables the intended loader cache behavior, and covers the opt-in with focused config and plugin-runtime tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and inspect the current runtime provider paths, which still inject bundled compatibility IDs while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term feature direction. A default-preserving opt-in is narrower than changing bundled allowlist semantics globally, while the larger manifest-first runtime-loader cleanup can remain separate follow-up work.
