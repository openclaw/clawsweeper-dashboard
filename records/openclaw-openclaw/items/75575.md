## Summary

Keep open. Current main still lacks the requested `plugins.bundledMode` config surface and retains the bundled-provider compatibility paths, while open PR #75644 is the active implementation candidate referenced with closing syntax.

## What This Changes

Add an opt-in `plugins.bundledMode: "respect-allow"` so bundled runtime provider paths honor `plugins.allow` and per-entry disablement instead of force-loading bundled providers.

## Best Possible Solution

Land PR #75644 or an equivalent narrow change that adds the config schema/types/docs contract, preserves the default compatibility mode, wires `respect-allow` through web-search, web-fetch, and LLM provider runtime paths, and covers the behavior with focused plugin runtime/config tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and trace the current runtime provider paths, which still inject bundled compatibility IDs while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in is narrower than changing bundled allowlist semantics globally, while the larger manifest-only runtime-loader cleanup can remain separate follow-up work.
