## Summary

Keep open. Current main still has the bundled runtime compatibility paths and no `plugins.bundledMode` config key, and open PR #75644 is already the referenced implementation candidate, so this issue should stay open until that PR is reviewed, merged, or closed.

## What This Changes

Request to add a default-compatible `plugins.bundledMode: "respect-allow"` option that makes web-search, web-fetch, and LLM provider runtime paths honor `plugins.allow` and `plugins.entries.<id>.enabled`, with runtime loader caching enabled for the affected web-provider paths.

## Best Possible Solution

Land a reviewed fix through PR #75644 or an equivalent narrow PR that adds the config/schema/docs contract, wires the opt-in through the runtime provider and web extraction paths, enables the intended runtime loader cache behavior, and covers the default compatibility mode plus `respect-allow` with focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and inspect the current runtime provider paths, which still inject bundled compatibility ids while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term shape. A default-preserving opt-in is a narrow maintainable fix compared with globally changing bundled allowlist semantics; the larger manifest-only runtime loader cleanup can remain a separate architecture follow-up.
