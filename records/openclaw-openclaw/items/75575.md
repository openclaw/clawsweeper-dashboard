## Summary

Keep open. Current main still lacks the requested config surface and retains the bundled-provider compatibility paths, while open PR #75644 is the active implementation candidate.

## What This Changes

The issue requests an opt-in `plugins.bundledMode: "respect-allow"` mode so runtime provider and web tool loading honors `plugins.allow` and per-plugin enablement for bundled plugins.

## Best Possible Solution

Land a reviewed fix through PR #75644 or an equivalent narrow PR that adds the config/schema/docs contract, wires `respect-allow` through the runtime provider paths, preserves the default compatibility mode, and covers it with focused plugin runtime/config tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a restrictive `plugins.allow` and trace the current runtime provider paths, which still inject bundled compatibility IDs while the strict schema has no `plugins.bundledMode` escape hatch.

## Solution Assessment

Yes for the near-term direction. A default-preserving opt-in is narrower than changing all bundled allowlist semantics globally, while the larger manifest-only runtime-loader cleanup can remain separate follow-up work.
