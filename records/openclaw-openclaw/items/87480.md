## Summary

Keep open: the PR is not merge-ready because it changes the legacy Anthropic SDK client path, while the reported embedded gateway path uses the OpenClaw Anthropic transport and guarded fetch path; the timeout tuning also risks worsening stale socket reuse and bypassing configured proxy routing.

## What This Changes

## Best Possible Solution

Rewrite the fix around a bounded retry-once for replayable pre-response `UND_ERR_SOCKET`/`fetch failed` failures in the guarded Anthropic transport path, preserving proxy-aware dispatcher behavior and adding focused regression plus redacted gateway proof.

## Reproduction Assessment

Yes for the source-level fallback behavior: current main routes embedded Anthropic Messages through `buildGuardedModelFetch`, which rethrows fetch errors into the fallback loop. I did not live-reproduce the multi-hour stale-socket trigger from the reporter's machine.

## Solution Assessment

No. The maintainable fix is a bounded retry in the guarded provider transport before model fallback, not extending idle pooling in a different Anthropic SDK client path.

  - body: The linked production issue is in the embedded gateway path: current main maps `anthropic-messages` to `createAnthropicMessagesTransportStreamFn`, which uses `buildGuardedModelFetch` and currently rethrows `[model-fetch]` errors before fallback.   - body: Undici's `keepAliveTimeout` is the idle time before a socket is closed, and server keep-alive hints can override it up to `keepAliveMaxTimeout`.   - confidence: 0.87
- **[P1] Preserve the proxy-aware dispatcher:** `src/llm/providers/anthropic.ts:850`
  - body: Passing a bare `Agent` through `fetchOptions.dispatcher` overrides undici's global dispatcher for these SDK requests. OpenClaw's gateway bootstrap installs the proxy-aware global dispatcher and docs promise normal runtime HTTP egress uses the configured proxy, so this can route Anthropic traffic around operator proxy/managed CA settings unless the dispatcher is built through the existing proxy-aware path.
  - confidence: 0.84
