- **[P1] Wire the fix into the embedded transport path:** `src/llm/providers/anthropic.ts:826-834`
  - body: The linked failure is from the embedded Anthropic path: its logs use `[model-fetch]`, and current main routes `anthropic-messages` through `createAnthropicMessagesTransportStreamFn()` and `buildGuardedModelFetch()`.   - confidence: 0.92
- **[P1] Do not lengthen stale socket retention:** `src/llm/providers/anthropic.ts:827-829`
  - body: Undici's `keepAliveTimeout` is the idle socket timeout, defaulting to 4s. The reported failure is stale idle socket reuse after the peer/network closes connections, so setting 60s with a 10-minute max keeps sockets eligible for reuse longer instead of evicting them before the peer does. Use a retry-once or keep-alive-safe strategy instead.
  - confidence: 0.87
- **[P1] Preserve the proxy-aware dispatcher:** `src/llm/providers/anthropic.ts:853`
  - body: Passing a bare `Agent` as `fetchOptions.dispatcher` overrides the dispatcher passed to fetch. That bypasses OpenClaw's existing env-proxy, managed TLS, and guarded dispatcher policy behavior for the SDK path, so proxy/TLS operators can fail and provider egress loses the security boundary this transport normally preserves.
  - confidence: 0.86
