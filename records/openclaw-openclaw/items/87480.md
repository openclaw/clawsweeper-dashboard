## Summary

Keep open: the linked provider bug is real and source-reproducible, but this PR patches the Anthropic SDK client path while current embedded Anthropic Messages traffic uses OpenClaw's guarded transport path, so the patch is not merge-ready.

## What This Changes

The PR adds a custom undici Agent to Anthropic SDK client construction and passes it through fetchOptions.dispatcher for all Anthropic client variants.

## Best Possible Solution

Replace the SDK dispatcher injection with a bounded retry-once path for clearly transient pre-response socket/fetch failures inside the guarded Anthropic transport, preserving guarded dispatcher policy and adding focused regression plus live/staging proof.

## Reproduction Assessment

Yes for source-level behavior: current main routes Anthropic Messages through guarded fetch, rethrows pre-response transport errors, and classifies bare fetch failures as timeout. I did not live-reproduce the multi-hour stale-socket trigger.

## Solution Assessment

No. The PR is a plausible mitigation for SDK callers, but the best fix is a capped same-provider retry in the guarded transport path before model fallback runs.
