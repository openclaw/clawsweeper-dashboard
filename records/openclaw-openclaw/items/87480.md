## Summary

Keep open: the linked Anthropic socket/fallback bug remains real on current main and in the latest release, but this PR is not merge-ready because it applies a raw SDK dispatcher while the reported embedded Anthropic Messages path uses the guarded model-fetch transport.

## What This Changes

The PR adds a custom undici Agent with extended keep-alive, timeout, pipelining, HTTP/2, and connection settings to Anthropic SDK clients via `fetchOptions.dispatcher`.

## Best Possible Solution

Move the mitigation into guarded model transport as a capped same-key retry for clearly transient pre-response socket/fetch failures before model fallback, preserving dispatcher/security policy and proving it with focused tests plus redacted live or staging gateway logs.

## Reproduction Assessment

Yes for the source-level failure path: current code and tests show bare transport failures can classify as timeout and advance the fallback chain. I did not live-reproduce the multi-hour Anthropic stale-socket trigger in this read-only review.

## Solution Assessment

No. This is a plausible mitigation, but the best fix is narrower in the guarded transport path so it retries the primary provider before fallback without replacing SDK dispatcher policy.
