## Summary

Keep open, but not merge-ready: the linked Anthropic socket/fallback bug is real and still active, yet this PR targets the SDK client dispatcher while the reported embedded Anthropic Messages path runs through guarded model fetch. The branch also lacks redacted after-fix real behavior proof from the affected gateway path.

## What This Changes

The branch adds an undici Agent to Anthropic SDK client creation and passes it as `fetchOptions.dispatcher` for Cloudflare, Copilot, OAuth, and API-key Anthropic client variants.

## Best Possible Solution

Replace the SDK dispatcher injection with a bounded retry-once path for clearly transient pre-response socket/fetch failures inside the guarded Anthropic/model transport, preserving guarded dispatcher policy and adding focused regression tests plus redacted live or staging proof.

## Reproduction Assessment

Yes for the source-level behavior: current main routes Anthropic Messages through guarded fetch, rethrows pre-response transport errors, and classifies bare fetch failures as timeout. I did not live-reproduce the multi-hour stale-socket trigger.

## Solution Assessment

No. The PR is a plausible SDK-layer mitigation, but the best fix is a capped same-provider retry in the guarded transport path before model fallback runs.
