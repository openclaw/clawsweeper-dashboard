## Summary

Keep open. Current `main` and the latest shipped release still omit `iproute2` from the default official Docker image; the documented custom build-arg path helps local image builders, but it does not solve the requested no-custom-image Synology/macvlan workflow, so this remains a maintainer product/security decision.

## What This Changes

The issue requests adding `iproute2` to the official OpenClaw Docker image so macvlan and custom route setups can use the `ip` command without rebuilding a custom image.

## Best Possible Solution

Make an explicit Docker image policy call: either include `iproute2` in the default runtime image with release-image proof and Docker docs covering required capabilities, or document macvlan route changes as a custom-image/operator recipe.

## Reproduction Assessment
