## Summary

Keep open. Current main and the latest release still omit `iproute2` from the official default Docker runtime image; the existing build-arg path only helps local/custom builds, so the remaining decision is Docker image package and capability policy.

## What This Changes

Request pre-installing `iproute2` in official OpenClaw Docker images so macvlan/proxy users can change container routes without custom rebuilds.

## Best Possible Solution

Make an explicit Docker image policy call: either add `iproute2` to the default runtime image with tests, docs, changelog, and capability guidance, or document macvlan route changes as a custom-image/host-networking recipe.

## Reproduction Assessment

Yes for the reported absence, via source inspection: current and latest-release Dockerfiles do not install `iproute2`, and official release builds do not pass the extra apt package build arg. This is a feature request rather than a broken documented behavior.

## Solution Assessment

Unclear. Adding the package is technically narrow, but changing the default official image package surface and route-capability guidance needs maintainer approval before it should be treated as a mechanical fix.
