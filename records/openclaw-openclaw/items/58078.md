## Summary

Keep open. Current main and the latest release still omit `iproute2` from the default official Docker image, while the existing workaround only supports custom local image builds; the remaining question is a maintainer product/security decision about default route-management tooling in every shipped image.

## What This Changes

The issue asks OpenClaw to pre-install `iproute2` in the official Docker image so macvlan and proxy-routing users can run `ip route` without maintaining a custom image.

## Best Possible Solution

Make an explicit Docker image policy call: either include `iproute2` in the default runtime image with release-image proof and Docker docs covering required capabilities, or document macvlan route changes as a custom-image/operator recipe.

## Reproduction Assessment

## Solution Assessment
