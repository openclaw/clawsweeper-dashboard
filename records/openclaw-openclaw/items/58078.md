## Summary

Keep open. Current main still builds the official runtime image without `iproute2`, and the official Docker release workflow does not pass the custom apt-package build arg that would let pulled `ghcr.io/openclaw/openclaw` images include it.

## What This Changes

Request to pre-install `iproute2` in the official Docker image so macvlan and proxy-routing users have the `ip` command without maintaining a custom image.

## Best Possible Solution

Add `iproute2` to the official runtime image's system utility package set, update the Dockerfile test/docs/changelog accordingly, and document that route replacement remains an explicit operator privilege/configuration choice.

## Reproduction Assessment

Yes. The current Dockerfile and release workflow provide a high-confidence static reproduction path: official builds do not install `iproute2`, and a Docker smoke would verify this with `command -v ip` inside the built image.

## Solution Assessment

Yes. The narrow maintainable fix is to add the package to the runtime apt install list and assert it in Dockerfile coverage, without changing default Compose capabilities or container user policy.
