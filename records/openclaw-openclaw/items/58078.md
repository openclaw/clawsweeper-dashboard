## Summary

Keep open. Current main and the latest release still omit `iproute2` from the default official Docker runtime image; the existing apt package hook only helps users who build a custom/local image, so the remaining question is default-image package and container capability policy.

## What This Changes

Request to pre-install Debian `iproute2` in the official Docker image so macvlan/NAS users can run `ip route` inside the container without rebuilding a custom image.

## Best Possible Solution

Make a Docker image policy call: either add `iproute2` to the default runtime image with tests, release-image proof, docs, and capability guidance, or document the macvlan routing setup as a custom-image/operator recipe.

## Reproduction Assessment

Yes for the reported package absence: current source, release workflow, and the `v2026.5.18` tag show `iproute2` is not installed by default and official builds do not pass the extra apt package build arg. I did not run a live GHCR container in this read-only review.

## Solution Assessment

Unclear. Pre-installing the package is technically narrow, but changing every official image and the route-capability guidance is a maintainer product/security decision rather than a mechanical bug fix.
