## Summary

Keep open. Current main and the latest release still omit iproute2 from the default official Docker image; the existing extra-APT path is build-time only for local/derived images, while the linked generic package-install issue only partially overlaps this Synology/macvlan request.

## What This Changes

The issue requests preinstalling iproute2 in the official Docker image so macvlan/routing users can run `ip route` without maintaining a custom image.

## Best Possible Solution

Maintainers should make one Docker image policy call: include iproute2 by default with tests, docs, and release-image proof, or explicitly choose a documented derived-image/prebuild path and let https://github.com/openclaw/openclaw/issues/85332 own generic package customization.
