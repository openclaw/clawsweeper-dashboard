## Summary

Keep open. Current main still does not bake `iproute2` into the official Docker release image, and the existing workaround is build-time customization rather than the requested preinstalled default; changing the official image package surface needs maintainer product/security review.

## What This Changes

Pre-install `iproute2` in the official OpenClaw Docker image so NAS/macvlan users can run route commands without maintaining a custom image.

## Best Possible Solution

Make an explicit Docker image policy decision: either add `iproute2` to the default runtime image with tests/docs/changelog, or document a supported macvlan custom-image and capability recipe.

## Reproduction Assessment

Yes. The source-level reproduction is clear from the current Dockerfile and Docker release workflow; a runtime confirmation would be pulling an official tag and checking `command -v ip` inside the container.

## Solution Assessment

Unclear. The code change would be narrow, but the default-image package policy, SBOM impact, and required route-changing capabilities need maintainer review before treating it as a mechanical fix.
