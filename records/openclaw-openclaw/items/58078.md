## Summary

Keep open. Current main and v2026.5.12 still omit `iproute2` from the default official Docker runtime image; the existing extra-apt build arg only helps local/custom builds, so the remaining question is default image package and capability policy.

## What This Changes

Request pre-installing `iproute2` in the official Docker runtime image so macvlan/NAS users can change container routes without rebuilding a custom image.

## Best Possible Solution

Make an explicit Docker image policy call: either add `iproute2` to the default runtime image with focused tests, docs, changelog, and capability guidance, or document macvlan route changes as a custom-image/host-networking recipe.

## Reproduction Assessment

Yes for the reported package absence: source and latest-release inspection show `iproute2` is not installed by default, and official release builds do not pass the extra apt package build arg. This is a feature request, not a broken documented behavior.

## Solution Assessment

Unclear. Adding the package is technically narrow, but changing the default official image package surface and route-capability guidance needs maintainer approval before it should be treated as a mechanical fix.
