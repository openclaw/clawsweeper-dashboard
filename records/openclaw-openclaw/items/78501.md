## Summary

Keep this PR open: the main heavy-check lock direction is source-backed, but the latest head adds a package preinstall pressure guard that can block normal installs, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch changes local heavy-check lock routing for type-aware oxlint, sharded oxlint, and scoped tooling tests, and additionally adds a preinstall host-pressure refusal guard.

## Best Possible Solution

Keep the heavy-check lock fixes inside the existing script helper seams, remove or narrowly gate the preinstall pressure guard so packaged installs stay reliable, then merge only after real after-fix proof and green checks.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main bypasses the lock for type-aware explicit oxlint targets, sharded oxlint children skip duplicate locks without a parent lock, and scoped tooling tests can skip the shared lock. I did not reproduce the reported Raspberry Pi RSS incident or run the PR branch.

## Solution Assessment

No, not as currently proposed: the heavy-check lock changes are narrow, but the added preinstall pressure guard can misclassify packaged installs under any consumer Git worktree and block installation. The safer path is to keep packaged lifecycle behavior non-fatal and scope pressure refusal only to true OpenClaw source checkouts if maintainers want that policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid treating any Git worktree as a source checkout:** `scripts/preinstall-package-manager-warning.mjs:124`
  - body: This preinstall guard uses `git rev-parse --is-inside-work-tree` from the lifecycle cwd to decide whether the install is a source checkout. During a normal packaged install under `node_modules/openclaw` inside any consumer project that is itself a Git repo, that command still returns true, so a low-memory or high-load machine can get an exit-code 1 from OpenClaw's preinstall and fail the install. Please gate this on an actual OpenClaw source checkout marker, or keep packaged preinstall warnings non-fatal.
  - confidence: 0.88
