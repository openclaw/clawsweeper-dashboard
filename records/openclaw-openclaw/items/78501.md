## Summary

Keep this PR open, but it is not merge-ready: the heavy-check lock hardening is useful and the supplied Raspberry Pi proof is relevant, while the new install pressure guard can falsely block published-package installs from inside any consumer Git worktree.

## What This Changes

The PR tightens local heavy-check locking for oxlint and scoped tests, adds a local install pressure guard, updates script tests, and includes a lint-only cron pagination cleanup.

## Best Possible Solution

Keep the heavy-check lock hardening, but replace the install guard's raw Git-worktree detector with an OpenClaw package-root/source-root detector and add regression coverage proving packaged installs under a consumer Git worktree are not refused.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: the preinstall hook is packaged and the PR treats any Git worktree as a source checkout. I did not run an install because this was a read-only review.

## Solution Assessment

No. The lock serialization pieces fit the script policy, but the install pressure guard should use an OpenClaw source-root/package-root contract instead of `git rev-parse --is-inside-work-tree`.
