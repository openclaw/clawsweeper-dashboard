## Summary

Keep open: the lock hardening is useful, but the proposed preinstall pressure guard can block published-package installs from inside consumer Git repositories, so this PR needs a narrow compatibility repair before merge.

## What This Changes

The PR tightens local heavy-check locking for type-aware oxlint and scoped tests, adds a pressured local install refusal guard, and adjusts related tests plus cron listing control flow.

## Best Possible Solution

Keep the lock hardening, but change the install pressure guard to use OpenClaw source-root markers and add regression proof that packaged installs under a consumer Git repo are not refused before merge.

## Reproduction Assessment

Yes for the PR defect by source inspection: the package `preinstall` hook calls a guard whose source-checkout check is satisfied by any Git worktree. I did not run an install because this was a read-only review.

## Solution Assessment

No. The lock serialization pieces fit the existing scripts policy, but the install pressure guard should use an OpenClaw source-root contract rather than `git rev-parse --is-inside-work-tree`.
