## Summary

Keep open: the lock-gap part is still source-reproducible on current main, but this PR is not merge-ready because the branch lacks real behavior proof and introduces blocking script/lifecycle regressions.

## What This Changes

The PR expands local heavy-check locking for type-aware oxlint, sharded oxlint, and scoped tooling tests, adds a local install pressure guard, and adjusts related script, cron, and Codex tests.

## Best Possible Solution

Keep the narrow lock fixes, rebase onto current main's skip-aware/worktree-lock behavior, remove or strictly source-checkout-gate the fatal preinstall guard, and require redacted after-fix proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main still lets type-aware explicit-file oxlint and scoped tooling-test runs bypass the shared lock, and the PR head's sharded wrapper can self-wait because it ignores the parent skip-lock marker.

## Solution Assessment

No, not as currently proposed. The serialization direction is maintainable, but the PR must preserve parent skip-lock behavior and avoid a fatal packaged preinstall lifecycle guard before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor the parent oxlint skip lock:** `scripts/run-oxlint-shards.mjs:8-12`
  - body: `check:changed` already acquires the shared heavy-check lock and runs child lint commands with `OPENCLAW_OXLINT_SKIP_LOCK=1`. This wrapper ignores that flag and immediately acquires the lock, so a changed-check lane that reaches sharded lint can queue behind its own parent until timeout.
  - confidence: 0.9
- **[P2] Gate the install refusal to real source checkouts:** `scripts/preinstall-package-manager-warning.mjs:124`
  - body: This treats any Git work tree as an OpenClaw source checkout. Because this script is packaged and runs as `preinstall`, installing OpenClaw inside a consumer Git repo can fail under host pressure; use a real OpenClaw source marker or keep packaged preinstall warnings non-fatal.
  - confidence: 0.88
