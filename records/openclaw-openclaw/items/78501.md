## Summary

Keep open: the heavy-check lock direction is useful and the updated PR body now supplies real behavior proof, but the new preinstall pressure guard can block installs outside the OpenClaw source tree because it treats any Git worktree as a source checkout.

## What This Changes

The PR tightens local heavy-check locking for type-aware oxlint and scoped tooling tests, adds a local install pressure refusal guard, and includes small lint-oriented cleanup in cron listing code.

## Best Possible Solution

Keep the lock hardening, but gate install pressure refusal with an OpenClaw source-root detector and regression tests that prove packaged installs are not refused.

## Reproduction Assessment

Yes for the PR defect by source inspection: the package `preinstall` hook calls a guard whose source-checkout check is satisfied by any Git worktree. I did not run an install because this was a read-only review.

## Solution Assessment

No. The lock serialization pieces fit the existing script policy, but the install-pressure guard should reuse an OpenClaw-root contract rather than `git rev-parse --is-inside-work-tree`.
