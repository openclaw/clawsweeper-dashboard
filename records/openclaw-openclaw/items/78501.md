## Summary

Keep open: the heavy-check hardening is useful, but the latest head still has a merge-blocking install lifecycle compatibility bug because the new preinstall pressure guard treats any Git worktree as an OpenClaw source checkout.

## What This Changes

The PR makes type-aware oxlint and scoped tooling test runs use the shared heavy-check lock, adds a pressured local install refusal path to the preinstall lifecycle script, and updates related script tests plus a cron-tool lint cleanup.

## Best Possible Solution

Keep the heavy-check lock hardening, but restrict the install pressure guard to real OpenClaw source/package roots and prove both source-checkout refusal and packaged-install compatibility.

## Reproduction Assessment

Yes, by source inspection: the packaged preinstall hook runs the new guard, and the guard treats any Git worktree as a source checkout. I did not run an install because this review is read-only.

## Solution Assessment

No as-is. The lock serialization changes fit the scripts policy, but the install pressure guard needs a narrower OpenClaw-root detector before the lifecycle hook is safe to ship.
