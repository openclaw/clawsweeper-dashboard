## Summary

Keep open for contributor or maintainer follow-up: the lock-gap direction is useful, but the PR currently has missing real behavior proof, merge conflicts, and two blocking correctness/security issues in the changed scripts.

## What This Changes

The PR expands local heavy-check locking for type-aware oxlint, sharded oxlint, and scoped tooling tests, adds a local install pressure guard, and adjusts related script, cron, and Codex tests.

## Best Possible Solution

Keep the narrow lock fixes, rebase onto current main's skip-aware sharded wrapper, remove or strictly source-checkout-gate the fatal preinstall guard, and require redacted after-fix proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main still lets type-aware explicit-file oxlint and scoped tooling-test runs bypass the shared local heavy-check lock, and the PR's new self-deadlock path is visible from `check:changed` plus the PR head sharded wrapper.

## Solution Assessment

No, not as currently proposed. The lock serialization direction is maintainable, but the PR must preserve parent skip-lock behavior and avoid a fatal packaged preinstall lifecycle guard before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor the parent oxlint skip lock:** `scripts/run-oxlint-shards.mjs:8-12`
  - body: `check:changed` already acquires the shared heavy-check lock and runs child commands with `OPENCLAW_OXLINT_SKIP_LOCK=1`. This wrapper ignores that flag and immediately calls `acquireLocalHeavyCheckLockSync`, so a changed-check lane that runs `pnpm lint` can queue behind its own parent lock until timeout.
  - confidence: 0.9
- **[P2] Gate the install refusal to real source checkouts:** `scripts/preinstall-package-manager-warning.mjs:124`
  - body: This uses `git rev-parse --is-inside-work-tree` from the lifecycle cwd to decide whether the install is a source checkout. A packaged install under `node_modules/openclaw` inside a consumer Git repo also returns true, so a low-memory or high-load host can fail OpenClaw's package `preinstall`; gate this on an actual OpenClaw source checkout marker or keep packaged preinstall warnings non-fatal.
  - confidence: 0.88
