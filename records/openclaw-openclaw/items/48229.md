## Summary

Keep open: current main and latest release still clear scopes for device-less shared-token/password WebSocket operator clients, and checked-in tests assert the resulting missing-scope behavior. The related open PR is not a safe covering fix for the regular token-auth LAN report, so this needs maintainer security/product follow-up rather than cleanup closure.

## What This Changes

The issue reports that a LAN-bound, token-authenticated WebSocket operator client loses `operator.write` scope and cannot dispatch A2A work from Docker.

## Best Possible Solution

Resolve the bounded token-scope or device-token policy in the canonical gateway auth path, with tests and docs for both retained-scope and still-cleared cases, before closing this issue.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` clear device-less shared-token/password WebSocket operator scopes, and checked-in tests assert the resulting missing-scope behavior.

## Solution Assessment

No for the unconditional scope-retention request. The safer solution is a maintainer-approved bounded token-scope or device-token contract rather than trusting client-declared scopes by default.
