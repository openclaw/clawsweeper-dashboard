## Summary

Keep open: current main and v2026.6.8 still clear scopes for device-less shared-token/password WebSocket operator clients, and the remaining fix changes a privileged Gateway auth boundary that is already under maintainer security/product review.

## What This Changes

The issue reports that LAN/Docker token-auth WebSocket operator clients lose `operator.write` scope after a successful connection while loopback clients work.

## Best Possible Solution

Resolve the bounded token-scope or device-token policy in the canonical Gateway auth path, with tests/docs for retained-scope and still-cleared cases, before closing this issue.

## Reproduction Assessment

Yes at source level. Current main and v2026.6.8 clear device-less shared-token/password WebSocket operator scopes, and checked-in tests assert the resulting missing-scope behavior; I did not run a live Docker LAN Gateway repro.

## Solution Assessment

No for the blanket scope-retention request. The safer solution is a maintainer-approved bounded token-scope or device-token contract rather than trusting client-declared scopes by default.
