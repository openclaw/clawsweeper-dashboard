## Summary

## What This Changes

The issue asks for LAN-bound token-authenticated gateway WebSocket operator clients to retain requested `operator.write`/`operator.admin` scopes like loopback clients do.

## Best Possible Solution

Resolve the security contract once in the canonical gateway auth thread: either land a tested, maintainer-approved scope-retention fix for non-local token-auth operators or document that device-less shared-token operator scopes are intentionally cleared and update linked PR references accordingly.
