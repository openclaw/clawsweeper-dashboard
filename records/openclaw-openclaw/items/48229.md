## Summary

Keep open: current main and the latest v2026.6.6 release still clear scopes for the reported device-less shared-token WebSocket operator path, and the linked open PR does not yet provide a clean maintainer-approved fix for the regular token-auth LAN case.

## What This Changes

The issue asks OpenClaw to preserve requested operator scopes for token-authenticated WebSocket operator clients when `gateway.bind` is `lan` instead of clearing scopes on device-less non-local sessions.

## Best Possible Solution

Resolve the gateway auth policy once in the canonical thread: either land a maintainer/security-approved scope-retention fix with regression coverage and real remote-token proof, or document that device-less shared-token WebSocket operators intentionally remain scope-empty.

## Reproduction Assessment

Yes, from source inspection: current main clears scopes for the reported device-less shared-token operator WebSocket path, and the default-token suite asserts the resulting `missing scope` behavior. I did not run the Docker/LAN flow live in this read-only review.

## Solution Assessment

Unclear/no. Preserving non-local token-auth operator scopes may fix the Docker/A2A workflow, but it changes a security-sensitive gateway authorization baseline, so the best fix needs maintainer/security approval rather than another automatic patch.
