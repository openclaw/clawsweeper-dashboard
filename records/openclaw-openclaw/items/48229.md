## Summary

Keep open: current main and the latest release still clear scopes for device-less shared-token/password WebSocket operator clients, and the checked-in gateway tests assert the resulting missing-scope behavior. The linked open PR is not a safe covering fix for this regular token-auth LAN report, so this needs maintainer security/product follow-up rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to preserve `operator.write`/`operator.admin` scopes for token-authenticated WebSocket operator clients when the Gateway is bound to LAN for Docker-style deployments.

## Best Possible Solution

Resolve the bounded token-scope or device-token policy in the canonical gateway auth path, with tests and docs for both retained-scope and still-cleared cases, before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.8` clear device-less shared-token/password WebSocket operator scopes, and checked-in tests assert the resulting missing-scope behavior.

## Solution Assessment

No for the unconditional scope-retention request. The safer solution is a maintainer-approved bounded token-scope or device-token contract rather than trusting client-declared scopes by default.
