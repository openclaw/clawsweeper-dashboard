## Summary

This should close as a duplicate: the reported Docker/LAN token-auth WebSocket scope loss is the same remaining trust-boundary problem now tracked by the broader open gateway issue, and current main still shows the behavior rather than an implemented fix.

## What This Changes

Token-authenticated LAN WebSocket operator clients lose requested operator scopes and then fail write-scoped gateway calls with `missing scope: operator.write`.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/51396 as the canonical open issue, preserving this Docker/A2A context there while maintainers decide the WebSocket token-auth scope policy.

## Reproduction Assessment

Yes, from source and checked-in tests. Current main connects a valid shared-token operator without device identity with cleared scopes, and the auth suite asserts a later scope-gated RPC fails with `missing scope`.
