- `impact:other`: The issue affects exec node routing and tool execution, which is maintainer-visible but outside the more specific owned impact labels.

## Summary

Keep open: current main and v2026.6.6 still compare raw bound/requested node selector strings before the shared resolver can canonicalize them, and open closing PRs already own the fix path.

## What This Changes

The issue reports that host=node exec rejects a valid request when bound and requested node selectors identify the same node but use different selector forms.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/66985 or an equivalent canonicalization fix, then close this issue with regression coverage for same-node/different-selector and different-node rejection cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: configure `boundNode` as a full node id and request the same node by display name, and current main throws before resolver canonicalization. I did not run a live paired-node setup in this read-only review.

## Solution Assessment

Yes. The best fix is to resolve both selectors inside `resolveNodeExecutionTarget` and compare canonical node ids, because that helper owns conversion from raw config/tool selectors to the concrete node execution target.
