## Summary

Keep open. Current main and the latest release still delegate TUI chat/tool Markdown through pi-tui without a dunder/underscore regression fix, and the only directly related fix PR was closed unmerged after a Markdown compatibility concern.

## What This Changes

The issue asks OpenClaw to preserve underscores verbatim in TUI-rendered fenced code, inline code, and Python identifiers such as __name__, __main__, and is_palindrome.

## Best Possible Solution

Keep this issue open and define a narrow TUI Markdown policy that preserves copyable code identifiers without unintentionally changing ordinary underscore-based Markdown emphasis.

## Reproduction Assessment

No high-confidence current-main reproduction yet. The report is concrete and source inspection shows the relevant Markdown path, but this read-only review did not run a ChatLog/HyperlinkMarkdown or PTY proof showing underscores stripped on current main.

## Solution Assessment

No proposed fix is under review. The closed repair PR was plausible but intentionally changed underscore-strong Markdown behavior, so the better path is a narrower renderer fix or explicit maintainer approval of that compatibility tradeoff.
