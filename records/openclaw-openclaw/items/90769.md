Keep open. Current main and v2026.6.8 still route assistant and tool-result Markdown through pi-tui without a focused final-render regression for Python dunder identifiers, and the only linked repair PR was closed unmerged after a Markdown compatibility concern.

## What This Changes

The issue asks for TUI Markdown to preserve Python dunder and snake_case identifiers verbatim in fenced code blocks and inline code spans.

## Best Possible Solution

Add a current-main final-render regression for Python dunder and snake_case output, then fix the TUI Markdown path narrowly for assistant and tool rows without broad Markdown semantics drift.

## Reproduction Assessment

No high-confidence final TUI reproduction yet. Source inspection shows assistant and tool rows pass raw text to pi-tui Markdown, whose inline strong/emphasis handling can consume underscores outside protected code tokens, but the reported fenced-code failure still needs ChatLog or PTY proof.

## Solution Assessment

No patch is currently under review. The best fix is a renderer-level regression and narrow TUI Markdown repair, not a broad underscore preprocessor or a NemoClaw/model workaround.
