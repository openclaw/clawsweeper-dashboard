## Summary

Keep open. Current main still stores a hardcoded 2,000 ms lane wait warning default, exposes no `diagnostics.laneWaitWarnMs` schema/type/help/docs surface, and the directly related implementation PR at https://github.com/openclaw/openclaw/pull/16366 is closed unmerged while https://github.com/openclaw/openclaw/issues/53023 is a broader concurrency/auto-yield umbrella, not a replacement for this focused log-noise control.

## What This Changes

Request to add a configurable lane wait diagnostic warning threshold so long-running cron work can avoid repeated non-actionable queue warnings.

## Best Possible Solution

Add a documented positive-millisecond `diagnostics.laneWaitWarnMs` default for future queue entries, keep explicit `opts.warnAfterMs` overrides authoritative, cover config reload behavior, and leave per-lane/adaptive/concurrency work to separate trackers.

## Reproduction Assessment

Yes, at source level. Enqueue a second task behind an active lane entry without `opts.warnAfterMs`; current main stores a 2,000 ms threshold and later emits `lane wait exceeded`, while the proposed config key is absent from schema, types, help, and docs.

## Solution Assessment

Yes. A narrow diagnostics threshold setting is the maintainable fix for this log-noise request; per-lane thresholds, adaptive timeouts, session concurrency, and auto-yield should remain separate work.

Implement the focused lane wait diagnostic threshold requested by https://github.com/openclaw/openclaw/issues/14747. Add a documented positive integer millisecond config field, preferably `diagnostics.laneWaitWarnMs`, that defaults to the current 2,000 ms behavior when unset and supplies the default warning threshold for future `enqueueCommandInLane` entries. Preserve explicit per-call `opts.warnAfterMs` overrides, including the manual cron run override in `src/cron/service/ops.ts`. Wire the config through the existing gateway startup/reload path so future queue entries can pick up the new value without requiring a restart, and add/update tests for command queue defaults, gateway lane/config application, config reload planning, and cron override behavior. Update schema/types/labels/help/generated docs and user-facing docs, plus `CHANGELOG.md` because this is a user-facing config feature.
