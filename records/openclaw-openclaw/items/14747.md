Request to make the lane wait diagnostic warning threshold configurable instead of always defaulting queued entries to about 2 seconds.

## Best Possible Solution

Land a focused refreshed implementation that preserves the 2,000 ms default and explicit per-call overrides, adds `diagnostics.laneWaitWarnMs` across schema/types/help/docs, and makes reload versus restart behavior explicit with coverage.

## Solution Assessment

Yes in direction, with conditions: a narrow global `diagnostics.laneWaitWarnMs` defaulting to 2,000 ms is the maintainable fix for this logging-control request. The implementation should not expand into session concurrency or auto-yield, and it must align config contract and reload behavior before merge.
