## What This Changes

Request a configurable lane wait diagnostic warning threshold so long-running cron or background lanes do not flood gateway logs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/86676 or an equivalent narrow fix that adds documented `diagnostics.laneWaitWarnMs`, preserves the 2,000 ms default and explicit per-entry overrides, and aligns schema, types, help, docs, tests, and reload semantics.

## Solution Assessment
