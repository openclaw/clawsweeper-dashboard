## Summary

## What This Changes

The issue requests a configurable lane wait diagnostic warning threshold so legitimate long-running queued work does not flood gateway diagnostics.

## Best Possible Solution

Land a refreshed focused PR that preserves the 2,000 ms default and explicit per-call overrides, adds canonical schema/type/help/docs/reload handling with tests, and then close this issue through the merged PR.
