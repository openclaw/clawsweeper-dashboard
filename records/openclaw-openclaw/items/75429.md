## Summary

Keep open. Current main still has the exact package.json fallback equality check described in the report, and the issue is paired with open PR #75431, which references it with closing syntax and targets the same predicate, so the issue should remain open until that PR is merged or closed.

## What This Changes

The issue reports that v2026.4.29 repeatedly restages bundled plugin runtime npm dependencies because package.json fallback materialization requires exact dependency-array equality instead of accepting an already-installed shared dependency superset.

## Best Possible Solution

Land a narrow fix that treats package.json fallback specs as a superset only after verifying every requested package is installed on disk, and add a regression test for a missing generated manifest with a shared package.json dependency list.

## Reproduction Assessment

Yes. A focused fixture with installed node_modules, no generated install manifest, and package.json containing a superset of one plugin's install specs reproduces the predicate because line 141 requires exact spec equality before `hasSatisfiedInstallSpecPackages()` can succeed.

## Solution Assessment

Yes. The paired PR's described switch from exact equality to the existing superset helper is the narrowest maintainable solution for this predicate, with the important follow-through being regression coverage for the package.json fallback case.
