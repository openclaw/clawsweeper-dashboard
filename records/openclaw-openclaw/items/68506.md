Overall confidence: 0.88

Full review comments:

- **[P1] Reuse the sanitized OAuth refresh helper:** `src/commands/status-all/diagnosis.ts:66`
  - body: `buildOAuthRefreshLoginCommand` interpolates provider text parsed from `healthErr` into a displayed login command. Current auth code sanitizes provider IDs, and doctor guidance aliases legacy `openai-codex` to canonical `openai`; bypassing that path can print unsafe or stale re-auth guidance. Use the existing OAuth refresh classification/safe-command path plus the aliasing logic instead.
  - confidence: 0.88
- **[P2] Match auth status codes before classifying:** `src/commands/status-all/diagnosis.ts:126-127`
  - body: The classifier treats any error text containing `401` or `403` as a gateway auth failure before checking transport errors. Strings like `ECONNREFUSED 127.0.0.1:4030` or `1401ms` would tell users to repair gateway credentials even though the gateway is unreachable; use bounded HTTP-status matching or prioritize transport failures.
  - confidence: 0.84
- **[P3] Avoid duplicate unreachable recovery guidance:** `src/commands/status-all/diagnosis.ts:394-397`
  - body: When `health.error` is `gateway unreachable` and `gatewayReachable` is false, the channel-skip branch prints one cause/fix block and the health classifier prints the same cause/fix again. That makes one failure look like two separate diagnoses; suppress one block or test the occurrence count.
  - confidence: 0.78
