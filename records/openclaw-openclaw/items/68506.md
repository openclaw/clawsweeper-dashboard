- **[P2] Reuse the sanitized OAuth refresh helper:** `src/commands/status-all/diagnosis.ts:68-71`
  - body: The new OAuth refresh guidance extracts a provider string from the health error and prints it directly in `openclaw models auth login --provider ...`.   - confidence: 0.9
- **[P2] Match auth status codes precisely:** `src/commands/status-all/diagnosis.ts:123-127`
  - body: The health recovery classifier treats any text containing `401` or `403` as an auth failure before transport failures are considered. Errors such as `ECONNREFUSED 127.0.0.1:4030` or `timeout after 1401ms` would incorrectly tell users to verify gateway credentials instead of checking reachability, so this should use bounded HTTP-status matching or prioritize transport errors first.
  - confidence: 0.88
- **[P3] Avoid repeating unreachable recovery guidance:** `src/commands/status-all/diagnosis.ts:386-398`
  - body: When `health.error` is `gateway unreachable` and the gateway is not reachable, the channel-skip block and the gateway-health block both emit the same cause/fix guidance. That makes one failure look like two separate recovery paths; emit it in one section or suppress the duplicate health recovery for this shared state.
  - confidence: 0.78
