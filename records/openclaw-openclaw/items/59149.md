## Summary

Keep open. Current main still has only global `tools.sessions.visibility` and `tools.agentToAgent` enforcement, and the paired open implementation PR is unmerged, so the issue should remain active for maintainer review.

## What This Changes

Add per-agent session visibility and agent-to-agent invocation scoping so multi-agent deployments can enforce least-privilege access between agents.

## Best Possible Solution

Land one reviewed access-control contract that covers schema/types, docs, requester-agent resolution, `sessions_list`, `sessions_history`, `sessions_send`, `session_status`, outbound A2A allowlists, and any chosen inbound invocation policy, with tests proving missing or malformed per-agent policy fails closed.

## Reproduction Assessment

Yes. The limitation is reproducible by inspection on current main: `agents.list[].tools` is strict and lacks the requested per-agent keys, while session visibility and A2A enforcement read only global `cfg.tools` values.

## Solution Assessment

Unclear. Per-agent scoping is the right direction, but the open implementation needs maintainer review for fallback semantics, requester identity, overlap with sandbox visibility work, and whether inbound `receivesFrom` belongs in the initial change.
