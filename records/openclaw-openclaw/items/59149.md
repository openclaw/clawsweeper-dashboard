## Summary

Keep open. Current main still exposes `tools.sessions.visibility` and `tools.agentToAgent` as global policy only, while the same-author implementation PR at https://github.com/openclaw/openclaw/pull/59170 is open, unmerged, and still needs access-control review.

## What This Changes

Add per-agent session visibility and agent-to-agent invocation scoping so multi-agent deployments can enforce least-privilege access instead of a global all-or-nothing policy.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/59170 with one fail-closed access-control contract covering schema/types, docs, requester identity, every session tool, outbound allowlists, and any chosen inbound policy.

## Reproduction Assessment

Yes. Source inspection on current main shows the per-agent tools schema lacks `sessions` and `agentToAgent`, while the runtime helpers and session tools read only global `cfg.tools` policy.

## Solution Assessment

Unclear. Per-agent scoping is the right direction, but the exact fallback semantics, requester identity trust, overlap with sandbox visibility work, and inbound invocation policy need maintainer review before merge.
