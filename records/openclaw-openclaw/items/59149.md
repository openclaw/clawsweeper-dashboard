## Summary

Keep open. Current main still exposes `tools.sessions.visibility` and `tools.agentToAgent` as global policy, while the paired implementation PR at https://github.com/openclaw/openclaw/pull/59170 remains open and unmerged.

## What This Changes

Allow session visibility and agent-to-agent invocation policy to be scoped per agent instead of only globally.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/59170 with one fail-closed access-control contract covering schema/types, docs, requester identity, all session tools, outbound A2A allowlists, and any chosen inbound policy.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict per-agent tools schema lacks the requested keys, while the runtime helpers and session tools read only global `cfg.tools` policy.

## Solution Assessment

Unclear. Per-agent scoping is the right direction, but fallback semantics, requester identity trust, overlap with sandbox visibility work, and inbound invocation policy need maintainer/security review before merge.
