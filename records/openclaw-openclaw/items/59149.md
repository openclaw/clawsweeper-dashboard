## Summary

Keep open. Current main still has only global `tools.sessions.visibility` and `tools.agentToAgent` enforcement, and the paired implementation PR at https://github.com/openclaw/openclaw/pull/59170 is open and unmerged.

## What This Changes

Request per-agent scoping for session visibility and agent-to-agent invocation so multi-agent deployments can enforce least-privilege access.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/59170 with one fail-closed access-control contract covering schema/types, docs, requester identity, all session tools, outbound A2A allowlists, and any chosen inbound policy.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict per-agent tools schema lacks `sessions` and `agentToAgent`, while the runtime helpers and session tools read only global `cfg.tools` policy.

## Solution Assessment

Unclear. Per-agent scoping is the right direction, but the exact fallback semantics, requester identity trust, overlap with sandbox visibility work, and inbound invocation policy need maintainer review before merge.
