## Summary

Keep open. Current main and the shipped v2026.6.1 code still have a direct-message fallback that can send the child/subagent result text to the external chat when the requester-agent completion handoff lacks the required delivery evidence, so the central bug remains present.

## What This Changes

The issue reports that a `sessions_spawn` run-mode child completion is delivered to the QQBot direct chat as raw worker output instead of staying internal for the parent/main session to summarize.

## Best Possible Solution

Decide the direct-message subagent completion contract, then update `src/agents/subagent-announce-delivery.ts` and its tests so child output is handed to the requester agent without leaking raw worker text to the external chat.

## Reproduction Assessment

Yes from source, but not from a live QQBot run in this review. Current main still has a test-backed direct-message fallback that externalizes child completion text when the completion agent lacks required delivery evidence, matching the reported failure mode.

## Solution Assessment

Unclear until maintainers choose the delivery contract. The likely fix is not a new feature, but a change to the existing announce/fallback behavior so parent summaries remain the only external user-facing output for run-mode subagents.
