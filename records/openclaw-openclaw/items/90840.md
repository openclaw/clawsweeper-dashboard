## Label Justifications

- `P1`: The report describes a live channel workflow where the user receives raw worker output and misses the intended parent/main summary.
- `impact:message-loss`: The intended parent summary is not delivered while a different raw child completion is delivered to the chat.
- `impact:session-state`: The failure is in routing completion state between child, parent, and requester sessions.

## Summary

Keep open: current main and v2026.6.5 still contain the direct-message `:text-direct` fallback that can publish the child `task_completion.result` when the parent completion handoff lacks message-tool evidence, and the latest discussion narrows this to a maintainer delivery-contract decision rather than a cleanup close.

## What This Changes

The issue requests that run-mode subagent completions return internally to the parent session so the chat user sees the parent/main summary instead of raw worker output.

## Best Possible Solution

Maintainers should choose the direct-message subagent completion contract, then update `src/agents/subagent-announce-delivery.ts` and focused tests so raw child output is never externalized and any user-visible reply comes from an explicit safe parent delivery path.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current source and tests still route the child `task_completion.result` through `deliverTextCompletionDirect` when the parent handoff lacks message-tool evidence. I did not run a live QQBot reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the delivery contract. A leak-stop is narrow, but a complete parent-summary delivery fix may require an explicit sanitized/public parent-reply signal rather than auto-publishing private final text.
