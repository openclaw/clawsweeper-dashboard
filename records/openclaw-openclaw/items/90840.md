## Summary

Keep open: current main and v2026.6.9 still contain the direct-message subagent completion fallback that can publish the child `task_completion.result` as a `delivery-mirror` when the announce run lacks message-tool delivery evidence. The related open yielded-completion PR only covers the `sessions_yield` sibling path, and this direct-run DM leak still needs maintainer delivery-semantics and security review.

## What This Changes

The issue reports that `sessions_spawn(mode="run")` can deliver raw child output to a direct chat instead of only exposing it internally for the parent/main session to summarize.

## Best Possible Solution

Choose the direct subagent completion contract, then implement a focused leak-stop or explicit sanitized public parent-reply signal with regression tests and live direct-message proof.

## Reproduction Assessment

Yes, at source level: current main still routes direct subagent completions through `message_tool_only` and falls back to `deliverTextCompletionDirect` with the child `task_completion.result` when message-tool evidence is missing. I did not run a live QQBot roundtrip in this read-only review.

## Solution Assessment

Unclear until maintainers choose the direct subagent completion contract. A narrow leak-stop is mechanically clear, but complete parent-summary delivery likely needs an explicit sanitized public-reply signal rather than automatic final-text publishing.
