## Summary

Keep open: current main and the latest release still preserve the direct-message `:text-direct` fallback that can publish raw child subagent output when the parent/announce run produced private final text but no message-tool delivery evidence. The code path is source-reproducible, but the safe fix still needs maintainer direction on the `message_tool_only` direct-completion contract.

## What This Changes

The issue asks OpenClaw to keep `sessions_spawn` run completions internal to the parent/main session and deliver only the parent-owned user-facing summary to the external chat.

## Best Possible Solution

Maintainers should settle the direct subagent completion delivery contract, then implement a structural guard or sanctioned public parent-reply signal that prevents raw child `:text-direct` delivery without auto-publishing private final text.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 still route direct-message subagent completions through the raw child-text fallback when message-tool evidence is missing. I did not run a live QQBot current-main roundtrip in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. A parent-final-text gate is a plausible leak stop, but the safer durable solution is explicit message-tool delivery or a sanctioned public parent-reply signal rather than auto-publishing private final text.
