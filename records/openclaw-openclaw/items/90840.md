## Summary

Keep open: current main and the latest release still contain the direct-message subagent text fallback that can send the raw child `task_completion.result`; the remaining change needs a maintainer decision on `message_tool_only` direct-completion semantics.

## What This Changes

The issue reports a subagent `sessions_spawn` run completion being delivered to the chat user as raw worker output instead of only being returned internally for the parent/main summary.

## Best Possible Solution

Maintainers should choose the direct-message subagent completion contract first, then implement either a narrow leak-stop gate or an explicit sanitized parent-public reply signal with focused tests and live DM proof.

## Reproduction Assessment

Yes at source level: current main still routes direct-message subagent completions through `deliverTextCompletionDirect` when message-tool evidence is missing, and the issue includes matching session trace evidence. I did not run a live QQBot roundtrip in this read-only review.

## Solution Assessment

Unclear until maintainers pick the contract: a gate on parent final text is a plausible leak stop, but it is not a complete parent-summary delivery fix. The safer end state is explicit message-tool delivery or a sanctioned public parent-reply signal rather than auto-publishing private final text.
