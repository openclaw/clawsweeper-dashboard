## Summary

Keep open: current `main` still has a source-supported path that can mirror the child subagent's raw `task_completion.result` to a direct chat when the parent announce run lacks message-tool delivery evidence. The issue is not implemented or safely superseded because the related open PR is blocked on broader `message_tool_only` contract concerns, and the issue comments correctly identify a maintainer behavior decision before a fix lands.

## What This Changes

The issue reports that a subagent spawned with `sessions_spawn` in `mode="run"` leaks the child worker's raw completion to the external chat instead of delivering only the parent/main response.

## Best Possible Solution

Choose the intended `message_tool_only` contract for subagent direct-message completions, then land a narrow fix that prevents raw child completion text from reaching external chats and proves the behavior with focused tests plus one live DM roundtrip.

## Reproduction Assessment

Yes from source inspection, not from a live local QQBot run. The current direct-message branch can still substitute the child `task_completion.result` when the parent announce run has no message-tool delivery evidence.

## Solution Assessment

No final fix has landed. The narrow leak-stop is plausible, but the best solution still requires maintainer confirmation of the `message_tool_only` behavior for subagent direct-message completions.
