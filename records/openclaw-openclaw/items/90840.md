## Summary

Keep open. Current main and the latest release still contain the direct-message text fallback that reads the child `task_completion.result` and can mirror that raw child output externally; the related open PR only closes a sibling `sessions_yield` issue, and the discussion identifies an unresolved delivery-contract decision.

## What This Changes

The issue asks OpenClaw to keep `sessions_spawn` `mode="run"` child completion output internal to the parent session and deliver only the parent/main user-facing summary to the QQBot direct chat.

## Best Possible Solution

Maintainers should decide the `message_tool_only` completion contract, then land a focused agent-delivery fix and regression tests that prevent raw child completion text from being mirrored externally without relying on text heuristics.

## Reproduction Assessment

Yes from source, though not with a live QQBot rerun in this read-only review. The logged path maps to current code: direct-message subagent completion forces `message_tool_only`, sees no messaging-tool evidence, and falls back to sending the child `task_completion.result`.

## Solution Assessment

Unclear until maintainers choose the contract. Suppressing the raw child fallback is necessary, but automatically delivering the parent's final text is not obviously safe because that text can still include private worker output.
