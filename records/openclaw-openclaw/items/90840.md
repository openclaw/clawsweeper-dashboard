## Summary

Keep open. Current main and the latest release still contain the direct-message subagent text fallback that can send the child `task_completion.result` externally, and the adjacent tests still assert that behavior; related open PRs need maintainer contract review/proof before this issue can be closed.

## What This Changes

The issue reports that a `sessions_spawn` run-mode subagent completion can be delivered to the chat user as raw child output instead of the parent/main session summary.

## Best Possible Solution

Make the parent/requester handoff authoritative for subagent completions: do not externally publish raw child text from the direct fallback when the parent owns the user-facing reply, and land the chosen behavior through a reviewed PR with live or equivalent transport proof.

## Reproduction Assessment

Yes from source inspection, but not from a fresh live QQBot run in this review. Current main still has the `message_tool_only` direct-completion branch that can fall back to sending the child `task_completion.result`, and tests assert that raw-child delivery path.

## Solution Assessment

No final fix shape can be selected yet. The narrow leak-stop is plausible, but the best complete solution needs maintainer direction on whether message-tool-only subagent direct completions should go silent without explicit message-tool send or gain a structural sanitized/public parent-reply signal.
