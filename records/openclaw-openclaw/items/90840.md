## Summary

Keep open: current main and the latest release still contain the direct-message subagent completion fallback that can mirror the raw child result, while the issue discussion shows the clean repair needs a maintainer decision about `message_tool_only` semantics versus a new sanitized public parent-reply signal.

## What This Changes

The issue reports that `sessions_spawn` `mode="run"` subagent completion in a direct chat leaks raw child output to the user instead of delivering only a parent-curated summary.

## Best Possible Solution

Make the parent/requester completion handoff the single visible-output owner, suppress raw child `:text-direct` delivery when the parent produced a completion, and settle the `message_tool_only` public-reply contract before changing code.

## Reproduction Assessment

Yes at source level: current main still has the same direct-message `:text-direct` raw child fallback, and the issue includes detailed transcript evidence plus a second user confirmation. I did not run a live current-main DM roundtrip.

## Solution Assessment

Unclear until maintainers choose the product contract. A leak-stop gate is mechanically narrow, but auto-delivering the parent's final text can re-leak private child output, so the best fix needs an explicit decision about silence versus a sanitized public reply signal.
