## Summary

Keep open. Current main and v2026.6.5 still contain the direct-message `:text-direct` fallback that can send a child subagent `task_completion.result` externally, tests still assert raw-child delivery in that path, and the latest discussion identifies an unresolved message-tool-only delivery contract decision rather than a completed fix.

## What This Changes

Subagent `sessions_spawn` run completions should return internally to the parent/main session and deliver only the parent user-facing summary, not raw worker output, to the chat user.

## Best Possible Solution

Make the parent/requester handoff authoritative for subagent completions, stop externally publishing raw child fallback text when the parent owns the user-facing reply, and land the chosen route contract with focused tests plus real transport proof.

## Reproduction Assessment

Yes, from source inspection and existing tests rather than a fresh QQBot run. Current main still routes subagent direct-message completions through `message_tool_only` and can fall back to sending the child `task_completion.result` when no message-tool delivery evidence is present.

## Solution Assessment

Unclear until maintainers choose the delivery contract. Suppressing raw child text is necessary, but the complete fix depends on whether no explicit message-tool call should mean silence or a new structural sanitized/public parent-reply signal.
