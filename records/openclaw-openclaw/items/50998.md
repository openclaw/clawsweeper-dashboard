## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Make Contacts permission waits cancellation-aware:** `apps/ios/Sources/Contacts/ContactsService.swift:105-109`
  - body: The new `.notDetermined` branch awaits `CNContactStore.requestAccess` through a bare continuation. `invokeWithTimeout` cancels the detached invoke task after timeout, but this continuation does not observe cancellation, so slow or unattended permission prompts can leave the handler suspended until the user responds; use a cancellation-aware permission gate and avoid starting the prompt once cancellation has won.
  - confidence: 0.9
- **[P2] Start Calendar prompts under the cancellation gate:** `apps/ios/Sources/Calendar/CalendarService.swift:183-184`
  - body: `awaitPermissionRequest` checks `box.isResolved` and then calls `start` as separate operations. If cancellation fires after the guard but before `start`, the caller can already return false while EventKit still launches the permission prompt after timeout; make install/check/start one atomic locked or tokened operation.
  - confidence: 0.86
