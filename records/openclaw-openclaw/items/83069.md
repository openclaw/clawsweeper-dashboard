## Summary

Keep open. The report is recent, detailed, and not covered by the closed DM/system-message echo issues; current main still has separate current-message audio preflight and reply-target media hydration paths, with no regression coverage for the forum-topic “reply with @mention to a voice note” workflow.

## What This Changes

The issue asks for Telegram forum-topic voice notes, especially mention replies to a voice note, to use the same transcription and echoTranscript behavior that works in direct chats.

## Best Possible Solution

Add a focused Telegram regression test for a forum-topic text @mention replying to a voice note, then ensure the replied-to audio is transcribed, echoed to the same topic, and not forwarded to the agent only as raw audio.

## Reproduction Assessment

No live reproduction was run. Source inspection gives a credible current-main path to reproduce with a Telegram forum topic voice message followed by a text @mention reply, but the shared replyMedia media-understanding path needs a targeted test to pin the exact failure.

## Solution Assessment

Yes, keeping this as a Telegram plugin bug is the best current direction. A docs-only workaround would be premature unless a focused test or live proof shows Telegram's Bot API cannot expose the replied-to voice media needed for transcription.
