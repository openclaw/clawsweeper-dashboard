## Label Justifications

- `P1`: The report describes a broken real Telegram forum-topic voice workflow where transcript routing fails while DMs work.
- `impact:message-loss`: The visible transcript echo is not delivered to the originating forum topic and downstream delivery can lose the topic target.
- `impact:session-state`: The root cause was flattened `OriginatingTo`, which mis-associates Telegram forum-topic context across inbound and follow-up routing.

## Summary

Current main now preserves Telegram forum-topic routing through the audio-preflight and echoTranscript path, so the reported forum-topic voice-note symptom is covered by the merged origin-routing fix. The fix is present on main but is not in a tagged release after v2026.5.12 yet.

## What This Changes

Telegram forum-topic voice notes should use the same audio transcription and echoTranscript routing as Telegram DMs, with transcript echoes delivered back to the originating topic.

## Best Possible Solution

Keep the merged Telegram origin-target fix on main and ship it in the next release; no separate core API or configuration change is needed for this report.

## Reproduction Assessment

Yes at source level for the reported released behavior: the matching canonical report showed v2026.5.x flattened Telegram forum-topic `OriginatingTo` in audio-preflight and follow-up contexts. Current main has targeted regression coverage proving the topic-qualified origin and thread id are preserved; I did not run a live Telegram repro.

## Solution Assessment

Yes. The current-main fix is the narrow owner-boundary solution: preserve topic-qualified Telegram origin targets in inbound, audio-preflight, and session reply contexts while keeping DM thread ids metadata-only.
