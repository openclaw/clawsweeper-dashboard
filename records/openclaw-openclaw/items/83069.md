## Summary

Keep open: current main still has a source-backed gap for the reported forum-topic workflow. Telegram reply-target media is fetched and attached to the inbound context, but audio preflight/transcript echo only considers media on the current message, so a text @mention reply to a voice note can still pass raw audio to the agent.

## What This Changes

The issue requests Telegram forum-topic voice notes to use the same media-understanding transcription and echoTranscript path that works for direct-chat voice notes.

## Best Possible Solution

Extend Telegram audio preflight/media-understanding to eligible reply-target voice-note media in forum topics, echo the transcript to the same topic/thread once, and preserve the existing authorization and no-double-STT guards.

## Reproduction Assessment

Yes from source inspection, though not from a live Telegram run. A forum-topic text reply with an @mention to a voice-note message produces replyMedia, while the transcription and echo path only considers allMedia from the current message.

## Solution Assessment

Yes, a code fix is the best direction. Documentation-only would leave the natural Telegram forum-topic workflow broken despite the existing audio and echoTranscript contract.
