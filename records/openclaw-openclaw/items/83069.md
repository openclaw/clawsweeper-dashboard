## Summary

Keep this open. The related DM and earlier outbound-routing reports are closed, but this report describes a narrower Telegram forum-topic reply-to-voice workflow; current main has no matching regression coverage, and source inspection shows replied audio is added after Telegram's preflight transcription decision.

## What This Changes

The issue reports that Telegram forum-topic voice notes reach the agent as raw audio while the same gateway transcribes and echoes direct-chat voice notes.

## Best Possible Solution

Add a focused Telegram forum-topic regression for replying with an @mention to a voice note, then make that path transcribe the replied audio exactly once and route any configured echoTranscript to the same topic/thread.

## Reproduction Assessment

Yes at code level, but not with a live Telegram run. A forum-topic text reply with an @mention and replied voice media reaches preflight with `allMedia` empty, while the replied audio is appended later as media context, matching the reported raw-audio handoff risk.

## Solution Assessment

Unclear because there is no proposed patch and the shared media path may still be intended to handle replied attachments. The safest solution is a targeted failing regression around the exact forum-topic reply workflow before changing routing or transcription behavior.
