## Summary

## What This Changes

The PR reorders systemd unavailability classification, routes `assertSystemdAvailable` through the shared classifier, adds WSL2-specific user D-Bus guidance, and covers the classifier/control path with daemon tests.

## Best Possible Solution

Land the focused daemon fix once maintainers either get redacted live WSL2 missing-bus terminal proof or explicitly accept the targeted test and Crabbox evidence as sufficient for this narrow error-path repair.

## Reproduction Assessment

Yes. Current main is source-reproducible: `classifySystemdUnavailableDetail('Failed to connect to bus: No such file or directory')` hits the broad missing-systemctl matcher before the user-bus matcher, which explains the reported wrong error.

## Solution Assessment

Yes. Reordering the specific bus classifier before the broad missing-binary classifier and making `assertSystemdAvailable` use that classifier is the narrow maintainable fix; the legacy `isSystemctlAvailable` follow-up can remain separate.
