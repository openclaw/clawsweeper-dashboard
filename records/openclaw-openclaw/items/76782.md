## Summary

Keep open: current main still exposes skills to the agent as name, description, and location only, so `triggers` would be a new public skill metadata and routing contract. The linked ClawHub `eyes` example is useful demand signal, but it does not make this implemented in OpenClaw core. ([clawhub.ai](https://clawhub.ai/skills/eyes))

## What This Changes

The issue requests an optional `triggers` field in `SKILL.md` frontmatter for exact keyword-based skill routing alongside semantic matching.

## Best Possible Solution

Have maintainers decide the public metadata shape and routing semantics, then implement optional normalized trigger parsing/rendering with docs and prompt-budget tests while preserving current behavior for skills that omit it.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source inspection gives a high-confidence absence check for parsed or prompt-rendered `triggers` on current main.

## Solution Assessment

Unclear: the proposed prompt field is plausible, but maintainers should first decide the public contract and exact-match behavior. A safer implementation should normalize triggers in the skill contract, update prompt budget handling, and document the compatibility rules.
