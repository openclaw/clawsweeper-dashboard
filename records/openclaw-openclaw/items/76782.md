## Summary

Keep open: current main still advertises skills by name, description, and location only, so `triggers` would be a new public skill metadata and routing contract rather than an implemented cleanup.

## What This Changes

Add optional `triggers` frontmatter metadata for skills and use it as an exact keyword signal alongside description-based skill routing.

## Best Possible Solution

Decide the public skill metadata contract first; if accepted, implement optional trigger-list parsing, prompt rendering, routing guidance, docs, and regression coverage while preserving behavior for skills that omit the field.
