## Summary

Keep open: current main does not consume `SKILL.md` `triggers` metadata or expose trigger keywords in `<available_skills>`, so this is a new public skill metadata and routing contract rather than conservative cleanup. The ClawHub `eyes` example in the issue is useful demand signal, but OpenClaw core needs maintainer API/product direction before implementation.

## What This Changes

Request optional `SKILL.md` `triggers` frontmatter, prompt rendering, and keyword-assisted skill routing.

## Best Possible Solution

Define and document trigger metadata semantics, then implement optional normalized parsing, prompt rendering, and routing coverage without changing behavior for skills that omit triggers.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives a high-confidence absence check for `triggers` consumption in current main.

## Solution Assessment

Unclear: the prompt-field approach is plausible, but maintainers should first choose the metadata shape, exact matching behavior, prompt-budget rules, and upstream formatter alignment.
