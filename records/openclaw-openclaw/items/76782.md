## Summary

Keep open: current main still lacks parsed or prompt-visible skill `triggers`, and adding it would define a new public skill metadata/routing contract rather than fix broken existing behavior. The ClawHub `eyes` example is useful demand signal, but this needs core prompt/metadata design.

## What This Changes

Request adding an optional `triggers` field to skill `SKILL.md` frontmatter and exposing it in `<available_skills>` for keyword-assisted routing.

## Best Possible Solution

Define an optional normalized trigger metadata contract, then update parsing, types, prompt rendering, docs, and focused regression tests while preserving current behavior for skills that omit it.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source inspection gives a high-confidence absence check for `triggers` in current skill parsing, prompt rendering, and docs.

## Solution Assessment

Unclear: the proposed prompt-field approach is plausible, but maintainers should first decide the public metadata shape, exact-match semantics, and how triggers interact with compact prompt mode.
