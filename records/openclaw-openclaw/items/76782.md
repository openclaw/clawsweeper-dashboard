## Summary

Keep open: current main still exposes skills to the model as name, description, and location only, so `triggers` would be a new public skill metadata and routing contract. The ClawHub `eyes` example and WSL2 40-skill report are useful demand signals, but the related implementation PR was closed unmerged and this needs maintainer API/product direction before cleanup can close it.

## What This Changes

The issue requests an optional `triggers` field in skill `SKILL.md` frontmatter for keyword-based skill routing.

## Best Possible Solution

Have maintainers decide the metadata shape, exact-match routing semantics, and prompt-budget contract, then implement optional normalized trigger parsing/rendering with docs and focused tests for full and compact skill prompt paths.

## Solution Assessment

Unclear: adding a prompt field is plausible, but field placement, exact-match priority, must-invoke behavior, and token-budget rules are public API/product decisions. A narrow implementation PR should follow that decision and update the docs/tests together.
