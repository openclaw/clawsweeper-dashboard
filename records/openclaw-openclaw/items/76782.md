## Summary

Keep open: current main still routes skills through name/description prompt metadata, and the requested `triggers` support would be a new public skill metadata and routing contract rather than an already-implemented cleanup. The ClawHub `eyes` reference in the issue is useful evidence of demand, but OpenClaw core does not yet consume that field.

## What This Changes

Add an optional `triggers` field to `SKILL.md` frontmatter and expose it for keyword-based skill routing.

## Best Possible Solution

Decide the skill metadata contract first; if accepted, implement optional trigger-list parsing, budget-aware prompt rendering, routing guidance, docs, and regression coverage without changing behavior for skills that omit the field.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection confirms current main lacks the requested field, but there is no broken existing contract to reproduce.

## Solution Assessment

Unclear: the proposed prompt field is plausible, but maintainers should choose whether `triggers` is top-level or namespaced metadata and how exact matching should interact with model-driven skill selection.
