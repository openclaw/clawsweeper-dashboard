## Summary

Keep open: current main still exposes skills to the agent through name, description, and location only, and adding `triggers` would create a new public skill metadata/routing contract. The linked ClawHub `eyes` skill is useful demand evidence, but OpenClaw core does not yet consume that field.

## What This Changes

The issue requests optional `triggers` frontmatter for skills and prompt/routing support for exact keyword activation.

## Best Possible Solution

Decide the skill metadata contract first; if accepted, implement optional trigger-list parsing, budget-aware prompt rendering/routing guidance, docs, and regression coverage without changing behavior for skills that omit it.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection on current main confirms no `triggers` contract is consumed or rendered, but there is no broken existing behavior to reproduce.

## Solution Assessment

Unclear: the proposed prompt field is plausible, but maintainers should decide whether `triggers` is top-level or namespaced metadata and how exact matching should interact with model-driven skill selection.
