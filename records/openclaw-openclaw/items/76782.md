## Summary

Keep open: current main still exposes skills to the agent through name, description, and location metadata only, so `triggers` would be a new public skill metadata and routing contract. The ClawHub `eyes` example is useful demand evidence, but OpenClaw core does not yet consume that field.

## What This Changes

The issue requests an optional `triggers` field in `SKILL.md` frontmatter and available-skills prompt metadata for exact keyword-based skill routing.

## Best Possible Solution

Decide the skill metadata contract first; if accepted, add optional trigger-list parsing, budget-aware prompt rendering, routing guidance, docs, and regression coverage while preserving behavior for skills without triggers.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection confirms current main lacks the requested `triggers` contract, but there is no broken existing behavior to reproduce.

## Solution Assessment

Unclear: a prompt-level trigger field is plausible, but maintainers should first decide whether it belongs as top-level frontmatter or namespaced metadata and how exact matching should interact with model-driven skill selection.
