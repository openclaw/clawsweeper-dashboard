## Summary

Keep open: current main still exposes skills to the agent through name/description/location metadata, and `triggers` would be a new public skill metadata and routing contract. The linked ClawHub `eyes` skill is useful demand evidence, but OpenClaw core does not yet consume that field.

## What This Changes

The issue requests an optional `triggers` field in skill `SKILL.md` frontmatter and prompt metadata for keyword-assisted skill routing.

## Best Possible Solution

Decide the skill metadata contract first; if accepted, implement optional trigger-list parsing, budget-aware prompt rendering, routing guidance, docs, and regression coverage without changing behavior for skills that omit the field.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection confirms current main lacks the requested trigger-list contract, but there is no broken existing behavior to reproduce.

## Solution Assessment

Unclear: adding a prompt field is plausible, but maintainers should first decide whether `triggers` is a top-level OpenClaw extension or namespaced metadata and how exact matching interacts with model-driven skill selection.
