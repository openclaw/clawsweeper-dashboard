## Summary

Keep open: current main still routes skills through name/description/location prompt metadata only, and `triggers` would be a new public skill metadata and routing contract rather than conservative cleanup.

## What This Changes

Request adding an optional `triggers` field to skill `SKILL.md` frontmatter and the `<available_skills>` prompt metadata for keyword-based skill routing.

## Best Possible Solution

Define the optional trigger metadata contract and matching semantics first, then parse a normalized string list, render it where appropriate, update docs/token-budget guidance, and add regression coverage while preserving behavior for skills that omit it.
