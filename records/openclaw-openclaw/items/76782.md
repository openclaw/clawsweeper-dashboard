## Summary

Keep open: current main still exposes skills through name, description, and location prompt metadata, and adding `triggers` would create a new public SKILL.md metadata and routing contract. The linked ClawHub `eyes` example is useful demand evidence, but OpenClaw core does not consume that field today.

## What This Changes

Request adds an optional `triggers` list to SKILL.md frontmatter and the `<available_skills>` prompt metadata for exact keyword-assisted skill routing.

## Best Possible Solution

Decide the skill metadata contract first; if accepted, parse an optional trigger list, render it budget-aware in `<available_skills>`, update routing guidance/docs/tests, and preserve behavior for skills without the field.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection verifies current main lacks trigger-list consumption, but there is no broken existing contract to reproduce.

## Solution Assessment

Unclear: the proposed prompt field is plausible, but maintainers should decide whether triggers are top-level or under `metadata.openclaw`, and how exact matching should combine with model-driven skill selection.
