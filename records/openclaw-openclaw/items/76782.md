## Summary

Keep open: current main still exposes prompt-facing skills as name, description, and location, and the system prompt tells the agent to scan descriptions. The ClawHub eyes example is useful demand signal, but `triggers` would be a new public skill metadata and routing contract rather than an already-implemented behavior.

## What This Changes

The issue requests an optional `triggers` field in `SKILL.md` frontmatter and in the available-skills prompt so exact keyword matches can help route skills.

## Best Possible Solution

Have maintainers define the trigger metadata shape and routing semantics, then implement optional normalized parsing, prompt rendering, docs, and regression coverage without changing behavior for skills that omit it.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source inspection of current main gives a high-confidence absence check for the requested `triggers` field.

## Solution Assessment

Unclear: the prompt-field approach is plausible, but maintainers should choose the public metadata shape, exact-match semantics, and prompt-budget contract before implementation.
