## What This Changes

Request to add an optional `triggers` field to `SKILL.md` frontmatter and use it for keyword-based skill routing alongside existing description matching.

## Best Possible Solution

Define the supported trigger metadata contract and routing semantics first, then update parsing, types, prompt rendering, compact-budget behavior, docs, Codex alignment, and focused tests together.

Yes for the source-level feature gap: current main and `v2026.6.10` omit trigger metadata from skill loading, types, prompt rendering, docs, and adjacent Codex skill contracts. This is not a failing bug reproduction because the issue requests new behavior.

## Solution Assessment

Unclear until maintainers choose the public contract. A `triggers` field is plausible, but the best implementation must settle field placement, exact-match semantics, compact prompt behavior, and Codex compatibility first.
