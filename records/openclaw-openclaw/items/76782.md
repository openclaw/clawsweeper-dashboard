## Summary

Keep open: current main still exposes skills to the agent through name, description, and location only, so `triggers` would be a new public skill metadata and routing contract rather than a cleanup close.

## What This Changes

Request to add an optional `triggers` field to `SKILL.md` frontmatter and include it in `<available_skills>` for keyword-based skill routing.

## Best Possible Solution

Have maintainers decide the public metadata shape and routing semantics, then implement optional normalized trigger parsing/rendering with docs and prompt-budget tests while preserving behavior for skills that omit it.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source inspection gives a high-confidence absence check for the requested `triggers` contract on current main.

## Solution Assessment

Unclear: adding a prompt field is plausible, but maintainers should first choose whether triggers are top-level frontmatter or `metadata.openclaw`, how exact matching interacts with model selection, and how the extra prompt tokens are budgeted.
