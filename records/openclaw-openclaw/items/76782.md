## Summary

Keep open: current main still exposes skills through name, description, and location only, so trigger support would add a new public skill metadata and routing contract. The ClawHub `eyes` example is useful demand signal, but this is not a ClawHub-only close because OpenClaw core does not currently consume that metadata.

## What This Changes

Request adding an optional `triggers` field to `SKILL.md` frontmatter and skill prompt/routing so exact keyword matches can supplement description matching.

## Best Possible Solution

Have maintainers settle the public metadata shape and routing semantics, then implement optional normalized trigger parsing/rendering with docs and focused regression coverage while preserving behavior for skills that omit it.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source inspection gives a high-confidence absence check for `triggers` in current skill prompt rendering, routing guidance, docs, and typed skill entries.

## Solution Assessment

Unclear: the proposed prompt field is plausible, but maintainers should first choose field placement, exact-match semantics, prompt-budget handling, and upstream formatter alignment.
