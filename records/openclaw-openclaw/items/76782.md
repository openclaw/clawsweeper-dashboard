## Summary

Keep open: current main still exposes skills through name/description/location prompt metadata, and `triggers` would be a new public skill metadata and routing contract. The ClawHub `eyes` reference is useful demand evidence, but OpenClaw core does not yet consume that field.

## What This Changes

The issue requests an optional skill `triggers` frontmatter field for keyword-based routing alongside existing skill name and description matching.

## Best Possible Solution

Decide the public skill metadata contract first; if accepted, add optional trigger parsing/rendering, budget-aware prompt guidance, docs, and regression coverage without changing behavior for skills that omit it.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection confirms current main lacks `triggers`, but there is no broken existing contract to reproduce.

## Solution Assessment

Unclear: the proposed prompt field is plausible, but maintainers should decide the metadata shape and how exact keyword matching should interact with upstream description-based skill selection.
