- `impact:session-state`: Plugin-created subagent runs are missing lifecycle registry state, so completion hooks cannot observe their end state.

## Summary

Keep open. Current main still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path without creating a `SubagentRunRecord`, while `subagent_ended` is emitted only from registry-tracked subagent records; the broader final-delivery behavior remains separately tracked at https://github.com/openclaw/openclaw/issues/55517.

## What This Changes

The issue requests that plugin SDK subagent runs created through `api.runtime.subagent.run()` participate in the existing subagent lifecycle so `api.on("subagent_ended")` handlers fire when the run finishes.

## Best Possible Solution

Register plugin-runtime subagent runs with the existing registry so `subagent_ended` fires, while keeping automatic final external delivery and `subagent_delivery_target` semantics separate unless maintainers resolve https://github.com/openclaw/openclaw/issues/55517.

## Reproduction Assessment

Yes, at source level. A plugin call to `api.runtime.subagent.run()` reaches Gateway `agent`, while only registered `SubagentRunRecord` entries can emit `subagent_ended`; I did not execute a live plugin harness in this read-only review.

## Solution Assessment

Yes for the narrow hook bug. The maintainable fix is to bridge plugin-created run ids into the existing registry with completion-message delivery disabled unless the broader delivery contract changes.
