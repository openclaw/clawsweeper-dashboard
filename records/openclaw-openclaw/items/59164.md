## Summary

Keep open. Current `main` still sends `api.runtime.subagent.run()` through the generic Gateway `agent` path, which tracks a CLI task but never registers the run with the subagent registry that emits `subagent_ended`.

## What This Changes

Plugin-created subagent runs should fire the existing `subagent_ended` hook when the run completes.

## Best Possible Solution

Register plugin-runtime subagent runs with the existing subagent registry lifecycle so `subagent_ended` fires, while keeping deferred external delivery behavior separate unless explicitly designed later.

## Reproduction Assessment

Yes. Source inspection on current `main` gives a high-confidence reproduction path: a plugin call to `api.runtime.subagent.run()` reaches Gateway `agent`, but only registered `SubagentRunRecord` entries can emit `subagent_ended`.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge plugin-created run ids into `registerSubagentRun()` or a small wrapper around it, with completion-message delivery disabled unless a separate API decision enables it.
