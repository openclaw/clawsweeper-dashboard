## Summary

Keep open. Current `main` still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path, which creates/finalizes a CLI task but does not register a `SubagentRunRecord`; the `subagent_ended` hook remains tied to registry entries created by the normal subagent spawn paths.

## What This Changes

The issue requests that subagent runs started with `api.runtime.subagent.run()` participate in the subagent completion lifecycle so `api.on("subagent_ended")` fires.

## Best Possible Solution

Add a narrow Gateway/plugin-runtime bridge that registers plugin-created subagent runs with the existing registry lifecycle so `subagent_ended` fires, with focused regression coverage and without changing default external delivery semantics.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. The reported plugin steps map directly to current `main`: `api.runtime.subagent.run()` reaches the generic Gateway `agent` task path, while the only paths that emit `subagent_ended` consume registered `SubagentRunRecord` entries.

## Solution Assessment

Yes. Reusing `registerSubagentRun()` or a small wrapper around it is the narrow maintainable fix; the implementation should keep completion-hook registration separate from any broader automatic delivery policy.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/59164. Current `api.runtime.subagent.run()` in `src/gateway/server-plugins.ts` dispatches Gateway method `agent`, and `src/gateway/server-methods/agent.ts` tracks that as a CLI task without registering a `SubagentRunRecord`, so plugin `api.on("subagent_ended")` handlers never fire. Add a narrow bridge so plugin-created subagent runs enter the existing subagent registry completion lifecycle and emit `subagent_ended` once when the run ends. Add regression coverage proving a plugin runtime `subagent.run()` completion invokes a registered `subagent_ended` hook. Likely files: `src/gateway/server-plugins.ts`, `src/gateway/server-methods/agent.ts`, `src/agents/subagent-registry.ts`, `src/agents/subagent-registry-completion.ts`, and tests under `src/gateway/`, `src/plugins/runtime/`, or `src/agents/`.
