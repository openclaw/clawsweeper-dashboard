## Summary

Keep open. Current `main` still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path, while `subagent_ended` is emitted only for registry-tracked `SubagentRunRecord` entries.

## What This Changes

The issue asks for `api.on("subagent_ended")` to fire when a plugin starts a run with `api.runtime.subagent.run()`.

## Best Possible Solution

Register plugin-runtime subagent runs with the existing subagent registry so `subagent_ended` fires, while leaving `subagent_delivery_target` and automatic final external delivery semantics to https://github.com/openclaw/openclaw/issues/55517.

## Reproduction Assessment

Yes. Source inspection on current `main` gives a high-confidence reproduction path: a plugin call to `api.runtime.subagent.run()` reaches Gateway `agent`, but only registered `SubagentRunRecord` entries can emit `subagent_ended`.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge plugin-created run ids into `registerSubagentRun()` or a small wrapper around it, with completion-message delivery disabled unless a separate API decision enables it.
