## Summary

Keep open. Current main still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path and returns a run id without creating a `SubagentRunRecord`, while `subagent_ended` is emitted only from registry-backed records; the related final-delivery issue is broader and does not supersede this narrow hook bug.

## What This Changes

Ensure plugin SDK subagent runs created by `api.runtime.subagent.run()` participate in the subagent lifecycle enough for `subagent_ended` handlers to fire.

## Best Possible Solution

Bridge plugin-created subagent run ids into the existing registry for lifecycle hook emission, with completion-message delivery disabled by default unless the broader delivery contract changes.

## Reproduction Assessment

Yes, at source level. A plugin call to `api.runtime.subagent.run()` reaches Gateway `agent`, while `subagent_ended` requires a registered `SubagentRunRecord`; I did not run a live plugin harness in this read-only review.

## Solution Assessment

Yes for the narrow hook bug. The maintainable fix is to register plugin-created runs with the existing subagent registry while keeping final external delivery disabled unless maintainers resolve the broader delivery contract separately.

Fix https://github.com/openclaw/openclaw/issues/59164. Plugin `api.runtime.subagent.run()` currently dispatches Gateway `agent` and returns a run id without registering a `SubagentRunRecord`, so `api.on("subagent_ended")` never fires for those runs. Add a narrow bridge that registers plugin-created subagent runs with the existing subagent registry after the Gateway `agent` run id is known, using compatibility-safe metadata and `expectsCompletionMessage: false` unless a separate maintainer decision changes final external delivery semantics. Do not solve or expand the broader final-delivery behavior tracked at https://github.com/openclaw/openclaw/issues/55517. Add focused tests that prove plugin runtime subagent runs can emit `subagent_ended` and that existing `sessions_spawn` behavior remains intact.
