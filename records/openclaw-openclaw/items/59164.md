## Summary

Keep open. Current main and the latest release still dispatch plugin subagent runs through Gateway `agent` without subagent registry lifecycle tracking, while `subagent_ended` is emitted only for registry-backed runs.

## What This Changes

The issue asks OpenClaw to make `api.runtime.subagent.run()`-created plugin subagent runs emit the existing `subagent_ended` hook on completion.

## Best Possible Solution

Register plugin-runtime subagent runs in the subagent registry for lifecycle-only tracking by default, add regression coverage for `subagent_ended`, and leave external completion delivery semantics to https://github.com/openclaw/openclaw/issues/55517.

## Reproduction Assessment

Yes, at source level. A plugin call to `api.runtime.subagent.run()` reaches Gateway `agent` without `registerSubagentRun()`, while `subagent_ended` is emitted from registry entries only.

## Solution Assessment

Yes for the narrow lifecycle bug if the bridge records plugin-runtime runs with completion delivery disabled by default. The broader question of automatic external delivery should stay with https://github.com/openclaw/openclaw/issues/55517.
