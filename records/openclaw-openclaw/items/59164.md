## Summary

Keep open. Current main still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path and never creates the registry record that the `subagent_ended` hook emission path requires; the related broader delivery issue remains open at https://github.com/openclaw/openclaw/issues/55517.

## What This Changes

The issue requests that subagent runs started through `api.runtime.subagent.run()` participate in the same registry-backed lifecycle that emits `subagent_ended` for internally spawned subagents.

## Best Possible Solution

Bridge plugin-runtime subagent runs into the existing registry lifecycle so `subagent_ended` fires once, while leaving automatic final external delivery semantics to the broader tracked work.

## Reproduction Assessment

Yes, at source level. A plugin call to `api.runtime.subagent.run()` reaches Gateway `agent`, while only registered `SubagentRunRecord` entries can emit `subagent_ended`; I did not execute a live plugin harness in this read-only review.

## Solution Assessment

Yes for the narrow hook bug. The maintainable fix is to bridge plugin-created run ids into the existing registry with completion-message delivery disabled unless maintainers separately change the broader delivery contract.
