## Label Justifications

- `P2`: This is a real plugin SDK lifecycle bug with limited blast radius and a clear source path, but it is not an emergency or core-runtime outage.
- `impact:session-state`: Plugin-created subagent runs are missing lifecycle registry state, so completion hooks cannot observe their end state.

## Summary

Keep open. Current main still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path without registering a `SubagentRunRecord`, while `subagent_ended` is emitted only from registry-tracked subagent records; the broader delivery policy remains tracked separately at https://github.com/openclaw/openclaw/issues/55517.

## What This Changes

Plugin SDK subagent runs should trigger the documented `subagent_ended` hook when the child run finishes.

## Best Possible Solution

Register plugin-runtime subagent runs with the existing subagent registry so `subagent_ended` fires, while keeping automatic final external delivery and `subagent_delivery_target` semantics separate unless maintainers explicitly resolve https://github.com/openclaw/openclaw/issues/55517.

## Reproduction Assessment

Yes, at source level. A plugin call to `api.runtime.subagent.run()` reaches Gateway `agent`, while only registered `SubagentRunRecord` entries can emit `subagent_ended`; I did not execute a live plugin harness in this read-only review.

## Solution Assessment

Yes for the narrow hook bug. The maintainable fix is to bridge plugin-created run ids into the existing registry or a small wrapper around it with completion-message delivery disabled unless the broader delivery contract changes.
