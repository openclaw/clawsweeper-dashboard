## Summary

Keep open: current main does not have this CLI subprocess inbound-turn file surface, and the implementation direction looks sound, but the new public runtime contract needs documentation before merge.

## What This Changes

The PR adds an `InboundTurnContext`, writes it to a per-session JSON file for CLI runs, exposes the file path via `OPENCLAW_INBOUND_TURN_FILE`, wires it from auto-reply execution, and adds tests plus a changelog entry.

## Best Possible Solution

Continue normal maintainer review; ClawSweeper found no patch-correctness issue.

## Reproduction Assessment

Not applicable: this is a feature PR rather than a broken existing contract. The after-fix proof gives a high-confidence live path for the proposed behavior: a WhatsApp turn through `claude-cli` writes the file and a wrapper consumes it.

## Solution Assessment

Yes, with documentation added. A stable env var pointing to a per-turn rewritten file is the maintainable shape for long-lived CLI sessions because spawn-time per-turn env values would go stale and current live-session fingerprinting includes env values.
