## Summary

## What This Changes

The PR adds an `InboundTurnContext`, writes current turn identifiers to a session-scoped JSON file, exposes that path to CLI subprocesses via `OPENCLAW_INBOUND_TURN_FILE`, wires auto-reply context into CLI runs, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land a hardened version with collision-free file identity, explicit failure semantics for turn-file refresh, no release-owned changelog edit, and focused validation before maintainer API-surface approval.
