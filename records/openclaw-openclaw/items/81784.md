## What This Changes

The PR adds an `InboundTurnContext` path from auto-reply into CLI agent preparation, writes the current turn identifiers to a session-scoped JSON file, exposes it via `OPENCLAW_INBOUND_TURN_FILE`, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the focused CLI inbound-turn file contract after normal maintainer review and fresh gates, keeping the identifier source channel-agnostic and covered by the added tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The PR body supplies after-fix live WhatsApp/`claude-cli` evidence showing the new file and wrapper path working in a real run.

## Solution Assessment

Yes. A stable env var pointing at a per-turn file fits long-lived CLI subprocesses better than per-turn env values, because the process env is captured when the subprocess is spawned while the file contents can be refreshed each turn.
