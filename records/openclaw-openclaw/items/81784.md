## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Fail closed when the inbound-turn file cannot refresh:** `src/agents/cli-runner/prepare.ts:252-257`
  - body: This catch only logs and continues the CLI run. For reused live-session subprocesses, the stable `OPENCLAW_INBOUND_TURN_FILE` path can still point at the previous turn's file, so a wrapper may read stale message/chat identifiers and send to the wrong target. Surface the failure or replace the file with a safe current-turn tombstone before allowing the run to proceed.
  - confidence: 0.88
- **[P1] Use a collision-free inbound-turn filename:** `src/agents/cli-runner/inbound-turn-file.ts:42`
  - body: Replacing every non `[A-Za-z0-9._-]` character with `_` is not injective: existing safe session ids can contain both `:` and `_`, so values like `a:b` and `a_b` resolve to the same JSON path. Because the payload carries sender/chat/message identifiers, a collision can expose another session's metadata and misroute replies. Use reversible encoding or a stable hash and add a regression test.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:17`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file, and the live review context reports this entry as the remaining merge conflict. Keep the release-note context in the PR body or squash message instead.
