## What This Changes

The PR adds a channel-agnostic inbound-turn metadata object, writes it to a per-session JSON file exposed through `OPENCLAW_INBOUND_TURN_FILE`, wires it into CLI agent runs, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land a hardened version with collision-free file identity, explicit failure semantics for turn-file refresh, no release-owned changelog edit, and focused validation of the CLI subprocess contract.

## Reproduction Assessment

## Solution Assessment

No: the file-based direction is reasonable for long-lived CLI sessions, but this patch is not the safest form until file identity is collision-free, failed refreshes cannot leave wrappers without trustworthy current-turn metadata, and release-owned changelog churn is removed.
