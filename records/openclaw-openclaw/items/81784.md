## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Fail closed when the inbound-turn file cannot refresh:** `src/agents/cli-runner/prepare.ts:252-257`
  - body: This catch only logs and then continues the CLI run. For a reused live-session subprocess, `OPENCLAW_INBOUND_TURN_FILE` is already fixed in the process environment and the old file remains at that path, so a wrapper can read the previous turn's message/chat identifiers and send to the wrong target. Surface the failure or replace the file with a safe current-turn tombstone before allowing the run to continue.
  - confidence: 0.88
- **[P1] Use a collision-free inbound-turn filename:** `src/agents/cli-runner/inbound-turn-file.ts:42`
  - body: Replacing every non `[A-Za-z0-9._-]` character with `_` is not injective: safe persisted session ids can contain both `:` and `_`, so values like `a:b` and `a_b` resolve to the same JSON path. Since this file carries sender/chat/message identifiers for subprocess reply routing, a collision can expose another session's turn data and misroute replies. Use a reversible encoding or hash and add a collision regression test.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:17`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file, and the live review context already reports this entry as the only merge conflict. Keep the release-note context in the PR body or squash message instead.
  - confidence: 0.93
