## Summary

Current main has stronger protected completion-event semantics than the reported June 6–7 behavior, but the exact harness-changing fallback remains unverified and an open draft PR still claims the issue. Keep this open until maintainers review, replace, or close that PR and establish whether current main reproduces the live failure.

## What This Changes

Prevent fallback turns from treating completed subagent results as new directives and re-executing or re-spawning already-finished work.

## Best Possible Solution

Use structured internal completion events as the single invariant, add a regression scenario that forces a harness-changing fallback after child completion, and add runtime-specific handling only if that shared-boundary test still fails.

## Reproduction Assessment

No, there is not yet a high-confidence failing reproduction on current main. The mechanism remains source-reproducible because model fallback can change runtime while completion events re-enter the active session, but the June 11 announcement refactor postdates the live incident.

## Solution Assessment

No, the linked CLI-only identity prompt is not yet the best demonstrated fix. The narrower maintainable direction is shared completion-event semantics with cross-harness regression proof.
