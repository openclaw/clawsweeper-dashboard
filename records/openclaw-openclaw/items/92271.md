## Summary

Current main does not enforce the reported parent/fallback invariant, and the linked draft PR remains open, conflicting, and narrower than the cross-harness problem. The issue should stay open for maintainer review of the shared completion-announcement boundary rather than spawning a second fix effort.

## What This Changes

Prevent fallback turns from misreading completed subagent announcements as fresh directives and re-executing already-finished work.

## Best Possible Solution

Carry an explicit non-directive completion state through the shared announcement/run boundary, apply the invariant consistently to every harness that can spawn work, and validate it with focused tests plus a real fallback session.

## Reproduction Assessment

No, not yet with high confidence. The live report is concrete and the source path is identifiable, but the duplicate spawn depends on model interpretation and reduced tool exposure rather than a deterministic current-main branch.

## Solution Assessment

No. The linked CLI identity prompt is a plausible mitigation for the observed runtime, but the narrowest maintainable fix should enforce completion semantics at the shared orchestration boundary and explicitly cover sibling spawning harnesses.
