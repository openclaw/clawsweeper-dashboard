## Summary

Keep this issue open: current main does not enforce the reported semantic completion invariant, and the paired draft PR remains the active but intentionally unfinished implementation path.

## What This Changes

Prevent fallback turns from treating completed background-task announcements as new work and spawning duplicate execution.

## Best Possible Solution

Represent completion announcements as explicit consumed task state at the shared run boundary, prevent only duplicate delegation of that completed work, and cover embedded, CLI/MCP, and Codex-native spawning through owner-specific enforcement or documented non-exposure.

## Solution Assessment
