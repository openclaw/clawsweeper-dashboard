## Summary

Keep open. Current main has added conservative stale-lane recovery and an extended embedded-run abort path, but the requested configurable active-stall recovery is not complete: active model/tool stalls still classify as non-recoverable, and open PR #73243 still specifically tracks this issue.

## What This Changes

The issue requests automatic gateway recovery for sessions stuck in `processing`, preferably with a separate abort threshold, so affected lanes resume without an external gateway restart.

## Best Possible Solution

Review and land PR #73243, PR #76115, or an equivalent source patch that adds a guarded configurable abort threshold, covers eligible active model/embedded stalls without killing healthy long turns, drains stale queued lanes, and updates tests, docs, schema, and changelog.

## Reproduction Assessment

Yes. The issue has production logs and deterministic Telegram/cron reproductions in comments, and current-main source inspection shows the remaining active model/tool stall path does not invoke recovery.

## Solution Assessment

No. Current main is a useful partial fix, but the narrowest complete solution still needs maintainer-approved abort policy and coverage for the eligible active-work stalls described in the report and comments.
