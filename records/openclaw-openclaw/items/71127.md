## Summary

Keep open. Current `main` has added partial stuck-session recovery, including stale-lane release and extended abort-drain for stalled embedded runs, but it still lacks the requested configurable abort threshold and this issue is paired with open closing PR #73243, so it should not be cleanup-closed before that PR or an equivalent fix is resolved.

## What This Changes

The issue requests automatic recovery for sessions that remain in `processing` after stuck-session diagnostics detect them, without requiring an external gateway restart.

## Best Possible Solution
