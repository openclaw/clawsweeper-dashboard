## Summary

Keep open. Current main has partial stuck-session recovery, but it still lacks the requested configurable abort threshold and leaves active model/tool stalls outside the recovery path, while open PR #73243 already tracks this issue with closing syntax.

## What This Changes

The issue asks for diagnostic stuck-session detection to abort or release sessions that remain in `processing` past a configurable threshold so the gateway can recover without an external restart.

## Best Possible Solution

Land PR #73243 or an equivalent source patch that adds a documented abort threshold, aborts only eligible stuck active work, drains stale lanes, and updates config, docs, tests, and changelog.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced in this read-only pass. The issue has deterministic production reports, and current main still shows active model/tool stall classifications that do not recover plus no independent abort threshold.

## Solution Assessment

Yes in direction, but not ready to close. A guarded diagnostics abort threshold with targeted lane/run recovery is the maintainable path; current main only partially implements it and the open closing PR still needs maintainer resolution.
