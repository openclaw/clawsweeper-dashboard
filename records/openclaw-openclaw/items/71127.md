## Summary

Keep open. Current main has meaningful partial recovery for stale/no-active-work and stalled embedded-run cases, but model/tool stall recovery and post-abort prompt cleanup are still not fully settled, and the issue discussion includes a maintainer note plus fresh release reports saying this remains live.

## What This Changes

Request automatic recovery for stuck `processing` sessions so diagnostics can abort or drain wedged work without an external gateway restart.

## Best Possible Solution

Keep this as the canonical core recovery-policy issue and land owner-reviewed fixes for the remaining model/tool stall, stale lane, and orphan prompt paths with focused regression coverage.

## Solution Assessment

Unclear as a single fix. The current embedded-run and stale-state recovery is a good base, but extending abort/drain semantics to model/tool stalls and orphan prompt cleanup needs maintainer-approved policy rather than a mechanical automation patch.
