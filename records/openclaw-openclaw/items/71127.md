Keep open. Current main has partial stuck-session recovery, but source still leaves stalled model calls outside diagnostic abort recovery, and recent production comments plus an open diagnostics PR show remaining stale-lane recovery cases.

## What This Changes

The issue requests self-recovery for sessions that stay in `processing` after diagnostics detect them as stuck, so gateway and channel progress resumes without an external restart.

## Best Possible Solution

Keep this as the canonical diagnostics recovery issue and land owner-reviewed fixes for remaining model/provider stall cancellation, stale idle or terminal embedded-run lane release, orphan prompt handling, and timeout policy with focused regression coverage.

## Solution Assessment

Unclear as an automated fix. The already-landed recovery path is useful, but expanding abort semantics across model calls, stale embedded-run handles, and transcript/orphan behavior needs maintainer-approved policy rather than a mechanical classifier flip.
