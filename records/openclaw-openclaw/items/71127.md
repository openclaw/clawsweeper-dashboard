## Summary

Keep open. Current main has partial stuck-session recovery for stale lanes and extended embedded-run stalls, but it still lacks the requested configurable abort threshold and still treats active model/tool stalls as non-recoverable.

## What This Changes

The issue requests aborting and recovering stuck `processing` sessions after a configurable diagnostic threshold instead of only logging repeated stuck/stalled warnings.

## Best Possible Solution

Review and land PR #73243, PR #76115, or an equivalent narrow patch that adds a separate guarded abort threshold, covers the reported active-stall cases, and updates schema, docs, tests, and changelog.

## Reproduction Assessment

Yes. The comments provide deterministic Telegram restart-window and cron `agentTurn` repros, and current-main source still leaves active model/tool stalls recovery-ineligible; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes in direction, but not yet on main. A separate guarded abort threshold is the narrow maintainable fix because warning, stale-lane release, and active-run termination have different operational risk.
