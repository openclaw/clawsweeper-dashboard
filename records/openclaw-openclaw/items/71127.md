Keep open. Current main has shipped partial stuck-session recovery, including `diagnostics.stuckSessionAbortMs` and stalled embedded-run aborts, but stale `model_call` and `tool_call` active work still remains observe-only, and the live maintainer note explicitly keeps this issue open for those follow-up recovery-policy surfaces.

The issue asks for diagnostics to abort or otherwise recover stuck processing sessions so gateway progress resumes without an external restart.

## Best Possible Solution
