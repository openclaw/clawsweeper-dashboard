## Summary

Keep open. Current main and v2026.5.12 still only cover the button-triggered Control UI confirmation; typed `/new`, typed `/reset`, shared channel reset handling, and Gateway agent resets remain immediate, and there is no shared configurable confirmation contract.

## What This Changes

Request a configurable confirmation step before destructive `/new` and `/reset` session reset flows.

## Best Possible Solution

Define one shared confirmation contract for user-authored destructive `/new` and `/reset`, with explicit automation/API bypasses and aligned config, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.12 shows immediate typed/shared/Gateway reset paths, and tests explicitly preserve immediate typed Control UI behavior; this is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear. The repeat-command timeout pattern discussed in comments is plausible, but maintainers still need to decide covered surfaces, timeout/config shape, and automation compatibility before implementation.
