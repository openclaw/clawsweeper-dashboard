## Summary

Keep open: current main still captures successful `assistantTexts` and records them to trajectory/result data, but I did not find a source path that reconciles a missing visible assistant turn back onto the active `SessionManager` branch. The merged #77033 overflow fix and current docs keep normal final-answer persistence owned by Pi, so they do not close this embedded-runner invariant gap.

## What This Changes

Ensure successful embedded PI runs that produce visible assistant text persist an equivalent visible assistant turn on the active transcript branch before later turns are assembled.

## Best Possible Solution

Add a focused embedded-runner transcript invariant that appends one equivalent visible assistant turn through the active session manager after a successful run with non-empty visible `assistantTexts`, while no-oping when Pi already persisted the turn.

## Reproduction Assessment

No: I did not establish a live current-main reproduction in this read-only review. The report and follow-up comments provide a concrete transcript/trajectory shape, and source inspection shows the successful `assistantTexts` output path without an active-branch reconciliation step.

## Solution Assessment

Yes: the narrowest maintainable fix is in the embedded PI transcript/session-manager path under the existing session lock. A channel or WebChat mirror would conflict with the current Pi ownership boundary and risks duplicate replay entries.

## Repair Work Prompt

Fix the embedded PI runner transcript invariant for https://github.com/openclaw/openclaw/issues/76990. After a successful embedded run produces non-empty visible `assistantTexts`, ensure the active persisted transcript branch contains exactly one equivalent visible assistant message reachable after the triggering prompt leaf before later user turns can append. Keep the fix in the embedded/Pi session-manager path under the existing session lock; do not add a config flag, do not add channel-specific behavior, and do not re-enable WebChat normal final-text mirroring. No-op when Pi already persisted an equivalent assistant turn, including whitespace-equivalent duplicates. Avoid using `delivery-mirror` entries for model replay because those are transcript-only. Add focused regression coverage for no-op when already persisted, append when missing, whitespace-equivalent duplicate suppression, and no-op when manager/session state is unavailable. Include a user-facing changelog fix entry unless the final patch is tests-only.
