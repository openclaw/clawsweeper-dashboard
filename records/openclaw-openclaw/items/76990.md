## Summary

Keep open: current main still collects successful embedded-run `assistantTexts` and records them in trajectory/output, but I found no current-main invariant repair that appends an equivalent visible assistant turn to the active PI transcript branch when that turn is missing. The related open PR #76969 addresses the neighboring orphaned-user leaf repair for #76968, not this successful-assistant-text persistence gap.

## What This Changes

This issue asks OpenClaw to persist each successful embedded-run visible assistant reply onto the active session transcript branch so later turns do not treat earlier user prompts as unanswered.

## Best Possible Solution

Add a focused embedded-runner transcript invariant repair after a successful run with non-empty visible `assistantTexts`, appending exactly one equivalent assistant message through the active `SessionManager` under the session lock only when the active branch is missing it.
