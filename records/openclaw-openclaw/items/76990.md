## Summary

Keep open: current main still captures successful embedded-run `assistantTexts` for trajectory/output, but I found no invariant repair that appends an equivalent visible assistant turn to the active PI transcript branch when PI did not persist it. The related open PR #76969 addresses the neighboring orphaned-user leaf repair for #76968, not this successful-assistant-text persistence gap.

## What This Changes

The issue asks OpenClaw to persist a visible assistant transcript turn after successful embedded PI runs when trajectory artifacts contain assistant text but the active session branch is missing it.

## Best Possible Solution

Add a focused embedded-runner transcript invariant repair: after a successful run with non-empty visible `assistantTexts`, ensure exactly one equivalent visible assistant turn exists on the active branch after the triggering prompt, appending through `SessionManager.appendMessage` under the session lock only when missing.
