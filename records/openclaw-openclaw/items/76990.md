## Summary

Keep open: current main still has a source-backed embedded-runner gap where successful `assistantTexts` are captured for output and trajectory, but no active-branch invariant ensures an equivalent visible assistant transcript entry is present for later turns. The related PRs/issues cover neighboring stale-replay, Codex mirror, or orphaned-user paths, not this successful embedded PI reply persistence path.

## What This Changes

The issue asks OpenClaw to persist visible assistant replies from successful embedded PI runs into the active transcript branch so later turns do not treat older prompts as unanswered.

## Best Possible Solution

Add a focused embedded-runner invariant that, after a successful run with non-empty visible `assistantTexts`, verifies the active branch after the prompt leaf and appends exactly one equivalent visible assistant message through the branch-safe session manager path when PI did not already persist it.
