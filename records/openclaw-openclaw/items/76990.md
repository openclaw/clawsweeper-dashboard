## Summary

Keep open: current main still captures successful embedded-run `assistantTexts` and records them in attempt artifacts, but I did not find the delivered-assistant transcript reconciliation on `main`. The exact repair appears to be in the still-open https://github.com/openclaw/openclaw/pull/77445 rather than already shipped.

## What This Changes

Ensure successful embedded PI runs that yield visible assistant text persist an equivalent assistant turn on the active transcript branch before later prompts are assembled.

## Best Possible Solution

Land or replace a narrow Pi embedded-runner/session-manager repair that persists exactly one normalized visible assistant turn for successful delivered `assistantTexts`, coordinated with the orphan-user persistence work.

## Reproduction Assessment

No live current-main reproduction was run. Source inspection plus multiple reporter comments show the path: `assistantTexts` can exist in attempt output while later prompt assembly relies on `activeSession.messages`, and current main lacks a reconciliation helper/test.

## Solution Assessment

Yes, the best direction is a narrow Pi-owned transcript invariant repair. It should avoid channel-specific delivery mirrors and use the open https://github.com/openclaw/openclaw/pull/77445 approach, or a successor, after resolving conflicts and adjacent orphan-user behavior.
