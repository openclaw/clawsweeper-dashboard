## Summary

Keep open: current main still has a source-backed gap between successful embedded-run `assistantTexts` capture and active-branch transcript persistence. Related #77033 fixed an overflow replay path and #76969 targets orphaned-user repair, but neither covers this successful-assistant persistence invariant.

## What This Changes

Ensure each successful embedded PI run with user-visible assistant text leaves an equivalent visible assistant turn on the active persisted transcript branch.

## Best Possible Solution

Add an idempotent embedded-runner transcript invariant that reconciles successful visible `assistantTexts` into the active SessionManager branch when Pi failed to persist them.

## Reproduction Assessment

No. I did not establish a live current-main reproduction, but source inspection shows the reported invariant gap: successful `assistantTexts` are recorded to trajectory without a post-success active-branch reconciliation step.

## Solution Assessment

Yes. The narrow maintainable fix is a Pi-owned, SessionManager-based idempotent reconciliation under the session lock; channel/WebChat mirroring is the less safe alternative because Pi owns normal agent-run persistence.

Fix openclaw/openclaw#76990 on current main. Add a focused embedded PI runner repair that, after a successful run with non-empty visible `assistantTexts`, verifies the active SessionManager branch after the prompt leaf contains an equivalent visible assistant reply and appends exactly one normal assistant message when it is missing. Keep the repair Pi/SessionManager-owned under the session lock; do not write raw JSONL, do not add a gateway/WebChat delivery mirror, and avoid duplicating already persisted or whitespace-equivalent assistant turns. Add regression coverage for no-op when already persisted, append when missing, no-op for whitespace-equivalent duplicates, and no-op when the manager/session path is unavailable. Include a user-facing changelog entry because this fixes visible stale replies. Coordinate with #76969 and #77033 but do not fold in their separate orphaned-user or overflow-compaction behavior.
