## Summary

Keep open: current main still lacks a post-success invariant that reconciles collected visible `assistantTexts` back into the active persisted transcript branch; related PR #76969 addresses the adjacent orphaned-user leaf path, not this embedded assistant-text persistence gap.

## What This Changes

The issue requests that successful embedded PI runner replies with visible assistant text always persist an equivalent assistant turn on the active transcript branch before later user turns are appended.

## Best Possible Solution

Add a small embedded-runner repair step that, after a successful attempt with non-empty visible assistant text and before releasing the session lock, verifies the active branch after the prompt leaf has an equivalent visible assistant message and appends exactly one when it is missing.
