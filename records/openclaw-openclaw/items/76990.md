## Summary

Keep open: current main still captures successful embedded-run visible assistant text for output/trajectory, but I did not find a current-main invariant that repairs a missing equivalent visible assistant turn on the active transcript branch. The later #77033 work addresses overflow replay and explicitly preserves Pi-owned transcript persistence, while comments on this issue report the broader missing-assistant condition persists outside overflow.

## What This Changes

The issue requests that every successful embedded Pi run with visible assistant text leave an equivalent visible assistant turn on the active persisted transcript branch before later turns assemble context.

## Best Possible Solution

Add a focused embedded-runner transcript invariant that appends one equivalent visible assistant message through SessionManager when a successful run produced visible assistantTexts but the active branch lacks that reply, while no-oping when Pi already persisted it.

## Repair Work Prompt

Fix issue #76990. Current embedded Pi runs can produce visible assistantTexts in trajectory/output while the active session transcript branch lacks an equivalent visible assistant reply, causing later turns to see prior user messages as unanswered. Add a focused embedded-runner invariant in the Pi/session-manager path: after a successful run with non-empty visible assistantTexts and no prompt error/abort/timeout, under the session write lock, check whether the active branch after the prompt leaf already contains an equivalent visible assistant message; if not, append exactly one normal visible assistant message through SessionManager.appendMessage or an existing wrapper. No-op when already persisted, when only whitespace-equivalent duplicates exist, when there is no manager/session path, or when the run is not successful. Do not add channel-specific mirroring, WebChat transcript writes, raw JSONL appends, or new config. Preserve the #77033 boundary that Pi owns persisted agent-run session entries. Add focused regression coverage for already persisted, missing append, whitespace-equivalent duplicate, and missing manager/session path. Add a user-facing changelog entry.
