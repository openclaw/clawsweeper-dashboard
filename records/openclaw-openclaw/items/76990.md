## Summary

Keep open: current main still has a source-backed gap where successful embedded-run `assistantTexts` are captured for delivery/trajectory, but there is no invariant repair that proves the same visible assistant text exists on the active transcript branch. The related #77033 release note covers overflow replay and WebChat duplicate-mirroring guardrails, while #76969 covers orphaned user repair, so neither fully covers this issue.

## What This Changes

Ensure every successful embedded PI run with visible assistant text persists one equivalent visible assistant turn on the active session transcript branch.

## Best Possible Solution

Add a focused embedded-runner/session-manager invariant that no-ops when the visible assistant reply is already on the active branch and appends one equivalent visible assistant turn under the existing session lock when a successful run produced visible `assistantTexts` but the active branch lacks them.
