## Summary

Keep open: current main still has a source-backed gap where embedded-run `assistantTexts` can be captured, delivered, and recorded in trajectory artifacts without a visible assistant turn reconciliation on the active persisted transcript branch.

## What This Changes

The issue reports successful embedded PI runner replies being present in run artifacts but missing from the active session transcript, causing later turns to answer old prompts again.

## Best Possible Solution

Enforce the embedded PI runner invariant under the session lock: a successful run with non-empty visible assistant text leaves exactly one equivalent visible assistant turn on the active persisted branch before later user turns can parent from it.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. The report and comments provide concrete trajectory/transcript evidence, and current source shows captured successful `assistantTexts` without a post-success active-branch reconciliation step.

## Solution Assessment

Yes: the narrowest maintainable fix is in the embedded PI runner/session-manager path, not channel-specific delivery or WebChat transcript mirroring. Related overflow and orphaned-user fixes should stay covered, but they do not replace the missing successful-assistant persistence invariant.
