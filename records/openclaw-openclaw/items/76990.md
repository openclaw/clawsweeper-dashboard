## Summary

Keep open: current main still has the reported transcript-invariant gap. The runner collects delivered `assistantTexts` and records them in attempt artifacts, but I did not find a main-branch reconciliation path that appends a missing visible assistant turn to the active session transcript; the focused fix remains the open, unmerged PR at https://github.com/openclaw/openclaw/pull/77445.

## What This Changes

The issue requests that every successful embedded PI run with user-visible assistant text persist an equivalent visible assistant turn on the active transcript branch before later turns are assembled.

## Best Possible Solution

Land or replace a narrow PI embedded-runner/session-manager repair that persists exactly one normalized visible assistant turn for successful delivered `assistantTexts`, without channel-specific transcript mirroring and while coordinating with orphan-user persistence work.

## Reproduction Assessment

No live current-main reproduction was run. The source path is reproducible by inspection: `assistantTexts` can be collected and written to attempt artifacts while future context is assembled from `activeSession.messages`, and current main lacks the reconciliation helper/test that would close the gap.

## Solution Assessment

Yes, the narrow PI-owned transcript reconciliation direction is the best fix shape. The existing open PR or a successor should handle it after review and real behavior proof, rather than adding channel-specific delivery mirrors or a config knob.
