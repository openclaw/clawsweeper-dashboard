Fix https://github.com/openclaw/openclaw/issues/76990. In the embedded PI runner, after a successful run with non-empty visible `assistantTexts`, ensure exactly one equivalent visible assistant message is present on the active persisted transcript branch after the triggering prompt. Prefer a focused helper under `src/agents/pi-embedded-runner/run/` and call it from `src/agents/pi-embedded-runner/run/attempt.ts` on the successful-run path. Use the active `SessionManager.appendMessage(...)` or an equivalent parent-linked/session-lock-safe path; do not raw-append Pi message entries in a way that can sever the parentId chain. No-op when an equivalent visible assistant message is already reachable on the active branch, when the text is whitespace-equivalent, or when the manager/session path is unavailable. Do not make this channel-specific, do not change provider fallback policy, and coordinate with but do not subsume #76969/#76968 orphaned-user repair. Add focused regression coverage for already persisted, missing append, whitespace-equivalent duplicate, and unavailable manager/session no-op.
