## Summary

Keep open: current main still has a source-backed gap where embedded-run `assistantTexts` are captured for payloads and trajectory, but the inspected successful-run path does not reconcile a missing visible assistant message back onto the active transcript branch. The related #77033 fix is intentionally scoped to overflow replay/WebChat mirroring, and #76969 targets the neighboring orphaned-user repair rather than this successful-assistant persistence invariant.

## What This Changes

The issue requests that successful embedded PI runs with visible assistant text persist an equivalent visible assistant turn on the active transcript branch used for later context.

## Best Possible Solution

Add a focused embedded-runner/session-manager invariant that appends exactly one equivalent visible assistant turn when a successful run produced visible assistantTexts but Pi did not leave one on the active branch.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. The issue provides concrete trajectory/transcript evidence, and current source shows assistantTexts captured and returned without a matching successful-run active-branch reconciliation step.

## Solution Assessment

Yes: the narrow maintainable fix belongs in the embedded PI runner/session-manager transcript path, not in Discord/Telegram/WebChat delivery mirroring. #77033 and #76969 should remain separate because they cover adjacent replay/orphaned-user paths, not this successful assistant persistence invariant.
