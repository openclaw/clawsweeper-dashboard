## Summary

Keep open: current main still has a source-backed gap where successful embedded-run `assistantTexts` are captured for delivery and trajectory, but no inspected PI success path enforces an equivalent visible assistant turn on the active transcript branch. The issue discussion adds confirming evidence across WebChat, Discord, Telegram, WhatsApp, and multiple providers, while related #77033 fixed only overflow replay and #76969/#76968 cover a neighboring orphaned-user repair.

## What This Changes

The issue reports that successful embedded PI runner replies can be present in run artifacts but absent from the persisted active transcript branch, causing later prompts to re-answer older user turns.

## Best Possible Solution

Add a focused embedded PI runner/session-manager invariant repair after successful attempts with non-empty visible assistant text, under the session lock, no-oping when an equivalent assistant turn already exists and avoiding WebChat/channel-side transcript mirroring.
