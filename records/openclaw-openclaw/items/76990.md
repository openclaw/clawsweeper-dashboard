## Summary

Keep open: current main still lacks a PI-owned invariant that reconciles successful run-local `assistantTexts` with a missing visible assistant turn on the active transcript branch, and the merged overflow replay fix does not cover the non-overflow missing-assistant case reported by two users.

## What This Changes

The issue requests that every successful embedded PI run with visible assistant text leaves an equivalent visible assistant message on the active persisted transcript branch before later turns are assembled.

## Best Possible Solution

Add a focused embedded-runner/session-manager invariant that appends exactly one equivalent visible assistant turn on the active branch after a successful run with non-empty visible `assistantTexts`, while preserving PI transcript ownership and avoiding duplicate turns.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. The issue provides concrete trajectory/session evidence from two reporters, and current source shows successful `assistantTexts` are captured and returned without an obvious active-branch reconciliation invariant.

## Solution Assessment

Yes. The narrow maintainable fix is a PI-owned transcript invariant using the session manager active branch, not a channel-specific workaround, Gateway delivery mirror, or raw JSONL append.
