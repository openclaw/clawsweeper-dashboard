## Summary

Keep this open. The WebSocket keepalive subcase has shipped, but current main still documents and tests Mattermost DMs as non-threaded, still suppresses DM reply targets/history/mentions, and lacks a Mattermost thread-history backfill path; the open related PRs only cover narrower context slices.

## What This Changes

Requests native Mattermost threading reliability support: configurable keepalive, DM replyToMode parity, thread history backfill, and bare-mention wake-up handling.

## Best Possible Solution

Keep the issue open and split the remaining Mattermost work into an explicit opt-in DM threading/history contract plus a separate bare-mention wake-up fix, while preserving the shipped keepalive path.

## Reproduction Assessment

Yes. Source inspection on current main shows direct chats still force replyToMode off, suppress effective reply targets and history, exclude direct chats from mention detection, and drop mention-only normalized bodies; I did not run a live Mattermost workspace.

## Solution Assessment

No current implementation fully solves the issue, and applying the proposed workaround as-is would reverse a documented and tested DM contract. The maintainable path is an explicit opt-in DM threading/backfill design with schema, docs, tests, and a narrow bare-mention fix.
