## Summary

Keep open. Current main has shipped Mattermost WebSocket keepalive, but the remaining DM threading, thread-history backfill, and bare-mention behavior are still not implemented and require a Mattermost product/compatibility decision rather than cleanup closure.

## What This Changes

Issue requests Mattermost WebSocket keepalive, configurable DM thread parity, thread history backfill after restart, and bare mention handling.

## Best Possible Solution

Keep the shipped keepalive behavior, then design an explicit opt-in Mattermost DM threading/history contract and a separate bare-mention wake-up fix with docs, schema, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main still forces direct chats to replyToMode "off", suppresses direct thread/session history paths, and drops mention-only normalized bodies; I did not run a live Mattermost server.

## Solution Assessment

No current implementation fully solves the issue. Applying the workaround directly would conflict with the documented flat-DM contract, so the safer path is an explicit opt-in design plus separate bare-mention handling.
