## Summary

Keep this open. Current main covers the WebSocket ping/pong keepalive subcase, but the requested DM threading parity, DM/thread history persistence, and bare-mention wake-up behavior remain unresolved and still conflict with the documented flat-DM contract.

## What This Changes

The issue requests native Mattermost support for configurable WebSocket keepalive, DM reply-threading parity with groups, thread-history backfill after memory loss, and handling bare bot mentions as wake-up messages.

## Best Possible Solution

Keep this as Mattermost-plugin-owned reliability work: preserve flat DMs by default, add explicit opt-in DM/threading semantics if desired, define safe thread-history backfill rules, and handle mention-only wake-ups with schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main for the remaining work: direct chats still force reply mode/effective reply id/history off, and mention-only messages still normalize to an empty body and return early; the WebSocket keepalive subcase no longer reproduces on main.

## Solution Assessment

No current implementation fully solves the issue. The keepalive fix is appropriate for one subcase, but the remaining DM threading and backfill behavior should be solved through an explicit opt-in contract and access-control review rather than silently changing existing DM defaults.
