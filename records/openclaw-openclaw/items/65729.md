## Summary

Keep this open. Current main covers the WebSocket keepalive subcase, but the remaining DM threading parity, DM/thread-history persistence, and bare-mention wake-up behavior are still unresolved and would change the documented Mattermost DM contract.

## What This Changes

The issue requests native Mattermost threading reliability improvements covering WebSocket keepalive, configurable DM threading parity, thread-history backfill after restarts, and bare-mention wake-up behavior.

## Best Possible Solution

Keep this as Mattermost-plugin reliability work: preserve the shipped keepalive path, then split the remaining changes into explicit opt-in DM threading/backfill semantics and a narrow bare-mention fix with schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main for the remaining requested behavior: direct chats still force reply mode, effective reply id, and history off, and mention-only messages still normalize to an empty body and return early.

## Solution Assessment

No current implementation fully solves the issue. The keepalive implementation is the right direction for one subcase, but DM threading and history backfill need an explicit opt-in contract and access-control review rather than silently changing default DM behavior.
