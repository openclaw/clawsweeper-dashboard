## Summary

Keep this open. The Mattermost WebSocket keepalive slice is implemented and shipped, but current main still documents and enforces flat direct messages, suppresses DM thread reply/history/mention paths, and lacks Mattermost thread-history backfill, so the remaining request needs owner/product review rather than cleanup closure.

## What This Changes

Request native Mattermost support for proxy-safe WebSocket keepalive, configurable DM threading parity, thread-history backfill after restart/session clear, and bare-mention wake-up handling.

## Best Possible Solution

Keep the shipped keepalive behavior, then split the remaining work into an explicit opt-in Mattermost DM threading/history contract and a separate bare-mention wake-up fix with docs, schema, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence path: direct chats still force replyToMode off, suppress effective reply targets/history/mention detection, and drop a mention-only normalized body; I did not run a live Mattermost server.

## Solution Assessment

No current implementation fully solves the issue. Applying the workaround as-is would override a documented and tested flat-DM contract; the maintainable route is explicit opt-in DM threading/backfill behavior plus a narrow bare-mention fix.
