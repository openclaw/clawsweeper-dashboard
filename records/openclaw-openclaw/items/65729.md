## Summary

Keep this open. Current main and v2026.5.7 cover the WebSocket keepalive subcase, but DM threading parity, DM/thread-history persistence, and bare-mention wake-up handling are still unresolved and would change the documented Mattermost DM contract.

## What This Changes

Request native Mattermost support for configurable WebSocket keepalive, DM replyToMode parity, thread-history backfill after restarts, and bare @mention wake-up handling.

## Best Possible Solution

Keep the issue open, preserve the shipped keepalive path, and split the remaining Mattermost plugin work into an explicit opt-in DM threading/backfill contract plus a narrow bare-mention fix with schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: direct chats still force reply mode, reply targets, mention detection, and history off, and mention-only messages normalize to an empty body and return early; I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

No current implementation fully solves this issue. The keepalive implementation is the right resolution for one subcase, but DM threading/backfill should be an explicit maintainer-approved contract rather than an automatic semantic change.
