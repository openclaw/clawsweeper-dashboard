## Summary

Keep open: current main and v2026.6.6 include Mattermost WebSocket ping/pong keepalive, but the remaining DM configurable threading, thread-history backfill, and bare-mention behavior are still missing and source-reproducible.

## What This Changes

Requests native Mattermost support for configurable DM threading parity, thread-history persistence/backfill, WebSocket keepalive behind proxies, and bare mention handling.

## Best Possible Solution

Keep the merged keepalive implementation, then have maintainers split or approve focused Mattermost-plugin tracks for dmReplyToMode, thread-history backfill, and bare-mention dispatch with docs and regression coverage.

## Reproduction Assessment

Yes, source-reproducible: direct chats still force reply threading off, use no history key, and empty normalized messages return before dispatch. I did not run a live Mattermost workspace reproduction.

## Solution Assessment

No, closing as implemented would be wrong because only the WebSocket keepalive slice is done. The best path is a maintainer-approved split that preserves existing non-threaded DM defaults unless a new explicit DM threading config is accepted.
