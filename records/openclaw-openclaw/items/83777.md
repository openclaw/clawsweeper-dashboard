## Summary

Keep open: the exact `groupPolicy: "open"` report conflicts with current source and existing tests, but there is a real source-visible diagnostic gap when WhatsApp group allowlisting drops an inbound group at default log levels.

## What This Changes

The issue reports WhatsApp group inbound messages being silently dropped unless the group is configured under `channels.whatsapp.groups`, even when `groupAllowFrom` is set.

## Best Possible Solution

Preserve the documented split between group admission and sender authorization, then add actionable default-level diagnostics when a WhatsApp group allowlist drops an inbound group.

## Reproduction Assessment

Unclear for the exact listed steps: current source and tests say `groupPolicy: "open"` with no populated `groups` map should route. Yes for the diagnostic gap: source shows a missing-group allowlist drop returns after only verbose logging.

## Solution Assessment

Yes for a warning/docs fix, not for changing routing semantics. `groupAllowFrom` should remain sender authorization, while missing group admission should produce an actionable warning when it blocks dispatch.
