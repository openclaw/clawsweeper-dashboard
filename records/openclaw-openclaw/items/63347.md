## Summary

Keep open. Current main still lacks MS Teams webhook host binding, so this PR is not obsolete; it remains the active narrow implementation candidate, but it needs small repair work before merge for bind logging, quick-start docs, generated config metadata, and changelog coverage.

## What This Changes

The PR adds optional `channels.msteams.webhook.host` support to MS Teams config schema/types, forwards it to the Express webhook listener, updates bind logging/docs, and adds lifecycle coverage.

## Best Possible Solution

Land the host-binding feature after preserving omitted-host behavior, logging the actual or explicitly unspecified bind address, keeping the baseline docs externally reachable, documenting loopback as an optional hardened tunnel/proxy setup, regenerating bundled channel config metadata, and adding the required changelog entry.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing host support across runtime, types, strict schema, generated metadata, and docs; the PR blockers are visible in the provided diff and review comments.

## Solution Assessment

No as-is. The narrow host-binding approach is the right maintainable solution, but the current patch needs the logging, docs, metadata, and changelog fixes before it is merge-ready.
