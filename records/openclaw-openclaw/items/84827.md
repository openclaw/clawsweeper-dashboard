Keep open: this is a focused Control UI shortcut feature, but the patch still leaves one document-level Escape path unguarded for editable targets. It is also member-authored with the protected maintainer label, so cleanup should not close it.

## What This Changes

Adds a Control UI dashboard keyboard shortcut dispatcher for `/`, `N`, `Esc`, and Cmd/Ctrl+K, updates the New messages affordance, adds UI tests, and adds a changelog entry.

## Best Possible Solution

Keep the centralized dispatcher, but make every document-level Escape path share the editable-target guard and cover mobile controls plus composer/search inputs in regression tests.

## Reproduction Assessment

Yes for the review finding: source inspection shows editable-target Escape returns no dashboard action in the new dispatcher, then the still-registered mobile-controls document handler can close the controls. I did not run a live browser because this review is read-only.

## Solution Assessment

No, not yet. The centralized dispatcher is the right shape, but the existing mobile-controls Escape handler must share the same text-entry guard or be folded into the dispatcher before this is the best fix.
