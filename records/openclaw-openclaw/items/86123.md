## What This Changes

The PR adds an AsyncLocalStorage-backed active same-session event-processing guard around Pi embedded session event queue waits and adds regression coverage for both the self-wait bypass and detached async queue-drain behavior.

## Best Possible Solution

Land the narrow same-session self-wait guard with its regression tests after maintainer review accepts the supplied runtime-helper proof for this P1 Gateway availability path.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: current main and v2026.5.22 unconditionally wait on the active _agentEventQueue, and the PR adds a RED/GREEN test showing the pre-fix path times out. I did not rerun tests in this read-only review.

## Solution Assessment

Yes, the proposed fix is the narrowest maintainable shape I found: skip the queue wait only for the active same-session event entry, then clear that active state so detached async work still drains normally. Broader timeout or lock-policy changes would be riskier than this targeted guard.
