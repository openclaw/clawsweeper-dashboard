## Label Justifications

- `P1`: The linked report describes a 2026.5.22 Gateway availability regression where Feishu replies and health/status probes can stall during embedded Pi tool calls.
- `merge-risk: 🚨 session-state`: The PR changes session event queue and write-lock ordering, and an overbroad skip can let same-session hook work bypass the existing queue-drain contract.

## Summary

Keep open: current main still has the session event queue self-wait path and this PR is a plausible narrow fix, but the guard should be made active-state bounded before merge so async context created during event processing cannot skip later queue drains.

## What This Changes

The PR adds an AsyncLocalStorage-based same-session event-processing guard around embedded Pi session event handling and adds a regression test for the beforeToolCall self-wait path.

## Best Possible Solution

Keep the same-session self-wait skip, but store a mutable active event-processing state and clear it after _processAgentEvent completes, with coverage proving in-entry hooks skip the wait while detached or external hooks still drain pending queue work.

## Reproduction Assessment

Yes; current main has beforeToolCall waiting on the same session queue that can contain the active _processAgentEvent call, and the PR supplies a RED/GREEN regression for that path. I did not rerun tests in this read-only review.

## Solution Assessment

No; the proposed boundary is right, but the implementation should clear or deactivate the AsyncLocalStorage state after the active event entry finishes to avoid leaking the skip into later async work.

- **[P2] Clear the active event context after processing returns:** `src/agents/pi-embedded-runner/run/attempt.session-lock.ts:493-495`
  - body: waitForSessionEventQueue now skips whenever the AsyncLocalStorage store equals the session.
