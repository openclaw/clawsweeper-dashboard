## Summary

Keep open: this is a focused fix for a plausible P1 Pi/Gateway self-wait hang, but it still needs maintainer review and stronger real-runtime proof before merge because the session queue guard changes session-state behavior.

## What This Changes

Adds an AsyncLocalStorage active session-event context around Pi `_processAgentEvent`, skips same-session queue waits during active event processing, and adds session-lock regression tests.

## Best Possible Solution

Land the narrow same-session active event guard only after redacted real Pi/Gateway runtime proof shows the tool-call path unblocks while detached and external queue drains still wait.

## Reproduction Assessment

Yes for a source-level reproduction: current main waits on `_agentEventQueue` before `beforeToolCall`, while the same queue entry can be processing the tool-call event. I did not establish a live Gateway/Feishu reproduction in this read-only review.

## Solution Assessment

Yes, the proposed boundary is the right narrow shape if the runtime path matches the helper: skip only the active same-session event-processing wait and preserve detached/external drains. The merge should wait for real runtime proof because the code relies on private Pi session queue behavior.
