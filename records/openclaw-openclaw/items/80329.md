## Summary

Keep open. Current main still lacks any request-scoped, model-hidden sandbox environment injection path from the OpenAI-compatible HTTP endpoint into sandboxed tool exec, and the requested secret-bearing header/config surface needs maintainer product and security design before implementation.

## What This Changes

The issue requests an off-by-default gateway option that lets trusted OpenAI-compatible HTTP callers pass per-request environment variables into sandboxed tool execution without exposing the values to the model context.

## Best Possible Solution

If maintainers accept the feature, add a documented off-by-default gateway option plus a command-context carrier that validates request env names and injects redacted, non-persisted values only at sandbox process spawn.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives high confidence that current main has no hidden per-call sandbox env path from `/v1/chat/completions` to sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. A trusted, opt-in header path fits nearby `x-openclaw-*` conventions, but the header format, validation, redaction, lifetime, and env precedence need owner review first.
