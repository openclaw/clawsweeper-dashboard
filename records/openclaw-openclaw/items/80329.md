## Summary

Keep open: current main does not have a request-scoped sandbox environment channel from the OpenAI-compatible HTTP ingress, and the request is a new security-sensitive feature rather than a stale or already-implemented bug fix.

## What This Changes

The issue requests an opt-in way for trusted operator callers to pass per-call environment variables into sandboxed tool execution without putting the values in model context.

## Best Possible Solution

Approve a single request-scoped sandbox env contract first, then implement it through OpenAI-compatible ingress, AgentCommandOpts, sandbox env construction, sanitizer/redaction, docs, and focused gateway/exec tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does give high confidence that current main has no header/config path that can carry per-call values into sandbox exec env.

## Solution Assessment

Unclear until maintainers approve the security contract. The proposed header approach is plausible, but the narrow maintainable solution should reuse the existing host-env safety checks and redaction paths rather than inheriting broad host or request state.
