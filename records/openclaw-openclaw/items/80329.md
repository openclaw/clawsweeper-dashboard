## Summary

Keep open. Current main still has no request-scoped, model-hidden sandbox env path from the OpenAI-compatible chat completions endpoint into sandbox exec, and the requested secret-bearing header/config surface needs maintainer security and product design rather than conservative cleanup closure.

## What This Changes

The issue requests an off-by-default OpenAI-compatible gateway option to pass request-scoped, model-hidden sandbox environment variables from trusted operator calls into tool exec.

## Best Possible Solution

If accepted, add a documented off-by-default gateway option and command-context carrier that validates request-scoped env, injects it only at sandbox exec spawn, and proves values bypass prompts, transcripts, persisted state, and logs.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high confidence that current main has no hidden per-call sandbox env path from `/v1/chat/completions` to sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. A trusted-header opt-in fits nearby `x-openclaw-*` conventions, but the exact wire shape, validation policy, redaction behavior, and interaction with existing sandbox env/SecretRef surfaces need design review first.

## Security Review

Status: needs_attention

Summary: The requested feature would move operator-supplied secrets into executable sandbox environments, so it needs explicit security design before implementation.

Concerns:

- **[medium] Define redaction and non-persistence before implementation:** `src/gateway/openai-http.ts:920`
  - body: Request-scoped secret values must not enter prompts, transcripts, runtime snapshots, logs, or persisted session state; tests should prove only redacted values or env names surface outside the spawned process env.
  - confidence: 0.88
- **[medium] Validate injected env names before spawn:** `src/infra/host-env-security.ts:95`
  - body: Request-supplied env names can alter process behavior if dangerous names such as loader hooks, shell startup vars, PATH-like controls, proxy/TLS controls, or package-manager config are accepted without a clear policy.
  - confidence: 0.86
- **[medium] Document the operator and tool-output boundary:** `docs/gateway/openai-http-api.md:47`
  - body: The endpoint is already documented as full operator access, but sandboxed skill code can still print env values, so any accepted contract should make that exposure and caller responsibility explicit.
  - confidence: 0.82
