## Summary

Keep open. Current main still lacks a request-scoped, model-hidden sandbox env path from the OpenAI-compatible HTTP endpoint into exec spawn, and the requested header/config surface is a security-sensitive feature that needs maintainer product design.

## What This Changes

Add an off-by-default gateway mechanism for trusted OpenAI-compatible HTTP callers to pass per-request sandbox environment variables into exec without putting values in model context.

## Best Possible Solution

If accepted, define a documented opt-in gateway contract that carries validated request-scoped env only to sandbox exec spawn, with regression coverage proving values bypass prompts, transcripts, persisted state, and logs.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives high confidence that current main has no hidden per-call sandbox env path from the OpenAI-compatible HTTP endpoint into sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. Header-based opt-in fits nearby x-openclaw conventions, but validation, redaction, lifetime, and interaction with configured sandbox env need design review first.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move operator-supplied secret values from HTTP headers into executable sandbox environments.

Concerns:

- **[medium] Define redaction and non-persistence before implementation:** `src/gateway/openai-http.ts:957`
  - body: Request-scoped secret values must not enter prompts, transcripts, config snapshots, logs, or persisted session state; the accepted design should include tests for those guarantees.
  - confidence: 0.89
- **[medium] Validate injected env names before spawn:** `src/infra/host-env-security.ts:95`
  - body: Request-supplied env names can alter runtime behavior if loader, shell startup, proxy/TLS, package manager, or workspace-control variables are accepted without the existing env security policy.
  - confidence: 0.87
- **[medium] Document the tool-output exposure boundary:** `docs/gateway/openai-http-api.md:51`
  - body: The endpoint is documented as full operator access, but sandboxed tools can still print injected env values; operators need explicit guidance that OpenClaw cannot hide env from code that receives it.
  - confidence: 0.82
