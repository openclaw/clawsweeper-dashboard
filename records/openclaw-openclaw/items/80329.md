## Summary

Keep open. Current main still has no request-scoped, model-hidden sandbox environment carrier from the OpenAI-compatible HTTP endpoint to sandbox exec, and the requested header/config path is a new secret-bearing operator surface that needs maintainer product and security design.

## What This Changes

Add an off-by-default OpenAI-compatible HTTP header/config path so trusted callers can inject request-scoped sandbox env vars without exposing values to the model.

## Best Possible Solution

If maintainers accept the feature, add a documented off-by-default gateway option and command-context carrier that injects validated request-scoped env only at sandbox exec spawn, with tests proving values bypass prompts, transcripts, persisted state, and logs.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high confidence that current main has no hidden per-call sandbox env path from /v1/chat/completions to sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. Header-based opt-in fits nearby x-openclaw conventions, but the header shape, validation, redaction, lifetime, and relation to configured sandbox env need design review first.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would move operator-supplied secret values into executable sandbox environments.

Concerns:

- **[medium] Define redaction and non-persistence first:** `src/gateway/openai-http.ts:898`
  - body: Request-scoped secret values must not enter prompts, transcripts, config snapshots, logs, or persisted session state; tests should prove only names or redacted values can surface outside the spawned process env.
  - confidence: 0.9
- **[medium] Validate injected env names before spawn:** `src/infra/host-env-security.ts:95`
  - body: Request-supplied env can alter process behavior if names such as PATH, loader hooks, shell startup vars, or proxy/TLS controls bypass the existing host env security policy.
  - confidence: 0.88
- **[medium] Document the operator and tool-output boundary:** `docs/gateway/openai-http-api.md:49`
  - body: The OpenAI-compatible endpoint is documented as full operator access, but env values handed to sandboxed tools can still be printed by skill code, so the accepted contract should make that exposure explicit.
  - confidence: 0.84
