## Summary

Keep open. Current main and the latest release still have no request-scoped, model-hidden sandbox env path from the OpenAI-compatible HTTP endpoint into sandbox exec, and the requested header/config surface is security-sensitive enough to need maintainer product review before implementation.

## What This Changes

The issue requests an off-by-default way for trusted OpenAI-compatible HTTP callers to pass request-scoped sandbox environment variables into tool exec without exposing values to model context.

## Best Possible Solution

If accepted, add a documented off-by-default gateway option plus a command-context carrier that injects validated request-scoped env only at sandbox exec spawn, with tests proving values bypass prompts, transcripts, persisted state, and logs.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and shipped-release inspection give high confidence that there is currently no hidden per-call sandbox env path from /v1/chat/completions to sandbox exec.

## Solution Assessment

Unclear until maintainers define the security and product contract. A header-gated opt-in fits the nearby x-openclaw-* convention, but the exact wire shape, validation, redaction, lifetime, and precedence rules need design approval first.

## Security Review

Status: needs_attention

Summary: The request would introduce operator-supplied secret material into sandbox process environments, so trust, validation, redaction, and output-exposure rules need maintainer approval before implementation.

Concerns:

- **[medium] Define redaction and non-persistence before implementation:** `src/gateway/openai-http.ts:957`
  - body: Request-scoped secret values must not enter prompts, transcripts, config snapshots, logs, or persisted session state; tests should prove that only names or redacted values can surface outside spawned process env.
  - confidence: 0.88
- **[medium] Validate injected env names before spawn:** `src/infra/host-env-security.ts:119`
  - body: Request-supplied env can alter process behavior if PATH, loader hooks, shell startup vars, or proxy/TLS controls are accepted without the existing host env security policy.
  - confidence: 0.86
- **[medium] Document tool-output exposure clearly:** `docs/gateway/openai-http-api.md:51`
  - body: The endpoint is already an operator-access surface, but env values handed to sandboxed tools can still be printed by skill code, so the accepted contract should state that boundary explicitly.
  - confidence: 0.82
