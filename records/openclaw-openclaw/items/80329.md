## Summary

Keep open. Current main still has no request-scoped, model-hidden sandbox env path from the OpenAI-compatible HTTP endpoint into sandbox exec, and the requested secret-bearing header/config surface needs maintainer security and product design rather than cleanup closure. The related custom-header request at https://github.com/openclaw/openclaw/issues/54177 is adjacent but broader; this issue has the more concrete sandbox-env contract.

## What This Changes

The issue requests an off-by-default gateway feature for request-scoped sandbox environment variables from trusted OpenAI-compatible HTTP callers.

## Best Possible Solution

If accepted, add a documented, off-by-default gateway option and command-context carrier that injects validated request-scoped env only at sandbox exec spawn, with tests proving values bypass prompts, transcripts, persisted state, and logs.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection on current main gives a high-confidence verification path showing no hidden per-call sandbox env carrier from `/v1/chat/completions` into sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. Header-based opt-in fits nearby `x-openclaw-*` conventions, but the exact header shape, validation, redaction, lifetime, and interaction with configured sandbox env need design review first.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move operator-supplied secret values into executable sandbox environments.

Concerns:

- **[medium] Define redaction and non-persistence before implementation:** `src/gateway/openai-http.ts:920`
  - body: Request-scoped secret values must not enter prompts, transcripts, config snapshots, logs, or persisted session state; tests should prove only names or redacted values surface outside the spawned process env.
  - confidence: 0.88
- **[medium] Validate injected env names before spawn:** `src/infra/host-env-security.ts:95`
  - body: Request-supplied env can alter process behavior if names such as PATH, loader hooks, shell startup vars, or proxy/TLS controls are accepted without the existing host env security policy.
  - confidence: 0.86
- **[medium] Document the operator and tool-output boundary:** `docs/gateway/openai-http-api.md:47`
  - body: The OpenAI-compatible endpoint is documented as full operator access, but env values handed to sandboxed tools can still be printed by skill code, so the accepted contract should make that exposure explicit.
  - confidence: 0.82
