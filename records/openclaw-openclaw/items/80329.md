## Summary

Keep open. Current main still lacks a hidden per-call env path from `/v1/chat/completions` to sandbox exec, and the request is a new security-sensitive header/config feature that needs maintainer design.

## What This Changes

Request to add an off-by-default OpenAI-compatible gateway header/config path for operator-trusted per-call sandbox environment injection that bypasses prompt context.

## Best Possible Solution

If accepted, add a documented, off-by-default gateway option and command-context carrier that injects validated request-scoped env only at sandbox exec spawn, with tests proving values bypass prompts/transcripts/logs and dangerous names are rejected.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection against current main gives high confidence that no per-call sandbox env carrier exists from `/v1/chat/completions` to sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. A header-based opt-in fits the existing `x-openclaw-*` convention, but the header shape, validation, redaction, lifetime, and precedence need design review first.
