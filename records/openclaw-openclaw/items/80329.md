## Summary

Keep open. Current main has the OpenAI-compatible operator HTTP surface and sandbox env composition seam, but no hidden per-call sandbox env path from request headers into sandbox exec; adding one is a new security-sensitive gateway/config feature that needs maintainer design.

## What This Changes

Request to add an off-by-default gateway header/config contract that lets trusted operator callers inject per-request environment variables into sandboxed tool execution without exposing values to the model.

## Best Possible Solution

If accepted, add a documented off-by-default gateway option and request-context carrier that injects validated per-request env only at sandbox exec spawn, with tests proving values bypass prompts, transcripts, logs, snapshots, and persisted session state.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives high confidence that current main has no hidden per-call sandbox env path from /v1/chat/completions to sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. A header-based opt-in fits the existing x-openclaw convention, but header shape, trust scope, validation, redaction, and precedence need design review first.
