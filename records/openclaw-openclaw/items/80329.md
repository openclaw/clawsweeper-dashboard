## Summary

Keep open. Current main still lacks a request-scoped, model-hidden sandbox env carrier from the OpenAI-compatible HTTP endpoint into sandbox exec, and the requested header/config surface would move secret values into executable environments, so it needs maintainer product and security design rather than cleanup closure.

## What This Changes

The issue requests an off-by-default OpenAI-compatible HTTP header/config path for operator-trusted callers to inject per-call sandbox environment variables without exposing values to the model context.

## Best Possible Solution

If accepted, add a documented off-by-default gateway option and command-context carrier that injects validated request-scoped env only at sandbox exec spawn, with coverage proving values bypass prompts, transcripts, persisted state, and logs.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives high confidence that current main has no hidden per-call sandbox env path from /v1/chat/completions to sandbox exec.

## Solution Assessment

Unclear until maintainers choose the product and security contract. Header-based opt-in fits nearby x-openclaw-* conventions, but the exact wire shape, validation, redaction, lifetime, and relation to configured sandbox env need design review first.
