## Summary

Keep open: current `main` and the latest release still contain the disputed alias sentence, while current docs, source, and tests show `/new` and `/reset` are intentionally different on Control UI and TUI surfaces.

## What This Changes

The issue requests correcting slash-command docs and related command copy so `/new` is described as fresh session creation and `/reset` as an in-place current-session reset, not as aliases.

## Best Possible Solution

Update the slash-command docs and nearby ACP/TUI/help copy to distinguish `/new` as fresh session creation or switching and `/reset` as current-session reset, while leaving command behavior and names unchanged.

## Reproduction Assessment

Yes. Reading current `main` reproduces the exact disputed docs line, and source plus regression tests show `/new` and `/reset` now take different paths.

## Solution Assessment

Yes. A narrow docs and command-copy correction is the maintainable fix; renaming `/reset` would be a separate product/API change and should not be bundled into this cleanup.
