## Summary

Keep open. Current main and the latest shipped tag still use one string-valued `tools.sessions.visibility` scope for send, history, list, and status access; the requested per-capability object form is not implemented, and the adjacent open A2A-send, allowlist, and per-agent scoping items do not supersede this permission-model request.

## What This Changes

The issue requests a backward-compatible structured `tools.sessions.visibility` object with separate send, history, list, and metadata controls.

## Best Possible Solution

Define one backward-compatible, fail-closed session-tool capability model that keeps string shorthands, validates an object form, preserves sandbox and A2A restrictions, updates schema/types/help/docs, and adds focused access-control tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source, docs, tests, and the v2026.5.19 tag give high confidence that the requested object form does not exist today.

## Solution Assessment

Unclear until maintainers choose the public permission model. The object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, sandbox clamps, and memory/status semantics.
