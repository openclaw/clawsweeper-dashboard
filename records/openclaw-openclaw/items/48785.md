## Summary

Keep open. Current main and latest release still expose one string-valued `tools.sessions.visibility` dial, and the related A2A, allowlist, and per-agent scoping items are adjacent rather than a merged or safe supersession of this per-capability permission model.

## What This Changes

Request to replace the single `tools.sessions.visibility` scope with per-capability session permission knobs for send, history, list, and metadata.

## Best Possible Solution

Define one coherent session-access model that preserves existing string shorthands, separates send/read/list/status only after security review, and updates schema, docs, generated help, and isolation tests together.

## Reproduction Assessment

Yes for the config gap: source inspection shows current main accepts only string visibility modes and routes send/history/list/status through one effective visibility model. It is a feature/config limitation rather than a failing current behavior repro.

## Solution Assessment

Unclear. The requested object form is a plausible least-privilege design, but maintainers need to choose between it, A2A-send-only gating, allowlists, and per-agent scoping before implementation.
