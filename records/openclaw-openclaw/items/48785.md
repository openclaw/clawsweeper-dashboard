## Summary

Keep open. Current main and the latest release still expose `tools.sessions.visibility` as a string-only, single-scope setting shared by send, history, list, and status; related A2A, allowlist, and per-agent proposals are adjacent but not a merged or safe canonical replacement for this per-capability permission model.

## What This Changes

Request to split `tools.sessions.visibility` into per-capability controls so cross-agent sends can be allowed without also granting transcript history or broad listing access.

## Best Possible Solution

Define one backward-compatible session-tools authorization model that preserves existing string shorthands, deliberately separates or rejects per-action send/read/list/status permissions, and updates schema, docs, generated help, and focused isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested object form and still enforces one visibility scope.

## Solution Assessment

Unclear until maintainers choose the permission model. The requested object form is plausible, but a narrower A2A-send exception, per-agent outbound model, or allowlist model may better fit OpenClaw's config-surface bar.
