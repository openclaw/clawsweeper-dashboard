## Summary

Keep open. Current main still exposes `tools.sessions.visibility` as one string-valued scope and applies it across send, history, list, and status access; the related A2A-send, allowlist, and per-agent scoping items are adjacent but do not supersede this per-capability permission-model request.

## What This Changes

Request adding backward-compatible per-capability session visibility flags for send, history, list, and metadata so cross-agent sends can be allowed without transcript/list/status access.

## Best Possible Solution

Define one backward-compatible, fail-closed session-tool capability model that preserves string shorthands, validates object-form permissions, keeps sandbox/A2A clamps intact, and updates schema, generated help, docs, and access-control tests together.

## Reproduction Assessment

Not applicable as a feature/config request rather than a bug report. Source inspection gives high confidence that current main still accepts only the string visibility model and uses one shared guard across session-tool actions.

## Solution Assessment

Unclear until maintainers choose the public session access-control model. The proposed object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, and sandbox clamp semantics.
