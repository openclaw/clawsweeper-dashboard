## Summary

Keep open. Current main and the latest release still expose `tools.sessions.visibility` as one string-valued scope that is shared across send, history, list, and status access; the adjacent A2A, allowlist, and per-agent scoping items do not implement or supersede the requested per-capability object model.

## What This Changes

Request to add a backward-compatible object form for `tools.sessions.visibility` with separate send, history, list, and metadata permissions.

## Best Possible Solution

Design one coherent, backward-compatible session-tool capability model that preserves string shorthands, validates a fail-closed object form, keeps sandbox/A2A restrictions intact, and updates schema, docs, generated config metadata, and focused access-control tests together.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection shows current main still has string-only `tools.sessions.visibility` and one shared visibility guard for send, history, list, and status.

## Solution Assessment

Unclear. The object model is plausible, but maintainers should choose it alongside the open A2A-send, allowlist, and per-agent scoping proposals before implementation.
