## Summary

Keep open. Current main still exposes `tools.sessions.visibility` as a single string scope shared by send, history, list, and status access, and the live related A2A/allowlist/per-agent items remain adjacent rather than superseding this permission-model request.

## What This Changes

This issue requests an additive object form for `tools.sessions.visibility` with separate send, history, list, and metadata permissions.

## Best Possible Solution

Define one backward-compatible session-tool capability model that keeps string shorthands, validates a fail-closed object form, preserves sandbox/A2A restrictions, and updates schema, docs, generated help, and access-control tests together.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing behavior. Source inspection gives high confidence that current main still has a string-only schema and one shared session visibility guard.

## Solution Assessment

Unclear until maintainers choose the public session access-control model. The proposed object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, sandbox clamps, and memory/session-discovery visibility.
