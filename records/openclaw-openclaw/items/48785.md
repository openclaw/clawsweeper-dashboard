## Summary

Keep open. Current main and v2026.6.6 still expose one string-valued session visibility scope shared by send, history, list, and status; the related open A2A, allowlist, and per-agent scoping items are useful context but do not implement this per-capability authorization model.

## What This Changes

Request to replace the single string-valued `tools.sessions.visibility` scope with backward-compatible per-capability session-tool visibility controls.

## Best Possible Solution

Define one maintainer-approved, backward-compatible session-tool authorization model that preserves legacy string shorthands, adds any approved capability granularity, keeps sandbox/A2A restrictions fail-closed, and updates schema, docs, and focused isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested object form and still enforces one shared visibility scope.

## Solution Assessment

Unclear until maintainers choose the permission model. The object form is plausible, but a narrower A2A-send exception, allowlist model, or per-agent outbound model may better satisfy OpenClaw's config-surface bar.
