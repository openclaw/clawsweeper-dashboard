## Summary

Keep open. Current main and v2026.5.18 still use one string-valued `tools.sessions.visibility` scope shared across send, history, list, status, and session-memory visibility, while the related open A2A-send, allowlist, and per-agent scoping items do not implement the requested structured per-capability model.

## What This Changes

Request a backward-compatible object form for `tools.sessions.visibility` with separate send, history, list, and metadata permissions.

## Best Possible Solution

Define one backward-compatible, fail-closed session-tool capability model that preserves string shorthands and sandbox/A2A restrictions, then update config schema/types/docs and access-control tests for send, history, list, status, and memory search.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, related-item, and v2026.5.18 tag checks give high confidence that the requested per-capability object form does not exist today.

## Solution Assessment

Unclear until maintainers choose the public permission model. The proposed object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, sandbox clamps, and memory/status semantics.
