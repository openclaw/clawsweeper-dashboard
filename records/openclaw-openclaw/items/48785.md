## Summary

Keep open. Current main and the latest release still expose one string-valued `tools.sessions.visibility` scope and apply it across send, history, list, and status access; the open A2A-send, allowlist, and per-agent scoping items are adjacent but do not supersede this per-capability permission model request.

## What This Changes

Request to extend `tools.sessions.visibility` from a single scope enum into per-capability send/history/list/metadata controls while preserving existing string shorthands.

## Best Possible Solution

Define one backward-compatible, fail-closed session-tool capability model that keeps string shorthands, validates a per-capability object form, preserves sandbox and A2A restrictions, updates schema/types/help/docs, and adds focused access-control tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, tests, and the v2026.5.20 tag give high confidence that the requested object form does not exist today.

## Solution Assessment

Unclear until maintainers choose the public permission model. The proposed object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, sandbox clamps, memory search, and status semantics.
