## Summary

Keep open. Current main and v2026.6.6 still use one string-valued `tools.sessions.visibility` scope for send, history, list, and status; the requested per-capability authorization object remains unimplemented and needs maintainer product/security review.

## What This Changes

Request to add per-capability session visibility controls for send, history, list, and metadata instead of one shared string enum.

## Best Possible Solution

Define one backward-compatible session-tool authorization model that preserves the existing string shorthands, adds any approved capability granularity, keeps sandbox and A2A restrictions fail-closed, and updates schema, docs, and focused isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested object form and still enforces one shared visibility scope.

## Solution Assessment

Unclear until maintainers choose the permission model. The proposed object form is plausible, but a narrower A2A-send exception, allowlist model, or per-agent outbound model may better satisfy OpenClaw's config-surface bar.
