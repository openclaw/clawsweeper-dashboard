## Summary

Keep open. Current main and the latest release still expose one string-valued `tools.sessions.visibility` scope and apply it across send, history, list, and status access; related A2A-send, allowlist, and per-agent scoping work is adjacent but does not implement the requested per-capability permission model.

## What This Changes

The issue requests a backward-compatible object form for `tools.sessions.visibility` with separate send, history, list, and metadata permissions.

## Best Possible Solution

Define one backward-compatible, fail-closed session-tool capability model that keeps string shorthands, adds a validated per-capability object form, preserves sandbox and A2A restrictions, updates schema/types/help/docs, and adds focused access-control tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, tests, and the `v2026.5.22` tag give high confidence that the requested object form does not exist today.

## Solution Assessment

Unclear until maintainers choose the public permission model. The proposed object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, sandbox clamps, memory search, and status semantics.
