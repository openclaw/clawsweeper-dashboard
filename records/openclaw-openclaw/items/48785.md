## Summary

Keep open. Current main and the latest release still expose `tools.sessions.visibility` as one string-valued scope, and the related A2A, per-agent, and allowlist items are adjacent product paths rather than a merged or exact supersession of this per-capability permission model.

## What This Changes

The issue requests a structured per-capability `tools.sessions.visibility` model so cross-agent sends can be allowed without also granting transcript history, listing, or metadata visibility.

## Best Possible Solution

Define one backward-compatible session-access model that preserves the current string shorthands, clearly separates send/read/list/status or intentionally chooses a narrower A2A-send path, and updates schema, docs, generated help, and focused isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does give high confidence that current main lacks the requested object form and still enforces a single visibility scope.

## Solution Assessment

Unclear until maintainers choose the permission model. The requested object form is plausible, but a narrower A2A-send exception or per-agent allowlist model may be the preferred design if it better fits OpenClaw's config-surface bar.
