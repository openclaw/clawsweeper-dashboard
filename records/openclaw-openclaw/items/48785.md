## Summary

Keep open. Current main and v2026.5.18 still model `tools.sessions.visibility` as one string scope shared by send, history, list, and status access; the requested per-capability object form is not implemented, and the related A2A-send, allowlist, and per-agent scoping items are adjacent rather than superseding.

## What This Changes

Add a backward-compatible structured `tools.sessions.visibility` form with separate send, history, list, and metadata/session-status capabilities.

## Best Possible Solution

Define one backward-compatible, fail-closed session-tool capability model that keeps string shorthands, validates a structured object form, preserves sandbox and A2A restrictions, and updates schema/types/help/docs plus focused access-control tests together.

## Reproduction Assessment

Not applicable as a feature/config request. Source, docs, tests, and the v2026.5.18 tag give high confidence that the requested object form does not exist today.

## Solution Assessment

Unclear until maintainers choose the public permission model. The proposed object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, sandbox clamps, and memory/status semantics.
