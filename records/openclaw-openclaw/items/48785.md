## Summary

Keep open. Current main and the latest release still expose one string-valued session visibility scope shared by send, history, list, and status, and the adjacent A2A, allowlist, and per-agent scoping threads do not implement or safely supersede this per-capability permission model.

## What This Changes

Request to replace the single `tools.sessions.visibility` enum with backward-compatible per-capability controls for send, history, list, and metadata access.

## Best Possible Solution

Define one coherent session-tool authorization model that preserves existing string shorthands, deliberately separates or rejects per-action send/read/list/status permissions, and updates schema, docs, generated help, and isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main and `v2026.6.6` lack the requested object form and still enforce one visibility scope.

## Solution Assessment

Unclear until maintainers choose the permission model. The requested object form is plausible, but a narrower A2A-send exception, per-agent outbound model, or allowlist model may better fit OpenClaw's high bar for new config surfaces.
