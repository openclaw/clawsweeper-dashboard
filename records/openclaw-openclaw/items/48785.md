## Summary

Keep open. Current main and v2026.6.10 still expose `tools.sessions.visibility` as one string enum and route send, history, list, and status through the same visibility guard; adjacent open items cover send-only, allowlist, per-agent, and sandbox scoping, but none implements the requested per-capability object model.

## What This Changes

Request to add backward-compatible per-capability session visibility controls so cross-agent sends can be allowed without granting transcript, list, or metadata access.

## Best Possible Solution

Define one maintainer-approved session authorization model that preserves existing string shorthands, adds any approved capability granularity, composes explicitly with sandbox and A2A policy, and updates schema, docs, generated help, and isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; yes for source verification. Current main and v2026.6.10 lack the object form and still route send/history/list/status through one visibility enum.

## Solution Assessment

Unclear until maintainers choose the permission model. The proposed object form is plausible, but a narrower send-only exception, allowlist model, or per-agent scoping model may be the better durable config surface.
