## Summary

Keep open. Current main and the latest release still expose one string-valued session visibility scope shared by send, history, list, and status; related A2A, allowlist, and per-agent scoping work is adjacent but does not implement this per-capability authorization model.

## What This Changes

The issue requests structured per-capability `tools.sessions.visibility` controls so cross-agent send can be allowed without granting transcript, list, or metadata access.

## Best Possible Solution

Define one backward-compatible session-tool authorization model, then update schema/types, guard semantics, all session-tool callers, docs/generated config help, and focused isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested object form and still enforces one visibility scope.

## Solution Assessment

Unclear until maintainers choose the permission model. The requested object form is plausible, but a narrower A2A-send exception, allowlist model, or per-agent outbound model may better fit OpenClaw's config-surface bar.
