## Summary

Keep open. Current main still exposes `tools.sessions.visibility` as one string-valued scope and applies that scope across send, history, list, and status/metadata access; the requested per-capability object model is not implemented, and the adjacent open A2A-send, allowlist, and per-agent scoping items do not supersede this permission-model request.

## What This Changes

Request to replace string-only `tools.sessions.visibility` with backward-compatible per-capability visibility flags for send, history, list, and metadata.

## Best Possible Solution

Define one backward-compatible session-tool capability model that keeps string shorthands, adds a validated fail-closed object form, preserves sandbox and A2A restrictions, and updates schema/types/help/docs plus focused access-control tests together.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that current main and the latest release still use a string-only visibility schema and one shared visibility guard.

## Solution Assessment

Unclear until maintainers choose the public session access-control model. The proposed object form is plausible, but it should be designed together with A2A send, allowlist, per-agent scoping, and sandbox clamp semantics.
