## Summary

Keep open: current main has default-hidden reasoning safeguards, but the source still leaves a plausible answer-first path for explicit visible Telegram reasoning with text-end block streaming, and the existing labels call for maintainer/security review plus live proof.

## What This Changes

The issue reports Telegram delivering Anthropic extended-thinking blocks as visible messages after the final assistant text instead of hiding them by default or ordering/coalescing them before the answer when visible.

## Best Possible Solution

Keep default Telegram reasoning hidden, and make explicit visible reasoning flush as one coalesced reasoning payload before any final answer or use the existing transient reasoning preview path without durable post-answer thinking messages.

## Reproduction Assessment

Partly. Source inspection gives a clear reproduction path for explicit visible reasoning with text-end block streaming, but this review did not run a live Telegram/Anthropic turn to reproduce the original default-session leak.

## Solution Assessment

No. Current main’s default-hidden behavior is the right safe default, but the explicit visible-reasoning path still needs ordering/coalescing proof or a small routing fix before the issue can close.
