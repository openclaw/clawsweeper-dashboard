## Summary

Keep open: current main appears to cover the default-hide direction for native thinking, but the issue also asks that explicitly visible reasoning be ordered/coalesced before the final Telegram answer, and the current default `text_end` block path still has source-level ordering risk without live Telegram proof.

## What This Changes

The issue reports Anthropic native thinking blocks becoming user-visible Telegram messages after the final assistant answer, with excessive message fanout.

## Best Possible Solution

Keep reasoning hidden by default, and when an operator explicitly enables visible reasoning on Telegram, deliver one coalesced reasoning lane before the final answer or use the transient `/reasoning stream` draft path with live Telegram proof.
