## Summary

Keep open: current main still uses the old array-style rate-limit summary, and the proposed branch is not safe to merge because it removes an SDK export, breaks Telegram's empty-response fallback, truncates active changelog content, and leaves Codex expectations stale.

## What This Changes

This PR changes the Codex command formatter to label account rate-limit windows from `windowDurationMins`, while also modifying Telegram native-command handling/tests, the session-store SDK re-export, and the changelog.

## Best Possible Solution

Land a narrow Codex rate-limit formatter that reads `RateLimitWindow.windowDurationMins`, updates Codex tests and changelog, and preserves Telegram native-command and SDK session-store behavior.

## Reproduction Assessment

Yes for source-level reproduction: current main routes Codex rate-limit values through `summarizeArrayLike`, which cannot render object-shaped `rateLimits.primary/secondary.windowDurationMins`. The PR-introduced blockers are also visible directly from the diff and current contracts.

## Solution Assessment

No. Duration labels are a reasonable target, but this branch is not the best solution because it carries unrelated SDK, Telegram, and changelog regressions; a narrow Codex formatter/test/changelog patch is safer.
