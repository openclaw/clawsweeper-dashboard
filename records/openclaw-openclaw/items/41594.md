## Summary

Keep open. Current main has partial knobs for context size and tool/skill filtering, but it does not provide the requested per-model or per-alias context profile, and the no-tools path still leaves default tool guidance in the system prompt.

## What This Changes

The issue requests per-model or per-alias profiles to control workspace file injection, tool visibility, and skill visibility.

## Best Possible Solution

Define a first-class model or alias profile contract that composes with model identity, fallback behavior, workspace bootstrap filtering, tool policy, and skill eligibility, while splitting any narrow no-tool prompt fix if maintainers want that landed sooner.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is source-verifiable on current main: config/schema surfaces lack contextProfile, and prompt assembly has no model-scoped workspace or skills filter.

## Solution Assessment

Unclear. The proposed alias-scoped contextProfile is plausible, but maintainers should choose how it relates to model-profile keys, fallback-agent context, existing per-agent skills/tools policy, and provider compat before implementation.
