## Summary

Keep open: current main still has global/per-agent prompt controls and provider-model tool compatibility knobs, but not the requested alias-scoped context profile for workspace files, skills, and tool visibility. Related open model-key and fallback-context threads overlap, but neither fully supersedes this feature request.

## What This Changes

Request an optional per-model alias or provider `contextProfile` that controls injected workspace files, tool visibility, and skill descriptions.

## Best Possible Solution

Define one backward-compatible alias/model-scoped profile contract, then thread it through config schema/help/docs, bootstrap filtering, skills prompt assembly, tool construction, Control UI, and `/model` switching coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. The missing capability is source-verifiable because current strict schemas and prompt assembly paths have no alias/model context-profile selector.

## Solution Assessment

Unclear until maintainers choose the schema and precedence rules. The requested opt-in profile is plausible, but it should be coordinated with https://github.com/openclaw/openclaw/issues/59168 and the fallback-context design in https://github.com/openclaw/openclaw/issues/15073.
