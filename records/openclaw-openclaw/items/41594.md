## Summary

Keep this open: current main still has global/per-agent prompt controls and provider-model tool compatibility knobs, but not the requested model-alias context profile for workspace files, skills, and tool visibility.

## What This Changes

The issue requests an optional alias/model-scoped context profile that controls workspace file injection, skill prompt inclusion, and tool visibility.

## Best Possible Solution

Define one backward-compatible alias/model-scoped profile surface, coordinate it with #59168, then thread it through config schema/help, bootstrap file filtering, skills prompt assembly, tool exposure, Control UI/docs, and focused `/model` switching coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. The missing capability is source-verifiable because the strict model-alias schema and prompt assembly paths have no alias/model context-profile selector.

## Solution Assessment

Unclear until maintainers settle the model-profile schema and precedence rules. Existing knobs are useful mitigations, but they are global, provider-model, or per-run controls rather than the requested alias-scoped profile.
