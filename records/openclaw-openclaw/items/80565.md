## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80565. On arm64 local Docker builds, the root Dockerfile runs `pnpm_config_verify_deps_before_run=false pnpm canvas:a2ui:bundle` after `COPY . .`, and the reporter sees pnpm retry optional native binary downloads for unrelated platforms before the stub fallback. Replace only that Docker A2UI layer with `node scripts/bundle-a2ui.mjs` or an equally narrow no-outer-pnpm path that preserves the existing non-fatal stub fallback. Update Dockerfile coverage so the A2UI Docker layer no longer uses `pnpm canvas:a2ui:bundle`, while keeping the existing `pnpm_config_verify_deps_before_run=false` guards for other build scripts. Add a CHANGELOG entry because this is a user-visible Docker build fix. Do not change the package manager, pnpm lockfile policy, Docker base images, bundled plugin dependency ownership, or add a new config flag unless the direct-node path proves impossible.
