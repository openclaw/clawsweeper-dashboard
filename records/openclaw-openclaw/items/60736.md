## Summary

Keep open. Current main has partial Google Vertex transport and ADC support, but it still does not satisfy the requested location fallback/default, onboarding/auth choice, or clear provider activation behavior; two open implementation PRs already target this issue with closing syntax.

## What This Changes

The issue requests real Google Vertex AI support in the bundled Google plugin, including Vertex-specific OAuth/scopes, project/location configuration, endpoint routing, and onboarding.

## Best Possible Solution

Land or consolidate one canonical Google plugin implementation that documents and tests project/location resolution, GOOGLE_CLOUD_LOCATION and CLOUD_ML_REGION handling with a safe us-central1 default, Vertex REST routing, ADC/OAuth bearer auth, onboarding selection, and model listing behavior.

## Reproduction Assessment

Yes. ## Solution Assessment

No. Closing this issue now or launching a third repair is not the best path; the narrow maintainable path is to review and land #60860, #62103, or a consolidated successor, then close this after the merged implementation covers the missing fallback and onboarding behavior.
