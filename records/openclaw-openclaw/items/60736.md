## Summary

Keep open. ## What This Changes

Request proper Google Vertex AI support in the Google provider, including location config/fallbacks, Vertex endpoint routing, OAuth/ADC auth, and onboarding.

## Best Possible Solution

Consolidate one Google-plugin-owned Vertex path that supports project/location resolution, GOOGLE_CLOUD_LOCATION and CLOUD_ML_REGION with a safe default, native endpoint routing, production ADC/OAuth bearer auth, onboarding/setup metadata, schema/runtime config acceptance, focused tests, and redacted real-run proof.

## Reproduction Assessment

Yes, source-level. With GOOGLE_CLOUD_PROJECT set but no GOOGLE_CLOUD_LOCATION, current main throws instead of using CLOUD_ML_REGION or a default, and a service_account ADC file cannot pass the authorized_user-only reader.

## Solution Assessment

No, current main is only a partial solution. The maintainable path is to land or supersede the linked Google Vertex PR work with one coordinated provider-owned implementation rather than close this issue now.
