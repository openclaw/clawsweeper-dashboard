## Summary

## What This Changes

The branch adds CLI startup policy so `openclaw acp` suppresses doctor stdout and hides the banner without `--json`, plus focused startup/config-guard tests.

## Best Possible Solution

Keep the command-policy approach, but scope protocol-stdout behavior to the bare ACP server command, preserve interactive `acp client` diagnostics, add regression coverage, and require redacted real Workbench/ACP proof before merge.

## Reproduction Assessment

Yes from source inspection: current main runs preaction config/bootstrap before the ACP server, and current startup policy only suppresses doctor stdout for JSON output mode. I did not run a live Workbench reproduction in this read-only review.

## Solution Assessment

No as submitted: the startup-policy layer is appropriate, but the ACP catalog override should be scoped so only the protocol server path gets protocol-stdout behavior.
