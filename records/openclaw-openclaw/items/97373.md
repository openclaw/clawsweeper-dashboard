## What This Changes

This PR adds a CLI command-path policy flag and ACP catalog override so ACP startup can suppress doctor stdout and the banner without requiring `--json`, with focused CLI regression tests.

## Best Possible Solution

Scope protocol-stdout startup behavior to the bare ACP server command, preserve normal human startup presentation for `openclaw acp client`, keep focused tests for both paths, and add redacted after-fix Workbench or terminal proof.

## Reproduction Assessment

## Solution Assessment

No as submitted: the startup-policy layer is the right owner boundary, but the PR should scope the policy to the bare server path or explicitly preserve the client subcommand behavior. The narrower fix avoids changing the human debug command while still keeping the protocol stream clean.
