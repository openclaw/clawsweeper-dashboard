## Summary

Current main already addresses the pairing lockout: CLI approval now requests admin scope when the pending request would mint or inherit admin access, the Gateway ownership guard exempts admin-scoped paired-device callers, and local loopback approval has a documented fallback.

## What This Changes

Allow upgraded users to approve new Control UI device-pairing requests from the CLI without being blocked by missing admin scope or device-ownership-mismatch.

## Best Possible Solution

Keep the current main behavior: exact-request approval escalates the CLI session to admin only when needed, admin callers bypass cross-device ownership checks, and local loopback fallback remains available for recovery.

## Reproduction Assessment

Yes. The report provides a high-confidence release reproduction with missing-scope and device-ownership-mismatch logs, and current main source/tests show the corrected approval path; I did not run a live Docker upgrade because this was a read-only review.

## Solution Assessment

Yes. The current implementation fixes the lockout without adding a force flag: the CLI requests the minimum sufficient approval scope, while the Gateway still blocks non-admin cross-device management.
