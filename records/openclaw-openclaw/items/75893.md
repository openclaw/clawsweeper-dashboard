## Summary

Keep open. The PR targets a real current Telegram bug, but the proposed routing change would pass default markdown text directly into an HTML-oriented chunk helper, regressing normal markdown formatting and raw HTML escaping; it also lacks the required changelog entry.

## What This Changes

The PR changes `extensions/telegram/src/send.ts` so default markdown-mode text would use the existing chunked send path instead of the single-message fallback branch.

## Best Possible Solution

Keep the Telegram fix narrow: chunk default markdown sends with rendered Telegram HTML plus the original plain fallback, preserve existing HTML-mode behavior, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. On current main, a default `sendMessageTelegram` call with text longer than 4096 characters follows the markdown branch that builds a single text chunk, and issue #75868 includes shipped-version logs showing Telegram rejects that payload as too long.

## Solution Assessment

No. The direction is right, but the submitted implementation should not feed raw markdown into `sendChunkedText`; a safer fix should use markdown-aware chunking or pass rendered HTML and plain fallback into the planner.
