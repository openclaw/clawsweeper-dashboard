## Summary

Keep open. The reported Telegram markdown chunking bug is valid, but this PR's current one-line routing change would regress existing markdown rendering by passing raw markdown into the HTML send path.

## What This Changes

The PR removes the `textMode === "html"` gate so default Telegram markdown text sends call the existing chunked text helper.

## Best Possible Solution

Add a markdown-aware chunk path that chunks rendered Telegram HTML while preserving the raw/plain-text fallback and last-chunk buttons, then cover the long markdown case and document the user-facing fix in the changelog.

## Reproduction Assessment

Yes. A mocked `sendMessageTelegram` unit case with default `textMode`, markdown such as `_oops_`, and a long `>4096` payload can reproduce both the current single-send bug and the PR's markdown-rendering regression without live Telegram access.

## Solution Assessment

No. Removing the gate is not the best fix because `sendChunkedText` currently assumes its input is already Telegram HTML; the safer fix is to separate rendered HTML chunk input from plain-text fallback input.
