## Summary

Close as implemented on main: current main and v2026.5.2 already split default Telegram markdown sends through the chunk planner while preserving the original plain-text fallback, and this PR branch should not merge because it regresses fallback semantics.

## What This Changes

The PR routes Telegram direct text sends through chunked outbound delivery for markdown-mode payloads over Telegram's 4096-character limit and adds a changelog entry.

## Best Possible Solution

Close this PR and keep the v2026.5.2/current-main Telegram chunking implementation that chunks rendered HTML from the original input while retaining the original plain fallback.

## Reproduction Assessment

No for current main: the old long default-markdown failure no longer reproduces from source inspection because direct sends route through sendChunkedText, and current tests cover the long markdown path. The PR branch fallback regression is source-reproducible from the diff.

## Solution Assessment

No for this PR branch. The current-main implementation is the safer solution because it sends the original text into the planner, rendering markdown for HTML chunks while preserving the original fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve the original markdown fallback:** `extensions/telegram/src/send.ts:967`
  - body: This branch passes rendered HTML into sendChunkedText, and the chunk planner derives the plain fallback from that same input when opts.plainText is absent.
