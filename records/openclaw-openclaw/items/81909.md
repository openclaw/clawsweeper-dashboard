## What This Changes

The PR changes edit recovery to read `file_path`/`filePath`/`filepath`/`file` aliases and adds a regression test for post-write recovery with `file_path`.

## Best Possible Solution

Make edit recovery and the production edit-tool parameter contract agree, with regression coverage through `createHostWorkspaceEditTool` or the sandboxed edit tool rather than only the bare recovery wrapper.

## Reproduction Assessment

No, not for the live Feishu report. Source inspection shows the narrow recovery failure mode with `file_path`, but current production edit tools reject aliases before recovery runs, so the PR's new test is not a high-confidence production reproduction.

## Solution Assessment

No. The patch changes recovery parsing only; the safer fix is to exercise and repair the validated production edit path, or deliberately update the alias contract and guards together.
