## Summary

Keep open: current main and v2026.5.18 only cover parts of the request, including Codex usage windows, WHAM cooldown handling, and manual re-auth guidance, but they still lack a stable quota/auth diagnostic state contract and any bounded brokered reauth execution surface.

## What This Changes

Request native Codex quota/auth diagnosis, 5h/weekly quota display, and optional bounded host-side reauth execution when OAuth credentials truly require reauthentication.

## Best Possible Solution

Define an additive provider auth/usage diagnostic contract, surface it through Gateway and Control UI, preserve automatic OAuth refresh and quota cooldown behavior, and allow host-side reauth only through a narrow security-reviewed adapter capability.

## Solution Assessment

No, not as an automated one-off fix. The direction is coherent, but the maintainable solution needs a maintainer-designed Gateway/provider/UI contract plus an explicit security policy for any host reauth adapter.
