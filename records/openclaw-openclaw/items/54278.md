## Summary

Keep open: current main has partial Codex quota visibility and single-provider cooldown recovery, but it still lacks the requested typed `quota_wait`/`reauth_required` diagnostic contract and brokered host-side reauth execution surface.

## What This Changes

The issue requests native Codex quota/auth state diagnosis, local-time quota display, and optional brokered host-side reauthentication when credentials actually need reauth.

## Best Possible Solution

Define one provider/Gateway/UI auth-quota contract for Codex that separates quota wait from credential failure, then add any host-side reauth adapter as a narrow, explicit, auditable extension point.

## Solution Assessment

Yes directionally, but not as an automation-ready patch: the proposed split between native diagnosis and bounded external execution is the right shape, while the permanent adapter boundary needs maintainer product and security decisions.
