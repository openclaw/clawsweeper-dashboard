## Summary

Keep open. Current main has partial Codex OAuth and usage support, but it still lacks the requested stable quota/auth diagnostic contract and bounded brokered reauth action, so there is no conservative close reason.

## What This Changes

Request native Codex quota/auth diagnosis states, local-time rate-limit display, and an optional bounded reauth adapter surfaced through Control UI.

## Best Possible Solution

Add an additive provider auth/usage diagnostic contract, expose it through gateway and Control UI, keep normal OAuth refresh automatic, and design a narrow audited broker API gated by a trusted reauth_required diagnosis.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path for the missing pieces across auth-health, models.authStatus, Codex WHAM usage handling, and the Codex app-server command surface.

## Solution Assessment

No automated fix is the best path yet. The narrow maintainable direction is a maintainer-designed additive provider/gateway/UI diagnostic contract plus an explicit security-reviewed broker policy.
