## Summary

Keep open: current main now has partial Codex quota visibility, WHAM cooldown handling, and manual reauth guidance, but it still lacks the requested quota_wait/reauth_required diagnostic contract and any brokered host-side reauth execution surface.

## What This Changes

The issue requests native Codex quota/auth diagnosis, local-time Codex quota display, and optional brokered reauthentication when credentials truly need reauth.

## Best Possible Solution

Define an additive provider auth/usage diagnostic contract, surface it through Gateway and Control UI, preserve existing automatic OAuth refresh and quota cooldown behavior, and allow host-side reauth only through a narrow security-reviewed adapter capability.

## Reproduction Assessment

Not applicable as a feature request; source inspection is the verification path. Current main shows partial Codex usage windows, WHAM cooldowns, and manual reauth guidance, but no stable quota_wait/reauth_required diagnostic state or brokered reauth execution surface.

## Solution Assessment

No, not as an automated one-off fix. The direction is coherent, but the maintainable solution needs a maintainer-designed Gateway/provider/UI contract plus an explicit security policy for any host reauth adapter.
