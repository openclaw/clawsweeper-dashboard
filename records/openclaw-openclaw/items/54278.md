## Summary

Keep open. Current main has partial Codex OAuth and quota support, but it still does not expose the requested native diagnostic state contract or a bounded brokered reauth action; the host-side reauth portion needs explicit product/security design before implementation.

## What This Changes

The issue requests native Codex quota/auth diagnosis, Codex rate-limit display, and bounded brokered reauth execution for true OAuth failures.

## Best Possible Solution

Add an additive provider auth/usage diagnostic contract, keep existing automatic OAuth refresh and WHAM parsing, surface local-time Codex rolling and weekly renewal details in Control UI, and design a narrow audited broker API that can run only after a trusted reauth_required diagnosis.
