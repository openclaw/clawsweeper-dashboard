## Summary

Keep open. Current main has partial Codex OAuth and quota support, but it still lacks the requested stable diagnostic state contract and bounded brokered reauth action, so there is no policy-valid close reason.

## What This Changes

The issue requests native Codex quota/auth diagnostics, local-time Codex quota display, and an optional bounded host-side reauth adapter for true OAuth failures.

## Best Possible Solution

Add an additive provider auth/usage diagnostic contract, expose it through gateway and Control UI surfaces, keep existing automatic OAuth refresh and WHAM usage parsing, format Codex quota windows in local time, and design a narrow audited broker API that can run only after a trusted reauth_required diagnosis.

## Reproduction Assessment

Not applicable as a feature request, but current-main source inspection gives a high-confidence verification path for the missing pieces. Existing tests also pin the generic 401 and 429 Codex usage behavior that would need to change.

## Solution Assessment

No automated fix is the best path yet. The current partial implementation should be extended only after maintainer product/security design settles the diagnostic contract and broker trust boundary.
