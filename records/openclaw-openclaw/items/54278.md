## Summary

Keep open. Current main has partial Codex usage display, WHAM-aware cooldowns, reset-time channel messages, and generic reauth guidance, but it still lacks the requested stable quota/auth diagnostic states and any bounded brokered host-side reauth execution path.

## What This Changes

Request native Codex quota/auth diagnostic states, Control UI quota display, and optional brokered reauth execution for true OAuth failures.

## Best Possible Solution

Add an additive provider auth/usage diagnostic contract exposed through Gateway and Control UI, preserve automatic OAuth refresh for normal expiry, and gate any host reauth broker behind a narrow security-reviewed adapter policy.
