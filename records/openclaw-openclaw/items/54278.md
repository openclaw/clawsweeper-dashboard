## Summary

Keep open: current main and the latest release have useful Codex quota display and usage-limit recovery pieces, but they still do not provide the requested typed quota/auth diagnosis contract or a brokered host-side reauth execution surface.

## What This Changes

The issue requests native Codex quota/auth state diagnosis, local-time quota display, and a bounded host-side reauth adapter path.

## Best Possible Solution

Define one provider/Gateway/UI auth-quota contract for Codex that separates quota wait from credential failure, then add any host-side reauth adapter as a narrow, explicit, auditable extension point.

Not applicable as a bug repro; this is a feature request. Source inspection gives high confidence that current main lacks the requested typed `quota_wait` / `reauth_required` contract and brokered reauth surface.

## Solution Assessment

Yes directionally, but not as an automation-ready change: native diagnosis plus bounded external execution is the right split, while the permanent adapter boundary needs maintainer product and security decisions.
