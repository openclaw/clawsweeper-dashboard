## Summary

Keep open. Current main has partial Codex OAuth, usage-window display, and reauth guidance, but it still lacks the requested stable quota/auth diagnostic contract and bounded brokered reauth execution, so there is no conservative close reason.

## What This Changes

The issue requests native Codex quota/auth diagnosis, native 5-hour and weekly Codex rate-limit display, and optional bounded brokered reauth execution for true OAuth failures.

## Best Possible Solution

Add an additive provider auth/usage diagnostic contract exposed through Gateway and Control UI, preserve automatic OAuth refresh for normal expiry, and gate any host reauth broker behind a narrow security-reviewed adapter policy.
