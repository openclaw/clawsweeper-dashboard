## Summary

Keep open. Current main still converts an exhausted image-description budget into a 1ms timeout, and the only related fix PRs I found are closed without merging, so the central reported bug remains a narrow repair candidate.

## What This Changes

The issue asks for image-understanding timeouts to stop turning exhausted per-call deadlines into repeated 1ms failures and instead fail clearly or reset the retry budget.

## Best Possible Solution

Keep the total image-description deadline, but make an exhausted budget return a clear terminal timeout before starting provider work, with focused regression coverage for runtime setup and internal retry exhaustion.

## Reproduction Assessment

Yes for the central 1ms collapse at source level: when elapsed time exceeds `timeoutMs`, the helper deterministically returns `1` and the timeout wrapper reports `1ms`. The full production multi-hour retry loop was not live-reproduced.

## Solution Assessment

Yes, a narrow media-understanding timeout helper fix is the best path. Avoid broad agent retry-policy changes; handle exhausted deadlines before provider side effects and add regression tests around that path.
