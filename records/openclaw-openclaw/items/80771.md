## Summary

Keep open. Current main and the latest release still clamp an exhausted image-description deadline to a 1ms timeout, and the two focused fix PRs linked from the issue were closed without merging.

## What This Changes

Fix image-understanding timeout budgeting so an exhausted per-call deadline does not collapse into repeated 1ms retries.

## Best Possible Solution

Keep the configured total image-description deadline, but return a clear terminal exhausted-deadline error before provider work starts and cover runtime setup plus retry exhaustion with focused tests.

Yes for the central source-level path: current main deterministically turns an exhausted timeout budget into `1`, and the wrapper reports that as `1ms`. The full production retry loop was not live-reproduced.

## Solution Assessment

Yes, a narrow media-understanding timeout helper fix is the best path. The closed work in https://github.com/openclaw/openclaw/pull/80807 is a useful model, but the live issue still needs an open or replacement fix.
