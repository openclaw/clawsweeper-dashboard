## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80771. Current `src/media-understanding/image.ts` clamps exhausted image-description timeout budgets to `1ms` via `Math.max(1, timeoutMs - elapsed)`, then schedules a near-immediate timeout and reports `image description timed out after 1ms`. Keep the existing configured timeout contract, but make exhausted budgets fail fast with a clear terminal timeout/deadline error before scheduling a 1ms timer or starting provider completion side effects. Add focused regression coverage in `src/media-understanding/image.test.ts` for runtime setup or an internal retry consuming the full budget, including an assertion that `complete()`/provider work is not invoked after the budget is already exhausted. Do not change timeout config schema, model fallback policy, or broad agent retry behavior unless a failing test proves it is necessary. Add a user-facing changelog entry for the bug fix.
