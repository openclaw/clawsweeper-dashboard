## Summary

Keep open. Current `main` and the latest release still clamp an exhausted image-description timeout budget to `1ms`, and the only focused fix PRs found for this report are closed unmerged, so the central timeout bug remains actionable.

## What This Changes

The issue reports that image-understanding retries can reuse an exhausted timeout budget and collapse to `image description timed out after 1ms`.

## Best Possible Solution

Keep the total image-description deadline, but return a clear terminal exhausted-deadline error before provider work or retry scheduling can turn the remaining budget into a 1ms timer.

## Reproduction Assessment

Yes for the central 1ms collapse at source level: when elapsed time exceeds `timeoutMs`, current `main` deterministically returns `1` and the timeout wrapper reports that value. The full production six-hour retry loop was not live-reproduced.

## Solution Assessment

Yes. A narrow media-understanding timeout helper fix is the best path; avoid broad agent retry-policy changes and handle exhausted deadlines before provider side effects or retry timers start.
