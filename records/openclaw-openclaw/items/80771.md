## Summary

Keep open. Current main and the latest shipped release still clamp an exhausted image-description deadline to 1ms, and the two related fix PRs were closed unmerged, so the narrow timeout-loop bug remains actionable.

## What This Changes

The issue asks OpenClaw to stop exhausted image-description timeout budgets from collapsing into repeated 1ms retry failures.

## Best Possible Solution

Make the exhausted image-description budget a terminal deadline result before provider work is started, preserve normal positive-budget timeouts, and add focused regression coverage for setup or retry paths that consume the full budget.

## Reproduction Assessment

Yes, at source level. Current main deterministically maps a negative remaining image-description budget to 1ms and then uses that value in the provider timeout path, though no live Slack/Anthropic loop was run.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `src/media-understanding/image.ts` with regression tests, not in broader agent retry policy or a new config option.
