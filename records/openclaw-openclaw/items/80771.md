## Summary

Keep open. Current main and the latest shipped release still clamp an exhausted image-description budget to 1ms, and the two focused fix attempts linked from the issue are both closed without merging, so the reported source-level bug remains actionable.

## What This Changes

The issue asks OpenClaw to stop image-understanding retries from converting an exhausted deadline budget into repeated 1ms timeouts.

## Best Possible Solution

Keep the configured total image-description deadline, but make an exhausted budget fail with a clear terminal timeout before provider work is created, with regression coverage for runtime setup and internal retry exhaustion.

## Reproduction Assessment

Yes for the source-level bug: current main deterministically returns `1` when elapsed time exceeds `timeoutMs`, and the timeout wrapper reports that value as a 1ms image-description timeout. I did not live-reproduce the full multi-hour production retry loop.

## Solution Assessment

Yes. A narrow fix in the media-understanding timeout helper is the best path; avoid broad agent retry-policy or config-default changes unless focused tests prove they are also required.
