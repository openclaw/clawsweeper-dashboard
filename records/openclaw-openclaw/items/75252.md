## Summary

Keep open: current main still has the Mattermost draft-preview overwrite behavior, and the issue is paired with open closing PR #75256. The right cleanup path is to review or revise that PR, while coordinating the overlapping #73252 streaming-mode config work, rather than closing this issue before a fix lands.

## What This Changes

Add Mattermost preview-streaming mode support and boundary splitting so draft preview phase transitions can preserve previously visible content instead of editing one post for the whole turn.

## Best Possible Solution

Land or revise the Mattermost-local fix in #75256, preserving the existing edit-in-place default while adding validated streaming-mode config and guarded boundary splitting for block-mode previews. Coordinate with #73252 so schema, account resolution, docs, and focused Mattermost tests describe one consistent streaming surface.

Yes. The issue gives a Mattermost DM/tool-call repro path, and current source provides a high-confidence code-level reproduction: phase hooks keep updating the same draft stream while the draft stream edits the existing streamPostId until forceNewMessage or cleanup clears it.

## Solution Assessment
