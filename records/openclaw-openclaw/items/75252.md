## Summary

Keep open: current main still routes Mattermost draft-preview phase transitions through one editable post, and the open closing PR #75256 is the implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests a Mattermost streaming preview mode that preserves prior phase content by splitting draft previews at reasoning, tool, and assistant-message boundaries instead of repeatedly editing one post.

## Best Possible Solution

Review and land or revise the Mattermost-local fix in #75256 while coordinating the overlapping #73252 config work: preserve the current edit-in-place default, add validated streaming mode config, split block-mode previews at content boundaries, and cover schema, account resolution, draft-stream, and monitor behavior with focused Mattermost tests plus a changelog entry.

## Reproduction Assessment

Yes. The issue gives a Mattermost DM/tool-call repro path, and current main provides a code-level reproduction because each phase transition updates the same draft stream post until forceNewMessage or cleanup clears the post id.

## Solution Assessment

Yes. A Mattermost-local preview mode plus boundary controller is the narrow maintainable solution; changing shared core reply dispatch would be broader than necessary because the overwrite behavior is owned by the Mattermost adapter.
