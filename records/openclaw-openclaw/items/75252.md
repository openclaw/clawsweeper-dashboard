## What This Changes

The issue requests a Mattermost-local preview streaming mode that can split draft preview posts at reasoning, assistant, and tool boundaries so previously visible content is not overwritten.

## Best Possible Solution

Review and land or revise the Mattermost-local fix in #75256 while coordinating the overlapping #73252 config work: preserve the current edit-in-place default, add validated streaming mode config, split block-mode previews at content boundaries, and cover schema, account resolution, draft-stream, and monitor behavior with focused Mattermost tests.

Yes. The issue gives a Mattermost DM/tool-call repro path, and current source provides a code-level reproduction because every phase transition updates the same draft stream post until forceNewMessage or cleanup clears the post id.

## Solution Assessment
