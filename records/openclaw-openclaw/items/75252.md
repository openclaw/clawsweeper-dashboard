## Summary

Keep open. Current main still edits one Mattermost draft-preview post across phase transitions, and the directly linked fix PR is open and unmerged while another open PR overlaps the same streaming config surface.

## What This Changes

The issue requests Mattermost draft preview streaming to preserve previously visible phase content instead of overwriting one edited post throughout a turn.

## Best Possible Solution

Review and land #75256 or an equivalent Mattermost-owned `streaming.mode` implementation, coordinated with #73252, with schema/types/docs and regression coverage aligned to the final mode semantics.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the current-main path: one Mattermost draft stream retains `streamPostId`, lifecycle hooks keep calling `update(...)`, and the only local splitter `forceNewMessage()` is unused by the monitor.

## Solution Assessment

Yes. A Mattermost-local preview-mode resolver plus guarded boundary-splitting controller is the narrow maintainable fix; shared dispatch/core behavior does not need to change for this adapter-owned preview behavior.
