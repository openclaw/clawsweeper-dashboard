## Summary

Keep open. Current main still routes Mattermost draft-preview phase changes through one editable post, and open PR #75256 is the closing implementation path rather than a reason to close this issue before merge; overlapping PR #73252 also needs coordination on the Mattermost streaming config surface.

## What This Changes

The issue requests a Mattermost preview streaming mode that preserves prior thinking, partial reply, and tool-status content by splitting draft preview posts at phase transitions instead of repeatedly overwriting one edited post.

## Best Possible Solution

Review and land or revise PR #75256 in coordination with PR #73252: keep the current edit-in-place default, accept a validated Mattermost streaming mode, split block-mode previews at meaningful assistant/reasoning/tool boundaries with forceNewMessage, and keep schema, tests, docs, and changelog aligned.

## Reproduction Assessment

Yes. The Mattermost DM/tool-call repro maps directly to current source: phase hooks call draftStream.update while streamPostId remains set, so later preview states edit the same post rather than creating a new one.

## Solution Assessment

Yes. A Mattermost-local streaming mode plus boundary controller is the narrow maintainable fix because the overwrite behavior is owned by the Mattermost adapter; changing shared dispatch would be broader than needed.
