## Summary

Keep open. Current main still edits one Mattermost draft-preview post across phase transitions, and open PR #75256 explicitly closes this issue while PR #73252 overlaps the same streaming config surface.

## What This Changes

Mattermost should offer a preview streaming mode that preserves prior reasoning, tool, and partial-reply phases instead of rewriting the same draft-preview post through a tool-call turn.

## Best Possible Solution

Review and land PR #75256, or an equivalent Mattermost-owned fix, in coordination with PR #73252 so schema, runtime behavior, tests, docs, and changelog all agree.

## Reproduction Assessment

Yes. The source path is high-confidence without a live Mattermost server: after the first preview post is created, later thinking, partial, and tool updates keep `streamPostId` set and therefore edit the same `/posts/{id}`.

## Solution Assessment

Yes. The narrow maintainable fix is Mattermost-local `streaming.mode` resolution plus guarded `forceNewMessage()` calls at real preview boundaries; shared core dispatch does not need to own this adapter-specific post-edit policy.
