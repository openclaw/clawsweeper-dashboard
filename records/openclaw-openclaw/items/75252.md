## Summary

Keep open. Current main still routes Mattermost draft-preview phase changes through one editable post, and the open closing PR is the implementation path rather than a reason to close this issue before merge.

## What This Changes

Request to add a Mattermost preview streaming mode that can split draft preview posts at reasoning, tool, and reply boundaries instead of rewriting one post.

## Best Possible Solution

Review and land or revise PR #75256 while coordinating with PR #73252: keep the current edit-in-place default, add validated Mattermost streaming mode config, split block-mode previews at meaningful boundaries with forceNewMessage, and cover schema, account resolution, draft-stream, monitor behavior, docs, and changelog together.

## Reproduction Assessment

Yes. The Mattermost DM/tool-call repro maps directly to current source: phase hooks call draftStream.update while streamPostId stays set, so later updates edit the same post rather than creating a new one.

## Solution Assessment

Yes. A Mattermost-local streaming mode plus boundary controller is the narrow maintainable fix; changing shared core dispatch would be broader than necessary because the overwrite behavior is owned by the Mattermost adapter.
