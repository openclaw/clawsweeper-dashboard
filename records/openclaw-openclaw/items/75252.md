## Summary

Keep open. Current main still rewrites a single Mattermost draft-preview post across thinking, tool, partial, and final transitions, and open PR #75256 explicitly closes this issue while PR #73252 overlaps the streaming-mode config surface.

## What This Changes

The issue requests a Mattermost streaming mode that preserves draft-preview content across reasoning, tool, and partial-reply transitions instead of repeatedly editing one post.

## Best Possible Solution

Land one coordinated Mattermost-owned streaming-mode implementation that aligns schema, resolver, docs, and boundary splitting while preserving the current default behavior.

## Reproduction Assessment

Yes. The source path is high-confidence without a live Mattermost server: one draft stream retains `streamPostId`, phase hooks call `update(...)`, and the monitor does not use the available `forceNewMessage()` boundary API.

## Solution Assessment

Yes. A Mattermost-local config resolver plus boundary-splitting controller is the narrow maintainable direction; the shared core dispatcher does not need to change for this adapter-owned overwrite behavior.
