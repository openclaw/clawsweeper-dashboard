## Summary

Keep open. Current main still edits a single Mattermost draft-preview post across phase transitions, and the open closing PR should be reviewed or merged before this issue is closed.

## What This Changes

The issue requests a Mattermost preview streaming mode that can split draft-preview transitions into new posts so earlier thinking, tool status, and partial-reply content remains visible.

## Best Possible Solution

Ship a Mattermost-owned `streaming.mode` implementation that preserves `partial` as the default, validates config, splits block-mode previews at real boundaries with `forceNewMessage()`, and updates tests, docs, and changelog in coordination with PR #73252.

## Reproduction Assessment

Yes. The source path is high-confidence without a live Mattermost server: preview transitions keep `streamPostId` set and never call `forceNewMessage()`, so later thinking, tool, and partial updates edit `/posts/{id}` instead of creating new posts.

## Solution Assessment

Yes. The narrow maintainable fix is Mattermost-local config resolution plus boundary splitting; shared core dispatch does not need to change because the overwrite behavior is created by the Mattermost draft stream and monitor hooks.
