## Summary

Keep open. Current main still routes Mattermost draft-preview phase transitions through a single editable post, and the open closing PR is the right implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests a Mattermost preview streaming mode that preserves prior transition content by splitting draft preview posts at reasoning, tool, and partial-reply boundaries.

## Best Possible Solution

Review and land or revise the Mattermost-local fix in #75256 while coordinating the overlapping #73252 config work: preserve the current default, add validated preview streaming mode config, split block-mode previews at content boundaries, and cover schema, account resolution, draft-stream, and monitor behavior with focused Mattermost tests.

## Reproduction Assessment

Yes. The issue gives a Mattermost DM/tool-call repro path, and current main provides a code-level reproduction because every phase transition updates the same draft stream post until forceNewMessage or cleanup clears the post id.

## Solution Assessment

Yes. A Mattermost-local streaming mode plus boundary controller is the narrow maintainable fix; changing shared core reply dispatch would be broader than necessary because the overwrite behavior is owned by the Mattermost draft-preview adapter.
