## Summary

Keep open. Current main still rewrites one Mattermost draft-preview post across reasoning, partial-reply, and tool-status transitions, and the directly linked closing PR is open and unmerged while another PR overlaps the same streaming-mode surface.

## What This Changes

The issue requests a Mattermost preview streaming mode that preserves prior transition content by splitting draft previews at boundaries instead of editing one post for the whole turn.

## Best Possible Solution

Review and land a Mattermost-owned streaming-mode implementation that keeps current default behavior, validates schema/types/docs, and uses guarded `forceNewMessage()` boundary splits when block mode is enabled.

## Reproduction Assessment

Yes. Source inspection on current main is enough: `createMattermostDraftStream` keeps one `streamPostId`, and the monitor's partial/reasoning/tool hooks call `draftStream.update(...)` without invoking `forceNewMessage()`.

## Solution Assessment

Yes. The narrow fix belongs in the Mattermost plugin: resolve the documented streaming mode and split the local draft stream on lifecycle boundaries; shared dispatch/core paths do not need to change.
