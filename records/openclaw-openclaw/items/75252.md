## Summary

Keep open. Current main still rewrites a single Mattermost draft-preview post across phase transitions, and the directly linked fix PR is open and unmerged with another open PR overlapping the same streaming-mode config surface.

## What This Changes

Mattermost draft previews should preserve previously visible phase content by supporting a block-style streaming mode that splits preview output at boundaries.

## Best Possible Solution

Land a Mattermost-owned `streaming.mode` implementation that keeps the current default, validates schema/types/docs, and uses guarded boundary splitting for real preview transitions in coordination with the overlapping PR.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the path: one draft stream retains `streamPostId`, lifecycle hooks keep calling `update(...)`, and the only local splitter `forceNewMessage()` is unused by the Mattermost monitor.

## Solution Assessment

Yes. A Mattermost-local streaming-mode resolver plus guarded boundary-splitting controller is the narrow maintainable fix; the shared dispatch/core path does not need to change for this adapter-owned preview behavior.
