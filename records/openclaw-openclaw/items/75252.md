## Summary

Keep open. Current main still routes Mattermost draft-preview phase changes through one editable post, and open PR #75256 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a Mattermost streaming draft-preview mode that preserves already-shown phase content by creating new posts at meaningful turn boundaries instead of repeatedly editing one post.

## Best Possible Solution

Review and land or revise PR #75256 while coordinating with PR #73252: keep the existing edit-in-place default, add a validated Mattermost streaming mode, split block-mode previews at meaningful boundaries with forceNewMessage, and cover schema, account resolution, monitor behavior, docs, changelog, and targeted Mattermost tests together.

## Reproduction Assessment

Yes. The reported Mattermost DM/tool-call reproduction maps directly to current source: phase hooks keep calling draftStream.update while streamPostId remains set, so subsequent preview states update the same Mattermost post instead of creating a new one.

## Solution Assessment

Yes. A Mattermost-local streaming mode plus boundary controller is the narrow maintainable fix; changing shared core dispatch would be broader than necessary because the overwrite behavior is owned by the Mattermost adapter.
