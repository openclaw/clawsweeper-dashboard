## Summary

Keep open. Current main still routes Mattermost draft-preview phase changes through one editable preview post, and open closing PR #75256 is the implementation path rather than a reason to close this issue before merge; overlapping PR #73252 also needs coordination around the Mattermost streaming config surface.

## What This Changes

The issue requests a Mattermost streaming mode that preserves prior draft-preview phase content by splitting Mattermost preview posts at reasoning, partial-reply, and tool-call boundaries.

## Best Possible Solution

Review and land or revise the Mattermost-local fix in #75256 while coordinating the overlapping #73252 config work: preserve the current edit-in-place default, add validated streaming mode config, split block-mode previews only after streamed content exists, and cover schema, account resolution, draft-stream, and monitor behavior with focused Mattermost tests.

## Reproduction Assessment

Yes. The issue gives a Mattermost DM/tool-call repro path, and current source provides a code-level reproduction because every phase transition updates the same draft stream post until forceNewMessage or cleanup clears the post id.

## Solution Assessment

Yes. A Mattermost-local streaming mode and boundary controller is the narrow maintainable solution; changing shared core reply dispatch would be broader than necessary because the overwrite behavior is owned by the Mattermost draft-preview adapter.
