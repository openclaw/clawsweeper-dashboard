## Summary

Keep open. Current main still has the Mattermost draft-preview overwrite path, and PR #75256 explicitly closes this issue while PR #73252 overlaps the same streaming config surface, so this should close only after the coordinated fix lands.

## What This Changes

Mattermost should offer a preview streaming mode that preserves prior draft-preview phases instead of overwriting one post through thinking, tool, partial, and final transitions.

## Best Possible Solution

Land or supersede the open Mattermost-owned implementation so schema/types/docs and monitor boundary splitting agree while preserving the current default behavior.

## Reproduction Assessment

Yes. A live server is not required for high-confidence source reproduction: the monitor sends all preview phases through one draft stream, and the stream edits the existing `streamPostId` until `forceNewMessage()` is called.

## Solution Assessment

Yes. A Mattermost-local streaming mode resolver plus boundary controller is the narrow maintainable fix; the overwrite behavior is created inside the Mattermost adapter, not in shared core dispatch.
