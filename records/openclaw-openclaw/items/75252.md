## Summary

Keep open. Current main still has the Mattermost draft-preview overwrite path, and PR #75256 explicitly closes this issue but has not merged yet.

## What This Changes

The issue requests a Mattermost preview-streaming mode that preserves prior thinking, tool, and partial-reply preview content instead of rewriting one post across phase transitions.

## Best Possible Solution

Land a Mattermost-owned streaming-mode fix that accepts the documented config, keeps the default backward compatible, and calls `forceNewMessage()` at real preview boundaries in block mode.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: once the first Mattermost preview post exists, later transition updates keep `streamPostId` set and therefore edit the same post instead of creating a new one.

## Solution Assessment

Yes. The narrow maintainable solution is Mattermost-local config resolution plus boundary splitting; shared core dispatch does not need to change because the overwrite is created by the Mattermost draft stream and monitor hooks.
