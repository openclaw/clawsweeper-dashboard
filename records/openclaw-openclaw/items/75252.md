## Summary

Keep open. Current main still rewrites one Mattermost draft-preview post across phase transitions, and the open closing PR should be reviewed or coordinated with the overlapping Mattermost streaming-mode PR before this issue closes.

## What This Changes

Request Mattermost preview streaming mode support that preserves prior preview content by splitting draft posts at phase boundaries instead of repeatedly editing one post.

## Best Possible Solution

Land a Mattermost-owned streaming-mode implementation that preserves the current default, validates schema/types/docs, and calls `forceNewMessage()` at real preview boundaries.

## Reproduction Assessment

Yes. The source path is high-confidence without a live Mattermost server: one draft stream retains `streamPostId`, phase hooks call `update(...)`, and the only local split API is unused by the monitor.

## Solution Assessment

Yes. A Mattermost-local config resolver plus boundary-splitting controller is the narrow maintainable fix; shared core dispatch does not need to change for this adapter-created overwrite behavior.
