## Summary

Keep open. Current main still rewrites one Mattermost draft-preview post across thinking, tool, and partial-reply transitions, and the explicitly linked fix PR remains open rather than merged.

## What This Changes

The issue asks Mattermost draft previews to preserve visible phase-transition content instead of replacing one edited post for the whole turn.

## Best Possible Solution

Land a coordinated Mattermost-local `streaming.mode` implementation that preserves the current default, validates schema/types/docs, and uses guarded boundary splitting for preview transitions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Mattermost turn with tool calls reuses one `streamPostId`, lifecycle hooks keep calling `update(...)`, and no Mattermost hook calls `forceNewMessage()`.

## Solution Assessment

Yes. A Mattermost-owned streaming-mode resolver plus guarded boundary-splitting controller is the narrow maintainable fix; shared core dispatch does not need to change for this adapter-owned preview behavior.
