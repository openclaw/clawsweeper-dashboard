## Summary

Keep open. Current main still routes Mattermost draft-preview phase changes through one editable post, and the open closing PR is the implementation path rather than a reason to close this issue before merge; overlapping streaming-mode work also needs coordination.

## What This Changes

The issue asks Mattermost to add a configurable preview streaming mode that splits draft preview posts at phase boundaries so prior thinking, tool-status, and partial-reply content remains visible.

## Best Possible Solution

Review and land or revise #75256 while coordinating with #73252: preserve the current edit-in-place default, add validated Mattermost streaming-mode config, split block-mode previews at meaningful boundaries with forceNewMessage, and cover schema, account resolution, draft-stream, monitor behavior, docs, and changelog in one coherent Mattermost-local fix.

## Reproduction Assessment

Yes. The reported Mattermost DM/tool-call repro maps directly to current source: phase hooks call draftStream.update while streamPostId stays set, so later updates edit the same post instead of creating a new one.

## Solution Assessment

Yes. A Mattermost-local streaming mode plus boundary controller is the narrow maintainable fix; changing shared core dispatch would be broader than necessary because the overwrite behavior is owned by the Mattermost adapter.
