## Summary

Keep open. Current main still creates one Mattermost draft stream that edits the same post across reasoning, partial-reply, and tool transitions, while the direct closing PR is still open and overlaps another Mattermost streaming-mode PR.

## What This Changes

The issue requests Mattermost preview streaming to preserve prior phase content by honoring a documented streaming mode and splitting draft previews at lifecycle boundaries.

## Best Possible Solution

Land a Mattermost-owned `streaming.mode` implementation that preserves the current default, validates schema/types/docs, and uses guarded boundary splitting for real preview transitions in coordination with the overlapping PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: one draft stream retains `streamPostId`, lifecycle hooks continue to call `update(...)`, and Mattermost never invokes the available `forceNewMessage()` splitter.

## Solution Assessment

Yes. The narrow maintainable direction is Mattermost-local: resolve the documented preview stream mode and split at guarded lifecycle boundaries, without changing the shared dispatch/core pipeline.
