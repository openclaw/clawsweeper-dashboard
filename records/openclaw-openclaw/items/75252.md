## Summary

Keep open. Current main still rewrites one Mattermost draft-preview post across thinking, tool, partial, and final transitions, and the open closing PR should be reviewed or coordinated with the overlapping streaming-mode PR before this issue closes.

## What This Changes

The issue asks Mattermost draft previews to preserve phase-transition content by adding a preview mode and splitting edited draft posts at boundaries.

## Best Possible Solution

Land a Mattermost-owned `streaming.mode` implementation that preserves the current default, validates schema/types/docs, and uses `forceNewMessage()` at real preview boundaries in coordination with the overlapping streaming-mode PR.

## Reproduction Assessment

Yes. The source path is high-confidence without a live Mattermost server: one draft stream retains `streamPostId`, phase hooks call `update(...)`, and the only `forceNewMessage()` implementation is unused by the monitor.

## Solution Assessment

Yes. A Mattermost-local config resolver plus boundary-splitting controller is the narrow maintainable fix; shared core dispatch does not need to change because the overwrite behavior is created by the Mattermost draft stream and monitor hooks.
