## Summary

Keep open. Current main still rewrites one Mattermost draft-preview post across phase transitions, and open PR #75256 explicitly closes this issue, so the issue should stay open until that PR or an equivalent coordinated fix lands.

## What This Changes

Mattermost draft preview should preserve already visible phase content by adding a block-style streaming mode that opens a new post at assistant, reasoning, and tool boundaries.

## Best Possible Solution

Land or supersede PR #75256 with a Mattermost-owned `streaming.mode` implementation that coordinates with #73252, preserves the default edit-in-place behavior, and splits preview posts at real phase boundaries for block mode.

## Reproduction Assessment

Yes. The source path is high-confidence without a live Mattermost server: one draft stream retains `streamPostId`, phase hooks call `update(...)`, and the only local split API is unused by the monitor.

## Solution Assessment

Yes. A Mattermost-local config resolver plus a guarded boundary controller is the narrow maintainable fix; shared core dispatch does not need to change because the overwrite behavior is created in the Mattermost adapter.
