Tighten the generic managed `web_search` contract and normalize one common single-query malformed call shape before provider dispatch, without changing provider selection or adding multi-query search behavior.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/82097. The managed `web_search` tool currently documents `query` but does not mark it required, and `{ queries: ["..."] }` is forwarded unchanged to providers such as Ollama that call `readStringParam(args, "query", { required: true })`. Implement a narrow repair in the managed web-search path: make the schema clearly require `query` and, if adding compatibility normalization, only map a single non-empty common alias such as `queries: [string]` to `query` without adding multi-query behavior. Do not change provider selection, fallback ordering, Ollama endpoint routing, or security/SSRF behavior. Add regression coverage for the managed schema and for the malformed single-query argument shape reaching provider execution. Add a `CHANGELOG.md` fix entry if the repair changes user-visible behavior.
