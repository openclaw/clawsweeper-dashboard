## Best Possible Solution

Land the focused generator, docs, and test change if maintainers accept the systemd availability behavior, while leaving broader gateway heap/OOM mitigation to https://github.com/openclaw/openclaw/issues/89315.

## Reproduction Assessment

Yes, source-reproducible: current generated units omit `OOMPolicy=continue`, systemd's documented default is `stop`, and `continue` is documented to keep the unit running when a process in the unit is OOM-killed. I did not re-trigger a live host OOM.

## Solution Assessment

Yes for the narrow child-OOM restart problem: the best layer is the canonical generated systemd unit plus matching docs and a renderer regression test. The broader gateway heap-growth problem remains separate work.
