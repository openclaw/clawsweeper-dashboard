## Summary

Keep open for normal merge review: the previous Gateway runbook docs mismatch is fixed, the patch is narrow, and the generated-unit behavior matches the documented systemd contract. The broader heap-growth report remains open because this PR only covers the child-OOM restart failure mode.

## What This Changes

The PR adds `OOMPolicy=continue` to generated Linux systemd Gateway units, aligns both public copyable systemd examples, and extends the systemd unit regression test.

## Best Possible Solution

Merge the targeted systemd policy and docs alignment if maintainers accept the availability tradeoff, while keeping https://github.com/openclaw/openclaw/issues/89315 open for the broader heap-growth and memory-mitigation work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source and incident evidence: current main omits `OOMPolicy=continue` from generated/manual systemd units, and systemd documents that `continue` is the mode that keeps the unit running after a process in the unit is OOM-killed.

## Solution Assessment

Is this the best way to solve the issue? Yes for the child-OOM restart failure mode: the generated unit is the right owner boundary, both manual docs now match it, and the broader heap-growth problem stays in the linked issue.
