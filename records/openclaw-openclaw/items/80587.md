The issue asks OpenClaw docs to resolve conflicting guidance about whether `openclaw browser wait --load networkidle` is supported across browser profile types.

## Best Possible Solution

Land or close the linked docs PR so both browser docs state that `networkidle` works for managed and remote-CDP profiles but not existing-session profiles.
