## Summary

Keep open. Current source shows a real docs clarity gap: the Playwright-backed browser path accepts `networkidle`, existing-session profiles reject it, and the browser-control reference advertises the option without the profile caveat.

The issue asks to resolve contradictory browser documentation about whether `openclaw browser wait --load networkidle` is supported.

## Best Possible Solution

Keep the runtime behavior and update the browser docs to state that `--load networkidle` is supported on managed/CDP profiles but unavailable on existing-session profiles.
