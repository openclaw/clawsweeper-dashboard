## Summary

Keep open: current main and the latest release still lack a WhatsApp owner/human-takeover config surface or per-conversation auto-reply pause gate. The linked implementation was closed unmerged and the duplicate was closed in favor of this issue, so the remaining work needs maintainer product/API direction rather than cleanup closure.

## What This Changes

Request owner takeover for DM auto-reply so outgoing owner messages can pause per-thread WhatsApp, and possibly Telegram, auto-replies until cooldown or explicit resume.

## Best Possible Solution

Define a WhatsApp-owned human-takeover config contract first, then add per-account/per-conversation pause state, an auto-reply suppression gate, docs, and focused regression coverage before considering a shared cross-channel helper.

## Reproduction Assessment

Not applicable as a feature request. Source and release inspection give high confidence that the requested config, pause state, and dispatch gate are absent, but there is no existing documented behavior to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the public API and resume policy. A WhatsApp-owned implementation with a small shared helper only if needed is safer than directly reviving the closed broad cross-channel PR.
