## Summary

Keep open: current main and the latest shipped release still lack the requested owner/human-takeover config surface and per-thread auto-reply pause gate. The linked implementation was closed unmerged, and the related duplicate was closed in favor of this issue, so this remains a product/API decision rather than cleanup-ready stale work.

## What This Changes

Requested owner/human handoff behavior that pauses DM auto-reply after the account owner manually replies in the same conversation.

## Best Possible Solution

Define a stable WhatsApp-owned config contract first, then add per-account/per-conversation pause state, an auto-reply suppression gate, docs, and focused regression coverage before considering a shared cross-channel abstraction.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested config, pause state, and dispatch gate are absent on current main, but there is no existing documented behavior to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the public API and resume policy. A WhatsApp-owned implementation with a small shared helper if needed is safer than directly reviving the closed broad cross-channel PR.
