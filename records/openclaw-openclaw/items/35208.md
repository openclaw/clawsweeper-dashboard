## Summary

Keep open: current main and the latest release still lack the requested WhatsApp owner handoff config and per-DM auto-reply pause gate. The linked implementation PR is closed unmerged, and the only related duplicate report was closed in favor of this issue as the canonical tracker.

## What This Changes

The issue requests a configurable owner-takeover handoff for personal-number DM auto-reply so manual outgoing owner messages pause AI replies per thread.

## Best Possible Solution

Define a WhatsApp-owned handoff contract first, then add per-account/per-DM pause state, an auto-reply suppression gate, docs, focused regression tests, and live WhatsApp proof; use the closed PR as prior art rather than reviving it wholesale.

## Reproduction Assessment

## Solution Assessment
