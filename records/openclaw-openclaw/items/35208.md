## Summary

Keep open: current main still lacks owner/human takeover config, pause state, and an auto-reply suppression gate; the only implementation PR is closed unmerged and the duplicate report points back here as the canonical tracker.

## What This Changes

Requests an opt-in owner/human handoff mechanism that detects manual outgoing DM messages and pauses auto-replies per conversation until cooldown or explicit resume.

## Best Possible Solution

Define a stable channel-owned config contract, implement per-account/per-conversation pause state where owner messages are reliable, and document/test resume behavior before broadening it across channels.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source inspection on current main gives a high-confidence gap because the config, schema, and dispatch paths do not contain owner takeover state or pause/resume handling.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the public config and resume policy; a WhatsApp-owned implementation with a small shared cooldown helper looks safer than a core-wide special case.
