## Summary

Keep open: current main still lacks the requested owner/human-takeover config surface and per-thread auto-reply pause gate, while the linked implementation was closed unmerged and the duplicate report was closed in favor of this issue.

## What This Changes

The issue requests owner/human takeover for DM auto-reply so manual outgoing messages pause per-thread bot replies until resume policy allows replies again.

## Best Possible Solution

Define a stable WhatsApp-owned config contract first, then add per-account/per-conversation pause state, an auto-reply suppression gate, docs, and focused regression coverage before considering any shared cross-channel abstraction.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested config, pause state, and dispatch gate are absent on current main, but there is no existing documented behavior to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the public API and resume policy. A WhatsApp-owned implementation with a small shared helper if needed is safer than a broad core-wide special case or directly reviving the closed cross-channel PR.
