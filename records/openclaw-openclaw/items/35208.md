## Summary

Keep open: current main and v2026.5.19 still lack a public owner/human-takeover config, per-conversation pause state, or auto-reply suppression gate for manual WhatsApp owner replies. The linked implementation at https://github.com/openclaw/openclaw/pull/58919 is closed unmerged, and the related duplicate was already closed in favor of this issue, so the remaining work needs maintainer API/product direction rather than cleanup closure.

## What This Changes

Request a configurable owner handoff for DM auto-reply that detects manual outgoing owner messages and pauses auto-replies per conversation until cooldown or explicit resume.

## Best Possible Solution

Define a WhatsApp-owned handoff contract first, then add per-account/per-conversation pause state, an auto-reply suppression gate, docs, focused regression tests, and live WhatsApp proof before considering a shared cross-channel helper.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, shipped-release, and related-item inspection give high confidence that the requested config and pause gate are absent, but there is no existing documented behavior to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the public API and resume policy. A WhatsApp-owned implementation with a small shared helper only if later justified is safer than reviving the closed broad cross-channel PR wholesale.
