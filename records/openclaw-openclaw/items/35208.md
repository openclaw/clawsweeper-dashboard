## Summary

Keep open: current main still lacks owner/human takeover config, pause state, and an auto-reply suppression gate; the related implementation PR is closed unmerged and the duplicate report points back here as the canonical tracker.

## What This Changes

Request owner/human handoff for DM auto-reply by detecting owner-authored outgoing messages and pausing or resuming per-thread automatic replies.

## Best Possible Solution

Define a stable channel-owned config contract, implement per-account/per-conversation pause state where owner messages are reliable, and document/test resume behavior before broadening it across channels.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives a high-confidence gap: the config/schema/dispatch paths do not contain owner takeover state or pause/resume handling.

## Solution Assessment

Unclear until maintainers choose the public config and resume policy. A WhatsApp-owned implementation with a small shared helper only for common cooldown bookkeeping looks safer than a core-wide special case.
