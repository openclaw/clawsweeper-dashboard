## Summary

Keep open: current main still lacks the requested owner/human-takeover config surface, per-thread pause state, and WhatsApp auto-reply suppression gate. The prior implementation at https://github.com/openclaw/openclaw/pull/58919 is closed unmerged, and https://github.com/openclaw/openclaw/issues/44148 is already closed as a duplicate pointing back to this issue.

## What This Changes

Request an opt-in owner/human handoff for WhatsApp/Telegram DM auto-reply that detects owner-authored outgoing messages and pauses AI replies per thread.

## Best Possible Solution

Define a stable channel-owned config contract, implement per-account/per-conversation pause state where owner messages are reliable, and document/test resume behavior before broadening it across channels.

## Reproduction Assessment

Not applicable as a feature request; source inspection on current main gives high confidence that the requested config, pause state, and dispatch gate are absent. The existing behavior is not a broken documented contract yet.

## Solution Assessment

Unclear until maintainers choose the public config and resume policy. A WhatsApp-owned implementation with a small shared helper looks safer than a core-wide special case or a broad cross-channel branch copied from the closed prior PR.
