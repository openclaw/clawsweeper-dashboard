## Summary

Keep open: current main and latest release v2026.5.20 still lack a WhatsApp owner/human-takeover config and per-DM auto-reply pause gate. The related implementation at https://github.com/openclaw/openclaw/pull/58919 is closed unmerged, and https://github.com/openclaw/openclaw/issues/44148 was already closed as a duplicate of this open tracker.

## What This Changes

The issue requests a configurable owner/human handoff mechanism that pauses DM auto-replies when the linked account owner manually replies in the same conversation.

## Best Possible Solution

Define a WhatsApp-owned handoff contract first, then add per-account/per-DM pause state, an auto-reply suppression gate, docs, focused regression tests, and live WhatsApp proof before considering a shared cross-channel helper.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, tests, release, and related-item inspection give high confidence that the requested config and pause gate are absent, but there is no existing documented behavior to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the public API, resume policy, and channel scope. A WhatsApp-owned implementation with a small shared helper only if later justified is safer than reviving the closed broad cross-channel PR wholesale.
