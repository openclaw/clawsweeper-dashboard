## Summary

Keep open: current main still lacks the requested owner/human handoff. WhatsApp currently drops outbound `fromMe` DMs as non-pairing traffic and later inbound messages still enter normal reply dispatch, while the linked implementation PR was closed unmerged.

## What This Changes

Request adds a configurable per-DM owner/human takeover mode that pauses auto-replies after owner-authored outbound messages and resumes by cooldown or explicit command.

## Best Possible Solution

Define a stable channel config contract, then implement per-thread takeover state in the relevant channel plugins with a shared helper only for common cooldown/session bookkeeping.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main gap: no config/schema/handoff helper exists and the WhatsApp/Telegram handlers do not persist a per-thread pause state.

## Solution Assessment

Unclear until maintainers choose the public config contract. A maintainable solution should be channel-owned, with a small shared cooldown helper only if multiple channel plugins adopt the same semantics.
