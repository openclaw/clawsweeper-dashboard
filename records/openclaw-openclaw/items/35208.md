## Summary

Keep open: current main still lacks owner/human takeover config, per-thread pause state, and an auto-reply suppression gate; the related implementation PR at https://github.com/openclaw/openclaw/pull/58919 is closed unmerged, and the duplicate report at https://github.com/openclaw/openclaw/issues/44148 points back here as the canonical tracker.

## What This Changes

Request a configurable owner/human takeover mode that detects manual outgoing DM messages and pauses auto-replies per thread until cooldown or explicit resume.

## Best Possible Solution

Define a stable channel-owned config contract, implement per-account/per-conversation pause state where owner messages are reliable, and document/test resume behavior before broadening it across channels.

## Reproduction Assessment

Not applicable as a feature request; source inspection on current main gives high confidence that the requested config, pause state, and dispatch gate are absent.

## Solution Assessment

Unclear until maintainers choose the public config and resume policy; a WhatsApp-owned implementation with a small shared helper looks safer than a core-wide special case.
