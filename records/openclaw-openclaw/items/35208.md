## Summary

Keep open: current main still has no owner/human takeover config, pause state, or auto-reply suppression gate, and the linked implementation PR is closed unmerged.

## What This Changes

Requests configurable owner/human takeover that pauses per-thread DM auto-replies when the account owner manually replies and resumes by policy or command.

## Best Possible Solution

Define a stable channel-owned config contract, implement per-account/per-conversation pause state where the channel reliably observes owner messages, and document/test resume behavior before broadening it across channels.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main gap: the config/schema/dispatch paths do not contain owner takeover state or pause/resume handling.

## Solution Assessment

Unclear until maintainers choose the public config and resume policy. A WhatsApp-owned implementation with a small shared helper only for common cooldown bookkeeping would be safer than a core-wide special case.
