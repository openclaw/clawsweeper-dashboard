Full review comments:

- **[P1] Gate unscoped deltas before touching this turn:** `extensions/codex/src/app-server/run-attempt.ts:1287`
  - body: This condition still lets unscoped `response.*.delta` notifications through whenever `matchesActiveTurn` is absent, which is exactly the broadcast case the new lease-count guard is trying to handle. With two active leases, a foreign custom-tool-input delta still calls `noteNotificationReceived(..., { attemptProgress: true })`, so a wedged turn can keep its completion watchdog alive while another turn streams. Split the condition so native response deltas only refresh activity when `nativeResponseStreamDeltaMatchesActiveTurn` is true.
  - confidence: 0.93
- **[P1] Retry aborted idle timeouts before returning timeout payload:** `src/agents/embedded-agent-runner/run.ts:2382`
  - body: The retry classification is still inside `promptError && !aborted`, but the retry-exhausted test added in this PR models `turn_completion_idle_timeout` attempts with `aborted: true`, matching the timeout shape used elsewhere in these run-attempt tests. That means a real aborted completion-idle timeout skips `resolveCodexAppServerRecoveryRetry`, so the first attempt can return the timeout payload instead of performing the promised replay-safe retry, and the new test cannot exercise the real branch. Allow Codex app-server completion-idle timeout retry handling before the generic aborted-timeout return path.
  - confidence: 0.88
