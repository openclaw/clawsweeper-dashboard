- **[P2] Guard timeout binding cleanup:** `extensions/codex/src/app-server/run-attempt.ts:1900`
  - body: This now clears the Codex app-server binding on every timed-out native turn. `thread-lifecycle.ts` deliberately preserves an existing binding when it starts a transient thread for native-tool-disabled or temporarily unavailable dynamic-tool turns; a timeout in that transient thread would delete the valid previous binding and make the next normal turn lose resume continuity. Only clear when the stored binding points at this timed-out thread, or track that this attempt wrote/resumed the binding before clearing it.
  - confidence: 0.88

That leaves config/UI users with three different contracts for the same setting; either restore the 60s behavior or update the docs and manifest to the five-minute default.
  - confidence: 0.92
