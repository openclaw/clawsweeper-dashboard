## Summary

Keep open: this is a plausible focused update-path bug fix, but it is not merge-ready because the new smoke check only verifies that `node_modules/openclaw` is a directory instead of proving it resolves to the current host package, and the external PR has no real behavior proof.

## What This Changes

The PR updates post-core plugin convergence and installed-plugin record recovery to relink managed npm OpenClaw peer dependencies and fail smoke checks when active plugins lack a local host peer link.

## Best Possible Solution

Keep the post-core convergence repair direction, tighten peer-link validation to require the current OpenClaw host target, and merge only with real update proof for stale managed npm plugin state.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection shows current main does not relink managed npm peer links during post-core convergence, and the PR adds tests for that path but no real update run.

## Solution Assessment

No, not yet. Relinking during post-core convergence is the right boundary, but the smoke check should prove the peer resolves to the current host package and the contributor needs real update proof.
