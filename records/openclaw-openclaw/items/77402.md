## Summary

Keep this PR open for normal merge handling: current main still has the store-derived colon channel crash path, while the PR applies the same narrow guard already used for explicit channel IDs and the earlier changelog-credit finding has been addressed.

## What This Changes

The PR filters colon-containing Active Memory session-store channel values before embedded recall channel resolution and adds a changelog entry.

## Best Possible Solution

Land the narrow Active Memory guard with the existing changelog credit, keeping bundled-plugin directory validation strict and adding focused regression coverage if maintainers want stronger proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a stored `lastChannel` or `channel` containing `:` can be used as the strong embedded recall channel, and the downstream bundled-plugin directory validator rejects `:`.

## Solution Assessment

Yes. Reusing the existing explicit-channel colon guard for store-derived channel values is the narrowest maintainable fix; weakening `normalizeBundledPluginDirName` or encoding conversation IDs there would broaden a safety boundary unnecessarily.
