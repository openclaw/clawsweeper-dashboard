Keep open. The PR is a narrow functional fix for a source-reproducible current-main crash path, but it still needs required contributor changelog credit before merge.

## What This Changes

The PR filters colon-containing `active-memory` session-store channel values before embedded recall channel selection and adds an unreleased changelog entry.

## Best Possible Solution

Land the narrow active-memory guard after adding required contributor credit and focused regression coverage, while keeping bundled-plugin `dirName` validation strict.

## Reproduction Assessment

Yes. Source inspection shows current main can promote a stored value like `c2c:...` to the embedded recall `messageChannel`, and downstream bundled-plugin public-surface validation rejects `:` in a `dirName`.

## Solution Assessment

Yes. Filtering colon-containing store-derived channel values at `resolveRecallRunChannelContext` mirrors the existing explicit `channelId` guard and is narrower than relaxing plugin directory validation or adding a new config mode.
