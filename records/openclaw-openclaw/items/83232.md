## Summary

Keep open: current main and the latest release still call `install_homebrew` before Node detection, and https://github.com/openclaw/openclaw/pull/83246 is an open closing PR for the same bug, so the issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue asks for `scripts/install.sh` on macOS to skip Homebrew setup when an already-supported Node is on PATH and only install Homebrew before brew-backed dependency installation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83246, or an equivalent conditional Homebrew fix with installer regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes at source level: current main calls `install_homebrew` before `check_node`, and `install_homebrew` can exit for a non-admin macOS user without brew before an adequate Node on PATH is considered. I did not run the live macOS non-admin environment.

## Solution Assessment

Yes: the narrow maintainable fix is to move Homebrew setup behind the failed Node check and before other macOS brew-backed consumers such as missing Git. The active linked PR is the right path to review rather than closing the issue early.
