## Summary

Keep open: current main and the latest release still run Homebrew setup before Node detection, and the linked fix PR is open, unmerged, and already has a review concern about another macOS brew consumer.

## What This Changes

The issue requests that macOS `install.sh` skip Homebrew setup when an existing Node satisfies the installer minimum and only prepare Homebrew for install steps that actually need it.

## Best Possible Solution

Keep this issue open until the linked PR or a replacement fix makes Homebrew conditional for every macOS brew consumer, adds focused installer coverage, and then closes the issue after merge.

## Reproduction Assessment

Yes, at source level: current main calls `install_homebrew` before `check_node`, and that function can exit for a missing-brew non-admin macOS user before a supported Node on PATH is considered. I did not run a live macOS non-admin installer.

## Solution Assessment

Yes on the issue direction: Homebrew should be conditional rather than unconditional. The current linked PR's Node-only placement appears incomplete because `install_git` also uses brew on macOS, so the fix should cover that path too.
