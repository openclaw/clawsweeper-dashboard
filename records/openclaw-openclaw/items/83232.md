## Summary

Keep open: current main and the latest release still run macOS Homebrew setup before Node detection, so the installer can fail for a no-admin/no-brew user before it checks an already-supported Node on PATH.

## What This Changes

The issue asks for `install.sh` to skip Homebrew setup on macOS when an existing Node runtime is already sufficient.

## Best Possible Solution

Make Homebrew setup conditional on the macOS branches that actually need it, add installer regression coverage, and update the installer flow docs to match.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main calls `install_homebrew` before `check_node`, and that Homebrew path can exit for a non-admin macOS user before a valid Node v24 runtime is checked.

## Solution Assessment

Is this the best way to solve the issue? Yes: moving Homebrew setup to the actual brew-dependent macOS install branches is the narrow maintainable fix, while preserving Linux behavior, nvm loading, and the existing Node runtime policy.
