## Summary

This is a well-documented duplicate of the existing Google Vertex routing/API mismatch tracked in the canonical open issue; the new curl and debug-log evidence matches that same source-level failure mode rather than a distinct root cause.

## What This Changes

The issue reports that `google-vertex/*` models invoked through `openclaw agent` return Google 404s while the same Vertex URL and ADC token succeed via direct curl.

## Best Possible Solution

Keep the remaining fix centralized under https://github.com/openclaw/openclaw/issues/58775 and its linked PRs so `google-vertex/*` preserves native Vertex API identity through config, dynamic model resolution, auth evidence, and stream selection.

## Reproduction Assessment

Yes, source-level: current tests and provider code show a `google-vertex` model can retain `api: "google-gemini-cli"` while the native Vertex stream only activates for `api: "google-vertex"`. I did not run a live GCP ADC request in this read-only review.

## Solution Assessment

Yes, duplicate closure is the best cleanup path because the same root cause is already tracked in the canonical open Google Vertex routing issue. The implementation should be consolidated there rather than split across another parallel issue.
