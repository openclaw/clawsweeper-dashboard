## Summary

Keep open: the stale Anthropic OAuth user-agent bug is valid, but this PR is not merge-ready because the resolver depends on `@anthropic-ai/claude-code/package.json` being resolvable from OpenClaw's module graph while OpenClaw supports external `claude` installs and does not declare that package as a runtime dependency. The PR also still lacks real patched OpenClaw request/header proof.

## What This Changes

The PR adds a shared Claude Code package-version resolver and uses it in both Anthropic OAuth header builders, with resolver unit tests for success and failure cases.

## Best Possible Solution

Resolve the Claude Code version from a maintainer-approved supported CLI/backend install source for both OAuth builders, or intentionally package the dependency, then land one canonical PR with header tests and redacted OpenClaw request proof.

## Reproduction Assessment

Yes for source-level stale-header behavior: current main and `v2026.6.8` still use `claude-cli/2.1.75` in the Anthropic OAuth header builders. I did not live-test Anthropic's credentialed rejection in this read-only review.

## Solution Assessment

No. Sharing one lazy helper across both headers is the right direction, but the package-graph lookup is not the best fix for OpenClaw's supported external CLI install layouts.
