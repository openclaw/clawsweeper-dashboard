## Summary

Keep open: the PR addresses the right stale Anthropic OAuth user-agent path, but this head resolves an undeclared external Claude Code package at module import time and still lacks redacted patched OpenClaw request proof, so it is not merge-ready.

## What This Changes

The branch adds a shared Claude Code package-version resolver, uses it in both Anthropic OAuth user-agent builders, and adds resolver unit tests.

## Best Possible Solution

Resolve the Claude Code version lazily only when constructing Anthropic OAuth Claude CLI headers, from a supported and package-proven source, then cover both builders with focused tests and redacted OpenClaw request proof.

Yes for the source-level stale-header path: current main still builds both Anthropic OAuth headers from `claude-cli/2.1.75`. I did not live-test Anthropic's credentialed rejection in this read-only review.

## Solution Assessment

No. A shared resolver is the right direction, but resolving an undeclared external package at module import time is not the narrowest maintainable fix; the resolver should be lazy to the OAuth header path and proven against supported install layouts.
