## Summary

Keep open. Current main still delegates Microsoft/Edge synthesis to `node-edge-tts@1.2.10`, whose published transport builds the same SSML WebSocket frame the reporter identified, and Feishu only delivers/transcodes the resulting audio.

## What This Changes

Fix Edge/Microsoft TTS so SSML/XML tags are not spoken aloud when Feishu sends synthesized voice replies.

## Best Possible Solution

Own or correct the Microsoft plugin's Edge transport so it sends protocol-compatible speech requests and proves generated audio/transcripts contain only the intended spoken text, leaving Feishu delivery unchanged.

Current main passes `req.text` through `extensions/microsoft/speech-provider.ts` into `extensions/microsoft/tts.ts`, which delegates transport to `node-edge-tts@1.2.10`; that dependency sends an SSML WebSocket frame that the reporter hears as spoken XML.
