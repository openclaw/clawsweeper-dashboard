## Summary

Keep open. Current main still delegates Microsoft/Edge speech synthesis to `node-edge-tts@1.2.10`, whose published transport builds the SSML WebSocket frame the reporter identified; Feishu only delivers/transcodes the resulting audio, so the provider bug remains unresolved.

## What This Changes

Fix Edge/Microsoft TTS so SSML/XML tags are not spoken aloud when Feishu sends synthesized voice replies.

## Best Possible Solution

Own or correct the Microsoft plugin's Edge transport so it sends protocol-compatible speech requests and proves generated audio/transcripts contain only the intended spoken text, leaving Feishu delivery unchanged.
