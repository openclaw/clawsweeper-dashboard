## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep callback queries on the serialized lane:** `extensions/telegram/src/active-run-ingress.ts:41`
  - body: `resolveTelegramSequentialMessage` includes `ctx.update.callback_query.message`, and the new candidate check only looks at that message text/caption and media fields. A non-approval button/select callback on a text message can therefore receive a unique `active-run-ingress` key while a reply fence is active, so callback work can interleave with the running turn instead of staying on the serialized callback lane. Exclude `callback_query` updates from this ingress path and add a regression test.
  - confidence: 0.92
- **[P2] Keep legacy file SecretRef repair out of runtime:** `src/secrets/resolve.ts:459-460`
  - body: This fallback resolves bare root-key file SecretRef ids even though the config/gateway/plugin contract and docs still require file ids to be absolute JSON pointers. That leaves schema-invalid legacy config silently working at runtime without a doctor migration, creating a second SecretRef contract. Move the compatibility handling to `openclaw doctor --fix` or align schema/docs with an explicit maintainer-approved runtime contract.
  - confidence: 0.88
