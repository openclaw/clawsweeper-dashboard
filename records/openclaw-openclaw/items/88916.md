Full review comments:

- **[P1] Keep callback queries on the serialized lane:** `extensions/telegram/src/active-run-ingress.ts:41`
  - body: `resolveTelegramSequentialMessage` includes `ctx.update.callback_query.message`, so a button/select callback on a text or captioned message can satisfy the active-run text candidate check. Once a reply fence is active, this returns a distinct `active-run-ingress` key, and grammY only serializes matching keys; callback/plugin/select/model work can therefore interleave with the active turn instead of staying on the serialized callback/chat lane. Exclude `callback_query` updates from this ingress path and add focused direct and isolated-path coverage.
  - confidence: 0.92
- **[P2] Keep legacy file SecretRef repair out of runtime:** `src/secrets/resolve.ts:453`
  - body: This fallback resolves bare root-key file SecretRef ids even though current config validation, gateway protocol schema, plugin SDK validation, and docs still require file ids to be `value` or absolute JSON pointers. That creates a runtime-only second contract for schema-invalid config and hides a migration problem that should be handled by doctor or by an explicit cross-surface compatibility decision.
  - confidence: 0.9
