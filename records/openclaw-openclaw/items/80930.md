## Summary

Keep open: this is a plausible bug-fix PR, but the current patch would drop substantive plain-`think` replies that end with `NO_REPLY`, and it still lacks real behavior proof.

## What This Changes

The PR adds reasoning-prefix detection to the shared silent-reply payload classifier and tests it in token and reply normalization coverage.

## Best Possible Solution

Adjust the shared silent-reply classifier so it suppresses only reasoning-prefix shapes without user-facing content after the marker, add regression coverage for plain `think` plus substantive answer plus trailing `NO_REPLY`, and provide redacted real outbound/channel proof.

## Reproduction Assessment

Yes. Source inspection of current main shows the classifier does not treat reasoning-prefixed `NO_REPLY` as silent, and the normalizer can deliver the remaining preamble after stripping the token; I did not run tests in this read-only review.

## Solution Assessment

No. The PR handles the reported sample, but its plain-marker path is broader than the established substantive trailing-token guard and can silently drop real answers.
