## Summary

Keep open: the narrow group-chat leak fix is plausible and the PR has terminal proof, but the patch broadens silent-payload classification without applying the existing direct-conversation rewrite/preserve policy to those newly classified payloads.

## What This Changes

The branch extends the shared silent-reply payload classifier to treat reasoning-prefixed final `NO_REPLY` outputs as silent and adds token/normalizer regression tests.

## Best Possible Solution

Land one canonical fix that classifies reasoning-prefixed silent payloads and applies existing silent-reply policy consistently across dispatcher, route, runner, and outbound surfaces, with regression coverage for group suppression, direct rewrite/preserve, and substantive-message false positives.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared classifier misses reasoning-prefixed `NO_REPLY`, and the normalizer can return the remaining reasoning preamble after stripping the token.

## Solution Assessment

No. The classifier direction is reasonable, but the PR must also route newly recognized silent final payloads through the existing direct-conversation rewrite/preserve policy before normalization drops them.

## Review Findings
