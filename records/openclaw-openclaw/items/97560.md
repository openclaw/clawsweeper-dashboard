## Summary

Keep open: the revised PR fixes the single-root outage path and now has sufficient copied terminal proof, but it still preserves an entire configured root when only a nested descendant is unreadable, so genuine deletions in readable siblings under that root can be retained instead of pruned.

## What This Changes

The PR changes memory-wiki unsafe-local sync to record unreadable configured roots, preserve matching source-sync entries during prune, and add regression coverage for transient outages, explicit files, mixed roots, and normal directory deletions.

## Best Possible Solution

Track unreadable scopes at the deepest failed source path, preserve only entries under those scopes, keep normal pruning for readable siblings and roots, and merge after maintainers accept the explicit-file retention tradeoff.
