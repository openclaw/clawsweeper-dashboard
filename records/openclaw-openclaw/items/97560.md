## Summary

Keep open: the PR addresses a real data-loss path and has sufficient real behavior proof, but the current branch still preserves an entire configured root when only a nested descendant is unreadable, so readable sibling deletions can stop pruning.

## What This Changes

The PR changes memory-wiki unsafe-local sync to record unreadable roots, preserve matching imported-source entries during prune, and add regression tests for transient outages plus normal deletions.

## Best Possible Solution

Track unreadable scopes at the deepest failed source path, preserve only entries under those scopes, keep normal pruning for readable siblings and roots, and merge after maintainers accept the explicit-file retention tradeoff.
