Full review comments:

- **[P1] Persist the matching anchor for retried payloads:** `apps/ios/Sources/Health/HealthExportService.swift:157-158`
  - body: When a retryable failure saves a pending body, the matching new anchor is discarded. After that body later succeeds, this branch clears it and falls through to a fresh read using the old anchor, so the same HealthKit window can be uploaded again before the anchor advances; store pending payloads with their anchor or otherwise advance exactly once on retry success.
  - confidence: 0.9
- **[P1] Use background-readable or explicitly deferred export state:** `apps/ios/Sources/Health/HealthExportFileStore.swift:45`
  - body: The export can run from BGProcessingTask/HKObserver wakes, but `anchor.bin` and `pending.json` are written with complete file protection and read failures return nil. Apple documents complete-protected files as accessible only while unlocked, so a locked wake can treat existing pending data or anchors as absent and upload the wrong HealthKit window instead of safely deferring. ([developer.apple.com](https://developer.apple.com/documentation/Foundation/NSData/WritingOptions/completeFileProtection?utm_source=openai))
  - confidence: 0.89
- **[P2] Gate Health Export scheduling behind saved configuration:** `apps/ios/Sources/OpenClawApp.swift:108`
  - body: This schedules the Health Export BGProcessingTask on every launch, and again when the app backgrounds, even when the user never enabled Health Export. Keep disabled users on the existing background path and schedule Health Export only after saved configuration, with scheduling avoided or canceled after clear where possible.
  - confidence: 0.86
- **[P2] Add the HealthKit background delivery entitlement:** `apps/ios/Sources/OpenClaw.entitlements:7-10`
  - body: `startObserving` calls `enableBackgroundDelivery` for each HealthKit sample type, but the entitlements file only adds the base HealthKit keys. Apple documents that this API path requires the HealthKit Background Delivery entitlement on iOS 15 and later, so add that entitlement when owning this background path or remove the background-delivery claim and prove the remaining BGProcessing behavior.
