## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use background-readable protection for export state:** `apps/ios/Sources/Health/HealthExportFileStore.swift:45`
  - body: The export is meant to run from BGProcessingTask/HKObserver wakes, but anchor.bin and pending.json are written with complete file protection. When the device is locked those reads can fail, and the reader then treats the missing anchor as an initial read, so background exports can replay or skip the wrong health batch instead of safely deferring.
  - confidence: 0.89
- **[P2] Gate health background work behind configuration:** `apps/ios/Sources/OpenClawApp.swift:108`
  - body: This schedules the Health Export BGProcessingTask on every launch, and the new background transition hook schedules it again even when the user never enabled Health Export. Keep disabled users on the existing background path and schedule Health Export only after a saved configuration, otherwise the new feature is not actually opt-in.
  - confidence: 0.86
