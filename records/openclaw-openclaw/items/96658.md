## Summary

Keep open: current main and v2026.6.10 still leave only a 2,048-token prompt budget for the reported 4,096-token model case, so the 3,544-token lightweight cron prompt still overflows into `compact_only` when there is no reducible content. Open fix candidates exist, but no merged fix is on main yet, so this issue should remain the canonical tracker until maintainers land or reject the fix path.

## What This Changes

The issue requests fixing lightweight isolated cron runs that overflow small model context windows because the reserve floor leaves too little prompt budget and routes into no-op compaction.

## Best Possible Solution

Land a focused compaction-budget fix that keeps lightweight isolated cron prompts on very small context windows out of no-op `compact_only` recovery while preserving shared caller behavior, then close this issue after merge.
