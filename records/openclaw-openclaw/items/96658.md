## Summary

Keep open: current main and v2026.6.10 still route the reported 4,096-token / 20,000-reserve / 3,544-prompt case to `compact_only`, and three open PRs reference this issue with closing syntax but none has merged yet.

## What This Changes

The issue reports that isolated cron sessions using `lightContext: true` can overflow context on small-context models because reserve budgeting leaves too little prompt budget and routes into no-op compaction.

## Best Possible Solution

Review the linked fix PRs and land one focused compaction-budget fix that lets history-free lightweight isolated cron prompts fit small context windows without creating a new config knob or weakening shared compaction behavior.
