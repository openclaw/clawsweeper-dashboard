## Summary

Current main and shipped releases already keep small-context models from being starved by the compaction reserve floor, so the reported 409-token prompt budget no longer matches the shared compaction path.

## What This Changes

The issue reports isolated cron jobs with `lightContext: true` failing on small-context models because reserve tokens leave only a 409-token prompt budget and trigger `compact_only` loops.

## Best Possible Solution

Keep the shipped shared reserve-floor and prompt-budget clamp, and ask for a new report only if a specific installed artifact still shows the old constants or 409-token precheck budget.

## Reproduction Assessment

No. Current main and the checked release source compute a 2,048-token prompt budget for a 4,096-token context window, not the reported 409-token budget.

## Solution Assessment

Yes. The existing fix lives in the shared compaction settings/precheck layer, which is narrower and more maintainable than a cron-only workaround.
