## Summary

Keep open: current main and v2026.6.10 still leave too little prompt budget for the reported 4,096-token lightweight cron case, and the viable fix path is still an open PR rather than merged main behavior.

## What This Changes

Fix isolated `lightContext` cron sessions on small-context models so near-empty sessions do not overflow into no-op `compact_only` recovery.

## Best Possible Solution

Review and land a focused compaction-budget fix, or replace the open candidate with a complete lightweight-cron fix that preserves shared reserve behavior for larger and tool-heavy runs.
