## Summary

Keep open: current main and v2026.6.9 still only mitigate the Feishu DM session-continuity problem. ## What This Changes

The issue asks OpenClaw to stop Feishu DM sessions from being rebuilt after restarts by addressing direct-session maintenance pruning and duplicate or malformed Feishu direct peer keys.

## Best Possible Solution

Keep this issue open until maintainers choose a Feishu DM retention policy and a Feishu-boundary identity repair plan, with `per-account-channel-peer` documented as the current multi-account workaround.
