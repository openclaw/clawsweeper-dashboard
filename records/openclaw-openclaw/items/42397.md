## Summary

Keep open: current main and latest release still expose only a run-level heartbeat model, parse HEARTBEAT.md tasks as name/interval/prompt, and batch all due tasks into one heartbeat turn, so the requested mixed-model per-task heartbeat behavior remains unimplemented and needs maintainer product/API direction.

## What This Changes

Request per-task model selection hints for HEARTBEAT.md tasks instead of a single run-level heartbeat model.

## Best Possible Solution

Define one canonical heartbeat task-routing contract, or expose the missing plugin/API seam, before adding task-level model selection to HEARTBEAT.md.

## Reproduction Assessment

Yes for the source-level feature gap, not for a bug repro: current main exposes only run-level heartbeat model selection and task parsing lacks a model field.

## Solution Assessment

Unclear: the HEARTBEAT.md hint syntax is plausible, but a parser-only field would not work while all due tasks are batched into one heartbeat run.
