## Summary

Keep open: current main still supports only a whole-heartbeat-run model, while HEARTBEAT.md tasks have no model metadata and due tasks are batched into one reply turn. This is a real feature/API request, not an implemented cleanup close.

## What This Changes

Add a HEARTBEAT.md task-level model hint so selected heartbeat tasks can use a different model than the run-level heartbeat model.

## Best Possible Solution

Define a documented task-level model field validated through the existing model alias/provider resolution path, then implement model-aware heartbeat execution after maintainers choose batching versus split-turn semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks task-level model metadata and uses one model selection for the heartbeat turn.

## Solution Assessment

Unclear until maintainers choose the contract. A typed model field validated through existing model alias/provider rules is safer than a magic rerun keyword, but batching versus split-turn behavior needs product/API review.
