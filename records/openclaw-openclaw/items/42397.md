## What This Changes

The issue requests a way for individual tasks in `HEARTBEAT.md` to override or escalate the model used during heartbeat execution.

## Best Possible Solution

Define a canonical task-level model reference or escalation syntax, then split heartbeat execution so tasks that request another model run with the right agent context and update state consistently.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken existing behavior. Source inspection confirms current main only supports one `heartbeat.model` per run, but there is no failing contract to reproduce.

## Solution Assessment
