## Summary

Keep open: current main and v2026.5.19 still support only a run-level heartbeat model, while HEARTBEAT.md task metadata has no model field and due tasks are batched into one heartbeat turn. This remains a feature/API request that needs maintainer product direction.

## What This Changes

Request a HEARTBEAT.md task-level model hint so selected heartbeat tasks can run on a different model than the run-level heartbeat.model.

## Best Possible Solution

Define a documented task model field validated through existing model alias/provider resolution, then implement model-aware heartbeat execution after maintainers choose batching versus split-turn semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection shows task metadata is limited to name, interval, and prompt, while heartbeat execution uses one model selection for the turn.

## Solution Assessment

Unclear until maintainers choose the contract. A typed model field validated through normal model alias/provider rules is safer than a magic rerun keyword, but batching versus split-turn behavior needs product/API review.
