## Summary

Keep open: current main still supports one run-level `heartbeat.model` and batches due `HEARTBEAT.md` tasks into a single prompt/model call. Per-task model hints would add new task metadata and mixed-model heartbeat execution semantics, so this still needs maintainer product/API direction.

## What This Changes

Request adding per-task model or escalation hints to `HEARTBEAT.md` so selected heartbeat tasks can run with a different model than the default heartbeat run.

## Best Possible Solution

Define the canonical task-level model or escalation semantics, including split-run behavior, session/task-state handling, provider alias rules, docs, and focused regression tests, before implementation.
