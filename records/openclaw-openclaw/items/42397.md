## Summary

Keep open: current main still supports only a run-level `heartbeat.model` and batches due `HEARTBEAT.md` tasks into one heartbeat turn. There is no task-level model metadata or mixed-model execution path, so this remains a feature and policy/schema decision.

## What This Changes

Request per-task model hints in `HEARTBEAT.md` so selected periodic heartbeat tasks can run on a different model than the run-level heartbeat model.

## Best Possible Solution

Define an explicit `HEARTBEAT.md` task metadata contract for model selection, then implement parsing, grouping/execution semantics, docs, and regression coverage around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main gives high confidence that task-level model metadata is not currently parsed or executed.

## Solution Assessment

Unclear until maintainers choose the contract. An explicit task metadata schema with controlled model references or aliases would be safer than a magic keyword such as `ESCALATE_TO_PRIMARY`.
