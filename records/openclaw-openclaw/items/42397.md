- `impact:auth-provider`: The requested behavior changes model choice and provider-routing semantics for heartbeat execution.

## Summary

Keep open: current main and v2026.5.22 still support only one heartbeat model per run. `HEARTBEAT.md` tasks carry only name, interval, and prompt metadata, and due tasks are batched into one heartbeat turn, so this remains a real feature request that needs product/API judgment.

## What This Changes

Allow individual `HEARTBEAT.md` task blocks to request a specific model or escalation to the agent's primary model.

## Best Possible Solution

Add a typed per-task model field resolved through existing model alias/provider rules, and define mixed-model due-task execution before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and v2026.5.22 inspection give high confidence that current OpenClaw lacks task-level heartbeat model metadata.

## Solution Assessment

Unclear until maintainers choose the contract. A typed task-level model field is safer than a magic rerun keyword, but batching versus split-turn behavior needs product/API review.
