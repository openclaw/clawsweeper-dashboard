## Summary

Keep open: current main and latest release still expose only a whole-heartbeat-run `heartbeat.model`, while `HEARTBEAT.md` task blocks carry only name, interval, and prompt and all due tasks are batched into one reply turn. This remains a concrete feature request that needs a maintainer product/API decision rather than conservative cleanup.

## What This Changes

Request to let individual `HEARTBEAT.md` task blocks specify a model override for higher-quality periodic tasks such as memory consolidation.

## Best Possible Solution

Define a documented typed task-level model field that resolves through the existing model alias/provider path, then implement grouped or split heartbeat execution with focused coverage for timestamps, response-tool behavior, delivery, and upgrade semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and latest-release inspection give high confidence that current OpenClaw lacks task-level model metadata and uses one model selection for each heartbeat reply turn.

## Solution Assessment

Unclear until maintainers choose the contract. A typed task-level model field resolved through existing alias/provider rules is safer than a magic rerun keyword, but batching versus split-turn behavior needs product/API review.
