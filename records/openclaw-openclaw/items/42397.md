## Summary

Keep open: current main and the latest release still support only a run-level heartbeat model override, while HEARTBEAT.md tasks carry no task-level model metadata and are executed as one batched heartbeat turn. This remains a valid feature request that needs maintainer product/API direction before implementation.

## What This Changes

The issue requests task-level model hints in HEARTBEAT.md so selected heartbeat tasks can use a different model than the run-level heartbeat model.

## Best Possible Solution

Define a documented HEARTBEAT.md task metadata contract, validate model references through the existing model alias/provider resolution rules, and implement model-aware heartbeat execution with docs and regression coverage once the semantics are approved.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that task-level model metadata is not parsed or executed.

## Solution Assessment

Unclear until maintainers choose the contract. A documented model field with controlled alias/provider-model resolution is safer than a magic keyword rerun path, but the batching versus split-turn behavior needs product/API review.
