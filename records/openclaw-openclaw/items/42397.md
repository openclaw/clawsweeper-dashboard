## Summary

Keep open: current main and the latest release still support only a run-level heartbeat model, while HEARTBEAT.md task parsing has no model metadata and due tasks are batched into one heartbeat turn. This is a real heartbeat model-routing feature request, but it needs maintainer product/API judgment before implementation.

## What This Changes

Request per-task model hints in HEARTBEAT.md so selected heartbeat tasks can use a different model than the run-level heartbeat.model setting.

## Best Possible Solution

Define a typed task-level model field in HEARTBEAT.md that resolves through existing alias/provider rules, then implement grouped or split heartbeat execution with explicit timestamp, delivery, response-tool, and docs semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and v2026.5.20 inspection give high confidence that OpenClaw currently lacks task-level heartbeat model metadata and uses one model selection per heartbeat turn.

## Solution Assessment

Unclear until maintainers choose the contract. A typed task-level model field resolved through the existing alias/provider rules is safer than a magic rerun keyword, but batching versus split-turn behavior needs product/API review.
