## Summary

Keep open: current main still supports only a run-level `heartbeat.model` and batches due `HEARTBEAT.md` tasks into one heartbeat turn, so task-level model hints remain an unimplemented feature and schema/policy decision.

## What This Changes

Request per-task model selection metadata in `HEARTBEAT.md` so high-value heartbeat tasks can use a stronger model without changing the whole heartbeat run.

## Best Possible Solution

Define a documented `HEARTBEAT.md` task metadata contract and execution model, then implement parser, runner, config/docs, and regression coverage around controlled model references or aliases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source inspection gives high confidence that task-level model metadata is not parsed or executed.

## Solution Assessment

Unclear until maintainers choose the contract. A documented task metadata schema with controlled model references or aliases would be safer than a magic keyword rerun path.
