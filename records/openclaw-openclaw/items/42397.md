## Summary

Keep open: current main still has one heartbeat-run model override, while `HEARTBEAT.md` tasks only carry name, interval, and prompt before due tasks are batched into one model call. Per-task model hints would add new task metadata and mixed-model execution semantics, so this needs maintainer product/API direction rather than cleanup closure.

## What This Changes

Request per-task model hints in `HEARTBEAT.md` so selected heartbeat tasks can use a stronger or primary model while routine heartbeat work stays on a cheaper model.

## Best Possible Solution

Define a canonical task-level model or escalation syntax, then split heartbeat execution so model-tagged tasks run with the intended provider/model while preserving task state and shared session context.
