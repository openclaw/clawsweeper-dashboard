## Summary

Keep open: current main still lacks HEARTBEAT.md task-level model metadata and mixed-model heartbeat execution. The existing before-model-resolution plugin hook covers whole-turn routing and supports related ClawHub/plugin solutions, but it does not fully close this heartbeat-specific API decision when several due tasks are batched into one heartbeat run.

## What This Changes

Request per-task model hints or escalation markers in HEARTBEAT.md so different periodic heartbeat tasks can use different models without losing shared context.

## Best Possible Solution

Maintainers should first decide whether heartbeat task model hints are a core HEARTBEAT.md contract or plugin policy; if core, split or group due heartbeat tasks by resolved model while preserving heartbeatTaskState, delivery behavior, and shared-session semantics.

## Reproduction Assessment

Yes for current-source behavior: HEARTBEAT.md tasks have no model metadata, and all due tasks are batched into one heartbeat model call. This is not a failing-bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the existing plugin hook is the best current seam for whole-turn model routing, but it does not fully answer per-task HEARTBEAT.md syntax or mixed-model execution when several tasks are due together.
