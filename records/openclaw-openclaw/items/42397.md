## Summary

Keep open: current main supports a run-level heartbeat model override and HEARTBEAT.md task batching, but it still has no per-task model field or mixed-model heartbeat execution path. This is a user-facing task-schema and model-policy feature request, not an already-fixed bug.

## What This Changes

Requested support for selecting a different model for specific HEARTBEAT.md tasks instead of using only the run-level heartbeat model.

## Best Possible Solution

Define an explicit HEARTBEAT.md task metadata contract with allowed model references or aliases, then implement parsing, grouping/execution semantics, docs, and regression coverage around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has no task-level model field or execution path.

## Solution Assessment

Unclear until maintainers choose the contract. An explicit task metadata schema and controlled model-selection policy would be safer than an implicit magic keyword such as `ESCALATE_TO_PRIMARY`.
