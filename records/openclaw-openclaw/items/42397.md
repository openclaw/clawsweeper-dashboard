## Summary

Keep open: current main and the latest release still support only a run-level heartbeat model, while structured HEARTBEAT.md tasks carry only name, interval, and prompt and are executed as one batched heartbeat turn. This remains a feature/API request needing maintainer product direction before implementation.

## What This Changes

Request per-task model hints in HEARTBEAT.md so selected heartbeat tasks can run on a different model than the default heartbeat run model.

## Best Possible Solution

Define a documented HEARTBEAT.md task model field, validate it through existing model alias/provider resolution, and implement model-aware heartbeat execution with docs and regression coverage after batching versus split-turn semantics are approved.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows task metadata is limited to name/interval/prompt and heartbeat execution uses one model selection per turn.

## Solution Assessment

Unclear until maintainers choose the contract. A typed model field validated through normal model alias/provider rules is safer than a magic rerun keyword, but batching versus split-turn behavior needs product/API review.
