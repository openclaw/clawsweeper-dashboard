## Summary

Keep open: current main still has only a run-level heartbeat model selection path, while `HEARTBEAT.md` tasks are parsed and batched without task-level model metadata. This is a real feature request, but it needs maintainer API/product decisions before implementation.

## What This Changes

Request to allow individual `HEARTBEAT.md` tasks to hint or escalate to a different model than the run-level `heartbeat.model`.

## Best Possible Solution

Define a documented `HEARTBEAT.md` task metadata contract and execution model, then implement parser, runner, config/docs, and regression coverage around controlled model references or aliases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that task-level model metadata is not parsed or executed.

## Solution Assessment

Unclear until maintainers choose the contract. A documented task metadata schema with controlled model references or aliases looks safer than a magic keyword rerun path.
