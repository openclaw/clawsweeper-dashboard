## Summary

Keep open: current main and v2026.5.22 still only support a single run-level heartbeat model, while `HEARTBEAT.md` task parsing carries no per-task model metadata and due tasks are batched into one heartbeat turn. This remains a valid feature request, but the task metadata contract and mixed-model execution semantics need maintainer product/API judgment.

## What This Changes

Request per-task model hints in `HEARTBEAT.md` so selected heartbeat tasks can use a different model from the run-level heartbeat model.

## Best Possible Solution

Add a typed task-level model field to `HEARTBEAT.md`, resolve it through the existing model alias/provider rules, and document whether mixed-model due tasks run as grouped turns or separate heartbeat turns.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and the v2026.5.22 tag give high confidence that OpenClaw currently lacks task-level heartbeat model metadata and uses one model selection per heartbeat turn.

## Solution Assessment

Unclear until maintainers choose the contract. A typed task-level model field resolved through existing alias/provider rules is safer than a magic rerun keyword, but batching versus split-turn behavior needs product/API review.
