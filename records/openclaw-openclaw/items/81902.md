## Summary

Keep open: current main still has the source-level failover path where a concrete `rate_limit` profile failure reason is replaced with `timeout`, and the open linked fix PR should remain the implementation path until it is proved and merged.

## What This Changes

Google rate-limited auth profiles should enter cooldown and rotate to a backup profile instead of being treated as repeated timeouts.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81927 or an equivalent narrow reason-preservation fix after real behavior proof shows the Google multi-profile cooldown path working, then close this issue as implemented.

## Reproduction Assessment

Yes, source-level: pass `timedOut` or `idleTimedOut` with `assistantProfileFailureReason: "rate_limit"` into `handleAssistantFailover`; current main resolves the persisted reason to `timeout` and skips cooldown marking. I did not run a live Google quota-exhaustion flow.

## Solution Assessment

Yes, preserving an already-classified actionable profile failure reason over the generic timeout is the narrowest maintainable fix. Header or parser work should be a follow-up only if real proof shows the classifier never receives the rate-limit signal.
