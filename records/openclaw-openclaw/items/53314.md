## Summary

Keep open: the reported retry-budget race is still source-reproducible on current main, and the PR is a plausible direction, but the current patch has two lifecycle correctness blockers plus missing external real behavior proof.

## What This Changes

This PR widens subagent announce results with a pending-descendants sentinel, routes that sentinel through cleanup without incrementing retries, broadens descendant deferral, and adds focused tests.

## Best Possible Solution

Rebase or replace the branch on current main, keep a typed blocked-by-descendants outcome through the completion hard-expiry window, preserve normal finalize-error recovery, and add focused regression coverage plus redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can return false from the inner descendant gate, then re-count zero descendants in cleanup and charge the announce retry budget without a real delivery attempt.

## Solution Assessment

No. The sentinel direction is reasonable, but this PR is not the best final fix until the pending-descendant branch honors completion hard-expiry semantics and reuses normal finalize failure recovery.

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor completion hard expiry for pending-descendant deferrals:** `src/agents/subagent-registry-lifecycle.ts:275`
  - body: For completion-message runs, this branch stops trusting the pending-descendants signal after `ANNOUNCE_EXPIRY_MS` even though the completion hard-expiry window is longer. From five to thirty minutes it can fall through to the old second-count path, race to zero descendants, and consume retries without a delivery attempt.

  - confidence: 0.82
