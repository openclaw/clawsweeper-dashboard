## Summary

Keep open: current main still has the descendant-blocked announce path that can be counted as a delivery failure, but this PR should not merge as-is because the proposed lifecycle branch shortens completion-message deferral, can swallow finalize failures, and lacks the required changelog entry.

## What This Changes

The PR changes subagent announce cleanup to return a distinct descendant-blocked signal, defer cleanup without incrementing announce retry count, broaden descendant deferral, and update related tests.

## Best Possible Solution

Preserve a typed descendant-blocked lifecycle outcome through cleanup with the correct per-entry expiry window, normal finalize-error recovery, focused regression coverage, and an active-version changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: make the inner announce descendant count positive so runSubagentAnnounceFlow returns false, then make the cleanup recheck see zero descendants so retry accounting is consumed instead of preserving the blocked state.

## Solution Assessment

No. The distinct blocked signal is the right direction, but this patch is not the best final form until it honors completion hard expiry, preserves finalize recovery, and records the user-visible fix in the changelog.
