## Summary

Keep open. Current main still has the reported source-reproducible subagent announce race, but this PR is not merge-ready because the latest head introduces two lifecycle regressions, is currently conflicting, and lacks after-fix real behavior proof.

## What This Changes

The PR changes subagent announce cleanup to return a typed `-1` when descendant gating blocks delivery, defer cleanup without retry increments, broaden descendant deferral, and add focused regression tests.

## Best Possible Solution

Keep this PR open until a rebased fix preserves the pending-descendants signal through the completion hard-expiry window, reuses normal finalize error recovery, includes focused regression coverage, and provides redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main returns `false` from the descendant gate and then re-counts descendants before incrementing retries, so a 1-to-0 transition between those checks consumes retry budget without a real delivery attempt.

## Solution Assessment

No. The typed blocked-by-descendants signal is the right direction, but this branch stops preserving it after five minutes for completion-message runs and swallows finalize errors instead of using the normal recovery path.
