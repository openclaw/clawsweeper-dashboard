## Summary

Keep open: current main and the latest release still have the reported two-check descendant announce race, but this external PR is not merge-ready because the patch has two cleanup-lifecycle correctness blockers and lacks required after-fix real behavior proof.

## What This Changes

The PR makes descendant-blocked subagent announces return a sentinel, defers cleanup without consuming announce retries, broadens descendant deferral, and adds focused cleanup tests.

## Best Possible Solution

Preserve a typed blocked-by-descendants outcome through cleanup, defer without consuming retries until the correct expiry window, retain normal finalization failure recovery, and add focused regression coverage on current main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can return false from the inner descendant gate, then the finalizer can re-count zero descendants and charge the announce retry budget instead of preserving a pending-descendant defer.

## Solution Assessment

No. The sentinel direction is reasonable, but this PR is not the best final fix until the pending-descendant branch honors the completion hard-expiry window and uses the normal finalize-error recovery path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor completion expiry for pending-descendant deferrals:** `src/agents/subagent-registry-lifecycle.ts:275`
  - body: This branch stops trusting the pending-descendants signal after `ANNOUNCE_EXPIRY_MS`. For completion-message runs older than five minutes but still before `ANNOUNCE_COMPLETION_HARD_EXPIRY_MS`, it falls through to the old second count check; if that count races to zero, retries are consumed without an attempted delivery. Use the completion hard-expiry window for completion entries.
  - confidence: 0.89
- **[P2] Restore finalize-error recovery for pending descendants:** `src/agents/subagent-registry-lifecycle.ts:415-417`
  - body: The new `-1` path swallows `finalizeSubagentCleanup` failures with an empty catch. If finalization rejects after `beginSubagentCleanup` set `cleanupHandled`, the run can stay handled but incomplete and later resumes will skip cleanup; reuse the normal warning/reset handler.
  - confidence: 0.82
