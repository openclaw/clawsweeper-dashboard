## Summary

Keep open: current main still has the descendant-announce retry race and this PR is directionally useful, but the branch is conflicting, missing after-fix real behavior proof, and still has two lifecycle correctness blockers.

## What This Changes

The PR makes descendant-blocked subagent announces return a sentinel, defers cleanup without incrementing announce retries, broadens descendant deferral, and adds focused cleanup tests.

## Best Possible Solution

Rebase or replace the branch on current main, preserve a typed blocked-by-descendants outcome through the completion hard-expiry window, keep normal finalize-error recovery, and add focused regression plus redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can return false from the inner descendant gate, then re-count zero descendants in cleanup and charge the announce retry budget without a real delivery attempt.

## Solution Assessment

No. The sentinel direction is reasonable, but this PR is not the best final fix until the pending-descendant branch honors the completion hard-expiry window and reuses normal finalize failure recovery.
