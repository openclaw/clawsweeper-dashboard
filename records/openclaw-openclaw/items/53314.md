## Summary

Keep open: current main still has a source-reproducible descendant-gate/retry race, and this PR is useful but still has lifecycle correctness blockers plus a missing changelog entry before it can merge.

## What This Changes

The PR adds a descendant-blocked announce outcome, routes it through cleanup as a defer without retry accounting, broadens descendant deferral logic/tests, and updates timeout expectations.

## Best Possible Solution

Preserve a typed descendant-blocked lifecycle outcome through cleanup, honor the completion-message hard-expiry window, route finalize failures through existing cleanup recovery, add regression coverage, and include an active-version changelog entry.

## Reproduction Assessment

Yes. Source inspection shows a current-main path where the inner announce gate returns `false` for pending descendants, while the later cleanup recheck can see zero descendants and count the blocked announce as a retry failure.

## Solution Assessment

No. The sentinel/defer direction is reasonable, but the proposed implementation should keep completion-message deferral through the 30-minute hard expiry, preserve finalize-error recovery, and add the required changelog entry.

  - body: The pending-descendants fast path only defers while `endedAgo <= ANNOUNCE_EXPIRY_MS`. If finalization rejects, later resumes can skip the run as handled but incomplete; route this through the same log and `cleanupHandled=false` recovery used by the normal finalize path.
