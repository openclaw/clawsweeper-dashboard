## Summary

Keep open: the reported subagent announce race is plausible from current source, but this PR is not mergeable as-is because the branch regresses completion hard-expiry handling, drops finalize-error recovery, and lacks required real behavior proof.

## What This Changes

The PR changes subagent announce cleanup to return a typed `-1` for descendant-gated announces, defer pending-descendant cleanup without retry increments, broaden deferred cleanup decisions, and update focused tests.

## Best Possible Solution

Rebase or replace the branch on current main, keep a typed blocked-by-descendants signal through the completion hard-expiry window, reuse normal finalize-error recovery, and add focused regression coverage plus redacted runtime proof.

## Reproduction Assessment

Yes. Current main source is enough to reproduce the race conceptually: the inner descendant gate returns `false`, then finalize re-counts descendants and can charge retry budget if the count has flipped to zero.

## Solution Assessment

No. The sentinel direction is reasonable, but this branch uses the five-minute expiry for completion-message pending-descendant deferral and swallows finalize failures instead of using the existing cleanup recovery path.
