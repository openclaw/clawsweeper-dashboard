## Summary

Keep open: the reported subagent announce race is still present on current main and in the latest release, but this PR still has correctness blockers and does not provide sufficient after-fix real behavior proof for an external contribution.

## What This Changes

The PR changes subagent announce cleanup so descendant-blocked announces return a sentinel and defer without consuming announce retry budget.

## Best Possible Solution

Preserve a typed blocked-by-descendants outcome through cleanup, defer without consuming retries until the correct expiry window, retain normal finalization failure recovery, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can return false from the inner descendant gate and then count zero descendants in the later cleanup decision, causing retry accounting instead of a pending-descendant defer.

## Solution Assessment

No. The sentinel approach is reasonable, but this PR is not the best final fix until the pending-descendant branch uses the completion hard-expiry window and preserves normal finalize-error recovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor completion hard expiry for pending descendants:** `src/agents/subagent-registry-lifecycle.ts:275`
  - body: The pending-descendants branch defers only while `endedAgo <= ANNOUNCE_EXPIRY_MS`. For completion-message runs blocked by descendants after 5 minutes but before the 30-minute hard expiry, this falls back to the old recheck path and can consume retry budget without a delivery attempt; use the completion hard-expiry window for completion entries.
  - confidence: 0.89
- **[P2] Restore cleanup recovery for pending-descendant failures:** `src/agents/subagent-registry-lifecycle.ts:415-417`
  - body: The new `didAnnounce === -1` path catches `finalizeSubagentCleanup` with an empty handler. If finalization rejects after `beginSubagentCleanup` set `cleanupHandled`, the run can remain handled but incomplete; route this through the same warning/reset logic as normal finalize failures.
  - confidence: 0.82
