## Summary

Current `main` still has the underlying two-check cleanup shape, so this PR is not obsolete; however, the latest diff still has lifecycle edge cases that should block merge.

## What This Changes

The PR widens `runSubagentAnnounceFlow` to return `-1` for descendant-blocked announces, routes that sentinel through lifecycle cleanup to defer retries, broadens cleanup decisions for active descendants, and updates related tests.

## Best Possible Solution

Preserve a typed descendant-blocked lifecycle outcome through cleanup with the completion-message hard-expiry window, normal finalize-error recovery, focused regression coverage, and an active-version changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: make the inner announce descendant gate return blocked while a later cleanup recheck sees zero descendants, and current main can count that as an announce failure retry instead of a pending-descendant defer.

## Solution Assessment

No. The sentinel approach is a reasonable direction, but this PR is not the best final fix until the pending-descendant branch uses the correct completion hard-expiry window, preserves finalize-error recovery, and adds the required changelog entry.
