## Summary

Keep open: current main still has a source-reproducible descendant-gated announce retry race, and this PR targets the right surface, but the current head has blocking lifecycle bugs and lacks inspectable after-fix real behavior proof.

## What This Changes

The branch changes subagent announce cleanup to return a pending-descendants sentinel, defer cleanup without consuming announce retries, broaden descendant deferral, and add focused tests.

## Best Possible Solution

Land a refreshed patch that preserves the typed pending-descendants outcome through the correct hard-expiry window, uses normal finalize-error recovery, and includes focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns false at the descendant gate, then finalization rechecks descendants and can count the later zero-descendant state as a retryable announce failure without any delivery attempt.

## Solution Assessment

No. The sentinel approach is reasonable, but this PR is not the best final fix until it honors the completion hard-expiry window, preserves finalize-error recovery, and provides after-fix real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the completion hard expiry for pending descendants:** `src/agents/subagent-registry-lifecycle.ts:275`
  - body: The pending-descendants branch only defers while `endedAgo <= ANNOUNCE_EXPIRY_MS`. For completion-message runs that remain descendant-blocked after 5 minutes but before `ANNOUNCE_COMPLETION_HARD_EXPIRY_MS`, this falls back to the old second count and can spend retry budget without a delivery attempt; keep the sentinel deferral until the completion hard-expiry window.
  - confidence: 0.89
- **[P2] Preserve cleanup recovery on pending-descendant finalize failures:** `src/agents/subagent-registry-lifecycle.ts:415-417`
  - body: This catch swallows `finalizeSubagentCleanup` errors, unlike the normal finalize path that logs and resets `cleanupHandled`. If finalization rejects after `beginSubagentCleanup` marked the run handled, later resumes skip it and cleanup can stall; route this through the same recovery path.
  - confidence: 0.84
