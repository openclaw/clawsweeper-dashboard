## Summary

Current main still has the reported false-as-failure boundary, so this PR addresses a real source-reproducible bug. The patch is not merge-ready because the latest head still has two lifecycle correctness blockers and lacks observable real behavior proof.

## What This Changes

The PR changes subagent announce cleanup so a descendant-gated announce returns a typed `-1` sentinel, defers cleanup without incrementing announce retries, broadens descendant deferral, and adds focused cleanup/announce tests.

## Best Possible Solution

Rebase or replace the branch on current main, keep a typed blocked-by-descendants outcome through the completion hard-expiry window, preserve normal finalize-error recovery, and add focused regression coverage plus redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can return `false` from the inner descendant gate, then re-count zero descendants in cleanup and charge the announce retry budget without a real delivery attempt.

## Solution Assessment

No. The sentinel direction is reasonable, but this PR is not the best final fix until the pending-descendant branch honors completion hard-expiry semantics and reuses normal finalize failure recovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor completion hard expiry for pending descendants:** `src/agents/subagent-registry-lifecycle.ts:275-276`
  - body: For completion-message runs, this branch stops trusting the pending-descendants signal after `ANNOUNCE_EXPIRY_MS` even though the cleanup contract allows descendant deferral until `ANNOUNCE_COMPLETION_HARD_EXPIRY_MS`. From five to thirty minutes it can fall through to the old second-count path, race to zero descendants, and consume retries without a real delivery attempt.
  - confidence: 0.89
- **[P2] Restore finalize-error recovery for pending descendants:** `src/agents/subagent-registry-lifecycle.ts:415-417`
  - body: The new `didAnnounce === -1` path swallows `finalizeSubagentCleanup` errors with an empty catch. If finalization rejects after `beginSubagentCleanup` set `cleanupHandled`, later resumes can skip cleanup while the run remains handled but incomplete; reuse the normal warning/reset handler.
  - confidence: 0.82
