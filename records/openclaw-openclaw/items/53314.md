## Summary

Keep open: current main still has the source-reproducible retry-budget race, but this PR is not merge-ready because the head is conflicting, lacks real behavior proof, and still has lifecycle correctness blockers.

## What This Changes

The PR changes subagent announce cleanup to return a typed `-1` when descendant gating blocks delivery, defer cleanup without retry increments, broaden descendant deferral, and add focused tests.

## Best Possible Solution

Rebase or replace the branch on current main, preserve the typed blocked-by-descendants signal through the completion hard-expiry window, reuse normal finalize failure recovery, and add focused regression coverage plus redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main returns `false` from the descendant gate and later re-counts active descendants in finalize; if the count flips to zero between those checks, it charges retry budget without a real delivery attempt.

## Solution Assessment

No. The sentinel direction is reasonable, but this branch stops honoring completion hard-expiry semantics after five minutes and swallows finalize errors instead of using the normal recovery path.

## Review Findings

Overall confidence: 0.86
