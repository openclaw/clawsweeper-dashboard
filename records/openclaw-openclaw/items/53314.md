## Summary

Keep open. Current main still has the source-reproducible descendant-gate retry race, but this PR is not merge-ready because the pending-descendant branch uses the wrong expiry window, swallows finalize failures, and does not include inspectable after-fix real behavior proof.

## What This Changes

The PR changes subagent announce cleanup so descendant-blocked announces return a sentinel, defer without consuming retry budget, broaden descendant deferral, and add focused tests.

## Best Possible Solution

Land a refreshed patch that preserves a typed descendant-blocked cleanup outcome through the completion hard-expiry window, uses normal finalize-error recovery, and includes focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns false at the descendant gate, then finalization rechecks descendants and can count the later zero-descendant state as a retryable announce failure without any delivery attempt.

## Solution Assessment

No. The sentinel direction is reasonable, but this PR is not the best final fix until it honors the completion hard-expiry window, preserves finalize-error recovery, and provides after-fix real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the completion hard expiry for pending descendants:** `src/agents/subagent-registry-lifecycle.ts:275`
  - body: The pending-descendants fast path defers only while `endedAgo <= ANNOUNCE_EXPIRY_MS`. For completion-message runs blocked by descendants after 5 minutes but before the 30-minute hard expiry, this falls back to the old recheck path and can consume retry budget without a delivery attempt; use the completion hard-expiry window for completion-message entries.
  - confidence: 0.89
- **[P2] Preserve cleanup retry state on finalize failures:** `src/agents/subagent-registry-lifecycle.ts:415-417`
  - body: The new `didAnnounce === -1` branch catches `finalizeSubagentCleanup` with an empty handler. If finalization rejects after `beginSubagentCleanup` set `cleanupHandled`, the run can remain handled but incomplete and later resumes will skip it; route this through the same warning/reset logic as the normal finalize path.
  - confidence: 0.84
