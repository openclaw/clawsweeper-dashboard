## Summary

Keep open. Current main still routes an explicit directory lookup for an installed channel plugin without a directory adapter into the install prompt path, so the reported Weixin reinstall prompt remains source-consistent and unfixed.

## What This Changes

The issue asks `openclaw directory peers list --channel openclaw-weixin` to use the installed plugin or report unsupported instead of prompting to reinstall it.

## Best Possible Solution

Keep the fix channel-agnostic: distinguish installed-but-directory-unsupported from missing/installable, report unsupported for the directory command, and leave any Weixin directory implementation to the external plugin.

## Reproduction Assessment

Yes. A focused unit reproduction can mock a catalog-backed channel whose scoped plugin resolves but lacks `directory`, then assert the explicit directory command or resolver does not call the installer; current main would fall through to `ensureChannelSetupPluginInstalled`.

## Solution Assessment

Yes. The narrowest maintainable solution is a resolver or directory-CLI guard for unsupported capability, not a Weixin special case or a new core WeChat directory implementation.
