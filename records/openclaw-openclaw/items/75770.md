## Summary

Keep open. Current main still routes an explicit directory channel through the installable-channel resolver with a directory-capability predicate, and the resolver still falls through to the installer when an installed/scoped channel plugin exists but lacks that capability. The existing ClawSweeper comment remains consistent with the checked-out code.

## What This Changes

The issue requests that `openclaw directory peers list --channel openclaw-weixin` recognize an already installed/running Weixin plugin and report unsupported directory capability instead of prompting to reinstall it.

## Best Possible Solution

Keep the behavior channel-agnostic: when the requested channel resolves to a trusted installed plugin but that plugin lacks the requested directory adapter, return the installed plugin and let the directory CLI report unsupported directory capability. Only genuinely missing installable channels should enter the installer prompt path.

## Reproduction Assessment

Yes. A focused unit reproduction can mock an explicit catalog-backed channel whose loaded/scoped plugin lacks `directory` and assert the resolver or directory CLI does not call `ensureChannelSetupPluginInstalled`; the current source path would call it.

## Solution Assessment

Yes. The narrow maintainable fix is a resolver or directory-CLI guard that distinguishes missing plugin from installed-but-unsupported capability, without special-casing `openclaw-weixin` or adding WeChat directory data to core.
