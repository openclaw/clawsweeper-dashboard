## Summary

Keep open. Current main still has the same resolver behavior reported against v2026.4.29: the directory CLI only accepts an explicit channel when the loaded plugin already exposes a directory adapter, and otherwise falls through to the install prompt path. That makes this a valid OpenClaw CLI/plugin-resolution bug rather than a stale support report or a Weixin-only plugin request.

## What This Changes

The issue asks OpenClaw to stop prompting to reinstall `openclaw-weixin` when `openclaw directory peers list --channel openclaw-weixin` is run against a plugin that is already installed, enabled, and running.

## Best Possible Solution

Keep the issue open and fix the core resolver boundary so explicit directory commands can distinguish “channel plugin is installed but this directory operation is unsupported” from “channel plugin is missing and may be installed.” The end state should return a clear unsupported/empty directory result for installed channels without a directory adapter while preserving install prompts for truly absent optional plugins.

## Reproduction Assessment

Yes. The issue has concrete release steps, and current source gives a high-confidence synthetic repro: model a catalog-backed explicit channel whose plugin is installed/resolved but lacks `directory`, then verify the command does not call the install prompt path.

## Solution Assessment

Yes, a narrow OpenClaw core fix is the best direction. The problem is in CLI install-resolution semantics, not in the external Weixin runtime; a small resolver/CLI change plus focused tests should be enough.
