## Summary

Keep open. Current main still routes explicit directory lookups for an installed catalog-backed channel plugin without a `directory` adapter into the install prompt path, so the reported Weixin reinstall prompt remains source-consistent and unfixed.

## What This Changes

The issue asks OpenClaw to stop prompting to install `openclaw-weixin` during `directory peers list` when that plugin is already installed and running but does not expose directory support.

## Best Possible Solution

Keep the fix channel-agnostic: distinguish installed-but-directory-unsupported from missing/installable, report unsupported for the directory command, and leave any Weixin directory implementation to the external plugin.

## Reproduction Assessment

Yes. A focused unit reproduction can mock a catalog-backed installed/scoped channel plugin with no `directory` adapter and assert the explicit directory command does not call the installer; current main falls through to the installer when the support predicate fails.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the resolver or directory CLI guard, not in a Weixin special case or a new core WeChat directory implementation.
