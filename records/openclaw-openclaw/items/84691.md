## Summary

Keep this PR open: the diagnostic feature is useful and the supplied live-output proof is adequate, but the current diff can report diagnostics from a different session-store state than the maintenance decision it is explaining and also adds an unused diagnostics scan to text-mode maintenance.

## What This Changes

The PR adds stale-running task maintenance JSON diagnostics with decision/reason fields and tests for backing-session retention plus durable cron recovery filtering.

## Best Possible Solution

Keep the JSON diagnostic shape, but compute it lazily for JSON output and tie it to the same session-store snapshot or ordering as the maintenance decision it explains.

## Reproduction Assessment

Yes for the review findings: source inspection shows diagnostics are computed after session maintenance and before the JSON branch. A focused stale-session preview/apply test plus a text-mode maintenance invocation would reproduce the mismatch and extra scan.

## Solution Assessment

No as written; the diagnostic surface is reasonable, but it should be lazy for JSON output and aligned with the maintenance decision snapshot before merge.
