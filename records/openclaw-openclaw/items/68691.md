## Summary

Keep this open. Current main and the latest release still create normal sandbox containers without Docker `--init`, while the default sandbox image runs `sleep infinity` without an image-level init/reaper; the focused implementation and rollout PRs are still open.

## What This Changes

The issue reports Docker sandbox sessions accumulating orphaned zombie children under PID 1 until sandbox process-limit pressure becomes possible.

## Best Possible Solution

Land or consolidate https://github.com/openclaw/openclaw/pull/74083 and https://github.com/openclaw/openclaw/pull/76590 so sandbox containers get Docker `--init` or an equivalent PID-1 reaper, with a create-args hash/epoch bump that recreates existing long-lived containers.

## Reproduction Assessment

Yes, source-reproducible. The issue includes PPID=1 zombie snapshots and a follow-up identifying PID 1 as `sleep infinity`; current source still creates sandbox containers in that shape without `--init`, though I did not run a live stress workload.

## Solution Assessment

Yes. A PID-1 init/reaper plus a deliberate create-args hash or epoch bump is the narrow maintainable fix for children already orphaned to PID 1; userland killing can worsen zombie counts and the Chromium live-parent leak is a separate path.
