## Summary

Keep this open. Current main still creates Docker sandbox containers without a PID-1 reaper and still uses `sleep infinity` as the long-running sandbox command, matching the reporter’s later PPID=1 zombie observation; the focused fix remains the open, unmerged PR #74083.

## What This Changes

The issue reports Docker sandbox sessions accumulating orphaned PPID=1 zombie processes because the sandbox container’s PID 1 is not reaping children.

## Best Possible Solution

New Docker sandbox and sandbox-browser containers should run with a real PID-1 reaper, preferably Docker `--init` or an equivalent init wrapper, with regression coverage in `buildSandboxCreateArgs` and an upgrade/recreate path for already-running containers.

## Reproduction Assessment

Yes: the issue includes PPID=1 zombie snapshots and a follow-up identifying sandbox PID 1 as `sleep infinity`; current main still constructs that exact Docker container shape without `--init`. A live stress run remains useful for validating the fix, but the source-level reproduction path is high confidence.

## Solution Assessment

Yes: for orphaned children already adopted by PID 1 in Docker sandboxes, adding an init/reaper at container creation is the narrow maintainable fix. The safer complete version also handles existing-container recreation instead of relying only on future new containers.
