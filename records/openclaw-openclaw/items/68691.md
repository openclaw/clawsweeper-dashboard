## Summary

Keep this open. Current main and v2026.5.12 still create Docker sandbox containers without Docker `--init` and run the normal sandbox as `sleep infinity` under PID 1, while the focused implementation and rollout PRs remain open.

## What This Changes

The issue reports PPID=1 zombie processes accumulating in Docker sandbox containers because PID 1 is not reaping orphaned children.

## Best Possible Solution

Land or consolidate the focused Docker `--init` change with a create-args hash/epoch rollout so both new and existing long-lived sandbox containers get an init-backed PID 1.

## Reproduction Assessment

Yes, source-level. The issue includes concrete PPID=1 zombie snapshots and a follow-up identifying PID 1 as `sleep infinity`; current main still creates normal sandbox containers in that shape without Docker `--init`.

## Solution Assessment

Yes. A PID-1 init/reaper plus deliberate container recreation is the narrow maintainable fix for children already orphaned to PID 1; the open PRs cover those two pieces and should be reviewed, refreshed, landed, or consolidated.
