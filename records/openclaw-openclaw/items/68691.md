## Summary

Keep this open. Current `main` and the latest release still create normal Docker sandbox containers without Docker `--init` while running `sleep infinity` as PID 1, so the orphaned-zombie reaping gap remains active and the linked implementation PRs are still open.

## What This Changes

The issue asks for Docker sandbox containers to reap orphaned zombie processes adopted by PID 1 so they do not accumulate toward the sandbox PID limit.

## Best Possible Solution

Land a focused sandbox runtime change that adds Docker `--init` or an equivalent PID-1 reaper, paired with a create-args hash or epoch bump so existing long-lived sandbox containers are recreated.

## Reproduction Assessment

Yes. The issue includes concrete PPID=1 zombie snapshots and a follow-up identifying PID 1 as `sleep infinity`; source inspection confirms current `main` still creates long-lived Docker sandboxes in that shape without `--init`, though I did not run a live stress workload in this read-only pass.

## Solution Assessment

Yes. A PID-1 init/reaper plus a deliberate recreate rollout is the narrow maintainable fix for children already orphaned to PID 1; the linked open PRs cover those two pieces and should be reviewed, landed, or consolidated.
