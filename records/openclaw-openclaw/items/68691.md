The issue reports that Docker sandbox sessions can accumulate orphaned zombie processes under PID 1 until sandbox process limits risk breaking normal command execution.

## Best Possible Solution

Land or replace the linked work with one coordinated sandbox rollout that adds a PID 1 reaper for Docker sandboxes, bumps the create-args hash in the same rollout, updates focused tests, and includes redacted real Docker proof for fresh and existing long-lived containers.
