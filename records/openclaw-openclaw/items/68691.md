## Summary

Keep this open. Current main still creates Docker sandbox containers without a PID-1 reaper and still runs the normal sandbox as `sleep infinity`; the focused open fix path remains PR #74083, which is not merged yet.

## What This Changes

The issue reports Docker sandbox sessions accumulating orphaned PPID=1 zombie processes because the sandbox container’s PID 1 is not reaping children.

## Best Possible Solution

New Docker sandbox and sandbox-browser containers should run with Docker `--init` or an equivalent PID-1 reaper, with regression coverage and an intentional rollout path for existing containers.

## Reproduction Assessment

Yes. The issue includes PPID=1 zombie snapshots and a follow-up identifying sandbox PID 1 as `sleep infinity`; current main still constructs that container shape without `--init`.

## Solution Assessment

Yes for the direction: a PID-1 init/reaper is the narrow maintainable fix for orphaned children already adopted by PID 1. The complete fix also needs existing-container recreation behavior, which PR #74083 still needs to address before merge.
