## Summary

Keep this open. Current main and the latest release still create normal Docker sandbox containers without a PID-1 reaper, while the focused `--init` fix and the create-args hash rollout remain open in https://github.com/openclaw/openclaw/pull/74083 and https://github.com/openclaw/openclaw/pull/76590.

## What This Changes

Docker sandbox containers should use an init/PID-1 reaper so orphaned children are reaped before zombie counts can exhaust the sandbox PID budget.

## Best Possible Solution

Land a focused sandbox runtime fix that adds Docker `--init` or an equivalent PID-1 reaper, pairs it with a create-args hash/epoch rollout for existing containers, and keeps the separate browser live-parent zombie work scoped to its own issue.

## Reproduction Assessment

Yes. The report includes PPID=1 zombie snapshots and a follow-up identifying PID 1 as `sleep infinity`; current main still creates long-lived sandbox containers as `sleep infinity` without `--init`, though I did not run a live stress reproduction in this read-only pass.

## Solution Assessment

Yes for the requested direction. A PID-1 init/reaper plus an intentional recreate rollout is the narrow maintainable fix for orphaned children already adopted by PID 1; current main lacks both pieces, and the open fix PRs should be reviewed, landed, or consolidated.
