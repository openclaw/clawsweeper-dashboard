## What This Changes

The issue requests Docker sandbox containers to reap orphaned zombie children adopted by PID 1 so zombie accumulation cannot exhaust the sandbox process budget.

## Best Possible Solution

Land or replace the linked work with one coordinated sandbox rollout that adds Docker init/reaping, bumps the create-args hash in the same rollout, updates focused tests, and includes redacted real Docker proof for fresh and existing long-lived containers.

## Solution Assessment
