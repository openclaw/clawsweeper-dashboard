## Summary

Keep this open. Current main and the latest release still create normal Docker sandbox containers without Docker `--init` while running `sleep infinity` as PID 1, so the orphaned-zombie reaping gap remains active and the focused implementation and rollout PRs are still open.

## What This Changes

The issue requests that Docker sandbox containers reap orphaned zombie processes adopted by PID 1 so normal sandbox workloads do not accumulate defunct children and exhaust the process limit.

## Best Possible Solution

Land or consolidate the focused sandbox runtime work so new Docker sandboxes run with a PID-1 reaper and long-lived containers are deliberately recreated when create flags change.

## Reproduction Assessment

Yes. The issue includes concrete PPID=1 zombie snapshots and a follow-up identifying PID 1 as `sleep infinity`; source inspection confirms current main still creates long-lived Docker sandboxes in that shape without `--init`, though this pass did not rerun a live stress workload.

## Solution Assessment

Yes. A PID-1 init/reaper plus a deliberate recreate rollout is the narrow maintainable fix for children already orphaned to PID 1; the linked open PRs cover those two pieces and should be reviewed, landed, or consolidated.
