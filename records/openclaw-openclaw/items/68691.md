## Summary

Keep this open: current main and latest release still create Docker sandbox containers without Docker `--init`, run the normal sandbox command as `sleep infinity`, and lack a create-args epoch to roll existing long-lived containers forward; the focused companion PRs remain open and conflicting.

## What This Changes

The issue asks OpenClaw sandbox containers to reap orphaned zombie children adopted by PID 1 so they do not accumulate toward the sandbox PID limit.

## Best Possible Solution

Land or consolidate the Docker `--init` change with a create-args epoch bump so both new and existing long-lived sandbox containers get an init-backed PID 1, with redacted Docker proof of the recreate path.

## Reproduction Assessment

Yes, source-level: the issue includes concrete PPID=1 zombie snapshots, and current main still creates sandbox containers as `sleep infinity` without Docker `--init`. I did not run a live stress workload in this read-only review.

## Solution Assessment

Yes. Docker `--init` is the narrow mechanism for orphaned children already adopted by PID 1, and the create-args epoch is needed so existing long-lived containers are recreated rather than staying on the old non-init PID 1.
