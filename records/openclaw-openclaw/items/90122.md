## Summary

## What This Changes

The PR adds a `turnSucceeded` signal to Control UI tool message groups and uses it to avoid primary error styling for failed tools when the turn produced a clean assistant reply.

## Best Possible Solution

Land this UI-boundary fix after maintainer review, keeping Codex exec failure semantics unchanged and leaving direct-chat suppression to the separate linked PR.

## Solution Assessment

Yes. The Control UI projection/render boundary is the narrow owner for deciding whether a failed internal tool is terminal to the visible turn; changing Codex exec failure semantics or direct-chat dispatch would be broader or the wrong surface.
