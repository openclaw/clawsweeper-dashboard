## Summary

Keep open. The PR is a useful candidate fix for the linked Control UI/WebChat bug, but it is not merge-ready because the new turn-success annotation runs after chat search filtering can remove the user boundary that separates turns.

## What This Changes

The PR adds a Control UI chat `turnSucceeded` signal and uses it to render failed tool groups as neutral collapsed output after a successful assistant reply while keeping terminal tool failures prominent.

## Best Possible Solution

Preserve true turn boundaries when computing tool outcome, add focused search-filter regression coverage, then land the UI projection fix separately from the adjacent direct-chat delivery work.

## Reproduction Assessment

Yes. Current main and v2026.6.8 source still map failed tool cards directly to `Tool error`, and the PR regression is source-reproducible because filtering can remove the user group that the new backward pass needs as a turn reset.

## Solution Assessment

No, not as written. The Control UI projection/render boundary is the right layer, but the implementation needs to compute or preserve turn boundaries before search filtering can change the classification input.
