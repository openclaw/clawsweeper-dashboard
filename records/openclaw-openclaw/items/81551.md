## Summary

Keep this PR open for maintainer review: it is protected by the `maintainer` label and the patch has a concrete blocker in the new MCP binding logic. The branch adds the right kind of app-server MCP plumbing, but it can preserve a Codex thread that already has native MCP servers after a later turn disables or excludes MCP tools.

## What This Changes

The PR adds shared Codex MCP server normalization, feeds configured MCP servers into Codex app-server thread config, tracks an MCP fingerprint on app-server bindings, and adds focused tests for projection and invalidation.

## Best Possible Solution

Keep the shared MCP normalization path, but make the native MCP fingerprint represent the desired effective state, including disabled and allowlist-excluded states, so Codex starts or rotates to a no-MCP thread when MCP tools should be absent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start a Codex app-server thread with MCP servers, then run a later turn with `disableTools` or a `toolsAllow` value that excludes MCP; the PR marks MCP config as not evaluated and resumes the old binding instead of rotating away from the MCP-backed thread.

## Solution Assessment

No. The direction is maintainable, but the new fingerprint/evaluation contract must handle disabled or excluded MCP as an evaluated empty state; otherwise the fix leaves prior native MCP capability available after policy changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Rotate MCP threads when tools are disabled:** `src/agents/codex-mcp-config.ts:100-104`
  - body: `loadCodexBundleMcpThreadConfig` returns `evaluated: false` for `disableTools` or an allowlist that excludes MCP, and the lifecycle code only invalidates when evaluation is true. If a Codex thread was started with native MCP servers, the next turn with tools disabled/excluded resumes that same thread, so those MCP tools can remain available. Treat the disabled/excluded state as an evaluated empty MCP config and rotate/start a no-MCP thread instead.
  - confidence: 0.86
