## Summary

Keep open. Current main still has partial skill security coverage, but native ClawHub skill install and update download, extract, and copy skill archives without a local scan or policy gate. The AgentShield and ClawGuard comments are useful implementation context, and protected #9256 plus open #18196 cover adjacent security work, but this remains unresolved maintainer policy work rather than stale cleanup.

## What This Changes

Add built-in security scanning and policy enforcement to ClawHub skill install, update/load, and publish flows, with AgentShield suggested as one possible scanner backend.

## Best Possible Solution

Define a first-class local skill supply-chain policy for native ClawHub install/update: scan before copy or activation, enforce scan-on-update, surface provenance and results through CLI/Gateway/UI, provide an explicit operator override path, document the threat model, and keep the scanner backend pluggable enough for AgentShield, ClawGuard, or the built-in scanner.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by tracing `openclaw skills install` or Gateway `skills.install` with `source: "clawhub"` into `src/agents/skills-clawhub.ts`, where the archive is downloaded, extracted, and copied without `scanSkillInstallSource`; update uses the same forced install path.

## Solution Assessment

Unclear for an AgentShield-only integration. The narrower maintainable direction is a maintainer-approved core scan/policy seam for native ClawHub install/update, with a deliberate default scanner, override model, and provenance contract.
