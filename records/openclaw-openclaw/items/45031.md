## Summary

Keep open. Current main partially addresses install-time security scanning through the built-in scanner and `before_install` hook, and commenters linked useful external scanner implementations like AgentShield and ClawGuard, but native ClawHub skill install/update still bypasses the local scan path and the remaining default policy is a security/product decision.

## What This Changes

The issue requests built-in security scanning and policy coverage for ClawHub skill install, update, upload, and load paths.

## Best Possible Solution

Define a maintained install/update security policy path that covers native ClawHub skill installs and updates, keeps scanner-specific logic pluggable, and documents the OpenClaw-versus-ClawHub publish/upload boundary.

## Reproduction Assessment

Yes for the source-verifiable gap: native ClawHub install/update reaches installExtractedSkillRoot with scan: false, while gateway-backed dependency installs use the scan runtime. This is a feature/security coverage gap rather than a broken documented behavior.

## Solution Assessment

Unclear. A hard-coded AgentShield integration is probably not the narrowest core solution; a stable scanner/policy hook contract with ClawHub or plugin-distributed scanner implementations is safer.
