## Label Justifications

- `P1`: The issue concerns a live untrusted-skill install/update security-policy gap affecting users who install third-party ClawHub skills.
- `impact:security`: The request covers supply-chain scanning, malicious skill detection, install blocking, provenance, and policy enforcement for untrusted skills.

## Summary

Keep open: current main and the latest release have scanner and `before_install` infrastructure for some skill install paths, but native ClawHub skill install/update still opts out. AgentShield and ClawGuard are useful external scanner context from the discussion, but the remaining default policy, update/load behavior, override model, and ClawHub publish boundary need maintainer security/product direction.

## What This Changes

The issue requests built-in security scanning for skill install, update, load, and ClawHub publish flows, with AgentShield suggested as one possible scanner.

## Best Possible Solution

Define a maintained, pluggable install/update security policy for native ClawHub skills that composes with the existing scanner and `before_install` hook, with explicit defaults, override behavior, provenance expectations, and docs separating local installs from ClawHub publish enforcement.

## Reproduction Assessment

Yes, as a source-verifiable coverage gap: native ClawHub install/update passes `scan: false`, while the shared archive installer only runs the scanner when `params.scan` is truthy. This is a security feature gap rather than a broken documented command with a simple live failure repro.

## Solution Assessment

Unclear. Built-in scanning is a valid direction, but hard-coding one community scanner into core is likely less maintainable than a generic scanner/policy contract with deliberate defaults and room for ClawHub-published scanners.
