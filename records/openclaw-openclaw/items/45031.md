## Summary

Keep open: current main still has a native ClawHub skill install/update path that bypasses the local install scanner and `before_install` policy hook, so the requested default security coverage is not implemented.

## What This Changes

The issue requests built-in security scanning for skill install, update, load, and ClawHub publish flows, with AgentShield suggested as one possible scanner.

## Best Possible Solution

Define a maintained, pluggable install/update security policy that covers native ClawHub skill installs and updates, composes with the existing built-in scanner and `before_install` hook, and documents which ClawHub service responsibilities live outside this repository.

## Reproduction Assessment

Yes, for a source-verifiable coverage gap: native ClawHub install/update reaches `installExtractedSkillRoot` with `scan: false`, while gateway dependency and upload installs can enter the scanner path. This is a security feature gap, not a broken documented runtime behavior with a simple failing command.

## Solution Assessment

Unclear. Hard-coding AgentShield into core is probably not the narrowest maintainable answer; a maintained scanner/policy hook contract with chosen defaults and pluggable scanner implementations is safer.
