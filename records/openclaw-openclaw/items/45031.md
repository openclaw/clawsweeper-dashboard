## Summary

Keep open. Current main and the latest shipped release still only partially cover the requested protection: gateway-backed skill dependency installs use the built-in scanner and `before_install` hook, but native ClawHub skill install/update still bypasses that local scan path. The discussion also links useful external scanner implementations, including AgentShield and ClawGuard, and a related duplicate was explicitly routed back to this issue as the canonical built-in skill-install scanning request.

## What This Changes

The issue requests built-in security scanning for skill install, update, load, and ClawHub publish/upload flows, with AgentShield suggested as one possible scanner.

## Best Possible Solution

Define a maintained, pluggable install/update security policy path that covers OpenClaw-native ClawHub skill installs and updates, composes with the existing scanner and before_install hook contract, and documents which ClawHub publish/upload responsibilities live outside this repository.

## Reproduction Assessment

Yes for the source-verifiable coverage gap: native ClawHub install/update reaches installExtractedSkillRoot with scan: false, while gateway-backed skill dependency installs use scanSkillInstallSource and before_install. This is a security feature coverage gap, not a broken documented behavior with a runtime repro script.

## Solution Assessment

Unclear. Hard-coding AgentShield into core is probably not the narrowest maintainable solution; a scanner/policy hook contract with maintainer-chosen defaults and pluggable scanner implementations is safer.
