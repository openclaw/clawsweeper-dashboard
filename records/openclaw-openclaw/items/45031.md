## Summary

Current main provides the generic fail-closed enforcement boundary for skill installation and updates, while OpenClaw’s documented architecture leaves scanner engines, registry scan state, provenance, and publisher review to ClawHub and maintained plugins. The discussion already links two usable scanner implementations, so a vendor-specific built-in scanner is no longer necessary in core.

## What This Changes

Add automatic security scanning and policy enforcement when skills are installed, updated, loaded, or published.

## Best Possible Solution

Keep the generic core install-policy contract, publish and maintain scanner engines such as AgentShield or ClawGuard through ClawHub, and evolve ClawHub-owned scanning and provenance independently of OpenClaw core.

## Reproduction Assessment

Not applicable; this requests a new default security capability rather than reporting broken established behavior, and the current behavior is directly documented and source-verifiable.

## Solution Assessment

No; embedding a particular scanner in core is not the narrowest maintainable solution because current main already exposes generic blocking policy surfaces and ClawHub hosts independently maintained scanner implementations.
