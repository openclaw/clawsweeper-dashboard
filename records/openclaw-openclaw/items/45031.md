## Summary

Keep open: current main has scanner and before_install policy infrastructure, but native ClawHub skill install/update still opts out of that path, and the requested default scanner/policy behavior needs maintainer security and product direction.

## What This Changes

Request built-in security scanning and policy enforcement for skills installed, updated, loaded, or published through ClawHub workflows.

## Best Possible Solution

Define a maintained, pluggable install/update security policy for native ClawHub skills that composes with the existing scanner and before_install hook, while documenting which publish/upload checks remain ClawHub service ownership.

## Reproduction Assessment

Yes, as a source-verifiable coverage gap: native ClawHub install/update reaches installExtractedSkillRoot with scan: false, while the archive installer only scans when params.scan is truthy. This is a security feature gap rather than a broken documented command with a simple live failure repro.

## Solution Assessment

Unclear. Hard-coding one community scanner into core is probably not the narrowest maintainable solution; a generic scanner/policy contract with deliberate defaults and room for ClawHub-published scanners is safer.
