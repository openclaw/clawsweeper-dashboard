## Summary

Keep open. Current main still has no local scan or policy gate in the native ClawHub skill install/update path, while the existing built-in scanner only covers Gateway-backed installer-metadata skill installs and plugin install/update flows. AgentShield and ClawGuard are useful implementation context from the discussion, and protected #9256 plus open #18196 cover adjacent security work, but neither has landed a current-main fix for this request.

## What This Changes

Request built-in security scanning and policy enforcement for ClawHub/manual skill installs, first load or update, and ClawHub publish, with AgentShield suggested as one scanner backend.

## Best Possible Solution

Define a first-class local skill supply-chain policy for native ClawHub install/update: scan before copy or activation, enforce scan-on-update, expose results and explicit overrides through CLI/Gateway/UI, document the threat model, and keep the scanner backend pluggable enough for AgentShield, ClawGuard, or the built-in scanner while coordinating publish-side checks with ClawHub.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by tracing openclaw skills install or Gateway skills.install with source: "clawhub" into src/agents/skills-clawhub.ts, where the archive is downloaded, extracted, and copied without scanSkillInstallSource; update uses the same forced install path.

## Solution Assessment

Unclear for an AgentShield-only integration. The narrower maintainable path is a maintainer-approved core scan/policy seam for native ClawHub install/update, with deliberate defaults, override behavior, provenance, and scanner backend choice.
