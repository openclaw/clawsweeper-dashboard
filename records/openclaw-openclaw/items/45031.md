## Summary

Keep open: current main and the latest shipped release still leave native ClawHub skill install/update outside the local scanner and `before_install` policy path, while the requested default scanner/policy behavior needs security and product decisions. The AgentShield and ClawGuard comments are useful ecosystem context, but an optional ClawHub-published scanner does not replace a maintained OpenClaw install/update policy boundary.

## What This Changes

Add built-in/default security scanning and policy enforcement for skill installation and update flows, especially ClawHub installs and uploads.

## Best Possible Solution

Define a maintained, pluggable install/update security policy that covers native ClawHub installs and updates, composes with the existing built-in scanner and `before_install` hook, and documents which upload/publish checks belong to ClawHub service ownership.

## Reproduction Assessment

Yes, as a source-verifiable coverage gap: native ClawHub install/update reaches `installExtractedSkillRoot` with `scan: false`, while dependency and upload installs can enter the scanner-backed hook path. This is a security feature gap rather than a broken documented runtime command with a simple live repro.

## Solution Assessment

Unclear. Hard-coding AgentShield into core is likely not the narrowest maintainable answer; a generic scanner/policy contract with deliberate defaults and room for ClawHub-published scanner implementations is safer.
