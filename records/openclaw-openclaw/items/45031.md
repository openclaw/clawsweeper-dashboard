## Summary

Keep open. Current main still only partially covers the requested protection: gateway-backed skill dependency installs and upload installs can enter the built-in scanner and `before_install` hook path, but native ClawHub skill install/update still downloads an archive and calls the archive installer with scanning disabled. The linked AgentShield and ClawGuard projects are useful prior art, but the remaining work is a security/product decision about OpenClaw's default install/update policy rather than a simple stale cleanup.

## What This Changes

Request built-in security scanning and policy enforcement for skill installation flows, with AgentShield proposed as one possible scanner implementation.

## Best Possible Solution

Define a maintained, pluggable install/update security policy path that covers native ClawHub skill installs and updates, composes with the existing built-in scanner and `before_install` hook contract, and documents which ClawHub publish/upload responsibilities live outside this repository.

## Reproduction Assessment

Yes for the source-verifiable coverage gap: native ClawHub install/update reaches `installExtractedSkillRoot` with `scan: false`, while gateway dependency and upload paths can use the scanner. This is a security feature coverage gap, not a broken documented behavior with a simple runtime repro script.

## Solution Assessment

Unclear. Hard-coding AgentShield into core is probably not the narrowest maintainable solution; a scanner/policy hook contract with maintainer-chosen defaults and pluggable scanner implementations is safer.
