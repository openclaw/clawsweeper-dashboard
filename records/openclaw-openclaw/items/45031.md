## Label Justifications

- `P1`: The issue concerns a live untrusted-skill install/update security-policy gap affecting users who install third-party ClawHub skills.
- `impact:security`: The request covers supply-chain scanning, malicious skill detection, install blocking, provenance, and policy enforcement for untrusted skills.

## Summary

Keep open: current main and v2026.5.22 still let native ClawHub skill install/update bypass the local skill scan and `before_install` policy path, while AgentShield and ClawGuard are useful scanner options rather than an OpenClaw default policy.

## What This Changes

The issue requests built-in security scanning for skill install/update/load/publish flows, including possible AgentShield integration and stronger provenance or per-agent policy controls.

## Best Possible Solution

Define a maintained, pluggable install/update security policy for native ClawHub skills that composes with the existing scanner and `before_install` hook, with explicit defaults, override behavior, provenance expectations, and docs separating OpenClaw installs from ClawHub publish enforcement.

## Reproduction Assessment

Yes. This is source-reproducible as a feature gap: native ClawHub install/update passes `scan: false`, while the shared archive installer only runs the scanner when `params.scan` is provided; v2026.5.22 has the same behavior.

## Solution Assessment

Unclear. Built-in scanning is a valid direction, but hard-coding AgentShield into core is less maintainable than a generic scanner/policy contract using the existing scanner and `before_install` path with deliberate defaults.
