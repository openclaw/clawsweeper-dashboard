## Summary

Keep open. Current main only partially covers the requested security scanning: gateway-backed skill dependency installs and uploaded private archives can run the built-in scan/hook path, but native ClawHub skill install/update still bypasses it, and the remaining built-in/default scanner policy overlaps open security lifecycle trackers that need maintainer product/security direction.

## What This Changes

Add built-in security scanning around skill install, update, load, and publish flows, with AgentShield suggested as one scanner option.

## Best Possible Solution

Define a stable install/update security policy path that covers native ClawHub skill installs, keeps scanner-specific integrations in plugins or ClawHub, and documents the OpenClaw-versus-ClawHub upload/publish boundary.

## Reproduction Assessment

Yes for the source-verifiable gap: native ClawHub install/update reaches installExtractedSkillRoot with scan:false, while gateway-backed dependency installs already call the scan runtime. This is a security feature gap rather than a broken documented behavior.

## Solution Assessment

Unclear. A hard-coded AgentShield integration is probably not the narrowest OpenClaw-core solution; the safer direction is a maintained policy hook/scanner contract plus ClawHub or plugin-distributed scanner implementations.
