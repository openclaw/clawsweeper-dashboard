## Label Justifications

- `P1`: The issue concerns a live untrusted-skill install/update security-policy gap affecting users who install third-party skills.
- `impact:security`: The request covers supply-chain scanning, malicious skill detection, install blocking, provenance, and security policy enforcement.

## Summary

Keep open: current main has a built-in scanner and `before_install` policy hook for some install flows, but native ClawHub skill install/update still opts out of that path. The discussion also points to AgentShield and ClawGuard as existing external scanners, but the remaining default scan policy and ClawHub ownership boundary need maintainer security/product direction.

## What This Changes

The issue asks OpenClaw to add built-in security scanning for skill installation, loading, and ClawHub publishing, with comments extending the request to update scans, provenance tracking, and per-agent policy.

## Best Possible Solution

Define a maintained, pluggable install/update security policy for native ClawHub skills that composes with the existing scanner and `before_install` hook, with explicit defaults, override behavior, provenance expectations, and docs separating local installs from ClawHub publish enforcement.

## Reproduction Assessment

Yes, as a source-verifiable coverage gap: native ClawHub install/update passes `scan: false`, while the shared archive installer only runs the scanner when `params.scan` is truthy. This is a security feature gap rather than a broken documented command with a simple live failure repro.

## Solution Assessment

Unclear. Hard-coding one community scanner into core is probably not the narrowest maintainable solution; a generic scanner/policy contract with deliberate defaults and room for ClawHub-published scanners is safer.
