## Summary

Keep open: current main still has a native ClawHub skill install/update path that bypasses the local skill scanner and `before_install` policy hook, while the requested default security coverage remains a product/security decision. The AgentShield and ClawGuard comments are useful external implementations, but they do not replace an OpenClaw install/update policy boundary.

## What This Changes

Request default security scanning and policy coverage for skill install/update, load-time, and ClawHub publish flows, with AgentShield suggested as one possible scanner.

## Best Possible Solution

Define a maintained, pluggable install/update security policy that covers native ClawHub installs and updates, composes with the existing scanner and `before_install` hook, and documents which ClawHub service responsibilities live outside this repository.

## Reproduction Assessment

Yes, as a source-verifiable coverage gap: native ClawHub install/update reaches `installExtractedSkillRoot` with `scan: false`, while gateway dependency and upload installs can enter the scanner path. This is a security feature gap rather than a broken documented runtime command with a simple live repro.

## Solution Assessment

Unclear. Hard-coding AgentShield into core is likely not the narrowest maintainable answer; a generic scanner/policy hook contract with deliberate defaults and pluggable scanner implementations is safer.
