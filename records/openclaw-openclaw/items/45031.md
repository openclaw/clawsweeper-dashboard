## Summary

Keep open. Current main has partial skill security coverage, but the native ClawHub skill install and update paths still download, extract, and copy skill folders without a local scan or policy gate. The AgentShield and ClawGuard discussion is useful implementation context, but the remaining decision is security-sensitive core policy work rather than stale cleanup or a ClawHub-only plugin move.

## What This Changes

Request to add built-in security scanning for ClawHub/manual skill installation, skill loading or updates, and ClawHub publish flows, with AgentShield suggested as one scanner backend.

## Best Possible Solution

Define a first-class OpenClaw skill supply-chain policy for native ClawHub install/update: local scan before copy or activation, scan-on-update, critical-finding blocking with explicit operator override, provenance/version evidence, CLI/Gateway/UI result surfacing, docs/threat-model alignment, and a backend-generic extension point so AgentShield, ClawGuard, or the built-in scanner can be selected deliberately.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by tracing openclaw skills install or Gateway skills.install with source: "clawhub" into src/agents/skills-clawhub.ts; update uses the same forced install path. No live ClawHub registry call is needed to verify the missing local scan gate.

## Solution Assessment

Unclear for an AgentShield-only integration. The narrower maintainable direction is a core scan/policy seam for native ClawHub install/update first, with a maintainer-approved default scanner and explicit override/provenance behavior layered on top.
