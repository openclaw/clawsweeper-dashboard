## Summary

Keep open: the PR has useful proof and coverage, but it is not merge-ready because it changes shipped thread-session reset defaults and does not safely handle existing Discord archived-thread close markers created by current releases.

## What This Changes

This PR changes unconfigured thread-scoped sessions from daily rollover to idle/no-expiry, adds a durable `sessionClosedAt` marker for archived Discord thread rollover, and updates docs/tests/callers to propagate that marker.

## Best Possible Solution

Preserve shipped daily rollover unless maintainers sponsor the new default; if they do, add an upgrade-safe path for old Discord `updatedAt: 0` close markers and release-note upgrade context.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.9` use daily fallback and Discord `updatedAt: 0` close markers, while PR head flips unconfigured thread sessions to idle/no-expiry and only forces closure from `sessionClosedAt`.

## Solution Assessment

No, not as-is. The maintainable path is either keep the shipped default as the implicit behavior or pair maintainer-approved default retention with an upgrade-safe translation for existing closed-thread rows.
