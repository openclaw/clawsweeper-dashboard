## What This Changes

The branch preserves the shipped daily reset default, documents explicit thread continuity through existing `session.resetByType.thread` idle/no-expiry config, and adds `sessionClosedAt` handling so closed Discord thread sessions roll over across session entry points.

## Best Possible Solution

Land this after maintainer acceptance of the additive `sessionClosedAt` state and green merge checks; keep thread continuity explicit through existing `session.resetByType.thread` instead of changing the shipped default.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.10` mark archived Discord thread sessions by zeroing `updatedAt`, while idle/no-expiry policies have no idle deadline, so those rows can remain reusable without an explicit closed marker.

## Solution Assessment

Yes. A shared `sessionClosedAt` freshness marker is narrower and more maintainable than overloading `updatedAt = 0`, while preserving the shipped daily default avoids the earlier compatibility break; cleanup/GC for orphan rows remains a separate product issue.
