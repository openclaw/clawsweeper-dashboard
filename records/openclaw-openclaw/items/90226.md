## Best Possible Solution

Choose the thread retention default intentionally: either sponsor idle/no-expiry as the new upgrade default, or preserve daily rollover as the implicit default and make continuity an explicit `session.resetByType.thread` configuration path.

## Reproduction Assessment

## Solution Assessment

No, not merge-ready as-is unless maintainers approve the new default. The safer alternative is preserving the shipped daily implicit default and making thread continuity an explicit `session.resetByType.thread` opt-in.
