## Summary

Keep open. The implementation is coherent and well-proven, but it intentionally changes the shipped implicit reset default for existing unconfigured thread sessions, so this needs maintainer product and upgrade approval before merge.

## What This Changes

The PR changes implicit thread-scoped session reset behavior from daily rollover to idle/no-expiry by default, adds a `sessionClosedAt` rollover marker path for externally closed threads, and updates related docs and tests.

## Best Possible Solution

Choose the thread retention default intentionally: either sponsor idle/no-expiry as the new upgrade default, or preserve daily rollover as the implicit default and make continuity an explicit `session.resetByType.thread` configuration path.

## Reproduction Assessment

Yes. Current main and v2026.6.9 source use daily fallback for omitted reset config, while PR head selects idle/no-expiry for unconfigured `resetType: "thread"`; Mantis also shows the visible Telegram before/after behavior.

## Solution Assessment

No, not merge-ready as-is unless maintainers approve the new default. The safer alternative is preserving the shipped daily implicit default and making thread continuity an explicit `session.resetByType.thread` opt-in.
