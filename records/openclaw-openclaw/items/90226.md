The PR changes unconfigured thread-scoped sessions to default to idle/no-expiry reset, updates session docs and regression tests, and includes small plugin/audit test compatibility cleanups.

Maintainers should either explicitly approve idle/no-expiry as the new unconfigured thread default with upgrade/release-note context, or rework the PR so thread persistence remains opt-in through existing reset configuration.
