## Summary

## What This Changes

The PR preserves the shipped daily reset default for unconfigured sessions, documents explicit thread continuity via existing `resetByType.thread` config, adds a durable `sessionClosedAt` marker for externally closed threads, and threads that marker through reply, gateway, command-session, plugin-helper, Discord, docs, and tests.

## Best Possible Solution

Land only after maintainer acceptance that `sessionClosedAt` is the canonical core-owned closed-thread lifecycle marker and that closed archived threads should roll over even under explicit idle/no-expiry continuity.

## Reproduction Assessment

Yes, at source level. Current main marks archived Discord thread sessions only by zeroing `updatedAt`, while an explicit idle/no-expiry thread policy has no idle deadline, so the next message can still reuse the closed thread session.

## Solution Assessment

Yes, with maintainer acceptance of the persisted state surface. A durable lifecycle marker read by shared freshness helpers is a cleaner fix than overloading `updatedAt`, and the branch preserves the shipped daily default while using the existing `resetByType.thread` config for continuity.
