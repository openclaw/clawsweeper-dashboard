## Summary

Keep open: the direction is useful, but the patch is not merge-ready because sibling gateway and command session paths now treat `sessionClosedAt` as stale without clearing it after minting a new session, which can cause repeated session rotation.

## What This Changes

The PR preserves the shipped daily reset default for unconfigured thread sessions, documents explicit thread continuity, and adds a durable `sessionClosedAt` marker so closed Discord thread sessions roll over.

## Best Possible Solution

Keep the default-preserving direction, but clear `sessionClosedAt` in every rollover writer that honors it and add sibling-path regressions before merge.

## Reproduction Assessment

## Solution Assessment

No, not yet. The auto-reply path is the right fix shape, but the sibling gateway and command-session paths need the same clear-on-rollover invariant before this is the best solution.
