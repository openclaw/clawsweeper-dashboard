## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/88304. On current `origin/main`, `extensions/browser/src/browser/chrome-mcp.ts` stores one `pendingSessions` promise per cache key and creates it with the first caller's `AbortSignal`, so aborting that caller can reject and tear down the shared attach for unrelated waiters. Replace the pending-session value with a small state object that owns an internal `AbortController`, shared attach promise, waiter registration/count, and pending-entry eviction. Each caller abort should release only that caller; abort/close the shared attach only when the last waiter cancels; evict a doomed pending entry before teardown so a fresh caller starts a fresh attach; preserve caching when at least one waiter remains and the session becomes ready. Add focused Vitest coverage in `extensions/browser/src/browser/chrome-mcp.test.ts` for one waiter aborting while another remains, all waiters aborting before factory resolve, all waiters aborting while `session.ready` is pending, a fresh caller after all-waiter abort, and keeping a ready-pending session cached when another waiter remains. Do not change plugin APIs, config, docs, or `CHANGELOG.md`; include release-note context in the PR body if the fix is user-visible.
