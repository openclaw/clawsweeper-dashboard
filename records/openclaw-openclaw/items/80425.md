## Summary

Keep open: the branch addresses a real source-reproducible session invalidation bug, but the auth-epoch fix bypasses the existing combined/profile-only ownership contract and can hide real local CLI account changes.

## What This Changes

The PR changes CLI auth epoch hashing to prefer auth-profile credentials over local CLI fingerprints, moves CLI OAuth status labels ahead of env API keys, adds regression tests, and updates a generated Swift protocol model.

## Best Possible Solution

Fix the storage-presence flip through the existing backend/prepare authEpochMode seam, or another explicit owner-approved rule, while preserving invalidation for real account/profile changes.

## Reproduction Assessment

Yes. Current main is source-reproducible by mocking the Claude local credential reader from null to an OAuth credential while the auth-profile credential stays unchanged; the resolver hashes different parts and session reuse invalidates on the epoch mismatch.

## Solution Assessment

No. The PR fixes the reported flip by changing resolver behavior globally, but the existing contract says combined mode includes host credential state, so the narrower maintainable fix should go through the authEpochMode/skipLocalCredential policy seam or explicitly update that contract.
