## Summary

Keep open: the reported auth-epoch bug is valid, but this PR should not merge as-is because the resolver-level fix bypasses the existing combined/profile-only auth ownership seam and still lacks observable real behavior proof.

## What This Changes

This PR changes CLI auth epoch hashing to treat auth-profile credentials as canonical over local CLI credentials, adds regression coverage, and moves CLI OAuth status labels ahead of aliased env API keys.

## Best Possible Solution

Fix the storage-presence flip through the backend/prepare authEpochMode ownership seam, or an explicit maintainer-approved contract change, with regression coverage that preserves invalidation for real account/profile changes.

## Reproduction Assessment

Yes. Source inspection of current main shows the local and profile parts are both hashed when skipLocalCredential is false, and cli-session invalidates on a versioned epoch mismatch; I did not run tests because this is a read-only review.

## Solution Assessment

No. The PR fixes the reported flip by suppressing local credentials whenever a profile exists, but the narrower maintainable solution should respect authEpochMode/skipLocalCredential or explicitly update that contract.
