## Summary

This should close as superseded: the branch mixes two separately tracked fixes, and its auth-epoch implementation conflicts with the documented combined/profile-owned auth boundary. The useful work is already preserved in narrower open PRs, so keeping this combined branch open is not useful.

## What This Changes

This PR changes CLI auth epoch hashing to drop local credential fingerprints whenever an auth-profile credential exists, adds regression tests, and also moves CLI OAuth status labels ahead of aliased env API keys.

## Best Possible Solution

Use the narrower auth-epoch policy-seam fix in https://github.com/openclaw/openclaw/pull/80265 and the status-label fixes in https://github.com/openclaw/openclaw/pull/80260 or https://github.com/openclaw/openclaw/pull/80417 instead of landing this mixed resolver-level branch.

## Reproduction Assessment

Yes. Current-main source shows the local credential fingerprint and auth-profile part are both hashed when the skip flag is false, and the downstream session-reuse path invalidates on a versioned auth-epoch mismatch.

## Solution Assessment

No. The PR fixes the storage-presence flip by suppressing local credentials for every selected profile, but the documented boundary keeps combined host-plus-profile epochs unless the prepare/authEpochMode seam decides otherwise; narrower open PRs already track those fixes.
