## Summary

Close this PR as superseded: its useful targets are already covered by narrower open PRs, while this branch bypasses the existing CLI auth-epoch ownership seam and would weaken session invalidation in combined mode.

## What This Changes

This PR changes CLI auth epoch hashing to skip local credential fingerprints when an auth-profile credential exists, adds regression tests, and reorders CLI OAuth auth labels ahead of env API keys.

## Best Possible Solution

Review and land the narrower canonical PRs for the auth-epoch and status-label bugs, and keep this combined branch out of the merge queue.

Yes. The auth-epoch bug is source-reproducible by flipping the local Claude credential reader from null to OAuth while the auth profile stays unchanged, and the status-label bug is source-reproducible through selected anthropic plus active claude-cli runtime alias rendering.

## Solution Assessment

No. This PR uses a broad resolver-level auth change and a partial status-label reorder, while narrower open PRs address the same user problems at the intended policy and renderer surfaces.
