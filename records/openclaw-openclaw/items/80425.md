## Summary

This PR should close as superseded: the auth-epoch change bypasses the existing authEpochMode/skipLocalCredential policy seam and the same remaining bug is better tracked by a narrower open PR, while the status-label change is already split into narrower status PRs.

## What This Changes

The branch changes CLI auth epoch hashing to ignore local CLI credentials when an auth profile exists, adds regression tests, and reorders auth-label resolution so CLI OAuth is checked before env API keys.

## Best Possible Solution

Continue the auth-epoch fix in https://github.com/openclaw/openclaw/pull/80265 and the status-label fix in https://github.com/openclaw/openclaw/pull/80260, rather than merging this mixed branch with a resolver-level auth-boundary change.

## Reproduction Assessment

Yes. Source inspection shows current main hashes both local and profile auth parts when skipLocalCredential is false, and resolveCliSessionReuse invalidates on a versioned auth-epoch mismatch; the linked report also provides live reset logs.

## Solution Assessment

No. The auth-epoch part of this PR suppresses local credentials at the resolver level, while the maintainable path is to preserve the backend/prepare auth ownership seam or explicitly change that contract; the useful status-label work is already split into narrower PRs.
