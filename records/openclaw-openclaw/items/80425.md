## Summary

Keep open: the reported session-invalidation bug is real on current main, but this PR is not safe to merge because it bypasses the existing authEpochMode/skipLocalCredential ownership seam, introduces a status-label precedence regression, and still lacks observable runtime proof.

## What This Changes

This PR changes CLI auth epoch hashing to treat auth-profile credentials as canonical over local CLI credentials, adds regression tests, and moves CLI OAuth status-label checks ahead of env API-key labels.

## Best Possible Solution

Fix the storage-presence flip through the prepare/backend auth ownership seam, or an explicit maintainer-approved contract change, while keeping status labels aligned with actual auth resolution and requiring redacted live Claude CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows the local credential part and auth-profile part are hashed together when skipLocalCredential is false, and cli-session invalidates on a versioned epoch mismatch; the linked issue also includes live reason=auth-epoch logs.

## Solution Assessment

No. The PR fixes the observed flip by suppressing local credentials whenever a profile exists, but the narrower maintainable fix should respect the authEpochMode/skipLocalCredential policy seam or explicitly update that contract.

## Review Findings
