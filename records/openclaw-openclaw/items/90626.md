## Summary

Keep open. The PR addresses a real, source-reproducible cron default-delivery regression and now has copied live-output proof, but the patch is still not merge-ready because it can suppress explicit legacy delivery targets and leaves canonical non-CLI cron creation on the old announce default.

## What This Changes

The PR changes cron delivery defaults for isolated agent-turn jobs and mode-less delivery objects from `announce` to `none`.

## Best Possible Solution

Decide the cron delivery default policy, then make one creation-time default change in the canonical cron create path while preserving explicit legacy delivery targets and leaving SQLite migration repair to the canonical issue or a separate focused PR.

## Reproduction Assessment

Yes. Current main and `v2026.6.8` still create omitted isolated cron delivery as `announce`, resolve mode-less delivery objects as `announce`, and hard-fail unresolved requested delivery when not best-effort; the linked issue and copied live output show the resulting channel-required failure mode.

## Solution Assessment

No. The PR is a plausible mitigation, but not the best fix as submitted because it suppresses explicit mode-less delivery targets and changes only the CLI fallback while canonical API/service creation still defaults omitted detached delivery to `announce`.
