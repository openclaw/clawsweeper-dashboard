## Summary

Keep open. ## What This Changes

The PR changes cron CLI and delivery-plan fallbacks so isolated agentTurn cron jobs without explicit delivery resolve toward delivery.mode="none" instead of "announce".

## Best Possible Solution

Yes. Source inspection shows current main and v2026.6.6 default isolated output cron delivery to announce/last, and unresolved non-best-effort delivery can still fail the run; I did not execute a live cron job in this read-only review.
