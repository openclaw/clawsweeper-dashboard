## Summary

Keep this PR open: the bounded `childSessions` shape is a plausible fix for session-list bloat, but it still needs real behavior proof and one row-shaped Gateway event metadata gap fixed before merge.

## What This Changes

The PR bounds gateway `childSessions` list output to 50 visible child keys, adds total/truncation metadata, and adds regression coverage for large controller sessions.

## Best Possible Solution

Land only after maintainers accept the capped `childSessions` contract, the new count/truncation metadata reaches list, tool, and row-shaped event surfaces, focused tests cover those surfaces, and the contributor adds redacted real setup proof of the after-fix behavior.

## Reproduction Assessment

No, not as a real runtime repro in this read-only review. Source inspection does show the current-main unbounded serialization path for child-session links with high confidence, but I did not run a large live store timeout scenario.

## Solution Assessment

Not yet. The central approach is narrow and maintainable, but the new metadata should be propagated through row-shaped session change events before the capped API contract is safe to merge.
