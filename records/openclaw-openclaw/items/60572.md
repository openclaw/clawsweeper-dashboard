## What This Changes

The issue requests replacing the single `plugins.slots.memory` selector with purpose-specific, optionally per-agent memory role slots for recall, compaction, capture, dreaming, and user modeling.

## Best Possible Solution

Review, rebase, and either land or replace https://github.com/openclaw/openclaw/pull/88504 with a maintainer-approved role-slot contract that preserves legacy `plugins.slots.memory` through doctor, docs, runtime, and upgrade coverage.

## Solution Assessment

Unclear. Multi-role memory slots are plausible, but the role taxonomy, per-agent semantics, legacy migration, and VISION.md direction need maintainer approval before calling this the best solution.
