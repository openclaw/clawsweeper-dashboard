## Summary

Keep open. Current main still has one exclusive active memory slot, plus additive supplement APIs and a narrow dreaming coexistence exception; it does not provide purpose-specific memory roles or per-agent memory slot overrides, and the issue discussion shows real composition needs that are not cleanup material.

## What This Changes

The issue requests replacing the single `plugins.slots.memory` selector with purpose-specific memory slots such as recall, compaction, capture, and user modeling, including per-agent overrides.

## Best Possible Solution

Keep this as the canonical memory-role composition RFC until maintainers define a backward-compatible contract that preserves `plugins.slots.memory`, supports additive memory layers intentionally, and decides whether per-agent slot scoping belongs in public config.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a runtime bug. Source, schema, tests, and docs give high-confidence current-main proof that memory still has one active slot plus supplements, not purpose-specific role slots.

## Solution Assessment

Unclear whether the exact four dotted slot names are the best final API. The maintainable path is a versioned memory capability/role contract with migration and docs, then a user-facing config shape that does not break existing memory plugins.
