## Summary

Keep open: current main and the latest release still expose a single scalar memory slot, while the open closing PR is the active candidate implementation rather than merged behavior.

## What This Changes

The issue requests purpose-specific, per-agent memory role slots so recall, compaction, capture, dreaming, and user modeling plugins can run side by side.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/88504 with a maintainer-approved plugin/config contract that preserves legacy `plugins.slots.memory` through doctor, docs, runtime, and upgrade coverage.

## Reproduction Assessment

Yes, at source level. Current main only accepts and routes a scalar `plugins.slots.memory`, so the requested memory role-slot architecture is absent without needing a live multi-agent setup.

## Solution Assessment

Unclear. Multi-role memory slots are a plausible plugin API extension, but the taxonomy, per-agent semantics, legacy migration, and relationship to OpenClaw's current one-memory-slot direction need maintainer approval before calling it the best solution.
