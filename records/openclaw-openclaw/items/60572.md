## Summary

Keep open: current main and the latest release still expose a single scalar memory slot, and the linked closing PR is open and conflicting, so the requested multi-slot memory architecture is not implemented yet.

## What This Changes

Request purpose-specific memory role slots and per-agent slot overrides so multiple memory providers can compose instead of competing for one global memory slot.

## Best Possible Solution

Review, rebase, and either land or replace the open memory role-slot implementation with a maintainer-approved plugin/config contract that preserves legacy `plugins.slots.memory` through doctor, docs, runtime, and upgrade coverage.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.8` only accept scalar `plugins.slots.memory` and resolve one memory runtime plugin, so the requested role-slot architecture is absent without needing a live multi-agent setup.

## Solution Assessment

Unclear. Multi-role memory slots are a plausible plugin API extension, but the role taxonomy, per-agent semantics, legacy slot migration, and relationship to the current one-memory-slot direction need maintainer approval before calling it the best solution.
