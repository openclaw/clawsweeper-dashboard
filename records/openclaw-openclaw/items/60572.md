## Summary

Keep open. Current main still exposes one exclusive active memory slot plus additive supplement APIs and a narrow dreaming coexistence exception, but it does not implement purpose-specific memory role slots or per-agent plugin slot overrides; the discussion remains active and production-backed rather than cleanup material.

## What This Changes

The issue requests replacing the single `plugins.slots.memory` owner with multiple purpose-specific memory slots such as recall, compaction, capture, and user modeling, including per-agent slot overrides.

## Best Possible Solution

Keep this as the canonical multi-slot memory RFC until maintainers design a backward-compatible memory role/capability contract that preserves `plugins.slots.memory`, integrates supplements and dreaming coexistence, and decides whether per-agent plugin slot scoping belongs in public config.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a runtime bug. The current-main source, docs, and tests provide high-confidence evidence that OpenClaw still models one active memory slot with supplements rather than purpose-specific memory role slots.

## Solution Assessment

Unclear whether the exact proposed taxonomy is the best final API. A maintainable solution should start with a versioned memory role/capability contract and migration story, then expose user-facing shorthand without breaking existing memory plugins.
