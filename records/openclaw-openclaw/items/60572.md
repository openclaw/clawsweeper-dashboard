## What This Changes

The issue requests purpose-specific memory role slots and per-agent memory slot overrides so multiple memory-related plugins can compose instead of competing for one `plugins.slots.memory` owner.

## Best Possible Solution

Review and either land or replace the open implementation with a maintainer-approved memory role-slot API that preserves `plugins.slots.memory` compatibility and includes schema, doctor, docs, runtime, and plugin SDK coverage.
