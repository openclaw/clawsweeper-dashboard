## Summary

Keep this PR open because it is a useful OpenRouter/Qwen prompt-caching feature, but the current matcher is still not aligned with OpenRouter's documented explicit-caching support and the user-facing prompt-caching docs do not move with the behavior.

## What This Changes

The PR adds a Qwen model-family matcher, wires it into the OpenRouter system prompt cache-control wrapper, and adds matcher unit tests.

## Best Possible Solution

Land a narrow update that keys Qwen marker injection to OpenRouter's documented explicit-caching support, adds included/excluded wrapper tests, and updates the public prompt-caching reference.

## Reproduction Assessment

Yes for the PR defect: the proposed regex can be evaluated directly and compared with current OpenRouter Qwen prompt-caching docs and model metadata. That shows it misses documented/cache-priced Qwen IDs and admits unsupported future snapshot-style IDs.

## Solution Assessment

No for the current diff. The wrapper is the right implementation seam, but the matcher should align with OpenRouter's documented explicit-caching support and the docs need to describe the new Qwen behavior.
