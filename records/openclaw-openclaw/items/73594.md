## Summary

Keep open: the PR is a useful OpenRouter/Qwen prompt-caching feature, and current main does not already implement it, but the proposed matcher still misses current Alibaba-only Qwen snapshot IDs that expose cache-write pricing.

## What This Changes

The PR adds a closed-source Qwen model matcher, extends the OpenRouter system-cache wrapper to inject Anthropic-style `cache_control` markers for selected Qwen plus/max/flash models, and adds matcher tests.

## Best Possible Solution

Land a narrow OpenRouter/Qwen cache-control update that covers current Alibaba-only cache-write Qwen plus/max/flash IDs, preserves uncertain-family exclusions, and documents the supported scope.

## Reproduction Assessment

Yes for the PR defect. The proposed regex rejects current OpenRouter IDs `qwen/qwen3.5-plus-02-15` and `qwen/qwen3.5-flash-02-23`, while OpenRouter's endpoints API lists both as Alibaba cache-write endpoints.

## Solution Assessment

No for the current diff. Extending the existing wrapper is the right implementation shape, but the matcher and docs need to cover the current supported Qwen snapshots before merge.
