## Summary

Keep open: this is a useful, focused OpenRouter/Qwen prompt-caching feature, and the PR now includes convincing live behavior proof, but the matcher is still inconsistent with current OpenRouter Qwen caching support and the public prompt-caching docs do not describe the new behavior.

## What This Changes

The PR adds a Qwen-family matcher, wires it into the OpenRouter system cache-control wrapper, and adds matcher unit tests.

## Best Possible Solution

Land a narrow update that keys Qwen marker injection to OpenRouter's documented explicit-caching support, adds included/excluded wrapper tests, and updates the public prompt-caching reference.

## Reproduction Assessment

Yes for the PR defect: the proposed regex can be evaluated directly and compared with current OpenRouter Qwen prompt-caching docs/API metadata. That shows it misses documented cache-supported Qwen IDs and admits snapshot-style IDs the docs say should not receive explicit caching.

## Solution Assessment

No for the current diff. The wrapper is the right implementation seam, but the matcher should be aligned with OpenRouter's documented explicit-caching support and the docs need to move with the behavior.
