## Summary

Keep open. Current main still only injects OpenRouter cache markers for Anthropic model refs, and this PR is useful, but the branch matcher does not line up with OpenRouter's current Alibaba Qwen explicit-cache documentation and the OpenClaw prompt-caching docs remain Anthropic-only.

## What This Changes

The PR adds a Qwen model-family matcher, wires it into the OpenRouter system prompt cache-control wrapper, and adds matcher tests.

## Best Possible Solution

Land a narrow update that matches the documented Alibaba Qwen explicit-cache set, keeps uncertain snapshots out unless proven supported, and updates prompt-caching docs with the supported OpenRouter/Qwen scope.

## Reproduction Assessment

Yes for the PR defect: evaluating the proposed regex shows it omits documented cache-capable Qwen IDs such as qwen/qwen-plus and qwen/qwen3-coder-plus while encoding snapshot support that OpenRouter's docs say is unsupported. Current-main behavior is also clear from source because the wrapper is still Anthropic-only.

## Solution Assessment

No for the current diff. Extending the existing OpenRouter wrapper is the right implementation shape, but the matcher and docs should be aligned with OpenRouter's current Qwen explicit-cache contract before merge.
