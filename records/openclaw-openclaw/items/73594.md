## Summary

Keep open: this is a useful, focused OpenRouter Qwen prompt-caching feature, but the current matcher is not aligned with OpenRouter's documented/API-backed explicit-caching contract and the user-facing docs still describe this path as Anthropic-only.

## What This Changes

The PR adds a Qwen model-family matcher, uses it to enable OpenRouter system/developer `cache_control` marker injection beyond Anthropic refs, and adds matcher tests.

## Best Possible Solution

Land Qwen cache-control support through the existing verified OpenRouter wrapper using a docs/API-backed supported model set, matching tests, and public docs.

## Reproduction Assessment

Yes for the PR defect: the matcher can be evaluated directly against the current OpenRouter docs/API-backed explicit-caching set. That source check shows supported IDs are excluded while unsupported future/preview-style IDs are admitted.

## Solution Assessment

No for the current diff. The wrapper seam and live proof are good, but the matcher/tests should track OpenRouter's explicit-caching contract and the docs should move with the behavior.
