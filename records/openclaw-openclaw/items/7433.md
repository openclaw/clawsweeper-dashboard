## Summary

Keep this issue open. Current main still has only the partial WhatsApp reconnect/group metadata fallback; the requested Baileys retry callback, socket-level group metadata cache, proactive group/participant event handling, and LID update handling remain absent from the WhatsApp plugin.

## What This Changes

The issue requests WhatsApp/Baileys reliability hooks for retry/decryption, socket-level group metadata caching, proactive group metadata updates, and LID mapping persistence.

## Best Possible Solution

Implement the missing reliability hooks inside the WhatsApp plugin, with bounded account-scoped message-content retention for retry support and tests for socket options plus group/LID event handling.

## Reproduction Assessment

Yes for the implementation gap: current-main source inspection shows the requested socket options and event handlers are absent while the pinned Baileys API exposes them. A live high-volume group retry failure was not reproduced.

## Solution Assessment

Yes. A focused WhatsApp plugin repair is the narrowest maintainable solution; it belongs in the plugin-owned session/socket, inbound monitor, send tracking, group metadata, and LID mapping paths without a core API change.
