Add a bounded, account-scoped recent message content store suitable for Baileys `getMessage`, including outbound content before `sendMessage` acknowledgement so sender-side retransmit requests can be answered. Preserve privacy by bounding TTL/size and avoiding durable message-content storage unless existing repo patterns already support it.
