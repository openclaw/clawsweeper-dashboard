## Repair Work Prompt

Fix the WhatsApp/Baileys group reliability gaps tracked by #7433 in the WhatsApp plugin only. Wire Baileys `getMessage` and `cachedGroupMetadata` into `createWaSocket` through typed plugin-owned options, connect the existing group metadata cache to the socket callback, and add proactive `groups.upsert`, `groups.update`, `group-participants.update`, and `lid-mapping.update` handling through the existing inbound monitor/LID resolution boundaries. For `getMessage`, keep a bounded privacy-preserving in-memory cache of recent WAMessageContent, store outbound content before `sendMessage` so retry receipts can be fulfilled, and avoid disk persistence or new config unless a maintainer explicitly asks. Add focused regression tests for socket options, retry message lookup, group metadata callback/event updates, participant updates, and LID mapping behavior. Update `CHANGELOG.md` because this is user-facing WhatsApp reliability work. Do not move WhatsApp-specific behavior into core or change unrelated channel/plugin APIs.
