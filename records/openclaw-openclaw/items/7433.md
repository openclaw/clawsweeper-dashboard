## Repair Work Prompt

Fix the WhatsApp/Baileys group reliability gap tracked at https://github.com/openclaw/openclaw/issues/7433. Current main does not pass `getMessage` or `cachedGroupMetadata` into `makeWASocket`, and `attachWebInboxToSocket` only wires `messages.upsert`/`connection.update` plus connect-time group hydration. Implement a focused WhatsApp plugin repair: provide a bounded, account-scoped, in-memory recent message content source for Baileys retry `getMessage`; pass existing bounded group metadata cache through `cachedGroupMetadata`; refresh that cache from `groups.upsert`, `groups.update`, and `group-participants.update`; evaluate `lid-mapping.update` against Baileys v7 signalRepository/auth-dir behavior and reuse existing LID mapping helpers instead of adding a parallel stale map. Do not change core SDK surfaces, add config, or persist message content to disk without explicit maintainer approval. Add focused tests for socket options and group/LID event handling, and keep the patch inside the WhatsApp plugin boundary.
