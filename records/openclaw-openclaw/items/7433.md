## Summary

Keep open. Current main still has only the partial WhatsApp reconnect/group metadata fallback; the WhatsApp plugin does not wire Baileys `getMessage`, `cachedGroupMetadata`, proactive group update events, participant updates, or `lid-mapping.update` handling.

## What This Changes

The issue requests WhatsApp/Baileys reliability improvements for group messages by adding retry lookup, socket-level group metadata caching, proactive group metadata/participant updates, and LID mapping persistence.

## Best Possible Solution

Land a focused WhatsApp plugin patch that wires the missing Baileys retry and group/LID cache hooks through existing session and inbound monitor boundaries with bounded, privacy-preserving retention.
