## Summary

Current main still has only the reconnect-time group metadata fallback; the Baileys retry callback, socket-level metadata cache, and proactive group/LID event integration requested here remain absent, so the reliability work is still necessary.

## What This Changes

Add Baileys retry-message lookup, socket-level group metadata caching, proactive group cache updates, and reliable LID mapping handling to the WhatsApp plugin.

## Best Possible Solution

Land a focused WhatsApp plugin repair that supplies Baileys with bounded recent-message lookup and the existing group metadata cache, keeps that cache current from supported events, and reuses the canonical Baileys LID/auth mapping rather than adding parallel persistent state.
