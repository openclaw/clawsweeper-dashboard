## Repair Work Prompt

Implement the WhatsApp/Baileys reliability hooks tracked by https://github.com/openclaw/openclaw/issues/7433 in the WhatsApp plugin only. Current main still does not pass `getMessage` or `cachedGroupMetadata` to `makeWASocket`, and `attachWebInboxToSocket` only listens to `messages.upsert` and `connection.update`. Add a bounded, privacy-preserving recent message content store sufficient for Baileys `getMessage`, making sure outbound message content needed for sender retransmit is available before the retry window can occur. Wire the existing group metadata cache into `cachedGroupMetadata`, update it from `groups.upsert`, `groups.update`, and `group-participants.update`, and handle `lid-mapping.update` through the existing LID resolution/persistence path where feasible. Keep the change inside `extensions/whatsapp`; do not add core bundled-plugin ids, do not add a new config option, and stop for maintainer review if the only viable design requires durable disk persistence of message bodies. Add focused regression coverage for socket options and event handling, then add a user-facing `CHANGELOG.md` entry under the active version because this improves WhatsApp reliability.
