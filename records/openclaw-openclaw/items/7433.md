Implement a focused WhatsApp plugin reliability patch for https://github.com/openclaw/openclaw/issues/7433. Current main has only the partial reconnect metadata fallback from https://github.com/openclaw/openclaw/pull/73732; it still lacks Baileys `getMessage`, socket-level `cachedGroupMetadata`, proactive `groups.upsert` / `groups.update` / `group-participants.update`, and `lid-mapping.update` handling. Add a single-line user-facing changelog entry. Do not change core plugin SDK APIs, add config knobs, or persist message content to disk.
