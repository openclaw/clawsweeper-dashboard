## Repair Work Prompt

Fix the WhatsApp plugin reliability gap tracked by https://github.com/openclaw/openclaw/issues/7433. Wire Baileys `getMessage` and `cachedGroupMetadata` into `createWaSocket`/socket construction, add proactive `groups.upsert`, `groups.update`, `group-participants.update`, and `lid-mapping.update` handling through the existing inbound monitor/cache/LID boundaries, and add focused regression coverage. Keep the implementation plugin-owned under `extensions/whatsapp`; do not add core SDK surface or new user config. For `getMessage`, cache `WAMessageContent` before send, not only after ack, keep retention bounded/TTL-based, and avoid persisting private message bodies beyond the minimum needed for retry. Preserve existing reconnect/group metadata fallback behavior and outbound echo suppression. Add or update changelog if the repair is user-facing.
