## Summary

Keep this issue open. Current main still has only the partial WhatsApp reconnect/group metadata fallback; the requested Baileys retry callback, socket-level group metadata cache, proactive group/participant update handlers, and LID update handler are still absent, and the later discussion specifically reinforces that `getMessage` needs sender-side content available before the ack window.
