Wire a bounded, account-scoped recent message-content cache into Baileys `getMessage` so retry resend requests can find recently sent content; connect a bounded `cachedGroupMetadata` provider to socket creation; add proactive handling for Baileys group metadata events and LID mapping updates without duplicating stale mappings already persisted by `sock.signalRepository.lidMapping`.
