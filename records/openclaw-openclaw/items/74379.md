## Summary

Keep open. Current main still lacks the requested `tools.exec.denyPathPatterns` config/schema/runtime gate, and the linked closing PR #75101 is open and unmerged, so this issue should remain the tracker until that implementation or a focused replacement lands.

## What This Changes

Request a config-level exec path-pattern denylist that blocks sensitive file arguments before command execution can leak secrets into transcripts or model context.

## Best Possible Solution

Review and land PR #75101 or a focused replacement that adds a shared pre-exec hard-deny gate for literal argv and shell-wrapper payload tokens, updates config schema/help/docs, adds targeted runtime tests, and documents bypass limits.

## Reproduction Assessment

Yes for the missing control: current main has no accepted `tools.exec.denyPathPatterns` schema key and no runtime deny-path phase in the inspected exec policy paths. The reported private token leak itself is not publicly reproducible from the available data.

## Solution Assessment

Unclear until the linked implementation is reviewed. A config-backed hard-deny gate before allowlist/approval is the narrow maintainable boundary for this class; optional warn or require-approval behavior should wait for explicit maintainer security design.
