## Summary

Keep open. Current main and the latest release still lack a `tools.exec.denyPathPatterns` config/schema/docs/runtime gate, and the current implementation candidate is open, unmerged, security-sensitive, and blocked by review/CI work.

## What This Changes

Request a config-level exec path denylist that blocks commands whose path-like arguments match sensitive glob patterns before exec policy allows or prompts them.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81827 or a focused replacement after it fixes the parser/security blockers, passes CI, and proves real OpenClaw exec or `system.run` denial behavior.

## Reproduction Assessment

Yes for the missing control: current-main source inspection shows no accepted `tools.exec.denyPathPatterns` schema key and no pre-exec path-deny phase. The private production leak itself is not publicly reproducible from the available artifacts.

## Solution Assessment

Unclear until the open implementation is repaired and reviewed. A default-off hard-deny gate before allowlist/approval is a plausible maintainable direction, but the current PR still needs security/parser fixes, green CI, and real behavior proof.
