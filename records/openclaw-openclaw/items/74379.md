## Summary

Keep open. Current main still lacks the requested exec path-deny config/schema/runtime gate, and PR #75101 explicitly targets this issue but is still open and unmerged, so the issue should remain open until that PR or a narrow replacement lands.

## What This Changes

The issue requests an operator-configurable `tools.exec.denyPathPatterns` denylist that blocks exec commands whose path-like arguments match sensitive file globs before execution.

## Best Possible Solution

Review and land PR #75101 or a narrow replacement that adds a shared pre-approval hard-deny gate for literal path-like argv and shell-wrapper payload tokens, updates config types/schema/help/docs, covers global and per-agent behavior, and documents v1 bypass limits.

## Reproduction Assessment

Yes for the missing control: current main has no accepted `tools.exec.denyPathPatterns` config key and no runtime deny-path phase in the node-host or gateway exec policy paths. The private production leak itself is not publicly reproducible from the available issue data.

## Solution Assessment

Unclear for the open implementation until PR #75101 is reviewed. The requested direction is a maintainable boundary if implemented as a shared pre-allowlist/pre-approval deny gate with conservative, documented matching semantics.
