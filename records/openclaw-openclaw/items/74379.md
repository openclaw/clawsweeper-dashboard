## Summary

Keep open. Current main still has no config/schema/docs/runtime support for an exec argument path denylist, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/75101 is still open and unmerged, so this issue remains the right tracker for maintainer security review.

## What This Changes

Add a config-level `tools.exec.denyPathPatterns` denylist that blocks exec commands whose arguments resolve to sensitive path globs before allowlist or approval execution.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75101 or a focused replacement that adds a default-off pre-exec hard-deny gate with schema/help/docs coverage, regression tests, and documented bypass limits.

## Reproduction Assessment

Yes for the missing control: current-main source inspection shows no accepted `tools.exec.denyPathPatterns` schema key and no pre-exec path-deny phase. The private token leak itself is not publicly reproducible from the available artifacts.

## Solution Assessment

Unclear until the linked PR is reviewed. A default-off config-backed hard-deny gate before allowlist/approval is the narrow maintainable boundary, but optional warning or approval modes and union semantics need maintainer security design.
