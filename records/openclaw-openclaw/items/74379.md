## Summary

Keep open. Current main still lacks the requested config key, strict schema support, shared runtime deny-path phase, and docs, and the linked closing PR is still open and unmerged.

## What This Changes

The issue requests an operator-configurable `tools.exec.denyPathPatterns` denylist that blocks exec commands whose path-like arguments match sensitive glob patterns before command execution.

## Best Possible Solution

Review and land PR #75101 or a focused replacement that adds a shared pre-exec hard-deny gate for literal argv and shell-wrapper payload tokens, updates config schema/help/docs, and documents bypass limits.

## Reproduction Assessment

Yes for the missing control: current main has no accepted `tools.exec.denyPathPatterns` schema key and no runtime deny-path phase in the inspected exec policy paths. The reported private token leak itself is not publicly reproducible from the available data.

## Solution Assessment

Unclear until the linked implementation is reviewed. A config-backed hard-deny gate before allowlist/approval is the narrow maintainable boundary for this class; optional warning or approval modes should wait for explicit maintainer security design.
