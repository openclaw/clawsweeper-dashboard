## Summary

Keep open. Current main still lacks the requested config/schema/runtime deny-path gate, and the linked closing PR #75101 remains open and unmerged, so this issue should stay open until that implementation or a narrow replacement lands.

## What This Changes

The issue requests an operator-configurable `tools.exec.denyPathPatterns` path denylist for exec command arguments so sensitive file paths can be blocked before command execution.

## Best Possible Solution

Review and land PR #75101 or a narrow replacement that adds a shared pre-approval hard-deny gate for literal path-like exec argv and shell-wrapper payload tokens, updates config schema/types/help/docs, includes focused runtime tests, documents bypass limits, and keeps the issue open until the fix merges.

## Reproduction Assessment

Yes for the missing control: current main has no accepted `tools.exec.denyPathPatterns` config key and no runtime deny-path phase in the inspected exec policy paths. The private production leak is not publicly reproducible from the available data.

## Solution Assessment

Unclear until the linked PR is reviewed. A shared deny gate before allowlist/approval is the right maintainable boundary for this class, but the final matching semantics and documented bypass scope need maintainer security review.
