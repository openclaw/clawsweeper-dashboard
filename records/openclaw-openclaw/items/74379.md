## Summary

Keep open. Current main still lacks a config/schema/docs/runtime path-deny gate for exec arguments, and the linked closing implementation PR is still open rather than merged.

## What This Changes

Request an operator-configurable `tools.exec.denyPathPatterns` denylist that blocks exec commands touching configured sensitive file paths before execution.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75101 or a focused replacement that adds a default-off pre-exec hard-deny gate with schema/help/docs coverage, regression tests, and documented bypass limits.

## Reproduction Assessment

Yes for the missing control: current-main source inspection shows no accepted `tools.exec.denyPathPatterns` schema key and no pre-exec path-deny phase. The private token leak itself is not publicly reproducible from the available artifacts.

## Solution Assessment

Unclear until the linked implementation is reviewed and repaired. A default-off config-backed hard-deny gate before allowlist/approval is a maintainable direction, but the exact semantics need maintainer security review.
