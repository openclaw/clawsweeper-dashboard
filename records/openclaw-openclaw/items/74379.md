## Summary

Keep open. Current main still lacks the requested config-level exec path denylist, and the open closing PR #75101 is unmerged, so this issue remains the right tracker for maintainer security review.

## What This Changes

Request an opt-in `tools.exec.denyPathPatterns` config that blocks exec commands when literal path-like arguments match sensitive file globs.

## Best Possible Solution

Review and land PR #75101 or a focused replacement that adds a shared pre-exec hard-deny gate, schema/help/docs coverage, regression tests, and documented bypass limits.

## Reproduction Assessment

Yes for the missing control: source inspection on current main shows no accepted `tools.exec.denyPathPatterns` schema key and no pre-exec path-deny phase. The reported private token leak itself is not publicly reproducible from the available artifacts.

## Solution Assessment

Unclear until PR #75101 is reviewed. A config-backed hard-deny gate before allowlist/approval is the narrow maintainable boundary; optional warning or approval modes should wait for explicit maintainer security design.
