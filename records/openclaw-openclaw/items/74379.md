## Summary

Keep open. Current main still lacks the requested config-level exec path denylist, and the open closing PR #75101 is still unmerged, so this issue remains the right tracker for maintainer security/config review.

## What This Changes

Add an operator-configurable `tools.exec.denyPathPatterns` denylist so exec commands that reference sensitive file paths can be blocked before execution.

## Best Possible Solution

Review and land PR #75101 or a focused replacement that adds a shared pre-exec hard-deny gate, schema/help/docs coverage, regression tests, and documented bypass limits, then close this issue after merge.

## Reproduction Assessment

Yes for the missing control: source inspection on current main shows no accepted `tools.exec.denyPathPatterns` schema key and no pre-exec path-deny phase. The reported private token leak itself is not publicly reproducible from the available artifacts.

## Solution Assessment

Unclear until PR #75101 is reviewed. A shared pre-exec hard-deny gate is the narrow maintainable direction; optional warn or require-approval modes should wait for explicit maintainer security design.
