## Summary

Keep open. Current main and the latest release still lack a `tools.exec.denyPathPatterns` config/schema/docs/runtime gate, while the active implementation path is the open, unmerged, failing, security-sensitive PR at https://github.com/openclaw/openclaw/pull/81827.

## What This Changes

The issue requests a config-level exec path denylist that blocks commands whose path-like arguments match configured sensitive-path globs before execution.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81827, or a focused replacement, only after parser/security blockers are fixed, CI is green, docs/schema/tests are aligned, and real OpenClaw exec or `system.run` denial behavior is proven.

## Reproduction Assessment

Yes for the missing control: current-main source inspection shows no accepted `tools.exec.denyPathPatterns` schema key and no pre-exec path-deny phase. The private production leak itself is not publicly reproducible from the available artifacts.

## Solution Assessment

Unclear until the open implementation is repaired and reviewed. A default-off hard-deny gate before allowlist/approval is a plausible maintainable direction, but the active PR still needs security/parser fixes, green CI, and real behavior proof.
