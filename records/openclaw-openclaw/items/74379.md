## Summary

Keep open. Current main still lacks the requested exec argument path denylist, and the linked closing PR #75101 is open and unmerged, so this remains the right tracker for maintainer security review.

## What This Changes

Request an operator-configurable `tools.exec.denyPathPatterns` denylist that blocks sensitive path arguments before exec commands run.

## Best Possible Solution

Review and land PR #75101 or a focused replacement that adds a shared pre-exec hard-deny gate with schema, help/docs, tests, and documented bypass limits.

## Reproduction Assessment

Yes for the missing control: current-main source inspection shows no accepted `tools.exec.denyPathPatterns` schema key and no pre-exec path-deny phase. The private token leak itself is not publicly reproducible from the available artifacts.

## Solution Assessment

Unclear until the linked implementation is reviewed. A config-backed hard-deny gate before allowlist/approval is the narrow maintainable boundary, while warning or approval modes should wait for explicit maintainer security design.
