## Summary

Keep open. Current `main` and latest shipped `v2026.5.7` still provide targeted read/write/edit validation and handler-level checks for tools like `exec`, but they do not implement the requested generic configurable pre-dispatch schema validation or automatic prompt/retry policy. The related report was closed as a duplicate pointing back to this issue, so this remains the canonical design thread.

## What This Changes

The issue requests configurable pre-dispatch validation of missing tool parameters, plus optional automatic prompting or retry before tool handlers run.

## Best Possible Solution

Add schema-derived pre-dispatch validation as a generic core seam while preserving richer per-tool validators, and treat automatic retry or prompting as a separate opt-in loop policy decision.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a strict bug: the report asks for a new configurable behavior. Source inspection on current `main` and `v2026.5.7` does confirm the requested generic schema-derived validation and retry/prompt config are absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear. Pre-dispatch schema validation is a plausible maintainable seam, but the proposed config names and automatic retry/prompt behavior need maintainer product/runtime review before implementation.
