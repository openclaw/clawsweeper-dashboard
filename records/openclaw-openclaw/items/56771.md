## Summary

Keep open. Current main has targeted validation for some core file tools and loop-detection guardrails, but it still lacks the requested generic configurable schema-derived pre-dispatch validation and automatic retry/prompt policy; the duplicate thread explicitly points back here as the canonical design item.

## What This Changes

The issue requests configurable schema-derived validation before tool dispatch plus optional automatic prompting or retrying when required tool parameters are missing.

## Best Possible Solution

Add a typed schema-derived validation seam in agent tool dispatch, preserve richer per-tool validators, and decide automatic prompting or retrying as a separate opt-in loop policy.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection confirms the requested generic schema-derived validation and config keys are absent on current main, while current behavior uses targeted validators and handler-level errors.

## Solution Assessment

Unclear. Pre-dispatch schema validation is a plausible maintainable direction, but the proposed config shape and automatic prompt/retry semantics need to be designed alongside existing per-tool validators and loop detection.
