## Summary

Keep open. Current main has targeted read/write/edit validation, structured tool error wrapping, and loop-detection/circuit-breaker machinery, but it still lacks the requested generic configurable pre-dispatch schema validation and automatic prompt/retry policy; the duplicate thread points back here as the canonical design item.

## What This Changes

Request to add configurable generic pre-dispatch tool parameter validation and optional auto-prompt/retry behavior for missing required tool arguments.

## Best Possible Solution

Add a typed schema-derived validation seam in agent tool dispatch, preserve richer per-tool validators, and decide automatic retry or prompting as a separate opt-in loop policy.
