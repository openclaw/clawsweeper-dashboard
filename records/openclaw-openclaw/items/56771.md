## Summary

Keep open. Current main still lacks the requested configurable, generic tool-parameter validation plus automatic prompt/retry policy; https://github.com/openclaw/openclaw/issues/56562 was closed as a duplicate pointing here, while https://github.com/openclaw/openclaw/pull/53329 and https://github.com/openclaw/openclaw/issues/72555 cover narrower exec/edit/circuit-breaker symptoms rather than the full design.

## What This Changes

The issue requests configurable pre-dispatch validation of required tool parameters and optional automatic prompting or retry when parameters are missing.

## Best Possible Solution

Add a typed schema-derived validation seam that composes with existing per-tool validators and before-tool-call hooks, then design automatic prompting or retrying as a separate explicit opt-in policy.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection confirms the requested config keys and generic validation policy are absent on current main, while existing behavior is a mix of schema normalization, targeted validators, handler-level errors, and loop detection.

## Solution Assessment

Unclear. Pre-dispatch validation is plausible, but the config names and auto-prompt/retry semantics need to be designed alongside pi-ai schema validation, OpenClaw before-tool hooks, per-tool validators, and loop detection.
