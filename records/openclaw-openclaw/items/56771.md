The issue requests configurable schema-derived validation before tool dispatch plus optional auto-prompt/retry handling for missing tool parameters.

## Best Possible Solution

Design a typed, schema-derived validation seam that composes with existing per-tool validators and before-tool-call hooks, then treat automatic prompting or retrying as a separate explicit opt-in policy.

Not applicable as a strict bug reproduction, but the missing capability is source-reproducible. Current main lacks the requested config keys and generic schema-derived validation policy while keeping targeted validators and loop-detection hooks.
