## Summary

Keep open: current main and the latest release have adjacent compaction notices, hooks, context-pressure checks, and silent memory flush, but they do not implement configurable pre-compaction warning thresholds, an agent-visible handoff turn, or an audited bounded deferral contract. This is new compaction lifecycle/API work that needs maintainer design rather than cleanup closure.

## What This Changes

The issue requests configurable pre-compaction warning thresholds, a structured pre-compaction handoff turn, and a bounded audited deferral mechanism.

## Best Possible Solution

Define a single additive compaction lifecycle contract that reuses existing hooks, notifyUser, memory flush, checkpoint, and context-pressure seams while adding approved threshold events, handoff semantics, and audited bounded deferral.
