## Summary

Keep open. Current main has adjacent compaction start/end notices, hook messages, checkpoints, and silent memory flush behavior, but it still lacks configurable pre-compaction threshold warnings, a structured agent-visible handoff turn, and a bounded audited deferral contract.

## What This Changes

Request configurable pre-compaction warning thresholds, a structured handoff turn, and a bounded audited deferral mechanism for long-running agent sessions.

## Best Possible Solution

Define one first-class compaction lifecycle contract that reuses the existing memory flush, hooks, notifier, checkpoint, and context-engine seams while adding explicit threshold events, handoff semantics, and audited bounded deferral.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a broken current behavior report. Source and docs inspection on current main show adjacent compaction mechanisms but not the requested contracts.

## Solution Assessment

Unclear until maintainers settle the lifecycle API and deferral policy. The maintainable direction is a unified compaction lifecycle surface rather than isolated warning flags or a standalone defer tool.
