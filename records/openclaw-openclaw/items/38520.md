## Summary

Keep open. Current main has several adjacent compaction lifecycle pieces, but it still lacks configurable pre-compaction warning thresholds, a visible structured handoff turn, and any bounded audited deferral contract.

## What This Changes

Request pre-compaction threshold warnings, a structured handoff window, and a bounded deferral mechanism for long-running agent sessions.

## Best Possible Solution

Define a single first-class compaction lifecycle contract that reuses existing memory flush, hook, notifier, context-engine, and checkpoint seams while adding explicit threshold events, handoff semantics, and audited bounded deferral.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a broken current behavior report. Source and docs inspection gives high confidence that current main has adjacent lifecycle behavior but not the requested warning, handoff, or deferral contracts.

## Solution Assessment

Unclear until maintainers settle the lifecycle API. The maintainable direction is a unified compaction lifecycle surface rather than an isolated defer tool or warning flag that bypasses existing hooks, memory flushes, notifier behavior, checkpoints, and context-engine ownership.
