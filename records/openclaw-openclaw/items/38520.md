## Summary

Keep open. Current main has adjacent compaction safeguards, user start/end notices, silent memory flushes, compaction hooks, checkpoints, and provider seams, but it still does not implement the requested configurable warning thresholds, visible structured handoff turn, or bounded audited deferral contract; #11055 only tracks the warning subset.

## What This Changes

The issue requests configurable pre-compaction warning events, a visible structured handoff window, and bounded audited compaction deferral.

## Best Possible Solution

Define one first-class compaction lifecycle contract that covers threshold events, visible handoff semantics, timeout/prompt policy, audited bounded deferral, and plugin/channel delivery rules while reusing or deliberately replacing the existing memoryFlush, notifyUser, hook, checkpoint, and context-engine seams.

## Reproduction Assessment

Not applicable as a feature/API request. Source and docs inspection on current main is high-confidence: the proposed config keys and deferral/handoff contracts are absent while adjacent compaction behavior exists.

## Solution Assessment

Unclear until maintainers settle the API shape. The narrow maintainable direction is a single compaction lifecycle surface rather than adding an isolated defer tool or warning flag that bypasses existing hooks, memory flushes, and channel delivery semantics.
