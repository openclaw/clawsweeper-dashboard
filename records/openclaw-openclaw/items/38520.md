## Summary

Keep open. Current main still has adjacent compaction safeguards, but it does not implement the requested configurable pre-compaction warning thresholds, visible structured handoff turn, or bounded/audited compaction deferral mechanism.

## What This Changes

The issue requests configurable pre-compaction warning thresholds, a structured pre-compaction handoff turn, and a bounded deferral mechanism for imminent compaction.

## Best Possible Solution

Define a first-class compaction lifecycle policy that validates warning/critical threshold config, specifies how warnings are delivered across channel/UI surfaces, decides whether core should run a visible pre-compaction handoff turn, and designs any deferral capability as a bounded audited contract rather than an ad hoc hook side effect.
