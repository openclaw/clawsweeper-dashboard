## Summary

Keep open. Current main has adjacent compaction safeguards, including silent memory flushes, user start/end notices, compaction hooks, checkpoints, and provider seams, but it still does not implement the requested configurable pre-compaction warning thresholds, visible structured handoff turn, or bounded/audited compaction deferral contract. The open #11055 covers only the warning subset, so it is not a complete duplicate for the handoff and deferral scope.

## What This Changes

Add configurable pre-compaction warning thresholds, a structured pre-compaction handoff turn, and a bounded audited deferral mechanism for imminent compaction.

## Best Possible Solution

Define a first-class compaction lifecycle contract in core: validated threshold config, deterministic delivery semantics for warning events across channel/UI surfaces, a visible pre-compaction handoff turn with timeout and prompt policy, and a bounded audited deferral mechanism if maintainers want agents or plugins to delay imminent compaction. The implementation should reuse or deliberately replace existing memoryFlush, notifyUser, session:compact hooks, compaction provider, and context-engine maintenance surfaces, with config schema/help/docs and focused runtime tests updated together.

## Reproduction Assessment

Not applicable as a feature/API request. The high-confidence verification path is source and docs inspection against current main, which shows the requested config keys and deferral/handoff contracts are absent while adjacent compaction behavior exists.

## Solution Assessment

Unclear until maintainers settle the API shape. The narrowest maintainable direction is not to bolt a tool onto memoryFlush alone, but to design one compaction lifecycle surface that covers thresholds, visible handoff, timeout policy, auditing, and plugin/channel semantics.
