## Summary

Keep open. Current main has adjacent compaction controls, start/end notices, hook messages, silent memory flushes, post-compaction section reinjection, and a bundled compaction notifier, but it still lacks configurable pre-warning thresholds, a visible structured pre-compaction handoff turn, and a bounded audited deferral contract; #11055 covers only the warning subset.

## What This Changes

Requests configurable compaction warning thresholds, a structured pre-compaction handoff turn, and a bounded audited compaction deferral mechanism.

## Best Possible Solution

Define one first-class compaction lifecycle contract that reuses the existing memory flush, hook, notifier, context-engine, and post-compaction context seams while adding explicit threshold events, handoff semantics, and audited bounded deferral.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a broken current behavior report. Source and docs inspection on current main is high-confidence that adjacent compaction lifecycle behavior exists but the requested contracts are absent.

## Solution Assessment

Unclear until maintainers settle the API shape. The maintainable direction is a single lifecycle surface rather than adding an isolated defer tool or warning flag that bypasses existing hooks, memory flushes, notifier behavior, and context-engine ownership.
