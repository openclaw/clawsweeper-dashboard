## Summary

Keep open: current main and v2026.6.10 have adjacent compaction start/end notices, memoryFlush, extension hooks, and stored compaction checkpoints, but they still do not provide configurable pre-threshold warnings, a structured agent handoff turn, or bounded agent-requested compaction deferral. ## What This Changes

Request to add pre-compaction warning events, a structured pre-compaction handoff turn, and bounded deferral controls for long stateful agent workflows.

## Best Possible Solution

Define one maintainer-approved compaction lifecycle contract that reuses existing checkpoints, hooks, memoryFlush, and notifyUser where appropriate, then add only narrow opt-in warning, handoff, or deferral surfaces with explicit audit and security policy.

Not applicable; this is a feature/design request rather than broken existing behavior. Source and release inspection give high-confidence evidence that the requested warning, handoff, and deferral surfaces are absent while adjacent compaction lifecycle features exist.
