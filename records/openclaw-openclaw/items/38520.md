## Summary

Keep open. Current main has adjacent compaction start/end notices, bundled compaction-notifier hook messages, checkpoints, and silent memory flush behavior, but it still lacks configurable pre-threshold warnings, a structured agent-visible handoff turn, and a bounded audited deferral contract.

## What This Changes

The issue requests configurable pre-compaction threshold warnings, a structured pre-compaction handoff turn, and bounded audited compaction deferral.

## Best Possible Solution

Define one first-class compaction lifecycle contract that reuses the existing memory flush, hooks, notifier, checkpoint, and context-engine seams while adding explicit threshold events, handoff semantics, and audited bounded deferral.

## Solution Assessment

Unclear until maintainers settle lifecycle API and deferral policy. The maintainable direction is a unified compaction lifecycle surface rather than isolated warning flags or a standalone defer tool.
