  - body: With Activity mode enabled and no configured token, this returns true after launch-context markers that can be supplied through query/cookie/referer, and Gateway canvas auth accepts that before normal bearer or canvas-capability checks.   - confidence: 0.96

  - body: This adds canvasHost.activity back to root Gateway config even though current main migrates root canvasHost into plugins.entries.canvas.config.host and treats the root key as legacy input. Put Activity settings under the Canvas plugin config or get an explicit migration/product decision before adding this public config surface.
  - confidence: 0.94
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:173-175`
  - body: The injected helper executes @discord/embedded-app-sdk from mutable third-party CDNs in the Canvas origin, outside the lockfile and release review path. Pin and bundle the SDK or use a maintainer-approved local/proxied loading model with CSP proof.
  - confidence: 0.93
- **[P1] Stop putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The A2UI bridge reads activityToken from the page URL, forwards it in the WebSocket query string, and falls back to ws:// on HTTP pages. Use a safer cookie or token-exchange path and fail closed for insecure transport before exposing this bridge.
  - confidence: 0.9
- **[P2] Bound Activity bridge payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: broadcastPush stores arbitrary JSONL and sends it to every Activity WebSocket without byte limits or bufferedAmount/backpressure handling. Add payload caps and slow-client behavior before this public Activity bridge can be considered safe.
  - confidence: 0.91
- **[P2] Reset failed SDK loads so callers can retry:** `src/canvas-host/a2ui.ts:202-210`
  - body: If the CDN import or ready() call fails once, state._loadPromise remains the rejected promise and every later load() call rethrows it until a full page reload. Clear the cached promise on failure so transient Activity SDK failures can recover.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security and supply-chain concerns around Activity auth bypass, URL token transport, remote script execution, and unbounded WebSocket fanout.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:151`
  - body: isCanvasActivityAccessAllowed returns true when Activity mode is enabled and no token is configured, and the HTTP/WS authorization path accepts that before normal Gateway auth.
  - confidence: 0.96
- **[high] Canvas helper executes mutable CDN JavaScript:** `src/canvas-host/a2ui.ts:173`
  - body: The injected helper imports the Discord SDK from public CDNs at runtime, so Canvas-origin code execution is not pinned by package review or the lockfile.
  - confidence: 0.93
- **[medium] Activity token is copied into URL query strings:** `src/canvas-host/a2ui/index.html:325`
  - body: The A2UI bridge reads and forwards activityToken through WebSocket URLs and can use ws:// on HTTP pages, which can leak credentials through logs, history, intermediaries, or cleartext transport.
  - confidence: 0.9
- **[medium] A2UI Activity fanout is unbounded:** `src/gateway/a2ui-activity.ts:240`
  - body: The Activity hub retains the last JSONL payload and broadcasts every payload to all connected clients without size limits or backpressure checks, creating memory and event-loop risk.
  - confidence: 0.91
