Full review comments:

- **[P1] Move Activity config under the Canvas plugin config:** `src/config/types.gateway.ts:41-59`
  - body: Current main migrates root `canvasHost` into `plugins.entries.canvas.config.host` and hides the root key from the public schema, so adding `canvasHost.activity` creates a competing Canvas config source. Put Activity settings under the Canvas plugin owner or get an explicit migration/product decision.
  - confidence: 0.95
- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:147-154`
  - body: With Activity enabled, spoofable launch markers can reach this branch and a missing token returns `true`, so Canvas/A2UI can be authorized before bearer or node-capability auth. Activity access needs a token or verifiable Discord assertion before bypassing normal Gateway auth.
  - confidence: 0.97
- **[P1] Bundle or pin the Discord SDK instead of importing CDNs:** `src/canvas-host/a2ui.ts:172-176`
  - body: The injected Canvas helper imports executable SDK code from jsDelivr or esm.sh at runtime inside the Canvas origin. That creates a supply-chain execution path outside the repo's package lock and should be replaced with a bundled/pinned dependency or maintainer-approved loading model.
  - confidence: 0.94
- **[P1] Stop forwarding Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge reads the Activity token from the page URL, appends it to the WebSocket query string, and falls back to `ws://` on HTTP pages. Use a safer cookie/exchange/header-capable path and fail closed for insecure transport before exposing this publicly.
  - confidence: 0.93
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every Activity WebSocket without payload limits or `bufferedAmount` handling. Add size caps and slow-client drop/close behavior before this public fanout path can be merged.
  - confidence: 0.91
- **[P2] Clear failed SDK loads before retrying:** `src/canvas-host/a2ui.ts:202-208`
  - body: When `OpenClaw.discord.load()` fails once, the rejected `_loadPromise` is retained and future calls keep awaiting the same failure. Clear the cached promise in the catch path so transient CDN or ready-timeout failures can recover without a full page reload.
  - confidence: 0.88
