## Security Review

Status: needs_attention

Summary: The diff introduces concrete auth-bypass, token-transport, and supply-chain script-loading concerns on the new Discord Activity Canvas path.

Concerns:

- **[high] Activity mode can bypass Canvas auth without a token:** `src/gateway/server/http-auth.ts:86`
  - body: The PR allows Activity Canvas/A2UI requests before normal Gateway bearer or plugin node-capability checks, while `isCanvasActivityAccessAllowed` returns true when no token is configured after weak launch-context markers.
  - confidence: 0.92
- **[high] Canvas helper loads mutable CDN JavaScript:** `src/canvas-host/a2ui.ts:173`
  - body: The injected helper imports `@discord/embedded-app-sdk` from third-party CDN URLs at runtime in the Canvas origin, which adds supply-chain script execution risk without pinning or bundling.

- **[medium] Activity token is placed in WebSocket URL:** `src/canvas-host/a2ui/index.html:327`
  - body: The A2UI bridge appends `activityToken` to the WebSocket URL and can use `ws://` on HTTP pages, exposing a credential to logs, browser history, intermediaries, and cleartext transport.
  - confidence: 0.89
- **[medium] A2UI fanout lacks payload and backpressure limits:** `src/gateway/a2ui-activity.ts:240`
  - body: The Activity hub retains full payload strings and broadcasts them to all clients without size caps or slow-client handling, which creates a denial-of-service path.
