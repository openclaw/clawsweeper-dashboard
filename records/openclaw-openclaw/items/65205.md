## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Require real activity auth before bypassing gateway auth:** `src/gateway/a2ui-activity.ts:151-154`
  - body: `isCanvasActivityAccessAllowed` returns true when no activity token is configured, and `hasDiscordActivityLaunchContext` accepts client-controlled query/cookie/referer markers. With the docs showing the token as optional, an external caller can append Activity-looking params or reuse the cookie path to reach Canvas/A2UI without normal gateway auth; make this fail closed unless a secret or signed Discord assertion verifies the request.
  - confidence: 0.95
- **[P1] Do not runtime-import the Discord SDK from CDNs:** `src/canvas-host/a2ui.ts:172-176`
  - body: The injected Canvas helper loads executable JavaScript from jsdelivr and esm.sh inside the gateway-served Canvas origin. That introduces unpinned third-party code execution; bundle/pin the SDK locally or require an app-provided helper under an explicit CSP.
  - confidence: 0.9
- **[P2] Bound A2UI activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every connected socket without a byte cap or `bufferedAmount` guard, while `node.invoke` can forward arbitrary A2UI payloads. A large update or slow clients can grow memory and stall the gateway; enforce payload limits and drop/close slow connections.
  - confidence: 0.86
- **[P2] Clear failed Discord SDK load attempts:** `src/canvas-host/a2ui.ts:202-206`
  - body: If the SDK import or `ready()` timeout fails once, the rejected `state._loadPromise` is kept forever and every later `OpenClaw.discord.load()` call rethrows until page reload. Clear the cached promise in the catch path so transient Activity load failures can recover.
  - confidence: 0.82
