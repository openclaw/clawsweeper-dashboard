## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Move Activity config onto the Canvas plugin surface:** `src/config/types.gateway.ts:41-59`
  - body: Current main serves Canvas/A2UI from `extensions/canvas` and keeps root `canvasHost` as a legacy migration input, but this PR adds new `canvasHost.activity` runtime config under core gateway types. That creates a competing config owner and will not survive the current Canvas plugin split; put Activity host settings under the Canvas plugin config and route/auth contract instead.
  - confidence: 0.94
- **[P1] Require a real Activity credential before bypassing gateway auth:** `src/gateway/a2ui-activity.ts:147-154`
  - body: With Activity enabled and no token configured, this returns `true` after accepting launch-context markers that come from query, cookie, or referer data. An external caller can spoof those markers on Canvas/A2UI paths and bypass normal Gateway auth, so the bypass needs a required secret or a verifiable signed Discord launch assertion.
  - confidence: 0.97
- **[P1] Bundle the Discord SDK instead of importing CDNs:** `src/canvas-host/a2ui.ts:172-175`
  - body: The injected Canvas helper dynamically imports executable JavaScript from jsDelivr or esm.sh inside the Canvas origin. That code bypasses the repo dependency/shrinkwrap review path and can change independently of OpenClaw releases, so use a pinned local dependency or reviewed bundled asset before exposing this helper.
  - confidence: 0.93
- **[P1] Keep Activity tokens out of WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The bridge copies `activityToken` from the page URL into the WebSocket query string and selects `ws://` when the page is served over HTTP. That can leak credentials through logs, history, intermediaries, and plaintext transport; require HTTPS/WSS for tokened access and use a cookie or handshake that does not put the token in URLs.
  - confidence: 0.91
- **[P2] Cap Activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:220-243`
  - body: `broadcastPush` stores the full last JSONL payload and sends every payload to every connected client without byte limits or `bufferedAmount` backpressure checks. A large A2UI push or slow client can grow memory or stall the event loop, so enforce payload caps and drop or close slow clients before merging this bridge.
  - confidence: 0.89
- **[P2] Clear failed Discord SDK load promises:** `src/canvas-host/a2ui.ts:202-208`
  - body: If `OpenClaw.discord.load()` fails once, the rejected `_loadPromise` remains cached and every later call re-awaits the same rejection. Clear the cached promise on failure so transient SDK load or `ready()` failures can recover without a full page reload.
  - confidence: 0.82
