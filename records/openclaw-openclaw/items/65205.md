## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require non-spoofable Activity auth before bypass:** `src/gateway/a2ui-activity.ts:151-152`
  - body: `authorizeCanvasRequest` returns success before normal gateway auth when `isCanvasActivityAccessAllowed` accepts the request, but the no-token path accepts caller-controlled launch markers from the URL, cookie, or referer. Activity mode therefore can expose Canvas/A2UI routes without a bearer token or node capability; require a configured secret or signed Discord assertion before granting this bypass, and cover HTTP plus WebSocket paths.
  - confidence: 0.94
- **[P1] Keep Activity config in the Canvas plugin contract:** `src/config/zod-schema.ts:660-667`
  - body: This adds `canvasHost.activity` under the retired root Canvas config surface, while current main routes Canvas through the bundled Canvas plugin and migrates legacy root `canvasHost` into `plugins.entries.canvas.config.host`. Adding new runtime behavior to the legacy root key creates a second source of truth and can break doctor/config/schema expectations; put the Activity host config under the Canvas plugin-owned contract instead.
  - confidence: 0.91
- **[P1] Bundle or remove runtime Discord SDK CDN imports:** `src/canvas-host/a2ui.ts:173-175`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from jsDelivr or esm.sh at runtime inside the Canvas origin. That gives public CDN responses script execution rights in the hosted app; bundle and pin the SDK through the owning plugin/package, or add a maintainer-approved controlled loading model with CSP/versioning proof.
  - confidence: 0.9
- **[P2] Bound Activity broadcasts before storing or sending:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every connected Activity WebSocket without a size cap or backpressure check, and `node.invoke` can forward large JSONL or stringify large objects into this path. Add payload limits, slow-client handling, and tests before exposing this hub on the gateway.
  - confidence: 0.88
- **[P2] Do not send Activity tokens over plain WebSockets:** `src/canvas-host/a2ui/index.html:327-329`
  - body: The browser bridge falls back to `ws://` on HTTP and places `activityToken` in the WebSocket query string. If the gateway is ever mis-served over HTTP during Activity testing or proxy setup, the token can leak through cleartext transport and logs; fail closed for tokened Activity mode without HTTPS/WSS or move the token into a safer server-set channel.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff introduces concrete auth-bypass, browser supply-chain, token transport, and broadcast resource-exhaustion concerns before merge.

Concerns:

- **[high] Spoofable Activity launch markers can bypass gateway auth:** `src/gateway/a2ui-activity.ts:151`
  - body: When Activity mode is enabled with no token, caller-controlled query/cookie/referer markers can satisfy launch-context enforcement and authorize Canvas/A2UI before bearer or node-capability auth.
  - confidence: 0.94
- **[high] Canvas helper executes SDK code from public CDNs:** `src/canvas-host/a2ui.ts:173`
  - body: The injected helper imports `@discord/embedded-app-sdk` from jsDelivr or esm.sh at runtime, giving those responses execution rights in the Canvas origin without pinning or a local package boundary.
  - confidence: 0.9
- **[medium] Activity token can be exposed in cleartext WebSocket URLs:** `src/canvas-host/a2ui/index.html:327`
  - body: The A2UI Activity bridge uses `ws://` on HTTP and sends `activityToken` in the query string, which can expose the token through transport, proxy, or log surfaces during misconfigured deployments.
  - confidence: 0.83
- **[medium] A2UI Activity hub lacks payload and backpressure limits:** `src/gateway/a2ui-activity.ts:240`
  - body: The hub stores and broadcasts unbounded payloads to all connected clients, creating a resource-exhaustion path for gateway memory or event-loop stalls.
  - confidence: 0.88
