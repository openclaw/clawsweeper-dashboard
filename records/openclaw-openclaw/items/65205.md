  - body: With Activity enabled and no token configured, this returns true after only spoofable launch-context markers, and the HTTP/WS auth path accepts that before Gateway bearer or Canvas capability auth. Require a token or verifiable Discord assertion before exposing public Canvas/A2UI.
  - confidence: 0.96

  - body: Current main migrates root canvasHost into plugins.entries.canvas.config.host and deletes the root key, so adding canvasHost.activity reopens a legacy config surface. Put Activity settings under the Canvas plugin owner or land an explicit migration/product decision.
  - confidence: 0.94
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper executes @discord/embedded-app-sdk from mutable third-party CDN URLs inside the Canvas origin. Bundle/pin the SDK through the plugin package or use a maintainer-approved loading policy before exposing this in production.
  - confidence: 0.91
- **[P1] Stop carrying Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge reads activityToken from the page URL and forwards it in the WebSocket query string, with ws:// on HTTP pages. Use a safer cookie/exchange/protocol path and fail closed for insecure transport so tokens do not leak through logs, history, or plaintext links.
  - confidence: 0.9
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: broadcastPush stores arbitrary JSONL and sends it to every Activity WebSocket without byte limits or bufferedAmount handling. Add payload caps and slow-client behavior before exposing this fanout path to public Activity clients.
  - confidence: 0.91
