## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Rebase Activity hosting onto the Canvas plugin:** `src/config/types.gateway.ts:41-59`
  - body: Current main and v2026.6.6 already serve Canvas/A2UI from `extensions/canvas` with plugin-owned config and route auth, and the public base schema intentionally hides root `canvasHost`. Adding `canvasHost.activity` back under core config creates a competing owner/default surface and bypasses the current plugin migration path.
  - confidence: 0.93
- **[P1] Require real Activity auth before bypassing gateway auth:** `src/gateway/a2ui-activity.ts:147-154`
  - body: With activity mode enabled, this returns true after spoofable query/cookie/referer launch markers, and also returns true when no token is configured. Because `authorizeCanvasRequest` accepts that before bearer/capability auth, an unauthenticated caller can reach Canvas/A2UI by adding Activity-looking markers.
  - confidence: 0.94
- **[P1] Do not runtime-import the Discord SDK from CDNs:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from jsDelivr or esm.sh at runtime, with no lockfile pin or local package ownership. That puts executable third-party code on the gateway-served Canvas origin and should be replaced with a bundled/pinned plugin-owned asset before merge.
  - confidence: 0.9
- **[P2] Bound A2UI Activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores the full JSONL payload and sends it to every client without a byte limit or `bufferedAmount`/slow-client handling; `node.invoke` can feed this path with arbitrary A2UI messages. A large payload or slow WebSocket client can turn one Canvas update into gateway memory/CPU pressure.
  - confidence: 0.88
- **[P2] Keep Activity tokens out of cleartext WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: When the page is served over HTTP, this builds `ws://.../__openclaw__/a2ui/ws?activityToken=...`, sending the Activity token in a query string over cleartext and into URL logs. Activity-token traffic should fail closed to HTTPS/WSS and use a cookie or other non-URL credential path.
  - confidence: 0.86
- **[P2] Reset failed Discord SDK load attempts:** `src/canvas-host/a2ui.ts:166-168`
  - body: After a transient CDN load or `ready()` timeout, `state._loadPromise` remains set to the rejected promise. Later calls re-await the same failure and cannot recover until page reload, so the helper should clear the promise in the failure path.
  - confidence: 0.82
