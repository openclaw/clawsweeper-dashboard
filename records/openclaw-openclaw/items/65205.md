Full review comments:

- **[P1] Require real Activity auth before bypassing gateway auth:** `src/gateway/a2ui-activity.ts:150-154`
  - body: When Activity mode is enabled and no token is configured, this returns true after only query/cookie/referer launch-marker checks. Those markers are client-controlled, and `authorizeCanvasRequest` then returns ok before normal gateway auth, so public Canvas/A2UI routes can become unauthenticated; require a token or signed Discord assertion before allowing this bypass.
  - confidence: 0.92
- **[P1] Keep Activity host config under the Canvas plugin:** `src/config/zod-schema.ts:654-660`
  - body: Current main moved Canvas host/A2UI ownership to `extensions/canvas` and `plugins.entries.canvas.config.host`; adding a new root `canvasHost.activity` schema reintroduces the retired root config path and bypasses the doctor migration/owner boundary. Rebase this onto the Canvas plugin config and HTTP route seam before merge.
  - confidence: 0.9
- **[P1] Do not runtime-import the Discord SDK from CDNs:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper loads executable SDK code from jsDelivr and falls back to esm.sh at runtime. That executes third-party CDN responses in the Canvas origin and is not pinned through the package/lockfile supply-chain path; vendor or serve the SDK from an owned, pinned asset instead.
  - confidence: 0.88
- **[P2] Bound A2UI Activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores the full JSONL payload and sends it to every connected socket without size limits or backpressure checks, while the node.invoke bridge can pass arbitrary JSONL/stringified objects into it. Add payload caps, validation, and slow-client handling before exposing this long-lived public Activity socket.
  - confidence: 0.86
- **[P2] Clear failed SDK load attempts so helpers can retry:** `src/canvas-host/a2ui.ts:166-210`
  - body: If `OpenClaw.discord.load()` fails once, the rejected promise remains in `state._loadPromise`, so later calls re-await the same failure and cannot recover from transient CDN/network/ready-timeout errors without a full page reload. Reset `_loadPromise` in the catch path.
  - confidence: 0.82
