  - body: Current main migrates root `canvasHost` into `plugins.entries.canvas.config.host` and serves Canvas/A2UI from the Canvas plugin. Adding a new root `canvasHost.activity` contract here creates a competing config surface and will not fit the current upgrade path after rebase.

  - body: With `activity.enabled` and no token, this returns `true` after only spoofable launch-context markers or a cookie, and `authorizeCanvasRequest` accepts it before normal Gateway bearer or node-capability auth.   - confidence: 0.96
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:172-175`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from jsDelivr or esm.sh at runtime. That makes every Activity page trust mutable third-party CDN code; use a pinned local dependency or an owner-approved integrity/CSP model before exposing this in production.
  - confidence: 0.92
- **[P1] Stop carrying Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The A2UI bridge reads `activityToken` from the page URL and forwards it in the WebSocket query string, falling back to cleartext `ws://` on HTTP pages. Use a safer cookie/exchange/protocol path and fail closed for insecure transport so tokens do not leak through logs, intermediaries, or browser history.
  - confidence: 0.9
