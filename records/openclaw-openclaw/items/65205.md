## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:151-154`
  - body: With Activity mode enabled and no token, any request that carries spoofable launch query/cookie/referer markers returns `true`, and `authorizeCanvasRequest` accepts that before bearer or capability auth. Fail closed unless a token or verifiable Discord assertion is configured and tested.
  - confidence: 0.96
- **[P1] Keep Activity config under the Canvas plugin:** `src/config/types.gateway.ts:41-59`
  - body: This adds `canvasHost.activity` to root config, but current main treats root `canvasHost` as legacy and migrates it into `plugins.entries.canvas.config.host`. Put Activity settings under the Canvas plugin config or get an explicit migration/product decision before creating a new root shape.
  - confidence: 0.94
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:171-176`
  - body: Canvas pages would execute `@discord/embedded-app-sdk` from jsDelivr or esm.sh in the Canvas origin, outside the lockfile and release review path. Use a pinned local dependency or controlled bundled asset instead of mutable runtime CDN imports.
  - confidence: 0.93
- **[P1] Stop putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The bridge reads `activityToken`/`a2uiToken` from the page URL, appends it to the WebSocket URL, and falls back to `ws://` on HTTP pages. That can leak credentials through logs, history, intermediaries, or cleartext local transports; use a safer cookie/exchange path and fail closed for insecure authenticated traffic.
  - confidence: 0.9
- **[P2] Bound Activity bridge fanout and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends each payload to every Activity WebSocket without byte caps or `bufferedAmount` handling. Add payload limits, drop/close behavior for slow clients, and coverage before exposing this hub.
  - confidence: 0.91
