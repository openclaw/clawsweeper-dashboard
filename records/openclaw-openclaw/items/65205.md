Overall confidence: 0.91

Full review comments:

- **[P1] Move Activity config to the Canvas plugin owner:** `src/config/zod-schema.ts:660-665`
  - body: This adds `canvasHost.activity` under the legacy root Canvas config, but current main hides root `canvasHost` and migrates it into `plugins.entries.canvas.config.host`. Adding a new documented root subtree creates a competing config owner and upgrade ambiguity; put the Activity settings under the Canvas plugin config or get an explicit maintainer migration/product decision.
  - confidence: 0.94
- **[P1] Require tokened Activity auth before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:150-154`
  - body: `isCanvasActivityAccessAllowed` returns true when no Activity token is configured, and the caller accepts that before normal bearer or canvas-capability auth. A request with spoofable launch-context markers can reach Canvas/A2UI without the existing Gateway auth boundary, so Activity access should fail closed unless tokened or backed by a verifiable Discord assertion.
  - confidence: 0.96
- **[P1] Bundle or pin the Discord Activity SDK:** `src/canvas-host/a2ui.ts:172-176`
  - body: The injected helper imports executable SDK code from jsDelivr and then esm.sh at runtime inside the Canvas origin. For this auth-sensitive iframe surface, use a bundled or explicitly pinned local dependency so released behavior is not controlled by mutable CDN responses.
  - confidence: 0.92
- **[P1] Bound Activity A2UI broadcasts before retaining payloads:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every connected Activity WebSocket without byte limits or `bufferedAmount` handling. A large `canvas.a2ui.pushJSONL` or `canvas.a2ui.push` result can retain memory and stall the gateway process, so add payload limits and slow-client behavior before merge.
  - confidence: 0.91
- **[P2] Avoid putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge copies `activityToken` into the WebSocket query and falls back to `ws://` on HTTP pages, which can leak tokens through logs, history, intermediaries, or cleartext proxy mistakes. Use a safer token exchange/cookie/header path and fail closed for insecure transport when the token is required.
  - confidence: 0.82
- **[P2] Clear failed SDK loads before retrying:** `src/canvas-host/a2ui.ts:202-208`
  - body: When `OpenClaw.discord.load()` fails once, the rejected promise remains cached in `state._loadPromise`, so later calls re-await the same failure and cannot recover from transient CDN or `ready()` errors without a full page reload. Clear the cached promise in the catch path or use a retryable loader state.
  - confidence: 0.87
