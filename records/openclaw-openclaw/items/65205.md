## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep Activity config on the Canvas plugin surface:** `src/config/types.gateway.ts:41-59`
  - body: Current main moved Canvas hosting/config to `plugins.entries.canvas.config.host` and plugin HTTP routes; adding `canvasHost.activity` back under core creates a competing config/default path and bypasses the Canvas plugin owner boundary. Port this Activity config and route handling into the Canvas plugin surface instead of reviving the retired root Canvas config.
  - confidence: 0.95
- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:147-154`
  - body: With Activity enabled and no token configured, this returns `true` after accepting launch-context markers that are just query/cookie/referer data. An external caller can spoof those markers on Canvas or A2UI paths and bypass normal Gateway auth, so Activity access needs a required secret or verifiable signed launch assertion before this early allow.
  - confidence: 0.97
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:171-175`
  - body: The injected Canvas helper dynamically imports executable JavaScript from jsDelivr or esm.sh inside the Canvas origin. That code is not pinned through the repo dependency/shrinkwrap review path, so a compromised or changed CDN response can run with page privileges; make the SDK a reviewed bundled dependency or local asset.
  - confidence: 0.93
- **[P1] Stop sending Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The bridge copies `activityToken` from the page URL into the WebSocket query string and uses `ws://` whenever the page is served over HTTP. That leaks credentials through logs/history/intermediaries and can transmit them in plaintext; require HTTPS/WSS for tokened Activity access and use a cookie or handshake that keeps the token out of URLs.
  - confidence: 0.91
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:220-243`
  - body: `broadcastPush` keeps the full last JSONL payload and sends every payload to every connected WebSocket without size limits or backpressure checks. A large `canvas.a2ui.pushJSONL` or slow client can grow memory or stall the event loop, so enforce payload caps and drop or close slow clients before merging this bridge.
  - confidence: 0.89
- **[P2] Reset failed Discord SDK load attempts:** `src/canvas-host/a2ui.ts:202-210`
  - body: If `load()` fails once, the rejected `_loadPromise` remains cached and every later call re-awaits the same rejection. Clear the cached promise on failure so transient CDN or `ready()` failures can recover without forcing a full page reload.
  - confidence: 0.82
