## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:147-154`
  - body: `hasDiscordActivityLaunchContext` accepts query/cookie/referer markers that any caller can spoof, and this branch then returns true when no Activity token is configured. That lets a request such as a Canvas/A2UI URL with `?platform=web` bypass the normal Gateway bearer or node-capability auth path.
  - confidence: 0.97
- **[P1] Move Activity config under the Canvas plugin owner:** `src/config/types.gateway.ts:41-59`
  - body: Current main moved Canvas host config to `plugins.entries.canvas.config.host` and keeps root `canvasHost` only as legacy migration input. Adding new `canvasHost.activity` runtime config here reopens a retired root config surface and bypasses the Canvas plugin's ownership and migration boundary.
  - confidence: 0.93
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:171-175`
  - body: This injects executable SDK code from mutable CDN URLs inside the Canvas page origin. A compromised CDN response or unexpected dependency resolution would run with the hosted Canvas page's privileges, so the helper needs a bundled/pinned dependency path or an explicitly approved loading policy before merge.
  - confidence: 0.94
- **[P1] Stop carrying Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The bridge copies `activityToken` from the page URL into the A2UI WebSocket query string and downgrades to `ws://` on HTTP pages. That leaks the token through URLs, logs, intermediaries, and cleartext transport in misconfigured deployments.
  - confidence: 0.95
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:220-243`
  - body: `broadcastPush` stores arbitrary JSONL as `lastJsonl` and sends it to every connected client with no byte limit or bufferedAmount/backpressure guard. A large A2UI update or slow client can retain memory and stall the event loop.
  - confidence: 0.9
- **[P2] Reset failed SDK loads so the helper can recover:** `src/canvas-host/a2ui.ts:166-210`
  - body: If the CDN import or `ready()` call rejects once, the rejected `state._loadPromise` is kept forever. Later `OpenClaw.discord.load()` calls re-await the same rejected promise, so transient startup/network failures cannot recover without a full page reload.
  - confidence: 0.88
