Overall confidence: 0.86

Full review comments:

- **[P1] Require non-spoofable Activity auth before bypass:** `src/gateway/a2ui-activity.ts:150-154`
  - body: `isCanvasActivityAccessAllowed` returns true for activity mode with no token after only caller-controlled launch markers are present, and `authorizeCanvasRequest` accepts that before normal gateway auth. Require a token or signed launch assertion by default before bypassing Canvas/A2UI bearer or node-capability auth.
  - confidence: 0.94
- **[P1] Move Activity config under the Canvas plugin:** `src/config/types.gateway.ts:41-59`
  - body: Current main moved Canvas host config to `plugins.entries.canvas.config.host` and treats root `canvasHost` as legacy migration input, but this PR adds new root `canvasHost.activity` settings. Put the Activity settings behind the plugin-owned Canvas/Discord config seam so upgrades and doctor migration do not get competing config sources.
  - confidence: 0.91
- **[P1] Bundle or pin the Discord Activity SDK:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from jsDelivr or esm.sh at runtime inside the Canvas origin. This makes Activity behavior and page privileges depend on mutable third-party CDN code; ship a pinned local dependency or another maintainer-approved trusted delivery path.
  - confidence: 0.9
- **[P1] Bound Activity A2UI broadcasts:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores the full JSONL payload and sends it to every connected WebSocket client without size limits or backpressure checks. Add payload bounds and slow-client handling before exposing this public Activity fanout path.
  - confidence: 0.88
- **[P2] Stop putting Activity tokens in ws URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge copies `activityToken` from the page URL into the WebSocket query string and downgrades to `ws://` on HTTP pages. Use a secure cookie, subprotocol, or other non-URL credential path and fail closed for authenticated Activity traffic on non-TLS pages.
  - confidence: 0.84
- **[P2] Clear failed SDK load promises so users can retry:** `src/canvas-host/a2ui.ts:202-208`
  - body: When SDK loading or `ready()` times out, `state._loadPromise` remains the rejected promise, so every later `OpenClaw.discord.load()` call rethrows until page reload. Clear the promise on failure so transient CDN or Discord readiness failures can recover.
  - confidence: 0.82
