- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:151-154`
  - body: With Activity mode enabled and no token configured, this returns true after spoofable launch-context markers, and Gateway canvas auth accepts that before normal bearer or canvas-capability checks. Activity access should fail closed unless a token or verifiable Discord assertion is configured and tested.
  - confidence: 0.96
- **[P1] Keep Activity config under the Canvas plugin:** `src/config/types.gateway.ts:41-59`
  - body: This adds canvasHost.activity to root Gateway config even though current main migrated Canvas host config to plugins.entries.canvas.config.host and treats root canvasHost as legacy doctor input. Put Activity settings under the Canvas plugin config or get an explicit migration/product decision.
  - confidence: 0.94
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:173-175`
  - body: The injected helper executes @discord/embedded-app-sdk from mutable third-party CDNs in the Canvas origin, outside the lockfile and release review path. Pin and bundle the SDK or use a maintainer-approved local/proxied loading model with CSP proof.
  - confidence: 0.93
- **[P1] Stop putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The A2UI bridge reads activityToken from the page URL, forwards it in the WebSocket query string, and falls back to ws:// on HTTP pages. Use a safer cookie/token exchange path and fail closed for insecure transport before exposing this bridge.
  - confidence: 0.88
- **[P2] Bound Activity bridge payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: broadcastPush stores arbitrary JSONL and sends it to every Activity WebSocket without byte limits or bufferedAmount/backpressure handling. Add payload caps and slow-client behavior before this public Activity bridge can be considered safe.
  - confidence: 0.91

## Security Review
