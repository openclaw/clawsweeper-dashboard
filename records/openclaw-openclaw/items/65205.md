  - body: With Activity enabled and no configured token, this returns true after only launch-context checks that can be supplied through query params, referer, or a cookie.   - confidence: 0.96

  - body: This adds new root canvasHost.activity config even though current main migrates root canvasHost into plugins.entries.canvas.config.host and registers Canvas routes from the Canvas plugin.   - confidence: 0.94
- **[P1] Stop carrying Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The bridge reads activityToken from the page query string, forwards it in the WebSocket URL, and uses ws:// when the page is served over HTTP. Use a safer cookie/token-exchange/protocol path and fail closed for insecure transport before exposing this publicly.
  - confidence: 0.92
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected helper executes @discord/embedded-app-sdk from mutable CDN URLs inside the Canvas origin. Bundle a pinned SDK or otherwise provide an owner-approved, version-pinned loading path before this runs in user-hosted Canvas pages.
  - confidence: 0.89
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: broadcastPush stores arbitrary JSONL and sends it to every Activity WebSocket without byte limits or bufferedAmount handling. Add payload caps and slow-client behavior before exposing a public fanout path.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces Activity auth, browser script execution, token transport, and WebSocket fanout concerns that need security and owner review before merge.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:151`
  - body: isCanvasActivityAccessAllowed returns true when Activity mode is enabled and no token is configured, allowing spoofable launch context to short-circuit normal Gateway or node-capability auth.
  - confidence: 0.96
- **[high] Injected helper loads executable code from mutable CDNs:** `src/canvas-host/a2ui.ts:173`
  - body: The Canvas helper imports @discord/embedded-app-sdk from jsDelivr with an esm.sh fallback, executing third-party CDN code in the Canvas origin without a pinned local dependency path.
  - confidence: 0.89
- **[medium] Activity token is copied into URL query strings:** `src/canvas-host/a2ui/index.html:325`
  - body: The browser bridge reads activityToken from the page URL and forwards it in the WebSocket URL, which can leak through logs, intermediaries, browser history, or plain HTTP deployments.
  - confidence: 0.92
- **[medium] A2UI Activity fanout is unbounded:** `src/gateway/a2ui-activity.ts:240`
  - body: The Activity hub retains and broadcasts arbitrary JSONL without size limits or slow-client backpressure, creating memory and event-loop risk for public Activity connections.
  - confidence: 0.91
