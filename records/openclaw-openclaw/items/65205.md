## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Move Activity config under the Canvas plugin owner:** `src/config/zod-schema.ts:654-660`
  - body: Current main keeps root `canvasHost` as a hidden legacy migration shape and resolves live Canvas host settings from `plugins.entries.canvas.config.host`. Adding `canvasHost.activity.*` here creates a second public config source that will conflict with the current Canvas plugin-owned surface after rebase and can strand existing users on the wrong shape.
  - confidence: 0.9
- **[P1] Require credentials before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:150-154`
  - body: When Activity mode is enabled, `authorizeCanvasRequest` accepts this helper before bearer or node-capability auth. Because `isCanvasActivityAccessAllowed` returns true with no configured token after only spoofable launch markers, a public caller can reach Canvas/A2UI endpoints without the normal Gateway credential path.
  - confidence: 0.94
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:171-176`
  - body: This injected script imports executable code from jsDelivr or esm.sh at runtime in the Canvas origin. A production Canvas page should not depend on mutable third-party CDN code for privileged Activity helpers; bundle a pinned dependency or otherwise use a maintainer-approved integrity/CSP model.
  - confidence: 0.9
- **[P1] Stop carrying Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The A2UI bridge copies `activityToken` into the WebSocket query string and downgrades to `ws://` when the page is served over HTTP. That can leak Activity credentials through logs, browser history, intermediaries, or cleartext local-network traffic; use a secure cookie or authenticated setup flow and fail closed for tokened Activity traffic without HTTPS/WSS.
  - confidence: 0.88
- **[P2] Bound Activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends the serialized payload to every connected Activity WebSocket without size or bufferedAmount checks. Since `node.invoke` can feed this hub from `canvas.a2ui.pushJSONL` and `canvas.a2ui.push`, a large payload or slow client can create memory and event-loop pressure outside the Gateway's existing backpressure protections.
  - confidence: 0.86
