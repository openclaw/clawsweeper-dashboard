  - body: Current main treats root `canvasHost` as a legacy migration path and serves Canvas through `extensions/canvas` plugin routes. Adding `canvasHost.activity` here creates a second public config surface and conflicts with the current owner boundary; rebase and put the Activity host/auth config under the Canvas plugin instead.
  - confidence: 0.92
- **[P1] Require real credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:145-147`
  - body: With Activity enabled and no token configured, any request with ordinary `instance_id`, `frame_id`, `platform`, cookie, or referer markers can pass before normal Gateway bearer or node-capability auth. Do not grant Canvas/A2UI access from spoofable launch markers alone; require a real credential or verifiable Discord assertion.
  - confidence: 0.95
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:169-176`
  - body: The injected helper imports executable code from mutable CDN URLs into the Canvas page origin. Add the SDK as a pinned/bundled dependency, or provide an owner-approved integrity/CSP story, before exposing this in hosted Canvas pages.
  - confidence: 0.9
- **[P1] Stop sending Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-330`
  - body: The A2UI bridge copies `activityToken` into the WebSocket query string and falls back to `ws://` for HTTP pages, which leaks credentials through URLs, logs, and cleartext transport. Use a secure cookie or another non-URL credential path and fail closed for authenticated Activity traffic without TLS.
  - confidence: 0.88
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:238-244`
  - body: `broadcastPush` stores the full JSONL payload and sends it to every client without size checks or `bufferedAmount` handling. Add payload limits and slow-client backpressure/termination before wiring node A2UI pushes into a public Activity WebSocket hub.
  - confidence: 0.9
