## Security Review

Status: needs_attention

Summary: The diff introduces concrete Activity auth, third-party script execution, token transport, and unbounded broadcast concerns that need security/owner review before merge.

Concerns:

- **[high] Activity auth can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:151`
  - body: Activity-mode Canvas/A2UI access can return authorized before normal bearer or capability auth, and the no-token path relies on spoofable launch markers from URL, cookie, or referer state.
  - confidence: 0.96
- **[high] Injected helper executes mutable CDN JavaScript:** `src/canvas-host/a2ui.ts:172`
  - body: The Canvas origin would dynamically import the Discord SDK from third-party CDNs at runtime, creating a supply-chain execution path outside normal dependency pinning and review.
  - confidence: 0.92
- **[medium] A2UI Activity fanout is unbounded:** `src/gateway/a2ui-activity.ts:240`
  - body: The hub retains the last JSONL payload and broadcasts to every WebSocket client without size or backpressure checks, creating a gateway availability risk from oversized or slow-client traffic.
  - confidence: 0.91
- **[medium] Activity token is placed in a WebSocket URL query:** `src/canvas-host/a2ui/index.html:325`
  - body: The browser bridge copies activityToken into the WebSocket query string and falls back to ws:// when the page is served over HTTP, increasing leakage risk through logs, intermediaries, or cleartext local/proxy traffic.
