Full review comments:

- **[P1] Keep Activity config in the Canvas plugin boundary:** `src/config/zod-schema.ts:657-664`
  - body: Current main moved Canvas host ownership under the Canvas plugin and treats root `canvasHost` as legacy migration input. Adding new `canvasHost.activity` runtime config here reopens the retired root surface and bypasses the plugin-owned config/migration path.
  - confidence: 0.93
- **[P1] Require real Activity credentials before bypassing auth:** `src/gateway/a2ui-activity.ts:151-154`
  - body: With Activity enabled and no configured token, this returns `true` after launch-context markers that can be supplied in query, cookies, or referer. That lets unauthenticated callers reach Canvas/A2UI before normal Gateway bearer or node-capability auth.
  - confidence: 0.96
- **[P1] Bundle or pin the Discord SDK instead of importing CDNs:** `src/canvas-host/a2ui.ts:173-175`
  - body: The injected helper executes `@discord/embedded-app-sdk` from jsdelivr or esm.sh at page runtime inside the Canvas origin. This introduces mutable third-party code execution without a pinned local dependency or enforceable integrity boundary.
  - confidence: 0.94
- **[P1] Stop putting Activity tokens in insecure WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-330`
  - body: The bridge copies `activityToken` into the WebSocket query string and falls back to `ws://` when the page is served over HTTP. That leaks bearer-like material through URLs/logs and can transmit it in cleartext on misconfigured deployments.
  - confidence: 0.91
- **[P2] Bound Activity broadcasts and slow WebSocket clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores any JSONL string as `lastJsonl` and sends it to every connected client without a size cap or `bufferedAmount` handling. A large A2UI push or slow clients can grow memory and stall the Gateway.
  - confidence: 0.9
