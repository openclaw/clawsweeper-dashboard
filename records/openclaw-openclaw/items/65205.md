## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Rebase Activity config onto the Canvas plugin:** `src/config/zod-schema.ts:660-665`
  - body: Current main has moved Canvas host config and routes under `extensions/canvas`, and root `canvasHost` is hidden legacy config for doctor migration. Adding new `canvasHost.activity.*` keys here revives the old root surface and will drift from the plugin-owned Canvas config and docs; move this under the Canvas plugin/runtime API before merge.
  - confidence: 0.96
- **[P1] Require real Activity auth before bypassing gateway auth:** `src/gateway/a2ui-activity.ts:147-154`
  - body: `authorizeCanvasRequest` returns `ok` before bearer or capability auth whenever `isCanvasActivityAccessAllowed` succeeds, but that helper returns true with no configured token after seeing spoofable query/cookie/referer launch markers. A public caller can add `instance_id`, `frame_id`, or `platform` and reach Canvas/A2UI without normal gateway auth; require a token or signed Discord assertion instead.
  - confidence: 0.97
- **[P1] Do not runtime-import the Discord SDK from CDNs:** `src/canvas-host/a2ui.ts:172-175`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from jsdelivr with an esm.sh fallback in the hosted page origin. That adds unpinned third-party code execution to every Activity Canvas page; bundle a pinned dependency through the owning plugin or otherwise use a maintainer-approved CSP/proxy design.
  - confidence: 0.94
- **[P2] Bound A2UI Activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores the full JSONL payload and sends it to every connected Activity WebSocket without a byte limit, rate limit, or `bufferedAmount` handling. Large `canvas.a2ui.pushJSONL` or slow clients can grow memory and stall the gateway; cap payloads and drop or terminate backed-up clients before exposing this path.
  - confidence: 0.91
- **[P2] Keep Activity tokens out of WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge copies `activityToken` into the WebSocket query string and falls back to `ws://` on HTTP pages. Query-string tokens leak through logs/history/referers, and the HTTP fallback sends them in cleartext; use HTTPS/WSS-only handling with a secure cookie or non-URL credential exchange.
  - confidence: 0.92
- **[P2] Reset failed SDK loads so Activity pages can recover:** `src/canvas-host/a2ui.ts:202-208`
  - body: When `OpenClaw.discord.load()` fails once, the rejected promise remains in `state._loadPromise`, so later calls re-await the same rejection even after a transient CDN/network/ready-timeout issue clears. Clear the cached promise on failure or avoid this lazy CDN loader entirely when bundling the SDK.
  - confidence: 0.86
