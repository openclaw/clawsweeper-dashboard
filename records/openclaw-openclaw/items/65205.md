## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Require real activity auth before bypassing gateway auth:** `src/gateway/a2ui-activity.ts:151-154`
  - body: `isCanvasActivityAccessAllowed` returns true with no configured token after accepting client-controlled launch markers, and `authorizeCanvasRequest` grants access before normal bearer/capability auth. Make Activity access fail closed unless a real secret or signed Discord assertion verifies the request.
  - confidence: 0.95
- **[P1] Move Activity config to the Canvas plugin boundary:** `src/config/types.gateway.ts:41-59`
  - body: Current main removed runtime top-level `canvasHost` ownership and routes Canvas through `extensions/canvas`; adding Activity config back under root Gateway types creates a competing config/source of truth. Put Activity host settings under the Canvas plugin config or a generic plugin-surface seam.
  - confidence: 0.9
- **[P1] Do not runtime-import the Discord SDK from CDNs:** `src/canvas-host/a2ui.ts:172-176`
  - body: The injected Canvas helper loads executable JavaScript from jsdelivr and esm.sh inside the gateway-served Canvas origin. Bundle or pin the SDK locally, or require an app-provided helper under an explicit CSP, before exposing this surface.
  - confidence: 0.9
- **[P2] Bound A2UI activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every connected socket without a byte cap or `bufferedAmount` guard, while `node.invoke` can forward arbitrary A2UI payloads. Enforce payload limits and drop or close slow clients.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff weakens Canvas/A2UI auth, adds unpinned remote browser code execution, and exposes availability-sensitive WebSocket broadcast behavior.

Concerns:

- **[high] Spoofable tokenless Activity auth bypass:** `src/gateway/a2ui-activity.ts:151`
  - body: Activity mode can bypass normal gateway auth without a configured token after accepting spoofable launch markers from query, cookie, or referer data.
  - confidence: 0.95

  - confidence: 0.9

  - confidence: 0.78
