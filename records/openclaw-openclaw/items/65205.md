## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Move Activity config under the Canvas plugin config:** `src/config/types.gateway.ts:38-55`
  - body: Current main treats root `canvasHost` as legacy migration input and resolves active Canvas host settings from `plugins.entries.canvas.config.host`. Adding `canvasHost.activity` creates a competing config/default path that doctor will migrate away and that does not follow the current plugin-owned Canvas surface.
  - confidence: 0.95
- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:148-156`
  - body: When Activity mode is enabled and no token is configured, spoofable query/cookie/referer launch markers let this helper return true before normal Gateway bearer or scoped capability auth runs. That exposes Canvas/A2UI routes to unauthenticated callers in the documented optional-token setup.
  - confidence: 0.96
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:169-176`
  - body: This injects runtime imports from jsdelivr and esm.sh into hosted Canvas pages, so the Canvas origin executes mutable third-party JavaScript that is not pinned by the repo lockfile or reviewed with the PR. Ship a local pinned dependency or a maintainer-approved loading model instead.
  - confidence: 0.93
- **[P1] Stop forwarding Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-330`
  - body: The bridge places `activityToken` in the WebSocket query string and uses `ws://` whenever the page is loaded over HTTP. That can leak the token through cleartext transport, logs, browser history, and intermediaries; use a secure cookie or handshake that does not put credentials in URLs.
  - confidence: 0.92
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:239-244`
  - body: The hub stores the last JSONL payload and sends every payload to every connected WebSocket without size limits or `bufferedAmount` handling. A large A2UI push or slow client can grow memory or stall the Gateway event loop, so cap payloads and terminate/drop slow clients before enabling this public bridge.
  - confidence: 0.9
- **[P2] Clear failed SDK load attempts before retrying:** `src/canvas-host/a2ui.ts:206`
  - body: If the Discord SDK import or `ready()` call fails once, the rejected promise stays in `state._loadPromise`, so later calls keep awaiting the same failure until a full page reload. Clear the cached promise on failure so transient CDN/network errors can recover.
  - confidence: 0.86
