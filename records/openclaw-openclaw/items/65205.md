## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Keep Activity config on the Canvas plugin surface:** `src/config/types.gateway.ts:41-59`
  - body: This adds Activity settings under root `canvasHost`, but current main treats root `canvasHost` as legacy input migrated into `plugins.entries.canvas.config.host`. Put Activity settings under the Canvas plugin owner or get an explicit migration/product decision before adding a new root config source.
  - confidence: 0.94
- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:150-154`
  - body: When Activity mode is enabled without a configured token, this returns `true`; `authorizeCanvasRequest` accepts that before normal Gateway bearer or capability auth. Spoofable launch markers must not authorize public Canvas/A2UI access by themselves.
  - confidence: 0.96
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper imports executable SDK code from jsDelivr and esm.sh at runtime in the Canvas origin. Use a pinned reviewed dependency or owned bundle so a CDN response cannot change the code OpenClaw executes.
  - confidence: 0.94
- **[P1] Stop sending Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge reads the token from the page query, forwards it in the WebSocket query string, and falls back to `ws://` on HTTP pages. Use a safer cookie/exchange/subprotocol path and fail closed for insecure transport.
  - confidence: 0.91
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every Activity WebSocket without byte limits or `bufferedAmount` handling. Add payload caps and slow-client behavior before exposing this bridge.
  - confidence: 0.91
- **[P2] Reset failed Discord SDK load attempts:** `src/canvas-host/a2ui.ts:166-210`
  - body: If `OpenClaw.discord.load()` fails once, the rejected promise stays in `_loadPromise`, so later calls re-await the same failure until page reload. Clear the promise on failure so transient network or ready-timeout errors can recover.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces concrete Activity auth, token transport, remote code execution, and WebSocket fanout concerns that need security/owner review before merge.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:151`
  - body: `isCanvasActivityAccessAllowed` returns true with Activity enabled and no token, and the Gateway accepts that before bearer or capability auth for Canvas/A2UI routes.
  - confidence: 0.96
- **[high] Injected helper executes mutable CDN JavaScript:** `src/canvas-host/a2ui.ts:173`
  - body: Canvas pages would dynamically import the Discord SDK from jsDelivr or esm.sh at runtime, giving third-party responses script execution in the Canvas origin.
  - confidence: 0.94
- **[medium] Activity token is copied into URL query strings:** `src/canvas-host/a2ui/index.html:325`
  - body: The A2UI bridge forwards `activityToken` in the WebSocket URL and permits `ws://` on HTTP pages, exposing credentials through logs, history, intermediaries, or cleartext transport.
  - confidence: 0.91
- **[medium] A2UI Activity fanout is unbounded:** `src/gateway/a2ui-activity.ts:240`
  - body: The hub retains and broadcasts arbitrary JSONL without size limits or slow-client backpressure, creating memory and event-loop risk for public Activity connections.
  - confidence: 0.91
