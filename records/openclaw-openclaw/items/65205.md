## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require credentials before activity auth bypass:** `src/gateway/a2ui-activity.ts:151-154`
  - body: `activity.token` is documented optional, and this branch returns `true` when it is absent after accepting spoofable query/cookie/referer launch markers as context. That lets an unauthenticated caller append `instance_id`, `frame_id`, or `platform` and reach Canvas/A2UI before normal Gateway bearer/capability auth, so Activity access needs a real token or verifiable Discord assertion before bypassing Gateway auth.
  - confidence: 0.95
- **[P1] Move Activity config to Canvas plugin config:** `src/config/zod-schema.ts:654-660`
  - body: Current main intentionally moved Canvas host settings to `plugins.entries.canvas.config.host` and keeps old root `canvasHost` only as doctor migration input. Adding `canvasHost.activity` to the root schema creates a new runtime config source that the current Canvas plugin boundary will not own, so the Activity config needs to live under the Canvas plugin surface or come with an explicit migration/product decision.
  - confidence: 0.93
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:172-176`
  - body: The injected Canvas helper imports executable SDK code from jsdelivr and then esm.sh at runtime, while the SDK is not pinned in the repository dependency graph. That gives mutable third-party CDN responses code execution in the Canvas origin and bypasses normal lockfile/supply-chain review, so the SDK should be bundled or otherwise pinned through an approved dependency path.
  - confidence: 0.94
- **[P1] Bound A2UI activity fanout:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores the full JSONL payload and sends it to every connected client with no byte cap, backpressure check, or slow-client handling. Since node.invoke can feed arbitrary A2UI payloads into this hub, a large message or slow clients can grow memory and stall the Gateway event loop.
  - confidence: 0.9
- **[P2] Avoid forwarding activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge copies `activityToken` from the page URL into the WebSocket query string and falls back to `ws://` on HTTP pages. That exposes the token to logs/intermediaries and cleartext local-network transport, so authenticated Activity bridge connections should require HTTPS/WSS and use a server-set secure cookie or another non-URL credential path.
  - confidence: 0.86
