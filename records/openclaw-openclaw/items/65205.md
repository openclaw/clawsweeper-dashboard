Full review comments:

- **[P1] Move Activity config under the Canvas plugin owner:** `src/config/zod-schema.ts:660-667`
  - body: Current main treats root `canvasHost` as legacy input and migrates it into `plugins.entries.canvas.config.host`; adding `canvasHost.activity` here creates a second runtime Canvas config surface. Please put Activity settings under the Canvas plugin config or get an explicit migration/product decision.
  - confidence: 0.94
- **[P1] Require credentials before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:150-154`
  - body: With Activity enabled and no token, this returns `true` after checking request-controlled query/cookie/referer launch markers, and `authorizeCanvasRequest` accepts it before normal bearer or capability auth. Public Activity access needs a real token or verifiable Discord assertion before exposing Canvas/A2UI.
  - confidence: 0.96
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected helper imports executable code from jsDelivr with an esm.sh fallback at runtime inside the Canvas origin. That creates a mutable third-party code execution path for a public Activity surface; use a pinned bundled dependency or an explicitly approved constrained loader.
  - confidence: 0.91
- **[P1] Stop putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The bridge reads `activityToken` from the page URL, forwards it in the WebSocket query string, and falls back to `ws://` on HTTP pages. Use WSS plus a safer server-set credential channel or token exchange so Activity credentials do not leak through URLs/logs or cleartext transport.
  - confidence: 0.9
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:240-244`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every connected Activity client without byte limits, rate limits, or `bufferedAmount` handling. Add payload caps and slow-client drop/terminate behavior before exposing this hub.
  - confidence: 0.88
