## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move Activity config off the retired root Canvas surface:** `src/config/zod-schema.ts:660-667`
  - body: Current main moved Canvas host ownership into the Canvas plugin and keeps root `canvasHost` only as legacy migration input. Adding `canvasHost.activity` here creates a new public config/default surface in the old core location, so upgraded configs using `plugins.entries.canvas.config.host` will not own the Activity settings correctly and doctor may migrate/delete the root shape. Put Activity config and route behavior under the Canvas plugin owner before merge.
  - confidence: 0.93
- **[P1] Require real Activity credentials before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:150-154`
  - body: When Activity mode is enabled without a token, `isCanvasActivityAccessAllowed` returns true after only checking ordinary query/cookie/referer launch markers, and `authorizeCanvasRequest` accepts that before bearer or node-capability auth. An external request can spoof those markers and reach Canvas/A2UI endpoints, so Activity access needs a mandatory secret or verifiable signed Discord launch assertion before this bypass is allowed.
  - confidence: 0.95
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper runtime-imports executable SDK code from mutable CDN URLs in the page origin. That adds an unpinned supply-chain execution path to every hosted Activity page; ship a bundled/pinned SDK path or an owner-approved constrained loading model before exposing this helper.
  - confidence: 0.9
- **[P1] Stop carrying Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The A2UI bridge copies `activityToken` from the page URL into the WebSocket query string and uses `ws://` on HTTP pages. That leaks tokens through URLs/logs and can send them in cleartext on misconfigured HTTP deployments; use a secure cookie or other non-URL credential path and fail closed for authenticated Activity transport without WSS.
  - confidence: 0.88
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:220-244`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every Activity WebSocket client without size limits or checking `bufferedAmount`, while `node.invoke` can stringify arbitrary object payloads into that path. Add byte caps and slow-client handling before exposing this as a public Activity bridge.
  - confidence: 0.9
- **[P2] Reset failed Discord SDK loads so helpers can recover:** `src/canvas-host/a2ui.ts:166-210`
  - body: If the dynamic SDK load or `ready()` call fails once, the rejected `state._loadPromise` is kept forever. Later calls re-await the same rejected promise instead of retrying, so a transient network or SDK readiness failure leaves `OpenClaw.discord.load()` broken until a full page reload.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces concrete auth, token-transport, and browser supply-chain risks that need owner-approved fixes before merge.

Concerns:

- **[high] Marker-only Activity auth can bypass Gateway auth:** `src/gateway/a2ui-activity.ts:150`
  - body: Activity requests can be accepted before normal Gateway auth when the token is absent and ordinary launch-context markers are present, which are not credentials.
  - confidence: 0.95
- **[high] Injected Canvas helper executes mutable CDN code:** `src/canvas-host/a2ui.ts:171`
  - body: The helper imports @discord/embedded-app-sdk from public CDN URLs at runtime in the Canvas page origin without bundling or pinning.
  - confidence: 0.9
- **[medium] Activity token is placed in WebSocket query URLs:** `src/canvas-host/a2ui/index.html:325`
  - body: The A2UI bridge forwards activityToken/a2uiToken in the WebSocket URL and allows ws:// on HTTP pages, increasing leakage and cleartext transport risk.
  - confidence: 0.88
- **[medium] A2UI Activity broadcast has no payload or backpressure bounds:** `src/gateway/a2ui-activity.ts:220`
  - body: The hub retains and sends arbitrary JSONL to every connected client without byte limits or slow-client handling, creating an avoidable denial-of-service surface.
  - confidence: 0.9
