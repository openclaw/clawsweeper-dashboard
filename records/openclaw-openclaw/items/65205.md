## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require real Activity auth before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:150-154`
  - body: With Activity mode enabled and no configured token, `isCanvasActivityAccessAllowed()` returns true after only spoofable launch markers, and `authorizeCanvasRequest()` then skips the normal bearer/capability checks for Canvas, A2UI, and WebSocket routes. Activity access needs a real shared secret or signed Discord assertion before it can bypass Gateway auth.
  - confidence: 0.96
- **[P1] Do not runtime-import the Discord SDK from CDNs:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from jsDelivr or esm.sh in the Canvas origin at runtime. That introduces unpinned third-party code execution into a Gateway-served surface; bundle or otherwise pin the SDK under OpenClaw/plugin ownership instead.
  - confidence: 0.93
- **[P1] Move Activity config into the Canvas plugin surface:** `src/config/zod-schema.ts:654-660`
  - body: Current main moved Canvas host/runtime ownership to `extensions/canvas` and migrates the old root `canvasHost` config into `plugins.entries.canvas.config.host`. Adding new `canvasHost.activity` fields on the retired root path revives the old surface and conflicts with the pluginized Canvas architecture.
  - confidence: 0.9
- **[P2] Bound A2UI Activity payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush()` stores the entire JSONL payload and sends it to every connected Activity WebSocket client without a size limit or bufferedAmount/backpressure guard. A large `canvas.a2ui.push*` result can be retained and fanned out enough to affect Gateway memory or event-loop health.
  - confidence: 0.89
- **[P2] Clear failed Discord SDK load attempts:** `src/canvas-host/a2ui.ts:202-208`
  - body: When `OpenClaw.discord.load()` fails once, the rejected promise remains in `state._loadPromise`; later calls re-await the same rejection instead of retrying after a transient CDN or ready-timeout failure. Clear the cached promise on failure so users can recover without a full page reload.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff changes Gateway auth, token handling, browser-executed SDK loading, and WebSocket fanout and needs security fixes before merge.

Concerns:

- **[high] Activity requests can bypass Gateway auth without a real secret:** `src/gateway/a2ui-activity.ts:151`
  - body: Activity mode can authorize Canvas/A2UI routes using spoofable launch markers when no activity token is configured, bypassing the normal Gateway bearer or node capability checks.
  - confidence: 0.96
- **[high] Canvas helper executes unpinned CDN code:** `src/canvas-host/a2ui.ts:173`
  - body: The injected helper imports the Discord SDK from third-party CDNs at runtime in the Canvas origin, creating a supply-chain execution path outside the repository/package lock.
  - confidence: 0.93
- **[medium] Activity token can travel in non-TLS WebSocket URLs:** `src/canvas-host/a2ui/index.html:327`
  - body: The A2UI bridge builds ws:// URLs on HTTP pages and includes the activity token in the query string, which can leak through logs or plaintext transport in misconfigured deployments.
  - confidence: 0.82
- **[medium] A2UI activity broadcasts are unbounded:** `src/gateway/a2ui-activity.ts:240`
  - body: The new hub retains and broadcasts arbitrary JSONL payloads to all connected clients without size or backpressure limits, creating a denial-of-service risk.
  - confidence: 0.89
