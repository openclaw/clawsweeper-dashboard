Full review comments:

- **[P1] Require non-spoofable Activity auth before bypass:** `src/gateway/a2ui-activity.ts:151-154`
  - body: With Activity enabled and no token, this returns `true` after accepting query/cookie/referer launch markers that any caller can spoof. Since `authorizeCanvasRequest` checks this before normal Gateway bearer or capability auth, an external request can reach Canvas/A2UI by adding those markers.
  - confidence: 0.95
- **[P1] Move Activity config under the Canvas plugin:** `src/config/types.gateway.ts:41-59`
  - body: Current main moved Canvas host config to the Canvas plugin and hides root `canvasHost` from the public schema except for migration. Adding `canvasHost.activity` creates a new public root config path that bypasses the shipped plugin-owned owner boundary.
  - confidence: 0.91
- **[P1] Bundle or pin the Discord Activity SDK:** `src/canvas-host/a2ui.ts:173-175`
  - body: The injected Canvas helper loads executable code from jsDelivr with an esm.sh fallback at runtime. That gives a third-party CDN response code execution in the Canvas origin, so this should be bundled/pinned or guarded by an explicit maintainer-approved CSP/loading design.
  - confidence: 0.91
- **[P2] Bound Activity A2UI broadcasts:** `src/gateway/a2ui-activity.ts:240-243`
  - body: The hub stores the last JSONL payload and broadcasts every payload to all Activity WebSocket clients without payload size or slow-client checks. A large node A2UI push can therefore create memory or event-loop pressure on the gateway.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security and supply-chain concerns around Activity auth bypass, unpinned SDK execution, token transport, and unbounded broadcast fanout.

Concerns:

- **[high] Spoofable Activity markers bypass Gateway auth:** `src/gateway/a2ui-activity.ts:151`
  - body: Activity access can return true without a configured token after checking only caller-controlled query, cookie, or referer launch markers, and Gateway auth accepts that before normal bearer/capability checks.
  - confidence: 0.95
- **[high] Injected helper executes CDN JavaScript:** `src/canvas-host/a2ui.ts:173`
  - body: The proposed Canvas helper imports the Discord embedded SDK from jsDelivr with an esm.sh fallback at runtime, adding unpinned third-party code execution to the Canvas origin.
  - confidence: 0.91
- **[medium] Activity token travels in URL query strings:** `src/canvas-host/a2ui/index.html:325`
  - body: The A2UI bridge copies activityToken into the WebSocket URL and can use ws:// on HTTP pages, exposing credentials through URL logs or cleartext transport in misconfigured deployments.
  - confidence: 0.86
- **[medium] Unbounded A2UI Activity fanout:** `src/gateway/a2ui-activity.ts:240`
  - body: The hub retains and broadcasts arbitrary JSONL without size limits or backpressure checks, creating a resource-exhaustion path once Activity WebSocket access is exposed.
  - confidence: 0.88
