## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require trusted Activity auth before bypassing Canvas auth:** `src/gateway/a2ui-activity.ts:151-154`
  - body: With Activity enabled and no token, any non-empty launch marker from the request, referrer, or cookie lets this path return true before normal bearer or Canvas-capability auth. Current main rejects unauthenticated Canvas/A2UI HTTP and WS by default, so this exposes the Canvas host on public gateway deployments.
  - confidence: 0.95
- **[P1] Keep Activity config out of legacy root canvasHost:** `src/config/zod-schema.ts:660-667`
  - body: Current main migrates root `canvasHost` into Canvas plugin config and hides the root key from public config surfaces. Adding `canvasHost.activity` here creates runtime/default surface in the legacy shape instead of `plugins.entries.canvas.config.host`, so upgrades and generated config will drift from the canonical Canvas owner.
  - confidence: 0.93
- **[P1] Bundle or pin the Discord Activity SDK:** `src/canvas-host/a2ui.ts:171-176`
  - body: This injects dynamic imports from jsDelivr and esm.sh into the Canvas origin. A mutable third-party CDN response can execute with OpenClaw helper privileges, so the SDK should come from an owned, pinned package path or an explicit maintainer-approved supply-chain model.
  - confidence: 0.9
- **[P2] Bound A2UI Activity fanout before broadcasting:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every Activity socket with no maxPayload or backpressure policy, while `node.invoke` can stringify arbitrary objects into this path. Existing gateway and Canvas WebSockets enforce payload and buffer limits; add equivalent bounds before exposing this route.
  - confidence: 0.86
- **[P2] Reset failed SDK load attempts so Activities can retry:** `src/canvas-host/a2ui.ts:202-208`
  - body: If `OpenClaw.discord.load()` fails once, the rejected `_loadPromise` is kept and every later call re-awaits the same failure. Clear the promise on failure so transient CDN, network, or ready-timeout errors can recover without a full page reload.
  - confidence: 0.82
