Full review comments:

- **[P1] Move Activity config under the Canvas plugin owner:** `src/config/types.gateway.ts:41-59`
  - body: This adds `canvasHost.activity` to the root config even though current main treats root `canvasHost` as legacy input migrated into `plugins.entries.canvas.config.host` and hides it from the public schema. Put Activity settings under the Canvas plugin config or get an explicit migration/product decision before exposing a new root surface.
  - confidence: 0.95
- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:147-154`
  - body: With Activity enabled, this returns `true` when no token is configured, and `authorizeCanvasRequest` accepts the Activity path before normal Gateway bearer or node-capability auth. Activity access should fail closed unless a token or verifiable Discord assertion is configured and tested.
  - confidence: 0.97
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from mutable third-party CDNs inside the Canvas origin, outside the lockfile and package review path. Add the SDK as a reviewed dependency or otherwise pin and own the executable source before exposing this in production.
  - confidence: 0.94
- **[P1] Stop putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The A2UI bridge reads `activityToken` from the page URL, forwards it in the WebSocket query string, and falls back to `ws://` for HTTP pages. Use a safer cookie/header/exchange path and fail closed for insecure transport when the Activity bridge is authenticated.
  - confidence: 0.9
- **[P2] Bound Activity bridge payloads and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every Activity WebSocket without byte limits or `bufferedAmount` handling. Add payload caps and slow-client drop/termination behavior before this bridge can be exposed to public Activity clients.
  - confidence: 0.92
