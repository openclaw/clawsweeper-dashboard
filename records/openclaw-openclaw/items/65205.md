## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:149-154`
  - body: When Activity mode is enabled without `activity.token`, this returns `true` after only query/cookie/referer launch-context markers are present. Those markers are caller-controlled, so an unauthenticated request can reach Canvas/A2UI before normal Gateway bearer or node-capability auth.
  - confidence: 0.94
- **[P1] Keep Activity config under the Canvas plugin:** `src/config/types.gateway.ts:41-59`
  - body: Current main moved Canvas host config to `plugins.entries.canvas.config.host` and hides root `canvasHost` as legacy migration input, but this adds new runtime fields under root `canvasHost.activity`. That reopens a retired config surface and bypasses the Canvas plugin owner boundary.
  - confidence: 0.92
- **[P1] Bundle the Discord SDK instead of loading CDNs:** `src/canvas-host/a2ui.ts:171-176`
  - body: The injected Canvas helper falls back to importing executable code from jsDelivr and esm.sh at runtime. That code runs in the Canvas origin outside the lockfile and package review path, so production Activity pages would trust mutable third-party CDN responses.
  - confidence: 0.9
- **[P1] Stop putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge reads `activityToken` from the page URL and appends it to the Activity WebSocket URL, using `ws://` whenever the page is served over HTTP. This leaks credentials through query logs/history/intermediaries and can send them in cleartext on a misconfigured host.
  - confidence: 0.9
- **[P2] Bound Activity bridge payloads and slow clients:** `src/gateway/a2ui-activity.ts:236-242`
  - body: `broadcastPush` stores the full JSONL payload and sends it to every connected socket without a byte limit or `bufferedAmount`/slow-client handling. A large `canvas.a2ui.pushJSONL` or object stringification can retain large data and stall the gateway while broadcasting.
  - confidence: 0.86
- **[P2] Reset failed SDK loads so callers can retry:** `src/canvas-host/a2ui.ts:202-210`
  - body: If the Discord SDK import or `ready()` call fails once, the rejected `_loadPromise` is kept and every later `OpenClaw.discord.load()` call re-awaits the same failure. Transient CDN/network failures therefore require a full page reload instead of allowing retry.
  - confidence: 0.82

## Security Review
