## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep Activity config under the Canvas plugin:** `src/config/zod-schema.ts:660`
  - body: Current main moved Canvas host config to `plugins.entries.canvas.config.host` and hides root `canvasHost` as legacy migration input. Adding `canvasHost.activity` here reopens a retired root runtime surface and conflicts with the Canvas plugin ownership model, so existing upgrade/migration behavior is ambiguous.
  - confidence: 0.94
- **[P1] Require real credentials before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:147-154`
  - body: When Activity mode is enabled, this returns true without any configured token after only query/cookie/referer launch markers. Those markers are client-controlled, so an unauthenticated caller can reach Canvas/A2UI routes before normal Gateway auth by spoofing the launch context.
  - confidence: 0.96
- **[P1] Bundle the Discord SDK instead of importing mutable CDNs:** `src/canvas-host/a2ui.ts:173-175`
  - body: The injected Canvas helper executes `@discord/embedded-app-sdk` from public CDNs at runtime inside the Canvas origin. That bypasses the lockfile, package review, and normal dependency pinning path for browser-executed code.
  - confidence: 0.93
- **[P1] Stop sending Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The A2UI bridge copies `activityToken` from the page URL into the WebSocket query string and downgrades to `ws://` on HTTP pages. That exposes credentials to logs/history/intermediaries and can send them in cleartext if the public Activity host is misconfigured.
  - confidence: 0.91
- **[P2] Bound A2UI Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` retains the full last JSONL payload and sends it to every client without a byte limit or `bufferedAmount`/slow-client handling. A large node A2UI payload or slow connected clients can make Activity mode consume unbounded memory and event-loop time.
  - confidence: 0.88
- **[P2] Clear failed SDK load promises so callers can retry:** `src/canvas-host/a2ui.ts:167-208`
  - body: If the CDN import or `DiscordSDK.ready()` fails once, the rejected `_loadPromise` remains cached and every later `OpenClaw.discord.load()` awaits the same failure until page reload. Reset the promise on failure so transient network or Discord iframe readiness errors can recover.
  - confidence: 0.84
