Overall confidence: 0.92

Full review comments:

- **[P1] Require credentials before Activity auth bypass:** `src/gateway/a2ui-activity.ts:151-154`
  - body: When Activity mode is enabled without a configured token, this returns `true` after only launch-context checks. Since `authorizeCanvasRequest` accepts this before normal bearer or node-capability auth, a spoofed Activity marker can expose Canvas/A2UI; require a token or verifiable Discord assertion before granting this bypass.
  - confidence: 0.96
- **[P1] Move Activity config under the Canvas plugin owner:** `src/config/types.gateway.ts:41-59`
  - body: Current main migrates root `canvasHost` into `plugins.entries.canvas.config.host` and deletes the root key, but this adds a new root `canvasHost.activity` surface. Put Activity settings on the Canvas plugin-owned config surface or get an explicit migration/product decision before adding a competing config path.
  - confidence: 0.94
- **[P1] Bundle or pin the Discord SDK before injecting it:** `src/canvas-host/a2ui.ts:171-175`
  - body: The injected helper imports executable SDK code from mutable CDN URLs inside the Canvas origin, and the PR does not add a reviewed package dependency or lockfile pin. Bundle the SDK as an owned dependency or otherwise pin and constrain the load path before exposing this helper.
  - confidence: 0.92
- **[P1] Stop carrying Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:325-329`
  - body: The browser bridge reads the Activity token from the page query string, forwards it in the WebSocket URL, and uses `ws://` on HTTP pages. Use a safer cookie/exchange/protocol path and fail closed for insecure transport so tokens do not leak through logs, intermediaries, or cleartext connections.
  - confidence: 0.9
- **[P2] Bound Activity broadcasts and slow clients:** `src/gateway/a2ui-activity.ts:240-243`
  - body: `broadcastPush` stores arbitrary JSONL and sends it to every Activity WebSocket without byte limits or `bufferedAmount` handling. Add payload caps and slow-client behavior before exposing this public Activity bridge.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces concrete auth, token-transport, supply-chain script-loading, and DoS risks in the Activity Canvas path.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:151`
  - body: `isCanvasActivityAccessAllowed` returns true when Activity mode is enabled without a configured token, allowing the pre-auth Canvas path to skip normal bearer or node-capability checks.
  - confidence: 0.96
- **[high] Canvas helper loads executable SDK code from CDNs:** `src/canvas-host/a2ui.ts:171`
  - body: The injected helper imports `@discord/embedded-app-sdk` from mutable CDN URLs at runtime inside the Canvas origin, creating a supply-chain script execution risk.
  - confidence: 0.92

  - body: The browser bridge reads the Activity token from the page URL and forwards it in the WebSocket query string, which can leak through logs, intermediaries, browser history, or non-TLS deployments.
  - confidence: 0.9

  - body: The Activity hub retains and broadcasts arbitrary JSONL without byte limits or slow-client backpressure, creating memory and event-loop risk for public Activity connections.
