## Summary

Keep this PR open. Current main and the latest release still contain the unsafe `trim().toLowerCase()` calls, and the PR's guard is a narrow, correct hardening for the OpenAI Codex legacy alias path; it still needs normal maintainer review and preferably focused regression coverage before merge.

## What This Changes

The PR adds runtime `typeof ... === "string"` guards before trimming/lowercasing `model.id` and `model.name` in the OpenAI Codex legacy alias canonicalizer.

## Best Possible Solution

Land this guard or an equivalent maintainer patch with a focused regression test for malformed OpenAI Codex resolved-model metadata, then keep the linked bootstrap-log issue open until the actual runtime source is confirmed or any adjacent stored-override hardening is handled.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by passing an `openai-codex` resolved model with undefined or non-string `id` into the `normalizeResolvedModel` path; current main reaches `params.model.id.trim()` at `src/agents/pi-embedded-runner/model.ts:144`, and a string alias id with malformed `name` reaches line 152.

## Solution Assessment

Yes for the canonicalizer crash path. The PR's `typeof` guards are the narrowest maintainable fix for these two string-method calls, but maintainers should not treat it as full proof for the linked issue until a regression test or runtime reproducer confirms the crashing source.
