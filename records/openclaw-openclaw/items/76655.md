## Summary

Keep this PR open: the cache approach is useful, but the latest head still misses the Gateway workspace-scoped metadata snapshot in the runtime-skipped startup path, so the intended hot-path cache can be bypassed.

## What This Changes

The PR adds snapshot-keyed caches for plugin manifest model-id normalization and bundled setup CLI backend metadata, threads manifest plugin metadata through model normalization/pricing helpers, and updates tests plus changelog.

## Best Possible Solution

Keep the snapshot-keyed cache design, but make the normal Gateway startup path preserve or pass defaultWorkspaceDir into both cache helpers and add regression coverage for runtime-skipped startup.

## Reproduction Assessment

Yes, source-level. The PR head helpers omit the Gateway workspace when runtime plugins are skipped, and the current metadata snapshot rejects workspace-scoped snapshots when callers omit workspaceDir.

## Solution Assessment

No. The cache direction is maintainable, but it is not the best fix until the Gateway workspace-compatible snapshot is used by the new hot-path helpers.
