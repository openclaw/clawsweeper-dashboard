## Summary

## What This Changes

The PR adds module-level caches for manifest model-id normalization policy collection and bundled setup CLI backend fallback lookup, plus adjusts the related unit tests.

## Best Possible Solution

## Solution Assessment

No, not as currently proposed. Caching is the right direction, but unkeyed module-level caches with test-only invalidation are not the narrow maintainable solution for plugin metadata that already has a snapshot/reload lifecycle.
