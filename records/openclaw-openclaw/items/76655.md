## Summary

Keep this PR open. Current main still has the source-level sessions.list performance path the PR targets, but the patch introduces stale module-global caches and misses the required user-facing changelog entry.

## What This Changes

The PR adds process-local caches for manifest model-id normalization policies and bundled setup CLI backend metadata, with related unit-test expectation changes.

## Best Possible Solution

Keep the narrow metadata caching approach, but make both caches lifecycle-aware or fingerprinted, restore stale-cache regression coverage, and add one Unreleased changelog line before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes: current main loops through session rows and reaches synchronous plugin metadata lookup helpers; live profiling was not run in this read-only pass.

## Solution Assessment

Is this the best way to solve the issue? No, not yet. Caching is the right direction, but module-global caches need production invalidation or metadata fingerprinting before this is the narrowest maintainable fix.
