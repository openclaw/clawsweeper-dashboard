## Summary

Keep open. The reported sessions.list performance problem is source-reproducible on current main, but this PR's module-global metadata caches can go stale across manifest edits, state-dir changes, or Gateway plugin reloads, and it is missing the required changelog entry.

## What This Changes

This PR adds module-level caches for manifest model-id normalization policies and bundled setup CLI backend metadata, with tests updated for reduced metadata loads.

## Best Possible Solution

Use a lifecycle-owned metadata snapshot or fingerprinted invalidation boundary for the expensive lookups, preserve freshness regression coverage, and document the user-visible performance fix in the changelog.

## Reproduction Assessment

Yes, at source level: current main loops over every sessions.list row and reaches model normalization and CLI provider helpers that load plugin manifest metadata. I did not run live CPU profiling in this read-only review.

## Solution Assessment

No, not as currently proposed. Caching is the right direction, but persistent module globals are not the narrowest maintainable fix unless they are keyed to, or invalidated with, the plugin metadata lifecycle.
