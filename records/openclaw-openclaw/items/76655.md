## Summary

Keep open: current main still has the source-level sessions.list metadata lookup hotspot, but this PR adds persistent module-level caches without production lifecycle invalidation and omits the required changelog entry, so it needs a narrow repair before merge.

## What This Changes

The PR adds module-level caches for manifest model-id normalization policies and bundled setup CLI backend metadata, with tests changed for manual cache invalidation and fewer metadata snapshot loads.

## Best Possible Solution

Keep the narrow metadata caches, but make them lifecycle-aware by keying or invalidating against the plugin metadata control-plane snapshot, preserve stale-cache regression coverage, and add one Unreleased changelog bullet.

## Reproduction Assessment

The PR's large-history Control UI polling reproduction is plausible but not independently executed here.

## Solution Assessment

Is this the best way to solve the issue? No, not as submitted. Caching is the right direction, but persistent module globals need production invalidation/fingerprinting and the required changelog before this is the narrowest maintainable fix.
