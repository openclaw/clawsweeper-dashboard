## Summary

Keep open. The PR targets a real `sessions.list` hot path and the cache direction is useful, but the current diff still misses Gateway's workspace-scoped metadata snapshot, so the reported cold manifest-scan cost can remain on the row path.

## What This Changes

The PR adds metadata-snapshot-keyed caches for manifest model-id normalization and bundled setup CLI backend lookup, plus tests and a changelog entry.

## Best Possible Solution

Keep the caching approach, but make both helpers consume the active workspace-compatible Gateway metadata snapshot or an explicit snapshot/lookup table, then cover the workspace-scoped case before merge.

## Reproduction Assessment

Yes, source-level. Gateway publishes a workspace-scoped current metadata snapshot, `getCurrentPluginMetadataSnapshot` rejects omitted workspace scope, and the PR calls it without `workspaceDir` in both hot-path cache helpers.

## Solution Assessment

No. Caching against the active metadata snapshot is the right direction, but this diff must use the same workspace-compatible snapshot that Gateway actually publishes instead of falling back to cold metadata loads.
