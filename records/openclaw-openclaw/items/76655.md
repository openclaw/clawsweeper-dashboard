## Summary

Keep open. The sessions-list performance problem is source-reproducible, but this PR still adds metadata-derived module-global caches without production lifecycle invalidation and it lacks the required changelog entry.

## What This Changes

The PR adds module-level caches for manifest model-id normalization policies and bundled setup CLI backend metadata, adjusts tests around those caches, and awaits an async TUI completion test.

## Best Possible Solution

Tie these metadata-derived lookups to the existing plugin metadata snapshot or fingerprinted reload lifecycle, preserve stale-cache regression coverage, and add a single Unreleased changelog entry for the Control UI/Gateway performance fix.

## Reproduction Assessment

Source-level yes, live profiling not run: current main loops over session rows and reaches manifest normalization paths that rescan plugin manifests. The large-history Control UI polling profile described by the PR is plausible, but this review did not execute it.

## Solution Assessment

No, not as currently proposed. Caching is the right direction, but module-global caches with test-only invalidation are not the narrow maintainable fix for metadata that changes on plugin reload, config changes, and state-dir changes.
