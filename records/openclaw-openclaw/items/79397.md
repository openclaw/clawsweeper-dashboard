## Summary

Keep open: the reported Nextcloud Talk bug is real and the patch direction is good, but latest head still drops visible non-bot user mentions from the text sent to the agent.

## What This Changes

The PR parses Nextcloud Talk structured webhook bodies, uses extracted text for mention and command dispatch, forces automatic source reply delivery for room replies, adds parser tests, and updates the changelog.

## Best Possible Solution

Land the focused Nextcloud Talk fix after reconstructing structured text so only the bot mention is removed while non-bot user mentions and rich objects remain visible to the agent.

## Reproduction Assessment

Yes. Current main is source-reproducible: it uses `rawBody` from `object.content` for command detection, mention matching, and agent context even though Talk sends a JSON `message`/`parameters` envelope.

## Solution Assessment

No. The PR's approach is maintainable, but the parser must distinguish the configured bot mention from other user mentions before merge.
