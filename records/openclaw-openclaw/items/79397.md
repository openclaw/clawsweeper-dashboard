## Summary

Keep open: the reported Nextcloud Talk structured-body bug is valid and the PR includes useful live proof, but the current patch has merge-blocking issues in rich-object rendering and mention identity matching.

## What This Changes

The PR adds structured Nextcloud Talk body parsing, explicit mention resolution, focused mention-parser tests, and a changelog entry for group-room mention handling.

## Best Possible Solution

Preserve the structured-body fix, render non-mention rich objects into the agent-visible text, restrict explicit mention matching to stable bot identifiers, and then validate with focused parser plus inbound dispatch coverage.

## Reproduction Assessment

Yes. Current main source passes the raw JSON `object.content` through command gating, mention matching, and agent context, while Nextcloud Talk documents that bot webhooks send a JSON-encoded `message`/`parameters` payload.

## Solution Assessment

No. Parsing structured bodies is the right direction, but this implementation drops non-mention rich-object text and treats a UI account label as a mention identity; the delivery-mode claim also needs reconciliation with current source-reply defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve non-mention rich object text:** `extensions/nextcloud-talk/src/inbound.ts:88`
  - body: The parser removes every `{key}` listed in `parameters`, but Talk webhooks use the same placeholder mechanism for non-user rich objects such as calls, files, and links whose `name` is rendered in the chat view. With this change a message like `look at {mention-call1}` is delivered to the agent as `look at`, losing content the user saw; strip only the explicit bot mention and substitute other placeholders with their display text.
  - confidence: 0.92
- **[P2] Stop treating account labels as mentions:** `extensions/nextcloud-talk/src/inbound.ts:111`
  - body: `account.name` is documented as a CLI/UI display label, but adding it to `expectedIds` lets any allowed room message that mentions another user with the same display name satisfy `requireMention`. Please match explicit mentions only against stable Nextcloud identifiers such as account id, `apiUser`, and mention-id fields unless a separate bot display-name setting is introduced.
  - confidence: 0.9

## Security Review
