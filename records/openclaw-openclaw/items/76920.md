## Summary

Keep this PR open: the forum-topic fail-closed direction and cron/heartbeat route preservation are well aligned with the reported Telegram issue, but the current head still broadens direct/DM topic fallback and the external real-behavior proof gate is not satisfied.

## What This Changes

The PR narrows Telegram thread-not-found fallback for forum topics, adds Telegram send/draft tests, preserves heartbeat thread IDs for group routes, and seeds explicit cron announce routes into isolated-session delivery context.

## Best Possible Solution

Land a revised patch that keeps explicit non-General forum topic sends fail-closed, preserves the cron/heartbeat topic route fixes, keeps intentional DM-topic sends fail-closed or proves a safe incidental-DM fallback boundary, and includes live Telegram proof or a maintainer override.

## Reproduction Assessment

Yes, source-level: current main retries explicit group/forum sends threadless after thread-not-found, and the contributor supplied base-fails/head-passes test evidence. I did not run tests or a live Telegram probe because this review is read-only.

## Solution Assessment

No, this is not merge-ready as written. The forum fail-closed and cron/heartbeat routing direction is narrow, but direct/private topic fallback should not be broadened without preserving topic metadata, and the live Telegram proof gate remains unsatisfied.
