## Summary

Keep open: the forum-topic and cron routing direction is useful, but this PR is not merge-ready because it reopens DM-topic threadless fallback that current main deliberately made fail-closed, and it still lacks after-fix live Telegram proof.

## What This Changes

The PR changes Telegram outbound thread-not-found fallback policy, adds forum-topic send/draft tests, preserves cron topic delivery context, and carries heartbeat thread IDs for plugin-owned group routes.

## Best Possible Solution

Keep the forum fail-closed and cron/heartbeat topic preservation pieces, restore current-main DM-topic fail-closed semantics, then add live Telegram forum proof or obtain an explicit maintainer override.

## Reproduction Assessment

Yes, from source inspection: current main retries explicit forum sends threadless, and the PR head shows base-fails/head-passes test proof for that boundary. The new DM-topic regression is also source-visible in the changed fallback helper and tests; I did not run a live Telegram probe in this read-only review.

## Solution Assessment

No, not as written. The forum-topic fail-closed direction is the right narrow fix, but the PR should not undo the merged DM-topic fail-closed behavior and still needs live Telegram proof or a maintainer override before merge.
