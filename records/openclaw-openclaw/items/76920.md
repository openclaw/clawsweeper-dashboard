## Summary

Keep open. The forum-topic fail-closed and cron/heartbeat route preservation work is useful, but the current PR head still enables threadless fallback for direct/DM topic sends and the external live Telegram proof gate remains unsatisfied.

## What This Changes

This PR narrows Telegram thread-not-found fallback for forum topics, preserves heartbeat/cron topic route context, and adds focused regression coverage plus a changelog entry.

## Best Possible Solution

Keep the forum-topic fail-closed and cron/heartbeat preservation pieces, but keep DM-topic sends fail-closed and land only after live Telegram proof or an explicit maintainer override.

## Reproduction Assessment

Yes. Source inspection and the author's base-fails/head-passes test evidence show the current forum threadless fallback bug boundary, and the PR's DM-topic regression is visible in the changed helper and tests.

## Solution Assessment

No, this is not merge-ready as written. The forum fail-closed direction is the right narrow fix, but direct/DM topic sends should not retry without topic metadata and the live Telegram proof gate is still only covered by synthetic tests.
