## Summary

Keep open: the timeout mitigation is small and relevant, but it changes Slack Socket Mode defaults for unconfigured users without aligning the public config contract, relies on stale reconnect rationale from before native reconnect was enabled, and still lacks after-fix real Slack proof.

## What This Changes

The PR raises Slack Socket Mode default client ping timeout from 15s to 30s, adds a 60s default server ping timeout, and updates the interop test expectation.

## Best Possible Solution

Land only after maintainers decide the new Slack Socket Mode defaults are intentional, align docs/generated metadata/tests, and add real Slack proof; otherwise keep serverPingTimeout pass-through-only and use per-workspace config for this mitigation.

## Solution Assessment

No; this is at best an acceptable mitigation once the default contract and proof gaps are fixed. Current main already has native reconnect enabled through https://github.com/openclaw/openclaw/pull/78158, so the PR needs updated rationale before changing defaults globally.
