## What This Changes

The PR replaces two `crypto.createHash("sha1")` calls with `crypto.createHash("sha256")` in bundled MCP catalog fingerprinting and the media-understanding temp-file test cache helper.

## Best Possible Solution

Land the scoped cleanup after fresh checks pass, and keep unrelated contract-specific SHA-1 uses out of this PR.

## Reproduction Assessment

Not applicable: this is a cleanup PR rather than a bug report. Source inspection on current main confirms the two targeted SHA-1 call sites remain and their consumers treat the digest as an opaque cache or fingerprint string.

## Solution Assessment

Yes for the scoped change: SHA-256 preserves the `createHash(...).digest("hex")` string contract, and the affected callers only compare or embed the value. It should not be framed as repo-wide SHA-1 removal because other contract-specific SHA-1 usages remain.
