## Summary

Keep this PR open: current main and the latest release still use SHA-1 at both targeted call sites, and the proposed two-line SHA-256 swap appears narrow, compatible, and non-security-sensitive.

## What This Changes

The PR changes the MCP catalog fingerprint helper and the media-understanding test temp-file hash helper from SHA-1 to SHA-256.

## Best Possible Solution

Merge or apply this narrow cleanup if maintainers want these internal opaque hash helpers aligned on SHA-256; track any broader SHA-1 audit separately.

## Reproduction Assessment

Not applicable as a cleanup PR rather than a bug report. Source inspection provides high-confidence verification that current main still has both SHA-1 calls and that SHA-256 preserves the existing string digest contract.

## Solution Assessment

Yes for the stated two-call cleanup: SHA-256 is a drop-in replacement for these non-persisted comparison/cache keys. It should not be treated as a complete repository-wide SHA-1 removal because other protocol, extension, and script-specific SHA-1 uses remain.
