## Summary

Keep open. Current main still builds Mattermost inbound media only from the WebSocket post's `file_ids`, and empty or missing IDs return no media; the earlier direct fix PR was closed unmerged, while the newer related Mattermost hydration PR is still open and not landed.

## What This Changes

The issue requests a Mattermost attachment fix so incoming WebSocket posts with empty or missing `file_ids` are re-fetched or hydrated before media resolution.

## Best Possible Solution

Land a Mattermost-plugin-local hydration or bounded re-fetch before media resolution for incomplete WebSocket posts, preserving debounced text and adding focused regression plus live Mattermost proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Mattermost `posted` payload whose parsed post has `file_ids: []` or no `file_ids`; current main passes that value to `resolveMattermostMedia`, which returns no media. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

Yes. A bounded plugin-local `GET /api/v4/posts/{post_id}` hydration before `resolveMattermostMedia` is the narrow maintainable fix; the open related PR should be reviewed or repaired before creating duplicate work.
