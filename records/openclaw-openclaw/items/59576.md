- `impact:message-loss`: Attached files are silently absent from the agent's inbound media context when Mattermost emits empty `file_ids`.

## Summary

Keep open. Current main and v2026.5.18 still resolve inbound Mattermost media only from the WebSocket post's `file_ids`, so an empty `file_ids` payload yields no media; the direct fix PR is closed unmerged and related hydration/thread PRs remain open.

## What This Changes

The issue reports that inbound Mattermost file attachments are missed when `posted` WebSocket events carry empty `file_ids` before REST post state is hydrated.

## Best Possible Solution

Land a Mattermost-plugin-local post hydration path before inbound media resolution, with bounded retry behavior, regression coverage for empty `file_ids`, and redacted live Mattermost proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Mattermost `posted` payload parsed as `MattermostPost` with `file_ids: []` or missing; current main passes that value to `resolveMattermostMedia`, which returns no media.

## Solution Assessment

Yes in direction. A bounded plugin-local `GET /api/v4/posts/{post_id}` hydration before media resolution is the narrow maintainable fix, but it should be coordinated with the open Mattermost hydration/thread PRs and validated for latency.
