## Summary

Keep open. Current main and the latest release still build Mattermost inbound media from the WebSocket post's `file_ids`, so an empty array still produces no attachment media; the direct fix PR is closed unmerged and the newer related Mattermost PR remains open.

## What This Changes

The issue reports that Mattermost inbound attachments are missed when WebSocket `posted` events arrive with empty `file_ids`.

## Best Possible Solution

Land a Mattermost-plugin-local hydration or bounded re-fetch before media resolution for incomplete WebSocket posts, preserving debounced text and adding focused regression plus redacted live Mattermost proof.

## Reproduction Assessment

Yes, from source inspection: a parsed Mattermost `posted` payload with `file_ids: []` reaches `resolveMattermostMedia(post.file_ids)`, and that helper returns no media for an empty list. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

Yes. A bounded plugin-local REST hydration using Mattermost's existing post endpoint before media resolution is the narrow maintainable fix; the open related PR should be reviewed or repaired before creating duplicate work.
