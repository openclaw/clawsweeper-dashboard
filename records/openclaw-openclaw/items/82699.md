## Summary

Keep open: the PR fixes the right raw-config CLI path, but the new capability-web target filter still accepts every plugin SecretRef target under `plugins.entries.*`, so web commands can resolve or fail on unrelated plugin credentials. The supplied terminal proof also exercises a helper script, not the actual CLI path or the final narrowed helper.

## What This Changes

This PR routes `infer web search` and `infer web fetch` through command-time SecretRef resolution, adds a capability-web target helper, focused tests, a coverage allowlist entry, and a repro script.

## Best Possible Solution

Resolve only exact web search/fetch credential target ids before local web commands, preserve selected-provider inactive semantics, and prove the fix with a redacted actual CLI web search or fetch run.

## Reproduction Assessment

Yes. Source inspection shows current main reads raw config for the local CLI web search/fetch commands, and the linked issue gives a concrete failing `openclaw infer web search --provider tavily ... --json` path with documented SecretRef semantics.

## Solution Assessment

No. Command-time secret resolution is the right direction, but the helper must drop the broad `plugins.entries.` prefix and rely on exact web credential targets before this is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Limit capability web targets to web credentials:** `src/cli/command-secret-targets.ts:65`
  - body: The new capability-web filter still includes the broad `plugins.entries.` prefix, so any bundled plugin SecretInput registry id can be selected for `infer web search` or `infer web fetch`. Non-web plugin refs such as voice-call Twilio or realtime API keys can then be resolved or fail before the selected web provider runs; remove the broad prefix and add a non-web plugin SecretInput regression.
  - confidence: 0.91
