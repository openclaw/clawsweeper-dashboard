## Summary

Keep this PR open: it targets a real current-main CLI SecretRef bug, but the patch resolves the full agent-runtime target set for web-only commands, which can fail on unrelated active model/skill/TTS SecretRefs and broadens credential materialization unnecessarily. The supplied after-fix proof is also only mocked/unit-test output, so it is not enough for an external PR merge gate.

## What This Changes

The PR routes local `infer web search` and `infer web fetch` through command secret resolution and adds CLI/coverage tests for plugin-scoped web SecretRefs.

## Best Possible Solution

Resolve only active web search/fetch credential paths for the selected provider, keep non-selected web providers inactive/non-fatal, add regression coverage for unrelated unresolved model SecretRefs, and include redacted real CLI/gateway proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main reads raw config for local web search/fetch, while the linked issue provides a concrete SecretRef config and command that should be resolved through the gateway snapshot.

## Solution Assessment

No, not as written. The direction is right, but the command should use a web-only target set or selected allowed paths instead of the full agent-runtime target set.
