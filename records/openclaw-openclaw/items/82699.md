## Summary

Keep open: the fix is aimed at the right CLI SecretRef bug, but the patch still routes web-only commands through the full agent-runtime secret target set, so unrelated unresolved model, memory, TTS, or skill refs can abort web search/fetch before the selected web provider runs.

## What This Changes

The PR routes `infer web search` and `infer web fetch` through command secret resolution, adds mocked CLI coverage and a coverage allowlist entry, and adds a repro script for Tavily SecretRef resolution.

## Best Possible Solution

Keep the command-time SecretRef resolution, but scope `infer web search` and `infer web fetch` to active web search/fetch credential paths for the selected or auto-selected provider instead of the full agent-runtime target set.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main reads raw config for local web search/fetch while the documented web SecretRef surface requires selected-provider command-time resolution; I did not execute the CLI in this read-only review.

## Solution Assessment

No, not as written. Resolving before invoking the web runtime is the right direction, but the target set must be web-only and provider-scoped rather than the full agent-runtime target set.

## Review Findings

Overall correctness: patch is incorrect
