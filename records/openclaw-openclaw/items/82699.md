## Summary

Keep open: the PR addresses a real CLI SecretRef gap, but the current diff is not mergeable because web commands resolve the full agent-runtime secret target set and can fail on unrelated model/skill/TTS SecretRefs. The supplied proof is unit/mock-only, so it also needs after-fix real behavior proof before merge.

## What This Changes

The PR routes local `infer web search` and `infer web fetch` through command secret resolution and adds mocked CLI coverage plus the secret-resolution coverage allowlist entry.

## Best Possible Solution

Resolve only active web search/fetch credential paths for the selected provider, keep non-selected web providers inactive, and prove the result with a redacted real CLI/gateway run.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main reads raw config for local web search/fetch while the documented plugin webSearch SecretRef surface requires command-time resolution. I did not run the live gateway command because this review is read-only.

## Solution Assessment

No, not as written. The direction is right, but the fix should use a web-only target set or selected allowed paths instead of the full agent-runtime target set.
