## Summary

Keep open. The PR addresses a real CLI SecretRef bug and now has sufficient terminal proof, but the implementation still resolves the full agent-runtime secret target set for web-only commands, which can require or materialize unrelated model, memory, TTS, or skill credentials before web search/fetch runs.

## What This Changes

The PR routes CLI web search and web fetch through command-time SecretRef resolution before invoking the local web runtime.

## Best Possible Solution

Keep command-time SecretRef resolution, but scope web search/fetch resolution to active web credential targets for the selected or auto-selected provider instead of the full agent-runtime target set.

## Reproduction Assessment

Yes. Source inspection shows current main reads raw config for CLI web search/fetch while the documented plugin web credential surface requires command-time SecretRef resolution for the selected provider.

## Solution Assessment

No, not as written. Resolving before invoking the web runtime is the right direction, but the target set must be web-only and provider-scoped to avoid unrelated credential failures or materialization.

## Review Findings
