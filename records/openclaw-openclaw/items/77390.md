## Summary

Current main still matches the report: official packaged installs are documented and advertised, but the service context grants internal diagnostics only to bundled diagnostics exporter services, so this should stay open for a trust-policy fix rather than maintainer cleanup closure.

## What This Changes

The issue asks for externally installed official `diagnostics-otel` plugins to receive internal diagnostics events or for OpenClaw to provide a documented trust/public-hook path.

## Best Possible Solution

Define an explicit trust contract for official external diagnostics exporters, update the service grant and regression tests to match it, and keep the internal diagnostics capability protected from arbitrary global plugins.

## Reproduction Assessment

Yes, from source inspection. A `global`-origin installed `diagnostics-otel` service fails the bundled-only grant in `src/plugins/services.ts`, and the exporter returns before subscribing when `ctx.internalDiagnostics` is missing.

## Solution Assessment

Unclear for automation. The narrow fix is likely in the plugin service trust path, but the best implementation needs maintainer policy on how external official packages prove trust or opt in.
