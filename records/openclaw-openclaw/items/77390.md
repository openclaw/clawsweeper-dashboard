## Summary

Keep open: current main still documents `diagnostics-otel` as an official ClawHub/npm install, but the service context grants `internalDiagnostics` only to bundled-origin services, so the externally installed exporter can start and then return before subscribing. The fix needs maintainer trust-policy review because it changes access to an internal diagnostics stream.

## What This Changes

The issue requests that externally installed official `diagnostics-otel` packages receive diagnostics events, or that the bundled-only gate be documented/replaced with an explicit opt-in or public hook.

## Best Possible Solution

Define and document a narrow trust policy for the official external diagnostics exporters, then align the service gate, exporter behavior, docs, and regression tests to that policy.

## Reproduction Assessment

Yes, from source inspection. A ClawHub/npm install is discovered as `origin: "global"`, fails the bundled-only grant in `src/plugins/services.ts`, and the exporter returns when `ctx.internalDiagnostics` is absent.

## Solution Assessment

Unclear for automation. The narrowest implementation depends on maintainer policy for trusting official external packages or requiring an explicit operator trust mechanism.
