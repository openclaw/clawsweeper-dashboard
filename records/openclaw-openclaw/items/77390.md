## Summary

Keep open: current main still denies `internalDiagnostics` to non-bundled `diagnostics-otel` services even though docs and package metadata advertise the official ClawHub/npm install path. The remaining fix crosses an internal diagnostics trust boundary, so it needs maintainer policy review before code changes.

## What This Changes

The issue asks for externally installed official `diagnostics-otel` packages to receive diagnostics events, or for OpenClaw to document/provide a safe opt-in or public hook path.

## Best Possible Solution

Make the packaged official diagnostics exporter path match the documented behavior while preserving the internal diagnostics trust model, then update the service test and docs to lock that policy.

## Reproduction Assessment

Yes. From source inspection, a non-bundled installed `diagnostics-otel` service has `origin: "global"`, fails the `origin === "bundled"` grant, and the exporter exits when `ctx.internalDiagnostics` is absent.

## Solution Assessment

Unclear for automation. The narrow code path is obvious, but the best fix depends on maintainer policy for trusting official external packages versus requiring explicit operator trust or a public diagnostics hook.
