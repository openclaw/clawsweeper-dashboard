## Summary

Keep open: current main still separates local skill loading from ClawHub search, and there is no registration/index path that makes local skill directories discoverable through `skills.search` or a `skill_search`-style surface.

## What This Changes

The issue requests a way to register local skill directories so they become discoverable through `skill_search`.

## Best Possible Solution

Define an additive local skill discovery contract that reuses existing local loaders, documents source semantics, preserves allowlists, and clarifies metadata/frontmatter compatibility.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that local skills are loaded for inventory/prompt/status paths while `skills.search` still searches ClawHub only.

## Solution Assessment

Unclear until maintainers choose the product/API shape. CLI registration, auto-scan, API registration, and source-filtered search have different protocol, metadata, and trust tradeoffs.
