## Summary

Keep open: current main still separates local skill inventory from ClawHub-backed search, so the requested local-skill search/indexing capability remains unimplemented and needs maintainer product/API direction rather than cleanup closure.

## What This Changes

The issue requests a way to register or index locally authored skills so `skill_search`/`skills.search` can discover them.

## Best Possible Solution

Define an additive local skill discovery/search contract that reuses existing local loaders, preserves agent allowlists, and clearly separates local inventory from ClawHub install semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that local skills are loaded for local inventory/prompt/status paths while search still delegates to ClawHub only.

## Solution Assessment

Unclear until maintainers choose the product/API shape. CLI registration, auto-scan, API registration, or a source-filtered search method have different protocol, metadata, and trust tradeoffs.
