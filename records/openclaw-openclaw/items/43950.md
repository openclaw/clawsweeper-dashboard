## Summary

Keep open: current main supports ClawHub skill search and local skill loading, but it does not provide a local registration or local-search index path for `skills.search`.

## What This Changes

Requested support for registering locally authored skill directories into the skill discovery/search index.

## Best Possible Solution

Define a documented local skill discovery contract that reuses existing local skill loaders, clarifies metadata/frontmatter compatibility, and makes `skills.search` source semantics explicit.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms local skills load for prompts/status, while `skills.search` currently searches only ClawHub.

## Solution Assessment

Unclear until maintainers choose the product shape. The issue lists CLI register, auto-scan, and API registration alternatives, each with different protocol and security implications.
