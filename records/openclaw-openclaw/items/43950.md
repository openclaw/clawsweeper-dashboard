## Summary

Keep open: current main still treats local skill inventory and ClawHub search as separate surfaces, so locally-authored skills are visible to local list/status/prompt flows but not discoverable through the existing search RPC or CLI search. This is a feature/API design request rather than a broken existing contract, and the remaining work needs maintainer direction on source semantics, metadata compatibility, and trust boundaries.

## What This Changes

Add a way to register or index locally-authored skill directories so they can appear in skill search results.

## Best Possible Solution

Define an additive local skill discovery contract that reuses existing local loaders, preserves agent allowlists, documents source semantics, and clarifies `SKILL.md` frontmatter versus `_meta.json` metadata compatibility.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that local skills are loaded for local inventory/prompt/status paths while search still delegates to ClawHub only.

## Solution Assessment

Unclear until maintainers choose the product/API shape. CLI registration, auto-scan, API registration, or a source-filtered search method have different protocol, metadata, and trust tradeoffs.
