## Summary

Keep open. Current main loads local/workspace skills and exposes them through local inventory paths, but `skills.search` remains ClawHub-only, so the requested local searchable registration/index behavior is not implemented and needs a product/API decision.

## What This Changes

The issue requests a way to register or auto-index locally authored skills so they are discoverable through skill search.

## Best Possible Solution

Decide whether local discovery belongs as a source option on `skills.search`, a separate local search/status endpoint, or documentation around `skills list/info`; then implement that chosen contract with tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction. Source review gives high confidence that local skills are loaded for status/list/prompt paths, while the current `skills.search` contract searches only ClawHub.

## Solution Assessment

Unclear until maintainers choose the product contract. The request names several possible designs, and the narrowest maintainable path depends on whether `skills.search` should remain remote discovery or gain local-source semantics.
