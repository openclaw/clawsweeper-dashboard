## Summary

Keep open: current main still treats `skills.search` as ClawHub discovery, while local skills are loaded through status/prompt/list surfaces and the newer upload path installs archives without adding a local search index.

## What This Changes

The issue requests a way to register or index locally authored skills so they are discoverable through skill search.

## Best Possible Solution

Define a documented local skill discovery contract that reuses existing local loaders, clarifies metadata/frontmatter compatibility, and makes `skills.search` source semantics explicit.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that local skills can be loaded for status/prompt surfaces, while `skills.search` currently searches only ClawHub.

## Solution Assessment

Unclear until maintainers choose the product shape. CLI registration, auto-scan, and API registration have different protocol, metadata, and trust implications.
