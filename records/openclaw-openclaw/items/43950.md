## Summary

Keep open: current main loads local skills into prompt/status/list surfaces, but the requested `skills.search`/skill-search discoverability remains ClawHub-only. This is a new product/API feature rather than an implemented bug fix or stale cleanup item.

## What This Changes

Request a way to register or auto-index locally authored skills so they are discoverable through skill search.

## Best Possible Solution

Define an explicit local discovery contract that reuses existing `SKILL.md` frontmatter and `loadWorkspaceSkillEntries`, with clear result source/path metadata or a separate local search endpoint instead of a parallel `_meta.json` index.

## Reproduction Assessment

Yes. Source inspection shows local skills are loaded for workspace status/prompt/list surfaces, while `skills.search` and `openclaw skills search` call only the ClawHub search path.

## Solution Assessment

Unclear until maintainers choose the API shape. The safest implementation would reuse the existing local skill loader and avoid inventing a second registration format.
