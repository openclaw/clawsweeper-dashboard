## Summary

Keep open: current main still keeps local skill inventory and ClawHub search as separate surfaces, so the requested local-skill search/indexing capability is not implemented and needs product/API direction before code changes.

## What This Changes

Add a way for locally authored skills in local skill roots to be registered or indexed so `skills.search` can discover them.

## Best Possible Solution

Define an additive local skill discovery/search contract that reuses existing local loaders, respects agent visibility rules, and keeps ClawHub discovery/install semantics distinct.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high confidence that local skills are loaded for inventory/status/prompt flows while search still delegates to ClawHub only.

## Solution Assessment

Unclear until maintainers choose the product/API shape. CLI registration, auto-scan, API registration, or source-filtered search have different protocol, metadata, trust, and agent-scope tradeoffs.
