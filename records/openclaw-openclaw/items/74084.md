## Summary

Keep open. Current main still keys the evaluated-bindings cache only by the config object, while this PR contains the proposed cross-config cache optimization and focused tests; it is useful but still needs the small eviction-policy cleanup and required changelog entry before normal maintainer review/merge.

## What This Changes

The PR adds bindings-array identity and SHA-256 content-fingerprint fallback caches to `getEvaluatedBindingsForChannelAccount`, plus routing tests for rebuilt config objects with identical or changed bindings content.

## Best Possible Solution

Keep the routing optimization if routing maintainers accept the fingerprinting/immutability tradeoff, make the fingerprint eviction behavior match the stated policy, add a changelog entry, and land only after focused routing tests plus the changed gate are green.

## Reproduction Assessment

Yes. Current-main source inspection shows fresh cfg objects miss `evaluatedBindingsCacheByCfg` and rebuild through `listBindings`; the PR’s added spy test is a high-confidence reproduction path for the intended performance case, though this read-only review did not run tests.

## Solution Assessment

Unclear. The proposed cache layers are narrow and plausible, but the eviction semantics should be corrected or accurately documented before treating this as the best maintainable fix.
