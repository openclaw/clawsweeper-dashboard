## What This Changes

The PR adds bindings-array and SHA-256 fingerprint fallback caches for routing evaluated-bindings lookup, plus tests for identical rebuilt configs and changed binding content.

## Best Possible Solution

Land a narrowed performance patch only after contributor runtime/perf proof, green relevant checks, and either true LRU refresh on fingerprint hits or FIFO wording.

## Reproduction Assessment

Yes. By source inspection, current main only reuses evaluated bindings for the same cfg object and bindings reference, so rebuilt configs with identical binding content rebuild through listBindings; I did not run tests because this is a read-only review.

## Solution Assessment

Unclear. The cache layering is narrow and plausible, but it should first align the stated bounded eviction behavior with the implementation and provide real perf evidence for the target path.
