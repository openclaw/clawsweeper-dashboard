## What This Changes

The branch adds binding-array and SHA-256 fingerprint fallback caches to routing evaluated-bindings lookup plus tests for rebuilt configs with identical or changed bindings.

## Best Possible Solution

Land a narrowed version after aligning the FIFO/LRU behavior or wording, adding real proof or perf evidence, and passing focused routing tests plus the normal changed gate.

## Reproduction Assessment

Yes, by source inspection: current main only reuses the evaluated-bindings cache for the same config object and bindings reference, so fresh config clones with identical bindings rescan through listBindings. I did not run a local test because this was a read-only review.

## Solution Assessment

Unclear. The implementation is narrow and plausible, but it should first make the bounded fingerprint eviction behavior honest or true LRU and provide real proof that the hashing tradeoff helps the target path.
