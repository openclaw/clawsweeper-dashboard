## Summary

Keep open: current main still has the source-reproducible no-delivery completion path, and an open fixing PR already references this issue with closing syntax, so this issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue reports that Nextcloud Talk direct-delivery turns can emit visible agent text through the `text_end`/block path without invoking the `message` tool and still mark the message completed.

## Best Possible Solution

Add and land a shared dispatch invariant that treats `message_tool_only` visible output without a successful message-tool send as an actionable no-delivery outcome or equivalent diagnostic, with regression coverage for the suppressed block/text_end path.

## Reproduction Assessment

Yes. A current-main source path maps Nextcloud Talk group turns to `message_tool_only`, sets `suppressDelivery`, drops block/text_end delivery, skips final sends, and still records the message as completed.

## Solution Assessment

Yes. The maintainable fix belongs in the shared source-reply/message-tool dispatch accounting rather than a Nextcloud-only workaround, and the open linked PR targets that boundary.
