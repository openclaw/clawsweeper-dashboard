## Summary

Keep open: current main still has the source-reproducible suppressed-delivery/completed-diagnostic path, and open draft PR #77020 already references this issue with closing syntax.

## What This Changes

The issue reports that Nextcloud Talk direct replies can be marked completed after visible generated output is emitted without a successful `message` tool delivery.

## Best Possible Solution

Land #77020 or an equivalent shared dispatch invariant so `message_tool_only` runs with generated visible output and no successful `message` tool send get an explicit non-delivery outcome instead of an ordinary completion.

## Reproduction Assessment

Yes. Source inspection shows a focused current-main path where a Nextcloud room turn resolves to `message_tool_only`, suppresses automatic block/final delivery, and still records the turn as completed when no message-tool send occurs.

## Solution Assessment

Yes. The best fix belongs in the shared source-reply/message-tool dispatch accounting path, with Nextcloud tests only as coverage, because the broken invariant is not specific to the Nextcloud transport adapter.
