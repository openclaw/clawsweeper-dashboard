## Summary

Keep open: current main still has a source-reproducible suppressed-delivery path, and open PR #77020 is the active closing fix candidate, so this issue should wait for that PR to merge or be replaced.

## What This Changes

The issue reports that Nextcloud Talk direct-channel runs can suppress visible final/block output under message-tool-only mode, log completion, and never post to the room when the agent does not call `message`.

## Best Possible Solution

Keep the issue open until #77020 or an equivalent core dispatch fix lands; the end state should distinguish suppressed visible output without a message-tool send from ordinary completed delivery.

## Reproduction Assessment

Yes. Source inspection on current main gives a focused reproduction path: a group/channel turn resolves `message_tool_only`, emits visible final/block text without a successful messaging-tool send, suppresses delivery, and still records the turn as completed.

## Solution Assessment

Yes. The best fix belongs in the shared source-reply/message-tool dispatch accounting, not in a Nextcloud-only workaround, because Nextcloud Talk enters the same core dispatch path as other direct channel plugins.
