## Summary

Keep open: current main still has a source-reproducible path where Nextcloud Talk/group turns use `message_tool_only`, generated visible output is suppressed, and the turn can be recorded as ordinary completion. A linked open fix PR with closing syntax is already tracking the issue, and a second user reported the same symptom on May 12, 2026.

## What This Changes

Ensure Nextcloud Talk direct-channel replies either go through the message tool or produce a clear no-delivery/suppressed-delivery diagnostic instead of silently completing without a chat post.

## Best Possible Solution

Land or revise the shared dispatch fix so generated visible output under `message_tool_only` without a successful message-tool send is surfaced as an actionable suppressed-delivery diagnostic while preserving privacy and avoiding Nextcloud-only special cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Nextcloud/group-style turn resolves to `message_tool_only`, block/final visible output is skipped under `suppressDelivery`, and current dispatch records ordinary completion without a successful message-tool delivery.

## Solution Assessment

Yes. The best fix belongs in shared dispatch accounting/diagnostics, not in a Nextcloud-only workaround, and the linked open PR is a narrow implementation candidate that maintainers can review or supersede.
