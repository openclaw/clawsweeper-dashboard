## What This Changes

Ensure Nextcloud Talk direct-delivery turns do not silently complete when visible output is generated without a successful `message` tool send.

## Best Possible Solution

Review and land #77020 or an equivalent shared dispatch fix that reports suppressed visible output without a successful message send, then close this issue after merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source: a Nextcloud Talk group-style inbound dispatch resolves to `message_tool_only`, suppresses block/final source delivery, and still records the turn as completed when no `message` tool evidence exists.

## Solution Assessment

Is this the best way to solve the issue? Yes: the fix belongs in shared source-reply delivery accounting rather than a Nextcloud-only workaround, and #77020 appears to target that boundary.
