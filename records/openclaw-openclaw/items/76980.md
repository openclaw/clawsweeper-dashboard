## What This Changes

The issue reports that Nextcloud Talk direct-channel replies can be marked completed after generated text is emitted without a successful `message` tool send.

## Best Possible Solution

Review and land PR #77020, or replace it with an equivalent shared dispatch fix that makes generated-but-undelivered message-tool-only output clearly visible in diagnostics without duplicating real `message` tool sends.

## Reproduction Assessment

Yes. A focused source reproduction exists on current main: a group/channel-style direct dispatch resolves to `message_tool_only`, suppresses block/final delivery, and still records the turn as completed without a distinct no-delivery reason.

## Solution Assessment

Yes. The narrow maintainable path is in shared source-reply dispatch accounting, because Nextcloud Talk already provides the direct send adapter and the silent completion comes from core suppression/diagnostic handling.
