## Summary

Keep open: current main still has a source-reproducible Nextcloud Talk path where group/channel source replies are processed under `message_tool_only`, visible generated output can be suppressed, and the diagnostic can still finish as ordinary `completed`; the linked closing PR is still open, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Nextcloud Talk replies in group rooms can complete after the agent emits text without calling the message tool, leaving no visible Nextcloud post or clear diagnostic.

## Best Possible Solution

Preserve the documented tool-only group/channel policy, but land or supersede the linked diagnostic fix so generated-but-suppressed source replies are visible to operators instead of looking like ordinary completed deliveries.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: Nextcloud Talk group rooms enter shared dispatch, resolve to `message_tool_only`, suppress automatic source delivery, and can finish with generated content but no source send; I did not run a live Nextcloud stack.

## Solution Assessment

Yes for the narrow issue. A shared dispatch diagnostic like the linked PR is the maintainable fix because the invariant is source-reply accounting, while auto-promoting hidden final text would change group/channel delivery policy and belongs in the broader tracker.
