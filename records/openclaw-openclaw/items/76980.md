## Summary

Keep open: current main still has a source-reproducible path where a message-tool-only Nextcloud Talk turn can generate visible text, suppress source delivery, and record the turn as completed; the active unmerged PR #77020 is the right place to resolve it.

## What This Changes

The issue reports that Nextcloud Talk direct-channel replies can be marked completed without posting when the agent emits final/text_end output instead of calling the `message` tool.

## Best Possible Solution

Land #77020 or an equivalent shared dispatch fix that marks generated visible output without a successful message-tool send as a delivery-suppressed diagnostic instead of an ordinary completed visible reply, while preserving intentional private replies and duplicate suppression.

## Reproduction Assessment

Yes. A focused source reproduction is `dispatchReplyFromConfig` with a group/channel or requested `message_tool_only` turn whose resolver emits visible final/block text but records no successful messaging-tool send; current main suppresses delivery and then records completion.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the shared source-reply/message-tool dispatch accounting and diagnostics path, not in a Nextcloud-only workaround, because the invariant applies to every direct channel using `message_tool_only`.
