## Summary

Keep open: current main still has a source-reproducible path where message-tool-only source delivery suppresses visible output while diagnostics can still end as completed, a second user confirmed the Nextcloud Talk symptom, and the linked fix PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to stop silently completing Nextcloud Talk direct-channel replies when the agent emits visible text without calling the `message` tool.

## Best Possible Solution

Land or replace the linked core dispatch/agent-delivery fix so suppressed visible output is reported as a non-successful source-reply-delivery outcome, with focused regression coverage for message-tool-only direct-channel turns.

## Reproduction Assessment

Yes for source-level reproduction: a direct/channel room event enters `message_tool_only`, suppresses automatic delivery, drops block replies under `suppressDelivery`, and can still record completed processing. I did not establish a live Nextcloud Talk repro in this read-only review.

## Solution Assessment

Unclear until the linked PR is rebased and reviewed: the fix belongs in core source-reply dispatch/diagnostics rather than the Nextcloud send adapter, but maintainers should decide whether diagnostics alone are sufficient or whether message-tool-only turns need a stronger delivery failure state.
