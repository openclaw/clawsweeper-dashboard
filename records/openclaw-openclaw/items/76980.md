## Summary

Keep open: current main still contains the source-reproducible message-tool-only suppression path that can finish a direct-channel turn as completed without visible delivery, and the linked fix PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to stop silently completing Nextcloud Talk direct-outbound turns when visible reply text is produced without a successful `message` tool send.

## Best Possible Solution

Land or rework #77020, or an equivalent shared dispatch fix, so message-tool-only turns with generated visible output and no recorded message-tool send produce an explicit no-delivery diagnostic while preserving duplicate suppression after real sends.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source: a Nextcloud/group-style direct channel resolves to `message_tool_only`, suppresses block/final delivery, and still records the turn as completed when no message-tool send is recorded.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix belongs in shared source-reply delivery accounting and diagnostics, not in a Nextcloud-only workaround.
