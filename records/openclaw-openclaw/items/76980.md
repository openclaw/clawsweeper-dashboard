## Summary

Keep open: current main still has a source-reproducible diagnostic gap for generated visible output suppressed by `message_tool_only`, and the linked implementation PR that closes this issue is still open.

## What This Changes

The issue asks OpenClaw to stop Nextcloud Talk direct-delivery turns from completing silently when assistant output is generated without a visible `message` tool send.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77020 for the diagnostic gap, while leaving behavior-changing auto-post fallback policy to the cross-channel follow-up.

## Reproduction Assessment

Yes at source level. Nextcloud Talk group turns enter shared dispatch, default to `message_tool_only`, suppress automatic block/final sends, and can still record a completed diagnostic when generated visible output was not posted.

## Solution Assessment

Yes for the narrow bug. A shared dispatch diagnostic is the maintainable fix because the invariant is generated-but-suppressed source reply accounting; automatic fallback would be a separate product-policy change.
