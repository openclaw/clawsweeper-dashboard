Keep open: current main and v2026.6.10 still keep substantive `message_tool_only` finals private when the model omits `message(action=send)`, and the open diagnostic/recovery PRs have not landed. ## What This Changes

The issue asks OpenClaw to stop Nextcloud Talk/direct-channel turns from finishing as completed after generated replies are not delivered through the `message` tool.

## Best Possible Solution

Resolve the open recovery/diagnostics path as the canonical maintainer decision, preserving warning-only privacy by default unless maintainers approve a narrow opt-in recovery mode.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the checked-in QA scenario forces a long `message_tool_only` final with no message tool call and expects no outbound plus the private-final warning; I did not run live Nextcloud Talk.

## Solution Assessment

Unclear for the final product choice. A default-off recovery path is plausible, but the repository policy deliberately preserves private finals unless `message(action=send)` is used, so maintainers need to accept the recovery semantics before this issue can close.
