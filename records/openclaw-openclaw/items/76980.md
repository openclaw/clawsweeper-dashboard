## What This Changes

Nextcloud Talk direct outbound turns should not report ordinary completion after generated visible output is suppressed without a successful message tool send.

## Best Possible Solution

Review, rebase if needed, and land https://github.com/openclaw/openclaw/pull/77020 or an equivalent shared-dispatch diagnostic fix that preserves message_tool_only privacy semantics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, as a source-level path: a Nextcloud Talk group/direct dispatch can resolve to message_tool_only, generate visible block or final output, suppress delivery, and still record ordinary completion without a message-tool send.

## Solution Assessment

Is this the best way to solve the issue? Yes: the maintainable fix belongs in shared dispatch delivery accounting and diagnostics rather than a Nextcloud-only workaround, and the existing open PR follows that direction.
