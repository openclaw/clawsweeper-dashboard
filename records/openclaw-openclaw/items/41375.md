## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.98

Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:205-226`
  - body: This new delivery runs before `resolveSendPolicy` and calls `routeReply` directly, so a session with `sendPolicy: "deny"` can still emit hook-generated outbound messages. Route these replies through the same policy-gated boundary as normal inbound replies and cover denied sessions.
  - confidence: 0.98
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:218-220`
  - body: The fallback `ctx.To ?? ctx.From` selects the address that received the inbound message—commonly the bot identity—before the user who sent it. Use the canonical originating conversation target or prefer `ctx.From` when `OriginatingTo` is absent, and correct the matching test expectation.
  - confidence: 0.98
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:24`
  - body: Managed hook code is runtime-loaded, so an accidental non-string entry can make `message.trim()` throw and abort reset/new handling. Narrow to strings before trimming so hook reply cleanup cannot crash an interactive command path.
  - confidence: 0.9
