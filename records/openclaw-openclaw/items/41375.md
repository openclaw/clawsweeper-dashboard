## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:210-226`
  - body: This new delivery runs immediately after `triggerInternalHook` and calls the direct reply route before the existing `resolveSendPolicy` and source-reply suppression decision can block outbound sends. A session configured with `sendPolicy: deny` would still send hook-generated text, so the drain needs to reuse or wait for the current policy-aware reply path before routing.
  - confidence: 0.91
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:218`
  - body: The new target falls back to `ctx.To` before `ctx.From`, but on inbound direct messages `ctx.To` is commonly the bot's address. The sibling command hook drain already uses the sender-first fallback, and this path should not route hook replies back to the bot identity.
  - confidence: 0.9
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:24`
  - body: Internal hook handlers are runtime-loaded code, so a bad or stale hook can push a non-string into `event.messages`. Calling `.trim()` on every element can throw and abort the awaited reset/new hook path or fail the fire-and-forget message hook delivery instead of dropping invalid values safely.
  - confidence: 0.82
