- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This new delivery runs immediately after `triggerInternalHook` and calls the direct reply route before the current `sendPolicy`/source-reply suppression decision can block outbound delivery.   - confidence: 0.9
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:219`
  - body: For direct inbound turns, `ctx.To` is commonly the bot address and `ctx.From` is the user who sent the message. The new hook reply target uses `ctx.To` before `ctx.From`, so hook replies can be sent to the bot identity instead of back to the user.
  - confidence: 0.88
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:24`
  - body: Internal hooks are runtime-loaded JavaScript, so a hook can accidentally push a non-string into `event.messages` despite the TypeScript type. Calling `message.trim()` directly can throw and abort the reset/new or inbound hook-reply path instead of skipping invalid hook output.
  - confidence: 0.78
