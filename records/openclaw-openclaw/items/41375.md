Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prefer the inbound sender for hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:218-220`
  - body: For inbound turns without `OriginatingTo`, `ctx.To` is typically the bot address and `ctx.From` is the user. This fallback would send hook replies to the bot identity instead of the sender, and the added test asserts the same wrong target.
  - confidence: 0.9
- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:214-220`
  - body: This new `message:received` delivery runs immediately after the internal hook and before the normal send-policy/source-delivery suppression path. A session configured with `sendPolicy: deny` can still emit hook-generated outbound messages through `routeReply`.
  - confidence: 0.86
- **[P2] Guard hook messages before trimming them:** `src/auto-reply/reply/internal-hook-replies.ts:27`
  - body: Internal hook handlers are runtime code and can accidentally push non-string values despite the TypeScript type. Calling `.trim()` directly can throw before delivery logging, and the command hook path awaits this helper.
  - confidence: 0.78
