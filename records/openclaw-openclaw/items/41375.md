Overall confidence: 0.91

Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This new hook drain runs immediately after `triggerInternalHook` and calls `routeReply` before the current source-delivery policy decision. That can send hook-generated outbound messages even when the session is configured with `sendPolicy: deny`; move this drain into the policy-aware delivery path or make the bypass an explicit documented contract with tests.
  - confidence: 0.9
- **[P2] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:218`
  - body: When `OriginatingTo` is absent, `ctx.To` is commonly the bot address for inbound direct messages and `ctx.From` is the user. This sends hook replies to the bot identity on those turns, and the added test currently asserts that wrong destination; prefer `ctx.From` before `ctx.To` and update the assertion.
  - confidence: 0.92
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:28`
  - body: Managed hooks and hook packs are runtime JavaScript boundaries, so they can accidentally push non-string values into `event.messages`. Calling `.trim()` directly can throw and abort the hook reply path; filter `typeof message === "string"` before trimming or routing.
  - confidence: 0.78
