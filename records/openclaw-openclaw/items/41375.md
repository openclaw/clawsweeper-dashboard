## Summary

Keep open. This maintainer-labeled PR targets a real source-confirmed hook reply delivery bug, but it still has blocker-level reply-target and send-policy issues before merge.

## What This Changes

The branch adds a shared helper to drain internal hook `event.messages`, wires it into reset/new and `message:received` reply routing, updates hook docs, and adds focused tests.

## Best Possible Solution

Rebase onto current main, drain hook replies only after the current source-reply/send-policy decision is available, route to the originating sender, and keep lifecycle-only hooks documented as non-replying unless maintainers explicitly widen that contract.

## Reproduction Assessment

Yes. Source inspection shows current main lets internal hook handlers mutate `event.messages`, but the `message:received` call site never retains or drains that event while `/new` and `/reset` do.

## Solution Assessment

No as written. The intended fix is useful, but the PR needs to route to the inbound sender and honor the current send-policy/source-reply suppression contract before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route hook replies to the inbound sender:** `src/auto-reply/reply/dispatch-from-config.ts:218-220`
  - body: On inbound turns where `OriginatingTo` is absent, `ctx.To` is typically the bot address and `ctx.From` is the user. This fallback sends hook replies to the bot instead of the sender, and the new test locks in that wrong target.
  - confidence: 0.94
- **[P1] Honor sendPolicy before draining hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:205-226`
  - body: This new `message:received` delivery path calls `routeReply` before the normal send-policy/source-reply suppression path can block user delivery, so sessions configured with `sendPolicy: deny` can still emit hook replies.
  - confidence: 0.9
- **[P2] Normalize hook messages before trimming them:** `src/auto-reply/reply/internal-hook-replies.ts:24-32`
  - body: Internal hooks can come from runtime JS hook packs, so a malformed value in `event.messages` will make `.trim()` throw and can abort the interactive command path. Filter to strings and trim after narrowing before routing.
  - confidence: 0.78
