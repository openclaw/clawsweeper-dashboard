## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prefer the inbound sender for hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:218-220`
  - body: This fallback picks `ctx.To` before `ctx.From`; on normal inbound turns `ctx.To` is often the bot identity, so hook replies can be routed back to the bot instead of the user who sent the message. Prefer `OriginatingTo`, then the inbound sender, then the bot-side `To` fallback, and update the test expectation accordingly.
  - confidence: 0.92
- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:211-218`
  - body: The new hook reply drain runs before `resolveSendPolicy` is applied, so a session configured with `sendPolicy: deny` can still send hook-generated replies through `routeReply`. Resolve and pass the send-policy decision before delivery, or gate this call site with focused denial coverage.
  - confidence: 0.88
- **[P2] Validate hook messages before trimming them:** `src/auto-reply/reply/internal-hook-replies.ts:25`
  - body: Hook handlers are runtime code, but this helper calls `.trim()` on every `event.messages` entry. A malformed or accidental non-string value can throw and disrupt command or inbound processing, so normalize only strings and skip invalid entries before joining.
  - confidence: 0.86
