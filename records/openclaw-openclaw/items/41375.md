## Review Findings

Overall confidence: 0.92

Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This new drain calls `deliverInternalHookMessages` before the same function computes `resolveSendPolicy`, so a session configured with `sendPolicy: deny` can still send hook-generated text. Move the drain behind the canonical policy/source-reply decision or pass an already-resolved allowed policy into the helper.
  - confidence: 0.94
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:218`
  - body: `ctx.To` is the bot-side address on direct inbound turns, while `ctx.From` is the user who sent the message. This fallback can send hook replies to the bot identity, and the new test currently asserts that wrong target.
  - confidence: 0.92
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:28`
  - body: Internal hook handlers are runtime code, so a malformed push into `event.messages` can make `.trim()` throw and abort command hook reply delivery. Normalize to strings before trimming and skip invalid entries.
  - confidence: 0.78
