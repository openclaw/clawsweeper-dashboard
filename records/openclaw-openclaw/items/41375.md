## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prefer the inbound sender for hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:218-220`
  - body: The new target falls back to `ctx.To` before `ctx.From`, but on normal inbound channel turns `To` is often the bot identity and `From` is the user. This can route the hook reply back to the bot; update the fallback and test expectation to prefer the sender when `OriginatingTo` is absent.
  - confidence: 0.91
- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:214-220`
  - body: This new automatic outbound path calls hook reply delivery without checking the current source-delivery/send-policy suppression state. A session with `sendPolicy` denied can still emit hook-generated replies unless this is gated by the same policy path as normal automatic replies.
  - confidence: 0.86
- **[P2] Guard hook messages before trimming them:** `src/auto-reply/reply/internal-hook-replies.ts:27`
  - body: Runtime hook code controls `event.messages`, so a non-string entry will throw on `.trim()` and can abort command hook delivery. Filter or normalize string entries before trimming and keep malformed hook output from crashing the turn.
  - confidence: 0.78
