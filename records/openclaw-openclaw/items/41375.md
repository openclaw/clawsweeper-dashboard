  - body: When `OriginatingTo` is absent, this target chooses `ctx.To` before `ctx.From`; on direct inbound turns `ctx.To` can be the bot-side address, so the hook reply can be sent to the bot instead of the user. The regression test should assert the sender fallback instead of `ctx.To`.

  - body: This new delivery runs before the current `sendPolicy`/source-reply suppression decision, so a session configured with `sendPolicy: "deny"` can still emit hook replies through `routeReply`. Compute or pass the same denial decision before calling the helper and add a denial regression test.
  - confidence: 0.9

  - body: Runtime hooks can push malformed values despite the TypeScript type, and calling `.trim()` on every array entry can throw during command or message hook reply delivery. Filter to strings, trim after narrowing, and keep bad hook output from aborting the turn.
  - confidence: 0.84
