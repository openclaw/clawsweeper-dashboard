## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This new delivery runs immediately after `triggerInternalHook` and calls the reply route before the existing `resolveSendPolicy` and source-reply suppression decision can block outbound delivery. A session configured with `sendPolicy: "deny"` can still emit hook-generated messages, so the drain needs to run through or after the same policy-aware delivery path as normal replies.
  - confidence: 0.94
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:218`
  - body: The fallback target uses `ctx.To` before `ctx.From`, but on direct inbound turns `ctx.To` is commonly the bot identity and `ctx.From` is the user. This can send a `message:received` hook reply to the bot's own address; the fallback should match the command path and prefer the sender when `OriginatingTo` is absent.
  - confidence: 0.9
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:28`
  - body: Internal hooks are runtime-loaded, so a handler can accidentally push a non-string into `event.messages`. Calling `.trim()` before narrowing throws and can abort awaited command-hook delivery paths; narrow to strings before trimming and joining.
  - confidence: 0.82
