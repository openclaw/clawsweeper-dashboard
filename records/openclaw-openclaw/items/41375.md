## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Prefer the inbound sender for hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:218-220`
  - body: This fallback chooses `ctx.To` before `ctx.From`, but on inbound provider turns `To` is often the bot identity. When `OriginatingTo` is absent, hook replies can be routed back to the bot instead of the user who sent the message.
  - confidence: 0.93
- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: The new hook delivery runs before the normal `sendPolicy` suppression path, so sessions configured to deny automatic outbound delivery can still send hook-generated replies. Gate this path on the same policy decision as other automatic replies.
  - confidence: 0.9
- **[P2] Filter hook reply messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:25`
  - body: Managed hooks can be plain JavaScript, so a bad handler can push a non-string into `event.messages`. Calling `.trim()` directly will throw and can abort `/new`, `/reset`, or the inbound hook delivery path instead of skipping the malformed entry.
  - confidence: 0.82
