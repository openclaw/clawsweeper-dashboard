## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Prefer the inbound sender for hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:218`
  - body: The new `message:received` target falls back to `ctx.To` before `ctx.From`. On inbound transports, `To` is often the bot address, and the added test asserts that wrong target, so hook replies can be sent to the bot instead of the user.
  - confidence: 0.94
- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This delivery runs before dispatch computes the normal `resolveSendPolicy` suppression decision. A session configured to deny automatic sends can still emit hook-generated text through `routeReply`, bypassing the same policy applied to normal replies.
  - confidence: 0.91
- **[P2] Validate hook messages before trimming them:** `src/auto-reply/reply/internal-hook-replies.ts:28`
  - body: Runtime hook handlers can push non-string values despite the TypeScript shape. Calling `.trim()` on each entry can throw and disrupt `/new`, `/reset`, or inbound processing, so the helper should filter to strings before trimming or routing.
  - confidence: 0.86
