Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This new hook drain runs immediately after `triggerInternalHook` and routes via `routeReply` before the current `sendPolicy` and source-reply suppression decision can block outbound delivery. A session configured with `sendPolicy: "deny"` can still emit hook-generated replies, so the drain needs to move through the existing policy-aware route and get deny coverage.
  - confidence: 0.9
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:220`
  - body: When `OriginatingTo` is absent, this falls back to `ctx.To` before `ctx.From`; on direct inbound messages `ctx.To` is commonly the bot address, while `ctx.From` is the user. The added test asserts the bot address, so this can send hook replies to the wrong target instead of catching the bug.
  - confidence: 0.88
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:24`
  - body: Runtime-loaded hook code can still push a non-string into `event.messages`, and calling `.trim()` here will throw before command/reset processing can complete. Normalize to strings before trimming so a bad hook reply cannot abort the turn.
  - confidence: 0.79
