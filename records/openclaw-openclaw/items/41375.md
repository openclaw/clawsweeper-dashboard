## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route hook replies to the sender fallback:** `src/auto-reply/reply/dispatch-from-config.ts:218-220`
  - body: This new target uses `ctx.To` before `ctx.From`. On normal inbound channel turns `To` is often the bot address and `From` is the user, so a `message:received` hook reply can be sent back to the bot; the new test also asserts the bot address, so CI would lock in the wrong route.
  - confidence: 0.9
- **[P1] Apply sendPolicy before hook reply delivery:** `src/auto-reply/reply/dispatch-from-config.ts:215-220`
  - body: The hook reply is delivered from this new block before the current dispatch path resolves sendPolicy and suppresses outbound delivery. A session configured with sendPolicy `deny` can still emit hook replies, bypassing the no-send policy that now guards normal replies, typing, plugin notices, and hook user delivery.
  - confidence: 0.88
- **[P2] Harden hook message normalization:** `src/auto-reply/reply/internal-hook-replies.ts:24`
  - body: `event.messages` is mutated by runtime hook code, including JS hook packs, but this helper assumes every entry has `.trim()`. A non-string push now throws outside `triggerInternalHook`'s handler guard and can abort `/new` or `/reset` processing instead of skipping the bad hook reply.
  - confidence: 0.78
