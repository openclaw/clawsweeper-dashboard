## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This new drain calls `deliverInternalHookMessages` immediately after `triggerInternalHook`, before `resolveSendPolicy` and the source-reply suppression decision can deny outbound delivery. A session configured with `sendPolicy: "deny"` can still send hook-generated text, so the drain needs to run through the same policy-gated path as ordinary replies.
  - confidence: 0.97
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:218`
  - body: The fallback target uses `ctx.To` before `ctx.From`, but on direct inbound messages `ctx.To` is commonly the bot identity while `ctx.From` is the user. That makes the new hook reply path send to the wrong address, and the added test locks in that wrong expectation.
  - confidence: 0.97
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:28`
  - body: Managed hook code can be runtime-loaded JavaScript, so it can accidentally push a non-string into `event.messages` despite the TypeScript type. Calling `.trim()` on that value will throw and can abort the hook reply path; skip or normalize malformed entries before trimming.
  - confidence: 0.82
