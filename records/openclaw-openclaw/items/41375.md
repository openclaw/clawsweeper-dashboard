## What This Changes

The PR adds an internal hook reply-drain helper, routes command and message:received event.messages through reply routing, updates hook docs, and adds focused tests.

## Best Possible Solution

Port the reply-drain onto current main after outbound policy resolution, route to the effective inbound sender, harden hook message normalization, and document that only policy-compliant reply-capable surfaces auto-deliver event.messages.

## Reproduction Assessment

Yes. Source inspection on current main shows message:received internal hooks are fired with an inline event and never drained, while command hooks already drain event.messages through routeReply.

## Solution Assessment

No as submitted. The helper direction is reasonable, but it must be integrated after current send-policy/source-reply decisions, route to the inbound sender, and harden runtime message values before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route hook replies to the inbound sender:** `src/auto-reply/reply/dispatch-from-config.ts:220`
  - body: The new message:received path falls back to `ctx.To` before `ctx.From`, but on inbound channel turns `ctx.To` is usually the bot address. This can route hook replies back to the bot instead of the user; prefer `ctx.From` before `ctx.To` and update the WhatsApp-style test to assert the sender.
  - confidence: 0.93
- **[P1] Honor sendPolicy before delivering hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: The PR delivers hook-added messages immediately after `triggerInternalHook`, before the normal send-policy/source-reply suppression decision gates outbound delivery. Sessions configured with `sendPolicy: deny` can still send hook replies unless this delivery is moved behind the same policy gate.
  - confidence: 0.89
- **[P3] Filter hook messages before trimming them:** `src/auto-reply/reply/internal-hook-replies.ts:29`
  - body: `event.messages` is populated by runtime hook code, but the helper calls `.trim()` on every entry. A malformed non-string value can throw and abort the command path, so normalize to strings before trimming and joining.
  - confidence: 0.78
