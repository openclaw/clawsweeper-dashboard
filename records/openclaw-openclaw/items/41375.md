## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor send policy before routing hook replies:** `src/auto-reply/reply/dispatch-from-config.ts:213-226`
  - body: This new delivery runs immediately after `triggerInternalHook` and calls the direct reply route before `resolveSendPolicy` / `resolveSourceReplyVisibilityPolicy` can suppress automatic outbound delivery. A session configured with `sendPolicy: "deny"` would still send hook-generated text, so the drain needs to run only after the same policy gate used by normal replies, with a deny regression test.
  - confidence: 0.91
- **[P1] Prefer the inbound sender over the bot address:** `src/auto-reply/reply/dispatch-from-config.ts:219`
  - body: When `OriginatingTo` is absent, this uses `ctx.To` before `ctx.From`. On direct inbound transports, `ctx.To` is often the bot/account address and `ctx.From` is the user, so hook replies can be routed to the bot identity; use the sender-first fallback and update the test expectation.
  - confidence: 0.9
- **[P3] Filter non-string hook messages before trimming:** `src/auto-reply/reply/internal-hook-replies.ts:24`
  - body: Runtime-loaded hook handlers can accidentally push malformed values into `event.messages`; calling `.trim()` on every entry can throw and abort the awaited `/new` or `/reset` hook path. Normalize to strings before trimming, or drop malformed entries without throwing.
  - confidence: 0.82
