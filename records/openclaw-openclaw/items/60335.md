## Summary

Keep open. Current main still hydrates Slack thread-starter files for text-only thread replies and forwards them as inbound media, while the paired closing PR remains open and unmerged.

## What This Changes

The issue requests that Slack thread replies include only media attached to the current reply, not media from the thread-starting parent message.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/60353 or a narrow rebased equivalent, with Slack docs updated to state the final thread-root media contract while preserving reply-owned media handling.

## Reproduction Assessment

Yes. Source inspection on current main shows a text-only Slack thread reply with starter files and no direct media hydrates `starter.files` into `threadStarterMedia`, then forwards that fallback into inbound media fields.

## Solution Assessment

Yes, with maintainer follow-through. The fix belongs in the Slack plugin's thread assembly, but it should land through the paired PR or a rebased equivalent with docs aligned to the final contract.
