## Summary

Keep this open. Current main still hydrates Slack thread-starter files for text-only thread replies and then falls back to that media in the inbound model payload; the linked fixing PR #60353 explicitly closes this issue but remains open and unmerged.

## What This Changes

Slack thread replies should only include media attached to the current reply, not rehydrate parent-message files as fresh model attachments.

## Best Possible Solution

Land #60353 or a narrow equivalent in the Slack plugin: stop hydrating thread-starter media for thread replies, keep starter text/history context, preserve media attached directly to replies, and add focused regression tests for first and later thread replies.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction path: a text-only Slack thread reply makes `effectiveDirectMedia` null, `prepare-thread-context.ts` hydrates the parent `starter.files`, and `prepare.ts` forwards that fallback as media fields.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Slack plugin’s inbound thread assembly, not core: remove parent media hydration/fallback while preserving direct reply media and thread text context.
