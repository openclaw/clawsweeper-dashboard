## Label Justifications

- `P2`: This is a real Slack media/context bug with limited channel-specific blast radius and an existing linked fix path.
- `impact:message-loss`: The issue is about duplicated stale Slack media being delivered as if it belonged to each text-only reply.
- `impact:session-state`: The model receives stale parent media as current turn context, causing thread context drift across replies.

## Summary

Keep open. Current main and v2026.5.19 still hydrate Slack thread-starter files for text-only thread replies and forward them as inbound media, while the linked closing PR remains open and conflicting.

## What This Changes

The issue asks Slack to stop treating thread starter media as fresh inbound media on every text-only thread reply.

## Best Possible Solution

Land a rebased Slack fix that stops repeated starter-media rehydration after initial root context is recorded, preserves reply-owned media and intentional first-turn root-media context, and updates docs/tests for the final contract.

## Reproduction Assessment

Yes, source-level. Current main hydrates `starter.files` when there is no direct media and forwards `threadStarterMedia` as effective media for text-only Slack thread replies.

## Solution Assessment

Unclear. The issue is real, but the broad never-hydrate direction conflicts with documented Slack root-media inheritance; a narrower fix or explicit contract change is safer.
