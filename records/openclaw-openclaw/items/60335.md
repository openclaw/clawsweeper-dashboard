## Summary

Keep open. Current main still hydrates Slack thread-starter files for text-only thread replies and forwards them through inbound media fields, while the paired closing PR remains open and unmerged; the current Slack docs also describe the existing root-file hydration behavior, so landing the fix should include docs/contract alignment.

## What This Changes

The issue reports that Slack text-only thread replies repeatedly reattach the thread starter's media to each new model turn instead of only including reply-owned media.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/60353 or a narrow equivalent, with Slack docs updated to state the final thread-root media contract while preserving reply-owned media handling.

## Reproduction Assessment

Yes, source-level. On current main, a text-only Slack thread reply with starter files and no direct media hydrates `starter.files` into `threadStarterMedia`, then forwards that fallback through the inbound media fields.

## Solution Assessment

Yes, with maintainer follow-through. The narrow fix belongs in the Slack plugin's thread assembly, but because current docs describe root-file hydration, the code change should land with docs/contract alignment.
