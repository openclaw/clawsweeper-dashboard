## What This Changes

The issue reports that macOS PortGuardian incorrectly terminates a valid launchd-managed Gateway process when it appears as `node .../openclaw/dist/index.js gateway --port 18789`.

## Best Possible Solution

Update PortGuardian with a conservative token-aware local-mode matcher for known OpenClaw entrypoints plus a standalone `gateway` argument, add positive and negative Swift regression coverage, and provide macOS launchd/PortGuardian proof before closing.

## Reproduction Assessment

Yes, at source level: current main rejects the reported `node .../openclaw/dist/index.js gateway --port 18789` shape, and that false result reaches the local-mode kill path. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

Yes: the best fix is in the existing PortGuardian expected-process matcher, but it should use argv/token-aware checks rather than broad substring matching. A focused regression test should cover both the valid launchd Node command and misleading non-Gateway Node commands.
