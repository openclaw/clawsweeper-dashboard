The issue reports that macOS PortGuardian terminates a valid launchd-managed Gateway when the listener appears as a Homebrew Node process running `dist/index.js gateway --port 18789`.

## Reproduction Assessment

Yes, source-reproducible: current main can launch a Gateway as `node ... dist/index.js gateway`, while `PortGuardian.isExpected` does not accept that local-mode command shape. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

Yes, the narrow owner-boundary fix is to update PortGuardian expected-process detection and add regression coverage for the runtime entrypoint shape. The open linked PR appears to take that path, so it should be reviewed rather than spawning a duplicate repair.
