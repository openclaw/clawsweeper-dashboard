## Summary

Keep open: current `main` and latest `v2026.6.8` still use the title-only local PortGuardian allowlist, and the issue already has an open closing PR targeting the missing `node .../openclaw/dist/index.js gateway` command shape.

## What This Changes

The issue reports that macOS PortGuardian can terminate a valid launchd-managed Gateway when the listener command line appears as Homebrew Node running `dist/index.js gateway`.

## Best Possible Solution

Review and land the linked PortGuardian matcher PR, or replace it with an equivalent conservative matcher that accepts OpenClaw entrypoint paths plus the `gateway` subcommand while still rejecting unrelated Node listeners.

## Reproduction Assessment

Yes from source inspection: current `isExpected` returns false for `command=node` with a full command containing `dist/index.js gateway --port`, while current macOS command construction can create that exact shape. I did not run a macOS launchd live repro in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow PortGuardian local-mode matcher plus regression coverage for the real command line; the open linked PR appears to target that exact path, so this issue should stay open until that fix is reviewed and merged.
