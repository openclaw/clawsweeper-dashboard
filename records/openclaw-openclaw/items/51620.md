- `impact:session-state`: The root behavior involves process-local followup queue and active session state disappearing or being rejected during gateway lifecycle transitions.

## Summary

Keep open: current main and the latest release still keep followup queues process-local, reject drain-window enqueue without durable inbound replay, and the linked recovery PRs are still under review rather than merged.

## What This Changes

Report requests durable handling so gateway restarts do not silently drop queued followup runs, drain-window arrivals, or downtime messages.

## Best Possible Solution

Land one maintainer-approved durable restart recovery contract that persists accepted followups and drain-window arrivals, replays them through typed untrusted dispatch paths with idempotent dedupe, and leaves provider history catch-up or ACK behavior to narrower provider-owned work.

## Reproduction Assessment

Yes, source-level: a pending item in FOLLOWUP_QUEUES is process-local and disappears across a real process restart, while markGatewayDraining makes new command enqueues reject with GatewayDrainingError and current main has no durable pending-message replay path.

## Solution Assessment

Unclear for the full request: persisting accepted followups is the narrow first repair, but drain-window arrivals, provider ACK/catch-up, replay ordering, and local-state security need a typed maintainer-approved contract rather than ad hoc replay.
