## Label Justifications

- `P2`: The underlying defect breaks a bounded doctor workflow for SecretRef-backed channels, while the submitted patch currently leaves that failure path intact.

## Summary

This PR should remain open because the materialized configuration never reaches the preview hook that reads the SecretRef, so the reported doctor failure remains. The fallback also dispatches the known-unresolved configuration after resolution fails, which can reproduce the same command-aborting exception.

## What This Changes

This PR attempts to resolve channel-scoped SecretRefs before dispatching doctor preview adapters and falls back to the original configuration when resolution fails.

## Best Possible Solution

Resolve once at the generic dispatcher boundary, pass `{ ...params, cfg: resolvedCfg }` to every preview hook, and convert resolution failure into a focused non-secret diagnostic or safely skip secret-dependent preview checks rather than redispatching the unresolved config.

## Reproduction Assessment

Yes. The linked issue supplies a concrete `openclaw doctor --non-interactive` failure, and source inspection shows the unresolved raw config reaches the Nextcloud Talk preview hook that reads `botSecret`.

## Solution Assessment

No. Resolving configuration only for plugin discovery does not repair the hook invocation, and falling back to the same unresolved configuration is not a safe non-fatal path.
