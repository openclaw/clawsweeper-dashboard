## Label Justifications

- `P1`: The linked bug can silently hide aborted subagent runs from a parent session, causing message-loss/session-state impact in an active agent workflow.
- `merge-risk: 🚨 compatibility`: The intended fix changes aborted terminal runs from successful `ok` completions to error/killed outcomes for callers that may have observed the old behavior.
- `merge-risk: 🚨 session-state`: Leaving gateway dedupe and client/lifecycle normalization inconsistent can make different wait paths report different terminal state for the same aborted run.

## Summary

Keep this PR open: it is a focused and useful fix for https://github.com/openclaw/openclaw/issues/72293, but it is not merge-ready because the gateway dedupe `agent.wait` path can still expose an aborted terminal snapshot as `ok`, and the external-PR real behavior proof is still test output only.

## What This Changes

The branch adds shared aborted-stop-reason handling across lifecycle wait snapshots, subagent completion, announcement fallback, and regression tests so aborted subagent runs become terminated errors.

## Best Possible Solution

Normalize aborted stop reasons at every terminal snapshot source, including gateway dedupe, then land after real behavior proof and clean validation confirm parent-visible terminated failure semantics.

## Reproduction Assessment

Yes, source-level with high confidence. Current main classifies lifecycle `end` snapshots and subagent listener completions with `stopReason: "aborted"` as successful unless a separate `aborted: true` or blocked liveness signal is present.

## Solution Assessment

Not yet. The PR is the right narrow direction for lifecycle and subagent completion, but it should also normalize gateway dedupe snapshots so `agent.wait` has one canonical aborted-terminal contract.
