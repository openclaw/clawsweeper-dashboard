## Summary

Keep open. Current main still has the default-agent Control UI write-path bug, and this PR is pointed at the right surface, but the branch still leaves stale default-agent list overrides able to mask or pollute the corrected defaults path and lacks after-fix real UI proof.

## What This Changes

This PR adds a Control UI helper and changes default-agent primary/fallback model saves to write `agents.defaults.model` while leaving non-default agents on `agents.list[index].model`, with focused helper tests.

## Best Possible Solution

Refresh the Control UI fix so default-agent primary and fallback saves target `agents.defaults.model`, clear or ignore stale default-agent list model overrides, preserve non-default override metadata, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes, source-level: current main labels the default-agent selector as `Primary model (default)` and sends it to `onModelChange`, which writes through `resolveAgentModelFormEntry()` to `agents.list[index].model` although docs define `agents.defaults.model` as the default surface.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch should also prevent stale default-agent list overrides from masking or copying into defaults before merge.
