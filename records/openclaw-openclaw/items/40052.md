## Summary

Keep open: the underlying UI bug is real on current main, but this PR is still incomplete because stale default-agent list overrides can continue to mask the saved default, fallback saves can still copy stale entry-model data, and the PR has no after-fix real behavior proof.

## What This Changes

The PR routes default-agent primary model and fallback edits in the Control UI to `agents.defaults.model`, adds a config-path helper, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, removes or ignores stale default-agent list model overrides for that edit path, preserves non-default overrides, and covers stale/default/null-default cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: select the default agent, change its model in the Control UI, and current main writes through `agents.list[index].model` while the UI and docs distinguish the default surface as `agents.defaults.model`.

## Solution Assessment

No. Routing writes to defaults is the right direction, but the patch is incomplete until default-agent saves clear or ignore stale list-model overrides and fallback saves stop reading stale entry primaries for the default agent.
