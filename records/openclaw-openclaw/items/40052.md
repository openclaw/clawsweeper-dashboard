## Summary

Keep open: this PR targets a real Control UI bug on current main, but the patch still lets stale default-agent list overrides win or get copied into defaults, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds a Control UI helper and updates primary/fallback model handlers so default-agent edits write to `agents.defaults.model`, with focused helper tests.

## Best Possible Solution

Refresh the Control UI bug fix so default-agent primary and fallback saves update `agents.defaults.model`, clear or ignore stale default-agent list model overrides, preserve non-default overrides/fallback metadata, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes, as a source-level reproduction: current main routes the default-agent selector through `onModelChange`, and that handler writes via `resolveAgentModelFormEntry()` to `agents.list[index].model` even though the documented default surface is `agents.defaults.model`.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch should also clear or ignore stale default-agent list overrides and read defaults first for default fallback saves before merge.
