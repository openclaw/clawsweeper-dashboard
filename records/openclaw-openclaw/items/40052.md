## Summary

Keep open: the bug is valid, but this PR is not merge-ready because stale default-agent list overrides can still shadow the saved defaults, fallback edits can copy a stale primary, and real Control UI proof is still absent.

## What This Changes

The branch adds a helper and updates Control UI model-change handlers/tests so default-agent model edits write to `agents.defaults.model` while non-default agents keep `agents.list[index].model`.

## Best Possible Solution

Refresh the UI fix so default-agent primary and fallback saves update `agents.defaults.model`, clear or ignore only stale default-agent list models created by the old bug, preserve intentional non-default overrides, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes. Source inspection against current main gives a high-confidence path: the default-agent selector is labeled as a default edit, the handlers write through `agents.list[index].model`, and the overview resolves entry models before defaults.

## Solution Assessment

No. Routing default-agent writes to `agents.defaults.model` is the right direction, but the PR needs stale override handling, default-first fallback-save logic, a rebase, and real UI proof before it is the maintainable fix.
