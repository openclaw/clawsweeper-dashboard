## Summary

Keep open. The PR targets a real current-main UI config bug, but it still leaves stale default-agent list overrides authoritative, can copy a stale primary during fallback edits, lacks after-fix real Control UI proof, and is currently conflicting.

## What This Changes

This PR changes the Agents Control UI model handlers so default-agent primary and fallback edits write to `agents.defaults.model`, keeps non-default agents on `agents.list[index].model`, and adds focused config-path helper tests.

## Best Possible Solution

Refresh the Control UI fix so default-agent saves target `agents.defaults.model`, clear or ignore stale matching list overrides, preserve non-default override behavior, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes, source-level. Current main labels the default-agent selector as `Primary model (default)` while routing saves through `agents.list[index].model`, and the docs define the default model surface as `agents.defaults.model`.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch also needs stale default-agent override handling and default-first fallback primary resolution before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Clear stale default-agent overrides when saving defaults:** `ui/src/ui/app-render.ts:801-804`
  - body: When an affected config already contains the old default-agent `agents.list[].model`, this writes the new selection to `agents.defaults.model` but leaves the override that the overview resolves first, so the save can appear to do nothing. Remove or migrate the matching default-agent list model when the default save succeeds.
  - confidence: 0.92
- **[P2] Read defaults first for default fallback saves:** `ui/src/ui/app-render.ts:812-814`

With a stale old override and missing defaults primary, saving fallbacks can copy the stale primary into `agents.defaults.model`; skip `entry.model` in the default-agent branch.
  - confidence: 0.9
