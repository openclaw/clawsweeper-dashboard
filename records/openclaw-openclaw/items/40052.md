## Summary

Keep this PR open: current main still has the default-agent model write-path bug and the PR moves in the right direction, but the proposed patch leaves stale default-agent list overrides able to mask or contaminate `agents.defaults.model`, and it still lacks real after-fix UI proof.

## What This Changes

The PR changes Control UI agent model save handlers so default-agent primary and fallback edits use `agents.defaults.model`, adds a config-path helper, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI fix that routes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list overrides, preserves non-default overrides and existing fallback metadata, adds regression coverage, and includes real UI proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: default-agent model saves go through `resolveAgentModelFormEntry()` to `agents.list[index].model`, while docs define `agents.defaults.model` as the defaults surface and the overview lets list-entry models take precedence.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the branch is incomplete until stale default-agent list overrides are cleared or ignored and default fallback saves stop reading entry primaries first.
