## Summary

Keep open. Current main still has the default-agent write-path bug, and this PR is aimed at the right surface, but the branch still leaves stale default-agent overrides able to mask the corrected defaults write and has no after-fix real Control UI proof.

## What This Changes

The PR changes the Control UI model save path so default-agent primary and fallback edits write to `agents.defaults.model` while non-default agents keep writing `agents.list[index].model`.

## Best Possible Solution

Refresh the Control UI fix so default-agent primary and fallback saves target `agents.defaults.model`, clear or ignore stale default-agent list model overrides, preserve non-default override behavior, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes, source-level. Current main labels the default-agent selector as `Primary model (default)` and sends it to `onModelChange`, which writes through `resolveAgentModelFormEntry()` to `agents.list[index].model` even though the docs define `agents.defaults.model` as the default model surface.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch should also prevent stale default-agent list overrides from masking the save and should read defaults first for default fallback saves.
