## Summary

Keep open: the PR targets a real Control UI bug that is still present on current main, but the patch leaves stale default-agent list overrides in place and can still copy stale entry primaries during fallback saves.

## What This Changes

This PR routes default-agent primary and fallback model edits toward `agents.defaults.model` through a new helper and adds focused helper tests.

## Best Possible Solution

Land an updated Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or migrates the matching stale list override, covers default/non-default/null-default/stale-override cases, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: default-agent model and fallback saves on current main write through `agents.list[index].model`, while the overview reads entry overrides before `agents.defaults.model`.

## Solution Assessment

No. The PR is aimed at the right config surface, but it is not the best complete fix until stale default-agent list overrides are cleared or migrated and default fallback saves stop reading stale entry primaries.
