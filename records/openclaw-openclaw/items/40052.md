## Summary

Keep open: the PR targets a real Control UI bug, but the current patch still lets stale default-agent list overrides win and lacks after-fix real behavior proof. Current main has not implemented the fix, and the related global-model PR is still open and not a shipped superseding fix.

## What This Changes

The PR changes Control UI model handlers so default-agent primary and fallback edits write to `agents.defaults.model`, adds a config-path helper, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list model overrides on that edit path, preserves non-default overrides, adds focused stale/default/null-default coverage, and includes real UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes the default-agent model editor through `agents.list[index].model`, while docs define `agents.defaults.model` as the defaults surface and the overview gives list entries precedence.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch is incomplete until stale default-agent list overrides are cleared or ignored and default fallback saves stop reading stale entry primaries.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Clear stale default-agent overrides before saving defaults:** `ui/src/ui/app-render.ts:802-804`

  - confidence: 0.9
