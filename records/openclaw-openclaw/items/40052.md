## Summary

Keep open: the bug remains valid on current main, but this PR is not merge-ready because stale default-agent list overrides can still mask or be copied into defaults, and contributor real behavior proof is missing.

## What This Changes

This PR routes default-agent model and fallback edits in the Control UI to `agents.defaults.model`, adds `resolveAgentModelConfigPath`, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI fix that routes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list overrides, preserves non-default overrides and fallback metadata, adds regression coverage, and includes real UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows default-agent model saves still go through `resolveAgentModelFormEntry()` to `agents.list[index].model`, while docs define `agents.defaults.model` as the defaults surface and the overview lets entry models win.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the branch should clear or ignore stale default-agent list overrides and read defaults first for default fallback saves before merge.
