## Summary

Keep open: current main still has the default-agent write-path bug, and this PR is a plausible focused fix, but it remains merge-blocked by stale-override correctness holes and missing real behavior proof.

## What This Changes

This PR changes the Control UI agent model handlers to route default-agent primary and fallback saves to `agents.defaults.model`, adds a config-path helper, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI bug fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list model overrides, preserves non-default overrides and fallback metadata, adds regression coverage, and includes real UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the default-agent model selector reaches `onModelChange(agent.id, ...)`, and that handler still writes through `resolveAgentModelFormEntry()` to `agents.list[index].model` while docs define `agents.defaults.model` as the default surface.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the branch should also clear or ignore stale default-agent list overrides and avoid reading entry primary first for default fallback saves.
