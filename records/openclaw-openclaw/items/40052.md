## Summary

Keep open. Current main still has the default-agent write-path bug, and this PR moves in the right direction, but the branch still leaves stale default-agent list overrides able to mask the saved defaults and lacks required after-fix real UI proof.

## What This Changes

The PR adds a helper that routes default-agent model and fallback saves to `agents.defaults.model`, keeps non-default agents on `agents.list[index].model`, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI bug fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list model overrides, preserves non-default override behavior and fallback metadata, adds regression coverage, and includes real UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the default-agent selector calls `onModelChange(agent.id, ...)`, and that handler still writes through `resolveAgentModelFormEntry()` to `agents.list[index].model` while docs define `agents.defaults.model` as the default surface.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the branch should also clear or ignore stale default-agent list overrides and avoid reading entry primary first for default fallback saves.
