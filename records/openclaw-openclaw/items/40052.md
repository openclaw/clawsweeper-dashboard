## Summary

Keep open. Current main still has the reported Control UI write-path bug, but this PR is not ready as-is because it routes default-agent edits to `agents.defaults.model` without clearing stale default-agent list overrides, and fallback saves can still read that stale entry model; the PR also lacks after-fix real UI/runtime proof.

## What This Changes

The PR routes default-agent model and fallback saves in the Control UI from per-agent config to `agents.defaults.model` and adds a helper with focused tests.

## Best Possible Solution

Land a refreshed Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, removes or ignores stale default-agent list-model overrides on that edit path, preserves non-default overrides, and covers stale/default/null-default cases.

## Reproduction Assessment

Yes, by source inspection. Current main writes default-agent model and fallback edits through `agents.list[index].model`, while the docs define `agents.defaults.model` as the default model surface and the overview resolves list entries before defaults.

## Solution Assessment

No. Routing writes to defaults is the right direction, but the patch is incomplete until default-agent saves clear or ignore stale list-model overrides and fallback saves stop using stale entry primaries for the default agent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Clear stale default-agent overrides before saving defaults:** `ui/src/ui/app-render.ts:802-804`
  - body: When a config already has the old `agents.list[].model` value for the default agent, this writes the new selection only to `agents.defaults.model`. The overview still resolves the entry model before defaults, so affected users can save a new default and still see the stale model win; remove the matching default-agent entry model when this default save succeeds.
  - confidence: 0.92

If the old per-agent override exists and defaults lacks a primary, saving fallbacks can copy that stale entry primary into `agents.defaults.model`; skip the entry primary in the default-agent branch.
  - confidence: 0.9
