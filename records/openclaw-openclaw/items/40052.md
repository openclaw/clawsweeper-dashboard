## Summary

Keep open. The PR addresses a real current-main Control UI bug, but the patch is still incomplete because stale default-agent list overrides can continue to win and default fallback saves can copy stale entry-model data into `agents.defaults.model`; after-fix real behavior proof is also missing.

## What This Changes

The PR changes Control UI agent model handlers to write default-agent primary and fallback edits to `agents.defaults.model`, adds a config-path helper, and covers the helper with focused tests.

## Best Possible Solution

Land a refreshed Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list model overrides on that edit path, preserves non-default overrides, and adds focused stale/default/null-default coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current main writes the default-agent model editor through `agents.list[index].model`, while the documented defaults surface is `agents.defaults.model` and the overview gives list entries precedence.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch is not complete until stale default-agent list overrides are cleared or ignored and default fallback saves stop reading stale entry primaries.
