## Summary

Keep open: current main still has the reported UI write-path bug, but this branch is not merge-ready because it leaves stale default-agent overrides authoritative, can copy stale primaries during fallback edits, has conflicts, and lacks real behavior proof.

## What This Changes

The PR routes default-agent model and fallback edits from the Control UI to `agents.defaults.model`, keeps non-default agents on `agents.list[index].model`, and adds helper coverage.

## Best Possible Solution

Refresh the UI fix so default-agent primary and fallback saves update `agents.defaults.model`, clear or ignore only the stale default-agent list model created by the old bug, preserve intentional non-default overrides, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main labels the default-agent selector as a default edit while writing through `agents.list[index].model`, and the overview resolves entry models before defaults.

## Solution Assessment

No. Routing default-agent writes to `agents.defaults.model` is the right direction, but this patch needs stale override handling, default-first fallback-save logic, a rebase, and real UI proof before it is the maintainable fix.
