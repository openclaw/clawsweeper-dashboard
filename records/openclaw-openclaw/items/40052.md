## Summary

Keep open. Current main still routes the default-agent Control UI model selector through the agent-list write path, but this PR is not ready because it leaves stale default-agent overrides able to mask the corrected defaults write and lacks inspectable real UI proof; the overlapping work in https://github.com/openclaw/openclaw/pull/40723 and https://github.com/openclaw/openclaw/issues/76128 does not supersede this UI save-path fix.

## What This Changes

The PR adds a Control UI helper to route default-agent primary model and fallback saves to `agents.defaults.model` while keeping non-default agents on `agents.list[index].model`, with focused helper tests.

## Best Possible Solution

Land a refreshed Control UI bug fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list model overrides, preserves non-default override behavior and fallback metadata, adds regression coverage, and includes real UI proof.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main shows the default-agent selector calls the shared model-change handler, and that handler writes through `resolveAgentModelFormEntry()` to `agents.list[index].model` while the documented default surface is `agents.defaults.model`.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch should also clear or ignore stale default-agent list overrides and read defaults first for default fallback saves before merge.
