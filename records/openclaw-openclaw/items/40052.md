## Summary

Keep open: current main still has the reported bug and this PR targets the right write surface, but the patch remains incomplete because stale default-agent list overrides can still win and fallback saves can still copy a stale primary into defaults.

## What This Changes

The PR adds a helper for default-vs-per-agent model config paths, routes default-agent primary/fallback saves toward `agents.defaults.model`, and adds focused helper tests.

## Best Possible Solution

Land an updated Control UI fix that saves default-agent primary and fallback edits to `agents.defaults.model`, clears stale default-agent list overrides, preserves non-default overrides, covers stale/default/null-default cases, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows default-agent model and fallback saves write through `agents.list[index].model`, while the docs expose `agents.defaults.model` as the default model config and the overview resolves entry model before defaults.

## Solution Assessment

No. The PR moves writes toward the right config surface, but it is not complete until it clears or migrates stale default-agent list overrides and stops default fallback saves from reading stale entry primaries.
