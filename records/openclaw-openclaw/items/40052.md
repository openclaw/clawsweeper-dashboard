## Summary

Keep this PR open. Current main still has the default-agent model save bug, and this PR is useful source work, but the proposed patch is not merge-ready because stale `agents.list[].model` overrides created by the old path can still shadow or be copied into `agents.defaults.model`.

## What This Changes

The PR routes default-agent primary and fallback model edits in the Control UI toward `agents.defaults.model`, keeps non-default agents on `agents.list[index].model`, and adds focused helper tests.

## Best Possible Solution

Land an updated Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, removes or migrates the matching stale default-agent `agents.list[].model` override during the save path, avoids reading stale entry primaries for default-agent fallback saves, covers default/non-default/null-default/stale-override cases, and adds the required changelog entry.

## Reproduction Assessment

Yes. The current-main code path is enough to reproduce with high confidence: editing the default agent writes through `agents.list[index].model`, and the overview prefers that entry value over `agents.defaults.model`. The linked #40065 minimal reproduction in the discussion matches this path.

## Solution Assessment

No. The PR points at the right config surface, but it is not the best complete fix until stale default-agent list overrides are cleared or migrated and fallback saves stop copying stale entry primaries into defaults.
