## Summary

Keep this PR open. Current main still has the default-agent model save bug, and the PR is useful source work, but the branch is not merge-ready because existing stale default-agent `agents.list[].model` overrides can still shadow or be copied into `agents.defaults.model`.

## What This Changes

The PR adds a helper for agent model config-path resolution, routes default-agent primary and fallback edits toward `agents.defaults.model`, keeps non-default agents on `agents.list[index].model`, and adds focused helper tests.

## Best Possible Solution

Land an updated Control UI fix that routes default-agent primary and fallback edits to `agents.defaults.model`, keeps non-default agent edits on `agents.list[].model`, removes or migrates the matching stale default-agent list override during the save path, and covers default-agent, non-default-agent, and `defaultId === null` path resolution.

## Reproduction Assessment

Yes. The current-main code path and the linked #40065 reproduction show a high-confidence mismatch: editing the default agent writes a per-agent override while `agents.defaults.model` remains unchanged, and the overview then prefers the override.

## Solution Assessment

No. The PR takes the right direction for the write path, but unchanged it is not the best fix because stale default-agent overrides can keep shadowing the new defaults value and fallback edits can still copy stale entry data into defaults.
