## Summary

Keep open: the source-level bug is real, but this branch is not merge-ready because it still leaves stale default-agent list overrides authoritative, can copy a stale entry primary during fallback edits, lacks after-fix real Control UI proof, and is currently conflicting with main.

## What This Changes

The PR changes the Control UI agent model handlers so default-agent primary and fallback edits write to `agents.defaults.model`, adds a path helper, and adds focused helper tests.

## Best Possible Solution

Refresh the UI fix so default-agent primary and fallback saves update `agents.defaults.model`, clear or ignore only stale default-agent list-model shadowing created by the old bug, preserve intentional non-default overrides, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes, at source level. Current main labels the default-agent selector as a default edit while writing through `agents.list[index].model`, and the overview resolves entry models before defaults.

## Solution Assessment

No. Routing default-agent writes to `agents.defaults.model` is the right direction, but the PR needs stale override handling, default-first fallback-save logic, a rebase, and real UI proof before it is the maintainable fix.
