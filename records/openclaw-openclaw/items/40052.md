## Summary

Keep open: the bug is real and the PR points at the right save path, but merging it as-is would leave stale default-agent overrides authoritative for affected users, and fallback saves can still copy stale entry models into defaults. It also lacks after-fix real Control UI proof and is currently conflicting.

## What This Changes

The PR adds a helper and updates Agents UI model handlers and tests so default-agent primary and fallback edits target `agents.defaults.model` instead of `agents.list[index].model`.

## Best Possible Solution

Refresh the Control UI fix so default-agent primary and fallback saves target `agents.defaults.model`, clear or ignore only the matching stale default-agent list model override, preserve non-default overrides, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main labels the selected default-agent control as `Primary model (default)`, but saves through the per-agent `agents.list[index].model` path while the documented default surface is `agents.defaults.model`.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch also needs stale default-agent override handling and default-first fallback-save logic before it is the narrow maintainable fix.
