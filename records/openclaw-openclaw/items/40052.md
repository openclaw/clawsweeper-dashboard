## What This Changes

The PR changes the Control UI default-agent primary model and fallback save paths to write `agents.defaults.model` instead of `agents.list[index].model`, with a helper and focused tests.

## Best Possible Solution

Refresh the UI fix so default-agent primary and fallback saves update `agents.defaults.model`, handle stale default-agent list-model shadowing without disturbing non-default overrides, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main labels the default-agent selector as a default edit while the save handler writes through `agents.list[index].model`, and the overview resolves entry models before defaults.

## Solution Assessment

No. Routing default-agent writes to `agents.defaults.model` is the right direction, but the current patch needs stale override handling, default-first fallback-save logic, a branch refresh, and real UI proof before it is the maintainable fix.
