## Summary

Keep open: current main still has the documented Control UI bug, but this PR is not ready because stale default-agent list overrides can still shadow saved defaults, fallback saves can still copy stale entry primaries, and the user-facing fix lacks a changelog entry.

## What This Changes

The PR routes default-agent primary and fallback model edits toward `agents.defaults.model`, adds a config-path helper, and adds focused helper tests.

## Best Possible Solution

Land an updated Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or migrates the matching stale list override, adds default/non-default/null-default/stale-override coverage, and records the fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path. On current main, default-agent model saves write through `agents.list[index].model` while the overview reads entry model before defaults, so the UI can show an override that does not update `agents.defaults.model`.

## Solution Assessment

No. The PR targets the right config surface, but it is not the best complete fix until stale default-agent list overrides are cleared or migrated and default fallback saves stop reading stale entry primaries.
