## Summary

Keep open: this targets a real current-main UI config bug, but the patch still lets stale default-agent list overrides mask the newly saved default and can copy a stale entry primary into defaults during fallback edits; it also lacks after-fix real UI proof.

## What This Changes

The PR changes the Control UI agents model handlers to route default-agent primary and fallback edits through `agents.defaults.model`, adds a path helper, and tests that helper.

## Best Possible Solution

Land a refreshed Control UI fix that writes default-agent primary and fallback edits to `agents.defaults.model`, removes or ignores stale default-agent list model overrides for that edit path, preserves non-default overrides, and covers stale/default/null-default cases.

## Reproduction Assessment

Yes, by source inspection. Current main writes default-agent model and fallback edits through `agents.list[index].model`, while docs define `agents.defaults.model` as the default model surface and the overview resolves list entries before defaults.

## Solution Assessment

No. Routing writes to defaults is the right direction, but the patch is not complete until default-agent saves clear or ignore stale list-model overrides and fallback saves stop using stale entry primaries for the default agent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Clear stale default-agent overrides before saving defaults:** `ui/src/ui/app-render.ts:802-804`
  - body: When a config already has the old `agents.list[].model` value for the default agent, this writes the new selection only to `agents.defaults.model`. The overview still resolves the entry model before defaults, so affected users can save a new default and still see the stale model win; remove the matching default-agent entry model when this default save succeeds.
  - confidence: 0.92
- **[P2] Read defaults first for default fallback saves:** `ui/src/ui/app-render.ts:812-814`
  - body: For default-agent fallback edits, `effectivePrimary` still reads `resolvedConfig.entry?.model` before defaults. If the old per-agent override exists and defaults lacks a primary, saving fallbacks can copy that stale entry primary into `agents.defaults.model`; skip the entry primary in the default-agent branch.
  - confidence: 0.9
