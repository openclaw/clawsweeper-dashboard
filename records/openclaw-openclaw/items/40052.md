## Summary

Keep open: current main still has the bug and this PR is a plausible focused fix, but the patch is incomplete because stale default-agent list overrides can still mask defaults and the PR lacks inspectable after-fix real behavior proof.

## What This Changes

The branch changes Control UI agent model handlers so default-agent primary/fallback edits write through a new defaults-path helper, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI fix that routes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list overrides on those edit paths, preserves non-default overrides, adds regression coverage, and includes real UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes the default-agent model editor through `agents.list[index].model`, while docs define `agents.defaults.model` as the defaults surface and the overview gives list entries precedence.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch is incomplete until stale default-agent list overrides are cleared or ignored and default fallback saves stop reading stale entry primaries.
