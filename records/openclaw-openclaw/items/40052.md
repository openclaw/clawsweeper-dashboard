## Summary

Keep open: this PR targets a real Control UI default-model save bug, but the branch still lets stale default-agent list overrides mask the new defaults write and it lacks required after-fix real behavior proof.

## What This Changes

The PR changes the Agents Control UI model handlers so default-agent primary and fallback edits write to `agents.defaults.model`, while non-default agents continue using `agents.list[index].model`, and adds focused helper tests.

## Best Possible Solution

Refresh the Control UI fix so default-agent primary and fallback saves target `agents.defaults.model`, remove or ignore stale default-agent list model overrides, preserve non-default override behavior, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes, source-level. Current main labels the default-agent selector as `Primary model (default)` and routes it through a helper that writes `agents.list[index].model`, while the docs define the default model surface as `agents.defaults.model`.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch also needs to clear or ignore stale default-agent list overrides and read defaults first for default fallback saves.
