## Summary

Keep open. The PR moves the default-agent write path in the right direction, but it still lets stale default-agent list overrides mask or pollute `agents.defaults.model`, and it lacks after-fix real Control UI proof.

## What This Changes

The PR adds a Control UI helper for resolving agent model config paths, routes default-agent model and fallback saves toward `agents.defaults.model`, and adds focused helper tests.

## Best Possible Solution

Land a refreshed Control UI bug fix that writes default-agent primary and fallback edits to `agents.defaults.model`, clears or ignores stale default-agent list model overrides, preserves non-default override behavior and fallback metadata, adds regression coverage, and includes real UI proof.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main shows the default-agent selector calls the shared model-change handler, and that handler writes through `resolveAgentModelFormEntry()` to `agents.list[index].model` while the documented default surface is `agents.defaults.model`.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but this branch should also clear or ignore stale default-agent list overrides and avoid reading entry primary first for default fallback saves.

## Review Findings
