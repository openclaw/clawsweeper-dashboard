## What This Changes

## Best Possible Solution

Refresh the Control UI fix so default-agent primary and fallback saves update `agents.defaults.model`, remove or ignore only the matching stale default-agent list model override, preserve non-default overrides, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main labels the default-agent selector as a default edit while writing through `agents.list[index].model`, and the overview resolves entry models before defaults.

## Solution Assessment

No. Routing default-agent writes to `agents.defaults.model` is the right direction, but the PR needs stale override handling, default-first fallback-save logic, and real UI proof before it is the maintainable fix.
