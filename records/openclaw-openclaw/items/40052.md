## Summary

Keep open: this is a valid UI model-selection bug, but the proposed patch is not merge-ready because it leaves the stale default-agent override path authoritative for existing affected configs and still lacks after-fix real Control UI proof.

## What This Changes

The PR changes the Agents Control UI model handlers so default-agent primary and fallback edits write to `agents.defaults.model`, keeps non-default edits on `agents.list[index].model`, and adds focused helper tests.

## Best Possible Solution

Refresh the Control UI fix so default-agent primary and fallback saves target `agents.defaults.model`, remove or ignore only the matching stale default-agent list model override, preserve non-default overrides, add regression coverage, and include real UI proof.

## Reproduction Assessment

Yes, source-level. Current main labels the default-agent selector as `Primary model (default)` but routes the save through the per-agent model path, while docs define the default model surface as `agents.defaults.model`.

## Solution Assessment

No. Routing default-agent writes to defaults is the right direction, but the patch also needs stale default-agent override handling and default-first fallback-save logic before it is the narrow maintainable fix.
