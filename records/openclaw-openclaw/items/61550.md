## Summary

Keep open. Current main still lets Telegram active-stream finalization mark a visible final preview as delivered without entering the normal `deliverReplies` path where `message_sending` rewrite/cancel hooks run; related broader hook issues do not replace this narrow Telegram finalization bug.

## What This Changes

The issue requests that Telegram streaming final delivery run `message_sending` hooks the same way non-streaming delivery does.

## Best Possible Solution

Wire the existing `message_sending` rewrite/cancel contract into Telegram streaming final preview finalization exactly once, while preserving one visible preview and leaving broader partial-edit hook semantics to the related design issue.

## Reproduction Assessment

Yes, from source inspection. Active Telegram streaming finals can return `preview-finalized` without `sendPayload`/`deliverReplies`, while normal Telegram delivery is the only path that runs `message_sending`.

## Solution Assessment

Yes for this narrow bug. The maintainable fix is to apply the existing Telegram outbound `message_sending` contract before final preview finalization, with once-only rewrite/cancel coverage and no extra final message bubble.
