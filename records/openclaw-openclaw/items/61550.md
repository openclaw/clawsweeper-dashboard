## Summary

Keep open. Current `main` still has a Telegram streaming finalization path that marks an active preview as delivered without passing through the normal hook-backed reply delivery path, so `message_sending` can still be skipped for visible final text.

## What This Changes

The issue asks for Telegram streaming final replies to fire `message_sending` hooks the same way non-streaming replies do.

## Best Possible Solution

Apply the same Telegram outbound `message_sending` contract before final preview edits or retained final previews, honoring rewrite and cancellation while preserving single-preview streaming behavior.

## Reproduction Assessment

Yes, from source inspection. On current `main`, an active Telegram stream can finalize visible final text as `preview-finalized` without `sendPayload`, while `message_sending` only runs in `deliverReplies`.

## Solution Assessment

Yes. The narrow maintainable fix is inside the Telegram streaming finalization path, with focused coverage for hook rewrite, cancellation, no-hook behavior, retained-preview cases, and normal send fallback.
