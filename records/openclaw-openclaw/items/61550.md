- `impact:security`: Outbound guardrail plugins such as content filters or URL validators can be bypassed when streamed Telegram finals skip `message_sending`.

## Summary

Keep open. Current main still allows Telegram streaming finalization to mark the visible preview as delivered without routing the final text through the `deliverReplies` path that applies `message_sending` rewrite/cancel hooks.

## What This Changes

The issue reports that Telegram partial/full streaming finalization bypasses the documented `message_sending` hook for the final delivered message.

## Best Possible Solution

Patch Telegram stream finalization to run the existing `message_sending` rewrite/cancel contract exactly once before marking a final preview delivered, while preserving one visible preview and leaving broader streaming-edit API design to https://github.com/openclaw/openclaw/issues/65535.

## Reproduction Assessment

Yes, from source inspection. Active Telegram streaming finals can return `preview-finalized` without `sendPayload`/`deliverReplies`, while normal Telegram delivery is the only inspected path that runs `message_sending`.

## Solution Assessment

Yes for this narrow issue. The maintainable fix is to apply the existing outbound hook contract before final preview finalization exactly once, rather than adding a second final bubble or solving the broader streaming-edit lifecycle here.
