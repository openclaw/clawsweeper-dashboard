## Summary

Keep open. Current main still lets Telegram streaming finalize text-only preview messages through `editMessageTelegram` without passing through `deliverReplies`, while `message_sending` rewrite/cancel handling remains in `deliverReplies`; closed PR #62676 is useful prior work but was not merged.

## What This Changes

The issue asks for Telegram streaming final preview delivery to run `message_sending` hooks so plugins can rewrite or cancel outbound content.

## Best Possible Solution

Apply `message_sending` inside the Telegram final preview-edit seam before any visible final edit, honoring rewrites and cancellation while avoiding a second hook pass on fallback sends.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the final text-only streaming branch edits or retains the preview without `sendPayload`, and `message_sending` is only applied in `deliverReplies`.

## Solution Assessment

Yes. The narrowest maintainable fix is Telegram-owned handling around final preview edits, not a broad core dispatcher rewrite for this issue.

## Repair Work Prompt

Fix the Telegram streaming final-preview `message_sending` bypass for this issue. On current main, text-only final replies can finalize by editing an existing Telegram preview instead of calling `sendPayload`/`deliverReplies`, so plugins cannot rewrite or cancel via `message_sending`. Keep the fix inside the Telegram plugin boundary: apply the same `message_sending` contract before final preview edits, use rewritten content for the final edit, honor cancellation without falsely marking a final delivery, and avoid running the hook twice when falling back to the normal `sendPayload` path. Inspect closed PR #62676 as source context, but do not copy unrelated additions such as new plugins, dependency/lockfile churn, or broad core API changes. Preserve existing duplicate-avoidance and fallback behavior for retained previews, archived previews, stale-preview fresh-final sends, regressive skips, and edit errors. Add focused regression tests for final preview rewrite, cancel, no-hook, fallback after rewrite, archived-preview handling, and normal send fallback. Add a user-facing `CHANGELOG.md` fix entry unless maintainers classify the final patch as test-only.
