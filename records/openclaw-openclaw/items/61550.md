## Summary

Keep open. Current main still lets Telegram streaming finalization edit or retain a preview without passing through the hook-backed `deliverReplies` path, so plugins using `message_sending` can still miss final Telegram replies.

## What This Changes

Ensure Telegram streaming final preview delivery fires `message_sending` hooks for outgoing messages just like non-streaming Telegram sends.

## Best Possible Solution

Telegram final preview edits should apply the same outbound hook contract as normal final sends before any visible final edit, honoring rewrite and cancel semantics inside the Telegram plugin boundary.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: with Telegram streaming preview active, a text-only final reply can edit or retain the preview and skip `sendPayload`, while `message_sending` only runs inside `deliverReplies`.

## Solution Assessment

Yes. The narrowest maintainable fix is a Telegram-owned hook application around final preview delivery, with regression tests for edit, rewrite, cancel, fallback, and no-hook behavior.
