Full review comments:

- **[P1] Normalize per-job hook paths before validating them:** `src/cron/hooks.ts:201-215`
  - body: `isValidJobHookPath` checks the raw persisted string, but `loadHookModule` later passes it through `resolveUserPath`, which trims and resolves against the base directory. A stored value like `" /etc/passwd"` or `" ../outside/hook.cjs"` can pass the raw absolute/traversal checks and then import outside the intended per-job relative scope.
  - confidence: 0.93
- **[P2] Narrow persisted hook entries before path operations:** `src/cron/hooks.ts:69-74`
  - body: `job.hooks?.[hookPoint]` comes from persisted job data, but the loop assumes every entry is a string before calling `path.isAbsolute`. A malformed stored row such as `[123]` can throw while building hook entries, before the normal cron run finalization path has a chance to complete cleanly.
  - confidence: 0.86
