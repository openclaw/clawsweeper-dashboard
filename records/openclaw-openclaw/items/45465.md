## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Normalize per-job hook paths before validation:** `src/cron/hooks.ts:201-212`
  - body: `isValidJobHookPath` checks the raw persisted string, but `loadHookModule` later imports the value after `resolveUserPath` trims and expands it. Values like `" /abs/hook.cjs"` or `" ../outside/hook.cjs"` can pass the guard and then load outside the intended relative per-job scope.
  - confidence: 0.94
- **[P2] Validate per-job hook entries before path checks:** `src/cron/hooks.ts:67-71`
  - body: `job.hooks` comes from persisted job data, but this loop assumes the selected hook point is a string array. A malformed value such as `[123]` can throw inside the path helper before normal hook isolation and finalization, leaving the run to stale-run recovery.
  - confidence: 0.89
- **[P2] Reuse stable hook import specifiers:** `src/cron/hooks.ts:192`
  - body: Every file-backed hook import uses `cacheBust: true`, so recurring cron hooks create a fresh ESM module instance on each run. Long-lived Gateway processes should reuse stable imports or expose an explicit reload mechanism instead of growing the module cache by default.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds unattended cron-triggered local module execution and still has concrete per-job path hardening gaps.

Concerns:

- **[medium] Per-job hook path restriction can be bypassed:** `src/cron/hooks.ts:201`
  - body: Per-job hooks are intended to be relative filesystem paths, but the safety check uses the raw value while import resolution trims and expands it later. Whitespace-prefixed absolute or traversal paths can evade the restriction and load code outside the intended scope.
  - confidence: 0.94
- **[medium] Define scheduled local module execution boundaries:** `src/cron/hooks.ts:181`
  - body: Global and per-job cron hook settings execute local modules during scheduled, startup catch-up, and manual runs. This unattended code-execution surface needs documented operator scope, path/env/secret handling, approval interaction, timeout/cancellation, resource limits, and logging behavior before merge.
  - confidence: 0.86
