## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Normalize hook paths before enforcing the job guard:** `src/cron/hooks.ts:201-212`
  - body: The per-job guard checks the raw value, but `loadHookModule` later resolves the same string through `resolveUserPath`, which trims first. A stored value like `" /etc/passwd"` or `" ../outside/hook.cjs"` can pass this check and then import outside the intended relative hook scope, so trim once and validate the exact string that will be resolved.
  - confidence: 0.91
- **[P2] Validate persisted hook arrays before iterating:** `src/cron/hooks.ts:67-74`
  - body: `job.hooks` comes from persisted jobs, and the loader assumes `job.hooks[hookPoint]` is a string array before iterating and passing entries to `path.isAbsolute`. A malformed stored value can throw before normal cron result finalization, leaving running state and scheduled work to stale recovery; normalize the shape or ignore invalid entries with a warning.
  - confidence: 0.88
- **[P2] Stop cache-busting every hook import:** `src/cron/hooks.ts:191-192`
  - body: Every hook run calls `importFileModule` with `cacheBust: true`, and the loader appends a timestamp query for that mode. In a long-lived gateway, repeated cron executions of the same hook create unique ESM module specifiers indefinitely; reuse stable imports or add an explicit bounded reload mechanism.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Needs attention: this PR adds unattended dynamic module execution and still has a per-job hook path validation bypass.

Concerns:

- **[high] Per-job path validation bypass:** `src/cron/hooks.ts:201`
  - body: The guard checks raw hook strings but the runtime resolver trims before importing, so whitespace-prefixed absolute or traversal values can bypass the intended relative-only jobs.json boundary.
  - confidence: 0.91
- **[medium] Unattended executable/module policy not settled:** `src/cron/hooks.ts:181`
  - body: Cron hooks would execute operator-provided modules around scheduled jobs, so the merge needs explicit policy for trust scope, cwd/path resolution, environment and secrets, timeouts, logging, and audit semantics.
  - confidence: 0.82
- **[medium] Unbounded dynamic import lifetime:** `src/cron/hooks.ts:192`
  - body: Cache-busted hook imports produce unique module specifiers per run, which can grow process memory in a long-lived Gateway even when the set of configured hook files is small.
