## Review Findings

Overall confidence: 0.94

Full review comments:

- **[P1] Normalize per-job paths before validating them:** `src/cron/hooks.ts:201-212`
  - body: `isValidJobHookPath` checks the raw value, but `loadHookModule` later imports the trimmed value through `resolveUserPath`. Values such as `" /absolute/hook.cjs"` or `" ../outside.cjs"` can pass this guard and then escape the intended relative-only per-job boundary.
  - confidence: 0.98
- **[P2] Narrow persisted hook entries before path checks:** `src/cron/hooks.ts:69-74`
  - body: This loop assumes every persisted hook entry is a string. A malformed jobs store entry such as `123` reaches `path.isAbsolute` and can throw before normal run finalization, leaving cron execution to recover through stale-run handling instead of a controlled skip.
  - confidence: 0.94

## Security Review

Status: needs_attention
