## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Persist hooks through the current SQLite cron store:** `src/cron/types.ts:146`
  - body: This PR adds `CronJob.hooks`, but current main persists cron jobs through the SQLite row codec and gateway-protocol package schemas, neither of which include hooks. Without porting the field through those current storage/protocol paths, user hook config can be rejected or dropped across reads, writes, and API calls.
  - confidence: 0.9
- **[P1] Normalize per-job hook paths before enforcing restrictions:** `src/cron/hooks.ts:201-217`
  - body: `isValidJobHookPath` checks the raw string, but `loadHookModule` later resolves it through `resolveUserPath`, which trims first. Values like `" /etc/passwd"` or `" ../outside/hook.cjs"` can pass this guard and then resolve outside the intended per-job relative path boundary.
  - confidence: 0.91
- **[P2] Validate per-job hook entry types before path checks:** `src/cron/hooks.ts:70`
  - body: `loadHookEntries` assumes persisted `job.hooks[hookPoint]` is a string array and sends each value directly into `path` checks. A malformed persisted entry such as `[123]` can throw before normal cron result finalization, leaving the run dependent on stale-run recovery.
  - confidence: 0.86
- **[P2] Avoid cache-busted imports for recurring hooks:** `src/cron/hooks.ts:192`
  - body: Every file hook import passes `cacheBust: true`, and the shared loader implements that by adding a unique query string. Recurring cron hooks will therefore accumulate distinct ESM module instances in the long-lived gateway process.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR adds gateway-local execution of user-configured hook modules and contains concrete path-boundary and long-lived-process resource concerns.

Concerns:

- **[high] Whitespace bypasses per-job path guard:** `src/cron/hooks.ts:201`
  - body: Per-job hook validation checks raw script paths before trimmed resolution, so whitespace-prefixed absolute or traversal paths can bypass the intended restriction and load files outside the relative hook scope.
  - confidence: 0.91
- **[medium] Executable cron hooks need an approved trust boundary:** `src/cron/hooks.ts:181`
  - body: The feature runs user-specified scripts inside the gateway process from both global config and persisted jobs, which is security-sensitive and needs a maintainer-approved API, path policy, and upgrade story.
  - confidence: 0.84
- **[medium] Cache-busted recurring imports can grow process memory:** `src/cron/hooks.ts:192`
  - body: Recurring hook execution imports a unique module URL for the same hook file each run, creating a denial-of-service style resource growth risk in a long-lived gateway.
  - confidence: 0.82
