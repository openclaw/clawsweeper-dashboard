## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Normalize per-job paths before validating them:** `src/cron/hooks.ts:202-214`
  - body: Per-job hooks are validated before `resolveUserPath()` trims and resolves them, so a value like ` /absolute/hook.cjs` or ` ../outside.cjs` can pass the raw checks and then import outside the intended relative hook boundary. Trim and resolve once, then validate that same value before loading.
  - confidence: 0.93
- **[P2] Narrow persisted hook entries before path operations:** `src/cron/hooks.ts:71-72`
  - body: `job.hooks?.[hookPoint]` is treated as strings from persisted state, but malformed JSON can put a non-string there; that reaches `path.isAbsolute` and throws before the job can finalize cleanly. Check `typeof script === "string"` or normalize the persisted shape before validation.
  - confidence: 0.86
