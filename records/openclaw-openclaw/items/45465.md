## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Normalize per-job paths before validating them:** `src/cron/hooks.ts:201`
  - body: Per-job hook paths are checked before `resolveUserPath()` trims them, so a value like `" /tmp/hook.cjs"` passes the relative-path guard here and is later trimmed into an absolute filesystem path for import. That bypasses the intended per-job workspace boundary.
  - confidence: 0.9
- **[P1] Stop cache-busting hook imports on every run:** `src/cron/hooks.ts:187`
  - body: `importFileModule` appends a timestamp query when `cacheBust` is true, and Node keeps each unique ESM specifier cached for the process lifetime. A recurring cron hook will therefore create a fresh module instance every run and can grow gateway memory indefinitely.
  - confidence: 0.84
- **[P2] Narrow persisted hook entries before path operations:** `src/cron/hooks.ts:69`
  - body: `job.hooks[hookPoint]` comes from persisted job data, but this loop assumes every element is a string. A malformed value such as `[123]` makes `path.isAbsolute()` throw before normal cron finalization, which can leave run markers uncleared until recovery.
  - confidence: 0.82
