## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Normalize per-job hook paths before validation:** `src/cron/hooks.ts:201-212`
  - body: `isValidJobHookPath` checks the raw value, but `loadHookModule` later imports the trimmed value through `resolveUserPath`. A persisted per-job hook like ` /etc/passwd` or ` ../outside/hook.cjs` passes the relative/traversal checks and is then imported outside the intended workspace boundary; trim and type-narrow once before both validation and import.
  - confidence: 0.98
- **[P2] Narrow persisted hook entries before path checks:** `src/cron/hooks.ts:69-74`
  - body: `job.hooks[hookPoint]` comes from persisted job state, but this loop assumes every entry is a string before calling path helpers. A malformed entry such as `123` can throw before normal run finalization; validate or quarantine bad entries at the store or hook boundary.
  - confidence: 0.94
- **[P2] Avoid cache-busted imports for recurring hooks:** `src/cron/hooks.ts:192`
  - body: Every file hook import uses `cacheBust: true`, which creates a fresh ESM specifier on each cron execution. Recurring jobs can retain unique module instances for the gateway lifetime; use stable imports with an explicit reload owner or another bounded lifecycle.
  - confidence: 0.88
