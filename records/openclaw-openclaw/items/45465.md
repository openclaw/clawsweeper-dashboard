## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Persist hook config through SQLite cron rows:** `src/cron/types.ts:146`
  - body: Adding `job.hooks` makes per-job hooks a runtime/persisted job field, but current main loads scheduler jobs from SQLite column projections in `rowToCronJob()` and does not merge `job_json` extras back into runtime jobs. After a current-main rebase, per-job hooks would be lost across persistence/restart unless the SQLite codec, normalization, protocol schemas, and upgrade tests all preserve this field.
  - confidence: 0.88
- **[P1] Reuse hook modules instead of cache-busting every run:** `src/cron/hooks.ts:192`
  - body: `loadHookModule()` imports every filesystem hook with `cacheBust: true`; the shared loader turns that into a timestamped file URL, so a scheduled hook imports a fresh ESM module instance on every run. Long-lived gateways can accumulate unbounded module instances for a recurring cron hook, so hook loading needs a stable module cache or explicit bounded reload mechanism.
  - confidence: 0.86
- **[P1] Normalize per-job hook paths before validation:** `src/cron/hooks.ts:201`
  - body: The per-job safety check validates the raw `scriptPath`, but the import path later goes through `resolveUserPath()`, which trims before resolving. A value with leading whitespace can bypass the absolute/traversal checks and then be imported as the trimmed path; malformed non-string persisted entries can also throw in the validator before normal run finalization.
  - confidence: 0.9
