## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Trim per-job hook paths before safety checks:** `src/cron/hooks.ts:201-212`
  - body: `loadHookEntries` validates the raw persisted path, but `loadHookModule` later resolves the same value through `resolveUserPath`, which trims first. A value such as `" /etc/passwd"` or `" ../outside/hook.cjs"` can pass the per-job guard and then import outside the intended relative hook scope; trim once and validate/import that same normalized value.
  - confidence: 0.92
- **[P2] Validate persisted hook entries before path checks:** `src/cron/hooks.ts:70-71`
  - body: `jobs.json` is persisted input, but this loop assumes every `job.hooks[hookPoint]` element is a string. A malformed entry like `[123]` can throw inside `path.isAbsolute` before normal cron result finalization, leaving runs dependent on stale-run recovery instead of cleanly skipping the bad hook entry.
  - confidence: 0.86
- **[P2] Avoid per-run cache-busted hook imports:** `src/cron/hooks.ts:192`
  - body: `cacheBust: true` makes each cron hook execution import a timestamped module specifier instead of reusing a stable module entry. For recurring jobs in a long-lived gateway this can grow loaded module instances over time; use stable imports by default and make reload semantics explicit if hot-reload is required.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces cron-triggered host-side script/module execution and still has a concrete per-job path-boundary bypass.

Concerns:

- **[high] Per-job hook path guard can be bypassed:** `src/cron/hooks.ts:201`
  - body: Per-job hook paths are meant to be relative-only because jobs.json may be less trusted than openclaw.json, but the guard checks the untrimmed raw value while import resolution trims later, allowing spaced absolute or traversal paths to escape the intended boundary.
  - confidence: 0.92
- **[medium] Define the cron hook execution trust boundary:** `src/cron/hooks.ts:181`
  - body: Global hooks may import arbitrary specifiers and per-job hooks run unattended around scheduled jobs, so maintainers need explicit policy for admin scope, cwd/base path, env and secret exposure, logging, timeout, cancellation, and audit behavior.
  - confidence: 0.86
