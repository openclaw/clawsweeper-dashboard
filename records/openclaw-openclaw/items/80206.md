## What This Changes

The issue asks the Skills UI/Gateway status path to recognize installed `clawhub` and `mcporter` binaries from npm/pnpm global locations instead of showing missing requirements.

## Best Possible Solution

Land or rework https://github.com/openclaw/openclaw/pull/85238, or an equivalent narrow PATH-builder fix, then close this issue after current-main proof shows package-manager bins remain after trusted system directories.

## Reproduction Assessment

Yes, source-reproducible: start the gateway with a PATH that lacks the npm/pnpm global bin directory while a skill declares that binary in `requires.bins`, and the status path will report it missing. I did not run a live macOS gateway reproduction.

## Solution Assessment

Yes, the best fix is in the gateway service/runtime PATH builders, not the UI rendering layer. The linked PR is the right current path, though it needs conflict handling or equivalent rework before landing.
