## Summary

Keep open: current main still has an Anthropic/Claude CLI inspection mismatch, at least in `models auth list --provider anthropic`, where the profile is summarized as `claude-cli` and then filtered out.

## What This Changes

After Claude CLI Anthropic login, Anthropic-facing auth inspection should treat `anthropic:claude-cli` as usable Anthropic auth.

## Best Possible Solution

Make model-auth inspection use a shared canonical provider/auth-alias path so Claude CLI profiles satisfy Anthropic-facing list/status output while preserving `claude-cli` as the runtime credential owner.

## Reproduction Assessment

Yes, at source level. Current main stores the Claude CLI credential under provider `claude-cli` and `models auth list --provider anthropic` still filters on that raw summarized provider string.

## Solution Assessment

No, the current inspection path is not the best final state. The safer fix is to centralize provider alias/profile resolution for auth inspection rather than special-casing one command or changing the stored profile shape.
