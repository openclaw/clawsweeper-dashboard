## Summary

Keep open: current main still has the Anthropic/Claude CLI auth-list mismatch, and the linked closing PR at https://github.com/openclaw/openclaw/pull/79602 is open rather than merged.

## What This Changes

The issue asks Anthropic-facing auth inspection commands to recognize usable Claude CLI auth profiles after successful `--provider anthropic --method cli` login.

## Best Possible Solution

Land or replace the linked fix so auth inspection resolves Claude CLI profiles through the canonical Anthropic provider while preserving `claude-cli` as the stored runtime credential owner.

## Reproduction Assessment

Yes, at source level. Current main stores Claude CLI credentials with provider `claude-cli`, while `models auth list --provider anthropic` still filters summarized profiles with strict raw provider equality.

## Solution Assessment

Yes, alias-aware auth inspection is the right boundary. The maintainable fix should reuse shared provider/auth alias resolution and include regression coverage for the Anthropic-facing list/status output rather than changing the stored credential shape.
