## Summary

Keep open: TTS provider fallback is documented existing behavior, and current main’s generic synthesis path still catches provider errors and continues to fallback providers, but this report does not include the exact TTS surface or MiniMax quota payload needed to identify the bypass.

## What This Changes

The issue reports that configured TTS fallback does not switch from MiniMax to Microsoft or another provider after MiniMax quota exhaustion.

## Best Possible Solution

Reproduce the failing MiniMax quota path first, then fix the specific provider or caller path that bypasses the existing fallback loop and add focused regression coverage.

## Solution Assessment

Unclear. The existing generic fallback loop is the right maintainable design, so the safer next step is to prove whether MiniMax quota handling or an explicit `disableFallback` caller path is bypassing it before changing code.
