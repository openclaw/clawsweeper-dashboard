## Repair Work Prompt

Fix the MiniMax TTS quota/billing fallback bug from https://github.com/openclaw/openclaw/issues/76904. Current main checks only HTTP status in `extensions/minimax/tts.ts`, then decodes `data.audio`; MiniMax can return HTTP 200 with non-zero `base_resp.status_code`, so the provider resolves instead of throwing and `packages/speech-core/src/tts.ts` never tries configured fallback providers. Add a MiniMax-local envelope guard before reading `data.audio`, throw `MiniMax TTS API error (<code>): <msg>` for non-zero status codes, and add focused regression tests for HTTP 200 envelope errors with empty and non-empty audio. Do not change the shared speech-core fallback loop, add config, or edit CHANGELOG.md.
