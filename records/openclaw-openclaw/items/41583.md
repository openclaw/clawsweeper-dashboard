## Summary

Keep open. Current main still injects inbound channel metadata into both the system prompt and user prompt without a disable/minimize mode, and the requested switch changes sender/channel provenance semantics enough to need maintainer product and security design.

## What This Changes

Add a configuration option to disable or minimize OpenClaw-generated inbound channel metadata for trusted channel or sender contexts.

## Best Possible Solution

Add a documented, default-on inbound metadata mode with narrow trusted-scope controls that applies consistently to system metadata and user-role metadata prefixes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the current behavior: source inspection shows direct external-channel messages flow through buildInboundMetaSystemPrompt and buildInboundUserContextPrefix without a config gate, and tests assert metadata-on behavior.

## Solution Assessment

Is this the best way to solve the issue? No current solution exists in main, and a simple global off switch would be too broad for provenance-sensitive metadata. The safer direction is a scoped, default-on mode with security review and regression coverage.
