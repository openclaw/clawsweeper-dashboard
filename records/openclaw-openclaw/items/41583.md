## Summary

Keep open. Current main still injects inbound channel metadata into both the system prompt and user prompt without a disable/minimize mode, and adding that switch would create new config and provenance policy.

## What This Changes

Request to add a config option that disables or minimizes OpenClaw-injected inbound channel/sender metadata for trusted channel conversations such as Telegram DMs.

## Best Possible Solution

Design a documented, default-on inbound metadata mode with narrowly scoped trusted-channel controls that covers both system metadata and user-role untrusted blocks consistently.

## Reproduction Assessment

Yes for the current behavior, but not as a failing bug: source inspection shows inbound system and user metadata are assembled without a config gate, and tests assert metadata-on behavior for direct external-channel messages.

## Solution Assessment

No current solution exists in main. The requested direction is reasonable, but the maintainable fix should be a scoped default-on mode rather than a broad provenance-disabling toggle.
