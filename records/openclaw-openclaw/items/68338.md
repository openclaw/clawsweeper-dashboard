## Summary

Keep open. Current main still only supports same-line `MEDIA:` payloads, while this PR adds a narrow continuation syntax and focused parser tests; the remaining blockers are maintainer acceptance of the expanded syntax and missing real behavior proof from the external contributor.

## What This Changes

The branch adds immediate-next-line continuation parsing for empty `MEDIA:` directives and parser tests for ordered text/media output preservation.

## Best Possible Solution

If maintainers want this syntax, land a rebased parser-local change that preserves current hardening and includes contributor-supplied real behavior proof; otherwise keep the documented same-line `MEDIA:` form.

## Reproduction Assessment

Yes. Source inspection on current main shows a bare `MEDIA:` line is kept as text because the regex requires a same-line payload and the parser does not inspect the next line.

## Solution Assessment

Yes, with caveats. Limiting continuation parsing to the immediate next line and reusing existing validation is the narrowest implementation, but it still needs maintainer approval for the new syntax and real behavior proof before merge.
