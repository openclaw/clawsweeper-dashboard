## Summary

Keep this PR open. Current main still does not parse a bare `MEDIA:` line as a continuation directive, and the PR is a narrow implementation of that behavior. I did not find a functional or security regression in the diff, but the user-facing fix is missing the required changelog entry before normal maintainer review and merge.

## What This Changes

The PR adds support for treating an empty `MEDIA:` directive as using the immediately following line as the media source, with parser tests for ordered segments and one-line continuation consumption.

## Best Possible Solution

Merge the narrow continuation parser support after adding the required changelog entry and validating it against current main, while keeping #75182 as the separate parser-hardening track unless maintainers choose to combine both safely.

## Solution Assessment
