## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve multiline MEDIA tails while streaming:** `src/media/parse.ts:211-212`
  - body: This adds a directive form that spans the newline after a bare `MEDIA:` line, but `createStreamingDirectiveAccumulator` only buffers when the current last line itself matches `MEDIA:`. If a streamed chunk ends as `MEDIA:\n`, the last line is empty, so the marker can be emitted as text and the following path arrives as ordinary text instead of a media attachment. Please update the streaming tail guard and add a chunk-boundary regression test for `MEDIA:\n<path>`.
  - confidence: 0.86
