## What This Changes

The branch changes the shared MEDIA parser and tests to treat one valid path on the line after an empty `MEDIA:` directive as a media segment while preserving following text.

## Best Possible Solution

Land a rebased, proof-backed parser update only if maintainers accept the multiline legacy directive syntax; otherwise keep the same-line `MEDIA:` and structured media surfaces as the supported paths.

## Reproduction Assessment

Yes, at source level: current main's MEDIA regex requires a same-line payload and `splitMediaFromOutput` keeps an empty `MEDIA:` line as text when no match is found. I did not run tests because this review is read-only.

## Solution Assessment

Unclear until maintainers accept this syntax: the shared parser is the right boundary if multiline legacy directives should work, but documented and preferred surfaces remain same-line `MEDIA:` or structured media fields.
