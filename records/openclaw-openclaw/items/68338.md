## Summary

Keep open. The patch is a focused parser change and the earlier greedy-continuation review feedback was addressed, but it expands the documented MEDIA syntax and still lacks contributor-provided real behavior proof from an OpenClaw run.

## What This Changes

The PR changes the shared MEDIA parser to accept an empty `MEDIA:` directive followed by a media path on the next line, and adds parser tests for ordered segments and one-line continuation consumption.

## Best Possible Solution

If maintainers want this syntax, land the parser-local change with real behavior proof and then update the public MEDIA directive docs; otherwise keep the documented same-line form.

## Reproduction Assessment

Yes. Source inspection on current main shows a bare `MEDIA:` line has no regex payload and is kept as text rather than consuming the next line.

## Solution Assessment

Yes, with maintainer approval. Limiting continuation parsing to the immediate next line while reusing existing media validation is the narrowest implementation, but accepting the new syntax is a product/docs decision.
