## What This Changes

## Best Possible Solution

Land the explicit WAV muxer change with focused regression coverage after the contributor adds real behavior proof and maintainers run the targeted media-understanding validation.

## Reproduction Assessment

Yes at source level, not live in this checkout: current main passes a `.wav.part` fs-safe temp path to ffmpeg for non-WAV `whisper-cli` audio without `-f wav`. The current test already captures the `.part` path shape, and FFmpeg's documented contract explains why format inference is fragile here.

## Solution Assessment

Yes: forcing WAV immediately before the output path is the narrowest maintainable fix because it preserves fs-safe staged writes while satisfying ffmpeg's output-format contract. Reverting the fs-safe wrapper would be broader and would weaken the recent file-safety hardening.
