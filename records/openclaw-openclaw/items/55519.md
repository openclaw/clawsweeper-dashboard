## Summary

Keep open: the bare agentId targeting idea is still useful, but this PR is not mergeable because label misses can retarget to a different visible session and Feishu-specific guidance leaks into global tool metadata. The ACP-only streamTo tolerance is already present on current main.

## What This Changes

The PR adds agentId-based sessions_send targeting, ignores ACP-only streamTo for native sessions_spawn, and adds Feishu-oriented orchestration prompt guidance with tests.

## Best Possible Solution

Land a narrowed change that supports explicit bare-agentId sessions_send only when no label is supplied, keeps label misses fail-closed, and gates any Feishu group orchestration text by the Feishu runtime channel.

## Reproduction Assessment

Yes. The PR diff itself is a high-confidence reproduction: mock sessions.resolve to miss and sessions.list to return a visible agent session, and the PR sends to that fallback target; current-main tests cover the fail-closed label-miss path.

## Solution Assessment

No. The proposed direction is useful, but the implementation should separate explicit agentId targeting from label lookup and keep channel-specific prompt text out of generic metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Fail label misses before agent fallback:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: When label resolution returns no key, this branch falls back to agent lookup and can deliver the message to the newest visible session for that agent. A missing or mistyped label should fail instead; keep agent lookup to calls that omit `label`.
  - confidence: 0.94
- **[P2] Gate Feishu summary text by channel:** `src/agents/system-prompt.ts:253-255`
  - body: This sessions_spawn summary is used for every non-ACP runtime, so Discord, Telegram, and other sessions would see Feishu-specific routing advice. Keep the generic summary channel-neutral and add this wording only when `runtimeChannel === "feishu"`.
  - confidence: 0.9
- **[P2] Remove Feishu guidance from generic spawn metadata:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description applies globally. Move the Feishu group guidance into channel-gated prompt logic instead of generic tool metadata.
  - confidence: 0.89
