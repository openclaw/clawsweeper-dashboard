## Summary

Keep open: the PR still has concrete cross-session routing and prompt-scope regressions, and it lacks real Feishu/session runtime proof. Current main already contains the streamTo tolerance part, but the agentId targeting and Feishu guidance need a narrower revision.

## What This Changes

The PR adds agentId-based sessions_send target resolution, strips streamTo for subagent sessions_spawn, and changes Feishu/subagent prompt guidance with tests.

## Best Possible Solution

Land a narrower revision that keeps label misses fail-closed, supports explicit bare-agentId targeting only with deliberate scope, keeps Feishu guidance channel-gated, and preserves current-main streamTo tolerance with real Feishu proof.

## Reproduction Assessment

Yes, at source level. The PR diff shows the label-miss fallback and ungated Feishu wording, while current main source and tests show label misses fail closed and streamTo tolerance has already landed.

## Solution Assessment

No. Bare agentId targeting may be useful, but this implementation broadens label fallback behavior and generic tool guidance; the safer solution is explicit no-label agentId routing plus channel-gated prompt text.
