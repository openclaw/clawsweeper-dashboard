## Summary

Keep open, but the PR should not merge as-is: it contains a useful Feishu/session direction, yet the current diff can retarget failed label sends to another visible session, leaks Feishu guidance into generic tool surfaces, overlaps streamTo handling already present on main, and lacks real Feishu/session runtime proof.

## What This Changes

This PR adds bare-agentId/fallback targeting for sessions_send, strips streamTo for subagent sessions_spawn, adds Feishu-oriented prompt guidance, and updates related tests.

## Best Possible Solution

Land a narrower revision that keeps label lookups fail-closed, supports explicit bare-agentId targeting only after owner approval, gates Feishu guidance by runtime channel, and preserves the current-main streamTo tolerance.

## Reproduction Assessment

Yes, at source level. The PR diff shows the label-miss fallback and globally emitted Feishu wording, while current main source/tests show missing labels fail closed and streamTo tolerance has already landed separately.

## Solution Assessment

No. The requested outcome may be useful, but this implementation broadens targeting semantics and generic guidance; the safer path is an explicit no-label agentId route plus channel-gated Feishu prompt text.
