## Summary

Keep this PR open for contributor revision. The intended Feishu/session orchestration work is plausible, but the current branch still has cross-session routing regressions, globally scoped Feishu prompt text, and no real after-fix behavior proof.

## What This Changes

The branch adds agentId-based `sessions_send` resolution, changes ACP-only `streamTo` handling for subagent spawns, and adds Feishu-oriented subagent orchestration prompt and test updates.

## Best Possible Solution

Narrow the branch to explicit bare-agentId targeting, keep label misses fail-closed, retain current streamTo tolerance, and place Feishu guidance only in channel-gated prompt context.

## Reproduction Assessment

Yes, at source level. The PR diff shows the label-miss fallback and global Feishu wording, while current-main source and tests show label misses fail closed and non-ACP prompt summaries stay channel-neutral.

## Solution Assessment

No. Bare agentId targeting may be useful, but this implementation changes label-targeting safety semantics and surfaces Feishu guidance outside Feishu contexts.

  - body: When label resolution fails, this branch can fall back to the newest visible session for an agent id derived from `agentId` or the label text.   - confidence: 0.83

  - body: This `sessions_spawn` summary is emitted for every non-ACP runtime channel, so Discord, Telegram, and other sessions would see Feishu-specific advice. Keep the generic summary channel-neutral and put this wording behind the Feishu runtime-channel check.
  - confidence: 0.91

  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description applies globally. Move Feishu group guidance into channel-gated prompt logic instead of generic tool metadata.
