## Summary

Keep open: the PR contains useful Feishu/session orchestration work, but the current patch introduces cross-session routing regressions, leaks Feishu-specific guidance into generic tool metadata, and lacks required real behavior proof. Current main already covers the ACP-only `streamTo` tolerance part, so the remaining value needs a narrower revision.

## What This Changes

The branch changes `sessions_send` targeting, `sessions_spawn` ACP-only parameter handling, and agent system-prompt/test guidance for Feishu group orchestration.

## Best Possible Solution

Land a narrower change that keeps label targeting exact, scopes agentId lookup to explicit agentId-only calls, gates Feishu guidance by runtime channel, and includes redacted Feishu/session runtime proof.

## Reproduction Assessment

Yes, at source level: the PR diff shows the unsafe label-miss fallback and global Feishu wording, while current main and tests show label misses fail closed and prompt summaries stay channel-neutral. I did not run tests because this is a read-only review.

## Solution Assessment

No. Bare agentId targeting may be useful, but the implementation should preserve fail-closed label semantics and the main-session A2A guard, then keep Feishu advice behind runtime-channel-specific prompt logic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require exact label targeting before agent fallback:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: When `sessions.resolve` cannot find the requested `label`, this branch falls back to the newest visible session for an agent id derived from `agentId` or the label text. A typoed or missing label can deliver message content to a different conversation, so keep agent lookup to calls that omit `label` or require an explicit opt-in.
  - confidence: 0.92
- **[P2] Preserve main-session A2A label sends:** `src/agents/tools/sessions-send-tool.ts:124-140`
  - body: The refactor runs the A2A denial whenever `requestedAgentId` differs, even when `requesterAgentId` is null. Root/main sessions without an agent-id component used to bypass this label+agentId gate; with A2A disabled they now become forbidden.
  - confidence: 0.83
- **[P2] Gate Feishu-specific summary text by channel:** `src/agents/system-prompt.ts:253-255`
  - body: This `sessions_spawn` summary is emitted for every non-ACP runtime channel, so Discord, Telegram, and other sessions would see Feishu-specific advice. Keep the generic summary channel-neutral and put this wording behind the Feishu runtime-channel check.
  - confidence: 0.91
- **[P2] Remove Feishu text from generic spawn metadata:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description applies globally. Move Feishu group guidance into channel-gated prompt logic instead of generic tool metadata.
  - confidence: 0.9
