## Summary

Keep this PR open, but it is not merge-ready: current main already covers the streamTo tolerance, while the remaining agentId and Feishu prompt changes introduce cross-session routing and channel-scoping problems.

## What This Changes

The PR adds bare agentId targeting for sessions_send, changes subagent handling of ACP-only streamTo, and adds Feishu group orchestration guidance to agent prompts and tool descriptions.

## Best Possible Solution

Keep the useful bare-agentId and Feishu-group goals, but redesign them so explicit agentId-only sends are fail-closed, Feishu guidance is channel-gated, and user-facing behavior gets a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main fails closed on missing labels, while the PR diff adds a label-miss fallback to newest visible agent session; I did not run tests because this review is read-only.

## Solution Assessment

No. The current PR is not the best solution because it mixes a useful explicit agentId feature with implicit label fallback and globally emitted Feishu guidance; the safer route is explicit-only targeting plus channel-gated prompt text.
