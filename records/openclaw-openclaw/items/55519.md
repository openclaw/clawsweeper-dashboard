## Summary

Keep open for maintainer handling: the PR has a plausible Feishu/session orchestration goal, but the current patch introduces unsafe label fallback retargeting, broad Feishu wording, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds sessions_send agentId-to-latest-visible-session routing, makes sessions_spawn ignore ACP-only streamTo for subagent runtime, and adds Feishu-oriented orchestration prompt/tool guidance plus tests.

## Best Possible Solution

Land a narrower revision that preserves fail-closed label targeting, limits agentId routing to an explicit accepted contract, keeps Feishu guidance channel-local, and includes redacted real Feishu/session proof.

## Reproduction Assessment

Yes, at source level. The PR head shows the label-miss fallback, A2A guard change, and ungated Feishu wording; current main shows label misses fail closed and streamTo tolerance already present.

## Solution Assessment

No. The safer path is explicit agentId routing without label-fallback retargeting, Feishu wording only in channel-gated prompt context, and real Feishu/session runtime proof before merge.
