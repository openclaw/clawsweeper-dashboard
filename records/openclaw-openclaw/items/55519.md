## Summary

This PR should stay open because it contains useful Feishu/session orchestration ideas, but the current branch still introduces a cross-session misrouting risk and leaks Feishu-specific guidance into generic tool surfaces; one streamTo-related part is already covered on current main.

## What This Changes

The branch adds agentId-based sessions_send targeting, changes ACP-only streamTo handling for subagent spawns, and adds Feishu-specific orchestration prompt guidance plus tests.

## Best Possible Solution

Narrow the branch to explicit bare-agentId targeting, keep label misses fail-closed, preserve current-main streamTo tolerance, and gate Feishu guidance to Feishu-specific prompt context.

## Reproduction Assessment

Yes, at source level. The PR diff shows the label-miss fallback and global Feishu text, while current-main source and tests show missing labels currently fail closed; I did not run tests because this review is read-only.

## Solution Assessment

No. Bare agentId targeting may be useful, but the current implementation should avoid changing label-targeting safety semantics and should keep Feishu guidance channel-gated.
