## Summary

Keep open: current main already covers the ACP-only `streamTo` tolerance, but the PR's remaining `agentId` and Feishu prompt changes introduce cross-session routing and channel-scoping regressions that should block merge.

## What This Changes

The branch adds `sessions_send` targeting by most-recent visible `agentId`, changes native `sessions_spawn` handling of ACP-only `streamTo`, and adds Feishu-oriented orchestration prompt/test coverage.

## Best Possible Solution

Land a narrowed change that supports explicit bare-`agentId` sends only when no label is supplied, preserves label-miss failures, gates Feishu guidance by runtime channel, and keeps the already-main native `streamTo` tolerance.

## Reproduction Assessment

Yes. A focused reproduction can mock `sessions.resolve` returning a label miss and `sessions.list` returning a visible agent session, then show the PR dispatching to that fallback session while current-main tests assert the label miss fails closed.

## Solution Assessment

No. The intended bare-`agentId` targeting and Feishu guidance are useful, but this diff is not the narrowest maintainable fix because it mixes them with unsafe label fallback, an A2A semantic regression, and ungated Feishu text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require explicit agent targeting after label misses:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: When label resolution fails, this fallback can use `agentId` or a valid-agent-looking label to pick the newest visible session. A missing or mistyped label should fail instead of delivering the message to a different conversation; keep agent lookup to calls that omit `label`.
  - confidence: 0.92
- **[P2] Preserve main-session A2A label sends:** `src/agents/tools/sessions-send-tool.ts:124-140`
  - body: The refactor runs the A2A denial whenever `requestedAgentId` differs, even when `requesterAgentId` is null. Root/main sessions without an agent-id component used to bypass this label+agentId gate; with A2A disabled they now become forbidden.
  - confidence: 0.83
- **[P2] Gate Feishu-specific summary text by channel:** `src/agents/system-prompt.ts:253-255`
  - body: This `sessions_spawn` summary is emitted for every non-ACP runtime channel, so Discord, Telegram, and other sessions would see Feishu-specific advice. Keep the generic summary channel-neutral and put this wording behind the Feishu runtime-channel check.
  - confidence: 0.91
- **[P2] Remove Feishu text from generic spawn metadata:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description applies globally. Move Feishu group guidance into channel-gated prompt logic instead of generic tool metadata.
  - confidence: 0.9
