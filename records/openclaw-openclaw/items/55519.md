## Summary

Keep open. Current main already has the native `streamTo` tolerance, but it does not implement the PR's bare `agentId` send target or Feishu group orchestration guidance, and the proposed branch is not merge-ready because it introduces unsafe label-fallback targeting and channel-specific prompt text in generic surfaces.

## What This Changes

The PR adds `sessions_send` targeting by most-recent visible `agentId` session, changes native `sessions_spawn` handling of ACP-only `streamTo`, and adds Feishu group orchestration prompt guidance with tests.

## Best Possible Solution

Revise the branch so `sessions_send` supports explicit bare `agentId` targeting with existing visibility and A2A checks, preserves failed-label lookup semantics, keeps generic tool metadata channel-neutral, and adds Feishu guidance only in Feishu-gated prompt logic while retaining the already-main native `streamTo` tolerance.

## Reproduction Assessment

Yes. A focused reproduction can mock `sessions.resolve` returning a label miss and `sessions.list` returning an agent session, then verify the PR sends to that fallback session instead of failing; current-main tests already cover the safer label-miss error and native `streamTo` ignore behavior.

## Solution Assessment

No. The PR addresses real gaps, but the label fallback and generic Feishu text are not the narrowest maintainable solution; explicit bare-`agentId` resolution plus channel-gated Feishu prompt guidance is safer.
