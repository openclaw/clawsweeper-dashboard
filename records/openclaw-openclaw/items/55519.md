## Summary

Keep open: the branch still contains unique `sessions_send` agentId targeting and Feishu prompt work, but the current diff has blocking cross-session routing and prompt-scope regressions plus missing real behavior proof. Current `main` already covers the `streamTo` tolerance portion, so this PR needs revision rather than cleanup closure.

## What This Changes

The branch adds agentId-based `sessions_send` targeting, ignores ACP-only `streamTo` for subagent spawns, and adds Feishu group orchestration prompt guidance with tests.

## Best Possible Solution

Keep current-main `streamTo` tolerance, add bare-agentId routing only for explicit no-label calls, keep label misses fail-closed, and put Feishu guidance only in channel-gated prompt context with real runtime proof.

## Reproduction Assessment

Yes, at source level. The PR diff shows the label-miss fallback and global Feishu wording, while current-main source and tests show label misses fail closed and non-ACP prompt summaries stay channel-neutral.

## Solution Assessment

No. Bare agentId targeting may be useful, but this implementation changes label-targeting safety semantics and leaks Feishu-specific guidance into generic runtime metadata; a narrower explicit-agentId path is safer.
