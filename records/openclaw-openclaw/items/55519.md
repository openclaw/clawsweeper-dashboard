## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require exact label targeting before fallback:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: When `sessions.resolve` misses a requested label, this branch falls back to the newest visible session for the requested agentId or a label that parses as an agent id. A typoed or missing label can deliver content to another conversation, so label targeting should fail closed unless the caller used a bare agentId path.
  - confidence: 0.92
- **[P2] Preserve main-session A2A label sends:** `src/agents/tools/sessions-send-tool.ts:124-133`
  - body: This refactor runs A2A denial whenever `requestedAgentId` differs from `requesterAgentId`, including null requester ids from main sessions. That can make label+agentId sends from main/orchestrator sessions fail when A2A is disabled, whereas current main only applies this policy when both agent ids are present.
  - confidence: 0.86
- **[P2] Gate Feishu summary by runtime channel:** `src/agents/system-prompt.ts:253-255`
  - body: This `sessions_spawn` summary is emitted for every non-ACP runtime channel, so Discord, Telegram, and other non-Feishu sessions would see Feishu-specific advice. Keep the summary generic or condition it on the Feishu runtime channel.
  - confidence: 0.91
- **[P2] Keep Feishu guidance out of generic spawn metadata:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description applies globally. Move Feishu group guidance into channel-gated prompt logic instead of generic tool metadata.
  - confidence: 0.89
