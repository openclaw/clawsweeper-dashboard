## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require exact label targeting before fallback:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: When `sessions.resolve` misses a requested label, this falls back to the newest visible session for the requested `agentId` or a label that parses as an agent id. A typoed or missing label can deliver content to another conversation, so label targeting should fail closed unless the caller used a bare `agentId` path.
  - confidence: 0.94
- **[P1] Preserve main-session A2A behavior:** `src/agents/tools/sessions-send-tool.ts:124-133`
  - body: This drops the requesterAgentId guard that let main/orchestrator sessions use label+agentId without requiring `tools.agentToAgent.enabled`. With a main requester, `null !== worker` enters the A2A block and can forbid sends that used to work.
  - confidence: 0.9
- **[P2] Gate Feishu summary by runtime channel:** `src/agents/system-prompt.ts:253-255`
  - body: This non-ACP `sessions_spawn` summary is emitted for every runtime channel, so Discord, Telegram, Slack, and other non-Feishu sessions would see Feishu-specific advice. Keep the summary generic or condition it on the Feishu runtime channel.
  - confidence: 0.92
- **[P2] Keep Feishu guidance out of generic spawn metadata:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description applies globally. Move Feishu group guidance into channel-gated prompt logic instead of generic tool metadata.
  - confidence: 0.9
