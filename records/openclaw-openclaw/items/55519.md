Full review comments:

- **[P1] Keep label lookups fail-closed:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: When `sessions.resolve` cannot find the requested `label`, this fallback tries `agentId` or even a label-shaped agent id and sends to the first listed session. A typoed or stale label should remain a lookup error; otherwise message content can be delivered to an unrelated latest-visible conversation.
  - confidence: 0.94
- **[P1] Avoid implicit latest-session agent targeting:** `src/agents/tools/sessions-send-tool.ts:273-281`
  - body: Current main treats bare `agentId` as the configured agent's main session, but this branch lists sessions and sends to the first returned row. Agents can have multiple visible group, DM, ACP, or subagent sessions, so this needs explicit maintainer-approved semantics, docs, and multi-session coverage before it is a safe routing API.
  - confidence: 0.9
- **[P2] Scope Feishu spawn guidance to Feishu:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: This tool description is registered for every channel, so Discord, Telegram, Slack, and other sessions would see Feishu-specific orchestration advice. Keep the generic `sessions_spawn` description channel-neutral and put Feishu guidance behind a Feishu runtime-channel prompt surface.
  - confidence: 0.88
