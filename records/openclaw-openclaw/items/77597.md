## Summary

Keep open: current PR head still has an interactive skip-selection ambiguity and the available real proof covers only non-interactive `--skill` apply, while the maintainer label requires explicit maintainer handling.

## What This Changes

The PR adds interactive Codex skill migration bulk on/off and skip controls, changes default skill selection for conflicts, and updates tests, docs, and changelog.

## Best Possible Solution

Keep the new Codex selector direction, but make the visible checkbox state and resolved migration action impossible to disagree before landing.

## Reproduction Assessment

Yes for the review finding by source inspection: select `Skip for now`, move to a skill, press Space, then Enter. The prompt keeps skip in `selectedValues` while the resolver applies the selected skill; I did not run the TTY interaction in this read-only review.

## Solution Assessment

No, not as currently proposed. A safer fix is to clear the skip sentinel whenever any skill or bulk control is selected, or otherwise make skip consistently honored so mixed UI states cannot proceed as normal migration.
