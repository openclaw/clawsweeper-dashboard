## Summary

Keep open: this PR is useful feature work, but the latest head still has an interactive skip-selection ambiguity and the PR body only proves the non-interactive `--skill` path, not the new full-screen controls. The protected `maintainer` label also means this needs explicit maintainer handling rather than cleanup closure.

## What This Changes

The PR replaces the Codex migration skill multiselect with a custom interactive prompt that adds bulk on/off and skip controls, defaults conflict skills unchecked, and updates migrate docs, tests, and changelog.

## Best Possible Solution

Make `Skip for now` mutually exclusive or consistently honored, then provide real terminal proof for the interactive bulk/skip path before maintainer handling.

## Reproduction Assessment

Yes for the PR finding from source inspection: select `Skip for now`, toggle a skill, then press Enter. The prompt leaves skip in `selectedValues` while the resolver proceeds with the selected skill; I did not run the interactive UI in this read-only review.

## Solution Assessment

No, not as currently proposed. The prompt should clear skip when another row is selected, or the resolver should honor skip whenever present, so the visible UI state cannot contradict the applied migration path.
