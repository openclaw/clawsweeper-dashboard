Keep open: the latest PR head fixes the earlier skip ambiguity and includes real prompt proof for skip handling, but source inspection shows the `a` shortcut still cannot clear an all-selected Codex skill list because `@clack/core` mutates the full enabled option list before the PR's reconciliation runs.

## Best Possible Solution

Keep the new selector direction, but handle shortcut state before or around `@clack/core`'s built-in mutation so `a` and `i` operate only on real skill rows, then add a targeted regression test and a short live prompt transcript.

## Reproduction Assessment

Yes for the review finding by source inspection: with a no-conflict Codex plan, all real skills start selected, and pressing `a` is handled by `@clack/core` before this PR's listener. I did not run the TTY path locally in this read-only review.

## Solution Assessment

No, not yet. The skip handling is improved, but the shortcut fix needs to preserve the previous select-all toggle semantics for real skill rows rather than reconciling only after sentinel rows have already changed the core prompt value.
