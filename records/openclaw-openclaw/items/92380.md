## Summary

This PR should remain open because flat staging addresses the reported tar conflict, but the changed CLI argument uploads the named random temporary directory, which upstream OpenShell preserves at the destination and therefore can place the file at the wrong remote path. The linked bug remains valid, and the branch also needs after-fix proof from a real OpenShell sandbox.

## What This Changes

The PR copies each mirrored file into a flat temporary directory and passes that directory to `openshell sandbox upload` targeting the file’s remote parent directory.

## Best Possible Solution

Keep basename-only staging, but upload the staging directory’s contents using the existing full-workspace source convention; add focused assertions for CLI arguments and cleanup, then prove the exact remote file path in a live OpenShell sandbox.

## Reproduction Assessment

Yes, at source level with high confidence: the linked report provides exact steps and logs, while current main directly exercises the nested upload shape implicated by the tar error. This review did not run a live OpenShell sandbox.

## Solution Assessment

No, not as currently implemented. Flat staging is the right boundary, but the named staging directory must not become part of the remote path; the existing directory-contents upload convention is the narrower correct fix.
