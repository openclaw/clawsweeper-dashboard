## Summary

Keep open: the PR targets a real `chat.history` hygiene problem, but the current patch would remove intentional compaction checkpoint dividers and filters by raw text prefixes without provenance; it also still lacks after-fix real behavior proof.

## What This Changes

The PR adds Gateway `chat.history` filtering that drops compaction markers and pre/post-compaction maintenance prompt text, plus sanitizer and gateway history tests.

## Best Possible Solution

Rework the PR so `chat.history` hides only trusted internal maintenance rows through provenance or structural metadata in the shared display projection, while preserving compaction checkpoint dividers.

## Reproduction Assessment

Yes, source-level. The PR predicate visibly drops every compaction marker and every message starting with the maintenance prefixes, while current main intentionally renders compaction markers as checkpoint dividers.

## Solution Assessment

No. The narrow maintainable fix is provenance-aware filtering in the shared display projection, not dropping all compaction markers or matching user-visible text prefixes globally.
