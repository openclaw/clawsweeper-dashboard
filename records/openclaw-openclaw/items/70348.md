## Summary

This PR should stay open because the branch still has blocking correctness issues: it removes compaction checkpoint markers that current main intentionally exposes as WebChat dividers, and it filters maintenance-looking user text by raw prefix without trusted internal provenance.

## What This Changes

The PR adds Gateway `chat.history` filtering for compaction markers and pre/post-compaction maintenance prompt text, plus sanitizer and gateway history tests.

## Best Possible Solution

Preserve compaction checkpoint dividers, and hide only trusted internal maintenance entries through shared chat display projection metadata or provenance rather than raw text prefixes.

## Reproduction Assessment

Yes, source-level. The PR predicate visibly drops every `__openclaw.kind === "compaction"` marker and any message whose text starts with the maintenance prefixes, while current main treats compaction markers as WebChat checkpoint dividers.

## Solution Assessment

No. The proposed raw marker and prefix filtering is not the narrowest maintainable fix; the safer solution is provenance-aware filtering in the shared display projection while preserving compaction boundary records.
