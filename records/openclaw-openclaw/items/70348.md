## Summary

Keep open. The PR targets a real chat.history artifact leak, but the current diff conflicts with current-main compaction-divider behavior and can hide legitimate user-authored prefix matches; it also lacks required after-fix real behavior proof.

## What This Changes

Adds Gateway chat.history filtering for synthetic compaction markers, pre-compaction memory-flush prompts, and post-compaction context-refresh messages, plus sanitizer and gateway history tests.

## Best Possible Solution

Rework the fix into the shared Gateway display projection, preserve compaction dividers, and drop only trusted internal maintenance entries using provenance or structural metadata with negative tests for user-authored prefix matches.

## Reproduction Assessment

Yes, source-level. A seeded transcript with a compaction marker shows the PR would remove a documented divider, and a normal user message beginning with the maintenance prefix would be dropped by the new predicate.

## Solution Assessment

No. Dropping all compaction markers and filtering by raw text prefix is not the narrowest maintainable fix; the repair should preserve current compaction UI affordances and require trusted internal provenance before hiding entries.
