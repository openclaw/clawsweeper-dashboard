## Summary

Keep open. The PR targets a real chat.history leak class, but the current diff is not mergeable because it drops current-main compaction dividers, uses raw text-prefix filtering that can hide legitimate user content, and lacks contributor real-behavior proof.

## What This Changes

Adds Gateway chat.history filtering for synthetic compaction markers, pre-compaction memory-flush prompts, and post-compaction context-refresh messages, plus sanitizer and gateway history tests.

## Best Possible Solution

Rebase onto the shared chat-display projection, preserve current compaction dividers, and drop only trusted internal maintenance entries using provenance or equivalent structural metadata with negative regressions for user-authored prefix matches.

## Reproduction Assessment

Yes, source-level. A seeded transcript can show the intended leak path, and the PR-introduced false positive is a normal user message beginning with the filtered maintenance prefix.

## Solution Assessment

No. Raw prefix filtering and dropping all compaction markers is not the narrowest maintainable fix; the fix should be provenance-aware and live in the current shared projection path.
