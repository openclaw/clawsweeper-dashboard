## Summary

Keep this PR open, but it should not merge as-is: it removes compaction checkpoint dividers that current main documents and renders, and it filters by raw text prefixes without trusted internal provenance.

## What This Changes

The PR adds Gateway `chat.history` filtering for compaction markers and maintenance prompt prefixes, with unit and gateway history tests.

## Best Possible Solution

Move the fix into the shared chat display projection, preserve compaction divider markers, and hide only trusted internal maintenance entries using provenance or structural metadata with negative tests for user-authored prefix matches.

## Reproduction Assessment

Yes, source-level. A seeded compaction transcript record is converted on current main into a `__openclaw.kind === "compaction"` marker that the UI renders as a divider, while the PR would drop it; the raw-prefix predicate also clearly drops normal user text with those prefixes.

## Solution Assessment

No. Filtering all compaction markers and matching maintenance prefixes by text alone is not the narrowest maintainable fix; the safer path is provenance-aware filtering in the shared display projection while preserving documented checkpoint dividers.
