## Summary

Keep open. Current main does not implement the PR's intended chat.history artifact filtering, but the proposed patch should not merge as-is because it filters by raw text prefixes and can hide legitimate user-authored messages; it also needs rebasing onto the current shared projection layout.

## What This Changes

The PR adds chat.history filtering for compaction markers, pre-compaction memory-flush prompts, and post-compaction context-refresh messages, plus focused Gateway sanitizer and history tests.

## Best Possible Solution

Move the fix to the current shared chat-display projection, filter synthetic/internal artifacts using structural metadata or explicit provenance, and add regressions that hide generated compaction artifacts while preserving ordinary visible messages that happen to start with the same text.

## Reproduction Assessment

Yes. A source-level reproduction exists by seeding a transcript with a type: "compaction" row and calling chat.history; current main projects the synthetic marker instead of dropping it. The PR-introduced false positive is reproducible from the patch with any user message starting with one of the filtered marker strings.

## Solution Assessment

No. The PR targets a real visibility problem, but raw prefix filtering is not the best fix because it lacks provenance and can suppress legitimate chat content. The safer solution is metadata/provenance-based filtering in the current shared projection layer, with #43567 guiding which internal messages should be hidden versus displayed as system/runtime UI.
