## Summary

Keep open. The PR targets a real history-visibility problem, but the current diff would remove documented compaction checkpoint dividers and can hide ordinary user-authored messages that happen to start with the maintenance prefixes.

## What This Changes

The PR adds Gateway `chat.history` filtering for compaction markers and pre/post-compaction maintenance prompt text, plus sanitizer and gateway history tests.

## Best Possible Solution

Rework this as a shared display-projection/provenance filter that preserves compaction checkpoint dividers and hides only trusted internal maintenance entries.

## Reproduction Assessment

Yes, source-level. The PR predicate visibly drops every compaction marker and any message whose text starts with the maintenance prefixes, while current main intentionally renders compaction markers as checkpoint dividers.

## Solution Assessment

No. The proposed raw marker and prefix filtering is not the narrowest maintainable fix; the safer path is provenance-aware filtering in the shared display projection while preserving compaction boundary records.
