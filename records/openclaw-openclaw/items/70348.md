## Summary

Keep open: the PR targets a real history hygiene problem, but the proposed filter now conflicts with current main's documented compaction checkpoint divider behavior and over-filters raw text without trusted internal provenance; it also still lacks required real behavior proof.

## What This Changes

The PR adds Gateway `chat.history` filtering and tests that drop compaction markers plus pre/post-compaction maintenance prompt text from history responses.

## Best Possible Solution

Rework the fix in the shared `chat-display-projection` boundary to hide trusted internal maintenance entries while preserving compaction checkpoint dividers and normal user-authored text.

## Reproduction Assessment

Yes, source-level. The PR predicate visibly drops every `__openclaw.kind === "compaction"` marker and any message whose text starts with the maintenance prefixes, while current main emits those markers and renders them as documented checkpoint dividers.

## Solution Assessment

No. The patch should preserve compaction dividers and filter trusted internal provenance or structural markers in the shared projection instead of adding raw text-prefix filtering in the older sanitizer path.
