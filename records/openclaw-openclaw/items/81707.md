Keep open: current main and the latest release do not implement configurable Control UI navigation links, and the existing plugin Control UI descriptor API does not cover persistent nav/header links. The request is concrete and aligned with frontend ergonomics, but adding a new `gateway.controlUi` config surface needs maintainer product/API direction before implementation.

## What This Changes

The issue requests a new `gateway.controlUi.navLinks` list that renders external self-hosted service shortcuts in the Control UI header or navigation.

## Best Possible Solution

Decide the product boundary first, then implement either a narrow core Control UI link surface or a plugin/descriptor extension that supports persistent navigation without ad hoc config growth.
