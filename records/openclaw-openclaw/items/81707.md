## Summary

Keep open. Current main does not provide configurable Control UI navigation shortcuts, and the request would add a new config and UI policy surface rather than fix broken existing behavior.

## What This Changes

The issue requests a `gateway.controlUi.navLinks` config array that renders persistent external shortcut links in the Control UI navigation/header.

## Best Possible Solution

Have maintainers choose whether this should be a small sanitized `gateway.controlUi.navLinks` contract or fold into the broader Control UI contribution/navigation design, then update config types/schema/docs, bootstrap data, sidebar rendering, and focused UI tests accordingly.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence the requested feature is absent because navigation, config, bootstrap, and docs have no custom link contract.

## Solution Assessment

Unclear. The proposed `gateway.controlUi.navLinks` shape is narrow and plausible, but maintainers should first decide whether arbitrary user-config links belong in core config or in a broader Control UI contribution system.
