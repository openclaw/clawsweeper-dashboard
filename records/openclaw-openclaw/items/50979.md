## Summary

Keep open. Current main still does not add outbound Slack message-tool formatting guidance, but the PR head's Markdown-only hint is overbroad because only plain message text is converted before Slack delivery; presentation and Block Kit paths are sent as Slack mrkdwn. The PR also lacks the required changelog entry for a user-facing Slack prompt fix.

## What This Changes

The PR appends Slack formatting guidance to `agentPrompt.messageToolHints` and updates Slack channel prompt tests to expect the added Markdown formatting hints.

## Best Possible Solution

Revise the Slack message-tool prompt so it distinguishes plain `message` text, which should use standard Markdown because OpenClaw converts it, from `presentation` text and native Slack block payloads, which must be authored as Slack mrkdwn because they are rendered or forwarded directly.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: plain Slack sends call `markdownToSlackMrkdwnChunks`, while block and presentation sends bypass that conversion and pass `mrkdwn` Block Kit fields directly.

## Solution Assessment

No. The PR targets the right prompt surface, but the current wording is not the narrowest maintainable fix because it applies Markdown guidance to Slack paths that expect native mrkdwn.
