## Summary

Keep this PR open, but it still needs changes before merge. The patch targets a real Slack outbound prompt gap, yet the current wording overstates Markdown conversion for Slack paths that forward Block Kit/presentation mrkdwn directly, and the user-facing prompt change still needs a changelog entry.

## What This Changes

The PR appends standard Markdown guidance to Slack `messageToolHints` and adjusts Slack prompt tests to expect the added hints.

## Best Possible Solution

Revise the Slack message-tool prompt so plain `message` text is described as standard Markdown that OpenClaw converts, while `presentation` text and native Slack block payloads are described as direct Slack mrkdwn, then update focused prompt tests and the active changelog without changing send/block rendering behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: plain Slack sends call `markdownToSlackMrkdwnChunks`, while block and presentation sends bypass that conversion and pass Slack `mrkdwn` fields directly.

## Solution Assessment

No. The PR uses the right prompt surface, but the current wording is not the best fix because it gives one Markdown-only rule for both converted plain text and direct mrkdwn Block Kit paths.
