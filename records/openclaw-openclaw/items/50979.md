## Summary

Keep this PR open, but do not merge it as-is. The remaining outbound Slack prompt gap is real, but the PR’s final wording is too broad: plain Slack message text is Markdown-converted, while Slack presentation/native block text is sent as Block Kit `mrkdwn`.

## What This Changes

This PR appends Slack formatting guidance to `messageToolHints` and updates Slack channel prompt tests to expect standard Markdown hints.

## Best Possible Solution

Add scoped Slack outbound guidance: plain `message` text should use standard Markdown because OpenClaw converts it before Slack delivery, while `presentation` text and legacy/native Slack block payloads are rendered as Slack `mrkdwn`. Cover both enabled and disabled interactive-reply prompt cases in tests.

## Reproduction Assessment

Yes. The source-level reproduction is clear: plain Slack sends call `markdownToSlackMrkdwnChunks`, while block and presentation sends forward `mrkdwn` blocks directly, so the PR’s unconditional “do not use mrkdwn” hint misguides block-based messages.

## Solution Assessment

No. The PR is pointed at the right prompt surface, but the safest solution is a narrower hint that distinguishes plain text from presentation/native block text.
