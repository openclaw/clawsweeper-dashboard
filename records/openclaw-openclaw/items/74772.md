The PR adds QQ Bot `clientSecret` env SecretRef resolution, secret-contract sidecars, QQ Bot SecretRef docs, and config tests.

## Best Possible Solution

Yes. Current main has a focused unit expectation that env SecretRefs throw, and the docs-matrix failure follows directly from the generated-matrix byte comparison; the predicate bug is reproducible with a QQ Bot account that has both `clientSecret` and `clientSecretFile` configured.
