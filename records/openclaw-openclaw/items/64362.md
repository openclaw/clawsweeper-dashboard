## Summary

Keep open. Current main has fixed adjacent long-held `sessions.json` store-lock contention via https://github.com/openclaw/openclaw/pull/68554 and added later delivery/transcript mitigations, but the transcript write-lock path still preserves the untracked current-PID/no-starttime reclaim behavior that matches the lock-cleanup theory; the targeted fix PR for that behavior closed unmerged.

## What This Changes

The report asks OpenClaw to stop losing or delaying Telegram main-agent replies after tool-running turns when session lock cleanup removes a dead-pid lock during active work.

## Best Possible Solution

Land an owner-reviewed session transcript write-lock fix that protects active no-starttime same-PID owners while preserving dead-PID, non-OpenClaw-owner, and verified recycled-PID recovery, with focused regression coverage.

## Reproduction Assessment
