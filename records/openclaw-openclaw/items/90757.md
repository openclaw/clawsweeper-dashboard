## Summary

Keep open: current main and v2026.6.8 have improved sidebar display fallbacks, CSS ellipsis, the `sessions.patch` label contract, and bulk delete plumbing, but the requested sidebar/picker rename affordance and per-row Sessions-table delete action are still missing and overlap active maintainer product/UI decisions.

## What This Changes

The issue requests more usable Control UI session management: better sidebar titles, inline/sidebar rename, and individual delete controls for session rows.

## Best Possible Solution

Keep the human-readable title fallbacks, choose one canonical picker/sidebar rename implementation that reuses `sessions.patch`, and add a narrowly-scoped per-row Sessions-table delete action backed by the existing `sessions.delete` helper if maintainers accept that UX.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.8 show switch-only sidebar/picker controls and bulk-only Sessions-table delete. This is not a failing-runtime bug repro; it is a feature/UX gap confirmed by source inspection.

## Solution Assessment

Unclear as one combined patch: the title fallback portion is already mostly solved, while rename and individual delete should be staged through focused Control UI changes. The rename path should reuse `sessions.patch`, and any delete button should reuse the existing confirmation and `sessions.delete` helper instead of adding a new backend path.
