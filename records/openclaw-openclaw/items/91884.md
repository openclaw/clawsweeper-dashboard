## Summary

The final branch correctly removes the earlier startup-lifecycle expansion, but its replacement ignore callback still fails for a common overlapping-root configuration: a broader collection is checked before a nested explicit `build`/`dist`/`vendor`/`.cache` root, so that root remains ignored. The PR also lacks after-fix real watcher evidence.

## What This Changes

The PR makes QMD watcher ignore checks relative to configured collection roots, adds regression tests for ignored-name roots, and isolates a deadcode pnpm fallback test from ambient PATH state.

## Best Possible Solution

Select the exact or longest containing configured root before evaluating ignored descendant segments, add a mixed broader-plus-nested-root regression test, and attach redacted runtime output showing an edit triggers QMD refresh and clean shutdown.

## Reproduction Assessment

Yes. Current main rejects an explicitly configured root when its absolute path contains an ignored directory name, and the proposed head's remaining overlapping-root failure follows directly from the callback's first-containing-root return.

## Solution Assessment

No. Root-relative filtering is the right ownership boundary, but the implementation must use the exact or most-specific matching configured root rather than whichever containing root appears first.
