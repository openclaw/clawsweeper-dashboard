## Summary

Keep open: current main and the latest release still have managed Chrome launch/stop and live cookie mutation paths, but no managed-profile cookie/session persistence across browser or gateway restarts; the earlier fix PR was closed because its default CDP cookie sidecar was the wrong security/product shape.

## What This Changes

The issue asks OpenClaw to persist managed-browser cookies and login sessions across browser and gateway restarts.

## Best Possible Solution

Keep this open for a current-path browser plugin fix that first improves graceful shutdown/profile recovery, and only adds explicit sensitive cookie persistence if maintainers accept the security model and storage owner.

## Reproduction Assessment

Yes at source level, but not from a live run in this review. Current main launches a dedicated managed Chrome profile and terminates it without any cookie save/restore path, and the linked CDP behavior explains why disk cookies can be missing after restart.

## Solution Assessment

No for the proposed default sidecar shape. The maintainable path is a browser-plugin lifecycle fix with tests and live proof, avoiding plaintext cookie duplication unless maintainers explicitly approve an opt-in sensitive-state design.
