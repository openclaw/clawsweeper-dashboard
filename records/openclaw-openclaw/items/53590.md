## Summary

Keep open. Current main has clearer runtime unknown-model handling and hook-scoped catalog warnings, but it still lacks the requested generic gateway-startup or doctor validation for invalid configured model IDs after updates; the open related PR only covers part of the doctor warning space.

## What This Changes

The issue requests startup warnings, doctor validation, and changelog guidance when configured model IDs become invalid or renamed after an npm update.

## Best Possible Solution

Add a shared configured-model diagnostics layer used by both Gateway startup and openclaw doctor: collect configured model refs from defaults, fallbacks, agents, hooks, channels, tools, and media settings; resolve aliases consistently; warn only when catalog evidence is authoritative; and document or changelog retired shipped model slugs when OpenClaw changes them.

## Reproduction Assessment

Yes. The issue gives a concrete invalid configured primary model, and current source inspection shows startup only prewarms metadata while doctor only validates hooks.gmail.model, so the missing-warning behavior is reproducible from the code path.

## Solution Assessment

Unclear for the full fix. PR #40355 is a useful doctor-only slice, but the best maintainable solution needs a shared diagnostics layer plus maintainer policy on catalog authority before broad startup warnings are added.
