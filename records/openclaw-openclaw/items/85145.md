## Summary

Keep this open: current main still lacks a `models auth` non-interactive API-key paste path, and the open linked fix PR uses closing syntax but has not merged yet.

## What This Changes

The issue requests a non-interactive OpenAI Codex API-key auth path because `paste-token` writes token-shaped credentials that the Codex app-server treats as ChatGPT/OAuth token material.

## Best Possible Solution

Review and land the linked PR if it remains correct; otherwise keep this issue open for a focused `models auth` API-key paste path that writes `type: "api_key"` with `key` and config mode `api_key` without changing token semantics.

## Reproduction Assessment

Yes. Source inspection shows `paste-token` always writes a token profile and mode, while the Codex app-server API-key path requires an `api_key` profile with `key`; no live credential request was needed to prove that shape mismatch.

## Solution Assessment

Yes. A separate API-key paste path is the narrowest maintainable fix because it preserves existing token/OAuth semantics and uses the already-supported API-key profile contract.
