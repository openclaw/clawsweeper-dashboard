Keep open. Current main still has the setup/provider-auth gap, and the linked closing PR is open and conflicting, so this issue should not be closed before a fix lands.

## What This Changes

Prevent Google Gemini auth setup from silently changing an existing default model so heartbeat background runs do not inherit paid Gemini usage.

## Best Possible Solution

Land a narrow setup fix that passes preserveExistingDefaultModel through provider auth when an existing primary is present, with focused setup regression coverage.

## Reproduction Assessment

Yes, by source inspection. Start with an existing agents.defaults.model.primary and heartbeat enabled without heartbeat.model, run setup with Google API-key auth, and current main can promote the Google default that heartbeat then inherits.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing preserveExistingDefaultModel seam in runSetupWizard, matching configure.gateway-auth, rather than special-casing Google or adding a new heartbeat policy.
