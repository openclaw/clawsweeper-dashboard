Fix the setup/provider-auth path for https://github.com/openclaw/openclaw/issues/64129. In `runSetupWizard`, provider auth should preserve an existing `agents.defaults.model` primary when applying a provider auth choice, matching the existing `configure gateway-auth` behavior, while first-run setups with no existing primary and explicit later model-picker selections must still work. Reuse the existing `preserveExistingDefaultModel` seam; do not add Google-specific logic, new config flags, heartbeat model special cases, or changelog edits. Add focused regression coverage in the setup wizard tests proving `applyAuthChoice` receives `setDefaultModel: true` and `preserveExistingDefaultModel: true`, and keep existing explicit default-setting behavior intact.
