## Summary

Keep open. Current main still has the setup/provider-auth gap, and the linked closing PR is open and conflicting, so the issue should stay open until that fix or an equivalent one lands.

## What This Changes

The issue requests that paid provider setup not silently move existing heartbeat/background traffic onto a newly configured Gemini default.

## Best Possible Solution

Land or update https://github.com/openclaw/openclaw/pull/81304, or an equivalent narrow setup fix, so runSetupWizard preserves an existing primary model during provider auth and has focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Start with an existing agents.defaults.model.primary and heartbeat enabled without heartbeat.model, run setup with Google API-key auth, and current main can promote the Google default that heartbeat then inherits.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing preserveExistingDefaultModel seam in runSetupWizard, matching configure.gateway-auth, rather than special-casing Google or adding a new heartbeat policy.
