## Summary

Keep open. Current main still omits `preserveExistingDefaultModel` from setup provider auth, so a provider default can replace an existing primary model and heartbeat can inherit it; the closing fix PR is still open and conflicting.

## What This Changes

The issue asks OpenClaw to stop provider setup from silently switching existing heartbeat traffic onto a newly configured paid Google Gemini default model.

## Best Possible Solution

Keep the issue open until https://github.com/openclaw/openclaw/pull/81304 or an equivalent narrow setup fix lands with regression coverage.

## Reproduction Assessment

Yes, by source inspection. Setup applies provider auth with default-setting enabled but without default preservation, and heartbeat without `heartbeat.model` falls through to the session/default model.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the existing `preserveExistingDefaultModel` seam through setup provider auth rather than adding Google-specific or heartbeat-specific policy.
