## Summary

Keep open: current main and the latest release still have the setup wizard applying provider auth with `setDefaultModel: true` but without the existing default-preservation flag, so paid Google auth can still replace an existing primary that heartbeat later inherits.

## What This Changes

The issue asks OpenClaw to stop paid Google Gemini setup from silently changing an existing default model used by heartbeat background runs.

## Best Possible Solution

Apply the existing `preserveExistingDefaultModel` contract to setup/provider auth for existing primaries, preserve first-run and explicit model-selection behavior, and add focused setup regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: setup applies Google provider auth with default-setting enabled, omits the preservation flag, and heartbeat can inherit the resulting default model when no heartbeat override exists.

## Solution Assessment

Yes. Reusing the existing setup/auth preservation seam is the narrow maintainable fix; broader paid-provider consent and preview-default policy belongs in the related product-policy issue.
