Keep open. Current main still has the source-level setup gap, and the issue is now paired with an open fixing PR that explicitly closes it, so the safe cleanup outcome is to leave the issue open until that PR is reviewed, rebased, and landed or rejected.

## What This Changes

Prevent Google provider setup from silently replacing an existing primary model with a paid Gemini default that heartbeat then inherits for background runs.

## Best Possible Solution

Review, rebase, and land the linked setup fix that forwards preserveExistingDefaultModel through runSetupWizard, while leaving the broader paid-preview migration policy to https://github.com/openclaw/openclaw/issues/74395.

## Reproduction Assessment

Yes, by source inspection. Use an existing agents.defaults.model.primary with heartbeat enabled and no heartbeat.model, run setup with a Google API-key auth choice, and the setup path can still apply the Google default that heartbeat later inherits.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the existing preserveExistingDefaultModel flag through setup/provider auth; changing heartbeat model resolution or adding a new confirmation policy is broader than needed for this specific bug.
