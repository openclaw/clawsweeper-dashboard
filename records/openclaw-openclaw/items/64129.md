## Summary

Keep open: current main still has the source-level setup path that can promote a provider default over an existing primary, and the exact narrow fix PR was closed unmerged.

## What This Changes

The issue reports that configuring a paid Google Gemini key through setup changed the default model and caused existing heartbeat background runs to use paid Gemini before an intentional chat.

## Best Possible Solution

Mirror configure-auth by passing preserveExistingDefaultModel through setup provider auth when an existing primary exists, add a setup regression test, and leave broader pricing-consent policy to https://github.com/openclaw/openclaw/issues/74395.

## Reproduction Assessment

Yes, source-level: setup forwards setDefaultModel without preserveExistingDefaultModel, the provider-auth helper only preserves when that flag is true, and heartbeat can inherit the resulting default. I did not run a live paid Google billing reproduction.

## Solution Assessment

Yes for the narrow bug: reusing the existing preservation seam in setup is the smallest maintainable fix and avoids Google-specific or heartbeat-specific policy changes. Broader price-tier consent and warning behavior belongs in the related product-policy issue.
