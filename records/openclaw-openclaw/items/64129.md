## Summary

Keep open. Current main still leaves the setup flow able to replace an existing primary model during Google API-key provider auth, so the next heartbeat without an explicit heartbeat model can run through the newly selected paid Gemini default; the later configure-specific preservation fix does not cover this setup path, and the related #75334/#74395 items do not supersede this report.

## What This Changes

Prevent Google Gemini provider setup from silently turning existing heartbeat background traffic into paid Gemini usage by changing default-model selection without explicit intent.

## Best Possible Solution

Apply the existing default-preservation contract to setup provider auth so adding Google credentials makes Gemini available without replacing an existing primary model, while leaving explicit model-picker choices and first-run no-primary setup behavior intact.

## Reproduction Assessment

Yes, by source inspection. Start with an existing agents.defaults.model.primary and heartbeat enabled without heartbeat.model, run setup with the Google Gemini API-key auth choice, and the current setup path can replace the primary while heartbeat later resolves through the normal default model path.

## Solution Assessment

Yes. The narrowest maintainable fix is to pass the existing preserveExistingDefaultModel seam through setup/provider auth and add focused regression coverage, rather than special-casing Google or changing heartbeat runtime model selection.
