## Summary

Keep open. Current main still leaves the setup provider-auth path able to replace an existing primary model with the provider default, and the linked closing fix is still an open, conflicting PR rather than merged code.

## What This Changes

The issue reports that configuring Google Gemini auth can silently promote a paid Gemini model to the default, causing existing heartbeat background runs to inherit paid usage.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81304 or an equivalent setup fix that passes preserveExistingDefaultModel through runSetupWizard while preserving first-run setup and explicit model-picker behavior.

## Reproduction Assessment

Yes, by source inspection. setup provider auth sets a default without the preservation flag, provider auth applies the Google default when that flag is absent, and heartbeat without a heartbeat.model falls through to the session/default model.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing default-preservation seam in setup provider auth rather than adding Google-specific or heartbeat-specific policy for this report.
