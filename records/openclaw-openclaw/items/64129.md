## Summary

Keep open. Current main still lets the setup wizard apply provider auth with default-model promotion but without the existing preservation flag, and heartbeat runs without `heartbeat.model` still resolve through the configured default model, so the reported paid-background-usage path remains source-reproducible.

## What This Changes

The issue asks OpenClaw to stop Google Gemini API-key setup from silently making existing heartbeat background runs use a newly configured paid Gemini default model.

## Best Possible Solution

Thread the existing default-preservation behavior through setup provider auth when an existing primary is present, while preserving first-run default selection and explicit model-picker choices.

## Reproduction Assessment

Yes, by source inspection. Start with an existing `agents.defaults.model.primary` and heartbeat enabled without `heartbeat.model`, run setup with Google Gemini API-key auth, and the setup path can replace the primary that later heartbeat turns inherit.

## Solution Assessment

Yes. Reusing the existing `preserveExistingDefaultModel` seam in setup is the narrow maintainable fix; changing heartbeat model inheritance globally or adding a new billing-confirmation feature would be broader than this report needs.
