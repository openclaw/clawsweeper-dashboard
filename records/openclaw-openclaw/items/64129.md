## Summary

Keep open. Current main still lets setup provider auth apply a provider default without `preserveExistingDefaultModel`, and the linked closing PR is still open and conflicting, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that configuring a paid Google Gemini key can silently promote Gemini to the default model, causing existing heartbeat background runs to consume paid provider usage.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81304 or an equivalent narrow fix that passes `preserveExistingDefaultModel: true` through setup provider auth with regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live billing run. Setup can still apply provider auth defaults without preserving an existing primary, and heartbeat without `heartbeat.model` falls through to the resolved session/default model.

## Solution Assessment

Yes. Reusing the existing `preserveExistingDefaultModel` contract in setup is the narrow maintainable fix; changing heartbeat policy or adding Google-specific behavior would be broader than necessary.
