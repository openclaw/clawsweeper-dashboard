## What This Changes

The PR normalizes `provider.defaultModel` before comparing it to synthesized media-generation catalog row models and adds a regression test for a whitespace-padded default model.

## Best Possible Solution

Land this narrow catalog normalization fix after required CI completes, keeping default-model and row-model normalization on the same shared helper.

## Reproduction Assessment

Yes. Current main and v2026.6.10 trim catalog row model ids through `uniqueModels` but compare the default marker to the raw `provider.defaultModel`, so a whitespace-padded default model loses `default: true`.

## Solution Assessment

Yes. Normalizing the default model with the same helper used for row model ids is the narrowest maintainable fix, and the touched helper is the shared synthesis point for the relevant callers.
