## Summary

Keep open: current main still writes anonymous transcript headers in the main transcript append path, reset/session creation paths, and compaction-related paths, and the previous duplicate cleanup accidentally left no open canonical tracker after the unmerged implementation attempt.

## What This Changes

Request to persist each transcript's logical `sessionKey` and related identity metadata in the JSONL session header so orphaned transcripts can be recovered without `sessions.json`.

## Best Possible Solution

Define a backward-compatible transcript header identity contract, probably `sessionKey` plus `agentId` first, then implement it consistently across every new-header and rotation path with docs and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the relevant header constructors write no `sessionKey` field; I did not run a live write because this review is read-only, but the write paths are clear.

## Solution Assessment

Yes in direction, but the exact contract needs maintainer agreement. Adding backward-compatible header fields is maintainable only if all header writers and rotation paths are covered and the privacy implications of raw session keys are accepted.
