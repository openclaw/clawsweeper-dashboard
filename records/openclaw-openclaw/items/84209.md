## Summary

Keep open: current `main` and `v2026.6.8` still create session transcript headers without `sessionKey` or `agentId`. The earlier duplicate issues are closed, the prior implementation PR was closed unmerged, and adjacent archive-history work does not solve portable transcript self-identification; the remaining blocker is maintainer approval for the durable metadata/privacy contract.

## What This Changes

Request to persist `sessionKey` and optionally `agentId` in the first JSONL session transcript header so orphaned or portable transcripts can identify their logical conversation.

## Best Possible Solution

Define a backward-compatible transcript-header contract for normalized `sessionKey` and derived `agentId`, document the privacy boundary, then update all header writers with focused transcript/reset/compaction tests.

## Reproduction Assessment

Yes, at source level: current `main`'s transcript header helpers and writer paths omit `sessionKey` and `agentId`. A live repro would create a session JSONL and inspect its first line, but this read-only review did not create files.

## Solution Assessment

Unclear until maintainers approve the durable metadata/privacy contract. If approved, the maintainable fix is a shared header contract used by every writer, not a plugin-local mapping file or a one-path patch.
