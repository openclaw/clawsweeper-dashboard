## Summary

Keep open: current `main` and the latest release still write anonymous JSONL session headers, and the linked implementation attempt was closed unmerged. The report is well grounded, but the exact persisted metadata contract and privacy implications need maintainer judgment before implementation.

## What This Changes

The issue requests persisting `sessionKey` and likely `agentId` in every session transcript JSONL header so transcript files remain self-identifying after reset, compaction, or loss of `sessions.json`.

## Best Possible Solution

Define a small transcript-header metadata contract, write normalized `sessionKey` and derived `agentId` on every new header path, preserve old-reader compatibility, and document the metadata/privacy implications with focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: the current header writers construct JSONL session headers without `sessionKey` or `agentId`, and the latest release has the same shape. I did not run a live gateway because this read-only review must not create session files.

## Solution Assessment

Unclear until maintainers settle the public contract. Adding `sessionKey` and derived `agentId` across all header creation paths looks like the right direction, but field set, versioning, and metadata exposure should be decided before a PR lands.
