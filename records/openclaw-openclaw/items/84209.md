- `P2`: The issue is a concrete platform improvement with source-confirmed impact on transcript recovery and tooling, but it is not an urgent runtime outage.
- `impact:data-loss`: The report is about losing the durable mapping between orphaned transcript files and their logical session identity after reset or store mutation.
- `impact:session-state`: The affected state is session transcript identity and reconstruction across session id rotations and compaction.

## Summary

Keep open: current main and v2026.5.19 still write JSONL session headers without `sessionKey` or `agentId`, and the earlier implementation attempt was closed unmerged. The request is valid, but it needs maintainer approval for the persisted metadata and privacy contract before implementation.

## What This Changes

Persist `sessionKey` and optionally `agentId` in session transcript JSONL headers so orphaned or rotated transcripts can self-identify without `sessions.json`.

## Best Possible Solution

Define a small backward-compatible transcript-header metadata contract, write normalized `sessionKey` and derived `agentId` on every new header path, and document the privacy/portability tradeoff with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows each production header writer still emits a JSONL session header without `sessionKey` or `agentId`; I did not run a live gateway because this read-only review cannot create session files in the checkout.

## Solution Assessment

Unclear until maintainers settle the public contract. Adding `sessionKey` and derived `agentId` across all header creation paths looks maintainable, but the exact fields, versioning, and metadata exposure need approval first.

## Risks / Open Questions

- Persisting `sessionKey` can expose channel, user, topic, or job identifiers wherever transcript files or support bundles are shared, so the field set and docs need an explicit privacy decision.
- A partial implementation would leave some new transcripts self-identifying and others anonymous unless normal append, direct append, embedded/bootstrap, reset, and compaction successor paths are covered together.
- The upstream `SessionHeader` type and docs do not currently include these fields, so OpenClaw should either extend its local contract deliberately or coordinate the compatibility story.
