- `impact:data-loss`: The issue is about losing the durable mapping between transcript files and their logical session identity after reset or store mutation.

## Summary

Keep open: current main and v2026.5.18 still write anonymous JSONL transcript headers, and the prior implementation attempt was closed unmerged. The CoClaw plugin context makes the consumer impact concrete, but the persisted metadata contract and privacy posture need maintainer judgment before implementation.

## What This Changes

The issue requests adding session identity metadata such as sessionKey and agentId to persisted session transcript JSONL headers so transcripts remain self-identifying after session store rotation.

## Best Possible Solution

Define a small transcript-header metadata contract, write normalized sessionKey and derived agentId on every new header path, preserve old-reader compatibility, document privacy implications, and cover reset/compaction/direct-append behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection shows current main constructs new JSONL session headers without sessionKey or agentId across the main append, direct append, embedded, and reset successor paths; I did not run a live gateway because this read-only review must not create session files.

## Solution Assessment

Unclear until maintainers settle the public contract. Adding sessionKey and a derived agentId appears directionally right, but field set, versioning, and metadata exposure should be decided before a PR lands.
