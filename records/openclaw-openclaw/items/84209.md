## Summary

Keep open: current main and the latest tagged release still write transcript headers without `sessionKey` or `agentId`. The older duplicate reports are closed, the previous implementation PR was closed unmerged, and the open family-history PR improves reset-history lookup but does not make each JSONL transcript self-identifying.

## What This Changes

Request to persist `sessionKey` and preferably `agentId` in the first JSONL session transcript header so orphaned transcripts can identify their logical conversation without `sessions.json`.

## Best Possible Solution

Approve a durable transcript metadata contract first, then implement it through the shared header helper and every current header writer with focused reset/compaction/embedded regression coverage and privacy guidance.

## Reproduction Assessment

Yes, at source level: current main’s header helpers and writer paths omit `sessionKey` and `agentId`. I did not create a live transcript in this read-only review, but the missing fields are clear from the current writers and tests.

## Solution Assessment

Unclear until maintainers approve the persistent metadata/privacy contract. If approved, the best fix is broader than reviving the old PR verbatim: update the shared header contract and all current writer/reset/embedded paths together.
