## Summary

Keep open. Current main has not implemented the PR’s no-CSS fallback, and the branch is stale against the current split markdown-it renderers, so the intent is still useful but the patch is not merge-ready.

## What This Changes

The PR adds a `hidden` fallback to code-block copy-button markup, toggles that hidden state after copy, allowlists the `hidden` attribute in markdown sanitization, and adds a regression test for fenced code blocks.

## Best Possible Solution

Refresh this against current main so both fenced and indented code-block renderers have the same non-CSS fallback, with focused markdown coverage for both forms and a narrow sanitizer approach unless global `hidden` allowlisting is explicitly needed.

## Reproduction Assessment

Yes, at source level: current main emits both copy labels without `hidden` in both code-block renderers and relies on CSS to hide `.code-block-copy__done`. Disabling, delaying, or failing to load that CSS would expose `CopyCopied!`, although I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No. The hidden fallback direction is reasonable, but this branch edits the old renderer path, misses current main’s split renderers, and needs real Control UI proof before it is a mergeable fix.
