## Summary

Keep open. The reported copy-button initial-state bug is still present on current main and in the latest release, while this PR is useful but stale against the current markdown-it renderer shape and needs one focused refresh before merge.

## What This Changes

The PR adds a `hidden` fallback to the code-block copy button's `Copied!` label, toggles that state in the chat copy handler, allowlists `hidden` during sanitization, and adds a markdown regression test.

## Best Possible Solution

Refresh or replace the branch with a current-main patch that centralizes the copy-button markup or updates both `fence` and `code_block`, gives the done label a non-CSS initial hidden state, keeps the click-state toggle symmetric, and covers both renderer paths in focused tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main emits both labels in generated copy-button HTML and relies on CSS alone to hide `Copied!`, so delayed or missing CSS exposes `CopyCopied!`.

## Solution Assessment

No. The direction is right, but this submitted patch is not the best current-main fix because it targets the older single renderer shape and does not cover the separate indented-code renderer now present on main.

## Review Findings
