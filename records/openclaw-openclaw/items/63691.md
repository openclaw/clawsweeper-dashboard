## Summary

Keep open. Current main has useful adjacent transcript-size and Gateway hot-path mitigations, but embedded run and compaction still repair and open the active session JSONL before any default oversized-transcript guard, which matches the reported Gateway freeze signature.

## What This Changes

The issue reports post-update Gateway hangs where WeChat/Feishu and other channel integrations stop responding, with follow-up diagnosis pointing to large active session JSONL parsing during bootstrap.

## Best Possible Solution

Add a default bounded pre-open active-transcript guard/recovery path before repairSessionFileIfNeeded and SessionManager.open, then rotate/quarantine/fail fast with actionable guidance instead of channel-specific fixes.

## Solution Assessment

Yes. The narrow maintainable fix is a generic pre-open active-transcript guard/recovery in the embedded run and compaction session path; release-process or channel-specific changes would not address the demonstrated event-loop blockage.
