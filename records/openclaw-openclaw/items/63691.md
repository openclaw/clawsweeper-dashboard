## What This Changes

The issue asks for update stability improvements after gateway hangs and WeChat/Feishu channel unresponsiveness following upgrades, with the discussion narrowing one reproducible cause to oversized active session JSONL files blocking the Gateway event loop.

## Best Possible Solution

Add bounded default recovery before active transcript repair/open in embedded Gateway runs: stat/classify oversized JSONL files before whole-file parsing, keep HTTP/RPC/channel handling responsive, and either rotate/compact through the existing semantic compaction path or fail fast with an actionable recovery message. Keep the fix generic to sessions/embedded runs, not WeChat or Feishu-specific.
