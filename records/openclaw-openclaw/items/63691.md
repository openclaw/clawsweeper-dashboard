## Repair Work Prompt

Fix the active-session large JSONL gateway freeze reported in https://github.com/openclaw/openclaw/issues/63691. Add a default bounded pre-open guard/recovery boundary before `repairSessionFileIfNeeded`, `prewarmSessionFile`, `SessionManager.open`, and compaction `readTranscriptFileState` paths can materialize an already-pathological active transcript. Prefer a generic session/transcript recovery helper that stats and streams enough metadata to rotate to a bounded successor transcript, quarantine the pathological active file, or fail fast with actionable logs; do not add channel-specific WeChat/Feishu/Telegram behavior and do not implement canary/rollback/release-process features in this PR. Coordinate the implementation with the broader doomloop tracker at https://github.com/openclaw/openclaw/issues/63998 so both paths share the same default safety model. Add regression coverage for oversized active transcript handling before embedded attempts and manual compaction, preserve existing opt-in `maxActiveTranscriptBytes` semantics unless intentionally replaced, and add a CHANGELOG fix entry if the behavior changes user-visible reliability.
