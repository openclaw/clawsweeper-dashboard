## Summary

Keep open. Current main still has the concrete large-active-transcript failure mode from the discussion: the embedded Gateway run path can whole-file repair and open a large JSONL before any default size guard, while the shipped active-transcript byte trigger remains opt-in.

## What This Changes

The issue requests safer OpenClaw updates after Gateway hangs and WeChat/Feishu channel integrations stop responding, plus stronger regression testing, known-issue notes, gradual rollout, and rollback support.

## Best Possible Solution

Add bounded default recovery before active transcript repair/open in the embedded Gateway run path: stat/classify oversized JSONL files before whole-file parsing, keep HTTP/RPC/channel handling responsive, and either rotate/compact through an existing bounded path or fail fast with an actionable recovery message. Preserve the existing opt-in maxActiveTranscriptBytes semantics unless a separate emergency default is introduced intentionally.

## Repair Work Prompt

Fix the Gateway/channel freeze tracked by https://github.com/openclaw/openclaw/issues/63691. The concrete failure is a very large active session JSONL causing the embedded Gateway run path to whole-file read/parse before `SessionManager.open`, monopolizing the Node event loop and freezing HTTP/RPC/channel handlers. Add a bounded default pre-open guard before `repairSessionFileIfNeeded` and `SessionManager.open` in `src/agents/pi-embedded-runner/run/attempt.ts`. Prefer a conservative stat/classification guard that keeps the Gateway responsive and either routes oversized active transcripts through existing bounded transcript rotation/compaction or fails fast with an actionable recovery message. Do not raw-split or silently delete user history, do not change WeChat/Feishu plugin behavior, and do not upgrade Pi/runtime dependencies unless a maintainer explicitly chooses that route. Preserve existing opt-in `agents.defaults.compaction.maxActiveTranscriptBytes` semantics unless intentionally adding a separate default emergency guard. Add focused regression coverage for an oversized active JSONL, update `docs/reference/session-management-compaction.md` if user-visible recovery/config behavior changes, and add a single-line `CHANGELOG.md` fix entry crediting Thanks @deciia and @HoroTW if the behavior is user-facing. Related refs: #63691, #71555, #63998, #49865. Validate with targeted tests and run `pnpm check:changed` in Testbox before handoff because this touches core agent/gateway recovery paths.
