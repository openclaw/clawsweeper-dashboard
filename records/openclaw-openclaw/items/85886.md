## Summary

Keep open for normal review and landing; the patch is a focused fix for a source-reproducible browser process leak, and the upstream Chrome DevTools MCP contract supports opting out with this flag.

## What This Changes

This PR adds `--no-usage-statistics` to the default Chrome DevTools MCP arg list, lets profile `mcpArgs` explicitly override usage-statistics behavior, and expands chrome-mcp arg-builder tests.

## Best Possible Solution

Land this narrow arg-builder fix after normal CI and maintainer review, with maintainer-owned release notes for the linked browser leak if it ships.

## Reproduction Assessment

Yes, source-reproducible: current main launches `chrome-devtools-mcp@latest` without an opt-out flag, while upstream enables usage statistics by default and spawns a detached watchdog. A direct current-main failing run was not executed in this read-only review.

## Solution Assessment

Yes; adding the documented upstream opt-out to OpenClaw's default Chrome MCP args while removing it when the user explicitly supplies usage-statistics args is a narrow maintainable fix.
