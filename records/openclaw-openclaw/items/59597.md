- **[P1] Restore shared --tools parsing in cron edit:** `src/cli/cron-cli/register.cron-edit.ts:447-450`

  - confidence: 0.93

  - body: The new preview path calls `cron.list` once without `limit` or `offset` and skips the diff if the target job is not in that returned page. Since current `cron.list` is paginated and capped, users with larger cron stores can still get a silent edit; reuse the current paginated lookup helper or a direct job lookup before printing the preview.
  - confidence: 0.86
- **[P2] Add the promised confirmation path or narrow the claim:** `src/cli/cron-cli/register.cron-edit.ts:557-560`
  - body: The linked request and PR body describe an apply confirmation with `--yes`/`--no-confirm`, but the patch prints the preview and then proceeds directly to `cron.update`. If this PR is meant to close that request, users still cannot abort after seeing the diff; otherwise the PR should stop claiming confirmation behavior.
