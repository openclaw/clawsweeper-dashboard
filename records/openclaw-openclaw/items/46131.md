## Summary

Keep this PR open but do not merge it as-is: the inline Anthropic provider default bug is still present on current main, while the branch mixes in an unrelated Telegram retry change that can retry ambiguous non-idempotent send failures.

## What This Changes

The PR applies provider-level API defaults to inline provider model construction with regression tests, and also broadens Telegram send retry handling for 429 and grammY failed-after errors.

## Best Possible Solution

Land a narrow current-main, model-only fix in the inline provider builder that applies provider-policy API defaults before explicit inline resolution, with focused regression tests and no Telegram retry changes.

## Reproduction Assessment

Yes, at source level. On current main, an Anthropic inline provider with models but no explicit API still yields an inline model with `api` undefined, and explicit resolution then skips the inline branch because it requires `inlineMatch?.api`.

## Solution Assessment

No, not as this PR branch. The model fix is the right direction, but the maintainable solution is a narrow current-main patch that uses the provider-policy default seam and leaves Telegram strict-send retry behavior unchanged.
