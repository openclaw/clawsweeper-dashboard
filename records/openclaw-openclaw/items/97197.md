## What This Changes

The PR updates the Telegram rich-text system prompt hint to spell out inline and block `<tg-math>` formula syntax and discourage common LaTeX delimiters.

## Best Possible Solution

Merge the narrow prompt hint once maintainers are satisfied with required checks; keep any converter-side LaTeX parsing as a separate product decision if prompting later proves insufficient.

## Reproduction Assessment

Yes at source level: current main builds a Telegram richText prompt that advertises formulas without tag syntax, while the formatter and Telegram API require `<tg-math>` / `<tg-math-block>` for rich math rendering. I did not run a live before-fix model turn.

## Solution Assessment

Yes. This is the narrowest maintainable fix for the prompt gap because the Telegram rich formatter already supports the target tags; a converter-side LaTeX parser would be broader and risk over-parsing code, currency, or literal text.
