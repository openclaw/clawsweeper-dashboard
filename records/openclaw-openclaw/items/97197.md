## What This Changes

The PR changes the Telegram rich-text system prompt hint in `src/agents/system-prompt.ts` to name inline and block `<tg-math>` tags and discourage common LaTeX delimiters.

## Best Possible Solution

Land the narrow prompt hint after normal CI and maintainer review; keep delimiter-to-tag conversion as separate product work only if maintainers want a deterministic parser.

## Reproduction Assessment

No live current-main model run was performed, but the source/API path is high-confidence: current main tells rich Telegram models only `formulas`, while Telegram rich HTML math uses `<tg-math>` and `<tg-math-block>` and OpenClaw already preserves those tags.

## Solution Assessment

Yes. This is the narrowest maintainable fix for the prompt gap because the rich formatter already supports the target tags, while delimiter parsing would be broader and easier to misapply to code or currency text.
