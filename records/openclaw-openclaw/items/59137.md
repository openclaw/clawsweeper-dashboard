## Summary

Keep this PR open: the memory reindex recovery fix is well-scoped and supported by real terminal proof, but the current head should not merge until it preserves current main's temp-DB cleanup contract during failed safe reindex rollback.

## What This Changes

The PR adds memory-core sync-state rollback restoration, a `sessionFullRetryPending` retry sentinel, per-batch embedding-cache mirror writes during safe reindex, cache-table seeding guards, regression tests, and a changelog entry.

## Best Possible Solution

Keep the retry-state and cache-mirroring design, but restore current-main `beforeTempCleanup`/`tempDbClosed` handling in `runSafeReindex` before this lands; leave broader transient transport retry and batch splitting to https://github.com/openclaw/openclaw/issues/44166.

## Reproduction Assessment

Yes. Current main is source-reproducible for the reported gap because cache writes are delayed until all direct embedding batches finish and rollback does not restore retry-state fields; the PR body also supplies real before/after terminal proof.

## Solution Assessment

Mostly yes, but not mergeable as-is. Snapshotting rollback-relevant sync state and mirroring successful cache batches is the narrow fix, but the safe-reindex implementation must preserve current main's temp DB cleanup before atomic rollback cleanup runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Close the temp DB before atomic cleanup:** `extensions/memory-core/src/memory/manager-sync-ops.ts:1270-1274`
  - body: Current main passes `beforeTempCleanup` so `runMemoryAtomicReindex` closes the temp SQLite handle before removing temp files after a failed build. This branch no longer provides that callback, and its catch closes `this.db` only after the helper has already tried to remove the temp files. On locked sqlite sidecars, rollback can become an `AggregateError` and leave temp files instead of cleanly surfacing the provider failure. Please restore the `beforeTempCleanup`/`tempDbClosed` handling while keeping the new retry-state restoration.
  - confidence: 0.91
