## Summary

Keep this PR open for review rather than closing: current main still has the reported Feishu/Lark missing-target gap, but the repaired branch still accepts unresolved session-derived targets and lacks real behavior proof.

## What This Changes

The PR adds cron service validation, regression tests, docs, and changelog text for rejecting Feishu/Lark announce jobs without an explicit or session-derived delivery target.

## Best Possible Solution

Land one consolidated fix that validates Feishu/Lark announce targets at the cron service boundary without accepting unresolved session keys, while preserving safe legacy cleanup edits.

## Reproduction Assessment

Yes, source-derived: current main still builds and accepts Feishu/Lark announce delivery without delivery.to, while runtime target resolution can later fail when no target is available.

## Solution Assessment

No. Service-level fail-fast validation is the right direction, but this branch still treats a session identifier as delivery proof and skips validation for some update paths that can make invalid jobs runnable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Verify session-derived Feishu targets before accepting them:** `src/cron/service/jobs.ts:225`
  - body: The new guard accepts missing delivery.to whenever resolveCronDeliverySessionKey(job) returns any string. That only proves a sessionTarget/sessionKey exists; resolveDeliveryTarget later loads the session store and can still fail with no deliverable Feishu/Lark recipient, so create/update can still accept jobs that fail at delivery time.
  - confidence: 0.9
- **[P2] Reject enabling invalid Feishu announce jobs:** `src/cron/service/jobs.ts:703-705`
  - body: patchTouchesDeliveryTarget skips the new validation for enabled-only updates, so cron enable on a legacy Feishu/Lark announce job without a target can make a runtime-failing job runnable again. Keep the disable remediation path, but reject enabling or otherwise activating unresolved delivery.
  - confidence: 0.82
