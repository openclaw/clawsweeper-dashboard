## What This Changes

The PR makes memory-wiki vault scans degrade malformed YAML frontmatter into a `frontmatterError` summary and `invalid-frontmatter` lint issue while leaving direct markdown parsing strict.

## Best Possible Solution

Merge one summary-scoped memory-wiki fix that keeps direct edit parsing strict, reports malformed frontmatter as lint, and allows the rest of the vault scan to continue.

## Reproduction Assessment

Yes. Source inspection shows malformed frontmatter reaches `YAML.parse`, `readPageSummaries` uses stop-mode concurrency and rethrows, and both `wiki_apply` and `wiki_lint` can reach that compile/lint path.

## Solution Assessment

Yes. Catching only in `toWikiPageSummary` is the narrowest maintainable fix because vault scans degrade the unrelated bad page while direct edit-target callers of `parseWikiMarkdown` still fail instead of rewriting metadata from an empty frontmatter object.
