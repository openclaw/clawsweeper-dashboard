Keep open: the PR is a focused, currently useful bug fix and current main still lacks the tolerant summary/lint behavior. The patch looks correct, but a human should reconcile it with the overlapping open candidate before merge.

## What This Changes

The PR makes memory-wiki vault scans degrade pages with unparsable YAML frontmatter into summaries with `frontmatterError`, reports them as `invalid-frontmatter` lint issues, and adds markdown/lint regression coverage.

## Best Possible Solution

Land one summary/lint-scoped fix that keeps direct edit/import parsing strict, reports malformed page frontmatter through lint, and then close or supersede the overlapping candidate and the canonical issue after merge.

## Reproduction Assessment

Yes. Source inspection shows malformed frontmatter reaches `YAML.parse`, `readPageSummaries` stops and rethrows, and the reported `wiki_apply` and `wiki_lint` tools both reach that compile path.

## Solution Assessment

Yes. Catching only at `toWikiPageSummary` is the narrowest maintainable fix because vault scans can degrade an unrelated bad page while direct edit/import parsing stays strict enough to avoid silently dropping metadata.
