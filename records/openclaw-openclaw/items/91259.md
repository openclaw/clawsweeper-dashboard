## Summary

Keep open: current main and the latest release still use agent-scoped QMD collection names, and docs/tests make that behavior part of the current cross-agent memory contract. Removing the suffix may be a good cleanup, but it needs a maintainer decision on the canonical collection-name and migration/rebuild path.

## What This Changes

The issue proposes removing the `-<agentId>` suffix from managed QMD memory collection names because each agent already has an isolated QMD index.

## Best Possible Solution

Maintain the current scoped behavior until maintainers choose a new canonical QMD collection-name contract; if accepted, land one focused cleanup that updates naming, migration/rebuild logic, docs, and regression tests together.

## Reproduction Assessment

Yes for the current behavior: source and tests show current main deterministically emits agent-suffixed QMD collection names. I did not run a live QMD instance because this read-only review can prove the behavior from source and regression tests.

## Solution Assessment

Unclear: removing the suffix is plausible because QMD state is already per-agent, but scoped names are currently documented and tested. Maintainers should first decide whether the suffix is a compatibility contract or cleanup debt.
