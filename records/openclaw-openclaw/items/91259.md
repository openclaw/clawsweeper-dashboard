## Summary

Keep open: current main and v2026.6.8 still use agent-scoped QMD collection names, and https://github.com/openclaw/openclaw/pull/91274 is the open implementation candidate that needs maintainer compatibility review before this issue can close.

## What This Changes

The issue requests dropping redundant agent-id suffixes from managed QMD memory, session, and extra collection names because each agent already has an isolated QMD index.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/91274 with maintainer compatibility sign-off, or deliberately keep scoped QMD collection names as a supported operational contract.

## Reproduction Assessment

Yes, source-level: current main deterministically emits agent-suffixed QMD collection names and migrates legacy unscoped names toward scoped names. I did not run live QMD because this is a cleanup/product-direction issue and the current behavior is proven by source, tests, docs, release inspection, and upstream QMD config-path contracts.

## Solution Assessment

Unclear until maintainer sign-off: the active PR is the best current implementation candidate if QMD collection names are accepted as rebuildable internal derived-index state. If those names are an operational contract, the safer solution is to keep scoped names or require a compatibility window.
