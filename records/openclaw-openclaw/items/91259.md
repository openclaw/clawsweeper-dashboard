## Summary

Keep open: current main and the latest release still use agent-suffixed QMD collection names, while the open linked PR now owns the proposed de-scoping and migration work. This issue should remain open until that PR is landed or maintainers decide to keep the existing collection-name contract.

## What This Changes

The issue asks to remove redundant agent-id suffixes from managed QMD collection names and reverse the legacy migration from scoped names to stable base names.

## Best Possible Solution

Review the linked PR as the canonical implementation path, and if maintainers accept the contract change, land it with the scoped-to-unscoped migration, docs updates, and regression coverage together.

## Reproduction Assessment

Yes for the current behavior: source, tests, docs, and the latest release all show agent-suffixed QMD collection names. This is a cleanup/product-contract issue rather than a failing runtime bug reproduction.

## Solution Assessment

Unclear until maintainer review: removing the suffix is technically plausible because QMD storage is per-agent, but it intentionally changes a shipped/documented collection identity and migration direction. The linked PR is the right place to evaluate that tradeoff.
