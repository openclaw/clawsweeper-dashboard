## Summary

Keep open: current main and v2026.6.11 still use agent-scoped QMD collection names, and https://github.com/openclaw/openclaw/pull/91274 is the active implementation candidate waiting on a maintainer product and compatibility decision.

## What This Changes

Request to remove the `-<agentId>` suffix from managed QMD memory, session, and extra collection names and migrate per-agent indexes to unscoped collection names.

## Best Possible Solution

Make the maintainer product call on QMD collection identity, then either land https://github.com/openclaw/openclaw/pull/91274 with compatibility sign-off or close the proposal and PR as an intentional scoped-name contract.

## Reproduction Assessment

Yes, source-level: current main deterministically appends the sanitized agent id to managed QMD collection names and tests still assert scoped names. I did not run live QMD because the remaining question is product direction and upgrade compatibility.

## Solution Assessment

Unclear until maintainers decide the product contract: the linked PR is the best current implementation candidate if QMD collection names are internal rebuildable index metadata. If scoped names are an operational or future shared-index contract, the better solution is to keep them or require a compatibility path.
