## What This Changes

The issue proposes removing redundant `-<agentId>` suffixes from managed QMD collection names because each agent already has an isolated QMD index.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/91274 with explicit compatibility sign-off, or decide scoped QMD collection names are intentional and close the proposal plus candidate PR.

## Reproduction Assessment

Yes, source-level: current main deterministically appends the sanitized agent id to managed QMD memory/custom/session collection names, and tests/docs still assert scoped examples. I did not run live QMD because the remaining question is product and upgrade compatibility, not whether the current code path exists.

## Solution Assessment

Unclear until maintainer sign-off: the linked PR is the best current implementation candidate if QMD collection names are internal rebuildable index metadata. If scoped names are an operational or future shared-index contract, keeping them or adding a compatibility window is safer.
