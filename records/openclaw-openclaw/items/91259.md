## Summary

Keep open: current main and the latest release still use agent-scoped QMD collection names, and https://github.com/openclaw/openclaw/pull/91274 is the active implementation candidate that needs maintainer compatibility review before this issue can close.

## What This Changes

The issue proposes removing the agent-id suffix from QMD memory, session, and custom collection names and migrating existing per-agent indexes from scoped to unscoped names.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/91274 with explicit compatibility sign-off, or keep scoped QMD collection names as the supported operational contract.

## Reproduction Assessment

Yes, source-level: current main deterministically appends the sanitized agent id to managed QMD collection names and tests/docs still assert scoped names. I did not run live QMD because the remaining question is cleanup direction and upgrade compatibility, not an unverified runtime failure.

## Solution Assessment

Unclear until maintainer sign-off: the linked PR is the best current implementation candidate if QMD collection names are rebuildable internal derived-index state. If operators rely on those names or the historical cross-agent collision risk still applies, the safer solution is to keep scoped names or require an explicit compatibility path.
