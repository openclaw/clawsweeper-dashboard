Request to remove redundant agent-id suffixes from managed QMD collection names and migrate per-agent derived indexes from scoped to unscoped collection identifiers.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/91274 as the canonical implementation path; if maintainers accept the migration tradeoff, land that focused cleanup with its docs and regression coverage.

## Reproduction Assessment

Yes. Source and tests on current main show deterministic agent-suffixed QMD collection names and scoped legacy migration behavior; I did not run QMD because this read-only review proves the current behavior from source.

## Solution Assessment

Unclear until maintainer review. The linked PR appears to be the best concrete implementation path, but the repository still needs an explicit decision that QMD collection names are rebuildable derived state rather than a shipped identifier contract.
