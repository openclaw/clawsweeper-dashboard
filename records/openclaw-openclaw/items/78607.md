## Summary

Keep open: current main still has a source-reproducible gateway agent-session path where daily rollover changes `sessionId` while preserving the old persisted `sessionFile`; related PR work covers `sessions.reset` but not this daily agent-send path.

## What This Changes

Daily reset should rotate both the stored `sessionId` and the active transcript `sessionFile` so long-lived sessions start writing to a fresh JSONL file.

## Best Possible Solution

Fix the gateway reset write paths so any generated session-id rollover derives or rewrites generated transcript filenames to the new id while preserving intentional custom transcript names, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed `agent:main:main` with stale `sessionStartedAt`, `sessionId=old`, and `sessionFile=<old>.jsonl`, then invoke the gateway agent path; current code generates a new `sessionId` but merges without a `sessionFile` patch, preserving the old path.

## Solution Assessment

Yes. The narrow maintainable fix is in the reset write paths, not in reset policy or configuration: set or rewrite `sessionFile` whenever the stored `sessionId` changes, and reuse the existing generated-filename rewrite semantics from compaction/reset work.
