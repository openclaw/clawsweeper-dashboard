## Summary

Keep this open: the reported stale `sessionFile` rollover is still source-reproducible on current main through the gateway agent ingress path, even though the older auto-reply resolver guard exists.

## What This Changes

Daily session reset should rotate both `sessionId` and the active transcript `sessionFile` instead of carrying the old JSONL path forward.

## Best Possible Solution

Make stale-session rollover force a new generated transcript path before the dispatched agent command can reload the already-updated row, with regression coverage for `agent:main:main` daily reset.

## Reproduction Assessment

Yes by source inspection: seed a stale `agent:main:main` row with `sessionId=old` and `sessionFile=old.jsonl`, then the gateway agent path writes `sessionId=new` while preserving `sessionFile`, and the later resolver reuses that old path. I did not execute a live gateway repro because this review is read-only.

## Solution Assessment

Yes: the narrow fix is to keep the existing resolver semantics but pass the pre-reset state, clear `sessionFile`, or otherwise force transcript rotation when the gateway agent path changes `sessionId`. Reusing the stale PR direction is reasonable only if it is extended to this daily ingress path and keeps custom filenames intact.

Fix https://github.com/openclaw/openclaw/issues/78607 on current main. The bug is in daily/stale gateway agent ingress rollover: `src/gateway/server-methods/agent.ts` generates a new `sessionId` for a stale row, persists `nextEntryPatch` with `mergeSessionEntry`, and because the patch omits `sessionFile`, the old transcript path survives. The dispatched `agentCommandFromIngress` then reloads the same row as fresh, so `resolveAndPersistSessionFile()` sees `baseEntry.sessionId === sessionId` and reuses the old `sessionFile`. Implement a narrow fix that forces transcript rotation when the gateway agent path changes session id: pass the pre-reset entry to the resolver, clear/rewrite `sessionFile` on `isNewSession`, or move transcript resolution before the store row is made fresh. Preserve custom-named explicit `sessionFile` behavior, preserve topic suffix behavior, and do not add a config option or broad session maintenance policy. Add regression coverage for a stale `agent:main:main` row with old `sessionId`, old `sessionFile`, and `sessionStartedAt` before the 4 AM daily boundary; after invoking the gateway agent send path, assert the persisted `sessionFile` basename follows the new `sessionId` and the old JSONL path is not reused. Check related stale PR https://github.com/openclaw/openclaw/pull/57126 for useful reset-service context, but do not depend on that stale branch.
