## Summary

Keep open: current main still has a source-reproducible daily gateway rollover path that changes `sessionId` while preserving the old `sessionFile`, and the linked fix PR is still open rather than merged.

## What This Changes

The issue reports that daily session rollover updates `sessionId` but keeps the old persisted `sessionFile`, so transcripts continue growing after reset boundaries.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78618 or a replacement lands; the fix should update the gateway rollover persistence path so generated transcript filenames follow a new `sessionId` while preserving intentional custom transcript paths, with regression coverage and changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed a gateway session entry with stale `sessionStartedAt`, `sessionId=old`, and `sessionFile=<old>.jsonl`, then invoke the gateway agent path; current main generates a new `sessionId` but persists a patch that omits `sessionFile`, so the merge keeps the old path.

## Solution Assessment

Yes, the maintainable direction is a narrow reset-write fix rather than a policy or configuration change. The linked PR is the active fix path, but the issue should not close until that PR or an equivalent replacement is merged.
