## Summary

Keep open. The PR addresses a real cron status/delivery problem, but the proposed normalization happens after channel delivery and can turn real Apply Patch/Edit failures into successful cron runs; the existing ClawSweeper review comment also correctly flagged this blocker.

## What This Changes

The PR changes cron delivery dispatch to use completed descendant summaries when parent text is empty and normalizes delivered Apply Patch/Edit warning results from `error` to `ok` in cron state, task completion, and finished events.

## Best Possible Solution

Move nonfatal Apply Patch/Edit warning classification into isolated-run payload/result handling using explicit success evidence, keep the descendant-summary delivery fix, and add an Unreleased changelog entry.

## Reproduction Assessment

Yes. A focused unit path can pass `status: "error"`, `delivered: true`, and an Apply Patch/Edit error into the new normalizer; current main shows `delivered` only proves channel delivery, so that path reproduces the masking risk.

## Solution Assessment

No. Normalizing in `timer.ts` after delivery is too late; the safer fix is to classify nonfatal warnings before `resolveRunOutcome` maps the isolated run to `ok` or `error`.

## Review Findings
