## Summary

Keep open. The PR addresses a real transcript/session-state regression path, but the new harness-rotation early return can write a successor transcript without emitting the post-compaction transcript update and memory-sync side effects that the existing compaction paths already run.

## What This Changes

The PR synthesizes a compaction boundary and rotates the active transcript after successful agent-harness compaction when the harness does not return its own successor transcript, with focused tests.

## Best Possible Solution

Keep the synthetic rotation approach, but make the harness-rotation path emit the same post-compaction transcript update and memory-sync side effects as the existing direct and context-engine compaction paths before landing.

## Reproduction Assessment

Yes, at source level: current main returns the harness compaction result before any transcript rotation when the harness does not delegate a successor. The PR body also supplies a real terminal smoke showing the intended after-fix behavior, but I did not rerun it in this read-only pass.

## Solution Assessment

No, not yet. The core fix direction is maintainable, but the new rotated harness path should also call the established post-compaction side-effect hook before returning.
