## Summary

Keep open: the MiniMax/CN failover gap is still present on current main, but this PR should not merge yet because it lacks real MiniMax proof and moves MiniMax-specific classification into core/provider-agnostic paths instead of the provider-owned failover hook.

## What This Changes

The PR adds MiniMax/CN rate-limit classifiers, a MiniMax assistant text-body failover classifier, runner wiring, tests, and a changelog entry.

## Best Possible Solution

Route MiniMax-specific 2062 and text-body recognition through the MiniMax provider-owned failover hook, keep the precise tests, and merge only after redacted live MiniMax proof is added.

## Reproduction Assessment

Yes for the source-level classifier gap: current main lacks the MiniMax Chinese rate-limit strings and 2062 patterns while fallback depends on classification. The live MiniMax provider path is not reproduced because the PR has no runtime proof.

## Solution Assessment

No as written. The tests and removal of standalone generic retry matching are useful, but MiniMax-specific classification should use the provider-owned hook rather than adding owner-specific logic to core, and merge still needs real behavior proof.
