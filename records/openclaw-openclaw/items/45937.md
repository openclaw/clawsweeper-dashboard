## Summary

Keep open: the MiniMax rate-limit gap is still real on current main, but the PR's current diff is too broad and can misroute successful or unrelated provider responses into failover.

## What This Changes

The PR adds MiniMax/CN rate-limit matchers to failover classification and changes assistant failover detection to scan short non-error assistant text for rate-limit or overload phrases.

## Best Possible Solution

Land a narrow revision that keeps precise MiniMax error indicators, avoids standalone generic retry text and ordinary assistant-prose scanning, and adds regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main does not match the exact MiniMax sample markers, and the PR diff's false-positive paths follow directly from the classifier order and runner reason derivation.

## Solution Assessment

No. The safer fix is to add only high-confidence MiniMax rate-limit indicators to the existing provider-error classifier, remove standalone `请稍后重试`, and avoid classifying normal non-error assistant content unless a structured provider error payload is available with reason propagation.
