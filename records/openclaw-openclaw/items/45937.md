## Summary

Keep open. Current main has broad CJK failover coverage from #56242, but it still does not include the exact MiniMax markers; this PR keeps a useful intent while the current diff introduces false-positive fallback behavior and loses failover-reason metadata.

## What This Changes

This PR adds MiniMax-specific Chinese rate-limit matchers and expands assistant failover detection to scan short non-error assistant text.

## Best Possible Solution

Land a narrow revision or replacement that keeps high-confidence MiniMax markers, removes generic retry-text matching, preserves reason metadata for any explicit content-as-error path, and adds negative tests plus changelog coverage.

## Reproduction Assessment

Yes, by source inspection rather than live execution. Current main lacks the exact MiniMax markers, and the PR diff clearly makes broad short assistant content eligible for failover matching.

## Solution Assessment

No. The safe solution is narrower than this diff: keep precise MiniMax indicators, avoid standalone `请稍后重试`, avoid broad assistant-content scanning, preserve the matched reason, and add tests/changelog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict text matching to provider error payloads:** `src/agents/pi-embedded-helpers/errors.ts:1049`
  - body: The new non-error branch scans any short assistant content with broad rate-limit and overload matchers. A successful answer that mentions `rate limit`, `usage limit`, or `high demand` can be treated as a provider failure, causing profile rotation or fallback instead of returning the response.
  - confidence: 0.91
- **[P2] Remove generic retry text from rate-limit matching:** `src/agents/pi-embedded-helpers/failover-matches.ts:21`
  - body: `请稍后重试` means "please retry later" and is commonly appended to non-rate-limit failures. Since rate-limit classification runs before timeout, billing, and auth paths, this can force unrelated Chinese provider errors into `rate_limit`.
  - confidence: 0.9
- **[P2] Preserve the matched reason for text-only failovers:** `src/agents/pi-embedded-helpers/errors.ts:1049`
  - body: The content branch returns only `true`, while the runner still derives `assistantFailoverReason` from `errorMessage`, which is usually empty for the targeted non-error case. These retries can be emitted as `unknown` and skip rate-limit or overload-specific handling.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/pi-embedded-helpers/failover-matches.ts:18-22`
  - body: This changes user-facing provider fallback behavior, but the PR only edits helper code. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing feature, fix, and performance changes.
  - confidence: 0.82
