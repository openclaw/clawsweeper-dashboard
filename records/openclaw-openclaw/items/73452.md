## Summary

Close this PR as superseded by the broader Mattermost streaming-mode PR. Current main still has the underlying Mattermost streaming-control gap, but this one-line patch would enable block replies while the draft-preview callbacks still own the turn, so it is not the right implementation path.

## What This Changes

The PR changes Mattermost's draft-preview dispatch path to derive `replyOptions.disableBlockStreaming` from `account.blockStreaming` instead of always forcing block streaming off there.

## Best Possible Solution

Use #73252 or an equivalent Mattermost-local fix as the canonical path: add schema/type support for `channels.mattermost.streaming`, gate draft-preview callbacks from the resolved preview mode, keep block replies and preview edits mutually exclusive for a turn, add focused tests, and include the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence code reproduction is available from the PR diff: set `account.blockStreaming` to true and the patched call passes `disableBlockStreaming: false` while `onPartialReply` and draft-preview tool/reasoning callbacks remain active.

## Solution Assessment

No. The one-line alignment matches non-preview paths but is not the best fix for the draft-preview path because it can route one reply through both block messages and the preview post; the safer solution is explicit Mattermost preview-mode handling as in #73252.
