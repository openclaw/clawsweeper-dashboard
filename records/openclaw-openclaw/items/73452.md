## Summary

Keep this PR open, but do not merge the current patch as-is. The code evidence supports the existing review finding: the change would let Mattermost block replies and draft-preview updates run during the same turn, while the related open issues show the intended Mattermost streaming behavior still needs maintainer confirmation.

## What This Changes

The PR changes Mattermost draft-preview reply dispatch to derive `replyOptions.disableBlockStreaming` from `account.blockStreaming` instead of always passing `true` in `extensions/mattermost/src/mattermost/monitor.ts`.

## Best Possible Solution

Implement Mattermost streaming as an explicit delivery-mode choice: honor `channels.mattermost.streaming` for draft previews, keep `streaming.block.enabled`/legacy `blockStreaming` as block-reply enablement, and ensure preview streaming and block replies cannot both own the same turn. Add focused Mattermost monitor coverage plus a single-line user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is available: apply the PR option mapping with `account.blockStreaming: true`, observe core resolve `disableBlockStreaming: false` to block streaming on, and note the same dispatch still registers Mattermost draft-preview callbacks.

## Solution Assessment

No. The sibling non-preview mapping is appropriate for plain reply paths, but applying it to the draft-preview path is not the safest fix because it can enable block replies while draft preview updates are active; the safer path is to choose one delivery mode before dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not enable block replies during draft preview:** `extensions/mattermost/src/mattermost/monitor.ts:1747-1750`
  - body: With this change, `account.blockStreaming: true` passes `disableBlockStreaming: false` while the same dispatch still registers draft-preview callbacks such as `onPartialReply`. Core treats that as block streaming enabled, and Mattermost's draft-preview finalizer delivers `kind: "block"` payloads normally, so one turn can surface through both block messages and the live draft preview. Keep block streaming suppressed while preview owns the turn, or skip preview when block streaming is selected.
