## Summary

Current main already implements the central fix for the built-in compaction safeguard: prior summaries are re-distilled with new messages, the old summary is not passed as SDK update state, and final summaries are capped. The fix is current-main-only based on tag containment; it is not in the latest release tag from the prompt.

## What This Changes

The issue requests changing the compaction summarizer so it re-compresses prior summaries within a stable budget instead of extending them monotonically.

## Best Possible Solution

Keep the current main implementation and let the next release carry the bounded redistill behavior; the separate preflight scheduler issue should remain tracked independently.

## Reproduction Assessment

No current-main failing reproduction remains: source and tests show the built-in safeguard now re-distills prior summaries and caps final output. The incident report is credible for older behavior, but the checked main branch no longer matches the reported prompt shape.

## Solution Assessment

Yes. Treating the previous summary as bounded input, unsetting the SDK `previousSummary` state, and capping the final summary is the narrow maintainable solution without adding config or changing product policy.
