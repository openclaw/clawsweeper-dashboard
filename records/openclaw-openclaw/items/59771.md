## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep canonical stream state raw:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` sanitizes snapshots before `mergeStreamingText()` stores them as canonical state. A partial like `Here is \`co` gets a synthetic closing backtick, so the next real snapshot `Here is \`code\`` no longer shares a clean prefix and can be concatenated/corrupted; sanitize only outbound Card Kit payloads.
  - confidence: 0.9
- **[P1] Reset stale streaming refs during recovery:** `extensions/feishu/src/reply-dispatcher.ts:314`
  - body: `clearStreamingProgress()` clears text but keeps the current `streaming` object and `streamingStartPromise`. If recovery enters with a closed or stale session, `startStreaming()` remains blocked by those refs and a recovered final can fall back to a separate visible reply.
  - confidence: 0.84
- **[P2] Serialize recovery cleanup before retry delivery:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: The new error path awaits `clearStreamingProgress()`, but the shared dispatcher invokes `onError` without awaiting it. A recovered final can therefore enter delivery before partial updates drain and buffers clear, leaving the stale-merge race in place.
  - confidence: 0.83
