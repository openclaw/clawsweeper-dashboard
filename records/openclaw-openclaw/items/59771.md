## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep sanitized output out of canonical stream state:** `extensions/feishu/src/streaming-card.ts:397`
  - body: `update()` sanitizes each partial before `mergeStreamingText` stores it. A partial like `Here is `co` gets a synthetic closing backtick, so the later raw snapshot `Here is `code`` no longer prefix-matches and can concatenate or corrupt the visible card; sanitize only the outbound CardKit payload.
  - confidence: 0.9
- **[P1] Reset stale streaming refs when recovery cannot reuse them:** `extensions/feishu/src/reply-dispatcher.ts:311`
  - body: `clearStreamingProgress()` clears text but leaves `streaming` and `streamingStartPromise` installed. If the session is already closed or `clearText()` returns, `startStreaming()` remains blocked by the stale object and a recovered final can fall back into a second visible reply.
  - confidence: 0.84
- **[P2] Serialize cleanup before retry delivery:** `extensions/feishu/src/reply-dispatcher.ts:491`
  - body: The Feishu handler awaits `clearStreamingProgress()`, but the shared dispatcher invokes `onError` without awaiting it. A retry final can run before queued updates drain and buffers reset, preserving the duplicate or corrupt reply race.
  - confidence: 0.83
