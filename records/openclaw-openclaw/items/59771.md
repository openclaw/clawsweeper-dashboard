Full review comments:

- **[P1] Keep raw streaming state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` sanitizes snapshots before `mergeStreamingText()` stores them as pending/current state. A later raw snapshot, such as an incremental code span, can stop sharing a clean prefix and be concatenated or corrupted; keep state raw and sanitize only outbound Card Kit payloads.
  - confidence: 0.9
- **[P1] Clear stale streaming refs during error recovery:** `extensions/feishu/src/reply-dispatcher.ts:314-315`
  - body: `clearStreamingProgress()` clears text but leaves the current streaming object and start promise installed. If recovery enters with a closed or stale session, `startStreaming()` remains blocked by the stale reference and retry delivery can fall back to a separate visible reply.
  - confidence: 0.84
- **[P2] Await error cleanup before retry delivery can run:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: This handler awaits `clearStreamingProgress()`, but the shared dispatcher invokes `onError` without awaiting it. A recovered final can enter delivery before queued partial updates drain and buffers clear, leaving the stale-merge race in place.
  - confidence: 0.83
