## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep canonical streaming state raw:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` sanitizes each streamed snapshot before `mergeStreamingText()` stores it as pending/current state. A later raw snapshot, such as `Here is `co` followed by `Here is `code``, no longer shares the same prefix and can concatenate or corrupt the card; sanitize only outbound Card Kit payloads.
  - confidence: 0.9
- **[P1] Reset stale streaming refs on recovery:** `extensions/feishu/src/reply-dispatcher.ts:314-315`
  - body: `clearStreamingProgress()` clears text but leaves `streaming` and `streamingStartPromise` installed. If a previous close failed after marking the session closed, `clearText()` returns and the stale session keeps `startStreaming()` from opening or reusing a card, so a recovered final can fall back to a separate visible reply.
  - confidence: 0.84
- **[P2] Serialize error cleanup before retry delivery:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: This handler awaits `clearStreamingProgress()`, but the shared dispatcher calls `options.onError?.()` without awaiting it. A recovered final can run before queued updates drain and buffers reset, leaving the stale merge and duplicate-reply race in place.
  - confidence: 0.83
