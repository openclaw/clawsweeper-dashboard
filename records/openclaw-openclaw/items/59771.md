## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` sanitizes snapshots before `mergeStreamingText()` stores them as canonical state. A later raw snapshot or final such as an incremental code span can stop sharing a clean prefix and be concatenated or corrupted; keep state raw and sanitize only outbound Card Kit payloads.
  - confidence: 0.9
- **[P1] Clear stale streaming refs on recovery cleanup:** `extensions/feishu/src/reply-dispatcher.ts:314`
  - body: `clearStreamingProgress()` clears text but leaves the current `streaming` object and `streamingStartPromise` installed. If recovery is entered with a closed/stale session, `startStreaming()` remains blocked by the stale reference and retry delivery can fall back to a separate visible reply.
  - confidence: 0.84
- **[P2] Serialize error cleanup before retry delivery:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: The Feishu error path awaits `clearStreamingProgress()`, but the shared dispatcher invokes `onError` without awaiting it. A recovered final can therefore enter delivery before partial updates drain and buffers clear, leaving the stale-merge race in place.
  - confidence: 0.83
