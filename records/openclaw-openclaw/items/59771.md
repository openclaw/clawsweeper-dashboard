## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep stream state raw before sanitizing:** `extensions/feishu/src/streaming-card.ts:398-400`
  - body: `sanitizeCardKitMarkdown()` runs before the snapshot is merged into `currentText`, so a partial like `Here is `co` gets a synthetic closing backtick stored as canonical state. The next real snapshot with the natural closing backtick no longer shares a clean prefix, and `mergeStreamingText` can concatenate/corrupt the visible card. Keep raw text in the stream state and sanitize only the outbound CardKit payload.
  - confidence: 0.9
- **[P1] Reset closed streaming sessions during recovery:** `extensions/feishu/src/streaming-card.ts:432-434`
  - body: `clearText()` returns when the session is already marked closed, but `close()` marks `closed = true` before later network work. If that close path throws before the dispatcher clears `streaming`, recovery leaves a stale session reference that blocks `startStreaming()` and can force a separate fallback/final message.
  - confidence: 0.83
- **[P2] Serialize cleanup before retry final delivery:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: `clearStreamingProgress()` is awaited inside Feishu `onError`, but the core dispatcher calls `onError` from its catch path without awaiting it. A recovered final can therefore run before stale buffers/session text are cleared, reintroducing the duplicate or corrupted recovery this PR is trying to prevent.
  - confidence: 0.82
