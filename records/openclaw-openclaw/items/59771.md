## Summary

Keep open. Current main still closes the active Feishu streaming card from the reply-error path, so the PR's intended recovery behavior is not already implemented; however, the PR head is not mergeable as-is because it can corrupt streamed markdown state and still has stale-session and async cleanup races.

## What This Changes

The PR adds Feishu streaming-session `clearText()` recovery, switches reply `onError` cleanup to clear buffers without closing the active streaming card, adds regression coverage, and bundles Feishu streaming markdown/reasoning sanitation changes.

## Best Possible Solution

Adapt the recovery idea as a focused Feishu fix: keep one streaming card alive through transient reply errors, reset stale text/session state in a serialized way, preserve the newer idle/final dedupe behavior, and keep markdown sanitizer work separate or sanitize only outbound Card Kit payloads without mutating canonical streaming state.

## Reproduction Assessment

Yes. The sanitizer corruption can be reproduced on the PR head with incremental snapshots such as `Here is `co` followed by `Here is `code``, and the async cleanup race is source-backed by the core dispatcher not awaiting `onError`; the broader duplicate-reply report is supported by #49381 logs but was not live-tested in this read-only pass.

## Solution Assessment

No. Keeping the card open and clearing stale buffers is a plausible direction, but this branch is not the best implementation until cleanup is serialized, closed/stale session references are handled, and sanitizer changes are separated or made state-safe.

- **[P1] Do not sanitize snapshots before merging:** `extensions/feishu/src/streaming-card.ts:399-400`
  - body: `sanitizeCardKitMarkdown()` appends synthetic closing backticks to partial snapshots before they become canonical streaming state. A later real snapshot no longer shares the same prefix, so `mergeStreamingText()` can fall back to concatenation and corrupt the visible reply; keep state raw and sanitize only the outbound Card Kit payload.
  - confidence: 0.9
- **[P1] Reset stale streaming refs after failed closes:** `extensions/feishu/src/reply-dispatcher.ts:314-315`
  - body: `clearStreamingProgress()` clears text but leaves `streaming` and `streamingStartPromise` installed. If the session was already marked closed by a failed close path, `clearText()` cannot revive it and `startStreaming()` is blocked by the stale reference, forcing a separate final reply.
  - confidence: 0.85
- **[P2] Serialize error cleanup before retry delivery:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: The new error path awaits `clearStreamingProgress()`, but core invokes `onError` without awaiting it. A recovered final can therefore run before partial updates drain and buffers clear, leaving the stale-merge race in place.
  - confidence: 0.82
