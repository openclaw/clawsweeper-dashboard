## What This Changes

This branch changes Feishu reply dispatching to clear streaming progress instead of closing the card on reply errors, adds clearText-based regression coverage, and bundles Card Kit markdown/reasoning sanitization changes.

## Best Possible Solution

Land a narrowed repaired Feishu fix that preserves one reply surface through transient errors, serializes error cleanup with retry delivery, resets stale session refs safely, keeps stream state raw, and includes redacted Feishu proof.

## Reproduction Assessment

Yes, at source/test-harness level rather than live: current main closes active Feishu streaming on reply errors and the shared dispatcher does not await async error cleanup. The linked issue adds sanitized failover logs showing duplicate visible Feishu replies.

## Solution Assessment

No. Keeping the card open through transient errors is the right direction, but this branch is not mergeable as-is until cleanup is serialized, stale refs are handled, sanitizer state mutation is removed or separated, and real proof is supplied.

  - body: `update()` sanitizes snapshots before `mergeStreamingText()` stores them as canonical state. A later raw snapshot or final, such as an incremental code span, can stop sharing a clean prefix and be concatenated or corrupted; keep state raw and sanitize only outbound Card Kit payloads.
  - confidence: 0.9

  - body: `clearStreamingProgress()` clears text but leaves the current `streaming` object and `streamingStartPromise` installed. If recovery is entered with a closed or stale session, `startStreaming()` remains blocked by the stale reference and retry delivery can fall back to a separate visible reply.
  - confidence: 0.84
- **[P2] Serialize error cleanup before retry delivery:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: The Feishu error path awaits `clearStreamingProgress()`, but the shared dispatcher invokes `onError` without awaiting it. A recovered final can enter delivery before partial updates drain and buffers clear, leaving the stale-merge race in place.
  - confidence: 0.83
