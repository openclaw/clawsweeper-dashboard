## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Reset the closed session before recovered finals:** `extensions/feishu/src/reply-dispatcher.ts:494`
  - body: `clearStreamingProgress()` clears buffers but leaves `streaming` and `streamingStartPromise` intact. If the failed delivery came from `closeStreaming()` after `FeishuStreamingSession.close()` has already set `closed = true`, `clearText()` returns early, the stale closed session still blocks `startStreaming()`, and the recovered final can fall back to a separate visible message instead of reusing the card.
  - confidence: 0.87
- **[P1] Keep raw snapshots out of sanitized merge state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: Sanitizing each partial before merging stores synthetic closing backticks in `currentText`. A later real snapshot with the natural closing backtick no longer has a clean prefix, so `mergeStreamingText` can concatenate and corrupt the card content, for example `Here is `co` followed by `Here is `code``.
  - confidence: 0.84
