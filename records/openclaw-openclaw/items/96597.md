## Summary

Keep open: current main still exposes Google Vertex through the bundled Google provider's static catalog, so the dynamic ADC-backed model discovery request remains unresolved even though manual configured-model overrides are now covered.

## What This Changes

Request dynamic Google Vertex model catalog refresh plus a supported override path for models not yet present in the bundled static catalog.

## Best Possible Solution

Have maintainers settle a provider-owned Google Vertex live catalog contract, then wire it through the existing provider catalog SDK with static fallback and keep manual additions on the existing `models.providers.google-vertex.models` surface unless a specific gap is proven.

## Reproduction Assessment

Yes at source level: current Google Vertex provider registration still exposes only `staticCatalog`, while the generic provider list path only runs live discovery when a provider has `catalog.run` or `discovery.run`. I did not run a live Google Vertex API call in this read-only review.

## Solution Assessment

Unclear. The manual override half already uses `models.providers.google-vertex.models`, but ADC-backed dynamic discovery needs a provider-owned contract for endpoint, auth, cache, timeout, fallback, and whether any new config option is warranted.
