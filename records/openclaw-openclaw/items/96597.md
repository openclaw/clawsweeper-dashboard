## Summary

Keep open: current main still builds `google-vertex` from a static Google catalog, and the requested ADC-backed live catalog behavior is not implemented. The existing `models.providers.google-vertex.models` surface partly covers manual overrides, while the linked override bug remains a narrower open follow-up.

## What This Changes

Request dynamic Google Vertex model catalog refresh via ADC and a manual override path for models missing from the bundled catalog.

## Best Possible Solution

Define a provider-owned Google Vertex live catalog contract using the existing live catalog SDK if maintainers approve it, and keep manual additions on the documented `models.providers.google-vertex.models` path with the linked override bug handled separately.

## Reproduction Assessment

Yes. Source inspection shows `google-vertex` is still produced from static catalog hooks on current main; I did not run a live Google Vertex API repro in this read-only review.

## Solution Assessment

Unclear. Dynamic discovery may be the right long-term feature, but the existing configured-provider override should remain the manual path unless maintainers decide it is insufficient.
