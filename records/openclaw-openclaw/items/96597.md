## Summary

Keep open: current main and v2026.6.10 still register Google Vertex through a static Google catalog, while live ADC-backed Vertex discovery and any extra override surface need maintainer decisions on provider behavior, auth, caching, and failure handling.

## What This Changes

The issue requests dynamic Google Vertex model catalog refresh plus a way to override the Vertex model list through configuration.

## Best Possible Solution

Settle a provider-owned Google Vertex live-catalog contract, reuse the guarded live catalog SDK with static fallback, and either fix or document the existing models.providers.google-vertex.models override path rather than adding duplicate config surface.
