## Summary

Keep open: current main still publishes `google-vertex` through a static Google catalog only, while the existing configured-model override path and the linked override bug do not settle the requested ADC-backed live catalog behavior.

## What This Changes

Request dynamic Google Vertex model discovery plus an `openclaw.json` override path for Google Vertex models not yet in the bundled catalog.

## Best Possible Solution

Define a provider-owned Google Vertex live catalog contract that reuses the guarded live catalog SDK, preserves static fallback, and treats the existing configured-model rows as the manual override unless maintainers decide that surface is insufficient.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 build `google-vertex` from a static catalog and do not expose a live Vertex catalog hook; I did not run a credentialed Google Vertex API call in this read-only review.

## Solution Assessment

Unclear. Dynamic discovery is plausible and VISION-aligned, but maintainers first need to choose ADC endpoint, cache, timeout, and fallback semantics and avoid duplicating the existing configured-model override path.
