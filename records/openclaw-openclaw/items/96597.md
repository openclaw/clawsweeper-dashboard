## Summary

Keep open: current main and v2026.6.10 still publish `google-vertex` from a static Google catalog. Manual additions are already partly covered by `models.providers.google-vertex.models`, but ADC-backed live Vertex discovery still needs maintainer decisions on provider behavior, auth, caching, and failure handling.

## What This Changes

Request live Google Vertex model catalog discovery and clearer use of configured model overrides so new Vertex models can be used without waiting for an OpenClaw release.

## Best Possible Solution

Settle a provider-owned Google Vertex catalog contract, then either document the existing `models.providers.google-vertex.models` override as the supported workaround or wire Vertex into the guarded live catalog SDK with static fallback and focused tests.

## Reproduction Assessment

Yes. Source-level inspection shows `google-vertex` is built from a static catalog hook and no Vertex runtime catalog hook exists; I did not run a live Google Vertex API call in this read-only review.

## Solution Assessment

Unclear. Dynamic discovery may be the right feature, but the existing configured-model override already covers manual additions, and ADC-backed live refresh needs endpoint, auth, cache, and failure-mode policy first.
