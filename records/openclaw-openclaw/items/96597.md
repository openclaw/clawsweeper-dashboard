## Summary

Keep open: current main still registers Google Vertex from the bundled static Gemini catalog, so live ADC-backed Vertex model discovery remains missing. The manual override part is now mostly covered by the existing `models.providers.google-vertex.models` path and recent auth-marker normalization, so the remaining work is a provider/product decision rather than a narrow bug fix.

## What This Changes

Request dynamic Google Vertex model catalog refresh and an `openclaw.json` model-list override path for newer Vertex models.

## Best Possible Solution

Keep the existing config override path, then design a provider-owned Google Vertex live catalog implementation through the guarded catalog SDK with static fallback and focused Vertex tests.

## Reproduction Assessment

Yes. Source inspection shows Google Vertex is still registered through static catalog hooks on current main, while the user-requested live ADC-backed catalog refresh is not implemented.

## Solution Assessment

Unclear as requested. Dynamic discovery may be the right feature, but a new override option is not the best first move because `models.providers.google-vertex.models` already exists and the remaining live-discovery behavior needs provider policy decisions.
