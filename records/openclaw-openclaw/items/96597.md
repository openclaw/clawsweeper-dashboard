## Summary

Keep open: current main and v2026.6.10 still publish `google-vertex` through the static Google catalog. The manual override part is now covered by configured provider rows and the related closed issue, but ADC-backed live Vertex catalog discovery remains a provider/config product decision.

## What This Changes

The issue requests dynamic Google Vertex model catalog discovery plus an `openclaw.json` override path for models missing from OpenClaw's static catalog.

## Best Possible Solution

Settle a provider-owned Google Vertex live catalog contract, then either document the existing configured-model override as the supported workaround or wire Vertex into the guarded live catalog SDK with static fallback and focused tests.

## Reproduction Assessment

Yes. Source inspection shows `google-vertex` is still built from a static Google catalog and no provider runtime discovery hook is registered for live Vertex model fetching.

## Solution Assessment

Unclear. Dynamic discovery may be the right provider feature, but the override portion already uses the existing configured-provider model path and a new override option could duplicate it.
