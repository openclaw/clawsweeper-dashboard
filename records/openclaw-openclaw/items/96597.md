## Summary

Keep open: current main still builds `google-vertex` from the static Google model list, while the requested manual override is already partly covered by documented `models.providers.google-vertex.models`; live Vertex discovery needs maintainer product and security decisions.

## What This Changes

The issue requests live Google Vertex model catalog refresh plus an `openclaw.json` override path for manually adding Vertex models.

## Best Possible Solution

Decide the Vertex catalog contract first, then document the existing `models.providers.google-vertex.models` override as the supported workaround or implement provider-owned live discovery through the guarded catalog SDK.

## Reproduction Assessment

Yes, source-level: `google-vertex` is built from `GOOGLE_GEMINI_TEXT_MODELS` through a static catalog hook, and no Vertex live catalog hook exists. I did not run a live Vertex API call in this read-only review.

## Solution Assessment

Unclear: dynamic discovery may be the right feature, but the existing configured-model catalog already covers manual additions, and ADC-backed live refresh needs endpoint, auth, cache, and failure-mode policy first.
