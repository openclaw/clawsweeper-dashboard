## Summary

Keep open: current main does not implement `after_agent_complete`, and the PR still has blocking API/security and delivery-semantics issues plus missing real behavior proof.

## What This Changes

The PR adds an `after_agent_complete` plugin hook and reply-runner wrapper that can suppress a completed response or reinject context for another agent pass.

## Best Possible Solution

Refit the lifecycle idea onto the current hook contract architecture, using or extending `before_agent_finalize` where revision is enough and adding any separate suppress/reinject hook only with explicit conversation-access gating and settled delivery semantics.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a new plugin lifecycle feature. The blocking PR issues are source-reproducible from the PR head plus current hook-policy and reply-delivery paths.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable form until it is rebased onto current hook architecture, gated for conversation access, and scoped against streaming/pre-delivery behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require conversation access for completion hooks:** `src/plugins/types.ts:1548`
  - body: The new event exposes the assistant `response`, but the hook is only added to prompt-injection policy. Non-bundled plugins could observe conversation content without `hooks.allowConversationAccess=true`; classify this hook with the raw conversation hooks before exposing it.
  - confidence: 0.91
- **[P1] Defer streaming until completion hooks decide:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: This wrapper runs the hook only after `runAgentTurnWithFallback()` returns, but block delivery can enqueue or send chunks during that call. A suppress or reinject verdict cannot retract already-visible output, so delivery must be deferred or the hook must be disabled for streaming modes.
  - confidence: 0.88
- **[P2] Return the active reinject pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:101-103`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still flushes and stops the original pipeline after return. Buffered output from the final reinjected attempt can be missed unless the active pipeline is returned or final flushing is owned inside the wrapper.
  - confidence: 0.84
