## Summary

Keep open: the PR proposes a useful lifecycle hook, but it is not merge-ready because the new response-text hook is not conversation-access gated, suppress/reinject runs after delivery and session side effects can already occur, and no real behavior proof is attached.

## What This Changes

The branch adds an `after_agent_complete` plugin hook and wraps reply-agent execution so plugins can suppress a completed response or rerun the agent with injected context before final payload handling.

## Best Possible Solution

Define one supported final-answer lifecycle contract: use `before_agent_finalize` where bounded revision is enough, or rebase a true pre-delivery completion API with conversation-access gating, explicit delivery/transcript semantics, docs, tests, and real proof.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new plugin lifecycle feature. The blocking risks are source-reproducible by tracing the wrapper after `runAgentTurnWithFallback` and comparing the response-text event to current conversation-access policy.

## Solution Assessment

No. The direction may be useful for Speedtrap, but this branch is not the best mergeable shape until the lifecycle boundary, security gate, delivery behavior, transcript semantics, and real proof are settled.

- **[P1] Gate completion hooks as conversation hooks:** `src/plugins/types.ts:1548-1549`
  - body: The new event exposes raw assistant `response` text, but the diff only classifies `after_agent_complete` as prompt-injection gated.   - confidence: 0.91

  - body: `runAgentTurnWithHooks` waits for `runAgentTurnWithFallback()` to finish before invoking the hook. In block-streaming setups, the inner run can already enqueue or send chunks, so later `suppress` or `reinject` cannot honor the advertised pre-delivery contract unless delivery is deferred or those modes are excluded for streamed/block-delivered output.
  - confidence: 0.88
- **[P1] Define transcript handling for suppressed drafts:** `src/auto-reply/reply/agent-runner-hooks.ts:86-87`
  - body: Returning an empty final payload after `runAgentTurnWithFallback()` suppresses delivery, but the completed assistant turn has already gone through the session path. The next turn can be conditioned on a response the user never saw unless the wrapper restores pre-hook state or the API explicitly documents transcript-preserving suppression.
  - confidence: 0.82
- **[P2] Return or flush the active block pipeline:** `src/auto-reply/reply/agent-runner-hooks.ts:101-103`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still owns and flushes the original pipeline after the wrapper returns. Buffered output from the final reinjected attempt can be missed or left unawaited unless the active pipeline is returned or flushed inside the wrapper.
  - confidence: 0.84
