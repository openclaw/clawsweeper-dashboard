## Summary

Keep open. Current main does not implement `after_agent_complete`; this PR is a new plugin lifecycle API and the branch still has blocking security and delivery-semantics issues plus missing real behavior proof.

## What This Changes

Adds an `after_agent_complete` plugin hook, hook runner, and reply-agent wrapper so plugins can inspect a completed response, suppress delivery, or reinject context.

## Best Possible Solution

Land a rebased API only if it uses current hook contract files, enforces conversation access, defines delivery/streaming/transcript semantics, and includes docs, tests, and real proof; use `before_agent_finalize` when final-answer revision is sufficient.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new plugin lifecycle feature. The blocking review findings are source-reproducible from the PR diff and current hook-policy and reply-delivery code paths.

## Solution Assessment

No. The direction may support speedtrap, but this branch is not the best mergeable form until it is rebased onto current hook contracts and the security, delivery, streaming, and transcript semantics are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate completion hooks as conversation hooks:** `src/plugins/types.ts:1537-1548`
  - body: The new event exposes raw assistant `response` text, but the hook is only added to prompt-injection policy. Current main blocks conversation-content hooks for non-bundled plugins unless `allowConversationAccess=true`, so this lets external plugins read assistant output without that explicit opt-in.
  - confidence: 0.91
- **[P1] Defer delivery before suppress or reinject:** `src/auto-reply/reply/agent-runner-hooks.ts:42-50`
  - body: `runAgentTurnWithHooks` calls `runAgentTurnWithFallback()` before invoking the completion hook, but fallback can enqueue or send block replies during the run. A later `suppress` or `reinject` result cannot honor the advertised pre-delivery contract unless delivery is deferred or those modes are excluded for streamed/block-delivered output.
  - confidence: 0.88
- **[P2] Return the active block pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:99-102`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still owns the original pipeline after the wrapper returns. Buffered output from the final reinjected attempt can be missed or left unawaited unless the active pipeline is returned or flushed inside the wrapper.
  - confidence: 0.84
