## Summary

Keep open: current main does not implement this new hook, but this PR is not mergeable as-is because it is stale/conflicting, lacks real behavior proof, and the proposed response-observing suppress/reinject API still has security and delivery-semantics blockers.

## What This Changes

The branch adds an `after_agent_complete` plugin lifecycle hook plus a reply-runner wrapper that can suppress a completed response or reinject context for another agent pass.

## Best Possible Solution

Rebase the lifecycle idea onto the current hook contract architecture, use or extend `before_agent_finalize` where final-answer revision is enough, and add any separate suppress/reinject hook only with conversation-access gating plus explicit streaming and transcript semantics.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a new plugin lifecycle feature. The blocking PR defects are source-reproducible from the PR head and current hook-policy/reply-delivery paths.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable form until it is rebased onto current hook architecture, gated for conversation access, and scoped against streaming/pre-delivery behavior.

  - body: The new event exposes the assistant `response`, but the hook is only added to prompt-injection policy. On current main, raw conversation hooks are protected by the conversation-access list, so this response-observing hook needs the same gate before non-bundled plugins can register it.
  - confidence: 0.91
- **[P1] Defer streaming until completion hooks decide:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: This wrapper calls `runAgentTurnWithFallback()` before `runAfterAgentComplete()`, but current reply delivery can enqueue or flush block replies during that run. A suppress or reinject verdict cannot honor the pre-delivery contract unless delivery is deferred or the hook is disabled for streaming paths.
  - confidence: 0.88
- **[P2] Return the active reinject pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:101-103`
  - body: On reinject, this replaces only the wrapper's local `blockReplyPipeline`; `runReplyAgent` still flushes and stops the original pipeline after the wrapper returns. Buffered output from the final reinjected attempt can be missed unless the active pipeline is returned or final flushing is owned inside the wrapper.
  - confidence: 0.84
