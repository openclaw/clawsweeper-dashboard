## Summary

Keep this PR open for maintainer review: the hook is a plausible plugin API addition, but the branch is now stale against current hook-contract code and still has blocking correctness/security gaps plus missing real behavior proof.

## What This Changes

The PR adds an `after_agent_complete` plugin hook that can inspect a draft agent response before delivery and either suppress delivery or reinject context for another agent run.

## Best Possible Solution

Design this against current main’s plugin lifecycle API, either by extending the existing finalize hook or by adding a documented, access-gated post-response hook with focused tests and real suppress/reinject proof.

## Reproduction Assessment

Yes for the review findings from source inspection: the PR can replace the block pipeline without returning it to the caller, and the new event exposes draft response text without current-main conversation-access gating. I did not establish live plugin behavior proof.

## Solution Assessment

No. The direction may be useful, but the current patch should not merge until it is rebased onto current hook contracts, privacy-gated, and made safe for block-streaming or explicitly excluded from it.
