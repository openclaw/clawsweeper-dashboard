## Summary

Keep open: this PR proposes a new core plugin lifecycle surface that is not on current main, and the branch still has blocking security, delivery, and proof gaps rather than being cleanup-closeable.

## What This Changes

The PR adds an `after_agent_complete` hook, hook runner, and reply-agent wrapper so plugins can inspect response text, suppress delivery, or reinject context.

## Best Possible Solution

Land a rebased API only if it uses current hook contract files, enforces conversation access, defines delivery, streaming, and transcript semantics, and includes focused tests, docs, and real proof; use `before_agent_finalize` where one extra final-answer pass is enough.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new plugin lifecycle feature. The blocking findings are source-reproducible by comparing the PR diff with current hook policy and reply-delivery code.

## Solution Assessment

No. The direction may be useful for Speedtrap, but this branch is not the best mergeable form until the security gate, delivery timing, pipeline ownership, transcript semantics, and proof requirements are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate completion hooks as conversation hooks:** `src/plugins/types.ts:1537-1548`
  - body: The new event exposes raw assistant `response` text, but the hook is only added to prompt-injection policy. Current main blocks conversation-content hooks for non-bundled plugins unless `allowConversationAccess=true`, so external plugins could read assistant output without that explicit opt-in.
  - confidence: 0.91
- **[P1] Defer delivery before suppress or reinject:** `src/auto-reply/reply/agent-runner-hooks.ts:44-50`
  - body: `runAgentTurnWithHooks` invokes `runAgentTurnWithFallback()` before the completion hook, but fallback can enqueue or send block replies during the run. A later `suppress` or `reinject` result cannot reliably honor the advertised pre-delivery contract unless delivery is deferred or those modes are excluded for streamed/block-delivered output.
  - confidence: 0.88
- **[P2] Return the active block pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:99-102`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still owns and flushes the original pipeline after the wrapper returns. Buffered output from the final reinjected attempt can be missed or left unawaited unless the active pipeline is returned or flushed inside the wrapper.
  - confidence: 0.84
