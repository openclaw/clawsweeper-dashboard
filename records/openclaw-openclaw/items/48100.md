## Summary

## What This Changes

The PR adds an `after_agent_complete` lifecycle hook and wraps reply-agent execution so plugins can suppress a completed response or append reinjection context before reply delivery.

## Best Possible Solution

Rebase onto the current hook-types architecture, reuse `before_agent_finalize` where final-answer revision is sufficient, and add any separate suppress/reinject completion hook only with explicit conversation-access, streaming, delivery, transcript, tests, and runtime-proof semantics.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new plugin lifecycle feature. The blocking review findings are source-reproducible by comparing the PR head wrapper with current main’s hook-policy and reply-delivery paths.

## Solution Assessment

No. The direction may support speedtrap, but this branch is not the best mergeable form until it is rebased onto current hook contracts and the policy, streaming, pipeline, and transcript semantics are made explicit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require conversation access for completion hooks:** `src/plugins/types.ts:1548`
  - body: The new event exposes assistant `response` text, but the hook is only added to prompt-injection policy. Non-bundled plugins without `allowConversationAccess=true` could observe raw assistant output by registering this hook; classify it with conversation-access gated hooks when rebasing.
  - confidence: 0.91
- **[P1] Defer delivery until the completion hook decides:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: `runAgentTurnWithHooks` calls `runAgentTurnWithFallback()` before `runAfterAgentComplete()`, but current reply delivery can enqueue or send block replies during that call. A later `suppress` or `reinject` result cannot honor the claimed pre-delivery contract unless delivery is deferred or the hook contract excludes those modes.
  - confidence: 0.88
- **[P2] Return the active reinject pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:101-103`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still owns the original pipeline after the wrapper returns. Buffered output from the final reinjected attempt can be missed unless the active pipeline is returned or final flushing is owned inside the wrapper.
  - confidence: 0.84
