## Summary

Keep this PR open for maintainer review rather than cleanup-close: current main has a related `before_agent_finalize` hook, but it does not cover this PR's suppress/delivery-gate semantics, and the branch still has privacy, streaming, and pipeline-ownership blockers.

## What This Changes

The branch adds an `after_agent_complete` plugin hook, hook runner wiring, and an agent-runner wrapper that can suppress a completed response or reinject context for another agent pass.

## Best Possible Solution

Refit the idea onto the current hook architecture or a deliberately scoped delivery-gate hook, with conversation-access gating, explicit streaming behavior, and defined transcript semantics for suppressed or revised drafts.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a new plugin lifecycle feature. The blocking PR issues are source-reproducible from the proposed wrapper, current hook policy, and reply delivery paths.

## Solution Assessment

No. The hook direction is plausible, but this PR is not the best mergeable form until privacy gating, streaming/pre-delivery behavior, and current hook architecture integration are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require conversation access for completion hooks:** `src/plugins/types.ts:1548`
  - body: The new event exposes the full assistant `response`, but the hook is not classified with raw conversation hooks. Non-bundled plugins could observe conversation content without `hooks.allowConversationAccess=true`; add this hook to the conversation-access policy when refitting it onto current `hook-types.ts`.
  - confidence: 0.91
- **[P1] Prevent streaming output before hook verdicts:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: This wrapper waits for `runAgentTurnWithFallback()` to finish before calling the hook, but that path can already enqueue or flush block replies. A later `suppress` or `reinject` result cannot retract content that reached the channel, so delivery must be deferred or streaming modes must be explicitly excluded.
  - confidence: 0.88
- **[P2] Return the active reinject pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:101-103`
  - body: On reinject this code replaces only the wrapper's local `blockReplyPipeline`; after the wrapper returns, `runReplyAgent` still flushes and stops the original pipeline. Buffered output from the final reinjected attempt can be missed unless the active pipeline is handed back or final flushing is owned inside the wrapper.
  - confidence: 0.84
