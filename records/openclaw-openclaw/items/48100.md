## Summary

Keep open: current main does not implement this hook, and the PR still needs security gating, streaming-semantics fixes, and real runtime proof before it is mergeable.

## What This Changes

The PR adds an `after_agent_complete` plugin lifecycle hook plus an agent-runner wrapper that can suppress a draft reply or reinject context for another agent pass.

## Best Possible Solution

Land a current-main refit that adds the hook through the current hook contracts, gates it as both prompt-injection and conversation-access, and either defers streaming delivery until the hook verdict or explicitly limits the hook to non-streaming modes.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a new plugin lifecycle feature. The blocking PR issues are source-reproducible from the wrapper running after `runAgentTurnWithFallback` and from the missing conversation-access classification.

## Solution Assessment

No. The hook direction is plausible, but this PR is not the best mergeable form until privacy gating and streaming/pre-delivery semantics are handled in the current hook architecture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require conversation access for completion hooks:** `src/plugins/types.ts:1365`
  - body: The new event exposes the full assistant `response`, but the hook is not added to the conversation-access hook classification. Non-bundled plugins without `hooks.allowConversationAccess=true` could observe model output through this handler, unlike `llm_output` or `agent_end`; classify this hook with the conversation hooks when registering it.
  - confidence: 0.9
- **[P1] Prevent streaming output before hook verdicts:** `src/auto-reply/reply/agent-runner-hooks.ts:48-50`
  - body: The wrapper waits for `runAgentTurnWithFallback()` to finish before calling the hook, but that path can already enqueue or flush block replies while the run is in progress. A later `suppress` or `reinject` result cannot retract chunks that reached the channel, so this needs a deferred delivery path or an explicit streaming guard.
  - confidence: 0.88
- **[P2] Return the active reinject pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:102-103`
  - body: On reinject this code replaces only the wrapper's local `blockReplyPipeline`; after the wrapper returns, `runReplyAgent` still flushes and stops the original pipeline. Buffered output from the final reinjected attempt can be missed unless the active pipeline is handed back or pipeline flushing stays inside the wrapper.
  - confidence: 0.84
