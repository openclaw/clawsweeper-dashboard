## Summary

Keep open: current main still lacks the requested hook, and this branch is not a safe merge because it is stale against the current hook contract and leaves policy, streaming, and transcript semantics unresolved.

## What This Changes

This PR adds an `after_agent_complete` plugin hook that can suppress a completed agent response or reinject context for another agent pass, and wires it into `runReplyAgent`.

## Best Possible Solution

Rework the capability as a current-main plugin API change in `hook-types.ts`, enforce both prompt-mutation and conversation-content policy, explicitly define streaming and transcript behavior, then add focused hook-runner, reply-runner, docs, and policy tests.

## Reproduction Assessment

Not applicable as an end-user bug, but the review issues are source-reproducible with high confidence. Current main lacks the hook, and source inspection shows the proposed post-fallback hook would run after streaming delivery and session mutation can already occur.

## Solution Assessment

No. The direction addresses a real plugin hook gap, but this implementation is not the best merge shape because it targets stale hook files, exposes raw assistant output without the current conversation-access gate, and does not settle streaming or transcript semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move hook definitions to hook-types.ts:** `src/plugins/types.ts:1365-1393`
  - body: Current main no longer owns `PluginHookName`, `PLUGIN_HOOK_NAMES`, or `PluginHookHandlerMap` in `src/plugins/types.ts`; those contracts live in `src/plugins/hook-types.ts`. Adding the new hook here leaves the typed hook catalog and handler map stale after rebase, so the hook will not type-check or register correctly.
  - confidence: 0.92
- **[P1] Gate raw response access with conversation policy:** `src/plugins/types.ts:1537-1547`
  - body: The new event passes the assistant `response` to plugin handlers, but the PR only classifies the hook as prompt-mutating. Non-bundled plugins that receive conversation content are currently gated by `allowConversationAccess`, so this hook needs the same classification, docs, and tests before exposing raw assistant output.
  - confidence: 0.88
- **[P1] Defer streaming until the hook verdict:** `src/auto-reply/reply/agent-runner-hooks.ts:49-60`
  - body: `runAgentTurnWithFallback` can emit block replies while the model run is still in progress, so invoking `runAfterAgentComplete` only after it returns is too late for `{ suppress: true }` or `{ reinject: true }` on streaming paths. Buffer or disable streaming until the hook decides, or make the hook explicitly unavailable for those modes.
  - confidence: 0.86
