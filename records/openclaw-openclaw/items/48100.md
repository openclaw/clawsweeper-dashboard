## Summary

Keep open: current main does not implement `after_agent_complete`, and this PR is still not merge-ready because it adds a new core plugin API without aligning to the current hook catalog, conversation-access policy, and reply delivery semantics.

## What This Changes

This PR adds an `after_agent_complete` plugin lifecycle hook that can suppress a completed agent response or reinject context for another agent pass before normal reply delivery.

## Best Possible Solution

Rework the capability against current main as a documented hook API in `hook-types.ts`, gate it under both prompt-mutation and conversation-content policy, and explicitly define streaming plus transcript behavior with focused tests and docs.

## Reproduction Assessment

Not applicable as an end-user bug. The review blockers are source-reproducible from the PR diff and current main: the hook is absent, the active hook contracts live in `hook-types.ts`, and streaming delivery can occur before the proposed post-run hook verdict.

## Solution Assessment

No. The hook gap is real, but this implementation is not the best merge shape because it targets stale hook files and does not fully settle conversation-access, streaming, or transcript semantics.

  - body: Current main no longer owns `PluginHookName`, `PLUGIN_HOOK_NAMES`, or `PluginHookHandlerMap` in `src/plugins/types.ts`; those contracts live in `src/plugins/hook-types.ts`.   - confidence: 0.92
- **[P1] Gate raw response access with conversation policy:** `src/plugins/types.ts:1537-1547`
  - body: The new event passes the assistant `response` to plugin handlers, but the PR only classifies the hook as prompt-mutating. Non-bundled plugins that receive conversation content are currently gated by `allowConversationAccess`, so this hook needs the same classification, docs, and tests before exposing raw assistant output.
  - confidence: 0.88
- **[P1] Defer streaming until the hook verdict:** `src/auto-reply/reply/agent-runner-hooks.ts:50-60`
  - body: `runAgentTurnWithFallback` can emit block replies while the model run is in progress, so invoking `runAfterAgentComplete` only after it returns is too late for `{ suppress: true }` or `{ reinject: true }` on streaming paths. Buffer or disable streaming until the hook decides, or make the hook contract explicitly unavailable for those modes.
  - confidence: 0.86
