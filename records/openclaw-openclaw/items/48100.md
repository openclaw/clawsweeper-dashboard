## Summary

Keep open: current main has the adjacent `before_agent_finalize` hook, but it does not implement this PR's suppress/reinject-after-agent-complete API, and the branch still has merge-blocking security, delivery, session-state, and proof gaps.

## What This Changes

The PR adds an `after_agent_complete` plugin hook and reply-agent wrapper so plugins can inspect assistant output, suppress delivery, or reinject context for another agent pass.

## Best Possible Solution

Rebase the design onto current hook contract files, enforce conversation access, define delivery/streaming/transcript semantics, add focused tests and docs, and use `before_agent_finalize` where one extra final-answer pass is sufficient.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new plugin lifecycle feature. The blocking problems are source-reproducible from the PR wrapper order plus current hook policy, delivery, and transcript code.

## Solution Assessment

No. The direction may be useful for Speedtrap, but this branch is not the best mergeable form until the security gate, delivery timing, pipeline ownership, transcript semantics, and proof requirements are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate completion hooks as conversation hooks:** `src/plugins/types.ts:1537-1548`
  - body: The new event exposes raw assistant `response` text, but the PR only classifies the hook as prompt-injection gated. Current main blocks conversation-content hooks for non-bundled plugins unless `allowConversationAccess=true`, so external plugins could read assistant output without that explicit opt-in.
  - confidence: 0.92
- **[P1] Defer delivery before suppress or reinject:** `src/auto-reply/reply/agent-runner-hooks.ts:44-50`
  - body: `runAgentTurnWithHooks` calls `runAgentTurnWithFallback()` before the completion hook, but fallback can already enqueue or direct-send block replies during the run. A later `suppress` or `reinject` result cannot honor the advertised pre-delivery contract unless delivery is deferred or those modes are excluded for delivered output.
  - confidence: 0.9
- **[P1] Define transcript state before rerunning:** `src/auto-reply/reply/agent-runner-hooks.ts:44-50`
  - body: The embedded run has already appended assistant output and fed `messagesSnapshot` to context-engine post-turn work before this hook runs. Suppressing leaves an unseen assistant turn in history, and reinjecting makes the next attempt condition on that stale draft unless the hook explicitly snapshots/restores or documents and tests those semantics.
  - confidence: 0.86
- **[P2] Return the active block pipeline:** `src/auto-reply/reply/agent-runner-hooks.ts:99-102`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still owns and flushes the original pipeline after the wrapper returns. Buffered output from the final reinjected attempt can be missed or left unawaited unless the active pipeline is returned or flushed inside the wrapper.
  - confidence: 0.84
