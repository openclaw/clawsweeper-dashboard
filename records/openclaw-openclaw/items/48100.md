## Summary

Keep open; this is a new plugin API and the current branch still needs maintainer design/security review because the raw response hook is not gated as conversation access, suppress/reinject can run after streaming delivery has begun, and no real behavior proof is posted.

## What This Changes

The PR adds an `after_agent_complete` plugin hook with suppress/reinject handling around reply-agent completion and wires it into plugin hook types and the reply-agent runner.

## Best Possible Solution

Settle the hook API against current `before_agent_finalize` and dispatch hooks, then land a gated, documented, streaming-safe contract with focused tests and real gateway proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff and current delivery path shows the hook sees raw response content without the conversation-access gate and runs after streamable reply payloads can already be queued.

## Solution Assessment

No. The current patch is not the narrowest maintainable solution until maintainers decide whether existing finalization/dispatch hooks cover the use case or a separate streaming-safe completion hook is needed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate raw response hooks with conversation access:** `src/plugins/types.ts:1545-1546`
  - body: The event exposes `response` to plugin handlers, but the diff only adds the hook to the prompt-injection list and never classifies it as a conversation-content hook. Non-bundled plugins could read raw replies without `allowConversationAccess`, unlike `llm_output`, `before_agent_finalize`, and `agent_end`.
  - confidence: 0.88
- **[P1] Run the hook before any streamable reply can leave:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: The wrapper waits for `runAgentTurnWithFallback()` before asking hooks whether to suppress or reinject. That inner run can enqueue block replies through the existing delivery handler, so the decision can arrive after stale output is already queued or sent.
  - confidence: 0.86
