## Summary

Keep open: this is a new core plugin lifecycle API that is not already implemented on current main, and the branch still has merge-blocking API/security, delivery-timing, pipeline-ownership, and proof gaps.

## What This Changes

The PR adds an `after_agent_complete` typed plugin hook and reply-agent wrapper so plugins can inspect response text, suppress delivery, or reinject context after an agent run.

## Best Possible Solution

Define the hook against current `hook-types.ts`/`hooks.ts`, enforce conversation-access policy, make streaming/delivery/transcript semantics explicit, add focused tests and docs, and use `before_agent_finalize` where one bounded final-answer revision pass is enough.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new lifecycle feature. The merge blockers are source-reproducible from the PR diff and current main hook policy, delivery, and pipeline ownership code.

## Solution Assessment

No. The direction may be useful, but this branch is not the best mergeable form until maintainers settle the API/security contract and the implementation proves delivery, streaming, transcript, and upgrade behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate completion hooks as conversation hooks:** `src/plugins/types.ts:1540-1549`
  - body: The new event exposes raw assistant `response` text, but the hook is only classified as prompt-injection gated. Non-bundled plugins could register it without `hooks.allowConversationAccess=true`, bypassing the current raw-conversation-content policy used for `llm_output`, `before_agent_finalize`, and `agent_end`.
  - confidence: 0.92
- **[P1] Defer delivery before suppress or reinject:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: `runAgentTurnWithHooks` calls `runAgentTurnWithFallback()` before asking the completion hook, but the wrapped run can already enqueue or send block replies. A later `{ suppress: true }` or `{ reinject: true }` cannot reliably satisfy the advertised pre-delivery contract for streamed or flushed block output.
  - confidence: 0.89
- **[P2] Return the active block pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:100-103`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still flushes and stops the original pipeline after the wrapper returns. Buffered output from the final reinjected attempt can be missed, unawaited, or delivered out of order unless the active pipeline is owned by or returned to the caller.
  - confidence: 0.85
