## Summary

Keep this PR open. Current main still lacks `after_agent_complete` or an equivalent post-agent suppress/reinject hook, but this branch should not merge as-is because it is stale against the current hook-contract split and still has security and streaming contract issues.

## What This Changes

The PR adds an `after_agent_complete` plugin hook that runs after an agent response and can suppress delivery or reinject extra context for another agent pass.

## Best Possible Solution

Keep the capability under maintainer review as a core plugin API change, but rework it as a current-main patch: define the contract in `hook-types.ts`, gate both prompt mutation and conversation access, settle streaming and transcript semantics, and add focused tests for no-hook fast path, suppress, reinject cap, streaming behavior, and session state.

## Reproduction Assessment

Not applicable as an end-user bug report, but there is a high-confidence static review path. Current-main searches and source inspection show the hook is absent, and the PR diff plus reply-delivery code show the merge and streaming/security issues without needing a live run.

## Solution Assessment

No. The proposed direction addresses a real hook gap, but this implementation is not the best merge shape because it targets stale hook files, exposes raw response content without the current conversation-access policy, and invokes the hook after streaming delivery may already have occurred.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move the hook contract into hook-types.ts:** `src/plugins/types.ts:1365-1393`
  - body: Current main no longer owns `PluginHookName`, `PLUGIN_HOOK_NAMES`, or `PluginHookHandlerMap` in `src/plugins/types.ts`; those contracts live in `src/plugins/hook-types.ts` and are imported by the registry and hook runner. As written, the branch is stale and the new `after_agent_complete` calls will not type-check or register correctly after rebase.
  - confidence: 0.92
- **[P1] Gate raw response access with conversation policy:** `src/plugins/types.ts:1537-1547`
  - body: The new event passes the assistant `response` to plugin handlers, but the PR only classifies the hook as prompt-mutating. Non-bundled plugins that receive conversation content are currently gated by `allowConversationAccess`; this hook needs the same classification, docs, and tests before exposing raw assistant output.
  - confidence: 0.88
- **[P1] Defer streaming until the hook verdict:** `src/auto-reply/reply/agent-runner-hooks.ts:45-60`
  - body: `runAgentTurnWithFallback` can call `onBlockReply` while the model run is still in progress, so invoking `runAfterAgentComplete` only after it returns is too late for `{ suppress: true }` or `{ reinject: true }` on streaming paths. Buffer or disable streaming until the hook decides, or make the hook explicitly unavailable for those modes.
  - confidence: 0.86
