## Summary

Keep open: current main does not implement the proposed `after_agent_complete` suppress/reinject hook, but the branch is not mergeable as-is because it has security, delivery-ordering, and pipeline-ownership defects and still lacks real behavior proof.

## What This Changes

The PR adds an `after_agent_complete` plugin lifecycle hook and reply-agent wrapper so plugins can inspect completed agent output, suppress delivery, or reinject context.

## Best Possible Solution

Land a rebased API only if it uses the current hook contract files, enforces conversation access, defines delivery, streaming, and transcript semantics, and includes focused tests, docs, and real proof; use `before_agent_finalize` where one extra final-answer pass is enough.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new plugin lifecycle feature. The blocking patch defects are source-reproducible by comparing the PR diff with current hook policy and reply-delivery code.

## Solution Assessment

No. The direction may be useful for Speedtrap, but this branch is not the best mergeable form until the security gate, delivery timing, pipeline ownership, transcript semantics, and proof requirements are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate completion hooks as conversation hooks:** `src/plugins/types.ts:1537-1548`
  - body: The new event exposes raw assistant `response` text, but the hook is only added to prompt-injection policy. Current main blocks raw conversation hooks for non-bundled plugins unless `allowConversationAccess=true`, so external plugins could read assistant output without that explicit opt-in.
  - confidence: 0.91
- **[P1] Defer delivery before suppressing or reinjecting:** `src/auto-reply/reply/agent-runner-hooks.ts:44-50`
  - body: `runAgentTurnWithHooks` invokes `runAgentTurnWithFallback()` before the completion hook runs. That path can stream partial replies or enqueue/direct-send block replies during execution, so a later `suppress` or `reinject` result cannot satisfy the advertised pre-delivery contract.
  - confidence: 0.88
- **[P2] Return the active block pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:99-102`
  - body: On reinject, the wrapper replaces only its local `blockReplyPipeline`. `runReplyAgent` still flushes/stops the original pipeline after the wrapper returns, so buffered output from the final reinjected attempt can be skipped or left unawaited.
  - confidence: 0.84
