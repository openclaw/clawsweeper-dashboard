## Summary

Keep open: the feature is plausible, but this PR is not mergeable as-is because it exposes raw agent responses without the conversation-access gate, runs suppress/reinject logic after streamable replies can already be queued, lacks after-fix real behavior proof, and overlaps the same author's open follow-up at https://github.com/openclaw/openclaw/pull/49870.

## What This Changes

The PR adds an `after_agent_complete` plugin hook and an agent-runner wrapper so plugins can suppress a completed response or reinject context for another agent pass.

## Best Possible Solution

Rebase onto the current hook SDK, decide the public API/privacy contract, gate raw response access with existing conversation-access policy, and either make completion hooks streaming-safe or explicitly exclude incompatible streaming modes with tests and docs.

## Reproduction Assessment

Yes. This is a feature PR, but the review blockers are source-reproducible from the PR head and current main: the hook exposes raw response text without the current conversation-access classification and fires after streamable delivery can be queued.

## Solution Assessment

No. The current PR direction is useful but not yet the best maintainable shape; it needs a current-main rebase, explicit privacy gating, and a settled streaming/pre-delivery contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate raw response hooks as conversation access:** `src/plugins/types.ts:1548-1549`
  - body: The new event exposes the full agent `response`, but the patch only classifies `after_agent_complete` as prompt-injection capable. Current main blocks raw conversation hooks unless non-bundled plugins opt into `allowConversationAccess`, so this hook should be added to that conversation-access category, with matching docs/tests, before exposing replies to plugins.
  - confidence: 0.9
- **[P1] Decide before streamable replies can be sent:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: The wrapper awaits `runAgentTurnWithFallback()` before asking hooks whether to suppress or reinject. That inner run can enqueue or send block-streamed payloads, so the hook can run after user-visible output has already started, violating the advertised pre-delivery suppress/reinject contract.
  - confidence: 0.86
