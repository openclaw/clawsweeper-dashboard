## Summary

Keep open for maintainer review: the hook is not on current main and may be useful, but this branch is conflicted/outdated and still has conversation-access, delivery-semantics, and real-proof blockers.

## What This Changes

The branch adds an `after_agent_complete` plugin lifecycle hook plus an agent-runner wrapper that can suppress a completed reply or reinject context for another agent pass.

## Best Possible Solution

Refit the idea onto current hook contracts, use or extend `before_agent_finalize` where it covers revision, and add any separate suppress/reinject hook only with explicit conversation-access gating and settled delivery semantics.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a new plugin lifecycle feature. The blocking PR issues are source-reproducible from the diff plus current hook-policy and reply-delivery paths.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable form until it is rebased onto current hook architecture, gated for conversation access, and scoped against streaming/pre-delivery behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require conversation access for completion hooks:** `src/plugins/types.ts:1548`
  - body: The new event exposes the assistant `response`, but the hook is not classified with current raw-conversation hooks. Non-bundled plugins could observe conversation content without `hooks.allowConversationAccess=true`; add this hook to the conversation-access classification when refitting it onto current main.
  - confidence: 0.91
- **[P1] Prevent streaming output before hook verdicts:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: This wrapper calls `runAfterAgentComplete` only after `runAgentTurnWithFallback()` returns, but current block delivery can enqueue or send chunks inside that call. A suppress or reinject verdict cannot retract already-visible content, so delivery must be deferred or the hook must be explicitly disabled for streaming modes.
  - confidence: 0.88
- **[P2] Return the active reinject pipeline to the caller:** `src/auto-reply/reply/agent-runner-hooks.ts:101-103`
  - body: On reinject the wrapper replaces only its local `blockReplyPipeline`; `runReplyAgent` still flushes and stops the original pipeline after return. Buffered output from the final reinjected attempt can be missed unless the active pipeline is returned or final flushing is owned inside the wrapper.
  - confidence: 0.84
