## Summary

Keep open: live official ClawHub metadata for `@openclaw/codex` still has no stable installable version, while current main docs and package metadata still advertise ClawHub as a Codex distribution route.

## What This Changes

The issue reports that `openclaw plugins install clawhub:@openclaw/codex` fails on v2026.5.2 even though docs advertise ClawHub for the Codex plugin.

## Best Possible Solution

Make `@openclaw/codex` stable-installable from ClawHub with latest/artifact metadata matching the documented route, or change the generated Codex docs and inventory to npm-only until ClawHub is ready.

## Reproduction Assessment

Yes, via source plus live metadata rather than an installed CLI run. Current source requires a latest ClawHub version for versionless specs, and live `@openclaw/codex` still has no stable latest or artifact for the reported release.

## Solution Assessment

No patch is proposed here. The npm workaround is valid for users, but the maintainable fix is to publish correct ClawHub metadata/artifacts or make the docs/inventory npm-only until that distribution path works.
