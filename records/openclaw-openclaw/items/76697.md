## Summary

Keep open: the reported failure is still valid against current source and live ClawHub state. Versionless ClawHub installs require a stable latest version, but the official Codex package still has no stable latest/artifact while current generated docs still advertise ClawHub as a distribution route.

## What This Changes

The issue requests that the documented Codex ClawHub install path work for OpenClaw 2026.5.2, matching the npm package route that already installs `@openclaw/codex`.

## Best Possible Solution

Publish a stable ClawHub ClawPack artifact/tag for `@openclaw/codex` that matches the supported release, or deliberately change Codex metadata/docs to npm-only until that route is ready.
