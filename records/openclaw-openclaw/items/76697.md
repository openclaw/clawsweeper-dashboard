## Summary

Keep open: current main still advertises ClawHub as a Codex plugin distribution route, while the live official ClawHub package has no stable latest version or installable artifact.

## What This Changes

The issue reports that the documented `clawhub:@openclaw/codex` install path fails on OpenClaw 2026.5.2 even though direct npm installation works.

## Best Possible Solution

Make a stable `@openclaw/codex` artifact installable from ClawHub, or change the generated Codex inventory/docs to npm-only until ClawHub readiness passes.

## Reproduction Assessment

Yes. Source inspection plus live ClawHub metadata gives a high-confidence reproduction path: versionless `clawhub:@openclaw/codex` needs a stable latest version, and the live package has none.

## Solution Assessment

No patch is proposed here. The npm workaround is valid for users, but the maintainable fix is to align the official ClawHub package state with the documented route or remove that route from docs until it works.
