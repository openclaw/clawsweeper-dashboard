## Summary

Keep open: current main still advertises Codex as ClawHub-distributed and the live official ClawHub package still has no stable installable version, so the reported 2026.5.2 install failure remains valid and needs maintainer release/registry follow-up.

## What This Changes

The issue reports that `openclaw plugins install clawhub:@openclaw/codex` fails on OpenClaw 2026.5.2 even though Codex is advertised as available through npm and ClawHub.

## Best Possible Solution

Make the official `@openclaw/codex` stable package installable from ClawHub, or make the Codex docs and generated inventory npm-only until the stable ClawHub artifact is ready.

## Reproduction Assessment

Yes. Source inspection plus live ClawHub metadata reproduces the failure path without writing to the checkout: versionless `clawhub:@openclaw/codex` needs a latest version or tag, and the live official package has neither.

## Solution Assessment

No patch is proposed here. The npm workaround is valid for users, but the maintainable fix is to align the official ClawHub package metadata/artifact with the documented route or remove that route from docs until it works.
