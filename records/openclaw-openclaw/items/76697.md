## Summary

Keep open: the live official ClawHub package for Codex still has no stable installable version, while current docs/package metadata still advertise ClawHub as a Codex distribution route.

## What This Changes

The issue reports that `openclaw plugins install clawhub:@openclaw/codex` fails on OpenClaw 2026.5.2 even though direct npm installation works.

## Best Possible Solution

Make `@openclaw/codex@2026.5.2` installable from ClawHub, or change the Codex docs/inventory to npm-only until a stable ClawHub artifact is actually ready.

## Reproduction Assessment

Yes. Without mutating the checkout, source inspection plus live ClawHub metadata shows the exact failure path: versionless ClawHub installs require a latest/tagged version, and live `@openclaw/codex` has no stable latest version.

## Solution Assessment

No patch is proposed here. The npm workaround is valid for users, but the maintainable fix is to bring the official ClawHub package metadata/artifact in line with the documented route or remove that route from docs until it works.
