## Summary

Keep open: the PR still lacks after-fix real behavior proof, and the proposed global fallback would regress configured local/self-hosted provider timeout behavior by bypassing the current no-default idle-watchdog exemption.

## What This Changes

The PR changes provider model resolution so missing or invalid `models.providers.<id>.timeoutSeconds` returns a hard-coded 120000 ms request timeout instead of leaving `requestTimeoutMs` unset.

## Best Possible Solution

Bound the slow cloud/provider HTTP request path while preserving the explicit local-provider no-default-watchdog behavior, then add focused regression coverage and real setup proof.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows a default `requestTimeoutMs` would be consumed before the current local-provider exemption. The original DeepSeek hang is not reproduced from the PR evidence because only a typecheck is reported.

## Solution Assessment

No. A global fallback in `resolveProviderRequestTimeoutMs` is broader than the current timeout contract; the safer fix should bound the cloud/provider request path while preserving local-provider opt-out behavior.
