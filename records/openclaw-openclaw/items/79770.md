## Summary

Keep open: the patch is not mergeable because it would shorten configured local/self-hosted provider runs that current tests and docs deliberately exempt from the default idle watchdog, and the contributor has not provided after-fix real behavior proof for the claimed Gateway/DeepSeek hang.

## What This Changes

The PR changes provider model resolution so missing, invalid, or non-positive `models.providers.<id>.timeoutSeconds` yields a 120000 ms `requestTimeoutMs` instead of leaving it unset.

## Best Possible Solution

Bound the slow remote provider HTTP request path while preserving local/self-hosted no-default-watchdog semantics, then add focused regression coverage and redacted real setup proof.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows a configured local provider without `timeoutSeconds` would receive `requestTimeoutMs=120000`, and the idle resolver consumes that before the local-provider exemption. The original DeepSeek hang is not proven by the PR evidence because only a typecheck is reported.

## Solution Assessment

No. The global fallback in model resolution is broader than the existing timeout contract; the safer fix should apply a default only to the cloud/provider path that needs bounding while preserving local-provider opt-out behavior.
