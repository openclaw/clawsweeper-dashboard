## What This Changes

The issue requests operator-facing memory lifecycle flows for reviewing, editing, forgetting, tombstoning/superseding, and resolving conflicting durable memories.

## Best Possible Solution

Define the active memory lifecycle contract first, then implement review/edit/forget/conflict flows through the owning memory plugin or SDK seam with non-destructive status transitions, provenance, and a single write resolver.
