## Summary

Keep open: current main and v2026.6.10 still do not provide the requested memory review, edit, non-destructive forget, tombstone/supersede, or conflict-resolution flows. Existing Dreaming, Memory Wiki, LanceDB forget, and the draft Memory Audit PR overlap pieces of the problem, but this remains a product/storage/API design item rather than a cleanup close.

## What This Changes

Request first-class operator flows for reviewing, editing, forgetting, tombstoning or superseding, and resolving conflicts in durable memory without direct database access.

## Best Possible Solution

Define a memory-plugin-owned lifecycle contract with SQLite-backed review state, provenance, explicit status transitions, safe forget/tombstone behavior, conflict resolution, reviewable CLI/UI flows, docs, and regression coverage before landing implementation.
