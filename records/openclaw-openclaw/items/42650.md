Keep open: current main still has adjacent memory search, promotion, Memory Wiki, and LanceDB memory surfaces, but it does not provide first-class review, edit, non-destructive forget/tombstone, or conflict-resolution operator flows. ## What This Changes

The issue requests first-class memory review, edit, forget, tombstone/supersede, and conflict-resolution operator flows.

## Best Possible Solution

Define the durable memory record/status/revision/provenance model first, then add small operator CLI/RPC or UI flows for batched review, edit-by-supersede, tombstone/forget, and explicit conflict resolution.

## Reproduction Assessment

## Solution Assessment

Unclear as a final implementation because the schema, write pipeline, and retention policy are still open. The requested direction is maintainable if it lands on durable statuses, revisions, provenance, and review surfaces instead of wrapping the current delete behavior.
