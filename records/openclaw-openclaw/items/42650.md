## Summary

Keep open: current main and the latest release have memory search, promotion, REM review/backfill, memory-wiki linting, and LanceDB store/recall/forget tools, but they do not implement the requested first-class review/edit/forget/conflict-resolution lifecycle with tombstone and superseded states. ## What This Changes

Request first-class operator flows for reviewing, editing, forgetting, and resolving conflicts in durable memory without destructive deletion.

## Best Possible Solution

Keep this open until maintainers choose a memory-plugin-owned v1 lifecycle model with explicit statuses, provenance, review/edit/forget commands, retrieval filtering, and conflict resolution semantics.
