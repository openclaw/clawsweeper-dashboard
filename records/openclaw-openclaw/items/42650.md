## Summary

Keep open: current main has adjacent memory search, promotion, REM/backfill, Memory Wiki, and LanceDB surfaces, but it still lacks the requested first-class operator review, edit, non-destructive forget/tombstone, and conflict-resolution lifecycle. ## What This Changes

Add operator-facing memory review, edit, non-destructive forget, and conflict-resolution flows for the Memory MVP.

## Best Possible Solution

Define canonical memory records, statuses, revisions, provenance, and conflict links together with the related schema/write-pipeline issues, then add narrow operator CLI/RPC or skill flows for batched review, superseding edits, soft tombstone forget, and explicit conflict resolution.
