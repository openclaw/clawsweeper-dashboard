## Summary

Keep open: current main has adjacent memory search, promotion, Memory Wiki, and LanceDB surfaces, but it still lacks the requested first-class operator review, edit, non-destructive forget/tombstone, and conflict-resolution flows. ## What This Changes

The issue requests first-class operator flows to review, edit, forget, and resolve conflicts for durable memories.

## Best Possible Solution

Define canonical memory records, statuses, revisions, provenance, and conflict links first, then add small operator CLI/RPC or UI flows for batched review, superseding edits, soft tombstone forget, and explicit conflict resolution.

## Solution Assessment
