## Summary

Keep open. Current main still has recall, promotion, and wiki-adjacent memory surfaces, but not first-class review, edit, tombstone/forget, or conflict-resolution flows; the KinthAI production note adds useful implementation context for batched review, supersede-not-delete, soft tombstones, and provenance.

## What This Changes

Add operator-facing memory review, edit, non-destructive forget, and conflict-resolution flows for the Memory MVP.

## Best Possible Solution

Sequence this after #42646, #42648, and #42651 with a canonical record/status/revision/provenance model, then expose small CLI/RPC or UI flows for batched review, edit-by-supersede, tombstone/forget, and explicit conflict resolution.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection against current main gives a high-confidence verification path: the requested operator flows and durable status schema are absent, and the existing LanceDB forget path hard deletes records.

## Solution Assessment

Unclear as a final implementation because the schema and write pipeline are not settled. The requested direction looks sound only if it lands on durable record statuses, revisions, and provenance instead of wrapping the existing delete path or wiki metadata edits.
