## Summary

Keep open. Current main has memory search, promotion, dreaming, and wiki-adjacent review surfaces, but it still lacks first-class review, edit, soft-forget/tombstone, and conflict-resolution operator flows; the KinthAI production comment adds useful design evidence rather than making the issue stale.

## What This Changes

Request to add first-class memory operator flows for review queues, edits, soft forget/tombstones, and conflict resolution.

## Best Possible Solution

Define the durable memory record/status/revision/provenance model first, then add small operator CLI/RPC or UI flows for batched review, edit-by-supersede, tombstone/forget, and explicit conflict resolution.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main gives a high-confidence verification path: the requested operator flows are absent and the existing LanceDB forget path still deletes records.

## Solution Assessment

Unclear as a final implementation because the schema and write pipeline are still open. The requested direction is maintainable if it lands on durable statuses, revisions, provenance, and review surfaces instead of wrapping the existing delete behavior.
