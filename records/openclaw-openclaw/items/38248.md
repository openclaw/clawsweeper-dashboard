## Summary

Keep this open. Current main has pricing metadata, usage/cost displays, transcript-backed cost aggregation, auth cooldowns, and a narrow idle-timeout runaway breaker, but it still lacks the requested pre-dispatch rolling spend ceiling for expensive fallback chains. The discussion adds useful external context from vibe-billing and KinthAI, and #42475 overlaps with gateway budget design, but this issue still carries distinct hourly/failover requirements that should not be swept away yet.

## What This Changes

Request a configurable cost guard that tracks model-call spend and pauses, rejects, downgrades, or alerts when hourly/daily ceilings would be exceeded.

## Best Possible Solution

Add a backward-compatible gateway/model-dispatch budget seam that checks estimated spend before each fallback candidate, reconciles actual usage after responses, exposes status/doctor visibility, and documents how it composes with broader per-agent budget tracking.

## Reproduction Assessment

Not applicable as a feature request; this is source-verifiable rather than a flaky runtime bug. Current main has no accepted cost-guard config field and no pre-dispatch spend gate in the fallback path.

## Solution Assessment

No; current cost reporting, auth cooldowns, and the idle-timeout breaker are not the best solution because they act after usage exists or only cover one timeout failure mode. The maintainable fix should gate model dispatch before the expensive fallback request is sent, coordinated with #42475.
