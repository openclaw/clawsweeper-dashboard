## Summary

Keep this open. Current main has cost metadata, reporting, usage aggregation, and auth cooldowns, but no pre-dispatch rolling spend ceiling for expensive fallback chains; the external vibe-billing and KinthAI discussion reinforces that this needs product/API design rather than cleanup closure.

## What This Changes

The issue requests a configurable per-hour spending ceiling that pauses, rejects, or notifies before failover retries can create runaway model costs.

## Best Possible Solution

Add a backward-compatible gateway/model-dispatch budget seam that checks estimated spend before each model or fallback attempt, reconciles actual usage afterward, and exposes status, doctor, and docs behavior for the chosen budget windows.

## Reproduction Assessment

Not applicable as a feature request; the missing behavior is source-verifiable. Current main has no accepted cost-guard config field and no pre-dispatch spend gate in the fallback path.

## Solution Assessment

No; current cost reporting and auth cooldowns are not the best solution because they act after usage exists or only cover provider failure reliability. The maintainable fix should gate dispatch before expensive fallback requests, coordinated with #42475.
