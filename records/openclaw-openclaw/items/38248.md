## Summary

Keep open. Current main still has cost metadata, usage display, transcript aggregation, and fallback cooldown behavior, but no native pre-dispatch rolling spend ceiling for expensive fallback chains. The external workarounds and production notes in the discussion, including vibe-billing and KinthAI gateway middleware, are useful evidence but do not implement this in OpenClaw core, and the broader #42475 tracker has not explicitly superseded this failover-specific request.

## What This Changes

The issue requests a native per-hour/per-day model-spend guard that can pause, reject, degrade, and notify before fallback chains run up unexpected provider costs.

## Best Possible Solution

Design one backward-compatible model-spend budget seam shared by gateway, CLI, cron, fallback, and subagent paths: strict config/schema/docs, pre-attempt estimated-cost admission, durable rolling accounting or reservations, post-response reconciliation, clear over-budget outcomes, status/doctor visibility, and targeted regression coverage for expensive rate-limit fallback cascades.

## Reproduction Assessment

Not applicable as a feature request; the gap is source-verifiable. The focused check is the strict config schemas plus runWithModelFallback/agent-command, where current main has no accepted budget field or pre-attempt spend gate.

## Solution Assessment

Unclear until maintainers choose the API shape. The requested native pre-dispatch guard is the right direction, but the specific costGuard fields and behavior should be coordinated with #42475 rather than added as a one-off fallback-only policy.
