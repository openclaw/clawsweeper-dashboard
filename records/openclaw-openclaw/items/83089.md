## Summary

Keep this open for maintainer product/API review. Current main and v2026.5.12 intentionally skip plugin tools whose normalized names collide with existing tool names, and current plugin docs tell authors to use unique non-colliding tool names; changing `memory_search` to override, merge, or auto-namespace would alter the public plugin contract and overlaps with the broader third-party memory-provider discussion at https://github.com/openclaw/openclaw/issues/82977.

## What This Changes

The issue asks OpenClaw to handle OpenViking's `memory_search` plugin-tool collision with the default memory tool instead of dropping the plugin's registration.

## Best Possible Solution

Keep the global unique tool-name contract unless maintainers intentionally design a generic memory-provider or tool-shadowing API; OpenViking should use unique/prefixed names or an existing memory slot instead of relying on duplicate `memory_search` registration.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: when memory-core or another earlier tool exposes `memory_search`, a later plugin registering the same normalized name is skipped by `resolvePluginTools`; I did not run a live OpenViking install.

## Solution Assessment

No for an automated bug fix. Override, merge, or auto-namespacing behavior would change the documented plugin API and memory-provider ownership model, so maintainers should decide the contract before code changes.
