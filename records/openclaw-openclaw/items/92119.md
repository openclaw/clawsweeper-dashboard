## Summary

Keep open: the PR targets a real Windows CLI-provider launch bug, but the patch is not merge-ready because it only maps `claude` and `gemini` to `.cmd` while the supervisor still spawns the resulting `.cmd` file directly. Current `main` and `v2026.6.6` remain unfixed, and the open alternative PR is not a safe superseding target yet.

## What This Changes

This PR adds `claude` and `gemini` to the supervisor Windows `.cmd` shim allowlist and adds unit tests for shim resolution.

## Best Possible Solution

Use one generic supervisor-side Windows invocation path that resolves npm-style shims and executes `.cmd`/`.bat` through the existing trusted `cmd.exe`/escaping pattern, with coverage for Claude and Gemini plus real Windows proof.

## Reproduction Assessment

Yes by source and linked user evidence. Current `main` passes bare Claude/Gemini CLI backend commands into the supervisor, and the PR branch would pass `.cmd` into direct `spawn()` without a Windows batch wrapper; I did not run a live Windows host in this read-only review.

## Solution Assessment

No. Suffixing the allowlist is only a plausible partial mitigation; the safer maintainable fix is a generic supervisor invocation path that matches the existing `runCommandWithTimeout` trusted `cmd.exe` wrapper behavior for Windows batch shims.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wrap resolved `.cmd` shims before spawning:** `src/process/supervisor/adapters/child.ts:17`
  - body: This changes `claude`/`gemini` into `claude.cmd`/`gemini.cmd`, but `createChildAdapter` then sends that argv through `spawnWithFallback`, which calls `spawn()` directly. Node's Windows contract says `.cmd` files are not executable on their own and need `shell`, `exec`, or an explicit `cmd.exe` wrapper, and this repo's `runCommandWithTimeout` already wraps `.cmd` through trusted `cmd.exe`; as written the Windows provider launch can still fail with `EINVAL` after the lookup failure is fixed. ([nodejs.org](https://nodejs.org/api/child_process.html?utm_source=openai))
  - confidence: 0.91
- **[P2] Keep provider commands out of the core allowlist:** `src/process/supervisor/adapters/child.ts:17`
  - body: Hard-coding `claude` and `gemini` in the core supervisor means every bundled or external CLI backend with an npm shim needs another core edit to work on Windows. The command ownership already lives in the CLI backend descriptors, so the fix should be generic for Windows batch shims or derive from a typed backend/runtime capability rather than adding provider names to this core list.
  - confidence: 0.78
