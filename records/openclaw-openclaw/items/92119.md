## Summary

Keep open: this is a focused alternative for the Windows Claude/Gemini CLI shim failure, but the branch only appends `.cmd` names and still sends the result through direct supervisor `spawn()`, so it can leave the provider workflow unavailable.

## What This Changes

The PR adds `claude` and `gemini` to the supervisor Windows `.cmd` shim allowlist and adds unit coverage for those resolver cases.

## Best Possible Solution

Use one generic supervisor-side Windows invocation path that resolves npm-style shims and executes `.cmd`/`.bat` through the existing trusted wrapper or shared spawn policy, with Claude/Gemini coverage and redacted Windows proof.

## Reproduction Assessment

Yes, at source level and with linked Windows reporter evidence. Current main passes bare CLI backend commands into a supervisor path that only suffixes package-manager shims, while the PR branch would pass `.cmd` names into direct spawn without a batch wrapper.

## Solution Assessment

No. The PR is a plausible lookup-only mitigation, but the maintainable fix needs a generic supervisor Windows spawn path that also handles `.cmd`/`.bat` execution safely.
