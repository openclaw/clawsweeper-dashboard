## Summary

Keep open. Current main has adjacent conservative stale-session recovery and diagnostics, but it still intentionally does not define the reported terminal timeout/session-health policy for active Discord-routed embedded runs.

## What This Changes

The report asks OpenClaw to make Discord-routed embedded-agent sessions unambiguous after a fatal timeout, including routing health and partial-success user copy.

## Best Possible Solution

Add a shared terminal timeout/session-health transition in the embedded reply/session pipeline that marks the affected route failed/dead or rotates/requires /new, with Discord copy warning that prior side effects may have completed.

## Reproduction Assessment

Yes for the current source-level gap: the classifier, heartbeat, recovery runtime, and tests show active embedded work is not recovered by default, while Discord direct raw-runner failures still use generic copy. A live Discord/Claude CLI timeout would strengthen timing proof but is not required to see the missing policy.

## Solution Assessment

Unclear because no patch is proposed and the issue lists multiple acceptable behaviors. The narrow maintainable direction is a shared session-health transition after maintainers choose the active-run timeout policy.
