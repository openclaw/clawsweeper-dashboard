## Summary

Keep open. Current main has adjacent cleanup that helps timeout/error runs return to idle, but it still does not implement the reported terminal-timeout policy: Discord direct raw-runner failures keep the generic failure copy, active embedded work is not recovered by default, and fresh stored sessions are reused without a timeout-dead or rotate gate.

## What This Changes

Request an unambiguous terminal timeout/session-health transition for Discord-routed agent turns that may complete side effects before the runner fails.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks or rotates the affected chat route before the next turn and uses Discord-facing copy that warns prior side effects may have completed.

## Reproduction Assessment

Yes for the source-level behavior gap: current code and tests show generic Discord direct failure copy, observe-only active recovery, and stored-session reuse without a timeout-dead gate. A live Discord/Claude CLI timeout would strengthen timing proof but is not required to see the missing policy.

## Solution Assessment

Unclear because no patch is proposed and the issue intentionally allows several acceptable policies. The narrow maintainable direction is to choose one shared timeout policy, then wire it through session health, routing, and user-facing copy.
