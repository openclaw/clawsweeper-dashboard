## Summary

Keep open. Current main still observes active stalled Discord/CLI work without terminal timeout recovery, reuses fresh stored sessions without a timeout-dead gate, and keeps Discord direct raw-runner failures on generic copy; the closed #73581 context is related but does not cover this remaining timeout-routing policy.

## What This Changes

Make Discord-routed CLI agent timeouts mark or rotate the affected session and surface partial-success/timeout state instead of leaving the session routable with generic failure copy.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks or rotates the affected chat route before the next turn, with Discord-facing copy warning that prior side effects may have completed.

## Reproduction Assessment

Yes for a source-level path: current code and docs show active stalled work is observed rather than recovered, fresh stored sessions lack a timeout-dead reuse gate, and Discord direct raw-runner failures still use generic copy. A live Discord/Claude CLI timeout would only strengthen timing proof.

## Solution Assessment

Unclear because no patch is proposed and the issue lists several acceptable policies. The narrow maintainable direction is a shared terminal timeout health transition plus explicit partial-success/timeout copy after maintainers choose the active-run policy.
