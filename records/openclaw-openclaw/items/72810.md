## Summary

Keep open. Current main has adjacent liveness, abort, and restart-recovery cleanup, but it still does not implement an unambiguous terminal-timeout health transition or partial-success state for Discord-routed agent turns.

## What This Changes

The issue asks OpenClaw to stop reusing or ambiguously reporting a Discord-routed agent session after a terminal embedded-agent timeout with possible side effects.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks or rotates the affected chat route before the next turn, with Discord-facing copy warning that prior side effects may have completed.

## Reproduction Assessment

Yes for the source-level behavior gap: current code, tests, and docs show active work is not recovered by default, Discord direct raw-runner failures still use generic text, and session reuse lacks a timeout-dead gate. A live Discord/Claude CLI timeout would only strengthen timing proof.

## Solution Assessment

Unclear because no patch is proposed and the issue intentionally leaves several acceptable policies. The narrow maintainable direction is a shared session-health transition plus timeout/partial-success copy after maintainers choose the active-run policy.
