## Summary

Keep open. Current main still observes stalled active Discord/CLI work and keeps direct raw-runner failures on generic copy, but it does not add a terminal timeout health transition, route rotation/dead-session gate, or partial-success warning; the closed #73581 context is related but broader and does not cover this remaining Discord timeout outcome.

## What This Changes

The issue asks for Discord-routed agent timeouts to leave an unambiguous session health/routing state and distinguish partial-success late failures from total failures.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks or rotates the affected chat route before the next turn, with Discord-facing copy warning that prior side effects may have completed.

## Reproduction Assessment

Yes for a source-level reproduction path: current code and docs show active stalled work is not recovered by default, reusable stored sessions lack a timeout-dead gate, and Discord direct raw-runner failures still use generic copy. A live Discord/Claude CLI timeout would strengthen timing proof but is not required to see the behavioral gap.

## Solution Assessment

Unclear because no patch is proposed and the issue lists multiple acceptable policies. The narrow maintainable direction is a shared terminal timeout health transition plus explicit partial-success/timeout copy after maintainers choose the active-run policy.
