## Summary

Keep open. Current main still observes active stalled embedded/model work instead of recovering it, reuses fresh stored sessions without a terminal-timeout/dead-session gate, and keeps normal Discord direct raw-runner failures on generic copy; the related #72434 and closed #73581 context does not cover this remaining Discord timeout routing and partial-success state gap.

## What This Changes

The issue asks OpenClaw to make Discord-routed agent sessions unambiguous after fatal timeout or pre-reply embedded-agent failure, especially when side effects may already have completed.

## Best Possible Solution

Add a shared terminal-timeout/session-health transition that marks or rotates the affected chat route before the next turn, with Discord-facing copy warning that prior side effects may have completed.

## Reproduction Assessment

Yes for a high-confidence source-level path: current main classifies active stalled work as non-recoverable, leaves active abort opt-in, reuses fresh stored sessions, and keeps Discord direct raw-runner failures on generic copy. A live Discord/Claude CLI timeout would add timing proof but is not required to see the remaining code-path gap.

## Solution Assessment

Unclear because no patch is proposed and the issue lists multiple acceptable recovery policies. The narrow maintainable direction is a shared terminal timeout health transition plus explicit partial-success/timeout copy after maintainers choose the active-run policy.
