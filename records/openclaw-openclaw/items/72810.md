## Summary

Keep open. Current main still treats active stalled Discord/CLI work as non-recoverable observation, only releases stale no-active-work sessions, reuses fresh stored sessions, and keeps Discord direct raw-runner failures on generic copy; the related #72434 and closed #73581 context do not close this remaining terminal-timeout routing and partial-success state gap.

## What This Changes

The issue asks OpenClaw to make Discord-routed agent sessions unambiguous after fatal timeout or pre-reply embedded-agent failure, including route health and partial-success user-facing copy.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks or rotates the affected chat route before the next turn, with Discord-facing copy warning that prior side effects may have completed.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction path: current code and docs show active stalled work is observed rather than recovered, reusable stored sessions lack a timeout-dead gate, and Discord direct raw-runner failures still use generic copy. A live Discord/Claude CLI timeout would add timing proof but is not required to see the remaining behavioral gap.

## Solution Assessment

Unclear because no patch is proposed and the issue intentionally lists multiple acceptable recovery policies. The narrow maintainable direction is a shared terminal timeout health transition plus explicit partial-success/timeout copy after maintainers choose the active-run policy.
