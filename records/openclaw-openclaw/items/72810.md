## Summary

Keep open. Current main has adjacent lane-timeout and stuck-session diagnostics, but it still does not make a timed-out Discord-routed active agent session unambiguously failed/dead or distinguish partial side effects from a generic pre-reply failure.

## What This Changes

The issue requests a clear terminal timeout/session-health policy for Discord-routed agent turns, including route reset or dead-session marking and partial-success user-facing copy.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks or rotates the affected chat route before the next turn, with Discord-facing copy warning that previous side effects may have completed.

## Reproduction Assessment

Yes for the source-level behavior gap: current code and tests show active embedded stalls are not recovered by default, session reuse has no timeout-dead gate, and Discord direct runner failures still use generic copy. A live Discord/Claude CLI timeout would strengthen timing proof but is not required to see the missing policy.

## Solution Assessment

Unclear because no patch is proposed and the issue intentionally lists several acceptable policies. The narrow maintainable direction is a shared session-health transition plus timeout/partial-success copy after maintainers choose the active-run policy.
