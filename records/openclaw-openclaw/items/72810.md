## Summary

Keep open. Current main has conservative stale-session recovery for stale bookkeeping, but active stalled Discord/CLI work remains observe-only, fresh session reuse lacks a terminal-timeout/dead-session gate, and Discord direct raw-runner failures still use generic copy; the closed #73581 work is related but does not cover this remaining terminal-timeout routing outcome.

## What This Changes

The report asks OpenClaw to make Discord/CLI agent sessions unambiguously failed, rotated, or clearly partial-success after fatal timeouts instead of reusing an ambiguous route with generic failure copy.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks or rotates the affected chat route before the next turn, with Discord-facing copy warning that prior side effects may have completed.

## Reproduction Assessment

Yes for a source-level reproduction path: current code shows active stalled work is not recovered by default, fresh stored sessions are reused without a terminal timeout gate, and Discord direct raw-runner failures still use generic copy. A live Discord/Claude CLI timeout would strengthen timing proof but is not required to see the remaining behavioral gap.

## Solution Assessment

Unclear because no patch is proposed and the issue lists several acceptable policies. The narrow maintainable direction is a shared terminal timeout health transition plus explicit partial-success/timeout copy after maintainers choose the active-run policy.
