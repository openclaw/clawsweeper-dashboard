## Summary

Keep open. Current main has adjacent liveness cleanup, but it still does not implement the reported terminal timeout session-health policy or partial-success user state for Discord-routed agent turns.

## What This Changes

The issue asks OpenClaw to make Discord-routed embedded-agent timeouts unambiguous when work may have completed side effects before a terminal failure.

## Best Possible Solution

Add a shared terminal timeout/session-health transition that marks the affected chat route failed or rotates it before the next turn, with Discord-facing copy that warns prior side effects may have completed.

## Reproduction Assessment

Yes for the source-level behavior gap: current code and tests show active work is not recovered by default, Discord direct raw-runner failures still use generic text, and session reuse lacks a timeout-dead gate. A live Discord/Claude CLI timeout would only strengthen timing proof.

## Solution Assessment

Unclear because no patch is proposed and the issue allows several policies. The narrow maintainable direction is a shared session-health transition plus timeout/partial-success copy after maintainers choose the active-run policy.
