## Summary

Keep this PR open: current main does not have the requested Nacos config source, but the proposed core remote-config change is still not merge-ready because source inspection shows runtime reload semantics and documented CLI behavior do not line up, real Nacos proof is missing, and the security model needs maintainer review.

## What This Changes

The PR adds an optional Nacos-backed gateway config source with env-based selection, file/Nacos adapters, long-poll reload subscription, CLI/doctor/startup wiring, docs, changelog, and tests.

## Best Possible Solution

If maintainers want remote config in core, land a generic source seam with file as the default and Nacos as an explicit adapter that preserves current reload/write semantics, documents security and persistence limits, and is proven against a real Nacos server.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection of PR head shows Nacos writes do not notify the reload planner, standalone CLI writes throw despite the docs, and Nacos snapshots bypass the file-backed shell-env fallback path. No live reproduction was established for the full Nacos runtime path because the PR has no real Nacos proof.

## Solution Assessment

No. The source abstraction is plausible, but this is not the best mergeable form until Nacos preserves existing config semantics, the remote-config security model is explicit, and docs match real CLI/RPC behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Notify the reloader for Nacos writes:** `src/config/io.ts:2481`
  - body: In Nacos mode, `writeConfigFile()` updates the runtime snapshot and can refresh derived state, then returns without emitting the existing write notification. Since subscribe mode only listens to the external Nacos subscription, successful `config.apply`/`config.patch` calls can update `config.get` while hot-reload-managed services stay stale until a remote Nacos edit or restart.
  - confidence: 0.87
- **[P3] Align CLI write docs with standalone Nacos behavior:** `docs/gateway/configuration.md:744`
  - body: The docs say CLI `config set` updates the in-memory snapshot in Nacos mode, but standalone config commands initialize the Nacos source and then `writeConfigFile()` throws when there is no runtime snapshot or refresh handler. Users following this section get an error instead of the documented process-local update.
  - confidence: 0.9
- **[P2] Preserve shell-env fallback for Nacos snapshots:** `src/config/sources/snapshot-from-raw.ts:83`
  - body: The file-backed loader imports shell env when `env.shellEnv.enabled` or `OPENCLAW_LOAD_SHELL_ENV` is set before validation, but the Nacos snapshot builder resolves env vars and validates without that fallback. A config that works from `openclaw.json` can fail or lose profile-provided credentials when moved to Nacos.
  - confidence: 0.78
