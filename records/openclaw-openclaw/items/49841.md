## Summary

Keep open: current main does not provide a Nacos config source, and this PR still has a source-reproducible reload correctness bug, missing real Nacos behavior proof, and an unsettled remote-config security model.

## What This Changes

The branch adds an optional Nacos-backed gateway config source with env selection, file/Nacos adapters, long-poll reload subscription, CLI/doctor/startup wiring, docs, changelog, and tests.

## Best Possible Solution

Land this only after the config-source seam preserves existing write/reload semantics, clearly documents persistence limits, defines remote-config security guardrails, and has real Nacos validation; otherwise move Nacos behind a future pre-start config-source plugin/API.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection of head 191251c8e028ed9d47522906d31514bbadf7bd4f shows Nacos writes return without notifying the reload planner while subscribe mode bypasses subscribeToWrites. No live Nacos end-to-end proof was provided.

## Solution Assessment

No. The abstraction direction is plausible, but this is not the best mergeable form until Nacos mode preserves current config semantics, has a settled remote-config security contract, and is proven against a real Nacos server.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Route Nacos writes through the reload planner:** `src/config/io.ts:2481-2501`
  - body: In Nacos mode this branch updates the runtime snapshot and may refresh secrets, then returns without emitting the existing config write notification; subscribe mode also does not register `subscribeToWrites`. Successful `config.set`, `config.patch`, or `config.apply` calls can therefore update `config.get` while leaving hot-reload-managed services such as hooks, cron, or channels on the old settings until a remote Nacos edit or restart occurs.
  - confidence: 0.87
- **[P3] Fix standalone CLI write behavior in Nacos docs:** `docs/gateway/configuration.md:744`
  - body: This sentence says CLI `config set` updates the in-memory snapshot in Nacos mode, but a standalone CLI process has no runtime snapshot or refresh handler, so the new Nacos write path throws instead of updating Nacos or the running gateway. The docs should distinguish running-gateway RPC writes from standalone CLI behavior.
  - confidence: 0.9
