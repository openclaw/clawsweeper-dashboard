Full review comments:

- **[P2] Route Nacos writes through the reload planner:** `src/gateway/config-reload.ts:474-475`
  - body: With `opts.subscribe` set, the reloader only registers the external Nacos callback and skips the in-process write listener; the Nacos `writeConfigFile()` branch also returns after runtime refresh without a notification. Successful `config.set` or `config.patch` can update `config.get` while hooks, cron, channels, or other hot-reloadable services stay on the old config until a remote Nacos edit or restart.
  - confidence: 0.88
- **[P2] Do not let restart-required writes self-revert:** `src/config/io.ts:2481-2486`
  - body: This branch stores the write only in process memory, but the existing gateway config write flow can still queue a SIGUSR1 restart for restart-required changes. After that restart, startup fetches the unchanged Nacos payload, so the successful `config.apply` or `config.patch` loses its own change unless the write is published to Nacos, rejected, or handled by a Nacos-aware restart policy.
  - confidence: 0.82
- **[P3] Correct standalone Nacos write documentation:** `docs/gateway/configuration.md:744`
  - body: The docs say CLI `config set` updates the in-memory snapshot in Nacos mode, but standalone CLI processes have no running gateway memory and this branch throws when there is no runtime snapshot or refresh handler. Document standalone writes as unsupported unless a Nacos publish or gateway RPC write path is added.
  - confidence: 0.82
