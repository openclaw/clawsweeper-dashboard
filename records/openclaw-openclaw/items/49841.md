## Summary

Keep open. This is a real feature proposal, but the PR still has blocking correctness findings, missing real Nacos runtime proof, and an unsettled remote-config security model before it can merge.

## What This Changes

The PR adds optional Nacos-backed gateway configuration loading with long-poll reloads, CLI/doctor/startup source resolution, docs, changelog, and tests.

## Best Possible Solution

Land either a core generic config-source seam with Nacos preserving existing reload/write/security semantics, or first define a pre-start plugin API and move the Nacos adapter outside core.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection of the PR head shows subscribe-mode reloads skip in-process write notifications, standalone CLI write docs conflict with the code, and Nacos snapshots omit the file-backed shell-env fallback. No live Nacos end-to-end reproduction was established because the PR has no real behavior proof.

## Solution Assessment

No. The abstraction direction is plausible, but this is not the best mergeable form until Nacos mode preserves current config semantics, has an explicit remote-config security contract, and is proven against a real Nacos server.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep write notifications active in subscribe mode:** `src/gateway/config-reload.ts:474-475`
  - body: In Nacos mode, the reloader registers only the external subscription and never subscribes to in-process config writes. A successful config.set/config.patch can update config.get while hot-reload-managed services stay stale until a remote Nacos edit or restart occurs.
  - confidence: 0.87
- **[P2] Import shell env before validating Nacos snapshots:** `src/config/sources/snapshot-from-raw.ts:83-85`
  - body: The file-backed loader can import login-shell env when env.shellEnv.enabled or OPENCLAW_LOAD_SHELL_ENV is set before validation, but this Nacos snapshot path only applies config.env. Moving the same config to Nacos can therefore fail validation or lose credentials that work from openclaw.json.
  - confidence: 0.78
- **[P3] Correct standalone CLI write docs for Nacos mode:** `docs/gateway/configuration.md:744`
  - body: This docs line says CLI config set updates the in-memory snapshot in Nacos mode, but a standalone CLI process has no runtime snapshot or refresh handler, so the new write path throws instead of updating anything.
  - confidence: 0.9
