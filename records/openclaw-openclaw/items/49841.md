## Summary

Keep open: current main still has no Nacos config source, and this PR remains a broad core gateway/config feature with a blocking write/reload bug, missing real Nacos proof, and unsettled remote-config security semantics.

## What This Changes

The PR adds an opt-in Nacos-backed gateway config source with env selection, file/Nacos adapters, long-poll reload subscription, CLI/doctor/startup wiring, docs, changelog, and tests.

## Best Possible Solution

If maintainers want remote config in core, land a generic source seam with file as the default and Nacos as a validated adapter that preserves startup, CLI/RPC reads and writes, reload planning, security guidance, and real Nacos validation; otherwise define the needed pre-start plugin API and move Nacos outside core.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection of PR head 191251c8e028ed9d47522906d31514bbadf7bd4f shows the Nacos write/reload notification gap and the standalone CLI documentation mismatch. Real Nacos runtime behavior is not proven by the contributor yet.

## Solution Assessment

No. The source abstraction is plausible, but this patch is not the best mergeable form until Nacos writes preserve or clearly reject reload semantics, standalone CLI behavior is documented accurately, and remote-config security expectations are settled.
