## Summary

Keep open: current main does not implement a Nacos config source, and this PR is still the active implementation candidate, but the patch is not mergeable while Nacos in-process writes bypass reload notifications and the remote-config security model is unsettled.

## What This Changes

This PR adds an optional Nacos-backed gateway config source with env-based source resolution, file/Nacos adapters, long-poll reload subscription, CLI/doctor/startup wiring, docs, changelog, and tests.

## Best Possible Solution

If maintainers want remote config in core, land a generic source seam with file as the default and Nacos as an explicit adapter that preserves read/write/reload semantics, documents persistence limits, handles auth/TLS/error/retry behavior, and has real Nacos validation.

## Reproduction Assessment

Yes. Enable Nacos mode, perform a gateway config write for a hot-reloadable setting through the running gateway, and the diff shows config.get can see the runtime snapshot update while subscribe-mode reloaders never receive an in-process write notification.

## Solution Assessment

No. The direction may be useful, but this branch is not the best mergeable form until Nacos writes drive the same reload planner as file writes and the remote-config security/deployment guardrails are settled.
