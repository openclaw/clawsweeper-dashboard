## Summary

Keep open. Current main still has only file-backed gateway configuration, while this PR is an unmerged Nacos remote-config feature with blocking write/reload behavior and unresolved security semantics for loading gateway config over operator-supplied HTTP.

## What This Changes

The PR adds an optional Nacos-backed gateway config source with env-based file/Nacos resolution, long-poll reload subscription, CLI/doctor/startup wiring, docs, changelog, and tests.

## Best Possible Solution

If maintainers want remote config in core, land a generic source seam where file remains the default and Nacos is an explicit, validated adapter that preserves existing read/write/reload semantics, handles auth/TLS/error/retry behavior, documents persistence limits accurately, credits the changelog, and has real Nacos validation. If this belongs outside core, first define the needed pre-start config-source plugin API and move the Nacos adapter to plugin space.

## Reproduction Assessment

Yes. Diff inspection gives a focused reproduction: enable Nacos mode, run a gateway config write that changes a hot-reloadable setting, and observe the runtime snapshot change without the subscribe-mode reloader receiving an in-process write notification. A live Nacos integration smoke is still missing for release confidence.

## Solution Assessment

No. The proposed direction may be useful, but this branch is not the best mergeable form yet because it applies the new source seam inconsistently across write/reload behavior and introduces remote-config security semantics before guardrails are settled.
