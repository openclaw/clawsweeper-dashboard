## Summary

Keep open: current main still has the source-reproducible duplicate prompt path, and the focused fix at https://github.com/openclaw/openclaw/pull/44973 remains open and unmerged rather than shipped.

## What This Changes

A main-session cron systemEvent with delivery mode none should inject only the raw system event and not also wrap the same text as a scheduled reminder.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent marker-and-skip fix with regression coverage and redacted real cron/heartbeat proof, coordinating with broader cron hardening only if maintainers choose that product direction.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: main-session cron queues a `cron:` system event, heartbeat wraps the same queued text as a scheduled reminder, and reply setup can drain it into `System:` lines.

## Solution Assessment

Yes. The narrow maintainable fix is to mark or otherwise identify main-session cron system events that are already injected into history and skip only those from cron reminder wrapping, without changing general delivery or heartbeat semantics.
