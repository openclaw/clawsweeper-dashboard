## Summary

Current main still has the duplicate model-context path, and two open pull requests explicitly target this issue. Keep the issue open until maintainers select and merge one canonical fix; the single-owner approach in https://github.com/openclaw/openclaw/pull/91287 better matches the existing exec-event pattern than adding a second abbreviated representation.

## What This Changes

Ensure a main-session cron `systemEvent` is surfaced only once to the model instead of through both generic system-event rendering and the heartbeat reminder prompt.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/91287 or an equivalent focused change that leaves `cron:` events queued for the existing dedicated heartbeat prompt, with coverage for generic events, isolated sessions, and direct-delivery mirrors; then close the competing implementation.

## Reproduction Assessment

Yes at source level. Current main routes a `cron:` event through generic `System:` rendering and the dedicated heartbeat cron prompt, and the linked before/after proof exercises those real assembly functions; this review did not run a live scheduled cron job.

## Solution Assessment

No, suppressing the reminder specifically because `delivery.mode` is `none` couples model-prompt ownership to an outbound-delivery setting. The narrowest maintainable solution gives cron-tagged content one owner, matching the established exec-completion pattern.
