## Summary

Keep open: current main still has the duplicate source path, and the linked remediation remains open/unmerged rather than an implemented fix.

## What This Changes

The issue asks for main-session cron `systemEvent` jobs with `delivery.mode: "none"` to avoid adding a second scheduled-reminder wrapper to the session/model context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent focused dedupe fix, unless maintainers deliberately choose the broader policy change in https://github.com/openclaw/openclaw/pull/48883, with regression coverage and real cron/heartbeat proof before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main queues a main-session `cron:` system event, heartbeat wraps `cron:` pending events as scheduled reminders, and reply setup can drain the same event into `System:` lines.

## Solution Assessment

Yes. The narrow maintainable solution is to distinguish main-session cron events already injected into history and skip only those from the reminder wrapper while preserving normal cron reminder wrapping elsewhere.
