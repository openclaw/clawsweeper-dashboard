## Summary

Keep open: current main still has the source-level duplicate main-session cron path, and the focused fix PR remains open and unmerged, so this issue is still the canonical tracker.

## What This Changes

Main-session cron `systemEvent` jobs with `delivery.mode: "none"` should avoid adding a second scheduled-reminder wrapper to the same session context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent focused marker-and-skip fix with regression coverage, preserving normal cron reminder wrapping for non-main-session events.

## Reproduction Assessment

Yes, at source level. Current main queues a `cron:` system event for main-session cron jobs, heartbeat wraps `cron:` pending events as scheduled reminders, and reply setup can drain the same queue into `System:` session context.

## Solution Assessment

Yes. The narrow maintainable fix is to mark main-session cron system events that are already injected into history and skip only those events from cron reminder wrapper construction.
