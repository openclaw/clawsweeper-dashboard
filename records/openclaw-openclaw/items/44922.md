## What This Changes

The issue requests that main-session cron `systemEvent` jobs with `delivery.mode: "none"` inject only the system event and not add a scheduled-reminder wrapper.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent marker-and-skip fix with regression coverage and redacted cron/heartbeat proof while preserving normal cron wrapper and delivery semantics.

## Reproduction Assessment

Yes, at source level. Current main queues a main-session `cron:` system event, heartbeat peeks and wraps the same text as a scheduled reminder, and reply setup can drain that queue into `System:` lines.

## Solution Assessment

Yes. The narrow maintainable solution is to identify main-session cron system events already injected into history and skip only those from cron reminder wrapping, without changing broader delivery or heartbeat behavior.
