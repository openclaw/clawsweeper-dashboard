## Summary

Keep open: current main and the latest shipped tag still have the cron-to-heartbeat duplicate source path, and the targeted fixes remain open PR work rather than landed code.

## What This Changes

The issue asks OpenClaw to suppress the extra scheduled-reminder heartbeat wrapper for main-session cron systemEvent jobs when delivery.mode is none.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent maintainer-chosen remediation such as https://github.com/openclaw/openclaw/pull/48883 with focused regression coverage and proof that normal cron reminder behavior still works.

## Reproduction Assessment

Yes, at source level. Current main queues a main-session `cron:` system event, heartbeat wraps cron-tagged pending events as scheduled reminders, and reply setup can drain the same queued text into the prompt as `System:` context.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish main-session cron system events that are already injected into history and skip only those from cron reminder wrapping, unless maintainers intentionally choose the broader constraint path.
