## Reproduction Assessment

Yes, source-reproducible. Current main queues a main-session `cron:` system event, heartbeat peeks and wraps the same text as a scheduled reminder, and reply setup can drain that queue into `System:` lines.

## Solution Assessment

Yes. The narrow maintainable fix is to identify main-session cron system events already injected into history and skip only those from cron reminder wrapping, without changing broader delivery or heartbeat behavior.
