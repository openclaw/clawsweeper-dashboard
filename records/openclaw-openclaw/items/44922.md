## Summary

Keep open: current main and the latest release still have the source-level duplication path, and the open narrow fix PR has not merged.

## What This Changes

Suppress the redundant scheduled-reminder wrapper for main-session cron `systemEvent` jobs with `delivery.mode: "none"`, leaving the injected system event as the only model-facing reminder text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent narrow marker-and-skip fix with regression coverage and real cron/heartbeat proof, while preserving ordinary cron reminder wrappers and `delivery.mode` semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron queues a `cron:` system event, heartbeat peeks that event into the cron wrapper, and reply setup drains the same queue into `System` lines.

## Solution Assessment

Yes. The linked marker-and-skip direction is the narrow maintainable fix because it suppresses only main-session cron events already injected into history while preserving normal cron reminder prompts.
