## Summary

Keep open: current main and the latest release still have the source path that can surface a main-session cron systemEvent twice, and the focused fix is still in the open, conflicting PR at https://github.com/openclaw/openclaw/pull/44973.

## What This Changes

Make main-session cron systemEvent jobs with `delivery.mode: "none"` avoid the extra scheduled-reminder heartbeat wrapper.

## Best Possible Solution

Rebase and validate https://github.com/openclaw/openclaw/pull/44973, or land an equivalent narrow marker/filter fix that skips only main-session injected cron systemEvents from reminder wrapping.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the cron service enqueues a main-session `systemEvent` with a `cron:` context key, heartbeat wraps that same pending event as a scheduled reminder, and reply setup drains the event into `System:` lines.

## Solution Assessment

Yes. The narrow maintainable solution is to distinguish main-session injected cron events, then exclude only those from cron reminder prompt construction while preserving ordinary cron reminder wrapping.
