## What This Changes

The issue reports that a main-session cron `systemEvent` with `delivery.mode: "none"` is surfaced both as a raw `System` event and as a scheduled-reminder heartbeat wrapper.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent narrow marker-and-skip fix with regression coverage and redacted real cron/heartbeat proof, while preserving ordinary cron reminder wrappers and `delivery.mode` semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron queues a `cron:` system event, heartbeat peeks that event into the cron wrapper, and reply setup can drain the same queue into `System` lines.

## Solution Assessment

Yes. The linked marker-and-skip direction is the narrow maintainable fix because it suppresses only main-session cron events already injected into history while preserving normal cron reminder prompts.
