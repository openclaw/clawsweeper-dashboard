## Summary

Keep open: current main still has the duplicate source path, and the existing narrow fix PR is open rather than merged.

## What This Changes

Main-session cron systemEvent jobs with delivery.mode none should not surface the same text both as a raw system event and as a scheduled-reminder heartbeat wrapper.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent marker-and-skip fix with regression coverage and real cron/heartbeat proof, while preserving normal cron reminder wrappers and delivery semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron queues a `cron:` system event, heartbeat peeks that event into the cron wrapper, and reply setup drains the same queue into `System` lines.

## Solution Assessment

Yes. The narrow maintainable fix is to identify main-session cron system events already injected into history and skip only those from cron reminder wrapping, instead of changing general delivery or heartbeat behavior.
