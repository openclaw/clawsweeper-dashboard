- `impact:session-state`: The reported failure duplicates internal cron text in the session transcript and model context, creating agent-state drift.

## Summary

Keep open: current main still has the source-level duplicate path for main-session cron system events, and https://github.com/openclaw/openclaw/pull/44973 remains an open, unmerged implementation candidate rather than a landed fix.

## What This Changes

The issue asks OpenClaw to avoid surfacing a main-session cron systemEvent twice, once as raw System context and again as a scheduled-reminder heartbeat wrapper, when delivery is disabled.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent marker-and-skip fix with focused regression coverage, preserving normal cron reminder wrapping and delivery semantics for other cron modes.

## Reproduction Assessment

Yes, at source level. Current main queues a main-session `cron:` system event, heartbeat wraps `cron:` pending events as scheduled reminders, and reply setup can drain the same event as `System:` context.

## Solution Assessment

Yes. The narrow maintainable fix is to mark main-session cron system events already injected into history and skip only those events from cron reminder wrapping, without changing broader delivery or heartbeat behavior.
