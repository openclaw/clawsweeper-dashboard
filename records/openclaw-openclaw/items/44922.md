## Summary

Keep open: current main still has the source-level duplication path, and the linked fix PR that targets it remains open rather than merged.

## What This Changes

The issue asks OpenClaw to avoid adding a heartbeat scheduled-reminder wrapper for main-session cron systemEvent jobs that already inject the same system event.

## Best Possible Solution

Rebase, validate, and land https://github.com/openclaw/openclaw/pull/44973 or an equivalent narrow marker-and-skip fix with regression coverage and real behavior proof.

## Reproduction Assessment

Yes, by source inspection: a main-session cron systemEvent is queued with a `cron:` context key, heartbeat peeks that same event into the cron wrapper, and reply setup drains it into `System` lines. I did not run a live cron schedule in this read-only review.

## Solution Assessment

Yes, the linked tag-and-skip direction is the narrow maintainable fix because it suppresses only main-session cron events already injected into history while preserving ordinary cron reminder prompts.
