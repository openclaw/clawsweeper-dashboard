## Summary

Keep open: current main and the latest shipped release still show the duplicate main-session cron system-event path, and the focused fix PR is open but unmerged.

## What This Changes

Suppress the redundant scheduled-reminder wrapper for main-session cron `systemEvent` jobs with `delivery.mode: "none"` so the session context sees the payload only once.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent targeted marker-and-skip fix with regression coverage, preserving normal cron reminder wrapping for non-main-session cron events.

## Reproduction Assessment

Yes, at source level. Current main enqueues a `cron:` main-session system event, heartbeat wraps `cron:` pending events as scheduled reminders, and reply setup can drain that same queue into model-visible `System:` lines.

## Solution Assessment

Yes. The narrow maintainable solution is to identify main-session cron system events that are already injected into history and skip only those from cron reminder wrapping, without changing broader delivery semantics.
