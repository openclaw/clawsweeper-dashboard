## What This Changes

The issue requests that main-session cron jobs with `systemEvent` payloads and `delivery.mode: "none"` inject the event once without adding the scheduled-reminder wrapper.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent marker-and-skip fix with regression coverage and redacted real cron/heartbeat proof, while preserving normal cron reminder wrappers and delivery semantics.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: main-session cron queues a `cron:` system event, heartbeat peeks and wraps it, and reply setup later drains the same queue into `System` lines.

## Solution Assessment

Yes. The narrow maintainable fix is to identify main-session cron system events that are already injected into history and skip only those from cron reminder wrapping, instead of changing general delivery or heartbeat behavior.
