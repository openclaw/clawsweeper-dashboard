## Summary

Keep open: current main still appears to have the duplicate main-session cron system-event path, and an open focused PR already targets the bug but has not merged.

## What This Changes

The issue reports that `sessionTarget: "main"` cron `systemEvent` jobs with `delivery.mode: "none"` add both the raw system event and a scheduled-reminder wrapper to the session context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent focused marker-and-skip fix with regression coverage, preserving normal cron reminder wrapping and delivery semantics for other cron modes.

## Reproduction Assessment

Yes, at source level. Current main still queues a main-session `cron:` system event, lets heartbeat wrap `cron:` pending events as scheduled reminders, and can also drain the same pending event into `System:` session context.

## Solution Assessment

Yes. The narrow maintainable fix is to mark main-session cron system events that are already injected into history and skip only those from cron reminder wrapping, without changing broader delivery or heartbeat semantics.
