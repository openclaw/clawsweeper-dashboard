## What This Changes

The issue asks OpenClaw to avoid adding the scheduled-reminder heartbeat wrapper when a main-session cron systemEvent with delivery.mode none is already injected as system context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent maintainer-chosen remediation such as https://github.com/openclaw/openclaw/pull/48883, with focused regression coverage and proof that normal cron reminder behavior remains intact.

## Reproduction Assessment

Yes, at source level. Current main queues a main-session `cron:` system event, heartbeat wraps cron-tagged pending events as scheduled reminders, and reply setup can drain the same event as `System:` context.

## Solution Assessment

Yes. The narrow maintainable fix is to mark main-session cron system events that are already injected into history and skip only those from cron reminder wrapping, unless maintainers intentionally choose the broader main-session system-event remediation instead.
