## Summary

Keep open: current main still has the source-level duplicate path for main-session cron system events, and the focused open PR is an implementation candidate rather than a merged fix.

## What This Changes

A main-session cron job with a systemEvent payload and `delivery.mode: "none"` should not also inject the scheduled-reminder wrapper into the same model context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent marker-and-skip fix with regression coverage, preserving normal cron reminder wrapping and delivery semantics for other cron modes.

## Reproduction Assessment

Yes, at source level. Current main queues a main-session `cron:` system event, heartbeat wraps `cron:` pending events as scheduled reminders, and reply setup can drain the same event as `System:` context.

## Solution Assessment

Yes. The narrow maintainable fix is to mark main-session cron system events already injected into history and skip only those events from cron reminder wrapping, without changing broader delivery or heartbeat behavior.
