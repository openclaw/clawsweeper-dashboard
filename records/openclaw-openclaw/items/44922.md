## Summary

Keep open: current main and the latest release still have the reported cron-to-heartbeat duplication path, and the targeted fix is still in an open PR rather than merged.

## What This Changes

The issue asks OpenClaw to suppress the scheduled-reminder heartbeat wrapper when a main-session cron job has `payload.kind: "systemEvent"` and `delivery.mode: "none"`.

## Best Possible Solution

Land, rebase, or replace the targeted fix in https://github.com/openclaw/openclaw/pull/44973, coordinating with https://github.com/openclaw/openclaw/pull/48883 if maintainers prefer the broader cron/main-session remediation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a main-session cron job enqueues a `cron:` system event, heartbeat peeks and wraps that event as a scheduled reminder, and reply setup later drains the same event into `System:` lines.

## Solution Assessment

Yes. The tag-and-filter approach in the linked fix PR is the narrow maintainable solution because it preserves normal cron reminders and delivery semantics while excluding only main-session system events that are already injected.
