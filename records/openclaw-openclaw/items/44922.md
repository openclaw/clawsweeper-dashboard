## Summary

Keep open: current main still has the source-reproducible duplicate model-context path, and the focused fix at https://github.com/openclaw/openclaw/pull/44973 is still open rather than merged or shipped.

## What This Changes

Suppress the redundant scheduled-reminder heartbeat wrapper for main-session cron `systemEvent` jobs with `delivery.mode: "none"` so the same event text is not surfaced twice.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44973 or an equivalent marker-and-skip fix with regression coverage and redacted real cron/heartbeat proof, while preserving normal cron wrapper and delivery semantics.

## Reproduction Assessment

Yes, source-reproducible. Current main enqueues a main-session cron system event under `cron:<job>`, heartbeat peeks and wraps the same text as a scheduled reminder, and reply setup can drain the queued event into `System:` lines.

## Solution Assessment

Yes. The narrow maintainable direction is to identify main-session cron system events already injected into history and skip only those from cron reminder wrapping, without changing broader delivery or heartbeat semantics.
