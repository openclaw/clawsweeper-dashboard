## Summary

Keep open: current main still has the reported cron-to-heartbeat duplication path, and the related targeted fix PR is open rather than merged.

## What This Changes

A main-session cron `systemEvent` with `delivery.mode: "none"` should inject only the raw system event and avoid an additional scheduled-reminder heartbeat prompt.

## Best Possible Solution

Review and land, rebase, or replace the #44973-style fix: tag main-session cron system events, exclude that tag from cron wrapper prompt construction, add regression coverage, and include a changelog entry if the fix is user-facing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a main-session cron job enqueues a `cron:` system event, then heartbeat reselects that event and builds the scheduled-reminder prompt; I did not execute a live cron run in this read-only review.

## Solution Assessment

Yes. The open #44973 approach is the narrow maintainable fix because it preserves delivery behavior while preventing only main-session-injected cron events from being wrapped a second time.
