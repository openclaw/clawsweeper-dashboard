The underlying cron cleanup bug is still valid and paired with https://github.com/openclaw/openclaw/issues/46369, but this PR is not currently safe to merge because it drops the removed-job hook snapshot, has in-flight run and queued run-log races, is conflicting, and still lacks contributor real behavior proof.

The PR adds asynchronous `cron.remove` cleanup for matching cron session-store entries and per-job run-log files, plus focused cron cleanup tests and small unrelated test/format cleanup.
