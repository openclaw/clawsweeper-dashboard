## Summary

Keep this PR open: the core bug is real and current main still leaves cron session/run-log artifacts on removal, but the proposed one-shot cleanup can be undone by a successful cron run that finishes after deletion, so it needs a focused repair before merge.

## What This Changes

The PR adds async cleanup after `cron.remove` to delete matching cron session-store entries and the per-job run log, adds cron cleanup tests, removes an unused agent model-compat test helper, and formats one gateway line.

## Best Possible Solution

Land a cron-owned cleanup path that removes deleted-job session records and run logs after removal and after any already-started run finishes, with coverage for idle removal, per-agent stores, canonical keys, and in-flight completion.

## Reproduction Assessment

Yes. Source inspection shows current main’s `cron.remove` removes only the job record, while cron isolated runs persist visible canonical session keys and gateway code appends run logs on `finished`.

## Solution Assessment

No. The proposed cleanup is directionally right for idle jobs, but it runs only once immediately after removal and can be undone by a successful in-flight cron run or a pending run-log append.
