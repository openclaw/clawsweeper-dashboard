## Summary

Keep open: current main still has the source-reproducible cron ghost-session/run-log gap, but this PR is not ready to merge because it drops the removed-event job snapshot, uses one-shot cleanup that can be undone by in-flight runs or pending log writes, and has no posted real behavior proof.

## What This Changes

The PR adds asynchronous cleanup of cron session-store entries and per-job run logs after `cron.remove`, with focused tests plus small unrelated test/format cleanup.

## Best Possible Solution

Repair or replace the branch with a narrow cron removal lifecycle fix that preserves removed-event snapshots, cleans deleted-job base and run session rows after active runs settle, drains pending run-log writes before deletion, and includes focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create or seed a cron session and run log, call `cron.remove`, and current main only removes the cron job record while session-store rows and run logs are separate artifacts.

## Solution Assessment

No. The cleanup direction is right, but the proposed branch is not the best merge path until it preserves hook payloads and coordinates cleanup with active runs and pending run-log writes.

  - confidence: 0.87

  - confidence: 0.84
