## Summary

Keep open: the PR targets a valid source-reproducible compaction alias bug, but it still resolves aliases before preserving existing literal bare compaction model overrides, which can silently change provider/model and auth-profile behavior on upgrade.

## What This Changes

The PR adds local alias resolution for bare `agents.defaults.compaction.model` values in embedded compaction and tests same-provider, cross-provider, and unmatched alias cases.

## Best Possible Solution

Preserve valid current-provider bare model IDs before alias routing, then resolve true aliases through the shared model-ref machinery with regression coverage for collisions, reported alias resolution, and cross-provider auth-profile dropping.

## Reproduction Assessment

Yes. Source inspection and the linked issue show current main and `v2026.6.5` forward a bare compaction alias as a literal model on the active provider, matching the `Unknown model: openai/gpt54mini` failure path.

## Solution Assessment

No. The target helper is the right layer, but the implementation should preserve literal current-provider bare model semantics before alias resolution to avoid upgrade-time rerouting.
