## Summary

Keep open: the PR fixes the reported alias path in the right general area, but the current patch still has a blocking compatibility issue because it can reinterpret existing bare compaction model overrides as unique configured-provider literals before alias resolution.

## What This Changes

The PR changes embedded auto-compaction target resolution so bare compaction model overrides can resolve configured aliases to canonical provider/model refs while preserving literal configured model-id precedence.

## Best Possible Solution

Resolve aliases through the shared model-selection helpers while preserving the shipped model-only compaction override contract unless maintainers explicitly choose a broader provider-inference upgrade.

## Solution Assessment

No. Reusing the shared alias resolver is the right owner boundary, but the provider-inference guard is broader than the shipped model-only compaction behavior and needs a narrower compatibility rule or explicit maintainer acceptance.
