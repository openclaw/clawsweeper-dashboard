## Summary

Keep this PR open: the reported compaction alias bug is valid and the updated terminal proof now exercises the production resolver, but the patch still has merge-blocking compatibility and duplicated-resolver issues.

## What This Changes

The PR changes `resolveEmbeddedCompactionTarget` so a bare `agents.defaults.compaction.model` value can be matched against `agents.defaults.models` aliases and returned as a canonical provider/model target.

## Best Possible Solution

Resolve non-colliding compaction aliases through the shared model-selection resolver while preserving shipped literal bare model overrides for the current provider, then cover alias, fallback, provider-change, and collision cases in focused tests.

## Reproduction Assessment

Yes, source-reproducible: current main returns a bare compaction override literally, and the linked issue supplies a concrete `Unknown model: openai/gpt54mini` auto-compaction log for that path.

## Solution Assessment

No. The PR fixes the sample alias case, but the best fix must reuse the shared model-selection resolver and preserve shipped bare model-only override behavior when a literal value collides with an alias.
