## Summary

Keep open: the bug-fix direction and helper-level proof are useful, but the patch is not merge-ready because it can silently reinterpret existing bare `agents.defaults.compaction.model` values and it reimplements model-alias resolution instead of using the shared model-selection contract.

## What This Changes

The PR adds a local compaction model alias resolver in `resolveEmbeddedCompactionTarget` plus tests for same-provider alias, cross-provider alias, and unmatched literal fallback.

## Best Possible Solution

Resolve true compaction aliases through the shared model-selection path after preserving existing current-provider literal model ids, keep provider-change auth-profile dropping, and add collision coverage before merge.

## Reproduction Assessment

Yes at source level: current main returns a bare compaction override as the model under the active provider, while shared model-selection helpers already resolve aliases through `buildModelAliasIndex` and `resolveModelRefFromString`. I did not run a live long-session compaction repro in this read-only review.

## Solution Assessment

No: the fix direction is right, but this branch is not the best implementation until it preserves existing bare literal semantics and reuses the shared model-selection resolver. The safer alternative is a narrow wrapper around the shared resolver with explicit collision tests.
