## Summary

Keep this open. Current main still lacks a general Control UI session-kind filter, and the linked implementation PR #73311 is open and unmerged with closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Add a Control UI session dropdown filter for session kinds such as main/direct, group, subagent, dreaming, cron, isolated, and other sessions.

## Best Possible Solution

Review and land PR #73311 or an equivalent focused Control UI patch that keeps filtering UI-local unless a protocol change is intentionally chosen, preserves the active session even when hidden by filters, covers desktop and mobile controls, and updates targeted UI tests/docs.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: the chat session dropdown renders without a general kind filter, and the only current filtering behavior is the cron-session hide toggle.

## Solution Assessment

Unclear until PR #73311 is reviewed. A UI-local filter appears to be the narrowest maintainable path against current main because the gateway protocol schema does not currently accept kinds.
