## Summary

Keep this issue open. Current main still only has the existing cron-hide toggle in the Control UI chat header/session dropdown, and the open unmerged PR #73311 is the active implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests a Control UI session dropdown filter that can hide or show sessions by kind, including main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Ship a focused Control UI filter, preferably by reviewing and landing #73311 or an equivalent patch, that classifies visible session rows, preserves the selected session even when hidden by filters, updates desktop/mobile UI consistently, and covers the behavior with UI tests and docs.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: the chat header renders only the session select/model/thinking controls plus the cron visibility toggle, and sessions loading never carries a general kinds filter.

## Solution Assessment

Unclear until #73311 is reviewed. A UI-local filter appears to be the narrowest maintainable fix against the current protocol; adding a server-side kinds parameter would need an explicit gateway schema and handler change.
