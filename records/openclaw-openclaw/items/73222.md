Keep this open. Current main still only has the existing cron visibility toggle in the Control UI chat session selector, and open PR #73311 is the unmerged implementation path that references this issue with closing syntax.

Add a Control UI session dropdown filter for session kinds such as main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Review and either land or close PR #73311; the desired end state is a Control UI chat-header session-kind filter that preserves the active session when hidden and keeps protocol changes explicit if maintainers want server-side filtering.
