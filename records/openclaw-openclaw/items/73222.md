## Summary

Keep this issue open. Current main still only has the existing cron-only Control UI session hiding path, and PR #73311 is open and unmerged with closing syntax for this issue, so the issue should stay open until that implementation candidate is reviewed, landed, or replaced.

## What This Changes

Add a session-kind filter to the Control UI chat-header session dropdown so users can hide or show noisy categories such as subagent, dreaming, cron, isolated, and main sessions.

## Best Possible Solution

Ship a focused desktop and mobile Control UI kind filter that classifies returned session rows, preserves the selected session even when its kind is hidden, updates focused UI/docs coverage, and only changes the gateway protocol if maintainers intentionally want server-side filtering.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main renders only the session select and cron visibility toggle, while the Control UI loader and gateway schema have no general kinds filter path.

## Solution Assessment

Unclear pending PR review. A UI-local filter appears to be the narrowest maintainable solution against the current protocol; server-side filtering would require an explicit additive schema and handler change.
