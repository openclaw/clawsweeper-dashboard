## Summary

Keep open because current main still only supports a hide-cron toggle in the Control UI chat session selector, not a general session-kind filter, and open PR #73311 references this issue with closing syntax but is unmerged.

## What This Changes

Add a Control UI session dropdown filter that can show or hide session kinds such as main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Review and either land or close PR #73311; the desired end state is a chat-header session-kind filter that preserves the active session when hidden and keeps any protocol change explicit.

## Reproduction Assessment

Yes. Source inspection on current main shows the chat session dropdown is filtered only by sessionsHideCron, and the Control UI sessions.list request path has no kinds parameter.

## Solution Assessment

Yes for the current tracking path. Keeping this issue open while PR #73311 is reviewed is the narrowest maintainer route because the remaining work is already paired with an open implementation PR.
