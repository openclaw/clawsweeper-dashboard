## Summary

Keep this open. Current main still has only the existing cron visibility toggle in the Control UI chat session selector, while the full main/subagent/dreaming/cron/other session-kind filter is already paired with open implementation PR #73311 that uses closing syntax for this issue.

## What This Changes

Request adding session-kind filtering to the Control UI session dropdown so users can hide or show main, subagent, dreaming, cron, isolated, or other session types.

## Best Possible Solution

Review and either land or close PR #73311. The desired end state is a UI-local Control UI chat-header session-kind filter that preserves the selected session even when its kind is hidden, updates user-facing docs/changelog as needed, and avoids a gateway protocol change unless maintainers explicitly want one.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main renders only a plain session select plus a cron-only hide toggle, and the Control UI sessions.list request cannot pass a kinds filter. I did not run tests because this review is read-only, but the inspected source and existing cron-only tests establish the behavior.

## Solution Assessment

Yes for the current path. Keeping this issue open while reviewing PR #73311 is the narrowest maintainable route; a UI-local filter fits the current gateway schema, while adding a gateway kinds parameter would require an explicit protocol change.
