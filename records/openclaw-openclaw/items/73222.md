## Summary

Keep this open. Current main still has only the existing cron visibility toggle in the Control UI session controls, not a general session-kind filter, and the linked implementation candidate #73311 is still open and unmerged with closing syntax for this issue.

## What This Changes

Request to add a Control UI session dropdown filter for session kinds such as main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Review and resolve #73311, or land an equivalent focused implementation that filters desktop and mobile Control UI session selectors by classified session kind, preserves the selected session even when hidden, and updates the relevant UI tests and docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: the Control UI renders only the session select and cron visibility toggle, while the session loader sends sessions.list without a general kinds filter.

## Solution Assessment

Unclear until the linked PR is reviewed. Given the current gateway schema lacks kinds, a UI-local filter appears to be the narrowest maintainable fix unless maintainers intentionally want an additive protocol change.
