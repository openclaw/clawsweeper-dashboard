## Summary

Keep this open. Current main still has only a cron visibility toggle around the Control UI chat session selector, while the broader main/subagent/dreaming/cron/other kind filter is being handled by the still-open linked PR #73311 that explicitly uses closing syntax for this issue.

## What This Changes

Add a Control UI session dropdown filter that lets users hide or show sessions by kind, including main/direct, subagent, dreaming, cron, isolated, and other sessions.

## Best Possible Solution

Land a reviewed UI-local session-kind filter in the Control UI chat header/dropdown, preserve the selected session even when its kind is hidden, keep the gateway protocol unchanged unless maintainers explicitly want a new API field, and let the linked PR close the issue after merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main renders the chat session selector with only cron/global/unknown filtering, and there is no general session-kind filter for subagent or dreaming sessions.

## Solution Assessment

Yes, with the open PR path rather than closing this issue now. Given the current gateway schema lacks kinds, a UI-local filter is the narrowest maintainable direction unless maintainers decide to expand the protocol.
