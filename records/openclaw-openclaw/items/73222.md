## Summary

Keep this open. Current main still has a plain Control UI chat session select plus a cron-only visibility toggle, while the linked implementation PR remains open and unmerged.

## What This Changes

Add a Control UI session-dropdown filter so users can show or hide sessions by kind, comparable to the agent `sessions_list` tool's kind filtering.

## Best Possible Solution

The maintainable end state is a chat-header kind filter with tests for hidden active sessions and an explicit choice between UI-local filtering and a documented gateway `kinds` parameter.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main renders only the plain session select and cron hide/show control, and the Control UI `sessions.list` request has no kind filter.

## Solution Assessment
