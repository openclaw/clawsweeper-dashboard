## Summary

Keep this issue open. Current main still lacks the requested multi-kind Control UI session dropdown filter, and the linked open PR #73311 is an active implementation candidate that references this issue, so the issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests a Control UI session dropdown filter for main, subagent, dreaming, cron, isolated, and related session kinds.

## Best Possible Solution

Land a focused Control UI filter from #73311 or an equivalent patch that preserves the selected session when hidden and updates the chat header docs/tests; if maintainers want server-side filtering later, add a separate gateway protocol extension intentionally.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main's chat header loads all visible sessions through `sessions.list` and only hides cron/global/unknown rows locally, so subagent and other noisy session rows remain in the dropdown.

## Solution Assessment

Yes. Keeping the issue open behind the linked PR is the right path; a UI-local filter is also consistent with the current gateway schema, which does not expose a `kinds` parameter even though the agent tool supports local kind filtering.
