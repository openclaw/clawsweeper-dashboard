## Summary

Keep this open. Current main still has only the existing cron-session hide toggle in the Control UI session dropdown, while the linked implementation PR #73311 is open and unmerged with closing syntax for this issue.

## What This Changes

Add a Control UI session-dropdown filter that lets users show or hide sessions by kind, including main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Ship a focused Control UI filter through #73311 or an equivalent patch, keeping the active session visible even when hidden by filters, covering desktop and mobile controls, and adding targeted UI tests plus docs for the new dropdown behavior.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: the chat session dropdown renders without a general kind filter, and the only existing filtering behavior is the cron-session hide toggle.

## Solution Assessment

Unclear until #73311 is reviewed. A UI-local filter appears to be the narrowest maintainable path against current main because the gateway protocol does not currently accept `kinds`.
