## Summary

Keep this open. Current main still lacks the requested general session-kind filter in the Control UI session dropdown, and the open implementation PR #73311 references this issue with closing syntax, so the issue should remain open until that PR is merged or explicitly rejected.

## What This Changes

Request adding a Control UI session-dropdown filter for main, subagent, dreaming, cron, isolated, and other session kinds to reduce noisy session lists.

## Best Possible Solution

Land a reviewed UI-local session-kind filter in the Control UI chat header/dropdown, preserve the selected session even when its kind is hidden, document the new behavior, and avoid changing the gateway protocol unless maintainers explicitly want a new API field.
