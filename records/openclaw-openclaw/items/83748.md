## Summary

Keep open: current main and the latest release still have Telegram sticker paths that can be downloaded but then omitted from agent-readable media context, and the prompt-context path still exposes only `telegram:file/...` refs without a hydrated workspace path.

## What This Changes

Telegram inbound stickers should be hydrated into agent-readable media paths rather than only appearing as Telegram provider file refs.

## Best Possible Solution

Preserve static Telegram sticker files as normal inbound media through current-message, reply-chain, prompt-context, and sandbox-staging paths while keeping animated/video sticker policy unchanged unless maintainers approve broader support.

## Reproduction Assessment

Yes, at source level: current main can download static sticker media but still has code paths that omit the saved path from `MediaPaths` and prompt context. I did not run a live Telegram sticker in docker/all during this read-only review.

## Solution Assessment

Yes: the maintainable fix belongs in the Telegram inbound media/context boundary, not in the image tool. The fix should keep the downloaded sticker path available anywhere the sticker is shown to the agent.
