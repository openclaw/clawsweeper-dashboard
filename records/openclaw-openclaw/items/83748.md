Keep open: current main still has Telegram sticker paths where cached or structured context can omit agent-readable MediaPaths while prompt context only carries the Telegram file ref, and I found no merged or closing fix for this issue.

## What This Changes

Telegram direct-chat sticker messages should hydrate into workspace-accessible inbound media paths instead of only provider file refs.

## Best Possible Solution

Preserve static Telegram sticker files as normal inbound media through current-message, reply-chain, prompt-context, and sandbox-staging paths, while keeping animated/video sticker policy unchanged unless maintainers separately expand support.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection gives a clear current-main path: sticker media can be pruned or omitted from MediaPaths while prompt context still exposes only `media_ref`.

## Solution Assessment

Yes, the best fix is in the Telegram inbound media/context boundary, not in the image tool. The repair should keep static sticker MediaPaths available for sandbox staging and prompt visibility without changing animated/video sticker policy.
