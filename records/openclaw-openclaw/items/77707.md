## Summary

Keep open: current main still has a source-level mismatch between daily-ingestion snippets and promotion rehydration for heading/list chunks, although the threshold half of the report is partly mitigated by current signal-count and recall-day logic.

## What This Changes

The issue asks for short-term memory promotion to successfully rehydrate dreaming-generated candidates and for default promotion gates to be reachable by normal recurring dreaming signals.

## Best Possible Solution

Fix promotion rehydration to normalize internally generated daily chunks consistently with ingestion, add regression coverage for heading-prefixed list chunks, and leave default thresholds unchanged unless a follow-up proves they still block valid candidates.

## Reproduction Assessment

Yes, source-reproducible: a meaningful heading plus Markdown list chunk is stored as a heading-prefixed, semicolon-joined snippet, but rehydration compares it to raw list-marker lines. I did not run a write-based fixture because this review was read-only.

## Solution Assessment

No, this should not be closed as-is. The narrow maintainable fix is to align rehydration normalization with daily-ingestion normalization and test that apply promotes those generated chunks; lowering thresholds should be avoided unless still failing after that fix.
