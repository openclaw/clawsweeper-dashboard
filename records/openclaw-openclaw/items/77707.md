## Summary

Current main still has a concrete memory-core rehydration gap for heading-prefixed daily snippets, so this should stay open as a valid bug. The threshold half is less certain because current ranking already counts total signals and recall-day diversity, and an existing test expects repeated dreaming-only daily signals to clear default gates.

## What This Changes

The issue asks for short-term memory promotion to work for dream-generated candidates whose stored snippets include heading/list context and for default promotion thresholds to be reachable from normal dreaming signals.

## Best Possible Solution

Fix memory-core’s rehydration comparison so heading-prefixed/list-joined daily snippets can relocate to their live daily-note source lines, then revisit threshold behavior only if a focused test proves it still blocks real candidates.

## Reproduction Assessment

Yes from source, but not from an executed command: daily ingestion can store `Heading: body; body` snippets while apply rehydration only accepts exact/includes matches against live body lines. That makes the heading-prefixed apply failure source-reproducible on current main.

## Solution Assessment

Yes for the rehydration half if kept narrow: normalize or compare against a body-only/list-normalized form inside the memory-core matcher and cover it with a focused apply test. Lowering default thresholds is not clearly the best first fix because current code already uses signal count and recall-day diversity.
