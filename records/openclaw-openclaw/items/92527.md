## Summary

Keep this PR open, but the current patch should not merge because it makes OpenClaw depend on `gog gmail watch serve` flags that are not present in the current gogcli command contract, so existing Gmail watcher installs can fail to start instead of delivering pushes.

## What This Changes

The PR changes Gmail watcher startup to write the push token and hook token into private temp files and pass `--token-file`/`--hook-token-file` to `gog gmail watch serve`, with tests for argv contents and file permissions.

## Best Possible Solution

Keep the security goal, but first make the secret-file inputs a supported gogcli contract or add a version/capability-gated OpenClaw upgrade path that preserves Gmail delivery.

## Reproduction Assessment

Yes. Source inspection shows current main passes Gmail watch secrets on argv, and the PR switches the startup path to flags missing from current gogcli Gmail watch serve source; I did not run a live gog process.

## Solution Assessment

No. The security direction is right, but the best fix needs gogcli support or an OpenClaw capability/version gate before the Gateway auto-start path depends on new serve flags.
