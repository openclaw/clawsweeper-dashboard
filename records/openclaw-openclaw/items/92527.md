## Summary

Keep open: the underlying raw-argv secret exposure is still present on current main and the latest release, but this PR is not merge-ready because it switches OpenClaw to unsupported gogcli flags, leaves the documented foreground runner unprotected, lacks inspectable real-environment proof, and can leave plaintext token files behind.

The PR changes Gmail watcher startup to write Gmail push and hook tokens to temporary files and pass those file paths to `gog gmail watch serve`, with mocked unit coverage for the new argv shape.

## Best Possible Solution

Coordinate or release compatible gogcli file-secret support first, then land a cleanup-safe OpenClaw implementation that covers both gateway auto-start and foreground Gmail runner paths with inspectable redacted live proof.

## Solution Assessment

No. File-backed secrets are the right direction, but this implementation is not the best fix until the gogcli contract exists, cleanup is failure-safe, the foreground runner shares the protection, and real behavior proof is inspectable.
