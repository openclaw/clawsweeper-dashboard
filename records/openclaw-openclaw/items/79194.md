## Summary

Keep open. Current main does not implement an operator-grant gate; the existing controls are `update.auto.*` default-off behavior, startup hints, manual `update.run`, and `OPENCLAW_NO_AUTO_UPDATE`, so this needs product/config design rather than cleanup closure.

## What This Changes

Add a configuration and approval workflow that holds platform auto-upgrades until an operator explicitly grants the upgrade.

## Best Possible Solution

Design the approval gate in the core update subsystem by extending `update.auto` policy, update availability events/status, operator notification/approval commands, and audit records while preserving the existing default-off and emergency kill-switch behavior.

## Reproduction Assessment

Not applicable; this is a new feature/config request rather than a bug with an existing expected behavior. Source review confirms current main auto-applies when configured and due, but has no operator-grant path.

## Solution Assessment

Unclear; an operator grant is a plausible direction, but the exact defaults, notification channels, approval identity, audit trail, and security-update exceptions need maintainer design before implementation.
