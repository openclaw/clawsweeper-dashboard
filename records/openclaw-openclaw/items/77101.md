## Summary

Current main already addresses the reported npm-plugin coexistence failure by preserving existing managed npm dependencies before running npm install, so installing a second plugin should no longer cause npm to prune the first as extraneous.

## What This Changes

Prevent installing one npm-based plugin, such as `@martian-engineering/lossless-claw`, from removing a previously installed npm-based plugin, such as `@openclaw/feishu`.

## Best Possible Solution

Keep the current managed-npm-root behavior and ship it in the next release so users affected on v2026.5.2 can upgrade to a release containing current main.

## Reproduction Assessment

No current-main failure path was established. Current source merges existing managed npm dependencies before npm install, and the regression test covers preserving an existing plugin dependency when adding another; I did not run a live install because this review is read-only.

## Solution Assessment

Yes. Keeping all managed npm plugin packages in the root `package.json` before invoking npm is the narrow maintainable fix because npm will not prune previously installed plugins as extraneous.
