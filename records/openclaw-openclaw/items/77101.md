## Summary

Current main has the managed npm-root install behavior needed for these plugins to coexist, and the same sibling-plugin pruning root cause is already recorded as fixed in the Unreleased changelog; the fix is not in the v2026.5.2 tag yet.

## What This Changes

The issue reports that installing `@openclaw/feishu` and `@martian-engineering/lossless-claw` on v2026.5.2 prunes the previously installed plugin from the managed npm root.

## Best Possible Solution

Keep the current main managed npm-root dependency-ledger behavior and ship it in the next release so npm-installed plugins can coexist reliably.

## Reproduction Assessment

Yes. The reporter provided concrete v2026.5.2 steps, and current main has source-level regression coverage for installing two npm plugins into the same managed root while preserving both package directories.

## Solution Assessment

Yes. The current fix is the narrow maintainable solution: keep the managed npm root's package manifest as the source of truth before running npm, rather than adding plugin-specific cleanup or special-casing Feishu/lossless-claw.
