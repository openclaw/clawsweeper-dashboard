## Summary

Current main fixes the reported plugin co-installation failure by installing from the managed npm-root manifest instead of asking npm to install only the newest package, which is the same root cause resolved by merged PR #76602.

## What This Changes

The issue reports that installing `@openclaw/feishu` and `@martian-engineering/lossless-claw` in either order removes the previously installed plugin from the shared plugin `node_modules` tree.

## Best Possible Solution

Keep the manifest-based npm install path on main and ship it in the next release so mixed official and third-party npm plugins can coexist in the managed plugin root.

## Reproduction Assessment

Yes. The reported `v2026.5.2` behavior matches the already-fixed #76571 path where sequential npm plugin installs pruned siblings; current main no longer passes the package spec to npm and has regression coverage for sequential installs.

## Solution Assessment

Yes. Installing from the managed manifest is the narrow maintainable fix because the manifest helper preserves all existing dependency names, including third-party packages such as `@martian-engineering/lossless-claw`.
