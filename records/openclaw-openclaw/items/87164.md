## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Clean backup sidecars after failed reacquire:** `src/agents/pi-embedded-runner/run/attempt.session-lock.ts:1017-1018`
  - body: The sidecar is removed only after `assertSessionFileFence()` succeeds. If that check throws for a valid takeover or failed restore, the catch releases the lock and rethrows while `backupPath` stays set; `dispose()` then returns early because `heldLock` is undefined, leaving a full session-file copy on disk.
  - confidence: 0.94
- **[P2] Discover the generated backup path in tests:** `src/agents/pi-embedded-runner/run/attempt.session-lock.test.ts:1308`
  - body: Production creates `${sessionFile}.prompt-bak-${...}`, but the new tests look for `${sessionFile}.prompt-bak`. Those stat/unlink assertions target a path that is never created, so the regression tests fail or stop before exercising restore cleanup.
  - confidence: 0.96
- **[P2] Write corrupt JSONL in the restore test:** `src/agents/pi-embedded-runner/run/attempt.session-lock.test.ts:1328`
  - body: The implementation restores only when parsing the current file throws, but this test appends a valid JSONL row and still expects recovery. It will take the protected valid-unowned-write path instead of proving corrupt-file recovery.
  - confidence: 0.94
