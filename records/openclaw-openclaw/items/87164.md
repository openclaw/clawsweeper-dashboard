## Summary

Keep open for review, but this PR should not merge as-is: the proposed rollback path can discard unowned session-file writes that current main deliberately treats as a takeover, and the external-PR proof is mock-only.

## What This Changes

The PR adds a `.prompt-bak` backup/restore path around embedded attempt session-lock release and reacquire, plus tests that expect restore after external session-file changes.

## Best Possible Solution

Keep the ownership fence fail-closed for unowned external writes; if recovery is still desired, design a targeted path that never overwrites unowned session data, uses collision-safe cleanup, and proves a real embedded retry run.

## Reproduction Assessment

Yes for the patch defect: source inspection shows the PR restores a backup after any non-benign fence mismatch, while current main tests define that same external edit as a takeover. I did not run the patched tests because this review is read-only.

## Solution Assessment

No. The proposed restore path is not the safest fix because it overwrites unowned session changes; a maintainable solution needs a narrower recovery design or to preserve the current takeover failure mode.
