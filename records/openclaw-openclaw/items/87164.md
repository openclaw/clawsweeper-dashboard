## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not restore over unowned session writes:** `src/agents/pi-embedded-runner/run/attempt.session-lock.ts:738-744`
  - body: This branch runs only after the current file failed the owned-write and benign-change checks, so copying `.prompt-bak` back to `sessionFile` discards the other writer's session records and then trusts the stale state. Current main intentionally treats that path as a takeover instead of rolling it back.
  - confidence: 0.94
- **[P2] Use a controller-owned backup artifact:** `src/agents/pi-embedded-runner/run/attempt.session-lock.ts:617-620`
  - body: Every controller for the same session file uses `${sessionFile}.prompt-bak`, so concurrent embedded attempts can overwrite each other's backup or unlink it before another controller restores. The current fence code supports multiple controllers by tracking per-generation ownership; the backup needs the same isolation or it can restore the wrong contents.
  - confidence: 0.87
