## What This Changes

Adds a bounded in-flight and 5-second TTL cache for `listSessionFilesForAgent`, invalidated by transcript updates, plus regression tests for coalescing, TTL refresh, invalidation, failed scans, and ENOENT caching.

## Best Possible Solution

Land this only after maintainers explicitly accept the 5-second out-of-process freshness bound and keep the companion failed-scan pruning guard tracked or landed separately.

## Reproduction Assessment

Yes for the source-level performance behavior: current main performs an uncached `readdir` on each `listSessionFilesForAgent` call, and the PR tests exercise the proposed coalescing/cache semantics. I did not reproduce the raw NFS syscall reduction in a live NFS setup.

## Solution Assessment

Yes, as an acceptable mitigation rather than a risk-free fix: centralizing the cache in `listSessionFilesForAgent` is the right owner boundary and avoids scattered caller caches. The safer alternative is to pause if maintainers cannot accept any stale listing window.
