## Summary

Keep this maintainer PR open: the coalescing/cache direction is useful, but the patch currently turns any failed session-directory scan into a cached empty snapshot, which is too risky for the NFS/session-state path it changes.

## What This Changes

The branch adds an in-flight plus short-TTL cache for `listSessionFilesForAgent`, invalidates it from transcript update events, and adds focused cache and chaos tests.

## Best Possible Solution

Keep the coalescing and event invalidation approach, but make failed directory scans non-cacheable or retryable before landing, then explicitly accept the remaining out-of-process TTL freshness tradeoff.

## Reproduction Assessment

Yes, from source inspection: make `fs.readdir` reject once and then succeed; the PR helper converts the rejection to `[]` and the cache publishes that empty value until the TTL expires. I did not run tests in this read-only sweep.

## Solution Assessment

No, not yet: coalescing plus event invalidation is a plausible owner-boundary fix, but failed directory scans should not become cacheable snapshots before the PR is merge-ready.
