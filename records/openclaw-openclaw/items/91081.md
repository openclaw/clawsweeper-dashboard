## What This Changes

The PR adds an in-flight, 1-second snapshot cache around memory host session-file directory listings and adds regression tests for coalescing, TTL refresh, event invalidation, transient scan failure, and absent-directory behavior.

## Best Possible Solution

Land the cache only after maintainers explicitly accept the 1-second session-list freshness tradeoff, align the PR body with the latest code, and keep the destructive failed-scan prune fix tracked in the linked follow-up.

## Reproduction Assessment

Yes, from source: current main calls `fs.readdir` inside `listSessionFilesForAgent` on every call, and several memory/session consumers share that helper. I did not run an NFS syscall-level repro in this read-only review.

## Solution Assessment

Yes, if maintainers accept the tradeoff: a bounded helper-local cache with event invalidation is the narrowest implementation point for the repeated scans. The safer alternative for stricter freshness would be no TTL cache or an explicit owner-approved freshness contract backed by NFS measurement.
