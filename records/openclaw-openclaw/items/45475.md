## Summary

Keep open: current main still treats Slack `message_changed` and `message_deleted` as session-scoped system events, and there is still no public `ignoreSubtypes` Slack config key. The request is valid but it adds a new config surface and needs maintainer approval on scope and Slack Assistant edit-event ordering before implementation.

## What This Changes

The issue requests a Slack configuration key that drops selected Slack message subtypes before they wake an agent session.

## Best Possible Solution

Add an opt-in Slack plugin config field, validate and document it, and filter matching edit/delete subtype system events before session enqueue while preserving the intended Slack Assistant DM path.

## Reproduction Assessment

Yes, at source level. Current main resolves Slack edit/delete subtype handlers and enqueues system events, while type, schema, docs, and search checks show no `ignoreSubtypes` config key.

## Solution Assessment

Yes, an event-layer opt-in filter is the narrow technical direction if maintainers approve the public config shape. The safer next step is maintainer review of account versus per-channel scope and Assistant edit-event ordering before a fix PR.
