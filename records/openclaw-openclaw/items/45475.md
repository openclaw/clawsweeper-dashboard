## Summary

Keep open: current main still routes Slack `message_changed` and `message_deleted` through the subtype system-event path, and there is still no `ignoreSubtypes` Slack config key. The prior discussion matches current source; this is a narrow but public Slack plugin config feature, not an implemented cleanup close.

## What This Changes

Request to add a Slack config key that drops configured `message_changed` and `message_deleted` subtype events before they wake an agent session.

## Best Possible Solution

Add an opt-in Slack plugin config field, validate and document it, and filter matching edit/delete subtype system events before authorization/session wake while preserving default behavior.

## Reproduction Assessment

Yes. Source inspection shows Slack `message_changed` and `message_deleted` events resolve through `resolveSlackMessageSubtypeHandler` and can enqueue system events; current tests assert that path for allowed edit events.

## Solution Assessment

Yes, the requested event-layer filter is the narrowest maintainable direction, but the public config shape and Assistant edit-event semantics need maintainer sign-off before implementation.
