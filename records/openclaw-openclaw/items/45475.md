## Summary

Keep open: current `main` and the latest release still map Slack `message_changed` and `message_deleted` into system events, and the requested `ignoreSubtypes` Slack config surface does not exist. The remaining blocker is a maintainer product/config decision, not lack of source evidence.

## What This Changes

Request to add a Slack `ignoreSubtypes` config key that drops selected Slack message subtypes before they enqueue system events and wake agent sessions.

## Best Possible Solution

Maintainers should decide the Slack subtype policy; if opt-in suppression is approved, add plugin-owned Slack schema/types/docs/tests and apply the filter after Assistant DM rehydration but before subtype enqueueing.

## Reproduction Assessment

Yes for source reproduction: current main resolves Slack `message_changed` and `message_deleted` to subtype handlers and enqueues system events before normal message handling. A live Slack workspace repro was not run in this read-only review.

## Solution Assessment

Unclear until maintainers choose whether edit/delete system events should be suppressible through a new config key or changed by default. If approved, the narrowest maintainable implementation is an event-layer Slack plugin filter with schema, docs, and regression tests.
