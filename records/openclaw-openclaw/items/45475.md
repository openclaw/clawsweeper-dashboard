## Summary

Keep open: current main and the latest release still intentionally map Slack `message_changed` and `message_deleted` events into session-scoped system events, and the requested `ignoreSubtypes` config key is not present. The remaining question is maintainer product direction for a new public Slack config surface, not whether the source path exists.

## What This Changes

Add Slack `channels.slack.ignoreSubtypes` configuration so `message_changed` and `message_deleted` events can be dropped before they wake an agent session.

## Best Possible Solution

Decide Slack edit/delete event policy explicitly; if accepted, add a narrow Slack plugin config field with schema, docs, and regression tests around the event-layer short-circuit.

## Reproduction Assessment

Yes. Source inspection shows ordinary Slack edit/delete events resolve a subtype handler and enqueue a session-scoped system event on current main; I did not run a live Slack Socket Mode reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed `ignoreSubtypes` key is a plausible narrow opt-in path, but it is not the best fix until maintainers choose the desired Slack edit/delete event policy and config surface.
