## Summary

Keep open: current main and v2026.6.6 still map Slack `message_changed` and `message_deleted` into system events, and the requested `ignoreSubtypes` Slack config surface is not present. The remaining decision is product/config direction, not whether the source behavior exists.

## What This Changes

Add a Slack `ignoreSubtypes` channel config key so configured `message_changed` and `message_deleted` events are dropped before waking an agent session.

## Best Possible Solution

Maintainers should decide whether Slack edit/delete events remain system events by default or gain opt-in suppression, then implement the accepted path after Assistant DM rehydration with schema, docs, and focused subtype tests.

## Reproduction Assessment

Yes for source reproduction: current main resolves Slack `message_changed` and `message_deleted` to subtype handlers and enqueues system events before regular message handling. A live Slack workspace repro was not run in this read-only review.

## Solution Assessment

Unclear until maintainers accept the new config surface or choose a default behavior change. If `ignoreSubtypes` is approved, the narrowest maintainable fix is an event-layer filter after Assistant DM `message_changed` rehydration and before subtype system-event enqueue.
