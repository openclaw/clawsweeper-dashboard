The remaining blocker is maintainer product direction on whether to add another Slack config key versus changing the default Slack system-event behavior.

## What This Changes

Request to add a Slack `channels.slack.ignoreSubtypes` configuration so selected Slack message subtypes are dropped before waking agent sessions.

## Best Possible Solution

Maintainers should decide the Slack product contract first, then implement the accepted path in the Slack plugin schema, monitor context, event filter, docs, and focused subtype tests while preserving `thread_broadcast` as a normal user message.

## Reproduction Assessment

Yes for source reproduction: current main resolves Slack `message_changed` and `message_deleted` to subtype handlers and enqueues system events before normal message handling. A live Slack workspace repro was not run in this read-only review.

## Solution Assessment

Unclear until maintainers accept the new config surface. The proposed event-layer filter is the narrowest implementation if `ignoreSubtypes` is approved, but changing default edit/delete behavior or declining the knob are product choices.
