## Summary

Keep open: current main and the latest release still route Slack `message_changed` and `message_deleted` events into system-event enqueue, and there is still no `ignoreSubtypes` config surface. This is a valid opt-in Slack config feature, but it needs maintainer product/API direction before implementation rather than conservative cleanup closure.

## What This Changes

The issue asks for a Slack channel `ignoreSubtypes` config key to drop selected Slack subtype events before they wake an agent session.

## Best Possible Solution

Add a maintainer-approved, opt-in Slack subtype filter in config, schema, docs, and the event layer after Assistant DM safeguards but before edit/delete subtype system-event enqueue.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.20` route `message_changed` and `message_deleted` through subtype system-event enqueue, and the strict Slack config schema does not accept `ignoreSubtypes`.

## Solution Assessment

Unclear for the exact public API until maintainers decide config scope and filter ordering. The maintainable direction is an opt-in event-layer filter, not unconditional suppression of all edit/delete events.
