## Summary

Keep open: current main and the latest release still route Slack `message_changed` and `message_deleted` through the subtype system-event path, and the requested `ignoreSubtypes` config key is absent. The request is valid, but it adds a new public Slack config surface and needs maintainer/product approval for config scope and Slack Assistant `message_changed` ordering.

## What This Changes

The issue requests a Slack channel config key that drops configured Slack message subtypes before they wake an agent session.

## Best Possible Solution

Add an owner-approved, opt-in Slack subtype filter in the Slack config/schema/docs and event layer, preserving valid Slack Assistant DM `message_changed` handling and keeping `thread_broadcast` on the normal message path.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.12` still route `message_changed` and `message_deleted` through subtype system-event enqueue, while focused config searches show no `ignoreSubtypes` key.

## Solution Assessment

Yes on direction, but not yet as an automated fix. An event-layer opt-in filter is narrow, but the public config shape and Slack Assistant `message_changed` ordering need maintainer approval first.
