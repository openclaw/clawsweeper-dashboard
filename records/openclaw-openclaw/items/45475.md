## Summary

Keep open. Current main still routes Slack `message_changed` and `message_deleted` through the subtype system-event path, while the requested opt-in `channels.slack.ignoreSubtypes` key is absent from the public config types, strict schema, docs, and focused tests.

## What This Changes

Add an opt-in Slack config key so configured edit/delete subtypes are dropped before waking an agent session.

## Best Possible Solution

Ship a Slack-owned, opt-in ignore list for supported metadata subtypes with unchanged defaults, strict validation, docs, config metadata, and regression coverage.

## Reproduction Assessment

Yes. Source and focused tests show current main enqueues system events for Slack edit/delete subtypes, and the proposed config key is absent from strict config validation.

## Solution Assessment

Yes. Filtering configured subtypes in the Slack event layer before system-event authorization/enqueue is the narrowest maintainable solution; prompt instructions and hard-coded suppression are weaker alternatives.
