## Summary

Current main intentionally converts authorized Slack message edits and deletions into session-scoped system events, and the only prior broad-ignore PR was closed because it would also block valid Slack Assistant DM handling; this issue remains a valid but product-sensitive request for an opt-in suppression policy.

## What This Changes

Add an opt-in Slack configuration policy that suppresses selected message edit/delete subtype events before they resolve or wake an agent session.

## Best Possible Solution

Define a typed Slack edit/delete system-event policy at the Slack plugin boundary, preserve the Assistant DM conversion before filtering, and apply suppression before authorization/session resolution with schema, documentation, and focused event tests.

## Reproduction Assessment

Yes, from source with high confidence: current main routes authorized Slack edit/delete events into session-scoped system events, although no live Slack workspace reproduction was performed during this read-only review.

## Solution Assessment

No, not exactly as proposed: filtering before session resolution is the correct boundary, but a typed edit/delete policy is safer than an arbitrary subtype string list and must preserve the existing Slack Assistant DM exception.
