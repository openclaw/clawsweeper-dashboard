## Summary

Keep open: current main and the latest release still route ordinary Slack `message_changed` and `message_deleted` subtypes into authorized system events, and the requested `ignoreSubtypes` Slack config surface does not exist. The source path is clear, but the remaining choice is a maintainer product/config decision, not an automatic strict bug fix.

## What This Changes

The issue requests a Slack channel config key that drops selected Slack message subtypes, especially edit/delete events, before they wake an agent session.

## Best Possible Solution

Choose the intended Slack policy, then either keep edit/delete as authorized system events or add a narrowly documented opt-in ignore list owned by the Slack plugin config surface.

## Reproduction Assessment

Yes from source inspection: ordinary Slack edit/delete events reach the subtype handler and can enqueue a session system event. I did not run a live Slack Socket Mode reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed config key is plausible, but it adds a public Slack configuration surface, so the best path is for maintainers to choose the product policy before a fix PR is queued.
