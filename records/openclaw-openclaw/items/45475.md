## Summary

Keep open: current main and the latest release still enqueue Slack `message_changed` and `message_deleted` as system events, and there is no supported `ignoreSubtypes` config key. This is a valid opt-in Slack configuration feature, but the public config shape and event-filter ordering need maintainer product/API direction before implementation.

## What This Changes

Add a Slack per-channel `ignoreSubtypes` config key so configured edit/delete subtypes can be dropped before waking agent sessions.

## Best Possible Solution

Add a maintainer-approved, opt-in Slack subtype filter in config, schema, docs, and the event layer after Assistant DM safeguards but before edit/delete subtype system-event enqueue.

## Reproduction Assessment

Yes, at source level. Current main recognizes `message_changed` and `message_deleted` handlers, the event tests still expect subtype system events to enqueue, and the strict Slack config schema does not accept `ignoreSubtypes`.

## Solution Assessment

Unclear for the exact public API until maintainers decide config scope and filter ordering. The maintainable direction appears to be an opt-in event-layer filter, not unconditional suppression of all edit/delete events.
