## Summary

Keep open: source inspection confirms Slack `message_changed` and `message_deleted` still flow through the subtype system-event path, and the requested `ignoreSubtypes` Slack config key is not present on current main or the latest release. This is valid feature work, but it needs owner approval because it adds public config surface and intersects with Slack Assistant `message_changed` handling.

## What This Changes

Add a Slack channel `ignoreSubtypes` config option to drop selected Slack message subtypes before they wake an agent session.

## Best Possible Solution

Add an owner-approved, opt-in Slack subtype filter in Slack config, schema, docs, and the event layer, preserving valid Slack Assistant `message_changed` handling and keeping the already-fixed `thread_broadcast` path as normal message input.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.12` still route `message_changed` and `message_deleted` through subtype system-event enqueue, while focused config searches show no `ignoreSubtypes` key.

## Solution Assessment

Unclear for the exact public API until an owner decides the config scope and ordering. The narrow maintainable direction is an opt-in event-layer filter placed after Slack Assistant `message_changed` rehydration safeguards.
