## Summary

Keep open: current main still intentionally maps Slack `message_changed` and `message_deleted` into session-scoped system events, and the requested `ignoreSubtypes` config key is still absent. This is a narrow Slack plugin config feature, but it needs maintainer sign-off on public config shape and whether ignored `message_changed` should also affect the special Slack Assistant DM rehydration path.

## What This Changes

The issue requests a Slack config option to ignore selected Slack message subtypes before they enqueue system events or wake sessions.

## Best Possible Solution

Add an opt-in Slack plugin config field, validate and document it, and filter matching edit/delete subtype system events before authorization/session enqueue while preserving default behavior and the intended Assistant DM path.

## Reproduction Assessment

Yes at source level. Current code and tests show Slack edit/delete subtypes resolve through the metadata system-event path and enqueue system events, while exact search shows no `ignoreSubtypes` config key.

## Solution Assessment

Yes, an event-layer opt-in filter is the narrowest maintainable direction. The exact public config location and whether it applies to Slack Assistant `message_changed` rehydration need maintainer approval first.
