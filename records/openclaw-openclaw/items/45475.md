## Summary

Keep open: current main and the latest release still map Slack edit/delete subtypes into system events, and the requested `ignoreSubtypes` config surface does not exist. The issue is valid, but it is a new opt-in Slack config/product decision rather than conservative cleanup closure.

## What This Changes

Request an opt-in Slack channel config key to drop configured Slack message subtypes, especially `message_changed` and `message_deleted`, before they enqueue system events or wake sessions.

## Best Possible Solution

Add a maintainer-approved, opt-in Slack subtype filter in Slack config, schema, docs, and the event layer after Assistant DM safeguards but before subtype system-event enqueue.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.20` route `message_changed` and `message_deleted` through subtype system-event enqueue, and the strict Slack config schema does not accept `ignoreSubtypes`.

## Solution Assessment

Unclear for the exact public API until maintainers decide config scope and filter ordering. The maintainable direction is an opt-in event-layer filter placed after Slack Assistant `message_changed` safeguards and before enqueueing subtype system events.
