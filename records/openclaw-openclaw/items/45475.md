## Summary

Keep open: current main still routes Slack `message_changed` and `message_deleted` through the Slack system-event path, and the requested `ignoreSubtypes` config surface is still absent. The request is valid, but it adds a new public Slack config option and needs maintainer/product sign-off on config scope and Slack Assistant `message_changed` ordering before a fix PR is safe.

## What This Changes

The issue requests an opt-in Slack config field to drop selected message subtypes before they wake an agent session.

## Best Possible Solution

Add an owner-approved, opt-in Slack subtype filter in the plugin config/schema/docs and event layer, with explicit ordering that preserves valid Slack Assistant DM `message_changed` handling and keeps `thread_broadcast` out of the default ignore set.

## Reproduction Assessment

Yes, at source level. Current code and tests show edit/delete subtypes can enqueue Slack system events, while schema/type/search checks show no `ignoreSubtypes` config key on current main.

## Solution Assessment

Yes, the event-layer opt-in filter is the narrowest maintainable direction, but the exact public config location and Assistant `message_changed` ordering need maintainer approval before implementation.
