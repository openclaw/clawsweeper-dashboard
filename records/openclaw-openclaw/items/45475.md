## Summary

Keep open: current main and v2026.6.10 still map ordinary Slack `message_changed` and `message_deleted` subtypes into session-scoped system events, and the requested `ignoreSubtypes` Slack config surface does not exist. The remaining blocker is maintainer product judgment about adding a public Slack config knob for edit/delete suppression.

## What This Changes

Request to add `channels.slack.ignoreSubtypes` so configured Slack edit/delete subtypes are dropped before they wake an agent session.

## Best Possible Solution

Decide the Slack edit/delete event policy explicitly; if accepted, implement a narrow opt-in Slack plugin config field with schema, docs, and regression tests while preserving current default behavior.

## Reproduction Assessment

Yes, at source level: ordinary Slack edit/delete events resolve a subtype handler and enqueue a session-scoped system event on current main. I did not run a live Slack Socket Mode reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed `ignoreSubtypes` key is a plausible narrow opt-in solution, but it is not the best fix until maintainers confirm the desired public Slack edit/delete event policy.
