## Summary

Keep open: current main still maps Slack `message_changed` and `message_deleted` into subtype system events, and the requested `ignoreSubtypes` public config key is not present. The request is valid, but it needs a Slack config/product decision rather than conservative cleanup closure.

## What This Changes

Add a Slack `channels.slack.ignoreSubtypes` config key so selected Slack subtype events such as `message_changed` and `message_deleted` are dropped before waking an agent session.

## Best Possible Solution

Add an owner-approved, opt-in Slack subtype filter in Slack config, schema, docs, and the event layer after Assistant DM safeguards but before subtype system-event enqueue, while keeping `thread_broadcast` as normal user message input.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.19` still route `message_changed` and `message_deleted` through subtype system-event enqueue, and focused config searches show no `ignoreSubtypes` key.

## Solution Assessment

Unclear for the exact public API until an owner decides the config scope and event ordering. The maintainable direction is an opt-in event-layer filter placed after Slack Assistant `message_changed` safeguards and before enqueueing subtype system events.
