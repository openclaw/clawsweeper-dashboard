## Summary

Keep this open: current main and v2026.5.12 still route Slack `message_changed` and `message_deleted` through the subtype system-event path, and there is still no `ignoreSubtypes` Slack config key to suppress them before session wake.

## What This Changes

Add a Slack `channels.slack.ignoreSubtypes` config key to drop selected Slack edit/delete subtype events before they wake agent sessions.

## Best Possible Solution

Add an opt-in Slack plugin config field, validate and document it, and short-circuit matching edit/delete subtype system events before authorization/session wake while preserving current default behavior.

## Reproduction Assessment

Yes. Source inspection and existing tests provide a high-confidence path: a Slack `message_changed` or `message_deleted` event reaches `resolveSlackMessageSubtypeHandler` and can enqueue a system event unless authorization blocks it.

## Solution Assessment

Yes. An opt-in event-layer filter is the narrowest maintainable direction; prompt instructions are fragile, and changing the default edit/delete behavior would be broader than requested.
