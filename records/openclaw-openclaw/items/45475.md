## Summary

Keep open: current main still intentionally converts Slack `message_changed` and `message_deleted` into session-scoped system events, while the requested `ignoreSubtypes` config surface is absent and needs a maintainer decision on public config shape and Assistant edit semantics.

## What This Changes

Request to add a Slack `ignoreSubtypes` config option that drops selected edit/delete subtypes before system-event enqueue and session wake.

## Best Possible Solution

Add an opt-in Slack plugin config field, validate and document it, and filter matching edit/delete subtype system events before authorization/session enqueue while preserving the intended Assistant DM path.

## Reproduction Assessment

Yes, at source level. Current code and tests show Slack edit/delete subtypes enqueue system events, and repository search plus schema/type inspection show no `ignoreSubtypes` config key.

## Solution Assessment

Yes, an event-layer opt-in filter is the narrowest maintainable direction. The config location and whether it applies before Slack Assistant `message_changed` rehydration need maintainer approval first.
