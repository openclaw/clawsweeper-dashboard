## Summary

Keep open: current main and the latest release still route ordinary Slack `message_changed` and `message_deleted` events into authorized system events, and there is still no `ignoreSubtypes` Slack config surface. The remaining decision is whether maintainers want a new public Slack config knob or prefer to keep edit/delete observability as-is.

## What This Changes

Requested an opt-in Slack channel config key that drops selected Slack message subtypes before they enqueue system events or affect agent sessions.

## Best Possible Solution

Choose the Slack event policy explicitly; if suppression is desired, add a narrowly documented opt-in Slack plugin config field for only selected edit/delete subtypes and keep existing behavior as the default.

## Reproduction Assessment

Yes, from source inspection: ordinary Slack edit/delete events resolve a subtype handler and enqueue a session-scoped system event on current main. I did not run a live Slack Socket Mode reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed `ignoreSubtypes` key is a plausible narrow opt-in solution, but adding a public Slack config knob is not automatically the best path without maintainer confirmation of the desired event policy.
