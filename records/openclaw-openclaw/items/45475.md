## Summary

Keep open. Current main still routes Slack `message_changed` and `message_deleted` through the subtype system-event path, and the requested `channels.slack.ignoreSubtypes` option is absent from config types, strict schema, channel resolution, UI hints, and docs.

## What This Changes

Add an opt-in Slack config key that drops configured message subtypes such as `message_changed` and `message_deleted` before system events wake agent sessions.

## Best Possible Solution

Ship a Slack-owned opt-in ignore list with unchanged defaults, strict validation, docs, config metadata, and regression tests that filter only the metadata subtype system-event branch before enqueue.

## Reproduction Assessment

Yes. Source and focused tests show current main enqueues system events for Slack edit/delete subtypes, and strict config validation has no accepted `ignoreSubtypes` key; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Filtering configured subtypes in the Slack event layer after Assistant `message_changed` rehydration and before system-event authorization/enqueue is the narrow maintainable path; prompt-only workarounds are weaker.
