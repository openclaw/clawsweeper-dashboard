## Summary

Keep open. Current main still treats Feishu broadcast-only `@all` as not being a bot mention, returns before Feishu broadcast fan-out under mention gating, and routes direct and group conversations into distinct session spaces; changing that behavior needs explicit Feishu product policy.

## What This Changes

The issue requests reliable Feishu group `@all` delivery to multiple OpenClaw agents and context continuity for the same agent across direct and group chats.

## Best Possible Solution

Define an opt-in Feishu multi-agent fan-out model covering `@all`, visible reply rules, loop control, and explicit direct/group memory linking with clear privacy boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: in a configured Feishu broadcast group with `requireMention=true`, a broadcast-only `@all` event leaves `ctx.mentionedBot=false` and returns before broadcast dispatch; DMs and groups also route through different peer kinds.

## Solution Assessment

Unclear. Treating `@all` as a normal bot mention would conflict with current docs and tests, so the safer path is an explicit opt-in Feishu fan-out and continuity design rather than a narrow mention-check change.
