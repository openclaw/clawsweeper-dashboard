## Summary

Keep open. Current main still does not satisfy the reported Feishu workflow: broadcast-only `@all` is intentionally not a bot mention, mention-gated group handling returns before broadcast fan-out, observer broadcast agents process silently, and direct/group conversations route to distinct session spaces.

## What This Changes

The issue asks Feishu group `@all` messages to reach every configured OpenClaw agent and the same agent to retain context across direct and group sessions.

## Best Possible Solution

Define an opt-in Feishu multi-agent fan-out model with loop control and visible-reply rules, and separately define explicit same-agent direct/group continuity or memory linking with clear privacy boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an `@all`-only Feishu group event is not a bot mention and returns at the mention gate before broadcast dispatch, while DM and group events route to different peer kinds.

## Solution Assessment

Unclear. Treating `@all` as a normal bot mention would reverse current documented and tested behavior, so the safer path is an explicit Feishu fan-out and continuity design rather than a narrow mention-check change.
