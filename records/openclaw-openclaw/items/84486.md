## Summary

Keep open: current main and v2026.5.18 still have the Feishu streaming-card path where a final payload assigns `streamText = text`, so pre-tool text accumulated by partial callbacks can be replaced. The attached closing PRs only touch ACP projector files, while the reporter and the linked Lark plugin issue provide useful live-workaround context for the Feishu-specific source fix.

## What This Changes

The issue reports Feishu streaming-card replies overwriting assistant text produced before a tool call when a later final payload arrives in the same turn.

## Best Possible Solution

Patch the Feishu dispatcher source to carry prior streamed answer text across assistant invocations, add a regression for partial pre-tool text followed by a final payload on the same streaming card, and reconcile the ACP-only PRs against that corrected scope.

## Reproduction Assessment

Yes, source-level: Feishu partial callbacks accumulate text into `streamText`, and a later active-card final delivery assigns `streamText = text`, so earlier pre-tool text can be replaced. I did not run a live Feishu credentialed repro.

## Solution Assessment

Yes for the issue's Feishu-dispatcher direction; no for treating the linked ACP-only PRs as sufficient. The narrow maintainable fix belongs in `extensions/feishu/src/reply-dispatcher.ts` with focused regression coverage.
