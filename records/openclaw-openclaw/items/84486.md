## Summary

Keep open: current main and v2026.5.19 still have the Feishu streaming-card path where a final payload replaces `streamText`, and the linked open PRs only change ACP projector files rather than the Feishu dispatcher surface.

## What This Changes

The issue requests preserving assistant text emitted before tool calls in Feishu streaming card replies.

## Best Possible Solution

Add a Feishu dispatcher source fix and regression for partial pre-tool text followed by a later final payload on the same active streaming card, then reconcile or close the ACP-only PRs separately.

## Reproduction Assessment

Yes, at source level: Feishu partial callbacks accumulate text into `streamText`, a later active-card final delivery assigns `streamText = text`, and card close uses that value. I did not perform a live Feishu credentialed run in this read-only review.

## Solution Assessment

Yes for the Feishu-dispatcher direction, but not for treating ACP-only PRs as sufficient. The maintainable fix belongs in `extensions/feishu/src/reply-dispatcher.ts` with regression coverage for partial-plus-final tool-call turns.
