## Summary

Keep open: current main still has the Feishu streaming-card path that can replace pre-tool streamed text with the later final payload, and the open closing PRs target the separate ACP projector surface rather than the Feishu dispatcher.

## What This Changes

The issue asks OpenClaw to preserve assistant text emitted before tool calls in Feishu streaming card replies.

## Best Possible Solution

Patch the Feishu dispatcher to carry or merge prior stream text when a later final payload updates the same active card, add a focused regression in extensions/feishu/src/reply-dispatcher.test.ts, and reconcile the ACP-only PRs separately.

## Reproduction Assessment

Yes, at source level: send a pre-tool onPartialReply into an active Feishu streaming card, then deliver a later final payload with only post-tool text; current code first accumulates streamText and then replaces it with the final text.

## Solution Assessment

Yes for a Feishu dispatcher fix, but no for treating the ACP-only linked PRs as sufficient. The narrow source fix belongs in extensions/feishu/src/reply-dispatcher.ts with partial-plus-final regression coverage.
