## Summary

Keep open: current `main` still has a source-plausible ACP `final_only` message-loss path around tool-use boundaries, and I did not find a merged fix or canonical duplicate that owns this exact default-delivery behavior.

## What This Changes

The issue reports that default ACP `final_only` delivery drops assistant text emitted before a tool call within the same user turn.

## Best Possible Solution

Make ACP delivery distinguish intermediate tool-use boundaries from the terminal turn result, preserving visible final-only text for one user turn without replaying stale pre-tool fragments when a canonical final answer is available.

## Reproduction Assessment

Yes, at source level: the final-only projector keeps visible text in a per-projector buffer and clears it on each done/error boundary, while existing tests only cover one terminal done. A live Feishu/ACP run was not executed in this read-only review.

## Solution Assessment

Unclear until patched, but the suggested turn-level accumulation direction is reasonable. The safer implementation should make terminal versus intermediate tool-use boundaries explicit rather than broadly appending every historical assistant fragment.
