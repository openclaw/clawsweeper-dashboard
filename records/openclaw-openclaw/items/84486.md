## Summary

Keep open: current main still has a source-reproducible ACP `final_only` path where buffered visible text is cleared on every `done`/`error` boundary, and I did not find a merged fix or canonical duplicate for this exact default-delivery behavior.

## What This Changes

The issue reports that ACP `final_only` delivery drops assistant text emitted before a tool-use boundary instead of including it in the final channel reply.

## Best Possible Solution

Make ACP `final_only` preserve visible output across intermediate tool-use boundaries for one user turn, with regression tests that prevent stale pre-tool fragments from leaking outside the intended ACP final reply.

## Reproduction Assessment

Yes, at source level: final-only text is buffered in `finalOnlyOutputText` and cleared by `resetTurnState()` after every projector `done`/`error` boundary. I did not run a live Feishu or ACP backend reproduction in this read-only review.

## Solution Assessment

Unclear until patched, but turn-level preservation with explicit intermediate-versus-terminal boundary handling is the right direction. A broad append-all-history fix would be riskier because adjacent issues intentionally suppress text-between-tools in other delivery paths.
