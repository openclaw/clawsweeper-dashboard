## Summary

Keep open: current main and v2026.6.8 still have the Feishu streaming-card final path overwriting accumulated pre-tool text with the later final payload, and the newer duplicate was already closed back to this issue as the canonical tracker.

## What This Changes

Feishu streaming card replies should preserve assistant text emitted before a tool call instead of showing only the later final assistant reply.

## Best Possible Solution

Patch the Feishu plugin streaming-card dispatcher to carry accumulated visible text across tool-use boundaries and cover partial-then-final payloads with regression tests.

## Reproduction Assessment

Yes, at source level: `onPartialReply` can accumulate pre-tool text, then final delivery normalizes to the later final payload and assigns it back to `streamText`; close renders from that current value. I did not run a live credentialed Feishu repro in this read-only review.

## Solution Assessment

Yes, the maintainable fix belongs in the Feishu plugin streaming-card dispatcher, not in ACP `final_only` dispatch. The patch should preserve accumulated visible text for later final payloads while coordinating with the broader Feishu streaming-card cluster.
