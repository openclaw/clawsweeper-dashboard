## Summary

Keep open: current main and v2026.5.18 still have the Feishu streaming-card path where a later final payload can replace pre-tool partial text, and the linked closing PRs target ACP `final_only` rather than the Feishu dispatcher surface.

## What This Changes

The issue reports that Feishu streaming cards overwrite assistant text emitted before a tool call when the later final reply updates the same card.

## Best Possible Solution

Fix the Feishu plugin dispatcher so streamed pre-tool text is carried through later final card updates without double-concatenation, add a focused dispatcher regression for partial-plus-final tool-call turns, and reconcile the ACP-only PRs separately.

## Reproduction Assessment

Yes, at source level: Feishu partial callbacks accumulate text into `streamText`, and a later active-card final delivery assigns `streamText = text`, so earlier pre-tool text can be replaced. I did not run a live Feishu credentialed repro in this read-only review.

## Solution Assessment

Yes for the issue's Feishu-dispatcher direction, but no for treating the ACP-only linked PRs as a complete fix. The narrow maintainable repair belongs in `extensions/feishu/src/reply-dispatcher.ts` with regression coverage for a pre-tool partial followed by a later final payload.
