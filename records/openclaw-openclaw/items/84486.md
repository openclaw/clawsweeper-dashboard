## Summary

Keep open: current main still has the Feishu streaming-card final path that can replace earlier streamed assistant text with the later final payload, and the latest release has the same overwrite shape. Related duplicate cleanup points back to this issue as the canonical narrow tracker.

## What This Changes

The issue asks OpenClaw to preserve text emitted before tool calls in Feishu streaming card reply mode instead of losing it when a later model invocation updates the same card.

## Best Possible Solution

Keep this canonical issue open until the Feishu dispatcher preserves earlier visible assistant text across tool-use boundaries without duplicating transient tool progress or stale card content.

## Reproduction Assessment

Yes, at source level: the current Feishu dispatcher can accumulate pre-tool partial text, then replace streamText with a later final payload before close. I did not run a live credentialed Feishu repro in this read-only review.

## Solution Assessment

Yes, the maintainable fix belongs in the Feishu streaming-card dispatcher, not ACP final_only dispatch. The patch should preserve accumulated visible assistant text across tool-use boundaries and add a focused regression test for that exact sequence.
