## Summary

Keep open. Current main still accepts plain assistant final text without enforcing a matching calendar/gog/exec invocation, and the newer finalization hook is only an optional adjacent seam rather than a fix for the reported Pi/Anthropic path.

## What This Changes

The issue asks OpenClaw to prevent factual or raw-output answers from being finalized unless the authoritative calendar or shell tool was actually invoked for that turn.

## Best Possible Solution

Define a narrow, generic evidence contract for explicit freshness or raw-output turns, then enforce it in the embedded runner, tool policy, or a documented hook boundary without hard-coding one user's gog/calendar wrapper.

## Reproduction Assessment

Yes for a source-level reproduction: simulate a plain assistant final answer to an explicit calendar or raw shell-output request with no structured tool call or tool metadata, and current main still has no guard that blocks the terminal payload. I did not rerun the reporter's live macOS gog shim setup.

## Solution Assessment

No, current prompt guidance plus optional hook surfaces are not yet the best complete fix for the reported behavior. The safer direction is a narrow enforceable evidence contract for explicit freshness/raw-output requests rather than unconditional tool forcing for every factual-looking question.
