## Summary

Keep open. Current main still has prompt guidance, tool execution, diagnostics, and hook seams, but no runtime contract that blocks or retries a plain final answer to an explicit calendar/raw-output request when no matching tool call was emitted.

## What This Changes

The issue asks OpenClaw to require authoritative live tool invocation for factual, time-sensitive calendar queries and explicit raw shell-output requests instead of accepting fabricated assistant text.

## Best Possible Solution

Define a narrow generic evidence contract for explicit freshness or raw-output turns, then enforce it in the embedded runner, tool policy, or a documented hook boundary without hard-coding one user's gog wrapper.

## Reproduction Assessment

Yes for a source-level reproduction: current main can turn plain assistant final text into a reply with no toolMetas, and the guard only repairs emitted tool calls. I did not run the reporter's live gog shim setup.

## Solution Assessment

No, the current prompt guidance plus optional finalization hook is not a complete fix. The safer solution is a narrow enforceable evidence contract for explicit freshness/raw-output requests rather than broad mandatory tool use for every factual-looking prompt.
