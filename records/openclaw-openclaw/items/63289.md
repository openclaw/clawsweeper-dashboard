## Summary

Keep open. Current main still gives advisory live-check guidance and handles structured tool calls after they exist, but it does not enforce a fresh calendar/gog/exec invocation before accepting a plain final answer to an explicit time-sensitive or raw-output request.

## What This Changes

The issue asks OpenClaw to prevent agents from answering calendar or raw-output factual requests unless the authoritative tool actually ran and left traceable output.

## Best Possible Solution

Define a narrow runtime contract for explicit freshness and raw-output turns, then enforce it in the embedded runner, tool policy, or plugin hook boundary without hard-coding calendar/gog specifics.

## Reproduction Assessment

Yes for a code-level reproduction: simulate an assistant final text answer to an explicit calendar/raw-output prompt with no structured tool call or tool metadata, and current main has no guard that blocks the terminal payload. The reporter's macOS gog shim path is concrete, but live gog validation was not rerun in this read-only review.

## Solution Assessment

No, the current prompt guidance and diagnostics are not the best solution for the reported behavior. The safer direction is a narrow enforceable evidence contract for explicit freshness/raw-output requests, not unconditional tool forcing for every factual-looking question.
