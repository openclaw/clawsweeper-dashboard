## Summary

Current main now implements the requested isolated-cron routing fix and adds focused coverage: canonical `anthropic/*` cron attempts are resolved through the configured `claude-cli` runtime before CLI detection. The proof commit is on `main` but not in a release tag in this checkout.

## What This Changes

Route isolated cron Anthropic model refs through the configured Claude CLI runtime when the agent uses `agentRuntime.id: "claude-cli"`.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; no further source change is needed for this item.

## Reproduction Assessment

Yes for the original regression by source inspection: the old failure mode was an unresolved `providerOverride` reaching `isCliProvider`. Current main no longer reproduces that path because isolated cron resolves the execution provider before CLI detection.

## Solution Assessment

Yes. Reusing `resolveCliRuntimeExecutionProvider` in isolated cron is the narrow maintainable fix because it aligns cron with the existing command and auto-reply runtime routing contract.
