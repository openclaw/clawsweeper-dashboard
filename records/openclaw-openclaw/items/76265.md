## Summary

Keep open: current main still has the isolated cron executor decide CLI versus embedded execution from the unresolved canonical provider, so `anthropic/*` with `agentRuntime.id: "claude-cli"` can bypass the configured CLI runtime while normal agent turns already resolve that mapping.

## What This Changes

The issue requests that isolated cron execution honor an agent's `claude-cli` runtime mapping when a cron payload uses canonical `anthropic/*` model IDs.

## Best Possible Solution

Teach isolated cron to resolve the execution provider through `resolveCliRuntimeExecutionProvider` before CLI detection, then add focused coverage for canonical `anthropic/*` plus `agentRuntime.id: "claude-cli"`.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with provider `anthropic` and configured runtime `claude-cli`, the current isolated cron callback checks `isCliProvider("anthropic")` and takes the embedded path, while the existing resolver would return `claude-cli`.

## Solution Assessment

Yes. Reusing the existing runtime execution-provider resolver is the narrowest maintainable fix because it aligns isolated cron with the already-covered command/chat path without adding a new mapping contract.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/76265. Isolated cron execution currently branches on `isCliProvider(providerOverride, cfg)` before applying the agent runtime mapping, so an agent configured with `agentRuntime.id: "claude-cli"` and a cron payload model like `anthropic/claude-haiku-4-5` can run through embedded Anthropic Messages instead of the Claude CLI backend. In `src/cron/isolated-agent/run-executor.ts`, resolve the execution provider with the existing `resolveCliRuntimeExecutionProvider({ provider, cfg, agentId })` helper before CLI detection and use the resolved provider consistently for CLI session lookup and `runCliAgent`, falling back to the original provider when no mapping applies. Add focused isolated-cron test coverage for canonical `anthropic/*` plus `agentRuntime.id: "claude-cli"`; avoid changing unrelated alias normalization, raw one-shot model runs, or provider catalog behavior. Add a user-facing `CHANGELOG.md` fix entry unless maintainer policy marks this as internal-only. Related refs: #76265, #75753, #76155.
