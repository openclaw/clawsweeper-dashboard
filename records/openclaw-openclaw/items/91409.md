## Summary

## What This Changes

The PR adds a gateway `update.run` post-core finalization step for successful git/source updates and extends `update finalize` to honor an effective, non-persisted update channel.

## Best Possible Solution

Land the CLI-parity finalizer path only after maintainers accept the fail-closed RPC restart behavior and are satisfied that the effective-channel bridge has enough upgrade proof.

## Solution Assessment

Yes, this is the best owner-boundary fix shape: reuse the existing hidden `update finalize` path instead of duplicating plugin convergence inside gateway code. The compatibility-sensitive part is the deliberate fail/no-restart behavior when finalization fails.
