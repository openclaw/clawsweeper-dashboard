## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve markers for soft-trimmed cached images:** `src/agents/pi-extensions/context-pruning/pruner.ts:475-479`
  - body: This callback records every image before head/tail trimming, but `writePrunedMediaCaches` can only replace placeholders that remain in the trimmed text. If an image lands in the elided middle and the message does not also hard-clear, the file is written but no `[media cached: ...]` marker is inserted, so `cacheMedia: true` still loses that media reference. Only collect refs whose marker survives, or emit compact marker slots for elided media in the soft-trim output.
  - confidence: 0.92
- **[P1] Add cleanup before persisting pruned media:** `src/media/media-cache.ts:46-47`
  - body: `cacheMediaToDisk` writes every pruned media block into `media/cache`, but this module has no TTL cleanup, session ownership, or removal path. That turns ephemeral pruned context into persistent sensitive files despite the PR body claiming TTL cleanup, so the feature needs an owned lifecycle through the existing media store/cleanup boundary or a documented retention policy before merge.
  - confidence: 0.9
- **[P2] Normalize MIME before classifying cached media:** `src/media/media-cache.ts:67-68`
  - body: `mediaKindFromMime` expects a normalized MIME string, but this helper passes raw values. A real image MIME such as `IMAGE/PNG` or `image/png; charset=binary` becomes `document`, and `detectImageReferences` only re-injects cached markers with `kind=image`, so those cached images will not be viewable again.
  - confidence: 0.87
- **[P3] Gate cached-media prompt guidance on the feature:** `src/agents/system-prompt.ts:457-459`
  - body: The system prompt always says media removed during pruning may be cached even though the new config defaults `cacheMedia` to false. That adds misleading instructions and token overhead for the default path where these markers cannot appear, so pass the effective setting into prompt rendering or remove the section until the feature is enabled.
  - confidence: 0.78
