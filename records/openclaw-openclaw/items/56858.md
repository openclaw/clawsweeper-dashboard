## Summary

## What This Changes

The PR adds an opt-in `agents.defaults.contextPruning.cacheMedia` setting that writes pruned media to disk, emits cached-media markers, rehydrates cached image markers, updates config schema/baselines, and adds focused tests.

## Best Possible Solution

Rework this as a current-main `pi-hooks` implementation that uses managed media-store primitives or opaque media refs, preserves default pruning semantics, documents opt-in retention behavior, and includes focused regression tests plus redacted real-session proof.

## Reproduction Assessment

Yes for the review blockers by source inspection: current main loads `pi-hooks`, while the PR head implements the feature under stale `pi-extensions`, and the new cache helper writes persistent files directly. I did not run a live multi-turn session in this read-only review.

## Solution Assessment
