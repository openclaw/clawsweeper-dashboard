  - confidence: 0.92

  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Needs attention because the patch persists pruned user media outside the managed media-store lifecycle and without proven retention bounds.

- **[medium] Unbounded persisted pruned media:** `src/media/media-cache.ts:46`
  - body: The new helper writes decoded pruned media into a cache directory without using the existing media store's max-byte limits, cleanup, and lifecycle hooks, so sensitive user media may remain longer than operators expect.
