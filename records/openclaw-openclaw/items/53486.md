## Summary

Keep open: current main and the latest release still send Feishu card-shaped JSON from the plain message text path as text, and the only active implementation path is the open linked fix PR rather than merged code.

## What This Changes

Feishu proactive `message(action=send)` renders valid Feishu interactive-card JSON from the `message` parameter as plain text instead of sending it as an interactive card.

## Best Possible Solution

Land or replace the linked Feishu PR with a plugin-owned parser that sanitizes supported native card JSON for `send` and `thread-reply` text while leaving malformed JSON and ordinary text on the existing text path.

## Reproduction Assessment

Yes, source-level: current Feishu `handleAction` only builds cards from structured `presentation`, and card-shaped text falls through to `sendMessageFeishu`. I did not run live Feishu delivery in this read-only review.

## Solution Assessment

Yes, the best fix belongs in the Feishu plugin action path: detect and sanitize supported Feishu card JSON before the existing text fallback. Moving Feishu card parsing into the shared message runner would cross the core/plugin owner boundary without a missing generic API.
