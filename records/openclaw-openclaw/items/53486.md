## Summary

Keep open. Current main still lacks Feishu plain-message card JSON detection, the source path sends JSON-looking `message` text through the plain text sender, and https://github.com/openclaw/openclaw/pull/77268 is an open unmerged closing fix candidate.

## What This Changes

The issue reports a regression where Feishu proactive `message(action=send)` sends valid interactive-card JSON as raw text instead of rendering it as a Feishu card.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/77268; after it merges, close this issue with merge commit and release provenance.

## Reproduction Assessment

Yes by source inspection. The issue provides a concrete JSON payload, and current main reads it as `text`/`message`, builds a Feishu card only from `presentation`, then calls `sendMessageFeishu` when no card is present.

## Solution Assessment

Yes for the proposed direction. A Feishu-local compatibility parser that reuses the existing native-card sanitizer is narrower than moving Feishu-specific parsing into shared core, but the linked PR still needs review and proof handling before merge.
