The issue asks OpenClaw to restore Feishu proactive sends so valid interactive-card JSON in the plain message parameter renders as an interactive card instead of raw text.

## Best Possible Solution

Add Feishu-local compatibility detection for valid native interactive-card JSON strings in send and thread-reply, sanitize through the existing native-card sanitizer, route valid cards to sendCardFeishu, and leave malformed, non-card, or oversized JSON on the text path.

## Solution Assessment

Yes for the fix direction. Keeping compatibility parsing inside the Feishu plugin runtime and reusing the existing sanitizer is narrower than adding Feishu-specific parsing to the shared message runner.
