## What This Changes

The issue reports a Feishu regression where proactive `message(action=send)` sends interactive-card JSON as raw text instead of rendering it as a Feishu interactive card.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/93706 after live Feishu proof or an explicit maintainer proof override, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: a plain card JSON `message` reaches the Feishu send path as text, no card is built unless `presentation` is provided, and the branch then calls `sendMessageFeishu`. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

Yes for the code direction: Feishu-local detection backed by the existing sanitizer is narrower than putting Feishu-specific JSON parsing in shared core. The issue should remain open until the linked PR lands with proof or an explicit proof override.
