## Summary

Keep open. Current main still sends valid Feishu interactive-card JSON in the plain `message` path as text, and open PR #77268 specifically references this issue as its closing fix candidate.

## What This Changes

Restore Feishu proactive `message(action=send)` card JSON detection so valid interactive-card JSON in `message` renders as an interactive card instead of plain text.

## Best Possible Solution

Review and land or close PR #77268; after it merges, close this issue as implemented with the merge commit and release provenance.

## Reproduction Assessment

Yes. The issue includes a concrete payload and current-main source shows that plain `message` JSON cannot become a Feishu card; I did not run live Feishu delivery in this read-only review.

## Solution Assessment

Yes. A Feishu-local compatibility parser that reuses the existing native-card sanitizer is the narrowest maintainable fix; moving Feishu-specific parsing into shared core would be broader than needed.
