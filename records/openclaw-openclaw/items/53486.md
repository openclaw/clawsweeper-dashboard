## Summary

Keep open. Current main still routes Feishu `message(action=send)` plain `message` text to the raw text sender unless structured presentation/card data is provided, and the active linked PR should be reviewed before this issue closes.

## What This Changes

The issue requests restoring Feishu interactive-card handling when proactive message-tool sends pass valid card JSON in the plain `message` or `text` parameter.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/93706, then close it only after a Feishu-local fix lands or a replacement main-branch fix supersedes it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass valid Feishu card JSON as `message` to Feishu `send`, and current main treats it as text before calling `sendMessageFeishu`.

## Solution Assessment

Yes. The best fix is a narrow Feishu-owned conversion from supported card JSON text into the existing sanitized card send path, without adding Feishu-specific parsing to the shared message runner unless maintainers decide a generic seam is needed.
