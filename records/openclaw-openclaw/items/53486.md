## Summary

## What This Changes

Restore Feishu `message(action=send)` so card JSON supplied as plain `message` or `text` renders as an interactive card instead of raw JSON text.

## Best Possible Solution

Adopt or reopen the sanitizer-backed Feishu-local parser approach from the closed candidate PRs, then land it with focused channel tests plus redacted live Feishu proof or an explicit maintainer proof override.

Yes, at source level: current main reads plain `message`/`text` but only builds a Feishu card from `presentation`, so card-shaped JSON falls through to `sendMessageFeishu`. I did not run a live Feishu send in this read-only review.

## Solution Assessment
