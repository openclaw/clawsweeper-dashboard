## Summary

Keep open. Current main still sends Feishu card-shaped JSON from the plain `message`/`text` parameter through the plain-text path, and the focused fix PR was closed unmerged because live Feishu proof was unavailable.

## What This Changes

Restore Feishu `message(action=send)` handling so valid card JSON in the plain `message` parameter renders as an interactive card instead of raw text.

## Best Possible Solution

Add Feishu-local compatibility detection for valid native interactive-card JSON strings in `send` and `thread-reply`, sanitize through the existing native-card sanitizer, route valid cards to `sendCardFeishu`, and leave malformed, non-card, or oversized JSON on the text path.

## Reproduction Assessment

Yes, by source inspection rather than live Feishu. The reported payload supplies only a plain `message` string, while current main only creates Feishu cards from `presentation` or structured `channelData` before falling through to `sendMessageFeishu`.

## Solution Assessment

Yes for the fix direction. Keeping compatibility parsing inside the Feishu plugin runtime and reusing the existing sanitizer is narrower than adding Feishu-specific parsing to shared core.
