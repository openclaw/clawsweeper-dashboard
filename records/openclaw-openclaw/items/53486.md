## Summary

Keep this issue open. Current main still supports Feishu cards only through structured `presentation`, `interactive`, or `channelData.feishu.card` paths; a valid native card JSON string passed as the plain `message` value remains text and is sent through Feishu post delivery. The related native-card payload work helps the eventual fix but does not supersede this compatibility regression.

## What This Changes

Restore Feishu `message(action=send)` compatibility so valid interactive-card JSON supplied in the `message` string is sent as a native interactive card rather than plain post text.

## Best Possible Solution

Add Feishu-owned compatibility parsing for valid native interactive-card JSON strings in `send` and `thread-reply`, route accepted cards through the same sanitizer and `sendCardFeishu` path used by native payload support, and leave malformed card JSON or ordinary JSON on the existing text path.

## Reproduction Assessment

Yes. The source-level reproduction is to call Feishu `message(action=send)` with the sample card JSON as `message`: current main treats it as `text`, the Feishu action handler does not convert it to `card`, and `sendMessageFeishu` emits a post payload.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Feishu plugin, likely by sharing the existing native-card sanitizer with the action handler; putting Feishu-specific JSON detection in shared core would broaden a provider-owned compatibility behavior.

## Repair Work Prompt

Fix the Feishu regression from https://github.com/openclaw/openclaw/issues/53486. Implement the fix in the Feishu plugin boundary, not shared core unless a generic seam proves necessary. Add a Feishu-local helper that recognizes valid native card JSON strings, including wrapped `{ "type": "interactive", "card": ... }` shapes and schema/v1 card object shapes already accepted by the native payload path. Reuse or extract the existing native-card sanitizer from `extensions/feishu/src/outbound.ts`; do not send arbitrary parsed JSON directly.
