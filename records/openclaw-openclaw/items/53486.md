## Summary

Keep this open: current main and the latest release still only build Feishu cards from structured `presentation` or `channelData.feishu.card`, while plain `message` JSON falls through to text sending. The directly related fix PR was closed unmerged for missing live Feishu proof, so it does not supersede the issue.

## What This Changes

The issue asks OpenClaw to render valid Feishu interactive card JSON passed through Feishu `message(action=send)` or `thread-reply` text as native cards instead of raw JSON text.

## Best Possible Solution

Land a narrow Feishu-owned parser/normalizer for card-shaped JSON in `send` and `thread-reply`, reusing the existing native-card sanitizer and preserving malformed or non-card JSON as plain text.

## Reproduction Assessment

Yes at source level. Current main routes plain `message` JSON through the Feishu text branch unless the caller supplies structured `presentation` or channelData card payloads; I did not run live Feishu delivery in this read-only review.

## Solution Assessment

Yes. The maintainable fix is Feishu-local parsing for Feishu card-shaped text using the existing sanitizer, not channel-specific JSON guessing in shared core.
