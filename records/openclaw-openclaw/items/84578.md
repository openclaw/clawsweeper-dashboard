## Summary

Keep this PR open: current main still drops final WhatsApp `isError` payloads at both the dispatcher and outbound adapter, while the patch is narrow and aligned with the channel’s opted-in error-payload routing. The PR lets WhatsApp deliver final `isError` reply payloads by relaxing inbound filtering and removing the outbound `sendPayload` error no-op, with regression tests for final delivery and non-final suppression.

## Best Possible Solution
