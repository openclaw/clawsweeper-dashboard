## Summary

Keep this PR open: the targeted Codex native-thread reuse fix is useful and the supplied terminal proof covers the intended oversized `thread_bootstrap` path, but the new early return is too broad and can preserve a stale oversized context-engine binding long enough to skip mirrored-history projection before lifecycle starts a fresh thread.

## What This Changes

The PR defers the Codex app-server startup native transcript token/byte guard for saved context-engine `thread_bootstrap` bindings and adds oversized bootstrap regression coverage.

## Best Possible Solution

Keep the narrow startup-guard deferral, but limit it to a current active context-engine bootstrap path or force mirrored-history projection whenever the saved binding will be invalidated before starting a fresh thread.

## Reproduction Assessment

Yes, from source inspection and the PR's targeted proof: current main clears oversized bindings before context-engine bootstrap compatibility can preserve them, and the PR demonstrates the intended token/byte oversized resume path. I did not execute tests because this review is read-only.

## Solution Assessment

No as written. The deferral is the right narrow direction for valid `thread_bootstrap` bindings, but it needs a stale/no-active context-engine guard or fallback projection before it is the safest fix.
