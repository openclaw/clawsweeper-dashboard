## Summary

## What This Changes

The PR adds a postinstall loopback gateway-port resolver/probe that prints a restart warning when a likely running gateway is detected after package installation.

## Best Possible Solution

Keep the postinstall warning only if it resolves ports through the same config and installed-service contracts OpenClaw already uses, with focused regression tests and no root proof artifact in the product branch.

## Reproduction Assessment

Yes for the patch defect: current source and tests show systemd writes the gateway port as an inline unit Environment value, while the PR reads only gateway.systemd.env. No full npm update/rollback gateway reproduction was run in this read-only review.

## Solution Assessment

No. A postinstall warning is an acceptable mitigation if maintainers want that boundary, but this implementation duplicates config/service parsing and still misses the current systemd custom-port contract.
