## Label Justifications

- `P1`: The PR targets a linked gateway regression where a healthy-looking process can silently lose inbound messages after update or rollback.
- `merge-risk: 🚨 compatibility`: The patch changes postinstall behavior and adds an env opt-out while missing supported configured gateway ports during upgrades.

## Summary

Keep this PR open, but it is not merge-ready: the warning-only mitigation is useful, yet the implementation only probes the default gateway port and misses documented/configured gateway ports for the same stale-module message-loss failure class.

## What This Changes

The PR adds a postinstall-time loopback TCP probe and ASCII restart warning when something is listening on `127.0.0.1:18789`, plus a new opt-out environment variable.

## Best Possible Solution

Resolve the same effective gateway port/service metadata OpenClaw uses at runtime, add focused postinstall coverage for default, custom, no-listener, timeout, and suppression cases, and keep the linked gateway issue open unless maintainers explicitly accept warning-only mitigation as sufficient.

## Reproduction Assessment

Yes for the PR defect: current docs and source show configured gateway ports are supported, while the PR source only probes `18789`. I did not run a live configured-port gateway in this read-only review.

## Solution Assessment

No. A postinstall warning is a plausible mitigation, but the current implementation is too narrow because it ignores the effective gateway port and lacks focused regression coverage.
