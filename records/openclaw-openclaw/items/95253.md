## Summary

Keep open: the PR is a focused mitigation, but the latest service-env repair still misses the Linux/systemd installed-service port path used by the reported WSL2 setup, and the current PR body only shows test output rather than real postinstall/gateway proof.

## What This Changes

The PR adds a postinstall restart warning when a local gateway port is reachable, resolves that port from env, service metadata, config, or default, and adds focused script tests.

## Best Possible Solution

Resolve the probed port through the same platform-specific installed-service metadata sources OpenClaw uses for daemon status/repair, add systemd coverage, keep the warning advisory, and require maintainer acceptance of the postinstall probe/env surface before merge.

## Reproduction Assessment

Yes for the PR defect at source level: Linux/systemd service installs persist `OPENCLAW_GATEWAY_PORT` through systemd unit/environment metadata while the new reader only scans `service-env/*.env`. I did not run a live package rollback in this read-only review.

## Solution Assessment

No, not yet. A postinstall advisory is a plausible mitigation, but this implementation should use the existing platform service metadata sources or explicitly limit its scope before it can be the best fix.
