Keep open. Current main still does not implement Android Health Connect or the requested `health.*` node commands, and adding permissioned health-data reads would be a new privacy-sensitive Android/gateway capability that needs maintainer product and security review.

## What This Changes

Request adding an optional Android Health Connect capability with read-only `health.*` node commands, Health Connect permission handling, gateway allowlisting, docs, and tests.

## Best Possible Solution

If maintainers accept the scope, implement a narrow Android-owned Health Connect command family with explicit per-record read permissions, read-only gateway defaults, synthetic fixtures, focused tests, and privacy-first docs; otherwise define the missing plugin/native extension seam first.
