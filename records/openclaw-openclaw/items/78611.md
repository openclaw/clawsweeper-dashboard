Keep open. Current main still does not implement Android Health Connect or `health.*` node commands, and this is a new privacy-sensitive Android/gateway capability that needs maintainer product and security review before implementation.

## What This Changes

Request adds optional read-only Android Health Connect capability and `health.*` node commands for selected permissioned health metrics.

## Best Possible Solution

If maintainers accept the privacy surface, implement a narrow Android-native read-only Health Connect command family with explicit per-data-type permissions, availability-gated advertisement, gateway allowlist entries, synthetic tests, and privacy-focused docs that avoid real health data.

## Solution Assessment
