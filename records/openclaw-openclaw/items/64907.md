The PR adds a synchronous text-only `tool_result_before_model` Plugin SDK hook, wires it into embedded Pi tool-result handling, updates docs/API baselines, and adds hook/runner tests.

## Best Possible Solution

Land a maintainer-approved, rebased Plugin SDK contract that preserves current bundled middleware, documents trust and ordering semantics, and leaves broader async/full-result mutation to the canonical follow-up issues if out of scope.

Not applicable as a bug reproduction. Source inspection confirms current main lacks a public `tool_result_before_model` hook, and the PR screenshots show the intended same-turn canonicalization behavior in a real UI demo.

## Solution Assessment

Unclear for this branch as-is. A narrow synchronous text-only hook is plausible, but this implementation must be rebased, preserve bundled middleware, and receive explicit Plugin SDK/security approval before it is the right solution.
