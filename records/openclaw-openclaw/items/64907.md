## Summary

Keep open: this PR addresses a real public Plugin SDK feature gap, and the attached screenshot is useful proof, but the branch is stale against current main and would regress the existing trusted tool-result middleware path. The public pre-model hook also crosses a trust boundary that current main deliberately keeps bundled-only, so maintainer Plugin SDK/security judgment is needed before merge.

## What This Changes

The branch adds a public synchronous text-only `tool_result_before_model` hook, wires it into Pi embedded tool-result processing, updates docs/API baselines, and adds hook/runner coverage.

## Best Possible Solution

Land only a rebased, maintainer-approved Plugin SDK contract that preserves current bundled middleware, clearly defines trust/order/failure/bounds/transcript semantics, and aligns with https://github.com/openclaw/openclaw/issues/29132.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection shows the public hook is absent on current main, and the inspected screenshot demonstrates the intended branch behavior in a real OpenClaw UI run.

## Solution Assessment

No for this branch as-is. The synchronous text-only hook may be a viable shape, but this PR must be rebased, must preserve bundled middleware, and needs maintainer approval for the public model-visible rewrite trust boundary.
