## Summary

Keep open: the patch is a focused fix for the unbounded successful provider JSON read, but the helper is exported through the plugin SDK, so maintainers should explicitly accept or adjust the new 16 MiB default before merge.

## What This Changes

The PR changes readProviderJsonResponse to read through readResponseWithLimit with a 16 MiB default cap and adds streaming JSON regression coverage.

## Best Possible Solution

Land the bounded JSON reader after maintainers intentionally accept the 16 MiB public-helper default or adjust the SDK contract and compatibility tests before merge.

## Reproduction Assessment

Yes. Current main and v2026.6.8 still call unbounded response.json() for successful provider JSON bodies, and the PR body includes a negative-control stream harness showing the old path reads the full 64 MiB body.

## Solution Assessment

Yes, with one maintainer-owned compatibility choice. The shared provider HTTP helper is the right owner boundary because callers already funnel through it and the binary sibling uses the same bounded reader; maintainers should confirm the exported SDK default cap.
