## Summary

Keep open: this PR has a blocking correctness issue because it would corrupt valid OpenRouter model slugs that current code and tests intentionally preserve.

## What This Changes

The PR changes OpenAI-compatible transport payload builders to remove a leading `openrouter/` segment from OpenRouter model IDs before sending Responses and Chat Completions requests.

## Best Possible Solution

Keep transport payloads using the resolved OpenRouter slug; if a full OpenClaw ref leaks into transport, fix model selection or provider resolution before the transport layer and add regression coverage for `openrouter/auto`.

## Reproduction Assessment

Yes for the blocking review finding: applying this patch to `openrouter/auto` or `openrouter/minimax/minimax-m2.7` would send `auto` or `minimax/minimax-m2.7` instead of the current resolved slug. No live failing OpenRouter request was provided for the PR's original claim.

## Solution Assessment

No. Stripping `openrouter/` in the shared transport builder is not the narrow maintainable fix because OpenRouter model slugs can legitimately start with that namespace and OpenClaw already resolves provider/model refs before transport.
