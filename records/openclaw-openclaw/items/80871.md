## Summary

Keep this PR open with changes requested: the patch strips valid OpenRouter model slugs in the request body, including the bundled default `openrouter/auto`, while current source, tests, and OpenRouter's docs all treat provider/model-style slugs as the body model value.

## What This Changes

The PR changes `src/agents/openai-transport-stream.ts` so OpenRouter request builders send `model.id` without a leading `openrouter/` segment.

## Best Possible Solution

Keep OpenRouter request bodies using the resolved OpenRouter slug; if a literal OpenClaw full ref ever leaks into transport, normalize that earlier in provider model resolution while preserving valid slugs such as `openrouter/auto`.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the patch would turn the canonical default `openrouter/auto` into `auto` before the Chat Completions request body is built. I did not establish a live failing OpenRouter request from the PR discussion.

## Solution Assessment

No. Stripping `openrouter/` inside the shared transport builder is not the narrow maintainable fix because OpenRouter model slugs can legitimately start with that namespace, and OpenClaw already normalizes model refs before the transport layer.
