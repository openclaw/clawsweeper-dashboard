## Summary

Keep open. Current main has a much stronger memory stack than the original report describes, including memory tools, multiple memory backends, opt-in transcript indexing, Active Memory, Dreaming, memory-wiki, and plugin seams. The core umbrella is still not fully resolved because graceful gateway-shutdown memory flush semantics and MEMORY.md-specific full/core-only/recall-only injection controls remain missing or only broadly approximated.

## What This Changes

The issue asks OpenClaw to improve long-term memory and knowledge management across persistence, shutdown flushing, automated consolidation, token-efficient recall, transcript search, and backend direction.

## Best Possible Solution

Keep this open or replace it with explicit canonical follow-ups: one for graceful gateway shutdown/session-stop memory flush semantics, one for MEMORY.md-specific full/core-only/recall-only injection configuration and docs, and one product/docs note that recommends the current memory-core, Active Memory, Dreaming, QMD, memory-wiki, LanceDB, and external plugin paths for larger memory architectures.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the two concrete remaining subclaims: source inspection shows shutdown proceeds through gateway_stop/shutdown without calling runMemoryFlushIfNeeded, and bootstrap loading still includes MEMORY.md with only broad contextInjection controls.

## Solution Assessment

Is this the best way to solve the issue? No, closing this umbrella as implemented or ClawHub-only would lose still-actionable core behavior. The maintainable path is to split or track the core reliability/config items while documenting plugin and external-memory options as ecosystem extensions.
