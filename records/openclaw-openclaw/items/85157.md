## Label Justifications

- `P2`: The bug is user-visible and can trap a plugin-bound conversation, but it is scoped to plugin-owned binding control rather than the whole runtime.
- `impact:message-loss`: The reported control command is delivered as ordinary text to the bound plugin session instead of reaching the command handler.
- `impact:session-state`: The affected command is supposed to inspect or remove conversation binding state, and the failure can leave that state stuck.

## Summary

Keep open: current main still routes plugin-owned conversation bindings before the normal command resolver, so an authorized `/codex detach` can be claimed as bound Codex text before the Codex command handler gets a chance to detach the binding.

## What This Changes

The issue requests an escape hatch so authorized native/plugin control commands bypass plugin-owned conversation binding delivery while ordinary bound text still routes to the owning plugin.

## Best Possible Solution

Add a generic pre-binding command escape check in the inbound dispatch path, reuse existing command/plugin command matching where possible, and add regression coverage for `/codex detach` plus ordinary bound text.

## Reproduction Assessment

Yes, from source inspection: a plugin-owned binding that returns `handled` prevents the downstream reply resolver from running, and `/codex detach` is only handled inside that downstream command path. I did not execute a live Telegram repro in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow command escape before plugin-owned binding delivery, not a Codex-only special case. The escape should be limited to authorized Gateway/plugin command turns so normal bound conversation text remains owned by the plugin.

## Repair Work Prompt

Fix the plugin-owned conversation binding command trap reported at https://github.com/openclaw/openclaw/issues/85157. In `src/auto-reply/reply/dispatch-from-config.ts`, add a generic escape hatch before plugin-owned binding `runInboundClaimForPluginOutcome` so authorized native/Gateway/plugin control commands, especially `/codex detach`, `/codex binding`, `/codex bind`, and `/codex stop`, fall through to existing command handling instead of being delivered as ordinary bound text.
