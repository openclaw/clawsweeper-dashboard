## Summary

Keep this open. Current `main` has adjacent local newer-config downgrade protection and the #74298 env-token migration diagnostic, but it still lacks a doctor/docs preflight that compares migrated source state or backup runtime version against the target binary before a machine cutover.

## What This Changes

Request a migration/DR preflight and documentation requiring OpenClaw source/target version alignment before transferring state between environments.

## Best Possible Solution

Add a focused doctor/docs migration preflight that reuses existing version parsing and available config/backup runtime metadata to warn or block unsafe source/target mismatches before cutover, while leaving normal startup and update policy unchanged.

Yes for the repository gap: current-main source inspection shows doctor/config/backup paths have no source-state or backup-runtime comparison against the target binary. The full incident cascade still needs the original production state to reproduce end to end.

## Solution Assessment
