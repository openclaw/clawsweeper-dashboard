## Summary

Keep this open. Current main and the latest release still have local future-config downgrade guards and adjacent CLI/runtime skew diagnostics, but not a migration preflight that compares transferred source state or backup runtime version against the target binary before a cutover proceeds.

## What This Changes

The issue requests migration documentation and doctor/preflight checks that require source and target OpenClaw versions to be aligned before transferring state.

## Best Possible Solution

Add a focused migration/cutover preflight that reuses existing version parsing, config metadata, and backup manifest runtime metadata, then document version alignment as required without adding broad startup blocking.
