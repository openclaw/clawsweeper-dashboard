## Summary

Keep this open. Current main still lacks the requested migration version-alignment guidance and a doctor-side source/target version preflight; the existing future-config guard is adjacent but only protects selected local service and gateway mutation paths.

## What This Changes

Add migration guidance and a doctor preflight that detects source/target OpenClaw version mismatches before accepting copied state.

## Best Possible Solution

Add a read-only doctor migration preflight that compares a supplied or stamped source version with the running OpenClaw version using the existing version helpers, then document same-version install before state transfer.

## Reproduction Assessment

Yes. The gap is reproducible by source inspection: current migration docs omit version alignment, doctor has no source/target version option, and the only version guard is wired to selected service/gateway mutation paths.

## Solution Assessment

Yes. A conservative doctor warning or failure plus docs update is the narrowest maintainable first fix; provider-auth repair and automatic state schema rewrites should remain separate follow-ups.
