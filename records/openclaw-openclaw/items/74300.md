## Summary

Keep this open. Current main has a newer-config guard and backup runtime metadata, but it still lacks a migration/cutover preflight that explicitly compares imported source state against the running target binary before operators accept transferred state.

## What This Changes

Add a migration preflight and docs so blue-green state transfers warn or fail when source and target OpenClaw versions are incompatible or misaligned.

## Best Possible Solution

Add an explicit doctor migration preflight that compares an imported source stamp or backup manifest runtimeVersion with the current binary, documents version alignment, and leaves normal startup/update behavior unchanged.

## Reproduction Assessment

Yes for the repository gap: current-main source inspection shows doctor reads local config snapshots but has no source/target migration version comparison. The full production cascade is only partially reproducible without the incident state files and environment.

## Solution Assessment

Yes, if scoped to a doctor/docs migration preflight rather than broad startup blocking. The safest repair is a high-signal warning or documented fail path for operator-driven state transfers, reusing existing version parsing and config metadata where possible.
