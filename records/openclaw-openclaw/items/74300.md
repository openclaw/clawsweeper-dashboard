## Summary

Keep open: current main and the latest release still lack a cross-environment source-version versus target-binary migration preflight, while existing guards only cover local newer-written config and specific legacy state migrations.

## What This Changes

The issue requests a migration/doctor preflight and docs so OpenClaw warns or blocks when copied state comes from a different source version than the target runtime can safely consume.

## Best Possible Solution

Define the supported machine-move version contract, then add a narrow doctor/docs preflight that reuses existing version parsing and current metadata where possible.

## Reproduction Assessment

Yes for the source-level diagnostic gap: current main has no source/target migration-version input in doctor preflight or doctor options. I did not run a live BLUE/GREEN VPS replay, so the incident mechanics remain supported by the report rather than freshly reproduced end to end.
