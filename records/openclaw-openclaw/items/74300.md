## Summary

Keep this open. Current main has adjacent config-version and missing-token safeguards, but it still lacks required migration version-alignment guidance and doctor-side source/target version preflight coverage.

## What This Changes

Add migration version-alignment guidance and a doctor preflight so copied state from a different OpenClaw version does not fail silently.

## Best Possible Solution

Add a read-only doctor migration version check that warns on any source/target mismatch, blocks downgrades consistently with the newer-config guard, and document same-version install before copying state.

## Reproduction Assessment

Yes. The reported gap is reproducible by source inspection on current main: migration docs omit version alignment, doctor has no version input, and the only version guard is limited to selected destructive local actions.

## Solution Assessment

Yes. A conservative doctor warning plus migration-doc update is the narrowest maintainable first fix; provider auth repair, env-token checks, and automatic state schema rewrites should stay separate.
