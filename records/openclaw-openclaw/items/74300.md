## Summary

Keep open: current main has local newer-config guards and backup runtime metadata, but it still lacks a cross-environment source-version versus target-binary preflight before copied state is accepted.

## What This Changes

The issue asks OpenClaw to catch source/target version mismatch during state migration so copied cron, auth, plugin, and session state cannot fail silently across deployments.

## Best Possible Solution

Define one supported machine-move version contract, then add a narrow doctor/docs preflight that compares imported source metadata against the running binary and distinguishes blocked downgrades from intentional upgrades.

## Reproduction Assessment

Yes. By source inspection, current main still lacks a doctor or migration-doc path that compares imported source-version metadata with the running target binary before copied state is accepted.

## Solution Assessment

Unclear. The doctor/migration boundary is the right place to solve it, but the source-version carrier and warn/fail semantics need maintainer product judgment before a safe implementation.
