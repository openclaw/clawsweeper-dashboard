## Summary

Keep this open. Current main still lacks the requested migration version-alignment guidance and doctor-side source/target version preflight; the existing version guard only covers selected older-binary destructive actions after config has already been imported.

## What This Changes

Request adding migration version-alignment docs and a doctor preflight to catch source/target OpenClaw version mismatches before accepting imported state.

## Best Possible Solution

Ship a read-only doctor migration preflight that compares an explicit or imported source OpenClaw version with the running binary, document same-version install before state transfer, and keep auth-profile/plugin schema repairs as separate targeted fixes.

## Reproduction Assessment

Yes. The gap is reproducible by source inspection: the migration docs omit version alignment, doctor cannot accept source/target version metadata, and the only current guard is limited to newer local config stamps on service/gateway mutation paths.

## Solution Assessment

Yes. A conservative doctor warning/blocking preflight plus migration-doc update is the narrowest maintainable first fix; automatic auth/profile or plugin state rewrites should remain separate follow-ups.
