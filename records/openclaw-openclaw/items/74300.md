## Summary

Keep this open. Current main and v2026.5.7 still have adjacent newer-config and CLI/gateway version-skew diagnostics, but no migration/cutover preflight that compares transferred source state or backup runtime version with the target OpenClaw binary before the operator proceeds.

## What This Changes

The issue requests migration preflight and documentation that detect source/target OpenClaw version mismatches before imported state is accepted.

## Best Possible Solution

Add a migration/cutover-scoped version preflight that reuses existing version parsing and backup/config metadata, then document version alignment as required before state transfer.

## Reproduction Assessment

Yes for the repository gap: current-main source inspection shows doctor/config/backup paths expose version metadata but do not compare a transferred source version or backup runtime version to the target binary. The full incident cascade is not independently reproducible without the reporter's real migrated state.

## Solution Assessment

Yes. A narrow doctor/backup/docs repair using existing version metadata is the maintainable path; a new general startup block or a required new doctor flag would be riskier than a migration-scoped preflight and clear operator docs.
