## Summary

Keep this open. Current main still lacks the requested migration version-alignment guidance and doctor-side source/target version preflight; the existing future-config guard is adjacent but only blocks selected local service/gateway mutation paths.

## What This Changes

Request to require source/target OpenClaw version alignment during machine migrations and add doctor-side compatibility preflight/docs.

## Best Possible Solution

Add a read-only doctor migration preflight that uses the existing version helpers to compare imported source/runtime version evidence with the running binary, then document same-version install before state transfer.

Yes. Source inspection on current main reproduces the gap: migration docs omit version alignment, doctor has no source/target version option, and the future-version guard only covers local lastTouchedVersion action blocks.

## Solution Assessment

Yes. A conservative doctor warning or downgrade block plus migration-doc update is the narrowest maintainable first fix; provider-auth repair and automatic state schema rewrites should stay separate.
