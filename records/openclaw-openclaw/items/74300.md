## Summary

Keep open: current main and v2026.6.6 still cover local newer-config/downgrade guards, but not a cross-environment source-version versus target-binary migration preflight before imported state is accepted.

## What This Changes

The issue asks for migration documentation and doctor/preflight checks that catch source and target OpenClaw version mismatch before transferred state causes silent incompatibility.

## Best Possible Solution

Define one supported machine-move version contract, then add a narrow doctor/docs path that compares imported source metadata against the running binary and distinguishes blocked downgrades from intended upgrades.

## Reproduction Assessment

Yes, by source inspection: current doctor preflight, migration docs, and backup verification do not enforce a cross-environment source/target version check. I did not run a live BLUE/GREEN VPS migration replay.

## Solution Assessment

Unclear that the suggested doctor flag is the best permanent shape. The safer path is for maintainers to choose the source-version carrier and warn/fail policy, then implement the narrow preflight and docs update.
