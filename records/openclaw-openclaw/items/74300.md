## Summary

Keep this open. Current main has adjacent downgrade protection and backup runtime metadata, but it still lacks an explicit migration/cutover preflight that compares imported source state against the running target binary before accepting transferred state.

## What This Changes

Request a migration/DR preflight and documentation that require or detect OpenClaw source/target version alignment before transferring state.

## Best Possible Solution

Add a narrow doctor migration preflight that reads a source-version stamp or backup manifest runtimeVersion, warns or blocks unsafe source/target mismatches with clear operator guidance, and document version alignment without changing normal startup/update migrations.

## Reproduction Assessment

Yes for the repository gap: current main can be inspected to show doctor lacks any source/target migration version comparison. The exact production cascade is only partially reproducible because the incident state files and environment are not included.

## Solution Assessment

Unclear for an exact required-alignment policy. The safest maintainable direction is an explicit doctor/cutover preflight plus docs, not a broad startup block or hidden load-time migration.
