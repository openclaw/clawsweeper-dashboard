## Summary

Keep this open. Current main has adjacent local downgrade protection and backup runtime metadata, but it still lacks a cross-environment source-version versus target-binary migration preflight, and the migration docs do not require version alignment before state transfer.

## What This Changes

The issue requests migration preflight and documentation to prevent silent state incompatibility when OpenClaw state is moved between different source and target versions.

## Best Possible Solution

Add a doctor migration preflight that compares imported source runtime metadata or a source-version stamp with the current binary, warns or blocks unsafe downgrade/cutover paths, and documents required version alignment before state transfer.

## Reproduction Assessment

Yes for the repository gap: current-main source inspection shows doctor and backup verify have no source/target version comparison. The full cutover incident is only partially reproducible without the original production state and environment.

## Solution Assessment

Yes, the requested doctor/docs preflight is the narrowest maintainable fix if it stays scoped to migration and backup verification paths. Broad startup blocking or general update policy changes would be riskier and are not necessary for this report.
