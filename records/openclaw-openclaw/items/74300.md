## Summary

Keep open: current main still has local future-config guards and backup runtime metadata, but it does not implement the requested cross-environment source-version versus target-binary preflight before copied state is accepted.

## What This Changes

The issue asks for migration preflight protection when copied OpenClaw state was written by a different source version than the target binary.

## Best Possible Solution

Define one supported machine-move version contract, then add a narrow doctor/docs preflight that compares imported source metadata against the running binary and distinguishes blocked downgrades from intentional upgrades.

## Reproduction Assessment

Yes. By source inspection, current main still lacks a doctor or migration-doc path that compares imported source-version metadata with the running target binary before copied state is accepted.

## Solution Assessment

Unclear. The issue identifies the right boundary, but the exact source-version carrier and warn/fail policy need maintainer product judgment before a safe implementation.
