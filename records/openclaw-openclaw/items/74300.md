## Summary

Keep this open. Current main still lacks the requested source/target migration version preflight and required version-alignment guidance; the newer local config guard and the closed token-env follow-up are adjacent but do not cover blue/green state transfer version mismatch.

## What This Changes

Request adding DR/migration version-alignment guidance and a doctor preflight or flag to compare source and target OpenClaw versions before accepting transferred state.

## Best Possible Solution

Add a read-only doctor migration preflight that compares a supplied or stamped source version with the running OpenClaw version, document same-version install before state transfer, and leave provider-auth/schema rewrites to separate follow-ups.

## Reproduction Assessment

Yes. The gap is reproducible by source inspection on current main: migration docs omit required version alignment, doctor has no migration version option, and the existing version guard only covers selected local destructive actions.

## Solution Assessment

Yes. A conservative doctor warning or failure plus docs update is the narrowest maintainable fix; automatic state rewrites and Anthropic auth-profile repair belong outside this issue.
