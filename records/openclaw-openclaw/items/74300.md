## Summary

Keep this open. Current main has newer-config downgrade protection and backup runtime metadata, but source inspection still shows no doctor or migration preflight that compares transferred source state or backup runtime version against the target binary before accepting the cutover state.

## What This Changes

The issue requests migration safeguards for source/target OpenClaw version mismatches during blue/green state transfer.

## Best Possible Solution

Add a narrow doctor and migration-docs preflight that reuses existing version parsing and state or backup metadata to warn or block unsafe cutovers before normal startup/update behavior changes.

## Solution Assessment
