## Summary

Keep open: current main still lacks a migration/cutover preflight that compares transferred source state or backup runtime version with the target OpenClaw binary. Adjacent version-skew diagnostics exist, but the migration guide still copies state before any version-alignment requirement and backup verification only reports the manifest runtime version.

## What This Changes

Add migration/DR version-alignment checks and documentation so source and target OpenClaw versions are compared before state transfer or import.

## Best Possible Solution

Add a migration/backup/doctor-scoped version compatibility warning or block using existing config and backup runtime metadata, then document version alignment as required before state transfer.

The full production incident cascade is not independently reproduced without the reporter's real migrated state.

## Solution Assessment
