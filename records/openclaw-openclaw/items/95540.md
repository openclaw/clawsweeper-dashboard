## Summary

Keep open: this PR is the current narrow candidate fix for a released upgrade regression, and current main plus v2026.6.9 still have the permissive missing-schema doctor migration path. I found no blocking patch defect, but changing the migration to fail closed for schemaless external channels is compatibility-sensitive and needs maintainer acceptance.

## What This Changes

The PR changes the doctor `groupAllowFrom` fallback migration to skip writes when bundled channel schema metadata is missing and adds a regression test for an external email channel.

## Best Possible Solution

Land the schema-verified doctor guard if maintainers accept plugin-owned external repairs; otherwise add an explicit external plugin doctor/schema opt-in before merging.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.9 permit missing-schema writes and then assign `groupAllowFrom` onto records. The PR body also includes before/after output from driving the actual migration helper, though this review did not run the full macOS ClawMail upgrade path.

## Solution Assessment

Yes, if maintainers accept the owner boundary: the changed guard is the exact write decision point and avoids mutating external plugin config that core cannot validate. The safer alternative is an explicit plugin doctor/schema opt-in if maintainers want schemaless external plugins to receive this migration.
