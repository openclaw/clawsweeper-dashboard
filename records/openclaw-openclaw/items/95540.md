## Summary

Keep open for maintainer review: current main and v2026.6.9 still have the schema-missing doctor write path, and this PR is a focused candidate fix for the linked upgrade regression, but changing the default to fail closed for schemaless external channels is compatibility-sensitive.

## What This Changes

The PR changes the doctor group allowFrom fallback migration to skip `groupAllowFrom` writes when a channel is absent from bundled schema metadata, and adds an external `email` channel regression test.

## Best Possible Solution

Land the schema-verified doctor guard after maintainer acceptance of the external-plugin compatibility tradeoff, or replace it with an explicit external plugin schema/doctor opt-in if maintainers want those migrations.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.9 permit missing-schema writes and then assign `groupAllowFrom` onto eligible records. The PR body also includes before/after output from driving the actual migration helper, though this review did not run a full packaged ClawMail upgrade.

## Solution Assessment

Yes, if maintainers accept the owner boundary: the changed guard is the exact write decision point and avoids mutating external plugin config that core cannot validate from bundled metadata. The safer alternative is an explicit plugin doctor/schema opt-in if maintainers want schemaless external plugins to receive this migration.
