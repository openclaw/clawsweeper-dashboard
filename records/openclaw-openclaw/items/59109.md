## Summary

Keep open. Current main and v2026.5.22 still have only partial primitives: explicit session selectors, ACP-only resume, internal subagent transcript fork, and compaction-checkpoint branch/restore, not the requested public SDK/Gateway fork, resume option, or continue-most-recent workflow.

## What This Changes

Request a public session lifecycle API comparable to open-agent-sdk: fork an existing session, resume by session ID, and continue the most recent session without knowing its ID.

## Best Possible Solution

Design one additive OpenClaw-native SDK and Gateway session lifecycle contract that reuses existing session selectors while explicitly defining public fork, resume-by-selector, and continue-most-recent behavior with docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, release-tag, related-issue, and reference-API checks give high confidence that OpenClaw has partial primitives but not the requested public fork/resume/continue workflow.

## Solution Assessment

Unclear pending maintainer API direction. The open-agent-sdk shape is useful prior art, but OpenClaw should define native fork, resume-by-selector, and continue-most-recent semantics that fit existing Gateway visibility, SDK, and snapshot work.
