## What This Changes

The issue requests public session forking, resume-by-ID ergonomics, and continue-most-recent session behavior similar to open-agent-sdk.

## Best Possible Solution

Design one additive OpenClaw-native SDK and Gateway session lifecycle contract that reuses `sessionId`, `sessionKey`, `sessions.resolve`, and session listing while explicitly defining arbitrary fork and continue-most-recent semantics with docs and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, release-tag, and related-issue inspection give high confidence that current OpenClaw has partial session primitives but not the requested public fork/resume/continue workflow.

## Solution Assessment

Unclear pending maintainer API direction. The open-agent-sdk shape is useful prior art, but OpenClaw should define native fork, resume-by-selector, and continue-most-recent semantics that fit existing Gateway visibility and snapshot work.
