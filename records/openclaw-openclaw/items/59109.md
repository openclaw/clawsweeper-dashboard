## Summary

Keep open. Current main and the latest release still expose only partial session primitives: explicit `sessionId`/`sessionKey`, internal/subagent forking, ACP-only resume, and checkpoint-scoped branch/restore, but not the requested public `forkSession(id)`, SDK `resume` option, or `continue: true` workflow.

## What This Changes

Request public OpenClaw session lifecycle APIs for forking an existing session, resuming by session ID, and continuing the most recent session without knowing its ID.

## Best Possible Solution

Define one additive OpenClaw-native session lifecycle contract across Gateway and the App SDK, reusing existing session IDs, keys, resolver, store metadata, checkpoint branch/restore, and subagent/ACP primitives where they fit.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, latest-release, and related-issue inspection give high confidence that current OpenClaw has partial session primitives but not the requested public fork/resume/continue workflow.

## Solution Assessment

Unclear pending maintainer API direction. The open-agent-sdk shape is useful prior art, but the maintainable solution is an OpenClaw-native contract that defines visibility, storage, lineage, and checkpoint behavior explicitly.
