## Summary

Keep open: current main and v2026.6.8 still only cover parts of the request. OpenClaw has explicit `--session-id` routing, internal transcript fork primitives, and compaction-checkpoint branch/restore, but no public SDK/Gateway/CLI contract for `forkSession(id)`, resume-by-ID semantics, or continue-most-recent behavior.

## What This Changes

Request a public session lifecycle surface for forking an existing session, resuming by session ID, and continuing the most recent session.

## Best Possible Solution

Design one maintainer-approved public session lifecycle contract across Gateway, SDK, and CLI that reuses existing session-id resolution, transcript branching, and checkpoint primitives while defining lineage, permissions, and continue-latest semantics.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, tests, latest-release inspection, and related issue context confirm that current OpenClaw has partial primitives but not the full public fork/resume/continue surface requested here.

## Solution Assessment

Unclear until maintainers choose the public API and UX. The maintainable path is to compose existing session-id resolution, Gateway protocol, SDK, and transcript-branch helpers instead of exposing an internal helper or adding a one-off CLI flag.
