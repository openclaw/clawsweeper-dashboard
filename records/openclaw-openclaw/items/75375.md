## Summary

Keep open. Current main still runs the bundled boot-md BOOT.md agent turn inside the Gateway process with no boot-specific background boundary or hard budget, so the reported restart availability risk remains actionable.

## What This Changes

The issue requests that the bundled boot-md gateway startup hook stop making Gateway responsiveness depend on BOOT.md agent work finishing.

## Best Possible Solution

Run BOOT.md execution on a bounded background path after Gateway readiness, preserving per-agent scope, workspace dedupe, session-key routing, silent reply semantics, and session mapping cleanup.

## Reproduction Assessment

Yes. Configure boot-md with a BOOT.md whose agent turn remains slow or pending; current main's handler -> runStartupTasks -> runBootOnce chain awaits that promise, and existing tests assert the awaited path rather than readiness isolation.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: detach or budget boot-md execution specifically rather than making all startup tasks fire-and-forget.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75375. The bundled boot-md gateway startup hook currently awaits runStartupTasks, which awaits runBootOnce, which awaits the BOOT.md agentCommand for each configured workspace. Change the boot-md path so a slow BOOT.md agent run cannot hold Gateway responsiveness after restart. Prefer a boot-md-specific bounded background scheduler or timeout/abort boundary; do not make all runStartupTasks callers fire-and-forget because restart-sentinel also uses that helper. Preserve per-agent workspace dedupe, cfg/deps propagation, session-key routing, silent reply prompt behavior, and session mapping snapshot/restore cleanup. Add regression coverage with a deferred or slow runBootOnce proving startup hook dispatch/Gateway readiness is not held by the BOOT.md run, and keep existing per-agent and #50721 outbound behavior intact. Add a user-facing CHANGELOG.md fix entry. Validate with the targeted boot/gateway tests and changed gate in Testbox before handoff.
