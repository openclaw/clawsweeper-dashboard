## Summary

Keep open. Current main still runs boot-md inside the Gateway process and awaits the BOOT.md agent turn through the startup-task path, with no boot-specific timeout, detached task ledger, or isolation for slow boot work.

## What This Changes

Make the bundled boot-md gateway startup hook stop degrading restart availability when BOOT.md agent work is slow or pending.

## Best Possible Solution

Run boot-md on a bounded detached background path after Gateway readiness while preserving per-agent workspace dedupe, session-key routing, silent reply behavior, outbound delivery, and session mapping cleanup.

## Reproduction Assessment

Yes. Configure boot-md with a BOOT.md whose agent turn is slow or pending; current main follows runBootChecklist -> runStartupTasks -> runBootOnce -> agentCommand and awaits that work inside the startup hook path.

## Solution Assessment

Yes. A boot-md-specific detached or budgeted execution path is the narrowest maintainable fix; changing all internal hook dispatch semantics would risk unrelated lifecycle hooks.
