## Summary

Keep open: this is a focused ACPX diagnostic bug-fix PR, but it is not merge-ready because the patch persists raw Codex/acpx stderr to a shared file before redaction, the latest head fails lint on a constant-condition test workaround, and the supplied proof does not show the patched OpenClaw `sessions_spawn` path producing the new sanitized diagnostic. The linked root bug remains tracked separately at https://github.com/openclaw/openclaw/issues/80079, with another open implementation PR at https://github.com/openclaw/openclaw/pull/80195.

## What This Changes

The PR captures generated Codex ACP wrapper stderr to a log, appends a redacted tail to generic ACP initialization and turn errors, and adds ACPX/spawn regression tests.

## Best Possible Solution

Surface Codex ACP diagnostics through a per-launch or per-session redacted buffer or diagnostic callback, keep stderr mirroring, align with upstream `codex-acp` changes where needed, and prove the patched `sessions_spawn` failure path with redacted real output.

## Reproduction Assessment

Yes, by source and linked-report evidence, but not by a live run in this read-only review. Current main has no stderr-tail enrichment path, and the linked issue shows direct acpx succeeds while OpenClaw `sessions_spawn` returns a generic `AcpRuntimeError: Internal error`.

## Solution Assessment

No. The diagnostic direction is useful, but this implementation persists raw stderr to a shared file, fails lint on the current head, and still needs real after-fix proof of the OpenClaw runtime path.

## Review Findings

Overall correctness: not a patch

Full review comments:

- **[P1] Redact stderr before writing the diagnostic log:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: This appends child stderr verbatim to `codex-acp-wrapper.stderr.log`. The later `readCodexWrapperStderrTail` redaction only protects the copied error message, so provider tokens or private config emitted by Codex/acpx can remain on disk; redact before persistence or keep only an in-memory redacted tail.
  - confidence: 0.92
- **[P2] Make the Codex stderr capture per launch:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: Every Codex wrapper invocation uses the same `codex-acp-wrapper.stderr.log` under the shared wrapper root, and startup truncates that file. Concurrent sessions can erase or mix another session's diagnostic tail and surface the wrong error; key the capture by lease/session/request and read that exact buffer.
  - confidence: 0.86
- **[P2] Remove the constant-condition generator workaround:** `extensions/acpx/src/runtime.test.ts:315-316`
  - body: The latest commit adds `if (false)` inside the async generator test, and CI is failing `check-lint` and `lint:extensions:bundled` with `Unexpected constant condition`. Use a generator shape that satisfies the type/lint rules without an always-false branch.
  - confidence: 0.9
