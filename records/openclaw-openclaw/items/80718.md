## What This Changes

The PR changes ACPX Codex runtime diagnostics to persist Codex wrapper stderr, attach redacted stderr tails to generic ACP internal errors, inherit selected Codex config, and add focused tests.

## Best Possible Solution

## Solution Assessment

No, not yet. Per-lease bounded diagnostics are the right direction, but this implementation needs stream-aware redaction, `close`-based stderr draining, and installed-path proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Redact stderr across stream chunks:** `extensions/acpx/src/codex-auth-bridge.ts:338`
  - body: The wrapper redacts each stderr `data` chunk before appending it to the per-lease log. Because child stderr is a Readable stream, a token, bearer header, URL secret, or PEM block can cross chunk boundaries and be persisted before the regex sees the full secret; keep a small raw carry/buffer and only write redacted finalized text.
  - confidence: 0.9
- **[P2] Wait for the stderr pipe before exiting:** `extensions/acpx/src/codex-auth-bridge.ts:481-494`
  - body: This changes child stderr from inherited stdio to a pipe, but the wrapper still calls `process.exit` from the child `exit` handler. Node can emit `exit` while stdio streams are still open, so diagnostics written right before adapter shutdown can be lost from both the mirror and the per-lease log; preserve the exit code and finish from `close` after stderr drains.
  - confidence: 0.87
