## Summary

Keep open. The PR targets a real ACPX diagnostics gap, but it still introduces a secret-at-rest risk by redacting stderr one stream chunk at a time, and the supplied proof does not exercise the patched `sessions_spawn` failure path.

## What This Changes

The PR captures Codex ACP wrapper stderr into per-lease logs, mirrors it to stderr, and appends redacted log tails to generic ACP session/turn errors with focused tests.

## Best Possible Solution

Capture bounded Codex wrapper diagnostics without persisting unredacted split secrets, enrich generic ACP errors, and prove the patched OpenClaw spawn path in a real install.

## Reproduction Assessment

Yes, at source level: current main inherits wrapper stderr and forwards generic `Internal error` failures from ACPX `ensureSession` and `runTurn` without retained wrapper diagnostics. I did not run a live installed-gateway reproduction.

## Solution Assessment

No, not yet. Per-lease diagnostics are the right direction, but persisted redaction must handle stream chunk boundaries and the PR needs proof that patched `sessions_spawn` surfaces a redacted stderr tail.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Redact stderr across chunk boundaries:** `extensions/acpx/src/codex-auth-bridge.ts:335`
  - body: The wrapper redacts each `stderr` `data` chunk before appending it to the per-lease log. Since child-process stderr is a stream and data can be split across arbitrary chunks, a token, bearer header, or PEM block split between chunks can be persisted in cleartext; keep a small raw carry/buffer and redact over joined text before writing.
