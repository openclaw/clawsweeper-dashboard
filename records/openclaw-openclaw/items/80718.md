## Summary

Keep open: this PR targets a real, source-backed acpx/Codex diagnostics bug, but the patch still persists raw shared stderr in plugin state and lacks after-fix proof for the patched `sessions_spawn` path.

## What This Changes

The PR captures Codex ACP wrapper stderr, appends a redacted tail to generic acpx session/turn errors, and adds focused acpx/acp-spawn tests.

## Best Possible Solution

Store bounded redacted diagnostics per Codex wrapper run, surface only the matching tail for generic failures, and prove the patched `sessions_spawn` path with redacted real output.

## Reproduction Assessment

Yes, at source level: current main inherits wrapper stderr and propagates generic `Internal error` from `ensureSession` and `runTurn` without attaching Codex stderr. I did not run a live patched gateway reproduction in this read-only review.

## Solution Assessment

No. Capturing diagnostics at the wrapper/runtime boundary is reasonable, but this implementation should not persist raw shared stderr and needs real proof of the patched `sessions_spawn` path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Redact stderr before writing the wrapper log:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: The generated wrapper appends each Codex stderr chunk to `codex-acp-wrapper.stderr.log` before any redaction. Codex/acpx diagnostics can include tokens or credential-bearing provider details, so a failed run can leave secrets in plugin state even though the surfaced `AcpRuntimeError` redacts the tail later.
  - confidence: 0.86
- **[P2] Use a per-run stderr log:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: Every Codex wrapper process writes to the same `codex-acp-wrapper.stderr.log` under the shared wrapper root. Concurrent spawned Codex sessions can clear or interleave another session's stderr before the runtime reads the tail, so the diagnostic can be missing or belong to the wrong run.
  - confidence: 0.79

## Security Review

Status: needs_attention
