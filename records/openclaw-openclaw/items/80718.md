## Summary

Keep open: the PR targets a real ACPX/Codex diagnostic gap, but the current patch persists raw adapter stderr before redaction, uses one shared log file across launches, and still lacks after-fix proof for the patched OpenClaw `sessions_spawn` path.

## What This Changes

The PR captures Codex ACP wrapper stderr into a generated-wrapper log and appends a redacted tail to generic ACP session or turn errors, with focused ACPX runtime and spawn-order tests.

## Best Possible Solution

Capture Codex ACP diagnostics per launch or per session, redact before persistence or avoid disk entirely, keep stderr mirroring, and prove a real redacted `sessions_spawn` failure surfaces the expected stderr tail.

## Reproduction Assessment

Yes, by source and linked-report evidence: current main has no stderr-tail propagation, and the linked issue shows direct acpx succeeding while OpenClaw `sessions_spawn` fails with generic `Internal error`. I did not live-run that gateway path in this read-only review.

## Solution Assessment

No. The PR improves the right diagnostic path, but the implementation should not persist raw shared stderr; a per-launch redacted buffer or redacted diagnostic file is the safer maintainable direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Redact stderr before writing the diagnostic log:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: This writes child stderr verbatim into `codex-acp-wrapper.stderr.log`; the later redaction only protects the `AcpRuntimeError` string. If Codex/acpx prints provider tokens, bearer tokens, endpoints, or config values, they can remain on disk in the plugin state directory.
  - confidence: 0.9
- **[P2] Key the stderr capture to each launch:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: All Codex wrapper invocations use the same `codex-acp-wrapper.stderr.log` and truncate it at startup. Concurrent sessions can erase or mix another session's failure tail, causing OpenClaw to surface the wrong diagnostic.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete secret-handling risk by persisting raw Codex/acpx stderr before redaction and reusing one diagnostic file across launches.

Concerns:

- **[medium] Raw stderr can persist secrets:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: Codex/acpx stderr may include provider keys, bearer tokens, private endpoints, or config values; this patch writes the raw stream to disk and only redacts when later copying the tail into an error message.
  - confidence: 0.9
- **[medium] Shared log can leak cross-session diagnostics:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: The fixed `codex-acp-wrapper.stderr.log` filename is shared by all Codex wrapper launches under the same ACPX state root, so overlapping sessions can read or overwrite each other's diagnostic tail.
