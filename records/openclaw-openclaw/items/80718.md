## Summary

Keep open: this is a plausible focused ACPX diagnostic fix for an open regression, but the current PR head has blocking lint failure and the stderr capture is not run-scoped. The supplied proof also does not demonstrate the changed `sessions_spawn` failure path after the patch.

## What This Changes

The PR captures Codex ACP wrapper stderr and appends a redacted tail to generic ACP initialization or first-turn internal errors, with focused runtime and spawn-order tests.

## Best Possible Solution

Keep the diagnostic fix in the ACPX plugin glue only if stderr capture is per-run, stale-safe, and sanitized appropriately, with lint-clean regression coverage and redacted proof from the patched OpenClaw ACP spawn path.

## Reproduction Assessment

Yes, there is a high-confidence source and issue path: direct acpx can work while OpenClaw’s generated Codex ACP wrapper path reports only a generic `Internal error`. I did not live-run the installed gateway flow in this read-only review.

## Solution Assessment

No, not as currently written. The owner boundary is right for OpenClaw-specific wrapper diagnostics, but the implementation must use run-scoped diagnostics, pass lint, and prove the changed runtime path before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Make the stderr capture per-run:** `extensions/acpx/src/codex-auth-bridge.ts:350`
  - body: All generated Codex wrapper processes use the same `codex-acp-wrapper.stderr.log` under the wrapper root. A second Codex ACP session can truncate that file at startup while another run is still active, and concurrent stderr can interleave, so the enriched `AcpRuntimeError` may show the wrong run's tail or no tail at all. Use a lease/session-scoped log path and read that same expected path when enriching the error.
  - confidence: 0.91
- **[P2] Remove the constant-false yield:** `extensions/acpx/src/runtime.test.ts:313-315`
  - body: The added async-generator stub contains `if (false) { yield {} as never; }`, and CI fails both core lint and bundled extension lint with `Unexpected constant condition`. Replace this with a lint-clean async generator test stub so the PR can pass the required checks.
  - confidence: 0.99

## Security Review

Status: needs_attention

Summary: The patch adds persistent raw stderr capture before redaction, which needs a safer handling strategy before merge.

Concerns:

- **[medium] Raw wrapper stderr is retained on disk:** `extensions/acpx/src/codex-auth-bridge.ts:181`
  - body: The generated wrapper appends raw Codex adapter stderr to a state-dir log, then the runtime redacts only when reading the tail into `AcpRuntimeError`. Provider or adapter stderr can include credentials or private endpoint details, so the persisted diagnostic file should be sanitized, scoped tightly, deleted after use, or otherwise explicitly protected.
  - confidence: 0.74
