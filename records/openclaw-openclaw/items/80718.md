## Summary

Keep open: this targets a valid source-backed ACPX diagnostics bug, but latest head still writes redacted stderr one stream chunk at a time and the supplied proof does not exercise the patched OpenClaw `sessions_spawn` failure path.

## What This Changes

The branch adds per-lease Codex ACP wrapper stderr logging, enriches generic ACP init/turn errors from that log, and adds focused ACPX/acp-spawn tests.

## Best Possible Solution

Capture bounded Codex wrapper diagnostics with stream-aware redaction before writing logs, enrich generic ACP errors from that safe log, and prove the installed OpenClaw spawn path after the patch.

## Reproduction Assessment

Yes, at source level: current main inherits wrapper stderr and rethrows generic ACP `Internal error` failures without retained diagnostics, while the linked issue adds installed-path reproduction. I did not rerun the live gateway flow in this read-only review.

## Solution Assessment

No, not yet. Per-lease diagnostics are the right direction, but the implementation needs stream-boundary-safe redaction before persisted writes and real installed-path proof of the improved error.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Redact stderr across stream chunks:** `extensions/acpx/src/codex-auth-bridge.ts:338`
  - body: The wrapper redacts each stderr `data` chunk before appending it to the per-lease log. Since child-process stderr is a Readable stream, a token, bearer header, or PEM block can be split across chunks and persisted before the regexes ever see the full secret; keep a small raw carry/buffer and write only redacted finalized text.
  - confidence: 0.9
