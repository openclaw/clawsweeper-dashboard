## Summary

Keep open. This is a plausible fix direction for the linked ACPX/Codex bug, but the latest head is not merge-ready because persisted stderr redaction is not stream-boundary safe, the wrapper can exit before a piped stderr stream is drained, and the supplied proof does not show the patched installed `sessions_spawn` path.

## What This Changes

The PR captures Codex ACP wrapper stderr into per-lease logs, appends redacted stderr tails to generic ACP internal errors, inherits selected Codex runtime config into isolated ACP config, and adds focused ACPX/acp-spawn tests.

## Best Possible Solution

Capture bounded Codex wrapper diagnostics with stream-aware redaction before any persisted write, wait for the stderr pipe to close before exiting the wrapper, and prove the installed OpenClaw ACP spawn path with redacted runtime output.

## Reproduction Assessment

Yes, at source level: current main lacks retained wrapper stderr and the PR's changed wrapper uses a piped Readable stderr stream while redacting each chunk independently. I did not run the live installed gateway flow in this read-only review.

## Solution Assessment

No, not yet. Per-lease diagnostics are the right direction, but the implementation should redact across stream boundaries, drain stderr before wrapper exit, and include real installed-path proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Redact stderr across stream chunks:** `extensions/acpx/src/codex-auth-bridge.ts:338`
  - body: The wrapper redacts each stderr `data` chunk before appending it to the per-lease log. Since child stderr is a Readable stream, a token, bearer header, URL secret, or PEM block can cross chunk boundaries and be persisted before the regex ever sees the full secret; keep a small raw carry/buffer and only write redacted finalized text.
  - confidence: 0.9
- **[P2] Wait for the stderr pipe before exiting:** `extensions/acpx/src/codex-auth-bridge.ts:416`
  - body: This changes child stderr from inherited stdio to a pipe, but the wrapper still calls `process.exit` from the child `exit` handler. Node can emit `exit` before stdio streams close, so diagnostics written right before adapter exit can be lost from both the mirror and the per-lease log; preserve the exit code and finish from `close` after stderr drains.
  - confidence: 0.87
