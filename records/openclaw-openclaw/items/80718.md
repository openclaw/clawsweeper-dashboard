## Summary

Keep open: the source-level ACPX diagnostic gap is real and this active PR is a plausible fix path, but the latest patch still has at-rest diagnostic redaction risks and lacks proof of the patched OpenClaw `sessions_spawn` failure path.

## What This Changes

The PR captures Codex ACP wrapper stderr in per-lease logs and appends redacted tails to generic ACP session initialization and turn failures, with ACPX and spawn-order tests.

## Best Possible Solution

Keep the per-lease diagnostic approach, but persist only a bounded buffer redacted with OpenClaw-equivalent coverage, avoid private-key-shaped source literals, and require redacted live or terminal proof of the patched OpenClaw spawn failure path.

## Reproduction Assessment

Yes at source level: current main inherits wrapper stderr and rethrows generic Codex ACP `Internal error` failures from `ensureSession` and `runTurn` without a retained stderr tail. I did not run the live installed-gateway reproduction.

## Solution Assessment

No, not yet. Per-lease stderr capture is the right direction, but the patch should not persist incompletely redacted diagnostics and still needs contributor-supplied proof of the patched OpenClaw path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Redact a buffered stderr tail before writing it:** `extensions/acpx/src/codex-auth-bridge.ts:335-338`
  - body: The wrapper redacts each `data` chunk independently before appending it. Stream chunks can split a credential across boundaries, so a token, JWT, or PEM block that spans chunks can be persisted raw in plugin state; the later read-side redaction only protects the thrown error message. Keep a bounded buffer and redact the accumulated tail before writing or rotating it.
  - confidence: 0.78
- **[P2] Avoid private-key-shaped source literals:** `extensions/acpx/src/codex-auth-bridge.ts:190`
  - body: This new production replacement string contains actual `-----BEGIN PRIVATE KEY-----` and `-----END PRIVATE KEY-----` delimiters. The repo's `detect-private-key` hook scans non-test files, and the latest `security-scm-fast` lane is failing at this head; express the replacement without committing a key-shaped block.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The patch adds persistent diagnostic stderr logs, so redaction must be safe before data reaches plugin state.

Concerns:

- **[medium] Chunk-local redaction can leak split secrets:** `extensions/acpx/src/codex-auth-bridge.ts:335`
  - body: Because stderr is redacted per stream chunk before append, secret values split across chunks can survive in the persisted log even if the later error-message path redacts the combined tail.
  - confidence: 0.78
- **[low] Private-key sentinel in production source:** `extensions/acpx/src/codex-auth-bridge.ts:190`
  - body: The production redaction replacement includes private-key block delimiters in source, which is incompatible with the repository's private-key detection policy and is consistent with the failing fast security lane.
  - confidence: 0.84
