## Summary

Keep open. The acpx-owned diagnostic fix is directionally useful for https://github.com/openclaw/openclaw/issues/80079, but this PR has a blocking security issue in raw stderr persistence and only mock/direct-smoke proof rather than real patched `sessions_spawn` behavior.

## What This Changes

The branch captures Codex ACP wrapper stderr to a per-run log, appends a redacted tail to generic ACP internal errors during session initialization and first turns, and adds focused acpx/acp-spawn tests.

## Best Possible Solution

Keep the acpx diagnostic approach, but store only redacted and bounded stderr, then prove the patched OpenClaw `sessions_spawn` path with redacted terminal/log output from a real Codex ACP failure.

## Reproduction Assessment

Source-reproducible. Current main forwards generic `Internal error` failures from `ensureSession`/`runTurn` without any captured wrapper stderr tail, and the linked issue supplies the observed OpenClaw/Codex failure path, but I did not run a live gateway repro.

## Solution Assessment

No. The acpx wrapper/runtime boundary is the right place to preserve diagnostics, but the implementation should not persist raw stderr and the proof should exercise the actual patched `sessions_spawn` path.

## Review Findings

Overall correctness: patch is incorrect
