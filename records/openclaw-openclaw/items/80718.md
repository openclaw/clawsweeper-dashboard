## Summary

Keep open: the PR is pointed at a real ACPX/Codex diagnostic gap, but the current patch still has a blocking generated-wrapper redaction bug and does not include proof of the patched `sessions_spawn` failure path surfacing a redacted stderr tail.

## What This Changes

The PR captures Codex ACP wrapper stderr in per-lease logs, appends a redacted tail to generic ACP session/turn errors, and adds focused ACPX/acp-spawn tests.

## Best Possible Solution

Fix the generated-wrapper redaction so stderr is safely redacted before persistence, keep the per-lease diagnostic boundary, and add redacted live output or logs proving the patched `sessions_spawn` error path.

## Reproduction Assessment

Yes, source-level: current main inherits Codex wrapper stderr and forwards generic `Internal error` failures from ACPX `ensureSession`/`runTurn` without a diagnostic tail. I did not run the live patched gateway path.

## Solution Assessment

No. Capturing bounded per-run wrapper diagnostics is the right direction, but this patch must first fix generated-wrapper redaction and prove the changed OpenClaw `sessions_spawn` failure path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Escape the generated redaction regexes:** `extensions/acpx/src/codex-auth-bridge.ts:206-210`
  - body: These regexes are inside the template literal that emits `codex-acp-wrapper.mjs`, so escapes like `\s` and `\b` are consumed while generating the file; CI is already flagging the `\s` cases. The emitted wrapper will not reliably redact bearer tokens or provider keys before writing stderr to the per-lease log, leaving raw secrets at rest. Use double escapes, `String.raw`, or a shared generated-safe redaction helper.
  - confidence: 0.91
- **[P2] Pass the lease id on Windows too:** `extensions/acpx/src/codex-auth-bridge.ts:196-199`
  - body: The wrapper only finds the lease id from `OPENCLAW_ACPX_LEASE_ID` or the injected lease arg, but `withAcpxLeaseEnvironment` returns the command unchanged on `win32`. In that path the wrapper falls back to a `pid-*` log while the runtime reads `codex-acp-wrapper.stderr.<leaseId>.log`, so Windows Codex failures still surface as plain `Internal error`.
  - confidence: 0.78
