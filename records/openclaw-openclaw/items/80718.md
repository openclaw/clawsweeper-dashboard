## Summary

Keep open: this is a focused ACPX diagnostic bug fix, but the current PR head still redacts stderr one stream chunk at a time before persisting it, so split credentials can be reconstructed in the per-lease log. The supplied proof also does not show the patched OpenClaw `sessions_spawn` failure path surfacing a redacted stderr tail.

## What This Changes

The PR captures generated Codex ACP wrapper stderr in per-lease logs, appends redacted tails to generic ACP session/turn errors, and adds focused ACPX and ACP spawn tests.

## Best Possible Solution

Keep the per-lease diagnostic direction, but redact over a buffered stream window before any log write and prove the patched `sessions_spawn` path in a real install with private details redacted.

## Reproduction Assessment

Yes, at source level: current main inherits wrapper stderr and forwards generic ACP `Internal error` failures without retained diagnostics. The PR's blocking security finding is also source-demonstrable by splitting a token across independently redacted chunks.

## Solution Assessment

No, not yet. Per-lease diagnostics are the right approach, but the write path must handle chunk-boundary redaction before persisting stderr and the changed OpenClaw spawn path still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Redact stderr across chunk boundaries:** `extensions/acpx/src/codex-auth-bridge.ts:338`
  - body: The generated wrapper redacts each `stderr` data chunk before appending it to the per-lease log. If a token, bearer header, or PEM block crosses chunk boundaries, neither chunk may match the redaction rule and the appended file can reconstruct the secret in cleartext; keep a small raw carry/buffer and redact over the joined window before writing.
  - confidence: 0.91
