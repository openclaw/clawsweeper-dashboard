## Summary

Keep open: this is a plausible narrow ACPX diagnostics fix, but the PR still has a security-sensitive redaction bug in persisted stderr logging and lacks after-fix proof for the changed installed `sessions_spawn` path.

## What This Changes

The branch adds per-lease Codex ACP wrapper stderr logging, surfaces a redacted stderr tail on generic ACP init/turn errors, and adds focused ACPX/acp-spawn tests.

## Best Possible Solution

Capture bounded per-lease Codex wrapper diagnostics with stream-aware redaction before persistence, enrich only generic Codex ACP errors from that safe log, and require installed `sessions_spawn` proof.

## Reproduction Assessment

Yes, at source level and from linked issue context: current main inherits wrapper stderr and propagates generic `Internal error` without retained diagnostics, while the linked issue/comment shows direct acpx works and OpenClaw `sessions_spawn` fails. I did not live-run the installed gateway flow in this read-only review.

## Solution Assessment

No, not yet. Per-lease diagnostics are the right direction, but persisted stderr needs stream-boundary-safe redaction before writes and installed-path proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Redact stderr across stream chunks:** `extensions/acpx/src/codex-auth-bridge.ts:338`
  - body: The wrapper redacts each stderr `data` chunk before appending it to the per-lease log. Since child-process stderr is chunked stream data, a token, bearer header, URL secret, or PEM block split across chunks can be persisted before the regexes see the full secret; keep a small raw carry/buffer and only write redacted finalized text.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR adds persisted stderr diagnostics, and the current per-chunk redaction can store split secrets in cleartext.

Concerns:

- **[medium] Split stderr chunks can persist secrets:** `extensions/acpx/src/codex-auth-bridge.ts:338`
  - body: Redacting one stderr chunk at a time is not sufficient for at-rest diagnostic logs because secret patterns can cross stream chunk boundaries before being appended to plugin state.
