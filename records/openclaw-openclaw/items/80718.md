## Summary

Keep open: the PR targets a real ACPX/Codex diagnosability gap, and the linked issue has useful reproduction context, but the patch is not merge-ready because it persists raw Codex stderr in a shared state-directory log and the supplied proof does not demonstrate the changed OpenClaw `sessions_spawn` path after the fix.

## What This Changes

The PR captures Codex ACP wrapper stderr in a generated wrapper log and appends a redacted tail to generic ACP initialization or turn errors, with focused runtime and spawn tests.

## Best Possible Solution

Surface Codex ACP adapter stderr through a per-launch or per-session diagnostic channel that redacts before persistence or avoids disk, keeps stderr mirroring, and is proven with a redacted real `sessions_spawn` failure path.

## Reproduction Assessment

Yes, by source and linked-report evidence, but not by a live run in this read-only review. Current main has no stderr-tail capture path, and the linked issue shows direct acpx succeeds while OpenClaw `sessions_spawn` returns a generic `AcpRuntimeError: Internal error`.

## Solution Assessment

No. The intended diagnostic improvement is the right area, but this implementation should not write raw stderr to a shared persistent log and still needs real after-fix proof of the OpenClaw runtime path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Redact stderr before writing the diagnostic log:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: This writes child stderr verbatim into `codex-acp-wrapper.stderr.log`. The later `readCodexWrapperStderrTail` redaction only protects the `AcpRuntimeError` string, so provider tokens or API keys printed by Codex/acpx can remain on disk in the plugin state directory; redact before persistence or keep only an in-memory redacted tail.
  - confidence: 0.9
- **[P2] Make the Codex stderr capture per launch:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: All Codex wrapper invocations use the same `codex-acp-wrapper.stderr.log` under the shared ACPX state directory. Since each wrapper truncates it at startup and appends to it, concurrent sessions can erase or mix another session's failure tail and surface the wrong diagnostic; key the capture by lease/session/request and read that exact buffer.
  - confidence: 0.84
