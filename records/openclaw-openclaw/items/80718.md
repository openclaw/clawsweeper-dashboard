## What This Changes

## Best Possible Solution

- **[P1] Fix the async generator test cast:** `extensions/acpx/src/runtime.test.ts:315`
  - body: `check-test-types` fails on this line with TS2352 because `never[]` is being cast directly to `AsyncIterable<never>`. Use an actual empty async generator or cast through `unknown` so the test suite typechecks again.
  - confidence: 0.99
- **[P2] Scope stderr logs to one wrapper run:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: Every Codex wrapper instance writes and truncates the same `codex-acp-wrapper.stderr.log` under the shared wrapper root. Concurrent sessions can erase or mix each other's diagnostics, so a generic `Internal error` can surface the wrong tail or no tail; pass a per-launch or lease-specific log path and read that matching file.
  - confidence: 0.86
- **[P2] Redact stderr before writing the log:** `extensions/acpx/src/codex-auth-bridge.ts:178-181`
  - body: The runtime redacts before putting stderr into `AcpRuntimeError`, but the generated wrapper first persists raw child stderr to disk. Codex/acpx stderr may include provider tokens or private endpoints, so store a redacted tail or a private ephemeral file instead of retaining raw stderr in plugin state.
  - confidence: 0.84
