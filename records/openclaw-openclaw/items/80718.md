## Summary

Keep open: the PR targets a source-backed ACP diagnostics bug, but the latest patch still persists raw Codex stderr to a shared file and the supplied proof does not exercise the patched `sessions_spawn` failure path.

## What This Changes

The PR captures Codex ACP wrapper stderr into a log, appends a redacted tail to generic ACP session/turn errors, and adds focused acpx/acp-spawn tests.

## Best Possible Solution

Capture bounded, redacted, per-run Codex wrapper diagnostics through the wrapper/runtime boundary, then prove the patched `sessions_spawn(runtime="acp", agentId="codex")` failure path with redacted live output or logs.

## Reproduction Assessment

Yes, at source level: current main inherits wrapper stderr and forwards generic `Internal error` failures from `ensureSession`/`runTurn` without an attached stderr tail. I did not run the live patched gateway path.

## Solution Assessment

No. Capturing diagnostics at the wrapper/runtime boundary is the right direction, but the implementation should not persist raw/shared stderr and should include real proof of the patched `sessions_spawn` path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Redact stderr before writing the wrapper log:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: The generated wrapper appends raw Codex stderr to `codex-acp-wrapper.stderr.log` before any redaction. Codex/acpx diagnostics can include tokens or credential-bearing request details, so a failed run can leave unredacted secrets in plugin state even though the surfaced `AcpRuntimeError` is redacted later.
  - confidence: 0.86
- **[P2] Use a per-run stderr log:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: All Codex wrapper processes use the same `codex-acp-wrapper.stderr.log` under the shared plugin state wrapper root. Concurrent or sequential sessions can clear, interleave, or reuse another run's stderr before the runtime reads the tail, so the diagnostic can be missing, stale, or from the wrong run.
  - confidence: 0.8
