## Summary

Keep open: the patch targets a source-backed acpx/Codex diagnostic gap, but it introduces blocking raw-stderr persistence and lacks real proof of the patched OpenClaw `sessions_spawn` path.

## What This Changes

The PR captures generated Codex ACP wrapper stderr, appends a redacted tail to generic session/turn `AcpRuntimeError` messages, and adds focused acpx/acp-spawn tests.

## Best Possible Solution

Store only redacted, bounded, per-run Codex wrapper diagnostics, then prove the patched OpenClaw `sessions_spawn` Codex ACP failure path with redacted terminal or log output.

## Reproduction Assessment

Yes, at source level: current main inherits wrapper stderr and forwards generic `Internal error` failures from `ensureSession`/`runTurn` without an attached Codex stderr tail. I did not run a live patched gateway reproduction.

## Solution Assessment

No. The wrapper/runtime boundary is the right place to preserve diagnostics, but the implementation should not persist raw/shared stderr and should include real proof of the patched `sessions_spawn` path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Redact stderr before writing the wrapper log:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: The generated wrapper appends raw Codex stderr to `codex-acp-wrapper.stderr.log` before any redaction. Codex/acpx diagnostics can include tokens or credential-bearing request details, so a failed run can leave unredacted secrets in plugin state even though the surfaced `AcpRuntimeError` is redacted later.
  - confidence: 0.86
- **[P2] Use a per-run stderr log:** `extensions/acpx/src/codex-auth-bridge.ts:347`
  - body: All Codex wrapper processes use the same `codex-acp-wrapper.stderr.log` under the plugin state directory. Concurrent spawned Codex sessions can clear or interleave another session's stderr before the runtime reads the tail, so the diagnostic can be missing or belong to a different run.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The diff introduces concrete secret-at-rest and cross-session diagnostic-leak risks in the Codex wrapper stderr capture.

Concerns:

- **[medium] Raw stderr can persist secrets in plugin state:** `extensions/acpx/src/codex-auth-bridge.ts:178`
  - body: The wrapper writes stderr chunks directly to `codex-acp-wrapper.stderr.log`; read-side redaction protects the thrown error but not the file at rest.
  - confidence: 0.86

  - body: Because the filename is constant under a shared wrapper root, concurrent Codex wrapper runs can overwrite or interleave diagnostics from another session, potentially exposing unrelated stderr in the next error.
  - confidence: 0.76
