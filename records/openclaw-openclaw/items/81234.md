## Summary

Keep open: current main covers part of the cron execution isolation and timeout cleanup path, but isolated jobs can still preserve and use a stored channel-shaped `sessionKey` for delivery/notification context, and the OpenAI/Codex timeout portion remains unresolved.

## What This Changes

Reports beta cron isolated agent jobs preserving stale Discord direct session keys and OpenAI/Codex turn-accepted timeouts after 2026.5.12-beta.3.

## Best Possible Solution

Split this into a narrow cron-session repair or doctor warning for isolated jobs with stale channel-shaped keys, plus a separate OpenAI/Codex runtime diagnostics investigation.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does show the remaining stale-key path: isolated jobs can keep a stored `sessionKey`, and delivery/notification resolution can still read it.

## Solution Assessment

Unclear: clearing every stored isolated-job `sessionKey` may break intentional current-chat delivery fallback. The safer path is a scoped maintainer decision that separates execution isolation, fallback delivery, doctor repair, and provider timeout diagnostics.
