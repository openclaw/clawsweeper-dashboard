## Summary

Keep open. Current main still uses unsigned JSON-RPC stdio frames for OpenClaw-managed MCP servers, has no supported MCPS signing configuration or MCP transport-decorator plugin seam, and the linked standalone `razashariff/mcps-openclaw` integration is useful prior art but does not remove the need for maintainer security/API design.

## What This Changes

Add opt-in MCPS message-level signing for OpenClaw-managed MCP stdio transport, including signed envelopes, nonce/timestamp replay protection, and agent identity.

## Best Possible Solution

Keep this open for a maintainer security/API decision: either design a generic opt-in MCP stdio signing/decorator contract with documented config and regression coverage, or explicitly keep MCPS external and expose any missing plugin API needed to wrap OpenClaw-managed frames safely.

## Reproduction Assessment

Yes for the current gap, from source inspection: OpenClaw serializes MCP stdio messages as raw JSON-RPC lines and the tests assert that framing. This is a feature/security-design request rather than a broken existing contract with a failing runtime reproduction.

## Solution Assessment

Unclear. The requested direction and linked external wrapper are plausible, but directly importing a signing dependency and permissive config is not established as the best OpenClaw fix without an approved threat model, key-handling story, and generic transport/decorator seam.
