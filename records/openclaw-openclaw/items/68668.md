## Summary

Keep open. Current main still has the unguarded browser cleanup wrapper path, and this issue is paired with the open closing PR https://github.com/openclaw/openclaw/pull/68669, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests deduping subagent browser-session cleanup wrapper dispatch so parallel embedded completion paths for the same runId invoke cleanup once.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68669, or an equivalent lifecycle-controller guard that dedupes only browser cleanup dispatch, resets on rearm, and includes focused regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: registering a subagent starts both the in-process lifecycle listener and gateway wait path, and both can call completeSubagentRun(..., triggerCleanup: true) for the same runId while current main has no browser cleanup dispatch flag.

## Solution Assessment

Yes, with a narrow caveat. A per-run dispatch marker in the shared lifecycle helper is the maintainable fix, but it should skip only the browser cleanup wrapper and still allow announce cleanup/finalization to proceed.
