## Summary

Keep open. Current main still has the unguarded browser cleanup wrapper path, and this issue is paired with an open closing PR from the same contributor, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Request a per-run guard so embedded subagent completion does not invoke `cleanupBrowserSessionsForLifecycleEnd` twice for the same `runId`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68669, or an equivalent lifecycle-controller guard with rearm reset and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows registration starts both `ensureListener()` and `waitForSubagentCompletion(...)`, both paths call `completeSubagentRun(..., triggerCleanup: true)`, and current main has no browser cleanup dispatch guard.

## Solution Assessment

Yes. A synchronous per-run dispatch marker in the shared lifecycle helper, reset when the run is rearmed, is the narrow maintainable fix because both completion paths converge there.
