## Summary

Keep open. Current main still has the unguarded browser-cleanup wrapper path, and this issue is explicitly paired with the open closing PR https://github.com/openclaw/openclaw/pull/68669, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Issue requests deduping browser-session cleanup wrapper dispatch when embedded subagent completion reaches the shared lifecycle helper from both listener and gateway wait paths for the same run.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68669, or an equivalent lifecycle-controller guard that dedupes only browser cleanup dispatch, resets on rearm, and keeps announce cleanup/finalization intact.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main starts both the in-process lifecycle listener and gateway wait path for a registered subagent, and both can call completeSubagentRun(..., triggerCleanup: true) while the browser cleanup wrapper has no per-run dispatch marker.

## Solution Assessment

Yes, the proposed direction is the right narrow fix if it skips only the browser cleanup wrapper. Returning before the existing announce cleanup/finalization tail would be too broad, which is why the linked PR still needs review or repair before landing.
