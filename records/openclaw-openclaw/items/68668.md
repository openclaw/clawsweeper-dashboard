## Summary

Keep open. Current main still has the unguarded browser cleanup wrapper path, and an open closing PR covers the narrow fix, so this issue should wait for that PR or an equivalent fix to merge or be rejected.

## What This Changes

Guard subagent browser lifecycle cleanup so embedded runs do not invoke `cleanupBrowserSessionsForLifecycleEnd` twice for the same `runId`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68669, or an equivalent lifecycle-controller guard with rearm reset and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: registration starts both the embedded listener and gateway wait, both call `completeSubagentRun(..., triggerCleanup: true)`, and current main has no browser cleanup dispatch guard.

## Solution Assessment

Yes. A synchronous per-run dispatch marker in the shared lifecycle helper, reset when a run is rearmed, is the narrow maintainable fix because both completion paths converge there.
