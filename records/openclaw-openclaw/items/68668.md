## Summary

Keep open. Current main still has the unguarded browser cleanup wrapper path, and this issue is paired with an open closing PR that has not merged yet.

## What This Changes

The issue requests deduping browser-session cleanup dispatch when embedded subagent completion reaches `completeSubagentRun` from both listener and gateway wait paths for the same `runId`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68669, or an equivalent guard that dedupes only the browser cleanup wrapper while preserving normal announce cleanup and rearm behavior.

## Reproduction Assessment

Yes, at source level. Current main starts both completion paths for a registered subagent and both can call `completeSubagentRun(..., triggerCleanup: true)` against an unguarded browser cleanup wrapper.

## Solution Assessment

Yes, the narrow maintainable fix is a synchronous per-run browser cleanup dispatch marker reset on rearm. The guard should suppress only the browser cleanup wrapper, not the shared announce cleanup/finalization path.
