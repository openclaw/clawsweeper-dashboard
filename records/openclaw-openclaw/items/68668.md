## Summary

Keep open. Current main still has the unguarded browser cleanup wrapper path, and this issue is paired with open closing PR #68669, so it should remain open until that PR or an equivalent fix is merged or rejected.

## What This Changes

Request a per-run guard so embedded subagent completion invokes `cleanupBrowserSessionsForLifecycleEnd` only once for the same `runId`.

## Best Possible Solution

Review and land #68669, or an equivalent narrow fix, with a synchronous per-record browser cleanup dispatch guard, rearm reset, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the gateway wait and embedded lifecycle listener can both call `completeSubagentRun(..., triggerCleanup: true)` for the same record, and current main has no per-run guard before the browser cleanup wrapper.

## Solution Assessment

Yes. A synchronous per-record dispatch flag reset on rearm matches the existing `endedHookEmittedAt` dedupe pattern and fixes the duplicated wrapper work at the converged lifecycle boundary.
