## Summary

Keep open. The PR is useful and current main does not already implement its central `session_status` group-activation fix, but the branch introduces two blocking regressions called out in review: `openclaw/main` can be rejected in default single-agent configs, and Gateway session defaults can lose configured/default context-token values.

## What This Changes

The PR shares model/context-token resolution across status and compat paths, accepts `openclaw/<provider>/<model>` compat overrides, derives `session_status` group activation defaults from channel config, and adds focused regression tests.

## Best Possible Solution

Keep the PR open and repair it by preserving current default-agent alias and context-token contracts while keeping the shared resolver and group-activation regression coverage; add the required user-facing changelog entry before merge.

## Reproduction Assessment

Yes. The blocking regressions have clear focused reproduction paths: default config without `agents.list` should still accept the advertised `openclaw/main` model id, and `getSessionDefaults` should still return configured `agents.defaults.contextTokens` or the default fallback when discovery has no value.

## Solution Assessment

No. The direction is maintainable, but this branch is not the best final fix until it passes the existing context-token override/fallback into the shared resolver and allows the resolved default agent alias even when no explicit agent list is configured.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve the default agent alias:** `src/gateway/http-utils.ts:224-230`
  - body: `/v1/models` still advertises `openclaw/<defaultAgentId>` even when `agents.list` is omitted, because the default agent falls back to `main`. This new membership check builds the accepted set only from `cfg.agents.list`, so `openclaw/main` is treated as invalid in the default config and compat/model lookup requests can reject a model id the server just listed. Include `resolveDefaultAgentId(cfg)` in the accepted single-segment targets or otherwise preserve the old default alias path.
  - confidence: 0.93
- **[P1] Pass the context override and fallback:** `src/gateway/session-utils.ts:948-953`
  - body: `getSessionDefaults` used to honor `agents.defaults.contextTokens` and fall back to `DEFAULT_CONTEXT_TOKENS`; the new shared call passes neither value. For users with a configured cap, or with a default model that has no discovery/cache entry, the session-default context window can become `null` and disappear from status/TUI displays. Pass `contextTokensOverride` and `fallbackContextTokens` into the shared resolver and restore the import.
  - confidence: 0.95
