## Summary

Keep open: current main still has the `session_status` group-activation gap this PR targets, but the branch regresses two existing status/compat contracts and the external PR is missing after-fix real behavior proof.

## What This Changes

The PR refactors status/session model resolution, updates compat model override parsing, derives `session_status` group activation defaults from channel config, and adds focused regression tests.

## Best Possible Solution

Land the shared resolver and config-aware group activation only after preserving the default-agent alias and session-default context-token contracts, with contributor-provided real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: default configs should still accept the advertised `openclaw/main` model id, and `getSessionDefaults` should still expose configured or fallback context tokens when discovery has no value.

## Solution Assessment

No. The refactor direction is maintainable, but this patch is not the best final fix until it passes the context-token override/fallback into the shared resolver and keeps the resolved default agent alias valid.

## Review Findings

Overall confidence: 0.92

- **[P1] Preserve the default agent alias:** `src/gateway/http-utils.ts:224-230`
  - body: `/v1/models` still advertises `openclaw/<defaultAgentId>`, and default configs resolve that id to `main` even without `agents.list`. This new guard accepts single-segment `openclaw/<target>` values only when they appear in `cfg.agents.list`, so `openclaw/main` can be rejected even though the server lists it as usable.
  - confidence: 0.93

  - confidence: 0.95
