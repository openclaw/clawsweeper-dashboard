## Summary

Keep open: current main still has the reported `session_status` group-activation fallback gap, so the PR has useful unique work, but the branch is not mergeable as-is because it introduces two blocking contract regressions and still lacks after-fix real behavior proof.

## What This Changes

The PR refactors status and OpenAI-compatible model resolution, derives group activation defaults from channel config for session status, and adds focused regression tests.

## Best Possible Solution

Keep the config-aware group activation direction, but update the branch to preserve the advertised default-agent alias and `getSessionDefaults` context-token contracts, then rebase and add redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main hard-codes `session_status` group fallback activation to `mention`, while this PR can reject advertised `openclaw/main` aliases and drop configured or default context-token values.

## Solution Assessment

No, not as-is. The config-aware fallback is the right narrow direction, but the PR must preserve the existing compat alias and session-default context-token contracts before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve advertised default-agent aliases:** `src/gateway/http-utils.ts:229-230`
  - body: `/v1/models` advertises `openclaw/<defaultAgentId>`, including default installs where the id is `main` and `agents.list` is absent. This guard rejects that same model id unless it appears in `cfg.agents.list`, so compat requests using the advertised default alias can fail as invalid.
  - confidence: 0.93
- **[P1] Pass the context-token override and fallback:** `src/gateway/session-utils.ts:948-953`
  - body: `getSessionDefaults` currently honors `agents.defaults.contextTokens` and falls back to `DEFAULT_CONTEXT_TOKENS`. This resolver call passes neither value, so configured caps or undiscovered models can return `null` and remove the status/TUI context-window display.
  - confidence: 0.95
