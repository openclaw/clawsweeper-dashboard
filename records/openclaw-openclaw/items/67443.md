## Summary

Keep open. The PR addresses a real current-main status gap, but the proposed patch still has two source-backed P1 regressions in default-agent alias validation and session-default context-token reporting, so it should be repaired before merge.

## What This Changes

The PR shares status/session model and context-token resolution, accepts `openclaw/<provider>/<model>` compat overrides, derives `session_status` group activation defaults from channel config, and adds focused regression tests.

## Best Possible Solution

Repair the PR by preserving the existing default-agent alias and context-token contracts while keeping the shared resolver and config-aware group activation coverage, then add the required changelog entry and run the targeted status/gateway tests before merge.

## Reproduction Assessment

Yes. The regressions have clear source-level reproduction paths: default config without `agents.list` should still accept the advertised `openclaw/main` alias, and `getSessionDefaults` should still return configured or default context tokens when discovery has no value.

## Solution Assessment

No. The overall refactor direction is maintainable, but the branch is not the best final fix until it passes the context-token override/fallback into the shared resolver and includes the resolved default agent alias in the accepted single-segment targets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve the default agent alias:** `src/gateway/http-utils.ts:224-230`
  - body: `/v1/models` still advertises `openclaw/<defaultAgentId>` even when `agents.list` is omitted, because the default agent falls back to `main`. This new membership check builds the accepted set only from `cfg.agents.list`, so `openclaw/main` is treated as invalid in the default config and compat/model lookup requests can reject a model id the server just listed. Include `resolveDefaultAgentId(cfg)` in the accepted single-segment targets or otherwise preserve the old default alias path.
  - confidence: 0.93
- **[P1] Pass the context override and fallback:** `src/gateway/session-utils.ts:948-953`
  - body: `getSessionDefaults` used to honor `agents.defaults.contextTokens` and fall back to `DEFAULT_CONTEXT_TOKENS`; the new shared call passes neither value. For users with a configured cap, or with a default model that has no discovery/cache entry, the session-default context window can become `null` and disappear from status/TUI displays. Pass `contextTokensOverride` and `fallbackContextTokens` into the shared resolver and restore the import.
  - confidence: 0.95
