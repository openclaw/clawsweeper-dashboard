## Summary

Keep open. The PR fixes a real status/session gap, but the current branch introduces two narrow regressions in compat model alias validation and session default context-token reporting, and it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR refactors status/session model and context-token resolution, accepts `openclaw/<provider>/<model>` compat overrides, derives group activation defaults from channel config, and adds focused regression tests.

## Best Possible Solution

Repair the branch by preserving the existing alias and context-token contracts while keeping the shared resolver and config-aware group activation coverage, then add the changelog entry and run the focused gateway/status tests.

## Reproduction Assessment

Yes. The regressions have source-level reproduction paths: default config without `agents.list` should still accept the advertised `openclaw/main` alias, and `getSessionDefaults` should still return configured or default context tokens when discovery has no value.

## Solution Assessment

No. The refactor direction is maintainable, but this is not the best final patch until it passes the context override/fallback into the shared resolver and includes the resolved default agent alias in accepted single-segment targets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve the default agent alias:** `src/gateway/http-utils.ts:224-230`
  - body: `/v1/models` still advertises `openclaw/<defaultAgentId>` when `agents.list` is omitted, because the default agent falls back to `main`. This new membership check builds the accepted set only from `cfg.agents.list`, so `openclaw/main` becomes invalid in the default config and compat/model lookup requests can reject a model id the server just listed. Include `resolveDefaultAgentId(cfg)` in the accepted single-segment targets or otherwise preserve the old default alias path.
  - confidence: 0.93
- **[P1] Pass the context override and fallback:** `src/gateway/session-utils.ts:948-953`
  - body: `getSessionDefaults` used to honor `agents.defaults.contextTokens` and fall back to `DEFAULT_CONTEXT_TOKENS`; the new shared call passes neither value. For users with a configured cap, or with a default model that has no discovery/cache entry, the session-default context window can become `null` and disappear from status/TUI displays. Pass `contextTokensOverride` and `fallbackContextTokens` into the shared resolver and restore the import.
  - confidence: 0.95
