## Summary

## What This Changes

The PR refactors status/compat model and context-token resolution, accepts `openclaw/<provider>/<model>` compat overrides, derives `session_status` group activation from channel config, and adds regression tests.

## Best Possible Solution

Repair the PR by preserving the default-agent alias and context-token contracts while keeping the config-aware group activation fix, then add a single-line changelog entry and run the targeted status/gateway checks.

## Reproduction Assessment

Yes. The regressions have source-level reproduction paths: default config without `agents.list` should still accept `openclaw/main`, and `getSessionDefaults` should still return configured or default context tokens when discovery has no value.

## Solution Assessment

No. The refactor direction is maintainable, but the branch is not the best final fix until it passes the context-token override/fallback into the shared resolver and includes the implicit default agent in accepted single-segment targets.
