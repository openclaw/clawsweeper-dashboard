- **[P1] Preserve advertised default-agent aliases:** `src/gateway/http-utils.ts:229-230`
  - body: `/v1/models` advertises `openclaw/<defaultAgentId>`, including default installs where the ID is `main` and `agents.list` is absent. This guard rejects that same model ID unless it appears in `cfg.agents.list`, so compat requests using the advertised default alias can fail as invalid.
  - confidence: 0.93
- **[P1] Pass the context-token override and fallback:** `src/gateway/session-utils.ts:948-953`
  - body: `getSessionDefaults` currently honors `agents.defaults.contextTokens` and falls back to `DEFAULT_CONTEXT_TOKENS`. This resolver call passes neither value, so configured caps or undiscovered models can return `null` and remove the status/TUI context-window display.
  - confidence: 0.95
- **[P1] Keep backend model overrides on the documented header:** `src/gateway/http-utils.ts:260-263`
  - body: The public OpenAI-compatible contract says the `model` field is an agent target and backend provider/model overrides belong in `x-openclaw-model`. Accepting `model: "openclaw/<provider>/<model>"` adds a second override source without proving the existing path is insufficient, which creates routing drift and user confusion.
  - confidence: 0.86
