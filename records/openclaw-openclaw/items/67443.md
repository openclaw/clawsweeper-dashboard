This PR contains a useful status fallback fix, but it should stay open because the current branch introduces two blocking regressions and still lacks required real behavior proof from the external contributor.

## What This Changes

The PR refactors status and OpenAI-compatible model resolution, derives `session_status` group activation defaults from channel config, adds a Codex context-token fallback, and extends focused tests.

## Best Possible Solution

Keep the config-aware group activation fix direction, but update the branch to preserve the advertised compat alias and `getSessionDefaults` context-token contracts before rebasing and adding redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main hard-codes the `session_status` group fallback to `mention`, while the PR diff can reject advertised `openclaw/main` aliases and drop configured/default context-token values.

## Solution Assessment

No, not as-is. The group activation change is a narrow maintainable fix, but the branch must preserve existing compat alias and session-default context-token behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve advertised default-agent aliases:** `src/gateway/http-utils.ts:229-230`
  - body: `/v1/models` advertises `openclaw/<defaultAgentId>`, including default installs where the ID is `main` and `agents.list` is absent. This guard rejects that same model ID unless it appears in `cfg.agents.list`, so compat requests using the advertised default alias can fail as invalid.
  - confidence: 0.93
- **[P1] Pass the context-token override and fallback:** `src/gateway/session-utils.ts:948-953`
  - body: `getSessionDefaults` currently honors `agents.defaults.contextTokens` and falls back to `DEFAULT_CONTEXT_TOKENS`. This resolver call passes neither value, so configured caps or undiscovered models can return `null` and remove the status/TUI context-window display.
  - confidence: 0.95
