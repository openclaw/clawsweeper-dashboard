## Summary

## Best Possible Solution

Preserve shipped session discovery while fixing cross-project adoption, either by keeping legacy lookup with cwd-aware resume/list filtering or by adding a tested migration/legacy fallback before adopting the new encoding.

No. Percent encoding is a plausible direction, but the mergeable fix must preserve or migrate shipped session directories and should validate header cwd types before resolving them.

  - body: Changing this encoder moves ordinary existing sessions from shipped names like `--home-alice-dev--` to new names like `--home%2Falice%2Fdev--`.   - confidence: 0.86
