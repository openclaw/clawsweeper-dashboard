## Summary

Keep open. Current main and the latest release still have the Feishu merge_forward parser gaps, and the linked fix PRs are open implementation candidates rather than merged fixes.

## What This Changes

Parse Feishu merge_forward sub-message content so forwarded cards, tables, and interactive messages reach the agent instead of collapsing to `[interactive]`.

## Best Possible Solution

Land or replace one narrow Feishu plugin fix that requests full card content for merge_forward fetches, includes non-root forwarded sub-messages, reuses the shared interactive-card parser, and adds regression coverage.

Yes, source-level: current main fetches merge_forward details, filters sub-messages by `upper_message_id`, and formats `interactive` through the default `[interactive]` branch. I did not run a live Feishu tenant reproduction in this read-only review.
