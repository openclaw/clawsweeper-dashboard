## Summary

Keep open: current main and latest v2026.6.8 still archive only primary session transcripts on reset, while two linked closing PRs are open and unmerged, so this issue remains the canonical tracker until one lands.

## What This Changes

The issue asks for session reset to preserve the matching trajectory runtime and pointer files instead of allowing them to be deleted after reset.

## Best Possible Solution

Land one canonical fix that archives the trajectory runtime and pointer through the shared reset artifact path, respects pointer/override ownership checks, and covers reset/delete retention with focused tests.

## Reproduction Assessment

Yes, the requested direction is the right fix, but it should be implemented in the shared reset artifact lifecycle rather than with outbound-specific detectors or a new config knob.
