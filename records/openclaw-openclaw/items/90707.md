## Summary

Keep open: current main and the latest release still archive only the primary transcript on reset, while trajectory runtime and pointer sidecars are not included. The only related implementation PR was closed unmerged after review found a missed default-path case and unrelated proof.

## What This Changes

The issue asks OpenClaw to preserve a session's `trajectory.jsonl` and pointer sidecar when a session reset archives the transcript.

## Best Possible Solution

Archive the trajectory runtime file and pointer alongside reset transcript archives using the same store-derived transcript candidate semantics, with tests for default and explicit session-file paths.

## Reproduction Assessment

Yes, by source inspection: create a session with a transcript plus `*.trajectory.jsonl`/`*.trajectory-path.json`, then follow the reset archive path. Current main archives the transcript candidates only, so the trajectory sidecars remain outside the archived reset artifact set.

## Solution Assessment

Yes for the narrow direction: preserving trajectory artifacts with the transcript archive is the maintainable fix. The optional outbound-only heuristic is a policy/config expansion and should not block the core reset artifact repair.
