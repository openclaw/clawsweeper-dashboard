## Summary

Keep open: current main and the latest release still archive only primary session transcripts on reset, while trajectory runtime and pointer sidecars remain outside the reset archive path; the related repair PR was closed unmerged after review found a real default-path gap.

## What This Changes

The issue requests preserving the matching trajectory JSONL, and the dangling trajectory pointer, when a session reset rotates the primary transcript.

## Best Possible Solution

Extend the existing session archive lifecycle to preserve trajectory runtime and pointer artifacts with the same path semantics as transcript archives, then leave retention and disk-budget cleanup to the existing maintenance paths.

## Reproduction Assessment

Yes, at source level: reset calls archiveSessionTranscriptsForSessionDetailed, and that helper currently archives only primary transcript candidates while trajectory runtime and pointer files live under separate path helpers. I did not run the filesystem repro because this review is read-only.

## Solution Assessment

Yes, the best fix is a narrow lifecycle/archive repair rather than a new outbound-specific heuristic or config option. The replacement should derive trajectory candidates from the same transcript candidates/store path semantics and cover pointer consistency.
