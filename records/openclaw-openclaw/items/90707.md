## Summary

Keep open: current main and the latest release still archive only the primary session transcript on reset, while the trajectory sidecar is not preserved; the only linked fix PR is closed unmerged and had a known default-path defect.

## What This Changes

The issue asks OpenClaw to preserve runtime trajectory sidecars when a session reset archives the primary transcript so outbound sends remain forensically reviewable.

## Best Possible Solution

Archive the reset session’s trajectory runtime artifact and pointer metadata as part of the same session artifact rotation, using the same resolved transcript path semantics as primary transcript archives.

## Reproduction Assessment

Yes, source-level: the reset path calls the transcript archive helper, and that helper currently archives only resolved transcript candidates before returning. I did not run a live reset because this review was read-only.

## Solution Assessment

Yes, the requested direction is the right core fix, but it should preserve trajectory artifacts through the existing archive path semantics rather than adding an outbound-specific detector or new config knob.

Current reset archives the primary session transcript through archiveSessionTranscriptsForSessionDetailed but does not preserve the matching trajectory sidecar, so outbound-forensics data can be lost or left unreachable after reset. Implement a narrow core session-artifact fix: preserve the reset session's trajectory runtime artifact, and handle the trajectory pointer so it does not become a dangling forensic reference. Use the same resolved transcript candidate semantics as primary archive handling, including explicit sessionFile and storePath-only default transcript cases; do not repeat the incomplete sessionFile-only lookup from https://github.com/openclaw/openclaw/pull/90934. Do not add a config option, env var, outbound-tool detector, or CHANGELOG entry. Add focused regression coverage for reset with a present trajectory, absent trajectory, and the default store-derived session path; include pointer or OPENCLAW_TRAJECTORY_DIR coverage if the implementation changes that discovery path. Keep the delete/prune trajectory cleanup behavior aligned with existing cleanup helpers and avoid broad session refactors.
