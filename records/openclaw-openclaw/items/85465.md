## Summary

Keep this PR open: it is a focused Discord/channel progress bug fix with useful final-delivery proof, but the command-output status-mode path is still incomplete because Discord does not pass channel config into the formatter for command-output events.

## What This Changes

The PR suppresses command-output titles in status-mode formatting, changes Discord progress-mode finals to clear the progress draft and send the final reply normally, suppresses late error finals after visible finals, and updates formatter and Discord handler tests.

## Best Possible Solution

Route Discord command-output progress through the config-aware formatter, preserve raw/default output, keep the separate final-delivery behavior, add a Discord status-mode command-output regression, and require redacted real Discord proof after the repair.

## Reproduction Assessment

Yes. Configure Discord progress with commandText status and emit an onCommandOutput end event with a title; on the PR head that callback still calls the raw formatter, so source inspection shows the title can remain visible.

## Solution Assessment

No. The formatter change and final-delivery direction are useful, but the maintainable fix must thread resolved channel config into the real Discord command-output callback and cover that path with a focused regression.
