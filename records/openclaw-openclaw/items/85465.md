## Summary

Keep open: the fix direction is right, but the branch only covers direct formatter calls. Current channel command-output callbacks still call the formatter without channel config, so status-mode channels can keep showing command-output titles.

## What This Changes

The PR changes shared channel progress formatting so command-output events omit their title when `commandText` is `status`, with a direct formatter regression assertion.

## Best Possible Solution

Preserve raw/default output, but route command-output progress through config-aware formatter helpers in the compact progress channel call sites and add coverage for at least one configured channel path.

## Reproduction Assessment

Yes, source-level: current main appends `input.title` for command-output progress, and channel callbacks for command-output do not pass config while tool/item callbacks do. I did not run a local repro because this review was read-only.

## Solution Assessment

No; the shared conditional is necessary but not sufficient for the documented channel behavior. The maintainable fix is to thread channel config into command-output formatting at the real channel call sites and test that path.
