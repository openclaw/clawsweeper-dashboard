## Summary

Keep this PR open: the formatter and Discord status-progress fixes are useful and the supplied live proof covers the main happy path, but the current patch introduces a message-delivery regression by deleting the progress draft before the normal final reply send has succeeded.

## What This Changes

This PR changes the shared channel progress formatter and Discord message handler so status-mode command-output titles are hidden, progress drafts are not used as final replies, pending progress drafts are canceled, and late Discord error finals are suppressed.

## Best Possible Solution

Keep the status formatter and late-error suppression work, but change progress-mode final cleanup so pending draft work is canceled before the final send and an existing visible draft is deleted only after the final reply is successfully delivered.

## Reproduction Assessment

Yes. Source inspection shows current main appends command-output titles in the shared formatter and Discord uses the raw formatter for `onCommandOutput`; the PR body also provides targeted test output and redacted live Discord proof for the intended happy path.

## Solution Assessment

No, not quite. The formatter fix is narrow, but the progress final cleanup should preserve an existing visible draft until the normal final reply send has succeeded.
