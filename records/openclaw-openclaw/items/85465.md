## Summary

Keep this PR open: the direction matches the documented status-mode contract, but the formatter-only change does not reach the real command-output channel paths because those callbacks still call the formatter without channel config.

## What This Changes

The PR changes the shared channel progress formatter to omit command-output titles when `commandText` is `status` and adds a formatter-level regression assertion.

## Best Possible Solution

Route command-output progress through the same config-aware formatter path as tool and item progress, preserve raw/default output, and add focused channel coverage for status mode.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a `command-output` event with a title still appends that title unless resolved `commandText: "status"` options reach the formatter. I did not run tests because this review is read-only.

## Solution Assessment

No, the formatter condition is necessary but incomplete for the stated channel behavior until command-output channel callbacks pass the same resolved config used by tool and item progress.
