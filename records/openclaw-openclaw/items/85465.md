## Summary

Keep open: the formatter change has the right intent, but configured channel command-output callbacks still bypass the config-aware helper, so real status-mode progress can continue appending command-output titles.

## What This Changes

The PR updates shared command-output progress formatting and changes Discord progress-mode final replies to clear the draft before normal final delivery.

## Best Possible Solution

Route command-output progress through the config-aware `ForEntry` helper in every compact progress channel, keep raw/default behavior unchanged, and add focused channel-level proof for status mode.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured channel with `commandText: "status"` reaches config-aware tool/item formatting, but command-output callbacks still call the raw formatter without that config.

## Solution Assessment

No. The direct formatter guard is necessary but incomplete; the maintainable fix is to pass resolved channel streaming options at the command-output call sites and cover the channel path.
