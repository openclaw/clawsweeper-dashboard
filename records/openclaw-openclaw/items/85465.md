The PR updates shared channel progress formatting and Discord progress final delivery so status-mode command-output lines omit titles and Discord progress drafts are cleared before sending the final reply normally.

## Best Possible Solution

Route command-output progress through the config-aware formatter path in every compact-progress channel, preserve raw/default output, add focused status-mode channel coverage, and require redacted real transport proof before merge.
