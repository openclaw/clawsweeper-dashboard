## What This Changes

The PR updates channel progress formatting, Discord command-output progress rendering, Discord progress-final delivery, late error-final suppression, and related SDK/Discord regression tests.

## Best Possible Solution

Keep the config-aware formatting, but make progress-final cleanup cancel pending and in-flight drafts before final delivery, retain late error suppression, and require redacted Discord status-mode proof.

## Reproduction Assessment

Yes. Source inspection shows current main appends command-output titles in status mode and Discord bypasses the config-aware formatter; the PR's own tests also encode the intended regressions.

## Solution Assessment

No. The formatter and Discord config-threading direction is right, but the progress-final cleanup should cancel pending or in-flight drafts even before a Discord message id exists.

## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P1] Clear progress drafts even before an id exists:** `extensions/discord/src/monitor/message-handler.process.ts:709-710`

Clear or discard the progress draft for final replies regardless of current id, and cover the pending/in-flight race.
