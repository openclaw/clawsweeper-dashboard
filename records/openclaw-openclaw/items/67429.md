## Best Possible Solution

Rebase onto current main and implement patch-only null clearing for `fallbacks` and `thinking` through the current gateway protocol package while preserving create-time null pruning and adding real gateway proof.

## Reproduction Assessment

Yes, from source inspection: current main drops `fallbacks: null` and `thinking: null` during patch normalization and the service never deletes those fields on null, so `cron.update` cannot clear them. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No, not as submitted. The narrow fix should preserve null only for update patches, not in the shared create/update normalizer, and should be rebased onto the current `packages/gateway-protocol` path.
