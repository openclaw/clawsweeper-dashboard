## Best Possible Solution

Forward the new validation options through the raw plugin-validation wrapper, add a discriminating regression that fails when those options are dropped, and keep the no-op dry-run and unrelated non-channel dry-run behavior.

## Reproduction Assessment

## Solution Assessment

No, not yet. The CLI layer is the right place to compute touched paths, but the validator wrapper must forward those options and the tests need one case that fails without that pass-through.
