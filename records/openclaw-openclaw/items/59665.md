## Summary

Keep open: current main and v2026.5.18 still intentionally generate a canonical macOS LaunchAgent PATH rather than copying the login shell, so the remaining request is a Gateway service PATH product/security decision, not an already-implemented bug fix.

## What This Changes

The issue requests that macOS Gateway LaunchAgent PATH generation reflect the current login-shell PATH instead of a curated generated PATH.

## Best Possible Solution

Define the macOS Gateway service PATH contract first, then align install, restart, doctor/audit diagnostics, docs, and tests around either the current canonical PATH policy or a deliberately sanitized login-shell capture design.

## Reproduction Assessment

Yes for source-level behavior: current main and v2026.5.18 deterministically build the macOS LaunchAgent PATH through buildMinimalServicePath rather than the login shell. No live launchd reinstall was run in this read-only review.

## Solution Assessment

Unclear. Mirroring login-shell PATH would satisfy the requested behavior, but it conflicts with the documented canonical service PATH and needs an explicit maintainer security/product decision before implementation.
