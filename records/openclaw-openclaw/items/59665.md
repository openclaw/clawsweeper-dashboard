## Summary

Keep open. Current main still builds the macOS Gateway service PATH from a curated service-environment policy, not from the current login-shell PATH, so the remaining work is a maintainer decision about the desired launchd PATH contract.

## What This Changes

The issue asks Gateway reinstall on macOS to generate the LaunchAgent PATH from the current login shell rather than a stale or curated service PATH.

## Best Possible Solution

Define the macOS service PATH contract, then implement either sanitized login-shell PATH capture or an improved documented curated PATH with launchd install, doctor/audit, and reinstall coverage.

## Reproduction Assessment

Yes for the code-level mismatch: current main deterministically builds launchd service PATH through buildMinimalServicePath rather than /bin/zsh -l -c 'echo $PATH'. A live macOS reinstall was not run in this read-only review.

## Solution Assessment

Unclear. The requested full login-shell mirroring would address the reporter's expectation, but a safer maintainable fix may be a sanitized import or an explicitly documented curated PATH policy.
