## Summary

Keep open. Current main still intentionally generates a curated macOS LaunchAgent service PATH, and the merged Homebrew-path work leaves the broader login-shell PATH contract unresolved.

## What This Changes

Request that `openclaw gateway install` generate a macOS LaunchAgent PATH matching the current login-shell PATH instead of a stale or curated service PATH.

## Best Possible Solution

Define the macOS Gateway service PATH contract, then implement either sanitized login-shell PATH capture or a documented curated PATH policy with launchd install, doctor/audit, and reinstall coverage.

## Reproduction Assessment

Yes for the code-level mismatch: current main deterministically builds the LaunchAgent PATH through `buildMinimalServicePath` rather than the login shell. A live macOS reinstall was not run in this read-only review.

## Solution Assessment

Unclear. Full login-shell mirroring would satisfy the reporter's expectation, but the existing implementation and docs intentionally favor a minimal service PATH, so maintainers need to choose the durable service PATH policy first.
