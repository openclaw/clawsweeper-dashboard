## Summary

Keep open: current main and v2026.5.12 intentionally generate a canonical macOS LaunchAgent PATH rather than copying the login shell, so the remaining request is a product/security policy decision rather than an already-implemented bug fix.

## What This Changes

The issue requests that macOS Gateway LaunchAgent installs capture the current login-shell PATH instead of writing a curated service PATH.

## Best Possible Solution

Define the macOS Gateway service PATH contract, then align install, restart, doctor/audit diagnostics, docs, and tests around either the current canonical PATH policy or a sanitized login-shell capture design.

## Reproduction Assessment

Yes for the source-level behavior: current main and v2026.5.12 deterministically build the macOS LaunchAgent PATH through buildMinimalServicePath rather than the login shell. No live launchd reinstall was run in this read-only review.

## Solution Assessment

Unclear. Mirroring login-shell PATH would satisfy the reporter's requested behavior, but it conflicts with the documented canonical service PATH and needs an explicit maintainer security/product decision first.
