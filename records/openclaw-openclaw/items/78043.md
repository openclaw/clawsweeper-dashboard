## Summary

Keep open: current `main` still appears to inline the node daemon gateway token into the Linux systemd unit, and the linked closing PR #78044 is open but not merged.

## What This Changes

The issue requests preventing Linux `openclaw node install` from embedding `OPENCLAW_GATEWAY_TOKEN` in the generated user systemd unit.

## Best Possible Solution

Land a reviewed fix that moves the node daemon gateway token into an owner-only service environment file and removes stale inline token copies from units/backups.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the node install environment includes `OPENCLAW_GATEWAY_TOKEN`, the node plan does not pass source metadata, and the Linux systemd renderer emits remaining values as inline `Environment=` lines.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing owner-only service env-file/source-metadata pattern for the node daemon instead of adding a new auth mode or config knob.
