## Summary

Keep open: current main still routes the Linux node daemon gateway token into inline systemd environment rendering, and the linked closing PR remains open and conflicting.

## What This Changes

The issue asks OpenClaw to stop Linux `openclaw node install` from embedding `OPENCLAW_GATEWAY_TOKEN` in user systemd unit files.

## Best Possible Solution

Land the linked PR or an equivalent narrow fix that writes the node gateway token only to the owner-only service env file and sanitizes stale inline unit and backup copies.

## Reproduction Assessment

Yes for a source-level path: current main copies `OPENCLAW_GATEWAY_TOKEN` into the node service environment, does not carry source metadata through the node install plan, and renders remaining systemd env inline. I did not run a live Linux user-systemd install in this read-only pass.

## Solution Assessment

Yes: reusing existing `environmentValueSources` and the 0600 systemd service env-file pattern is the narrow maintainable fix. The linked PR points in that direction, but should not close this issue until merged with real Linux behavior proof.
