## Summary

Keep open: current main still carries the reported Linux node systemd token exposure, and the paired closing PR is open but not merged.

## What This Changes

The issue reports that `openclaw node install` on Linux persists the gateway bearer token as an inline `Environment=` entry in the generated user systemd unit.

## Best Possible Solution

Land a reviewed fix that moves the node daemon gateway token into an owner-only service environment file and removes stale inline token copies from units and backups.

## Reproduction Assessment

Yes. Source inspection on current main shows the node install environment includes `OPENCLAW_GATEWAY_TOKEN`, the node install plan carries no value-source metadata, and the Linux systemd renderer emits remaining environment values as inline `Environment=` lines.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing owner-only service env-file/source-metadata path for the node daemon instead of adding a new auth mode or config knob.
