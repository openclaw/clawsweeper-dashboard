## Summary

Keep open: current main and the latest tagged source do not resolve the Android manual LAN setup problem. The Android UI/docs still say private-LAN `ws://` is supported, while current Android resolver/runtime tests enforce loopback-only cleartext and the manual composer still treats the host field as host text rather than a full WebSocket URL.

## What This Changes

The issue requests Android manual setup to handle full WebSocket LAN endpoints without converting a `ws://` scheme into host `ws` or an HTTP endpoint.

## Best Possible Solution

Make Android setup consistent end to end: either accept documented private-LAN `ws://` URLs through parsing, preview, TLS resolution, and tests, or update UI/docs to reject them clearly without ever resolving a pasted scheme as a hostname.

## Reproduction Assessment

Yes for source-level reproduction: current Android source composes manual host text into HTTP/HTTPS URLs, rejects private-LAN cleartext in resolver tests, and still shows UI/docs claiming private-LAN `ws://` support. I did not run a live Android device repro in this read-only review.

## Solution Assessment

Unclear. The scheme-preservation parser fix is narrow, but the private-LAN cleartext behavior conflicts with current loopback-only tests and docs, so maintainers should choose the intended policy before landing a repair.
