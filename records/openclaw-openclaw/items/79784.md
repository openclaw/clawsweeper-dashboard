## Summary

Kept open: current main still uses Telegram message-edit preview streaming with no transport selector, and reintroducing `sendMessageDraft` would be a Telegram plugin feature/config decision after prior duplicate-finalization problems.

## What This Changes

Request to add a Telegram streaming transport option that uses Bot API `sendMessageDraft` for private-chat typewriter previews while falling back to `editMessageText` elsewhere.

## Best Possible Solution

Have Telegram channel owners decide whether native draft streaming is supportable again; if accepted, implement it inside the Telegram plugin with explicit transport config, docs, regression tests for DM/group fallback/finalization, and live DM proof.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection verifies the current-main baseline: Telegram preview streaming uses `sendMessage` plus `editMessageText` and has no `streaming.transport` selector.

## Solution Assessment

Unclear: `sendMessageDraft` is a plausible user experience improvement, but current main deliberately removed native drafts in https://github.com/openclaw/openclaw/pull/75073 after duplicate/finalization reports, so the maintainable path needs Telegram owner review and live proof.
