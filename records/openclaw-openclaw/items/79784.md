## Summary

Kept open: current main does not implement a native Telegram `sendMessageDraft` streaming transport, and the request is a new Telegram plugin config/product choice that reopens a transport path recently removed for duplicate-message risk.

## What This Changes

Request to add an opt-in Telegram `sendMessageDraft` streaming transport/config for native typewriter-style private-chat previews while retaining message-edit fallback.

## Best Possible Solution

Have Telegram channel owners decide whether to reintroduce native drafts as an explicit opt-in transport, then implement it in the Telegram plugin with docs and regression coverage for DM-only fallback, duplicate finalization, draft expiry, and throttling.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken documented behavior. Source inspection does confirm the baseline: current Telegram preview streaming uses `sendMessage` plus `editMessageText` and has no active `sendMessageDraft` transport.

## Solution Assessment

Unclear: Telegram and grammY support the API, but prior OpenClaw history removed this transport because finalization could duplicate messages. The maintainable solution needs owner judgment on whether an opt-in config is worth the UX and support tradeoff.
