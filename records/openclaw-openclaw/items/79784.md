## Summary

Kept open: current main does not implement the requested Telegram `sendMessageDraft` transport or config selector, and reintroducing it would reverse a recent intentional transport-removal direction that needs Telegram owner/product review.

## What This Changes

Add a Telegram streaming transport/config option that uses Bot API `sendMessageDraft` for native typewriter previews in private chats with edit-message fallback.

## Best Possible Solution

Telegram channel owners should decide whether native drafts should return as an opt-in plugin-owned transport with fallback and finalization coverage, or whether the durable edit-preview direction should remain canonical.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection does confirm the baseline: Telegram preview streaming currently uses `sendMessage` plus `editMessageText` and has no `sendMessageDraft` transport/config path.

## Solution Assessment

Unclear: the proposed transport selector is plausible, but the best maintainable path needs owner review because it would reintroduce native draft state after https://github.com/openclaw/openclaw/pull/75073 removed it to avoid duplicate-looking draft transitions.
