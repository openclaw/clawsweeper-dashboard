## Summary

Kept open: current main does not implement native Telegram `sendMessageDraft` preview streaming, and the request is a new Telegram plugin feature/config decision after the prior native-draft path was deliberately removed for duplicate-final UX problems.

## What This Changes

The issue requests a native Telegram `sendMessageDraft` streaming transport, with fallback to the existing `sendMessage` plus `editMessageText` preview path.

## Best Possible Solution

Have Telegram channel owners decide whether to reintroduce a plugin-owned native draft transport, then implement it with explicit fallback, finalization, expiry, and duplicate-prevention tests plus Telegram docs if accepted.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken documented behavior. Source, tests, and docs do verify the current-main baseline: Telegram preview streaming is `sendMessage` plus `editMessageText`, not `sendMessageDraft`.

## Solution Assessment

Unclear: `sendMessageDraft` support is plausible and grammY exposes the API, but the maintainable shape needs owner review because current main intentionally removed native draft preview transport after duplicate-final/flicker issues.
