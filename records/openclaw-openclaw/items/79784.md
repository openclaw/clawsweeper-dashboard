## Summary

Keep open for maintainer product review: current main intentionally uses Telegram message-edit streaming after the native draft transport was removed, and restoring it would reintroduce a transport/config policy that maintainers previously closed as not planned due duplicate-message reports.

## What This Changes

Request to add a Telegram `sendMessageDraft` native streaming transport option for smoother private-chat typewriter-style previews while falling back to `editMessageText` elsewhere.

## Best Possible Solution

Maintainers should decide whether Telegram native draft streaming is now safe enough to reintroduce, likely as a guarded Telegram-plugin opt-in with live DM proof, finalization/duplicate safeguards, fallback behavior, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs verification do show the current main behavior: Telegram streaming uses `sendMessage` plus `editMessageText`, and the requested `streaming.transport` config key is absent.

## Solution Assessment

Unclear. Native drafts may be the right UX direction, but reintroducing them is not a narrow automated fix because merged https://github.com/openclaw/openclaw/pull/75073 intentionally removed that transport after duplicate-message reports.
