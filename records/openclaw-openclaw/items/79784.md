## Summary

Keep open: current main intentionally uses Telegram `sendMessage` plus `editMessageText` for preview streaming, while the requested `sendMessageDraft` transport is a new Telegram-plugin feature/config choice with known duplicate-finalization history and fresh maintainer/contributor discussion.

## What This Changes

Request a Telegram streaming transport option that can use native `sendMessageDraft` for private-chat typewriter-style previews while retaining message-edit fallback behavior.

## Best Possible Solution

Have Telegram channel owners decide whether native drafts should return as an opt-in plugin-owned transport, then require focused runtime/tests/docs coverage for fallback and finalization semantics before landing.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, not broken documented behavior. Source inspection does verify the baseline that current main streams Telegram previews through `sendMessage` plus `editMessageText`.

## Solution Assessment

Unclear: native `sendMessageDraft` is a plausible transport, but adding it as a config option is not obviously the best fix until maintainers resolve the known duplicate-finalization tradeoff and live Telegram behavior after the May 8, 2026 Bot API update.
