## Summary

Kept open: current main does not implement the requested native Telegram `sendMessageDraft` transport, and the request would add a user-facing Telegram plugin transport/config path that needs maintainer review.

## What This Changes

Add a Telegram streaming transport selector that can use native `sendMessageDraft` for private-chat typewriter previews while falling back to `editMessageText`.

## Best Possible Solution

If maintainers want native drafts again, implement it inside the Telegram plugin behind an explicit transport config with fallback, finalization, expiry, and docs coverage; otherwise keep the current message-edit-only direction documented.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection confirms the current-main baseline: Telegram preview streaming uses `sendMessage` plus `editMessageText` and has no native `sendMessageDraft` transport.

## Solution Assessment

Unclear: the requested transport is technically plausible, but the best maintainable path needs Telegram owner review because current main deliberately removed native draft previews and the proposal adds a new user-facing config/default decision.
