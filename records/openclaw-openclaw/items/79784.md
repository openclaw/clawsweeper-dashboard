## Summary

Kept open: current main does not implement the requested native Telegram `sendMessageDraft` transport, and recent discussion invites a contributor PR. This needs Telegram owner/product judgment because native drafts were previously removed after duplicate-finalization reports.

## What This Changes

Request to add a Telegram streaming transport option that uses Bot API `sendMessageDraft` for DM typewriter previews while falling back to `editMessageText` elsewhere.

## Best Possible Solution

Telegram channel owners should decide whether to reintroduce native drafts; if accepted, keep it plugin-owned with docs, regression tests, and live DM finalization proof, otherwise keep the durable message/edit transport from https://github.com/openclaw/openclaw/pull/75073.
