## Summary

Keep open. Current main still derives an `http://` Mattermost native slash callback when `channels.mattermost.commands.callbackUrl` is omitted and then proceeds to registration; the cross-trigger token replay portion is already hardened, but the remaining cleartext transport issue is still paired with the open closing PR at https://github.com/openclaw/openclaw/pull/65655.

## What This Changes

The issue requests hardening Mattermost native slash-command callbacks so omitted callback URLs do not default to cleartext HTTP and captured command tokens cannot be replayed across commands.

## Best Possible Solution

Ship or rework the Mattermost plugin-local transport fix via the linked PR or a narrower replacement, preserving current per-command validation and adding focused docs/tests.

## Reproduction Assessment

Yes, at source level. Enable Mattermost native slash commands without `channels.mattermost.commands.callbackUrl`; current main derives `http://...` in `resolveCallbackUrl` and registration proceeds after the warning path.

## Solution Assessment

Yes for the remaining direction, but not by closing this issue. The maintainable path is to review, rebase, or replace the linked PR with a narrow HTTPS fail-closed callback transport fix on current main.
