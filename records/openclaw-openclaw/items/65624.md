## Summary

Keep open: current `main` and the latest release still derive `http://...` Mattermost slash-command callback URLs when `commands.callbackUrl` is omitted, while the cross-command token replay portion is already hardened. The remaining transport fix is paired with an open closing PR and needs maintainer/security review for the fail-closed upgrade impact.

## What This Changes

Harden Mattermost native slash callbacks so omitted callback URLs do not register cleartext HTTP endpoints and slash-command tokens cannot be reused across commands.

## Best Possible Solution

Review and land or replace the linked Mattermost security PR with a narrow HTTPS fail-closed transport fix that preserves current per-command token validation and documents the upgrade path for existing HTTP callback deployments.

## Reproduction Assessment

Yes, source-reproducible: current `main` still derives `http://...` when `commands.callbackUrl` is omitted and proceeds toward registration. I did not run a live Mattermost network capture in this read-only review.

## Solution Assessment

Unclear until maintainers accept the compatibility tradeoff. A narrow Mattermost-plugin HTTPS fail-closed gate appears to be the right fix shape, but it should land through the linked PR or a replacement after security and upgrade review.
