## Summary

Keep open. Current main has hardened cross-trigger token replay, but it still derives `http://` Mattermost native slash callback URLs when `commands.callbackUrl` is omitted and proceeds with registration; open closing PR #65655 still tracks the remaining HTTPS fail-closed work.

## What This Changes

The issue asks to stop Mattermost native slash commands from deriving cleartext callback URLs and accepting captured slash-command tokens across commands.

## Best Possible Solution

Ship a Mattermost plugin fix that rejects non-HTTPS resolved native slash callback URLs before command registration, while preserving the current per-command/current-token validation and updating docs/tests.

## Reproduction Assessment

Yes. A source-level reproduction remains on current main for the cleartext half: enable Mattermost native slash commands without `channels.mattermost.commands.callbackUrl`, and `resolveCallbackUrl` derives an `http://` URL that the monitor still registers after only a warning. Cross-trigger replay no longer reproduces because current main now binds payloads to the resolved registered command token.

## Solution Assessment

Yes for the requested direction, but it must be reconciled with current main. The narrow maintainable solution is still Mattermost-plugin-local HTTPS fail-closed registration plus focused docs/tests; no core gateway API change is required.
