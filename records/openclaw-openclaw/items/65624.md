## Summary

Keep open. Current main and the latest release still derive and register a plain-HTTP Mattermost native slash callback when `channels.mattermost.commands.callbackUrl` is omitted; the cross-trigger token replay portion is already hardened, and the remaining transport-security issue is paired with the open closing PR https://github.com/openclaw/openclaw/pull/65655.

## What This Changes

The issue asks OpenClaw to stop defaulting Mattermost native slash-command callbacks to cleartext HTTP and to prevent reusable callback-token replay across slash triggers.

## Best Possible Solution

Land or replace the linked security fix with a maintainer-approved HTTPS fail-closed policy for Mattermost native slash callbacks, plus upgrade-facing docs and tests for secure configuration and rejected insecure callbacks.

## Reproduction Assessment

Yes. Source inspection reproduces the remaining path: enable Mattermost native slash commands, omit `channels.mattermost.commands.callbackUrl`, and current main derives an `http://` callback URL before registration; I did not run a live Mattermost replay in this read-only review.

## Solution Assessment

Unclear until maintainer policy is confirmed. Failing closed on derived non-HTTPS callbacks is the safer security direction, but the exact upgrade behavior for existing non-HTTPS deployments needs maintainer approval because it intentionally changes registration behavior.
