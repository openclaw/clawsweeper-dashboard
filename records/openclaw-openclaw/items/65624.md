## Summary

Keep open: current main and the latest release still derive and register HTTP Mattermost slash-command callbacks when `commands.callbackUrl` is omitted. The cross-command token replay portion is now hardened, but the remaining cleartext transport issue is still tracked by an open linked PR and needs maintainer/security review for the fail-closed upgrade impact.

## What This Changes

The issue asks OpenClaw to stop registering Mattermost native slash callbacks over HTTP when `callbackUrl` is omitted and to prevent replay of Mattermost command tokens across slash triggers.

## Best Possible Solution

Review and land or replace the linked Mattermost security PR with a narrow HTTPS fail-closed transport gate that preserves current per-command token validation and documents the upgrade path for existing HTTP callback deployments.

## Reproduction Assessment

Yes, source-reproducible: current main still derives an `http://` Mattermost slash callback when `commands.callbackUrl` is omitted and then proceeds toward registration after only a warning. I did not run a live Mattermost network capture in this read-only review.

## Solution Assessment

Unclear until maintainers accept the compatibility tradeoff. A narrow Mattermost-plugin HTTPS fail-closed gate appears to be the right fix shape, but it should land through the linked PR or a replacement after security and upgrade review.
