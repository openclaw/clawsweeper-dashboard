## Summary

Keep open. Current main still derives plain-HTTP Mattermost native slash callback URLs when `commands.callbackUrl` is omitted, proceeds after a reachability warning, stores registered command tokens as an account-wide set, and then trusts the caller-supplied slash trigger after token membership passes. The issue is also paired with open, unmerged closing PR #65655, so it should remain open for maintainer/security review until that PR or an equivalent fix lands.

## What This Changes

Mattermost native slash callbacks should fail closed instead of defaulting to cleartext callback URLs, and each command token should authorize only its registered slash trigger.

## Best Possible Solution

Ship a Mattermost plugin fix that rejects non-HTTPS resolved native slash callback URLs before command registration, preserves token-to-trigger bindings from Mattermost registration through HTTP dispatch, updates the public Mattermost docs, and adds regression tests for HTTPS enforcement and cross-trigger token replay.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: omit `channels.mattermost.commands.callbackUrl` to get a derived `http://...` callback, then send the slash HTTP handler any registered token with a different `payload.command`; the code checks token membership before using the supplied trigger.

## Solution Assessment

Yes. Enforcing HTTPS and binding tokens to registered triggers inside the Mattermost plugin is the narrowest maintainable fix; it does not require a core gateway or plugin API change.

## Security Review

Status: needs_attention

Summary: The issue describes a concrete security-sensitive Mattermost slash callback weakness that is still present on current main.

Concerns:

- **[high] Reject cleartext derived slash callback URLs:** `extensions/mattermost/src/mattermost/slash-commands.ts:592`
  - body: When no explicit callback URL is configured, current main derives an `http://` Mattermost slash callback and proceeds to registration. That can transport reusable command tokens in cleartext on the Mattermost-to-gateway path.
  - confidence: 0.96
- **[high] Bind command tokens to registered triggers:** `extensions/mattermost/src/mattermost/slash-http.ts:263`
  - body: The callback path accepts any registered token for the account, then trusts `payload.command` to choose the trigger. A captured token for one registered slash command can therefore be replayed against a different trigger unless token-to-trigger binding is added.
  - confidence: 0.94
