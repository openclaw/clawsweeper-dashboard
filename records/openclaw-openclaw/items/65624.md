## Summary

Keep open. Current main and the latest release still derive and register a plain-HTTP Mattermost native slash callback when `channels.mattermost.commands.callbackUrl` is omitted; the cross-trigger token replay part is already hardened, and the remaining transport issue is paired with the open closing PR at https://github.com/openclaw/openclaw/pull/65655.

## What This Changes

The issue requests hardening Mattermost native slash callbacks so omitted callback URLs do not fall back to cleartext transport and callback tokens cannot be replayed across slash triggers.

## Best Possible Solution

Review, land, or replace the linked Mattermost plugin-local fix so derived native slash callbacks fail closed unless the callback URL is HTTPS, while preserving the current per-command token validation and docs/tests.

## Reproduction Assessment

Yes, at source level. Enable Mattermost native slash commands without `channels.mattermost.commands.callbackUrl`; current main derives `http://...` in `resolveCallbackUrl` and registration proceeds with that URL.

## Solution Assessment

Yes for the remaining direction, but not by closing this issue now. The narrow maintainable solution is a Mattermost plugin fix that rejects insecure derived callback transport before registration and keeps the existing command-specific token gate.
