## Summary

Keep open. Current main still derives and registers a cleartext Mattermost native slash callback when `channels.mattermost.commands.callbackUrl` is omitted; the cross-trigger token replay half is already hardened, but the remaining transport issue is still paired with the open closing PR at https://github.com/openclaw/openclaw/pull/65655.

## What This Changes

Require Mattermost native slash callbacks to avoid cleartext derived URLs and prevent captured command tokens from replaying across slash triggers.

## Best Possible Solution

Ship a Mattermost plugin-local fix that rejects non-HTTPS resolved native slash callback URLs before registration, preserves current command-specific token validation, and updates focused docs/tests.

## Reproduction Assessment

Yes. A source-level reproduction remains on current main: enable Mattermost native slash commands without `channels.mattermost.commands.callbackUrl`, and `resolveCallbackUrl` derives `http://...` while the monitor warns and continues to registration.

## Solution Assessment

Yes for the remaining direction, but not by closing this issue. The maintainable path is to review, rebase, or replace the open PR with a narrow HTTPS fail-closed transport fix on current main.
