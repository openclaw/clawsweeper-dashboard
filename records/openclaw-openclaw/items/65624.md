- `impact:security`: The issue concerns reusable Mattermost command tokens sent over cleartext callback transport and the authorization boundary for native slash-command callbacks.

## Summary

Keep open: current main and v2026.6.10 still derive an HTTP Mattermost slash-command callback when `commands.callbackUrl` is omitted and still pass it into registration; the older cross-command token replay part is already hardened, and the remaining transport fix is tracked by the open linked PR.

## What This Changes

Mattermost native slash commands should stop defaulting to cleartext callback URLs that can expose reusable command tokens.

## Best Possible Solution

Review and land or replace the linked Mattermost PR with a narrow HTTPS fail-closed transport fix that preserves the current per-command token validation and documents the upgrade path.

## Reproduction Assessment

Yes. Current main source shows `resolveCallbackUrl()` deriving `http://...` when `commands.callbackUrl` is omitted and the registration path passing that URL into Mattermost command registration; I did not run a live network capture.

## Solution Assessment

Unclear until maintainers accept the compatibility tradeoff. A narrow Mattermost-plugin HTTPS fail-closed gate appears to be the right fix shape, but it should land through the linked PR or a replacement after security and upgrade review.
