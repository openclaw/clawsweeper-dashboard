## What This Changes

The PR changes Tavily API-key resolution to use inspect-mode SecretRef handling, fall back to `process.env.TAVILY_API_KEY` only for matching allowed env refs or missing config, and adds resolver regression tests.

## Best Possible Solution

Land one canonical guarded inspect-mode Tavily resolver that preserves explicit non-env and nonmatching credential boundaries while restoring the documented `TAVILY_API_KEY` fallback for the matching env-ref path.

## Reproduction Assessment

## Solution Assessment

Yes, this is a strong fix candidate at the right plugin-owned resolver boundary. A broader SDK helper could be a later cleanup, but the guarded inspect-mode Tavily change is the narrowest maintainable repair for this shipped bug.
