## Summary

Keep open. Current main still lacks a channel slash command that manually starts or resumes Lobster or Task Flow workflows, and the remaining work needs maintainer decisions around side-effectful command authorization and workflow argument handling.

## What This Changes

Request a slash command that can manually execute Task Flow or Lobster workflows from chat, with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command or ClawHub plugin pattern that reuses registerCommand and managedFlows once maintainers settle the safety and UX contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: Lobster is tool-only, /tasks rejects arguments, and docs only expose Task Flow inspection/cancellation commands.

## Solution Assessment

Unclear until maintainers choose the command contract. A Lobster-owned plugin command reusing registerCommand and managedFlows looks narrower than a new core built-in, but the side-effect policy must be defined first.

## Risks / Open Questions

- The command contract is not settled: command name, run vs resume shape, JSON argument schema, and error/reporting behavior all need product agreement.
- Channel-triggered workflow execution can perform side effects, so authorization, owner-only behavior, approval routing, and session binding need explicit maintainer design.
- It is unclear whether the first implementation should be a bundled Lobster plugin command, a ClawHub plugin/skill pattern, or a narrower Task Flow API extension.
