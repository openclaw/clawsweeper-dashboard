## Summary

Keep open. Current main still exposes Lobster execution through the optional agent tool and exposes Task Flow through inspect/cancel/status surfaces, but I found no explicit `/lobster`, `/taskflow`, or `/tasks ... run` channel command. The plugin command API is available, so the feature looks feasible, but the side-effectful command contract needs maintainer product/security decisions before a safe automated repair PR.

## What This Changes

Add a channel slash command that manually runs or resumes Lobster workflows or Task Flow flows with optional JSON arguments, without requiring terminal or curl access.

## Best Possible Solution

Approve a narrow Lobster-owned runtime slash command contract, then wrap the existing Lobster runner and managed Task Flow APIs with explicit JSON parsing, authorized active-session binding, preserved approval/resume semantics, focused command/tool tests, and docs for channel usage and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path. Source and docs inspection on current main show no `/lobster`, `/taskflow`, or flow-run chat command, and `/tasks` rejects arguments rather than starting flows.

## Solution Assessment

Unclear until maintainers settle the command contract. A plugin-owned command around the existing Lobster/Task Flow execution paths appears narrower than adding a core workflow command, but the security and UX details are not specified enough for an automated fix PR.
