## Summary

Keep open. Current main still exposes Lobster execution through the optional agent tool and Task Flow through inspect/cancel/status surfaces, but it does not provide an explicit channel slash command to run or resume Lobster/Task Flow work. The plugin command API is present, so this is feasible, but the side-effectful command contract needs maintainer design before a safe repair PR is queued.

## What This Changes

Add a slash or channel command that manually executes Lobster workflows or Task Flow runs, optionally with JSON arguments.

## Best Possible Solution

Approve a narrow Lobster-owned plugin command contract, then implement it by reusing the existing Lobster runner and managed Task Flow runtime with deliberate JSON parsing, authorized active-session binding, preserved approval/resume semantics, focused tests, and docs for channel usage and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path. Current source, tests, and docs show no /lobster, /taskflow, or flow-run command, and /tasks with arguments returns Usage: /tasks.

## Solution Assessment

Unclear until maintainers settle the command contract. A plugin-owned command wrapping the existing Lobster/Task Flow execution paths appears narrower than adding a core workflow command, but the UX and security details are not specified enough for an automated fix PR.
