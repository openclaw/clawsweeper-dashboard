## Summary

Keep open. Re-checking current main shows the prior ClawSweeper assessment still holds: Lobster can run/resume workflows as an optional agent tool and Task Flow can be inspected or cancelled, but there is still no explicit channel slash command for manually starting or resuming Lobster/Task Flow work. The existing plugin command seam looks capable, but the command contract and security model need maintainer decisions first.

## What This Changes

Request a chat slash command to manually execute Task Flow or Lobster workflows, optionally with JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the embedded runner and managed Task Flow runtime, validates JSON strictly, binds to an authorized active conversation/session, preserves approval and resume semantics, and documents channel behavior and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but the missing feature is high-confidence from source and docs inspection. Current main has no /lobster or /taskflow command, and /tasks with arguments is explicitly rejected.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a generic core workflow command, but the security and UX details are not specified enough for an automated repair PR.
