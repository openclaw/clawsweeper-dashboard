## Summary

Keep open. Current main still has tool-based Lobster execution and status-only Task Flow slash/CLI surfaces, so the requested channel command would be a new side-effectful capability that needs maintainer UX and safety decisions.

## What This Changes

The issue asks for a channel slash command to manually execute Task Flow or Lobster workflows with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the existing run/resume and managed-flow helpers after maintainers settle authorization, JSON parsing, session binding, and approval behavior.

## Reproduction Assessment

Not applicable as a runtime bug. Source verification is high confidence: current main has no Lobster `registerCommand`, `/tasks` rejects arguments, and docs only expose Task Flow inspection/cancellation plus the Lobster tool path.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin command using `api.registerCommand` and `api.runtime.tasks.managedFlows` appears narrower than a new core built-in, but the side-effect policy must be decided first.
