## Summary

Keep open. Current main still has Lobster execution as an optional agent tool and Task Flow inspection/cancellation surfaces, but no explicit channel slash command that manually starts or resumes Lobster or Task Flow workflows. The existing plugin command API can support the feature, but channel-triggered workflow execution is side-effectful enough that maintainers should settle the command contract, authorization, session binding, JSON parsing, and approval/resume behavior first.

## What This Changes

Request a chat slash command to manually execute Task Flow or Lobster workflows, optionally with JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the existing embedded Lobster runner and managed Task Flow runtime, validates JSON strictly, binds to an authorized active conversation/session, preserves approval and resume semantics, and documents channel behavior and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence source verification path. Current main has no /lobster, /taskflow, or Task Flow run command, and /tasks with arguments is explicitly rejected.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a core workflow command, but the security and UX details are not specified enough for an automated repair PR.
