## Summary

Keep open. Current main still lacks a channel slash command for manually running or resuming Lobster/Task Flow workflows, and the remaining work needs maintainer decisions around side-effectful command authorization, approval routing, and JSON argument handling.

## What This Changes

Add a slash command that manually executes or resumes Task Flow/Lobster workflows from chat channels, with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin slash command or ClawHub plugin pattern that reuses `registerCommand` and `api.runtime.tasks.managedFlows`, with owner-gated execution, explicit approvals, and validated JSON arguments.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection gives a high-confidence current-state check: Lobster is tool-only, `/tasks` rejects arguments, and docs expose only inspection/cancel commands for Task Flow.

## Solution Assessment

Unclear until maintainers choose the command contract. A Lobster-owned plugin command looks narrower than a new core built-in, but the side-effect policy and JSON argument shape need explicit product/security review first.
