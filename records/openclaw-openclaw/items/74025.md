## Summary

Keep open. Current main still lacks a channel slash command that starts or resumes Lobster or Task Flow workflows, and the remaining work needs maintainer decisions around side-effectful command authorization, session binding, approval routing, and argument validation.

## What This Changes

Request adds a chat slash command to manually run or resume Task Flow/Lobster workflows with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that wraps the existing embedded runner and Task Flow runtime after maintainers settle the safety and UX contract.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: current main has no Lobster registerCommand, /tasks rejects arguments, and docs only document Task Flow inspection/cancellation paths.

## Solution Assessment

Unclear until maintainers choose the command contract. A Lobster plugin command reusing api.registerCommand and api.runtime.tasks.managedFlows appears narrower than a new core built-in, but direct channel-triggered workflow execution needs explicit product/security design first.
