## Summary

Keep open. Current main still lacks a channel slash command to start or resume Lobster or Task Flow workflows; the existing tool/runtime pieces are present, but chat-triggered workflow execution needs maintainer product and security design before cleanup can close it.

## What This Changes

The issue requests a slash command that manually runs Task Flow or Lobster workflows from a chat channel, optionally with JSON arguments.

## Best Possible Solution

Define a plugin-owned Lobster command that wraps the embedded runner and managed Task Flow runtime after maintainers settle the authorization, session, approval, and channel-delivery contract.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: current main has no Lobster `registerCommand`, `/tasks` rejects arguments, and docs only document Task Flow inspect/cancel commands.

## Solution Assessment

Unclear. A Lobster-owned plugin command appears narrower than a new core built-in, but the proposed slash command is not complete until maintainers define the security and product contract for chat-triggered workflow execution.
