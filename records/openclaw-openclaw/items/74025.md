## Summary

Keep open. Current main still exposes Lobster execution through the optional agent tool surface and Task Flow through inspect/cancel task surfaces, but I found no chat slash command that manually starts or resumes Lobster/Task Flow workflows; the remaining work needs maintainer design on syntax, authorization, session binding, JSON parsing, and approval/resume semantics.

## What This Changes

Request a chat slash command that manually executes Task Flow or Lobster workflows, optionally accepting JSON arguments.

## Best Possible Solution

Define a narrow Lobster-owned slash-command contract that reuses the existing embedded Lobster runner and managed Task Flow runtime, preserves approval/resume behavior, binds execution to an authorized active session, validates JSON strictly, and documents channel usage and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but yes for source-level verification. Current main has a clear inspection path showing no `/lobster`, `/taskflow`, or Task Flow run slash command while `/tasks` is status-only and rejects arguments.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a core workflow command, but the side-effectful security and UX semantics need explicit maintainer approval first.
