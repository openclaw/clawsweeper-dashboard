## Summary

Keep open. Current main still lacks a channel slash command that manually starts or resumes Lobster or Task Flow workflows, while the existing Lobster tool/runtime and plugin command API show this is actionable in the repo after maintainer product/security design.

## What This Changes

Request a slash command that manually executes a Task Flow or Lobster workflow, with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the embedded runner and managed Task Flow runtime, validates JSON strictly, binds to an authorized active conversation/session, preserves approval/resume behavior, and updates slash-command/Lobster/Task Flow docs and tests.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence source verification path: current main has no `/lobster` or `/taskflow` command, `/tasks` rejects arguments, and docs expose only Task Flow list/show/cancel.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a new core built-in, but the side-effect and authorization model should be settled first.
