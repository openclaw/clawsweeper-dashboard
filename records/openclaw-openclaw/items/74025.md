## Summary

Keep open. Current main still exposes Lobster execution through the optional agent tool surface and Task Flow through inspect/cancel/status surfaces, but there is no explicit chat slash command to manually start or resume Lobster or Task Flow workflows. The existing plugin command API makes this feasible, but the command would directly trigger side effects from channels, so maintainers should settle the syntax, JSON parsing, authorization, session binding, and approval/resume behavior before implementation.

## What This Changes

Request to add a chat slash command that manually runs or resumes Task Flow/Lobster workflows, optionally with JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command contract and implement it by reusing the embedded Lobster runner plus managed Task Flow runtime, with strict JSON parsing, authorized active-session binding, preserved approval/resume semantics, focused tests, and user-facing docs for channel usage and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence source verification path. Current main has no /lobster, /taskflow, or Task Flow run command, and /tasks with arguments is explicitly rejected.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than adding a core workflow command, but the security and UX decisions are not specified enough for an automated repair PR.
