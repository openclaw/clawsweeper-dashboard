## Summary

Keep open. Current main still exposes Lobster execution through the optional agent tool surface and Task Flow through inspect/cancel CLI and /tasks status surfaces, but there is no explicit channel slash command that manually starts or resumes a Lobster or Task Flow workflow. The existing plugin command API makes this feasible, but the side-effectful command contract needs maintainer security and UX decisions before an automated fix PR is appropriate.

## What This Changes

Request a chat slash command to manually execute Task Flow or Lobster workflows, optionally with JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the existing embedded Lobster runner and managed Task Flow runtime, validates JSON strictly, binds to an authorized active conversation/session, preserves approval/resume semantics, and documents the channel behavior and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence source verification path. Current main has no /lobster, /taskflow, or Task Flow run command, and /tasks with arguments is explicitly rejected.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a core workflow command, but the security and UX details are not specified enough for an automated repair PR.
