## Summary

Keep open. Current main still exposes Lobster execution through an optional agent tool and Task Flow through runtime/CLI inspection, not a channel slash command, and the requested side-effectful command needs maintainer decisions around authorization and argument format.

## What This Changes

Add a channel slash command that manually starts or resumes Task Flow or Lobster workflows with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command contract that reuses existing Plugin SDK command and managed Task Flow APIs, with explicit owner/auth checks and structured argument validation.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: Lobster is tool-only, /tasks rejects arguments, and docs expose inspection/tool-mode paths rather than a run slash command.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned command looks narrower than a new core built-in, but the side-effect policy and argument schema need explicit design first.
