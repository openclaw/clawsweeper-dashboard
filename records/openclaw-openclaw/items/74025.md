## Summary

Keep open. Current main still lacks a channel slash command that manually starts or resumes Lobster or Task Flow workflows; existing behavior is tool-based or status/cancel-only, and the remaining work needs maintainer product/security decisions for a side-effectful channel command.

## What This Changes

Request a slash command that manually executes Task Flow or Lobster workflows from a channel, with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the embedded runner and session-bound Task Flow runtime after maintainers settle the authorization, approval, and UX contract.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence source verification path. Current main has no Lobster `registerCommand`, `/tasks` rejects arguments, and the docs only expose Task Flow inspection/cancellation paths.

## Solution Assessment

Unclear until maintainers choose the command contract. A Lobster plugin command using `api.registerCommand` looks narrower than a new core built-in, but direct channel-triggered workflow execution needs explicit safety and UX design first.
