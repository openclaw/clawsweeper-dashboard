## Summary

Keep open. Current main still exposes Lobster execution through an optional agent tool and Task Flow through inspect/status/cancel surfaces, but it does not provide an explicit channel slash command to manually run or resume a Lobster/Task Flow workflow. The existing plugin command API appears sufficient for a future implementation, but the side-effectful command contract needs maintainer design before this is safe to queue as an automated repair.

## What This Changes

Request a channel slash command that manually executes Task Flow or Lobster workflows, optionally with JSON arguments.

## Best Possible Solution

Define a narrow Lobster-owned plugin command contract, then implement it by reusing the existing Lobster runner and managed Task Flow helpers with strict JSON parsing, authorized active-session binding, preserved approval/resume semantics, focused tests, and docs for channel usage and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path. Source and docs inspection on current main show no /lobster, /taskflow, or flow-run command, while /tasks accepts only the bare status form.

## Solution Assessment

Unclear until maintainers settle the command contract. A plugin-owned command wrapping the existing Lobster/Task Flow execution paths looks narrower than adding a core workflow command, but the security and UX details are not specified enough for an automated fix PR.
