## Summary

Keep open. Current main still exposes Lobster run/resume through the optional agent tool and Task Flow inspection/cancellation through `/tasks` plus CLI commands, but it does not provide a channel slash command to start or resume Lobster or Task Flow workflows; the remaining decision is the product/security contract for side-effectful workflow execution from chat.

## What This Changes

Request a channel slash command to manually execute Task Flow or Lobster workflows, with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the embedded runner and managed Task Flow runtime after maintainers settle authorization, session binding, approval/resume, and channel behavior.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: current main has no Lobster `registerCommand`, `/tasks` rejects arguments, and docs only document Task Flow inspect/cancel commands.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a new core built-in, but direct channel-triggered workflow execution needs explicit product/security design first.
