## Summary

Keep open. Current main still has the Lobster run/resume path as an optional agent tool and Task Flow inspection/cancellation through `/tasks` and CLI commands, but it does not expose a channel slash command that manually starts or resumes Lobster or Task Flow workflows.

## What This Changes

Request a chat slash command that manually runs or resumes Task Flow/Lobster workflows, with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the embedded runner and `api.runtime.tasks.managedFlows.bindSession` after maintainers settle the authorization, approval, and channel-delivery contract.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: current main has no Lobster `registerCommand`, `/tasks` rejects arguments, and docs only document Task Flow inspect/cancel commands.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a core built-in, but direct channel-triggered workflow execution needs explicit product/security design first.
