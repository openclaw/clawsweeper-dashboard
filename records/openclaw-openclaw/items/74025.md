## Summary

Keep open. Current main still lacks a channel slash command that manually starts Lobster or Task Flow workflows, and the requested chat-triggered execution path needs maintainer decisions around authorization, approval routing, and JSON argument handling.

## What This Changes

The issue requests a slash command that manually executes Task Flow or Lobster workflows, optionally with JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command or documented ClawHub plugin pattern that reuses registerCommand and the existing managed Task Flow execution path after maintainers settle the authorization and argument contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: Lobster is exposed as a tool, /tasks rejects arguments, and docs only expose Task Flow inspection/cancellation commands.

## Solution Assessment

Unclear until maintainers choose the command contract. A Lobster-owned plugin command reusing registerCommand and managed Task Flow APIs looks narrower than a new core built-in, but the side-effect policy must be defined first.
