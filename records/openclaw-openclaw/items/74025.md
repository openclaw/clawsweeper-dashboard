## Summary

Keep open. Current main still exposes Lobster execution through the optional agent tool and Task Flow inspection/cancellation through `/tasks` and the CLI, but it does not provide a channel slash command that manually starts or resumes Lobster or Task Flow workflows; the existing plugin command seam can support this, but the command contract needs maintainer product/security design first.

## What This Changes

Add a chat slash command to manually start or resume Task Flow/Lobster workflows, with optional JSON arguments.

## Best Possible Solution

Define a Lobster-owned plugin command that reuses the embedded runner and managed Task Flow runtime, strictly validates JSON, binds to an authorized active conversation/session, preserves approval/resume behavior, and documents channel-specific behavior.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence source verification path: current main has no `/lobster` or `/taskflow` command, `/tasks` rejects arguments, and docs only expose Task Flow list/show/cancel.

## Solution Assessment

Unclear until maintainers choose the command contract. A plugin-owned Lobster command appears narrower than a new core built-in, but the side-effect and authorization model should be settled before implementation.
