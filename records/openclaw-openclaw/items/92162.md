## What This Changes

The branch changes `reparseProgramFromActionArgs` to reparse from the Commander root command and updates the action-reparse unit expectations.

## Best Possible Solution

Land the narrow root-reparse fix after normal CI/maintainer validation, and let https://github.com/openclaw/openclaw/issues/92069 close only after the PR merges.

## Reproduction Assessment

Yes, from source inspection: current main can reparse an intermediate lazy command after Commander has cleared that command's `rawArgs`, which makes unknown-option handling depend on the wrong argv source. I did not execute the CLI because the checkout must remain read-only.

## Solution Assessment

Yes, this appears to be the best fix: reparsing from the true Commander root uses the object that owns `_prepareUserArgs` state and avoids duplicating command-path reconstruction logic. The main alternative, expanding fallback argv reconstruction for nested commands, would be a more fragile parallel parser.
