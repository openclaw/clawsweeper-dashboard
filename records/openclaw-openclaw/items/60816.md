## Summary

Keep this open: current main and v2026.6.10 still do not return a distinct already-at-target noop for identical edit followers, and the live closing PR is still open and not merge-ready.

## What This Changes

Request distinct edit-tool result semantics for concurrent identical edits: first writer succeeds, the follower reports noop when the file is already at target, and true mismatches remain errors.

## Best Possible Solution

Review and land a repaired version or narrow replacement of https://github.com/openclaw/openclaw/pull/77638 that implements substring-safe already-at-target noop classification in the active session edit tool and covers both single replace and edits[] inputs.

## Reproduction Assessment

Yes, at source level. Same-file edits serialize, an identical follower can read the leader's already-updated content, and the active mismatch path rethrows with a hint instead of returning a distinct noop.

## Solution Assessment

Yes for the requested semantics, but not via the current linked PR as submitted. The maintainable fix belongs in `src/agents/sessions/tools/edit.ts` with active-runtime tests and true mismatch/write-failure behavior preserved.
