## Summary

Keep this open: current main and v2026.6.10 still lack the requested distinct noop result, and the linked closing PR is open, conflicting, and targets legacy edit-wrapper files rather than the active session edit tool.

## What This Changes

The issue requests that identical concurrent edit followers return a distinct noop result when the file is already at the desired target state, while preserving success for real mutations and errors for true mismatches.

## Best Possible Solution

Land a repaired version of https://github.com/openclaw/openclaw/pull/77638, or a narrow replacement, that implements substring-safe noop classification in `src/agents/sessions/tools/edit.ts` with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Same-file edits serialize, an identical follower can read the leader's already-updated file, and current mismatch handling throws with a hint instead of returning a distinct noop.

## Solution Assessment

Yes for the requested semantics, but not for the open PR as currently shaped. The maintainable fix should live in the active session edit tool and preserve true mismatch/write-failure errors.
