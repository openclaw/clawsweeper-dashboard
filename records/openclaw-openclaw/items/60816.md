## What This Changes

The issue requests distinct edit-tool semantics for concurrent identical edits so the first mutation reports success, followers whose desired state is already present report noop, and true mismatches still fail.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77638 after conflict resolution, or replace it with an equivalent focused post-state classifier and regression tests, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: same-file edits are serialized upstream, and a follower whose old text was already replaced reaches OpenClaw's recovery wrapper with no distinct noop branch on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a post-state classifier in wrapEditToolWithRecovery that returns noop only when the requested final state is already satisfied while preserving true mismatch and write-failure errors.
