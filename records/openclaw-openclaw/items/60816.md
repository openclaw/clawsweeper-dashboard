## Summary

Keep open: current main still has only success/error edit recovery classification, and live GitHub shows https://github.com/openclaw/openclaw/pull/77638 remains open as the closing implementation path for this issue.

## What This Changes

The issue requests a distinct noop result when concurrent identical edit requests find the file already in the requested final state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77638, or replace it with an equivalent narrow wrapper/test fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: same-file edit calls are queued upstream, and a follower whose target is already present reaches OpenClaw’s recovery wrapper with no distinct noop branch on current main.

## Solution Assessment

Yes. The narrow maintainable solution is a post-state classifier in the existing edit recovery wrapper that returns noop only when the requested final state is satisfied while preserving real mismatch and write failures.
