## What This Changes

The issue asks the edit tool to classify concurrent identical edits that find the file already in the desired final state as a distinct noop while preserving success for actual writes and errors for true mismatches.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77638 after resolving conflicts, or apply an equivalent focused post-state classifier with regression tests, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an unchanged already-at-target follower bypasses the success recovery branch and reaches the exact-match mismatch throw path instead of returning a distinct noop.

## Solution Assessment

Yes. The narrow maintainable fix is a post-state classifier in wrapEditToolWithRecovery that returns noop only when the requested final state is already satisfied while preserving real success and true failure behavior.
