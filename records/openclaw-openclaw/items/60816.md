## Summary

Keep open: current main still lacks the requested distinct noop result, and an open linked fix PR already owns the implementation path but has not merged.

## What This Changes

Classify already-at-target identical edit followers as a distinct noop result while preserving normal success and true edit failures.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/77638; after it merges, close this issue with the merged commit and release/current-main proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an `edits[]` follower whose requested `oldText` is already gone but whose `newText` is present still goes through the mismatch/rethrow path on current main because there is no already-at-target noop classifier.

## Solution Assessment

Yes. A narrow post-state classifier in the existing edit recovery wrapper is the maintainable fix; broad serialization or suppressing all mismatch errors would hide real failures.
