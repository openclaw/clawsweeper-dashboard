## What This Changes

The issue asks the edit tool to distinguish a real write, an already-at-target noop from an identical concurrent edit, and a true mismatch/error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77638 after rebase/conflict resolution, or replace it with an equivalent narrow wrapper/test fix, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: same-file edits are serialized upstream, and a follower whose old text was already replaced reaches OpenClaw's recovery wrapper with no distinct noop branch on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a post-state classifier in `wrapEditToolWithRecovery` that returns noop only when the requested final state is already satisfied while preserving real mismatch and write failures.
