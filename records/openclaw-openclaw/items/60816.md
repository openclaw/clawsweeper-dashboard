## Summary

Keep this open: current main and v2026.6.10 still lack the requested distinct noop result for an already-at-target identical edit follower, and the linked closing PR is still open and not merged.

## What This Changes

The issue requests that identical concurrent edit followers return a distinct noop result when the file is already in the requested final state, while real mutations remain success and true mismatches remain errors.

## Best Possible Solution

Land a repaired version of https://github.com/openclaw/openclaw/pull/77638, or a narrow replacement, in `src/agents/sessions/tools/edit.ts` with focused regression coverage for single-edit and edits[] noop classification.

## Reproduction Assessment

Yes, at source level. Same-file edits serialize, an identical follower can read already-updated content, and current mismatch handling throws with a hint instead of returning a distinct noop.

## Solution Assessment

Yes for the requested semantics, but the repair should be applied to the active session edit tool rather than the stale legacy wrapper path. The open PR can still be the candidate if it is ported or replaced narrowly.
