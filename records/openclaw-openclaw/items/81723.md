## Summary

Keep open: current main still has a source-reproducible path where a full same-channel WeChat direct session key can be accepted as a Weixin target and then classified as a group route, so the central message-loss bug is not fixed.

## What This Changes

The issue requests that `message(action=send)` deliver to a WeChat direct session target instead of nesting that session key under a synthetic group session.

## Best Possible Solution

Add a generic core guard or normalization path so same-channel direct agent session keys cannot be treated as group targets; preferably unwrap supported direct session keys to the provider peer id and otherwise fail with a clear error.
