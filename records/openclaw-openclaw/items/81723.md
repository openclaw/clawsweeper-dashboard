## Summary

Keep open. Current main still has a plausible generic message-target normalization gap: a full OpenClaw agent session key can pass into provider target resolution and outbound session mirroring as a raw group target, which matches the reported doubled `group:agent:...:direct:...` session shape.

## What This Changes

Report that proactive `message(action=send)` to a WeChat direct session key is routed/mirrored as a group session instead of the direct peer.

## Best Possible Solution

Normalize canonical OpenClaw session-key targets before provider target resolution, mapping direct/group/channel/account-scoped session keys to the correct channel target when safe and failing closed with clear guidance for unsupported shapes.

## Solution Assessment

Yes in direction: the maintainable fix is generic session-key target normalization or fail-closed validation before provider target resolution. A WeChat-only workaround would be the wrong owner boundary.
