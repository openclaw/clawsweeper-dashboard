## Label Justifications

- `P1`: The reported path can make a successful WeChat send miss the intended direct user and split a live conversation into a phantom group session.
- `impact:message-loss`: The issue says the API returns success while the WeChat direct user does not receive the message because it is routed to the wrong session type.
- `impact:session-state`: The same WeChat peer can be represented by conflicting direct and group session keys, which drifts conversation state and reset behavior.

## Summary

This issue can close as duplicate/superseded because the same WeChat direct-peer-to-phantom-group session failure is now tracked by the canonical open issue at https://github.com/openclaw/openclaw/issues/92384, with active implementation work at https://github.com/openclaw/openclaw/pull/92995.

## What This Changes

The issue requests that `message(action=send)` to a WeChat direct target deliver and mirror into the direct session instead of creating or using a group session.

## Best Possible Solution

Close this duplicate and keep the remaining fix and proof discussion on https://github.com/openclaw/openclaw/issues/92384 and its open implementation PR.
