## Summary

Keep open: current main and v2026.5.22 still lack an agent-facing browser-tool action for request listing, response bodies, CDP Network, Fetch, or interception. The lower-level browser control API already has passive request and response-body routes, so the remaining work is a security-sensitive model-facing API decision rather than cleanup to close.

## What This Changes

The issue requests browser-tool support for CDP Network-style request capture/interception so agents can inspect SPA traffic beyond fetch hooks.

## Best Possible Solution

Expose a narrow passive request and response-body inspection action through the browser tool first, then design mutating CDP/Fetch interception separately with explicit security policy, redaction, scoping, and regression tests.

## Reproduction Assessment

Yes for the source-level feature gap: the browser tool schema and dispatcher still do not accept network, requests, responsebody, CDP, Fetch, or intercept actions, while lower-level `/requests` and `/response/body` routes exist. I did not run a live browser session because this review is read-only.

## Solution Assessment

Unclear as proposed. Passive inspection can reuse existing control routes, but raw CDP Network or mutating interception is broader than necessary and needs maintainer API/security design first.
