## Summary

Keep open: current main and latest release still do not expose model-facing CDP Network, request-listing, response-body, or interception actions through the browser tool. Lower-level CLI/control routes exist for passive request metadata and response bodies, but they do not satisfy the requested agent-facing API and interception capability.

## What This Changes

Expose browser tool actions for CDP Network request capture, request listing, and request interception during browser automation.

## Best Possible Solution

Define a narrow browser-plugin network-inspection API with explicit redaction and permission boundaries, expose passive request/response inspection first, and add interception only after maintainer-approved security semantics.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.8` shows the browser tool schema and dispatcher do not expose network, requests, responsebody, CDP, Fetch, or intercept actions, while lower-level control routes exist separately.

## Solution Assessment

No as proposed. Raw CDP Network-domain exposure is broader than the safest maintainable solution; a narrow audited browser-plugin API with redaction and permission semantics is the better direction.
