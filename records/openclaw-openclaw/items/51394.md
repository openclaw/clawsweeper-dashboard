## Summary

Keep open: current main and the latest release still do not expose request listing, response-body lookup, CDP Network, Fetch, or interception through the model-facing browser tool. The lower-level browser control API already has passive request and response-body routes, so the remaining work is a security-sensitive product/API decision rather than cleanup to close.

## What This Changes

The issue requests a model-facing browser tool API for CDP Network request capture and interception.

## Best Possible Solution

Define a narrow browser-plugin API for safe network inspection, and only add interception if maintainers approve explicit security boundaries, redaction, permissioning, and tests for agent-facing use.

## Reproduction Assessment

Yes. Source inspection on current main shows the browser tool schema and dispatch do not expose requests, response bodies, CDP Network, Fetch, or interception, while lower-level control routes exist separately.

## Solution Assessment

Unclear as an implementation plan. The issue identifies a real capability gap, but the best fix needs maintainer product/security design before exposing network traffic or interception to the model-facing tool.
