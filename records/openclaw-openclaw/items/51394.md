## Summary

Keep open: current main still lacks an agent-facing browser Network/intercept action. The control API and CLI already provide passive request listing and response-body capture, but exposing request interception to agents is a new feature with product and security policy still unresolved.

## What This Changes

Request an agent-facing browser tool API for CDP Network request capture/interception and request/response debugging.

## Best Possible Solution

Expose a narrow passive request/response inspection action first, and handle mutating interception as a separately scoped design with target scoping, permission policy, and tests.

## Reproduction Assessment

Yes for the source-level feature gap. BrowserToolSchema and createBrowserTool do not accept network, requests, responsebody, or intercept actions, while lower-level /requests and /response/body routes exist.

## Solution Assessment

Unclear as proposed. Passive inspection can reuse the existing control routes, but raw CDP Network or request interception exposure is broader than necessary and needs maintainer API/security design first.
