## Summary

Keep open: current main has some lower-level Playwright-backed request observation and CLI debugging commands, but the model-facing browser tool still does not expose the requested Network-domain request listing/interception API. This is security-sensitive browser capability work, so it needs maintainer product and security review before implementation.

## What This Changes

Request to expose CDP Network-domain request capture and interception through the browser agent tool for SPA/API debugging.

## Best Possible Solution

Design a narrow browser-plugin network inspection API that reuses existing guarded Playwright request/responsebody plumbing where safe, keeps profile limitations explicit, and adds security review, docs, and regression coverage before model-facing exposure.

## Reproduction Assessment

Not applicable as a runtime bug, but yes for the feature gap: current main's browser tool schema and docs do not expose network, requests, or intercept actions. The existing request/responsebody paths are CLI/debug and Playwright/profile-limited, so source inspection is enough to confirm the requested surface is still missing.

## Solution Assessment

No, not as the raw proposed `browser act kind=network` shape without design review. The maintainable path is to decide the browser-plugin API and security policy first, then reuse existing guarded Playwright request/response plumbing where it fits.
