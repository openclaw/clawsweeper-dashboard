## Summary

Keep open: current main has CLI/control-API request observation and response-body helpers, but the agent `browser` tool still has no network/intercept action and no canonical issue or PR owns the remaining API work.

## What This Changes

The issue requests exposing CDP Network-domain capabilities in the browser tool so automation can capture, inspect, and intercept requests.

## Best Possible Solution

Design a permission-aware browser-plugin network capture/interception surface that extends the existing request/debug routes and agent tool schema without bypassing browser security policy.

## Reproduction Assessment

Not applicable as a bug reproduction. Source review gives high confidence that current main lacks the requested browser-tool network/intercept capability while retaining narrower request observation helpers.

## Solution Assessment

Unclear. The capability is plausible, but the proposed `act kind=network` shape is not obviously the best fit because current `act` is UI-oriented and network helpers currently live in debug/control routes.
