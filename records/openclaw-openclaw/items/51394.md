## Summary

Keep open: current main and `v2026.6.6` still do not expose request listing, response-body lookup, CDP Network, Fetch, or interception through the model-facing browser tool. Lower-level browser control routes and SSRF route-interception work exist, but they do not resolve the requested agent-facing API or its security boundary.

## What This Changes

The issue requests CDP Network-domain request capture and interception through the model-facing browser tool.

## Best Possible Solution

Define a narrow browser-plugin network-inspection API with explicit redaction and permission boundaries, expose passive request/response inspection first, and add interception only after maintainer-approved security semantics, docs, and tests.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.6` shows the model-facing browser tool schema and dispatch do not expose network, requests, responsebody, CDP Network, Fetch, or intercept actions, while lower-level control routes exist separately.

## Solution Assessment

Unclear as proposed. A raw CDP Network-domain exposure is broader than the safest maintainable solution; a narrow audited browser-plugin API with redaction and permission semantics is the better direction.
