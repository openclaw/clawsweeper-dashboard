## Summary

Keep open: current main still lacks an agent-facing browser Network/intercept action. Passive request listing and response-body routes already exist below the browser tool, but exposing them, especially mutating interception, needs maintainer product/security design.

## What This Changes

Request a browser tool API that lets agents inspect and intercept CDP Network traffic for SPA request discovery and debugging.

## Best Possible Solution

Expose a narrow passive request and response-body inspection action through the browser tool first, then design mutating CDP/Fetch interception separately with explicit target scoping, permission policy, redaction, SSRF interaction, and regression tests.

## Reproduction Assessment

Yes for the source-level feature gap. Current BrowserToolSchema and createBrowserTool do not accept or route network, requests, responsebody, intercept, CDP, or Fetch actions, while lower-level /requests and /response/body routes exist.

## Solution Assessment

Unclear as proposed. Passive inspection can reuse existing control routes, but raw CDP Network or request interception exposure is broader than necessary and needs maintainer API/security design first.
