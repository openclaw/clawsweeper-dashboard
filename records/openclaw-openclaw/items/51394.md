## Summary

Keep open: current main and the latest release still do not expose request listing, response-body lookup, CDP Network, Fetch, or interception through the model-facing browser tool. The lower-level browser control API has passive request and response-body routes, and related SSRF/proxy-capture work exists, but none of that resolves the requested agent-facing network API or its security boundary.

## What This Changes

The issue requests a browser-tool network capability for request capture, request/response inspection, and URL-pattern interception during browser automation.

## Best Possible Solution

Define a narrow browser-plugin network-inspection API with explicit redaction and permission boundaries, expose passive request/response inspection first, and add interception only after maintainer-approved security semantics, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.6 shows the model-facing browser tool schema and dispatch do not expose network, requests, responsebody, CDP Network, Fetch, or intercept actions, while lower-level control routes exist separately.

## Solution Assessment

Unclear as proposed. A raw CDP Network-domain exposure is not yet the best solution; the safer direction is a narrow, audited browser-plugin API with redaction, permissions, and explicit maintainer security choices.
