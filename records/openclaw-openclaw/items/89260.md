The PR adds a derived `platformIncompatible` skill-status flag, uses it to exclude OS-mismatched skills from doctor missing-requirements counts and doctor --fix disable targets, and adds focused regression coverage.

## Best Possible Solution

Land this focused doctor fix after normal maintainer and CI review; any broader `openclaw skills check` presentation change can stay separate unless maintainers want the same rollup outside doctor.

## Reproduction Assessment

## Solution Assessment

Yes. Deriving the classification from the shared requirement evaluator is the best bounded fix for the doctor bug, and updating both the status rollup and disable target covers the reported user-visible paths without adding config or duplicate platform logic.
