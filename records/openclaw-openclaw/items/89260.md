## Summary

Keep this PR open: the direction is right, but the patch is not merge-ready because it adds a local-only platform flag and applies it only to the workspace summary while doctor repair/readiness still treats platform-mismatched skills as unavailable.

## What This Changes

The PR adds a skill-status platform mismatch flag and changes `openclaw doctor` workspace output so OS-incompatible skills are rolled up separately from missing requirements.

## Best Possible Solution

Derive the platform-mismatch classification from the existing requirement evaluation semantics, then use it consistently in doctor summary and doctor repair/readiness filtering.

## Reproduction Assessment

Yes. Source inspection shows current `buildWorkspaceSkillStatus` marks OS-mismatched skills ineligible with `missing.os`, and current `noteWorkspaceStatus` counts them in Missing requirements.

## Solution Assessment

No. The proposed fix is plausible but too narrow because it adds a separate local-only OS check and does not update the doctor repair/readiness path that can still disable those skills.
