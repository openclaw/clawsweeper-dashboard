## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible: with `cron.minInterval: "5m"`, a 30s retry backoff, and a recurring every-10m job failing with a retryable `429`, the PR head schedules `endedAt + 30s` and returns before applying the floor.

## Solution Assessment

No; creation validation plus a partial fire-time floor is not the best guardrail while one automatic retry branch bypasses it. The narrow maintainable fix is to apply `minIntervalFloorAtMs` to the retry timestamp or intentionally narrow the public contract.
