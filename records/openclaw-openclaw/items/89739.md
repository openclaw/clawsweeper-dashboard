## Best Possible Solution

Land only after maintainers accept the new config contract and either clamp recurring transient retries to the floor with regression coverage or explicitly document and test that retries are exempt.

## Reproduction Assessment

Yes. From source inspection, configure `cron.minInterval` above `cron.retry.backoffMs` and trigger a retryable recurring error such as a rate limit; the retry branch stores `endedAt + backoffMs` and returns before the floor logic.

## Solution Assessment

No. The current patch is close, but the best fix is to make every automatic recurring re-arm honor one contract or deliberately narrow the public contract and tests around a retry exception.
