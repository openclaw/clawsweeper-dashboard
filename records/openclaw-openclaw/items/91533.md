## Summary

Keep open: the per-agent storage direction and scoped-clear sentinel are right, but PR head still leaves the Control UI bootstrap local-override read on the global key, so reload/startup can miss or misapply a scoped avatar; the supplied proof also does not cover that live Control UI/bootstrap path.

## What This Changes

The PR scopes Control UI assistant avatar localStorage by agent ID, threads the agent ID through render and assistant identity controller paths, stores scoped clears as explicit nulls, and adds controller regression tests.

## Best Possible Solution

Land one canonical fix that scopes every assistant-avatar localStorage caller, including bootstrap, preserves legacy fallback intentionally, and proves upload/switch/clear/reload in a real Control UI session.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.6 use one unscoped assistant-avatar localStorage key, and PR head source shows the bootstrap reload path still reads that unscoped key.

## Solution Assessment

No. Per-agent storage is the right layer, but this branch must also scope the bootstrap caller and prove the live Control UI reload path before it is the best fix.
