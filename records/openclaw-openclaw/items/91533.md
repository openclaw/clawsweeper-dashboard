## Summary

This PR is still useful, but it is not merge-ready: the per-agent storage direction is right, while the legacy global fallback can still make a cleared agent avatar reappear for upgraded browsers.

## What This Changes

The PR scopes Control UI assistant avatar localStorage keys by agent ID, threads the agent ID through render and assistant identity controller paths, and adds controller tests for scoped storage plus global fallback.

## Best Possible Solution

Land one canonical per-agent avatar storage fix that migrates or suppresses the legacy global fallback after scoped clears, with focused upgrade coverage and redacted live Control UI proof.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.6 read/write a single assistant-avatar localStorage key, and the render/controller callers use that single value across agents.

## Solution Assessment

No. Per-agent keys are the right owner boundary, but the fallback must be migration-aware or clear-aware so a scoped clear cannot resurrect the legacy global avatar.
