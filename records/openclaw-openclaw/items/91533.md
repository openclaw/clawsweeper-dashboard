## Summary

Keep open: the PR is aimed at the right Control UI storage surface, but it still has upgrade/clear semantics that can reapply a legacy global avatar and it does not provide real two-agent Control UI proof.

## What This Changes

The PR changes Control UI assistant avatar localStorage from one global key to agent-scoped keys, threads the active agent id through render/controller paths, and adds controller regression tests.

## Best Possible Solution

Land a narrow Control UI storage fix that scopes overrides per agent, handles the shipped global key deterministically, updates every visible reader, and includes real two-agent browser proof.

## Reproduction Assessment

## Solution Assessment

No. Per-agent keys are the right layer, but this branch is not the best fix until legacy fallback clears, remaining visible readers, and real two-agent browser proof are handled.
