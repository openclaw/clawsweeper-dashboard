## Summary

Keep open: the code shape looks like a valid fix for the linked Control UI avatar bug, but the contributor proof is still mock-only and several open branches are competing for the same bugfix.

## What This Changes

The PR scopes Control UI assistant avatar localStorage by assistant agent ID, preserves legacy global fallback, stores a scoped null on clear, and adds regression/proof tests.

## Best Possible Solution

Land one canonical per-agent assistant-avatar storage fix that preserves legacy fallback and includes live Control UI proof for upload, switch, clear, and reload/bootstrap.

## Reproduction Assessment

Yes, from source inspection: current main and `v2026.6.6` store every local assistant avatar override under the same global localStorage key, and active callers read that same key regardless of agent.

## Solution Assessment

Yes for code shape, with merge-readiness caveats. Scoping storage by the existing normalized assistant agent ID matches the local key-prefix pattern in `ui/src/ui/storage.ts`; the remaining work is proof and canonical-branch selection, not a different implementation layer.
