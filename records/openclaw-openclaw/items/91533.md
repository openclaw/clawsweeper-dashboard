## Summary

Keep open: the per-agent storage direction is right, but the branch still lets the shipped global avatar fallback reappear after a scoped clear and the supplied proof is mock-only rather than a live Control UI flow.

## What This Changes

This PR scopes Control UI assistant-avatar localStorage by optional agent ID, threads the active agent ID through render/controller calls, and adds controller regression tests.

## Best Possible Solution

Land a narrow per-agent avatar storage fix that handles the legacy global key deterministically after scoped clears, keeps visible readers scoped to the active agent, and includes redacted browser proof for a two-agent set/switch/clear/reload flow.

## Reproduction Assessment

Yes. Current main and v2026.6.6 read/write assistant avatar overrides through one localStorage key, so the two-agent overwrite path is source-reproducible from the storage helper and render/controller callers.

## Solution Assessment

No. Per-agent keys are the right layer, but this branch is not the best fix until legacy global fallback clear semantics and live two-agent Control UI proof are handled.
