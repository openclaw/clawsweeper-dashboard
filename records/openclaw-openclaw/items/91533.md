## Summary

Keep this PR open: the code shape is a plausible canonical fix for the per-agent assistant avatar bug, but it is not merge-ready because contributor proof is still mock-based, a changed proof test has a lint-blocking unbraced conditional, and maintainers still need to accept the browser-local upgrade semantics among duplicate branches.

## What This Changes

The PR scopes Control UI assistant-avatar localStorage reads and writes by assistant agent ID, keeps a legacy global fallback, stores scoped clears as explicit nulls, and adds regression/proof tests.

## Best Possible Solution

Land one canonical per-agent assistant-avatar storage fix after the lint defect is fixed, redacted browser proof covers the visible multi-agent flow, and maintainers accept the legacy fallback/scoped-clear semantics.

## Reproduction Assessment

Yes. Current main and v2026.6.6 store all local assistant avatar overrides under one unscoped localStorage key, and active callers read/write that same key while the linked issue reports cross-agent avatar overwrites.

## Solution Assessment

Yes for the code shape, but not yet for merge readiness. Scoping by the existing assistant agent ID and threading it through render, controller, and bootstrap is the narrow fix; the lint defect, browser proof, duplicate branch choice, and upgrade semantics still need resolution.
