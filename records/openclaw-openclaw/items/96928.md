## Summary

Keep open: the Browser/Ollama package-subpath regression is real, but this branch is not merge-ready because it promotes private SSRF/CDP bypass helpers into the public Plugin SDK and still lacks real built/runtime Browser lazy-load proof.

## What This Changes

## Best Possible Solution

Repair the packaged Browser/Ollama lazy-load path while keeping these SSRF/CDP helpers owner-scoped, or deliberately promote a documented public SDK/security contract with maintainer approval, replacement guardrails, and package smoke proof.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.10` import `openclaw/plugin-sdk/ssrf-runtime-internal` from Browser/Ollama while package metadata excludes that private artifact and does not export the subpath. I did not run a built Browser CLI reproduction in this read-only review.

## Solution Assessment
