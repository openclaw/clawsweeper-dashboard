## Summary

Keep open. Current main still lacks the requested `agents.defaults.bootstrapTier` schema/config/docs surface and tier-aware resolver, and open PR #22439 is already the direct implementation candidate with closing syntax. Recent discussion adds concrete wire-level evidence that bootstrap size can degrade tool dispatch, so this should stay with maintainer review rather than cleanup closure.

## What This Changes

Request to add configurable `agents.defaults.bootstrapTier` tiers (`minimal`, `standard`, `full`) for progressive workspace bootstrap file loading across main, subagent, cron, and extra-bootstrap-file flows.

## Best Possible Solution

Review and either land or deliberately replace PR #22439 with an additive, documented bootstrap-tier contract that preserves current defaults unless maintainers choose otherwise, resolves the effective tier from session type plus config override, wires tier selection through bootstrap filtering and hook-loaded extras, updates schema/types/help/docs, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source-inspection verification path: current main has no tier symbol, schema key, or resolver path that reads a bootstrap-tier config value. The latest discussion also supplies runtime evidence for the token-pressure problem this feature is meant to address.

## Solution Assessment

Unclear until maintainer review of PR #22439 and adjacent bootstrap-extra/default-context work. A config-driven tier surface is a plausible narrow solution, but the default behavior and `standard` versus `full` semantics need explicit product agreement.
