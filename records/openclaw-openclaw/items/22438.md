## Summary

Keep open: current main still lacks the requested `agents.defaults.bootstrapTier` config surface and resolver, and the issue is actively paired with open implementation work plus recent user evidence. Existing allowlists, lightweight mode, skip controls, and character budgets partially reduce prompt pressure, but they do not provide the declarative tiered loading contract requested here.

## What This Changes

The issue requests a configurable `agents.defaults.bootstrapTier` setting with `minimal`, `standard`, and `full` tiers for controlling bootstrap-file loading across main, subagent, and cron sessions.

## Best Possible Solution

Consolidate the open implementation work into one additive, documented bootstrap-tier contract that preserves current defaults, wires tier resolution through filtering and hook-loaded extras, updates schema/types/help/docs/generated metadata, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives a high-confidence verification path for the gap: current main has fixed allowlists, lightweight mode, skip controls, and character budgets, but no `bootstrapTier` setting or resolver.

## Solution Assessment

Unclear until maintainers settle the API semantics and overlapping PRs. The config-tier direction is plausible and belongs in core bootstrap prompt injection, but it should be reviewed against lightweight mode, extra bootstrap files, and prompt-size mitigation work.
