## Summary

Keep open. Current main still lacks the requested `agents.defaults.bootstrapTier` config/schema/docs surface and tier-aware bootstrap resolver, and open PR #22439 is already the direct implementation candidate with closing syntax; recent discussion also adds concrete runtime evidence that oversized bootstrap context can affect tool dispatch.

## What This Changes

Add an `agents.defaults.bootstrapTier` option with `minimal`, `standard`, and `full` tiers to control workspace bootstrap file loading for main, subagent, and cron sessions.

## Best Possible Solution

Review and either land or deliberately replace #22439 with an additive, documented bootstrap-tier contract that preserves current defaults unless maintainers choose otherwise, wires tier resolution through bootstrap filtering and hook-loaded extras, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source-inspection check: current main has no tier symbol, schema key, docs entry, or resolver path that reads a bootstrap-tier config value.

## Solution Assessment

Unclear until maintainer review of #22439. A config-driven tier surface is a plausible narrow solution, but defaults and `standard` versus `full` semantics need explicit product agreement.
