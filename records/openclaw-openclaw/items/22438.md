## Summary

Keep open: current main still has fixed session-based bootstrap filtering and no `agents.defaults.bootstrapTier` schema, resolver, docs, or hook-aware tier policy. The issue is also actively paired with open implementation work, especially https://github.com/openclaw/openclaw/pull/22439, so it should stay open until maintainers land, replace, or close that work.

## What This Changes

The issue requests configurable tiered bootstrap file loading for main, subagent, and cron sessions via an `agents.defaults.bootstrapTier` option.

## Best Possible Solution

Consolidate the open implementation work into one additive, documented bootstrap-tier contract that preserves current defaults, wires tier resolution through filtering and hook-loaded extras, updates schema/types/help/docs/generated metadata, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives a high-confidence verification path for the gap: current main has fixed allowlists, lightweight mode, skip controls, and character budgets, but no `bootstrapTier` setting or resolver.

## Solution Assessment

Unclear until maintainers settle the public API. A tiered config contract is plausible and belongs in core bootstrap prompt injection if accepted, but the broad implementation and the narrower minimal-tier PR need consolidation before this issue is resolved.
