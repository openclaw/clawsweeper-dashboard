## What This Changes

The issue requests configurable tiered workspace bootstrap loading so main, subagent, and cron sessions can choose minimal, standard, or full bootstrap context.

## Best Possible Solution

Consolidate the open implementation work into one additive, documented bootstrap-tier contract that preserves current defaults, updates schema/types/help/docs/generated metadata, and covers main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives a high-confidence verification path for the gap: current main has fixed allowlists, lightweight mode, skip controls, and character budgets, but no `bootstrapTier` setting or resolver.

## Solution Assessment

Unclear until maintainers settle the API semantics and overlapping PRs. The config-tier direction is plausible in the core bootstrap prompt-injection path, but it should be reconciled with lightweight mode, extra bootstrap files, and the related prompt-size bug reports.
