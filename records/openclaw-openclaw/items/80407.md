## Summary

## What This Changes

The branch adds a bundled policy plugin, doctor lint and health-check plumbing, oc-path document API exports, docs, tests, and channel deny-rule policy checks with optional repair and attestation/watch output.

## Best Possible Solution

Keep the policy design under maintainer review, make untrusted policy hashes terminal for policy-driven repair, align package metadata to the current host line, and then validate the chosen doctor-health versus standalone-lint direction across the stacked policy drafts.

## Solution Assessment

No, not as-is. The conformance direction may be viable, but policy repairs should not consume hash-mismatched artifacts, package metadata needs alignment, and the protected draft still needs maintainer architecture review.
