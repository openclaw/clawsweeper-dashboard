## Summary

Keep this PR open: it is a draft maintainer-labeled feature proposal, current main does not already contain the policy/health-check system, and the latest head still has blocking correctness and security issues in the proposed conformance path.

## What This Changes

The PR adds a bundled policy plugin with `policy check`, doctor lint/repair health-check plumbing, channel deny-rule conformance, an oc-path document API seam, docs, tests, and SDK/export metadata.

## Best Possible Solution

Have maintainers decide the policy/doctor-health architecture, then make policy fail closed on malformed artifacts and align targeted repair behavior with the documented safety model before merge.

## Reproduction Assessment

Yes for the review findings: source inspection of the latest PR head shows malformed policy diagnostics are discarded and `doctor --fix --only` is accepted but not honored. The broader PR is a new feature proposal, so there is no current-main bug reproduction path for the product request itself.

## Solution Assessment

No as-is: the proposed direction may be acceptable, but the implementation should fail closed on malformed policy files and either wire repair selectors through or reject them outside lint mode. The larger policy/doctor layering still needs maintainer judgment.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Report malformed policy files as failures:** `extensions/policy/src/doctor/register.ts:150-152`
  - body: When `policy.jsonc` has JSONC parse errors, this branch returns the existing empty finding set, so `policy check` can emit `ok: true` and doctor lint will not block even though the policy artifact could not be evaluated. Add an explicit parse/diagnostic finding and cover invalid policy input so conformance fails closed.
  - confidence: 0.93
- **[P2] Honor repair selectors or reject them:** `src/cli/program/register.maintenance.ts:31-32`

Either thread selected check IDs into structured repairs or fail fast when these selectors are used without `--lint`, so a targeted policy repair command cannot silently run unrelated repairs.
  - confidence: 0.86
