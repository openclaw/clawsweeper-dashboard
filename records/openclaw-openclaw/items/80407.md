## Summary

Keep open: this is a draft, maintainer-labeled feature proposal that is not implemented on current main, and the latest PR head still has blocking security/correctness issues in the proposed policy and doctor repair behavior.

## What This Changes

The branch adds a bundled policy extension, `policy check`, channel deny-rule evidence and attestations, doctor lint/repair health-check plumbing, an oc-path document API, docs, tests, and SDK/export metadata.

## Best Possible Solution

Have maintainers decide the policy/doctor-health architecture, then require malformed policies to fail closed and make repair selector behavior explicit before this stack can merge.

## Reproduction Assessment

Yes for the review findings: source inspection of the latest PR head shows malformed policy diagnostics are discarded and `doctor --fix --only` is accepted but not honored. The broader PR is a new feature proposal, so there is no current-main bug reproduction path for the product request itself.

## Solution Assessment

No as-is: the proposed direction may be acceptable, but the implementation should fail closed on malformed policy files and either wire repair selectors through or reject them outside lint mode. The larger policy/doctor layering still needs maintainer judgment.
