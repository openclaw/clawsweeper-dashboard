## Summary

Keep open: current main still has the reported Matrix MAS cross-signing failure path, and the same reporter has an open companion implementation PR, #74509, that should be reviewed or superseded before this issue is closed.

## What This Changes

The issue asks the Matrix plugin to handle MAS-fronted Synapse cross-signing bootstrap by honoring MSC3967 first-publish behavior, parsing UIA 401 challenges, preserving UIA sessions, and surfacing the MAS reset URL instead of suggesting password UIA.

## Best Possible Solution

Review, land, or replace #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, preserves UIA sessions, surfaces `org.matrix.cross_signing_reset` URLs, keeps MSC3967 first-publish uploads available, and updates focused tests plus operator docs/changelog.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading fallback error, and the issue supplies the concrete MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this in the Matrix plugin bootstrap path through UIA parsing, session preservation, and MAS capability/reset URL handling is the narrow maintainable direction; #74509 is the active implementation candidate.
