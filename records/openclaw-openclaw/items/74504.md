Keep open: current main still has the reported Matrix MAS cross-signing bootstrap gap, and #74509 is already the same-reporter open implementation candidate.

## What This Changes

The issue requests MAS-aware Matrix cross-signing bootstrap for PAT-authenticated bots, including UIA challenge parsing, MAS reset URL surfacing, UIA session propagation, and MSC3967 first-publish handling.

## Best Possible Solution

Review, land, or supersede #74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves UIA sessions, surfaces MAS reset URLs, keeps the MSC3967 upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction: exercise the current auth callback with no configured password while null and dummy requests reject, and it throws the old password-only UIA message. I did not live-test against a MAS-fronted Synapse in this read-only pass.

## Solution Assessment

Yes at the design level, but it is not implemented on current main. The narrow maintainable solution is Matrix bootstrap-owned UIA challenge parsing/session propagation plus typed MAS reset URL errors, with #74509 as the active candidate to review, land, or replace.
