## Summary

Keep open. Current main still has the reported Matrix UIA fallback path and lacks MAS-specific cross-signing reset handling, while #74509 remains the open companion implementation candidate from the same reporter.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse by parsing UIA challenges, preserving UIA sessions, surfacing MAS reset URLs, and keeping the MSC3967 first-publish path usable.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, preserves UIA sessions, returns a typed MAS reset URL for operator approval, keeps first-publish and post-approval upload paths available, and updates focused tests, Matrix docs, and the changelog.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current cross-signing auth callback with no configured password while the no-auth and dummy requests fail; current tests assert the same misleading password-fallback error, and the issue supplies the concrete MAS 401 challenge shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this in the Matrix plugin bootstrap path with UIA parsing and a homeserver capability probe is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
