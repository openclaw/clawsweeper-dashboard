## What This Changes

The issue requests Matrix plugin support for MAS-fronted Synapse cross-signing bootstrap by parsing UIA 401 challenges, surfacing `org.matrix.cross_signing_reset` recovery URLs, threading UIA sessions, and preserving MSC3967 first-publish behavior.

## Best Possible Solution

Review and land or supersede #74509 with a Matrix plugin-owned fix that parses Matrix UIA 401 bodies, threads the UIA session through supported stages, surfaces a typed MAS reset URL for human approval, keeps MSC3967 first-publish uploads available, updates Matrix operator docs, and preserves credential handling semantics noted in the PR review.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading password fallback, and the issue supplies the concrete MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. The narrow maintainable path is in the Matrix plugin cross-signing bootstrap and SDK capability layer; #74509 is the active candidate to review, adjust, land, or replace.
