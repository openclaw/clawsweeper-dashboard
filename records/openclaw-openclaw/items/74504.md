## Summary

Keep open: current main and the latest stable release still use the old Matrix cross-signing UIA fallback that ignores MAS stages, UIA sessions, and reset URLs. The same reporter's open implementation candidate at https://github.com/openclaw/openclaw/pull/74509 is the right place to resolve it before this issue closes.

## What This Changes

The issue asks Matrix cross-signing bootstrap to handle MAS-fronted Synapse by honoring MSC3967 first-publish, surfacing MAS reset URLs, and avoiding the misleading password fallback.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves UIA sessions, surfaces MAS reset URLs, keeps the MSC3967 upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes, at source level: current main's `authUploadDeviceSigningKeys` callback can be exercised with no configured password while null and dummy requests reject. That path throws the reported password fallback error and never inspects the MAS UIA body, reset URL, or session.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: keep the Matrix plugin responsible for UIA parsing/session propagation and typed MAS reset URL errors. It should be delivered through the open implementation PR or a direct successor, not by starting a parallel repair lane.
