## Summary

Keep open: current main still has the old Matrix cross-signing UIA fallback path with no MAS reset-stage handling, and #74509 is an open same-author implementation candidate for this report.

## What This Changes

The issue asks the Matrix plugin to handle MAS-fronted Synapse cross-signing bootstrap by parsing UIA challenges, preserving sessions, surfacing reset URLs, and keeping the MSC3967 first-publish path working.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA challenges, propagates sessions, surfaces MAS reset URLs, preserves the first-publish upload path, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; the current test asserts the same misleading fallback error, and the issue supplies the MAS 401 body shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this inside the Matrix bootstrap path with typed MAS/UIA errors, session propagation, and capability probing is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
