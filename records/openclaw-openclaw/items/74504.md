## Summary

Keep open: current main and the latest release still have the old Matrix cross-signing UIA path, and the matching fix is still an open PR rather than merged behavior.

## What This Changes

The issue reports that Matrix cross-signing bootstrap fails on MAS-fronted Synapse homeservers for PAT-authenticated bot accounts, leaving cross-signing unpublished and E2EE recovery/decryption broken.

## Best Possible Solution

Review and land a repaired version of https://github.com/openclaw/openclaw/pull/74509, or close that PR with a narrower Matrix-plugin fix that handles MAS UIA stages and preserves existing credential behavior.

## Reproduction Assessment

Yes at source level: current main's Matrix UIA callback can be driven with a 401 UIA body advertising MAS reset stages and no password, and it will fall into the old password-fallback path without reading stages, params, reset URL, or session. I did not run a live MAS homeserver scenario in this read-only review.

## Solution Assessment

Yes directionally: fixing the Matrix plugin bootstrapper/UIA callback is the right owner boundary for this existing Matrix channel bug. The current open PR is the active path, but it needs review/proof repairs before this issue can close.
