## Summary

Keep open. Current main still matches the reported failure path, and the same author has an open implementation candidate in #74509, so this should not be closed before that PR lands or is superseded.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse by handling MSC3967 first-publish, MAS reset UIA challenges, UIA sessions, and clearer operator-facing errors.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, preserves UIA sessions, reports a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading fallback error, and the issue supplies the concrete MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this inside the Matrix plugin cross-signing bootstrap path plus a homeserver capability probe is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
