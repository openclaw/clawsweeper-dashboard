## Summary

Keep open. Current main still matches the reported Matrix UIA failure path, and the issue is paired with open implementation PR #74509, so closing before that PR lands or is superseded would be premature.

## What This Changes

The issue requests a Matrix plugin fix so cross-signing bootstrap on MAS-fronted Synapse handles MSC3967 first-publish, parses UIA challenges, preserves UIA sessions, and surfaces MAS reset URLs instead of recommending password UIA.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 challenges, threads the UIA session through supported stages, surfaces a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists by invoking the current cross-signing upload auth callback with no configured password while the no-auth and dummy UIA attempts reject; the existing test asserts the same misleading password fallback error, and the issue supplies a concrete MAS 401 shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this in the Matrix plugin's cross-signing auth callback and homeserver-capability path is the narrowest maintainable solution; #74509 is the active candidate to review, land, or replace.
