## Summary

Keep open: current main still lacks Matrix MAS cross-signing reset handling, MSC3967-aware UIA parsing, and operator-facing MAS reset URL propagation, while #74509 remains the same-author open implementation candidate for this issue.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse PAT accounts, including MSC3967 first-publish behavior, UIA challenge parsing, UIA session propagation, and clear MAS reset URL errors.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves UIA sessions, surfaces the MAS reset URL, keeps the first-publish upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to exercise the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests still assert the same misleading fallback error, and the issue supplies the MAS 401 body shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this inside the Matrix bootstrap path with UIA parsing, session propagation, MAS capability detection, and typed operator-facing errors is the narrow maintainable solution; #74509 is the active candidate to review, land, or replace.
