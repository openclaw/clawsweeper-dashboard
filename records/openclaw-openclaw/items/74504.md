## Summary

Keep open: current main and the latest release still use the old Matrix cross-signing UIA fallback that ignores MAS reset stages, UIA sessions, and reset URLs. The active paired implementation candidate at https://github.com/openclaw/openclaw/pull/74509 is the right path, but this issue should stay open until that PR lands or is superseded.

## What This Changes

The report asks Matrix cross-signing bootstrap to handle MAS-fronted Synapse for PAT-authenticated bots by honoring MSC3967 first-publish and surfacing MAS reset URLs instead of the password fallback.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves sessions, surfaces MAS m.oauth/reset URLs, keeps MSC3967 first upload available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes, at source level: current main's authUploadDeviceSigningKeys callback can be exercised with null and dummy failures plus no password, and it throws the reported fallback error without inspecting UIA flows, session, or reset URL. I did not run a live MAS-fronted Synapse environment in this read-only pass.

## Solution Assessment

Yes in direction: a stage-aware Matrix plugin fix is the narrow maintainable solution. It should land through the active PR or a superseding PR rather than closing this issue first.
