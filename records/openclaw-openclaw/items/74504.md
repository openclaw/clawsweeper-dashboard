## Summary

Keep open: current main still has the reported Matrix MAS cross-signing failure path, and #74509 is an open same-reporter implementation candidate that should be reviewed, landed, or superseded before this issue closes.

## What This Changes

Fix Matrix cross-signing bootstrap for MAS-fronted Synapse PAT bots by honoring MSC3967 first-publish, parsing UIA reset challenges, surfacing the MAS reset URL/session, and avoiding the misleading password fallback.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, preserves UIA sessions, returns a typed MAS reset URL for operator approval, keeps the MSC3967 first-publish upload path available, and updates focused tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading fallback error, and the issue supplies the concrete MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this inside the Matrix plugin bootstrap path with UIA parsing, session preservation, and MAS capability recovery is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
