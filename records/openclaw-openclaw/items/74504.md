## Summary

Keep open. Current main still follows the reported Matrix cross-signing UIA fallback path, lacks MAS `org.matrix.cross_signing_reset` handling or a homeserver capability probe, and the same author has an open implementation PR (#74509) for this exact Matrix plugin-owned fix.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse/PAT bots by honoring MSC3967 first-publish behavior, parsing UIA 401 challenges, threading UIA sessions, and surfacing MAS reset URLs.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses Matrix UIA 401 bodies, preserves and passes UIA sessions, reports a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests, docs, and changelog.

Yes at the boundary level, with PR review still needed for the exact implementation. The narrow maintainable fix belongs in the Matrix plugin cross-signing bootstrap/UIA path, plus a Matrix homeserver capability probe and operator documentation.
