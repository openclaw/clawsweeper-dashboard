## Summary

Keep open. Current main still has the reported Matrix UIA fallback behavior, does not parse MAS `org.matrix.cross_signing_reset` challenges or expose a MAS reset URL, and the same author has an open implementation candidate in #74509 that should be reviewed, landed, or superseded before this issue closes.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse/PAT bot accounts by parsing UIA challenges, preserving UIA sessions, surfacing MAS reset URLs, and keeping the MSC3967 first-publish path available.

## Best Possible Solution

Review, land, or replace #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, preserves UIA sessions, reports a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading fallback error, and the issue supplies the concrete MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this inside the Matrix plugin cross-signing bootstrap path with a homeserver capability probe is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
