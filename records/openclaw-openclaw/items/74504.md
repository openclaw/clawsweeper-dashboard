## Summary

Keep open: current main still follows the old Matrix cross-signing UIA fallback path and lacks MAS reset/MSC3967-specific handling, while #74509 remains an open same-author implementation candidate for this bug.

## What This Changes

The issue requests a Matrix plugin fix for MAS-fronted Synapse cross-signing bootstrap so MSC3967 first-publish succeeds and MAS reset challenges surface an actionable URL instead of the misleading password fallback.

## Best Possible Solution

Review, harden, and land or supersede #74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves sessions, surfaces the MAS reset URL, keeps the first-publish upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current authUploadDeviceSigningKeys callback with no configured password while null and dummy auth requests reject; the current test covers that path and the issue supplies the MAS 401 body shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this in the Matrix bootstrap path with UIA parsing, session propagation, MAS capability detection, and typed operator-facing errors is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
