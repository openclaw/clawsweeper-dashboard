## Summary

Keep open: current main still follows the old Matrix cross-signing UIA fallback path and lacks MAS reset/MSC3967-specific handling, while #74509 remains an open same-author implementation candidate for this bug.

## What This Changes

Handle MAS-fronted Synapse cross-signing bootstrap for PAT-authenticated Matrix bots by honoring MSC3967 first-publish and surfacing the MAS reset URL instead of the password fallback.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves sessions, surfaces the MAS reset URL, keeps the first-publish upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; the current test asserts the same misleading fallback error, and the issue supplies the MAS 401 body shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this in the Matrix bootstrap path with UIA parsing, session propagation, MAS capability detection, and typed operator-facing errors is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
