## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse, including MSC3967 first-publish behavior, UIA challenge parsing, UIA session propagation, and operator-facing MAS reset URL errors.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves UIA sessions, surfaces MAS reset URLs, keeps the MSC3967 upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction: exercise the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject. Current tests still assert the same misleading password fallback error instead of MAS stage handling.
