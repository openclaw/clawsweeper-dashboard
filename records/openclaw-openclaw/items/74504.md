## Summary

Keep open. Current main still matches the reported Matrix MAS cross-signing failure path, and #74509 is an open companion implementation candidate from the same reporter, so this issue should remain open until that PR lands or is superseded.

## What This Changes

The issue requests a Matrix plugin fix for MAS-fronted Synapse cross-signing bootstrap so PAT-authenticated bots can use the MSC3967 first-publish path, preserve UIA sessions, and surface the MAS cross-signing reset URL instead of a password fallback error.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, preserves UIA sessions, returns a typed MAS reset URL for operator approval, keeps the MSC3967 first-publish upload path available, and updates focused tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading fallback error, and the issue supplies the concrete MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this inside the Matrix plugin bootstrap path, with UIA parsing and a homeserver capability probe, is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
