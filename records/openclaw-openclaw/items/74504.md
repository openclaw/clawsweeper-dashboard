## Summary

Current main still has the reported Matrix cross-signing UIA fallback: it retries no-auth, dummy, and password without parsing MAS UIA challenges or preserving the UIA session. The linked implementation PR #74509 remains the active path, so this issue should stay open until that PR lands or is superseded.

## What This Changes

The issue requests Matrix plugin support for MAS-fronted Synapse cross-signing bootstrap by honoring MSC3967 first-publish behavior, surfacing MAS cross-signing reset URLs, preserving UIA sessions, and replacing the misleading password fallback.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, preserves UIA sessions, reports a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading password fallback, and the issue supplies the MAS 401 response body shape.

## Solution Assessment

Yes at the design level, but not yet on main. Handling this in the Matrix plugin's cross-signing UIA callback and homeserver-capability plumbing is the narrowest maintainable solution; #74509 is the active candidate to review, land, or replace.
