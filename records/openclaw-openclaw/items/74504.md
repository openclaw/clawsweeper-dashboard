## Summary

Keep open: current main still lacks Matrix MAS cross-signing reset handling, MSC3967-aware UIA parsing, UIA session propagation, and operator-facing MAS reset URL output; #74509 is the paired open implementation candidate from the same reporter.

## What This Changes

The issue asks the Matrix plugin to support MAS-fronted Synapse cross-signing bootstrap for PAT-authenticated bots by parsing UIA challenges, preserving UIA sessions, and surfacing the MAS reset URL.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 challenges, preserves UIA sessions, surfaces MAS reset URLs, keeps the MSC3967 first-publish upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to exercise the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests still assert the reported fallback error, and the issue supplies the MAS 401 body shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this inside the Matrix bootstrap path with UIA parsing, session propagation, MAS capability detection, and typed operator-facing errors is the narrow maintainable direction; #74509 is the active implementation candidate to review, land, or replace.
