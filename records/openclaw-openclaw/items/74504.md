## Summary

Keep open: current `main` and the latest release still use the password-only Matrix UIA fallback that cannot inspect a MAS `org.matrix.cross_signing_reset` challenge, and the related fix PR is still open rather than merged.

## What This Changes

Matrix MAS/PAT cross-signing bootstrap should handle UIA reset stages and first-publish behavior so encrypted Matrix bots can publish keys and decrypt messages.

## Best Possible Solution

Land a proven Matrix-plugin fix that parses UIA 401 flows/session, surfaces the MAS reset URL, and preserves first-publish behavior, either through the existing open PR or a narrower replacement if that branch cannot be repaired.

## Reproduction Assessment

Yes from source inspection, not from a live MAS deployment: current main retries no-auth, dummy, then password without inspecting the UIA response body or session.

## Solution Assessment

Yes on owner boundary, pending implementation quality: Matrix-plugin bootstrap code is the right place to handle Matrix UIA stages, but the issue should stay open while the related PR is unmerged.
