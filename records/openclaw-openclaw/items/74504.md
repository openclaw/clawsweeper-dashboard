## Summary

Keep open. Current main still matches the reported Matrix MAS cross-signing failure path, and #74509 is an open implementation candidate from the same contributor, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw’s Matrix plugin to handle MAS-fronted Synapse cross-signing bootstrap by honoring MSC3967 first-publish behavior, preserving UIA sessions, and surfacing `org.matrix.cross_signing_reset` recovery URLs instead of a password fallback error.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 bodies, carries UIA sessions, reports a typed MAS reset URL for operator approval, preserves the MSC3967 no-auth first-publish upload path, and updates focused tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to invoke the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject; current tests assert the same misleading fallback error, and the issue supplies the concrete MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main. Handling this in the Matrix plugin bootstrap path, with a homeserver capability probe only where needed, is the narrow maintainable direction; #74509 is the active candidate to review, land, or replace.
