## Summary

Keep open: current main still has the reported Matrix MAS cross-signing bootstrap gap, and #74509 is already the same-reporter implementation candidate for this issue.

## What This Changes

Support Matrix MAS-fronted Synapse cross-signing bootstrap for PAT-authenticated bots by preserving first-publish behavior and surfacing reset URLs instead of password fallback errors.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses UIA challenges, threads sessions through auth submissions, surfaces MAS reset URLs, keeps the first-publish upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction: exercise the current `authUploadDeviceSigningKeys` callback with no configured password while no-auth and dummy requests reject. Current tests still assert the same misleading password fallback error instead of MAS stage handling.

## Solution Assessment

Yes at the design level, but it is not implemented on main. The narrow maintainable path is Matrix bootstrap-owned UIA parsing/session propagation plus typed MAS reset URL errors, with #74509 as the active candidate to review, land, or replace.
