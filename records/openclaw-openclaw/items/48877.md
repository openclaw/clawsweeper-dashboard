## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Recheck command auth before forwarding menu callbacks:** `extensions/telegram/src/bot-handlers.runtime.ts:1531-1532`
  - body: When a menu button belongs to a custom command but is not listed in `routes`, this code skips the menu block and falls through to generic callback-to-agent handling after only callback-scope authorization. A restricted command can be opened by an authorized user in a group and then driven by another callback-authorized room member, so recheck the owning custom command's authorization before forwarding fallback payloads.
  - confidence: 0.88
- **[P2] Reject invalid menu callback payloads:** `src/config/zod-schema.providers-core.ts:130-131`
  - body: The schema accepts any string for `callback_data`, while the new send/edit paths pass `menu.buttons` directly to Telegram. Existing inline-button paths sanitize callback data before sending, so blank or over-64-byte configured payloads should fail config validation instead of making a menu command fail at runtime.
  - confidence: 0.83
