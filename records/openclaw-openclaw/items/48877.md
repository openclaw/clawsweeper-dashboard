## Summary

Keep this PR open. Current main still lacks declarative Telegram custom-command menus, so the contribution addresses a real missing feature, but the proposed patch is not merge-ready because it is stale against the current Telegram SDK/config layout and still accepts invalid callback payloads that can fail at runtime.

## What This Changes

The PR adds Telegram `customCommands` `menus`/`routes` config, menu-backed slash-command handlers, callback-query menu routing, typed config/schema fields, and a helper for filtering validated custom commands.

## Best Possible Solution

Land a refreshed Telegram plugin implementation that keeps the feature owned by the Telegram/config surfaces, adds typed `menus` and `routes`, validates button text and callback payload constraints before Telegram API calls, preserves existing command authorization and callback scoping, and updates generated config metadata, Telegram docs, tests, and changelog.

## Reproduction Assessment

Yes. Current main can be verified by source/docs inspection: the strict schema rejects `menus`/`routes`, the runtime has no custom menu command/router, and the PR’s helper import is not exported by the current SDK surface.

## Solution Assessment

No, not as currently proposed. The feature direction is appropriate, but the branch should be refreshed onto the current `src/plugin-sdk/telegram-command-config.ts` and `bot-core.ts` layout and should validate callback payloads before merge.
