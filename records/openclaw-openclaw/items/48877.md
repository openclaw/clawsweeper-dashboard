  - body: Current `main` has `extensions/telegram/src/bot.ts` as a thin wrapper and registers native commands/handlers inside `createTelegramBotCore`. - **[P1] Recheck custom-command auth before fallback:** `extensions/telegram/src/bot-handlers.runtime.ts:1502`
  - body: When a menu button is not in `routes`, the patch lets it fall through to the synthetic-message callback path after only generic callback authorization. In groups, that does not reapply the menu command's allowlist/`commands.allowFrom` authorization, so another group member can click an already-posted restricted menu button and drive an agent turn.
  - confidence: 0.86
- **[P2] Validate callback payloads before sending menus:** `src/config/zod-schema.providers-core.ts:130`
  - body: `callback_data: z.string()` accepts empty and over-64-byte values, but the new menu sender/editor passes those buttons straight to Telegram. A malformed configured menu can therefore turn `/command` or submenu navigation into a Telegram API failure instead of a config error.
  - confidence: 0.88
