## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Recheck command auth before forwarding menu callbacks:** `extensions/telegram/src/bot-handlers.runtime.ts:1535-1536`
  - body: When a configured menu button is not in `routes`, this branch falls through to the generic callback-to-agent path after only callback-scope authorization. In shared groups, a command that is restricted by command auth can therefore be opened by an authorized user and then driven by another callback-authorized room member, so recheck the owning custom command's auth before forwarding menu-owned fallback payloads.
  - confidence: 0.88
- **[P2] Reject invalid menu callback payloads:** `src/config/zod-schema.providers-core.ts:130`
  - body: The schema accepts any string for `callback_data`, but the new command and edit paths pass `menu.buttons` directly to Telegram. Existing inline-button code sanitizes callback data before sending, so reject blank or overlong configured payloads here instead of letting one bad menu button fail at runtime.
  - confidence: 0.83
