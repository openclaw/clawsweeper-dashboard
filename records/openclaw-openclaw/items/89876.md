## Summary

Keep open: current main and v2026.6.8 have Telegram inline buttons, plugin callback dispatch, and managed select primitives, but they do not implement the requested multi-question decision-card runtime, draft/refinement state, or structured packet delivered to the agent.

## What This Changes

Request a Telegram-native multi-question decision packet card flow that lets a user revise answers/refinements and submits one structured input to the agent.

## Best Possible Solution

Define a maintainer-approved packet contract first, then implement the renderer, draft state, callback tokens, refinement handling, and targeted Telegram tests in the Telegram plugin, adding a shared SDK seam only if the agent/plugin contract needs to be portable.

## Reproduction Assessment

Not applicable: this is a feature/UX request rather than a broken existing behavior report. Source inspection shows the requested decision-packet flow is absent while related Telegram primitives are present.

## Solution Assessment

Unclear until maintainers choose the permanent contract. A Telegram-local first slice is plausible, but the best implementation depends on how agents request and receive the structured packet and whether a shared SDK seam is needed.
