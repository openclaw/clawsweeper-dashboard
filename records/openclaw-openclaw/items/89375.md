Keep open: current main still has the reported gap between listener-scoped stale gateway PID cleanup and conservative live OpenClaw-owned session-lock handling, and no merged related PR or canonical issue fully owns this restart-specific failure.

## What This Changes

The issue asks OpenClaw to handle gateway restarts where an old orphaned gateway process is missed and continues holding a session write lock.

## Best Possible Solution

Keep this open until maintainers define and prove a restart/session-lock policy that detects old same-instance gateway processes beyond listener ownership or safely invalidates locks from old gateway lifecycles.

## Reproduction Assessment

Yes at source level, but not as a live macOS reproduction in this review. Current stale-process cleanup is port-listener based, while current session-lock cleanup preserves live OpenClaw-owned stale locks, matching the reported unresolved failure shape.

## Solution Assessment

No, current main is not a complete solution for this issue. A safer fix needs maintainer-approved lifecycle policy for old gateway processes and their locks, plus macOS/service restart proof.
