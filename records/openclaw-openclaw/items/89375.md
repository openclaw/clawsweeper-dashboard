## Summary

Keep open: current main and the latest release still clean stale gateways through port/listener-scoped process discovery and preserve live OpenClaw-owned stale session locks, so the reported orphaned gateway process holding a session write lock is not fixed or superseded.

## What This Changes

The issue asks OpenClaw to detect and retire all stale same-instance gateway processes during restart so orphaned processes cannot keep session write locks and block later requests.

## Best Possible Solution

Add a maintainer-approved restart reliability path that detects same-instance orphaned gateway lock holders beyond port listeners, reclaims or terminates only proven-stale owners, and proves it with live macOS/service restart coverage.

## Reproduction Assessment

No high-confidence live current-main reproduction was established in this read-only review. The issue logs are concrete and current source makes the failure plausible because restart cleanup is listener-scoped while live OpenClaw lock owners are preserved.

## Solution Assessment

Unclear: the requested detection and verification direction is plausible, but the best fix needs a maintainer-chosen owner boundary for safely retiring same-instance orphaned gateways without stealing legitimate transcript writes.
