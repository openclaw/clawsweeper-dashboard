## Summary

Keep open: current main still has listener-scoped stale gateway PID cleanup and conservative live OpenClaw-owned session-lock handling, so the restart-specific stale-process/session-lock policy is not fully solved by current code or the overlapping related issues.

## What This Changes

The issue reports that SIGUSR1 gateway restart can miss stale or orphaned gateway processes, letting an old process hold session JSONL write locks and cause `SessionWriteLockTimeoutError`.

## Best Possible Solution

Define and prove one restart lifecycle policy that either detects old same-instance gateway processes beyond listener ownership or safely invalidates locks from old gateway lifecycles, with focused tests and macOS/service proof.

## Reproduction Assessment

Yes at source level, but not as a live current-main macOS restart reproduction in this review. The source still combines listener-scoped stale-PID cleanup with preservation of live OpenClaw-owned stale locks, matching the reported failure shape.

## Solution Assessment

No, current main is not a complete solution. The maintainable fix needs a restart/session-lock ownership policy that avoids stealing legitimate writes while handling old gateway lifecycles.
