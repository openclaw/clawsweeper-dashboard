## Summary

Keep open. The current main branch still runs the embedded agent preparation, tool setup, skill prompt resolution, provider prompt transform, and system-prompt report construction in the Gateway process, while the issue now has corroborating event-loop starvation data from multiple environments. Recent unreleased runtime-deps fixes likely reduce one related hot path, but they do not prove that the agent-prep starvation reported here is fixed.

## What This Changes

The issue requests fixing Gateway API hangs and Node event-loop starvation during agent preparation in OpenClaw 2026.4.29.

## Best Possible Solution

Treat this as a Gateway performance incident: profile packaged v2026.4.29 and current main during an agent turn, then make agent preparation non-starving by caching, yielding, bounding, or isolating the prompt/skill/tool/runtime setup so Gateway RPCs and active streams remain responsive.
