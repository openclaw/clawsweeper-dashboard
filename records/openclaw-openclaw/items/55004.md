## Summary

Keep open. The local dashboard preflight bug is still present on current main and this PR is the canonical fix attempt, but the branch is conflicting and would regress current token-redaction and TLS dashboard-link contracts.

## What This Changes

The branch adds a local-mode dashboard reachability preflight, daemon/config remediation hints, a loaded daemon-status field, and dashboard command tests.

## Best Possible Solution

Rework or replace the preflight on current main so local auto-open side effects are gated while preserving token redaction, SecretRef behavior, TLS-aware links, --no-open behavior, and custom-bind behavior.

## Reproduction Assessment

Yes. Current main still reaches dashboard copy/open side effects before any reachability gate, and the linked issue provides concrete fresh-install logs for the unreachable local gateway path.

## Solution Assessment

No. The preflight direction is appropriate, but this branch is not the best implementation until it is rebased or replaced without regressing token redaction and TLS-aware link contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep bearer tokens out of dashboard logs:** `src/commands/dashboard.ts:149`
  - body: This logs dashboardUrl, which includes #token=... when token auto-auth is enabled. Current main logs only the clean base URL and has regression coverage for that, so this would re-expose gateway auth tokens in runtime logs.
  - confidence: 0.91
- **[P1] Stop passing tokens into SSH hints:** `src/commands/dashboard.ts:187-190`
  - body: When browser open fails, this passes the bearer token into formatControlUiSshHint and then logs the returned hint. Current main deliberately calls that helper without a token, so this would leak the token on the SSH fallback path.
  - confidence: 0.89
- **[P2] Preserve TLS-aware dashboard links:** `src/commands/dashboard.ts:136-141`
  - body: This call omits tlsEnabled, so gateway.tls.enabled would still produce plaintext dashboard links. Current main passes TLS state and the dashboard docs require https/wss in that mode.
  - confidence: 0.87
