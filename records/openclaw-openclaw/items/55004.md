## Summary

Keep this PR open: the reported dashboard reachability bug still exists on current main, and the PR has useful source work, but the branch is stale against current dashboard token-redaction and TLS behavior and would regress both if merged as-is.

## What This Changes

The PR adds a local-mode dashboard reachability preflight before clipboard/browser side effects, prints config and daemon remediation when unreachable, adds dashboard command tests, and exposes daemon loaded state in status summaries.

## Best Possible Solution

Reapply the local dashboard reachability preflight on current main while preserving token redaction, SecretRef handling, TLS-aware link resolution, --no-open behavior, and custom/tailnet bind semantics, then land it with focused tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. The original bug has a high-confidence reproduction path from the issue logs and current main source: with no reachable local gateway, dashboardCommand still reaches clipboard/browser side effects without probing first. The PR regressions are also source-reproducible from the PR head because it logs tokenized dashboardUrl, passes token into the logged SSH hint helper, and omits tlsEnabled.

## Solution Assessment

No. The intended preflight is the right direction, but this branch is not the best way to land it because it is stale against current token-redaction and TLS changes; a narrow reapply on current main is safer.

## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Keep bearer tokens out of dashboard logs:** `src/commands/dashboard.ts:149`
  - body: This branch logs `dashboardUrl`, which includes `#token=...` when token auto-auth is enabled. Current main logs only `links.httpUrl` and has a regression test that logged dashboard lines never contain the bearer token or `#token=`, so merging this branch would re-expose dashboard auth tokens.
  - confidence: 0.91
- **[P1] Stop passing tokens into SSH hints:** `src/commands/dashboard.ts:187-190`
  - body: When browser open fails, this branch passes the bearer token into `formatControlUiSshHint`; that helper includes an authenticated URL in the returned string, and `dashboardCommand` logs the hint. Current main deliberately calls the helper without a token and tests the SSH path for token redaction.
  - confidence: 0.89
- **[P2] Preserve TLS-aware dashboard links:** `src/commands/dashboard.ts:136-141`
  - body: This branch calls `resolveControlUiLinks` without `tlsEnabled`, so `gateway.tls.enabled: true` would still produce `http://` and `ws://` dashboard links and probe the wrong scheme. Current main passes TLS state, and docs/tests require `https://` and `wss://`.
  - confidence: 0.87
