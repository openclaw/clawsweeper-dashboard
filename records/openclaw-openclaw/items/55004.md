- **[P1] Keep bearer tokens out of dashboard logs:** `src/commands/dashboard.ts:149`
  - body: This branch logs `dashboardUrl`, which contains `#token=...` when token auto-auth is enabled. Current main logs only `links.httpUrl` and has regression coverage rejecting tokens in logged lines, so merging this would re-expose gateway auth tokens through runtime logs.
  - confidence: 0.91
- **[P1] Stop passing tokens into SSH hints:** `src/commands/dashboard.ts:187-190`
  - body: When browser open fails, this branch passes the bearer token into `formatControlUiSshHint`; that helper includes an authenticated URL in the returned hint, and `dashboardCommand` logs it. Current main deliberately calls the helper without a token and tests the SSH path for redaction.
  - confidence: 0.89
- **[P2] Preserve TLS-aware dashboard links:** `src/commands/dashboard.ts:136-141`
  - body: This branch calls `resolveControlUiLinks` without `tlsEnabled`, so `gateway.tls.enabled: true` would still produce plaintext dashboard links. Current main passes TLS state, and the dashboard docs require `https://` and `wss://` in that mode.
  - confidence: 0.87
