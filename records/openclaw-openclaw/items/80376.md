## Summary

Keep open. The patch fixes part of the resumed CLI arg assembly, but the default Anthropic backend uses the Claude stdio live-session path, and that path still strips `--append-system-prompt-file` and `--append-system-prompt` whenever `useResume` is true; the supplied proof is also unit-harness output rather than a real resumed Claude CLI run.

## What This Changes

The PR removes resume guards from CLI system-prompt resolution/arg building and adds helper tests for resumed Claude CLI prompt flags.

## Best Possible Solution

Update the CLI runner and Claude live-session argument builder so resumed default Claude CLI invocations keep the system prompt file/arg, then prove it with final spawned argv coverage and a redacted real resume run.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main suppresses system-prompt resolution and prompt-file argv on resume, and the linked issue shows the fresh-vs-resumed Claude argv difference. I did not execute tests because this review is read-only.

## Solution Assessment

No. The helper-level change is necessary but incomplete because the default Claude CLI live-session path strips the same prompt flags on resume before spawning Claude.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep prompt flags through Claude live resume:** `src/agents/cli-runner/execute.ts:281-285`
  - body: The new resume prompt work stops at `buildCliArgs`, but default `claude-cli` runs go through `runClaudeLiveSessionTurn` because the Anthropic backend is `liveSession: "claude-stdio"`. That path calls `buildClaudeLiveArgs`, which still strips both `systemPromptArg` and `systemPromptFileArg` whenever `useResume` is true, and the adjacent spawn test still asserts the resumed live argv lacks those flags. Please carry the prompt file/arg through the live-session resume path and add final argv coverage, not only helper-level `buildCliArgs` coverage.
