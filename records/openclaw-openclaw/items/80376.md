## Summary

This PR should stay open because it only repairs the helper/one-shot argv path; default Claude CLI resumes use the live-session path, which still strips prompt flags on resume. The supplied proof is unit-harness output rather than a real resumed Claude CLI run.

## What This Changes

The branch changes `executePreparedCliRun` and `buildCliArgs` to write and pass system prompt flags on resumed sessions, and adds helper tests for resumed prompt injection.

## Best Possible Solution

Update both the CLI runner and Claude live-session argv builder so resumed default Claude CLI processes receive the generated harness prompt file or arg, then validate with final argv coverage and real resume proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main suppresses system-prompt resolution/file creation on resume, and the default Claude live-session builder strips prompt flags when `useResume` is true. I did not run a live Claude CLI resume in this read-only review.

## Solution Assessment

No. The helper and `execute.ts` changes are necessary, but they are incomplete because the default `claude-cli` live-session resume path still removes the same prompt flags before spawning Claude.
