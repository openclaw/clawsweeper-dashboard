## Summary

Keep open: the PR targets a real Claude CLI resume regression, but the already-noted live-session argv gap remains and the supplied proof is still helper-unit output rather than a real Claude resume run.

## What This Changes

The PR removes resume guards from CLI runner system-prompt resolution and argument building, always writes the resolved CLI system-prompt temp file, and adds buildCliArgs resume tests.

## Best Possible Solution

Carry prompt file or arg injection through buildClaudeLiveArgs for resumed Claude live sessions, update spawned-argv coverage, and add redacted real Claude resume proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main suppresses prompt creation/flags on resume, and the default Claude live-session path strips those flags before spawn; I did not run a live Claude CLI session in this read-only review.

## Solution Assessment

No. The helper and execute changes are necessary but incomplete because the default claude-stdio live-session argv builder still removes the prompt flags on resumed sessions.
