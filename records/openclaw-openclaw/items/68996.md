## Summary

Keep open: this targets a real Google Gemma routing bug and has sufficient copied live-output proof, but the current diff still introduces blocking transport/baseUrl regressions that need repair before merge.

## What This Changes

The branch forces Gemma dynamic resolution and explicit Google provider config normalization from OpenAI-compatible transport to `google-generative-ai`, with a provider-policy regression test.

## Best Possible Solution

Switch Gemma to native Google transport atomically in the Google plugin, updating `api` and compatible `baseUrl` only for affected Gemma entries while preserving explicit OpenAI-compatible sibling models.

## Reproduction Assessment

Yes, at source level. Current main and the PR diff expose the relevant Gemma model/config paths, and the PR discussion provides before/after endpoint checks plus copied after-fix E2E output; I did not run a credentialed Google call.

## Solution Assessment

No. The PR targets the right Google plugin surfaces, but the diff must keep `api` and `baseUrl` compatible and gate provider-level baseUrl normalization on an actual Gemma API switch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Patch Gemma baseUrl when forcing native API:** `extensions/google/provider-models.ts:175-176`
  - body: Setting `api: "google-generative-ai"` for every Gemma clone can preserve a CLI template `baseUrl`, so native requests may be built against `https://cloudcode-pa.googleapis.com` instead of the Generative Language endpoint. Patch a compatible baseUrl together with the API switch or avoid switching CLI-sourced clones.
  - confidence: 0.91
- **[P1] Gate Gemma baseUrl normalization on an API switch:** `extensions/google/provider-policy.ts:108-115`
  - body: This check strips `/openai` from the provider baseUrl whenever any Gemma id exists, even when a model omits `api` and still inherits `providerConfig.api = "openai-completions"`. That sends OpenAI-compatible requests to the native path and can break sibling OpenAI-compatible models under the same provider config.
  - confidence: 0.89
