Full review comments:

- **[P1] Patch Gemma baseUrl when forcing native API:** `extensions/google/provider-models.ts:177`
  - body: This sets only `api` on every Gemma clone. For `google-gemini-cli`, the resolver can clone the CLI template first, keeping its `https://cloudcode-pa.googleapis.com` baseUrl, but the native Google transport expects the Generative Language host; that makes OAuth/CLI Gemma runs fail or route to an incompatible endpoint. Please either scope the override to native Google templates or patch the baseUrl together with the API.
  - confidence: 0.9
- **[P1] Gate baseUrl normalization on an API switch:** `extensions/google/provider-policy.ts:111-115`
  - body: This new Gemma trigger runs provider-level native baseUrl normalization even when a Gemma model omits `api` and still inherits `provider.api = "openai-completions"`. That leaves OpenAI transport pointing at `/v1beta` instead of `/v1beta/openai`, and can also affect sibling OpenAI-compatible rows in the same provider config. Please only normalize the baseUrl when the affected model is actually switched to `google-generative-ai`.
  - confidence: 0.9
