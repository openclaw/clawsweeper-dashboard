## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Patch Gemma baseUrl/auth with the native API switch:** `extensions/google/provider-models.ts:177`
  - body: This unconditionally changes every Gemma clone to `google-generative-ai`, but `google-gemini-cli` resolution can still pick a CLI template first and carry its Cloud Code baseUrl/OAuth-backed provider state. That produces a model whose API says native Generative AI while its route/auth still belong to the CLI backend, so Gemini CLI Gemma users can fail before reaching the intended endpoint. Scope the override to compatible Google API-key templates or patch the route/auth coherently with tests for the CLI provider.
  - confidence: 0.89
- **[P1] Gate Gemma baseUrl normalization on a real API switch:** `extensions/google/provider-policy.ts:117-118`
  - body: `hasExplicitGemmaModel` makes the provider normalize its baseUrl to native `/v1beta` based only on the model id, but `normalizeProviderModels` switches `model.api` only when the row explicitly says `openai-completions`. If a Gemma row omits `api`, it still inherits provider-level `openai-completions` and sends OpenAI requests to the native path, and the same provider baseUrl can affect sibling models. Only strip `/openai` when the model API is actually switched, and cover omitted-api and mixed-model configs.
  - confidence: 0.91
