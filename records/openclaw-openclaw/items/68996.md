## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Patch Gemma baseUrl with the native API override:** `extensions/google/provider-models.ts:177`
  - body: For Gemma, this forces `api: "google-generative-ai"` for every cloned template but leaves the selected template `baseUrl` intact. When `google-gemini-cli` supplies the first matching template, the cloned model can keep `https://cloudcode-pa.googleapis.com` while the Google native transport builds `:streamGenerateContent` URLs from that baseUrl, so CLI Gemma requests are routed to an incompatible host. Patch the baseUrl together with the API override or avoid the override for CLI-sourced templates.
  - confidence: 0.93
- **[P1] Gate Gemma baseUrl normalization on an API switch:** `extensions/google/provider-policy.ts:111-118`
  - body: This new Gemma trigger returns true for any explicit Gemma model before checking whether the model will actually use `google-generative-ai`. If a Gemma entry omits `model.api` under a provider-level `api: "openai-completions"`, `normalizeProviderModels` leaves the model on OpenAI transport while provider normalization strips `/openai` from the baseUrl, causing OpenAI-compatible requests and sibling models to hit the native path. Only normalize the provider baseUrl when the Gemma model API is actually switched or declared native.
  - confidence: 0.91
