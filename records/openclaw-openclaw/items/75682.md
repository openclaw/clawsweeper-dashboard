## Summary

Keep this PR open, but do not merge as-is: the patch is broader than the stated nil-safety goal and would regress existing session status model identity behavior for runtime providers and configured custom-provider inference.

## What This Changes

The PR removes `resolveSessionModelIdentityRef` from `session_status`, derives card provider/model labels locally, and uses stored `entry.model` during model override resolution.

## Best Possible Solution

Keep the shared model identity resolver in the status path and add only the narrow guard needed for any real undefined-provider case, with focused regression coverage and a changelog entry if observable status output changes.

## Reproduction Assessment

Yes. Existing current-main tests reproduce the relevant behavior by exercising `session_status` with stored `modelProvider`, configured custom-provider-only models, and providerless runtime models.

## Solution Assessment

No. Replacing the resolver with local direct field checks is not the narrowest maintainable fix because it drops established provider inference; preserving the resolver and guarding only the undefined path is safer.
