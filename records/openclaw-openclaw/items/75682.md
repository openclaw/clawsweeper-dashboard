## Summary

Keep this PR open, but the current patch is not merge-ready. It removes the existing session model identity resolver and treats any stored runtime model without explicit overrides as providerless, which regresses current `session_status` behavior for sessions that already have `modelProvider` or an unambiguous configured custom provider.

## What This Changes

The PR changes `session_status` model-card resolution to stop importing `resolveSessionModelIdentityRef`, derive provider/model labels directly from the session entry and configured defaults, and consider stored `entry.model` during model override resolution.

## Best Possible Solution

Keep the existing model identity resolver behavior and make only the narrow nil-safety adjustment needed for any real `providerForCard` undefined path, with focused regression coverage for providerless runtime models, runtime `modelProvider`, configured-provider inference, and a user-facing changelog entry if the fix changes observable status output.

## Reproduction Assessment

Yes. The current main test cases in `src/agents/openclaw-tools.session-status.test.ts` provide a high-confidence reproduction path: run the session status tool against entries with `modelProvider` and configured custom-provider-only models, then assert the status card primary model label and auth lookup metadata.

## Solution Assessment

No. Replacing `resolveSessionModelIdentityRef` with local direct field checks is broader than the claimed nil-safety fix and drops established provider inference; the safer solution is to keep the resolver and guard only the undefined value path that can actually occur.
