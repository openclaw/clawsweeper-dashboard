## Summary

Current main already handles the providerless `session_status` model-card path safely and with broader behavior than this PR: it keeps the centralized model identity resolver, preserves configured-provider inference, and has coverage for both providerless and inferred-provider runtime models. The PR branch is superseded and would regress configured custom-provider status cards while also adding an unrelated empty root file.

## What This Changes

The PR adds an empty top-level `git` file and rewrites `src/agents/tools/session-status-tool.ts` model-card selection to bypass `resolveSessionModelIdentityRef`, use `entry.model` directly, and pass an empty provider when the model is treated as providerless.

## Best Possible Solution

Keep the current main implementation centered on `resolveSessionModelIdentityRef`. If a new undefined-provider crash is found, add a focused regression test for that exact session entry shape without removing configured-provider inference or adding unrelated repository artifacts.

## Reproduction Assessment

Original crash reproduction is not provided: the PR body is still the blank template. For the branch regression, yes: the existing `session_status` custom-provider runtime-model test exercises the affected path and would fail if runtime-only models were always treated as providerless.

## Solution Assessment

No, this PR is not the best way to solve the issue. Current main already handles providerless cards safely while preserving configured-provider inference; the safer path is to keep that implementation and only add a targeted test if a distinct crash is reported.
