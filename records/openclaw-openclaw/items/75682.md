## Summary

Keep open. The branch is a narrow status-card change, but it regresses established model-provider resolution by replacing the shared resolver with local checks that treat provider-backed runtime models as providerless.

## What This Changes

The PR removes `resolveSessionModelIdentityRef` from `session_status`, derives card provider/model labels locally, and uses stored `entry.model` during model override resolution.

## Best Possible Solution

Keep the shared model identity resolver in the status path and add only the narrow guard needed for any real undefined-provider case, with focused regression coverage and a changelog entry if observable status output changes.

## Reproduction Assessment

Yes for the PR regression: source inspection plus existing tests show that applying this branch would drop provider identity for stored `modelProvider` and inferred custom-provider runtime models. The original crash path is not separately documented in the PR body.

## Solution Assessment

No. Replacing `resolveSessionModelIdentityRef` with direct local field checks is not the narrowest maintainable fix because it bypasses existing provider inference and slash-model parsing semantics.
