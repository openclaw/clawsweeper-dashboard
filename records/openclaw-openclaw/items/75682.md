## Summary

Keep this PR open: the branch is not safe to merge because it replaces the shared session model identity resolver with a local providerless check that drops runtime provider identity and configured-provider inference already covered by current tests.

## What This Changes

The PR changes `session_status` model override and status-card label resolution to derive provider/model labels locally and to use stored `entry.model` as a model fallback.

## Best Possible Solution

Keep the shared resolver in the status path and add only the narrow guard needed for any real undefined-provider case, with focused regression coverage and a changelog entry if observable output changes.

## Reproduction Assessment

Yes. Current-main tests exercise stored `modelProvider`, configured custom-provider inference, and providerless runtime models; this branch would fail the provider-backed cases by rendering them providerless.

## Solution Assessment

No. Replacing `resolveSessionModelIdentityRef` with local direct field checks is not the narrowest maintainable fix because it discards established provider inference; preserving the resolver and guarding only the undefined path is safer.
