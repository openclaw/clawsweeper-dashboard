## Summary

Keep this PR open, but do not merge it as-is. The branch tries to make session status tolerate a missing card provider, but it drops the existing model identity resolver and would regress runtime provider labels and model-auth lookup for sessions that already record `modelProvider` or have an inferable configured provider.

## What This Changes

The PR changes `src/agents/tools/session-status-tool.ts` to remove `resolveSessionModelIdentityRef`, derive the session-status card provider/model from local session/default fields, consider persisted `entry.model` during model override resolution, and pass an empty provider string when no provider is selected.

## Best Possible Solution

Keep the existing identity resolver as the source of truth and make only a narrow nil-safety adjustment for any real undefined-provider path, with focused tests that lock runtime `modelProvider`, configured-provider inference, slash-prefixed runtime models, and providerless legacy runtime models.

## Reproduction Assessment

Yes. The current tests in `src/agents/openclaw-tools.session-status.test.ts` give a high-confidence reproduction path for the regression: run the session status tool with stored `modelProvider`, configured custom-provider-only models, and providerless runtime models, then assert the primary label and model-auth metadata.

## Solution Assessment

No. Replacing `resolveSessionModelIdentityRef` with local direct field checks is broader than the claimed nil-safety fix and loses established provider inference; the safer repair is to keep the resolver and guard only the consumer path that can actually receive a missing provider.
