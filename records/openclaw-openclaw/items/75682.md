## Summary

Keep open: the proposed patch introduces a status-card regression by bypassing the shared model identity resolver, and the original undefined-provider crash path is not documented beyond the title.

## What This Changes

The PR removes shared session model identity resolution from `session_status` and replaces status-card provider/model selection with local providerless runtime-model logic.

## Best Possible Solution

Keep the shared resolver in the status path and add only a narrow guard for a real undefined-provider case, with focused regression coverage and a changelog entry if output changes.

## Reproduction Assessment

Yes for the PR regression: source inspection plus existing tests show this branch would drop provider identity for stored `modelProvider` and inferred custom-provider runtime models. No high-confidence reproduction is provided for the original undefined-provider crash path.

## Solution Assessment

No. Replacing `resolveSessionModelIdentityRef` with local field checks is not the narrowest maintainable fix because it bypasses existing provider inference, stored-provider preservation, and slash-model parsing semantics.
