## Summary

## What This Changes

The issue requests that interactive Telegram session lanes be released after a successful visible reply during provider failover rather than waiting for failover cleanup or timeout.

## Best Possible Solution

Add an explicit post-visible-delivery boundary for interactive channel replies that releases same-session intake only after durable delivery, while preserving transcript/finalization ordering and keeping provider timeout/escalation policy in the failover layer.
