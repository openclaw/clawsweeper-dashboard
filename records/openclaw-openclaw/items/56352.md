## Summary

Keep open: current main and the latest release still gate visible same-session turns behind active reply/run ownership, while the merged recovery fixes only cover stale or wedged cleanup paths after delay.

## What This Changes

The issue asks OpenClaw to release same-session lane ownership after a successful channel-visible reply during model failover, with additional provider timeout and escalation improvements.

## Best Possible Solution

Define and test a durable reply/lane lifecycle contract that admits later inbound turns after safe visible delivery while preserving transcript, failover, terminal-run, and channel-delivery consistency.

## Reproduction Assessment

Yes from source: visible turns wait for active reply operations, and embedded runs own the same-session lane until completion or stale recovery. No live Telegram/provider-outage reproduction was run in this read-only review.

## Solution Assessment

No: the literal release-on-any-reply rule is not yet proven as the safest fix. The better path is a maintainer-owned lifecycle contract with regression coverage for reply delivery, failover cleanup, transcript durability, and queued inbound ordering.
