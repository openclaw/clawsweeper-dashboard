## Summary

Keep open. Current main still renders Feishu card notes from fixed Agent/Model/Provider fields, has no Feishu footer configuration or card-footer directive, and the existing usage/status surfaces do not feed the Feishu card footer requested here. This remains a real bundled Feishu plugin feature gap, but the public config/API shape and any model-emitted directive behavior need maintainer direction before cleanup can safely close it.

## What This Changes

Request Feishu card replies to support configured or dynamic custom footer fields such as context usage and compaction counts.

## Best Possible Solution

Implement this in the Feishu plugin as a typed `channels.feishu.cardFooter` option for controlled built-in fields first, then compose the same note for streaming and non-streaming card paths with focused tests and Feishu docs. If maintainers want model-supplied footer text, add it later behind an explicit opt-in with strict sanitization and size limits.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path: inspect current main with Feishu `renderMode: "card"`/streaming paths and search the config/directive surfaces. Those checks show no custom footer support today.

## Solution Assessment

No current solution is present on main, and the issue lists multiple possible APIs. The safest maintainable direction appears to be a typed Feishu config for built-in footer fields first, rather than starting with a free-form assistant directive.
