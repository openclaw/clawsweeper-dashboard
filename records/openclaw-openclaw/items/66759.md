## Summary

Keep open. Current main still renders Feishu card footers from fixed Agent/Model/Provider metadata, has no Feishu footer config or card-footer directive, and existing usage/status surfaces do not feed the Feishu card footer requested here.

## What This Changes

Request Feishu message cards support configurable custom footer fields such as context usage, compaction count, or session age.

## Best Possible Solution

Implement a typed Feishu plugin `channels.feishu.cardFooter` option for curated built-in fields first, wire it through streaming and non-streaming card notes, and update Feishu docs/tests; defer free-form model directives unless explicitly designed.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path: inspect Feishu `renderMode: "card"`/streaming delivery and config/directive surfaces on current main. Those checks show no custom footer support today.

## Solution Assessment

No current solution is present on main, and the issue lists multiple possible APIs. The narrowest maintainable path is likely a typed Feishu config for built-in footer fields before any assistant-controlled directive.
