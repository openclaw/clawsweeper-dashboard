## Summary

Keep open. Current main and the latest release still contain the macOS LaunchAgent restart path that rejects a busy configured gateway port before `kickstart`, while the open scoped fix at https://github.com/openclaw/openclaw/pull/89096 has not landed and the closed intent-only attempt did not cover the full reported restart ordering.

## What This Changes

The issue reports that macOS LaunchAgent-managed gateway restarts can race with KeepAlive and leave the gateway stopped instead of restarted.

## Best Possible Solution

Keep this as the canonical macOS LaunchAgent restart-race tracker; land or revise a launchd-aware fix that treats the managed listener as the restart target, uses unambiguous restart routing, re-bootstrap recovery where needed, and proves the behavior on real macOS.

## Reproduction Assessment

Yes at source level: current main still contains and tests the busy-port-before-kickstart failure, and SIGTERM still routes to stop without a matching restart intent. I did not run a live macOS LaunchAgent KeepAlive reproduction in this read-only review.

## Solution Assessment

Unclear as a final fix: the expected behavior is established, but the best implementation needs maintainer-approved launchd restart semantics and live macOS proof for busy-port, SIGTERM, and re-bootstrap timing.
