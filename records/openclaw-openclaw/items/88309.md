## Summary

Keep open. Current main and v2026.6.9 still contain the macOS LaunchAgent restart path that rejects a busy configured gateway port before launchd restart, and SIGTERM still routes to shutdown unless a restart-intent payload is present; the open busy-port PR is a useful scoped candidate but has not merged and does not settle the full restart-intent/re-bootstrap behavior.

## What This Changes

The issue requests that macOS LaunchAgent-managed gateway restarts survive KeepAlive and port-release races instead of leaving the gateway shut down.

## Best Possible Solution

Keep this as the canonical LaunchAgent restart-race tracker; review, land, or revise the scoped busy-port PR, then settle SIGTERM intent ordering and re-bootstrap behavior with live macOS proof.

## Reproduction Assessment

Yes at source level: current main still contains and tests the busy-before-LaunchAgent-restart failure, and SIGTERM still stops without a consumable restart intent. A live macOS KeepAlive timing reproduction was not run in this read-only review.

## Solution Assessment

Unclear as a final fix. The open busy-port PR is the right scoped direction for one branch, but the best overall solution needs maintainer-approved launchd restart semantics and live proof for SIGTERM, stale cleanup, and re-bootstrap timing.
