## Summary

Keep open. Current main and the latest release still reject a busy configured LaunchAgent gateway port before launchctl restart, SIGTERM still becomes shutdown unless a restart-intent payload is consumed, and the related restart-intent PR was closed after its author identified an unhandled pre-intent SIGTERM path.

## What This Changes

The issue asks OpenClaw to make macOS LaunchAgent-managed gateway restarts recover reliably instead of treating the expected gateway listener or KeepAlive race as a fatal shutdown.

## Best Possible Solution

Keep this as the focused LaunchAgent restart-race tracker; the end state should route managed macOS restarts through one launchd-aware path that treats the expected gateway listener as the restart target, avoids SIGTERM-as-stop, re-bootstraps when launchd is unloaded, and proves the behavior on real macOS.

## Reproduction Assessment

Yes at source level: current main still contains and tests the busy-before-LaunchAgent-restart failure, and SIGTERM still stops without a consumable restart intent. A full live macOS KeepAlive timing reproduction was not run in this read-only review.

## Solution Assessment

Unclear as a final fix. A simple retry or intent-only patch is not enough by itself; the safer solution is a maintainer-approved unified launchd restart path with real macOS proof.
