## Summary

Keep open. Current main and the latest release still contain the macOS LaunchAgent restart path that rejects a busy configured gateway port before launchd restart, while the linked busy-port fix remains open and the broader SIGTERM/restart-intent ordering still needs maintainer review.

## What This Changes

The issue asks for macOS LaunchAgent gateway restarts with KeepAlive=true to avoid converting restart requests into a stopped or offline gateway when the managed port is still occupied.

## Best Possible Solution

Keep this as the canonical LaunchAgent restart-race tracker; review, land, or revise the scoped busy-port PR, then settle SIGTERM intent ordering and re-bootstrap behavior with live macOS proof.
