## Summary

Keep open. Current main and the latest release still contain the macOS LaunchAgent restart path that rejects a busy configured gateway port before `kickstart`, and SIGTERM still becomes stop unless a matching restart intent is consumed; the open scoped fix at https://github.com/openclaw/openclaw/pull/89096 is useful but has not landed and does not close the broader LaunchAgent restart-race design by itself.

## What This Changes

This issue reports macOS LaunchAgent `KeepAlive` restart requests leaving the gateway shut down or unloaded instead of reliably restarting when port ownership and SIGTERM ordering race.

## Best Possible Solution

Keep this as the canonical macOS LaunchAgent restart-race tracker; land or revise a launchd-aware fix that treats the managed listener as the restart target, uses unambiguous restart routing, re-bootstrap recovery where needed, and proves the behavior on real macOS.

## Reproduction Assessment

Yes at source level: current main still contains and tests the busy-before-LaunchAgent-restart failure, and SIGTERM still stops without a consumable restart intent. A live macOS KeepAlive timing reproduction was not run in this read-only review.

## Solution Assessment

Unclear as a final fix. The open busy-port PR is the right scoped direction for one branch, but the best overall solution needs maintainer-approved launchd restart semantics and live proof for SIGTERM, stale cleanup, and re-bootstrap timing.
