## What This Changes

The PR adds a SHA-256-keyed, byte-bounded in-process resize-result cache in `src/agents/tool-images.ts` plus regression tests and log-test cache reset coverage.

## Best Possible Solution

Keep this PR open for maintainer review and either land or adapt the shared-sanitizer cache while preserving full-payload keying, no-op MIME preservation, and byte-bounded eviction tests.

## Reproduction Assessment

Yes. Current-main source shows session replay calls `sanitizeSessionMessagesImages`, that helper routes historical image blocks through `sanitizeContentBlocksImages`, and the resize helper still has no cache before image probe/resize work.

## Solution Assessment

Yes, likely. The shared sanitizer boundary is the narrowest maintainable place because CLI, tools, session replay, and prompt-image callers already route through it; per-channel or per-session caches would duplicate the invariant, but maintainers still need to accept process-local retention.
