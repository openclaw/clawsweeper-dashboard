## Best Possible Solution

Keep this PR open for maintainer review and either land or adapt the shared-sanitizer cache while preserving full-payload keying, no-op MIME preservation, fixed byte-bounded eviction, and regression coverage.

## Reproduction Assessment

Yes. Current-main source shows session replay calls `sanitizeSessionMessagesImages`, that helper routes historical image blocks through `sanitizeContentBlocksImages`, and the resize helper still has no cache before image probe/resize work.

## Solution Assessment

Yes, likely. The shared sanitizer boundary is the narrowest maintainable place because tool results, prompt images, and session replay already route through it; per-channel or per-session caches would duplicate the invariant, but maintainers still need to accept process-local retention.
