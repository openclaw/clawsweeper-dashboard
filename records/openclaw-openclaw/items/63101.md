## Summary

Keep open. Current main and the latest release still have strict Feishu channel/account config validation without a Feishu stale-key migration for the reported legacy keys, and the live closing PR remains open and conflicting rather than merged.

## What This Changes

The issue asks whether Feishu upgrade failures from legacy config keys such as `installs`, `plugins`, `ackReaction`, `threadSession`, `mediaLocalRoots`, and `botName` are intentional, and requests either documentation or a compatibility fix.

## Best Possible Solution

Land or replace the linked open PR with a narrow doctor/legacy migration that handles the agreed Feishu legacy keys before strict validation, keeps unknown-key rejection for new mistakes, and adds regression coverage plus upgrade notes.

## Reproduction Assessment

Yes. A config containing one of the reported legacy keys under `channels.feishu` or `channels.feishu.accounts.<id>` is source-reproducible because current Feishu schemas are strict and no current Feishu doctor normalizer strips those keys before channel validation.

## Solution Assessment

Unclear. A pre-validation doctor/legacy migration is the right direction, but the current open PR is conflicting and may not cover the broader stale-key set reported in comments.
