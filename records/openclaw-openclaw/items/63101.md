## Summary

Keep open. Current main and the latest release still have strict Feishu config validation without a doctor compatibility migration for the reported legacy keys, and the linked closing PR remains open/dirty rather than merged.

## What This Changes

The issue asks OpenClaw to preserve or document upgrade compatibility for legacy Feishu channel config keys that strict v4.8+ validation now rejects.

## Best Possible Solution

Keep strict validation, but land a pre-validation doctor compatibility migration for the agreed stale Feishu keys with regression coverage and upgrade proof.

## Reproduction Assessment

Yes, source-backed. A Feishu config containing legacy keys such as installs, plugins, ackReaction, threadSession, or mediaLocalRoots becomes additional properties under the strict schema, validation reports channels.feishu invalid config, and current doctor compatibility code does not remove them.

## Solution Assessment

Yes for the compatibility-repair direction. The maintainable fix is a doctor/shared or Feishu doctor-contract migration before validation, not relaxing strict schemas at runtime.
