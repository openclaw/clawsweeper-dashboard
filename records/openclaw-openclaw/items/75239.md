WebChat should stop rendering duplicate assistant bubbles after a completed reply when later transcript/session events reload history.

Add a focused UI regression for renderable final followed by same-session transcript/index events, then make final/history reconciliation idempotent for that ordering while preserving idle refreshes and tool-result history reloads.
