Fix the WebChat duplicate assistant reply regression reported in https://github.com/openclaw/openclaw/issues/75239. Add a focused Control UI regression where a renderable assistant `chat` final for the active run is processed, clears `chatRunId`, and then a same-session `session.message` and/or `sessions.changed` arrives; the UI must not duplicate the already-rendered assistant bubble. Keep legitimate idle transcript refreshes working, preserve history reloads for silent/non-renderable final payloads and aborted/error terminal cases, and avoid a blanket no-op for all `session.message` events.
