## Summary

Keep open: current main fixes fresh Bedrock plugin installs by registering `bedrock-converse-stream` in the plugin, but the reported bare package-manager host upgrade path still has no source-backed repair that updates or warns on an older installed Bedrock provider before runtime use.

## What This Changes

The issue reports a 2026.5.28 upgrade regression where an older installed Amazon Bedrock provider plugin no longer registers `bedrock-converse-stream` after the host removed Pi-era provider registration.

## Best Possible Solution

Keep the issue open and choose an upgrade-safe repair path that updates or blocks/warns stale official Bedrock provider plugins before gateway restart; avoid a long-lived core Bedrock shim unless maintainers explicitly choose a temporary hotfix.

## Reproduction Assessment

Yes from source, though not from a live AWS run: current core does not register `bedrock-converse-stream`, the current plugin does, and package postinstall explicitly does not update plugin packages. A stale 2026.5.26 provider plugin can therefore remain without the new registration after a host-only npm upgrade.

## Solution Assessment

Unclear until maintainers choose the upgrade policy. A package-update/plugin-convergence repair or explicit stale-plugin warning fits the current plugin boundary better than restoring a permanent core Bedrock registration shim.
