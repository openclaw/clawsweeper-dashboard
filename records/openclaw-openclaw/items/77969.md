## Summary

Keep open. Current main has TTS cleanup for directives, markdown, media/short replies, and long-reply summarization, but no documented or typed option to omit emoji, arbitrary symbols, or file paths; adding one is a new TTS policy/config feature that needs maintainer design.

## What This Changes

The issue requests a TTS option to omit or summarize symbols, emoji, and long file paths before speech output.

## Best Possible Solution

Define a core-owned TTS text-preparation policy under messages.tts, with config/schema/docs and regression tests covering emoji, symbol, and path cases.

## Reproduction Assessment

Not applicable; this is an enhancement request, not a report of broken existing behavior. Source inspection shows current main lacks the requested emoji/symbol/path filter option rather than failing an established contract.

## Solution Assessment

Unclear as proposed because the issue asks for a broad option but does not define exact filtering semantics. A maintainable solution should be a core TTS text-preparation policy, not a provider-specific workaround.
