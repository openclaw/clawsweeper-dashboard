## Summary

Keep open. Current main has Auto-TTS skips for media, short replies, long-reply handling, and Markdown cleanup, but no supported policy for emoji/symbol/path filtering or for suppressing gateway/system status text from Auto-TTS.

## What This Changes

Request configurable TTS filtering for emoji, symbols, paths, and optional Auto-TTS suppression for gateway/system messages, possibly per agent.

## Best Possible Solution

Define a core TTS spoken-text preparation and suppression policy under messages.tts, with schema/docs coverage and tests for emoji, symbols, paths, and gateway/system status messages.

Not applicable; this is an enhancement request rather than broken existing behavior. Source inspection gives high confidence that current main lacks the requested configurable filter/suppression policy.

## Solution Assessment
