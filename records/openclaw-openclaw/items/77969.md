## Best Possible Solution

Define one shared TTS text-preparation policy in the speech-core runtime, document the config contract, and decide whether path elision and restart/system-message suppression are part of the same option or separate scoped work.

## Reproduction Assessment

## Solution Assessment

Unclear as a complete solution: an opt-in shared filter is plausible for emoji/symbols, but path replacement and auto-mode system-message suppression need an explicit maintainer policy and config contract first.
