## Summary

Keep open: current main still learns Telegram forum topic names but does not carry that value into stored session display metadata, gateway session rows, or the Control UI selector.

## What This Changes

Request to show human-readable Telegram forum topic names in the session dropdown instead of raw topic-qualified session keys.

## Best Possible Solution

Add a generic session display/thread-label metadata path, have Telegram populate it from cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label without hard-coding Telegram-only UI formatting.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, but not as a Telegram-specific UI formatter. The maintainable solution is a generic display-label or thread-label seam populated by the Telegram plugin from cached TopicName.
