## Summary

Keep open because current main and the latest release still expose Telegram forum topic names only as `TopicName`, while the open linked PR is the candidate that adds the missing `ThreadLabel` handoff into existing session display metadata.

## What This Changes

The issue requests human-readable Telegram forum topic names in session dropdown labels instead of raw Telegram topic session keys.

## Best Possible Solution

Review and land the linked candidate PR, or close it and replace it with the same narrow `TopicName` to `ThreadLabel` session-display handoff plus focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram forum topic names are available as `TopicName`, but the Telegram payload does not set `ThreadLabel`, while the session store uses `ThreadLabel` to populate `displayName`.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing generic `ThreadLabel` to `displayName` path rather than adding Telegram-specific UI parsing or a new config option.
