## Summary

Keep open. Current main and v2026.6.8 still expose Telegram TopicName only to inbound context/prompt metadata, while persisted session metadata, gateway session rows, and the Control UI selector still choose label/displayName/group fallbacks that do not include the human topic name.

## What This Changes

Request to show Telegram forum topic sessions in the session dropdown with human-readable topic names instead of raw numeric topic-key labels.

## Best Possible Solution

Carry a generic explicit thread/session display label from known Telegram TopicName through session metadata, sessions.list, and the Control UI selector while preserving existing non-Telegram label behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a Telegram forum message can populate TopicName, but session metadata, gateway rows, and UI display resolution never consume that field. I did not run a live Telegram/Desktop repro in this read-only review.

## Solution Assessment

Yes, the best fix is to add a narrow generic thread/session display-label path rather than a Telegram-specific UI string parser. Telegram should populate that label from the existing cached TopicName and the UI should keep consuming normal row label/displayName data.
