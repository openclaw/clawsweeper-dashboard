## Summary

Keep open: current main captures Telegram forum topic names, but the value still stops at inbound context and is not carried into stored session display metadata, gateway session rows, or the Control UI selector. The old implementation PRs are closed unmerged, and the open UI cleanup PR is adjacent rather than a complete metadata fix.

## What This Changes

Request to show Telegram forum-topic sessions with human-readable group and topic labels in the session dropdown instead of raw topic IDs or internal session keys.

## Best Possible Solution

Add a generic session thread/display-label path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list/GatewaySessionRow, and have the Control UI prefer that explicit label without Telegram-only key parsing.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram can produce TopicName for forum messages, but SessionEntry, gateway row construction, and the Control UI selector do not consume it for display labels.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing topic-name cache and generic ThreadLabel/display metadata path; a UI-only formatter or Telegram-specific core key parser would leave backend session metadata incomplete.
