## Summary

Keep open: current main can learn and persist Telegram forum topic names, but the stored session metadata, gateway session row, and Control UI selector still do not consume that topic name for the visible session label.

## What This Changes

The issue requests human-readable Telegram forum topic names in the session dropdown instead of raw or numeric topic session labels.

## Best Possible Solution

Add a generic session thread/display-label path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list, and have the Control UI prefer that explicit label without hard-coding Telegram-only formatting in core.

## Solution Assessment
