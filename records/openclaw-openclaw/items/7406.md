## Summary

Keep open: current main still learns Telegram forum topic names but only exposes them as inbound context metadata; the stored session row and Control UI session selector still use generic group labels instead of the topic name.

## What This Changes

Request to show human-readable Telegram forum topic names in the session dropdown instead of raw topic session keys or indistinguishable group labels.

## Best Possible Solution

Carry the existing cached Telegram topic name through a generic thread/display-label session metadata path, expose it in sessions.list, and have the Control UI prefer that explicit label while preserving non-Telegram labels.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, the maintainable path is to carry the already-known topic label through a generic session display/thread-label seam. A Telegram-only UI formatter or new per-topic config name field would create a weaker source of truth.
