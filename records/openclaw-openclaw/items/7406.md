## Summary

Keep open. ## What This Changes

Request to show Telegram forum-topic sessions in the session dropdown with human-readable group/topic names instead of raw session keys or numeric topic IDs.

## Best Possible Solution

Keep this issue open and implement an explicit generic thread/display-label path that stores known Telegram topic names in session metadata, projects them through sessions.list, and has the Control UI prefer that explicit label while keeping existing non-Telegram labels intact.

## Reproduction Assessment

Yes. Source inspection shows Telegram forum messages can populate `TopicName`, while session metadata and gateway/UI label paths do not consume it for the session selector label.

## Solution Assessment

Yes, with a slightly more generic shape than the issue’s initial Telegram-specific field proposal. The maintainable fix is an explicit thread/display-label metadata path, not broad Control UI string parsing or a new per-topic config name option.
