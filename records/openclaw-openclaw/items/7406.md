## Summary

Keep open: current main still learns Telegram forum topic names, but it does not persist or expose that value as session display metadata for the gateway row or Control UI selector. The related duplicate reports are already closed in favor of this canonical tracker, and the open UI cleanup PRs are adjacent rather than a complete metadata bridge.

## What This Changes

Request to show Telegram forum topic names in the session dropdown instead of raw topic-scoped keys or indistinguishable group labels.

## Best Possible Solution

Add a generic session thread/display-label metadata path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list, and have the Control UI prefer that explicit label while preserving existing non-Telegram behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram can produce TopicName for forum messages, but the session metadata, gateway row, and Control UI selector do not consume it for display labels.

## Solution Assessment

Yes. The maintainable direction is a generic thread/display-label bridge from Telegram-owned topic metadata into session rows; a UI-only key formatter or Telegram-specific core parser would leave stored metadata incomplete.
