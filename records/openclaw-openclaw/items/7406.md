## Summary

Keep open: current main and v2026.5.20 still learn Telegram forum topic names only for inbound/prompt metadata, while stored session metadata, gateway rows, and the Control UI selector do not surface the topic name as the session label.

## What This Changes

The issue requests human-readable Telegram forum topic names in the Control UI session dropdown instead of raw or indistinguishable topic IDs.

## Best Possible Solution

Add a generic session display/thread-label metadata path, have Telegram populate it from cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label without hard-coding Telegram-only UI formatting.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, if implemented as a generic display-label or thread-label seam populated by Telegram from cached TopicName. A Telegram-specific UI formatter or a new per-topic config name field would be a weaker long-term fit.
