## Summary

Keep open: current main still learns Telegram forum topic names, but the value stops at inbound/prompt metadata and is not carried into stored session display metadata, gateway session rows, or the Control UI session selector. Related duplicates and older PRs are closed or only adjacent, so this issue remains the canonical tracker.

## What This Changes

Show Telegram forum-topic sessions in the session dropdown with human-readable group/topic labels instead of raw topic-key identifiers.

## Best Possible Solution

Add a generic session display/thread-label metadata path, have Telegram populate it from cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label without hard-coding Telegram-only formatting.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, if implemented as a generic display-label or thread-label seam populated by Telegram from cached TopicName. A Telegram-specific UI formatter or new per-topic config name field would be a weaker long-term fit.
