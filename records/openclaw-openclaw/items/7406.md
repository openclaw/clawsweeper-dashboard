## Summary

Keep open: current main and v2026.6.5 still learn Telegram TopicName but do not propagate it into persisted session display metadata, the Gateway session row, or the Control UI selector; related fix PRs are closed unmerged and no open replacement owns the remaining work.

## What This Changes

The issue asks for Telegram forum-topic sessions to show human-readable group and topic names in the session dropdown instead of raw topic IDs.

## Best Possible Solution

Carry the existing cached Telegram topic name through a generic thread/display-label session metadata path, expose it in sessions.list, and have the Control UI prefer that explicit label without adding per-topic config.

## Reproduction Assessment

Yes, source-level: current main can populate TopicName for Telegram forum topics, but the inspected metadata, Gateway, and UI display paths do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, the best fix is to propagate the already-known topic label through a generic session display/thread-label seam. A Telegram-only UI formatter or a new per-topic config name field would create a weaker source of truth.
