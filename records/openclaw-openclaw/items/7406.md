## Summary

Keep open. Current main learns Telegram forum topic names and exposes them as `TopicName`, but the value still does not reach stored session display metadata, gateway session rows, or the Control UI session selector; the exact linked fix PRs are closed unmerged, so this issue remains the canonical tracker.

## What This Changes

Show Telegram forum topic sessions in the session dropdown with human-readable group/topic names instead of raw `agent:...:topic:<id>` keys.

## Best Possible Solution

Add a narrow generic session display-label/thread-label path, populate it from Telegram’s cached topic name for forum-topic sessions, expose it through `sessions.list`, and have the UI prefer that explicit label without changing non-Telegram labels.

## Reproduction Assessment

Yes. Source inspection on current main shows `TopicName` is created from Telegram forum metadata, but stored session metadata, gateway rows, and the UI selector do not consume it.

## Solution Assessment

Yes. The best fix is the generic session display-label/thread-label handoff, not a Telegram-only UI parser or a new config option.
