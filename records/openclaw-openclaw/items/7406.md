## Summary

Keep open: current main still learns Telegram forum TopicName, but it does not carry that value into persisted session display metadata, gateway session rows, or the Control UI session selector. Recent discussion and duplicate closures already treat this issue as the canonical tracker, while the open selector cleanup PR is adjacent rather than a complete metadata bridge.

## What This Changes

Request human-readable Telegram forum topic names in session dropdown labels instead of raw topic IDs or indistinguishable group labels.

## Best Possible Solution

Carry known Telegram topic names through a generic session thread/display label in session metadata, gateway rows, and selector rendering while preserving existing non-Telegram labels.

## Reproduction Assessment

Yes, source-level. Current main can populate Telegram TopicName for forum messages, but the session metadata, gateway row, and Control UI selector paths do not consume it for visible labels.

## Solution Assessment

Yes. The narrow maintainable direction is a generic session thread/display-label bridge; UI-only key formatting or a Telegram-specific parser in core would leave stored metadata incomplete.
