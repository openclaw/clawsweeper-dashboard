## Summary

Keep open. Current main and v2026.6.10 still learn Telegram forum topic names as `TopicName`, but they do not pass that value into the generic `ThreadLabel`/session display path; the open linked PR is the current candidate fix and should land or be closed before this issue is resolved.

## What This Changes

Request human-readable Telegram forum topic names in session dropdowns instead of raw group/topic session keys.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96793 after redacted live Telegram proof confirms topic labels appear while group context and routing stay intact.

## Reproduction Assessment

Yes, source inspection is enough to reproduce the gap: current main learns `TopicName`, but the visible session path consumes `ThreadLabel`/`displayName` and Telegram does not populate `ThreadLabel`. I did not run a live Telegram forum group in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow existing seam: populate generic `ThreadLabel` from the already-known Telegram topic name while preserving `ConversationLabel` for the parent group. Adding config or UI key parsing would create a second source of truth.
