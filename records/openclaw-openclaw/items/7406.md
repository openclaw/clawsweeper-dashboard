## Summary

Keep open. Current main and the latest release still stop at Telegram `TopicName` and do not carry it into the `ThreadLabel`/`displayName` path that the session dropdown uses; the open linked PR is the right candidate path but is not merged yet.

## What This Changes

Request to show known Telegram forum topic names instead of raw topic session keys in session dropdowns.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94303 after proof and any regression coverage are acceptable, or keep this issue canonical until an equivalent `ThreadLabel`/displayName fix lands.

## Reproduction Assessment

Yes, at source level: current main resolves Telegram forum `TopicName` but does not set `ThreadLabel`, while session storage and UI listing prefer `displayName` derived from `ThreadLabel`. I did not run a live Telegram forum group repro in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable solution is to reuse the existing generic `ThreadLabel` to `displayName` seam and keep Telegram-specific topic-name extraction inside the Telegram plugin. The remaining action is to review or land the linked PR, not queue a second fix.
