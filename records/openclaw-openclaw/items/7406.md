## Summary

Keep open: current main and the latest release still learn Telegram forum topic names, but that value stops at Telegram inbound context and prompt metadata instead of becoming stored session display metadata, a gateway session row label, or a Control UI picker label.

## What This Changes

Show human-readable Telegram forum topic names in session labels instead of raw topic-qualified session keys or indistinguishable group labels.

## Best Possible Solution

Add a generic thread/display-label metadata path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label while preserving existing non-Telegram labels.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, while session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, the maintainable path is to carry the already-known topic label through a generic session display/thread-label seam. A Telegram-only UI formatter, topic-id parser, or new per-topic config name field would create a weaker source of truth.
