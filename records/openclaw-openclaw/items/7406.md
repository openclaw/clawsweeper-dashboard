## Summary

Keep open: current main learns Telegram forum topic names but only exposes them to inbound context metadata; stored session metadata, gateway session rows, and the Control UI selector still do not consume that topic name for display.

## What This Changes

The issue asks for Telegram forum-topic sessions to show human-readable group/topic names in session dropdowns instead of raw numeric topic keys.

## Best Possible Solution

Implement a generic session display/thread-label metadata path that Telegram populates from cached TopicName, then expose it through sessions.list and make the Control UI prefer it while preserving explicit labels and non-Telegram behavior.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics but session metadata, gateway rows, and UI display resolution do not consume it. I did not run live Telegram or browser proof because this is a read-only review.

## Solution Assessment

Yes, but not as a Telegram-specific UI formatter. The maintainable path is a narrow generic display-label/thread-label metadata seam filled by the Telegram plugin from cached TopicName.
