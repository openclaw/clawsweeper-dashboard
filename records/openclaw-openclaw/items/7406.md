## Summary

Keep open: current main can learn Telegram forum topic names, but that value still stops in inbound context metadata and is not carried into stored session display metadata, gateway session rows, or the Control UI selector.

## What This Changes

Request to show cached Telegram forum topic names as human-readable labels in the session dropdown instead of raw Telegram topic session keys.

## Best Possible Solution

Add a generic session display/thread-label metadata path, have Telegram populate it from cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label without hard-coding a Telegram-only UI formatter.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, if implemented as a generic display-label or thread-label seam populated by Telegram from cached TopicName. A Telegram-specific UI formatter or new per-topic config name field would be a weaker long-term fit.
