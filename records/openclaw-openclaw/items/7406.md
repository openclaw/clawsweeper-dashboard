Implement a generic session display/thread-label metadata path that Telegram fills from cached TopicName, expose it through sessions.list, and make the Control UI prefer that explicit label while preserving non-Telegram labels.

## Solution Assessment

Yes, but not as a Telegram-specific UI formatter. The maintainable solution is a generic display-label or thread-label seam populated by the Telegram plugin from cached TopicName.
