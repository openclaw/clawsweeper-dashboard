## Summary

Keep open: current main and the latest release still learn Telegram forum TopicName, but that value is not bridged into persisted session display metadata, gateway session rows, or the Control UI selector. The related duplicate issues are closed in favor of this tracker, and the open UI cleanup PRs are adjacent rather than a complete fix.

## What This Changes

The issue requests human-readable Telegram forum topic names in the session dropdown instead of raw or indistinguishable topic session keys.

## Best Possible Solution

Add a generic thread/display-label metadata bridge from Telegram-owned TopicName into stored session entries, gateway sessions.list rows, and the Control UI selector while preserving existing non-Telegram labels.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram can produce TopicName for forum messages, but session metadata, gateway rows, and the Control UI selector do not consume that value for display labels.

## Solution Assessment

Yes. A generic session thread/display-label bridge is the narrow maintainable direction; UI-only key formatting or a Telegram-specific core parser would leave stored metadata incomplete.
