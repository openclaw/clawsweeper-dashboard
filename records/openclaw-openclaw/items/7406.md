## Summary

Keep open. Current main has the Telegram topic-name source and cache, but the known topic name still does not flow into persisted session display metadata, gateway session rows, or the Control UI selector.

## What This Changes

Request to show Telegram forum-topic sessions in the session dropdown with human-readable group and topic names instead of raw topic-qualified session keys.

## Best Possible Solution

Add a narrow generic session thread/display-label path, populate it from Telegram's cached topic name for forum-topic sessions, expose it through `sessions.list`, and have the Control UI prefer that explicit label while preserving current non-Telegram labels.

## Reproduction Assessment

Yes, at source level. A Telegram forum message can carry `TopicName`, but the session metadata, gateway row, and UI selector paths inspected on current main never consume that field for the visible session label.

## Solution Assessment

Yes, with one adjustment. The maintainable fix should avoid Telegram-specific formatting in core and instead use a generic thread/display-label seam populated by the Telegram plugin.
