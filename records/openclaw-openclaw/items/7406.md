## Summary

Keep open: current main captures Telegram forum topic names, but the known topic name still stops at Telegram inbound metadata and is not carried into stored session labels, gateway session rows, or the Control UI selector.

## What This Changes

Show human-readable Telegram forum topic names in the session dropdown instead of raw topic-qualified session keys.

## Best Possible Solution

Add a generic explicit thread/session display-label path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that label without parsing raw Telegram keys.

## Reproduction Assessment

Yes, source-level: Telegram forum messages can set TopicName, but current session metadata, gateway rows, and the Control UI selector never consume TopicName as a display label. I did not run a live Telegram/UI reproduction in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is a generic explicit thread/session display label populated by Telegram from the cached topic name; adding Telegram-specific core formatting or UI key parsing would be a weaker direction.
