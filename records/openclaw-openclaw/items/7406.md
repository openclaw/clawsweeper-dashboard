## Summary

Keep open: current main still captures Telegram forum `TopicName`, but the stored session metadata, gateway session row, and Control UI selector do not consume it for the visible session label. The open UI cleanup at https://github.com/openclaw/openclaw/pull/51124 is adjacent and has no closing reference for this issue.

## What This Changes

Request to show Telegram forum-topic sessions in the session dropdown with a human-readable group/topic label instead of raw topic IDs or internal session keys.

## Best Possible Solution

Add a generic session thread/display-label path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list, and have the Control UI prefer that explicit label without hard-coding Telegram-only formatting in core.

## Reproduction Assessment

Yes. By source inspection, current main can produce TopicName for Telegram forum topics, but the stored SessionEntry, gateway row, and Control UI selector paths do not consume it for display labels.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing topic-name source and add a generic session display/thread-label seam; the open UI cleanup PR is useful adjacent work but does not complete the backend metadata path.
