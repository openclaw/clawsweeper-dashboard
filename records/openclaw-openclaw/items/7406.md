## Summary

Keep open: current main captures and persists Telegram forum topic names for inbound context, but the known topic name is still not carried into stored session display metadata, gateway session rows, or the Control UI session selector, so the central dropdown problem remains.

## What This Changes

The issue requests human-readable Telegram forum topic names in the session dropdown instead of raw topic-scoped session keys or indistinguishable group labels.

## Best Possible Solution

Add a generic session thread/display-label metadata path, have Telegram populate it from the cached forum TopicName, expose it through sessions.list, and make the Control UI selector prefer that explicit label without adding Telegram-specific formatting policy to core.

## Reproduction Assessment

Yes, at source level: current main captures TopicName for Telegram forum messages, but the session metadata, gateway row, and UI selector paths only use label/displayName/group-derived fields. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes for the direction, but the maintainable fix is a generic thread/display-label seam rather than embedding a Telegram-only spaced-colon formatter in core session code.
