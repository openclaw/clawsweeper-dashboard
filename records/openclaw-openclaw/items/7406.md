## Summary

Keep open: current main learns Telegram forum topic names, but the value still stops in inbound context metadata and is not carried into stored session display metadata, gateway session rows, or the Control UI selector.

## What This Changes

Request to show human-readable Telegram forum topic names in the session dropdown instead of raw `agent:...:telegram:group:...:topic:...` keys.

## Best Possible Solution

Carry the already-known Telegram topic name through a generic thread/display-label session metadata path, expose it in `sessions.list`, and make the Control UI prefer that explicit label without adding a per-topic config name fallback.

## Reproduction Assessment

Yes, source-level: current main populates `TopicName` for Telegram forum topics, but session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes for the generic metadata propagation shape: reusing the existing topic-name source and the existing label/displayName UI path is narrower than Telegram-only UI formatting. A new per-topic config name field would add another source of truth and should not be the first fix.
