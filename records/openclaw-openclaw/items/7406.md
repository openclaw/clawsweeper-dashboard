## Summary

Keep open: current main still learns Telegram forum topic names, but the value remains prompt/context metadata and is not propagated into stored session metadata, gateway session rows, or the Control UI session dropdown. The directly linked implementation attempts are closed unmerged, and the remaining open UI cleanup PR is adjacent rather than a complete fix.

## What This Changes

Request to show human-readable Telegram group and forum topic names in the session dropdown instead of raw topic session keys.

## Best Possible Solution

Add a generic explicit thread/session display-label path, populate it from Telegram cached topic names for forum-topic sessions, expose it through sessions.list, and have the UI prefer that label while preserving existing non-Telegram labels.

## Reproduction Assessment

Yes, source-level. Current main can set TopicName for a Telegram forum message, but the session metadata, gateway row, and UI selector code never consume TopicName as a display label.

## Solution Assessment

Yes. The maintainable direction is to reuse the existing generic ThreadLabel or an explicit session display-label path; Telegram-specific UI key parsing would create a second source of truth.
