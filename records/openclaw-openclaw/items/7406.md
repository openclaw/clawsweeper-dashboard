## Summary

Keep open. Current main captures and caches Telegram forum topic names, but it still does not carry the known topic name into persisted session display metadata, gateway session rows, or the Control UI session selector.

## What This Changes

Show known Telegram forum topic names in session dropdown labels instead of raw topic-scoped Telegram session keys.

## Best Possible Solution

Add a generic explicit thread/session display-label path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that label without parsing raw Telegram keys in the UI.

## Reproduction Assessment

Yes, source-level: Telegram forum messages can set TopicName, but current session metadata, gateway rows, and the Control UI selector never consume TopicName as a display label. I did not run a live Telegram/UI repro in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable direction is a generic explicit thread/session display label that Telegram can populate from the cached topic name. Parsing Telegram session keys in the UI or adding Telegram-specific core formatting would be a weaker fix.
