## Summary

Keep open. Current main and v2026.6.5 learn and persist Telegram forum topic titles, but the value remains inbound context metadata and is not propagated into stored session display metadata or the Control UI selector; all directly related implementation PRs are closed unmerged, so this remains the canonical tracker.

## What This Changes

Show known Telegram forum topic titles as human-readable labels in the Control UI session selector.

## Best Possible Solution

Promote the Telegram-observed title into a generic thread/session display-label field, persist it canonically, expose it through `sessions.list`, and let the selector prefer it without changing non-Telegram labels.

## Reproduction Assessment

Yes. Current source deterministically shows `TopicName` being produced while canonical session display metadata and selector rows do not carry it.

## Solution Assessment

Yes. A generic typed thread/session label path is the narrowest maintainable solution; new configuration or Telegram-specific UI key parsing would duplicate state and ownership.
