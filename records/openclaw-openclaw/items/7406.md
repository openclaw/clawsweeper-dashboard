## Summary

Keep open: current main learns Telegram forum topic names, but they still do not flow into stored session display metadata, gateway session rows, or Control UI labels; the closed implementation PRs were not merged, and the open UI cleanup PR is adjacent rather than complete.

## What This Changes

Request to show Telegram forum topic sessions with human-readable group/topic names in the session dropdown instead of raw or indistinguishable topic IDs.

## Best Possible Solution

Add a generic session thread/display-label path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list/GatewaySessionRow, and have the Control UI prefer that explicit label without Telegram-only key parsing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Telegram forum messages can produce TopicName, but SessionEntry, gateway row construction, and the Control UI selector do not consume that value for display labels.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing topic-name cache and add a generic session display/thread-label bridge; a UI-only formatter or Telegram-specific core key parser would leave the backend metadata gap unresolved.
