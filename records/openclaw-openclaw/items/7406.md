## Summary

Keep open: current main and the latest release learn Telegram forum topic names, but that value still stops at Telegram context metadata and is not carried into stored session display metadata, gateway session rows, or the Control UI selector.

## What This Changes

Show known Telegram forum topic names as human-readable labels in the session dropdown instead of raw topic-qualified session keys.

## Best Possible Solution

Add a generic thread/display-label metadata path, populate it from Telegram cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label while preserving non-Telegram labels.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but session metadata, gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, the maintainable path is to carry the already-known topic label through a generic session display/thread-label seam. A Telegram-only UI formatter or new per-topic config name field would create a weaker source of truth.
