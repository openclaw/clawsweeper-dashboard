## Summary

Keep open. Current main and v2026.6.8 still learn Telegram forum topic names only as `TopicName`, while the visible session-label path depends on `ThreadLabel` or stored `displayName`; the open linked PR is the current candidate fix and should land or be closed before this issue is closed.

## What This Changes

Request to show Telegram forum topic names in the session dropdown instead of raw topic-qualified session keys.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/94303; the final fix should reuse the existing `ThreadLabel` to `displayName` path instead of adding Telegram-specific UI parsing.

## Reproduction Assessment

Yes at source level: current main resolves Telegram forum `TopicName` but does not set `ThreadLabel`, while session initialization and the gateway/UI session selector already consume `displayName`. I did not run a live Telegram forum repro in this read-only review.

## Solution Assessment

Yes, the best current solution is the narrow linked PR path that maps the existing Telegram `topicName` into the generic `ThreadLabel`/`displayName` seam. A broader Telegram-specific UI parser or new config field would be a worse first fix because the generic display-label path already exists.
