## Summary

Keep open: current main and the latest release know Telegram forum topic names, but that value is still only prompt/hook metadata and is not projected into stored session display metadata, gateway session rows, or the Control UI selector. The live discussion and related searches point back to this issue as the canonical remaining tracker.

## What This Changes

Request to show human-readable Telegram forum topic names in the session dropdown instead of raw topic-qualified Telegram session keys.

## Best Possible Solution

Add a generic thread/topic display-label path through session metadata, gateway sessions.list rows, and Control UI display resolution, populated by Telegram from the existing cached TopicName.

## Reproduction Assessment

Yes, at source level: a Telegram forum topic with a cached TopicName reaches inbound context, while session metadata, gateway rows, and the UI selector do not consume it for the visible label. I did not run a live Telegram or visual Control UI reproduction in this read-only review.

## Solution Assessment

Yes. The best path is a narrow generic session/thread display-label projection, not Telegram-specific key parsing in the UI or a new user config option.
