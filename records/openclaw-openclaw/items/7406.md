## Summary

Keep open. Current main already captures Telegram forum topic names for inbound agent context, but that value still does not flow into persisted session display metadata, Gateway session rows, or the Control UI session selector.

## What This Changes

Show Telegram forum topic sessions in the session dropdown with human-readable group/topic labels instead of raw topic-qualified session keys.

## Best Possible Solution

Add an additive generic thread/topic display-label path through Telegram context, session metadata, Gateway sessions.list, and the Control UI selector, with Telegram populating it from the cached topic name.

## Reproduction Assessment

Yes. Source inspection on current main is enough: Telegram forum messages can populate ctx.TopicName, while SessionEntry, deriveGroupSessionPatch, Gateway row construction, and the UI selector do not consume that value for visible session labels.

## Solution Assessment

Yes. A generic display-label or ThreadLabel propagation path with Telegram supplying the known topic name is the narrowest maintainable direction; a UI-only key formatter or Telegram-specific core rule would leave other session consumers inconsistent.
