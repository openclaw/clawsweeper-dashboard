## Summary

Keep open. Current main still learns Telegram forum topic names but only emits them as `TopicName`, while the visible session-label path depends on `ThreadLabel` or stored `displayName`; the open linked PR is the current fix candidate and should land or be closed before this issue is closed.

## What This Changes

The issue requests human-readable Telegram forum topic names in session dropdown labels instead of raw Telegram topic session keys.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/94303; if it lands, close this issue as implemented by that merged fix.

## Reproduction Assessment

Yes, at source level: current main resolves Telegram forum topic names into `TopicName`, and session storage promotes `ThreadLabel` into `displayName`, but Telegram does not set `ThreadLabel`. I did not run a live Telegram forum/TUI setup in this read-only review.

## Solution Assessment

Yes. The best path is the narrow existing-seam fix that populates `ThreadLabel` from Telegram’s cached topic name, rather than parsing Telegram keys in the UI or adding a new per-topic config option.
