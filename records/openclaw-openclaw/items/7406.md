## Summary

Keep open: current main has the Telegram topic-name source and cache, but it still does not carry the known topic name into stored session display metadata, gateway session rows, or the Control UI session selector; the related implementation PRs are closed unmerged and the exact duplicates point back here.

## What This Changes

Request to show Telegram forum topic sessions in the session dropdown with a human-readable topic label instead of raw topic/session identifiers.

## Best Possible Solution

Add a narrow generic thread/topic display-label path through session metadata, gateway sessions.list rows, and Control UI display resolution, populated by Telegram from the existing cached TopicName.

## Reproduction Assessment

Yes, at source level: a Telegram forum topic with a cached TopicName reaches inbound context, but the session metadata, gateway row, and UI display helpers do not consume it for the selector label. I did not run a live Telegram or UI repro in this read-only review.

## Solution Assessment

Yes. The best fix is a generic session/display-label projection for known thread names rather than Telegram-specific string parsing in the UI or a new user config option.
