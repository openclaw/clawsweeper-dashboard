## Summary

Keep open. Current main still learns Telegram forum topic names and exposes `TopicName` to inbound/prompt metadata, but it does not carry that known name into persisted session display metadata, gateway session rows, or the Control UI session selector.

## What This Changes

The issue requests human-readable Telegram forum topic names in the Control UI session dropdown instead of raw or ambiguous Telegram topic session labels.

## Best Possible Solution

Add a generic explicit thread/session display-label path, populate it from Telegram cached `TopicName` for forum-topic sessions, expose it through `sessions.list`, and have the Control UI prefer that explicit label without parsing Telegram keys in the UI.

## Reproduction Assessment

Yes, source-level: Telegram forum messages can set `TopicName`, but current session metadata, gateway rows, and the Control UI selector never consume it as a display label. I did not run a live Telegram/UI repro in this read-only pass.

## Solution Assessment

Yes, the best fix is a generic explicit thread/session label flow, likely through the existing `ThreadLabel`/`displayName` path, rather than a Telegram-specific UI parser or raw session-key formatting.
