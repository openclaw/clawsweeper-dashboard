## Summary

Keep open: current main and the latest release still learn Telegram forum topic names, but the learned value stops at inbound context metadata and is not projected into stored session display metadata, Gateway session rows, or the Control UI selector.

## What This Changes

The issue requests human-readable Telegram forum topic names in the session dropdown instead of raw Telegram topic session keys.

## Best Possible Solution

Carry the already-known Telegram TopicName through a generic session display/thread-label metadata path, expose it through sessions.list, and make the Control UI prefer that explicit label while preserving existing non-Telegram labels.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but session metadata, Gateway rows, and the UI selector do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, the maintainable path is to carry the already-known topic label through a generic session display/thread-label seam. A Telegram-only UI formatter or new per-topic config name field would create a weaker source of truth.
