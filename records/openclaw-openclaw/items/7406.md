## Summary

Keep open. Current main still learns Telegram forum topic names but does not pass them into the existing `ThreadLabel`/session-display path, and the open linked PR is the active fix candidate rather than a reason to close this issue early.

## What This Changes

The issue requests human-readable Telegram forum topic names in session dropdown labels instead of raw topic IDs.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/94303; once a fix merges to main, this issue can close as implemented with release/main provenance.

## Reproduction Assessment

Yes, from source inspection: current main can obtain and expose `TopicName`, but the display path consumes `ThreadLabel`, `label`, or `displayName`, so the topic name is not shown by the stock session selector path. I did not run a live Telegram forum group reproduction in this read-only review.

## Solution Assessment

Yes, the best narrow path is to feed the already-known Telegram topic name into the existing generic `ThreadLabel`/session-display seam instead of adding Telegram-specific UI parsing or new config. The open linked PR is the current candidate for that path.
