## Summary

Current main still knows Telegram forum topic titles but does not carry them into the stored display metadata consumed by the Control UI selector. This remains the canonical, vision-aligned tracker; the direct implementation PRs are closed unmerged and no open PR currently owns the fix.

## What This Changes

Show known Telegram forum topic names in the Control UI session selector instead of indistinguishable group-only or raw-key labels.

## Best Possible Solution

Populate the existing generic `ThreadLabel` from the known Telegram forum topic title, let the canonical session path persist it as `displayName`, and verify that `sessions.list` and the Control UI selector preserve it without changing existing fallback labels.

## Reproduction Assessment

Yes, at source level with high confidence: Telegram writes the known title to `TopicName`, session persistence promotes only `ThreadLabel`, and the selector reads persisted `label` or `displayName`. A real Telegram forum remains the appropriate final behavioral verification.

## Solution Assessment

Yes, with a narrower implementation than originally proposed: reuse the existing generic `ThreadLabel` to `displayName` path rather than adding a parallel `topicName` session field, new configuration, or Telegram-specific formatting in core.
