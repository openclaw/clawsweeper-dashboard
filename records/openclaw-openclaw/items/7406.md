## Summary

Keep open: current main already learns Telegram forum topic names, but the name still stops at inbound context/prompt/hook metadata and is not carried into stored session display metadata, gateway session rows, or the Control UI session selector.

## What This Changes

The issue requests human-readable Telegram forum topic names in the Control UI session dropdown instead of raw topic IDs.

## Best Possible Solution

Implement a generic session display/thread-label metadata path that Telegram fills from cached TopicName, expose it through sessions.list, and make the Control UI prefer that explicit label while preserving existing labels and non-Telegram behavior.

## Reproduction Assessment

Yes, source-level: current main populates TopicName for Telegram forum topics, but the session metadata, gateway row, and UI selector code do not consume it. I did not run live Telegram or browser proof because this review is read-only.

## Solution Assessment

Yes, but not as a Telegram-specific UI formatter. The maintainable solution is a narrow generic display-label or thread-label seam populated by the Telegram plugin from cached TopicName.
