## Label Justifications

- `P2`: The sessions remain usable, but operators with multiple Telegram forum topics cannot reliably distinguish them in the selector.
- `impact:session-state`: Distinct persisted topic sessions lack complete human-readable identity in session metadata and listing state.

## Summary

Current main caches Telegram forum topic names for inbound context but still does not propagate them into canonical session display metadata or the Control UI selector; no open replacement PR or issue owns the remaining work.

## What This Changes

Add human-readable Telegram forum topic names to session dropdown labels instead of showing raw topic identifiers.

## Best Possible Solution

Persist Telegram’s known topic title through one generic canonical thread/session label, expose it through gateway session rows, and let the existing UI label preference display it while retaining current fallbacks.

## Reproduction Assessment

Yes. Current source obtains and caches the Telegram topic title, while the stored session and selector path only carries generic label fields that are not populated from that title.

## Solution Assessment

Yes in direction, provided the implementation uses a generic session/thread-label contract; Telegram-specific formatting in core or a new configuration option would be less maintainable.
