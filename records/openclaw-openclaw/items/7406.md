## Summary

Keep open: current main and v2026.5.12 still learn Telegram forum TopicName, but that value is not bridged into persisted session display metadata, gateway rows, or the Control UI selector. The linked UI cleanup PRs are adjacent rather than complete fixes, and the duplicate reports were already closed in favor of this canonical tracker.

## What This Changes

Show Telegram forum topic sessions in the session dropdown with human-readable topic names instead of raw topic IDs.

## Best Possible Solution

Carry known Telegram topic names through a generic session thread/display label in session metadata, gateway rows, and selector rendering while preserving existing non-Telegram labels.

## Reproduction Assessment

Yes, source-level. Current main can populate Telegram TopicName for forum messages, but the session metadata, gateway row, and Control UI selector paths do not consume it for visible labels.

## Solution Assessment

Yes. The narrow maintainable direction is a generic session thread/display-label bridge; UI-only key formatting or a Telegram-specific parser in core would leave stored metadata incomplete.
