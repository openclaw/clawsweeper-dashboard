Keep open. Current main and the latest release still learn Telegram forum topic names as `TopicName`, but they do not pass that value into the existing `ThreadLabel`/session-display path, and all linked fix attempts are closed unmerged.

## What This Changes

The issue requests human-readable Telegram forum topic names in the session dropdown instead of raw or numeric Telegram topic session labels.

## Best Possible Solution

Use the existing Telegram topic-name cache and generic `ThreadLabel`/`displayName` seam so known forum topic names become stored session labels without new config or Telegram-specific UI parsing.

## Reproduction Assessment

Yes, from source inspection: current main obtains and exposes `TopicName`, while the visible session path consumes `ThreadLabel`, `displayName`, or `label`. I did not run a live Telegram forum group in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to feed the already-known Telegram topic name into the existing generic `ThreadLabel`/session-display seam, not add new config or parse Telegram keys in the UI.
