## Best Possible Solution

Land the narrow SDK normalization after the draft is ready and exact-head SDK unit/e2e or CI validation confirms the helper wire shapes, while keeping generic request semantics unchanged.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? ## Solution Assessment

Is this the best way to solve the issue? Yes, this is an acceptable narrow fix because it normalizes only the affected no-arg list helpers to the object-param contract and does not change generic request forwarding or mask explicit malformed `null` values.
