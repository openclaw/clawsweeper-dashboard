## Summary

Keep open. The PR is a focused backup retry change and matches the node-tar ENOENT failure path, but it still needs real behavior proof from a backup run where a planned state file disappears during archive creation; the current proof only shows a normal backup and mocked retry tests.

## What This Changes

The PR adds ENOENT-with-path detection to the backup tar retry wrapper and unit tests for retrying that error class.

## Best Possible Solution

Land a backup archiver fix after real proof shows backup creation survives a planned state file disappearing mid-archive; if retries are insufficient, switch to per-entry skip/reporting.

## Reproduction Assessment

Yes for source-level reproduction: current main calls tar.c over live state roots, tar 7.5.16 emits lstat errors before the filter can skip a vanished path, and linked issue comments confirm released failures. I did not run a live timing race in this read-only review.

## Solution Assessment

Unclear. Extending the existing retry wrapper is the narrowest code change, but issue comments and current tests leave per-entry skip/reporting as a possibly more robust fix for repeated live churn.
