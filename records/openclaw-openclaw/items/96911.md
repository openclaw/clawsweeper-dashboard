## Summary

Keep open. The PR is a focused candidate fix for the open backup ENOENT race, but the current proof is only a normal backup run plus mocked retry coverage, not an after-fix run where a planned state file disappears during archive traversal.

## What This Changes

The PR updates backup archive retry handling to treat tar ENOENT errors with a path as retryable and adds focused unit coverage for that retry path.

## Best Possible Solution

Land a backup archiver fix after real proof shows backup creation survives a planned state file disappearing mid-archive; if retry-only behavior is not robust enough, switch to per-entry ENOENT skip/reporting in the archive layer.

## Reproduction Assessment

Yes for source-level reproduction: current main sends live state roots into node-tar and only retries EOF-class errors, while the linked issue includes release-confirmed ENOENT failures from disappearing state files. I did not run a live timing race in this read-only review.

## Solution Assessment

Unclear. Extending the existing retry wrapper is the narrowest code change, but the related issue’s repeated-churn reports leave per-entry skip/reporting as a potentially better archive-layer fix.
