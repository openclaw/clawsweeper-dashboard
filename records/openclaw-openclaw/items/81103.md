## Summary

Keep open: current main still leaves Feishu JSON-derived `file_name` values on the raw path that bypasses CJK mojibake recovery, and https://github.com/openclaw/openclaw/pull/81125 is an open closing PR for the same fix.

## What This Changes

Recover CJK Feishu inbound filenames that arrive through JSON `file_name` metadata rather than the already-fixed Content-Disposition header path.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81125; the durable fix should reuse the conservative Feishu recovery helper for JSON and fallback filename paths while preserving valid Latin-1 names in tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the JSON `file_name` path bypasses the recovery helper, and the focused Feishu media test currently asserts the mojibake value is preserved.

## Solution Assessment

Yes. A narrow Feishu-plugin fix that reuses the existing conservative CJK recovery heuristic and updates the focused tests is the maintainable path; the linked open PR is the right place to resolve it before closing this issue.
