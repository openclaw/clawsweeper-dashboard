## Summary

Keep this PR open, but it is not mergeable as-is. The latest head still adds a security-sensitive install/update code-execution surface with env-policy gaps, unstable archive `SKILL_DIR` semantics, mock-only proof, and unrelated generated Canvas drift; the linked feature request also remains active with a plugin-shaped alternative under review.

## What This Changes

This PR adds `metadata.openclaw.setup.script` parsing and runs skill-supplied setup scripts after archive/ClawHub and metadata-based skill install paths, with tests, a changelog entry, a restricted command-env option, and a Canvas bundle hash change.

## Best Possible Solution

Pick one canonical setup-hook contract, reuse or expose the existing skills/frontmatter/env-safety helpers, define stable install/update timing and trust semantics, document the public contract, and require redacted real install/update proof before merge.

## Reproduction Assessment

Yes for the PR defects by source inspection: the env forwarding path, archive staging path, unused test local, and unrelated generated hash are visible at the PR head. For the linked setup-hook request itself, reproduction is not applicable because it adds a new capability.

## Solution Assessment

No. The feature direction may be reasonable, but this direct core auto-execution patch is not the best mergeable solution until env-policy parity, stable path/timing semantics, docs/security contract, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Filter setup env through the skill env policy:** `src/agents/skills-setup.ts:112-115`
  - body: `runSkillSetupHook` copies every declared `requires.env` name from `process.env` into the setup script environment. A skill can declare execution-affecting keys such as `NODE_OPTIONS`, `BASH_ENV`, or `OPENSSL_CONF` and bypass the dangerous env filtering already used for skill env overrides.
  - confidence: 0.93
- **[P2] Run archive hooks with a stable installed path:** `src/agents/skills-archive-install.ts:184-187`
  - body: Archive installs invoke the hook from `installPackageDir.afterInstall`, which receives the temporary stage directory before it is moved to the canonical skill path. Scripts that persist `SKILL_DIR`, write external config, or register paths can store a path that no longer exists after install succeeds.
  - confidence: 0.88
- **[P2] Remove the unused timeout local:** `src/agents/skills-setup.test.ts:34`
  - body: The added test helper computes `timeout` and never reads it. Core test typechecking enables unused-local checks, so this leaves the new test file unable to pass the normal typecheck lane.
  - confidence: 0.9
- **[P2] Remove the unrelated Canvas bundle hash drift:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This PR changes only the Canvas A2UI `.bundle.hash` without changing the Canvas bundle inputs or generated output. That generated drift is unrelated to setup hooks and can be rewritten by package/build validation.
  - confidence: 0.82
- **[P3] Document the setup hook contract before exposing it:** `src/agents/skills/types.ts:35`
  - body: The PR adds a public `metadata.openclaw.setup` field, but the skills docs still describe only installer specs. The timing, env forwarding, timeout ownership, failure/rollback behavior, and trust model need documentation before this becomes a skill-author surface.
  - confidence: 0.8
