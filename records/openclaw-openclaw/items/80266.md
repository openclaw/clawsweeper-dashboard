## Summary

This PR should stay open but is not ready to merge: it implements a plausible new skill setup-hook feature, but the diff leaks ambient host environment variables to skill scripts, can remove the existing skill on failed updates, only wires archive installs, and lacks real behavior proof.

## What This Changes

Adds `metadata.openclaw.setup.script` parsing, a setup-hook runner, archive install/update invocation, tests, and a changelog entry.

## Best Possible Solution

Define the setup hook as a constrained lifecycle contract with a controlled base environment, scanner/trust policy, rollback-safe install composition, docs, and coverage for both archive installs and metadata installer runs.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the runner inherits ambient env, runs after archive publish/backup cleanup, and is only wired into archive install. As a feature request, current-main bug reproduction is otherwise not applicable.

## Solution Assessment

No. The direction may be viable, but this implementation needs env confinement, rollback-safe composition, existing installer coverage, docs, real behavior proof, and maintainer security review before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Pass only declared setup env to hook scripts:** `src/agents/skills-setup.ts:118-127`
  - body: The setup runner builds `scriptEnv` from `process.env`, and the process helper also merges command env over ambient `process.env`. Any installed skill with `setup.script` can therefore read unrelated gateway secrets instead of only `SKILL_DIR`, `OPENCLAW_HOOK_KIND`, and allowlisted `requires.env` values.
  - confidence: 0.97
- **[P2] Keep update rollback intact until setup succeeds:** `src/agents/skills-archive-install.ts:195-197`
  - body: On update, `installPackageDir` has already published the new tree and removed its backup before this setup failure path runs. If the hook fails, this cleanup removes `targetDir`, leaving the user with neither the previous skill nor the updated one; run the hook inside the install rollback window or preserve the backup until setup succeeds.
  - confidence: 0.92
- **[P2] Invoke setup from the existing skills.install flow:** `src/agents/skills-archive-install.ts:190-194`
  - body: This only invokes the hook after archive installs. The existing `installSkill()` path that runs `metadata.openclaw.install` still returns immediately after the installer command, so a skill installed through the current dependency-install flow will never run its declared setup script.
  - confidence: 0.88
- **[P3] Document the setup metadata contract:** `src/agents/skills/types.ts:19-22`
  - body: The PR adds a new public `SKILL.md` metadata field, timeout, failure behavior, and env contract, but only updates the changelog. Skill authors need the public skills documentation updated before this API ships so the supported shape and security expectations are clear.
  - confidence: 0.82
