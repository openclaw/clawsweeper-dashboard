## Summary

Keep this PR open for normal review, not cleanup-close it. It is a recent active implementation for an open feature request, but the patch still has blocking security/env-policy, archive install semantics, documentation, generated-file, and contributor proof issues.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing and setup-hook execution for skill install/archive flows, adds tests and a changelog entry, extends `runCommandWithTimeout` with `baseEnv`, and changes an unrelated Canvas A2UI bundle hash.

## Best Possible Solution

Land one maintainer-approved setup-hook contract that matches existing env sanitization, has stable install/update and rollback semantics, documents timing/failure/trust behavior, removes unrelated generated drift, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner bypasses existing env-policy checks, archive hooks receive the staging directory, docs are missing for the new public contract, and the Canvas hash drift is visible in the diff. For the linked request itself, bug reproduction is not applicable because it adds a new capability.

## Solution Assessment

No. The direction may be viable, but this patch is not the best mergeable solution until env-policy parity, stable `SKILL_DIR` semantics, docs/security contract, generated-file cleanup, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Filter setup env through the skill env policy:** `src/agents/skills-setup.ts:112-115`
  - body: `runSkillSetupHook` copies every declared `requires.env` name from `process.env` into the hook without reusing the existing skill env sanitizer. A skill can declare execution-affecting keys such as `NODE_OPTIONS` or `OPENSSL_CONF` and receive values that current skill env handling intentionally blocks.
  - confidence: 0.94
- **[P2] Avoid exposing staged SKILL_DIR to archive hooks:** `src/agents/skills-archive-install.ts:184-187`
  - body: Archive installs run the setup hook from `installPackageDir.afterInstall`, which receives the temporary staging directory before it is moved to the final install target. Scripts that persist or report `SKILL_DIR` will capture a path that disappears after success instead of the installed skill directory promised by the contract.
  - confidence: 0.88
- **[P2] Remove the unrelated Canvas bundle hash drift:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This PR changes only the Canvas A2UI `.bundle.hash` without changing the A2UI bundle inputs or output. The bundler derives that file from package, lockfile, and A2UI app sources, so validation can rewrite it back and leave generated drift unrelated to the setup-hook feature.
  - confidence: 0.82
- **[P3] Document the setup hook contract before exposing it:** `src/agents/skills/types.ts:38`
  - body: The patch adds public `metadata.openclaw.setup` fields without documenting timing, env forwarding, timeout ownership, failure behavior, rollback semantics, or the trust model in the skill docs. Skill authors and operators need that contract before this becomes an exposed SKILL.md surface.
  - confidence: 0.82
