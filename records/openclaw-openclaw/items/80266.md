## Summary

Keep open: this PR adds a new skill setup-hook capability that is not implemented on current main, but the patch still has blocking lifecycle and env-boundary issues plus only test/check proof. The related open PR at https://github.com/openclaw/openclaw/pull/80276 tracks the same request, but neither PR has merged or been chosen as canonical.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing, setup-hook execution for skill archive and metadata installs, restricted command environment support, tests, and a changelog entry.

## Best Possible Solution

Define the setup-hook lifecycle deliberately: durable `SKILL_DIR`, rollback semantics, sanitizer reuse, ClawHub and metadata installer coverage, docs for trust/env behavior, and real CLI/archive proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the archive hook receives `stageDir` before publish, and setup env forwarding bypasses the existing sanitizer path. As a feature request against current main, a bug reproduction is not applicable.

## Solution Assessment

No. The direction may be viable, but this patch is not yet the narrowest maintainable solution because lifecycle ordering, env sanitization, trust policy, docs, and real behavior proof still need work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter setup env through the skill sanitizer:** `src/agents/skills-setup.ts:105-107`
  - body: `runSkillSetupHook` copies every declared `requires.env` value straight from `process.env`. Current skill env injection blocks host execution and credential override names through the sanitizer/host-env policy, so this new setup path can forward values such as `GITHUB_TOKEN`, `NODE_OPTIONS`, or package-manager override vars if a skill lists them.
  - confidence: 0.88
- **[P2] Run archive hooks with the durable skill directory:** `src/agents/skills-archive-install.ts:184-186`
  - body: `installPackageDir` invokes `afterInstall` with the temporary staging directory before publishing it to the final target. Passing that path as `SKILL_DIR` means archive and ClawHub setup scripts can persist a `.openclaw-install-stage-*` path into vault/config state, which is wrong after install or update succeeds.
  - confidence: 0.9
- **[P3] Document the setup hook contract before exposing it:** `src/agents/skills/types.ts:35`
  - body: This adds a new author-facing `SKILL.md` lifecycle field and env/trust contract, but the PR only updates code and changelog. Skill authors need docs for when the hook runs, what env is forwarded, rollback behavior, failure handling, and the security model before this manifest surface ships.
  - confidence: 0.78
