The PR adds `metadata.openclaw.setup.script` parsing and post-install/update skill setup execution, with tests, changelog text, env isolation changes, and an unrelated Canvas bundle hash update.

## Best Possible Solution

Define one documented setup-hook lifecycle that reuses existing env sanitizer rules, provides durable `SKILL_DIR` semantics with rollback-safe archive installs, includes skill-author docs, and lands through one canonical PR with real install/update proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Filter setup env through the host env policy:** `src/agents/skills-setup.ts:112-115`
  - body: `runSkillSetupHook` copies every declared `requires.env` value from `process.env` into the setup hook environment. Current skill env handling blocks dangerous names such as `NODE_OPTIONS` and `OPENSSL_CONF` even when declared, so this install-time path needs the same sanitizer before spawning third-party setup scripts.
  - confidence: 0.92
- **[P2] Pass archive hooks a durable skill directory:** `src/agents/skills-archive-install.ts:185-187`
  - body: For archive/ClawHub installs, `installPackageDir` invokes `afterInstall` with the temporary `.openclaw-install-stage-*` directory before publishing. Passing that as `targetDir` makes `SKILL_DIR` transient, so setup scripts that persist the skill path can write a location that is removed or moved after install.
  - confidence: 0.9
- **[P2] Remove the unrelated Canvas bundle hash change:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This PR changes only the Canvas A2UI bundle hash without changing Canvas bundle inputs. That unrelated generated-asset drift can make bundle validation expect a bundle state this setup-hook PR does not produce.
  - confidence: 0.82
- **[P2] Remove the unused setup timeout variable:** `src/agents/skills-setup.test.ts:34`
  - body: `timeout` is declared but never read in the new test helper. Core test typechecking has `noUnusedLocals` enabled, so this keeps the changed test surface from typechecking cleanly.
  - confidence: 0.85
- **[P3] Document the setup hook contract before exposing it:** `src/agents/skills/types.ts:38`
  - body: This adds a new author-facing `metadata.openclaw.setup` field, but the skill docs still list only the existing metadata and installer fields. Skill authors need the timing, env, failure, rollback, and security model documented before this surface ships.
  - confidence: 0.78
