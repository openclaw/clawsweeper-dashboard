## Summary

Keep open: current main still lacks the requested skill-authored setup hook, but this PR is not merge-ready because the branch still has merge-blocking env policy, archive install path, unrelated generated drift, documentation, proof, and security-contract gaps.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing and setup-hook execution after skill install/update flows, plus tests, a restricted command `baseEnv` option, a changelog entry, and a Canvas bundle hash change.

## Best Possible Solution

Land one canonical setup-hook contract only after it matches existing env safety policy, uses stable installed-directory semantics, documents trust and failure behavior, removes unrelated generated drift, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner bypasses the existing env policy, archive hooks receive the staging directory, and the branch changes unrelated generated Canvas state. For the linked request itself, reproduction is not applicable because it is a new feature request.

## Solution Assessment

No. The feature direction may be reasonable, but this patch is not the best mergeable solution until env-policy parity, stable `SKILL_DIR` semantics, public docs/security contract, generated-file cleanup, and real behavior proof are addressed.

  - body: `runSkillSetupHook` copies every declared `requires.env` value directly from `process.env` into the hook.   - body: Archive installs call the setup hook from `installPackageDir.afterInstall`, whose argument is the temporary `.openclaw-install-stage-*` directory before publish. Scripts that persist or report `SKILL_DIR` get a path that is removed after success instead of the installed skill directory promised by the contract.
  - confidence: 0.9

  - body: This PR changes only the Canvas A2UI `.bundle.hash` without changing Canvas bundle inputs or output. That generated file is unrelated to skill setup and can be rewritten by package/build validation, leaving avoidable generated drift in the branch.
  - confidence: 0.84
- **[P3] Document the setup hook contract:** `src/agents/skills/types.ts:38`
  - body: The PR adds public `metadata.openclaw.setup` metadata without documenting timing, env forwarding, failure behavior, rollback semantics, timeout ownership, or the trust model. Skill authors need that contract before relying on an install-time execution surface.
  - confidence: 0.82
