## Best Possible Solution

Land only after every setup-hook entry point preserves the same config/env/secret contract, focused regression coverage proves the gateway installer action path, and maintainers explicitly accept the core setup-hook API over the plugin/RPC alternative.

## Solution Assessment

No, not yet. The default-disabled opt-in approach is a plausible implementation, but the non-download installer action path does not preserve the setup env contract and maintainers still need to approve the core-vs-plugin owner boundary.
