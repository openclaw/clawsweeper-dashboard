## Summary

Keep open: the linked feature gap is real, but this PR still has blocking correctness and security issues, lacks real install/update behavior proof, and overlaps with another open implementation that maintainers should consolidate.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing, a setup-hook runner invoked from skill install/archive flows, tests, a changelog entry, and a `runCommandWithTimeout` base-env option.

## Best Possible Solution

Define one documented setup-hook lifecycle with host-env sanitizer parity, durable installed-path semantics, rollback-safe install/update behavior, ClawHub/archive plus metadata-installer coverage, and real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner copies declared env from `process.env`, the archive callback receives a staging directory, and the PR head contains CI-blocking drift. For the linked request itself, reproduction is not applicable because it is a new capability.

## Solution Assessment

No. The direction may be viable, but this patch is not yet the best merge candidate because the env policy, durable path contract, docs, real behavior proof, and product/security trust model still need work.
