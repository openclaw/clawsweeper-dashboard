## Summary

## Best Possible Solution

Maintainers should first choose the setup contract; if core auto-run is accepted, implement it with typed `baseEnv` forwarding, shared host env sanitization, realpath confinement, final target semantics, docs, and redacted real install/update proof, otherwise continue with the admin-only plugin/ClawHub path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire baseEnv through the exec helper:** `src/agents/skills-setup.ts:115`
  - body: `baseEnv: {}` is passed here to prevent setup scripts from inheriting ambient process secrets, but `CommandOptions` does not declare it and `runCommandWithTimeout` currently calls `resolveCommandEnv({ argv, env })` without forwarding any base env. This should fail type checking and, if forced through, still gives the child process the default `process.env`; add `baseEnv` to the command options, pass it into `resolveCommandEnv`, and cover that a hook cannot see unrelated env keys.
  - confidence: 0.94
- **[P1] Filter setup env through the host policy:** `src/agents/skills-setup.ts:97-101`
  - body: Copying every declared `requires.env` value directly from `process.env` bypasses the existing host execution env policy that rejects dangerous or invalid names such as loader and override variables. Setup hooks should reuse the same sanitizer/diagnostics path before forwarding declared values to a child process.
  - confidence: 0.9
- **[P1] Resolve script realpaths before executing:** `src/agents/skills-setup.ts:61-72`
  - body: The containment check compares only the lexical `path.resolve()` result, then `accessSync` and spawn follow symlinks. A skill can declare an inside-looking symlink such as `scripts/setup.sh` that points outside the skill directory, so resolve both the skill dir and script realpaths and assert the canonical target remains inside before execution.
  - confidence: 0.88
- **[P2] Avoid passing a staging SKILL_DIR for archive hooks:** `src/agents/skills-archive-install.ts:181`
  - body: `installPackageDir` invokes `afterInstall` with the staging directory before publishing it to the canonical target, so ClawHub archive setup scripts receive a temporary `SKILL_DIR`. Any script that persists or registers that path will record a dead location after install; run setup with final-path semantics or explicitly stage only file-local writes before publishing.
  - confidence: 0.84

## Security Review

Status: needs_attention
