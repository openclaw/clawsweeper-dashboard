## What This Changes

The PR adds skill setup hook metadata parsing and post-install/update execution, plus tests, changelog text, and a restricted process-env option.

## Best Possible Solution

Define one documented setup-hook lifecycle that reuses the existing env sanitizer, provides stable `SKILL_DIR` semantics while preserving rollback, and lands through one canonical PR with real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner copies declared env names directly from `process.env`, and archive setup receives `installPackageDir`'s staging directory. For the linked feature request itself, reproduction is not applicable because it adds a new capability.

## Solution Assessment

No. The setup-hook direction may be viable, but this patch is not the best mergeable solution until sanitizer parity, durable path semantics, documentation, and real install/update proof are addressed.
