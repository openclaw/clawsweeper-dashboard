## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire baseEnv through the exec helper:** `src/agents/skills-setup.ts:111-116`
  - body: `baseEnv` is passed here to prevent setup scripts from inheriting ambient process secrets, but `CommandOptions` does not declare it and `runCommandWithTimeout` currently calls `resolveCommandEnv({ argv, env })` without forwarding any base env. This should fail type checking and, if forced through, still gives the child process the default `process.env`; add `baseEnv` to the command options, pass it into `resolveCommandEnv`, and cover that a hook cannot see unrelated env keys.
  - confidence: 0.94
- **[P1] Filter setup env through the host policy:** `src/agents/skills-setup.ts:91-100`
  - body: Copying every `requires.env` name directly from `process.env` bypasses the existing host execution env policy that blocks keys such as `NODE_OPTIONS`, `LD_*`, `HOME`/override-style variables, and invalid names. Setup hooks should reuse the same sanitizer/diagnostics path as other host command env handling before forwarding declared values.
  - confidence: 0.9
- **[P1] Resolve script realpaths before executing:** `src/agents/skills-setup.ts:61-72`
  - body: The containment check only compares the lexical `path.resolve()` result, then `accessSync`/spawn will follow symlinks. A downloaded or archived skill can declare `scripts/setup.sh` as a symlink to a file outside the skill directory and still pass this check, so resolve the script and skill directory realpaths and assert the canonical target stays inside the skill before running it.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch adds install/update-time script execution and currently has concrete env-isolation, env-policy, and path-confinement security defects.

  - body: `baseEnv: {}` is passed from the new setup runner, but the current exec helper neither types nor forwards that option, so setup scripts can still inherit process environment if the type error is bypassed.

  - body: The setup runner forwards declared `requires.env` values directly from `process.env` instead of using the existing sanitizer that blocks dangerous host-execution env keys and prefixes.
  - confidence: 0.9
