## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Wire env isolation into the exec helper:** `src/agents/skills-setup.ts:134`
  - body: `baseEnv: {}` is passed to `runCommandWithTimeout`, but `CommandOptions` does not include that property and the helper still calls `resolveCommandEnv({ argv, env })`. This fails strict typechecking and, if bypassed, still merges ambient `process.env` into setup scripts.
  - confidence: 0.96
- **[P1] Filter setup env through host policy:** `src/agents/skills-setup.ts:112-114`
  - body: The hook copies every declared `requires.env` name from `process.env`. Existing host-env policy blocks execution-sensitive names such as `NODE_OPTIONS`, `LD_*`, and Git override variables, so setup hooks need the same filtering and diagnostics before values reach a child process.
  - confidence: 0.94
- **[P2] Run archive hooks against the final skill path:** `src/agents/skills-archive-install.ts:184-186`
  - body: `installPackageDir.afterInstall` receives the temporary staging directory before publish, so archive and ClawHub setup scripts see `SKILL_DIR` as `.openclaw-install-stage-*` rather than the installed skill directory. Anything that persists or reports that path will break after publish.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR adds install/update-time script execution and still has concrete env-isolation and env-policy gaps.

Concerns:

- **[high] Ambient env can still leak to setup scripts:** `src/agents/skills-setup.ts:134`
  - body: The setup runner relies on `baseEnv: {}`, but the current exec options do not accept that field and `runCommandWithTimeout` does not forward it to `resolveCommandEnv`, so ambient process env would still be merged into the child process.
  - confidence: 0.96
- **[high] Declared setup env bypasses blocked host env names:** `src/agents/skills-setup.ts:112`
  - body: Declared `requires.env` values are copied directly from `process.env`, allowing execution-affecting names that existing host-env policy blocks to reach third-party setup scripts.
  - confidence: 0.94
- **[medium] Install-time script execution contract is unsettled:** `src/agents/skills-setup.ts:130`
  - body: Automatically running skill-supplied scripts during install/update is a new local code-execution capability, so consent, trust, logging/redaction, rollback, and docs need explicit maintainer approval before shipping.
