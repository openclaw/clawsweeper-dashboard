## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate setup hooks before auto-running them:** `src/agents/skills-install.ts:526-530`
  - body: This new call runs a skill-supplied setup script after any successful gateway dependency install, but this path does not have the `allowSetupHooks` gate used for archives or another maintainer-approved trust decision. A skill can therefore start executing bundle code and forwarded `requires.env` values during dependency install; keep this disabled until the core/plugin/admin setup model is explicit.
  - confidence: 0.9
- **[P1] Wire hooks into the actual install/update path:** `src/agents/skills-archive-install.ts:186-198`
  - body: The archive hook is hidden behind `allowSetupHooks`, but the current ClawHub/upload/local install callers never set that flag, and the ClawHub update path also goes through that caller. As a result, the user path this feature targets still ignores `metadata.openclaw.setup`; either wire the approved path with tests or narrow the PR away from claiming install/update support.
  - confidence: 0.88
- **[P1] Reuse the shared host environment sanitizer:** `src/agents/skills-setup.ts:115-153`
  - body: This local blocklist does not match the existing host env security policy; for example shell-init variables such as `BASH_ENV`/`ENV` are covered by the shared sanitizer but can still be forwarded here if a skill lists them in `requires.env`. Setup hooks should build env through the existing sanitizer or an equivalent shared allowlist so new local execution does not drift from the hardened exec boundary.
  - confidence: 0.86
- **[P2] Run archive setup after publishing the stable target:** `src/agents/skills-archive-install.ts:186-198`
  - body: `installPackageDir` invokes `afterInstall` on the staging directory before it is moved into the final skill location, so this callback gives setup scripts a temporary `SKILL_DIR`. Scripts that seed external state or write absolute paths can persist the staging path or run side effects before the update has actually published; move setup to a post-publish stable-target hook if archive installs are enabled.
  - confidence: 0.8
