## Summary

Keep open: current main does not implement the requested setup-hook feature, and the latest PR head still has blocking env-policy, archive install semantics, typecheck, generated-file, documentation, security, and real-behavior-proof gaps.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing and setup-hook execution after skill install/update flows, with tests, a changelog entry, a restricted command `baseEnv` option, and an unrelated Canvas A2UI bundle hash change.

## Best Possible Solution

Land one canonical setup-hook contract only after it matches the existing env safety policy, gives stable install/update semantics, documents the public trust model, removes unrelated generated drift, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner forwards declared env names directly, archive hooks receive the staging directory, the added test has an unused local, and the branch changes a generated Canvas hash without Canvas inputs. For the linked feature request itself, reproduction is not applicable because it adds a new capability.

## Solution Assessment

No. The feature direction may be reasonable, but this patch is not the best mergeable solution until env-policy parity, stable `SKILL_DIR` semantics, docs/security contract, generated-file cleanup, and real behavior proof are addressed.

Full review comments:

  - body: `runSkillSetupHook` copies every declared `requires.env` name from `process.env` into the hook. Existing skill env handling blocks execution-affecting keys such as `NODE_OPTIONS` and `OPENSSL_CONF`, so setup hooks can currently bypass that policy.
  - confidence: 0.94
- **[P2] Keep SKILL_DIR stable for archive hooks:** `src/agents/skills-archive-install.ts:184-187`
  - body: Archive installs call the setup hook from `installPackageDir.afterInstall`, which receives the temporary `.openclaw-install-stage-*` directory before publish. Scripts that persist or report `SKILL_DIR` get a path that is removed after success instead of the installed skill directory promised by the feature contract.
  - confidence: 0.9
- **[P2] Remove the unused timeout local:** `src/agents/skills-setup.test.ts:34`
  - body: The added test helper computes `timeout` and never reads it. Core test typechecking enables `noUnusedLocals`, so this leaves the new test surface unable to typecheck cleanly.
  - confidence: 0.9
- **[P2] Remove the unrelated Canvas bundle hash drift:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This PR changes only the Canvas A2UI `.bundle.hash` without changing the bundle inputs or output. The Canvas bundler derives that file from package, lockfile, and A2UI app sources, so build/package validation can rewrite it back and leave generated drift.
  - confidence: 0.84
- **[P3] Document the setup hook contract before exposing it:** `src/agents/skills/types.ts:38`
  - body: The PR adds the public `metadata.openclaw.setup` field without documenting timing, env forwarding, failure behavior, rollback semantics, or the trust model in the skills docs.
  - confidence: 0.8
