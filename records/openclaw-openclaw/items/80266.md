## Summary

Keep open: the PR is a plausible implementation of the requested skill lifecycle feature, but the current patch still has correctness and security blockers plus insufficient real behavior proof for a new install-time code execution path.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing and runs skill-supplied setup scripts after skill install or ClawHub update flows with scoped environment forwarding.

## Best Possible Solution

Land one canonical setup-hook implementation only after it reuses the existing env safety policy, gives stable `SKILL_DIR` semantics, documents the public contract, removes unrelated drift, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner forwards declared env names directly, archive hooks run against the staging directory, and the added test helper still has an unused local. For the linked feature request itself, reproduction is not applicable because it adds a new capability.

## Solution Assessment

No. The direction may be acceptable, but this patch is not the best mergeable solution until env policy parity, stable archive path semantics, public docs, unrelated generated-file drift, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Filter setup env through the host env policy:** `src/agents/skills-setup.ts:112-115`
  - body: `runSkillSetupHook` copies declared `requires.env` names straight from `process.env` into third-party setup scripts. That bypasses the existing skill/host env blocking for execution-affecting names like `NODE_OPTIONS` or `OPENSSL_CONF`, so the setup runner should reuse the same sanitizer before spawning.
  - confidence: 0.93
- **[P2] Pass archive hooks a stable skill directory:** `src/agents/skills-archive-install.ts:184-187`
  - body: Archive installs call `runSkillSetupHook` from `installPackageDir`'s `afterInstall`, but that callback receives the temporary `.openclaw-install-stage-*` path before publish. Setup scripts that persist or report `SKILL_DIR` get a path that is moved away instead of the final skill directory promised by the feature contract.
  - confidence: 0.9
- **[P2] Remove the unused timeout helper local:** `src/agents/skills-setup.test.ts:34`
  - body: `timeout` is computed in the new test helper but never read. The core test typecheck uses unused-local checks, so this leaves the added test surface unable to typecheck cleanly.
  - confidence: 0.88
- **[P2] Remove the unrelated Canvas bundle hash change:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This PR changes only the Canvas A2UI bundle hash without changing Canvas bundle inputs. That unrelated generated-file drift can make bundle validation expect a bundle state this setup-hook PR does not produce.
  - confidence: 0.82
- **[P3] Document the setup hook contract before exposing it:** `src/agents/skills/types.ts:38`
  - body: The PR adds a new author-facing `metadata.openclaw.setup` field, but the skill docs do not describe timing, env forwarding, failure behavior, rollback semantics, or the security model. Please document the public contract alongside the type.
  - confidence: 0.78
