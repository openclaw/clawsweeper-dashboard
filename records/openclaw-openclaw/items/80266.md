Full review comments:

- **[P1] Gate setup hooks before auto-running them:** `src/agents/skills-install.ts:526-530`
  - body: This new branch runs any declared `metadata.openclaw.setup.script` after a successful gateway installer action, but unlike the archive helper it has no explicit `allowSetupHooks`/trusted decision.   - confidence: 0.9

  - body: The archive hook only runs when `allowSetupHooks` is passed, but the current ClawHub/upload archive callers do not pass it and `skills.update` still only updates ClawHub slugs or config. A ClawHub skill with `setup.script` therefore will not get the install/update behavior this PR claims to add, so either wire the approved user path end to end or narrow the PR scope.
  - confidence: 0.88
- **[P1] Reuse the shared host environment sanitizer:** `src/agents/skills-setup.ts:115-132`
  - body: This local `BLOCKED_ENV` list misses variables already covered by OpenClaw's shared host-exec policy, such as shell-init and `GIT_*` loader/config keys, and separately re-injects `PATH`. Because setup hooks execute local code on the host, using a parallel blocklist can drift and reopen env-based execution risks that the shared sanitizer already tracks.
