## Summary

Keep open: this PR targets a real missing skill lifecycle feature, but the latest head still has merge blockers around env-policy parity, archive `SKILL_DIR` semantics, public docs, unrelated generated drift, and contributor real behavior proof.

## What This Changes

The PR adds a `metadata.openclaw.setup.script` skill metadata field, parses it from frontmatter, runs the declared script after skill install/archive update paths, and adds setup-hook tests.

## Best Possible Solution

Land one canonical setup-hook design that reuses existing env sanitization, has stable and documented `SKILL_DIR`/rollback semantics, removes unrelated generated drift, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner copies declared env names directly from `process.env`, and archive hooks receive the staging directory from `installPackageDir`. For the linked feature itself, bug reproduction is not applicable because this is a new capability.

## Solution Assessment

No. The direction may be viable, but this diff is not the best mergeable solution until env policy parity, stable path semantics, public docs, unrelated hash drift, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Filter setup env through the host policy:** `src/agents/skills-setup.ts:112-115`
  - body: `runSkillSetupHook` copies every declared `requires.env` name from `process.env` into the child env. Even with `baseEnv: {}`, execution-affecting names that current skill env policy blocks, such as `NODE_OPTIONS` or `OPENSSL_CONF`, can reach a third-party setup script; reuse the existing sanitizer or reject blocked names before spawning.
  - confidence: 0.93
- **[P2] Pass archive hooks a stable skill directory:** `src/agents/skills-archive-install.ts:184-187`
  - body: Archive installs run the setup hook from `installPackageDir`'s `afterInstall` callback, which receives the temporary `.openclaw-install-stage-*` path before publish. Any setup script that persists or registers `SKILL_DIR` will record a path that disappears after install/update.
  - confidence: 0.9
- **[P3] Document the setup hook contract:** `src/agents/skills/types.ts:38`
  - body: The PR adds a public `metadata.openclaw.setup` field, but the skill docs still describe only installer specs. Skill authors need the timing, env forwarding, failure, timeout, rollback, and trust model documented with the new metadata surface.
  - confidence: 0.82
- **[P3] Remove the unrelated Canvas hash change:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This PR changes the Canvas A2UI `.bundle.hash` without changing Canvas bundle inputs. That generated-file drift is unrelated to skill setup hooks and can force or validate a bundle state this branch does not own.
  - confidence: 0.8
