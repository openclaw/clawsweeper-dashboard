## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve setup scripts by realpath before execution:** `src/agents/skills-setup.ts:71-73`
  - body: The containment check only verifies the lexical `path.resolve()` string starts with the skill directory. A skill can place `scripts/setup.sh` as a symlink to a file outside the skill directory and still pass this check, so this new install-time execution path can run code outside the intended bundle. Resolve both the target directory and script to realpaths and require the script realpath to remain inside the skill directory before spawning.
  - confidence: 0.91
- **[P1] Sanitize declared setup environment before spawning:** `src/agents/skills-setup.ts:106-114`
  - body: The loop forwards any declared `requires.env` value from `process.env` into the setup child. That lets a skill declare execution-control names such as `NODE_OPTIONS`, `LD_PRELOAD`, or other host-policy-blocked variables and receive them in a new command-execution path. Reuse the existing host env security/sanitizer policy before adding declared env vars to `scriptEnv`.
  - confidence: 0.88
- **[P2] Do not give archive hooks a disappearing SKILL_DIR:** `src/agents/skills-archive-install.ts:181-183`
  - body: For archive installs, `installPackageDir` invokes `afterInstall(stageDir)` before publishing the stage to the final target. That makes `SKILL_DIR` point at a temporary `.openclaw-install-stage-*` directory that is moved away after success, so setup scripts that seed external config or vault state with the bundle path will persist a stale path. The hook contract needs a stable final path or a different rollback-safe publish flow.
  - confidence: 0.84
- **[P1] Gate automatic ClawHub setup execution behind a trust contract:** `src/agents/skills-archive-install.ts:181`
  - body: This `afterInstall` hook runs for ClawHub/archive installs, while the current ClawHub path passes `scan: false` and docs describe normal skill installs as copying skill folders rather than executing metadata. Automatically running skill-supplied scripts during install/update is a security-boundary change; keep it out of the default path until maintainers approve an admin/consent/config contract and docs, or use the admin-only setup plugin direction.
  - confidence: 0.82
