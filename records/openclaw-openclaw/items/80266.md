## Summary

Keep this PR open, but the patch is not ready to merge: it still has security-sensitive setup-env handling, archive install path semantics, unrelated Canvas generated-state drift, and missing real behavior proof for an actual skill install/update run.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing and runs skill-supplied setup scripts after skill install/update paths, with tests, changelog text, and a process env isolation helper.

## Best Possible Solution

Define one documented setup-hook lifecycle that uses durable skill paths, rollback-safe execution, host-env sanitizer parity, explicit trust/consent policy, ClawHub/archive plus metadata-installer coverage, and real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner still copies declared env values directly from `process.env`, and the archive hook still receives `stageDir` before publish. For the linked request itself, reproduction is not applicable because it is a new capability.

## Solution Assessment

No. The direction may be viable, but this patch is not yet the best maintainable implementation because the security contract, durable path semantics, docs, and real behavior proof still need work.
