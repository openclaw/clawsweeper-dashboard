## Summary

Keep open for maintainer review: the related ClawHub plugin is useful context, but this PR's core auto-run path is a security-boundary change with blocking path/env defects and only test-output proof.

## What This Changes

The PR adds `metadata.openclaw.setup.script` parsing and automatically runs skill-supplied setup scripts after skill download, archive, and command install paths.

## Best Possible Solution

Converge on one setup execution contract, preferably explicit and admin-controlled unless maintainers intentionally approve auto-run installs, then reuse realpath confinement and the existing env security policy with docs and real install/update proof.

For the linked feature request itself, reproduction is not applicable because this is a new capability.

## Solution Assessment

No. The current PR is not the safest maintainable solution until maintainers choose the setup trust boundary; if core auto-run is accepted, the implementation still needs realpath, env policy, final-path, docs, and real behavior proof repairs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Resolve setup scripts by realpath before execution:** `src/agents/skills-setup.ts:71-73`
  - body: The containment check only verifies the lexical `path.resolve()` string starts with the skill directory. A skill can make `scripts/setup.sh` a symlink to a file outside the skill directory and still pass this check, so this new install-time execution path can run code outside the intended bundle.
  - confidence: 0.91
- **[P1] Sanitize declared setup environment before spawning:** `src/agents/skills-setup.ts:106-114`
  - body: The loop forwards any declared `requires.env` value from `process.env` into the setup child. That lets a skill declare execution-control names such as `NODE_OPTIONS`, `BASH_ENV`, or other host-policy-blocked variables and receive them in a new command-execution path.
  - confidence: 0.89
- **[P1] Gate automatic ClawHub setup execution behind a trust contract:** `src/agents/skills-archive-install.ts:184`
  - body: This `afterInstall` hook runs for archive installs, while current docs frame normal ClawHub installs as copying skill folders rather than executing metadata. Automatically running skill-supplied scripts during install/update is a security-boundary change that needs an approved admin, consent, or opt-in contract before defaulting on.
  - confidence: 0.84
- **[P2] Do not give archive hooks a disappearing SKILL_DIR:** `src/agents/skills-archive-install.ts:184-186`
  - body: `installPackageDir` invokes `afterInstall(stageDir)` before publishing the stage to the final target. That makes `SKILL_DIR` point at a temporary install-stage directory, so setup scripts that seed external config or vault state with the bundle path can persist a stale path.
  - confidence: 0.84
