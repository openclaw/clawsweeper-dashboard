## Summary

## What This Changes

The PR threads a derived agent-tool `callerScope` through cron RPC schemas, the agent cron tool, Gateway cron handlers, and tests to limit agent-originated cron reads and mutations to the calling agent while preserving unscoped operator behavior.

## Best Possible Solution

Land the server-side scoping fix with regenerated protocol artifacts committed and an explicit maintainer acceptance of the mixed-version fail-closed behavior for scoped agent-tool cron calls.
