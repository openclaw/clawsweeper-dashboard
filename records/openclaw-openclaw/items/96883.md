## Risks / Open Questions

- The new Gateway connect auth field intentionally fails closed when an older Gateway rejects it, so mixed-version local backend cron calls need maintainer-accepted upgrade behavior.
- OPENCLAW_TOOLS_MCP_AGENT_SESSION_KEY becomes required for the openclaw-tools MCP server path, so unmanaged/direct invocation needs an explicit support or documentation decision.
- The PR changes a model-mediated cron authorization boundary; mistakes could expose another agent's jobs, hide same-agent jobs, or deny valid operations.
- The broader scheduling-authority RFC remains open for playbooks, quotas, expiry policy, provenance receipts, and execution lineage, so this PR is a scoped mitigation rather than the final design.
