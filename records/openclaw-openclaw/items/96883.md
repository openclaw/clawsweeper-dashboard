## What This Changes

The PR scopes agent-originated cron list/read/mutate/run operations to the calling agent by adding callerScope to cron RPCs, deriving it in the cron tool and ACPX MCP bridge, enforcing it in Gateway handlers, and updating tests/generated Swift models.

## Best Possible Solution

Land this only after maintainers accept the callerScope/env contract and the PR body includes redacted real runtime proof for same-agent allowed paths, cross-agent rejected paths, and unchanged unscoped operator behavior; keep the broader provenance design in https://github.com/openclaw/openclaw/issues/71712.

## Solution Assessment

Mostly yes: the agent tool and Gateway are the right layers for deriving and enforcing cron caller scope while preserving unscoped operator behavior. The remaining question is maintainer acceptance of the public callerScope/env contract plus real runtime proof.
