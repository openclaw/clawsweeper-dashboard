- OPENCLAW_TOOLS_MCP_AGENT_SESSION_KEY becomes required for the managed openclaw-tools MCP server path, so direct or unmanaged invocation needs an explicit support or documentation decision.
- The PR changes a model-mediated cron authorization boundary; mistakes here could expose another agent's jobs, hide same-agent jobs, or deny valid operations.
- The broader scheduling-authority RFC remains open for playbooks, quotas, expiry policy, provenance receipts, and execution lineage, so this PR is a scoped mitigation rather than the final design.
