## Summary

Keep open: the fix is useful, but the branch currently makes sessionless cron tool instances behave as main-scoped callers, which can break the documented OpenClaw-tools MCP bridge and other unscoped tool callers.

## What This Changes

The PR adds an optional cron `callerScope` protocol field, stamps it from the agent cron tool, and enforces same-agent cron list/get/add/update/remove/run/runs behavior in the Gateway.

## Best Possible Solution

Keep the Gateway-side scoped enforcement, but make `callerScope` a real caller-identity contract: stamp it only for cron tool instances with an actual agent session, or plumb the MCP bridge's agent identity before applying scope there.

## Reproduction Assessment

Yes. Source inspection shows current main sends agent cron tool operations without caller identity, while PR head also shows a sessionless cron tool instance will resolve caller scope to `main`.

## Solution Assessment

No, not yet. The tool and Gateway are the right layers for scoped agent cron calls, but the implementation should not apply caller scope when no caller identity exists.
