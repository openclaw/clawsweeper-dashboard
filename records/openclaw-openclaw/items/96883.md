## Risks / Open Questions

- The new Gateway connect auth field intentionally fails closed when an older Gateway rejects it, so mixed-version local backend cron calls need maintainer-accepted compatibility behavior.
- OPENCLAW_TOOLS_MCP_AGENT_SESSION_KEY becomes required for the managed openclaw-tools MCP server path, so direct or unmanaged invocations that used to start without session context need an explicit support decision or documentation.
- The PR changes a model-mediated cron authorization boundary; a mistake could expose another agent's jobs, hide same-agent jobs, or deny valid operations.
- The broader scheduling-authority RFC remains open for playbooks, quotas, expiry policy, provenance receipts, and execution lineage, so this should be treated as a scoped mitigation rather than the final scheduling design.
