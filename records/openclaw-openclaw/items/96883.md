## Summary

Keep open: this is a useful scoped cron authorization mitigation, but it is a draft PR with a protected maintainer label and the remaining decision is maintainer acceptance of the new Gateway auth field, managed MCP session-key contract, and security boundary behavior.

## What This Changes

The PR adds a local agent-runtime identity token path, wraps agent-tool Gateway cron calls with the calling agent identity, injects per-session MCP env for the ACPX openclaw-tools bridge, scopes Gateway cron operations server-side, and expands focused cron/Gateway/ACPX tests.

## Best Possible Solution

Land this scoped mitigation only after maintainers accept the Gateway auth and managed MCP env contracts, while keeping the broader schedule-authority design in https://github.com/openclaw/openclaw/issues/71712.

## Reproduction Assessment

Yes, at source level. Current main exposes the openclaw-tools MCP cron path and Gateway cron CRUD without a verified agent runtime identity, while PR head threads and enforces the caller identity through local Gateway context.

## Solution Assessment

Mostly yes. The agent tool, Gateway client, and Gateway server are the right enforcement layers for a scoped mitigation, but maintainers still need to accept the new auth/env contract or choose a different internal identity channel.
