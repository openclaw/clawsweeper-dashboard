## Risks / Open Questions

- The new Gateway connect auth field intentionally fails closed against older Gateways, so mixed-version local backend cron calls can stop until the Gateway is restarted or updated.
- `OPENCLAW_TOOLS_MCP_AGENT_SESSION_KEY` becomes required for the managed openclaw-tools MCP server path; managed ACPX injects it, but direct unmanaged invocation needs an explicit support or documentation decision.
- The PR changes a model-mediated cron authorization boundary, where mistakes could expose another agent's jobs, hide same-agent jobs, or deny valid operations.
- The broader scheduling-authority RFC remains open for playbooks, quotas, expiry policy, provenance receipts, and execution lineage, so this PR is an interim mitigation rather than the final design.
