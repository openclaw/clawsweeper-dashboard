## Summary

The PR addresses a real runtime-prompt bug, but it is not merge-ready because it adds an uncached Darwin `sw_vers` subprocess to prompt construction and changes the broader serialized OS summary `release` field outside the reported prompt surface.

## What This Changes

The PR routes three agent runtime prompt callers through a new Darwin-aware OS product label helper and changes `resolveOsSummary()` to use the macOS product version as its `release` on Darwin.

## Best Possible Solution

Keep the runtime prompt fix, but implement it through a cached helper that leaves `resolveOsSummary().release` as the raw OS release and adds focused prompt and OS-summary regression tests before merge.

## Solution Assessment

No: the direction is right, but this patch should avoid uncached prompt-time subprocesses and preserve the shared OS summary `release` contract.

That whole object is reused in status JSON and trajectory metadata, so existing consumers lose the canonical Darwin release while `label` already carries the product-version display string.
