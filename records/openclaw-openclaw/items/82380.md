## Summary

Keep open: the PR targets a real OpenRouter replay bug, but the patch only prevents newly streamed non-JSON signatures from being persisted and does not recover existing OpenRouter openai-completions transcripts that already contain field-name sentinels.

## What This Changes

The PR changes `src/agents/openai-transport-stream.ts` to skip replaying or persisting `thinkingSignature` values unless they start with `{`.

## Best Possible Solution

Fix OpenRouter openai-completions replay so field-name sentinels are never sent, preserve valid OpenRouter reasoning contracts where supported, and cover both new streams and existing poisoned transcripts with focused tests plus real after-fix proof.

## Reproduction Assessment

Yes at source level: the linked issue gives a concrete two-turn OpenRouter path, and current main still writes and replays field-name reasoning sentinels through the OpenRouter completions sanitizer. I did not run the live OpenRouter/WhatsApp reproduction in this read-only pass.

## Solution Assessment

No: this PR is not yet the best fix because it prevents future non-JSON persistence but does not sanitize existing poisoned openai-completions history. The narrower maintainable fix should add the completions replay guard and regression tests around the OpenRouter sentinel cases.
