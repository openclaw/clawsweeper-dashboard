## Summary

Keep this PR open: it targets a plausible OpenRouter second-turn bug, but the patch is not mergeable because it drops provider-owned OpenAI-completions replay signatures broadly, hardens the wrong replay path for existing transcripts, and lacks after-fix real behavior proof.

## What This Changes

The PR changes `src/agents/openai-transport-stream.ts` to skip non-JSON `thinkingSignature` values when persisting OpenAI-completions thinking blocks and when replaying Responses reasoning items.

## Best Possible Solution

Narrow the fix in the OpenAI-completions replay contract so non-replayable OpenRouter Anthropic/xAI provenance tags are stripped or rewritten while documented replay fields for OpenRouter, DeepSeek, Z.AI, MiMo, and Kimi remain covered by regression tests.

## Reproduction Assessment

Yes at source level: the related issue gives a concrete OpenRouter second-turn path, and current main still stores field-name signatures from completions deltas before replay sanitization. I did not run a live OpenRouter/WhatsApp repro in this read-only review.

## Solution Assessment

No. The patch uses a broad JSON-looking guard that removes provider-owned replay markers and puts the recovery guard in the Responses conversion path instead of the OpenAI-completions follow-up path that the report exercises.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve provider replay signatures that are not JSON:** `src/agents/openai-transport-stream.ts:1838-1843`
  - body: The write-side guard now drops every streamed OpenAI-completions signature unless it starts with `{`. Those signatures are field keys by design for providers that current main replays (`reasoning_content`, `reasoning`, `reasoning_details`, etc.); existing tests expect OpenRouter, DeepSeek/Z.AI, MiMo, and Kimi follow-up payloads to carry them. This would make future turns lose required reasoning replay for those routes instead of only suppressing the bad Anthropic/xAI OpenRouter cases.
  - confidence: 0.89
- **[P2] Harden the completions replay path for old transcripts:** `src/agents/openai-transport-stream.ts:521-527`
  - body: The added replay guard is in `convertResponsesMessages`, but the reported follow-up uses the `openai-completions` path through `buildOpenAICompletionsParams`. Existing sessions that already contain `thinkingSignature: "reasoning_details"` still pass through the completions replay sanitizer, so the claimed recovery for poisoned transcripts is not covered.
  - confidence: 0.83
