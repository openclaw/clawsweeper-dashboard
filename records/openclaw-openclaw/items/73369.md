## Summary

Keep open. The Discord skill is a reasonable place for this clarification, but the current wording still ties automatic runtime delivery to `autoThread: true`; current main defaults Discord guild/channel turns to message-tool-only unless visible replies are configured automatic, so the patch can suppress the only visible output path.

## What This Changes

The PR adds a Discord skill guideline and changelog entry telling agents not to repeat final conversational replies with `message.send` when `autoThread` runtime delivery will post to a created thread.

## Best Possible Solution

Revise the skill and changelog so duplicate-avoidance applies to resolved automatic source-delivery turns, while preserving `message.send` as the visible-output path for default Discord guild/channel and autoThread turns.

## Reproduction Assessment

Yes. Source inspection and existing tests show current main defaults Discord guild/channel source replies to `message_tool_only`; the proposed text claims `autoThread` alone means automatic runtime delivery.

## Solution Assessment

No. A Discord-specific prompt clarification is a narrow maintainable boundary, but it should key off resolved automatic delivery mode rather than `autoThread: true` alone.
