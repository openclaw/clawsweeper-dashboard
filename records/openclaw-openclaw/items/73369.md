## Summary

Keep open. The PR is a plausible scoped prompt fix for the linked Discord duplicate-reply bug, but the current wording still treats `autoThread: true` as automatic final delivery and the contributor has not shown after-fix real Discord proof.

## What This Changes

This PR adds one Discord skill guideline and one changelog entry to reserve `message.send` for sideband traffic when runtime delivery will post the final reply.

## Best Possible Solution

Make the Discord guidance delivery-mode-aware and verify the patched autoThread path in a real Discord setup; if the linked bug is fully fixed, align or explicitly preserve the broader Codex prompt behavior too.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main shows automatic configured Discord autoThread delivery can target the created thread, while the existing prompt and skill text can still steer the agent to also call `message.send`.

## Solution Assessment

No. A Discord skill clarification is a reasonable part of the fix, but this wording should be scoped to automatic source delivery rather than treating `autoThread: true` alone as proof that final delivery will be visible.
