## Summary

Keep open. The PR targets a real Discord duplicate-reply problem, but the current wording still treats `autoThread: true` as if it implies automatic final-response delivery; current main keeps default Discord guild/channel turns in `message_tool_only`, where skipping `message.send` can make the reply private or invisible. The PR also needs after-fix real Discord behavior proof before merge.

## What This Changes

This PR adds one Discord skill guideline and one changelog entry to discourage duplicate conversational replies through `message.send` in Discord autoThread turns.

## Best Possible Solution

Make the Discord skill and changelog delivery-mode-aware: warn against duplicate same-answer `message.send` only when automatic source delivery is active, while preserving `message(action=send)` as the visible-output path for `message_tool_only` Discord channel turns.

## Reproduction Assessment

Yes. Source inspection and existing tests show default Discord guild/channel turns resolve to `message_tool_only`, while automatic final delivery requires explicit automatic visible replies or fallback behavior.

## Solution Assessment

No. A Discord skill clarification is a reasonable boundary, but this implementation should key the no-repeat guidance to resolved automatic source delivery rather than `autoThread: true` alone.
