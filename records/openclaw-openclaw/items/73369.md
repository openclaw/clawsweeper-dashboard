## Summary

Keep open. The patch still overstates that `autoThread: true` itself means automatic final delivery, while current main makes Discord guild/channel replies message-tool-only by default unless automatic visible replies are configured, and the PR still lacks after-fix Discord proof.

## What This Changes

This branch adds one Discord skill guideline and one changelog entry saying agents should avoid same-answer `message.send` duplicates in Discord autoThread turns.

## Best Possible Solution

Land one canonical delivery-mode-aware Codex prompt and Discord skill update that reserves `message.send` for explicit sideband sends while preserving tool-only visible replies for default group/channel turns.

## Reproduction Assessment

Yes, source-reproducible rather than live-run: current main shows automatic Discord autoThread delivery can target a created thread, while the prompt/skill guidance can still steer an agent to also call `message.send`. Tests and docs also show the default group/channel mode is tool-only, which is the boundary this PR must preserve.

## Solution Assessment

No. A Discord skill clarification is a reasonable part of the fix, but the wording should key off automatic source delivery rather than `autoThread: true` alone, and the broader Codex instruction remains part of the same bug surface.
