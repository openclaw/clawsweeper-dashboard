## Summary

Keep open. The PR is aimed at a real Discord duplicate-reply prompt problem, but the current wording still treats `autoThread: true` as sufficient proof that final assistant text is automatically delivered, while current main resolves Discord guild/channel turns to message-tool-only by default. That can make the skill suppress the only visible reply in default Discord channel configurations.

## What This Changes

This PR adds a Discord skill prompt guideline reserving `message.send` for sideband traffic when final replies auto-deliver, plus a changelog fix entry for Discord autoThread duplicate replies.

## Best Possible Solution

Revise the skill and changelog so duplicate-avoidance applies when the per-turn/runtime context says final replies are automatically delivered to the source thread or channel, while preserving `message.send` as the required visible-output path for default Discord guild/channel tool-only turns.

## Reproduction Assessment

Yes. The patch-level regression is reproducible from current source and tests: Discord guild messages resolve as `ChatType: "channel"`, default delivery resolves to `message_tool_only`, and tests only expect automatic delivery after explicit visible-reply configuration.

## Solution Assessment

No. A scoped Discord skill prompt clarification is the right boundary, but the current text should key off resolved delivery mode or explicit automatic visible-reply context rather than `autoThread: true` alone.
