## Summary

Keep open. The PR is scoped to the right Discord skill/changelog surface, but the current wording still treats `autoThread: true` as sufficient proof that final replies auto-deliver; current main keys that behavior to the resolved source delivery mode, and default Discord guild/channel turns remain message-tool-only.

## What This Changes

This PR adds a Discord skill guideline and changelog entry telling agents not to repeat final conversational replies with `message.send` when autoThread runtime delivery will post to the created thread.

## Best Possible Solution

Revise the skill and changelog so the duplicate-avoidance rule applies when the current turn is in automatic source-delivery mode, while preserving `message.send` as the visible-output path for default Discord channel turns.

## Reproduction Assessment

Yes. Current source and tests reproduce the patch-level mismatch: Discord guild/channel turns default to `sourceReplyDeliveryMode: "message_tool_only"`, while automatic final delivery requires explicit visible-reply configuration or message-tool fallback.

## Solution Assessment

No. A Discord skill prompt clarification is a good boundary, but this implementation should key off resolved automatic delivery mode rather than `autoThread: true` alone.
