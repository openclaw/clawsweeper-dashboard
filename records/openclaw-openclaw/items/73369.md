## Summary

Keep open. The PR is still relevant to the linked Discord duplicate-reply bug, but the current diff is not merge-ready because its new prompt text overstates when `autoThread: true` gets automatic final delivery on current main.

## What This Changes

This PR adds a Discord skill guideline warning agents not to duplicate autoThread-delivered replies with `message.send` and adds a matching Unreleased changelog fix entry.

## Best Possible Solution

Revise the Discord skill and changelog so the “do not use `message.send` for the final reply” guidance applies only when automatic source delivery is active, such as `messages.groupChat.visibleReplies: "automatic"` or an explicit automatic delivery mode. Under the current default `message_tool_only` mode, the prompt should still allow `message.send` for visible Discord output.

## Reproduction Assessment

Yes. The focused reproduction for this PR finding is source-backed: a default Discord guild/channel turn resolves to `message_tool_only`, so the proposed autoThread wording would tell the agent not to use the mechanism current main expects for visible output.

## Solution Assessment

No. The PR’s narrow prompt/docs approach is reasonable, but the wording needs to be conditional on automatic delivery before it is the best fix for current main.
