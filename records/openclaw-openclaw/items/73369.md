## Summary

Keep open. The PR is not obsolete, but its current one-line Discord skill guidance overstates current behavior: on current main, Discord guild/channel replies default to message-tool-only visible delivery, so `autoThread: true` does not by itself mean the runtime will post final assistant text to the created thread. That makes the proposed prompt potentially suppress the only visible reply in the default autoThread path.

## What This Changes

This PR adds a Discord skill guideline and changelog entry intended to prevent duplicate parent-channel plus auto-created-thread replies when Discord `autoThread: true` sessions use `message.send` and also produce a final assistant response.

## Best Possible Solution

Revise the PR so the Discord skill and changelog key the no-repeat guidance to automatic source-delivery turns, such as when the per-turn prompt/config says final replies are automatically sent, while preserving message-tool guidance for the default Discord guild/channel and autoThread path where final replies are private.

## Reproduction Assessment

Yes. The PR-specific regression is reproducible from source and tests: Discord guild messages resolve as `ChatType: "channel"`, current defaults produce `message_tool_only`, and tests assert automatic delivery only when configured.

## Solution Assessment

No in its current wording. A scoped skill prompt clarification is a reasonable fix boundary, but it must not say `autoThread: true` alone causes automatic final delivery; the safer fix is to defer to the resolved per-turn delivery mode or explicit `messages.groupChat.visibleReplies: "automatic"` configuration.
