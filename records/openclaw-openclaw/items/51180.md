The branch changes `src/commands/channels/status.ts` so live gateway status output shows up to five `allowFrom` entries and appends a `+N more` overflow marker.

## Best Possible Solution

Land one tested CLI status formatter policy that removes silent truncation, uses an unambiguous overflow marker, keeps identifier exposure intentional, and includes redacted after-fix CLI proof.

## Reproduction Assessment

Yes at source level. Current main still slices live `account.allowFrom` to two entries, and the documented `channels status --probe` path logs that formatter for reachable-gateway status output.

## Solution Assessment

Unclear. The patch fixes the central live truncation, but the overflow marker, identifier exposure policy, focused coverage, and redacted real CLI proof need to align before this is the best merge shape.
