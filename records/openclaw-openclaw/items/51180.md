## Summary

Keep this PR open. Current main still has the live `channels status --probe` truncation this PR targets, so the branch is not obsolete; it needs maintainer consolidation with the overlapping open PR and a clear allowlist display/privacy decision before merge.

## What This Changes

This PR changes the live `channels status --probe` formatter to show up to five `allowFrom` entries and append a `,+N more` suffix for longer lists.

## Best Possible Solution

Use one canonical allowlist display helper for the live gateway status formatter and the config/status summary path, choose a clear full/capped/redacted policy, render overflow with an unmistakable marker, and cover two-entry, three-entry, exact-limit, and overflow cases in focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to pass a live gateway status payload with three `allowFrom` entries into `formatGatewayChannelsStatusLines()` on current main; the formatter slices to two entries at `src/commands/channels/status.ts:96`.

## Solution Assessment

No, not yet. The PR is a narrow useful formatter change, but the maintainable solution should coordinate with #51179, share the display policy, avoid the ambiguous overflow token, and add regression tests before merge.
