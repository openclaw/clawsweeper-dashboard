## Summary

Keep open: current main and the latest release still run the primary memory/session branch before the wiki/supplement branch under one 15s tool deadline, and timeout failures still fall through to the generic embedding/provider warning. Two open linked PRs now own the central fix and diagnostics, so the issue should close only after an implementation lands or maintainers reject that path.

## What This Changes

The issue asks for `memory_search` with `corpus="all"` to avoid all-results timeouts when individual corpora succeed, return better partial/debug information, and stop misattributing aggregation timeouts to the embedding provider.

## Best Possible Solution

Review and land the focused memory-core concurrency fix in https://github.com/openclaw/openclaw/pull/92833 if maintainers accept it, then coordinate the timeout attribution/debug follow-up in https://github.com/openclaw/openclaw/pull/92647 or an equivalent narrower patch before closing this issue as fully handled.
