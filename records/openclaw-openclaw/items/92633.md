Two open mergeable PRs already reference this issue with closing syntax, so the issue should stay open until that fix path lands or maintainers choose a different resolution.

## What This Changes

The issue asks for `memory_search` with `corpus="all"` to avoid timing out when individual memory/session/wiki corpora succeed and to report all-corpus timeout causes accurately.

## Best Possible Solution

Land a focused memory-core fix that overlaps the memory/session and wiki/supplement searches under the shared deadline and returns branch-aware timeout diagnostics, then close this issue from the merged PR.
