## Summary

Keep open. Current main has partial fixes for exact silent tokens and commentary-phase output, but the shared user-facing normalizer still has no guard for envelope-shaped tool-call text such as `to=functions.*`, `recipient_name`, `parameters`, or short tool-argument JSON before delivery.

## What This Changes

Prevent Discord from delivering internal sentinel/control tokens and tool-call envelope text as visible channel messages.

## Best Possible Solution

Add a shared normalizer or sanitizer guard that narrowly suppresses internal envelope traces across channels while preserving media, channelData, interactive payloads, and legitimate prose examples.

## Reproduction Assessment

Yes. Source inspection shows current main handles exact silent tokens and commentary phase, but still lacks a suspicious-envelope guard before user-visible delivery for `to=functions.*`, `recipient_name`, `parameters`, and short tool-argument JSON.

## Solution Assessment

Yes. A shared sanitizer/normalizer boundary is the maintainable fix; adding Discord to plain-text HTML sanitization or only adding a Discord send guard would miss shared envelope-shaped text paths.
