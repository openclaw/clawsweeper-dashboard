## Summary

Keep open. Current main still has a source-reproducible path where envelope-shaped internal text can pass through the shared reply normalizer/sanitizer as deliverable content, and the active related fix remains in review rather than merged.

## What This Changes

The issue asks OpenClaw to stop Discord and other user-facing channel delivery from exposing internal control tokens, tool-routing envelopes, commentary markers, and raw tool-call JSON.

## Best Possible Solution

Land or replace the channel-agnostic reply-normalization guard so malformed internal envelopes are suppressed before any channel delivery, with regression tests for Discord-shaped and cross-channel text payloads.

## Reproduction Assessment

Yes, source-reproducible. The current shared normalizer still delivers sanitized text for envelope-shaped strings because neither the skip reasons nor sanitizeUserFacingText() recognize the reported to=functions.*, recipient_name, parameters, commentary, or short JSON forms.

## Solution Assessment

Yes, the best fix direction is a shared reply/delivery guard rather than a Discord-only plain-text sanitizer change. The open related PR targets that boundary but still needs review and proof before this issue can close.
