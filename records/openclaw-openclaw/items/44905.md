## Summary

Keep open. Current main has partial fixes for commentary-phase output and exact `NO_REPLY` media captions, but the Discord/shared outbound path still lacks a delivery-boundary sanitizer or send backstop for raw tool-routing lines, envelope field names, generic tool-call JSON, and `REPLY_SKIP`/`ANNOUNCE_SKIP`-style control tokens.

## What This Changes

The issue asks OpenClaw to stop Discord from delivering internal LLM tool-call traces, silent/control tokens, and tool-routing envelope text as visible channel messages.

## Best Possible Solution

Add a conservative shared delivery-boundary sanitizer for internal trace/control artifacts, preserve normal prose/code/media delivery, and add a Discord send-level backstop for bypass paths. Cover `to=functions.*`, envelope field names, raw tool-call JSON shapes, and skip/control tokens with focused outbound and Discord regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a Discord outbound payload whose text is `to=functions.memory_search`, `recipient_name`, a small tool-call argument JSON object, `REPLY_SKIP`, or `ANNOUNCE_SKIP`; current main shows those values can reach shared normalization and Discord send without a matching sanitizer or final guard.

## Solution Assessment

No. Current main is only a partial solution; the narrow maintainable path is a maintainer-reviewed shared outbound filter plus a Discord backstop, not adding Discord to the plain-text HTML sanitizer list or relying only on commentary-phase suppression.
