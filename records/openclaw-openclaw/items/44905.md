## Summary

Keep open. Current main has partial fixes for commentary-phase output and exact `NO_REPLY` payload handling, but it still lacks a shared delivery-boundary or Discord send backstop for raw `to=functions.*`, envelope field names, generic tool-call JSON, and `REPLY_SKIP`/`ANNOUNCE_SKIP`-style control sentinels.

## What This Changes

The issue asks OpenClaw to prevent Discord from delivering internal tool-call traces, commentary markers, raw tool envelopes, and control sentinels as visible channel messages.

## Best Possible Solution

Add a conservative shared delivery-boundary sanitizer for internal trace/control artifacts, preserve normal prose/code/media delivery, and add a Discord send-level backstop for bypass paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a Discord outbound payload whose text is `to=functions.memory_search`, `recipient_name`, a small tool-call JSON object, `REPLY_SKIP`, or `ANNOUNCE_SKIP`; current main shows those values can reach normalization and Discord send without a matching sanitizer or final guard.

## Solution Assessment

No. Current main is only a partial solution; the maintainable path is a maintainer-approved shared outbound policy plus a Discord backstop, not adding Discord to the plain-text HTML sanitizer list or relying only on commentary-phase suppression.
