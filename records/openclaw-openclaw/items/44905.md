## Label Justifications

- `P1`: The issue reports user-visible leakage of internal orchestration text in Discord, a real channel workflow and trust-boundary failure.
- `impact:security`: Visible tool-routing envelopes and raw tool arguments can expose internal implementation details or sensitive contextual data to channel users.
- `impact:message-loss`: The channel can receive internal routing payloads instead of only the intended user-facing reply, which is wrong-target message delivery.

## Summary

Keep open: current main has partial hardening for exact silent/control tokens and commentary-phase output, but the remaining envelope-shaped Discord leak variants are still not covered by the shared sanitizer path.

## What This Changes

The issue requests that Discord never deliver internal control tokens, commentary markers, tool-routing envelopes, or raw tool-call JSON as visible channel messages.

## Best Possible Solution

Add a maintainer-approved, channel-neutral internal-trace sanitizer with regression coverage for the reported envelope forms, while preserving legitimate prose/code and existing channel formatting.

## Reproduction Assessment

Yes at source level, but not with a live Discord run in this review. Current code only strips known runtime blocks and narrower tool-call syntaxes, so the reported envelope-shaped strings can still pass if they reach the delivery sanitizer.

## Solution Assessment

No complete fix is present yet. The best fix is not a Discord-only send guard or an ad hoc regex, but a shared outbound/user-facing trace sanitizer with tests and maintainer-reviewed false-positive boundaries.
