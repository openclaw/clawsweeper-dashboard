## Summary

Keep open. Exact control-token and commentary-phase subsets are partially fixed, but current main still lacks a shared guard for envelope-shaped tool-call text, and the related fix PRs are still open rather than merged.

## What This Changes

The issue requests preventing Discord/user-visible delivery of internal control tokens and tool-call envelope strings such as `NO_REPLY`, `commentary`, `to=functions.*`, `recipient_name`, `parameters`, and short tool-argument JSON.

## Best Possible Solution

Land a rebased shared normalizer or user-facing sanitizer that narrowly suppresses internal envelope traces while preserving media, channelData, and literal prose examples.

## Reproduction Assessment

Yes. Source inspection shows current main handles exact silent tokens and commentary phase, but still lacks a suspicious-envelope branch for `to=functions.*` or `recipient_name`/`parameters` text before user-visible delivery.

## Solution Assessment

Yes, the best direction is a shared sanitizer/normalizer guard. Adding Discord to plain-text HTML sanitization would not cover raw tool-envelope strings, and a Discord-only send guard would miss other delivery paths.
