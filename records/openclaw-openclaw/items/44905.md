## Summary

Keep open. Current main has partial coverage for exact silent tokens and commentary-phase output, but the shared reply path still lacks a guard for envelope-shaped internal text such as `to=functions.*`, `recipient_name`, `parameters`, and short tool-argument JSON; the active related fix path is https://github.com/openclaw/openclaw/pull/43679.

## What This Changes

The issue requests preventing Discord and shared channel delivery from exposing internal control tokens and tool-call envelope text to users.

## Best Possible Solution

Land or replace the shared channel-agnostic normalizer/sanitizer guard so internal envelope traces are suppressed before delivery while preserving legitimate prose examples and non-text reply payload content.

## Reproduction Assessment

Yes, source-reproducible. The current normalizer only skips `empty`, `silent`, and `heartbeat` before passing text through a sanitizer that does not recognize the reported envelope-shaped tool-routing strings.

## Solution Assessment

Yes, the shared normalizer/sanitizer boundary is the right fix direction. A Discord-only plain-text or exact-token send guard would miss cross-channel `to=functions.*` and JSON-envelope text paths.
