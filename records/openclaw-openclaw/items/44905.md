## Summary

Keep open. Current main has partial fixes for commentary-phase delivery and exact silent/control-token handling, but the broader envelope-shaped sanitizer remainder is still source-reproducible and is also called out by a fresh current-main retest in the issue discussion.

## What This Changes

The issue reports Discord-visible leakage of internal tool-call traces and control tokens such as `NO_REPLY`, `commentary`, `to=functions.*`, `recipient_name`, `parameters`, and short tool-argument JSON.

## Best Possible Solution

Land a shared, channel-agnostic sanitizer or normalizer guard that suppresses raw internal routing/envelope text before outbound delivery, while preserving the existing commentary-phase and silent-token safeguards.

## Reproduction Assessment

Yes, source-reproducible. Current main has active paths through `normalizeReplyPayload()` and `sanitizeUserFacingText()` where envelope-shaped text such as `to=functions.exec {"query":"x"}` or `recipient_name: functions.exec` is not stripped, and the issue has a fresh current-main retest reporting the same remainder.

## Solution Assessment

No, this is not solved yet. Adding Discord to `PLAIN_TEXT_SURFACES` would be the wrong layer; the maintainable fix is a shared envelope/internal-trace guard with regression coverage, or a rebased equivalent of the open related PR work.
