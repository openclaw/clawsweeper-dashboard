## Summary

Keep open: current main has fixed the commentary-phase and silent-token media subsets, but the Discord/outbound path still lacks a shared guard for raw tool-routing or envelope text once it reaches a normal reply payload.

## What This Changes

The issue requests that Discord and channel delivery suppress `NO_REPLY`, commentary, `to=functions.*`, raw tool-call JSON, and related internal envelope fields before user-visible sends.

## Best Possible Solution

Define a conservative shared visible-delivery sanitizer for known internal control/tool-routing envelopes, cover it with outbound and Discord regression tests, and keep broader audience-boundary work coordinated with #25592 and #74928.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows that a normal reply payload containing `to=functions.*` or envelope-field text would pass through the current Discord/outbound sanitizers, while exact `NO_REPLY` and commentary-phase subsets are already covered.

## Solution Assessment

Unclear until maintainers choose the stripping policy. Adding Discord to `PLAIN_TEXT_SURFACES` is not the right fix; the maintainable path is a shared, narrowly scoped delivery sanitizer plus existing phase-aware suppression.
