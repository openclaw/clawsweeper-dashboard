## Summary

Keep open. Current main still lacks a shared suspicious-envelope guard for raw `to=functions.*`, `recipient_name`, `parameters`, and short tool-argument JSON text; exact silent tokens and commentary-phase output are only partially covered, and https://github.com/openclaw/openclaw/pull/43679 remains open and conflicting rather than landed.

## What This Changes

Prevent Discord/channel delivery from exposing internal silent tokens, commentary/tool-routing markers, and raw tool-call envelopes as visible messages.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43679 with a rebased shared normalizer/sanitizer guard that blocks internal envelope traces before any channel delivery while preserving media, interactive, presentation, and channel data payloads.

## Reproduction Assessment

Yes, source-reproducible. Current main routes replies through the shared normalizer, and neither the skip reasons nor `sanitizeUserFacingText()` recognizes the reported raw envelope forms.

## Solution Assessment

Yes, the maintainable fix shape is a channel-agnostic reply-normalization or sanitizer guard rather than a Discord-only plain-text conversion change; the active related PR targets that boundary but still needs rebase, proof, and review.
