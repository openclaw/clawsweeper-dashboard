The PR changes Unix process-tree termination so omitted `detached` uses direct-pid signaling, opts `runCommandWithTimeout` back into detached group-kill, and updates kill-tree unit tests.

## Best Possible Solution

Land the safe direct-pid default only after preserving group cleanup for known detached callers or adding safe process-group-leader inference, with focused tests and real process proof.

## Reproduction Assessment

Yes at source level: current main and the latest release default omitted Unix kills to group-kill, and current main has detached callers that omit `detached`. I did not run the live macOS launchd gateway scenario in this read-only review.

## Solution Assessment

No: the safe-default direction is good, but this patch is still too one-sided because it preserves only one detached caller while changing the shared helper contract. The best fix is the helper change plus caller audit, or helper-side process-group-leader detection.
