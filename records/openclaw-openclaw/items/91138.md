## Summary

## What This Changes

The issue requests a global or per-agent way to disable exec credential masking, or a heredoc/quoted-string exemption, so inline API keys are not redacted before use.

## Best Possible Solution

Keep secure default redaction, collect an exact command/output reproduction for any real shell-input mutation, then design a narrow trusted credential-passing or display exemption only if maintainers approve the security boundary.

## Reproduction Assessment

No. The issue says the behavior is consistent on v2026.6.1, but it does not include an exact command/output pair; current main source shows display/event/approval redaction and raw exec command execution as separate paths.

## Solution Assessment

No. A global tools.exec credential-masking off switch is too broad for the current security boundary; a targeted parser/display exemption or explicit trusted credential-passing path would be safer if a concrete repro proves execution mutation.
