## Summary

Keep open for maintainer product/security review: current main still deliberately redacts credential-like exec command text in approval/tool-display and persisted surfaces, while the raw shell launch path appears separate; the requested disable switch is a new security-sensitive config surface and still lacks an exact command/output repro for the claimed pre-shell mutation.

## What This Changes

The issue requests a global or per-agent option, or quoted/heredoc-aware parsing, to prevent exec credential masking from rewriting inline secret-looking command text.

## Best Possible Solution

Define a narrow trusted credential-passing or display-only redaction boundary that preserves safe defaults, then implement it with focused tests only after maintainers accept the security tradeoff and the reporter supplies a concrete repro.

## Reproduction Assessment

No. Source inspection proves command text can be masked in approval/display and persisted metadata paths, but the report does not include an exact command/output pair proving current main mutates the shell command before execution.

## Solution Assessment

No. A global disable switch is broader than the proven need; the safer direction is a maintainer-approved, scoped trusted path or parser refinement that keeps safety-boundary redaction intact where required.
