## What This Changes

The PR adds optional `engineCandidates` and `corpus` fields to `MemoryCorpusSupplement`, forwards engine search results to corpus supplements, re-exports `MemorySearchResult`, updates the SDK API baseline hash, and adds focused memory-core tests.

## Best Possible Solution

Rebase over current main, preserve the latest `memory_search` behavior, and merge only if maintainers approve the additive `engineCandidates`, `corpus`, and `MemorySearchResult` SDK semantics.

## Reproduction Assessment

## Solution Assessment

Mostly yes, pending maintainer SDK approval. Passing the already available filtered search results through an optional field is the narrowest path for post-pipeline supplement roundtrip reduction, while the broader rerank-provider work is complementary rather than a substitute.
