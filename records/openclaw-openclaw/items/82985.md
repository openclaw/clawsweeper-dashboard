## Summary

Keep open: the PR addresses a real memory supplement SDK gap and has sufficient terminal proof, but the current patch drops the existing runtime `corpus` value delivered to supplements, so it needs a small repair before merge.

## What This Changes

The PR adds optional `engineCandidates` to `MemoryCorpusSupplement.search`, forwards raw memory search results to registered supplements, re-exports `MemorySearchResult`, and adds passthrough tests.

## Best Possible Solution

Preserve the existing supplement params while adding optional `engineCandidates`, add a regression assertion for `corpus`, and then let maintainers decide the public SDK shape.

## Reproduction Assessment

Yes by source inspection: current main lacks `engineCandidates` in `MemoryCorpusSupplement.search` while the engine already has `rawResults` before calling supplements. The PR regression is also source-reproducible because PR head constructs `supplementParams` without `corpus`.

## Solution Assessment

No as patched. The optional field is a plausible narrow API addition, but the implementation should forward the full params object or include `corpus: params.corpus` and cover that compatibility path in tests.
