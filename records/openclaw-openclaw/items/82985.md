The PR adds optional `engineCandidates` and `corpus` fields to the memory corpus supplement contract, forwards memory search `rawResults` to supplements, re-exports `MemorySearchResult`, updates the SDK baseline hash, and adds focused memory-core tests.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, pending maintainer SDK approval: passing the existing in-scope results through an optional supplement field is a narrow way to remove the redundant search for supplement consumers. The broader rerank-provider work is adjacent and complementary, not a replacement for this post-pipeline passthrough.
