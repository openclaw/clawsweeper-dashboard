## Summary

Keep open: the PR addresses a real memory supplement API gap and has sufficient terminal proof, but the current patch introduces a P2 regression by wrapping supplement search params without preserving the existing runtime `corpus` value.

## What This Changes

The PR adds optional `engineCandidates` to `MemoryCorpusSupplement.search`, forwards memory search results through the memory-core tool path, re-exports `MemorySearchResult`, and adds passthrough tests.

## Best Possible Solution

Preserve the existing supplement params while adding optional `engineCandidates`, cover both fields in focused tests, and then have maintainers decide whether this SDK expansion should land.

## Reproduction Assessment

Yes by source inspection: current main has `rawResults` in scope before the supplement call and no `engineCandidates` contract, and the PR head source shows the new wrapper omits the existing runtime `corpus` field. I did not run tests because this was a read-only review.

## Solution Assessment

No as patched. The optional `engineCandidates` API is a plausible narrow solution, but the implementation should forward the full params object or include `corpus: params.corpus` and add regression coverage.
