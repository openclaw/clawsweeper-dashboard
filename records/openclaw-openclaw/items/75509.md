## Summary

Keep open. Current main still has the `announceTimeoutMs` docs gap, but the PR wording is too broad because the runtime applies the configured timeout to each retried gateway `agent` call, not to the whole announce step.

## What This Changes

## Best Possible Solution

Keep the docs-only scope, but describe `announceTimeoutMs` as the per-call gateway announce timeout and note that transient retries can extend total wait before fallback or failure.

## Solution Assessment

No, not as written. A docs-only update is the right scope, but the wording should avoid implying that the configured value bounds the entire announce step.
