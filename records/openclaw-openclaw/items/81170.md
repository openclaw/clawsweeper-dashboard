## Summary

Keep open. The PR is a focused bug fix: current main still forces OpenAI memory embedding lookups through the `openai` provider id, while the branch preserves the caller's custom provider id and includes targeted regression tests plus sufficient terminal proof.

## What This Changes

The PR changes the OpenAI memory embedding adapter and resolver to pass `options.provider ?? "openai"` into remote-client lookup/cache data and adds custom-provider regression tests.

## Best Possible Solution

Land the focused provider-id preservation with regression tests after normal CI, then close https://github.com/openclaw/openclaw/issues/47884 once the fix is merged.

## Reproduction Assessment

Yes. Current main source shows a custom `memorySearch.provider` can select the OpenAI adapter through `models.providers.<id>.api`, but the adapter/resolver then force `provider: "openai"` before the remote client reads `models.providers[params.provider]`.

## Solution Assessment

Yes. Preserving `options.provider ?? "openai"` at both OpenAI embedding call sites is the narrowest maintainable fix because it keeps adapter identity stable while changing only the provider-config lookup key and cache key.
