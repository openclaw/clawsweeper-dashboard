## Summary

Keep this PR open, but it is not merge-ready: the hardening goal is valid and not already implemented on current main for normal fresh/resumed Claude CLI launches, but the patch unconditionally forces a Claude Code flag that older host-installed Claude binaries do not support and the PR still lacks after-fix real launch proof.

## What This Changes

The PR adds `--safe-mode` to Anthropic `claude-cli` fresh/resume defaults and re-adds it during backend config normalization after user overrides.

## Best Possible Solution

Keep the Claude CLI isolation goal, but land it only with a maintainer-approved minimum-version policy or a version/setup/doctor gate, focused regression coverage, branch refresh, and real fresh/resume launch proof.

## Reproduction Assessment

Yes by source and dependency inspection: current main does not enforce `--safe-mode` for normal fresh/resumed Claude CLI launches, the PR would force it after user overrides, and Claude Code 2.1.168 lacks the flag strings that appear in 2.1.169. I did not run a live OpenClaw-spawned Claude CLI turn in this read-only review.

## Solution Assessment

No. Unconditional flag insertion is a plausible implementation, but the best fix needs a maintainer-approved minimum-version policy or a narrow version/setup/doctor gate before OpenClaw forces the new host-CLI requirement.
