## Summary

Keep this PR open, but do not merge it as-is: the intent is useful, yet it unconditionally raises the host Claude Code CLI requirement without upgrade handling, and the contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds `--safe-mode` to the Anthropic `claude-cli` backend's default fresh and resume args and normalizes user overrides to re-add the flag.

## Best Possible Solution

Confirm the desired minimum Claude Code version, then either gate `--safe-mode` behind version/support detection with doctor/docs coverage or deliberately document and enforce the new host-version floor before changing the default.

## Reproduction Assessment

Yes for the merge risk: source inspection shows every Claude CLI spawn uses the backend args, and dependency inspection shows `--safe-mode` appears in 2.1.169 but not 2.1.168. I did not execute a live older Claude install in this read-only review.

## Solution Assessment

No, not as currently proposed. Enforcing safe mode may be the right product direction, but the maintainable fix needs explicit version support, docs/doctor handling, or maintainer acceptance of the new host CLI floor.
