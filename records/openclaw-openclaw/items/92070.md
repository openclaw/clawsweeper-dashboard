## Summary

Keep open: the normal and resumed Claude CLI safe-mode behavior is still not on current main, but this PR is not merge-ready because it unconditionally adds a new host Claude Code flag without an upgrade gate, real launch proof, or a refresh against current main.

## What This Changes

The PR adds `--safe-mode` to Anthropic `claude-cli` default fresh/resume args and normalizes user overrides to reinsert the flag.

## Best Possible Solution

Land safe-mode isolation for the Anthropic plugin only after maintainers choose a hard Claude Code 2.1.169+ minimum or the branch adds a version/setup/doctor compatibility gate, with fresh and resumed launch proof.

## Reproduction Assessment

Yes, by source and dependency inspection: current main omits `--safe-mode` from normal fresh/resume defaults and normalization, while the PR would force it into both paths. The native Claude Code 2.1.168 package lacks the flag strings that appear in 2.1.169.

## Solution Assessment

No. Unconditional safe-mode insertion is a plausible hardening direction, but the best fix needs a compatibility gate or explicit maintainer-approved minimum-version policy before forcing a new host CLI requirement.
