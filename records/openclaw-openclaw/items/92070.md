## Summary

Keep open: the safe-mode isolation goal is useful, but the patch is not merge-ready because it unconditionally forces a newly introduced Claude Code flag onto all host-installed `claude` binaries without a compatibility gate, and real fresh/resume launch proof is still missing.

## What This Changes

The PR adds `--safe-mode` to Anthropic `claude-cli` default fresh/resume arguments and normalizes user overrides to re-add it.

## Best Possible Solution

Land safe-mode isolation in the Anthropic plugin only after maintainers either approve a hard Claude Code 2.1.169+ minimum or the branch adds a version/setup/doctor compatibility gate with focused fresh/resume proof.

## Reproduction Assessment

Yes by source and dependency inspection: current main omits `--safe-mode` from normal fresh/resume defaults and normalization, while the PR would force it into both paths. The upstream 2.1.168 native package lacks the exact flag strings that appear in 2.1.169.

## Solution Assessment

No. Unconditional safe-mode insertion is a plausible hardening direction, but the best fix needs a version/setup/doctor gate or explicit maintainer-approved minimum-version policy before forcing a new host CLI requirement.
