## Summary

Keep this PR open, but it is not merge-ready: the hardening direction is plausible, while the branch still lacks real OpenClaw-spawned launch proof, is currently conflicting, and can break existing users with older host Claude Code binaries.

## What This Changes

This PR adds `--safe-mode` to Anthropic `claude-cli` fresh/resume defaults and re-adds it during backend config normalization after user overrides.

## Best Possible Solution

Land Claude safe-mode isolation only after maintainers either approve a hard Claude Code 2.1.169+ minimum or the branch adds a version/setup/doctor compatibility gate, with focused fresh/resume proof.

## Reproduction Assessment

Yes by source and dependency inspection: current main omits `--safe-mode` from normal fresh/resume launches, while the PR forces it and Claude Code 2.1.168 lacks the flag. I did not run a live failing launch with an older binary.

## Solution Assessment

No. Unconditional insertion is a plausible hardening direction, but the best fix needs either maintainer-approved minimum-version policy or a version/setup/doctor compatibility gate before forcing the new host CLI requirement.
