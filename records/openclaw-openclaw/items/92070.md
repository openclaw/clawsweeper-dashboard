## Summary

Keep open: the central `claude-cli` safe-mode behavior is not implemented for normal current-main launches, but this PR is not merge-ready because it unconditionally requires a newer host Claude Code flag, is currently dirty against main, and still lacks real behavior proof.

## What This Changes

The PR adds `--safe-mode` to the Anthropic `claude-cli` default fresh/resume args and normalizes user overrides to re-add the flag.

## Best Possible Solution

Keep the safe-mode isolation goal, but land it only after a maintainer-approved minimum-version policy or version/setup/doctor gate, branch refresh, focused coverage, and real fresh/resume launch proof.

## Reproduction Assessment

Yes by source and dependency inspection: current main does not enforce `--safe-mode` for normal fresh/resumed launches, the PR would force it through normalization, and the 2.1.168 Claude Code linux-x64 package lacks the flag strings that appear in 2.1.169. I did not run a live OpenClaw-spawned Claude CLI turn in this read-only review.

## Solution Assessment

No. Unconditional flag insertion is a plausible hardening path, but the best fix needs a version/setup/doctor gate or explicit maintainer-approved minimum-version policy before forcing the new host-CLI requirement.
