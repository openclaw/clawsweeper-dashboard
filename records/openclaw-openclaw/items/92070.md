## Summary

Keep open: current main and the latest release do not force `--safe-mode` on normal fresh/resumed `claude-cli` launches, but this PR is not merge-ready because it unconditionally requires a newly supported host Claude Code flag, lacks real behavior proof, and is conflicting against main.

## What This Changes

The PR adds `--safe-mode` to Anthropic `claude-cli` fresh/resume defaults and re-adds it during backend config normalization after user overrides.

## Best Possible Solution

Keep the safe-mode isolation goal, but land it only after branch refresh, real fresh/resume launch proof, and a maintainer-approved version/setup/doctor gate or explicit minimum Claude Code CLI policy.

## Reproduction Assessment

Yes by source and dependency inspection: current main does not enforce `--safe-mode` for normal fresh/resumed launches, the PR would force it into spawned argv, and Claude Code 2.1.168 lacks the flag strings that appear in 2.1.169. I did not run a live OpenClaw-spawned Claude CLI turn in this read-only review.

## Solution Assessment

No. Unconditional flag insertion is a plausible implementation, but the best fix needs a maintainer-approved minimum-version policy or a narrow version/setup/doctor gate before OpenClaw forces the new host-CLI requirement.
