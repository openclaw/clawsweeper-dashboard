## Summary

Keep open. Current main still normalizes proactive WhatsApp phone-number targets to PN JIDs, while the focused outbound PN-to-LID fix is represented by open, unmerged PR #74925.

## What This Changes

Fix WhatsApp proactive outbound sends so phone-number targets resolve to LID JIDs via the active account auth directory before falling back to PN JIDs.

## Best Possible Solution

Review and land #74925, or an equivalent WhatsApp-plugin-only fix, so createWebSendApi resolves phone-number recipients through the active account authDir/Baileys forward LID mapping while preserving qualified JIDs and the PN fallback.

## Reproduction Assessment

Yes. The issue provides concrete live WhatsApp steps, and current main has a source-level reproduction: a phone-number target still reaches createWebSendApi as digits@s.whatsapp.net even when a forward LID mapping file would exist.

## Solution Assessment

Yes. Resolving at the active Web send boundary with the listener account authDir is the narrowest maintainable fix because text/media sends, polls, and composing presence all flow through createWebSendApi.
