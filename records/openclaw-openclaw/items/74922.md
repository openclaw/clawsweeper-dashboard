## Summary

Keep open: current main still suppresses configured Discord ack reactions for guild/channel turns that resolve to `message_tool_only`, and the open closing PR should be reviewed or replaced before this issue is closed.

## What This Changes

Restore automatic Discord ack reactions for guild/channel inbound messages with `ackReactionScope: "all"` after v2026.4.27.

## Best Possible Solution

Review and land #75883 if maintainers agree that the initial configured ack is allowed in tool-only guild routes; otherwise document the suppression and add clearer logging/tests for that contract.

## Reproduction Assessment

Yes. Current main has a high-confidence source/test reproduction: guild/channel messages default to `message_tool_only`, `processDiscordMessage` gates the ack on `!sourceRepliesAreToolOnly`, and the focused Discord process test asserts no reaction for `ackReactionScope: "all"`.

## Solution Assessment

Unclear until maintainer review. The open PR is a narrow maintainable fix because it decouples the initial configured ack from status/final replies, but it still needs an explicit decision on visible emoji behavior in guild channels.
