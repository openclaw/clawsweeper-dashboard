## Summary

Keep open: current main still suppresses automatic Discord ack/status reactions for guild-channel turns that resolve to `message_tool_only`, including the reported `ackReactionScope: "all"` shape. The reported channel-resolution failure path preserves configured guild entries, so the remaining issue is the visible-output policy for always-on guild rooms rather than a stale or already-fixed config-loss bug.

## What This Changes

Restore or clarify Discord automatic `ackReaction` behavior in guild channels when `ackReactionScope` is configured for all messages after the v2026.4.27 always-on guild reply changes.

## Best Possible Solution

Define the Discord guild/channel reaction contract explicitly. If configured ack reactions are allowed as processing feedback in tool-only rooms, decouple the initial ack from final-reply suppression and add focused Discord tests/docs; if they are intentionally private-output suppression, document the `ackReaction`, `ackReactionScope`, `statusReactions`, and `messages.groupChat.visibleReplies` interaction and log the suppression clearly.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: an always-on guild/channel route with `ackReactionScope: "all"` resolves to `message_tool_only`, and the current Discord process test asserts no reaction calls are made.

## Solution Assessment

Unclear. Re-enabling `ackReactionScope: "all"` for tool-only rooms would address the reported behavior, but the best fix depends on whether maintainers classify ack/status reactions as allowed processing feedback or suppressed visible output in Discord guild channels.
