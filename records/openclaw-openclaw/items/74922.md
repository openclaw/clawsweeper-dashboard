## Summary

Keep open: current main still suppresses automatic Discord ack/status reactions for guild/channel source turns that resolve to `message_tool_only`, including the reported `ackReactionScope: "all"` configuration. The reported channel-resolution fallback preserves configured guild entries, so the remaining question is product/privacy intent: whether configured ack reactions should bypass the v2026.4.27 group-channel visible-output default.

## What This Changes

The issue reports that Discord automatic `ackReaction` stopped firing in guild/channel messages after v2026.4.27 when `ackReactionScope: "all"` is configured and channel resolution logs fetch failures.

## Best Possible Solution

Define the Discord/group-channel contract explicitly. If configured ack/status reactions are allowed in tool-only rooms, narrow only the Discord reaction gate and update the matching tests/docs; if they are intentionally private output, document the `ackReaction`, `ackReactionScope`, `statusReactions`, and `messages.groupChat.visibleReplies` interaction and add clearer suppression logging.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: a guild/channel route with `ackReactionScope: "all"` resolves to `message_tool_only` and current tests assert no Discord reaction call.

## Solution Assessment

Unclear. Letting `ackReactionScope: "all"` bypass tool-only suppression would address the report, but it may conflict with the shipped privacy policy that keeps automatic visible output private in always-on group/channel rooms.
