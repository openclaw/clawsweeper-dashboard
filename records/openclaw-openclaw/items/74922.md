## Summary

Keep open: current main still suppresses configured Discord ack reactions for guild/channel turns that resolve to `message_tool_only`, and the linked closing PR is open but not merged.

## What This Changes

The issue reports Discord guild/channel `ackReaction` no longer firing with `ackReactionScope: "all"` after v2026.4.27.

## Best Possible Solution

Land a narrow Discord change that allows explicitly configured initial ack reactions in tool-only guild/channel turns while keeping final replies and status/lifecycle reactions suppressed, or document the intentional suppression if maintainers reject visible acks.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main: a guild/channel turn resolves to `message_tool_only`, the ack gate requires `!sourceRepliesAreToolOnly`, and the focused Discord process test asserts no reaction for `ackReactionScope: "all"`.

## Solution Assessment

Yes. The linked PR’s direction is the narrowest maintainable fix shape because it decouples the initial configured ack from status/lifecycle reaction gating without re-enabling automatic visible final replies.
