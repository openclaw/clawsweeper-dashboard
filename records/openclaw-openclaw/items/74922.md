## Summary

Keep open: current main still suppresses automatic Discord ack reactions for guild/channel messages resolved as tool-only, and open PR #75883 already references this issue with closing syntax, so the issue should stay open until that fix is reviewed and merged.

## What This Changes

The issue reports that configured Discord `ackReaction` no longer fires in guild channels after v2026.4.27 when guild/channel routing uses allowlist configuration and `ackReactionScope: "all"`.

## Best Possible Solution

Review and land or revise #75883 so configured initial ack reactions work in tool-only guild/channel routes without re-enabling automatic final replies or lifecycle/status reaction churn, then update focused Discord tests and docs if the policy is confirmed.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: the focused Discord process test sets `ackReactionScope: "all"` for a guild/channel route, resolves source delivery to `message_tool_only`, and currently expects no reaction client calls.

## Solution Assessment

Unclear until maintainer policy is confirmed. The linked PR’s approach is narrow and plausible because it decouples the initial configured ack from status/final-reply output, but the visible-output contract for tool-only guild rooms is the policy boundary that needs review.
