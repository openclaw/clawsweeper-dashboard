## Summary

Keep open: current main still suppresses automatic Discord ack/status reactions for guild/channel source turns that resolve to message_tool_only, including the reported ackReactionScope: "all" configuration. The channel-resolution fallback preserves configured guild entries, so the remaining blocker is deciding whether configured ack reactions should bypass the group-channel visible-output privacy default introduced around #73046.

## What This Changes

The issue asks OpenClaw to restore automatic Discord ackReaction delivery in guild channels after v2026.4.27 when ackReactionScope is configured as all and guild/channel allowlist resolution may fall back to config entries.

## Best Possible Solution

Define the Discord/group-channel contract explicitly. If configured ack/status reactions are allowed in tool-only rooms, narrow the Discord reaction gate and update tests/docs; if they are intentionally private output, document the ackReaction, ackReactionScope, statusReactions, and messages.groupChat.visibleReplies interaction and add clearer suppression diagnostics.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: a guild/channel route with ackReactionScope: "all" resolves to message_tool_only and current tests assert no Discord reaction call.

## Solution Assessment

Unclear. Decoupling ack/status reactions from message-tool-only suppression would address the report, but it may conflict with the shipped privacy default that prevents automatic visible output in always-on group/channel rooms.
