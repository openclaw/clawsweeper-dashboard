## Summary

Keep open: current main still suppresses configured Discord ack reactions in default guild/channel tool-only routes, and the issue is already paired with open closing PR #75883 rather than being implemented on main.

## What This Changes

Restore automatic Discord `ackReaction` behavior for guild channels using `ackReactionScope: "all"` after the v2026.4.27 tool-only guild reply change.

## Best Possible Solution

Review and land #75883 or an equivalent narrow Discord handler change that allows the configured initial ack reaction in tool-only guilds while preserving private final replies and suppressed lifecycle/status reactions.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main: a guild/channel route with `ackReactionScope: "all"` resolves to `message_tool_only`, and the focused process test asserts no Discord reaction is sent.

## Solution Assessment

Yes. The narrowest maintainable direction is to decouple the initial configured ack from lifecycle/status reaction suppression in the Discord handler, rather than changing the generic group/channel source reply delivery policy.
