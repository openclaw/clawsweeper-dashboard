## Summary

Keep open: current main still suppresses Discord automatic ack/status reactions for guild/channel source turns that resolve to `message_tool_only`, including the reported `ackReactionScope: "all"` shape. The reported channel-resolve fallback preserves configured guild entries, and the linked #73133 path is a separate reaction-notification listener, so this remains a product/behavior decision rather than an implemented fix.

## What This Changes

The issue requests restoring Discord automatic `ackReaction` in guild channels after v2026.4.27 when `ackReactionScope: "all"` is configured and channel resolution falls back to configured guild/channel entries.

## Best Possible Solution

Define the Discord group/channel contract explicitly. If configured ack reactions are allowed in tool-only rooms, decouple the initial ack reaction from final-reply/status-reaction suppression and update focused tests/docs; if they are intentionally private output, document the `ackReaction`, `ackReactionScope`, `statusReactions`, and `messages.groupChat.visibleReplies` interaction and add clearer suppression logging.

## Reproduction Assessment

Yes. Current main has a source-backed reproduction: a Discord guild/channel route with `ackReactionScope: "all"` resolves to `message_tool_only`, and the focused process test asserts no reaction is sent.

## Solution Assessment

Unclear. Treating this solely as a channel-resolve or #73133 regression is not the best fix because the fallback keeps config entries and #73133 is a separate reaction-event path; the maintainable path is a narrow reaction-gate/docs change after deciding the intended tool-only privacy contract.
