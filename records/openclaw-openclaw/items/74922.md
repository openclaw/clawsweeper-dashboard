## Summary

Keep open: current main still suppresses Discord ack/status reactions for guild/channel source turns that resolve to `message_tool_only`, including the reported `ackReactionScope: "all"` shape. The channel-resolution failure path returns configured guild entries, so the unresolved question is whether configured reactions should bypass the v2026.4.27 group/channel visible-output privacy default.

## What This Changes

The issue requests restoring automatic Discord `ackReaction` in guild/channel messages after v2026.4.27 when `ackReactionScope: "all"` is configured and channel resolution falls back to configured guild entries.

## Best Possible Solution

Define the Discord/group-channel reaction contract explicitly. If configured ack/status reactions are allowed in tool-only rooms, narrow the Discord reaction gate and update tests; if they are intentionally suppressed, document the interaction with `messages.groupChat.visibleReplies` and add clearer logging when reactions are skipped by source reply mode.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: a guild/channel route with `ackReactionScope: "all"` resolves to `message_tool_only` and expects no Discord reaction call, matching the reported symptom.

## Solution Assessment

Unclear. Exempting ack/status reactions from tool-only suppression would address the report, but it may conflict with the shipped group/channel privacy policy from #73046; maintainer product judgment should choose the contract first.
