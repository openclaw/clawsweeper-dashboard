## Summary

Keep this PR open; the skills-lane direction is useful, but the final branch now risks making non-bundled skills disappear from native Codex and misses the persisted-snapshot upgrade path for the new trusted prompt field.

## What This Changes

The PR routes a trusted subset of OpenClaw skills into Codex turn-scoped developer instructions, removes the skills section from Codex user input, updates prompt accounting/observer surfaces, and adds tests/snapshots for that routing.

## Best Possible Solution

Keep the skills-token lane split, but preserve non-bundled skills through a non-authoritative current-turn/reference contract or an explicit maintainer-approved compatibility change, and force-refresh or migrate legacy skill snapshots before reading `trustedDeveloperPrompt`.
