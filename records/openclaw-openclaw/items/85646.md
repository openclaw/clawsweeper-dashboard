## Summary

This is a useful Phase 1 fix for the Codex skills-history growth, but it is not merge-ready because the PR currently drops supported non-bundled skill visibility and can make existing sessions lose even bundled skills until their persisted snapshot is rebuilt.

## What This Changes

The PR routes a trusted OpenClaw-bundled skills prompt into Codex turn-scoped collaboration developer instructions, removes skills from Codex turn user input, updates prompt accounting and observers, and adds regression tests and prompt snapshots.

## Best Possible Solution

Keep the skills lane split, preserve supported non-bundled skill discovery through a non-authoritative current-turn surface, rebuild or version-gate persisted snapshots missing trustedDeveloperPrompt, and leave the linked issue open for the history sanitizer follow-up.
