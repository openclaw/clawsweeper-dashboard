Keep open. Current main still lacks the requested live abort-source diagnostic, the linked implementation PR is closed unmerged, and the stack-logging/diagnostics-flag surface needs maintainer approval before it is a safe automated repair.

## What This Changes

The issue requests a default-off embedded-runner diagnostic that classifies abort sources and logs bounded caller stack frames when enabled.

## Best Possible Solution

If maintainers accept the diagnostic, implement it at the shared abort entry point using the approved diagnostics flag surface, bounded/redacted stack output, docs, and focused timeout/compaction coverage.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The source check is high-confidence: current main has the shared abort path and abort-state metadata, but not the requested env gate, classifier, `[abort-source]` marker, stack capture, or tests.

## Solution Assessment

Unclear until maintainers approve the diagnostic surface. A single helper called from `abortRun` is technically narrow, but extending the existing diagnostics-flag model with explicit stack redaction is safer than adding a separate env-only stack logger without review.
