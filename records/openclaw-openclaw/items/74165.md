## Summary

Keep open. Current main still lacks the requested abort-source diagnostic, the linked implementation PR is closed unmerged, and the stack-frame logging/env-flag surface needs maintainer product and security review before implementation.

## What This Changes

Add an opt-in diagnostic for embedded runner aborts that classifies the abort source and logs bounded caller stack frames when enabled.

## Best Possible Solution

If maintainers accept the diagnostic, implement it narrowly at the shared abortRun entry point using the approved diagnostics flag surface, bounded/redacted stack output, docs, and focused abort-source tests.

## Reproduction Assessment

Not applicable; this is an opt-in diagnostics feature request rather than a broken existing behavior report. Source inspection gives high-confidence current-main proof that the requested flag, classifier, log marker, and tests are absent.

## Solution Assessment

Unclear until maintainers approve the diagnostic surface. A single helper at abortRun is technically narrow, but using the existing diagnostics flag system with explicit stack redaction is safer than adding a standalone env-only stack logger by default.
