## Summary

Keep open. Current main still lacks the requested live abort-source diagnostic, the linked implementation PR is closed unmerged, and stack-frame abort logging is a core diagnostics/security surface that needs maintainer approval before automation is safe.

## What This Changes

Add a default-off structured diagnostic for embedded runner aborts that classifies the abort source and logs bounded caller stack frames.

## Best Possible Solution

Decide the diagnostic surface first; if accepted, add a default-off abort-source diagnostic at abortRun using existing diagnostics flags, bounded/redacted stack frames, docs, and focused helper tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers approve the diagnostic surface. A single helper at abortRun is technically narrow, but using the existing diagnostics flag model with explicit redaction is safer than adding a new env-only stack logger without review.
