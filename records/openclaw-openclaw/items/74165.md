## Summary

Keep open. Current main still lacks the requested live abort-source diagnostic, the linked implementation PR is closed unmerged, and stack-frame abort logging is a core diagnostics/security surface that needs maintainer approval before a repair PR is safe.

## What This Changes

Request to add a default-off structured diagnostic that logs embedded-run abort source and caller stack frames when `OPENCLAW_LOG_ABORT_SOURCES=1` is enabled.

## Best Possible Solution

If maintainers accept the diagnostic, implement it as a narrow default-off helper at the shared `abortRun` entry point using the approved diagnostics flag surface, bounded/redacted stack output, docs, and focused timeout/compaction tests.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The source check is high-confidence: current main has the shared abort path and abort-state metadata, but not the requested env gate, classifier, `[abort-source]` marker, stack capture, or tests.

## Solution Assessment

Unclear until maintainers approve the diagnostic surface. A single helper called from `abortRun` is technically narrow, but using the existing diagnostics flag model with explicit stack redaction is safer than adding a new env-only stack logger without review.
