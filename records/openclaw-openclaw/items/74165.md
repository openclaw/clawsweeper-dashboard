## Summary

Keep open: current main still lacks the requested embedded-runner abort-source diagnostic, and the linked implementation PR is closed unmerged. This is a new core diagnostics/config surface that would log abort reasons and stack frames, so it needs maintainer product and security review before implementation.

## What This Changes

Request an opt-in embedded-runner diagnostic, gated by `OPENCLAW_LOG_ABORT_SOURCES=1`, that logs one structured abort-source line with caller stack frames whenever `abortRun()` is entered.

## Best Possible Solution

If maintainers accept the feature, implement it through the existing diagnostics flag model with bounded/redacted reason and stack output plus focused classifier and emission tests.

## Reproduction Assessment

Not applicable: this is a feature request for a new diagnostics surface, not a broken existing contract. Source inspection on current main confirms the requested env flag, log marker, classifier, stack capture, and tests are absent.

## Solution Assessment

Unclear until maintainers approve the diagnostic surface. The single `abortRun` hook is technically narrow, but routing it through `diagnostics.flags`/`OPENCLAW_DIAGNOSTICS` with explicit redaction is safer than adding an unreviewed standalone stack logger.
