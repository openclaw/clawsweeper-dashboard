## Summary

Keep open. Current main and latest release v2026.6.6 still provide automatic cache-TTL pruning, pressure/overflow truncation, manual compaction, and whole-session reset, but not a caller-directed clear/archive/recover operation for a stable tool-result entry; the duplicate at https://github.com/openclaw/openclaw/issues/82054 already routes the same design here, and https://github.com/openclaw/openclaw/pull/81190 only addresses overflow recovery ordering.

## What This Changes

Manual context clearing for tool results requests a caller-directed way to purge large, no-longer-needed tool outputs from active context before automatic TTL or overflow pruning runs.

## Best Possible Solution

Define one session/context-engine-owned lifecycle operation for stable tool-result release or archival, including persistence, recoverability, transcript integrity, and authorization semantics, then expose only approved caller surfaces as adapters.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.6 expose automatic TTL pruning, pressure-driven truncation, compaction, and reset, but no caller-directed release by stable tool-result identity.

## Solution Assessment

No, not as independent CLI, agent function, and per-tool-call implementations. The maintainable path is one canonical session/context-engine lifecycle operation with explicit persistence, recovery, authorization, and transcript-integrity semantics.
