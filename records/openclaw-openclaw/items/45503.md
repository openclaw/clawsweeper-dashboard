## Label Justifications

## Summary

Keep open. Current main and v2026.6.6 still provide automatic in-memory tool-result pruning, overflow truncation, manual compaction, and session reset, but not caller-directed clear/archive/recover for a stable tool-result entry; the duplicate manual-free issue was closed into this thread, while the open overflow PR only addresses recovery ordering.

## What This Changes

Manual context clearing for tool results is requested so large one-off tool outputs can be removed from active context before automatic TTL or overflow pruning runs.

## Best Possible Solution

Define one session/context-engine-owned lifecycle operation for stable tool-result release or archival, then expose any approved CLI, agent, or tool-call surfaces as adapters over that operation.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.6 expose automatic TTL/pressure pruning and manual compaction/reset, but no caller-directed release by stable tool-result identity in the context engine or session protocol.

## Solution Assessment

No, not as separate CLI, agent function, and per-tool-call implementations. The maintainable path is one canonical context/session lifecycle operation with explicit persistence, recovery, authorization, and transcript-integrity semantics.
