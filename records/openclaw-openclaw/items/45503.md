## Summary

Keep open: current main still has TTL/policy-driven pruning, manual session-wide compaction, and overflow truncation paths, but no targeted user or agent control surface for clearing, archiving, or recovering a specific completed tool result. The closed related request at https://github.com/openclaw/openclaw/issues/82054 explicitly points back here as the canonical design thread.

## What This Changes

Requests a user or agent-accessible way to manually purge or archive completed tool results from active context before TTL pruning expires.

## Best Possible Solution

Design an additive prompt-view clear/archive/recover mechanism for completed tool results that preserves raw transcript integrity, provider pairing, compaction behavior, and a documented user or agent API.

## Reproduction Assessment

Not applicable: this is a feature request, not broken behavior under an existing contract. Source inspection gives high-confidence evidence of the feature gap because pruning is cache-TTL/policy driven, `/context` is inspect-only, `/compact` is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The suggested CLI command, agent function, and clear-after-use flag are plausible entry points, but the maintainable solution needs maintainer API design around prompt-view archive/recover semantics rather than deleting transcript history.
