## Summary

Keep open. Current main still has automatic cache-TTL/tool-result pruning and manual session compaction, but no user- or agent-addressable way to clear, archive, or recover a specific completed tool result; the remaining work is a feature/API design decision, not stale cleanup.

## What This Changes

Add a manual way to purge large completed tool results from active context before TTL-based pruning expires.

## Best Possible Solution

Design an additive prompt-view clear/archive/recover mechanism for completed tool results that preserves raw transcripts and provider pairing, works with context engines and compaction, and documents the supported user or agent API.

## Reproduction Assessment

Not applicable as a runtime bug, but the feature gap is high-confidence from source inspection: cache-TTL pruning is policy-driven, /context is inspect-only, /compact is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The suggested CLI command, agent function, and clearAfterUse flag are plausible entry points, but the maintainable solution should be designed around prompt-view archive/recover semantics rather than deleting transcript history.
