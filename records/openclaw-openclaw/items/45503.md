## Summary

Keep open: current main still offers TTL/policy-driven tool-result pruning, manual session-wide compaction, and overflow truncation, but no user- or agent-addressable clear/archive/recover API for a specific completed tool result. The duplicate at https://github.com/openclaw/openclaw/issues/82054 was already routed back here, while https://github.com/openclaw/openclaw/pull/81190 only addresses overflow recovery ordering, not the requested lifecycle contract.

## What This Changes

The issue requests a user- or agent-addressable way to purge completed tool results from active context once they are no longer useful.

## Best Possible Solution

Design an additive prompt-view clear/archive/recover mechanism for completed tool results that preserves raw transcript integrity and provider pairing, works with context engines and compaction, and documents the supported user or agent API.

## Reproduction Assessment

Not applicable: this is a feature request, not broken behavior under an existing contract. Source inspection gives a high-confidence feature gap because pruning is cache-TTL/policy driven, /context is inspect-only, /compact is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The suggested CLI command, agent function, and clearAfterUse flag are plausible entry points, but the maintainable solution needs maintainer API design around prompt-view archive/recover semantics rather than deleting transcript history.
