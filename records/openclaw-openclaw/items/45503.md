## Summary

Keep open. Current main has automatic context-bloat mitigations, but it still lacks a user- or agent-addressable operation to clear, archive, or recover a specific completed tool result, and the remaining API/replay semantics need maintainer design.

## What This Changes

Request a manual way to purge or archive large completed tool results from active model context once they are no longer needed.

## Best Possible Solution

Define a core prompt-view clear/archive mechanism for completed tool results that preserves the raw transcript and provider pairing invariants, supports intentional recovery or durable lookup, and documents interaction with /context, /compact, cache-TTL pruning, context engines, and session replay.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature gap. The gap is reproducible by source inspection on current main: pruning is TTL/policy-driven, /context is reporting-only, /compact is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The proposed CLI command, agent function, and clearAfterUse flag are plausible entry points, but the safer maintainable solution is a prompt-view archive/clear design rather than deleting transcript history.
