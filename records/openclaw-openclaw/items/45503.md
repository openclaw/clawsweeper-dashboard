## Summary

Keep open: current main and the latest release still only provide TTL/policy-driven tool-result pruning, `/context` inspection, and session-wide `/compact`; the requested targeted clear/archive/recover lifecycle is still a new product/API decision.

## What This Changes

The issue requests a user or agent-controlled way to clear completed tool results from active context before TTL-based pruning expires.

## Best Possible Solution

Design an additive prompt-view clear/archive/recover mechanism for completed tool results that preserves raw transcript integrity, provider pairing, compaction behavior, authorization, and a documented user or agent API.

## Reproduction Assessment

Not applicable: this is a feature request, not broken behavior under an existing contract. Source inspection gives high-confidence evidence of the gap because pruning is cache-TTL/policy-driven, `/context` is inspect-only, and `/compact` is session-scoped.

## Solution Assessment

Unclear: the suggested CLI command, agent function, and clear-after-use flag are plausible entry points, but the maintainable solution needs maintainer API design around prompt-view archive/recover semantics rather than deleting transcript history.
