## Summary

Keep open. Current main has automatic tool-result truncation and cache-TTL pruning, but no user- or agent-addressable way to clear, archive, or recover a specific completed tool result; the remaining work is a core product/API design choice rather than a small bug fix.

## What This Changes

The issue requests a manual way to purge completed tool results from active context once they are no longer useful.

## Best Possible Solution

Define a prompt-view clear/archive mechanism for completed tool results that preserves raw transcript and provider pairing invariants, supports intentional recovery, and documents interaction with /context, /compact, cache-TTL pruning, context engines, and session replay.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature gap. The gap is source-reproducible on current main because pruning is TTL/policy-driven, /context is inspect-only, /compact is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The suggested CLI command, agent function, and clearAfterUse flag are plausible entry points, but the narrow maintainable solution should be designed around prompt-view archive/recover semantics rather than deleting transcript history.
