## Summary

Keep open. Current main still has automatic cache-TTL tool-result pruning, manual `/compact`, and session reset controls, but no user- or agent-addressable way to clear, archive, or recover a specific completed tool result; this is a feature/API design item, not stale cleanup.

## What This Changes

Request a user or agent mechanism to manually purge completed tool results from active context instead of waiting for cache-TTL pruning.

## Best Possible Solution

Design an additive prompt-view clear/archive/recover mechanism for completed tool results that preserves raw transcripts and provider pairing, works with context engines and compaction, and documents the supported user or agent API.

## Reproduction Assessment

Not applicable as a runtime bug. The feature gap is high-confidence from source inspection: cache-TTL pruning is policy-driven, `/context` is inspect-only, `/compact` is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The suggested CLI command, agent function, and `clearAfterUse` flag are plausible entry points, but the maintainable solution should likely use prompt-view archive/recover semantics rather than deleting transcript history.
