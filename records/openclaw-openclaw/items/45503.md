## Summary

Keep open. Current main still provides automatic cache-TTL pruning, broad session compaction/reset, and overflow-oriented tool-result truncation, but no user- or agent-addressable API to clear, archive, or recover a specific completed tool result.

## What This Changes

Request a manual way to purge or archive completed tool results from active context once they are no longer useful.

## Best Possible Solution

Design an additive prompt-view clear/archive/recover mechanism for completed tool results that preserves raw transcripts and provider pairing, works with context engines and compaction, and documents the supported user or agent API.

## Reproduction Assessment

Not applicable as a runtime bug. The feature gap is high-confidence from current-main source: cache-TTL pruning is policy-driven, `/context` is inspect-only, `/compact` is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The proposed CLI command, agent function, and `clearAfterUse` flag are plausible entry points, but the maintainable solution needs maintainer API design around prompt-view archive/recover semantics rather than deleting transcript history.
