## Summary

Keep open: current main still provides TTL/policy-driven pruning, overflow truncation, `/context` inspection, and session-wide compaction/reset controls, but no targeted user or agent API to clear, archive, or recover a specific completed tool result.

## What This Changes

Request a manual lifecycle control for large completed tool results so they can be removed from active prompt context before TTL-based pruning runs.

## Best Possible Solution

Design an additive prompt-view clear/archive/recover mechanism for completed tool results that preserves raw transcript integrity, provider pairing, compaction behavior, and a documented user or agent API.

## Reproduction Assessment

Not applicable: this is a feature request, not broken behavior under an existing contract. Source inspection gives high-confidence evidence of the gap because pruning is cache-TTL/policy driven, `/context` is inspect-only, `/compact` is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The suggested CLI command, agent function, and clear-after-use flag are plausible entry points, but the maintainable solution needs maintainer API design around prompt-view archive/recover semantics rather than deleting transcript history.
