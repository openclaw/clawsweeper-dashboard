## Summary

Keep open. Current main only supports policy/TTL-based in-memory tool-result pruning plus session-wide compaction; a targeted user- or agent-addressable clear/archive/recover path is still absent and needs core product/API design.

## What This Changes

Requested a manual way to purge or archive large completed tool results from active context before automatic cache-TTL pruning expires.

## Best Possible Solution

Define a prompt-view clear/archive mechanism for completed tool results that preserves raw transcript history and provider pairing, supports intentional recovery, and documents its interaction with `/context`, `/compact`, cache-TTL pruning, context engines, and session replay.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature gap. The gap is source-reproducible on current main because pruning is TTL/policy-driven, `/context` is inspect-only, `/compact` is session-scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The suggested CLI command, agent function, and `clearAfterUse` flag are plausible entry points, but the safer maintainable direction is prompt-view archive/recover semantics rather than deleting transcript history.
