## Summary

Keep open. Current main and v2026.6.6 provide automatic cache-TTL pruning, pressure/overflow truncation, compaction, and whole-session reset, but they still do not provide a caller-directed way to release or recover a stable tool-result entry; the newer duplicate already routes the same design back to this issue, and the related overflow pull request only covers recovery ordering.

## What This Changes

Requested a way to manually purge no-longer-needed tool results from active context before automatic TTL, overflow truncation, or compaction runs.

## Best Possible Solution

Define one session/context-engine-owned lifecycle operation for stable tool-result release, including persistence, recoverability, transcript integrity, and authorization semantics, then expose only approved CLI or agent callers as thin adapters.

## Reproduction Assessment

Yes, at the source level. Current main and v2026.6.6 expose automatic TTL pruning, pressure-driven truncation, compaction, and reset, but no caller-directed release by stable tool-result identity.

## Solution Assessment

No, not as three independent implementations. The maintainable solution is one canonical session/context-engine lifecycle operation with explicit persistence, recovery, authorization, and transcript-integrity semantics, followed by whichever caller surfaces maintainers approve.
