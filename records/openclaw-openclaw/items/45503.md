## Summary

Keep open. Current main has several context-bloat mitigations, but it still lacks a user- or agent-addressable operation to clear, archive, or recover a specific completed tool result; the discussion also raises API and replay semantics that need maintainer product judgment.

## What This Changes

Request a manual way for users or agents to remove completed tool results from active context after they are no longer useful.

## Best Possible Solution

Define a core prompt-view clear/archive mechanism for completed tool results that preserves the raw transcript and provider pairing invariants, supports an intentional recoverable placeholder or durable lookup, and documents its interaction with /context, /compact, cache-TTL pruning, context engines, and session replay.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature gap. The gap is reproducible by source inspection on current main: pruning is TTL/policy-driven, /context is inspect-only, /compact is whole-session scoped, and no targeted clear/archive/recover selector exists.

## Solution Assessment

Unclear. The proposed CLI command, agent function, and clearAfterUse flag are plausible entry points, but the safer maintainable direction is a prompt-view archive/clear design that does not physically delete history or violate provider replay requirements.
