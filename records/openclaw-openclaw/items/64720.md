## Summary

The PR addresses a real prompt-boundary gap, but it should stay open because the current diff still has blocker-level trust-classification bugs and lacks after-fix real behavior proof for an external contributor PR.

## What This Changes

The branch adds `wrapExternalContent` handling for channel message bodies and thread context, adds a `channel` external-content source label, and adds inbound-text unit coverage.

## Best Possible Solution

Land a revised implementation that wraps genuinely external channel body and thread text using explicit ingress/source trust metadata, while preserving trusted scheduler/system-event prompts and leaving broader memory provenance to the canonical follow-up.

## Reproduction Assessment

Yes, by source inspection. Current main sends channel body and thread context into prompt assembly without `wrapExternalContent`, and the PR's remaining regressions are reproducible from the route/provider fields and scheduler provider values.

## Solution Assessment

No. Nonce-backed external-content wrapping is the right direction, but this patch should derive wrapping from explicit ingress/source trust metadata instead of `OriginatingChannel` and an incomplete allowlist.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Derive wrapping from ingress trust:** `src/auto-reply/reply/get-reply-run.ts:270-271`
  - body: `OriginatingChannel` is reply-route metadata, but this branch uses it before `Provider` to decide whether prompt text is attacker-supplied. Heartbeat or cron turns routed to an external channel can therefore have trusted scheduler prompts wrapped as untrusted channel content, which can distort automated actions.
  - confidence: 0.9
- **[P1] Trust scheduler providers:** `src/auto-reply/reply/inbound-text.ts:7`
  - body: The new allowlist omits `heartbeat` and `cron-event`, even though current main emits both as system-event providers. Internal scheduler runs without an originating channel would now be fenced as untrusted external content, changing trusted automation prompts.
  - confidence: 0.88
