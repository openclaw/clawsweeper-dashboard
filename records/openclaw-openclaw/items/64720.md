## Summary

Keep open: the security goal is valid and current main still lacks channel body/thread wrapping, but this branch introduces P1 trust-boundary regressions for heartbeat and cron/system-event runs, is conflicting with current main, and still lacks after-fix real behavior proof.

## What This Changes

The PR wraps channel message body and thread context with `wrapExternalContent`, adds a `channel` external-content source label, and adds inbound text unit tests.

## Best Possible Solution

Define an owner-approved ingress trust boundary, wrap only external channel body/thread text with the existing nonce-backed helper, preserve scheduler/system-event prompts, and add regression tests plus redacted real behavior proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main sends sanitized channel body/thread text into prompt assembly without external-content framing, and the PR diff shows heartbeat/cron-event misclassification through `OriginatingChannel` and the provider allowlist.

## Solution Assessment

No, not as currently implemented. The nonce-backed wrapper is the right primitive, but the trust decision needs explicit ingress/system-event provenance rather than mutable routing strings, plus upgrade proof for scheduler paths.
