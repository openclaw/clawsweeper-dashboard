## Summary

Keep open. Current main still lacks channel-body wrapping, and this PR is directionally useful but not mergeable because it can misclassify trusted heartbeat/cron prompts as external channel text; the follow-up commit appears to have addressed the earlier queue-summary and thread-context comments.

## What This Changes

The PR adds external-content wrapping helpers and tests for channel message bodies/thread context, plus a new channel source label for wrapped content.

## Best Possible Solution

Revise the PR to derive wrapping from an explicit ingress/source trust class, keep wrapping for real external channel body and thread context, preserve raw queue summaries, add the changelog entry, and cover heartbeat/cron/exec trusted paths in tests.

## Reproduction Assessment

Yes. Static inspection is enough: current main emits heartbeat/cron MsgContext with Provider set to an internal source while OriginatingChannel can be an external delivery route, and the PR diff chooses OriginatingChannel first.

## Solution Assessment

No. The direction is right, but route metadata plus an incomplete trusted-provider allowlist is not the narrowest maintainable fix; an explicit ingress trust/source seam is safer.
