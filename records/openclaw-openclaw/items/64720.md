## Summary

Keep open. Current main still leaves channel bodies unwrapped, so this PR targets a real security gap, but the current patch misclassifies internal heartbeat/cron turns as external channel content and only provides test proof, not real after-fix behavior proof.

## What This Changes

The PR wraps channel message bodies and thread context with `wrapExternalContent`, adds a `channel` external-content source label, and adds unit coverage for inbound text helpers.

## Best Possible Solution

Revise this PR to derive wrapping from an explicit ingress/source trust class, wrap only real external channel body and thread context, preserve trusted scheduler/CLI/node prompts, and coordinate memory provenance through https://github.com/openclaw/openclaw/issues/69401.

## Reproduction Assessment

Yes. Static source inspection reproduces the PR regression: internal heartbeat/cron turns can have trusted `Provider` values while retaining external `OriginatingChannel` delivery routes, and the PR uses the route first.

## Solution Assessment

No. The direction is right, but route metadata plus a hard-coded incomplete trusted-provider allowlist is not the narrowest maintainable trust-boundary fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use ingress trust, not reply route, for wrapping:** `src/auto-reply/reply/get-reply-run.ts:268`
  - body: `OriginatingChannel` is reply-routing metadata. Heartbeat and cron turns can set an external route for delivery while the prompt source remains trusted, so this wraps scheduler prompts as untrusted channel content and can degrade scheduled actions.
  - confidence: 0.9
- **[P1] Keep internal event providers out of channel wrapping:** `src/auto-reply/reply/inbound-text.ts:7`
  - body: `TRUSTED_PROVIDERS` omits current internal providers such as `heartbeat` and `cron-event`. When no originating channel is present, those trusted automation prompts now fall through as external channel messages.
  - confidence: 0.88
