## Summary

Keep open. The security goal is valid, but this PR is not merge-ready because it conflicts with current main, lacks real behavior proof, and the diff can reclassify trusted heartbeat/cron prompts as external channel content.

## What This Changes

The branch adds external-content wrapping for channel message bodies and thread context in auto-reply prompt assembly, adds a channel source label, and adds inbound text unit tests.

## Best Possible Solution

Land a rebased security-boundary implementation that wraps genuinely external channel body and thread text from explicit ingress trust metadata, preserves heartbeat/cron/exec prompts, and coordinates broader memory provenance through the canonical issue.

## Reproduction Assessment

Yes. Source inspection shows current main lacks structural wrapping for channel body/thread text, and the PR diff shows heartbeat/cron-event misclassification through OriginatingChannel-first routing and a narrow trusted-provider allowlist.

## Solution Assessment

No. Nonce-backed wrapping is the right direction, but this patch should derive trust from explicit ingress/source metadata and preserve trusted scheduler providers before merge.
