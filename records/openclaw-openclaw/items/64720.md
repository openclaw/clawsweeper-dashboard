## Summary

Keep open. The security hardening direction is valid, but this branch is conflicting and still classifies scheduler/system-event prompts from route metadata as external channel content while lacking real behavior proof.

## What This Changes

This branch adds channel external-content wrapping helpers, applies them to auto-reply base and thread context text, extends external-content source labels, and adds inbound text unit tests.

## Best Possible Solution

Land a rebased security-boundary implementation that wraps genuinely external channel body and thread text from explicit ingress trust metadata, preserves heartbeat/cron/exec prompts, and leaves broader memory provenance to the canonical design issue.

## Reproduction Assessment

Yes. Source inspection shows current main lacks structural wrapping for channel body/thread text, and the PR diff shows heartbeat/cron-event misclassification through OriginatingChannel-first routing and a narrow trusted-provider allowlist.

## Solution Assessment

No. Nonce-backed wrapping is the right direction, but this patch should derive trust from explicit ingress/source metadata and preserve trusted scheduler providers before merge.
