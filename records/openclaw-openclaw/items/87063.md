## Summary

Keep open: current main has fixed the direct CLI-runtime alias selection throw, but the Claude CLI path still exposes the same OpenClaw skills through both the system prompt and a Claude plugin, so the context-bloat half remains actionable.

## What This Changes

The issue reports that forcing `agentRuntime.id` to `claude-cli` can hit `MissingAgentHarnessError` and that Claude CLI runs duplicate the skills catalog enough to inflate context use.

## Best Possible Solution

Make Claude CLI use one authoritative skills exposure path, preferably the native `--plugin-dir` path with a prompt fallback only when the plugin cannot be prepared, and update the tests/docs while keeping the current CLI-alias passthrough fix.

## Reproduction Assessment

## Solution Assessment
