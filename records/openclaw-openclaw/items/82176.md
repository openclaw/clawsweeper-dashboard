The PR adds a built-in `sessions_search` agent tool and wires it into session tool catalogs, policies, display metadata, docs, and tests.

## Best Possible Solution

Keep the tool only if it searches and returns sanitized, redacted, bounded transcript text consistent with `sessions_history`, with regression coverage and real OpenClaw tool-run proof.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows raw transcript files are read and previews are built directly from matching raw lines. I did not run a PR-branch repro because this review is read-only.

## Solution Assessment

No. Reusing `sessions_list` for visibility is a good direction, but returned search previews should pass through the same sanitization and redaction boundary as `sessions_history` before becoming a built-in tool result.
