## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/84817. Current `gateway install --force` can persist a launchd ProgramArguments entry using `dist/index.js`; the reporter shows that this breaks bundled plugin loading under launchd while the package launcher wrapper works. Update gateway service entrypoint resolution so packaged managed gateway installs prefer the supported package launcher/entry wrapper (`openclaw.mjs` or an equivalent wrapper path) when available, while preserving stable symlink/package-manager paths and keeping dist fallbacks for source/dev or missing-wrapper cases. Do not remove the stable pnpm path handling from prior gateway entrypoint fixes. Add regression tests for package-root wrapper selection and for the fallback behavior, and add a changelog entry because this is user-visible gateway install behavior. Validate with focused Vitest coverage plus real macOS launchd proof that `openclaw gateway install --force` starts the gateway and loads bundled plugins without the reported `Unknown system error -11`.
