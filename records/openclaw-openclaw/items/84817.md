## Summary

Keep open. Current `main` and the latest release still default managed gateway service arguments to Node plus a resolved `dist/index.js` entrypoint when no wrapper is configured, so the reported launchd failure is not implemented or superseded.

## What This Changes

The issue reports that `openclaw gateway install --force` writes a launchd plist using the `dist/index.js` entrypoint, which causes gateway plugin load failures under launchd while the npm-runtime wrapper works.

## Best Possible Solution

Make the managed macOS gateway service select a launchd-safe packaged launcher or preserve a detected working wrapper, while keeping explicit `--wrapper`/`OPENCLAW_WRAPPER` semantics intact and proving `gateway install --force` under launchd loads plugins cleanly.

## Reproduction Assessment

No, not as a live macOS launchd reproduction in this read-only review. Source inspection does confirm current `main` and `v2026.6.6` still emit the reported default `dist/index.js` entrypoint when no wrapper is configured.

## Solution Assessment

Unclear until a patch is proven on macOS. The narrow maintainable path is to adjust gateway service entrypoint/wrapper resolution and preserve explicit wrapper behavior, not to add another user-facing setting.
