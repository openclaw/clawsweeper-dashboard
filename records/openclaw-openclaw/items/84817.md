## Summary

Keep open. The report matches current `main`: managed gateway install still resolves the service command through `resolveGatewayProgramArguments`, tests explicitly prefer `dist/index.js`, and the wrapper path that matches the workaround is only opt-in or preserved from `OPENCLAW_WRAPPER`.

## What This Changes

The issue asks OpenClaw to stop `gateway install --force` from rewriting macOS launchd to a `dist/index.js` entrypoint that causes plugin load failures and a restart loop.

## Best Possible Solution

Make the managed macOS gateway service prefer the packaged `openclaw.mjs`/npm-runtime launcher or preserve a detected working wrapper when present, keep explicit `--wrapper` and `OPENCLAW_WRAPPER` semantics intact, and prove `gateway install --force` starts under launchd with plugins loaded.

## Reproduction Assessment

No, not as a live macOS launchd reproduction in this review. Source inspection does confirm the current install path still emits the reported `dist/index.js` entrypoint by default, while the working wrapper path is only opt-in or preserved from service environment state.

## Solution Assessment

Unclear until a patch is proven on macOS. The narrowest likely fix is to adjust the supervised launchd entrypoint resolution or wrapper preservation instead of adding another user-facing knob.
