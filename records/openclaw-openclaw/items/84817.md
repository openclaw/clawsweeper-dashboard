## Summary

Keep open. Current `main` and latest release `v2026.6.8` still route wrapperless managed gateway installs through a resolved `dist/index.js` entrypoint, and related items cover adjacent errno, wrapper-diagnostics, or PortGuardian behavior rather than this launchd plugin-load failure.

## What This Changes

The issue reports that `openclaw gateway install --force` writes a launchd service command using `dist/index.js`, which can leave the macOS launchd-managed gateway in a plugin-load restart loop while the npm runtime wrapper works.

## Best Possible Solution

Make macOS managed gateway install and repair choose a launchd-safe packaged launcher or preserve an explicit working wrapper, while keeping stable update paths and proving plugins load cleanly under launchd.

## Reproduction Assessment

No, not as a live macOS launchd reproduction in this read-only review. Source inspection does confirm current `main` and `v2026.6.8` still emit the reported default `dist/index.js` entrypoint when no wrapper is configured.

## Solution Assessment

Unclear until a patch is proven on macOS launchd. The maintainable direction is to fix gateway service entrypoint/wrapper resolution, not to add a new user-facing setting or treat plugin loaders as the primary fault site.
