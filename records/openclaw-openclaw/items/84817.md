## Summary

Keep open. Current main and the latest release still route managed gateway installs through `dist/index.js` when no wrapper is supplied, tests lock in that preference, and the workaround path is only available through explicit or preserved `OPENCLAW_WRAPPER` state, so the reported launchd restart-loop failure remains actionable.

## What This Changes

The issue reports that `gateway install --force` writes a launchd service entrypoint through `dist/index.js`, causing plugin and secrets loading failures under launchd while the npm-runtime `openclaw.mjs` wrapper works.

## Best Possible Solution

Make the managed macOS gateway install use or preserve a launchd-safe packaged `openclaw.mjs`/npm-runtime entrypoint when available, keep explicit `--wrapper` and `OPENCLAW_WRAPPER` semantics intact, and prove `gateway install --force` starts under launchd with plugins loaded.
