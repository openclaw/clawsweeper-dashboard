## Label Justifications

- `P1`: A forced macOS gateway service reinstall can leave affected launchd gateways in a restart loop with plugins failing to load.
- `impact:crash-loop`: The reported failure mode is a launchd-managed gateway repeatedly failing during startup after installation.

## Summary

Keep open. Current main and latest release still route wrapperless managed gateway installs through an `index.js` dist entrypoint, while the open related PRs/issues cover diagnostics or adjacent gateway-service behavior rather than this launchd plugin-load failure.

## What This Changes

Issue reports that `gateway install --force` writes a launchd plist entrypoint pointing at `dist/index.js`, causing gateway plugin load failures and a launchd restart loop on macOS.

## Best Possible Solution

Make macOS managed gateway install and repair choose a launchd-safe packaged launcher or preserve an explicit working wrapper, then prove plugin loading under launchd on a real macOS service install.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live macOS launchd reproduction was run in this read-only review, but source inspection confirms the reported wrapperless `dist/index.js` install path remains present on current main and latest release.

## Solution Assessment

Is this the best way to solve the issue? Unclear until a patch is proven on macOS launchd; the maintainable direction is to fix gateway service entrypoint or wrapper resolution rather than adding a new user-facing setting or treating plugin loaders as the primary fault site.
