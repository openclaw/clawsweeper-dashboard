Keep open. Current main and the latest shipped release still lack a modifying plugin hook between final assistant text and TTS directive parsing, and the requested seam needs maintainer API plus conversation-access security review before implementation.

## What This Changes

Request a new modifying plugin hook between final assistant reply generation and TTS directive parsing so plugins can inject per-message voice directives or skip TTS.

## Best Possible Solution

Add a documented, backward-compatible pre-TTS transform hook only after maintainer approval of the hook contract, access policy, TTS scope, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks a before_tts or reply_transform-style modifying seam before TTS processing.

## Solution Assessment

Unclear for the exact API name and result shape. The maintainable direction is a generic documented hook with explicit access gating, not forcing plugins to own reply_dispatch or rely on model prompt instructions.
