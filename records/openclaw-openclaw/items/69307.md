## Summary

Keep open. Current main still lacks a plugin-modifying seam between final assistant text creation and TTS directive parsing; the existing hooks are either observational, too early, after TTS, or require owning dispatch entirely, so this is a real plugin API design item rather than cleanup closure.

## What This Changes

Request to add a plugin hook that can modify final assistant text before TTS directive parsing so plugins can route or skip voices per message.

## Best Possible Solution

Add a documented, backward-compatible pre-TTS modifying hook in the plugin SDK, with focused dispatch/TTS tests for directive injection and TTS suppression while keeping voice-routing policy in plugins.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested plugin-modifying seam between final reply text creation and TTS processing.

## Solution Assessment

Yes on the direction, but not yet on the exact API. A generic pre-TTS modifying hook is narrower than forcing plugins to own reply_dispatch or prompting the model to emit directives, but the SDK contract and access-control policy need maintainer approval.
