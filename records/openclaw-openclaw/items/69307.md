The issue requests a pre-TTS plugin hook that can transform final assistant reply text before TTS directive parsing for per-message voice routing.

## Best Possible Solution

Add a documented, backward-compatible pre-TTS modifying hook in the plugin SDK only after maintainers approve the API shape and conversation-access gating, with focused hook-runner and reply-dispatch/TTS coverage.
