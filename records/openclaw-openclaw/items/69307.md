Keep open. Current main still lacks a plugin-modifying seam between final assistant text and TTS directive parsing, and the related TTS and hook items do not supersede this specific pre-TTS plugin API request.

## What This Changes

Request a new modifying plugin hook before TTS processing so plugins can inject voice directives or skip TTS per message.

## Best Possible Solution

Add a documented, backward-compatible pre-TTS modifying hook in the Plugin SDK, with focused dispatch/TTS tests for directive injection and TTS suppression while keeping voice-routing policy in plugins.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested modifying seam between final reply text creation and TTS processing.

## Solution Assessment

Yes on the direction, but unclear on the exact API. A generic pre-TTS modifying hook is narrower than forcing plugins to own reply_dispatch or prompting the model to emit directives, but naming, access gating, and scope need maintainer approval.
