## Summary

Keep open. Current main still lacks a modifying plugin seam between final reply creation and TTS directive parsing, so this remains a real plugin API gap rather than a ClawHub-only extension request.

## What This Changes

Request a new pre-TTS plugin hook so plugins can inject per-message `[[tts:voice=...]]` directives or skip TTS based on external routing state.

## Best Possible Solution

Add a documented, backward-compatible modifying hook at the final-reply/pre-TTS boundary, with focused tests for directive injection and TTS suppression while keeping voice-routing policy in plugins.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that the requested pre-TTS transform seam is absent on current main.

## Solution Assessment

Yes on the overall direction: a generic pre-TTS modifying hook is the narrowest maintainable way to let plugins inject `[[tts:...]]` directives without owning full reply dispatch. The exact public SDK contract still needs maintainer approval before implementation.
