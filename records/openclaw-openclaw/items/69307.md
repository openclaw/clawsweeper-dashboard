## Summary

Keep open. Current main still lacks a plugin-modifying seam between final reply payload creation and TTS directive parsing, and this is a real public plugin API gap that needs maintainer design review rather than cleanup closure.

## What This Changes

The issue requests a plugin hook that can modify final assistant text before TTS directive parsing so plugins can inject or suppress per-message voice routing.

## Best Possible Solution

Add a documented, backward-compatible pre-TTS modifying hook in the plugin SDK, with focused dispatch/TTS tests for directive injection and TTS suppression while keeping voice-routing policy in plugins.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that the requested pre-TTS modifying seam is absent on current main.

## Solution Assessment

Yes on the direction: a generic pre-TTS modifying hook is narrower than forcing plugins to own reply_dispatch or prompt the model to emit voice directives. The exact SDK contract and access-control model still need maintainer approval before implementation.
