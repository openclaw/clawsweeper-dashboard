## Summary

Keep open. Current main still lacks a modifying plugin seam between final reply text creation and TTS directive parsing, so plugins cannot reliably inject per-message voice directives before audio generation.

## What This Changes

Request adding a plugin hook between final LLM reply generation and TTS directive parsing so plugins can rewrite reply text or skip TTS per message.

## Best Possible Solution

Add a documented, backward-compatible modifying hook at the final-reply/pre-TTS boundary, with tests covering directive injection and TTS suppression while keeping voice-routing policy in plugins.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that the requested pre-TTS transform seam is absent on current main.

## Solution Assessment

Yes on the overall direction: a generic modifying hook before TTS is the narrowest maintainable way to let plugins inject [[tts:...]] directives without owning full reply dispatch. The exact public SDK contract still needs maintainer approval before implementation.
