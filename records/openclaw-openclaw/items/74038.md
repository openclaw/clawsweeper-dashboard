## Summary

Keep open. Current main still enables model-pricing bootstrap for replace-mode configs when `models.pricing.enabled` is omitted, so this PR addresses a valid remaining bug, but the branch is dirty against main and exact-head checks have failures that need rebase and maintainer review.

## What This Changes

The PR changes gateway model-pricing enablement so omitted pricing defaults off in `models.mode: "replace"`, with cache/startup tests, generated schema/help, docs, baseline hash, and changelog updates.

## Best Possible Solution

Rebase onto current main, keep the shared helper default change, preserve current after-ready activation semantics, update regression coverage as needed, and land after exact-head gates pass.

## Reproduction Assessment

Yes. Source-level reproduction on current main is `models: { mode: "replace" }` with omitted `models.pricing.enabled`: the shared helper returns true, and scheduled activation plus direct refresh use that value before remote pricing work.

## Solution Assessment

Yes. The helper-level default is the narrowest maintainable fix because the activation, bootstrap, and direct refresh paths already share `isGatewayModelPricingEnabled`, and the PR preserves explicit pricing overrides.
