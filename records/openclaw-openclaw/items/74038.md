## Summary

Keep open. Current main still enables the remote pricing bootstrap when `models.mode: "replace"` omits `models.pricing.enabled`, and this PR addresses that at the shared gateway helper with matching startup/cache coverage and docs; I found no discrete patch bug, but exact-head CI is red and the branch is behind current main.

## What This Changes

The PR defaults gateway model-pricing bootstrap off for replace-mode catalogs unless explicitly enabled, with regression tests, config help/schema/docs, baseline hash, and changelog updates.

## Best Possible Solution

Land this helper-level default change, or an equivalent maintainer revision, after rebasing and getting exact-head checks green; keep broader pricing resilience work in #75067.

## Reproduction Assessment

Yes. A source-level reproduction on current main is `models: { mode: "replace" }` with omitted `models.pricing.enabled`: the helper returns enabled, and startup/direct refresh use that helper before remote pricing bootstrap work.

## Solution Assessment

Yes. The PR’s shared-helper default is the narrowest maintainable fix because startup bootstrap and direct refresh already share that enablement path, and explicit pricing overrides remain intact.
