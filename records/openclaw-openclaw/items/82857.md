## What This Changes

Routes queued followup runs whose model resolves to a configured CLI backend through `runCliAgent` and adds a focused followup-runner regression test.

## Best Possible Solution

Land a narrow followup-runner fix that reuses the direct CLI dispatch contract, including room-event binding isolation and stable CLI session hash inputs, with real queued-followup CLI proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure a queued followup model with `agentRuntime.id` set to a CLI backend and current `main` still routes the queued attempt into `runEmbeddedPiAgent`. I did not execute the full live followup path in this read-only review.

## Solution Assessment

No, not yet: the PR is in the right narrow area, but the implementation omits direct-path CLI session semantics for room events and stable prompt hashing. The safer fix is to mirror the existing direct-run CLI parameter contract before merging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve room-event CLI session isolation:** `src/auto-reply/reply/followup-runner.ts:413-415`
  - body: The primary reply CLI path explicitly skips `getCliSessionBinding` for `currentInboundEventKind === "room_event"` and strips the returned session id, so message-tool-only room events do not resume or rewrite the user's live CLI session. This new followup path always passes the active binding, so queued room-event drains on CLI runtimes can contaminate the normal CLI session and persist the wrong binding. Mirror the direct path's room-event guard before passing `cliSessionId`/`cliSessionBinding`.
  - confidence: 0.89
- **[P2] Forward the stable extra prompt hash:** `src/auto-reply/reply/followup-runner.ts:409`
  - body: Queued runs carry `extraSystemPromptStatic` so CLI session reuse hashes ignore per-message inbound metadata. The new CLI branch forwards only `extraSystemPrompt`; `prepareCliRunContext` will hash the dynamic prompt instead and invalidate reusable CLI sessions as `system-prompt` across queued group or room followups. Pass `run.extraSystemPromptStatic` as the direct path does.
  - confidence: 0.86
