## Summary

Keep open. The patch is relevant, but it still leaves important cancellation paths uncovered: RPC `chat.abort` is not forwarded into manual compaction, harness-owned compactions return before the new active-run registration, and the bundled profile change removes explicit launchd label overrides.

## What This Changes

The PR wires queued embedded `/compact` into active-run cancellation, adds active-run exports and a previous-run drain guard, and bundles `--profile` / `OPENCLAW_LAUNCHD_LABEL` behavior changes with tests.

## Best Possible Solution

Land a focused compact cancellation fix that forwards the existing chat abort signal into manual compaction, registers an abortable handle for both harness-owned and queued compaction at the point work is actually active, cascades same-session reply abort when needed, cleans up in `finally`, and adds targeted cancellation regression coverage. Split the `OPENCLAW_LAUNCHD_LABEL` / `--profile` behavior for #65643 into a separate maintainer-reviewed change.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main and the PR head both show manual `/compact` can run without receiving the gateway `chat.abort` signal, and the PR head returns from harness-owned compaction before active-run registration.

## Solution Assessment

No. The PR is a useful start, but it is not the best complete fix because it only covers part of the queued embedded path, misses RPC chat abort propagation, leaves harness compaction unregistered, and bundles unrelated launchd/profile behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Forward chat abort into manual compaction:** `src/auto-reply/reply/commands-compact.ts:119-156`
  - body: `chat.abort` aborts the active chat controller signal, and the command pipeline carries that signal on `params.opts`. The branch's queued compaction only forwards `params.abortSignal`, but this call still omits `abortSignal: params.opts?.abortSignal`, so dashboard/RPC cancel can report success while manual compaction keeps running.
  - confidence: 0.88
- **[P2] Register harness compactions before returning:** `src/agents/pi-embedded-runner/compact.queued.ts:69-98`
  - body: The new active-run handle is registered only after `maybeCompactAgentHarnessSession(params)` returns undefined. Harnesses that implement `compact` still return before this handle exists, so `/stop` and `abortEmbeddedPiRun(sessionId)` continue to miss those in-flight compactions.
  - confidence: 0.87
- **[P2] Preserve custom launchd label overrides:** `src/cli/profile.ts:116-117`
  - body: This deletes every `OPENCLAW_LAUNCHD_LABEL` that differs from the profile-derived label, including intentional custom plist labels. Existing launchd and restart-helper code treats that variable as an explicit override, so profile-scoped commands can target the wrong custom service after this change.
  - confidence: 0.9
