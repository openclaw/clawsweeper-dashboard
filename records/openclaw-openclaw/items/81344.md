## Summary

Keep open: the PR fixes the main WebChat composer trim path and includes useful proof, but the queued-message Steer action still trims queued text before sending, leaving a user-visible whitespace-loss path in the same queue surface.

## What This Changes

The PR keeps WebChat message text untrimmed through `handleSendChat`, queue storage, optimistic user messages, and `chat.send`, with regression tests for multi-line diagram indentation.

## Best Possible Solution

Preserve the exact user-entered text for every normal WebChat send path, including queued and steered queued messages, while using trimmed text only for blank-message and command guards.
