## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register task and team in the core catalog:** `src/agents/openclaw-tools.ts:235-239`
  - body: The PR constructs `task` and `team` as core tools, but it never adds them to `CORE_TOOL_DEFINITIONS`. Profiles, `group:openclaw`, tool catalog output, and `isKnownCoreToolId()` are generated from that catalog, so these tools will not behave as first-class core tools under normal policy/catalog resolution.
  - confidence: 0.88
- **[P2] Emit flat enum schemas for task:** `src/agents/tools/task-tool.ts:31-39`
  - body: `TaskToolSchema` uses `Type.Union(...Type.Literal...)` for `action`, `runtime`, and `notifyPolicy`, which emits `anyOf`. Existing tool schemas use `stringEnum`/`optionalStringEnum` because some providers reject `anyOf`, so this can make the new task tool unusable with those providers.
  - confidence: 0.93
- **[P2] Emit a flat enum schema for team:** `src/agents/tools/team-tool.ts:21`
  - body: `TeamToolSchema.action` also uses `Type.Union(...Type.Literal...)`, producing the same provider-incompatible `anyOf` shape. Use the shared flat string enum helper here as the other action-dispatched tools do.
  - confidence: 0.92
- **[P2] Classify allowed-mount escapes as workspace boundaries:** `src/agents/fs-permission-denied.ts:22-25`
  - body: The classifier never matches `Sandbox path escapes allowed mounts`, but this PR wraps that exact current local and remote sandbox bridge message. Those denials become `reason=unknown` instead of `workspace_boundary`, defeating the standardized reason metadata.
  - confidence: 0.91
- **[P2] Reject unknown task actions before output lookup:** `src/agents/tools/task-tool.ts:232-241`
  - body: After the explicit task actions, the handler falls through to the output lookup path. Direct surfaces such as `/tools/invoke` call `execute` without TypeBox validation, so a typo like `ouptut` becomes a state-dependent `task not found` error instead of a clear unsupported-action client error.
  - confidence: 0.9
- **[P2] Translate missing team updates to ToolInputError:** `src/agents/tools/team-tool.ts:88-94`
  - body: `updateTeam()` throws a plain `Error` for an unknown team id, and this branch does not translate it. `/tools/invoke` maps only input/authorization errors to 4xx responses, so a user-correctable missing team update becomes a 500 `tool execution failed`.
  - confidence: 0.88
