  - confidence: 0.93

  - body: `openReadableFile` now wraps every failed boundary open as `permission_denied`, including ordinary missing-file opens. Current sandbox read behavior preserves not-found diagnostics, so this changes normal ENOENT/no-such-file reads into permission failures.
  - confidence: 0.91
- **[P2] Classify allowed-mount escapes as workspace boundaries:** `src/agents/fs-permission-denied.ts:22-25`
  - body: The classifier does not match the bridge phrase `Sandbox path escapes allowed mounts`, while this PR wraps local and remote sandbox boundary errors using that exact cause text. Those denials become `reason=unknown` instead of `workspace_boundary`.
  - confidence: 0.92

  - confidence: 0.9

  - body: `updateTeam()` throws a plain `Error` for an unknown team id, and this branch does not translate it. Direct tool invocation maps input errors differently from execution failures, so a user-correctable missing team update can surface as a tool failure.
  - confidence: 0.88
