## Summary

Keep open: the standalone telemetry filter is useful and the PR body gives real live proof for that case, but the new gate runs after Discord debounce coalescing and can drop an entire mixed bot batch before dispatch.

## What This Changes

The PR adds a Discord preflight helper that drops bot-authored operational telemetry-shaped messages and adds focused preflight tests for dropped telemetry and kept conversational bot text.

## Best Possible Solution

Filter operational telemetry at the original Discord inbound-entry level before debounce coalescing, while preserving conversational bot messages in mixed batches and adding a regression for that path.

## Reproduction Assessment

Yes for the review finding by source inspection: set a positive `messages.inbound.debounceMs`, allow bot messages, then send the same bot a telemetry line followed by conversational text within the debounce window. The handler coalesces them into one synthetic message, and the new preflight gate drops it because the joined text starts with telemetry.

## Solution Assessment

No. Filtering in preflight is acceptable for a single message, but it is too late for debounced batches; the maintainable fix is to filter original entries before coalescing or otherwise make the drop gate batch-aware.
