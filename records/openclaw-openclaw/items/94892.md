## Summary

Keep open: the Discord telemetry-filtering goal is valid, but the current patch can suppress legitimate bot-authored Discord conversation because it uses broad prefix matching before mention handling and after same-author debounce batching.

## What This Changes

Adds a Discord preflight detector that drops bot-authored operational telemetry-shaped messages and adds preflight tests for telemetry drops plus one conversational bot keep case.

## Best Possible Solution

Keep the Discord-owned telemetry filter, but make it a precise standalone-record detector and apply it before or around batching so telemetry-only entries are dropped without suppressing later conversational bot messages.

## Reproduction Assessment

Yes by source inspection: same-author Discord debounce can synthesize one message from multiple entries, and the PR-head prefix gate then returns null for the whole batch. I did not run tests in this read-only review.

## Solution Assessment

No. Filtering telemetry belongs in Discord preflight, but the implementation should match exact standalone telemetry shapes and account for original entries before the debouncer combines messages.
