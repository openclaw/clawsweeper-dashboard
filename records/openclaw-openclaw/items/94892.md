## Summary

Keep open: the latest head still filters Hermes telemetry in preflight after Discord may have coalesced same-author messages, so a telemetry line followed by a real conversational bot mention can be dropped as one batch.

## What This Changes

The PR adds a Hermes Discord operational-telemetry detector in Discord preflight plus tests for standalone telemetry, non-Hermes bot messages, and regular bot-authored collaboration text.

## Best Possible Solution

Keep the Discord-owned telemetry filter, but make it exact and batch-safe by dropping standalone telemetry per original Discord entry before constructing or dispatching a synthetic debounced batch.

## Reproduction Assessment

Yes from source: same-author Discord messages can be debounced into one synthetic message, and PR-head preflight drops the whole normalized text when it starts with a Hermes telemetry prefix. I did not run tests because this is a read-only review.

## Solution Assessment

No. Preflight is an acceptable owner for single-message telemetry, but this patch is not the best fix because it filters after coalescing and uses broad prefix matching instead of exact standalone telemetry records.
