## Summary

Keep open: the intended standalone telemetry drop has useful proof, but current head is still not merge-ready because the new Discord preflight gate can discard legitimate Hermes bot conversation in same-author debounced batches or label-leading messages before mention policy runs.

## What This Changes

Adds a Discord preflight filter and tests to drop Hermes-style standalone operational telemetry from a bot before model dispatch while keeping ordinary bot-authored collaboration.

## Best Possible Solution

Keep the Discord-owned telemetry filter, but constrain it to exact standalone telemetry records and handle debounced entries before a telemetry-only entry can suppress later conversation.

## Reproduction Assessment

Yes by source inspection: current main batches same-author Discord messages before preflight, and the PR-head null return would commit all replay keys for a batch that starts with Hermes telemetry. I did not run tests in this read-only review.

## Solution Assessment

No. Discord preflight is the right owner boundary, but this implementation should match exact standalone telemetry and account for original debounced entries before discarding an entire preflight input.
