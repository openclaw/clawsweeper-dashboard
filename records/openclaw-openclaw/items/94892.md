## What This Changes

## Best Possible Solution

Keep the Discord-owned telemetry filter, but make it an exact standalone-record detector and apply telemetry dropping at original-message granularity before or around batching.

## Solution Assessment

No. Discord preflight is the right owner boundary, but the implementation needs exact standalone telemetry grammar and original-entry handling so conversational bot messages still reach the existing allowBots policy.
