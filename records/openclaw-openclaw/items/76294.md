## Summary

Keep this PR open: it fixes the simple array case, but the current patch is type-blind and can make numeric object-key unsets silently no-op.

## What This Changes

The PR changes `runConfigUnset` to omit write-time `unsetPaths` for numeric leaf paths and adds a CLI regression test for `agents.list[1]`.

## Best Possible Solution

Keep the focused fix, but decide whether to omit `unsetPaths` from the resolved parent container type, then cover numeric object keys, nested array leaves, and the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows `unsetAtPath` splices array leaves before `replaceConfigFile`, and the write layer can apply the same numeric unset path again.

## Solution Assessment

No. The PR's current solution is too broad because it treats any numeric leaf segment as an array index instead of checking whether the parent value is an array.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
