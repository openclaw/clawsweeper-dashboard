## Summary

Keep open: the PR targets the right double-splice bug, but the current diff is not mergeable because it treats any numeric leaf as an array index, which can make numeric object-key unsets silently fail; it also still needs the required user-facing changelog entry.

## What This Changes

The branch changes `runConfigUnset` to skip `writeOptions.unsetPaths` for numeric leaf paths and adds a CLI regression test for unsetting `agents.list[1]`.

## Best Possible Solution

Land a focused fix that suppresses write-time unset paths only when the resolved parent container is an array, preserves numeric object-key unsets, covers nested array and numeric object-key cases, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows the original double-splice path, and source inspection of the PR shows a high-confidence regression path for `openclaw config unset agents.defaults.models.30` because the regex-only numeric leaf skips `unsetPaths` while preservation can restore the deleted model key.

## Solution Assessment

No. The direction is right for real array parents, but the implementation should check the resolved parent value instead of the leaf string before omitting `unsetPaths`.

## Review Findings

Overall correctness: patch is incorrect
