## Summary

Keep open. The reported bug is still source-reproducible on current main and the PR's fix shape is useful, but the branch is not merge-ready because it covers only embedded payload wrapping, imports the obsolete pi-agent-core package name, and lacks inspectable after-fix provider proof.

## What This Changes

The PR adds an embedded pi-runner onPayload wrapper and wrapper tests that rewrite OpenAI Chat Completions assistant tool-call messages from `content: null` to `content: ""`.

## Best Possible Solution

Port the null-to-empty assistant tool-call normalization to the shared OpenAI Chat Completions payload boundary so both direct and embedded paths run it before strict-key stripping, with focused regression tests and redacted DeepSeek or equivalent provider proof.

## Reproduction Assessment

Yes, source-level. Current main can still turn an assistant tool-call replay with no surviving text into `content: null`, and an existing strict-key regression test preserves that null payload shape.

## Solution Assessment

No, not as submitted. A scoped outbound payload normalization is the right fix shape, but this branch needs a current-main port, direct completions coverage, package-name updates, and real provider proof before merge.
