## Summary

Keep open. Current main still lacks the assistant tool-call `content: null` to `""` normalization, and the pinned `@mariozechner/pi-ai` converter still emits `null` for this replay shape; this PR is useful but needs a small merge-ready update before landing.

## What This Changes

The PR adds a Chat Completions payload wrapper and tests that convert assistant messages with `tool_calls` from `content: null` to `content: ""` before provider submission.

## Best Possible Solution

Current main should normalize assistant tool-call replay content at the OpenAI-compatible Chat Completions payload boundary, with focused no-op/rewrite tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence code reproduction is an `openai-completions` replay turn whose assistant message has only blank text plus a tool call; the pinned pi-ai converter filters the blank text, leaves `content: null`, and keeps the message because `tool_calls` exists.

## Solution Assessment

Yes in shape, but not merge-ready as submitted. A scoped payload wrapper is a maintainable fix, but it should be ported onto the current post-plugin wrapper chain and include the required changelog entry.
