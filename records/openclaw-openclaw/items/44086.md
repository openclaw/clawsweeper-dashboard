## What This Changes

The PR adds an embedded pi-runner payload wrapper and tests to rewrite OpenAI Chat Completions assistant tool-call messages from `content: null` to `content: ""`.

## Best Possible Solution

Port the null-to-empty assistant tool-call normalization to the shared OpenAI Chat Completions payload boundary so both direct and embedded paths run it before strict-key serialization, with focused regression coverage and redacted real-provider proof.

## Reproduction Assessment

Yes, source-level. Current `main` can still turn an assistant tool-call replay with no surviving text into `content: null`, and existing strict-key regression coverage preserves that payload shape.

## Solution Assessment

No, not as submitted. A scoped outbound payload normalization is the right fix shape, but this branch needs a current-main port, direct completions coverage, package-name updates, and real provider proof before merge.
