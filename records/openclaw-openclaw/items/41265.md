## Summary

Keep open: current main still has the reported Discord/WebP media-loss behavior, and this PR remains useful but is not merge-ready because the head is dirty against current main and the new tracked Plugin SDK type options are missing the generated API baseline update.

## What This Changes

The PR threads detected media content types into Discord upload blobs, preserves WebP/AVIF and animated image bytes in shared media loading, adds APNG/WebP detection coverage, and updates the changelog.

## Best Possible Solution

Rebase the contributor branch onto current main, move the interaction-reply pieces into the current reply helper split, update the Plugin SDK API baseline if the public options remain, and then land it as the canonical fix for #41256.

## Reproduction Assessment

Yes. Source inspection on current main shows Discord upload blobs are constructed without a MIME type and shared media loading still optimizes non-GIF images, which is enough to reproduce the reported WebP conversion and typed-upload failure path without live Discord credentials.

## Solution Assessment

Mostly yes. Preserving typed blobs and bypassing lossy optimization for Discord-preserved/animated formats is the narrow maintainable direction, but the PR still needs conflict resolution and SDK baseline alignment before merge.
