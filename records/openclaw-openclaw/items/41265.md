## Summary

Keep open: current main and the latest release still have the Discord/WebP media bug, and this PR remains the open linked implementation candidate, but the patch is not merge-ready because it has two concrete correctness blockers and only test/CI proof.

## What This Changes

The PR updates Discord send/component/native-reply media paths, shared media/MIME helpers, Plugin SDK outbound media options, tests, and changelog to preserve upload MIME types and skip destructive optimization for WebP, AVIF, APNG, and animated images.

## Best Possible Solution

Land a rebased current-main fix that threads detected content types through all Discord upload paths, preserves WebP/APNG/animated media without weakening fetch and size caps, updates the Plugin SDK baseline if public options remain, and includes focused tests plus redacted real Discord proof.

## Reproduction Assessment

Yes. Source inspection on current main shows regular/component Discord sends wrap `media.buffer` in untyped Blobs and the shared loader still optimizes non-GIF images such as WebP; I did not run a live Discord upload in this read-only review.

## Solution Assessment

No, not yet. The direction is right, but this branch still needs the Plugin SDK baseline update, a better preserved-image fetch cap, a current-main port for native replies, and real after-fix Discord behavior proof before it is the best mergeable solution.
