## What This Changes

The PR preserves detected media MIME types in Discord uploads and changes shared media loading so WebP, AVIF, APNG, and animated image uploads can avoid lossy optimization.

## Best Possible Solution

Refresh this PR or a narrow replacement against current main that forwards media.contentType through every Discord upload path, preserves animated media without lossy re-encoding, and either documents the SDK contract change or keeps the new knobs internal.

## Reproduction Assessment

Yes. Current main still passes media buffers into untyped Discord upload Blobs while the REST serializer only preserves MIME type when a file content type or typed Blob reaches it; I did not run a live Discord bot upload in this read-only review.

## Solution Assessment

Unclear. The direction is right, but the best current-main fix may be narrower by using the existing Discord file contentType path and avoiding or explicitly documenting the new public SDK media options.
