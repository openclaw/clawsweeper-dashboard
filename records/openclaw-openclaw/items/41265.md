  - body: The new fetch cap only treats URL extensions as preserved-image hints. Signed or extensionless WebP/AVIF/APNG URLs still download up to the document cap before `clampAndFinalize` rejects them against the Discord cap, adding avoidable bandwidth and memory pressure.
  - confidence: 0.84
- **[P2] Port native reply uploads to the active helper:** `extensions/discord/src/monitor/native-command.ts:1370-1378`
  - body: Current main now uses `native-command-reply.ts` for interaction replies, but this branch repairs the older in-file helper in `native-command.ts`. After rebase, native reply uploads would still need the same typed Blob and preserve flags in the active helper.
  - confidence: 0.9
- **[P2] Refresh the Plugin SDK API baseline:** `src/plugin-sdk/outbound-media.ts:13-16`
  - body: Adding `preserveWebp` and `preserveAvif` to exported `OutboundMediaLoadOptions` changes the public Plugin SDK surface, but the generated API baseline hash is not updated, so the SDK API check should reject this intentional contract change.
  - confidence: 0.9
