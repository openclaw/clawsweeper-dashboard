## Summary

Keep open: the linked Discord media bug is still source-reproducible on current main and this PR remains the canonical fix path, but the branch is conflicting and still needs native-helper porting, fetch-cap/API-baseline repairs, and real Discord proof before merge.

## What This Changes

The PR threads detected MIME types into Discord upload blobs, adds WebP/AVIF/APNG and animated-image preservation in shared media loading, updates Discord/WhatsApp/media tests, and adds a changelog entry.

## Best Possible Solution

Rebase and repair this PR or land an equivalent current-main patch that threads contentType through all active Discord upload paths, preserves WebP/animated media with bounded downloads, updates or avoids public SDK API drift, and includes redacted live Discord proof.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.12 still construct typeless Discord upload Blobs and still route WebP/non-GIF images through an optimizer that emits PNG or JPEG; live Discord proof is still missing.

## Solution Assessment

No. The direction is right, but the current branch edits a stale native helper after main moved the active path, leaves preserved-image remote fetches under-bounded for signed or extensionless URLs, changes public SDK shape without the baseline update, and lacks real behavior proof.
