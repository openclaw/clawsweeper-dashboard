## Best Possible Solution

Rebase and repair the canonical PR so active standard, component, and native Discord upload paths thread `contentType`, preserved image downloads stay bounded, SDK baselines are updated if the public flags remain, and redacted live Discord proof is posted.

## Solution Assessment

No. The direction is right, but the current branch changes a stale native helper, leaves a preserved-image fetch-cap gap, changes public SDK shape without baseline follow-through, and lacks live proof.
