## Summary

Keep open: current main still has the reported Discord/WebP media bug, and this PR is the active linked implementation path, but it is not merge-ready because the branch lacks real behavior proof and still has concrete media-cap and SDK-baseline blockers.

## What This Changes

The branch updates Discord send/component/native-reply media paths plus shared media/SDK helpers, tests, and changelog to preserve upload MIME types and avoid destructive optimization for WebP, AVIF, APNG, and animated images.

## Best Possible Solution

Land a rebased Discord/media fix that threads detected content types through all Discord upload paths, skips destructive optimization for WebP/APNG/animated media, bounds preserved-image fetches correctly, updates the SDK API baseline, and includes focused tests plus redacted real Discord proof.

## Reproduction Assessment

Yes. Source inspection of current main shows Discord media sends create untyped upload Blobs and the shared loader still optimizes non-GIF images, which gives a high-confidence path for MIME loss and WebP/APNG flattening without live bot credentials.

## Solution Assessment

No, not yet. The direction is right, but this branch still needs the SDK baseline update, a stronger preserved-image fetch cap, a current-main port, and real after-fix Discord behavior proof before it is the best mergeable solution.
