Yes. Current main has a source-level reproduction path: Discord media loading produces media.contentType but the upload Blob remains untyped, and non-GIF images still pass through PNG/JPEG optimization.

## Solution Assessment

No, not as currently proposed. The implementation direction is right, but it must be moved onto the active native-command-reply.ts helper and aligned with the current 100 MB Discord default before it is the narrowest maintainable fix.
