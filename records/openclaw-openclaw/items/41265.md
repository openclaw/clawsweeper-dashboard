The PR updates Discord media upload helpers and shared outbound media loading to preserve MIME types and bypass optimization for WebP/AVIF/APNG or animated images, with related tests and changelog text.

## Best Possible Solution

Land a rebased fix that preserves MIME type and animated/WebP media across standard sends, component sends, and native replies while bounding remote fetches, updating SDK baselines, and making raw-metadata behavior explicit.

## Reproduction Assessment

Yes. Source inspection on current main shows untyped Discord upload Blobs and WebP/APNG media still reaching the optimizer path; I did not run a live Discord upload in this read-only review.

## Solution Assessment

No. The direction is right, but this PR still needs the current-main native reply port, fetch-cap hardening, Plugin SDK baseline update, and real Discord behavior proof before it is the best merge path.
