## Best Possible Solution

Land a rebased implementation that uses the current typed heartbeat wake API, treats node-supplied location text as untrusted, adds focused coverage, and includes redacted real behavior proof.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the implementation should be rebased onto current `requestHeartbeat` and preserve `sanitizeInboundSystemTags` plus owner downgrade for node-supplied event text.
