## Summary

Keep open: current main still has the location event gap, but the patch should not merge until node-originated location events are queued as untrusted and after-fix real behavior proof is added.

## What This Changes

The branch adds iOS/gateway logging, a `location.update` node-event handler, heartbeat wake/prompt plumbing, and node location documentation.

## Best Possible Solution

Land a rebased implementation that validates and sanitizes location payloads, queues them as untrusted system events, routes them through current heartbeat prompt/wake plumbing, and documents the node event contract.

## Reproduction Assessment

Yes, source-reproducible: current main's iOS monitor sends `location.update`, while the gateway node-event switch has no matching case. I did not run a live device or simulator path in this read-only review.

## Solution Assessment

No, not as currently proposed. The direction is narrow and useful, but omitting `trusted: false` violates the established remote node-event trust boundary.
