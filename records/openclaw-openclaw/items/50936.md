## Summary

Keep open: this PR addresses a real current-main gap, but it should not merge yet because the new node-originated location event is still enqueued as trusted and the external contributor has not attached real after-fix iOS/gateway proof.

## What This Changes

The branch adds iOS/gateway logging, a `location.update` gateway node-event handler, heartbeat wake/prompt plumbing, test mock updates, and node protocol documentation.

## Best Possible Solution

Land a rebased implementation that treats `location.update` like other remote node events: validate and sanitize the payload, enqueue it as an untrusted system event, deliver it through current heartbeat prompt plumbing, and document the event contract.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced here: current main's iOS monitor sends `location.update`, while the current gateway node-event switch has no matching handler. A live reproduction would run an iOS node through the gateway and observe the event being dropped before this PR.

## Solution Assessment

No, not as currently proposed. The direction is useful, but node-supplied location events must be marked untrusted and covered before this is the maintainable fix.
