## Summary

This PR is a good narrow direction for the linked Dashboard update bug, but it should stay open: the patch does not preserve post-restart verification for the real managed-service handoff payload, and the contributor proof is unit-test-only.

## What This Changes

The branch changes the Dashboard update controller to treat successful `update.run` responses as pending even when the result status is `skipped`, adds guidance for `managed-service-handoff-started`, and adds a regression test.

## Best Possible Solution

Keep the no-skipped-banner behavior, but make the pending expected version come from the real Dashboard or gateway contract and prove the managed-service Dashboard handoff path after the patch.

## Reproduction Assessment

Yes. Current main can be source-reproduced: the gateway returns `ok: true` for started managed-service handoff while the Dashboard checks `result.status === "ok"` before treating the update as pending.

## Solution Assessment

No as-is. Treating a started handoff as non-error is the right direction, but the patch should model the real gateway payload and keep post-restart version verification active.
