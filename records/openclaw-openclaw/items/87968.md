## Summary

The patch appears functionally correct and narrowly fixes a source-reproducible Dashboard regression, but this external PR should remain open until the managed-service restart/reconnect behavior has real proof or a maintainer explicitly overrides that gate.

## What This Changes

The PR makes the Control UI treat a successfully started managed-service update handoff as pending, retains the expected target version from update availability, adds handoff guidance, and adds two controller regression tests.

## Best Possible Solution

Keep the patch's explicit handoff-success recognition and authoritative availability fallback, then prove the complete Dashboard handoff, restart, sentinel, and reconnect-verification lifecycle in a managed-service environment.

## Reproduction Assessment

Yes. Current-main gateway tests establish the successful handoff payload, and the Control UI source shows how the absent after.version loses the expected verification target.

## Solution Assessment

Yes. Handling the existing explicit handoff contract in the Control UI and using the already-advertised update target is narrower and more maintainable than weakening all skipped-response handling or changing the gateway protocol solely for this UI bug.
