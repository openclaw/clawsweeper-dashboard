## What This Changes

The PR changes the Control UI update controller to treat `ok: true` `update.run` responses as pending updates, derive the expected version from `updateAvailable` when `after.version` is absent, and add controller tests for managed-service handoff payloads.

## Best Possible Solution

Merge the narrow controller/test fix only after a maintainer accepts live managed-service Dashboard proof or records a proof override for the managed-service handoff flow.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main's Dashboard treats the documented started handoff payload as skipped because it keys on `result.status` instead of payload `ok`. I did not run a live managed-service supervisor repro.

## Solution Assessment

Yes, the PR is the narrowest maintainable fix I found: align the Dashboard with the gateway `ok` contract and use the already-loaded update-available version when the handoff payload has no `after.version`. Changing the gateway to report `status: "ok"` would conflict with the documented handoff result shape.
