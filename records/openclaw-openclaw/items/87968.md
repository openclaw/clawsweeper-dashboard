## What This Changes

The PR updates the Control UI update controller and tests so managed-service handoff responses can use `updateAvailable.latestVersion` as the pending expected version and avoid the skipped-update banner.

## Best Possible Solution

Land the narrow UI controller/test fix only after redacted real managed-service Dashboard proof, Mantis/Crabbox validation, or a maintainer proof override confirms no skipped banner and correct final update status.

## Reproduction Assessment

## Solution Assessment

Yes for the code shape, with a proof caveat. The controller is the right boundary for classifying the documented gateway response and using the existing update-availability snapshot; the remaining blocker is live behavior proof, not a different implementation path.
