## What This Changes

The PR updates the Control UI config controller and tests so `update.run` responses with `ok: true` are treated as pending updates, with managed-service handoffs falling back to the cached latest update version when `after.version` is absent.

## Best Possible Solution

Land the narrow UI fix after redacted live managed-service Dashboard proof or an explicit maintainer proof override confirms the skipped banner is gone and post-restart version verification runs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source: current main branches on `status === "ok"` while the gateway contract and tests show a started handoff returns `ok: true` with `result.status: "skipped"`; I did not run a live managed-service update.

## Solution Assessment

Is this the best way to solve the issue? Yes for the code path: using `ok: true` as the pending/success signal matches the gateway contract, and falling back to `updateAvailable.latestVersion` is the narrow available source when handoff payloads omit `after.version`.
