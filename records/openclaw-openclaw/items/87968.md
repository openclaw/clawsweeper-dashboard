## What This Changes

The PR updates the Dashboard update controller to treat `ok: true` gateway update responses as pending and to use the cached latest update version when a managed-service handoff payload lacks `after.version`.

## Best Possible Solution

Merge the focused UI fix after redacted live managed-service Dashboard proof is added, or after a maintainer explicitly records a proof override.

Yes from source: current main treats a documented `ok: true` started handoff as skipped because it gates pending-update handling on `result.status === "ok"`. I did not run a live managed-service Dashboard repro in this read-only review.

Yes, the proposed code path is the narrow maintainable fix: follow the gateway-level `ok` contract and keep post-restart verification alive with the cached update target version. The remaining gap is proof, not a different implementation direction.
