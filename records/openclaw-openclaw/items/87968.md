## What This Changes

The PR updates the Control UI update controller and tests so `update.run` `ok: true` managed-service handoff responses are treated as pending updates and use `updateAvailable.latestVersion` when `after.version` is absent.

## Best Possible Solution

Land this narrow UI controller fix after live managed-service Dashboard proof or a maintainer proof override confirms the handoff click path no longer shows the skipped banner and pending version verification stays active.

## Reproduction Assessment

Yes for source reproduction: current main gates success on `status === "ok"`, while Gateway source and docs return `ok: true` with `status: "skipped"` for a started managed-service handoff. I did not run a live managed-service Dashboard repro in this read-only review.

## Solution Assessment

Yes, this appears to be the best narrow fix: the UI follows the top-level Gateway `ok` contract and uses already-populated `updateAvailable` state for version verification. Special-casing only the handoff reason would be more brittle.
