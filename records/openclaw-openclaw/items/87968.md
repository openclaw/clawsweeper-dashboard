## What This Changes

This PR updates the Control UI update controller to treat `update.run` `ok: true` responses as pending updates, including managed-service handoffs with `status: "skipped"`, and adds regression coverage for the fallback expected-version path.

## Best Possible Solution

Land this UI controller change after live managed-service Dashboard proof or a maintainer proof override confirms the handoff click path no longer shows the skipped banner and post-restart verification stays active.

## Reproduction Assessment

Yes, this is source-reproducible: current main gates success on `status === "ok"`, while the documented Gateway handoff returns `ok: true` with `result.status: "skipped"`. I did not establish a live managed-service Dashboard repro in this read-only review.

## Solution Assessment

Yes, the patch is the best narrow fix: the UI now follows the Gateway `ok` contract and uses existing `updateAvailable` state for version verification. Special-casing only the handoff reason would duplicate the contract and leave future `ok: true` handoff-like responses at risk.
