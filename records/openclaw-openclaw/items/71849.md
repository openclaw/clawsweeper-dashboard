## Summary

Close as implemented on current main. Voice Call now has a default-off realtime fast-context path, runtime lookup before the full consult agent, partial transcript handoff, tests, docs, and an Unreleased changelog entry; the fix is on main but not in a tagged release yet.

## What This Changes

Add a low-latency Voice Call realtime memory/session context lookup before falling back to the full `openclaw_agent_consult` path.

## Best Possible Solution

Keep the current plugin-owned, default-off fast-context implementation on main and let the next release carry it to users; any deeper retrieval-quality work should stay in narrower memory-search follow-ups.

## Reproduction Assessment

Yes. The original gap was statically reproducible by checking the Voice Call realtime config/runtime/webhook paths; current main now shows those exact paths implemented and covered by focused tests.

## Solution Assessment

Yes. The current solution is the narrow maintainable fix: a Voice Call plugin setting using existing memory-host search before full consult, without moving policy into core or removing the full consult fallback.
