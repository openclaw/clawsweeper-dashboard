## Summary

Keep open. Current main still lacks the requested Voice Call realtime fast-context path: remembered-context turns still go through the full consult tool, non-final realtime transcript chunks are ignored before call-state processing, and the strict config, manifest, and docs expose no bounded fast-context setting. The maintainer discussion also explicitly confirms this is a real current-main gap rather than a duplicate of existing realtime voice work.

## What This Changes

Request an opt-in Voice Call realtime fast-context path that performs low-latency memory/session-context retrieval before falling back to full `openclaw_agent_consult` during live phone calls.

## Best Possible Solution

Implement a default-off, plugin-owned `realtime.fastContext` feature in Voice Call that accumulates and normalizes partial user transcript chunks, detects remembered-context turns, performs a hard-deadline memory/session lookup through public memory-host surfaces, injects snippets or an explicit miss into the realtime provider session, resets for follow-up questions, and keeps full `openclaw_agent_consult` available as fallback.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main has no `fastContext` config/docs/manifest support, routes realtime remembered-context work through the full consult handler, and drops non-final transcript chunks before call-state transcript processing.

## Solution Assessment

Yes. The narrow maintainable direction is an opt-in Voice Call plugin path using existing memory-host/search surfaces; moving this into core or replacing the full consult tool would be broader than the issue requires.
