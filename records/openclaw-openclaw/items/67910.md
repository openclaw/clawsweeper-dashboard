## Summary

Keep open. Current main still has the deep-only `memory status` dreaming summary, but this PR head still hides light-only and REM-only configurations behind the managed deep-promotion gate and removes existing deep-promotion troubleshooting details, so it should not merge unchanged.

## What This Changes

The PR changes memory-core CLI dreaming status to read full dreaming config and phase toggles, adds `memory status --force` for indexed status reindexing, trims blank `/dreaming status` timezone labels, and adds related tests.

## Best Possible Solution

Make `memory status` show global dreaming state, configured/effective light/deep/REM phase state and limits, plus a separate managed deep-promotion or sweep status for deep-disabled and `limit=0` cases; update focused tests, docs, and changelog together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: on current main, a light-only config reaches the deep-derived `resolveShortTermPromotionDreamingConfig` path and `formatDreamingSummary` returns `off`; the PR head preserves that result through its `shortTermDreaming.enabled` gate and added light-only test.

## Solution Assessment

No. Reading the full dreaming config is the right direction, but this patch is not the best solution unchanged because it keeps the deep gate for phase visibility, removes deep limit/threshold context, and bundles an undocumented `status --force` behavior change.
