## Summary

Keep this PR open for maintainer review. Current main still has the deep-only `memory status` renderer, but the latest PR head still does not resolve the linked light-only false-negative and has reviewable status-accuracy regressions before it can merge.

## What This Changes

This PR changes memory-core status output to render dreaming phase state from the full dreaming config, adds phase-oriented CLI tests, exposes `memory status --force` for indexed status runs, and trims blank timezone labels in `/dreaming status`.

## Best Possible Solution

Revise this PR or replace it with a narrower memory-core status change that reports configured light/deep/REM phase state while separately preserving effective deep-promotion disabled reasons such as deep off and `limit=0`, with targeted tests for light-only, REM-only, deep-disabled, global-disabled, and limit-zero configurations.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with global dreaming enabled, light enabled, and deep disabled, current main reads the deep-derived short-term promotion config and returns `Dreaming: off`; the PR head still does that through the added `shortTermDreaming.enabled` gate.

## Solution Assessment

No. The current PR direction is close, but the latest patch is not the best solution unchanged because it still hides light-only/REM-only phase state and drops effective deep disabled detail; a safer fix separates configured phase state from managed deep-promotion status.
