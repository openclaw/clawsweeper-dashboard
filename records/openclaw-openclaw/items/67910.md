## Summary

Keep open. Current main still renders `memory status` dreaming from the deep-promotion config, and the PR head still reports light-only and REM-only configs as `off` through its added deep gate, so neither the linked issue nor this PR is resolved yet.

## What This Changes

This PR changes memory-core CLI status rendering to use full dreaming config, adds phase-oriented status tests, exposes `memory status --force`, and trims blank timezone suffixes in `/dreaming status`.

## Best Possible Solution

Revise this PR or replace it with a narrower memory-core status fix that shows configured phase state while preserving effective deep-promotion status, limits, disabled reasons, tests, and the required changelog entry.

## Reproduction Assessment

Yes. With global dreaming enabled, light enabled, and deep disabled, current main reads the deep-only resolver and prints `Dreaming: off`; the PR head still returns `off` because it also gates on `shortTermDreaming.enabled`.

## Solution Assessment

No. Consulting the full dreaming config is the right direction, but the proposed patch still hides light-only/REM-only configs and drops deep limit details that operators need for troubleshooting.
