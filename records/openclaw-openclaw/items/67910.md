This PR changes memory-core status rendering to use the full dreaming config and phase toggles, adds `memory status --force`, trims blank `/dreaming status` timezone labels, and adds memory-core CLI tests.

## Best Possible Solution

Revise or replace this with a focused memory-core status fix that reports global dreaming plus light/deep/REM effective phase state while preserving managed deep-promotion limit and threshold context.

## Reproduction Assessment

Yes. Source inspection is high confidence: current main uses the short-term/deep resolver for `memory status`, and the PR head still returns `off` when the deep-derived `shortTermDreaming.enabled` is false.

## Solution Assessment
