## What This Changes

This PR changes memory-core `memory status` dreaming output to use full dreaming config phase state, adds CLI tests, adds `status --force`, and trims blank timezone labels in `/dreaming status`.

## Best Possible Solution

Keep the canonical issue/PR open and narrow this branch or replace it with a focused memory-core status fix that reports effective light/deep/REM state while preserving deep-promotion diagnostics and targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `memory status` reads the deep-derived short-term resolver while light and REM phase configs resolve and run independently, so the linked false-negative path is clear without running the CLI.

## Solution Assessment

No. Reading the full dreaming config is the right direction, but this PR still collapses light-only and REM-only configurations to `off` and drops deep-promotion limit/threshold context that status currently exposes.
