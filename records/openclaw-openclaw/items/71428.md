Request an additive plugin manifest permission field for declaring conversation-content hook access needs without requiring external plugins to patch host config schemas.

## Best Possible Solution

Define a manifest-level permission/request contract that preserves explicit operator approval semantics, carries through manifest registry metadata, updates hook/config docs, and includes tests for manifest-only, config-only, both-present, and default-deny behavior.

## Solution Assessment

Unclear. Manifest metadata is a plausible direction, but the best fix depends on maintainer security semantics for request-only disclosure versus approval and explicit host override or denial.
