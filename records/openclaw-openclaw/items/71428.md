## Summary

Keep open: current main still grants raw conversation typed hooks only from host config, the native manifest/registry contract has no `permissions.conversationAccess` path, and the linked closing PR remains open rather than merged.

## What This Changes

Add a native plugin manifest `permissions.conversationAccess` declaration so plugins can express raw conversation hook needs without patching host config schemas.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/72690 with a narrow manifest metadata contract that preserves explicit host approval for raw conversation hooks.

## Reproduction Assessment

Yes. Source inspection shows a non-bundled plugin that only declares manifest `permissions.conversationAccess: true` would still receive `hookPolicy: entry?.hooks`, and `registerTypedHook` blocks `agent_end` unless host config grants `allowConversationAccess`.

## Solution Assessment

No for the original auto-grant sketch. The safer maintainable path is request/disclosure metadata in the manifest plus explicit host-config approval or a clearly reviewed deny-precedence policy.
