## What This Changes

Add a plugin manifest field for conversation-access needs so non-bundled plugins can declare raw conversation hook access without maintaining host config schema patches.

## Best Possible Solution

Review and land or replace the linked implementation with a narrow manifest metadata contract that documents the trust model and preserves explicit host approval unless maintainers deliberately approve a stronger policy.

Unclear. A request-only manifest field appears safer than manifest self-authorization, but maintainers still need to accept that product/security scope before the linked PR can close this issue.
