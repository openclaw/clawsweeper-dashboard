## Summary

Keep open. Current main still lacks the requested exec identity environment contract: `OPENCLAW_AGENT_ID`, `CLAWDBOT_AGENT_ID`, and `OPENCLAW_SESSION_KEY` are not injected into spawned command environments, while the existing implementation only derives agent/session metadata for routing/descriptions and injects `OPENCLAW_SHELL=exec` as the runtime marker.

## What This Changes

Expose spawned subagent identity to child command environments via `OPENCLAW_AGENT_ID` and possibly compatible session/context variables.

## Best Possible Solution

Define a documented core exec identity contract, then inject it in the shared exec environment path after agent/session context is resolved so gateway, node, sandbox, PTY, background, and approval-gated runs behave consistently. Add focused tests and docs that state the variables are identity markers and clarify whether request-provided `env` can override them.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: inspect the current exec env builders and runtime spawn path, or run an exec from a spawned subagent that prints the requested variables; current main has no code path that sets them.

## Solution Assessment

Unclear. The requested direction is plausible, but the best maintainable fix needs a maintainer decision on security/advisory semantics and legacy compatibility before implementation.
