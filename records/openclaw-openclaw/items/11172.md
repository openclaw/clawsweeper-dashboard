## Summary

Keep open. Current main still does not inject the requested agent/session identity variables into exec child-process environments, and the remaining trust/override semantics need maintainer judgment before implementation.

## What This Changes

Request to expose subagent identity to spawned commands via `OPENCLAW_AGENT_ID` or `CLAWDBOT_AGENT_ID`, with optional session identity variables.

## Best Possible Solution

Define a documented core exec identity contract, then inject stable agent/session identifiers consistently across exec hosts with explicit advisory, override, and legacy-compatibility semantics.

## Reproduction Assessment

Yes. Source inspection on current main is enough: exec has agent/session metadata, but environment construction only merges inherited/request/sandbox env and the runtime only adds `OPENCLAW_SHELL`.

## Solution Assessment

Unclear. Environment injection is plausible, but the safest maintainable solution needs maintainer decisions on spoofing, override precedence, and `CLAWDBOT_AGENT_ID` compatibility first.
