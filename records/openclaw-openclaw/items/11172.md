## Summary

Keep open. Current main still does not provide the requested agent/session identity environment contract; the source resolves subagent identity for routing and policy, but exec child-process environments only receive inherited/request/sandbox env plus existing OpenClaw runtime markers.

## What This Changes

Request to set `OPENCLAW_AGENT_ID`, legacy `CLAWDBOT_AGENT_ID`, and optionally `OPENCLAW_SESSION_KEY` in subagent-spawned exec sessions so external tools can identify the calling agent.

## Best Possible Solution

Define and document a core exec identity contract, then inject stable agent/session markers in the shared env construction paths with explicit advisory, override, and legacy-compatibility semantics plus focused tests and docs.

## Reproduction Assessment

Yes. Source inspection on current main shows subagent identity is resolved before exec, but child-process env construction only passes inherited/request/sandbox env and the runtime adds only `OPENCLAW_SHELL`.

## Solution Assessment

Unclear. Environment injection is a plausible implementation, but the safest maintainable solution needs maintainer decisions on spoofing, override precedence, and `CLAWDBOT_AGENT_ID` compatibility before coding.
