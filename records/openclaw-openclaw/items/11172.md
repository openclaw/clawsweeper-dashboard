## Summary

Keep open. Current main still does not inject the requested agent/session identity env vars into exec child processes; related PR #11497 is closed unmerged, #11477 was closed in favor of this issue, and #53262 covers `sessions_spawn.agentId` schema behavior rather than exec environment identity.

## What This Changes

Request to expose spawned subagent identity to external commands via variables such as `OPENCLAW_AGENT_ID`, `CLAWDBOT_AGENT_ID`, and optionally `OPENCLAW_SESSION_KEY`.

## Best Possible Solution

Define a documented core exec identity contract, then inject stable agent/session identifiers consistently across exec hosts with explicit advisory, override, and legacy-compatibility semantics.

## Reproduction Assessment

Yes. Source inspection on current main is enough: subagent identity is resolved before exec, but child-process env construction only merges inherited/request/sandbox env and the runtime only adds `OPENCLAW_SHELL`.

## Solution Assessment

Unclear. Environment injection is plausible, but the safest maintainable solution needs maintainer decisions on spoofing, override precedence, and `CLAWDBOT_AGENT_ID` compatibility before coding.
