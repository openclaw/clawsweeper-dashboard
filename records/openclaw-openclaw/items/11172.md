## Summary

Keep open. Current main still does not expose agent/session identity through exec child-process environments; the implemented markers are generic exec/runtime markers, and the related implementation PR is closed unmerged while #53262 addresses schema behavior instead.

## What This Changes

Request to automatically expose subagent agent/session identity, such as `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY`, to commands launched from spawned sessions.

## Best Possible Solution

Define a documented core exec identity contract, then inject it consistently in the shared exec environment path across gateway, node, sandbox, PTY, background, approval-gated, and subagent-spawned runs with focused tests and docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: inspect the current env builders or run a spawned subagent exec that prints the requested variables; current main has no code path that sets them.

## Solution Assessment

Unclear. Injecting env markers is plausible, but the best maintainable fix needs a maintainer decision on advisory/security semantics, override precedence, and legacy compatibility before implementation.
