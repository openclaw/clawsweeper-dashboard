## Summary

Keep open. Current main still lacks the requested exec identity environment contract, and related implementation work is either closed unmerged or addresses a different `sessions_spawn.agentId` schema issue; the remaining work needs maintainer decisions because the use case treats env vars as authorization-adjacent identity.

## What This Changes

Request to expose a spawned subagent's `agentId` and optionally session key to exec child processes via environment variables.

## Best Possible Solution

Define a documented core exec identity contract, then inject stable agent/session markers consistently across gateway, node, sandbox, PTY, and subagent-spawned exec paths with explicit advisory, override, and legacy-compatibility semantics.

## Reproduction Assessment

Yes. Source inspection at current main shows subagent identity is resolved before exec, but child-process env construction only passes inherited/request/sandbox env and the runtime adds only `OPENCLAW_SHELL`.

## Solution Assessment

Unclear. Environment injection is plausible, but the safest maintainable solution needs maintainer decisions on spoofing, override precedence, and `CLAWDBOT_AGENT_ID` compatibility before coding.
