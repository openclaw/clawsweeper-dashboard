## Summary

Keep open. Current main and latest release still derive session titles from display name, subject, the first transcript user message, then a session-id/date fallback; the linked implementation PR is still open and unmerged, so the requested compaction-summary or later meaningful-message behavior is not implemented yet.

## What This Changes

Issue requests richer session derived titles that prefer compaction summaries or meaningful later user messages instead of the first transcript user message.

## Best Possible Solution

Resolve the open implementation path in https://github.com/openclaw/openclaw/pull/81948, or replace it with a maintainer-approved bounded title-source design that preserves displayName/subject priority and uses meaningful transcript or compaction context safely.

## Reproduction Assessment

Yes from source inspection, although this is a feature request rather than a strict bug. Current main still passes only `fields.firstUserMessage` into `deriveSessionTitle()`, so greeting-first sessions cannot derive a title from later meaningful content or compaction summaries.

## Solution Assessment

Unclear until maintainers choose the scope. A greeting filter is a plausible partial mitigation, but the requested best solution likely belongs in bounded transcript title extraction or an explicit compaction-summary title source.
