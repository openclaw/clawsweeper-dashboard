## Summary

Keep open. Current main and the latest release still derive titles from display name, subject, then the first transcript user message, while the requested compaction-summary or later meaningful-message behavior remains unmerged and is already paired with an open closing PR.

## What This Changes

The issue requests better session derived-title generation using compaction summaries or later meaningful user messages instead of low-information first user messages.

## Best Possible Solution

Derived titles should keep metadata precedence, then choose a bounded high-quality conversation source such as a compaction summary or meaningful user message, with Gateway/session-list coverage and real picker proof.

## Reproduction Assessment

Yes. Source inspection shows current main still uses the first transcript user message directly for derived titles, so a session that starts with a greeting can still get that greeting as its picker title.

## Solution Assessment

Unclear. The linked PR is a plausible partial mitigation for generic greetings, but maintainers still need to decide whether the best solution must include compaction summaries, later-message scanning, or a narrower scoped fix.
