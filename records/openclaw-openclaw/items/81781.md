## Summary

Keep open. Current main and v2026.6.6 still derive session titles from display name, subject, then the first transcript user message; the linked closing PR remains open and only covers generic-greeting filtering, not the broader compaction-summary or later-meaningful-message title source requested here.

## What This Changes

The issue requests better derived session titles using compaction summaries and meaningful recent user messages instead of only the first user message.

## Best Possible Solution

Land or replace the linked implementation with a bounded Gateway title-source path that preserves displayName/subject precedence, can use stored compaction summaries when appropriate, and supplies a meaningful transcript candidate instead of low-information first prompts.

## Reproduction Assessment

Yes, from source inspection. A session whose first user message is a greeting still flows through readSessionTitleFieldsFromTranscript into deriveSessionTitle, which returns that first message before any session-id fallback.

## Solution Assessment

Unclear. The direction is plausible, but maintainers need to decide whether the acceptable fix is the full compaction-summary and meaningful-message source path or a narrower greeting-filter mitigation in the linked PR.
