## Summary

Keep open. Current main and the latest release still derive session titles from display name, subject, the first transcript user message, then the session-id/date fallback; the linked implementation PR remains open and unmerged, so this issue is still necessary.

## What This Changes

Improve derived session titles so session pickers prefer compaction summaries or meaningful later user messages over low-information first user messages.

## Best Possible Solution

Resolve the active implementation path in https://github.com/openclaw/openclaw/pull/81948 or a replacement PR with a bounded title-source contract that can use compaction summaries and later meaningful user messages without broad session-list reads.

## Reproduction Assessment

Yes. Source inspection shows current main still passes only fields.firstUserMessage into deriveSessionTitle(), so a greeting-first transcript can still become a low-information session title.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers need to decide whether the linked greeting-filter PR is enough or whether the best fix must also cover compaction summaries and bounded later-message scanning.
