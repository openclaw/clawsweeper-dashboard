## Summary

Keep open: current main still derives titles from display name, subject, the first transcript user message, then the session-id/date fallback; the closed greeting-filter PR did not land, and the open adjacent PR does not implement compaction-summary or later-message title selection.

## What This Changes

Request to improve session `derivedTitle` quality by using compaction summaries or later meaningful user messages instead of relying on the first user transcript message.

## Best Possible Solution

Define a bounded session-title source contract that preserves displayName/subject precedence, can use stored compaction summaries when available, and falls back to a later meaningful transcript candidate without unbounded session-list reads.

## Reproduction Assessment

Yes, at source level. Current main still passes only `fields.firstUserMessage` into `deriveSessionTitle`, so a greeting-first transcript can still produce a low-information session title.

## Solution Assessment

Unclear. The direction is useful, but maintainers should choose the bounded title-source contract before implementation because compaction-summary use and later-message scanning affect a hot session-list path.
