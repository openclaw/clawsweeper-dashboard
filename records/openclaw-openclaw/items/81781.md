## Summary

Keep open: current main and the latest release still derive session titles from display name, subject, the first transcript user message, then the session-id/date fallback. The closed greeting-filter PR did not merge, and the open adjacent session-title PR only changes label/origin-label precedence, not compaction-summary or later-message title selection.

## What This Changes

Improve derived session titles so session pickers show the conversation’s meaningful topic instead of a low-information first user message.

## Best Possible Solution

Define a bounded title-source contract that preserves displayName/subject precedence, can use stored compaction summaries when appropriate, and avoids unbounded transcript reads in session listing.

## Reproduction Assessment

Yes at source level: current Gateway derives titles from the first transcript user message once displayName and subject are absent. I did not run a live TUI or Control UI reproduction because this review is read-only.

## Solution Assessment

Unclear. The requested direction is useful, but maintainers should choose the bounded title-source contract before implementation because compaction-summary use and later-message scanning affect a hot session-list path.
