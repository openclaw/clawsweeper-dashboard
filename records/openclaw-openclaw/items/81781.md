## Summary

Keep open. Current main still derives session titles from metadata, then the first transcript user message, then the session-id/date fallback, and the open linked PR that references this issue has not merged and still has review/proof blockers.

## What This Changes

Improve derived session-title generation so session pickers can use compaction summaries or later meaningful user messages instead of only the first user message.

## Best Possible Solution

Define a bounded derived-title contract that preserves displayName/subject precedence, then uses a stored compaction summary or clearly meaningful recent user prompt, with coverage for gateway listing, the sessions_list tool, and the TUI picker.
