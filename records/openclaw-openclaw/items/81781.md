## Summary

Keep open: current main and v2026.6.10 still derive titles from display name, subject, then the first transcript user message before falling back to the session id/date, and the related open PRs/issues only cover UI wiring or narrower title-precedence bugs rather than compaction-summary or later-message title quality.

## What This Changes

The issue requests better dynamic session titles that use compaction summaries or meaningful later user messages instead of generic first user messages.

## Best Possible Solution

Define a bounded derived-title contract that preserves displayName/subject precedence, can use stored compaction summaries when appropriate, skips low-information first messages, and keeps session-list transcript reads capped.

## Reproduction Assessment

Yes, at source level. Current main still passes only fields.firstUserMessage into deriveSessionTitle, and deriveSessionTitle still uses that before the session-id/date fallback when displayName and subject are absent.

## Solution Assessment

Unclear. The requested direction is useful, but maintainers should choose the bounded title-source and precedence contract before implementation because compaction-summary use, later-message scanning, and AI-generated titles have different performance and product tradeoffs.
