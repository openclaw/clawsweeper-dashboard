## Summary

Keep open. Current main still derives session titles from display name, subject, the first transcript user message, then a session-id/date fallback; the requested compaction-summary or later meaningful-message behavior is not implemented, and the open linked PR is an implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Improve computed session titles so session pickers prefer compaction summaries or meaningful recent user messages over greeting-like first-message fallbacks.

## Best Possible Solution

Define a bounded derived-title contract that prefers existing metadata, then a stored compaction summary or clearly meaningful recent user prompt, with regression coverage for gateway listing, sessions_list, and the TUI picker.

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main still uses only firstUserMessage for derived titles and does not use compaction summaries or later meaningful user messages.
