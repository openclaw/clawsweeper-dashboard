## Summary

Keep open. Current main still derives session titles from display metadata, subject, the first transcript user message, then a session-id fallback; compaction-summary and meaningful-message title sources are not implemented, and related AI-title/UI-title issues do not directly supersede this specific non-LLM title-quality request.

## What This Changes

Request to improve session `derivedTitle` generation so session pickers show compaction summaries or meaningful conversation content instead of a greeting or first-message fallback.

## Best Possible Solution

Define a bounded derived-title contract that keeps manual/display metadata first, then uses stored compaction summaries when available, then falls back to a clearly meaningful user prompt, with focused gateway, tool, TUI, and Control UI coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main confirms the reported current behavior and the absence of compaction-summary or meaningful-message title sources.

## Solution Assessment

Unclear.
