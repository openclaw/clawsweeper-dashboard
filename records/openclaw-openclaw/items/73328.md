## Summary

Keep this open: current main still falls back to the Bedrock discovery default for the reported unlisted Claude 3.x/3.5 IDs, and open PRs #73382 and #73959 already target this issue with closing syntax.

## What This Changes

Amazon Bedrock discovery should report correct per-model context windows for Claude 3.x/4.x models instead of applying the 32000 fallback to every unlisted discovered model.

## Best Possible Solution

Land one Bedrock plugin-owned discovery fix with regression coverage for the reported Claude 3.x/3.5 IDs and region-prefixed inference profiles, keeping schema/types/docs aligned if a public override knob is chosen.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a discovered ID such as `anthropic.claude-3-opus-20240229-v1:0` is absent from `KNOWN_CONTEXT_WINDOWS`, so `resolveKnownContextWindow` returns undefined and discovery uses the 32000 fallback.

## Solution Assessment

Unclear. The requested outcome is valid, but maintainers still need to choose between a narrow built-in Claude-family resolver and a user-configurable override surface.
