## Summary

Keep this open. Current main still maps reported unlisted Bedrock Claude 3.x/3.5 model IDs through the Amazon Bedrock discovery fallback, there is still no per-model override config, and two open PRs already use closing syntax for the same issue, so maintainers should choose and land one canonical fix before cleanup.

## What This Changes

The issue requests that Amazon Bedrock discovery respect per-model context windows for Claude 3.x/4.x models instead of applying the global 32000-token fallback to every unlisted discovered model.

## Best Possible Solution

Land one Bedrock plugin-owned fix with regression coverage for the reported Claude 3.x/3.5 IDs and region-prefixed inference-profile variants. If maintainers choose a public override knob, keep the plugin manifest schema, exported config type, docs, cache key, and tests aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: an unlisted model such as `anthropic.claude-3-opus-20240229-v1:0` is absent from the known-context table, so discovery falls through to `defaults.contextWindow`, which defaults to 32000.

Unclear. The requested outcome is valid, but maintainers still need to choose between the narrower family resolver in #73382 and the broader config override surface in #73959.
