## Summary

Keep this open: current main still sends unlisted Bedrock Claude 3.x/3.5 model IDs through the Amazon Bedrock discovery fallback, no per-model override surface exists, and open PRs #73382 and #73959 already target the same bug with closing syntax.

## What This Changes

Amazon Bedrock discovery should respect Claude 3.x/4.x model context windows, or expose a per-model override, instead of applying the 32K fallback to every unlisted discovered model.

## Best Possible Solution

Land one Bedrock plugin-owned discovery fix with regression coverage for the reported Claude 3.x/3.5 IDs and region-prefixed inference-profile variants; if a public override knob is chosen, keep schema, types, docs, and cache behavior aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a model such as anthropic.claude-3-opus-20240229-v1:0 is not in the known-context table, so resolveKnownContextWindow returns undefined and discovery falls back to 32000.

## Solution Assessment

Unclear. The requested outcome is valid, but maintainers should pick the narrow provider-owned model/family resolver or the broader configurable regex override surface before landing one fix.
