## Summary

Keep this open. Current main still falls back to the Amazon Bedrock discovery default for unlisted Claude 3.x/3.5 model IDs, and two open, unmerged PRs already reference this issue as their closing target, so the issue should remain open until maintainers choose and land one canonical fix.

## What This Changes

The issue requests per-model Amazon Bedrock context-window handling so Claude 3.x/4.x discovery does not advertise the 32000-token default for models that should use larger windows.

## Best Possible Solution

Land one canonical Bedrock plugin-owned fix with regression coverage for the reported Claude 3.x/3.5 IDs and region-prefixed inference-profile variants; if the chosen fix adds a public override knob, keep the manifest schema, exported config type, docs, UI hints, and discovery cache key aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: an unlisted Claude 3.5 Sonnet Bedrock ID is absent from `KNOWN_CONTEXT_WINDOWS`, so discovery falls through to `defaults.contextWindow`, which is 32000 by default.

## Solution Assessment

Unclear. The requested outcome is valid, but the open PRs propose different contracts; a family-aware resolver is the narrowest behavioral fix unless maintainers explicitly want a new user-facing regex override surface.
