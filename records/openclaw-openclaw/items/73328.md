## Summary

Keep this open: current main still gives the reported unlisted Bedrock Claude 3.x/3.5 IDs the 32000 fallback, no per-model override surface exists, and open PRs #73382 and #73959 already target this issue with closing syntax.

## What This Changes

Amazon Bedrock discovery should assign correct per-model context windows for Claude 3.x/4.x models or expose an override knob instead of applying the 32000 fallback.

## Best Possible Solution

Land one Bedrock plugin-owned discovery fix with focused regression coverage for the reported Claude 3.x/3.5 IDs and region-prefixed inference profiles, keeping schema/types/docs aligned if an override knob is chosen.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a discovered model such as `anthropic.claude-3-opus-20240229-v1:0` is absent from the known-context table, so discovery uses the 32000 fallback path.

## Solution Assessment

Unclear. The requested outcome is valid, but the maintainable implementation choice is still between a narrow built-in Claude-family resolver and a broader user-configurable override surface.
