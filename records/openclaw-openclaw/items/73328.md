## Summary

Keep this open: current `main` still falls back to the Bedrock discovery default for the reported Claude 3/3.5 IDs, and two open PRs already target this issue with closing syntax.

## What This Changes

The issue requests Bedrock discovery to use accurate per-model context windows for Claude 3.x/4.x models instead of applying the 32000 fallback to every unlisted discovered model.

## Best Possible Solution

Land one Bedrock plugin-owned discovery fix with regression coverage for the reported Claude 3/3.5 base IDs and region-prefixed inference profiles, keeping schema/types/docs aligned if maintainers choose a public override knob.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: a discovered ID such as `anthropic.claude-3-opus-20240229-v1:0` is absent from the known table, so discovery resolves no known context window and uses the 32000 fallback.

## Solution Assessment

Unclear. The requested outcome is valid, but maintainers still need to choose whether the best fix is a narrow built-in Claude-family resolver or a new user-configurable override surface.
