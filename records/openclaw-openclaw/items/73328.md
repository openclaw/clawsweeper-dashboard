## Summary

Keep this open. Current main still resolves unlisted Bedrock Claude 3.x/3.5 IDs through the 32000 default context-window path, and two open PRs already reference this issue as their closing target, so the issue should remain open until maintainers choose and land one canonical fix.

## What This Changes

The issue asks Amazon Bedrock discovery to assign correct per-model context windows, especially 200K for Claude 3.x/4.x models, instead of applying the global 32000 fallback to every unlisted model.

## Best Possible Solution

Land one canonical Bedrock plugin-owned fix with regression coverage for the reported Claude 3.x/3.5 IDs and their region-prefixed inference-profile variants. If maintainers choose a public override knob, keep the manifest schema, exported config type, docs, UI hints, discovery cache key, and tests aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a discovered `anthropic.claude-3-5-sonnet-20241022-v2:0` or `anthropic.claude-3-opus-20240229-v1:0` model is absent from `KNOWN_CONTEXT_WINDOWS`, so current discovery falls through to the 32000 default.

## Solution Assessment

Yes for keeping this issue open until a linked fix lands. The narrowest maintainable code direction appears to be a plugin-owned known-model/family resolver, while the regex override approach needs explicit maintainer product approval because it adds a new config contract.
