## Summary

Keep open. Current main still sends unlisted Claude 3.x/3.5 Bedrock model IDs through the 32K discovery fallback, and open closing PRs #73382 and #73959 already cover the competing implementation paths.

## What This Changes

Amazon Bedrock discovery should assign correct per-model context windows for Claude 3.x/4.x models instead of applying the 32K default to every unlisted discovered model.

## Best Possible Solution

Land one canonical Bedrock plugin-owned fix with regression coverage for the reported Claude 3.x/3.5 model IDs and region-prefixed inference-profile variants. If maintainers choose a public override knob, keep the manifest schema, exported config type, docs, UI hints, validation, and discovery cache key aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the reported Claude 3 Opus/Sonnet and Claude 3.5 Sonnet IDs are absent from the known table and discovery falls through to defaultContextWindow, currently 32000 unless globally overridden.

## Solution Assessment

Unclear. The requested outcome is valid, but the two open PRs implement different contracts; the known-model/family resolver is the smaller behavioral fix unless maintainers explicitly want a new user-facing override surface.
