## Summary

Keep this open: current main still routes reported unlisted Bedrock Claude 3.x/3.5 model IDs through the Amazon Bedrock discovery fallback, no per-model override surface exists, and open PRs #73382 and #73959 already target the same bug with closing syntax.

## What This Changes

The issue requests that Amazon Bedrock discovery respect per-model context windows for Claude 3.x/4.x models instead of applying the global 32000-token fallback to every unlisted discovered model.

## Best Possible Solution

Land one Bedrock plugin-owned discovery fix with regression coverage for the reported Claude 3.x/3.5 IDs and region-prefixed inference-profile variants, keeping schema/types/docs/cache behavior aligned if a public override knob is chosen.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: an unlisted model such as `anthropic.claude-3-opus-20240229-v1:0` is absent from `KNOWN_CONTEXT_WINDOWS`, so discovery falls through to `defaults.contextWindow`, which defaults to 32000.

## Solution Assessment

Unclear. The requested outcome is valid, but maintainers still need to pick the narrower provider-owned model/family resolver or the broader configurable regex override surface.
