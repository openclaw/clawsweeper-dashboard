## Summary

Keep open: current main still falls back to the Bedrock discovery default for the reported unlisted Claude 3/3.5 IDs, and open PRs #73382 and #73959 already target this issue with closing syntax.

## What This Changes

The issue asks Amazon Bedrock discovery to honor 200K context windows for Claude 3.x/4.x models instead of using the 32000 discovery default.

## Best Possible Solution

Land one Bedrock plugin-owned discovery fix with regression coverage for the reported Claude 3/3.5 base IDs and region-prefixed inference profiles, keeping schema/types/docs aligned only if a public override knob is chosen.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a discovered ID such as anthropic.claude-3-opus-20240229-v1:0 is absent from KNOWN_CONTEXT_WINDOWS, so resolveKnownContextWindow returns undefined and discovery uses the 32000 fallback.

## Solution Assessment

Unclear. The Bedrock plugin is the right boundary, but maintainers need to choose whether the narrowest maintainable fix is a built-in Claude-family resolver or the broader modelContextWindowOverrides config surface proposed by #73959.
