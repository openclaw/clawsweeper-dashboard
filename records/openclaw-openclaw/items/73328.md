## Summary

Keep open: current main and the latest release still miss the reported Bedrock Claude 3/3.5 context-window rows and still fall back to the 32000 discovery default; the active linked PR is the right implementation path but has not merged.

## What This Changes

The issue asks for Amazon Bedrock discovery to respect 200K context windows for Claude 3.x/4.x models instead of assigning the 32K fallback to unlisted Claude IDs.

## Best Possible Solution

Land one Bedrock plugin discovery fix with regression coverage for the reported Claude 3.x/3.5 IDs and region-prefixed inference profiles, then close this issue after the fix merges.

Yes, at source level. Current main lacks the reported known-window entries and any override/family fallback, and the discovery path falls back to defaultContextWindow for missing foundation models and inference profiles.

## Solution Assessment
