## Summary

Keep open: current main still has the reported Slack lifecycle gap. Slack dispatch only signals dispatch idle, while the shared typing controller requires both run-complete and dispatch-idle state, and Discord already signals them in that order.

## What This Changes

The issue asks Slack to mark inbound reply runs complete before marking dispatch idle so typing/status cleanup happens promptly after completion.

## Best Possible Solution

Align Slack dispatch with the shared and Discord lifecycle by calling markRunComplete before markDispatchIdle, then add focused Slack regression coverage for the ordering.

## Reproduction Assessment

Yes at the source level: current Slack dispatch omits markRunComplete and only calls markDispatchIdle, while the shared typing controller requires both flags before cleanup. I did not establish a live Slack run in this read-only review.

## Solution Assessment

Yes, the suggested direction matches the existing shared contract and Discord implementation. The maintainable fix is narrow: carry markRunComplete through the Slack handler and assert the run-complete-before-idle ordering in focused tests.
