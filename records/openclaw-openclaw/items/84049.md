## Summary

Keep open: current main and the latest shipped tag still have the Slack lifecycle gap, and the linked closing PR is open but not merged.

## What This Changes

Slack reply dispatch should mark a run complete before marking dispatch idle so Slack typing/status cleanup follows the normal completion hold instead of the fallback TTL.

## Best Possible Solution

Land the linked PR or an equivalent narrow Slack fix that calls `markRunComplete()` before `markDispatchIdle()` in both settle paths, with focused regression coverage and real Slack proof.

## Reproduction Assessment

Yes at the source level: current main Slack dispatch only marks dispatch idle, while typing cleanup requires both run-complete and dispatch-idle. I did not establish a live Slack run in this read-only review.

## Solution Assessment

Yes, the requested direction matches the existing shared lifecycle contract and Discord behavior. The issue should stay open until https://github.com/openclaw/openclaw/pull/84061 or an equivalent fix merges.
