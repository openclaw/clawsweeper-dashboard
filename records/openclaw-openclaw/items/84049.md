## Summary

Keep open: current main still has the reported Slack lifecycle gap, and the linked fixing PR has not merged. Slack only marks dispatch idle, while the shared typing controller requires both run-complete and dispatch-idle before cleanup.

## What This Changes

The issue asks Slack's reply pipeline to call `markRunComplete()` before `markDispatchIdle()` so typing/status cleanup follows `doneHoldMs` instead of the fallback TTL.

## Best Possible Solution

Review and land a narrow Slack fix that calls `markRunComplete()` before `markDispatchIdle()` in both Slack cleanup paths, with focused regression coverage and redacted real Slack proof.

## Reproduction Assessment

Yes at source level: current Slack dispatch omits `markRunComplete()`, while the shared typing controller stops only after both `runComplete` and `dispatchIdle` are true. I did not run a live Slack workspace reproduction.

## Solution Assessment

Yes: aligning Slack with Discord and the shared lifecycle marker order is the narrow maintainable fix. The linked PR is the right implementation path once its proof and PR-level blockers are resolved.
