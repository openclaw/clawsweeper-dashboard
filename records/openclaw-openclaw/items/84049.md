## What This Changes

Slack reply dispatch should mark the run complete before marking dispatch idle so typing/status cleanup honors doneHoldMs after replies.

## Best Possible Solution

Land an equivalent of https://github.com/openclaw/openclaw/pull/84061 after its proof and check blockers are resolved, then close this issue with the merged commit and release provenance.

## Reproduction Assessment

Yes at the source level: current Slack dispatch omits markRunComplete where shared typing cleanup requires both run-complete and dispatch-idle state. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to align Slack with the shared and Discord lifecycle order by calling markRunComplete before markDispatchIdle. Because an open linked PR already attempts that fix, the issue should remain open until that PR or an equivalent lands.
