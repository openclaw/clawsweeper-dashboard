## What This Changes

Slack reply dispatch should call `markRunComplete()` before `markDispatchIdle()` so Slack typing/status cleanup follows the configured completion hold instead of the fallback TTL.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84061 after real Slack proof and required checks, or replace it with the same narrow lifecycle fix before closing this issue.

## Reproduction Assessment

Yes, at source level: current Slack dispatch only marks dispatch idle while the shared typing controller requires both run-complete and dispatch-idle before normal cleanup. I did not establish a live Slack run in this read-only review.

## Solution Assessment

Yes; matching Discord by calling `markRunComplete()` before `markDispatchIdle()` is the narrow maintainable fix. The issue should remain open while the linked closing PR is reviewed and proven.
