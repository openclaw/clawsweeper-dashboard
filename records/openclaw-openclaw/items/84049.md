## Summary

Keep open: current main and the latest shipped release still have the Slack lifecycle gap, and https://github.com/openclaw/openclaw/pull/84061 is an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

Slack reply dispatch should mark the run complete before marking dispatch idle so typing/status cleanup clears after `doneHoldMs`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84061 or an equivalent narrow fix that aligns Slack with the shared lifecycle contract and adds focused regression coverage.

## Reproduction Assessment

Yes at source level: current main Slack dispatch only marks dispatch idle, while the shared typing controller requires both run-complete and dispatch-idle before cleanup. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to call `markRunComplete()` before `markDispatchIdle()` in Slack, matching Discord and the shared lifecycle contract. The existing open PR is the right implementation path, pending real Slack proof and normal review.
