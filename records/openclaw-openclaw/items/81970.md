## What This Changes

The branch adds final write-boundary ACP metadata reconciliation in the session store and stale-writer regression tests.

## Best Possible Solution

Land a narrow ACP-relevant write-boundary reconciliation that preserves fresh disk ACP metadata, keeps explicit helper removals intact, respects the non-ACP writer-cache contract, and includes redacted real ACP/Gateway proof.

## Reproduction Assessment

Yes, from source: current main snapshots ACP metadata before the mutator and can save a stale session entry without re-reading disk. I did not execute tests because this review had to keep the checkout read-only.

## Solution Assessment

Not yet: final write-boundary reconciliation is the right boundary, but the current patch unconditionally breaks the writer-cache no-read contract and has lint failures. A narrower ACP-relevant read with the cache-contract test kept meaningful is safer.
