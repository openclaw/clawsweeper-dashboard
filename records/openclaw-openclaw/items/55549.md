## Summary

Keep this PR open: the exec stale-render fix is pointed at the right problem, but the current patch loses the approval kind when resolving a captured ID after queue pruning, so plugin approvals can still be routed to the exec resolver and remain stuck.

## What This Changes

The PR updates the Control UI approval buttons to pass the rendered approval ID into the approval decision handler and widens the view-state signature to accept it.

## Best Possible Solution

Carry render-time approval identity as ID plus kind or resolver method, cover stale-render exec and plugin approval clicks, then merge after redacted real Control UI proof shows the decision is submitted and the blocked work proceeds.

## Reproduction Assessment

Yes by source inspection: render an approval, let the expiry timeout prune `execApprovalQueue`, then click the still-rendered button. Current main reads live queue state and returns before sending an approval RPC.

## Solution Assessment

No, not as currently patched. Passing render-time identity is the right direction, but the handler must preserve approval kind or resolver method instead of re-deriving it from queue state that may already be gone.
