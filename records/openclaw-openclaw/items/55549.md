## Summary

Keep this PR open: current main still has the Control UI approval stale-render race, but the proposed patch should not merge as-is because it captures only the approval ID and can route stale plugin approvals to the exec resolver.

## What This Changes

The PR changes the Control UI approval buttons and shared decision handler so rendered approval IDs are passed into `handleExecApprovalDecision`.

## Best Possible Solution

Keep the render-time capture approach, but capture stable resolver identity such as kind or method with the ID, preserve the legacy decision-only fallback, add stale-render exec and plugin tests, and add the required changelog line.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render an approval, let the expiry timeout remove it from `execApprovalQueue`, then click a still-rendered button; current main reads the now-empty queue and returns without sending the approval RPC.

## Solution Assessment

No, not as-is. Passing render-time identity is the right narrow fix shape, but the PR needs to carry the approval kind or resolver method too so stale plugin approvals do not fall back to `exec.approval.resolve`.
