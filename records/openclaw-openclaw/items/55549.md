## What This Changes

The PR passes render-time approval IDs from the Control UI approval buttons into `handleExecApprovalDecision` and widens the view-state handler signature.

## Best Possible Solution

Carry render-time approval identity as ID plus kind or resolver method, cover stale-render exec and plugin approval clicks, then merge after redacted Control UI proof shows the decision is submitted and blocked work proceeds.

## Reproduction Assessment

Yes, by source inspection: render an approval, let the expiry timeout prune `execApprovalQueue`, then click the still-rendered button. Current main returns before sending an approval RPC; PR head fixes ID capture for exec approvals but loses plugin kind in that stale window.

## Solution Assessment

No, not as currently patched. Passing render-time identity is the right direction, but the handler must preserve approval kind or resolver method instead of re-deriving it from queue state that may already be gone.
