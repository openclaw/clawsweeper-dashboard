## Summary

Keep open: current main still has the reported stale-render approval bug, but this PR is not merge-ready because the proposed captured-id path loses plugin approval routing when the queue entry has already been pruned.

## What This Changes

The PR changes Control UI approval buttons to pass a render-time approval id into the shared approval decision handler.

## Best Possible Solution

Keep the render-time capture approach, but capture stable resolver identity such as kind or method with the id, preserve the legacy decision-only fallback, add stale-render tests for exec and plugin approvals, and add the required Unreleased/Fixes changelog line.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render an approval, let the timeout remove it from `execApprovalQueue`, then click a still-rendered button; current main reads the now-empty queue and returns without sending the approval RPC.

## Solution Assessment

No, not as-is. Passing only the id fixes the exec path but loses the approval kind for stale plugin approvals, so the maintainable fix is to capture and route by stable id plus kind or resolver method.
