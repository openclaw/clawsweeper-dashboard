## Summary

Keep this PR open: it targets a real Control UI stale-render bug, but the patch is not mergeable as-is because captured plugin approvals can be routed to the exec resolver, and the external PR does not include visible after-fix runtime proof.

## What This Changes

The PR changes the Control UI approval buttons to pass a render-time approval ID into `handleExecApprovalDecision` and updates the handler/type signature accordingly.

## Best Possible Solution

Capture stable approval identity, including resolver kind or method plus ID, at render time for approval actions and add focused stale-render coverage for exec and plugin approvals while preserving the decision-only fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render an approval, let the expiry timeout remove it from `execApprovalQueue`, then click the still-rendered button; current main reads the now-empty queue and returns before sending the RPC.

## Solution Assessment

No, not as-is. Capturing render-time identity is the right direction, but passing only the ID loses plugin resolver identity after pruning; the safer fix captures ID plus kind or method.
