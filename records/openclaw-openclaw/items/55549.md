## Summary

Keep this PR open: the exec approval direction is useful, but the current patch is not merge-ready because captured plugin approval IDs can still be routed to the exec resolver after queue expiry, and the PR still lacks inspectable real Control UI proof.

## What This Changes

The branch updates the Control UI approval view and handler signature so rendered approval buttons pass an approval ID into `handleExecApprovalDecision`.

## Best Possible Solution

Carry render-time approval identity as ID plus kind or resolver method, cover stale-render exec and plugin approval clicks, then merge after redacted real Control UI proof shows the decision is submitted and blocked work proceeds.

## Reproduction Assessment

Yes, by source inspection: render an approval, let the expiry timeout prune `execApprovalQueue`, then click the still-rendered button. Current main returns before sending an approval RPC; PR head fixes ID capture for exec approvals but keeps the stale-window failure for plugin approvals by losing kind.

## Solution Assessment

No, not as currently patched. Passing render-time identity is the right direction, but the handler must preserve approval kind or resolver method instead of re-deriving it from queue state that may already be gone.
