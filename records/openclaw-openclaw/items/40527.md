## What This Changes

The issue reports macOS app-backed node remote skill bin probes timing out during co-located gateway startup and early node reconnect.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/83429 with a readiness-aware gateway/macOS-node fix that prevents premature automatic probes while preserving valid remote macOS shell execution.

Yes. Current main can start automatic remote-bin probing before the Swift app starts its normal receive loop, and the April 2 live comment confirms automatic probe timeouts while manual `system.which` succeeds.

## Solution Assessment

Unclear for the open PR path. The maintainable fix should address node readiness or a narrow co-located non-exec case rather than globally removing macOS app-backed `system.run`/`system.which` support.
