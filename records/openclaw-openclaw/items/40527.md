## Summary

Keep open: current main and v2026.6.10 still start automatic remote bin probing for macOS nodes that advertise `system.which` or `system.run`, and the remaining fix needs a maintainer decision on readiness, co-location, or backoff behavior with live macOS proof.

## What This Changes

The issue asks OpenClaw to avoid remote skill bin probe timeouts for co-located and macOS app-backed nodes while preserving legitimate remote Mac node execution.

## Best Possible Solution

Preserve the declared macOS app node execution contract and land a narrow readiness, deferral, backoff, or co-location-aware probe fix with redacted live macOS app-node proof.

## Reproduction Assessment

Partially. Source inspection confirms the automatic probe still runs on connect for macOS nodes that advertise `system.which` or `system.run`, but no live current-main macOS reproduction was run here.

## Solution Assessment

Unclear as an implementation. The safer direction is not a broad config flag or disabling app-node exec commands, but a readiness/order/backoff or co-location-specific probe fix that preserves declared macOS app node execution.
