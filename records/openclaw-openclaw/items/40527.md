## Summary

Keep open: current main still starts the automatic remote-bin refresh during node connect before the Swift client begins its normal event receive loop, and this issue is paired with an open closing PR that should be reviewed or replaced before the issue closes.

## What This Changes

The issue reports remote skill binary probe timeouts for macOS app-backed nodes during gateway startup or reconnect, especially when the app node is co-located with the gateway.

## Best Possible Solution

Fix the node readiness contract so automatic remote-bin refresh runs only after app-backed nodes can receive normal events, or skip only a proven co-located non-exec case while preserving valid macOS system.run and system.which support.

## Reproduction Assessment

Yes. Current main can start automatic remote-bin probing before the Swift app starts its normal listener, and the April 2 live comment confirms automatic probe timeouts while manual system.which succeeds.

## Solution Assessment

Unclear for the exact open PR path. The maintainable fix should address node readiness or a narrow co-located non-exec case rather than globally removing macOS app-backed system.run/system.which support.
