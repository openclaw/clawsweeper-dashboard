## Summary

Keep open: current main still starts the remote macOS bin probe during node connect before the Swift client has entered its normal receive loop, and the linked churn-reduction PR is open and does not prove or close the first-connect readiness bug.

## What This Changes

The issue requests a fix for connect-time remote macOS skill bin probe timeouts when an app-backed node is co-located with, or newly connected to, the gateway.

## Best Possible Solution

Fix the node readiness contract so automatic remote-bin refresh runs only after an app-backed node can receive normal events, or make the Swift channel safely buffer and dispatch handshake-time events.

## Reproduction Assessment

Yes, as a source-level reproduction: current main can emit node.invoke.request for the remote bin probe before the Swift app starts its normal receive loop. A user also confirmed the automatic probe times out while manual system.which succeeds.

## Solution Assessment

Yes: a readiness/order fix is the narrow maintainable solution. Loopback-only skips, deny-command workarounds, or a global disable flag would mask valid remote macOS nodes rather than fixing the race.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/40527. Current main registers a node and starts refreshRemoteNodeBins during the connect handler before the Swift macOS app starts its normal receive loop, so the automatic system.which/system.run probe can time out even though manual system.which later works. Implement the narrow readiness/order fix: either defer automatic remote-bin refresh until the node can receive normal events, or make the Swift GatewayChannel/GatewayNodeSession safely handle/buffer node.invoke.request events received during the connect handshake. Do not add a global disable flag or broad loopback skip as the primary fix, and do not duplicate the TTL/backoff work from https://github.com/openclaw/openclaw/pull/79438. Add focused regression coverage for the gateway connect/probe ordering and the Swift handshake/event handling path. Changelog is expected if the fix changes user-visible gateway/macOS node behavior.
