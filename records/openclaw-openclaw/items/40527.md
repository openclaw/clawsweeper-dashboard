## Summary

Keep open. Current main still starts the automatic macOS remote-bin refresh during node connect before the gateway sends `hello-ok`, while the Swift client starts its normal receive loop only after the connect response; newer user evidence also shows manual `system.which` can work while the automatic connect-time probe times out.

## What This Changes

The issue reports remote macOS app node skill-bin probes timing out during gateway/node connect, especially when the app node is co-located with the gateway host.

## Best Possible Solution

Gate the automatic remote-bin refresh behind a confirmed post-`hello-ok` node readiness point, or buffer pre-listen node events in the Swift channel, while preserving node command policy and leaving TTL/backoff churn work to the related PR.

## Reproduction Assessment

Yes, by source inspection and user evidence. Current main can emit the automatic `node.invoke.request` before the Swift app starts its normal receive loop, and a newer report confirms manual `system.which` can work while the automatic probe times out.

## Solution Assessment

Yes. A readiness/order fix is the narrow maintainable direction; loopback-only skips, broad deny-command workarounds, or a global disable flag would mask legitimate remote macOS nodes instead of fixing the race.

## Repair Work Prompt

Fix the macOS app-backed remote-bin probe readiness race for this issue. On current main, `src/gateway/server/ws-connection/message-handler.ts` registers a node and schedules `refreshRemoteNodeBins` before sending `hello-ok`, while `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift` starts its normal listener only after the connect response. Move the automatic refresh behind a post-hello readiness point or buffer/dispatch pre-listen `node.invoke.request` events in the Swift channel. Keep node command declaration/approval semantics intact, do not add a global config disable flag, do not skip all loopback nodes in a way that hides real remote-capable nodes, and do not duplicate the TTL/backoff/cache work from https://github.com/openclaw/openclaw/pull/79438. Add focused regression coverage for the ordering and update changelog only if the final change is user-visible.
