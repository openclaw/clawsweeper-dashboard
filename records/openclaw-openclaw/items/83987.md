## Summary

Keep open for maintainer review: the patch is focused and has sufficient targeted proof, and I did not find a discrete blocking correctness bug in the diff. ## Best Possible Solution

## Solution Assessment

Yes, with maintainer acceptance of the relay-lifetime tradeoff. Deferring cleanup only for successful turns while keeping aborted, timed-out, error, and interrupted paths immediate is a narrow fix, and generation-aware handles address the stable-ID replacement race.
