## Summary

Keep open for normal maintainer review. The patch appears to address a real Codex native-hook cleanup race with focused source changes, regression coverage, and sufficient exact-head terminal proof; the remaining decision is explicit acceptance of the bounded post-success relay lifetime.

## What This Changes

The PR defers Codex native-hook relay unregister after successful turns, makes relay cleanup and renewal generation-aware, and adds regression coverage for late callbacks and fail-closed cleanup paths.

## Best Possible Solution

Land after maintainers explicitly accept the bounded successful-turn relay grace and keep the fail-closed immediate cleanup behavior for aborted, timed-out, interrupted, and error paths.

## Reproduction Assessment

Yes at source level: current main synchronously unregisters the relay during final cleanup, and the PR body supplies before/after relay logs plus exact-head targeted late-callback output. I did not run a live gateway Codex agent in this read-only review.

## Solution Assessment

Yes, with maintainer risk acceptance: deferring only successful cleanup and making handles generation-aware is a focused fix for late callbacks without keeping aborted/error turns open. The safer alternative would be an upstream hook-subprocess drain signal, but that does not appear available in the current contract.
