## Summary

Keep open: the PR targets a real Codex native-hook cleanup race with useful runtime logs, but the timer-only fix is not merge-ready because an old delayed unregister can delete a newer active relay for the same deterministic session id.

## What This Changes

The PR changes Codex app-server end-of-run cleanup to defer `nativeHookRelay.unregister()` for 10 seconds with an unref'd timer.

## Best Possible Solution

Keep the delayed-cleanup direction only after unregister is generation-aware or superseded timers are cancelled, with regression coverage for late invokes and immediate second runs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main unregisters synchronously, the PR body includes a concrete live command plus before/after lifecycle logs, and existing tests prove same-session relay id reuse.

## Solution Assessment

Is this the best way to solve the issue? No, not as written; delaying cleanup is plausible, but the unregister must be tied to the registration generation or cancelled when a newer run registers the same relay id.
