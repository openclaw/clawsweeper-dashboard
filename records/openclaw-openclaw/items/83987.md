## Summary

Keep open: the reported Codex native-hook cleanup race is plausible and backed by useful live trace evidence, but the current timer-only patch can unregister a newer relay for the next run because Codex relay IDs are deterministic per session.

## What This Changes

The PR changes Codex app-server end-of-run cleanup to schedule `nativeHookRelay.unregister()` on an unref'd 10-second timer instead of unregistering synchronously.

## Best Possible Solution

Keep the delayed-cleanup idea only if unregister is generation-aware or superseded timers are cancelled, with regression coverage for both late invokes and an immediate second run.

## Reproduction Assessment

Yes, there is a concrete live command and instrumented before/after log path in the PR body, and current main source matches the synchronous unregister path. I did not rerun the live Codex harness during this read-only review.

## Solution Assessment

No. Deferring cleanup may be the right direction, but the current handle timer is not safe with reused relay IDs; cleanup needs to be tied to the registration generation or cancelled when a new run registers.
