## Summary

Keep open for maintainer review: current main still unregisters the Codex native-hook relay synchronously at normal turn cleanup, and the PR is a focused fix with no blocking review findings. The remaining decision is whether maintainers accept the bounded 10-second post-turn relay lifetime for a security-sensitive hook path.

## What This Changes

The PR defers normal Codex app-server native-hook relay unregister by 10 seconds, makes relay handle renew/unregister generation-aware, tightens expired cleanup, and adds regression tests.

## Best Possible Solution

Land this focused fix once maintainers accept the bounded post-turn relay grace window; exact-head live gateway proof would reduce risk but no code repair is currently identified.

## Reproduction Assessment

Yes for source-level reproduction: current main unregisters the relay synchronously during normal cleanup, and the PR body supplies real before/after lifecycle logs for a late `nativeHook.invoke` callback. I did not rerun the live gateway path in this read-only review.

## Solution Assessment

Yes, the proposed shape is the narrow maintainable fix: defer only normal completion cleanup while keeping abort, timeout, startup, and turn-start failure cleanup immediate. The generation-aware handle changes are necessary to avoid old deferred cleanup deleting or renewing a replacement relay with the same deterministic id.
