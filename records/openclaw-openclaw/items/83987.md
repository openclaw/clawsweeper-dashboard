## Summary

Keep this PR open: the successful-turn grace path is useful, but the current head still lets a Codex `turn/completed` terminal status of `interrupted` take the delayed unregister path because that status is intentionally not projected as aborted or errored.

## What This Changes

The PR defers Codex app-server native hook relay unregister after successful turns, makes relay cleanup and renewal handles generation-aware, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Keep the bounded successful-turn grace, but make the cleanup decision explicitly exclude Codex `interrupted` terminal status and cover that path with a focused relay cleanup regression test before automerge continues.

## Reproduction Assessment

Yes, at source level. Existing projector coverage says `turn.status === "interrupted"` alone leaves `aborted=false` and `promptError=null`, while the PR's new cleanup gate delays unregister whenever those flags stay clear.

## Solution Assessment

No, not as written. The bounded grace is the right narrow shape for successful turns, but it must exclude the plain Codex interrupted terminal status before the implementation matches its fail-closed cleanup contract.
