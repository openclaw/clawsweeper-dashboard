## Summary

Keep open for maintainer review: this PR addresses a real Codex native-hook cleanup race with a focused patch and sufficient log proof; I found no blocking correctness defect, but merging intentionally extends the relay lifetime and needs maintainer acceptance.

## What This Changes

The PR defers normal Codex app-server native-hook relay unregister, makes relay renew/unregister generation-aware, hardens expired cleanup, and adds regression tests.

## Best Possible Solution

Land this focused fix once maintainers accept the bounded post-turn relay grace window; exact-head live gateway proof would reduce risk but no code repair is currently identified.

## Reproduction Assessment

Yes, at source level: current main unregisters the relay synchronously during normal cleanup, and the PR body supplies before/after lifecycle logs for a late `nativeHook.invoke` callback. I did not rerun the live gateway path in this read-only review.

## Solution Assessment

Yes, this is the narrow maintainable fix shape: defer only normal completion cleanup while keeping abort, timeout, startup, and turn-start failure cleanup immediate. The generation-aware handle changes are necessary to prevent old deferred cleanup from deleting or renewing a replacement relay with the same deterministic id.
