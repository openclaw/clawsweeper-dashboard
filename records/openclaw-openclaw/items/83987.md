## Summary

Keep open for maintainer merge decision: the patch appears focused and correct with sufficient exact-head terminal proof, but it intentionally extends a security-sensitive Codex native hook relay lifetime after successful turns.

## What This Changes

The PR defers Codex native-hook relay unregister after successful app-server runs, makes relay cleanup and renewal generation-aware, and adds regression coverage for late callbacks and fail-closed cleanup paths.

## Best Possible Solution

Land after maintainers explicitly accept the bounded successful-turn relay grace while preserving immediate cleanup for aborted, timed-out, interrupted, and error-terminal runs.

## Reproduction Assessment

Yes at source level: current main synchronously unregisters the relay during final cleanup while Codex command hooks have their own subprocess timeout. The PR body also supplies exact-head terminal output for a late `PreToolUse` callback reaching `before_tool_call`; I did not rerun the full live gateway agent in this read-only review.

## Solution Assessment

Yes, with maintainer risk acceptance: delaying only successful cleanup and making relay handles generation-aware is a focused fix for late callbacks without retaining aborted or error turns. The safer long-term alternative would be an upstream hook-subprocess drain signal, but that does not appear available in the current contract.
