## Summary

Keep open: current main and latest release still have both fail-open existing-page reads, and https://github.com/openclaw/openclaw/pull/98360 is the unmerged closing fix candidate. This is a source-backed memory-wiki data-loss bug, not stale cleanup.

## What This Changes

Prevent memory-wiki source re-ingests from treating transient existing-page read failures as an empty page and overwriting the user's Notes block.

## Best Possible Solution

Land or replace the open memory-wiki fix with a narrow fail-closed or bounded-retry read path for both source-page writers, with regression coverage for transient and persistent read failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: make the existing source page unreadable at the read site, and the catch returns an empty string so the writer stores regenerated content without the prior Notes block.

## Solution Assessment

Yes. The best fix is local to the two memory-wiki existing-page read sites; changing unsafe-local pruning or adding configuration would be the wrong boundary for this report.
