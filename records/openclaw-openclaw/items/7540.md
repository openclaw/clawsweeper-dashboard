## Summary

Keep open. Current main still lacks WhatsApp Web/Baileys call-event handling, the pinned Baileys contract exposes a typed call event, and the live discussion scopes this as valid WhatsApp-plugin-owned work that still needs privacy/API decisions.

## What This Changes

Add WhatsApp Web call-event detection via Baileys so WhatsApp voice/video calls can count as contact activity alongside messages.

## Best Possible Solution

Implement default-off, sanitized call-activity handling inside the WhatsApp plugin, reusing existing account, DM/group, self-chat, allowlist, and plugin-hook privacy boundaries.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers credential, connection, and message listeners for WhatsApp Web while the pinned Baileys rc10 contract exposes a typed call event that OpenClaw does not consume.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, opt-in call tracking through a sanitized documented event/tool/session surface rather than folding WhatsApp Web call activity into the PSTN voice-call plugin.
