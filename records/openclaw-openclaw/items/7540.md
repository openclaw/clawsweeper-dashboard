## Risks / Open Questions

- A raw call stream exposes sensitive communication metadata: contact identifiers, timestamps, group context, call type, status, and offline/latency data.
- The issue proposes multiple product surfaces, including session metadata, daily history, a message-tool query, or plugin hooks; current code does not define which surface should own non-message activity.
- Persisting `callsToday` or `lastCallWith` would introduce retention and stored-state semantics, which should not be implemented as ad hoc sidecar state.
- Baileys emits call status transitions, so counting contact requires explicit deduplication and status policy rather than treating every event as one contact.
