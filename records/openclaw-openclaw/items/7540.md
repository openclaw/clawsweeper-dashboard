## What This Changes

The issue requests WhatsApp Web/Baileys call-event support so calls can count as contact activity alongside messages.

## Best Possible Solution

Implement this inside `extensions/whatsapp`: subscribe to Baileys `call`, normalize and sanitize the payload, apply account/DM/group/self-chat allowlist boundaries, add an explicit opt-in surface if maintainers choose one, and document/test the resulting activity API without routing it through the PSTN `voice-call` plugin.
