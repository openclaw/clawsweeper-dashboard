## Summary

Keep open. Current main still lacks WhatsApp Web/Baileys call-event handling, the pinned Baileys package exposes a typed `call` event, and the live discussion scopes this as valid WhatsApp-plugin-owned work that still needs privacy and API decisions.

## What This Changes

Add WhatsApp Web support for subscribing to Baileys call events so voice/video calls can count as contact activity.

## Best Possible Solution

Implement default-off, sanitized WhatsApp call-activity handling inside `extensions/whatsapp`, reusing the existing account, DM/group, self-chat, allowlist, and plugin-hook privacy boundaries.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers WhatsApp credential, connection, and message listeners while the pinned Baileys rc11 contract exposes a typed `call` event that OpenClaw does not consume.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, opt-in call tracking through a sanitized documented event/tool/session surface rather than folding WhatsApp Web call activity into the PSTN voice-call plugin.
