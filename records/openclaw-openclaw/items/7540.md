## Summary

Keep open. Current main still lacks WhatsApp Web/Baileys call-event handling, the dependency exposes a typed `call` event, and the discussion plus source evidence scope this as valid WhatsApp-plugin-owned feature work with privacy and API decisions still unresolved.

## What This Changes

Request to add WhatsApp Web/Baileys voice and video call-event tracking so calls can count as contact activity.

## Best Possible Solution

Implement default-off, sanitized WhatsApp call-activity handling inside `extensions/whatsapp`, reusing existing account, DM/group, self-chat, allowlist, and plugin-hook privacy boundaries.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers WhatsApp credential, connection, and message listeners while the pinned Baileys rc11 contract exposes a typed `call` event that OpenClaw does not consume.

## Solution Assessment

Unclear on the exact final API, but the maintainer direction is sound: keep this WhatsApp-plugin-owned and expose sanitized, opt-in call activity rather than folding WhatsApp Web calls into the PSTN voice-call plugin.
