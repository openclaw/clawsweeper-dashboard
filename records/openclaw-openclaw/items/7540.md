## Summary

Keep this open: current main still lacks a WhatsApp/Baileys call-event subscription or exposed call activity surface, and the discussion already scopes the work as valid WhatsApp-plugin-owned product/privacy work.

## What This Changes

Add WhatsApp Web support for subscribing to Baileys call events so voice/video calls can count as contact activity alongside messages.

## Best Possible Solution

Implement this inside `extensions/whatsapp` with sanitized Baileys call-event handling, inbound access-control parity, an explicit opt-in/exposure policy, focused tests, and public docs for the resulting hook/tool/session-event surface.

## Reproduction Assessment

Yes, at source level: current main subscribes to WhatsApp message and connection events only, while the pinned Baileys contract exposes a `call` event that OpenClaw does not consume. I did not run a live WhatsApp call.

## Solution Assessment

Yes on direction, but not as an automated patch yet: the work belongs in the WhatsApp plugin rather than the PSTN voice-call plugin, and maintainers still need to decide the opt-in, privacy, and exposure surface.
