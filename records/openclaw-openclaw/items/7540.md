## Summary

Keep open: current main still lacks WhatsApp/Baileys call-event support, Baileys exposes the event, and maintainer discussion already scoped this as valid WhatsApp-plugin-owned work rather than a PSTN voice-call or ClawHub item.

## What This Changes

Add WhatsApp Web/Baileys call-event tracking so voice/video calls can count as contact activity through a controlled plugin/session/tool surface.

## Best Possible Solution

Implement this inside `extensions/whatsapp`: add opt-in call tracking, subscribe to Baileys `call`, normalize and sanitize `WACallEvent`, apply account/DM/group/self/allowlist privacy boundaries, expose a documented plugin/session/tool surface, and add focused tests plus docs.

## Reproduction Assessment

Yes, source-reproducible. Current main attaches only message and connection listeners in the WhatsApp monitor, a targeted source search found no call-event surface, and Baileys 7.0.0-rc11 exposes `call: WACallEvent[]`.

## Solution Assessment

Yes at the ownership boundary: this belongs in the WhatsApp plugin, not the PSTN voice-call plugin or a separate core channel. The exact opt-in config and sanitized exposure surface still need maintainer product/privacy judgment.
