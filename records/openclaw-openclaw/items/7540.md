## Summary

Keep open: current main and the latest release still lack WhatsApp/Baileys call-event handling, while the pinned Baileys dependency exposes a `call` event; the remaining work is valid but needs product, privacy, and security review before implementation.

## What This Changes

Request support for detecting and exposing WhatsApp voice/video call events from Baileys in the bundled WhatsApp plugin.

## Best Possible Solution

Implement this inside `extensions/whatsapp` with opt-in call tracking, sanitized `WACallEvent` normalization, existing account/DM/group/self/allowlist boundaries, a documented exposure surface, focused tests, and redacted live or fixture-backed proof.

Yes, at source level. Current main registers `creds.update`, `connection.update`, and `messages.upsert` but no `call` listener, while the pinned Baileys version exposes `Events.call` as `WACallEvent[]`.

## Solution Assessment

Partly. WhatsApp plugin ownership is the right boundary, but maintainers still need to choose the opt-in setting and runtime exposure surface before implementation.
