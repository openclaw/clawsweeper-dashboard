## Summary

Keep open. Current main still lacks WhatsApp Web/Baileys call-event handling, the dependency exposes the event, and the discussion already scopes this as valid WhatsApp-plugin-owned work that needs privacy/API decisions.

## What This Changes

Request adding support for Baileys WhatsApp `call` events so WhatsApp calls can count as contact activity.

## Best Possible Solution

Implement default-off, sanitized call-activity handling inside `extensions/whatsapp` with the same account, DM/group, self-chat, allowlist, and plugin-hook privacy boundaries as inbound messages.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main only registers credential, connection, and message listeners for WhatsApp Web, while the pinned Baileys rc10 source exposes a typed `call` event.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, opt-in call tracking through a sanitized documented surface rather than folding WhatsApp Web call events into the PSTN voice-call plugin or a separate third-party outbound-call plugin.
