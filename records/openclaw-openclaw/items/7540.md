## Summary

Keep open: current main and the latest release still lack WhatsApp/Baileys call-event handling, while the issue discussion scopes this as valid WhatsApp-plugin work that needs privacy, security, and product decisions before implementation.

## What This Changes

The issue requests opt-in WhatsApp/Baileys call-event tracking so WhatsApp voice/video calls can count as contact activity for agents.

## Best Possible Solution

Implement this inside `extensions/whatsapp` with opt-in call tracking, sanitized `WACallEvent` normalization, existing DM/group/self/allowlist checks, a documented exposure contract, focused tests, docs, and redacted runtime proof.

## Reproduction Assessment

Yes, for source-level reproduction. Current main registers WhatsApp credential, connection, and message-upsert events but no Baileys `call` listener, while the pinned Baileys tag exposes `call: WACallEvent[]`.

## Solution Assessment

Unclear. The WhatsApp plugin boundary is right, but maintainers still need to choose the opt-in config and whether call activity is exposed as session metadata, a tool query, a plugin hook, or another event surface.
