## Summary

Keep open. Current main still lacks a WhatsApp/Baileys call listener or documented call-event surface, and maintainer discussion scopes this as valid WhatsApp-plugin-owned feature work that needs privacy and API decisions.

## What This Changes

Add WhatsApp Web/Baileys voice and video call-event tracking so calls can count as contact activity alongside messages.

## Best Possible Solution

Implement this inside extensions/whatsapp behind an explicit opt-in, normalize and sanitize WACallEvent payloads, reuse existing account/access/privacy boundaries, document the chosen hook/tool/session surface, and add focused tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers WhatsApp credential, connection, and message listeners but no Baileys call listener, while the pinned Baileys version exposes a typed call event.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is to keep this WhatsApp-plugin-owned with an explicit privacy opt-in, then expose sanitized call activity through a documented surface rather than routing it through the PSTN voice-call plugin.
