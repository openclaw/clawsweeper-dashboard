## Summary

Keep open. Current main still handles WhatsApp Web inbound activity as message-only, the pinned Baileys source exposes a typed `call` event, and recent maintainer context scopes this as bundled WhatsApp-plugin work that needs privacy/API decisions before implementation.

## What This Changes

Request adding WhatsApp Web/Baileys call-event tracking so voice/video calls can count as contact activity alongside messages.

## Best Possible Solution

Implement this inside `extensions/whatsapp` as an opt-in, sanitized call-activity surface that reuses the existing account, DM/group, self-chat, allowlist, and privacy boundaries, with focused tests and docs.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers credential, connection, and message listeners but no Baileys `call` listener, while the pinned Baileys dependency exposes a typed call event.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, default-off call tracking through a sanitized documented surface rather than folding WhatsApp Web calls into the PSTN voice-call plugin.
