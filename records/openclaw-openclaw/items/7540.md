## Summary

## What This Changes

The issue requests WhatsApp Web support for Baileys `call` events so call activity can count as contact activity alongside messages.

## Best Possible Solution

Implement this inside `extensions/whatsapp` with opt-in call tracking, sanitized `WACallEvent` normalization, existing account/DM/group/self/allowlist boundaries, a documented exposure surface, and focused tests plus redacted live or fixture-backed proof.

Yes for source-level reproduction. Current main registers `creds.update`, `connection.update`, and `messages.upsert` but no `call` listener, while the pinned Baileys version exposes `Events.call` as `WACallEvent[]`.

## Solution Assessment

Partly. WhatsApp plugin ownership is the right boundary, but maintainers still need to choose the opt-in setting and runtime exposure surface before implementation.
