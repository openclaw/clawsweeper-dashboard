## Summary

Keep open. Current main and the latest release still do not subscribe to Baileys `call` events or expose WhatsApp call activity, while the pinned Baileys package supports that event; the remaining work is a valid WhatsApp-plugin-owned feature that needs privacy/security and product-contract review.

## What This Changes

The issue requests WhatsApp Web/Baileys call-event support so calls can count as contact activity alongside messages.

## Best Possible Solution

Approve a WhatsApp-plugin-owned call activity contract, then add a sanitized Baileys `call` listener with opt-in privacy gating, access-control parity, focused tests, and docs.

## Reproduction Assessment

Yes, at source level. Current WhatsApp code subscribes to and processes message events only, while the pinned Baileys event map includes a separate `call` event that OpenClaw does not consume.

## Solution Assessment

Unclear at final contract detail. The right boundary is the WhatsApp plugin, but maintainers still need to settle the opt-in config, payload shape, and whether the exposed surface is a hook, tool, or session activity model.
