## Summary

Keep open. Current main still lacks WhatsApp Web/Baileys call-event handling, Baileys rc10 exposes a typed `call` event, and the issue discussion scopes this as valid WhatsApp-plugin-owned work that needs privacy/API decisions.

## What This Changes

Add WhatsApp Web/Baileys call-event subscription and expose sanitized call activity so agents can count WhatsApp calls as contact activity.

## Best Possible Solution

Implement default-off, sanitized call-activity handling inside `extensions/whatsapp`, reusing existing account, DM/group, self-chat, allowlist, and plugin-hook privacy boundaries.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers credential, connection, and message listeners for WhatsApp Web, while the pinned Baileys rc10 contract exposes a typed `call` event that OpenClaw does not consume.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, opt-in call tracking through a sanitized documented event/tool/session surface rather than folding WhatsApp Web call activity into the PSTN voice-call plugin.
