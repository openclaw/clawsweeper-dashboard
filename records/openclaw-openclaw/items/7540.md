## Summary

Keep open: current main and the latest release still subscribe only to WhatsApp message/connection events, while the pinned Baileys contract exposes call events and maintainer discussion already scoped this as valid WhatsApp-plugin-owned work.

## What This Changes

Add WhatsApp Web/Baileys call-event tracking so WhatsApp calls can count as contact activity alongside messages.

## Best Possible Solution

Implement this inside extensions/whatsapp with opt-in call tracking, sanitized WACallEvent normalization, existing account/DM/group/self/allowlist boundaries, a documented exposure surface, and focused tests plus docs.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp subscribes to messages.upsert and connection.update only, targeted search found no call-event surface, and the pinned Baileys version exposes a call event.

## Solution Assessment

Yes at the ownership boundary: the work belongs in the WhatsApp plugin rather than the PSTN voice-call plugin or core. The opt-in default and sanitized exposure surface still need maintainer product/privacy review before implementation.
