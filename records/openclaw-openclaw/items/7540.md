## Summary

Keep open: current main and v2026.6.5 still lack a WhatsApp/Baileys `call` listener or call exposure contract, and prior maintainer discussion scoped this as valid WhatsApp-plugin work that needs privacy and product direction.

## What This Changes

Add WhatsApp/Baileys call-event tracking so calls can count as contact activity alongside messages.

## Best Possible Solution

Implement this in `extensions/whatsapp` with opt-in call tracking, sanitized event normalization, existing DM/group/self/allowlist checks, a documented exposure contract, and focused tests/docs after maintainer privacy direction.

## Reproduction Assessment

Yes, source-level: current main registers WhatsApp message and connection listeners but no Baileys `call` listener, while the pinned Baileys package exposes `call: WACallEvent[]`. I did not run a live WhatsApp call scenario.

## Solution Assessment

Unclear. The WhatsApp plugin boundary is the right place, but maintainers still need to choose the opt-in setting and whether call activity is exposed as session metadata, a tool query, a plugin hook, or another event surface.
