## Summary

Keep open. Current main still lacks WhatsApp/Baileys call-event handling, and maintainer discussion scopes this as valid WhatsApp-plugin-owned work that needs privacy and API-surface decisions before implementation.

## What This Changes

Add WhatsApp Web/Baileys voice and video call-event tracking so call activity can count as contact alongside messages.

## Best Possible Solution

Implement this inside `extensions/whatsapp` behind explicit opt-in, applying existing inbound privacy/access controls and documenting the chosen call-activity surface.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to inspect current main and confirm WhatsApp registers credential, connection, and message listeners but no Baileys `call` listener, while the pinned Baileys API exposes a typed `call` event.

## Solution Assessment

Unclear on the exact API. The maintainable direction is clear: keep the work WhatsApp-plugin-owned with an explicit privacy opt-in rather than routing it through the PSTN Voice Call plugin or the closed outbound WhatsApp-call PR.
