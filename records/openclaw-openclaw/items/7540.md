## Summary

Keep open: current main still lacks Baileys call-event handling, the latest release also omits it, and maintainer discussion scopes this as privacy-sensitive WhatsApp plugin work that needs a product/security decision before implementation.

## What This Changes

The issue requests WhatsApp Web support for subscribing to Baileys `call` events so voice/video calls can count as contact activity.

## Best Possible Solution

Implement this inside `extensions/whatsapp` with opt-in call tracking, sanitized `WACallEvent` normalization, existing account/DM/group/self/allowlist boundaries, a documented exposure surface, and focused tests plus redacted live or fixture-backed proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main registers `creds.update`, `connection.update`, and `messages.upsert` but no call listener, while the pinned Baileys package exposes `Events.call` as `WACallEvent[]`.

## Solution Assessment

Partly. WhatsApp plugin ownership is the right boundary, but maintainers still need to choose the opt-in setting and runtime exposure surface before implementation.
