## Summary

Keep this open: current main still has no WhatsApp/Baileys call-event subscription or exposure path, and the existing discussion already scopes the work as valid WhatsApp-plugin-owned product/privacy work rather than stale cleanup.

## What This Changes

The issue requests support for subscribing to Baileys WhatsApp call events and exposing WhatsApp call activity to OpenClaw agents.

## Best Possible Solution

Implement this inside `extensions/whatsapp` by subscribing to Baileys `call`, normalizing and sanitizing payloads, applying the same account/DM/group/self-chat/allowlist boundaries as inbound messages, adding an opt-in setting if maintainers want that privacy gate, and documenting the resulting hook/tool/session-event surface.

## Reproduction Assessment

Yes, at source level: current main subscribes to WhatsApp message and connection events only, while the pinned Baileys contract exposes a `call` event that OpenClaw does not consume. I did not run a live WhatsApp call.

## Solution Assessment

Yes on direction, but not as an automated patch yet: the work belongs in the WhatsApp plugin, not the PSTN voice-call plugin, and the exact opt-in/privacy/exposure surface needs maintainer product judgment.
