## Summary

Keep open. Current main still registers WhatsApp credential, connection, and message listeners only; the pinned Baileys version exposes call events, and maintainer discussion scopes this as valid WhatsApp-plugin-owned work that needs privacy and API-surface decisions.

## What This Changes

Add WhatsApp Web/Baileys call-event tracking so WhatsApp voice/video calls can count as contact activity.

## Best Possible Solution

Implement this in extensions/whatsapp behind an explicit opt-in, normalize sanitized WACallEvent payloads, reuse existing DM/group/self-chat/account boundaries, document the chosen surface, add focused tests, and include a user-facing changelog entry.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main shows WhatsApp registering credential, connection, and message listeners but no Baileys call listener, while the pinned Baileys version exposes a typed call event.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is clear: keep this WhatsApp-plugin-owned with an explicit privacy opt-in instead of routing it through core or the PSTN voice-call plugin.
