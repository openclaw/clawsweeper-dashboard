## Summary

Keep open. Current main still lacks a WhatsApp/Baileys call-event listener or call-activity surface, and prior maintainer context scopes this as bundled WhatsApp plugin work that needs privacy/API decisions.

## What This Changes

Add support for WhatsApp Web/Baileys call events so WhatsApp call activity can be counted alongside messages.

## Best Possible Solution

Implement this in extensions/whatsapp behind an explicit opt-in, normalize and sanitize WACallEvent payloads, reuse account/access/privacy boundaries, document the chosen call-activity surface, and add focused tests plus changelog coverage.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers credential, connection, and message listeners but no Baileys call listener, while the pinned Baileys package exposes a typed call event.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, default-off call tracking through a sanitized documented surface, not the closed outbound WhatsApp-call plugin proposal or the PSTN voice-call plugin.
