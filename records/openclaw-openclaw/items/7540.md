## Summary

Keep open. Current main still lacks a WhatsApp/Baileys call-event listener or call-activity surface, and prior maintainer context explicitly scoped this as bundled WhatsApp-plugin work that needs privacy/API decisions.

## What This Changes

Request support for detecting WhatsApp voice/video calls from Baileys `call` events and exposing that contact activity to agents.

## Best Possible Solution

Implement this inside `extensions/whatsapp` behind an explicit opt-in, normalize and sanitize `WACallEvent` payloads, reuse existing account/access/privacy boundaries, document the chosen activity surface, and add focused tests plus changelog coverage.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers credential, connection, and message listeners but no Baileys `call` listener, while the pinned Baileys package exposes a typed call event.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, default-off call tracking through a sanitized documented surface, not the closed outbound WhatsApp-call plugin proposal or the PSTN voice-call plugin.
