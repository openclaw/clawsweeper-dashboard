## Summary

Keep open: current main and the latest release still lack WhatsApp/Baileys call-event handling, and the issue discussion already scopes this as valid WhatsApp-plugin work that needs privacy, security, and product decisions before implementation.

## What This Changes

Add WhatsApp Web support for Baileys `call` events so agents can count WhatsApp voice/video calls as contact activity.

## Best Possible Solution

Implement this inside the WhatsApp plugin with opt-in call tracking, sanitized `WACallEvent` normalization, existing account/DM/group/self/allowlist checks, a documented exposure contract, focused tests, and redacted live or fixture-backed proof.

## Reproduction Assessment

Yes, for source-level reproduction. Current main registers `creds.update`, `connection.update`, and `messages.upsert` but no `call` listener, while the pinned Baileys version exposes `Events.call` as `WACallEvent[]`.

## Solution Assessment

Unclear. The WhatsApp plugin boundary is right, but maintainers still need to choose the opt-in config and whether call activity is exposed as session metadata, a tool query, a plugin hook, or another event surface.
