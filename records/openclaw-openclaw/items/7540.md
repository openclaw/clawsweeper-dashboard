## Summary

Keep open. Current main still has no WhatsApp/Baileys call-event subscription or call-activity surface, and maintainer discussion already scoped this as valid WhatsApp-plugin-owned feature work that needs privacy and API decisions.

## What This Changes

Add support for WhatsApp Web/Baileys voice and video call events so call activity can be detected alongside inbound messages.

## Best Possible Solution

Implement this inside extensions/whatsapp behind an explicit opt-in, normalize and sanitize WACallEvent payloads, reuse existing account/access/privacy boundaries, document the chosen hook/tool/session surface, and add focused tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers WhatsApp credential, connection, and message listeners but no Baileys call listener, while the pinned Baileys version exposes a typed call event.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, default-off call tracking that exposes sanitized activity through a documented surface rather than routing WhatsApp calls through the PSTN voice-call plugin.
