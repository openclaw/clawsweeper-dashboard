## Summary

Keep open. Current main still lacks WhatsApp/Baileys call-event handling: the WhatsApp plugin subscribes to credential, connection, and message events only, while maintainer discussion scopes this as valid WhatsApp-plugin-owned work that needs privacy and API-surface decisions.

## What This Changes

Add WhatsApp Web support for detecting Baileys voice/video call events and exposing that call activity to agents.

## Best Possible Solution

Implement this inside `extensions/whatsapp` behind an explicit opt-in, normalize sanitized Baileys call events, apply existing account/DM/group/self-chat privacy boundaries, document the chosen surface, and add focused tests.

## Reproduction Assessment

Yes, as a source-level gap rather than a failing bug. Inspecting current main shows WhatsApp registers credential, connection, and message listeners but no Baileys `call` listener, while the pinned Baileys version exposes a typed `call` event.

## Solution Assessment

Unclear on the exact API. The maintainable direction is clear: keep the work WhatsApp-plugin-owned with an explicit privacy opt-in instead of routing it through core or the PSTN Voice Call plugin.
