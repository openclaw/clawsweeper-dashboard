The issue requests WhatsApp Web/Baileys call-event tracking so call activity can count as contact activity alongside messages.

## Best Possible Solution

Implement a WhatsApp-plugin-owned, opt-in sanitized call-activity path that applies existing account, DM/group, self-chat, allowlist, and privacy boundaries before recording contact activity or emitting a documented hook/tool surface.

## Reproduction Assessment

Yes for the current gap: source inspection shows OpenClaw subscribes to WhatsApp messages and connection/group events but not Baileys call events. No live WhatsApp call flow was run.

## Solution Assessment

Unclear as a final product shape. The WhatsApp plugin is the right owner boundary, but maintainers still need to choose one sanitized opt-in activity surface rather than adding ad hoc session metadata plus a parallel tool API.
