## Summary

Keep open. Current main still lacks WhatsApp/Baileys call-event support, the issue discussion keeps this scoped as valid WhatsApp-plugin-owned work, and the remaining design needs privacy, config, and API decisions.

## What This Changes

Add WhatsApp Web support for subscribing to Baileys `call` events so voice/video call activity can count as contact activity.

## Best Possible Solution

Implement default-off, sanitized WhatsApp call-activity handling inside `extensions/whatsapp`, reusing the existing account, DM/group, self-chat, allowlist, and plugin-hook privacy boundaries.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main registers WhatsApp credential, connection, and message listeners while the pinned Baileys contract exposes a typed `call` event that OpenClaw does not consume.

## Solution Assessment

Unclear on the exact final API. The maintainable direction is WhatsApp-plugin-owned, opt-in call tracking through a sanitized documented event/tool/session surface rather than folding WhatsApp Web call activity into the PSTN voice-call plugin.
