## Summary

Keep open: current main and the latest release still lack WhatsApp/Baileys call-event handling, and the existing discussion scopes this as valid WhatsApp-plugin work that needs privacy, security, and product decisions before implementation.

## What This Changes

Add opt-in WhatsApp Web call-event tracking for Baileys `call` events so voice/video calls can count as contact activity.

## Best Possible Solution

Implement this inside `extensions/whatsapp` with opt-in call tracking, sanitized `WACallEvent` normalization, existing account/DM/group/self/allowlist boundaries, a documented exposure surface, focused tests, and redacted live or fixture-backed proof.

## Reproduction Assessment

Yes for source-level reproduction. Current main registers `creds.update`, `connection.update`, and `messages.upsert` but no `call` listener, while the pinned Baileys version exposes `Events.call` as `WACallEvent[]`.

## Solution Assessment

Unclear. The WhatsApp plugin boundary is right, but maintainers still need to choose the opt-in config and whether call activity is exposed as session metadata, a tool query, or a plugin hook.
