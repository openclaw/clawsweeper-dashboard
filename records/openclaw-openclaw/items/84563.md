## What This Changes

The branch changes Telegram partial-stream final delivery to wait for successful continuation chunks, repairs truncated preview-finalized text from the transcript, adds regression tests, and suppresses active abort recovery for queued embedded diagnostic sessions.

## Best Possible Solution

Land the Telegram delivery-state fix only after redacted live Telegram proof confirms the partial-stream final fallback or repair path, and either split the diagnostics abort-policy change or have a maintainer explicitly accept that recovery tradeoff.

## Reproduction Assessment

Yes at source level: current main marks a Telegram streamed final delivered before sending continuation chunks, and the PR tests model both thrown and false-return follow-up failures. I did not run tests because this review is read-only and the missing piece is real Telegram proof.

## Solution Assessment

Mostly yes for the Telegram fix: moving markDelivered after successful continuation sends matches the one-preview streaming design. The diagnostics change is less clearly coupled, so the best path is maintainer acceptance with proof or splitting that policy change into a separate review.
