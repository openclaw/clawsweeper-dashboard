## Summary

Keep open. The reported memory-index noise is real, but this PR is not merge-ready: it targets legacy memory files, does not cover the active memory-host SDK or QMD collection path, adds an unrelated IPv4 runtime helper, and still lacks after-fix real behavior proof.

## What This Changes

The branch adds a `memorySearch.excludeDirs` config/schema/status path and legacy directory-walk filtering, plus an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Define one documented exclusion contract across active SDK indexing, QMD collections/traversal, watcher behavior, status output, docs, and focused regression tests, then remove the unrelated network helper.

## Reproduction Assessment

Yes, by source inspection. Current main still recursively scans configured builtin `extraPaths` without an exclusion parameter, and QMD maps those paths to plain collections with no traversal or exclusion policy.

## Solution Assessment

No. The direction is plausible, but this branch is not the best current-main solution because it changes obsolete files, leaves QMD traversal semantics unresolved, exposes a public config key prematurely, and mixes in unrelated network behavior.
