## Summary

Keep open. The requested exclusion feature is plausible, but this PR is not merge-ready on current main because it patches obsolete memory files, leaves the active memory-host/QMD contract unresolved, lacks real after-fix behavior proof, and includes an unrelated root network helper.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface for excluding directory basenames during memory indexing and also adds a root `ipv4-fix.cjs` helper.

## Best Possible Solution

Define the exclusion contract once on current main, wire it through the active memory host SDK and any intended QMD behavior, update config/docs/status/tests together, and drop unrelated network changes.

## Reproduction Assessment

Yes, by source inspection. Current main recursively scans configured `extraPaths` through the active memory host SDK without a directory-basename exclusion input, so a configured directory containing `node_modules/*.md` would still be included by the builtin path.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best current-main solution because it targets obsolete memory files, leaves QMD and basename-versus-pattern semantics unresolved, and mixes in unrelated network behavior.
