## Summary

This PR should stay open for maintainer review because the feature is plausible, but the branch is stale against the active memory-host SDK/QMD implementation, includes an unrelated network helper, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface intended to skip configured directory basenames during recursive memory indexing.

## Best Possible Solution

Define one memory exclusion contract across builtin and QMD indexing, implement it in the active memory-host SDK/QMD boundary before traversal, document it in the public memory config reference, and drop unrelated network changes.

## Reproduction Assessment

Yes, by source inspection. Current main still has no `memorySearch.excludeDirs` config and the active builtin scanner recursively scans normalized `extraPaths` without a directory-basename exclusion input, while QMD extra paths are resolved as plain collections.

## Solution Assessment

No. The feature direction is plausible, but this patch targets removed `src/memory/*` files, leaves active SDK/QMD semantics unresolved, does not skip excluded extra path roots, and includes unrelated process-wide network behavior.
