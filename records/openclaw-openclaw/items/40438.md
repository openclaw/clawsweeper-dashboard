Keep open: the feature remains useful and current main still lacks an OpenClaw `excludeDirs` setting, but this branch is not mergeable as-is because it targets the old memory walker, misses the active SDK/QMD paths, and includes an unrelated network hook. ## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface for pruning named directories during memory indexing, and also adds an unrelated root IPv4 DNS/socket helper.

## Best Possible Solution

Rebase or replace the branch with a current-main implementation that defines builtin/QMD exclusion semantics, wires the active memory host SDK plus memory-core status/docs/schema/changelog, and removes unrelated network changes.

Yes.
