## Summary

Keep open. The underlying memory-index pollution problem is still real, but this branch is not mergeable as-is because it patches obsolete legacy memory files, does not define the active builtin/QMD traversal contract, lacks real after-fix behavior proof, and includes an unrelated root network helper.

## What This Changes

The PR adds `memorySearch.excludeDirs` config/schema/status/test plumbing for memory indexing and also adds a root `ipv4-fix.cjs` DNS/socket helper.

## Best Possible Solution

Define one backend-consistent exclusion/traversal contract in the active memory-host SDK and QMD paths, document basename-vs-pattern semantics, add focused regressions, and remove unrelated network behavior.

## Reproduction Assessment

Yes, by source inspection. Current main still recursively scans configured builtin `extraPaths` without an exclusion parameter, and QMD maps those paths to plain collections with no traversal/exclusion policy.

## Solution Assessment

No. The direction is plausible, but this branch is not the best current-main solution because it changes obsolete files, leaves QMD traversal semantics unresolved, exposes a public config key prematurely, and mixes in unrelated network behavior.
