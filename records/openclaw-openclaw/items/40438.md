The PR adds a `memorySearch.excludeDirs` config/status/test surface for pruning named directories from memory indexing and also adds a root `ipv4-fix.cjs` DNS/socket helper.

## Best Possible Solution

Land a current-main replacement that implements normalized default/custom excludes through config, generated schema/help, docs/status, `packages/memory-host-sdk`, and `extensions/memory-core`, defines QMD behavior explicitly, tests root and nested pruning, and omits unrelated network changes.

No. The feature direction is reasonable, but this patch is not the best fix as-is because it targets old memory files, misses generated/public surface follow-through, does not skip excluded extra path roots, and includes an unrelated IPv4 hook.
