## Summary

Keep open. The feature is still valid, but this PR is not merge-ready because it targets the old memory walker, misses the active memory-host SDK and memory-core paths on current main, and includes an unrelated process-wide network hook.

## What This Changes

The PR branch adds a `memorySearch.excludeDirs` option for memory indexing config/status/tests and also adds an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Land a current-main implementation in the active memory-host SDK and memory-core plugin with normalized default/custom excludes, root and nested pruning, generated config/docs/changelog updates, focused tests, and explicit QMD semantics.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure `memorySearch.extraPaths` to a directory containing `node_modules/*.md`, and current main's active `listMemoryFiles` path will recursively walk it because there is no exclude-dir input or basename pruning.

## Solution Assessment

No. The direction is reasonable, but this PR is not the best merge path as-is; the safer path is a current-main rebase or replacement through `packages/memory-host-sdk` and `extensions/memory-core`, with the unrelated network hook removed.
