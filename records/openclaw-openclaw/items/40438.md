## Summary

Keep open. Current main still lacks a `memorySearch.excludeDirs` contract, and this PR remains a plausible feature direction, but the branch is not mergeable as-is because it targets removed legacy memory files, leaves QMD traversal semantics unresolved, adds an unrelated root network helper, and lacks contributor real behavior proof.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface and tries to skip excluded directory basenames during memory file discovery.

## Best Possible Solution

Define one exclusion/traversal contract across the active memory-host SDK and QMD paths, document basename-vs-pattern semantics, add focused regression coverage, and remove unrelated network changes.

## Reproduction Assessment

Yes, by source inspection. Current main still scans configured builtin `extraPaths` recursively without an exclusion parameter and maps QMD extra paths to plain path collections; I did not run a live indexing repro.

## Solution Assessment

No. The direction is plausible, but this branch is not the best current-main solution because it changes obsolete files, leaves QMD traversal semantics unresolved, exposes a public config key prematurely, and mixes in unrelated network behavior.
