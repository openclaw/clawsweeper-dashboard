## Summary

Keep open for maintainer review: the feature direction is plausible, but this branch is stale against current `main`, targets removed memory files, exposes a new config contract before QMD/builtin traversal semantics are settled, lacks real behavior proof, and includes an unrelated network hook.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface for basename-based directory pruning in memory indexing, plus a root `ipv4-fix.cjs` helper.

## Best Possible Solution

Define one exclusion/traversal contract across builtin SDK and QMD, implement it in the active current-main paths before exposing config/docs/status, and leave the QMD symlink-loop boundary aligned with the related runtime investigation.

## Reproduction Assessment

Yes, by source inspection. Current `main` scans configured `extraPaths` through the memory-host SDK without a directory-basename exclusion input, and QMD maps `extraPaths` to path-only collections; no live indexing run was performed.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best current-main solution because it changes obsolete files, leaves QMD traversal semantics unresolved, exposes a new public config key prematurely, and mixes in unrelated network behavior.
