## Summary

Keep open. Current main still lacks a `memorySearch.excludeDirs` contract, but this branch is not mergeable as-is because it edits obsolete memory files instead of the active SDK/QMD path, includes unrelated network code, and lacks real behavior proof.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface intended to skip directory basenames during recursive memory indexing, plus an unrelated root IPv4 helper.

## Best Possible Solution

Define the exclusion contract once, implement it in the active memory-host SDK and any intended QMD path before traversal, update config/docs/status/tests together, and drop unrelated network changes.

## Reproduction Assessment

Yes, by source inspection. Current main recursively scans configured `extraPaths` through the active memory-host SDK without an `excludeDirs` input, so markdown under a dependency directory in an indexed extra path can still enter the builtin index.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best current-main solution because it targets obsolete `src/memory` paths, leaves QMD and pattern semantics unresolved, and mixes in unrelated network behavior.
