## Summary

Keep open for human review: the feature direction addresses real memory-index noise, but this branch targets removed legacy memory files, does not define the active QMD traversal contract, adds an unrelated root network helper, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface for basename-based memory indexing exclusions and also adds an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Define one exclusion/traversal contract across the active memory-host SDK and QMD paths, document the chosen basename-vs-pattern semantics, add focused regression coverage, and remove unrelated runtime/network changes.

## Reproduction Assessment

Yes, by source inspection. Current main still recursively indexes configured memory `extraPaths` through the active SDK and maps QMD extra paths to plain path collections; no live indexing run was performed in this read-only review.

## Solution Assessment

No. The direction is plausible, but this branch is not the best current-main solution because it changes obsolete files, leaves QMD traversal semantics unresolved, exposes a new public config key prematurely, and mixes in unrelated network behavior.
