## Summary

Keep open. The feature addresses a real memory-index noise problem, but this branch is not merge-ready on current main because it patches obsolete memory files, leaves the active memory-host SDK and QMD semantics unresolved, lacks after-fix real behavior proof, and includes an unrelated root network helper.

## What This Changes

The branch adds a `memorySearch.excludeDirs` config/status/test surface for skipping named directories during memory indexing and also adds a root IPv4 DNS/socket helper.

## Best Possible Solution

Define the exclusion contract once in the current memory-host SDK, decide how it applies to builtin versus QMD indexing, update config/docs/status/tests together, and drop the unrelated IPv4 helper.

## Reproduction Assessment

Yes for the reported problem. Source inspection shows current main still recursively scans configured `extraPaths` through the active memory-host SDK without a directory-basename exclusion input, so dependency-directory Markdown can still be included by the builtin path.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best current-main solution because it patches obsolete files, does not settle QMD or pattern semantics, misses excluded root paths, and mixes in unrelated networking behavior.
