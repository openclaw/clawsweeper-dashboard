## Summary

Keep open. The user problem is plausible and still not implemented on current main, but this PR is not mergeable as-is because it patches legacy memory files, leaves the current SDK/QMD paths unsettled, and includes an unrelated root network hook.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface for skipping named directories during recursive memory indexing, plus an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Rework this on current main by defining the exclusion contract once, wiring it through the active memory host SDK and any intended QMD behavior, updating config/docs/status/tests, and dropping the unrelated network helper.

## Reproduction Assessment

Yes. Source inspection shows current main recursively scans configured `extraPaths` through the active memory host SDK without a directory-basename exclude input, so a configured directory containing `node_modules/*.md` would still be included by the builtin path.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best current-main solution because it patches obsolete files, leaves QMD and basename-versus-pattern semantics unresolved, and mixes in unrelated network behavior.
