## Summary

Keep this PR open because the user problem is still real, but the patch is not mergeable against current main: it targets the old memory implementation, misses current generated/config and backend surfaces, and includes an unrelated network hook.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface for excluding named directories during memory indexing and also adds an unrelated root IPv4 DNS/socket helper.

## Best Possible Solution

Rework this as a current-main replacement that defines the exclude contract once, wires it through the active memory host SDK and memory-core surfaces, documents builtin versus QMD behavior, adds focused tests and changelog, and omits unrelated network changes.

## Reproduction Assessment

Yes. Source inspection shows current builtin `listMemoryFiles` recursively walks configured `extraPaths` without a directory-basename exclude input, so an extra path containing `node_modules/*.md` would still be indexed.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best current-main solution because it modifies removed legacy files, leaves QMD semantics unclear, misses generated/public follow-through, and mixes in an unrelated network hook.
