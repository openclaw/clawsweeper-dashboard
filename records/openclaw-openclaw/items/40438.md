## Summary

Keep open. The feature is still missing on current main, but this PR is not merge-ready: it targets pre-refactor memory paths, omits generated config artifacts, leaves a root extraPath bypass, and includes an unrelated root network hook that should not ship with a memory feature.

## What This Changes

The PR branch adds a `memorySearch.excludeDirs` option with default directory-basename filtering for memory indexing config/status/tests, and also adds an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Land the feature as a current-main implementation in the memory host SDK and memory-core plugin: normalize and merge default/custom exclude directory names, prune excluded root and nested directory paths before indexing, regenerate config schema/docs artifacts, add focused builtin and status tests, include a user-facing changelog entry, and either implement or explicitly document QMD behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: configure `memorySearch.extraPaths` to a directory containing `node_modules/*.md`, and the active `listMemoryFiles` path will recursively walk it because there is no exclude-dir input or basename pruning.

## Solution Assessment

No. The direction is reasonable, but this PR is not the best merge path as-is; the safer path is a current-main rebase or replacement through `packages/memory-host-sdk` and `extensions/memory-core`, with the unrelated network hook removed and QMD semantics deliberately handled.
