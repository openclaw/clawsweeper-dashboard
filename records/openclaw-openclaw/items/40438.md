## Summary

Keep open: the requested exclusion feature is still not implemented on current main, but this PR is not merge-ready because it targets removed legacy memory files, does not wire the active SDK/QMD paths, and includes unrelated process-wide network behavior.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config/status/test surface for skipping directory basenames during recursive memory indexing, plus an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Define the exclusion contract once, implement it in the active memory-host SDK and any intended QMD traversal path before scanning, update config/docs/status/tests together, and drop unrelated network changes.

## Reproduction Assessment

Yes, by source inspection. Current main recursively scans configured `extraPaths` through the active memory-host SDK without a directory-basename exclusion input, and QMD extra paths are plain collections with no exclusion metadata.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best current-main solution because it targets obsolete `src/memory` files, leaves active SDK/QMD semantics unresolved, fails to skip excluded extra path roots, and mixes in unrelated network behavior.
