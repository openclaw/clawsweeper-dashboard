## Summary

## What This Changes

The branch adds a top-level gateway `send.components` field, forwards it into Discord outbound payloads as raw native components, exposes extra outbound context fields through the runtime/Discord adapter path, re-exports `ChannelOutboundContext`, regenerates Swift protocol models, and adds a changelog line.

## Best Possible Solution

Maintainers should choose the gateway boundary first, likely either a Discord-owned nested `channelData.discord` input for this narrow fix or a typed presentation/action API, and keep internal outbound context types out of the SDK unless they are deliberately promoted.

## Solution Assessment
