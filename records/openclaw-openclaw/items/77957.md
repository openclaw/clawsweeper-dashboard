## Summary

Keep open: current `main` and v2026.6.10 still finalize completed Voice Call records by persisting terminal state and clearing active indexes, without a supported post-call Telegram/session notification path. The prior relay PR was closed unmerged as the wrong vehicle, so this remains a real product gap that needs maintainer direction.

## What This Changes

Completed inbound voice calls persist transcript records but do not produce a visible Telegram or session notification to the user.

## Best Possible Solution

Define a plugin-owned, opt-in post-call delivery contract for target session/channel, summary content, privacy defaults, and failure reporting, then implement it in Voice Call with focused regression and Telegram/channel proof.

Yes. Source inspection gives a high-confidence feature-gap path: completed calls reach `call.ended`, then `finalizeCall`, which persists and removes active state without any post-call delivery callout; I did not place a live Twilio/Telegram call in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. An opt-in plugin-owned post-call delivery path is safer than unconditional Telegram delivery or a core-specific shortcut.
