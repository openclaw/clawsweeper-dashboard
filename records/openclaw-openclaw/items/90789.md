## Summary

Keep open. Current main and the latest release still lack a synthetic-placeholder marker or non-GPT fallback classification for the reported Claude CLI stream-json shape, and the open closing PR remains the active fix candidate rather than a merged implementation.

## What This Changes

The issue requests that Claude CLI synthetic placeholder turns such as `No response requested.` stop disappearing silently in Telegram and instead become observable or fallback-worthy.

## Best Possible Solution

Land a focused PR or replacement that detects exact Claude CLI synthetic placeholders, preserves that metadata through CLI run results, and makes the outcome observable or fallback-worthy without suppressing legitimate `NO_REPLY` turns.

## Reproduction Assessment

Yes, at source level. A Claude stream-json `<synthetic>` assistant record followed by an empty result is currently normalized as plain empty text with no synthetic metadata, and the non-GPT fallback branch then returns `null`.

## Solution Assessment

Unclear. Parser-to-runner metadata is the right boundary, but maintainers still need to choose the exact recovery semantics and ensure the silent-policy path cannot turn this into an intentional `NO_REPLY`.
