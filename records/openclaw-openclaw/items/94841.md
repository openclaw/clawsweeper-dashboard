## Summary

## What This Changes

Adds a QMD command-token helper that preserves bare Windows drive-letter and UNC absolute paths before `splitShellArgs`, plus regression tests for Windows path cases and quoted flag support.

## Best Possible Solution

Narrow the raw-return branch to the Windows backslash and UNC forms that `splitShellArgs` actually mangles, keep tokenization for forward-slash drive-letter command strings with trailing tokens, and then use this branch as the canonical fix if maintainers prefer it over the sibling PR.

## Reproduction Assessment

Yes. Current main still routes `memory.qmd.command` through `splitShellArgs`, whose unquoted backslash handling strips Windows separators, and the PR body supplies Windows before/after terminal output using the real config resolver and QMD availability probe.

## Solution Assessment

No, not yet. Preserving Windows backslash absolute executable paths at the QMD config parser boundary is the right layer, but the current predicate also changes forward-slash drive-letter command strings that did not need the raw-return path.
