## Best Possible Solution

Land one canonical fix that preserves Windows absolute QMD executable paths before POSIX parsing, keeps the existing no-shell Windows spawn policy, includes focused regression tests, and has redacted Windows OpenClaw/QMD proof.

## Solution Assessment

Yes on fix boundary: preserving Windows absolute command paths before POSIX parsing is the narrow maintainable repair. It is not merge-ready until maintainers pick one canonical PR and get after-fix Windows OpenClaw/QMD proof.
