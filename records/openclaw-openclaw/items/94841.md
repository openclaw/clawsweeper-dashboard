## Summary

Keep this PR open: it fixes the right QMD config-parser boundary and has strong Windows proof, but the new drive-letter predicate also captures forward-slash Windows paths and can break currently tokenizable command strings before spawn.

## What This Changes

The PR preserves bare Windows drive-letter and UNC QMD command paths before shell-style tokenization, plus regression tests for backslash, UNC, spaced, and quoted Windows command forms.

## Best Possible Solution

Repair the predicate so backslash drive-letter and UNC paths are preserved without breaking already-tokenizable `C:/...` command strings, then land this branch as the canonical fix if maintainers choose it over the sibling PR.

## Reproduction Assessment

Yes. Current main still routes `memory.qmd.command` through `splitShellArgs`, whose unquoted backslash handling strips Windows separators, and the PR body supplies Windows before/after terminal output using the real config resolver and QMD availability probe.

## Solution Assessment

No, not yet. Preserving Windows backslash absolute executable paths at the QMD config parser boundary is the right layer, but the current predicate also changes forward-slash drive-letter command strings that did not need the raw-return path.
