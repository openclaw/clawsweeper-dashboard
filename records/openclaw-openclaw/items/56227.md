## Summary

Keep open: current main still restarts or closes the active gateway before replacement health is proven, so the requested old-instance-preserving handoff is not implemented.

## What This Changes

Request safer gateway restart semantics that start and health-check a replacement before stopping the currently reachable gateway.

## Best Possible Solution

Define a maintainer-approved cross-platform restart continuity contract, then implement a handoff that keeps the current listener available until replacement health succeeds or the restart is rejected safely.

## Reproduction Assessment

Yes at source level: managed and in-process restart paths close, restart, or signal the active gateway before replacement health is proven. I did not run a live supervisor repro.

## Solution Assessment

Unclear as an implementation-ready proposal: the atomic handoff goal is sound for availability, but the best fix needs a designed cross-platform supervisor, port, lock, rollback, and notification contract.
