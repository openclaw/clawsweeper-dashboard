## Summary

Keep open. Current main still shuts down the active Gateway before replacement readiness is proven, and the existing health checks cover post-restart diagnostics rather than preserving the old listener through a failed restart.

## What This Changes

The issue asks for Gateway restarts to keep the previous instance running until a replacement has started and passed a health check.

## Best Possible Solution

Design a supervisor-aware transactional restart model that stages or validates the replacement, proves readiness, transfers endpoint and channel ownership, and rolls back cleanly without dropping the old listener.

## Reproduction Assessment

Yes, at source level. A SIGUSR1/config restart closes the active server before replacement readiness is proven, and a later startup failure leaves `server` null rather than restoring the old listener.

## Solution Assessment

Unclear. The no-disconnect goal is sound, but the literal start-new-first sequence is not obviously the narrowest maintainable solution because same-port binding, service supervisors, config commit timing, and channel handoff all need design.
