## Summary

Keep open: current main has config preflight, safe restart deferral, restart health diagnostics, supervisor handoff diagnostics, and update-specific replacement health checks, but ordinary Gateway restarts still do not keep the previous live instance serving until a replacement is proven healthy.

## What This Changes

Request an atomic Gateway restart handoff that starts and health-checks a replacement before stopping the previous instance.

## Best Possible Solution

Design a Gateway lifecycle handoff that preserves current drain and update-restart semantics while keeping the old listener alive until a replacement has bound, authenticated, passed health checks, and can report failure back through an active channel.

## Reproduction Assessment

No live high-confidence managed-service reproduction was run in this read-only review. Source and tests show ordinary restarts close or signal the current instance before replacement health is established, while unhealthy restarts report diagnostics instead of retaining the previous instance.

## Solution Assessment

Unclear. The requested atomic-swap direction is reasonable, but the maintainable solution needs owner design around port binding, locks, supervisors, update restarts, and operator notification before implementation is safe.
