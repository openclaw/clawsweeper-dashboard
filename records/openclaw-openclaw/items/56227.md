## Summary

Keep open: current main still tears down the serving gateway before ordinary restart replacement health is established. The merged safe-restart coordinator helps defer active work, and update restarts have a specialized child-health fallback, but the requested old-instance-preserving handoff is still a lifecycle design gap.

## What This Changes

The issue requests a gateway restart handoff that keeps the existing instance reachable until a replacement instance starts and passes health checks.

## Best Possible Solution

Design a supervised and unmanaged gateway handoff that preserves active-work drain, validates replacement health, keeps service-manager semantics intact, and drops the old serving path only after verified replacement or clear failure notification.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows ordinary restarts close the listener before replacement health is proven, but user-visible proof still needs a managed-service failing-replacement scenario.

## Solution Assessment

Unclear. The atomic-swap direction is reasonable, but the maintainable fix needs lifecycle decisions around ports, locks, supervisors, update restarts, and notification before implementation is safe.
