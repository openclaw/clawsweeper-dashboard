## Summary

Keep open: current main still has safe restart deferral, diagnostics, and update-specific replacement health checks, but ordinary Gateway restarts do not keep the previous serving instance alive until a replacement is proven healthy.

## What This Changes

Request an atomic Gateway restart handoff that starts and verifies the replacement before stopping the existing instance.

## Best Possible Solution

Design a Gateway lifecycle handoff that preserves current drain, update, and supervisor semantics while keeping the old listener alive until a replacement has bound, authenticated, passed health checks, and can report failure through an active channel.

## Reproduction Assessment

No high-confidence live reproduction was run. Source and regression tests show the current ordinary restart path closes or restarts the serving process before replacement health is established, while unhealthy replacements are handled as diagnostics/failure.

## Solution Assessment

Unclear. The requested atomic-swap direction is reasonable, but the maintainable fix needs lifecycle ownership decisions around ports, locks, supervisors, update restarts, and operator notification before implementation is safe.
