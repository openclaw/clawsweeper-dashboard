## Summary

Keep open. Current main added safe restart preflight and update-respawn health checks, but the normal Gateway restart path still closes the active server before replacement startup is proven and has no old-listener rollback; the prior ClawSweeper keep-open review remains accurate.

## What This Changes

Gateway restart should preserve the currently running instance until a replacement is healthy, so a failed restart does not disconnect all channels and API access.

## Best Possible Solution

Design a supervisor-aware transactional restart model that validates before committing where possible, stages or probes the replacement Gateway, proves readiness, transfers endpoint/channel ownership, and only then stops the old instance or rolls back cleanly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a SIGUSR1/config restart reaches `server.close` before any replacement is proven healthy, and a later `params.start` failure leaves `server` null with diagnostics rather than restoring the old listener.

## Solution Assessment

Unclear. The no-disconnect target is sound, but the literal start-new-first sequence is not clearly the narrowest maintainable fix because same-port binding, lock ownership, service supervisors, and channel handoff all need design.
