## Summary

Keep open. Current `main` still closes the active Gateway before normal replacement startup is proven healthy, while the documented safe restart path only preflights active work rather than preserving the old listener on startup failure.

## What This Changes

Request a transactional Gateway restart that starts and health-checks a replacement before stopping the previous instance, preserving the old Gateway if startup fails.

## Best Possible Solution

Design a supervisor-aware transactional restart model that validates what can be validated before commit, proves replacement readiness, transfers endpoint/channel ownership, and rolls back cleanly when readiness fails.

## Reproduction Assessment

Yes. Source inspection on current `main` shows a SIGUSR1/config restart closes the active server before replacement health is proven, and a later startup failure leaves `server` null with diagnostics instead of restoring the old listener.

## Solution Assessment

Unclear. The no-disconnect target is maintainable, but the literal start-new-first sequence is not yet proven best because same-port binding, service supervisors, config commit timing, and channel handoff need design.
