## Summary

Keep open: the previous ClawSweeper review identified a real gap, and current main plus the latest release still provide restart deferral, post-restart health checks, and duplicate-starter protection rather than a previous-instance-preserving atomic swap.

## What This Changes

The issue requests an atomic gateway restart path that starts and health-checks a replacement before stopping the previous gateway instance.

## Best Possible Solution

Define a maintainer-approved gateway supervisor handoff contract, then implement restart continuity with explicit coverage for replacement startup failure and old-instance preservation.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes: current restart paths close or replace the active server and then verify health, while safe restart only defers SIGUSR1 scheduling. I did not run a live supervisor failure reproduction.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the supervisor contract; the atomic-swap target is reasonable, but the mechanism needs owner design across service managers, locks, ports, and user notification.
