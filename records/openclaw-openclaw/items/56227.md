## Summary

Keep open: current main has useful restart deferral, health checks, and update-recovery mitigations, but normal gateway restart paths still replace or exit the active instance before a replacement is proven healthy, so the requested previous-instance-preserving restart contract is not implemented.

## What This Changes

The issue asks for gateway restarts to keep the current working gateway alive until a replacement instance is started and health-verified.

## Best Possible Solution

Keep this open as the canonical product/design item until OpenClaw has a maintainer-approved restart contract that either atomically swaps a healthy replacement or explicitly documents a safer compatible alternative across supervisors.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does show the current restart paths still replace or exit the active gateway instead of proving a parallel replacement healthy first.

## Solution Assessment

No final implementation path is established yet. The atomic-swap idea targets the right availability problem, but the best fix needs maintainer design across supervisor ownership, port binding, locks, health checks, update handoff, and fallback notification.
