## Summary

Keep open: current main has safer restart drains, config preflight, health diagnostics, and an update-specific respawn fallback, but ordinary gateway restarts still do not keep the previous live Gateway serving until a replacement is health-checked.

## What This Changes

Request gateway restarts to start and health-check a replacement before stopping the existing Gateway, leaving the old instance available on failure.

## Best Possible Solution

Design a Gateway lifecycle handoff that preserves the existing safe-restart drain semantics while keeping the old listener alive until a replacement has bound, authenticated, and passed health checks, with a clear fallback and operator notification path.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source and tests show the current restart behavior: the old server is closed before ordinary replacement startup is proven, and unhealthy restarts fail with diagnostics.

## Solution Assessment

Unclear. The requested start-new-then-stop-old direction is reasonable, but the maintainable solution needs owner design around port binding, locks, supervisors, and how much overlap OpenClaw should support.
