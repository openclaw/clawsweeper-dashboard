## Summary

Keep open: current main and the latest release still use service-manager or SIGUSR1 restart handoff with health checks after the restart, not a previous-instance-preserving atomic swap. The request fits OpenClaw stability/setup goals, but it needs a maintainer-approved gateway supervisor contract before implementation.

## What This Changes

Request gateway restarts to keep the current gateway running until a replacement instance has started and passed health checks.

## Best Possible Solution

Keep this open until maintainers define and implement a restart contract that either atomically swaps to a healthy replacement or documents a compatible safer fallback across supported supervisors.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows restart requests schedule SIGUSR1, drain active work, close or replace the current server, and then prove health after the restart rather than proving a parallel replacement healthy first.

## Solution Assessment

Unclear until maintainers choose the restart contract. The atomic-swap idea targets the right availability problem, but the best fix needs maintainer design across supervisor ownership, port binding, locks, health checks, update handoff, and fallback notification.
