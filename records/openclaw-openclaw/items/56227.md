## Summary

Keep open: current main and v2026.6.10 still validate restart health only after the active gateway has already been closed, restarted, or signaled, so the requested old-instance-preserving handoff is not implemented.

## What This Changes

The issue requests an atomic gateway restart flow that starts and health-checks a replacement gateway before stopping the current instance, preserving connectivity on restart failure.

## Best Possible Solution

Define a maintainer-approved cross-platform gateway handoff contract, then implement restart continuity so the current listener remains available until replacement health succeeds or the restart is safely rejected.

## Reproduction Assessment

Yes at source level: managed and in-process restart paths close, restart, or signal the active gateway before replacement health is proven. I did not run a live supervisor repro.

## Solution Assessment

Unclear as an implementation-ready proposal: the atomic handoff goal is sound for availability, but the best fix needs a designed cross-platform supervisor, port, lock, rollback, and notification contract.
