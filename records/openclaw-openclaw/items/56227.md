## Summary

Keep open. Current main has active-work restart preflight and update-respawn health checks, but ordinary Gateway restart still closes the active listener before a replacement is proven healthy and has no rollback to the previous listener after startup failure.

## What This Changes

Request transactional Gateway restart behavior that keeps the previous Gateway instance reachable until the replacement is healthy.

## Best Possible Solution

Design a supervisor-aware transactional restart model that stages or validates the replacement, proves readiness, transfers listener and channel ownership, and rolls back or reports failure through the still-running gateway.

## Reproduction Assessment

Yes, at source level. A SIGUSR1/config restart reaches `server.close` before replacement readiness is proven, and a later replacement startup failure leaves `server` null with no previous listener to restore.

## Solution Assessment

Unclear. The no-disconnect goal is sound, but the literal start-new-first sequence is not obviously the narrowest maintainable fix because same-port binding, lock ownership, service supervisors, and channel handoff all need design.
