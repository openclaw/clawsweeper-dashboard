## Summary

The PR hardens installed plugin index record building so malformed stale contribution and startup metadata is ignored while valid metadata is preserved.

## Best Possible Solution

Make `buildContractContributionInfo` prototype-safe or filter blocked object keys, add blocked-key regression coverage, then land after focused installed-index validation.

No. The hardening direction is right, but the best fix needs the new contract map to use a prototype-safe representation or blocked-key filtering before this branch merges.
