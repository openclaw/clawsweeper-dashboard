Full review comments:

- **[P1] Recompute fallback due time at schedule gaps:** `src/infra/heartbeat-runner.ts:1152-1155`
  - body: When a boundary moves from a scheduled window into a gap, `resolveScheduleIntervalMs(...)` returns `null`, so this guard skips recalculating `nextDueMs` from the fallback base interval. A `2h` overnight window ending at `08:00` with fallback `every: "30m"` can keep the previous window's due time and wait until around `09:50` instead of running around `08:20`; recompute against `schedule interval ?? agent.intervalMs` so gaps honor the documented fallback.
  - confidence: 0.93
- **[P2] Reconcile DST boundary behavior with docs:** `docs/gateway/heartbeat.md:219`
  - body: The docs promise minute-accurate boundary transitions with at most about 60 seconds of delay, but the runtime comment says the fixed `24 * 60` minute arithmetic can be off by about 60 minutes on DST transition days. For users setting an IANA timezone, either compute boundary delays with timezone-aware instants or document and test the DST limitation in the user-facing heartbeat docs.
  - confidence: 0.81
