## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Honor fallback intervals after schedule gaps:** `src/infra/heartbeat-runner.ts:1151-1157`
  - body: When a boundary moves from a scheduled window into a gap, `resolveScheduleIntervalMs(...)` returns `null`, so this block skips recalculating `nextDueMs` from the fallback base interval. That can keep the prior window's due time, for example making an overnight `2h` window that ends at `08:00` wait until much later instead of using `every: "30m"` as documented.
  - confidence: 0.88
- **[P2] Compute boundary delays from timezone instants:** `src/infra/heartbeat-active-hours.ts:189-201`
  - body: `resolveNextWindowBoundaryMs` uses minute-of-day math with a fixed `24 * 60` day, so DST transition days can schedule boundary wakes about an hour early or late. Because this is a public scheduler feature, either compute the next local boundary as a real timezone instant or make the limitation an explicit maintainer-approved contract with focused tests.
  - confidence: 0.78
