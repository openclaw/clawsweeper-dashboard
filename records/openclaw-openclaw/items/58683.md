## Summary

Keep this PR open: the daypart heartbeat feature is not on current main, but the branch is stale/conflicting, still has source-visible scheduler defects, and lacks required real behavior proof for an external PR.

## What This Changes

The PR adds optional `heartbeat.schedule[]` time-of-day interval blocks across heartbeat config schema/types/help/docs, runner scheduling logic, and tests.

## Best Possible Solution

Implement daypart heartbeat intervals as a first-class core scheduler extension on top of the current phase-aligned runner, with schema/docs/tests and redacted real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced scheduler defects by source inspection: fallback gaps skip recalculation, and boundary wakes use fixed minute-of-day arithmetic. The requested feature itself is absent on current main, so there is no current-main user path to exercise.

## Solution Assessment

No. The capability likely belongs in core heartbeat scheduling, but this patch needs a rebase onto the current phase scheduler plus fixes for fallback gaps and DST boundary wakes before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor fallback intervals after schedule gaps:** `src/infra/heartbeat-runner.ts:1152-1155`
  - body: When a boundary moves from a scheduled window into an uncovered gap, `resolveScheduleIntervalMs(...)` returns `null`, so this guard leaves `nextDueMs` from the prior window. A `2h` overnight window ending at `08:00` with fallback `every: "30m"` can wait until the stale due time instead of recomputing from the fallback cadence; recalculate with `scheduleMs ?? agent.intervalMs`.
  - confidence: 0.92
- **[P2] Compute boundary delays from timezone instants:** `src/infra/heartbeat-active-hours.ts:190`
  - body: `resolveNextWindowBoundaryMs` uses fixed `24 * 60` minute arithmetic, so DST transition days can schedule a local boundary about an hour early or late. For a time-of-day feature, compute the next local boundary as a real epoch instant or narrow the documented guarantee and tests accordingly.
  - confidence: 0.8
