## What This Changes

Adds gateway active-run projection to `session.message` snapshots and tests that Control UI preserves the local WebChat run while plugin finalization delays terminal delivery.

## Best Possible Solution

Land the gateway-owned active-run projection only after contributor-supplied real WebChat/plugin-hook proof and normal maintainer review accept the event-order risk.

## Reproduction Assessment

No high-confidence real-environment reproduction was established. Source inspection and the focused tests show the race path: current main can apply a terminal-looking `session.message` without `hasActiveRun`, and the UI clears the local run when that nested value is absent or false.

## Solution Assessment

Yes, this looks like the best narrow fix: projecting the existing gateway active-run registry into `session.message` keeps the owner boundary in the gateway instead of adding a broader UI workaround. The remaining gap is real behavior proof, not an obvious code defect.
