## Summary

Keep open. Current main still preserves unsupported reminder-like language and appends the transparency note after reply payload construction, while the requested pre-send gate would add a new core policy/API behavior.

## What This Changes

The issue requests a pre-send enforcement gate that either removes unsupported future-commitment language or creates a concrete scheduled action before delivery.

## Best Possible Solution

Define an opt-in core finalization gate or typed hook contract that reuses the existing detector and cron evidence, with bounded revision behavior and regression tests for unsupported promises, successful cron adds, and existing same-session cron coverage.
