## Summary

Keep open. Current main still implements an append-only transparency note for unsupported reminder/follow-up language, while the requested pre-send forced-choice gate remains a product/API decision across reply finalization, scheduling evidence, and plugin hook contracts.

## What This Changes

The issue requests an opt-in pre-send gate for unsupported reminder or follow-up commitments instead of appending a post-hoc transparency note.

## Best Possible Solution

Implement a deliberate opt-in pre-send finalization gate that reuses the existing detector plus successfulCronAdds and same-session cron coverage, or first extend the public finalization hook contract with the scheduling evidence needed for a plugin to enforce the same policy.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists in the current runReplyAgent test path: a mocked reply of "I'll remind you tomorrow morning." with successfulCronAdds set to 0 returns the same promise plus the transparency note. I did not run tests because this review was explicitly read-only.

## Solution Assessment

No. The current append-only disclaimer is not the best solution for the requested trust contract because it still sends the unsupported promise; the narrow maintainable direction is an opt-in pre-send gate or an explicit hook/API contract that can make the same scheduling-backed decision before delivery.
