## Summary

Keep open. Current main still sends unsupported reminder/follow-up language and appends the transparency note afterward; the newer commitments path is opt-in background extraction and explicitly excludes exact reminders, so the requested pre-send gate remains unimplemented and needs maintainer product/API choice.

## What This Changes

The issue requests replacing the current post-reply unsupported-reminder disclaimer with a pre-send gate that forces the agent to remove unsupported commitment language or create a scheduled action.

## Best Possible Solution

Implement a deliberate opt-in pre-send finalization gate in the reply pipeline, or first extend the public finalization hook contract to expose scheduling evidence so a plugin can make the same decision before delivery.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists in the current runReplyAgent test path: a mocked reply of “I'll remind you tomorrow morning.” with successfulCronAdds set to 0 returns the same promise plus the transparency note. I did not run tests because this review was explicitly read-only.

## Solution Assessment

No. The current append-only disclaimer does not satisfy the requested trust contract because it still delivers the unsupported promise; the maintainable direction is an opt-in gate or a documented hook/API contract that can revise before delivery.
