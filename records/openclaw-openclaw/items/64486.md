## Summary

Keep open. Current main still sends unsupported reminder/follow-up language with an appended note, while inferred commitments are opt-in, post-reply background extraction and explicitly skip exact reminders; the requested pre-send enforcement remains unresolved.

## What This Changes

The issue requests an opt-in pre-send gate that either removes unsupported future-commitment wording or requires a real scheduled action before delivery.

## Best Possible Solution

Implement a deliberate opt-in pre-send finalization gate that reuses the existing detector plus successfulCronAdds and same-session cron coverage, with tests proving unsupported commitment text is revised or backed by a real schedule before delivery.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists in the current runReplyAgent tests: a mocked reply of "I'll remind you tomorrow morning." with successfulCronAdds set to 0 returns the same promise plus the transparency note; I did not run tests because this review was read-only.

## Solution Assessment

No. The current append-only transparency note is not the best solution for the requested trust contract because it still sends the unsupported promise; the maintainable path is an opt-in gate or a typed hook/API extension that can make the same decision before delivery.
