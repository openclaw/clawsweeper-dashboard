## Summary

Keep open. Current main still implements an append-only transparency note for unsupported reminder/follow-up language; it does not gate delivery, force a rewrite, or require creation of a backing schedule before the reply is returned.

## What This Changes

The issue requests a pre-send gate for agent replies that contain future-commitment language without a confirmed cron, task, or heartbeat-backed action.

## Best Possible Solution

Implement a deliberate opt-in pre-delivery finalization gate that reuses the existing detector and confirmed scheduling signals, or first expose those signals through a stable plugin hook contract if maintainers want this outside core.

## Reproduction Assessment

Yes. A static reproduction path exists in the current runReplyAgent test: a mocked reply of "I'll remind you tomorrow morning." with successfulCronAdds set to 0 returns that same text plus the transparency note. I did not run tests because this review was explicitly read-only.

## Solution Assessment

No. The current append-only note is useful transparency, but it is not the requested trust contract because the unsupported promise still ships; the maintainable fix needs an explicit pre-delivery gate or a plugin API that can make the same scheduling-backed decision.
