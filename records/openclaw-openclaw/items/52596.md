## Summary

Keep this open. Current main and v2026.6.8 still have the before-tool-call enforcement seam and durable session lifecycle primitives, but not the requested core PIN config, framework-owned authenticated session state, or trusted TypeScript verification path.

## What This Changes

Request adding a `session.auth.pin` configuration and framework-level session PIN gate that blocks tool calls until a trusted verification path authenticates the session.

## Best Possible Solution

Maintainers should decide whether OpenClaw wants a core session-auth primitive; if yes, implement it narrowly with schema/docs alignment, trusted verification outside model context, reset semantics, secret handling, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/security design request. Source inspection confirms the requested core PIN config, trusted verifier, and authenticated session state are absent on current main and v2026.6.8.

## Solution Assessment

Unclear until maintainers choose the security/product boundary. The before-tool-call hook is a plausible enforcement layer, but the requested PIN gate is a new config, secret, session-state, and verification contract.
