## Summary

Keep this open. Current main still has the before-tool-call enforcement seam and session lifecycle primitives, but it does not implement a core session PIN/auth config, trusted TypeScript verification path, or framework-owned authenticated session state; the remaining work needs maintainer security and product direction.

## What This Changes

The issue requests a core session PIN authentication gate that blocks tool calls at the before-tool-call layer until trusted framework code authenticates the session.

## Best Possible Solution

Maintainers should decide whether OpenClaw wants a core session-auth primitive; if yes, implement it narrowly with schema/docs alignment, trusted verification outside model context, reset semantics, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/security design request. Source inspection confirms the requested core PIN/auth config and authenticated session state are not present on current main.

## Solution Assessment

Unclear until maintainers choose the product boundary. The existing before-tool-call hook is the right enforcement seam, but a core PIN gate is a new security/config contract that should be designed deliberately.
