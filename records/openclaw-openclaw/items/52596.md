## Summary

Keep this open. Current main and v2026.6.8 still have the before-tool-call enforcement seam, plugin policy hooks, and session-reset/session-state primitives, but they do not implement a core session PIN config, framework-owned authenticated session state, or trusted TypeScript verification path; the remaining work needs maintainer security and product direction.

## What This Changes

Requested framework-level session PIN authentication that blocks tool calls until a trusted TypeScript verification path authenticates the current session.

## Best Possible Solution

Maintainers should decide whether OpenClaw wants a core session-auth primitive; if yes, implement it narrowly with schema/docs alignment, trusted verification outside model context, reset semantics, secret handling, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/security design request. Source inspection confirms the proposed core PIN/auth config, trusted verification route, and authenticated session state are absent on current main and v2026.6.8.

## Solution Assessment

Unclear until maintainers choose the security/product boundary. The existing before-tool-call hook is a plausible enforcement seam, but a core PIN gate is a new config, secret, session-state, and verification contract that should be designed deliberately.
