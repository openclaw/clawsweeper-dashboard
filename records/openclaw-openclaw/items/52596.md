## Summary

Keep open. Current main and v2026.6.10 still have the before-tool-call enforcement seam and session/reset primitives, but not the requested core PIN config, framework-owned authenticated session state, or trusted TypeScript PIN verification path.

## What This Changes

Requests a core session PIN authentication gate that blocks tool calls until a trusted TypeScript verifier marks the session authenticated.

## Best Possible Solution

Keep this as a security/product design item; if maintainers accept a core primitive, implement a narrow secret-safe verifier, authenticated session state cleared on new/reset, before-tool-call enforcement, docs, and regression coverage.

## Solution Assessment
