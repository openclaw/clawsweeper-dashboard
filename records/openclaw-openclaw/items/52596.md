## Summary

Keep open: this is a security-oriented feature request, not a stale, duplicate, or implemented cleanup item. Current main has pre-tool and pre-agent blocking primitives plus reset/session state surfaces, but no `session.auth.pin` config or TypeScript-side session PIN verifier; the remaining work creates a new auth/config surface and needs maintainer design.

## What This Changes

The issue requests a framework-level session PIN gate that blocks tools until a trusted TypeScript-side authentication step succeeds.

## Best Possible Solution

Define a maintainer-approved auth gate design that keeps secret comparison outside the model, stores only session-authenticated state, clears it on session rotation, and enforces the gate through the trusted pre-run/pre-tool path with focused tests and docs.
