## Summary

Keep open. Current main still has the useful pre-tool and session primitives, but it does not implement the requested framework-owned session PIN gate or trusted verifier, and the security/UX/config shape needs maintainer design before implementation.

## What This Changes

The issue requests a core `session.auth.pin` capability that blocks tool calls for each new or reset session until a trusted TypeScript-side PIN check authenticates the session.

## Best Possible Solution

Add a documented core session-auth capability with schema/config for the secret source, framework-owned authenticated state on `SessionEntry`, reset clearing on `/new` and `/reset`, pre-tool enforcement before ordinary hooks, and a trusted command/gateway/internal verifier that compares PIN attempts outside model context with redaction coverage.

## Reproduction Assessment

Not applicable as a conventional bug reproduction. The missing feature is high-confidence by source inspection: the proposed config key, session auth state, and trusted verifier are absent from current main.

## Solution Assessment

Unclear on exact UX and config shape, but the requested enforcement boundary is sound. A prompt-only ritual is not a trusted boundary, and an external-plugin-only workaround does not settle whether OpenClaw should expose this as a core security primitive.
