## Summary

Keep open: current main still has the pre-tool blocking and session/reset primitives, but it does not implement a `session.auth.pin` config surface or a trusted TypeScript-side session PIN verifier. This remains a security-sensitive feature/config request that needs maintainer design before code.

## What This Changes

Request a session-scoped PIN authentication gate that blocks tool execution at the framework layer until a trusted verifier authenticates the session.

## Best Possible Solution

Define a maintainer-approved auth gate that compares secrets outside the model, stores only session-authenticated state, clears it on session rotation, enforces through the trusted pre-tool path, and documents the config/secret contract.

Not applicable: this is a security/auth feature request, not a broken existing behavior report. Source inspection establishes the current-main gap by showing the requested config and trusted verifier are absent.

## Solution Assessment
