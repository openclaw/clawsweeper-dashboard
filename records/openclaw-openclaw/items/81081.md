Keep open. The PR appears to be the narrow correct code fix for the linked macOS dashboard bug, but the external PR still has only mocked Linux-hosted platform coverage and needs real macOS after-fix behavior proof before merge.

## What This Changes

The PR changes `resolveBrowserOpenCommand()` so Darwin skips the SSH/no-display short-circuit and adds focused browser-open regression tests for Darwin stale SSH envs and preserved Linux behavior.

## Best Possible Solution

Land this narrow resolver change after contributor-supplied real macOS behavior proof is added, while preserving the Linux SSH fallback and Windows launch hardening tests.

## Reproduction Assessment

Yes, from source inspection: on current main, Darwin with any `SSH_*` variable and no `DISPLAY` or `WAYLAND_DISPLAY` returns `ssh-no-display` before the macOS `open` branch. I did not run a live macOS shell in this read-only review.

## Solution Assessment

Yes. Excluding Darwin from the SSH/no-display short-circuit is the smallest maintainable fix for macOS, and the added tests cover the regression plus Linux preservation; merge should wait for real behavior proof.
