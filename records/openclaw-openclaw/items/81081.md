The PR changes `resolveBrowserOpenCommand()` so Darwin skips the SSH/no-display short-circuit and adds regression tests for the macOS stale-SSH case and Linux preservation.

## Best Possible Solution

## Solution Assessment

Yes. Excluding Darwin from the SSH/no-display short-circuit is the narrowest maintainable fix for the reported macOS behavior, and the added tests cover the regression plus Linux preservation; merge should wait for real macOS proof.
