## Summary

Keep open: the PR is a focused agents-runtime fix with credible real behavior proof, but the current diff still permits assistant text attached to a tool-call turn to be selected as terminal subagent output, leaving a central premature-completion case unresolved.

## What This Changes

The PR changes subagent lifecycle and announcement handling to classify unresolved tool-boundary endings as errors, preserve hosted-tool handoffs, and add regression coverage.

## Best Possible Solution

Keep the lifecycle hardening, but make output selection wait for a later assistant message without tool calls before treating child text as terminal completion.

## Reproduction Assessment

Yes, at source level: a chat history ending with an assistant message that contains both text and a non-`sessions_yield` tool call still returns that text from `readSubagentOutput` on the PR head. I did not execute the fixture because this review is read-only.

## Solution Assessment

No. The lifecycle-error and hosted-tool parts look directionally right, but the output-selection change should not return assistant text while the latest turn is still waiting for tool continuation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep tool-use progress text out of terminal output:** `src/agents/subagent-announce-output.ts:305-307`

That preserves the premature-completion path from the linked bug instead of waiting for a later assistant message without tool calls, so parent sessions can still announce in-progress text as a successful completion.
