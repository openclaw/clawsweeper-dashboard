## Summary

Keep open: the latest PR head still returns assistant text from a tool-call turn while `waitingForContinuation` is true, so the branch can still freeze progress text as terminal child output despite useful lifecycle hardening and real-world proof.

## What This Changes

The PR changes subagent lifecycle and announce-output handling around unresolved tool-boundary completions, with regression coverage for lifecycle errors, hosted client-tool handoff, and stale output suppression.

## Best Possible Solution

Keep the lifecycle-error and hosted-tool hardening, but make every assistant turn with tool calls non-terminal until a later assistant-only response arrives, then land with focused announce-output and lifecycle coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `chat.history` assistant message with text plus a non-`sessions_yield` tool call sets `waitingForContinuation` but is still returned from `latestAssistantText` on the PR head.

## Solution Assessment

No. The lifecycle and hosted-tool direction is right, but the proposed patch is incomplete until output selection stops returning text from assistant turns that are still at a tool boundary.
