## Summary

Keep open: the PR fixes important lifecycle pieces, but the current head still allows assistant text attached to a tool-call turn to be selected as terminal subagent output, preserving the premature-completion path tracked by the related open report.

## What This Changes

The PR changes agent/subagent lifecycle handling for unresolved tool-call boundaries, carries hosted client-tool metadata, adjusts subagent completion output selection, and adds regression tests.

## Best Possible Solution

Keep the lifecycle-error hardening and hosted-tool distinction, but make output selection treat every assistant turn with tool calls as non-terminal progress until a later assistant message without tool calls arrives.

## Reproduction Assessment

Yes. Source inspection on the PR head gives a high-confidence path: a `chat.history` assistant message with text plus a non-`sessions_yield` tool call leaves `waitingForContinuation` true while still returning that assistant text as selected output.

## Solution Assessment

No. The lifecycle hardening and hosted-tool handling are useful, but the patch is incomplete until the output selector stops returning text from assistant turns that are still at a tool boundary.
