## Summary

Needs changes before merge: the latest PR head still preserves the premature completion-output path for assistant messages that contain both text and tool calls, so it can still freeze progress text as a successful subagent result.

## What This Changes

The PR changes agent lifecycle and subagent announcement handling to classify unresolved tool-boundary endings as errors, preserve hosted tool handoffs, defer empty completion announcements, and add regression coverage.

## Best Possible Solution

Keep the lifecycle-error hardening and hosted-tool exception, but make output selection suppress all assistant text from tool-call turns until a later assistant message without tool calls provides the final result.

## Reproduction Assessment

Yes, at source level: current main and the latest PR head both expose a fixture/path where assistant text on a non-`sessions_yield` tool-use turn can be selected as completion output. I did not execute a live reproduction in this read-only review.

## Solution Assessment

No. The lifecycle hardening is useful, but the output-selection fix is incomplete because it still returns assistant text while `waitingForContinuation` is true.

## Review Findings

Overall correctness: patch is incorrect
