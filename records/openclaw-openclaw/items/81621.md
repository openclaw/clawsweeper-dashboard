## Summary

Keep open: the PR improves several lifecycle paths, but it still selects assistant text from a non-`sessions_yield` tool-use turn as terminal completion output, which preserves a central premature-completion failure mode from the related report.

## What This Changes

The PR hardens subagent completion handling around unresolved tool-call boundaries and adjusts announcement output selection to avoid stale tool/raw text.

## Best Possible Solution

Keep the lifecycle-error hardening, but make output selection wait for a later assistant message without tool calls before freezing or announcing successful completion text.

## Reproduction Assessment

Yes. The PR head itself contains a source-level fixture that returns assistant text from a non-`sessions_yield` tool-use turn as completion output, matching the related premature-completion path.

## Solution Assessment

No. The lifecycle hardening is useful, but the proposed output-selection path is incomplete because it still treats assistant text on a tool-use turn as terminal completion text.
