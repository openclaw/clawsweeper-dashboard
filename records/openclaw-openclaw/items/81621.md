## Summary

Keep open: this is a focused P1 subagent completion fix with useful live proof, but the current branch still has a source-level correctness blocker where assistant progress text from a tool-use turn can be frozen as terminal child output.

## What This Changes

The branch changes agent/subagent lifecycle handling for unresolved tool-boundary terminal states, adds hosted client-tool handoff metadata, adjusts subagent completion output selection, and adds regression tests.

## Best Possible Solution

Keep the lifecycle-error and hosted-tool hardening, but make output selection wait for a later assistant message without tool calls before freezing child completion text.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `chat.history` assistant message with text plus a non-`sessions_yield` tool call leaves `waitingForContinuation` true but is still returned from `latestAssistantText`.

## Solution Assessment

No. The direction is maintainable, but the branch is incomplete until assistant turns with tool calls are treated as non-terminal progress for output selection.
