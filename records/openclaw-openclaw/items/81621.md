## Summary

Keep open: the PR is a focused fix for a real agents/subagent completion bug, but the current head still has a source-reproducible path that can freeze assistant text from an unfinished tool-use turn as terminal output.

## What This Changes

The PR changes agent/subagent lifecycle handling so unresolved tool-boundary endings become errors, adjusts completion output selection around tool-use and yield turns, preserves hosted client-tool handoffs, and adds regression tests.

## Best Possible Solution

Keep the lifecycle-error hardening and hosted-tool distinction, but make output selection treat every assistant turn with tool calls as non-terminal until a later assistant message without tool calls arrives.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a `chat.history` assistant message with text plus a non-`sessions_yield` tool call leaves `waitingForContinuation` true while `selectSubagentOutputText` still returns that assistant text.

## Solution Assessment

No. The lifecycle hardening and hosted-client-tool preservation are good, but the PR is incomplete until the output selector stops returning assistant text from turns that are still at a tool boundary.
