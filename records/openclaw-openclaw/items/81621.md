## Summary

Keep open: this is a focused agents bug-fix PR, but the current head still preserves a false-completion path for assistant text attached to a tool-call turn, and the supplied real proof is from a hotpatched older install rather than the current PR head on latest main.

## What This Changes

The branch changes agent/subagent lifecycle handling and announcement output selection so unresolved tool-boundary completions become lifecycle errors and stale wait/tool output is not announced, with targeted regression tests.

## Best Possible Solution

Keep the lifecycle error classification, remove the waiting-state assistant-text fallback, add a text-plus-tool-call regression test, and land only after redacted latest-main Telegram proof.

## Reproduction Assessment

Yes, source-level: PR head can still select assistant text from a turn that also has tool calls while `waitingForContinuation` is true. Current main's incomplete-turn contract treats terminal tool-use pre-tool text as incomplete rather than final output.

## Solution Assessment

No; the lifecycle classification direction is narrow and appropriate, but the output-selection branch keeps part of the false-completion behavior alive. The safer fix is to make waiting-for-continuation a hard stop for terminal output selection unless a later non-tool assistant answer arrives.
