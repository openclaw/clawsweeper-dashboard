## Summary

Keep open: the PR is useful but not merge-ready because PR head still allows pre-tool assistant text to be selected as terminal completion output and broadly classifies legitimate hosted client-tool continuations as lifecycle errors; the copied Telegram proof does not cover those current-head paths.

## What This Changes

The PR changes agent/subagent lifecycle handling and completion-announcement output selection around unresolved tool boundaries, yield waits, stale output, and empty success announcements, with focused regression tests.

## Best Possible Solution

Narrow the fix so truly abandoned internal tool-boundary subagent turns become errors, hosted client-tool handoffs remain continuations, and terminal output selection waits for a later non-tool assistant response before announcing success.

## Reproduction Assessment

Yes, source-level: PR head stores latestAssistantText for an assistant turn that can also contain tool calls, then returns it while waitingForContinuation is true. The hosted-tool lifecycle concern is also source-visible through run.ts producing tool_calls/pendingToolCalls and openresponses-http consuming them as function_call continuations.

## Solution Assessment

No; the lifecycle error direction is useful for abandoned internal tool turns, but this patch is not yet the narrowest maintainable solution. It should distinguish hosted client-tool handoff from unresolved internal tool failure and should block pre-tool text until a later non-tool assistant answer arrives.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not return tool-turn text while waiting:** `src/agents/subagent-announce-output.ts:304-306`
  - body: When an assistant message contains both text and a tool call, the summarizer marks the snapshot as waiting but still stores that text, and this branch returns it as completed output. That keeps the false-completion path alive; wait for a later assistant turn without tool calls before selecting terminal output.
  - confidence: 0.88
- **[P2] Preserve hosted client-tool turns as continuations:** `src/agents/agent-command.ts:1131-1132`
  - body: Current main uses `stopReason: "tool_calls"` with `pendingToolCalls` for OpenResponses hosted client tools, and the responses gateway converts that into function_call output. Treating every pending/tool_calls result as lifecycle error changes a valid continuation boundary into failure for lifecycle subscribers, so distinguish hosted client-tool handoff from abandoned internal tool turns.
  - confidence: 0.78
