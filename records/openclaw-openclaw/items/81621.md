## Summary

Keep open: the patch still preserves the premature-completion path for assistant text attached to a tool-use turn, so it should not merge as-is despite useful lifecycle hardening and sufficient real behavior proof.

## What This Changes

The PR changes agents lifecycle and subagent announce handling to classify unresolved tool-boundary endings, preserve hosted tool handoffs, defer empty completion announces, and add regression coverage.

## Best Possible Solution

Keep the lifecycle-error hardening, but make output selection wait for a later assistant message without tool calls before freezing or announcing successful completion text.

## Reproduction Assessment

Yes. The PR head and current main both expose a source-level fixture/path where assistant text on a non-`sessions_yield` tool-use turn can be treated as completion output.

## Solution Assessment

No. The lifecycle hardening is useful, but the proposed output-selection path is incomplete because it still returns assistant text while waiting for tool continuation.
