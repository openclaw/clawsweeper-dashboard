## Summary

Keep open: the PR is a useful, focused hardening pass, but the current head still preserves the premature completion-output path that the earlier review flagged. A child assistant turn with both text and a tool call can still be selected as terminal subagent output while the snapshot is explicitly waiting for continuation.

## What This Changes

The PR changes agent/subagent lifecycle handling to emit errors for unresolved tool-call terminal states, preserve hosted client-tool handoffs, adjust subagent announcement output selection, and add focused regression tests.

## Best Possible Solution

Keep the lifecycle-error hardening, but make output selection treat every assistant turn with tool calls as non-terminal progress until a later assistant message without tool calls arrives.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a `chat.history` assistant message with text plus a non-`sessions_yield` tool call leaves `waitingForContinuation` true while still returning that assistant text as selected output.

## Solution Assessment

No. The lifecycle hardening and hosted-tool distinction are good, but the PR is incomplete until the output selector stops returning text from assistant turns that are still at a tool boundary.
