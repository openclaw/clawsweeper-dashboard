## Summary

Keep open: current main still has the reported IRC failures, and this PR contains useful fixes, but it is not merge-ready because it introduces a QUIT delivery regression, leaves delayed reclaim timers outside client lifetime, and lacks inspectable real behavior proof.

## What This Changes

The PR updates the bundled IRC plugin client, transient send path, and NickServ tests to surface failed sends, add a transient drain delay, and improve fallback-nick NickServ reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that preserves the useful send/NickServ fixes while preserving QUIT delivery, tying delayed reclaim work to client shutdown, avoiding duplicate reclaim attempts, and adding focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main still uses password-only NickServ IDENTIFY, silently returns from sanitized-empty sendPrivmsg, and immediately quits transient sends; I did not perform a live IRC run in this read-only review.

## Solution Assessment

No. The repair direction is narrow and appropriate, but this patch should not merge until it preserves QUIT delivery, bounds reclaim timer lifetime, avoids duplicate reclaim attempts, and provides inspectable real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new sendRaw guard rejects whenever closed is true, but quit() sets closed = true before it calls sendRaw("QUIT ..."). Every graceful shutdown now throws inside quit(), swallows the error, and closes the socket without sending a QUIT frame; move the closed transition until after QUIT is written or bypass the guard for shutdown.
  - confidence: 0.94
- **[P2] Cancel delayed NickServ reclaim timers:** `extensions/irc/src/client.ts:343-358`
  - body: The delayed NickServ GHOST/NICK setTimeout calls are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after quit() and can fire against a client that has already been closed, so track these timers and clear or unref them during shutdown.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:346-370`
  - body: tryRecoverNickCollision() can already send GHOST/NICK during collision recovery. If fallback login still reaches welcome, this delayed block can send a second reclaim sequence for the same desired nick, so gate it on whether recovery was already attempted.
  - confidence: 0.74
