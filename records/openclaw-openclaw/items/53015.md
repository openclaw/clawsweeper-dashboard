## Summary

Keep open. Current main still has the IRC reliability gaps this PR is trying to fix, but the PR is not mergeable as-is because the new closed-send guard breaks graceful QUIT delivery, the delayed NickServ reclaim timers are not tied to client lifetime, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR changes the bundled IRC plugin client, transient send path, and NickServ tests to surface empty/closed outbound send failures, delay transient shutdown, use account-qualified NickServ IDENTIFY for fallback nicks, and schedule post-identify GHOST/NICK reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful failure visibility, dead-socket detection, transient drain delay, account-qualified NickServ identify, and fallback-nick reclaim behavior while preserving graceful QUIT delivery, binding delayed reclaim work to client lifetime, avoiding duplicate reclaim attempts, and documenting the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The code-level reproduction is high confidence: current main shows password-only IDENTIFY, silent sanitized-empty `sendPrivmsg`, and immediate transient quit; the PR diff then makes `quit()` call `sendRaw` after `closed` is true and schedules untracked reclaim timers.

## Solution Assessment

No. The requested direction is sound, but this PR is not the best form as-is because it regresses graceful QUIT, lets delayed reclaim timers outlive the client, can duplicate NickServ reclaim attempts, and lacks the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new `sendRaw` guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before it sends `QUIT` through `sendRaw`. Every graceful shutdown now swallows that error and closes the socket without sending the QUIT frame, so move the closed transition until after QUIT is written or bypass this guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after `quit()` and can fire against a client that has already been closed, so tie them to client lifetime.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed reclaim path can run after `tryRecoverNickCollision()` already attempted GHOST/NICK during collision recovery. Gate the post-identify path on that attempt state or otherwise prevent a second noisy reclaim attempt for the same desired nick.
  - confidence: 0.73
- **[P3] Add the required changelog entry:** `extensions/irc/src/client.ts:221`
  - body: This is a user-facing IRC fix, but the PR changes only IRC code/tests and does not add a `CHANGELOG.md` entry. Repo policy requires an Unreleased Fixes entry with contributor credit so release notes cover the behavior change.
  - confidence: 0.88
