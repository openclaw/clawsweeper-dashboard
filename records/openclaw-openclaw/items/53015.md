## Summary

Keep open: current main still lacks the requested IRC reliability and NickServ fallback-nick behavior, but this PR cannot merge as-is because it introduces QUIT/timer regressions and lacks inspectable real behavior proof.

## What This Changes

The PR changes the bundled IRC client, transient send path, and NickServ tests to surface empty/closed sends, delay transient shutdown, use account-qualified IDENTIFY, and reclaim fallback nicks.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful send/NickServ fixes while preserving QUIT delivery, tying delayed reclaim timers to client lifetime, avoiding duplicate reclaim attempts, and adding focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main uses password-only NickServ IDENTIFY, silently returns on sanitized-empty sendPrivmsg, and immediately quits transient sends. I did not perform a live IRC run in this read-only review.

## Solution Assessment

No. The direction is sound, but this branch should not merge until QUIT ordering, reclaim timer lifecycle, duplicate reclaim behavior, conflicts, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:166-168`
  - body: The new sendRaw guard rejects whenever closed is true, but quit() sets closed = true before calling sendRaw("QUIT ..."). Every graceful shutdown now swallows the error and closes the socket without sending a QUIT frame, so move the closed transition until after QUIT is written or bypass this guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed NickServ GHOST/NICK setTimeout calls are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after quit() and can fire against a client that has already been closed, so track these timers and clear or unref them as part of quit()/close().
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:338-343`
  - body: tryRecoverNickCollision() can already send GHOST/NICK during collision recovery. If fallback login still reaches welcome, this delayed block can send a second reclaim sequence for the same desired nick; gate it on whether recovery was already attempted.
  - confidence: 0.74
