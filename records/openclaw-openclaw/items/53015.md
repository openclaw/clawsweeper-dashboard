## Summary

Keep open: current main and the latest checked release still have the reported IRC behavior, and this PR contains useful fixes, but the head cannot merge as-is because it regresses QUIT delivery, leaves delayed NickServ timers unmanaged, lacks inspectable real IRC proof, and is currently conflicting.

## What This Changes

The PR changes bundled IRC client/send code and tests to throw on empty or closed sends, delay transient shutdown, use account-qualified NickServ IDENTIFY, and reclaim fallback nicks with delayed GHOST/NICK.

## Best Possible Solution

Merge a repaired IRC plugin fix that preserves the useful send failure visibility, closed-socket detection, transient drain delay, account-qualified NickServ IDENTIFY, and fallback-nick reclaim behavior while fixing shutdown ordering, timer lifetime, duplicate recovery, and focused coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still has password-only IDENTIFY, sanitized-empty send drop, and immediate transient quit paths; the PR head blockers are visible in the changed client code, but I did not run a live IRC server.

## Solution Assessment

No. The repair direction is narrow and useful, but this patch should not merge until it preserves QUIT delivery, ties delayed reclaim timers to client lifetime, avoids duplicate reclaim attempts, and provides inspectable real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before calling `sendRaw("QUIT ...")`. Every graceful shutdown now throws inside `quit()`, swallows the error, and closes the socket without sending a QUIT frame; move the closed transition until after QUIT is written or bypass the guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or `unref()`'d. Once scheduled, they can keep one-shot IRC sends alive after `quit()` and can fire against a client that has already been closed; track these timers and clear or unref them during shutdown.
  - confidence: 0.87
- **[P3] Skip duplicate reclaim after collision recovery:** `extensions/irc/src/client.ts:346-369`
  - body: `tryRecoverNickCollision()` can already send GHOST/NICK during collision handling. If fallback login still reaches welcome, this delayed block can send a second reclaim sequence for the same desired nick; gate it on whether recovery was already attempted.
  - confidence: 0.74
