## Summary

Keep open: current main still has the IRC failures this PR targets, but the PR head is not safe to merge because it introduces a graceful QUIT regression, leaves delayed reclaim timers outside client shutdown, can duplicate NickServ reclaim attempts, and lacks inspectable after-fix proof.

## What This Changes

The PR updates bundled IRC client/send code and NickServ tests to surface failed sends, delay transient disconnects, identify fallback nicks with account-qualified NickServ, and reclaim desired nicks.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful reliability fixes while preserving QUIT delivery, binding delayed reclaim timers to client lifetime, avoiding duplicate reclaim attempts, adding focused tests, and including redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main builds password-only NickServ IDENTIFY, returns from sanitized-empty `sendPrivmsg`, and quits transient sends immediately. I did not perform a live IRC run in this read-only sweep.

## Solution Assessment

No. The direction is sound, but this branch is not the best merge form until QUIT ordering, reclaim timer lifecycle, duplicate reclaim behavior, conflicts, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:166-168`
  - body: The new guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before sending QUIT through `sendRaw`. Every graceful shutdown now swallows the error and closes without a QUIT frame, so move the closed transition until after QUIT is written or bypass this guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed GHOST/NICK timers are not stored, cleared, or unref'd. They can keep one-shot IRC sends alive after `quit()` and can fire against a closed client, so track them and clear or unref them during shutdown.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:338-343`
  - body: `tryRecoverNickCollision()` can already send GHOST/NICK during collision recovery. If fallback login still reaches welcome, this delayed block can send a second reclaim sequence for the same desired nick; gate it on whether recovery was already attempted.
  - confidence: 0.74
