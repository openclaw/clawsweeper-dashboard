## Summary

Keep open: current main still has the IRC NickServ and silent-send failure paths, but this PR is not mergeable because its closed-socket guard suppresses QUIT, delayed reclaim timers can outlive shutdown, and contributor real behavior proof is missing.

## What This Changes

This PR changes the bundled IRC client/send path and NickServ tests to surface empty/closed send failures, delay transient shutdown, use account-qualified IDENTIFY, and schedule fallback-nick reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful empty-send visibility, closed-socket detection, transient drain delay, account-qualified NickServ IDENTIFY, and fallback-nick reclaim behavior while fixing shutdown ordering, timer lifetime, duplicate recovery, and focused coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still has the password-only IDENTIFY, sanitized-empty send drop, and immediate transient quit paths, and the PR head blockers are visible in the changed client code.

## Solution Assessment

No. The repair direction is narrow and useful, but this patch should not merge until it preserves QUIT delivery, ties delayed reclaim timers to client lifetime, avoids duplicate reclaim attempts, and provides inspectable real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before calling `sendRaw("QUIT ...")`. Every graceful shutdown now throws inside `quit()`, swallows the error, and closes the socket without sending a QUIT frame; move the closed transition until after QUIT is written or bypass the guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or `unref()`'d. Once scheduled, they can keep a one-shot IRC send alive after `quit()` and can fire against a closed client; track these timers and clear or unref them during shutdown.
  - confidence: 0.87
- **[P3] Skip duplicate reclaim after collision recovery:** `extensions/irc/src/client.ts:346-369`
  - body: `tryRecoverNickCollision()` can already send GHOST/NICK during collision handling. If fallback login still reaches welcome, this delayed block can send a second reclaim sequence for the same desired nick; gate it on whether recovery was already attempted.
  - confidence: 0.74
