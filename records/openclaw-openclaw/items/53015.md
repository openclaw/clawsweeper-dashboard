## Summary

Keep open: current main still lacks the requested IRC fixes, but this PR head has blocking shutdown/timer regressions and no inspectable real behavior proof, so it should not be closed or merged as-is.

## What This Changes

The PR changes the bundled IRC plugin client, transient send path, and NickServ tests to throw on empty/closed sends, delay transient shutdown, use account-qualified NickServ IDENTIFY, and schedule fallback-nick reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that preserves the useful empty-send visibility, closed-socket detection, transient drain delay, account-qualified NickServ IDENTIFY, and fallback-nick reclaim behavior while fixing shutdown ordering, timer lifetime, duplicate reclaim attempts, and real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main still has the reported password-only IDENTIFY, sanitized-empty send drop, and immediate transient quit paths, and the PR head blockers are visible in the changed client code.

## Solution Assessment

No. The repair direction is narrow and useful, but this patch should not merge until it preserves QUIT delivery, ties delayed reclaim timers to client lifetime, avoids duplicate reclaim attempts, and provides inspectable real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before calling `sendRaw("QUIT ...")`. Every graceful shutdown now throws inside `quit()`, swallows the error, and closes the socket without sending a QUIT frame; move the closed transition until after QUIT is written or bypass the guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client lifetime:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or `unref()`'d. Once scheduled, they can keep a one-shot IRC send alive after `quit()` and can fire against a closed client; track these timers and clear or unref them during shutdown.
  - confidence: 0.87
- **[P3] Skip delayed reclaim after collision recovery:** `extensions/irc/src/client.ts:346-370`
  - body: `tryRecoverNickCollision()` can already send GHOST/NICK during collision handling. If fallback login still reaches welcome, this delayed block can send a second reclaim sequence for the same desired nick; gate it on whether recovery was already attempted.
  - confidence: 0.74
