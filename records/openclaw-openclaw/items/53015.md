## Summary

Keep open: current main still shows the IRC send/NickServ problems by source inspection, but this PR is not mergeable as-is because it introduces graceful-QUIT and delayed-timer regressions and lacks inspectable real behavior proof.

## What This Changes

The PR updates the bundled IRC client, transient send path, and NickServ tests to throw on empty/closed sends, delay transient shutdown, use account-qualified IDENTIFY, and reclaim fallback nicks.

## Best Possible Solution

Merge a repaired IRC plugin change that preserves the useful send/NickServ fixes while preserving QUIT delivery, tying reclaim timers to client lifetime, avoiding duplicate reclaim attempts, and adding focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main uses password-only NickServ IDENTIFY, returns silently on sanitized-empty sendPrivmsg, and immediately quits transient sends. I did not perform a live IRC run in this read-only review.

## Solution Assessment

No. The direction is sound, but the branch should not merge until QUIT ordering, reclaim timer lifecycle, duplicate reclaim behavior, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new sendRaw guard rejects whenever closed is true, but quit() sets closed = true before it calls sendRaw("QUIT ..."). Graceful shutdown now swallows the error and closes without writing QUIT, so move the closed transition until after QUIT is written or bypass this guard for the shutdown frame.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ reclaim setTimeout calls are not stored, cleared, or unref'd. Once scheduled, they can keep transient sends alive after quit() and can fire against a closed client, so track these timers and clear or unref them as part of quit()/close().
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:346-350`
  - body: tryRecoverNickCollision() can already send GHOST/NICK during collision recovery. If fallback login still reaches welcome after that attempt, this delayed block can send a second reclaim sequence, so gate it on whether recovery was already attempted.
  - confidence: 0.74
