- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-176`
  - body: The new guard rejects `sendRaw` after `quit()` has set `closed = true`, so every graceful shutdown swallows the QUIT write and only ends the socket. Move the closed transition until after QUIT is written, or use a dedicated shutdown write path.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK `setTimeout` calls are never stored, cleared, or unref'd. A fallback-nick connection can keep one-shot sends alive after `quit()` and later fire against a closed client; track and clear or unref these timers with client shutdown.
  - confidence: 0.87
- **[P2] Only account-qualify IDENTIFY on fallback nick:** `extensions/irc/src/client.ts:338`
  - body: `connectIrcClient` now passes `desiredNick` into `buildIrcNickServCommands` for every connection, so normal NickServ logins change from password-only IDENTIFY to account-qualified IDENTIFY even when no fallback nick was used. The bug only needs the account argument for fallback nick; preserve the old form for normal connects or provide explicit compatibility proof.
  - confidence: 0.82
