## Summary

Keep open: current main still has the IRC send/NickServ gaps, but this PR introduces clear shutdown and timer-lifetime regressions, so it should be repaired before merge rather than closed or merged as-is.

## What This Changes

The PR updates the bundled IRC plugin client, transient send path, and NickServ tests to surface empty/closed sends, delay transient shutdown, identify fallback nicks with an account nick, and schedule post-identify nick reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that preserves the useful reliability fixes while keeping graceful QUIT delivery, binding delayed reclaim work to client lifetime, preventing duplicate reclaim attempts, and adding focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current main still emits password-only NickServ IDENTIFY, returns from sanitized-empty `sendPrivmsg`, and transient sends quit immediately; the PR diff then makes `quit()` trip the new closed-send guard and schedules untracked reclaim timers.

## Solution Assessment

No. The direction is sound, but this branch is not the best form until QUIT ordering, reclaim timer lifecycle, duplicate reclaim behavior, mergeability, and changelog coverage are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before calling `sendRaw("QUIT ...")`. Every graceful shutdown now swallows that error and closes the socket without sending the QUIT frame, so move the closed transition until after QUIT is written or bypass this guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after `quit()` and can fire against a client that has already been closed, so track these timers and clear or unref them during shutdown.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:346-354`
  - body: `tryRecoverNickCollision()` can already send GHOST/NICK during collision recovery. If that path has already run and the fallback nick still reaches welcome, this delayed block sends a second GHOST/NICK sequence for the same desired nick; gate it on the earlier recovery state.
  - confidence: 0.74
- **[P3] Add the required changelog entry:** `extensions/irc/src/client.ts:224`
  - body: This is a user-facing IRC reliability fix, but the PR only changes IRC code and tests. Add an Unreleased Fixes entry in `CHANGELOG.md` so release notes cover the behavior change.
  - confidence: 0.88
