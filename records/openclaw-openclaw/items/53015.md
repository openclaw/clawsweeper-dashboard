## Summary

Keep open. The PR addresses real IRC reliability gaps, but it is not mergeable as-is because the new closed-send guard drops graceful QUIT frames and the delayed NickServ reclaim timers are not tied to client lifetime.

## What This Changes

The PR updates the bundled IRC plugin client, transient send path, and NickServ tests to surface empty/closed outbound sends, delay transient shutdown, identify with an account nick, and reclaim fallback nicks.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful reliability fixes while preserving graceful QUIT delivery, binding delayed reclaim work to client shutdown, preventing duplicate reclaim attempts, and documenting the user-facing fix.

## Reproduction Assessment

Yes. The code-level reproduction is high confidence: current main lacks the requested behavior, and applying the PR diff makes `quit()` call `sendRaw` after `closed` is already true while also scheduling untracked reclaim timers.

## Solution Assessment

No. The direction is sound, but this branch is not the best form until the QUIT ordering, reclaim timer lifetime, duplicate reclaim behavior, and changelog coverage are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve QUIT before rejecting closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new `sendRaw` guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before sending `QUIT` through `sendRaw`. Every graceful shutdown now swallows that error and closes the socket without sending the QUIT frame, so move the closed transition until after QUIT is written or bypass this guard for shutdown.
  - confidence: 0.94
- **[P2] Cancel delayed NickServ reclaim timers on shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed GHOST/NICK `setTimeout` calls are not stored, cleared, or unref'd. Once scheduled, they can keep a one-shot IRC send alive after `quit()` and can fire against a client that has already been closed, so tie these timers to client lifetime.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:343-360`
  - body: `tryRecoverNickCollision()` can already send GHOST/NICK during collision recovery. If that path has already run and the fallback nick still reaches welcome, this delayed post-identify block sends a second GHOST/NICK sequence for the same desired nick, so gate it on the earlier recovery attempt state.
  - confidence: 0.74
- **[P3] Add the required changelog entry:** `extensions/irc/src/client.ts:221`
  - body: This is a user-facing IRC reliability fix, but the PR only changes IRC code and tests. Add an Unreleased Fixes entry in `CHANGELOG.md` so the release notes cover the behavior change.
  - confidence: 0.88
