## Summary

This PR should stay open: current main still has the IRC send/NickServ gaps the branch targets, but the branch introduces blocking QUIT/timer regressions and still lacks inspectable real-behavior proof from the contributor.

## What This Changes

The PR updates the bundled IRC client, transient send path, and NickServ tests to throw on empty/closed sends, delay transient shutdown, identify with an account nick, and reclaim fallback nicks with delayed GHOST/NICK.

## Best Possible Solution

Merge a repaired IRC plugin change that preserves the useful empty-send visibility, closed-socket detection, transient drain delay, account-qualified IDENTIFY, and fallback-nick reclaim behavior while fixing shutdown ordering, timer lifetime, and focused coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still has password-only NickServ IDENTIFY, sanitized-empty send drop, and immediate transient quit paths, and the PR-head regressions are visible in the changed client code.

## Solution Assessment

No. The direction is narrow and useful, but this patch should not merge until it preserves QUIT delivery, ties delayed reclaim timers to client lifetime, and supplies inspectable real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:166-168`
  - body: The new sendRaw guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before it calls `sendRaw("QUIT ...")`. Every graceful shutdown now throws inside `quit()`, swallows the error, and closes the socket without sending a QUIT frame; move the closed transition until after QUIT is written or bypass the guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or `unref()`'d. Once scheduled, they can keep a one-shot IRC send alive after `quit()` and can fire against a closed client; track these timers and clear or unref them during shutdown.
  - confidence: 0.87
