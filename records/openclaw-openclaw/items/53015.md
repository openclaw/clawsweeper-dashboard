## Summary

Keep open: current main still has the reported IRC failure modes and this PR contains useful fixes, but the branch is not merge-ready because it introduces a QUIT delivery regression, leaves delayed reclaim timers outside client lifetime, and lacks inspectable real behavior proof.

## What This Changes

The PR modifies bundled IRC client/send code and NickServ tests to throw on empty/closed sends, delay transient disconnects, use account-qualified NickServ IDENTIFY, and schedule fallback-nick reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful send/NickServ fixes while preserving QUIT delivery, tying delayed reclaim timers to client shutdown, avoiding duplicate reclaim attempts, and adding focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main uses password-only NickServ IDENTIFY, silently returns from sanitized-empty sendPrivmsg, and immediately quits transient sends. I did not perform a live IRC run in this read-only review.

## Solution Assessment

No. The direction is sound, but this branch should not merge until QUIT ordering, delayed timer lifetime, duplicate reclaim behavior, rebase preservation, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new sendRaw guard rejects whenever closed is true, but quit() sets closed = true before it sends QUIT through sendRaw. Every graceful shutdown now swallows that error and closes the socket without sending a QUIT frame, so move the closed transition until after QUIT is written or bypass this guard for shutdown.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK setTimeout calls are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after quit() and can fire against a client that has already been closed, so track these timers and clear or unref them as part of shutdown.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:346-350`
  - body: tryRecoverNickCollision() can already send GHOST/NICK during collision recovery. If fallback login still reaches welcome, this delayed block can send a second reclaim sequence for the same desired nick; gate it on whether recovery was already attempted.
  - confidence: 0.74
