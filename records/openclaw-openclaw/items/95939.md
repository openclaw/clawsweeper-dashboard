## Summary

Current OpenClaw no longer persists startup-derived runtime defaults over user config, and the app/local fallback writer now preserves unrelated existing config keys, so the reported 2026.4.23 restart overwrite behavior is obsolete in current main and shipped releases.

## What This Changes

The issue reports that desktop startup rewrites existing `agents.defaults` values in `openclaw.runtime.json` instead of preserving customized settings.

## Best Possible Solution

Keep the shipped runtime-only startup behavior and protected config writer; users still seeing this on 2026.4.23 should upgrade to a current release rather than adding another compatibility path.

## Reproduction Assessment

No; the reported 2026.4.23 restart path is obsolete against current main because startup-derived config changes are runtime-only and protected fallback writes preserve existing keys. The old failure is source-explained by later config-writer/startup hardening rather than currently reproducible.

## Solution Assessment

Yes; making startup-derived changes runtime-only and preserving existing keys on fallback writes is the narrow maintainable fix for avoiding restart-time config clobbering without adding new config surface.
