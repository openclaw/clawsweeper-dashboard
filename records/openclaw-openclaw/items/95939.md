## Summary

Close: current main already prevents the reported restart-time config clobbering, keeps startup-derived config runtime-only, and preserves unrelated existing config keys during desktop fallback writes; the fix is also in a shipped release.

## What This Changes

The issue asks OpenClaw to preserve user-edited `agents.defaults` runtime config fields such as `bootstrapMaxChars` across desktop app restarts.

## Best Possible Solution

Keep the shipped runtime-only startup behavior and protected desktop config writer; users still seeing this on 2026.4.23 should upgrade to a current release instead of adding another compatibility path.

## Reproduction Assessment

No; there is no high-confidence current-main failing reproduction path because current source and regression coverage show startup changes are runtime-only and desktop fallback writes preserve existing keys. The reported 2026.4.23 behavior is source-explained by later config/startup hardening.

## Solution Assessment

Yes; making startup-derived changes runtime-only and preserving existing keys during fallback writes is the narrow maintainable fix for restart-time config clobbering without adding new config surface.
