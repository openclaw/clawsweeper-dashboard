## Summary

Keep open. Current main and the latest release still route active-memory recall through variable `prependContext` in user-prompt space; the OpenAI-compatible full-body hash explanation is not proven by source, but the cache-boundary concern remains real and now has a related stronger sibling report at https://github.com/openclaw/openclaw/issues/91223.

## What This Changes

The issue asks OpenClaw to prevent active-memory recall from breaking prompt-cache reuse for OpenAI-compatible providers that opt into `compat.supportsPromptCacheKey`.

## Best Possible Solution

Define the cache contract for dynamic plugin context, then either document active-memory as cache-hostile on affected providers or add a tested below-boundary path that preserves stable-prefix caching without weakening untrusted-context semantics.

## Reproduction Assessment

No high-confidence live current-main reproduction for the exact OpenAI-compatible provider claim yet. Source inspection does reproduce the active-memory `prependContext` injection path, and the sibling report adds related live provider cache metrics.

## Solution Assessment

Unclear. A documentation warning is the narrowest safe mitigation, while any code fix needs a maintainer decision about whether dynamic memory belongs in user-prompt space, system-prompt suffix space, or a new explicit cache-boundary contract.
