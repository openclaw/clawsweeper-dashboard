## Summary

Keep open. Current main still routes active-memory recall through `prependContext` into the model-facing user prompt, so the source-level cache-boundary concern remains; however OpenClaw itself forwards `prompt_cache_key` from the caller/session rather than hashing the full request body, so the provider-side cache-hit collapse still needs live trace evidence and maintainer direction.

## What This Changes

The issue asks OpenClaw to preserve OpenAI-compatible prompt-cache behavior, or document the limitation, when active-memory injects per-turn recalled context.

## Best Possible Solution

Add an owner-approved cache-aware active-memory path or clear public limitation explaining how `prependContext` interacts with OpenAI-compatible prompt caching.

## Reproduction Assessment

Source-level reproduction is high-confidence for active-memory modifying the user prompt through `prependContext`; full provider cache-hit collapse is not yet reproduced without a live OpenAI-compatible trace.

## Solution Assessment

Unclear; moving active-memory below the cache boundary is plausible, but changing `prependContext` semantics or plugin hook placement is product-sensitive and should be owner-approved.
