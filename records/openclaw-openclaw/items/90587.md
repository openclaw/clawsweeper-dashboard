## What This Changes

The issue reports that OpenClaw starts successfully on Ubuntu RISC-V64 but the Codex app-server exits because upstream @openai/codex rejects linux/riscv64.

## Best Possible Solution

Make the unsupported Codex platform path explicit in OpenClaw while tracking upstream RISC-V64 binary support: add a clear preflight/error and docs for custom app-server or direct OpenClaw runtime routing, or defer full managed support until upstream publishes a compatible binary.

## Solution Assessment
