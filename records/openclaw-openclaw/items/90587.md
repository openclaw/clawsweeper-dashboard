## Summary

Keep open: current main and v2026.6.9 still route official OpenAI agent turns through the managed Codex app-server, while OpenClaw and upstream Codex still have no linux/riscv64 managed launcher target.

## What This Changes

The issue reports that OpenClaw on Linux riscv64 fails after setup because the managed Codex app-server launcher rejects `linux (riscv64)`.

## Best Possible Solution

Keep this open until maintainers choose the platform policy: clearer unsupported-platform preflight/docs, an explicit custom app-server path, or waiting for upstream Codex linux/riscv64 binaries.

## Reproduction Assessment

Yes, from source rather than a live RISC-V64 host. Current OpenClaw still starts the managed @openai/codex launcher for official OpenAI agent turns, and upstream Codex deterministically throws for linux/riscv64 because no target triple is selected.

## Solution Assessment

Unclear as an implementation choice. A narrow OpenClaw preflight or documentation improvement is maintainable, but successful default turns on linux/riscv64 require upstream binary support or a maintainer-approved fallback/custom app-server policy.
