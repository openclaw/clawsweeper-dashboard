## Summary

Keep open: current main still lets the TUI streaming watchdog clear `activeChatRunId`, switch to idle, and emit the reported prompt after a quiet window. Related open config/notice PRs address adjacent symptoms, but they do not yet implement the requested authoritative active-run check before marking a quiet run idle.

## What This Changes

Reports that the TUI streaming watchdog can mark a quiet but still-active chat run idle and prompt the user to continue before later stream/final events arrive.

## Best Possible Solution

Before the watchdog clears a run or prompts the user, refresh or consult authoritative session/run state and keep or rearm streaming when the current session still has an active run.

## Reproduction Assessment

Yes. Source inspection plus the existing fake-timer harness gives a high-confidence path: a chat delta arms the watchdog, timer expiry clears `activeChatRunId`, sets idle, and logs the reported prompt.

## Solution Assessment

No. Current main is not the best fix because it treats event silence as stale without consulting the gateway's active-run state; config-only or notice-only changes may reduce annoyance but do not satisfy the reported expected behavior.
