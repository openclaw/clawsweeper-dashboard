## Summary

Keep open: current main still lets the TUI watchdog clear an active chat run and emit the reported timeout prompt based on stream silence alone, while the gateway already exposes active-run state that the TUI does not consume.

## What This Changes

The issue asks the TUI to keep quiet but active runs visibly streaming or working until authoritative run/session state says the run is terminal.

## Best Possible Solution

Before the watchdog clears a run or prompts the user, refresh or consume `sessions.list.hasActiveRun` for the current session; keep or rearm streaming while the gateway says the run is active, and only return to idle after terminal state or no active run.

## Reproduction Assessment

Yes. The high-confidence source path is a chat delta that arms the watchdog followed by timer expiry; current tests assert the active run is cleared and the reported prompt is emitted, though I did not run a live TUI in this read-only review.

## Solution Assessment

No. The current implementation treats stream silence as enough to reset the UI; the narrower maintainable fix is to use gateway/session active-run state before clearing TUI run state, while config-only or notice-dismissal PRs are adjacent but incomplete.

## Repair Work Prompt

Fix the TUI streaming watchdog bug reported at https://github.com/openclaw/openclaw/issues/78360. On current main, `src/tui/tui-event-handlers.ts` clears `activeChatRunId`, sets idle, and emits the timeout prompt solely because no stream delta arrived. Use authoritative session/run state instead: carry `sessions.list.hasActiveRun` through the TUI session-info path, refresh or consult it before the watchdog clears an active run, and keep/rearm streaming while the current session still has an active run. Only fall back to idle and show the timeout prompt when the run is terminal or no active run remains. Add focused regression coverage in TUI event/session tests. Do not solve this by only raising the timeout, adding a new config option, or only dismissing the notice; those are tracked by related config/UX items. Coordinate with the same-file open PRs https://github.com/openclaw/openclaw/pull/77375 and https://github.com/openclaw/openclaw/pull/79997 if they are still open. Add a concise changelog entry because this changes user-visible TUI behavior.
