## Label Justifications

- `P1`: The reported regression can fail an interactive embedded-agent turn before any reply and consume the configured fallback chain on a local coordination error.
- `impact:session-state`: The failure is triggered by active session JSONL mutation while the embedded prompt lock is released.
- `impact:auth-provider`: The observable bug is in provider/model fallback classification, where a non-provider session coordination error is counted against model candidates.

## Summary

Keep open: current main still lets the embedded session-takeover error enter the generic model-fallback path, and the active fix PR is not merged yet.

## What This Changes

The issue asks OpenClaw to treat embedded session-file mutation during the released prompt-lock window as a local session coordination failure instead of a provider/model fallback failure.

## Best Possible Solution

Land an amended fix that rethrows embedded session-takeover and recursive session-ownership errors before model fallback while preserving generic provider fallback behavior, with regression tests and redacted runtime proof.

## Reproduction Assessment

Yes, at source level: current main has the takeover throw path and the fallback loop still advances non-final unknown errors to the next candidate. I did not run a live macOS gateway reproduction in this read-only review.

## Solution Assessment

Yes for the issue direction: a narrow non-model fail-fast classification is the best first fix. Re-reading and rebasing the turn is a broader behavior change and should not be required to stop fallback exhaustion.
