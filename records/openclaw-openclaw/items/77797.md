## What This Changes

The issue requests wildcard or glob entries in plugin `contracts.tools` so dynamically registered tool namespaces do not require every tool name to be listed explicitly.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/78084 so `contracts.tools` has one documented, tested, bounded wildcard contract that preserves explicit allowlist filtering.

## Reproduction Assessment

Yes, at source level: current `main` exact-matches `contracts.tools` and rejects runtime names not exactly declared. Because this is a feature request, runtime bug reproduction is not the deciding gate.

## Solution Assessment

Unclear: bounded terminal wildcard ownership is plausible, but unrestricted glob semantics would be too broad for a tool-exposure contract. The best fix should preserve manifest-first loading and concrete allowlist filtering, with docs, tests, and real runtime proof.
