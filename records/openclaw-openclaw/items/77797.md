## What This Changes

Add support for bounded wildcard/glob patterns in plugin manifest `contracts.tools` so dynamic tool catalogs do not require enumerating every runtime tool name.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/78084 so `contracts.tools` has one documented, tested, bounded wildcard contract that preserves manifest-first loading and concrete allowlist filtering.

## Reproduction Assessment

Yes, at source level: current main exact-matches `contracts.tools` and rejects runtime names not exactly declared. Because this is a feature request, live runtime reproduction is not the deciding gate.

## Solution Assessment

Unclear: bounded terminal wildcard ownership is plausible, but unrestricted glob semantics would be too broad for a tool-exposure contract. The best fix should preserve manifest-first loading and concrete allowlist filtering, with docs, tests, and real runtime proof.
