## What This Changes

Support wildcard/glob entries in plugin `contracts.tools` so dynamic plugins can declare a tool namespace instead of enumerating every runtime tool name.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/78084 so `contracts.tools` has one documented, tested, bounded wildcard contract that preserves manifest-first discovery and concrete allowlist filtering.

## Reproduction Assessment

Yes, at source level: current main exact-matches `contracts.tools`, so a manifest entry like `xxx-*` does not match a registered `xxx-search` tool. This is a feature/API reproduction rather than a broken-runtime bug repro.

## Solution Assessment

Unclear: bounded wildcard ownership is plausible, but the permanent grammar and allowlist/security semantics need maintainer product/API judgment before it is the best fix.
