## Summary

Keep open: current main still treats `contracts.tools` as exact static tool names, and the remaining work is an unmerged public plugin-manifest feature with an active implementation candidate at https://github.com/openclaw/openclaw/pull/78084. ## What This Changes

Request support for wildcard or glob entries in `contracts.tools` so dynamic plugin tool namespaces do not need every runtime tool name listed statically.

## Best Possible Solution

Review and land a bounded dynamic-tool ownership contract, likely through https://github.com/openclaw/openclaw/pull/78084 or an equivalent branch, with exact-match compatibility, explicit allowlist filtering, docs, and regression coverage.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken documented behavior. Source inspection confirms current main uses exact `contracts.tools` matching, which explains why wildcard entries are unsupported today.

## Solution Assessment

Unclear; wildcard support is plausible, but it changes the public plugin manifest contract and tool ownership policy. The safer path is maintainer review of the bounded approach in https://github.com/openclaw/openclaw/pull/78084 with the allowlist fix and real behavior proof.
