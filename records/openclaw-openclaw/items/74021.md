## Summary

Keep open. Current main has adjacent runtime handling for reasoning deltas, reasoning-only terminal outcomes, and vLLM/Nemotron visible-content request shaping, but it still lacks the requested `oc llm` reporting schema, extraction-source counters, split scoring modes, first-class profile policy, and routing behavior; the reporter's follow-up also confirms this should remain separate from #74017.

## What This Changes

The issue requests opt-in reasoning-aware handling for native reasoning model outputs, including benchmark score splits, extraction-source metadata, profile flags, and visible-content enforcement.

## Best Possible Solution

Keep this as a maintainer design item: implement redaction-safe reporting metadata first, then add explicit profile and routing policy once the schema is agreed.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature/reporting request. Source inspection on current main shows the requested reporting and profile surfaces are missing, while exact score reproduction needs the reporter's redacted fixtures.

## Solution Assessment

Unclear because there is no proposed patch yet. The direction is sound, but the safest first implementation is reporting-only metadata with raw reasoning disabled or redacted by default.
