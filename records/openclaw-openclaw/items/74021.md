## Label Justifications

- `P2`: This is a real reasoning-model compatibility and reporting gap with production examples, but it needs design work rather than an emergency patch.
- `impact:security`: Reasoning-field ingestion and reporting could expose hidden provider reasoning unless opt-in and redaction rules are defined.
- `impact:message-loss`: `content:null` reasoning-only outputs can leave OpenClaw without a visible assistant reply or trigger fallback instead of delivering useful output.
- `impact:auth-provider`: The request touches provider/model reasoning payload policy, model-profile fields, and routing/fallback behavior.

## Summary

Keep open: current main has partial runtime mitigations for reasoning-only outputs and provider request shaping, but it still lacks the requested reasoning-aware benchmark/reporting schema, extraction-source telemetry, redaction policy, and explicit opt-in runtime/profile contract. The recent OpenRouter production comment adds current evidence rather than making the umbrella request obsolete.

## What This Changes

Request reasoning-aware handling and reporting for native reasoning outputs, including opt-in extraction from `message.reasoning`, visible-content compatibility metrics, and model/profile policy.

## Best Possible Solution

Ship a redacted reporting-first measurement layer, then add explicit provider-profile/runtime opt-ins for reasoning-field extraction and visible-content enforcement.

## Reproduction Assessment

Yes for the core source path, but not for the exact Nemotron score split. Current parsing keeps provider reasoning as thinking and visible extraction reads text/final-answer blocks, so a reasoning-only response has no visible answer unless recovery policy retries or a provider-specific wrapper promotes it.

## Solution Assessment

Unclear as a complete implementation because there is no patch and the request spans reporting schema, redaction, provider profiles, and runtime routing. The safest direction is reporting-only redacted instrumentation before any opt-in runtime extraction from reasoning fields.
