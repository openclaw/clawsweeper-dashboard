## Summary

Keep open: current main has partial native-reasoning ingestion, retry, and provider-specific visible-output handling, but the central benchmark/reporting fields, extraction-source metrics, and explicit visibility/redaction policy requested here are still missing.

## What This Changes

The issue requests reasoning-aware model output handling for native reasoning models, including opt-in `message.reasoning` extraction, visible-content enforcement, model/profile policy, and separate compatibility scores in `oc llm` reporting.

## Best Possible Solution

Add reporting-first, opt-in extraction-source metrics and visible-content compatibility scoring, then decide runtime/profile use of reasoning fields with explicit redaction and visibility rules.

## Solution Assessment
