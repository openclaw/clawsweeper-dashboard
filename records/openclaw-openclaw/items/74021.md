## Summary

Keep open. Current main has adjacent runtime handling for reasoning fields and vLLM/Nemotron visible-content request shaping, but it still does not implement the requested benchmark/report schema, extraction-source counters, separate reasoning-aware versus visible-content scores, or first-class model/profile routing policy.

## What This Changes

Request explicit reasoning-aware response handling, visible-content compatibility scoring, extraction-source reporting, and model/profile policy for native reasoning model outputs.

## Best Possible Solution

Keep this as a maintainer design item. The safest path is to first add reporting-only instrumentation for content/reasoning presence, extraction source, and separate visible-content, strict-content, and reasoning-aware score breakdowns, with raw reasoning disabled by default until retention and redaction policy is documented; runtime routing/profile changes can follow that measurement surface.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature/reporting request. Current-main verification is high confidence through source/docs/test search and inspection, while the reported benchmark percentages need the reporter's local fixtures or redacted replacements.

## Solution Assessment

Unclear as an implementation because there is no proposed patch. The requested direction is sound, but the best maintainable solution is a staged design: measurement/report schema first, then opt-in raw reasoning retention and runtime routing/profile policy.
