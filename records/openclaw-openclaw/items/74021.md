## Summary

Keep open. Current main has runtime mitigation for reasoning-only turns and vLLM/Nemotron visible-content request shaping, but it does not implement the requested `oc llm` scoring/reporting modes, extraction-source telemetry, or opt-in profile contract.

## What This Changes

Request opt-in reasoning-field ingestion, visible-content enforcement, extraction-source telemetry, and separate reasoning-aware versus visible-content scores for native reasoning models.

## Best Possible Solution

Design an explicit opt-in measurement layer first, then add runtime/profile policy only after privacy, redaction, and tool/JSON compatibility semantics are settled.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature/design request backed by local benchmark evidence outside the checkout. Source inspection does confirm current main lacks the requested `oc llm` score split and extraction-source reporting fields.

## Solution Assessment

Unclear as an implementation because there is no patch and the request spans runtime policy, model profile config, and benchmark report schema. The reporter's suggested reporting-only first step is the narrowest maintainable direction before enabling reasoning-field extraction in normal runtime paths.
