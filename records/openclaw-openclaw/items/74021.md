## Summary

Keep open. Current main has adjacent reasoning-field ingestion, Nemotron/vLLM request shaping, and reasoning-only fallback handling, but it still does not implement the requested benchmark/report schema, extraction-source counters, separate reasoning-aware versus visible-content scores, or first-class profile/routing policy.

## What This Changes

Request explicit reasoning-aware handling and reporting for native reasoning models whose useful final answer may appear in reasoning fields instead of visible message content.

## Best Possible Solution

Keep this as a maintainer design item. The safest first slice is reporting-only instrumentation for content/reasoning presence, extraction source, and separate visible-content, strict-content, and reasoning-aware score breakdowns, with raw reasoning disabled by default until redaction and retention policy is documented.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature/reporting request. Current-main verification is high confidence through source/docs search and inspection; the benchmark percentages need the reporter's local fixtures or redacted replacements.

## Solution Assessment

Unclear as an implementation because there is no proposed patch yet. The requested direction is sound, but runtime routing/profile behavior should follow after the measurement schema and raw-reasoning policy are settled.
