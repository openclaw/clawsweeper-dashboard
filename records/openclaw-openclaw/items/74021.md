## Summary

Keep open. Current main has adjacent reasoning stream parsing, reasoning-only retry, and vLLM/Nemotron visible-content shaping, but it still lacks the requested `oc llm` report schema, score splits, extraction-source counters, opt-in reasoning extraction policy, and model/profile routing contract; the reporter also agreed this should stay separate from #74017.

## What This Changes

Request opt-in reasoning-field ingestion, visible-final-answer enforcement, model/profile policy, and `oc llm` reporting/scoring splits for native reasoning outputs.

## Best Possible Solution

Design a redaction-safe reporting layer first for content/reasoning presence, extraction source, and separate visible/strict/reasoning-aware scores, then add explicit runtime/profile routing policy once the measurement contract is stable.

## Reproduction Assessment

Not applicable as a normal runtime bug reproduction because this is a feature/reporting request. Source inspection on current main verifies the requested schema and score/reporting modes are missing, while the exact Nemotron Lane B percentages require redacted benchmark fixtures.

## Solution Assessment

Unclear because there is no patch yet. The direction is reasonable, but the safest first slice is reporting-only metadata with raw reasoning disabled or redacted by default before runtime routing changes.
