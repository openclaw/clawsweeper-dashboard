## Summary

Keep open. Current main has adjacent reasoning-field ingestion, reasoning-only fallback classification, vLLM/Nemotron visible-content request shaping, and OpenAI Responses final-answer phase handling, but it still lacks the requested benchmark/report schema, extraction-source counters, separate reasoning-aware versus visible-content scores, and first-class profile policy knobs. The issue discussion also explicitly preserves this as broader product/runtime/reporting work separate from the narrower scorer bug in #74017.

## What This Changes

Request support for opt-in reasoning-field extraction, visible-content compatibility scoring, extraction-source reporting, and model/profile policy controls for native reasoning model outputs.

## Best Possible Solution

Keep this as a maintainer design item and start with an opt-in reporting-first slice: record content/reasoning presence and extraction source, compute visible-content versus reasoning-aware versus strict-content scores, default away from raw reasoning retention, then extend provider/profile routing once privacy and compatibility semantics are settled.
