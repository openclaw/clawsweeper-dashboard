## Summary

Keep open. Current main has adjacent runtime reasoning-field handling and vLLM/Nemotron visible-content request shaping, but it still lacks the requested `oc llm`/report schema, extraction-source counters, score split, and first-class profile/routing policy; raw reasoning capture also needs maintainer privacy design.

## What This Changes

The issue requests reasoning-aware response handling and benchmark/reporting support for native reasoning models where useful answers may appear in `message.reasoning` while visible content is null or incomplete.

## Best Possible Solution

Keep this as a maintainer design item and start with redaction-safe reporting metadata for content/reasoning presence, extraction source, and separate visible, strict, and reasoning-aware score breakdowns before adding runtime routing policy.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature/reporting request. Source inspection verifies the requested surfaces are missing on current main, while the exact benchmark percentages need the reporter's redacted fixtures.

## Solution Assessment

Unclear because there is no proposed patch yet. The direction is sound, but the safest first implementation slice is reporting-only metadata with raw reasoning disabled or redacted by default.
