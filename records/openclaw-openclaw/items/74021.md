## Summary

Keep open. Current main has adjacent reasoning-stream handling, provider shaping, and terminal-turn classification, but it still lacks the requested reasoning-aware benchmark/report schema, score splits, extraction-source counters, and opt-in runtime/profile policy.

## What This Changes

The issue requests support for reasoning-field ingestion, visible-final-answer enforcement, extraction-source reporting, model profile policy, and separate reasoning-aware versus visible-content compatibility scores.

## Best Possible Solution

Design a redaction-safe reporting schema for reasoning/content presence, extraction source, and separate visible, strict, and reasoning-aware scores, then layer runtime/profile routing policy onto the existing provider reasoning seams.

## Solution Assessment
