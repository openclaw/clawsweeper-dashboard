## Summary

Keep open: current main and v2026.6.8 still skip agent session transcript JSONL files during backup, so the chat-history restore contract remains unresolved. The broader native backup/restore umbrella at https://github.com/openclaw/openclaw/issues/13616 overlaps but does not replace this narrower transcript-preservation decision.

## What This Changes

Request to preserve agent session transcript JSONL in backup archives or explicitly define safe support for chat-history restore.

## Best Possible Solution

Choose and land a maintainer-approved transcript backup contract: staged, quiesced, or inactive-only transcript inclusion, or explicit user-facing docs that chat transcript restore remains outside `openclaw backup create` until native restore/export work lands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: any state tree with `agents/<agentId>/sessions/*.jsonl` hits the volatile backup filter and is excluded from the archive, and current tests assert that omission.

## Solution Assessment

Unclear. The issue identifies a real restore-semantics gap, but simply reverting the skip is not the best fix because it would reopen the live tar EOF race; maintainers need to choose the snapshot and sensitive-data contract first.
