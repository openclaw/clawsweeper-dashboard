## Label Justifications

- `P2`: This is a normal-priority backup/restore semantics gap with real disaster-recovery impact but no current core outage.
- `impact:data-loss`: Backup archives can omit durable chat transcript content that users may expect to recover after restore or migration.
- `impact:session-state`: The affected files are OpenClaw session transcripts and trajectory records tied to chat history state.
- `impact:security`: Any transcript backup path must handle sensitive transcript content such as private messages, tool output, pasted secrets, and file contents.

## Summary

Keep open: current main and v2026.6.9 still skip agent session transcript JSONL from backup archives, so the chat-history restore contract remains unresolved and needs maintainer product/security direction before a fix shape is chosen.

## What This Changes

Request to preserve or explicitly support agent session transcript JSONL and trajectory JSONL in `openclaw backup create` so chat history can be restored without reintroducing live tar races.

## Best Possible Solution

Define a first-party transcript backup contract, then implement staged, quiesced, or inactive-only transcript inclusion, or explicitly document transcript omission as the supported backup behavior.

## Reproduction Assessment

Yes at source level. Current main classifies `agents/<agentId>/sessions/*.jsonl` as volatile and the backup test asserts those files are absent from the archive.

## Solution Assessment

Unclear. The report correctly avoids a blind revert, but maintainers need to choose the snapshot consistency and sensitive-data contract before implementation.
