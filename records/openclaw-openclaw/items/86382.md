## Summary

Keep open: current main and v2026.6.8 still omit agent session transcript JSONL from backup archives, so the chat-history restore contract remains unresolved and needs maintainer product/security direction rather than closure.

## What This Changes

Request to preserve or explicitly support session transcript JSONL in `openclaw backup create` so chat history can be restored.

## Best Possible Solution

Define a first-party transcript backup contract, then implement staged, quiesced, or inactive-only transcript inclusion, or make the omission explicit in user-facing backup/restore docs.

## Reproduction Assessment

Yes. Source inspection shows `agents/<agentId>/sessions/*.jsonl` matches the volatile filter and is dropped by the tar filter; the current backup test asserts that omission.
