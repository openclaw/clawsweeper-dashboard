## Summary

Keep open: the PR is aimed at the right provider-owned OpenAI TTS parser gap, but it still lets out-of-range model-emitted speed directives throw out of directive parsing instead of returning directive warnings.

## What This Changes

The PR adds OpenAI TTS speed directive parsing, focused tests for speed directives and telephony override propagation, and a changelog entry for the linked OpenAI TTS speed request.

## Best Possible Solution

Merge a provider-owned OpenAI speed directive fix once invalid and out-of-range speed values are handled as directive warnings while valid overrides still serialize into normal and telephony OpenAI speech requests.

## Reproduction Assessment

Yes, from source inspection. Current main routes directive tokens to the selected provider parser, but the OpenAI parser falls through for speed, so providerOverrides.openai.speed is never written.

## Solution Assessment

No, not as-is. The owner-local parser and telephony coverage are the right shape, but out-of-range OpenAI speed values should follow the directive warning contract instead of throwing out of parseTtsDirectives.

## Review Findings
