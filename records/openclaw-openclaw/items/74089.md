## Summary

Keep this PR open: it fixes a real current-main OpenAI TTS directive gap, but the latest diff introduces a non-fatal directive validation path that can throw through `parseTtsDirectives` and should be fixed before merge.

## What This Changes

The PR adds OpenAI TTS `speed`/`openai_speed` directive parsing, focused OpenAI parser tests, telephony override test coverage, and a changelog entry for #12163.

## Best Possible Solution

Land the provider-owned OpenAI speed parser with warning-safe validation and focused regression coverage, preserving the current main telephony override forwarding.

## Reproduction Assessment

Yes, by source inspection. Current main routes `[[tts:speed=1.5]]` to provider parsers, but OpenAI only handles voice/model keys, so no OpenAI speed override is recorded.

## Solution Assessment

No, not as currently patched. The provider-owned parser is the right fix shape, but out-of-range speed validation should return a directive warning instead of throwing through the shared parser.
