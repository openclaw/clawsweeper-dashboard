## Summary

Keep open: the PR targets the right OpenAI-owned TTS directive parser gap and includes sufficient happy-path request-body proof, but it still makes out-of-range model-emitted speed directives throw out of directive parsing instead of returning warnings.

## What This Changes

The PR adds OpenAI `speed` directive alias parsing, OpenAI speech provider tests, telephony override test coverage, and a changelog entry.

## Best Possible Solution

Merge a scoped OpenAI provider fix after out-of-range speed directives become non-fatal warnings while valid speed overrides still serialize through normal and telephony OpenAI speech requests.

## Reproduction Assessment

Yes. Source inspection shows current main's OpenAI parser does not claim `speed`, while the request builder already supports `speed`; the PR body supplies after-fix request-body proof for the happy path.

## Solution Assessment

No, not as-is. The provider-local parser and telephony override coverage are the right direction, but range validation should follow the directive-parser warning contract instead of throwing out of `parseTtsDirectives`.

## Review Findings

Overall correctness: patch is incorrect
