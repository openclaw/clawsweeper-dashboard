## Summary

Keep open: the PR targets a real update CLI failure on legacy config, but the new repair call runs before dry-run handling and would write config during a documented preview path; the draft, maintainer-labeled PR needs follow-up before merge.

## What This Changes

This PR adds an update CLI preflight that migrates doctor-migratable legacy Slack/Telegram streaming config before persisting a requested update channel, plus regression coverage and a changelog entry.

## Best Possible Solution

Keep the legacy repair path, but make `--dry-run` preview-only for both migration and channel persistence, with regression coverage that asserts no config write occurs in that mode.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `openclaw update --channel ...` when the snapshot is invalid, and the PR test models a doctor-migratable Slack/Telegram legacy config; the new dry-run regression is also source-reproducible from the PR line order.

## Solution Assessment

No. The repair direction is narrow and maintainable, but it currently runs before `opts.dryRun`; the safer fix is to skip the write during dry-run or compute a non-mutating preview candidate.
