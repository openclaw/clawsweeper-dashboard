## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve before_message_write for user transcript updates:** `src/gateway/server-methods/chat.ts:3036`
  - body: This new update path emits the built user message directly, but the existing user-turn transcript path applies `before_message_write` before the inline transcript update and persistence. A plugin that blocks or redacts user messages can now still have the unredacted content published to transcript consumers; route this through the hook-aware recorder result or suppress the update when the hook blocks it.
  - confidence: 0.91
- **[P2] Scope media rewrites to the created user turn:** `src/gateway/server-methods/chat.ts:1157-1172`
  - body: The media rewrite searches backward for the newest user row with the same text and no media paths. If two identical `chat.send` calls are in flight, this can attach media to the wrong user turn; target the specific created message by id, idempotency key, or run identity instead of message text.
  - confidence: 0.86
- **[P2] Trim stale assistants before turns grouping:** `src/gateway/server-methods/chat.ts:2253`
  - body: Display clients now default to `turns`, but the stale leading-assistant trim is skipped when `historyMode === "turns"`. A pre-session assistant reply can therefore become an assistant-only turn instead of being filtered like the legacy message path; apply the timestamp trim before turn grouping as well.
  - confidence: 0.84
- **[P2] Honor raw-messages history limits:** `src/gateway/server-methods/chat.ts:2290-2291`
  - body: For `mode: "raw-messages"`, `historyMode` is `messages`, `scanMax` becomes `limit + 1`, and this branch returns `rawMessages.slice(-scanMax)`. That means an admin diagnostic request with `limit: 10` can receive 11 raw rows; raw mode should still trim to the requested limit after any internal overread.
  - confidence: 0.88
