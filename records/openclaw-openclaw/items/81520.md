## Summary

Keep open: current main and the latest release still preserve the latest GitHub Copilot Claude assistant `thinking` block during replay, matching the reported follow-up 400 payload shape. The issue is already paired with open implementation PRs, so it should close only after one provider-scoped fix lands.

## What This Changes

The issue reports that GitHub Copilot Claude multi-turn replay sends persisted `thinking` or `redacted_thinking` blocks back to the provider and gets HTTP 400 on follow-up turns.

## Best Possible Solution

Land one provider-scoped fix that strips `thinking` and `redacted_thinking` from every GitHub Copilot Claude replay turn while preserving native Anthropic and Bedrock signed-thinking replay semantics.

## Reproduction Assessment

Yes, source-reproducible: current main sets Copilot Claude to `dropThinkingBlocks`, but the helper and tests preserve latest assistant thinking blocks. I did not run a live Copilot credential repro in this read-only review.

## Solution Assessment

Yes, the best fix direction is the linked provider-scoped all-thinking drop for GitHub Copilot Claude replay. A global helper change would risk breaking native Anthropic or Bedrock signed-thinking continuation semantics.
