## Label Justifications

- `P1`: Affected GitHub Copilot Claude conversations fail on follow-up turns after reasoning output, breaking a real multi-turn provider workflow.
- `impact:message-loss`: The second turn fails before an assistant response can be delivered to the user.
- `impact:session-state`: The failure is caused by persisted assistant thinking blocks being replayed from conversation history.
- `impact:auth-provider`: The incompatibility is specific to the GitHub Copilot provider's Claude replay payload contract.

## Summary

Keep open: current main and v2026.6.6 still route GitHub Copilot Claude replay through the latest-turn-preserving thinking sanitizer, matching the reported second-turn HTTP 400 shape. Two open linked PRs already own the implementation path, so this issue should close only after one provider-scoped fix lands.

## What This Changes

The issue asks OpenClaw to strip `thinking` and `redacted_thinking` blocks from GitHub Copilot Claude replay history so multi-turn reasoning conversations do not fail with HTTP 400.

## Best Possible Solution

Land one canonical provider-owned fix that strips `thinking` and `redacted_thinking` from every GitHub Copilot Claude replay and tool-continuation payload while preserving native Anthropic/Bedrock signed-thinking replay semantics.

## Reproduction Assessment

Yes, at source level. Current main sets Copilot Claude to `dropThinkingBlocks`, the helper preserves latest assistant thinking, and current tests still assert that unsafe replay shape; this review did not use live Copilot credentials.

## Solution Assessment

Yes, the best fix direction is provider-scoped all-turn thinking removal for GitHub Copilot Claude. A global helper change would risk breaking native Anthropic or Bedrock signed-thinking continuation semantics.
