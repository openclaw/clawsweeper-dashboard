## Summary

Keep open: current main still preserves the prior assistant `thinking` block in the normal GitHub Copilot Claude follow-up transcript shape, matching the reported 400-causing replay payload.

## What This Changes

The issue asks OpenClaw to strip `thinking` and `redacted_thinking` blocks from GitHub Copilot Claude replay history before follow-up provider requests.

## Best Possible Solution

Add a GitHub Copilot Claude replay policy or sanitizer mode that strips completed prior assistant `thinking` and `redacted_thinking` blocks before outbound follow-up requests, with focused regression coverage for the second-turn transcript shape.

## Reproduction Assessment

Yes at source level: current main's Copilot sanitizer test explicitly preserves a `thinking` block in `[user, assistant, user follow-up]`, and the runtime policy applies that helper before provider send. I did not establish a live Copilot 400 in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is provider-scoped Copilot replay sanitization plus regression tests. A global change to `dropThinkingBlocks` would be riskier because other provider paths intentionally preserve signed thinking.
