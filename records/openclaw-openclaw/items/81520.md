## What This Changes

The issue reports that GitHub Copilot Claude reasoning sessions replay stored `thinking` / `redacted_thinking` blocks on the second turn, causing HTTP 400 responses.

## Best Possible Solution

Land a narrowed provider-owned Copilot replay policy that strips all Copilot Claude thinking blocks, preferably by finishing https://github.com/openclaw/openclaw/pull/81534, while keeping Anthropic-family signed-thinking replay unchanged.

## Reproduction Assessment

Yes at source level, but not with a live API call: current main's Copilot Claude replay policy resolves to `dropThinkingBlocks`, and that helper preserves the latest assistant `thinking` block in the follow-up transcript shape. I did not run a live GitHub Copilot request in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a Copilot-scoped replay sanitizer that drops all Copilot Claude thinking blocks while preserving Anthropic/Bedrock latest-turn behavior. The open closing PR is the active implementation vehicle, but the issue should remain open until a fix merges.
