## Summary

Keep open: current main still routes GitHub Copilot Claude history through a sanitizer path that preserves the prior assistant `thinking` block on a follow-up turn, matching the reported 400-causing payload shape.

## What This Changes

The issue asks OpenClaw to strip `thinking` and `redacted_thinking` content blocks from GitHub Copilot Claude replay history before follow-up requests.

## Best Possible Solution

Add a Copilot-scoped replay policy or sanitizer mode that strips thinking blocks from completed prior assistant turns before follow-up Copilot requests, while preserving signed thinking where direct Anthropic-family providers require it.

## Reproduction Assessment

Yes at source level: current tests explicitly preserve a `thinking` block in a `[user, assistant, user follow-up]` GitHub Copilot transcript, and the runtime path applies the same helper. I did not establish a live API 400 in this read-only review.

## Solution Assessment

Yes: the best fix is a narrow provider-scoped replay sanitization change plus regression coverage for Copilot Claude follow-up turns. A global change to `dropThinkingBlocks` would be riskier because other providers use signed thinking replay differently.
