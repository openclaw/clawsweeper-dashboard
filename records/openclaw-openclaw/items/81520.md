## What This Changes

The issue requests stripping `thinking` and `redacted_thinking` blocks from GitHub Copilot Claude replay history so follow-up turns do not fail with HTTP 400.

## Best Possible Solution

Review and land the provider-scoped fix in https://github.com/openclaw/openclaw/pull/81534 after its proof and test-mock blocker are resolved, while preserving direct Anthropic and Bedrock signed-thinking replay semantics.

## Reproduction Assessment

Yes, at source level: current main and `v2026.5.12` route GitHub Copilot Claude through `dropThinkingBlocks`, and the current test for `[user, assistant thinking-only, user follow-up]` expects the thinking block to remain. I did not run a live Copilot API request in this read-only review.

## Solution Assessment

Yes, the maintainable direction is a Copilot-scoped replay sanitizer or policy that strips all thinking blocks for Copilot without changing the shared latest-preserving helper used by Anthropic-family replay. The open linked PR is the right vehicle once its proof and test-mock issue are addressed.
