## Summary

Keep open. Current main already has a GitHub Copilot Claude replay policy that asks to drop thinking blocks, but the shared helper preserves the latest assistant turn even when a follow-up user is present; existing tests currently assert that Copilot keeps the same thinking block shape reported here.

## What This Changes

The issue requests stripping `thinking` and `redacted_thinking` assistant content from GitHub Copilot Claude replay history so multi-turn requests do not return HTTP 400.

## Best Possible Solution

Keep the provider-owned Copilot policy but make normal follow-up replay strip prior assistant thinking, while preserving signed thinking only for providers and tool-continuation cases that require it.

## Reproduction Assessment

Yes at source level: the current Copilot sanitizer path preserves a thinking-only prior assistant turn in a `[user, assistant, user]` follow-up transcript. I did not run tests or live Copilot because this review is read-only, but the failing shape is directly covered by current source and tests.

## Solution Assessment

Yes, the narrow fix should be in replay sanitization semantics, not a new config option. The safest path is to strip Copilot Claude replay thinking after a user follow-up while keeping the existing signed-thinking preservation for Anthropic/Bedrock current tool continuations.
