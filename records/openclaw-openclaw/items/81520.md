## Summary

Keep open: current main and the latest release still route GitHub Copilot Claude replay through the latest-turn-preserving `dropThinkingBlocks` path, and two open closing PRs still own the implementation decision.

## What This Changes

The issue reports that GitHub Copilot Claude reasoning turns replay persisted `thinking` or `redacted_thinking` blocks on the next message, causing HTTP 400 and broken multi-turn conversations.

## Best Possible Solution

Land one canonical provider-owned fix, preferably https://github.com/openclaw/openclaw/pull/87060 or an equivalent, that strips Copilot Claude thinking blocks from replay and continuation payloads while preserving native Anthropic and Bedrock signed-thinking behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Copilot Claude selects `dropThinkingBlocks`, `sanitizeSessionHistory` calls the helper, and current tests assert latest assistant thinking remains in GitHub Copilot replay.

## Solution Assessment

Yes. The best fix direction is provider-scoped all-turn thinking cleanup for GitHub Copilot Claude; changing the global helper would risk breaking native Anthropic and Bedrock signed-thinking replay.
