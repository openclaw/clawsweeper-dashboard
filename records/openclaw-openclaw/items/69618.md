  - body: `resolveGatewaySessionStoreTarget()` can choose a different canonical key for deleted-main or legacy session mappings, but this return still exposes the pre-target `canonicalKey`.   - confidence: 0.9
- **[P2] Update the lockfile for the new dependency:** `package.json:1662`
  - body: `package.json` adds `@types/qrcode-terminal`, but the lockfile importer only updates `@types/node`. Frozen installs will treat the lockfile as stale until the new dev dependency is represented in `pnpm-lock.yaml` or removed.
  - confidence: 0.9
- **[P2] Regenerate the base config schema:** `src/config/zod-schema.session.ts:151`
  - body: The runtime schema now accepts `messages.responseFooter`, and the PR also adds `agents.defaults.responseUsage`, but the generated base config schema consumed by schema-driven clients is not refreshed. Those clients will not discover or describe the new settings.
  - confidence: 0.86

  - body: Channel `/usage` now always persists an explicit value, but this parser still accepts only `off|tokens|full|cost`. After a chat user overrides the mode, they cannot clear it back to inheriting `agents.defaults.responseUsage` like the TUI can.
  - confidence: 0.83

  - body: `shouldResolveLateTemplateContext` becomes true whenever `onResponseTemplateContextResolved` is present, and the new channel pipeline provides that callback broadly.   - confidence: 0.82
