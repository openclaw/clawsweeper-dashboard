Full review comments:

- **[P1] Return target store data from loadSessionEntry:** `src/gateway/session-utils.ts:461-463`
  - body: `resolveGatewaySessionStoreTarget()` can select a different backing store and canonical key for deleted-main or legacy sessions, but this code still searches and returns the pre-target store/canonical key. Callers can then read or patch the wrong session even though the target helper found the intended store.
  - confidence: 0.9
- **[P2] Keep the unsupported-platform fast fail:** `extensions/memory-lancedb/lancedb-runtime.ts:292-299`
  - body: The new `explicitPlatformOverride` condition means normal runtime usage no longer throws immediately on unsupported native platforms such as `darwin-x64`; it falls through to runtime install attempts before failing. Preserve the existing fast-fail unless the platform is actually supported.
  - confidence: 0.88
- **[P2] Compose caller response-template callbacks:** `src/plugin-sdk/inbound-reply-dispatch.ts:143-146`
  - body: `params.replyOptions` may already include `onResponseTemplateContextResolved`, but this assignment overwrites it after spreading caller options. SDK callers that supply the callback will silently lose their hook, so compose both callbacks or omit this field from accepted options.
  - confidence: 0.86
- **[P2] Regenerate the base config schema:** `src/config/zod-schema.session.ts:162`
  - body: The PR adds new accepted config keys, but the generated base schema still lacks `messages.responseFooter` and `agents.defaults.responseUsage`. Schema-driven clients and config help cannot discover the new settings until the generated artifact is refreshed.
  - confidence: 0.84
- **[P2] Gate late-template work on configured consumers:** `src/auto-reply/reply/agent-runner.ts:1475-1478`
  - body: Gateway and inbound dispatch now pass `onResponseTemplateContextResolved`, so this condition becomes true for default replies even when `/usage` is off and no footer or late-bound prefix is configured. That forces fresh session-store reads and late usage assembly on the hot reply path for no visible output.
  - confidence: 0.82
- **[P2] Add an inherit path for channel usage commands:** `src/auto-reply/reply/commands-session.ts:355-358`
  - body: Channel text commands now persist an explicit `off|tokens|full` value, but the parser still does not accept an `inherit` or `default` mode that clears the override. After a chat user changes `/usage`, that session cannot return to inheriting `agents.defaults.responseUsage`.
  - confidence: 0.82
