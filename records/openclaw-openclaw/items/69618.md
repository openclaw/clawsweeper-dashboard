  - body: `resolveGatewaySessionStoreTarget()` can choose a different store path and canonical key for deleted-main or legacy sessions, but this code still returns the pre-target `storePath`, `store`, and `canonicalKey`. Callers can then read or patch the wrong session even though the target helper found the intended backing store.
  - confidence: 0.9

  - body: `package.json` adds `@types/qrcode-terminal`, but the shown lockfile importer patch only changes the `@types/node` specifier. Frozen installs will treat the lockfile as stale until the new dev dependency is represented in `pnpm-lock.yaml` or removed from this PR.

  - body: `params.replyOptions` may already include `onResponseTemplateContextResolved`, but this assignment is written after the spread and overwrites the caller's callback. SDK callers that supply the hook will silently lose it; compose both callbacks or omit this field from accepted options.
  - confidence: 0.86
