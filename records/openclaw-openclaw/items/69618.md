  - body: The regenerated schema drops `sensitive: true` from `messages.tts.providers.*.apiKey` and removes the persona-provider API-key hint even though these are credential fields.   - body: Because `shouldResolveLateTemplateContext` is true whenever `onResponseTemplateContextResolved` is present, and the shared channel pipeline now supplies that callback broadly, default replies still do fresh session-store reads and late usage/cost/context assembly even when `/usage` is off and no footer or late-bound prefix is configured.
  - confidence: 0.88
- **[P2] Preserve workspace auth for footer costs:** `src/auto-reply/reply/agent-runner.ts:1835-1836`
  - body: The late footer path recomputes `resolveModelAuthMode(providerUsed, cfg)` without the `workspaceDir` option used by the existing usage formatter. Workspace-scoped provider keys can therefore show cost in the normal `/usage` path but drop `{cost}` or cost-bearing footer output in the new path.
  - confidence: 0.84
- **[P2] Add chat inherit/default support:** `src/auto-reply/reply/commands-session.ts:355-357`
  - body: The text-command `/usage` path now makes explicit `off` meaningful, but it still rejects `inherit`/`default` and always persists a concrete value. After a Telegram, WhatsApp, or other chat user overrides usage once, they cannot return that session to inheriting `agents.defaults.responseUsage`.
  - confidence: 0.9
- **[P2] Add the changelog entry:** `src/config/types.messages.ts:161`
  - body: This PR adds user-facing config and reply behavior, but `CHANGELOG.md` has no entry for the response footer/default usage changes. OpenClaw policy requires user-facing feature/fix changes to land with a changelog bullet in the active section.
