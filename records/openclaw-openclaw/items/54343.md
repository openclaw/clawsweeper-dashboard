## Summary

Keep open: the security-hardening goal may be worth considering, but this branch is not mergeable because it calls a helper that current main does not export or import and it removes the existing sender authorization gate. The intended `/stop` scope policy still needs maintainer confirmation before repair.

## What This Changes

The PR replaces the auto-reply `/stop` sender authorization gate with a gateway-client `operator.admin` scope check in `src/auto-reply/reply/commands-session-abort.ts`.

## Best Possible Solution

If maintainers approve stricter `/stop` hardening, preserve `rejectUnauthorizedCommand`, add the exported gateway scope gate after it, and cover internal write/admin plus external unauthorized sender cases with tests and live proof.

## Reproduction Assessment

Yes for the PR defects: the live diff and current source show the missing exported/imported helper and the removed sender gate. The underlying product-policy bug is not fully established because current main still scopes direct abort RPCs as `operator.write`.

## Solution Assessment

No. Replacing the sender gate is not the narrow maintainable fix; any approved scope hardening should be additive after existing sender authorization and should include focused regression coverage.
