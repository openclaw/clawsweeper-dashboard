## Summary

Keep open: the proposed hardening is security-sensitive but the branch is currently incorrect, removes the existing sender authorization path, lacks real behavior proof, and needs maintainer confirmation because direct abort RPCs are still scoped to operator.write.

## What This Changes

The PR changes the auto-reply /stop handler to require operator.admin through a gateway scope check instead of the existing sender authorization gate.

## Best Possible Solution

If maintainers want this hardening, keep rejectUnauthorizedCommand(params, "/stop"), add the exported requireGatewayClientScope check after it, and cover internal operator.write rejected, internal operator.admin allowed, and external unauthorized sender blocked.

## Reproduction Assessment

Yes for the PR defects: the live diff and current source show the unexported helper call and the removed sender gate. No high-confidence current-main product bug is established because current direct abort RPCs remain operator.write scoped.

## Solution Assessment

No. Replacing the sender gate is not the narrow safe fix; the maintainable shape is an additive gateway scope check after existing sender authorization, pending maintainer approval of the stricter /stop policy.
