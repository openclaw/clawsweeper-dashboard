## Summary

Keep open: the branch still has blocking compile and authorization defects, and the stricter `/stop` scope policy needs maintainer confirmation because current main keeps `/stop` sender-gated while direct abort RPCs remain `operator.write` scoped.

## What This Changes

The PR changes the auto-reply `/stop` handler to require `operator.admin` for internal gateway clients instead of using the existing sender authorization gate.

## Best Possible Solution

Confirm the desired `/stop` internal scope policy; if approved, keep the existing sender gate, add the `operator.admin` internal-channel check immediately after it, and include focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes for the PR defects: static inspection of the provided diff and current helper contract shows the missing import and the dropped external sender gate. The underlying hardening request is only partly reproducible as a product bug because direct abort RPCs are still intentionally `operator.write` scoped on current main.

## Solution Assessment

No. Replacing `rejectUnauthorizedCommand` is not the maintainable fix; the safer shape is an additive internal-channel `operator.admin` check after the existing sender authorization, pending maintainer confirmation of the stricter policy.
