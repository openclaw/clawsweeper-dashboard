## Summary

Keep open: the hardening goal may be valid, but the current patch is not mergeable because it calls an unimported helper and removes the existing external-sender authorization gate; the desired `operator.admin` policy for `/stop` still needs maintainer confirmation.

## What This Changes

The PR changes the auto-reply `/stop` handler to call `requireGatewayClientScopeForInternalChannel` for `operator.admin` instead of the existing `rejectUnauthorizedCommand` sender gate.

## Best Possible Solution

If maintainers approve the stricter `/stop` policy, preserve `rejectUnauthorizedCommand`, import and add the internal-channel `operator.admin` check after it, add focused tests, and add a changelog entry.

## Reproduction Assessment

Yes for the PR defects: static inspection of the provided diff and current helper contract shows the missing import and the sender-gate regression. The underlying admin-scope request is source-reproducible only as a policy gap, not a high-confidence current-main bug, because direct abort RPCs remain `operator.write`.

## Solution Assessment

No. Replacing the sender gate is not the best fix shape; the maintainable version is an additive internal-channel scope check after the existing sender authorization, pending maintainer confirmation of the `/stop` scope policy.
