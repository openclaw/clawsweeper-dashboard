## Summary

Keep open: the PR still has blocking correctness and security issues, and current main has not already made `/stop` require `operator.admin`. ## What This Changes

The PR changes the auto-reply `/stop` handler to use an internal gateway `operator.admin` scope check in `src/auto-reply/reply/commands-session-abort.ts`.

## Best Possible Solution

If maintainers approve stricter `/stop` authorization, add the internal `operator.admin` check after the existing sender gate, import the helper, and add focused `/stop` auth coverage.

## Reproduction Assessment

Yes for the PR defects: PR-head source inspection shows the unimported helper call and the dropped sender authorization gate. No high-confidence current-main product bug is established because abort RPCs remain intentionally `operator.write` scoped.

## Solution Assessment
