## Summary

Close as duplicate/superseded: current main already lifts the cron `timeoutSeconds` resume path out of the 180s cap, while the remaining default-cap and watchdog-policy work is already tracked by the open canonical watchdog issue and implementation PR.

## What This Changes

Requests cron agent-turn resume watchdogs honor a per-job no-output override and loosen the 180s resumed CLI cap for long silent work.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/40982 as the canonical watchdog policy tracker, with any maintainer-approved cap/default or per-run override change landing through that issue or its replacement PR.
