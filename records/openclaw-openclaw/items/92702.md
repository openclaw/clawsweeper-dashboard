## What This Changes

The branch disposes the previous registered, unleased session MCP runtime when a sessionKey is rebound to a new sessionId and adds manager regression tests for rebind disposal and leased-runtime preservation.

## Best Possible Solution

Land the manager-level rebind cleanup if maintainer review agrees, while treating https://github.com/openclaw/openclaw/pull/92696 as a separate reset-time cleanup path.

## Reproduction Assessment

Yes. Source inspection shows current main can preserve a stable sessionKey while compaction records a new sessionId, and getOrCreate then remaps that key while the prior runtime remains registered until idle sweep.

## Solution Assessment

Yes. The session MCP runtime manager owns the sessionKey-to-sessionId invariant, and disposing only the prior registered, unleased runtime is narrower and safer than per-server restart config or MCP-server self-healing.
