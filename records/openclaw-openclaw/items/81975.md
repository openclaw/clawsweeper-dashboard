## What This Changes

The branch initializes the grammY bot before isolated Telegram polling ingress drains spooled updates and adds a regression test asserting init precedes handleUpdate.

## Best Possible Solution

Land the narrow init-before-drain fix after redacted live Telegram proof shows isolated polling drains and replies successfully in a real gateway setup.

## Reproduction Assessment

Yes from source and dependency inspection, though not from a fresh live gateway run. Current main drains isolated spooled updates through handleUpdate without runner initialization, and grammY throws until init() or constructor botInfo provides bot info.

## Solution Assessment

Yes. Calling grammY's idempotent bot.init() before the isolated drain starts is the narrow maintainable fix; the remaining blocker is real behavior proof, not a different code shape.
