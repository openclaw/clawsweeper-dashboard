## Summary

Keep this PR open: it addresses a real source-level unbounded response-read problem, but it duplicates the existing provider JSON helper, overlaps an already reviewed Tavily PR, and lacks required real oversized-stream proof.

## What This Changes

The PR changes Perplexity and Tavily successful web-search JSON response parsing to read through a 16 MiB bounded response helper instead of calling response.json() directly.

## Best Possible Solution

Use readProviderJsonResponse for both providers, keep focused normal and oversized-stream coverage, coordinate or drop the Tavily overlap with https://github.com/openclaw/openclaw/pull/96026, and add real oversized-response terminal/log proof before merge.

## Reproduction Assessment

Yes at source level: current main has unbounded response.json() calls in the Perplexity and Tavily success JSON helpers. I did not run a live oversized streaming repro in this read-only checkout.

## Solution Assessment

No, not as submitted. The right fix is to reuse the existing readProviderJsonResponse contract and add real oversized-stream proof; the Tavily portion also needs coordination with the already open Tavily PR.
