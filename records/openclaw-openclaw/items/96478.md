## Summary

Keep open: the patch is a focused and likely correct security hardening for the Perplexity and Tavily web-search success paths, but the PR currently has only test/CI-style proof and needs real oversized-response proof before merge.

## What This Changes

The PR replaces unbounded Perplexity and Tavily success JSON response parsing with bounded body reads capped at 16 MiB.

## Best Possible Solution

Land this bounded-read fix after contributor-supplied oversized-response proof confirms both changed providers cancel early and still parse normal responses under the cap.

## Reproduction Assessment

Yes, source-reproducible: current main calls unbounded response.json() in both changed helpers, and both helpers sit on external web-search response paths. I did not run a live oversized-response repro in this read-only review.

## Solution Assessment

Yes for these two providers: using the existing bounded reader matches the sibling Exa and Parallel fixes and stays inside the plugin runtime boundary. The merge-ready path still needs provider-specific real behavior proof and maintainer acceptance of the cap.
