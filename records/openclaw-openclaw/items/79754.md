## Summary

Keep open: current main still has the reported Bedrock policy gap. The Bedrock runtime hook advertises adaptive Claude 4.6 thinking, but directive/session validation depends on a lightweight provider-policy artifact and `extensions/amazon-bedrock` still does not provide one; the only linked fix PR is closed and unmerged.

## What This Changes

Make `/think:adaptive` and `agents.defaults.thinkingDefault: "adaptive"` work for Amazon Bedrock Claude 4.6 by exposing Bedrock's adaptive thinking profile to directive and session validation.

## Best Possible Solution

Add a Bedrock-owned lightweight provider-policy surface that reuses the Bedrock-aware Claude profile logic, with regression coverage for prefixed Claude 4.6 refs and no core Bedrock special case.

## Solution Assessment
