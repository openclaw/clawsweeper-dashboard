The issue reports that Amazon Bedrock Claude 4.6 rejects `/think:adaptive` and remaps `thinkingDefault: "adaptive"` to `medium` because the Bedrock plugin was missing a lightweight provider-policy surface.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. A lightweight plugin-owned provider-policy artifact is the narrow boundary-correct fix because directive/session validation can see Bedrock thinking policy without loading the full Bedrock runtime.
