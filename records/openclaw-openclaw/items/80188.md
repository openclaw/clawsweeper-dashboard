## Summary

Keep open: current main still lacks a generic structured `api.runtime.llm.completeStructured(...)` surface. The merged media-understanding structured extraction path is image-first and intentionally narrower, while https://github.com/openclaw/openclaw/pull/80203 is an open closing implementation candidate that should be reviewed instead of closing this issue early.

## What This Changes

The issue requests a generic host-owned structured `api.runtime.llm` inference API for plugins, covering text-only and image-plus-text inputs without exposing credentials.

## Best Possible Solution

Review and either land or intentionally close https://github.com/openclaw/openclaw/pull/80203; if accepted, it should add a versioned `runtime.llm` structured-completion seam while keeping https://github.com/openclaw/openclaw/pull/79334 and the merged media-understanding path image/provider focused.
