## Summary

Keep open: current main still lacks a generic structured `api.runtime.llm` inference helper, and the related media-understanding PR remains open and intentionally narrower. The remaining question is API/security design and canonical tracking, not cleanup closure.

## What This Changes

Request a generic host-owned plugin runtime API for structured text and image model inference, likely under `api.runtime.llm.completeStructured(...)`.

## Best Possible Solution

Have maintainers choose the canonical API and tracking issue, then design an additive structured inference seam that reuses host auth/model policy, supports text and image inputs, returns controlled schema/error envelopes, and keeps plugin prompts, schemas, routes, and storage plugin-owned.
