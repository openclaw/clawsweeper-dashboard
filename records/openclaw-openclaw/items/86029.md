## What This Changes

This PR adds `meta.agentMeta` usage, cost, provider, and model plumbing to gateway `agent.wait` snapshots through run dispatch, dedupe parsing, and wait responses.

## Best Possible Solution

Land the additive `agent.wait` telemetry contract only after metadata is tied to the same terminal snapshot source used by `agent.wait`, same-runId chat isolation is preserved, lifecycle and dedupe paths are both covered by focused tests, and redacted real output proves the response.

## Reproduction Assessment

Yes. Source inspection plus existing same-runId gateway tests show how the new fallback bypasses ignored agent snapshots, and the lifecycle/dedupe race shows how lifecycle-first waits can still miss metadata.

## Solution Assessment

No, not yet. The response shape is a reasonable additive direction, but the implementation should carry normalized telemetry through the selected terminal snapshot contract instead of sampling `context.dedupe` opportunistically.

Full review comments:

- **[P1] Keep ignored agent metadata out of chat waits:** `src/gateway/server-methods/agent.ts:2136-2141`
  - body: When `hasActiveChatRun` is true, current code intentionally ignores stale `agent:<runId>` snapshots. This new fallback unconditionally reads that agent entry and can attach stale provider/model/cost metadata to a chat wait response for the same reused run ID. Please derive metadata from the same snapshot source that won the wait, or respect `ignoreAgentTerminalSnapshot` here as well.
  - confidence: 0.9
- **[P2] Make lifecycle waits carry agentMeta reliably:** `src/gateway/server-methods/agent.ts:2205-2210`
  - body: If the lifecycle promise wins, this branch only samples whatever is already in `context.dedupe`; lifecycle snapshots themselves still do not include `agentMeta`. A lifecycle end that arrives before the terminal dedupe write can therefore return without `meta.agentMeta`, contrary to the PR's lifecycle-path claim. Please carry telemetry into lifecycle snapshots or wait for the terminal dedupe snapshot when metadata is missing.
  - confidence: 0.87
