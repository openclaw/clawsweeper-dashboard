## Summary

The branch adds gateway `agentMeta` usage/cost/provider/model extraction and returns it under `payload.meta.agentMeta` from `agent.wait` responses.

## Best Possible Solution

Normalize `agentMeta` onto the selected terminal snapshot only, preserve chat/agent collision ownership, and add focused coverage for cached, live dedupe, and lifecycle-first waits before merging.

Yes. Source inspection shows current main omits `agentMeta` from `agent.wait`, and the PR defects are reproducible from the selected-snapshot and lifecycle race paths without needing external services.

## Solution Assessment

No. The API idea is reasonable, but the implementation should carry telemetry through the selected snapshot source instead of re-reading raw agent dedupe metadata after ownership decisions have already been made.
