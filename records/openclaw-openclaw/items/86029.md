## Best Possible Solution

Keep the additive response shape, but source telemetry from a canonical terminal result path or make lifecycle snapshots carry the same normalized metadata, with focused tests for dedupe-first and lifecycle-first waits.

Yes from source inspection: lifecycle snapshots are built without `agentMeta`, while the PR aborts the dedupe wait when lifecycle wins and only checks the dedupe entry once. I did not run a live gateway repro because this review is read-only and the PR also lacks contributor runtime proof.

## Solution Assessment

No: the current patch is not the best mergeable form because it makes telemetry depend on which wait source wins. The maintainable fix is to normalize metadata once and prove both terminal snapshot paths return the same shape.
