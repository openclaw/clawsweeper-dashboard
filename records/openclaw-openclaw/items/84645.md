## Summary

Keep open: the PR is a focused node-host exec approval change that is not implemented on current main, but the proposed argv rewrite still changes supported inline-eval interpreter semantics and lacks real behavior proof.

## What This Changes

The branch materializes direct Python and Node inline eval commands into private state-dir script files before node-host exec approval planning, then dispatches the approved generated script argv.

## Best Possible Solution

Keep the materialization approach only if it faithfully preserves or explicitly rejects eval-only flags and argv behavior, validates the prepared `system.run` dispatch path, and includes redacted real node-host proof.

## Reproduction Assessment

Yes for the PR defect: the changed source rewrites inline eval argv, and local Node/Python probes show eval forms have semantics that are not preserved by a file-command rewrite.

## Solution Assessment

No. Materializing to a stable file is plausible, but this patch must first preserve or fail-close eval-only semantics and prove the real node-host path after approval.
