## Summary

Keep this PR open for maintainer review. The current head addresses the earlier stored-plan replay concern with targeted dispatch tests, and I did not find a concrete blocking bug, but the change intentionally expands the node-host exec approval boundary and should be accepted by an exec/security owner before merge.

## What This Changes

The PR materializes exact direct Python and Node inline eval commands into private deterministic script files, binds those scripts in node-host approval plans, and reuses the approved argv during dispatch.

## Best Possible Solution

Land the materialized-script approval path after exec/security owner acceptance, keeping generated scripts private and hash-bound and keeping ambiguous eval forms fail-closed.

## Reproduction Assessment

Yes. Current main source shows approval planning denies interpreter/runtime commands when it cannot bind a concrete local file operand; I did not run mutating prepare commands in this read-only review.

## Solution Assessment

Yes, subject to owner acceptance. Materializing exact inline eval into private hash-bound scripts fits the existing file-binding model better than approving raw inline code, and the PR keeps ambiguous forms fail-closed.
