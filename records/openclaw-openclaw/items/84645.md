## Summary

Keep open: the materialization direction is useful and the PR now has live proof, but the diff still misses the gateway-approved stored-plan replay path, so a venv interpreter shown in the inline-eval approval can be replaced by its realpath at execution.

## What This Changes

The branch materializes exact direct Python and Node inline eval commands into deterministic private script files before node-host system-run approval and adds approval/dispatch coverage.

## Best Possible Solution

Keep the materialization approach, but make the stored `systemRunPlan` carry or imply enough inline-eval context that direct node calls and gateway-approved replay preserve the approved interpreter path and script hash binding.

Yes from source inspection: gateway replay forwards the materialized plan argv and command text, while the PR's resolved-path pin mode only triggers when the parsed command text equals the original inline preview. I did not run a local mutating test in this read-only review.

## Solution Assessment
