## What This Changes

The PR materializes exact direct Python and Node inline-eval `system.run.prepare` commands into private generated scripts, binds those scripts as mutable file operands, and reuses the approval-bound argv during `system.run` dispatch.

## Best Possible Solution

Land only after maintainer security approval accepts the narrow generated-script approval model and the state-dir artifact lifecycle.

## Reproduction Assessment

Yes from source: current main still returns `SYSTEM_RUN_DENIED` when direct interpreter inline eval cannot be bound to a stable script operand, and the branch adds a generated-script binding path for exact Python/Node forms.

## Solution Assessment

Yes for the implementation shape, with a policy caveat: materializing the snippet to a private file and reusing the stored approval plan fits the existing `systemRunPlan` and mutable-file binding contract. The remaining question is whether maintainers accept the security-boundary and at-rest artifact model.
