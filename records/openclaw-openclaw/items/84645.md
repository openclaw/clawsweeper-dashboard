## What This Changes

The branch changes node-host `system.run.prepare` to materialize direct Python and Node inline eval commands into private state-dir scripts and adds focused approval-plan tests.

## Best Possible Solution

Keep the materialization approach only if it preserves or fail-closes eval-only interpreter semantics, adds dispatch-level coverage for prepared plan execution, and includes redacted real node-host proof.

## Reproduction Assessment

Yes for the PR defect: the changed source rewrites inline eval argv, and a Node 24 probe shows eval-only module input works with `--eval` but fails after the same option is carried to a file command.

## Solution Assessment

No: materializing to a stable file is a plausible direction, but this patch does not yet preserve supported inline-eval semantics or prove the real node-host path.
