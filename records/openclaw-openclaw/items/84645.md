## Summary

Keep open: the latest head is a focused node-host approval change, but it still has merge-blocking issues in a security-sensitive path and lacks real behavior proof. The patch can execute a forwarded approval-plan argv after only a preview-text match, and the supported inline eval forms still change Python/Node argv semantics when rewritten to script files.

## What This Changes

The PR materializes direct Python and Node inline eval commands into state-dir scripts for node-host approval planning/dispatch and adds focused approval-plan and dispatch tests.

## Best Possible Solution

Keep the materialization approach only after node-host derives or validates execution argv from the current materialized command, preserves or explicitly fail-closes eval argv semantics, and includes redacted real node-host proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the new branch executes approvalPlan.argv after a preview-only match, and local Python/Node probes show inline eval and script-file argv values differ. I did not run repository tests because this was a read-only review.

## Solution Assessment

No: materializing to stable scripts is a plausible direction, but this patch should bind execution to a freshly validated materialized argv and preserve or fail-close interpreter semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Validate inline plans before using their argv:** `src/node-host/invoke-system-run.ts:369`
  - body: This branch switches approved inline-eval execution to `approvalPlan.argv` when only `commandPreview` equals the requested command. The plan is normalized from the forwarded approval shape, and this code does not compare it with the freshly materialized argv or mutable-file binding, so a stale or mismatched plan can make approval for one inline command execute a different argv. Use the freshly materialized argv or require an exact plan/binding match before replacing `command.argv`.
  - confidence: 0.82
- **[P2] Preserve eval argv semantics when materializing:** `src/node-host/invoke-system-run-plan.ts:282`
  - body: The supported `python -c` and `node --eval` forms are rewritten to real script files, but those interpreters expose different argv values for inline eval versus script execution. Snippets that inspect `sys.argv` or `process.argv` therefore run differently from the approved command; adjust the wrapper or fail closed for forms whose semantics cannot be preserved.
  - confidence: 0.86
