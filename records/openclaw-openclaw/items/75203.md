## Summary

Keep open. Current main still exposes isolated heartbeat session rows as normal selectable sessions and sends Control UI WebChat input to the selected key; open PR #75356 is the active closing implementation candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to prevent Control UI WebChat from selecting or sending user messages into heartbeat-isolated `:heartbeat` sessions when the base `main:main` session has been pruned.

## Best Possible Solution

Land or replace the narrow marker-backed fix: project a heartbeat-isolated base-session marker through the gateway session-list contract, hide or normalize synthetic heartbeat rows in the Control UI picker, and make stale heartbeat selections route to the base session while preserving real user sessions that intentionally end in `:heartbeat`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable solution because it can use the stored heartbeat marker instead of suffix-only matching, normalize stale sends to the base session, and cover the UI/gateway behavior with focused tests.
