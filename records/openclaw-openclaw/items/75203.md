## Summary

Keep open. Current main still has the reported Control UI/session-routing failure shape, and open PR #75356 directly tracks the fix with closure linkage, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that Control UI WebChat can select and send user messages to an isolated `:heartbeat` session after the base `main:main` session is pruned.

## Best Possible Solution

Land or replace the narrow marker-backed fix: project `heartbeatIsolatedBaseSessionKey` or an equivalent synthetic-session flag through the gateway session-list contract, hide or normalize heartbeat-isolated rows in the Control UI picker, and guard `chat.send` so stale heartbeat selections route to the base session without changing real user sessions that merely end in `:heartbeat`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable solution because it uses the stored heartbeat-isolated marker and covers both picker visibility and stale send routing; the issue should not close until that or an equivalent fix is merged.
