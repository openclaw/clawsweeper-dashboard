## Summary

Keep open: current main still exposes the reported source path, and open PR #75356 remains the active unmerged fix candidate with closing syntax for this issue.

## What This Changes

The issue asks Control UI webchat to avoid selecting or sending user chat into isolated heartbeat sessions when the base `agent:main:main` session has been pruned.

## Best Possible Solution

Land or replace PR #75356 with a marker-backed fix that projects isolated heartbeat base keys, hides synthetic heartbeat rows from the picker, and normalizes stale heartbeat sends back to the base session.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: seed the session list with only `agent:main:main:heartbeat`, the picker can add that row, and webchat sends `state.sessionKey` directly to `chat.send`.

## Solution Assessment

Yes. The marker-backed direction in PR #75356 is the narrow maintainable solution because storage already distinguishes synthetic heartbeat rows without special-casing every key that ends in `:heartbeat`.
