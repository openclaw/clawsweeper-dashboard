## Summary

Keep open. Current main still allows heartbeat-isolated session rows to appear as ordinary Control UI session choices and still sends WebChat input to the selected session key; open PR #75356 is the active fix candidate and should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks Control UI/WebChat to avoid selecting or sending user input into heartbeat-isolated `:heartbeat` sessions when the base `main:main` session has been pruned.

## Best Possible Solution

Land or replace PR #75356 with the narrow marker-backed fix: project an explicit heartbeat-isolated base-session marker through the gateway session-list contract, hide or normalize those rows in the Control UI picker, and route stale heartbeat selections back to the base session without suffix-only matching.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable solution because it uses the stored heartbeat-isolated marker rather than treating every `:heartbeat` suffix as synthetic. Current main is not fixed yet.
