## Summary

Keep open. Current main still exposes isolated heartbeat session rows as ordinary selectable Control UI sessions and sends WebChat input to the selected key; PR #75356 is open, unmerged, and directly tracks the fix with closing syntax, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks for Control UI/WebChat to avoid showing or sending user input into isolated `:heartbeat` sessions when the base main session has been pruned.

## Best Possible Solution

Land or replace the narrow marker-backed fix: project `heartbeatIsolatedBaseSessionKey` or an equivalent synthetic-session flag through the gateway session-list contract, hide or normalize heartbeat-isolated rows in the Control UI picker, and guard `chat.send` so stale heartbeat selections route to the base session while preserving real user sessions that merely end in `:heartbeat`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable solution because it uses the stored heartbeat-isolated marker rather than suffix-only matching and covers both picker visibility and stale send routing. The issue should not close until that fix lands or an equivalent replacement is merged.
