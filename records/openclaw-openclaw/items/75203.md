## Summary

Keep open. Current main still allows the reported source-level path: heartbeat-isolated session rows can appear in the Control UI picker and WebChat sends to the selected session key, while open PR #75356 directly references this issue with closing syntax and is the active fix candidate.

## What This Changes

Prevent Control UI/WebChat from listing or sending user messages into isolated `:heartbeat` sessions when the base main session has been pruned.

## Best Possible Solution

Land or replace PR #75356 with the marker-backed fix that projects heartbeat base-session metadata, hides heartbeat-only rows from the picker, and normalizes stale heartbeat sends back to the base session.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes. The marker-backed direction in PR #75356 is the narrow maintainable solution because storage already distinguishes synthetic heartbeat rows and avoids treating every key ending in `:heartbeat` as special.
