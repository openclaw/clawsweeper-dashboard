## Summary

Keep open. Current main still has the reported source-level path, and the provided timeline shows open unmerged PR #75356 with closing syntax as the active fix candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Ensure Control UI/WebChat does not display or route user input into synthetic isolated heartbeat sessions when the base main session has been pruned.

## Best Possible Solution

Land or replace PR #75356 with a marker-backed fix that projects the base-session marker, hides heartbeat-only rows, and normalizes stale chat routing back to the base session without changing heartbeat execution or pruning policy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: after pruning agent:main:main so sessions.list returns only agent:main:main:heartbeat, current main adds that row to the picker and sends chat input with state.sessionKey.

## Solution Assessment

Yes for the active PR direction, not for current main. The marker-backed approach is the narrow maintainable fix because stored session metadata already distinguishes synthetic heartbeat rows without relying on a raw :heartbeat suffix.
