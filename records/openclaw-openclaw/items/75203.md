## Summary

Keep open. Current main still exposes isolated heartbeat session rows as normal selectable Control UI sessions and sends WebChat input to the selected key, while open PR #75356 directly tracks the fix with closing syntax.

## What This Changes

Control UI webchat should hide or normalize isolated heartbeat sessions and route stale heartbeat selections back to the primary session when the base session was pruned.

## Best Possible Solution

Land or replace PR #75356 with a marker-backed fix that projects isolated heartbeat metadata through the session-list contract, hides or normalizes heartbeat-only picker rows, and routes stale heartbeat selections back to the base session without suffix-only matching.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes. The marker-backed direction in PR #75356 is the narrow maintainable fix because the stored `heartbeatIsolatedBaseSessionKey` avoids breaking legitimate user session keys that happen to end in `:heartbeat`.
