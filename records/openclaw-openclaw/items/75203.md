## Summary

Keep open: current main still exposes the source-level routing path, and open PR #75356 is the unmerged closing fix candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that Control UI can select an isolated heartbeat session after the primary session is pruned and then send user webchat messages into that synthetic heartbeat session.

## Best Possible Solution

Land or replace PR #75356 with a marker-backed fix that exposes the isolated heartbeat base key, hides synthetic heartbeat rows from the picker, and normalizes stale heartbeat sends back to the base session.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: seed sessions.list with only agent:main:main:heartbeat, the picker adds that row, and webchat sends state.sessionKey directly to chat.send.

## Solution Assessment

Yes. The marker-backed direction is the narrow maintainable solution because storage already distinguishes heartbeat-created isolated rows via heartbeatIsolatedBaseSessionKey without treating every key ending in :heartbeat as special.
