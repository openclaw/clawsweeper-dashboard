## Summary

Keep open: current main still exposes the reported source-level path, and open PR #75356 remains the active unmerged fix candidate with closing syntax for this issue.

## What This Changes

The issue reports that Control UI webchat can select and send user input into an isolated `:heartbeat` session after the primary `agent:main:main` session row is pruned.

## Best Possible Solution

Land or replace PR #75356 with a marker-backed fix that projects the isolated heartbeat base key, hides synthetic heartbeat rows from the picker, and normalizes stale heartbeat sends back to the base session.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: seed `sessions.list` with only `agent:main:main:heartbeat`, the picker adds that row, and webchat sends `state.sessionKey` directly to `chat.send`.

## Solution Assessment

Yes. The marker-backed direction in PR #75356 is the narrow maintainable solution because storage already distinguishes synthetic heartbeat rows without treating every key ending in `:heartbeat` as special.
