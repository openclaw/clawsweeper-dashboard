## Summary

Keep open: current main still exposes the reported source-level path, and no related issue or PR appears to own the remaining Control UI webchat picker/send binding.

## What This Changes

The issue reports that Control UI webchat can select and send user messages into an isolated `:heartbeat` session after the primary main session is pruned.

## Best Possible Solution

Project the stored heartbeat-isolated marker through the gateway session-list contract, keep marked heartbeat-owned rows out of normal writable picker choices, and normalize marked heartbeat sends back to the base main session while preserving legitimate user sessions ending in `:heartbeat`.

## Reproduction Assessment

Yes, at source level: seed the session list with only a row keyed `agent:main:main:heartbeat`, and current main can offer that row in the picker and pass it unchanged to `chat.send`. I did not run a live browser/gateway reproduction in this read-only review.

## Solution Assessment

Yes. A marker-backed fix is the narrow maintainable path because storage already records `heartbeatIsolatedBaseSessionKey`, avoiding brittle suffix-only handling for real session keys that happen to end in `:heartbeat`.
