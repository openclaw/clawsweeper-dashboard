## Summary

Keep open. Current main at a1054fbe still exposes the reported source-level path, and open PR #75356 uses closing syntax for this issue and remains the active fix candidate.

## What This Changes

The issue requests preventing Control UI WebChat from selecting or sending user messages into heartbeat-isolated session rows when the base main session was pruned.

## Best Possible Solution

Land or replace PR #75356 with a marker-backed fix that projects heartbeat base-session metadata, hides heartbeat-only rows from the picker, and normalizes stale heartbeat sends back to the base session.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by listing only agent:main:main:heartbeat after the base row is pruned: current main adds that row to the picker and sends WebChat input with sessionKey: state.sessionKey.

## Solution Assessment

Yes. The marker-backed direction in PR #75356 is the narrow maintainable solution because storage already distinguishes synthetic heartbeat rows and avoids treating every key ending in :heartbeat as special.
