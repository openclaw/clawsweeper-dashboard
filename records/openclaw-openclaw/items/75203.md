## Summary

Keep open. Current main still exposes isolated heartbeat session rows as selectable Control UI sessions and sends WebChat input to the selected key, while PR #75356 is open and directly tracks the fix with closing syntax.

## What This Changes

Control UI WebChat should hide or normalize isolated `:heartbeat` sessions so user input routes to the primary base session after session pruning.

## Best Possible Solution

Land or replace PR #75356 with the marker-backed fix: project the isolated heartbeat base-session marker through the session-list contract, hide or normalize heartbeat-only rows in the picker, and route stale heartbeat selections back to the base session without changing normal user-created session keys.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes. The marker-backed direction in PR #75356 is the narrow maintainable fix because it uses stored heartbeat metadata rather than suffix-only matching and covers both picker visibility and stale send routing.
