## Summary

Keep open. Current main still allows the reported path: heartbeat-isolated session rows can be offered by the Control UI picker, and WebChat sends user input to the selected session key; open PR #75356 is the active closing fix candidate.

## What This Changes

Ensure Control UI/WebChat does not display or send user input into synthetic isolated heartbeat sessions when the base main session has been pruned.

## Best Possible Solution

Land or replace PR #75356 with the marker-backed fix that projects heartbeat base-session metadata, hides heartbeat-only rows from the picker, and normalizes stale heartbeat sends back to the base session.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists when sessions.list contains only agent:main:main:heartbeat after the base row is pruned: the picker adds that row and WebChat sends with state.sessionKey.

## Solution Assessment

Yes for the linked marker-backed direction. It uses the stored heartbeatIsolatedBaseSessionKey marker instead of suffix-only matching and avoids changing heartbeat creation or pruning policy.
