## Summary

Keep open: this PR remains useful and appears technically sound. Current main and the latest release still call MiniMax's redirected `api.*` OAuth endpoints, while the PR keeps the fix inside the MiniMax provider-owned OAuth path and adds focused global/CN endpoint coverage.

## What This Changes

The PR changes MiniMax global/CN OAuth device-code and token requests from `api.* /oauth/*` to `account.* /oauth2/*`, adds regression coverage, and adds a release-note entry.

## Best Possible Solution

Merge the scoped endpoint switch once required checks pass and maintainers accept or verify the MiniMax account-host OAuth flow while keeping post-login API resource URLs on the existing API hosts.

## Reproduction Assessment

Yes. Current source posts MiniMax OAuth forms to redirected `api.*` endpoints, live endpoint checks show those hosts now 307 to `account.*`, and guarded fetch strips cross-origin 307 bodies by default.

## Solution Assessment

Yes. Calling MiniMax's account OAuth endpoints directly is narrower than enabling unsafe cross-origin redirect replay and keeps the vendor-specific behavior inside the MiniMax plugin.
