## What This Changes

The PR changes embedded-agent model-call diagnostics to count common stream delta chunks by incremental delta bytes, skip full `partial` snapshots in fallback accounting, and add regression tests.

## Best Possible Solution

Land this as a documented, maintainer-accepted diagnostic accounting change while keeping the broader Windows local-provider starvation issue and companion mitigation PRs separate.

## Reproduction Assessment

Yes for the PR's narrow diagnostic contributor: current main serializes every stream chunk while stream types carry growing `partial` snapshots beside incremental `delta` strings. The full Windows local-provider starvation report was not reproduced here.

## Solution Assessment

Mostly yes: using the typed delta payload is the narrowest fix for repeated full-partial serialization. The remaining gap is documenting or explicitly accepting the changed diagnostic metric semantics before merge.
