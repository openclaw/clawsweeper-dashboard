## What This Changes

The branch changes embedded agent model-call diagnostics so streamed chunk byte counting omits growing `partial` snapshots, catches byte-inspection failures, and adds regression tests.

## Best Possible Solution

Land after aligning public docs and OTEL metric descriptions with the bounded `responseStreamBytes` semantics and after maintainers explicitly accept the metric rebaseline, while keeping the broader local-model starvation issue open.

## Reproduction Assessment

Yes, from source inspection: current main serializes every streamed chunk with `JSON.stringify`, while model stream deltas can carry growing `partial` assistant snapshots. I did not execute the test in this read-only review, but the PR adds focused regression coverage for that path.

## Solution Assessment

Mostly yes: excluding `partial` snapshots from byte accounting is a narrow fix for the diagnostics overhead. The missing piece is contract alignment and maintainer acceptance for the exposed metric semantic change.
