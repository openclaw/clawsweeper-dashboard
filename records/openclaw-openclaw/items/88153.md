## What This Changes

The branch changes embedded-agent model-call diagnostic accounting to omit accumulated `partial` snapshots from streamed chunk byte counts and adds focused regression tests for incremental delta counting and opaque chunks.

## Best Possible Solution

Ship the incremental counter with explicit docs/plugin-hook semantics, or add a separate incremental metric while preserving `responseStreamBytes`, and leave the broader local-provider starvation issue tracked separately.

## Reproduction Assessment

Yes. Current main passes each stream chunk to `utf8JsonByteLength(chunk)`, so a delta chunk carrying an accumulated `partial` snapshot serializes the full snapshot repeatedly; I did not rerun the Windows local-daemon reproduction.

## Solution Assessment

No, not yet as a merge-ready solution. The code path is narrow, but the exposed `responseStreamBytes` contract needs a docs/API decision before changing what plugin hooks and telemetry consumers see.
