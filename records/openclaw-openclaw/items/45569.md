## Summary

This should stay open because current `main` still source-reproduces the one-Enter `/status` problem, and an open fixing PR already references the issue with closing syntax but has unresolved review concerns about changing `/status` semantics.

## What This Changes

The issue reports that selecting `/status` from the web UI slash-command menu requires pressing Enter twice instead of executing on the first Enter.

## Best Possible Solution

Keep `/status` on the canonical status path and fix slash-menu selection so one Enter submits the selected no-argument command without rebuilding status output in the UI.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/status` is not `executeLocal`, and the Enter handler only sends selected no-arg slash commands when `executeLocal` is true.

## Solution Assessment

No for the currently linked PR direction as reviewed. A narrower slash-menu submit fix, or a call into the canonical status path, is safer than moving `/status` to a separate local UI formatter.
