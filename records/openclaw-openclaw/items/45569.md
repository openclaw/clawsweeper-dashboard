## Summary

Keep open. Current main still has the source-level two-Enter path for Control UI `/status`, and an open implementation PR already references this issue, so this should remain the tracker until that PR is landed, revised, replaced, or closed.

## What This Changes

The issue requests that selecting `/status` in the Control UI slash menu execute with one Enter instead of requiring a second Enter.

## Best Possible Solution

Preserve the canonical `/status` behavior and fix slash-menu selection so selecting the no-arg status command submits once, with focused regression coverage for the menu-selection flow.

## Reproduction Assessment

Yes by source inspection. On current main, `/status` is present in the menu but is not `executeLocal`, and the Enter handler only sends no-arg commands when that flag is true, so the first Enter completes the draft and a later Enter sends it.

## Solution Assessment

No for closing this issue yet. The narrow maintainable fix is to make the slash-menu selection submit `/status` once while preserving the existing status command path; the already-linked PR should be reviewed, revised, replaced, or closed first.
