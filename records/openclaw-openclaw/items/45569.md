The issue reports that selecting `/status` from the Control UI slash menu requires a second Enter press to submit.

## Best Possible Solution

## Solution Assessment

No for closing this issue now. The narrow maintainable fix is to submit the selected no-argument `/status` command once through the existing status path, not to close the tracker before the linked PR is resolved.
