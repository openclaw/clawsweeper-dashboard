## What This Changes

The issue reports that selecting `/status` from the Control UI slash menu requires a second Enter instead of executing immediately.

## Best Possible Solution

Review and land the focused replacement PR that makes Control UI `/status` execute locally through `session_status`, then close this issue after that fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows `/status` is in the slash menu but is not local-executable, and the Enter handler only submits selected no-argument commands when `executeLocal` is true.

## Solution Assessment

Yes for the linked replacement direction, but not for closing this issue yet. The maintainable fix is to make `/status` local while preserving the canonical `session_status` output path.
