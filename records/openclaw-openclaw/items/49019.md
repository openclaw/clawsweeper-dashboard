## Summary

Keep open: current main and v2026.5.22 still expose async, runId-only `/hooks/agent`, and the matching implementation remains open at https://github.com/openclaw/openclaw/pull/67433 rather than merged.

## What This Changes

The issue requests an opt-in `/hooks/agent` mode that waits for an isolated agent run and returns a deterministic machine-readable result while optionally suppressing main-session announcement.

## Best Possible Solution

Review, rebase, and either land or redirect https://github.com/openclaw/openclaw/pull/67433 with documented hook auth, session policy, retry/idempotency, timeout/error, result-shape, and announcement-suppression semantics.

## Solution Assessment

Unclear until Gateway API and security review completes. Extending `/hooks/agent` is a plausible narrow surface, but retry, timeout, error, trust, result, and announcement-suppression semantics should be settled in the linked PR.
