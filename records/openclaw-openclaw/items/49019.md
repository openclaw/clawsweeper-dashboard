## Summary

Keep open: current main and the latest release still expose POST /hooks/agent as an asynchronous hook ingress and do not provide the requested opt-in result-return or announce-suppression contract. The open implementation at https://github.com/openclaw/openclaw/pull/67433 is the canonical next item, but the API/security shape still needs maintainer review before this issue can close.

## What This Changes

The issue requests an opt-in wait/result mode for POST /hooks/agent, plus optional suppression of main-session announcements for machine callback workflows.

## Best Possible Solution

Adopt a reviewed opt-in hook result contract with focused gateway tests, docs, and explicit security semantics, or explicitly keep the supported workaround as /hooks/agent plus sessions_history polling.

## Reproduction Assessment

Yes for source-level behavior: current main normalizes POST /hooks/agent payloads without waitForResult, resultMode, or announceToMain and the handler returns only an async run id. I did not run a live gateway because the read-only source path and release tag both show the missing capability.

## Solution Assessment

Unclear pending maintainer decision: extending /hooks/agent is a plausible core seam because the existing endpoint already owns authenticated external agent ingress. The best next step is to review the linked PR’s API and security semantics rather than create a second implementation path.
