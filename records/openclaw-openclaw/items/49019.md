## Summary

Keep open: current main and v2026.6.6 still expose `/hooks/agent` as an async run-id-only endpoint, and the requested opt-in result-return mode is not implemented. The open implementation candidate is https://github.com/openclaw/openclaw/pull/67433, but the API/security shape still needs maintainer review before this issue can close.

## What This Changes

Request to add opt-in wait/result fields to `POST /hooks/agent` so external automations can receive a deterministic agent result without polling `sessions_history`.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/67433 as the canonical implementation path; land it with explicit security/session-result docs and tests if maintainers accept the API, otherwise close this feature as rejected.
