## Label Justifications

- `P2`: This is a normal-priority Gateway API feature with real automation impact but not an urgent regression in an existing workflow.
- `impact:session-state`: The requested mode changes how hook-triggered isolated agent runs expose session keys, results, and main-session announcements.
- `impact:security`: The feature touches authenticated external hook ingress and result disclosure without default-enabling broader `sessions_send` over HTTP.

## Summary

Keep open: current `main` and `v2026.5.18` still expose async, runId-only `POST /hooks/agent`; the active implementation is in https://github.com/openclaw/openclaw/pull/67433 and is currently open/conflicting, so this needs Gateway API and security review rather than cleanup closure.

## What This Changes

Add an opt-in synchronous result-return mode to `POST /hooks/agent` so external automations can wait for an isolated agent run and receive a deterministic result.

## Best Possible Solution

Review, rebase, and either land or redirect https://github.com/openclaw/openclaw/pull/67433 with a documented result-return contract that preserves hook auth, idempotency, timeout, error, trust, and announcement behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new Gateway API mode request. Source, docs, latest-release inspection, and the open implementation PR give high confidence that the requested mode is absent from current `main`.

## Solution Assessment

Unclear until Gateway API and security review completes. Extending `/hooks/agent` is a plausible narrow surface, but the final retry, timeout, error, trust, result, and announcement-suppression semantics should be settled in the linked implementation PR.
