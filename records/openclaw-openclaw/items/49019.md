## What This Changes

The issue requests an opt-in result-return mode for `POST /hooks/agent`, including synchronous result output and optional suppression of main-session announcements for machine callback workflows.

## Best Possible Solution

Review and either land or explicitly redirect https://github.com/openclaw/openclaw/pull/67433 with a documented `/hooks/agent` result-return contract covering response shape, retries, errors, timeouts, announcement suppression, and existing hook security boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new gateway API mode request. Source, docs, release, and test inspection give high confidence that current `main` and v2026.5.7 still expose only the async runId-returning hook behavior.

## Solution Assessment

Unclear until maintainer API review completes. Extending `/hooks/agent` is a plausible narrow surface for this use case, but the final retry, timeout, error, and documentation contract should be settled in the linked implementation PR.
