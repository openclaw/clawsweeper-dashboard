## Summary

Keep open: current main and the latest release still expose `POST /hooks/agent` as an async run-id-only endpoint, while the requested wait/result API is only present in the open implementation PR at https://github.com/openclaw/openclaw/pull/67433. ## What This Changes

The issue requests an opt-in synchronous result-return mode for `POST /hooks/agent`, including `waitForResult`, result text, and optional suppression of main-session announcements.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/67433 as the canonical implementation path, land it with docs/tests/security proof if maintainers accept the API, or reject/reshape that PR and then close or update this issue accordingly.
