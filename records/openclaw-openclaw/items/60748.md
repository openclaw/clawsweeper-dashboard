## Summary

Keep open. Current main still only suppresses the generic `main` Feishu card header; it does not provide the requested reply-to/custom/hidden header modes, elapsed/status footer rendering, footer hiding, or per-field footer toggles. Related #71079 is closed as a duplicate of this broader tracker, while open PR #54979 covers only the header-hide subset.

## What This Changes

Request Feishu card mode to expose configurable header and footer behavior, including hiding or replacing the agent label, showing reply-to context, rendering elapsed time and completion status, and hiding or toggling footer metadata.

## Best Possible Solution

Keep this issue as the canonical Feishu card customization tracker. The maintainable end state is a Feishu-plugin-owned, backward-compatible config surface for header mode and footer fields, wired through streaming and static card replies, with quoted-content handling reused from existing reply context logic, generated config metadata updated, docs added at `https://docs.openclaw.ai/channels/feishu`, and focused schema/dispatcher tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main’s Feishu card helpers are config-free, card paths always use those helpers, and tests still assert the hard-coded `Agent:` note while only covering the generic `main` header omission.

## Solution Assessment

Unclear for the exact option names, but yes on the implementation boundary: this belongs in the Feishu plugin rather than core. A header-only patch such as #54979 is useful but not sufficient for this broader issue unless maintainers intentionally split the footer and reply-to work into narrower follow-ups.
