## Summary

Keep open. Current main only omits the generic `main` Feishu card header; it still lacks reply-to/custom/hidden header modes, elapsed/status footer rendering, footer hiding, and per-field footer toggles. Related #71079 is already closed as a duplicate of this broader tracker, while open PR #54979 covers only the header-hide subset.

## What This Changes

Add Feishu card header and footer configuration for reply-to/custom/hidden headers, elapsed/status footer text, metadata toggles, and full header/footer hiding.

## Best Possible Solution

Keep this as the canonical Feishu card customization tracker, then implement a Feishu-plugin-owned backward-compatible config surface across streaming and static card replies with schema, generated metadata, docs, and focused dispatcher/outbound tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main’s Feishu card helpers are config-free, card reply paths call those helpers directly, and tests still assert the hard-coded note behavior.

## Solution Assessment

Unclear for the exact option names, but the implementation boundary is clear: this belongs in the Feishu plugin rather than core. A header-only patch such as #54979 is useful but not sufficient for this broader request unless maintainers intentionally split footer and reply-to work into separate follow-ups.
