## Summary

Keep open. Current main only omits the generic `main` Feishu card header; it still lacks the requested reply-to/custom/hidden header modes, elapsed/status footer rendering, footer hiding, and per-field footer toggles, while open PR #54979 covers only a header-hide subset.

## What This Changes

Request Feishu card-mode controls to replace or hide the agent label, show elapsed time and completion status in the footer, and surface reply-to context.

## Best Possible Solution

Keep this as the canonical Feishu card customization tracker, then implement a Feishu-plugin-owned, backward-compatible card presentation config across streaming and static replies with schema, generated metadata, docs, and focused tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: `renderMode: "card"` replies call fixed header/footer helpers, and the schema/docs expose no requested card presentation controls.

## Solution Assessment

Unclear for the exact option names, but the implementation boundary is clear: this belongs in the Feishu plugin. Header-only PR #54979 is useful but not sufficient for this broader request unless maintainers intentionally split the footer and reply-to work into follow-ups.
