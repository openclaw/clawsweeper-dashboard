## Summary

Keep open. Current main still only suppresses the unconfigured generic `main` card header; it does not provide reply-to/custom/hidden header modes, elapsed/status footer rendering, footer hiding, or per-field footer controls. The related cleanup chain points to this issue as the canonical tracker, while open PR https://github.com/openclaw/openclaw/pull/54979 covers only the narrower header-hide subset.

## What This Changes

Request Feishu card customization for header modes, elapsed/status footer rendering, footer metadata toggles, and reply-to context.

## Best Possible Solution

Keep this as the canonical Feishu card header/footer customization tracker, then implement a Feishu-plugin-owned backward-compatible config surface across streaming and static cards with schema, generated metadata, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main’s Feishu card helpers and card reply paths are config-free for the requested controls, and tests still assert fixed Agent/Model/Provider note behavior; no live Feishu send was run in this read-only review.

## Solution Assessment

Unclear for the exact option names, but the owner boundary is clear: this belongs in the Feishu plugin. A header-only patch such as https://github.com/openclaw/openclaw/pull/54979 is useful but insufficient unless maintainers intentionally split footer/status/reply-to work into separate follow-ups.
