## Summary

Keep this issue open: current main and the latest release still allow stale finalizer handoff env values to survive, and the linked open PR is the live fix candidate rather than a reason to close before merge.

## What This Changes

The issue reports that the RPC post-core finalizer child can inherit stale source-config and compatibility-host env values when the current update run has no fresh handoff payload.

## Best Possible Solution

Review and land the linked narrow finalizer-env isolation fix, or an equivalent change that clears stale handoff vars before applying the current update context, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows current main copies the parent env into the finalizer child, and `update finalize` reads the inherited source-config path before falling back to the current config snapshot; I did not run the script because this review is read-only.

## Solution Assessment

Yes. Clearing internal handoff env vars in the finalizer env builder before setting this run's values is the narrow owner-boundary fix, and the linked PR implements that path.
