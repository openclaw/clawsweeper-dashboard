## Summary

Keep this issue open: current main and the latest release still allow stale post-core finalizer handoff env vars to leak, and the linked open PR is the candidate fix rather than implementation evidence.

## What This Changes

The issue asks OpenClaw to prevent RPC post-core finalization from inheriting stale source-config and compatibility-host environment variables when the current update run has no fresh values.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/94560; once that PR merges, this issue can close as implemented by the merge commit.

## Reproduction Assessment

Yes. Source inspection shows current main copies the parent environment and only conditionally overwrites fresh handoff values, while update finalize reads the source-config env var before fallback; I did not run the provided script because this review is read-only.

## Solution Assessment

Yes. Clearing only the internal finalizer handoff env vars before adding current-run values is the narrow maintainable fix, and the linked open PR targets that boundary.
