## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows current main copies the parent env into the finalizer child while `update finalize` reads the source-config path from env before fallback; I did not run the script because this cleanup review is read-only.

## Solution Assessment

Yes. Clearing these internal handoff env vars in `buildFinalizeEnv` before setting current-run values is the narrow owner-boundary fix, and the linked PR implements that approach.
