## What This Changes

The issue requests isolating the RPC post-core finalizer child environment so stale source-config and compatibility-host values are not inherited from the gateway process.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/94560 so `buildFinalizeEnv` clears stale handoff vars before applying current-run source-config and host-version context.

## Reproduction Assessment

Yes. Source inspection shows the finalizer child env inherits the parent env and `update finalize` reads the inherited source-config path before fallback; I did not run the provided script to avoid writing artifacts in this read-only review.

## Solution Assessment

Yes. The linked PR’s approach of clearing stale handoff env in `buildFinalizeEnv` before applying this run’s values is the narrow owner-boundary fix; filtering later in `update finalize` would leave the child env contract ambiguous.
