The issue reports that RPC update finalization can inherit stale source-config and compatibility-host environment values when the current update run has no fresh handoff values.

Yes. Clearing only the internal finalizer handoff env vars in buildFinalizeEnv before applying this run's values is the narrow owner-boundary fix, and the linked PR targets that boundary.
