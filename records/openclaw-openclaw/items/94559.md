Keep this open: current main and the latest release still inherit stale finalizer handoff env values, and the linked open PR is the active fix candidate rather than a reason to close the issue before merge.

## What This Changes

The issue reports that RPC post-core finalizer children can inherit stale source-config and compatibility-host env values from the gateway process.

## Best Possible Solution

Keep this issue open as the canonical bug and resolve it by reviewing and landing https://github.com/openclaw/openclaw/pull/94560 or an equivalent finalizer-env isolation fix.

## Reproduction Assessment

Yes, source-reproducible: current main copies parent env into the finalizer child and `update finalize` reads the inherited source-config path before falling back to the current config snapshot. I did not execute the repro because this review is read-only and the source path directly shows the leak.

## Solution Assessment

Yes, the narrow fix is to clear internal finalizer handoff env vars before applying the current run's source-config path and host version. The issue should remain open because that fix is only in an open PR, not on main or in the latest release.
