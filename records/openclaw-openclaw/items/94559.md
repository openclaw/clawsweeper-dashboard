## Summary

Keep this issue open: current main still allows stale post-core finalizer handoff env vars to reach the child process, latest release v2026.6.9 has the same behavior, and https://github.com/openclaw/openclaw/pull/94560 is still open as the linked fix candidate.

## What This Changes

The issue reports that RPC post-core finalization can inherit stale source-config and compatibility-host env vars from the gateway process when the current update run has no fresh handoff values.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/94560; after it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Source inspection shows current main copies the parent env into the finalizer child and update finalize reads the inherited source-config path before fallback; I did not execute the script in this read-only review.

## Solution Assessment

Yes. Clearing only the internal finalizer handoff env vars in buildFinalizeEnv before applying this run's values is the narrow owner-boundary fix, and the linked PR targets that boundary.
