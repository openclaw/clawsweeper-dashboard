The issue asks to isolate the RPC post-core finalizer child environment so stale inherited source-config and compatibility-host vars are not forwarded when the current update run lacks fresh values.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94560, or an equivalent narrow patch that clears finalizer handoff env vars before setting this run's values.

## Reproduction Assessment

Yes. The source path is high-confidence: current main copies parent env into the finalizer child, the RPC path may omit a fresh pre-update config, and the finalizer consumes the source-config env path before fallback.

## Solution Assessment
