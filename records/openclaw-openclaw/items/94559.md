## Summary

Keep this issue open: current main and the latest release still allow stale post-core finalizer handoff env values to survive, and the linked open PR is the candidate fix rather than close evidence.

## What This Changes

The issue reports that RPC-triggered post-core finalization can inherit stale source-config and compatibility-host environment values when the current update run has no fresh payload.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94560, or an equivalent patch that clears stale finalizer handoff env vars before applying this run's values.

## Reproduction Assessment

Yes. Source inspection shows current main copies the parent env into the finalizer child, the RPC path can omit a fresh pre-update config, and the finalizer reads the source-config env path before falling back.

## Solution Assessment

Yes. Clearing stale internal handoff variables in `buildFinalizeEnv` before applying this run's values is the narrow owner-boundary fix; the linked PR is the current candidate for that solution.
