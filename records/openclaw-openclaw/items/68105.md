## Summary

Keep open. Current main and the latest release still keep RTL isolation inside the TUI formatter only, while non-TUI outbound delivery sanitizers continue to return/send RTL text without shared bidi isolation; the open closing PR should be reviewed, rebased, landed, or replaced before this issue closes.

## What This Changes

Outbound non-TUI assistant replies should apply RTL bidi isolation like the TUI so Hebrew/Arabic punctuation renders correctly in messaging channels.

## Best Possible Solution

Land or replace the linked outbound RTL isolation PR so the shared delivery sanitizer wraps RTL lines while history/internal profiles remain byte-clean.

## Reproduction Assessment

Yes. Source inspection shows TUI output wraps RTL lines with RLI/PDI while shared delivery and plain-text outbound sanitizers used by Discord, WhatsApp, and direct channel delivery do not.

## Solution Assessment

Yes. Extracting the existing TUI RTL isolation helper into a shared text module and applying it only at the delivery profile is the narrow maintainable fix; the linked open PR appears to target that seam but must be reviewed/rebased before the issue closes.
