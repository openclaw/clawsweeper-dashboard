## Summary

Keep open: this targets a real OpenRouter image-tool attribution gap, but the PR diff is too broad because it applies attribution from provider id alone and it still needs real behavior proof before merge.

## What This Changes

The PR adds resolved provider attribution headers to the generic image-understanding `complete()` call.

## Best Possible Solution

Use the centralized provider request policy for image completions so attribution is based on provider, API, base URL, transport, and capability, then cover OpenRouter plus non-native OpenAI-compatible routes with regression tests and redacted real OpenRouter proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: the generic image runtime calls `complete()` without OpenRouter attribution while current policy already knows how to gate attribution by endpoint. I did not run a live OpenRouter billing/free-tier check in this read-only review.

## Solution Assessment

No. Passing `resolveProviderAttributionHeaders(model.provider)` directly is not the best fix because it bypasses existing route gating; the safer path is to resolve request-policy headers using the model API/base URL and media-understanding capability.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate image attribution by request route:** `src/media-understanding/providers/image.ts:189-201`
  - body: This resolves attribution from `model.provider` alone, so `openai` or `openai-codex` image models configured with a non-native OpenAI-compatible `baseUrl` would receive hidden `originator`/`User-Agent` headers. The stream path already gates those headers to verified native hosts; use the provider request policy with API/baseUrl/capability, or an OpenRouter-only helper, before passing headers here.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff changes outbound provider headers and can send attribution headers to routes that current policy intentionally gates by endpoint.

Concerns:

- **[medium] Gate attribution headers by endpoint:** `src/media-understanding/providers/image.ts:201`
  - body: Provider-id-only attribution can disclose hidden OpenAI/OpenAI-Codex attribution headers, or OpenRouter app headers, to custom proxy routes that are not verified for that header contract. Keep the policy check tied to the resolved base URL before emitting these headers.
  - confidence: 0.86
