## Summary

Keep open. The reported bug is still present on current main, but this PR should not merge as-is because it patches an obsolete image-understanding path and uses provider-only attribution instead of endpoint-aware request-policy headers.

## What This Changes

The branch imports `resolveProviderAttributionHeaders` into `src/media-understanding/providers/image.ts` and passes provider-only attribution headers into that file's image `complete()` call.

## Best Possible Solution

Rework the fix in `src/media-understanding/image.ts` so both image completion attempts pass endpoint-aware request headers for the final resolved model, with regression tests for native OpenRouter attribution and custom base URL non-injection.
