## What This Changes

The PR imports `resolveProviderAttributionHeaders` into the old media-understanding provider image file and forwards those headers to the direct image `complete()` call.

## Best Possible Solution

Replace this stale patch with a focused current-main fix in the live image completion helper that uses endpoint-aware request policy headers and adds regression coverage plus a changelog entry.

Yes, by source inspection. Current main reaches the image `complete()` call without headers, while the shared policy/test surface shows OpenRouter attribution must be applied only through endpoint-aware request policy.

## Solution Assessment

No. The PR intent is right, but the implementation targets a removed file and resolves attribution from provider identity alone; the safer fix belongs in `src/media-understanding/image.ts` using provider/API/base URL/capability/transport policy.
