## Summary

Keep open. Current main and the latest release still lack first-class Batch API routing for delayed/background agent turns, and the related closed reports are duplicate context rather than shipped fixes.

## What This Changes

Requested support for routing non-real-time background agent turns through provider Batch APIs for cron jobs, subagents, heartbeats, and other async work.

## Best Possible Solution

Define an additive delayed/batch agent-turn contract in the provider runtime and public Plugin SDK, then add explicit cron/session/subagent/heartbeat opt-ins plus provider implementations and docs for polling, cancellation, fallback, accounting, and delivery semantics.
