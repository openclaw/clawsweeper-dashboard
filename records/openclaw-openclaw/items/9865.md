## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the discussion plus related closed reports still identify this issue as the canonical product/API tracker rather than a shipped fix or duplicate.

## What This Changes

Add provider Batch API support for async/background agent work so cron jobs, sub-agents, and heartbeat-style tasks can trade latency for lower provider cost.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract across provider runtime/SDK, protocol opt-ins, cron/sessions/subagents/heartbeat delivery, provider implementations, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high-confidence proof that current main has no delayed/batch agent-turn opt-in in the protocol schemas, CLI payload builder, provider runtime contract, or cron/session/subagent execution paths.

## Solution Assessment

Yes as a canonical tracker, but not as a one-shot repair. The narrowest maintainable path is a reviewed provider/runtime and Plugin SDK contract before provider-specific OpenAI or Anthropic batch routing is added.
