## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the existing discussion already treats this as the canonical product/API tracker rather than a shipped fix or ClawHub-only plugin request.

## What This Changes

The issue requests Batch API support for delayed background agent work such as cron jobs, subagents, and heartbeat checks.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract across provider runtime, public Plugin SDK, protocol/config opt-ins, provider implementations, and documented async delivery semantics.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: strict cron/session/subagent schemas and stream-oriented provider hooks show no batch opt-in or delayed-turn lifecycle for agent turns.

## Solution Assessment

Unclear until maintainers choose the contract. The maintainable path is a shared provider/runtime and Plugin SDK seam first, not a provider-specific cron flag that bypasses cancellation, fallback, accounting, and delayed delivery behavior.
