## Summary

Keep open. Current main and the latest release still lack first-class provider Batch API routing for delayed/background agent turns, and the issue remains the canonical product/API tracker rather than a duplicate or shipped fix.

## What This Changes

Request support for routing non-real-time background agent work through provider Batch APIs for lower cost and separate async execution semantics.

## Best Possible Solution

Define an additive delayed/batch agent-turn contract in the provider runtime and public Plugin SDK first, then add explicit cron/session/subagent/heartbeat opt-ins plus OpenAI and Anthropic implementations with documented polling, cancellation, fallback, accounting, and delivery semantics.

## Reproduction Assessment

Not applicable. This is a feature/API request, and current-main source inspection shows the requested delayed routing fields and provider lifecycle hooks are absent rather than exposing a failing existing contract.

## Solution Assessment

Yes for keeping it open as a design tracker; no for a one-shot automated patch. The maintainable path is a reviewed additive provider/runtime and SDK contract before adding cron, sessions, subagent, heartbeat, and provider-specific behavior.
