## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the live discussion already treats this as the canonical tracker rather than a duplicate or shipped fix.

## What This Changes

Add Batch API support for background agent work such as cron jobs, subagents, scheduled maintenance, and heartbeat checks.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract, then implement documented opt-ins across cron, sessions_send, sessions_spawn/subagents, heartbeat/background runs, and provider plugins.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that cron, sessions_send, sessions_spawn, and heartbeat/background agent turns have no Batch API opt-in path today.

## Solution Assessment

No, a one-off provider or cron-only patch is not the best solution yet. The maintainable path is a reviewed additive provider/runtime and Plugin SDK contract before wiring OpenAI or Anthropic Batch API adapters into background agent turns.
