## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the live discussion treats this as the canonical design tracker rather than an implemented or duplicate cleanup item.

## What This Changes

Add opt-in provider Batch API routing for non-real-time background work such as cron agent turns, subagents, heartbeats, and scheduled maintenance.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract, then implement documented opt-ins across cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs with provider support.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat agent turns.

## Solution Assessment

No one-shot repair is the best path yet. The maintainable solution is a design-reviewed additive runtime/API contract before provider-specific OpenAI or Anthropic batch adapters are implemented.
