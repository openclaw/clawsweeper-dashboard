## Summary

Keep open. Current main and v2026.6.10 still lack first-class provider Batch API routing for delayed/background agent turns, and the existing related reports are closed duplicate context rather than shipped fixes.

## What This Changes

Requests opt-in provider Batch API routing for non-real-time background agent work such as cron jobs, subagents, and heartbeat checks to reduce provider cost at acceptable latency.

## Best Possible Solution

Define an additive delayed/batch agent-turn lifecycle shared by provider runtime and the Plugin SDK, then expose explicit background opt-ins with provider implementations, docs, accounting, cancellation, fallback, and delivery semantics.
