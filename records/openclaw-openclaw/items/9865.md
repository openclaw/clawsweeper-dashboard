## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the live discussion plus related closed reports confirm this issue is the canonical product/API tracker rather than a shipped fix or duplicate.

## What This Changes

Request to add provider Batch API routing for non-real-time background LLM workloads such as cron jobs, heartbeats, scheduled maintenance, and spawned subagents.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract, then add explicit cron, sessions_send, sessions_spawn/subagent, and heartbeat/background opt-ins with provider implementations and docs once the contract is agreed.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection does establish the missing current-main capability because protocol schemas, CLI payload construction, provider runtime hooks, and cron/session execution paths have no delayed batch-turn opt-in.

## Solution Assessment

Yes as a canonical tracker, but not as a one-shot automated implementation. The narrowest maintainable path is a reviewed provider/runtime and Plugin SDK contract before OpenAI or Anthropic batch routing is added.
