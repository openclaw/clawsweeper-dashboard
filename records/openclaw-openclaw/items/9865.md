Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the linked discussion/search context supports this as the canonical design tracker rather than a duplicate, ClawHub-only item, or implemented feature.

## What This Changes

Add Batch API routing for delayed/background agent turns such as cron jobs, subagents, heartbeats, and scheduled maintenance.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract, then implement documented opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs with provider support.

Not applicable as a feature request. Source inspection gives high confidence that current main has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat agent turns.

## Solution Assessment

No one-shot repair is the best path yet. The maintainable solution is a design-reviewed additive runtime/API contract before provider-specific OpenAI or Anthropic batch adapters are implemented.
