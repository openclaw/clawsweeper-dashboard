## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the live discussion/search context supports this issue as the canonical design tracker rather than an implemented, duplicate, or ClawHub-only cleanup item.

## What This Changes

Add provider Batch API support for delayed/background LLM agent turns, with opt-ins for cron jobs, subagents, heartbeats, and other non-real-time work.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract, then implement documented opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs with provider support.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat agent turns.

## Solution Assessment

No one-shot repair is the best path yet. The maintainable solution is a design-reviewed additive runtime/API contract before provider-specific OpenAI or Anthropic batch adapters are implemented.
