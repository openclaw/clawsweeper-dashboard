## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the issue discussion already treats this as the canonical design tracker rather than a shipped fix, duplicate, or plugin-only item.

## What This Changes

Request support for routing non-real-time background agent work through provider Batch APIs with cron, subagent, heartbeat, or config opt-ins.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract, then implement documented opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs with OpenAI and Anthropic provider support.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat agent turns.

## Solution Assessment

No, not as a one-shot repair. The maintainable solution is a design-reviewed additive runtime/API contract before provider-specific OpenAI or Anthropic batch adapters are implemented.
