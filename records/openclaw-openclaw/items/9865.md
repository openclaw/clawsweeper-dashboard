## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the closed related reports are duplicate context rather than shipped fixes.

## What This Changes

Request support for routing non-real-time background agent work such as cron jobs, subagents, and heartbeats through provider Batch APIs for lower cost at higher latency.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract across provider runtime/SDK, protocol/config opt-ins, cron/sessions/subagents/heartbeat delivery, provider implementations, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high-confidence evidence that cron, sessions, subagents, heartbeats, and provider runtime contracts have no delayed/batch agent-turn path.

## Solution Assessment

Yes as a canonical tracker, but no as a one-shot repair. The maintainable path is a reviewed provider/runtime and public Plugin SDK contract before provider-specific OpenAI or Anthropic batch routing is added.
