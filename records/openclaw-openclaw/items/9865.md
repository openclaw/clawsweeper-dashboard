## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the existing related reports are useful duplicate context rather than shipped fixes.

## What This Changes

Add opt-in provider Batch API routing for non-real-time background work such as cron agent turns, subagents, and heartbeats.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract with documented cron, sessions_send, sessions_spawn/subagent, and heartbeat opt-ins plus OpenAI and Anthropic provider implementations.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat agent turns.

## Solution Assessment

No one-shot repair is the best path yet. The maintainable solution is a design-reviewed additive runtime/API contract before provider-specific OpenAI or Anthropic batch adapters are implemented.
