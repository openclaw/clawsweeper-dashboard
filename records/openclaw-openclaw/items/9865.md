The issue requests opt-in provider Batch API routing for non-real-time background work such as cron jobs, subagents, heartbeats, and scheduled maintenance tasks.

## Best Possible Solution

Keep this as the canonical tracker for an additive delayed/batch agent-turn contract across provider runtime/SDK, protocol/config opt-ins, cron/sessions/subagents/heartbeat delivery, provider implementations, and docs.
