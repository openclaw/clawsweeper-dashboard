## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the related closed reports are narrower duplicate context rather than shipped fixes.

## What This Changes

Request Batch API support for non-real-time background workloads such as cron jobs, subagents, and heartbeat checks.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract with provider implementations and documented cron, session, subagent, and heartbeat opt-ins.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat agent turns.

## Solution Assessment

No one-shot repair is the best path yet. The maintainable solution is a design-reviewed additive runtime/API contract before provider-specific OpenAI or Anthropic batch adapters are implemented.
