## Summary

Keep open. Current main and the latest release still lack first-class provider Batch API routing for delayed/background agent turns, and the related closed reports are narrower duplicate context rather than shipped fixes.

## What This Changes

Request support for provider Batch APIs for non-real-time OpenClaw background work such as cron, heartbeat, scheduled maintenance, and sub-agent runs.

## Best Possible Solution

Define an additive delayed/batch agent-turn lifecycle shared by provider runtime and the Plugin SDK, then expose explicit background opt-ins with provider implementations, docs, accounting, cancellation, fallback, and delivery semantics.

## Reproduction Assessment

Not applicable: this is a feature request for a new delayed provider execution mode, not a broken existing behavior. Source inspection gives high-confidence evidence that current main and v2026.6.8 have no such opt-in or lifecycle.

## Solution Assessment

No, a simple cron-only or provider-local batch flag is not the best complete solution. The maintainable path is a generic delayed-turn provider and Plugin SDK contract shared by cron, heartbeat, sessions, and subagents.
