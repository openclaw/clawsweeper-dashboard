## Summary

Keep open. Current `main` still does not provide first-class Batch API routing for delayed/background agent turns, and the issue remains a concrete enhancement that needs maintainer API/product design rather than cleanup closure.

## What This Changes

Request support for provider Batch APIs on async/background OpenClaw workloads such as cron jobs, subagents, scheduled maintenance, and heartbeat checks.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract, then implement documented opt-ins across cron, sessions_send, sessions_spawn/subagents, heartbeat/background runs, and provider plugins.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current `main` has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat/background agent turns.

## Solution Assessment

No, a one-shot implementation is not the best path yet. The maintainable solution needs design review for an additive runtime/API contract before provider-specific OpenAI or Anthropic Batch API adapters are implemented.
