## Summary

Keep open. Current main still lacks first-class delayed or batch model-turn routing for background agent work, and the live discussion already treats this as the canonical product/API tracker rather than a cleanup close.

## What This Changes

Request support for routing non-real-time background agent tasks, such as cron jobs, subagents, and heartbeats, through provider Batch APIs to trade latency for lower cost.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract spanning provider runtime and Plugin SDK support, protocol/config opt-ins, cron/sessions/subagents/heartbeat delivery, provider implementations, and docs for fallback, cancellation, accounting, and delayed result delivery.

## Solution Assessment

Yes as a canonical tracker, but not as a one-shot repair. The maintainable path is a reviewed delayed-turn provider/runtime and public Plugin SDK contract before provider-specific OpenAI or Anthropic batch routing is added.
