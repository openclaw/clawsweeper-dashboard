## Summary

Keep open. Current main and the latest release still lack first-class Batch API routing for delayed/background agent turns, and the live discussion plus duplicate-search pass support keeping this as the canonical product/API tracker.

## What This Changes

Add opt-in provider Batch API support for non-real-time background work such as cron jobs, sub-agents, heartbeats, and scheduled maintenance.

## Best Possible Solution

Define an additive delayed/batch agent-turn contract in the provider runtime and public Plugin SDK, then add explicit cron/session/subagent/heartbeat opt-ins plus provider implementations and docs for polling, cancellation, fallback, accounting, and delivery semantics.
