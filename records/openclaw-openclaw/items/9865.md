## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the discussion/search context supports this as the canonical design tracker rather than a shipped fix or duplicate.

## What This Changes

Request support for provider Batch APIs for non-real-time background agent turns, including cron jobs, subagents, heartbeats, and scheduled maintenance work.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract, then implement documented opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs with OpenAI and Anthropic provider support.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no batch opt-in path for cron, sessions_send, sessions_spawn, or heartbeat agent turns.

## Solution Assessment

No one-shot repair is the best path yet. The maintainable solution is a design-reviewed additive runtime/API contract before provider-specific OpenAI or Anthropic batch adapters are implemented.
