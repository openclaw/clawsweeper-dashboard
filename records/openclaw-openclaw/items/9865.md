## Summary

Keep open: current main still lacks first-class delayed or provider Batch API routing for background agent turns, and the live discussion/search context treats this as the canonical design tracker rather than something already shipped or superseded.

## What This Changes

Add opt-in provider Batch API routing for latency-tolerant background agent work such as cron jobs, subagents, and heartbeat checks.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract, then implement documented protocol/config opt-ins and provider adapters after the runtime and SDK contract is agreed.

## Reproduction Assessment

Not applicable; this is an enhancement request rather than broken existing behavior. Source inspection is high-confidence that current main has no accepted batch opt-in for cron, sessions_send, or sessions_spawn agent turns.

## Solution Assessment

No, not as a one-off cron flag or provider adapter. The maintainable route is a reviewed additive provider-runtime, public Plugin SDK, and gateway protocol contract before OpenAI or Anthropic Batch API execution is wired in.
