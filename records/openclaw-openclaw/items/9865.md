## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the existing discussion plus closed related reports point to this issue as the canonical product/API tracker rather than a shipped or duplicate item.

## What This Changes

Request Batch API support for delayed/background LLM work such as cron jobs, subagents, heartbeats, and scheduled maintenance.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract, then add explicit cron, sessions_send, sessions_spawn/subagent, and heartbeat/background opt-ins plus OpenAI/Anthropic implementations and docs once the contract is agreed.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection does establish the missing current-main capability because the protocol schemas, CLI payload builder, provider runtime contract, and cron/session execution paths have no delayed batch-turn opt-in.

## Solution Assessment

Yes as a canonical product/API tracker, but not as a one-shot automated fix. The narrowest maintainable solution is a reviewed provider/runtime and Plugin SDK contract before provider-specific OpenAI or Anthropic batch routing is added.
