## Summary

Keep open. Current main still lacks first-class delayed/batch routing for background agent turns, and prior discussion treats this as the canonical tracker rather than a shipped or plugin-only item.

## What This Changes

Request support for provider Batch APIs on non-real-time background work such as cron jobs, subagents, and heartbeat-style tasks.

## Best Possible Solution

Keep this as the canonical design tracker for an additive delayed/batch agent-turn contract with documented cron, sessions_send, sessions_spawn/subagent, and heartbeat/background opt-ins plus provider implementations.

## Solution Assessment
