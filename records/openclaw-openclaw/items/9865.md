Keep this as the canonical tracker for an additive delayed/batch agent-turn contract across provider runtime/SDK, protocol/config opt-ins, cron/sessions/subagents/heartbeat delivery, provider implementations, and docs.

## Solution Assessment

Yes as a canonical tracker, but no as a one-shot repair. The maintainable path is a reviewed provider/runtime and public Plugin SDK contract before provider-specific OpenAI or Anthropic batch routing is added.
