## Summary

Keep open. Current main and the latest release still lack first-class Batch API routing for delayed/background agent turns; existing batch support is limited to memory/media-style provider paths, and related reports are closed duplicate/provider-specific context rather than shipped fixes.

## What This Changes

The issue requests Batch API support for non-real-time background agent work such as cron jobs, subagents, scheduled maintenance, and heartbeat checks.

## Best Possible Solution

Define an additive delayed/batch agent-turn contract in the provider runtime and public Plugin SDK, then add explicit cron/session/subagent/heartbeat opt-ins plus OpenAI and Anthropic implementations and docs.
