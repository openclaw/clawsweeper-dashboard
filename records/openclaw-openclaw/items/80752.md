## Summary

Keep open: current main and the latest release still do not expose a commitments-specific model override, and implementing one would add a new public config/model-routing contract that needs maintainer approval.

## What This Changes

The issue requests optional `commitments.model` and possible fallback config so hidden commitment extraction can use a different model than the main agent.

## Best Possible Solution

Make a maintainer decision on the commitments model-routing contract, then implement the chosen schema, resolver, docs/help, and focused regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main and `v2026.5.12` confirms that `commitments.model` is not accepted and extraction inherits the agent/default model.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback and fast-mode semantics should be decided before implementation.
