## Summary

Keep open: current main still does not expose a commitments-specific model override, and the request would add a new public config/model-routing feature that needs maintainer judgment around exact override and fallback semantics.

## What This Changes

Add optional commitments model-routing config so hidden commitment extraction can use a model different from the owning agent's default.

## Best Possible Solution

Decide the commitments model-routing contract, then add the schema/help/docs/tests and resolver/runtime behavior while preserving current agent-default inheritance when unset.

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main has no commitments.model key and that hidden extraction inherits the agent default through resolveCommitmentDefaultModelRef.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback behavior and interaction with fast-mode/model fallback need approval first.
