## Summary

Keep open: current main still lacks a commitments-specific model override, and the request is a concrete new public config/model-routing feature rather than an already-implemented cleanup. The issue is supported by current source, docs, adjacent active-memory/compaction precedent, an open related fast-mode PR that does not cover this request, and a follow-up user timeout report.

## What This Changes

Add an optional commitments extraction model override, and possibly fallback semantics, so background commitments can use a different model from the main agent.

## Best Possible Solution

Decide the commitments model-routing contract, then add the config type/schema/help/docs plus resolver and focused regression coverage for override precedence, default inheritance, and interaction with the fast-mode path.

## Reproduction Assessment

Source-level yes: current main has no `commitments.model` config key and the extractor inherits the resolved agent/default model. This is not a strict runtime bug reproduction because the requested behavior is a new public config option.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback semantics and fast-mode interaction need explicit product approval first.
