The issue requests an optional commitments-specific model override so background commitment extraction can use a different model than the owning agent default.

## Best Possible Solution

Decide the commitments model-routing contract, then add the config type/schema/help/docs plus resolver and focused regression coverage for override precedence and default inheritance.
