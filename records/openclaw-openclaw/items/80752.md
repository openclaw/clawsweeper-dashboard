## Summary

Keep open: current main still has no commitments-specific model override, and the request would add a new public config/model-routing contract rather than close out an already-implemented cleanup item. The later user comment reinforces that commitments extraction remains painful in real setups, while the related fast-mode PR only addresses a different extraction-routing bug.

## What This Changes

Add optional commitments model routing fields so hidden commitment extraction can use a dedicated provider/model instead of always inheriting the agent default.

## Best Possible Solution

Decide the commitments model-routing contract, then implement the approved override through config type/schema/help/docs plus focused resolver and extraction tests, coordinated with the fast-mode fix path.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a new config feature request. Source inspection gives high confidence that current main lacks commitments.model and that extraction inherits the agent/default model today.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback and fast-mode interactions need product approval first.
