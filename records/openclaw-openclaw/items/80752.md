## Summary

Keep open: current main still lacks a commitments-specific model override, the strict config schema does not accept `commitments.model`, and hidden extraction still resolves through the agent/default model path. The request is narrow and well-supported by adjacent precedent, but it adds a new public config/model-routing contract that needs maintainer decision before implementation.

## What This Changes

The issue requests a commitments-specific model override, and possibly fallback, so hidden commitment extraction can use a faster or cheaper model than the main agent.

## Best Possible Solution

Decide the commitments model-routing contract, then implement it in the commitments resolver with schema/help/docs updates and focused tests for override precedence and default inheritance.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main has no `commitments.model` key and that extraction inherits the agent/default model.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback semantics and interaction with fast-mode behavior should be approved first.
