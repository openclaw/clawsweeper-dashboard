## Summary

Keep open: current main still does not expose a commitments-specific model override, and hidden commitment extraction inherits the agent/default model through the commitments resolver. The request is narrow and well-supported by adjacent precedent, but it adds a public config/model-routing option and needs maintainer approval for the exact contract.

## What This Changes

Add an optional commitments-specific provider/model override, and possibly a fallback, so hidden commitment extraction can use a different model than the main agent default.

## Best Possible Solution

Decide the commitments model-routing contract, then implement the approved fields in the type/schema/help/docs plus resolver and extraction tests for override precedence and default inheritance.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main has no `commitments.model` key and that hidden extraction inherits the agent/default model.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback semantics and interaction with fast-mode behavior need approval first.
