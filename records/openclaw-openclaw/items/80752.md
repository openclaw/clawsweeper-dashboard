## Summary

Keep open: current main and the latest release still do not expose a commitments-specific model override, and adding one is a new public config/model-routing feature rather than conservative cleanup. The request is narrow and backed by adjacent precedent, but the contract for `model` versus `modelFallback` and interaction with fast-mode/fallback behavior needs maintainer approval.

## What This Changes

The issue requests optional commitments model-routing config so hidden extraction can use a cheaper or faster model than the main agent default.

## Best Possible Solution

Keep this open for a maintainer decision on the commitments model-routing contract, then implement the approved config, schema/help/docs, resolver, and focused tests in the commitments surface.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main confirms `commitments.model` and `commitments.modelFallback` are absent and hidden extraction inherits the agent/default model.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback semantics and coordination with commitments fast-mode behavior need product approval first.
