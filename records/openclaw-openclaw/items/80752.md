## Summary

Keep open: current main still has no commitments-specific model override, and the request adds a new public config/model-routing capability rather than closing an already-fixed bug. The issue is concrete and now has an additional user report of commitments causing message timeouts, but maintainers still need to decide the public contract and coordinate it with the separate fast-mode work.

## What This Changes

Request adding optional commitments model and fallback config so hidden commitment extraction can use a dedicated model instead of inheriting the agent default.

## Best Possible Solution

Decide and implement a commitments model-routing contract in the config type, schema, help/docs, resolver, and focused tests, coordinated with current fast-mode semantics.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main cannot accept `commitments.model` and currently routes extraction through the agent/default model.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback behavior and interaction with fast-mode extraction need explicit product approval first.
