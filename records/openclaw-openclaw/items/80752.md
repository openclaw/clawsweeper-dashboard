## Summary

Keep open: current main and the latest release still lack a commitments-specific model override, while the related fast-mode PR addresses a separate extractor payload bug. The request is concrete and plausible, but it adds public config/model-routing semantics that need maintainer approval before implementation.

## What This Changes

Request adding optional `commitments.model` and possibly `commitments.modelFallback` so hidden commitment extraction can use a cheaper or faster model instead of inheriting the agent default.

## Best Possible Solution

Decide the commitments model-routing contract, then add the config type/schema/help/docs, resolver behavior, and focused tests for override precedence and default inheritance.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main and `v2026.5.12` verifies there is no `commitments.model` key and extraction inherits the agent/default model.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change matches existing precedent, but fallback semantics and interaction with the fast-mode path need explicit approval first.
