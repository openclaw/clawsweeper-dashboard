## Summary

Keep open: current main and latest release still lack a commitments-specific model override, while the request is concrete and supported by adjacent model-routing precedent. This adds a public config/model-routing contract, and prior commitments history shows an earlier extraction model knob was removed during simplification, so maintainers should decide the final contract before implementation.

## What This Changes

Add optional commitments model-routing fields so hidden commitment extraction can use a dedicated provider/model instead of inheriting the active agent default.

## Best Possible Solution

Approve a narrow commitments model-routing contract, then update the type, Zod schema, labels/help/docs, resolver, and focused tests using existing model-ref parsing semantics rather than ad hoc string splitting.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main has no `commitments.model` key and no resolver branch for a commitments-specific override.

## Solution Assessment

Unclear until maintainers choose the public contract. A small resolver/schema/docs/test change is plausible, but the proposed fallback semantics and parser shape should align with existing model-routing helpers.
