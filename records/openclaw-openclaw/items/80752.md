## Summary

Keep open: current main still has no commitments-specific model override, and the request would add a new public config/model-routing contract rather than close out an already-implemented fix. The added user comment reinforces the impact, but maintainer judgment is still needed on the exact `model`/fallback/fast-mode semantics.

## What This Changes

The issue requests an optional commitments-specific model override so background commitment extraction can use a different model from the main agent default.

## Best Possible Solution

Decide the public commitments model-routing contract, then implement it through config types, schema/help/docs, resolver logic, and focused tests for override precedence plus default inheritance.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main lacks `commitments.model` and that extraction inherits the agent default through `resolveCommitmentDefaultModelRef`.

## Solution Assessment

Unclear until maintainers choose the config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback semantics and interaction with existing fast-mode behavior need explicit product approval first.
