## Summary

Keep open: current main and the latest release still lack a commitments-specific model override, and the request adds a public config/model-routing feature. The related fast-mode work solves a different extractor payload bug, so this needs maintainer contract approval rather than cleanup closure.

## What This Changes

The issue requests an optional commitments model override, with possible fallback semantics, so commitment extraction can use a different model from the interactive agent default.

## Best Possible Solution

Maintainers should choose the commitments model-routing contract, then add the aligned config type, Zod schema, help/labels, docs, resolver behavior, and focused tests if the new public option is approved.
