## Summary

Keep open: current main still has no commitments-specific model override, and this is a public config/model-routing feature rather than an already-fixed bug. The request is narrow and supported by adjacent precedent and user reports, but it needs maintainer approval for the config contract and fallback semantics before implementation.

## What This Changes

The issue asks OpenClaw to add optional `commitments.model` and possibly `commitments.modelFallback` config so commitments extraction can use a different model than the main agent default.

## Best Possible Solution

Define the commitments model-routing contract, then add the config type, schema/help/docs, resolver behavior, and focused tests for override precedence, default inheritance, and any approved fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main lacks `commitments.model` and that extraction currently inherits the agent default through `resolveCommitmentDefaultModelRef`.

## Solution Assessment

Unclear until maintainers choose the config contract. A narrow resolver/schema/docs/test implementation looks maintainable, but adding `modelFallback` or changing fallback behavior needs explicit product approval.
