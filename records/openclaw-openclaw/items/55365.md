## What This Changes

The issue requests early rejection/redaction of browser tool HTTP(S) navigation URLs containing embedded username/password credentials before they reach Chromium or proxy/runtime logs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82490 or an equivalent narrow browser guard/redaction fix, while preserving documented remote CDP credential handling.

## Reproduction Assessment

Yes, at source level. Current main parses and forwards HTTP(S) browser navigation URLs without checking `URL.username` or `URL.password`, though I did not run a live Chromium reproduction in this read-only review.

## Solution Assessment

Yes. The shared browser navigation guard with redacted diagnostics is the right fix boundary, provided remote CDP `cdpUrl` credential support remains separate from page-navigation URL rejection.
