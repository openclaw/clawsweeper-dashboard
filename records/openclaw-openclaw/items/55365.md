## What This Changes

Request browser open/navigate inputs to reject credential-bearing HTTP(S) URLs and redact credentials from surfaced browser errors or logs.

## Best Possible Solution

Review and land, or explicitly replace, the linked browser PR so current main rejects credentialed browser navigation URLs before browser dispatch and redacts URL diagnostics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call browser `open` or `navigate` with a URL like `https://user:secret@example.com/path`, and current main forwards it before any userinfo rejection; live Chromium was not run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is a shared userinfo guard at browser-tool input and navigation-guard surfaces, which the linked PR attempts; opening a separate new fix PR would duplicate the active fix path.
