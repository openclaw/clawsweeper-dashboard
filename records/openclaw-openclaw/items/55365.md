## What This Changes

Reject browser tool HTTP(S) navigation URLs with embedded username/password credentials and redact credential-bearing diagnostics before runtime or logs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82490 or an equivalent narrow browser guard/redaction fix while preserving documented remote CDP credential handling.

## Reproduction Assessment

Yes, at source level. Current main parses and forwards HTTP(S) browser navigation URLs without checking `URL.username` or `URL.password`, though I did not run a live Chromium reproduction in this read-only review.

## Solution Assessment

Yes in direction. A shared browser navigation guard with redacted diagnostics is the right fix boundary, provided remote CDP `cdpUrl` credential support remains separate.
