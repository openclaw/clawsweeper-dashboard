## Best Possible Solution

Land this after redacted real macOS hatch/chat proof confirms a delayed OpenAI 5.5 or slow first reply stays pending past 30 seconds and clears correctly, while keeping the ACK wait short and using the existing agent.wait contract.

## Solution Assessment

Yes, this is the right fix shape: split the quick WebSocket ACK deadline from the assistant run budget and use the existing agent.wait completion contract. It is not merge-ready until the real macOS behavior proof gap is closed or maintainers explicitly accept that risk.
