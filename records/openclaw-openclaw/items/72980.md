## Summary

Keep open: current main does not implement the requested session-key env marker, but this PR should not merge as-is because the marker contract overlaps the broader open #40423 work and the diff has concrete env-resolution and node-host scope bugs.

## What This Changes

The PR injects `defaults.sessionKey` as `OPENCLAW_SESSION_KEY` into exec child environments and adds gateway-focused tests for injection, absence, and caller override behavior.

## Best Possible Solution

Define one documented runtime marker contract, likely via a shared resolver, with explicit gateway/sandbox/PTY/node scoping, inherited-env override semantics, docs/changelog coverage, and regression tests consolidated with #40423.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by setting an ambient or sandbox default `OPENCLAW_SESSION_KEY` to a stale value and observing that the PR guard preserves it over `defaults.sessionKey`; the node mismatch is visible from the current request-env versus requested-env code paths.

## Solution Assessment

No. The requested capability is useful, but this patch is not the best final shape because it treats inherited env as caller override and injects before node dispatch instead of using a scoped, documented runtime-marker resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Override inherited session markers:** `src/agents/bash-tools.exec.ts:1615-1617`
  - body: The guard checks the already-merged env, so an inherited `OPENCLAW_SESSION_KEY` from the gateway process or sandbox defaults prevents `defaults.sessionKey` from being written. That breaks the promised runtime guarantee and can pass a stale marker to child processes unless the caller explicitly overrides it.
  - confidence: 0.9
- **[P2] Keep session markers out of node host env:** `src/agents/bash-tools.exec.ts:1615-1617`
  - body: This block runs before the `host === "node"` branch, but node execution forwards only `requestedEnv` to remote `system.run` while approval analysis reads the full request env. A node command can therefore be analyzed with `OPENCLAW_SESSION_KEY` present and run remotely without it, so the injection should be gated to the intended hosts or moved after node dispatch.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff has no dependency, CI, or supply-chain change, but it intentionally exposes a session identifier through subprocess environment variables and needs maintainer acceptance of that security contract.

Concerns:

  - body: The new env var is inherited by affected child processes and descendants, where it can be printed, logged, captured in diagnostics, or exposed to tools that inspect process environments. This may be acceptable, but it should be explicitly documented and approved before merge.
