## Summary

Keep open. The capability is useful, but this PR is not ready because the injected marker can be spoofed by inherited ambient environment, the node-host path gets inconsistent analysis/runtime env state, and the work overlaps an open broader runtime-marker PR.

## What This Changes

The PR injects `defaults.sessionKey` as `OPENCLAW_SESSION_KEY` for exec commands and adds gateway-focused tests for default, absent, and caller override behavior.

## Best Possible Solution

Define one documented runtime marker resolver, scope it explicitly to supported hosts, prefer runtime session metadata over inherited ambient values unless `params.env` explicitly overrides, and land docs/changelog plus gateway, sandbox, PTY, and node-scope coverage.

## Reproduction Assessment

Yes. A static reproduction is enough: set an ambient `OPENCLAW_SESSION_KEY` on the gateway process, run an exec command with a different `defaults.sessionKey`, and the PR guard preserves the stale ambient value; for `host=node`, the added marker is present during approval analysis but absent from the remote `requestedEnv` channel.

## Solution Assessment

No. The requested env marker is reasonable, but this patch is not the narrowest maintainable solution because it relies on post-merge env state and runs before node-host dispatch instead of a scoped runtime-marker resolver.

## Security Review

Status: needs_attention

Summary: The PR intentionally exposes a session identifier through child-process environment and needs a maintainer-approved contract before merge.

Concerns:

- **[medium] Session key becomes subprocess environment data:** `src/agents/bash-tools.exec.ts:1615`
  - body: `OPENCLAW_SESSION_KEY` would be inherited by spawned commands and descendants, where tools can print it, include it in diagnostics, or expose it through process-environment inspection; that may be intended, but it needs explicit product/security acceptance and documentation.
  - confidence: 0.88
- **[medium] Ambient env can spoof the runtime marker:** `src/agents/bash-tools.exec.ts:1615`
  - body: Because the guard preserves inherited `OPENCLAW_SESSION_KEY`, a stale or operator-injected ambient value can be forwarded as if it were the current runtime session marker, undermining downstream session-aware verification.
