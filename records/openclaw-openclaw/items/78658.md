## What This Changes

## Best Possible Solution

Land this PR or an equivalent Gateway-owned patch only after maintainer acceptance of the public bypass policy, keeping the stuck-task maintenance fix separate from the operator restart escape hatch.

## Reproduction Assessment

Yes, source-reproducible: current main has internal scheduler support for `skipDeferral`, but the safe restart coordinator, RPC handler, and CLI path do not pass it. The PR body also supplies after-fix terminal proof from a real Linux Gateway setup.

## Solution Assessment

Yes, pending maintainer acceptance: the patch reuses the existing scheduler option and threads it through the existing authenticated RPC/CLI path instead of inventing a parallel restart mechanism. The safer alternative is to decline the public bypass and rely only on the already-fixed stale-task maintenance path.
