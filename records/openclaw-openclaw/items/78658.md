## Summary

Keep open: current main still lacks this public bypass, and the PR is a plausible narrow implementation with real terminal proof. No blocking code-review findings remain after the changelog cleanup, but the new operator escape hatch and current CI state still need maintainer handling before merge.

## What This Changes

The PR adds an optional `skipDeferral` parameter to `gateway.restart.request`, threads it through safe restart coordination, exposes `openclaw gateway restart --safe --skip-deferral`, and updates docs, tests, and changelog.

## Best Possible Solution

If Gateway maintainers accept the public escape hatch, land this narrow plumbing after the branch is current and checks are green; otherwise decline the public bypass rather than inventing a second restart mechanism.

## Reproduction Assessment

Yes, source-reproducible: current main has internal `scheduleGatewaySigusr1Restart({ skipDeferral })` support, but the safe coordinator, RPC handler, and CLI do not accept or forward it. The PR body also supplies after-fix terminal proof of SIGUSR1 emission with active blockers.

## Solution Assessment

Mostly yes: threading the existing scheduler option through the existing ADMIN-scoped safe-restart RPC and CLI is the narrow maintainable implementation. The open question is whether maintainers want this public bypass policy at all.
