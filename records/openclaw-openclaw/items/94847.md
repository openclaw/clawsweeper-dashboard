Full review comments:

- **[P2] Use the active reply run context for quota loading:** `src/auto-reply/reply/agent-runner.ts:2284-2288`
  - body: This call only passes `config`, while `/status` resolves provider usage with credential type and forwards `agentDir`/`workspaceDir` into `loadProviderUsageSummary`. Those fields are used by auth/profile and plugin usage resolution, so provider windows that work in `/status` can silently disappear from `/usage full` in agent-scoped or workspace-plugin setups. Mirror the status call using the current reply run context.
  - confidence: 0.86
- **[P3] Clear the footer quota timeout:** `src/auto-reply/reply/agent-runner.ts:2289-2294`
  - body: When `loadProviderUsageSummary` resolves before two seconds, the competing timeout remains scheduled until it fires. In the reply path this leaves an extra live timer for every successful `/usage full` quota lookup; capture and clear it in `finally`, or rely on the loader's existing timeout helper.
  - confidence: 0.83
