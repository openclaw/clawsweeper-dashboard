## Summary

Keep open. Current main and v2026.6.8 still provide picker/sidebar session switching and Cmd/Ctrl+K, but not guarded positional or cyclic Control UI session/agent switching shortcuts; the open guarded-shortcut PR is adjacent infrastructure for different dashboard actions, not this exact feature.

## What This Changes

Request guarded Control UI keyboard shortcuts for switching sessions or agents by position/cycle without firing from editable fields.

## Best Possible Solution

Choose the shortcut contract, then extend the guarded dashboard shortcut dispatcher to switch active-agent-filtered sessions through switchChatSession with editable-target, ordering, Escape, picker, and composer regression coverage.

## Reproduction Assessment

Not applicable: this is a new ergonomics feature request rather than broken documented behavior. Source inspection gives a high-confidence current-main comparison showing picker/sidebar switching exists but the requested guarded positional and cyclic keyboard shortcuts do not.

## Solution Assessment

Unclear until maintainers choose the shortcut contract. The maintainable implementation should extend the guarded dashboard shortcut dispatcher and reuse switchChatSession for an active-agent-filtered ordered session list, not add another uncoordinated document listener.
