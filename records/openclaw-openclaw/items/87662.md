## Summary

Keep open. Current main and latest release still lack guarded dashboard shortcuts for positional or cycling session/agent switching; the open guarded-shortcut work at https://github.com/openclaw/openclaw/pull/84827 is relevant infrastructure but only covers palette, composer focus, new-message jump, and Escape behavior, so this request still needs maintainer UX direction.

## What This Changes

Request guarded Control UI keyboard shortcuts for switching between agents or sessions without firing while text input is focused.

## Best Possible Solution

Extend the maintainer-approved guarded dashboard shortcut dispatcher to call the existing switchChatSession path with an approved binding and ordering contract, plus focused tests for editable-focus no-ops, agent scoping, session order, and existing picker behavior.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence current-main comparison showing session switching exists through picker/sidebar UI but not through the requested guarded dashboard shortcuts.

## Solution Assessment

Unclear until maintainers choose the shortcut contract. The maintainable path is to extend the central guarded shortcut dispatcher rather than add another document-level listener or bypass the existing switchChatSession helper.
