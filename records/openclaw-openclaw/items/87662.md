## Summary

Keep open. Current main and v2026.6.8 still provide command-palette, picker, and sidebar-click session switching, but not guarded positional or cyclic Control UI session/agent switching shortcuts; the open guarded-dashboard-shortcut PR is adjacent infrastructure for different actions, not this exact feature.

## What This Changes

The issue requests guarded Control UI keyboard shortcuts for switching sessions or agents without firing while text input is focused.

## Best Possible Solution

Choose the shortcut contract, then extend the guarded dashboard shortcut dispatcher to route active-agent ordered session switching through switchChatSession with editable-target, picker, and session-ordering tests.

## Reproduction Assessment

Not applicable: this is a new ergonomics feature request rather than broken documented behavior. Source inspection gives a high-confidence current-main comparison showing picker/sidebar switching exists but the requested guarded positional and cyclic keyboard shortcuts do not.

## Solution Assessment

Unclear until maintainers choose the shortcut contract. The maintainable implementation should extend the guarded dashboard shortcut dispatcher and reuse switchChatSession for an active-agent-filtered ordered session list, not add another uncoordinated document listener.
