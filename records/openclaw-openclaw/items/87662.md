## Summary

Keep open. Current main and the latest release still only provide command-palette, picker, and pointer-based session switching; the requested guarded positional or cyclic Control UI session/agent switching shortcuts remain a product/UX decision and are only adjacent to the open guarded-dashboard-shortcuts work.

## What This Changes

The issue requests guarded Control UI keyboard shortcuts for switching sessions or agents without firing while text input is active.

## Best Possible Solution

Choose the shortcut contract first, then extend the single guarded dashboard shortcut path to switch through the active-agent-scoped session order using switchChatSession with focused guard and ordering tests.

## Reproduction Assessment

Not applicable: this is a new ergonomics feature request rather than broken documented behavior. Source inspection of current main and v2026.6.9 gives a high-confidence comparison showing picker/sidebar switching exists but the requested guarded positional and cyclic keyboard shortcuts do not.

## Solution Assessment

Unclear until maintainers choose the shortcut contract. The maintainable shape is to extend one guarded dashboard dispatcher and reuse switchChatSession, not to add another uncoordinated document listener.
