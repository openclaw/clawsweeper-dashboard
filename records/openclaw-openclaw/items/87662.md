Keep open. Current main and v2026.5.27 still only have the command-palette shortcut plus picker-local keyboard behavior, while session/agent switching by guarded dashboard shortcuts remains unimplemented; the active related shortcut work at https://github.com/openclaw/openclaw/pull/84827 is useful infrastructure but does not cover this requested session-switching scope.

## What This Changes

Request guarded Control UI dashboard shortcuts for switching sessions/agents by position or cycle while ignoring editable focus targets.

## Best Possible Solution

Use the maintainer-approved guarded shortcut dispatcher as the single entry point, then add session-switch actions with browser-safe bindings, discoverability, and focused tests for editable-focus no-ops, session ordering, picker Escape behavior, and existing composer handling.

## Solution Assessment

Unclear until maintainers choose the shortcut contract. The narrow implementation should extend the guarded dashboard dispatcher rather than adding a second keyboard listener, and the requested chords need UX approval because they can conflict with browser tab navigation.
