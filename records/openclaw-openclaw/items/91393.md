## What This Changes

The PR adds a compaction-retry heartbeat that emits `run.progress` from `runEmbeddedAttempt` and adds focused regression coverage for aggregate-timeout and diagnostic session-attention behavior.

## Best Possible Solution

Refresh diagnostic activity synchronously when the compaction heartbeat fires, keep the aggregate timeout fallback, and require redacted runtime proof from a real embedded compaction retry before merge.

## Reproduction Assessment

Yes at source level: current main extends active compaction retry waits without refreshing `run.progress`, while session attention depends on `lastProgressAgeMs` for stalled versus long-running classification. I did not run a live current-main compaction retry session in this read-only review.

## Solution Assessment

No: the direction is right, but async-only emission is not the best freshness fix because sibling model-stream diagnostics update the in-memory activity clock before emitting the async event. The safer fix is direct `markDiagnosticRunProgress` plus diagnostic emission and a pre-drain regression assertion.
