## Summary

Keep open. Current main still has the unbounded provider-auth wait before JSON status can emit usage output, and the issue is already paired with open fix PR #74185, so it should remain open until that PR or an equivalent narrow fix lands.

## What This Changes

The issue requests that `openclaw status --usage --json` return machine-readable JSON reliably from non-TTY subprocess automation instead of hanging or exiting with no output.

## Best Possible Solution

Review and land PR #74185 or an equivalent narrow change that bounds provider usage auth resolution before JSON status output waits on it, returns parseable degraded JSON on auth timeout, and adds a regression test where auth resolution never settles.

## Reproduction Assessment

Yes. The issue includes a concrete non-TTY `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits for usage auth resolution before it can write output.

## Solution Assessment

Yes for triage. Keeping this issue open while PR #74185 or an equivalent narrow fix is reviewed is the right path; current main is not sufficient because the timeout only applies after auth resolution completes.
