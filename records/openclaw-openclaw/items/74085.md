## Summary

Keep open. Current main still lets `status --usage --json` wait on unbounded provider auth resolution before writing JSON stdout, and open PR #74185 already references this issue with closing syntax for the same focused fix.

## What This Changes

The issue reports that `openclaw status --usage --json` hangs or exits without machine-readable output when run from a non-TTY subprocess after the 2026.4.26 release.

## Best Possible Solution

Land PR #74185 or an equivalent focused fix that bounds provider auth resolution and returns degraded, parseable status JSON when auth lookup times out.

## Reproduction Assessment

Yes. The issue includes a concrete `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider usage auth before stdout is written.

## Solution Assessment

Yes. A focused timeout around `resolveProviderAuths` with degraded empty-provider JSON is the narrowest maintainable fix; the issue should remain open until that PR or equivalent code lands.
