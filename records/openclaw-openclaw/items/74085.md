## Summary

Keep open. Current main still has an unbounded provider-usage auth await before JSON status output can be written, and open PR #74185 already references this issue with closing syntax for the same narrow timeout gap, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw status --usage --json` hangs or exits without output when invoked from a non-TTY subprocess after upgrading to 2026.4.26.

## Best Possible Solution

Land PR #74185 or an equivalent narrow change that applies the existing timeout budget to provider usage auth resolution, returns parseable degraded JSON on auth timeout, and adds regression coverage for a never-settling auth resolver.

## Reproduction Assessment

Yes. The issue includes a concrete non-TTY `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider usage auth before writing stdout.
