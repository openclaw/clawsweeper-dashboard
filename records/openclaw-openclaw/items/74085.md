Restore reliable machine-readable `openclaw status --usage --json` output from non-TTY subprocesses after the reported 2026.4.26 regression.

## Best Possible Solution

Yes. The issue provides a concrete `spawnSync('openclaw', ['status', '--usage', '--json'])` repro, and current main still shows JSON output waiting on unbounded provider auth resolution before stdout is written.

## Solution Assessment

Yes. Bounding provider auth resolution before the usage JSON path waits is the narrowest maintainable fix, and the linked PR targets that seam directly.
