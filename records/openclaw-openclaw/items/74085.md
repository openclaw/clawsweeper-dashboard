## Summary

Keep open. Current main still has an unbounded provider-usage auth await before JSON output is written, and open PR #74185 already references this issue with closing syntax for the same narrow timeout fix.

## What This Changes

The issue asks for `openclaw status --usage --json` to return machine-readable output from non-TTY subprocesses instead of hanging or exiting with empty stdout/stderr.

## Best Possible Solution

Land PR #74185 or an equivalent narrow change that applies the existing timeout budget to provider usage auth resolution, returns parseable degraded JSON on auth timeout, and adds regression coverage for a never-settling auth resolver.

## Reproduction Assessment

Yes. The issue includes a concrete non-TTY `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current-main source still has a path where JSON output waits on unbounded provider usage auth before stdout is written.

## Solution Assessment

Yes. Bounding provider auth resolution inside `loadProviderUsageSummary` is the narrow maintainable fix; a broader status rewrite or TTY-specific branch is not needed for the observed failure.
