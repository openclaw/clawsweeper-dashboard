## Summary

Keep open. Current main still awaits provider usage auth resolution before writing `status --usage --json` output, and the linked open PR #74185 references this issue with closing syntax but has not merged, so this issue should remain open until that fix or an equivalent lands.

## What This Changes

The issue requests that `openclaw status --usage --json` return machine-readable JSON from non-TTY subprocesses instead of hanging or failing after the 2026.4.26 regression.

## Best Possible Solution

Land PR #74185 or an equivalent narrow change that bounds provider usage auth resolution, returns parseable degraded JSON on auth timeout, and adds regression coverage for a never-settling auth resolver.

Yes. The issue includes a concrete non-TTY `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider usage auth before writing stdout.
