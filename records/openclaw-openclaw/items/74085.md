## Summary

Keep open. Current main and the latest release still have an unbounded provider-usage auth await before status JSON can write stdout, and open PR #74185 already references this issue with closing syntax for the same narrow fix.

## What This Changes

The issue requests restoring reliable machine-readable `openclaw status --usage --json` output from non-TTY subprocesses after the 2026.4.26 regression.

## Best Possible Solution

Land PR #74185 or an equivalent focused fix that bounds provider usage auth resolution, returns degraded parseable JSON on auth timeout, and keeps regression coverage for a never-settling auth resolver.

## Reproduction Assessment

Yes. The issue includes a concrete `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider usage auth before stdout is written.

## Solution Assessment

Yes. A focused timeout around `resolveProviderAuths` with degraded empty-provider JSON is the narrowest maintainable fix; the issue should remain open until that PR or equivalent code lands.
