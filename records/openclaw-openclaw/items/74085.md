## Summary

Keep open. Current main still lets `status --usage --json` wait on unbounded provider-usage auth before writing JSON, and open PR #74185 already tracks the narrow timeout fix with closing syntax.

## What This Changes

Investigate a regression where `openclaw status --usage --json` hangs or exits without JSON when invoked from a non-TTY subprocess.

## Best Possible Solution

Land PR #74185 or an equivalent focused fix that applies the existing timeout budget to provider usage auth resolution, returns degraded parseable JSON on auth timeout, and adds regression coverage.

## Reproduction Assessment

Yes. The issue includes a concrete `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where a never-settling auth resolver prevents JSON output.

## Solution Assessment

Yes. A focused timeout around `resolveProviderAuths` with a never-settling-auth regression test is the narrow maintainable fix; a broader status rewrite is not needed.
