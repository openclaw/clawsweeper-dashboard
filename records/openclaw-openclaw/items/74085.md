## Summary

Keep open. Current main still has an unbounded provider-usage auth await before status JSON can write stdout, and open PR #74185 already references this issue with closing syntax for the same timeout fix.

## What This Changes

Make `openclaw status --usage --json` reliably emit machine-readable JSON from non-TTY subprocesses instead of hanging or exiting empty.

## Best Possible Solution

Land PR #74185 or an equivalent narrow fix that applies the existing timeout budget to provider usage auth resolution, returns degraded parseable JSON on auth timeout, and adds regression coverage.

## Reproduction Assessment

Yes. The issue includes a concrete `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider usage auth before stdout is written.

## Solution Assessment

Yes. A focused timeout around `resolveProviderAuths` with a never-settling-auth regression test is the narrow maintainable fix; a broader status rewrite is not needed for the observed failure.
