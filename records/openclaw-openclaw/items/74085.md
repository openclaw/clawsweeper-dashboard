## Summary

Keep open. Current main still routes `openclaw status --usage --json` through an unbounded provider-auth resolution before JSON is written, and open PR #74185 already references this issue with a focused timeout fix.

## What This Changes

Make `openclaw status --usage --json` return machine-readable JSON reliably from non-TTY subprocesses instead of hanging or exiting with empty output.

## Best Possible Solution

Review and land PR #74185 or an equivalent focused timeout around provider-usage auth resolution, then close this issue after merge and release provenance are clear.

## Reproduction Assessment

Yes. The issue includes a concrete `spawnSync('openclaw', ['status', '--usage', '--json'])` repro, and current main shows that JSON output can wait indefinitely on provider auth resolution before stdout is written.

## Solution Assessment

Yes. Bounding `resolveProviderAuths` in the provider-usage loading path is the narrowest maintainable fix because it preserves the CLI JSON contract and leaves provider fetch timeouts on their existing boundary.
