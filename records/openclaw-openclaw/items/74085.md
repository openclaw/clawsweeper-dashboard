## Summary

Keep open. The checked-out main still has the same unbounded provider-usage auth await before JSON status output can be written, and open PR #74185 already references this issue with closing syntax for the same narrow fix.

## What This Changes

The issue requests that `openclaw status --usage --json` reliably return machine-readable JSON from non-TTY subprocesses instead of hanging or failing after provider usage auth resolution.

## Best Possible Solution

Apply a narrow timeout around provider auth resolution, return parseable degraded JSON with empty usage providers on auth timeout, and keep regression coverage for a never-settling auth resolver before closing this after the fix lands.

## Reproduction Assessment

Yes. The issue provides a concrete `spawnSync('openclaw', ['status','--usage','--json'])` non-TTY reproduction, and current main still has a source path where JSON output waits on unbounded provider usage auth before writing stdout.

## Solution Assessment

Yes. PR #74185 or an equivalent focused change is the narrow maintainable solution; a broader status rewrite or TTY-specific branch is not needed for the observed failure.
