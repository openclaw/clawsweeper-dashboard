## Summary

Keep open. Current main still waits for provider usage auth resolution before the timeout-bound provider fetch phase, and open PR #74185 is already paired to this report with the narrow intended fix.

## What This Changes

`openclaw status --usage --json` should return parseable JSON from non-TTY subprocesses instead of hanging or exiting with empty output after the 2026.4.26 regression.

## Best Possible Solution

Land PR #74185 or an equivalent narrow change that bounds provider usage auth resolution before JSON status output waits on it, returns degraded parseable JSON on timeout, and adds a regression test for never-settling auth resolution.

## Reproduction Assessment

Yes. The issue provides a concrete non-TTY `spawnSync('openclaw', ['status','--usage','--json'])` repro, and source inspection on current main shows a path where JSON output waits on unbounded usage auth resolution.

## Solution Assessment

Yes for triage. Keeping the issue open while PR #74185 or an equivalent narrow fix is reviewed is the maintainable path; closing now would be premature because current main still lacks the auth timeout boundary.
