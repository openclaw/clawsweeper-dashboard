## Summary

Keep open. Current main still lets `status --usage --json` wait on provider auth resolution before writing JSON, and open PR #74185 already references this issue with closing syntax for the focused fix.

## What This Changes

Ensure `openclaw status --usage --json` returns machine-readable JSON from non-TTY subprocesses instead of hanging or failing.

## Best Possible Solution

Land PR #74185 or an equivalent focused fix that bounds provider auth resolution and returns degraded parseable JSON with regression coverage for a never-settling auth resolver.

## Reproduction Assessment

Yes. The issue provides a concrete `spawnSync('openclaw', ['status', '--usage', '--json'])` reproduction, and current main still has a source path where stdout waits on unbounded provider auth resolution.

## Solution Assessment

Yes. A focused timeout around `resolveProviderAuths` with degraded empty-provider JSON is the narrowest maintainable fix; this issue should remain open until that PR or equivalent code lands.
