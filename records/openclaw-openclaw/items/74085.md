## Summary

Keep open. Current main still awaits provider auth before writing `status --usage --json` output, the latest local release tag shows the same unbounded path, and open PR #74185 already references this issue with closing syntax for the focused fix.

## What This Changes

Make `openclaw status --usage --json` return machine-readable output from non-TTY subprocesses instead of hanging or failing when provider auth resolution stalls.

## Best Possible Solution

Review and land PR #74185 or an equivalent focused auth-timeout fix, then close this issue once the implementation is merged and release provenance is clear.

## Reproduction Assessment

Yes. The issue includes a concrete `spawnSync('openclaw', ['status', '--usage', '--json'])` subprocess repro, and current source shows JSON stdout can wait indefinitely on provider auth resolution before being written.

## Solution Assessment

Yes. Bounding `resolveProviderAuths` before `status --usage --json` waits is the narrowest maintainable fix, and the already-linked PR targets that path directly.
