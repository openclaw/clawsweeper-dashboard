## Summary

Keep open. Current main has fixed several adjacent causes, including Claude CLI runtime migration and sticky auto-fallback recovery, but the root/systemd Claude CLI permission posture is still unresolved: the docs do not mention UID 0 or the reporter's `IS_SANDBOX=1` workaround, and the Claude CLI permission mapper still adds `bypassPermissions` from OpenClaw YOLO exec policy without a root-aware branch.

## What This Changes

The issue requests actionable Claude CLI skip diagnostics, root/systemd Claude CLI backend guidance, safer fallback session persistence, and an optional root opt-in path for systemd-managed gateways.

## Best Possible Solution

Keep the merged runtime-migration and fallback-recovery fixes, then add maintainer-approved root/systemd Claude CLI guidance and targeted diagnostics that explain UID 0 `bypassPermissions` failures without silently encouraging unsafe root operation.
