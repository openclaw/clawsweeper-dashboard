## Summary

Keep open. Current main has generic fallback-skip observability, but the root/systemd Claude CLI constraint is still not documented, the Claude permission mapping is not root-aware, auto fallback session persistence is intentionally documented behavior, and the related silent-runtime-misroute class remains active in https://github.com/openclaw/openclaw/issues/74500.

## What This Changes

The issue asks OpenClaw to surface pre-dispatch claude-cli skips, document root/systemd Claude CLI constraints, and avoid persisting fallback model state when the primary CLI was not attempted.

## Best Possible Solution

Add maintainer-approved CLI candidate skip diagnostics, document the supported root/systemd Claude CLI posture, and define the policy for when auto fallback session pins should persist or be cleared.

## Reproduction Assessment

Yes for source-level triage: current main still has a deterministic pre-dispatch skip path when auth profiles are unavailable, plus the reporter supplied detailed root/systemd steps and a working host workaround. No live root/systemd VPS run was performed here, so the exact packaged host behavior was not reproduced end to end.

## Solution Assessment

Unclear as a single bundled fix. CLI skip observability and docs are narrow, but root permission behavior and auto fallback persistence require maintainer security/product judgment before an automated patch is safe.
