## Summary

Keep open. Current main still has source-reproducible pre-dispatch skip behavior, generic model-fallback logging rather than a CLI-backend-specific skipped diagnostic, and root-unaware Claude CLI permission-mode mapping; the reporter’s workaround and related open issues show this still needs maintainer policy work.

## What This Changes

The issue asks for clearer diagnostics, root/systemd Claude CLI documentation, and safer fallback model persistence when a claude-cli primary is skipped before dispatch.

## Best Possible Solution

Add maintainer-approved, provider-scoped diagnostics for unattempted CLI candidates, document the supported root/systemd Claude posture, and define when auto fallback pins are persisted or cleared.

## Reproduction Assessment

Yes for source-level triage, but not as a live root/systemd VPS run. Current main still has deterministic auth skip and fallback persistence paths, and the reporter supplied detailed root/systemd steps plus a working workaround.

## Solution Assessment

Unclear as one bundled fix. Skip observability and docs are narrow, but root behavior and fallback persistence need explicit maintainer security/product judgment before implementation.
