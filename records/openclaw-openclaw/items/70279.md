Keep open. Current main partially improves the root and diagnostic story, but the remaining asks are not fully implemented: CLI-backend-specific skip diagnostics, safe root/systemd Claude guidance, and the fallback override persistence policy still need maintainer handling.

Add diagnostics, documentation, and fallback-session handling so Claude CLI primaries on root/systemd gateways do not silently skip to fallback models.

## Best Possible Solution

Split this into maintainer-approved pieces: align systemd/root docs with the root guard, add a CLI-visible pre-dispatch skip diagnostic, and decide whether auto fallback persistence should apply when the configured primary was skipped before execution.
