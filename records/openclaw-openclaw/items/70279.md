## Summary

Keep open. Current main has adjacent external CLI credential-discovery cleanup and generic fallback observability, but the issue’s remaining maintainer requests are not fully implemented: there is still no CLI-backend-facing pre-dispatch skip diagnostic, the Claude CLI docs still do not explain the root/systemd constraint, and fallback session persistence remains intentional behavior rather than gated on a primary CLI dispatch attempt.

## What This Changes

The issue requests visible diagnostics and documentation for Claude CLI backend skips on root/systemd gateways, plus a policy change to avoid persisting fallback model selections when the configured primary was skipped.

## Best Possible Solution

Split the remaining work into maintainer-approved pieces: add a CLI-backend-visible skip diagnostic for pre-dispatch auth/cooldown skips, document the safe supported Claude CLI root/systemd operating model on the CLI backend page, and make an explicit product decision on whether auto fallback persistence should differ when the configured primary was skipped rather than executed.

## Reproduction Assessment

Yes for code-level triage: the issue gives concrete root+systemd steps, and current main has a deterministic pre-dispatch skip path when candidate auth profiles are unavailable or cooldowned. A live VPS/systemd reproduction would still be needed before blessing root-specific Claude CLI behavior.

## Solution Assessment

Unclear as one bundled fix. Logging and documentation are narrow, but changing fallback persistence and adding or documenting root bypass behavior are policy/security decisions that should be reviewed before implementation.
