## Summary

Keep open. The reporter found a workable configuration, but current main still leaves the maintainer requests unresolved: pre-dispatch skips are logged as generic model-fallback decisions, Claude CLI docs omit root/systemd guidance, and auto fallback session persistence remains documented and tested behavior.

## What This Changes

The issue asks for clearer Claude CLI skip diagnostics, root/systemd Claude CLI documentation, and a policy change or decision around persisting fallback model selections after a skipped primary CLI candidate.

## Best Possible Solution

Split the work into maintainer-approved pieces: add a CLI-backend-visible pre-dispatch skip diagnostic, document the safe Claude CLI systemd operating model, and make an explicit product decision on fallback override persistence when the configured primary was skipped rather than executed.
