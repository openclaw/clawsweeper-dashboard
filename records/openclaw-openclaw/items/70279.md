## Summary

Keep open. Current main has generic model-fallback skip diagnostics and some related hardening, but the root/systemd Claude CLI posture is still not documented in the CLI backend guide, the Claude permission mapping is not UID-0-aware, fallback session pinning remains documented behavior, and the related silent runtime-misroute report is still open.

## What This Changes

The issue requests CLI-backend skip diagnostics, root/systemd Claude CLI guidance, safer fallback session persistence, and possible root opt-in handling for claude-cli.

## Best Possible Solution

Add maintainer-approved CLI candidate skip diagnostics, document the supported root/systemd Claude CLI posture, and define when auto fallback session pins should persist or be cleared.

## Reproduction Assessment

Yes for source-level triage, but not for a live end-to-end root/systemd host. Current main still has a deterministic auth/cooldown skip path before CLI dispatch, and the reporter supplied detailed root/systemd steps plus a working host workaround.

## Solution Assessment

Unclear as a single automated fix. CLI skip observability and docs are narrow, but UID 0 Claude permission handling and fallback-session persistence need maintainer security/product policy first.
