## Summary

Keep open. Current main still has source-reproducible pre-dispatch skip and auto fallback-persistence paths, while the CLI backend docs lack root/systemd Claude guidance and current diagnostics are generic model-fallback logs rather than a CLI-backend-specific skipped event; the reporter solved their host but kept the maintainer asks active.

## What This Changes

The issue asks OpenClaw to surface Claude CLI pre-dispatch skips on a root systemd gateway, document the root/supervisor constraint, avoid sticky fallback session pins when the primary was not attempted, and optionally add a root opt-in.

## Best Possible Solution

End state should be explicit provider-scoped diagnostics for unattempted CLI candidates, safe documented Claude CLI deployment under supervisors, and a maintainer-approved rule for when auto fallback pins are persisted or reset.

## Reproduction Assessment

Yes for source-level triage, but not as a live root/systemd VPS run. Current main still has deterministic auth-cooldown skip and fallback override persistence paths, and the reporter supplied detailed root/systemd steps plus a working workaround.

## Solution Assessment

Unclear as one bundled fix. CLI skip observability and docs are narrow, but root bypass guidance, an allow-root option, and fallback persistence policy require maintainer security/product judgment before code changes.
