## Summary

Keep open: the reported Gemini replay bug is real and this PR is in the right area, but the patch is not currently mergeable because it sanitizes away the existing unsigned Gemini 3 replay sentinel and only supplies test output rather than after-fix real behavior proof. It also overlaps with the broader active replay-preservation work in https://github.com/openclaw/openclaw/pull/75713.

## What This Changes

The PR adds native Google Gemini tool-call thought-signature recovery from prior messages, merges split SSE signature parts, sanitizes outbound signatures, and adds focused tests in `extensions/google/transport-stream.test.ts`.

## Best Possible Solution

Land one corrected Google replay repair that preserves real tool-call signatures, keeps a validated unsigned-replay fallback strategy, includes the split-SSE parser merge if confirmed, and adds real behavior proof plus focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. The linked issue includes direct API T3/T4 evidence for missing versus present `thoughtSignature`, and current main/PR source shows the serializer branches that can emit or drop that field; I did not run a live Gemini request in this read-only review.

## Solution Assessment

No. The proposed direction is close, but this patch removes the unsigned replay sentinel while trying to validate signatures; a safer fix should preserve valid signatures and only replace the fallback after proving the replacement behavior against the affected runtime path.
