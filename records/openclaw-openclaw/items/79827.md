## Summary

Keep open: the PR targets a real Gemini replay bug, but it cannot merge as-is because the new sanitizer drops the documented Gemini dummy-signature fallback and the PR currently fails test typechecking. It also still needs after-fix real behavior proof from the Gemini/OpenClaw replay path.

## What This Changes

The PR changes native Google Gemini streaming parsing and replay serialization to preserve, reattach, and validate tool-call thought signatures, with regression tests.

## Best Possible Solution

Land a native Google replay repair that preserves real Gemini thought signatures, uses Google's documented dummy fallback only where supported or otherwise repairs invalid replay, and includes type-clean tests plus after-fix real Gemini/OpenClaw replay proof.

## Reproduction Assessment

Yes. The linked issue gives a direct Gemini replay matrix showing missing `thoughtSignature` causes the reported 400, and current-main source shows Gemini 3 tool-call replay depends on the stored or fallback signature; I did not run a live Gemini request in this read-only review.

## Solution Assessment

No. The proposed real-signature recovery and SSE merge direction is useful, but the current patch drops the documented fallback for unsigned replay and fails test typechecking, so it is not yet the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the Gemini fallback signature:** `extensions/google/transport-stream.ts:519-524`
  - body: The fallback branch still supplies `GEMINI_THOUGHT_SIGNATURE_VALIDATOR_SKIP` for Gemini 3 tool calls when no real signature was recovered, but this patch immediately passes that value through `sanitizeGeminiToolCallThoughtSignature`, whose new guard rejects the exact fallback. That leaves transferred or legacy unsigned replay as a bare `functionCall` with no `thoughtSignature`, which is the 400 path this PR is meant to fix; keep the documented dummy on supported endpoints or repair/drop that replay shape explicitly.
  - confidence: 0.9
- **[P2] Fix the filter callback type:** `extensions/google/transport-stream.test.ts:764`
  - body: `params.contents` is typed as `Array<Record<string, unknown>>`, so a `filter` callback that requires `{ role: string }` is not assignable and CI fails with TS2769. Use a type guard or read `role` as optional/unknown before comparing so the added test compiles.
  - confidence: 0.99
