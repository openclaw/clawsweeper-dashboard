## Summary

Keep open: the PR targets a real native Google Gemini replay bug, but the current patch still has blocking issues in the touched transport/test surface, including erasing the documented skip fallback, trusting stored signatures without matching `api`, and exact-head type/lint failures.

## What This Changes

The PR updates the Google transport to merge streamed tool-call thought signatures, replay same-id Gemini signatures, sanitize outbound signature values, and add regression tests.

## Best Possible Solution

Keep the Google-owner transport fix, but preserve documented fallback behavior, make signature trust route-exact by provider/api/model, and repair the new test type/lint failures before merge.

## Reproduction Assessment

Yes. The linked issue has a live Gemini T1-T4 matrix for the missing `thought_signature` replay 400, and source inspection shows the native Google serializer/parser path involved; I did not rerun the paid Gemini API in this read-only sweep.

## Solution Assessment

No. The owner-module direction is right, but this exact patch removes a documented fallback, still misses `api` in one stored-signature trust guard, and fails exact-head test/type checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the Gemini skip fallback:** `extensions/google/transport-stream.ts:537-541`
  - body: `sanitizeGeminiToolCallThoughtSignature` rejects `skip_thought_signature_validator`, so the fallback expression immediately below is always removed. Current main and Google's docs rely on that sentinel for manually constructed function-call replay, so this can reintroduce missing-signature failures for unsigned Gemini 3 tool calls.
  - confidence: 0.9
- **[P2] Match api when trusting stored signatures:** `extensions/google/transport-stream.ts:536`
  - body: `ownSignature` still trusts `isSameProviderAndModel`, which checks provider/model but not `api`. A `google` OpenAI-compatible history entry with the same model id can therefore pass a non-native route signature into the native Gemini serializer; make this guard match provider/api/model like the collector.
  - confidence: 0.86
- **[P2] Use typed find or narrowing for model turns:** `extensions/google/transport-stream.test.ts:764`
  - body: Exact-head CI reports type failures on the new `params.contents.filter((c: { role: string }) => ...)` callbacks, and lint also flags the line 823 filter/index pattern. Replace these lookups with a typed predicate or `find` helper so the test file compiles and passes oxlint.
  - confidence: 0.96
