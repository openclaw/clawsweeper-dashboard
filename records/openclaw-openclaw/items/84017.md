- **[P1] Authenticate scoped tokens on GET and DELETE:** `src/gateway/mcp-http.request.ts:223-227`
  - body: The generated Streamable HTTP MCP client sends the same scoped `OPENCLAW_MCP_TOKEN` on GET notification streams and DELETE teardown, but these branches still only accept the old owner/non-owner runtime tokens before scoped-token lookup runs. That can make bundled loopback clients 401 on non-POST transport paths even though POST tools/list works.
  - confidence: 0.9
- **[P1] Include every token-bound field in the live fingerprint:** `src/agents/cli-runner/claude-live-session.ts:279-287`
  - body: The scoped bearer binds channel/thread/message/audio, sessionId, reply-delivery mode, explicit-target policy, and owner state, but the Claude live-session fingerprint hashes only five fields. Reusing a warm Claude process can therefore keep an old scoped token when one of the omitted fields changes.
  - confidence: 0.92
- **[P2] Revoke scoped tokens on later prepare failures:** `src/agents/cli-runner/prepare.ts:431-433`
  - body: The cleanup guard only revokes when `prepareCliBundleMcpConfig` throws or after a successful returned cleanup exists. If later preparation work throws before return, the scoped bearer remains registered until loopback runtime reset.
  - confidence: 0.86
