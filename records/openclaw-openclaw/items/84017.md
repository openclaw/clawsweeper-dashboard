## Summary

Keep this PR open: the token-bound MCP loopback direction is the right security boundary, but the current branch can reuse a Claude live MCP process with a stale token scope when only per-message/topic context changes.

## What This Changes

The PR moves MCP loopback session/account/channel/event context from mutable request headers and CLI env values into server-issued scoped bearer tokens, with tests and Claude live-session fingerprint changes.

## Best Possible Solution

Bind every token-scoped MCP context field into Claude live-session reuse, add a regression for an omitted field, then let maintainers decide the stricter loopback compatibility tradeoff.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: prepare issues a token with all ten scope fields, but Claude live-session reuse fingerprints only five and reuses the old process token when the omitted fields change.

## Solution Assessment

No as written. Moving loopback scope into server-issued tokens is the right layer, but the live-session reuse key must match the full token scope before the branch is safe to merge.
