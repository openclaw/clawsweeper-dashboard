## Summary

Keep this PR open: the token-bound MCP loopback scope fix is useful, but the current head still lets Claude live sessions reuse a process whose original scoped token can be stale for several context fields, and the legacy header/env removal remains an explicit compatibility decision.

## What This Changes

The PR moves MCP loopback context from generated request headers and OPENCLAW_MCP_* scope env vars into server-issued scoped bearer tokens, updates CLI live-session handling and live probes, and adds gateway/CLI regressions.

## Best Possible Solution

Land one token-bound loopback-scope fix after live-session reuse depends on the full token scope, regression coverage proves an omitted-field change restarts the process, and maintainers explicitly accept or adjust the legacy header/env compatibility break.
