## Summary

Keep this PR open, but it is not merge-ready: token-bound loopback scope is the right security direction, while the current patch still splits scoped-token auth across MCP lifecycle methods, can reuse stale scoped context in warm Claude sessions, can leave tokens registered on prepare failures, and keeps obsolete mutable context headers in generated config.

## What This Changes

The PR binds MCP loopback request context to server-issued scoped bearer tokens, updates CLI runner MCP env/live-session handling, and adds gateway/agent regression tests.

## Best Possible Solution

Land a security-reviewed token-bound loopback implementation that authenticates scoped bearers consistently across POST, GET/SSE, and DELETE, fingerprints every token-scope field, revokes on all failure paths, and removes obsolete mutable scope headers while preserving required diagnostic capture behavior.

  - body: The scoped-token lookup is only reached on POST.   - confidence: 0.96

  - body: The live-session fingerprint hashes only five fields, but the scoped token also binds session id, channel/thread/message ids, inbound audio, delivery mode, and explicit-target policy.   - confidence: 0.94

  - body: The token is registered before later preparation work runs, but only the bundle-MCP config catch revokes it. If later preparation throws before cleanup is returned, the scoped bearer stays registered in the process token map.
  - confidence: 0.9
- **[P2] Remove obsolete scope headers from MCP config:** `src/gateway/mcp-http.loopback-runtime.ts:449-461`
