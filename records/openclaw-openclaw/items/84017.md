- **[P1] Accept scoped tokens on lifecycle requests:** `src/gateway/mcp-http.request.ts:223-227`
  - body: The scoped-token lookup runs only after the GET and DELETE branches have already authenticated against the legacy owner/non-owner tokens and returned. A client using the scoped bearer accepted by POST can still get 401 on SSE startup or session teardown; share the scoped-token auth path across POST, GET, and DELETE.
  - confidence: 0.93

  - body: The scoped bearer now binds session id, channel, thread, message, audio, delivery, explicit-target, and owner context, but Claude live-session reuse hashes only accountId, inboundEventKind, messageProvider, senderIsOwner, and sessionKey. A warm process can keep an old OPENCLAW_MCP_TOKEN when one of the omitted scope fields changes.
  - confidence: 0.94
- **[P2] Revoke scoped tokens on every prepare failure:** `src/agents/cli-runner/prepare.ts:442-458`
  - body: The scoped token is issued before prepareExecution, auth-epoch resolution, reference-path work, and skills setup. If any later step throws before the returned cleanup is installed, the bearer remains registered in the loopback token table.
  - confidence: 0.89
- **[P2] Remove obsolete scope headers from generated config:** `src/gateway/mcp-http.loopback-runtime.ts:447-461`

  - confidence: 0.86
