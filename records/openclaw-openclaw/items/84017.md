## Summary

Keep this PR open because it addresses a real MCP loopback security boundary on current main, but it is not merge-ready: Claude live-session reuse can still preserve a bearer token bound to stale current channel/thread/message/audio/reply-delivery context, and the intentional header/env compatibility break still needs maintainer acceptance.

## What This Changes

The PR moves MCP loopback scope from generated x-* request headers and OPENCLAW_MCP_* env values into server-issued scoped bearer tokens, updates CLI live-session handling and live probes, and adds gateway/CLI regressions.

## Best Possible Solution

Land one token-bound loopback-scope fix after the live-session fingerprint covers every token scope field, regression coverage proves restarts on omitted-field changes, and maintainers explicitly accept the legacy header/env scope removal.

## Reproduction Assessment

Yes. The central spoofing bug is source-reproducible on current main through header-derived MCP loopback scope, and the new stale live-session bug is source-reproducible because the reuse fingerprint omits five fields carried by the scoped token.

## Solution Assessment

No, not yet. Token-bound loopback scope is the right layer, but the implementation must make live-session reuse depend on the full token scope and leave the header/env compatibility break as an explicit maintainer decision.
