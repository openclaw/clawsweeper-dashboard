## Label Justifications

- `P2`: This is a focused gateway/agent security hardening PR with limited but real compatibility and session-state blast radius.
- `merge-risk: 🚨 compatibility`: The PR changes the behavior of static loopback bearer tokens with scope headers, which can break existing internal callers that relied on header-scoped context.
- `merge-risk: 🚨 session-state`: Random scoped tokens feed into the Claude live-session env fingerprint and can restart bundled-MCP live sessions even when the logical scope is unchanged.

## Summary

Keep this PR open: the security direction is sound, but it needs real behavior proof and one session-safety fix before merge.

## What This Changes

The PR moves MCP loopback session/account/channel scope from mutable headers into server-issued bearer-token context and updates gateway/CLI tests around spoofed scope headers.

## Best Possible Solution

Bind loopback context to bearer tokens while preserving deliberate session-reuse semantics: scoped tokens should be stable or lifecycle-aware for identical live-session scope, and the PR should include real CLI/MCP proof showing spoofed headers no longer affect context.

## Reproduction Assessment

Yes from source inspection: current main resolves MCP loopback session/channel/account context from mutable request headers after bearer authentication, and the PR adds tests showing spoofed scope headers no longer change context. I did not run tests because this was a read-only review.

## Solution Assessment

Not yet. Token-bound scope is the right security direction, but the implementation needs explicit live-session/token lifecycle handling and real behavior proof before it is the safest merge path.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Preserve live-session reuse for unchanged token scope:** `src/agents/cli-runner/prepare.ts:219-226`
  - body: This issues a fresh random MCP token every time `prepareCliRunContext` runs. Please make the scoped token lifecycle stable for identical scope, or explicitly key/restart only on scope changes with regression coverage.
