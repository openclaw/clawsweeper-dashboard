## Summary

Keep open: the security direction is right and real proof is supplied, but the current patch still lets Claude live MCP sessions reuse a stale scoped token when several token-bound context fields change.

## What This Changes

The PR moves MCP loopback session/account/channel/event context from mutable request headers and CLI env values into server-issued scoped bearer tokens, updates gateway/CLI handling, and adds regression coverage.

## Best Possible Solution

Bind every token-scoped MCP context field into Claude live-session reuse, add a regression for an omitted field, then make the legacy header/env compatibility removal an explicit maintainer-approved security tradeoff.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: prepare issues a token with all ten scope fields, but Claude live-session reuse fingerprints only five and reuses the old process token when the omitted fields change.

## Solution Assessment

No as written. Moving loopback scope into server-issued tokens is the right layer, but the live-session reuse key must match the full token scope before the branch is safe to merge.
