## Summary

Keep open: the iOS first-run onboarding work is useful and has real visual proof, but the new Gateway routing helper can send existing configured-but-offline users into onboarding instead of Settings / Gateway recovery. The protected `maintainer` label also means this should stay on the maintainer review path, not cleanup-close.

## What This Changes

The branch adds an iOS first-run Gateway onboarding and preview flow with seeded demo data, setup instructions, new artwork, route wiring, and focused simulator/source tests.

## Best Possible Solution

Keep the new first-run onboarding and preview experience, but narrow Gateway entry routing so true first-run users see onboarding while saved/manual/previously paired offline configurations stay on Settings / Gateway recovery.

## Reproduction Assessment

Yes, from source inspection: with a saved/manual/previous Gateway config but `isOperatorGatewayConnected == false`, any retargeted Gateway entry point calls `openGatewayEntryPoint()` and presents onboarding. I did not run the iOS simulator for that exact edge case.

## Solution Assessment

No: the new onboarding flow is a plausible and well-proven UX improvement, but the helper is too broad. The best fix is to reuse the existing configured-Gateway distinction for manual Gateway entry routing.
