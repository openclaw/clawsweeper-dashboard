## Summary

Keep open. Current main still supports model request proxy and `allowPrivateNetwork` only under each `models.providers.<id>.request` block, while the requested inherited/global default layer is not implemented and needs explicit maintainer/security design before automation should change SSRF policy inheritance.

## What This Changes

Add a global/default model-provider request policy so proxy routing and private-network allowance can be configured once, inherited by remote model providers, overridden per provider, and inspected in effective config.

## Best Possible Solution

Design a models-scoped provider request defaults layer that merges before `models.providers.<id>.request`, keeps per-provider overrides authoritative, defines remote versus local/self-hosted provider treatment, exposes the resolved policy in effective config inspection, and ships with docs plus SSRF, redaction, and transport regression coverage.

## Reproduction Assessment

Yes. Source inspection against current main gives a high-confidence reproduction for the missing feature: the strict models schema lacks global/default request keys, and runtime model resolution only forwards the selected provider's `request` block into the request policy resolver.

## Solution Assessment

Unclear. A models-scoped defaults layer is a plausible fix and safer than relying on process-wide proxy routing, but the exact config shape and private-network inheritance semantics need maintainer/security review before implementation is the best solution.
