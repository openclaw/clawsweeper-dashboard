## Summary

Keep open. Current main still has only per-provider model request overrides, while the requested inherited proxy/private-network policy would add a new config surface and change SSRF semantics; the newer fake-IP hostname allowlist reduces one failure mode but does not provide global provider request defaults.

## What This Changes

Add a global/default model-provider request policy so proxy and private-network SSRF settings can be inherited instead of repeated under each `models.providers.<id>.request` block.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before provider-local `request`, preserves explicit overrides and opt-outs, defines local-provider handling, exposes effective config, and updates schema, docs, redaction, and SSRF tests together.

## Reproduction Assessment

Yes for the current behavior: source inspection shows schema, types, docs, and runtime resolution only accept/pass `models.providers.<id>.request`. I did not run a live proxy/Fake-IP environment because the missing inherited config surface is clear from current main.

## Solution Assessment

Unclear. A provider request defaults layer is plausible, but the key shape, merge precedence, local-provider defaults, audit output, and inherited SSRF semantics need maintainer/security design before implementation.
