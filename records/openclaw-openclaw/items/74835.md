## Summary

Keep open. Current main has improved provider SSRF handling for exact configured origins and fake-IP provider hosts, but it still has no inherited model-provider request defaults for proxy or `allowPrivateNetwork`, and adding that surface needs config and security design.

## What This Changes

The issue requests an inherited model-provider request policy so operators can configure proxy and private-network SSRF settings once while preserving per-provider overrides.

## Best Possible Solution

Design a models-scoped request-defaults layer that merges before provider-local `request`, preserves explicit overrides and opt-outs, defines local-provider handling, and updates schema, docs, redaction, effective config, and SSRF tests together.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows only provider-local `models.providers.<id>.request` is accepted and passed into runtime request resolution. A live proxy/Fake-IP environment is not needed to verify that the inherited config surface is missing.

## Solution Assessment

Unclear. A model-provider defaults layer is plausible, but the exact key shape, merge precedence, local-provider defaults, audit output, and inherited SSRF semantics need maintainer/security design before implementation.
