## Summary

Keep open. Current main still only supports model-provider request proxy/private-network policy under each `models.providers.<id>.request`, and the requested inherited default would widen SSRF-sensitive behavior across providers, so this needs maintainer/security design rather than cleanup.

## What This Changes

Add a global/default model-provider request policy so proxy routing and `allowPrivateNetwork` can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before provider-local `request`, preserves explicit overrides and opt-out, defines local-provider treatment, exposes effective config, and updates docs, redaction, and SSRF tests.

## Reproduction Assessment

Yes. Source inspection against current main shows schema support only under `models.providers.<id>.request`, and runtime model resolution passes provider-local request overrides into the request policy resolver without a defaults merge.

## Solution Assessment

Unclear. A model-provider defaults layer is plausible, but the safest config shape, precedence, audit surface, and private-network SSRF semantics need maintainer/security review before implementation.
