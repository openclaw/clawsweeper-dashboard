## Summary

Keep open. Current main still supports model-provider request proxy and private-network SSRF overrides only at `models.providers.<id>.request`, and the requested inherited default policy changes security-sensitive provider routing semantics.

## What This Changes

The issue requests a global/default model-provider request policy for proxy and private-network SSRF settings with provider-level overrides.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before provider-local `request`, preserves explicit overrides and opt-out, defines local-provider treatment, exposes effective provider config, and updates docs, redaction, and SSRF tests.

## Reproduction Assessment

Yes. Source inspection against current main shows schema support only under `models.providers.<id>.request`, and all inspected model resolution paths pass only provider-local request overrides into the request policy resolver.

## Solution Assessment

Unclear. A `models.defaults.request` layer is plausible, but the safest config shape, precedence model, audit surface, and private-network SSRF semantics need maintainer/security review before implementation.
