## Summary

Keep open. Current main still supports request proxy and `allowPrivateNetwork` only at the individual model-provider level, and turning that into an inherited default SSRF/proxy policy needs maintainer security and config-design review.

## What This Changes

The issue requests a global/default model-provider request policy so proxy routing and private-network allowance can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before provider-local `request`, preserves explicit opt-out, defines local-provider handling, exposes effective config, and updates schema, docs, redaction, and SSRF tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the only supported operator config path is `models.providers.<id>.request`, and runtime model resolution passes that provider-local request into the policy resolver without a defaults merge.

## Solution Assessment

Unclear. A model-provider defaults layer is plausible, but the safest key shape, override precedence, audit output, local-provider behavior, and private-network SSRF semantics need maintainer/security design first.
