## Summary

Keep open. Current main still lacks a global/default model-provider request policy; the implemented and documented behavior remains per-provider request overrides plus the separate process-wide proxy feature, and the requested inheritance semantics are security-sensitive enough to need maintainer design before a repair PR is queued.

## What This Changes

The issue requests a global/default model-provider request policy so proxy and private-network SSRF settings can be configured once and inherited by model providers unless overridden.

## Best Possible Solution

Design a models-scoped provider request defaults layer that merges before `models.providers.<id>.request`, preserves explicit per-provider overrides, defines local/self-hosted provider opt-out semantics, appears in effective config inspection, and ships with schema, docs, redaction, and SSRF policy tests.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current strict schema has no accepted global/default provider request key, and model resolution only forwards the selected provider's `request` object into request policy resolution.

## Solution Assessment

Unclear. A global defaults layer is plausible and narrower than relying on process-wide proxy routing, but the config shape and private-network inheritance semantics need maintainer/security design before implementation can be considered the best fix.
