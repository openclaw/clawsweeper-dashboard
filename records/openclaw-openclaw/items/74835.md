## Summary

Keep open. Current main still has only provider-local model request overrides, and adding inherited proxy plus private-network SSRF defaults needs maintainer/security design before implementation.

## What This Changes

The issue requests a global/default request policy for model providers so proxy and private-network SSRF allowances can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before `models.providers.<id>.request`, preserves per-provider override authority, defines local/self-hosted opt-out behavior, and ships with docs, redaction, schema, and SSRF tests.

## Reproduction Assessment

Yes. Static inspection of current main shows the strict schema has no global/default model-provider request key, and all model resolution paths pass only provider-local `request` into the request-policy resolver.

## Solution Assessment

Unclear. A model-scoped defaults layer is a plausible maintainable direction, but the exact config shape, inheritance precedence, and private-network security semantics need maintainer/security review first.
