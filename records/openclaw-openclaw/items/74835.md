## Summary

Keep open. Current main still has only provider-local model request overrides, so the requested global/default proxy plus private-network SSRF policy is not implemented; the related per-provider fix in #63671 helps the workaround but does not remove the repeated-config problem, and #74809 means the documented process-wide proxy is not a reliable substitute for this fleet.

## What This Changes

The issue requests a global/default model-provider request policy so proxy settings and `allowPrivateNetwork` can be configured once, inherited by model providers, overridden per provider, and inspected in effective config output.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before `models.providers.<id>.request`, keeps provider overrides authoritative, defines local/self-hosted provider opt-out behavior, exposes the resulting policy through config inspection, and ships with schema, docs, redaction, and SSRF/proxy precedence tests.

## Reproduction Assessment

Yes. A static check against current main is enough: the proposed global/default keys are absent from the strict models schema, and runtime model resolution only forwards `providerConfig.request` into the provider request policy resolver.

## Solution Assessment

Unclear. A global defaults layer is a plausible narrower solution than relying on process-wide proxy routing, but the exact config shape and private-network inheritance semantics need maintainer/security design before implementation is the best path.
