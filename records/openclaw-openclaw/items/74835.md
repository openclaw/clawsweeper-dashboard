## Summary

Keep open. Current main still only supports provider-local model request overrides, and the requested inherited provider request policy is absent from the strict config schema and runtime model resolution path. The request is valid but security-sensitive because it would change how proxy routing and private-network SSRF exceptions propagate across providers.

## What This Changes

The issue requests a models-wide default request policy so model providers can inherit explicit proxy and private-network SSRF settings while preserving provider-level overrides.

## Best Possible Solution

Add a models-scoped request-default layer that merges before models.providers.<id>.request, keeps provider overrides authoritative, supports explicit false opt-out for local/self-hosted providers, appears in effective config inspection, and ships with schema/help/docs plus SSRF, proxy, and redaction tests.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the strict models schema has no inherited request key, and both normal and fallback model resolution only pass providerConfig.request into the request-policy resolver.

## Solution Assessment

Unclear. A models-scoped defaults layer is plausible and narrower than relying on process-wide proxy routing, but inherited private-network semantics and the final config shape need maintainer/security design before implementation.
