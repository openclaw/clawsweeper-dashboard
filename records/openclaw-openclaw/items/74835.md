## Summary

Keep open. Current main still only supports provider-local model request overrides; the requested global/default model-provider request policy is absent, and inheriting proxy plus private-network SSRF behavior needs maintainer/security design before implementation.

## What This Changes

Request a global/default model-provider request policy so proxy settings and private-network SSRF allowances can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped provider request defaults layer that merges before models.providers.<id>.request, keeps per-provider overrides authoritative, defines local/self-hosted provider opt-out behavior, appears in config schema/effective inspection, and ships with docs plus SSRF and redaction coverage.

## Reproduction Assessment

Yes. Against current main, the requested global/default keys are absent from the strict models schema, and runtime model resolution only forwards providerConfig.request into the provider request policy resolver.

## Solution Assessment

Unclear. A model-scoped defaults layer is a plausible narrower solution than relying on process-wide proxy routing, but the exact config shape and inherited private-network semantics need maintainer/security review before it is the best fix.
