## Summary

Keep open. Current main still only supports model-provider request proxy and private-network SSRF opt-in at the individual `models.providers.<id>.request` level, and making those settings inherited defaults requires maintainer security and config-design review.

## What This Changes

The issue requests a global/default request policy for model providers so proxy and private-network SSRF settings can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before provider-local `request`, preserves explicit overrides and opt-outs, defines local-provider handling, exposes effective config, and updates schema, docs, redaction, and SSRF tests together.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows only `models.providers.<id>.request` is accepted and passed into runtime resolution. I did not run a live proxy/Fake-IP environment because the missing inherited config surface is clear from schema, types, docs, and call sites.

## Solution Assessment

Unclear. A model-provider defaults layer is plausible, but the exact key shape, merge precedence, local-provider defaults, audit output, and inherited SSRF semantics need maintainer/security design before implementation.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would make proxy routing and private-network SSRF allowances inheritable across model providers.
