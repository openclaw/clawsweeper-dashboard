## Summary

Keep open. Current main still only supports model-provider request proxy and private-network SSRF opt-in at `models.providers.<id>.request`; adding an inherited default layer would be a new security-sensitive config surface that needs maintainer design review.

## What This Changes

Add a global/default model-provider request policy so proxy routing and `allowPrivateNetwork` can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before provider-local `request`, preserves explicit per-provider override and opt-out, defines local/self-hosted behavior, exposes effective config, and updates schema, docs, redaction, and SSRF tests.

## Reproduction Assessment

Yes for the missing config surface. Source inspection on current main shows the schema, config types, docs, and runtime model resolution all require provider-local `models.providers.<id>.request` with no defaults merge.

## Solution Assessment

Unclear. A model-provider request defaults layer is plausible, but the safest key shape, merge precedence, audit output, local-provider behavior, and private-network SSRF semantics need maintainer/security design first.
