## Summary

Keep open: current main still lacks a global/default model-provider request policy. The shipped per-provider `models.providers.*.request` support and related Fake-IP reports are useful evidence, but they do not satisfy this narrower request to configure proxy and private-network SSRF posture once across model providers.

## What This Changes

Request to add a global/default model-provider request policy so proxy and private-network SSRF settings can be inherited by providers while preserving per-provider overrides.

## Best Possible Solution

Design a models-scoped provider request defaults layer that merges before `models.providers.<id>.request`, preserves explicit per-provider overrides, defines local/self-hosted provider opt-out semantics, appears in effective config inspection, and ships with schema, docs, redaction, and SSRF policy tests.

## Reproduction Assessment

Yes. Against current main, the proposed global/default keys are not accepted by the strict models schema, and runtime model resolution only forwards `providerConfig.request` to the request policy resolver.

## Solution Assessment

Unclear. A global defaults layer is plausible and narrower than relying on process-wide proxy routing, but the config shape and private-network inheritance semantics need maintainer/security design before implementation is the best fix.
