## Summary

Keep open. Current main still only supports model-provider request overrides per provider, and adding inherited proxy/private-network defaults is core security-policy work rather than a cleanup close.

## What This Changes

Add a global or model-scoped default request policy so model providers can inherit proxy and `allowPrivateNetwork` settings unless overridden.

## Best Possible Solution

Design a model-scoped request defaults layer that merges before `models.providers.<id>.request`, preserves provider overrides, defines local/self-hosted opt-out behavior, and ships with config docs plus SSRF/redaction tests.

## Reproduction Assessment

Yes. Source inspection against current main shows strict schema support only under `models.providers.<id>.request`, and both configured and fallback model resolution pass only provider-local request overrides into the request policy resolver.

## Solution Assessment

Unclear. A `models.defaults.request` shape is plausible, but the exact inheritance precedence, audit visibility, and private-network SSRF semantics need maintainer/security review before implementation.
