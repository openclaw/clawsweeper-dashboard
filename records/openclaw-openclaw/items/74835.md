## Summary

Keep open. Current main still supports model-provider request proxy and `allowPrivateNetwork` only on individual providers, and making those settings inheritable needs explicit maintainer security and config-design review.

## What This Changes

Request a models-scoped/global request policy so model providers can inherit proxy and `allowPrivateNetwork` defaults while preserving per-provider overrides.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before provider-local `request`, supports explicit opt-out, defines local-provider behavior, exposes effective provider config, and updates schema, docs, redaction, and SSRF tests.

## Reproduction Assessment

Yes. Source inspection on current main shows only `models.providers.<id>.request` is schema-supported and runtime model resolution passes only the provider-local request into the request-policy resolver.

## Solution Assessment

Unclear. A models-scoped defaults layer is plausible, but the safest config key, override precedence, local-provider exclusion, effective-config output, and SSRF semantics need maintainer/security design before implementation.
