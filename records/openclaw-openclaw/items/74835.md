## Summary

Keep open. Current main still only supports provider-local model request overrides, while inherited proxy plus private-network SSRF defaults remain absent and need maintainer/security design before implementation.

## What This Changes

Add a global/default request-policy layer for model providers so proxy and `allowPrivateNetwork` settings can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before `models.providers.<id>.request`, keeps per-provider overrides authoritative, defines local/self-hosted opt-out behavior, and ships with config docs plus SSRF/redaction tests.

## Reproduction Assessment

Yes. Against current main, strict schema inspection and runtime resolution show no accepted global/default model-provider request key and no merge source before provider-local `request`.

## Solution Assessment

Unclear. A model-scoped defaults layer is a plausible maintainable direction, but the exact config shape, inheritance precedence, and private-network security semantics need maintainer/security review first.
