## Summary

Keep open. Current main still lacks an inherited model-provider request defaults layer; because the requested `allowPrivateNetwork` inheritance touches SSRF policy, this needs maintainer/security design rather than automatic cleanup.

## What This Changes

Request a global/default model-provider request policy so proxy overrides and private-network allowance can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before `models.providers.<id>.request`, keeps provider overrides authoritative, defines local/self-hosted opt-out behavior, and ships with config docs plus SSRF/redaction tests.

## Reproduction Assessment

Yes. Source inspection against current main shows strict schema support only under `models.providers.<id>.request`, and runtime model resolution passes only provider-local request overrides into the request policy resolver.

## Solution Assessment

Unclear. A model-scoped defaults layer is plausible, but the exact config shape, inheritance precedence, audit visibility, and private-network SSRF semantics need maintainer/security review first.
