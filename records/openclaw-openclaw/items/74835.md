## Summary

Keep open. Current main still only supports model-provider request overrides per provider, and the requested inherited proxy/private-network policy would change SSRF behavior enough that it needs maintainer/security design instead of cleanup closure.

## What This Changes

The issue requests a global/default model-provider request policy so proxy and `allowPrivateNetwork` settings can be inherited by providers with per-provider overrides.

## Best Possible Solution

Design a models-scoped request defaults layer that merges before `models.providers.<id>.request`, keeps provider overrides authoritative, defines local/self-hosted opt-out behavior, and ships with config docs plus SSRF/redaction tests.

## Reproduction Assessment

Yes. Source inspection against current main shows the proposed default policy keys are absent from the strict schema, and runtime model resolution passes only provider-local `request` values into the request policy resolver.

## Solution Assessment

Unclear. A model-scoped defaults layer is plausible, but the exact shape, precedence, effective-config visibility, and inherited private-network semantics need maintainer/security review first.
