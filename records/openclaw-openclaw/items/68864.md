## Summary

Keep open. Current `main` still reads compaction checkpoints from stored session metadata only, while the active closing fix remains open at https://github.com/openclaw/openclaw/pull/81108.

## What This Changes

The issue requests that Web UI checkpoint lists include existing `{session}.checkpoint.{checkpointId}.jsonl` files even when historical session metadata lacks `compactionCheckpoints`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81108, or an equivalent gateway-side reconciliation that discovers valid disk checkpoint files and keeps row summaries, list/get, branch, and restore consistent.

## Reproduction Assessment

Yes. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will produce an empty row count and empty `sessions.compaction.list` response because the gateway reads only stored metadata.

## Solution Assessment

Yes. Server-side disk discovery behind the existing gateway API is the narrow maintainable fix; a UI-only workaround would still leave get, branch, and restore inconsistent.
