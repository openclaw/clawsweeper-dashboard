## Summary

Keep open: current main and the latest release still expose checkpoint lists from stored session metadata only, and the open closing PR that adds disk checkpoint discovery has not merged.

## What This Changes

The issue asks for the Web UI session checkpoint list to show existing `{sessionId}.checkpoint.{checkpointId}.jsonl` files for historical sessions whose session-store checkpoint metadata is missing.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81108 so the gateway session APIs reconstruct valid disk checkpoint transcripts when stored checkpoint metadata is absent.

## Reproduction Assessment

Yes, source-backed. A current-main session entry with missing or empty `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will return an empty checkpoint list because rows and compaction APIs only read stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the right boundary; this issue should remain open until the linked PR lands or an equivalent fix replaces it.
