## Summary

Keep open: current main and the latest shipped release still read checkpoint listings from stored session metadata only, while the open linked PR owns the disk-discovery fix path and references this issue with closing syntax.

## What This Changes

The issue reports that the Web UI Show Checkpoints action displays no historical compaction checkpoints even when checkpoint JSONL files exist on disk.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81108 so the existing gateway session APIs discover valid disk checkpoint transcripts when stored metadata is missing.

## Reproduction Assessment

Yes, source-backed. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will return an empty checkpoint list because row and compaction APIs read only stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the maintainable boundary; this issue should stay open until the linked PR lands or an equivalent fix replaces it.
