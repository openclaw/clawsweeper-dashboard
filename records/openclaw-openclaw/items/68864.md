## Summary

Keep this issue open. Current main still serves the checkpoint modal and row summary from session-store `compactionCheckpoints` metadata only, so historical disk-only checkpoint files remain invisible; the directly related fix PR is closed unmerged.

## What This Changes

The issue reports that the Web UI session-list checkpoint modal is empty for historical sessions even though valid checkpoint `.jsonl` files exist on disk.

## Best Possible Solution

Add gateway-side checkpoint discovery or reconciliation for valid checkpoint transcript files when stored metadata is empty or stale, then cover list/get and session-row summaries with disk-only regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a session-store entry without `compactionCheckpoints` plus valid `{sessionId}.checkpoint.{checkpointId}.jsonl` files will make `sessions.compaction.list` return an empty array. The UI then renders the empty checkpoint state from that RPC result.

## Solution Assessment

Yes. The narrowest maintainable fix is server-side gateway discovery or reconciliation using the existing checkpoint filename and transcript contracts, leaving the UI on the existing `sessions.compaction.*` API.
