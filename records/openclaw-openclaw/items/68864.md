## Summary

Keep open because current main still serves checkpoint rows and compaction APIs from stored `compactionCheckpoints` metadata only, and the open closing PR is the active fix path rather than a merged implementation.

## What This Changes

The issue reports that the Web UI checkpoint modal is empty for historical compacted sessions whose `{session}.checkpoint.{uuid}.jsonl` files exist on disk but are missing session-store checkpoint metadata.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81108 after its checkpoint timestamp repair, or replace it with equivalent gateway-side disk discovery that keeps session rows, list/get, branch, and restore consistent.

## Reproduction Assessment

Yes. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will produce empty row/list data because the gateway and UI consume only stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the narrow maintainable fix; a UI-only workaround would leave get, branch, and restore inconsistent.
