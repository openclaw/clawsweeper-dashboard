## Summary

Keep open: current main still derives checkpoint rows and compaction APIs from stored `compactionCheckpoints` metadata only, while the disk-discovery fix path is still an open, unmerged PR.

## What This Changes

The issue reports that the Web UI's Show Checkpoints list is empty for historical compacted sessions whose checkpoint JSONL files exist on disk but are not indexed in session metadata.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81108 after its pending review fix, or replace it with equivalent gateway-side disk discovery that keeps rows, list/get, branch, and restore consistent.

## Reproduction Assessment

Yes. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will produce empty row/list data because the gateway and UI consume only stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the narrow maintainable fix; a UI-only workaround would leave get, branch, and restore inconsistent.
