## Summary

Keep open: current main still lists checkpoint data from stored session metadata only, and the open closing PR that adds disk checkpoint discovery has not merged.

## What This Changes

The issue reports that the Web UI checkpoint modal is empty for historical compacted sessions even though `{session}.checkpoint.{uuid}.jsonl` files exist on disk.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81108 so the existing gateway session APIs and row projection discover valid disk checkpoint transcripts when stored metadata is missing.

## Reproduction Assessment

Yes, source-backed. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will return an empty checkpoint list because row and compaction APIs read only stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the right boundary; this issue should stay open until the linked PR lands or an equivalent fix replaces it.
