## Summary

Keep open. Current main still returns checkpoint rows and `sessions.compaction.*` results from stored `compactionCheckpoints` metadata only, and https://github.com/openclaw/openclaw/pull/81108 is now the active closing fix PR, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks the Web UI and gateway session APIs to show existing disk checkpoint JSONL files for historical sessions even when `sessions.json` lacks checkpoint metadata.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81108, or an equivalent gateway-side reconciliation that validates disk checkpoint files and keeps row summaries, list/get, branch, and restore behavior consistent.

## Reproduction Assessment

Yes. A current-main session-store entry with no `compactionCheckpoints` but valid `{sessionId}.checkpoint.{checkpointId}.jsonl` files will produce an empty `sessions.compaction.list` response because the source reads only stored metadata.

## Solution Assessment

Yes. Server-side checkpoint discovery or reconciliation behind the existing gateway API is the narrow maintainable fix; the issue should remain open while the linked implementation PR is reviewed.
