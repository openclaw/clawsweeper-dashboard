## Summary

Keep open. Current main still reads compaction checkpoints from session-store metadata only, and an open closing PR targets the disk-discovery fix, so the issue should remain open until that lands or is replaced.

## What This Changes

Show Checkpoints in the Web UI should list existing disk checkpoint `.jsonl` files for historical sessions even when session-store metadata is missing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81108, or an equivalent gateway-side reconciliation that discovers valid disk checkpoint files and keeps rows, list/get, branch, and restore consistent.

## Reproduction Assessment

Yes. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will produce empty checkpoint row/list data because the gateway and UI consume only stored metadata.

## Solution Assessment

Yes. Server-side disk discovery behind the existing gateway API is the narrow maintainable fix; a UI-only workaround would still leave get, branch, and restore inconsistent.
