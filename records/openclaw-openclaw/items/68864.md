## Summary

Keep open: current main still builds checkpoint counts and `sessions.compaction.*` responses from stored `compactionCheckpoints` metadata only, while the open closing PR owns the disk-discovery fix and has not merged.

## What This Changes

The issue asks the Web UI checkpoint list to show existing `{sessionId}.checkpoint.{checkpointId}.jsonl` files for historical sessions even when checkpoint metadata is missing from the session store.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81108 so the existing gateway session APIs and row projection discover valid disk checkpoint transcripts when stored metadata is missing.

## Reproduction Assessment

Yes, source-backed. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will return an empty list because rows and compaction APIs only read stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the right boundary; this issue should remain open until the linked PR lands or an equivalent fix replaces it.
