## Summary

Keep open. Current main still routes the Web UI checkpoint modal through gateway session-store metadata only, with no disk-discovery fallback for historical `.checkpoint.*.jsonl` files, and the directly related fix PR is closed unmerged.

## What This Changes

The issue asks for the Web UI checkpoint list to show existing historical compaction checkpoint files on disk when session-store checkpoint metadata is missing or stale.

## Best Possible Solution

Add a narrow gateway-side reconciliation path that discovers parser-valid checkpoint transcript files for the requested session when stored checkpoint metadata is empty or stale, synthesizes safe list/get metadata, and preserves the existing branch/restore contracts.

## Reproduction Assessment

Yes. Create or preserve valid `.checkpoint.*.jsonl` files for a session whose store entry lacks `compactionCheckpoints`, then call `sessions.compaction.list` or open the Sessions UI checkpoint modal; current main returns an empty checkpoint array.

## Solution Assessment

Yes. Server-side discovery/reconciliation is the narrowest maintainable solution because the UI already calls the correct RPC and the config layer already defines the safe checkpoint filename shape.
