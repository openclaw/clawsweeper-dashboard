## Summary

Keep this issue open. Current main still serves the Web UI checkpoint modal from session-store `compactionCheckpoints` metadata only, while the reported disk-only historical checkpoint files have no gateway discovery or reconciliation path; the directly related fix PR was closed unmerged.

## What This Changes

Fix the Web UI session checkpoint list so historical sessions with valid `.checkpoint.*.jsonl` files on disk show their checkpoints even when session-store metadata is empty or stale.

## Best Possible Solution

Add a narrow gateway-side reconciliation path for valid checkpoint transcript files when stored checkpoint metadata is empty or stale, then cover list/get behavior with a disk-only regression test.

## Reproduction Assessment

Yes. Create or preserve valid `{sessionId}.checkpoint.{checkpointId}.jsonl` files for a session whose store entry lacks `compactionCheckpoints`, then call `sessions.compaction.list` or open the Sessions UI checkpoint modal and observe an empty list.

## Solution Assessment

Yes. The narrowest maintainable fix is server-side gateway discovery/reconciliation because the UI already calls the right RPC and the existing artifact parser defines the safe filename shape.
