## Summary

Keep this issue open. Current main still serves the Web UI checkpoint modal from session-store `compactionCheckpoints` metadata only, so valid historical checkpoint files on disk remain invisible when that metadata is absent; the directly related fix PR is closed unmerged.

## What This Changes

The issue requests that the Web UI session checkpoint list show existing checkpoint `.jsonl` files for historical sessions even when session-store checkpoint metadata is missing.

## Best Possible Solution

Add a narrow gateway-side checkpoint reconciliation path for valid checkpoint transcript files when stored metadata is empty or stale, then cover list/get and row summary behavior with disk-only regression tests.

## Reproduction Assessment

Yes. A session-store entry without `compactionCheckpoints` but with valid `{sessionId}.checkpoint.{checkpointId}.jsonl` files will make `sessions.compaction.list` return an empty array because current main only reads stored metadata.

## Solution Assessment

Yes. The narrowest maintainable fix is server-side gateway discovery or reconciliation using the existing checkpoint filename/parser contracts, while leaving the UI to keep consuming the existing RPC.
