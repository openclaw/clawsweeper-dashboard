## Summary

Keep open. Current main still serves checkpoint details from session-store metadata only, so disk-only historical `{sessionId}.checkpoint.{checkpointId}.jsonl` files remain invisible in the Sessions UI, and the directly related repair PR #68883 is closed unmerged.

## What This Changes

The issue asks the Web UI Sessions checkpoint modal to show existing on-disk compaction checkpoint files for historical sessions whose store metadata is missing.

## Best Possible Solution

Add gateway-side reconciliation from valid checkpoint transcript files into the existing checkpoint RPC and session-row path, with regression coverage for disk-only historical checkpoints.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a session-store entry without `compactionCheckpoints` but with valid checkpoint files will make `sessions.compaction.list` return an empty array because current main reads only stored metadata.

## Solution Assessment

Yes. The narrowest maintainable fix is server-side discovery or reconciliation using the existing checkpoint filename/session-file contracts while leaving the UI on the current RPC surface.
