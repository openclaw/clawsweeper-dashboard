## Summary

Keep open. Current main still serves checkpoint rows and `sessions.compaction.*` responses from stored `compactionCheckpoints` metadata only, so historical disk checkpoint files remain invisible when that metadata is missing or stale; the directly related fix PR is closed unmerged.

## What This Changes

The issue asks for the Web UI Sessions compaction checkpoint list to show existing `{sessionId}.checkpoint.{checkpointId}.jsonl` files for historical or inactive sessions whose store metadata is missing or stale.

## Best Possible Solution

Add gateway-side discovery or reconciliation of valid checkpoint transcript files into checkpoint list/get/branch/restore and row summaries, using existing filename/transcript contracts with disk-only regression coverage.

## Reproduction Assessment

Yes. A session-store entry with no `compactionCheckpoints` plus valid `{sessionId}.checkpoint.{checkpointId}.jsonl` files will make the current `sessions.compaction.list` source path return an empty array, which the UI renders as no checkpoints.

## Solution Assessment

Yes. The narrow maintainable fix is server-side checkpoint discovery or reconciliation behind the existing `sessions.compaction.*` contract; the closed PR is a useful sketch, but the final fix should use the existing parser and cover row summaries as well as list/get behavior.
