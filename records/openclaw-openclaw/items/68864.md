## Summary

Keep open: current main and the latest release still build checkpoint rows and `sessions.compaction.*` responses from stored metadata only, while the open closing PR owns the exact disk-discovery fix but has not merged.

## What This Changes

The issue asks the Web UI/session APIs to show existing disk checkpoint transcripts for historical sessions when stored compaction checkpoint metadata is missing.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81108 so gateway session rows and `sessions.compaction.*` discover valid disk checkpoint transcripts when stored metadata is missing.

## Reproduction Assessment

Yes, source-backed. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will return an empty list because rows and compaction APIs only read stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the right boundary; this issue should remain open until the linked PR lands or an equivalent fix replaces it.
