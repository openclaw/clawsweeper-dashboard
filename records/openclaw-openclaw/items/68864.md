## Summary

Keep open: current main still derives checkpoint rows and compaction APIs from stored `compactionCheckpoints` metadata only, while https://github.com/openclaw/openclaw/pull/81108 references this issue with the exact gateway disk-discovery fix and is still open.

## What This Changes

The issue reports that Web UI checkpoint lists are empty for historical sessions even when checkpoint transcript files exist on disk.

## Best Possible Solution

Gateway session APIs should discover valid disk checkpoint transcripts for metadata-missing sessions and keep rows, list/get, branch, and restore consistent.

## Reproduction Assessment

Yes. A current-main session entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will return an empty `sessions.compaction.list` result because the gateway and row projection consume only stored metadata.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the narrow maintainable fix; a UI-only workaround would still leave get, branch, and restore inconsistent.
