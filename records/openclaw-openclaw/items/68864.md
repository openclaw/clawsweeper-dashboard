## Summary

Keep open. Current main still derives checkpoint rows and `sessions.compaction.*` results from stored `compactionCheckpoints` metadata, and the active closing fix is still open at https://github.com/openclaw/openclaw/pull/81108.

## What This Changes

The issue asks the Web UI and Gateway to show existing on-disk compaction checkpoint JSONL files for historical sessions whose session-store metadata lacks checkpoint entries.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81108 or an equivalent gateway-side reconciliation that validates disk checkpoint files and keeps row summaries, list/get, branch, and restore consistent.

## Reproduction Assessment

Yes. A current-main session-store entry with no `compactionCheckpoints` but valid `{session}.checkpoint.{uuid}.jsonl` files will produce empty row and `sessions.compaction.list` results because the gateway reads only stored metadata.

## Solution Assessment

Yes. Server-side disk checkpoint discovery behind the existing gateway API is the narrow maintainable fix; this issue should remain open while the linked implementation PR is reviewed.
