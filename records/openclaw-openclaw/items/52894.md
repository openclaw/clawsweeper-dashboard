## Summary

Keep open. Current main still matches the report: skill onboarding only offers to show the Homebrew install command, while gateway-backed skill dependency installs return a missing-brew failure when no brew executable is available. Automatically bootstrapping Homebrew from these flows would execute a remote installer, so this needs maintainer product/security direction before a fix PR is safe.

## What This Changes

Request skill onboarding and dependency installation flows to install Homebrew automatically when `brew` is missing instead of only showing the Homebrew install command.

## Best Possible Solution

Keep the issue open until maintainers decide the product/security policy. If approved, implement a shared explicit opt-in Homebrew bootstrap used by skill onboarding and gateway-backed skill installs, then re-detect brew before running the selected installer; if not approved, improve the manual requirement and failure messaging instead.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: select a missing brew-backed skill with no brew executable available, and `setupSkills` only shows the Homebrew command while `installSkill` returns `brew not installed` for the brew spec.

## Solution Assessment

Unclear. Automatic Homebrew installation would address the reported friction, but it is not clearly the best maintainable solution until maintainers approve executing that remote installer from OpenClaw-controlled skill flows.
