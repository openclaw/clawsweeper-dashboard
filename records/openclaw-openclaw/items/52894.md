## Summary

Keep open. Current main and the latest release still intentionally show or return Homebrew guidance instead of bootstrapping Homebrew during skill installs, and changing that behavior is a security-sensitive setup policy decision.

## What This Changes

The issue requests that OpenClaw automatically install Homebrew when a skill install path needs brew and brew is not installed.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway/UI, macOS, Linux, and container behavior.

## Reproduction Assessment

Yes, at source level. Selecting a brew-backed missing skill without brew reaches manual Homebrew guidance, and direct installSkill returns a missing-brew failure instead of bootstrapping Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve onboarding, but it is only the best solution after maintainers approve the remote-installer trust, consent, platform, and failure policy.
