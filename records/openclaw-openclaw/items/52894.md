## Summary

Keep open. Current main still keeps skill-triggered Homebrew setup manual, and turning the displayed Homebrew command into an executed bootstrap needs maintainer product and security approval rather than an automated fix.

## What This Changes

The issue requests that OpenClaw automatically install Homebrew when a skill install needs `brew` but Homebrew is absent.

## Best Possible Solution

Define a maintainer-approved, explicit-consent Homebrew bootstrap policy shared by interactive onboarding and gateway-backed skill installs; otherwise keep the manual requirement and improve guidance.

## Reproduction Assessment

Yes, at source level: selecting a brew-backed missing skill with no `brew` shows the Homebrew command but does not execute it, and direct install returns a missing-brew failure. This is a feature gap against the request, not a bug against current docs, because the docs explicitly say skill installs do not auto-install Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation would reduce onboarding friction, but it is only the best solution if maintainers approve the remote-installer trust, consent, platform, and gateway policy; otherwise clearer manual guidance is safer.
