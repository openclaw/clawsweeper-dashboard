## Summary

Keep open. Current main and the latest release still show or report manual Homebrew installation instead of bootstrapping it, and changing that behavior needs maintainer product/security policy because it would run Homebrew's remote installer from skill install flows.

## What This Changes

The issue requests automatic Homebrew installation when brew-backed skill installation needs brew and no brew executable is present.

## Best Possible Solution

Keep this open for maintainer product/security direction; if approved, add an explicit opt-in shared Homebrew bootstrap path for skill onboarding and gateway-backed skill installs, otherwise improve the manual Homebrew requirement and failure messaging.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: select a brew-backed missing skill with no brew executable, and onboarding only displays the Homebrew command while backend `skills.install` returns `brew not installed`.

## Solution Assessment

Unclear. Automatic Homebrew installation would address the friction, but it is not clearly the best maintainable fix until maintainers decide whether OpenClaw should run that remote installer from skill flows.
