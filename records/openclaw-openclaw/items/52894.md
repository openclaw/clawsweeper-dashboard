## Summary

Keep open. Current main and v2026.6.10 still deliberately show Homebrew guidance or return a missing-brew failure, while changing that into automatic Homebrew bootstrap needs a maintainer security/product decision.

## What This Changes

The issue requests that OpenClaw install Homebrew automatically when a skill install or onboarding flow detects that brew is missing.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install lifecycle with docs and coverage for onboarding, direct install, macOS, Linux, and container cases.

## Reproduction Assessment

Yes. Source inspection shows the selected brew-backed missing-skill path displays Homebrew guidance, and direct brew installer specs return a missing-brew failure instead of bootstrapping Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve onboarding, but it is only the best solution after maintainers approve the remote-installer trust, consent, platform, config, and failure policy.
