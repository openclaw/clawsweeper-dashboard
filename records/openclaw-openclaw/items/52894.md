## Summary

Keep open. Current main still matches the report: skill onboarding shows the Homebrew install command instead of running it, and gateway-backed skill installs return a missing-brew failure when no brew executable is available. The remaining decision is product/security policy, because automatic Homebrew bootstrap would execute Homebrew's remote installer from OpenClaw-controlled flows.

## What This Changes

Request that skill onboarding or skill dependency installation automatically install Homebrew when a selected brew-backed skill dependency needs brew and no brew executable is available.

## Best Possible Solution

Keep this open for maintainer product/security direction. If approved, add an explicit opt-in shared Homebrew bootstrap path for skill onboarding and gateway-backed skill installs, re-detect brew after installation, preserve install-source scanning and user consent, and update tests/docs; if not approved, improve the manual Homebrew requirement and failure messaging instead.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists on current main: select a missing brew-backed skill with no brew available, and `setupSkills` only shows the Homebrew command while `installSkill` returns `brew not installed` for the brew spec.

## Solution Assessment

Unclear. Automatically running the Homebrew installer may solve the user friction, but it is not clearly the best maintainable fix until maintainers decide whether OpenClaw should execute that remote installer from onboarding or gateway-controlled skill flows.
