## Summary

Keep open. Current main and the latest release still deliberately show Homebrew guidance or return a missing-brew failure, while automatic Homebrew bootstrap is a security-sensitive setup policy change that needs maintainer approval.

## What This Changes

The issue asks OpenClaw to automatically install Homebrew when skill dependency installation needs `brew` and Homebrew is missing.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway/UI, macOS, Linux, and containers.

## Reproduction Assessment

Yes. Source inspection on current main shows onboarding only displays the Homebrew command and the shared installer returns a brew-not-installed failure; no live terminal run was performed.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve first-run UX, but it is only the best solution after maintainers approve the trust, consent, platform, and failure policy.
