## Summary

Keep open. Current main and v2026.6.6 still keep Homebrew bootstrap manual for skill installs, and changing that would be a security-sensitive product decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to automatically install Homebrew when skill installation encounters a missing brew dependency.

## Best Possible Solution

Keep manual skill-install guidance unless maintainers approve an explicit-consent Homebrew bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway, macOS, and Linux/container behavior.

## Reproduction Assessment

Yes, at source level. Selecting a brew-backed skill with no brew reaches onboarding guidance that only displays the Homebrew command, then installSkill returns missing-brew guidance instead of bootstrapping Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve first-run UX, but it is only the best solution if maintainers approve the remote-installer trust, consent, platform, and failure policy.
