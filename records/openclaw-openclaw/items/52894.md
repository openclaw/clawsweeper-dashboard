## Summary

Keep open. Current main and v2026.5.20 still treat Homebrew bootstrap in skill installs as manual guidance, and turning that displayed command into an executed installer is a security-sensitive product decision rather than conservative cleanup.

## What This Changes

Request to let skill installation automatically install Homebrew when a selected brew-backed skill needs it and brew is missing.

## Best Possible Solution

Keep the manual skill-install behavior until maintainers approve an explicit-consent Homebrew bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway, macOS, and Linux/container behavior.

## Reproduction Assessment

Yes, at source level. Current main shows selected brew-backed skills with no brew only get a Homebrew command note, and installSkill still returns manual missing-brew guidance; this is a feature gap, not a bug against current docs.

## Solution Assessment

Unclear. Automatic Homebrew installation would reduce onboarding friction, but it is only the best solution if maintainers approve the remote-installer trust, consent, and platform policy; otherwise clearer manual guidance is safer.
