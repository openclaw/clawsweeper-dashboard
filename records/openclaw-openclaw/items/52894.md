## Summary

Keep open. Current main and the latest release still show or report manual Homebrew installation for brew-backed skill installs, and automatically bootstrapping Homebrew from skill flows requires maintainer product/security direction.

## What This Changes

The issue requests that OpenClaw automatically install Homebrew when a brew-backed skill install needs `brew` and no `brew` executable is present.

## Best Possible Solution

Keep this open for maintainer product/security direction; if approved, add an explicit opt-in Homebrew bootstrap shared by skill onboarding and gateway-backed skill installs, otherwise improve the manual Homebrew requirement and failure messaging.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: select a brew-backed missing skill with no `brew` executable, and onboarding only displays the Homebrew command while backend `skills.install` returns `brew not installed`.

## Solution Assessment

Unclear. Automatic Homebrew installation would address the friction, but maintainers need to approve the remote-installer trust and consent model before this becomes the best maintainable fix.
