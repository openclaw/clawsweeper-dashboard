## Summary

Keep open. Current main and v2026.6.8 still deliberately show Homebrew guidance or a missing-brew failure during skill installs, while automatic Homebrew bootstrap is a security-sensitive feature and product-policy decision.

## What This Changes

Request that skill installation automatically install Homebrew when a brew-backed skill dependency is selected and brew is unavailable.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway/UI, macOS, Linux, and containers.

## Reproduction Assessment

Yes. Source inspection shows the selected brew-backed missing-skill path displays Homebrew guidance, and the shared installer returns a missing-brew failure instead of bootstrapping Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve onboarding, but it is only the best solution after maintainers approve the remote-installer trust, consent, platform, and failure policy.
