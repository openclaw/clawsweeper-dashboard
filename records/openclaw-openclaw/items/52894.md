## Summary

Keep open. Current main and v2026.6.10 still show Homebrew guidance or return a missing-brew failure instead of bootstrapping Homebrew, and the remaining choice is a product/security decision rather than conservative cleanup.

## What This Changes

The issue requests automatic Homebrew installation when a skill dependency installer needs `brew` and no brew executable is installed.

## Best Possible Solution

Keep explicit Homebrew guidance unless maintainers approve a consent-first bootstrap policy; if approved, implement it through the shared skill install lifecycle with docs and coverage for onboarding, direct install, macOS, Linux, and container cases.

## Reproduction Assessment

Yes. Source inspection shows onboarding displays the Homebrew install command as text, and the direct skill installer returns a missing-brew failure when no brew executable resolves.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve onboarding, but it is only the best solution after maintainers approve the remote-installer trust, consent, platform, logging, timeout, and failure policy.
