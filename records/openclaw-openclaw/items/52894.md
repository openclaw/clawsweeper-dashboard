## Summary

Keep open. Current main and v2026.6.8 still intentionally show Homebrew guidance or return a missing-brew failure during skill installs, while automatic Homebrew bootstrap is a security-sensitive product decision rather than a narrow bug cleanup.

## What This Changes

Request to change skill onboarding so OpenClaw installs Homebrew automatically when a selected brew-backed skill needs it and brew is missing.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install lifecycle with docs and coverage for onboarding, direct install, macOS, Linux, and container cases.

## Reproduction Assessment

Yes. Source inspection shows the selected brew-backed missing-skill path displays Homebrew guidance, and direct brew installer specs return a missing-brew failure instead of bootstrapping Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve onboarding, but it is only the best solution after maintainers approve the remote-installer trust, consent, platform, and failure policy.
