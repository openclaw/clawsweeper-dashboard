## Summary

Keep open. Current main and v2026.6.10 still intentionally show Homebrew guidance or a missing-brew failure, and changing that into executing the Homebrew bootstrapper needs maintainer product/security direction.

## What This Changes

Request to let OpenClaw install Homebrew automatically when a selected skill needs a brew-backed installer and brew is missing.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install lifecycle with docs and coverage for onboarding, direct install, macOS, Linux, and container cases.

## Reproduction Assessment

Yes. Source inspection shows onboarding displays the Homebrew install command and direct brew-backed skill installs return missing-brew guidance instead of installing Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation may improve first-run UX, but it is only the best solution after maintainers approve the consent, supply-chain, platform, and failure policy.
