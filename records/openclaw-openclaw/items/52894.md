## Summary

Keep open. Current main and the latest release still leave Homebrew bootstrap as manual skill-install guidance, and executing that displayed installer from onboarding or gateway-backed skill installs is a security-sensitive product decision rather than conservative cleanup.

## What This Changes

The issue requests that OpenClaw automatically install Homebrew when a skill install needs brew but brew is missing.

## Best Possible Solution

Keep the manual skill-install behavior until maintainers approve an explicit-consent Homebrew bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway, macOS, and Linux/container behavior.

## Reproduction Assessment

Yes, at source level. Current main shows selected brew-backed skills with no brew only get a Homebrew command note, and `installSkill` still returns manual missing-brew guidance instead of executing a bootstrap installer.

## Solution Assessment

Unclear. Automatic Homebrew installation could reduce onboarding friction, but it is only the best solution if maintainers approve the remote-installer trust, consent, platform, and failure policy; otherwise clearer manual guidance is safer.

- Adjacent skill-install UX in https://github.com/openclaw/openclaw/issues/57555 may constrain the final design for brew-only skills and Docker/Linux environments.
