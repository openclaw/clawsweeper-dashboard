## Summary

Keep open. Current main and the latest release still deliberately show Homebrew guidance or return a missing-brew failure during skill installs, while the requested automatic Homebrew bootstrap crosses a security-sensitive setup policy boundary that needs maintainer approval.

## What This Changes

Request that skill onboarding and skill install flows automatically install Homebrew when a brew-backed skill is selected and `brew` is missing.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway/UI, macOS, Linux, and containers.

## Reproduction Assessment

Yes. Source inspection shows selecting a brew-backed missing skill without `brew` reaches manual Homebrew guidance, and the shared installer returns a missing-brew failure instead of bootstrapping Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve onboarding, but it is only the best solution after maintainers approve the remote-installer trust, consent, platform, and failure policy.
