## Summary

Keep open. Current main and v2026.5.19 still keep Homebrew bootstrapping out of skill installs, and turning the displayed command into an executed installer is a security-sensitive product decision rather than a conservative cleanup close.

## What This Changes

The issue requests that skill onboarding install Homebrew automatically when a selected skill needs brew and brew is missing.

## Best Possible Solution

Keep the manual skill-install behavior until maintainers approve an explicit-consent Homebrew bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway, macOS, and Linux/container behavior.

## Reproduction Assessment

Yes for the current feature gap: source inspection shows selected brew-backed skills with no brew only get a Homebrew command note, and installSkill still returns manual missing-brew guidance. This is not a bug against current docs, which explicitly say skill installs do not auto-install Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation would reduce onboarding friction, but it is only the best solution if maintainers approve the remote-installer trust, consent, and platform policy; otherwise clearer manual guidance is safer.
