## Summary

Keep open. Current main and v2026.6.6 still deliberately show or return Homebrew guidance instead of bootstrapping Homebrew during skill installs, and changing that crosses a security-sensitive setup policy boundary that needs maintainer approval.

## What This Changes

The issue asks OpenClaw skill/install onboarding to automatically install Homebrew when a brew-backed skill is selected and brew is missing.

## Best Possible Solution

Keep manual Homebrew guidance unless maintainers approve an explicit-consent bootstrap policy; if approved, implement it through the shared skill install boundary with docs and coverage for onboarding, gateway/UI, macOS, Linux, and container behavior.

## Reproduction Assessment

Yes, at source level. Selecting a brew-backed missing skill without brew reaches manual Homebrew guidance, and the shared installer returns a missing-brew failure instead of bootstrapping Homebrew.

## Solution Assessment

Unclear. Automatic Homebrew installation could improve onboarding, but it is only the best solution after maintainers approve the remote-installer trust, consent, platform, and failure policy.
