## Summary

Keep open: current main still has the Nostr setup-status path delegate into the full setup surface, while this PR keeps that status path on the lightweight setup entry and adds focused regression coverage. I found no blocking correctness or security issue in the patch; the remaining action is maintainer review and landing validation.

## What This Changes

The PR replaces Nostr's delegated setup wizard proxy with a local lightweight status wizard, adds a regression test that fails if the full setup surface loads, and updates the changelog.

## Best Possible Solution

Land the focused Nostr plugin fix after normal maintainer review and current-head validation, keeping setup status dependency-light while leaving full setup and runtime behavior unchanged.

## Reproduction Assessment

Yes. Source inspection shows current main's setup-status collector calls Nostr status resolvers that the proxy delegates through `loadWizard`, and the full setup import chain reaches `nostr-tools`.

## Solution Assessment

Yes. Keeping Nostr status resolvers local to `channel.setup.ts` is narrower than installing `nostr-tools` in the root package or changing the shared proxy contract for every channel.
