## Summary

Keep open: current main still delegates Nostr setup status through the full setup surface, and this PR is a focused, correct fix for that remaining dependency boundary with supplied real behavior proof and no blocking review findings.

## What This Changes

The PR replaces Nostr's setup-only delegated wizard status proxy with a local lightweight status wizard, adds a regression test that fails if `setup-surface.js` loads, and adds a changelog entry.

## Best Possible Solution

Land this small Nostr plugin patch after normal CI and maintainer merge, keeping setup status dependency-light while leaving full setup and runtime behavior unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows setup status collection calls resolvers that the Nostr setup-only proxy delegates through `loadWizard`, which imports `setup-surface.ts` and then `nostr-tools`; the PR test encodes that boundary by mocking the full surface to throw.

## Solution Assessment

Yes. Keeping only Nostr setup status local is narrower than changing shared proxy semantics or root-installing `nostr-tools`, and it preserves the full setup/runtime path for actual Nostr configuration.
