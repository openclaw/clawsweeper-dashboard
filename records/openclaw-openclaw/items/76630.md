## Summary

Current main has the generic external web-search runtime path needed for the Brave plugin case: agent-managed `web_search` late-binds the active runtime config/metadata and resolves runtime plugin providers instead of being limited to bundled provider metadata. The proof commit is main-only, so this should be treated as fixed on current main but not yet proven shipped in a release.

## What This Changes

Brave selected as the configured `web_search` provider should keep working after the 2026.5.2 external-plugin cutover, including doctor repair and agent-managed search execution.

## Best Possible Solution

Keep the generic external-plugin runtime resolution on main and let it ship in the next release; any remaining failure after that should be reported with fresh release logs.

## Reproduction Assessment

Yes. The issue includes a concrete v2026.5.2 path with Brave installed/configured, capability search succeeding, and agent `web_search` failing with `web_search is disabled or no provider is available`; I did not run a live Brave API call in this read-only sweep.

## Solution Assessment

Yes. The current solution is the narrow maintainable fix because it routes agent `web_search` through generic runtime plugin-provider metadata rather than adding a Brave-specific special case.
