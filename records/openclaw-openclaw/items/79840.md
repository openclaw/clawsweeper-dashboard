## Summary

Keep open: current main still parses `sessions_spawn.cwd` but the native `runtime="subagent"` path does not pass it into native spawn state, and the live fix path is an open PR that uses closing syntax but has not merged yet.

## What This Changes

The issue requests that native `sessions_spawn({ runtime: "subagent", cwd })` run delegated same-workspace subagents from the explicit `cwd`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81896 after final maintainer checks, then close this issue with the merge commit and any live native runtime proof.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd` and forwards it to ACP, while native spawn params lack `cwd` and cross-agent workspace resolution falls back away from the requested workspace. I did not run the live macOS/Telegram canary in this read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: carry `cwd` as internal native spawn state, resolve one child workspace, reuse it for run metadata and attachments, and keep Gateway agent params schema-clean.
