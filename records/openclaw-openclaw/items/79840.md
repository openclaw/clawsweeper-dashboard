## Summary

Keep open: current main still drops the explicit `cwd` on the native `sessions_spawn(runtime="subagent")` path, and the open implementation PR that closes this issue has not merged.

## What This Changes

Make native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` so delegated subagents run in the requester-specified workspace.

## Best Possible Solution

Review or repair the open fix PR so native subagent cwd is plumbed through one validated child-workspace resolver shared by spawned metadata and attachment materialization.

## Reproduction Assessment

Yes, at source level: current main parses `cwd` and forwards it to ACP, while the native `spawnSubagentDirect` call omits it and cross-agent workspace resolution drops the caller workspace before resolving the target agent workspace. I did not run the macOS/Telegram canary in this read-only review.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. Honoring explicit `cwd` is the right end state, but the implementation should validate the requested path and reuse the final child workspace consistently rather than passing arbitrary cwd through multiple paths.
