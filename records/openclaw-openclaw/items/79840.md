## Summary

Keep open: current main still reads a `cwd` argument for `sessions_spawn`, but only the ACP path forwards it; the native subagent path still resolves cross-agent spawns to the target agent workspace. The closest open fix, https://github.com/openclaw/openclaw/pull/58112, is not merged and was previously reviewed as incomplete around workspace resolution and attachments.

## What This Changes

The issue requests that native `sessions_spawn` with `runtime: "subagent"` honor an explicit `cwd` so delegated workers can run in the requester workspace rather than the target agent default workspace.

## Best Possible Solution

Implement bounded native-subagent cwd handling that validates allowed workspace boundaries, reuses one final child workspace for metadata and attachments, keeps `workspaceDir` internal to Gateway, and adds focused regression coverage plus docs/changelog.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and `cwd` set to the requester workspace, then inspect the child workspace. Current main reads `cwd` but only forwards it to ACP, while native cross-agent spawns resolve the target workspace.

## Solution Assessment

Unclear until a maintainer confirms the workspace-boundary policy. The best fix is not an arbitrary cwd pass-through; it should validate the explicit cwd and preserve the prior internal-only spawned-workspace hardening.
