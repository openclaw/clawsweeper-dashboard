## Summary

Keep open: current main still exposes and reads `sessions_spawn.cwd`, but the native subagent branch does not forward it, and the cross-agent native path deliberately resolves to the target agent workspace. The older closed reports fixed default target-agent workspace selection, not explicit native `cwd` overrides for same-workspace delegation.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` for delegated same-workspace execution.

## Best Possible Solution

Add bounded native-subagent cwd handling that passes explicit `cwd` separately from inherited workspace metadata, validates it against the approved workspace boundary, resolves the final child workspace once, and updates focused tests plus docs/changelog.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and an explicit `cwd`, then inspect the child workspace. Current main reads `cwd` but only forwards it to ACP, while native cross-agent spawns resolve to the target agent workspace.

## Solution Assessment

Unclear until maintainers confirm the workspace-boundary policy. The maintainable fix is not a raw `ctx.workspaceDir` pass-through; it should validate explicit cwd and preserve the prior internal-only workspace override hardening.
