## Summary

Keep open: current main still accepts and reads `sessions_spawn.cwd`, but only the ACP path forwards it; native cross-agent subagent spawns still resolve the target agent workspace instead of an explicit requested cwd. The linked older reports are adjacent target-workspace fixes, not a terminal fix for this explicit-cwd same-workspace delegation case.

## What This Changes

The issue requests native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` for same-workspace delegated execution.

## Best Possible Solution

Implement bounded native-subagent cwd handling only within approved requester/target workspace boundaries, keep workspace metadata internal to session state, then add regression tests and docs for native versus ACP cwd semantics.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`, then inspect the child workspace/cwd. Current main parses `cwd` but does not pass it to the native spawn path, and cross-agent native spawns resolve the target agent workspace.

## Solution Assessment

Unclear until a maintainer confirms the workspace-boundary policy. The maintainable fix is bounded validation of explicit native cwd, not an arbitrary model-supplied cwd pass-through.
