## Summary

Keep open: current main still parses `sessions_spawn.cwd` and forwards it for ACP spawns, but the native `runtime="subagent"` path drops it before workspace inheritance. The focused implementation PR is still open, so this issue should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue asks for native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` when delegating to another agent in the same workspace.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80053 with bounded native-subagent `cwd` handling, regression coverage, and proof that the child observes the requested workspace.

## Reproduction Assessment

Yes. A source-level reproduction is to call native `sessions_spawn` with a different `agentId` and explicit `cwd`; current main reads the value but never forwards it into native spawn params, so cross-agent workspace inheritance wins.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. The maintainable direction is to carry a validated explicit workspace fact through the native path, not to add arbitrary host-path steering.
