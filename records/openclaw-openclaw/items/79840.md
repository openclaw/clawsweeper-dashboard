## Summary

Keep open: current main still drops the parsed `cwd` on the native subagent path, and the directly matching fix PR was closed unmerged.

## What This Changes

The issue asks for `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` for same-workspace delegated subagent execution.

## Best Possible Solution

Thread explicit `cwd` as native spawn state, resolve the final child workspace once, use that workspace for metadata and attachments, keep Gateway params public-safe, and add focused regression coverage plus live canary proof.

## Reproduction Assessment

Yes, at source level: native `sessions_spawn` parses `cwd`, but the `spawnSubagentDirect` params object and type have no `cwd` field, so cross-agent workspace resolution still uses target or inherited workspace instead.

## Solution Assessment

Yes for a narrow fix: add `cwd` to `SpawnSubagentParams`, pass the parsed tool value into native spawn resolution as `explicitWorkspaceDir`, and preserve the internal-only Gateway workspace boundary.
