## Summary

Keep open: current main still exposes and reads `sessions_spawn.cwd`, and ACP forwards it, but the native `runtime="subagent"` path still drops it before workspace resolution; the direct fix PR was closed unmerged.

## What This Changes

The issue asks for native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` when delegating into the requester workspace.

## Best Possible Solution

Thread `cwd` as explicit native spawn state, keep the public Gateway schema unchanged, align spawned metadata and attachment workspace handling, and add focused regression plus live/native proof where feasible.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd` and ACP forwards it, while the native `spawnSubagentDirect` contract and call path drop it before workspace inheritance. I did not run a live macOS/Telegram canary in this read-only review.

## Solution Assessment

Yes for the code direction: add `cwd` to native spawn params and pass it as `explicitWorkspaceDir` into existing workspace resolution. Avoid the older context `workspaceDir` shortcut from https://github.com/openclaw/openclaw/pull/58112 and keep Gateway params schema-clean.
