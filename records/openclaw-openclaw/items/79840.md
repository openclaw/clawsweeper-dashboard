## Summary

This should stay open: current main accepts a `cwd` argument for `sessions_spawn`, but the native `runtime="subagent"` path still does not forward it, and cross-agent native spawns intentionally resolve to the target agent workspace instead.

## What This Changes

Request native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` for same-workspace delegated execution.

## Best Possible Solution

Add bounded native-subagent `cwd` handling that validates the requested directory against approved workspace boundaries, preserves the internal-only workspace metadata contract, and adds regression coverage plus docs for the supported semantics.

## Reproduction Assessment

Yes, source-level: call native `sessions_spawn` with `runtime="subagent"`, a different `agentId`, and explicit `cwd`; current main reads the value but only forwards it to ACP, while the native branch resolves cross-agent workspace from the target agent.

## Solution Assessment

Unclear until maintainers confirm the workspace-boundary policy. The narrow fix should not be an unconstrained `cwd` pass-through; it should validate the requested workspace and preserve the prior internal-only override hardening.
