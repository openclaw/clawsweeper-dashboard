## Summary

Keep open: current main still exposes and reads `sessions_spawn.cwd`, but the native `runtime="subagent"` path does not carry it into `SpawnSubagentParams` or workspace resolution. The linked fix PR is open and currently does not contain the runtime change, so this issue should remain available as the canonical bug report.

## What This Changes

Native `sessions_spawn(runtime="subagent")` should honor an explicit `cwd` for delegated same-workspace execution instead of falling back to the target agent workspace.

## Best Possible Solution

Carry a validated `cwd` through the native spawn contract into workspace inheritance, add focused native regression coverage for explicit `cwd` overriding cross-agent target workspace, and preserve the internal-only gateway boundary.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with `runtime="subagent"`, a different `agentId`, and explicit `cwd`; current main reads `cwd` but never forwards it into the native spawn params, so workspace resolution receives no explicit override. I did not run a live multi-agent canary because this review is read-only.

## Solution Assessment

Yes, if implemented narrowly: the maintainable fix is to pass a validated native `cwd` into `resolveSpawnedWorkspaceInheritance` and cover it with regression tests. The current open PR is not yet that solution because its latest diff is snapshot-only.
