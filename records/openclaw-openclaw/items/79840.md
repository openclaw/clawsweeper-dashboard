## Summary

Keep open: current main still reads `sessions_spawn.cwd` and forwards it only for ACP, while the native subagent path cannot receive it. An open PR now uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` so delegated agents run in the caller-selected workspace for same-workspace handoff flows.

## Best Possible Solution

Review and land or replace the linked fix PR with bounded native-subagent `cwd` handling that validates the requested workspace and adds regression coverage for cross-agent explicit cwd.

## Reproduction Assessment

Yes, at source level: invoke native `sessions_spawn` with a different `agentId` and explicit `cwd`, then inspect the child workspace. Current main reads `cwd`, forwards it only to ACP, and resolves native cross-agent runs through target/default workspace data.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. The maintainable solution is not arbitrary cwd passthrough; it should validate the explicit path and preserve the internal-only spawned workspace model.
