## Summary

Keep open: current main still drops `sessions_spawn.cwd` on the native subagent path, and the linked closing PR at https://github.com/openclaw/openclaw/pull/80053 is open rather than merged.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` for delegated same-workspace execution.

## Best Possible Solution

Review, revise, or replace the linked PR with bounded native `cwd` handling that canonicalizes and validates the requested workspace, adds regression coverage, and includes real same-workspace canary proof before this issue closes.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a cross-agent `agentId` and explicit `cwd`; current main reads `cwd` but only forwards it to ACP while native subagent spawning resolves to the target workspace.

## Solution Assessment

Unclear for the exact requested arbitrary `cwd` behavior. The maintainable solution should honor explicit native `cwd` only within an approved requester/target workspace boundary rather than trusting any tool-supplied host path.
