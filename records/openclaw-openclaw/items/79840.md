## Summary

Keep open: current main still accepts and reads `sessions_spawn.cwd`, but the native `runtime="subagent"` path does not carry it into native spawn parameters or workspace resolution. Open PRs are already attempting this area, but no merged/current-main fix is present.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` for delegated same-workspace execution.

## Best Possible Solution

Land or replace the open fix attempts with bounded native-subagent `cwd` handling that carries the explicit workspace through internal spawn metadata, preserves the gateway boundary, and adds regression coverage.

## Reproduction Assessment

Yes, at source level: current main reads `cwd` and forwards it to ACP, but the native spawn object/type omit it and cross-agent workspace resolution receives `explicitWorkspaceDir` as undefined. I did not run a live canary in this read-only review.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. The narrow maintainable fix is to carry a validated internal cwd/workspace fact through the native path, not to expose or reintroduce public gateway `workspaceDir` injection.
