## Summary

Keep open: current main at ac482047f5331c2ca5fd0876ac7235c02b1bf1e2 still reads `sessions_spawn.cwd`, but the native subagent path does not pass it into native workspace resolution; the overlapping fix PR remains open with unresolved review feedback and the workspace-boundary policy needs maintainer validation.

## What This Changes

The issue asks native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` for same-workspace delegated execution.

## Best Possible Solution

Implement bounded native-subagent `cwd` handling that validates the requested path against approved requester/target workspace boundaries, resolves the child workspace once, reuses it for metadata and attachments, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with `agentId` different from the requester and `cwd` set to the requester workspace. Current main reads `cwd`, forwards it only to ACP, and resolves native cross-agent spawns to the target agent workspace instead.

## Solution Assessment

Unclear until maintainers confirm the workspace-boundary policy. The safe direction is not arbitrary native `cwd` passthrough; it should validate the requested workspace and preserve the internal-only spawned workspace boundary.
