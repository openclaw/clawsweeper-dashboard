## Summary

This should stay open: current main still accepts and reads `sessions_spawn.cwd`, but the native `runtime="subagent"` path drops it before calling the subagent runner, while the ACP path forwards it. The remaining work is real, but it needs maintainer policy review because it changes spawned-workspace boundary behavior.

## What This Changes

The issue asks native `sessions_spawn(runtime="subagent")` cross-agent delegation to honor an explicit `cwd` so the child runs in the requester-selected same workspace.

## Best Possible Solution

Implement bounded native-subagent cwd handling that validates the requested directory against approved requester/target/sandbox workspace boundaries, then add regression coverage for explicit cross-agent `cwd`.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and an explicit `cwd`, then inspect the child workspace. Current main reads `cwd` but only the ACP branch forwards it; the native cross-agent path resolves to the target agent workspace.

## Solution Assessment

Unclear until maintainers confirm the workspace-boundary policy. The best fix is not an arbitrary cwd pass-through; it should validate explicit native cwd against the intended workspace/sandbox contract.
