## Summary

Keep open: current main still exposes and reads `sessions_spawn.cwd`, but the native `runtime="subagent"` path only forwards it to ACP and cross-agent native spawns intentionally resolve to the target agent workspace. An open implementation PR exists, but it needs maintainer review because this path overlaps prior workspace-boundary hardening.

## What This Changes

The issue asks native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` so delegated same-workspace subagents run in the requester-selected workspace.

## Best Possible Solution

Review and land a bounded fix, likely via https://github.com/openclaw/openclaw/pull/58112 or a replacement, that honors explicit native subagent cwd only inside approved workspace policy and keeps `workspaceDir` internal.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`, then inspect the child workspace/cwd. Current main parses `cwd`, forwards it only to ACP, and native cross-agent spawns resolve the target agent workspace instead.

## Solution Assessment

Unclear until maintainers confirm the workspace-boundary policy. The requested outcome is valid, but the best fix is a constrained native cwd handoff with regression coverage, not an unrestricted `cwd` to workspace override.
