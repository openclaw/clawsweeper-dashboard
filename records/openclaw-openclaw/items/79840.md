## Summary

Keep open: current main still drops the explicit `cwd` on the native `runtime="subagent"` path, and the current implementation candidate is open and unmerged with review/proof work still outstanding.

## What This Changes

Honor an explicit `sessions_spawn` `cwd` for native subagent delegation so same-workspace child runs resolve relative paths inside the requested workspace.

## Best Possible Solution

Carry the explicit cwd through the native subagent path as internal spawned workspace metadata, keep Gateway `agent` params schema-clean, and add focused regression coverage plus live proof.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`; current main reads the value but never forwards it into native spawn params, so workspace resolution falls back to existing inheritance rules.

## Solution Assessment

Yes, the narrow maintainable fix is to thread `cwd` into native spawned workspace resolution while preserving the internal-only gateway boundary; the open PR still needs cleanup and real behavior proof.
