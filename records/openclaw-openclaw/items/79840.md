## Summary

Keep open: current main still exposes and reads `sessions_spawn.cwd`, but the native `runtime="subagent"` path does not forward it, and the linked closing PR is still open rather than merged.

## What This Changes

Honor explicit `cwd` for native `sessions_spawn(runtime="subagent")` so delegated same-workspace subagents run in the requested workspace.

## Best Possible Solution

Land or replace the linked fix with bounded native-subagent `cwd` handling that validates the requested workspace against approved spawned-run boundaries and adds regression coverage.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`; current main reads the value but never forwards it into the native spawn params, so the child workspace remains resolved by existing inheritance rules.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. The narrow maintainable fix is not arbitrary `cwd` passthrough; it should carry a validated spawned workspace fact through the native path while preserving the internal-only gateway boundary.
