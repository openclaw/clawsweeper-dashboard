## Summary

Keep open: live main still accepts and reads `cwd`, but the native `runtime="subagent"` path drops it before workspace resolution. There are open implementation PRs for the same area, so the issue should remain as the tracking report until one lands or is replaced.

## What This Changes

The issue asks for native `sessions_spawn(runtime="subagent")` calls to honor an explicit `cwd` for same-workspace delegated execution.

## Best Possible Solution

Land or replace the open implementation work with a bounded native-subagent cwd path that carries the validated workspace fact once, uses it for child workspace resolution, avoids leaking `workspaceDir` to gateway `agent` params, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`; live main reads the value but never forwards it into `SpawnSubagentParams`, so workspace resolution cannot use it.

## Solution Assessment

Yes, the requested direction is the narrow fix if implemented through the existing workspace inheritance helper with validation. The open PRs still need review or replacement before the issue can close.
