## Summary

Keep open: current main still reads `sessions_spawn.cwd` but the native `runtime="subagent"` path does not forward it, and the two related fix PRs are not merged. https://github.com/openclaw/openclaw/pull/58112 remains open on the same surface but still needs repair/proof; https://github.com/openclaw/openclaw/pull/80053 was closed unmerged.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` when delegating same-workspace work to another agent.

## Best Possible Solution

Thread `cwd` as explicit native spawn state, resolve the final child workspace once, use that same value for spawned metadata and attachment materialization, add focused same-agent and cross-agent regression coverage, and require redacted live native subagent proof before closing the issue.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`; current main reads the value but never carries it into native spawn params, and cross-agent workspace resolution still drops inherited context. I did not run a live native subagent canary in this read-only review.

## Solution Assessment

Yes for the narrow direction: add `cwd` to native spawn params and pass it as `explicitWorkspaceDir` into the existing workspace resolver. The open context-workspace shortcut in https://github.com/openclaw/openclaw/pull/58112 is not sufficient for the reported cross-agent case.
