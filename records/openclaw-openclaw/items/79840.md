## Summary

Keep open: current main still reads `sessions_spawn.cwd` but drops it on the native `runtime="subagent"` path, and the two related fix PRs were closed without merging. The issue remains source-reproducible against the checked-out main branch.

## What This Changes

The issue asks for native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` for same-workspace delegated subagent execution.

## Best Possible Solution

Land a focused native-subagent cwd threading fix that carries `cwd` as explicit spawn state, preserves the existing Gateway boundary, and adds regression coverage for native `sessions_spawn` cwd precedence.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`; current main reads the value but never passes it to `spawnSubagentDirect`, so target-workspace inheritance wins.

## Solution Assessment

Yes, the narrow maintainable direction is to add `cwd` to `SpawnSubagentParams`, pass it from the tool layer, use it as `explicitWorkspaceDir`, and keep internal workspace metadata out of Gateway `agent` params.
