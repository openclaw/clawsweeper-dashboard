## What This Changes

Request that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` for same-workspace delegated execution.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81896 after maintainer handling and real native runtime proof, or keep this issue as the canonical bug if that PR is rejected.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd` and forwards it to ACP, but the native params have no `cwd` and cross-agent workspace resolution falls back to the target agent workspace. I did not run the live same-workspace canary.

## Solution Assessment

Yes, the narrow maintainable direction is to thread `cwd` as internal native spawn state, resolve one child workspace, reuse it for attachments and metadata, and keep `workspaceDir` out of public Gateway params.
