## Summary

Keep open: current main still exposes and reads `sessions_spawn.cwd`, but the native `runtime="subagent"` path does not forward it. The focused implementation path is the still-open https://github.com/openclaw/openclaw/pull/80053, which should be reviewed, proven, landed, or closed before creating a replacement.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` so delegated subagents run in the requester-specified workspace.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80053 with bounded native-subagent `cwd` forwarding, regression coverage, and redacted live proof, then close this issue from the merge commit.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`; current main reads `cwd` but never passes it into `spawnSubagentDirect`, whose params cannot carry `cwd`.

## Solution Assessment

Yes for the code direction: threading `cwd` through `SpawnSubagentParams` into the existing explicit-workspace resolver is the narrow maintainable fix, as long as it preserves the internal spawned-workspace boundary.
