## Summary

Keep open: current main still parses `sessions_spawn.cwd` but drops it on the native subagent path, while the older closed reports only fixed the default cross-agent target-workspace behavior. There is an open related implementation PR, but the fix needs maintainer review because cwd propagation intersects prior spawned-workspace boundary hardening.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` for same-workspace delegated execution.

## Best Possible Solution

Implement bounded native-subagent cwd handling that validates explicit cwd against approved requester/target workspace policy, keeps workspace metadata internal, and adds regression coverage for same-workspace delegation.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with `agentId` different from the requester and an explicit `cwd`; current main reads `cwd` but only forwards it to ACP, while native cross-agent resolution falls back to the target agent workspace. I did not run the macOS/Telegram canary locally.

## Solution Assessment

Unclear until a maintainer confirms the workspace-boundary policy. The best fix is a constrained native cwd handoff, not an arbitrary `cwd` to `workspaceDir` pass-through.
