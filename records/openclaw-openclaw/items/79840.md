## What This Changes

The issue asks native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` so delegated same-workspace runs execute in the requested workspace.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81896 after maintainer handling and redacted runtime proof that a native spawned child observes the requested cwd.

## Reproduction Assessment

Yes, at source level: call `sessions_spawn` with `runtime="subagent"`, a different `agentId`, and explicit `cwd`; current main reads `cwd` but never forwards it into native spawn params. I did not run the live macOS/Telegram canary in this read-only review.

## Solution Assessment

Yes, the best path is the focused open PR that threads `cwd` as native spawn state, resolves one child workspace, and keeps Gateway agent params schema-clean. Starting a competing repair would be worse while that closing PR is open.
