## Summary

Keep open: current main still exposes and reads `sessions_spawn.cwd`, but only ACP receives it; the native subagent path drops it before `spawnSubagentDirect`, and the overlapping PR remains open with unresolved workspace-boundary concerns.

## What This Changes

The issue asks native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` so delegated workers can run in the requester workspace instead of the worker agent default workspace.

## Best Possible Solution

Define bounded native-subagent `cwd` handling that validates the requested workspace against approved requester/target boundaries, reuses one resolved child workspace consistently, and adds regression coverage.

## Reproduction Assessment

Yes, at source level: invoke native `sessions_spawn` with a cross-agent `agentId` and explicit `cwd`, then inspect the child workspace. Current main reads `cwd`, forwards it only to ACP, and resolves native cross-agent spawns to the target agent workspace.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. The maintainable fix is bounded validation and single child-workspace resolution, not arbitrary `cwd` passthrough.
