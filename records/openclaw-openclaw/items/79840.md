## Summary

This should stay open: current main still exposes a `cwd` parameter for `sessions_spawn`, but the native subagent path does not forward it, and cross-agent native spawns intentionally resolve to the target agent workspace instead.

## What This Changes

Request that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` so a cross-agent worker can run inside the requester workspace for same-workspace handoff flows.

## Best Possible Solution

Implement bounded native-subagent cwd handling that honors explicit cwd only within approved workspace boundaries, then add regression tests and document the parameter.

## Reproduction Assessment

Yes, at source level: invoke native `sessions_spawn` with `agentId` different from the requester and `cwd` set to the requester workspace, then inspect the child cwd/workspace. Current main reads `cwd` but only forwards it to ACP, while native cross-agent spawns resolve the target workspace.

## Solution Assessment

Unclear until a maintainer confirms the workspace-boundary policy. The narrow maintainable fix is not an arbitrary cwd pass-through; it should validate the explicit cwd and preserve prior spawned-workspace security constraints.
