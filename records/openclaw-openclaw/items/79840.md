## Summary

Keep this open: current main still accepts a `cwd` argument for `sessions_spawn`, but the native `runtime="subagent"` path does not pass it into `spawnSubagentDirect`, while cross-agent native spawns intentionally resolve the child workspace from the target agent.

## What This Changes

Honor explicit `cwd` for native `sessions_spawn(runtime="subagent")` same-workspace delegation instead of forcing cross-agent spawns to the target agent workspace.

## Best Possible Solution

Implement bounded native-subagent `cwd` handling that validates the requested directory against approved workspace boundaries, then add regression tests and document the semantics.

## Reproduction Assessment

Yes. At source level, invoke native `sessions_spawn` with a different `agentId` and an explicit `cwd`; current main reads the argument but only forwards it to ACP, while native cross-agent spawns resolve the target agent workspace.

## Solution Assessment

Unclear until maintainers confirm the workspace-boundary policy. The maintainable fix should not be an arbitrary path pass-through; it should validate explicit `cwd` and preserve spawned-run security constraints.
