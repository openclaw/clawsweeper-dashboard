## Summary

Keep this open: current main still accepts and reads `cwd` for `sessions_spawn`, but the native subagent path only forwards it to ACP, while cross-agent native spawns resolve the target agent workspace instead of the explicit requested cwd.

## What This Changes

Honor explicit `cwd` for native `sessions_spawn(runtime="subagent")` same-workspace delegated execution.

## Best Possible Solution

Add bounded native-subagent cwd handling that validates the explicit cwd against approved workspace boundaries, threads it into native spawn metadata, and adds focused regression tests plus docs for the parameter.

## Reproduction Assessment

Yes, at source level: invoke native `sessions_spawn` with `runtime: "subagent"`, a different `agentId`, and `cwd` set to the requester workspace. Current main reads `cwd` but forwards it only to ACP, while native cross-agent spawns resolve the target workspace.

## Solution Assessment

Unclear until a maintainer confirms the workspace-boundary policy. The maintainable fix is not arbitrary cwd pass-through; it should validate the explicit cwd and preserve spawned-run sandbox/workspace constraints.
