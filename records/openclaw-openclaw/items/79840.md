## Summary

Keep open: current main still accepts and reads `sessions_spawn.cwd`, but only the ACP path forwards it; the native subagent path does not, and the related open PR remains unresolved.

## What This Changes

The issue asks native `sessions_spawn(runtime="subagent")` to honor an explicit `cwd` so a delegated worker can run in the requester workspace.

## Best Possible Solution

Define and implement bounded native-subagent `cwd` handling that validates the requested workspace against approved requester/target boundaries, reuses the resolved child workspace consistently, and adds regression coverage.

## Reproduction Assessment

Yes, at source level: invoke native `sessions_spawn` with `agentId` different from the requester and `cwd` set to the requester workspace, then inspect the child workspace. Current main reads `cwd`, forwards it only to ACP, and resolves native cross-agent spawns to the target agent workspace.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. The maintainable fix is not arbitrary `cwd` passthrough; it should validate the explicit path and preserve the internal-only spawned workspace boundary.
