## Summary

Keep open: current main still drops the explicit `cwd` on the native `runtime="subagent"` path, and an open fixing PR with closing syntax is already linked, so this issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that native `sessions_spawn(runtime="subagent")` use an explicit `cwd` for same-workspace delegated execution instead of resolving to the target agent default workspace.

## Best Possible Solution

Review and land a bounded version of https://github.com/openclaw/openclaw/pull/80053 that validates explicit cwd against approved requester/target workspace boundaries and adds regression coverage for native subagent cwd precedence.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and an explicit requester-workspace `cwd`, then inspect the child workspace. Current main parses `cwd` but only forwards it to ACP, while native cross-agent spawns still resolve through target workspace inheritance.

## Solution Assessment

Unclear until the workspace-boundary policy is confirmed. The best fix is a bounded native-subagent cwd override with validation, not an unconstrained arbitrary path passthrough.
