## Summary

Keep open: current main still reads `sessions_spawn.cwd`, but the native `runtime="subagent"` path drops it before workspace resolution, and the only direct fix PR, https://github.com/openclaw/openclaw/pull/80053, is closed unmerged.

## What This Changes

Request that native `sessions_spawn(runtime="subagent")` honor an explicit `cwd` for same-workspace delegated subagent execution.

## Best Possible Solution

Thread a validated explicit `cwd` through native subagent spawn as internal spawned-workspace metadata, keep `workspaceDir` out of public Gateway `agent` params, and add regression plus live canary proof.

## Reproduction Assessment

Yes, at source level: call native `sessions_spawn` with a different `agentId` and explicit `cwd`; current main reads the value but never forwards it into native spawn params, so workspace resolution falls back to existing inheritance rules.

## Solution Assessment

Yes for the direction: adding `cwd` to the native spawn params and passing it to `resolveSpawnedWorkspaceInheritance` as `explicitWorkspaceDir` is the narrow fix, provided it stays internal and does not expand the public Gateway schema.
