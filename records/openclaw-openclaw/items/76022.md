## Summary

Current main already addresses the reported v2026.4.29 packaged-channel dependency failure: still-bundled Slack/Telegram runtime dependencies are root-owned again, Discord/Nostr have moved to installable plugin packages with update-time bridge handling, and status/config paths no longer depend on the removed bundled runtime-deps repair path.

## What This Changes

The issue reports that upgrading from 2026.4.27 to 2026.4.29 caused bundled channel loading to fail for Discord, Nostr, Slack, and Telegram due to missing runtime modules.

## Best Possible Solution

Keep the current main dependency architecture: bundled plugins that remain packaged must have root-owned runtime deps, externalized plugins should migrate through the install/update bridge, and runtime/status should stay read-only rather than repairing package installs.

## Reproduction Assessment

Yes. The issue provides concrete missing-module errors after upgrading to v2026.4.29, and current-main inspection shows the exact missing Slack/Telegram dependencies are now root-owned while Discord/Nostr no longer ship as bundled dist trees.

## Solution Assessment

Yes. The current implementation is the maintainable fix because it resolves packaged runtime deps at install/package boundaries and uses update-time migration for externalized bundled plugins instead of reviving startup-time dependency repair.
