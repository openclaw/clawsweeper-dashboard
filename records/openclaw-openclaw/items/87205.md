## Summary

Keep this PR open for maintainer review. Current main still collapses a gateway protocol mismatch into the generic dead-gateway path, while this patch is focused and now has inspectable Windows terminal-output proof; the remaining question is whether maintainers accept the changed doctor repair behavior for upgrades and service fallback states.

## What This Changes

The PR classifies gateway protocol mismatch in doctor health checks, passes that state into daemon repair, skips dead-gateway repair on mismatch while still allowing stale service-definition repair, exports the service-repair issue collector, and adds focused regression tests.

## Best Possible Solution

Land a narrow doctor/gateway repair change only after maintainers accept the upgrade behavior and any desired Scheduled Task or non-Windows proof; keep https://github.com/openclaw/openclaw/issues/87156 open for the broader Windows stale-service/update repair problem.
