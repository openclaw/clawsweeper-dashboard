## Best Possible Solution

Forward enum-validated `search_context_size` through both Perplexity transports where upstream supports it, reject only documented conflicts with token budget parameters, and refresh docs/tests/proof for both paths.

## Reproduction Assessment

Yes for source-level review: current main auto-selects the native Search API for direct Perplexity credentials, and the PR head rejects `search_context_size` on that route. I did not run a live Perplexity call because credentials are not available in this review environment.

## Solution Assessment

No. The implementation is close, but the best fix should support the native Search API's documented `search_context_size` field instead of making the newly shared parameter fail for default direct Perplexity users.
