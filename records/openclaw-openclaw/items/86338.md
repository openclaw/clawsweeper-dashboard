## Summary

Keep this PR open, but it is not merge-ready: the branch adds a useful `search_context_size` tool hint, yet rejects it on the default direct Perplexity Search API route even though Perplexity documents the field on `/search`; the supplied proof only covers chat-completions body construction.

## What This Changes

Adds a shared `web_search.search_context_size` parameter, forwards it on Perplexity chat-completions, rejects it on the native Search API path, and updates Perplexity docs, tests, and prompt snapshots.

## Best Possible Solution

Forward enum-validated `search_context_size` through both Perplexity transports, reject only documented token-budget conflicts on the native path, then refresh docs, tests, and proof around both routes.

## Reproduction Assessment

Yes, at source level: current main routes direct Perplexity credentials to `search_api`, and the PR head returns `unsupported_search_context_size` for that route before any network call.

## Solution Assessment

No. The implementation is close, but the best fix should support Perplexity's documented native Search API field and reject only the documented conflict with explicit token-budget fields.
