## Summary

Keep this PR open, but do not merge it as-is. Current main has not superseded the intended iOS LAN TLS and Swift auth-precedence changes, but this PR breaks the iOS test target and needs a maintainer security-policy decision before allowing plaintext WebSocket routes for Tailscale-style hosts.

## What This Changes

The PR changes iOS manual gateway TLS host classification, adds Swift gateway security tests, and changes OpenClawKit GatewayChannel auth selection so explicit passwords suppress bootstrap-token auth.

## Best Possible Solution

Revise this PR or a successor before merge: move the added tests inside the suite, update the stale .local expectation, keep private-LAN plaintext behavior explicitly bounded, preserve Tailscale/public routes on wss:// or Serve/Funnel unless maintainers approve a policy change, and coordinate any remaining setup-code or TypeScript auth-precedence work through the linked issue.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main upgrades non-loopback manual iOS hosts to TLS, the test suite expects openclaw.local to force TLS, and mixed Swift auth still selects bootstrap before password; the PR's new tests would also fail to compile because they are outside the suite.

## Solution Assessment

No, not as-is. The password-over-bootstrap change is narrow and plausible, but the LAN TLS change overreaches into Tailscale/plaintext policy and the test patch is structurally broken, so this needs revision before it is the best maintainable fix.
