## Summary

Keep open. Current main still has the iOS TLS and Swift auth behavior this PR targets, and the linked #47887 issue remains open, so the PR is not obsolete. The branch is not merge-ready because it breaks the iOS test target, conflicts with an existing expectation, broadens plaintext WebSocket eligibility to tailnet endpoints contrary to current mobile transport docs, and lacks the required changelog coverage.

## What This Changes

The PR changes iOS manual gateway TLS host classification, adds Swift gateway security tests for LAN/Tailscale hosts, and changes OpenClawKit gateway auth selection so an explicit password suppresses bootstrap-token auth.

## Best Possible Solution

Revise this PR or a successor before merge: keep private-LAN plaintext behavior bounded and consistent with the shared local-network helper, preserve tailnet/public mobile routes on `wss://` or Tailscale Serve/Funnel unless maintainers approve a policy change, fix the Swift tests, add the active-version changelog entry, and coordinate the remaining setup-code and TypeScript auth work through #47887.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists from current main: manual iOS non-loopback hosts still force TLS, mixed Swift password/bootstrap auth still selects bootstrap first, and the PR's added tests would compile outside the suite while calling a private helper.

## Solution Assessment

No, not as-is. The password precedence change is narrow and plausible, but the TLS change overreaches into tailnet plaintext policy and the test patch is structurally broken, so the maintainable fix should be narrower and policy-aligned.
