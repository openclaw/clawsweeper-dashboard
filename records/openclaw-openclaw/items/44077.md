## Summary

Keep open. Current main still has the reported sandbox write behavior: the sandbox mutation helper creates a 0600 temp file and atomically replaces the target without applying the original target mode in the write path. The closed fix PRs were not merged, and the related open PR addresses host-side default modes rather than this sandbox helper path.

## What This Changes

Sandbox file-tool edit/write should preserve existing workspace file permissions so host-side OpenClaw file tools do not lose access after sandbox-backed edits.

## Best Possible Solution

Patch `write_atomic` in the sandbox mutation helper to capture the existing regular target's mode and apply it to the temp file before `os.replace`, while preserving the current symlink, hardlink, pinned-parent, and boundary checks. Keep any new-file default-mode policy separate unless #44260 or maintainer review makes that product decision explicit.

## Reproduction Assessment

Yes. The reporter's POSIX stat sequence has a high-confidence source match: sandbox edit writes currently flow to `write_atomic`, which creates a 0600 temp file and replaces the target without applying the original 0644 mode.

## Solution Assessment

Yes.
