## Summary

Keep open. Current main still only blocks service-marker based package updates, while this PR adds the missing active-Gateway PID ancestry guard for the related update footgun; the patch looks functionally sound, but it is missing the required user-facing changelog entry.

## What This Changes

This PR exports the existing bounded PID ancestry helper, uses it to abort restartful package-manager updates when the managed Gateway runtime PID is in the caller ancestry, and adds CLI regression coverage for that abort path.

## Best Possible Solution

Land one coherent updater guard that rejects unsafe Gateway-owned restartful package updates before service stop or package mutation, keeps the existing external-shell update path intact, and records the user-facing fix in the Unreleased changelog.

## Reproduction Assessment

Yes. #75691 provides concrete systemd/Gateway-owned exec steps, current main shows only the inherited service-marker guard, and the PR adds a focused mocked-runtime PID ancestry test that exercises the missing path.

## Solution Assessment

Yes, with the changelog gap fixed. Reusing the existing bounded ancestry walker and failing before the managed-service stop is the narrowest maintainable fix for the reported lifecycle failure; a detached restart helper would be a larger separate design.
