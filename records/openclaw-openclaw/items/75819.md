## Summary

Keep open for normal PR review and exact-head CI. Current main still lacks the no-marker PPID ancestry guard tracked by #75691, and this PR now adds the focused guard, regression coverage, and changelog entry; I did not find a concrete diff bug.

## What This Changes

This PR exports the existing PID ancestry walker, uses it in `openclaw update` to abort restartful package updates when the active Gateway PID is in the caller ancestry, adds a regression test, and documents the fix in the changelog.

## Best Possible Solution

Land this narrow ancestry guard after exact-head CI is reconciled, preserving the merged #75729 service-marker guard and the new Unreleased changelog entry.

## Reproduction Assessment

Yes. #75691 gives systemd plus agent-exec steps for a no-marker Gateway-owned child, current main only has the service-marker guard, and the PR comments include live VPS proof plus a regression test with the active runtime PID in the caller ancestor set.

## Solution Assessment

Yes. Reusing the existing bounded ancestry walker and failing before service stop or package replacement is the narrowest maintainable fix for the reported restart footgun; a detached restart helper would be a larger separate design.
