## What This Changes

Adds Codex CLI transcript discovery/parsing for Gateway `chat.history`, multi-provider CLI history merging, reset-time CLI import suppression metadata, tests, and a changelog entry.

## Best Possible Solution

Extend the Codex history importer to parse current rollout `response_item` message records as well as event messages, keep the reset-suppression contract, and refresh proof against those transcript shapes before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `chat.history` only imports Claude CLI transcripts, so a `codex-cli`-bound session with an empty local transcript has no Codex import source.

## Solution Assessment

No. The seam and reset-suppression direction are sound, but the parser should support persisted Codex `response_item` message records instead of intentionally ignoring them.
