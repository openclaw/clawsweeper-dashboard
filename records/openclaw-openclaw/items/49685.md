## Summary

Keep open: the progressive-disclosure direction is reasonable and prior review feedback was partly addressed, but the current PR head still has two concrete merge blockers in the skill guidance and metadata. Both are narrow documentation/config-contract fixes rather than reasons to close the PR.

## What This Changes

This PR splits bundled `coding-agent`, `gh-issues`, `skill-creator`, and `xurl` skill instructions into shorter `SKILL.md` files with on-demand reference files for the larger examples and workflows.

## Best Possible Solution

Keep the lean `SKILL.md` plus `references/` pattern, but land it only after the skill-creator layout diagram matches the documented sibling resource layout and the `gh-issues` dependency metadata matches the declared curl-only workflow or retains a valid installer for any required CLI.

## Reproduction Assessment

Yes. The two blockers reproduce by static inspection of PR head `ad1ecc166999ad69a048b8d0f5809de4f7d0f8ab` against current `main`: the skill tree nests sibling directories incorrectly, and `gh-issues` metadata gates on `gh` while the rewritten instructions say not to use it.

## Solution Assessment

No. The overall split is a maintainable direction, but the current patch is not the best mergeable version until the resource layout and dependency metadata are corrected in the changed files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep scripts and assets as skill-root siblings:** `skills/skill-creator/SKILL.md:28-30`
  - body: The new tree shows `scripts/` and `assets/` under `references/`, but the initializer and the existing skill guidance create `scripts/`, `references/`, and `assets/` as sibling directories at the skill root. Agents following this diagram will build the wrong layout, and later instructions that reference top-level `scripts/...` paths become inconsistent.
  - confidence: 0.96
- **[P2] Align gh-issues metadata with the curl-only workflow:** `skills/gh-issues/SKILL.md:6`
  - body: The rewritten skill says not to use the `gh` CLI, but the frontmatter still requires `gh` and now omits the previous install metadata. Since OpenClaw checks `requires.bins` for eligibility and uses `metadata.openclaw.install` for missing-bin remediation, users without `gh` can still lose the skill even though the workflow is documented as curl-only. Remove `gh` from the required bins if it is no longer needed, or keep the installer metadata if it remains a hard dependency.
  - confidence: 0.84
