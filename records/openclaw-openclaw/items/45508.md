## Summary

Keep open: current main removed the old browser speech controls and has gateway TTS/Talk primitives, but the requested per-message gateway Read aloud action is still only in an open draft PR, and WebChat server-side dictation is not present on current main.

## What This Changes

Request WebChat voice controls that route Read aloud and mic dictation through gateway-backed configured TTS/STT instead of browser Web Speech APIs.

## Best Possible Solution

Land or replace the gateway Read aloud work using `tts.convert`, then decide whether WebChat dictation should be restored through `tools.media.audio` or folded into Talk without adding `messages.stt`.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks both the requested Read aloud action and `chat.transcribeAudio`, while the stale browser speech buttons are gone.

## Solution Assessment

No, not exactly as proposed. TTS should reuse `tts.convert`, and STT should reuse `tools.media.audio` or Talk contracts instead of introducing a parallel `messages.stt` config block.
