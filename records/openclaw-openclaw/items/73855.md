## Summary

Keep open. Current main does not have an equivalent hosted-installer sync workflow, and the drift guard is useful for the related hosted-installer report, but the current PR head still has the two CI blockers already identified in review: installer-source PRs would compare proposed bytes against deployed CDN bytes, and scheduled checks can share cancellation with main push runs.

## What This Changes

The PR adds `.github/workflows/check-hosted-installer-sync.yml`, which hashes `scripts/install.sh`, downloads `https://openclaw.ai/install.sh`, compares SHA-256 checksums, and runs on main pushes, selected pull requests, a daily schedule, and manual dispatch.

## Best Possible Solution

Keep the checksum guard, but make the live source-vs-host comparison a post-merge, scheduled, or manual signal rather than a PR check for `scripts/install.sh` changes. The workflow should also separate schedule/push concurrency and avoid closing #73837 until the hosted installer sync problem is resolved or explicitly tracked by the deployment pipeline.

## Reproduction Assessment

Yes. The current PR head can be inspected directly at `64534f91b08ae0afe1919bcf20474f5257d153b0`: line 11 still enables installer PR checks, line 22 still has event-agnostic concurrency, and current source/hosted installer hashes differ.

## Solution Assessment

No. The proposed workflow is a useful guard, but this is not the best mergeable form because it compares in-flight PR bytes to deployed CDN bytes and can suppress the daily schedule through cross-event cancellation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Skip live installer comparison for installer PRs:** `.github/workflows/check-hosted-installer-sync.yml:11`
  - body: This workflow runs on pull requests that change `scripts/install.sh`, but the PR checkout contains the proposed installer while `openclaw.ai` still serves the last deployed installer. Since the compare step exits on any hash mismatch, future installer-source PRs will fail until after merge and deployment. Remove this path from `pull_request.paths`, or skip the live checksum comparison for pull request events.
  - confidence: 0.97
- **[P2] Separate schedule and push concurrency:** `.github/workflows/check-hosted-installer-sync.yml:22`
  - body: `schedule` and `push` runs on main can share `refs/heads/main` in `github.ref`, so `cancel-in-progress: true` lets a main push cancel an in-flight scheduled drift check. Include `github.event_name` or another event discriminator in the concurrency group.
  - confidence: 0.91
