## Summary

Keep open. Current main does not contain an equivalent hosted-installer drift guard, but this PR head is not mergeable because the live diff still has the two previously reported CI blockers: installer-source PRs would compare in-flight bytes against the already-deployed CDN, and scheduled checks can share a cancellation group with main pushes. The author follow-up comment saying both were fixed is not reflected in the current PR head.

## What This Changes

The PR adds a new GitHub Actions workflow that hashes `scripts/install.sh`, fetches `https://openclaw.ai/install.sh`, compares the two digests, and runs on push, pull request, daily schedule, and manual dispatch.

## Best Possible Solution

Keep the checksum guard, but make the live source-vs-host comparison a post-merge, scheduled, or manual signal rather than a pull-request check for installer source changes. The workflow should also isolate scheduled concurrency from push concurrency and preserve the repo’s existing low-permission, workflow-lintable CI conventions.

## Reproduction Assessment

Yes for the introduced CI failure mode. The current PR head still includes `scripts/install.sh` in the `pull_request` path filter, the source and hosted installer hashes differ, and the PR head’s new sync check is failing.

## Solution Assessment

No. The idea is useful, but this implementation is not the best mergeable form because it compares PR checkout bytes against deployed CDN bytes and can cancel scheduled drift alerts from unrelated main pushes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Limit live-host comparisons on pull requests:** `.github/workflows/check-hosted-installer-sync.yml:11`
  - body: This workflow runs on pull requests that change `scripts/install.sh`, but the PR checkout contains the proposed installer while `openclaw.ai` still serves the last deployed installer. Because the job fails on any hash mismatch, future installer-source PRs will fail until after merge and deployment. Remove this path from `pull_request` validation, or skip the live checksum comparison for pull request events.
  - confidence: 0.97
- **[P2] Separate scheduled checks from push cancellation:** `.github/workflows/check-hosted-installer-sync.yml:22-23`
  - body: `schedule` and `push` runs on `main` both use `refs/heads/main` for `github.ref`, so `cancel-in-progress: true` can let a main push cancel the daily drift check. Include `github.event_name` in the concurrency group or otherwise avoid cross-event cancellation.
  - confidence: 0.91
