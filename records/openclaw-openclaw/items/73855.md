## Summary

Keep open. The drift guard is useful, but the proposed live comparison still runs directly on installer-source pushes, which can race the existing website sync pipeline and leave main red before the CDN is updated.

## What This Changes

Adds a GitHub Actions workflow that fetches `openclaw.ai/install.sh` and compares its SHA256 to `scripts/install.sh` on schedule, manual dispatch, main pushes, and workflow-file PRs.

## Best Possible Solution

Keep the drift guard, but run the live CDN comparison after `Website Installer Sync` succeeds for a trusted main push, or keep live checks to schedule/manual while PRs only validate workflow syntax.

## Reproduction Assessment

Yes. Source inspection shows the PR's push trigger can fetch the live CDN before `website-installer-sync` publishes, and the PR check log shows the workflow failing on a live checksum mismatch.

## Solution Assessment

No. The direction is useful, but running the live CDN comparison directly on installer-source pushes is not the narrowest maintainable solution because it can race the existing sync workflow.
