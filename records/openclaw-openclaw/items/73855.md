## Summary

Keep open. The latest PR head still contains the two earlier workflow defects despite the author’s follow-up comment, its own hosted-installer check is failing, and the external PR has no after-fix real behavior proof.

## What This Changes

The PR adds a `check-hosted-installer-sync` GitHub Actions workflow that downloads the hosted installer and compares its SHA256 to `scripts/install.sh` on push, pull request, schedule, and manual dispatch.

## Best Possible Solution

Keep the live drift guard idea, but make it schedule/post-merge/manual only or fold it into `website-installer-sync.yml`, with event-separated concurrency and real behavior proof from a successful run.

## Reproduction Assessment

Yes. The live PR diff still contains the problematic `pull_request.paths` entry and concurrency group, and the PR head’s added hosted-installer check is failing.

## Solution Assessment

No. The current implementation checks undeployed PR bytes against deployed CDN bytes and lets scheduled drift checks share cancellation with main pushes; a safer fix would restrict live comparison to schedule/post-merge/manual events and reconcile it with the existing website installer sync workflow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Limit live-host comparisons on pull requests:** `.github/workflows/check-hosted-installer-sync.yml:11`
  - body: This workflow runs on pull requests that change `scripts/install.sh`, but the PR checkout contains the proposed installer while the hosted URL still serves the last deployed installer. Future installer-source PRs will fail before deployment can make the hosted copy match, so remove this path from pull request validation or skip the live checksum comparison for pull request events.
  - confidence: 0.97
- **[P2] Separate scheduled checks from push cancellation:** `.github/workflows/check-hosted-installer-sync.yml:22-23`
  - body: `schedule` and `push` runs on `main` both use `refs/heads/main` for `github.ref`, so `cancel-in-progress: true` can let a main push cancel the daily drift check. Include `github.event_name` in the concurrency group or otherwise prevent cross-event cancellation.
  - confidence: 0.91
