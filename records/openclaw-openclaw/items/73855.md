## Summary

Keep open: the earlier PR-trigger and concurrency defects were fixed, but the new live CDN checksum still runs on the same `push` that changes `scripts/install.sh`, before the existing website sync workflow can publish that source to openclaw.ai.

## What This Changes

The PR adds a GitHub Actions workflow that compares `scripts/install.sh` against the hosted `openclaw.ai/install.sh` on scheduled, push, manual, and workflow-file PR events.

## Best Possible Solution

Keep the hosted-installer drift guard, but run the live CDN comparison only after the installer sync/deploy path can publish the same source, or keep it to schedule/manual drift checks with normal workflow syntax validation handled separately.

## Reproduction Assessment

Yes. Source inspection shows the proposed workflow runs a live CDN checksum on `push` for `scripts/install.sh`, while current main only syncs that file to openclaw.ai later in a separate workflow; the live source and hosted hashes differ today.

## Solution Assessment

No. The direction is useful, but running the live comparison directly on the installer-source push races the existing deployment workflow; a post-sync `workflow_run`, scheduled/manual guard, or explicit wait/retry tied to deployment would be narrower.
