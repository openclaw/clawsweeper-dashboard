## Summary

Keep open: the fix is focused and the new MiniMax proof is sufficient, but the patch changes provider precedence by trying MiniMax text candidates before existing generic image candidates, which can silently reroute users who already have an authenticated image-capable provider such as OpenAI.

## What This Changes

The branch updates PDF model auto-resolution and tests so MiniMax text chat models become PDF extraction fallback candidates and MiniMax-VL-01 is filtered out of automatic PDF fallback candidates.

## Best Possible Solution

Keep the MiniMax text fallback, but select it only for MiniMax-default or MiniMax-only cases, or otherwise after the existing generic image-provider order, with regression coverage for mixed OpenAI-plus-MiniMax auth.

## Reproduction Assessment

Yes by source inspection for the MiniMax-only bug: current main can choose MiniMax-VL-01 for automatic PDF fallback while the PDF runtime resolves the model before extraction. The contributor also supplied redacted after-fix MiniMax runtime proof, though I did not run a live MiniMax call myself.

## Solution Assessment

No, not as currently ordered. Using MiniMax text models for extraction fallback is the right direction, but the implementation should preserve existing generic image-provider precedence unless MiniMax is the resolved default or the only usable candidate.
