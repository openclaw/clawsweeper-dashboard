## What This Changes

The PR adds a 30-second WhatsApp monitor interval that drains pending deliveries for the connected account and clears the interval on non-keepalive shutdown or connection close.

## Best Possible Solution

Land the WhatsApp-local periodic drain with existing drain/backoff guards intact after adding the required single-line Unreleased Fixes changelog entry, with a fake-timer monitor regression test if maintainers want stronger coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: enqueue a WhatsApp delivery while the account is already connected and no reconnect occurs, because current main only invokes the WhatsApp delivery-queue drain from the connect/reconnect path.

## Solution Assessment

Yes. The proposed WhatsApp-local periodic drain is the narrowest maintainable fix because it keeps connection-readiness policy inside the WhatsApp monitor and reuses the existing drainKey, claim, selectEntry, and backoff behavior instead of teaching enqueueDelivery about provider connection state.
