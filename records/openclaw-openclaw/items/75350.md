## Summary

Keep open; the revised patch is still not merge-ready. It changes a shipped DeepSeek V4 replay contract that current main documents and tests for plain assistant replay, while the PR's older-endpoint 400 claim is not backed by linked live evidence.

## What This Changes

The PR changes the DeepSeek V4 OpenAI-compatible thinking wrapper so enabled-thinking requests strip `reasoning_content` from assistant messages without `tool_calls` while preserving or backfilling it for assistant messages with tool calls.

## Best Possible Solution

Keep the current DeepSeek V4 enabled-thinking replay behavior unless maintainers obtain live evidence for an endpoint-specific split. If that split is real, implement a turn-aware rule with targeted unit/plugin/live coverage, update provider docs, and add a changelog entry while preserving the #71372 and #73417 replay protections.

## Reproduction Assessment

Yes. Current unit, plugin, and live-test fixtures construct DeepSeek V4 enabled-thinking replay payloads with prior plain assistant messages and assert or demonstrate that `reasoning_content` is present; this diff would remove it from those payloads.

## Solution Assessment

No. Preserving tool-call messages is an improvement over the first blanket strip, but using `tool_calls` on the individual assistant message is not enough to model DeepSeek's turn-level replay contract, and it conflicts with current shipped tests and docs.
