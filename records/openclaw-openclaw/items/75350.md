## Summary

Keep open, but the patch should not merge as-is: current main intentionally backfills `reasoning_content` for plain assistant replay, current tests and docs lock that contract, and the PR still has no real DeepSeek V4 proof for the claimed older-endpoint failure.

## What This Changes

The PR changes the DeepSeek V4 enabled-thinking wrapper to remove `reasoning_content` from assistant replay messages unless they contain tool calls.

## Best Possible Solution

Keep the shipped DeepSeek V4 replay backfill unless maintainers receive live provider proof for a narrowly gated endpoint exception, with matching tests and docs updated together.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the regression introduced by this patch: source, unit tests, plugin tests, docs, and a live fixture all show current main preserving blank `reasoning_content` on plain assistant replay. No high-confidence reproduction exists for the PR's claimed older-endpoint 400.

## Solution Assessment

Is this the best way to solve the issue? No. Stripping plain assistant replay is broader than the evidence supports and conflicts with the shipped DeepSeek V4 replay contract; the safer path is provider proof first, then a narrow gated rule if needed.
