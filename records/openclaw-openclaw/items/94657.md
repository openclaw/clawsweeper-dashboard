Keep open: current main and the latest release still expose and report workspace bootstrap budgets in characters, while the requested token-based limit/reporting path remains missing and needs a maintainer product/config decision before implementation.

## What This Changes

The issue requests token-based bootstrap budget config and doctor output for workspace bootstrap files, replacing or superseding the current character-based caps.

## Best Possible Solution

Design a token-aware bootstrap budget model that keeps config/docs/doctor/runtime reports aligned, preserves or migrates the shipped char keys through doctor, and uses one maintainer-approved estimator contract.

## Reproduction Assessment

## Solution Assessment

Unclear. Token-based budgeting is a plausible direction, but replacing char keys outright is not proven best until maintainers choose the estimator/tokenizer contract and migration strategy.
