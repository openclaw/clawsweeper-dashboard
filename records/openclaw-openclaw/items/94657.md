## Summary

Keep open: current main and v2026.6.10 still expose, enforce, document, and report workspace bootstrap budgets in characters. Current main has a shared token-estimate helper and recent CJK/fullwidth estimator work, but there is still no bootstrap token-limit config key or doctor token output for this issue's requested contract.

## What This Changes

The issue requests token-based workspace bootstrap limits and doctor reporting instead of, or ahead of, the existing character-based bootstrap budget controls.

## Best Possible Solution

A maintainer should choose the canonical bootstrap budget unit and migration/deprecation path, then implement shared runtime, doctor, docs, and test coverage that keeps existing char-based users safe or migrates them through doctor.
