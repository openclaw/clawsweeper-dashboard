## Summary

Keep open: current main and the latest release still use character-based bootstrap limits in config, runtime budget analysis, doctor output, and docs. The request is useful, but it adds or replaces public config semantics, so it needs maintainer product direction before an implementation PR.

## What This Changes

The issue requests token-based workspace bootstrap limits and token estimates in doctor output instead of relying only on character-based `bootstrapMaxChars` limits.

## Best Possible Solution

Define a maintainer-approved token-budget design that either supplements or deliberately migrates the shipped char keys, reuses the existing estimate utility where appropriate, and updates runtime analysis, doctor output, `/context`, schema/help, tests, and docs together.
