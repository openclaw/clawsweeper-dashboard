## Summary

Keep open: current main and the latest release still use character-based bootstrap config, runtime budget analysis, doctor output, and docs; the requested token-based bootstrap limit/reporting is a valid core context-budget feature but needs maintainer product direction before implementation.

## What This Changes

The issue requests replacing or supplementing `bootstrapMaxChars` bootstrap budgeting with token-based limits and doctor reporting.

## Best Possible Solution

Define one canonical token-budget contract for bootstrap context, then implement shared runtime/doctor/docs/test coverage that either migrates from char limits or clearly keeps char limits as the compatibility fallback.
