## Summary

Keep open: current main has isolated agents, per-agent workspaces, sandbox/tool policy, routing bindings, tasks, commitments, and heartbeat-related surfaces, but it does not provide the requested supervisor-declared family-agent mode, structured operating queue, escalation states, morning-sync digest delivery, or `openclaw agents queue` tooling.

## What This Changes

Request a first-class family-support agent mode with supervisor routing, structured queue capture, escalation levels, daily digest delivery, and CLI inspection/resolution.

## Best Possible Solution

Keep this as a maintainer product/API design item; if accepted, define a generic supervised agent-state/digest contract that composes with existing tasks, commitments, SQLite state, and plugin boundaries instead of hard-coding a family-specific mode.

## Reproduction Assessment

Not applicable: this is a feature request, not a current-main bug report. Source and docs inspection confirm the requested supervisor queue and digest contract is not implemented today.

## Solution Assessment

Unclear: the requested behavior is coherent, but a literal core family-assistant mode is not proven to be the best OpenClaw design. A generic supervised agent-state/digest API or plugin-boundary decision should come first.
