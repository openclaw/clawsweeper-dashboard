## Summary

## What This Changes

The issue asks for a first-class supervised family-agent mode with a structured queue, supervisor configuration, daily digest, escalation states, family-safe response contract, and CLI inspection/resolution commands.

## Best Possible Solution

Decide the product boundary first, then design a single SQLite-backed supervision queue/digest contract that composes with heartbeat, commitments/tasks, and the open A2A/session-state tracks instead of adding a parallel family-only orchestration layer.
