## Summary

The PR should stay open for maintainer handling: it adds a meaningful Control UI feature, but the proposed implementation is not merge-ready because it lacks real behavior proof and misses current main's task-ledger contract.

## What This Changes

This PR adds a read-only Control UI `/tasks` tab with task-board state, session and cron adapters, Lit components, navigation wiring, and focused UI tests.

## Best Possible Solution

Refresh the Control UI Task Board around `tasks.list`/`TaskSummary`, keep cron/session details as secondary facets where useful, route visible copy through the Control UI i18n catalog, and require real `/tasks` browser proof before merge.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the branch omits `tasks.list`, derives waiting durations for error sessions, and can mark never-run cron jobs stale from `updatedAtMs`. The actual browser route still lacks after-fix real behavior proof.

## Solution Assessment

No. A Control UI Task Board is a plausible feature, but this branch should be updated to current main's task-ledger and i18n contracts rather than shipping a sessions/cron-only reconstruction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the task ledger for the task board:** `ui/src/ui/task-board/service.ts:23-50`
  - body: Current main exposes `tasks.list`/`TaskSummary` as the durable background-task contract, but this service only reads sessions and cron APIs. The new general Task Board would omit non-cron task-ledger work, including ACP, subagent, CLI, and media tasks.
  - confidence: 0.9
- **[P2] Only show waiting time for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-141`
  - body: Failed, killed, and timed-out rows map to `status === "error"`, but this condition sets `waitingForSec` for every non-running status. Those cards would render an error blocker plus a misleading waiting duration; restrict the duration to actual waiting rows.
  - confidence: 0.94
- **[P2] Base never-run cron health on the schedule:** `ui/src/ui/task-board/adapters/cron-adapter.ts:91-95`
  - body: For an enabled job with no recorded run, this falls back to `job.updatedAtMs`, so a daily or weekly job created more than an hour ago can show warning or stale even if its first `nextRunAtMs` is still in the future.
  - confidence: 0.89
- **[P2] Move Task Board copy into the i18n catalog:** `ui/src/ui/components/task-board/page.ts:46-47`
  - body: The new page hard-codes Chinese strings directly in the component, so English and other locale users can see mixed-language UI. Add English source strings through the Control UI i18n catalog instead.
  - confidence: 0.9
- **[P2] Exclude session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:129`
  - body: This filter only removes cron-namespace session keys. Cron jobs can target `current` or `session:<key>` sessions that reuse ordinary session keys, so the same scheduled work can appear in both lanes and be mislabeled as an independent active session.
  - confidence: 0.82
