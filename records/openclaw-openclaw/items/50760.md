## Summary

Keep open. Current main still has no Control UI Task Board route, so this PR is not obsolete, but the proposed patch has user-visible adapter and localization bugs and still needs a maintainer decision on whether this surface should aggregate Sessions/Cron data or consume the canonical task ledger.

## What This Changes

This PR adds a read-only Control UI `/tasks` Task Board tab backed by `sessions.list` and `cron.*` adapters, Lit components, app/navigation state, and targeted tests.

## Best Possible Solution

Keep the PR open as an implementation candidate, but require a rebase, i18n integration, adapter fixes, and an explicit maintainer decision on task-ledger versus Sessions/Cron aggregation before merge.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting navigation, render branches, refresh logic, and docs; the PR-specific bugs are reproducible from the diff with failed session rows, never-run future cron jobs, and cron jobs targeting `current` or `session:<id>`.

## Solution Assessment

No, not as currently proposed. The feature direction may be useful, but the patch should be rebased and hardened, and the source-of-truth question should be settled before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Only show waiting duration for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-140`
  - body: Failed, killed, or timed-out sessions become `status: "error"`, but this condition still populates `waitingForSec` for every non-running row. The card then renders an error badge plus "waiting for", misreporting terminal failures as pending work.
  - confidence: 0.93
- **[P2] Base never-run cron health on the next scheduled run:** `ui/src/ui/task-board/adapters/cron-adapter.ts:91`
  - body: When a job has no recorded run, health falls back to `job.updatedAtMs`. A daily or weekly job created more than an hour ago but scheduled for the future is marked warning/stale before it ever had a chance to run; use `nextRunAtMs` or another not-yet-due state for never-run enabled jobs.
  - confidence: 0.9
- **[P2] Exclude session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:129`
  - body: Filtering out only cron-prefixed session keys misses cron jobs that target `current` or `session:<key>`. Those runs can surface from `sessions.list` while the same job is also rendered from `cron.list`/`cron.runs`, so scheduled automation is double-counted and mislabeled as independent active work.
  - confidence: 0.86
- **[P2] Route Task Board copy through i18n:** `ui/src/ui/components/task-board/page.ts:46-47`
  - body: The new page hardcodes Chinese text while navigation adds separate hardcoded English labels instead of using the Control UI translation system. In the default English UI this renders mixed-language copy, and non-English locales cannot translate the new surface.
  - confidence: 0.88
