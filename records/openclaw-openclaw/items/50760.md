## Summary

Keep this PR open. Current main still does not provide the requested browser Control UI Task Board tab/page, so the branch is not obsolete, but the proposed patch has concrete data-display and localization defects that should block merge until maintainers decide the right task data contract and the branch is rebased.

## What This Changes

This PR adds a read-only Control UI Task Board tab and `/tasks` route with session/cron adapters, a fan-out service, Lit components, and targeted page/adapter tests.

## Best Possible Solution

Keep the PR open as a potential implementation candidate, but rework it around the maintained Control UI patterns, route all user-facing copy through the existing i18n system, fix the adapter correctness issues, and either consume the first-class task ledger through an explicit Gateway/UI contract or document why the Phase 1 board intentionally aggregates sessions and cron.

## Reproduction Assessment

Yes. The missing-main behavior is reproducible by inspecting current navigation, app state, render branches, and UI files: `/tasks` is not a Control UI route on main. The PR-specific adapter bugs are also reproducible from the diff with failed session rows and never-run cron rows.

## Solution Assessment

No, not as currently proposed. The direction may be useful, but the patch should be rebased and should either use the shipped task-ledger contract or explicitly harden the sessions/cron aggregation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Only show waiting duration for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-141`
  - body: Failed, killed, or timed-out sessions become `status: "error"`, but this code still populates `waitingForSec` for every non-running row. The card then renders an error badge plus "waiting for", which misreports terminal failures as pending work.
  - confidence: 0.93
- **[P2] Base never-run cron health on the next scheduled run:** `ui/src/ui/task-board/adapters/cron-adapter.ts:83`
  - body: For jobs with no recorded run, health falls back to `job.updatedAtMs`. A daily or weekly job created more than an hour ago but scheduled for the future is marked warning/stale before it ever had a chance to run; use `nextRunAtMs` for never-run enabled jobs instead of aging from creation/update time.
  - confidence: 0.9
- **[P2] Avoid double-counting session-targeted cron jobs:** `ui/src/ui/task-board/adapters/session-adapter.ts:125`
  - body: Filtering out only `cron:*` session keys misses cron jobs that target `current` or `session:<key>`. Those runs can appear in `sessions.list` as active cards while the same cron job is also rendered from `cron.list`/`cron.runs`, so scheduled automation is shown twice and mislabeled as independent active work.
  - confidence: 0.86
- **[P2] Route Task Board copy through i18n:** `ui/src/ui/components/task-board/page.ts:43-44`
  - body: The new page hardcodes Chinese page text and separate English tab labels instead of using the Control UI translation system. In the default English UI, the page would render mixed-language copy and non-English locales cannot translate the new surface.
  - confidence: 0.88
