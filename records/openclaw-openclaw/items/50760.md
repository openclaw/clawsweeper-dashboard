## Summary

Keep open: current main does not implement the Control UI `/tasks` route, so the PR still contains useful feature work, but it should not merge as-is because it misses the current task-ledger source of truth, has user-visible adapter and i18n defects, and lacks real behavior proof.

## What This Changes

The PR adds a read-only Control UI Task Board tab with navigation, app state, rendering, sessions/cron adapters, service loading, and focused UI tests.

## Best Possible Solution

Refresh the Task Board around `tasks.list`/`TaskSummary` as the primary source, keep sessions and cron as secondary facets where useful, move visible copy through the i18n catalog, and add redacted browser proof before merge.

## Reproduction Assessment

Yes for the review findings by source inspection: the proposed loader omits `tasks.list`, error sessions get `waitingForSec`, never-run cron health falls back to update time, and visible copy bypasses i18n. No after-fix browser proof currently reproduces the full `/tasks` route in a real setup.

## Solution Assessment

No. A Control UI Task Board is a plausible feature, but this branch should be refreshed around current main's task-ledger and i18n contracts instead of shipping a sessions/cron-only reconstruction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the task ledger as the board source:** `ui/src/ui/task-board/service.ts:23-45`
  - body: Current main exposes `tasks.list`/`TaskSummary` as the durable background-task contract, but this service only reads sessions and cron APIs. The new general Task Board would omit non-cron task-ledger work such as ACP, subagent, CLI, and media tasks.
  - confidence: 0.9
- **[P2] Restrict waiting duration to waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-141`
  - body: Failed, killed, and timed-out rows map to `status === "error"`, but this condition sets `waitingForSec` for every non-running status. Those cards would render an error blocker plus a misleading waiting duration; compute it only for actual waiting rows.
  - confidence: 0.94
- **[P2] Use next run time for never-run cron health:** `ui/src/ui/task-board/adapters/cron-adapter.ts:91-98`
  - body: For an enabled job with no recorded run, this falls back to `job.updatedAtMs`, so a daily or weekly job created more than an hour ago can show warning or stale even if its first `nextRunAtMs` is still in the future.
  - confidence: 0.88
- **[P2] Move Task Board strings into i18n:** `ui/src/ui/components/task-board/page.ts:43-47`
  - body: The new page and toolbar hard-code user-visible Chinese and English strings directly in components, bypassing the Control UI locale source of truth. Add English source strings through the i18n catalog so other locales do not see mixed-language UI.
  - confidence: 0.9
- **[P2] Exclude session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:129`
  - body: This filter only removes cron-namespace session keys. Cron jobs can target `current` or `session:<id>` sessions that reuse ordinary session keys, so the same scheduled work can appear in both lanes and be mislabeled as an independent active session.
  - confidence: 0.8
