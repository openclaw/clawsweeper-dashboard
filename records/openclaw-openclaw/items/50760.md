## Summary

Keep open. Current main still lacks the Control UI Task Board surface, so this PR is not obsolete, but the branch is not merge-ready because the proposed adapter behavior misreports several cron/session states and the new visible UI copy bypasses the Control UI i18n contract.

## What This Changes

This PR adds a read-only `/tasks` Control UI tab, Task Board app state/service/adapters over `sessions.list`, `cron.list`, and `cron.runs`, Lit components, and targeted page/adapter tests.

## Best Possible Solution

Define the Task Board around the authoritative task/session/cron model, then land a rebased, i18n-ready read-only slice with regression coverage for the adapter edge cases.

## Reproduction Assessment

Yes. Current-main absence is reproducible from the navigation, render, and refresh paths; the PR-specific blockers are source-reproducible from the diff against current session and cron contracts.

## Solution Assessment

No. The feature direction may be useful, but this patch needs adapter fixes, Control UI i18n integration, and a maintainer decision on the Task Board source of truth before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Only show waiting duration for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:130-133`
  - body: Failed, killed, or timed-out rows map to `status: "error"`, but this condition still assigns `waitingForSec` for every updated non-running row. The card then renders a terminal error together with a waiting duration, misreporting failed work as pending.
  - confidence: 0.93
- **[P2] Use nextRunAtMs for never-run cron health:** `ui/src/ui/task-board/adapters/cron-adapter.ts:85-91`
  - body: When a job has no recorded run, health falls back to `job.updatedAtMs`. A daily or weekly job created more than an hour ago but scheduled for the future is marked warning/stale before it has had a chance to run; base the never-run case on `state.nextRunAtMs` or an explicit not-yet-due state.
  - confidence: 0.9
- **[P2] Exclude custom session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:121`
  - body: Filtering only cron-prefixed session keys misses cron jobs bound to `current` or `session:<id>`, which run on ordinary session keys. Those rows can appear from `sessions.list` while the same job is rendered from `cron.list`/`cron.runs`, double-counting scheduled automation as active work.
  - confidence: 0.86
- **[P2] Route Task Board copy through i18n:** `ui/src/ui/components/task-board/page.ts:41-42`
  - body: The new page hardcodes visible Chinese strings instead of adding English source keys and generated locale output. In the default English UI this produces mixed-language copy and leaves the new surface outside the translation pipeline.
  - confidence: 0.88
