## Summary

Keep this PR open: current main still does not have a Control UI Task Board route, but the proposed implementation is not merge-ready because it bypasses the current task-ledger contract, has user-visible mapping bugs, and lacks real browser/runtime proof.

## What This Changes

The PR adds a read-only Control UI `/tasks` tab with task-board state, sessions/cron adapters, Lit components, navigation wiring, and focused tests.

## Best Possible Solution

Rework the Control UI task board around `tasks.list`/`TaskSummary`, route copy through the Control UI i18n catalog, add focused tests, and provide real browser proof for `/tasks`.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the service omits `tasks.list`, failed sessions get `waitingForSec`, never-run cron jobs can degrade from `updatedAtMs`, and new copy bypasses i18n. The actual `/tasks` browser route still lacks real after-fix proof.

## Solution Assessment

No. A Control UI task board is a plausible feature, but this branch should be updated to current main's task-ledger and i18n contracts rather than shipping a sessions/cron-only reconstruction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the task ledger for the task board:** `ui/src/ui/task-board/service.ts:23-50`
  - body: Current main exposes `tasks.list` as the durable background-task contract for ACP, subagent, CLI, media, and cron work. This service only queries `sessions.list`, `cron.list`, and `cron.runs`, so the new general Task Board would omit most non-cron background tasks.
  - confidence: 0.9
- **[P2] Only show waiting time for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-141`
  - body: Failed, killed, and timed-out rows are mapped to `status === "error"`, but this condition sets `waitingForSec` for every non-running status. Those cards render an error blocker plus a misleading waiting duration; restrict the duration to actual waiting rows.
  - confidence: 0.94
- **[P2] Base never-run cron health on the schedule:** `ui/src/ui/task-board/adapters/cron-adapter.ts:91-95`
  - body: For an enabled job with no recorded run, this falls back to `job.updatedAtMs`, so a daily or weekly job created more than an hour ago can show warning or stale even if its first `nextRunAtMs` is still in the future.
  - confidence: 0.89
- **[P2] Move Task Board copy into the i18n catalog:** `ui/src/ui/components/task-board/page.ts:46-47`
  - body: The new page hard-codes Chinese strings directly in the component, so English and other locale users will see mixed-language UI. Add English source strings under the Control UI i18n catalog and regenerate locale outputs instead.
  - confidence: 0.9
- **[P2] Exclude session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:129`
  - body: This filter only removes cron-namespace session keys. Cron jobs can target `current` or `session:<key>` sessions that reuse ordinary session keys, so the same scheduled work can appear in both lanes and be mislabeled as an independent active session.
  - confidence: 0.82
