## Summary

Keep open, but the current patch should not merge as-is: the Control UI task-board feature is still absent on current main, while this branch is stale against the current task-ledger and i18n contracts, has user-visible mapping bugs, and lacks real behavior proof.

## What This Changes

This PR adds a read-only Control UI `/tasks` tab with task-board state, sessions/cron adapters, Lit task-board components, navigation wiring, and focused adapter/page tests.

## Best Possible Solution

Rebase and redesign the Control UI task board around current `tasks.list`/`TaskSummary` semantics, route user-visible copy through the Control UI i18n catalog, and prove the resulting `/tasks` page in a real browser session.

## Reproduction Assessment

Yes for the PR review findings by source inspection: failed session rows with `updatedAt` get `waitingForSec`, never-run cron jobs can degrade from `updatedAtMs`, and the service omits `tasks.list`. The new route itself still lacks after-fix browser proof.

## Solution Assessment

No. A Control UI task board is plausible, but this branch should be updated to current main's task-ledger and i18n contracts instead of shipping a partial sessions/cron reconstruction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the task ledger for the task board:** `ui/src/ui/task-board/service.ts:22-41`
  - body: Current main exposes `tasks.list` as the durable background-task contract for ACP, subagent, CLI, media, and cron work. This service only queries `sessions.list`, `cron.list`, and `cron.runs`, so the new `/tasks` page would omit most non-cron background tasks while presenting itself as a general Task Board.
  - confidence: 0.88
- **[P2] Only show waiting time for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-141`
  - body: Failed, killed, and timed-out rows are mapped to `status === "error"`, but this condition sets `waitingForSec` for every non-running status. Those cards render an error blocker plus a misleading waiting duration; restrict the duration to actual waiting rows and cover it in the failed-session test.
  - confidence: 0.94
- **[P2] Base never-run cron health on the schedule:** `ui/src/ui/task-board/adapters/cron-adapter.ts:95`
  - body: For an enabled job with no recorded run, this falls back to `job.updatedAtMs`, so a daily or weekly job created more than an hour ago can show warning/stale even when its first `nextRunAtMs` is still in the future. Use the next scheduled run when there is no last run before degrading health.
  - confidence: 0.89
- **[P2] Move Task Board copy into the i18n catalog:** `ui/src/ui/components/task-board/page.ts:44-46`
  - body: The new page hard-codes Chinese strings directly in the component, so English and other locale users will see mixed-language UI. Add English source strings under the Control UI i18n catalog and regenerate locale outputs instead of embedding user-visible copy here.
  - confidence: 0.9
- **[P2] Avoid counting session-targeted cron runs as active sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:129`
  - body: This filter only removes cron-namespace session keys. Cron jobs can target `current` or `session:<key>` sessions that reuse ordinary session keys, so the same scheduled work can appear in both lanes and be mislabeled as an independent active session.
  - confidence: 0.82
