## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Only show waiting duration for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-140`
  - body: Failed, killed, or timed-out sessions become `status: "error"`, but this condition still populates `waitingForSec` for every non-running row. The card can render an error badge plus a waiting duration, misreporting terminal failures as pending work.
  - confidence: 0.93
- **[P2] Base never-run cron health on the next scheduled run:** `ui/src/ui/task-board/adapters/cron-adapter.ts:91`
  - body: When a job has no recorded run, health falls back to `job.updatedAtMs`. A daily or weekly job created more than an hour ago but scheduled for the future is marked warning or stale before it ever had a chance to run.
  - confidence: 0.9
- **[P2] Exclude session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:129`
  - body: Filtering out only cron-prefixed session keys misses cron jobs that target `current` or `session:<key>`. Those runs can surface from `sessions.list` while the same job is also rendered from `cron.list`/`cron.runs`, double-counting scheduled automation.
  - confidence: 0.86
- **[P2] Route Task Board copy through i18n:** `ui/src/ui/components/task-board/page.ts:46-47`
  - body: The new page hardcodes Chinese copy while navigation adds separate hardcoded English labels instead of the Control UI translation system. That mixes languages in the default UI and leaves the new surface unavailable to locale bundles.
  - confidence: 0.88
