## Summary

Keep open. The Control UI Task Board feature is not present on current main, but this PR is not merge-ready because it lacks real behavior proof and the branch now needs a current-main design refresh around the task ledger plus fixes for user-visible logic and i18n issues.

## What This Changes

Adds a read-only Control UI `/tasks` tab with task-board state, a sessions/cron fan-out service, Lit task-board components, navigation wiring, and focused adapter/page tests.

## Best Possible Solution

Rebase and redesign the Control UI task board around current `tasks.list`/`TaskSummary` semantics, keep localized UI copy in the i18n catalog, and add focused adapter plus browser proof for the resulting `/tasks` route.

## Reproduction Assessment

Yes for the review findings by source inspection: failed session rows with `updatedAt` get `waitingForSec`, and never-run enabled cron jobs can be degraded from `updatedAt` even with a future `nextRunAtMs`. The PR still lacks after-fix browser proof for the new route.

## Solution Assessment

No. A Control UI task board is plausible, but this branch should be updated to current main’s task-ledger contract and i18n rules instead of shipping a partial sessions/cron reconstruction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the task ledger for the task board:** `ui/src/ui/task-board/service.ts:22-41`
  - body: Current main exposes `tasks.list` as the durable background-task contract for ACP, subagent, CLI, media, and cron work. This service only queries `sessions.list`, `cron.list`, and `cron.runs`, so the new `/tasks` page would omit most non-cron background tasks while presenting itself as a general Task Board.
  - confidence: 0.86
- **[P2] Only show waiting time for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-141`
  - body: Failed, killed, and timed-out rows are mapped to `status === "error"`, but this condition sets `waitingForSec` for every non-running status. Those error cards will render a danger blocker and a misleading “waiting for” duration; restrict the duration to actual waiting rows and cover it in the failed-session test.
  - confidence: 0.94
- **[P2] Base never-run cron health on its schedule:** `ui/src/ui/task-board/adapters/cron-adapter.ts:82`
  - body: For an enabled job with no recorded run, this falls back to `job.updatedAtMs`, so a daily or weekly job created more than an hour ago can show warning/stale even when its first `nextRunAtMs` is still in the future. Use the next scheduled run when there is no last run before degrading health.
  - confidence: 0.89
- **[P2] Move Task Board copy into the i18n catalog:** `ui/src/ui/components/task-board/page.ts:44-46`
  - body: The new page hard-codes Chinese strings directly in the component, so English and other locale users will see mixed-language UI. Add English source strings under the Control UI i18n catalog and regenerate locale outputs instead of embedding user-visible copy here.
  - confidence: 0.9
