## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Only show waiting duration for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:130-133`
  - body: Failed, killed, or timed-out sessions map to `status: "error"`, but this condition still sets `waitingForSec` for every updated non-running row. The card can then render a terminal error together with a waiting duration, misreporting failed work as pending.
  - confidence: 0.93
- **[P2] Base never-run cron health on the next scheduled run:** `ui/src/ui/task-board/adapters/cron-adapter.ts:85-91`
  - body: When a job has no recorded run, health falls back to `job.updatedAtMs`. A daily or weekly enabled job created more than an hour ago but scheduled for the future will show warning/stale before it ever had a chance to run; use `nextRunAtMs` or an explicit not-yet-due state for never-run jobs.
  - confidence: 0.9
- **[P2] Exclude session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:121`
  - body: Filtering only cron-prefixed session keys misses cron jobs targeting `current` or `session:<id>`, which run on ordinary agent session keys. Those rows can appear from `sessions.list` while the same job is also rendered from `cron.list`/`cron.runs`, double-counting scheduled automation as active work.
  - confidence: 0.86
- **[P2] Route Task Board copy through Control UI i18n:** `ui/src/ui/components/task-board/page.ts:41-42`
  - body: The new page hardcodes visible Chinese strings instead of adding English source keys and generated locale output. In the default English UI this produces mixed-language copy and leaves the new surface outside the translation pipeline.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `ui/src/ui/navigation.ts:9`
  - body: This adds a user-facing Control UI tab, but the PR does not update `CHANGELOG.md`. Repo policy requires user-facing feature changes to have a changelog entry so release notes do not omit the new surface.
  - confidence: 0.82
- **[P3] Document the new Control UI surface:** `ui/src/ui/navigation.ts:9`
  - body: The PR adds a browser `/tasks` page, but the Control UI docs still describe Sessions and Cron as separate surfaces only. Update the Control UI docs so the shipped browser feature is discoverable and matches the new navigation.
  - confidence: 0.8
