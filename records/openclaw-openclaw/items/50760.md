## Summary

Keep open: current main still has no Control UI `/tasks` tab, so the PR is not obsolete, but the branch is not merge-ready because it misses the current task-ledger contract, has source-visible UI mapping bugs, and lacks contributor real behavior proof.

## What This Changes

The PR adds a read-only Control UI Task Board tab with navigation, state, rendering, sessions/cron adapters, service loading, and focused UI tests.

## Best Possible Solution

Refresh the Task Board around `tasks.list`/`TaskSummary` as the primary source, keep sessions and cron as secondary facets where useful, move visible copy through i18n, and add redacted browser proof before merge.

## Reproduction Assessment

Yes for the review findings by source inspection of the PR diff against current-main contracts: the service omits `tasks.list`, error sessions get `waitingForSec`, never-run cron health falls back to update time, and new UI copy bypasses i18n. The actual browser route still lacks after-fix real behavior proof.

## Solution Assessment

No. A Control UI Task Board is plausible, but this branch should be refreshed around current main's task-ledger and i18n contracts rather than shipping a sessions/cron-only reconstruction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the task ledger for the task board:** `ui/src/ui/task-board/service.ts:24-45`
  - body: Current main exposes `tasks.list`/`TaskSummary` as the durable background-task contract, but this service only reads sessions and cron APIs. The new general Task Board would omit non-cron task-ledger work, including ACP, subagent, CLI, and media tasks.
  - confidence: 0.9
- **[P2] Restrict waiting duration to waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:138-141`
  - body: Failed, killed, and timed-out rows map to `status === "error"`, but this condition sets `waitingForSec` for every non-running status. Those cards would render an error blocker plus a misleading waiting duration; restrict the duration to actual waiting rows.
  - confidence: 0.94
- **[P2] Base never-run cron health on its schedule:** `ui/src/ui/task-board/adapters/cron-adapter.ts:87-98`
  - body: For an enabled job with no recorded run, this falls back to `job.updatedAtMs`, so a daily or weekly job created more than an hour ago can show warning or stale even if its first `nextRunAtMs` is still in the future.
  - confidence: 0.88
- **[P2] Move Task Board copy into the i18n catalog:** `ui/src/ui/components/task-board/page.ts:43-47`
  - body: The new page and toolbar hard-code user-visible Chinese and English strings directly in components, bypassing the Control UI locale source of truth. Add English source strings through the i18n catalog so other locales do not see mixed-language UI.
  - confidence: 0.9
- **[P2] Filter session-targeted cron runs from active cards:** `ui/src/ui/task-board/adapters/session-adapter.ts:129`
  - body: This filter only removes cron-namespace session keys. Cron jobs can target `current` or `session:<id>` sessions that reuse ordinary session keys, so the same scheduled work can appear in both lanes and be mislabeled as an independent active session.
  - confidence: 0.8
