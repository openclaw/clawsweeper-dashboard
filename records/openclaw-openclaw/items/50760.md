## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Build the board from task ledger records:** `ui/src/ui/task-board/service.ts:24-42`
  - body: Current main defines background task inspection around durable task records exposed by `tasks.list`, covering ACP, subagent, cron, and CLI work. This service only combines `sessions.list`, `cron.list`, and `cron.runs`, so the new board would miss non-session tasks and can double-count cron-backed sessions; use `tasks.list` as the primary source and add supplemental calls only when needed.
  - confidence: 0.86
- **[P2] Show waiting duration only for waiting sessions:** `ui/src/ui/task-board/adapters/session-adapter.ts:145-148`
  - body: `deriveActiveStatus` maps failed, killed, timeout, and aborted rows to `error`, but this condition still populates `waitingForSec` for every non-running session. Error cards then render a misleading "waiting for" duration next to the failure blocker; restrict this to `status === "waiting"`.
  - confidence: 0.91
- **[P2] Treat never-run jobs as healthy until their first due time:** `ui/src/ui/task-board/adapters/cron-adapter.ts:91`
  - body: When an enabled cron job has never run, `latestRun` and `lastRunAtMs` are absent, so health is aged from `job.updatedAtMs`. A daily or weekly job created hours ago with a future `nextRunAtMs` will be marked warning or stale even though no run is overdue; consider `state.nextRunAtMs` before aging it.
  - confidence: 0.83
