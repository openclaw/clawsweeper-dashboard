## Summary

Keep open: the reported bug is plausible and still source-reproducible, but this PR does not fix the zero-usage path end to end and lacks real after-fix behavior proof.

## What This Changes

The PR removes the `hasNonzeroUsage` import and relaxes `recordAssistantUsage` in `src/agents/pi-embedded-subscribe.ts` to accept any normalized usage object.

## Best Possible Solution

Track observed provider usage separately from nonzero counters through embedded aggregation, session persistence, and display handoff, then add focused regression coverage and redacted real output.

## Reproduction Assessment

Yes, by source inspection. Current main rejects all-zero normalized usage in the embedded resolver, collapses all-zero totals to `undefined`, and gates persistence on `hasNonzeroUsage`.

## Solution Assessment

No. Changing only the first guard is incomplete; the narrow maintainable fix needs explicit observed-usage state carried through totals, persistence, and status/UI handoff.
