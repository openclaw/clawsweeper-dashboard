## Summary

Keep open: the reported zero-usage token display problem is source-reproducible, but this branch does not fix it end-to-end and lacks after-fix real behavior proof.

## What This Changes

The PR relaxes a usage guard in `src/agents/pi-embedded-subscribe.ts` and removes the now-unused `hasNonzeroUsage` import so zero-valued provider usage can enter the subscription path.

## Best Possible Solution

Carry an explicit observed-usage state through subscription aggregation, runner metadata, session persistence, and status formatting so valid reported zeros render as fresh `0/context` while absent usage remains unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass normalized `{ input: 0, output: 0, total: 0 }` through the assistant usage path; current nonzero-only guards and zero-collapsing totals drop it before status can render a fresh zero.

## Solution Assessment

No. Changing one subscription guard is not the best fix because the downstream accumulator, totals, session persistence, and status freshness logic still need an explicit observed-usage signal.
