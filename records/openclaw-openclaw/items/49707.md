Keep open: the bug report is valid, but this PR is incomplete and stale against current `main`; it removes one nonzero guard while other usage aggregation, persistence, and status paths still collapse observed all-zero usage into unknown usage.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `recordAssistantUsage()` records normalized zero-valued usage instead of requiring a nonzero usage field.

## Best Possible Solution

Carry an explicit observed-usage state through subscription aggregation, runner metadata, session persistence, and status formatting so valid reported zeros render as fresh `0/context` while absent usage remains unknown.

No. This PR changes only one guard, but the maintainable fix needs to track whether usage was observed independently from whether any counter is greater than zero.
