## Summary

Keep open. The PR targets a real zero-valued usage bug, but the patch is incomplete against current main: zero usage is still filtered or collapsed in embedded aggregation, session persistence, and status fallback, and the PR lacks after-fix real behavior proof.

## What This Changes

The branch removes the `hasNonzeroUsage` import and changes `recordAssistantUsage` in `src/agents/pi-embedded-subscribe.ts` to accept any normalized usage object.

## Best Possible Solution

Track provider usage presence separately from nonzero token totals and carry that observed-usage state through embedded aggregation, session persistence, and status display with regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main rejects all-zero normalized usage in the embedded resolver, collapses all-zero totals to `undefined`, and gates persistence on `hasNonzeroUsage`.

## Solution Assessment

No. Changing only the first guard is incomplete; the maintainable fix needs an explicit observed-usage state carried through totals, persistence, and status/UI handoff.

## Review Findings

- **[P2] Carry zero-valued usage past the first guard:** `src/agents/pi-embedded-subscribe.ts:274-276`
