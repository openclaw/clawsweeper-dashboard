## Summary

Keep open for rework rather than cleanup-close: the branch targets a real session-usage problem, but the patch only relaxes the first guard while current main still drops all-zero usage in the resolver, totals, persistence, and status paths, and the PR still lacks real after-fix proof.

## What This Changes

The PR removes the `hasNonzeroUsage` import and changes embedded assistant usage recording to accept any normalized usage object in `src/agents/pi-embedded-subscribe.ts`.

## Best Possible Solution

Track usage presence separately from nonzero token totals through embedded resolution, accumulation, session persistence, and status display, while leaving the broader unavailable-usage estimate work to https://github.com/openclaw/openclaw/issues/54996.

## Reproduction Assessment

Yes, at source level. A normalized all-zero usage object is still rejected or converted back to `undefined` by current-main resolver, totals, persistence, and status code; I did not run a live Qwen session.

## Solution Assessment

No. Relaxing only the first guard is not the best fix because downstream code still uses nonzero totals as the signal that usage exists, so the user-visible `/status` behavior remains unchanged for the zero-usage case.
