## Summary

Keep open. The bug is valid, but this PR is still incomplete: it relaxes one guard while current token aggregation, session persistence, and status paths still treat all-zero usage as absent, so the stated `/status` behavior remains unresolved.

## What This Changes

The PR removes the `hasNonzeroUsage` import and changes `recordAssistantUsage()` to skip only absent normalized usage, intending to preserve all-zero provider usage.

## Best Possible Solution

Carry an explicit observed-usage state through subscription aggregation, runner metadata, session persistence, and status display so reported zeros render as fresh `0/context` while absent usage remains unknown.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to feed `{ input: 0, output: 0, total: 0 }` through the current subscription or runner path; the nonzero guards drop it or serialize it to `undefined`, and status then treats usage as unknown.

## Solution Assessment

No. The PR removes only one guard on an older path; the maintainable fix should track whether usage was observed independently from whether any counter is greater than zero.
