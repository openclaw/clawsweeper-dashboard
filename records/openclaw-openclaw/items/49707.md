## Summary

Keep open: the branch is still an incomplete fix for the reported zero-valued usage path, and the PR also lacks after-fix real behavior proof from a real run.

## What This Changes

The PR removes `hasNonzeroUsage` from the import and changes `recordAssistantUsage` in `src/agents/pi-embedded-subscribe.ts` to accept any normalized usage object.

## Best Possible Solution

Track observed provider usage separately from nonzero token totals, then carry that state through embedded aggregation, session persistence, and status display with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still rejects or collapses all-zero normalized usage in the embedded resolver, totals aggregation, session persistence, and status fallback; I did not run a live Qwen/free-tier reproduction in this read-only review.

## Solution Assessment

No. Relaxing only the first guard is not the narrow complete fix because downstream code still uses nonzero-token semantics to mean usage presence.
