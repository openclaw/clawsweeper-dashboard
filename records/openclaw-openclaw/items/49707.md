## Summary

Keep open: the patch is not mergeable because it only relaxes the first usage guard, while the later totals and persistence paths still treat all-zero usage as missing, and the PR has no after-fix real behavior proof.

## What This Changes

The PR removes the nonzero-usage guard from embedded PI subscription token accounting so normalized zero-valued provider usage can enter aggregation.

## Best Possible Solution

Track whether provider usage was observed separately from whether any counter is nonzero, return and persist zero-valued totals when observed, and add focused regression coverage for embedded subscription plus session persistence.

## Reproduction Assessment

Yes, from source inspection. A normalized all-zero provider usage object is rejected by the current nonzero predicate, and the PR still leaves `getUsageTotals()` and session persistence unable to distinguish observed zero usage from missing usage.

## Solution Assessment

No. The PR changes only the first guard; the maintainable fix needs an explicit observed-usage state carried through totals and persistence so zero remains distinct from absent data.
