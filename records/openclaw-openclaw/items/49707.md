## Summary

Keep open. The PR targets a real token-accounting bug, but the patch still collapses all-zero usage to missing usage downstream and the external proof gate is still empty.

## What This Changes

The PR removes the nonzero-usage guard in `src/agents/pi-embedded-subscribe.ts` so normalized zero-valued provider usage can enter embedded subscription token accounting.

## Best Possible Solution

Track observed provider usage separately from nonzero counters through embedded aggregation and session persistence, return/persist zero-valued totals when observed, and add focused regression coverage plus redacted real output.

## Reproduction Assessment

Yes, by source inspection. A normalized all-zero provider usage object is rejected on current main, and the PR still leaves totals and persistence unable to distinguish observed zero usage from missing usage.

## Solution Assessment

No. Changing only the first guard is incomplete; the maintainable fix needs an explicit observed-usage state carried through totals, persistence, and display handoff.
