The PR branch removes the `hasNonzeroUsage` guard from `recordAssistantUsage()` in `src/agents/pi-embedded-subscribe.ts` so normalized zero-valued provider usage is accepted instead of skipped.

## Best Possible Solution

Land a complete current-main fix that carries an explicit observed-usage state through subscription aggregation, runner metadata, session persistence, and status display, while preserving the existing stale fallback for absent usage and adding focused regression coverage for all-zero provider snapshots.
