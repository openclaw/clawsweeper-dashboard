## Summary

Keep open: current main still forwards the guard-created dispatcher through Slack media fetch, and https://github.com/openclaw/openclaw/pull/64022 is still open with closing syntax rather than merged.

## What This Changes

Slack inbound attachments in container sandboxes should be fetched and materialized as agent media instead of degrading to `[Slack file: ...]` placeholders with `invalid onRequestStart method`.

## Best Possible Solution

Review, rebuild, and land or replace https://github.com/openclaw/openclaw/pull/64022 with a Slack media fix that removes the Undici dispatcher conflict while preserving URL validation, Authorization headers, redirect behavior, and the intended SSRF contract.

## Solution Assessment

Unclear for the exact patch shape. Stripping the forwarded dispatcher matches the observed failure, but maintainers still need to approve the Slack SSRF transport contract or choose a replacement that preserves pinned-dispatcher enforcement.
