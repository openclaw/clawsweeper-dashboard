## Summary

Keep open: current main and the latest release still forward the SSRF guard dispatcher through the Slack media fetch wrapper, and the target fixing PR was closed unmerged after becoming stale, conflicting, and security-sensitive.

## What This Changes

The issue reports Slack inbound files in container sandboxes resolving only to placeholders because the automatic Slack media fetch path fails with an Undici dispatcher error.

## Best Possible Solution

Land a refreshed Slack media fix that removes the Undici dispatcher conflict while preserving Slack URL validation, auth headers, redirect behavior, and an explicitly approved SSRF enforcement contract.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The issue includes Slack container logs for `invalid onRequestStart method`, and current main still forwards the guard-created dispatcher through `createSlackMediaFetch`; I did not run a live Slack workspace.

## Solution Assessment

No, not yet for the previously proposed direct strip as-is. Stripping the dispatcher matches the reported failure, but maintainers need to approve a Slack preflight-only SSRF contract or request an enforcement-preserving replacement before implementation.
