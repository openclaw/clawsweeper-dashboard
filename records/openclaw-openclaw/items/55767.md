## Summary

Keep open. Current main still lacks a reserved internal session-namespace guard for external gateway hooks, and this PR appears to address a valid security-sensitive boundary without an obvious blocking defect. The remaining action is maintainer security review plus CI/Testbox validation on the final merge ref.

## What This Changes

The PR adds a gateway hook session-key guard and regressions that reject reserved `subagent:`, `acp:`, and `cron:` namespaces for direct hook requests, default hook session keys, hook mappings, and transform-provided session keys.

## Best Possible Solution

Land a maintainer-reviewed version of the reserved-namespace guard that keeps detection centralized at the hook session-key boundary, preserves wake mapping compatibility, and validates request, default, static mapping, templated mapping, and transform-provided hook session keys on the final merge ref.

## Solution Assessment

Yes. The proposed approach is the narrow maintainable fix because it normalizes hook-supplied keys, unwraps routed agent prefixes before reserved-prefix checks, and applies the guard at the config-load and dispatch boundaries that can supply hook session keys.
