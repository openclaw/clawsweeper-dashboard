## Summary

Keep open: current main still accepts hook session keys that can target internal session namespaces, so this PR is useful, but it still needs real behavior proof and a compatibility fix or explicit maintainer sign-off before merge.

## What This Changes

This PR adds reserved internal namespace validation for gateway hook session keys across config, request, mapping, and test paths.

## Best Possible Solution

Land the reserved namespace guard after moving static mapping fail-fast validation onto the effective-mapping policy path, preserving runtime rejection for any dispatched key and adding redacted real hook-request proof.

## Reproduction Assessment

Yes, source-reproducible: current main returns prefix-allowed hook session keys without checking reserved internal namespaces, and both direct and mapped hook paths use that resolver. I did not run a live gateway request in this read-only review.

## Solution Assessment

Mostly yes, but not as-is: a shared reserved-prefix guard at config and dispatch boundaries is the right shape, while static mapping fail-fast validation should preserve effective-mapping shadowing or receive explicit maintainer sign-off.
