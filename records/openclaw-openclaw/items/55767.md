## Best Possible Solution

Land the centralized reserved-namespace guard only after static mapping validation respects effective mapping shadowing, redacted real hook-runtime proof is posted, and maintainers accept the fail-closed policy for reserved namespaces.

Not yet: a shared reserved-prefix guard at config and dispatch boundaries is the right direction, but static mapping fail-fast validation needs to preserve the existing effective-mapping shadowing compatibility or get explicit maintainer sign-off.
