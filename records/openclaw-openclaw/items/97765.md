## Summary

Keep open: this PR targets the valid linked tool-call text-transform bug, but the patch only rewrites streamed tool-call events and still leaves final assistant-message `ToolCall.arguments` untransformed before tool execution; it also still needs real behavior proof from a plugin/tool-call run.

## What This Changes

Adds output replacement handling for `toolcall_delta` strings and `toolcall_end.toolCall.arguments` in `src/agents/plugin-text-transforms.ts`, with focused unit coverage in `src/agents/plugin-text-transforms.test.ts`.

## Best Possible Solution

Extend one shared message/tool-call transform so streamed events, partial/message/error snapshots, and `stream.result()` all rewrite string leaves inside `ToolCall.arguments` without touching ids, names, numbers, or metadata, then add real behavior proof.

## Reproduction Assessment

Yes, source-level: current main's final assistant message transform skips `ToolCall.arguments`, and the agent loop executes tool calls from `response.result()`. I did not run the reported live provider/chat-space scenario in this read-only review.

## Solution Assessment

No, this is only a partial fix because it rewrites the streamed `toolcall_end.toolCall` field but leaves final assistant message content unchanged. The narrower maintainable solution is to reuse the same tool-call argument transform wherever assistant messages are transformed.
