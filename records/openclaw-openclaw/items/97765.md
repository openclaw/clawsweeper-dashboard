## Summary

Keep open: the patch is in the right boundary but is incomplete because final assistant-message tool-call arguments still flow through `transformMessageText`, which does not rewrite `toolCall` blocks before `agent-loop` executes `response.result()`.

## What This Changes

The PR adds output text replacement for streamed `toolcall_delta` strings and `toolcall_end.toolCall.arguments`, with focused coverage in `plugin-text-transforms.test.ts`.

## Best Possible Solution

Extend the shared message-content transform so streamed events, partial snapshots, done/error messages, and `stream.result()` all rewrite string leaves inside `ToolCall.arguments` without touching names, ids, numeric values, or metadata.

## Reproduction Assessment

Yes, source-level: current main skips `ToolCall.arguments` in the message transform while `agent-loop` executes tool calls from `response.result()`. I did not run the reported live DeepSeek/chat-space scenario in this read-only review.

## Solution Assessment

No, this PR is only a partial fix because it rewrites the streamed `toolcall_end.toolCall` field but leaves the final assistant message path unchanged. The best fix is one shared transform for tool-call blocks wherever assistant messages are transformed.
