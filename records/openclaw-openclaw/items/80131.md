The issue asks OpenClaw to reduce embedded-run TTFT by avoiding repeated Vertex auth resolution and repeated plugin/tool bundling work on each request.

Add current-main substage timing for the reported setup, then land narrowly scoped caching or prewarming only for the measured hot paths while preserving auth refresh, provider hooks, and plugin/MCP invalidation semantics.
