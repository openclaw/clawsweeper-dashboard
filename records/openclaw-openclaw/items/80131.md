Keep open. The report is recent and credible, current main still has request-time work in both the embedded tool-prep path and the Google Vertex auth/header path, and the related trackers split the auth and tool-latency pieces rather than terminally superseding this combined Vertex TTFT report.

## What This Changes

The issue requests reducing gateway TTFT by avoiding repeated Vertex auth resolution and repeated plugin/bundled tool loading or serialization on every embedded run.

## Best Possible Solution

Reprofile this exact Vertex workload on current main, then land or supersede the Google Vertex ADC PR with credential-scoped token caching and make a separate profiling-driven tool-prep/bundle cache change that preserves plugin invalidation and workspace/session isolation.
