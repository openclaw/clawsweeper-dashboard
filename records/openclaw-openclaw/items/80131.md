## Summary

Keep open: current main has partial work around authorized-user ADC token caching and plugin tool descriptor caching, but it does not fully cover the reported service-account Vertex auth path or the remaining per-request tool preparation and provider schema normalization costs.

## What This Changes

Request to reduce gateway TTFT by caching Google Vertex auth resolution and per-request plugin/bundled tool construction and serialization.

## Best Possible Solution

Split this into measured perf work: credential-type-aware Google Vertex auth caching or warmup plus stable tool descriptor/provider payload caching with explicit invalidation and current-main TTFT benchmarks.

## Reproduction Assessment

No high-confidence live reproduction was run for the exact latency numbers. Source inspection does confirm the current request-time auth and tool-prep paths named by the report, but the Mac mini/Vertex/50-tool setup is needed for timing proof.

## Solution Assessment

Unclear. The proposed caching direction is plausible, but the maintainable fix needs design around service-account versus authorized-user ADC, credential rotation, plugin invalidation, and stable provider payloads.
