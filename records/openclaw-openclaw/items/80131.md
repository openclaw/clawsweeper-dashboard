Keep open: current main has partial caching in this area, but source inspection still shows the reported request-time auth and tool-prep surfaces are not fully covered, especially the Vertex service-account case and per-attempt tool normalization/provider payload work.

## What This Changes

The issue requests reducing gateway TTFT by caching repeated Vertex auth resolution and repeated plugin/tool schema bundling work across requests.

## Best Possible Solution

Split the work into measured, credential-type-aware Vertex auth caching/warmup and stable tool descriptor/provider payload caching with explicit invalidation and benchmark proof.

## Solution Assessment

Unclear. The proposed caching direction is plausible, but the safest implementation needs maintainer decisions around service-account versus authorized-user ADC, credential rotation, plugin hot reload, dynamic tool context, and provider payload semantics.
