## Summary

Keep open: current main has partial plugin descriptor caching and authorized-user ADC token caching, but it does not cover the reported Vertex service-account auth path or the remaining per-run tool construction, normalization, and bundling costs.

## What This Changes

The issue asks to reduce gateway TTFT by avoiding repeated Vertex service-account auth work and repeated plugin/tool descriptor bundling on each embedded run request.

## Best Possible Solution

Split this into measured perf work: service-account-aware Vertex auth caching and warmup, stable provider/tool payload caching with explicit invalidation, and focused benchmark/regression proof on current main.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms the relevant current-main paths still only cache authorized_user ADC and still do per-attempt tool construction/normalization, but the exact latency numbers require the reporter's Mac/Vertex service-account setup or a targeted benchmark.

## Solution Assessment

Unclear. The proposed caching direction is plausible, but the maintainable fix needs separate design for credential type support, cache lifetime/rotation, plugin hot reload, dynamic tool context, and provider payload stability.
