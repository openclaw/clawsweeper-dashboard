## Summary

Keep open. The report is fresh and credible on v2026.5.7: current main still has per-attempt embedded tool construction/normalization and per-request Google Vertex header resolution paths, while the service-account Vertex work is still tied to the open Google Vertex tracker/PR rather than shipped on main.

## What This Changes

The issue requests reducing Gateway TTFT by reusing Vertex auth and stable embedded-run tool construction/serialization across requests.

## Best Possible Solution

Split the remaining work into measured Google Vertex service-account token reuse and embedded-run tool-definition amortization, preserving plugin hot-reload and request-scoped tool context boundaries.

## Reproduction Assessment

No, not as a high-confidence live reproduction from this review. The issue includes detailed release timing data and source inspection confirms the per-attempt auth/tool paths still exist, but the exact current-main Mac/Vertex service-account workload was not replayed.

## Solution Assessment

Unclear. Singleton auth and tool memoization are directionally plausible, but the maintainable fix should first separate stable descriptors from request-scoped tool context and align the Vertex service-account path with the existing open Google Vertex work.
