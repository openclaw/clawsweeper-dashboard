## Summary

Keep open. The report is recent and detailed, and current main still has per-run auth and tool-prep work on the exact embedded-run stages being timed; related reports only cover pieces of the broader TTFT problem.

## What This Changes

Reduce gateway TTFT by avoiding repeated Vertex auth resolution and repeated embedded-run tool construction/bundling work.

## Best Possible Solution

Profile current main with the reported Vertex and heavy-plugin workload, then split the work into provider auth caching/prewarm and tool descriptor/bundle reuse with explicit invalidation semantics.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The reporter supplied repeated timings, and current source confirms auth and tool construction still sit on the per-run embedded path, so the issue is source-reproducible enough to keep open.

## Solution Assessment

Unclear. The proposed caching directions are plausible, but current main already has partial descriptor and authorized-user token caches, so a maintainer should profile the remaining service-account auth and tool-bundle costs before choosing cache keys or invalidation behavior.
