## Summary

Keep open. Current main has improved adjacent pieces, including session-scoped bundle MCP catalog reuse and authorized_user Vertex ADC token caching, but it still keeps embedded-run auth preparation and tool construction/materialization on the request path, and the related open items cover narrower slices rather than fully superseding this combined Vertex auth plus tool-bundling performance report.

## What This Changes

The issue requests reducing embedded-run TTFT by moving repeated Vertex auth and tool bundling work off the per-request hot path.

## Best Possible Solution

Capture redacted current-main auth/core-plugin-tools/bundle-tools substage timings, then land narrow caches or prewarming only for proven invariant work while preserving provider auth and plugin invalidation contracts.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. The reporter supplied repeated v2026.5.7 timings, and current source confirms the auth, core-plugin-tools, and bundle-tools buckets still sit on the embedded-run request path.

## Solution Assessment

Unclear. Singleton auth and broad tool memoization are plausible directions, but the maintainable fix should be driven by current-main spans and must preserve provider runtime auth, Vertex credential handling, tool availability, MCP catalog reuse, and explicit invalidation semantics.
