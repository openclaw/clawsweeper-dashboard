## Summary

Keep open. Current main has partial caching for plugin descriptors and Google Vertex authorized_user ADC, but the reported hot paths are still present in source and there is no proof that the Vertex service-account auth or per-run tool bundling costs from v2026.5.7 are fixed.

## What This Changes

The issue requests reducing gateway TTFT by avoiding repeated Vertex auth resolution and repeated plugin/tool bundle work on each embedded run.

## Best Possible Solution

Capture current-main substage timings for the reported Vertex and embedded-run setup, then land targeted caching or prewarming only for the proven hot paths while preserving provider auth refresh and plugin/tool invalidation semantics.

## Reproduction Assessment

No high-confidence live reproduction was run. The reporter supplied repeated v2026.5.7 timings, and current source confirms auth setup plus tool and bundle materialization remain on the embedded-run request path, so the issue is source-reproducible but needs fresh profiling for exact current-main timing.

## Solution Assessment

Unclear. The singleton auth and memoized tool-bundle direction is plausible, but the best maintainable fix should be driven by narrower profiling and must preserve credential rotation, provider auth contracts, plugin availability, and explicit invalidation.
