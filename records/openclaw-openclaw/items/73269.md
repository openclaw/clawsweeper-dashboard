## Summary

Keep open. Current main has a useful partial mitigation through `messages.visibleReplies` and group/channel message-tool-only defaults, but direct chats still default to automatic visible delivery, Feishu still wires partial/final text callbacks for automatic delivery, and there is no per-agent `directives.autoReply` or explicit public/internal assistant audience contract. The related #74928 remains the best broader tracker for the shared boundary decision, but this issue still carries concrete Feishu/direct-chat reproduction context.

## What This Changes

Request to prevent assistant text in mixed text/tool-call responses from being automatically delivered to channels, either through per-agent `autoReply=false` or by delivering only final assistant text after tool calls complete.

## Best Possible Solution

Define a shared visible-delivery contract that separates internal/runtime assistant text from public channel text, then apply it consistently across partial previews, block streaming, tool progress, final delivery, replay, and channel dispatchers. If maintainers choose #74928 as canonical, preserve this issue’s Feishu/direct-chat context there before closing this as superseded.

## Reproduction Assessment

Partly. Source and tests provide a high-confidence reproduction of the residual policy gap: direct chats default to automatic delivery while `message_tool_only` suppresses automatic callbacks and finals. The exact Feishu mixed text-then-tool-call leak still needs a focused fixture or live check.

## Solution Assessment

No. The current `messages.visibleReplies` mitigation is useful but not the complete requested fix because it is global/direct opt-in, not per-agent, and it does not add an explicit public/internal or final-only assistant-text boundary.
