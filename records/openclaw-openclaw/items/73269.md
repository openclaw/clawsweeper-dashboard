## What This Changes

The issue requests a way to prevent assistant text that precedes tool calls from being automatically delivered as visible channel output, such as a per-agent `directives.autoReply=false` setting or final-only delivery.

## Best Possible Solution

Use #74928 as the canonical tracker for the remaining work: define a shared visible-delivery contract that separates internal/runtime assistant text from public channel text, then validate it against Feishu direct chats, group/channel message-tool-only mode, provider stream parsing, replay/history helpers, and final channel dispatch.

## Reproduction Assessment

Partly. The exact live Feishu mixed text-then-tool-call transcript was not rerun.
