## Summary

Keep open: current main still has the reported usage aggregation problem, and this PR contains useful direction and tests, but the branch is not merge-ready because it mixes overview scopes, regresses the no-overview limit warning fallback, and lacks after-fix real behavior proof.

## What This Changes

The branch changes gateway `sessions.usage` and Control UI usage helpers to add matched/returned overview metadata, rank zero-cost channel and agent usage by tokens, display token-primary insight cards, and add regression tests.

## Best Possible Solution

Rework the fix into a documented usage contract where `limit` trims only returned rows, overview totals/aggregates/diagnostics share one matched-session scope, no-overview responses preserve the legacy warning fallback, zero-cost token-heavy channel and agent rankings stay visible, and channel attribution is coordinated with https://github.com/openclaw/openclaw/pull/52521.

## Reproduction Assessment

Yes, as a source-level path: seed more matched sessions than `limit`, include zero-cost token-heavy channel or agent usage plus peak-hour data outside the returned slice, then compare current-main totals/aggregates and Control UI diagnostics. I did not run tests in this read-only review.

## Solution Assessment

No. The PR's direction is useful, but this patch is not the best final solution because it leaves overview diagnostics split across full and limited scopes and drops the compatibility warning fallback when `overview` is unavailable.
