## Summary

Keep open. The reported usage aggregation bug is valid on current main, but this PR still has merge-blocking correctness issues in the dashboard scope handling and lacks required real behavior proof for an external PR.

## What This Changes

The PR changes `sessions.usage` and the Control UI usage overview to aggregate over all matched sessions, rank zero-cost channel/agent results by tokens, add overview scope fields, and cover the behavior with regression tests.

## Best Possible Solution

Revise or split the fix so returned rows stay limited while totals, rankings, warnings, duration, peak-hour, and mosaic diagnostics use explicit matched versus returned scopes, with focused tests and redacted real usage proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create more matching sessions than `limit` and include a zero-cost token-heavy channel or agent; current main aggregates only `limitedEntries` and cost-sorts channel rankings.

## Solution Assessment

No. The PR direction addresses valid bugs, but the current diff still mixes full-scope overview aggregates with limited-row diagnostics and regresses the no-overview warning fallback.
