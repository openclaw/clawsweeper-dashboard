## Summary

Keep open. Current main still has the usage aggregate undercount and cost-only zero-cost channel ranking, but this PR still has UI scope regressions and lacks required real behavior proof for an external contribution.

## What This Changes

The PR changes gateway `sessions.usage` aggregation/ranking, shared usage-ranking helpers/types, Control UI usage overview rendering, and regression tests for limit-scoped totals plus zero-cost channel/agent visibility.

## Best Possible Solution

Finish a usage contract where `limit` trims returned rows while totals, rankings, warnings, duration, peak-hour, and mosaic diagnostics use explicit matched or limited scopes, then add focused tests and real dashboard or gateway proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed more matched sessions than `limit` and include a zero-cost token-heavy channel or agent; current main aggregates only `limitedEntries` and cost-sorts channel rankings.

## Solution Assessment

No. The PR direction addresses valid bugs, but the current diff still mixes full-scope overview aggregates with limited-row diagnostics and regresses the no-overview warning fallback; the safer fix is to make every displayed metric's scope explicit.
