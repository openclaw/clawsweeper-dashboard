## Summary

Keep this open: the prior review comment is still accurate on current main, because `before_model_resolve` hook context exposes only effective model fields and not the persisted session override/source data that routing plugins need to yield to a user-pinned model.

## What This Changes

Expose session-scoped provider/model override state to `before_model_resolve` plugin hook context so routing plugins can respect explicit `/model` choices.

## Best Possible Solution

Add a backwards-compatible plugin hook context contract that carries session override values and source semantics, then document the precedence contract for routing plugins.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to inspect a `before_model_resolve` handler path: the hook context lacks session override/source fields while persisted session entries already store them.

## Solution Assessment

Yes, with one refinement. The narrow maintainable fix is additive SDK/runtime plumbing, but it should expose source semantics so plugins can distinguish user choices from auto fallback state.
