## Summary

Keep this open: current main still lacks session override/source fields in the plugin hook context, even though session state already persists the data routing plugins need to yield to a user-pinned model.

## What This Changes

Expose session-level provider/model override state to `before_model_resolve` hook context so routing plugins can yield to explicit `/model` choices.

## Best Possible Solution

Add a backwards-compatible hook context contract for session provider/model override plus source semantics, plumb it from persisted session state into `before_model_resolve`, and document the precedence contract for routing plugins.

## Reproduction Assessment

Yes, source-level. Current main shows the hook context and runtime `hookCtx` omit session override/source fields while `SessionEntry` persists `providerOverride`, `modelOverride`, and `modelOverrideSource`.

## Solution Assessment

Yes, with one refinement. Additive SDK/runtime plumbing is the narrow maintainable path, but the contract should include source/user-pinned semantics rather than only raw override values.
