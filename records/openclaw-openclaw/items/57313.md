## Summary

Keep this open. Current main still does not expose persisted session provider/model override data, or its source, to `before_model_resolve` plugin hooks, so routing plugins cannot reliably yield to a user-pinned `/model` choice without reading internal session storage.

## What This Changes

Request to add session-scoped provider/model override information to `PluginHookAgentContext` for `before_model_resolve` hooks.

## Best Possible Solution

Add a backwards-compatible plugin hook context contract that exposes session-scoped provider/model override values plus source semantics to `before_model_resolve`, then document the precedence so routing plugins can yield to explicit user selections without reading internal session files.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create or inspect a `before_model_resolve` hook and its `ctx` can only see effective `modelProviderId`/`modelId`, while session entries already persist override/source fields outside that context.

## Solution Assessment

Yes, with one refinement. Exposing the session override fields is the narrow maintainable fix, but the implementation should also expose `modelOverrideSource` or a user-pinned boolean so plugins can distinguish user selections from auto fallback state.
