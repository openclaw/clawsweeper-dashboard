## Summary

Keep open. The additive `resumable` field is the right general direction for the linked session-state bug, but this PR currently contradicts the existing send contract for `killed` sessions and tells agents that `sessions_send` interrupts active sessions even though the Gateway documents `sessions.steer` as that path.

## What This Changes

The PR adds a `resumable` boolean to `sessions_list` rows, updates the agent-facing tool description, and refreshes related tests and prompt snapshots.

## Best Possible Solution

Keep the additive metadata approach, but align `resumable` and its prompt wording with the actual Gateway send/reuse contract and prove the patched `sessions_list` behavior in OpenClaw.

## Reproduction Assessment

Yes, at source level. Current main exposes terminal-looking `status` values without resumability metadata, and the PR defects are visible by comparing the new mapping and description against the current send, lifecycle, and protocol code.

## Solution Assessment

No, not as written. Additive metadata is the maintainable path, but the field and wording need to match the actual `sessions_send`/`sessions.steer` contract before merge.
