## Summary

Keep open. The PR addresses a real session-state bug, but the submitted patch is not merge-ready because it marks `killed` sessions as non-resumable despite current `sessions.send` still targeting existing sessions, and it adds a row field without updating the shared `SessionListRow` type.

## What This Changes

The PR adds a `resumable` field to `sessions_list` rows, updates the sessions-list tool description, and adds mocked status-mapping tests.

## Best Possible Solution

Keep the additive `resumable` direction, but make it match the actual sendability contract, update the shared row/test types, and prove it with a real session-list plus follow-up send flow.

## Reproduction Assessment

Yes, at source level. Current `sessions_list` exposes last-run status without resumability metadata, while `sessions.send` can target existing sessions and reactivate completed subagent records after an acknowledged send.

## Solution Assessment

No, not as submitted. The additive field is the right shape, but it must update the shared types and avoid contradicting the current `sessions_send` contract for killed-status sessions.
