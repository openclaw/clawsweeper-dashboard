## Summary

Keep open: current main still lacks a `before_steering` hook, and active-run steering still queues text into the runtime without a dedicated per-steering blocking or rewrite hook. This is a security-sensitive public plugin API addition, so it needs maintainer design instead of cleanup closure or an automated fix lane.

## What This Changes

The issue requests a new blocking plugin hook that can inspect, block, or rewrite steering messages before they are injected into active agent turns.

## Best Possible Solution

Design a documented, generic steering input gate that covers normal inbound steering, explicit `/steer`, and runtime steering paths, with tests for block/rewrite behavior and clear failure-policy semantics.

## Reproduction Assessment

Yes, by source inspection: current main has no `before_steering` hook, while inbound steer mode and explicit `/steer` still call the runtime queue path directly. I did not run a live exploit because the actionable finding is a public security-hook design gap rather than a narrow runtime crash.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should first decide the public API contract, failure policy, metadata exposure, and relationship to the existing `before_agent_run` gate.
