## Summary

Keep open: current main still lacks a dedicated pre-queue steering hook, while active-run steering and explicit `/steer` continue to enqueue steering text directly into the runtime. This is a security-sensitive public plugin API addition, so it needs maintainer/API design rather than cleanup closure or an automatic fix lane.

## What This Changes

The issue requests a new blocking plugin hook that can inspect, rewrite, or reject steering messages before they enter active agent turns.

## Best Possible Solution

Design a documented, generic steering input-gate plugin API covering inbound steering and explicit `/steer`, with clear block/rewrite semantics, redaction/audit policy, and regression tests across Pi and Codex steering paths.

## Reproduction Assessment

Yes, by source inspection: current main has no `before_steering` hook, and active inbound plus explicit steer paths call `queueEmbeddedPiMessageWithOutcome` before any dedicated per-steering gate. I did not run a live exploit because the remaining work is a security-sensitive public hook design.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should first decide pre-queue versus model-boundary placement, failure policy, metadata exposure, rewrite semantics, and docs/test coverage.
