## Summary

Keep open: current main still lacks a `before_steering` hook, and both inbound active-run steering and explicit `/steer` queue text directly into Pi steering without a dedicated plugin gate. This is a security-sensitive public plugin API change, so it needs maintainer design rather than cleanup closure or an automatic repair lane.

## What This Changes

Add a public plugin hook that can inspect, rewrite, or block steering messages before they are queued into active agent turns.

## Best Possible Solution

Design a generic documented input-gate seam for active-run steering that covers normal inbound steering and explicit `/steer`, with regression coverage for block, rewrite, redaction, and failure-policy behavior before runtime injection.

## Reproduction Assessment

Yes, by source inspection: current main has no `before_steering` registration, active inbound steering queues `followupRun.prompt`, and `/steer` queues command text into the same runtime path. I did not run a live exploit; this is a source-reproducible security/API gap rather than a narrow failing test reproduction.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should decide the public API, fail-open versus fail-closed behavior, metadata/redaction contract, and queue-boundary versus model-boundary placement before implementation.
