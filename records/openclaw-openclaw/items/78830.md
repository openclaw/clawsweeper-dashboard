## Summary

Keep open: current main still has no `before_steering` hook, and the active inbound and explicit steer paths queue text into the active runtime without a dedicated blocking plugin gate. This is a security-sensitive public plugin API addition, so it should stay with maintainer/API review rather than cleanup closure or an automatic fix lane.

## What This Changes

The issue requests a blocking plugin hook that can inspect, rewrite, or reject steering messages before they are injected into active agent turns.

## Best Possible Solution

Design a documented, generic steering input-gate seam for active inbound steering and explicit `/steer`, with regression coverage for block and rewrite behavior before runtime injection.

## Reproduction Assessment

Yes, by source inspection: current main lacks `before_steering`, active inbound steering calls `queueEmbeddedPiMessageWithOutcome` with the inbound prompt, and `/steer` calls the same queue path with the command text. I did not run a live exploit because this is a security-sensitive public API gap rather than a narrow failing test case.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should decide the public hook contract, failure policy, metadata fields, permission gating, and runtime placement before implementation.
