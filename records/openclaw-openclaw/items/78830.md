## Label Justifications

- `P1`: The issue describes a default active-run channel path where security plugins cannot currently block steering input at the requested boundary.
- `impact:security`: The requested hook is for prompt-injection screening and security-policy enforcement before steering input reaches the active runtime queue.

## Summary

Keep open: current main and the latest release still lack a steering-specific plugin hook, while normal active-run steering and explicit `/steer` still queue user text directly. The shipped `before_agent_run` gate is an important mitigation before model submission, but it does not settle the requested pre-steering API, metadata, failure policy, or transcript/queue boundary.

## What This Changes

The issue requests a public `before_steering` plugin hook so plugins can inspect, rewrite, or block normal active-run steering and explicit `/steer` inputs before they are queued.

## Best Possible Solution

Design a documented steering input-gate contract that covers normal inbound steering and explicit `/steer`, or explicitly document why `before_agent_run` is the supported security boundary for steering inputs.

## Reproduction Assessment

Yes, by source inspection. Current main defaults inbound active-run behavior to `steer`, and both normal steering and explicit `/steer` queue text without a steering-specific block/rewrite hook; I did not run a live prompt-injection exploit.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should first decide whether the right security boundary is pre-queue, pre-transcript, `before_agent_run`, or a shared runtime steering gate.
