## Summary

Keep open: current main does not implement a `before_steering` hook, and source inspection shows steering messages still enter the active runtime queue without a dedicated per-steering blocking hook. This is a security-sensitive public plugin API change, so it needs maintainer design rather than cleanup closure or an automatic fix lane.

## What This Changes

The issue requests a new `before_steering` plugin hook that can inspect, modify, or block active-run steering messages before they are queued into the runtime.

## Best Possible Solution

Design a generic, documented plugin input-gate seam for active-run steering across normal inbound steering and explicit `/steer`, with regression coverage for blocking and modification before runtime injection.

## Reproduction Assessment

Yes, by source inspection: current main defaults inbound queueing to `steer`, active inbound and explicit `/steer` paths call `queueEmbeddedPiMessage`/`activeSession.steer`, and `before_steering` is absent from the hook registry. I did not run a live exploit because this is a security-sensitive plugin API gap rather than a narrow failing test case.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should first decide the public API, failure policy, metadata contract, and whether the gate belongs before queuing or at the runtime model-boundary layer.
