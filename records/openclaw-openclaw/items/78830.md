## Summary

Keep open: current main has no `before_steering` hook, and active steering still reaches Pi/Codex runtime queues without a dedicated per-steering blocking plugin API. Existing `before_agent_run` helps at Pi model submission, but it does not provide the requested pre-queue seam or visibly cover the Codex `turn/steer` path.

## What This Changes

The issue requests a new `before_steering` plugin hook that can block or rewrite active-run steering messages before runtime injection.

## Best Possible Solution

Design a documented, runtime-neutral steering input gate that covers normal inbound steering and explicit `/steer`, with tests for block, rewrite, audit behavior, and Codex/Pi parity.

## Reproduction Assessment

Yes, by source inspection: current main lacks `before_steering`, direct inbound and explicit steer paths call `queueEmbeddedPiMessage`, and the Codex app-server path sends `turn/steer` without a visible blocking gate. I did not run a live exploit because the report is a security-sensitive API gap.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should decide whether the right seam is pre-queue, model-boundary, or an extension of `before_agent_run`, including failure policy and metadata exposure.
