Keep open: current main and the latest v2026.6.5 release still contain the provider-local Kimi delimiter sanitizer and the generic user-facing sanitizer, but neither covers the reported markerless plain-text reasoning preamble. ## What This Changes

The issue reports internal reasoning text leaking into Telegram/user-facing replies on OpenClaw 2026.6.5 with Ollama/Kimi, with a later comment reporting the same markerless class on Gemini before 2026.6.5.

## Best Possible Solution

Keep this issue open while maintainers decide and land a narrowly tested delivery-level sanitizer, with reasoning hidden by default and any optional visibility toggle handled as an explicit product decision.

Yes, by source inspection: the reported markerless preamble is not matched by the current Kimi delimiter sanitizer or the current user-facing delivery sanitizer. I did not run a live Telegram/Ollama/Gemini reproduction in this read-only review.

## Solution Assessment

Unclear as a final product path: delivery-level sanitization is the right boundary for cross-provider markerless leaks, but maintainers still need to choose the false-positive tolerance and optional reasoning-visibility behavior.
