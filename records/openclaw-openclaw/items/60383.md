## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve later element-array fallthrough:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. With the new title join, a card that has a title plus an empty first `elements` array returns the title immediately and never reaches later nonempty arrays or post fallback content. Return title-only after all element/post candidates are exhausted.
  - confidence: 0.87
- **[P2] Bound nested card-array traversal:** `extensions/feishu/src/send.ts:228-229`
  - body: The new array branch recurses back into `extractInteractiveElementsText()` for every nested array without a depth or node bound. Feishu message content is external JSON, so a deeply nested array that current main ignores can now overflow the stack during message reads. Flatten the one-level row shape or use a bounded iterative traversal.
  - confidence: 0.78
