## Summary

Keep open: current main still misses the reported Feishu fallback shape, and this branch is not merge-ready because it can short-circuit later fallback candidates, recurses unbounded over external card JSON, and lacks inspectable real Feishu proof.

## What This Changes

The PR adds Feishu interactive-card parsing for titles, nested fallback rows, text/plain_text/link elements, and focused `getMessageFeishu` tests.

## Best Possible Solution

Land a Feishu-plugin-local parser/test fix that handles top-level title plus nested text/link fallback rows, preserves empty-array and locale fallthrough, uses bounded traversal, and includes redacted Feishu behavior proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` follows current main to `[Interactive Card]`.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the final fix should preserve candidate fallthrough, bound nested-array traversal, and include real Feishu after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve later element-array fallthrough:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. With the new title join, a card that has a title plus an empty first `elements` array returns the title immediately and never reaches later nonempty arrays or post fallback content. Return title-only after all element/post candidates are exhausted.
  - confidence: 0.87
- **[P2] Bound nested card-array traversal:** `extensions/feishu/src/send.ts:228-229`
  - body: The new array branch recurses back into `extractInteractiveElementsText()` for every nested array without a depth, node, or output bound. Feishu message content is external JSON, so a deeply nested array that current main ignores can now overflow the stack or burn CPU during message reads. Flatten the known one-level row shape or use a bounded iterative traversal.
  - confidence: 0.78
