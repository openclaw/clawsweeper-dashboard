## Summary

Keep open. Current main and the latest release still miss the reported Feishu interactive-card fallback shape, and this PR is useful but not merge-ready because it can skip later extractable card content, recursively traverses external card arrays without bounds, and still lacks inspectable real Feishu after-fix proof.

## What This Changes

The PR extends Feishu interactive-card parsing in `extensions/feishu/src/send.ts` and adds `getMessageFeishu` tests for nested fallback rows, card titles, and anchor text.

## Best Possible Solution

Land one Feishu-plugin parser fix that handles top-level titles, bounded one-level fallback rows, `text`/`a` nodes, existing fallthrough behavior, focused tests, and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` reaches current main's parser path, which rejects the array row and has no `tag: "text"` branch.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the final fix should preserve later candidate and post-fallback content, bound nested-array traversal, and include real Feishu after-fix proof.
