## What This Changes

This PR updates Feishu interactive-card parsing in `extensions/feishu/src/send.ts` to extract titles, nested post-format text/link elements, and adds `getMessageFeishu` regression tests.

## Best Possible Solution

Land one Feishu-plugin-local parser fix that handles top-level titles, bounded one-level fallback rows, `text`/`a` nodes, existing fallthrough behavior, focused tests, and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` reaches current main's parser path, which rejects the array row and has no `tag: "text"` branch.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the final fix should preserve later candidates and fallback content, bound nested-array traversal, and include real Feishu after-fix proof.

## Review Findings

Full review comments:

- **[P2] Preserve later card fallback candidates:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. Because this line joins the title into every candidate, a card with a title and an empty first `elements` array returns only the title and never reaches later nonempty arrays or fallback content.
  - confidence: 0.87

A malformed card can now overflow the stack or burn CPU during message reads; flatten the known row shape or use bounded iteration.
  - confidence: 0.8
