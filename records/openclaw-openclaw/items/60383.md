## What This Changes

The PR updates Feishu read-message interactive-card parsing and tests to extract titles, nested post-format text rows, plain-text nodes, and anchor labels from fallback card content.

## Best Possible Solution

Keep the Feishu parser fix in the plugin, repair it with bounded traversal and preserved fall-through semantics, then require redacted real Feishu read-message proof before merge.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 route read-message `msgType === "interactive"` through `parseInteractiveCardContent`, and the exact top-level `title` plus `elements: [[{ tag: "text" }]]` shape is neither handled by the element parser nor accepted by the post fallback.

## Solution Assessment

No, not as currently proposed. The Feishu parser is the right boundary and the branch addresses a real gap, but the recursive array parsing needs explicit bounds and title handling must preserve fallback across all element arrays before merge.
