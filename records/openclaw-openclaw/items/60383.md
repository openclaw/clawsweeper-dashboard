## Summary

Keep open. Current main and the latest release still skip the reported Feishu post-format fallback shape, and this PR is useful but not merge-ready because it lacks inspectable real Feishu proof and introduces two concrete parser risks.

## What This Changes

The PR updates Feishu interactive-card message parsing to extract titles, nested post-format fallback rows, text/plain_text/link elements, and adds focused `getMessageFeishu` tests.

## Best Possible Solution

Land one Feishu-plugin-local parser fix that handles top-level titles, bounded one-level fallback rows, `text`/`a` nodes, current fallthrough behavior, focused tests, and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` reaches current main's parser and falls back instead of extracting the text.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the final fix should preserve later candidate fallthrough, bound nested-array traversal, and include real Feishu after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve later element-array fallthrough:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. With the new title join inside the candidate loop, a card with a title plus an empty first `elements` array returns the title immediately and never reaches later nonempty arrays or post fallback content.
  - confidence: 0.87
- **[P2] Bound nested card-array traversal:** `extensions/feishu/src/send.ts:228-229`
  - body: This array branch recurses back into `extractInteractiveElementsText()` for every nested array from Feishu message JSON without a depth, node-count, or output limit. A malformed card can now overflow the stack or burn CPU during message reads; flatten the known row shape or use bounded iteration.
  - confidence: 0.78
