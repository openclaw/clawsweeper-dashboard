The PR changes the Feishu plugin parser and tests to extract card titles, nested fallback rows, text/plain-text nodes, and links from interactive-card message reads.

## Best Possible Solution

Land one bounded Feishu plugin parser/readback fix that handles known nested fallback rows and text/link/title shapes, preserves existing template and post fallbacks, adds regression tests, and has redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. A getMessageFeishu() response with msg_type:"interactive" and body.content containing { title, elements: [[{ tag:"text", text:"..." }]] } reaches current main's parser, where the nested row is ignored and tag:"text" is unsupported.

## Solution Assessment

No, not as currently patched. The Feishu plugin parser is the right boundary, but this branch should preserve later fallback candidates, avoid unbounded recursion over external card JSON, and provide inspectable real Feishu proof before merge.
