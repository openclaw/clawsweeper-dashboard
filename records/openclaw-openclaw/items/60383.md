## Summary

Keep open. Current main still misses the reported Feishu fallback shape, and this PR contains useful parser/test work, but the branch has blocking parser/security findings and no inspectable real Feishu after-fix proof.

## What This Changes

The PR changes Feishu interactive-card message parsing to extract card titles, nested post-format fallback rows, text/plain_text/link elements, and adds focused `getMessageFeishu` tests.

## Best Possible Solution

Land one Feishu-plugin-local parser/test fix that handles title plus nested text/link fallback rows, preserves later fallthrough, uses bounded traversal, and includes redacted real Feishu behavior proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` follows current main to `[Interactive Card]`.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the final fix should preserve candidate fallthrough, bound nested-array traversal, and include real Feishu after-fix proof.
