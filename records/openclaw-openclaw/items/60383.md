## Summary

Keep open. The PR targets a real Feishu message-read parser gap, but the current patch still has blocking parser fallthrough and external-input traversal issues, and the external PR has no inspectable real Feishu after-fix proof.

## What This Changes

The PR extends Feishu interactive-card message parsing in `extensions/feishu/src/send.ts` and adds `getMessageFeishu` tests for nested fallback rows, card titles, and anchor text.

## Best Possible Solution

Land one Feishu-plugin-local parser fix that handles top-level titles, bounded one-level fallback rows, `text`/`a` nodes, existing fallthrough behavior, focused tests, and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` reaches current main's parser path, which rejects the array row and has no `tag: "text"` branch.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the final fix should preserve later candidate and post-fallback content, bound nested-array traversal, and include real Feishu after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve later element-array fallthrough:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. Because this line joins the title into every candidate, a card with a title and an empty first `elements` array returns only the title and never reaches later nonempty arrays or fallback content.
  - confidence: 0.87
- **[P2] Bound nested card-array traversal:** `extensions/feishu/src/send.ts:228-229`
  - body: This array branch recurses back into `extractInteractiveElementsText()` for every nested array from Feishu message JSON without a depth, node-count, or output limit. A malformed card can now overflow the stack or burn CPU during message reads; flatten the known row shape or use bounded iteration.
  - confidence: 0.8
