## What This Changes

The PR updates Feishu interactive-card message parsing and tests to extract titles, nested post-format fallback rows, `text` and `a` nodes, and anchor labels from `message(action=read)` responses.

## Best Possible Solution

Land one bounded Feishu-plugin parser/readback fix that handles top-level titles, one-level fallback rows, `text` and `a` nodes, current fallthrough behavior, focused regression tests, and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. A `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` parsed as `{ title, elements: [[{ tag: "text", text: "..." }]] }` reaches current main's parser, which rejects the row array and has no `tag: "text"` branch.

## Solution Assessment

No, not as currently patched. The parser boundary is right, but the final fix should use bounded one-level row handling, preserve later candidate fallthrough, and include real Feishu after-fix proof.

## Review Findings

Full review comments:

- **[P2] Preserve later element-array fallthrough:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. Because this line joins the title into every candidate, a card with a title and an empty first `elements` array returns only the title and never reaches later nonempty arrays or fallback content.
  - confidence: 0.87

  - confidence: 0.8
