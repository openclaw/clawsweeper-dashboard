## Summary

Keep open. Current main still misses the reported top-level nested Feishu fallback shape, and the latest PR head fixes the earlier whitespace mismatch but introduces a title-only early return that can skip later nonempty element arrays; it also needs an active changelog entry for the user-facing fix.

## What This Changes

The PR changes the Feishu message parser and tests to extract title, text, plain-text, and link content from interactive-card fallback payloads with nested top-level elements.

## Best Possible Solution

Land a narrow Feishu parser/test update that collects nonempty candidate content before prepending title text, preserves empty-array fallthrough, and records the actual landing fix in the active changelog.

## Reproduction Assessment

Yes. A mocked getMessageFeishu response with msg_type interactive and body.content containing { title, elements: [[{ tag: "text" }]] } still follows current main to the placeholder because nested arrays are ignored and the post fallback does not accept that shape.

## Solution Assessment

No. The extraction direction is narrow and appropriate, but this patch should choose the first nonempty element/post fallback content before returning title-only content, otherwise it regresses existing fallthrough behavior.
