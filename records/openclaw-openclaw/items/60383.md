## Summary

Keep open. Current main still misses the reported Feishu top-level post-format fallback shape, but this PR is not ready to merge because it introduces unbounded recursive traversal, can return title-only before checking later fallback candidates, and lacks external-contributor real behavior proof.

## What This Changes

The branch updates Feishu interactive-card message parsing to include top-level/header titles, text/link fallback nodes, nested element arrays, and focused parser tests.

## Best Possible Solution

Land a Feishu-plugin-local parser/test fix that handles top-level title plus nested text/link fallback rows, preserves empty-array/i18n/post fallthrough, uses bounded or one-level traversal, and includes redacted Feishu after-fix proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and content shaped as `{ title, elements: [[{ tag: "text", text: "..." }]] }` follows current main to `[Interactive Card]` because array rows are rejected and the post fallback requires a `content` array.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the branch should preserve later fallback candidates, bound or limit nested traversal, and provide real Feishu after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve later element-array fallthrough:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. With the new title join inside the loop, a card with a title plus an empty first `elements` array returns the title immediately and never reaches later nonempty arrays or post fallback content. Return title-only after all element/post candidates are exhausted.
  - confidence: 0.87
- **[P2] Bound nested card-array traversal:** `extensions/feishu/src/send.ts:228-229`
  - body: The new array branch recurses back into `extractInteractiveElementsText()` for every nested array without a depth, node, or output bound. Feishu message content is external JSON, so a deeply nested array that current main ignores can now overflow the stack or burn CPU during message reads. Flatten the one-level row shape or use a bounded iterative traversal.
  - confidence: 0.78
