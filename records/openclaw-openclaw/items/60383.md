## Best Possible Solution

Land one bounded Feishu-plugin parser/readback fix that preserves existing fallback ordering, handles top-level titles plus known one-level post-format row shapes, includes focused tests, and has redacted real Feishu proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently patched. The Feishu plugin parser is the right boundary, but this branch should preserve later fallback candidates, avoid unbounded recursion over external card JSON, and provide inspectable real Feishu proof before merge.
