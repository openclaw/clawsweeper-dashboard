## Summary

Keep open: the PR targets a real Feishu parser gap that is still present on current main, but the branch regresses existing element-candidate fallthrough, adds unbounded recursive parsing of external card JSON, and still lacks inspectable real Feishu behavior proof.

## What This Changes

The PR updates Feishu interactive-card parsing to extract title, text, and link content from nested post-format fallback payloads and adds focused parser tests.

## Best Possible Solution

Land a Feishu-plugin-local parser/test fix that handles title, text, and link fallback rows after all candidates are exhausted, uses bounded or one-level traversal, and includes redacted real Feishu read proof.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` follows current main to `[Interactive Card]`.

## Solution Assessment

No, not as currently patched. The parser direction is right, but the branch should preserve later candidate fallthrough, bound nested-array traversal, and provide real Feishu after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve later element-array fallthrough:** `extensions/feishu/src/send.ts:335-337`
  - body: `readInteractiveElementArrays()` visits `parsed.elements` before `body.elements` and i18n. With the new title join, a card that has a title plus an empty first `elements` array returns the title immediately and never reaches later nonempty arrays or post fallback content. Return title-only after all element/post candidates are exhausted.
  - confidence: 0.87
- **[P2] Bound nested card-array traversal:** `extensions/feishu/src/send.ts:228-229`
  - body: The new array branch recurses back into `extractInteractiveElementsText()` for every nested array without a depth or node bound. Feishu message content is external JSON, so a deeply nested array that current main ignores can now overflow the stack during message reads. Flatten the one-level row shape or use a bounded iterative traversal.
  - confidence: 0.78
