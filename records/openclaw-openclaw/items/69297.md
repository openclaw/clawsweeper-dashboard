- **[P1] Return admission for the admin allowlist accept path:** `extensions/whatsapp/src/inbound/access-control.ts:83-89`

  - confidence: 0.9
- **[P1] Validate admin config before accepting the privacy key:** `src/config/zod-schema.providers-whatsapp.ts:26`
  - body: The new public `admin` field is documented as an E.164 privacy boundary, but the schema accepts any string while the runtime ignores values that do not normalize to E.164. A typo can pass config validation and silently leave the intended admin protection inactive, so the config should reject invalid values or produce an explicit diagnostic before this ships.
  - confidence: 0.86
- **[P1] Align admin docs with preserved owner command access:** `docs/channels/whatsapp.md:276`
  - body: This sentence says the configured admin becomes the only user allowed to run slash commands, but the PR code preserves explicitly configured owner command access in admin-scoped groups. Either the code should enforce the admin-only model or the docs/UI should state the owner exception so users do not rely on a stronger privacy boundary than OpenClaw enforces.
  - confidence: 0.84
