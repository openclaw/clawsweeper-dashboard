- **[P1] Return admission for admin allowlist accepts:** `extensions/whatsapp/src/inbound/access-control.ts:83-89`

  - confidence: 0.91

  - body: `groups.*.admin` is documented and used as a WhatsApp group privacy boundary, but the schema accepts any string and the runtime later ignores values that do not normalize to an E.164-shaped number. A typo can pass config validation while silently leaving the admin restriction inactive or falling back, so this should reject invalid admin values or surface an explicit config diagnostic before merge.
  - confidence: 0.84
- **[P2] Align docs with preserved owner command access:** `docs/channels/whatsapp.md:276`
  - body: The docs say the configured admin becomes the only user allowed to run commands, but the implementation deliberately keeps explicit owner command access when an admin is configured. Please update the WhatsApp docs and config UI hint to describe the final admin-plus-owner semantics, or change the code/tests if the intended contract is truly admin-only.
  - confidence: 0.87
