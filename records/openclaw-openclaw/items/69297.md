## Summary

Keep open: current main does not implement the WhatsApp group admin config, and this PR is the active canonical implementation path, but it still needs real WhatsApp behavior proof and the public docs currently overstate command exclusivity compared with the runtime owner exception.

## What This Changes

The PR adds `channels.whatsapp.groups.<jid>.admin` and wildcard admin config with WhatsApp mention/ingress/command gating, schema/docs/UI metadata, regression tests, and several non-WhatsApp stabilization changes.

## Best Possible Solution

Align the WhatsApp admin privacy contract so docs and runtime agree on admin versus explicit-owner command access, keep the schema/generated/docs/test coverage, and land only after redacted real WhatsApp proof is posted.

## Reproduction Assessment

Not applicable as a direct bug reproduction because this is a new WhatsApp config/privacy feature. Source inspection confirms current main lacks `groups.*.admin`, while the PR adds and tests that feature path without real WhatsApp proof.

## Solution Assessment

No, not yet. The runtime direction is plausible, but the public docs overpromise that only the admin can run slash commands while the PR intentionally preserves explicit owner command access; that contract needs alignment before merge.
