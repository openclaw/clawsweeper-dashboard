## Summary

## What This Changes

The PR guards MCP loopback union schema `properties` reads, skips unreadable variants, and adds a regression for an unreadable `properties` getter under `anyOf`.

## Best Possible Solution

Make union-variant flattening transactional: read `properties`, property entries, and `required` before mutating the merged schema, skip unreadable variants, and keep focused gateway regression coverage.

## Reproduction Assessment

Yes. A focused test can construct an `anyOf` variant whose `properties` or `required` getter throws; current main reads those getters during MCP `tools/list` projection, and PR head still mishandles the unreadable-`required` variant from source inspection.

## Solution Assessment

No. The direction is right, but the patch is not the best fix until it reads all variant fields transactionally and skips the whole variant on any unreadable field needed for projection.
