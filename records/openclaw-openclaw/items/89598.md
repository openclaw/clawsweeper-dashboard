## Summary

Keep open: this is a focused gateway bug-fix PR, but the branch still treats an unreadable union-variant `required` field as an empty required set after merging that variant's properties. That can make MCP loopback `tools/list` advertise required arguments as optional, so the prior ClawSweeper blocker is still valid.

## What This Changes

The branch adds guarded schema reads for MCP loopback union variants and a regression for an unreadable `properties` getter under `anyOf`.

## Best Possible Solution

Make each union variant merge transactional: read `properties`, property entries, and `required` before mutating `mergedProps`; skip unreadable variants with a warning and add the missing throwing-`required` regression.

## Reproduction Assessment

Yes. Source inspection shows current main can throw while reading union variant schema fields, and the PR demonstrates the unreadable-`properties` path; the remaining patch defect is reproduced by the analogous readable-properties plus throwing-`required` variant.

## Solution Assessment

No. The PR is close, but the best fix is to skip a variant when any field needed for its merge is unreadable rather than merging its properties with an empty required set.
