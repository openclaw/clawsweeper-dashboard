## Summary

Keep open: this focused maintainer PR is still useful, but the current head has one merge-blocking schema correctness issue because unreadable `required` is treated as an empty set after a variant's properties may already have been merged.

## What This Changes

The branch guards MCP loopback union-schema flattening against unreadable nested `properties` fields and adds a regression that preserves usable sibling variants.

## Best Possible Solution

Make each union variant merge transactional: read `properties`, property entries, and `required` before mutating the merged schema, skip unreadable variants with a warning, and keep focused gateway regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main can throw while reading unreadable union-variant schema fields, and the PR regression covers the unreadable `properties` path; the remaining reviewed failure is the analogous unreadable `required` path on the PR head.

## Solution Assessment

No. The fix is close, but the best fix is to skip a variant when any field needed for its merge is unreadable rather than merging its properties with an empty required set.
