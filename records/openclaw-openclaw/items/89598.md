## Best Possible Solution

Make union-variant flattening transactional: read `properties`, property entries, and `required` before merging, skip unreadable variants, and keep focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. Current main directly reads union variant schema fields during MCP `tools/list` construction, and the PR regression covers the unreadable-`properties` path while the remaining unreadable-`required` case is source-reproducible on the PR head.

## Solution Assessment

No. The patch is close, but treating unreadable `required` as absent can mis-advertise required arguments; the maintainable fix is to skip that variant before merging any of its fields.
