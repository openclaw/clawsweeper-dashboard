## What This Changes

## Best Possible Solution

Make union-variant flattening transactional: read `properties`, property entries, and `required` before mutating the merged schema, skip unreadable variants, and preserve focused gateway regression coverage.

Yes. Current main directly reads union variant schema fields during MCP `tools/list` construction, and the PR regression covers the unreadable-`properties` path while the remaining unreadable-`required` path is source-reproducible on the PR head.

## Solution Assessment

No. The patch is close, but treating unreadable `required` as absent can mis-advertise required arguments; the maintainable fix is to skip that variant before merging any of its fields.
