## Summary

Keep open: current main still has the source-reproducible MCP loopback schema bug, but this PR still has a narrow blocker because unreadable `required` is treated as an empty set after properties may already be merged.

## What This Changes

The PR guards MCP loopback union schema flattening against unreadable nested variant fields and adds a regression for an unreadable `properties` getter.

## Best Possible Solution

Make each union-variant merge transactional: read properties, property entries, and required before mutating the merged schema, skip unreadable variants with a warning, and keep focused gateway regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main can throw while reading union variant schema fields, and the PR regression covers the unreadable `properties` path; the remaining source-reproducible blocker is the analogous unreadable `required` path.

## Solution Assessment

No. The fix is close, but the best fix is to skip a variant when any field needed for its merge is unreadable rather than merging its properties with an empty required set.
