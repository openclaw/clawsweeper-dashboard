## Summary

Keep open: current main still has the MCP loopback schema crash class, and this PR is a focused fix, but the patch still has one merge-blocking schema-accuracy bug when `required` is unreadable.

## What This Changes

The PR guards MCP HTTP loopback union schema flattening against unreadable `properties` variants and adds a regression for an unreadable `properties` getter under `anyOf`.

## Best Possible Solution

Make each union variant merge transactional: read `properties`, property entries, and `required` before mutating the merged schema, skip unreadable variants with a warning, and keep focused gateway regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is `anyOf` with one variant that has readable `properties` and a throwing `required` getter plus a healthy sibling; the PR path merges the bad variant's properties and treats its required set as empty.

## Solution Assessment

No. The PR is close, but the best fix is to skip a variant when any field needed for its merge is unreadable rather than merging its properties with an empty required set.
