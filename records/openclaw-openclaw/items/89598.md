## Summary

Keep open: this maintainer-labeled draft still addresses a real current-main MCP loopback schema crash, but the branch has one narrow correctness blocker because unreadable `required` is treated as optional after properties are merged.

## What This Changes

The PR adds guarded MCP loopback union-schema reads in `flattenUnionSchema` and a regression for an unreadable `properties` getter under `anyOf`.

## Best Possible Solution

Make each union-variant merge transactional: read `properties`, property entries, and `required` before mutating the merged schema, skip unreadable variants with a warning, and keep focused gateway regression coverage.

## Reproduction Assessment

Yes, from source inspection. Current main directly reads union variant schema fields during MCP `tools/list` construction, and the PR regression covers the unreadable `properties` path while the remaining unreadable-`required` case is still source-reproducible.

## Solution Assessment

No. The patch is close, but the best fix is to skip a variant when any field needed for its merge is unreadable rather than merging its properties with an empty required set.
