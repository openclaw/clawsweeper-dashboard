## Summary

Keep open. Current main and v2026.6.8 still let an exact semver `--tag` resolve in dry-run before npm availability is proven, and the two linked candidate fixes remain open rather than merged.

## What This Changes

Make `openclaw update --dry-run --tag <version>` warn or fail when the requested package version is not yet available on npm, so dry-run and apply do not disagree during registry propagation.

## Best Possible Solution

Review and land one canonical update CLI fix that checks confirmed missing public npm targets before dry-run/apply while preserving GitHub specs, package-spec overrides, tarballs, and custom registry behavior.

## Reproduction Assessment

Yes. Source inspection shows exact semver tags still resolve locally before dry-run returns, and a live npm registry read for an absent version-like target returns 404 while `latest` resolves.

## Solution Assessment

Yes, the right layer is update CLI planning/preflight. Automatic fallback to a GitHub package spec would change update source semantics and should require explicit maintainer product sign-off.
