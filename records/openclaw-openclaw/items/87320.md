## Summary

Keep open. Current main and the latest release still let exact semver `--tag` values resolve in dry-run before npm availability is proven, and the two linked candidate PRs remain open rather than merged.

## What This Changes

The issue reports that `openclaw update --dry-run --tag <version>` can approve a version before npm has published it, while apply later fails with `ETARGET`.

## Best Possible Solution

Review and land one canonical update CLI fix that checks confirmed missing public npm targets before dry-run/apply while preserving GitHub specs, package-spec overrides, tarballs, and custom registry behavior.

## Reproduction Assessment

Yes, at source level. Current `main` still returns exact semver tags before npm availability is checked, dry-run returns before package preflight/install, and a live npm read shows absent exact targets return 404 while `latest` resolves.

## Solution Assessment

Yes, the right layer is update CLI planning/preflight so dry-run and apply share the same package-target authority. Automatic fallback to a GitHub package spec would change update source semantics and should require explicit maintainer product sign-off.
