## Summary

Keep open. Current main and latest release still resolve exact semver `--tag` values locally in dry-run before npm availability is proven, and the two linked candidate fixes are open rather than merged.

## What This Changes

`openclaw update --dry-run --tag <version>` should warn or fail when the requested npm target is not yet published, instead of passing dry-run and failing during apply.

## Best Possible Solution

Land one canonical updater preflight that blocks confirmed missing public npm targets before both dry-run preview and mutable apply, while preserving non-registry specs and custom/transient registry behavior.
