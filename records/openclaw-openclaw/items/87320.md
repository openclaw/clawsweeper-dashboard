## What This Changes

The issue asks for `openclaw update --dry-run --tag <exact-version>` to fail or warn when the target package version is not yet published on npm, instead of passing dry-run and failing apply later with `ETARGET`.

## Best Possible Solution

Land one focused updater fix that makes dry-run and apply share an exact public npm target availability gate while preserving non-registry specs, custom registries, and transient network-error behavior.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, the right end state is a shared availability preflight or equivalent warning/failure in the updater's package target planning path. The remaining choice is maintainer policy around hard-fail versus warning and the exact compatibility boundary for public npm, custom registries, and non-registry specs.
