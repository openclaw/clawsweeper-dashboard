## Label Justifications

- `impact:auth-provider`: The failure is caused by gateway operator scope resolution and token authorization for a plugin RPC.

## Summary

Keep open: current main and the latest release still show the reported Workboard CLI and gateway scope mismatch, and the linked fix PR is open rather than merged.

## What This Changes

The issue reports that `openclaw workboard dispatch` is rejected for `operator.admin` even though the Workboard gateway method is documented and registered for `operator.write`.

## Best Possible Solution

Land or replace the linked PR with a least-privilege fix that makes the gateway classify `workboard.cards.dispatch` as `operator.write` or otherwise keeps CLI dispatch on the documented read/write scope path.

## Reproduction Assessment

Yes. Source inspection shows the current CLI sends only `operator.write`/`operator.read`, the Workboard gateway method declares `operator.write`, and the server fallback rejects unclassified methods as `operator.admin`; I did not run a live Gateway in this read-only review.

## Solution Assessment

No, the issue is not solved on current main. The best fix is a least-privilege gateway/plugin descriptor classification repair or an equivalent narrow dispatch scope path, not closing while the linked PR is still open and blocked.
