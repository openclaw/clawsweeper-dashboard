## Summary

Keep open: current main and latest release still carry the explicit Workboard CLI scope override, and the linked unmerged fix PR is the active path, so this issue is not implemented or safe to close yet.

## What This Changes

The issue reports that `openclaw workboard dispatch` is rejected as requiring `operator.admin` even though the Workboard dispatch RPC is documented and registered as `operator.write`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92066 or an equivalent narrow fix that lets the CLI gateway runtime choose scopes for this plugin RPC, then close this issue after the fix merges.

## Reproduction Assessment

Yes from source inspection, but not from a live command in this read-only review. Current main hardcodes write/read scopes in the Workboard CLI while unresolved gateway methods fall back to admin authorization.

## Solution Assessment

Yes, the narrowest maintainable solution is to remove the Workboard-specific explicit scope override or otherwise make plugin method descriptors available to the CLI scope resolver. The existing linked PR takes the small CLI-side path and should be reviewed rather than opening a duplicate repair job.
