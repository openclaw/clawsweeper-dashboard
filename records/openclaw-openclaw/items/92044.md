## What This Changes

The issue reports that `openclaw workboard dispatch` is rejected for `operator.admin` even though the Workboard dispatch Gateway method is documented and registered for `operator.write`.

## Best Possible Solution

Keep this issue open while https://github.com/openclaw/openclaw/pull/92066 is reviewed; close it only after a fix lands or if maintainers choose a different canonical repair.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only pass. Source review supports the reported path with medium confidence: the CLI sends only read/write scopes, while Gateway falls back to admin if the plugin method descriptor is not resolved.

## Solution Assessment

Unclear whether the linked PR is the best final fix. Removing the CLI's explicit narrow scopes is a plausible narrow mitigation, but maintainers should compare it against fixing plugin Gateway method descriptor visibility more generally.
