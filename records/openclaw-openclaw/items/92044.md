## Summary

Keep open: current main and the latest release still have the reported Workboard CLI scope-selection shape, and the open linked fix PR is the active implementation path rather than grounds to close this issue.

## What This Changes

The issue reports that `openclaw workboard dispatch` fails with `missing scope: operator.admin` even though the Workboard dispatch Gateway method is documented and registered as `operator.write`.

## Best Possible Solution

Review and land or replace the linked fix PR with a narrow repair that preserves `operator.write` as the Workboard dispatch contract and prevents plugin RPC methods from being silently promoted to admin-only.

## Reproduction Assessment

Yes, source-backed rather than live-run here: enable Workboard, make a ready card, then run `openclaw workboard dispatch` against a token-auth Gateway where the dispatch descriptor is not resolved. Current main still has the CLI hardcoded write/read scopes and the Gateway admin fallback for unresolved methods.

## Solution Assessment

Unclear which repair is best until the linked PR is reviewed. Removing the Workboard CLI's explicit scopes is a plausible narrow mitigation, but maintainers should verify whether the more general descriptor propagation path is needed for other plugin RPC methods.
