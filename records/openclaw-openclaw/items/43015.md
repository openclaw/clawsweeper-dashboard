## Summary

Keep open: current main and v2026.6.6 still reject ordinary `action="send"` payloads when channel-specific poll metadata such as `pollDurationSeconds: 60` is present, and Discord still throws on an empty `components.modal` skeleton. An open linked PR claims to close this but is conflicting, so closure would be premature.

## What This Changes

The issue requests narrowing the shared message send schema/runtime so schema-padded poll fields and empty Discord component/modal skeletons do not block ordinary sends.

## Best Possible Solution

Refresh one canonical owner-reviewed fix that treats only content-bearing poll question/options as send-time poll intent, normalizes wholly empty Discord component skeletons as absent, preserves strict validation for real poll actions, and proves ordinary send delivery plus real-poll rejection on live channels.

## Reproduction Assessment

Yes, at source level: current main classifies `pollDurationSeconds: 60` as poll intent and `runMessageAction` throws on `action="send"`; the Discord parser also deterministically rejects an empty modal fields array. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

No, current main is only a partial fix because shared poll modifier padding is handled but channel poll metadata and empty Discord modal skeletons still fail. The best path is one consolidated maintainer-approved fix rather than another parallel repair branch.
