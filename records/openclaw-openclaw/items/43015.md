## Summary

Keep open: current main and v2026.6.6 still contain the send-blocking poll-duration and empty-modal validation paths, and the linked closing PR is open but conflicting and not maintainer-modifiable.

## What This Changes

The issue asks for `message.send` schema and runtime behavior to stop treating schema-padded poll, component, and modal fields as explicit advanced-action intent on plain sends.

## Best Possible Solution

Land a maintainable core/plugin fix that keeps plain `message.send` robust against schema-padded poll and empty component/modal params while preserving explicit poll and Discord component actions.

## Reproduction Assessment

Yes, source-reproducible: current main still rejects `action="send"` when `pollDurationSeconds` is treated as poll intent, and the Discord parser still throws on an empty `components.modal.fields` skeleton. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No settled best fix yet: the right endpoint is likely a schema/runtime normalization fix at the shared message-tool boundary plus plugin-specific coverage, but the linked closing PR is conflicting and not maintainer-modifiable. Maintainers should review whether to narrow schemas, strip schema-padded fields, or combine both without weakening explicit advanced actions.
