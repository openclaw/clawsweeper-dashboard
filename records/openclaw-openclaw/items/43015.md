## Summary

Keep open. Current main still has source-visible poll-param send rejection and empty Discord modal skeleton failure paths, while several open repair PRs overlap with this cluster.

## What This Changes

The issue asks for the generic message send schema/runtime to stop treating auto-populated poll and Discord component/modal fields as explicit advanced send intent.

## Best Possible Solution

Consolidate on one canonical repair PR that strips or ignores model-filled advanced defaults for ordinary sends, scopes poll/component schema by action and channel where practical, and keeps strict validation for explicit poll and real modal requests.
