## Summary

Keep open. Current main still has source-visible false-positive paths for poll-populated `send` calls and empty Discord modal skeletons, while an open linked repair PR already references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to keep generic message sends reliable when schema-driven models auto-populate poll, Discord components, or modal fields that were not explicitly requested.

## Best Possible Solution

Land or replace one canonical narrow fix that strips or ignores model-filled advanced defaults for `send`, scopes poll/component schema by action and channel where practical, and preserves strict validation for explicit poll and real modal requests.
