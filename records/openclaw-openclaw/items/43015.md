## What This Changes

The issue asks OpenClaw to stop plain `message.send` calls from failing when GPT-family models auto-populate poll fields or empty Discord component/modal skeletons.

## Best Possible Solution

Review, repair, land, or replace the linked fix so plain sends strip or ignore model-filled advanced-field defaults while explicit poll and real modal validation stays strict.

## Reproduction Assessment

Yes. Source inspection on current main shows `action="send"` plus poll-looking params still reaches the reported guard, and focused tests still expect that rejection; the Discord empty-modal path is also source-visible when a non-empty components skeleton is supplied.

## Solution Assessment

Yes, the narrow schema/normalization direction is the best fit for the remaining bug. The issue should stay open while the linked repair PR is reviewed or replaced rather than creating another parallel fix lane.
