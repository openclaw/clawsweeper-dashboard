## Summary

Keep open. Current main has partial fixes for this cluster, but the remaining source path is still clear: mixed-action message schemas expose poll fields, `message.send` still throws when those fields look non-empty, and Discord still rejects empty modal skeletons if they reach the component parser; the canonical target appears to be https://github.com/openclaw/openclaw/pull/75652, which is still open and needs changes.

## What This Changes

The issue asks for ordinary `message.send` calls to avoid exposing or hard-failing on poll, Discord components, and modal fields that models may auto-populate.

## Best Possible Solution

Land a narrow semantic normalization fix, preferably through https://github.com/openclaw/openclaw/pull/75652 after its blockers are resolved, and keep any workspace media-root policy changes separate from the poll and Discord default-field fix.

## Reproduction Assessment

Yes. A high-confidence source reproduction is available: `message.send` with non-zero/default poll fields still reaches the poll-only guard, and a Discord components object with `modal.fields: []` still reaches a parser that throws.

## Solution Assessment

Yes for the semantic normalization direction, but not yet on current main. The safest path is to fix the open normalization PR or replace it with the same narrow helper/parser boundary changes rather than closing this issue early.
