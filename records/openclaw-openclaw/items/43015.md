## Summary

Keep open. Current main still exposes poll fields in the unified message tool schema, still rejects ordinary `action: "send"` calls when non-empty poll-looking params are present, and still treats a present Discord empty modal skeleton as a hard validation error; PR #46455 remains the direct open implementation candidate.

## What This Changes

The issue requests that ordinary `message.send` calls avoid schema-exposed poll, component, and modal parameters unless the user explicitly asks for those advanced actions.

## Best Possible Solution

Review, land, or explicitly supersede PR #46455 with a focused current-main patch that makes advanced poll/component fields action-aware for ordinary sends while preserving explicit poll and modal validation.

## Reproduction Assessment

Yes, source-reproducible. Current main exposes poll params in the shared message schema, routes populated poll-looking send params into the send-side guard, and has tests that still assert the rejection.

## Solution Assessment

Unclear. The requested direction is plausible, but the narrowest maintainable fix should be settled through PR #46455 or a focused replacement so ordinary sends tolerate model-populated noise without weakening explicit advanced-action validation.
