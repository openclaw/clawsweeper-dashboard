## Summary

The issue remains necessary: current main still rejects plain sends carrying schema-padded channel poll metadata, and it still strictly rejects empty Discord modal skeletons. The linked repair branches are open but conflicting, so the issue should remain the canonical user-impact tracker while maintainers consolidate a refreshed fix.

## What This Changes

Prevent generic `message.send` calls from failing when models auto-populate unused poll or Discord component fields.

## Best Possible Solution

Refresh one canonical PR that treats only non-empty poll question/options as send-time poll intent, retains strict validation for actual `poll` actions, normalizes wholly empty Discord component skeletons to absent, and demonstrates ordinary attachment delivery plus real-poll rejection on live Discord and Telegram.

## Reproduction Assessment

Yes at source level: current main classifies `pollDurationSeconds: 60` as poll intent and rejects `action="send"`, and the Discord parser deterministically rejects an empty modal fields array. This read-only review did not execute a live transport reproduction.

## Solution Assessment

Not yet as submitted: the content-bearing question/options rule is the narrowest maintainable core behavior, but the conflicting repair branches must be consolidated with plugin-owned empty-component handling and live delivery proof.
