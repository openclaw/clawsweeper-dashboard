## Summary

Keep open. Current `main` and the latest shipped tag still preserve the source-visible poll false-positive path for mixed `message` schemas, and the issue is paired with an open linked repair PR that has closing references for this issue and the Feishu companion.

## What This Changes

The issue reports that generic `message.send` exposes advanced poll and Discord component/modal fields, causing schema-driven models to populate fields that break ordinary sends.

## Best Possible Solution

Review, land, or replace the linked repair with a narrow fix that strips or ignores model-filled advanced defaults for `send`, scopes poll/component schema by action and channel, and keeps explicit poll plus real modal validation strict.

## Reproduction Assessment

Yes. Source inspection on current `main` and `v2026.5.12` shows mixed message schemas still expose poll fields and `action="send"` still reaches the poll guard; current focused tests also still expect the rejection.

## Solution Assessment

Yes. The linked repair direction is the narrow maintainable path for the central bug, while the newer Telegram `actions.poll` config-schema gap is already covered on current `main`.
