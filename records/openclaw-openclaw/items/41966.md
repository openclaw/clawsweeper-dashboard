## Summary

Keep open. Current main and v2026.5.22 still skip fenced `MEDIA:` directives without a runtime warning; docs now explain the rule and an open warn-only PR exists, so the remaining work is maintainer review of the parser contract rather than closing this issue.

## What This Changes

The issue asks OpenClaw to stop silently ignoring `MEDIA:` directives when models wrap them in Markdown code fences, via extraction, warning logs, or template/docs guidance.

## Best Possible Solution

Keep fenced-code extraction disabled by default, land a narrow warning diagnostic with parser coverage and real behavior proof, and treat extraction from fences as a separate opt-in contract change.

## Reproduction Assessment

Yes, at source level. Current main skips fenced lines before `MEDIA:` matching, and parser coverage asserts that a fenced directive remains visible text; this read-only review did not run tests.

## Solution Assessment

Unclear until maintainers choose the parser contract. Warn-only diagnostics are the narrowest safe fix for the silent failure, while extraction from fences needs an explicit behavior decision and regression coverage.
