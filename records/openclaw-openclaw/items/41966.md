## Summary

Keep open. Current main and v2026.5.2 still skip fenced `MEDIA:` lines before media validation, current tests assert that fenced tokens remain visible text, and the prompt/docs still describe own-line `MEDIA:` delivery without warning that Markdown fences suppress it; the related extraction PR #41980 was closed unmerged.

## What This Changes

The issue requests that fenced `MEDIA:` directives stop failing silently, either by extracting them, warning when they are skipped, or documenting that code fences suppress media delivery.

## Best Possible Solution

Define and document the fenced `MEDIA:` contract, then implement either constrained extraction through the existing validation path or an explicit warning plus prompt/docs guidance.

## Reproduction Assessment

Yes. Source inspection on current main shows fenced lines are skipped before `MEDIA:` matching, and the current parser test asserts a fenced `MEDIA:` line remains visible text.

## Solution Assessment

Unclear. Extraction is plausible if limited to valid own-line directives, but warning plus docs may be safer if code fences are intentionally treated as examples rather than delivery metadata.
