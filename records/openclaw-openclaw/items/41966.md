## Summary

Keep this issue open. Current main and the latest release still skip fenced `MEDIA:` lines before media validation, tests explicitly assert that behavior, and the user-facing prompt/docs still describe own-line `MEDIA:` as attachment delivery without warning that Markdown fences suppress it. The related fix PR was closed unmerged, so it remains useful prior art rather than a superseding implementation.

## What This Changes

The issue asks OpenClaw to make fenced code-block `MEDIA:` tokens observable by extracting them, warning about skipped tokens, or documenting that fences prevent attachment delivery.

## Best Possible Solution

Define the intended fenced `MEDIA:` contract, then either extract valid own-line fenced directives through the existing validation path with updated parser tests/docs, or preserve the skip behavior while adding an observable warning plus prompt and public docs guidance.

## Reproduction Assessment

Yes. The current source path shows fenced lines are skipped before `MEDIA:` matching, and the current parser test asserts that a fenced `MEDIA:` line remains text.

## Solution Assessment

Unclear. Extraction is a plausible fix if constrained to valid own-line directives, but warning plus docs is safer if code fences are intentionally treated as examples rather than delivery metadata.
