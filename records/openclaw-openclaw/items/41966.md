## Summary

Keep open. ## What This Changes

The issue asks OpenClaw to make fenced-code `MEDIA:` directives debuggable or deliverable instead of silently leaving them as visible text.

## Best Possible Solution

Keep fenced-code extraction disabled by default, review and land a narrow warn-only diagnostic with proof, and treat extraction from fences as a separate explicit contract change.

## Reproduction Assessment

Yes, at source level. Current main skips fenced lines before `MEDIA:` matching, and parser coverage asserts that a fenced directive remains visible text; this read-only review did not run tests.

## Solution Assessment

Unclear until maintainers choose the parser contract. Warn-only diagnostics are the narrowest safe fix for the silent failure, while extraction from fences needs explicit behavior approval and regression coverage.
