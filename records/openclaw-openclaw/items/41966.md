## Summary

Keep open. Current `main` and `v2026.5.2` still preserve fenced `MEDIA:` lines as visible text before media validation, and the related extraction PR was closed unmerged, so no close reason applies yet.

## What This Changes

The issue asks OpenClaw to make `MEDIA:` directives inside Markdown code fences either deliver media, warn visibly, or be documented as unsupported.

## Best Possible Solution

Define the fenced-`MEDIA:` contract, then either extract valid own-line fenced directives through the existing validation path with tests/docs, or preserve the skip behavior while adding observable warning and prompt/docs guidance.

## Reproduction Assessment

Yes. Source inspection on current `main` shows fenced lines are skipped before `MEDIA:` matching, and the parser test explicitly asserts that a fenced `MEDIA:` line remains visible text.

## Solution Assessment

Unclear. Extraction is plausible if tightly constrained, but warning plus documentation may be safer if code fences are intentionally examples rather than delivery metadata.
