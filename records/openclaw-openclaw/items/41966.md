## Summary

Keep open. Current main documents the plain-text `MEDIA:` requirement, but the runtime parser still skips fenced directives without a warning and the open warn-only PR has not landed, so maintainers still need to settle the parser/logging contract.

## What This Changes

The issue asks OpenClaw to stop silently ignoring model-produced `MEDIA:` directives inside Markdown code fences, either by extracting them, warning, or documenting the constraint.

## Best Possible Solution

Land a narrow warn-only diagnostic with focused parser coverage while keeping fenced extraction disabled by default; treat extraction-from-fences or any config flag as a separate product decision.

## Reproduction Assessment

Yes, at source level. Current `splitMediaFromOutput()` skips fenced lines before `MEDIA:` matching, and parser coverage asserts that a fenced directive remains visible text; this read-only review did not run tests.

## Solution Assessment

Unclear until maintainers choose the parser contract. Warn-only diagnostics preserve the current safety boundary, while extraction-from-fences needs an explicit safety decision and regression coverage.
