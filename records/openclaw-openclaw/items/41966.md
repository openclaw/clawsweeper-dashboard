## Summary

Keep open. Current main and the latest release still skip fenced `MEDIA:` lines before token validation, regression coverage asserts that behavior, and the only extraction PR was closed unmerged; the remaining work needs a maintainer parser/docs contract decision.

## What This Changes

The issue asks OpenClaw to make `MEDIA:` directives inside Markdown code fences actionable or at least diagnosable instead of silently leaving them as visible text.

## Best Possible Solution

Define the fenced-`MEDIA:` contract, then update `splitMediaFromOutput()`, prompt/docs guidance, and regression coverage consistently.

## Reproduction Assessment

Yes. Source inspection of current main shows fenced lines are skipped before `MEDIA:` matching, and the parser test asserts that a fenced `MEDIA:` line remains visible text; I did not run tests because this review is read-only.

## Solution Assessment

Unclear. Extraction would address the silent delivery failure, but current tests intentionally protect code examples, so a maintainer should choose extraction, warning plus docs, or a narrow hybrid before implementation.
