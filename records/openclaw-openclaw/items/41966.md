## Summary

Keep open. Current main and v2026.5.2 still skip fenced `MEDIA:` lines before media validation, the parser test explicitly preserves that behavior, and the prompt/docs still describe own-line `MEDIA:` delivery without warning that Markdown fences suppress it. The related extraction PR was closed unmerged, so it remains prior art rather than a shipped fix.

## What This Changes

The issue asks OpenClaw to make `MEDIA:` tokens inside Markdown code fences observable by extracting them, warning about skipped tokens, or documenting that fences suppress delivery.

## Best Possible Solution

Define the intended fenced `MEDIA:` contract, then either extract valid own-line fenced directives through the existing validation path with updated parser tests/docs, or preserve the skip behavior while adding an observable warning plus prompt and public docs guidance.

## Reproduction Assessment

Yes. Source inspection shows fenced lines are skipped before `MEDIA:` matching, and the current parser test asserts that a fenced `MEDIA:` line remains visible text.

## Solution Assessment

Unclear. Extraction is a plausible fix if tightly constrained, but warning plus docs may be safer if code fences are intentionally treated as examples rather than delivery metadata.
