## What This Changes

The issue asks OpenClaw to stop silently ignoring `MEDIA:` directives that a model places inside Markdown code fences, by extracting them, warning about them, or documenting the constraint.

## Best Possible Solution

Keep fenced-code extraction disabled by default, land a narrow warn-only runtime diagnostic with focused parser coverage and real proof, and treat any extraction-from-fences behavior as a separate maintainer-approved contract change.

## Reproduction Assessment

Yes, at source level. Current main skips fenced lines before `MEDIA:` matching, and parser coverage asserts that a fenced directive remains visible text; this read-only review did not run tests.

## Solution Assessment

Unclear until maintainers choose the parser contract. Warn-only diagnostics are the narrowest safe fix for the silent failure, while extraction from fences needs an explicit behavior decision and regression coverage.
