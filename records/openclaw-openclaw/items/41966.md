## Summary

Keep open. Current main and the latest release still skip fenced `MEDIA:` lines, the parser test asserts that behavior, and the only linked extraction PR was closed unmerged; the remaining work needs a maintainer decision on parser behavior versus warning/docs.

## What This Changes

The issue requests making `MEDIA:` directives inside Markdown code fences not fail silently, via extraction, warnings, or documentation.

## Best Possible Solution

Codify the fenced-`MEDIA:` contract, then update `splitMediaFromOutput()`, prompt/docs guidance, and regression coverage consistently with that contract.

## Reproduction Assessment

Yes. Feeding `splitMediaFromOutput()` a fenced `MEDIA:` line is source-reproducible because current main skips fenced spans before token matching, and the parser test encodes that expected output.

## Solution Assessment

Unclear. Extracting fenced tokens would solve the silent delivery failure, but it conflicts with the current tested intent to ignore code examples; warning plus docs may be safer if fenced directives remain unsupported.
