## Summary

Keep open. Current main still skips fenced `MEDIA:` lines before token validation, regression coverage asserts that the fenced token remains visible text, the latest release carries the same behavior, and the prior extraction PR was closed unmerged; the remaining work is a parser/docs contract choice rather than cleanup-close material.

## What This Changes

The issue asks OpenClaw to make fenced `MEDIA:` directives visible by extracting them, warning about them, or documenting that Markdown code fences suppress delivery.

## Best Possible Solution

Codify the fenced-`MEDIA:` contract, then update `splitMediaFromOutput()`, prompt/docs guidance, and regression coverage consistently with that contract.

## Reproduction Assessment

Yes, source-reproducible: current main skips lines inside Markdown fence spans before matching `MEDIA:`, and the parser test expects a fenced `MEDIA:` line to remain visible text. I did not run tests in this read-only review.

## Solution Assessment

Unclear. Extraction would solve the reported silent delivery failure, but it conflicts with the current tested intent to ignore code examples; warning plus docs may be the safer contract if fenced directives remain unsupported.
