## Summary

Keep open. Current main documents the plain-text-only MEDIA contract, but the runtime parser still leaves fenced MEDIA lines as visible text with no warning, and the open warn-only PR has not landed.

## What This Changes

The issue asks OpenClaw to avoid silent media-delivery failures when a MEDIA directive appears inside a Markdown code fence, either by extracting it, warning, or documenting the constraint.

## Best Possible Solution

Keep the current parser safety boundary by default, land a warn-only runtime diagnostic with focused parser coverage, and treat extraction from fences as a separate explicit contract decision.

## Reproduction Assessment

Yes, at source level. Current main skips fenced lines before MEDIA matching, and parser coverage asserts that a fenced directive remains visible text; this read-only review did not run tests.

## Solution Assessment

Unclear until maintainers choose the parser contract. Warn-only diagnostics preserve the current safety boundary, while extraction from fences needs an explicit behavior decision and regression coverage.
