## Summary

Keep open. Current main and v2026.5.19 still skip fenced `MEDIA:` lines before extraction with no warning, the parser tests preserve that behavior, and the open follow-up PRs have not settled the parser/docs contract yet.

## What This Changes

The issue asks OpenClaw to stop silently ignoring fenced `MEDIA:` directives by extracting them, warning, and/or documenting the required plain-line format.

## Best Possible Solution

Settle the parser contract, then land warn-only runtime diagnostics plus docs/prompt clarification, or an explicit extraction policy with focused parser tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the parser contract. Warn-only is the lowest-risk first step already proposed, while extraction-from-fences needs an explicit safety decision and regression coverage.
