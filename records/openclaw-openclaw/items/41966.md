## Summary

Keep open. Current main still skips fenced `MEDIA:` lines before media matching, existing tests preserve that behavior, and the linked warn-only fix PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to stop silently ignoring `MEDIA:` directives when LLMs place them inside Markdown code fences, via extraction, warning logs, or documentation.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80396 with the chosen parser/docs contract, then close this issue after the behavior or guidance is on main.
