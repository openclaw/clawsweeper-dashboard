## Summary

Keep open. Current main and v2026.5.7 still skip fenced `MEDIA:` lines before token validation, the parser test asserts that the fenced token remains visible text, the earlier extraction PR was closed unmerged, and the active related PR addresses indented-code false positives rather than this fenced-token contract.

## What This Changes

The issue asks OpenClaw to avoid silent fenced `MEDIA:` delivery failures by extracting those tokens, warning about them, or documenting that fences suppress extraction.

## Best Possible Solution

Define the fenced-`MEDIA:` contract, then implement the chosen narrow behavior in `splitMediaFromOutput()` with prompt/docs updates and regression coverage.

## Reproduction Assessment

Yes, from source inspection. Current main skips lines inside Markdown fence spans before matching `MEDIA:`, and the parser test expects a fenced `MEDIA:` line to remain visible text.

## Solution Assessment

Unclear. Extraction would solve the reported failure, but it conflicts with the current tested intent to ignore code examples; warning plus documentation may be the safer contract if fenced directives stay unsupported.
