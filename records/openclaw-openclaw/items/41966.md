## Summary

Keep open. Current main and the latest release still skip fenced `MEDIA:` lines before media matching and keep them as visible text, while the prior extraction PR was closed unmerged; because current tests assert the skip behavior, this needs a maintainer contract decision rather than cleanup closure.

## What This Changes

The issue requests extraction, warning, or docs guidance for `MEDIA:` directives that appear inside Markdown code fences.

## Best Possible Solution

Define the fenced-`MEDIA:` contract, then either extract tightly constrained valid own-line fenced directives through the existing validation path or preserve the skip behavior while adding observable warning and docs/prompt guidance.

## Reproduction Assessment

Yes. Source inspection on current main shows fenced lines are skipped before `MEDIA:` matching, and the parser test explicitly asserts that a fenced `MEDIA:` line remains visible text.

## Solution Assessment

Unclear. Extraction would solve the visible failure, but current tests intentionally protect code blocks and the adjacent indented-code PR highlights false-positive risk; a maintainer should choose the contract before implementation.
