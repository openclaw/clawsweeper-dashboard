## Summary

Current main and the latest release still preserve fenced `MEDIA:` directives as visible text without any diagnostic. The linked warn-only PR remains open and needs updating against current main, so this issue remains necessary until maintainers land or reject that direction.

## What This Changes

Make fenced `MEDIA:` directives diagnosable or deliverable instead of silently leaving them as text.

## Best Possible Solution

Keep fenced code inert, update and review the linked warn-only PR, and add concise agent/workspace guidance if maintainers determine logs alone are not discoverable enough.

## Reproduction Assessment

Yes. Current source and an explicit test establish that fenced `MEDIA:` lines remain visible text and are not emitted as attachments; the absence of a warning follows directly from that branch.

## Solution Assessment

The linked warn-only approach is safer than extracting directives from code fences because it preserves inert examples, but maintainers must decide whether a log warning adequately solves the user-facing discoverability problem.
