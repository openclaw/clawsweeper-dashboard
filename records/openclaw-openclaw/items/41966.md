## Summary

Keep open. Current main now documents the plain-text `MEDIA:` requirement, but `splitMediaFromOutput()` still skips fenced `MEDIA:` lines before parsing and emits no runtime warning; the latest release has the same runtime behavior, and https://github.com/openclaw/openclaw/pull/80396 is the active warn-only fix candidate.

## What This Changes

The issue asks OpenClaw to avoid silently losing intended media delivery when an assistant emits a `MEDIA:` directive inside a Markdown code fence.

## Best Possible Solution

Keep the plain-text contract, review or land the warn-only diagnostic with focused parser tests, and treat extraction from fences as a separate explicit product decision.

## Reproduction Assessment

Yes, at source level. Current main skips fenced lines before `MEDIA:` matching, and parser coverage asserts that a fenced `MEDIA:` line remains visible text; this read-only review did not run tests.

## Solution Assessment

Unclear until maintainers choose the parser contract. Warn-only diagnostics plus the already-landed docs/prompt warnings are the lowest-risk immediate fix, while extraction-from-fences needs explicit safety coverage.
