## Summary

Keep this PR open: it addresses a valid source-reproducible subagent session-state bug, but the submitted patch still has a stale-finalization accounting defect and the updated proof does not exercise the real gateway restart/orphan-recovery path.

## What This Changes

The PR removes the `abortedLastRun` stale-run pruning exemption, adds a stale liveness gate to subagent orphan recovery, and updates docs plus persistence coverage for stale aborted subagent runs.

## Best Possible Solution

Use the shared stale-run cutoff in both restore and orphan recovery, but make stale finalization awaited and retry-accounted, then back it with focused orphan-recovery coverage plus redacted real restart proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main exempts `abortedLastRun` from stale restore pruning and orphan recovery has no stale liveness gate before resume. I did not run a live multi-hour gateway restart scenario in this read-only review.

## Solution Assessment

No, not as submitted: the shared liveness cutoff is the right direction, but the stale recovery branch must await and account for finalization failures before it is the best fix. The safer shape is to preserve the existing retry/finalization contract while changing the cutoff behavior.
