## Summary

Keep open. Current main still exposes the noisy heartbeat exec-completion path, and this PR targets the right seam, but the patch should not merge until the SIGTERM suppression boundary is tightened or explicitly accepted, the branch is rebased onto current heartbeat code, and the user-facing fix is recorded in the changelog. https://github.com/openclaw/openclaw/pull/72253

## What This Changes

The PR adds local-exec provenance to async exec completion system events, filters structured code 0 and SIGTERM completions out of heartbeat user relay, JSON-encodes relayable exec details, and adds focused heartbeat/exec tests.

## Best Possible Solution

Land a rebased fix that marks only proven benign local exec completions internal, preserves user relay for actionable signal/code failures, keeps current heartbeat response-tool behavior, and documents the user-facing change.

## Reproduction Assessment

Yes. Source inspection on current main shows output-bearing structured code 0 and SIGTERM exec completions are relayable, and existing tests still assert delivery for an output-bearing successful completion.

## Solution Assessment

No, not yet. The local-exec provenance gate is the right narrow direction, but treating all local structured SIGTERM completions as benign is broader than the reported restart-cleanup case unless cleanup provenance is added or maintainers explicitly accept that boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve relay for actionable SIGTERM failures:** `src/infra/heartbeat-events-filter.ts:12`
  - body: This regex makes every structured `Exec failed (..., signal SIGTERM)` completion internal-only once the runner sees local-exec provenance, but `maybeNotifyOnExit` emits the same shape for any background exec that exits by SIGTERM and currently classifies signaled exits as failures. A user-started background job killed by SIGTERM would lose its completion output; carry cleanup/restart provenance from the producer or limit suppression to a known cleanup exit reason.
