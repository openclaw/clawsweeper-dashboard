## Summary

Keep open: this is a focused fix for real heartbeat chat noise, but the branch is currently conflicting, lacks after-fix real behavior proof, and would silently drop output-bearing successful exec completion notices that current main and docs still treat as relayable.

## What This Changes

The branch tags local exec completion system events and changes heartbeat exec-event filtering so code 0 and SIGTERM completions stay internal while code 1 failures remain relayable.

## Best Possible Solution

Rebase onto current main and implement a narrower classifier that suppresses proven cleanup/no-output noise without dropping captured success output, then attach real chat or Gateway proof.

## Reproduction Assessment

Yes from source inspection, but not from a live run in this read-only review. Current main treats output-bearing SIGTERM and successful structured exec completions as relayable through `isRelayableExecCompletionEvent`, which matches the reported noisy chat path.

## Solution Assessment

No. The proposed direction is plausible for cleanup noise, but the current branch is too broad because it also suppresses ordinary successful completion output; the best fix is narrower and rebased onto the current heartbeat runner.
