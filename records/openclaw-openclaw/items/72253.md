## Summary

Keep open: the PR targets a real noisy heartbeat path, but the current branch is not merge-ready because it broadens suppression beyond the reported cleanup case and can hide currently relayable exec completion messages.

## What This Changes

This PR tags local exec completion system events and changes heartbeat filtering so structured code 0 and SIGTERM completions stay internal while non-SIGTERM failures remain relayable.

## Best Possible Solution

Preserve the current output-aware relay boundary, then suppress only cleanup-proven SIGTERM or no-output benign completion cases with real chat proof.

## Solution Assessment

No. A text-only code 0/SIGTERM classifier is too broad; the safer solution needs output-aware delivery preserved and cleanup provenance before hiding SIGTERM failures.
