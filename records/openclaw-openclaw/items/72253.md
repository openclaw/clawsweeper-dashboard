## Summary

Keep open: the PR targets a real heartbeat exec-delivery bug, but the branch is conflicting, proof is test-only, and the patch would suppress currently relayable output-bearing success and SIGTERM completions without cleanup provenance.

## What This Changes

The PR adds local-exec metadata and internal-only heartbeat filtering for structured code 0 and SIGTERM exec completions, plus regression tests.

## Best Possible Solution

Carry explicit cleanup/internal provenance from the exec producer, suppress only proven cleanup or metadata-only completions, preserve output-bearing success and actionable signal failures, then verify the behavior in a real chat/runtime path.

## Reproduction Assessment

Yes, at source level. Current main can enqueue output-bearing code 0 and SIGTERM exec summaries through `maybeNotifyOnExit`, and the heartbeat predicate treats output-bearing completions and non-success structured completions as relayable.

## Solution Assessment

No. The local-exec origin/context gate improves spoofing resistance, but exit-shape matching is still too broad; the maintainable fix is explicit cleanup/internal provenance before suppressing user delivery.

A user-started command terminated by SIGTERM could lose its completion output; carry cleanup provenance from the producer before suppressing it.
  - confidence: 0.87
