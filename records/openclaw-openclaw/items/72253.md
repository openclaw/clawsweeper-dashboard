## Summary

Keep this PR open, but it is not merge-ready: the focused suppression is useful, current main has not implemented the central behavior, and the same P2 SIGTERM boundary plus missing real behavior proof still block merge.

## What This Changes

The PR adds local-exec provenance to system events, filters structured code 0 and SIGTERM exec completions out of heartbeat user relay, JSON-encodes relayable exec details, and adds focused tests.

## Best Possible Solution

Land a rebased, provenance-backed fix that internalizes successful or known-cleanup completions while preserving user relay for actionable signal/code failures and includes real channel or runtime-log proof.

## Reproduction Assessment

Yes, at source level. On current main, an output-bearing `Exec completed (..., code 0)` or `Exec failed (..., signal SIGTERM)` event reaches the relayable-exec heartbeat path; I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not yet. The local-exec provenance gate is a good narrow direction, but suppressing every structured local SIGTERM by text is broader than the reported restart-cleanup case unless cleanup provenance is added or maintainers explicitly accept that boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84
