## Summary

Keep this PR open, but it should not merge as-is: the reported heartbeat exec-noise problem is real, yet the patch suppresses by broad `code 0` and `SIGTERM` shapes rather than by cleanup/internal provenance, which would drop user-relevant exec output that current main deliberately preserves.

## What This Changes

The PR adds local-exec origin/context metadata, filters structured code 0 and SIGTERM exec completions from heartbeat user relay, JSON-encodes remaining exec details, and adds focused heartbeat/exec tests.

## Best Possible Solution

Carry explicit cleanup or internal-audience provenance from the exec producer, suppress only proven-benign completions, preserve output-bearing success and actionable signal failures, and verify the delivery behavior in a real chat/runtime path.

## Reproduction Assessment

Yes, at source level. Current main can enqueue output-bearing `Exec completed (..., code 0)` or `Exec failed (..., signal SIGTERM)` events through `maybeNotifyOnExit`, and the heartbeat filter/runner path treats output-bearing structured completions as relayable.

## Solution Assessment

No. The local-exec source gate is useful, but the PR uses broad exit-shape matching as policy; the safer fix is to carry cleanup/internal provenance and keep output-bearing success or actionable signal completions relayable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve relay for output-bearing success completions:** `src/infra/heartbeat-events-filter.ts:7-8`
  - body: This classifies every local `Exec completed (..., code 0) :: <output>` as internal-only. Current main deliberately treats structured completions with captured output as relayable and has regression coverage that sends that output back to the original chat target, so successful background commands with meaningful `notifyOnExit` output would now be dropped. Restrict suppression to metadata-only successes or events with explicit internal provenance.
  - confidence: 0.88
- **[P2] Preserve relay for actionable SIGTERM failures:** `src/infra/heartbeat-events-filter.ts:9-10`
  - body: This treats every structured local `Exec failed (..., signal SIGTERM)` completion as benign cleanup, but `maybeNotifyOnExit` emits the same shape for any background process killed by SIGTERM. A user-started command terminated by SIGTERM would lose its completion output; carry cleanup/restart provenance from the producer or keep signaled failures relayable unless the event is known cleanup.
  - confidence: 0.87
