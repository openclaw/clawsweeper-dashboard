## Summary

Keep open. Current main still treats output-bearing successful and SIGTERM exec completions as relayable heartbeat events, so this PR is still addressing a real user-visible noise path; however the branch is reported unmergeable against current main and still needs the required changelog entry plus maintainer confirmation of the SIGTERM suppression boundary.

## What This Changes

The PR adds local-exec provenance to queued exec completion events, uses that provenance to keep structured `code 0` and `signal SIGTERM` completions internal during heartbeat delivery, JSON-encodes relayable exec prompt data, and adds focused regression tests.

## Best Possible Solution

Land a rebased narrow heartbeat fix that marks only trusted benign local exec completions internal, preserves relay for actionable failures, records the user-facing fix in `CHANGELOG.md`, and leaves the broader event-audience model to #69492 unless maintainers choose to fold this slice into that design.

Yes. ## Solution Assessment

Unclear as submitted. The local-exec provenance gate is the right narrow seam, but the branch needs a current-main rebase and maintainer confirmation that all trusted local SIGTERM completions should be internal before this is the best final fix.
