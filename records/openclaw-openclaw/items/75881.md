## Summary

## What This Changes

The PR adds `openclaw browser start --headed`, browser client/tool `headless` override plumbing, node-proxy forwarding, and focused tests for headed/headless start behavior.

## Best Possible Solution

Land the implementation after adding public docs for the headed handoff flow and a single-line Unreleased changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the internal `/start` route already accepts `headless=false`, while the public CLI, client helper, and browser tool cannot currently send that value.

## Solution Assessment

Yes. Reusing the existing request-local `/start?headless=` contract is the narrowest maintainable solution; the submitted PR is incomplete only in docs and changelog coverage.
