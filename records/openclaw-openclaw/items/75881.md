## Summary

Keep open. Current main still lacks the public headed browser start override, and this PR is a narrow implementation candidate, but it needs docs and changelog follow-through before merge.

## What This Changes

The PR adds `openclaw browser start --headed`, forwards `headless=false` through browser client/tool start paths including node proxy requests, and adds schema plus focused tests.

## Best Possible Solution

Land the runtime implementation after adding public docs for the headed handoff flow and a single-line Unreleased changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the public CLI, client, and browser tool cannot send `headless=false`, while the internal `/start` route already accepts that request-local value.

## Solution Assessment

Yes. Reusing the existing `/start?headless=` request-local contract is the narrowest maintainable solution; the submitted PR is incomplete only in docs and changelog coverage.
