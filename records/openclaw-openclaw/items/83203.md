## Summary

Keep open: this PR targets a real Gateway SSE compatibility bug, but the branch is conflicting and not safe as written against current main because it drops the current usage/tool-call-aware stream finalizer shape and lacks after-fix real behavior proof.

## What This Changes

The PR splits streamed `/v1/chat/completions` assistant content chunks from terminal `finish_reason: "stop"` chunks and updates Gateway SSE tests.

## Best Possible Solution

Rework the split on current main by preserving `maybeFinalize()`, parameterized terminal finish reasons, usage chunk ordering, tool-call streaming, and focused Gateway coverage for the generic error path.

## Reproduction Assessment

Yes, source-level: current main's generic streaming catch path writes `Error: internal error` with `finishReason: "stop"` in the same content chunk, and the current test expects that shape. I did not run a live Gateway request in this read-only review.

## Solution Assessment

No, not as written. The fix should be rebased onto current main's finalizer and keep non-stop finish reasons rather than hard-coding a direct stop-only close path.
