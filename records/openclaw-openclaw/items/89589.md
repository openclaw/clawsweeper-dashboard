## Summary

Keep open: current main and the latest release still route the reported update-check state write through a JSON writer that can chmod the parent directory to an umask-derived mode, and the linked open PR does not currently cover that exact writer.

## What This Changes

The issue asks OpenClaw to stop update-check JSON writes from relaxing a hardened state directory back to group-writable permissions under permissive umask settings.

## Best Possible Solution

Cover the update-check state write with a private parent-directory mode, add a regression test for that path under permissive umask, refresh current-head proof on the linked PR or a replacement, and close this issue only after merge.

## Reproduction Assessment

Yes. Source inspection shows `update-check.json` is written under `resolveStateDir()` through `writeJson` without `dirMode`, while the pinned writer path can chmod the parent to an umask-derived mode.

## Solution Assessment

Yes for the issue direction: preserving private state-root permissions on the update-check write is the narrow maintainable fix. The current linked PR is not sufficient until it covers that exact writer and refreshes proof.
