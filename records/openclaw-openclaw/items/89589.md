## What This Changes

The issue reports that update-check JSON writes can relax the OpenClaw state directory from 0700 back to 0775 under umask 0002.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/89635 or a narrower replacement that keeps private state-root writes at 0700 with regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows update-check writes under the state directory through writeJson without dirMode, while the helper and pinned fs-safe dependency can chmod the parent directory to an umask-derived 0775.

## Solution Assessment

Yes, the requested direction is the right fix target. The active PR is the candidate implementation, but maintainers still need to confirm the exact compatibility scope before merge.
