## Summary

Keep open: current main and the latest release still use the umask-derived parent-directory mode on the update-check write path, while the linked fixing PR is open and not merged.

## What This Changes

The issue reports that OpenClaw update-check JSON writes can relax a hardened state directory from 0700 back to 0775 under umask 0002.

## Best Possible Solution

Keep this issue open while https://github.com/openclaw/openclaw/pull/89635 is reviewed; land a private-state permission fix with regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows update-check writes under resolveStateDir() through writeJson without dirMode, while the pinned fs-safe writer derives the parent directory mode from umask and chmods that parent.

## Solution Assessment

Yes, the requested direction is the right fix target, but the active PR still needs the scope question resolved. The safest end state is private state-root writes preserving or enforcing 0700 without unintentionally changing public SDK-visible helper defaults.
