## Summary

## What This Changes

The PR adds a raw `current` shortcut to `resolveCurrentSessionClientAlias` and updates resolver tests so `sessionKey: "current"` resolves to the requester without calling `sessions.resolve`.

## Best Possible Solution

Preserve literal `current` key/sessionId precedence while adding a narrow semantic-current no-round-trip path, then prove it with the affected agent-tool tests and a live gateway/tool check.

## Reproduction Assessment

Yes, source-level: current main still lets raw `current` reach `sessions.resolve` in the wrapper path, and the linked issue includes live logs. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, this is not the best fix as written: the shortcut is in a shared helper used before `session_status` literal lookup. The safer repair preserves literal lookup first and only short-circuits semantic-current fallback probes.
