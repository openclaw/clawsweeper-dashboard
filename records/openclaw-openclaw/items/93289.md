## Best Possible Solution

## Solution Assessment

No: the requested direction is right, but this branch is not the best fix because it bypasses the existing activation-policy gate and has no new regression test. A safer shape is to include the explicit external channel path only before the existing `canStartConfiguredChannelPlugin` decision and add allowlist/deny coverage.
