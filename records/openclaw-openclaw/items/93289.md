## Summary

## What This Changes

The PR adds a non-bundled manifest-channel startup predicate so `plugins.entries.<id>.enabled=true` can route an external channel plugin into the gateway startup plan without a `channels.<id>` block.

## Best Possible Solution

Land one canonical startup-planner fix that uses the shared effective activation contract for non-bundled manifest channel plugins, keeps non-channel plugins lazy, adds focused startup-policy tests, and includes redacted real gateway boot proof.
