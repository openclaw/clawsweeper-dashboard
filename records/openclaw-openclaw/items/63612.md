Keep open. Current main still routes active session history through the pinned dependency token estimator without malformed-message guards before replay normalization, and https://github.com/openclaw/openclaw/pull/63636 remains the open implementation path with closing syntax for this issue.

## What This Changes

The issue requests that malformed long-lived session history stop crashing pre-provider compaction token estimation when missing fields are read with `.length`.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/63636 with a shared guarded estimator or pre-estimation normalization path plus regression coverage for malformed assistant, toolResult/custom, bashExecution, and summary messages.

## Reproduction Assessment

Yes for the source-level failure shape: malformed assistant, toolResult/custom, bashExecution, or summary messages can still reach unchecked token estimation before provider dispatch. The exact reporter session was not replayed because no sanitized damaged transcript is available.

## Solution Assessment

Yes. A shared guarded estimator or normalization before any compaction/pre-prompt estimation is the narrow maintainable fix; the open linked PR is the current review path, but it needs maintainer handling before the issue should close.
