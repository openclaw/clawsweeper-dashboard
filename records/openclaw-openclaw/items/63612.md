## Summary

Keep open. Current main still routes compaction/pre-prompt token estimation through the pinned upstream estimator without malformed-message guards before replay normalization, and https://github.com/openclaw/openclaw/pull/63636 remains the open, conflicting implementation path with closing syntax for this issue.

## What This Changes

Long-lived main sessions can crash before provider dispatch when malformed replay history reaches compaction token estimation and unchecked `.length` reads.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/63636 with a shared guarded estimator or pre-estimation normalization path that preserves current runtime-context stripping, pre-prompt routing contracts, and malformed-history regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main can pass malformed `activeSession.messages` to `estimateMessagesTokens()`/upstream `estimateTokens()` before replay normalization, and the pinned dependency has unchecked `.length` reads. I did not replay the exact reporter transcript because no sanitized damaged session is available.

## Solution Assessment

Yes in direction: a shared guarded estimator or normalization before any compaction/pre-prompt estimation is the narrow maintainable fix. The open linked PR targets that boundary, but it needs review, rebase, or replacement before this issue should close.
