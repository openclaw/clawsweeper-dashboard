## Summary

Keep open. Current main still sends active history through the dependency token estimator before replay normalization, and an open implementation PR already targets this issue with closing syntax.

## What This Changes

The issue reports a long-lived main session crashing before provider dispatch when compaction token estimation reads `.length` from malformed history blocks.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/63636 with a shared guarded estimator or pre-estimation normalization path plus malformed-history regression coverage.

## Reproduction Assessment

Yes for the source-level failure shape: malformed assistant, toolResult/custom, bashExecution, or summary messages can still reach unchecked token estimation before provider dispatch. The exact reporter session was not replayed because no sanitized damaged transcript is available.

## Solution Assessment

Yes. The narrow maintainable fix is to use a shared guarded estimator or normalize history before any compaction/pre-prompt estimation; the open linked PR is the current review path for that fix.
