## Summary

Keep open. Current main still routes compaction and pre-prompt estimation through the pinned upstream estimator without malformed-message guards, and an open implementation PR already targets this issue with closing syntax.

## What This Changes

A long-lived main session can crash before provider dispatch when compaction token estimation reads `.length` on malformed replay-history message blocks.

## Best Possible Solution

Provide one OpenClaw-owned safe token-estimation adapter, or normalize all persisted history before every compaction/precheck estimate, so malformed blocks cannot crash while valid messages continue to use upstream-equivalent estimates with focused regression coverage.

## Reproduction Assessment

Yes for the source-level failure shape: a malformed assistant, toolResult/custom, bashExecution, or summary message passed through `estimateMessagesTokens()` or `shouldPreemptivelyCompactBeforePrompt()` can still reach unchecked upstream `.length` reads. The exact long-lived reporter session is unavailable, so the full session replay is not high-confidence.

## Solution Assessment

Yes, the narrow maintainable direction is a shared guarded estimator or equivalent pre-estimation normalization used by both compaction and preemptive compaction. Since PR #63636 already implements that direction with closing syntax, the best next step is review, landing, or superseding that PR.
