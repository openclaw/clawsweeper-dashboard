## Summary

Keep open. Current main still routes pre-prompt compaction token estimation through the pinned upstream estimator without malformed-message guards, and open PR #63636 already targets this issue with closing syntax.

## What This Changes

The issue reports that malformed long-lived session history can crash pre-prompt compaction token estimation before provider dispatch.

## Best Possible Solution

Land or supersede #63636 with one shared guarded estimator or equivalent pre-estimation normalization used by all compaction and pre-prompt estimation paths, with focused malformed-history regression coverage.

## Reproduction Assessment

Yes for the source-level failure shape: malformed assistant, toolResult/custom, bashExecution, or summary messages can still reach unchecked upstream token estimation before provider dispatch. The exact long-lived reporter session is unavailable, so I did not replay the full damaged session.

## Solution Assessment

Yes, the narrow maintainable fix is a shared guarded estimator or equivalent normalization before any estimator call. Because #63636 already implements that direction with closing syntax, the next step is review, landing, or superseding that PR.
