## Summary

Keep open. Current main still exposes the reported pre-provider crash path: compaction token estimation delegates to the upstream estimator without malformed-message guards, replay normalization still happens after the pre-prompt check, and an open implementation PR already targets this issue with closing syntax.

## What This Changes

The issue reports that malformed long-lived session history can make pre-prompt compaction token estimation throw before provider dispatch.

## Best Possible Solution

Land or supersede the open guarded-estimator PR with a shared safe token-estimation path used by compaction and preemptive compaction, plus focused regression tests for malformed assistant, toolResult/custom, bashExecution, and summary messages.

## Reproduction Assessment

Yes for the estimator crash path: a malformed assistant/toolResult/bashExecution/summary message passed through estimateMessagesTokens or shouldPreemptivelyCompactBeforePrompt can still hit unchecked upstream .length reads on current main. The exact damaged long-lived session is not available, so only the source-level failure shape is high-confidence.

## Solution Assessment

Yes, the narrow maintainable direction is a shared guarded estimator or equivalent normalization before every compaction/pre-prompt estimate. Because PR #63636 already implements that direction with closing syntax, the next step is review, landing, or superseding that PR rather than opening another repair lane.
