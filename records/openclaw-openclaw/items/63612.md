## Summary

Keep open: current main still sends active history through the upstream token estimator before replay normalization, and the live implementation PR at https://github.com/openclaw/openclaw/pull/63636 is still open with closing syntax for this issue.

## What This Changes

The issue requests hardening compaction token estimation so malformed long-lived session history cannot crash before provider dispatch.

## Best Possible Solution

Use one shared guarded estimator, or normalize all history before any compaction/pre-prompt estimation, and cover malformed assistant, toolResult/custom, bashExecution, and summary messages with regression tests.

## Reproduction Assessment

Yes, for the core failure path by source construction: malformed history can reach `estimatePrePromptTokens()` and the pinned upstream `estimateTokens()` before replay normalization. The original damaged main-session transcript was not available for an end-to-end replay.

## Solution Assessment

Yes. The narrow maintainable fix is the shared guarded-estimator direction in https://github.com/openclaw/openclaw/pull/63636, or an equivalent current-main replacement, rather than closing the issue before that implementation lands.
