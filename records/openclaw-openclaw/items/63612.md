## Summary

Keep open. Current main still sends active session history through the pinned upstream compaction token estimator before replay normalization, the pinned estimator still has unchecked `.length` reads, and the open closing PR remains unmerged and conflicting.

## What This Changes

The issue reports that long-lived main sessions can crash before provider dispatch when compaction token estimation reads `.length` on malformed replayed history blocks.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/63636 with a shared guarded estimator or pre-estimation normalization path plus malformed-history regression coverage and redacted real-run proof.

## Reproduction Assessment

Yes, by source inspection: malformed active history can reach `estimateMessagesTokens()` before replay normalization, and the pinned upstream estimator still has unchecked `.length` reads. I did not replay the exact damaged transcript because no sanitized session data is available.

## Solution Assessment

Yes in direction: guarding token estimation or normalizing history before any compaction/pre-prompt estimate is the narrow maintainable fix. The linked PR targets that boundary but needs rebase, proof, or replacement before this issue should close.
