## Best Possible Solution

Keep the shipped Telegram startup fix and direct users still on 2026.4.24 to update to v2026.5.3 or newer; a remaining update failure should be filed separately with updater logs.

No current-main failure remains to reproduce. The old retry-loop path is represented by source inspection and regression coverage that forces deleteWebhook timeout and confirms polling starts.

## Solution Assessment

Yes. The shipped fix removes the pre-poll retry blocker without adding a new config knob, while preserving getUpdates conflict handling for an actually active webhook.
