## Summary

Keep open: the ledger-backed replay direction is useful and the latest head adds canonical session-key handling, but `loadSession` can still trust a complete ledger after a Gateway disconnect/reconcile path that may have missed chat/tool stream events.

## What This Changes

Adds an ACP event ledger with in-memory/file-backed storage, ledger-first `loadSession` replay, canonical session-key reconciliation, tests, docs, and changelog coverage.

## Best Possible Solution

Keep the ledger-backed replay approach, but invalidate ledger completeness whenever the bridge may have missed Gateway stream events so `loadSession` falls back to the Gateway transcript.

## Reproduction Assessment

Not applicable to the feature as a user bug; yes for the PR-introduced regression path by source inspection. A send accepted before a Gateway disconnect can finish through `agent.wait` while the bridge records no missed chat/tool events, then `loadSession` trusts the complete ledger.

## Solution Assessment

No. Ledger-first replay is a maintainable direction, but this PR should first mark affected ledgers incomplete or force transcript fallback after stream gaps.

## Review Findings
