## What This Changes

The branch changes Feishu's module-load User-Agent interceptor registration to use `defaultHttpInstance.interceptors.request.use(...)` and adds regression tests that fail on direct axios `handlers` access.

## Best Possible Solution

Land the public-interceptor implementation after normal CI or maintainer validation, keeping the focused regression coverage that prevents future private `handlers` access.

## Reproduction Assessment

Yes. Source inspection of current main shows import-time direct assignment to `interceptors.request.handlers`, and the PR's mocked interceptor path would throw on that access before the production change.

## Solution Assessment

Yes. Registering a later public request interceptor that always sets the Feishu User-Agent preserves the SDK interceptor stack and avoids the axios internal `handlers` property while staying inside the Feishu plugin boundary.
