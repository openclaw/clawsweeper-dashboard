## Summary

Keep open, but do not merge as-is: the branch records recovery checkpoints without wiring recovered bundles into the real recreated-session prompt paths, and the persisted free-text path still needs shared redaction, non-blocking writes, and real behavior proof.

## What This Changes

The PR adds session recovery prompt guidance, local task-ledger/recovery-bundle persistence, and checkpoint writes from update_plan, sessions_yield, and sessions_spawn.

## Best Possible Solution

Define an end-to-end passive recovery contract, then wire scoped bundle loading into real recreated-session prompts with shared redaction, asynchronous best-effort persistence, focused tests, and explicit user confirmation semantics.

## Reproduction Assessment

Yes. Static source inspection is a high-confidence path: the PR writes checkpoint state but does not load or pass recovered bundles through the embedded or CLI prompt builders, and the redaction plus synchronous I/O issues are visible in the added module.

## Solution Assessment

No. The direction is useful, but this implementation records recovery data before it can reach real prompt creation and uses bespoke blocking persistence; the safer path is a maintainer-approved recovery contract with shared redaction and async writes.
