## Summary

Keep open: this is a useful session-continuation prototype, but it is not merge-ready because it adds new persistent recovery state outside the canonical SQLite owner, does not prove a production path that loads recovered bundles into recreated sessions, and still lacks real behavior proof.

## What This Changes

The PR adds session recovery prompt discipline, JSONL/JSON recovery checkpoint persistence, and passive checkpoint writes from `update_plan`, `sessions_yield`, and `sessions_spawn`.

## Best Possible Solution

Define one canonical checkpoint/recovery path that uses OpenClaw's SQLite-backed state owner, shared redaction, non-blocking or transaction-safe writes, real recovered-bundle prompt loading, and explicit no-approval-inheritance semantics aligned with the native checkpoint umbrella.
