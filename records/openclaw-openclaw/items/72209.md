## Summary

Keep open: the direction is useful, but the PR still lacks end-to-end recovery wiring, has persisted-text redaction and hot-path synchronous I/O issues, and needs real behavior proof before merge.

## What This Changes

The PR adds session recovery prompt discipline, local task-ledger/recovery-bundle persistence, and checkpoint writes from `update_plan`, `sessions_yield`, and `sessions_spawn` with focused tests.

## Best Possible Solution

Define an end-to-end passive recovery contract, wire scoped bundle loading into real recreated-session prompt builders, reuse shared redaction, move persistence off hot paths, and prove the behavior in a real recreated session.

## Reproduction Assessment

Yes. Static source inspection is a high-confidence path: the PR writes recovery state and adds a leaf prompt input, but the live embedded/CLI prompt builders do not load or pass recovered bundles, and the redaction plus synchronous I/O behaviors are directly visible.

## Solution Assessment

No. The direction is useful, but the current branch records recovery data before it can reach real recreated-session prompts and should be narrowed around a maintainer-approved recovery contract with shared redaction and non-blocking persistence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire recovered bundles into live prompt creation:** `src/agents/system-prompt.ts:502`
  - body: This only adds `sessionRecovery` to the leaf prompt builder. The embedded and CLI prompt builders do not accept or load a recovery bundle, so persisted checkpoints cannot reach recreated-session prompts and the advertised recovery context remains unavailable.
  - confidence: 0.93
- **[P2] Use shared redaction for persisted recovery text:** `src/agents/session-recovery-state.ts:115`
  - body: The local secret regex both redacts ordinary recovery text like `authorization: waiting for review` and misses common secret forms covered by the shared logging redaction path. Since this text is persisted, reuse the shared redaction helper or tighten this to high-confidence secret shapes.
  - confidence: 0.86
- **[P2] Move checkpoint writes off the tool hot path:** `src/agents/session-recovery-state.ts:205-206`
  - body: Every enabled plan update, yield, and accepted spawn can call `appendTaskLedgerEvent`, which performs synchronous directory creation and append before the tool returns. Best-effort checkpoints should be queued or written asynchronously so slow state storage does not block agent progress.
  - confidence: 0.88
