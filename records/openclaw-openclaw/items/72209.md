  - body: `buildSessionRecoveryPromptSection` only renders `params.sessionRecovery`, but this PR only writes bundles; no production caller loads a saved bundle and passes it into `buildAgentSystemPrompt`.   - confidence: 0.86
- **[P2] Make checkpoint writes non-blocking on tool paths:** `src/agents/session-recovery-state.ts:205-206`
  - body: Every enabled checkpoint path calls `recordSessionRecoveryCheckpoint`, which performs synchronous `mkdirSync` and `appendFileSync` before returning. Because these writes run from `update_plan`, `sessions_yield`, and accepted spawn handling even though checkpoints are best-effort, a slow state directory can stall the active agent turn.
  - confidence: 0.87
- **[P2] Reuse the shared secret redaction contract:** `src/agents/session-recovery-state.ts:112-115`
  - body: Recovery summaries and source refs are free text, but `SECRET_PATTERNS` is a local partial redactor instead of OpenClaw's shared `redactSecrets`/`redactToolPayloadText` behavior. That leaves structured secret shapes and operator-configured redaction out of the persistence path.
