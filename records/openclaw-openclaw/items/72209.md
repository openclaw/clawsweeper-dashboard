## Summary

Keep open. The PR is coherent, but it is not merge-ready because it adds new recovery persistence outside the canonical state boundary, does not wire saved bundles into production prompt construction, lacks real behavior proof, and overlaps the broader checkpoint design still tracked in https://github.com/openclaw/openclaw/issues/17211.

## What This Changes

The branch adds passive recovery checkpoint writes for `update_plan`, `sessions_yield`, and `sessions_spawn`, plus recovery prompt discipline, JSONL/JSON state files, and focused tests.

## Best Possible Solution

Define one canonical checkpoint/recovery path using SQLite-backed state, shared redaction, async or transaction-safe writes, production prompt loading, and maintainer alignment with https://github.com/openclaw/openclaw/issues/17211.
