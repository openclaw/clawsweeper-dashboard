## What This Changes

This PR adds session recovery prompt guidance, local task ledger/recovery bundle persistence, and checkpoint writes for update_plan, sessions_yield, and sessions_spawn.

## Best Possible Solution

Define passive recovery end to end: write scoped checkpoints, load scoped bundles into real prompt builders, use shared redaction, make writes async best-effort, and prove recreated-session behavior.

## Solution Assessment

No. The current patch is a useful direction, but the maintainable solution needs a complete recovery-loading contract, shared redaction, non-blocking persistence, and real recreated-session proof before merge.

  - confidence: 0.88
