Full review comments:

- **[P1] Move recovery persistence into canonical state storage:** `src/agents/session-recovery-state.ts:205-206`
  - body: `appendTaskLedgerEvent` creates a JSONL ledger under the state directory, and the same module writes per-task JSON bundles. This is OpenClaw-owned runtime checkpoint state, so it should use the canonical SQLite-backed state owner with migration and cleanup behavior instead of sidecar files.
  - confidence: 0.91
- **[P1] Load saved recovery bundles in production prompts:** `src/agents/system-prompt.ts:698-700`
  - body: The new prompt section only renders `params.sessionRecovery`, but the PR does not add a production caller that loads a saved recovery bundle and passes it into prompt construction. Automatically recreated sessions therefore would not receive the recovered context this feature promises.
  - confidence: 0.88
- **[P1] Use shared redaction before persisting recovery text:** `src/agents/session-recovery-state.ts:112-115`
  - body: Recovery summaries, source refs, and bundle items are free text from agent/tool paths, but this local `SECRET_PATTERNS` list does not use OpenClaw's shared structured/config-aware redaction helpers. Supported secret shapes or operator redact patterns can be missed before persistence.
  - confidence: 0.86
- **[P2] Keep checkpoint writes off active tool paths:** `src/agents/session-recovery-state.ts:205-206`
  - body: Every enabled checkpoint reaches `recordSessionRecoveryCheckpoint`, which performs `mkdirSync` and `appendFileSync` before the tool result returns. Because checkpoints are best-effort, slow state storage should not block `update_plan`, `sessions_yield`, or accepted spawn handling.
  - confidence: 0.87
