## Summary

Keep open: current main still lacks passive session recovery checkpoints, and this PR is useful but not merge-ready because the recovered bundle is not wired into live prompt creation and the new persistence path still has redaction, hot-path I/O, and changelog blockers.

## What This Changes

This PR adds session recovery prompt guidance, local Task Ledger/Recovery Bundle persistence, and checkpoint writes from `update_plan`, `sessions_yield`, and `sessions_spawn`.

## Best Possible Solution

Complete passive recovery end to end: load scoped bundles for recreated sessions, pass them to embedded/CLI prompt creation as informational-only context, keep approvals expired, make persistence non-blocking with tighter redaction, and add tests plus a changelog entry.

## Reproduction Assessment

Yes. Static inspection is high-confidence here: the PR writes checkpoint state, but the live embedded/CLI prompt builders do not load or pass recovered bundles, and the redaction/I/O issues are directly visible in the added module.

## Solution Assessment

No. The direction is plausible, but recording checkpoints before recovered data reaches real recreated-session prompts leaves the user-facing recovery behavior incomplete; the safer fix includes live loading, non-blocking persistence, narrower redaction, and changelog coverage in one coherent contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire recovered bundles into real prompt creation:** `src/agents/system-prompt.ts:502`
  - body: The PR adds `sessionRecovery` to `buildAgentSystemPrompt`, but the production embedded/CLI prompt builders do not accept a recovery bundle and no recreated-session caller loads `loadRecoveryBundle`. Checkpoints can be written but never reach live prompts, so the advertised recovery context is unavailable.
  - confidence: 0.93
- **[P2] Tighten redaction to high-confidence secrets:** `src/agents/session-recovery-state.ts:115`
  - body: The bare `token|secret|password|authorization` branch redacts ordinary recovery text such as `authorization: waiting for review`, so saved summaries can lose the context this feature needs to restore. Require stronger value shapes or keep only high-confidence secret forms.
  - confidence: 0.86
- **[P2] Move checkpoint writes off the tool hot path:** `src/agents/session-recovery-state.ts:205-206`
  - body: Every enabled checkpoint reaches `appendTaskLedgerEvent`, which uses synchronous `mkdirSync` and `appendFileSync` before the tool returns. Since this is best-effort state, queue the write or use async persistence so slow state directories do not block normal agent progress.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/agents/session-recovery.ts:64-70`
  - body: This adds visible session recovery behavior and persisted local task context, but the PR does not modify `CHANGELOG.md`. OpenClaw policy requires a user-facing feature entry before merge.
  - confidence: 0.78
