- **[P1] Persist recovery checkpoints in SQLite:** `src/agents/session-recovery-state.ts:106-108`
  - body: This introduces OpenClaw-owned runtime checkpoint state as `session-recovery/task-ledger.jsonl` plus per-task JSON bundles under the state dir.   - confidence: 0.92
- **[P1] Wire recovery bundles into prompt callers:** `src/agents/system-prompt.ts:502`
  - body: The PR writes recovery bundles and renders them only when `buildAgentSystemPrompt` receives `sessionRecovery`, but real CLI and embedded prompt builders never load a bundle or pass that value. Recreated sessions therefore cannot consume the persisted recovery context this PR creates.
  - confidence: 0.93
- **[P2] Use shared redaction before persisting recovery text:** `src/agents/session-recovery-state.ts:112-116`
  - body: Recovery summaries and bundles are persisted through a local regex list instead of the shared redaction helpers. That can miss supported secret shapes already covered elsewhere and makes recovery persistence a separate privacy contract.
  - confidence: 0.88
- **[P2] Move checkpoint writes off the tool hot path:** `src/agents/session-recovery-state.ts:205-206`
  - body: `appendTaskLedgerEvent` performs synchronous mkdir and append work before update_plan, sessions_yield, or sessions_spawn returns. These checkpoints are best-effort, so slow state storage should not block normal tool execution.
  - confidence: 0.88
