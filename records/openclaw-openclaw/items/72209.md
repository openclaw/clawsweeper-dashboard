## Summary

Keep open. The PR is a useful recovery direction but is not merge-ready: it writes recovery bundles without wiring them into real prompt creation, persists free text through bespoke redaction, performs synchronous hot-path writes, lacks contributor real behavior proof, and is currently conflicting with main.

## What This Changes

Adds session-recovery prompt discipline, local task ledger/recovery bundle persistence, and checkpoint writes from update_plan, sessions_yield, and sessions_spawn.

## Best Possible Solution

Define an end-to-end passive recovery contract, wire scoped bundle loading into recreated CLI and embedded prompts, use shared redaction, make persistence asynchronous best-effort, and require focused tests plus real behavior proof.

## Reproduction Assessment

Yes for the PR review findings. Static PR-head inspection shows bundles are written but not loaded by real prompt callers, and the redaction plus synchronous I/O issues are visible in the added recovery-state module.

## Solution Assessment

No. The direction is useful, but the current patch records recovery data before it can reach recreated sessions and uses bespoke blocking persistence; the safer path is a maintainer-approved recovery contract with shared redaction and async writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire recovery bundles into prompt callers:** `src/agents/system-prompt.ts:502`
  - body: The PR writes bundles and only renders recovery context when buildAgentSystemPrompt receives sessionRecovery, but the CLI and embedded prompt builders never load a bundle or pass that value. Recreated sessions therefore cannot consume the persisted recovery context this PR creates.
  - confidence: 0.93
- **[P2] Use shared redaction before persisting recovery text:** `src/agents/session-recovery-state.ts:115`
  - body: Recovery summaries and bundles are persisted through a local regex list that can redact ordinary prose while missing structured secret forms already covered by the shared redaction helpers. Persisting this free text should reuse the shared redaction path or equivalent key-aware coverage.
  - confidence: 0.86
- **[P2] Move checkpoint writes off the tool hot path:** `src/agents/session-recovery-state.ts:205-206`
  - body: appendTaskLedgerEvent performs mkdirSync and appendFileSync before update_plan, sessions_yield, or sessions_spawn returns. Since these checkpoints are best-effort, slow state storage should not block normal tool execution.
  - confidence: 0.88
