## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use canonical SQLite storage for recovery checkpoints:** `src/agents/session-recovery-state.ts:108`
  - body: This adds `task-ledger.jsonl` and bundle JSON files under the OpenClaw state dir for runtime session recovery data. OpenClaw's current storage contract says runtime state/checkpoints belong in SQLite-owned state, plugin KV, the agent DB, or a designed SQLite schema, not new JSON/JSONL sidecars; otherwise session recovery can drift from the ongoing session/transcript storage migration and has no clear migration or doctor owner.
  - confidence: 0.9
- **[P2] Load bundles before rendering recovered candidates:** `src/agents/system-prompt.ts:502`
  - body: The branch writes recovery bundles and adds a `sessionRecovery` prompt parameter, but the runtime never loads the bundle and passes it into recreated sessions; `loadRecoveryBundle` and `buildSessionRecoveryPromptInputFromBundle` are only exercised by the new tests. As a result, an automatically recreated session gets generic recovery discipline but not the saved candidate context the PR is meant to restore.
  - confidence: 0.86
- **[P2] Make checkpoint writes non-blocking on tool paths:** `src/agents/session-recovery-state.ts:205-206`
  - body: `recordSessionRecoveryCheckpoint` eventually calls `mkdirSync` and `appendFileSync`, and the PR invokes it from `update_plan`, `sessions_yield`, and `sessions_spawn`. Those tools run on the agent turn path, so slow state storage can block the event loop even though checkpoints are described as best-effort.
  - confidence: 0.84
- **[P2] Tighten free-text secret redaction keywords:** `src/agents/session-recovery-state.ts:115`
  - body: The broad `token|secret|authorization|password` pattern runs over free-text plan/yield summaries, so normal text like `authorization: waiting for review` or `secret management` can be silently rewritten in the saved recovery context. Use more specific credential shapes or structured sensitive fields so recovery does not corrupt the task state it is trying to preserve.
  - confidence: 0.78
