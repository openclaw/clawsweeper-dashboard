## Summary

Keep open. The PR is a useful direction, but it is not already implemented on current main and the branch still does not deliver end-to-end passive recovery: it writes checkpoint state without wiring recovered bundles into real recreated-session prompt paths, uses local secret redaction before persistence, and performs synchronous disk writes in tool execution paths.

## What This Changes

Adds session-recovery prompt text, local task-ledger/recovery-bundle persistence, and checkpoint writes from update_plan, sessions_yield, and sessions_spawn.

## Best Possible Solution

Define an end-to-end passive recovery contract, then wire scoped bundle loading into real recreated-session prompts with shared redaction, asynchronous best-effort persistence, focused tests, and explicit user-confirmation semantics.

## Reproduction Assessment

Yes for the review findings. Static PR-head inspection shows recovery bundles are written but not loaded into CLI or embedded prompt creation, and the redaction plus synchronous I/O issues are visible in the added recovery-state module.

## Solution Assessment

No. The direction is useful, but the current patch records recovery data before it can reach real prompt creation and uses bespoke blocking persistence; the safer path is a maintainer-approved recovery contract with shared redaction and async writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire recovered bundles into real prompt creation:** `src/agents/system-prompt.ts:502`
  - body: The patch only adds a sessionRecovery parameter to buildAgentSystemPrompt. The CLI and embedded prompt builders never call loadRecoveryBundle or pass sessionRecovery, so persisted checkpoints cannot appear in automatically recreated sessions.
  - confidence: 0.93
- **[P2] Use shared redaction before persisting recovery text:** `src/agents/session-recovery-state.ts:115`
  - body: Recovery summaries are persisted through a local regex list that can redact ordinary prose while missing structured secret forms already covered by the shared redaction helpers. Use the shared redaction path or equivalent key-aware coverage before writing bundles or ledger events.
  - confidence: 0.86
- **[P2] Move checkpoint writes off the tool hot path:** `src/agents/session-recovery-state.ts:205-206`
  - body: appendTaskLedgerEvent performs mkdirSync and appendFileSync before tool execution returns. Since these checkpoints are best-effort, queue or perform the writes asynchronously so slow state storage does not block update_plan, sessions_yield, or sessions_spawn.
  - confidence: 0.88
