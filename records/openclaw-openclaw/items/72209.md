## Summary

Keep open. The direction is useful, but the patch is not merge-ready: the recorded recovery bundle is not wired into real recreated-session prompt construction, persistence uses a bespoke weaker redaction path, and checkpoint writes block the tool path.

## What This Changes

The PR adds session recovery prompt guidance, local task-ledger/recovery-bundle persistence, and passive checkpoint writes from update_plan, sessions_yield, and sessions_spawn.

## Best Possible Solution

Define an end-to-end passive recovery contract, wire scoped bundle loading into real recreated-session prompts, use shared redaction and asynchronous best-effort persistence, and add focused tests plus real behavior proof.

## Reproduction Assessment

Yes. Static source inspection is a high-confidence path: the PR writes recovery state but no real embedded or CLI prompt builder loads and forwards a recovered bundle, and the redaction and synchronous I/O issues are visible in the added module.

## Solution Assessment

No. The approach is directionally useful, but the current patch records recovery data before it can reach recreated-session prompts and uses bespoke blocking persistence; a maintainer-approved contract with shared redaction and async writes is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire recovered bundles into real prompt creation:** `src/agents/system-prompt.ts:502`
  - body: This adds sessionRecovery to buildAgentSystemPrompt, but the embedded and CLI prompt builders still have no recovery parameter or loadRecoveryBundle path. As a result, persisted checkpoints cannot appear in recreated sessions, so the advertised recovered context is unavailable.
  - confidence: 0.93
- **[P2] Use shared redaction before persisting recovery text:** `src/agents/session-recovery-state.ts:115`
  - body: The local SECRET_PATTERNS branch can redact ordinary recovery prose like authorization: waiting for review while missing common secret forms already handled by the repo redaction helpers. Since these summaries are persisted, use the shared redaction path or equivalent coverage.
  - confidence: 0.86
- **[P2] Move checkpoint writes off the tool hot path:** `src/agents/session-recovery-state.ts:205-206`
  - body: Every enabled plan update, yield, and accepted spawn can reach appendTaskLedgerEvent, which performs mkdirSync and appendFileSync before the tool returns. Best-effort checkpoints should be queued or written asynchronously so slow state storage does not block agent progress.
  - confidence: 0.88
