## Summary

Keep open. Current main still lets a managed macOS LaunchAgent restart enter the Gateway restart drain with a default 300000 ms budget, so the reported multi-minute pre-start gap remains possible despite related startup and launchd hardening.

## What This Changes

The issue asks OpenClaw to investigate and reduce macOS LaunchAgent Gateway restarts that wait about 3-4 minutes between SIGTERM and the next Gateway startup on v2026.4.26.

## Best Possible Solution

Define managed Gateway restart semantics, then separate explicit operator/service-manager restarts from ordinary config/update drains with observable logging and a maintainer-approved timeout or force mode.

## Reproduction Assessment

Yes for the likely mechanism: current source and tests show service-manager SIGTERM restart enters the restart drain and can wait 300000 ms while active work remains. The exact reporter environment was not live-reproduced.

## Solution Assessment

Unclear because there is no proposed patch and the remaining choice is policy-sensitive. The current five-minute bound prevents indefinite hangs, but it still permits managed operator restarts to look stuck.
