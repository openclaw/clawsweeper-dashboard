## Summary

Keep open. Current main has update-specific restart improvements, but macOS LaunchAgent service restarts can still turn SIGTERM into the normal active-work drain with a default 300000 ms budget, so the reported multi-minute pre-start gap remains possible.

## What This Changes

Investigate and reduce macOS LaunchAgent Gateway restarts that can wait several minutes between SIGTERM and the next startup.

## Best Possible Solution

Define the managed Gateway restart contract, then implement a fast and observable operator restart path with explicit graceful-drain versus force-restart semantics and documented timeout controls.

## Reproduction Assessment

Yes for the likely mechanism: current source and tests show a service-manager SIGTERM restart enters the normal restart drain and can wait 300000 ms while active work remains. The exact reporter macOS/plugin log slice was not live-reproduced.

## Solution Assessment

Unclear because there is no proposed patch and the remaining fix is a lifecycle policy choice. The current update.run fast path is appropriate for package swaps, but it does not fully solve managed LaunchAgent restarts that should relaunch promptly.
