## Summary

Keep open. Current main still routes managed Gateway SIGTERM restarts through the active-work restart drain, so a multi-minute pre-start wait remains possible when tasks or embedded runs are active.

## What This Changes

Investigate macOS LaunchAgent Gateway restarts that can wait several minutes between SIGTERM and the next process startup.

## Best Possible Solution

Define managed restart semantics, then separate explicit operator/service-manager restarts from routine config or update deferrals with observable logging and a maintainer-approved timeout or force mode.

## Reproduction Assessment

Yes for the likely mechanism: current source and tests show service-manager SIGTERM restart enters the restart drain and can wait up to 300000 ms while active work remains. The exact reporter environment was not live-reproduced.

## Solution Assessment

Unclear because there is no patch and the remaining choice is policy-sensitive. The current five-minute bound prevents indefinite hangs, but it does not make managed operator restarts prompt.
