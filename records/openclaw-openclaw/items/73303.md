## Summary

Keep open. Current main still lets a managed macOS LaunchAgent restart enter the Gateway restart drain with a default 300000 ms budget, so the reported multi-minute pre-start gap remains possible despite related restart and startup fixes.

## What This Changes

The issue asks maintainers to investigate and fix slow macOS LaunchAgent Gateway restarts where SIGTERM is logged several minutes before the replacement Gateway starts loading config.

## Best Possible Solution

Define managed Gateway restart semantics, then separate explicit operator/service-manager restarts from ordinary config/update drains with observable logging and a maintainer-approved timeout or force mode.

## Reproduction Assessment

Yes for the likely mechanism: current source and tests show service-manager SIGTERM restart enters the restart drain and can wait 300000 ms while active work remains. The reporter's exact macOS/plugin environment was not live-reproduced.

## Solution Assessment

Unclear because there is no proposed patch and the remaining choice is policy-sensitive. The current five-minute bound prevents indefinite hangs, but it does not make managed operator restarts prompt.
