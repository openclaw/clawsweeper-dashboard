## Summary

Keep open. Current main has bounded and more observable restart behavior, but managed Gateway restarts can still wait on the normal active-work drain before the replacement process starts, which matches the reported multi-minute pre-start gap.

## What This Changes

A macOS LaunchAgent Gateway restart can wait minutes between SIGTERM and the next Gateway startup when active work keeps the restart drain open.

## Best Possible Solution

Define the managed Gateway restart contract, then implement a fast and observable operator restart path with explicit graceful-drain versus force-restart semantics and documented timeout controls.

## Reproduction Assessment

Yes for the likely mechanism: current source and tests show a service-manager SIGTERM restart enters the normal restart drain and can wait 300000 ms while active work remains. The exact reporter macOS/plugin log slice was not live-reproduced.

## Solution Assessment

Unclear because there is no proposed patch and the remaining fix is a lifecycle policy choice. The current bounded drain is useful, but it does not fully solve managed LaunchAgent restarts that should relaunch promptly.
