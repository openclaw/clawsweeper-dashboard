## Summary

Keep open. Current main still allows a managed LaunchAgent restart to become a normal Gateway restart drain with a default 300000 ms budget before the old process exits, so the reported multi-minute pre-start gap remains possible.

## What This Changes

The issue reports multi-minute macOS LaunchAgent Gateway restart delays in v2026.4.26 and asks whether shutdown, plugin teardown, or a known regression is responsible.

## Best Possible Solution

Define managed Gateway restart semantics, then implement an observable operator restart path with explicit graceful-drain versus force/short-timeout behavior.

## Reproduction Assessment

Yes for the likely mechanism: current source and tests show service-manager SIGTERM restart enters the restart drain and can wait 300000 ms while active work remains. The reporter's exact macOS/plugin timing was not live-reproduced.

## Solution Assessment

Unclear because there is no proposed patch and the remaining fix is a lifecycle policy choice. The current bounded drain is safer than an unbounded hang, but it does not fully solve prompt managed restarts.
