## Summary

Keep open: current main and the latest release still contain the reported lifecycle path, and the linked fixes are open candidate PRs rather than merged implementations.

## What This Changes

The issue reports that active-memory timeout cleanup can close the shared QMD-backed memory_search manager, making chat memory_search unavailable while CLI QMD search still works.

## Best Possible Solution

Merge one focused fix that makes active-memory timeout cleanup request-scoped, preserves disposable local resource cleanup, and does not retire the shared QMD full manager used by chat memory_search.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main routes active-memory timeout cleanup into full manager close, and memory-core closes the cached QMD manager that later chat memory_search calls can reuse.

## Solution Assessment

Yes at the issue-routing level. The maintainable fix is to narrow timeout cleanup so it does not close the shared QMD manager; the exact API shape should be decided while reviewing the linked candidate PRs.
