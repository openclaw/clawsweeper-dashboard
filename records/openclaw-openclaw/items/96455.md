- `impact:session-state`: The affected state is the agent's shared memory_search manager and subsequent chat recall availability.

## Summary

Keep open: current main still contains the reported lifecycle path, and https://github.com/openclaw/openclaw/pull/96459 is an open closing candidate rather than a merged fix.

## What This Changes

An active-memory timeout can close the shared QMD-backed memory_search manager, leaving chat memory_search unavailable while CLI QMD search still succeeds.

## Best Possible Solution

Preserve the long-lived shared QMD full manager during request-scoped active-memory timeouts, keep the disposable local embedding-provider cleanup, and land focused regression coverage through the linked fix PR if review passes.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: active-memory timeout cleanup currently routes into closeMemorySearchManager, and that path closes the cached QMD manager. I did not run a live QMD gateway reproduction in this read-only review.

## Solution Assessment

Yes at the issue-routing level: the best fix is to narrow scoped cleanup so request timeouts do not tear down the shared QMD full manager while preserving local provider cleanup. The open linked PR implements that direction and should be reviewed rather than opening a duplicate fix lane.
