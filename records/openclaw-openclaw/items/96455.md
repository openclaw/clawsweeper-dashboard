## Summary

Keep open: current main still contains the reported active-memory timeout cleanup path, and the linked candidate fixes are open and unmerged rather than an implemented fix.

## What This Changes

The issue reports that active-memory timeout cleanup can close the shared QMD memory_search manager, making chat memory_search return a closed-manager error while CLI QMD search still works.

## Best Possible Solution

Land one narrow linked fix that makes active-memory timeout cleanup release only disposable recall resources while preserving the shared QMD manager for chat and startup paths.

## Reproduction Assessment

Yes from source, not by a live run: current main still routes active-memory timeout cleanup to closeActiveMemorySearchManager, and QMD managers are cached by agent for default chat use before that close path deletes and closes them.

## Solution Assessment

Yes on the issue direction: the best fix is a narrow lifecycle separation, not a new feature or config option; maintainers still need to choose between the two open linked PR implementations.
