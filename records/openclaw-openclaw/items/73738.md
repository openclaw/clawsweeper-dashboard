## Summary

Keep open: current main does not implement `workspaceLifecycle`, and this active feature PR still has source-reproducible blockers in media staging and `package_url` address filtering plus insufficient real behavior proof for the runtime lifecycle.

## What This Changes

The PR adds `workspaceLifecycle: persistent | ephemeral` sandbox config/docs/schema/runtime cleanup and tests, plus package URL download hardening and small gate-related fixture/trust annotations.

## Best Possible Solution

Keep this PR as the implementation path only after media staging uses the same run-scoped workspace, package URL address filtering covers IPv4-mapped blocked ranges, changelog credit is corrected, and maintainers accept the public sandbox mode with real runtime proof.

## Reproduction Assessment

Yes. The blockers are source-reproducible: PR head derives ephemeral workspaces only with `runId`, while media staging still resolves a workspace without `runId`, and the IPv6 blocker visibly omits IPv4-mapped link-local/private ranges.

## Solution Assessment

No, not yet. The config/lifecycle shape is plausible, but the patch is not the narrow maintainable solution until staged media targets the run-scoped workspace and the package URL blocker delegates IPv4-mapped addresses through the IPv4 classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Stage media in run-scoped sandbox workspaces:** `src/agents/sandbox/context.ts:429-433`
  - body: When `workspaceLifecycle` is `ephemeral`, `resolveSandboxContext()` derives the real workspace from `runId`, but `ensureSandboxWorkspaceForSession()` still has no run id and is what inbound media staging uses. Attachments are copied into the persistent per-scope workspace while the agent runs in a different run workspace, so documented `media/inbound/*` paths will be missing for ephemeral runs.
  - confidence: 0.9
- **[P2] Block all IPv4-mapped package URL ranges:** `scripts/resolve-openclaw-package-candidate.mjs:215-218`
  - body: The new DNS blocker treats `::ffff:*` as IPv6 and only covers a few private ranges. Addresses like `::ffff:169.254.169.254` pass validation, so `package_url` can still connect to metadata/link-local hosts before tarball hash checking.
  - confidence: 0.86
- **[P3] Credit the contributor in the changelog entry:** `CHANGELOG.md:1070`
  - body: This user-facing feature adds an active changelog entry without the required external contributor credit. Add `Thanks @Protocol-zero-0` to the one-line entry so it matches the repo changelog policy for contributor PRs.
  - confidence: 0.95

## Security Review

Status: needs_attention

Summary: The PR improves package URL fetching, but the new address filter still leaves an IPv4-mapped link-local/private bypass.
