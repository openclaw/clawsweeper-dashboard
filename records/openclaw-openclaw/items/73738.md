## Summary

Keep open. The PR is not obsolete because current main still lacks the sandbox `workspaceLifecycle` surface, but the latest head still has a security-sensitive package URL redirect gap that should block merge until redirects are rejected or revalidated.

## What This Changes

The PR adds sandbox `workspaceLifecycle` config/schema/docs/runtime cleanup for run-scoped ephemeral workspaces, updates sandbox tests and cleanup helpers, annotates system events, tweaks Docker E2E setup, hardens package URL downloads, and restores a tool-display config export.

## Best Possible Solution

Keep this PR as the implementation path for #14804, fix the package URL resolver to reject redirects or apply the same SSRF/DNS pinning policy to every redirect target, add focused redirect coverage, then have secops review the sandbox cleanup semantics before merge.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR head plus Undici 8.1.0 source shows redirects are followed by default while only the original host is validated. A focused test can reproduce this with an allowed HTTPS URL that returns a 30x to a private/internal host.

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle design is a plausible narrow direction, but the package URL hardening is incomplete until redirects are rejected or each redirect target is revalidated and repinned with the same policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject package URL redirects before downloading:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The new package URL guard validates and pins only the initial host, but `undiciFetch` follows redirects by default. Because the custom lookup falls back to normal DNS for a different redirected host, an allowed URL can 30x to a private/internal target and bypass the blocked-host/address checks; reject redirects or validate and repin each redirect target before downloading.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff still has a concrete package download redirect bypass in a CI/package acceptance path.

Concerns:

- **[high] Package URL redirects bypass host checks:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The PR checks DNS and blocked address ranges only for the original `package_url` host, but Undici follows redirects by default and the custom lookup delegates redirected hosts to normal DNS. A public allowed URL can therefore redirect the package download to a private/internal host without the same validation.
