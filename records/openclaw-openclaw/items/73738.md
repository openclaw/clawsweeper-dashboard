## Summary

Keep open. Current main still lacks the requested sandbox workspace lifecycle, and this PR is the active implementation path for #14804, but the latest head still has a blocking package URL redirect validation gap on a CI/package download surface. The sandbox cleanup review comments appear addressed, CI is broadly green, and the remaining blocker needs author/security maintainer handling before merge.

## What This Changes

The PR adds a `workspaceLifecycle: "persistent" | "ephemeral"` sandbox config option with run-scoped workspace/runtime cleanup, schema/docs/tests, Discord trust annotations, a Docker E2E fixture tweak, and package URL download hardening.

## Best Possible Solution

Keep this PR as the implementation path for #14804, but require the package URL resolver to reject redirects or manually validate and repin every redirected HTTPS target with the same blocked-host/address policy. Add focused redirect-to-private/internal-host coverage, then rerun the package resolver tests, sandbox/config tests, schema check, core typecheck, and security gates before maintainer/security review.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the latest PR head plus Undici 8.1.0 source shows fetch follows redirects by default while the PR validates only the original host. A minimal reproduction would use an allowed HTTPS `package_url` that returns a 30x to an internal/private host; no live exploit was run because this review is read-only.

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle shape is a plausible narrow solution for disposable isolated workspaces, but the bundled package URL hardening is incomplete until redirect targets are rejected or revalidated with the same SSRF policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject unvalidated package URL redirects:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The new dispatcher validates and pins only the original `package_url` host, but `undiciFetch(parsed, { dispatcher })` follows redirects by default. When a 30x points at another host, `createPinnedLookup` delegates that host to normal DNS, so an allowed URL can redirect the CI package download to a private/internal address without the blocked-host/address checks. Set redirects to `error`/`manual`, or validate and repin each redirect target before downloading.
  - confidence: 0.9
