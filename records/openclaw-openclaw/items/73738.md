## Summary

Keep open: current main does not implement ephemeral sandbox workspaces, and this PR remains the active implementation path for #14804, but the patch still has a source-reproducible media staging bug and lacks required real behavior proof.

## What This Changes

The PR adds a `workspaceLifecycle` sandbox config/schema/docs/runtime path for run-scoped ephemeral workspaces, cleanup coverage, a changelog entry, package URL hardening, and small gate follow-ups.

## Best Possible Solution

Keep this PR as the implementation path after media staging/prestaging uses the same run-scoped workspace as the agent run, the contributor adds real behavior proof, and maintainers review the new public sandbox config mode.

## Reproduction Assessment

Yes for the blocking PR bug: source inspection shows media staging callers use `ensureSandboxWorkspaceForSession`, whose PR merge version cannot accept `runId`, while the embedded runner resolves the actual sandbox with `runId`. I did not run a live sandbox in this read-only review.

## Solution Assessment

No, not yet. The lifecycle shape is reasonable, but the implementation must stage inbound media into the same run-scoped workspace before it is the narrow maintainable solution.
