## Summary

Keep open. Current main does not implement disposable sandbox workspaces, and this active PR is the implementation path for #14804, but the head still has a merge-blocking package download redirect gap plus a missing changelog entry.

## What This Changes

The branch adds a sandbox `workspaceLifecycle` config with run-scoped ephemeral workspaces and cleanup, schema/docs/tests, Discord trust annotations, Docker E2E/package URL hardening, and a restored tool-display export.

## Best Possible Solution

Keep this PR as the implementation path for #14804 after fixing package URL redirects, adding redirect-to-private-host coverage and the changelog entry, then rerunning the targeted sandbox/config/package gates.

## Reproduction Assessment

Yes. A public allowed HTTPS `package_url` that returns a 30x redirect to a private/internal HTTPS host exercises the P1 path; source inspection of the PR diff plus Undici 8.1.0 redirect behavior establishes the bypass.

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle shape is a plausible narrow solution, but the package URL hardening must reject redirects or revalidate and repin each redirect hop before merge.
