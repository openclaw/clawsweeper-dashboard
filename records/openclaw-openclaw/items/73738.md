## Summary

Keep open. Current main still lacks the ephemeral sandbox workspace lifecycle, and this live contributor PR is the paired implementation for #14804, but the patch is not merge-ready because its package URL hardening still follows unvalidated redirects and it lacks the required changelog entry.

## What This Changes

The PR adds a sandbox `workspaceLifecycle` config with run-scoped ephemeral workspaces and cleanup, schema/docs/tests, related Discord trust annotations, package URL hardening, and a restored tool-display export.

## Best Possible Solution

Keep this PR as the implementation path for #14804 after fixing redirect handling, adding redirect-to-private-host coverage and the changelog entry, then rerun targeted sandbox/config/package validation.

## Reproduction Assessment

Yes. A public allowed HTTPS `package_url` that returns a 30x redirect to a private/internal host exercises the P1 path; source inspection plus Undici 8.1.0 redirect behavior gives a high-confidence reproduction path.

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle design is a plausible narrow solution, but the package download path must reject redirects or revalidate and pin each redirect hop before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject redirected package URL downloads:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The new host/address checks pin only the original `package_url` hostname, but this fetch still follows redirects by default and the custom lookup falls back to normal DNS for different redirected hosts. An allowed public URL can 30x to a private/internal host and bypass the blocked-host/address checks; set redirects to `error`/`manual`, or validate and pin every redirected target before writing the tarball.
