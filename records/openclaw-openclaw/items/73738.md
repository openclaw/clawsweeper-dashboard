## Summary

Keep open: the PR is not obsolete on current main, but source review found blocking mismatches in run-scoped media staging and package URL address blocking.

## What This Changes

The PR adds `workspaceLifecycle: persistent | ephemeral` for sandbox configs, derives run-scoped workspaces/runtimes, wires cleanup, updates schema/docs/tests, and hardens package URL downloads.

## Best Possible Solution

Keep this PR as the implementation path after inbound media staging uses the same run-scoped workspace, package URL blocked-address coverage is complete, and maintainers sign off on the new public sandbox config mode.

## Reproduction Assessment

Yes for the blocking PR bugs: source inspection shows staging helpers still resolve a session workspace without `runId`, while the PR's actual sandbox context uses `runId`, and the new IPv6 blocker omits IPv4-mapped link-local ranges. I did not run a live Docker sandbox in this read-only review.

## Solution Assessment

No, not yet. The lifecycle shape is plausible, but the PR needs to stage media into the run-scoped workspace and close the package URL address-blocking gap before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Stage inbound media in run-scoped workspaces:** `src/agents/sandbox/context.ts:429-433`
  - body: When `workspaceLifecycle` is `ephemeral`, the runner derives the actual sandbox workspace from `runId`, but `ensureSandboxWorkspaceForSession()` still has no run id and the media staging/prestaging callers use it before the run. Attachments are copied into the persistent per-scope workspace while the agent executes in a different run workspace, so documented `media/inbound/*` files are missing; thread the run id through staging or stage against the resolved sandbox workspace.
  - confidence: 0.9
- **[P2] Block IPv4-mapped link-local package URLs:** `scripts/resolve-openclaw-package-candidate.mjs:204-219`
  - body: The new `isBlockedIpv6()` only rejects some `::ffff:` private ranges. An AAAA record such as `::ffff:169.254.169.254` is treated as IPv6 by Node and passes both DNS validation and the pinned dispatcher, so `package_url` can still fetch link-local metadata hosts despite the new blocked-address check. Reuse the IPv4 classifier for IPv4-mapped addresses or cover the same ranges.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR improves package URL fetching, but the new blocked-address check still misses IPv4-mapped link-local and related blocked IPv4 ranges.

Concerns:

- **[medium] IPv4-mapped link-local package URL bypass:** `scripts/resolve-openclaw-package-candidate.mjs:204`
  - body: `isBlockedIpv6()` does not map `::ffff:` addresses through the IPv4 blocker, so a package URL host resolving to `::ffff:169.254.169.254` can pass the new validation before the tarball hash check runs.
  - confidence: 0.78
