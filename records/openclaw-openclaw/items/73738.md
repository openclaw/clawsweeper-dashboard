## Summary

Keep open. Current main still lacks the sandbox `workspaceLifecycle` feature, and the PR is the active implementation path for #14804, but the latest head still has a concrete security blocker in the package URL downloader because redirects are followed without revalidating the redirected host.

## What This Changes

The PR adds a sandbox `workspaceLifecycle` config option, run-scoped ephemeral sandbox workspace/runtime cleanup, schema/docs/tests, Discord trust annotations, a Docker E2E fixture tweak, package URL download hardening, and a restored tool-display export.

## Best Possible Solution

Keep this PR as the implementation path for #14804, but fix package URL downloads to reject redirects or manually validate and repin every redirected HTTPS target, add redirect-to-private-host coverage, add the required changelog entry, then rerun the targeted package, sandbox, config, and typecheck gates before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head plus Undici 8.1.0 source gives a high-confidence reproduction path: use an allowed HTTPS `package_url` that returns a 30x redirect to a private/internal HTTPS host and observe that the redirected host is not checked by the new policy.

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle design is a plausible narrow solution, but the package URL hardening is incomplete until redirect handling is rejected or revalidated; the changelog gap is a separate mechanical merge blocker.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject redirects for package URL downloads:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The new dispatcher validates and pins only the original `package_url` host, but this fetch still uses Undici's default redirect-following behavior. Because `createPinnedLookup` delegates different redirected hostnames to normal DNS, an allowed URL can 30x to a private/internal host and bypass the blocked-host/address checks. Set redirects to `error`/`manual`, or revalidate and repin every redirect target before writing the tarball.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `docs/gateway/sandboxing.md:306`
  - body: This PR changes user-facing sandbox config/runtime behavior and touches non-doc/test files, but it does not update `CHANGELOG.md`. The repo merge gate marks this changed-file set as changelog-required, so add a single-line entry for the active release referencing this PR and crediting the external contributor.
  - confidence: 0.9
