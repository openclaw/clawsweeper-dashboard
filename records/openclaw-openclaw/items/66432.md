## Summary

Keep open. Current main still leaves Feishu thread-history media as parsed placeholder text, while this PR is a focused Feishu channel fix that reuses the existing media download/save path and attachment-hint contract. The patch looks functionally sound, with remaining small review polish and validation before merge.

## What This Changes

The PR adds raw Feishu thread-message body content, resolves media for media-typed thread-history messages before formatting `ThreadHistoryBody`, appends existing `[media attached: ...]` hints, and adds an image-history regression test.

## Best Possible Solution

Land a polished version of this PR in the Feishu plugin: preserve raw thread message content, resolve history media through the existing helper, keep the current attachment-hint contract, add the requested failure-path regression, and validate the Feishu plugin lane before merge.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. Reusing `resolveFeishuMediaList` is the narrow maintainable fix because it keeps Feishu-owned behavior inside the Feishu plugin and matches the existing current-message media contract; the safer alternative is polish within this PR, not a core API change.
