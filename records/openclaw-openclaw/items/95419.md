## Summary

## What This Changes

The PR reroutes nonfatal isolated cron delivery-dispatch errors through the successful execution path, merges delivery diagnostics, and adds a regression test for status ok plus failed delivery metadata.

## Best Possible Solution

Keep the narrow execution/delivery split, carry a delivery-specific error through cron state and run-log persistence with focused coverage, then let maintainers confirm ok/not-delivered versus a first-class warning status.

## Reproduction Assessment

## Solution Assessment

No as submitted. finalizeCronRun is the right narrow layer for the status split, but the best fix must carry delivery failure text through a delivery-specific persistence path without reusing execution error, and maintainers still need to accept the public ok/not-delivered semantics.
