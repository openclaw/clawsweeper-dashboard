## Summary

Keep this PR open: the security hardening direction is useful and matches recent roadmap discussion, but the patch parses config backups with a strict JSON helper instead of OpenClaw's config parser, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds secrets-audit scanning for adjacent `openclaw.json` backup artifacts and a regression test for a `.bak` file containing a plaintext provider API key.

## Best Possible Solution

Keep the backup-audit feature, but parse backup config artifacts with the same JSON5-compatible config parsing path as active configs and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes for the review finding from source: the PR sends backup configs through `readJsonObjectIfExists`, which uses `JSON.parse`, while current active config loading uses `parseConfigJson5`. I did not execute tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not yet: scanning backup artifacts is a reasonable security-audit extension, but it should use OpenClaw's config parser rather than the strict JSON sidecar helper. The added regression should cover JSON5 backup syntax as well as the `.bak` plaintext case.
