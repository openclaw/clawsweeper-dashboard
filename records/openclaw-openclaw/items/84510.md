## Summary

Keep open: the newly supplied terminal proof covers the happy-path CLI behavior, but the patch still parses config backups as strict JSON while OpenClaw configs are JSON5, so valid backup artifacts can become unresolved and hide plaintext secret findings.

## What This Changes

The PR extends `openclaw secrets audit` to scan adjacent config backup files, report plaintext secrets in those backup configs, and add regression coverage for `.bak` provider API keys.

## Best Possible Solution

Keep the backup-audit feature, but parse config backup artifacts with OpenClaw's JSON5-compatible config parser and add regression coverage for a JSON5 `.bak` containing a plaintext provider API key.

## Reproduction Assessment

Yes for the review finding from source: the PR sends backups through `readJsonObjectIfExists`, which uses `JSON.parse`, while current config loading uses `parseConfigJson5`. I did not execute tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not yet: scanning backups is a reasonable security-audit extension, but config backup artifacts should use the same JSON5 parser as active config loading rather than the strict JSON sidecar helper.
