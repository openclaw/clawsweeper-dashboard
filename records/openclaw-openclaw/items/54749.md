## Summary

Keep open: current main still has a narrow report-card content navigation gap. Report-kind Memory Palace pages are listed, but primary card clicks still toggle inline metadata while the actual wiki content preview remains behind a nested button; the exact mobile-only Command Overview fallback is still not visible in source or discussion evidence.

## What This Changes

Mobile report-list selection should open report detail/content instead of showing the Command Overview.

## Best Possible Solution

Make primary selection of Memory Palace report cards open the existing wiki.get-backed preview, while keeping metadata expansion on a separate details control if that affordance is still useful.

Yes, the narrowest maintainable fix is in `ui/src/ui/views/dreaming.ts` with a focused view regression test. The backend list and content APIs already exist through `wiki.palace` and `wiki.get`, so no gateway or plugin API change is indicated.
