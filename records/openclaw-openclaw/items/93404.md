## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Confine package paths before copying or removing files:** `packages/agent-sdk/src/commands/enable.ts:25-29`
  - body: `entry.src` and `entry.dest` are resolved directly from manifest strings, and disable later removes the same destinations. A package can use absolute paths or `..` segments to read from, write to, or remove files outside the intended package/workspace; validate canonical package-relative sources and workspace-relative destinations before any filesystem operation.
  - confidence: 0.95
- **[P1] Keep quarantine tools read-only:** `packages/agent-sdk/src/quarantine/mutation.ts:138-140`
  - body: The helper documents quarantine as read-only, but its allowlist includes `write` and `edit`. A package quarantined for instruction mutation would still be able to modify files through those tools, so the quarantine boundary does not enforce the stated restriction.
  - confidence: 0.94
- **[P1] Compile only schema-approved config paths:** `packages/agent-sdk/src/compiler/compiler.ts:33-64`
  - body: The compiler labels these entries as actual `openclaw.json` paths and always emits `agentPackages.*`, but current main's strict config schema has no `agentPackages` block. Dry-run output should be derived from or validated against the current schema so future writes do not create unsupported config contracts.
  - confidence: 0.93
- **[P1] Reuse the shared net-policy classifier:** `packages/agent-sdk/src/policy/network.ts:100-102`
  - body: This package exports a smaller string-based private IP check even though current main already centralizes special-use, metadata, mapped, and embedded IP handling in `@openclaw/net-policy/ip`. Package network enforcement should reuse the shared classifier to avoid a weaker security boundary.
  - confidence: 0.92
- **[P2] Wire the CLI into the package build:** `packages/agent-sdk/package.json:23-24`
  - body: The package advertises CLI commands and exports `dist/*`, but the build omits `src/cli.ts` and `src/commands/pack.ts`, and there is no `bin` entry. As written, the documented `pack` CLI is not produced by the supported package build.
  - confidence: 0.91
- **[P2] Guard `manifest.files` before nested validation:** `packages/agent-sdk/src/commands/validate.ts:21-23`
  - body: When `manifest.files` is missing, validation records an error and immediately reads `manifest.files.copy` and `manifest.files.mutable`, which throws instead of returning structured validation errors for malformed manifests.
  - confidence: 0.96
- **[P2] Run pack tests against an isolated fixture copy:** `packages/agent-sdk/src/pack.test.ts:50-60`
  - body: The pack test runs the CLI against the checked-in `valid-pack` fixture and asserts that `openclaw.integrity.json` appears there, so running the test dirties the source tree and can make later checks depend on generated fixture state.
  - confidence: 0.88
- **[P2] Keep unimplemented enable flows out of workflow docs:** `docs/specs/agent-sdk-packaging.md:301-307`
  - body: The spec documents `openclaw agent enable` applying config, binding channels, creating schedules, and recording `agentPackages`, but this PR does not add a supported OpenClaw CLI integration or config schema for those flows. Mark the section as planned/RFC-only or land it with the implementation it describes.
