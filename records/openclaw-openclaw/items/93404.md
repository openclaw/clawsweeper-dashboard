  - body: `compileManifest` emits package-scoped settings under `agentPackages.packages.<name>.*`, but the new strict `AgentPackagesSchema` never declares `packages`. Any package using package-scoped policy/tools/sandbox output will generate config that OpenClaw's own schema rejects.

  - body: `enable` reports that it wrote config, but it writes `agent-sdk-config.json` in the workspace; current OpenClaw reads `OPENCLAW_CONFIG_PATH` or the state-dir `openclaw.json`, so a successful enable can leave the gateway ignoring the package config.
  - confidence: 0.92
- **[P1] Fail unsupported required secret mappings:** `packages/agent-sdk/src/compiler/compiler.ts:241-245`

  - confidence: 0.9
- **[P2] Validate proof against the real config schema:** `packages/agent-sdk/src/test.ts:477-484`
  - body: The `agent.configCompiler.dryRunValidates` proof only checks for a non-empty diff and no compiler-unsupported fields. It does not parse the generated config with `OpenClawSchema`, so the current schema mismatch can still report PASS.
