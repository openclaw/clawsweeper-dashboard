## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Constrain package paths before copying or removing files:** `packages/agent-sdk/src/commands/enable.ts:24-29`
  - body: `resolve(workspacePath, entry.dest)` and `resolve(packagePath, entry.src)` are used without checking that the resolved paths stay under the intended roots. A manifest can use `../` segments to copy files from outside the package or write/remove files outside the workspace, which is not acceptable for a packaging security boundary.
  - confidence: 0.95
- **[P1] Compile only approved current config paths:** `packages/agent-sdk/src/compiler/compiler.ts:33-64`
  - body: The compiler hard-codes draft paths such as `agentPackages.*`, `agents.defaults.tools.*`, `secrets.mapping`, and `cron.jobs` as actual `openclaw.json` fields, but current main's strict config schema and types do not expose those surfaces. A dry-run compiler needs to share or probe the real schema and reject unsupported mappings instead of producing diffs that future config writes would reject or misapply.
  - confidence: 0.93
- **[P1] Keep package state out of JSON sidecars:** `packages/agent-sdk/src/commands/enable.ts:42-65`
  - body: The enable path writes `agent-sdk-config.json`, `agent-sdk-registry.json`, `agent-sdk-bindings.json`, and `agent-sdk-schedules.json` into the workspace as OpenClaw-owned state. Current repository policy keeps runtime state in canonical config or SQLite-owned stores, so this would create a second package/config source of truth outside the approved migration path.
  - confidence: 0.89
- **[P1] Reuse the shared network policy for DNS rebinding checks:** `packages/agent-sdk/src/policy/network.ts:100-149`
  - body: `isPrivateIp` is exported as part of the package security boundary but only performs string checks for a small range list, while current main already has `@openclaw/net-policy` handling special-use ranges, metadata IPs, mapped/embedded IPv4, and legacy forms. Use the shared classifier and fixtures so future enforcement cannot drift weaker than core policy.
  - confidence: 0.92
- **[P2] Wire the package and CLI into a real build surface:** `packages/agent-sdk/package.json:23-24`
  - body: The package exposes only `dist/*`, but the package build omits `src/cli.ts` and `src/commands/pack.ts`, has no `bin`, and the shared tsdown package list does not include `agent-sdk`. As written, the documented pack/validate CLI is not produced by the normal build or by this package build.
  - confidence: 0.93
- **[P2] Guard missing files before reading nested manifest fields:** `packages/agent-sdk/src/commands/validate.ts:21-23`
  - body: When `manifest.files` is absent, validation records `files is required` and then immediately reads `manifest.files.copy` and `manifest.files.mutable`, so malformed manifests throw instead of returning structured validation errors. Return early or guard nested checks after the top-level object exists.
  - confidence: 0.95
- **[P2] Run pack tests against an isolated fixture copy:** `packages/agent-sdk/src/pack.test.ts:50-61`
  - body: This test runs `pack` against the checked-in fixture and expects `openclaw.integrity.json` to appear there, so the test dirties the source tree and later validation can depend on generated fixture state. Copy the fixture to a temp directory and clean it up instead.
  - confidence: 0.88
