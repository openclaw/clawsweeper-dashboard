- **[P1] Confine package file destinations to the workspace:** `packages/agent-sdk/src/commands/enable.ts:26-29`
  - body: `entry.dest` is resolved directly under `workspacePath` without rejecting absolute paths or `..`, and `disable` uses the same unchecked destination for removal.   - confidence: 0.94
- **[P1] Compile only approved current config paths:** `packages/agent-sdk/src/compiler/compiler.ts:33-64`
  - body: The compiler labels its hard-coded set as actual `openclaw.json` paths and emits `agentPackages.*`, `agents.defaults.tools.*`, `secrets.mapping`, and `cron.jobs`, but current main's strict config schema does not expose several of those surfaces. Dry-run output must share or probe the current schema and reject unsupported mappings instead of producing diffs future writes would reject or misapply.
  - confidence: 0.93
- **[P1] Reuse the shared network policy for DNS rebinding checks:** `packages/agent-sdk/src/policy/network.ts:100-102`
  - body: `isPrivateIp` is exported as package security logic but only checks a small string-based range list, while current main already uses `@openclaw/net-policy/ip` for special-use, metadata, mapped, and embedded IP forms. Use the shared classifier and fixtures so package enforcement cannot drift weaker than core policy.
  - confidence: 0.92
- **[P2] Wire the package and CLI into a real build surface:** `packages/agent-sdk/package.json:24`
  - body: The package exports only `dist/*`, but its build omits `src/cli.ts` and `src/commands/pack.ts`, has no `bin`, and root `tsdown.config.ts` has no `agent-sdk` package entry. As written, the documented pack CLI is not produced by supported builds.
  - confidence: 0.91
- **[P2] Guard missing files before reading nested manifest fields:** `packages/agent-sdk/src/commands/validate.ts:21-23`
  - body: When `manifest.files` is absent, validation records `files is required` and then immediately reads `manifest.files.copy` and `manifest.files.mutable`, so malformed manifests throw instead of returning structured validation errors. Return early or guard nested checks after the top-level object exists.
  - confidence: 0.96
- **[P2] Run pack tests against an isolated fixture copy:** `packages/agent-sdk/src/pack.test.ts:52-60`
  - body: This test runs `pack` against the checked-in `valid-pack` fixture and asserts that `openclaw.integrity.json` appears there, so the test dirties the source tree and can make later checks depend on generated fixture state. Copy the fixture to a temp directory and clean it up instead.
  - confidence: 0.88
- **[P2] Keep unimplemented enable flows out of workflow docs:** `docs/specs/agent-sdk-packaging.md:301-307`
