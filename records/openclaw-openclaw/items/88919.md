## Summary

## What This Changes

This PR adds `allowReentrant: true` to embedded preflight compaction's session write-lock acquisition and adds a regression guard that checks the compaction lock call keeps that opt-in.

## Best Possible Solution

Land the scoped compaction reentrant-lock opt-in after maintainer acceptance of the same-logical-writer semantics, and keep broader Feishu/subagent lock contention tracked separately.

## Reproduction Assessment

Yes. Current main acquires the compaction session lock without `allowReentrant`, existing lock tests show non-reentrant same-process acquisition rejects, and the PR body supplies live before/after output for the affected Telegram topic.

## Solution Assessment

Yes, this is an acceptable narrow fix for the reported preflight compaction failure because OpenClaw's own docs require explicit reentrant opt-in for nested same-logical-writer lock acquisition. A broader default-reentrant rewrite would be a separate session-lock design decision.
