## Summary

Keep open: the branch moves the retry helper toward per-attempt replay metadata, but it still does not route classified empty provider 5xx/server_error turns into that helper, and the submitted proof remains mocked runner/unit output rather than real runner or gateway behavior.

## What This Changes

The PR adds optional current-attempt replay metadata to embedded attempt results, derives it during normalization, and makes empty-error retry prefer that per-attempt metadata over cumulative replay metadata.

## Best Possible Solution

Land one proof-backed fix that routes classified empty provider 5xx/server_error failures through retry while preserving the current-attempt side-effect guard and conservative fallback for harnesses without per-attempt metadata.

## Reproduction Assessment

Yes at source level: current main accumulates replayMetadata across attempts and rejects retry when that metadata reports side effects, while classified provider 500/server_error failures are filtered before this PR's helper change can help. I did not run a live OpenRouter 5xx reproduction in this read-only review.

## Solution Assessment

No as submitted: splitting current-attempt metadata is the right direction, but the branch still leaves classified provider 5xx/server_error failures outside the retry path. The best fix should combine the current-attempt side-effect guard with caller-side eligibility for the linked transient provider-error shape.
