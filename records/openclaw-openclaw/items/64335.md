## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Avoid storing ZAI_API_KEYS as the persisted profile ref:** `extensions/zai/index.ts:325`
  - body: `ZAI_API_KEYS` is a delimited rotation list, but this branch can persist it as the `zai:default` env ref in `--secret-input-mode ref`. If the user later adds another key to that variable, the default profile resolves the raw `key1,key2` string as one API key before rotation profiles are tried, so auth can fail instead of rotating. Keep the list var runtime-only or require a single-key env var for persisted refs.
  - confidence: 0.82
