- **[P1] Preserve external CLI overlays when widening auth profiles:** `src/agents/auth-profiles/external-auth.ts:111-128`
  - body: This rewrite of `overlayExternalAuthProfiles` only resolves plugin-provided profiles, while current main also folds in external CLI runtime profiles and passes the external-CLI options through persistence filtering.   - confidence: 0.82
- **[P2] Keep ZAI_API_KEYS out of persisted default refs:** `extensions/zai/index.ts:325`
  - body: `ZAI_API_KEYS` is a comma/semicolon list variable, but this path can store it as the `zai:default` env ref. If a user later expands the variable from one key to multiple keys, the default profile resolves the raw list as one API key before runtime rotation profiles are useful, so auth can fail instead of rotating.
  - confidence: 0.84
- **[P1] Keep Z.AI setup writes locked:** `extensions/zai/index.ts:346-350`
  - body: Current main writes the Z.AI auth profile through `upsertAuthProfileWithLockOrThrow`, but this head uses the unlocked `upsertAuthProfile` path. Reintroducing the unlocked write can race gateway/auth-store updates and silently lose or overwrite profile state during non-interactive setup.
  - confidence: 0.81
