## Summary

Keep open: current main still lacks catalog feedback in `models set`, but this PR is not mergeable because it bypasses the existing config mutation path and has no attached real CLI behavior proof.

## What This Changes

The PR changes `openclaw models set` to load the model catalog, warn when the resolved model is missing, and then save the configured default model.

## Best Possible Solution

Keep warning-only catalog-miss feedback, route the final config mutation through the existing base-hash/include-aware write path, preserve current Codex runtime repair behavior, and add focused command coverage plus real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main’s `modelsSetCommand` accepts a syntactically valid model ref through `applyDefaultModelPrimaryUpdate` without catalog lookup; I did not run the CLI because this read-only review must not mutate user config.

## Solution Assessment

No. Warning-only catalog feedback matches the runtime allowlist contract, but the PR should not replace the existing `updateConfig`/`replaceConfigFile` mutation semantics with a direct `writeConfigFile` call from a pre-catalog snapshot.
