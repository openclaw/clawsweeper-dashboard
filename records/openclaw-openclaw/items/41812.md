## Summary

Keep open: the underlying CLI feedback gap is real, but this PR still bypasses current config mutation and catalog side-effect contracts, drops existing Codex repair warnings, and lacks external-contributor real behavior proof.

## What This Changes

The PR changes `models set` to resolve the requested model, load the model catalog, warn when the model is absent, and then write the updated default model directly.

## Best Possible Solution

Implement warning-only catalog feedback with `loadModelCatalog({ readOnly: true })` inside the existing `updateConfig` and Codex repair-warning flow, then land it with real CLI proof.

## Reproduction Assessment

Yes, by source inspection: current main's `modelsSetCommand` writes the default model through `updateConfig` without any catalog lookup or typo warning. I did not run the CLI because this read-only review must not mutate user config.

## Solution Assessment

No, this is not the best way to solve it as-is. The safer implementation is warning-only catalog feedback using `readOnly: true`, while preserving include-aware config writes and existing Codex repair warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use a read-only catalog lookup:** `src/commands/models/set.ts:18`
  - body: `loadModelCatalog({ config: cfg })` takes the mutable catalog path. Current source shows that path can ensure `models.json` and invoke provider plugin catalog augmentation, so a warning-only `models set` check should pass `readOnly: true` to avoid catalog refresh/runtime side effects.
  - confidence: 0.88
- **[P2] Preserve include-aware config writes:** `src/commands/models/set.ts:28`
  - body: The patch reads a validated config, mutates it, and writes it with `writeConfigFile(updated)`. Current model commands use `updateConfig`, which clones the source config and routes through `replaceConfigFile` with base-hash and top-level include handling; bypassing that path can write the wrong config surface.
  - confidence: 0.94
- **[P2] Keep Codex repair warnings:** `src/commands/models/set.ts:28`
  - body: Current main calls `repairCodexRuntimePluginInstallForModelSelection` after updating the default so OpenAI/Codex model selections can surface missing-plugin repair warnings. This patch writes the config and returns success without that check, regressing existing CLI feedback.
  - confidence: 0.88
