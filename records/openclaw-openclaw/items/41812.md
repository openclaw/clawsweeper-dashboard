The PR changes `openclaw models set` to load the model catalog, warn when the resolved model is absent, and then save the selected default model.

## Best Possible Solution

Keep warning-only catalog-miss feedback, route the final mutation through the existing `updateConfig`/`replaceConfigFile` path, preserve current Codex runtime repair behavior, and add focused command coverage plus redacted real CLI proof.

## Solution Assessment

No. Warning-only feedback is safer than hard rejection under the current runtime allowlist contract, but this implementation should not replace the existing base-hash/include-aware config mutation path with a direct write.
