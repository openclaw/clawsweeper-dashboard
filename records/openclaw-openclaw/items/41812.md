## Summary

Keep open; current main still has the original config-time feedback gap, but this PR is not mergeable as-is because it bypasses the protected config write path and lacks required user-facing changelog coverage.

## What This Changes

The branch makes `models set` load the model catalog, log a warning for unresolved provider/model refs, and persist the selected default through a direct config write.

## Best Possible Solution

Keep the catalog-miss warning, preserve the existing config mutation contract, and add targeted tests plus a `CHANGELOG.md` fix entry.

## Reproduction Assessment

Yes. Current main reproduces the original gap by source inspection because `modelsSetCommand` never loads the model catalog; the PR regression is visible from the diff whenever config changes during the awaited catalog load or the target config section is include-owned.

## Solution Assessment

No. Warning on catalog miss is a maintainable direction, but the direct write is not; the safer fix is to keep the warning and route the final mutation through `updateConfig` or `replaceConfigFile` with the original snapshot/hash.
