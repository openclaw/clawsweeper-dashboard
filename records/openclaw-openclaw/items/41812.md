## Summary

Keep open: current main still lacks catalog feedback for `models set`, but this branch is not merge-ready because it bypasses existing command/config mutation contracts and needs contributor proof plus maintainer policy confirmation.

## What This Changes

The PR adds a model catalog lookup to `models set`, logs a warning when the selected provider/model is absent, and writes the updated default model directly.

## Best Possible Solution

Implement side-effect-free warning-only catalog feedback inside the existing `updateConfig` and Codex repair-warning path, or explicitly close this direction if maintainers decide config-time catalog validation is not wanted.

## Reproduction Assessment

Yes, by source inspection: current main's `modelsSetCommand` updates config through `applyDefaultModelPrimaryUpdate` and has no catalog lookup or typo warning. I did not run the CLI because this read-only review must not mutate user config.

## Solution Assessment

No. Warning-only catalog feedback may be the maintainable direction, but this diff should use a read-only catalog lookup, preserve `updateConfig`/include-aware writes, keep Codex repair warnings, and align the PR body with the implemented behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use a read-only catalog lookup:** `src/commands/models/set.ts:18`
  - body: `loadModelCatalog({ config: cfg })` uses the mutable catalog path; `loadModelCatalog` calls `ensureOpenClawModelsJson(cfg)` unless `readOnly: true`, so a typo warning in `models set` can create or refresh `models.json` as a side effect. Pass `readOnly: true` for this check.
  - confidence: 0.86
- **[P2] Preserve include-aware config writes:** `src/commands/models/set.ts:28-29`
  - body: This reads a validated config, waits for catalog loading, mutates that object, and writes it with `writeConfigFile(updated)`. Current model commands use `updateConfig`, which preserves source-config shape, single top-level include writes, and base-hash conflict checks; this direct write can overwrite the wrong config surface.
  - confidence: 0.94
- **[P2] Keep Codex repair warnings:** `src/commands/models/set.ts:29`
  - body: Current `models set` calls `repairCodexRuntimePluginInstallForModelSelection` after updating the default so OpenAI/OpenAI-Codex selections can surface missing Codex plugin repair warnings. The branch writes the config and returns success without that check, regressing this CLI feedback.
  - confidence: 0.88
