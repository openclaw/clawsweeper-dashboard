## Summary

Keep this PR open because it targets a real current-main gap, but it is not merge-ready: the head writes config through a weaker path than current `models set`, and the external PR still lacks after-fix CLI proof.

## What This Changes

This PR changes `openclaw models set` to load the model catalog, warn when the selected provider/model is absent, and then persist the default model.

## Best Possible Solution

Add warning-only catalog-miss feedback in `models set` while preserving the include-aware/base-hash config mutation path and current Codex runtime repair behavior, with focused command coverage and redacted CLI proof.

## Reproduction Assessment

Yes, by source inspection: current main calls `updateConfig(...applyDefaultModelPrimaryUpdate...)`, and `resolveModelTarget` only parses and resolves aliases without catalog lookup. I did not run the CLI because `models set` mutates user config in this read-only review.

## Solution Assessment

No. Warning-only catalog feedback fits the runtime allowlist contract, but the branch must preserve `updateConfig`/`replaceConfigFile` semantics and current Codex repair behavior before it is the narrow maintainable fix.
