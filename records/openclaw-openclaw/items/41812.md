## Summary

## What This Changes

The PR adds a catalog lookup to `models set`, warns when the requested provider/model is absent, and saves the updated default model through a direct config write.

## Best Possible Solution

Add side-effect-free, warning-only catalog feedback inside the existing `updateConfig` and Codex repair-warning path, or make an explicit maintainer decision if strict rejection is desired.

## Reproduction Assessment

Yes, by source inspection: current main's `modelsSetCommand` updates config through `applyDefaultModelPrimaryUpdate` and has no catalog lookup or typo warning. I did not run the CLI because this read-only review must not mutate user config.

## Solution Assessment

No. The warning-only direction may be maintainable after the prior hard-reject revert, but this diff should use a read-only catalog lookup, preserve include-aware `updateConfig` writes, and keep Codex repair warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use a read-only catalog lookup:** `src/commands/models/set.ts:18`
  - body: `loadModelCatalog({ config: cfg })` uses the mutable catalog path, which can call `ensureOpenClawModelsJson(cfg)` and provider catalog augmentation before returning. A warning check in `models set` should pass `readOnly: true` so inspecting a typo does not create or refresh catalog state as a side effect.
  - confidence: 0.86
- **[P2] Preserve include-aware config writes:** `src/commands/models/set.ts:28-29`
  - body: This reads a validated config, waits for catalog loading, mutates that object, and persists it with `writeConfigFile(updated)`. Current model commands use `updateConfig`, which clones the source config and routes through `replaceConfigFile` for hash conflict checks and single top-level include preservation; this direct write can overwrite the wrong config surface.
  - confidence: 0.94

  - body: Current `models set` calls `repairCodexRuntimePluginInstallForModelSelection` after updating the default so OpenAI/Codex selections can report missing Codex plugin repair warnings. The branch writes the config and returns success without that check, regressing this CLI feedback.
  - confidence: 0.88
