## Summary

Keep open: current main still lacks catalog-miss feedback in `models set`, but this PR should not merge as-is because it bypasses the established conflict-safe config mutation path and is missing the required user-facing changelog entry.

## What This Changes

The PR changes `models set` to load the model catalog, warn when the selected provider/model is absent, and then save that model as the default.

## Best Possible Solution

Keep the warning-only catalog-miss feedback, preserve `updateConfig`/`replaceConfigFile` semantics for the final write, and add focused tests plus an Unreleased changelog fix entry.

## Reproduction Assessment

Yes by source inspection. Current main's `modelsSetCommand` never loads the catalog before saving a syntactically valid model ref; I did not run the CLI because this read-only review avoids commands that can write user config.

## Solution Assessment

No. Warning on catalog miss is aligned with the runtime allowlist contract, but writing with `writeConfigFile(updated)` from a pre-catalog snapshot is not the narrowest maintainable fix; the final mutation should go through the existing config mutation helper contract.

## Review Findings

Overall correctness: patch is incorrect
