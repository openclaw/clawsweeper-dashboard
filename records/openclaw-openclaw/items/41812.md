## Summary

Keep open: the CLI typo-feedback goal is useful, but this PR still bypasses established config mutation and Codex repair-warning paths, and it lacks contributor real behavior proof.

## What This Changes

The PR changes `models set` to load the model catalog, warn when the resolved provider/model is absent, and then write the updated default model directly.

## Best Possible Solution

Add warning-only catalog feedback through the existing read-only catalog and `updateConfig` flow, preserve Codex repair warnings, and align the PR body/proof with the warning behavior.

## Reproduction Assessment

Yes, by source inspection: current main's `modelsSetCommand` writes through `updateConfig` and Codex repair handling without any catalog lookup or typo warning. I did not run the CLI because this read-only review must not mutate user config.

## Solution Assessment

No, this diff is not the best way to solve it as-is. The maintainable path is warning-only catalog feedback using `readOnly: true`, while preserving include-aware config writes and the existing Codex repair warning path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use a read-only catalog lookup:** `src/commands/models/set.ts:18`
  - body: `loadModelCatalog({ config: cfg })` takes the mutable catalog path; the loader runs `ensureOpenClawModelsJson(cfg)` unless `readOnly: true` and can invoke provider plugin catalog augmentation. A warning-only check in `models set` should not create or refresh catalog state as a side effect.
  - confidence: 0.87

Current model commands use `updateConfig`, which clones the source config and routes through `replaceConfigFile` with base-hash and top-level include handling; bypassing that path can write the wrong config surface.
  - confidence: 0.94

  - body: Current `models set` calls `repairCodexRuntimePluginInstallForModelSelection` after updating the default so OpenAI/Codex model selections can surface missing-plugin repair warnings. The PR writes the config and returns success without that check, regressing this CLI feedback.
  - confidence: 0.88
