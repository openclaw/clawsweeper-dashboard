## Summary

This PR should stay open, but it is not currently merge-ready: the useful direction is warning-only catalog feedback, yet the patch bypasses the current config mutation helper contract and still lacks real CLI behavior proof.

## What This Changes

The branch changes `openclaw models set` to load the model catalog, warn when the selected provider/model is absent, and then save the selected default model.

## Best Possible Solution

Add warning-only catalog-miss feedback in `models set` while preserving the existing base-hash/include-aware config mutation path and current Codex runtime repair warnings.

## Reproduction Assessment

Yes, by source inspection: current main resolves and writes `models set` values without a catalog lookup, and the PR diff can be seen bypassing the existing helper with `writeConfigFile(updated)`. I did not run the CLI because this read-only review avoids mutating user config.

## Solution Assessment

No. Warning-only catalog feedback fits the current runtime allowlist contract, but this branch should route the final save through `updateConfig`/`replaceConfigFile` or an equivalent base-hash/include-aware mutation path before merge.
