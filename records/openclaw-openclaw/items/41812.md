The PR changes `openclaw models set` to load the model catalog, warn when the selected provider/model is missing, and then save the selected default model.

## Best Possible Solution

Add warning-only catalog-miss feedback in `models set` while preserving `updateConfig`/`replaceConfigFile` semantics and current Codex runtime repair warnings.

## Reproduction Assessment

Yes, by source inspection: current main resolves and writes `models set` values without any catalog lookup. I did not run the CLI because the command mutates user config in this read-only review.

## Solution Assessment

No. Warning-only catalog feedback fits the current runtime allowlist contract, but this branch must keep the existing base-hash/include-aware mutation path before it is the narrow maintainable fix.
