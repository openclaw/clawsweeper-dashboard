The PR updates shared self-hosted provider discovery so configured vLLM/SGLang endpoints can still run catalog discovery when `agents.defaults.models` contains `provider/*`, with focused tests and a changelog entry.

## Best Possible Solution

Land one canonical fix that keeps exact configured model entries manual-only, enables `provider/*` discovery through configured self-hosted endpoints, and updates the vLLM/SGLang docs to match.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure `models.providers.vllm` or `models.providers.sglang` plus `agents.defaults.models["provider/*"]`, and the shared helper returns before the provider builder can query `/models`.

## Solution Assessment

Yes for the code path: fixing the shared self-hosted discovery helper covers both affected providers without duplicating plugin-local policy. The remaining safer step is to update the user-facing provider docs and canonicalize against the later related draft PR.
