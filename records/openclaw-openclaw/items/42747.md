## Summary

Keep open. Current main and the latest shipped release still use static Slack credentials with no token-rotation runtime, and adding Slack OAuth refresh-token persistence plus runtime credential mutation needs maintainer security/product design rather than cleanup closure.

## What This Changes

The issue requests native Slack OAuth token rotation support, including refresh-token storage, oauth.v2.access refresh handling, config updates, and CLI/status support.

## Best Possible Solution

Implement this in the official Slack plugin with reviewed SecretRef-aware credential storage, OAuth exchange/init/status/docs coverage, and serialized refresh/reload behavior for both Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and v2026.5.12 confirms Slack token-rotation config, refresh-token secret targets, and oauth.v2.access runtime are absent.

## Solution Assessment

Unclear pending maintainer/security design. The feature belongs in the official Slack plugin, but raw config patching and SIGUSR1-style reload should be replaced by an approved credential-persistence and runtime-reload contract.
