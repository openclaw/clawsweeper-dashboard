## Summary

Keep open. Current main still has static Slack credential handling only: no Slack token-rotation schema/config surface, no Slack refresh-token secret targets, and no `oauth.v2.access` refresh runtime; this is a security-sensitive feature request that needs maintainer design rather than cleanup closure.

## What This Changes

Add native Slack token rotation using `oauth.v2.access` refresh-token flow, including config, secret storage, refresh runtime, setup/status CLI, and reload behavior.

## Best Possible Solution

Implement the feature in the official Slack plugin with reviewed SecretRef-aware credential storage, schema/docs/status/doctor coverage, and serialized refresh plus runtime reload behavior for both Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence evidence that Slack token-rotation config, refresh-token secret targets, and OAuth refresh runtime are absent.

## Solution Assessment

Unclear pending maintainer/security design. The feature direction belongs in the official Slack plugin, but raw config patching and SIGUSR1-style reload should be refined into an approved credential-persistence and runtime-reload contract.
