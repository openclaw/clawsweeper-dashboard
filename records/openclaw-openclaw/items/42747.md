## Summary

Keep open. Current main still treats Slack credentials as static configured tokens, and native refresh-token storage plus an oauth.v2.access refresh loop would add security-sensitive auth behavior that needs maintainer product and security review before implementation.

## What This Changes

Add Slack token rotation support so OpenClaw can store refresh metadata and automatically refresh expiring Slack bot tokens with oauth.v2.access.

## Best Possible Solution

Define a Slack-plugin-owned token-rotation design that uses secure credential storage, follows Slack's one-use refresh-token contract, updates runtime clients safely in Socket and HTTP modes, exposes status/docs, and includes focused unit plus live Slack proof.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection confirms current main has no Slack tokenRotation schema or oauth.v2.access runtime path rather than a failing existing contract.

## Solution Assessment

Unclear. The requested direction is reasonable, but the exact fix should be decided by maintainers because refresh-token storage, Bolt OAuth integration, config mutation, and restart semantics are security- and product-sensitive.
