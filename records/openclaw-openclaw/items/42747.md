## Summary

Keep open. Current main still models Slack credentials as static tokens and lacks Slack OAuth token-rotation config, secret targets, refresh logic, setup command, or reload behavior; this is a security-sensitive feature request that needs maintainer design rather than cleanup closure.

## What This Changes

Add native Slack token rotation to the bundled Slack plugin, including OAuth refresh credentials, automatic refresh/reload behavior, and setup/status support.

## Best Possible Solution

Implement this in the Slack plugin with reviewed SecretRef-aware persistence, schema/docs/status/doctor coverage, serialized refresh, a safe bootstrap flow, and runtime reload semantics for both Socket Mode and HTTP mode.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source inspection on current main confirms there is no accepted config, schema, secret target, setup command, or runtime path for Slack OAuth token rotation.

## Solution Assessment

Is this the best way to solve the issue? Directionally yes that the bundled Slack plugin should own the feature, but the proposed raw config patching and SIGUSR1 flow should be refined into an approved secret-persistence and runtime-reload design.
