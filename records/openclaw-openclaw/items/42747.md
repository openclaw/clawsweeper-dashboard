## Summary

Keep open. Current main still supports static Slack credentials only; the requested OAuth token rotation fields, refresh loop, bootstrap command, and status/doctor surfaces are absent, and `token_expired` is still classified as a non-recoverable Slack auth error.

## What This Changes

Add native Slack OAuth token rotation support, including refresh-token storage, proactive `oauth.v2.access` refresh, runtime credential updates, and setup/status commands.

## Best Possible Solution

Implement this in the bundled Slack plugin with SecretRef-aware schema/persistence, bootstrap/status/doctor/docs coverage, serialized `oauth.v2.access` refresh, atomic credential updates, and runtime client refresh for both Socket Mode and HTTP mode; add a generic SDK seam only if current plugin persistence cannot support it safely.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source inspection on current main verifies the Slack config, secret registry, runtime startup, docs, and auth-error tests lack token-rotation or OAuth refresh paths.

## Solution Assessment

Is this the best way to solve the issue? Directionally yes that the bundled Slack plugin should own token rotation, but the proposed raw config-patching and SIGUSR1 shape should be refined into reviewed SecretRef-aware persistence and runtime reload behavior.
