## Summary

Keep open. Current main and v2026.5.22 still model Slack credentials as static bot/app/user/signing-secret inputs, while the requested Slack token rotation support would add new mutable OAuth credential storage, refresh runtime behavior, and setup/status UX that need maintainer security and product review.

## What This Changes

The issue requests native Slack OAuth token rotation with refresh-token storage, proactive `oauth.v2.access` refresh, and Slack token-rotation setup/status commands.

## Best Possible Solution

Design and implement this in the official Slack plugin with an explicit OAuth credential-persistence contract, atomic refresh/reload behavior, redaction/audit coverage, and focused bootstrap, refresh, failure, and status tests.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, command-surface search, and v2026.5.22 inspection show the requested Slack OAuth token-rotation capability is absent rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer security/product design. The capability belongs in the Slack plugin, but the proposed direct config-file patch plus SIGUSR1 workflow should be replaced or formalized through a reviewed credential-persistence and provider-reload contract.
