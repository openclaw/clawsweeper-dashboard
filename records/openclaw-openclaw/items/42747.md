## Summary

Keep open. Current main and v2026.5.12 still expose only static Slack bot/app/signing/user token configuration and no native `oauth.v2.access` refresh loop, so this remains a security-sensitive product/design feature rather than cleanup closure.

## What This Changes

Request native Slack token rotation support with config/secret storage, `oauth.v2.exchange` bootstrap, `oauth.v2.access` refresh, runtime token update, retry, and status behavior.

## Best Possible Solution

Implement this in the official Slack plugin with reviewed credential storage, OAuth exchange/refresh/status/docs coverage, and serialized atomic refresh plus channel/client reload behavior for both Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source and v2026.5.12 inspection confirm the requested Slack token-rotation config, secret targets, refresh runtime, and setup/status command surface do not exist.

## Solution Assessment

Unclear pending maintainer/security design. The feature belongs in the official Slack plugin, but raw config patching and SIGUSR1-style reload should be replaced by an approved credential-persistence and runtime-reload contract.
