## Summary

Keep open. Current main still only supports static Slack bot/app/user/signing-secret credentials, while this request adds a new security-sensitive Slack OAuth rotation flow with mutable credential persistence, runtime refresh, and CLI/status surface that needs maintainer security/product design first.

## What This Changes

Add native Slack token rotation using Slack oauth.v2.access refresh-token flow, including config, secret handling, runtime refresh, and setup/status CLI support.

## Best Possible Solution

Implement this in the official Slack plugin after security/product approval, with reviewed mutable credential storage, OAuth exchange/refresh/status/docs coverage, and serialized atomic refresh plus Slack client rebuild behavior for Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source and release inspection confirm current main and v2026.5.12 lack Slack tokenRotation config, refresh-token secret targets, oauth.v2.access runtime refresh, and setup/status CLI surface.

## Solution Assessment

Unclear pending maintainer/security design. The feature likely belongs in the official Slack plugin, but the proposed direct config patch plus SIGUSR1 workflow should be replaced with an approved credential-persistence and runtime-reload contract.
