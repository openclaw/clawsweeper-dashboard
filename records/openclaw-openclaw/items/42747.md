## Summary

Keep open. Current main and the latest source evidence still show static Slack token configuration only; native Slack OAuth token rotation would add sensitive OAuth persistence, refresh scheduling, setup/status CLI, and runtime reload semantics that need maintainer design.

## What This Changes

The issue requests native Slack bot token rotation using Slack OAuth V2 refresh tokens, including config schema, refresh runtime, bootstrap CLI, and operational status support.

## Best Possible Solution

Implement this in the Slack plugin behind a reviewed SecretRef-aware credential persistence and reload design, with schema/docs/status/doctor coverage and serialized refresh for both Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence evidence that OpenClaw does not accept Slack token-rotation config, register refresh-token secrets, or run an OAuth refresh path today.

## Solution Assessment

Unclear pending maintainer/security design. The broad direction belongs in the Slack plugin, but raw config patching and SIGUSR1-style reload should be refined into an approved secret-persistence and runtime-reload contract.
