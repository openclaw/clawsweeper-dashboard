## Summary

Keep open for maintainer product and security design: current main and v2026.6.10 still model Slack credentials as static bot/app/signing/user tokens, while this issue asks for a new Slack OAuth refresh-token lifecycle with secure persistence and runtime token adoption.

## What This Changes

Add native Slack token rotation support using oauth.v2.access, including refresh-token storage, runtime refresh, setup/bootstrap, status, and failure handling.

## Best Possible Solution

Design a Slack-plugin-owned OAuth credential lifecycle first, then add transactional refresh, runtime token adoption, redacted status/recovery UX, docs, and live Slack proof while preserving existing static-token setups.
