## Summary

Keep open. Current main and v2026.5.12 still only support static Slack credential configuration, while the requested Slack rotation flow would add mutable OAuth credential storage and runtime refresh behavior that needs maintainer, security, and product review.

## What This Changes

The issue requests native Slack OAuth token rotation with config/secret storage, oauth.v2.exchange bootstrap, oauth.v2.access refresh, runtime update, and CLI/status support.

## Best Possible Solution

Implement this in the official Slack plugin with reviewed OAuth credential storage, exchange/refresh/status/docs coverage, and serialized atomic refresh plus Slack client rebuild behavior for Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Current source and v2026.5.12 inspection confirm the requested Slack token-rotation config, secret targets, refresh runtime, and setup/status command surface do not exist.

## Solution Assessment

Unclear pending maintainer/security design. The feature belongs in the official Slack plugin, but direct openclaw.json patching and SIGUSR1-style reload should be replaced by a reviewed credential-persistence and runtime-reload contract.
