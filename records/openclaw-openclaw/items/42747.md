## Summary

Keep open for maintainer product and security design: current main and the latest release still use static Slack token configuration, while Slack token rotation needs a refresh-token OAuth lifecycle; the related https://github.com/openclaw/openclaw/issues/80346 tracks reload reliability for manually changed static tokens, not native OAuth rotation.

## What This Changes

Request native Slack token rotation support, including refresh-token storage, proactive oauth.v2.access refresh, runtime token adoption, setup/status UX, and Socket Mode plus HTTP Mode coverage.

## Best Possible Solution

Design Slack-plugin-owned OAuth installation storage first, then implement transactional refresh, runtime token adoption, redacted status/recovery UX, docs, and live proof for both Socket Mode and HTTP Mode.

## Reproduction Assessment

Not applicable: this requests a new Slack OAuth credential lifecycle rather than a broken existing OpenClaw contract. Current source, docs, release inspection, and Slack's API contract show the capability is missing.

## Solution Assessment

No for the exact proposed openclaw.json mutation shape. Native Slack-plugin support is a reasonable direction, but managed rotation state should use secure transactional credential storage and explicit runtime adoption rather than periodic mutation of general config.

## Risks / Open Questions

- Slack refresh tokens are effectively one-use, so a crash after Slack accepts a refresh but before OpenClaw durably stores the replacement refresh token could strand the installation.
- The proposed openclaw.json fields would expand mutable sensitive config with client secrets, refresh tokens, and expiry state unless maintainers choose a narrower credential store.
- Socket Mode, HTTP Mode, inbound Bolt authorization, outbound Web API clients, probes, status, and reload behavior need one refreshed-token source of truth.
- The bootstrap path changes rollback behavior because the old long-lived token stops being the steady-state credential once rotation is enabled.
