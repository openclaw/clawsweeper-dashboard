## Summary

Keep open. Current main still has only static Slack credential handling, no Slack token-rotation config/schema, no refresh-token secret registry entries, and no runtime `oauth.v2.access` refresh path; the request is a real security-sensitive feature that needs maintainer design rather than cleanup closure.

## What This Changes

Add native Slack token rotation support for refreshing expiring Slack bot tokens with `oauth.v2.access`, including config, setup/status CLI, persistence, and runtime refresh behavior.

## Best Possible Solution

Implement this in the official Slack plugin with a reviewed SecretRef-aware credential store, schema/docs/status/doctor coverage, and serialized refresh plus client reload behavior for both Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence evidence that Slack token-rotation config, refresh-token secret targets, and OAuth refresh runtime are absent.

## Solution Assessment

Unclear pending maintainer/security design. The direction belongs in the official Slack plugin, but raw config patching and SIGUSR1-style reload should be refined into an approved secret-persistence and runtime-reload contract.
