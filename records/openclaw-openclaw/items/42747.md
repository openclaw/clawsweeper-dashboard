## Summary

Keep open: current main and v2026.6.10 still treat Slack credentials as static bot/app/user/signing tokens, while this request asks for a new Slack OAuth refresh-token lifecycle with security, storage, and runtime-adoption design work.

## What This Changes

Request native Slack token rotation so OpenClaw can exchange and refresh expiring Slack bot tokens with oauth.v2.access instead of relying on external config-patching daemons.

## Best Possible Solution

Design a Slack-plugin-owned OAuth rotation lifecycle with secure persistence, transactional refresh-token replacement, active runtime token adoption, redacted status/recovery UX, docs, and live Slack proof while preserving static-token setups.
