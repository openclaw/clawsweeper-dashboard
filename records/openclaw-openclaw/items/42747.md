## Summary

Keep open. Current main still treats Slack credentials as static bot/app/signing/user token inputs, while this request would add native Slack OAuth token rotation with mutable refresh-token storage, runtime refresh, setup/status UX, and security-sensitive persistence decisions.

## What This Changes

Add native Slack OAuth token rotation support using `oauth.v2.access` refresh-token flow, including config fields, setup/status commands, runtime refresh, and token persistence.

## Best Possible Solution

Implement this in the official Slack plugin with reviewed OAuth credential storage, exchange/refresh/status/docs coverage, and serialized refresh plus Slack client rebuild behavior for Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, and command-surface inspection confirm the requested Slack token-rotation config, secret targets, refresh runtime, and setup/status command surface do not exist on current main.

## Solution Assessment

Unclear pending maintainer/security design. The feature belongs in the official Slack plugin, but direct config-file patching and SIGUSR1-style reload should be replaced by a reviewed credential-persistence and runtime-reload contract.
