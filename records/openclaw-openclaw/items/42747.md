## Summary

Keep open. Current main still exposes static Slack credentials only, with no Slack token-rotation config shape, refresh-token secret registry, OAuth refresh runtime path, or setup/status CLI; implementing this is a security-sensitive feature that needs maintainer design.

## What This Changes

This issue requests native Slack OAuth token rotation for the Slack plugin, including refresh-token storage, proactive oauth.v2.access refresh, and setup/status support.

## Best Possible Solution

Implement this in the official Slack plugin with reviewed SecretRef-aware storage, schema/docs/status/doctor coverage, and serialized refresh/reload behavior for Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence evidence that Slack token-rotation config, refresh-token secret targets, and OAuth refresh runtime are absent.

## Solution Assessment

Unclear pending maintainer/security design. The broad direction belongs in the Slack plugin, but raw config patching and SIGUSR1-style reload should be refined into an approved secret-persistence and runtime-reload contract.
