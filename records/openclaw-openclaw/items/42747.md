## Summary

Keep open. Current main and the checked shipped tag still expose static Slack credentials only; this request adds sensitive OAuth persistence and runtime credential mutation, so it needs maintainer security/product design rather than cleanup closure.

## What This Changes

Request native Slack token rotation for the official Slack plugin using Slack's oauth.v2.access refresh-token flow.

## Best Possible Solution

Implement this in the official Slack plugin with reviewed SecretRef-aware credential storage, OAuth exchange/init/status/docs coverage, and serialized refresh/reload behavior for both Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms Slack token-rotation config, refresh-token secret targets, and oauth.v2.access runtime are absent.

## Solution Assessment

Unclear pending maintainer/security design. The feature direction belongs in the official Slack plugin, but raw config patching and SIGUSR1-style reload should be refined into an approved credential-persistence and runtime-reload contract.
