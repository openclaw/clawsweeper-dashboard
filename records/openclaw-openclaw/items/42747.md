## Summary

Keep open. Current main and v2026.5.22 still model Slack credentials as static bot/app/user/signing-secret inputs, while native Slack token rotation remains a new, security-sensitive Slack plugin capability that needs maintainer product and security design.

## What This Changes

The issue requests native Slack OAuth token rotation support, including refresh-token storage, oauth.v2.access refresh behavior, and setup/status UX.

## Best Possible Solution

Design and implement this inside the official Slack plugin with an explicit OAuth credential-persistence contract, atomic refresh/reload behavior, redaction/audit coverage, and focused bootstrap, refresh, failure, and status tests.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, canonical search, and v2026.5.22 inspection show the requested Slack OAuth token-rotation capability is absent rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer security and product design. The capability belongs in the Slack plugin, but the proposed direct openclaw.json mutation plus SIGUSR1-style reload should be replaced or formalized through a reviewed credential-persistence and provider-reload contract.
