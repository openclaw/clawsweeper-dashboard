## Summary

Keep open. Current main still exposes Slack as static bot/app/user token configuration, while this request adds a new security-sensitive OAuth rotation flow with mutable refresh-token persistence, proactive refresh, setup/status UX, and runtime Slack client rebuild behavior.

## What This Changes

Add native Slack OAuth token rotation so the Slack plugin can store refresh state, refresh expiring bot tokens, and expose setup/status UX.

## Best Possible Solution

Implement this inside the official Slack plugin with reviewed OAuth credential storage, serialized refresh, setup/status/docs coverage, and explicit Slack client rebuild behavior for Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a feature request. Source inspection and GitHub search provide high-confidence current-main proof that the requested Slack token-rotation config, refresh runtime, and CLI/status surface do not exist.

## Solution Assessment

Unclear pending maintainer and security design. The Slack plugin is the right owner, but direct openclaw.json patching and SIGUSR1-style reload should be replaced by a reviewed credential-persistence and runtime-rebuild contract.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because it would add mutable Slack OAuth refresh-token storage and automated runtime credential mutation.

Concerns:

- **[medium] Design mutable Slack OAuth credential storage:** `docs/reference/secretref-credential-surface.md:14`
