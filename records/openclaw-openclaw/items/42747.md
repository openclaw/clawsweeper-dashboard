## Summary

Keep open for maintainer product and security design. Current main and latest release still configure Slack with static bot/app/signing/user tokens, while Slack token rotation requires a refresh-token OAuth lifecycle; the related https://github.com/openclaw/openclaw/issues/80346 tracks static-token reload reliability, not native OAuth rotation. citeturn0view0

## What This Changes

Add Slack plugin support for OAuth token rotation, including refresh-token storage, proactive oauth.v2.access refresh, bootstrap/status UX, and runtime token adoption.

## Best Possible Solution

Design Slack-plugin-owned OAuth installation storage first, then implement transactional refresh, runtime token adoption, redacted status/recovery UX, and live proof for Socket and HTTP modes.

## Reproduction Assessment

Not applicable: this requests a new Slack OAuth credential lifecycle rather than a broken existing OpenClaw contract. Current source, docs, release inspection, and Slack’s official API contract show the capability is missing.

## Solution Assessment

No for the exact proposed config-mutation shape. Native Slack-plugin support is a reasonable direction, but managed rotation state should use secure transactional credential storage and explicit runtime adoption rather than periodic mutation of general config.
