## Summary

Keep open for maintainer product and security design: current main and v2026.6.8 still model Slack as static bot/app/signing/user tokens, while this asks for a new OAuth refresh-token lifecycle with secure persistence and runtime adoption.

## What This Changes

Add native Slack token rotation support that refreshes expiring bot tokens through Slack oauth.v2.access instead of relying on an external daemon.

## Best Possible Solution

Design a Slack-plugin-owned OAuth credential store first, then implement transactional refresh, runtime token adoption, redacted status/recovery UX, docs, and live proof for Socket Mode and HTTP mode.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new Slack OAuth credential lifecycle. Source, docs, shipped-tag inspection, and Slack docs confirm the capability is still missing.

## Solution Assessment

No for the exact proposed openclaw.json mutation shape. Native Slack rotation is a reasonable direction, but the durable implementation should first choose plugin-owned secure storage and runtime adoption semantics.
