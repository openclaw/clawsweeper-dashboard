## Summary

Keep open: current main appears to mitigate plain cron tool-warning recovery for Telegram/final-text paths, but the broader reported status/delivery contract is still not proven fixed, especially run-level fatal state and the inter-agent reply facet.

## What This Changes

The issue asks OpenClaw to deliver produced cron or inter-agent output even when only an ancillary backgrounded tool fails non-zero.

## Best Possible Solution

Define one cron and isolated-agent delivery contract that delivers valid final assistant output after non-terminal ancillary failures while preserving true fatal failure suppression and matching inter-agent reply semantics.

Unclear: the issue is valid, but a per-job option is not clearly the best fix. The safer direction is a maintainer-owned terminal-status and delivery contract with focused cron plus inter-agent coverage.
