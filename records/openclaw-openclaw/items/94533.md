## Summary

Keep open: current main still has source-level evidence that fatal run/prompt state can override produced cron output into error/not-delivered state, and v2026.6.8 contains the same relevant cron delivery behavior.

## What This Changes

The issue asks OpenClaw to preserve delivery of a produced cron or isolated-agent reply when only ancillary background/process failure state is unrelated to the deliverable.

## Best Possible Solution

Define one cron/isolated terminal-outcome contract that delivers valid final assistant output or source replies when only ancillary diagnostics failed, while preserving true fatal denials/errors and covering cron plus source-delivery tests.

## Reproduction Assessment

Not as a high-confidence live cron/Telegram reproduction in this review. Source inspection does reproduce the remaining status/delivery coupling on current main and v2026.6.8.

## Solution Assessment

Unclear as an implementation direction: the bug is valid, but the best fix is a maintainer-owned terminal-status and delivery contract rather than a narrow per-job option or one partial classifier change.
