## Summary

Keep open: current main still has the source-level path where a Codex failed turn, run-level error, or fatal failure signal can override produced assistant output and persist the cron run as error/not-delivered, and the only directly related PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to deliver cron or isolated-agent output when a non-final background tool fails after useful assistant output has already been produced.

## Best Possible Solution

Define one cron and isolated-agent delivery outcome contract that delivers valid final assistant output after ancillary non-terminal failures, preserves true fatal failures, and keeps inter-agent reply semantics consistent.

## Solution Assessment
