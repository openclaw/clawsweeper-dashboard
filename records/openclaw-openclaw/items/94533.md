## Summary

Keep open: current main and v2026.6.9 still have source-level cron outcome and delivery gates that can turn produced output into error/not-delivered state, and the related fix PRs are still open or closed unmerged rather than landed fixes.

## What This Changes

The issue asks OpenClaw to deliver valid cron or isolated-agent output even when an unrelated background/tool failure marks the final turn failed.

## Best Possible Solution

Define and land one maintainer-approved cron/isolated-agent delivery outcome contract that delivers valid final output after non-terminal ancillary failures, preserves true fatal-error suppression, and aligns inter-agent reply delivery semantics.

## Reproduction Assessment

Yes at source level: current main still propagates run-level/fatal classifier state into cron error status and not-delivered persistence, and the A2A path remains an open related surface. I did not run a fresh live scheduled background-process or inter-agent reproduction in this read-only review.

## Solution Assessment

Unclear: the bug is valid, but the best fix is a maintainer-owned terminal-status and delivery contract rather than a new per-job option or one narrow classifier tweak.
