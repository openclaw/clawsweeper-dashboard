Keep open: current main and v2026.6.8 still have the reported status/delivery coupling, and the overlapping open PRs do not yet settle the background-process and inter-agent delivery contract.

## What This Changes

Request to prevent scheduled isolated cron/announce runs from discarding produced assistant output when only an ancillary backgrounded exec/tool process exits non-zero.

## Best Possible Solution

Define and land a structured cron terminal-outcome contract that delivers valid final assistant output when only ancillary background diagnostics failed, while preserving true fatal run failures, denials, and explicit fail-on-tool-failure behavior.

## Reproduction Assessment

Yes, source-level: current main still propagates run-level errors or fatal failure signals into an error cron outcome, and error outcomes force requested primary delivery into not-delivered state. I did not run a live cron plus Telegram reproduction in this read-only review.

Unclear. A per-job option is not the best first step; maintainers should first settle the core terminal-status contract for ancillary background failures versus true fatal failures and reconcile the open related PRs.
