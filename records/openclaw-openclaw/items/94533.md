## Summary

Keep open: current main and v2026.6.8 still let cron run-level or fatal outcome state turn produced output into an error/not-delivered result, and the related PRs are open or only partial candidates rather than a merged fix for this issue.

## What This Changes

The issue requests that scheduled and isolated agent deliverables remain deliverable when only an ancillary background process fails, while true fatal failures remain explicit.

## Best Possible Solution

Define one isolated cron terminal-outcome contract that delivers valid final assistant payloads when only ancillary diagnostics failed, preserves true fatal run failures and denials, and reconciles the overlapping recovered-tool and fail-on-tool-failure work.

## Reproduction Assessment

Yes at source level: current main still propagates run-level or fatal classifier output into cron error status and not-delivered state, but I did not run a fresh live scheduled background-process repro.

## Solution Assessment

Unclear as an implementation direction: the issue is real, but the best fix is a maintainer-owned status/delivery contract rather than simply adding a per-job option or merging one partial classifier change.
