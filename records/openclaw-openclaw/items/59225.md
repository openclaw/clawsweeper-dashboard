## Summary

Keep open. Current main still lacks per-agent parameter allow/deny filtering for model-visible tool schemas; the merged work only made ACP-only `sessions_spawn` fields runtime-tolerant, and the related open schema-stripping PR is narrower than this public per-agent config contract.

## What This Changes

The issue requests per-agent allow/deny filtering of individual tool schema parameters before model-visible schemas are sent to subagents.

## Best Possible Solution

Define a generic schema-visibility layer after effective tool selection and before provider schema normalization, then document and test the per-agent config contract across `sessions_spawn` and at least one generic or plugin-owned tool.

## Reproduction Assessment

Yes. Source inspection shows current main still advertises the broad `sessions_spawn` schema and the strict per-agent/global tools config has no parameter-filter fields, so the requested contract is absent without needing a live model run.

## Solution Assessment

Unclear. `allowedParams` or `deniedParams` is plausible, but the best fix needs maintainer agreement on config naming, required-field handling, precedence, and generic/plugin schema semantics before implementation.
