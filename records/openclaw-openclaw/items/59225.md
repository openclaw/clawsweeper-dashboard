## Summary

Keep open. Current main has narrowed the ACP-only `sessions_spawn` failure mode, but it still does not provide per-agent parameter allow/deny filtering for advertised tool schemas; solving this would add a new public config contract that needs maintainer API judgment.

## What This Changes

Request a per-agent config surface that hides selected tool schema parameters, especially for `sessions_spawn`, before schemas are sent to a model.

## Best Possible Solution

Define a generic schema-visibility layer after effective tool selection and before provider schema normalization, then document and test the per-agent config contract across `sessions_spawn` and at least one generic or plugin-style tool.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is that current main still builds and advertises the broad `sessions_spawn` schema, while the strict per-agent tools config has no accepted parameter-filter fields.

## Solution Assessment

Unclear. `allowedParams` or `deniedParams` is plausible, but the best fix needs maintainer agreement on the public config name, required-field behavior, and generic/plugin schema semantics before implementation.
