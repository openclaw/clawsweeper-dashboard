## Summary

Keep open: current main and the latest release still lack a per-agent or global allow/deny mechanism for individual model-visible tool schema parameters; the related `sessions_spawn` fixes only narrow availability or execution tolerance.

## What This Changes

The issue requests per-agent filtering of model-visible tool schema parameters so subagents can receive slimmed `sessions_spawn` definitions.

## Best Possible Solution

Define a generic schema-visibility layer after effective tool selection and before provider schema normalization, then document and test the chosen per-agent/global contract across `sessions_spawn` and at least one generic or plugin-owned tool.

## Reproduction Assessment

Yes for the missing contract: current main and `v2026.5.22` still advertise broad `sessions_spawn` parameters and the strict config schema has no `allowedParams`/`deniedParams` equivalent. I did not live-reproduce the Grok/MiMo auto-fill behavior, but the source path proves the model-visible filtering capability is absent.

## Solution Assessment

Unclear. `allowedParams` or `deniedParams` is plausible, but maintainers should choose the public naming, required-field behavior, precedence, and plugin/provider schema semantics before implementation.
