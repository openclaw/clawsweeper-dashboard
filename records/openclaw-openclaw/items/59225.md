## Summary

Keep open. Current main has the narrower `sessions_spawn` hardening for ACP-only fields, but it does not implement per-agent parameter allow/deny filtering for advertised tool schemas; the current config and docs still support tool-level allow/deny, not schema-property filtering.

## What This Changes

The issue requests per-agent configuration to hide selected `sessions_spawn` parameters from the tool schema sent to subagents/models.

## Best Possible Solution

Add a generic schema-visibility layer after effective tool selection and before provider schema normalization, backed by documented per-agent config, generated schema/help updates, and focused tests for `sessions_spawn` plus at least one generic/plugin-style tool schema.

## Reproduction Assessment

Yes. A code-level reproduction is high confidence: current main constructs `sessions_spawn` with the broad optional schema and the strict per-agent tools schema has no accepted parameter-filter config.

## Solution Assessment

Unclear. The requested `allowedParams`/`deniedParams` direction is plausible, but the exact public config shape and required-field behavior should be settled before implementation; the existing ACP-only runtime workaround is not a complete solution for this feature.
