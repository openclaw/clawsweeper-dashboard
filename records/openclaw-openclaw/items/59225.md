## Summary

Keep open. Current main still lacks a per-agent allow/deny mechanism for individual model-visible tool schema parameters; the merged `sessions_spawn` work only made ACP-only fields runtime-tolerant, and the open schema-stripping PR is narrower than this public config contract.

## What This Changes

Add per-agent config to allow or deny individual model-visible parameters for tools such as `sessions_spawn` so subagents can receive slimmer schemas.

## Best Possible Solution

Define a generic schema-visibility layer after effective tool selection and before provider schema normalization, then document and test the per-agent contract across `sessions_spawn` and at least one generic or plugin-owned tool.

## Reproduction Assessment

Yes. Source inspection shows current main still advertises broad `sessions_spawn` parameters and the strict per-agent/global tool config has no parameter-filter fields, so the requested contract is absent without a live model run.

## Solution Assessment

Unclear. `allowedParams` or `deniedParams` is plausible, but the best fix needs maintainer agreement on naming, required-field handling, precedence, and generic/plugin schema behavior before implementation.
