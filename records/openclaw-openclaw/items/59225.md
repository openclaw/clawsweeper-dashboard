## Summary

Keep open. Current main only hardened the ACP-only `sessions_spawn` runtime failure; it still has no per-agent parameter allow/deny filtering for model-visible tool schemas, and adding that would create a new public config contract.

## What This Changes

The issue requests per-agent allow/deny filtering of tool schema parameters so dispatcher-style agents can see slimmer `sessions_spawn` definitions.

## Best Possible Solution

Define a generic schema-visibility layer after effective tool selection and before provider schema normalization, then document and test the per-agent config contract across `sessions_spawn` and at least one generic or plugin-owned tool.

## Reproduction Assessment

Yes. Source inspection shows current main still advertises a broad `sessions_spawn` schema and the strict per-agent tools config has no accepted parameter-filter fields; I did not need a live model run to verify that missing contract.

## Solution Assessment

Unclear. `allowedParams` or `deniedParams` is plausible, but the best fix needs maintainer agreement on config naming, required-field handling, and generic/plugin schema semantics before implementation.
