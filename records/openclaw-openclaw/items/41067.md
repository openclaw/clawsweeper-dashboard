## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Filter active runs before exposing them:** `src/gateway/server/ws-connection/message-handler.ts:104-109`
  - body: `listActiveChatRuns` publishes every abort-controller entry into the hello snapshot. Current main's active-run helpers filter out non-Control-UI-visible and non-project-active runs; without the same filter, reconnect can recover a hidden/side-result/internal run and attach the chat UI to the wrong lifecycle.
  - confidence: 0.86
- **[P1] Select the newest valid active run:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching entry, but the server tracks active runs by run id and can have more than one entry for a session. Recovering an older run id makes later deltas/finals from the newer run fail the runId checks, leaving the chat stuck or stale.
  - confidence: 0.84
- **[P2] Return false when transcript injection fails:** `src/gateway/server-restart-sentinel.ts:32-33`
  - body: If `ensureGatewayTranscriptFile` or transcript append throws, this helper never returns `false`, so the callers skip the `enqueueSystemEvent` fallback after consuming the restart sentinel. Catch injection errors inside the helper and return `false` so the fallback notice is still queued.
  - confidence: 0.82
