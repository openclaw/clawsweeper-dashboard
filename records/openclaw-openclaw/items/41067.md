## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Filter active runs before exposing them:** `src/gateway/server/ws-connection/message-handler.ts:100-111`
  - body: `listActiveChatRuns` publishes every abort-controller entry into the hello snapshot, but current main's active-run helpers filter out non-Control-UI-visible and non-project-active runs. Without the same filter, reconnect can recover a hidden or side-result run and attach Dashboard state to the wrong lifecycle.
  - confidence: 0.86
- **[P2] Select the newest valid active run:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching entry, but active runs are tracked by run id and a session can have more than one entry. Recovering an older run id makes later deltas/finals from the newer run fail the runId checks, leaving chat stuck or stale.
  - confidence: 0.84
- **[P2] Return false when transcript injection fails:** `src/gateway/server-restart-sentinel.ts:32-33`
  - body: If `ensureGatewayTranscriptFile` throws, this helper never returns `false`, so callers skip the `enqueueSystemEvent` fallback after consuming the restart sentinel. Catch injection errors here and return `false` so the fallback notice is still queued.
  - confidence: 0.82
