## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Recover the newest active run for the session:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `activeChatRuns` is built from every `chatAbortControllers` entry, and current registration only rejects duplicate run IDs, not duplicate session keys. Returning the first matching session can restore an older run; later deltas/finals for the newer run are then ignored by `handleChatEvent` because the run ID no longer matches. Prefer the existing matching run when present, otherwise choose the newest `startedAtMs` for the session.
  - confidence: 0.88
- **[P1] Preserve the fallback when transcript injection throws:** `src/gateway/server-restart-sentinel.ts:32`
  - body: `injectRestartNoticeIntoSession` can throw before it returns `false`, for example when transcript file creation fails. In the no-route paths the sentinel has already been consumed, so the caller skips `enqueueSystemEvent(...)` and the restart notice is lost; catch helper errors and return `false` so the fallback path still runs.
  - confidence: 0.9
