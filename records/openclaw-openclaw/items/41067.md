## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve the fallback when transcript injection fails:** `src/gateway/server-restart-sentinel.ts:32`
  - body: `injectRestartNoticeIntoSession` is used as a boolean fallback guard after the restart sentinel has been consumed. If `ensureGatewayTranscriptFile` or transcript append setup throws, the exception exits the wake path and `enqueueSystemEvent(...)` is skipped, so the user gets neither a transcript notice nor the fallback system event. Catch helper errors and return `false` so the existing fallback still runs.
  - confidence: 0.9
- **[P2] Prefer the current or newest active run:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching active run from the snapshot. Because the server snapshots active runs by run id, multiple entries can exist for one session; choosing an older one makes the UI adopt a stale `chatRunId` and then ignore deltas/finals from the newer run. Prefer the existing host run when present, otherwise choose the highest `startedAtMs`.
  - confidence: 0.78
