  - body: This loop now aborts every active chat run and injects a restart-style transcript notice on any close path, including normal shutdowns with no restartExpectedMs. Current main intentionally gates active-run abort cleanup behind restart reply drain and filters agent-kind runs, so carrying this stale implementation forward would misreport shutdowns and disrupt current restart semantics.

  - body: injectRestartNoticeIntoSession can throw from ensureGatewayTranscriptFile, but callers only enqueue the fallback system event when the helper returns false. If file creation or path access fails after the sentinel is consumed, the recovery notice is dropped instead of being queued through the fallback path.
  - confidence: 0.9
- **[P2] Choose a deterministic active run on reconnect:** `ui/src/ui/app-gateway.ts:126-136`
  - body: resolveActiveChatRunForSession returns the first matching run from a run-id-keyed snapshot. When a session has more than one active run entry, reconnect can restore an older run id and then ignore deltas or finals for the newer run due to run-id mismatch; keep the current run when still active or choose the newest startedAtMs.
