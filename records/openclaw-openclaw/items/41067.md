- **[P1] Filter active runs before exposing them:** `src/gateway/server/ws-connection/message-handler.ts:100-111`
  - body: `listActiveChatRuns` publishes every abort-controller entry into the hello snapshot, but current main filters out non-Control-UI-visible and non-project-active runs.   - confidence: 0.86

  - body: `resolveActiveChatRunForSession` returns the first matching entry, but active runs are tracked by run id and a session can have more than one entry. Recovering an older run id makes later deltas/finals from the newer run fail run-id checks, leaving chat stuck or stale.
  - confidence: 0.84

  - confidence: 0.82
