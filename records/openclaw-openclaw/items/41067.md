## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Filter recovered chat runs before restoring state:** `src/gateway/server/ws-connection/message-handler.ts:104-110`
  - body: `listActiveChatRuns` serializes every abort-controller entry, while current gateway session APIs only expose Control UI-visible active runs. Combined with the UI returning the first matching session entry, reconnect can restore a hidden, inactive, or older run for the selected session and make Stop/Queue target the wrong work. Filter to the visible active-run contract and choose the current/newest matching run before setting `chatRunId`.
  - confidence: 0.87
- **[P1] Catch transcript injection failures before fallback:** `src/gateway/server-restart-sentinel.ts:32-33`
  - body: `injectRestartNoticeIntoSession` is awaited inside the condition that enqueues the fallback system event. If `ensureGatewayTranscriptFile` throws on mkdir/access/write, the helper never returns `false`, so the fallback is skipped after the sentinel has been consumed and the restart notice can still disappear. Catch helper errors and return `false` so the existing fallback path runs.
  - confidence: 0.9
