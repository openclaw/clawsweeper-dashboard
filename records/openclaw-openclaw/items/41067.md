## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Select recovered runs deterministically:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `activeChatRuns` is built from every chat abort controller entry, so one session can have multiple active run IDs. Returning the first matching entry can restore an older run and make later events for the newer run look unrelated; keep the current run when present or choose the newest `startedAtMs`.
  - confidence: 0.9
- **[P2] Preserve restart fallback on injection errors:** `src/gateway/server-restart-sentinel.ts:32-39`
  - body: `injectRestartNoticeIntoSession` can reject before it returns `false`, but callers only enqueue the fallback system event when it returns `false`. A transcript path or filesystem error can consume the sentinel and drop the visible restart notice; catch helper errors and return `false`.
  - confidence: 0.9
- **[P2] Use the current transcript append contract:** `src/gateway/server-methods/chat-transcript-file.ts:3`
  - body: Current main uses the current transcript append helper and package contract, but this branch adds a helper importing the old package name and duplicating header creation. Rebase this onto `appendInjectedAssistantMessageToTranscript`/`appendSessionTranscriptMessage` so it does not fail module resolution or drift from parentId-safe transcript writes.
  - confidence: 0.86
