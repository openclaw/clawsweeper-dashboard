- **[P1] Preserve fallback when transcript setup fails:** `src/gateway/server-restart-sentinel.ts:54-55`
  - body: `injectRestartNoticeIntoSession` can throw while ensuring the transcript file, but callers only enqueue the fallback system event when the helper returns `false`. If file setup throws after the sentinel is consumed, the restart notice is dropped instead of falling back, so catch helper errors and return `false` for the fallback path.
  - confidence: 0.88
- **[P2] Choose a deterministic active run for the session:** `ui/src/ui/app-gateway.ts:126-137`
  - body: `resolveActiveChatRunForSession` returns the first matching activeChatRuns entry, but the server can track multiple active entries for the same session. If the UI adopts an older runId, later deltas/finals for the newer run are ignored, so select the newest matching run with a deterministic tie-break.
  - confidence: 0.84
