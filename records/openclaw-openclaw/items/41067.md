## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve fallback when transcript injection fails:** `src/gateway/server-restart-sentinel.ts:32`
  - body: `injectRestartNoticeIntoSession` is used to decide whether to enqueue the system-event fallback, but `ensureGatewayTranscriptFile` can reject before the helper returns `false`. In the no-route branches this means the restart sentinel has already been consumed and the fallback notice is never queued; catch injection errors inside the helper and return `false`.
  - confidence: 0.9
- **[P2] Recover the newest matching active run:** `ui/src/ui/app-gateway.ts:126-136`
  - body: The reconnect helper returns the first activeChatRuns entry for a session. The server tracks active runs by runId, so a retry or stale-controller race can leave multiple entries for the same session; adopting an older run makes later events for the newer run fail the runId check and leaves the chat stuck busy or stale. Choose the newest matching run, or preserve an existing matching run when present.
  - confidence: 0.83
