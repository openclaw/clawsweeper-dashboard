## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Filter active-run snapshots before restoring chat state:** `src/gateway/server/ws-connection/message-handler.ts:100-111`
  - body: `listActiveChatRuns` serializes every chatAbortControllers entry, including hidden agent runs, entries no longer projected active, and stale older runs. The UI then returns the first session match, so reconnect can adopt the wrong run and show Stop/Queue for work that is not the visible active chat run. Reuse the same filtering/newest-run contract as the current active-run/session projection.
  - confidence: 0.9
- **[P1] Only abort active chat runs on restart shutdown:** `src/gateway/server-close.ts:124-138`
  - body: This close handler aborts every tracked chat run before broadcasting shutdown, regardless of whether the close is a restart. Current main deliberately keeps normal shutdown from aborting active runs, so this would turn ordinary SIGTERM/close paths into interrupted runs with restart notices.
  - confidence: 0.88
- **[P1] Keep the fallback when transcript injection fails:** `src/gateway/server-restart-sentinel.ts:83-84`
  - body: `injectRestartNoticeIntoSession` can throw from session loading/path resolution/file creation before it returns `false`. Because the caller awaits it inside the condition, a failure skips `enqueueSystemEvent` after the sentinel has been consumed, dropping the restart notice instead of falling back.
  - confidence: 0.86
