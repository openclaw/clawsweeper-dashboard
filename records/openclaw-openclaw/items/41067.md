  - body: This loop aborts every active chat run and writes a gateway restart notice for every close path. Normal shutdown should not create restart-interruption transcript notices or shutdown abort events unless the close is actually a restart.
  - confidence: 0.87

  - body: `activeChatRuns` is built from every abort-controller entry. Current main filters hidden/background and terminal-guard runs before projecting Control UI-visible active session state, so this snapshot can make the UI adopt a run whose events it should not receive.
  - confidence: 0.86
- **[P1] Fall back when transcript setup fails:** `src/gateway/server-restart-sentinel.ts:32`

  - confidence: 0.84
