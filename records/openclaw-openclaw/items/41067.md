## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose the recovered chat run deterministically:** `ui/src/ui/app-gateway.ts:126-136`
  - body: activeChatRuns can contain multiple entries for the same session because the server snapshots every run-id entry from chatAbortControllers. Returning the first matching entry can recover an older run, causing later events for the newer run to fail the run-id guard and leave stale busy/stream state; prefer the current host run when present, otherwise choose the highest startedAtMs.
  - confidence: 0.9
- **[P2] Preserve the restart-notice fallback on injection errors:** `src/gateway/server-restart-sentinel.ts:22-39`
  - body: injectRestartNoticeIntoSession can throw from session lookup, path resolution, file creation, or transcript append, but callers enqueue the fallback only when it returns false. Catch helper errors and return false so a consumed sentinel does not lose the user-visible restart notice.
  - confidence: 0.92
