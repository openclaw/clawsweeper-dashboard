## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P1] Filter active runs before exposing them:** `src/gateway/server/ws-connection/message-handler.ts:104-110`
  - body: `listActiveChatRuns` serializes every abort-controller entry into the hello snapshot. Current session projections filter out inactive, hidden, aborted, and internal agent runs before showing a session as active; without the same filter here, the Control UI can recover a run it will never receive terminal events for and stay stuck on the wrong busy state.
  - confidence: 0.86
- **[P2] Select the newest valid active run:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching run for the session. If two entries exist for the same session, reconnect can adopt an older run id and then ignore later deltas/finals for the newer run, so this should pick the newest valid run deterministically or preserve an existing matching run.
  - confidence: 0.83
- **[P2] Return false when transcript setup fails:** `src/gateway/server-restart-sentinel.ts:32-33`
  - body: `injectRestartNoticeIntoSession` can throw before it returns `false`, for example while creating or opening the transcript file. The caller only enqueues the fallback system event when the helper returns `false`, so a filesystem/setup error can consume the restart sentinel and drop the user-visible notice.
  - confidence: 0.78
