## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Fall back when transcript injection setup fails:** `src/gateway/server-restart-sentinel.ts:29`
  - body: `injectRestartNoticeIntoSession` can still throw before returning `false`, for example from `ensureGatewayTranscriptFile` on filesystem errors. Because `scheduleRestartSentinelWake` already consumed the sentinel before this call, the existing `enqueueSystemEvent` fallback is skipped and the restart notice is lost. Catch setup/injection failures inside this helper and return `false` so the fallback path still runs.
  - confidence: 0.86
- **[P2] Pick the recovered run deterministically:** `ui/src/ui/app-gateway.ts:122-131`
  - body: `listActiveChatRuns` sends every active controller, but `resolveActiveChatRunForSession` returns the first matching entry. Retry/restart races can leave multiple active entries for one session, so reconnect can adopt an older run and then ignore newer deltas/finals due to runId mismatch. Prefer the existing matching run when present, otherwise choose the newest `startedAtMs` with a deterministic tie-break.
  - confidence: 0.83
