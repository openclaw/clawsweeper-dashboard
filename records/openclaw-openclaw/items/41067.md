## Summary

Keep open: current main still has the dashboard reconnect gap, and this PR is useful source work, but it needs changes before merge because it can recover the wrong active run and can drop restart notices when transcript injection throws.

## What This Changes

The PR adds active chat run metadata to Gateway hello snapshots, Control UI reconnect/busy-state recovery, sidebar chat session preservation, shutdown/restart transcript notices, generated protocol artifacts, and tests.

## Best Possible Solution

Land a rebased implementation that exposes active dashboard chat runs in the additive hello snapshot, restores the correct run deterministically, keeps controls busy while abort is available, preserves restart-notice fallback delivery, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes, from source inspection and the linked issue steps. Current main clears chat run state on hello, does not expose active chat runs in the snapshot, and computes chat busy state without canAbort.

## Solution Assessment

No, not as currently proposed. The direction is maintainable, but the branch needs deterministic active-run selection, exception-safe fallback for transcript injection, a changelog entry, and rebase work against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose the recovered chat run deterministically:** `ui/src/ui/app-gateway.ts:126-136`
  - body: activeChatRuns can contain multiple entries for the same session because the server tracks active runs by runId. Returning the first matching entry can recover an older run; later events for the newer run then fail the run-id guard and leave stale busy or stream state. Prefer the existing host run if still active, otherwise choose the newest startedAtMs.
  - confidence: 0.9
- **[P2] Return fallback when transcript injection fails:** `src/gateway/server-restart-sentinel.ts:22-39`
  - body: injectRestartNoticeIntoSession can throw from session lookup, transcript path resolution, file creation, or append. Because callers only enqueue the system-event fallback when this helper returns false, an exception after the sentinel is consumed drops the restart notice. Catch helper errors and return false so the fallback still runs.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `ui/src/ui/views/chat.ts:243-244`
  - body: This is a user-facing Control UI/Gateway reliability fix, but the PR does not update CHANGELOG.md. Add a single-line entry under Unreleased Fixes; do not force a bot or maintainer handle into the thanks text.
  - confidence: 0.86
