- **[P1] Keep restart-sentinel fallback on transcript failures:** `src/gateway/server-restart-sentinel.ts:32`
  - body: `scheduleRestartSentinelWake` consumes the sentinel before this helper runs, and callers only enqueue the system-event fallback when the helper returns `false`.   - confidence: 0.88

  - body: The server snapshots every active controller, so a session can have more than one matching entry during retry/restart races. Returning the first entry can recover an older run id, after which chat events for the newer active run are ignored as unrelated; select the newest matching run, with the same visible-run/session scoping used by current main's in-flight run helper.
  - confidence: 0.82
