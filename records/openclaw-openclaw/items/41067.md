## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose the recovered chat run deterministically:** `ui/src/ui/app-gateway.ts:126-136`
  - body: activeChatRuns can contain multiple entries for the same session because the server snapshots every run-id entry from chatAbortControllers. Returning the first match can recover an older run, causing events for the newer run to fail the run-id guard and leave stale busy state; keep the current host run when present, otherwise choose the newest startedAtMs.
  - confidence: 0.9
- **[P2] Preserve fallback when restart notice injection fails:** `src/gateway/server-restart-sentinel.ts:22-39`
  - body: injectRestartNoticeIntoSession can throw from session lookup, path resolution, file creation, or transcript append, but callers only enqueue the fallback system event when it returns false. Catch helper errors and return false so a consumed sentinel does not lose the user-visible restart notice.
  - confidence: 0.92
