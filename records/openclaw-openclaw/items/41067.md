## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate restart abort cleanup to restart closes:** `src/gateway/server-close.ts:124-147`
  - body: This loop runs for every gateway close, including normal SIGTERM/SIGINT shutdowns where `restartExpectedMs` is null, and it writes a durable "gateway restart" notice for each active run. Keep this abort/notice path behind the restart close path so ordinary shutdowns do not mislabel or prematurely clear active chat state.
  - confidence: 0.87
- **[P1] Fall back when transcript injection setup fails:** `src/gateway/server-restart-sentinel.ts:32-33`
  - body: `injectRestartNoticeIntoSession` can throw from `ensureGatewayTranscriptFile`, but callers only enqueue the fallback system event when the helper returns `false`. Catch setup/injection failures and return `false` so a consumed restart sentinel still produces a visible fallback notice.
  - confidence: 0.86
- **[P2] Pick the recovered run deterministically:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching entry even though the server can expose multiple active runs for one session. Prefer an existing matching run or the newest `startedAtMs` with a deterministic tie-break so reconnect does not adopt an older run and ignore newer deltas/finals.
  - confidence: 0.83
