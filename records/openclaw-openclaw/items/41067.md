- **[P1] Gate restart aborts to restart shutdowns:** `src/gateway/server-close.ts:125-147`
  - body: This loop now aborts every active chat run and injects a "gateway restart" transcript notice on any close path, including normal SIGTERM/no `restartExpectedMs`, and it also sweeps agent-kind runs. Current main intentionally limits this to restart-drain cleanup and filters agent runs, so this stale implementation would misreport normal shutdowns and disrupt current shutdown semantics if carried forward.
  - confidence: 0.89
- **[P1] Fallback when transcript injection throws:** `src/gateway/server-restart-sentinel.ts:32-33`
  - body: `injectRestartNoticeIntoSession` can throw from `ensureGatewayTranscriptFile`, but callers only enqueue the fallback system event when the helper returns `false`. If file creation/path access fails after the sentinel is consumed, the recovery notice is dropped instead of being queued through the fallback path.
  - confidence: 0.9
- **[P2] Choose a deterministic active run on reconnect:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching run from a run-id-keyed map snapshot. When a session has more than one active run entry, reconnect can restore an older run id and then ignore deltas/finals for the newer run due to run-id mismatch; keep the current run when still active or choose the newest `startedAtMs`.
  - confidence: 0.86
