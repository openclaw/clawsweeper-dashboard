## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Filter active runs before exposing them:** `src/gateway/server/ws-connection/message-handler.ts:104-110`
  - body: `listActiveChatRuns` serializes every abort-controller entry, but current visible session projections filter out inactive, hidden, aborted, and internal agent runs. If the UI adopts one of those entries, it can recover a run that will not receive normal chat events and leave the session stuck or mis-associated.
  - confidence: 0.9
- **[P2] Select the newest recovered active run:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching run for a session. Current main's in-flight snapshot resolver deliberately chooses the newest `startedAtMs` with a deterministic tie-break, so this can adopt stale run IDs and drop newer deltas or finals.
  - confidence: 0.87
- **[P2] Return false when transcript setup fails:** `src/gateway/server-restart-sentinel.ts:32-33`
  - body: `injectRestartNoticeIntoSession` can throw before returning `false`, for example from transcript file setup, but callers only enqueue the fallback system event when the helper returns false. Catch setup/injection errors and return false so a consumed restart sentinel still has a delivery fallback.
  - confidence: 0.82
