- **[P1] Gate restart abort cleanup to restart closes:** `src/gateway/server-close.ts:124-147`
  - body: This loop aborts every active chat run and writes a gateway restart notice for every close path. Normal shutdown should not create restart-interruption transcript notices or shutdown abort events unless the close is actually a restart.
  - confidence: 0.87
- **[P1] Filter active-run snapshots before sending hello:** `src/gateway/server/ws-connection/message-handler.ts:1038-1040`
  - body: `activeChatRuns` is built from every abort-controller entry. Current main filters hidden/background and terminal-projected runs before projecting Control UI-visible active session state, so this snapshot can make the UI adopt a run whose events it should not receive.
  - confidence: 0.86
- **[P1] Fall back when transcript setup fails:** `src/gateway/server-restart-sentinel.ts:32`
  - body: `injectRestartNoticeIntoSession` can throw before returning false, but callers only enqueue the fallback system event on a false result. Catch setup errors so a consumed restart sentinel notice is not dropped.
  - confidence: 0.84
- **[P2] Pick the recovered run deterministically:** `ui/src/ui/app-gateway.ts:126-136`
  - body: `resolveActiveChatRunForSession` returns the first matching run. If multiple active entries exist for the same session, the UI can adopt an older run and ignore newer deltas or finals by run-id mismatch.
  - confidence: 0.8
