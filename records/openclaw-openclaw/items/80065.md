## Summary

Keep open: current main has the underlying auth cooldown config, runtime rotation caps, docs, generic config labels/help, CLI profile listing, and a provider-level Model Auth card, but it still lacks the requested configure/dashboard profile-count controls, per-profile cooldown reason/status in the dashboard payload, and an authorized clear-cooldown UI action.

## What This Changes

Expose auth-profile rotation settings, per-profile cooldown state, a clear-cooldown action, and inline cascade help in the configure wizard or dashboard.

## Best Possible Solution

Add a designed auth-profile management surface that exposes rotation limits with provider profile counts, includes cooldownUntil/cooldownReason in the status data path, and provides an explicit authorized clear-cooldown mutation if maintainers approve it.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing config and runtime state, not broken documented behavior. Source inspection confirms the underlying config and state exist while the requested dashboard/wizard and mutation surfaces remain absent.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending models.authStatus is the narrowest likely read path, but clear-cooldown should be an explicit authorized gateway mutation rather than hidden generic config or state-file editing.
