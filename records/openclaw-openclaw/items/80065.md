## Summary

Keep open: current main has the underlying auth cooldown config, runtime rotation limits, schema/help metadata, CLI cooldown listing, and a provider-level Model Auth dashboard card, but it still lacks the requested purpose-built configure/dashboard UX, per-profile cooldown fields in the dashboard payload, and an authorized clear-cooldown mutation.

## What This Changes

Expose existing `auth.cooldowns.*ProfileRotations` settings, per-profile cooldown state, clear-cooldown action, and inline cascade help in `openclaw configure` and/or the dashboard auth UI.

## Best Possible Solution

Add a designed auth-profile management surface that extends the status payload with cooldown metadata, exposes rotation limits with effective profile counts, and adds an explicitly authorized clear-cooldown mutation if maintainers approve the semantics.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing config and runtime state, not broken documented behavior. Source inspection confirms the config and state plumbing exist while the requested dashboard/configure controls and mutation do not.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending `models.authStatus` is the narrowest likely read path, but clearing cooldowns should be an explicit authorized gateway mutation rather than hidden state-file editing.
