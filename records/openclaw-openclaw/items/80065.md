## Summary

Keep open: current main has the cooldown config, runtime rotation behavior, docs, CLI cooldown visibility, and a provider-level Model Auth card, but it still lacks the requested per-profile dashboard cooldown data, rotation-count UI with profile counts, and an authorized clear-cooldown UI action. The related open config-UI PR only improves generic config discoverability and does not cover the auth-profile management request.

## What This Changes

The issue requests configure-wizard or dashboard exposure for auth profile-rotation limits, per-profile cooldown indicators, clear-cooldown controls, and inline failover help.

## Best Possible Solution

Add a designed auth-profile management surface that shows rotation limits with provider profile counts, includes cooldownUntil/cooldownReason in the dashboard status path, and provides an operator-authorized clear-cooldown mutation if approved.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing config and runtime state, not a broken documented behavior. Source inspection confirms the underlying knobs and state exist while the requested dashboard/configure surfaces remain absent.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending models.authStatus is the narrowest likely read path, but clear-cooldown should be an explicit authorized gateway mutation rather than raw state-file editing or a hidden generic config side effect.
