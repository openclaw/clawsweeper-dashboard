## Summary

Keep open: current main has the auth cooldown config, runtime behavior, docs, CLI inspection, and a low-level clear helper, but it still lacks the requested configure/dashboard controls, cooldown fields in the dashboard auth-status payload, and an authorized per-profile clear-cooldown UI action.

## What This Changes

The issue asks to expose existing auth profile-rotation cooldown settings and per-profile cooldown state in the configure wizard or dashboard UI.

## Best Possible Solution

Add a designed auth-profile management surface that exposes rotation limits with profile counts, includes cooldownUntil/cooldownReason in the status data path, and provides an explicit authorized clear-cooldown mutation if maintainers approve it.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing config and runtime state, not a broken documented behavior. Source inspection on current main confirms the underlying data exists while the requested dashboard/configure surfaces are absent.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending models.authStatus is the narrowest likely read path, but clear-cooldown should be an explicit authorized gateway mutation rather than hidden generic config or state-file editing.
