## Summary

Keep open: this is a current UI/UX feature request, not an obsolete or already-implemented bug report. Current main has the underlying auth cooldown config, runtime behavior, docs, CLI/doctor visibility, and a low-level clear helper, but it still lacks the requested dashboard/configure controls, per-profile cooldown status in `models.authStatus`, and a per-profile clear-cooldown gateway/UI action.

## What This Changes

Expose existing auth cooldown rotation limits and per-profile cooldown state in `openclaw configure` and/or the dashboard, including a per-profile clear-cooldown control.

## Best Possible Solution

Add a designed auth-profile status surface that extends `models.authStatus` with cooldown metadata and exposes existing rotation knobs with profile counts, then add a narrowly authorized clear-cooldown mutation only if maintainers approve that control.

## Reproduction Assessment

Not applicable: this is a feature request for new UI exposure of existing runtime/config state, not a broken documented behavior. Source inspection confirms the data and knobs exist while the requested dashboard/configure controls are absent on current main.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending `models.authStatus` is the narrowest likely data path, but the clear-cooldown button should be designed as an explicit gateway mutation rather than hidden in generic config editing.
