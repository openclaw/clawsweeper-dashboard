## Summary

Keep open: current main has the auth cooldown config, runtime behavior, docs, CLI inspection, and an Overview auth card, but it still lacks the requested configure/dashboard controls, cooldown fields in the dashboard auth-status payload, and a per-profile clear-cooldown gateway/UI action. Targeted searches did not find a canonical duplicate or merged UI follow-up that owns this request.

## What This Changes

Expose existing auth cooldown rotation settings and per-profile cooldown state in the configure/dashboard UI, including a per-profile clear-cooldown action.

## Best Possible Solution

Add a designed auth-profile management surface that extends models.authStatus with cooldown metadata and profile counts, exposes the existing rotation knobs with cascade help, and adds an explicit authorized clear-cooldown mutation if maintainers approve it.
