## Summary

Keep open. Current main already has the auth cooldown config and runtime rotation behavior, plus CLI observability, but the requested dashboard/configure-wizard controls, per-profile cooldown indicators, and clear-cooldown UI are still missing and need product/authorization review.

## What This Changes

The issue requests UI or configure-wizard exposure for auth cooldown rotation limits, per-profile cooldown state, clear-cooldown actions, and cascade-order help.

## Best Possible Solution

Add an owner-reviewed settings/auth-profile surface that reuses the existing config schema and auth-store APIs, extends dashboard auth status with cooldown reason/expiry, and gates clear-cooldown as an explicit authorized mutation with focused tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for new UI/wizard exposure of existing auth cooldown and profile-state behavior. Source inspection confirms the config/runtime pieces exist while the requested UX surface does not.

## Solution Assessment

Unclear until maintainers choose the product surface. A narrow implementation should probably extend `models.authStatus` and dashboard/settings first, then decide whether `openclaw configure` should expose advanced auth cooldown tuning.
