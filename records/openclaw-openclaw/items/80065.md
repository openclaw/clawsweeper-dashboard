## Summary

Keep open: current main and the latest release still do not provide the requested profile-aware configure/dashboard workflow, even though the underlying cooldown config, runtime rotation behavior, CLI auth listing, and generic config metadata exist. The old `auth-state.json` wording is now obsolete because auth state is in the per-agent SQLite store, but the UX/product request remains valid.

## What This Changes

The issue requests exposing existing auth profile-rotation cooldown knobs, per-profile cooldown state, and a guarded clear-cooldown action in the configure wizard or dashboard settings UI.

## Best Possible Solution

Add a product-approved model-auth settings/status flow that reuses the existing config schema and `models.authStatus`, with any clear-cooldown action gated by an explicit auth-state mutation policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/UX request. Source inspection gives a high-confidence gap: current main has config/runtime/CLI status pieces but not the requested configure/dashboard controls or clear-cooldown action.

## Solution Assessment

Unclear until maintainers choose the UX and mutation policy. The maintainable path is to extend the existing config metadata and `models.authStatus`/Control UI flow rather than build a parallel auth-state editor.
