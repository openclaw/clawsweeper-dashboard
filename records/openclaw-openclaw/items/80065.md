## Summary

Keep open: current main still has the underlying auth cooldown config/runtime and a provider-level Model Auth card, but it does not expose rotation-count controls, per-profile cooldown reason/until fields in the dashboard payload, or an authorized clear-cooldown UI action.

## What This Changes

Request to expose existing auth-profile rotation limits, per-profile cooldown state, and a clear-cooldown action in the configure wizard or dashboard UI.

## Best Possible Solution

Add a designed auth-profile management surface that shows rotation limits with profile counts, carries cooldown state through models.authStatus or an equivalent safe read API, and exposes clear-cooldown only through an explicit authorized gateway mutation.
