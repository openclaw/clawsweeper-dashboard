## Summary

Keep open: current main and the latest shipped release still expose auth cooldown rotation mostly through config/docs/CLI, while the requested configure/dashboard controls, per-profile cooldown fields in the dashboard payload, and authorized clear-cooldown UI mutation remain missing.

## What This Changes

The issue requests UI and configure-wizard exposure for existing auth profile rotation limits, per-profile cooldown state, and a clear-cooldown action.

## Best Possible Solution

Add a designed auth-profile management surface that shows rotation limits with provider profile counts, sends cooldownUntil/cooldownReason through the status API, and exposes a scoped clear-cooldown mutation if maintainers approve that behavior.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing config and auth-state behavior, not a broken documented behavior. Source inspection confirms the underlying settings and state exist while the requested user-facing surfaces are absent.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending models.authStatus is the narrowest likely read path, but clear-cooldown should be an explicit authorized gateway mutation rather than hidden state-file editing.
