Keep open. Current main has the auth cooldown config, runtime limits, generic config-form editing, and CLI status pieces, but it does not provide the requested first-class configure/dashboard UX for profile counts, cooldown reason visibility, or a per-profile clear action.

Add a deliberate auth-profile UX surface that reuses existing auth.cooldowns and auth-state contracts: show per-provider profile counts, active cooldown until/reason per profile, editable rotation limits, and an owner-approved clear action backed by the existing locked helper.
