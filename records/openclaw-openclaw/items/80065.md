## Summary

Keep open: current main has the auth cooldown config, runtime behavior, docs, generic dashboard config editing, CLI inspection, and a low-level cooldown-clear helper, but it still lacks the requested first-class auth-profile UI/API surface for cooldown metadata, profile counts, and a per-profile clear-cooldown action. Targeted GitHub searches did not find an exact duplicate or merged PR that owns this UI exposure.

## What This Changes

Expose existing auth profile-rotation settings and per-profile cooldown state in the configure wizard and/or dashboard auth UI.

## Best Possible Solution

Add a first-class auth profile management surface that reuses the existing config schema for rotation limits, extends `models.authStatus` with safe cooldown metadata and profile counts, and adds a scoped clear-cooldown RPC only if maintainers approve the mutation semantics.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing config and runtime state, not a broken documented behavior. Source inspection confirms the runtime/config/CLI pieces exist while the requested first-class dashboard/configure controls and mutation are absent.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending `models.authStatus` is the likely narrow data path, but the clear-cooldown action should be an explicit authorized gateway mutation rather than generic config editing or direct auth-state file writes.
