## Summary

Keep open: current main exposes these config keys through the generic dashboard config editor, but the requested auth-profile UX is not complete because the configure wizard lacks a rotation/cooldown section and the dashboard auth-status surface does not expose per-profile cooldown reason/time or a clear-cooldown action.

## What This Changes

The issue asks to expose existing auth profile-rotation cooldown settings and per-profile cooldown state in the configure wizard or dashboard UI.

## Best Possible Solution

Add a focused auth/profile settings surface that shows each provider's profile count, rotation limits, active cooldown/disabled windows, and an explicit admin-gated per-profile clear action while keeping the existing generic config editor as the fallback path.
