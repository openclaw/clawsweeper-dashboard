## Summary

Keep open: current main has the auth cooldown config, runtime behavior, docs, CLI inspection, and a low-level clear helper, but it still lacks the requested configure/dashboard controls, cooldown fields in the dashboard auth-status payload, and a per-profile clear-cooldown gateway/UI action. The earlier ClawSweeper review reached the same keep-open conclusion, and targeted searches did not find a canonical duplicate or merged UI follow-up that now owns this request.

## What This Changes

Request to expose existing auth profile rotation limits, per-profile cooldown state, and clear-cooldown actions in the configure wizard or dashboard.

## Best Possible Solution

Add a designed auth-profile management surface that extends `models.authStatus` with cooldown metadata and profile counts, exposes the existing rotation knobs with cascade help, and adds an explicit authorized clear-cooldown mutation if maintainers approve it.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing runtime state, not a broken documented behavior. Source inspection confirms the runtime/config data exists while the requested dashboard/configure controls are absent on current main.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending `models.authStatus` is the narrowest likely data path, but the clear-cooldown button should be an explicit authorized gateway mutation rather than hidden generic config editing.
