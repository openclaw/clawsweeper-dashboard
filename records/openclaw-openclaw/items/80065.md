## Summary

Keep open: current main has the auth cooldown config, runtime rotation behavior, docs, CLI profile listing, and a provider-level Model Auth dashboard card, but it still lacks the requested profile-aware dashboard/configure controls, cooldown reason/status fields in the dashboard payload, and an authorized clear-cooldown UI mutation.

## What This Changes

Request to expose existing auth profile rotation limits and per-profile cooldown state in the configure wizard or dashboard, including a clear-cooldown affordance.

## Best Possible Solution

Add a designed auth-profile management surface that shows rotation limits with provider profile counts, includes cooldownUntil and cooldownReason in the status data path, and exposes an explicit authorized clear-cooldown mutation if maintainers approve it.

Not applicable: this is a feature request for new UI/API exposure of existing config and runtime state, not broken documented behavior. Source inspection confirms the underlying config/state exists while the requested UI and mutation surfaces are absent.

## Solution Assessment
