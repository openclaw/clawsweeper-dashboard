## Summary

Keep open: current main implements and documents the auth cooldown/profile-rotation knobs, and it has CLI/status pieces that expose some cooldown state, but the requested configure-wizard and Control UI affordances are still not present. This is a useful core auth UX feature rather than stale cleanup or a ClawHub/plugin item.

## What This Changes

Request to expose existing auth profile-rotation cooldown settings, per-profile cooldown state, and a clear-cooldown action in the configure wizard or dashboard UI.

## Best Possible Solution

Add a core-owned auth settings/status experience that reuses the existing config keys and auth-state contract, with safe profile counts, cooldown reason/until display, and a maintainer-approved clear-cooldown admin action.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main has the underlying config/runtime/docs and partial CLI diagnostics, but not the requested wizard/dashboard UI surface.

## Solution Assessment

Yes for keeping it in core auth/UI rather than closing or moving to ClawHub; the safest path is to extend existing `auth.cooldowns`, `models.authStatus`, and configure/dashboard surfaces instead of adding a parallel plugin-owned state path.
