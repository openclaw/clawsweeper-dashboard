## Summary

Keep open: current main and the latest shipped release already have the auth cooldown config, runtime rotation caps, docs, CLI profile listing, and a provider-level Model Auth dashboard card, but the requested configure/dashboard controls, per-profile cooldown fields in the dashboard payload, and clear-cooldown UI mutation are still missing.

## What This Changes

Expose existing auth profile-rotation settings and per-profile cooldown state in the configure wizard or dashboard UI.

## Best Possible Solution

Add a designed auth-profile management surface that exposes rotation limits with profile counts, includes cooldownUntil/cooldownReason in the status data path, and provides an explicit authorized clear-cooldown mutation if maintainers approve it.
