## Summary

Keep open: current main has the cooldown config keys, documentation, CLI cooldown listing, and a provider-level dashboard auth card, but it still lacks the requested profile-aware configure/dashboard workflow, per-profile cooldown reason display, profile-count guidance, and guarded clear-cooldown action.

## What This Changes

Request to expose auth profile rotation limits and per-profile cooldown state/actions in the configure wizard or dashboard UI.

## Best Possible Solution

Add a product-approved model-auth settings/status flow that reuses the existing config schema and models.authStatus path, includes per-profile cooldown reason/expiry and profile counts, and only adds clear-cooldown after maintainers define safe mutation semantics.

Not applicable as a bug reproduction; this is a feature/UX request. Source inspection gives a high-confidence gap: current main has config/docs/CLI/status pieces but not the requested profile-aware configure/dashboard controls or clear-cooldown action.

## Solution Assessment

Unclear until maintainers choose the UX and mutation policy. The maintainable direction is to extend existing config metadata and models.authStatus rather than build a parallel auth-state editor.
