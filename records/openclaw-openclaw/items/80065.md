## Summary

Keep open: current main already has the underlying auth cooldown configuration, docs, CLI visibility, and a dashboard auth-status card, but it does not expose the requested rotation limits, per-profile cooldown state, or clear-cooldown action in the configure wizard or dashboard UI.

## What This Changes

Expose existing auth.cooldowns rotation limits and per-profile cooldown state/actions in openclaw configure and/or the dashboard settings UI.

## Best Possible Solution

Add a first-class operator-authenticated auth profile settings/status surface that reuses the existing auth.cooldowns schema, exposes cooldown state from auth-state.json, and calls the existing clear-cooldown helper through a narrow gateway method.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has config/docs/CLI pieces for auth cooldowns, but no dedicated wizard or dashboard controls for the requested rotation counts and clear-cooldown UI.

## Solution Assessment

Unclear until maintainers choose the UX/API scope. The narrowest maintainable direction is to extend the existing models.authStatus/dashboard auth surface and optionally the configure wizard, rather than asking users to edit auth-state.json directly.
