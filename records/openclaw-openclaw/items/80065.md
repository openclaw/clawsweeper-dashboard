## Summary

Keep open: current main has the underlying auth cooldown config, runtime rotation behavior, docs, CLI inspection, a generic settings editor path, and a low-level cooldown-clear helper, but it still lacks the requested designed configure/dashboard profile-management surface, per-profile cooldown metadata in `models.authStatus`, and an authorized gateway/UI clear-cooldown action.

## What This Changes

Feature request to expose existing auth profile-rotation and cooldown settings in the configure wizard or dashboard.

## Best Possible Solution

Add a designed auth-profile management surface that extends `models.authStatus` with cooldown metadata and profile counts, exposes the rotation knobs with cascade help, and adds an explicit authorized clear-cooldown mutation if maintainers approve it.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing runtime state, not a broken documented behavior. Source inspection confirms the runtime/config data exists while the requested profile-aware configure/dashboard workflow is absent on current main.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending `models.authStatus` is the narrowest likely data path, but cooldown clearing should be an explicit authorized gateway mutation rather than hidden generic config editing.
