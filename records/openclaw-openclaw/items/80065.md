## Summary

Keep open: current main has the auth cooldown config, runtime behavior, docs, CLI inspection, and generic config-editor hints, but it still lacks the requested configure-wizard flow, dedicated dashboard profile-rotation controls, per-profile cooldown metadata in `models.authStatus`, and a gateway/UI clear-cooldown action. The existing ClawSweeper comment reached the same conclusion, and fresh searches found related auth-status/runtime work but no canonical duplicate that owns this exact UI/API exposure.

## What This Changes

Request to expose existing auth profile rotation limits, per-profile cooldown state, and cooldown clearing in the configure wizard or dashboard.

## Best Possible Solution

Add a designed auth-profile management surface that extends `models.authStatus` with profile counts and cooldown metadata, exposes the two rotation knobs with cascade help, and adds an explicit authorized clear-cooldown mutation if approved.

## Reproduction Assessment

Not applicable: this is a feature request for new UI/API exposure of existing runtime state, not a broken documented behavior. Source inspection confirms the config/runtime data exists while the requested configure/dashboard controls and clear action are absent on current main.

## Solution Assessment

Unclear pending maintainer UX/API direction: extending `models.authStatus` for read-side cooldown metadata looks like the narrowest likely data path, but clearing cooldowns should be an explicit authorized gateway mutation rather than hidden config or state-file editing.
