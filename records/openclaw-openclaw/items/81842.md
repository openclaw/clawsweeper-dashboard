## What This Changes

This PR forwards `threadName` from the Discord message tool send adapter into the Discord send runtime and adds a warning plus focused tests when send cannot rename existing threads.

## Best Possible Solution

Land the warning-only Discord plugin fix after contributor-supplied real behavior proof; automatic thread rename can remain a separate product choice if maintainers want it later.

## Reproduction Assessment

Yes. Current main exposes `threadName` in the shared message-tool schema, but the Discord `send` adapter/runtime path does not read, forward, rename, or warn on it; I verified this from source rather than by mutating a live Discord thread.

## Solution Assessment

Yes for the code direction: returning an explicit warning is the narrowest safe fix for the accepted minimum behavior and avoids silently renaming arbitrary targets from a send call. It is not merge-ready until the external PR proof gate is satisfied.
