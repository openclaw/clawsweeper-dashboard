## What This Changes

The PR changes isolated cron payload outcome classification and adds helper tests so recovered early tool-warning errors can bypass fatal delivery classification.

## Best Possible Solution

Narrow recovery to proven non-terminal tool-warning cases, suppress run-level-error delivery synthesis only for the accepted recovered path, and preserve fatal run-level/provider/retry/context behavior with focused tests.

## Reproduction Assessment

## Solution Assessment

No: the resolver is the right layer, but this implementation is incomplete because fatality and selected delivery/output text are no longer kept in sync. A safer fix should update both the predicate and the run-level-error payload selection with regression coverage.
