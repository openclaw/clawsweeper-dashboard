- **[P1] Preserve fatal tool signals in the recovery escape:** `src/cron/isolated-agent/helpers.ts:304`
  - body: The new recovery escape clears `hasFatalStructuredErrorPayload` whenever there is terminal output and the last error has the cron tool-warning prefix, but it does not also require `params.failureSignal?.fatalForCron !== true` or that every error payload is a tool warning. Please gate this branch with the same fatal-signal and all-tool-warning invariants as recovered tool-warning detection.
