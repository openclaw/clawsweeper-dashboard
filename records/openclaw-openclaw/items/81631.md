Keep open: current main still has only `systemEvent` and `agentTurn` cron payloads; the closest adjacent change is Codex-specific prompt guidance for command-style cron, not a direct scheduler exec payload. ## What This Changes

Request adding a cron `payload.kind: "exec"` that runs shell commands or scripts directly with cwd, PATH, env, timeout, and output controls.

## Best Possible Solution

Design an additive cron protocol/CLI/UI feature that reuses the existing `exec` tool security policy and documents delivery/logging semantics, or explicitly make `agentTurn` plus `--tools exec` the supported path.
