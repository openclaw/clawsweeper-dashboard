## Summary

Keep this issue open as the canonical stable-release tracker. The reply-delivery repair is present on current `main` and both `2026.6.6` betas, but npm `latest` still resolves to affected `2026.6.5`, so the reported stable-user problem remains real.

## What This Changes

Track the stable npm release gap where `/compact` changes session state but its final acknowledgement is dropped because `v2026.6.5` lacks the native fast-path delivery marker.

## Best Possible Solution

Publish a stable release whose tagged native slash fast path contains `markCommandReplyForDelivery`, verify npm `latest` points to that version, and then close this issue with the exact stable release tag.

## Reproduction Assessment

Yes at the shipped-source level: tagged `v2026.6.5` lacks the delivery marker required by dispatch, matching the supplied Feishu and WeChat `queuedFinal=false` and `replies=0` observations. This read-only review did not reproduce a failure on current `main`, where the marker is present.

## Solution Assessment

Yes. Reusing the existing command-delivery marker in the native fast path is the narrow canonical repair; the remaining work is stable release publication and verification rather than another runtime implementation.
