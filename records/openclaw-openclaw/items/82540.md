## Summary

Keep open: the latest head fixes the stale-account review path, but it now excludes the `channelConfigUpdatedAt` reload path that `openclaw-weixin` uses after QR login, so the central WeChat account A+B bug remains and the supplied proof is still mocked regression output rather than live provider proof.

## What This Changes

The PR adds an internal `includeKnownAccounts` channel restart option for plugin-reload handoffs, updates WeChat multi-account docs, and adds gateway plus agent regression tests.

## Best Possible Solution

Keep the fix channel-agnostic, but add a narrow account-index reload handoff for owner-declared external account refreshes, preserve the stale-account guards, and require live WeChat A+B proof or explicit maintainer acceptance before merge.

## Reproduction Assessment

Yes at source level: the published WeChat plugin writes `channels.openclaw-weixin.channelConfigUpdatedAt`, while the PR head's non-plugin reload branch starts the channel without `includeKnownAccounts`. I did not run a live WeChat A+B provider session.

## Solution Assessment

No. The branch is a plausible plugin-replacement handoff fix, but it no longer covers the WeChat QR-login channel-config reload path that the title, docs, and related report describe.
