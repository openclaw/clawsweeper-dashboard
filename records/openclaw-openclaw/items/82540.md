## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clear stale known accounts when plugin reload disables a channel:** `src/gateway/server-reload-handlers.ts:487`
  - body: When a plugin reload disables or removes a channel, the restart loop returns before starting it, but the pre-replace non-manual stop has left its accounts as known-live/restartPending in the manager. On a later plugin reload that re-enables the channel, this `includeKnownAccounts` restart unions those stale accounts back into the fresh plugin list, so an account removed while the plugin was inactive can be started again instead of honoring the new listing. Clear the preserved handoff state for inactive channels, or only use this safety net for channels that stayed active across the same reload.
  - confidence: 0.86
