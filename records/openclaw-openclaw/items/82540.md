## Summary

Keep open: current main and the latest release still lack the exact account-index known-account restart path, and the PR looks like a focused Gateway fix, but the external PR proof is still test/build output rather than live WeChat A+B behavior proof.

## What This Changes

The PR changes Gateway channel reload handling so plugin-reload handoffs and WeChat `channelConfigUpdatedAt` account-index reloads can include manager-known accounts while ordinary config edits do not revive removed accounts.

## Best Possible Solution

Land the generic Gateway lifecycle fix only after redacted live WeChat A+B proof is supplied, or after maintainers explicitly accept the narrower gateway-regression proof for this provider lifecycle bug.

## Reproduction Assessment

## Solution Assessment

Yes at the code level: the PR fixes the host Gateway lifecycle boundary instead of adding WeChat-specific core policy, and it adds guards for ordinary config edits and inactive-plugin reloads. The safer pre-merge state is to add live provider proof or obtain explicit maintainer acceptance of the proof gap.
