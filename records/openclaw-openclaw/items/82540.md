## Summary

Keep open: the current PR head fixed the stale `connected` feedback, but plugin-reload pre-replace stops still use the default manual stop path, so the patch can still drop previously running WeChat accounts before `includeKnownAccounts` runs. The PR also still lacks contributor-provided real WeChat proof.

## What This Changes

The PR changes gateway channel hot reload to use a non-manual stop/restart handoff with known-account preservation, updates WeChat multi-account docs, and adds gateway regression tests.

## Best Possible Solution

Carry the manager-owned hot-reload handoff through both the pre-replace plugin stop path and the normal restart path, add a regression for plugin reload with a temporarily incomplete account list, then provide redacted real WeChat proof.

## Reproduction Assessment

No for a live WeChat setup: this review did not run the external account flow. The linked report at https://github.com/openclaw/openclaw/issues/82091 plus source inspection give a source-level reproduction path, and the current PR head has a clear pre-replace stop gap.

## Solution Assessment

No. The non-manual restart direction is good, but the PR only applies it to the later restart branch; the pre-replace plugin stop path should use the same handoff semantics and have focused regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve accounts stopped before plugin reload:** `src/gateway/server-reload-handlers.ts:401`
  - body: When `plan.reloadPlugins` is true, `beforeReplace` stops channels with `params.stopChannel(channel)`, which defaults to a manual stop, and records those channels in `channelsStoppedBeforePluginReload`. The changed restart branch then skips the non-manual stop for those channels, so `includeKnownAccounts` can run after the old account was marked manually stopped/cleared and still drop account A when the reloaded plugin temporarily lists only account B. Pass the handoff options through the pre-replace stop and cover that path.
  - confidence: 0.91
