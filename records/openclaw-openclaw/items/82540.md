## Summary

Keep open: this is a useful gateway bugfix for the WeChat multi-account hot-reload failure, but it is not merge-ready because the proof is mock-only and the new known-account safety net can still revive stale account state after a disabled/re-enabled plugin reload.

## What This Changes

The PR adds a gateway hot-reload account preservation path for WeChat/external channel plugins, extends channel manager start/stop options, updates gateway/agent tests, and documents multi-account WeChat behavior.

## Best Possible Solution

Keep the generic gateway handoff approach, but clear preserved known-account state for channels that do not remain active across a plugin reload, add that regression coverage, and require redacted live WeChat A+B proof or an explicit maintainer acceptance of targeted proof.

## Reproduction Assessment

Yes, source-reproducible. Current main starts only plugin-listed accounts after a clean non-manual reload stop, and the PR's remaining stale-account defect follows from preserving restart-pending known accounts across disabled-plugin reloads; I did not run a live WeChat session.

## Solution Assessment

No, not yet. The generic gateway handoff direction is the right layer, but the patch still needs stale preserved-account cleanup for inactive plugin reloads and real provider proof before it is the best merge candidate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Clear preserved accounts when plugin reload disables a channel:** `src/gateway/server-reload-handlers.ts:487`
  - body: At this plugin-reload start call, `includeKnownAccounts` trusts any runtime account left `restartPending` by a prior manager-owned stop. If an earlier plugin reload disabled the channel, the restart loop skipped `startChannel`, so those preserved ids remain in the store; re-enabling the plugin later with a different account list can union and restart the stale id. Clear preserved known-account state when a plugin reload leaves a channel inactive, or gate this option to channels active across the same reload.
  - confidence: 0.83
