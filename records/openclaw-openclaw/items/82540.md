## Label Justifications

- `P1`: The PR targets a user-facing channel lifecycle bug that can leave an already-running WeChat account offline and stop message delivery after adding another account.

- `merge-risk: 🚨 message-delivery`: If the handoff is wrong in live WeChat, a previously running account can remain offline or a stale account can restart, directly affecting message delivery.

## Summary

Keep this PR open: current main still lacks the account-index known-account restart path this branch adds, but the external PR still needs live WeChat A+B proof or explicit maintainer acceptance before merge.

## What This Changes

The PR adds an internal Gateway `includeKnownAccounts` restart path for channel/plugin hot reloads, wires it to WeChat account-index reload markers, updates WeChat docs, and adds gateway plus agent regression tests.

## Best Possible Solution

Land a bounded, channel-agnostic Gateway lifecycle fix only after redacted live WeChat A+B proof is attached or maintainers explicitly accept the gateway-only proof for this provider lifecycle path.

## Solution Assessment
