## Summary

Current main still does not address the reported child-process shutdown blocker. The gateway has a force-exit watchdog and bounded restart drains for tasks, embedded runs, and runtime-deps installs, but there is no shutdown path that enumerates, cancels, or waits for ProcessSupervisor-managed child/PTY runs; the discussion also explicitly separated the merged websocket close fix in #61565 from this remaining problem.

## What This Changes

The issue asks OpenClaw to prevent gateway shutdown timeouts caused by active child-process and pipe handles from exec/tool subprocesses, and to expose debugging for the stuck child command.

## Best Possible Solution

Add a narrow core shutdown contract for ProcessSupervisor-managed child/PTY runs: privacy-safe active-run summaries, bounded cancel/drain operations, gateway stop/restart integration before the force-exit watchdog fires, and focused tests that preserve existing command-queue, embedded-run, runtime-deps, and websocket shutdown behavior.
