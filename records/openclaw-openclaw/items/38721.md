## Summary

Keep open. Current main still has a gateway shutdown watchdog and supervised exec/PTY child processes, but gateway cleanup only drains command-queue tasks and embedded runs; the ProcessSupervisor has per-run cancellation, not a gateway-wide bounded drain/cancel hook for active child handles.

## What This Changes

Fix gateway stop/restart so active supervised child or PTY processes do not keep referenced handles alive until shutdown dirty-exits.

## Best Possible Solution

Add an internal ProcessSupervisor active-run snapshot plus bounded drain/cancel operation, then invoke it from gateway stop and restart cleanup before server close and before the watchdog can dirty-exit.

## Reproduction Assessment

Yes, at source level. Current main can have supervised exec or PTY children active while gateway stop/restart cleanup has no ProcessSupervisor-wide bounded drain/cancel path, matching the reported referenced child-process handles.

## Solution Assessment

Yes. A narrow internal supervisor drain/cancel integration in the gateway lifecycle is the maintainable fix; the websocket close fix and the public session-kill feature are adjacent but do not solve this shutdown blocker.
