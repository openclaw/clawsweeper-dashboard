## Summary

Keep open: current main still has a bounded gateway shutdown watchdog and supervised exec/process children, but the gateway close path does not cancel active exec/process sessions. The merged websocket shutdown fix is adjacent, but the live issue discussion correctly preserves this child-process handle blocker as a separate root cause.

## What This Changes

The issue reports dirty gateway shutdowns on systemd when active child-process handles from exec or internal subprocess work keep shutdown from completing before the watchdog fires.

## Best Possible Solution

Add a bounded internal shutdown step that cancels active supervised exec/process runs and records safe diagnostic metadata, while leaving the public session-wide list/kill API design to https://github.com/openclaw/openclaw/issues/12554.

## Reproduction Assessment

No, not as a live systemd reproduction in this read-only review. Source inspection does show the reported path: current main can create supervised exec child processes, and gateway shutdown does not cancel or drain those active process sessions.

## Solution Assessment

Yes, the best maintainable direction is a narrow internal gateway shutdown cleanup using existing supervisor/process primitives. It should not be stretched into the broader public `sessions_kill` operator request.
