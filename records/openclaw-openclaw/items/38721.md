## Summary

Keep this open: current main and the latest release still preserve the normal shutdown behavior that leaves active runs un-aborted, while the linked websocket shutdown fix covers a different close blocker.

## What This Changes

The issue reports gateway shutdown timing out on normal stop when active child-process handles and pipes remain referenced, causing dirty shutdown and later lane waits.

## Best Possible Solution

Define one gateway stop policy that rejects new work, drains or aborts active runs through their canonical owners, cancels supervisor-owned process scopes with bounded grace, and documents any preserved background-process exception.
