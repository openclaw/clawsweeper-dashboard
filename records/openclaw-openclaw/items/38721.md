## Summary

Keep open: current main still has the reported shutdown watchdog path and normal shutdown intentionally does not abort active runs; the merged websocket-close fix only covered an adjacent blocker, while child-process shutdown policy remains unresolved.

## What This Changes

Report asks OpenClaw to diagnose and fix dirty gateway shutdowns where active child-process handles keep the Node process alive until the shutdown watchdog fires.

## Best Possible Solution

Define one normal gateway stop policy that rejects new work, drains or aborts active runs through their canonical owners, cancels supervisor-owned child-process scopes with bounded grace, and documents any preserved background-process exception.

## Reproduction Assessment

No. The reporter provided strong diagnostic symptoms and current source shows normal shutdown leaves active runs un-aborted, but I did not run a live Ubuntu/systemd reproduction or identify the exact child command.

## Solution Assessment

Unclear. The current websocket-close fix is not the complete solution; the best path needs an owner-approved normal-stop policy for active exec/process children before code changes are safe.
