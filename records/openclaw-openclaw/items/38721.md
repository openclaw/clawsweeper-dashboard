## Summary

Keep open. Current main still has the shutdown watchdog and the bounded websocket close fix from #61565, but gateway stop/restart still does not enumerate, cancel, or bounded-drain ProcessSupervisor child/PTY runs, which matches the reporter's active process-handle evidence.

## What This Changes

Gateway shutdown can time out with active supervised child/PTY process handles because shutdown does not cancel or drain ProcessSupervisor runs.

## Best Possible Solution

Add a small internal ProcessSupervisor shutdown contract and wire it into gateway stop/restart before the force-exit watchdog, preserving existing command-queue, embedded-run, websocket, HTTP, MCP, and LSP cleanup behavior.

## Reproduction Assessment

Yes, at source level. The report includes concrete Node diagnostic reports with active process and pipe handles, and current main still has a path where a long-running supervised exec/PTY run is not canceled or drained during gateway shutdown.

## Solution Assessment

Yes. A narrow supervisor active-run drain/cancel API plus gateway lifecycle wiring is the maintainable fix; relying on systemd or launchd cleanup would still leave OpenClaw with a dirty shutdown and no bounded in-process cleanup path.
