## Summary

Keep open: current main and v2026.5.12 have important shutdown mitigations for service-managed children, but the gateway close path still does not explicitly cancel or drain active exec/process sessions, which is the child-handle blocker reported here. The related session-wide process-control feature remains open and should not absorb this narrower shutdown cleanup bug.

## What This Changes

The issue reports dirty gateway shutdowns where Node diagnostic reports show referenced child-process and pipe handles surviving shutdown, followed by lane wait delays.

## Best Possible Solution

Add a bounded internal shutdown cleanup step that cancels active supervised exec/process sessions during gateway close, with process-tree fallback where a pid is known, while leaving the public session-wide list/kill API to https://github.com/openclaw/openclaw/issues/12554.

## Reproduction Assessment

Yes at source level, but not as a live systemd reproduction: current main can create supervised exec/process child handles, and the gateway close path does not cancel those active process sessions. The exact reporter child process is still unidentified because the full diagnostic reports are not attached.

## Solution Assessment

Unclear as a submitted patch because there is no patch here; the best maintainable direction is a narrow internal shutdown cleanup, not closing this as the broader public `sessions_kill` operator request.
