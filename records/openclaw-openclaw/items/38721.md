## Summary

This should stay open: current main bounds several shutdown phases and restart drains, but normal gateway stop still does not visibly cancel or reap active exec/process children before the shutdown watchdog can fire.

## What This Changes

The issue reports dirty gateway shutdown on OpenClaw 2026.3.2 where an active child-process handle appears to keep shutdown from completing before the internal timeout.

## Best Possible Solution

Add an owner-approved gateway stop policy that rejects new command work, aborts or drains active agent/process children through the canonical owners, and gives them a bounded settle window before the shutdown watchdog exits dirty.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection does show a plausible current-main path: normal stop skips the restart active-work drain while exec/process children only terminate when their owner calls abort or cancel.

## Solution Assessment

No, current main is not the complete solution for this report. The websocket close timeout fix and restart drain are useful, but the best fix should define one shutdown-owned child-process cancellation policy instead of relying on unrelated close steps.
