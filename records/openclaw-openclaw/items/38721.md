## Summary

Keep open. Current main still has the gateway shutdown watchdog and supervised exec/PTY child runs, but gateway stop/restart cleanup drains task and embedded-run counters only; it still lacks a ProcessSupervisor-wide bounded drain/cancel path for active child processes.

## What This Changes

Investigate and fix gateway shutdown timing out when supervised child process or pipe handles remain active, causing dirty shutdown and lane waits after restart.

## Best Possible Solution

Add an internal ProcessSupervisor active-run count plus bounded cancel/drain operation and wire it into gateway stop/restart cleanup before the watchdog fires.

## Reproduction Assessment

Yes, at source level. Current main can have supervised exec or PTY children active while gateway stop/restart cleanup only drains tasks and embedded runs, matching the reporter's child-process and pipe-handle diagnostics.

## Solution Assessment

Yes. A narrow internal supervisor drain/cancel integration in the gateway lifecycle is the maintainable fix; the WebSocket close fix and the public session-kill feature are adjacent but do not solve this shutdown blocker.
