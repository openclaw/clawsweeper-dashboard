## Summary

Keep open: current main still has the reported dirty-shutdown watchdog and normal SIGTERM shutdown intentionally leaves active runs un-aborted; the merged websocket shutdown fix addressed an adjacent blocker, not the child-process/pipe-handle policy in this report.

## What This Changes

The issue asks for gateway shutdown to avoid timing out dirty when active child-process handles from exec/tool work are still referenced, and for guidance on whether such child processes should be canceled or detached on shutdown.

## Best Possible Solution

Define one normal gateway stop policy that rejects new work, drains or aborts active runs through their canonical owners, cancels supervisor-owned child-process scopes with bounded grace, and documents any preserved background-process exception.

No high-confidence live reproduction is available from this review. The reporter provided strong diagnostics and current source shows normal shutdown leaves active runs un-aborted, but the exact child process behind the Node report was not identified on current main.
