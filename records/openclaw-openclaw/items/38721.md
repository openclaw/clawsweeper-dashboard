## Best Possible Solution

Define a bounded gateway shutdown contract for active exec/supervisor children, then implement it with grace-period cancellation, registry cleanup, and regression coverage for active child handles.

## Reproduction Assessment

No live high-confidence current-main reproduction was established. Source inspection does show the reported class remains plausible: exec children are supervisor-owned, manually killable through the process tool, and not cancelled as a class during gateway shutdown.

## Solution Assessment

Unclear until maintainers choose the shutdown contract. The narrowest maintainable path is probably a gateway-owned drain/cancel hook for active process-supervisor runs, not treating the websocket fix or service-managed attachment as the complete solution.
