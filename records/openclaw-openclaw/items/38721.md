## Summary

Keep open: current main and the latest release still have bounded websocket/session shutdown work, but no gateway-close drain or cancellation contract for active process-supervisor child runs. The related websocket fix and open signal/operator-control issues are useful adjacent context, but they do not close this child-process shutdown report.

## What This Changes

The issue asks OpenClaw to prevent gateway shutdown timeouts caused by active child process handles, and to clarify or implement child-process detach/cancel behavior during stop or restart.

## Best Possible Solution

Confirm the symptom on current main under a managed systemd gateway; if it still reproduces, add a bounded gateway-owned supervisor child-run drain/cancel path that preserves service-managed process-group safety and keeps broader signal policy or operator controls in their canonical issues.

## Reproduction Assessment

No high-confidence current-main live reproduction has been run. Source inspection still shows supervised child runs can exist while gateway close lacks a process-supervisor drain hook, but service-managed child changes mean the original Ubuntu/systemd symptom needs fresh confirmation.

## Solution Assessment

Unclear until current-main live proof is collected. The likely maintainable fix is a bounded gateway-owned supervisor drain/cancel hook, not treating the websocket close timeout, signal escalation policy, or session-kill operator API as a complete substitute.
