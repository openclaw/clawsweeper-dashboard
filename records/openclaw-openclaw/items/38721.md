## Summary

Keep open: current main still has a credible gateway shutdown timeout path where supervised exec/CLI child processes can remain active, while the gateway close path only drains chat/session lifecycle work and does not cancel or drain process-supervisor runs. ## What This Changes

The issue asks for gateway stop/restart to cancel or drain active child processes so shutdown does not hit the watchdog and leave lane waits.

## Best Possible Solution

Add a bounded internal gateway shutdown step that cancels or drains active gateway-owned supervised process runs, while leaving the public session process-control API to https://github.com/openclaw/openclaw/issues/12554 and signal escalation policy to https://github.com/openclaw/openclaw/issues/66399.
