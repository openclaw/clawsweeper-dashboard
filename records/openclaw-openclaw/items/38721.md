## Summary

Keep open: current main still has the gateway shutdown watchdog and supervised exec/process children, but the gateway close path is not wired to cancel active process-supervisor runs. The merged websocket shutdown fix is adjacent, not a fix for the child-process/pipe-handle blocker described here.

## What This Changes

The issue reports gateway shutdown timing out when active child-process and pipe handles keep the process alive during stop or restart.

## Best Possible Solution

Add a bounded internal gateway shutdown step that cancels active supervised exec/process runs and records safe diagnostic metadata, while leaving the broader public session exec-list/kill operator UX to https://github.com/openclaw/openclaw/issues/12554.

## Reproduction Assessment

No, not as a live systemd reproduction in this read-only review. Source inspection does show a credible current-main path: supervised exec child processes can remain active, and gateway shutdown is not wired to cancel those active process sessions.

## Solution Assessment

Yes, the best maintainable direction is a narrow internal gateway shutdown cleanup using existing supervisor/process primitives. It should not be stretched into the broader public `sessions_kill` operator request.
