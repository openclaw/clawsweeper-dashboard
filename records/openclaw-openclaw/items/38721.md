## Summary

Keep open. Current main still has the gateway shutdown watchdog and bounded WebSocket close fix, but it does not count, drain, or cancel active ProcessSupervisor exec/PTY children during stop or restart, so the reporter's child-process handle evidence still maps to an unfixed source path.

## What This Changes

The issue asks for gateway shutdown to handle active child process handles from exec/tool subprocesses instead of timing out and leaving a dirty stop or lane wait.

## Best Possible Solution

Add a small internal ProcessSupervisor active-run count/drain/cancel contract and wire it into gateway stop/restart before the force-exit watchdog, leaving public session kill APIs and signal-escalation policy to their own tracks.

## Reproduction Assessment

Yes, at source level. The report gives concrete Node diagnostic evidence with active process and pipe handles, and current main still has supervised exec/PTY child runs without a gateway-wide shutdown drain or cancel path.

## Solution Assessment

Yes. A narrow internal supervisor drain/cancel hook in the gateway lifecycle is the maintainable fix; the merged WebSocket close fix and the open per-subsystem timeout PR address adjacent blockers but not active child-process handles.
