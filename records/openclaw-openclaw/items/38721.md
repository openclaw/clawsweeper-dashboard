## Summary

Keep open. Current main still has the gateway shutdown watchdog and supervised exec/PTY child runs, but the gateway lifecycle only drains task and embedded-run counters and has no ProcessSupervisor active-run drain/cancel hook; related WebSocket and signal-escalation work does not cover this blocker.

## What This Changes

The issue asks OpenClaw to avoid dirty gateway shutdowns when active supervised exec/PTY child processes keep child-process and pipe handles referenced.

## Best Possible Solution

Add an internal ProcessSupervisor active-run count plus bounded cancel/drain operation and wire it into gateway stop/restart cleanup before the watchdog fires, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Run the gateway with an active supervised exec or PTY child and then stop or restart it; current main has no gateway-wide ProcessSupervisor drain/cancel path, matching the reporter's active child-process and pipe-handle diagnostics.

## Solution Assessment

Yes. A narrow internal supervisor drain/cancel integration in the gateway lifecycle is the maintainable fix; WebSocket close bounding, public session process control, and SIGTERM-first cancellation policy are adjacent but different work.
