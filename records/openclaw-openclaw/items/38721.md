## Summary

Keep open. Current main still has the gateway shutdown watchdog and bounded websocket/runtime cleanup, but it does not expose or call a ProcessSupervisor-wide drain/cancel path for active exec or PTY child processes, which matches the reporter's process-handle evidence and the prior maintainer comment separating this from #61565.

## What This Changes

The issue reports gateway stop/restart timeouts when active exec child-process handles keep shutdown from draining cleanly.

## Best Possible Solution

Add a small internal ProcessSupervisor shutdown contract and wire it into gateway stop/restart before the force-exit watchdog, preserving existing command-queue, embedded-run, websocket, HTTP, MCP, and LSP cleanup behavior.

## Reproduction Assessment

Yes, at source level. The report provides concrete Node diagnostic evidence with active process and pipe handles, and current main still has supervised child/PTY runs with no gateway-wide drain or cancel path.

## Solution Assessment

Yes. A narrow supervisor active-run drain/cancel API plus gateway lifecycle wiring is the maintainable fix; relying on systemd cleanup or folding in the broader session-kill feature would leave this shutdown bug either dirty or over-scoped.
