## Summary

Keep open: current main still has a credible shutdown timeout path where supervised exec child processes can stay referenced, and the gateway close path is not wired to cancel or drain process-supervisor runs. The merged websocket shutdown fix addresses an adjacent hang, but not the child-process handle blocker reported here.

## What This Changes

Reports dirty gateway shutdown on Ubuntu/systemd when active child-process handles from exec/tool runs keep the process referenced, often followed by lane wait timeouts.

## Best Possible Solution

Add a bounded internal gateway shutdown step that cancels or drains active supervised exec/process runs using supervisor primitives, while leaving the broader public session-process operator to https://github.com/openclaw/openclaw/issues/12554.

## Reproduction Assessment

No high-confidence live systemd reproduction was run in this read-only review. Source inspection plus the reporter's Node reports provide a credible current-main path: supervised exec children can remain active while gateway shutdown lacks a process-supervisor drain/cancel hook.
