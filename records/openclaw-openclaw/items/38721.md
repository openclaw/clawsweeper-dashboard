## Summary

Keep open: current main still has a credible gateway shutdown path where active supervised exec child processes can remain referenced, while the gateway close handler is not wired to cancel or drain ProcessSupervisor/process sessions. The websocket close fix and launcher orphan fix are adjacent but do not cover this child-process handle blocker.

## What This Changes

The issue asks OpenClaw to prevent gateway shutdown timeouts caused by active child-process and pipe handles, likely from exec/tool subprocesses.

## Best Possible Solution

Add a bounded internal gateway shutdown step that cancels or drains active supervised exec/process runs using existing process primitives, with safe diagnostics and without adding a public sessions_kill API here.

## Solution Assessment
