## Summary

Keep open: current main still has the reported shutdown watchdog, normal gateway stop still avoids the restart active-run abort path, and background exec abort behavior is intentionally preserved; the websocket-close fix and related session-kill/orphan reports only cover adjacent parts of the same process-lifecycle area.

The issue reports dirty gateway shutdown on Ubuntu/systemd when active child-process handles and pipes appear to block cleanup, followed by lane wait warnings.

## Best Possible Solution

Define one normal gateway stop policy that rejects new work, drains or aborts active runs through their canonical owners, cancels supervisor-owned child-process scopes with bounded grace, documents any background-process exception, and proves it on a live Ubuntu/systemd stop path.
