## Summary

Keep open: current main still has the shutdown timeout watchdog and supervised exec children, but gateway close is not wired to cancel active ProcessSupervisor/process sessions. The websocket shutdown fix at https://github.com/openclaw/openclaw/pull/61565 and the launcher orphan-child fix at https://github.com/openclaw/openclaw/pull/77481 are adjacent, but neither covers the child-process/pipe-handle blocker reported here.

## What This Changes

The issue reports gateway shutdown timeouts on OpenClaw 2026.3.2 where Node diagnostic reports show active child-process and pipe handles during shutdown.

## Best Possible Solution

Add a bounded internal gateway shutdown step that cancels active supervised exec/process runs via existing supervisor/process primitives, records safe diagnostics, and leaves the broader operator-facing session kill UX to https://github.com/openclaw/openclaw/issues/12554.

## Reproduction Assessment

No, not as a live systemd reproduction in this read-only review. Source inspection shows a credible current-main path: supervised exec child processes can remain active, and gateway shutdown is not wired to cancel those active process sessions.

## Solution Assessment

Yes, the best maintainable direction is a narrow internal gateway shutdown cleanup using existing supervisor/process primitives. It should not be expanded into the broader public `sessions_kill` operator request.
