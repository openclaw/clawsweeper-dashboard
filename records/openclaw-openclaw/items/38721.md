## Summary

Keep open: current main still has the shutdown watchdog and supervised exec/process children, but the gateway close handler does not cancel or drain active exec/process sessions. The websocket shutdown fix in https://github.com/openclaw/openclaw/pull/61565 is related cleanup, but the maintainer comment correctly preserves this separate child-handle blocker.

## What This Changes

The issue asks OpenClaw to prevent gateway shutdown timeouts caused by active child-process handles from exec or internal subprocess work, and to provide useful diagnostics for the responsible child.

## Best Possible Solution

Add a bounded internal shutdown step that cancels active supervised exec/process sessions during gateway close, with pid process-tree fallback where needed, while leaving the public session-wide operator design to https://github.com/openclaw/openclaw/issues/12554.

## Reproduction Assessment

Yes at source level, but not as a live systemd reproduction: current main can create supervised exec/process child handles, and the gateway close path does not cancel those active process sessions. The exact reporter child process is still unidentified because the full diagnostic reports are not attached.

## Solution Assessment

No patch is proposed here; the best maintainable direction is a narrow internal shutdown cleanup, not closing this as the broader public `sessions_kill` operator request.
