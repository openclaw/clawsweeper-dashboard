## Summary

Keep open: current main still lacks the requested generic `inbound_claim` broadcast, but this PR's current insertion point would regress abort handling, plugin-bound fallback behavior, delivery-suppression guarantees, and inbound dedupe cleanup.

## What This Changes

The PR adds a broadcast `runInboundClaim` call to `dispatchReplyFromConfig` so non-plugin-bound inbound messages can be claimed before agent dispatch.

## Best Possible Solution

Land generic inbound-claim support for ordinary non-plugin-bound dispatch while preserving fast-abort precedence, plugin-bound fallback semantics, delivery suppression, dedupe cleanup, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main has no ordinary-path `runInboundClaim` call, and the PR regressions follow directly from the existing dispatch ordering, tests, and dedupe helpers without executing the suite.

## Solution Assessment

No. The direction is correct, but this patch is not the best solution because the generic claim must be placed or gated so it cannot preempt fast-abort, plugin-bound fallback, suppressed-delivery processing, or dedupe finalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve fast-abort before plugin claims:** `src/auto-reply/reply/dispatch-from-config.ts:420-423`
  - body: This generic claim runs before `tryFastAbortFromMessage()`. A global `inbound_claim` handler that buffers overlapping turns can also claim `stop`, `abort`, or `/stop`, so dispatch returns here before the active run is cancelled.
  - confidence: 0.91
- **[P1] Skip generic claims after plugin-bound fallback:** `src/auto-reply/reply/dispatch-from-config.ts:420-423`
  - body: When a plugin-owned binding falls through `missing_plugin` or `no_handler`, dispatch has already chosen the OpenClaw fallback path. Broadcasting here lets an unrelated plugin claim the turn before the promised fallback resolver runs.
  - confidence: 0.88
- **[P2] Respect delivery suppression for generic claims:** `src/auto-reply/reply/dispatch-from-config.ts:420-423`
  - body: Current main skips plugin-bound claim hooks under suppressed delivery because plugin handlers can emit outbound messages and the agent should still process inbound. This unconditional generic claim has the same bypass and can stop the suppressed agent path.
  - confidence: 0.84
- **[P2] Commit claimed inbound dedupe before returning:** `src/auto-reply/reply/dispatch-from-config.ts:422-426`
  - body: `claimInboundDedupe()` has already put the message key in the in-flight set before this new early return. Returning without `commitInboundDedupeIfClaimed()` leaves that key in flight, so repeated delivery is skipped as `inflight`.
  - confidence: 0.89
- **[P2] Update dispatch tests for the new broadcast contract:** `src/auto-reply/reply/dispatch-from-config.ts:420-423`
  - body: The existing no-broadcast test makes `runInboundClaim` return handled, then asserts it is not called and the resolver still runs. This PR changes that contract but does not update the test or add coverage for the handled early return.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff has no dependency, CI, secret, or supply-chain change, but its hook ordering can let plugin code intercept cancellation and suppressed-delivery turns before core guards run.

Concerns:

- **[medium] Plugin claims can preempt abort messages:** `src/auto-reply/reply/dispatch-from-config.ts:420`
  - body: Because `runInboundClaim` is inserted before fast-abort handling, a global claim handler returning handled can intercept `stop`, `abort`, or `/stop` and prevent cancellation of an active run.
  - confidence: 0.88
- **[low] Generic claims can bypass suppressed delivery processing:** `src/auto-reply/reply/dispatch-from-config.ts:420`
  - body: The new generic claim has no `suppressDelivery` guard, unlike the current plugin-bound path, so plugin code can claim a turn that should still be processed by the agent with delivery suppressed.
  - confidence: 0.78
