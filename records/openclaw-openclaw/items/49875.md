## Summary

Close this PR as superseded: current main still does not add the exact generic `runInboundClaim` call, but the shipped `before_dispatch` hook covers the central global pre-model interception use case, and later broadcast attempts were closed on that basis. The branch is also not mergeable as-is because its insertion point can preempt abort handling, fallback dispatch, suppressed-delivery processing, and dedupe cleanup.

## What This Changes

The PR adds a broadcast `runInboundClaim` call to `dispatchReplyFromConfig` so non-plugin-bound inbound messages can be claimed before agent dispatch.

## Best Possible Solution

Keep `inbound_claim` scoped to plugin-bound ownership and use `before_dispatch` for global plugin interception; any exact `inbound_claim` broadcast should come back as a fresh hook-semantics/API decision with tests.

## Reproduction Assessment

Yes. Source inspection shows current main has no ordinary-path `runInboundClaim` call, while the PR regressions follow directly from the proposed insertion point and existing dispatch ordering.

## Solution Assessment

No. This branch is not the best way to solve the underlying use case because current main already ships a dedicated `before_dispatch` interception hook, and the proposed generic claim introduces ordering and dedupe regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve fast-abort before plugin claims:** `src/auto-reply/reply/dispatch-from-config.ts:420-423`
  - body: This generic claim runs before `tryFastAbortFromMessage()`. A global `inbound_claim` handler that buffers overlapping turns can also claim `stop`, `abort`, or `/stop`, so dispatch returns here before the active run is cancelled.
  - confidence: 0.91
- **[P1] Skip generic claims after plugin-bound fallback:** `src/auto-reply/reply/dispatch-from-config.ts:420-423`
  - body: When a plugin-owned binding falls through `missing_plugin` or `no_handler`, dispatch has already chosen the OpenClaw fallback path. Broadcasting here lets an unrelated plugin claim the turn before the fallback resolver runs.
  - confidence: 0.88
- **[P2] Respect delivery suppression for generic claims:** `src/auto-reply/reply/dispatch-from-config.ts:420-423`
  - body: Current main skips plugin-bound claim hooks under suppressed delivery because plugin handlers can emit outbound messages and the agent should still process inbound. This unconditional generic claim has the same bypass and can stop the suppressed agent path.
  - confidence: 0.84
- **[P2] Commit claimed inbound dedupe before returning:** `src/auto-reply/reply/dispatch-from-config.ts:422-426`
  - body: `claimInboundDedupe()` has already put the message key in the in-flight set before this new early return. Returning without `commitInboundDedupeIfClaimed()` leaves that key in flight, so repeated delivery is skipped as `inflight`.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff has no dependency, CI, secret, or supply-chain change, but hook ordering can let plugin code intercept cancellation and suppressed-delivery turns before core guards run.

Concerns:

- **[medium] Plugin claims can preempt abort messages:** `src/auto-reply/reply/dispatch-from-config.ts:420`
  - body: Because `runInboundClaim` is inserted before fast-abort handling, a global claim handler returning handled can intercept `stop`, `abort`, or `/stop` and prevent cancellation of an active run.
  - confidence: 0.88
- **[low] Generic claims can bypass suppressed delivery processing:** `src/auto-reply/reply/dispatch-from-config.ts:420`
  - body: The new generic claim has no `suppressDelivery` guard, unlike the current plugin-bound path, so plugin code can claim a turn that should still be processed by the agent with delivery suppressed.
  - confidence: 0.78
