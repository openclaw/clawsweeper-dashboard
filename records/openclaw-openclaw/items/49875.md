## Summary

Keep this PR open because the requested behavior is still missing on current main, but the proposed patch is not merge-ready. The generic inbound_claim wiring is useful source work, yet the current insertion point can preempt fast-abort cancellation, lets unrelated global claim hooks intercept plugin-bound fallback turns, and returns without the cleanup performed by comparable early exits.

## What This Changes

The PR adds a broadcast runInboundClaim call to the general auto-reply dispatch path and returns early when a plugin reports the inbound message as handled.

## Best Possible Solution

Land generic inbound_claim support for ordinary non-plugin-bound dispatch only, while keeping fast-abort ahead of claim hooks, preserving plugin-owned fallback behavior, and matching existing early-return cleanup and diagnostics. The fix should include focused dispatch regression tests plus the hook-runner tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. Current main has a high-confidence source/test reproduction: the ordinary dispatch test mocks runInboundClaim as handled and still asserts it is not called, while the /stop fast-abort tests and FAQ establish the cancellation path that the PR insertion would preempt.

## Solution Assessment

No. The requested wiring is the right direction, but this exact patch is not the best solution because it places the generic claim before fast-abort, after plugin-bound fallback decisions, and without the cleanup used by comparable early exits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve fast-abort before plugin claims:** `src/auto-reply/reply/dispatch-from-config.ts:419-423`
  - body: This claim runs before tryFastAbortFromMessage. A global plugin that claims overlapping messages can also claim `stop`, `abort`, or `/stop`, causing dispatch to return here before the active run is cancelled.
  - confidence: 0.91
- **[P1] Skip generic claims after plugin fallback:** `src/auto-reply/reply/dispatch-from-config.ts:419-423`
  - body: When a plugin-bound conversation falls through `missing_plugin` or `no_handler`, dispatch has already chosen the OpenClaw fallback. Broadcasting here lets an unrelated plugin claim the turn before the fallback reply resolver runs.
  - confidence: 0.88
- **[P2] Clean up claimed-turn state before returning:** `src/auto-reply/reply/dispatch-from-config.ts:421-423`
  - body: The handled return records completion but does not call `markIdle` or `commitInboundDedupeIfClaimed`. Other completed early returns do both, and skipping the dedupe commit can leave the inbound key in-flight.
  - confidence: 0.86
