## Repair Work Prompt

Fix the gateway/session transcript hot path for https://github.com/openclaw/openclaw/issues/76123 without broad plugin, provider, runtime-deps, or queue architecture changes. Current main calls `resolveSessionKeyForTranscriptFile()` from `createTranscriptUpdateBroadcastHandler()` before checking `update.message`, and `resolveSessionKeyForTranscriptFile()` loads the combined session store before consulting `TRANSCRIPT_SESSION_KEY_CACHE`. Implement a safe narrow fix that removes combined-store loads from message-less transcript updates and cached repeat transcript lookups while preserving transcript-only message routing for shared transcript paths. Do not simply return stale cached keys unless invalidation or lightweight validation makes that correct. Add focused tests proving: file-only/no-message transcript updates do not load the combined session store; repeated cache-hit lookups avoid repeated combined-store loads; stale/shared transcript ownership behavior remains correct. Add a user-facing CHANGELOG.md fix entry with appropriate human attribution from the issue discussion if available. Validate with targeted gateway tests first, then the changed gate in Testbox per repo policy.
