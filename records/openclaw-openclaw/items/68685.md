## Best Possible Solution

Land the scoped runtime/doctor migration for the known retired keys, keep it out of global unknown-key stripping, and add the missing changelog plus a schema-valid regression test survivor.

Yes. A config with `agents.defaults.memorySearch.chunkSize` or `agents.list[0].memorySearch.maxResults` reaches the strict `MemorySearchSchema` on current main, and current runtime migrations do not strip those keys.

## Solution Assessment

Yes, with small cleanup. A targeted strip migration after the existing root move is narrower than loosening config validation globally, and the PR now covers the per-agent override gap raised in review.
