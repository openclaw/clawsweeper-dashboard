## What This Changes

The issue asks for the Control UI chat session picker to show only sessions for the active chat agent while preserving existing cron and subagent filtering.

## Best Possible Solution

Review and land a focused UI fix that scopes picker `sessions.list` calls to the active agent and renders picker rows through the existing grouped-session filtering.

## Solution Assessment

Yes. The linked PR's direction is the narrow maintainable fix: add active-agent scoping to picker requests and reuse the existing grouped session projection for visible rows.
