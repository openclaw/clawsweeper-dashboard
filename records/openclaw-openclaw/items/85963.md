## Summary

Current main and the latest release still match the reported source-level bug: chat picker search/load-more calls are unscoped and the popover renders raw result rows. The issue should stay open because the linked focused PR is still open and explicitly owns the fix path.

## What This Changes

The issue requests that the Control UI chat session picker scope session search/load-more results to the active agent and filter out unrelated operational sessions.

## Best Possible Solution

Land a focused UI fix that passes the active agent id into chat picker sessions.list calls and renders picker rows through the same filtered projection used by the grouped session selector.

## Reproduction Assessment

Yes, by source inspection: current main builds chat picker sessions.list params without agentId and then renders result.sessions directly in the popover. I did not run a live UI repro in this read-only review.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable path: scope the picker request and reuse the existing filtered session option logic instead of changing gateway sessions.list globally.
