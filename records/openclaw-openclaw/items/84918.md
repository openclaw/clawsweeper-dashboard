## Summary

Keep open: current main fixed the older image_url parser drop, but the reported text-model plus configured imageModel path still has a source-backed gap because OpenAI HTTP images can be discarded when the selected session model is text-only.

## What This Changes

The issue reports that OpenWebUI image uploads through `/v1/chat/completions` reach the OpenClaw image path as an empty image when the default chat model is text-only and an image model is configured.

## Best Possible Solution

Apply the existing imageModel override-plan pattern to OpenAI chat-completions image turns so text-only default agents route current-turn images through a verified vision-capable `agents.defaults.imageModel` without changing config or URL-fetch policy.

## Reproduction Assessment

Yes, by source inspection: send `/v1/chat/completions` with an `image_url` while the selected/default model is text-only and `agents.defaults.imageModel` is configured. Current parser code can pass `images`, but `detectAndLoadPromptImages` drops existing images for non-vision models, and a current test locks in that behavior.

## Solution Assessment

Yes, the best fix is to reuse the existing Gateway imageModel override plan for this HTTP ingress rather than adding another parser or new config. The older parser fix is already present; the remaining gap is model routing for text-only primaries.
