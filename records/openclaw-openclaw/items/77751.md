## Summary

Keep open: the mitigation targets a concrete EBADF path, but the patch leaves no-input non-TTY child stdin open and lacks attached real gateway proof. The linked MCP-startup surface also remains outside the diff.

## What This Changes

The PR changes the child process adapter to use pipe stdin for non-TTY default spawns and adds an all-pipe EBADF fallback.

## Best Possible Solution

Revise the service-mode mitigation to preserve EOF for no-input children, add focused regression coverage, attach redacted launchd/gateway proof, and handle direct MCP stdio retry if this PR is meant to close https://github.com/openclaw/openclaw/issues/77750.

## Reproduction Assessment

Unclear for the original EBADF failure: the related issue logs and source path are concrete, but I did not recreate the high-FD macOS launchd setup. The stdin regression introduced by the patch is source-reproducible and backed by a direct Node spawn probe.

## Solution Assessment

No. Retrying with a pipe is a plausible direction, but the patch should preserve closed/EOF stdin semantics for no-input children and should not claim the linked MCP-startup issue without covering direct MCP stdio spawn.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Close no-input pipes after non-TTY fallback:** `src/process/supervisor/adapters/child.ts:41`
