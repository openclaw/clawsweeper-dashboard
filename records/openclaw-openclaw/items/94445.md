## Summary

Keep open for maintainer review: the patch is a narrow, plausible fix for cron cloud runs that currently bypass the implicit idle watchdog, but it changes existing no-timeout cron behavior and lacks after-patch live cron proof.

## What This Changes

The branch removes the cron-wide no-timeout return from the embedded LLM idle-timeout resolver and updates focused tests so cron cloud calls get the default watchdog while cron local-provider calls still opt out.

## Best Possible Solution

Land the narrow resolver change only after maintainers accept the no-explicit-timeout cloud-cron default and live or log proof shows cloud cron stalls fail or recover while local providers remain opted out.

## Reproduction Assessment

## Solution Assessment

Yes for the narrow no-timeout cloud-cron resolver bug: deleting the cron-wide bypass lets the existing cloud/local provider classification decide the default. It is not the full solution for related explicit-timeout stream-stall fallback policy.
