## Label Justifications

- `P1`: The PR addresses a maintainer-reported cron agent wedge that can leave queued work behind an active model call with no recovery.

## Summary

Keep open for maintainer review: the patch is small and I found no blocking code defect, but it intentionally changes no-explicit-timeout cloud cron behavior and the PR carries a protected maintainer label.

## What This Changes

The PR removes the cron-wide idle-watchdog opt-out for no-explicit-timeout embedded cloud model runs, adds resolver regression tests, and updates agent-loop docs while retaining explicit timeout precedence and the local-provider opt-out.

## Best Possible Solution

Land this only after a maintainer explicitly accepts the restored default watchdog for no-explicit-timeout cloud cron model calls, keeping the PR's explicit-timeout precedence, local-provider opt-out, docs, and focused regression coverage intact.

## Reproduction Assessment

Yes for the source-level failure path: current main returns 0 from resolveLlmIdleTimeoutMs({ trigger: "cron" }) when no timeout is configured, and cron embedded runs pass trigger="cron" into that resolver. I did not recreate a forced live Codex app-server stall because the PR explains that the original production stall is not safely deterministic.

## Solution Assessment

Yes, conditional on maintainer acceptance of the behavior change. The patch is the narrowest runtime fix I found: remove the cron-wide default opt-out while preserving explicit provider/run/agent timeout precedence and local-provider opt-out behavior.
