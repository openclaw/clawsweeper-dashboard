## Summary

Keep this PR open: it targets a real Control UI PWA gap on current main, but the proposed CSS is incomplete because the inner shell still sizes itself to `100dvh` after the body is padded for safe areas, so the UI can still be clipped or fail to compact correctly in standalone mode.

## What This Changes

The PR adds `viewport-fit=cover` and standalone-mode CSS to lock the Control UI document viewport and apply iOS safe-area padding.

## Best Possible Solution

Keep the PR direction, but finish the containment by sizing the app shell to the safe-area-adjusted content box in a later or equally authoritative stylesheet, then add the required changelog entry and verify on iOS standalone PWA.

## Reproduction Assessment

Yes for the review finding: the static render path shows body safe-area padding would shrink the content box while `.shell` still uses `100dvh` inside `openclaw-app`. The original iOS bug has clear manual repro steps in the PR body, but this review did not run a live iPhone PWA check.

## Solution Assessment

No; the proposed fix is directionally right but not yet the narrowest complete solution. The safer path is to keep `viewport-fit=cover` and standalone document locking, while also overriding the inner shell height/min-height to fit the safe-area-adjusted app container.
