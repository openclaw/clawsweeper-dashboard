## Summary

Keep this PR open, but do not merge as-is: the proposed standalone PWA containment is aimed at a real missing surface on current main, yet it constrains `openclaw-app` while the later-imported shell layout still sizes `.shell`/`.shell--chat` to `100dvh`, which can clip bottom content when safe-area padding is nonzero.

## What This Changes

The PR adds `viewport-fit=cover` and standalone-mode CSS to lock the Control UI document/body, apply safe-area padding, and constrain `openclaw-app`.

## Best Possible Solution

Keep the CSS-only PWA direction, but size the actual shell layer to the safe-area-adjusted app box without clipping inner panes, then add the required active-version changelog entry.

## Reproduction Assessment

Yes for review purposes: the PR body gives a concrete iPhone Add-to-Home-Screen standalone path, and current main lacks both `viewport-fit=cover` and standalone document containment. I did not run a live iOS PWA session in this read-only review.

## Solution Assessment

No as written: the body/app containment is incomplete because the later `.shell` rules still use viewport height. A safer narrow fix should make `.shell`/`.shell--chat` fill the padded app content box in standalone mode and preserve existing inner scroll containers.
