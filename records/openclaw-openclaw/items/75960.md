## Summary

Keep this PR open. The session-store size reduction is useful, but the current patch is incomplete because stripped cold-loaded snapshots can still be reused by the Claude CLI path without the hydrated skill bodies it needs.

## What This Changes

The PR strips `skillsSnapshot.resolvedSkills` from persisted session entries, documents it as runtime-only, adds session-store stripping tests, and updates the changelog.

## Best Possible Solution

Keep the disk-size fix, but add runtime hydration for consumers that need full `Skill[]` data when a persisted snapshot only has lightweight metadata.

## Reproduction Assessment

Yes. A cold-resumed `claude-cli` session with a persisted snapshot containing `prompt`, `skills`, and current `version` but no `resolvedSkills` will reuse that snapshot and then return no `--plugin-dir` args from `prepareClaudeCliSkillsPlugin`.

## Solution Assessment

No. Stripping at the persistence boundary is the right direction, but every runtime consumer that still needs skill file paths must rebuild or hydrate them without re-persisting the cache.
