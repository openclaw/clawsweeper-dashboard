## Summary

Keep open; the storage reduction is directionally right, but the patch would break Claude CLI native skill materialization after cold resumes because it strips the only full skill data that the CLI plugin path currently uses while the agent run can reuse a matching stripped snapshot.

## What This Changes

The PR drops `skillsSnapshot.resolvedSkills` from session-store persistence, documents it as runtime-only, adds focused stripping tests, and updates the changelog.

## Best Possible Solution

Keep `resolvedSkills` off disk, but hydrate or rebuild the runtime-only full skill list before every consumer that needs skill paths, especially Claude CLI plugin materialization.

## Reproduction Assessment

Yes. A cold-loaded session entry with `prompt`, `skills`, `skillFilter`, and `version` but no `resolvedSkills` will pass the version/filter reuse check, then `prepareClaudeCliSkillsPlugin` will see an empty array and skip `--plugin-dir`.

## Solution Assessment

No. Stripping at the persistence chokepoint is a good storage boundary, but the patch is incomplete until non-embedded runtime consumers can rebuild the cache without re-persisting it.

## Review Findings

Overall correctness: patch is incorrect
