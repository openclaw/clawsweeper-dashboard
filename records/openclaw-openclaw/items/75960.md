## Summary

Keep this PR open: the storage-size direction is good, and the latest force-push covers the auto-reply `ensureSkillSnapshot` path, but the direct `agentCommand` resume path still reuses a stripped snapshot and can drop Claude CLI native skill materialization.

## What This Changes

The PR strips `skillsSnapshot.resolvedSkills` from persisted session-store entries, documents it as runtime-only, hydrates one resume path, adds focused persistence tests, and updates the changelog.

## Best Possible Solution

Keep the persistence strip, but hydrate or rebuild missing `resolvedSkills` through a shared snapshot resolver used by both auto-reply and direct `agentCommand` before any CLI runtime consumes the snapshot.

## Reproduction Assessment

Yes. A focused unit test can load a persisted direct agent session with a current `skillsSnapshot` containing `prompt`, `skills`, and `version` but no `resolvedSkills`, then verify the `claude-cli` path still receives populated skills or emits `--plugin-dir`.

## Solution Assessment

No. Stripping at the persistence chokepoint is the right boundary, but the current repair is incomplete because it hydrates `ensureSkillSnapshot` while leaving the separate direct command path on the stripped cached object.
