## Summary

Keep open: the latest head still leaves the `agent-command` Claude CLI path without hydration after the persistence strip, so cold-resumed CLI sessions can lose native Claude Code skill plugin args.

## What This Changes

The PR strips `skillsSnapshot.resolvedSkills` from session-store persistence, adds auto-reply hydration coverage, tests, JSDoc, and a changelog entry.

## Best Possible Solution

Keep the persistence-size reduction, but hydrate stripped snapshots anywhere they are reused for runtime execution, especially `agent-command` before Claude CLI/plugin consumers see the snapshot.

## Reproduction Assessment

Yes. A stored `SessionEntry` with `skillsSnapshot.prompt`, `skills`, and `version` but no `resolvedSkills` will be reused by `agent-command` when version/filter match, then `prepareClaudeCliSkillsPlugin` will see an empty skill list and return no `--plugin-dir` args.

## Solution Assessment

No. Stripping at the persistence boundary is the right storage fix, but the current patch is incomplete until the non-auto-reply CLI path rebuilds or hydrates `resolvedSkills` without re-persisting it.
