## Summary

Keep this PR open. The persistence-size goal is valid, but the current diff would regress Claude CLI resumes because it strips the only field that the Claude skill-plugin materializer uses while the session reuse path will not rebuild it.

## What This Changes

The PR strips `skillsSnapshot.resolvedSkills` during session-store normalization, documents the field as runtime-only, adds focused persistence tests, and updates the changelog.

## Best Possible Solution

Keep the disk-size reduction, but add a runtime-only hydration path for non-embedded consumers such as Claude CLI and cover stripped-snapshot resume behavior in tests.

## Reproduction Assessment

Yes. A cold-resumed Claude CLI session with a persisted snapshot containing `prompt`, `skills`, and `version` but no `resolvedSkills` will reuse that snapshot and then pass an empty skill list to `prepareClaudeCliSkillsPlugin`. This can be reproduced with a focused unit test without external services.

## Solution Assessment

No. Stripping at the persistence chokepoint is the right storage boundary, but the proposed patch is incomplete until every runtime consumer that still needs full skill paths can rebuild or hydrate them without re-persisting the cache.
