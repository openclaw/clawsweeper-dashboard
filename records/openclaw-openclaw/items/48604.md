## Summary

Keep open: the feature direction is valid, but the PR is incomplete against current main. Main already has the schema half from #71247, while the runtime context-window resolver and compaction/extension budget paths are still agent-blind, and the PR’s resolver precedence would not let a per-agent value replace a stricter global default.

## What This Changes

This PR adds `agents.list[].contextTokens` schema support, extends context-window resolution with an optional agent id, passes it through one embedded-runner path, and adds resolver tests.

## Best Possible Solution

Land a rebased implementation that treats `agents.list[].contextTokens` as authoritative before defaults, clamps it to the provider/model base window, and threads the effective session agent id through main run setup, direct compaction, queued/context-engine compaction, context-pruning, and compaction-safeguard setup.

## Reproduction Assessment

Yes for the review findings. Source inspection shows current main has no agent-aware resolver path, and the supplied PR diff updates only one runtime call while leaving precedence and compaction/extension budget paths incomplete.

## Solution Assessment

No. The requested direction is sound, but this patch is not the narrow complete fix because per-agent values are not authoritative over defaults and the agent id is not threaded through every context-budget consumer.
