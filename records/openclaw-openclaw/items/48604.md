## Summary

Keep open. The feature is useful and partly overlapped by current main, but this PR is still incomplete: current main only has the schema side, while the PR’s runtime wiring is stale/incomplete and its override precedence is wrong.

## What This Changes

The branch adds `agents.list[].contextTokens`, extends context-window resolution with an optional agent id, threads it through one embedded-runner call, and adds resolver tests.

## Best Possible Solution

Land a rebased implementation that treats `agents.list[].contextTokens` as authoritative before defaults, clamps it to the provider/model base window, and threads the effective session agent id through main run setup, direct compaction, context-engine compaction, context-pruning, and compaction-safeguard setup.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR branch and current main shows the agent id is not threaded through every context-budget consumer, and the PR resolver can still let global defaults override an explicit per-agent value.

## Solution Assessment

No. The requested direction is sound, but this patch is not the narrow complete implementation because it only updates one runtime call and does not make per-agent values authoritative over defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread agent ids through all context budget paths:** `src/agents/pi-embedded-runner/run.ts:395`
  - body: Adding `agentId` only to this resolver call leaves direct compaction, context-engine compaction, context-pruning, and compaction-safeguard setup on the old agent-blind budget path. Those paths still compute limits without the active agent, so the new per-agent cap is ignored where compaction decisions are made.
  - confidence: 0.88
- **[P2] Make per-agent contextTokens override defaults:** `src/agents/context-window-guard.ts:52-58`
  - body: When an agent value exists but is not below `baseInfo.tokens`, this falls through to `agents.defaults.contextTokens`. With defaults=100k, agent=196k, and model=180k, the resolver returns 100k instead of treating the agent value as authoritative and clamping to the base window.
  - confidence: 0.9
