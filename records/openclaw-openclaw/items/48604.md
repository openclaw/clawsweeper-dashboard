## Summary

Keep open: the feature is useful and supported by prior user reports and comments, but this branch is not merge-ready because it applies agent identity to only one context-budget path, treats per-agent values as a lower cap rather than an authoritative override, and lacks real behavior proof.

## What This Changes

The branch adds `agents.list[].contextTokens` schema support, an `agentId` parameter to context-window resolution, one embedded-runner call-site update, and resolver tests.

## Best Possible Solution

Land a rebased implementation that treats explicit `agents.list[].contextTokens` as the authoritative per-agent budget across run setup, direct and queued compaction, context-pruning, and compaction-safeguard setup, with redacted real behavior proof.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR diff and current main shows the branch passes agent identity through only one runtime path and lets a stricter global default override a larger explicit per-agent value.

## Solution Assessment

No. The requested direction is sound, but the patch is incomplete; the maintainable fix is an agent-aware budget resolver used by every context-budget consumer, with the already-shipped schema support kept from current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread agent ids through every context budget path:** `src/agents/pi-embedded-runner/run.ts:392`
  - body: Passing `agentId` only to the main run resolver leaves direct compaction, queued/context-engine compaction, context-pruning, and compaction-safeguard setup on agent-blind budget resolution. Those paths still compute thresholds from defaults/model metadata, so an agent-specific cap is ignored in the compaction/pruning flows this feature is meant to protect.
  - confidence: 0.9
- **[P2] Treat per-agent contextTokens as authoritative:** `src/agents/context-window-guard.ts:50-52`
  - body: The new lookup returns the per-agent value only when it is lower than the model/config base, then falls through to `agents.defaults.contextTokens`. With defaults=100k, agent=196k, and model=180k, the resolver still returns 100k, so an explicit agent override cannot replace a stricter global default.
  - confidence: 0.9
