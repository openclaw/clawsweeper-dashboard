## Summary

Keep open. The feature direction still has user signal, but this branch is stale/conflicting, the schema portion already landed on main, and the remaining runtime implementation is incomplete and still needs real behavior proof.

## What This Changes

The PR adds `agents.list[].contextTokens` schema support, extends context-window resolution with an optional `agentId`, threads that id through one embedded runner call, and adds resolver tests.

## Best Possible Solution

Land a rebased implementation that uses the existing per-agent config field as an agent-aware budget input across run setup, direct and queued compaction, context-pruning, and compaction-safeguard setup, while coordinating model-safety clamp semantics with https://github.com/openclaw/openclaw/issues/71136.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR diff and current main shows the branch adds agent identity to only one runtime path and lets `agents.defaults.contextTokens` still win over some explicit per-agent values.

## Solution Assessment

No. The direction is useful, but the patch is no longer the best current solution because schema support already landed and the remaining runtime behavior needs a rebased, centralized, agent-aware resolver across all context-budget consumers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Thread agent ids through every context budget path:** `src/agents/pi-embedded-runner/run.ts:392`
  - body: Passing `agentId` only to this main run resolver leaves direct compaction, queued compaction, context-pruning, and compaction-safeguard setup on agent-blind context-window resolution. Those paths still compute thresholds from defaults or model metadata, so an agent-specific cap is ignored in the flows this feature is meant to protect.
  - confidence: 0.9
- **[P2] Honor explicit per-agent caps over defaults:** `src/agents/context-window-guard.ts:50-56`
  - body: The new lookup returns the per-agent value only when it is lower than the base window, then still applies `agents.defaults.contextTokens`. With a larger explicit per-agent budget and a smaller global default, the default still wins, so the per-agent setting cannot actually override defaults.
  - confidence: 0.9
