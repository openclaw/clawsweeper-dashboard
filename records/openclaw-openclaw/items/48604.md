## Summary

Keep this PR open. Current main already contains the schema/config acceptance for per-agent `contextTokens`, but the runtime resolver and embedded runner budget paths remain agent-blind; the proposed patch is directionally useful but still leaves compaction and extension budgets incomplete and does not make the per-agent value authoritative over a stricter global default.

## What This Changes

The PR adds `agents.list[].contextTokens` schema support, extends context-window resolution with an optional agent id, passes that id through one embedded runner path, and adds resolver tests for per-agent/default precedence.

## Best Possible Solution

Land a corrected, rebased implementation that keeps the already-shipped schema support, resolves matching `agents.list[].contextTokens` before `agents.defaults.contextTokens`, caps the per-agent value to the provider/model base window, and threads the resolved session agent id through main run, direct compaction, queued/context-engine compaction, context-pruning, and compaction-safeguard setup.

## Reproduction Assessment

Yes. A code-level reproduction is clear: configure a lower `agents.defaults.contextTokens` plus a higher matching `agents.list[].contextTokens`, then the PR resolver can still return the global default; current main also shows compaction and extension budget calls cannot receive any agent id.

## Solution Assessment

No. The requested direction is maintainable, but this patch is not the best complete solution because it updates only one runtime path and still lets a global default override an explicit per-agent value in common configurations.
