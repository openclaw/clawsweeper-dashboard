## Summary

Keep this PR open, but it needs changes before merge. Current main already has the config-schema half from #71247, while the runtime context-budget resolver and embedded runner/compaction/extension paths are still agent-blind; the proposed patch also treats per-agent values as only a lower cap instead of an authoritative override over `agents.defaults.contextTokens`.

## What This Changes

This PR adds per-agent `agents.list[].contextTokens` schema support, extends context-window resolution with an optional agent id, passes that id through one embedded runner path, and adds resolver tests.

## Best Possible Solution

Land a rebased implementation that resolves `agents.list[].contextTokens` before global defaults, clamps it to the provider/model base window, and threads the effective session agent id through main run, direct compaction, queued/context-engine compaction, context-pruning, and compaction-safeguard setup.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is available: configure a lower `agents.defaults.contextTokens` plus a higher matching `agents.list[].contextTokens`; the PR resolver can still return the global default, and current main cannot pass any agent id into the resolver or compaction/extension budget paths.

## Solution Assessment

No. The feature direction is sound, but this patch is not the best complete solution because it updates only one runtime path, includes schema work already merged separately, and does not make per-agent values authoritative over global defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread agent ids through every context budget path:** `src/agents/pi-embedded-runner/run.ts:392`
  - body: Adding `agentId` only to this resolver call leaves direct compaction, queued/context-engine compaction, context-pruning, and compaction-safeguard setup on the old agent-blind budget path. Those paths still compute limits without the active agent, so the new per-agent cap is ignored where compaction decisions are made.
  - confidence: 0.88
- **[P2] Honor per-agent contextTokens before global defaults:** `src/agents/context-window-guard.ts:50-52`
  - body: This only returns the per-agent value when it is lower than the base window, then falls through to `agents.defaults.contextTokens`. With defaults=100k, agent=196k, and model=180k, the resolver still returns 100k, so an agent cannot raise or replace a stricter global cap.
  - confidence: 0.9
