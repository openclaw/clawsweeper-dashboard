## Summary

Keep open: current main and the latest release still route silent-allowed group reasoning-only turns through the silent reply path, and the only direct fix PR was closed unmerged. The remaining blocker is maintainer confirmation of whether to change the documented group silent-reply contract.

## What This Changes

The issue requests that reasoning-only no-text assistant turns in group chats trigger the visible-answer retry instead of being absorbed by the default group silent-reply policy.

## Best Possible Solution

Maintainers should decide the group silent-reply contract, then either update the helper, tests, and docs to retry reasoning-only group turns visibly or close the issue as intentionally silent behavior.

## Reproduction Assessment

Yes, at source level: current main still lets silent-allowed reasoning-only turns return `NO_REPLY` without the retry instruction, and tests assert that behavior. I did not run a live Telegram group reproduction in this read-only review.

## Solution Assessment

Unclear: a narrow helper change is plausible, but it would reverse documented group-chat silence behavior unless maintainers choose that new contract.
