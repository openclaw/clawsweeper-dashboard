## Summary

Keep open. Current main confirms the reported source path, but docs and tests also intentionally define reasoning-only group turns as silent when silent replies are allowed, so this needs maintainer policy review rather than cleanup close or an automated fix.

## What This Changes

Request to let reasoning-only group-chat assistant turns trigger the visible-answer retry instead of being absorbed by the default group silent-reply policy.

## Best Possible Solution

Maintainers should choose the group-silence contract, then either exempt signed reasoning-only turns from silent-empty handling with focused runner and Telegram regressions or explicitly keep the documented `NO_REPLY` behavior and handle any stuck-ack cleanup separately.

## Reproduction Assessment

Yes, at source level. Current main marks signed reasoning-only, non-visible turns as silent when `allowEmptyAssistantReplyAsSilent` is true, then skips the reasoning-only retry instruction; I did not run a live Telegram/provider reproduction.

## Solution Assessment

Unclear as an automated fix. The suggested code direction is plausible, but current docs and tests deliberately preserve reasoning-only group turns as `NO_REPLY`, so maintainers need to decide the policy boundary first.
