## Summary

Keep open: current main and v2026.6.9 still contain the reported source path, and the direct fix attempt was closed unmerged. The remaining blocker is maintainer direction on whether reasoning-only hidden-output failures should override the documented group silent-reply contract.

## What This Changes

The issue asks OpenClaw to retry reasoning-only no-visible-text assistant turns in group chats instead of treating them as silent replies under the default group silent-reply policy.

## Best Possible Solution

Maintainers should decide the group silent-reply contract; if approved, split reasoning-only hidden-output failures from deliberate silence in the helper, update focused runner coverage, and align the group docs.

## Reproduction Assessment

Yes, at source level: current main wires default group silent replies into the runner, classifies reasoning-only turns as silent-eligible, and then suppresses the reasoning-only retry. I did not run a live Telegram group reproduction in this read-only review.

## Solution Assessment

Unclear pending maintainer direction: the narrow helper/test change is plausible, but it changes the documented group silent-reply behavior rather than only fixing an isolated implementation typo.
