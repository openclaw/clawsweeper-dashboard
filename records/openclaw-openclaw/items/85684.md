## What This Changes

The issue asks OpenClaw to retry group-chat reasoning-only turns with visible-answer continuation instead of swallowing them under the default silent-reply policy.

## Best Possible Solution

Maintainers should decide the group silent-reply contract; if approved, split deliberate silence from reasoning-only hidden-output failure in the helper and update focused runner tests and group docs together.

## Reproduction Assessment

Yes. Current main source and tests show a reasoning-only assistant turn with `allowEmptyAssistantReplyAsSilent: true` skips retry instructions and returns `NO_REPLY` after one attempt.

Unclear pending maintainer direction. The proposed helper/test change is the narrow code path, but current docs/tests intentionally describe reasoning-only group turns as silent-eligible, so the best fix depends on the desired product contract.
