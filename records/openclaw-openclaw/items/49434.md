## Summary

Keep open: current main still has account-scoped Discord binding lookup, and this PR is the canonical fix path, but the PR head still has a blocking stale-config false-drop risk and lacks live Discord multi-bot proof.

## What This Changes

The branch adds a Discord cross-account active thread-binding lookup, suppresses preflight routing for non-owning bot accounts, exports the helper, and adds focused regression tests.

## Best Possible Solution

Land a current-main Discord fix that enforces exclusive active-thread ownership across Discord accounts, uses the current applicable config for stale-owner filtering, preserves same-account behavior, and includes focused regression coverage plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves Discord bindings through the inbound account id and account-scoped thread manager, so another configured bot can miss an active binding before generic routing proceeds.

## Solution Assessment

No. The direction is right, but PR head should pass current/fresh config into the foreign-owner check and be reconciled with current main before merge.
