## Summary

Keep this PR open: the Discord thread isolation bug is still present on current main, but the branch still needs a stale-config fix, live multi-bot proof, and reconciliation with current main before it is mergeable.

## What This Changes

The PR adds a Discord cross-account thread-binding lookup, suppresses preflight routing for non-owning bot accounts, exports the helper, and adds regression tests.

## Best Possible Solution

Land a rebased Discord fix that enforces exclusive active-thread ownership across accounts, filters stale owners with the current config semantics, preserves same-account routing, and includes focused tests plus redacted live proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves Discord bindings through the inbound account id and the thread manager key includes accountId, so a second bot can miss another account's active thread binding before generic routing proceeds.

## Solution Assessment

No. The direction is right, but PR head should use the applicable current config for foreign-owner filtering and be reconciled with current main before merge.
