## Summary

Keep open. The Discord multi-bot thread-isolation bug is still valid, but this PR is not ready because the new foreign-owner suppression uses the preflight/startup config and can drop valid messages after a foreign account is disabled or removed.

## What This Changes

Adds a Discord thread-binding lookup by thread id, drops messages for non-owning Discord accounts when another account owns an active binding, and adds preflight regression tests.

## Best Possible Solution

Implement exclusive cross-account Discord thread ownership in the current routing path using active runtime config for stale-owner checks, with focused regressions and an Unreleased changelog entry.

## Reproduction Assessment

Yes. The source path is high-confidence: two Discord bot accounts can observe the same thread, bot A owns the bound subagent thread, and current main only resolves bot B's account-scoped binding before generic routing.

## Solution Assessment

No. The PR targets the right owner boundary, but the suppression should consult active runtime config or an equivalent fresh config selector and should not lock in startup/preflight config behavior.
