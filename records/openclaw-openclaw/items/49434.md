## Summary

Keep open: current main still has account-scoped Discord thread-binding routing, and this PR is the canonical fix path, but the latest head is not mergeable as written because it can drop valid messages after live config changes and lacks real multi-bot proof.

## What This Changes

The PR adds Discord preflight suppression for thread messages owned by another account's active thread binding, exports a cross-account binding lookup, and adds regression tests for active and stale foreign bindings.

## Best Possible Solution

Adapt the cross-account lookup on current main, use live runtime config for stopped or stale foreign owners, preserve same-account routing, and land with focused regressions plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, at source level. Current main resolves Discord bindings through the current account and the thread manager lookup is account-scoped, so a second bot can miss another account's active thread binding before generic routing continues.

## Solution Assessment

No, not as written. Cross-account exclusive suppression is the right direction, but stale foreign-owner filtering should use live runtime config rather than the handler's startup preflight snapshot, and the contributor still needs real multi-bot proof.
