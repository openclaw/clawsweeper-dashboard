## Summary

Keep open: the Discord multi-bot thread-isolation bug is valid and this PR is the canonical remaining fix, but the latest head is not mergeable as-is because it still has a stale-config suppression bug, is conflicting with current main, and lacks real Discord multi-bot proof.

## What This Changes

The branch adds a Discord cross-account active thread-binding lookup, suppresses non-owning account routing in preflight, exports the helper, and adds focused regression tests for stale or disabled foreign bindings.

## Best Possible Solution

Land a current-main Discord fix that enforces exclusive active-thread ownership across Discord accounts, uses live/applicable config for stale-owner filtering, preserves same-account behavior, and includes focused regression coverage plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves Discord bindings through the inbound account id and account-scoped thread manager, so a second bot account can miss another account's active thread binding before generic routing proceeds.

## Solution Assessment

No. The direction is right, but the PR head passes stale `params.cfg` into the new foreign-owner lookup and also needs current-main reconciliation plus real Discord multi-bot proof.
