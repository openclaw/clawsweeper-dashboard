## What This Changes

Adds a Discord cross-account thread-binding lookup, suppresses non-owning bot routing for bound threads, exports the helper, and adds focused preflight regression tests.

## Best Possible Solution

Land a rebased Discord fix that enforces exclusive active-thread ownership across accounts using current config semantics, preserves same-account routing, and includes focused tests plus redacted live multi-bot Discord proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves Discord bindings through the inbound account id and the thread manager key also includes account id, so another bot can miss an active foreign binding and fall through to generic routing.

## Solution Assessment

No. The direction is right, but the PR head should use the fresh/current config for foreign-owner filtering and be rebased onto current main before merge.
