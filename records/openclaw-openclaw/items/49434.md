## Summary

Keep open. Current main still lacks the cross-account Discord thread-owner suppression this PR targets, but the PR head still uses a stale preflight config for the new foreign-owner lookup and the external real-behavior proof gate is not satisfied.

## What This Changes

The PR adds cross-account Discord thread-binding lookup and preflight suppression so non-owning bot accounts drop messages in another account's active bound thread, plus regression tests for stale or disabled foreign bindings.

## Best Possible Solution

Land a current-main version of the active-thread exclusive ownership check that uses live config for stale-owner filtering, keeps same-account binding behavior unchanged, and includes focused regression coverage plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, source-reproducible. In a two-bot Discord setup, Bot A can own a bound thread while Bot B's inbound preflight resolves bindings using Bot B's accountId, so current main cannot see Bot A's binding before falling through to generic routing.

## Solution Assessment

No. The direction is right, but the submitted branch should pass the fresh/current config into the foreign-owner lookup, be adapted to current main's preflight routing split, and include real Discord multi-bot proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use fresh config for foreign owner checks:** `extensions/discord/src/monitor/message-handler.preflight.ts:368`
  - body: The function loads `freshCfg` for route and configured-binding decisions, but this suppression passes `params.cfg`. After a foreign account is removed, disabled, or turns off thread bindings and its manager stops, the stale handler snapshot can still mark its old binding active and drop valid messages for another bot until expiry. Pass `freshCfg` into `findActiveThreadBindingByThreadId` and cover the hot-config case.
