## Summary

The PR should stay open because the Discord multi-bot thread-isolation bug is still source-reproducible on current main, but the branch is conflicting, still has a message-delivery correctness issue in the new suppression path, and lacks required live multi-bot Discord proof.

## What This Changes

The PR adds a cross-account Discord thread-binding lookup, suppresses non-owning bot routing for active bound threads, exports the helper, and adds focused preflight regression tests.

## Best Possible Solution

Land a rebased Discord fix that aligns foreign-owner filtering with the same current config snapshot used by preflight routing, preserves same-account binding behavior, and includes focused tests plus redacted live multi-bot Discord proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves Discord bindings through the inbound account id and the thread manager key also includes account id, so another bot can miss an active foreign binding and fall through to generic routing. I did not run a live two-bot guild reproduction in this read-only review.

## Solution Assessment

No, not yet. The direction is maintainable, but the branch needs a current-main rebase, the foreign-owner check must use the same config snapshot as preflight routing, and live multi-bot Discord proof is still missing.
