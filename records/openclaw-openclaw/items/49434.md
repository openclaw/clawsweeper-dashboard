## What This Changes

The branch adds a Discord cross-account thread-binding lookup, suppresses generic route fallback for non-owning bot accounts, exports the helper, and adds preflight regression tests.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main resolves Discord bindings through the inbound account id, and the thread manager lookup is also account-scoped, so the reported two-bot blind spot is source-reproducible without a live guild run.

## Solution Assessment

No, not as-is. The direction is maintainable, but the branch needs a current-main rebase, the foreign-owner check must use the same fresh config snapshot as route resolution, and live multi-bot Discord proof is still missing.
