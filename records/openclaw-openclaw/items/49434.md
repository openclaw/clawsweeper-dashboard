## Summary

Keep this PR open: the bug is real and source-reproducible on current main, but the branch still has a blocking stale-config false-drop path and lacks required real Discord multi-bot behavior proof.

## What This Changes

The PR adds cross-account Discord thread-binding lookup and preflight suppression so non-owning bot accounts drop messages in another account's active bound thread, with regression tests for stale and disabled foreign bindings.

## Best Possible Solution

Land a current-main Discord fix that enforces exclusive active-thread ownership with live/applicable config for stale-owner filtering, preserves same-account behavior, and includes focused regression coverage plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves Discord thread/session bindings through the inbound account id and account-scoped thread manager, so Bot B cannot see Bot A's active binding before generic routing proceeds.

## Solution Assessment

No. The direction is right, but this patch still uses the stale handler config for foreign-owner filtering and needs current-main reconciliation plus real Discord multi-bot proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use live config for foreign owner checks:** `extensions/discord/src/monitor/message-handler.preflight.ts:368`
  - body: The new suppression path passes `params.cfg` into `findActiveThreadBindingByThreadId` even though this function has already loaded the fresh config for routing.
