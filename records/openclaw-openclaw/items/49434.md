## Summary

Keep this PR open: the Discord multi-bot thread isolation bug is still source-reproducible on current main, but the submitted patch has a real stale-config false-drop bug and lacks required live multi-bot proof from the contributor.

## What This Changes

The PR adds cross-account Discord thread-binding lookup and preflight suppression so non-owning bot accounts drop messages in another account's active bound thread, with tests for stale and disabled foreign bindings.

## Best Possible Solution

Land a current-main version of exclusive active-thread ownership that uses live config for stale-owner filtering, preserves same-account behavior, and includes focused regression coverage plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves Discord bindings through the inbound account's adapter and manager, so a thread bound by Bot A is invisible to Bot B before generic routing proceeds.

## Solution Assessment

No. The direction is right, but this patch still needs the live-config stale-owner fix, reconciliation with current main's preflight split, and real Discord multi-bot proof before merge.
