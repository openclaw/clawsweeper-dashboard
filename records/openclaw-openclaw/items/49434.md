## Summary

Keep open: current main still has account-scoped Discord thread-binding resolution, and this PR is the canonical fix path, but the patch still has a blocking stale-config suppression bug and lacks required live multi-bot Discord proof.

## What This Changes

The branch adds a Discord cross-account active thread-binding lookup, suppresses non-owning account routing during preflight, exports the lookup helper, and adds focused regression tests.

## Best Possible Solution

Land a current-main Discord fix that enforces exclusive active-thread ownership across Discord accounts, uses live applicable config for stale-owner filtering, preserves same-account behavior, and includes focused regression coverage plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves Discord bindings through the inbound account id and account-scoped thread manager, so a second bot account can miss another account's active thread binding before generic routing proceeds.

## Solution Assessment

No. The direction is right, but the PR head should use fresh/current config for foreign-owner filtering and still needs current-main reconciliation plus real Discord multi-bot proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87
