## Summary

Keep open. Current main still lacks cross-account Discord thread-owner suppression, so the PR remains relevant, but the latest patch is not ready because it checks stale foreign owners against `params.cfg` instead of the active runtime config and it omits the required changelog entry.

## What This Changes

The PR adds Discord preflight logic and a thread-binding manager lookup to drop thread messages for non-owning bot accounts when another Discord account owns an active binding, with regressions for active, stale, removed, and disabled foreign bindings.

## Best Possible Solution

Implement the Discord fix in the active preflight/routing layout with a cross-account active-owner lookup that uses live config to ignore stale or disabled owners, then add focused tests and a changelog entry.

## Reproduction Assessment

Yes. The source-level path is high-confidence: run two Discord bot accounts that can see the same thread, bind the thread under bot A with `sessions_spawn(thread=true, mode="session")`, then send a follow-up visible to bot B; current main only checks bot B's account-scoped binding adapter and can fall through to generic routing.

## Solution Assessment

No, not as written. The PR targets the right owner boundary, but the foreign-owner stale-binding decision should use the active runtime config or config selector instead of the startup/preflight config.

## Review Findings
