## Summary

Keep this PR open: current main still has the account-scoped Discord thread-binding blind spot and the linked issue was closed in favor of this canonical PR, but the branch conflicts with current main, lacks required real Discord proof, and still has a P1 message-delivery defect.

## What This Changes

The PR adds a cross-account Discord thread-binding lookup, drops generic routing for non-owning bot accounts in bound threads, exports the helper, and adds preflight regression tests.

## Best Possible Solution

Rebase the Discord fix onto current main, keep exclusive bound-thread ownership only for active bindings in the modern routing-preflight path, use live config for stale foreign-owner decisions, and require focused regressions plus redacted two-bot Discord proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main resolves Discord bindings through the inbound account id, and the thread manager keys `getByThreadId` by account id. I did not run a live two-bot Discord guild reproduction in this read-only review.

## Solution Assessment

No, not as currently proposed: the direction is plausible, but the branch needs a rebase onto current main and the foreign-owner stale-binding check should use live config. The safer end state is an owner-bound cross-account lookup with tests for active owners, stopped managers, disabled accounts, and hot config changes.
