## Summary

Keep open: current main still lacks cross-account Discord thread ownership suppression, and this PR is useful, but the latest head still uses stale preflight config for the foreign-owner check and can drop valid Discord messages after runtime config changes; it also needs changelog coverage.

## What This Changes

The PR adds a cross-account active Discord thread-binding lookup in preflight, exports it, and adds regressions for foreign-owner suppression plus stale, removed, and disabled owner bindings.

## Best Possible Solution

Land a current-main adaptation that preserves exclusive active thread ownership across Discord accounts, consults live runtime config when filtering stale owners, keeps same-account behavior unchanged, and includes focused regressions plus a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main resolves session bindings with the current Discord accountId and the manager getByThreadId lookup is account-scoped, so a second bot seeing the same thread can miss the active owner and continue to generic routing.

## Solution Assessment

No, not as written. The exclusive foreign-owner suppression direction is the narrow maintainable fix, but the branch should use active runtime config for stale-owner filtering and add the required changelog entry before merge.
