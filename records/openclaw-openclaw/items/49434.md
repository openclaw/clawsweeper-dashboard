## What This Changes

The PR adds a cross-account Discord thread-binding lookup, suppresses non-owning bot routing for active bound threads, exports the helper, and adds focused preflight regression tests.

Yes. Current main resolves Discord bindings with the inbound account id and the thread manager lookup is account-keyed, so a second bot account cannot see a foreign active binding and can fall through to generic routing; I did not run a live two-bot guild reproduction.

## Solution Assessment

No, not as currently proposed. The direction is maintainable, but the branch must be rebased onto current main, use one config snapshot for foreign-owner filtering, and include live multi-bot Discord proof.
