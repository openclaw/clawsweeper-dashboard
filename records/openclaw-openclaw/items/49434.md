## Summary

Keep open: current main still lacks cross-account Discord thread-ownership suppression, but this PR head is not merge-ready because it still uses the stale handler config for foreign-owner filtering and the external contributor has not provided real Discord multi-bot behavior proof.

## What This Changes

The branch adds a Discord cross-account thread-binding lookup, preflight suppression for non-owning bot accounts, an export, and regression tests for stale or disabled foreign bindings.

## Best Possible Solution

Land a current-main Discord fix that enforces exclusive active-thread ownership with live/applicable config for stale-owner filtering, preserves same-account behavior, and includes focused regression coverage plus redacted live multi-bot proof.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves Discord thread/session bindings through the inbound account id and account-scoped thread manager, so a second bot account cannot see another account's active binding before generic routing proceeds.

## Solution Assessment

No. The direction is right, but the current PR head passes stale params.cfg into the foreign-owner lookup even after loading freshCfg for routing, and it still needs current-main reconciliation plus real Discord multi-bot proof.
