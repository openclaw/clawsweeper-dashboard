Keep open for maintainer review. Current main still has the cron destination-session transcript gap, while this PR supplies a focused fix, regression coverage, green CI, and real WhatsApp after-fix proof; the main remaining action is merge sequencing with the overlapping custom-session PR.

## What This Changes

The PR resolves direct cron announce delivery to the destination outbound session, mirrors delivered assistant text into that transcript after successful delivery, tightens fallback target classification, and adds cron/outbound regression tests plus two provider-registry test isolation fixes.

## Best Possible Solution

Land one coordinated cron-owned fix that mirrors only successfully delivered final payloads into the same durable destination session normal outbound sends use, while preserving isolated worker-session boundaries and avoiding duplicate main-session records.

## Reproduction Assessment

Yes. Current main is source-reproducible because direct cron delivery sends with the cron session context and only queues main-session awareness, and the linked before-fix proof shows the delivered WhatsApp token missing from the destination DM transcript.

## Solution Assessment

Yes. Resolving the outbound destination session before send and appending the delivered payload projection after full success is the narrow maintainable fix; the safer maintainer action is to coordinate landing with https://github.com/openclaw/openclaw/pull/74913.
