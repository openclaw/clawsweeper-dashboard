## Summary

Keep open. The report has a concrete packaged-release failure, and current main still has a plausible generic bundled-channel loader gap: startup stages and registers runtime dependencies, but the channel-entry loader path used for bundled Slack/Telegram plugin materialization can still opt out of runtime-deps preparation and does not visibly consume the runtime dependency Jiti alias map that the main plugin loader uses.

## What This Changes

Slack and Telegram bundled channel providers should resolve `json5` from the staged `plugin-runtime-deps` root after gateway startup instead of crashing under the provider launcher.

## Best Possible Solution

Fix the generic bundled plugin runtime-dependency loading path so staged root dependencies and Jiti aliases are honored consistently for bundled channel entry/runtime loads, then cover Slack/Telegram through a packaged staged-deps smoke rather than adding Slack- or Telegram-specific `json5` workarounds.

## Reproduction Assessment

Yes. The issue gives a high-confidence packaged reproduction path on OpenClaw 2026.4.29 with Slack/Telegram enabled, and source inspection identifies the loader path that can diverge from direct Node resolution.

## Solution Assessment

Yes, a generic bundled runtime-deps loader fix is the right direction. Bundling or inlining `json5` only for Slack/Telegram would mask the shared staged-dependency resolution bug and leave other mirrored root chunks exposed.

## Repair Work Prompt

Fix the bundled channel/plugin runtime-dependency resolution regression reported in https://github.com/openclaw/openclaw/issues/75623. The observable bug is that packaged OpenClaw 2026.4.29 stages `json5` under `plugin-runtime-deps/<version-hash>/node_modules`, direct Node imports from that staged root work, but Slack and Telegram provider startup crashes resolving `json5` from a mirrored root `dist/frontmatter-*.js` chunk. Keep the fix generic: do not special-case Slack, Telegram, or `json5`; do not add core knowledge of bundled plugin ids beyond existing generic manifest/runtime-deps seams. Inspect `src/gateway/server-startup-plugins.ts`, `src/plugins/bundled-runtime-root.ts`, `src/plugins/bundled-runtime-staging.ts`, `src/plugins/loader.ts`, `src/plugin-sdk/channel-entry-contract.ts`, and `src/channels/plugins/module-loader.ts`/`bundled.ts`. Ensure staged runtime dependency Node paths and Jiti alias maps are honored by bundled channel entry/runtime loads that currently use `installRuntimeDeps: false` after pre-stage. Add a focused regression test that simulates a packaged external stage root with a mirrored root chunk importing `json5` through the bundled channel entry path, not only direct Node import. Add a user-facing `CHANGELOG.md` fix entry with reporter credit to @cjbubblz.
