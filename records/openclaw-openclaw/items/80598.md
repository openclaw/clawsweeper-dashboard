## Summary

Keep open: this is a valid narrow backup bug-fix PR, but the current head still has blocking signal-cleanup correctness issues and the supplied proof does not show the real interrupted CLI path.

## What This Changes

The PR moves backup temp archive creation into the staging temp directory, adds CLI SIGINT/SIGTERM cleanup wiring, updates backup docs/changelog, and adds a temp-path regression test.

## Best Possible Solution

Keep the temp archive under the staging temp root, expose an actual synchronous cleanup closure for CLI signal handlers, preserve 130/143 signal exit semantics, and prove the interrupted CLI path with redacted terminal/log output.

## Reproduction Assessment

Yes, source-backed: current main writes the active tar output beside `outputPath` and cleans it only during normal `finally` unwinding. Running `openclaw backup create` long enough to enter `tar.c`, then terminating it before normal unwinding, matches the orphaned `.tmp` report.

## Solution Assessment

No. Moving the temp archive into the staging temp directory is the right direction, but the current PR does not wire a real cleanup function to the signal handler and returns success on interruption.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass the real cleanup closure to onCleanup:** `src/infra/backup-create.ts:504`
  - body: `onCleanup` is supposed to give the CLI a synchronous cleanup function, but this passes a setter-like callback instead. The CLI stores that function as `signalCleanup`; on SIGINT/SIGTERM it is called with no argument and never removes `tempArchivePath` or `tempDir`, and the intended `rmSync` import remains unused under `noUnusedLocals`.
  - confidence: 0.96
- **[P2] Exit with signal-derived status codes:** `src/commands/backup.ts:30`
  - body: The signal handler exits with `0` for both SIGINT and SIGTERM, so an interrupted backup can look successful to scripts or supervisors even though no archive was completed. Use the existing CLI convention of 130 for SIGINT and 143 for SIGTERM.
  - confidence: 0.86
