## Summary

Keep open: the PR is aimed at a valid backup bug, but the latest head still has merge-blocking correctness issues in the signal cleanup path and lacks sufficient after-fix interrupted-run proof.

## What This Changes

The PR moves `openclaw backup create` temp archive creation into the staging temp directory, adds CLI SIGINT/SIGTERM cleanup wiring, updates backup docs/changelog, and adds a temp-path regression test.

## Best Possible Solution

Keep the temp archive under the staging temp root, pass a real synchronous cleanup closure to CLI-owned signal handlers, preserve SIGINT/SIGTERM exit semantics, and prove the interrupted CLI path with redacted terminal/log output.

## Reproduction Assessment

Yes, source-backed: current main writes the active tar output beside `outputPath` and only cleans it in `finally`. Running `openclaw backup create` long enough to enter `tar.c`, then terminating it before normal unwinding, matches the linked report's orphaned `.tmp` files.

## Solution Assessment

No. Moving the temp archive is the right direction, but this patch currently passes the wrong callback shape to `onCleanup` and exits successful on signal interruption.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass an actual cleanup closure to onCleanup:** `src/infra/backup-create.ts:501`
  - body: `onCleanup` is supposed to give the CLI a synchronous cleanup function, but this passes a setter-like callback instead. The CLI stores that function as `signalCleanup`; on SIGINT/SIGTERM it is called with no argument and never removes `tempArchivePath` or `tempDir`, so the signal-cleanup part of the fix does not work. It also leaves the new `rmSync` import unused under `noUnusedLocals`.
  - confidence: 0.96
- **[P2] Return signal exit codes instead of success:** `src/commands/backup.ts:33`
  - body: The new handler calls `process.exit(0)` for both SIGINT and SIGTERM, so an interrupted backup can look successful to a shell script or supervisor even though no archive was completed. Use the usual signal-derived codes, matching the existing CLI shutdown path: 130 for SIGINT and 143 for SIGTERM.
  - confidence: 0.86
