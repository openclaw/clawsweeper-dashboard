## Summary

Keep open: current main still has the reported adjacent backup temp-file behavior, but this PR is not mergeable yet because the patch does not compile, puts process-exit signal handling inside an exported helper, overpromises cleanup in docs, and still lacks convincing after-fix real interruption proof.

## What This Changes

The PR moves the backup temp archive into the staging temp directory, adds SIGINT/SIGTERM cleanup, updates backup docs and changelog, and adds a focused atomic-write regression test.

## Best Possible Solution

Move the active archive under the selected staging temp root, keep graceful-interruption cleanup owned by the CLI or injectable runtime boundary, document hard-kill limits, and prove it with targeted regression coverage plus redacted real interrupted-run output.

## Reproduction Assessment

Yes, source-backed: current main derives the temp archive from the output path and only removes it during normal `finally` cleanup, matching the linked timeout report. I did not run a live signal repro in this read-only review.

## Solution Assessment

No. Moving the active archive into the staging temp directory is the right direction, but this implementation must compile, avoid process exits in the shared helper, narrow the docs claim, and include stronger real interrupted-run proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import rmSync before using signal cleanup:** `src/infra/backup-create.ts:497-498`
  - body: The new signal cleanup calls `rmSync`, but the PR head still imports only `constants as fsConstants` from `node:fs`. That leaves `rmSync` undefined for typecheck/build, so the backup command cannot compile until the import is fixed.
  - confidence: 0.98
- **[P2] Keep process exits out of the archive helper:** `src/infra/backup-create.ts:500-501`
  - body: `createBackupArchive` is exported through `openclaw/plugin-sdk/runtime`, so installing SIGINT/SIGTERM handlers here that call `process.exit()` can bypass an embedding caller's graceful shutdown. Put process-exit behavior in the CLI layer or make cleanup injectable while keeping this helper non-exiting.
  - confidence: 0.86
- **[P3] Narrow the cleanup guarantee in backup docs:** `docs/cli/backup.md:60`
  - body: The docs promise cleanup for timeout or signal interruption, but this patch only handles graceful SIGINT/SIGTERM and cannot clean up after SIGKILL, crashes, or hard timeout kills. Please describe the guarantee as graceful-interruption cleanup.
  - confidence: 0.78
