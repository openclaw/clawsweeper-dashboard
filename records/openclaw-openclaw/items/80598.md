## Summary

Keep open: this is a valid narrow backup bug, but the current PR head has a compile blocker and needs more work before merge.

## What This Changes

The PR moves `openclaw backup create`'s active temp archive into the backup staging temp directory, adds SIGINT/SIGTERM cleanup, and updates backup docs, changelog, and an atomic-write regression test.

## Best Possible Solution

Move the active archive into the selected backup temp directory, add scoped graceful-interruption cleanup without forcing `process.exit()` from the shared helper, document hard-kill limits, and prove it with targeted tests plus redacted real interrupted-run output.

## Reproduction Assessment

Yes, source-backed: current `origin/main` derives the temp archive from `outputPath` and removes it only in `finally`, so termination before unwinding can leave the adjacent `.tmp` file. I did not run a live signal repro because this review is read-only.

## Solution Assessment

No. Moving the temp archive into the staging temp directory is the right direction, but this PR is not mergeable as-is because it does not compile and puts process-exit signal behavior inside a shared helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import rmSync before using signal cleanup:** `src/infra/backup-create.ts:497-498`
  - body: The patch calls `rmSync` in the new signal cleanup handler, but the PR head still imports only `constants as fsConstants` from `node:fs`. That leaves `rmSync` undefined at typecheck/build time, so the backup command cannot compile.
  - confidence: 0.98
- **[P2] Keep process exits out of the shared archive helper:** `src/infra/backup-create.ts:500-501`
  - body: This helper is exported through `openclaw/plugin-sdk/runtime` and used by Matrix migration snapshots, so SIGINT/SIGTERM handlers here that call `process.exit()` can bypass an embedding caller's graceful shutdown. Put process-exit behavior in the CLI wrapper or make cleanup injectable while leaving `createBackupArchive` non-exiting.
  - confidence: 0.86
- **[P3] Narrow the temp-file cleanup guarantee:** `docs/cli/backup.md:60`
  - body: The docs promise cleanup for any timeout or signal, but the patch only installs JS handlers for SIGINT/SIGTERM and cannot run on SIGKILL, crashes, or hard timeouts. Please describe this as graceful interruption cleanup so users do not rely on an impossible guarantee.
  - confidence: 0.78
