## Summary

Keep open: the backup bug is valid and the PR is aimed at the right surface, but the latest head still has a concrete signal-cleanup implementation bug and only mocked test proof. The earlier changelog deletion was addressed on the updated head.

## What This Changes

The PR moves backup temp archive creation into the staging temp directory, adds SIGINT/SIGTERM cleanup, adds an atomic backup regression test, and updates backup docs and changelog.

## Best Possible Solution

Keep the temp archive under a backup-owned temp root, implement cleanup without surprising SDK or in-process callers, add focused signal-path coverage, and refresh real behavior proof before merge.

## Reproduction Assessment

Yes, source-backed: current main writes the active archive as `<outputPath>.<uuid>.tmp` and cleans it only in `finally`, so terminating during tar can bypass cleanup. I did not run the live signal path in this read-only review.

## Solution Assessment

No. Moving the temp archive into the staging temp directory is the right direction, but this implementation uses an unimported `rmSync` and puts `process.exit()` inside an exported helper rather than a CLI-owned signal boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import rmSync before using signal cleanup:** `src/infra/backup-create.ts:497-498`
  - body: The new signal cleanup calls `rmSync`, but this module still only imports `constants` from `node:fs` and `fs/promises`. A focused test that never emits the signal can pass, but the SIGINT/SIGTERM path will throw `ReferenceError` instead of removing the temp archive, and TypeScript checks should reject the unbound name.
  - confidence: 0.99
- **[P2] Keep signal exits out of the exported archive helper:** `src/infra/backup-create.ts:500-501`
  - body: `createBackupArchive` is exported through the plugin SDK runtime, but the added handler calls `process.exit()` from inside that shared helper. Any in-process caller that starts a backup would now have its host process terminated on SIGINT/SIGTERM before its own signal handlers or cleanup can run; put the process-level exit behavior at the CLI boundary or make the cleanup hook injectable.
  - confidence: 0.86
