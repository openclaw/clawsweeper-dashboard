## Summary

Keep open: the latest push preserves the changelog entry and moves signal handling toward the CLI boundary, but the PR head still wires `onCleanup` backward, exits with success on signals, and lacks sufficient proof of a real interrupted CLI run.

## What This Changes

The PR stages backup temporary archives under the system temp directory, adds CLI SIGINT/SIGTERM cleanup wiring, updates backup docs/changelog, and adds a temp-path regression test.

## Best Possible Solution

Keep the temp archive under the staging temp root, expose a real synchronous cleanup closure that deletes the temp archive and staging dir, preserve signal-derived exit codes, and prove the interrupted CLI path with redacted terminal/log output.

## Reproduction Assessment

Yes. Source inspection shows current main writes the active tar output beside `outputPath` and only cleans it in `finally`; running `openclaw backup create` until tar writing begins, then terminating before normal unwinding, matches the linked orphaned `.tmp` report.

## Solution Assessment

No. Moving the temp archive into the staging temp directory is the right direction, but the current PR does not pass a real cleanup handler to the CLI signal hook and makes interrupted backups exit successfully.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass a real cleanup handler to onCleanup:** `src/infra/backup-create.ts:504`
  - body: `onCleanup` is meant to give the CLI a synchronous cleanup closure, but this passes a setter-like function instead. The CLI stores that function as `signalCleanup`, so SIGINT/SIGTERM calls it with no argument and never removes `tempArchivePath` or `tempDir`; the unused `rmSync` import also makes this likely to fail core type checking.
  - confidence: 0.96
- **[P2] Return signal exit codes instead of success:** `src/commands/backup.ts:30`
  - body: The signal handler exits with `0` for both SIGINT and SIGTERM, so an interrupted backup can look successful to scripts or supervisors even though no archive was completed. Use signal-derived codes such as 130 for SIGINT and 143 for SIGTERM, matching existing CLI shutdown behavior.
  - confidence: 0.88
