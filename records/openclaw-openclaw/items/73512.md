## Summary

Keep this PR open, but not as-is: the patch changes the documented BM25-only QMD contract and still treats the default resolved embed interval as explicit user intent, so it needs maintainer product review and a narrower implementation before merge. https://github.com/openclaw/openclaw/pull/73512

## What This Changes

The PR changes QMD embed execution, timer scheduling, and tests so a positive resolved embed interval can run `qmd embed` even when `searchMode` is `search`.

## Best Possible Solution

Settle the intended QMD config contract first; if explicit `embedInterval` should opt into embedding, preserve explicitness in resolved config and update tests, docs, and changelog to match.

## Reproduction Assessment

Yes, source-level. Current main has a lexical search-mode test with `embedInterval: "5m"` expecting no QMD update/embed calls, and the docs define `searchMode: "search"` as BM25-only with no embedding maintenance.

## Solution Assessment

No. The PR guard is not the best fix because it uses a resolved positive interval as explicit intent; the safer path needs explicit-config tracking plus docs/tests aligned with the chosen contract.
