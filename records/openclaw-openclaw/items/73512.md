## Summary

Keep open: the PR targets a real reported QMD pain point, but the current patch changes the documented lexical-mode contract and infers explicit user intent from a resolved default interval, so it needs maintainer product review before merge.

## What This Changes

The PR changes QMD embed execution and timer scheduling so a positive resolved embed interval bypasses lexical search-mode gating, with colocated QMD manager tests updated for that behavior.

## Best Possible Solution

Settle the lexical QMD/vector-mode contract first; if explicit `embedInterval` is accepted as an opt-in, preserve raw-config explicitness in resolved QMD config and align tests, docs, and changelog.

## Reproduction Assessment

Yes, source-level. Current main has a focused `searchMode: "search"` plus `embedInterval: "5m"` test expecting no QMD commands, while this PR adds the opposite expectation for the same effective setup.

## Solution Assessment

No. The proposed guard is not the narrowest maintainable fix because it treats the default resolved embed interval as explicit user intent; a safer path needs explicit-config tracking and contract-aligned docs/tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Track explicit embedInterval before bypassing search mode:** `extensions/memory-core/src/memory/qmd-manager.ts:1649`
  - body: The new guard uses the resolved `embedIntervalMs > 0` value to infer opt-in, but `resolveEmbedIntervalMs(undefined)` returns the default 60m and resolved config stores no explicitness flag. Default lexical QMD configs can therefore run `qmd embed` through boot or first update paths, contrary to the current BM25-only contract.
  - confidence: 0.94
- **[P3] Keep the shared QMD fixture on production defaults:** `extensions/memory-core/src/memory/qmd-manager.test.ts:238`
  - body: Adding `embedInterval: "0s"` to the shared fixture means most tests no longer exercise production's default 60m embed interval. That masks the default lexical-mode regression above; disable embeds only in tests that require that setup.
  - confidence: 0.83
