## Summary

Keep open: the PR addresses useful context from #73432, but current main defines QMD `searchMode: "search"` as BM25-only in code, tests, and docs, and this patch would bypass that contract based on a defaulted embed interval rather than explicit user intent.

## What This Changes

The branch changes QMD embed execution and timer scheduling, plus QMD manager tests, so a positive resolved embed interval can run embedding in `searchMode: "search"`.

## Best Possible Solution

Decide the QMD lexical-mode contract first; if the override is accepted, carry explicit `embedInterval` intent through config resolution, preserve default lexical no-embed behavior, and align tests, docs, and changelog.

## Reproduction Assessment

Yes, source-level. Current main has a focused `searchMode: "search"` plus `embedInterval: "5m"` test expecting no QMD commands, while this PR adds the opposite expectation for the same effective setup.

## Solution Assessment

No. The proposed guard infers explicit user intent from a resolved default value; a safer implementation would preserve raw-config explicitness and update the documented contract only if maintainers accept the behavior change.
