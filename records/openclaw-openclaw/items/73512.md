## Summary

Keep open. This is a plausible focused fix for https://github.com/openclaw/openclaw/issues/73432, but the current patch still changes the documented lexical QMD contract, loses explicit `embedInterval` intent after config resolution, and lacks after-fix real behavior proof.

## What This Changes

The PR changes QMD embed run/timer guards so a positive resolved embed interval schedules `qmd embed` in lexical `search` mode, and adjusts QMD manager tests around that behavior.

## Best Possible Solution

Carry explicit `embedInterval` intent through resolved QMD config and align docs/tests if maintainers want lexical mode to opt into embedding; otherwise keep lexical mode BM25-only and direct users to `query` or `vsearch` for embedding maintenance.

## Reproduction Assessment

Yes, source-level. Current main resolves omitted/default QMD search mode to lexical `search`, then the manager blocks both embed execution and timer scheduling even with a positive `embedInterval`; the focused current test preserves that behavior.

## Solution Assessment

No. The proposed guard uses resolved `embedIntervalMs`, which defaults to 60 minutes and loses explicitness; the safer path is an explicit-config signal plus maintainer-confirmed docs/test semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Track explicit embedInterval before bypassing search mode:** `extensions/memory-core/src/memory/qmd-manager.ts:1649`
  - body: The new guard treats `embedIntervalMs > 0` as opt-in, but omitted `embedInterval` resolves to the default 60m and the resolved config stores no explicitness flag. Default lexical QMD configs can therefore start running `qmd embed` after ordinary updates, contradicting the documented BM25-only mode and current tests.
  - confidence: 0.94
- **[P3] Keep the shared QMD fixture on production defaults:** `extensions/memory-core/src/memory/qmd-manager.test.ts:238`
  - body: Adding `embedInterval: "0s"` to the shared fixture means most manager tests stop covering production's default 60m embed cadence. That masks the default lexical-mode regression above; set it only in tests that need embeds disabled.
  - confidence: 0.83
