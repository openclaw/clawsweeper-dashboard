## Summary

Keep this PR open. The core/media plumbing is directionally useful, but the patch needs security fixes and contributor-supplied real behavior proof before maintainer review can proceed.

## What This Changes

The PR propagates agent/media read context into Feishu outbound delivery, expands Feishu local media root resolution, converts local file paths in Feishu text replies into uploads, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the agent-scoped media plumbing, but route Feishu host reads through the shared media load-options contract and narrow path auto-detection so incidental text cannot authorize new local roots without explicit maintainer-approved policy.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show current main lacks non-image text path conversion and the reply dispatcher does not pass scoped local roots for reply media.

## Solution Assessment

No. The context propagation is a reasonable part of the fix, but automatically uploading any existing absolute path found in reply text and bypassing the shared host-read validation path is not the safest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve host-read media validation:** `extensions/feishu/src/media.ts:857`
  - body: Passing `mediaReadFile` directly into `loadWebMedia` does not set `hostReadCapability`, so the host-read validator that rejects unsafe/plaintext-like local documents never runs for this new readFile path. Use the shared `buildOutboundMediaLoadOptions` path or pass the equivalent validated options when Feishu reads host files.
  - confidence: 0.83
- **[P1] Do not auto-upload arbitrary paths from reply text:** `extensions/feishu/src/local-file-reply.ts:56-63`
  - body: The new embedded-path parser treats any existing absolute path mentioned in normal reply text as an attachment candidate; the caller then expands media roots from that path before upload. That turns incidental text like a host path mention into a file upload attempt, whereas current behavior only auto-converts pure image-path payloads.
  - confidence: 0.79
