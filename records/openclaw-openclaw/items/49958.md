## Summary

Keep open: the Feishu command bug is source-reproducible, but this PR changes a shared command-authorization helper too broadly, which can widen unrelated DM/control-command access and leaves root SDK shim semantics inconsistent. The PR also lacks after-fix real Feishu/OpenClaw runtime proof.

## What This Changes

The PR changes the shared command-gating helper so access-groups mode allows commands when no authorizer is configured, and updates command-gating unit tests.

## Best Possible Solution

Land a narrow Feishu or explicit channel-ingress command-authorization fix that authorizes admitted Feishu senders when no command allowlist is configured, while preserving fail-closed behavior for explicit allowlists, owner-only commands, DM command gates, and SDK/root-alias consistency.

## Reproduction Assessment

Yes, at source level: current Feishu command handling can carry an admitted slash command into `CommandAuthorized=false`, and the reply path then drops unauthorized whole-message control commands. I did not run a live Feishu workspace in this read-only review.

## Solution Assessment

No. The intended Feishu fix direction is valid, but this PR's global helper fallback is not the narrowest maintainable solution because shared DM and SDK command gates also depend on the current fail-closed semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

Open-DM paths can build all-unconfigured authorizers when no allowlists are set; this branch would flip those control commands to authorized and bypass current fail-closed tests. Fix this at the Feishu call site or add an explicit scoped mode that preserves the existing DM contract.
  - confidence: 0.94

  - body: Changing only the TypeScript helper leaves the root CJS alias with the old `resolveControlCommandGate` semantics, so root-SDK importers and subpath importers would authorize the same no-config command case differently. If the shared helper semantics change, update `src/plugin-sdk/root-alias.cjs` and its contract tests too.
  - confidence: 0.9
