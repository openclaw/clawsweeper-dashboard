## Summary

Keep open: this PR targets a real Feishu slash-command authorization bug, but the patch broadens a shared command gate beyond Feishu and the external PR only supplies test claims, not real after-fix behavior proof.

## What This Changes

The PR changes shared command-gating semantics to allow all-unconfigured authorizers when access groups are enabled, and updates the helper unit tests.

## Best Possible Solution

Use a narrow Feishu command-authorization fix, or an explicit mirrored helper mode, that authorizes admitted group senders when no command allowlist is configured while preserving DM fail-closed and SDK mirror contracts.

## Reproduction Assessment

Yes, source-reproducible. Current main can route a Feishu group command with no command allowlist into an unconfigured command authorizer that resolves false, and the PR's shared fallback source-reproduces a DM authorization regression against existing tests.

## Solution Assessment

No. A broad shared fallback is not the narrowest maintainable fix; Feishu should handle the no-command-allowlist admitted-sender case directly, or the shared contract must be made explicit and mirrored in the root SDK.

## Review Findings

Full review comments:

- **[P1] Scope the no-config fallback to Feishu group commands:** `src/channels/command-gating.ts:31-32`

  - confidence: 0.86
