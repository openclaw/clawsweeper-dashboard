## Summary

Keep open: the underlying Feishu slash-command bug is still source-reproducible on current main, but this PR is not merge-ready because it changes shared command authorization globally, can widen unrelated DM/SDK command access, and lacks after-fix real Feishu/OpenClaw runtime proof.

## What This Changes

The PR changes the shared command-gating helper to allow all-unconfigured access-group authorizers and updates command-gating unit tests.

## Best Possible Solution

Land a narrow Feishu or explicitly scoped command-authorization fix that authorizes admitted Feishu senders when no command allowlist is configured, while preserving fail-closed behavior for explicit allowlists, owner-only commands, DM command gates, and SDK/root-alias consistency.

## Reproduction Assessment

Yes, at source level: current main can carry an admitted Feishu slash command into `CommandAuthorized=false`, and the reply path drops unauthorized whole-message control commands. I did not run a live Feishu workspace in this read-only review.

## Solution Assessment

No. The intended Feishu fix direction is valid, but this PR's global helper fallback is not the narrowest maintainable solution because shared DM, generic ingress, and SDK command gates also rely on the current fail-closed semantics.
