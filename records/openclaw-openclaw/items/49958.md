## Summary

Keep this PR open, but do not merge the current patch. It targets a real Feishu group-command bug, yet the branch changes a shared command-authorization helper in a way that can authorize control commands outside the intended Feishu group flow and leaves the root SDK mirror inconsistent.

## What This Changes

The PR adds a no-configured-authorizer allow fallback to the shared command-gating helper and updates its unit tests for `useAccessGroups=true`.

## Best Possible Solution

Implement the Feishu no-`allowFrom` group-command fix at a scoped group-command boundary, or add an explicit helper mode that preserves DM fail-closed behavior; align the root SDK mirror if the shared contract changes, add regression coverage, and include a changelog entry.

## Reproduction Assessment

Yes, from source inspection. On current main, Feishu group slash commands with no configured `allowFrom` pass an unconfigured authorizer into the shared helper, and that helper returns false under `useAccessGroups=true`; the current Feishu test still asserts `CommandAuthorized: false`.

## Solution Assessment

No. The proposed patch fixes the Feishu symptom by changing shared `useAccessGroups=true` semantics for every caller and leaving the root SDK CJS mirror unchanged; a scoped group-command fix or explicit helper mode is safer.
