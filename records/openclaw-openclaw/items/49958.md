## Summary

Keep open: the linked Feishu slash-command bug is valid, but this branch is not merge-ready because it broadens a shared command-authorization boundary and lacks after-fix Feishu/OpenClaw runtime proof.

## What This Changes

The PR changes shared command gating so `useAccessGroups=true` allows commands when every authorizer is unconfigured and updates command-gating unit tests.

## Best Possible Solution

Land a narrow Feishu command-authorization fix that authorizes admitted senders when no explicit command allowlist is configured, while preserving explicit allowlist denial and SDK entrypoint consistency.

## Reproduction Assessment

Yes, at source level: current main can carry an admitted Feishu slash command with no command allowlist into `CommandAuthorized=false`, and the reply path then returns no response for unauthorized whole-message control commands. I did not run a live Feishu workspace.

## Solution Assessment

No. The bug-fix direction is valid, but this implementation changes a shared helper too broadly and leaves the root SDK shim divergent; a Feishu-scoped fix or explicit shared mode with parity is safer.
