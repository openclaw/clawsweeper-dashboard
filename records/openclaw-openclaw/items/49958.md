## Summary

Keep open: the underlying Feishu slash-command bug is source-reproducible, but this PR fixes it by broadening shared command authorization and still lacks required after-fix real behavior proof.

## What This Changes

The PR changes the shared command-gating helper to allow all-unconfigured `useAccessGroups` authorizers and updates command-gating unit tests.

## Best Possible Solution

Land a Feishu-scoped command-authorization fix, or an explicit scoped helper mode, that authorizes admitted Feishu senders only when no explicit command allowlist is configured while preserving DM, SDK, owner-only, and explicit allowlist fail-closed behavior.

## Reproduction Assessment

Yes, at source level: current main can carry an admitted Feishu slash command into `CommandAuthorized=false`, and the reply path drops unauthorized whole-message control commands. I did not run a live Feishu workspace in this read-only review.

## Solution Assessment

No. The intended Feishu fix direction is valid, but changing the global helper is not the narrowest maintainable solution because DM, generic ingress, and SDK command gates rely on the current fail-closed semantics.
