## Summary

Keep open: the PR targets a real, source-reproducible Feishu slash-command bug, but the current diff is not merge-ready because it broadens shared command authorization and still lacks real behavior proof.

## What This Changes

The PR changes the shared command-gating helper to treat all-unconfigured `useAccessGroups=true` authorizers as allowed and updates its unit tests.

## Best Possible Solution

Land a narrow Feishu command-authorization fix, or an explicit shared mode with root SDK parity and contract tests, after redacted runtime proof.

## Reproduction Assessment

Yes, at source level: current main can carry an admitted Feishu slash command with no configured command allowlist into `CommandAuthorized=false`, and the shared reply path then returns no response for unauthorized whole-message control commands. I did not run a live Feishu workspace in this read-only pass.

## Solution Assessment

No. The bug-fix direction is valid, but this implementation changes a shared helper too broadly and leaves the root SDK shim divergent; a Feishu-scoped fix or explicit shared mode with parity is safer.
