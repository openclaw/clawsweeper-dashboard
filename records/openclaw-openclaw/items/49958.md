## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main can carry a Feishu slash command with no configured command allowlist into `CommandAuthorized=false`, and the reply path then returns no response for unauthorized whole-message control commands. I did not run a live Feishu workspace in this read-only pass.

## Solution Assessment

No. The bug-fix direction is valid, but this implementation changes a shared helper too broadly and leaves the root SDK shim divergent; a Feishu-scoped fix or explicit shared mode with parity is safer.
