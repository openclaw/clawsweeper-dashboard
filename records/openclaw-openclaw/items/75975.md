## Summary

Keep open. Current main still splits Telegram private-chat messages with `message_thread_id` into thread-suffixed sessions, and Telegram has no DM-specific `threadReplies`/flattening policy while the docs and tests still assert thread-aware DM sessions.

## What This Changes

Request to keep Telegram one-on-one DMs on a flat session by default when inbound messages include `message_thread_id`, while preserving thread metadata for replies and forum groups.

## Best Possible Solution

Add a Telegram DM threading policy matching Matrix semantics, default DMs to flat sessions, and update Telegram config types/schema, docs, tests, and changelog while keeping forum topics isolated.

## Reproduction Assessment

Yes. A static reproduction exists in current main: a private Telegram message with `message_thread_id: 42` is expected by the test suite to route to `agent:main:main:thread:1234:42` instead of the flat DM session.

## Solution Assessment

No. The current thread-aware DM default is the reported failure mode; a DM-specific `off|inbound|always` policy with default `off` is narrower and safer than dropping thread metadata globally.
