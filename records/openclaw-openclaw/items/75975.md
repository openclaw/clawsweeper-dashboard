## Summary

Keep open. Current main still derives a separate Telegram DM session when a private-chat message carries `message_thread_id`, and existing Telegram docs/tests assert that behavior rather than the requested DM-flat default.

## What This Changes

Telegram DM messages that carry `message_thread_id` should stay in the main DM session by default instead of deriving a separate thread-scoped session key.

## Best Possible Solution

Add a Telegram DM threading policy matching Matrix semantics, default direct chats to flat sessions, and keep forum-topic group isolation plus outbound `message_thread_id` preservation intact.

## Reproduction Assessment

Yes. The current unit-test expectation and routing code provide a high-confidence static reproduction: a private Telegram message with `message_thread_id: 42` produces a thread-suffixed session key instead of the main DM session.

## Solution Assessment

No. The current default is not the best solution for reply-with-quote DMs; a DM-specific `off|inbound|always` policy with default `off` is narrower than dropping all Telegram thread metadata and safer than continuing to split every private-chat thread id.
