## Summary

Keep open. Current main still derives a thread-suffixed Telegram DM session when a private-chat message carries `message_thread_id`, and the current tests/docs assert that behavior while Telegram has no Matrix-style DM threading policy.

## What This Changes

The issue requests Telegram direct-message routing that does not split sessions by inbound `message_thread_id` by default while retaining thread metadata for replies and topic-aware use cases.

## Best Possible Solution

Add a Telegram DM threading policy matching Matrix `off|inbound|always`, default direct chats to flat sessions, and keep forum-topic group isolation plus outbound thread metadata intact.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: the current Telegram DM thread unit test builds a private message with `message_thread_id: 42` and expects the thread-suffixed session key that the issue reports as fragmented.

## Solution Assessment

No. The current default is not the best solution for reply-with-quote DMs; a DM-specific policy with default `off` is narrower and more compatible than globally ignoring Telegram thread metadata.
