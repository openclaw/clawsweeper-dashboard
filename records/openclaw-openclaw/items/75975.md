## Summary

Keep open. Current main still routes Telegram private-chat messages with `message_thread_id` into thread-suffixed session keys, and the Telegram docs/tests currently assert that behavior rather than the requested DM-flat default.

## What This Changes

Request that Telegram private chats ignore inbound `message_thread_id` for session-key derivation by default, with a Matrix-style DM threading policy or equivalent override.

## Best Possible Solution

Add a Telegram DM threading policy matching Matrix semantics, default direct chats to flat sessions, and keep forum-topic group isolation plus outbound `message_thread_id` preservation intact.

## Reproduction Assessment

Yes. The current unit-test expectation and routing code provide a high-confidence static reproduction: a private Telegram message with `message_thread_id: 42` produces a thread-suffixed session key instead of the main DM session.

## Solution Assessment

Yes. A Telegram `dm.threadReplies` policy defaulting to `off` is the narrowest maintainable fix because it matches Matrix, preserves an opt-in path for DM topic isolation, and avoids dropping outbound thread metadata.
