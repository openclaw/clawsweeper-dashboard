## Summary

Keep open: current main still has the Google Chat auth transport path without header normalization, and #76791 is an open, unmerged closing PR for the same regression.

## What This Changes

Restore Google Chat inbound webhook auth after a late-April regression causes google-auth certificate retrieval to fail on non-`Headers` request headers.

## Best Possible Solution

Review and land #76791, or an equivalent plugin-local fix that preserves the SSRF-guarded transport while normalizing google-auth request headers.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection is enough to mark it source-reproducible: current main routes app-url auth to `OAuth2Client.verifyIdToken` through `getGoogleAuthTransport`, and that transport lacks the header normalization proposed for the reported failure.

## Solution Assessment

Yes: a plugin-local normalization around `getGoogleAuthTransport` is the narrowest maintainable fix. Because #76791 already carries that approach and closes this issue, this issue should stay open until that PR lands or is replaced.
