## What This Changes

The PR adds a manual Mantis Discord Status Reactions workflow, docs, and a QA Lab Discord live scenario that captures REST reaction timelines plus HTML/PNG artifacts.

## Best Possible Solution

Keep the PR open and harden the Mantis oracle so pass requires a stable, ordered sequence of SUT-authored reactions before maintainer merge review.

Yes, source-reproducible for the review findings: the PR head's collector/pass code accepts unordered presence and any matching reactor, while current Discord handling restores the done reaction immediately by default. I did not run the live Discord scenario.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Hold the done reaction for the REST oracle:** `extensions/qa-lab/src/live-transports/discord/discord-live.runtime.ts:364-371`
  - body: This scenario expects the REST poller to observe `👀 -> 🤔 -> 👍`, but the config leaves `messages.removeAckAfterReply` at its default. Current Discord processing calls `setDone()` and then immediately restores the initial reaction, so `👍` can disappear before the 250 ms poll catches it. Hold or clear the terminal reaction on a known timer, or observe a stable lifecycle signal.
  - confidence: 0.84
- **[P2] Require ordered reactions before passing:** `extensions/qa-lab/src/live-transports/discord/discord-live.runtime.ts:1160-1166`
  - body: The scenario is documented as `queued -> thinking -> done`, but the result only checks that each expected emoji appeared at least once. A timeline like `👀 -> 👍 -> 🤔` leaves `missing` empty and is marked pass, producing false-positive Mantis evidence. Compare the observed sequence to the expected order before returning pass.
  - confidence: 0.88
- **[P2] Validate reactions came from the SUT:** `extensions/qa-lab/src/live-transports/discord/discord-live.runtime.ts:547-552`
  - body: The REST snapshot records only emoji/count/current-token state, and the poll uses the driver bot token. In a shared QA channel, another user or bot adding `🤔` or `👍` can satisfy the oracle even if the SUT never emitted status reactions. Fetch the reaction users or otherwise filter the evidence to the SUT bot.
  - confidence: 0.78
