## What This Changes

The issue asks OpenClaw to fix a regression where OpenAI Codex-backed coding/background agent runs stop producing final results or completion follow-up after work starts.

## Best Possible Solution

Add a focused heartbeat dispatch fix so targeted `exec-event` wakes with captured delivery context can run even when periodic heartbeat cadence is disabled by `every: "0m"`, while preserving the global heartbeat enable switch and normal disabled alert visibility. Cover the scheduler and relay behavior with regression tests, then only open a separate follow-up if Codex-specific planning/final-answer suppression still reproduces afterward.

## Repair Work Prompt

Fix the narrowed regression from https://github.com/openclaw/openclaw/issues/62505: background exec completions are lost when `agents.defaults.heartbeat.every = "0m"`. Current `notifySessionExit` / `emitExecSystemEvent` enqueue an exec completion and call `requestHeartbeatNow({ reason: "exec-event", sessionKey, ... })`, but `src/infra/heartbeat-runner.ts` only schedules agents with a non-null interval and `runHeartbeatOnce` also returns disabled when the interval is null. Implement a narrow one-shot path for targeted `exec-event` wakes so completion events with captured delivery context can dispatch to the originating agent/session even when periodic heartbeat cadence is disabled. Preserve the global `areHeartbeatsEnabled()` kill switch, normal non-exec disabled behavior, active routing/delivery-context semantics, and existing interval scheduling. Do not reintroduce codex-cli, do not change provider/model selection, and do not broaden Codex runtime behavior beyond what is required for exec completion delivery. Add focused tests equivalent to the closed unmerged PR #67913: scheduler dispatch with disabled heartbeat interval, completion relay with captured delivery context when disabled, and no fan-out to unrelated agents. Because this is user-facing, add a single-line `CHANGELOG.md` fix entry crediting the reporters/contributors such as `Thanks @drpau, @gwwallace, @ToniDonDoni, @lml2468` if the changelog policy applies.
