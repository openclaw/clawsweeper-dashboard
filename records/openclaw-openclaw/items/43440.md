## What This Changes

The issue requests changing iMessage/imsg execution so imsg can read Messages chat.db under its own Full Disk Access grant without also granting FDA to the parent Node/OpenClaw process.

## Best Possible Solution

Track the macOS TCC process-context redesign or supported signed-app/LaunchAgent guidance in https://github.com/openclaw/openclaw/issues/51299, preserving explicit user consent for Full Disk Access and Automation.

## Reproduction Assessment
