## Summary

Keep this PR open. Current main still lacks the proposed npm_execpath test isolation and config-validation doctor-registry mocks, but the PR should be revised before merge because the new migration mocks do not match the real doctor-registry return contract and the diff adds an unexplained root production dependency in a test-only change.

## What This Changes

The PR updates config validation tests to mock doctor-contract registry discovery, changes one npm release-check test to save/delete/restore process.env.npm_execpath, and adds markdown-it-task-lists to the root package manifest and lockfile.

## Best Possible Solution

Keep the useful test stabilization, but make it strictly test-scoped: restore the npm_execpath isolation, make the doctor-registry mocks either avoid applyPluginDoctorCompatibilityMigrations or return { config, changes }, remove or separately justify the root dependency change, then rerun the targeted config validation and npm release-check tests.

## Reproduction Assessment

Yes. The npm test flake has a code-level reproduction by setting process.env.npm_execpath to an npm CLI path before the current win32 fallback test; the PR review findings are reproducible by comparing the PR mock return values against the current doctor-registry contract and root import graph.

## Solution Assessment

No, not as submitted. The intended stabilization direction is narrow and reasonable, but the current implementation should first align mocks with the real contract and remove the unrelated production dependency change.
