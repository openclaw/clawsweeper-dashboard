## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Budget active-memory QMD deadlines from the effective mode:** `extensions/memory-core/src/tools.ts:549-552`
  - body: `activeMemoryDeadline` is computed before `qmdSearchModeOverride` is resolved, but that override is what the manager actually receives. When active-memory overrides the QMD mode, the new timeout can be based on the wrong command lifecycle, so please compute the budget from the effective mode used for the search.
  - confidence: 0.9
- **[P1] Use one total timeout for QMD close cleanup:** `extensions/memory-core/src/memory/qmd-manager.ts:1705-1706`
  - body: `close(timeoutMs)` races `pendingUpdate` and `queuedForcedUpdate` independently with the full timeout, so cleanup can take about twice the requested cap when both promises exist. The one-shot cleanup path needs one shared deadline across all awaited QMD cleanup work.
  - confidence: 0.88
- **[P3] Keep resolver tests from mutating global Git config:** `test/scripts/resolve-openclaw-ref.test.ts:12-14`
  - body: This test adds `/tmp` to the user's global Git `safe.directory` list, permanently changing the runner or developer account outside the test process. Use a test-local `GIT_CONFIG_GLOBAL` or equivalent isolated config instead.
  - confidence: 0.82
