Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fallback searches on the default deadline:** `extensions/memory-core/src/memory/search-manager.ts:514-516`
  - body: `getSearchTimeoutMs()` returns the primary QMD whole-search budget while `primaryFailed` is still false, but `search()` can then catch a QMD failure and run the builtin fallback during that same call. That makes a hung builtin fallback wait for the QMD budget instead of the existing 15s guard, so resolve the deadline after fallback selection or keep fallback work on the default timeout.
  - confidence: 0.88
- **[P2] Check aborted signals before spawning QMD children:** `packages/memory-host-sdk/src/host/qmd-process.ts:207-209`
  - body: The new signal handling checks `params.signal?.aborted` only after `spawn(...)` has already launched the child. An already-cancelled caller can still start a qmd or mcporter process and then kill it, so the exported helper should reject before spawning when the signal is already aborted.
  - confidence: 0.86
