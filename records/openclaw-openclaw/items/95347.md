Overall correctness: patch is incorrect

Full review comments:

- **[P2] Budget QMD deadlines from the effective mode:** `extensions/memory-core/src/tools.ts:549-552`
  - body: activeMemoryDeadline is created before qmdSearchModeOverride is resolved, and QmdMemoryManager.getSearchTimeoutMs() budgets from the manager's configured searchMode even though the actual search may run the active-memory override. That can give active-memory QMD searches a deadline for a different command lifecycle than the one executed.
  - confidence: 0.88
- **[P2] Use one shared close timeout:** `extensions/memory-core/src/memory/qmd-manager.ts:1705-1706`
  - body: close(timeoutMs) races pendingUpdate and queuedForcedUpdate separately with the full timeout, so one-shot cleanup can take roughly twice the requested cap when both promises are present. The close path should spend one shared remaining budget across all cleanup awaits.
  - confidence: 0.86
