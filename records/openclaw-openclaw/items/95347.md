The branch derives QMD-backed memory_search deadlines from manager-owned budgets, adds optional memory-manager timeout/close hooks, bounds builtin fallback and one-shot CLI cleanup waits, and adds memory-core regression tests.

## Best Possible Solution

Land the QMD timeout fix after tightening the close-timeout helper, with maintainer acceptance for manager-owned search budgets and the additive SDK hooks while leaving QMD subprocess cancellation to its separate PR.

## Solution Assessment

Mostly yes. Manager-owned whole-search budgeting is the right boundary because QMD owns collection/search/update/embed timing, but close-timeout accounting still needs a narrow fix and maintainers must accept the additive SDK shape.
