## Summary

Keep this PR open: it is still a useful candidate for the QMD memory_search timeout bug, but the current patch still grants QMD a fresh long manager timeout for the initial search, forced sync, and retry instead of spending one whole-search deadline.

## What This Changes

The PR adds a manager-provided memory_search timeout hook, computes a QMD whole-search budget, wraps memory search phases with deadline helpers, bounds builtin fallback search, and adds memory-core tests.

## Best Possible Solution

Land a QMD timeout fix that spends one manager-provided whole-search deadline across initial search, optional forced sync, fallback, and retry while preserving the default 15s cap for setup, supplements, builtin fallback, and non-QMD paths.

## Reproduction Assessment

## Solution Assessment

No, not yet: the manager-owned QMD timeout is the right layer, but the current patch resets a long timeout across QMD search, sync, and retry phases instead of enforcing one whole-search deadline.
