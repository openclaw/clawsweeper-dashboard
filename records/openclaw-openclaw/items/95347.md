## What This Changes

This PR derives QMD-backed `memory_search` deadlines from manager-owned budgets, bounds QMD fallback and one-shot cleanup waits, and adds memory-core regression coverage.

## Best Possible Solution

Land this focused QMD timeout fix after maintainers accept manager-owned search budgets and the additive SDK hook, with the close-timeout accounting tightened and QMD subprocess cancellation left to https://github.com/openclaw/openclaw/pull/93394.

## Solution Assessment

Mostly yes. Manager-owned whole-search budgeting is the right boundary because QMD owns collection/search/update/embed timing, but maintainers still need to accept the additive SDK hook and the close helper should use one cleanup deadline.
