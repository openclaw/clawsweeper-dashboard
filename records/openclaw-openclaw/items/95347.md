## Summary

Keep this PR open, but it is not merge-ready: the current diff calls `getSearchTimeoutMs()` as an unbound class method, which breaks the QMD manager timeout path, and the external PR still only has test/mock proof rather than latest-head real QMD behavior proof.

## What This Changes

The PR changes memory-core so QMD-backed `memory_search` can use manager-reported whole-search deadlines, bounds search/sync/cleanup waits, and extends `MemorySearchManager` with optional timeout and close parameters.

## Best Possible Solution

Fix the unbound method call, get maintainer approval for the manager-owned timeout/cleanup contract, and require latest-head real QMD terminal or log proof before this branch can land.

## Reproduction Assessment

Yes. Source inspection shows current main and the latest release still use the fixed 15s `memory_search` wrapper while QMD config/docs expose longer `memory.qmd.limits.timeoutMs`, and CI/source inspection reproduce a PR-introduced unbound-method failure.

## Solution Assessment

No, not as written. Reusing the existing QMD timeout is plausible, but the current implementation must fix the unbound receiver bug and still needs maintainer agreement on the API and availability tradeoff.
