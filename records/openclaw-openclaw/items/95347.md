- The optional MemorySearchManager.getSearchTimeoutMs hook and close(timeoutMs?) signature expand the memory-host SDK/interface surface and need maintainer acceptance as a permanent contract shape.
- QMD subprocess cancellation remains separate in https://github.com/openclaw/openclaw/pull/93394, so this PR should not be treated as closing orphaned-process work.
- QmdMemoryManager.close currently uses separate un-cleared timeout timers for pending update and queued forced update, which can emit stale warnings and spend the cleanup grace twice.
