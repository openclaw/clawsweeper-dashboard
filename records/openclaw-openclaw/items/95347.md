Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Budget QMD deadlines from the effective mode:** `extensions/memory-core/src/tools.ts:549-552`
  - body: `activeMemoryDeadline` is created before `qmdSearchModeOverride` is resolved, and `QmdMemoryManager.getSearchTimeoutMs()` budgets from `this.qmd.searchMode` even though the search may run the override. Active-memory QMD searches can therefore get a deadline for a different command lifecycle than the one actually executed.
  - confidence: 0.88
- **[P2] Use one total QMD close timeout:** `extensions/memory-core/src/memory/qmd-manager.ts:1705-1706`
  - body: `close(timeoutMs)` races `pendingUpdate` and `queuedForcedUpdate` separately with the full timeout, so one-shot cleanup can take about twice the requested cap when both are present. The close path needs a single shared deadline across all QMD cleanup awaits.
  - confidence: 0.86
