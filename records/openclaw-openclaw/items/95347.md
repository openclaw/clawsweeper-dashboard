Full review comments:

- **[P2] Keep runtime loading inside the deadline:** `extensions/memory-core/src/tools.ts:415-416`
  - body: `loadMemoryToolRuntime()` used to run inside `runMemorySearchToolWithDeadline`, so the timeout guard was armed before memory runtime setup. This change awaits the runtime before creating the deadline, which can leave `memory_search` hanging without the existing 15s fail-open behavior if runtime loading stalls.
  - confidence: 0.82
- **[P1] Do not use one QMD command timeout as the tool deadline:** `extensions/memory-core/src/tools.ts:421`
  - body: `memory.qmd.limits.timeoutMs` is applied to individual QMD command calls, while the QMD manager can do setup, pending-update waits, multiple collection-group searches, fallback, and repair. Using that one value as the whole `memory_search` deadline can still abort valid configured searches and repeats the design already rejected on the overlapping timeout PR.
  - confidence: 0.9
