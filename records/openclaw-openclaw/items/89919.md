## Summary

Keep open: the patch is close, but it is not merge-ready because it can suppress configured same-target failure destinations and still lacks after-fix real Matrix/cron runtime proof.

## What This Changes

The PR adds a cron timeout classifier and uses it in cron state resolution and gateway notification dispatch to suppress primary fallback failure announcements for the exact `tool-execution-started` watchdog timeout while adding targeted cron/gateway tests.

## Best Possible Solution

Keep the exact timeout classifier, but suppress primary fallback delivery only when no job/global failure destination was configured at all, including same-target deduped destinations, and add redacted real Matrix/cron proof before merge.

## Reproduction Assessment

Yes at source level: current main can format the exact phase-specific timeout and routes cron errors through configured failure destinations or primary announce fallback. I did not reproduce the private Matrix incident in a live environment.

## Solution Assessment

No, not yet: the patch is a plausible mitigation, but it conflates no explicit failure destination with an explicit destination deduped to the primary target. The safer fix preserves configured failure-destination intent while suppressing only the unconfigured primary fallback.
