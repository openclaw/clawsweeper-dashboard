## Summary

Keep this PR open. ## What This Changes

The PR adds a cron timeout classifier and uses it to suppress the primary fallback cron failure announce for the exact `tool-execution-started` watchdog timeout while recording the run error and testing configured failure-destination paths.

## Best Possible Solution

Keep the exact timeout classifier, but suppress primary fallback delivery only when no job/global failure destination was configured at all, including same-target deduped destinations, and add redacted real Matrix/cron proof before merge.

## Reproduction Assessment

Yes at source level: current main can format the exact phase-specific timeout and routes cron errors through configured failure destinations or primary announce fallback. I did not reproduce the private Matrix incident in a live environment.

## Solution Assessment

No, not yet: the current patch is close, but it still conflates no explicit failure destination with an explicit destination deduped to the primary target. The maintainable fix is to preserve explicit/global failure-destination intent while suppressing only the unconfigured primary fallback.
