The PR adds a cron timeout classifier used by cron state resolution and gateway notification dispatch to suppress primary announce fallback for the exact tool-execution-started watchdog timeout, with gateway and cron regression tests.

## Best Possible Solution

Land a narrow suppression for the exact timeout that preserves all configured failure destinations, including same-target deduped routes, with tests for fallback-only, distinct, same-target, and global destinations.

## Solution Assessment

No as written. The suppression is in the right general area, but it needs to preserve configured same-target failure destinations or make that product decision explicit with tests and docs.
