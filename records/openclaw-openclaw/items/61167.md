## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not force Docker bundled plugin discovery:** `docker-compose.yml:10`
  - body: A later merged Docker fix removed the packaged-runtime OPENCLAW_BUNDLED_PLUGINS_DIR override so Docker images use the compiled bundled artifacts through the normal resolver. Setting the Compose default back to /app/extensions can reintroduce source-plugin loading in packaged Docker runs.
  - confidence: 0.93
- **[P1] Keep service UID aligned with image-owned runtime paths:** `docker-compose.yml:4`
  - body: Running both services as the host UID/GID means common hosts with UID other than 1000 run as a user that does not own /home/node or the preinstalled Playwright cache. The new WhatsApp setup then installs/uses browsers under /home/node/.cache/ms-playwright and can fail with EACCES.
  - confidence: 0.86
- **[P1] Gate destructive teardown behind an explicit data-removal opt-in:** `docker-teardown-web-whatsapp.sh:69`
  - body: The teardown helper removes the entire HOST_ROOT with sudo whenever it exists, while only HOME_BIND_DIR removal is opt-in. With the default root this can delete the user's OpenClaw workspace and persisted state instead of only helper-created container data.
  - confidence: 0.95
