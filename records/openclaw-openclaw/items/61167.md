## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Validate UID/GID before root chown:** `scripts/docker/setup.sh:508-509`
  - body: `OPENCLAW_UID` and `OPENCLAW_GID` are environment-controlled and persisted, then expanded into a root `sh -c` command. Values containing shell metacharacters can alter the command run in the setup container while host-mounted OpenClaw state is attached; validate numeric IDs or avoid string-building the root shell command.
  - confidence: 0.95
- **[P1] Do not delete HOST_ROOT by default:** `docker-teardown-web-whatsapp.sh:69`
  - body: The teardown helper always runs `sudo rm -rf "$HOST_ROOT"` when the path exists. Since setup puts the workspace under that root, running teardown can delete user workspace/project data instead of only generated container state; full-root removal needs explicit opt-in or confirmation and should preserve workspace by default.
  - confidence: 0.92
- **[P2] Preserve valid host paths with spaces:** `docker-setup-web-whatsapp.sh:61-63`
  - body: The new helper rejects whitespace in `OPENCLAW_HOST_ROOT` and derived paths, but current Docker setup supports quoted host persistence paths with spaces. This makes common user-profile paths fail before setup instead of using the existing mount quoting behavior.
  - confidence: 0.86
