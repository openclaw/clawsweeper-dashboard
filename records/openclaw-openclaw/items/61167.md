## Summary

Keep open. Current main still does not include the PR's Docker env passthrough or WhatsApp setup helpers, but the latest PR head is not merge-ready because it interpolates caller-controlled UID/GID values into a root `sh -c` permission-repair command.

## What This Changes

This PR adds Docker Compose passthrough for bundled plugin and Playwright cache paths, adds WhatsApp Docker setup/teardown helpers, and changes Docker setup permissions to use configurable host UID/GID values.

## Best Possible Solution

Keep the useful Docker env/helper direction, but rework UID/GID handling so IDs are validated as numeric or passed as quoted shell arguments, then add focused Docker setup coverage for env passthrough, Playwright cache behavior, ownership repair, and teardown safety.

## Reproduction Assessment

Yes. For the blocking review finding, set `OPENCLAW_UID` or `OPENCLAW_GID` to a value containing shell metacharacters and inspect the root `sh -c` command constructed by the PR; I did not run destructive Docker setup commands during this read-only review.

## Solution Assessment

No. The env passthrough and helper scripts are plausible, but the current PR is not the safest implementation until the root permission repair stops constructing shell code from raw environment values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Validate UID/GID before using them in root chown:** `scripts/docker/setup.sh:508-509`
  - body: `OPENCLAW_UID` and `OPENCLAW_GID` are caller-controlled environment values, but this change interpolates them directly into the string passed to `sh -c` in a root prestart container. A value containing shell syntax can alter the command while host-backed OpenClaw directories are mounted, so validate numeric IDs or pass them as quoted shell arguments before using them in `chown`.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete shell-command injection risk in Docker setup permission repair.

Concerns:

- **[high] Unvalidated UID/GID enters root shell command:** `scripts/docker/setup.sh:508`
  - body: The proposed setup change expands `OPENCLAW_UID` and `OPENCLAW_GID` into a command string executed by `sh -c` as root inside the prestart container. Because those values can come from the caller environment, shell metacharacters can change the command while OpenClaw bind mounts are attached.
