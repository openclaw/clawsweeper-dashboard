## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.98

Full review comments:

- **[P1] Use interpolated Compose scalars instead of single quotes:** `docker-compose.yml:4`
  - body: Docker Compose treats single-quoted values literally, so the new `user` value will not resolve to numeric IDs and can prevent both services from starting. The same reformatting makes the port and `--bind` expressions literal; restore unquoted or double-quoted values and verify the rendered result with `docker compose config`.
  - confidence: 0.99
- **[P1] Do not delete the entire configurable host root:** `docker-teardown-web-whatsapp.sh:69`
  - body: The teardown helper runs `sudo rm -rf` on `OPENCLAW_HOST_ROOT`, whose default also contains the persistent workspace and whose override may point at a reused directory. Quoting and whitespace rejection do not prevent deletion of a valid but overly broad path; remove whole-root deletion or require a sentinel and strict containment around helper-created directories.
  - confidence: 0.97
- **[P1] Preserve the documented container ownership contract:** `docker-compose.yml:4`
  - body: Running the gateway and CLI as arbitrary host UID/GID conflicts with the current Docker contract, which keeps uid 1000 and fixes bind-mount ownership because plugin loading validates ownership. Existing mounts and image-owned browser/plugin files can become inaccessible or blocked; keep the image user and repair only the documented bind mounts unless an upgrade-safe identity change is explicitly approved and proven.
  - confidence: 0.93
