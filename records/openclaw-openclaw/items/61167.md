## Summary

Keep open. Current main still lacks the Docker WhatsApp helper/env wiring, and the PR branch is not merge-ready because it expands unvalidated UID/GID values into a root `sh -c` permission repair while overlapping recent Docker setup changes.

## What This Changes

The PR adds Docker Compose env passthrough for bundled plugin and Playwright cache paths, adds WhatsApp Docker setup/teardown helpers, and changes Docker setup/services to run and chown with host UID/GID.

## Best Possible Solution

Keep the PR open, update it against current main, validate UID/GID as numeric or pass them as shell arguments before any root chown repair, and add focused Docker setup coverage for env passthrough, Playwright cache persistence, UID/GID ownership, and teardown safety.

## Reproduction Assessment

Yes. The blocking security finding is reproducible by inspecting PR head `scripts/docker/setup.sh:508-509` or by setting `OPENCLAW_UID`/`OPENCLAW_GID` to shell metacharacters before setup and observing the generated root `sh -c` command. The original env-passthrough gap is also reproducible by reading current main's compose/setup files, where those variables are absent.

## Solution Assessment

No. The env plumbing is directionally useful, but the current PR is not the best mergeable solution until raw UID/GID values are validated or passed without shell parsing and the branch is reconciled with current Docker setup behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate UID/GID before chown repair:** `scripts/docker/setup.sh:508-509`
  - body: `OPENCLAW_UID` and `OPENCLAW_GID` are caller-controlled environment values, but this expands them into the string executed by `sh -c` in a root prestart container. A value containing shell syntax can alter the command while host-backed OpenClaw directories are mounted. Validate numeric IDs, or pass them as shell arguments and quote them inside the container command before using them in `chown`.
  - confidence: 0.88
