## Summary

This PR should stay open because its central Docker env/helper work is not already on current main, but the latest branch is not mergeable as-is: it still expands caller-controlled UID/GID values into a root `sh -c` command. The prior ClawSweeper review identified the same blocker, and no later branch commit is present.

## What This Changes

The PR adds Docker Compose/env passthrough for bundled plugin and Playwright cache paths, adds WhatsApp Docker setup/teardown helpers, and changes Docker services plus setup permission repair to use configurable host UID/GID values.

## Best Possible Solution

Keep the useful Docker env/cache/helper direction, but make UID/GID handling numeric-safe or pass values as shell arguments, preserve current Docker defaults, and add focused Docker setup coverage for env passthrough, Playwright cache, ownership repair, and teardown safety.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Validate UID and GID before root chown:** `scripts/docker/setup.sh:508-509`
