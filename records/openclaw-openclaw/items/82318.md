## Summary

Keep open: current main and v2026.6.10 still let local CLI session-store resolution follow an isolated process HOME when canonical OpenClaw state env is absent, and Doctor still has only the generic multiple-state-dir warning rather than a targeted Codex shadow session-store guard or repair path.

## What This Changes

The issue asks OpenClaw session recovery and Doctor paths to avoid mutating a Codex/ACP isolated HOME shadow state tree instead of the Gateway's canonical per-agent session store.

## Best Possible Solution

Online maintenance should target the running Gateway's canonical session state, while offline writes should require explicit canonical state or refuse/warn with the exact target path; Doctor should detect Codex shadow session stores and report a safe repair path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run OpenClaw session maintenance with HOME under an agent codex-home/home tree and without OPENCLAW_STATE_DIR or OPENCLAW_CONFIG_PATH, and default store resolution points under that shadow HOME.

## Solution Assessment

Unclear until maintainers choose the recovery UX contract. The narrow maintainable shape is a shared state-target guard plus Gateway-canonical routing for online recovery, not a one-off sessions command patch.
