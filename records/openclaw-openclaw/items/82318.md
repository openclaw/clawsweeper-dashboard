## Summary

Keep open: current main still lets CLI session-maintenance paths derive the default OpenClaw state root from the process HOME when canonical OpenClaw env is absent, and doctor has only a generic multiple-state-dir warning rather than a Codex shadow session-store detector. The related reset-transcript and ACPX orphan fixes are merged, but they do not cover this issue's remaining split-brain recovery path.

## What This Changes

The issue asks OpenClaw to prevent session recovery commands from silently reading or mutating a shadow session store when run from a Codex/ACP environment with an isolated HOME.

## Best Possible Solution

Add a shared recovery-state guard and doctor detector: online maintenance should use the running Gateway's canonical state, while offline writes should require explicit canonical env or an explicit store and print the exact target path before mutation.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: run OpenClaw session commands with HOME under an agent codex-home/home tree and without OPENCLAW_STATE_DIR or OPENCLAW_CONFIG_PATH, and default session stores resolve under that shadow HOME. I did not run a live mutating command because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the UX contract. The narrowest maintainable shape appears to be a shared guard/detector plus Gateway-canonical routing for online recovery, not a one-off sessions command patch.
