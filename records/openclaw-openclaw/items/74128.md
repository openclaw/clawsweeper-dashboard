## Summary

Keep open. Current main still starts the optional model-pricing refresh from the early Gateway runtime-services path before the post-attach sidecar/channel ready boundary, so the reported default startup ordering is not proven fixed; the config opt-out is only a mitigation and #73486 remains the focused implementation candidate.

## What This Changes

The issue asks for Gateway startup to avoid blocking channel/sidecar readiness on OpenRouter and LiteLLM pricing fetch timeouts.

## Best Possible Solution

Land or adapt #73486 so optional pricing refresh cannot run before sidecar/channel readiness, keep `models.pricing.enabled: false` as the mitigation, and leave broader Gateway network degradation to #73323.

## Reproduction Assessment

Yes. The issue provides concrete restart steps and timing logs, and a focused repro can force the pricing endpoints to hang while measuring whether `gateway ready` is delayed on current main.

## Solution Assessment

Yes, as a direction. Deferring optional pricing refresh until after the ready boundary is the narrowest maintainable fix for this startup bug; the opt-out helps affected users but does not make the default path robust.
