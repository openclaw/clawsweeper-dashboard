## Summary

Keep open. Current main still starts the optional model-pricing refresh from the early Gateway runtime-services path while pricing remains enabled by default, so the reported default startup ordering is not proven fixed; the existing opt-out is only a mitigation and open PR #73486 remains the focused implementation candidate.

## What This Changes

The issue reports a macOS Gateway boot delay where OpenRouter/LiteLLM pricing fetch timeouts still appear to block readiness after the earlier manifest-reuse fix.

## Best Possible Solution

Land or adapt #73486 so optional pricing refresh cannot run before sidecar/channel readiness, keep `models.pricing.enabled: false` as the mitigation, and leave broader Gateway network degradation to #73323.

## Reproduction Assessment

Yes. The issue provides concrete restart steps and timing logs, and current main still has the same default early pricing-refresh path with 60s remote fetch timeouts; a focused repro can force those endpoints to hang and verify whether readiness is delayed.

## Solution Assessment

Yes, as a direction. Deferring optional pricing refresh until after the ready boundary is the narrowest maintainable fix for this startup bug; the opt-out helps affected users but does not make the default boot path robust.
