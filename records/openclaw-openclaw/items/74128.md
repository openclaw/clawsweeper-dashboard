## Summary

Keep open. Current main still starts the optional model-pricing refresh from the early Gateway runtime-services path with pricing enabled by default, before the post-attach sidecar/channel ready boundary. The documented `models.pricing.enabled: false` setting is a mitigation, and open PR #73486 is the focused implementation candidate, but the default startup behavior is not implemented on main yet.

## What This Changes

This issue asks OpenClaw to stop default macOS Gateway startup from waiting about 75 seconds for optional OpenRouter/LiteLLM model-pricing fetch timeouts after the earlier manifest-reuse fix.

## Best Possible Solution

Land or adapt #73486 so the default model-pricing refresh cannot run until after Gateway sidecars/channels are ready, keep `models.pricing.enabled: false` as the restricted-network mitigation, and leave the broader runtime degradation investigation with #73323.

## Reproduction Assessment

Yes. The issue includes concrete macOS restart steps and timestamped logs, and current main still has default early pricing refresh plus 60-second remote-fetch timeouts; a focused repro can force the pricing endpoints to hang and verify whether sidecar/channel readiness is delayed.

## Solution Assessment

Yes. Deferring optional pricing refresh until after the ready boundary is the narrowest maintainable fix for this startup bug; the config opt-out helps affected users but does not make the default boot path robust.
