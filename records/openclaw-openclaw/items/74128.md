## Summary

Keep open. Current main still starts the optional model-pricing refresh from the early Gateway runtime-services path while pricing is enabled by default, and channel/sidecar readiness is still reached later in the post-attach startup path. The documented pricing opt-out is a mitigation, while open PR #73486 remains the focused implementation candidate for the startup deferral.

## What This Changes

The issue asks for Gateway startup to stop delaying channel and sidecar readiness when default OpenRouter/LiteLLM model-pricing refreshes hang or hit their 60s timeout after the earlier manifest-reuse fix.

## Best Possible Solution

Land or adapt #73486 so the default pricing refresh starts only after Gateway sidecars and channels are ready, keep `models.pricing.enabled: false` as the documented restricted-network mitigation, and leave the broader process/network degradation investigation in #73323.

## Reproduction Assessment

Yes. The issue includes concrete macOS restart steps and timing logs, and current main preserves the same default early pricing-refresh ordering with 60s remote-fetch timeouts; a focused repro can force the pricing endpoints to hang under default config and verify whether sidecar/channel readiness is delayed.

## Solution Assessment

Yes, as a direction. Deferring optional pricing refresh until after the ready boundary is the narrowest maintainable fix for this startup bug; the existing config opt-out helps affected users but does not make the default boot path robust.
