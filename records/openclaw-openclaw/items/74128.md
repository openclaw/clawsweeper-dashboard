## Summary

Keep open. Current main has added a documented pricing opt-out, but the default Gateway startup path still starts the optional model-pricing refresh from early runtime services before the post-attach sidecar/channel ready boundary, so the reported default boot-delay risk is not resolved.

## What This Changes

The issue reports that Gateway startup still waits roughly 75 seconds on optional OpenRouter/LiteLLM model-pricing fetch timeouts after the earlier manifest-reuse fix.

## Best Possible Solution

Land or adapt #73486 so the default model-pricing refresh cannot start until after Gateway sidecars/channels reach the ready path; keep models.pricing.enabled: false as a documented mitigation and leave the deeper runtime/network investigation to #73323.

## Reproduction Assessment

Yes. The issue gives a concrete restart timeline, and a focused reproduction is to make the OpenRouter/LiteLLM pricing fetches hang under default config and verify whether channel/sidecar readiness waits; current source and tests still show the refresh starts before the ready boundary with 60s timeouts.

## Solution Assessment

Yes. Deferring the optional pricing refresh until after sidecars/channels are ready is the narrowest maintainable fix for this report; the opt-out is useful but does not solve the default startup path.
