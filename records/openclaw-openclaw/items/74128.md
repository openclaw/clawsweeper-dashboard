## Summary

Keep open. Current main still starts the default-enabled model-pricing refresh from the early Gateway runtime-services path, while the ready boundary still waits for post-attach sidecars/channels; the shipped pricing opt-out is useful mitigation but does not make the default boot path robust. Open PR #73486 remains the focused implementation candidate for deferring the optional refresh.

## What This Changes

The issue reports that Gateway startup remains blocked by optional OpenRouter/LiteLLM model-pricing fetch timeouts on macOS despite the earlier manifest-reuse fix.

## Best Possible Solution

Land or adapt #73486 so the default pricing refresh starts only after Gateway sidecars and channels are ready, keep models.pricing.enabled: false as the documented restricted-network mitigation, and leave the broader runtime/network degradation investigation in #73323.

## Reproduction Assessment

Yes. The issue includes concrete macOS restart steps and timing logs, and current main preserves the same default early pricing-refresh ordering with 60s remote-fetch timeouts; a focused repro can force the pricing endpoints to hang and verify whether gateway ready is delayed.

## Solution Assessment

Yes as a direction. Deferring optional pricing refresh until after the ready boundary is the narrowest maintainable fix for this startup bug; the config opt-out helps affected users but does not make the default boot path robust.
