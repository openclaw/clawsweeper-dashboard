## Summary

Keep this PR open: the Mattermost gap is source-backed and the patch is in the right plugin-owned area, but it currently has blocking type/metadata issues and the supplied real-behavior proof still contains a placeholder instead of actual output.

## What This Changes

The PR adds Mattermost per-group `systemPrompt` config support, resolves exact and wildcard prompts into inbound contexts, adds Mattermost tests, and documents the new config field.

## Best Possible Solution

Mattermost should accept and forward trusted per-group prompts with typed config, updated generated metadata, matching exact-over-wildcard semantics, a changelog entry, and redacted real after-fix proof.

## Reproduction Assessment

Yes from source inspection: current main has a strict Mattermost group schema that rejects `systemPrompt`, and the Mattermost inbound context paths do not set `GroupSystemPrompt`. I did not run a live Mattermost instance in this read-only review.

## Solution Assessment

No. The owner-local Mattermost fix is the right direction, but the current patch needs type, generated metadata, changelog, wildcard semantics, and real-proof follow-up before merge.
