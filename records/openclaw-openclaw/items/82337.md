## Summary

Keep open: current main still builds the inbound metadata envelope for the current turn, submits it to the model, and the replay sanitizer does not strip those envelope blocks from historical user messages.

## What This Changes

Strip OpenClaw inbound metadata envelopes from historical user-role messages during model prompt replay while keeping current-turn metadata available.

## Best Possible Solution

Add replay-history sanitization that strips OpenClaw inbound metadata from historical user messages only, preserving current-turn context and assistant/tool turns, with focused regression coverage.

## Reproduction Assessment

Yes from source inspection: current-turn metadata is prefixed into `promptForModel`, and the historical user replay sanitizer does not call `stripInboundMetadata`. I did not run a live Telegram replay in this read-only pass.

## Solution Assessment

Yes: the narrowest maintainable fix is to strip only historical user-role inbound metadata at the replay-hygiene boundary, not to change Telegram chat-window creation or remove current-turn context.
