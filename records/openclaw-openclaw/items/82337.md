## Summary

Keep open: current main still preserves inbound metadata in historical user-role replay messages, while the current-turn envelope is still built and submitted through the embedded runner path. The reporter also referenced an exact fix commit, but it is not on main and has no associated open PR, so this remains actionable in the repository.

## What This Changes

The issue asks to strip OpenClaw inbound metadata envelopes from historical user-role messages during model prompt replay while preserving the current turn envelope.

## Best Possible Solution

Add replay-history sanitization at the embedded-runner boundary that applies `stripInboundMetadata` to historical user-role text while preserving current-turn context and existing provider replay repairs.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the current-turn envelope is prepended into `promptForModel`, and the historical user replay sanitizer does not strip that envelope from persisted user messages. I did not run a live Telegram replay in this read-only pass.

## Solution Assessment

Yes: the narrowest maintainable fix is to strip inbound metadata only from historical user-role replay text at `sanitizeSessionHistory`, leaving Telegram context construction and current-turn context intact.
