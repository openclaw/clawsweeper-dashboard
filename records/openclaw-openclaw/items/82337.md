## Summary

Keep open: current main still replays historical user messages without stripping inbound metadata, and the reporter’s follow-up confirms the missing sanitizer coverage for the new Delivery and chat_window envelope blocks. No open PR appears to own this exact replay fix yet.

## What This Changes

The issue asks OpenClaw to strip inbound metadata envelopes from historical user-role messages during model prompt replay while preserving the current turn’s metadata.

## Best Possible Solution

Add replay-history sanitization for historical user-role text only, extend stripInboundMetadata for the newer Delivery and chronological chat_window blocks, and cover both string and text-block replay cases with focused tests.

## Reproduction Assessment

Yes from source inspection: current main builds current-turn inbound metadata, stores/replays historical user messages, and sanitizeUserReplayContent does not strip metadata from those past user turns. I did not run a live Telegram replay in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to strip OpenClaw-generated inbound metadata at the historical user replay boundary while preserving current-turn context, assistant turns, tool results, and Telegram context generation.
