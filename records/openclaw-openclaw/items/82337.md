## Summary

Keep open: current main still prefixes inbound metadata into model-bound user prompts and the replay sanitizer does not strip those blocks from historical user messages. The referenced patch is useful context, but it is not merged and a complete fix also needs strip-helper coverage for newer Delivery/chat_window envelope blocks.

## What This Changes

Strip OpenClaw-generated inbound metadata from historical user-role replay while preserving the current turn's metadata envelope.

## Best Possible Solution

Implement historical user-turn replay sanitization at the replay-history boundary, extending inbound metadata stripping to cover the current generated envelope while leaving currentTurnContext, assistant/tool turns, and persisted transcript records intact.

## Reproduction Assessment

Yes from source inspection: `buildCurrentTurnPrompt` prefixes current-turn context and `promptActiveSession` submits it, while `sanitizeUserReplayContent` returns nonblank historical user content unchanged. I did not run a live Telegram replay in this read-only review.

## Solution Assessment

Yes, with one caveat: the narrow fix belongs in replay-history sanitization, but it should not rely on the current strip helper without adding coverage for Delivery and chat_window/generated structured prefix blocks.
