## Summary

Keep open: the PR targets a real subagent cleanup bug, but it is not mergeable as-is because the active cleanup path on current main is split across run-manager, lifecycle, and helper modules while the branch patches the older monolithic registry path. It also still needs contributor-supplied real behavior proof before merge.

## What This Changes

The PR refactors subagent cleanup bookkeeping in `src/agents/subagent-registry.ts` and adds a steer-restart test for delete-mode termination idempotency.

## Best Possible Solution

Port per-run cleanup finalization and orphan-prune deferred retry wakeup into the current split registry modules, with focused regression coverage for shared child-session delete cleanup and orphan-prune retry behavior.

## Reproduction Assessment

Yes, at source level: current main groups terminated records by `childSessionKey` but delete cleanup removes only the representative run ID, and orphan pruning has no deferred retry wakeup. I did not execute a runtime repro because this sweep is read-only.

## Solution Assessment

No, not as-is. The invariant is the right fix direction, but it must be ported from the older monolithic registry patch into `subagent-registry-run-manager.ts`, `subagent-registry-lifecycle.ts`, and `subagent-registry-helpers.ts`.
