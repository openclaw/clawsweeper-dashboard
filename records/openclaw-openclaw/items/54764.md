## Summary

Keep open: the cleanup bug still appears valid on current main, but this branch applies the invariant to the old monolithic registry while the active path now lives in split run-manager, lifecycle, and helper modules.

## What This Changes

The PR refactors subagent cleanup bookkeeping in `src/agents/subagent-registry.ts` and adds a steer-restart regression test for delete-mode termination idempotency.

## Best Possible Solution

Port the cleanup invariant into `subagent-registry-run-manager.ts`, `subagent-registry-lifecycle.ts`, and `subagent-registry-helpers.ts` with focused tests for shared-child delete cleanup and orphan-prune retry wakeups.
