## Summary

Keep open: the intended cleanup invariant is valid, but this external PR is not merge-ready because it targets the older monolithic registry while current main uses split registry modules, and it still lacks required real behavior proof.

## What This Changes

The PR extracts final subagent cleanup bookkeeping into a helper in `src/agents/subagent-registry.ts` and adds a steer-restart test for delete-mode termination idempotency.

## Best Possible Solution

Port per-run cleanup finalization and orphan-prune deferred retry wakeup into the current split registry modules, with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. On current main, two delete-mode run records sharing a `childSessionKey` can both be marked ended, but cleanup bookkeeping runs once for the grouped child entry and deletes only that representative run id.

## Solution Assessment

No, not as-is. The invariant is the right direction, but current main's active implementation lives in `subagent-registry-run-manager.ts`, `subagent-registry-lifecycle.ts`, and `subagent-registry-helpers.ts`, and the branch also lacks real behavior proof.
