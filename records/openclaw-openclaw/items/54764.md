## Summary

Keep open. The branch targets a real subagent cleanup invariant, but it changes the old monolithic registry while current main routes the active behavior through split run-manager, lifecycle, and helper modules; the external PR also lacks required real behavior proof.

## What This Changes

The PR refactors subagent cleanup bookkeeping so kill and orphan-prune paths share final cleanup handling, delete-mode terminations are idempotent, and deferred cleanup retries are preserved.

## Best Possible Solution

Port the cleanup invariant into the split registry modules so every updated run id is finalized, orphan pruning wakes deferred cleanup retry handling, and focused regression tests cover both shared-child-session and orphan-prune cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two delete-mode runs sharing a childSessionKey, terminate by that child session, and the run manager updates both run ids while cleanup bookkeeping runs only for one grouped child entry.

## Solution Assessment

No, not as-is. The intended invariant is the right direction, but it needs to be ported from the stale monolithic file into current main's run-manager, lifecycle, and helper modules.
