## What This Changes

The issue asks for a plugin-side API to invoke registered gateway methods for inter-plugin request/response communication.

## Best Possible Solution

Resolve this through SDK/runtime design review, either by landing a revised #46485 or a narrower replacement seam with docs, API baseline updates, tests, and authorization semantics.

## Reproduction Assessment

## Solution Assessment

Unclear for the exact requested API name. A documented runtime dispatcher or typed service/capability seam may be maintainable, but it needs maintainer review around caller identity and scopes before implementation.
