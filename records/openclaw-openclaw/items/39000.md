## Summary

Keep open: current main still lacks a plugin-side caller for registered Gateway methods, and the only existing dispatch bridge is intentionally scoped to authenticated plugin HTTP routes. The closed raw-dispatch PR was superseded by this issue, while the related host-read RPC issue is narrower and does not cover plugin-to-plugin registered methods.

## What This Changes

The issue requests a public plugin SDK API that lets one plugin invoke Gateway methods registered by another plugin.

## Best Possible Solution

Design a narrow documented plugin interop or Gateway-call seam with caller attribution, entitlement checks, method-scope policy, plugin-handler routing, and focused regression coverage.

## Reproduction Assessment

Yes for the capability gap. Source inspection on current main shows plugins can register Gateway methods and use a request-scope-only dispatch bridge, but neither OpenClawPluginApi nor PluginRuntime exposes a general plugin-side caller.

## Solution Assessment

Unclear for the exact requested API. The real gap should be solved through a maintainer-approved, security-reviewed SDK seam rather than exposing arbitrary Gateway method strings without caller policy.

## Security Review

Status: needs_attention

Summary: Plugin-side Gateway dispatch is security-sensitive because it would expose control-plane method access across plugin trust boundaries.

Concerns:

- **[medium] Avoid arbitrary Gateway RPC access from plugins:** `src/gateway/method-scopes.ts:150`
  - body: Gateway authorization lets admin scope call anything and falls back to operator.admin for methods without a resolved scope, so a public plugin caller needs an explicit allowlist, method-scope mapping, or entitlement policy before implementation.
  - confidence: 0.9
- **[medium] Preserve plugin caller identity outside HTTP routes:** `src/plugin-sdk/gateway-method-runtime.ts:32`
  - body: The current bridge only runs inside an authenticated plugin request scope that opted into Gateway method dispatch; a broader caller must define how plugin identity and scopes are carried outside that scope.
  - confidence: 0.88
