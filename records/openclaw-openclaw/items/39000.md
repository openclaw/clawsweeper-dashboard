## Summary

Keep open: current main still lacks a plugin-side gateway caller, and #46485 remains an open design candidate rather than a merged implementation.

## What This Changes

The issue requests a public plugin-side API for dispatching registered gateway methods so plugins can communicate through the gateway RPC surface.

## Best Possible Solution

Land a reviewed public runtime gateway dispatch seam, or intentionally reject it, with documented caller identity, scope behavior, SDK/API baseline updates, tests, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection against current main shows registerGatewayMethod exists while OpenClawPluginApi, PluginRuntime, and createPluginRuntime expose no call-side gateway method or runtime.gateway namespace.

## Solution Assessment

Unclear. Exposing the existing dispatcher may be the narrowest implementation, but the public API name, caller identity, and operator scope model require maintainer review before this should be implemented.
