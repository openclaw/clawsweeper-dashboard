## Summary

Keep open: current main still lacks the requested general plugin-side Gateway caller. The existing dispatch helper is deliberately limited to authenticated plugin HTTP routes, and the related raw dispatcher PR was closed in favor of keeping this as the canonical SDK/security design discussion.

## What This Changes

Request to add a plugin-side API for invoking registered Gateway methods from another plugin.

## Best Possible Solution

Define a narrow documented plugin interop/Gateway-call seam with explicit entitlement, caller attribution, plugin-handler routing, method-scope policy, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `registerGatewayMethod` and a reserved HTTP-route dispatch helper, but no general `api.callGatewayMethod()` or plugin runtime caller for hooks/services like the reported use case.

## Solution Assessment

Unclear for the exact requested API. The maintainable path should be narrower than arbitrary Gateway RPC access and should carry plugin identity, caller scope, and an explicit entitlement policy.
