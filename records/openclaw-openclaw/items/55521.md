## Summary

Keep open: the patch has blocking Feishu access-policy and typing-lifecycle regressions, lacks contributor real behavior proof, and should wait for maintainer Feishu/API judgment because native typing work is tracked separately.

## What This Changes

The PR adds a Feishu `Typing` `messageReaction.create` call in `handleFeishuMessage()` immediately after receipt logging, before routing and policy checks.

## Best Possible Solution

If immediate Feishu feedback remains desired, implement it through a managed Feishu typing path only after reply eligibility is known, respecting `typingIndicator` and cleanup, or replace reactions with the native typing/status approach tracked at https://github.com/openclaw/openclaw/issues/69572.

## Reproduction Assessment

Yes. Applying the PR diff makes the reaction call execute before current-main early returns for disabled or disallowed groups, unmentioned mention-required group messages, and blocked DMs; this is clear from source inspection.

## Solution Assessment

No. A receipt-time unmanaged reaction is not the narrowest maintainable solution because it bypasses Feishu ingress policy, config, and cleanup semantics; the safer path is a managed eligible-reply signal or the native typing/status work tracked separately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Move the ack reaction after reply eligibility gates:** `extensions/feishu/src/bot.ts:327-336`
  - body: The new reaction runs before disabled-group, group allowlist, sender allowlist, mention-required, and unauthorized-DM returns. Messages OpenClaw is configured to ignore would still get a visible processing reaction with no reply, which is especially noisy in mention-required group chats.
  - confidence: 0.94
- **[P2] Pair the immediate reaction with cleanup:** `extensions/feishu/src/bot.ts:330-334`
  - body: This direct create call does not keep the returned `reaction_id` or register it with the existing stop path. Feishu typing reactions persist until removed, so eligible messages can retain stale `Typing` reactions after the bot finishes replying.
  - confidence: 0.9
- **[P2] Honor the typingIndicator setting:** `extensions/feishu/src/bot.ts:330-334`
  - body: The new path unconditionally sends a `Typing` reaction, while the current Feishu typing path skips reaction calls when `account.config.typingIndicator` is false. Users who disabled typing indicators would still get visible reactions and API calls.
  - confidence: 0.91
