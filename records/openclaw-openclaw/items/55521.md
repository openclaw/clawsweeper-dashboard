## Best Possible Solution

Rework Feishu acknowledgement through the existing typing lifecycle after ingress, mention, and DM gates, preserving `typingIndicator: false` and coordinating with the native typing/API direction tracked in https://github.com/openclaw/openclaw/issues/69572.

## Solution Assessment

No. The branch adds a raw message-reaction write before authorization and bypasses existing typing configuration/helper behavior; the safer solution is a gated lifecycle change or a maintainer decision to use the native typing/acknowledgement paths tracked by related items.
