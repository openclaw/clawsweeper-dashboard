## Best Possible Solution

Implement any Feishu acknowledgement as managed post-eligibility feedback that respects `typingIndicator`, cleans up through the existing lifecycle or native status path, and is reconciled with https://github.com/openclaw/openclaw/issues/69572.

## Solution Assessment

No. A direct receipt-time reaction is not the narrowest maintainable solution because it bypasses ingress policy, the existing `typingIndicator` setting, and managed cleanup; a post-eligibility lifecycle hook or native typing/status path is safer.
