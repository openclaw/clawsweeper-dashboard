## Summary

Keep open, but the PR is not mergeable as-is: it emits a visible Feishu reaction before ingress gates and outside the managed typing lifecycle, and it still lacks real Feishu proof.

## What This Changes

The PR adds an unconditional Feishu `Typing` `messageReaction.create` call in `handleFeishuMessage()` immediately after receipt logging.

## Best Possible Solution

If immediate feedback remains desired, implement it as a managed post-eligibility Feishu signal that respects `typingIndicator` and cleanup, or defer it to the native typing/status work tracked at https://github.com/openclaw/openclaw/issues/69572.

## Reproduction Assessment

Yes. Applying the PR diff places the reaction call before current-main early returns for disabled or disallowed groups, unmentioned mention-required group messages, and blocked DMs; this is source-reproducible from `extensions/feishu/src/bot.ts`.

## Solution Assessment

No. The direct receipt-time reaction bypasses Feishu ingress policy, the existing `typingIndicator` toggle, and managed cleanup; a post-eligibility managed signal or native typing/status implementation is safer.

  - confidence: 0.91
