## Summary

Keep open: this external PR still has blocking correctness findings, lacks after-fix real Feishu/Lark behavior proof, and overlaps an unresolved maintainer choice about native typing behavior at https://github.com/openclaw/openclaw/issues/69572.

## What This Changes

The PR adds an immediate Feishu `Typing` message reaction in `handleFeishuMessage()` right after receipt logging and before routing or policy checks.

## Best Possible Solution

If immediate Feishu feedback remains desired, implement it through a managed post-eligibility path that respects `typingIndicator` and cleanup, or defer reaction work to the native typing/status direction tracked at https://github.com/openclaw/openclaw/issues/69572.

## Reproduction Assessment

Yes. Applying the PR diff places the reaction call before current-main early returns for disabled or disallowed groups, unmentioned mention-required group messages, and blocked DMs; this is source-reproducible from `extensions/feishu/src/bot.ts`.

## Solution Assessment

No. The proposed unmanaged receipt-time reaction bypasses Feishu ingress policy, the `typingIndicator` toggle, and cleanup semantics; a managed eligible-reply signal or native typing/status implementation is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Move the ack reaction after reply eligibility gates:** `extensions/feishu/src/bot.ts:327-336`
  - body: The new reaction runs before disabled-group, group allowlist, sender allowlist, mention-required, and unauthorized-DM returns. Messages OpenClaw is configured to ignore would still get a visible processing reaction and API call with no reply, which is especially noisy in mention-required group chats.
  - confidence: 0.94
- **[P2] Pair the ack reaction with managed cleanup:** `extensions/feishu/src/bot.ts:330-334`
  - body: This direct create call does not keep the returned `reaction_id` or register it with the existing stop path. The current Feishu typing helper stores reaction state and deletes it on stop, so eligible messages can retain stale `Typing` reactions after the bot finishes.
  - confidence: 0.9
- **[P2] Honor the typingIndicator setting:** `extensions/feishu/src/bot.ts:330-334`
  - body: The new path unconditionally sends a `Typing` reaction, while the existing Feishu typing path skips reaction calls when `account.config.typingIndicator` is false. Users who disabled typing indicators would still get visible reactions and API calls.
  - confidence: 0.91
