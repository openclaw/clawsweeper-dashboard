## Summary

Keep open: the PR’s direct Feishu `Typing` reaction is placed before current reply-eligibility gates and outside the managed typing lifecycle, and the related open Feishu typing API issue means the reaction-based direction needs maintainer/API judgment before merge.

## What This Changes

The PR inserts a Feishu `im.messageReaction.create` `Typing` reaction in `handleFeishuMessage()` immediately after the received-message log.

## Best Possible Solution

If immediate Feishu feedback remains desired, implement it through a Feishu-owned managed typing path only after reply eligibility is known, honoring `typingIndicator` and cleanup, or switch to the native typing/status API tracked by #69572.

## Reproduction Assessment

Yes. Applying the PR diff places the reaction before current-main group/DM no-dispatch gates, and existing tests cover several of those drop-without-dispatch paths.

## Solution Assessment

No. A direct unconditional reaction in `handleFeishuMessage()` bypasses the existing managed typing lifecycle, config toggle, and cleanup semantics; the safer direction is managed Feishu typing after eligibility or the native API path from #69572.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Move the ack reaction after reply eligibility gates:** `extensions/feishu/src/bot.ts:327-336`
  - body: The new `Typing` reaction runs before disabled-group, group allowlist, sender allowlist, mention-required, and unauthorized-DM returns. Messages OpenClaw is configured to ignore would still receive a visible processing reaction with no reply.
  - confidence: 0.94
- **[P2] Clean up the immediate typing reaction:** `extensions/feishu/src/bot.ts:330-334`
  - body: This direct create call discards the returned `reaction_id` and is not registered with the existing stop path. Feishu typing reactions persist until removed, so eligible messages can retain a stale `Typing` reaction after the reply finishes.
  - confidence: 0.9
- **[P2] Honor the typingIndicator setting:** `extensions/feishu/src/bot.ts:330-334`
  - body: The new path unconditionally sends a `Typing` reaction, while current Feishu typing skips reaction calls when `account.config.typingIndicator` is false. Operators who disabled typing indicators would still get visible reactions and API calls.
  - confidence: 0.91
