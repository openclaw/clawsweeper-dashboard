## Summary

Keep open, but not merge-ready. The patch sends a raw Feishu Typing reaction before admission and mention gates, ignores the existing `typingIndicator` opt-out, and bypasses the managed typing lifecycle that handles cleanup and backoff.

## What This Changes

This PR adds a raw Feishu `Typing` message reaction immediately after an inbound message is logged in `handleFeishuMessage`.

## Best Possible Solution

Decide whether Feishu should use native typing/status or a managed reaction-based acknowledgement, then implement it behind existing admission and `typingIndicator` semantics with cleanup, backoff coverage, tests, and live proof.

## Reproduction Assessment

Yes at source level. The PR inserts the reaction at lines 327-336, while the same function returns later for disabled groups, disallowed groups/senders, and missing mentions before dispatch.

## Solution Assessment

No. The current raw reaction is not the safest fix; the maintainable path is a managed Feishu typing/ack lifecycle or the canonical native-typing direction, with admission and config gates preserved.
