## Summary

Keep open. Current main still executes a repeated read when the model asks for it, existing repeated-tool protection is optional outside post-compaction recovery, and the issue lacks a transcript or trajectory proving a current-main regression before compaction.

## What This Changes

Request that OpenClaw avoid re-reading files that should already be present in active session context when the context window is not full.

## Best Possible Solution

Define the intended product policy first: either document the current model-autonomy behavior, or add a scoped unchanged-file repeat-read warning/cache through the agent tool layer with provider-agnostic coverage.

## Reproduction Assessment

No. The issue has only broad steps and no transcript or trace; source inspection shows current main will execute a repeated read when the model requests one, but that is not a high-confidence current-main regression reproduction.

## Solution Assessment

Unclear. Blocking all repeat reads is not clearly safe because files can change and re-reading can be intentional; a scoped unchanged-file policy, warning, or documentation update would be safer after maintainer direction.
