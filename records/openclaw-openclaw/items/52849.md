## Summary

Keep open. Current main still lacks the requested public post-execution/pre-consumption plugin hook; the existing pre-model tool-result middleware covers part of the need but is intentionally limited to bundled plugins. The related #62123 is useful overlapping context, but it is still open and not established as the canonical replacement for this issue.

## What This Changes

Request to add an additive plugin hook that can allow, replace, or block tool results after execution and before same-turn model consumption.

## Best Possible Solution

Keep the issue open for a maintainer-owned plugin API design, preferably consolidated with #62123. The end state should be a documented, versioned public seam or an explicit trust-policy decision explaining why only bundled middleware can rewrite tool results before model consumption.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence code-inspection path: the public hook catalog and docs lack the requested hook, while the only current pre-model result middleware is bundled-only.

## Solution Assessment

Unclear pending maintainer design. A new public hook could be the right narrow API, but the current bundled middleware boundary shows this is a trust-sensitive surface that should not be opened without explicit semantics for blocking, replacement, async failure, provenance, and runtime coverage.
