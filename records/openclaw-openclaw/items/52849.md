## Summary

Keep open. Current main still lacks a public post-execution/pre-consumption plugin hook; the only live tool-result rewrite seam is bundled-only, and exposing that capability needs maintainer API and security design.

## What This Changes

Add a typed `before_tool_result_emit`-style plugin hook that can allow, replace, or block tool results before same-turn model consumption.

## Best Possible Solution

Keep this open for a maintainer-owned plugin API design, preferably coordinated with #62123, ending in either a documented versioned public seam or an explicit policy that pre-model result rewriting stays bundled-only.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence source check is that current public hook names omit the requested hook while the only pre-model result middleware is bundled-only.

## Solution Assessment

Unclear pending maintainer design. A public hook may be the narrow API, but it needs explicit trust, blocking/replacement, async failure, provenance, and runtime coverage semantics first.
