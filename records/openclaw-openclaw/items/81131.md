## Summary

Keep open: this PR targets a real Telegram TTS streaming regression and includes visible Telegram proof, but the patch deletes and clears the already visible streamed text before the replacement voice message is confirmed delivered, which can lose the answer on send failure.

## What This Changes

The PR changes Telegram final voice-payload delivery so streamed text is removed and the audioAsVoice payload is sent with the spoken text as the voice caption.

## Best Possible Solution

Keep the intended Telegram TTS delivery contract explicit, then only remove the streamed preview after the replacement voice message is confirmed delivered and covered by a focused regression test.

## Solution Assessment
