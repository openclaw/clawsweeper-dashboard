## Summary

This PR should stay open: the Telegram-local direction matches the streaming contract, but the patch deletes the existing preview before the replacement voice send is known to have landed and it still lacks real Telegram after-fix proof.

## What This Changes

The PR modifies Telegram reply dispatch so a final `audioAsVoice` payload after a streamed preview deletes the preview and sends the spoken text as the voice caption.

## Best Possible Solution

Keep the fix local to Telegram dispatch, but send the captioned voice payload first, delete/reset the streamed preview only after successful replacement delivery, and require real Telegram proof before merge.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection shows the current-main path that can finalize the streamed text and then send a final TTS voice payload separately.

## Solution Assessment

No as written: the Telegram dispatch boundary is appropriate, but cleanup should be ordered after successful combined delivery and backed by real Telegram proof.
