## Summary

## What This Changes

The PR changes Telegram final reply dispatch so a streamed text message followed by an `audioAsVoice` final payload is replaced by a single captioned voice note.

## Best Possible Solution

Keep the shipped default until maintainers approve the delivery-mode direction; if combined captioned voice is accepted, send the replacement voice successfully before deleting the streamed preview and cover success plus failure paths.

## Reproduction Assessment

Yes from source inspection: the patch deletes the streamed preview before `sendPayload` completes, while current voice delivery can rethrow non-forbidden send failures. The happy-path UI change is also demonstrated by the contributor's Telegram screenshot.

## Solution Assessment

No as-is: the source fix needs to preserve the preview until the replacement send succeeds, and the bundled-versus-separate TTS default should be decided through a maintainer-approved delivery mode rather than a silent default change.
