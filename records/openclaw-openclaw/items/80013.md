## What This Changes

The branch adds checkpoint throttling to `refreshCostUsageCache`, a regression test for reduced cache rewrites/no-op refreshes, and a proof script for production usage-cost cache refresh/load paths.

## Best Possible Solution

Land the cache checkpointing change after maintainer review and required checks, preserving the existing cache format, lock, and atomic writer semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows `refreshCostUsageCache` writes the full cache for every stale file and again after the loop; the PR proof script provides a concrete on-disk corpus path for the production refresh/load code.

## Solution Assessment

Yes. Batching checkpoint writes inside `refreshCostUsageCache` is the narrow maintainable fix because it leaves the cache contract, lock, and atomic writer intact while reducing repeated full-file serialization.
