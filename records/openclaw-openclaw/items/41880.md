## Summary

Keep open. Current main still sends Feishu text post payloads through a single Markdown element, so the requested behavior is not implemented, but this PR is now unmergeable and overlaps with the broader open Feishu URL-boundary candidate in #68514; maintainers should choose the canonical scope before landing or closing either branch.

## What This Changes

This PR changes Feishu post payload construction so a trimmed single bare HTTP(S) URL is emitted as an explicit rich-text link element while other text keeps the Markdown path, and adds regression tests for the URL and normal Markdown cases.

## Best Possible Solution

Land one Feishu plugin-local URL preservation fix that keeps underscore-containing URLs fully clickable without regressing ordinary Markdown, existing Markdown links, sends, or edits; the canonical fix should be broad enough for the user-facing Feishu message paths maintainers want to support, then #41860 and overlapping PRs can be resolved together.

## Solution Assessment

Unclear as the final maintainer choice. This PR is a narrow maintainable fix for URL-only payloads, while #68514 appears to cover the broader case of bare URLs embedded inside normal Markdown text with more boundary tests.
