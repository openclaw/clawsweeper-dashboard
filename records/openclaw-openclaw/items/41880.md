## Summary

Keep this PR open. Current main still sends Feishu text posts as raw `md` content, so the linked underscore-URL bug remains present; this PR is a focused candidate for the exact bare-URL report, but it overlaps with the broader open Feishu URL-boundary PR and has one non-blocking test-isolation issue for maintainer review.

## What This Changes

The PR changes Feishu post payload construction so a trimmed single bare HTTP(S) URL is emitted as an explicit rich-text `a` link element, while other text keeps the existing Markdown path, and adds regression tests for the URL and normal Markdown cases.

## Best Possible Solution

Land one Feishu plugin-local URL preservation fix that keeps full underscore URLs clickable without regressing normal Markdown, existing links, sends, or edits; then close #41860 and explicitly supersede the other Feishu URL PR candidates.

## Reproduction Assessment

Yes. #41860 provides a concrete Feishu send command with an underscore-containing URL, and current main routes that text through the raw Markdown post builder; I did not run a live tenant smoke test in this read-only pass.

## Solution Assessment

Unclear as the final maintainer choice. This PR is a narrow maintainable fix for the exact bare URL-only command, while #68514 is the broader candidate if maintainers want URL boundary preservation inside longer Markdown messages too.
