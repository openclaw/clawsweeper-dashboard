## Summary

Keep open: the bug is real and the branch has useful proof, but the current helper can truncate valid URL basenames that contain escaped slash or backslash characters.

## What This Changes

The branch decodes percent escapes in remote media URL fallback basenames, preserves malformed escapes, adds focused saveRemoteMedia coverage, and adds a changelog entry.

## Best Possible Solution

Decode only after selecting the final URL path segment, replace decoded / and \ with safe filename characters instead of re-baselining, and add regressions for %2F and %5C.

## Reproduction Assessment

Yes. Source inspection plus a read-only Node probe show current main preserves My%20Report.pdf because URL.pathname is passed directly to basenameFromAnyPath, and the same probe shows the PR truncates encoded separator cases.

## Solution Assessment

No, not as written. The decode direction is right, but the implementation should preserve or sanitize decoded separators inside the selected URL segment instead of running basename extraction a second time.

## Review Findings
