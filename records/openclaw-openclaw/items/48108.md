## Summary

Keep open. Current main still has the stale Control UI dropdown path: after a fresh `sessions.list`, the dropdown resolver unconditionally re-adds the active session key even when that deleted direct session is absent from the Gateway response, and the linked implementation PR is closed unmerged.

## What This Changes

The issue asks for the Control UI sessions dropdown to stop showing deleted Gateway sessions that are no longer present in `sessions.json`.

## Best Possible Solution

Make fresh `sessions.list` rows authoritative for normal direct sessions, normalize a missing active direct key to a valid fallback, and preserve intentional grouped or ephemeral fallbacks.

## Reproduction Assessment

Yes. A source-level reproduction is clear: set the Control UI active `sessionKey` to a deleted direct Discord key, return a fresh `sessions.list` response without that key, and `resolveSessionOptionGroups` still adds it to the dropdown.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI session selection/dropdown reconciliation path; fetching fresh data alone is insufficient while the renderer reinserts a missing active direct key.
