## What This Changes

The issue requests that the Control UI sessions dropdown stop showing sessions that were deleted from the Gateway sessions store.

## Best Possible Solution

Keep the shipped Control UI session-source-of-truth behavior; reopen only if a new report shows a distinct stale-dropdown path on current builds.

## Reproduction Assessment

No, not as a current-main bug. The historical stale path is source-proven, but current main removes deleted rows and no longer synthesizes a missing active non-main session option.

## Solution Assessment

Yes. Treating `sessions.list` and `sessions.changed` delete events as authoritative, while preserving only the main-session fallback, is the narrow maintainable fix for the visible stale dropdown entry.
