## Summary

Keep open. Current main still refreshes `sessions.list` after a session mutation, but the Control UI session selector then re-adds the active session key even when that deleted direct session is absent from the fresh Gateway response; the related fix PR was closed unmerged.

## What This Changes

The issue requests that the Control UI sessions dropdown stop showing sessions that have been deleted from the Gateway session store.

## Best Possible Solution

Normalize a missing active direct session to a valid fallback after fresh `sessions.list`, stop rendering missing direct keys in the dropdown, preserve grouped subagent/cron fallbacks, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to set the Control UI active `sessionKey` to a deleted direct session, return a fresh `sessions.list` response without that key, and observe `resolveSessionOptionGroups` add that missing key back into the dropdown.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI session loading and option resolver path; fetching fresh data alone is insufficient while the renderer reinserts missing active direct keys.
