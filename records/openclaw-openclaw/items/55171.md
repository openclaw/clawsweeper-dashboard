## Summary

Keep open: the PR’s agent-mismatch cleanup is too broad and would remove valid explicit cross-agent Feishu bindings used by `/focus` and ACP thread/session routing. Current main routes runtime bindings through a shared helper that preserves the bound target session, so the patch needs a narrower stale-binding predicate before merge.

## What This Changes

The PR changes Feishu inbound routing to unbind runtime conversation bindings when the bound session agent differs from the resolved route agent, updates Feishu lifecycle fixtures to same-agent bindings, and adds a targeted regression test.

## Best Possible Solution

Implement cleanup only for bindings that are provably stale legacy Feishu route-shaped records, while preserving explicit ACP, subagent, and plugin-owned cross-agent runtime bindings and restoring regression coverage for both positive and negative cases.

## Reproduction Assessment

Yes, from source inspection. Create a Feishu runtime conversation binding with `/focus` or `/acp spawn --thread here` to a target session whose agent differs from the normal resolved route, then process the next inbound message through this PR branch; the new mismatch path deletes that valid binding.

## Solution Assessment

No. Blanket cleanup on agent mismatch is not the best solution because agent mismatch is valid for explicit runtime bindings; the safer fix is a route-shaped stale-binding detector with preservation tests for explicit cross-agent bindings.
