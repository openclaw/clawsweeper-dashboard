## Summary

Keep open, but do not merge as written: the PR treats every Feishu bound-agent versus route-agent mismatch as stale, which conflicts with current main's runtime binding contract, tests, and docs that allow explicit cross-agent ACP/focus bindings. It also lacks inspectable real behavior proof and is currently merge-conflicting.

## What This Changes

The PR changes Feishu message routing to unbind runtime conversation bindings whose target session agent differs from the resolved route agent, with regression and fixture test updates.

## Best Possible Solution

Preserve explicit runtime conversation bindings and add stale cleanup only when the target is invalid or dispatch rejects it, preferably in the shared binding or ACP dispatch path with focused Feishu coverage and redacted live proof.

## Reproduction Assessment

Yes for the PR-introduced regression: from source inspection, a Feishu runtime binding to another agent would enter the mismatch branch and be unbound even when it is a valid ACP or focus binding. No live Feishu reproduction was run, and the original stale-binding claim still lacks inspectable proof.

## Solution Assessment

No. Agent mismatch is not a reliable stale-binding signal; the safer solution is to preserve explicit bindings and clean up only invalid targets or failed dispatches.
