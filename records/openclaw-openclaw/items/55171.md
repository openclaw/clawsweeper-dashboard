## Summary

Keep open, but do not merge as-is. Current main treats explicit cross-agent runtime bindings as valid Feishu behavior, while this PR deletes any binding whose bound agent differs from the resolved route agent; the external PR also still lacks inspectable real behavior proof.

## What This Changes

The branch adds Feishu-local unbinding of runtime conversation bindings when the bound session agent differs from the resolved route agent, plus a routing regression test and same-agent fixture updates.

## Best Possible Solution

Preserve explicit runtime conversation bindings, and add cleanup only for provably dead or stale targets through the shared runtime binding route with Feishu proof.

## Reproduction Assessment

Yes. From source inspection, create a Feishu runtime conversation binding to a session whose agent differs from ordinary route resolution, then process the next inbound message on this branch; the mismatch branch unbinds that valid binding and falls back to the resolved route.

## Solution Assessment

No. Agent mismatch is not the best stale-binding signal because ACP, focus, subagent, and plugin-owned runtime bindings can intentionally target another agent; cleanup needs a narrower invalid-target predicate.
