## Summary

Keep open: the PR is a plausible Feishu bug-fix attempt, but the current diff still uses agent mismatch as a stale-binding signal, which can remove valid cross-agent runtime bindings and misroute follow-up messages.

## What This Changes

The PR changes Feishu inbound routing to unbind conversation bindings whose target session agent differs from the resolved route agent and updates Feishu routing tests.

## Best Possible Solution

Preserve explicit runtime conversation bindings, and clean stale Feishu bindings only after a narrower invalid-target, dispatch-failure, or explicit rebind signal proves the record is obsolete.

## Reproduction Assessment

Yes for the PR regression via source review: a route resolved to one agent and a runtime binding targeting another would hit the new mismatch branch, while current main routes to the bound session. No inspectable live Feishu artifact proves the original stale-binding setup.

## Solution Assessment

No. Agent mismatch is not a safe stale signal because current main intentionally preserves explicit cross-agent ACP and focus bindings; cleanup should use a narrower invalid-target, dispatch-failure, or explicit rebind signal.
