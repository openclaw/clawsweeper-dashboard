## Summary

The PR should stay open because its proposed mismatch cleanup conflicts with current runtime binding semantics: explicit Feishu ACP and focus bindings may intentionally route to another agent, and the PR lacks inspectable Feishu proof.

## What This Changes

The PR adds Feishu logic to unbind a runtime conversation binding when its target session agent differs from the resolved route agent, plus regression and fixture test updates.

## Best Possible Solution

Preserve explicit runtime conversation bindings and add stale cleanup only when the target is invalid or dispatch rejects it, ideally in shared binding or ACP dispatch code with focused Feishu coverage.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows a Feishu runtime binding to an ACP or focus session on another agent would be unbound when ordinary routing resolves elsewhere. I did not run live Feishu, and the contributor has not provided inspectable proof.

## Solution Assessment

No. Agent mismatch is not a reliable stale-binding signal; stale cleanup should key off an invalid target or dispatch failure, not intentional cross-agent binding.
