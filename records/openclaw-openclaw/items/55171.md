## Summary

Keep open: the stale Feishu binding cleanup is plausible, but this PR’s blanket agent-mismatch unbind would also remove intentional cross-agent ACP, subagent, and focus bindings that current main supports.

## What This Changes

The PR changes Feishu inbound routing to unbind runtime conversation bindings whose target session agent differs from the resolved route agent, updates Feishu lifecycle fixtures, and adds a stale-binding regression test.

## Best Possible Solution

Implement a narrow Feishu stale route-shaped binding detector that removes only legacy route-shaped bindings, preserves explicit ACP/subagent/focus/plugin-owned bindings, and adds changelog plus positive and negative regression coverage.

## Reproduction Assessment

Yes. A high-confidence review reproduction is to create a Feishu runtime conversation binding via `/focus` or `/acp spawn --bind/--thread` to a target session whose agent differs from the resolved route, then process the next inbound message through this PR branch and observe the binding being unbound.

## Solution Assessment

No. Blanket cleanup on agent mismatch is too broad; the maintainable fix is a route-shaped stale predicate like the current Discord path, with explicit-binding preservation tests.
