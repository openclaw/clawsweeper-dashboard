## Summary

Keep this PR open, but the proposed fix is currently unsafe: it treats every Feishu binding whose session-key agent differs from the resolved route agent as stale, while current runtime bindings intentionally support cross-agent routing.

## What This Changes

The PR changes Feishu routing to unbind conversation bindings whose session-key agent differs from the currently resolved route agent, and updates Feishu tests for that cleanup path.

## Best Possible Solution

Keep conversation bindings authoritative for explicit/subagent cross-agent targets, and solve stale rebinding with metadata or route-generation evidence that distinguishes stale configured-route leftovers from intentional runtime bindings.

## Reproduction Assessment

Yes from source inspection. Current main has an executable contract where a `main` route plus a bound `agent:review:...` session rewrites to `review`, and the PR’s agent-mismatch unbind would take the opposite branch for that same shape.

## Solution Assessment

No. The PR’s broad `boundAgentId !== route.agentId` rule is not the narrowest maintainable fix because current Feishu subagent and ACP conversation bindings can intentionally target a different agent than the initial route.
