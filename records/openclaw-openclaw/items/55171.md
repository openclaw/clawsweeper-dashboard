## Summary

Keep open: the PR targets a real Feishu stale-binding cleanup problem, but the proposed blanket agent-mismatch unbind would delete valid explicit cross-agent runtime bindings, and current main has not implemented the intended cleanup safely.

## What This Changes

The PR changes Feishu inbound routing to unbind runtime conversation bindings when the bound session agent differs from the resolved route agent, updates lifecycle fixtures to same-agent bindings, and adds a targeted bot regression test.

## Best Possible Solution

Implement Feishu cleanup only for provably stale route-shaped legacy bindings, analogous to the Discord detector, while preserving explicit ACP, subagent, and plugin-owned cross-agent runtime bindings.

## Reproduction Assessment

Yes, from source inspection of the PR diff. Create a Feishu runtime conversation binding via /focus or ACP --thread here to a session whose agent differs from the resolved route, then handle the next inbound message on this PR branch; the new mismatch branch calls unbind and falls back.

## Solution Assessment

No. Blanket cleanup on agent mismatch is not the narrowest maintainable fix because current runtime binding contracts allow explicit cross-agent bindings; a route-shaped stale-binding detector with preservation tests is safer.
