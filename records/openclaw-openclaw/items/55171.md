## Summary

Keep open: the PR still introduces a Feishu routing regression because current main deliberately preserves explicit runtime bindings to cross-agent target sessions, while the branch unbinds any bound-agent versus route-agent mismatch. The external PR also lacks inspectable real behavior proof and live metadata reports the branch as conflicting.

## What This Changes

The PR adds Feishu-local cleanup for conversation bindings whose target session agent differs from the resolved route agent, plus regression tests and fixture changes.

## Best Possible Solution

Preserve explicit runtime conversation bindings and add any stale cleanup through the shared binding route only when the target is provably dead, invalid, or rejected by dispatch.

## Reproduction Assessment

Yes. From source inspection, create a Feishu runtime conversation binding whose target session agent differs from ordinary route resolution; on this PR branch, the mismatch path unbinds that valid binding and falls back to the resolved route.

## Solution Assessment

No. Agent mismatch is not a safe stale-binding signal because ACP, focus, subagent, and plugin-owned runtime bindings can intentionally target another agent; cleanup needs a narrower invalid-target predicate.
