## Summary

Keep this PR open; the cleanup goal is valid, but the current patch still removes valid explicit cross-agent Feishu conversation bindings and is not safe to merge as written.

## What This Changes

The PR changes Feishu inbound routing to unbind runtime conversation bindings when the bound session agent differs from the resolved route agent, adds a regression test, and changes Feishu lifecycle fixtures to same-agent bindings.

## Best Possible Solution

Implement a narrow Feishu stale-binding detector that removes only legacy route-shaped stale bindings, preserves explicit ACP/subagent/plugin-owned bindings, restores cross-agent regression coverage, and adds the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence review reproduction is to create a Feishu runtime conversation binding via /focus or /acp spawn --bind/--thread to a target session whose agent differs from the resolved route agent, then process the next inbound message through this PR branch; the new mismatch path unbinds that valid binding.

## Solution Assessment

No. Blanket cleanup on agent mismatch is too broad because agent mismatch is valid for intentional cross-agent bindings; a route-shaped stale predicate with positive and negative regression tests is the narrower maintainable fix.
