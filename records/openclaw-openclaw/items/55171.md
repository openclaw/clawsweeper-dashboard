## Summary

Keep this PR open for changes: the Feishu stale-binding problem is plausible, but the proposed blanket agent-mismatch cleanup would remove valid explicit cross-agent /focus and ACP conversation bindings and the PR is missing the required changelog entry.

## What This Changes

The PR changes Feishu inbound runtime binding routing to unbind bound conversations when the bound session agent differs from the resolved route agent, adds a regression test, and adjusts lifecycle fixtures to same-agent targets.

## Best Possible Solution

Implement a narrow Feishu stale-route cleanup that removes only legacy route-shaped bindings after route-agent changes while preserving explicit focus, ACP, subagent, and plugin-owned bindings, with positive and negative regression coverage plus changelog.

## Reproduction Assessment

Yes. On this PR branch, create a Feishu runtime conversation binding with /focus or /acp spawn --thread here to a target session whose agent differs from the resolved route agent, then process the next inbound message; the new mismatch path unbinds that valid binding.

## Solution Assessment

No. The broad mismatch cleanup is not the narrowest maintainable fix; the safer direction is a route-shaped stale predicate like the Discord implementation, with explicit binding intent excluded.
