## Summary

Keep this PR open, but it is not mergeable as written: the diff treats any Feishu bound-session agent mismatch as stale even though current routing uses cross-agent runtime bindings intentionally for `/focus` and ACP thread/session bindings.

## What This Changes

The branch adds Feishu agent-mismatch cleanup for bound conversations, updates routing tests/mocks, and changes lifecycle fixtures to same-agent bound session keys.

## Best Possible Solution

Implement cleanup only for bindings that are provably stale or dead while preserving explicit ACP, subagent, and plugin-owned cross-agent runtime bindings, with regression coverage for both cleanup and preservation paths.

## Reproduction Assessment

Yes, from source inspection. Create a Feishu runtime conversation binding with `/focus` or `/acp spawn --thread here` to a target session whose agent differs from the normal resolved route, then process the next inbound message through this PR branch; the mismatch path deletes that valid binding.

## Solution Assessment

No. Agent mismatch alone is not a safe stale-binding signal; the maintainable fix is a narrow stale/dead binding cleanup path layered into current shared binding routing while preserving explicit cross-agent bindings.
