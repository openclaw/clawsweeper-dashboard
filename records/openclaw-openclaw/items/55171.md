## Summary

Keep this PR open, but it should not merge as-is: the proposed Feishu-local agent-mismatch cleanup would delete valid explicit cross-agent conversation bindings, and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds Feishu cleanup that unbinds runtime conversation bindings when the bound session agent differs from the resolved route agent, plus regression and fixture updates.

## Best Possible Solution

Keep explicit runtime conversation bindings authoritative, and add stale cleanup only through a shared invalid-target or dispatch-failure predicate that cannot delete valid ACP, focus, subagent, or plugin-owned bindings.

## Reproduction Assessment

Yes for the introduced regression: source inspection shows a Feishu runtime binding to another agent would be unbound by the PR's mismatch branch even though current main treats that binding as valid. I did not run live Feishu because this is a read-only review and the PR lacks inspectable proof.

## Solution Assessment

No. Agent mismatch is not a safe stale-binding signal; the narrower maintainable path is to preserve explicit bindings and clean them up only when the target is provably stale, invalid, or rejected by dispatch.
