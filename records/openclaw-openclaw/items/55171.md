## Summary

This PR should stay open because the proposed Feishu cleanup treats any route-agent mismatch as stale, but current OpenClaw behavior and docs allow explicit cross-agent focus and ACP conversation bindings.

## What This Changes

The PR adds Feishu runtime conversation-binding cleanup on agent mismatch, adjusts related mocks/tests, and changes lifecycle fixtures to same-agent bound sessions.

## Best Possible Solution

Implement Feishu cleanup only for provably stale route-shaped bindings, preserve explicit focus/ACP/subagent/plugin-owned bindings, and add regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. From source inspection, create an explicit Feishu `/focus` or `/acp spawn --thread here` binding to an agent different from the normal route; the PR branch's mismatch path deletes that valid binding on the next inbound message.

## Solution Assessment

No. Blanket cleanup on agent mismatch is not the best fix because agent mismatch is valid for explicit runtime bindings; the safer direction is the route-shaped stale-binding heuristic already used by Discord.
