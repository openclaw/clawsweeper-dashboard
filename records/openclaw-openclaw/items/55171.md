## Summary

Keep this PR open, but the current patch should not merge as-is. The stale Feishu binding cleanup goal is valid, but the proposed agent-mismatch unbind would also delete intentional cross-agent runtime bindings used by Feishu `/focus` and ACP bound conversations.

## What This Changes

The PR changes Feishu inbound routing to unbind runtime conversation bindings when the bound session agent differs from the resolved route agent, adds a regression test, and adjusts Feishu lifecycle fixtures.

## Best Possible Solution

Implement Feishu stale cleanup with a narrow route-shaped predicate: remove only legacy session bindings whose parsed Feishu route matches the current conversation route except for agent id, while preserving explicit ACP, subagent, plugin-owned, and metadata-backed conversation bindings.

## Reproduction Assessment

Yes. A high-confidence review reproduction is to create a Feishu runtime binding via `/focus` or `/acp spawn --bind/--thread` to a target session whose agent differs from the resolved route agent, then process the next inbound message through this PR branch; the new mismatch path unbinds that valid binding.

## Solution Assessment

No. Blanket cleanup on agent mismatch is not the best fix because agent mismatch is valid for intentional cross-agent bindings; the safer fix is a narrow stale-route detector with regression coverage for both stale legacy leftovers and valid explicit bindings.
