## Summary

Keep open: the proposed Feishu mismatch cleanup is unsafe because current main treats explicit cross-agent runtime bindings as valid, and the external PR still lacks inspectable real behavior proof.

## What This Changes

The branch adds Feishu logic to unbind runtime conversation bindings when the bound session agent differs from the resolved route agent, plus routing regression tests and same-agent fixture updates.

## Best Possible Solution

Handle stale or dead binding cleanup only when the target is provably invalid, preferably in the shared runtime binding route or Feishu binding manager, while preserving ACP, `/focus`, subagent, and plugin-owned cross-agent bindings.

## Reproduction Assessment

Yes, for the PR regression from source inspection: a Feishu runtime binding to a target session whose agent differs from ordinary route resolution would enter the new mismatch branch, delete the valid binding, and fall back. The original stale-binding claim still lacks inspectable live proof against current main.

## Solution Assessment

No. Agent mismatch alone is not a safe stale-binding signal; the narrower maintainable fix must prove the binding target is invalid while preserving intentional cross-agent binding routes.
