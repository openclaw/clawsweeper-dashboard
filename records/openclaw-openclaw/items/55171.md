## Summary

Keep open: the branch still has a blocking correctness issue because it treats agent mismatch as a stale-binding signal, while current main and Feishu docs preserve explicit cross-agent conversation bindings. The external PR also lacks inspectable real behavior proof.

## What This Changes

The branch adds Feishu runtime-conversation unbinding when a bound session agent differs from the resolved route agent, plus routing regression tests and same-agent fixture changes.

## Best Possible Solution

Implement stale or dead binding cleanup only when the target is provably invalid, preferably through the shared runtime binding route, while preserving explicit ACP, subagent, focus, and plugin-owned cross-agent bindings.

## Reproduction Assessment

Yes, from source inspection. Create a Feishu runtime conversation binding with `/focus` or `/acp spawn --thread here` to a target session whose agent differs from ordinary route resolution, then process the next inbound message on this branch; the mismatch branch deletes the valid binding and falls back to the wrong route.

## Solution Assessment

No. Agent mismatch alone is not a safe stale-binding signal; the maintainable fix needs a narrower stale/dead binding check that preserves explicit cross-agent runtime bindings.
