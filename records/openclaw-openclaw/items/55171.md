## Summary

Keep open: the cleanup goal may still be useful, but this PR is not mergeable because it treats any bound-agent mismatch as stale even though Feishu supports intentional cross-agent ACP, focus, and subagent bindings.

## What This Changes

The PR updates Feishu inbound routing to actively unbind bound conversations whose target session agent differs from the resolved route and adds or adjusts Feishu tests for that path.

## Best Possible Solution

Implement Feishu stale-binding cleanup only for legacy route-shaped bindings whose session-key suffix matches the current route while preserving explicit ACP, focus, subagent, and plugin-owned bindings.

## Reproduction Assessment

Yes. Bind a Feishu conversation via /focus or /acp spawn --thread here to a target session whose agent differs from the configured route, then process another inbound message through this PR branch; the new mismatch path unbinds that valid binding.

## Solution Assessment

No. Blanket cleanup on agent mismatch is broader than the binding contract; a Discord-style route-shaped stale predicate with positive and negative Feishu regression tests is the narrower maintainable fix.
