## Summary

## What This Changes

Request relevance-aware bootstrap selection, incremental compaction, and response-derived observation extraction.

## Best Possible Solution

Split the work into a focused generic-bootstrap selection decision, retain compaction strategies behind the existing provider/context-engine contracts, and keep observation extraction plugin-owned unless a concrete missing lifecycle hook is demonstrated.

## Reproduction Assessment

Yes for generic bootstrap files: current source and tests deterministically preserve only the beginning and end once a file exceeds its budget. Incremental compaction and observation extraction are separate architecture proposals, not one failing current-main reproduction.

## Solution Assessment

No as one bundled request; the narrow maintainable path is a product decision for generic bootstrap selection while using existing plugin contracts for compaction and observation behavior.
