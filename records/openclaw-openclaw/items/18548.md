## What This Changes

Request relevance-aware bootstrap file selection, incremental compaction architecture, and observation extraction from existing LLM responses.

## Best Possible Solution

Split this into narrower work: a focused bootstrap relevance or structure-aware selector, plus separate plugin/API discussions for compaction and observation extraction only if current seams are insufficient.

## Reproduction Assessment

## Solution Assessment

No as one combined core change: the maintainable path is to split the bootstrap relevance work from plugin-owned compaction and observation decisions. Current plugin/context-engine surfaces should be extended only if a concrete missing seam is proven.
