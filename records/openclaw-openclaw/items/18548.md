## Summary

Keep open. Current main has fixed the original bootstrap ratio arithmetic and added compaction/context plugin surfaces, but oversized bootstrap files are still selected by fixed prefix/suffix slicing and this issue bundles several architecture decisions that need maintainer scoping.

Request proposes relevance-aware bootstrap context selection, incremental compaction improvements, and observation extraction from existing LLM output.

## Best Possible Solution

Split the work into narrower trackers: define a relevance- or structure-aware bootstrap selector, keep compaction changes aligned with existing compaction-provider/context-engine contracts, and leave observation extraction plugin-owned unless a specific SDK hook gap is identified.

## Reproduction Assessment

## Solution Assessment

No, not as one bundled implementation item. The maintainable route is to split bootstrap relevance from compaction-provider/context-engine work and observation-hook/plugin policy before any fix PR.
