## Summary

Keep open. Current main still has the reported behavior: `generic_repeat` is explicitly warn-only, warning-level detections do not block tool execution, and `process` with `action: "list"` is still not treated as a known polling call. PR #60248 remains open and unmerged as the narrow implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `tools.loopDetection` escalate `genericRepeat` to critical/blocking at `criticalThreshold`, consider treating `process(action="list")` as known polling, and evaluate a separate tool-only-turn circuit breaker.

## Best Possible Solution

Land or replace the scoped loop-detection fix so `generic_repeat` can produce a critical blocking result at `criticalThreshold` when repeated calls show no progress, update unit/e2e expectations and public config docs, and handle `process(action="list")` plus any tool-only-turn breaker as explicit follow-up scope if they are not included.
