Keep open: this is a fresh Control UI/WebChat history-loss bug report, and current main still has a plausible source path where a provider-specific assistant turn can be present in JSONL but become non-renderable after `chat.history` reload. ## What This Changes

Fix Control UI/WebChat so earlier assistant replies remain visible after later sends and reloads when the backend transcript still contains those turns.

## Best Possible Solution

Add a focused regression for the JSONL-present disappearing assistant turn, then fix the provider/runtime projection so true final answer text is persisted or projected into supported display fields while internal reasoning remains hidden.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection gives a medium-confidence fixture path: an assistant transcript row with visible text only in `reasoning_content` or thinking-like fields lacks the `content`/`text` shape that `chat.history` and the Control UI render after reload.

## Solution Assessment

Unclear until a failing regression captures the actual JSONL shape. The safest direction is not to display raw reasoning fields, but to normalize the provider/runtime output so durable final answers use the existing display contract.
