## What This Changes

The PR adds a fallback in steer session resolution that tries sibling direct and DM lane keys when a slash-lane target has no active embedded run.

## Best Possible Solution

Refresh this PR or replace it with the same narrow fallback on current main, add a focused steer regression test, and provide redacted real Discord/native-command proof or an explicit maintainer proof override before merge.

## Reproduction Assessment

Yes for a source-level path: use a non-native/text /steer command whose sessionKey is a :slash: lane while the only active embedded run is on a sibling :direct: or :dm: key. Current main resolves the slash key first and then continues with the payload as a normal prompt when that lane has no active run.

## Solution Assessment

Mostly yes, but not merge-ready: resolveSteerSessionId() is the narrow place that sees active run state and can preserve current behavior when no sibling is active. The branch still needs refresh, a regression test, and real behavior proof before maintainers can accept the session retargeting behavior.
