Yes, source-level: a text command whose chosen target key is `:slash:` returns undefined when only a sibling `:direct:` or `:dm:` key has an active run. I did not run a live Discord or Telegram replay in this read-only review.

Mostly yes, but not merge-ready: `resolveSteerSessionId()` is the narrow place that sees active-run state and preserves existing fallback behavior when no sibling is active. The branch still needs refresh, regression coverage, and real behavior proof before maintainers can accept the session retargeting behavior.
