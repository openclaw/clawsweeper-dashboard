## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep validation from decoding whole attachments:** `src/gateway/chat-attachments.ts:122`
  - body: `roundTripBase64` fully decodes the canonical payload during validation after only the generic `maxBytes` check. That means an image above `MAX_IMAGE_BYTES` but under `agents.defaults.mediaMaxMb` is decoded before the later image cap rejects it, weakening the existing estimate-before-decode resource guard; use a non-decoding character validator instead.
  - confidence: 0.88

  - body: The validation round trip decodes attachment content before the later `MAX_IMAGE_BYTES` image guard, so large rejected images can still allocate decoded buffers during validation.
