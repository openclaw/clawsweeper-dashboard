## Summary

Keep open. Current main already covers meaningful pieces of this request, including configurable DM session isolation, context usage UI, compaction, memory flush, and compaction checkpoints, but the requested smart archival/recovery workflow still bundles new product behavior and policy choices that are not implemented as described.

Split this into scoped maintainer-owned work: keep the existing `dmScope` isolation path, and design the remaining context archival/recovery UX around compaction, memory search, and context-engine ownership before implementation.
