## Summary

Keep open: current main still rejects unsupported persisted cron payload kinds and can later persist the filtered store, while the linked fix PR is open and unmerged.

## What This Changes

The issue asks OpenClaw to preserve persisted cron jobs whose `payload.kind` is not `systemEvent` or `agentTurn`, or at least warn instead of silently dropping them.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/84971 so unsupported persisted payload kinds survive load and full persist, then handle runnable legacy conversions separately through doctor migration coverage.

## Reproduction Assessment

Yes from source inspection. A `jobs.json` row with an unsupported string `payload.kind` is rejected during `ensureLoaded`, omitted from `state.store.jobs`, and can be absent from later full cron-store writes.

## Solution Assessment

Yes for the lossless-persistence portion. The narrow maintainable fix is to preserve unsupported rows through reload and full persist while leaving execution of non-canonical kinds to a separate, explicit doctor migration decision.
