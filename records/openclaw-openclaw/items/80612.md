## Summary

Keep open. The feature direction is plausible, but the patch still has source-reproducible WhatsApp lifecycle regressions and the updated proof only shows the queued ack, not terminal cleanup or the Telegram emoji-category behavior.

## What This Changes

The PR adds WhatsApp lifecycle status reactions, extends shared status-reaction emoji categories/config keys, and updates Telegram reaction variants plus SDK exports.

## Best Possible Solution

Use a WhatsApp-specific replace-only lifecycle path, preserve audio preflight feedback, update the public config docs, and add redacted live proof for the full WhatsApp lifecycle plus Telegram reaction-category behavior.

## Reproduction Assessment

Yes for the blocking review findings from source inspection: the shared controller's per-emoji cleanup flows into WhatsApp's empty-emoji removal path, and the audio preflight ack is skipped before any status controller exists. I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

No. The safer implementation is a WhatsApp replace-only adapter or finalizer that does not expose per-emoji removal, plus a preflight path that still sends immediate receipt feedback before slow audio transcription.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve WhatsApp's current reaction during cleanup:** `extensions/whatsapp/src/auto-reply/monitor/status-reaction.ts:97-99`
  - body: The shared controller calls removeReaction for tracked prior emojis after setDone/setError and restoreInitial. This adapter turns each removal into sendReactionWhatsApp(..., ""), which clears WhatsApp's single current reaction, so the final or restored reaction can disappear immediately.
  - confidence: 0.91
- **[P2] Keep audio preflight receipt feedback enabled:** `extensions/whatsapp/src/auto-reply/monitor/on-message.ts:141-163`
  - body: When statusReactions.enabled is true, this skips the preflight ack before audio transcription, but the status controller is only created later in processMessage. Audio messages that run early STT can therefore lose the immediate receipt feedback that the current path sends before slow transcription.
  - confidence: 0.82
- **[P3] Update the public status reaction docs:** `src/config/schema.help.ts:1824`
  - body: This feature adds WhatsApp status reactions and new emoji override keys, but the public config guide still lists only Slack, Discord, and Telegram support. Please update the user-facing docs alongside the schema/help changes.
  - confidence: 0.74
