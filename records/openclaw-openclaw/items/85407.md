Keep open: current main still has the same side-effectful root undici load path, and the reported failure matches how undici 8.3.0 initializes a process-global dispatcher before plugin code calls native fetch.

## What This Changes

The issue reports that loading OpenClaw's undici runtime through `createRequire("undici")` can overwrite Node's native undici dispatcher and make channel plugins using `globalThis.fetch` fail with `TypeError: fetch failed`.

## Best Possible Solution

Preserve plugin native `globalThis.fetch` compatibility while keeping OpenClaw's proxy and dispatcher-aware fetch paths explicit and covered by a regression test for gateway startup plus an external-style channel fetch.

## Reproduction Assessment

No high-confidence live reproduction was run here. Source inspection does show a concrete current-main path from gateway proxy bootstrap to a side-effectful undici 8.3.0 root load, which matches the reporter's failure theory.

## Solution Assessment

No patch is proposed yet. The best fix should isolate OpenClaw's undici runtime usage or initialize/use dispatchers without poisoning Node's native fetch dispatcher for plugin code.
