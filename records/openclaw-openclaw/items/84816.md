## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Bypass the warm map for workspace-scoped auth checks:** `src/agents/model-provider-auth.ts:44-49`
  - body: The warmed map is built for `resolveDefaultAgentWorkspaceDir()`, but `matchesWarmedScope` ignores `params.workspaceDir`. `buildModelsProviderData` and model-pickers pass agent-specific workspaces into the auth checker, and provider env/plugin auth resolution uses that workspace, so a non-default workspace can now reuse the default workspace's auth answer and show or hide the wrong providers.
  - confidence: 0.91
- **[P1] Guard warm publication with a current-context token:** `src/agents/model-provider-auth.ts:137`
  - body: Startup and reload both schedule `warmCurrentProviderAuthState` in the background, but the function writes the module global after slow async catalog/auth work without checking that its config or reload generation is still current. A startup warm for the old config can finish after a reload rewarm and reinstall stale provider visibility.
  - confidence: 0.87
- **[P2] Rewarm provider auth when plugin auth surfaces reload:** `src/gateway/server-reload-handlers.ts:320-323`
  - body: The reload hook only clears and rewarms for `models` and default-model paths. Provider auth can come from plugin setup env vars, plugin auth evidence, and synthetic provider auth, while `plugins.entries.*` hot reloads the plugin registry; after this PR those plugin-auth changes can leave the global map stale until a model config change or restart.
  - confidence: 0.82
