## Summary

Keep open: current main still has the user-visible reminder-note behavior and this PR targets a real pain point, but the branch is not merge-ready because it imports a non-exported test constant, reads an untyped config key, and leaves the generated config/help/changelog surface incomplete. ## What This Changes

This PR gates `appendUnscheduledReminderNote` behind a new `messages.unscheduledReminderNote` boolean, adds the key to the session Zod schema and field labels, and adds reminder-guard unit tests.

## Best Possible Solution

Settle the reminder-note UX first, then land one complete implementation with runtime wiring, public types, schema/help metadata, docs or config reference coverage, changelog, and focused integration coverage.

## Solution Assessment

No, not as submitted. A default-on opt-out may be a maintainable direction, but this branch must compile and complete the public config contract, and maintainers should first choose it over the competing strip-before-delivery approach in #39330.
