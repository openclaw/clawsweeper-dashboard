## Summary

Keep open: current main still has no opt-out for the user-visible reminder note, and this PR addresses a valid false-positive path, but the branch is not merge-ready because the new config key is incomplete in the public config contract and competes with another open UX approach.

## What This Changes

This PR adds a `messages.unscheduledReminderNote` boolean gate around the unscheduled-reminder note append path, adds the key to the session Zod schema and field labels, and adds focused reminder-guard tests.

## Best Possible Solution

Settle the reminder-guard UX, then land one complete implementation with runtime wiring, public types, generated schema/help metadata, docs or config reference coverage, changelog, and focused integration coverage.

## Reproduction Assessment

Yes. A `runReplyAgent` result containing text like `I'll remember that` or `I'll remind you tomorrow morning`, with `successfulCronAdds: 0` and no enabled same-session cron, reaches the current append path; current tests assert the appended note behavior.

## Solution Assessment

Unclear. A default-on config opt-out is narrow and backward compatible, but it creates public API surface and should be chosen deliberately against the competing delivery-stripping approach in #39330.
