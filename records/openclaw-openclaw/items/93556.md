## Summary

Keep open: current main and the latest release still have the manifest account listing/account resolution mismatch on the Gateway startup path, while https://github.com/openclaw/openclaw/pull/93686 is an open candidate fix rather than merged implementation.

## What This Changes

The issue reports an external openclaw-weixin account whose raw provider ID is normalized for routing but fails to resolve before Gateway `startAccount`, preventing the configured Weixin session from starting.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/93686, or an equivalent generic fix that makes manifest account listing and resolution share the same normalized lookup invariant, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: current main lists manifest account keys after route normalization but resolves by direct lookup of the normalized key in the raw `accounts` object before `startAccount`. I did not run a live Weixin repro in this read-only checkout.

## Solution Assessment

Yes, the best fix shape is a generic core resolver change that aligns manifest account listing and resolution, not a Weixin-specific special case. The linked open PR appears to target that path, so the issue should remain open until it or an equivalent fix merges.
