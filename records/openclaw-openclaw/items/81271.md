## Summary

Keep open: current main still supports explicit exec host/node selection only, and sender/device-derived node defaults remain a new security-sensitive routing policy rather than implemented behavior.

## What This Changes

The issue requests automatic per-sender exec node routing so a TUI on a node can default shell execution to that same node in multi-node Gateway setups.

## Best Possible Solution

Design an explicit opt-in sender-device routing policy that reuses authenticated Gateway device identity and node presence while preserving the current explicit host/node safety model by default.

## Reproduction Assessment

Not applicable as a bug reproduction: current source and docs define `host=auto` as sandbox-or-Gateway routing and require explicit node selection for multi-node `host=node` use. The reported behavior is the current contract, while the requested behavior is a new routing policy.

## Solution Assessment

Unclear until maintainers choose the policy. Same-device matching is plausible, but the maintainable solution should be opt-in and define identity trust, multiple-node resolution, sandbox interactions, and fallback behavior before implementation.
