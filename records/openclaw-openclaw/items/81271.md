## Summary

Keep this open: current main does not implement per-sender device-to-node exec routing, and the requested behavior is a new exec routing policy with config and security implications rather than a stale or already-fixed bug.

## What This Changes

Requested per-sender exec routing so a TUI connected from the same device as a node can default exec calls to that node in shared-gateway multi-node setups.

## Best Possible Solution

Design an opt-in sender-device-to-node exec routing policy, thread requester device identity into the exec default-resolution path, and add gateway/agent tests for matching-node and fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does confirm the reported current behavior boundary: current exec routing uses session, agent, global, or explicit node selectors, not sender device identity.

## Solution Assessment

Unclear pending maintainer decision. The requested direction is plausible, but the default, opt-in config shape, fallback, and security interaction should be decided before implementation.
