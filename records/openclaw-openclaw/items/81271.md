## Summary

Keep open: current main and the latest shipped docs still define exec routing as explicit host/node policy, while per-sender same-device routing would add a new command-execution policy surface with security and config implications.

## What This Changes

Request per-sender exec routing so a TUI connected from the same device as a node defaults shell execution to that node instead of the Gateway.

## Best Possible Solution

Design an explicit opt-in sender-device routing policy that reuses Gateway device identity and node presence while preserving the current explicit host/node safety model by default.

## Reproduction Assessment

Not applicable as a bug reproduction: current source and docs define `host=auto` as sandbox-or-Gateway routing, with node execution selected explicitly. The issue is a requested new routing mode rather than broken documented behavior.

## Solution Assessment

Unclear until maintainers choose the policy: same-device matching is plausible, but it should be opt-in and specify identity trust, multiple-node resolution, disconnected-node fallback, and sandbox behavior before implementation.
