## Summary

Keep open. Current main still has the false-overflow path, but this PR is not merge-ready because its persistence change is stale against current main's stronger whole-object sanitizer and the user-facing fix needs a changelog entry.

## What This Changes

The PR removes `toolResult.details` from tool-result context estimation, adds top-level persisted detail field truncation, and adds regression coverage.

## Best Possible Solution

Land a rebased, narrow fix that excludes model-invisible `toolResult.details` from estimation, keeps the current 8 KiB whole-object persisted-details sanitizer, adds regression coverage, and records the fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows large hidden `toolResult.details` is counted by the estimator and then used by the preemptive overflow guard, while the LLM boundary strips those details before provider conversion.

## Solution Assessment

No, not as-is. Excluding details from estimation is the right narrow fix, but the persistence portion should preserve current main's whole-object byte cap instead of relying on a top-level field whitelist.
