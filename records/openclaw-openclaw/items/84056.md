## Summary

Keep open: the PR addresses a real embedded-runner cleanup precedence bug, but the current patch drops the session-takeover signal when a prompt error already exists, which can let model fallback continue into a session that has been taken over. The PR body proof is also only a focused Vitest harness run, not after-fix real behavior proof.

## What This Changes

The PR changes embedded-runner cleanup error precedence so a cleanup-time session takeover no longer replaces an earlier provider prompt error, and adds regression tests for both precedence paths.

## Best Possible Solution

Preserve the provider-facing error text while still carrying or throwing the takeover signal so model fallback aborts, then cover that with a fallback-level regression and redacted real behavior proof.

## Reproduction Assessment

Yes from source inspection: current main rejects cleanup errors after prompt errors, and the PR's harness simulates a cleanup lock release throwing takeover. I did not run the test in this read-only review.

## Solution Assessment

No: preserving the provider error is the right direction, but swallowing the takeover is not the best fix because it breaks the existing fallback abort contract. The safer solution is to keep the provider-facing message while carrying the takeover as a cause/wrapper or explicit coordination signal.
