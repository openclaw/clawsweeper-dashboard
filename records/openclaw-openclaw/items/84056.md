## Summary

Keep open: the PR fixes a real cleanup-error precedence problem, but the current patch drops the session-takeover signal that model fallback intentionally uses to stop retrying against a taken-over session.

## What This Changes

The PR preserves an earlier provider prompt error when embedded-attempt cleanup later throws `EmbeddedAttemptSessionTakeoverError`, and adds focused regression tests for provider-error and cleanup-only precedence paths.

## Best Possible Solution

Preserve the provider error for user-facing diagnostics while also carrying the cleanup takeover signal through a cause, wrapper, or explicit attempt field so fallback still aborts; add regression coverage for fallback-classified provider errors plus cleanup takeover.

## Reproduction Assessment

Yes, at source level: current main rethrows cleanup takeover over a prior prompt error, while the PR suppresses that throw and the fallback code only aborts when it can still see the coordination error. I did not run a live gateway race in this read-only review.

## Solution Assessment

No, not as written. The right shape is to preserve the provider error for display without removing the takeover signal from fallback classification.
