## Summary

Keep open: the report has detailed live evidence for wrong-agent manual cron routing, but current main's inspected manual-run path already appears to thread the stored job agent through isolated execution, so there is no terminal close reason or safe guessed fix yet.

## What This Changes

Manual cron triggers should dispatch isolated agent-turn jobs to the cron job's configured `agentId`, matching scheduled cron runs.

## Best Possible Solution

Add a focused regression for a multi-agent cron job force-run through the same queued `cron.run` path used by CLI and Control UI, asserting the run session key, transcript path, delivery lookup, and failure-alert route all use the configured job agent.
