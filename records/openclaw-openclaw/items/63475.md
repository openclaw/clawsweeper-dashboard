## Summary

Keep open. Current main still allows gateway-started Feishu accounts to enter the per-account bot-identity probe path concurrently, and the linked fix PR at https://github.com/openclaw/openclaw/pull/63481 is still open rather than merged.

## What This Changes

The issue requests serializing or staggering Feishu startup bot-info probes across multiple accounts to avoid Feishu API timeouts during gateway startup.

## Best Possible Solution

Land or replace the focused Feishu-owned startup bot-identity throttling fix so gateway per-account startup and aggregate startup share the same rate-limit-safe probe behavior.

## Reproduction Assessment

Yes, at source level. Configure multiple Feishu accounts and start the Feishu channel through gateway startup: current main can run account start tasks concurrently, each accountId path fetches bot identity directly, and live Feishu credentials are only needed to observe the external timeout itself.

## Solution Assessment

Yes. The narrow maintainable solution is a Feishu-owned startup bot-identity probe queue or equivalent throttling path, not serializing all gateway channel startup in core.
