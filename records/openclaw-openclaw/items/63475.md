## Summary

Keep open. Current main still lets gateway-started Feishu accounts reach the account-specific bot-identity fetch path concurrently, and the focused closing PR remains open and unmerged.

## What This Changes

The issue asks OpenClaw to serialize or stagger Feishu startup bot identity probes across multiple accounts to avoid rotating Feishu API timeouts.

## Best Possible Solution

Review, rebase, and land or replace the Feishu-owned shared startup probe queue so the gateway accountId path avoids bot-info bursts without detaching monitor lifecycle or changing gateway-wide concurrency.

## Reproduction Assessment

Yes, at source level. Configure multiple Feishu accounts and start Feishu through gateway startup: current main can fan account tasks out concurrently, and each accountId path can fetch bot identity without a shared queue.

## Solution Assessment

Yes. The narrow maintainable solution is Feishu-local shared probe serialization or bounded staggering, not broad gateway serialization or detached monitor lifecycle changes.
