## Summary

Keep open. Current main still has a source-level path where gateway-started Feishu accounts can fetch bot identity concurrently, and the focused closing PR at https://github.com/openclaw/openclaw/pull/63481 is still open rather than merged.

## What This Changes

The issue asks OpenClaw to serialize or stagger Feishu startup bot identity and tenant-token probes across multiple configured accounts.

## Best Possible Solution

Review, rebase, and land or replace the Feishu-local shared startup probe throttling fix so the gateway accountId path and aggregate path both avoid bot-info probe bursts without serializing unrelated channel startup.

## Reproduction Assessment

Yes, at source level. Configure multiple Feishu accounts and start the Feishu channel through gateway startup: current main can fan account tasks out concurrently, and each accountId path can fetch bot identity without a shared queue.

## Solution Assessment

Yes. The narrow maintainable fix is Feishu-owned shared probe throttling or an equivalent bounded queue, not broad core gateway serialization.
