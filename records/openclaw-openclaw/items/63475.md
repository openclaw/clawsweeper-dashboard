## Summary

Keep open. Current main still lets gateway-started Feishu accounts reach the per-account bot-identity probe path concurrently, and the focused closing PR at https://github.com/openclaw/openclaw/pull/63481 is still open rather than merged.

## What This Changes

The issue reports Feishu multi-account startup bot info probes timing out because concurrent startup requests hit Feishu API rate limits.

## Best Possible Solution

Review, rebase, and land or replace the Feishu-local shared startup probe throttling fix so the gateway accountId path and aggregate path both avoid bot-info probe bursts without serializing unrelated gateway startup.

## Reproduction Assessment

Yes, at source level. Configure multiple Feishu accounts and start the Feishu channel through gateway startup: current main can fan account tasks out concurrently, and each accountId path can fetch bot identity without a shared queue.

## Solution Assessment

Yes. The narrow maintainable solution is a Feishu-owned shared probe queue or equivalent startup throttling path, not a broad core gateway serialization change.
