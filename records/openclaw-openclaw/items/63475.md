## Summary

Keep open. Current main still lets multiple Feishu gateway account starts enter the per-account bot-identity probe path concurrently, and the linked implementation PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to serialize or stagger Feishu startup bot-identity probes across multiple configured accounts to avoid rotating startup timeouts.

## Best Possible Solution

Land a Feishu-owned startup bot-identity throttling path that covers gateway startAccount fan-out and the aggregate monitor path, with regression coverage for concurrent gateway account startup.

## Reproduction Assessment

Yes, at source level. Configure multiple Feishu accounts and start the Feishu channel through gateway startup: current main can run up to four account start tasks concurrently, each accountId path fetches bot identity directly, and live Feishu credentials are only needed to observe the vendor timeout itself.

## Solution Assessment

Yes. The narrow maintainable solution is in the Feishu plugin's startup bot-identity probing layer, shared by gateway-driven and aggregate startup paths, rather than serializing all channel account startup in core.
