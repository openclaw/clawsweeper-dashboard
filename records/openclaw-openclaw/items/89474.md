## Summary

Keep open: current main and v2026.6.9 still preserve Codex context usage as current/last snapshots, mirror Codex tool-call transcript messages with zero usage, and build cost summaries from transcript-visible usage entries. Upstream Codex exposes total and last usage, so the remaining work is a maintainer contract decision for Codex run/billing usage versus visible transcript/context usage.

## What This Changes

The issue asks OpenClaw to include or clearly disclose Codex Runtime tool-loop usage in local usage and cost reporting for tool-heavy sessions.

## Best Possible Solution

Define a separate Codex run or billing usage contract that preserves context-window semantics while exposing or clearly labeling transcript-visible versus cumulative Codex usage.

## Reproduction Assessment

Yes for the source-level accounting shape: current main still mirrors Codex tool-call messages with zero usage and `/usage cost` scans only transcript-visible usage entries. No live sanitized Codex run was available in this read-only review to prove the reported multiplier.

## Solution Assessment

Unclear until maintainers choose the Codex usage contract. The safer solution is a separate run/billing usage path or explicit visible-reported-usage labeling, not reusing cumulative Codex totals as context usage.
