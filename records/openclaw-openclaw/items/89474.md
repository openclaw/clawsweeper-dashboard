## Summary

Keep open: current main still projects only Codex last/current token usage into assistant usage, mirrors Codex tool-call messages with zero usage, and the usage-cost cache sums only transcript-visible usage. Sibling Codex exposes both cumulative and last-call usage, so the accounting gap is not implemented on main and needs a maintainer contract decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to make Codex Runtime usage/cost accounting reflect tool-loop model invocations or clearly label the current transcript-derived values as incomplete reported usage.

## Best Possible Solution

Define a separate Codex run/billing usage accumulator or clearly labeled reported-usage path that preserves last-call context semantics while making /usage cost and usage UI honest for tool-loop runs.

## Reproduction Assessment

Yes for the structural failure path: source inspection shows Codex tool-call mirror records are zero usage, only last/current usage is projected, and usage-cost sums transcript-visible records. A live sanitized Codex run is still needed to independently measure the exact multiplier reported by the user.

## Solution Assessment

Unclear until maintainers choose the usage contract. The safest direction is not to reuse cumulative Codex totals as context usage, but to add a separate billing/run-usage path or explicit reported-usage labeling.
