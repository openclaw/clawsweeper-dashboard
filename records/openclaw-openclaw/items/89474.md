## Summary

Keep open: current main and v2026.6.8 still keep Codex context usage on last/current token snapshots, mirror Codex tool-call transcript messages with zero usage, and build usage-cost summaries from transcript-visible usage records. ## What This Changes

The issue requests accurate or clearly labeled Codex Runtime usage accounting for tool-heavy sessions where transcript toolUse messages record zero usage while only higher-level completion events carry nonzero usage.

## Best Possible Solution

Keep this open until maintainers define and implement a separate Codex run/billing usage path, or explicitly relabel Codex transcript totals as visible reported usage without regressing context-window semantics.

## Solution Assessment

Unclear until maintainers choose the Codex usage contract. The safer direction is a separate run/billing usage path or explicit visible-reported-usage labeling rather than reusing cumulative totals as context usage.
