## Summary

Keep this PR open: the option-description escaping is a useful bug fix with screenshot proof, but the patch is still incomplete because zsh subcommand descriptions remain on the same expansion surface without `$` or backtick escaping.

## What This Changes

The PR adds zsh escaping for `$` and backticks in generated option descriptions and adds regression assertions for that option-description output.

## Best Possible Solution

Finish context-aware zsh escaping for all generated descriptions, including subcommand descriptions, while keeping the option regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main emits `$` and backtick-bearing descriptions into generated zsh strings without escaping them, and the contributor screenshots show the `openclaw models <TAB>` failure before and menu after. I did not run a live zsh repro locally.

## Solution Assessment

No. The helper is the right narrow direction for option descriptions, but the patch leaves subcommand descriptions on the same zsh expansion surface, so it is not the complete maintainable fix yet.
