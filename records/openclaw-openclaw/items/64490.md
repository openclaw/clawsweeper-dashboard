## Summary

Keep open: the PR fixes zsh escaping for option descriptions, but it leaves subcommand descriptions on the same expansion surface, so the patch is not complete yet.

## What This Changes

The PR adds zsh escaping for dollar signs and backticks in option descriptions emitted by `src/cli/completion-cli.ts` and adds regression assertions for those option-description characters.

## Best Possible Solution

Finish one zsh completion escaping patch that covers both option and subcommand description strings, preserves context-appropriate quoting, and validates generated output with focused tests.

## Reproduction Assessment

Yes. Current main source emits `$` and backtick-bearing descriptions into generated zsh completion strings without complete escaping, and the contributor provided before/after zsh screenshots; I did not run a live zsh repro because zsh is unavailable here.

## Solution Assessment

No. The helper is the right direction for option descriptions, but this patch leaves subcommand descriptions on the same zsh expansion surface, so it is not the narrow complete fix yet.
