## Summary

Keep this PR open: it fixes a real zsh completion escaping bug for option descriptions, but the branch still leaves subcommand descriptions vulnerable to the same `$`/backtick expansion path.

## What This Changes

The PR adds a zsh option-description escaping helper and regression assertions for `$` and backticks in generated completion output.

## Best Possible Solution

Land one zsh escaping fix that covers both option and subcommand descriptions while preserving the `_values` quoting contract and adding focused `$`/backtick regression tests.

## Solution Assessment
