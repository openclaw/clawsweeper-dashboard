## Summary

Keep open: this is a valid bug-fix direction with usable screenshot proof, but the patch still leaves subcommand descriptions on the same zsh expansion surface, so it needs one focused repair before merge.

## What This Changes

The PR adds a zsh option-description escaping helper in `src/cli/completion-cli.ts` and regression assertions for `$` and backticks in `src/cli/completion-cli.test.ts`.

## Best Possible Solution

Finish one zsh completion escaping patch that covers both option and subcommand description contexts with focused regression coverage, then land it through the normal maintainer changelog and CI flow.

## Reproduction Assessment

Yes. Current main source shows `$` and backtick-bearing descriptions can flow into generated zsh completion strings without complete escaping, and the contributor provided before/after zsh screenshots; I did not run a live zsh repro because `zsh` is unavailable here.

## Solution Assessment

No. The helper is the right direction for option descriptions, but the patch should also protect subcommand descriptions before this is the narrow complete fix.
