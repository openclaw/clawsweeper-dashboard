## Summary

Keep open: this is a focused CLI bug fix with sufficient screenshot proof for the visible option-description path, but the patch is still incomplete because subcommand descriptions can still expand `$` or backticks in generated zsh specs.

## What This Changes

The PR adds zsh escaping for `$` and backticks in option descriptions and regression assertions in the completion CLI tests.

## Best Possible Solution

Apply expansion-safe zsh description escaping to both option and subcommand descriptions, preserving the current `_values` command action shape and adding focused `$` and backtick regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main has `$OPENCLAW_STATE_DIR` and backtick-containing descriptions flowing through zsh description paths without complete expansion escaping, and the screenshots show the visible option-path error before and after. I did not run a live zsh session in this read-only review.

## Solution Assessment

No, not as-is. Escaping option descriptions is the right direction, but the narrow complete fix must also protect subcommand descriptions generated by `generateZshSubcmdList`.
