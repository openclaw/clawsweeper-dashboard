## Summary

Keep open: current main and the latest release still emit zsh option descriptions without escaping dollar signs or backticks, and this focused PR adds the right generator-side escaping plus regression coverage and real terminal proof.

## What This Changes

The PR routes zsh option descriptions through a dedicated escaping helper and adds regression assertions for backticks and `$OPENCLAW_STATE_DIR` in generated completions.

## Best Possible Solution

Land the focused zsh option-description escaping fix after normal maintainer validation, while keeping positional-argument completion and profile-install path fixes separate.

## Solution Assessment

Yes. Escaping at `generateZshArgs`, where option descriptions are embedded into double-quoted zsh specs, is the narrow maintainable fix; sibling positional-argument and install-profile issues are separate completion problems.
