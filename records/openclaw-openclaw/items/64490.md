## Summary

Keep open: this is a real zsh completion bug and the PR fixes the option-description path, but it leaves subcommand descriptions on the same unescaped zsh evaluation path, so the patch is not ready to merge.

## What This Changes

The PR adds a zsh option-description escaping helper for dollar signs and backticks, plus option-level regression assertions.

## Best Possible Solution

Use one parser-safe zsh description escaping helper for both option and subcommand descriptions, cover both paths with focused tests, and land after normal validation.

## Reproduction Assessment

Yes, by source inspection and contributor screenshots. Current main emits `$` and backtick-bearing descriptions into generated zsh double-quoted specs, and the screenshots show the before/after `openclaw models <TAB>` symptom.

## Solution Assessment

No, not yet. The option-description helper is the right direction, but the same escaping must be applied to subcommand descriptions before this is the narrow complete fix.
