## Summary

Keep open: this is a focused, viable bug-fix PR, and current main plus the latest release still lack dollar-sign and backtick escaping for zsh option descriptions. The related zsh completion PRs are closed-unmerged partial overlap or adjacent behavior, so they do not supersede this PR.

## What This Changes

The PR factors zsh option-description escaping into a helper that escapes double-quoted zsh metacharacters and adds regression coverage for dollar signs, backticks, and literal single quotes.

## Best Possible Solution

Land this focused zsh option-description escaping fix after required branch checks or proof gates are satisfied, while keeping adjacent zsh autoload and positional-completion changes separate.

## Reproduction Assessment

Yes by source inspection and contributor screenshot proof: current main and v2026.6.9 emit zsh option descriptions without escaping dollar signs or backticks. I did not run a local zsh repro in this read-only review.

## Solution Assessment

Yes. Escaping once in `generateZshArgs` is the narrowest maintainable fix because zsh option descriptions enter the double-quoted `_arguments` spec there; subcommand descriptions and other shells already use separate quoting paths.
