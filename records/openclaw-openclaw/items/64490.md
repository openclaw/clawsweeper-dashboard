## Summary

Keep open: current main still has the zsh description-expansion bug and this PR is a useful fix path, but the branch remains incomplete because subcommand descriptions still flow through the old escaping path.

## What This Changes

The PR adds a zsh option-description escaping helper for `$` and backticks and regression assertions for those characters in option descriptions.

## Best Possible Solution

Finish one zsh escaping helper that covers option and subcommand description contexts, add focused regression coverage for both, and land after normal validation.

## Solution Assessment

No, not yet. The option-description helper is the right direction, but the same escaping must also be applied to subcommand descriptions before this is the narrow complete fix.
