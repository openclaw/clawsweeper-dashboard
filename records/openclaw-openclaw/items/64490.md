## Summary

Keep open: current main and the latest release still leave `$` and backticks unescaped in double-quoted zsh option descriptions, while this PR applies a focused generator fix with regression coverage and sufficient before/after screenshot proof.

## What This Changes

The PR adds zsh option-description escaping for dollar signs and backticks in the CLI completion generator and adds regression assertions in the completion CLI tests.

## Best Possible Solution

Land the focused zsh option-description escaping fix after a fresh CI and mergeability refresh, leaving compdef placement and positional completion work to their separate PRs.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.9` leave `$` and backticks unescaped in double-quoted zsh option specs, and the attached screenshots show the before/after zsh completion behavior; I did not run a local zsh repro because zsh is not installed in this checkout.

## Solution Assessment

Yes. Escaping where Commander option descriptions enter `generateZshArgs` is the narrowest maintainable fix for this bug; sibling subcommand descriptions and other shell generators use different quoting contexts and do not need this exact change.
