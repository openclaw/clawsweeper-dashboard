## Summary

Keep open: the patch fixes the root option-description case and includes real zsh screenshot proof, but it still leaves the same escaping bug in subcommand descriptions, so it needs a narrow follow-up before merge.

## What This Changes

The PR adds parser-safe zsh escaping for option descriptions and regression assertions for `$OPENCLAW_STATE_DIR` and backticks in generated option output.

## Best Possible Solution

Use one zsh description-escaping path for both option and subcommand descriptions, add focused coverage for both paths, and land the CLI completion fix through normal validation.

## Reproduction Assessment

Yes for source-level reproduction. Current main embeds `$` and backticks in generated zsh description strings without complete escaping, and the PR screenshots show the visible `openclaw models <TAB>` failure before and after the option-description fix.

## Solution Assessment

No, not as-is. The helper is the right direction for option descriptions, but the complete narrow fix must also apply equivalent escaping to subcommand descriptions.
