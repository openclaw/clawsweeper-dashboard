## Summary

Keep open: current main and the latest release still emit zsh option descriptions without escaping `$` or backticks, while this focused PR fixes that generator boundary with regression coverage and terminal screenshot proof. The remaining work is maintainer landing review and a current CI refresh, not cleanup closure.

## What This Changes

The PR adds a zsh option-description escaping helper for `$` and backticks and adds regression assertions for a description containing both.

## Best Possible Solution

Land the narrow zsh option-description escaping fix on current main, keeping positional-argument completion work separate.

## Reproduction Assessment

Yes, from source rather than a live local zsh repro: current main emits unescaped backticks and `$` from option descriptions into double-quoted zsh `_arguments` specs. The contributor screenshot also shows the after-fix `openclaw models` completion menu without the prior `command not found: models` error.

## Solution Assessment

Yes. Escaping at `generateZshArgs` is the right boundary because changing the `models` descriptions would leave the zsh generator unsafe for future option descriptions.
