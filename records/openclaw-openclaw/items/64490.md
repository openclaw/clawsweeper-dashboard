## Summary

Keep open: the reported zsh escaping bug is real on current main, but this branch is still incomplete because subcommand descriptions take the same expansion-sensitive zsh path without escaping.

## What This Changes

The PR adds zsh option-description escaping for `$` and backticks plus regression assertions for generated completion output.

## Best Possible Solution

Apply expansion-safe zsh description escaping to both option and subcommand descriptions while preserving the existing `_values` command action shape, then cover both `$` and backtick cases in focused completion tests.

## Reproduction Assessment

Yes for source-level reproduction. Current main contains `$OPENCLAW_STATE_DIR` and backtick-containing descriptions flowing through zsh description paths that do not escape `$` or backticks, and the PR screenshots show the visible `openclaw models <TAB>` error disappearing for the option path.

## Solution Assessment

No, not as-is. Escaping option descriptions is the right direction, but the complete narrow fix must also protect subcommand descriptions generated by `generateZshSubcmdList`.
