## Summary

Keep open: the PR fixes the option-description half of the zsh escaping bug, but the same `$`/backtick expansion hazard remains in generated subcommand descriptions.

## What This Changes

The PR adds `$` and backtick escaping for zsh option descriptions and regression assertions for a root option description.

## Best Possible Solution

Extend the shared zsh description escaping to both option and subcommand description paths, preserve the parser-specific quoting shape, and cover both paths with focused `$`/backtick tests.

## Reproduction Assessment

Yes for source-level reproduction. Current main emits zsh completion descriptions from the inspected generator without escaping `$` or backticks, and the attached screenshots show the visible `openclaw models <TAB>` error disappearing for the option-description case.

## Solution Assessment

No, not as-is. The option-description helper is the right direction, but it should also be applied to subcommand descriptions before this is the narrow complete fix.
