## Summary

Keep open: the option-description escaping fix is useful and is not on current main, and the contributor provided real zsh screenshot proof. The patch is still incomplete because subcommand descriptions remain on a separate escaping path even though they are emitted into the same double-quoted zsh specs; current hooks subcommands contain backtick descriptions, and the closed overlapping PR already showed that sibling coverage.

## What This Changes

The PR routes zsh option descriptions through a new escaper for `$` and backticks and adds regression assertions for those characters in completion output.

## Best Possible Solution

Land one shared zsh description escaper for option and subcommand specs, preserve this PR's option coverage, and add the useful sibling backtick coverage from the closed overlapping PR before merge.

## Reproduction Assessment

Yes. Current main has a `models` option description with backticks and emits zsh option descriptions without escaping backticks or `$`; local zsh is unavailable here, but the source path and contributor screenshot make the bug path high-confidence.

## Solution Assessment

No, not as submitted. Routing option descriptions through a dedicated zsh escaper is the right shape, but the same double-quoted description escaping needs to cover subcommand descriptions before this is the best fix.
