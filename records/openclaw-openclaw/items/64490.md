## Summary

Keep open: this is a focused CLI completion bug fix, but the branch is not merge-ready because it protects zsh option descriptions while leaving subcommand descriptions exposed to the same `$` and backtick expansion hazard.

## What This Changes

The PR adds `$` and backtick escaping for generated zsh option descriptions and adds regression assertions for a root option description.

## Best Possible Solution

Extend zsh description escaping to both option and subcommand generation, preserve the subcommand parser quoting rules, and cover `$` plus backticks in both paths.

## Reproduction Assessment

Yes. The bug is source-reproducible from current main because zsh completion descriptions flow into generated double-quoted shell words without escaping `$` or backticks, and the PR screenshots show the visible option-description failure disappearing after the branch change.

## Solution Assessment

No, not as-is. The option-description helper is the right direction, but the same escaping must be applied to subcommand descriptions before this is the narrow maintainable fix.

## Review Findings
