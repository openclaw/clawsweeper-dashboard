The PR adds zsh escaping for option descriptions in generated completions and regression assertions for dollar signs and backticks in those option descriptions.

## Best Possible Solution

Land one shared zsh description-escaping path for both option and subcommand descriptions, with regression coverage for dollar signs and backticks in both generated spec paths.

## Solution Assessment

No, not as submitted. Escaping option descriptions is the right direction, but the same helper needs to cover subcommand descriptions rendered by generateZshSubcmdList before this is the maintainable fix.
