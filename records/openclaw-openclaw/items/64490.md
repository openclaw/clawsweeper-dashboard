## Summary

Keep open: current main still has the zsh description escaping gap, this branch contains a focused fix with regression coverage and after-fix zsh proof, and the related PRs are either separate positional-argument work or closed partial duplicates.

## What This Changes

The branch changes the zsh completion generator so option descriptions escape `$` and backticks via a helper and adds assertions for a description containing a backticked command and `$OPENCLAW_STATE_DIR`.

## Best Possible Solution

Land a narrow zsh option-description escaping fix on top of current main, preserving the newer completion-cli helpers and keeping positional-argument completion work separate.

## Reproduction Assessment

Yes from source: current main generates double-quoted zsh `_arguments` specs with unescaped `$` and backticks in option descriptions, and zsh treats those as expansion syntax. A live zsh session was not available in this read-only environment, but the author provided local zsh screenshot proof.

## Solution Assessment

Yes, with a merge-refresh caveat: the fix belongs at the zsh option-description escaping boundary and does not need to change bash, fish, PowerShell, or the separate positional-argument PR.
