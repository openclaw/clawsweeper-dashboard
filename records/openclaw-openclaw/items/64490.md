## Summary

Keep open. Current main and v2026.6.9 still generate zsh option specs without escaping dollar signs or backticks, while this PR fixes that narrow path with matching regression tests and direct terminal proof; no code or security findings were found, but merge should wait for maintainer/check-state handling.

## What This Changes

The PR adds zsh option-description escaping for dollar signs and backticks in completion generation, plus regression tests for those metacharacters and single quotes.

## Best Possible Solution

Land this focused zsh option-description escaping fix after required checks and proof automation are green or consciously refreshed, keeping adjacent zsh autoload and positional-completion work separate.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.9 leave dollar signs and backticks unescaped in double-quoted zsh option specs, and the attached terminal screenshots show the before/after completion behavior.

## Solution Assessment

Yes. Escaping at generateZshArgs is the narrowest maintainable fix because option descriptions enter the double-quoted zsh _arguments context there; subcommand descriptions and other shells use separate quoting paths.
