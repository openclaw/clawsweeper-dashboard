## What This Changes

The PR adds zsh double-quoted option-description escaping for `$` and backticks in `src/cli/completion-cli.ts` and extends completion CLI tests for those cases.

## Best Possible Solution

Land this focused zsh option-description escaping fix after exact-head checks and proof automation agree with the inspected screenshot proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main embeds `$OPENCLAW_STATE_DIR` and backtick-wrapped command examples in zsh double-quoted option specs without escaping. ## Solution Assessment

Yes, this is the best narrow fix for the submitted bug: escaping at `generateZshArgs` protects every zsh option description at the generation boundary without changing other shells or command behavior. Subcommand descriptions use a different single-quoted zsh context, so leaving that path unchanged is reasonable.
