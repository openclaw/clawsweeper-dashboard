## Summary

Keep open: the PR fixes zsh option descriptions, but subcommand descriptions still sit on the same expansion-prone zsh completion path and need a small repair before merge.

## What This Changes

The PR adds escaping for dollar signs and backticks in generated zsh option descriptions and regression assertions for that output.

## Best Possible Solution

Finish the zsh escaping fix with context-aware coverage for both option and subcommand descriptions while preserving the existing single-quote handling needed by subcommand value specs.

## Reproduction Assessment

Yes. Current main source emits `$` and backtick-bearing descriptions into zsh completion strings without escaping them, and contributor screenshots show the before/after `openclaw models <TAB>` behavior; I did not run a live zsh repro locally.

## Solution Assessment

No. The helper is the right direction for option descriptions, but this patch leaves subcommand descriptions on the same expansion surface, so it is not the narrow complete fix yet.
