## Summary

Keep open. This is the canonical active PR for the zsh description-escaping bug and the attached terminal proof is useful, but the patch is incomplete because it applies the new double-quoted escaping only to option descriptions while subcommand descriptions still use the old path.

## What This Changes

The PR escapes dollar signs and backticks in generated zsh option descriptions and adds focused regression assertions for those characters.

## Best Possible Solution

Use one shared zsh description-escaping helper for every description emitted into generated zsh completion specs, with regression coverage for both option and subcommand descriptions containing `$` and backticks.

## Reproduction Assessment

Yes from source inspection and the zsh grammar: current main emits `$` and backtick-bearing descriptions into zsh completion strings without complete escaping. I did not run a live current-main zsh failure in this read-only review.

## Solution Assessment

No, not yet. The helper is the right direction for option descriptions, but the same escaping needs to cover subcommand descriptions before the bug is fully fixed.
