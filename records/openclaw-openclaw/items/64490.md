## Summary

Keep this PR open: it targets a real zsh completion escaping bug and includes useful option-description coverage, but the same expansion surface remains for subcommand descriptions.

## What This Changes

The PR adds a zsh double-quoted description escaping helper for option descriptions and tests `$` plus backtick escaping in generated completion output.

## Best Possible Solution

Use one shared zsh description-escaping path for every description emitted into double-quoted `_arguments` specs, with regression coverage for option and subcommand descriptions containing `$` and backticks.

## Reproduction Assessment

Yes. Current main emits `$` and backtick-bearing descriptions into zsh double-quoted completion specs, and the zsh manual confirms parameter and command substitution occur in that context.

## Solution Assessment

No. The helper is the right direction for option descriptions, but the patch leaves subcommand descriptions on the same unescaped zsh double-quoted path.
