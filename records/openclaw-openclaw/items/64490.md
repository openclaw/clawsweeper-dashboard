## Summary

Keep open: the option-description fix is useful and the reported zsh completion bug is real, but the PR is incomplete because subcommand descriptions still use the old escaping path and reachable subcommands contain backtick examples.

## What This Changes

The PR adds zsh escaping for `$` and backticks in generated option descriptions and adds regression assertions for a root option description.

## Best Possible Solution

Extend the zsh escaping to both option and subcommand descriptions, preserve the parser-specific subcommand quoting semantics, and add focused `$`/backtick coverage for both paths.

## Reproduction Assessment

Yes. Current main is source-reproducible through the generated zsh completion paths, and the PR screenshots show the visible `openclaw models <TAB>` error disappearing for the option-description case.

## Solution Assessment

No, not as-is. The option-description helper is the right direction, but the same `$`/backtick expansion hazard remains in `generateZshSubcmdList` until that path is escaped and tested.
