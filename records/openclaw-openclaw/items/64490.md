## Summary

Keep open: the reported zsh completion escaping bug is valid and the option-description fix is useful, but the PR still leaves subcommand descriptions on the old escaping path, so the patch is incomplete.

## What This Changes

The PR adds a helper to escape `$` and backticks in generated zsh option descriptions and adds option-level regression assertions.

## Best Possible Solution

Reuse one zsh description escaping helper for both option and subcommand description contexts, add focused subcommand regression coverage, and land the narrow zsh-only fix after scoped validation.

## Reproduction Assessment

Yes. Current main can be source-reproduced by inspecting generated zsh completion paths: `$` and backtick-bearing descriptions flow into zsh strings without complete escaping, and the PR includes before/after zsh screenshots for the visible `models` completion symptom.

## Solution Assessment

No, not yet. The option-description helper is the right direction, but the same escaping must also cover subcommand descriptions before this is the narrow complete fix.
