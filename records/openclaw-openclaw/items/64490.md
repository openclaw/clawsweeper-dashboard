## Summary

Keep this PR open: it fixes the option-description half of a real zsh completion escaping bug, but the current patch still leaves subcommand descriptions exposed to the same `$`/backtick expansion problem.

## What This Changes

The PR adds a helper to escape `$` and backticks in zsh option descriptions and adds regression assertions in `src/cli/completion-cli.test.ts`.

## Best Possible Solution

Land one zsh escaping fix that covers both option and subcommand descriptions while preserving the subcommand `_values` quoting contract, with focused regression tests for `$` and backticks.

## Reproduction Assessment

Yes. Current main source shows unescaped `$OPENCLAW_STATE_DIR` and backtick-wrapped descriptions flowing into zsh double-quoted completion specs, and the contributor supplied before/after zsh screenshots for the reported completion path.

## Solution Assessment

No, not as submitted. Escaping option descriptions is the right direction, but the same expansion-safe handling must be applied to `generateZshSubcmdList` before the fix is complete.
