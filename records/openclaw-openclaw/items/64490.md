## Summary

Keep open: current main and the latest release still emit zsh option descriptions without escaping dollar signs or backticks, while this PR is a focused fix with sufficient real zsh screenshot proof and no blocking review findings.

## What This Changes

The PR routes zsh option descriptions through a double-quoted-context escaper and adds regression assertions for dollar signs and backticks in generated zsh completion output.

## Best Possible Solution

Land the focused option-description escaping fix after normal maintainer validation, and keep positional-argument completion work separate in the related zsh positional-completion PR.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 embed dollar signs and backtick-wrapped examples into double-quoted zsh option specs without escaping them. The contributor's before/after screenshots also show the reported zsh completion error disappearing on the PR branch.

## Solution Assessment

Yes; escaping at the zsh option-description generator is the narrow maintainable fix for this bug. The sibling subcommand list uses single-quoted zsh values, so it does not need the same double-quoted escaping rule, and positional-argument completion should remain separate.
