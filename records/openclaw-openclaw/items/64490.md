## Summary

Keep this PR open, but it still needs a small correctness repair before merge: the option-description escaping is useful and has real zsh proof, while subcommand descriptions in the same generated zsh `_arguments` path still leave backticks and dollar expansion reachable.

## What This Changes

The branch adds zsh escaping for `$` and backticks in generated option descriptions and adds regression assertions for the root option fixture.

## Best Possible Solution

Apply expansion-safe zsh description escaping to both option and subcommand description generation while preserving the existing `_values` quoting rules, then cover both `$` and backtick cases in focused completion tests.

## Reproduction Assessment

Yes for source-level reproduction. Current main emits `$` and backticks into generated zsh description strings without complete escaping, and the PR screenshots show the visible `openclaw models <TAB>` error disappearing after the option-description fix.

## Solution Assessment

No, not as-is. Escaping option descriptions is the right direction, but the complete narrow fix must extend equivalent expansion escaping to subcommand descriptions too.
