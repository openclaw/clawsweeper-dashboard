## Summary

Keep open: this is a useful zsh completion fix with real screenshot proof for the model-completion command-substitution symptom, but it is not merge-ready because the same unescaped-description path remains for subcommand descriptions and current CLI subcommands already contain backtick examples.

## What This Changes

The PR adds a zsh option-description escaping helper for `$` and backticks and adds regression assertions for option descriptions in the completion generator test.

## Best Possible Solution

Use one zsh description escaping path for every description emitted inside double-quoted `_arguments` specs, with regression coverage for both option and subcommand descriptions.

## Reproduction Assessment

Yes. Source inspection shows current option and subcommand descriptions can contain backticks inside generated zsh specs, and the author’s screenshot shows the option-description backtick symptom disappearing after the PR; I did not run a local zsh repro in this read-only review.

## Solution Assessment

No. Escaping option descriptions is the right direction, but the best fix also applies the same escaping to subcommand descriptions because they share the same zsh double-quoted completion-spec context.
