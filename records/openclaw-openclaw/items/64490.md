## Summary

Keep open: this is a real CLI completion bug and the PR fixes the option-description path, but the same double-quoted zsh expansion problem remains for subcommand descriptions that already contain backticks.

## What This Changes

The PR extracts zsh option-description escaping into a helper, escapes `$` and backticks, and adds focused regression assertions in the completion CLI test.

## Best Possible Solution

Apply one shared zsh double-quoted-description escaping path to every description emitted inside double-quoted `_arguments` specs, with tests covering option and subcommand descriptions containing `$` and backticks.

## Reproduction Assessment

Yes. Current main places `$` and backtick-bearing descriptions into double-quoted zsh completion specs, and the zsh manual confirms parameter and command substitution occur there.

## Solution Assessment

No, not yet. The option-description helper is the right direction, but the same escaping needs to be applied to subcommand descriptions before the zsh description-expansion bug is fully fixed.
