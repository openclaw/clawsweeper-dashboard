## Best Possible Solution

Make the cron diagnostic check effective `web_search` availability, including native provider ownership, then add focused regression coverage and redacted real cron-run proof before merge.

## Solution Assessment

No. A preflight warning is the right general shape, but checking only `listWebSearchProviders()` is too narrow because native OpenAI `web_search` can be valid with an empty managed-provider list.
