## Best Possible Solution

Land the split shutdown fix after maintainers receive or explicitly waive redacted real proof that the awaited `--json` path completes fully and exits naturally with the guarded `ws._socket` dependency.

## Reproduction Assessment

## Solution Assessment

Yes, the revised split is the best code shape I found: it avoids forced `process.exit()`, preserves awaited shutdown, and confines the private `ws` dependency behind a guarded helper. The remaining blocker is real behavior proof for the revised awaited path, not a line-level code defect.
