## Summary

Keep this PR open for normal maintainer review. Current main still lacks frozen-final metadata on `subagent_ended`, and the PR is a narrow API addition with focused tests, a changelog entry, and no concrete correctness or security finding from this review.

## What This Changes

The PR adds a privacy-minimal `final` metadata object to `subagent_ended` hook events, containing availability, SHA-256, UTF-8 byte length, and capture timestamp without raw final text.

## Best Possible Solution

Land the narrow hook metadata addition after normal CI and maintainer approval, preserving the no-raw-final-text boundary.

## Reproduction Assessment

Not applicable as a bug reproduction. The behavior is verifiable by emitting `subagent_ended` from a run record with `frozenResultText`, and the PR adds focused assertions for hash, byte length, timestamp, blank omission, and no raw text leakage.

## Solution Assessment

Yes. Reusing the existing frozen-result capture and extending the existing ended-hook payload is narrower than adding delivery/routing behavior or exposing raw final content.
