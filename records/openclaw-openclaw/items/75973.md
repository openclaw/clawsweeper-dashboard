## Summary

Keep open: current main does not expose frozen-final metadata on `subagent_ended`, and source review of the PR found no blocking correctness issue.

## What This Changes

The PR adds optional privacy-minimal frozen final metadata to `subagent_ended` hook events, plus focused tests and a changelog entry.

## Best Possible Solution

Land the narrow hook metadata contract after maintainer acceptance, keeping raw final text out of hook payloads.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The behavior is checkable by emitting `subagent_ended` from a run record with `frozenResultText` and inspecting the hook event payload.

## Solution Assessment

Yes, if maintainers want this public hook contract. Reusing the already-frozen completion result and exposing only digest, byte length, and capture time on the existing hook is narrower than adding delivery behavior or raw content access.
