## Summary

Current main already fixes the underlying SSH remote regression by keeping SSH transport on a loopback `gateway.remote.url` and storing the discovered host in `gateway.remote.sshTarget`; this PR is a superseded alternate that weakens the fail-closed WebSocket guardrail.

## What This Changes

The PR rewrites `gateway.remote.transport=ssh` `gateway.remote.url` values to a loopback URL before WebSocket security validation and adds one gateway connection-details test.

## Best Possible Solution

Keep the shipped SSH transport contract: persist a loopback `gateway.remote.url`, store remote hosts in `gateway.remote.sshTarget`, and leave WebSocket URL validation fail-closed.

## Reproduction Assessment

Yes for the proposed patch defect: source inspection shows malformed SSH remote URLs are caught and rewritten to `ws://127.0.0.1:18789` before validation. The underlying normal SSH flow is already fixed on main by preventing the macOS app from persisting non-loopback URLs for SSH transport.

## Solution Assessment

No. Rewriting remote-host SSH URLs inside the security check conflicts with the shipped config contract; the safer solution is the existing loopback-url plus `sshTarget` model with strict validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Fail closed on malformed SSH URLs:** `src/gateway/connection-details.ts:27-29`
  - body: `resolveSshTunnelLoopbackUrl` catches any parse error and returns `ws://127.0.0.1:18789`, so malformed nonempty SSH remote URL config no longer reaches the existing validator and can connect to a different local gateway with remote credentials. Throw or leave the original value to the validator, and cover the malformed case.
  - confidence: 0.89
- **[P3] Preserve pathless URL shape:** `src/gateway/connection-details.ts:25-26`
  - body: For a bare URL like `ws://203.0.113.10:18789`, `URL.pathname` is `/`, so this helper returns `ws://127.0.0.1:18789/` instead of preserving the configured endpoint shape. Rebuild the URL without inventing a path, or add coverage if the slash is intentional.
  - confidence: 0.82
