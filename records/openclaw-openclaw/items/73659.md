## Summary

Keep open: current main still rejects the reported SSH remote config path, but this patch is not merge-ready because it weakens malformed-url handling and conflicts with the documented loopback/sshTarget contract.

## What This Changes

The PR rewrites SSH-transport remote gateway URLs to a 127.0.0.1 tunnel URL inside `buildGatewayConnectionDetails` and adds one gateway call test for that mapping.

## Best Possible Solution

Settle the SSH remote-url contract, then either keep the documented loopback/sshTarget model or implement a fail-closed effective-endpoint mapping with aligned tests, docs, and changelog.

## Reproduction Assessment

Yes. Current main can be reasoned through with `gateway.mode=remote`, `gateway.remote.transport=ssh`, and `gateway.remote.url=ws://203.0.113.10:18789`: the helper selects that non-loopback URL and the security check rejects it before any tunnel endpoint is used.

## Solution Assessment

No as written. Rewriting to an effective loopback endpoint may be viable, but malformed config must fail closed and the implementation must preserve endpoint shape and match the documented SSH transport contract.
