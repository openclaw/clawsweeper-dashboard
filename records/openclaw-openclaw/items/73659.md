## Summary

Keep open: the PR targets a real current-main failure mode, but the patch is not merge-ready because it weakens malformed URL handling and conflicts with the documented SSH transport contract.

## What This Changes

The PR rewrites `gateway.remote.transport=ssh` remote URLs to a `127.0.0.1` tunnel endpoint in gateway connection details and adds one unit test for that mapping.

## Best Possible Solution

Decide the SSH remote-url contract, then either preserve the documented loopback-plus-`sshTarget` model or implement an effective endpoint mapper that fails closed, preserves URL shape, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes. With current main, `gateway.mode=remote`, `gateway.remote.transport=ssh`, and `gateway.remote.url=ws://203.0.113.10:18789` flow into `isSecureWebSocketUrl()` as a non-loopback plaintext URL and are rejected before any tunnel endpoint is considered.

## Solution Assessment

No as written. Rewriting to an effective loopback endpoint may be viable, but malformed config must not silently fall back to a trusted local URL and the behavior must align with the documented SSH transport contract.
