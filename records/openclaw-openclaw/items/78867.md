Keep this open: current main still leaves non-Claude GitHub Copilot Responses models without a provider policy to drop historical reasoning, and the active closing path is the open linked PR rather than closing the issue before merge.

## What This Changes

The issue reports that GitHub Copilot GPT-5.5 fails across turns because OpenClaw replays prior `encrypted_content` reasoning items that Copilot rejects.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79176, or an equivalent provider-owned Copilot replay policy, after redacted live two-turn Copilot proof confirms cross-turn reasoning works.

## Reproduction Assessment

No, not as a high-confidence live reproduction in this review. The issue gives concrete two-turn steps and current source shows the non-Claude Copilot Responses path would replay prior `thinkingSignature` reasoning items, so it is source-reproducible.

## Solution Assessment

Yes, the linked provider-owned replay-policy approach is the narrowest maintainable fix path because it targets GitHub Copilot without changing native OpenAI Responses replay semantics. It should wait for live Copilot proof or explicit maintainer acceptance of that proof gap.
