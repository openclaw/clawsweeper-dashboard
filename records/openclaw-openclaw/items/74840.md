## Summary

Keep open. The linked cron summary bug is valid, but this PR is not merge-ready because its fallback-only change does not bypass the existing fatal structured-error override that still replaces the summary/output/delivery with the last raw error payload.

## What This Changes

The PR changes the isolated cron outcome fallback so `finalAssistantVisibleText` is considered before the first payload text when payload-derived summary or output text is missing.

## Best Possible Solution

Repair the cron/embedded-runner boundary so final assistant-visible text wins only for a proven recovered or self-corrected mutating-tool failure, while real unresolved payload errors, run-level errors, denials, and non-presentation failures remain fatal.

## Reproduction Assessment

Yes. A high-confidence unit-level reproduction is `resolveCronPayloadOutcome` with only `isError: true` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; current tests already assert that shape returns the raw error, and this PR's fallback change does not reach the fatal override.

## Solution Assessment

No. This is not the best way to solve the issue because it changes fallback text selection after payload summary extraction, while the reported path is decided by fatal structured-error precedence; the safer fix is to classify recovered versus unresolved mutating failures before producing or honoring a fatal cron payload.
