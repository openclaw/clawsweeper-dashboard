## Summary

Keep open. The PR targets a real cron summary bug, but the current diff is still incorrect: it adds final assistant text after the payload fallback helpers, and current main still treats the all-error payload plus recovered final assistant text shape as fatal.

## What This Changes

This PR changes `resolveCronPayloadOutcome` to use `finalAssistantVisibleText` as an intermediate fallback for cron summary and output text selection.

## Best Possible Solution

Repair `resolveCronPayloadOutcome` so trusted recovered final assistant text can win for self-corrected all-error payload summaries while unresolved fatal payloads, run-level failures, and structured delivery continue to report failure, with focused resolver regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only `isError` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; the current helper test asserts the error wins.

## Solution Assessment

No. The PR adds final assistant text too late in the fallback flow; the safer fix is to classify recovered versus unresolved fatal payloads before the fatal delivery override selects the error text.
