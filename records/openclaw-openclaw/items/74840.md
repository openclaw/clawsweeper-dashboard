## Summary

This PR should stay open because the proposed fallback does not fix the linked cron self-correction bug: error payload text is still selected before the new fallback can run, and the fatal delivery override still replaces the final assistant text.

## What This Changes

The PR changes cron isolated-agent outcome fallback selection to consider `finalAssistantVisibleText` before falling back to raw payload text.

## Best Possible Solution

Repair the cron outcome boundary so final assistant-visible text wins for recovered or self-corrected mutating-tool failures while unresolved fatal tool errors remain fatal, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible. Current main’s helper logic and existing test cover the all-error payload shape where `finalAssistantVisibleText` is present but the last error still becomes the output and delivery payload.

## Solution Assessment

No. The PR adds final assistant text too late in the fallback chain; the maintainable fix is to classify recovered versus unresolved tool failures before selecting fatal delivery text.
