## Summary

Keep open. The PR targets a real cron isolated-run delivery bug from #74807, but the submitted diff is not merge-ready because it changes fallback text selection without changing the fatal structured-error path that still replaces the result with the last error payload.

## What This Changes

The PR adds `finalAssistantVisibleText` as an intermediate fallback for cron isolated-agent summary and output text in `resolveCronPayloadOutcome`.

## Best Possible Solution

Repair the cron outcome boundary so final assistant-visible text wins only when a trailing tool error is proven recovered or self-corrected, while unresolved fatal tool, denial, and run-level failures continue to deliver as errors, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is `resolveCronPayloadOutcome` with only `isError: true` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; current main already pins that shape to return the raw error.

## Solution Assessment

No. This PR is not the best fix because the new fallback is placed after helpers that already return error text and before a fatal-error override that still replaces the result; the safer solution is to classify recovered versus unresolved errors before choosing fatal delivery text.
