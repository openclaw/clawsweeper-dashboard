## Summary

Keep open. The branch targets the right cron outcome surface, but it does not fix the linked bug because unresolved all-error payloads are still classified as fatal and then replace the final assistant text with the last error payload.

## What This Changes

The PR changes `resolveCronPayloadOutcome` so `finalAssistantVisibleText` is considered as a fallback summary/output text before falling back to payload text.

## Best Possible Solution

Repair the cron outcome boundary so final assistant-visible text wins only for a proven recovered or self-corrected mutating-tool failure, while unresolved fatal tool errors continue to report failure, and cover both paths with focused cron helper and embedded-runner regressions.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is `resolveCronPayloadOutcome` with only `isError: true` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; current main already pins that shape to return the raw error.

## Solution Assessment

No. The proposed fallback change is not enough because fatal structured-error precedence still runs afterward and overrides the result; the safer fix is to classify recovered versus unresolved mutating failures before selecting fatal delivery text.
