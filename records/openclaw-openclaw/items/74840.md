## Summary

Keep open. The PR targets a real cron summary bug, but the proposed fallback does not fix the reported path because current payload helpers still choose error text first and the fatal-error override still replaces the final assistant text.

## What This Changes

The PR adds `finalAssistantVisibleText` as an intermediate fallback summary/output source in `resolveCronPayloadOutcome`.

## Best Possible Solution

Repair the cron outcome boundary so final assistant-visible text wins only for recovered or self-corrected mutating-tool failures, while unresolved fatal tool errors continue to report failure.

## Reproduction Assessment

Yes, source-reproducible. `resolveCronPayloadOutcome` with only `isError` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true` still resolves to the last error on current main, and the helper test pins that behavior.

## Solution Assessment

No. The proposed fallback is too late in the selection flow; the safer fix is to classify recovered versus unresolved mutating-tool failures before selecting fatal delivery text.
