## Summary

Keep open. This is a valid cron bug-fix direction, but the patch is still too late in the outcome selection flow: all-error payloads with final assistant-visible text still resolve to the last error through the fatal delivery override.

## What This Changes

The PR adds `finalAssistantVisibleText` as an intermediate fallback source for cron outcome summary and output text selection.

## Best Possible Solution

Repair the cron outcome boundary so recovered or self-corrected runs can use final assistant-visible text, while unresolved fatal tool/run errors still report failure.

## Reproduction Assessment

Yes, source-reproducible. On current main, `resolveCronPayloadOutcome` with only `isError` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true` still returns the last error, and the helper test pins that behavior.

## Solution Assessment

No. The proposed fallback is not the best fix because it runs after payload fallbacks and before an unchanged fatal-error override; the safer fix is to classify recovered versus unresolved failures before fatal delivery text is selected.
