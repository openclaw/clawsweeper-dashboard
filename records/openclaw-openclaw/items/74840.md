## What This Changes

This PR changes `resolveCronPayloadOutcome` so `finalAssistantVisibleText` can be considered as a fallback summary and output text before falling back to the first payload text.

## Best Possible Solution

Repair the cron outcome boundary so final assistant-visible text wins only for a proven recovered or self-corrected mutating-tool failure, while unresolved fatal tool errors continue to report failure, and cover both paths with focused cron helper and embedded-runner regression tests.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is `resolveCronPayloadOutcome` with only `isError: true` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; current main's helper test already pins that shape to return the raw error.

## Solution Assessment

No. This PR is not the best fix because it adjusts fallback text selection, but the reported path is governed by fatal structured-error precedence; the safer fix is to classify recovered versus unresolved mutating failures before choosing fatal delivery text.
