## Summary

Keep open. This PR targets a real cron summary bug, but the diff still does not fix the all-error self-correction path and the contributor has not supplied after-fix real behavior proof.

## What This Changes

The PR adds `finalAssistantVisibleText` as an intermediate fallback for cron summary and output text selection in `resolveCronPayloadOutcome`.

## Best Possible Solution

Repair `resolveCronPayloadOutcome` so trusted recovered final assistant text can win for self-corrected all-error tool payload summaries while real run-level and unresolved fatal failures remain errors, with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only `isError` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; the current helper test pins the raw error result.

## Solution Assessment

No. The PR adds final assistant text too late in the fallback flow, so all-error payloads still select the error and the fatal delivery override replaces the result; the safer fix is to classify recovered versus unresolved fatal payloads before the override.
