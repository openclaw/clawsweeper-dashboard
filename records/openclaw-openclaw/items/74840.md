## Summary

Keep open. The PR is a useful narrow attempt for the linked issue #74807, but the diff only changes fallback text selection and leaves the fatal structured-error override in place, so the reported recovered assistant reply can still lose to the tool-error payload; the user-facing fix also needs a changelog entry.

## What This Changes

The PR adds `finalAssistantVisibleText` as an intermediate fallback for cron payload summary/output selection in `resolveCronPayloadOutcome`.

## Best Possible Solution

Repair the cron outcome boundary so final assistant-visible text wins only after a mutating-tool failure is proven recovered or explicitly self-corrected, while unresolved fatal errors continue to report failure.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is `resolveCronPayloadOutcome` with `finalAssistantVisibleText`, `preferFinalAssistantVisibleText: true`, and only `isError` payloads or an assistant payload followed by a mutating-tool error; current main pins that shape to return the raw error.

## Solution Assessment

No. The fallback-only patch is not the best solution because fatal structured-error precedence still overrides it; a safer fix classifies recovered versus unresolved tool failures before building delivery text.
