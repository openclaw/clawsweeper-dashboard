## Summary

Keep open. Current main still lacks the prompt cap, restored-launch fingerprinting, and SavedStateHandle-backed assistant routing from this PR, but the branch is not merge-ready because it reintroduces auto-send for externally supplied ASK_OPENCLAW prompts, conflicting with the shipped #70714 security fix.

## What This Changes

The PR adds Android assistant prompt sanitization, restored-launch fingerprint handling, SavedStateHandle-backed transient chat routing state, and unit tests for those paths.

## Best Possible Solution

Preserve the useful hardening direction but update it on current main: keep ASK_OPENCLAW prompts draft-only, apply the length cap to the draft path, persist requested destination and draft state through SavedStateHandle, pass restored launch fingerprints explicitly into assistant intent handling, and cover both draft-only capped prompts and restored launch state in Android unit tests.

## Reproduction Assessment

Yes. There is a high-confidence source/unit reproduction path: current main lacks the new sanitizer and lifecycle state helpers, while the PR head shows ASK_OPENCLAW parsing to autoSend for nonblank prompts and tests that assert that unsafe behavior.

## Solution Assessment

No. The proposed hardening is directionally right, but this branch is not the best mergeable solution because it conflicts with #70714; the safer solution is a rebase or replacement that keeps external App Action prompts as drafts while adding prompt bounds and lifecycle restoration.
