## Summary

Keep open. The branch still contains useful Android assistant lifecycle hardening that is not on current main, but it cannot merge as-is because it conflicts with the shipped draft-only security fix for exported `ASK_OPENCLAW` prompts and lacks real Android behavior proof.

## What This Changes

The PR hardens Android assistant launch handling by adding prompt capping, restored-launch fingerprinting, SavedStateHandle persistence for transient chat routing, and Android unit coverage.

## Best Possible Solution

Land a rebased or replacement patch on current main that keeps `ASK_OPENCLAW` prompts draft-only, caps sanitized draft text, persists/restores transient assistant routing, avoids the implicit restored-fingerprint instance field, and verifies the real Android launch path.

No. The hardening direction is useful, but this stale branch is not the best mergeable solution because it would undo shipped exported-intent protection; the safer path is a current-main patch that preserves draft-only app actions.
