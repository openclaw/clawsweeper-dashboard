## Summary

Keep open. The lifecycle and prompt-bounding hardening remains useful because current main still lacks it, but this PR is not merge-ready because its head treats exported `ASK_OPENCLAW` prompts as auto-send, conflicting with the shipped #70714 draft-only security fix.

## What This Changes

This PR adds Android assistant prompt trimming and capping, assistant launch fingerprint restoration, `SavedStateHandle` persistence for transient chat routing state, and unit coverage for those paths.

## Best Possible Solution

Update this PR or replace it with a narrow current-main patch that preserves `ASK_OPENCLAW` as `autoSend = false`, applies the prompt cap to the draft path, persists transient launch routing through `SavedStateHandle`, passes restored fingerprints explicitly, and covers both draft-only app actions and restoration behavior in Android unit tests.

## Reproduction Assessment

Yes. The blocking issue has a high-confidence source/unit reproduction path: PR head returns `autoSend = true` for `ASK_OPENCLAW` and adds an assertion for that, while current main and #70714 require `autoSend = false`.

## Solution Assessment

No. The hardening direction is appropriate, but this PR is not the best mergeable solution as-is because it conflicts with shipped exported-intent security behavior; the safer path is a rebase/update that keeps external prompts draft-only while adding lifecycle hardening.
