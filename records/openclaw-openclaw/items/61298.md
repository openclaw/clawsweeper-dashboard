## Summary

Keep open: the Android assistant hardening direction is still useful because current main lacks the prompt cap, restored-launch fingerprinting, and SavedStateHandle persistence, but this PR is not merge-ready because it conflicts with the shipped #70714 draft-only protection for exported App Action prompts.

## What This Changes

The PR adds Android assistant prompt trimming/capping, restored-launch fingerprinting, SavedStateHandle-backed transient chat routing state, and unit tests.

## Best Possible Solution

Rebase or replace the PR with a narrow patch that preserves draft-only `ASK_OPENCLAW`, caps sanitized draft text, persists transient launch routing through `SavedStateHandle`, passes restored fingerprints explicitly, and keeps Android unit coverage focused on those behaviors.

## Reproduction Assessment

Yes. The blocking finding is source-reproducible from the PR head: `ASK_OPENCLAW` parses with `autoSend = prompt != null`, while current main and #70714 require app-action prompts to remain drafts.

## Solution Assessment

No. The hardening direction is appropriate, but this is not the best mergeable solution as-is because it conflicts with shipped exported-intent security behavior; the safer path is a current-main rebase or replacement that keeps App Actions draft-only.
