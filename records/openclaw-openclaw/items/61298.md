## Summary

Keep this PR open but not mergeable as-is: the lifecycle hardening is still useful and not present on main, but the branch conflicts with the shipped draft-only `ASK_OPENCLAW` security fix, has a restored/fresh launch ambiguity, and lacks real Android proof.

This PR trims and caps Android assistant prompts, fingerprints restored launches, persists assistant routing state through `SavedStateHandle`, and adds assistant launch unit tests.

## Best Possible Solution

Rebase or replace the branch so `ASK_OPENCLAW` remains draft-only, sanitized draft text is capped, restored-launch handling is freshness-aware, transient routing survives recreation, and redacted real Android launch proof is provided.
