## Summary

Keep open. Current main still exposes prompt-visible skills through a model-facing SKILL.md read/follow flow, while the requested instruction isolation, role locking, per-skill memory/tool capability enforcement, and monitoring remain unresolved security design work.

## What This Changes

Add an instruction-isolation and permission boundary for untrusted SKILL.md content used by agents.

## Best Possible Solution

Define a maintainer-approved trusted/untrusted skill precedence model, add typed per-skill capability metadata, enforce it through prompt assembly and runtime policy, and cover malicious SKILL.md cases with docs and focused regression tests.

## Reproduction Assessment

Yes for the missing-control behavior: current main source shows prompt-visible skills are listed with SKILL.md locations and the model is told to read and follow one. No deterministic exploit transcript was run, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear as a final implementation plan. The requested isolation direction is valid, but maintainers should choose a typed capability and prompt-precedence contract rather than landing only delimiter or pattern-filter prompt text.
