## Summary

Keep open. Current main still exposes prompt-visible skills through a model-facing catalog and instructs the model to read and follow selected SKILL.md files, while current skill metadata and prompt assembly do not provide the requested per-skill instruction-isolation, role, memory, or tool-capability boundary.

## What This Changes

Requests an instruction-isolation and permission-boundary design for SKILL.md content so skill instructions cannot override agent role, memory, or tool policy.

## Best Possible Solution

Define a maintainer-approved trusted/untrusted precedence model for skills, add typed per-skill capability metadata, enforce it in prompt assembly and runtime policy, and cover malicious SKILL.md cases with docs and focused regression tests.

## Reproduction Assessment

Yes for the missing-control behavior: current main source and docs show prompt-visible skills are listed for model selection and read/follow use. A full exploit transcript is less deterministic because it depends on model behavior and installed skill content.

## Solution Assessment

Unclear as a direct implementation plan. The requested direction is valid, but delimiter or pattern filtering alone would be weaker than a typed capability contract with explicit prompt precedence, runtime enforcement, compatibility tests, and docs.
