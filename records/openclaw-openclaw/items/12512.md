## Summary

## Best Possible Solution

Define one versioned skill trust and capability contract, then enforce it consistently through prompt selection, SKILL.md loading, tool execution, memory access, sandbox policy, tests, and docs.

Unclear as a final implementation path: the requested direction is valid, but the best fix needs a typed trust and capability model enforced at runtime boundaries rather than prompt wording alone.
