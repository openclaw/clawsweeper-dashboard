## Summary

Keep open. Current main still lists prompt-visible skills and instructs the model to read/follow selected SKILL.md files, while metadata and prompt assembly only control eligibility/visibility and do not implement the requested isolation or capability boundary.

## What This Changes

Request adds instruction isolation and capability enforcement for SKILL.md content so skills cannot override agent role, memory, or tool policy.

## Best Possible Solution

Define a maintainer-approved trusted/untrusted skill precedence model, add typed capability metadata, enforce it in prompt assembly and runtime policy, and cover malicious SKILL.md cases with docs and focused regression tests.

## Reproduction Assessment

Yes for the missing-control behavior: current main source and docs show prompt-visible skills are listed with SKILL.md locations and the system prompt tells the model to read/follow a selected file. A full malicious-skill exploit transcript is model/content dependent, so I did not mark a deterministic runtime exploit as reproduced.

## Solution Assessment

Unclear as a direct implementation plan. The requested direction is valid, but delimiters or pattern filtering alone would be weaker than a typed capability contract with explicit prompt precedence, runtime enforcement, compatibility tests, and docs.
