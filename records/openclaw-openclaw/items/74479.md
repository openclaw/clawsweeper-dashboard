## Summary

Keep open. Current main still lets malformed DuckDuckGo numeric HTML entities reach `String.fromCodePoint`, which can throw and abort result parsing, while this PR is a narrow plugin-local fix with focused regression coverage and no blocking review findings.

## What This Changes

The PR adds a DuckDuckGo numeric HTML entity guard and tests so invalid decimal and hex entities are preserved while valid numeric entities still decode.

## Best Possible Solution

Land the plugin-local parser guard after maintainer review and CI, keeping malformed numeric entities preserved, valid numeric entities decoded, and core/plugin contracts, provider config, cache, and network behavior unchanged.

## Reproduction Assessment

Yes. Current main shows the DuckDuckGo decoder passing `&#9999999999;` and `&#x110000;`-style values into `String.fromCodePoint`, and a Node check confirms those values throw `RangeError`.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable solution: guard numeric entity decoding inside the DuckDuckGo parser and add unit coverage at the same plugin boundary.
