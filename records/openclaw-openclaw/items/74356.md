## What This Changes

The issue asks OpenClaw to stop installing Bun in the root Dockerfile via an unverified `curl | bash` installer and instead use a pinned or verified Bun source aligned with CI.

## Best Possible Solution

Adopt a maintainer-approved pinned or verified Bun install path in the root Dockerfile, update the Docker docs and update policy in the same change, then close this issue through PR #74359 or an equivalent merged fix.

Yes. Static inspection of current main is enough: `Dockerfile:47` still pipes `https://bun.sh/install` to `bash`, while `Dockerfile:15-16` digest-pin the Node base images.

  - body: The root Dockerfile fetches `https://bun.sh/install` and pipes it to `bash` in the build stage, so a compromised upstream script or delivery path could execute code in the build container and affect the produced image.
