## Summary

Keep open. Current main still executes the Bun installer from `bun.sh` during Docker builds despite digest-pinned Node base images, and PR #74359 is open with closing syntax for this issue, so closing before that security-sensitive fix merges would be premature.

## What This Changes

The issue requests replacing the root Dockerfile’s unverified Bun `curl | bash` installer with a pinned or verified Bun source consistent with the repository’s Docker image pinning policy.

## Best Possible Solution

The desired end state is a maintainer-approved Docker build path that sources Bun from a pinned or verified artifact, aligns Docker docs and CI Bun version policy, and extends digest/update coverage so future Bun updates are explicit.

## Reproduction Assessment

Yes. Static inspection of current main is enough: `Dockerfile:47` still pipes `https://bun.sh/install` to `bash`, while the same Dockerfile pins the Node base images to SHA256 digests.

## Solution Assessment

Yes for the requested direction, but it is not implemented on main. A pinned or verified Bun source is the narrow maintainable fix; the exact source, digest, and update policy should be settled in PR #74359 or an equivalent maintainer-reviewed change.
