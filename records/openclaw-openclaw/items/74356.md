## Security Review

Status: needs_attention

Summary: Current main still has a Docker build supply-chain gap because it executes an unverified remote Bun installer during image builds.

Concerns:

- **[high] Unverified remote installer executes during Docker build:** `Dockerfile:47`
  - body: The root Dockerfile fetches `https://bun.sh/install` and pipes it to `bash` in the build stage, so a compromised upstream script or delivery path could execute code in the build container and affect the produced image.
  - confidence: 0.97
- **[medium] Docker docs repeat the curl-pipe Bun installer pattern:** `docs/install/docker.md:352`
  - body: The public Docker faster-rebuild example teaches the same unverified Bun installer pattern, so users may copy the unsafe build guidance even after the root Dockerfile is fixed.
  - confidence: 0.9
