## What This Changes

The issue asks to replace the root Dockerfile's unverified curl-pipe Bun install with a pinned or verified Bun source aligned with the repository's CI Bun version.

## Best Possible Solution

Adopt a maintainer-approved pinned or verified Bun source in the root Dockerfile, update Docker docs and update policy in the same change, and validate the Docker build path before merge.

## Security Review

Status: needs_attention

Summary: Current main still has a Docker build supply-chain exposure because Bun is installed by executing an unverified remote installer script.

Concerns:

- **[high] Unverified remote installer executes during Docker build:** `Dockerfile:47`
  - body: The root Dockerfile fetches https://bun.sh/install and pipes it to bash in the build stage, so a compromised upstream script or delivery path could execute code in the build container and affect the produced image.
  - confidence: 0.97
- **[medium] Docker docs repeat the curl-pipe installer pattern:** `docs/install/docker.md:352`
  - body: The public faster-rebuild example teaches the same unverified Bun installer pattern, so fixing only the root Dockerfile would leave unsafe copyable guidance for users.
  - confidence: 0.9
