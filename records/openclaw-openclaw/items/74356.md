## What This Changes

The issue requests hardening the root Dockerfile Bun installation by replacing the unverified `curl | bash` installer with a pinned or verified Bun source aligned with the CI Bun version.

## Best Possible Solution

Review and land #74359 or an equivalent maintainer-approved Docker hardening change that removes build-time execution of the remote Bun installer, pins or verifies the Bun source, updates the Docker docs and coverage, and validates the root Dockerfile build path.
