## Summary

Keep open. Current main still executes the Bun installer from `bun.sh` during Docker builds, and open PR #74359 is already paired with this issue by closing syntax, so this should remain open for maintainer security review rather than cleanup closure.

## What This Changes

The issue requests replacing the root Dockerfile's unverified Bun `curl | bash` install with a pinned or verified Bun source aligned with the CI Bun version.

## Best Possible Solution

Adopt a maintainer-approved Docker Bun install path that avoids executing an unverified remote installer, pins or verifies the Bun source, updates the Docker docs and update policy in the same change, and validates the root Dockerfile build path before merge.

## Reproduction Assessment

Yes. Static inspection of current main is enough: `Dockerfile:47` still pipes `https://bun.sh/install` to `bash`, while the same Dockerfile pins the Node base images to SHA256 digests.

## Solution Assessment

Partly. Replacing `curl | bash` with a pinned or verified Bun source is the right narrow direction, but the complete solution should also settle digest/update coverage and remove the repeated docs example.
