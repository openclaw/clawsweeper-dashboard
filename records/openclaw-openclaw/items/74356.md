## Summary

Keep open. Current main still executes the Bun installer from `bun.sh` during Docker builds while the same Dockerfile pins Node base images by digest, and PR #74359 is still an open implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to replace the Dockerfile's curl-piped Bun installer with a pinned or verified Bun source aligned with the CI Bun version.

## Best Possible Solution

Adopt a maintainer-approved pinned or verified Bun source in the root Dockerfile, update Docker docs and update coverage, and stop teaching or executing unverified installer scripts in Docker build paths.

## Reproduction Assessment

Yes. Static inspection of current main is enough: `Dockerfile:47` still pipes `https://bun.sh/install` to `bash`, while the same file pins Node base images to SHA256 digests.

## Solution Assessment

Yes for the requested direction, but it is not implemented on main. A pinned or verified Bun source is the narrow maintainable fix; PR #74359 should be reviewed, landed, or replaced with an equivalent maintainer-approved change.
