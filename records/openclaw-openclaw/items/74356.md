## Summary

Keep open. Current main still contains the reported Docker build supply-chain gap, the public Docker docs repeat the same Bun installer pattern, and the already-linked PR #74359 remains the active implementation candidate with closing syntax rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to replace the root Dockerfile's unpinned `curl | bash` Bun bootstrap with a pinned or verified Bun source aligned with the Bun version used in CI.

## Best Possible Solution

Land PR #74359 or an equivalent reviewed Docker hardening change that removes build-time execution of the remote Bun installer, pins or verifies the Bun source, updates the Docker docs and update policy, and validates the root Dockerfile build path before closing this issue.

## Reproduction Assessment

Yes. Static inspection of current main is enough to reproduce the reported condition: `Dockerfile:47` still pipes `https://bun.sh/install` into `bash`, while the same Dockerfile pins Node image digests and CI pins Bun to `1.3.9`.

## Solution Assessment

Yes for the requested direction, but it is not implemented on main. Replacing the remote installer with a pinned or verified Bun source is the narrow maintainable fix; the exact source, digest, and docs policy should be settled in the linked PR or an equivalent maintainer-approved change.
