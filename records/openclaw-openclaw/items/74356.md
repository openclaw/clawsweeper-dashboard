Status: needs_attention

Summary: Current main still has a Docker build supply-chain gap because it executes an unverified remote Bun installer script.

- **[high] Unverified remote installer executes during Docker build:** `Dockerfile:47`
  - body: The root Dockerfile fetches `https://bun.sh/install` and pipes it to `bash` in the build stage, so a compromised upstream script or delivery path could execute code in the build container and influence the produced image.
