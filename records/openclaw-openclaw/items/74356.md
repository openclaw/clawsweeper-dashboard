## What This Changes

The issue asks to replace the root Dockerfile’s unverified Bun curl-pipe installer with a pinned or verified Bun source aligned with the Bun version used by CI.

## Best Possible Solution

Review and land PR #74359 or an equivalent maintainer-approved Docker hardening change that removes build-time execution of the remote Bun installer, pins or verifies the Bun source, updates the Docker docs and digest/update coverage, and validates the root Dockerfile build path.

## Reproduction Assessment

Yes. Static inspection of current main is enough: `Dockerfile:47` still pipes `https://bun.sh/install` to `bash`, while the same Dockerfile digest-pins Node base images and CI separately pins the Bun version.

## Solution Assessment
