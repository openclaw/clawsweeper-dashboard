## Summary

Keep open. Current main still leaves the reported stale session artifact set without a proactive startup/hourly sweep, and the open closing PR remains unmerged and conflicting rather than being implementation proof on main.

## What This Changes

The issue asks OpenClaw to stop accumulating stale `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` files under agent session directories.

## Best Possible Solution

Land or replace the linked cleanup PR with a focused implementation that covers configured session-store roots and stale archive/temp artifacts while preserving or explicitly updating maintenance-mode semantics.

## Reproduction Assessment

Yes, at source level. Current main has no gateway startup/hourly session artifact sweep, and the existing cleanup paths leave the full `.deleted.*`, `.reset.*`, orphan `.tmp`, and `.bak.*` set uncollected for default warn-mode users.

## Solution Assessment

Unclear for the exact policy, but the cleanup direction is valid. The maintainable solution should reuse the session maintenance/artifact helpers and either respect `mode: "enforce"` or include an explicit maintainer-approved docs/product change for warn-mode deletion.
