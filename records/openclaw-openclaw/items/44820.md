## Summary

Keep open. Current main still only cleans some session artifacts through save-path, CLI, or disk-budget paths, while the reported normal-usage accumulation of stale `.deleted.*`, `.reset.*`, `.tmp`, and `.bak.*` files is not proactively swept during gateway uptime; an open closing PR already tracks the main implementation path.

## What This Changes

The issue reports that stale session artifact files accumulate on disk instead of being automatically cleaned up during normal OpenClaw use.

## Best Possible Solution

Review, narrow, or replace the open cleanup PR with a focused implementation that covers configured session-store roots and safely handles stale `.deleted.*`, `.reset.*`, orphan `.tmp`, and excess `.bak.*` artifacts while preserving or explicitly updating the warn-mode contract.

## Reproduction Assessment

Yes, at source level. Current main lacks a gateway startup/hourly session artifact sweep, and the existing cleanup paths leave `.tmp` and ordinary `.bak.*` accumulation outside configured disk-budget or manual cleanup paths.

## Solution Assessment

Unclear for the exact policy, but the direction is right. A proactive cleanup is maintainable if it covers configured store roots and either respects `mode: "warn"` or lands with explicit docs/product approval for changing that behavior.
