## Summary

Keep open: current main and v2026.6.8 have meaningful partial cleanup for deleted/reset archives and stale store temps, but the requested automatic cleanup for every reported artifact class is still not complete, especially `.bak.*` archives and a general startup/periodic sweep.

## What This Changes

The issue asks OpenClaw to automatically clean stale `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` session artifacts so normal use does not cause disk bloat.

## Best Possible Solution

Keep this as the canonical cleanup tracker until maintainers land or replace the active cleanup PRs with one core session-maintenance path that safely handles stale deleted, reset, temp, and backup artifacts under the documented retention contract.

## Reproduction Assessment

Yes at source level, but not with a fresh multi-day live run here. Current main tests and cleanup code show `.bak.*` archives remain outside normal maintenance and no general gateway startup/hourly sweep covers every reported artifact class.

## Solution Assessment

Unclear. Current implementation is a partial fix; the best solution is likely a single canonical cleanup contract used by normal maintenance and explicit cleanup, with maintainer-approved deletion policy for `.bak.*` and background cadence.
