## Label Justifications

- `impact:session-state`: The issue concerns retained session transcript archives, backups, and atomic session-store artifacts.

## Summary

Current main substantially reduces the reported bloat: enforced store writes clean expired `.deleted.*` and `.reset.*` archives, stale atomic-write temps and all archive classes are eligible for disk-budget cleanup, and cron sessions have a reaper. The issue should remain open because idle non-cron gateways still have no general lifecycle sweep, `.bak.*` retention is only enforced under an optional disk budget, and the open closing PR is conflicting and has not demonstrated the reported real accumulated-directory case.

## What This Changes

Automatically remove stale `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` session artifacts so normal long-running use does not accumulate disk bloat.

## Best Possible Solution

Extend the canonical session-maintenance owner with a bounded startup/periodic pass for all unambiguously stale generated artifacts, reuse the existing classifiers and retention settings, and prove active transcripts plus fresh atomic-write temps survive in both idle and concurrent-write scenarios.

## Reproduction Assessment

Yes at source level: current main has no general gateway lifecycle sweep, and `.bak.*`/temp removal is conditional on disk-budget enforcement; a high-confidence live reproduction of multi-day growth on current main is still missing.

## Solution Assessment

Unclear. Proactive lifecycle cleanup is the right direction, but the open PR’s separate gateway timer should be reconciled with canonical session maintenance and backed by real idle/concurrent behavior proof before it is treated as the best fix.
