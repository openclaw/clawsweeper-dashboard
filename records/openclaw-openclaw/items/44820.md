## Summary

Keep this open. Current main has meaningful partial cleanup, but it still lacks a general gateway startup or periodic lifecycle sweep for the full reported artifact set, and the open linked PR that explicitly targets this issue is still unmerged and conflicting.

## What This Changes

The issue asks OpenClaw to automatically clean stale `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` session artifacts so normal use does not bloat agent session directories.

## Best Possible Solution

Extend the canonical session-maintenance owner with one bounded startup or periodic cleanup path for unambiguously stale generated artifacts, reusing existing classifiers and retention settings with tests for active transcripts and fresh temp-file survival.

## Reproduction Assessment

Yes at source level, but not as a live multi-day reproduction. Current main has no general gateway lifecycle sweep for all reported stale artifacts, while some cleanup only runs on save-triggered maintenance, CLI cleanup, or optional disk-budget paths.

## Solution Assessment

Unclear. Proactive cleanup is directionally right, but maintainers still need to decide the lifecycle owner and either review, fix, or close the open linked PR before a replacement repair is queued.
