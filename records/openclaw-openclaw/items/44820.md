## Summary

Keep open. Current main has manual and write-triggered cleanup paths, but it still does not proactively sweep the full reported `.deleted.*`, `.reset.*`, orphan `.tmp`, and `.bak.*` set at gateway startup or on an idle timer, and https://github.com/openclaw/openclaw/pull/47635 remains an open closing implementation candidate.

## What This Changes

The issue requests automatic cleanup of stale session artifact files that accumulate under agent session directories.

## Best Possible Solution

Land or replace the linked cleanup PR with a focused implementation that sweeps configured session-store roots for stale `.deleted.*`, `.reset.*`, orphan `.tmp`, and `.bak.*` artifacts while preserving or explicitly updating warn/enforce and recovery semantics.

Yes, at source level. Current main lacks a gateway startup/hourly session artifact sweep, and existing cleanup paths do not proactively remove the full reported `.deleted.*`, `.reset.*`, orphan `.tmp`, and `.bak.*` set.

## Solution Assessment

Unclear for the exact policy, but the cleanup direction is valid. The maintainable fix should reuse existing session maintenance/artifact helpers over configured store roots and either honor warn mode as non-mutating or update that product contract explicitly.
