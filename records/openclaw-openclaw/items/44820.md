## Summary

Keep open: current main and v2026.6.10 only partially address the disk-bloat report. Deleted/reset archive retention exists, stale store temps and unreferenced primary/trajectory/checkpoint artifacts have cleanup paths, but `.bak.*` archives are still explicitly preserved in current tests and broader automatic startup/timer cleanup remains owned by open cleanup work and maintainer direction.

## What This Changes

The issue requests automatic cleanup for stale session `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` files that accumulate under agent session directories.

## Best Possible Solution

Land or close the targeted deleted/reset archive cleanup PR first, then make a maintainer decision on whether `.bak.*` purging and proactive startup/timer sweeps belong in core session maintenance.

## Reproduction Assessment

Yes at source level, but not by a live filesystem run in this read-only review. Current tests and code show `.bak.*` archives are preserved while deleted/reset cleanup is only partial, and docs confirm startup reads do not run full pruning.

## Solution Assessment

Unclear for the full issue: the narrow deleted/reset cleanup reporting fix should reuse existing archive-retention helpers, but broad `.bak.*` deletion and proactive startup/timer cleanup need maintainer product direction.
