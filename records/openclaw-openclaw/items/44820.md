## Summary

Keep open. Current main has write-path and CLI cleanup for some session artifacts, but it still does not proactively clean the full reported `.deleted.*`, `.reset.*`, `.tmp`, and `.bak.*` set during normal gateway uptime, and an open closing PR already tracks the main fix path.

## What This Changes

The issue requests automatic cleanup of stale session `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` artifacts that accumulate under agent session directories.

## Best Possible Solution

Land, narrow, or replace the open cleanup PR with a focused implementation that respects session maintenance mode, covers configured store roots, and safely handles stale `.deleted.*`, `.reset.*`, orphaned `.tmp`, and excess `.bak.*` artifacts.

## Reproduction Assessment

Yes, at source level. Current main has CLI/write-path cleanup for some artifacts, but no gateway startup/hourly sweep, no `.tmp` cleanup in the session cleanup paths, and current tests preserve old `.bak.*` artifacts.

## Solution Assessment
