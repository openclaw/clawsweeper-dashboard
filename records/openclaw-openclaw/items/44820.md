## Summary

Keep open. Current main handles part of the bloat on enforced session-store writes and manual cleanup, but the central report remains because idle gateways still lack a general session-artifact lifecycle sweep, `.bak.*` and stale temp cleanup are conditional, and an open conflicting PR explicitly claims the fix path.

## What This Changes

The issue asks OpenClaw to automatically clean stale `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` session artifacts that accumulate under agent session directories.

## Best Possible Solution

Extend the canonical session-maintenance owner with a bounded startup/periodic pass for unambiguously stale generated artifacts, reuse existing classifiers and retention settings, and prove active transcripts plus fresh atomic-write temps survive idle and concurrent-write cases.

## Reproduction Assessment

No live high-confidence reproduction was established in this review. Source inspection gives a credible current-main path: idle gateways have no general session-artifact sweep, and `.bak.*`/temp cleanup is conditional on budget or manual cleanup.

## Solution Assessment

Unclear. Proactive lifecycle cleanup is the right direction, but the open PR’s separate gateway timer should be reconciled with canonical session maintenance and backed by idle/concurrent real behavior proof before it is treated as the best fix.
