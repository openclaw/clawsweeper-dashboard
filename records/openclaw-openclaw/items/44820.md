## Summary

Keep open. Current main still lacks a proactive gateway startup/hourly sweep for the full stale session artifact set, and an open closing PR already tracks the implementation path but remains unmerged and conflicting.

## What This Changes

The issue requests automatic cleanup of stale session artifacts such as `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` files that accumulate under agent session directories.

## Best Possible Solution

Review, narrow, or replace the open cleanup PR so it safely covers configured session-store roots and stale `.deleted.*`, `.reset.*`, orphan `.tmp`, and excess `.bak.*` artifacts while preserving or explicitly updating warn/enforce semantics.

## Reproduction Assessment

Yes, at source level. Current main has no gateway startup/hourly session artifact sweep, and existing cleanup paths still leave orphan `.tmp` and ordinary `.bak.*` accumulation outside manual/enforce or optional disk-budget paths.

## Solution Assessment

Unclear for the exact policy, but the direction is right. A proactive cleanup is maintainable if it covers configured store roots and either respects `mode: "enforce"` or lands with an explicit maintainer-approved behavior/docs change.
