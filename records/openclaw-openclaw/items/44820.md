## Summary

Keep open: current main still lacks proactive cleanup for orphaned session `.tmp` files and routine `.bak.*` rotation trimming, while `.deleted.*` and `.reset.*` cleanup is only wired through session-store maintenance writes. A linked open PR, #47635, specifically targets this issue, so the issue should remain open until that implementation is reviewed, landed, or replaced.

## What This Changes

Request automatic cleanup of stale session archive artifacts such as `.deleted.*`, `.reset.*`, `.tmp`, and `.bak.*` under agent session directories to prevent disk bloat.

## Best Possible Solution

Land a narrow gateway/session maintenance cleanup that runs at startup and on a guarded timer, reuses existing retention settings for archives, age-gates orphan `.tmp` deletion, trims `.bak.*` safely, and preserves active transcripts.

## Reproduction Assessment

Yes, source-reproducible: current main creates session-adjacent `.tmp` files and recognizes `.bak.*` artifacts, but normal cleanup paths do not proactively remove orphan `.tmp` files or trim `.bak.*` files. I did not run a live several-day gateway reproduction in this read-only review.

## Solution Assessment

Yes, the maintainable direction is a narrow proactive cleanup in gateway/session maintenance, but this issue should not be closed before the linked open PR lands or is replaced by an equivalent focused fix.
