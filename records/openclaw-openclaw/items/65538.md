## Summary

Keep open. Current main and the latest release still render the Control UI chat transcript as a polite live log while streamed text is appended inside it, and open PR #65575 remains the focused unmerged implementation candidate.

## What This Changes

Request to stop Control UI screen readers from announcing every streamed assistant token by suppressing chat transcript live-region updates while a response is sending or streaming.

## Best Possible Solution

Land #65575 or an equivalent narrow UI fix that turns the chat transcript live region off while sending or streaming, restores polite idle behavior, and keeps focused DOM tests.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: `renderChat` exposes `.chat-thread` as a polite live log, and `buildChatItems` appends stream items inside it whenever `props.stream !== null`.

## Solution Assessment

Yes. The #65575 approach of toggling the existing chat transcript live region from `polite` to `off` only while busy is the narrowest maintainable fix for this defect.
