## Summary

Keep open. Current main and the latest release still render the Control UI chat transcript as a polite live log while streaming assistant text is appended inside it, and open PR #65575 remains the focused closing implementation candidate rather than a merged fix.

## What This Changes

The issue reports that Control UI chat streaming causes screen readers to announce each incremental assistant-token update because the transcript is a polite live region.

## Best Possible Solution

Land #65575 or an equivalent narrow Control UI fix that quiets the chat transcript live region while sending or streaming, restores the intended idle announcement behavior after the final message is rendered, and adds focused DOM tests for idle, sending, streaming, and final states.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: current `renderChat` exposes `.chat-thread` as a polite live log, and `buildChatItems` appends streaming text inside that log whenever `props.stream !== null`, matching the reported screen-reader announcement trigger.

## Solution Assessment

Yes. The narrow UI-only live-region change described by #65575 is the right maintainable scope; the current main implementation is not sufficient because it leaves the transcript polite during streaming.
