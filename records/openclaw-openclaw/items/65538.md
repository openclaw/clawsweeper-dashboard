## Summary

Keep open. Current main still renders the Control UI chat transcript as a polite live log while streaming text is appended inside it, and open PR #65575 remains the focused unmerged fix candidate.

## What This Changes

The issue requests that Control UI chat stop announcing streaming assistant tokens to screen readers and announce only the completed response.

## Best Possible Solution

Land #65575 or an equivalent narrow UI fix that disables the chat transcript live region while sending or streaming, restores polite idle behavior afterward, and keeps focused DOM tests.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: current `renderChat` exposes `.chat-thread` as a polite live log, and streaming text is appended and rendered inside it whenever `props.stream !== null`.

## Solution Assessment
