## Summary

Keep open. Current main and the latest release still render the Control UI chat transcript as a polite live log while streaming text is appended inside it, and open PR #65575 remains the focused unmerged implementation candidate.

## What This Changes

The issue asks the Control UI chat transcript to stop announcing intermediate streaming response updates to screen readers and announce only the completed assistant response.

## Best Possible Solution

Land #65575 or an equivalent narrow UI fix that disables the chat transcript live region while sending or streaming, restores polite idle behavior after the final response, and keeps focused DOM tests.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: current `renderChat` exposes `.chat-thread` as a polite live log, and `buildChatItems` appends streaming text inside it whenever `props.stream !== null`.

## Solution Assessment

Yes. The focused live-region change in #65575 is the narrowest maintainable fix; broader accessibility PRs can proceed separately without replacing this targeted path.
