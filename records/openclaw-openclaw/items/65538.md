## Summary

Current main and the latest release still render the Control UI chat transcript as a polite live region, so the reported accessibility bug is not fixed. Keep this issue open as the canonical report while maintainers decide whether to refresh the linked fork PR or land an equivalent narrow fix with live screen-reader proof.

## What This Changes

The issue reports that the Control UI chat transcript announces every streamed token to screen readers because the chat thread live region remains polite during streaming.

## Best Possible Solution

Land one narrow Control UI fix that suppresses chat transcript live-region announcements while sending or streaming, restores readable final transcript behavior when idle, and includes focused DOM coverage plus live screen-reader proof when feasible.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. The source path is clear: current main keeps `.chat-thread` as a polite live region while streaming content mutates inside it, matching the reporter's NVDA/Firefox steps.

## Solution Assessment

Yes for the fix boundary: the maintainable solution belongs in the Control UI chat thread render path with focused tests. The exact accessibility behavior should be confirmed by maintainers because `aria-live="off"` versus `aria-busy` and final announcement timing are user-facing assistive-technology choices.
