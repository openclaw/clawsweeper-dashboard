Keep this open: current main and the latest release still render streamed Control UI chat updates inside a polite live log, and the remaining fix path is still pending maintainer accessibility-semantics review plus live screen-reader proof.

## What This Changes

The issue requests that Control UI streaming chat stop causing screen readers to announce every incremental token and instead preserve usable final-response announcement behavior.

## Best Possible Solution

Land one canonical Control UI live-region fix with focused DOM coverage and redacted live screen-reader/browser proof, then close this issue after the fix merges and ships.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainer review chooses the canonical accessibility pattern. The proposed aria-live toggle is narrow, but live proof should decide whether it is sufficient or whether `aria-busy` or completed-message-only transcript semantics are safer.
