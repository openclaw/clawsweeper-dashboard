## Summary

Keep open: current main still has separate Codex CLI MCP projection but the Codex app-server thread config only merges native hook config and curated plugin app config, so user-defined OpenClaw `mcp.servers` are not carried into the app-server runtime.

## What This Changes

The issue asks the Codex app-server runtime to propagate user-defined `mcp.servers` from OpenClaw config into Codex `mcp_servers`, matching the existing Codex CLI bridge.

## Best Possible Solution

Teach the bundled Codex app-server harness to project OpenClaw `mcp.servers` into the Codex thread config as `mcp_servers`, within the plugin boundary, with regression coverage for new and resumed threads.

## Reproduction Assessment

Yes, at source level: current main has Codex CLI MCP projection but the app-server path only sends native hook config plus curated plugin app config. I did not run a live app-server session in this read-only review.

## Solution Assessment

Yes, the narrow fix is to add an app-server config projection for existing OpenClaw `mcp.servers` rather than adding a new config option or startup migration. The implementation should respect the plugin boundary and avoid deep-importing core private helpers into `extensions/codex`.
