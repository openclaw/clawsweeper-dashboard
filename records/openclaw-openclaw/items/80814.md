## Summary

Keep open: current main still has the Codex CLI MCP projection but the Codex app-server path does not project user `mcp.servers` into `mcp_servers`. The open fixing PR at https://github.com/openclaw/openclaw/pull/81026 references this issue with closing syntax, so this issue should stay open until that PR, or an equivalent fix, lands.

## What This Changes

The issue asks for Codex app-server agents to project user-defined OpenClaw `mcp.servers` into Codex `mcp_servers`, matching the existing `codex-cli` behavior.

## Best Possible Solution

Review and land a corrected version of https://github.com/openclaw/openclaw/pull/81026, or an equivalent plugin-boundary fix, so Codex app-server thread start and resume both receive normalized user MCP servers.

## Reproduction Assessment

Yes, source-reproducible: current main never projects `cfg.mcp.servers` into Codex app-server `thread/start` or `thread/resume`, while the Codex CLI path already projects normalized servers as `mcp_servers`. I did not run a live app-server session.

## Solution Assessment

Yes, the narrow direction is to project the existing `mcp.servers` contract into the Codex app-server thread config within the Codex plugin boundary. Because a closing PR is already open, the issue should wait for that review and merge rather than be closed early.
