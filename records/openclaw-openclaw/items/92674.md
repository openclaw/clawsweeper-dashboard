## Summary

Keep open: current `main` and the latest release still have the reported MiniMax M3 fallback path, and no open or merged canonical fix owns the specific non-adaptive-to-adaptive escalation.

## What This Changes

The issue reports that unsupported `medium` thinking requests fall back to `adaptive` for `[off, adaptive]` model profiles, silently increasing MiniMax token use.

## Best Possible Solution

Make the shared thinking fallback cost-aware for strategy levels: non-adaptive budget requests should not resolve to `adaptive`, while explicit `adaptive` and MiniMax M3's provider default remain supported.

## Reproduction Assessment

Yes for the fallback path: current source shows MiniMax M3 exposes `off`/`adaptive`, `adaptive` ranks equal to `medium`, and cron applies the shared fallback. No live MiniMax run was executed to verify the exact token-cost multiplier.

## Solution Assessment

Unclear without maintainer policy choice: the right area is the shared thinking fallback/profile contract, but warning-only would not prevent the cost escalation. A narrow fix should avoid selecting `adaptive` for non-adaptive budget requests and add regression coverage.
