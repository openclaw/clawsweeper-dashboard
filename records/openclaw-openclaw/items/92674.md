## Summary

Keep open: current main and v2026.6.8 still have the reported MiniMax M3 adaptive-only fallback path, and the linked fix PR was closed unmerged after maintainer review found the proposed off fallback could make M3 return empty output.

## What This Changes

The issue asks OpenClaw to stop silently resolving a user-requested fixed thinking budget such as `medium` to provider-managed `adaptive` on models that advertise only `off` and `adaptive`.

## Best Possible Solution

Define a cost-safe fallback contract for adaptive-only profiles that prevents non-adaptive budget requests from silently escalating to `adaptive` while preserving MiniMax M3's required-thinking path, then cover cron/reply/CLI behavior with tests and MiniMax live or payload proof.

## Reproduction Assessment

Yes for the source path: current main has MiniMax M3 advertising only `off`/`adaptive`, ranks `adaptive` equal to `medium`, and applies the shared fallback in cron/reply paths. I did not live-run MiniMax to verify the reported token multiplier.

## Solution Assessment

No current fix is ready: the closed PR's off fallback was not the best safe solution because current MiniMax wrapper/docs evidence says disabled M3 can return empty output. The best path is a maintainer-approved provider/fallback policy backed by live or payload proof.
