## Summary

Keep open: current main and v2026.6.6 still contain the MiniMax M3 adaptive-only fallback path, and the only linked fix PR was closed unmerged after maintainer review because the obvious `off` fallback can make MiniMax M3 return empty output.

## What This Changes

The issue requests a safer thinking-level fallback so stored/default fixed-budget levels such as `medium` do not silently resolve to provider-managed `adaptive` for adaptive-only models like MiniMax M3.

## Best Possible Solution

Implement a canonical provider-aware fallback that prevents non-adaptive budget defaults from silently using adaptive while keeping MiniMax M3 on a provider-supported non-empty path, with payload or live MiniMax proof and regression tests.

## Reproduction Assessment

Yes for the source path: current main and v2026.6.6 show MiniMax M3 exposes `off|adaptive`, `adaptive` has the same rank as `medium`, and cron/session code applies the shared fallback. No live MiniMax run was performed here to reproduce the exact token-cost multiplier.

## Solution Assessment

No final fix is selected yet: the shared fallback is the right choke point, but the closed PR showed that resolving fixed-budget defaults to `off` can make MiniMax M3 produce empty output. The safer direction needs maintainer-approved fallback semantics plus MiniMax payload or live proof.
