## Summary

Keep open: current main and v2026.6.9 still contain the reported fallback path, while the only focused fix PR was closed unmerged after maintainer review because its `off` fallback could make MiniMax M3 return empty content.

## What This Changes

The issue asks OpenClaw to prevent unsupported fixed-budget thinking defaults such as `medium` from silently resolving to provider-managed `adaptive` on adaptive-only MiniMax M3 profiles.

## Best Possible Solution

Maintainers should choose a provider-safe fallback policy that prevents silent adaptive escalation without disabling MiniMax M3, then land focused resolver, payload, cron/session, docs, and live MiniMax proof.

## Reproduction Assessment

Yes for the source path: current main has MiniMax M3 as `off|adaptive`, ranks `adaptive` like `medium`, and cron/session paths apply the shared fallback. I did not run a live MiniMax token-usage reproduction in this read-only review.

## Solution Assessment

No current fix is in place. The closed candidate at https://github.com/openclaw/openclaw/pull/92803 used the right shared choke point but was not the best complete solution because `medium -> off` can break MiniMax M3 without payload/live proof.
