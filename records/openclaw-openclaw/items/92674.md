## Summary

Keep open: current main and v2026.6.6 still have the MiniMax M3 `medium` to `adaptive` fallback path, and the only linked fix PR was closed unmerged after maintainer review found its `off` fallback shape unsafe for MiniMax M3.

## What This Changes

The issue reports that unsupported fixed-budget thinking levels such as `medium` fall back to provider-managed `adaptive` for adaptive-only profiles, causing unexpectedly higher MiniMax token usage.

## Best Possible Solution

Prevent unsupported fixed-budget defaults from silently escalating to provider-managed adaptive while preserving MiniMax M3's live-required thinking behavior, with MiniMax payload or live-provider proof before landing.

## Reproduction Assessment

Yes for the source path: current main has MiniMax M3 exposing `off|adaptive`, `adaptive` ranked equal to `medium`, and cron/session paths applying the shared fallback. I did not live-run MiniMax to verify the exact token-cost multiplier.

## Solution Assessment

Unclear without a maintainer/provider decision. The shared fallback is the right choke point, but the closed related PR showed that mapping unsupported fixed budgets to `off` can hit MiniMax M3's disabled-thinking empty-output path.
