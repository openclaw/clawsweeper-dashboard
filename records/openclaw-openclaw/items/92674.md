## Summary

Keep open: current main and v2026.6.6 still select `adaptive` through the shared fallback for MiniMax M3 fixed-budget defaults, and the only linked fix PR was closed unmerged as unsafe because `off` can make M3 return empty output.

## What This Changes

The issue reports that configured `thinkingDefault: "medium"` for adaptive-only MiniMax M3 profiles is silently remapped to `adaptive`, causing higher token use.

## Best Possible Solution

Prevent silent fixed-budget-to-adaptive escalation for adaptive-only profiles while keeping MiniMax M3 on a response-producing path, with focused resolver/runtime tests and live or payload-level MiniMax proof.

## Reproduction Assessment

Yes, source-level: current main ranks `adaptive` like `medium`, MiniMax M3 advertises `off|adaptive`, and cron mutates unsupported defaults through the shared fallback. I did not live-run MiniMax to verify the exact token multiplier.

## Solution Assessment

Unclear until maintainer policy is chosen; the prior off-fallback patch was not safe because MiniMax M3 disabled thinking can return empty content. The best fix likely needs a shared fallback/runtime policy rather than a pure resolver change.
