## Summary

Keep open: current main and v2026.6.8 still have the MiniMax M3 adaptive-only fallback path, and the linked fix PR was closed unmerged after review found that resolving stored `medium` to `off` could make MiniMax M3 return empty output.

## What This Changes

The issue reports that unsupported fixed-budget thinking levels such as `medium` resolve to provider-managed `adaptive` on adaptive-only profiles, causing unexpectedly high MiniMax token use.

## Best Possible Solution

Keep this open for a maintainer-approved MiniMax/provider-thinking fix that prevents silent cost escalation without sending MiniMax M3 down the disabled-thinking empty-output path.

## Reproduction Assessment

Yes for the fallback path: current source shows MiniMax M3 exposes `off|adaptive`, `adaptive` has the same rank as `medium`, and cron/session paths call the shared fallback. No live MiniMax token-usage run was performed here.

## Solution Assessment

No closeable solution is on main. The closed candidate PR avoided `adaptive` by resolving to `off`, but maintainer review rejected that as unsafe for MiniMax M3 because disabled thinking can return empty content.
