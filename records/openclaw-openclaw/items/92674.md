## Summary

Keep open: current main and v2026.6.8 still have the reported MiniMax M3 adaptive-only fallback path, and the linked fix PR was closed unmerged as an unsafe fallback-to-off shape.

## What This Changes

The issue reports that `thinkingDefault: "medium"` on MiniMax M3 resolves to provider-managed `adaptive`, causing much higher token use than the user-requested fixed-budget level.

## Best Possible Solution

Keep this open for a maintainer-approved fallback policy that prevents silent non-adaptive budget requests from becoming provider-managed adaptive while preserving MiniMax M3's required active-thinking path.

## Reproduction Assessment

Yes for the source path: current main still ranks `adaptive` like `medium`, MiniMax M3 exposes `off|adaptive`, and cron applies the shared fallback. No live MiniMax run was performed here to verify the exact token-cost multiplier.

## Solution Assessment

No current implementation fully solves it. The closed PR's resolver-only fallback-to-off shape was rejected because it can disable MiniMax M3 output, so the best fix needs an explicit provider-thinking policy choice plus focused regression coverage.
