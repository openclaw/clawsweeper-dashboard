## Summary

Current main deliberately sends model-fallback transitions through normal reply surfaces, including suppressed-source delivery paths, so the reported recipient-facing disclosure is real but contradicts an established and shipped product contract. This should remain open for a shared visibility-policy decision rather than a channel-specific patch.

## What This Changes

Prevent model fallback and recovery banners from reaching external message recipients while preserving internal fallback telemetry.

## Best Possible Solution

Define a shared typed visibility policy at the reply-presentation boundary that preserves logs and lifecycle events while explicitly distinguishing operator/internal surfaces from external-recipient delivery across every channel.

## Reproduction Assessment

Yes at source level. Current main deterministically constructs fallback status payloads and permits them to bypass source-reply suppression, matching the reported recipient-visible behavior.

## Solution Assessment

No final implementation direction is established yet. A shared surface-aware policy is safer than deleting the notices or patching individual channels, but maintainers must choose whether notices remain visible by default, become opt-in, or vary by recipient surface.
