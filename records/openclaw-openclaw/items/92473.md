## Summary

Current main and shipped documentation intentionally deliver model-fallback transitions on the normal reply surface even with verbose mode off, so the report identifies a valid privacy/presentation concern but requests a change to an established product contract. The item should remain open for a shared visibility-policy decision rather than a channel-specific suppression patch.

## What This Changes

Change model-fallback notice visibility so external recipients do not receive provider/model routing banners by default while internal telemetry remains available.

## Best Possible Solution

Define one shared reply-presentation policy—prefer opt-in fallback banners outside verbose or operator surfaces—and verify that both transition notices remain available through logs and lifecycle events while external-recipient replies omit them by default.

## Reproduction Assessment

Yes at source level. Current main deterministically creates and delivers user-visible fallback and fallback-cleared payloads outside verbose mode, matching the reported recipient-facing behavior.

## Solution Assessment

Not yet. Shared visibility policy is safer than per-channel gating, but maintainers must decide whether external notices become opt-in, verbose-only, or surface-aware before implementation.
