## Summary

Keep open: the PR is a useful draft, but current main still does not provide the requested structured delivery observability and the branch still has merge-blocking JSON-output and hook-cancellation issues.

## What This Changes

The PR adds deliveryStatus tracking, diagnostic delivery-failure logging, --json --deliver status output, and expanded command delivery tests around deliverAgentCommandResult.

## Best Possible Solution

Land one rebased delivery observability change that preserves parseable --json output on strict failures, treats hook-cancelled sends as non-failures via outbound outcome metadata, and records the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: strict --json --deliver can throw before the PR's delayed JSON block runs, and message_sending cancellation paths return [] that the PR maps to failed delivery.

## Solution Assessment

No. The delivery observability direction is good, but this patch is not the best mergeable solution until it preserves the JSON contract, consumes cancellation metadata or waits for it, and reconciles with related delivery-status work.

  - body: The PR suppresses the existing early JSON envelope whenever --deliver is set. In strict delivery mode, deliverOutboundPayloads can throw before the later JSON block runs, so openclaw agent --json --deliver can produce no parseable JSON even though stdout is reserved for the JSON response.
