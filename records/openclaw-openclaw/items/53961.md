## Summary

Keep open: current main still lacks the structured deliveryStatus/logging surface this PR is trying to add, but the draft patch is not mergeable because it can break the documented JSON stdout contract and misclassify expected hook cancellations as delivery failures.

## What This Changes

This PR adds deliveryStatus tracking, diagnostic delivery-failure logging, --json --deliver status output, and expanded tests around deliverAgentCommandResult.

## Best Possible Solution

Land one rebased delivery observability change that preserves parseable JSON on strict delivery failures, treats hook-cancelled sends as non-failures through outbound outcome metadata, and records the user-facing change in CHANGELOG.md.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: strict --json --deliver can throw before the PR’s delayed JSON block runs, and message_sending cancellation paths return [] that the PR maps to failed delivery.

## Solution Assessment

No. The observability direction is sound, but this patch is not the best mergeable solution until it preserves the JSON contract, consumes cancellation metadata or waits for it, and reconciles with the related delivery-status work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit JSON before rethrowing strict delivery failures:** `src/agents/command/delivery.ts:325`
  - body: The PR suppresses the existing early JSON envelope whenever --deliver is set, then only writes JSON after deliverOutboundPayloads returns. In non-bestEffort mode a transport throw exits before that later block, so openclaw agent --json --deliver can produce no parseable JSON despite the documented stdout contract.
  - confidence: 0.9
- **[P2] Do not mark hook-cancelled sends as failures:** `src/agents/command/delivery.ts:426`
  - body: deliverySucceeded = results.length > 0 treats every empty result set as failed delivery, but deliverOutboundPayloads intentionally returns no result when message_sending hooks cancel all payloads. Expected cancellations would be logged and surfaced as failed delivery even though current main treats an empty no-error delivery as success.
  - confidence: 0.89
- **[P3] Add the required changelog entry:** `src/agents/command/delivery.ts:458-465`
  - body: This PR adds user-visible CLI diagnostics and a new JSON field, but the diff does not update CHANGELOG.md. OpenClaw policy requires changelog coverage for user-facing fixes before merge.
  - confidence: 0.82
