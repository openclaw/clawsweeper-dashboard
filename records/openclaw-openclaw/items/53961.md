## Summary

Keep open. Current main still lacks this PR's deliveryStatus and diagnostic logging behavior, but the draft PR is not mergeable as-is because it regresses strict --json --deliver failure output and treats expected hook cancellations as delivery failures.

## What This Changes

This PR adds deliveryStatus tracking, delivery-failure diagnostics, --json --deliver status output, and tests for deliverAgentCommandResult.

## Best Possible Solution

Land one rebased delivery observability change that preserves the single parseable --json stdout contract, consumes cancellation metadata for expected hook no-ops, and includes the required changelog entry.

## Reproduction Assessment

Yes. Code inspection gives high-confidence paths: strict --json --deliver can throw before the PR's delayed JSON block runs, and current message_sending cancellation paths return [] that this PR maps to failed delivery.

## Solution Assessment

No. The delivery observability direction is useful, but this patch is not the best mergeable solution until it fixes strict JSON error output, treats hook cancellations as non-failures, and reconciles the JSON contract with the related deliveryStatus work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit JSON before rethrowing delivery failures:** `src/agents/command/delivery.ts:330`
  - body: The PR suppresses the existing early JSON envelope whenever --deliver is set. In strict delivery mode, deliverOutboundPayloads can throw before the later JSON block runs, so openclaw agent --json --deliver can produce no parseable stdout even though the command reserves stdout for JSON output.
  - confidence: 0.9
- **[P2] Treat hook-cancelled sends as non-failures:** `src/agents/command/delivery.ts:426`
  - body: deliverySucceeded = results.length > 0 treats every empty result set as a failed send, but deliverOutboundPayloads intentionally returns [] when message_sending hooks cancel all payloads. That creates false failure telemetry/status for expected cancellations, so this should consume cancellation metadata or wait for that delivery outcome work.
  - confidence: 0.86

  - body: This PR changes user-visible CLI delivery diagnostics and JSON output, but the file list contains only delivery code and tests. OpenClaw policy requires a CHANGELOG.md entry for user-facing fix/feat/perf changes before merge.
  - confidence: 0.82
