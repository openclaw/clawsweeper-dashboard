## Summary

Keep open: this is a useful draft for delivery observability, but the current patch has blocking regressions around durable delivery cleanup, strict JSON error output, and hook-cancellation reporting.

## What This Changes

This PR adds deliveryStatus tracking, JSON envelope status, and diagnostic delivery-failure logging to deliverAgentCommandResult, with expanded delivery tests.

## Best Possible Solution

Keep the draft, but update it to preserve or migrate the deliverySucceeded cleanup contract, guarantee one parseable --json envelope on strict delivery errors, and consume DeliveryOutcome cancellation metadata once the prerequisite delivery outcome work lands.

## Reproduction Assessment

Yes, at source level: current main can return from --deliver with no structured delivery status or diagnostic when delivery produces no useful result. I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

No. The direction is useful, but this patch is not yet the best mergeable solution because it breaks durable cleanup, can drop JSON on strict delivery throws, and intentionally misclassifies hook cancellations until the related outcome metadata exists.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve the deliverySucceeded success signal:** `src/agents/command/delivery.ts:484-488`
  - body: agent-command.ts still clears pendingFinalDelivery only when deliverAgentCommandResult returns deliverySucceeded === true. This PR no longer returns that field, so successful main-session deliveries leave the pending final reply in the session store and later recovery paths can replay it. Keep the compatibility flag or update the caller and regression coverage together.
  - confidence: 0.92
- **[P2] Emit JSON before rethrowing delivery failures:** `src/agents/command/delivery.ts:433-435`
  - body: With --json --deliver, the early envelope is skipped, but strict delivery errors rethrow here before the later JSON write runs. That regresses the documented single parseable stdout contract for --json callers on transport failures.
  - confidence: 0.9
- **[P2] Do not mark hook cancellations as failed delivery:** `src/agents/command/delivery.ts:426-432`
  - body: deliverySucceeded = results.length > 0 treats all empty result sets as failed sends, but deliverOutboundPayloads intentionally returns no result when message_sending hooks cancel payloads. Wait for or consume the DeliveryOutcome cancellation metadata before surfacing this as failure telemetry.
  - confidence: 0.88
