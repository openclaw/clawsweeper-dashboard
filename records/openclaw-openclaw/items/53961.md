## Summary

Keep open. Current main still lacks the requested deliveryStatus/diagnostic tracking, and this draft PR is still a useful implementation candidate, but it is not mergeable while it can drop parseable --json output on strict delivery errors and reports intentional hook cancellations as failed delivery.

## What This Changes

This PR adds deliveryStatus tracking and diagnostic delivery-failure logging to deliverAgentCommandResult, includes that status in --json --deliver output, and expands command delivery tests.

## Best Possible Solution

Land one rebased delivery observability change that preserves parseable --json output on strict failures, consumes cancellation metadata for expected hook no-ops, and records the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Yes. Code inspection gives high-confidence paths: strict --json --deliver can throw before the PR's delayed JSON block runs, and current message_sending cancellation paths return [] that this PR maps to failed delivery.

## Solution Assessment

No. The delivery observability direction is useful, but this patch is not the best mergeable solution until it preserves the JSON contract, treats hook cancellations as non-failures, and reconciles with the related DeliveryOutcome work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit JSON before rethrowing delivery failures:** `src/agents/command/delivery.ts:330`
  - body: The PR suppresses the existing early JSON envelope whenever --deliver is set. In strict delivery mode, deliverOutboundPayloads can throw before the later JSON block runs, so openclaw agent --json --deliver can produce no parseable stdout even though the command reserves stdout for JSON output.
  - confidence: 0.9
- **[P2] Do not mark hook cancellations as failed delivery:** `src/agents/command/delivery.ts:426`
  - body: deliverySucceeded = results.length > 0 treats every empty result set as a failed send, but deliverOutboundPayloads intentionally returns [] when message_sending hooks cancel all payloads. That creates false failure telemetry/status for expected cancellations, so this should consume cancellation metadata or wait for the DeliveryOutcome work.
  - confidence: 0.86

  - body: This PR changes user-visible CLI delivery diagnostics and JSON output, but the diff contains only delivery code and tests. OpenClaw policy requires a CHANGELOG.md entry for user-facing fix/feat/perf changes before merge.
