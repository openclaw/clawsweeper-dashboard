## Summary

Keep this PR open. Current main still misses self-closing `<final/>` in the streaming and sanitizer paths, and the PR targets the right user-visible regression, but the proposed streaming regex still weakens strict final-tag suppression on malformed tag text and the user-facing fix is missing a changelog entry.

## What This Changes

The PR broadens final-tag matching in user-facing sanitization and streaming, treats self-closing final tags as closers in enforced streaming, and adds sanitizer plus streaming regression tests.

## Best Possible Solution

Land this work only after constraining or centralizing final-tag parsing so ordinary tags, spaced tags, self-closing tags, and intended attributes are stripped while malformed partial tags fail closed in strict streaming mode. Keep focused tests for sanitizer stripping, non-enforced streaming, enforced streaming, slash-containing attributes, malformed `>` inside attribute text, and the chosen persistence path, then add the required changelog entry and resolve the overlapping PR.

## Reproduction Assessment

Yes. Current main can be reproduced with a read-only regex check showing `<final/>` remains visible, and the PR regression can be reproduced by emulating the changed strict-mode scanner with `<final reason="a>b">SECRET`.

## Solution Assessment

No, not as-is. The PR changes the correct surfaces, but the safest maintainable solution should use a stricter complete-tag parser or shared helper so strict final-tag enforcement continues to fail closed on malformed tag text.
