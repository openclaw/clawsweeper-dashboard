## Summary

Keep open: current main still has the reported self-closing final-tag gap, but this PR has a blocking strict-parser regression and lacks after-fix real behavior proof.

## What This Changes

The PR updates embedded final-tag regex handling in the streaming subscriber and user-facing sanitizer to strip self-closing `<final/>` tags and adds regression tests.

## Best Possible Solution

Land one reconciled tag parser/sanitizer that handles intended bare, spaced, self-closing, and attributed final tags while failing closed on malformed or custom tag names, with focused regression coverage and redacted real setup proof.

## Reproduction Assessment

Yes, at source level. Current main leaves `<final/>` visible, and a read-only PR-style parser probe emits suppressed text for custom or malformed final-tag text; no live Gemini run was performed in this read-only review.

## Solution Assessment

No, not as-is. The PR targets the right duplicated surfaces, but the widened strict scanner should be constrained or shared so final-tag enforcement fails closed before merge.

Overall correctness: patch is incorrect
