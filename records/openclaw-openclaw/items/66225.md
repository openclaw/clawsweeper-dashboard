## Summary

Keep this PR open: it targets a real current-main leak where self-closing `<final/>` tags survive the streaming and sanitizer regexes, but the latest patch still weakens the strict final-tag suppression path by accepting malformed partial tag matches that can emit hidden text. Maintainer review should choose the canonical parser/helper shape before merge, especially because the work overlaps another open implementation.

## What This Changes

The PR broadens final-tag matching in the streaming subscriber and user-facing sanitizer, treats self-closing `<final/>` as a closer in enforcement mode, and adds sanitizer plus streaming regression tests.

## Best Possible Solution

Land this PR or a narrow replacement after tightening final-tag parsing into a shared helper that strips ordinary, self-closing, spaced, and intended attributed final tags without treating malformed partial tags as real final blocks. Keep focused coverage for sanitizer stripping, non-enforced streaming, enforced streaming, slash-containing attributes, malformed attribute text, and any chosen persistence path, then resolve the overlapping implementation PR.

## Reproduction Assessment

Yes. Current main is reproducible with the existing final-tag regex: `<final>Hello<final/>` leaves `<final/>` visible. The PR regression is also reproducible by emulating the changed strict-mode scanner with `<final reason="a>b">SECRET`, which emits `b">SECRET`.

## Solution Assessment

No, not as-is. The PR targets the right surfaces, but the safest maintainable fix should constrain or centralize final-tag parsing so strict `enforceFinalTag` mode continues to fail closed for malformed tag text.
