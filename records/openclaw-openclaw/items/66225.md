## Summary

Keep open: the underlying bug is still present on current main, but this PR is not mergeable as-is because the widened strict-mode scanner can emit trailing text from malformed final tags and the contributor has not supplied real behavior proof.

## What This Changes

The PR broadens final-tag stripping in the embedded streaming subscriber and user-facing sanitizer to handle self-closing `<final/>` tags, adds self-closing handling in enforced mode, and adds focused regression tests.

## Best Possible Solution

Land one reconciled final-tag parser/helper fix that strips bare, spaced, self-closing, and attributed final tags while failing closed on malformed tag text, with sanitizer, streaming strict/non-strict, and persistence coverage.

## Reproduction Assessment

Yes, at source level. Current main's narrow regex leaves self-closing and attributed `<final>` tags visible, and the PR's strict-mode regression is reproducible by emulating the changed loop with malformed tag text.

## Solution Assessment

No, not as-is. The PR targets the right bug, but the scanner should be constrained or centralized so malformed final-tag text does not open a final block and leak suppressed content.
