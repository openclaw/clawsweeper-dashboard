## Summary

Keep this PR open. Current main still has the self-closing `<final/>` leak the PR is trying to fix, but the proposed streaming scanner can weaken enforced final-tag suppression by accepting malformed partial tags as real final blocks; it also needs the required user-facing changelog entry before merge.

## What This Changes

The PR broadens final-tag stripping in the streaming subscriber and user-facing sanitizer, treats self-closing final tags as closers in enforced streaming mode, and adds sanitizer/streaming regression tests.

## Best Possible Solution

Land this PR or a narrow replacement after tightening final-tag parsing into a shared helper or complete-tag matcher that strips `<final/>`, `<final />`, ordinary tags, and intended attributed forms without treating malformed partial tags as valid final blocks. Keep focused coverage for sanitizer stripping, non-enforced streaming, enforced streaming, slash-containing attributes, malformed attribute text, and add the required `CHANGELOG.md` Fixes entry.

## Reproduction Assessment

Yes. Current-main regex inspection plus a read-only Node check reproduces self-closing final tags surviving the affected regexes, and the PR parser risk is reproducible by emulating the changed strict-mode scanner with `<final reason="a>b">SECRET`.

## Solution Assessment

No, not as-is. The PR targets the right bug and call sites, but the safest maintainable fix should constrain or centralize final-tag parsing so enforced mode continues to fail closed for malformed model output.
