## Summary

Keep open: current main still misses the self-closing final-tag leak, but this PR still has a strict parser regression and lacks required after-fix real behavior proof.

## What This Changes

The PR broadens embedded final-tag stripping in the streaming subscriber and user-facing sanitizer to handle self-closing `<final/>` tags and adds focused regression tests.

## Best Possible Solution

Land one reconciled final/reasoning tag parser fix that strips bare, spaced, self-closing, and attributed final tags while failing closed on malformed tag text, with streaming/sanitizer coverage and contributor real-world proof.

## Reproduction Assessment

Yes, at source level. Current main's regex leaves `<final/>` visible, and a read-only probe of the PR's strict loop returns `b\">SECRET` for malformed final-tag text instead of suppressing it.

## Solution Assessment

No, not as-is. The PR targets the right duplicated surfaces, but the widened strict scanner should be constrained or shared so malformed final-tag text fails closed.
