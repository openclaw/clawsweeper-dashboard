## Summary

Keep open: current main still has narrow final-tag matchers, and this PR targets the right bug family but is not merge-ready because the widened strict parser can misclassify malformed/custom final tags and after-fix real behavior proof is missing.

## What This Changes

The PR broadens embedded final-tag regexes in the streaming subscriber and user-facing sanitizer, adds self-closing enforcement handling, and adds focused regression tests.

## Best Possible Solution

Land or replace this with one constrained/shared final-tag parser used by streaming, sanitizer, persistence/replay, and display paths, with regression tests and redacted real setup proof.

## Reproduction Assessment

Yes, at source level. Current main's sanitizer and streaming regexes leave `<final/>` visible, and a read-only probe also shows the PR-style strict parser can emit text for malformed/custom final tags.

## Solution Assessment

No, not as-is. The PR touches the right duplicated surfaces, but the strict scanner should be constrained or shared so malformed/custom tags fail closed before merge.
