## Summary

Keep open: the PR targets a real current-main final-tag leak, but the proposed scanner still has a strict-mode parser/security regression and the contributor has not provided after-fix real behavior proof.

## What This Changes

The PR broadens final-tag stripping in the embedded streaming subscriber and user-facing sanitizer to handle self-closing variants, adds enforced-mode self-closing handling, and adds focused regression tests.

## Best Possible Solution

Land one reconciled final-tag parser/helper fix that strips bare, spaced, self-closing, and attributed final tags while failing closed on malformed tag text, with sanitizer, streaming, strict-mode, and related persistence/UI coverage.

## Reproduction Assessment

Yes, at source level. Current main's narrow regex leaves self-closing and attributed final tags visible, and a read-only probe shows the PR-style strict loop leaks trailing text after malformed final-tag syntax.

## Solution Assessment

No, not as-is. The PR targets the right surfaces, but the broadened regex should be constrained or centralized so strict final-tag enforcement fails closed instead of accepting partial malformed tags.
