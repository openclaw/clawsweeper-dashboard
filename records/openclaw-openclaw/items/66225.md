## Summary

Keep open: the bug is still real on current main and the PR targets the right agent surfaces, but it still lacks contributor-provided real behavior proof and has a blocking strict-parser finding in a content-suppression path.

## What This Changes

The PR broadens final-tag stripping in the embedded agent streaming and sanitizer paths to handle self-closing final tags and adds focused sanitizer/subscriber regression tests.

## Best Possible Solution

Land one reconciled final-tag parser/helper fix that strips bare, spaced, self-closing, and attributed final tags while failing closed on malformed tag text, with focused sanitizer, streaming, strict-mode, and replay/persistence coverage.

## Reproduction Assessment

Yes, at source level. Current main's regex leaves self-closing and attributed final tags visible, and the PR's strict-mode parser regression is reproducible with the changed loop on malformed final-tag text.

## Solution Assessment

No, not as-is. The PR targets the right surfaces, but the broadened scanner should be constrained or centralized so malformed final tags do not open a final block and leak suppressed text.
