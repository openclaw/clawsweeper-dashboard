## Summary

Keep open: current main still has the narrow final-tag matchers that miss self-closing and attributed variants, but this PR still has a blocking strict-mode parser defect and no after-fix real behavior proof. The remaining fix also overlaps https://github.com/openclaw/openclaw/pull/48593, so maintainers should pick one canonical parser path.

## What This Changes

The PR broadens final-tag stripping in the embedded agent streaming and sanitizer paths to handle self-closing `<final/>` variants, adds self-closing handling in enforced mode, and adds focused regression tests.

## Best Possible Solution

Land one reconciled final-tag parser/helper fix that strips intended bare, spaced, self-closing, and attributed final tags while failing closed on malformed tag text, with focused sanitizer, streaming, strict-mode, and persistence coverage.

## Reproduction Assessment

Yes, at source level. Current main's regex leaves self-closing and attributed final tags visible, and the PR's strict-mode parser regression is reproducible by emulating the changed loop with malformed tag text.

## Solution Assessment

No, not as-is. The PR targets the right surfaces, but the broadened regex should be constrained or centralized so malformed final tags do not open a final block and leak suppressed text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain final-tag matches to complete tag forms:** `src/agents/pi-embedded-subscribe.ts:44`

  - confidence: 0.88
