## Summary

Keep this PR open. Current main still has the narrow final-tag regexes, so the user-visible self-closing `<final/>` leak remains valid, but the proposed streaming scanner should not merge as-is because it can partially accept malformed final tags and leak trailing text in strict final-tag mode.

## What This Changes

The PR broadens final-tag stripping in the streaming subscriber and user-facing sanitizer, treats self-closing final tags as closers in enforcement mode, and adds sanitizer/streaming regression tests.

## Best Possible Solution

Land this PR or a narrow replacement after tightening final-tag parsing into a shared helper that strips `<final/>`, `<final />`, ordinary tags, and intended attributed forms without accepting malformed partial tags as real final blocks. Keep focused coverage for sanitizer stripping, non-enforced streaming, enforced streaming, slash-containing attributes, and malformed attribute text, then resolve the overlapping implementation PR.

## Reproduction Assessment

Yes. Current-main inspection plus a read-only Node regex check reproduces self-closing final tags surviving the affected regexes, and the PR parser risk is reproducible by emulating the changed strict-mode scanner with `<final reason="a>b">SECRET`.

## Solution Assessment

No, not as-is. The PR targets the right bug, but the safest maintainable fix should constrain or centralize final-tag parsing so strict mode continues to fail closed for malformed tags.
