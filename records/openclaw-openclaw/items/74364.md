## What This Changes

Request to strip `<relevant-memories>...</relevant-memories>` blocks from assistant responses before delivery to outbound messaging channels such as Telegram, Discord, Slack, and iMessage.

## Best Possible Solution

Land #74436 or a replacement that centralizes the existing code-fence-aware memory-tag stripping at shared outbound delivery boundaries, with coverage for normal final replies and confirmed streaming/block reply paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass assistant text containing a `<relevant-memories>...</relevant-memories>` block through `normalizeReplyPayload` or the block reply path on current main and observe that those paths do not call `stripRelevantMemoriesTags`.

## Solution Assessment

Yes, the best direction is to reuse the existing shared assistant-visible sanitizer at the outbound boundary. A Telegram-only regex would be too narrow for the cross-channel leak and easier to drift from Web UI behavior.
