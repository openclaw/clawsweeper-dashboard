## Summary

Keep open. Current main still misses `<relevant-memories>` stripping on outbound channel delivery paths, and #74436 is an open, unmerged closing PR rather than a shipped fix.

## What This Changes

The issue requests stripping reflected `<relevant-memories>` memory-context blocks from outbound channel replies across Telegram, Discord, Slack, iMessage, and similar messaging surfaces.

## Best Possible Solution

Land #74436 or a replacement that reuses the existing code-fence-aware memory-tag sanitizer at every outbound channel delivery boundary, including normal replies and streaming block replies.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass assistant text containing `<relevant-memories>...</relevant-memories>` through `normalizeReplyPayload` or the `emitBlockChunk` block-reply path and observe that current main does not apply `stripRelevantMemoriesTags` there.

## Solution Assessment

Yes. The maintainable fix is a central outbound sanitizer reuse, not a Telegram-only regex; the final patch should cover both normal and streaming delivery paths.
