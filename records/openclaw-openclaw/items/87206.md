- **[P1] Resolve trajectory files by run or session id:** `extensions/memory-core/src/dreaming-narrative.ts:373-375`
  - body: This recovery builds candidate trajectory filenames from the memory-core session key, but runtime sidecars are named from the transcript session id/session file or `OPENCLAW_TRAJECTORY_DIR`.   - confidence: 0.91
- **[P1] Verify trajectory identity before using assistant text:** `extensions/memory-core/src/dreaming-narrative.ts:417-430`
  - body: The loop returns `model.completed` assistant text from whichever candidate file matched without checking the event's `sessionId`, `sessionKey`, `runId`, or `workspaceDir`. A wrong match can write another session's generated text into this workspace's `DREAMS.md`.
  - confidence: 0.87

  - body: Passing `{ workspaceDir }` as the env argument to `resolveStateDir` drops `OPENCLAW_STATE_DIR` and `OPENCLAW_HOME`, so installs using a configured state directory will scan the default state tree and miss the active trajectory files.
  - confidence: 0.8
