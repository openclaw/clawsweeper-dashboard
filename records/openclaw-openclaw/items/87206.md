## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Resolve the trajectory by run or session id:** `extensions/memory-core/src/dreaming-narrative.ts:374-375`
  - body: This recovery builds paths from `successfulSessionKey`, but OpenClaw trajectory sidecars are named from the transcript session id (`<sessionId>.trajectory.jsonl`), and the reported failure returns `[]` from `getSessionMessages`, leaving no session id to derive here. As written this will normally miss the archived trajectory and fall through to the synthetic fallback text.
  - confidence: 0.91
- **[P1] Match trajectory events before using assistant text:** `extensions/memory-core/src/dreaming-narrative.ts:417-424`
  - body: When a candidate file is found, the loop returns the last `model.completed` text without checking that the event belongs to the successful session key or run id. Please require the successful run/session identifier before writing the recovered text to this workspace's DREAMS.md.
  - confidence: 0.84
- **[P2] Use the active runtime state directory:** `extensions/memory-core/src/dreaming-narrative.ts:363-364`
  - body: Passing `{ workspaceDir }` into `resolveStateDir` treats that object as an environment map and drops the real `OPENCLAW_STATE_DIR`, so configured state-dir installs will scan the default home state instead of the active runtime state. Use the existing runtime state resolver without a fake env object.
  - confidence: 0.8
