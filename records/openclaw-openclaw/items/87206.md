Full review comments:

- **[P1] Resolve the trajectory by run or session id:** `extensions/memory-core/src/dreaming-narrative.ts:374-375`
  - body: This recovery builds paths from `successfulSessionKey`, but OpenClaw trajectory sidecars are named from the transcript session id or configured trajectory dir. In the reported failure `getSessionMessages` can return `[]`, so this usually has no session id to derive and can fall through to fallback text.
  - confidence: 0.91
- **[P1] Match trajectory events before using assistant text:** `extensions/memory-core/src/dreaming-narrative.ts:417-424`
  - body: When a candidate file is found, the loop returns the last `model.completed` text without checking that the event belongs to the successful session key or run id. Require the successful run/session identifier before writing recovered text to this workspace's `DREAMS.md`.
  - confidence: 0.84
- **[P2] Use the active runtime state directory:** `extensions/memory-core/src/dreaming-narrative.ts:363-364`
  - body: Passing `{ workspaceDir }` into `resolveStateDir` treats that object as an environment map and drops the real `OPENCLAW_STATE_DIR`, so configured state-dir installs scan the default home state instead of the active runtime state.
  - confidence: 0.8
