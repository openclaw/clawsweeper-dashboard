## Summary

Keep open: current main still marks failed auth profiles before rotating in both prompt and assistant failover, and this PR's direction matches the linked bug, but the branch is dirty/stale after helper extraction and needs a rebased or replacement patch before merge.

The PR reorders embedded Pi auth-profile failover so profile rotation happens before failed-profile cooldown marking, with cooldown writes deferred asynchronously.

## Best Possible Solution
