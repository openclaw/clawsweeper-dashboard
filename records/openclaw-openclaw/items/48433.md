## Summary

Keep open: current main still lacks the requested per-host exec allowlist feature, but this external feature PR is not ready because it has blocking command-execution policy regressions and no inspectable real behavior proof.

## What This Changes

The PR adds `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist buckets, doctor migration support, docs, and tests for mixed gateway/sandbox/node exec routing.

## Best Possible Solution

Rework this on top of current `ExecTarget`/`auto` routing so per-host allowlists preserve ask, obfuscation, durable approval, sandbox execution, and node-host map semantics, then land it with redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced defects via source inspection: sandbox wildcard returns before `requiresExecApproval`, and node-host checks still read a flat allowlist that omits host-specific `node` buckets. The original request is a feature, not a current-main bug, although current tests confirm gateway-configured agents still cannot request sandbox.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it is rebased onto current exec routing and preserves existing approval policy boundaries across sandbox, gateway, and node hosts.

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve approval checks for sandbox wildcards:** `src/agents/bash-tools.exec-host-gateway.ts:95-99`
  - body: This early return exits before obfuscation detection and `requiresExecApproval`, so a sandbox request with a `"*"` allowlist runs immediately even when `ask: "always"` or an obfuscated command should prompt. Keep the host-side path-resolution workaround, but still run the approval decision path before execution.
  - confidence: 0.94
