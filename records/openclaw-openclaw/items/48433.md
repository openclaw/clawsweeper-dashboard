## Summary

Keep open: current main does not implement the requested `allowedHosts`/per-host allowlist feature, and this PR still has blocking command-exec correctness and security issues plus missing real behavior proof.

## What This Changes

The PR adds optional `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist maps, doctor migration, docs, and tests for mixed gateway/sandbox/node exec routing.

## Best Possible Solution

Rebase the feature onto current main’s `ExecTarget` routing and preserve approval, obfuscation, node-host, and `auto` semantics before maintainer review of the new policy surface.

## Reproduction Assessment

Yes for the PR-introduced defects by source inspection: the sandbox wildcard path returns before approval checks, node-host paths still read a flat allowlist, and the branch schema omits current `auto` target support. The original request is a feature rather than a current-main bug.

## Solution Assessment

No. The direction may be useful, but this patch is not the best solution until it is rebased onto current exec target routing and preserves the existing command-approval security boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve approval checks for sandbox wildcards:** `src/agents/bash-tools.exec-host-gateway.ts:95-99`
  - body: This early return runs sandbox wildcard commands before obfuscation detection and `requiresExecApproval`, so `ask: "always"` or obfuscated sandbox exec can skip explicit operator approval. Keep the wildcard handling, but still run the approval decision path before execution.
  - confidence: 0.94
- **[P2] Include node buckets in the flat node-host fallback:** `src/infra/exec-approvals.ts:589-598`
  - body: Map-form allowlists populate the legacy flat `resolved.allowlist` only from `default`, but the node-host evaluators still read that flat list. A map such as `{ "node": [...] }` with no default will miss and deny node commands that the per-host format should allow.
  - confidence: 0.88
- **[P2] Preserve current auto exec host support:** `src/config/zod-schema.agent-runtime.ts:425-426`
  - body: Current main accepts `tools.exec.host: "auto"` and uses `ExecTarget` routing, but this branch's schema only enumerates sandbox/gateway/node. Reconcile `allowedHosts` with the current model so valid current configs are not rejected after rebase.
  - confidence: 0.82
