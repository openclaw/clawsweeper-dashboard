## Summary

Keep open: current main still lacks the requested `allowedHosts` and per-host approval-map feature, while the PR head is not mergeable as-is because it introduces command-approval regressions and lacks inspectable real behavior proof.

## What This Changes

The PR adds optional `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist maps, doctor migration/docs, and gateway/sandbox/node approval plumbing with tests.

## Best Possible Solution

Rebase and rework per-host allowlists against current exec target routing so approval, obfuscation, sandbox execution, and node-host map semantics are preserved, then land with redacted real-environment proof.

## Reproduction Assessment

Yes for the PR-introduced defects via source inspection: the sandbox wildcard return precedes approval checks, and node-host checks still read a flat allowlist that omits host-specific `node` buckets. The original request is a feature, not a current-main bug.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it is rebased onto current exec routing and preserves existing approval-policy boundaries across sandbox, gateway, and node hosts.
