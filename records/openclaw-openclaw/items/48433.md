Keep open: current main does not contain this PR's `tools.exec.allowedHosts` or per-host exec approval map feature, and the PR still has source-reproducible command-execution policy regressions plus missing external real behavior proof.

## What This Changes

The PR adds `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist maps, doctor migration/docs, and sandbox/gateway/node approval plumbing.

## Best Possible Solution

If maintainers want this capability, rebase it onto current exec target/defaults and approval internals, then preserve ask, obfuscation, durable approval, and node-host map semantics with focused regression coverage.

## Reproduction Assessment

Yes, for the PR regressions. Source inspection on the PR head shows the sandbox wildcard path returns before approval checks, and node-host allowlist evaluation still consumes a flat fallback that omits `node` buckets.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the best current solution because it bypasses explicit approval policy for sandbox wildcard requests and does not fully adapt node-host allowlist resolution.
