## Summary

Keep this PR open. Current main still does not implement `tools.exec.allowedHosts` or per-host exec approval buckets, but the PR head is not mergeable because it introduces a sandbox wildcard approval bypass and leaves node-host map allowlists incompatible with the flat fallback path.

## What This Changes

The PR adds optional `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist maps, doctor migration, tests, docs, and sandbox approval plumbing for mixed gateway/sandbox/node exec routing.

## Best Possible Solution

Rebase the branch onto the current `host=auto` routing model, confirm the intended explicit multi-host semantics with a maintainer, and make per-host approval buckets preserve existing ask, allowlist, obfuscation, durable approval, and node-host guarantees with focused regression tests.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: on PR head, a sandbox bucket containing `"*"` returns before approval checks, and a map containing only `node` entries is not visible to the node path that still reads the flat allowlist.

## Solution Assessment

No. The direction may be useful, but this patch is not the best fix until it preserves approval policy and node-host compatibility; the safer path is an additive, host-aware approval contract that keeps the current routing security tests green.
