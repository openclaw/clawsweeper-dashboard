## Summary

Keep open: current main does not contain the proposed `tools.exec.allowedHosts`/per-host allowlist feature, and the PR still has blocking approval-policy regressions in the command execution path.

## What This Changes

The PR adds `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist maps, doctor migration/docs, and plumbing through gateway, sandbox, node, and embedded approval flows.

## Best Possible Solution

Rebase onto current main, keep the new permission model only if maintainers accept it, and preserve existing approval prompts plus node-host allowlist compatibility with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. The PR returns before approval checks for sandbox wildcard allowlists, and node-host evaluators still read the flat `approvals.allowlist` while the PR only flattens map `default` buckets.

## Solution Assessment

No. The direction may be useful, but this patch is not merge-ready until it preserves the approval contract, handles node-host buckets, and is reconciled with current main's exec target model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve approvals for sandbox wildcard runs:** `src/agents/bash-tools.exec-host-gateway.ts:99`
  - body: This early return exits before `requiresExecApproval(...)` and the inline-eval/obfuscation checks run. With a sandbox `"*"` bucket plus `ask: "always"` or an approval-triggering command, the request proceeds without the operator prompt that the exec policy explicitly requires. Keep the wildcard path inside the normal approval flow and only avoid host-side path resolution.
  - confidence: 0.93
- **[P2] Include node host buckets in flat fallback:** `src/infra/exec-approvals.ts:598`
  - body: Map-format agents only copy the `default` bucket into `resolved.allowlist`, but the node-host allowlist evaluators still read that flat field. A config like `{ allowlist: { node: [...] } }` with no `default` bucket is treated as empty and allowed node commands miss under `security=allowlist`; either include node buckets in the fallback or route node-host through host-aware resolution.
  - confidence: 0.89
