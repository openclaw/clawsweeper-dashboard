## Summary

Keep open. Current main covers the related broad exec-routing issue differently, but it does not implement this PR's per-host allowlist surface, and the current PR head still has source-proven command-approval regressions plus missing external real behavior proof.

## What This Changes

The PR adds `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist buckets, doctor migration, docs, and tests for mixed exec-host routing.

## Best Possible Solution

If maintainers want this capability, rebase it onto current exec target/defaults and implement per-host approval resolution without bypassing ask, obfuscation, durable approval, or node-host map semantics.

## Reproduction Assessment

Yes, for the PR-introduced regressions via source inspection. The sandbox wildcard branch returns before approval checks, and node-host consumers still read a flat allowlist that omits host-specific `node` buckets.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it is rebased and preserves the existing approval policy boundaries across gateway, sandbox, and node hosts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve approval checks for sandbox wildcards:** `src/agents/bash-tools.exec-host-gateway.ts:95-99`
  - body: This early return exits before obfuscation detection and `requiresExecApproval`, so a sandbox request with a `"*"` allowlist runs immediately even when `ask: "always"` or an obfuscated command should prompt. Keep the host-side path-resolution workaround, but still run the approval decision path before execution.
  - confidence: 0.94
- **[P2] Include node buckets in the flat allowlist fallback:** `src/infra/exec-approvals.ts:589-598`
  - body: Map-form allowlists only contribute the `default` bucket to `resolved.allowlist`, but node-host prechecks still evaluate that flat list. A config such as `{ "node": [{ "pattern": "..." }] }` with no default will deny node commands that the new per-host format says are allowed.
  - confidence: 0.88
