## Summary

Keep open. Current main does not include `tools.exec.allowedHosts` or per-host exec approval buckets, but this PR still has blocking command-execution policy defects and lacks inspectable real behavior proof.

## What This Changes

The PR adds optional per-host exec routing and per-host exec approval allowlists through `tools.exec.allowedHosts`, `exec-approvals.json` map buckets, doctor migration, docs, and tests.

## Best Possible Solution

Rework this on top of current `ExecTarget`/`auto` routing so per-host allowlists preserve ask, obfuscation, durable approval, sandbox execution, and node-host map semantics, then land it with redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced defects via source inspection: the sandbox wildcard return precedes approval checks, and node-host checks still read a flat allowlist that omits host-specific `node` buckets. The original request is a feature, not a current-main bug.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it is rebased onto current exec routing and preserves existing approval policy boundaries across sandbox, gateway, and node hosts.
