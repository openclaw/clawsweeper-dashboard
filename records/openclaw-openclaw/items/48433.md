## Summary

Keep this PR open. Current main still does not implement `tools.exec.allowedHosts` or per-host `exec-approvals.json` allowlist buckets, and the proposed patch is not landable as-is because its sandbox wildcard fast path bypasses approval checks and its node-host compatibility path can miss node-specific buckets.

## What This Changes

The PR adds an optional `tools.exec.allowedHosts` config field, per-host exec approval allowlist maps, doctor migration for legacy approval files, tests, and docs for mixed gateway/sandbox/node exec routing.

## Best Possible Solution

Rebase the branch onto current main's `host=auto` routing model, keep any explicit multi-host permission surface additive and schema-documented, and make per-host approval buckets preserve all existing ask/allowlist/obfuscation/node-host guarantees with focused regression tests.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: on PR head, configure sandbox per-host allowlist with `"*"` and `ask: "always"`; `processGatewayAllowlist` returns at lines 95-99 before the approval decision path at lines 158-173, so no approval prompt can be registered.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it is rebased onto `host=auto` and fixes the approval-bypass and node-bucket compatibility defects.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve approval checks for sandbox wildcard allowlists:** `src/agents/bash-tools.exec-host-gateway.ts:95-99`
  - body: This early return exits before `requiresExecApproval`, heredoc handling, and obfuscation checks run. With a sandbox `"*"` bucket and `ask: "always"` or an obfuscated command, the command runs immediately instead of producing the required approval prompt, which breaks the operator approval policy for sandbox exec.
  - confidence: 0.92
- **[P2] Include host-specific node buckets in the flat fallback:** `src/infra/exec-approvals.ts:589-599`
  - body: Node-host consumers still evaluate `resolved.allowlist`, but map-format agents only copy the `default` bucket into that flat list. A config such as `{ allowlist: { node: [...] } }` therefore appears empty to the node path and causes allowlist misses for commands that should be permitted by the new per-host format.
  - confidence: 0.84
