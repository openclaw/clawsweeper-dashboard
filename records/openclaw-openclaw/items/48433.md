## Summary

Keep open for maintainer review: current main does not implement the `allowedHosts` surface, but this PR is not ready as-is because the head still has command-exec approval/security regressions and lacks inspectable real behavior proof.

## What This Changes

The branch adds optional `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist maps, doctor migration, docs, and tests for routing exec across gateway, sandbox, and node hosts.

## Best Possible Solution

Rebase and rework the feature on current main so allowed host selection, per-host allowlists, approval prompts, obfuscation checks, sandbox execution, node-host compatibility, and `auto` target support preserve the existing exec security boundaries.

## Reproduction Assessment

Yes for the PR-introduced defects by source inspection: the sandbox wildcard return precedes approval checks, node-host checks still read a flat allowlist that omits host-specific `node` buckets, and the branch schema omits current `auto` host support. The original request is a feature, not a current-main bug.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it preserves approval-policy boundaries across sandbox, gateway, and node hosts, reconciles current `auto` routing, and includes real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve approval checks for sandbox wildcards:** `src/agents/bash-tools.exec-host-gateway.ts:95-99`
  - body: This early return bypasses obfuscation detection and `requiresExecApproval`, so a sandbox request with `"*"` in its allowlist runs immediately even when `ask: "always"` or an obfuscated command should prompt. Keep the sandbox wildcard behavior, but still run the approval decision path before execution.
  - confidence: 0.94
- **[P2] Include node buckets in the flat node-host fallback:** `src/infra/exec-approvals.ts:589-598`
  - body: Map-form allowlists populate the legacy flat `resolved.allowlist` only from the `default` bucket, but node-host evaluators still read that flat list. A map such as `{ "node": [...] }` with no default will miss and deny node commands that the per-host format should allow.
  - confidence: 0.88
- **[P2] Preserve current auto exec host support:** `src/config/zod-schema.agent-runtime.ts:425-426`
  - body: Current main accepts `tools.exec.host: "auto"` and tests exercise auto target selection, but this branch's schema and types only enumerate sandbox/gateway/node. Reconcile `allowedHosts` with the current `ExecTarget` model so valid current configs are not rejected after rebase.
  - confidence: 0.82
