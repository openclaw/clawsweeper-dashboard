## Summary

Keep open. Current main does not implement `tools.exec.allowedHosts`, but the latest PR head still has blocking command-exec approval defects and lacks inspectable real behavior proof for this security-sensitive feature.

## What This Changes

Adds optional `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlists, doctor migration, docs, and approval plumbing across gateway, sandbox, and node exec paths.

## Best Possible Solution

Rework the feature against current exec target routing so allowed host selection, per-host allowlists, approval prompts, obfuscation checks, sandbox execution, and node-host compatibility preserve the existing security boundaries, then land only with redacted real-environment proof and maintainer security review.

## Reproduction Assessment

Yes for the PR-introduced defects via source inspection: the sandbox wildcard return precedes approval checks, and node-host checks still read a flat allowlist that omits host-specific `node` buckets. The original request is a feature, not a current-main bug.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it preserves approval-policy boundaries across sandbox, gateway, and node hosts and includes real proof.
