## Summary

Keep open: the requested mixed-host exec/allowlist feature is not implemented on current main, but this PR is not mergeable as-is because it introduces source-reproducible exec approval policy defects and still lacks inspectable real behavior proof.

## What This Changes

The PR adds optional `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist buckets, doctor migration, docs, tests, and sandbox/gateway/node approval plumbing.

## Best Possible Solution

Rework the feature on top of current exec target routing so per-host allowlists preserve ask, obfuscation, durable approval, sandbox execution, and node-host map semantics, then land it with redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced defects via source inspection: the sandbox wildcard return precedes approval checks, and node-host checks still read a flat allowlist that omits host-specific `node` buckets. The original request is a feature, not a current-main bug.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it is rebased onto current exec routing and preserves existing approval policy boundaries across sandbox, gateway, and node hosts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve approval checks for sandbox wildcards:** `src/agents/bash-tools.exec-host-gateway.ts:95-99`
  - body: This early return exits before obfuscation detection and `requiresExecApproval`, so a sandbox request with `"*"` in its allowlist runs immediately even when `ask: "always"` or an obfuscated command should prompt. Keep the host-side path-resolution workaround, but still run the approval decision path before execution.
  - confidence: 0.94
- **[P2] Include node buckets in the flat allowlist fallback:** `src/infra/exec-approvals.ts:589-598`
  - body: Map-form allowlists only contribute the `default` bucket to `resolved.allowlist`, but node-host prechecks still evaluate that flat list. A config such as `{ "node": [{ "pattern": "..." }] }` with no default will deny node commands that the new per-host format says are allowed.
  - confidence: 0.88
