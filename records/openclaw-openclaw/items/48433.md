Keep this PR open for maintainer handling: the feature is still unique and not implemented on current main, but the current branch has command-exec policy defects, lacks inspectable real behavior proof, and predates current `tools.exec.host="auto"` routing.

## What This Changes

The PR adds optional `tools.exec.allowedHosts`, per-host `exec-approvals.json` allowlist maps, doctor migration, docs, and tests for gateway/sandbox/node exec routing.

## Best Possible Solution

Rebase the feature onto current main and rework it against current exec target routing so allowed host selection, per-host allowlists, approval prompts, obfuscation checks, sandbox execution, node-host compatibility, and `auto` host support preserve existing security boundaries.

## Reproduction Assessment

Yes for the PR-introduced defects by source inspection: the sandbox wildcard return precedes approval checks, node-host checks still read a flat allowlist that omits host-specific `node` buckets, and the branch schema omits current `auto` host support. The original request is a feature, not a current-main bug.

## Solution Assessment

No. The direction may be useful, but this patch is not the best current solution until it preserves approval-policy boundaries across sandbox, gateway, and node hosts, reconciles current `auto` routing, and includes real proof.
