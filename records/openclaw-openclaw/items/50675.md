## Summary

Close as implemented on main: the plugin-command continuation behavior has already shipped as the public `continueAgent` flag in v2026.4.27. This PR is now stale against that API shape and would regress current plugins by reading `shouldContinue` instead.

## What This Changes

The PR adds a `shouldContinue?: boolean` plugin command result flag, maps it through command handling to continue the agent, updates JSDoc, and adds command tests.

## Best Possible Solution

Keep the shipped `continueAgent` API and close this stale PR; handle the narrower undefined-handler-result hardening through the canonical bug report #74800 or a focused replacement that preserves the current public contract.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: a plugin command returning `{ continueAgent: true }` is mapped to internal `shouldContinue: true`, and the inline reply path continues to the agent.

## Solution Assessment

No. This PR is not the best way to solve the issue now because it changes the public field to `shouldContinue`; the maintainable solution is the already-shipped `continueAgent` contract plus any separate nullish-result hardening that preserves it.
