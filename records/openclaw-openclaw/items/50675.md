## Summary

Keep this PR open for maintainer handling, but do not merge it as-is. Current main already implements plugin-command agent continuation through the public `continueAgent` field, while this branch still proposes the older `shouldContinue` API; however, the branch and prior review discussion still point at useful cleanup around stale command docs/JSDoc and nullish handler-result hardening.

## What This Changes

The PR adds a `shouldContinue` plugin command result flag, wires plugin command handling to continue the agent when that flag is true, adds a nullish result guard, and updates tests/JSDoc for that behavior.

## Best Possible Solution

Keep the public API as `continueAgent`, update the stale `PluginCommandResult` and `registerCommand` docs/JSDoc to explain continuation and reply suppression, and land one canonical nullish-result hardening fix through this PR or the newer overlapping PRs.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a plugin command returning `continueAgent: true` is covered on current main, while a handler resolving `undefined` still reaches `result.continueAgent` without a guard.

## Solution Assessment

No. This PR solves the older feature shape, but current main standardized the public field as `continueAgent`; the safer path is a focused rebase/follow-up that preserves that API and ports only the remaining docs/nullish cleanup.
