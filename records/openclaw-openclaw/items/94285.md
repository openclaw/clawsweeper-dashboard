## Label Justifications

- `P3`: This is a low-risk ergonomic feature request for Discord progress visibility, not an urgent regression or core runtime outage.
- `impact:message-loss`: The issue is about progress/reasoning/tool-progress content disappearing from the Discord-visible message at finalization.
- `impact:security`: The requested durable preservation can keep reasoning, tool args, URLs, and diagnostics in channel history where current guidance treats them as sensitive debug output.

## Summary

Keep open: current main and v2026.6.8 still treat Discord progress drafts as transient finalization previews, and the requested preserve/append/send-separate behavior would add new user-facing config and security-sensitive channel history semantics that need maintainer direction.

## What This Changes

Request adding a Discord progress-streaming finalization setting that preserves the live progress draft and appends or separates the final answer instead of replacing it.

## Best Possible Solution

Choose a maintainer-approved Discord progress finalization policy, then implement an opt-in setting with safe defaults, security docs, message-limit handling, and focused Discord/shared finalizer tests.

## Reproduction Assessment

Yes, source-backed: Discord progress mode creates a draft stream and current finalization either edits that preview to final text or clears it before normal final delivery, while config/docs have no preserve/final behavior setting. I did not run a live Discord proof in this read-only review.

## Solution Assessment

Unclear as proposed: a finalBehavior enum is plausible, but maintainers need to choose whether durable history should append in place, send a separate message, or stay transient given security, message-limit, and notification semantics.
