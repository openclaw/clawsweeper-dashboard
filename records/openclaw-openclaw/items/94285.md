## Summary

Keep open: current main and v2026.6.9 still finalize Discord progress drafts by editing the preview message to the final answer, and the current config/docs expose no preserve, append, or separate-final setting. This is a valid Discord UX feature request, but it needs maintainer product/config direction before implementation.

## What This Changes

The issue requests a Discord progress-streaming finalization setting that preserves the live progress draft and appends or sends the final answer separately instead of replacing the draft.

## Best Possible Solution

Have maintainers choose the Discord-owned finalization policy first, then implement the selected default-preserving option with schema, docs, regression tests, and live Discord proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection confirms current Discord progress drafts are finalized by editing the preview message to final text, with no preserve/append/separate setting in the current schema.

## Solution Assessment

Unclear. An opt-in finalization policy is plausible, but maintainers should first choose append versus send-separate versus keeping current replacement behavior before adding config surface.
