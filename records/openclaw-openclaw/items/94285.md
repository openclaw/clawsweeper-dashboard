## Summary

Keep open. Current main intentionally edits a Discord progress draft into the final answer when safe, and neither current config nor docs expose the requested preserve, append, or send-separate finalization behavior; adjacent open issues overlap but do not supersede this exact Discord setting request.

## What This Changes

The issue requests a Discord progress-streaming option that keeps the progress draft and appends or separately sends the final assistant answer instead of replacing the draft.

## Best Possible Solution

Have maintainers decide the finalization policy first; if accepted, add a default-preserving typed final behavior with Discord implementation, schema/UI hints, docs, and focused progress-finalization tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new finalization mode. Source and docs do confirm current main lacks the requested preserve/append/send-separate option and currently replaces the draft when safe.

## Solution Assessment

Unclear. The requested behavior is plausible, but root policy sets a high bar for new config surface, so maintainers should choose between a shared progress finalization mode, a Discord-specific setting, or leaving progress drafts transient by design.
