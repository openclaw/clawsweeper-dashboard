## Summary

Keep open: current main already has Discord progress drafts, but they are intentionally finalized by handing off to the final reply rather than preserving the progress content. The requested preserve/append behavior would add a new Discord streaming finalization policy and config surface, so it needs maintainer product direction before implementation.

## What This Changes

The issue requests a Discord setting that keeps the progress draft visible at finalization and appends or sends the final assistant answer separately instead of replacing the progress message.

## Best Possible Solution

Add a maintainer-approved Discord progress finalization policy to the existing streaming progress config, schema, docs, and live-preview finalizer, defaulting to the current replace behavior.

## Reproduction Assessment

Yes, from source inspection: progress mode creates a transient Discord draft and the final delivery path finalizes or replaces it rather than retaining the progress content. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

Unclear: a preserve/append mode is plausible, but the exact public config shape and Discord finalization semantics need maintainer product approval before this is the best fix.
