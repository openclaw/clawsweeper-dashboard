## Summary

Keep open: current main and latest release still finalize Discord progress drafts by editing the preview into the final answer or clearing/falling back, and no preserve/append/separate-final setting exists. The request is valid, but it adds user-facing Discord streaming/config policy and should stay with maintainer product review rather than an automated fix lane.

## What This Changes

The issue requests a Discord setting that keeps the live progress draft visible and appends or separates the final answer instead of replacing the draft.

## Best Possible Solution

Have maintainers choose the Discord-owned finalization policy first, then implement the selected default-preserving option with schema/types/docs, regression coverage, and live Discord proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: Discord progress draft tests update the draft with tool progress and then expect the final answer to replace it via preview edit. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

Unclear. A default-off finalization policy is plausible, but maintainers should first choose whether the supported shape is append, send-separate, or a narrower existing-mode adjustment before adding config surface.
