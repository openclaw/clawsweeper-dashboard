## Summary

Keep open: current main and the latest release still finalize Discord progress drafts by editing or clearing the preview rather than preserving the progress transcript, and the request is a real but product-sensitive Discord ergonomics feature rather than an already-defined bug fix.

## What This Changes

The issue requests a Discord progress-streaming option that preserves the live progress draft and appends or sends the final answer instead of replacing the draft.

## Best Possible Solution

If maintainers want this behavior, define one canonical opt-in Discord progress finalization mode that keeps the current replace behavior as the default, then update schema, UI hints, docs, and focused Discord progress-finalization tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current behavior is source-verifiable: progress drafts are transient and final delivery replaces or clears them rather than preserving the progress transcript.

## Solution Assessment

Unclear until maintainers choose the product shape. The proposed finalBehavior enum is more complete than a one-off boolean, but any new config knob should pass the repo's high bar for channel config surface.
