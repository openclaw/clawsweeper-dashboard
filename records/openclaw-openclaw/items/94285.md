## Summary

Keep open: current main still finalizes Discord progress drafts by replacing the preview with the final answer or clearing/discarding it before normal delivery, and there is no `preserveOnFinal` or `finalBehavior` option. The request is clear and aligned with Discord ergonomics, but it adds a new channel streaming/config policy and needs maintainer product direction before implementation.

## What This Changes

The issue requests a Discord progress-streaming option to preserve the live reasoning/tool-progress draft and append or send the final answer instead of replacing it.

## Best Possible Solution

A maintainer-approved additive policy should keep current replacement behavior as the default, add one documented opt-in finalization mode if accepted, handle Discord length/chunking limits, and cover source behavior plus live Discord proof in a future PR.

## Reproduction Assessment

Not applicable as a bug reproduction, but the current behavior is source-reproducible: Discord progress drafts are finalized by editing to final text or clearing/discarding before normal delivery. No current-main path implements the requested preserve-and-append mode.

## Solution Assessment

Unclear as proposed: the user-visible behavior is plausible, but the exact `preserveOnFinal` versus `finalBehavior` shape and whether this should be Discord-only or shared preview policy need maintainer product judgment.
