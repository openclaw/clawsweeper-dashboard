## Summary

Keep open: current main and v2026.6.8 still replace the Discord progress draft with the final text and expose no preserve, append, or separate-final setting. This is a valid Discord UX feature request, but it needs maintainer product/config direction before an implementation PR.

## What This Changes

Request an opt-in Discord progress-streaming finalization mode that preserves the live reasoning/tool-progress draft and appends or sends the final answer separately instead of replacing the draft.

## Best Possible Solution

Have maintainers choose the Discord-owned finalization policy first, then implement the selected default-preserving option with schema/types/docs, regression tests, and live Discord proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current behavior is nevertheless source-confirmed: progress drafts are finalized by editing the preview message to the final answer, and no preserve/append setting exists.

## Solution Assessment

Unclear. An opt-in finalization policy is plausible, but maintainers should first choose whether the supported shape is append, send-separate, or a narrower existing-mode adjustment before adding config surface.
