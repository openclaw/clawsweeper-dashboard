## Summary

Keep open: current main and the latest release still finalize Discord progress drafts by replacing or clearing the draft, and there is no preserve/append/send-separate setting. The request is plausible and aligned with channel UX priorities, but it adds public Discord streaming/config behavior and needs maintainer product direction before implementation.

## What This Changes

The issue requests a Discord streaming setting that keeps the progress draft visible on final delivery and appends or sends the final assistant answer separately instead of replacing the draft.

## Best Possible Solution

Maintainers should first choose the public Discord progress-finalization contract; if accepted, implement it in the existing Discord draft-preview/finalizer path with default current behavior, strict schema/docs updates, and focused Discord regression coverage.

## Reproduction Assessment

Yes, source-level. Current Discord progress finalization edits the draft message into the final answer when possible, otherwise the shared finalizer clears the draft after normal delivery, and no preserve/append setting exists.

## Solution Assessment

Unclear until maintainers choose the product shape. The narrow implementation should extend the existing Discord progress finalization path rather than add a parallel delivery system, but the exact `preserveOnFinal` versus `finalBehavior` contract is a product/config decision.
