## Summary

Keep open: current main and v2026.6.10 still finalize Discord progress drafts by editing the preview into the final answer, and there is no preserve, append, or send-separate finalization setting. The request is valid, but it adds Discord streaming/config UX policy and needs maintainer direction before implementation.

## What This Changes

The issue requests a Discord progress-streaming option that preserves the live progress/reasoning draft and appends or sends the final assistant answer instead of replacing it.

## Best Possible Solution

Have maintainers choose the Discord-owned finalization policy first, then implement the selected default-preserving option with schema/types/docs, focused regression coverage, and live Discord proof.

## Reproduction Assessment

Yes, at source level. Discord progress-mode tests and implementation show progress lines are written into a draft, then final delivery edits that preview to the final answer content; I did not run a live Discord reproduction.

## Solution Assessment

Unclear. A default-off finalization policy is plausible, but maintainers should choose append, send-separate, or an existing verbose-lane alternative before adding public config surface.
