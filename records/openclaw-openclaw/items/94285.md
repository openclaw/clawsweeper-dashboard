## Summary

Keep open: current main and v2026.6.10 still replace the Discord progress draft with the final answer and expose no preserve, append, or separate-final setting. This remains a valid Discord UX/config feature request, but it needs maintainer product direction before implementation.

## What This Changes

Request an opt-in Discord progress-streaming finalization setting that preserves the progress draft and appends or sends the final assistant answer separately instead of replacing the draft.

## Best Possible Solution

Have maintainers choose the Discord-owned finalization policy first, then implement the selected default-preserving option with schema/types/docs, focused regression tests, and live Discord proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection still confirms the current behavior: Discord progress drafts are finalized by editing the preview message to the final answer, and no preserve/append setting exists.

## Solution Assessment

Unclear. An opt-in finalization policy is plausible, but maintainers should first choose append versus send-separate behavior and decide whether the new config surface is worth the compatibility cost.
