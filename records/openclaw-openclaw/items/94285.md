## Summary

Keep open. Current main and the latest release still replace/finalize Discord progress drafts into the final answer path and expose no `preserveOnFinal` or `finalBehavior` setting, while the requested behavior is a new Discord UX/config choice rather than an already-implemented fix.

## What This Changes

Request to add an opt-in Discord progress-streaming finalization mode that preserves the live progress draft and appends or separately sends the final assistant answer.

## Best Possible Solution

Keep this as the canonical issue for normal-answer Discord progress-draft preservation, choose one opt-in finalization enum or equivalent API, then implement it in the Discord draft-preview path with schema, docs, and focused tests.

## Reproduction Assessment

Yes for source-level behavior: current docs and code show Discord progress drafts are transient and can finalize by editing to the final answer. I did not run a live Discord session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API. The enum-style `finalBehavior` direction looks more future-proof than a boolean because it can distinguish replace, append, and separate-message behavior explicitly.
