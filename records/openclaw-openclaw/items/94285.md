## Summary

Keep open: current main still finalizes Discord progress drafts by replacing the draft with the final answer, the config schema/docs do not include the requested preserve/append/separate finalization setting, and no linked PR owns this exact request.

## What This Changes

Request to add an opt-in Discord progress-streaming finalization mode that preserves the live progress draft and appends or separately sends the final assistant answer instead of replacing the draft.

## Best Possible Solution

Keep this open for a maintainer decision on Discord progress finalization policy; if accepted, add an explicit opt-in finalBehavior-style setting with schema, docs, and focused Discord progress/finalization tests while preserving the current replace default.

## Reproduction Assessment

Yes, source-level: current main edits the preview message to the final answer and the Discord process test asserts the progress draft is replaced by final content. I did not run a live Discord reproduction because this read-only review did not execute tests or external channel flows.

## Solution Assessment

Unclear until maintainers choose the product behavior; a finalBehavior enum looks more maintainable than a boolean because it models replace, append, and separate-final explicitly. The implementation should stay opt-in and preserve the current replace default.
