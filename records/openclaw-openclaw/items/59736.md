## Summary

Keep open. Current main has only a one-target Makefile and `make help` fails, so the requested self-documenting common development task surface is not implemented; the package scripts and contributor docs still expose contributors to raw `pnpm` commands.

## What This Changes

The issue requests a self-documenting Makefile or justfile with common development tasks such as help, build, test, check, and dev, plus contributor documentation.

## Best Possible Solution

Expand the existing top-level Makefile into a small optional contributor entry point that delegates to established `pnpm` scripts, adds `make help`, and documents the wrapper without making it the source of truth over `package.json`.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a DX feature request. Current-main verification is direct: `make help` fails and the Makefile only wraps `pnpm build`.

## Solution Assessment

Yes, expanding the existing Makefile is the best narrow direction now that main already has a top-level Makefile. A new justfile would add another tool requirement without solving more of the current request.
