## Summary

Keep open. Current `main` still does not apply `cron.sessionRetention` to default keep-mode subagent session-store rows, and ACP cleanup is handled by a separate task-maintenance path rather than a generic retention sweep. The related reports are closed into this tracker and the direct fix PR was closed unmerged, so this remains the canonical item.

## What This Changes

The issue requests restart-safe retention cleanup for completed subagent and ACP sessions instead of pruning only cron-run sessions.

## Best Possible Solution

Keep this as the canonical tracker and design a lifecycle-aware, restart-safe retention path for completed native subagent and ACP sessions. The finished implementation should preserve active and intentionally persistent sessions, reuse `sessions.delete` or existing transcript archive helpers, run during startup/store-write or durable maintenance paths, and document the selected retention knob.

## Reproduction Assessment

Yes for the code-path reproduction: current `main` still has a cron-only reaper guard, default run-mode subagents resolve to `cleanup: "keep"`, and keep-mode subagent cleanup only removes registry records after TTL. I did not live-reproduce RSS growth because this review was read-only.

## Solution Assessment

No, the simple suggested guard expansion is not clearly the best fix because subagent keep/session modes and ACP lifecycle rules have separate ownership and terminal-state semantics. The safer solution is a durable lifecycle-aware retention sweep with tests for completed keep-mode subagents, active/thread-bound preservation, parent-owned ACP cleanup, and transcript handling.
