## Summary

Keep open. Current main and v2026.6.8 still do not provide optional secondary token/password auth on top of verified Tailscale Serve identity; the existing supported workaround is to disable Tailscale identity auth and use token/password instead.

## What This Changes

Request optional secondary token/password authentication, runtime warnings, and documentation for Tailscale Serve deployments that currently trust verified tailnet identity without shared-secret credentials.

## Best Possible Solution

Define a maintainer-approved layered Gateway auth contract, then implement schema/types/help metadata, auth enforcement, runtime warning or audit messaging, docs, and focused HTTP plus WebSocket tests.

## Reproduction Assessment

Yes, at source level. Current main lacks a secondary-auth config/schema field, and the WS Control UI path has tests and code that accept verified Tailscale identity without token/password credentials when `allowTailscale` is enabled.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix needs maintainer/security agreement on the permanent layered-auth contract rather than a one-off flag added directly to the current branch.
