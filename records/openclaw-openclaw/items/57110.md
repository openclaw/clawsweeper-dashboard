## Summary

Keep open. Current main and v2026.6.10 still do not provide optional secondary token/password authentication for verified Tailscale Serve identity; the related fix for https://github.com/openclaw/openclaw/issues/50630 closed the explicit `auth.mode="none"` exposure gap, but not this layered-auth request.

## What This Changes

The issue requests an opt-in Gateway auth mode that layers token/password verification on top of Tailscale Serve identity, plus warnings and docs for the current trust boundary.

## Best Possible Solution

Define a maintainer-approved layered Gateway auth contract, then update config types/schema/help, WebSocket auth enforcement, warning or audit messaging, public docs, and focused auth tests together.

## Reproduction Assessment

Yes, at source level for the current behavior: current main and v2026.6.10 both accept verified Tailscale Serve identity for ws-control-ui without token/password credentials when `allowTailscale` is enabled. No live Tailscale runtime repro was run in this read-only sweep.

## Solution Assessment

Unclear. The requested opt-in flag is plausible, but maintainers should settle the permanent layered-auth contract instead of adding a one-off config key around one auth branch.
