## Summary

Keep open. Current main and the latest release document and partially harden the Tailscale Serve trust boundary, but they still do not provide the requested opt-in mode that requires both verified Tailscale/trusted-proxy identity and a token/password.

## What This Changes

The issue requests opt-in secondary token/password authentication plus clearer warnings/docs for Tailscale Serve or trusted-proxy deployments.

## Best Possible Solution

Make a maintainer security/product decision on whether OpenClaw should support identity-plus-shared-secret gateway auth, then implement one explicit current-schema contract with docs, audit guidance, and focused gateway auth tests.

## Reproduction Assessment

Yes at source level, but this is a feature gap rather than a broken existing contract. Current main has no secondary-auth config key and the auth path treats verified Tailscale identity and token/password auth as alternatives.

## Solution Assessment

Unclear. The proposed `requireSecondaryAuth` shape may be reasonable, but current docs already offer `allowTailscale: false` for secret-only auth, so maintainers should decide the identity-plus-secret contract before implementation.
