## Summary

Keep open. Current main and the latest release still support tokenless verified Tailscale Serve identity for the Control UI/WebSocket path and still have no optional secondary token/password-auth contract; the already-merged no-auth exposure fix only covers auth.mode=none with Tailscale exposure.

## What This Changes

Requests optional secondary token/password authentication, runtime warning, and documentation hardening for Tailscale Serve trusted identity access.

## Best Possible Solution

Define a maintainer-approved layered Gateway auth contract, then update config schema/types/help, WebSocket auth enforcement, warning or audit messaging, public docs, and focused auth tests together.

## Reproduction Assessment

Yes, at source level. Current main lacks a secondary-auth config/schema field, and code plus tests accept verified Tailscale identity for ws-control-ui without token/password credentials when allowTailscale is enabled.

## Solution Assessment

Unclear. The requested opt-in flag is plausible, but maintainers should decide the permanent layered-auth contract instead of adding a one-off config key around one auth branch.
