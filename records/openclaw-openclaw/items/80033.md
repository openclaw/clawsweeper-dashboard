## Summary

This PR should stay open because the patch is a plausible owner-path fix for opencode-go DeepSeek V4 reasoning metadata, but it does not yet make `max` a supported thinking level in the validation/profile path and it lacks after-fix real behavior proof.

## What This Changes

The PR adds `supportedReasoningEfforts` arrays including `off`, `minimal`, `low`, `medium`, `high`, `xhigh`, and `max` to the opencode-go DeepSeek V4 Pro and Flash supplemental model compat entries.

## Best Possible Solution

Expose both `xhigh` and `max` through the opencode-go DeepSeek V4 thinking profile or a tested generic compat-profile path, then add regression coverage for validation and stream payload mapping.

## Reproduction Assessment

Yes at source level: current main validates session thinking levels against `resolveThinkingProfile`, while opencode-go has no provider profile and the generic compat fallback only appends `xhigh`, not `max`. I did not run a live provider flow, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No, not yet: adding `supportedReasoningEfforts` is in the right area, but the current profile builder still will not expose `max`. The narrower maintainable fix is to add an opencode-go DeepSeek V4 thinking profile or extend the generic compat profile logic with tests for both `xhigh` and `max`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Expose `max` in the thinking profile too:** `extensions/opencode-go/provider-catalog.ts:39`
  - body: This adds `max` to the model compat list, but the validator does not build options from every compat entry: `resolveThinkingProfile` only uses `supportedReasoningEfforts` to append `xhigh`. Since opencode-go does not register a provider thinking profile, `/think max` or a session patch with `thinkingLevel: "max"` can still hit the same unsupported-level error before the stream wrapper maps it to `reasoning_effort: "max"`. Add a DeepSeek V4 thinking profile for opencode-go, or update the generic compat profile builder and tests so `max` is actually supported.
