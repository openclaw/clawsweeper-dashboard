## Summary

Keep open: this PR targets a valid browser bug and matches the previously identified repair direction by bounding Playwright-backed remote tab enumeration and evicting the scoped connection on timeout. The runtime patch looks sound, but the public timeout docs should be updated because the PR expands what the existing CDP timeout settings cover.

## What This Changes

The PR adds a timeout parameter to Playwright-backed browser tab listing, passes the remote CDP timeout budget from remote profile tab ops, evicts scoped CDP connections on timeout, adds regression tests, and updates the changelog.

## Best Possible Solution

Land the browser-plugin runtime fix after updating the public browser configuration docs so the existing remote CDP timeout settings clearly cover persistent Playwright tab enumeration as well as reachability and tab-opening paths.

## Reproduction Assessment

Yes, source-reproducible rather than locally executed in this read-only review. Current main awaits the Playwright page enumeration path without a timeout, and the linked issue plus regression mock describe a never-resolving CDP session that keeps the request pending.

## Solution Assessment

Mostly yes. Reusing the existing remote CDP timeout budget and evicting the scoped Playwright connection is the narrow maintainable fix; the remaining improvement is documenting the expanded timeout scope.
