## Summary

Keep open: this is a focused bug-fix PR for a still-open browser CDP timeout issue, and source plus dependency inspection supports the timeout-and-scoped-disconnect approach; I found no blocking correctness or security finding.

## What This Changes

The PR adds an operation timeout to Playwright-backed remote CDP tab listing, passes the existing remote CDP timeout budget from tab ops, evicts the scoped connection on timeout, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land a rebased version of this browser plugin fix after required checks pass, preserving the scoped timeout cleanup, SSRF checks, and existing remote CDP configuration semantics.

## Reproduction Assessment

Yes at source level: current main awaits Playwright-backed remote tab enumeration without an operation timeout, and Playwright’s newCDPSession/CDPSession.send APIs do not provide per-call timeout options. The PR body also supplies after-fix terminal proof against a deliberately hanging loopback CDP endpoint.

## Solution Assessment

Yes. Reusing the existing remote CDP timeout budget and evicting only the affected scoped Playwright connection is the narrow owner-local fix; returning an empty tab list would hide a real remote browser failure.
