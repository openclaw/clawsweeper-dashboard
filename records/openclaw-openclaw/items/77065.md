## Summary

Keep open: this PR targets a valid source-level bug from #77039, but the current restore guard still keeps stale corpus supplements alive for service-owning plugins that intentionally stop registering them, and the changelog links the fix to the wrong PR.

## What This Changes

The branch snapshots pre-load memory corpus supplements, restores non-re-registered supplements for plugins that still register services during activating loads, adds loader regression tests, and adds a changelog entry.

## Best Possible Solution

Track corpus supplement provenance or lifecycle phase so only service-start-owned supplements survive activating reloads, while register-time or config-disabled supplements can disappear normally.

## Reproduction Assessment

Yes, source-level. Current main clears memory plugin state at the start of an activating load, while service-start corpus registrations happen outside loadOpenClawPlugins and are not replayed by a later lazy registry load.

## Solution Assessment

No. Snapshotting before the clear is a reasonable direction, but restoring by service ownership is too broad; the fix should be provenance-aware and the changelog reference should be corrected.
