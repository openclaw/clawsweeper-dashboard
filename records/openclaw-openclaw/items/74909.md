## Summary

## What This Changes

The PR changes the PI embedded runner auth-profile failure policy to return null for non-auth or ambiguous failover reasons and adds focused unit coverage for filtered and retained reasons.

## Best Possible Solution

Land a provider-aware narrower cooldown policy that filters GitHub Copilot format/model-path failures while preserving OpenAI Codex WHAM-probed ambiguous failures, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. The source path is high-confidence: current main maps 400/422 payload rejections to format, non-null reasons flow into markAuthProfileFailure, and the PR's newly null ambiguous reasons would return before the WHAM-backed Codex path.

## Solution Assessment

No. The requested narrowing is maintainable, but the blanket mapping is too broad; the safer fix is provider-aware cooldown eligibility or a separate WHAM probe path that keeps Codex ambiguous failures covered.
