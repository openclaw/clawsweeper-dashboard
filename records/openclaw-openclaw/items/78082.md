## Summary

Keep open: current main and the latest release still skip Discord slash-command deploy and cleanup when native commands are disabled, so stale specialist app commands can remain a real multi-account routing problem rather than an implemented fix.

## What This Changes

The issue requests that disabling native Discord commands for a specialist account remove or otherwise prevent stale registered slash commands from hijacking generic commands such as `/side`.

## Best Possible Solution

A maintainer should choose a Discord deploy cleanup policy that can reconcile stale application commands for disabled specialist accounts without surprising operators who rely on the current disabled-native zero-REST behavior.

## Reproduction Assessment

Yes from source inspection: current main skips deploy/cleanup when native commands are disabled, and the docs/tests explicitly preserve that behavior. I did not run a live Discord account reproduction in this read-only review.

## Solution Assessment

No automatic fix is clearly best yet. Deleting stale commands would address the reported hijack, but it changes the documented disabled-native no-cleanup contract and needs a maintainer-approved deploy policy or cleanup mode.
