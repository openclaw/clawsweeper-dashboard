## Summary

Keep open. Current main and the latest shipped behavior intentionally skip Discord slash-command registration and cleanup when native commands are disabled, while this report asks for a different multi-account ownership/cleanup policy that still needs maintainer judgment.

## What This Changes

Request that per-account Discord native-command disable reconcile or delete stale commands, or provide account-level ownership controls so specialist apps cannot keep generic commands like `/side`.

## Best Possible Solution

Define the Discord native-command ownership and cleanup policy first, then implement an explicit cleanup, deploy-mode, or per-account command ownership path with docs, tests, and redacted live Discord proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register commands for a specialist account, later set that account's `commands.native=false`, and current main skips all command lifecycle calls, leaving provider-side registrations untouched unless cleaned externally.

## Solution Assessment

Unclear. Cleanup-on-disable would address stale registrations, but it conflicts with the shipped low-power no-REST startup contract; an explicit cleanup/deploy/ownership policy is the safer maintainable direction.
