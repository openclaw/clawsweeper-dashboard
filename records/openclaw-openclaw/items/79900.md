## Summary

This PR fixes a real, source-reproducible HTML 401 misclassification, but it should not merge as-is because the new auth HTML 401 kind is still missing from the existing raw HTML suppression paths for lifecycle and failover console messages.

## What This Changes

The PR adds an `auth_html_401` provider runtime failure kind, maps HTML HTTP 401 responses to it, adds re-authentication formatter copy, regression tests, and a changelog entry.

## Best Possible Solution

Land the narrow classifier and formatter fix together with matching raw HTML suppression for `auth_html_401` in lifecycle and failover console paths plus focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `401 <!DOCTYPE html>...</html>` enters the HTML branch and falls to `upstream_html` because only status 403 is special-cased; I did not run tests because this review is read-only.

## Solution Assessment

No as submitted. Adding `auth_html_401` is the right narrow direction, but the maintainable fix should also mirror the existing `auth_html_403` raw HTML suppression paths so auth/login HTML is not previewed in console messages.
