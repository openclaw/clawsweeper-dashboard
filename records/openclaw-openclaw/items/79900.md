## Summary

Keep open: the core HTML 401 classification fix is useful and the supplied terminal proof is sufficient, but the patch misses existing raw-error suppression paths, so auth/login HTML can still be appended to lifecycle and failover console messages.

## What This Changes

The branch adds an `auth_html_401` provider runtime failure kind, maps HTTP 401 HTML bodies to it, adds re-authentication copy and formatter tests, and updates the changelog.

## Best Possible Solution

Land a version that treats HTML 401 like existing auth HTML across classifier, user copy, console suppression, and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main routes `401 <!DOCTYPE html>...</html>` through the HTML branch but only maps status 403 to `auth_html_403`, so 401 falls to `upstream_html`. I did not run tests because this review is read-only.

## Solution Assessment

No as submitted: adding `auth_html_401` is the right narrow user-facing fix, but the maintainable solution also mirrors the existing `auth_html_403` raw-error console suppression and tests.
