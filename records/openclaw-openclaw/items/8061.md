## Summary

Keep open: current main still exposes and enforces only the global subagent concurrency cap, so the requested per-target-agent limit remains absent and not superseded by the adjacent lane/concurrency items.

## What This Changes

## Best Possible Solution

Add a validated per-target-agent subagent concurrency setting under the agreed config namespace, document it, and enforce it alongside the existing global subagent lane limit.

## Solution Assessment
