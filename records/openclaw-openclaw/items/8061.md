## Summary

Keep open. Current main still has only a global subagent concurrency cap and no typed, validated, documented, or enforced per-target-agent cap; the related concurrency items cover adjacent lane, per-parent, or session-lifetime scopes rather than this TTS-style target-agent limit.

## What This Changes

Request a per-agent subagent concurrency cap in `agents.list` so one target agent can be serialized while other subagents remain parallel.

## Best Possible Solution

Add a documented per-target subagent concurrency setting, likely under `agents.list[].subagents` if maintainers keep the namespace pattern, with strict schema/type/help/docs updates and native plus ACP enforcement alongside the global cap.

## Reproduction Assessment

Yes, for source-level capability proof: current main lacks any per-target maxConcurrent field in the strict schema, exported types, docs, and runtime resolver. No live TTS/local-model reproduction was attempted.

## Solution Assessment

Unclear as proposed. The capability is well scoped, but maintainers should choose the public config shape and define how a per-target cap composes with the global subagent lane and adjacent concurrency controls.
