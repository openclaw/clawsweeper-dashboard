## Summary

Keep open: current main still has only a global subagent concurrency cap, and related items cover different main-lane or per-parent scopes rather than this per-target-agent limit.

## What This Changes

Allow a per-agent subagent concurrency limit in agent-list configuration so one target agent can serialize spawns without globally throttling other subagents.

## Best Possible Solution

Add a validated per-target cap under the existing per-agent subagents namespace, document it, and enforce it alongside the current global subagent lane for native and ACP spawns.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is source-verifiable: strict per-agent config has no accepted cap and both native and ACP spawn paths use the shared subagent lane.

## Solution Assessment

Unclear until maintainers confirm the public API shape. The narrow maintainable direction appears to be `agents.list[].subagents.maxConcurrent`, matching existing per-agent subagent controls instead of adding an unrelated top-level agent-list key.
