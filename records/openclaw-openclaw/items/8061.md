## Summary

Keep open. Current main still exposes only a global subagent concurrency cap and lacks a typed, validated, documented, or enforced per-target-agent cap; the related concurrency issues and PRs cover adjacent lane/session scopes rather than this TTS-style target-agent limit.

## What This Changes

The issue requests a per-agent subagent concurrency cap so one target agent can be serialized without limiting all other subagents.

## Best Possible Solution

Add a documented per-target subagent concurrency setting, likely under agents.list[].subagents if maintainers keep the current namespace, with strict schema/type/help/docs updates and native plus ACP enforcement alongside the global cap.

## Reproduction Assessment

## Solution Assessment

Unclear as proposed. The capability is well scoped, but maintainers should choose the public config shape and define how a per-target cap composes with the global subagent lane and adjacent concurrency controls.
