## Summary

Keep open. Current main still has only a global subagent concurrency lane cap, while `agents.list[]` has no typed, validated, documented, or enforced per-target-agent `maxConcurrent`; related #52655 was closed unmerged and #73870/#43235 cover different lane scopes.

## What This Changes

Allow a specific configured target agent to have its own subagent concurrency limit while other subagents can remain more parallel.

## Best Possible Solution

Add a validated per-target cap under the existing per-agent subagents namespace, document it, and enforce it alongside the global lane for native and ACP spawns with focused concurrency coverage.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is high-confidence source-verifiable: strict per-agent config has no accepted cap and all spawn paths use the shared subagent lane.

## Solution Assessment

Unclear until a maintainer confirms the public API shape. The narrow maintainable direction appears to be `agents.list[].subagents.maxConcurrent`, matching existing per-agent subagent controls instead of adding unrelated main-lane policy.
