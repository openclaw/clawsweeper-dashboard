## Summary

The requested target-agent cap is still absent on current main and the neighboring open reports cover broader or different concurrency scopes, so this issue remains a useful narrow product decision.

## What This Changes

Requests a per-target-agent concurrency cap for subagents while preserving broader global concurrency.

## Best Possible Solution

Keep the global subagent pool as the safety ceiling and add an optional target-agent cap under the established agents.list[].subagents namespace, enforced consistently for native and ACP spawns after maintainers confirm composition semantics.

## Reproduction Assessment

Yes at source level: current main has only the global subagent limit and no typed, validated, documented, or enforced per-target limit. This is a missing capability rather than a failure of an existing contract.

## Solution Assessment

The target-agent limit is the narrowest solution to the reported TTS use case, but the exact field shape and its interaction with global, per-parent, and named-lane controls require maintainer product judgment.
