## Summary

Keep open. Current main still has only a global subagent concurrency cap, while the requested per-target-agent cap is neither typed nor validated nor enforced, and related items cover different queue-lane or per-parent limits.

## What This Changes

Add an optional per-agent subagent concurrency limit so a target agent such as a local TTS agent can be capped without limiting other subagents.

## Best Possible Solution

Add a validated per-target-agent subagent concurrency setting, document it, and enforce it for native and ACP sessions_spawn while preserving the existing global subagent safety cap.

## Reproduction Assessment

Yes. Source inspection on current main shows strict per-agent config and AgentConfig have no per-agent maxConcurrent, while runtime dispatch applies only the shared agents.defaults.subagents.maxConcurrent lane cap.

## Solution Assessment

Unclear until maintainers choose the public config contract. The requested direction is reasonable, but agents.list[].subagents.maxConcurrent appears more consistent with the existing per-agent subagents namespace than a top-level agents.list[].maxConcurrent.
