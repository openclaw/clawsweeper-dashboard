## Summary

Keep open. Current main still lacks a typed, validated, documented, or enforced per-target-agent subagent concurrency cap, and the related items cover different lane or per-parent limits.

## What This Changes

Request adding an optional per-agent subagent concurrency limit for a TTS/local-model target without limiting other subagents.

## Best Possible Solution

Add a validated per-target-agent subagent concurrency setting, document it, and enforce it for native and ACP spawns while preserving the existing global subagent cap.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: strict per-agent config and generated schema have no cap field, and runtime dispatch uses one shared `subagent` lane cap.

## Solution Assessment

Unclear until maintainers choose the public config contract. The requested direction is reasonable, but `agents.list[].subagents.maxConcurrent` appears more consistent with the existing namespace than a top-level `agents.list[].maxConcurrent`.
