## Summary

Keep open. Current main still has only a global subagent concurrency lane plus active-child/depth guardrails; it has no typed, validated, documented, or enforced per-target-agent subagent maxConcurrent setting, and the related items cover different scheduler scopes.

## What This Changes

Request adding an optional per-target subagent concurrency limit to agent list entries so one target agent can be serialized while other subagents remain parallel.

## Best Possible Solution

Add a documented per-target subagent concurrency setting with agreed config shape, strict schema/type/help/docs updates, and enforcement in native plus ACP spawn paths while preserving the global subagent lane cap.

## Solution Assessment
