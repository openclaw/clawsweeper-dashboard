## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat user-selected reply workflow: there is no right-click/context-menu handler, no message-level Reply action, no selected quote state in the composer, and the existing reply pill is display-only assistant directive metadata.

## What This Changes

The issue requests a right-click Dashboard/WebChat message reply action that quotes or references the selected message and focuses the composer.

## Best Possible Solution

Implement a narrow Control UI quote-reply flow: add accessible message-level Reply controls plus a right-click menu, selected quote preview, composer focus management, and safe quoted-Markdown insertion, while leaving durable reply metadata to the separate metadata/threading work.
