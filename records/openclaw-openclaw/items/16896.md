## Summary

Keep open. Current main still lacks the requested Dashboard/Control UI WebChat user-selected reply workflow: there is no right-click handler, no Reply action in message actions, no selected quote state in the composer, and the existing reply pill only renders inbound assistant directive metadata; the related duplicates were already closed back to this issue while broader reply metadata work remains separate.

## What This Changes

Add a right-click/contextual Reply workflow in Dashboard WebChat that quotes or references the selected message and focuses the composer.

## Best Possible Solution

Land a narrow Control UI quote-reply flow with accessible message Reply controls, a right-click menu, selected quote preview, composer focus, and safe quoted-Markdown insertion while leaving durable reply metadata to the separate protocol/threading work.

## Solution Assessment
