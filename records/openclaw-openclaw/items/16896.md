## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat user-selected reply workflow: there is no context-menu handler, no reply callback or selected quote state in the composer, and the existing reply pill only renders inbound directive metadata.

## What This Changes

Add a Dashboard/WebChat reply workflow so users can right-click or choose Reply on a message, quote it in the composer, and send a contextual response.

## Best Possible Solution

Implement the first pass in the Control UI chat owner area: accessible message-level Reply controls plus a right-click menu, local selected-message state, textarea focus, dismissible quote preview, and sanitized quoted-Markdown prepending on send; defer persisted metadata/threading to a separate design.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by inspecting current main's Control UI chat renderer, composer, grouped message actions, and send schema and confirming the requested reply/context-menu workflow is absent.

## Solution Assessment

Yes. A UI-only quoted-Markdown reply workflow is the narrowest maintainable first fix for the Dashboard/WebChat problem; native threaded reply metadata would be a broader follow-up rather than a prerequisite.
