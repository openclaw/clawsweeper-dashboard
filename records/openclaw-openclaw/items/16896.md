## Summary

Keep open. Current main still lacks the requested Dashboard/Control UI WebChat user-selected reply workflow: there is no right-click/context-menu handler, no Reply action in message actions, no selected quote state in the composer, and the existing reply pill is inbound/display metadata only.

## What This Changes

The issue requests a right-click Reply action in Dashboard/Control UI webchat that quotes a selected message into the composer.

## Best Possible Solution

Land a narrow Control UI quote-reply flow with accessible message Reply controls, a right-click menu, selected quote preview, composer focus, and safe quoted-Markdown insertion while leaving durable reply metadata to the separate protocol/threading work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection on current main gives high-confidence evidence that the requested right-click/contextual reply workflow is absent.

## Solution Assessment

Yes at the direction level: a UI-only quoted-Markdown reply flow with accessible controls is the narrowest maintainable first step. Structured reply metadata should stay coordinated with https://github.com/openclaw/openclaw/issues/38364 and https://github.com/openclaw/openclaw/pull/62262.
