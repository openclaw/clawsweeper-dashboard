## Summary

The PR is still a useful candidate for the linked sidebar discoverability bug, but it should not merge yet because the current head still routes non-sidebar chat/gateway session refreshes through a sidebar-only all-agents helper, which can overwrite shared session state with cross-agent rows.

## What This Changes

The branch adds a Web UI Recent Sessions sidebar All agents toggle, sidebar-specific session-list helpers, filtering/rendering changes, and focused UI/session regression tests for cross-agent subagent session visibility.

## Best Possible Solution

Keep the default-off sidebar All agents behavior, but restore scoped helpers for non-sidebar chat/gateway refresh callers and leave any Sessions-table all-agents mode to an explicit maintainer-approved follow-up.

Yes, at source level. The PR head shows sidebarRecentSessionsListParams* returns an empty override when the sidebar toggle is on, and three non-sidebar refresh callers then write the unscoped result into shared sessionsResult.

## Solution Assessment

No, not yet. A default-off sidebar reveal is a reasonable fix shape, but reusing the sidebar-wide helper from shared chat/gateway refresh paths is not the best boundary because it changes non-sidebar session state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep post-send refreshes scoped:** `ui/src/ui/app-chat.ts:1233`
  - body: When the sidebar All agents toggle is enabled, this helper returns `{}`, so a successful chat send now reloads `sessions.list` without the run's `agentId`. `loadSessions` writes that unscoped result into shared `sessionsResult`, which the Sessions tab also renders, so a sidebar-only control can leak cross-agent rows into non-sidebar state. Use the scoped refresh-target helper here and keep the sidebar helper on sidebar-owned refreshes only.
  - confidence: 0.92
- **[P1] Keep terminal refreshes scoped:** `ui/src/ui/app-gateway.ts:1018`
  - body: Terminal final events drain `refreshSessionsAfterChat` for a specific run target, but this line now widens that refresh whenever the sidebar toggle is on. That makes a terminal completion overwrite shared `sessionsResult` with an all-agent list even though the PR says All agents is sidebar-only, so this path should keep `scopedAgentListParamsForRefreshTarget`.
  - confidence: 0.9
- **[P1] Keep deferred message refreshes scoped:** `ui/src/ui/app-gateway.ts:1202`
  - body: This deferred `session.message` refresh runs while a chat run is active and then replays the pending reload, but with the sidebar toggle enabled it now refreshes sessions without the current session's agent scope. Because the refreshed result feeds the Sessions table and chat controls, this non-sidebar path should continue using `scopedAgentListParamsForSession`.
  - confidence: 0.88
