## Summary

Keep open: the linked bug is real and this is the viable fix candidate, but the current PR head still lets a sidebar-only toggle remove agent scoping from shared chat/gateway session refreshes.

## What This Changes

The PR adds a Web UI Recent Sessions sidebar All agents toggle and related helper/test changes so child-spawned subagent sessions can be found from the sidebar without changing gateway protocol or storage.

## Best Possible Solution

Keep the All agents behavior limited to sidebar-owned refreshes while all chat, gateway, checkpoint, and Sessions-tab refreshes derive scope from the current session or refresh target.

## Reproduction Assessment

Yes at source level: current main filters subagent/spawned rows out of Recent Sessions and scoped list loads can exclude cross-agent rows before the sidebar can render them. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No: the default-off sidebar toggle is the right narrow product shape, but shared chat/gateway refresh paths must not call sidebar-only helpers because that can widen session-list state outside the sidebar.
