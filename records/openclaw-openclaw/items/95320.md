Full review comments:

- **[P1] Keep the all-agents override out of shared loads:** `ui/src/ui/controllers/sessions.ts:1128`
  - body: This makes every `loadSessions(state, { agentId })` call drop that explicit scope whenever the sidebar toggle is on. Other callers such as Skill Workshop revision loading and chat refreshes rely on that `agentId`, so using the sidebar control can turn unrelated agent-scoped refreshes into unscoped session-state updates; make only the sidebar toggle request unscoped and keep explicit overrides authoritative.
  - confidence: 0.9
- **[P2] Keep the toggle reachable when the scoped list is empty:** `ui/src/ui/app-render.ts:581-582`
  - body: The new section renders only when the currently scoped `sessionsResult` already has a non-global/non-cron row. If a user's current agent has no visible recent rows and the only sessions to rediscover are hidden cross-agent subagent rows, the All agents button never appears, so the reported workflow remains blocked. Render the scope control independently of the scoped row count or add a focused empty-scope regression.
  - confidence: 0.84
