## Summary

## What This Changes

The PR expands `browser.upload` preflight to accept `getMediaDir()/inbound`, exports `getMediaDir` through the browser runtime barrel, and adds mocked upload unit tests.

## Best Possible Solution

Use one browser-owned upload path resolver across tool preflight, `/hooks/file-chooser`, and Playwright/Chrome-MCP execution that accepts the existing upload root plus verified managed inbound media, with escape tests and real WebChat-to-browser proof.

## Reproduction Assessment

Yes. Source inspection shows WebChat attachments are staged as `media/inbound/*`, while current `browser.upload`, the server route, and Playwright upload runtime only accept `DEFAULT_UPLOAD_DIR`.

## Solution Assessment

No. The direction is plausible, but this PR updates only the agent-side preflight; the maintainable fix needs the same resolver in the authoritative browser server/runtime path.
