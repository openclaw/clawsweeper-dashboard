## Summary

Keep open: the PR is aimed at a valid source-reproducible WebChat attachment upload bug, but it only widens the browser tool preflight while the browser route and Playwright upload validators still enforce the old uploads-only root.

## What This Changes

The PR allows `browser.upload` preflight to accept files under the managed inbound media directory and adds mocked unit coverage for uploads-root, inbound-media, and outside-root paths.

## Best Possible Solution

Move the inbound-media allowance into a shared browser-owned upload path resolver used by tool preflight, `/hooks/file-chooser`, and Playwright upload helpers, with tests for uploads-root, verified inbound media, and escape/rejection cases.

## Reproduction Assessment

Yes, source-level: WebChat stages attachments as `media/inbound/*`, while current browser upload validation accepts only `DEFAULT_UPLOAD_DIR`. I did not run a live WebChat-to-browser upload flow in this read-only review.

## Solution Assessment

No, not yet: the direction is right, but the implementation changes only the tool preflight and misses the downstream server/runtime validators that enforce uploads.
