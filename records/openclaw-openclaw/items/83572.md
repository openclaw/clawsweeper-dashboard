## Summary

Keep open: the PR targets a real source-reproducible browser upload bug, but the patch only relaxes the agent-facing preflight and leaves the browser control server and Playwright upload paths enforcing the old uploads-only root.

## What This Changes

The PR allows `browser.upload` preflight to accept paths under the managed inbound media directory and adds mocked unit coverage for uploads-root, inbound-media, and outside-root paths.

## Best Possible Solution

Move the inbound-media allowance into a shared browser-owned upload path resolver used by tool preflight, `/hooks/file-chooser`, and Playwright upload helpers, with tests that cover uploads-root, verified inbound media, and escape/rejection cases.

## Reproduction Assessment

Yes, source-level: WebChat can stage attachments as `media/inbound/*`, while current browser upload validation accepts only `DEFAULT_UPLOAD_DIR`. I did not run a live WebChat-to-browser upload flow in this read-only review.

## Solution Assessment

No, not yet: the direction is right, but the implementation only changes the tool preflight and misses the server-side/runtime validators that actually enforce uploads.
