## What This Changes

The issue requests a native, configurable same-run recovery step that gives the agent structured failed-tool context before finalization.

## Best Possible Solution

Define an opt-in core recovery contract that classifies structured failed tool results, bounds same-run continuations, preserves side-effect metadata, integrates with final-answer policy, and documents embedded/Codex capability limits.

Unclear pending product direction; native run-loop ownership is plausible, but safe design needs maintainer decisions on failure classes, mutating tools, budgets, finalization, and Codex/native fallback.
