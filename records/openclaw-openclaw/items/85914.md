## Summary

Keep open: current main still has hook observation, warning/finalization handling, loop guards, and narrow replay-safe runtime retries, but not the requested native tool-failure-specific same-run recovery step before final answer selection.

## What This Changes

The issue requests a native, configurable OpenClaw run-loop recovery step that preserves structured failed tool-call context and gives the same agent a bounded continuation before finalization.

## Best Possible Solution

Define a maintainer-approved, opt-in core recovery contract that classifies structured failed tool results, bounds same-run continuations, preserves side-effect metadata, and documents runtime-specific capability limits.

## Reproduction Assessment

Not applicable as a feature request. Source review confirms current main has observation, warnings, loop guards, and plugin follow-up primitives, but no existing native in-loop recovery contract to reproduce as broken.

## Solution Assessment

Unclear pending maintainer product direction. Native run-loop ownership is a plausible best solution, but the final contract needs decisions on failure classes, mutation safety, attempt budgets, final-answer policy, and Codex/native runtime boundaries.
