## Summary

Keep open: current main and the latest release expose hook observation, next-turn primitives, warning finalization, and bounded retry/failover pieces, but they do not provide the requested native in-loop recovery continuation before final answer selection. The remaining work is a product and architecture decision, not stale cleanup.

## What This Changes

Request to add a native, configurable tool-call failure recovery step inside the OpenClaw agent run loop.

## Best Possible Solution

Define a core-owned, opt-in recovery contract that re-enters the same agent turn after structured failed tool results, preserves side-effect metadata, honors loop/replay guards, and treats Codex-native transcript boundaries as a first-class design constraint.

## Reproduction Assessment

Not applicable as a feature request. Source review confirms current main has observation and follow-up primitives, but no native in-loop recovery continuation to reproduce as a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product contract. Native run-loop ownership is the plausible surface, but the implementation needs an approved policy for finalization, mutation safety, attempt budgets, and native runtime boundaries.
