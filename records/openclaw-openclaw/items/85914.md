## Summary

Keep open: current main and the latest release still provide hook observation and follow-up-turn primitives, but not the requested native, bounded same-run recovery continuation before finalization.

## What This Changes

Request a native OpenClaw run-loop capability that preserves structured tool-call failure context and lets the same agent perform a bounded recovery continuation before the turn finalizes.

## Best Possible Solution

Define a generic core run-loop recovery contract that feeds structured failure context back into the active agent under bounded, idempotency-aware limits while preserving plugin hooks as observation and follow-up extension points.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection do give high-confidence evidence that current main has observation/follow-up primitives rather than native same-run recovery semantics.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested native boundary is plausible for same-turn recovery, but the safe solution needs explicit decisions about retry limits, mutating tools, final-answer policy, runtime coverage, and plugin API interaction.
