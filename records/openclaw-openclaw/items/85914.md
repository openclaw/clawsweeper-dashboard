## Summary

Keep open: current main and v2026.5.22 have structured tool failure observation, warning/finalization handling, plugin follow-up primitives, and loop guards, but they still do not provide the requested native same-turn recovery continuation before final answer selection.

## What This Changes

The issue requests a configurable native tool-call failure recovery step inside the active agent run loop, with structured failure context, bounded continuation, and loop/side-effect safety.

## Best Possible Solution

Define an opt-in core-owned recovery contract that re-enters the same agent turn after structured failed tool results, preserves side-effect metadata, honors loop/replay guards, and treats native runtime transcript boundaries as a first-class constraint.

## Reproduction Assessment

Not applicable as a feature request. Source review confirms current main has observation, warning, and follow-up primitives but no existing native in-loop recovery contract to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the product contract. Native run-loop ownership is plausible, but the implementation needs approved policy for finalization, mutation safety, attempt budgets, config defaults, and native runtime boundaries.
