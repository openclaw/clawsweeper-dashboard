## Summary

Issue #85914 should stay open because current main records failed tool state, warns or suppresses final payloads, and exposes plugin follow-up hooks, but it does not implement the requested native same-run reflect-and-continue recovery step before final answer selection; the remaining work is a core run-loop product design, not a stale duplicate or ClawHub-only plugin idea.

## What This Changes

The issue requests a native, configurable OpenClaw run-loop capability that classifies failed tool calls and gives the agent a bounded same-run recovery continuation before choosing a final answer.

## Best Possible Solution

Define an opt-in core recovery contract that classifies structured failed tool results, bounds same-run continuations, preserves side-effect metadata, and documents embedded/Codex capability limits.

## Reproduction Assessment

Not applicable as a feature request; source inspection confirms current main has failure recording, plugin follow-up hooks, and finalization warnings but no native same-run reflect-and-continue recovery path.

## Solution Assessment

Unclear pending maintainer product direction; native run-loop ownership is plausible because plugin follow-up cannot guarantee same-run recovery, but the exact policy needs design for failure classes, mutating tools, budgets, final-answer suppression, cron behavior, and Codex capability fallbacks.
