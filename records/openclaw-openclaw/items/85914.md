## Summary

Keep open: current main and v2026.5.22 expose hook observation, final-answer revision hooks, follow-up-turn primitives, warning finalization, and retry/failover guards, but they still do not provide the requested native in-loop tool-failure recovery continuation before final answer selection.

## What This Changes

Add a native, configurable agent run-loop step that returns structured failed tool-call state to the same agent for a bounded recovery continuation before finalization.

## Best Possible Solution

Define an opt-in core-owned recovery contract that re-enters the same agent turn after structured failed tool results, preserves side-effect metadata, honors loop/replay guards, and treats Codex-native transcript boundaries as a first-class constraint.

## Reproduction Assessment

Not applicable as a feature request. Source review confirms current main has observation and follow-up primitives, but no native in-loop recovery continuation that is already an expected contract to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the product contract. Native run-loop ownership is plausible, but the implementation needs approved policy for finalization, mutation safety, attempt budgets, config defaults, and native runtime boundaries.
