## Summary

## What This Changes

Requests an opt-in native run-loop capability that feeds structured tool-call failures back to the same agent before finalization.

## Best Possible Solution

Define an explicit, opt-in core-owned recovery contract that classifies structured failed tool results, bounds one same-turn continuation, preserves side-effect metadata, and treats Codex-native transcript boundaries as first-class constraints.

## Reproduction Assessment

Not applicable as a feature request. Source review confirms current main has observation, warnings, loop guards, and plugin continuation primitives, but no existing native in-loop recovery contract to reproduce as broken.

## Solution Assessment

Unclear until maintainers choose the product contract. Native run-loop ownership is plausible, but the implementation needs approved policy for finalization, mutation safety, attempt budgets, and native runtime boundaries.
