## Summary

Keep open: current main and latest v2026.6.6 still route Feishu doc markdown through Feishu document.convert with no equation/backslash handling or regression coverage, so the reported data corruption is not fixed.

## What This Changes

The issue asks Feishu document creation/update to preserve LaTeX backslashes inside block formulas and equation tags.

## Best Possible Solution

Add owner-local Feishu doc math-block handling that preserves LaTeX source for '$$' and '<equation>' inputs, backed by focused tests and live Feishu proof, or explicitly document the unsupported limitation if the API cannot support it safely.

## Reproduction Assessment

No, not as a live end-to-end repro. Source inspection gives a credible path because write/append/insert still pass markdown directly through Feishu document.convert and there is no equation/backslash coverage, but a real Feishu document test is still needed.

## Solution Assessment

Unclear until live Feishu behavior is confirmed. The likely best fix is not a core markdown change; it should stay in the Feishu plugin's document conversion path and preserve math blocks before inserting Feishu docx blocks.
