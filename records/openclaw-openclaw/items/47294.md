## Summary

Keep open: current main and latest v2026.6.6 still send Feishu document markdown through `document.convert` without equation/backslash preservation or focused regression coverage, so the reported silent LaTeX content corruption remains unresolved.

## What This Changes

The issue reports that Feishu document writes strip LaTeX backslashes from complex `$$...$$` and `<equation>` formula content.

## Best Possible Solution

Add Feishu-plugin-local math-block handling that preserves LaTeX content for markdown formula inputs, backed by focused unit coverage and live Feishu proof, or explicitly document the limitation if the API cannot support it safely.

## Reproduction Assessment

No, not as a live end-to-end reproduction. Source inspection gives a credible path because current write/append/insert still pass markdown through Feishu `document.convert` without math-specific handling, but a real Feishu document run is still needed.

## Solution Assessment

Unclear until live Feishu behavior is confirmed. The likely best fix is Feishu plugin-local equation preservation, but maintainers need to choose the permanent strategy rather than queueing an unproven generic markdown change.
