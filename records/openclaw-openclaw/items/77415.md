## Summary

Keep open. The PR targets a real source-reproducible subagent lifecycle bug that is still missing on current `main`, but the branch is not merge-ready because it introduces a cleanup lease ordering bug and still only provides test-harness proof rather than real aborted-startup behavior proof.

## What This Changes

The PR adds first-progress/startup-failed subagent lifecycle handling across embedded runs, registry, gateway wait/session state, docs, changelog, and focused tests.

## Best Possible Solution

Land a consolidated lifecycle fix after moving the cleanup lease to the point where cleanup actually starts and adding redacted real-runtime proof from an observed aborted subagent startup.

## Reproduction Assessment

Yes, at source level: current main emits and consumes only start/end/error in the affected paths and maps aborted end events to timeout. I did not run a live aborted startup in this read-only review.

## Solution Assessment

No, not as currently written. The additive lifecycle direction is the right narrow fix, but the cleanup lease should be delayed until just before announce cleanup starts, and the external PR still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Move cleanup preclaim after the hook path:** `src/agents/subagent-registry-lifecycle.ts:789-796`
  - body: This preclaims `cleanupHandled` before later awaited lifecycle work, including the non-deferred ended hook. If that hook rejects for a no-completion run, `completeSubagentRun` exits before `startSubagentAnnounceCleanupFlow`, but the persisted record now says cleanup is already handled, blocking later announce/finalize cleanup. Claim the lease immediately before starting cleanup, after those awaited paths.
  - confidence: 0.87
