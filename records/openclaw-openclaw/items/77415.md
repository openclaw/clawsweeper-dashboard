## Summary

Keep open: the PR addresses a valid subagent startup-silence bug that is still absent from current main, but the branch still has a blocking lifecycle-consumer gap where `agent.wait` can lose the new startup failure signal.

## What This Changes

The PR adds first-progress and startup-failed embedded subagent lifecycle events, updates subagent announce/cleanup handling, and adds related docs, changelog, and agent lifecycle tests.

## Best Possible Solution

Land the lifecycle signal work after gateway wait/cache consumers preserve startup-failed as the terminal startup error across the startup-failed -> end sequence, with focused regression coverage for agent.wait.

## Reproduction Assessment

Yes, source-reproducible: PR head emits startup-failed and then end for an aborted no-progress run, while waitForAgentJob still ignores startup-failed in active waits and can treat the trailing end as ok. I did not run tests in this read-only review.

## Solution Assessment

No: additive first-progress/startup-failed phases are the right direction, but this branch is incomplete until the gateway wait path treats startup-failed as terminal and guards it from the follow-up end event.
