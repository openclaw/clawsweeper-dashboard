## Summary

Keep open with a blocking finding: the missing hook is a real embedded-runner bug, but this PR calls the finalize hook after embedded terminal output and lifecycle work may already have been flushed, so `revise` can be too late to prevent the first answer from being observed.

## What This Changes

The PR wires the existing `before_agent_finalize` plugin hook into embedded agent terminal success, adds hidden continuation retries for `revise`, and adds focused harness tests.

## Best Possible Solution

Move the finalize-hook decision to the pre-terminal embedded lifecycle boundary, or otherwise hold terminal output/lifecycle emission until the hook result is known, with a regression proving `revise` delivers only the revised answer.

## Reproduction Assessment

Yes from source inspection: current `main` has the hook helper and native relay path, but the embedded runner has no `before_agent_finalize` call. I did not run tests because this review is read-only.

## Solution Assessment

No. The hook belongs in the embedded terminal lifecycle, but this PR invokes it after the attempt has already had a chance to flush visible output and terminal lifecycle events.
