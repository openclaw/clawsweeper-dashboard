## Summary

Keep open. Current main still steers or enqueues active same-session messages without returning a channel-visible busy acknowledgement, and the closed related work does not replace the requested cross-channel busy-reply contract.

## What This Changes

Request an immediate session-busy reply with optional cancel/new-session guidance, status continuity, and stable busy reason metadata when a message arrives during an active same-session run.

## Best Possible Solution

Keep this as the canonical tracker and design a generic busy-ack payload/config contract across queue policy, runReplyAgent, dispatch, docs, and focused channel-safe tests.

## Reproduction Assessment

Yes, at source level: active same-session turns currently steer or enqueue and return undefined, and dispatch converts that into no outbound reply. I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers settle the cross-channel contract. The safer direction is a generic runner/dispatch busy-reply payload and config shape, not a one-channel bridge patch.
