## Summary

Keep this issue open. Current main still lacks the requested channel-visible busy acknowledgement: active same-session input is steered or enqueued and returns no final reply payload, while the related implementation PR is closed unmerged and the overlapping issue was closed in favor of this canonical tracker.

## What This Changes

Request to add immediate channel-visible session-busy status replies with optional cancel/new-session guidance for messages sent to locked active sessions.

## Best Possible Solution

Keep this as the canonical tracker and design one generic busy-ack contract across queue policy, runner return payloads, dispatch, config/docs, and channel-safe tests.

## Reproduction Assessment

Yes, at source level: active same-session turns currently steer or enqueue and return undefined, and dispatch converts that into no outbound final reply. I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers settle the cross-channel contract. The maintainable direction is a generic runner/dispatch payload and config shape that preserves locking, fallback, /stop, /new, collect, followup, steer, and heartbeat semantics.
