## Summary

Keep this issue open. Current main still lacks a channel-visible busy/status acknowledgement for active same-session input, and the related implementation PR is closed unmerged while the overlapping issue was closed in favor of this canonical tracker.

## What This Changes

Request an immediate cross-channel session-busy reply, optional cancel/new-session guidance, typing heartbeat, and stable busy reason metadata when a user messages a locked session.

## Best Possible Solution

Keep this as the canonical tracker and design one generic busy-ack contract across queue policy, runner return payloads, dispatch, config/docs, and focused channel-safe tests.

## Reproduction Assessment

Yes, at source level: active same-session turns can steer or enqueue and return undefined, and dispatch converts that into no outbound final reply. I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers settle the cross-channel contract. The maintainable direction is a generic runner/dispatch payload and config shape that preserves locking, fallback, /stop, /new, collect, followup, steer, and heartbeat semantics.
