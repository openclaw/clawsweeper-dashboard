## Summary

Keep open. Current main still queues active same-session messages without a channel-visible busy acknowledgement, and the overlapping implementation PR is open with review blockers rather than merged or shipped.

## What This Changes

The issue requests an immediate scripted busy/status reply, optional cancel/new-session guidance, heartbeat/status continuity, and busy reason metadata when a same-session message arrives during an active long-running response.

## Best Possible Solution

Define one generic busy-ack contract across queue policy, runner return payloads, dispatch, config/docs, and channel-safe tests, then reconcile or replace the overlapping implementation PR.

## Reproduction Assessment

Yes. Source inspection shows active same-session followup/steer/collect paths can resolve to enqueue-followup, runReplyAgent returns undefined, and dispatch maps undefined to no outbound replies; I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation path. The likely boundary is a generic queue/runner/dispatch contract, but defaults, config shape, cancel guidance, busy reason metadata, and the relationship to https://github.com/openclaw/openclaw/pull/39117 need maintainer design.
