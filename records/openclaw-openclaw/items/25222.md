## Summary

Keep open. Current main still lacks a generic channel-visible busy/status acknowledgement for active same-session messages, and the overlapping implementation PR is still open with review concerns, so this is not implemented or safely superseded.

## What This Changes

Request an immediate scripted busy/waiting reply with optional cancel/new-session guidance when a user messages a session that is already handling a long model or tool turn.

## Best Possible Solution

Define one generic busy-ack contract across queue policy, runner return payloads, dispatch, config/docs, and channel-safe tests, then reconcile or replace the overlapping implementation PR.

## Reproduction Assessment

Yes, from source inspection. Active same-session messages can resolve to enqueue-followup, runReplyAgent returns undefined on that path, and dispatch maps undefined to no outbound replies; I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation path. The likely boundary is a generic queue/runner/dispatch contract, but defaults, config shape, busy reason metadata, and the relationship to https://github.com/openclaw/openclaw/pull/39117 need maintainer design.
