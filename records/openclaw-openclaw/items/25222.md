## Summary

Keep open. Current main still enqueues active same-session messages without returning a channel-visible busy/status acknowledgement, and the related implementation PR remains open with unresolved review concerns rather than merged or shipped.

## What This Changes

The issue requests an immediate scripted busy-session reply, optional cancel or new-session guidance, and busy reason/status metadata when a message arrives while a session run is already active.

## Best Possible Solution

Define one generic active-session busy acknowledgement contract across queue policy, runner return payloads, dispatch, config/docs, and channel-safe tests, then reconcile or replace the overlapping implementation PR.

## Reproduction Assessment

Yes. Source inspection shows active same-session followup/steer/collect paths can resolve to enqueue-followup, runReplyAgent returns undefined, and dispatch maps undefined to no outbound replies; I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation path. The likely boundary is a generic queue/runner/dispatch contract, but defaults, config shape, cancel guidance, busy reason metadata, and the relationship to https://github.com/openclaw/openclaw/pull/39117 need maintainer design.
