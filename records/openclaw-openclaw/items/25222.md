## Summary

Keep this issue open. Current main still lacks a channel-visible busy acknowledgement for active same-session input: the queue path steers or enqueues and returns no final reply payload, while the related implementation PR is closed unmerged and the overlapping issue was closed in favor of this tracker.

## What This Changes

Request an immediate scripted busy/status reply, optional cancel or new-session guidance, heartbeat continuity, and stable busy reasons when a message arrives during an active same-session run.

## Best Possible Solution

Keep this issue as the canonical tracker and design one generic busy-ack contract across queue policy, runner return payloads, dispatch, config/docs, and focused channel-safe tests.

## Reproduction Assessment

Yes, at source level: active same-session turns currently steer or enqueue and return undefined, and dispatch converts that into no outbound final reply. I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers settle the cross-channel contract. The maintainable direction is a generic runner/dispatch payload and config shape that preserves locking, fallback, /stop, /new, collect, followup, steer, and heartbeat semantics.
