## Summary

Current main still lacks the requested visible busy/status acknowledgement: active same-session turns are steered or enqueued without returning a channel reply, and there is no generic cancel/new-session guidance or busy reason contract. The issue should remain open because the remaining work is a cross-channel feature/config design, not a stale cleanup candidate.

## What This Changes

Add a scripted session-busy/status reply, optional cancel/new-session guidance, heartbeat behavior, and busy reason metadata when a user messages a locked long-running session.

## Best Possible Solution

Keep this as the canonical tracker and design one generic busy-ack contract across queue policy, runner return payloads, dispatch, config/docs, and focused channel-safe tests.

## Reproduction Assessment

Yes, at source level: active same-session turns currently steer or enqueue and return undefined, and dispatch converts that into no outbound reply. I did not run a live chat-channel reproduction in this read-only review.

## Solution Assessment

Unclear as an implementation path until maintainers settle the cross-channel contract. A narrow fix should add a generic runner/dispatch payload and config shape without changing locking, fallback, /stop, /new, collect, followup, steer, or heartbeat semantics accidentally.
