## Summary

This PR remains necessary: current main still loses cron-tool wake origin and defaults no-target heartbeat delivery to suppression. The patch uses the existing session, gateway target, session-store delivery-context, and heartbeat turn-source seams, and the supplied real Telegram proof demonstrates the fix. No discrete correctness or security defect remains; maintainers should explicitly accept the compatibility-sensitive absent-target origin fallback before landing.

## What This Changes

This PR routes cron wake calls through the originating session and agent, carries stored channel delivery context into the wake event, and permits that explicit origin to receive the heartbeat reply when no heartbeat target is configured.

## Best Possible Solution

Use the branch’s canonical origin propagation and explicitly approve the rule that event-carried origin delivery applies only when heartbeat target is absent, while preserving explicit target:none and no-origin suppression.

## Solution Assessment

Yes, with a maintainer compatibility decision. The implementation reuses canonical session and delivery-context ownership rather than adding a competing route; the only policy choice is whether absent heartbeat target should yield to an explicit event origin.
