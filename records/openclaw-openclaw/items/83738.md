## Summary

Keep this PR open: it targets a real cron wake routing bug, but as submitted it can misroute explicit cross-agent sessionKey wakes and still only provides synthetic harness proof rather than a real non-main channel/CLI wake path.

## What This Changes

The branch threads optional sessionKey and agentId through the cron wake tool, gateway wake schema/handler, cron service enqueue/heartbeat routing, and adds regression tests plus a proof harness.

## Best Possible Solution

Land the focused origin-capture design only after explicit sessionKey precedence is fixed, regression coverage proves cross-agent explicit targeting, and redacted live route proof shows a non-main wake returning to the originating lane.

## Reproduction Assessment

Yes by source inspection: current main's cron tool wake call drops origin metadata, and the PR's explicit-sessionKey path can be traced into the existing agent/session mismatch fallback. I did not run tests because this review is read-only.

## Solution Assessment

No as submitted: threading session and agent metadata through the existing wake path is the right narrow direction, but the tool-level precedence bug and synthetic-only proof block merge. The safer fix is to honor explicit sessionKey independently, add a regression test for that case, and prove the real route.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.86

Full review comments:

- **[P1] Do not pair explicit session keys with the inferred agent:** `src/agents/tools/cron-tool.ts:847`
  - body: The gateway cron adapter falls back to the requested agent's main session when a supplied `sessionKey` belongs to another agent. Because this line still sends the caller's inferred `agentId` whenever `sessionKey` is explicit but `agentId` is omitted, `cron({ action: "wake", sessionKey: "agent:coding:..." })` from `main` is rewritten back to `main` instead of honoring the explicit target. Only infer `agentId` when no explicit `sessionKey` is supplied, or derive it from that key.
  - confidence: 0.89
