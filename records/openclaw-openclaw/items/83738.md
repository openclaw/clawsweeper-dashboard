## Summary

The branch fixes a source-confirmed cron wake routing bug and the primary Telegram topic behavior is convincingly demonstrated, but the final commit broadens unset heartbeat-target delivery beyond cron wake and still lacks real current-head proof, so owner confirmation and exact-head validation remain merge gates.

## What This Changes

The PR routes cron wake requests to the originating agent and session, carries stored channel delivery context, extends the wake protocol with `agentId`, and delivers origin-carrying heartbeat replies when no target is configured.

## Best Possible Solution

Keep the canonical agent/session and stored delivery-context routing, preserve explicit `target: "none"`, and merge only after a current-head live Telegram run proves omitted-target delivery and the outbound/heartbeat owner confirms that explicit origin context should override historical unset-target suppression for all affected event producers.
