## Summary

Keep open: this is a useful cron/gateway routing fix, but it is not merge-ready because agentId-only wake overrides can still carry the inferred current session key, and the supplied proof is a synthetic harness rather than live route proof.

## What This Changes

The branch threads optional sessionKey and agentId through the cron agent tool, gateway wake schema/handler, cron service wake path, regression tests, and a synthetic proof script.

## Best Possible Solution

Keep the origin-capture direction, make target precedence unambiguous for agentId-only calls, and land only after redacted live route proof confirms the non-main wake path.

Yes, source inspection gives a high-confidence reproduction path: current main's cron tool wake sends only {mode, text}, and the gateway cron target resolver falls back when no origin is supplied. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

No, not yet. The implementation uses the right gateway and cron-service seam, but the agentId-only override semantics need a small fix and the transport behavior still needs live proof before merge.
