## Summary

Keep open: current main still drops orphan Anthropic-compatible text deltas before they become assistant content, and the Kimi provider still routes through that accumulator on the checked-out branch.

## What This Changes

The issue requests preserving Kimi/Anthropic-compatible `content_block_delta` text when no matching `content_block_start` has been accumulated, preventing an empty terminal result and `incomplete_result` fallback.

## Best Possible Solution

Add a defensive text-block recovery path in `src/agents/anthropic-transport-stream.ts` with a focused mocked-SSE regression test, while keeping Kimi endpoint-routing decisions separate from this accumulator fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked SSE stream with `message_start`, a `content_block_delta` text event for index 0 without a prior `content_block_start`, `content_block_stop`, and a stop `message_delta`; current main will skip the delta and leave `content` empty.

## Solution Assessment

Yes. The suggested accumulator fallback is the narrowest maintainable fix because it preserves visible text from valid provider deltas without relying on a Kimi-only transport switch or changing unrelated provider behavior.
