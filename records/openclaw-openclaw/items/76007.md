## Summary

Keep open: current main still drops Anthropic-compatible text deltas when the matching `content_block_start` was not accumulated, and the bundled Kimi provider still routes through that Anthropic messages stream path.

## What This Changes

The issue requests preserving Kimi Anthropic-style streaming text deltas so completed responses do not end as empty content and get classified as incomplete.

## Best Possible Solution

Add a defensive recovery path in the Anthropic messages stream accumulator that creates a text block and emits `text_start` before appending an orphan `text_delta`, with a focused regression test.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked Anthropic SSE stream with `message_start`, `content_block_delta` text for index 0 without a prior `content_block_start`, `content_block_stop`, and a stop `message_delta`; current main skips the delta and leaves assistant content empty.

## Solution Assessment

Yes. The suggested accumulator fallback is the narrowest maintainable fix because it preserves valid provider text without changing Kimi endpoint selection or unrelated provider policy.

## Repair Work Prompt

Fix issue #76007 by hardening the Anthropic messages stream accumulator against orphan text deltas. In `src/agents/anthropic-transport-stream.ts`, when a `content_block_delta` has `delta.type === "text_delta"` and a string `delta.text` but no matching existing block for `event.index`, create a text `TransportContentBlock`, push it into `output.content`, emit `text_start`, append/sanitize the delta consistently with existing text handling, and allow the later `content_block_stop` to emit `text_end`. Add a focused mocked-SSE regression test in `src/agents/anthropic-transport-stream.test.ts` that currently would end with `content: []` and proves the result contains the streamed text plus `text_start`, `text_delta`, and `text_end`. Add a user-facing `CHANGELOG.md` fix entry. Keep Kimi endpoint/API selection separate; do not implement PR #76002's image endpoint change in this fix.
