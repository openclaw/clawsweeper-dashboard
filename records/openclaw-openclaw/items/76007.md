## Summary

Keep open: current main still drops orphan Anthropic Messages text deltas before they become assistant content, and the bundled Kimi Coding provider still routes through that accumulator.

## What This Changes

Preserve Kimi/Anthropic-compatible text deltas when `content_block_delta` arrives without a prior matching `content_block_start`, so successful replies do not end as `content: []` / `incomplete_result`.

## Best Possible Solution

Add a defensive text-block creation path in the Anthropic Messages accumulator for orphan `text_delta` events, with a mocked SSE regression test and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked SSE stream with `message_start`, `content_block_delta` text for index 0 without `content_block_start`, `content_block_stop`, and a stop `message_delta`; current code leaves `output.content` empty.

## Solution Assessment

Yes. The accumulator fallback is the narrowest maintainable fix because it preserves visible text for Kimi and other Anthropic-compatible streams without conflating this bug with Kimi endpoint-selection work.
