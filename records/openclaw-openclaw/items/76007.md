## Summary

Keep open: current main still drops orphan Anthropic-compatible `text_delta` events before they become assistant content, and Kimi Code still defaults through that Anthropic messages accumulator. The report is narrow, source-backed, and suitable for a focused regression fix.

## What This Changes

Preserve Kimi Code streaming text when a `content_block_delta` arrives without a matching Anthropic `content_block_start`, avoiding an empty assistant `content` result and `incomplete_result` classification.

## Best Possible Solution

Add a defensive text-block recovery path in `src/agents/anthropic-transport-stream.ts` with a focused mocked-SSE regression test, while keeping Kimi endpoint selection and runtime-deps repair as separate concerns.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked Anthropic SSE stream with `message_start`, a `content_block_delta` text event for index 0 without a prior `content_block_start`, `content_block_stop`, and a stop `message_delta`; current main skips the text and leaves `content` empty.

## Solution Assessment

Yes. The requested accumulator fallback is the narrowest maintainable fix because it preserves visible text from provider deltas without changing Kimi-only routing or unrelated provider contracts.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/76007. Current main drops Anthropic-compatible `content_block_delta` text when no matching `content_block_start` was recorded, which can make Kimi Code sessions end with `content: []` and `incomplete_result` despite visible upstream SSE text. Implement a defensive fallback in `src/agents/anthropic-transport-stream.ts`: when handling `content_block_delta`, if no matching block exists and `delta.type === "text_delta"` with string text, create a text content block at the event index, emit the corresponding stream start event before appending text, then continue through the existing text delta path. Keep the change generic to the Anthropic messages accumulator; do not switch Kimi endpoints, change provider catalog behavior, or address plugin-runtime-deps repair in this PR. Add a focused regression test in `src/agents/anthropic-transport-stream.test.ts` using a mocked SSE sequence with an orphan text delta and assert final assistant text/content plus stream events. Add a user-facing `CHANGELOG.md` fix entry because this affects provider responses. Validate with targeted formatter check and the Anthropic stream test; per repo policy, run `pnpm check:changed` in Blacksmith Testbox before handoff.
