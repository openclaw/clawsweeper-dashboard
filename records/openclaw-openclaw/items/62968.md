The broader queue-first completion PR overlaps this area, but it is still open, draft, and dirty, so it is not a safe reason to close this issue.

## What This Changes

The issue requests surfacing a warning when a failed direct subagent completion announce is recovered by fallback delivery so operators can see the recovered direct-send failure.

## Best Possible Solution

Add a narrow warning for failed direct completion phases that are recovered by fallback, or fold equivalent phase-aware diagnostics into the queue-first completion work if maintainers choose to land that path.
