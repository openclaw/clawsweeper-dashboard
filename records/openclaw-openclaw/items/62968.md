## Summary

Keep open: current main and latest release still preserve a recovered direct-send failure only in `delivery.phases`, while the caller logs only top-level failed direct delivery. The overlapping queue-first completion PR is open, draft, and conflicting, so it is not a close-safe replacement for this narrow diagnostics bug.

## What This Changes

The issue asks for warning-level logging when subagent announce direct delivery fails but fallback steering/queue delivery succeeds.

## Best Possible Solution

Add a phase-aware warning for failed direct completion phases that are recovered by fallback, or fold the same diagnostic into the queue-first completion work if maintainers land that path.

## Reproduction Assessment

Yes. Source inspection shows a failed `direct-primary` phase can be followed by a delivered fallback result, while the caller only logs top-level failed direct delivery; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as a standalone landing path. A narrow phase-aware warning is the smallest fix on current main, but maintainers should decide whether to land it separately or inside the overlapping queue-first completion PR.
