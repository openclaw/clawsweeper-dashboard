## Summary

Keep open. The report matches OpenClaw's silent-reply contract and describes a privacy-impacting Telegram leak, but current source inspection still does not isolate the exact path that turns a successful empty agent result into user-visible raw run-envelope JSON.

## What This Changes

The issue reports that Telegram sends the entire successful agent run envelope when a final assistant reply is `NO_REPLY`, instead of suppressing the channel reply.

## Best Possible Solution

Preserve the documented silent-reply behavior and add focused Telegram/gateway regression coverage proving that successful empty or silent agent results cannot be adapted into user-visible run metadata.

## Reproduction Assessment

Unclear. The issue includes concrete live evidence from OpenClaw 2026.5.6, but I did not reproduce it on current main and source inspection did not reveal the exact Telegram branch that sends the gateway envelope as text.

## Solution Assessment

Unclear because there is no proposed patch and the root path is not isolated. The likely best fix is a narrow delivery-boundary guard plus regression test, not a broad change to `NO_REPLY` policy.
