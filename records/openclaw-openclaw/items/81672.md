## Summary

Keep open for maintainer review: the PR is a focused source-scoped bug fix with sufficient real loader proof and no blocking code/security finding, but it changes the managed/personal skill-root trust boundary and has one low-priority docs alignment gap.

## What This Changes

The PR changes the skills loader, tests, docs, and changelog so managed and personal skill roots can load skill-directory symlinks while preserving stricter containment for workspace, extra, project, bundled, and per-skill SKILL.md paths.

## Best Possible Solution

Land the source-scoped loader fix after maintainer trust-boundary approval and align the remaining symlink troubleshooting guidance with the new managed/personal exception.

## Reproduction Assessment

Yes, source-based. On current main, managed and personal skill roots go through `resolveContainedSkillPath()`, so a skill-directory symlink whose realpath leaves that root is rejected unless the target is explicitly allowlisted.

## Solution Assessment

Yes, with one documentation cleanup. The PR is narrower than broadening the global allowlist because it separates root-directory trust by source and keeps every `SKILL.md` checked against its resolved skill directory.
