## Summary

Keep open: current main still routes managed and personal skill roots through the same realpath containment path that rejects this ClawHub/local-manager symlink layout, while this PR carries the focused source-scoped fix with docs, regression coverage, and no blocking review findings.

## What This Changes

The PR updates the skill loader, tests, docs, and changelog so managed/personal skill roots accept directory symlink targets while workspace/project/extra/bundled roots and per-skill `SKILL.md` containment remain guarded.

## Best Possible Solution

Land this source-scoped loader fix after normal CI and maintainer trust-boundary approval, preserving `allowSymlinkTargets` for workspace/project/extra roots and per-skill `SKILL.md` containment for every source.

## Reproduction Assessment

Yes, source-based. On current main, a managed or personal skill-directory symlink whose realpath leaves its root reaches `resolveContainedSkillPath()` and is rejected unless the target is explicitly allowlisted.

## Solution Assessment

Yes, assuming maintainers accept the managed/personal root trust policy. The PR composes the narrow source-scoped root exemption with the existing allowlist path and keeps the per-skill `SKILL.md` containment check fail-closed.
