Fix the custom OpenAI-compatible onboarding bug from https://github.com/openclaw/openclaw/issues/44870. Current main can manually run custom providers with `api: "openai-responses"`, but interactive and non-interactive custom onboarding can only choose `openai|anthropic`, verifies non-Azure OpenAI-compatible endpoints via `/chat/completions`, and persists `openai-completions`.
