## Summary

Keep open. The reported Zalouser mid-word splitting bug is real, but this PR only changes a helper fallback that production reply paths bypass; normal monitor and raw outbound sends still pass the shared `length` default explicitly, and the user-facing fix also needs a changelog entry.

## What This Changes

The PR changes the Zalouser send helper default chunk mode to `newline` and adds a helper-level regression test for paragraph-boundary splitting.

## Best Possible Solution

Wire Zalouser’s production reply and raw outbound defaults to natural-boundary chunking while preserving explicit `chunkMode` overrides, then cover those paths and add the user-facing changelog entry.

## Reproduction Assessment

Yes, by source inspection. Current main’s normal Zalouser monitor and outbound adapter resolve `chunkMode` to the shared `length` default and pass it explicitly, so long replies still hard-cut unless configured otherwise.

## Solution Assessment

No. Changing only the helper fallback is too narrow because runtime delivery bypasses it; the fix should update the Zalouser delivery default or resolver path while preserving explicit config overrides.
