## Summary

Keep open. The PR targets a real Zalouser long-message chunking bug, but the patch is too narrow because normal monitor replies and raw outbound sends already pass an explicit shared default of `length`, so changing only the helper fallback does not fix ordinary bot delivery; it also lacks the required user-facing changelog entry.

## What This Changes

The PR changes Zalouser's send helper default text chunk mode from `length` to `newline` and adds a regression test for paragraph-boundary splitting.

## Best Possible Solution

Make Zalouser's real delivery paths default to natural-boundary chunking when no explicit Zalouser override is configured, keep configured overrides intact, cover monitor/outbound paths, and add the changelog entry.

## Reproduction Assessment

Yes, by source inspection. With normal config, `resolveChunkMode(config, "zalouser", accountId)` returns the shared `length` default and the monitor/outbound paths pass that value into `sendMessageZalouser`, so a helper-only fallback change does not affect ordinary bot replies.

## Solution Assessment

No. The PR changes the right helper behavior for direct omitted-option callers, but it is not the best complete fix because the production Zalouser delivery paths bypass that fallback; the safer fix is a Zalouser-specific default at the resolver/call-site boundary plus coverage.
