## Summary

Keep open: the reported Zalouser long-message split problem is plausible, but this PR is incomplete because current main’s normal monitor and outbound delivery paths pass an explicit shared chunk mode that still defaults to `length`. The PR also has only unit-test proof, not after-fix real Zalouser delivery proof.

## What This Changes

The PR changes the Zalouser send helper default text chunk mode from `length` to `newline` and adds a helper-level regression test for paragraph-boundary splitting.

## Best Possible Solution

Make Zalouser production reply and outbound adapter delivery default to natural-boundary chunking while preserving explicit config overrides, then cover those runtime paths with focused regression tests.

## Reproduction Assessment

Yes, by source inspection. With normal config, current main resolves the Zalouser chunk mode to the shared default `length` and passes it explicitly into `sendMessageZalouser`, so changing only the helper fallback does not affect ordinary long bot replies.

## Solution Assessment

No. The PR changes a fallback used only when callers omit `textChunkMode`; the maintainable fix needs to update the Zalouser runtime delivery default/resolver path and preserve explicit overrides.
