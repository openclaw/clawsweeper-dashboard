## What This Changes

The PR changes the Zalouser send helper default from `length` to `newline` and adds a unit test for paragraph-boundary splitting.

## Best Possible Solution

Align the Zalouser delivery default at the resolver/runtime boundary, update production-path tests and docs for the chosen default, and preserve explicit `chunkMode` overrides.

## Reproduction Assessment

Yes, source-reproducible: normal Zalouser monitor delivery resolves the shared `length` default and passes it explicitly to `sendMessageZalouser`, whose length mode hard-cuts at the limit. I did not run a live Zalo account in this read-only review.

## Solution Assessment

No. The PR is a plausible helper-level mitigation, but the best fix must update the production resolver/default path or explicitly document that only helper/tool sends change.
