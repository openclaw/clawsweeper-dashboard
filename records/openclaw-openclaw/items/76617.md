## Summary

Keep open: the PR targets a real Zalouser readability bug, but it is not merge-ready because it changes only the send-helper fallback while normal replies and resolver-backed outbound sends still use the shared `length` default, and real Zalouser after-fix proof is still missing.

## What This Changes

The PR changes Zalouser's send-helper default text chunk mode from `length` to `newline` and adds a helper-level regression test for paragraph-boundary splitting.

## Best Possible Solution

Align the Zalouser default at the shared resolver/runtime boundary, preserve explicit `chunkMode` overrides, add production-path regression coverage, and require redacted real Zalouser proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current `main` returns `length` from the shared resolver when no Zalouser override is configured, and normal replies pass that into `sendMessageZalouser`. I did not run a live Zalouser account in this read-only review.

## Solution Assessment

No. The helper-level change is a plausible mitigation, but the best fix aligns the shared runtime/resolver default used by normal replies and outbound sends, preserves explicit overrides, and covers that production path in tests.
