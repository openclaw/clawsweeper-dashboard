## What This Changes

The PR changes the Zalouser send helper default text chunk mode from `length` to `newline` and adds a helper-level regression test for paragraph-boundary splitting.

## Best Possible Solution

Align the Zalouser default at the shared runtime/resolver boundary, preserve explicit `chunkMode` overrides, add production-path regression coverage, and require redacted real Zalouser proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main defaults to `length`, and normal Zalouser replies/outbound sends pass that explicit shared value into `sendMessageZalouser`; I did not run a live Zalouser account in this read-only review.

## Solution Assessment

No. The helper fallback change is a plausible mitigation for direct helper callers, but the best fix is to align the shared runtime/resolver default used by normal replies and outbound sends while preserving explicit overrides.
