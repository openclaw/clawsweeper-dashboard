The PR changes the Zalouser send helper default text chunk mode from `length` to `newline` and adds a unit test for paragraph-boundary splitting.

## Best Possible Solution

Align the Zalouser default at the shared resolver/runtime boundary, update production-path tests and any relevant docs for the chosen default, preserve explicit `chunkMode` overrides, and add redacted real Zalouser proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main's shared chunk resolver defaults to `length`, and normal Zalouser delivery passes that explicit mode into `sendMessageZalouser`. I did not run a live Zalo account in this read-only review.

## Solution Assessment

No. The PR is a plausible helper-level mitigation, but the best fix should align the resolver/runtime default and production-path tests, or explicitly document that only helper/tool sends change.
