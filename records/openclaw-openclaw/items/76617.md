## Summary

Keep open: the PR targets a real Zalouser readability problem, but the patch is too narrow because it changes only the helper fallback while current main and the latest release still send normal replies and resolver-backed outbound messages with the shared `length` chunk-mode default; it also still lacks real Zalouser behavior proof.

## What This Changes

The PR changes Zalouser's `sendMessageZalouser` fallback chunk mode from `length` to `newline` and adds a helper-level regression test for paragraph-boundary splitting.

## Best Possible Solution

Align the Zalouser default at the shared resolver/runtime boundary, preserve explicit `chunkMode` overrides, update production-path tests, and add redacted real Zalouser proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main's shared chunk resolver defaults to `length`, and normal Zalouser delivery passes that explicit mode into `sendMessageZalouser`. I did not run a live Zalouser account in this read-only review.

## Solution Assessment

No. The PR is a plausible helper-level mitigation, but the best fix should align the resolver/runtime default and production-path tests, or explicitly document that only helper/tool sends change.
