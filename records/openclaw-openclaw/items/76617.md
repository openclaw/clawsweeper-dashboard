## Summary

Keep open. The PR is useful but incomplete: it changes the helper fallback, while the normal Zalouser reply paths still pass the shared default `length` mode explicitly, so the reported bot-response behavior remains on current `main`.

## What This Changes

The PR changes Zalouser's send helper default text chunk mode from `length` to `newline` and adds a regression test for paragraph-boundary splitting.

## Best Possible Solution

Make Zalouser's production reply and outbound-adapter paths default to natural-boundary chunking, preserve any explicit override semantics, add coverage for those paths, and include the required changelog entry.

## Reproduction Assessment

Yes, by source inspection. With normal config, `resolveChunkMode(cfg, "zalouser", accountId)` returns `length`, and the monitor/outbound paths pass that value into `sendMessageZalouser`, which makes the helper default change ineffective for ordinary bot replies.

## Solution Assessment

No. Changing `DEFAULT_TEXT_CHUNK_MODE` is too narrow because the current production paths bypass that fallback; the fix should wire the Zalouser default through the actual delivery call sites or a Zalouser-specific resolver.
