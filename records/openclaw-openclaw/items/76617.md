## Summary

Keep open. The PR is aimed at the right Zalouser readability problem, but the current patch only changes the send-helper fallback while ordinary reply and outbound paths pass an explicit shared chunk mode that still defaults to `length`.

## What This Changes

The PR changes the Zalouser send helper default chunk mode from `length` to `newline` and adds a send-helper regression test for paragraph-boundary splitting.

## Best Possible Solution

Wire Zalouser’s production reply and outbound adapter defaults to natural-boundary chunking while preserving explicit `chunkMode` overrides, then cover those paths and document the user-facing fix in the changelog.

## Reproduction Assessment

Yes, by source inspection. With normal config, the monitor and outbound adapter resolve the shared chunk mode to `length` and pass it explicitly to `sendMessageZalouser`, so the helper fallback change does not affect ordinary bot replies.

## Solution Assessment

No. Changing `DEFAULT_TEXT_CHUNK_MODE` is too narrow because the runtime delivery call sites bypass that fallback; the maintainable fix should update the Zalouser-specific delivery default or resolver path and add coverage there.
