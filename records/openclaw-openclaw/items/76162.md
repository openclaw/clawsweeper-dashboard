## What This Changes

The issue asks for exec approval follow-up CLI tasks whose embedded run has ended to be reconciled instead of being kept running by a persistent parent session row.

## Best Possible Solution

Land a narrow task-maintenance fix that treats run-backed CLI follow-ups as unbacked once their run context disappears, with regression coverage for the `agent:main:main` exec follow-up case.

## Reproduction Assessment

Yes, source-reproducible: seed a stale `runtime: "cli"` task with inactive `sourceId`/`runId`, `childSessionKey: "agent:main:main"`, and a matching session-store row. Current `hasBackingSession` can still return true after the grace period, preventing `shouldMarkLost` from reconciling it.

## Solution Assessment

Yes, the best fix belongs in task-maintenance CLI liveness. The implementation should make run-backed follow-ups depend on live run context without broadly removing intended child-session fallback behavior for unrelated CLI records.
