## Summary

Keep open. Current main still has the reported CLI task-maintenance gap, and the related repair PRs #76199 and #76656 are closed unmerged, so this should become a narrow fix PR rather than a cleanup close.

## What This Changes

Request to reconcile stale `exec-approval-followup` CLI task runs after their embedded gateway agent run has ended instead of treating the persistent parent session as live backing.

## Best Possible Solution

Land a narrow task-maintenance fix that treats run-backed CLI follow-ups as unbacked once their live run context is gone, with regression coverage for `agent:main:main` and no new config surface.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: seed a stale active `runtime: "cli"` task with inactive `sourceId`/`runId`, `childSessionKey: "agent:main:main"`, and a matching session-store row; current maintenance will keep it running after the grace period.

## Solution Assessment

Yes, with a scope constraint. The best fix belongs in `hasBackingSession`, making run-backed CLI follow-ups depend on live run context while preserving documented child-session behavior for CLI records without run metadata.
