## Summary

Keep open: current main still has the `agents add` optimistic-write race path, and the linked fix PR is open, unmerged, and previously reviewed as needing corrections before merge.

## What This Changes

Requests concurrency-safe `openclaw agents add` config writes so simultaneous agent creation does not lose agents or surface raw `ConfigMutationConflictError`.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/76601 that retries `ConfigMutationConflictError` from a fresh valid snapshot, preserves duplicate and binding checks, adds regression coverage and changelog, then close this issue and update https://github.com/openclaw/openclaw/issues/43367 as appropriate.

## Reproduction Assessment

Yes, at source level. Current main reads the config hash once for `agents add` and the config writer throws `ConfigMutationConflictError` when another process changes the hash before commit; I did not run a live race because this review is read-only.

## Solution Assessment

Yes, retrying after a config mutation conflict is the narrow maintainable direction and matches the configure wizard precedent. The current linked PR needs source corrections and after-fix proof before it is the mergeable solution.
