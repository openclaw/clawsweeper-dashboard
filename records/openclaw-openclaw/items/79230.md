## Summary

Keep open: current main still has the `agents add` config-race path, and the same-author fix at https://github.com/openclaw/openclaw/pull/76601 is still open rather than merged.

## What This Changes

The issue reports that concurrent `openclaw agents add` invocations race on the config file, losing agent entries or surfacing `ConfigMutationConflictError`.

## Best Possible Solution

Land a corrected `agents add` retry implementation that rebuilds from a fresh valid config snapshot on conflict, preserves binding outcomes, adds regression coverage, and then close this issue with the merged fix.

## Reproduction Assessment

Yes, at source level: current main reads one config snapshot/base hash, then commits with that same hash while the writer rejects stale hashes with `ConfigMutationConflictError`. I did not run the parallel CLI repro because this review is read-only.

## Solution Assessment

Yes for the proposed direction, but not by closing this issue now. The narrow maintainable solution is retry handling around `agents add` config commits, with the paired PR repaired and validated before merge.
