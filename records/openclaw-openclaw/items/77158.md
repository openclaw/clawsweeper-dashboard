## Summary

Keep this PR open: the performance problem is real from source inspection, but the proposed persistent fast path needs one correctness fix and changelog coverage before merge.

## What This Changes

The PR persists QMD session export state in `.export-state.json`, skips unchanged session exports by source size/mtime, and adds QMD manager tests for cache reload and exportDir invalidation.

## Best Possible Solution

Land the QMD export cache with a fast path that verifies the cached markdown target still exists, plus focused regression coverage and a release-note entry.

## Reproduction Assessment

No high-confidence live benchmark was run. Source inspection on current main shows `exportSessions()` rebuilds every session entry before the write-skip check, so the expensive path is source-reproducible even though the exact performance numbers remain unverified.

## Solution Assessment

No, not as currently written. A versioned persistent cache plus stat fast path is a narrow fit, but it should fall through to the slow path when the cached markdown target is missing and it needs changelog coverage.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Verify the cached export still exists:** `extensions/memory-core/src/memory/qmd-manager.ts:2238-2244`
  - body: After loading a valid `.export-state.json`, this fast path skips rebuilding solely because the source JSONL size and mtime match. Check that the cached markdown target exists before continuing, or fall through to the slow path.
