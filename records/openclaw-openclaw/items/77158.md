## What This Changes

The PR persists QMD session export state in `.export-state.json`, adds a source size/mtime fast path before `buildSessionEntry()`, and updates QMD tests, changelog, and the generated plugin SDK baseline hash.

## Best Possible Solution

Land the QMD-local export-state cache after maintainer approval, keeping the slow render/redaction path authoritative when source stats change, cache metadata is invalid, or cached targets are missing.

## Reproduction Assessment

Yes, at source level: current main calls `buildSessionEntry()` for every listed session before it can skip a write, and that helper reads/parses/redacts/hashes the JSONL. The PR also supplies live after-fix output showing fast-path reuse and missing-target rebuild behavior.

## Solution Assessment

Yes: the implementation keeps the cache local to the QMD exporter, invalidates it on schema/render/exportDir mismatch, verifies cached targets before skipping, and falls back to the existing slow path for changed or missing exports.
