## Summary

Keep open: this is a focused QMD performance PR that is not implemented on current main, has a source-backed hot path, sufficient after-fix live proof, focused tests, and no blocking functional or security findings from this review.

## What This Changes

The PR persists QMD session-export state and adds a size/mtime fast path so unchanged transcript exports can skip the full entry-build pipeline across boots and interval syncs.

## Best Possible Solution

Land the QMD-local export-state cache with its focused tests after maintainer review, keeping the existing slow render/redaction path authoritative whenever source stats change or cached targets are missing.

## Reproduction Assessment

Yes, at source level: current main calls `buildSessionEntry()` for every listed transcript before it can skip markdown writes, and that helper reads, parses, redacts, hashes, and renders the JSONL data. The PR also supplies live after-fix output for the cache reuse and missing-target rebuild behavior.

## Solution Assessment

Yes: the patch keeps the cache inside the QMD exporter, invalidates it on schema/render/exportDir mismatch, verifies the cached target before skipping, and falls back to the existing slow path for changed or missing exports.
