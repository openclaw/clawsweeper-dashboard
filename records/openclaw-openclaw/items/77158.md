## Summary

Keep this PR open for maintainer merge handling: current main still rebuilds every session export before checking whether the markdown write can be skipped, while the PR provides a focused QMD-local cache, regression coverage, and sufficient live runtime proof with no blocking review findings.

## What This Changes

The branch adds a persistent QMD session-export state file, a size/mtime fast path before `buildSessionEntry()`, focused cache tests, and a changelog entry.

## Best Possible Solution

Land the QMD-local cache after required CI passes and maintainers accept the bounded size/mtime cache tradeoff, keeping the slow render/redaction path authoritative on stat changes, invalid cache metadata, or missing targets.

## Reproduction Assessment

Yes. Current main is source-reproducible because `exportSessions()` calls `buildSessionEntry()` for each listed transcript before it can skip a markdown write, and `buildSessionEntry()` performs the expensive transcript read/parse/render/hash path.

## Solution Assessment

Yes. The PR keeps the cache inside the QMD exporter, invalidates persisted state by schema/render version/exportDir, checks that cached markdown still exists before skipping, and uses the existing slow path whenever source stats change or cache metadata is invalid.
