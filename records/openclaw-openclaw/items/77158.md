## Summary

Keep open: current main still rebuilds every listed QMD session entry before it can skip writes, while this PR adds a focused QMD-local persistent cache, target-existence guard, tests, changelog entry, and real-runtime proof. I found no blocking correctness or security findings in the proposed patch.

## What This Changes

The PR persists QMD session export state to `.export-state.json`, uses source size/mtime to skip rebuilding unchanged session markdown, adds cache regression tests, updates the changelog, and includes a no-op import reorder.

## Best Possible Solution

Land the QMD-local persistent cache and stat fast path after normal branch, CI, and maintainer gates, keeping the slow render/redaction path as the canonical correctness path.

## Reproduction Assessment

Yes for the source-level performance path: current main calls `buildSessionEntry()` for every listed session file, and that helper reads and parses the JSONL transcript. I did not run a production-sized timing benchmark in this read-only review.

## Solution Assessment

Yes. The PR keeps the optimization inside the QMD exporter, invalidates by state format/render version/export directory, verifies the cached markdown target exists before skipping, and leaves the slow rendering/redaction path unchanged.
