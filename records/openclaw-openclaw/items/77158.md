## Summary

This PR is still useful but should stay open: current main has the expensive QMD session export path the PR targets, while the proposed persistent cache needs one correctness fix plus release-note coverage before merge.

## What This Changes

The PR persists QMD session export state in `.export-state.json`, adds a size/mtime fast path before rebuilding unchanged session Markdown, and adds QMD manager tests for cache reload and exportDir invalidation.

## Best Possible Solution

Land a focused QMD export cache that falls back to rebuilding when the cached Markdown target is missing, includes regression coverage for that case, and documents the perf improvement in the changelog.

## Reproduction Assessment

No high-confidence live performance reproduction was run. Source inspection does show current main rebuilds every listed session entry before the write-skip check, which explains why large transcript stores are expensive.

## Solution Assessment

No, not as written. The persistent cache plus stat fast path is a good fit, but it should verify the exported Markdown target exists before skipping rebuild and should add the required changelog entry.

- **[P2] Verify exports exist before taking the cache fast path:** `extensions/memory-core/src/memory/qmd-manager.ts:2243-2248`
