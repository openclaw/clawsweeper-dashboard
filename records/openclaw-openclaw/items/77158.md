## Best Possible Solution

Land the QMD-local export-state cache after green current-head checks and maintainer acceptance of the stat-based freshness tradeoff, keeping the slow render/redaction path authoritative whenever source stats change, cache metadata is invalid, or the cached target is missing.

## Reproduction Assessment

## Solution Assessment

Yes: the proposed cache stays inside the QMD exporter, invalidates on format/render/exportDir mismatch, verifies the cached target before skipping, and falls back to the existing slow path for changed or missing exports.
