- `P2`: The optional QMD performance improvement has limited blast radius, but its persistent-state and freshness contracts need resolution before merge.
- `merge-risk: 🚨 compatibility`: The PR adds a durable sidecar and versioned cache contract with backup, cleanup, corruption, and upgrade implications.

## Summary

The core performance problem is credible and the supplied live proof is strong, but the branch establishes two unsafe permanent contracts: noncanonical persistent cache storage and transcript freshness based only on size and mtime. Both require QMD owner direction before merge.

## What This Changes

The PR persists QMD session-export metadata across restarts and skips transcript parsing, redaction, hashing, and rendering when cached source size and mtime match.

## Best Possible Solution

Keep the demonstrated performance improvement only with lifecycle-owned per-agent database metadata and a writer-owned generation or other content-safe validity signal; otherwise retain the current content-hash export path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence same-size/same-mtime stale-export path, and the supplied live run confirms the persistent fast path operates across manager restarts.

## Solution Assessment

No. The speedup is valuable, but this implementation makes durable cache ownership and transcript correctness depend on contracts weaker than the repository's canonical state and session-freshness model.
