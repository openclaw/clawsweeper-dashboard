Keep the QMD performance direction, but store export-cache state in the proper SQLite owner or keep it process-local, use a freshness key that cannot skip changed transcript bytes, and remove the `CHANGELOG.md` edit.

- **[P1] Store the export cache in SQLite:** `extensions/memory-core/src/memory/qmd-manager.ts:2579`
  - body: This writes `.export-state.json` under the QMD sessions export directory as persistent runtime cache state. OpenClaw's state policy says app state and caches under state dirs belong in SQLite, so this sidecar creates a second state owner that future repair, cleanup, and migration code must special-case; move this cache into the agent/shared SQLite state owner or keep it process-local.
  - confidence: 0.9

If a session JSONL is restored or replaced with different bytes while preserving size and mtime, QMD keeps serving the old exported markdown indefinitely; include a stronger freshness token such as ctime/source state, or fall back to hashing when content freshness is not proven.
  - confidence: 0.78
