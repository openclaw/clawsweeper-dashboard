## Summary

Keep this draft PR open. The optimization targets a real source-visible hot path, but the new persistent fast path can preserve a cache entry while the exported Markdown file is missing, leaving session memory absent until the JSONL source changes.

## What This Changes

The PR persists QMD session export state to `.export-state.json` and adds a size/mtime fast path to skip rebuilding unchanged session Markdown, with tests for cache persistence and exportDir invalidation.

## Best Possible Solution

Keep the optimization direction, but make the fast path prove the cached Markdown target still exists before skipping the rebuild, add the required changelog entry, and then run the focused QMD manager test plus changed gate.

## Reproduction Assessment

Yes, source-level. Current main clearly rebuilds every session entry before write-skipping, and the PR diff clearly skips rebuilds based only on cached source stat metadata.

## Solution Assessment

No, not yet. Persisting the cache is a narrow maintainable approach, but the skip path should also validate that the cached Markdown export still exists before trusting it.
