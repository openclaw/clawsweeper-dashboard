## Summary

Keep open: the performance goal is meaningful and the contributor supplied real-runtime proof, but the current branch still has merge-blocking storage and session-state correctness issues that need maintainer direction before this can land.

## What This Changes

The PR adds a durable QMD session-export state cache, a size/mtime fast path in `exportSessions()`, and tests for cache reload, invalidation, and missing-export rebuild behavior.

## Best Possible Solution

Land only after the cache is moved to agent-owned SQLite state, or an explicit maintainer exception documents the file artifact, and the fast path preserves current content-hash correctness for rewritten transcripts.

## Reproduction Assessment

Yes for the PR-introduced fast-path defect from source: preserve a transcript JSONL file’s size and mtime while changing message bytes, then the branch can take the cached `continue` without rebuilding the content hash. I did not run it locally because this review is read-only.

## Solution Assessment

No; the performance direction is plausible, but this implementation is not the best landing shape because it adds an unowned JSON cache and weakens current content-hash validation. The safer solution is a SQLite-owned cache plus content-safe invalidation, or an explicit maintainer exception before merge.
