## Summary

Keep open: the performance direction is well scoped and current main still has the expensive QMD export path, but the patch should verify the cached markdown export still exists before skipping rebuild from persistent state.

## What This Changes

The branch persists QMD session export state under `qmd/sessions/.export-state.json`, adds a source stat fast path before rebuilding unchanged session markdown, and covers the cache with QMD manager tests and a changelog entry.

## Best Possible Solution

Keep the render-versioned persistent cache, but make the fast path validate the cached markdown export and add a stale-state/missing-export regression test before merge.

## Reproduction Assessment

Yes from source inspection: a valid persisted `.export-state.json` entry plus an unchanged JSONL and missing `sessions/*.md` export will take the fast path without regenerating the markdown. I did not run a live filesystem reproduction because this review is read-only.

## Solution Assessment

No as written: a persistent source-stat cache is the right narrow direction, but the skip branch must fall back to the slow path when the cached target export is missing or unreadable.
