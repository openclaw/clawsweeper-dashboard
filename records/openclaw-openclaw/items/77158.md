## Summary

This PR addresses a real QMD performance problem and includes credible runtime proof, but it is not merge-ready: the implementation adds noncanonical persistent cache state and can serve stale transcript exports when different content retains the same size and modification time. Likely related people: @vignesh07 (high confidence, introduced QMD and its existing export cache) and @vincentkoc (medium confidence, recent contributor to the current QMD manager path).

## What This Changes

The PR persists QMD session-export cache entries across restarts and bypasses transcript parsing and rendering when source size and modification time match the cached values.

## Best Possible Solution

Retain the fast export path only with canonical per-agent SQLite metadata and a content-safe validity signal; otherwise preserve the current hash-based export behavior.
