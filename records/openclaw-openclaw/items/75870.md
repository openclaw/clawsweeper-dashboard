## Summary

Keep open. Current main still blocks the reported Signal linked-device Note-to-Self path, and #75890 is the active closing implementation path but still has loop-protection review work before this issue should close.

## What This Changes

Request Signal linked-device Note-to-Self chat support so self-addressed `syncMessage.sentMessage` envelopes can reach the agent instead of being blanket-dropped.

## Best Possible Solution

Keep this issue open until #75890 or a replacement lands a Signal-owned Note-to-Self implementation with loop/echo guards, regression tests, and docs/changelog, then close as implemented.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Signal receive envelope from the configured account with `syncMessage.sentMessage` addressed back to that same account; current main returns at the own-account guard and also drops any sync envelope.

## Solution Assessment

Unclear. The Signal plugin is the right boundary, but the destination/source carve-out needs a maintainer-approved guard against self-addressed sentTranscript echoes before it is the best maintainable fix.
