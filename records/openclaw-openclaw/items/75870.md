Keep open. Current main still drops the Signal Note-to-Self sync-envelope path, and https://github.com/openclaw/openclaw/pull/75890 is the active open implementation vehicle for this issue.

## What This Changes

The issue requests Signal Note-to-Self support for linked-device personal-account setups so self-addressed `syncMessage.sentMessage` envelopes can reach the inbound channel pipeline.

## Best Possible Solution

Land a focused Signal channel change through the linked PR or a replacement that accepts only intentional self-chat sync messages, preserves loop protection, adds regression tests, and updates the public Signal docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a signal-cli receive envelope whose sender is the configured account and whose `syncMessage.sentMessage` is addressed back to that account; current main returns at the own-account guard and also drops any `syncMessage` envelope.

## Solution Assessment

Unclear. The bundled Signal plugin is the right implementation boundary, but destination/source equality alone needs maintainer review against sentTranscript echo and loop-protection semantics before it is accepted.
