## Summary

Keep open. Current main still drops all Signal sync envelopes and own-account messages before inbound dispatch, while the public Signal docs explicitly say personal-account self-messages are ignored; the related #5722 context provides a concrete Note-to-Self safety condition rather than evidence that the request is already implemented or superseded.

## What This Changes

The issue asks OpenClaw's bundled Signal channel to receive messages sent through Signal Note to Self / syncMessage flows so a single personal Signal account can be used safely.

## Best Possible Solution

Add a Signal-plugin-owned, default-off Note-to-Self/self-messaging mode that preserves current behavior by default, extracts syncMessage.sentMessage only when the sync destination is the sender's own Signal identity, leaves ordinary sent-to-friend sync transcripts ignored, and documents the safer single-number setup path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by feeding the Signal handler a receive event with syncMessage.sentMessage for a Note-to-Self payload: current main returns on syncMessage and also returns on own-account sender before dispatch.

## Solution Assessment

The pasted patch direction is plausible but not complete as-is. The safer maintainable fix is an opt-in config surface with destination/source gating, existing allowlist or pairing enforcement, regression tests for Note-to-Self versus ordinary sent transcripts, and docs that keep the separate bot number as the default recommendation.
