## Summary

Keep open. Current main still intentionally drops Signal sync envelopes and own-account messages before inbound dispatch, while the related discussion and upstream signal-cli payload shape show a concrete Note-to-Self path that is not implemented.

## What This Changes

Allow Signal personal-account Note-to-Self messages from syncMessage.sentMessage to reach OpenClaw without processing ordinary messages sent to other Signal recipients.

## Best Possible Solution

Add a Signal-plugin-owned, default-off Note-to-Self mode that extracts syncMessage.sentMessage only when the sync destination is the sender's own Signal identity, keeps ordinary sent transcripts ignored, and documents the safer setup path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by feeding the Signal receive handler a syncMessage.sentMessage Note-to-Self payload from the configured account; current main returns on own-account sender and on syncMessage before dispatch.

## Solution Assessment

The pasted patch is directionally useful but not complete as-is. The maintainable fix is an opt-in config surface with destination/source gating, regression tests for Note-to-Self versus sent-to-friend sync transcripts, and docs that keep separate bot numbers as the default recommendation.

## Repair Work Prompt

Fix the Signal Note-to-Self/self-messaging gap reported in https://github.com/openclaw/openclaw/issues/75870, carrying forward the safety refinement from #5722. Implement this inside the bundled Signal plugin and shared config surface, not as a core channel special case. Add a default-off config option such as channels.signal.allowSelfMessaging or a similarly clear name. When enabled, extract syncMessage.sentMessage into the inbound data path only when the sync destination identifies the sender's own Signal account (destinationNumber/destination equals sourceNumber/source, and UUID equivalents if supported by signal-cli); continue ignoring read receipts, null syncMessage, bot reply echoes, and sync transcripts for messages sent to other people. Preserve existing default behavior when the option is unset. Update Signal types/schema/help docs, docs/channels/signal.md, focused regression tests for accepted Note-to-Self and ignored sent-to-friend sync envelopes, and CHANGELOG.md if the change is user-facing. Do not weaken dmPolicy/allowFrom/pairing checks and do not add bundled IDs or Signal-specific policy to core.
