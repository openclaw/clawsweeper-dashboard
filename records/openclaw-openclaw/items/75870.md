## What This Changes

Request support for using Signal Note-to-Self linked-device messages as an inbound Signal chat surface without replying to non-self sent transcripts.

## Best Possible Solution

Review and land or close #75890 after it adds regression coverage and keeps the Signal handler's non-self syncMessage loop protection intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed the Signal receive handler an envelope from the configured account containing syncMessage.sentMessage where destination matches source; current main returns before inbound dispatch at the own-account or syncMessage checks.

## Solution Assessment

Yes, directionally. The narrow maintainable fix belongs in the bundled Signal plugin handler with tests for self syncMessages, non-self sent transcripts, and direct own-account loop protection.
