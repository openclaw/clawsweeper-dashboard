## Summary

Keep open: current main still drops Signal sync envelopes and own-account messages before inbound dispatch, and open PR #75890 is the paired implementation candidate rather than a merged fix.

## What This Changes

Request Signal Note-to-Self support for linked-device personal accounts by handling self-addressed `syncMessage.sentMessage` payloads without replaying ordinary sent transcripts.

## Best Possible Solution

Review and land PR #75890 or an equivalent Signal-plugin fix with regression tests that promotes only self-addressed sync sent messages and preserves existing loop protection.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed the Signal handler a linked-device Note-to-Self receive envelope with `syncMessage.sentMessage`; current main returns on own sender or `syncMessage` before dispatch.

## Solution Assessment

Yes on the product boundary: this belongs in the bundled Signal plugin, not core. The safest implementation should gate on destination/source equality, keep ordinary sent transcripts ignored, and add tests/docs before closing the issue.
