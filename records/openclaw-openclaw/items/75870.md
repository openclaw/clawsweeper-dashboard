## Summary

Keep open. Current main still drops Signal Note-to-Self-style sync envelopes before inbound dispatch, and open PR #75890 is the active closing implementation path.

## What This Changes

Support using Signal Note-to-Self on a linked personal Signal account by routing self-addressed `syncMessage.sentMessage` envelopes as inbound messages while preserving loop protection for other sync envelopes.

## Best Possible Solution

Review #75890 as the canonical implementation path; if accepted, update the Signal handler, regression tests, and docs so only self-addressed sent transcripts are admitted while other sync envelopes remain dropped.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Signal receive envelope whose sender is the configured account and whose `syncMessage.sentMessage` is addressed back to that same account; current main returns at the own-account guard and also drops any `syncMessage` envelope.

## Solution Assessment

Yes, if maintainers accept the product change. The narrow maintainable boundary is the bundled Signal plugin, with tests and docs proving the `destination === source` carve-out while preserving loop protection.
