## Summary

Keep open. Current main still drops Signal own-account and sync envelopes before inbound dispatch, and the issue is paired with open closing PR #75890, so this should be resolved by reviewing that PR rather than closing this issue now.

## What This Changes

Support Signal Note-to-Self for linked-device personal-account usage by accepting self-addressed syncMessage.sentMessage envelopes instead of blanket-dropping them.

## Best Possible Solution

Review and either land or close #75890 after it adds adequate regression coverage for self-addressed syncMessage envelopes while preserving loop protection for other sync transcripts.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed the Signal event handler a Note-to-Self-style syncMessage.sentMessage envelope from the configured account; current main returns before inbound dispatch at the own-account and syncMessage guards.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the bundled Signal plugin handler with tests around the Signal envelope routing rules, not in generic core channel code.
