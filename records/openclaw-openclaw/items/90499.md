## Summary

Keep open: current main and `v2026.6.6` still have the reported Discord read-target guard behavior, and two open closing PRs now own the implementation path but have not merged yet.

## What This Changes

The issue asks the Discord plugin to allow `message.read`/`readMessages` for one-to-one DM channels when the fetched DM recipient is authorized by `channels.discord.allowFrom`.

## Best Possible Solution

Review and land the strongest open closing PR, currently https://github.com/openclaw/openclaw/pull/90558 if its proof gate is satisfied, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection traces `readMessages` and `fetchMessage` into `assertReadTargetAllowed`, where fetched one-to-one DM metadata without a guild id can only reach the guild-channel allowlist fallback; no live Discord DM was run in this read-only review.

## Solution Assessment

Yes. The maintainable fix belongs in the Discord plugin read guard: trust freshly fetched `ChannelType.DM` metadata, require exactly one recipient matching the effective account allowlist, and keep malformed, channel-id, and group-DM cases fail-closed.
