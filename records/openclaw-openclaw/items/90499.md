## What This Changes

The issue asks OpenClaw to allow Discord message reads from one-to-one DM channel targets when freshly fetched DM metadata identifies an allowlisted peer.

## Best Possible Solution

Review and land one canonical linked PR that authorizes only freshly fetched one-to-one DM metadata through the effective Discord DM allowlist policy while preserving guild, thread, malformed-DM, and group-DM rejection behavior.

## Reproduction Assessment

Yes. Source inspection traces `readMessages` and `fetchMessage` into `assertReadTargetAllowed`, where fetched one-to-one DM metadata without a guild id can only reach the guild-channel allowlist fallback.

## Solution Assessment

Yes. The maintainable fix belongs in the Discord plugin read guard, using freshly fetched `ChannelType.DM` metadata, exactly one recipient, and the effective account DM authorization path rather than caller-supplied metadata or DM channel ids.
