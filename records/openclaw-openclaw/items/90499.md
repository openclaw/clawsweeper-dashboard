## What This Changes

The issue asks for Discord `readMessages`/`message.read` to allow one-to-one DM channel targets when the fetched DM recipient is authorized by `channels.discord.allowFrom`.

## Best Possible Solution

Land a narrow Discord plugin fix through the linked PR that authorizes only freshly fetched one-to-one DM metadata with exactly one effective-allowlist recipient and keeps malformed, group, disabled, and channel-id-only cases fail-closed.

## Reproduction Assessment

Yes. Current main can be traced from `readMessages` into `assertReadTargetAllowed`, where a no-guild target falls through to guild channel-id checks and never evaluates fetched one-to-one DM recipients.

## Solution Assessment

Yes, the linked fix direction is the right layer: keep this in the Discord plugin read guard, use fresh Discord channel metadata, and reuse the existing effective DM access helper rather than adding a second policy source.
