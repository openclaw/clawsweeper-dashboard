## What This Changes

Discord DM read-target authorization should allow a one-to-one DM history read when freshly fetched DM metadata identifies an already allowlisted Discord peer.

## Best Possible Solution

Land one canonical Discord read-guard fix, preferably the strongest linked PR if its proof gate is satisfied, that authorizes only freshly fetched one-to-one DM metadata through the effective DM access resolver and keeps malformed, DM-channel-id, and group-DM cases fail-closed.

## Reproduction Assessment

Yes. Source inspection traces `readMessages` and `fetchMessage` into `assertReadTargetAllowed`, where fetched one-to-one DM metadata without a guild id can only reach the guild-channel allowlist fallback; no live Discord DM was run in this read-only review.

## Solution Assessment

Yes. The maintainable solution belongs in the Discord plugin read guard, using freshly fetched `ChannelType.DM` metadata, exactly one recipient, and the effective account DM authorization path rather than caller-supplied metadata or DM channel ids.
