## Summary

This issue should stay open: current main and the latest release still reject the reported Discord one-to-one DM read path because the read-target guard never preserves or checks fetched DM recipients against `channels.discord.allowFrom`. The report is narrow and source-reproducible, but it touches a Discord read authorization boundary, so it should get maintainer/security review before a repair PR.

## What This Changes

Allow Discord `message.read`/`readMessages` to read a one-to-one DM channel only when the fetched DM recipient is authorized by the effective Discord `allowFrom` list.

## Best Possible Solution

Patch the Discord plugin read-target guard to authorize only freshly fetched `ChannelType.DM` targets with exactly one recipient matching the effective Discord `allowFrom`, while preserving guild/thread behavior and fail-closed DM edge cases in tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a fetched one-to-one DM has no guild id, so the current guard reaches the non-guild fallback and checks only configured guild channel ids. I did not run a live Discord DM in this read-only review.

## Solution Assessment

Yes, the suggested fix direction is the right boundary: authorize from fresh Discord channel metadata, require a one-to-one DM recipient match against effective `allowFrom`, and keep missing metadata, channel-id allowlisting, and group DMs fail-closed.
