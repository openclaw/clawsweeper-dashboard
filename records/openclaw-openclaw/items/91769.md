## Summary

Keep open: current main and latest release still have the reported shape: Slack suppresses `isReasoning` replies, Slack progress-draft reasoning is wired only through `onReasoningStream`, and core only enables that stream for reasoning mode `stream`, so the mode-`on` visibility regression remains source-reproducible.

## What This Changes

The issue requests restored Slack pre-final visibility for `reasoningDefault: "on"` after Slack reasoning-message suppression, either through a transient progress-draft line or an explicit opt-out.

## Best Possible Solution

Define a Slack-safe transient reasoning visibility contract for mode `on`, preferably as progress-draft-only output that finalizes away and is documented alongside the Slack exception and safe progress defaults.

## Reproduction Assessment

Yes. Source inspection shows mode `on` reasoning still becomes an `isReasoning` block reply that Slack suppresses, while the Slack progress-draft reasoning feed only receives the mode `stream` callback.

## Solution Assessment

Unclear. The requested outcome is valid, but the best fix needs maintainer choice between a transient safe progress-draft signal and any explicit opt-in surface that could expose reasoning.
