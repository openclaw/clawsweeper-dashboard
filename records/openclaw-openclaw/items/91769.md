## Summary

Keep open: current main and v2026.6.8 still suppress Slack `isReasoning` payloads, Slack progress-draft reasoning is fed only by `onReasoningStream`, and core only enables that callback for reasoning mode `stream`; the open Slack-safe progress visibility issue overlaps but does not fully own this narrower mode-`on` behavior and docs mismatch.

## What This Changes

Slack reasoning mode `on` loses all pre-final visible feedback on tool-less progress-mode turns after Slack began suppressing `isReasoning` payloads.

## Best Possible Solution

Define a Slack-safe transient reasoning visibility contract for mode `on`, preferably progress-draft-only output that finalizes away and is documented with the Slack exception and safe progress defaults.

## Reproduction Assessment

Yes. Source inspection shows mode `on` reasoning still becomes an `isReasoning` block reply that Slack suppresses, while the Slack progress-draft reasoning feed only receives the mode `stream` callback.

## Solution Assessment

Unclear. The requested outcome is valid, but maintainers need to choose between a transient safe progress-draft signal and any explicit opt-in surface that could expose reasoning.
