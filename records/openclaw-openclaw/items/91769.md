## Summary

Keep open: current main and v2026.6.9 still have the reported Slack mode-`on` gap, and the broader Slack-safe progress visibility issue only partially overlaps rather than owning this narrower reasoning-default regression and docs mismatch.

## What This Changes

The issue reports that Slack users with `reasoningDefault: "on"` lost all pre-final visible feedback on tool-less turns after Slack reasoning payload suppression, because the replacement progress-draft reasoning feed only handles `"stream"` mode.

## Best Possible Solution

Define a Slack-safe transient mode-`on` reasoning visibility contract, preferably progress-draft-only output that finalizes away and is documented with the Slack exception and safe progress policy.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.9 shows mode `on` reasoning still emits an `isReasoning` block reply that Slack suppresses, while Slack progress reasoning only receives the mode `stream` callback.

## Solution Assessment

Unclear. The desired outcome is valid, but the best fix needs maintainer agreement on a transient Slack-safe progress signal rather than a broad opt-out that could expose reasoning again.
