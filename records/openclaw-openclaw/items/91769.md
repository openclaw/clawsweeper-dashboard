## Summary

Keep open: current main still suppresses Slack reasoning reply payloads while the progress-draft reasoning feed is driven only by stream-mode callbacks, so tool-less turns using reasoning level `on` retain the reported pre-final visibility gap. The prior suppression prevents durable hidden-reasoning leaks and should remain; maintainers need to decide whether mode `on` should instead produce a transient Slack preview.

## What This Changes

Restore a safe, transient pre-final Slack signal for tool-less turns when reasoning level is `on`, without persisting hidden reasoning text.

## Best Possible Solution

Preserve durable `isReasoning` suppression, define the Slack contract for reasoning level `on`, and—if visibility is intended—feed completed reasoning into the existing finalized-away progress-draft lifecycle with focused tool-less-turn tests and updated thinking/progress documentation.

## Reproduction Assessment

Yes, at source level with high confidence: Slack consumes reasoning only through the stream callback, while plain text-only turns have no other progress-draft trigger. This read-only review did not execute a real Slack turn, so live confirmation remains useful.

## Solution Assessment

The transient draft direction is the safest and narrowest likely fix because it preserves the reasoning-leak suppression boundary. Restoring durable reasoning messages or adding a suppression opt-out is not the preferred solution; maintainers still need to approve the event ownership and mode-`on` UX contract.
