## Summary

Keep open. Current main has the internal queue data needed to compute sub-agent lane capacity, but no agent-visible capacity/remaining field in `session_status`, no `sessions_capacity` tool, and no structured global-capacity error from `sessions_spawn`; the related open issues cover adjacent session-budget and multi-lane work rather than this exact preflight capacity request.

## What This Changes

The issue asks OpenClaw to expose sub-agent concurrency capacity through `session_status`, a new capacity tool, or structured `sessions_spawn` capacity errors.

## Best Possible Solution

Define a documented additive capacity contract backed by the command-lane snapshot for `CommandLane.Subagent`, preserve existing visibility boundaries, and cover it with focused tool, queue, docs, and spawn-path tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence current-behavior check: current tools expose sub-agent activity and config, but not global remaining capacity or a preflight capacity contract.

## Solution Assessment

Unclear until maintainers choose the public API shape. The internal queue snapshot makes a narrow additive implementation feasible, but choosing `session_status`, `subagents`, a new tool, or spawn-error semantics is a product/API decision.
