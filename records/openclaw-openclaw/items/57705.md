## Summary

Keep open: current main still has a real global sub-agent lane budget and an internal queue snapshot, but no agent-facing `session_status`, `sessions_capacity`, `subagents list`, or `sessions_spawn` response exposes the sub-agent limit, active count, queued count, or remaining slots.

## What This Changes

The issue requests an agent-facing way to inspect sub-agent concurrency capacity before or during `sessions_spawn`.

## Best Possible Solution

Expose the existing `CommandLane.Subagent` snapshot as additive structured read-only data, preferably from `session_status.details`, while preserving current `sessions_spawn` queueing semantics.

## Reproduction Assessment

Yes as a current-behavior check: source inspection of `session_status`, `subagents`, `sessions_spawn`, docs, and the latest release shows no agent-facing sub-agent capacity fields. It is not a strict bug reproduction because the request is for a new structured feature surface.

## Solution Assessment

Yes. The narrow maintainable path is to expose the existing lane snapshot as additive read-only structured data; changing `sessions_spawn` to reject at global capacity would alter queueing semantics and should be avoided unless separately requested.
