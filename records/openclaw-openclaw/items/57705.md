## Summary

Keep open. Current main still has the internal lane snapshot needed to compute sub-agent capacity, but no agent-visible remaining/capacity field, no `sessions_capacity` tool, and no structured global-capacity error from `sessions_spawn`; the related open issues are adjacent rather than superseding this request.

## What This Changes

Expose sub-agent concurrency limit, active count, and remaining capacity to agents via `session_status`, a new `sessions_capacity` tool, or structured `sessions_spawn` errors.

## Best Possible Solution

Define a documented additive capacity contract backed by the `CommandLane.Subagent` snapshot, preserve existing visibility boundaries, and cover it with focused tool, queue, docs, and spawn-path tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence current-behavior check: current tools expose sub-agent activity/config in other forms, but not global remaining capacity or a preflight capacity contract.

## Solution Assessment

Unclear until maintainers choose the public API shape. The internal queue snapshot makes a narrow additive implementation feasible, but choosing `session_status`, `subagents`, a new tool, or structured spawn-error semantics is a product/API decision.
