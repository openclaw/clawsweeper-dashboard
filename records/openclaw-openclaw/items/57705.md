## Summary

Keep open. Current main still has the queue/concurrency internals needed to compute sub-agent lane capacity, but no agent-visible remaining-capacity field, no `sessions_capacity` tool, and no structured global-capacity error from `sessions_spawn`; the related open issues are adjacent rather than superseding.

## What This Changes

Expose sub-agent concurrency capacity to agents, either through `session_status`, a new `sessions_capacity` tool, or structured `sessions_spawn` capacity errors.

## Best Possible Solution

Define a documented additive capacity contract backed by the `CommandLane.Subagent` snapshot, preserve existing visibility boundaries, and cover the selected tool/spawn surface with focused tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives a high-confidence current-behavior check: the tools expose status text and subagent lists, but not global sub-agent remaining capacity or a preflight capacity contract.

## Solution Assessment

Unclear until maintainers choose the public API shape. An additive capacity contract backed by `CommandLane.Subagent` snapshots looks maintainable, but choosing `session_status`, `subagents`, a new tool, or structured spawn-error semantics is a product/API decision.
