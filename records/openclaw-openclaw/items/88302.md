## Summary

Keep open: this is a valid gateway probe bug-fix direction, but the patch currently gives inactive configured remote probes the full timeout and the external proof is Vitest-only, so it should not merge as-is.

## What This Changes

The PR changes gateway probe budget resolution so configured and explicit remote targets receive the full caller timeout, with helper and command regression tests.

## Best Possible Solution

Land a narrowed fix that gives full budget to explicit remotes and active configRemote targets, preserves short caps for inactive configured remotes and SSH tunnels, adds coverage for that split, and includes redacted real remote CLI proof.

## Reproduction Assessment

Yes: current source shows non-loopback configRemote probes are capped at 1500ms and the command path passes that budget into probeGateway. I did not run the live remote WSS environment, but the source path and linked report make the bug reproducible from code inspection.

## Solution Assessment

No, not yet: full timeout for explicit remotes and active configRemote targets is the right repair shape, but inactive configured remotes should keep the short cap and the PR needs real remote CLI proof.
