## Summary

Keep this PR open: the linked bug is source-reproducible and the fix direction is useful, but the current patch still widens inactive configured remote probes and only provides targeted test output instead of real remote CLI proof.

## What This Changes

The PR changes gateway probe budget resolution so configured and explicit remote targets receive the full caller timeout, with helper and command regression tests.

## Best Possible Solution

Land a narrowed fix that gives full budget to explicit remotes and active configRemote targets, preserves short caps for inactive configRemote and SSH tunnel probes, covers that split in tests, and includes redacted real CLI proof.

Yes for source-level reproduction: current main caps non-loopback configRemote probes at 1500ms and passes that budget into probeGateway. I did not run the reporter's live remote WSS environment.

## Solution Assessment

No, not yet: full budget for explicit remotes and active configRemote targets is the right repair shape, but the unconditional configRemote return changes inactive configured remotes too and the PR still needs real remote CLI proof.
