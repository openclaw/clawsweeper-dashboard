## Summary

Keep open: the PR is not safe to merge because it broadens Google auth SSRF policy to all private/internal DNS answers and lacks real behavior proof. Some CI-contract pieces have since shifted on current main, but the branch still carries unique Google/Gemini behavior changes that need contributor proof and maintainer/security review.

## What This Changes

The PR changes Google OAuth, Google Chat, and Google Meet SSRF policies plus Gemini CLI defaults, adds a Gemini CLI provider test, updates Matrix test runtime and plugin config-boundary tests, and adds Google docs/changelog/package metadata.

## Best Possible Solution

Keep any still-valid CI fixes, narrow fake-IP support to the existing host-scoped RFC2544/ULA policy pattern, and require redacted real runtime proof for the affected OAuth/proxy/Gemini CLI paths.

## Reproduction Assessment

Yes, source-reproducible: on the PR, a lookup returning a private/internal address for an allowed Google auth hostname would pass because `allowPrivateNetwork` skips resolved-address validation. I did not run a mutating test in this read-only review.

## Solution Assessment

No: the CI-contract fixes may be useful, but broad `allowPrivateNetwork` is not the narrowest maintainable solution for fake-IP DNS. The safer path is the existing host-scoped RFC2544 and IPv6 ULA policy pattern, with matching tests and real behavior proof.
