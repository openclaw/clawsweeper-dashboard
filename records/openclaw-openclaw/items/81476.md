## Summary

Keep open: the CI contract repairs may be useful, but the branch also adds a security-sensitive Google auth fake-IP policy that accepts any private/internal DNS answer for allowed Google hostnames, and the external PR still lacks real runtime proof.

## What This Changes

This PR changes Google OAuth/fake-IP and Gemini CLI defaults, Google docs/changelog/package metadata, Matrix test runtime helpers, and plugin registry/config-boundary tests.

## Best Possible Solution

Keep any valid CI contract fixes, but rework OAuth fake-IP support through the existing host-scoped RFC2544/ULA policy pattern and require redacted real runtime proof for the affected login/proxy path.

## Reproduction Assessment

Yes, source-reproducible: current main skips resolved-address checks when allowPrivateNetwork is true, and the PR adds that flag to Google auth policies. A focused fake lookup returning a private address for an allowed Google hostname would exercise the issue.

## Solution Assessment

No: the CI contract changes may be valid, but the Google auth network fix is broader than the repo's established fake-IP policy. The safer path is a host-scoped RFC2544/ULA fake-IP policy or equivalent narrow helper.
