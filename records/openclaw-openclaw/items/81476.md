## Summary

Keep open: the branch is not ready because it broadens Google auth SSRF policy beyond the existing fake-IP contract and only provides test assertions, not real after-fix behavior proof.

## What This Changes

This PR combines Google fake-IP OAuth and Gemini CLI default changes with plugin registry/config-boundary, Matrix test-runtime, docs, package metadata, and changelog updates.

## Best Possible Solution

Keep any valid CI contract fixes, but rework OAuth fake-IP support through the existing host-scoped RFC2544/ULA policy pattern and add redacted real runtime proof for the affected login/proxy path.

## Reproduction Assessment

Yes, source-reproducible: current guard code skips resolved-address checks when `allowPrivateNetwork` is true, and the PR adds that flag to Google auth policies. A focused fake lookup returning a private address for an allowed Google hostname would exercise the issue.

## Solution Assessment

No: the CI contract changes may be valid, but the Google auth network fix is broader than the repo's established fake-IP policy. The safer path is a host-scoped RFC2544/ULA fake-IP policy or equivalent narrow helper.
