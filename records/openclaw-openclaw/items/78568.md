## Summary

This can close as ClawHub/plugin work: current main already exposes the hook semantics needed to claim selected inbound messages and skip model dispatch, and a reusable inbound-to-webhook forwarder is optional integration code outside OpenClaw core.

## What This Changes

The issue asks whether a third-party plugin should use `before_dispatch` to forward selected WhatsApp inbound messages to an authenticated HTTP webhook and suppress model dispatch.

## Best Possible Solution

Keep the core hook API as-is, publish the inbound webhook forwarder as an external ClawHub or npm plugin, and handle any generic `before_dispatch` docs wording improvement as a small docs-only follow-up.

## Reproduction Assessment

Not applicable; this is a feature/guidance request rather than a bug report. Source inspection and existing tests give high-confidence proof that current main supports the requested hook behavior.

## Solution Assessment

Yes; `before_dispatch` is the narrow maintainable primitive for a hook-only code plugin to claim selected inbound messages and suppress model dispatch. Bundling a generic webhook forwarder in core is not the best fit under the current plugin and ClawHub policy.
