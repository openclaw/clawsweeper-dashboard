## Summary

Keep open: current main still sends array-wrapped Nostr DM filters and the Nostr start task still resolves into the generic auto-restart path; open PR #42832 targets the same root causes but has not merged.

## What This Changes

The issue reports that the Nostr channel connects to relays, immediately restarts without a useful error, and eventually stops instead of listening for NIP-04 DMs.

## Best Possible Solution

Keep this as the active Nostr restart-loop tracker until PR #42832 or an equivalent fix lands, with regression coverage for filter shape, channel lifetime, and relay cleanup.

## Reproduction Assessment

Yes. Source inspection of current main gives a high-confidence reproduction path: configure Nostr and start the gateway; the Nostr bus still supplies the malformed subscribeMany filter while the gateway start task resolves into the auto-restart supervisor.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Nostr plugin path: pass a plain Filter object, keep the account task alive until abort/stop, close pool resources, and add targeted regression tests without core Nostr-specific special cases.
