## Summary

Keep open: current main still matches the reported Nostr restart loop root causes, and the related implementation PR is open rather than merged.

## What This Changes

The issue reports that the Nostr channel connects to relays, sends malformed DM subscription filters, immediately exits into auto-restart, and can leave relay connections behind.

## Best Possible Solution

Land PR #42832 or an equivalent Nostr plugin fix that passes a single filter object, keeps the gateway account task pending until abort/stop, closes relay pool resources, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure Nostr and start the gateway; current main still passes an array-wrapped DM filter and returns from `startAccount`, which the supervisor converts into an auto-restart loop.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Nostr plugin path rather than core special cases: correct the filter shape, keep the account lifecycle alive until abort, clean up relay connections, and cover those behaviors with targeted tests.
