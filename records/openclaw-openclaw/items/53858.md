## Summary

Keep open: current main still matches the reporter's narrowed root cause. The Nostr bus still passes an array-wrapped DM filter into `nostr-tools` 2.23.3, the gateway still treats a resolved channel `startAccount` promise as a clean exit and auto-restarts it, and the related fix PR is open rather than merged.

## What This Changes

The issue reports that the Nostr channel connects to relays but immediately exits into the gateway auto-restart loop because inbound DM subscriptions are sent with an invalid nested filter shape and relay connections are not fully cleaned up.

## Best Possible Solution

Keep this as the canonical Nostr restart-loop tracker until PR #42832 or an equivalent maintainer-approved fix lands. The end state should pass a plain DM `Filter` object to `SimplePool.subscribeMany`, keep the Nostr gateway account task pending until abort/stop, close subscription and relay pool resources during cleanup, and add targeted regression tests for REQ filter shape plus channel lifecycle.
