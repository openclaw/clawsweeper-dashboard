## Summary

Keep this open because current main still matches the reporter's root-cause comments: the Nostr bus passes an array-wrapped DM filter into nostr-tools, the Nostr gateway start task resolves immediately and therefore fits the generic channel auto-restart path, and relay-pool cleanup still appears incomplete. A related implementation PR is open, so the issue should remain available for review/landing rather than be closed.

## What This Changes

Fix the Nostr channel so it sends valid DM subscription filters, stays running instead of entering the channel restart loop, and cleans up relay WebSocket resources.

## Best Possible Solution

Keep this as an active Nostr restart-loop tracker until PR #42832 or an equivalent maintainer-approved fix lands. The durable fix should pass a plain DM Filter object to SimplePool.subscribeMany, keep startAccount pending until abort/stop, close subscription and pool resources during cleanup, and add targeted regression coverage for REQ filter shape plus channel lifecycle.

Yes. The current source gives a high-confidence reproduction path: configure Nostr, start the gateway, and the immediate resolution of startNostrGatewayAccount feeds the generic auto-restart supervisor while the subscribeMany call still supplies the malformed array-wrapped filter.

## Solution Assessment
