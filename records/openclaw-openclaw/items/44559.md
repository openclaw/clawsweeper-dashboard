## Summary

Keep open. Current main still uses the Windows Scheduled Task gateway backend with `gateway.cmd`, `ONLOGON`, and interactive `/IT` registration, while the related PowerShell ScheduledTasks PR remains open and not yet accepted as the product-safe fix path.

## What This Changes

The issue reports that a Windows gateway installed by `openclaw gateway install` disconnects when the launching PowerShell window is closed.

## Best Possible Solution

Define the supported native Windows gateway persistence contract, then land or replace the Scheduled Task implementation so it survives terminal/window closure, preserves or explicitly changes fallback behavior, updates docs/tests, and includes real Windows proof.

## Reproduction Assessment

Yes for source-level reproduction: current main stages `gateway.cmd` and registers it as an `ONLOGON` interactive Scheduled Task with `/IT`. Live Windows proof is still needed for the exact close-PowerShell symptom and any fix.

## Solution Assessment

No. Current main does not satisfy the requested background persistence, and the existing PowerShell ScheduledTasks PR is not yet the best merge path because it changes fallback policy and still has unresolved review/proof blockers.
