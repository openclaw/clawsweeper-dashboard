## Summary

Keep open. Current main still installs the Windows gateway through the Scheduled Task backend with a default `gateway.cmd` target, `ONLOGON`, and interactive `/IT` flags; the related PowerShell ScheduledTasks PR remains open and is not merge-ready because it changes fallback policy and has unresolved correctness, security, and proof blockers.

## What This Changes

Windows gateway installs should keep the gateway running after the launching PowerShell window is closed.

## Best Possible Solution

Define the supported native Windows gateway persistence contract, then land or replace the Scheduled Task implementation so it survives terminal/window closure, preserves or explicitly changes the fallback contract, updates docs/tests, and includes real Windows proof.

## Reproduction Assessment

Yes for source-level reproduction: current main writes `gateway.cmd` and registers the gateway with `ONLOGON` plus interactive `/IT` flags. Live Windows proof is still needed for the exact close-PowerShell symptom and for any fix.

## Solution Assessment

No. Current main does not satisfy the requested background persistence, and the open PowerShell ScheduledTasks PR is not yet the best merge path because it changes fallback policy and has unresolved blockers.
