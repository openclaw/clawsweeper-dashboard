## Summary

Keep open. Current main and the latest release still use a `gateway.cmd` Windows Scheduled Task with `ONLOGON`/interactive flags plus the documented Startup-folder fallback, and the related PowerShell ScheduledTasks implementation remains open and blocked at https://github.com/openclaw/openclaw/pull/68149.

## What This Changes

Windows gateway install should keep the Gateway running in the background after the launching PowerShell window is closed.

## Best Possible Solution

Define the supported native Windows gateway persistence contract, then land a Scheduled Task or hidden-launcher design that survives terminal/window closure, preserves or explicitly replaces the no-admin fallback, updates docs/tests, and includes real Windows proof.
