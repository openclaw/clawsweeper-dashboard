## Summary

Keep open: current main and the latest shipped release still use the native Windows `gateway.cmd` logon Scheduled Task path with interactive `/IT` registration, and the related PowerShell ScheduledTasks implementation remains open with proof and product-policy blockers.

## What This Changes

The issue asks for a native Windows Gateway installed by `openclaw gateway install` to keep running after the launching PowerShell or terminal window is closed.

## Best Possible Solution

Define the supported native Windows Gateway persistence contract, then land or replace the Scheduled Task implementation so it survives terminal/window closure, preserves or explicitly migrates fallback behavior, updates docs/tests, and includes real Windows proof.

## Reproduction Assessment

Yes for source-level reproduction: current main and `v2026.5.19` still stage `gateway.cmd` and register an `ONLOGON` interactive Scheduled Task with `/IT` when a user is available. I did not run a live Windows close-PowerShell test in this Linux read-only review.

## Solution Assessment

No current shipped solution satisfies the requested background persistence. The existing PowerShell ScheduledTasks PR may be the right direction, but it changes fallback policy and still has unresolved review/proof blockers.
