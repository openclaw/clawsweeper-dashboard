## Summary

Keep open. Current main and the latest release still use the Windows Scheduled Task gateway backend with a default `gateway.cmd` target, `ONLOGON`, and `/IT` interactive user flags; the related PowerShell ScheduledTasks PR is open and still has product-policy, correctness/security-review, and real Windows proof concerns rather than a merged fix.

## What This Changes

The issue requests that a Windows gateway installed by `openclaw gateway install` remain connected after the launching PowerShell window is closed.

## Best Possible Solution

Define the supported native Windows gateway persistence contract, then land or replace the Scheduled Task implementation so it survives terminal/window closure, preserves or explicitly changes the fallback contract, updates docs/tests, and includes real Windows proof.

## Reproduction Assessment

Yes for source-level reproduction: current main and latest release write `gateway.cmd` and register the gateway with `ONLOGON` plus interactive `/IT` flags. Live Windows proof is still needed for the exact close-PowerShell symptom and for any fix.

## Solution Assessment

No. Current main does not satisfy the requested background persistence, and the open PowerShell ScheduledTasks PR is not yet the best merge path because it changes fallback policy and has unresolved blockers.
