## Summary

Keep open: current main and the latest release still use the native Windows Scheduled Task path with `ONLOGON` and `/IT`, so the reported Gateway persistence problem remains unresolved and needs maintainer policy review before a safe fix.

## What This Changes

The issue asks for `openclaw gateway install` on native Windows to keep the Gateway running after the launching PowerShell or visible task window is closed.

## Best Possible Solution

Define the supported native Windows Gateway persistence contract, then land or replace the Scheduled Task implementation so it survives terminal/window closure, preserves or deliberately migrates fallback behavior, updates docs/tests, and includes real Windows proof.

## Reproduction Assessment

Yes for source-level reproduction: current main and `v2026.5.22` still stage `gateway.cmd` and register an `ONLOGON` task with `/IT` when a user is available. I did not run a live Windows close-PowerShell test in this Linux read-only review.

## Solution Assessment

No shipped solution currently satisfies the requested background persistence. The best path is a maintainer-approved Windows persistence contract plus a focused implementation that preserves or explicitly migrates current fallback behavior and includes real Windows proof.
