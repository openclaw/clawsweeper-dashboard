## Summary

Keep open. Current main and the latest stable release still install the native Windows gateway through a `gateway.cmd` logon Scheduled Task with interactive `/IT` registration, and the related PowerShell ScheduledTasks PR remains open with unresolved proof and product-policy concerns.

## What This Changes

The issue asks for native Windows `openclaw gateway install` to keep the gateway connected after the launching PowerShell or visible task window is closed.

## Best Possible Solution

Define the supported native Windows gateway persistence contract, then land or replace the Scheduled Task implementation so it survives terminal/window closure, preserves or explicitly changes fallback behavior, updates docs/tests, and includes real Windows proof.

## Reproduction Assessment

Yes for source-level reproduction: current main and `v2026.5.18` stage `gateway.cmd` and register it as an `ONLOGON` interactive Scheduled Task with `/IT`. Live Windows proof is still needed for the exact close-PowerShell symptom and any fix.

## Solution Assessment

No current shipped solution satisfies the requested background persistence. The existing PowerShell ScheduledTasks PR may be the right direction, but it changes fallback policy and still has unresolved review/proof blockers.
