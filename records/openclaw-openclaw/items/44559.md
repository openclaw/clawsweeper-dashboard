## Summary

Keep open: current main and the latest release still route native Windows Gateway install through an interactive `ONLOGON` Scheduled Task/`gateway.cmd` model, and the related PowerShell ScheduledTasks PR is open but conflicting and not merge-ready. The linked duplicate report at https://github.com/openclaw/openclaw/issues/81017 was already closed into this issue, so this remains the canonical thread for the Windows Gateway window/persistence policy.

## What This Changes

The issue reports that the Windows Gateway installed with `openclaw gateway install` disconnects when the launching PowerShell or visible task window is closed.

## Best Possible Solution

Define the supported native Windows Gateway persistence contract, then land or replace the Scheduled Task implementation so it survives terminal/window closure, preserves or explicitly migrates fallback behavior, updates docs/tests, and includes real Windows proof.

## Reproduction Assessment

Yes for source-level reproduction: current main and `v2026.5.20` still stage `gateway.cmd` and register an `ONLOGON` interactive Scheduled Task with `/IT` when a user is available. I did not run a live Windows close-PowerShell test in this Linux read-only review.

## Solution Assessment

No shipped solution currently satisfies the requested background persistence. The related PowerShell ScheduledTasks PR may be useful source work, but the best path must preserve or deliberately migrate current fallback behavior, fix its review blockers, and include real Windows proof.
