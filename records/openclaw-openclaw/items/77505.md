## Summary

Keep open: the feature is plausible and the submitted real-behavior proof is sufficient, but this is a new Control UI browser rendering policy and the docs still describe `[embed ...]` as the only agent-facing rich render path.

## What This Changes

The PR renders complete assistant and expanded tool-result HTML documents in Control UI as sandboxed iframe previews with a collapsible source view.

## Best Possible Solution

If maintainers accept automatic complete-document previews, document the rule and sandbox/CSP contract in the rich-output and Control UI docs, then merge after owner review and focused UI/Testbox validation; otherwise keep complete HTML on the escaped markdown or explicit hosted-embed path.

Not applicable: this is a feature PR rather than a bug report. Source inspection shows current main keeps raw HTML on the escaped markdown path, and the linked proof demonstrates the proposed before/after behavior.

## Solution Assessment

Unclear: the sandboxed `srcdoc` iframe approach is focused and tested, but it is not the best maintainable path until the rendering policy and documentation contract are accepted. The safer fallback is the existing explicit `[embed ...]` rich-output contract.
