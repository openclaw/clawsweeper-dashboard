- **[P2] Mirror runtime transport precedence in MCP summaries:** `ui/src/ui/views/mcp.ts:41-43`
  - body: When a saved MCP server has both `command` and `url`, this summary reports HTTP because `url` wins here, but `resolveMcpTransportConfig` returns stdio first when `command` is valid.   - confidence: 0.94

  - confidence: 0.95

  - body: A saved `{ enabled: false }` entry is the supported tombstone shape for disabling bundled/default MCP servers, but doctor still adds `server transport is invalid` after the disabled warning because the tombstone has no transport.
