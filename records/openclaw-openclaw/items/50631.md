## Summary

Keep open: this PR targets a real current-main gateway-auth exposure and is paired with open issue #50630, but the runtime-only patch still leaves service install/onboarding able to accept the same config before writing a daemon that will fail on boot; it also lacks the required changelog entry.

## What This Changes

The PR adds a runtime gateway configuration guard rejecting Tailscale Serve with auth mode none, plus regression tests for Serve/Funnel auth combinations.

## Best Possible Solution

Update this PR so Serve + none is rejected consistently at runtime and before managed service writes, preserve valid loopback-only none mode and Serve with real auth, and add a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows Serve + auth mode none resolves successfully, an existing test expects that configuration to pass, and none auth authorizes without credentials.

## Solution Assessment

No. The runtime guard is necessary, but the maintainable fix also needs install/onboarding preflight coverage so OpenClaw does not write a managed service that is known to fail at startup.
