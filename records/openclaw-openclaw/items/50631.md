## Summary

Keep open: this PR addresses a real source-reproducible Gateway auth exposure, but the patch is incomplete because install/onboarding preflight still accepts the dangerous Serve plus no-auth configuration, and the external PR still lacks real Tailscale Serve after-fix proof.

## What This Changes

The PR adds a runtime-config guard rejecting Tailscale Serve with `gateway.auth.mode: "none"` and adds focused runtime-config tests for Serve/Funnel auth combinations.

## Best Possible Solution

Centralize validation so Tailscale Serve plus explicit no-auth is rejected in both runtime startup and install/onboarding preflight, while preserving loopback-only no-auth and Serve with token/password, then land it with regression tests and redacted real Serve proof.

## Reproduction Assessment

Yes, at source level. Current main accepts `gateway.auth.mode="none"` with `gateway.tailscale.mode="serve"`, has a test asserting that shape succeeds, and then authorizes mode none before Tailscale identity or shared-secret checks run.

## Solution Assessment

No. The runtime guard is necessary, but the patch should also reject the same configuration before managed service install/onboarding writes a daemon plan that will fail on boot.
