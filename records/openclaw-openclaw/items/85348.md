## Summary

Keep open for maintainer review: this is a maintainer-labeled draft that addresses a real crash-risk path, and I did not find a discrete patch bug, but the worker-default and new GPU policy are upgrade-sensitive enough to need packaged/runtime proof before merge.

## What This Changes

The PR moves local memory embeddings behind a worker process by default, adds `memorySearch.local.gpu`, propagates CPU policy into local embeddings and QMD, and degrades to fallback or FTS-only search on local worker failure.

## Best Possible Solution

Land this only after maintainer review confirms the new worker default and GPU policy are the intended public contract and packaged install/upgrade proof shows local memory still starts, falls back, and searches correctly.

## Reproduction Assessment

Partially: the related issue has concrete crash logs and current main still runs local `node-llama-cpp` embeddings in-process, but I did not establish a live current-main Apple Silicon Metal crash reproduction in this review.

## Solution Assessment

Unclear as a final merge path: worker isolation is the right maintainable boundary for native teardown crashes, but the default/runtime/config change needs packaged upgrade proof and maintainer approval before it is the best shipped solution.
