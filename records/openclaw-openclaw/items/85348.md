## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Strip eval flags before forking the worker:** `packages/memory-host-sdk/src/host/embeddings-worker.ts:124-126`
  - body: `resolveWorkerExecArgv` returns a new execArgv array but only removes `--input-type`. Node only strips `--eval`/`-e` automatically when `fork()` uses the original `process.execArgv`; with this explicit copy, eval-launched runtimes can run the parent eval payload instead of the worker script, so local embeddings hang or fail even though the sidecar path is valid.
  - confidence: 0.9
- **[P2] Activate fallback after probe-time local degradation:** `extensions/memory-core/src/memory/manager.ts:331-342`
  - body: `markLocalEmbeddingProviderDegraded` clears `this.provider`, and `probeEmbeddingAvailability` catches that failure without activating fallback. A later `search()` then enters the FTS-only branch before the query-time fallback code can run, so a configured `memorySearch.fallback` can be stranded after a status/probe failure; activate fallback from the degraded lifecycle or before the FTS-only return and cover probe-failure-then-search.
  - confidence: 0.86
