## Summary

Keep open: this maintainer-labeled MEMBER draft addresses a real linked Apple Silicon local-memory crash by moving local embeddings into a worker, but the default runtime boundary is upgrade-sensitive and still needs maintainer review plus packaged/live proof before merge.

## What This Changes

The PR makes local memory embeddings use a forked worker by default, adds structured worker-failure degradation and fallback handling, exports worker failure helpers, emits the worker sidecar in the build, and adds focused memory/config tests.

## Best Possible Solution

Land after maintainer review confirms worker isolation is the intended local-memory contract and packaged fresh-install/upgrade proof shows the sidecar resolves, starts, and degrades safely.

## Reproduction Assessment

Yes, source-reproducible: current main still constructs local node-llama-cpp embeddings in-process, and the linked issue includes concrete Apple Silicon crash logs. I did not run a live Apple Silicon Metal teardown reproduction in this review.

## Solution Assessment

Yes, with a merge gate: process isolation is the narrowest maintainable fix for preventing native node-llama-cpp aborts from killing the gateway without adding new public config. The safer path is to land it only after maintainer approval and packaged upgrade proof.
