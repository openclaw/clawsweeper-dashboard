## Summary

Keep this PR open: the worker-isolation fix is still needed because current main runs local node-llama-cpp embeddings in-process, but the branch has a blocking packaged-sidecar path defect that can make the new default local provider fail in an installed package.

## What This Changes

The PR moves local memory embeddings behind a worker process by default, adds structured worker-failure degradation and fallback/FTS behavior, keeps the local GPU config surface rejected, and adds focused memory/config tests.

## Best Possible Solution

Keep the worker-isolation design, align the sidecar emission and default resolution for installed packages, and require a packaged local-embedding smoke after that fix before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not fully live: the linked report has concrete Apple Silicon crash logs and current main clearly runs node-llama-cpp in-process, but this read-only review did not reproduce the native Metal teardown crash.

## Solution Assessment

Is this the best way to solve the issue? Not as submitted: worker isolation is the right bounded crash-safety direction, but the packaged sidecar path must be fixed before the PR is merge-safe.
