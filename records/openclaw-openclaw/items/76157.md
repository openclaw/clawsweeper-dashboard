## Summary

Keep open. Current main still hardcodes the 20000 overflow hint, and the latest PR head still has the same primary-model fallback bug that the earlier ClawSweeper review called out, with no accompanying test or changelog update.

## What This Changes

The PR changes the generic context-overflow recovery message to compute a reserveTokensFloor hint from context-window tiers in src/auto-reply/reply/agent-runner-execution.ts.

## Best Possible Solution

Derive the generic overflow hint from primary model metadata first, fall back to active session context only when metadata is unavailable, preserve heartbeat-specific guidance, and add focused tests plus a scoped changelog bullet.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence path: configure a primary model with 1,000,000 context tokens and pass an active session entry with 32,768 contextTokens; the PR still selects the 20k tier because session context wins before metadata lookup.

## Solution Assessment

No. Context-aware tiers are the right direction, but this PR is not the best implementation until it fixes the fallback order and covers the behavior with tests and release notes.
