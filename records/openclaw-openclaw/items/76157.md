## Summary

Keep open. The branch is now narrow, but the generic overflow hint still lets stale session context override primary model metadata, so a 1M primary model can still get the 20k recommendation this PR is meant to avoid.

## What This Changes

The PR changes overflow recovery copy to compute a reserveTokensFloor hint from context-window tiers instead of always recommending 20000.

## Best Possible Solution

Land a narrow patch that derives the generic reserveTokensFloor hint from primary model metadata first, falls back to session context only when metadata is unavailable, preserves heartbeat-specific guidance, and adds focused tests plus a scoped changelog bullet.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction: configure primary model metadata at 1,000,000 tokens and pass an active session entry with 32,768 contextTokens; the PR still selects the 20k tier because the session value wins before metadata lookup.

## Solution Assessment

No. Context-aware tiers are the right direction, but this implementation is not the best fix until the generic path prefers primary model metadata and only uses session context as a fallback.
