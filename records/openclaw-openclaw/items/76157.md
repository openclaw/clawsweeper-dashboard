## Summary

Keep open: the PR targets a real remaining main-branch gap, but the current head still lets stale session `contextTokens` override primary model metadata in the generic overflow hint path and is missing the required changelog entry.

## What This Changes

The PR changes the generic context-overflow recovery hint to compute `agents.defaults.compaction.reserveTokensFloor` from context-window tiers instead of always recommending `20000`.

## Best Possible Solution

Land a narrow patch that derives the generic reserve floor from primary model metadata first, falls back to active session context only when metadata is unavailable, preserves heartbeat-specific guidance, and adds focused tests plus a single-line changelog entry.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence path: configure a 1,000,000-token primary model and pass an active session entry with `contextTokens: 32768`; the PR head selects the session value before metadata and recommends the 20k tier instead of 100k.

## Solution Assessment

No. Context-aware tiers are the right direction, but the current implementation is not the best fix until the generic path prefers primary model metadata and has regression coverage for that ordering.
