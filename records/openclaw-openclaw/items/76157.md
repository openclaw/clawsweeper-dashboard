## Summary

Keep open. The context-aware hint direction is useful, but the current PR head still removes current-main runner policy, attribution, app-server session-key, and lifecycle plumbing, including security-sensitive no-tools propagation.

## What This Changes

The PR adds a context-aware `reserveTokensFloor` helper and tests while also removing current-main agent-runner policy, fallback attribution, app-server session-key, and embedded lifecycle plumbing.

## Best Possible Solution

Rebuild a narrow patch on current main that only makes the generic overflow reset hint context-aware, uses primary model metadata before fallbacks, preserves heartbeat-specific guidance and all runner policy plumbing, and includes focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. Static inspection against PR head `09a13cf1c3c0871db4b172aaf534593677f40f72` reproduces the blockers: the diff omits fields that current downstream code and tests consume, and the new generic hint path can select active-session context before primary metadata.

## Solution Assessment

No. The requested hint improvement is reasonable, but this branch is not the best implementation because it mixes the hint change with unrelated reversions of current-main runner behavior and an unsafe fallback order.
