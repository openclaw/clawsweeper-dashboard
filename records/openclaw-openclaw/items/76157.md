## Summary

Keep open. The intended overflow-hint fix is still relevant on current main, but this branch is stale and would regress current agent-runner safety, delivery, lifecycle, and failover attribution behavior if merged as-is.

## What This Changes

The PR changes the overflow reset hint to recommend tiered `reserveTokensFloor` values by context window and adds tests/changelog, while the branch diff also removes several current-main agent-runner parameters and lifecycle tests.

## Best Possible Solution

Rebuild a narrow patch on current main that only makes the generic overflow reset hint context-aware, preserves heartbeat-specific guidance and all runner policy plumbing, and covers known primary-model metadata plus fallback cases.

## Reproduction Assessment

Yes. Static inspection is enough to reproduce the blockers: the PR branch omits currently consumed runner fields, and the new hint path can be exercised with a known 1M primary model plus a stale 32k active session to show it chooses the session tier instead of the primary tier.

## Solution Assessment

No. The direction is useful, but this branch is not the best implementation because it mixes the hint change with stale reversions and uses a helper whose fallback order is wrong for the generic primary-model hint.
