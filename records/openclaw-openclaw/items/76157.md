## Summary

Keep open. Current main still emits the static 20000 overflow hint, but this PR is not ready because the generic path can still choose stale session context over primary model metadata, and the latest head lacks the claimed tests and changelog entry.

## What This Changes

The PR changes context-overflow recovery copy to compute the suggested `reserveTokensFloor` from context-window tiers.

## Best Possible Solution

Land a narrow corrected patch that derives the generic hint from primary model metadata first, falls back to session context only when metadata is unavailable, preserves heartbeat-specific guidance, and adds focused tests plus a changelog bullet.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction: use primary model metadata at 1,000,000 tokens with an active session entry at 32,768 tokens and no heartbeat-bleed match; the PR path still selects the 20k tier because session context wins first.

## Solution Assessment

No. Context-aware tiers are a reasonable direction, but the implementation needs the primary-metadata fallback order fixed before it is the narrowest maintainable solution.
