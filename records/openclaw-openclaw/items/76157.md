## Summary

Keep open, but do not merge this head as-is. Current main still has the static overflow hint, while this PR’s latest patch is much broader than the stated fix and introduces several static-diff-backed regressions in release validation, plugin status/install behavior, skills visibility, config writes, and status auth labels.

## What This Changes

The PR adds context-aware `reserveTokensFloor` hint tiers and also changes release workflows, plugin install/status behavior, skills status, config writes, docs, Docker scripts, and changelog entries.

## Best Possible Solution

Land a narrow current-main patch that derives the generic overflow hint from primary model metadata first, falls back to session/default context only when metadata is unavailable, preserves heartbeat-specific guidance, adds focused tier tests, and limits the changelog to one relevant bullet.

## Reproduction Assessment

Yes. Static inspection reproduces both the current-main gap and the PR bug: main hardcodes `20000`, while the PR still lets stale session context override a larger primary model when selecting the generic tier.

## Solution Assessment

No. Context-aware tiers are a reasonable direction, but this branch is not the best solution because it mixes unrelated release/plugin/config/skills changes with the overflow copy fix and still gets the generic fallback order wrong.
