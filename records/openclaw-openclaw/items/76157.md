## Summary

Keep open. The hint improvement is useful and current main still has the static `20000` recovery hint, but the latest PR head is dirty and still removes current-main runner policy, lifecycle, and session-attribution plumbing, including no-tools propagation on both runner paths.

## What This Changes

The PR adds context-window-aware `reserveTokensFloor` tiers and tests while also changing agent-runner fallback, CLI, embedded-run, and changelog surfaces.

## Best Possible Solution

Land a rebased narrow patch that only makes the generic overflow reset hint context-aware, prefers primary model metadata before session fallback, preserves heartbeat-specific guidance, and leaves all runner policy plumbing intact.

## Reproduction Assessment

Yes. Static inspection of PR head `4fe2cf3cfb3d4cbda278e37bb97c0b77a195a18a` against current main reproduces the blockers: removed parameters are consumed by downstream runner code and covered by current tests.

## Solution Assessment

No. The requested hint direction is reasonable, but this branch is not the best implementation because it mixes the hint change with unrelated reversions of current-main runner behavior and an unsafe fallback order.

  - confidence: 0.9

  - confidence: 0.89
- **[P2] Preserve message-tool-only delivery plumbing:** `src/auto-reply/reply/agent-runner-execution.ts:1386-1387`
  - body: The patch removes `sourceReplyDeliveryMode` and `forceMessageTool` from embedded reply runs. That breaks the `message_tool_only` contract where the message tool remains available while automatic source delivery is suppressed.
  - confidence: 0.87
- **[P2] Prefer primary metadata before session fallback:** `src/auto-reply/reply/agent-runner-execution.ts:549-557`
  - body: The generic reset hint asks `resolveContextWindowForHint` with the active session, and that helper prefers `activeSessionEntry.contextTokens` before resolving the primary model. A stale smaller session can therefore recommend the wrong lower tier even when primary metadata is available.
  - confidence: 0.85
- **[P2] Restore the embedded lifecycle backstop:** `src/auto-reply/reply/agent-runner-execution.ts:1363-1366`
  - body: The branch deletes the embedded terminal lifecycle backstop. If the embedded runner returns without a terminal lifecycle event, downstream consumers can remain stuck waiting after a timed-out or failed run.
  - confidence: 0.84
- **[P2] Keep fallback attribution on model fallback:** `src/auto-reply/reply/agent-runner-execution.ts:1161-1164`
  - body: The `runWithModelFallback` call no longer passes `sessionId` and `lane`, reverting current attribution used by fallback observations and wrapped fallback errors. Exhausted fallback logs would lose the originating session/lane metadata.
  - confidence: 0.82
- **[P2] Forward app-server session keys:** `src/auto-reply/reply/agent-runner-execution.ts:1435-1441`
  - body: The app-server event bridge now emits only `runId`, `stream`, and `data`, dropping `evt.sessionKey`. Current tests cover that session key so subagent-scoped app-server events remain attributable to the child session.
  - confidence: 0.8
- **[P2] Forward CLI provenance and source delivery mode:** `src/auto-reply/reply/agent-runner-execution.ts:1254-1262`
  - body: The CLI runner call drops `inputProvenance` and `sourceReplyDeliveryMode`. Current CLI preparation uses those to annotate inter-session prompts and build the correct source-delivery system prompt, so CLI-backed replies would lose that context.
  - confidence: 0.78
