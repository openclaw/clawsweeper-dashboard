## Summary

Keep open. Current `main` still has the static overflow hint, so the requested fix is useful, but this PR head is incorrect and unsafe to merge because it still chooses stale session context before primary model metadata and carries broad unrelated release, plugin, status, skills, config, and changelog regressions.

## What This Changes

The PR proposes replacing the static context-overflow `reserveTokensFloor` hint with context-window-aware 100k/50k/35k/20k tiers.

## Best Possible Solution

Land a narrow current-main patch that computes the generic overflow hint from primary model metadata first, falls back to session/default context only when metadata is unavailable, preserves heartbeat-specific guidance, adds focused tier coverage, and keeps changelog scope to one bullet.

## Reproduction Assessment

Yes. Static inspection reproduces the current-main gap and the PR bug: `main` hardcodes 20000, while the PR's generic path still lets a stale lower-context active session override a larger primary model.

## Solution Assessment

No. Context-aware tiers are a reasonable direction, but this branch is not the best solution because it is far broader than the stated fix and still gets the generic fallback order wrong.
