## Summary

Keep open: current main still has a source-level path where an oversized agent contextTokens value can be reported as the effective context budget, and PR #77861 is already open with closing syntax for this issue.

## What This Changes

The issue asks for reported context budgets to treat agents.defaults.contextTokens as a downward cap rather than an override above the selected model/catalog window.

## Best Possible Solution

Review and land PR #77861, or an equivalent fix, so reporting and session persistence use the lower of the agent cap and known model window when a model window is available.

## Reproduction Assessment

Yes, as a source-level reproduction: current main returns the agent override before model lookup, and status/reply paths feed agents.defaults.contextTokens into that override. I did not run a live /status turn in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to change the shared context-token resolver semantics and cover the status/reply callers, which is exactly the scope of the linked PR #77861.
