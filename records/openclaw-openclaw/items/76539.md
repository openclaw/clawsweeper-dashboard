The observable bug is that the core `openclaw` package can still carry externalized/downloadable plugin runtime code or dependency metadata for WhatsApp/Baileys, which can trigger bundled runtime dependency installation failures even when WhatsApp is disabled and only Discord/Codex are enabled. Keep core extension-agnostic and do not work around this by setting pnpm `blockExoticSubdeps=false` or adding a new config option. Add regression coverage that proves WhatsApp remains package-local for `@openclaw/whatsapp`, root pack artifacts do not contain `@whiskeysockets/baileys`/`libsignal` or legacy runtime-dep staging paths, and existing lightweight/core bundled plugins still ship.
