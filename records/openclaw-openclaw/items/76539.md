## Summary

Current main has already removed the failing root-package bundled runtime path for externalized plugins: WhatsApp/Discord/Codex sidecars are no longer part of the root package sidecar baseline, and publishable plugins now build package-local npm runtimes with their own dependency graph.

## What This Changes

The issue reports that v2026.5.2 packaged installs can still pull Baileys runtime artifacts into the root OpenClaw package, causing pnpm exotic-subdependency failures before Discord/Codex replies.

## Best Possible Solution

Keep the current package-local runtime/externalized sidecar implementation and let the next release carry the fix to packaged installs.

## Reproduction Assessment

No high-confidence current-main reproduction remains. The reported v2026.5.2 path is source-supported by the old sidecar baseline, but current main filters externalized plugin sidecars out of the root package and builds plugin-local runtimes.

## Solution Assessment

Yes. Package-local npm runtimes keep Baileys under the WhatsApp plugin package and remove the disabled-plugin root package sidecar path without adding a user-facing config knob.
