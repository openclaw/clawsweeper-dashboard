## Summary

## What This Changes

The PR changes web UI session reconciliation to suppress stale running session rows after terminal chat, adds controller regressions, updates transcripts tool-display metadata, and removes redundant image-tool casts.

## Best Possible Solution

Keep the controller-side stale-row suppression, but preserve any known previous terminal `SessionRunStatus` and cover failed or timeout regressions before automerge.

## Reproduction Assessment

Yes at source level: current main can clear `chatRunId`, then accept a stale `sessions.list` or `sessions.changed` row with `status: "running"`, and `isSessionRunActive` treats that row as active. I did not run the live browser flow during this read-only review.

## Solution Assessment

No as submitted: the session controller is the right fix surface, but the stale-row normalizer should preserve known `failed` and `timeout` terminal statuses instead of collapsing every non-done terminal to `killed`.
