## Summary

Close as ClawHub/plugin scope: current main already exposes plugin hooks for per-turn context injection and pre-model blocking, while the proposed core gateway shell hook would add a deployment-specific config and process-execution surface contrary to the core/plugin boundary.

## What This Changes

The issue requests an optional gateway inbound context hook that can inject or reject per-channel or per-agent context before LLM dispatch.

## Best Possible Solution

Keep core unchanged and implement this as a code plugin published through ClawHub or loaded locally, using sanitized `before_prompt_build` injection plus `before_agent_run` pass/block decisions.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a reported broken existing behavior. Source inspection shows existing plugin hooks already cover the normal channel-turn injection and pre-model blocking paths.

## Solution Assessment

No; adding a gateway command hook in core is not the best fit because current code-plugin hooks already own deployment-specific policy. A ClawHub/plugin implementation is narrower, with a new issue only if a concrete routing case cannot be expressed by current hooks.
