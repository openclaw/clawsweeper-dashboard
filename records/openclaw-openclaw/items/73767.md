## Summary

Keep open: this is useful RuntimePlan cleanup, but the current PR head still regresses current-main auth-profile isolation and default plugin-harness profile forwarding, and it lacks the required changelog entry for a public Plugin SDK API addition.

## What This Changes

This PR extracts embedded-runner model/auth and stream-loop helpers, canonicalizes neutral embedded-runner imports/docs, adds native and Plugin SDK Harness V2 factory registration for Codex, and expands regression coverage around registry rollback and WS pooling.

## Best Possible Solution

Preserve current-main auth semantics inside the extracted helper, keep the owner-scoped Harness V2 registry fixes, add the active changelog entry, then rerun focused auth/plugin tests and the changed gate before maintainer review decides whether to land the stack whole or split it.

## Reproduction Assessment

Yes for the PR regressions. A source comparison against current main shows the extracted helper switched from no-external auth loading to external-overlaid loading and dropped the default plugin-harness profile lookup; a focused unit test can mock a Codex-style plugin harness with no explicit `authProfileId`.

## Solution Assessment

No, not yet. The extraction and Harness V2 direction are maintainable, but the patch must preserve current auth discovery/profile-forwarding behavior and document the public SDK API before it is the best way to solve the cleanup.
