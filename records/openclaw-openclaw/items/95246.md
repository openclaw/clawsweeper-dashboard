The remaining merge question is API compatibility, because this adds a 4 MiB limit to a public plugin SDK runtime path.

## What This Changes

The PR changes `fetchLiveProviderModelRows` to parse live model catalog success bodies through a 4 MiB bounded response reader and adds overflow/stall regression tests.

## Best Possible Solution

Land the bounded success-body read if maintainers accept 4 MiB as the public SDK live-catalog limit; otherwise adjust or document the cap before merge.
