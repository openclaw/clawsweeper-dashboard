## Summary

Keep open: the patch appears correct and well-proven, but it intentionally adds a public plugin SDK live-catalog size limit, so maintainers should accept, adjust, or document that compatibility contract before merge.

## What This Changes

The PR routes live provider model catalog success JSON through the shared bounded response reader with a 4 MiB cap and idle timeout, plus overflow/stall regression tests.

## Best Possible Solution

Land the bounded success-body read if maintainers accept 4 MiB as the live-catalog SDK limit; otherwise adjust or document the cap before merge.
