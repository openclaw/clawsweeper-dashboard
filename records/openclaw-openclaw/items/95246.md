## Summary

Keep open for normal PR review: current main still reads the live catalog success body with unbounded `response.json()`, and this patch appears to use the existing bounded stream helper correctly with focused regression coverage. ## What This Changes

The PR changes the plugin SDK live provider catalog helper to read successful catalog JSON bodies through the shared bounded response reader and adds streamed overflow/stall tests.

## Best Possible Solution

Land the bounded success-body read once maintainers accept 4 MiB as the public SDK live-catalog limit and the exact-head checks pass.
