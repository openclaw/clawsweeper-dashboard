## What This Changes

The PR changes `fetchLiveProviderModelRows` to parse successful live model catalog responses through a 4 MiB bounded reader with idle timeout and adds overflow/stall regression tests.

## Best Possible Solution

Land the bounded success-body read after maintainers accept the 4 MiB SDK live-catalog limit and the exact-head checks pass.

Yes. Reusing the existing media-core bounded response reader at the live catalog fetch boundary is the narrowest maintainable fix; the remaining decision is the public SDK cap value.
