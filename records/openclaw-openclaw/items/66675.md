## What This Changes

The issue reports that `openclaw gateway restart` can return failure on Linux/systemd even though the restarted gateway is healthy.

## Best Possible Solution

Land a focused Linux/systemd self-restart handoff, keep external restarts synchronous, preserve the scheduled-restart contract, and coordinate the expired approval followup warning cleanup with live user-systemd restart and update smoke proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection plus the systemd contract show a clear plausible path: current main restarts the same `KillMode=control-group` unit synchronously and treats post-restart health failure as command failure.

## Solution Assessment

No current-main solution is sufficient. The best direction is the open systemd self-restart handoff in https://github.com/openclaw/openclaw/pull/66735, with https://github.com/openclaw/openclaw/pull/66685 as related log-noise cleanup rather than the root fix.
