Keep open: current main still has the synchronous Linux/systemd restart path and the noisy expired approval followup path, while the related root-fix and warning-cleanup PRs remain open rather than merged.

## What This Changes

The issue asks for `openclaw gateway restart` on Linux/systemd to avoid reporting failure when systemd restarts successfully and the gateway is healthy afterward.

## Best Possible Solution

Land a Linux/systemd self-restart handoff that returns the existing scheduled restart outcome for in-service callers, then handle the expired approval followup warning cleanup as companion signal work.

## Reproduction Assessment

No, not as a live high-confidence repro in this read-only pass. The issue gives a concrete Linux/systemd setup and current source still shows the plausible synchronous restart, hard post-check failure, and followup-warning paths, but I did not run a real systemd user service.

## Solution Assessment

No current-main solution is sufficient. A #66735-style systemd self-restart handoff is the narrow root fix, while #66685-style followup suppression is useful cleanup but not enough by itself.
