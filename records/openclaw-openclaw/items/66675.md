## Summary

Keep open: current main still has the Linux/systemd synchronous restart path, the hard post-restart health failure path, and the expired approval followup warning path described here; the related fix PRs are still open rather than merged.

## What This Changes

Ensure `openclaw gateway restart` on Linux/systemd does not return a non-zero false failure after a successful healthy restart.

## Best Possible Solution

Land a narrow Linux/systemd self-restart handoff so in-unit restarts return scheduled/success semantics while preserving external synchronous restarts, and separately suppress benign expired approval followup noise.

## Reproduction Assessment

Unclear: the reporter provides a concrete Linux/systemd setup and current source shows a plausible false-failure path, but this read-only review did not run a live systemd user service or establish the exact non-zero command result on current main.

## Solution Assessment

No current-main solution is sufficient. A #66735-style systemd self-restart handoff is the narrow root fix, while #66685-style followup suppression is useful cleanup but not enough by itself.
