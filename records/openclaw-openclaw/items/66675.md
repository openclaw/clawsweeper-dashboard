## Summary

Keep open: current main still has the synchronous Linux/systemd restart path, hard restart-health failure reporting, and unsuppressed expired approval followup warning path described here. The related handoff and warning-suppression PRs are still open, and a later reporter added an overlapping package-update false-negative case.

## What This Changes

The issue reports that Linux/systemd gateway restart and update flows can return failure even when the managed gateway is running and healthy afterward.

## Best Possible Solution

Land a focused Linux/systemd self-restart handoff, keep external restarts synchronous, preserve the scheduled-restart contract, and coordinate the expired approval followup warning cleanup with live user-systemd restart and update smoke proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The reporter provided a concrete Linux/systemd setup and current source still shows the synchronous restart, control-group kill semantics, and hard post-restart failure path that make the false-negative plausible.

## Solution Assessment

No current-main solution is sufficient. The best direction is the open systemd self-restart handoff in https://github.com/openclaw/openclaw/pull/66735, with https://github.com/openclaw/openclaw/pull/66685 as related log-noise cleanup rather than the root fix.
