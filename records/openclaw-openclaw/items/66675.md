## Summary

Keep open: current main still lacks a systemd self-restart handoff, still converts post-restart health failures into a failed restart result, and still logs expired approval followup dispatch failures. The linked open PRs target the relevant surfaces, but the root behavior is not implemented on main yet.

## What This Changes

The issue asks OpenClaw to avoid reporting `openclaw gateway restart` as failed on Linux/systemd when the restart succeeds and follow-up `status`/`probe` checks show the gateway is healthy.

## Best Possible Solution

Review and land a #66735-style Linux/systemd self-restart handoff that reports a scheduled restart instead of letting the service cgroup terminate the caller, with focused regression coverage for systemd restart semantics. The #66685 warning cleanup can land alongside or separately, but it should not be treated as the root fix by itself.

## Reproduction Assessment

Unclear. The issue gives a concrete Linux/systemd setup and current source shows a plausible failure path, but this read-only review did not run a live systemd user service or find an existing end-to-end failing regression test for the false non-zero restart.

## Solution Assessment

No current-main solution is sufficient yet. The maintainable direction appears to be the systemd self-restart handoff in #66735; suppressing expired approval followup warnings alone is useful cleanup but not enough to solve the restart false failure.
