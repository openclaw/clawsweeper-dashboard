## Summary

Keep open: current main still has the synchronous Linux/systemd restart path, the hard post-restart health failure path, and the expired approval followup warning path described here; the relevant fix PRs are still open rather than merged.

## What This Changes

The issue asks OpenClaw to stop returning a failed `openclaw gateway restart` result after a healthy Linux/systemd restart and to clean up related expired exec-approval followup noise.

## Best Possible Solution

Review, rebase, and land a Linux/systemd self-restart handoff that returns a scheduled restart for in-unit callers, then coordinate the expired approval followup warning cleanup as a separate or companion signal fix.

## Reproduction Assessment

Unclear. The issue gives a concrete Linux/systemd setup and current code shows a plausible false-failure path, but this read-only review did not run a live systemd user service or find an existing regression test that reproduces the exact non-zero restart result.

## Solution Assessment

No current-main solution is sufficient. A #66735-style systemd self-restart handoff is the narrow root fix, while #66685-style followup suppression is useful cleanup but not enough by itself.
