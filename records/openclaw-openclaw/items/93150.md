## Summary

Keep open: current main and the latest release still have no keyword fallback when the builtin SQLite manager cannot initialize, and the open linked PR is the active implementation candidate for this issue.

## What This Changes

The issue requests a bounded keyword fallback for `memory_search` over local memory files when `node:sqlite` is unavailable.

## Best Possible Solution

Review and either land or decline https://github.com/openclaw/openclaw/pull/93260; if it lands, close this issue as implemented by that PR.

## Reproduction Assessment

Yes, at source level: missing `node:sqlite` throws through `requireNodeSqlite()`, and current `getBuiltinMemorySearchManager()` converts that into no manager plus an error instead of a fallback search path. I did not run a separate downlevel Node binary in this read-only review.

## Solution Assessment

Yes, conditionally: the narrowest plausible implementation belongs at the memory-core manager-resolution seam and should reuse the existing memory file helpers. It still needs maintainer acceptance because fallback behavior is a product and compatibility choice.
