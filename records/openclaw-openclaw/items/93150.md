## Summary

Keep open: current main and the latest release still route builtin memory search through the SQLite-backed manager and return an unavailable result when that manager cannot initialize; https://github.com/openclaw/openclaw/pull/93260 is the open closing implementation candidate for the requested keyword fallback.

## What This Changes

Request a bounded keyword fallback over local memory files when `memory_search` cannot initialize SQLite because `node:sqlite` is unavailable.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/93260 as the candidate implementation, then close this issue only after that PR lands or maintainers choose a different canonical path.

Yes at source level: current main returns an unavailable memory result when the builtin SQLite manager cannot initialize, and the issue provides a missing-`node:sqlite` command showing the runtime failure. I did not live-run a downlevel Node build; the local Node v24.16.0 has `node:sqlite` available.

## Solution Assessment
