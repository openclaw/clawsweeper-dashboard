## Best Possible Solution

Land one canonical backend-config fix that preserves Windows absolute QMD command paths before shell parsing, keeps bare PATH commands unchanged, and includes redacted Windows QMD availability plus index or search proof.

## Solution Assessment

No. Repairing drive-relative output after shell parsing is too late, and resolving every selected command through `resolvePath` breaks the documented PATH default.
