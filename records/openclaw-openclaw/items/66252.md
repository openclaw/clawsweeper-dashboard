## Summary

Keep open because current main implements the per-agent TTS half, but there is still no per-agent STT config contract or active-agent audio transcription overlay; the related implementation PRs are closed unmerged.

## What This Changes

Request adds per-agent TTS/STT overrides so different agents can use different speech voices, transcription languages, and speech providers.

## Best Possible Solution

Define a backward-compatible per-agent STT surface or equivalent runtime contract, document its precedence over `tools.media.audio`, and pass the active-agent audio config into transcription while preserving global defaults.

## Reproduction Assessment

Yes, at source level. Current main accepts and resolves `agents.list[].tts`, but strict agent config and the transcription runner have no `agents.list[].stt` contract or agent-aware STT merge.

## Solution Assessment

Unclear for the exact JSON shape proposed in the issue. The best path is a maintainer-approved STT config contract and precedence review rather than treating the existing TTS-only implementation as complete.
