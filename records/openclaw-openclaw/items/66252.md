## Summary

Keep open: current main implements the per-agent TTS portion, but `agents.list[].stt` is still absent and audio transcription still resolves language/provider settings from request, model-entry, and global `tools.media.audio` config.

## What This Changes

Add per-agent TTS/STT overrides so agents in one OpenClaw instance can use different speech providers, voices, and transcription languages.

## Best Possible Solution

Land a maintainer-approved per-agent STT contract that merges into agent-aware audio transcription entry points with schema/types/docs/tests and unchanged global defaults unless explicitly configured.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check: TTS has an agent override path, while `agents.list[].stt` and agent-aware transcription config are absent.

## Solution Assessment

Unclear for the exact proposed JSON shape. The maintainable path is a generic per-agent STT/audio-understanding contract with explicit precedence over global `tools.media.audio`, not reviving either closed unmerged PR as-is.
