## Summary

Keep open. Current main implements the per-agent TTS half, but the requested per-agent STT/language/provider override is still missing from the agent config contract and from the transcription runtime path; the issue discussion already records this split and keeps the remaining STT work open for separate review.

## What This Changes

Add per-agent TTS and STT overrides so different agents in one OpenClaw instance can use different speech providers, voices, languages, or transcription defaults.

## Best Possible Solution

Keep this issue open until OpenClaw has a backward-compatible per-agent STT contract in schema/types/docs and the batch transcription paths resolve an effective audio config for the active agent while preserving global `tools.media.audio` fallback behavior.

## Reproduction Assessment

Yes. A static reproduction is enough: configure `agents.list[].stt` as proposed and current main rejects it through the strict agent schema, while transcription still reads global `tools.media.audio`.

## Solution Assessment

No. The current TTS-only implementation is the right direction for voices, but it does not solve per-agent STT language/provider overrides; the maintainable fix is a separate STT config contract plus shared agent-aware audio config resolution.
