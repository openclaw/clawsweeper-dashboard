## Summary

Keep open. Current main has the per-agent TTS half, but the requested per-agent STT language/provider override is still absent from the agent config contract and transcription path, matching the issue discussion that explicitly left STT open.

## What This Changes

The issue requests per-agent TTS and STT overrides so different agents can use different voices, languages, or providers.

## Best Possible Solution

Keep this issue open until schema/types/docs define a backward-compatible `agents.list[].stt` surface and active-agent transcription paths merge it over `tools.media.audio` without changing existing global fallback behavior.

## Reproduction Assessment

Yes. A static current-main check is enough: `agents.list[].stt` is not accepted by the strict agent schema/type, and transcription still resolves language/provider settings from request/model/global audio config only.

## Solution Assessment

No. The current TTS-only implementation is the right direction for voice overrides, but the requested STT language/provider override needs a separate public config contract and agent-aware audio resolution path.
