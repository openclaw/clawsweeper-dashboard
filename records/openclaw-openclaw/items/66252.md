## Summary

Keep open. Current main implements the per-agent TTS half, but per-agent STT/language/provider overrides remain missing: agent config exposes `tts` only, transcription still reads the global `tools.media.audio` path, and the linked implementation PRs are closed unmerged.

## What This Changes

Request to let each configured agent override TTS and STT provider, voice, model, language, and related speech settings instead of sharing one global speech configuration.

## Best Possible Solution

Keep this issue open until maintainers choose the per-agent STT contract, then implement schema/types/docs/tests plus an agent-aware transcription merge path that preserves global `tools.media.audio` defaults.

## Reproduction Assessment

Yes for source-level confirmation: current main accepts `agents.list[].tts`, but there is no `agents.list[].stt` schema/type field or agent-specific transcription config path. This is a missing feature/config contract rather than a broken existing behavior bug.

## Solution Assessment

Unclear for the exact proposed JSON shape. The maintainable solution is a maintainer-approved STT override contract and precedence model, not reviving either closed PR branch as-is.
