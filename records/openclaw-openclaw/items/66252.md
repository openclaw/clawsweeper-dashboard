## Summary

Keep open because current main still covers only the per-agent TTS half; the per-agent STT language/provider override contract and transcription runtime path remain absent, and the related implementation PRs are closed unmerged.

## What This Changes

The issue asks for per-agent TTS and STT overrides so different agents can use distinct voices, languages, and speech providers in one OpenClaw instance.

## Best Possible Solution

Define a backward-compatible `agents.list[].stt` surface, document its precedence over `tools.media.audio`, and pass the resolved active-agent audio config through transcription without changing existing global fallbacks.

## Reproduction Assessment

Yes, at source level. Current main accepts and resolves `agents.list[].tts`, but the strict agent schema, resolved agent config, and transcription path have no `agents.list[].stt` contract or agent-aware STT merge.

## Solution Assessment

Unclear for the exact proposed JSON shape. The maintainable solution is an explicit public STT config contract and precedence review rather than closing this as solved by the existing TTS-only implementation.
