## Summary

Keep open because current main and the latest release implement the per-agent TTS half, but there is still no per-agent STT config contract or agent-aware transcription merge path; the related implementation PRs are closed unmerged.

## What This Changes

Request per-agent TTS and STT overrides so configured agents can use different speech providers, voices, languages, and transcription settings.

## Best Possible Solution

Define a backward-compatible per-agent STT surface or equivalent runtime contract, document its precedence over `tools.media.audio`, and pass the effective active-agent audio config into transcription while preserving existing global defaults.

## Reproduction Assessment

Yes, at source level. Current main accepts and resolves `agents.list[].tts`, but the strict agent schema, resolved agent config, and transcription path have no `agents.list[].stt` contract or agent-aware STT merge.

## Solution Assessment

Unclear for the exact JSON shape proposed in the issue. The best path is a maintainer-approved STT config contract and precedence review rather than treating the existing TTS-only implementation as complete.
