## Summary

Keep open. Current main implements the per-agent TTS half, but per-agent STT language/provider overrides are still absent from the agent config contract and transcription runtime path; the related implementation PRs are closed unmerged and the live discussion explicitly leaves STT open.

## What This Changes

Request per-agent TTS/STT overrides so different agents can use different speech providers, voices, languages, and transcription settings.

## Best Possible Solution

Define a backward-compatible agents.list[].stt surface with explicit precedence over global tools.media.audio, document it, and pass the resolved active-agent audio config through transcription without changing existing global fallbacks.

## Reproduction Assessment

Yes, at source level. AgentConfig and AgentEntrySchema accept agents.list[].tts but not agents.list[].stt, and the transcription path still resolves language/provider settings from request, model entry, and global tools.media.audio config only.

## Solution Assessment

No, this should not be closed as solved. The TTS-only implementation is a useful partial solution, but the remaining STT half needs a public config contract and agent-aware transcription resolution path.
