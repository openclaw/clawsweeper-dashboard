## Summary

Keep open. Current main implements the per-agent TTS half, but the requested per-agent STT language/provider override is still missing from the agent config contract and transcription runtime path; the two related implementation PRs are closed unmerged and no narrower open STT issue appears to supersede this one.

## What This Changes

Add per-agent TTS and STT overrides so different agents can use different speech providers, voices, and transcription languages.

## Best Possible Solution

Define a backward-compatible agents.list[].stt config surface, document its precedence, and merge it into active-agent transcription paths without changing existing global audio fallback behavior.

## Reproduction Assessment

Yes. A static current-main check is enough: agents.list[].stt is not accepted by the strict agent schema/type, and transcription still resolves provider/language settings from request, model entry, capability config, and global tools.media.audio only.

## Solution Assessment

Not yet. The TTS-only implementation is the right partial direction, but the remaining STT half needs a separate public config contract and agent-aware audio resolution path before this issue is solved.
