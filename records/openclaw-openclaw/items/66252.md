## Summary

Keep open. Current main implements the per-agent TTS half, but per-agent STT language/provider overrides are still absent from the agent config contract and transcription runtime path, matching the maintainer discussion that explicitly left STT open.

## What This Changes

The issue asks for per-agent TTS and STT overrides so different agents can use different speech providers, voices, and transcription languages in one OpenClaw instance.

## Best Possible Solution

Keep this issue open until schema/types/docs accept a backward-compatible `agents.list[].stt` surface and active-agent transcription paths merge it over `tools.media.audio` without changing existing global fallback behavior.

## Reproduction Assessment

Yes. A static reproduction is enough: configure `agents.list[].stt` as proposed and current main has no accepted schema/type field, while transcription still reads global `tools.media.audio`.

## Solution Assessment

No. The current TTS-only implementation is the right direction for voices, but it does not solve per-agent STT language/provider overrides; the maintainable fix is a separate STT config contract plus agent-aware audio config resolution.
