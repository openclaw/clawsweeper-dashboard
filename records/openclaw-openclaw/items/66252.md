## Summary

Keep open because current main has the per-agent TTS half, but the requested per-agent STT language/provider override is still absent and the related implementation PRs are closed unmerged.

## What This Changes

The issue asks for agent-specific TTS and STT overrides so different agents can use different speech providers, voices, and transcription languages in one OpenClaw instance.

## Best Possible Solution

Define a backward-compatible `agents.list[].stt` surface, document its precedence over `tools.media.audio`, and pass the resolved active-agent audio config through transcription without changing existing global fallbacks.

## Reproduction Assessment

Yes, at source level. Current main accepts and resolves `agents.list[].tts`, but the strict agent schema, resolved agent config, and audio transcription path have no `agents.list[].stt` contract or agent-aware STT merge.

## Solution Assessment

No, closing this as solved is not the best outcome. The TTS-only implementation is a useful partial solution, but the remaining STT half needs an explicit public config contract and transcription precedence review.
