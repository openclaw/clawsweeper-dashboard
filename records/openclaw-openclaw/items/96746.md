## Summary

Keep open for maintainer review: current main and the latest release still split long note words by code-point count, while this patch aligns `splitLongWord` with the visible-width wrapping contract and adds focused CJK regression coverage with sufficient real behavior proof.

## What This Changes

This PR changes terminal-core note wrapping to split oversized non-copy-sensitive words by grapheme visible width and adds a CJK regression test.

## Best Possible Solution

Land this narrow terminal-core fix, or an equivalent visible-width split, while preserving the existing copy-sensitive-token behavior and public API.

## Reproduction Assessment

Yes for source-level reproduction: current main uses a visible-width budget for wrapping decisions but chunks oversized words by code-point count in `splitLongWord`. I did not run local tests in this read-only review; the PR body supplies before/after terminal output.

## Solution Assessment

Yes. `splitLongWord` is the only fallback still using code-point counts, and reusing `splitGraphemes` plus `visibleWidth` fixes the owner boundary without changing Clack, table wrapping, or API surface.
