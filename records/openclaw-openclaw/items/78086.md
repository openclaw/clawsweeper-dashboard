## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Advance suspension TTLs before checking resume state:** `src/agents/pi-embedded-runner/run/attempt.ts:2189-2192`
  - body: This load uses `loadSessionStore(storePath)` without maintenance, then only injects the handoff when the stored state is already `resuming`. Since the new `suspended -> resuming` transition only runs behind load-time maintenance and is not durably written here, normal resume attempts can keep skipping the handoff after the TTL.
  - confidence: 0.95
- **[P2] Restore the configured lane concurrency:** `src/agents/session-suspension.ts:22`
  - body: Auto-resume always sets the lane back to concurrency `1`. Main and subagent lanes can be configured higher, and current defaults are 4 and 8, so a suspended lane can remain downgraded after the TTL until some later gateway-level config pass happens to reapply it.
  - confidence: 0.9
- **[P2] Capture a real handoff summary before suspending:** `src/agents/pi-embedded-runner/run.ts:1834-1843`
  - body: The new suspension calls omit `summary`, and the added `summarizeForHandoff` helper is not wired into the suspension flow. The resume message therefore falls back to generic text instead of the recovery briefing promised by the PR body.
  - confidence: 0.9
- **[P2] Export model.failover through OTLP:** `extensions/diagnostics-otel/src/service.ts:2318-2319`
  - body: The diagnostics-otel subscriber currently returns immediately for `model.failover`, so the new event is not exported through this plugin even though the PR description lists OTLP/ring-buffer visibility as part of the feature.
  - confidence: 0.88
- **[P2] Preserve non-quota cooldown reasons:** `src/agents/model-fallback.ts:945`
  - body: This maps every non-billing cooldown suspension to `quota_exhausted`. Transient or overloaded cooldowns from `resolveProfilesUnavailableReason` will be persisted and surfaced as quota exhaustion, which can mislead the fallback model, diagnostics, and users.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/session-suspension.ts:1`
  - body: This is a user-facing feature that changes failover and queue behavior, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active release section before shipping.
  - confidence: 0.87
