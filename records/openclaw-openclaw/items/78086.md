Full review comments:

- **[P1] Gate summary suspension to suspend-worthy failures:** `src/agents/model-fallback.ts:412-420`
  - body: This all-fallbacks-exhausted path suspends any attributed session with multiple attempts, including auth, format, model-not-found, or other non-quota failures. That can drop the lane to concurrency 0 for the TTL even though no quota, billing, or real circuit-open condition was tripped.
  - confidence: 0.9
- **[P2] Emit failover diagnostics for runner suspensions:** `src/agents/pi-embedded-runner/run.ts:1882-1892`
  - body: The prompt and assistant runner paths call `suspendSession` directly but never emit `model.failover`. A real provider 429/402 suspension from these paths therefore will not appear in the diagnostic ring or OTLP exporter despite the feature contract.
  - confidence: 0.86
- **[P2] Persist a real handoff summary before suspension:** `src/agents/pi-embedded-runner/run.ts:1883-1892`
  - body: These `suspendSession` calls omit `summary`, and the new `summarizeForHandoff` helper is not called anywhere. The resume path will inject `No recovery briefing was captured.` instead of the structured recovery briefing promised by this PR.
  - confidence: 0.9
- **[P2] Preserve suspended records until handoff is consumed:** `src/config/sessions/store-maintenance.ts:241-244`
  - body: Maintenance clears any `quotaSuspension` once the cleanup deadline passes before checking whether it is still `suspended`. If the process or session is idle past that second TTL, the record is deleted without ever entering `resuming`, so the next attempt gets no handoff.
  - confidence: 0.84
