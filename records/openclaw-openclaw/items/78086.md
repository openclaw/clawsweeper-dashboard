## Summary

Keep open: the latest head still has blocking state-machine defects and lacks acceptable real behavior proof; the supplied proof text remains placeholder-only and the author is asking for an override.

## What This Changes

The PR adds quota-driven lane suspension, persisted `quotaSuspension` state, failover diagnostics, and handoff-message injection across agent failover, session-store maintenance, diagnostics, and command-queue paths.

## Best Possible Solution

Make the failover state machine explicit and tested: persist or await a consumable handoff state, run the TTL transition on the actual resume path, generate a real recovery summary, scope lane suspension to intended quota or billing cases, add a changelog entry, and provide live proof or an explicit maintainer override.

## Reproduction Assessment

Yes for the PR defects: source inspection of head `9fc43b7e1a555c92868ac4a677abfe7003c9609a` shows the suspension is stored as `suspended`, normal attempt loading does not run quota-suspension maintenance, and the new handoff summary helper has no call site. I did not run live failover because this was a read-only review.

## Solution Assessment

No; the current patch is not the best way to solve the issue because the handoff state is not reliably reachable and terminal fallback can suspend non-quota failures. A safer path is to make the transition explicit, await or otherwise order persistence before fallback, generate the summary, and cover the state machine with regression tests and live proof.
