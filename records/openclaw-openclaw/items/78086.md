## Summary

Keep open: current main does not implement this feature, and the PR head still has merge-blocking source issues plus placeholder real-behavior proof for the new failover path.

## What This Changes

The PR adds state-aware model failover with zero-concurrency lane suspension, persisted `quotaSuspension` state, resume handoff injection, `model.failover` diagnostics/OTLP export, tests, and a changelog entry.

## Best Possible Solution

Finish the branch so only validated quota, billing, or true circuit-open conditions suspend lanes, every suspension path emits diagnostics and stores a real recovery summary, and the branch includes maintainer-accepted runtime proof.

## Reproduction Assessment

Yes for the PR defects: PR-head source inspection shows the broad suspension path and direct suspension calls without diagnostics or handoff summaries. No current-main bug reproduction applies because this is new feature work absent from main.

## Solution Assessment

No; the direction is plausible, but this is not the best mergeable solution while non-quota fallback summaries can suspend lanes and some suspension paths omit diagnostics and real handoff summaries. The safer path is to repair those contracts before maintainer approval.
