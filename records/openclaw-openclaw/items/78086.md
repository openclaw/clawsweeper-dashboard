## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Advance suspension TTLs before checking resume state:** `src/agents/pi-embedded-runner/run/attempt.ts:2191-2194`
  - body: This path loads the session store without running the new quota-suspension maintenance, then injects only when the stored state is already `resuming`. A normal `suspended` record can therefore remain skipped after its TTL, so the fallback handoff may never be injected.
  - confidence: 0.95
- **[P2] Restore the configured lane concurrency:** `src/agents/session-suspension.ts:22`
  - body: Auto-resume always sets the lane back to concurrency `1`. Main and subagent lanes can be configured higher, and current defaults are 4 and 8, so a suspended lane can stay under-capacity after the TTL until another gateway config pass happens to reapply it.
  - confidence: 0.9
- **[P2] Capture a real handoff summary before suspending:** `src/agents/pi-embedded-runner/run.ts:1834-1843`
  - body: The new suspension calls do not pass `summary`, and the added `summarizeForHandoff` helper is not wired into the suspension flow. The resume message can therefore fall back to generic text instead of the recovery briefing promised by the PR body.
  - confidence: 0.9
- **[P2] Export model.failover through OTLP:** `extensions/diagnostics-otel/src/service.ts:2380-2381`
  - body: The diagnostics-otel subscriber returns immediately for `model.failover`, so the new diagnostic event is not exported by this plugin even though the PR describes OTLP visibility as part of the feature.
  - confidence: 0.88
- **[P2] Preserve non-quota cooldown reasons:** `src/agents/model-fallback.ts:945`
  - body: This maps every non-billing cooldown suspension to `quota_exhausted`. Transient reasons such as `overloaded` or `timeout` can then be persisted and surfaced as quota exhaustion, misleading the fallback model, diagnostics, and users.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/session-suspension.ts:1`
  - body: This is a user-facing feature that changes failover and queue behavior, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active release section before shipping.
  - confidence: 0.87
