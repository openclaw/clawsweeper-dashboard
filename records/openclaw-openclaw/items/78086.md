## Summary

## What This Changes

The PR adds quota-driven session suspension, lane throttling, failover diagnostics, and handoff-message injection across agent failover and session-store paths.

## Best Possible Solution

Implement a documented failover state machine that makes the handoff state reachable on the actual fallback/resume path, captures a real summary, adds regression tests and a changelog entry, and ships with live proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes for the PR defects: source inspection of head 7eaa9e4808 shows suspension is written as `suspended`, the injection only runs for `resuming`, and normal attempt loading does not run the TTL maintenance transition. I did not run live failover because this was a read-only review.

## Solution Assessment

No; the current patch is not the best way to solve it because the state machine does not put the handoff on the actual fallback/resume path. A safer path is to make the transition explicit, generate a real summary, add tests, and get live proof or maintainer override.
