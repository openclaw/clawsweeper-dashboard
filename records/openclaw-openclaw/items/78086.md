## Summary

Keep open: current main does not implement this feature, and the PR still has source-reproducible failover state-machine defects plus placeholder real-behavior proof.

## What This Changes

The PR adds quota/billing-driven lane suspension, persisted session `quotaSuspension` state, failover diagnostics, and a handoff message for fallback model resumes.

## Best Possible Solution

Keep the PR open and rework it so quota/billing suspensions transition on the actual resume path, preserve configured lane concurrency, generate and persist a real handoff summary, add regression tests and a changelog entry, and attach real runtime proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the resume path loads the store without maintenance while the TTL transition is behind `opts.runMaintenance`, and the suspend calls do not capture a summary. I did not run a live failover scenario because this was a read-only review and the PR proof is still placeholder text.

## Solution Assessment

No; the current patch is not the best way to solve the issue because the handoff state is not reliably reached, the recovery briefing is not generated, and unrelated failure modes can suspend lanes. A safer implementation would make the state transition explicit on the resume path, scope suspension to quota/billing, preserve configured concurrency, and prove it with focused tests plus live output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Run suspension TTL maintenance on the resume path:** `src/agents/pi-embedded-runner/run/attempt.ts:2189-2192`
  - body: This loads the session store without `runMaintenance: true`, but the new `suspended -> resuming` transition only runs behind the store loader's maintenance flag. After the TTL or a process restart, a suspended session still loads as `suspended`, so this handoff branch is skipped.
  - confidence: 0.94
- **[P2] Capture the handoff summary before suspending:** `src/agents/pi-embedded-runner/run.ts:1834-1843`
  - body: The new suspension calls omit `summary`, and the added `summarizeForHandoff` helper has no call site. If the resume branch runs, it can only inject the generic fallback text instead of the recovery briefing promised by the PR body.
  - confidence: 0.91
- **[P2] Limit lane suspension to quota and billing failures:** `src/agents/model-fallback.ts:409-418`
  - body: When all candidates fail, this suspends any attributed session as `circuit_open` regardless of the recorded attempt reasons. A transient timeout, auth, or format failure can now throttle the lane for the suspension TTL even though the feature is scoped to quota and billing exhaustion.
  - confidence: 0.88
- **[P2] Restore the lane's configured concurrency:** `src/agents/session-suspension.ts:10-23`
  - body: Auto-resume always sets the lane back to concurrency `1`. Gateways can configure higher main, subagent, cron, and cron-nested concurrency, so suspending one of those lanes leaves it downgraded until a later config reapplies.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/session-suspension.ts:1`
  - body: This is a user-facing feature that changes failover and queue behavior, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active release section before shipping.
  - confidence: 0.88
