## Summary

Keep open: current main still matches the reported Dreaming cleanup residue path. The memory-core narrative key remains timestamped and reused as the idempotency key, the Dreaming scrubber only archives primary `.jsonl` transcripts, and the direct `sessions.delete` path archives transcripts without invoking trajectory sidecar cleanup; the recent Windows confirmation and the v2026.5.7 duplicate cleanup thread keep this current.

## What This Changes

The issue asks OpenClaw to fully clean up memory-core Dreaming narrative sessions, registry rows, and trajectory sidecars after managed Dreaming sweeps.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/70464, then add explicit trajectory sidecar cleanup for Dreaming/session deletion so this issue remains the canonical v2026.5.x residue evidence until the full lifecycle is covered.

## Solution Assessment
