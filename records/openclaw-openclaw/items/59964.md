Full review comments:

- **[P1] Keep invalid config recovery in doctor:** `src/gateway/config-reload.ts:247-258`
  - body: Current main intentionally skips invalid hot reloads and documents that Gateway does not rewrite openclaw.json; this branch restores from a backup during reload and applies the recovered config. That reintroduces whole-file rollback outside doctor and can discard unrelated user edits that should stay loud until openclaw doctor --fix runs.
  - confidence: 0.93
- **[P1] Scope recovery alerts to the triggering write:** `src/gateway/config-reload.ts:110-116`
  - body: The reload notifier reads sessionKey from the cached lastWriteEvent for every recovery alert. If a later filesystem-only edit corrupts config, the warning can be delivered to the previous session/thread and expose config failure details to an unrelated conversation.
  - confidence: 0.91
- **[P2] Clear selectedSource for invalid requested backups:** `src/config/io.ts:2886-2887`
  - body: When --from names an invalid candidate, this branch still returns selectedSource/restoredPath from that unusable backup. runConfigRecover treats selectedSource as the dry-run success signal, so dry-run can report it would recover and exit successfully when recovery cannot happen.
  - confidence: 0.88
- **[P2] Remove the draft proposal from user docs:** `docs/PR-PROPOSAL.md:3-8`
  - body: docs/PR-PROPOSAL.md is a draft design note in Chinese and says it should be rebased before submission. Repo docs are English user-facing content, so this should stay in the PR discussion or be replaced by final English docs for an accepted command.
  - confidence: 0.86
