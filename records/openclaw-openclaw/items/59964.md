## Summary

Keep open: the PR adds useful config-recovery work that is not on current main, but it is not safe to merge because the branch has a privacy-sensitive reload-notification bug, incorrect dry-run success semantics, a draft/non-English docs file, and no real behavior proof.

## What This Changes

The PR adds an `openclaw config recover` command, backup-candidate recovery helpers, startup and hot-reload recovery hooks, channel recovery notifications, and a draft proposal document.

## Best Possible Solution

A maintainable path should build on the current last-known-good and doctor-owned recovery architecture, add any manual recovery surface only after maintainer approval, and keep recovery notices scoped to the triggering context or a central policy.

Yes for the PR findings: source inspection shows the stale `lastWriteEvent` notification path and the invalid requested-backup dry-run success path. The broader feature was not live-run in this read-only review.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope recovery notices to the triggering write:** `src/gateway/config-reload.ts:112-124`
  - body: The reload notifier reuses `lastWriteEvent` for every recovery alert. A later filesystem-only invalid edit can therefore send config failure details to the previous session/thread instead of the context that triggered the bad write.
  - confidence: 0.91
- **[P2] Clear selectedSource for invalid requested backups:** `src/config/io.ts:2889-2890`
  - body: When `--from` selects an invalid candidate, this branch still returns `selectedSource` and `restoredPath`. The CLI treats `selectedSource` as dry-run success, so it can say it would recover from an unusable backup.
  - confidence: 0.88
- **[P2] Remove the draft proposal from user docs:** `docs/PR-PROPOSAL.md:3-8`
  - body: `docs/PR-PROPOSAL.md` is a draft proposal with non-English content and a note that it should be rebased before submission. It should stay in the PR discussion or become final English user documentation for a shipped command.
  - confidence: 0.86
