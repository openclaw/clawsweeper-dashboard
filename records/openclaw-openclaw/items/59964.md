## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Scope recovery notices to the triggering write:** `src/gateway/config-reload.ts:112-124`
  - body: `notifyReloadFailure` reads `sessionKey` and delivery context from cached `lastWriteEvent` for every recovery alert. A later filesystem-only invalid edit can therefore send config failure details to the previous session or thread instead of the context that caused the bad write.
  - confidence: 0.91
- **[P1] Harden backup candidates before reading them:** `src/config/io.ts:992-993`
  - body: `validateRecoveryCandidate` reads `.bak` files directly, and the sync path does the same, before checking that the candidate is a regular bounded trusted file. A bad backup can hang or exhaust startup/reload recovery before fail-closed handling runs.
  - confidence: 0.84
- **[P2] Return no selected source for invalid backups:** `src/config/io.ts:2881-2890`
  - body: When `--from` selects an invalid candidate, this branch still returns `selectedSource` and `restoredPath`. `runConfigRecover` uses `selectedSource` as the dry-run success signal, so it can report that it would recover from an unusable backup and exit successfully.
  - confidence: 0.88
- **[P2] Remove the draft proposal from docs:** `docs/PR-PROPOSAL.md:3-8`
  - body: `docs/PR-PROPOSAL.md` is a draft proposal with non-English content and a rebase-before-submit note. It should stay in the PR discussion or become final English user documentation for a shipped command.
  - confidence: 0.86
