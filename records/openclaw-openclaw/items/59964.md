## Summary

Keep open. Current main intentionally keeps whole-file config recovery doctor-owned, while this PR introduces a new recovery CLI/runtime policy and still has missing real behavior proof plus blocking correctness and security findings.

## What This Changes

The PR adds a config recovery CLI, broader backup-based config recovery, gateway startup and hot-reload rollback with channel notices, and a draft proposal document.

## Best Possible Solution

Keep whole-file recovery in doctor unless maintainers explicitly approve a new CLI/runtime recovery surface; if approved, scope notices to the triggering write, fix dry-run semantics, remove the proposal doc, and prove the real channel flow.

## Reproduction Assessment

Yes for the PR findings: PR-head source inspection shows the stale lastWriteEvent notification path, invalid requested-backup dry-run result, and committed draft doc. The broader CLI and gateway recovery behavior was not live-run in this read-only review.

## Solution Assessment

No. This branch is not the best current path until maintainer product direction, notification scoping, CLI result semantics, draft docs removal, and real behavior proof are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope recovery notices to the triggering write:** `src/gateway/config-reload.ts:112-124`
  - body: notifyReloadFailure reads sessionKey and delivery context from cached lastWriteEvent for every recovery alert. A later filesystem-only invalid edit can therefore send config failure details to the previous session or thread instead of the context that caused the bad write.
  - confidence: 0.91
- **[P2] Clear selectedSource when requested backups are invalid:** `src/config/io.ts:2881-2890`
  - body: When --from selects an invalid candidate, this branch still returns selectedSource and restoredPath. runConfigRecover treats selectedSource as the dry-run success signal, so it can report that it would recover from an unusable backup and exit successfully.
  - confidence: 0.88
- **[P2] Remove the draft proposal from user docs:** `docs/PR-PROPOSAL.md:3-8`
  - body: docs/PR-PROPOSAL.md is a draft proposal with non-English content and a rebase-before-submit note. It should stay in the PR discussion or become final English user documentation for a shipped command.
  - confidence: 0.86
