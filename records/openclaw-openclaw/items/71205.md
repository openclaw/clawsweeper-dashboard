## Summary

Keep open. The PR was plausible on its original base, but current main now writes through @openclaw/fs-safe's atomic writer, which chmods the config directory to 0o700 after the proposed guard returns, so the opt-out no longer preserves the 02770 shared-directory mode it is meant to protect; the PR also still needs real multi-uid proof and maintainer/security approval.

## What This Changes

The PR adds an `OPENCLAW_SKIP_STATE_DIR_HARDEN` env-var guard with accepted-env logging, a regression test, and a changelog entry for multi-uid sidecar deployments that need to preserve group access to the state directory.

## Best Possible Solution

Rework the opt-out so the accepted env var also prevents the atomic writer from chmodding the state/config directory on current main, then land it only with maintainer/security approval and redacted real multi-uid runtime proof; leave the broader chmod policy to the linked configurable-permissions issue.

## Reproduction Assessment

Yes. Source inspection plus a focused @openclaw/fs-safe 0.2.4 probe show the current atomic writer turns a 02770 directory into 0700 after the proposed guard would return.

## Solution Assessment

No. The proposed guard is no longer sufficient on current main because the later atomic write also chmods the parent directory; the fix needs to cover that write path and be approved as an explicit filesystem trust-boundary exception.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve the directory mode after the atomic write:** `src/config/io.ts:258-270`
  - body: On current main this guard returns before `tightenStateDirPermissionsIfNeeded`, but `writeConfigFile` then calls `replaceFileAtomic({ dirMode: 0o700 })`, and @openclaw/fs-safe 0.2.4 chmods the parent directory to that mode. A focused probe turns a 02770 directory into 0700, so the new test should fail after rebase and the opt-out would still lock out the sibling uid. Thread the skip through the atomic write or avoid that later directory chmod when the opt-out is accepted.
