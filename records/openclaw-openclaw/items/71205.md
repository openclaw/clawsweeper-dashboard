## Summary

Keep open. Current main still has no state-dir hardening opt-out, and the PR is a narrow implementation that preserves the default chmod path while adding an explicit trusted-env escape hatch for multi-uid shared-volume deployments; the remaining decision is maintainer acceptance of that operational exception and any docs expectation.

## What This Changes

The PR adds a truthy `OPENCLAW_SKIP_STATE_DIR_HARDEN` guard with accepted-env logging before config state-dir chmod hardening, adds POSIX regression coverage for preserving a `0o2770` state directory, and records a changelog entry.

## Best Possible Solution

Land a maintainer-approved version if OpenClaw wants this operational exception: keep default state-dir hardening unchanged, allow skipping only through a trusted operator environment variable, log accepted use, and retain tests for both default `0o700` hardening and setgid preservation with the opt-out enabled.

## Reproduction Assessment

Yes. On POSIX current main, a config path inside the state dir with mode `0o2770` reaches `tightenStateDirPermissionsIfNeeded`, sees group bits via `stat.mode & 0o777`, and chmods the directory to `0o700`; the PR adds the direct preservation case with the skip env enabled.

## Solution Assessment

Yes for the implementation shape. A truthy env guard before stat/chmod, plus accepted-env logging and a focused POSIX regression test, is the narrowest maintainable way to preserve secure defaults while supporting explicitly managed multi-uid shared-volume deployments.
