## Summary

Keep open. Current main still has the bundled-tool allowlist gap, and the updated PR is on the right path, but it still has blocking test type errors and incomplete `group:plugins` runtime behavior.

## What This Changes

The PR reapplies runtime `toolsAllow` to the embedded runner's combined base plus bundled MCP/LSP tool list, makes that filter plugin-aware, adds focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the plugin-aware final runtime allowlist, but make the fixture metadata match production and align bundle MCP startup gating with every allowlist key the PR supports.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main leak path, and the PR blockers are directly visible from the head diff plus the current `PluginToolMeta` contract and bundle MCP startup guard.

## Solution Assessment

No. The implementation direction is maintainable, but it needs a small correction before merge: use the real plugin metadata shape and either fully support `group:plugins` during MCP runtime creation or stop advertising that key.
