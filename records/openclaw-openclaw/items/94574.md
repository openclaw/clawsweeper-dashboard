## What This Changes

The PR removes the explicit `token: botToken` argument from Slack monitor startup `auth.test` and adds a regression test asserting the token is not passed in call arguments.

## Best Possible Solution

Land the narrow Slack monitor fix after redacted patched-OpenClaw startup, proxy, log, or request-adapter proof shows `auth.test` succeeds and the serialized request body omits the token.

## Reproduction Assessment

Yes, by source inspection. Current main and `v2026.6.8` pass `botToken` as `auth.test` options, and the checked Slack Web API source serializes options into the POST body while the client already has an Authorization header.

## Solution Assessment

Yes. Removing the per-call token is the narrowest maintainable fix because the Bolt app client already owns the bot token and sibling Slack `auth.test` paths use token-bound clients without passing a token argument.
