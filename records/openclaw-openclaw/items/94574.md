## Best Possible Solution

Land the canonical contributor PR only after redacted real OpenClaw Slack startup proof shows auth.test still succeeds and the request body no longer carries the bot token.

## Reproduction Assessment

Yes from source inspection: current main and v2026.6.8 pass { token: botToken } to auth.test, while the locked Slack Web API client serializes call options into the POST body and already carries the constructor token in Authorization. I did not run live Slack credentials in this read-only review.

## Solution Assessment

Yes, the patch is the narrowest maintainable code fix: keep using the App/WebClient constructor token and remove the duplicate per-call token. A wrapper or redaction-only fix would leave the token in the request payload.
