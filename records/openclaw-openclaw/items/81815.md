## Summary

## What This Changes

This PR switches `/sessions/:key/history` REST and SSE authorization to the OpenAI-compatible shared-secret scope resolver and adds REST/SSE regression coverage plus a revocation mock update.

## Best Possible Solution

Land the narrow resolver/test change after maintainer acceptance of the session-history bearer boundary and redacted real GET/SSE proof.

## Reproduction Assessment

Yes, by source inspection. Current main routes both initial history reads and SSE reauth through `resolveTrustedHttpOperatorScopes`, and shared-secret bearer auth sets declared scopes as untrusted, so the route lacks `operator.read` before returning history.

## Solution Assessment

Yes, if maintainers want this route in the compatible shared-secret bucket. Reusing `resolveOpenAiCompatibleHttpOperatorScopes` at both the initial request and SSE reauth points is the narrow maintainable fix for that policy.
