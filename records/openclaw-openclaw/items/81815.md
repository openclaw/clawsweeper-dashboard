## Summary

## What This Changes

The PR switches session-history REST/SSE authorization to the OpenAI-compatible HTTP scope resolver and adds bearer REST/SSE regression coverage plus a revocation mock update.

## Best Possible Solution

Land the narrow resolver/test change only after maintainer acceptance of the session-history bearer boundary and redacted real GET/SSE proof.

## Reproduction Assessment

Yes. Current main clearly routes /sessions/:key/history and its SSE reauth through resolveTrustedHttpOperatorScopes, so shared-secret bearer requests without trusted declared scopes fail chat.history authorization before reading history.

## Solution Assessment

Yes, if maintainers want this route in the compatible shared-secret bucket. Reusing resolveOpenAiCompatibleHttpOperatorScopes at both the initial request and SSE reauth points is the narrow maintainable implementation for that policy.
