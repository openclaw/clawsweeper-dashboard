## Summary

Keep open. Current main still has the reported deep-status hang path, and this PR is useful but not merge-ready because its CLI-side race does not cancel or avoid the inner embedding batch timeout.

## What This Changes

The PR adds an 8-second CLI-side embedding probe budget for `memory status --deep`, a `timedOut` probe field, timeout rendering/docs/tests, and a changelog entry.

## Best Possible Solution

Implement the short diagnostic budget in a manager/provider probe path that avoids or cancels the normal batch timeout, keep non-TTY status progress on `log`, preserve indexing's longer batch budget, and cover timeout rendering plus real timer behavior in focused tests.

## Reproduction Assessment

Yes. The related issue gives user-level steps, and current source inspection confirms `memory status --deep` awaits a cache-miss embedding probe that runs a batch ping under the local 600-second timeout path.

## Solution Assessment

No. A top-level CLI `Promise.race` is not the best fix because it does not cancel or avoid the inner manager batch timeout; a manager-level short-budget or cancellable diagnostic probe is the safer maintainable direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid leaving the inner batch timeout alive:** `extensions/memory-core/src/cli.runtime.ts:122`
  - body: This races `manager.probeEmbeddingAvailability()` from the CLI, but on a cache miss that method has already entered `embedBatchWithRetry(["ping"])` with the normal local 600-second timer. The CLI can report the 8-second timeout while that inner timer still keeps the process alive until it settles.
  - confidence: 0.92
- **[P2] Keep verbose non-TTY progress visible:** `extensions/memory-core/src/cli.runtime.ts:732`
  - body: When `--verbose` is set this chooses `fallback: "line"`, but `createCliProgress` only renders `line` on TTY streams and returns a no-op for non-TTY unless the fallback is `log`. `openclaw memory status --deep --verbose | cat` would still hide the probe labels during the diagnostic wait.
  - confidence: 0.9
- **[P2] Add the required changelog credit:** `CHANGELOG.md:9`
  - body: The new user-facing fix entry lacks a contributor attribution. Repo changelog policy requires a real non-bot author credit for this kind of entry, so it should credit `@DanWebb1949` before merge.
  - confidence: 0.97
