## Summary

Keep open. Current main still drives the Google Gemini CLI backend in headless `--prompt` mode without an approval-mode flag, and the focused fix PR was closed unmerged after a real native-tool approval/security posture concern.

## What This Changes

The issue requests restoring file write/edit capability for OpenClaw sessions using the `google-gemini-cli` OAuth backend on Linux.

## Best Possible Solution

Adopt a maintainer-approved Gemini CLI approval-mode mapping for headless runs, preserve explicit user backend args, document the security tradeoff, add regression coverage, and verify with a live OAuth edit smoke.

## Reproduction Assessment

Yes, by source and dependency contract rather than a live OAuth run. Current main invokes `gemini --prompt` without an approval mode, while Gemini CLI default/non-interactive policy excludes prompt-requiring write and shell tools.

## Solution Assessment

Unclear. The closed PR's unconditional `--yolo` likely fixes the observable write failure, but the safer maintainable fix needs maintainer approval of the Gemini native-tool approval posture, possibly with `auto_edit` or a policy-derived mapping.
