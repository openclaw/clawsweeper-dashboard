## Summary

Keep open for maintainer triage: current main already has the generic channelRuntime and no-proxy undici fixes relevant to the Weixin symptoms, but the in-repo Codex Computer Use `marketplace_missing` status path still does not clearly distinguish an existing bundled marketplace from an unregistered isolated Codex home.

## What This Changes

The issue asks to address external Weixin startup/fetch regressions and improve Codex Computer Use status/remediation when an isolated Codex home has cached/enabled Computer Use but lacks the registered marketplace.

## Best Possible Solution

Split the scopes: route Weixin monitor/header fixes to the external plugin, and decide a narrow OpenClaw Codex Computer Use status/remediation change that explains an unregistered bundled marketplace without weakening the read-only status contract.

## Reproduction Assessment

Partly. Source inspection gives a high-confidence path for the Codex `marketplace_missing` status when no registered marketplace is returned, but the Weixin runtime and fetch failures require the external plugin and live setup to reproduce end to end.

## Solution Assessment

No single fix is clearly best yet. The Weixin fixes belong in the external plugin, while the OpenClaw change should be a maintainer-approved Codex status/remediation improvement rather than an all-in-one repair for the whole issue.
