## What This Changes

## Best Possible Solution

Split the owner paths: route the Weixin startup/header fixes to the external plugin package, and keep any OpenClaw change narrowly focused on Codex Computer Use status/remediation for unregistered bundled marketplaces.

## Reproduction Assessment

Partly. Source inspection gives a clear current-main path for the Codex Computer Use `marketplace_missing` status when status is read-only and no marketplace is listed, but the Weixin runtime and fetch failures need the external plugin and live setup.

## Solution Assessment

No single in-repo fix is clearly the best solution. The channel runtime contract is already present in core, the Weixin changes belong to the external plugin, and any OpenClaw work should be a split Codex Computer Use diagnostic/status improvement.
