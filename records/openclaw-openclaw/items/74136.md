## Summary

## What This Changes

The PR extends `isAnnounceSkip()` to treat `ANNOUNCE_SKIP` on its own final line as silent, and adds unit/e2e coverage plus docs and changelog updates.

## Best Possible Solution

Land this focused parser, regression coverage, and docs update after the normal CI and maintainer merge gate, then close the linked bug as fixed.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the current-main failure: `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` cannot pass the exact trimmed equality check, and the announce flow gates suppression through that helper.

## Solution Assessment

Yes. Centralizing final-line token support in `isAnnounceSkip()` is the narrowest maintainable fix because the announce paths already share that predicate, and leaving `isReplySkip()` strict avoids changing unreported A2A reply semantics.
