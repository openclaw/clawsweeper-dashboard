## Summary

Keep open for normal maintainer review and CI. Current main still only accepts whole-text `ANNOUNCE_SKIP`, while this PR applies the focused parser change, adds targeted unit and announce-flow coverage, updates the docs/changelog, and has sufficient after-fix terminal plus production-use proof.

## What This Changes

The PR broadens `isAnnounceSkip()` to treat a trimmed final-line `ANNOUNCE_SKIP` as silent and adds focused parser/e2e tests plus subagent docs and changelog updates.

## Best Possible Solution

Land this PR after maintainer review and CI so the parser, announce behavior, regression tests, docs, and changelog agree on final-line sentinel suppression.

## Reproduction Assessment

Yes, source-reproducible. Current main trims and compares the entire reply to `ANNOUNCE_SKIP`, while announce suppression paths rely on that helper, so `DM summary\nANNOUNCE_SKIP` is not suppressed on main.

## Solution Assessment

Yes. Centralizing the final-line token check in `isAnnounceSkip()` is the narrow maintainable fix because the affected announce paths already route through that helper, and the PR keeps `isReplySkip()` strict.
