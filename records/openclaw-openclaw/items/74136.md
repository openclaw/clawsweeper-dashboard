## What This Changes

The PR broadens `isAnnounceSkip()` to suppress announce delivery when `ANNOUNCE_SKIP` is on a standalone final line, and adds helper/e2e tests plus docs and changelog text.

## Best Possible Solution

Merge after the contributor adds redacted real setup proof and CI/security-audit status is resolved or waived, preserving final-line-only `ANNOUNCE_SKIP` suppression and strict `REPLY_SKIP` behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `isAnnounceSkip("summary\nANNOUNCE_SKIP")` returns false because the helper compares the whole trimmed string to the token, and the announce delivery paths depend on that helper.

## Solution Assessment

Yes. The PR's final-line-only check is the narrow maintainable fix: it preserves exact-token behavior, handles whitespace, rejects embedded or non-final tokens, leaves `REPLY_SKIP` strict, and adds focused regression coverage.
