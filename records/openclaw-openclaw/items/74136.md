## Summary

Keep open. The PR is the right narrow code fix for the reported sub-agent announce leak, and the parallel PR discussion supports this as the canonical shape, but it still needs the required user-facing changelog entry, public sub-agent docs alignment, and exact-head CI follow-up before merge.

## What This Changes

The PR broadens `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` as a standalone trimmed final line of multi-line sub-agent output and adds focused unit plus announce-flow regression coverage.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding an Unreleased Fixes changelog bullet, updating the sub-agent announce docs to describe final-line `ANNOUNCE_SKIP`, preserving strict `REPLY_SKIP` behavior, and clearing exact-head validation.

## Reproduction Assessment

Yes. On current main, `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` returns false because the helper compares the entire trimmed string to the token, and the announce callsite uses that helper to decide whether to deliver output.

## Solution Assessment

Yes. The proposed final-line-only predicate is the narrowest maintainable fix for the observed leak because it preserves exact-token behavior, rejects embedded or non-final token text, and leaves reply suppression unchanged; docs/changelog and CI cleanup are the remaining follow-through.
