## Summary

Keep open. Current main still has the reported strict `ANNOUNCE_SKIP` behavior and this PR is the canonical focused fix, but the external-contributor real behavior proof gate is still missing and CI has a separate production dependency audit failure before merge.

## What This Changes

This PR broadens `isAnnounceSkip()` to suppress a standalone final-line `ANNOUNCE_SKIP` after multi-line sub-agent output, adds focused helper and announce-flow regression tests, and updates sub-agent docs plus the changelog.

## Best Possible Solution

Merge this PR after redacted real sub-agent behavior proof is added and the failing proof/security gates are green or explicitly handled by maintainers.

## Reproduction Assessment

Yes. Source inspection on current main shows `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` fails because the helper compares the whole trimmed string to the token, and both direct announce and output-capture paths depend on that helper.

## Solution Assessment

Yes. The final-line-only check is the narrow maintainable fix: it preserves exact-token behavior, handles trailing whitespace through trimming, rejects embedded or non-final tokens, and leaves `REPLY_SKIP` strict.
