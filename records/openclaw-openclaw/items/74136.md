## What This Changes

The PR changes the shared sessions send token parser to treat `ANNOUNCE_SKIP` on its own final line as silent, adds focused unit/e2e coverage, and updates subagent docs/changelog.

## Best Possible Solution

Land this PR after normal maintainer and CI gating so the shared announce-skip parser, regression tests, and subagent docs agree on the final-line sentinel behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `isAnnounceSkip()` only accepts a whole-text token match, while the announce path suppresses delivery only when that helper returns true; `DM summary\nANNOUNCE_SKIP` therefore falls through as deliverable text.

## Solution Assessment

Yes. Centralizing the final-line check in `isAnnounceSkip()` is the narrow maintainable fix because the affected announce paths already route through that helper, and the PR preserves strict `isReplySkip()` semantics.
