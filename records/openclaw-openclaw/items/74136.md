## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Centralizing the final-line check in `isAnnounceSkip()` is the narrow maintainable fix because all affected announce paths already route through that helper, and the PR deliberately leaves `isReplySkip()` strict.
