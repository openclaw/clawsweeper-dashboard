## Summary

Keep this PR open. Current main still uses strict equality for `isAnnounceSkip()`, so the reported multi-line final-token leak remains real, and this PR is the canonical open implementation after the parallel PRs were closed. The branch still needs the required release note and public docs alignment before it is ready to land.

## What This Changes

The PR broadens `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` as the trimmed standalone final line of multi-line output, adds focused token-helper unit tests, and adds a subagent announce-flow regression test.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding the missing changelog entry, updating the sub-agent announce docs to describe standalone final-line `ANNOUNCE_SKIP`, and verifying the targeted agents tests plus the changed gate. Keep `REPLY_SKIP` strict and continue rejecting embedded or non-final `ANNOUNCE_SKIP` text.

## Reproduction Assessment

Yes. On current main, `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` compares the whole trimmed string to the token and returns false, and the announce call sites only suppress delivery when that helper returns true.

## Solution Assessment

Yes. The PR’s final-line-only check is the narrowest maintainable fix for the reported leak because it preserves exact-token behavior, rejects mid-text and trailing-content token uses, and leaves reply suppression unchanged; the safer follow-through is docs, changelog, and CI cleanup rather than a broader parser.
