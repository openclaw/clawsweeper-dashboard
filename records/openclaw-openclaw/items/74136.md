## What This Changes

The PR broadens `isAnnounceSkip()` so multi-line sub-agent output ending with `ANNOUNCE_SKIP` on its own final line suppresses announcement delivery, and adds focused tests plus docs alignment.

## Best Possible Solution

Land the centralized parser/docs/test change after maintainer review and green merge gates; leave broader deterministic spawn-side suppression to the separate feature issue.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` exact-compare the fully trimmed text to `ANNOUNCE_SKIP`, so a multi-line summary ending with the token falls through to announcement delivery.

## Solution Assessment

Yes. The shared `isAnnounceSkip` helper is the narrowest maintainable owner boundary because completion delivery and output summarization already consume it; caller-specific filtering or changing `REPLY_SKIP` would be broader than the reported bug.
