## Summary

Keep open. Current main still has the strict `ANNOUNCE_SKIP` helper behavior, and this PR is the active canonical focused fix for https://github.com/openclaw/openclaw/issues/74071; the diff looks correct, but external-contributor real behavior proof is still only tests/CI and the PR merge commit currently has a failing dependency-audit gate.

## What This Changes

The PR broadens `isAnnounceSkip()` to treat `ANNOUNCE_SKIP` on its own final line as silent, and adds helper tests, an announce-flow regression, docs, and changelog text.

## Best Possible Solution

Merge this PR after redacted after-fix real sub-agent proof is added and the dependency-audit gate is resolved, preserving the narrow final-line `ANNOUNCE_SKIP` contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` fails because the helper compares the entire trimmed string to the token, and the direct announce and output-capture paths both depend on that helper.

## Solution Assessment

Yes. The final-line-only check is the narrow maintainable fix: it preserves exact-token behavior, trims the final token line, rejects embedded or non-final tokens, and leaves `REPLY_SKIP` strict.
