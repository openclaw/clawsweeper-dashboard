## Summary

Keep open. Current main still has the strict `ANNOUNCE_SKIP` equality behavior, and this PR remains the canonical narrow fix for #74071 after the parallel #74157 was closed in its favor; it still needs the missing docs and changelog follow-through before merge.

## What This Changes

The PR broadens `isAnnounceSkip()` to treat a standalone final-line `ANNOUNCE_SKIP` after multi-line sub-agent output as silent, and adds focused token-helper plus announce-flow regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding a single-line changelog Fixes entry and aligning the sub-agent docs/FAQ with the final-line announce-skip contract.

## Reproduction Assessment

Yes. On current main, `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` fails because the helper compares the entire trimmed string to the token, and both announce delivery paths depend on that helper.

## Solution Assessment

Yes. The final-line standalone-token check is the narrowest maintainable fix because it preserves exact-token behavior, keeps `REPLY_SKIP` strict, and rejects embedded or non-final `ANNOUNCE_SKIP` text.
