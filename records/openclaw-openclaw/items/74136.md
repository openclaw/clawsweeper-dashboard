## What This Changes

The PR updates `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` on its own final line after multi-line sub-agent output, adds helper and announce-flow regression tests, and documents/changelogs the behavior.

## Best Possible Solution

Merge this PR after redacted after-fix sub-agent proof is added and CI/security gates are green, preserving strict `REPLY_SKIP` and final-line-only `ANNOUNCE_SKIP` semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows multi-line text ending in `ANNOUNCE_SKIP` fails the helper because it compares the whole trimmed string to the token, and both direct announce and captured-output paths use that helper.

## Solution Assessment

Yes. The final-line-only helper check is the narrow maintainable fix: it preserves exact-token behavior, rejects embedded or non-final tokens, and leaves `REPLY_SKIP` strict.
