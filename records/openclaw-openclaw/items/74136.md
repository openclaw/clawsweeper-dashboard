## Summary

Keep open: the PR is a narrow, correct implementation for the current-main `ANNOUNCE_SKIP` false negative and appears to be the canonical open fix, but it still needs external-contributor real behavior proof before merge.

## What This Changes

The PR broadens `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` on its own final line, adds focused helper and announce-flow regression tests, and updates docs/changelog text.

## Best Possible Solution

Merge this PR after redacted after-fix sub-agent proof is added and CI/security gates are green, preserving strict `REPLY_SKIP` and final-line-only `ANNOUNCE_SKIP` semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` fails because the helper compares the whole trimmed string to `ANNOUNCE_SKIP`, and direct announce/output-capture paths use that helper.

## Solution Assessment

Yes. The final-line-only helper check is the narrow maintainable fix: it preserves exact-token behavior, rejects embedded or non-final tokens, and leaves `REPLY_SKIP` strict.
