## Summary

Keep open. Current main still only suppresses exact `ANNOUNCE_SKIP`, and this PR is the canonical narrow fix for the linked bug after the parallel PRs were closed, but it still needs release notes and docs alignment before merge.

## What This Changes

The PR broadens `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` as the trimmed standalone final line of multi-line output and adds focused unit plus announce-flow regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding the release-note entry, updating the sub-agent announce docs, and keeping `REPLY_SKIP` strict plus embedded or non-final `ANNOUNCE_SKIP` text rejected.

## Reproduction Assessment

Yes. On current main, `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` fails because the helper compares the entire trimmed string to the token, and both announce delivery paths depend on that helper.

## Solution Assessment

Yes. The final-line-only check is the narrowest maintainable fix for the reported leak because it preserves exact-token behavior, handles trailing whitespace via trimming, and avoids broadening reply suppression.
