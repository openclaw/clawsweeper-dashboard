## Summary

Keep open. The patch is the right narrow fix for a valid current-main bug, but it should not merge until contributor-supplied real behavior proof and the failing gates are addressed.

## What This Changes

The PR updates `isAnnounceSkip()` to suppress multi-line sub-agent announce output ending with `ANNOUNCE_SKIP`, and adds focused tests, docs, and a changelog entry.

## Best Possible Solution

Merge this PR after redacted after-fix real behavior proof is added and targeted agents validation plus the failing gates are green.

## Reproduction Assessment

Yes. Source inspection on current main shows `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` fails because the helper compares the entire trimmed string to the token, and both direct announce and output-capture paths depend on that helper.

## Solution Assessment

Yes. The final-line-only check is the narrow maintainable fix: it preserves exact-token behavior, handles trailing whitespace through trimming, rejects embedded or non-final tokens, and leaves `REPLY_SKIP` strict.
