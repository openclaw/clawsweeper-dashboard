## Summary

That fix shipped after the reporter’s 2026.3.13 version, and source search found no built-in `Cursor support assistant` injection string in OpenClaw.

## What This Changes

The issue asks OpenClaw to prevent Cursor-related links or assistant behavior from hijacking a custom Feishu agent session and clearing or rerouting context.

## Best Possible Solution

Keep the shipped route-preservation implementation; if incidental Cursor links still trigger ACP on current releases, that should be reported as a new focused ACP-intent bug with Feishu/WebUI logs.

## Reproduction Assessment

Yes for the actionable route-corruption path: the attachment identifies Feishu `deliveryContext` being rewritten to `webchat`, and current regression tests model that webchat-overwrites-external-route scenario. I did not run a live Feishu/WebUI session in this read-only review.

## Solution Assessment

Yes. Preserving established external delivery routes is the narrow maintainable fix for the observed hijack, while removing ACP/Cursor support or adding a broad opt-in config would overcorrect explicit harness workflows.
