## Summary

Keep this open: current main still teaches agents to route explicit Cursor harness requests through ACP, Feishu can bind the current conversation to an ACP session when that tool path is called, and there is still no negative coverage proving incidental Cursor links stay in the existing custom-agent session.

## What This Changes

The issue asks OpenClaw to preserve custom-agent Feishu session context when Cursor-related links or keywords are mentioned, instead of switching into a Cursor/ACP assistant flow.

## Best Possible Solution

Add an explicit-intent boundary for ACP harness routing so clear requests to run or bind work in Cursor still use ACP, while incidental Cursor links or mentions remain in the existing custom-agent Feishu conversation.

## Reproduction Assessment

No high-confidence live reproduction is available because the report has no logs and I did not run a Feishu/ACP session. Source inspection does show the risky path: Cursor ACP routing remains advertised and Feishu can current-bind an ACP session when `sessions_spawn(runtime="acp")` is called.

## Solution Assessment

No for removing ACP or adding a broad opt-in config as the default fix. The narrower maintainable fix is to require explicit Cursor/ACP harness intent and add regression coverage for incidental Cursor-link text staying in the current custom-agent session.
