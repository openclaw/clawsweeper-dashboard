## Summary

Keep open: current main still lacks an agent-callable current-conversation bind for ACP `sessions_spawn`; the tool only exposes `thread`, which keeps Discord on child-thread placement, while `/acp spawn --bind here` already has the in-place bind behavior.

## What This Changes

Request an ACP `sessions_spawn` option that binds a spawned ACP session to the current Discord conversation instead of creating a child thread.

## Best Possible Solution

Add an additive ACP-only current-conversation binding mode to `sessions_spawn` that reuses `/acp spawn --bind here` validation and binding behavior while preserving `thread: true` child-thread placement.

## Reproduction Assessment

Yes, source-level. Current main exposes and forwards only `thread` for ACP `sessions_spawn`, while Discord `thread: true` still resolves to child placement and the slash command path separately supports current binds.

## Solution Assessment

Yes. An additive `bindTo: "current"` or equivalent ACP-only option is the narrow maintainable fix; changing `thread: true` would risk existing child-thread semantics.
