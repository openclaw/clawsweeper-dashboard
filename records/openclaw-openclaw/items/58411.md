## Summary

Keep open: current `main` still has no agent-callable ACP current-conversation bind option for `sessions_spawn`; `thread: true` still follows the Discord child-thread path while `/acp spawn --bind here` remains a separate slash-command-only current-bind path.

## What This Changes

The issue requests an agent-callable `sessions_spawn` option that can bind an ACP session to the existing Discord thread instead of creating a child thread.

## Best Possible Solution

Add an ACP-only current-conversation binding mode to `sessions_spawn` that reuses existing `/acp spawn --bind here` validation and binding behavior while preserving `thread: true` child-thread placement.

## Reproduction Assessment

Yes. Source inspection shows the tool schema and ACP params expose no current-bind field, execution forwards only `thread`, and tests still distinguish Discord child placement from slash-command current placement.

## Solution Assessment

Yes. An additive `bindTo: "current"` or equivalent ACP-only option is the narrow maintainable fix because changing `thread: true` would break existing child-thread semantics.
