## Summary

Keep open: the bug is still present on current main, but this PR is stale against the current shared helper layout and needs a narrow rebase/repair before merge.

## What This Changes

The PR makes missing session override-field deletion opt-in so generic session writes preserve model/provider overrides unless reset code explicitly clears them.

## Best Possible Solution

Land the opt-in clearing behavior in `src/agents/command/attempt-execution.shared.ts`, thread it through the current wrapper/reset paths, and add regression tests for both preserve-by-default and explicit clearing semantics.

## Reproduction Assessment

Yes. Seed a session entry with `providerOverride` and `modelOverride`, then persist a later partial entry through the current wrapper that omits those fields; current main merges the old values and then deletes them because they are in `clearedFields`.

## Solution Assessment

Yes in direction, but not as submitted. The narrow maintainable fix is an explicit clear flag, but it needs to be applied to the current shared helper and wrapper layout rather than the older `attempt-execution.ts` helper shape.
