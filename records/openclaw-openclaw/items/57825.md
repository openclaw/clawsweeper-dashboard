## Summary

Keep open: the underlying session override loss is still source-reproducible on current main, but this PR is stale against the current helper split and needs real after-fix behavior proof before merge.

## What This Changes

The PR makes missing session override-field deletion opt-in so generic session writes preserve model/provider overrides unless reset code explicitly clears them.

## Best Possible Solution

Move the opt-in clearing behavior into `src/agents/command/attempt-execution.shared.ts`, thread it through the current wrapper/reset paths, and keep regression coverage for both preserve-by-default and explicit clearing semantics.

## Reproduction Assessment

Yes. By source inspection, seed a session entry with `providerOverride` and `modelOverride`, then persist a later partial entry through the current wrapper; current main merges the old values and deletes the omitted keys because they are in `clearedFields`.

## Solution Assessment

Yes in direction, but not as submitted. The explicit-clear flag is the narrow maintainable fix, but it needs to be rebased onto the active shared helper and current wrapper call sites.
