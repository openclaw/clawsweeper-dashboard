## Summary

Keep open: current `main` and v2026.6.8 still lack the requested typed subagent allowlist mutation, and https://github.com/openclaw/openclaw/pull/89656 is the open closing implementation path awaiting API/security review and proof.

## What This Changes

Add a typed Gateway method for patching `agents.list[].subagents.allowAgents` so clients do not need raw `config.patch`.

## Best Possible Solution

Review, prove, and either land or reject https://github.com/openclaw/openclaw/pull/89656 as a narrow admin-scoped Gateway/protocol/SDK/Admin HTTP API while keeping existing `sessions_spawn` runtime policy semantics unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows `subagents.allowAgents` runtime/config behavior exists, but no typed `agents.subagents.patch` Gateway method, protocol schema, SDK helper, or Admin HTTP exposure exists.

## Solution Assessment

Yes in direction, pending maintainer acceptance. A narrow additive admin-scoped RPC is the right owner-boundary if maintainers approve the API and security contract; raw `config.patch` remains only a workaround that leaks config shape to clients.
