## Summary

Keep open: current main still routes classic voice-call TTS through a fully awaited embedded run, and that run can wait on post-turn compaction after the assistant text is available.

## What This Changes

The issue asks for voice-call embedded responses to be handed to TTS before post-turn compaction can add phone-call latency or contaminate later voice context.

## Best Possible Solution

Add a narrow voice-call early-delivery path, likely via the existing embedded-runner flush/callback boundary, so spoken output reaches TTS before compaction waits while post-turn maintenance still completes safely.
