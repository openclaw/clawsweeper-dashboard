## Summary

Keep open. Current main and v2026.6.8 still make voice-call wait for the completed embedded-agent result while the runner can wait on compaction; the linked open pull request is the candidate fix, but the issue should not close until that lands.

## What This Changes

The issue asks for voice-call to hand the spoken embedded-agent response to TTS before post-turn compaction can add latency or contaminate later context.

## Best Possible Solution

Review and land or replace the existing narrow candidate fix that collects early block replies for voice-call TTS, while keeping the broader sentence-level streaming TTS work in https://github.com/openclaw/openclaw/issues/8355.
