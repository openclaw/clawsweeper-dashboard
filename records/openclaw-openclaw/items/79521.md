## Summary

Keep open: current `main` has the separate voice-agent `tools.allow` prerequisite, but the classic voice-call path still awaits the full embedded run before speaking, and the embedded runner still waits on post-turn compaction unless a pre-compaction flush callback is supplied. The merged fix for the same sequencing problem on messaging channels is a useful pattern, but it does not currently cover voice-call.

## What This Changes

This issue asks for voice-call embedded responses to be handed to TTS before post-turn compaction blocks the turn.

## Best Possible Solution

Add a narrow voice-call low-latency handoff that delivers the final spoken response before the embedded runner waits on post-turn compaction, while keeping compaction and transcript cleanup safe for later turns.

## Reproduction Assessment

Yes for source-level reproduction: current `main` still awaits `generateVoiceResponse()` before `manager.speak()`, and `generateVoiceResponse()` awaits the full embedded runner while the runner waits on compaction unless an early flush callback is supplied. No fresh live Twilio/Ollama reproduction was run.

## Solution Assessment

No current fix is present for voice-call. The best repair is to reuse or mirror the existing pre-compaction delivery seam for the voice response path, rather than disabling compaction globally or adding a new config knob.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79521. In classic voice-call conversation mode, do not wait for post-turn embedded-runner compaction before handing the final spoken response to TTS. Current `main` already forwards routed voice-agent `tools.allow`; preserve that behavior. Use the existing `onBlockReply`/`onBlockReplyFlush` delivery-before-compaction seam or an equivalently narrow voice-call handoff so the first final caller-facing spoken text is delivered before `waitForCompactionRetryWithAggregateTimeout(...)` can delay the run. Avoid globally disabling compaction, avoid adding a config knob, avoid double-speaking, and do not speak partial/reasoning/error payloads. Add regression coverage that simulates a delayed embedded-run completion/compaction wait and proves the voice reply can be sent to `manager.speak()` before the embedded run promise resolves or before compaction wait completion. Also keep or extend coverage for `toolsAllow: []`. Add a user-facing changelog Fixes entry for the voice-call latency fix.
