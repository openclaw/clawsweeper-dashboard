## Summary

Keep open: current main has the separate routed-agent `toolsAllow` fix, but the reported voice latency path still exists because classic voice-call waits for the full embedded run before speaking and does not wire an early block-reply delivery callback before the runner's compaction wait.

## What This Changes

The issue asks for voice-call embedded responses to be handed to TTS before post-turn compaction delays or contaminates the phone session.

## Best Possible Solution

Add a narrow classic voice-call early delivery path, likely via the existing embedded-runner block-reply callbacks, so the spoken JSON response is sent to TTS before post-turn compaction waits while later cleanup remains safe.

## Reproduction Assessment

No full current-main live reproduction was run. The reporter supplied detailed live Twilio/Ollama logs, and current source still shows the same blocking shape after the separate tools-allow fix landed.

## Solution Assessment

Yes, the best direction is a scoped early voice handoff before compaction wait, not a global compaction disable or timeout weakening. The existing block-reply callback seam is the safest starting point, with voice-call-specific parsing and duplicate-delivery guards.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79521. Classic voice-call embedded responses currently await the full `runEmbeddedPiAgent()` result before `manager.speak()`, so post-turn compaction can delay TTS. Implement a narrow voice-call early delivery path that speaks the parsed `{"spoken":"..."}` response before embedded-runner compaction waits, preferably by wiring the existing `onBlockReply`/`onBlockReplyFlush` seam or an equally scoped callback. Preserve routed agent `toolsAllow` behavior, avoid duplicate speaking when the final result returns, do not globally disable or weaken compaction, and keep transcript/session cleanup safe for later turns. Add focused regression tests for delayed embedded completion/compaction with early speech and for no duplicate speech. Add a changelog entry because this is user-visible.
