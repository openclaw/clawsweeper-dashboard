## Summary

Keep open: current main still omits `messages` from the direct `before_compaction` plugin-hook payload, so the PR is solving a real API parity gap, but the current patch is not merge-ready because it passes the live session array and only weakly asserts the non-empty payload.

## What This Changes

The PR adds `messages: session.messages` to the direct compaction `before_compaction` hook payload and adds focused hook-test assertions for non-empty and empty payloads.

## Best Possible Solution

Update the current `runBeforeCompactionHooks` helper to pass a defensive pre-compaction message snapshot, then cover non-empty, empty, and post-compaction stability cases.

## Reproduction Assessment

Yes. Source inspection shows current main computes direct pre-compaction state from `session.messages` but sends only counts to `before_compaction`; I did not execute the focused tests in this read-only review.

## Solution Assessment

No. Adding the field is the right direction, but the current patch should pass a stable snapshot through the current helper boundary and strengthen the regression assertions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Pass a message snapshot to hooks:** `src/agents/pi-embedded-runner/compact.ts:762`
  - body: This passes the live `session.messages` array into plugin hooks. Because hooks run before diagnostics, the real-content guard, and `session.compact(...)`, a handler that splices, sorts, or pushes `event.messages` can change the actual compaction input; pass a shallow pre-compaction snapshot instead.
  - confidence: 0.92
- **[P3] Assert the message payload contents:** `src/agents/pi-embedded-runner/compact.hooks.test.ts:376`
  - body: The new non-empty assertion accepts any array, so always sending `[]` or a stale array would still pass. Assert the expected pre-compaction length or representative contents, and preferably that the captured payload stays stable after compaction mutates session state.
  - confidence: 0.84
