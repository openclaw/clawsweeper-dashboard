## Summary

Keep this PR open because current main still omits `messages` from the direct `before_compaction` plugin-hook payload, but the branch is not mergeable as-is: it passes a live transcript array, has weak payload assertions, is conflicting against main, and only provides mocked test proof.

## What This Changes

The PR adds the direct compaction session messages array to the `before_compaction` hook payload and updates focused hook tests to assert the field.

## Best Possible Solution

Implement this at the current `runBeforeCompactionHooks` helper boundary by passing a defensive pre-compaction message snapshot and adding assertions for exact pre-compaction contents and snapshot stability.

## Reproduction Assessment

Yes, by source inspection: current main computes direct-compaction metrics from `session.messages` but sends only counts to plugin `before_compaction` hooks. I did not run tests because this review is read-only.

## Solution Assessment

No, not as patched. The feature direction is sound, but the branch passes a live transcript array from an outdated call site; the safer fix is a defensive snapshot through the current helper plus stronger tests.
