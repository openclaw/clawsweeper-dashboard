## Summary

Keep open: current main still lacks the direct-path `messages` payload, but this PR should not merge as-is because it passes a live mutable session array, has a weak payload assertion, and still lacks real behavior proof.

## What This Changes

The PR adds the current session message array to the direct compaction `before_compaction` hook payload and extends focused hook tests to assert the field.

## Best Possible Solution

Thread a defensive, validated/limited pre-compaction message snapshot through `runBeforeCompactionHooks`, then cover non-empty, empty, and post-compaction stability cases.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main sends only counts in the direct helper while the hook type already allows `messages` and the wired compaction path asserts a message payload. I did not execute the focused test in this read-only review.

## Solution Assessment

No, not in the current PR form. Adding the field is the right direction, but the maintainable fix is to pass a stable snapshot through the helper boundary and strengthen the regression assertions.
