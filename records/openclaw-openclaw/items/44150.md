## Summary

Keep open. Current main still omits `messages` from the direct `before_compaction` hook payload, and the PR targets a real contract/runtime gap. The branch is not ready as-is because it passes the live `session.messages` array to plugin hooks before compaction continues; it should be refreshed for the current helper split and changed to pass a stable pre-compaction snapshot with stronger payload assertions.

## What This Changes

The PR adds `messages: session.messages` to the direct compaction `before_compaction` hook payload and adds focused hook tests for the new payload field.

## Best Possible Solution

Expose a defensive pre-compaction message snapshot through the direct compaction hook helper, preserve the additive plugin hook contract, and cover both non-empty and empty transcript payloads without changing compaction policy, transcript sanitization, or `after_compaction` behavior.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the gap: the direct helper omits `messages`, while the direct compaction path has `session.messages` available immediately before hooks run and the test harness shows later in-place mutation risk.

## Solution Assessment

No, not in the PR's current form. The feature direction is right, but the maintainable implementation is to pass a pre-compaction snapshot through the current helper rather than a live `session.messages` reference from the stale call site.
