## Summary

Keep open: current main still lacks the requested direct `before_compaction` plugin payload, but this PR should not merge as-is because it passes a live session array, has weak payload coverage, is conflict/dirty against main, and only provides mock/test proof.

## What This Changes

The PR adds `messages: session.messages` to the direct compaction `before_compaction` hook payload and extends hook tests to assert a messages field for non-empty and empty cases.

## Best Possible Solution

Rebase the branch onto the current helper and pass a defensive pre-compaction message snapshot through `runBeforeCompactionHooks`, with tests for exact contents and stability after compaction.

## Reproduction Assessment

Yes by source inspection. `origin/main` builds pre-hook metrics from `session.messages` but `runBeforeCompactionHooks` sends only counts to plugin `before_compaction`; no focused test was executed in this read-only review.

## Solution Assessment

No. The feature direction is right, but this patch passes a live array from an old call site; the maintainable fix belongs in the current helper with snapshot semantics and stronger tests.

## Review Findings

Overall confidence: 0.9
