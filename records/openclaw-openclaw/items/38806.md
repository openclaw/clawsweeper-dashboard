- `impact:message-loss`: The reported failure misroutes bot replies outside the intended Feishu thread, putting messages in the wrong conversation context.

## Summary

Keep open: current main and the latest shipped tag still group-gate Feishu thread-aware reply routing, so P2P thread replies can still fall back to standalone direct messages; the linked open PR is an implementation candidate, not proof of a landed fix.

## What This Changes

The issue requests that Feishu P2P direct-message thread replies stay inside the thread instead of being sent as standalone direct messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent scoped Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes, at source level. A Feishu P2P event can carry root_id/thread_id, but current main still sets replyInThread/threadReply only for groups and passes skipReplyToInMessages: !isGroup, so the direct-message path cannot reach the Feishu thread reply send path.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to treat P2P thread_id as the thread signal, target the thread root, preserve root-id-only P2P quote replies, and leave group behavior unchanged.
