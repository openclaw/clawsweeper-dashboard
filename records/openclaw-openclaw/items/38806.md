## Summary

Keep open: current main still routes Feishu P2P thread events through the non-thread direct-message path, and https://github.com/openclaw/openclaw/pull/73972 is the active fix candidate rather than a reason to close before merge.

## What This Changes

Make Feishu P2P direct-message thread replies stay inside the thread while preserving plain quote replies and existing group behavior.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73972 with a mergeable Feishu plugin patch that treats direct messages carrying thread_id as thread replies, keeps root-id-only DMs as quote replies, and preserves group behavior.

## Reproduction Assessment

Yes, source-level. A Feishu p2p event with root_id and thread_id reaches ctx.rootId/ctx.threadId, then current main sets replyInThread/threadReply false for non-groups and drops reply metadata before send.

## Solution Assessment

Yes at the issue level. The narrow Feishu plugin routing fix in https://github.com/openclaw/openclaw/pull/73972 is the maintainable owner-boundary path; broader Feishu DM session isolation remains separate at https://github.com/openclaw/openclaw/issues/70512.
