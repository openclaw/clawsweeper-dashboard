- `impact:message-loss`: The reported failure misroutes bot replies outside the intended Feishu thread, making messages appear in the wrong conversation context.

## Summary

Keep open: current main still preserves Feishu P2P thread metadata but routes direct-message replies through the non-thread send path, and the open linked PR is the implementation candidate rather than proof that the bug is fixed.

## What This Changes

The issue requests that Feishu P2P direct-message thread replies stay inside the thread instead of being sent as standalone direct messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent scoped Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a Feishu `p2p` event can carry `thread_id` and `root_id`, but current routing leaves `replyInThread` and `threadReply` false and strips reply metadata before sending.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to treat P2P `thread_id` as the thread signal, target the thread root, preserve root-id-only P2P quote replies, and leave group behavior unchanged.
