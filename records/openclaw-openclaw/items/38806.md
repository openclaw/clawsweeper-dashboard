## Summary

Keep open: current main still routes Feishu P2P thread replies through the flat direct-message path, and the linked open repair PR remains the active fix path rather than proof that the issue is implemented.

## What This Changes

The issue asks Feishu P2P direct-message thread replies to stay inside the thread instead of being sent as standalone direct messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent scoped Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes, source-level. A Feishu p2p event with root_id and thread_id reaches ctx, but current direct-message routing leaves replyInThread/threadReply false and strips reply metadata before sending.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to use P2P thread_id as the thread signal, target the thread root, preserve root-id-only quote replies, and leave group behavior unchanged.
