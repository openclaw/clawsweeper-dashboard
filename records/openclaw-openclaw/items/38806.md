## Summary

Keep this open: current main still routes Feishu P2P direct-message thread replies through the non-thread direct-message path, and an open fixing PR is already paired with the issue. The right cleanup is to land one canonical Feishu plugin fix, then close this as implemented.

## What This Changes

The issue requests that Feishu P2P direct-message replies stay inside the originating Feishu thread instead of being sent as standalone direct messages.

## Best Possible Solution

Land one canonical Feishu plugin patch, preferably #73972 if review is clean, that treats direct messages with ctx.threadId as thread replies, targets the thread root, enables replyInThread/threadReply, keeps root-id-only P2P quote replies unchanged, and preserves existing group behavior. Leave broader Feishu DM session isolation to #70512.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu event with chat_type p2p plus root_id and thread_id: current main parses those fields, then because isGroup is false it disables replyInThread/threadReply and strips reply metadata before sending.

## Solution Assessment

Yes. The narrow Feishu plugin routing fix described by the linked PR is the best path because the send layer already supports im.message.reply with reply_in_thread; no core API or cross-plugin change is needed.
