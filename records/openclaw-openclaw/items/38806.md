## Summary

Keep open: current main still routes Feishu P2P thread replies through the direct-message non-thread path, and open PR #73972 is already the canonical fix candidate for this behavior.

## What This Changes

The issue reports that Feishu P2P direct-message thread replies are sent as standalone direct messages instead of staying inside the thread.

## Best Possible Solution

Land one canonical Feishu plugin patch, preferably #73972 if review is clean, that treats direct-message events with thread_id as thread replies while preserving root_id-only P2P quote replies and existing group behavior.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu p2p event with both root_id and thread_id: current main preserves those fields, then direct-chat routing disables replyInThread/threadReply and removes the reply target before the send layer can use im.message.reply.

## Solution Assessment

Yes. The narrow Feishu plugin routing fix described by the paired PR is the best path; broader Feishu DM topic/session isolation remains separate work in #70512.
