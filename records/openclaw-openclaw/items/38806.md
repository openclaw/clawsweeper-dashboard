- `impact:message-loss`: The reported failure misroutes bot replies outside the intended Feishu thread, placing messages in the wrong conversation context.

## Summary

Keep open: current main and the latest release still group-gate Feishu thread reply routing, so P2P thread replies cannot reach the Feishu reply API; https://github.com/openclaw/openclaw/pull/73972 is an open fix candidate but has not landed.

## What This Changes

Feishu P2P direct-message thread replies should stay inside the Feishu thread instead of being sent as standalone main-chat messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent scoped Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes, source-level: current main accepts Feishu p2p events with root_id/thread_id but still computes replyInThread and threadReply only for groups and strips direct-message reply metadata before send. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

Yes at the issue level: the narrow maintainable fix is to treat P2P thread_id as the thread signal, target the thread root, preserve root-id-only P2P quote replies, and leave group behavior unchanged.
