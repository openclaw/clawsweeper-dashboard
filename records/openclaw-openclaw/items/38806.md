## What This Changes

The issue requests that Feishu P2P direct-message thread replies stay inside the thread instead of being sent as standalone direct messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent Feishu plugin patch, so direct messages carrying thread_id target the thread root with thread reply metadata while preserving root-id-only quote replies and group behavior.
