## What This Changes

## Best Possible Solution

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to treat P2P thread_id as the thread signal, target the thread root when available, preserve root-id-only P2P quote replies, and leave group behavior unchanged.
