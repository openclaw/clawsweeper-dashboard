## Summary

Keep open: current main and latest shipped release still route Feishu P2P thread replies through the flat direct-message path, and the open linked repair PR is the active implementation path rather than proof that this issue is fixed.

## What This Changes

The issue asks Feishu P2P direct-message thread replies to stay inside the originating thread instead of being sent as standalone DM messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent scoped Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes, source-level. A Feishu p2p event with thread_id/root_id is parsed into context, but current routing leaves replyInThread/threadReply false and strips reply metadata before sending.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to treat P2P thread_id as the thread signal, target the thread root, preserve root-id-only P2P quote replies, and leave group behavior unchanged.
