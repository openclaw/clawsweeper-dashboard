## Summary

Keep open: current main and the latest release still preserve Feishu P2P thread metadata but route direct-message replies through the flat DM path, while the linked open repair PR is the active fix path rather than proof that this issue is implemented.

## What This Changes

The issue reports that Feishu P2P direct-message thread replies are sent as standalone direct messages instead of staying inside the thread.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent scoped Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes, source-level. A Feishu P2P event with root_id and thread_id reaches ctx, but current direct-message routing leaves replyInThread/threadReply false and strips reply metadata before sending.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to use P2P thread_id as the thread signal, target the thread root when available, preserve root-id-only quote replies, and leave group behavior unchanged.
