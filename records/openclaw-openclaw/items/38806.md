## Summary

Keep open: current main and the latest release still route Feishu P2P thread replies through the non-thread direct-message path, while https://github.com/openclaw/openclaw/pull/73972 is an open unmerged fix candidate.

## What This Changes

The requested change is to make Feishu bot replies in P2P direct-message threads stay inside the thread instead of creating standalone DM messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or replace it with an equivalent Feishu plugin patch that treats direct messages carrying thread_id as thread replies while preserving root-id-only quote replies and group behavior.

## Reproduction Assessment

Yes, source-level. A Feishu p2p event with root_id and thread_id reaches ctx.rootId and ctx.threadId, then current main leaves direct-message replyInThread/threadReply false and strips reply metadata before send.

## Solution Assessment

Yes at the issue level. The narrow Feishu plugin routing fix is the maintainable path because the send layer already supports reply_in_thread and the remaining work is selecting the correct P2P thread reply target.
