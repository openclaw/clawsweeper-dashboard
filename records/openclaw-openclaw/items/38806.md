## Summary

Keep open: current main still parses Feishu P2P thread metadata but direct-message dispatch disables the thread reply path, and https://github.com/openclaw/openclaw/pull/73972 is already the paired canonical fix candidate.

## What This Changes

The issue asks Feishu P2P direct-message thread replies to stay inside the thread instead of being sent as standalone main-chat messages.

## Best Possible Solution

Land one Feishu plugin patch, preferably https://github.com/openclaw/openclaw/pull/73972 if review is clean, that treats direct events with thread_id as thread replies while preserving root_id-only P2P quote replies and existing group behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu p2p event with root_id and thread_id: current main preserves those fields, then direct-message routing disables replyInThread and strips reply metadata before sending.

## Solution Assessment

Yes at the issue level. The narrow Feishu plugin-owned routing fix is the maintainable path because the send layer already supports Feishu thread replies; broader DM topic session isolation remains separately tracked at https://github.com/openclaw/openclaw/issues/70512.
