## Summary

Keep open: current main still parses Feishu P2P thread metadata but routes direct-message replies through the non-thread path, and open PR #73972 is already paired as the canonical fix candidate.

## What This Changes

The issue asks Feishu/Lark P2P thread messages to receive bot replies inside the existing direct-message thread instead of as standalone direct messages.

## Best Possible Solution

Land one canonical Feishu plugin patch, preferably #73972 if review is clean, that treats direct events with thread_id as thread replies while preserving root_id-only P2P quote replies and existing group behavior.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu event with chat_type p2p plus root_id and thread_id: current main preserves those fields, then isGroup=false disables replyInThread/threadReply and strips reply metadata before sending.

## Solution Assessment

Yes. The narrow Feishu plugin routing fix is the best path because the send layer already supports im.message.reply with reply_in_thread; broader Feishu DM session isolation belongs to #70512.
