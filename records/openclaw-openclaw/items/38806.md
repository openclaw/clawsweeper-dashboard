## Summary

Keep open. Current main still preserves Feishu `thread_id`/`root_id` on inbound messages, but the outbound reply routing remains group-gated, so a P2P direct-message thread cannot reach the existing Feishu thread-reply send path. The issue is also paired with open fixing PRs #73972 and #74001, so it should stay open until one canonical fix lands.

## What This Changes

Feishu P2P direct-message thread replies should be sent inside the originating thread instead of as standalone direct messages.

## Best Possible Solution

Land one canonical Feishu plugin patch that treats direct messages with `ctx.threadId` as P2P thread messages, replies to the thread root with `replyInThread` and `threadReply` enabled, preserves the root-id-only P2P quote-reply path, leaves group/topic behavior unchanged, and adds focused regression tests before closing this issue as implemented.
