## Summary

Keep open: current main still routes Feishu P2P thread replies through the direct-message non-thread path, and the linked open PR is an implementation candidate rather than proof that the issue is fixed.

## What This Changes

The issue asks OpenClaw to keep Feishu/Lark P2P direct-message replies inside the originating thread instead of sending a standalone DM reply.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent scoped Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes, source-level. Current main can accept a Feishu p2p event with thread_id/root_id but still computes replyInThread and threadReply only for groups and passes skipReplyToInMessages: !isGroup, so direct replies cannot reach the Feishu thread reply API.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to treat P2P thread_id as the thread signal, target the thread root, preserve root-id-only P2P quote replies, and leave group behavior unchanged.
