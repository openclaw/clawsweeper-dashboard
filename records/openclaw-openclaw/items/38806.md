## Summary

Keep open: current main still routes Feishu/Lark P2P thread replies through the flat direct-message path, and live GitHub data shows an open closing PR already owns the fix path.

## What This Changes

The issue requests Feishu P2P direct-message thread replies stay inside the originating thread instead of posting as standalone main-DM messages.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/73972, or an equivalent current-main Feishu plugin patch, with regression coverage for P2P thread replies and root-id-only P2P quote replies.

## Reproduction Assessment

Yes, source-level. A Feishu p2p event with root_id and thread_id reaches ctx, but current direct-message routing leaves replyInThread/threadReply false and strips reply metadata before sending.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to treat P2P thread_id as the thread signal, target the thread root, preserve root-id-only quote replies, and leave group behavior unchanged.
