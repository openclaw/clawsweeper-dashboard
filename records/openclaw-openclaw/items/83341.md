## Summary

Keep this PR open, but it is not merge-ready: it targets a valid config-set bug, yet the current diff globally disables numeric array container creation and would break documented bracket-index paths.

## What This Changes

The PR changes config set/patch path mutations to always pass `numericObjectKeys: true` and adds a regression test for setting a Discord guild snowflake key.

## Best Possible Solution

Teach the config path mutation layer to preserve bracket/index intent or make schema-aware container choices so numeric Record keys become objects while documented array paths remain arrays.

## Reproduction Assessment

Yes, at source level: current main passes `numericObjectKeys: false` for `config set`, and `setAtPath` can materialize `channels.discord.guilds.<snowflake>` as an array parent even though the schema requires a record. I did not run the CLI because this review is read-only.

## Solution Assessment

No. The fix direction is right for Discord snowflake keys, but forcing object-key semantics globally removes the array auto-creation behavior that documented bracket-index paths depend on.

## Review Findings
