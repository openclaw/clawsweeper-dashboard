## Summary

Keep open: the branch targets a real config set bug, but the current diff forces object-key semantics for every set/merge path and would break documented bracket-index array writes; the supplied proof is a helper probe rather than the patched CLI path.

## What This Changes

The PR changes config set/patch path writes to always use numeric object-key semantics and adds a regression test for setting a Discord guild snowflake key.

## Best Possible Solution

Teach the config path mutation layer to preserve bracket/index intent or make schema-aware container choices so numeric Record keys become objects while documented array paths remain arrays, with regression coverage for both cases.

## Reproduction Assessment

Yes, at source level: current main passes `numericObjectKeys: false` for `config set`, and `setAtPath` can materialize `channels.discord.guilds.<snowflake>` as an array parent even though the schema requires a record. I did not run the CLI because this review is read-only.

## Solution Assessment

No. Preserving numeric Record keys is the right direction, but forcing object-key semantics globally loses bracket-array intent for documented config paths.

## Review Findings

Overall correctness: patch is incorrect
