## Summary

Keep open: the PR targets a real config-set bug, but the branch is not merge-ready because it globally disables numeric-index parent creation and can break documented bracket-array config paths.

## What This Changes

The PR changes `runConfigOperations` to pass `numericObjectKeys: true` for set and patch writes, plus a regression test for a Discord snowflake key under `channels.discord.guilds`.

## Best Possible Solution

Preserve bracket/index intent or use schema-aware container selection so numeric Record keys remain object keys while real array paths still create arrays, with focused tests and redacted CLI proof.

## Reproduction Assessment

Yes. At source level, current `config set` still creates array parents for all-digit next segments while `channels.discord.guilds` is a schema-backed object record; I did not run the CLI in this read-only review.

## Solution Assessment

No. The fix direction is right for snowflake keys, but the implementation should preserve bracket-index intent or choose containers from schema instead of forcing every numeric-looking segment to be an object key.
