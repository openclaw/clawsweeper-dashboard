## Summary

Keep open: the PR addresses the numeric snowflake-key symptom, but the current patch breaks documented bracket-index array creation and the supplied proof does not exercise the real patched CLI path.

## What This Changes

The PR changes `runConfigOperations` so config set/patch always pass `numericObjectKeys: true` and adds a regression test for setting a Discord guild snowflake key.

## Best Possible Solution

Preserve bracket/index intent in parsed path segments, or make container creation schema-aware, so `channels.discord.guilds.<snowflake>` creates an object record while `agents.list[0]` still creates or updates an array; then prove both with focused tests and real CLI terminal output.

## Reproduction Assessment

Yes, at source level: current main shows the original `config set` bug, and the PR diff shows why the proposed fix changes bracket-array creation. I did not execute the CLI because this review is read-only.

## Solution Assessment

No. The fix direction is right for Discord snowflake keys, but the implementation should preserve bracket-index intent or use schema-aware container choices instead of forcing all numeric segments to object keys.
