## Summary

Keep open: current main and the latest release already support manual session labels through the Control UI Sessions table and Gateway `sessions.patch`, but the request still has meaningful unsolved product scope: inline WebChat rename, a CLI rename command, automatic title generation, and tags.

## What This Changes

Request manual session title renaming for WebChat and CLI session management, with improved automatic titles and optional tags for easier session discovery.

## Best Possible Solution

Use the existing `sessions.patch` label contract as the storage/API base, then make a maintainer product call on a narrow inline WebChat rename UI and optional CLI alias while tracking automatic titles and tags separately.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms the remaining gaps because unlabeled dashboard sessions still fall back to key-derived labels and no CLI rename command exists.

## Solution Assessment

Unclear as one bundled change: the existing label API is the right foundation, but inline WebChat rename, CLI rename, automatic title generation, and tags should be split or explicitly prioritized before implementation.
