## Summary

Keep open: the underlying mkdir error-swallowing bug is still present on current main, but this PR edits the obsolete `src/memory/internal.ts` path, so merging it would not change the active memory runtime.

## What This Changes

The PR removes the bare `catch {}` around `mkdirSync(dir, { recursive: true })` in the old `src/memory/internal.ts` `ensureDir` helper.

## Best Possible Solution

Apply the same catch removal to `packages/memory-host-sdk/src/host/internal.ts`, cover propagated mkdir failures in the package tests, and add the required Unreleased changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the active `ensureDir` still catches and discards `mkdirSync` failures, and the runtime path reaches that helper before opening the memory SQLite database.

## Solution Assessment

No as-is. Removing the catch is the right narrow fix, but this PR must be retargeted from the obsolete `src/memory/internal.ts` file to the active memory host SDK helper and include regression coverage plus changelog.
