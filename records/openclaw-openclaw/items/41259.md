## What This Changes

The PR removes the empty catch around memory host SDK `ensureDir` mkdir creation, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Land the focused helper fix after normal maintainer approval and current-head checks so memory indexing reports directory creation failures at the source.

## Reproduction Assessment

Yes, by source inspection and dependency-contract proof. Current main catches every recursive `mkdirSync` failure in `ensureDir`; a file-as-intermediate path would throw `ENOTDIR` from Node but the helper currently returns after the empty catch.

## Solution Assessment

Yes. Removing the bare catch is the narrowest maintainable fix because recursive mkdir already treats an existing directory as success while real filesystem failures should propagate at the helper boundary.
