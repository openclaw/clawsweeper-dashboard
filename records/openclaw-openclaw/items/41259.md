## Summary

This is a focused, valid bug-fix PR: current main still swallows `mkdirSync({ recursive: true })` failures in the memory host SDK helper, while the PR propagates those errors and adds a matching regression test. No blocking review or security findings were found, and the supplied terminal proof is sufficient, so this should stay open for normal maintainer approval rather than cleanup closure.

## What This Changes

The PR removes the empty `catch` from memory host SDK `ensureDir`, adds a mkdir failure regression test, and adds a changelog entry.

## Best Possible Solution

Land the focused helper fix after normal maintainer approval and required checks so memory indexing reports directory creation failures at the source.

## Reproduction Assessment

Yes, by source inspection. Current main catches every recursive `mkdirSync` failure in `ensureDir`, and memory database setup calls that helper before opening SQLite, so file-as-intermediate, permission, read-only, or disk-full failures can be swallowed.

## Solution Assessment

Yes. Removing the bare catch is the narrowest maintainable fix because recursive mkdir already treats an existing directory as success while real filesystem failures should propagate at the helper boundary.
