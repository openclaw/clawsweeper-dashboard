## Summary

Keep open: current main still has the swallowed filesystem error on the active memory path, and the PR is a focused live fix with sufficient real behavior proof and no introduced correctness or security defect.

## What This Changes

The PR changes the memory host SDK `ensureDir` helper to propagate recursive `mkdirSync` failures, adds focused regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the focused helper fix so memory indexing fails at directory creation time instead of returning an unusable directory as ready.

## Reproduction Assessment

Yes, by source inspection. Current main catches every recursive mkdir failure in `ensureDir`, and the active memory database path calls that helper before opening SQLite, so a file-as-intermediate, permission, disk-full, or read-only-filesystem failure would be swallowed at the helper boundary.

## Solution Assessment

Yes. Removing the bare catch is the narrowest maintainable fix because recursive mkdir already handles existing-directory success while real filesystem failures should propagate immediately.
