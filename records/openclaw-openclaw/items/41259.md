## What This Changes

The PR removes a bare try/catch around `mkdirSync(dir, { recursive: true })` in `src/memory/internal.ts` so directory creation failures propagate.

## Best Possible Solution

Retarget the same semantic fix to `packages/memory-host-sdk/src/host/internal.ts`, add focused regression coverage, and add an Unreleased changelog Fixes entry.

## Reproduction Assessment

Yes. Source inspection shows the active `ensureDir` still catches and discards `mkdirSync` failures, and `openMemoryDatabaseAtPath` reaches that helper through the SDK facade; a focused unit test can mock `fsSync.mkdirSync` to throw and assert propagation.

## Solution Assessment

No as-is. Removing the catch is the narrow maintainable fix, but it must be applied to the active memory host SDK helper rather than the removed `src/memory/internal.ts` path.
