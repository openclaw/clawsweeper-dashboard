## Summary

Keep open: current main still swallows `mkdirSync` failures in the active memory host SDK helper, and this PR is a narrow fix with regression coverage, terminal real behavior proof, and clean relevant checks.

## What This Changes

The PR removes the bare catch around memory host `ensureDir` recursive mkdir, adds a regression test for propagated mkdir failures, and adds a changelog fix entry.

## Best Possible Solution

Merge the focused helper fix after normal maintainer landing review, preserving immediate filesystem failure reporting before SQLite opens the memory index.

## Reproduction Assessment

Yes. Current main’s active helper catches and ignores `mkdirSync` failures before returning, and a Node 24 probe shows a file-as-intermediate path throws `ENOTDIR` while existing directories still succeed under recursive mkdir.

## Solution Assessment

Yes. Removing only the catch in the active helper is the narrowest maintainable fix because recursive mkdir already handles existing directories and real filesystem failures should propagate immediately.
