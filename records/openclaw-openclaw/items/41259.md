The PR removes the empty catch around memory host SDK `ensureDir` directory creation, adds regression coverage for surfaced `mkdirSync` failures, and adds a changelog entry.

## Best Possible Solution

Land the focused helper fix after normal maintainer review and required-check gates so memory indexing reports directory creation failures at the source.

## Reproduction Assessment

Yes, by source inspection. Current main catches every recursive mkdir failure in `ensureDir`, and the memory database path calls that helper before opening SQLite, so an unusable directory can be returned as ready.

## Solution Assessment

Yes. Removing the bare catch is the narrowest maintainable fix because recursive mkdir already handles existing-directory success while genuine filesystem failures should propagate at the helper boundary.
