## Summary

Keep open. Current main still swallows `mkdirSync({ recursive: true })` failures in the active memory host SDK helper, while this focused PR removes that catch, adds regression coverage, has sufficient real behavior proof, and has no blocking review or security findings.

## What This Changes

The PR removes the empty `catch` around memory host SDK `ensureDir`, adds a mkdir failure regression test, and adds a changelog entry.

## Best Possible Solution

Land the focused helper fix after normal maintainer approval and required checks so memory indexing reports directory creation failures at the source.

## Reproduction Assessment

Yes, by source inspection. Current main catches every recursive `mkdirSync` failure in `ensureDir`, and memory database setup calls that helper before opening SQLite, so a file-as-intermediate, permission, read-only, or disk-full failure can be swallowed.

## Solution Assessment

Yes. Removing the bare catch is the narrowest maintainable fix because recursive mkdir already treats an existing directory as success while real filesystem failures should propagate at the helper boundary.
