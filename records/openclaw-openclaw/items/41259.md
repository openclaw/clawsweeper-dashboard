## Summary

Keep open: current main still swallows `mkdirSync({ recursive: true })` failures in the memory host SDK helper, while this PR makes the narrow helper change, adds focused regression coverage, and has sufficient after-fix terminal proof.

## What This Changes

The PR removes the empty `catch` around memory host SDK `ensureDir`, adds a regression test for surfaced mkdir failures, and adds a changelog entry.

## Best Possible Solution

Land the focused helper fix after normal required checks are green so memory indexing reports directory creation failures at the source.

## Reproduction Assessment

Yes, by source inspection. Current main catches every recursive mkdir failure in `ensureDir`, and the memory database path calls that helper before opening SQLite, so a permission, read-only, disk-full, or file-as-intermediate failure can be returned as a ready directory.

## Solution Assessment

Yes. Removing the bare catch is the narrowest maintainable fix because recursive mkdir already handles existing-directory success while real filesystem failures should propagate at the helper boundary.
