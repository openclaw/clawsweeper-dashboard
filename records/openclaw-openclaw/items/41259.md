## What This Changes

The PR removes the empty `mkdirSync({ recursive: true })` catch in the memory host SDK `ensureDir()`, adds a regression test for surfaced mkdir failures, and adds a changelog entry.

## Best Possible Solution

Land the focused helper fix through the exact-head automerge path so real memory directory creation failures are reported at the storage boundary instead of being hidden until a later SQLite or indexing error.

## Reproduction Assessment

Yes, from source inspection rather than a locally executed repro. Current main swallows every recursive mkdir failure in `ensureDir()`, and the active memory database path calls that helper before opening SQLite.

## Solution Assessment

Yes. Removing the bare catch is the narrowest maintainable fix because recursive mkdir already handles existing directories, while real filesystem failures such as `ENOTDIR`, permission errors, or disk exhaustion should surface immediately.
