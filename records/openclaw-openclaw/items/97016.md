## Summary

Keep open: current main and the latest release still omit `requestedContextLength` from LM Studio embedding preload, and the open linked PR is the active fix candidate rather than a reason to close the issue before merge.

## What This Changes

The issue asks OpenClaw to make LM Studio embedding preload honor configured `contextTokens`/`contextWindow` so LM Studio does not load embedding models with an overlarge context and trigger CUDA OOM.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/97033; the merged fix should stay provider-local, consume existing model/provider context caps, pass `requestedContextLength` into the load helper, and include focused regression coverage plus redacted load-request proof when feasible.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure an LM Studio embedding model with a lower context cap and current main still calls the load helper without `requestedContextLength`; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes. The best fix is provider-local: resolve the configured LM Studio embedding model or provider `contextTokens`/`contextWindow` and pass it to the existing load helper rather than adding a new config surface.
