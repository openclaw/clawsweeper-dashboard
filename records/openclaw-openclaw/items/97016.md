## Summary

Keep open: current main still omits `requestedContextLength` from LM Studio embedding preload, the latest release has the same omission, and https://github.com/openclaw/openclaw/pull/97033 is the open closing PR that should land or be rejected before this issue closes.

## What This Changes

The issue reports that LM Studio embedding preload ignores configured `contextWindow` or `contextTokens`, causing LM Studio to load a larger context and trigger CUDA OOM during memory search.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/97033; after a fix merges, close this issue as implemented with the merge commit and release provenance.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.10` call the LM Studio model-load helper from the embedding path without `requestedContextLength`, while the helper defaults upward before sending `context_length`; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes. The best fix is provider-local: resolve existing LM Studio embedding model or provider `contextTokens`/`contextWindow` and pass that value to the existing load helper, rather than adding new config or a core fallback.
