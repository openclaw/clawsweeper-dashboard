## Summary

Keep open: current main and `v2026.6.10` still omit `requestedContextLength` from LM Studio embedding preload, and https://github.com/openclaw/openclaw/pull/97033 is the open closing PR that should land or be closed before this issue is resolved.

## What This Changes

The issue requests that LM Studio memory-search embedding preload respect configured `contextTokens`/`contextWindow` instead of loading with the helper default or maximum context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/97033 or an equivalent provider-local fix with focused regression coverage and redacted LM Studio load-request proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure an LM Studio embedding model with a smaller `contextWindow`, then current main still calls the load helper without `requestedContextLength`, causing the helper to default upward; I did not live-reproduce the CUDA OOM.

## Solution Assessment

Yes for the fix direction. The narrow maintainable solution is to resolve existing LM Studio model/provider context config in the embedding preload path and pass it to the existing load helper, not to add a new config surface or core fallback.
