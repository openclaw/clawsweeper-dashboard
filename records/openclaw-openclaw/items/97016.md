## Summary

Keep open: current main and the latest release still omit the configured LM Studio context length when preloading the memory embedding model, and the linked fix remains an open, unmerged PR.

## What This Changes

LM Studio memory embedding preload should honor configured `contextWindow` or `contextTokens` when loading the embedding model for `memory_search`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/97018, or an equivalent provider-local fix, after redacted LM Studio load proof confirms the configured context length is sent.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure an LM Studio embedding model with a smaller context cap and trigger `memory_search`; current main calls the load helper without `requestedContextLength`. I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes, the best fix is provider-local: resolve the configured LM Studio embedding model or provider context cap and pass it to the existing load helper without adding a new config surface.
