## Summary

Keep open: current main and the latest release still omit `requestedContextLength` from LM Studio embedding preload, and the open linked PR is the active fix candidate rather than a reason to close the issue before merge.

## What This Changes

The issue reports that LM Studio embedding preload ignores configured `contextWindow`/`contextTokens`, causing model preload to request an oversized context and trigger CUDA OOM in LM Studio.

## Best Possible Solution

Land one narrow LM Studio provider fix that resolves the configured embedding model or provider context cap and passes it into the existing load helper, with focused regression coverage and redacted LM Studio load-request proof when feasible.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main and `v2026.6.10` omit `requestedContextLength` in LM Studio embedding preload while the load helper defaults upward before sending `context_length`; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes. The best fix is provider-local: resolve the existing LM Studio embedding model or provider `contextTokens`/`contextWindow` and pass it to the existing load helper rather than adding a new config surface.
