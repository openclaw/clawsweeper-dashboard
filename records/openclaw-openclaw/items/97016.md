## Summary

Keep open: current main still creates the LM Studio memory embedding provider without passing the configured model context length into the native load call, so the reported OOM path remains a real provider-runtime bug.

## What This Changes

Memory search's LM Studio embedding warmup should honor the configured embedding model context budget when preloading the model.

## Best Possible Solution

Mirror the inference preload context resolution in the LM Studio embedding factory, add focused regression coverage, and avoid adding a new config surface.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure an LM Studio embedding model with a lower context window and trigger memory search, and current main will call the load helper without `requestedContextLength`. I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes, the best fix is provider-local: resolve the configured LM Studio model's `contextTokens` or `contextWindow` and pass it to the existing load helper, with tests proving the request body context length.
