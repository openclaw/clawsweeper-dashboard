- `impact:auth-provider`: The issue affects LM Studio provider model-loading behavior and configured runtime context selection for embeddings.

## Summary

Keep open: current main and `v2026.6.10` still omit `requestedContextLength` from LM Studio embedding preload, and https://github.com/openclaw/openclaw/pull/97033 is already the open closing fix candidate.

## What This Changes

LM Studio embedding preload should honor configured `contextTokens`/`contextWindow` for `memory_search` model loading so it does not load at the model's maximum context and exhaust GPU memory.

## Best Possible Solution

Land a narrow provider-local fix that resolves existing LM Studio model/provider `contextTokens` or `contextWindow` and passes it into `ensureLmstudioModelLoaded`, with focused regression coverage and redacted load-request proof when feasible.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure an LM Studio embedding model with a lower context cap and current main still calls the load helper without `requestedContextLength`; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes. The best fix is provider-local: resolve existing LM Studio `contextTokens`/`contextWindow` config for the embedding preload path and pass it to the existing load helper rather than adding a new config surface.
