## Summary

Keep open: current main and the latest release still omit the configured LM Studio context length from the memory embedding preload call, while the related preload opt-out work only avoids preload instead of fixing the enabled-preload context length.

## What This Changes

The issue reports that LM Studio memory-search embedding preload ignores configured model context length and can load a larger context that exhausts GPU memory.

## Best Possible Solution

Keep this open for a narrow LM Studio provider fix that reuses existing context-length semantics for embedding preload and adds regression coverage for the preload request body.

## Reproduction Assessment

Not fully with live hardware; source inspection does show a deterministic path where embedding preload omits `requestedContextLength` while the load helper falls back to the larger default target. The reporter's LM Studio logs and CUDA OOM are plausible but were not independently reproduced here.

## Solution Assessment

Yes, the best fix is a narrow provider-owned repair that passes existing configured model context metadata into the LM Studio load helper. Adding a new config key or a second preload mechanism would be the wrong direction.
