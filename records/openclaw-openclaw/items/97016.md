## Summary

Keep open: current main and the latest release still omit the configured context length from LM Studio embedding preload, and the linked fix is an open candidate PR rather than a merged implementation.

## What This Changes

The issue reports that LM Studio embedding preload ignores the configured model context window during memory search, causing LM Studio to load the embedding model with a larger context and potentially exhaust CUDA memory.

## Best Possible Solution

Keep the issue open until https://github.com/openclaw/openclaw/pull/97018 or an equivalent LM Studio plugin fix lands with focused regression coverage for embedding preload context length.

## Reproduction Assessment

Yes from source: current main's embedding preload omits `requestedContextLength`, and the load helper falls back to the default load context when that value is missing. I did not run a live LM Studio CUDA repro in this read-only review.

## Solution Assessment

Yes, the narrow best fix is in the LM Studio provider's embedding preload path: resolve the configured embedding model's effective context length and pass it to `ensureLmstudioModelLoaded`, matching the inference preload invariant.
