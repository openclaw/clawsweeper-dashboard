## Summary

Keep open: current main and the latest release still omit the configured context length from LM Studio embedding preload, and the open linked pull request is the active fix candidate rather than a reason to close the issue before it merges.

## What This Changes

Respect configured LM Studio model/provider context caps when preloading embedding models for memory search so the load request does not default to a larger context.

## Best Possible Solution

Review and either land or close the linked provider-local fix; the desired end state is for LM Studio embedding preload to pass the existing configured model/provider context cap into the existing load helper with focused regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main and `v2026.6.10` omit `requestedContextLength` in LM Studio embedding preload while the load helper defaults upward before sending `context_length`; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes. The best fix is provider-local: resolve the configured LM Studio embedding model or provider `contextTokens`/`contextWindow` and pass it to the existing load helper rather than adding a new config surface.
