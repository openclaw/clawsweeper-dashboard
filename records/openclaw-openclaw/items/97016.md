## Summary

Keep open: current main and the latest release still omit `requestedContextLength` from LM Studio embedding preload, and https://github.com/openclaw/openclaw/pull/97033 is the active open closing PR rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to make LM Studio embedding preload honor configured `contextTokens` or `contextWindow` so memory search does not load embedding models at an excessive context length.

## Best Possible Solution

Land one narrow LM Studio provider fix that resolves model/provider `contextTokens` or `contextWindow` and passes `requestedContextLength` into `ensureLmstudioModelLoaded`, with focused regression coverage and redacted LM Studio load-request proof when feasible.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence because current main calls LM Studio embedding preload without `requestedContextLength`, while the load helper defaults absent values upward before sending `context_length`; I did not run a live CUDA reproduction.

## Solution Assessment

Yes. The best fix boundary is provider-local: resolve existing LM Studio model/provider context config and pass it to the existing load helper, then keep this issue open until the active linked PR lands or is rejected.
