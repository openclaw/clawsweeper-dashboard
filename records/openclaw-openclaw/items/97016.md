## Summary

Keep open: current main and the latest release still omit the configured context length from LM Studio embedding preload, and the open linked fix PR is the active candidate rather than a reason to close this issue before merge.

## What This Changes

The issue reports that LM Studio embedding preload ignores configured `contextWindow`/`contextTokens`, causing LM Studio to load a larger context and OOM on limited CUDA GPUs.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/97033; the end state should be a provider-local fix that passes existing LM Studio context config into embedding preload with focused regression coverage and, ideally, redacted live LM Studio load-request proof.

## Reproduction Assessment

Yes for source reproduction: current main’s LM Studio embedding preload omits `requestedContextLength`, while the load helper defaults missing values to the larger load target. I did not run a live LM Studio CUDA setup, so the OOM symptom itself remains user-reported rather than locally reproduced.

## Solution Assessment

Yes. Passing the already-configured LM Studio model or provider context cap into the plugin-owned embedding preload path is the narrow maintainable fix; the open linked PR is the current implementation candidate.
