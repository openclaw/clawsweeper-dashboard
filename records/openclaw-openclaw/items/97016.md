## Summary

Keep open: current main and the latest release still omit the configured LM Studio context length during embedding preload, and the open closing PR is only a candidate fix until it merges.

## What This Changes

The issue asks OpenClaw to pass the configured LM Studio model context cap into memory-search embedding preload so LM Studio does not load the embedding model at its maximum context length.

## Best Possible Solution

Review and land one narrow LM Studio plugin fix, likely https://github.com/openclaw/openclaw/pull/97033, with regression coverage and redacted LM Studio load-request proof, then close this issue as implemented.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main calls the LM Studio embedding preload helper without `requestedContextLength`, so a lower configured context cap is ignored. I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes, the best fix is provider-local: reuse existing `contextTokens`/`contextWindow` config and pass the resolved value into the existing LM Studio load helper, with focused regression coverage.
