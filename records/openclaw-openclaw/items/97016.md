## What This Changes

The issue reports that LM Studio embedding preload ignores the configured embedding context window and can load too large a context, causing CUDA OOM during memory search.

## Best Possible Solution

Review and land the narrow provider-local fix in https://github.com/openclaw/openclaw/pull/97033 after it proves the configured context_length reaches LM Studio, or close that PR if maintainers reject the fix shape.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main still omits requestedContextLength in LM Studio embedding preload while the helper defaults upward before sending context_length; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes for the code direction. The best fix is provider-local: resolve existing LM Studio model/provider contextTokens or contextWindow and pass it to the existing load helper, with live LM Studio request proof if feasible.
