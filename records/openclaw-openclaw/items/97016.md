## Summary

Keep open: current main and v2026.6.10 still omit `requestedContextLength` from LM Studio embedding preload, and https://github.com/openclaw/openclaw/pull/97033 is the open closing PR rather than a reason to close this issue before merge.

## What This Changes

The issue reports that LM Studio `memory_search` embedding preload ignores configured `contextWindow` and loads too large a context, causing CUDA OOM.

## Best Possible Solution

LM Studio embedding preload should resolve the effective model/provider context cap and pass it to the existing load helper, with focused regression tests and redacted LM Studio load-request proof where feasible.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure an LM Studio embedding model with a lower context window and current main still calls the load helper without `requestedContextLength`; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes. The best fix is provider-local: resolve the configured LM Studio embedding model/provider context cap and pass it to the existing load helper rather than adding a new config surface.
