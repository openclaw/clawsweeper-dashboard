## Summary

Keep open: current main and the latest release still omit `requestedContextLength` from LM Studio embedding preload, and the open linked PR is the active fix candidate rather than a reason to close the issue before merge.

## What This Changes

The issue reports that LM Studio embedding preload ignores configured context length and can load embeddings at the model's maximum context, causing CUDA out-of-memory failures.

## Best Possible Solution

Land the open linked fix PR once maintainers accept its LM Studio behavior proof, or close that PR and then queue one narrow replacement fix against the same provider-owned embedding preload path.

## Reproduction Assessment

Yes from source inspection: the embedding preload path omits `requestedContextLength`, while the shared LM Studio load helper falls back to its default load context when the parameter is absent. I did not run a live LM Studio CUDA repro in this read-only review.

## Solution Assessment

Yes in shape: the fix belongs in the LM Studio plugin embedding provider, resolving existing model/provider context config and passing it into `ensureLmstudioModelLoaded`. The open linked PR owns that repair path, so the issue should stay open until that PR lands or is replaced.
