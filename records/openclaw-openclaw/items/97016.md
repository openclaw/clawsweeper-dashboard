## Summary

Keep open: current main and the latest release still omit the configured LM Studio embedding preload context length, and https://github.com/openclaw/openclaw/pull/97033 is the open closing PR that should land or close before this issue is resolved.

## What This Changes

The issue reports that LM Studio embedding preload ignores configured `contextWindow`/`contextTokens` and can load embeddings with an oversized context that causes CUDA OOM.

## Best Possible Solution

Land one narrow LM Studio provider fix that resolves the configured embedding model or provider context cap and passes it into the existing load helper, with focused regression coverage and redacted LM Studio load-request proof when feasible.

## Reproduction Assessment

Yes. Current main calls the LM Studio embedding load helper without `requestedContextLength`, while the helper defaults and sends `context_length`; I did not run a live CUDA reproduction.

## Solution Assessment

Yes. The narrow maintainable solution is provider-local: resolve existing LM Studio `contextTokens`/`contextWindow` config for the embedding model or provider and pass it to the existing load helper instead of adding a new config surface.
