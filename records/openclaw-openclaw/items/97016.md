## Summary

Keep open: current main and v2026.6.10 still preload LM Studio embedding models without passing the configured context length, and https://github.com/openclaw/openclaw/pull/97033 is the open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks OpenClaw to respect configured LM Studio embedding model/provider `contextTokens` or `contextWindow` during `memory_search` preload so LM Studio does not load the model at its maximum context and trigger CUDA OOM.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/97033 or an equivalent narrow LM Studio plugin fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main's embedding provider omits `requestedContextLength`, while the load helper defaults absent values upward before sending `context_length`; I did not reproduce the CUDA OOM live.

## Solution Assessment

Yes. The best fix is provider-local: resolve the configured LM Studio embedding model or provider `contextTokens`/`contextWindow` and pass it to the existing load helper without adding a new config surface.
