## Summary

Keep open. The feature is still not implemented on current main, but this PR should not merge as-is because it introduces a public prompt/config surface with unresolved docs/schema metadata, CLI session reuse, real-proof, and prompt-file trust-boundary issues.

## What This Changes

The branch adds an optional `agents.defaults.systemPromptFile` config field, reads the configured local file into CLI and embedded runner system prompts, and adds helper tests.

## Best Possible Solution

Define one maintainer-approved additive prompt-default contract with explicit file/include semantics, session-reuse hashing, docs/schema metadata, and trust-boundary validation before landing this branch or a replacement.

## Reproduction Assessment

Yes. Source review is enough for the PR defects: current main lacks `agents.defaults.systemPromptFile`, and the PR reads the file during prompt building after CLI session reuse hashing has already been computed.

## Solution Assessment

No. The runner hook points are plausible, but this is not the best merge shape until config docs/metadata, CLI session invalidation, real behavior proof, and the file mutability boundary are resolved and aligned with the broader prompt-default design.
