## Summary

Keep this PR open for contributor and maintainer follow-up: the requested file-based prompt config is not on current main, but the branch still has merge-blocking config-surface, session-reuse, trust-boundary, and real-behavior-proof gaps.

## What This Changes

The branch adds an optional `agents.defaults.systemPromptFile` config key, reads that local file into CLI and embedded-runner system prompts, and adds helper tests.

## Best Possible Solution

Define one approved agent-level prompt-injection contract with file/include semantics, schema/types/docs/help/baseline coverage, session-reuse hashing, and an explicit trust boundary before landing this branch or a replacement.

## Reproduction Assessment

Yes for the PR blockers. Source inspection shows current main lacks `agents.defaults.systemPromptFile`, and the PR reads the file during prompt building after CLI session reuse hashing has already been computed.

## Solution Assessment

No. The runner hook points are plausible, but this is not the best merge shape until config docs/metadata, CLI session invalidation, real behavior proof, and the file mutability boundary are resolved and aligned with the broader prompt-default design.
