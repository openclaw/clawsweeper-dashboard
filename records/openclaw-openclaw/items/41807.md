## Summary

Keep open: current main does not already implement `agents.defaults.systemPromptFile`, and the PR still has merge-blocking gaps around public config surfaces, CLI session invalidation, the prompt-file trust boundary, and contributor proof.

## What This Changes

The PR adds an optional `agents.defaults.systemPromptFile` key, reads the configured file, and prepends its contents to CLI and embedded runner system prompts with helper tests.

## Best Possible Solution

Define one maintainer-approved agent-level prompt-injection contract covering file/include semantics, trust boundary, session-reuse invalidation, public config metadata/docs, and both CLI and embedded session creation paths.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main hashes CLI prompt inputs before prompt construction, while the PR reads the file later, and the branch omits public config follow-through. The requested behavior itself is a new feature rather than a current-main bug reproduction.

## Solution Assessment

No. The hook points are plausible, but this is not the best merge shape until the config contract, prompt-file trust boundary, session invalidation behavior, and real runtime proof are settled.
