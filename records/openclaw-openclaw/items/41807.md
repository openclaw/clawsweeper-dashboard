## Summary

Keep open: current main still lacks a global file-based prompt default, but this PR is not merge-ready because the public config/schema surface is incomplete, CLI session reuse can miss the injected file prompt, and the prompt-file trust boundary needs maintainer judgment alongside #50103.

## What This Changes

The PR adds `agents.defaults.systemPromptFile`, reads the configured local file, prepends its trimmed contents to CLI and embedded runner system prompts, and adds helper coverage for file reading.

## Best Possible Solution

Land a maintainer-reviewed global prompt-default design that defines file/path mutability semantics, updates generated config schema/help/docs and changelog, includes prompt-file identity or content in CLI session invalidation, and reconciles the final API with #50103.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main lacks the `agents.defaults.systemPromptFile` key and runtime file read, and the CLI reuse defect follows from hashing `params.extraSystemPromptStatic` before the PR's later file read.

## Solution Assessment

No. The hook point is plausible, but this is not the best merge shape until public config metadata/docs, prompt-file trust semantics, CLI session invalidation, and changelog coverage are handled.
