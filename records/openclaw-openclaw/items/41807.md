## Summary

Keep this open for maintainer review: current main still lacks file-backed agent prompt injection, but the branch needs real behavior proof and a maintainer-approved config/security contract before merge.

## What This Changes

Adds an optional `agents.defaults.systemPromptFile` config key that reads an operator-specified file into CLI and embedded system prompts, plus type/schema entries and reader tests.

## Best Possible Solution

Define one maintainer-approved agent-level prompt-injection contract covering file/include semantics, trust boundary, precedence, session invalidation, config metadata/docs, and both CLI and embedded session creation paths.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main computes CLI reuse hashes before any prompt-file read, and public config metadata/docs still lack the new key. The requested behavior is a feature rather than a current-main bug.

## Solution Assessment

No. The hook points are plausible, but the safer merge shape is a central prompt-config contract with an explicit trust boundary and session invalidation behavior, not runner-local file reads plus a schema/type addition.
