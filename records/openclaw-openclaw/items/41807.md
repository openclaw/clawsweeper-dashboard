## Summary

Keep open. Current main still lacks a global file-based prompt default, and the PR addresses a real feature request, but the patch is not merge-ready because the public config surface is incomplete, CLI session reuse can miss the new prompt content, and the prompt-file trust boundary overlaps the broader open design in #50103.

## What This Changes

The PR adds an optional `agents.defaults.systemPromptFile` config key, reads the configured local file, and prepends its contents to CLI and embedded runner system prompts.

## Best Possible Solution

Land a maintainer-approved global prompt-default design that updates schema/help/docs/changelog, includes prompt-file identity or content in CLI session reuse, and explicitly defines whether this is a file-only API or part of the broader #50103 rules surface.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main lacks the requested config key, and the PR's CLI reuse defect follows from computing `extraSystemPromptHash` before the later prompt-file read.

## Solution Assessment

No. The hook point is plausible, but this is not the best merge shape until public config metadata, docs/changelog, CLI session invalidation, and prompt-file trust semantics are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Regenerate the public config schema surface:** `src/config/zod-schema.agent-defaults.ts:41`
  - body: Adding a public `agents.defaults` key only to Zod/types leaves the generated schema, UI labels/help, config docs, and docs baseline stale. `buildBaseConfigSchema()` serves the generated schema, so `openclaw config schema` and Control UI metadata will not expose this key until those surfaces are updated.
  - confidence: 0.91
- **[P2] Include prompt-file content in CLI session reuse:** `src/agents/cli-runner/helpers.ts:75-81`
  - body: The file content is appended inside `buildSystemPrompt()`, after `prepare.ts` has already computed `extraSystemPromptHash`. Reusable CLI sessions are therefore not invalidated when this config is added or the file changes, so resumed CLI runs can keep running without the operator prompt.
  - confidence: 0.86
- **[P2] Define an immutable prompt-file boundary:** `src/agents/system-prompt-file.ts:13-16`
  - body: The helper rereads the configured path each session while the feature is positioned as persistent, agent-unmodifiable guidance. If the path is relative, workspace-local, or otherwise agent-writable, the agent can affect future prompts, so the patch should enforce, snapshot, validate, or clearly document the required outside-workspace deployment model.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/config/zod-schema.agent-defaults.ts:41`
  - body: This is a user-facing feature, but the branch does not update `CHANGELOG.md`. OpenClaw policy requires shipped `feat` changes to include a changelog entry with appropriate contributor credit before merge.
  - confidence: 0.88
