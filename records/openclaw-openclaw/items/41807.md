## Summary

Keep open: current main still does not implement the requested file-backed agent-default prompt injection, but this PR is not merge-ready because it leaves config follow-through, CLI session reuse invalidation, prompt-file trust-boundary, and real behavior proof gaps.

## What This Changes

The PR adds an optional `agents.defaults.systemPromptFile` key that reads a configured local file and prepends its contents to CLI and embedded runner system prompts.

## Best Possible Solution

Define one maintainer-approved agent-level prompt-injection contract covering include/file semantics, trust boundary, precedence, session invalidation, config metadata/docs, and both CLI and embedded session creation paths.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main decides CLI saved-session reuse from `extraSystemPromptHash` before the PR's helper would read the prompt file, and public config metadata/docs still lack the new key. The requested behavior itself is a feature rather than a current-main bug reproduction.

## Solution Assessment

No. The hook points are plausible, but this is not the best merge shape until the config contract, prompt-file trust boundary, CLI reuse invalidation behavior, and real runtime proof are settled.
