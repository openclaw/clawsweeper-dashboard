## Summary

Keep this PR open: current main still does not implement `agents.defaults.systemPromptFile`, but the branch is not merge-ready because it lacks inspectable real behavior proof and leaves prompt-file trust, public config, and CLI session-reuse behavior unsettled. The broader prompt-config request remains open at https://github.com/openclaw/openclaw/issues/50103, so this is useful source work rather than cleanup-close material.

## What This Changes

The PR adds an optional `agents.defaults.systemPromptFile` config key that reads a local file and prepends its contents to CLI and embedded runner system prompts, with helper tests.

## Best Possible Solution

Define one maintainer-approved agent-level prompt-injection contract covering file/include semantics, trust boundary, session-reuse invalidation, public config metadata/docs, and both CLI and embedded session creation paths.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main hashes CLI prompt inputs before prompt construction, while the PR reads the file later, and the branch also omits public config follow-through. The requested behavior itself is a new feature rather than a current-main bug reproduction.

## Solution Assessment

No. The hook points are plausible, but this is not the best merge shape until the config contract, prompt-file trust boundary, session invalidation behavior, and real runtime proof are settled.
