## Summary

Close: the requested arbitrary basename support is an optional bootstrap hook capability that can already be implemented outside core through the mutable `agent:bootstrap` hook context, while the bundled `bootstrap-extra-files` hook intentionally stays constrained to recognized bootstrap basenames.

## What This Changes

Request support for configured user-defined `bootstrap-extra-files` basenames such as `SESSION_STATE.md`, either with an `allowArbitraryBasenames` option or by adding `SESSION_STATE.md` to the recognized bootstrap filenames.

## Best Possible Solution

Keep the bundled hook’s conservative basename allowlist and publish a focused ClawHub hook pack or plugin that injects `SESSION_STATE.md` or configured filenames via `agent:bootstrap` with explicit safety documentation.

## Reproduction Assessment

Not applicable as a bug reproduction: the issue requests a new config capability. Source and docs confirm the current allowlist behavior, and tests confirm arbitrary bootstrap entries are possible through a custom hook.

## Solution Assessment

No, adding a new arbitrary-basename knob to the bundled core hook is not the best default path. The narrower maintainable solution is a ClawHub hook/plugin using the existing `agent:bootstrap` extension point.
