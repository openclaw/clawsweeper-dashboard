## Summary

Keep open: the bug is valid, but this PR targets removed legacy browser-tool files rather than the active `extensions/browser` plugin, so it would not make `browser:screenshot` honor `agents.defaults.imageMaxDimensionPx` on current `main`. It also needs real behavior proof before merge.

## What This Changes

The branch adds config plumbing and tests to the old core browser tool so screenshot results pass `imageSanitization` from `agents.defaults.imageMaxDimensionPx`.

## Best Possible Solution

Port the behavior into `extensions/browser`: pass plugin context config into the active browser tool and pass `resolveImageSanitizationLimits(config)` to the screenshot `imageResultFromFile()` call with focused plugin tests.

## Reproduction Assessment

Yes. A current-main source trace shows the docs define the config, plugin context exposes it, and the active browser screenshot path returns `imageResultFromFile()` without `imageSanitization`; I did not run a live browser screenshot in this read-only review.

## Solution Assessment

No. The sanitizer idea is the right narrow fix, but changing the old core browser tool is not the maintainable path because current `main` owns browser tooling under `extensions/browser`.
