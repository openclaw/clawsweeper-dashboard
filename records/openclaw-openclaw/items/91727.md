## Summary

Keep open: current main and v2026.6.8 still skip the history reload when the terminal final carries renderable assistant text, so the persisted reasoning block is not repainted until a manual history refresh; https://github.com/openclaw/openclaw/pull/91810 is the open closing PR that owns the focused fix.

## What This Changes

The issue reports a WebChat regression where persisted reasoning/thinking blocks are absent from the live display for `reasoningDefault: "on"` until the browser refreshes history.

## Best Possible Solution

Review and land or intentionally replace https://github.com/openclaw/openclaw/pull/91810, then close this issue once the deferred-thinking repaint fix reaches main.

## Reproduction Assessment

Yes, by source inspection rather than a live browser run: a thinking-bearing `session.message` during an active run followed by a renderable text-only final leaves `shouldReloadHistoryForFinalEvent` false, so current main does not reload the persisted transcript.

## Solution Assessment

Yes, the best current path is the linked PR's narrow behavior: track thinking-bearing deferred session messages and reload only when the terminal final lacks that thinking, preserving the no-reload path for ordinary renderable finals.
