## Label Justifications

- `P2`: This is a normal-priority WebChat regression with a focused open fix and limited blast radius.
- `impact:session-state`: The failure is caused by live session/message reconciliation not repainting persisted reasoning state until history is reloaded.
- `impact:message-loss`: Reasoning content is captured and persisted but effectively absent from the live WebChat message display until manual refresh.

## Summary

Keep open: current main and the latest release still suppress the history reload when a matching terminal final contains renderable assistant text, so persisted WebChat reasoning is not repainted until a manual history refresh; the open closing PR at https://github.com/openclaw/openclaw/pull/91810 owns the focused fix.

## What This Changes

WebChat should repaint persisted reasoning/thinking in the live transcript after a run completes, without requiring a manual refresh.

## Best Possible Solution

Review and land or intentionally replace https://github.com/openclaw/openclaw/pull/91810, then close this issue once the deferred-thinking repaint fix reaches main.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: a thinking-bearing `session.message` during an active run followed by a renderable text-only final leaves `shouldReloadHistoryForFinalEvent` false, so current main does not reload the persisted transcript.

## Solution Assessment

Yes. The best current path is the linked PR's narrow behavior: track thinking-bearing deferred session messages and reload only when the terminal final lacks that thinking, while preserving the no-reload path for ordinary renderable finals.
