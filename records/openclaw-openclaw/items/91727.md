## Label Justifications

- `P2`: This is a normal-priority WebChat regression with a focused open fix and limited blast radius to Control UI chat rendering.
- `impact:session-state`: The failure is caused by live session/message reconciliation not repainting persisted reasoning state until history is manually reloaded.
- `impact:message-loss`: Reasoning content is captured and persisted but effectively absent from the live WebChat message display until manual refresh.

## Summary

Keep open: current main and v2026.6.9 still skip the WebChat history reload when the terminal final has renderable assistant text, so persisted reasoning can remain hidden until manual refresh; https://github.com/openclaw/openclaw/pull/91810 is the open focused closing PR.

## What This Changes

The issue asks WebChat to show persisted reasoning in the live/repainted transcript when a thinking-bearing session update is followed by a text-only final event, without requiring manual refresh.

## Best Possible Solution

Review and land or intentionally replace https://github.com/openclaw/openclaw/pull/91810, then close this issue once the deferred-thinking repaint fix reaches main.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: a thinking-bearing session.message during an active run followed by a renderable text-only final leaves shouldReloadHistoryForFinalEvent false, so current main does not reload the persisted transcript.

## Solution Assessment

Yes. The best current path is the linked PR's narrow behavior: track thinking-bearing deferred session messages and reload only when the terminal final lacks that thinking, while preserving the no-reload path for ordinary renderable finals.
