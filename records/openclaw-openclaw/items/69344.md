## Summary

Keep open. Current main has not implemented this PR's intended TUI copy-preservation change, but the proposed patch weakens the existing narrow-terminal sanitizer and has actionable correctness findings before merge; open #39565 overlaps the dotted-token portion without superseding the broader PR.

## What This Changes

The PR raises the TUI render sanitizer long-token threshold from 32 to 128, adds a dot/parenthesis/bracket copy-sensitive exemption, and updates formatter tests.

## Best Possible Solution

Revise before merge: keep the generic 32-character wrapping guard for ordinary tokens, add a constrained copy-sensitive classifier for real code/entity/file-like fragments, and retain regression tests for both preserved copy-sensitive examples and generic overflow protection.

## Reproduction Assessment

Yes. Static reproduction against the PR head is high-confidence: a 90-character ordinary token no longer matches the 129+ regex, and a >128-character token with incidental punctuation reaches the new copy-sensitive branch and bypasses chunkToken.

## Solution Assessment

No. The PR preserves some code-like snippets by weakening the generic layout guard; the safer solution is to keep the existing guard and narrowly expand isCopySensitiveToken for specific copy-sensitive token shapes.
