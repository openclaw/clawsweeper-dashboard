## Summary

Keep open: current main now has a merged managed-npm generation-retention slice, managed update handoff, and lifecycle eager-loading, but the issue discussion and source still show the broader stale hashed-import invariant is not settled for raw/manual package or plugin swaps, shutdown paths, and in-flight/recovered runtimes carrying old module graphs.

## What This Changes

The issue reports stale hashed bundle imports after OpenClaw updates when a live gateway or recovered runtime keeps an old module graph while package files rotate on disk.

## Best Possible Solution

Keep this issue open, narrow it to raw/manual package or plugin swaps plus in-flight/recovered runtimes, and land a maintainer-approved recovery boundary with live cross-platform package-swap proof.

## Reproduction Assessment

Yes at source level, but not as a fresh live run after the latest managed-npm merge. The reproduction path is a raw/manual package or plugin swap, shutdown, or in-flight/recovered runtime retaining an old module graph and then hitting a lazy import for a removed hashed chunk.

## Solution Assessment

No complete solution is merged yet. The managed-npm retention and lifecycle eager-load work are useful partial fixes, but the best fix still needs a process/update boundary or bounded stale-own-dist detection that covers raw swaps and request/task paths without restart thrash.
