## Summary

Keep open: the latest v2026.6.8 systemd retest still reports stale hashed imports during the old gateway shutdown, and current main only shows partial handoff/recovery mitigation rather than proof that the old-process/new-files window is closed.

## What This Changes

The issue reports that auto-update can rotate hashed OpenClaw and bundled plugin chunks while a gateway still has cached imports to the removed files.

## Best Possible Solution

Keep this as the canonical lifecycle issue and choose a durable update contract: quiesce old gateway work before package rotation, preserve old chunks until exit, or validate/restart before any recovered or shutdown path can touch deleted hashed imports.

## Reproduction Assessment

No, not as a live current-main repro: the review did not run a packaged systemd upgrade/restart-recovery lane. The issue is source-reproducible at medium confidence from the v2026.6.8 logs plus current package-root rotation and lazy-import boundaries.

## Solution Assessment

Unclear. The current managed-service handoff is the right direction, but the best durable fix still needs a maintainer decision on update/shutdown/recovery policy rather than a one-off chunk shim or reply-loader warning.
