## Summary

Keep open: current main and v2026.5.20 have broad package-update stop/handoff safeguards, but they do not prove the exact reported Codex run-attempt stale hash is covered. The searched tree has no `run-attempt-*.js` legacy alias, stale post-swap imports remain acknowledged in update smoke tooling, and a same-author release-stability consolidation issue is still open.

## What This Changes

The issue asks OpenClaw to prevent auto-update from leaving a live gateway with cached modules that import removed hashed Codex bundle chunks.

## Best Possible Solution

Keep the managed-service stop/handoff path as the default, then either add a focused Codex stale-chunk compatibility path with the old hash or prove through installed-artifact update tests that the live gateway cannot keep referencing removed chunks.

## Reproduction Assessment

No high-confidence current-main live reproduction was established. The report is plausible and source/tests show stale post-swap imports are a real update hazard, but the exact old `run-attempt` hash and packaged auto-update path are missing.

## Solution Assessment

Unclear. Current safeguards are in the right ownership area, but maintainers need to choose whether the remaining fix is a targeted Codex chunk alias, a stronger restart/handoff guarantee, or an installed-artifact integrity check.
