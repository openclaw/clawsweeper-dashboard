## Summary

## What This Changes

The issue asks for auto-update/package-swap behavior that does not leave a live gateway or recovered turn holding dynamic imports to removed hashed bundle chunks.

## Best Possible Solution

Choose one durable update boundary and prove it live: stop old gateway work before package rotation, preserve old chunks until exit, or make all long-lived runtime imports stable/detected with a visible restart-required state across gateway, plugin, and Codex recovery paths.
