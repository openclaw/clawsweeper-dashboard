## Summary

Keep open: current main and v2026.6.8 have managed update handoff mitigations, but the latest release retest and current source still leave old-process/new-files stale import windows during shutdown or interrupted Codex recovery, and no merged canonical fix owns the durable policy yet.

## What This Changes

The issue asks OpenClaw to prevent auto-update/package swaps from leaving a live gateway or recovered Codex turn with stale content-hashed import targets.

## Best Possible Solution

Choose and implement one durable update boundary: stop old gateway work before replacing dist, preserve old chunks until exit, or make recovery/shutdown imports stable and verified across package swaps.

## Reproduction Assessment

No live current-main reproduction was run in this read-only review. Source inspection plus the v2026.6.8 retest gives a credible reproduction shape: an old gateway or recovered Codex turn can still reach lazy imports after the installed package tree has rotated.

## Solution Assessment

Unclear until maintainers choose the update policy. A one-off stale chunk alias or warning-only mitigation is not the best durable fix because the failure spans package swap, gateway shutdown/restart, and Codex recovery boundaries.
