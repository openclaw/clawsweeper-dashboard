## Summary

Keep open: current main has managed-service handoff, staged package swaps, restart verification, stale-PID cleanup, and one eager-load mitigation, but the latest 2026.6.10-beta.1 retest plus current lazy Codex/runtime boundaries still show no merged durable fix for stale hashed imports across update, shutdown, and recovered turns.

## What This Changes

Prevent auto-update or package/plugin swaps from leaving a running gateway or recovered Codex turn with cached imports to removed hashed dist chunks.

## Best Possible Solution

Choose and implement a durable update boundary: stop or reexec the gateway before package/plugin dist rotation, or retain old chunks until exit, with restart-required detection for raw swaps and live cross-platform upgrade proof.

## Reproduction Assessment

No live current-main reproduction was run here, but recent Linux and Windows upgrade retests plus current lazy import/package-rotation source give a credible source-level reproduction path.

## Solution Assessment

Current mitigations are not yet the best complete fix; the durable solution needs a lifecycle/update boundary rather than a narrow warning-only loader guard.
