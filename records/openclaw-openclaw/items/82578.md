## Summary

Keep open: the reported corruption is source-reproducible on current main, but this PR needs a small correctness fix before merge and still lacks after-fix real behavior proof from a patched run.

## What This Changes

This PR changes `persistSessionUsageUpdate` so stale usage updates mark `totalTokensFresh=false` without overwriting an existing `totalTokens` value.

## Best Possible Solution

Preserve `totalTokens` only for stale no-snapshot updates, retain the numeric fresh guard for snapshot paths, add focused regression coverage, and verify the patched behavior in the reported small-context messaging run.

## Reproduction Assessment

Yes at source level: current main explicitly patches `totalTokens: undefined` when usage exists without a fresh context snapshot, and the session merge overwrites existing fields. I did not run a live Feishu/Webchat reproduction in this read-only review.

## Solution Assessment

No, not yet: preserving the stale compaction token value is the right direction, but the PR should keep the existing numeric guard before marking a snapshot fresh. It also needs focused regression coverage and after-fix real behavior proof before merge.
