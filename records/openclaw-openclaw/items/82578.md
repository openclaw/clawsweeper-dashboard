## Summary

Keep open: the PR targets a real source-backed session accounting bug, but the patch is not mergeable as written because it drops the explicit unknown marker for no-snapshot sessions and currently fails the relevant session test. It also lacks required real behavior proof from an after-fix OpenClaw run.

## What This Changes

The PR changes `persistSessionUsageUpdate` to update `totalTokens` and `totalTokensFresh` only when a fresh context snapshot is available, preserving existing token fields for stale usage updates.

## Best Possible Solution

Revise this PR to preserve an existing valid token snapshot when stale usage has nothing better, explicitly mark unknown when there is no prior snapshot, cover both cases with regression tests, and add redacted real-session proof that repeated compaction stops.

## Reproduction Assessment

Yes for the PR defect: PR CI reproduces a failing `persistSessionUsageUpdate` test where `totalTokensFresh` becomes `undefined` instead of `false`. The original repeated-compaction report is source-reproducible from current `main`, but it has not been live-reproduced in this review.

## Solution Assessment

No, not as written. The narrow direction is right, but the implementation should preserve existing compaction totals only when there is a valid prior snapshot and still write the explicit unknown marker when nothing can be preserved.

## Review Findings

Overall correctness: patch is incorrect
