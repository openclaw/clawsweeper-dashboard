## Summary

Keep open. Current main still exposes `node.invoke` and plugin node-host commands as request/final-result primitives, and the related timeout or inter-session streaming work does not provide the requested request-correlated progress stream.

## What This Changes

Add an opt-in long-running or streaming node-host command path that can deliver progress/intermediate updates before the final result.

## Best Possible Solution

Design an additive, version-aware node invoke progress contract and update the protocol schemas, Gateway handler, node-host dispatcher, plugin runtime/SDK typing, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is static inspection of current `main` protocol schemas, method discovery, Gateway invoke handling, node-host dispatch, plugin runtime typing, docs, and related items.

## Solution Assessment

Unclear pending maintainer design. A shared protocol/SDK primitive is the maintainable direction; timeout configuration, polling, or generic `node.event` alone does not give callers request-correlated progress with clear lifecycle semantics.
