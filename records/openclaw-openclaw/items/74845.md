## Summary

Keep open. Current main still exposes node-host invocation as one request plus one final result; the requested request-correlated progress or streaming primitive remains additive protocol and SDK design work, and the related timeout PR is narrower.

## What This Changes

Add an opt-in long-running or streaming node-host command path that can report intermediate progress before returning a final result.

## Best Possible Solution

Design an additive, version-aware node invoke progress contract that preserves one-shot behavior while updating protocol schemas, Gateway handling, node-host dispatch, plugin SDK typing, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request. Static inspection of current main gives a high-confidence verification path: node invoke, plugin node-host commands, plugin runtime typing, method discovery, node-host dispatch, and docs remain final-result oriented.

## Solution Assessment

Unclear pending maintainer design. A shared additive protocol/SDK primitive is the maintainable direction; increasing timeouts or relying on generic `node.event` alone would not provide request-correlated invoke progress semantics.
