## Summary

Keep open. Current main and v2026.6.8 still treat `node.invoke` as one request followed by one final result or timeout, and plugin node-host commands still return a single string; the related timeout-config work does not provide caller-correlated progress updates.

## What This Changes

Requested an additive node-host command path that can report progress or intermediate updates before returning a final result.

## Best Possible Solution

Design an additive, permission-checked node-host progress contract that preserves one-shot `node.invoke` while updating protocol schemas, gateway handling, node-host dispatch, plugin SDK/runtime typing, docs, and focused tests together.

## Reproduction Assessment

Yes, as a source-level feature gap rather than a failing bug repro. Current main has no request-correlated `node.invoke` progress event, streaming invoke API, or plugin node-host command result stream.

## Solution Assessment

Unclear final API shape. A shared additive protocol/SDK primitive is the right layer, while a timeout config or ad hoc generic `node.event` workaround would not provide reusable caller-correlated progress semantics.
