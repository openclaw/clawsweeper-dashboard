## Summary

Keep open. Current main and v2026.6.8 still expose `node.invoke` as a one-shot request with one final result or timeout, and plugin node-host commands still return a single final string; the timeout-config PRs overlap but do not provide caller-correlated progress updates.

## What This Changes

The issue requests an opt-in long-running or streaming node-host command path that can send intermediate progress updates before the final result.

## Best Possible Solution

Design an additive, permission-checked node-host progress contract that preserves one-shot `node.invoke` while updating protocol schemas, gateway handling, node-host dispatch, plugin SDK/runtime typing, docs, and focused tests together.

## Reproduction Assessment

Yes, at source level. Current main has no request-correlated `node.invoke` progress event, streaming invoke API, or plugin node-host command result stream.

## Solution Assessment

Unclear final API shape. A shared additive protocol and SDK primitive is the right layer, while a timeout config, polling, or ad hoc generic `node.event` workaround would not provide reusable caller-correlated progress semantics.
