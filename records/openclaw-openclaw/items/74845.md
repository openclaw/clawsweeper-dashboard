## Summary

Keep open. Current main still exposes node invoke and plugin node-host commands as a one-shot request/final-result flow, so the requested request-correlated progress or streaming primitive remains real platform design work.

## What This Changes

Add an opt-in long-running node-host command path that can send progress or intermediate updates before returning a final result.

## Best Possible Solution

Design an additive, version-aware node invoke progress contract that preserves existing one-shot behavior and updates the protocol schemas, Gateway handler, node-host dispatch, plugin SDK typing, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is static inspection of protocol schemas, method discovery, Gateway invoke handling, node-host dispatch, plugin command typing, docs, and related items on current main.

## Solution Assessment

Unclear pending maintainer design. A shared protocol/SDK primitive is the maintainable direction; only increasing timeouts, polling, or relying on generic `node.event` would not give the pending invoke caller request-correlated progress and lifecycle semantics.
