## Summary

Keep open. Current main still treats node-host invocation as a request with one final result; the existing node-originated exec lifecycle events and the related timeout PR do not provide a request-correlated progress/update primitive for callers.

## What This Changes

Add a first-class platform path for long-running node-host commands to emit intermediate progress before returning a final result.

## Best Possible Solution

Design an additive, version-aware node invoke progress contract that preserves one-shot behavior while updating protocol schemas, Gateway handling, node-host dispatch, plugin SDK typing, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request. Static inspection gives a high-confidence verification path: node invoke, protocol schema, plugin node-host commands, plugin runtime typing, current node events, and docs are still final-result oriented.

## Solution Assessment

Unclear pending maintainer design. A shared additive protocol/SDK primitive is the maintainable direction; a timeout config or generic best-effort node.event usage alone would not provide reusable request-correlated progress semantics.
