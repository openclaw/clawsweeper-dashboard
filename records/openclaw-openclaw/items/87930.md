## What This Changes

The issue asks for Mattermost `message(action="send")` attachment-shaped inputs to upload the requested file or fail explicitly instead of returning success without the file.

## Best Possible Solution

Wire Mattermost send attachments through the existing scoped media upload path and return explicit unsupported/upload errors for attachment-shaped inputs that cannot be delivered.

## Reproduction Assessment

Yes for a source-level repro: current main still routes send attachment hints into Mattermost without passing the media access context needed by the upload path, and upload failure can still produce a post without `file_ids`. I did not run a live Mattermost server repro in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing Mattermost file upload flow with the generic send media context, then fail loudly for unsupported or failed attachment shapes instead of posting success without the file.
