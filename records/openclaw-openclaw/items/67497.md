## Summary

Keep open. Current main and the latest release still send Feishu setup/status through the normal SDK client constructor without logger suppression, and https://github.com/openclaw/openclaw/pull/67734 is the open same-author implementation PR with closing syntax, so this issue should wait for that PR to land, be revised, or be closed.

## What This Changes

The issue asks OpenClaw to stop showing Feishu/Lark SDK internal `client ready` info logs during interactive setup/status probes while preserving useful Feishu connection status.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/67734, or an equivalent Feishu plugin fix, so setup/status probes use a quiet SDK logger path without changing shared runtime client behavior.

## Reproduction Assessment

Yes. Although I did not run a credentialed wizard session, the current source path is clear: setup status calls `probeFeishu`, the probe constructs the normal SDK client, and the pinned SDK defaults to info logging and emits `client ready`.

## Solution Assessment

Yes. A Feishu-owned probe-only silent/custom SDK logger is the narrow maintainable fix; suppressing general wizard output or changing core channel status rendering would be broader than necessary.
