## Summary

Keep open. Current main still routes Feishu setup/status through the default SDK client without logger suppression, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/67734 is still open with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to suppress Feishu SDK info logs during setup/status probes without hiding the useful connected status line.

## Best Possible Solution

Review, rebase, and land or replace the linked Feishu fix so setup/status probes use a quiet SDK logger while preserving the connected status line and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main calls `probeFeishu` during Feishu setup status, that probe constructs `createFeishuClient` without logger suppression, and SDK 1.64.0 defaults to info logging and emits `client ready`.

## Solution Assessment

Yes for the direction. A Feishu probe-only quiet/custom SDK logger is narrower than suppressing wizard output globally or changing normal runtime Feishu logging, and the linked PR is the implementation path to review first.
