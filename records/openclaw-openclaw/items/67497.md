## Summary

Keep open. Current main and the latest shipped tag still route Feishu setup/status probing through the default SDK client without logger suppression, and the linked same-author implementation PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to stop Feishu setup/status probes from printing the Lark SDK's raw `[info]: [ 'client ready' ]` log line in the interactive wizard.

## Best Possible Solution

Land or replace the linked Feishu-owned fix so setup/status probes use a quiet SDK logger, preserve the connected status line, and add focused regression coverage for the probe/client path.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main calls `probeFeishu` during Feishu setup status, `probeFeishu` constructs `createFeishuClient` without a logger option, and SDK 1.64.0 defaults to info logging and emits `client ready`.

## Solution Assessment

Yes for the direction. A Feishu probe-only quiet/custom SDK logger is narrower than suppressing wizard output globally or changing normal runtime Feishu logging, and the linked PR is the current implementation path to review first.
