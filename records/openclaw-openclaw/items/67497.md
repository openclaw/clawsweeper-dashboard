## Summary

Keep open. Current main and the latest release still route Feishu setup/status through the normal SDK client constructor without logger suppression, and the same-author implementation PR at https://github.com/openclaw/openclaw/pull/67734 remains open with closing syntax for this issue.

## What This Changes

The issue requests suppressing Feishu SDK info output during setup/status probes while preserving the normal Feishu connection status line.

## Best Possible Solution

Review, update, and land or close https://github.com/openclaw/openclaw/pull/67734 so Feishu setup/status probes use a quiet SDK logger path while normal runtime Feishu client behavior stays unchanged.

## Reproduction Assessment

Yes. Current main calls `probeFeishu` from Feishu setup status, the probe constructs the SDK client without logger suppression, and SDK 1.63.1 logs `client ready` at info level by default; I did not run a credentialed wizard session.

## Solution Assessment

Yes. A Feishu-owned probe-only silent/custom SDK logger path is the narrow maintainable fix; suppressing core wizard output or changing global runtime logging would be broader than necessary.
