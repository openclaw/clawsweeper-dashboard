## Summary

Keep open. Current main still routes Feishu setup/status resolution through the default SDK client without logger suppression, and the same-author implementation PR at https://github.com/openclaw/openclaw/pull/67734 is still open with closing syntax for this issue.

## What This Changes

The issue asks for Feishu setup/status probes to suppress `@larksuiteoapi/node-sdk` internal info logs while preserving useful wizard status output.

## Best Possible Solution

Land or replace the linked Feishu-owned fix so setup/status probes use a quiet SDK logger, preserve the connected status line, and cover the probe/client path with focused regression tests.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main calls `probeFeishu` during Feishu setup status, that probe constructs the SDK client without a logger option, and SDK 1.64.0 defaults to info logging and emits `client ready`; I did not run a credentialed wizard session.

## Solution Assessment

Yes for the direction. A Feishu probe-only quiet/custom SDK logger path is narrower than suppressing core wizard output or changing normal runtime Feishu logging, and the open linked PR is the implementation path to review first.
