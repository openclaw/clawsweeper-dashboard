## Summary

Keep this issue open. Current `main` still sends Feishu setup/status through the probe path that constructs the default SDK HTTP client without a quiet logger, and the same-author implementation PR remains open with closing syntax for this issue. https://github.com/openclaw/openclaw/issues/67497

## What This Changes

The issue requests that Feishu setup/status probing stop printing raw `@larksuiteoapi/node-sdk` info output in the interactive setup wizard while preserving useful Feishu connection status.

## Best Possible Solution

Land or revise the Feishu-owned fix in #67734, or an equivalent narrow change, so setup/status probes create a quiet/custom SDK logger path, avoid leaking SDK info output into the wizard, preserve `Feishu: connected as ...`, and add focused regression coverage in the Feishu client/probe/setup-surface tests.

## Reproduction Assessment

Yes. The high-confidence source path is `collectChannelStatus`/`Channel status` -> Feishu `resolveStatusLines` -> `probeFeishu` -> `createFeishuClient` with no logger override; the live wizard was not rerun in this read-only review.

## Solution Assessment

Yes, the currently linked approach is the best known direction: keep the fix inside the Feishu plugin, add a probe-only quiet/custom SDK logger path, and avoid mutating or reusing the shared runtime client when a custom probe logger is needed.
