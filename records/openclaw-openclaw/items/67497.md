## Summary

Keep open. Current main and the latest release still route Feishu setup/status probing through the default SDK client constructor without logger suppression, and the same-author implementation PR at https://github.com/openclaw/openclaw/pull/67734 remains open with closing syntax for this issue.

## What This Changes

The issue requests hiding raw `@larksuiteoapi/node-sdk` info logs from Feishu setup/status probes while preserving normal Feishu status output.

## Best Possible Solution

Feishu setup/status probes should use a quiet probe-specific SDK logger path while preserving normal runtime Feishu logging and the visible `Feishu: connected as ...` status.

## Reproduction Assessment

Yes. Current main calls `probeFeishu` from Feishu setup status, the probe constructs the default SDK client, and the pinned SDK defaults to info logging and emits `client ready`; I did not run a credentialed wizard session.

## Solution Assessment

Yes. A Feishu-owned probe-only silent/custom SDK logger path is the narrow maintainable fix; suppressing core wizard output or changing global runtime Feishu logging would be broader than necessary.
