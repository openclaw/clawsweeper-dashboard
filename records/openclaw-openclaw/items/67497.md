## Summary

Keep open. Current main and the latest release still route Feishu setup/status through the normal SDK client constructor without logger suppression, and the same-author closing PR at https://github.com/openclaw/openclaw/pull/67734 remains open, so this issue should wait for that PR or an equivalent Feishu plugin fix.

## What This Changes

Prevent Feishu setup/status probes from leaking the SDK's raw `[info]: [ 'client ready' ]` log line into the interactive setup wizard.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/67734 so setup/status probes use a quiet SDK logger path while preserving normal runtime Feishu client behavior and the useful `Feishu: connected as ...` status line.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main's setup status calls `probeFeishu`, the probe constructs the default SDK client, and the pinned SDK defaults to info logging and emits `client ready`; I did not run a credentialed wizard session.

## Solution Assessment

Yes. A Feishu-owned probe-only silent/custom SDK logger is the narrow maintainable fix; suppressing core wizard output or changing global runtime logging would be broader than necessary.
