## Summary

Keep open. Current main and the latest release still route Feishu setup/status probing through the default SDK client without logger suppression, and the linked same-author implementation PR at https://github.com/openclaw/openclaw/pull/67734 is still open with closing syntax for this issue.

## What This Changes

Suppress raw @larksuiteoapi SDK info logs from Feishu setup/status probes while preserving the useful Feishu connectivity status line.

## Best Possible Solution

Land or replace the linked Feishu-owned fix so setup/status probes use a quiet SDK logger path, preserve `Feishu: connected as ...`, and add focused regression coverage.

## Reproduction Assessment

Yes. Current main shows setup status calling probeFeishu, probeFeishu constructing the default SDK client, and the pinned SDK defaulting to info logging and emitting `client ready`; I did not run a credentialed wizard session.

## Solution Assessment

Yes. A Feishu probe-only quiet/custom SDK logger path is the narrow maintainable fix; suppressing core wizard output or changing normal runtime Feishu logging would be broader than necessary.
