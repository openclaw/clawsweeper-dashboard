## Summary

Keep open. Current main and the latest release still route Feishu setup/status through `probeFeishu` and `createFeishuClient` without SDK logger suppression, while the same-author implementation PR at https://github.com/openclaw/openclaw/pull/67734 is open and uses closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to hide the Feishu SDK `client ready` info log during setup/status probing while preserving the useful Feishu connection status line.

## Best Possible Solution

Land or replace the linked Feishu-owned fix so setup/status probes use a quiet SDK logger, preserve the connected status line, and keep focused client/probe regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main calls `probeFeishu` during Feishu setup status, `probeFeishu` constructs `createFeishuClient` without a logger option, and SDK 1.64.0 defaults to info logging and emits `client ready`; I did not run a credentialed wizard session.

## Solution Assessment

Yes for the fix direction. A Feishu probe-only quiet/custom SDK logger is narrower than suppressing wizard output globally or changing normal runtime Feishu logging, and the open linked PR is the implementation path to review first.
