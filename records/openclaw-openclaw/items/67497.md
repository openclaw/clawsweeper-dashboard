## Summary

Keep open. Current main still sends Feishu setup/status through the probe path that constructs the normal Lark SDK client without logger suppression, and the same-author PR #67734 remains the open implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks for Feishu setup/status probes to stop printing the Lark SDK `[info]: [ 'client ready' ]` line while preserving useful Feishu connection status output.

## Best Possible Solution

Land or revise #67734, or an equivalent Feishu-owned fix, so setup/status probes use a quiet/custom SDK logger path without changing core channel status rendering or shared runtime client behavior.

## Reproduction Assessment

Yes. The reproduction is source- and dependency-backed: setup status calls `resolveStatusLines`, which calls `probeFeishu`, which constructs the SDK `Client` whose pinned constructor defaults to info logging and emits `client ready`.

## Solution Assessment

Yes. A narrow Feishu plugin fix that passes a quiet/custom logger or logger level for setup/status probing is the maintainable solution; suppressing general wizard output or changing core status rendering would be broader than necessary.
