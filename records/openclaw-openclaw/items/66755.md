## What This Changes

The PR adds reasoning-wrapper handling to `isSilentReplyText` for `think` and `<think>` payloads ending in the silent token, plus token tests for positives and false-positive guards.

## Best Possible Solution

Land one canonical shared silent-reply classifier that suppresses reasoning-prefixed silent final replies, preserves substantive-reply guards, includes focused regression coverage, and has redacted dispatcher or channel proof.

## Solution Assessment

No, not as written. The PR is a focused direction, but it misses `think\nNO_REPLY`, lacks external after-fix proof, and should be selected or superseded alongside the related open silent-reply PRs.
