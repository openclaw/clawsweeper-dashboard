## Summary

Keep open because the follow-up comments make this a plausible current Web UI regression: two users report that local web chat history at `127.0.0.1:18789` collapses to only the latest model reply, and current history code has several source-level paths that can hide or bound transcript messages.

## What This Changes

The issue reports that Web UI AI-agent messages disappear from chat history, leaving only the latest reply visible after short conversations or new sessions.

## Best Possible Solution

Preserve normal short-session Web UI history across sends, reloads, and new-session navigation while keeping documented caps, silent-message filtering, and active-branch behavior intact.

## Reproduction Assessment

Yes, at source level with medium confidence: the Web UI and Gateway history paths can be traced, and the reporter plus another user describe the same last-reply-only behavior. I did not establish a live failing run or inspect an affected transcript file, so the exact current-main reproduction is still incomplete.

## Solution Assessment

Unclear until the transcript shape is known. The narrowest likely fix is in `chat.history` transcript reading or UI history reload preservation, not a broad cap increase or new history feature.
