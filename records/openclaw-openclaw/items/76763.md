## Summary

Keep open: the added comments make this a plausible Web UI chat-history regression reported by two users, and current main still has several intentional history caps and filters that could mask messages, but there is not yet a high-confidence reproduction for actual transcript loss or the “only latest reply remains” symptom.

## What This Changes

The issue reports that Web UI conversations on `http://127.0.0.1:18789` often hide or clear all but the latest AI-generated message during short agent chats.

## Best Possible Solution

Preserve all recent visible user and assistant turns within the existing 200-message and response-size budgets, and make any intentional omissions explicit instead of letting the Web UI appear to clear prior replies.

## Reproduction Assessment

No. The comments identify the Web UI surface and current source shows several plausible hiding paths, but there is still no concrete current-main sequence, transcript, or log proving why only the latest model reply remains visible.

## Solution Assessment

Unclear. A cap increase alone is not a defensible fix; the narrow maintainable path is to reproduce the Web UI collapse with a transcript or focused test, then patch the specific gateway projection, renderer filtering, or history-refresh race that causes it.
