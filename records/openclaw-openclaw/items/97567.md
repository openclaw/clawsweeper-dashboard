## Summary

Keep open. Current main and v2026.6.10 still implement agent and channel/topic skill filters as final allowlists, while this issue asks for a distinct opt-in additive composition contract that needs maintainer product/API direction.

## What This Changes

The issue asks for opt-in add/remove or merge-style composition for agent and topic/channel skill allowlists instead of requiring each scoped filter to restate the full final set.

## Best Possible Solution

Define one maintainer-approved skill-filter composition contract for agents and channel/topic scopes, keep replacement as the default, and update schema, docs, diagnostics, and regression tests together.

## Reproduction Assessment

Yes, as a source-level feature gap: current main and v2026.6.10 still resolve and document explicit skill allowlists as replacements, with tests covering that behavior.

## Solution Assessment

Unclear. Add/remove composition is a plausible solution, but maintainers need to pick the canonical config shape and scope interaction before a safe implementation.
