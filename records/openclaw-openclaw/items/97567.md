## Summary

Keep open. Current main deliberately implements skill allowlists as replacement/intersection filters and does not provide an opt-in add/remove or inherited-merge composition contract, so this is a real feature request that needs maintainer product/API direction before implementation.

## What This Changes

Request additive/merge composition for agent and channel/topic skill allowlists so scoped configurations can add to or subtract from inherited skill sets without restating the full list.

## Best Possible Solution

Define one opt-in skill composition contract that preserves current replace behavior by default and applies consistently across agents and channel/topic filters with docs and regression tests.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main has replacement-style agent skills, first-defined channel/topic skills, and channel-agent intersection, with no add/remove or merge-mode skill composition surface.

## Solution Assessment

Unclear. The need is well supported, but maintainers still need to choose the canonical config shape and whether channel/topic deltas compose before or after the current agent-channel intersection.
