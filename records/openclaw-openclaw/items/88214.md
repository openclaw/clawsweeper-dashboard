## Summary

Keep open: current main and the latest release still render the sidebar Recent list from the loaded session rows without a current-agent filter, while the chat session picker already scopes its `sessions.list` request by active agent.

## What This Changes

The issue requests filtering the Control UI dashboard sidebar Recent sessions to the currently selected agent.

## Best Possible Solution

Reuse the existing active-agent session filtering semantics for the sidebar Recent list, add a mixed-agent sidebar regression test, and defer the optional All agents toggle unless maintainers explicitly want that extra UI.

## Solution Assessment
