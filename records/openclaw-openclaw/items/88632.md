## Summary

Keep open: current main and v2026.6.6 still retain gateway `model-run-*` sessions under the generic 30-day/500-entry session maintenance policy, and https://github.com/openclaw/openclaw/pull/91057 is the open closing implementation path.

## What This Changes

The issue requests model-run-specific session lifecycle or cleanup so one-shot gateway model probes do not saturate the default session store cap.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/91057 with a maintainer-approved lifecycle cleanup that protects durable direct/group/cron sessions and safely clears accumulated gateway probe rows.

## Reproduction Assessment

Yes. Source inspection shows each gateway model probe persists a fresh explicit `model-run-*` session, while cleanup still applies only the global 30-day/500-entry policy; the issue also includes production counts showing probe rows dominating the store.

## Solution Assessment

Unclear as a final implementation choice. The linked PR is a plausible fix path, but maintainers still need to choose the durable-versus-ephemeral contract for gateway model-run history before closing this issue.
