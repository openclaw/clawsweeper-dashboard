## Summary

Keep open: current main and the latest release still create explicit gateway model-run session rows, while session cleanup only applies the normal global age/count policy with no model-run/probe lifecycle. The code supports the reported session-state maintenance gap, but the exact retention shape needs maintainer direction.

## What This Changes

The issue requests a short or non-durable lifecycle for one-shot gateway model-run/probe sessions so repeated `openclaw infer model run --gateway` calls do not consume normal conversation-session retention.

## Best Possible Solution

Add core-owned model-run/probe lifecycle handling that prevents `agent:*:explicit:model-run-*` rows from consuming normal conversation retention, with audited cleanup output and tests that preserve durable direct/group/cron sessions.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: current main creates `agent:<agent>:explicit:model-run-<uuid>` rows for gateway probes and cleanup only applies the global 30-day/500-entry policy. I did not run a live Gateway/store reproduction because this review was read-only.

## Solution Assessment

Unclear as an implementation choice: the report identifies a real lifecycle gap, but maintainers should choose between non-persistence, a fixed short TTL, or an explicit cleanup policy before a PR changes session retention semantics.
