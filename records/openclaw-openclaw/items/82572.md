  - body: `originatingReplyToMode` is part of the queue routing contract: `resolveFollowupReplyAnchor` treats `off` specially, and the delivery-context key includes the mode. This serializer persists the reply target and chat type but drops the mode, so after restart a queued message that intentionally disabled reply anchoring can restore with the default behavior and reply/thread differently. Persist the mode and add a restart round-trip test for `off`.
  - confidence: 0.94
- **[P2] Persist execution context fields used by restored runs:** `src/auto-reply/reply/queue/persist.ts:129-130`

Because `PersistedRunFields` omits them, a queued group/channel turn or explicit timeout override can be lost across restart. Add these fields to the allowlist and cover them in the SQLite round-trip tests.
  - confidence: 0.9
