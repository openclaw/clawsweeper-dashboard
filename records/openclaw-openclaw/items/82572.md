## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate queue restore process-wide:** `src/auto-reply/reply/queue/state.ts:171`
  - body: `state.ts` can be bundled/evaluated in more than one runtime chunk, which is why the queue itself uses a `Symbol.for` global map. Calling `restoreFollowupQueues()` from each module evaluation lets a later chunk reread the still-present state file and `FOLLOWUP_QUEUES.set()` over a live restored queue or in-flight drain, risking duplicate redelivery or lost queued items. Please make restore idempotent with a process-global once flag before running it from module init.
  - confidence: 0.86
