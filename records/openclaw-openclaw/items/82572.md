## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Persist the reply-mode selector:** `src/auto-reply/reply/queue/persist.ts:150-168`
  - body: `FollowupRun` carries `originatingReplyToMode`, and drain/delivery code uses it to decide whether a reply target participates in threading. This allowlist persists the reply id and chat type but drops the mode, so after restart a queued reply whose original mode was `off`, `first`, or `all` can be reinterpreted from current config and thread or reply differently. Add `originatingReplyToMode` to the persisted descriptor and cover the restore path.
  - confidence: 0.92
- **[P2] Persist the run chat type:** `src/auto-reply/reply/queue/persist.ts:94-141`
  - body: `FollowupRun.run.chatType` is part of the execution and delivery context key, and embedded context reconstruction falls back to it when route/session chat type is absent. `PersistedRunFields` omits it, so restored group/channel followups can resume without the chat type that current-main execution had before restart. Include `chatType` in the run allowlist and add a round-trip test.
  - confidence: 0.86
