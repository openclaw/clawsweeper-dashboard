## Summary

Keep open. Current main still resolves Docker sandbox workspace paths inside the Gateway namespace and formats them directly as Docker bind sources; this remains the canonical Docker gateway/DooD workspace-bind failure, and the direct remap PRs were closed unmerged.

## What This Changes

The issue requests Docker gateway sandbox `workspaceAccess` to mount host-resolvable workspace paths instead of container-internal `/home/node` paths when using Docker-outside-of-Docker.

## Best Possible Solution

Adopt one Docker-out-of-Docker sandbox contract end to end: safely remap managed workspace/browser bind sources to host paths, or change Docker setup/docs/diagnostics to require and enforce host-path parity.

## Reproduction Assessment

Yes, from source inspection: current main resolves sandbox workspace paths in the Gateway namespace and formats them directly into Docker `-v` bind sources. I did not run a live Docker daemon scenario in this read-only review.

## Solution Assessment

Unclear. Runtime remapping matches the reported failure and the closed fix PRs, but current docs codify host-path parity, so the maintainable fix should follow a maintainer decision on that contract.
