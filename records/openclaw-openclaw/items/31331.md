## Summary

Keep open. Current main still routes gateway-visible workspace paths directly into Docker sandbox bind creation, and the previously linked fix PRs for host-source remapping were closed unmerged; this remains the canonical Docker gateway/DooD sandbox workspaceAccess issue.

The issue asks OpenClaw to make Docker gateway sandbox workspaceAccess mount host-resolvable workspace paths instead of gateway-container-internal paths.

## Best Possible Solution

Keep this canonical issue open until maintainers choose and implement one durable contract: secure automatic host-source remapping, or a Docker setup/doctor path that makes host-path parity work without dangerous custom binds.
