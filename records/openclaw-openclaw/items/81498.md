## What This Changes

The PR adds a session-maintenance preserve-key provider for active and pending subagent child sessions, wires it into save/load/disk-budget maintenance, and adds regression coverage.

## Best Possible Solution

Land the lifecycle-aware preserve-key approach after aligning cleanup dry-run preview behavior and getting green relevant checks; leave archive fallback and extra instrumentation as separate follow-ups if maintainers still want them.

Yes. Source inspection gives a high-confidence current-main path: enforce `maxEntries`, leave a completed or pending-delivery subagent child row outside `activeSessionKey`, and `capEntryCount` can remove it before announce/result delivery reads it.
