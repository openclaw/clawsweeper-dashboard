## Summary

Keep open. Current main still exposes only the allowlisted `agents.files.*` bootstrap-file surface, and open PR #62417 is already paired with this issue using closing syntax, so cleanup should wait for that implementation to be reviewed, merged, replaced, or rejected.

## What This Changes

Request a full path-based `agents.workspace.*` Gateway API plus a Control UI workspace file manager for remote and container deployments.

## Best Possible Solution

Review and land #62417 or a narrower replacement that adds the typed Gateway protocol, handlers, scopes, UI, docs, tests, changelog, and security limits before closing this issue.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection is high-confidence: `agents.workspace.*` is absent, while `agents.files.*` remains limited to allowlisted bootstrap filenames.

## Solution Assessment

Unclear until the linked implementation is reviewed. The maintainable path should be additive, typed, permission-scoped, documented, resource-limited, and covered by focused path-boundary and destructive-operation tests.
