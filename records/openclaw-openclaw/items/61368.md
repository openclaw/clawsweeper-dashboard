## Summary

Keep open. Current main still lacks the requested full path-based workspace API and Workspace file manager, and open PR #62417 is already linked as the implementation candidate, so this issue should remain open until that PR is reviewed, merged, replaced, or closed.

## What This Changes

The issue requests an additive `agents.workspace.*` Gateway API for list/read/write/delete/mkdir/move/stat plus a Control UI Workspace Files manager for remote/container deployments.

## Best Possible Solution

Review PR #62417 as the canonical implementation candidate, or replace it with a narrower additive protocol, server, UI, docs, tests, changelog, and security-reviewed file-boundary design.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection is high-confidence: `agents.workspace.*` is absent, while `agents.files.*` remains limited to allowlisted bootstrap file names.

## Solution Assessment

Unclear until the linked implementation is reviewed. The maintainable solution should be additive, typed, permission-scoped, resource-limited, documented, and covered by focused path-boundary and destructive-operation tests.
