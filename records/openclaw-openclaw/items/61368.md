## Summary

Keep open. Current main still only exposes the bootstrap-file `agents.files.*` surface and the existing Control UI files panel, not the requested full `agents.workspace.*` API or workspace file manager. The issue is also paired with open PR #62417 using closing syntax, so the right cleanup outcome is to keep this issue open while that implementation candidate is reviewed, landed, replaced, or closed.

## What This Changes

Add an `agents.workspace.*` gateway API and Control UI workspace file manager for full list/read/write/delete/mkdir/move/stat operations inside agent workspaces for remote/container deployments.

## Best Possible Solution

Review PR #62417 as the canonical implementation candidate and either land a security-reviewed additive gateway protocol/UI/docs/tests change or replace it with a narrower maintained PR; close this issue only after that work is merged or intentionally rejected.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source inspection: `agents.workspace.*` is absent, while `agents.files.*` is limited to a fixed bootstrap-file allowlist.

## Solution Assessment

Unclear until maintainer review of the linked implementation. The requested direction is plausible because `fs-safe` has relevant primitives, but a broad workspace API should land only with additive protocol schemas, scoped permissions, explicit resource limits, docs, and focused security tests.
