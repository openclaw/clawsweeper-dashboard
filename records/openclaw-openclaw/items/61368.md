## Summary

Keep open. Current main still exposes only the allowlisted `agents.files.*` bootstrap-file surface, the requested path-based `agents.workspace.*` API and Workspace file manager are absent, and open PR #62417 already uses closing syntax for this issue.

## What This Changes

Request a full path-based agent workspace file-management API and Control UI file manager for remote or container deployments.

## Best Possible Solution

Review, land, or replace #62417 with an additive typed Gateway protocol, safe server handlers, Control UI, docs/changelog, and focused path-boundary tests, then let GitHub close this issue after merge.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks `agents.workspace.*` while `agents.files.*` remains limited to allowlisted bootstrap filenames.

## Solution Assessment

Unclear until the linked implementation is reviewed. The maintainable solution should be additive, permission-scoped, resource-limited, documented, and covered by focused tests for traversal, link handling, binary limits, and destructive operations.
