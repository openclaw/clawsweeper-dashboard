## Summary

Current main has the underlying agent identity, secure avatar resolution, avatar serving, and Control UI rendering paths, but it still lacks the requested Agents-tab image upload workflow. The request remains useful and focused; the broader related issue also asks for favicon and display-name customization, while the prior upload PR was closed unmerged due to inactivity, so neither safely supersedes this issue.

## What This Changes

Add an Agents-tab image picker that saves an avatar in the agent workspace, updates persistent agent identity metadata, and displays it in Control UI.

## Best Possible Solution

Add a first-class picker to the existing Agents identity editor, backed by one authenticated and size-limited binary-upload contract that writes only a canonical workspace-relative avatar file and updates identity through the existing agent mutation pipeline.

## Reproduction Assessment

Not applicable as a bug reproduction. Current source confirms that avatars can already be configured and rendered, but no documented or implemented contract promises an Agents-tab image upload control.

## Solution Assessment

No, not exactly as proposed. The user-facing picker is appropriate, but a new parallel RPC and HTTP route should not be adopted until it is reconciled with the existing identity mutation, workspace-safe file access, avatar policy, and serving contracts.
