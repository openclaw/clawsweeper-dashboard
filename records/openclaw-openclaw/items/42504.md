## What This Changes

The issue requests a Control UI Agents-tab workflow to upload a per-agent avatar image, save it under the agent workspace, update `IDENTITY.md`, and render it through the avatar route.

## Best Possible Solution

Design a narrow authenticated upload flow that stores a bounded image artifact under the selected agent workspace, writes a workspace-relative avatar value through the existing identity path, and reuses the current avatar policy, auth, and path-safety checks.
