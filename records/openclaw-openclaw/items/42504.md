## Summary

Keep open: current main and the latest release already support configured avatar resolution and read-only Control UI avatar serving, but they still do not implement the requested Agents-tab file picker or `agents.avatar.upload`/binary upload path. ## What This Changes

The issue requests an Agents-tab UI flow and admin backend path to upload per-agent avatar images, store them in the workspace, update identity metadata, and render the result in Control UI.

## Best Possible Solution

Approve a narrow, authenticated workspace-persistent avatar upload design that reuses the existing avatar resolver/serving policy, writes bounded images inside the selected agent workspace, and updates identity through the canonical agent identity path.

Yes for the missing capability from source inspection: current main has read-only avatar serving and identity string mutation paths, but no Agents-tab upload picker or `agents.avatar.upload` RPC. This is not a bug reproduction because no documented upload behavior exists today.

Unclear. The proposed picker plus admin upload route is plausible, but the best solution needs maintainer-approved storage, validation, auth, and identity precedence before implementation.
