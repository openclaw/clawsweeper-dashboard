## Summary

Keep open: current main and the latest release still lack the requested Agents-tab, server-side avatar upload flow. Request a Control UI Agents-tab picker and admin upload path that stores an image in the selected agent workspace and updates agent identity metadata.

## Best Possible Solution

Settle a narrow core Control UI/gateway avatar upload design, then implement authenticated bounded image upload inside the agent workspace, update canonical identity metadata, and document source precedence with focused tests.

Unclear. A picker plus upload path is plausible, but the best fix needs maintainer-approved upload API, storage, auth, validation, and identity-write semantics before implementation.
