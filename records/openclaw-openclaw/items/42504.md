## Summary

Keep this open. Current main still lacks the requested Agents-tab workspace avatar upload flow; it only has read-only avatar serving plus browser-local assistant avatar overrides, and the linked closed PR/gist remain design context rather than shipped behavior.

## What This Changes

Add an Agents-tab avatar picker/upload flow that stores selected images in each agent workspace and updates identity metadata for `/avatar/<agentId>` rendering.

## Best Possible Solution

Add a maintainer-approved admin-scoped avatar upload path with typed protocol/schema coverage, strict image validation, bounded symlink-safe workspace writes, identity updates through existing helpers, Agents-tab picker/refresh behavior, docs, and focused gateway/UI regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The negative source check is high-confidence: current main has no avatar upload RPC, no POST avatar route, and no Agents-tab workspace avatar picker.

## Solution Assessment

Unclear whether the exact `agents.avatar.upload` RPC or a `POST /avatar/<agentId>` route is the best final API. The user-facing direction is valid, but maintainers should choose the contract, auth scope, validation rules, and persistence model first.
