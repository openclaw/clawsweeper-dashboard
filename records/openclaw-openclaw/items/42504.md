## Summary

Keep open: current main and v2026.6.6 still lack the requested persistent Agents-tab avatar upload flow. The code can resolve and display configured avatar sources, and Quick Settings has a browser-local assistant override, but the requested server-side upload/storage path remains a product and security decision.

Request an Agents-tab file picker and admin upload flow that stores a selected avatar image in the agent workspace and updates the agent identity.

## Best Possible Solution

Design a narrow authenticated upload flow that writes a bounded image artifact inside the selected agent workspace, updates the canonical agent identity path, reuses existing avatar auth/path policy, and includes focused UI, gateway, docs, and upgrade tests.
