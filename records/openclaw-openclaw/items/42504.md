## Summary

Keep open. Current main has avatar identity parsing, validation, read-only authenticated `/avatar/<agentId>` serving, and browser-local assistant avatar overrides, but it still lacks the requested Agents-tab picker plus server-side image upload/write contract; that is a new privileged feature needing maintainer product and security design.

## What This Changes

Request an Agents-tab avatar picker and admin upload path that saves selected agent avatar images in the workspace and updates `IDENTITY.md`.

## Best Possible Solution

Add a maintainer-approved admin-scoped Agents-tab avatar upload flow that reuses existing avatar policy and identity helpers, validates and stores bounded images under the agent workspace, updates identity consistently, and refreshes authenticated `/avatar/<agentId>` rendering.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks `agents.avatar.upload`, rejects non-read `/avatar` methods, and has no Agents-tab upload callback.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` RPC is the best final contract. The user-facing direction is valid, but maintainers should choose RPC versus HTTP shape, auth scope, content policy, persistence model, and identity update path before implementation.
