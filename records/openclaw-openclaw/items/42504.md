## Summary

Keep open. Current main can read and render configured agent avatars and has browser-local assistant avatar overrides, but it still has no Agents-tab image picker or server-side upload/write contract; the remaining work is a product and security-scoped feature rather than cleanup to close.

## What This Changes

Request adding an Agents-tab avatar picker plus an `agents.avatar.upload` path that stores an image under the agent workspace and updates `IDENTITY.md`.

## Best Possible Solution

Add a maintainer-approved admin-scoped Agents-tab upload flow that reuses existing avatar policy and identity helpers, stores bounded images safely under the agent workspace, updates identity consistently, and refreshes authenticated avatar rendering.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks `agents.avatar.upload`, rejects non-read `/avatar` methods, and has no Agents-tab upload callback.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` RPC is the best final contract. The user-facing direction is valid, but maintainers should choose the protocol shape, auth scope, content policy, persistence model, and identity update path before implementation.
