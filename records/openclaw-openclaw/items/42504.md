## Summary

Keep open. Current main supports avatar identity values, read-only authenticated avatar serving, and browser-local assistant avatar overrides, but it still lacks the requested Agents-tab image picker plus server-side avatar upload/write contract; that privileged upload surface needs maintainer product and security design before implementation.

## What This Changes

Request an Agents-tab avatar picker that uploads a local image, stores it under the agent workspace, updates identity metadata, and renders it through the Control UI avatar route.

## Best Possible Solution

Add a maintainer-approved admin-scoped workspace avatar upload flow that reuses existing avatar policy and identity helpers, with typed protocol coverage, strict validation, safe writes, UI refresh behavior, docs, and focused gateway/UI tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives a high-confidence baseline that current main lacks an Agents-tab workspace avatar upload RPC, writable avatar route, and picker callback.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` RPC is the best final contract. The user-facing direction is valid, but maintainers should choose the HTTP/RPC shape, auth scope, content policy, persistence model, and `IDENTITY.md` update path before implementation.
