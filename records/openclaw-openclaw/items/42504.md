## Summary

Keep open. Current main and the latest release still have manual/config avatar support, authenticated read-only avatar serving, and browser-local assistant avatar overrides, but they do not implement the requested Agents-tab image picker plus server-side workspace upload/write contract.

## What This Changes

Request to add an Agents-tab avatar picker that uploads a local image, stores it under the agent workspace, updates agent identity, and renders it through the Control UI avatar route.

## Best Possible Solution

Define a maintainer-approved authenticated upload path that stores bounded images safely under the agent workspace, updates canonical agent identity consistently, and exposes a clear Agents-tab picker without weakening the existing read-only avatar serving policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested upload method, upload endpoint, and Agents-tab file picker while manual avatar configuration and browser-local overrides already exist.

## Solution Assessment

Unclear whether the proposed agents.avatar.upload shape is the best final API. Maintainers should choose the protocol shape, authorization scope, image validation policy, storage model, and canonical identity update behavior before implementation.
