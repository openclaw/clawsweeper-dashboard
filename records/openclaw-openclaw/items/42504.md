## Summary

Keep open. Current main supports manual/configured avatar rendering and browser-local avatar overrides, but the requested Agents-tab picker plus admin-scoped persisted workspace upload/write contract is still missing.

## What This Changes

Request to add an Agents-tab avatar picker that stores a chosen image under the agent workspace, updates agent identity state, and renders it through Control UI.

## Best Possible Solution

Define a maintainer-approved admin avatar upload contract that safely stores bounded images under the agent workspace, updates canonical agent identity consistently, and reuses the authenticated avatar rendering policy from the Agents tab.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested upload RPC/endpoint and Agents-tab file picker while manual/configured avatar support and browser-local overrides already exist.

## Solution Assessment

Unclear whether the proposed agents.avatar.upload plus IDENTITY.md update is the best final contract. Maintainers should choose the protocol shape, authorization scope, image validation policy, storage model, and canonical identity update behavior before implementation.
