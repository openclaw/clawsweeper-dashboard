## Summary

Keep this open. Current main still supports configured/manual avatar references and authenticated read-only avatar serving, but it does not provide the requested Agents-tab image picker or workspace avatar upload RPC; the missing piece is a new upload/write surface that needs maintainer approval for API and security policy.

## What This Changes

Add a Control UI Agents-tab avatar picker that uploads a local image, stores it under the agent workspace, updates `IDENTITY.md`, and renders it via `/avatar/<agentId>`.

## Best Possible Solution

Add a maintainer-approved admin-scoped upload flow that reuses the existing avatar policy and identity helpers, with typed protocol coverage, strict image validation, bounded workspace writes, UI refresh behavior, docs, and focused gateway/UI tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection against current main gives a high-confidence check that there is no upload RPC, writable avatar route, or Agents-tab workspace avatar picker.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` RPC is the best final contract. The user-facing direction is valid, but maintainers should choose the auth scope, validation policy, SVG/content handling, persistence model, and identity update path before implementation.
