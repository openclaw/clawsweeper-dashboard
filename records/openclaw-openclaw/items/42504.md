## What This Changes

The issue requests an Agents-tab UI flow and backend contract for uploading per-agent avatar images into the agent workspace and updating identity metadata.

## Best Possible Solution

Define and implement a maintainer-approved admin-scoped upload flow that reuses avatar policy, stores bounded images safely under the agent workspace, updates IDENTITY.md consistently, and refreshes authenticated avatar rendering.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks agents.avatar.upload, rejects non-read /avatar methods, and has no Agents-tab upload callback.

## Solution Assessment

Unclear whether the proposed agents.avatar.upload RPC is the best final contract. The user-facing direction is valid, but maintainers should choose the protocol shape, authorization scope, image policy, persistence model, and identity update path before implementation.
