## Summary

Keep open. Current main supports configured/manual avatar rendering and browser-local assistant avatar overrides, but it still lacks the requested Agents-tab picker plus an authenticated workspace avatar upload/write contract; the broader related avatar/favicon issue points back here for this narrower upload track.

## What This Changes

Request an Agents-tab avatar picker and admin upload path that saves selected images under the agent workspace and updates agent identity for `/avatar/<agentId>` rendering.

## Best Possible Solution

Define a maintainer-approved authenticated upload contract that stores bounded images safely under the agent workspace, updates canonical identity consistently, and reuses the existing authenticated avatar rendering policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested upload method, upload endpoint, and Agents-tab file picker while manual avatar configuration and browser-local overrides already exist.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` plus `IDENTITY.md` update is the best final contract. Maintainers should choose the protocol shape, authorization scope, image validation policy, storage model, and canonical identity update behavior before implementation.
