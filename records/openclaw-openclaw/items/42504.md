## Summary

Keep open: current main and the latest release still lack the requested Agents-tab file picker plus server-side avatar upload/storage workflow. Existing code can read and display configured avatars, and related avatar issues cover narrower display or browser-local override problems, but no open or merged related item owns this exact persistent upload flow.

## What This Changes

The issue requests a Control UI Agents-tab avatar picker that saves an image under the selected agent workspace and updates the agent identity so `/avatar/<agentId>` can render it.

## Best Possible Solution

Design a narrow authenticated upload flow that writes a bounded image artifact inside the selected agent workspace, updates the canonical agent identity through the existing identity path, and reuses the current avatar auth and path-safety policy.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing UI capability rather than broken documented behavior. Source inspection gives high confidence that current main lacks both an Agents-tab upload control and a server upload RPC.
