## Summary

Keep open. Current main has manual/configured avatar rendering and a browser-local assistant avatar override, but it still lacks the requested Agents-tab file picker and an admin-scoped workspace avatar upload/write contract.

## What This Changes

The issue requests an Agents-tab avatar upload flow that stores a selected image under the agent workspace and updates agent identity metadata.

## Best Possible Solution

Define a maintainer-approved admin-scoped avatar upload contract that safely stores bounded images under the agent workspace, updates identity consistently, and reuses the authenticated `/avatar/<agentId>` rendering policy from the Agents tab.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested upload RPC/endpoint and Agents-tab file picker, while manual identity avatar configuration already exists.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` RPC is the best final contract. The user-facing direction is valid, but maintainers should choose the protocol shape, authorization scope, image validation policy, storage model, and identity update behavior first.
