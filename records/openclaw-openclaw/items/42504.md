## Summary

Keep this open. Current main still lacks the requested Agents-tab workspace avatar upload flow; the shipped behavior covers avatar strings, read-only avatar serving, manual identity edits, and browser-local Control UI overrides, but not storing a chosen image into the agent workspace from the Agents tab.

## What This Changes

Requests an Agents-tab avatar picker backed by an admin upload path that saves images under the agent workspace and updates `IDENTITY.md`.

## Best Possible Solution

Add a maintainer-approved admin-scoped avatar upload flow with typed protocol coverage, strict image validation, symlink-safe writes under `workspace/avatars/`, identity updates through existing helpers, Agents-tab UI refresh behavior, docs, and focused gateway/UI regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The negative source check is high-confidence because current main has no upload RPC, no POST avatar route, and no Agents-tab file picker for workspace avatars.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` RPC is the best final API. The user-facing direction is valid, but maintainers should choose the contract, auth scope, validation rules, and persistence model before implementation.
