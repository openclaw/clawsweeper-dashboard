## Summary

Keep open. Current main still lacks the requested Agents-tab avatar picker and server-side workspace upload/write contract; existing code supports configured/manual avatar identity, read-only avatar serving, and browser-local overrides only.

## What This Changes

Requested an Agents-tab avatar upload flow that saves selected images under `workspace/avatars/`, updates `IDENTITY.md`, and renders the result through `/avatar/<agentId>`.

## Best Possible Solution

Define a maintainer-approved authenticated upload contract that safely stores bounded images under the agent workspace, updates canonical identity consistently, and reuses the existing authenticated avatar rendering policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested upload method, upload endpoint, and Agents-tab file picker while manual avatar configuration and browser-local overrides already exist.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` shape is the best final API. Maintainers should choose the protocol shape, authorization scope, image validation policy, storage model, and canonical identity update behavior before implementation.
