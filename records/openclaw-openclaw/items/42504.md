## Summary

Keep this open. Current main still lacks the requested Agents-tab workspace avatar upload flow; existing behavior supports avatar strings, read-only avatar serving, manual identity edits, and browser-local Control UI overrides, but not uploading an image into the agent workspace from the Agents tab.

## What This Changes

Request an Agents-tab avatar picker and admin upload RPC that stores selected images in the agent workspace and updates IDENTITY.md.

## Best Possible Solution

Add a maintainer-approved admin-scoped upload flow with typed protocol coverage, strict image validation, symlink-safe bounded writes under workspace avatars, identity updates through existing helpers, UI refresh behavior, docs, and focused gateway/UI regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main has no upload RPC, writable avatar route, or Agents-tab workspace avatar picker.

## Solution Assessment

Unclear whether the proposed agents.avatar.upload RPC is the best final API. The user-facing direction is valid, but maintainers should choose the contract, auth scope, validation policy, and persistence model before implementation.
