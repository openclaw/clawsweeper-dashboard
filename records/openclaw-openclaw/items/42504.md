## Summary

Keep open. Current main supports configured/manual avatar rendering and browser-local assistant avatar overrides, but it still lacks the requested Agents-tab picker plus an admin-scoped workspace avatar upload/write contract.

## What This Changes

The request would add an Agents-tab avatar picker that stores a selected image under `workspace/avatars/`, updates `IDENTITY.md`, and renders the result through `/avatar/<agentId>`.

## Best Possible Solution

Define a maintainer-approved admin avatar upload contract that safely stores bounded images under the agent workspace, updates canonical identity state consistently, and reuses the authenticated avatar rendering policy from the Agents tab.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested upload RPC/endpoint and Agents-tab file picker while manual avatar configuration and browser-local overrides already exist.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` plus `IDENTITY.md` update is the best final contract. Maintainers should choose the protocol shape, authorization scope, image validation policy, storage model, and canonical identity update behavior before implementation.
