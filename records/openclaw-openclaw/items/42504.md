## Summary

Keep this open. Current main still supports avatar references, read-only avatar serving, manual `IDENTITY.md` edits, and a browser-local assistant override, but it does not provide the requested Agents-tab workspace avatar upload flow.

## What This Changes

The issue requests a Control UI picker on the Agents tab plus an admin upload API that saves an agent avatar image under the workspace, updates `IDENTITY.md`, and renders it via `/avatar/<agentId>`.

## Best Possible Solution

Add a maintainer-approved admin-scoped upload flow with typed protocol coverage, strict image validation, symlink-safe bounded writes under workspace avatars, identity updates through existing helpers, UI refresh behavior, docs, and focused gateway/UI tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection against current main gives high confidence that there is no upload RPC, writable avatar route, or Agents-tab workspace avatar picker.

## Solution Assessment

Unclear whether the proposed `agents.avatar.upload` RPC is the best final API. The user-facing direction is valid, but maintainers should choose the contract, auth scope, validation policy, and persistence model before implementation.
