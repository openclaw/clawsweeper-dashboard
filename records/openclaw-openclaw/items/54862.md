## Summary

Keep open. The feature is useful and current main does not already provide this OpenRouter Auto Router allowlist surface, but the PR head is not safe to merge because it injects on non-Auto OpenRouter requests, bypasses the plugin API seam from a core test, is stale against current OpenRouter stream wrappers, and misses required docs/changelog coverage.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` option that injects an OpenRouter `auto-router` plugin payload and extends OpenRouter/extra-param tests.

## Best Possible Solution

Land a revised OpenRouter plugin implementation that injects Auto Router allowlists only for canonical Auto requests, preserves current OpenRouter stream wrappers and reasoning guards, exposes any shared helper through `extensions/openrouter/api.ts`, and documents/changelogs the new config.

## Reproduction Assessment

Yes. The PR-head blockers are source-reproducible: shared default params flow into all model calls, and the new wrapper injects the Auto Router plugin whenever those params include `allowedModels` without checking the requested OpenRouter model.

## Solution Assessment

No. The feature belongs in the OpenRouter plugin stream layer, but the current patch needs Auto-only gating, current-main wrapper preservation, API-seam cleanup, wildcard reasoning handling, docs, and changelog before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:99-107`
  - body: `autoRouter.allowedModels` is documented for OpenRouter Auto, but this wrapper runs for every OpenRouter model when merged params contain a non-empty allowlist. Because shared default params are merged into fixed-model calls too, this can add the Auto Router plugin payload and alter reasoning on non-Auto requests; gate this injection to canonical Auto model ids.
  - confidence: 0.92
- **[P2] Layer the change onto the current wrapper chain:** `extensions/openrouter/stream.ts:120-130`
  - body: This branch's wrapper returns the shared thinking hook directly and does not include current main's Anthropic prefill stripping, DeepSeek V4 reasoning wrapper, or `isOpenRouterProxyReasoningUnsupportedModel` guard. Rebase the feature onto the current stream chain so Auto Router support does not drop those existing OpenRouter behaviors.
  - confidence: 0.88
- **[P2] Expose the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. Core tests that need bundled plugin helpers should consume the plugin `api.ts` surface or move the assertion into plugin-owned tests, otherwise entrypoint refactors can break unrelated core coverage.
  - confidence: 0.87
- **[P2] Treat provider-agnostic Grok wildcards as unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter's Auto Router pattern syntax supports provider-agnostic wildcards such as `*/claude-*`, so patterns like `*/grok-*` can still route to x-ai/Grok models. This check only recognizes explicit `x-ai/` patterns, leaving proxy reasoning enabled for wildcard pools that can select unsupported models.
  - confidence: 0.83
- **[P3] Document and changelog the new config:** `extensions/openrouter/stream.ts:7-16`
  - body: This adds a user-facing `params.autoRouter.allowedModels` setting, but the PR does not update `docs/providers/openrouter.md` or `CHANGELOG.md`. Add both so users can discover the config and release notes capture the feature.
  - confidence: 0.86
