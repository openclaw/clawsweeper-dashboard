## Summary

Keep open: the feature is useful and current main still lacks OpenRouter Auto Router allowlist support, but the PR is not merge-ready because the request wrapper is too broad and the public seam/docs work is incomplete.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` option that injects an `auto-router` plugin payload and adds OpenRouter extra-param tests.

## Best Possible Solution

Land a revised OpenRouter plugin implementation that gates Auto Router payload injection to canonical Auto model refs, exposes any shared helper through the OpenRouter API seam, handles wildcard reasoning conservatively, and documents/changelogs the new config.

## Reproduction Assessment

Yes. Under the PR head, configure `agents.defaults.params.autoRouter.allowedModels` and call a fixed OpenRouter model; current extra-param merging passes that global param into the OpenRouter wrapper, which injects the plugin without a model guard.

## Solution Assessment

No. The feature direction is maintainable, but the current diff is not the narrowest safe merge shape until the Auto-model guard, public seam, wildcard reasoning handling, docs, and changelog are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard auto-router injection to Auto model requests:** `extensions/openrouter/stream.ts:99-106`
  - body: `autoRouter.allowedModels` is documented for the OpenRouter Auto route, but this block runs for every OpenRouter model whenever params contain a non-empty allowlist. Since `agents.defaults.params` are merged into all model requests, a shared Auto Router config can add the plugin payload and alter reasoning on fixed-model calls. Gate this to canonical Auto model ids before wrapping.
  - confidence: 0.9
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` directly from the OpenRouter extension entrypoint. The extension boundary requires core tests that need bundled plugin helpers to use `api.ts` or an SDK facade, otherwise entrypoint refactors can break unrelated core tests.
  - confidence: 0.86
- **[P2] Treat Grok wildcard allowlists as reasoning unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter allowed-model patterns support provider-agnostic wildcards, so patterns such as `*/grok-*` can still route Auto Router to x-ai/Grok models. This helper only flags entries with an explicit `x-ai/` path segment, leaving proxy reasoning enabled for wildcard pools that can select unsupported models.
  - confidence: 0.82
- **[P3] Document the new OpenRouter config surface:** `extensions/openrouter/stream.ts:14-16`
  - body: This adds a user-facing `params.autoRouter.allowedModels` config, but the PR does not update the OpenRouter provider docs or `CHANGELOG.md`. Add both before merge so users can discover the setting and release notes capture the feature.
  - confidence: 0.88
