## Summary

Keep open: the feature is useful and not implemented on current main, but the PR branch still has source-backed correctness blockers and lacks inspectable after-fix real OpenRouter proof.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` config path that injects an Auto Router `allowed_models` plugin into request payloads, plus OpenRouter and agent extra-params tests.

## Best Possible Solution

Land provider-local support that injects `allowed_models` only for `openrouter/auto`, preserves current OpenRouter wrappers, exposes any shared helper through `api.ts` or plugin-owned tests, documents the config contract, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows unconditional `autoRouter.allowedModels` injection and a stale wrapper chain compared with current main. The actual OpenRouter after-fix behavior is not reproducible from attached artifacts because none are provided.

## Solution Assessment

No. The feature direction is reasonable, but this branch should be rebased and narrowed so Auto Router handling only applies to Auto requests, composes with current wrappers, respects the plugin API boundary, and documents the new config surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:98-106`
  - body: `autoRouter.allowedModels` is read from merged params and injected for every OpenRouter model. Shared defaults or reused params can make fixed OpenRouter models receive an Auto Router plugin payload and alter reasoning behavior, so normalize the model id and only inject for `openrouter/auto`.
  - confidence: 0.91
- **[P2] Preserve the current OpenRouter wrapper chain:** `extensions/openrouter/stream.ts:120-122`
  - body: This fallback now returns only the partially wrapped stream, and the main path returns only the shared thinking hook result. Current main also wraps OpenRouter requests with Anthropic prefill cleanup and DeepSeek V4 reasoning replay, so rebasing this as written would drop those protections.
  - confidence: 0.89
- **[P2] Expose the helper through a public plugin seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. Core tests should use a plugin `api.ts`/SDK seam or keep the assertion in plugin-owned tests, so routine entrypoint refactors do not break core coverage.
  - confidence: 0.88
- **[P3] Document the public OpenRouter config:** `extensions/openrouter/stream.ts:11-19`
  - body: The new user-facing `params.autoRouter.allowedModels` key is only described in source comments, while the OpenRouter provider docs still omit the config shape and wildcard limits. Add public docs so users do not have to read provider internals to use it safely.
  - confidence: 0.76
