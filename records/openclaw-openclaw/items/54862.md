## Summary

Keep open: current main does not implement OpenRouter Auto Router allowlists, but this external PR still has merge-blocking provider-wrapper, model-gating, plugin-boundary, wildcard-contract, documentation, and real-proof gaps.

## What This Changes

The PR adds OpenRouter `params.autoRouter.allowedModels` support by injecting an Auto Router plugin into OpenRouter request payloads, exporting the helper, and adding OpenRouter plus agent extra-params tests.

## Best Possible Solution

Land provider-local support that injects `allowed_models` only for `openrouter/auto`, preserves current OpenRouter wrappers, exposes any shared helper through `api.ts` or plugin-owned tests, documents the config, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the PR injects whenever `autoRouter.allowedModels` exists, while current main routes merged params into provider wrapping and lacks this feature. The real OpenRouter after-fix behavior is not proven because no runtime artifact is attached.

## Solution Assessment

No. The feature direction is useful, but this branch should gate to Auto requests, compose with the current wrapper chain, respect the plugin API boundary, define the wildcard/reasoning contract, add public docs, and provide real proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:98-106`
  - body: `autoRouter.allowedModels` is read from merged params and injected without checking that `ctx.modelId` is `openrouter/auto`. Defaults or reused model params can make fixed OpenRouter models receive the Auto Router plugin and suppress reasoning; normalize the model id and only inject for Auto requests.
  - confidence: 0.91
- **[P2] Preserve current OpenRouter wrappers:** `extensions/openrouter/stream.ts:120-122`
  - body: This fallback returns the partially wrapped stream, and the final path returns only the shared thinking hook. Current main also composes Anthropic prefill cleanup and DeepSeek V4 reasoning replay, so rebasing this as written would drop those protections for OpenRouter requests.
  - confidence: 0.89
- **[P2] Expose the helper through a public plugin seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core test imports the helper from the extension entrypoint. The extension boundary requires core tests to use `api.ts` or SDK seams for bundled plugin helpers, so export it from `extensions/openrouter/api.ts` or keep the assertion in plugin-owned tests.
  - confidence: 0.88
- **[P2] Constrain broad wildcard reasoning behavior:** `extensions/openrouter/stream.ts:57-59`
  - body: The PR accepts wildcard allowlists, but this check only recognizes entries that explicitly mention `x-ai/`. Provider-agnostic patterns such as `*/grok-*` can still route to x-ai models while reasoning stays enabled; reject broad unsafe patterns, resolve them, or document and enforce a narrower contract.
  - confidence: 0.78
- **[P3] Document the public OpenRouter config:** `extensions/openrouter/stream.ts:11-16`
  - body: The new user-facing param is only described in source JSDoc, while `docs/providers/openrouter.md` remains silent on `params.autoRouter.allowedModels`. Add public docs for exact IDs, wildcard limits, and the `openrouter/auto` config shape.
  - confidence: 0.76
