Keep this PR open: the feature is not already on current main and the upstream OpenRouter API shape is real, but the branch is still blocked by missing real behavior proof, stale/conflicting OpenRouter stream composition, and a plugin-boundary issue.

## What This Changes

The branch adds provider-local OpenRouter Auto Router allowed model payload injection from `params.autoRouter.allowedModels`, exports a helper, and adds OpenRouter/agent extra-params tests.

## Best Possible Solution

Land provider-local support that injects `allowed_models` only for `openrouter/auto`, preserves the current OpenRouter wrapper chain, uses a public plugin API seam or plugin-owned tests, documents the config shape, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows unconditional Auto Router injection and a stale wrapper chain compared with current main. No for the claimed live OpenRouter result, because no inspectable artifact is attached.

## Solution Assessment

No. The feature direction is reasonable, but the branch needs to be rebased and narrowed so it composes with current OpenRouter wrappers, gates behavior to Auto Router requests, respects the plugin boundary, and documents the new config.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:98-106`
  - body: `autoRouter.allowedModels` is read from merged params and injected for every OpenRouter model. If shared defaults or reused params are present, fixed OpenRouter models can receive an Auto Router plugin payload and have reasoning behavior changed, so normalize the model id and only apply this for `openrouter/auto`.
  - confidence: 0.91
- **[P2] Preserve the current OpenRouter wrapper chain:** `extensions/openrouter/stream.ts:120-129`
  - body: This branch returns only `streamFn` when the shared thinking hook is absent and otherwise returns only the shared hook result. Current main also wraps OpenRouter requests with Anthropic prefill cleanup and DeepSeek V4 reasoning replay, so rebasing this as written would drop those protections.
  - confidence: 0.89
- **[P2] Expose the helper through a public plugin seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. Core tests should use a plugin `api.ts`/SDK seam or keep the assertion in plugin-owned tests, so entrypoint refactors do not break unrelated core coverage.
  - confidence: 0.88
- **[P3] Document the public OpenRouter config:** `extensions/openrouter/stream.ts:11-19`
  - body: The new user-facing `params.autoRouter.allowedModels` key is only described in source comments, while the OpenRouter provider docs omit the config shape and wildcard limits. Add public docs so operators can use it without reading provider internals.
  - confidence: 0.76
