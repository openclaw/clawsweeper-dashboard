  - body: `wrapOpenRouterProviderStream` consumes `ctx.extraParams.autoRouter` for every OpenRouter model.   - confidence: 0.9

  - body: The new return path feeds only the routed/auto-router stream into the generic thinking hook and returns it directly.   - confidence: 0.88
- **[P2] Use a public OpenRouter test seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `../../extensions/openrouter/index.js` directly, which crosses the documented plugin boundary for core tests. Move the helper through `extensions/openrouter/api.ts` or keep the coverage in plugin-owned tests so core tests do not depend on the extension entrypoint shape.
  - confidence: 0.86
- **[P2] Handle provider-agnostic x-ai wildcards before enabling reasoning:** `extensions/openrouter/stream.ts:53-59`
  - body: The guard only detects explicit `x-ai/` entries, but OpenRouter's allowed-model pattern syntax supports provider-agnostic wildcards. A pattern such as `*/grok-*` can still route to x-ai/Grok while reasoning remains enabled, causing the same unsupported proxy-reasoning failure this guard is meant to avoid.
  - confidence: 0.78
