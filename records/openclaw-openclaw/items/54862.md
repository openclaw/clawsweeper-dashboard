## Summary

Keep open: this is a useful OpenRouter feature, but the PR is not mergeable as-is because source inspection still shows provider-scope regressions, a core/plugin boundary violation, wildcard reasoning gaps, and missing external real behavior proof.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` option that injects an `auto-router` `allowed_models` plugin payload and adds provider/agent tests.

## Best Possible Solution

Land provider-local support that injects `allowed_models` only for canonical `openrouter/auto`, preserves the current wrapper chain, uses the `api.ts` seam or plugin-local tests, documents the canonical config, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers: PR-head source shows unconditional `autoRouter.allowedModels` injection, and current main shows shared params are merged before provider wrapping. A live OpenRouter request is not needed to prove those source-level merge blockers.

## Solution Assessment

No. The feature direction matches OpenRouter's Auto Router API, but this patch is not the safest shape until Auto-only gating, current wrapper preservation, API-seam usage, wildcard reasoning handling, docs, and proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:106`
  - body: `autoRouter.allowedModels` is an Auto Router setting, but this wraps every OpenRouter model whenever the param is present. Because shared/default params are merged before provider wrapping, a global allowlist can add the `auto-router` plugin and alter reasoning behavior for fixed-model calls; gate the injection to canonical Auto requests.
  - confidence: 0.91
- **[P2] Preserve the current OpenRouter wrapper chain:** `extensions/openrouter/stream.ts:121-122`
  - body: When the shared thinking hook is unavailable, this returns the raw stream function and drops current-main Anthropic prefill cleanup plus DeepSeek V4 thinking replay wrappers. Add the auto-router wrapper inside the existing `wrapOpenRouterProviderStream` composition so rebasing this PR does not remove those protections.
  - confidence: 0.87
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. The scoped extension guide requires bundled plugin helpers used by core tests to be exposed through `api.ts`, so expose the helper there or keep the assertion in plugin-owned tests.
  - confidence: 0.88
- **[P2] Handle provider-agnostic Grok wildcard allowlists:** `extensions/openrouter/stream.ts:57-59`
  - body: The PR advertises wildcard allowlists, but this helper only recognizes explicit `x-ai/` entries. A provider-agnostic pattern such as `*/grok-*` can still route Auto Router to Grok while proxy reasoning stays enabled, so either reject/disable reasoning for broad unsafe patterns or resolve them before enabling reasoning.
  - confidence: 0.84
- **[P3] Document the canonical OpenRouter Auto config:** `extensions/openrouter/stream.ts:14-16`
  - body: The new example uses `openrouter/openrouter/auto`, while current docs and OpenRouter's own API use `openrouter/auto`. Move the user-facing config into `docs/providers/openrouter.md` with the canonical model ref so users do not copy a legacy-key-only shape.
  - confidence: 0.78
