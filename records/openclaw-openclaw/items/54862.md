## Summary

Keep open: current main does not implement OpenRouter Auto Router allowlists, and this PR is useful but still needs changes before merge because the injection is not scoped to Auto requests, the core test crosses the plugin boundary, wildcard reasoning handling is incomplete, and docs/changelog are missing.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` option that injects an OpenRouter `auto-router` plugin payload and adds OpenRouter extra-param tests.

## Best Possible Solution

Land a revised OpenRouter plugin implementation that gates allowlist injection to canonical Auto routes, preserves current reasoning wrappers and unsupported-model guards, exposes any shared helper through `extensions/openrouter/api.ts`, and documents/changelogs the new config.

## Reproduction Assessment

Yes. Under the PR head, configure `agents.defaults.params.autoRouter.allowedModels` and call a fixed OpenRouter model; source inspection shows the shared extra params flow into the PR's unguarded OpenRouter wrapper, which then injects the plugin.

## Solution Assessment

No. The feature direction matches OpenRouter's documented API, but the current diff is not the narrowest safe merge shape until Auto-only gating, API-seam usage, wildcard reasoning handling, current-main wrapper preservation, docs, and changelog are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard auto-router injection to Auto model requests:** `extensions/openrouter/stream.ts:99-107`
  - body: `autoRouter.allowedModels` is documented for the OpenRouter Auto route, but this block runs for every OpenRouter model whenever merged params contain a non-empty allowlist. Since `agents.defaults.params` are merged into fixed-model calls too, a shared Auto Router config can add the plugin payload and alter reasoning on non-Auto requests; gate this wrapper to canonical Auto model ids before injecting.
  - confidence: 0.91
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. Core tests that need bundled plugin helpers should use the plugin `api.ts` seam or move the assertion into the plugin test area, otherwise entrypoint refactors can break unrelated core tests.
  - confidence: 0.88
- **[P2] Treat provider-agnostic Grok wildcards as reasoning unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter's Auto Router pattern syntax supports provider-agnostic wildcards, so entries such as `*/grok-*` can still route to x-ai/Grok models. This helper only detects explicit `x-ai/` patterns, leaving proxy reasoning enabled for wildcard pools that can select unsupported models.
  - confidence: 0.84
- **[P2] Preserve OpenRouter unsupported-model reasoning guards:** `extensions/openrouter/stream.ts:134`
  - body: Current main intentionally disables proxy reasoning for stale OpenRouter Hunter Alpha refs. This replacement thinking-level expression only considers the new allowlist check before calling the shared wrapper, whose generic guard does not know about OpenRouter's Hunter Alpha exception, so a rebase can re-enable reasoning for a route current tests/docs say must skip it.
  - confidence: 0.8
- **[P3] Document the new OpenRouter config surface:** `extensions/openrouter/stream.ts:7-16`
  - body: This adds a user-facing `params.autoRouter.allowedModels` config, but the diff does not update `docs/providers/openrouter.md` or `CHANGELOG.md`. Add both so users can discover the setting and the release notes capture the feature.
  - confidence: 0.86
