## Summary

Keep open: current main does not implement OpenRouter Auto Router allowlists, and this PR is useful but still unsafe to merge because the diff applies the new plugin payload outside the Auto model path, crosses the plugin boundary from a core test, and leaves wildcard reasoning behavior incomplete.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` option that injects an `auto-router` plugin payload into OpenRouter requests and adds OpenRouter/provider extra-param tests.

## Best Possible Solution

Land a revised OpenRouter plugin implementation that gates the auto-router plugin payload to canonical Auto model refs, keeps helper exports on public plugin surfaces or plugin-local tests, treats broad x-ai/Grok wildcard allowlists conservatively for reasoning, and documents/changelogs the new config.

## Reproduction Assessment

Yes. Under the PR head, configuring `agents.defaults.params.autoRouter.allowedModels` and then using a fixed OpenRouter model is enough to reproduce the unintended payload injection, because current main merges global params before the PR wrapper injects the plugin without a model guard.

## Solution Assessment

No. The feature direction is reasonable, but this diff is not the best merge shape until the Auto-model guard, public plugin seam, wildcard reasoning handling, docs, and changelog are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard auto-router injection to Auto model requests:** `extensions/openrouter/stream.ts:99-106`
  - body: `autoRouter.allowedModels` is documented for `openrouter/auto`, but this block runs for every OpenRouter model whenever params contain a non-empty allowlist. Since `agents.defaults.params` are merged into every model request, a shared Auto Router config can add the plugin payload and alter reasoning on fixed-model OpenRouter calls. Gate the injection and reasoning change to the Auto model path.
  - confidence: 0.9
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` directly from the OpenRouter extension entrypoint. The repo boundary requires core tests needing bundled plugin helpers to use `api.ts` or an SDK facade, otherwise entrypoint refactors can break unrelated core tests. Export the helper through the public OpenRouter API seam or keep this coverage in OpenRouter plugin tests.
  - confidence: 0.86
- **[P2] Treat broad Grok wildcard allowlists as reasoning unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter allowed-model patterns support provider-agnostic wildcards, so patterns such as `*/grok-*` can still route Auto Router to x-ai/Grok models. This helper only flags entries that explicitly contain `x-ai/`, leaving proxy reasoning enabled for those wildcard pools and allowing the same x-ai reasoning failure the guard is meant to avoid.
  - confidence: 0.82
- **[P3] Document the new OpenRouter config surface:** `extensions/openrouter/stream.ts:14-16`
  - body: This adds a user-facing `params.autoRouter.allowedModels` config, but the PR does not change provider docs or the changelog even though the body says docs were updated. Add OpenRouter provider docs and a credited changelog entry for this feature before merge.
  - confidence: 0.9
