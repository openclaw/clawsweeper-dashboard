## Summary

Keep open: current main does not implement OpenRouter Auto Router allowed-model constraints, but this external PR is still blocked by missing real behavior proof and source-level issues in the stale provider wrapper/test changes.

## What This Changes

The branch adds OpenRouter `params.autoRouter.allowedModels` handling that injects an Auto Router `plugins` payload, exports a helper, and adds OpenRouter plus agent extra-params tests.

## Best Possible Solution

Land a rebased provider-local implementation that injects `allowed_models` only for `openrouter/auto`, preserves the current OpenRouter wrapper chain, respects the plugin API boundary, documents the params shape, and includes redacted live OpenRouter proof.

## Reproduction Assessment

Yes for the review blockers: PR-head source inspection shows unconditional Auto Router injection, stale wrapper composition, and a core test import across the plugin boundary. No for the claimed live OpenRouter result, because no inspectable artifact is attached.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best merge path until it is rebased, gated to Auto Router requests, composed with current wrappers, moved through an appropriate plugin seam or plugin-owned test, documented, and backed by live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:98-106`
  - body: `autoRouter.allowedModels` is read from merged params and injected for every OpenRouter model. Shared defaults or reused params can therefore add an Auto Router plugin payload and suppress reasoning on fixed-model OpenRouter requests, so normalize the model id and apply this only for `openrouter/auto`.
  - confidence: 0.91
- **[P2] Preserve the OpenRouter wrapper chain:** `extensions/openrouter/stream.ts:120-129`
  - body: This branch returns only `streamFn` when the shared thinking hook is absent and otherwise returns only the shared hook result. Current main also wraps OpenRouter requests with Anthropic prefill cleanup and DeepSeek V4 reasoning replay, so rebasing this as written would drop those protections.
  - confidence: 0.9
- **[P2] Use a public OpenRouter test seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: The new core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. Core tests should use a plugin `api.ts` seam or keep this assertion in plugin-owned tests, so entrypoint refactors do not break unrelated core coverage.
  - confidence: 0.88
- **[P3] Document the public OpenRouter config:** `extensions/openrouter/stream.ts:7-19`
  - body: The new user-facing `params.autoRouter.allowedModels` key is only described in source comments while OpenClaw's public provider docs omit the config shape and wildcard behavior. Add docs so operators can use the feature without reading provider internals.
  - confidence: 0.76
