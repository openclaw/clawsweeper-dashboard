## Summary

Keep open. The feature is not implemented on current main and is plausibly useful, but this PR still needs merge-blocking changes around OpenRouter Auto-only scoping, current stream-wrapper preservation, core/plugin test boundaries, docs, and real behavior proof.

## What This Changes

This PR adds an OpenRouter `params.autoRouter.allowedModels` option that injects an `auto-router` plugin payload, exports/tests the helper, and adds agent extra-param coverage.

## Best Possible Solution

Rebase on current main and land provider-local support that injects `allowed_models` only for canonical OpenRouter Auto requests, preserves existing stream wrappers, exposes test helpers through `extensions/openrouter/api.ts` when core tests need them, documents the config, and includes redacted live proof.

## Reproduction Assessment

Yes for the blocking review defects: source inspection of the PR head shows unconditional `autoRouter.allowedModels` injection, and current main shows shared params are merged before provider wrapping. I did not run a live OpenRouter request because the merge-blocking defects are visible in source.

## Solution Assessment

No. The feature direction matches OpenRouter's documented Auto Router API, but this patch is not the safest merge shape until Auto-only gating, wildcard reasoning handling, API-seam usage, current wrapper preservation, docs, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:106`
  - body: `autoRouter.allowedModels` is an Auto Router setting, but this block wraps every OpenRouter model when the param is present. Since shared/default params are merged before provider wrapping, a global allowlist can add the `auto-router` plugin and alter reasoning behavior for fixed-model OpenRouter calls; gate this to canonical Auto model requests before injecting.
  - confidence: 0.91
- **[P2] Preserve the current OpenRouter wrapper chain:** `extensions/openrouter/stream.ts:121-122`
  - body: When the shared thinking hook is unavailable, this returns the raw stream function, and the PR head also omits current-main Anthropic prefill cleanup and DeepSeek V4 thinking replay wrappers. Add the auto-router wrapper inside the current `wrapOpenRouterProviderStream` chain so rebasing this PR does not drop those existing OpenRouter protections.
  - confidence: 0.86
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. The scoped extension guide says core tests needing bundled plugin helpers should use a stable `api.ts` seam, so expose this helper there or keep the assertion in plugin-owned tests.
  - confidence: 0.88
- **[P2] Handle provider-agnostic Grok wildcard allowlists:** `extensions/openrouter/stream.ts:57-59`
  - body: The PR advertises wildcard allowlists, but this helper only treats patterns containing `x-ai/` as reasoning-unsafe. A provider-agnostic pattern such as `*/grok-*` can still route to Grok while proxy reasoning stays enabled, so either reject/disable reasoning for those broad patterns or resolve them against a model registry before enabling reasoning.
  - confidence: 0.84
- **[P3] Document the new OpenRouter Auto config:** `extensions/openrouter/stream.ts:14-16`
  - body: This adds a user-facing provider config option, but the PR does not update `docs/providers/openrouter.md`, and the only example here uses the stale duplicated `openrouter/openrouter/auto` ref. Add provider docs with the canonical `openrouter/auto` model ref and the `params.autoRouter.allowedModels` shape.
  - confidence: 0.8
