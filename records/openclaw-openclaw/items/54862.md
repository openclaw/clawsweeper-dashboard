## Summary

Keep open. Current main does not implement OpenRouter Auto Router allowlist support, but this external PR is not merge-ready because the branch is stale against the OpenRouter stream code, imports removed PI package names, applies the Auto Router plugin too broadly, crosses the plugin API boundary in a core test, and lacks real behavior proof.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` config path that injects an OpenRouter `auto-router` plugin payload, exports the helper, and adds OpenRouter plus agent extra-params tests.

## Best Possible Solution

Land a refreshed provider-local implementation that targets only `openrouter/auto`, composes with the existing OpenRouter wrapper chain, uses `api.ts` or plugin-local tests for helper coverage, documents the supported config, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows stale imports, ungated injection, wrapper-chain regression, and a core-test deep import. Current main lacks the requested feature, so this is a feature PR review rather than a current-main bug reproduction.

## Solution Assessment

No. The feature direction matches OpenRouter's Auto Router API, but this patch is not the narrow maintainable implementation until it is rebased, gated to Auto requests, composed with current wrappers, routed through the plugin API seam, documented, and proven live.

  - body: This branch still imports `@mariozechner/pi-agent-core`/`pi-ai`, but current main only declares the renamed `@earendil-works/pi-*` packages. As written, the changed TypeScript files cannot resolve these imports on current main; rebase the branch and update the imports consistently.

  - confidence: 0.92

  - body: When the shared OpenRouter thinking hook is absent, this now returns the raw stream function. Current main still applies Anthropic prefill cleanup and DeepSeek V4 reasoning replay in that path, so this rebase would drop existing OpenRouter request-shaping protections.
  - confidence: 0.9
- **[P2] Expose the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` directly from the OpenRouter plugin entrypoint. Core tests should use a plugin public seam such as `extensions/openrouter/api.ts`, or the helper coverage should stay in OpenRouter-owned tests, to avoid coupling core tests to bundled plugin internals.
  - confidence: 0.88
- **[P2] Handle broad wildcard allowlists before reasoning:** `extensions/openrouter/stream.ts:57-59`
  - body: The feature accepts wildcard allowlists, but the unsupported-reasoning check only detects patterns containing `x-ai/`.   - confidence: 0.82
