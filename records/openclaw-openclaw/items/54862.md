## Summary

Keep open: the requested OpenRouter Auto allowlist feature is not implemented on current main and the PR is a plausible successor to a dirty predecessor, but the branch still has blocking source-level issues and missing external real behavior proof before merge.

## What This Changes

The PR adds OpenRouter `params.autoRouter.allowedModels` support by injecting an Auto Router plugin into OpenRouter request payloads, exports the helper, and adds OpenRouter plus agent extra-params tests.

## Best Possible Solution

Land provider-local support that injects `allowed_models` only for canonical `openrouter/auto`, preserves the current OpenRouter wrapper chain, uses `api.ts` or plugin-owned tests for helper coverage, documents the canonical config, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows `autoRouter.allowedModels` is injected whenever present, and current main shows shared/default/model/agent params reach provider wrapping. The requested feature itself is not implemented on current main.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best implementation until it gates to Auto requests, preserves current wrapper composition, respects the plugin API seam, clarifies wildcard reasoning behavior, documents the config, and provides real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:106`
  - body: `autoRouter.allowedModels` is an Auto Router setting, but this wrapper applies it to every OpenRouter request whenever the param is present. Because defaults and model params are merged before provider wrapping, fixed-model calls can receive the Auto Router plugin and have reasoning suppressed; normalize the model id and inject only for canonical `openrouter/auto`.
  - confidence: 0.91
- **[P2] Preserve existing OpenRouter wrapper composition:** `extensions/openrouter/stream.ts:121-122`
  - body: The rebase drops current-main wrapper protections: when the shared thinking hook is absent this returns the raw stream function, and the final path no longer composes Anthropic prefill cleanup plus DeepSeek V4 reasoning replay. Add auto-router injection inside the existing wrapper chain so those protections stay active.
  - confidence: 0.88
- **[P2] Expose the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the extension entrypoint. The extension boundary requires core tests to use `api.ts` or SDK seams for bundled plugin helpers, so export the helper from `extensions/openrouter/api.ts` or keep this assertion in plugin-owned tests.
  - confidence: 0.87
- **[P2] Constrain wildcard reasoning behavior:** `extensions/openrouter/stream.ts:57-59`
  - body: The PR accepts wildcard allowlists, but this helper only detects entries that explicitly contain `x-ai/`. Provider-agnostic patterns such as `*/grok-*` can still route to models that need reasoning suppressed, so reject broad unsafe patterns, resolve them before sending reasoning, or document and enforce a narrower pattern contract.
  - confidence: 0.78
- **[P3] Document the canonical OpenRouter Auto config:** `extensions/openrouter/stream.ts:14-16`
  - body: The new source example uses the legacy-looking `openrouter/openrouter/auto` shape while the current provider docs use `openrouter/auto`, and this PR does not change provider docs. Add docs coverage for `autoRouter.allowedModels` with the canonical model ref so users do not copy the wrong shape.
  - confidence: 0.74
