## Summary

Keep open: current main still lacks OpenRouter Auto Router allowed_models support, but this PR is not merge-ready because it applies the auto-router payload too broadly, drops current OpenRouter wrapper composition, crosses the plugin/core test boundary, and lacks external after-fix proof.

## What This Changes

The PR adds OpenRouter params.autoRouter.allowedModels payload injection, exports the helper, and adds OpenRouter plus agent extra-params tests.

## Best Possible Solution

Land provider-local support that injects allowed_models only for canonical openrouter/auto, preserves the existing OpenRouter wrapper chain, uses api.ts or plugin-local tests for helper coverage, documents the canonical config, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers: source inspection shows autoRouter.allowedModels is injected whenever present, while current main merges default/model/agent params before provider wrapping. Current main also has no implementation of this feature, so a live OpenRouter request is not needed to prove the patch blockers.

## Solution Assessment

No. The feature direction matches OpenRouter's Auto Router API, but this patch is not the safest implementation until it gates to Auto requests, preserves current wrapper composition, respects the plugin API seam, handles wildcard reasoning edge cases, documents the supported config, and provides real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:107`
  - body: autoRouter.allowedModels is an Auto Router setting, but this path wraps every OpenRouter request whenever the param is present. Because shared/default params are merged before provider wrapping, fixed-model calls can receive the auto-router plugin and have reasoning disabled; normalize the model id and inject only for OpenRouter Auto.
  - confidence: 0.91
- **[P2] Preserve current OpenRouter wrapper composition:** `extensions/openrouter/stream.ts:121-122`
  - body: The rebase drops current-main wrapper protections: when the shared thinking hook is absent this returns the raw stream function, and the final path no longer composes Anthropic prefill cleanup plus DeepSeek V4 reasoning replay. Add auto-router injection inside the existing wrapOpenRouterProviderStream chain so those protections stay active.
  - confidence: 0.88
- **[P2] Expose the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports injectAutoRouterPlugin from extensions/openrouter/index.js. The extension boundary requires core tests to use api.ts or SDK seams for bundled plugin helpers, so export the helper from extensions/openrouter/api.ts or keep this assertion in plugin-owned tests.
  - confidence: 0.87
- **[P2] Handle provider-agnostic wildcard allowlists:** `extensions/openrouter/stream.ts:57-59`
  - body: The PR supports wildcard allowlists, but the reasoning suppression helper only detects explicit x-ai/ entries. Patterns such as */grok-* can still route Auto Router to Grok while reasoning injection remains enabled, so reject broad unsafe patterns, resolve them before sending reasoning, or enforce a narrower documented pattern contract.
  - confidence: 0.84
- **[P3] Document the canonical OpenRouter Auto config:** `extensions/openrouter/stream.ts:14-16`
  - body: The new example uses openrouter/openrouter/auto in source comments while current provider docs use openrouter/auto and do not document autoRouter.allowedModels. Add docs/providers/openrouter.md coverage with the canonical model ref so users do not copy a legacy-only shape.
  - confidence: 0.76
