  - body: The PR head calls `isProxyReasoningUnsupported` in the OpenRouter stream wrapper, but the added helper is named differently, so the provider path can throw a `ReferenceError` and the OpenRouter test job is already annotated on this line.   - confidence: 0.93
- **[P2] Guard auto-router injection to Auto model requests:** `extensions/openrouter/stream.ts:99-107`
  - body: `autoRouter.allowedModels` is documented for `openrouter/auto`, but this wrapper runs for every OpenRouter model when merged params contain a non-empty allowlist. Since `agents.defaults.params` are shared, fixed-model calls can receive the Auto Router plugin payload and altered reasoning behavior; gate the injection to canonical Auto model IDs.
  - confidence: 0.91
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: The core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. Core tests that need bundled plugin helpers should use `extensions/openrouter/api.ts` or move the coverage into the plugin tests, otherwise entrypoint refactors can break unrelated core tests.
  - confidence: 0.88
- **[P2] Treat provider-agnostic Grok wildcards as reasoning unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter's Auto Router pattern syntax supports provider-agnostic wildcards, so entries such as `*/grok-*` can still route to x-ai/Grok models. The PR's unsupported-model check only detects explicit `x-ai/` patterns, leaving proxy reasoning enabled for wildcard pools that can select unsupported models.
  - confidence: 0.84
- **[P2] Preserve OpenRouter unsupported-model reasoning guards:** `extensions/openrouter/stream.ts:228`
  - body: Current main disables proxy reasoning for stale OpenRouter Hunter Alpha refs. The PR's replacement thinking-level expression needs to keep that existing `isOpenRouterProxyReasoningUnsupportedModel(ctx.modelId)` guard, otherwise a rebase can re-enable reasoning for a route current docs say must skip it.
  - confidence: 0.8
- **[P3] Document and changelog the new config surface:** `extensions/openrouter/stream.ts:7-16`
  - body: This adds a user-facing `params.autoRouter.allowedModels` config, but the PR files do not update the OpenRouter provider docs or `CHANGELOG.md`. Add both so users can discover the setting and release notes capture the feature.
  - confidence: 0.86
