## Summary

Keep open: current main still lacks the requested OpenRouter Auto Router allowlist surface, but this PR needs provider-runtime fixes and real behavior proof before it is mergeable.

## What This Changes

This PR adds an OpenRouter stream wrapper for `params.autoRouter.allowedModels`, exports and tests the payload injection helper, and adds agent extra-param coverage.

## Best Possible Solution

Land a rebased OpenRouter plugin change that exposes documented `params.autoRouter.allowedModels` only for canonical Auto requests, preserves current wrapper behavior, and includes focused tests, docs, and redacted live proof.

## Reproduction Assessment

Yes. Source inspection on the PR head shows `autoRouter.allowedModels` is injected without an Auto-model guard, and current main merges shared default params before provider wrapping.

## Solution Assessment

No. The feature direction matches OpenRouter's API, but this patch is not the safest merge shape until Auto-only gating, wildcard reasoning handling, API-seam usage, current wrapper preservation, docs, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:106`
  - body: `autoRouter.allowedModels` is an Auto Router setting, but this block runs for every OpenRouter model with a non-empty allowlist. Since shared/default params are merged before provider wrapping, fixed-model requests can receive the `auto-router` plugin and have reasoning behavior changed; gate this wrapper to canonical Auto model ids before injecting.
  - confidence: 0.91
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. Core tests that need bundled plugin helpers should use a stable plugin `api.ts` seam or keep the assertion inside plugin tests, otherwise entrypoint refactors can break unrelated core tests.
  - confidence: 0.88
- **[P2] Treat provider-agnostic Grok wildcards as reasoning unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: The PR advertises wildcard allowlists, but this helper only recognizes explicit `x-ai/` entries. A pattern like `*/grok-*` can still route Auto Router to Grok while leaving proxy reasoning enabled, so requests can fail on the same unsupported-reasoning path this guard is meant to avoid.
  - confidence: 0.85
- **[P2] Preserve current OpenRouter stream wrappers when rebasing:** `extensions/openrouter/stream.ts:122-128`
  - body: Current main wraps OpenRouter streams with Anthropic prefill cleanup, DeepSeek V4 thinking replay handling, and the Hunter Alpha reasoning guard. The PR head does not carry those current-main protections, so the rebase should add auto-router injection inside the current wrapper chain rather than replacing it.
  - confidence: 0.82
- **[P3] Document the canonical OpenRouter Auto config:** `extensions/openrouter/stream.ts:14-16`
  - body: This adds a user-facing provider config option, but the PR has no provider-docs change and the JSDoc example uses the stale duplicated `openrouter/openrouter/auto` ref. Document `params.autoRouter.allowedModels` in `docs/providers/openrouter.md` using the canonical `openrouter/auto` ref so users can discover the setting safely.
  - confidence: 0.8
