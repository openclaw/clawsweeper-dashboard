Full review comments:

- **[P2] Gate auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:106`
  - body: `autoRouter` comes from effective params, which can include shared `agents.defaults.params` or agent-level params. Because this wrapper injects the plugin for every OpenRouter model, a fixed model like `openrouter/openai/gpt-5.4` can unexpectedly get an Auto Router plugin payload and reasoning suppression; gate this path to `auto` / `openrouter/auto` before wrapping.
  - confidence: 0.92
- **[P2] Preserve the current OpenRouter wrapper chain:** `extensions/openrouter/stream.ts:121-129`
  - body: Current main wraps OpenRouter streams with Anthropic prefill cleanup and DeepSeek V4 reasoning replay after provider routing/family hooks. This branch returns `streamFn` or the family hook result directly, so rebasing it as-is would drop those current OpenRouter compatibility wrappers.
  - confidence: 0.9
- **[P2] Use a public OpenRouter test seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: Core agent tests should not deep-import bundled plugin entrypoints. `extensions/AGENTS.md` says helpers needed by core tests should be exposed through `api.ts` or another public artifact, so importing `../../extensions/openrouter/index.js` couples a core test to a private plugin entrypoint shape.
  - confidence: 0.88
- **[P2] Handle provider-agnostic Grok wildcards:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter documents provider-agnostic wildcard patterns, but this guard only catches strings containing `x-ai/`. An allowlist like `*/grok-*` can still route Auto Router to Grok while reasoning injection stays enabled, which reintroduces the x-ai incompatibility this guard is meant to avoid.
  - confidence: 0.78
