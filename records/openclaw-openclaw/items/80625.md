## Summary

Keep open. Current main still has the Anthropic thinking recovery helper and tests, but the Claude 4.6 production path does not install that helper when signed thinking replay is preserved; the linked fix PR was closed unmerged after live Anthropic proof became non-reproducible.

## What This Changes

The issue reports that Anthropic `thinking` or `redacted_thinking` blocks can permanently poison a session after a prompt-cache miss unless OpenClaw strips and retries the replay.

## Best Possible Solution

Land or reopen an equivalent to https://github.com/openclaw/openclaw/pull/80745: a bounded Anthropic exact-error retry in the production stream path, with focused regression coverage and maintainer-accepted proof if live Anthropic remains non-reproducible.

## Reproduction Assessment

No high-confidence live current-main reproduction is available. The issue has a concrete provider log and current source shows the no-retry path, but the later live Anthropic attempt on May 12, 2026 could not reproduce the exact rejection.

## Solution Assessment

Yes, the one-shot retry on the exact Anthropic thinking-block modification error is the narrowest maintainable direction. Stripping all thinking blocks at persistence time would risk breaking valid signed-thinking replay and prompt-cache behavior.
