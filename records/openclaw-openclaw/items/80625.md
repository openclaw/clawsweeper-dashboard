## Summary

Keep open. Current main still has the exact-error Anthropic recovery helper only in tests/helper code, while the Claude 4.6 production replay path preserves latest thinking blocks and does not install the one-shot retry needed for this cache-miss failure; the related fix PR was closed unmerged, and the reporter has fresh 2026.5.19 impact evidence.

## What This Changes

Recover Anthropic Claude sessions that become stuck after cache-miss replay of persisted `thinking` or `redacted_thinking` blocks.

## Best Possible Solution

Wire a bounded Anthropic thinking-block recovery retry into the production Claude replay path, preserving normal signed-thinking replay but stripping thinking blocks once after the exact cache-miss rejection.

## Reproduction Assessment

No live high-confidence reproduction was run here. The issue has a concrete provider error log plus source evidence that current main can replay preserved Claude 4.6 thinking blocks without the existing exact-error recovery wrapper.

## Solution Assessment

Yes, the best fix direction is the narrow one-shot recovery used by the existing helper and the closed related PR. Stripping all thinking at persistence time would be broader and could break valid Claude signed-thinking replay.
