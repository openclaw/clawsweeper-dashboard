## Summary

Keep open. Current main still preserves Claude 4.6 signed thinking replay and has no production caller for the exact-error recovery helper, while the linked fix PR was closed unmerged after live Anthropic proof became non-reproducible.

## What This Changes

The issue requests automatic recovery or clearer graceful handling when Anthropic rejects replayed `thinking` or `redacted_thinking` blocks after a prompt-cache miss.

## Best Possible Solution

Keep normal Claude signed-thinking replay, then add one bounded retry that strips thinking only after the exact Anthropic immutability rejection and covers both request-time and pre-content transport error shapes.

## Reproduction Assessment

No high-confidence live reproduction is available. The issue has a concrete provider log and current source explains the remaining stuck path, but the linked PR's later live Anthropic attempt did not reproduce the exact cache-miss rejection.

## Solution Assessment

Yes, the best code direction is the narrow one-shot exact-error recovery rather than stripping all thinking at persistence time. Maintainers still need to decide whether simulated regression proof is enough while the live provider behavior is not reproducing.
