## Summary

Keep open. Current main has the pieces for Anthropic thinking recovery, but the Claude 4.6 production path still preserves latest signed thinking blocks and does not wire the exact-error retry helper, so the reported cache-miss failure remains actionable.

## What This Changes

The issue requests automatic recovery when Anthropic rejects replayed `thinking` or `redacted_thinking` blocks after a prompt cache miss.

## Best Possible Solution

Wire a bounded Anthropic thinking-block recovery retry into the production Claude replay path, preserving normal signed-thinking replay but stripping thinking blocks once after the exact cache-miss rejection.

## Reproduction Assessment

No. The issue has a concrete real log and source inspection shows the current Claude 4.6 path can replay preserved thinking blocks without a production retry, but I did not establish a live Anthropic cache-miss reproduction.

## Solution Assessment

Yes. A one-shot retry on the exact Anthropic thinking-block modification error is narrower and safer than stripping all thinking at persistence time or changing the general Claude 4.6 replay policy.
