## Summary

Keep open. Current main and the latest release still have the Anthropic thinking recovery helper but do not install it on the production Claude 4.6 stream path; the follow-up fix attempt at https://github.com/openclaw/openclaw/pull/80745 was closed unmerged after live Anthropic proof could not reproduce the exact rejection.

## What This Changes

The issue reports that Anthropic `thinking` or `redacted_thinking` blocks can poison a persisted session after prompt-cache invalidation and leave future turns failing until reset.

## Best Possible Solution

Keep signed-thinking replay for modern Claude models, but add a one-shot production recovery wrapper that strips thinking blocks only after the exact Anthropic modification rejection and before any content has been emitted.

## Reproduction Assessment

No high-confidence live current-main reproduction is available. The reporter supplied the exact provider error and source inspection shows the current Claude 4.6 path can replay preserved thinking blocks without installing the existing recovery helper, but the later live Anthropic attempt did not reproduce the exact rejection.

## Solution Assessment

Yes, the code direction is the narrowest maintainable fix. Retrying once on the exact Anthropic thinking-block rejection is safer than stripping all thinking at persistence time or changing the normal Claude 4.6 replay policy.
