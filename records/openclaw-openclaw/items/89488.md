The PR updates shared and native Anthropic payload shaping to prefer the latest real user text/image cache marker over trailing tool-result carriers, preserves a trailing tool-result marker when the marker budget allows, and adds shared policy regression tests.

## Best Possible Solution

## Solution Assessment

Yes, this is an acceptable best current fix: it changes the existing Anthropic payload-shaping owners rather than adding a new mode, preserves the tool-result fallback, and bounds message markers by the documented four-breakpoint limit. A native provider regression test would make the final repair easier to land confidently.
