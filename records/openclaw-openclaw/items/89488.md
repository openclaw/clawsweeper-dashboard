## Summary

Keep open: this PR is a focused bug fix for the tool-result-tail cache-marker problem, and I found no blocking patch defect. Current main still tags the trailing user-role tool_result, while the PR updates both the shared Anthropic payload policy and the native Anthropic path to prefer the latest real user text/image block.

## What This Changes

The PR changes Anthropic cache-control placement so trailing user-role tool_result messages no longer receive the conversation cache marker when an earlier real user text/image block is available, with focused regression coverage for the shared payload policy.

## Best Possible Solution

Merge the narrow tool-result cache-marker fix after CI and maintainer validation, while keeping the broader inbound-metadata cache instability tracked on https://github.com/openclaw/openclaw/issues/86063.
