## Summary

Keep open for normal review. Current main still drops detail for empty provider error bodies, treats SSE `data:` error envelopes as opaque text, and can let body-reader failures escape; the PR is a narrow shared-helper fix with focused tests and a changelog entry.

## What This Changes

The PR updates shared provider HTTP error extraction to handle empty or unreadable bodies and SSE `data:` error envelopes, adds unit coverage, and adds a changelog entry.

## Best Possible Solution

Land this shared-helper patch if CI stays green, keeping provider error detail handling centralized in `src/agents/provider-http-errors.ts` rather than adding per-provider forks.

## Reproduction Assessment

Yes, from source inspection. Current main returns no detail for empty bodies, preserves SSE frames as raw text after bare JSON parsing fails, and does not catch `readResponseTextLimited` failures before formatting.

## Solution Assessment

Yes. Updating the shared provider HTTP helper is the narrowest maintainable fix because Google streaming and many other provider surfaces already use `createProviderHttpError`.
