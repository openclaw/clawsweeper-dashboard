## Summary

Keep open for normal maintainer review. Current main still loses provider HTTP error detail for empty, unreadable, or SSE-framed bodies, and this PR fixes that in the shared helper with focused regression coverage and no blocking review findings.

## What This Changes

The branch updates shared provider HTTP error parsing to catch body-read failures, parse SSE `data:` error envelopes, fall back to `statusText` for empty bodies, and add tests plus a changelog entry.

## Best Possible Solution

Land the shared-helper fix after maintainer review and exact-head CI, keeping the behavior generic in `src/agents/provider-http-errors.ts` with the targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection against current main shows empty/whitespace error bodies return no detail, body-reader failures can escape formatting, and the Google streaming transport routes non-OK responses through the same helper.

## Solution Assessment

Yes. Centralizing the fallback and SSE-envelope parsing in the shared provider HTTP helper is the narrowest maintainable solution because the SDK and provider call sites already share that path.
