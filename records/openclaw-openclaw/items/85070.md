## Summary

Keep open. The report is recent, has a concrete live Telegram forum-topic failure on v2026.5.19, and current source still has a source-reproducible path where group/topic subagent completions require message-tool evidence and are marked failed when the announce agent emits only normal final text.

## What This Changes

The issue requests reliable Telegram forum-topic delivery for subagent completion announcements when the announce agent does not call the message tool.

## Best Possible Solution

Make subagent completion announce delivery deterministic for Telegram group topics while preserving message-tool-only visibility policy: the requester announce agent must receive/obey explicit message-tool delivery context or the runtime must surface a clear failed/blocked delivery state without silent loss or duplicate sends.

## Reproduction Assessment

Yes, at source level and with the reporter's live v2026.5.19 logs. Current code and tests show group/channel subagent completions fail with the reported error when the announce agent returns normal text without message-tool evidence.

## Solution Assessment

Unclear for the exact patch, but the best direction is narrow: preserve explicit message-tool-only group visibility and fix the completion announce path so the message-tool contract is reliably available and enforced. A raw automatic final-text fallback would need extra care to avoid wrong-surface or duplicate group sends.
