## What This Changes

The issue asks OpenClaw to reliably deliver Telegram forum-topic subagent completion announcements when the completion agent returns final text instead of calling the message tool.

## Best Possible Solution

Adopt a maintainer-approved Codex subagent completion delivery contract that either reliably forces/elicits `message(action="send")` for message-tool-only handoffs or safely converts valid completion text into a visible Telegram topic message, then prove it with live Telegram forum-topic subagent completion output.

## Solution Assessment
