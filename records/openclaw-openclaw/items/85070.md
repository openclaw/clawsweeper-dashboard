## Summary

Keep open. This is a fresh, source-reproducible Telegram forum-topic message-loss bug: current main still treats a group/channel subagent completion as failed when the requester announce agent returns normal final text without message-tool delivery evidence, so the shipped user problem is not fully resolved by the earlier hardening work.

## What This Changes

The issue asks OpenClaw to reliably deliver Telegram forum-topic subagent completion announcements when the completion announce agent misses the required `message(action="send")` call.

## Best Possible Solution

Keep the mediated delivery policy, but make subagent completion announce handoffs deterministically produce a `message(action="send")` call for Telegram group/forum topics, with regression coverage for missed-tool final text and proof from a real or Crabbox Telegram topic flow.

## Reproduction Assessment

Yes at source/log level: the issue includes a concrete 2026.5.19 Telegram forum-topic run, and current main contains the same failure branch when a message-tool-only completion lacks messaging-tool evidence. I did not perform a live Telegram reproduction in this read-only pass.

## Solution Assessment

No for the current behavior: it prompts for the message tool but still accepts repeated model misses as failed delivery. The narrower maintainable fix is to preserve message-tool-only routing while adding a deterministic forced-tool retry or equivalent mediated delivery path.
