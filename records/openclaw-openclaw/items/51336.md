Keep this open: current main and the latest release still emit provider-free transient overload/rate-limit copy, and the remaining work is already owned by the open linked PR https://github.com/openclaw/openclaw/pull/55851.

## What This Changes

The issue asks for user-facing overload and rate-limit errors to include the active API provider name so users can distinguish provider-side failures from local or gateway failures.

## Best Possible Solution

Resolve or replace the linked PR’s implementation so the existing formatter path appends safe active-provider context for overloaded and rate-limit errors without leaking request/profile identifiers.

## Reproduction Assessment

Yes: source inspection shows the transient error formatter still returns provider-free constants even when provider/model options are available to the caller. The focused source path is an assistant error that reaches formatAssistantErrorText with an overloaded or generic rate-limit payload.
