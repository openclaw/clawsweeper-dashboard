## Summary

Keep this open: current main and the latest release still use provider-agnostic transient error copy, and an open implementation PR explicitly links this issue but has not merged.

## What This Changes

The issue requests provider names in user-facing overload and rate-limit errors so users can distinguish upstream provider failures from local or gateway failures.

## Best Possible Solution

Land a narrow provider-aware transient formatter fix through the linked PR or a replacement: use the active failed provider/model context, preserve provider reset details, omit raw request IDs from chat-visible copy, and add focused tests.
