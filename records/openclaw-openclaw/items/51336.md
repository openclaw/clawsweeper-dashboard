## What This Changes

Request to include the active API provider name in user-facing overload, rate-limit, and similar provider error messages so users can distinguish provider-side failures from gateway or local failures.

## Best Possible Solution

Refresh and review https://github.com/openclaw/openclaw/pull/55851, preserving the shared sanitized transient-formatter approach, or replace it with a narrower current-main PR that surfaces only the active failed provider/model without leaking profile, session, trigger, or request identifiers.

## Solution Assessment

Yes, with maintainer approval and a refreshed branch. Extending the shared transient formatter and passing already-prepared active provider/model context is the narrow maintainable shape; the open PR should land or be replaced before this issue closes.
