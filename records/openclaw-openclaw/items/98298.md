## Summary

Keep open. Current main still uses contextPruning thinking stripping only for estimation, not as a replay-view mutation, and the signed-thinking replay problem remains tied to open Anthropic replay work rather than a merged fix.

## What This Changes

The issue requests in-memory pruning of older assistant thinking blocks in contextPruning, with optional config for how many recent assistant turns retain thinking.

## Best Possible Solution

Settle the Anthropic signed-thinking replay policy in https://github.com/openclaw/openclaw/issues/92201 and https://github.com/openclaw/openclaw/pull/94493 first; add a narrow contextPruning replay-view pass or config knob only if maintainers still want operator-visible control after that provider decision.

## Reproduction Assessment

Yes at source level: current contextPruning only uses thinking stripping for estimation, while current Anthropic replay paths can still preserve completed-turn signed thinking on Claude 4 models. I did not run live Anthropic traffic in this read-only review.

## Solution Assessment

Unclear: a new contextPruning.thinking config object is not clearly the best first fix because provider replay policy already owns the signed-thinking contract. The safer path is to land, reject, or reshape the focused Anthropic replay fix first, then decide whether context pruning needs a user-facing knob.
