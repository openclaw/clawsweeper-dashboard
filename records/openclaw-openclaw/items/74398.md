## Summary

## What This Changes

The PR threads `chat.history.verboseLevel` into Control UI state and uses `full` to auto-expand chat tool cards, with reset-path tests and a changelog entry.

## Best Possible Solution

Land this narrow Control UI state/render plumbing after normal CI and mergeability checks, leaving the gateway API and manual tool-card toggle semantics unchanged.

## Solution Assessment

Yes. Consuming the existing `chat.history.verboseLevel` in Control UI state and feeding the existing `autoExpandToolCalls` seam is the narrowest maintainable fix.
