## What This Changes

The PR adds a warning-only model-catalog check to selected `openclaw config set` model paths, adds a path-detection test, and changes skills eligibility for parse-error entries.

## Best Possible Solution

Keep the warning-only UX, but implement it inside the current operation-based config-set pipeline, reuse the existing model resolution/catalog helpers, add `runConfigSet` coverage plus a changelog entry, and leave skills parse-error eligibility to a separate typed change.

Yes. Static inspection gives a high-confidence reproduction of the PR defects: current main routes writes through `runConfigSet` and `runConfigOperations`, while the PR hooks a stale inline writer, and `SkillEntry` has no `parseError` field.

## Solution Assessment
