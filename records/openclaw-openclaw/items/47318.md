## What This Changes

The PR adds a non-blocking `openclaw config set` warning for model-reference paths, checks the model catalog, adds path-detection tests, and also changes skills eligibility for parse-error entries.

## Best Possible Solution

Rebase onto the current config operation path, implement one sanitized warning for documented default and per-agent primary model refs using runtime-equivalent catalog lookup, and keep the skills parse-error change in its separate PR.

## Reproduction Assessment

Yes, by source inspection. Current main has no config-set model catalog warning, and the PR-head defects are visible in the diff: the stray skills line references a missing type field and the detector skips `agents.list[].model.primary`.

## Solution Assessment

No. Warning-only validation is a reasonable direction, but this branch is not the best implementation until it is rebased into the active config mutation path, covers the documented per-agent object form, sanitizes output, and removes unrelated skills churn.
