## Summary

Keep this PR open: the config-set warning is a useful direction and current main still lacks it, but the branch is not merge-ready because it checks raw model text instead of the runtime resolver, misses a supported per-agent primary path, still has unsafe terminal interpolation, and carries unrelated generated/runtime churn.

## What This Changes

The PR adds a non-blocking `openclaw config set` warning for unknown model refs, plus focused path-detection tests and unrelated formatting/generated-file changes.

## Best Possible Solution

Keep the narrow config-set warning, but base it on the canonical post-mutation model resolver and alias index, cover default and per-agent primary paths, sanitize displayed refs, remove unrelated files, and prove the real CLI path.

## Reproduction Assessment

Yes. Source inspection shows current main routes `config set` through `runConfigOperations`, normalizes/writes the config, and has no model-catalog warning hook for invalid model refs.

## Solution Assessment

No. A non-blocking warning is a reasonable fix shape, but this patch should use the canonical resolver, cover per-agent `.model.primary`, sanitize terminal output, and drop unrelated generated/runtime changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve model refs with the runtime resolver before warning:** `src/cli/config-cli.ts:770`
  - body: `warnIfModelNotInCatalog` parses the raw CLI value with `DEFAULT_PROVIDER` and checks the catalog directly. Runtime model selection also honors configured aliases through `buildModelAliasIndex`/`resolveModelRefFromString`, so supported alias values can be warned as missing; validate the normalized post-mutation value through the same resolver contract before emitting a miss.
  - confidence: 0.9
- **[P2] Detect per-agent primary model paths:** `src/cli/config-cli.ts:752`
  - body: The detector matches `agents.list.<n>.model` but not `agents.list.<n>.model.primary`. Agent model config supports object-shaped primaries, so setting a bad per-agent primary through that path still takes the old silent-success path this PR is meant to cover.
  - confidence: 0.87
- **[P3] Sanitize model refs before warning:** `src/cli/config-cli.ts:793-794`
  - body: The warning interpolates model text derived from CLI/config input without `sanitizeForLog`. A ref containing ANSI or control characters can spoof or obscure terminal diagnostics, so sanitize any displayed model ref before writing the warning.
  - confidence: 0.84
