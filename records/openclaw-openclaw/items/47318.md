## Summary

Keep open: current main still lacks the requested `openclaw config set` model-catalog warning, and this PR is useful but not merge-ready because it patches a stale config-set path, misses supported per-agent primary paths, includes unrelated skills code, lacks real CLI proof, and needs catalog-warning semantics reviewed.

## What This Changes

The PR adds a non-blocking model-catalog warning to `openclaw config set` for selected agent model paths, adds a path-detection test, and includes an unrelated skills eligibility change.

## Best Possible Solution

Rebase the warning into `runConfigOperations`, use runtime-equivalent sanitized model resolution for default and per-agent primary refs, remove unrelated skills changes, coordinate with the doctor diagnostic work, and include redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts these config writes through `runConfigOperations` and emits only the generic restart message, while the PR-head defects are visible in the diff and documented model/config contracts.

## Solution Assessment

No. Warning-only validation is a reasonable direction, but this branch is not the best implementation until it is moved to the shared config operation path, sanitized, scoped to supported model forms, split from unrelated skills work, and proven with real CLI output.
