## Summary

This PR should stay open because current main still lacks a config-set model-catalog warning, but the latest branch is not ready to merge: it still validates the raw value outside the runtime resolver contract, misses per-agent primary paths, emits unsanitized terminal text, and includes unrelated generated/runtime churn.

## What This Changes

The PR adds a non-blocking `openclaw config set` warning for unknown model references, plus focused config CLI tests and broad unrelated formatting/generated-file changes.

## Best Possible Solution

Keep the narrow config-set warning, but resolve the post-mutation model value through the canonical runtime resolver and alias index, cover default and per-agent primary paths, sanitize warning output, remove unrelated churn, and prove the real CLI path.

## Reproduction Assessment

Yes. Source inspection shows current main routes `config set` through `runConfigOperations`, normalizes/writes the config, and has no model-catalog warning hook for invalid model refs.

## Solution Assessment

No. A non-blocking warning is a reasonable fix shape, but this patch should validate through the canonical post-mutation resolver, include per-agent `.model.primary`, sanitize displayed refs, and drop unrelated generated/runtime changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve the written model ref before warning:** `src/cli/config-cli.ts:770`
  - body: `warnIfModelNotInCatalog` parses the raw CLI value and checks the catalog directly before the config is normalized. Runtime model selection also honors configured aliases through `buildModelAliasIndex`/`resolveModelRefFromString`, so supported alias values or normalized values can be warned as missing; validate the post-mutation config value through the same resolver contract before emitting a miss.
  - confidence: 0.9
- **[P2] Detect per-agent primary model paths:** `src/cli/config-cli.ts:752`
  - body: The detector matches `agents.list.<n>.model` but not `agents.list.<n>.model.primary`. Agent model config supports object-shaped primaries, so setting a bad per-agent primary through that path still silently succeeds without the warning this PR is meant to add.
  - confidence: 0.87
- **[P3] Sanitize model refs before warning:** `src/cli/config-cli.ts:793-794`
  - body: The warning interpolates model text derived from CLI/config input without `sanitizeForLog`. A ref containing ANSI or control characters can spoof or obscure terminal diagnostics, so sanitize any displayed model ref before writing the warning.
  - confidence: 0.84
