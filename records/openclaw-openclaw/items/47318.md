## Summary

Keep open, but this PR is not merge-ready. The requested warning is still useful and not implemented on current main, but the branch currently has resolver false positives, misses a supported per-agent model path, echoes unsanitized model text, carries broad unrelated generated/formatting churn, is merge-conflicting, and still lacks real `openclaw config set` proof.

## What This Changes

This PR adds a non-blocking `openclaw config set` model-catalog warning for selected agent model paths plus focused detector/config CLI tests, while also carrying unrelated generated and formatting changes.

## Best Possible Solution

Implement this diagnostic through the shared post-change model-resolution path, cover default and per-agent string/object primary shapes, sanitize terminal output, keep the branch scoped to config/model diagnostics, and require redacted real CLI proof for invalid and valid refs.

## Reproduction Assessment

Yes, by source inspection. Current main’s `runConfigOperations` applies and normalizes config changes without a model-catalog warning hook, so the linked invalid-model scenario would still be accepted silently by `config set`.

## Solution Assessment

No, not as currently implemented. The useful direction is a warning-only diagnostic, but it should use the shared resolver against the post-change config and avoid unrelated branch churn.
