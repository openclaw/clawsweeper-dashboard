## Summary

Keep open. Current main still lacks this config-set model-catalog warning, but this branch is not mergeable as-is because it patches a stale config-set path, misses documented per-agent primary refs, includes unrelated skills work, and lacks real CLI proof.

## What This Changes

The PR adds a non-blocking `openclaw config set` warning for selected agent model refs, a path-detection test, and an unrelated skills eligibility change.

## Best Possible Solution

Retarget a focused warning into `runConfigOperations`, use runtime-equivalent sanitized model resolution for default and per-agent primary refs, remove unrelated skills changes, coordinate with the doctor diagnostic work, and include redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts these config writes through `runConfigOperations` and emits only the generic restart message, while the PR-head defects are visible in the live diff and documented model/config contracts.

## Solution Assessment

No. Warning-only validation is a reasonable direction, but this branch is not the best implementation until it is moved to the shared config operation path, sanitized, scoped to supported model forms, split from unrelated skills work, and proven with real CLI output.
