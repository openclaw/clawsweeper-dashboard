## Summary

Keep open: current main still lacks this specific config-set model warning, but this PR is not mergeable as written because it hooks a stale config writer, includes an unrelated untyped skills change, prints unsanitized user-controlled warning text, and lacks real CLI proof.

## What This Changes

The PR adds non-blocking model-catalog warnings for selected `openclaw config set` model paths, a path-detection test, and an unrelated skills eligibility change.

## Best Possible Solution

Implement a sanitized warning-only model catalog advisory in the shared `runConfigOperations` path, cover single and batch config-set cases with focused tests, remove the unrelated skills hunk, and coordinate the UX boundary with the open doctor and override-warning PRs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the PR defects: the branch hooks an old inline writer while current main routes `config set` through `runConfigSet` and `runConfigOperations`, and `SkillEntry` has no `parseError` field.

## Solution Assessment

No. Warning rather than blocking is a reasonable direction, but this patch needs to target the current operation pipeline, drop the unrelated skills change, sanitize terminal output, and include real CLI proof before merge.
