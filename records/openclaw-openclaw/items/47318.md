## Best Possible Solution

Retarget a focused advisory into `runConfigOperations`, use runtime-equivalent sanitized model resolution for default and per-agent primary refs, remove unrelated skills changes, coordinate with the doctor diagnostic work, and include redacted real CLI proof.

## Reproduction Assessment

## Solution Assessment

No. Warning-only validation is a reasonable direction, but this branch should be rebased into the shared config operation path, sanitized, expanded to per-agent `model.primary`, split from unrelated skills work, and proven with real CLI output.
