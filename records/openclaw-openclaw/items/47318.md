## Summary

Keep open. Current main still lacks the requested `openclaw config set` invalid-model warning, but this PR is not ready to merge because its warning is attached to the old inline config writer instead of the current operation-based set pipeline, and it includes an unrelated untyped skills-status change.

## What This Changes

The PR adds a non-blocking model-catalog warning for selected `openclaw config set` model paths, adds a small path-detection test, and changes skills eligibility for parse-error entries.

## Best Possible Solution

Keep the warning-only user experience, but implement it against current main's config operation pipeline so value and batch `config set` writes are covered, reuse the existing model-ref/profile/catalog lookup helpers, and leave skills parse-error eligibility to a separate typed change.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction of the PR defects: current main routes `config set` through `runConfigSet` and `runConfigOperations`, while the PR hooks the old inline writer, and `SkillEntry` has no `parseError` field.

## Solution Assessment

No. The requested warning is a reasonable narrow fix, but this patch is not the best solution because it misses the current write path and mixes unrelated skills-status behavior into a CLI config change.
