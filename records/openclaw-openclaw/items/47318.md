## Summary

Keep open: the requested warning is still useful and not implemented on current main, but this PR needs changes before merge because it can warn on valid configured aliases, misses valid per-agent primary paths, still lacks real CLI proof, and carries unrelated generated/runtime churn.

## What This Changes

The PR adds a non-blocking model-catalog warning to `openclaw config set` for selected model config paths plus a small path-detector test, while the live branch also carries unrelated formatting and generated asset changes.

## Best Possible Solution

Implement the warning in the shared config mutation path using the same configured model resolver and alias index as runtime, cover default and per-agent primary shapes, sanitize displayed refs, keep it warn-only, remove unrelated churn, add focused regression coverage, and provide redacted CLI output showing the warning path.

## Reproduction Assessment

Yes, by source inspection: current main routes `config set` through `runConfigOperations` and emits only the generic update message, with no catalog-warning helper present. I did not run the CLI because this review is read-only.

## Solution Assessment

No. A non-blocking warning is a reasonable direction, but the patch should use the canonical model-ref resolver, cover the object primary config shapes, sanitize terminal output, and drop unrelated generated/runtime changes before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve configured aliases before warning:** `src/cli/config-cli.ts:770`
  - body: `parseModelRef` skips the configured alias index that runtime model selection uses. A config where `agents.defaults.models` defines alias `Opus` can validly resolve `agents.defaults.model.primary: "Opus"`, but this warning will check `openai/Opus` against the catalog and report a false miss. Use the same configured resolver/alias index as runtime, ideally against the post-mutation config.
  - confidence: 0.9
- **[P2] Check per-agent primary model paths:** `src/cli/config-cli.ts:752`
  - body: `AgentConfig.model` may be an object with `primary`, and current audit/doctor code treats `agents.list.<id>.model.primary` as a model-ref source. This detector only matches `agents.list.<n>.model`, so `openclaw config set agents.list.0.model.primary <bad-ref>` still silently skips the new warning.
  - confidence: 0.84
- **[P3] Sanitize model refs before printing warnings:** `src/cli/config-cli.ts:793`
  - body: The warning interpolates CLI-provided model text into terminal output. Route displayed refs through the existing terminal/log sanitizer before passing them to `warn()` so ANSI or control characters cannot spoof diagnostics.
  - confidence: 0.8
