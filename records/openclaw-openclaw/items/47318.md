## Summary

## What This Changes

The PR adds a non-blocking `openclaw config set` warning for selected agent model paths when the parsed model ref is missing from the loaded model catalog, adds path-detection tests, and currently also changes skills eligibility.

## Best Possible Solution

Rebase the warning into `runConfigOperations`, use runtime-equivalent sanitized catalog lookup for default and per-agent primary model refs, remove unrelated skills changes, add focused tests, and include redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main lacks a config-set model-catalog warning, and the PR-head defects are visible in the live diff and surrounding config/model contracts; I did not run the CLI because this review is read-only.

## Solution Assessment

No. Warning-only validation is a reasonable direction, but this branch is not the best implementation until it is rebased onto the shared config operation path, aligned with supported model forms and catalog semantics, sanitized, scoped, and proven with real CLI output.
