## Summary

Keep open: current main still lacks the requested `config set` model-catalog warning, but this PR is not correct as-is because it hooks the old inline config writer, carries an unrelated untyped skills change, lacks the required changelog entry, and prints unsanitized model text.

## What This Changes

The PR adds a warning-only model-catalog check for selected `openclaw config set` model paths, a path-detection unit test, and an unrelated skills eligibility change.

## Best Possible Solution

Implement a warning-only catalog advisory in the current config-set operation pipeline with regression tests and a changelog entry, while keeping custom and forward-compatible model refs non-blocking.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction of the PR defects: the PR hooks an old inline writer, while current main routes `config set` through `runConfigSet` and `runConfigOperations`, and `SkillEntry` has no `parseError` field.

## Solution Assessment

No. Warning rather than blocking is a maintainable direction, but this patch needs to target the current operation pipeline, drop the unrelated skills hunk, sanitize warning output, and add the required changelog entry.
