## Solution Assessment

No. A non-blocking warning is a reasonable direction, but this patch should validate the normalized post-mutation value through runtime resolver/catalog semantics, cover per-agent primary paths, sanitize output, and drop unrelated files.

  - body: `warnIfModelNotInCatalog` parses the raw CLI value with `DEFAULT_PROVIDER` and checks the catalog directly.   - body: The detector matches `agents.list.<n>.model` but not `agents.list.<n>.model.primary`. Per-agent model config supports object-shaped primaries, so setting a bad per-agent primary through that path still follows the old silent-success path this PR is meant to improve.
  - confidence: 0.87
