## Summary

Keep open. Current main still lacks a generic `config set` warning for arbitrary invalid agent model refs, but this PR is not merge-ready: it is conflicting/outdated, inserts the warning into an old write path, relies on catalog data that can include user-configured rows, prints unsanitized user input, lacks real behavior proof, and carries an unrelated skills-status change from a closed PR.

## What This Changes

The PR adds a non-blocking model-catalog warning to `openclaw config set` for agent default/per-agent model paths, adds a path-detection test, and also includes an unrelated skills-status eligibility change.

## Best Possible Solution

Rebase this into the current shared config mutation pipeline and implement a sanitized warning that uses runtime-equivalent model resolution plus a trusted catalog source, while coordinating semantics with the related doctor and model-override warning PRs.

## Reproduction Assessment

Yes, by source inspection: current main writes `config set` changes through `runConfigOperations` and emits only the generic restart message, while schema validation is structural except for static suppressions. I did not run the command because this was a read-only review.

## Solution Assessment

No, not as currently patched. A warn-only diagnostic is a reasonable direction, but the branch needs to target the current write path, avoid trusting user-configured catalog rows, sanitize output, remove unrelated changes, and provide real behavior proof.

- **[P2] Move the warning into runConfigOperations:** `src/cli/config-cli.ts:498-500`
  - body: Current main no longer uses the older inline write action this branch patches; `runConfigSet` builds operations and delegates to `runConfigOperations`. If the warning stays at this old insertion point after rebase, active single and batch `config set` flows can miss the advisory.
  - confidence: 0.91

  - body: `loadModelCatalog()` appends models declared in local config, so a bogus model already present under `models.providers.*.models` can be treated as found and skip the warning.   - confidence: 0.86
- **[P3] Sanitize model refs before printing warnings:** `src/cli/config-cli.ts:315-317`
  - body: The warning interpolates model refs from user-controlled CLI/config input directly into terminal output. Route the displayed model value through the existing terminal sanitizer so ANSI or control characters cannot spoof the diagnostic text.
  - confidence: 0.78
