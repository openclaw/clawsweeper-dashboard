## Summary

Keep open: the underlying DX problem still exists on current main, but this branch still has blocking false-positive warning paths, misses a supported per-agent model shape, needs output sanitization, and carries unrelated generated/formatting churn.

## What This Changes

The branch adds a non-blocking model-catalog warning to `openclaw config set` for selected agent model config paths, plus focused detector tests and a large set of unrelated generated/formatting changes.

## Best Possible Solution

Implement the config-set warning against the canonical post-mutation model resolver and alias index, cover default and per-agent primary shapes, sanitize displayed refs, keep the warning non-blocking, and remove unrelated generated/formatting changes.

## Reproduction Assessment

Yes, by source inspection: current main routes `config set` through `runConfigOperations` and emits only generic update output, with no model-catalog warning hook. I did not run the CLI because this was a read-only review.

## Solution Assessment

No. A non-blocking warning is a reasonable direction, but this patch should resolve the normalized post-mutation model through the same alias/profile-aware resolver as runtime, cover per-agent `.model.primary`, sanitize terminal output, and drop unrelated churn.
