## Summary

Keep open. The UI readability goal is useful, but the patch moves truncation into the shared session-key parser, which can hide exact fallback identifiers in existing consumers that do not expose the raw key, and the PR only provides unit-test evidence rather than after-fix Control UI proof.

## What This Changes

The PR formats Web Control UI activity and exec-approval session keys with friendly channel labels and truncated direct-chat identifiers, and adds session-display tests.

## Best Possible Solution

Keep the friendly UI labels, but preserve full identifiers by moving shortening to display-site helpers or returning full and short forms, then add real Control UI proof and avoid closing the broader session-label issue until its precedence path is handled or split.

## Reproduction Assessment

Yes, at source level: current activity and exec-approval rendering paths display raw sessionKey directly. I did not run a browser reproduction, and the PR's possible truncation regression is source-visible through shared helper consumers.

## Solution Assessment

No. The friendly-label direction is reasonable, but the best fix should keep canonical parsing full-fidelity and apply shortening only where the UI can preserve the raw key.
