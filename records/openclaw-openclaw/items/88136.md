## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Merge pending hard timeouts before scheduling new timeouts:** `src/gateway/server-methods/agent-job.ts:272-273`
  - body: When a provider/preflight hard timeout is pending during the 15s grace window, a later `end` with `aborted=true` but without hard-timeout metadata still enters this branch and calls `schedulePendingAgentRunTimeout`, which clears the pending hard timeout and replaces it with an ordinary timeout. Move the preservation check ahead of timeout scheduling or make `schedulePendingAgentRunTimeout` merge against the existing pending snapshot.
  - confidence: 0.9
- **[P1] Merge sticky dedupe outcomes across sibling keys:** `src/gateway/server-methods/agent-wait-dedupe.ts:286-300`
  - body: `setGatewayDedupeEntry` only checks the existing entry for `params.key`, but reads choose the freshest terminal snapshot across `agent:<runId>` and `chat:<runId>`. A hard timeout stored under one key can still be hidden by a later completion/error under the sibling key, so compare both sibling snapshots before accepting terminal overwrites.
  - confidence: 0.88
