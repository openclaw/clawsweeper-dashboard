## What This Changes

The branch tags main-session cron system events, filters those tagged events out of heartbeat cron reminder prompt construction, and adds a heartbeat regression test.

## Best Possible Solution

Land this PR or an equivalent marker-and-skip fix after rebasing, updating cron context-key expectations, and adding redacted real cron/heartbeat proof while preserving normal cron wrapper and delivery semantics.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: main-session cron queues a `cron:` system event, heartbeat peeks and wraps it, and reply setup drains the same queue into `System:` lines.

## Solution Assessment

Mostly yes. The marker-and-skip approach is narrow and maintainable for duplicated model input, while an alternative cron prompt that references the existing system message would be a broader behavior change.
