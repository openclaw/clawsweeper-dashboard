## Summary

Keep open. Current main still lacks ambient env-proxy mode in the provider guarded model-fetch path, and maintainer context says the recent fake-IP fix did not cover this remaining HTTP(S)_PROXY routing gap while #72480 remains the open focused fix candidate.

## What This Changes

The issue asks for Gemini CLI/WebUI calls to respect HTTP(S)_PROXY in fake-IP proxy environments and to surface explicit failures instead of hanging silently.

## Best Possible Solution

Review, rebase, and land #72480 or an equivalent shared provider-fetch change that preserves #76887 fake-IP policy and explicit-proxy validation with regression coverage.

## Reproduction Assessment

Yes for the source-level failure path: Google model requests call buildGuardedModelFetch, which calls fetchWithSsrFGuard without trusted env-proxy mode while strict mode ignores ambient HTTP_PROXY/HTTPS_PROXY. I did not live-reproduce the UI hang against Gemini and a local proxy.

## Solution Assessment

Yes, the narrow maintainable fix is in the shared provider guarded-fetch seam, gated the same way as the existing media-understanding helper. #72480 appears to be that candidate, but it needs rebasing over #76887 rather than closing this issue now.
