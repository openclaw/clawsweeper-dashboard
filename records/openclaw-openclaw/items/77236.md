## Summary

Keep open: the PR addresses the main Telegram fallback bug, but it still gates configured reasoning defaults on a Telegram flag that does not carry owner-only authorization in the text-message path, and it still lacks real behavior proof.

## What This Changes

The PR updates Telegram reasoning-level resolution to fall back from session state to per-agent and global reasoning defaults, adds Telegram dispatch tests, and adds a changelog entry.

## Best Possible Solution

Land a narrow Telegram fix that carries or reuses canonical owner-aware reasoning authorization, preserves session/per-agent/global/off precedence, keeps focused regression coverage, and includes redacted live Telegram proof.

## Reproduction Assessment

Yes, from source inspection. Current main resolves Telegram reasoning from stored session state only and returns `off` for a fresh session, despite docs and the shared resolver supporting configured per-agent/global defaults for authorized users.

## Solution Assessment

No. The fallback order is close, but the PR should derive `canUseReasoningState` from the same owner/authorized/admin semantics as the shared resolver, or carry owner authorization into Telegram dispatch before landing.
