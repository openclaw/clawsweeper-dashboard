## Summary

Keep open: the PR fixes the Telegram reasoning-default gap in the right area, but the patch is not merge-ready because configured defaults bypass the shared authorization gate and the user-facing fix lacks a changelog entry.

## What This Changes

The PR changes Telegram reply dispatch to fall back from stored session reasoning state to per-agent/global `reasoningDefault` values and adds dispatch tests for precedence.

## Best Possible Solution

Thread Telegram authorization state into the reasoning resolver, apply configured defaults only when allowed, preserve the intended precedence, and cover both authorized and unauthorized Telegram cases plus changelog.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh Telegram session with no stored `reasoningLevel` reaches `return "off"` before checking configured reasoning defaults; the PR tests cover that fallback path, but I did not execute them.

## Solution Assessment

No. The fallback order is directionally correct, but the patch should match the shared authorization semantics before exposing configured reasoning defaults and should include the required changelog entry.
