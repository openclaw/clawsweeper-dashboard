## Summary

This PR should stay open: it addresses a real Telegram reasoning-default bug, but the proposed fallback currently bypasses the documented authorization gate and the user-facing fix is missing a changelog entry.

## What This Changes

The branch updates Telegram reply dispatch to fall back from session reasoning state to per-agent and global reasoning defaults, with dispatch tests for precedence.

## Best Possible Solution

Thread Telegram command authorization into the reasoning resolver for configured defaults, preserve the intended precedence, add focused auth/precedence tests, and add the changelog entry before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh Telegram session with no stored reasoningLevel reaches "off" before checking configured reasoning defaults; the PR's added tests target that path, though I did not execute them in this read-only review.

## Solution Assessment

No. The fallback order is directionally correct, but the implementation should apply configured defaults only when the Telegram sender is command-authorized and should include the required changelog entry.
