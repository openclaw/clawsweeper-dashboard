## Summary

Keep open. The PR fixes the documented Telegram fallback gap, but it applies configured reasoning defaults without the shared authorization gate and is missing the required user-facing changelog entry.

## What This Changes

The PR changes Telegram reply dispatch to fall back from session reasoning state to per-agent and global `reasoningDefault` config, with precedence tests.

## Best Possible Solution

Preserve the fallback behavior, thread Telegram authorization into the reasoning resolver for configured defaults, add unauthorized coverage and the changelog entry, then land it against the linked bug.

## Reproduction Assessment

Yes. Source inspection of current main shows `resolveTelegramReasoningLevel` returns `off` after a missing or empty session `reasoningLevel` instead of consulting per-agent/global defaults; the PR adds tests for that path, but I did not run them in this read-only review.

## Solution Assessment

No. The fallback order is directionally right, but the resolver should apply configured defaults only for Telegram senders authorized to use reasoning state and the fix needs an Unreleased changelog entry.
