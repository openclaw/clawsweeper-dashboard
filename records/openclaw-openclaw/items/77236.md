## Summary

Keep open: the PR targets a real Telegram bug, but the proposed fallback applies configured reasoning defaults without the shared authorization gate and also lacks the required changelog entry for a user-facing fix.

## What This Changes

The PR updates Telegram message dispatch to fall back from session reasoning state to per-agent and global reasoning defaults, with dispatch tests for precedence.

## Best Possible Solution

Thread Telegram command authorization into the reasoning resolver, apply configured defaults only when allowed, preserve the intended session/per-agent/global precedence, and add focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh Telegram session with no stored reasoningLevel reaches return off before checking configured reasoning defaults; the PR tests cover that path, but I did not execute them in this read-only review.

## Solution Assessment

No. The fallback order is directionally right, but the PR should also match the shared authorization semantics before exposing configured reasoning defaults and should include the required changelog entry.
