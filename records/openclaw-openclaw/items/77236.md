## Summary

Keep this PR open: it targets a valid, source-reproducible Telegram bug, but the current patch still misses the documented owner authorization path for configured reasoning defaults.

## What This Changes

The PR updates Telegram message dispatch to fall back from session reasoning state to per-agent and global `reasoningDefault` config, adds Telegram dispatch regression tests, and records the fix in the changelog.

## Best Possible Solution

Update Telegram message dispatch to use the same owner-aware authorization fact as the documented shared resolver, with an owner-only regression test and the existing precedence coverage preserved.

## Reproduction Assessment

Yes, from source inspection. Current main returns `off` for a fresh Telegram session without checking per-agent or global `reasoningDefault`, while docs and the shared resolver define those fallbacks for authorized/owner contexts.

## Solution Assessment

No. The fallback order is close, but using only `ctxPayload.CommandAuthorized` misses Telegram owners authorized through `commands.ownerAllowFrom`; the safer fix is to carry or recompute owner-aware authorization before resolving reasoning state.
