## Summary

Keep open: the PR targets a real Telegram bug, but the current head still lets explicit higher-precedence `off` settings fall through to a lower-precedence global `stream` default.

## What This Changes

The PR adds Telegram dispatch fallback from session reasoning state to per-agent/global `reasoningDefault`, gates configured defaults on Telegram command authorization, adds dispatch tests, and adds a changelog entry.

## Best Possible Solution

Land the Telegram resolver fallback with `off` treated as a valid resolved value at each precedence level, plus regression tests for explicit `off` overrides.

## Reproduction Assessment

Yes. Source inspection reproduces the original main-branch gap and the PR-head regression path: an authorized Telegram message with session or per-agent `off` and global `stream` resolves to `stream`.

## Solution Assessment

No. The PR’s direction is right, but the implementation must preserve documented precedence for explicit `off` before checking lower-precedence defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Honor explicit off before falling through:** `extensions/telegram/src/bot-message-dispatch.ts:222-243`
  - body: The resolver only returns higher-precedence session/per-agent values when they are `on` or `stream`. If a session has `reasoningLevel: "off"` or the matching agent has `reasoningDefault: "off"` while `agents.defaults.reasoningDefault` is `"stream"`, this falls through and enables reasoning despite the documented precedence. Treat `off` as a resolved value before checking lower-precedence defaults.
  - confidence: 0.95
