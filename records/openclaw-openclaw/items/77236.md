## Summary

Keep open: the PR is directionally correct and now includes the earlier authorization gate and changelog entry, but it still mishandles an explicit per-agent `reasoningDefault: "off"` when a global default is enabled.

## What This Changes

The PR updates Telegram dispatch to fall back from session reasoning state to per-agent and global reasoning defaults, gates configured defaults on Telegram command authorization, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Implement Telegram reasoning-default resolution with nullish precedence for all valid configured values, including explicit `"off"`, while keeping the authorization gate and changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: configure the selected Telegram agent with `reasoningDefault: "off"`, set `agents.defaults.reasoningDefault: "stream"`, use an authorized fresh session, and the PR code falls through to the global stream default.

## Solution Assessment

No. The PR should use the same nullish per-agent-then-global precedence as the shared resolver so `"off"` is respected before falling back to the global default.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Respect explicit per-agent off defaults:** `extensions/telegram/src/bot-message-dispatch.ts:242`
