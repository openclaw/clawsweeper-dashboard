## Summary

Keep open: the PR fixes the Telegram fallback order and explicit `off` precedence, but the new configured-default gate still uses raw Telegram `CommandAuthorized` instead of the resolved owner/authorized-sender contract documented for reasoning defaults.

## What This Changes

The PR updates Telegram message dispatch to fall back from session reasoning state to per-agent and global reasoning defaults, adds Telegram dispatch regression tests, and adds a changelog entry.

## Best Possible Solution

Land a revised PR or replacement that preserves the fallback and explicit-`off` precedence but computes the configured-default gate from resolved command authorization before applying Telegram reasoning defaults.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh Telegram session with configured `reasoningDefault` falls through to `off` because the resolver only checks stored session values.

## Solution Assessment

No. The fallback order is the right direction, but the proposed gate is narrower than the documented resolved authorization contract and should use owner/authorized-sender/operator-admin semantics.
