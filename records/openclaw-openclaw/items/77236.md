## Summary

Keep open: the PR addresses a real Telegram default-resolution bug, but it is not merge-ready because configured reasoning defaults need the same authorization gate as the shared reply resolver and the user-facing fix needs a changelog entry.

## What This Changes

The PR changes Telegram reply dispatch to fall back from session reasoning state to per-agent and global reasoning defaults, with precedence tests.

## Best Possible Solution

Thread Telegram command authorization into the reasoning resolver, apply configured defaults only when authorized, preserve session/per-agent/global precedence, and add focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh Telegram session with no stored reasoningLevel reaches "off" before checking per-agent or global reasoningDefault, matching the linked issue's repro path.

## Solution Assessment

No. The fallback order is directionally correct, but the patch should match the documented shared authorization semantics and include the required changelog entry before merge.
