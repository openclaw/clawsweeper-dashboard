The issue asks for cron announce deliveries to be visible in the destination channel session transcript so later `sessions_history` calls can see prior cron reports.

## Best Possible Solution

Review and land one coordinated fix that resolves the normal outbound destination session, appends only successfully delivered cron assistant output with an idempotency key, and preserves the isolated worker transcript boundary.
