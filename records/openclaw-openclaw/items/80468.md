## Summary

Keep open. Current main still sends isolated cron announce delivery with the cron worker session and only queues main-session awareness, while an open closing PR, https://github.com/openclaw/openclaw/pull/80786, is already the focused fix path with real after-fix proof.

## What This Changes

Cron direct announce messages should be visible in the destination channel session transcript so later channel turns and sessions_history can reference them.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80786 or an equivalent focused patch, coordinating overlap with https://github.com/openclaw/openclaw/pull/74913 so only one coherent cron transcript mirror implementation reaches main.

Yes. Current main is source-reproducible because direct cron announce delivery uses the cron worker session and only queues main-session awareness, and the issue discussion adds live release proof where delivered channel output had zero hits in the destination session transcript.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the normal outbound destination session and append only successfully delivered cron output idempotently; the open fix PR follows that direction, so the issue should remain open until it merges.
