## Summary

Keep open. Current main still has the reported split-brain path: isolated cron direct delivery sends with the cron run session and only queues main/global awareness, while `sessions_history` reads the requested channel session transcript. The linked open fix PR already uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Cron announce deliveries should be recorded in the destination channel session transcript so later channel turns can see prior cron reports.

## Best Possible Solution

Mirror only successfully delivered cron final payloads into the resolved destination channel session transcript with an idempotency key, while keeping the isolated worker transcript separate.

## Reproduction Assessment

Yes. Source inspection on current main shows the direct cron delivery path uses the isolated cron session and only queues main/global awareness, and the issue has a recent live WhatsApp reproduction showing 0 destination-session hits.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse outbound destination-session resolution and transcript mirroring after successful delivery; https://github.com/openclaw/openclaw/pull/80786 is the active candidate but needs normal maintainer review because it is not merged.
