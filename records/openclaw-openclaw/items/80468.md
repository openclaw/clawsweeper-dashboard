## Summary

Keep open. Current main still sends isolated cron announce delivery with the cron worker session context and only queues main-session awareness, while an open closing PR specifically targets destination channel-session mirroring but has not merged.

## What This Changes

Cron announce deliveries should be visible in the destination channel session transcript so later `sessions_history` reads can find what the agent posted.

## Best Possible Solution

Land or replace the focused destination-session mirror fix so successful cron announce payloads are appended idempotently to the same channel session that later `sessions_history` reads, while keeping the isolated worker transcript separate.

## Reproduction Assessment

Yes. Current main is source-reproducible because direct cron delivery uses `params.agentSessionKey` and only queues main-session awareness, and the discussion adds a live WhatsApp repro with zero token hits in the destination session transcript.

## Solution Assessment

Yes. The maintainable fix is to resolve the outbound destination session used by normal sends, ensure its entry exists, and append only the successfully delivered assistant payload with an idempotency key after delivery succeeds.
