## Summary

Keep open. Current main still has the source-proven gap: isolated cron direct delivery sends with the cron run session context and only queues main-session awareness, while the linked open PR https://github.com/openclaw/openclaw/pull/80786 specifically uses closing syntax for this issue and is the active fix path.

## What This Changes

Cron announce deliveries into a channel should be visible or queryable in that channel session history after successful delivery.

## Best Possible Solution

Land one coordinated fix, likely the linked closing PR or a replacement, that idempotently mirrors only successfully delivered cron assistant output into the resolved destination channel session.

## Reproduction Assessment

Yes. Current main is source-reproducible because direct cron delivery uses the cron run session and only queues main-session awareness, with a recent live 2026.5.7 WhatsApp repro showing the delivered token absent from the destination session transcript.

## Solution Assessment

Yes. Resolving the same outbound destination session used by normal sends and appending the delivered assistant text after successful delivery is the narrow maintainable fix; the issue should close only after the linked implementation lands or is superseded.
