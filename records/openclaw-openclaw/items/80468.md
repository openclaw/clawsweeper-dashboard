## Summary

Keep open. Current main still sends isolated cron announce delivery with the cron run session context and only queues main-session awareness, while later `sessions_history` reads the requested channel session transcript; the open closing PR https://github.com/openclaw/openclaw/pull/80786 is the right item to review or land before closing this issue.

## What This Changes

Cron announce deliveries should be mirrored into the destination channel session transcript so later `sessions_history` reads can see delivered reports.

## Best Possible Solution

Review and land a focused fix like https://github.com/openclaw/openclaw/pull/80786 that resolves the outbound destination session, ensures its session entry, and idempotently appends only successfully delivered assistant payloads there.

## Reproduction Assessment

Yes. Source inspection on current main shows direct cron delivery uses the cron run session and only queues main-session awareness, and the discussion adds a live before-fix channel delivery with zero token hits in the destination session transcript.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror the successfully delivered final payload into the resolved destination channel session after delivery succeeds; the active closing PR is the current implementation candidate.
