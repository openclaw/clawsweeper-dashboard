## Summary

Keep open. Current main still has the cron/channel transcript gap, and https://github.com/openclaw/openclaw/pull/80786 is an open closing PR for the same bug; this issue should stay open until that PR or an equivalent fix merges.

## What This Changes

Cron announce deliveries should be visible in the destination channel session transcript so later channel turns can see prior cron reports.

## Best Possible Solution

Review and land one coordinated fix that resolves the normal outbound destination session, ensures the session entry, and idempotently appends only the successfully delivered cron assistant payload into that transcript.

## Reproduction Assessment

Yes. Current main is source-reproducible because cron direct delivery uses the cron run session key and only queues main/global awareness, while the issue discussion adds a live stable repro with zero hits in the destination session transcript.

## Solution Assessment

Yes. The best path is the narrow destination-session mirror implemented by the open fix PR, with coordination against the narrower `session:<id>` mirror PR before merge.
