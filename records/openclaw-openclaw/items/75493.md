## Summary

Keep open. The PR targets a real current-main gap in the MCP loopback cron path, but the proposed patch is incomplete: it adds a current-channel value at the HTTP/request/tool-resolution edges while the loopback tool cache still neither accepts nor forwards that value, so the new context would be dropped before `resolveGatewayScopedTools` and the added regression assertion would not hold.

Keep the regression test and changelog entry, then land it to resolve #75261.
