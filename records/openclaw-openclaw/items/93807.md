## What This Changes

The issue reports that `tools.web.fetch.useTrustedEnvProxy` respects proxy env vars but not `NO_PROXY`, so local/internal `web_fetch` targets are proxied or blocked instead of reached directly.

## Best Possible Solution

Keep this issue open until maintainers either keep the documented strict fallback or land one security-reviewed shared-guard PR that scopes NO_PROXY local access and preserves metadata/link-local protections.
