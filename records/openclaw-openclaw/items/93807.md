## What This Changes

The issue requests that `web_fetch` honor `NO_PROXY`/`no_proxy` when `tools.web.fetch.useTrustedEnvProxy` is enabled, including for local and private-network targets.

## Best Possible Solution

Keep this issue open until maintainers either keep the documented strict fallback or land one security-reviewed canonical PR with scoped NO_PROXY local access, metadata/link-local protections, docs, tests, and real web_fetch proof.

Unclear until maintainers decide the security policy. The maintainable fix belongs in the shared guarded-fetch layer, not a web_fetch wrapper-only workaround, and must preserve metadata/link-local protections if direct local access is approved.
