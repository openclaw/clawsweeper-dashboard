## Summary

Keep open: current main and v2026.6.8 still clear scopes for device-less shared-token/password WebSocket operator clients, and checked-in tests assert the resulting missing-scope behavior. The issue is paired with open implementation attempts and still needs a maintainer security/product decision rather than cleanup closure.

## What This Changes

The issue asks for non-local WebSocket clients with successful gateway token auth to retain requested operator scopes so `chat.send` and other write-scoped RPCs work.

## Best Possible Solution

Resolve the bounded token-scope or device-token policy in the Gateway auth path, then update source, tests, and docs for both retained-scope and still-cleared cases before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 clear device-less shared-token/password WebSocket operator scopes, and checked-in tests assert the resulting missing-scope behavior.

## Solution Assessment

No for unconditional scope retention. The maintainable solution is a maintainer-approved bounded token-scope or device-token contract rather than trusting client-declared scopes by default.
