## Summary

Keep open. Current main still clears self-declared operator scopes for device-less shared-token/password WebSocket operator sessions outside narrow local/trusted exceptions, and open closing PRs need maintainer/security policy review before this issue can be resolved.

## What This Changes

The issue asks OpenClaw to preserve requested operator scopes for non-local WebSocket operator clients that authenticate successfully with a shared gateway token.

## Best Possible Solution

Define one explicit WebSocket auth policy, then either land a reviewed scoped fix with regression coverage and docs or document that remote operator clients must use device identity or device tokens.

## Reproduction Assessment

Yes, from current source and checked-in tests. A valid shared-token operator connection without device identity is accepted with scopes cleared, and the tests expect a scoped RPC to fail with `missing scope`.

## Solution Assessment

Unclear. Preserving scopes would address the report, but it changes the remote device-less operator trust boundary, so the best path is maintainer/security approval followed by aligned code, tests, and docs.
