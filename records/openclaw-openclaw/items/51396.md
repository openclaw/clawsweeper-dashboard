## Summary

Keep open: current main and v2026.6.6 still clear scopes for device-less shared-token/password WebSocket operator clients, and the remaining fix is tied to an open auth/security product decision rather than a landed implementation.

## What This Changes

The issue requests that non-local token-authenticated WebSocket operator clients retain requested operator scopes so `chat.send` and other `operator.write` methods work after `hello-ok`.

## Best Possible Solution

Resolve the bounded token-scope or device-token policy in the open maintainer-reviewed auth path, with docs and proof for both retained-scope and still-cleared cases, before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 clear device-less shared-token/password WebSocket operator scopes, and checked-in tests assert the resulting missing-scope behavior.

## Solution Assessment

No for the unconditional scope-retention request. The safer solution is a maintainer-approved bounded token-scope or device-token contract rather than trusting client-declared scopes by default.
