## Summary

Keep open: current main and v2026.6.6 still clear scopes for the reported device-less shared-token WebSocket operator path, and the linked open PR still needs maintainer/security review before this issue should close.

## What This Changes

The issue asks Gateway WebSocket auth to retain requested operator scopes for non-local device-less clients that successfully authenticate with a shared token so write-scoped RPCs such as chat.send do not fail.

## Best Possible Solution

Settle the Gateway auth policy, then either land a maintainer-approved, tested WebSocket scope-retention fix or document the current device-binding requirement for LAN token-auth operator clients.

## Reproduction Assessment

Yes from source inspection: current main clears scopes for the reported device-less shared-token operator WebSocket path, and the default-token/compat tests assert missing-scope behavior. I did not run a live Docker/LAN repro.

## Solution Assessment

Unclear/no: preserving non-local token-auth operator scopes may fix the workflow, but it changes a security-sensitive auth boundary. The best fix needs an explicit maintainer policy decision plus focused tests for both retained-scope and still-cleared cases.
