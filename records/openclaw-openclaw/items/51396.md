## What This Changes

The issue reports remote device-less WebSocket clients losing requested operator scopes after valid shared-token auth, causing write-scoped RPCs such as `chat.send` to fail.

## Best Possible Solution

## Solution Assessment

No for blanket retention of client-declared scopes. The maintainable fix needs a bounded maintainer-approved token-scope or device-token contract rather than trusting arbitrary remote device-less operator scope claims by default.
