- **[P1] Preserve message-tool-only completion gating:** `src/agents/subagent-announce-delivery.ts:1243-1247`
  - body: Adding `!isSubagentCompletion` makes configured group/channel subagent completions skip `completionRequiresMessageToolDelivery`, so the direct agent call can deliver normal final text into rooms where docs and current tests require `message(action=send)` evidence.   - confidence: 0.93
- **[P1] Keep hosted tool-call turns out of lifecycle failure:** `src/agents/agent-command.ts:1632-1644`
  - body: This branch converts any pending tool-call metadata or `tool_calls`/`toolUse` stop reason into lifecycle `phase: "error"`. OpenResponses/Codex tool-call turns can be valid non-terminal follow-up states, and current SDK/UI/gateway/subagent consumers treat lifecycle error as a terminal failure, so this compatibility change needs a narrower classifier or sibling contract updates.
  - confidence: 0.85
- **[P2] Apply fallback text before retrying empty completions:** `src/agents/subagent-announce.ts:402-405`
  - body: The new empty-reply guard returns `false` before the existing fallback assignment runs. When an ok completion has an empty model reply but `fallbackReply` contains the frozen child result, the PR preserves the child for retry and still delivers nothing instead of using the available fallback text.
  - confidence: 0.88
- **[P3] Restore the specific gateway handoff assertion:** `src/agents/subagent-announce-delivery.test.ts:4289-4293`
  - body: This test now only checks that the gateway was called, so it would pass even if the handoff lost the channel, target, deliver flag, or delivery mode. Keep the focused `expectGatewayAgentParams` assertion and update only the fields that intentionally change.
