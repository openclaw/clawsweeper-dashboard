Full review comments:

- **[P1] Preserve message-tool-only completion gating:** `src/agents/subagent-announce-delivery.ts:1246`
  - body: Adding this guard makes every `subagent_announce` completion skip `completionRequiresMessageToolDelivery`, so a group/channel configured with `visibleReplies: "message_tool"` can receive normal final text without `message(action=send)`. Current docs and maintainer notes define that mode as keeping final text private unless the message tool sends it, so the fix needs to preserve `sourceReplyDeliveryMode` or land an explicit approved exception.
  - confidence: 0.91
- **[P1] Keep hosted tool-call turns out of lifecycle failure:** `src/agents/agent-command.ts:1628-1633`
  - body: This branch treats `pendingToolCalls` and `stopReason: "tool_calls"` as lifecycle errors. OpenClaw's OpenResponses gateway uses that same metadata to return normal `function_call` output, and the SDK maps lifecycle errors to `run.failed`, so hosted-tool turns can look failed to subscribers even when the response contract is satisfied.
  - confidence: 0.86
- **[P2] Apply fallback text before retrying empty completions:** `src/agents/subagent-announce.ts:405-407`
  - body: The new empty-reply guard returns `false` before the existing non-silent `fallbackReply` assignment. For an `ok` completion whose model reply is empty but whose frozen fallback contains the child result, current main would deliver the fallback; this patch instead keeps the child session and delivers nothing.
  - confidence: 0.82
- **[P3] Restore the specific gateway handoff assertion:** `src/agents/subagent-announce-delivery.test.ts:4289-4291`
  - body: This test now only checks that the gateway was called, so wrong `deliver`, channel, target, or delivery-mode parameters would still pass. Keep the specific `expectGatewayAgentParams` assertion and update only the fields that intentionally changed.
  - confidence: 0.78
