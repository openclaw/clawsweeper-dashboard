  - confidence: 0.92

  - body: This branch treats any `pendingToolCalls`/`tool_calls` terminal result as `phase: "error"`, but OpenAI-compatible client tools are a normal boundary: `openai-http` maps the same meta to `finish_reason: "tool_calls"`, and SDK normalization turns lifecycle `phase: "error"` into `run.failed`. This will make successful external tool-call turns look failed to event consumers; only truly unrecoverable unresolved child boundaries should emit lifecycle error.
  - confidence: 0.78
