## Summary

Keep open and block merge: the fresh-install path guard is useful and source-proven, but switching existing explicit `.openclaw` homes to read the parent state root can strand already-written nested config, credentials, sessions, and SQLite state without doctor recovery.

## What This Changes

This PR changes state/config path resolution so explicit `OPENCLAW_HOME` values ending in `.openclaw` or `.clawdbot` are used directly, extracts a leaf state-dir helper, and updates regression and mock tests.

## Best Possible Solution

Land the basename guard together with conflict-safe doctor recovery that migrates safe `$OPENCLAW_HOME/.openclaw` layouts, reports conflicts without overwriting user state, and covers upgrade tests.

Yes. Source inspection of current main and `v2026.6.10` shows explicit `OPENCLAW_HOME=/path/.openclaw` still flows through helpers that append another `.openclaw`, matching the linked issue's reproduction path.

## Solution Assessment
