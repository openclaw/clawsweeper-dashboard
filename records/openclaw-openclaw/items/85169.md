## Summary

Keep open for normal PR handling: the patch is a focused fix for the open Microsoft Teams managed-identity auth bug and I found no blocking code findings, but the auth-path compatibility risk should be maintainer-accepted before merge.

## What This Changes

This PR changes the Microsoft Teams plugin’s federated managed-identity path to use a managed-identity assertion with `ClientAssertionCredential`, adds safer delivery-warning metadata, and covers the paths with focused tests.

## Best Possible Solution

Merge the narrow token-exchange fix once maintainers accept the documented FIC-based managed-identity behavior for Teams, with focused CI plus optional live Azure proof if owners want exact-environment confirmation.

## Reproduction Assessment

Yes, source-reproducible: current main uses `ManagedIdentityCredential` directly for the Teams token provider while the documented managed-identity setup requires an `api://AzureADTokenExchange` FIC. I did not run a live Azure/Teams repro in this read-only review, but the linked issue includes live deployment evidence.

## Solution Assessment

Yes, the PR’s `ClientAssertionCredential` wrapper is the narrowest maintainable fix for the documented FIC flow. A broader `DefaultAzureCredential` redesign or new config mode would be a separate product decision, not necessary for this bug fix.
