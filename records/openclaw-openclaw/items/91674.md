## Summary

The PR fixes a real exec-event delivery bug in the right area, but it is not merge-ready because terminal exec-event runs now set the next scheduled interval slot to the 30s min-spacing floor, causing an unintended scheduled heartbeat soon after the event on long-interval configs.

## What This Changes

The branch changes heartbeat cooldown, wake retry handling, and scheduler tests so exec-event wakes bypass the interval nextDueMs gate and min-spacing skips are retryable.

## Best Possible Solution

Keep the source-aware exec-event nextDueMs bypass, but after terminal exec-event attempts advance nextDueMs by the configured interval, add scheduler coverage for no extra interval wake, and include min-spacing in the exported retryable type.

## Reproduction Assessment

Yes. Current main source shows post-run exec-event wakes hit the nextDueMs not-due gate and the wake layer does not retry not-due; the contributor also supplied live gateway transcript/log proof for the fixed delivery path.

## Solution Assessment

No. The source-aware cooldown bypass is the right layer, but using min-spacing as the next scheduled interval slot is broader than needed and should be replaced with configured-interval advancement after terminal exec-event runs.
