## Summary

Keep open: the patch targets a real source-reproducible stale auto-fallback pin path and is scoped to the right model/session surfaces, but it still needs real behavior proof and a small docs update before merge.

## What This Changes

The PR adds a `modelOverrideExpiresAt` session field, stores 30s expirations on automatic fallback model pins, clears expired pins before model selection, reserves the new slot key, and adds fallback-expiry tests.

## Best Possible Solution

Land a documented fallback-pin expiry that preserves explicit user overrides and legacy sessions, after real setup proof shows an auto fallback pin expiring and the next turn retrying the configured primary/default model.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main writes auto fallback overrides without expiry and resolves stored overrides on later turns, so the stale-pin behavior is source-reproducible.

## Solution Assessment

Mostly yes: persisting expiry with fallback-owned state and clearing it before model selection is the narrow maintainable area. The remaining gaps are documenting the new expiry rule, confirming the 30s policy, and adding real behavior proof.
