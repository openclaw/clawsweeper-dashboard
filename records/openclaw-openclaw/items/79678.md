## Summary

Keep this PR open: the TTL idea is a focused fallback-policy change with real proof, but current main does not already implement it and the branch needs maintainer policy review plus a conflict-aware rebase over newer fallback-origin behavior.

## What This Changes

The PR adds a 30-second expiry for auto-created model fallback session overrides, clears expired auto pins before model selection, updates session typing/reserved keys/tests, and documents the policy.

## Best Possible Solution

Rebase the TTL implementation onto current main, preserve fallback-origin and heartbeat semantics, and land it only if maintainers choose short TTL failback as the desired sticky-fallback policy.

## Reproduction Assessment

Yes. Current main source and docs show auto fallback pins persist until reset with no TTL, and the PR body includes terminal proof exercising the real createModelSelectionState path with an expired auto pin.

## Solution Assessment

Mostly yes, if the policy is accepted. Expiring only auto-owned fallback state before stored override resolution is a narrow direction, but the implementation must be rebased over current fallback-origin metadata before merge.
