## Summary

Keep open for normal PR review: current main still persists automatic fallback model overrides without an expiry, and this PR now adds the documented TTL behavior plus contributor terminal proof. I did not find a blocking correctness or security issue in the diff.

## What This Changes

The PR adds a 30s expiry field for automatic model fallback session pins, clears expired auto pins before model selection, reserves the session key, updates focused tests, and documents the behavior.

## Best Possible Solution

Land a documented, tested automatic fallback-pin expiry once maintainers accept the 30s policy and CI validates the touched agent/session paths.

## Reproduction Assessment

Yes. Current main writes auto fallback overrides without an expiry and later model selection reads stored overrides, so the stale-pin behavior is source-reproducible without mutating the read-only checkout.

## Solution Assessment

Mostly yes. Keeping expiry on auto-owned fallback state and clearing it before model selection is the narrow maintainable path; the remaining judgment is whether 30 seconds is the desired product policy.
