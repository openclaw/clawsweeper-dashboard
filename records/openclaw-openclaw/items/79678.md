## Summary

Keep this PR open for maintainer review. The TTL approach addresses a real sticky fallback symptom and has terminal proof, but it is a user-visible fallback policy choice, conflicts with current main, and needs to be reconciled with newer fallback-origin state plus competing open fix paths.

## What This Changes

The PR adds a 30-second expiry for auto-created model fallback session pins, clears expired pins before model selection, updates session types/tests, and documents the TTL behavior.

## Best Possible Solution

Choose one fallback recovery policy, then rebase the selected implementation onto current main so session override lifetime, fallback-origin metadata, auth-profile preservation, status/docs, and regression tests all describe the same behavior.

## Reproduction Assessment

Yes. Current main source, docs, and tests show auto fallback overrides persist until reset, and the PR body demonstrates the proposed expired-pin path using createModelSelectionState and a temporary session store.

## Solution Assessment

No, not as currently mergeable. The TTL direction is plausible, but the branch must be rebased to preserve current fallback-origin behavior and maintainers still need to choose TTL versus the competing immediate-cleanup policy.
