## Summary

Keep open: current main has timeout-compaction and failed-send idempotency mitigations, but source/tests still show successful replayed `message` sends receive a fresh autogenerated idempotency key, so compaction retry can still duplicate visible channel sends.

## What This Changes

The issue reports compaction timeout recovery wedging sessions and replaying already-sent outbound messages during retry.

## Best Possible Solution

Carry a durable per-logical-message idempotency identity across compaction/timeout retries and outbound queue recovery, with regression coverage for successful tool-send replay rather than only failed-send retry.
