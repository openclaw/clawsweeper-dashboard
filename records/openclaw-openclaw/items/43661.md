## Summary

Keep open: current main and v2026.6.10 still only dedupe visible sends when a stable idempotency key is reused, while compaction timeout progress is covered by an open partial fix and durable Telegram/logical-send replay semantics remain unsettled.

## What This Changes

The issue requests that compaction timeouts stop wedging sessions and stop replaying the same outbound Telegram message repeatedly.

## Best Possible Solution

Land or reject https://github.com/openclaw/openclaw/pull/92899 for the compaction-progress subset, then define a maintainer-approved logical-send replay contract for Telegram/message-tool delivery with focused tests and real transport proof.

## Solution Assessment
