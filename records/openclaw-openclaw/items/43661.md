## Summary

Keep open: current main has partial timeout-compaction and failed-send idempotency mitigations, but successful replayed message-tool sends still intentionally receive fresh autogenerated idempotency keys, so the central duplicate visible-send problem remains.

## What This Changes

The issue asks OpenClaw to prevent compaction-timeout retries from wedging a session and repeatedly sending the same outbound message.

## Best Possible Solution

Carry a durable per-logical-message idempotency identity across compaction and timeout retries, with tests for successful replay and legitimate repeated sends plus real channel proof before close.

## Reproduction Assessment

Yes, source-level. Current main retries after timeout compaction and tests prove successful identical message-tool sends intentionally receive fresh autogenerated keys; no live channel repro was run in this read-only review.

## Solution Assessment

No, current main is only a partial mitigation. The durable fix should define stable logical-send identity across compaction retries rather than only reusing keys for unresolved failed sends.
