## Summary

Keep open: current `main` and `v2026.6.9` still leave `secrets audit` scoped to config/auth/model/`.env` storage and not the generated LaunchAgent service-env file, while the related open PR fixes a different `.env` scrub over-deletion path.

The issue reports that `openclaw secrets audit --json` can report clean after SecretRef migration while OpenClaw-managed macOS LaunchAgent service-env files still contain plaintext migrated credentials.

## Best Possible Solution

Keep this open and first make `secrets audit` truthful about generated service-env plaintext without exposing secret values, then handle service-env repair and OAuth residue cleanup as separate maintainer-approved flows.
