## What This Changes

The PR extends operator-admin `chat.inject` with optional metadata fields, persists and broadcasts them, updates generated Swift models, docs, and focused gateway tests.

## Best Possible Solution

Land the additive metadata seam only after idempotency is enforced inside the serialized append path or an equivalent lock, with concurrent replay coverage, generated protocol/docs updates, and redacted live Gateway proof.

## Reproduction Assessment

## Solution Assessment

No, not yet. The generic metadata seam is a maintainable direction if accepted, but the at-most-once guarantee needs to move into the locked append boundary and the PR still needs real Gateway behavior proof.
