This PR should stay open: current main does not yet expose the requested `chat.inject` metadata, and the PR has useful real Gateway proof, but the latest diff still makes the new idempotency contract non-atomic under concurrent retries.

## What This Changes

The PR adds optional `idempotencyKey`, `command`, `interactive`, and `channelData` fields to `chat.inject`, persists and broadcasts them, updates Swift protocol models, adds focused tests, and documents the contract.

## Best Possible Solution

## Reproduction Assessment

Not applicable as a bug reproduction for the requested API feature. For the PR defect, yes: source inspection gives a high-confidence path where concurrent `chat.inject` calls with the same `idempotencyKey` both pass the pre-lock scan before either append is visible.

## Solution Assessment

No, not yet: the metadata plumbing is the right narrow direction, but idempotency must be enforced inside the locked append path so duplicate retries are at-most-once under concurrency.
