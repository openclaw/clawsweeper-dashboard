## Summary

Keep this PR open: current main still lacks the requested Gateway metadata surface, but the PR branch still has a correctness issue in its new idempotency contract because duplicate calls can race before the transcript append queue/write lock. The change is focused and useful, but it needs the atomic dedupe repair before merge.

## What This Changes

The PR adds optional `idempotencyKey`, `command`, `interactive`, and `channelData` support to Gateway `chat.inject`, persists and broadcasts those fields, updates generated Swift models and docs, and adds focused Gateway tests.

## Best Possible Solution

Keep the generic metadata contract, but enforce idempotency inside the transcript append critical section and add concurrent replay coverage before maintainers decide whether to land the public Gateway API.

## Reproduction Assessment

Not applicable as a bug reproduction for the requested API feature. For the PR defect, yes: source inspection shows duplicate calls can both pass the pre-lock `transcriptHasIdempotencyKey` check before either append is visible.

## Solution Assessment

No, not yet: the metadata plumbing is the right narrow direction, but the idempotency guarantee must live inside the locked transcript append path. The related host API issue should remain the maintainer product/API coordination point.
