## Summary

Keep open: current main does not expose the requested `chat.inject` metadata contract, and the linked host-API issue shows this is a generic core seam needed by an external ClawHub plugin. The PR is not ready to merge because idempotent replay is still not atomic and the supplied proof is only protocol/test output, not real Gateway behavior proof.

## What This Changes

The PR extends operator-admin `chat.inject` schema, handler, transcript append, docs, generated Swift models, and tests to persist idempotency and UI/status metadata for plugin approval cards.

## Best Possible Solution

Land a narrow generic Gateway metadata contract only after idempotency is enforced inside the serialized transcript append path, concurrent replay coverage exists, real Gateway proof is supplied, and the broader approval API shape is accepted.

## Reproduction Assessment

Not applicable as a bug reproduction for the requested API feature. For the PR defect, yes: source inspection shows duplicate detection still occurs before the serialized transcript append path, so concurrent calls can both append.

## Solution Assessment

No, not yet: adding generic `chat.inject` metadata is the right core seam if maintainers accept the API, but the at-most-once guarantee must move into the locked append boundary and the PR still needs real Gateway behavior proof.
