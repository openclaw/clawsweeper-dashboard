## Summary

Keep open. Current main does not expose the requested `chat.inject` metadata, and this PR is a plausible core seam for the related ClawHub AgentKit API gap, but the branch still has a blocking idempotency race and only test/protocol output for real behavior proof.

## What This Changes

The PR adds optional `idempotencyKey`, `command`, `interactive`, and `channelData` fields to operator-admin `chat.inject`, persists and broadcasts them on injected assistant messages, updates Swift protocol/docs, and adds focused gateway tests.

## Best Possible Solution

Keep the generic Gateway metadata seam only if maintainers accept it, make `chat.inject` idempotency atomic at the serialized transcript append boundary, and require live Gateway event/history proof before merge.

## Reproduction Assessment

Not applicable as a user bug reproduction for the requested API feature. For the PR defect, yes: source inspection shows the duplicate scan remains before the serialized transcript append queue, so concurrent duplicate `chat.inject` calls can race.

## Solution Assessment

No, not yet. The metadata plumbing is a plausible core API seam, but the at-most-once guarantee needs to be enforced inside the locked append boundary or an equivalent inflight dedupe before this is the best fix.
