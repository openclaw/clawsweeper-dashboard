## Summary

Keep open: current main still lacks the proposed `chat.inject` metadata contract, and the PR is useful, but its idempotency guarantee is still race-prone because duplicate detection happens before the transcript append lock.

## What This Changes

The PR adds optional `chat.inject` metadata fields, persists and broadcasts them, adds idempotency dedupe, refreshes Swift models, and documents/tests the contract.

## Best Possible Solution

Keep the generic metadata contract, but enforce idempotency inside the locked transcript append path and add concurrent replay coverage before broader approval API review continues in https://github.com/openclaw/openclaw/issues/82336.

## Reproduction Assessment

Not applicable as a bug reproduction for the requested API feature. For the PR defect, yes: source inspection shows concurrent duplicate calls can both pass the pre-lock transcript scan before either append is visible.

## Solution Assessment

No, not yet: the metadata plumbing is the right narrow direction, but the at-most-once guarantee must live inside the transcript append critical section.
