## Summary

Keep this PR open: it covers a missing host API slice and includes real Gateway proof, but the implementation still has a source-provable race in the promised `idempotencyKey` at-most-once behavior.

## What This Changes

The PR adds optional `idempotencyKey`, `command`, `interactive`, and `channelData` to `chat.inject`, persists and broadcasts them, refreshes generated Swift/docs, and adds focused Gateway tests.

## Best Possible Solution

Keep the generic metadata contract, but enforce idempotency inside the locked transcript append path and keep the broader approval surfaces tracked in https://github.com/openclaw/openclaw/issues/82336.

## Reproduction Assessment

Not applicable as a bug reproduction for the requested API feature. For the PR defect, yes: source inspection shows concurrent duplicate calls can both pass the pre-lock transcript scan before either append is visible.

## Solution Assessment

No, not yet: the metadata plumbing is the right narrow direction, but the at-most-once guarantee must live inside the transcript append critical section.
