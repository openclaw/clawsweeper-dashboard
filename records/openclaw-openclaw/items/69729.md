## Summary

Keep this PR open because the catalog update is useful and not present on current main, but the branch is not merge-ready: it reintroduces a runtime suppression hook that current manifest-based model resolution intentionally avoids, and it changes the Global Standard onboarding endpoint to a host OpenClaw does not otherwise advertise.

## Best Possible Solution

Keep the model catalog additions and qwen3.6-plus reasoning correction, but express endpoint-specific suppression through the Qwen plugin manifest, preserve the current Global Standard endpoint host, and update focused Qwen tests and docs.
