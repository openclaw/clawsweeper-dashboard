## Summary

Keep open: the catalog additions are useful and not implemented on current main, but the PR still conflicts with the current manifest-first model-suppression contract and changes the advertised Global Standard endpoint to a host that the provider otherwise does not use.

## What This Changes

The PR adds qwen3.6-flash and qwen3.6-max-preview to the Qwen model catalog, marks qwen3.6-plus as reasoning-capable, exports new IDs, extends Standard-only filtering, and changes one Global/Intl onboarding endpoint note.

## Best Possible Solution

Port the useful catalog additions onto current main with manifest-based qwen/modelstudio suppressions for the new Standard-only IDs, keep the existing Global Standard endpoint host, update focused Qwen tests/docs/changelog, and preserve explicit qwen3.6-plus opt-in behavior.

## Reproduction Assessment

Not applicable as a bug report, but yes for the PR defects: the PR head reintroduces a runtime suppression hook while current main documents and tests manifest-only suppression, and it changes one endpoint string away from current constants/docs.

## Solution Assessment

No. The direction is useful, but the best implementation is a small rebase/repair that adds manifest suppressions and tests instead of restoring runtime suppressBuiltInModel, while leaving the Global/Intl Standard host unchanged.
