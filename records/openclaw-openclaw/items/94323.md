## Best Possible Solution

Keep the add/remove move to maintenance recompute, but call it with recomputeExpired: true and add focused coverage so unexecuted due slots are preserved while already-executed expired slots still advance.

No as submitted. Moving add/remove to maintenance recompute is the right boundary, but those calls should preserve the existing recomputeExpired repair for already-executed expired slots.
