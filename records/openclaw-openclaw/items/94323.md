## Summary

Keep open. Current main still lets cron add() and cold-store remove() reach the full next-run recompute path, while this focused PR moves those mutation paths onto the existing maintenance recompute invariant with sufficient real-service proof and no blocking findings.

## What This Changes

The PR updates cron add/remove to skip full load recompute, run maintenance recompute before persisting, and adds focused add/remove regression tests.

## Best Possible Solution

Land this focused cron ops fix after current-head required checks complete, keeping add/remove on the established maintenance recompute path with the new regression tests.

## Reproduction Assessment

Yes. Source inspection of current main shows add() and cold-store remove() can reach full recompute, whose due-or-missing branch advances present past-due nextRunAtMs; the PR body also supplies before/after real CronService and SQLite output, though I did not run tests in this read-only review.

## Solution Assessment

Yes. Moving only add/remove to skipRecompute plus recomputeNextRunsForMaintenance matches the established read/timer/startup invariant and preserves missing nextRunAtMs backfill; changing recomputeNextRuns globally would be a broader and riskier fix shape.
