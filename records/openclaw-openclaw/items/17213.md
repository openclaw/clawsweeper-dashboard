## Summary

Keep open. Current main has adjacent lifecycle, liveness, timeout, and SDK normalization safeguards, but it still does not expose the requested atomic begin/exec/commit-or-abort run contract with structured abort blockers.

## What This Changes

Request an additive run lifecycle contract that guarantees announced actions reach explicit execution and commit-or-abort status with machine-readable abort details.

## Best Possible Solution

Define an additive protocol-level lifecycle contract that distinguishes intent, execution evidence, committed completion, and aborted/blocker completion, then wire it through event types, Gateway schemas, runtime emitters, wait behavior, SDK/client normalization, session state, docs, and regression tests without breaking existing lifecycle consumers.

## Reproduction Assessment

No for a runtime failure: the issue does not include concrete interruption steps, logs, or a minimal trace. Source and docs do verify that current main exposes only lifecycle start/end/error and agent.wait snapshots, so the requested public contract is absent.

## Solution Assessment

Unclear: the transaction-style lifecycle is a plausible maintainable direction, but the exact protocol states, blocker payload, and rollout plan need maintainer design. Another local heuristic would not solve the cross-surface contract gap.
