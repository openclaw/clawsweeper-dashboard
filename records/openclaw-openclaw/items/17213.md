## Summary

Keep open. Current main and v2026.6.6 have stronger incomplete-turn detection, delivery evidence, and normalized terminal outcomes, but they still do not guarantee that an announced action proceeds to execution and resolves through a typed commit or structured abort. The related implementation proposals were closed unmerged, and the remaining cross-protocol design requires maintainer judgment.

## What This Changes

Require every announced agent action to proceed through execution and resolve with an explicit commit or structured abort.

## Best Possible Solution

Define one additive typed run-intent contract whose mandatory terminal resolution reuses canonical terminal outcomes, carries structured abort blockers, and propagates consistently through gateway events, wait responses, session projection, clients, plugin-visible APIs, documentation, and regression tests.

## Reproduction Assessment

Yes for the contract gap: current source shows generic lifecycle events and downstream terminal normalization without mandatory intent-to-execution binding. The intermittent orphan-notice scenario itself was not reproduced live because the report provides no concrete trace or deterministic interruption point.

## Solution Assessment

No, the standalone transaction vocabulary is not yet the best fully specified implementation. The maintainable direction is an additive extension of the existing canonical terminal-outcome architecture rather than a parallel lifecycle state machine.
