## Summary

Keep open. Current main has adjacent strict-agentic, liveness, and incomplete-turn safeguards, but it still lacks the requested atomic run.begin/run.exec/mandatory run.commit-or-run.abort protocol contract with structured abort blockers.

## What This Changes

The issue requests a transaction-style run lifecycle so announced execution intent always ends in explicit commit or abort state.

## Best Possible Solution

Define an additive Gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, client/plugin-visible surfaces, docs, and regression tests.

## Reproduction Assessment

No high-confidence live reproduction exists for the reported orphan pre-action notice because no trace or step-by-step runtime repro is attached. Source inspection does confirm that the requested transaction contract is absent from current main and the latest release.

## Solution Assessment

Yes to keeping this open for maintainer design; no to closing or queueing a narrow automated patch. The maintainable path is an additive protocol/runtime lifecycle contract rather than another local false-start heuristic.
