## Summary

Keep open. Current main has adjacent strict-agentic and liveness safeguards, but it still lacks the requested public begin/exec/commit-or-abort lifecycle contract with structured abort blockers.

## What This Changes

The issue requests an atomic intent-to-execution run lifecycle so announced actions always end in an explicit commit or abort state.

## Best Possible Solution

Define an additive Gateway lifecycle contract for intent, execution evidence, committed completion, and aborted or blocked completion, then wire it through events, schemas, runtime emitters, wait behavior, clients, docs, and regression tests.

## Reproduction Assessment

No high-confidence runtime reproduction path is available because the issue provides no concrete interruption steps, logs, or trace. Source inspection does confirm that current main lacks the requested public transaction lifecycle contract.

## Solution Assessment

Unclear. The transaction-style lifecycle is plausible, but the wire contract, structured blocker payload, compatibility behavior, and optional completion-blocking policy need maintainer design before implementation.
