## Summary

Keep open. Current main still has narrower strict-agentic, liveness, and item-event safeguards, but the public AgentEvent/Gateway lifecycle remains a generic start/end/error snapshot flow rather than the requested atomic run.begin/run.exec/run.commit-or-run.abort contract with structured abort blockers.

## What This Changes

The issue asks OpenClaw to add an atomic run lifecycle contract so announced action intent must end in explicit execution commit or abort status with structured blockers.

## Best Possible Solution

Define an additive Gateway lifecycle contract for intent, execution evidence, committed completion, and aborted or blocked completion, then wire it through events, schemas, runtime emitters, wait behavior, clients, docs, and regression tests.

## Reproduction Assessment

No for a concrete runtime bug reproduction: the issue has no interruption trace or minimal scenario. Source inspection does provide high-confidence proof that the requested public transaction contract is absent on current main.

## Solution Assessment

Unclear as an implementation plan. A transaction-style lifecycle is plausible, but maintainers still need to choose the protocol shape, abort-blocker payload, compatibility behavior, client/plugin exposure, and optional completion-blocking policy.
