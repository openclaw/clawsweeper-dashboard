## Summary

Keep open. Current main and the latest release still have adjacent liveness, incomplete-turn, abort, and terminal-outcome safeguards, but they do not define the requested public intent-to-execution-to-terminal contract or structured abort blocker.

## What This Changes

The issue requests an atomic run lifecycle so announced action intent always resolves to explicit execution evidence plus commit or abort status.

## Best Possible Solution

Extend the canonical terminal-outcome and gateway lifecycle contract with a stable intent identifier, execution evidence, terminal reason, and structured blocker details projected consistently through events, wait results, clients, and focused interruption tests.
