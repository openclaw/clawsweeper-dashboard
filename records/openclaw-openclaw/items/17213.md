## Summary

Keep open. The issue is a broad core reliability contract request, and current main still exposes legacy/generic lifecycle events and wait snapshots rather than the requested atomic begin/exec/commit-or-abort flow with structured abort blockers; recent discussion also explicitly kept this open for maintainer design.

## What This Changes

The issue requests an enforced transaction-style run lifecycle so pre-action notices cannot be orphaned without a mandatory commit or abort status.

## Best Possible Solution

Define an additive Gateway lifecycle contract for intent, execution evidence, committed completion, and aborted or blocked completion, then wire it through events, schemas, runtime emitters, wait behavior, clients, docs, and regression tests.

## Solution Assessment

Unclear. A transaction-style lifecycle is plausible, but maintainers still need to choose the Gateway wire types, abort-blocker payload, compatibility behavior, and optional completion-blocking policy before implementation.
