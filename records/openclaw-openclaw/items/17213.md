## Summary

Keep open. Current main has narrower strict-agentic, liveness, and incomplete-turn safeguards, but it still does not define the requested atomic run.begin/run.exec/run.commit-or-abort protocol with structured abort blockers across agent events, gateway schema, lifecycle state, and agent.wait.

## What This Changes

Requests an atomic core run lifecycle so announced execution intent always reaches an explicit commit or abort state.

## Best Possible Solution

Define an additive protocol-level lifecycle contract, document it, then wire begin intent, execution evidence, mandatory commit or abort, and structured blockers through events, gateway schemas, runtime emitters, wait behavior, clients, and tests.
