## Summary

Keep open. Current main and v2026.6.5 still have adjacent strict-agentic, liveness, abort, and incomplete-turn safeguards, but not the requested atomic run.begin/run.exec/mandatory run.commit-or-run.abort protocol with structured abort blockers.

## What This Changes

The issue requests a transaction-style agent run lifecycle that prevents user-visible intent notices from being orphaned when execution is interrupted.

## Best Possible Solution

Define an additive core lifecycle contract at the Gateway/runtime boundary, then wire begin intent, execution evidence, mandatory commit or abort, and structured blocker details through AgentEvent types, protocol schemas, runtime emitters, wait results, clients, docs, and regression tests.
