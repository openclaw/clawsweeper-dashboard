## Summary

Keep open. Current main has narrower strict-agentic false-start and liveness safeguards, but it still lacks the requested atomic begin/exec/commit-or-abort run lifecycle with structured abort blockers; the earlier maintainer discussion also kept this broader product contract open after treating the related false-start PR as only partial context.

## What This Changes

Request an atomic intent-to-execution run lifecycle that makes every announced action end in an explicit commit or abort state with machine-readable blockers.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway/protocol boundary, then wire begin intent, execution evidence, mandatory commit or abort terminal state, and structured blocker details through events, runtime emitters, wait results, clients, docs, and regression tests.

## Reproduction Assessment

No. Source inspection confirms the requested contract is absent on current main, but the issue still lacks a concrete runtime trace that reproduces an orphan pre-action notice end to end.

## Solution Assessment

Yes to keeping this open for design, no to a narrow automated patch. The maintainable fix needs an additive protocol/runtime contract and compatibility plan rather than another heuristic false-start guard.
