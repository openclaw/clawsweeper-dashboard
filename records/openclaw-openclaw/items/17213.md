## Summary

Keep open. Current main and the latest release still have adjacent strict-agentic, liveness, and terminal-outcome safeguards, but they do not define the requested atomic intent-to-execution contract with mandatory commit or structured abort; this remains a core protocol/product design item.

## What This Changes

The issue requests an atomic run lifecycle so announced action intent must either execute and commit or abort with a machine-readable blocker.

## Best Possible Solution

Define one additive public run-intent contract that reuses canonical terminal outcomes, carries structured abort blockers, and propagates consistently through gateway events, wait responses, session projection, SDK/plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main has no typed begin/exec/commit/abort lifecycle state or structured abort blocker in the inspected event, protocol, lifecycle, session, SDK, and wait paths. The intermittent runtime orphan-notice symptom was not live-reproduced because the issue does not include a deterministic trace or interruption point.

## Solution Assessment

Unclear. The transaction-style direction fits the reliability goal, but the best implementation needs maintainer design so it extends the existing terminal-outcome architecture instead of adding a parallel lifecycle state machine.
