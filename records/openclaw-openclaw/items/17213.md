## Summary

Keep open. Current main has narrower strict-agentic and liveness safeguards, but it still lacks the requested atomic begin/exec/mandatory commit-or-abort lifecycle contract with structured abort blockers across agent events, Gateway protocol, runtime lifecycle emission, session-state mapping, and `agent.wait`.

## What This Changes

The issue requests an atomic agent run lifecycle so announced execution intent always ends in an explicit commit or abort state with machine-readable blocker details.

## Best Possible Solution

Define an additive Gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, client/plugin-visible surfaces, docs, and regression tests.

## Solution Assessment
