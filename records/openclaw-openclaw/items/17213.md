## Summary

Keep open. Current main has adjacent incomplete-turn and liveness safeguards, but the public agent/Gateway contract still uses legacy lifecycle start/end/error snapshots rather than the requested atomic begin/exec/commit-or-abort transaction with structured abort blockers.

## What This Changes

The issue requests an atomic run lifecycle contract that guarantees each announced action reaches an explicit commit or abort state.

## Best Possible Solution

Define an additive protocol-level lifecycle contract for intent, execution evidence, committed completion, and aborted/blocker completion, then wire it through event types, Gateway schemas, runtime emitters, wait behavior, SDK/client normalization, docs, and regression tests.

## Reproduction Assessment

No high-confidence runtime reproduction path is available because the issue provides no concrete interruption steps, logs, or trace. Source inspection does confirm that the requested public transaction contract is absent on current main.

## Solution Assessment

Unclear. The transaction-style lifecycle is a plausible direction, but the public protocol states, structured blocker payload, compatibility behavior, and optional completion-blocking policy require maintainer design before implementation.
