## Summary

Keep open. Current main still has only adjacent strict-agentic and liveness safeguards; the requested atomic run.begin/run.exec/run.commit-or-run.abort lifecycle with structured abort blockers is not defined in the event, gateway protocol, runtime lifecycle, session state, or agent.wait contract.

## What This Changes

The issue requests a transaction-style run lifecycle so every announced action reaches execution and an explicit commit or abort state.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway/protocol boundary, then wire begin intent, execution evidence, mandatory commit or abort terminal state, and structured blocker details through runtime emitters, wait results, clients, docs, and tests.

## Reproduction Assessment

No. Source inspection confirms the requested contract is absent on current main, but the issue does not provide a high-confidence end-to-end trace that reproduces an orphan pre-action notice.

## Solution Assessment

Yes to keeping this open for design, no to a narrow automated patch. The maintainable solution is an additive protocol/runtime lifecycle contract rather than another heuristic false-start guard.
