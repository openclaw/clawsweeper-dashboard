## Summary

Keep open. Current main has adjacent strict-agentic, liveness, and terminal-state safeguards, but it still does not define the requested atomic run.begin/run.exec/run.commit-or-abort contract with structured abort blockers across the agent event bus, Gateway protocol, runtime lifecycle, session state, and agent.wait.

## What This Changes

The issue requests an atomic run lifecycle contract so action intent notices always terminate in explicit execution commit or abort state.

## Best Possible Solution

Define an additive Gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, clients, plugin-visible surfaces, docs, and regression tests.

## Reproduction Assessment

No high-confidence live reproduction exists for the reported orphan pre-action notice failure because no trace or step-by-step runtime repro is attached. Source inspection does confirm the requested transaction contract is absent on current main.

## Solution Assessment

Yes to keeping this open for maintainer design; no to closing or queueing a narrow automated patch. The maintainable solution is an additive protocol/runtime lifecycle contract rather than another local heuristic around false starts.
