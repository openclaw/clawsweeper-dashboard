## Summary

Keep open. Current main and v2026.5.22 have adjacent strict-agentic, liveness, and lifecycle safeguards, but they still do not provide the requested typed run.begin/run.exec/mandatory run.commit-or-run.abort contract with structured abort blockers.

## What This Changes

The issue asks for an atomic intent-to-execution lifecycle so announced actions always finish with an explicit commit or abort state.

## Best Possible Solution

Define an additive Gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, client/plugin-visible surfaces, docs, and regression tests.
