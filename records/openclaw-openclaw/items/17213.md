## Summary

Keep open. Current main and v2026.5.12 still expose only generic lifecycle events plus adjacent strict-agentic/liveness safeguards, while this issue asks for a new atomic transaction contract across protocol, runtime lifecycle, session state, and wait surfaces; prior discussion and related items treat existing fixes as partial context, not closure evidence.

## What This Changes

Request a transaction-style run lifecycle so announced action intent must lead to execution evidence and a mandatory commit or abort with machine-readable blocker details.

## Best Possible Solution

Define an additive gateway/runtime lifecycle contract, then wire begin intent, execution evidence, mandatory commit or abort state, and structured blockers through events, protocol schemas, wait results, clients, plugin-visible surfaces, docs, and regression tests.

## Reproduction Assessment

No for the reported orphan pre-action notice failure. Source inspection confirms the requested contract is absent on current main, but the issue does not provide a high-confidence runtime trace that reproduces an orphan notice end to end.

## Solution Assessment

Yes to keeping this open for maintainer design; no to a narrow automated patch. The maintainable path is an additive protocol/runtime lifecycle contract rather than another local false-start heuristic.
