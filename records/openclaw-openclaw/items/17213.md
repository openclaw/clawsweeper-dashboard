## Summary

Keep open. Current main has adjacent strict-agentic, incomplete-turn, liveness, and terminal-outcome safeguards, but it still does not define the requested public atomic begin/exec/commit-or-abort contract with structured abort blockers across agent events, gateway schema, runtime lifecycle emitters, session projection, docs, and agent.wait.

## What This Changes

The issue requests an atomic agent run lifecycle contract that guarantees announced action intent reaches execution and a mandatory commit or abort state.

## Best Possible Solution

Define an additive public run lifecycle contract that reuses the existing terminal-outcome normalization, carries structured abort blockers, and propagates through gateway events, wait responses, session projection, SDK/plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main still lacks typed transaction lifecycle phases in the event, protocol, runtime, session, and wait paths. The intermittent orphan-notice runtime symptom itself was not live-reproduced from a deterministic trace.

## Solution Assessment

Unclear. The transaction-style direction fits the reliability goal, but the best implementation needs maintainer design so it extends the current terminal-outcome architecture instead of adding a parallel lifecycle state machine or premature policy knob.
