## Summary

Keep open. Current main has narrower strict-agentic and liveness safeguards, and related reports were closed by narrower fixes, but the core AgentEvent/Gateway lifecycle still lacks the requested atomic begin/exec/commit-or-abort contract with structured abort blockers.

## What This Changes

The issue requests an atomic agent run lifecycle where announced intent must be followed by execution evidence and a mandatory commit or abort terminal state with structured blocker details.

## Best Possible Solution

Define an additive Gateway lifecycle contract for intent, execution evidence, committed completion, and aborted or blocked completion, then wire it through events, schemas, runtime emitters, wait behavior, clients, docs, and regression tests.

## Solution Assessment

Unclear. A transaction-style lifecycle is plausible, but maintainers still need to choose the Gateway wire types, abort-blocker payload, compatibility behavior, client/plugin exposure, and any completion-blocking policy before implementation.
