## Summary

Keep open. Current main has adjacent strict-agentic and liveness safeguards, but it still lacks the requested atomic run.begin/run.exec/run.commit-or-abort contract with structured abort blockers across events, gateway protocol, runtime lifecycle, session state, and agent.wait; the live discussion already routes this as maintainer product-contract work.

## What This Changes

The issue requests a transaction-style run lifecycle so announced action intent must end in explicit execution commit or abort status.

## Best Possible Solution

Define an additive gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, clients, plugin-visible surfaces, docs, and regression tests.

## Reproduction Assessment

No high-confidence live reproduction exists for the reported orphan pre-action notice failure because there is no trace or step-by-step runtime repro. Source inspection does confirm the requested transaction contract is absent on current main.

## Solution Assessment

Yes to keeping this open for maintainer design; no to closing or queueing a narrow automated patch. The maintainable solution is an additive protocol/runtime lifecycle contract rather than another local heuristic around false starts.
