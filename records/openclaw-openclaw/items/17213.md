## Summary

Keep open. Current main still has adjacent strict-agentic, incomplete-turn, liveness, restart, and terminal-outcome safeguards, but it does not define the requested public atomic begin/exec/commit-or-abort lifecycle with structured abort blockers across agent events, gateway protocol, runtime emitters, session projection, docs, and agent.wait.

## What This Changes

The issue requests an atomic run lifecycle so announced actions always end with explicit execution, commit, or abort state instead of orphan pre-action notices.

## Best Possible Solution

Define an additive public run lifecycle contract that extends the existing terminal-outcome architecture, carries structured abort/blocker details, and propagates through gateway events, wait responses, session projection, SDK/plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main still lacks typed transaction lifecycle phases in the event, protocol, runtime, session, docs, and wait paths. The intermittent orphan-notice runtime symptom itself was not live-reproduced from a deterministic trace.

## Solution Assessment

Unclear. The transaction-style direction fits the reliability goal, but the best implementation needs maintainer design so it extends current terminal-outcome handling instead of adding a parallel lifecycle state machine or premature policy knob.
