## Summary

Keep open. Current main has shipped narrower strict-agentic false-start safeguards, but it still does not define the requested atomic begin/exec/mandatory commit-or-abort lifecycle contract with structured abort blockers across event, protocol, runtime, session-state, and wait surfaces.

## What This Changes

The issue requests a transaction-style run lifecycle so user-visible intent/pre-action notices always reach explicit execution, commit, or abort status.

## Best Possible Solution

Define an additive Gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, client/plugin-visible surfaces, docs, and regression tests.

## Reproduction Assessment

No high-confidence live reproduction exists for the specific orphan pre-action notice failure because no trace or step-by-step runtime repro is attached. Source inspection does confirm that the requested transaction contract is absent on current main.

## Solution Assessment

Yes to keeping this open for maintainer design; no to closing or queueing a narrow automated patch. The maintainable path is an additive protocol/runtime lifecycle contract rather than another local false-start heuristic.
