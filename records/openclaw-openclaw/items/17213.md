## Summary

Keep open. Current main has strict-agentic and liveness safeguards, but it still does not define the requested atomic run.begin/run.exec/run.commit-or-abort contract with structured abort blockers across events, gateway protocol, runtime lifecycle, session state, or agent.wait; the live issue discussion also already routes this as maintainer product-contract work.

## What This Changes

Request a transaction-style run lifecycle so announced action intent always reaches an explicit commit or abort state instead of leaving orphan pre-action notices.

## Best Possible Solution

Define an additive gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through events, protocol schemas, wait results, clients, plugin-visible surfaces, docs, and regression tests.

## Reproduction Assessment

No for the reported orphan pre-action notice failure: there is no high-confidence live trace or step-by-step runtime repro. Source inspection does confirm the requested transaction contract is absent on current main.

## Solution Assessment

Yes to keeping this open for maintainer design; no to a narrow automated patch. The maintainable solution is an additive protocol/runtime lifecycle contract, not another local false-start heuristic.
