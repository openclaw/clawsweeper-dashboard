Keep open. Current main and the latest release still expose generic lifecycle start/end/error events and wait snapshots, while the requested atomic run.begin/run.exec/mandatory run.commit-or-run.abort contract with structured abort blockers remains absent; prior maintainer-side discussion also kept this as a product-contract item.

## What This Changes

Request an atomic run lifecycle contract so announced execution intent must resolve to explicit execution, commit, or abort status with machine-readable blockers.

## Best Possible Solution

Define an additive Gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, client/plugin-visible surfaces, docs, and regression tests.
