## Summary

Keep open. Current main has adjacent strict-agentic, lifecycle, liveness, and terminal-outcome safeguards, but it still does not define the requested public atomic begin/exec/commit-or-abort contract with structured abort blockers across agent events, gateway schema, runtime emitters, session projection, docs, and agent.wait.

## What This Changes

The issue requests an atomic run lifecycle where an announced execution intent must resolve to a machine-readable execution, commit, or abort state instead of leaving orphan pre-action notices.

## Best Possible Solution

Define an additive public run lifecycle contract that reuses canonical terminal outcomes, carries structured abort blockers, and propagates through gateway events, wait responses, session projection, SDK/plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main still lacks typed transaction lifecycle phases in the event, protocol, runtime, session, docs, and wait paths. The intermittent orphan-notice runtime symptom itself was not live-reproduced from a deterministic trace.

## Solution Assessment

Unclear. The transaction-style direction fits the reliability goal, but the best implementation needs maintainer design so it extends the current terminal-outcome architecture instead of adding a parallel lifecycle state machine.
