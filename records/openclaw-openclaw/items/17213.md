## Summary

Keep open. Current main still has legacy lifecycle, liveness, yield, replay-invalid, and wait-snapshot safeguards, but no public atomic run.begin to run.exec to mandatory run.commit/run.abort contract with structured abort blockers; the prior maintainer discussion also explicitly kept this as open design work.

## What This Changes

Request an atomic intent-to-execution run lifecycle so announced actions always end in explicit commit or abort status.

## Best Possible Solution

Define an additive protocol-level lifecycle contract for intent, execution evidence, committed completion, and aborted/blocker completion, then wire it through event types, Gateway schemas, runtime emitters, wait behavior, SDK/client normalization, docs, and regression tests.

## Reproduction Assessment

No high-confidence runtime reproduction path is provided: the issue has no concrete interruption steps, logs, or trace. Source inspection does confirm that the requested public transaction contract is absent on current main.

## Solution Assessment

Unclear: the proposed transaction lifecycle is plausible, but the exact states, blocker schema, and backward-compatible rollout require maintainer design. A local heuristic like the closed #40710 execution gate would not cover the cross-surface contract gap.
