## Summary

Keep open. Current main still has adjacent liveness, incomplete-turn, wait-snapshot, and SDK normalization safeguards, but it does not define the requested public atomic run.begin to run.exec to mandatory run.commit/run.abort lifecycle contract with structured abort blockers; recent maintainer discussion also explicitly kept this as design work.

## What This Changes

Request a transaction-style run lifecycle so announced execution intent always reaches an explicit commit or abort outcome.

## Best Possible Solution

Define an additive protocol-level lifecycle contract for intent, execution evidence, committed completion, and aborted/blocker completion, then wire it through event types, Gateway schemas, runtime emitters, wait behavior, SDK/client normalization, docs, and regression tests.

## Reproduction Assessment

No high-confidence runtime reproduction path is provided: the issue has no concrete interruption steps, logs, or trace. Source inspection does confirm that the requested public transaction contract is absent on current main.

## Solution Assessment

Unclear: the proposed transaction lifecycle is plausible, but the exact states, blocker payload, and compatibility semantics need maintainer design. A local heuristic like the closed #40710 execution gate would not cover the cross-surface contract gap.
