## Summary

Keep open. Current main and the latest release still have adjacent strict-agentic, liveness, and abandoned-turn safeguards, but they do not define the requested atomic begin/exec/commit-or-abort lifecycle contract with structured abort blockers.

## What This Changes

The issue requests an atomic run lifecycle contract so announced action intent cannot be left without execution evidence or an explicit commit/abort outcome.

## Best Possible Solution

Define an additive public lifecycle contract that reuses the existing terminal-outcome model while carrying intent, execution evidence, mandatory commit or structured abort, and client/plugin migration guidance.

## Reproduction Assessment

Yes for the source-level contract gap: current main has no typed begin/exec/commit/abort lifecycle state or structured abort blocker in the inspected event, protocol, lifecycle, session, and wait paths. The intermittent orphan-notice runtime symptom was not live-reproduced because no deterministic interruption trace is attached.

## Solution Assessment

Unclear. The transaction-style direction fits the reliability problem, but the best implementation needs maintainer design so it extends the existing terminal-outcome architecture instead of adding a competing lifecycle state machine.
