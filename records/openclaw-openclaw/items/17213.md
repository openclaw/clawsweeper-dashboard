## Summary

Keep open. Current main and v2026.6.6 still have adjacent false-start, liveness, and terminal-outcome safeguards, but they do not define the requested atomic intent-to-execution contract with mandatory commit or structured abort; this remains a core protocol/product design item rather than conservative cleanup.

## What This Changes

Request to guarantee that announced agent execution intent is followed by execution evidence and a mandatory commit or structured abort state.

## Best Possible Solution

Define one additive public run-intent contract that reuses canonical terminal outcomes, carries structured abort blockers, and propagates consistently through gateway events, wait responses, session projection, plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main and v2026.6.6 have no typed begin/exec/commit/abort lifecycle state or structured abort blocker in the inspected agent event, gateway schema, lifecycle emitter, session projection, and wait paths. The intermittent orphan-notice runtime symptom itself was not live-reproduced because no deterministic trace or interruption point is attached.

## Solution Assessment

No, the request is not yet a fully specified best implementation. The safer direction is an additive extension of the existing terminal-outcome architecture, not a parallel lifecycle state machine.
