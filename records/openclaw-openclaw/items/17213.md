## Summary

Keep open. Current main and the latest release still have adjacent liveness, incomplete-turn, and terminal-outcome safeguards, but they do not provide the requested atomic begin/exec/commit-or-abort lifecycle contract with structured abort blockers; this remains a core protocol/product design item.

## What This Changes

The issue requests a transaction-style run lifecycle so announced execution intent always ends in explicit commit or abort state.

## Best Possible Solution

Define one additive public run-intent contract that reuses canonical terminal outcomes, carries structured abort blockers, and propagates consistently through gateway events, wait responses, session projection, plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main and v2026.6.8 have no typed begin/exec/commit/abort lifecycle state or structured abort blocker in the inspected event, protocol, lifecycle, session projection, and wait paths. The intermittent orphan-notice runtime symptom itself was not live-reproduced because no deterministic trace or interruption point is attached.

## Solution Assessment

Unclear as an implementation proposal: the requested transaction shape is directionally valid, but the exact protocol and compatibility shape is not specified. The safer solution is an additive extension of the existing terminal-outcome architecture, not a parallel lifecycle state machine.
