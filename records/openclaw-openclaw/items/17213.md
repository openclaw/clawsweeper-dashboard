## Summary

Keep open. Current main and the latest release still have adjacent strict-agentic and liveness safeguards, but they do not define the requested atomic run.begin/run.exec/run.commit-or-run.abort contract with structured abort blockers across events, gateway protocol, runtime lifecycle, session state, and agent.wait.

## What This Changes

Request an atomic run lifecycle so announced action intent must be followed by execution evidence and an explicit commit or abort state.

## Best Possible Solution

Define an additive gateway/runtime lifecycle contract, then wire begin intent, execution evidence, mandatory commit or abort state, and structured blockers through events, protocol schemas, wait results, clients, plugin-visible surfaces, docs, and regression tests.

## Reproduction Assessment

No. Source inspection confirms the requested contract is absent on current main, but the issue does not provide a high-confidence runtime trace that reproduces an orphan pre-action notice.

## Solution Assessment

Yes to keeping this open for maintainer design; no to a narrow automated patch. The maintainable solution is an additive protocol/runtime contract rather than another local false-start heuristic.
