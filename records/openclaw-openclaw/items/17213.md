## Summary

Keep open. Current main and the latest release still use generic lifecycle start/end/error events plus wait snapshots, while the requested atomic run.begin/run.exec/mandatory run.commit-or-run.abort contract with structured abort blockers remains absent.

## What This Changes

The issue asks OpenClaw to guarantee that announced action intent is followed by execution evidence and an explicit commit or abort terminal state.

## Best Possible Solution

Define an additive Gateway/runtime lifecycle contract that carries begin intent, execution evidence, mandatory commit or abort, and structured blockers through protocol schemas, AgentEvent types, runtime emitters, wait results, client/plugin-visible surfaces, docs, and regression tests.

## Solution Assessment
