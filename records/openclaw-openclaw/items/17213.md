## Summary

Keep open. Current main and latest release still have adjacent lifecycle, liveness, incomplete-turn, and terminal-outcome safeguards, but they do not define the requested atomic begin/exec/commit-or-abort contract with structured abort blockers.

## What This Changes

The issue requests a transaction-style agent run lifecycle so every announced action has execution evidence and a mandatory commit or structured abort state.

## Best Possible Solution

Define one additive public run-intent contract that reuses canonical terminal outcomes, carries structured abort blockers, and propagates consistently through gateway events, wait responses, session projection, SDK/plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main has no typed begin/exec/commit/abort lifecycle state or structured abort blocker in the inspected event, protocol, lifecycle, session, and wait paths. The intermittent runtime orphan-notice symptom was not live-reproduced because the issue lacks a deterministic trace or interruption point.

## Solution Assessment

Unclear. The transaction-style direction fits the reliability goal, but the best implementation needs maintainer design so it extends the existing terminal-outcome architecture instead of adding a parallel lifecycle state machine.
