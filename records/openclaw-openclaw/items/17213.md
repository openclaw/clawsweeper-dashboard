## Summary

Keep open. Current main has meaningful adjacent safeguards for strict-agentic false starts, incomplete tool-use turns, liveness, and `agent.wait`, but it still lacks the requested public begin/exec/commit-or-abort lifecycle contract with structured abort blockers.

## What This Changes

Request an atomic run lifecycle contract so announced execution intent always resolves to explicit execution, commit, or abort status.

## Best Possible Solution

Define an additive Gateway lifecycle contract for intent, execution evidence, committed completion, and aborted or blocked completion, then wire it through events, schemas, runtime emitters, wait behavior, clients, docs, and regression tests.
