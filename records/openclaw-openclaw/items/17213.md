## Summary

Keep open. Current main and v2026.6.8 still have adjacent strict-agentic, liveness, abort, and terminal-outcome safeguards, but they do not define the requested atomic intent-to-execution contract with mandatory commit or structured abort; this remains a core protocol/product design item.

## What This Changes

The issue asks for an atomic run lifecycle where announced action intent must be followed by execution and a mandatory commit or abort state.

## Best Possible Solution

Define one additive core run-intent contract that reuses canonical terminal outcomes, carries structured abort blockers, and propagates consistently through gateway events, wait responses, session projection, SDK/plugin-visible APIs, docs, and regression tests.
