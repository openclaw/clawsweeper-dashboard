## Summary

Keep open. Current main still has adjacent safeguards for planning-only turns, incomplete turns, liveness, abort snapshots, and wait behavior, but it does not define the requested public atomic run lifecycle contract with explicit intent, execution, and mandatory commit-or-abort terminal states plus structured abort blockers.

## What This Changes

The issue requests a transaction-style agent run lifecycle so announced pre-action intent always proceeds to execution evidence and then an explicit commit or abort state.

## Best Possible Solution

Define an additive protocol-level lifecycle contract that distinguishes intent, execution evidence, committed completion, and aborted/blocker completion, then wire it through AgentEvent types, Gateway schemas, embedded runtime emitters, agent.wait, SDK/client normalization, session state, docs, and regression tests without breaking existing lifecycle consumers.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No concrete runtime interruption reproduction is supplied, but current-main source, schema, docs, and tests provide high-confidence verification that the requested lifecycle contract is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers settle the public contract; the maintainable direction appears to be an additive lifecycle protocol rather than another local heuristic, but the exact states and rollout need human review.
