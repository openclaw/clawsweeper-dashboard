## Summary

Keep open. Current main and the latest release still have adjacent lifecycle, liveness, and terminal-outcome safeguards, but they do not define the requested atomic begin/exec/commit-or-abort contract with structured abort blockers.

## What This Changes

The issue requests a transaction-style run lifecycle so announced execution intent always resolves to explicit execution plus commit or abort status.

## Best Possible Solution

Define an additive public run lifecycle contract that reuses canonical terminal outcomes, carries structured abort blockers, and propagates through gateway events, wait responses, session projection, SDK/plugin-visible APIs, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level contract gap: current main has no typed begin/exec/commit/abort lifecycle state or structured abort blocker in the inspected event, protocol, lifecycle, session, and wait paths. The intermittent orphan-notice runtime symptom was not live-reproduced because the issue lacks a deterministic interruption trace.

## Solution Assessment

Unclear. The transaction-style direction fits the reliability goal, but the best implementation needs maintainer design so it extends existing terminal-outcome architecture instead of adding a parallel lifecycle state machine.
