## Summary

Keep open. Current main and the latest release still leave `waitForWaConnection` unbounded, and this PR is directionally useful but incomplete: controller startup remains unbounded, monitor timeout failure can orphan the socket, and the external contributor has not provided real WhatsApp runtime proof.

## What This Changes

The PR adds an optional timeout parameter to WhatsApp `waitForWaConnection`, applies a 60s timeout in the inbound monitor, and adds unit tests for timeout cleanup.

## Best Possible Solution

Add a bounded connection wait for reconnectable startup paths, keep QR login intentionally unbounded or governed by its login TTL, and ensure every timeout path closes the socket before retrying.

## Reproduction Assessment

Yes, source-level. An EventEmitter-backed socket that never emits `connection.update` with `open` or `close` leaves `waitForWaConnection` pending, and current controller and monitor startup paths await that helper.

## Solution Assessment

No. The helper timeout and tests are useful, but the patch needs to cover controller startup and close sockets abandoned by a monitor timeout before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the timeout through controller startup:** `extensions/whatsapp/src/session.ts:194`
  - body: Because `timeoutMs` now defaults to no deadline, production callers that should fail fast must opt in. `WhatsAppConnectionController.openConnection` still awaits `waitForWaConnection(sock)` before listener and watchdog ownership, so gateway startup can still hang indefinitely in the same no-terminal-event state.
  - confidence: 0.91
- **[P2] Close sockets on monitor timeout failure:** `extensions/whatsapp/src/inbound/monitor.ts:53`
  - body: If `waitForWaConnection(sock, 60_000)` rejects here, `monitorWebInbox` exits before `attachWebInboxToSocket` returns the listener whose close method owns socket teardown. Close the created socket in this failure path before rethrowing to avoid orphaned Baileys sockets across retries.
  - confidence: 0.9
