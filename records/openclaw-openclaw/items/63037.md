## Summary

Keep open. Current main still has the unbounded WhatsApp connection wait, and this PR is useful, but the branch remains partial because controller startup is still unbounded, monitor timeout failure can leave the new socket without an owner, and the PR body still lacks real WhatsApp runtime proof.

## What This Changes

The PR adds an optional timeout to `waitForWaConnection`, opts `monitorWebInbox` into a 60s wait, and adds unit tests for timeout cleanup.

## Best Possible Solution

Add a bounded connection wait for reconnectable startup paths, preserve QR login’s longer scan window, and ensure every timeout path closes the socket it created before retrying or failing.

## Reproduction Assessment

Yes, source-level. An EventEmitter-backed socket that never emits `connection.update` with `open` or `close` leaves current main’s helper pending; I did not run a live WhatsApp account reproduction.

## Solution Assessment

No. The helper timeout and unit tests are useful, but the patch must opt in controller startup and close abandoned monitor sockets before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the timeout through controller startup:** `extensions/whatsapp/src/session.ts:194`
  - body: Because the helper now defaults to `0`, `WhatsAppConnectionController.openConnection` still awaits `waitForWaConnection(sock)` before listener and watchdog ownership. A socket that never emits a terminal connection update can still hang gateway startup through that path, so the reconnectable controller wait needs to opt into the bounded timeout too.
  - confidence: 0.91
- **[P2] Close the socket when the monitor wait times out:** `extensions/whatsapp/src/inbound/monitor.ts:53`
  - body: If `waitForWaConnection(sock, 60_000)` rejects here, `monitorWebInbox` exits before `attachWebInboxToSocket` returns the listener that owns teardown. Close the created socket before rethrowing so repeated timeouts do not accumulate orphaned Baileys sockets across retries.
  - confidence: 0.9
