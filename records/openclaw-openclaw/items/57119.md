## Summary

Keep open. Current main still treats `enabled:false` as skill eligibility and prompt filtering, while file tools, exec, and sandbox skill sync do not share a centralized disabled-skill realpath deny policy; the remaining choice is a security/product contract decision.

## What This Changes

Request to make disabled skills inaccessible to agent file and exec tools, not only hidden from skill eligibility.

## Best Possible Solution

Define the disabled-skill contract explicitly; if it is access control, enforce one centralized realpath-aware disabled-skill policy across skill sync, file tools, exec, sandbox/remote paths, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. Current main filters disabled skills out of eligibility and prompts, but does not carry disabled skill realpaths into file-tool, sandbox-sync, or exec deny policy.

## Solution Assessment

Unclear. The requested enforcement is plausible, but maintainers first need to decide whether `enabled:false` is an access-control contract or a visibility switch with separate sandbox/tool controls.
