## Summary

Keep this PR open: the server-side Codex resume approval gate is the right narrow fix and the supplied logs are convincing, but the patch still leaves the Web Control UI offering a disallowed `allow-always` decision for restricted plugin approvals.

## What This Changes

The PR gates Codex CLI session resume behind plugin approval, restricts resume approvals to `allow-once` or `deny`, forwards `allowedDecisions` through the gateway node-invoke policy, and adds focused regression tests.

## Best Possible Solution

Carry plugin approval `allowedDecisions` through the Control UI model and render only permitted actions, then land the Codex resume gate with gateway, Codex, and UI regression coverage.

## Reproduction Assessment

Yes, as a source-level reproduction: current main dispatches `codex.cli.session.resume` directly to the node, and the node handler can run `codex exec resume` after validating the request. I did not run a live current-main repro in this read-only review.

## Solution Assessment

No, not yet. Gating resume in the plugin node-invoke policy is the narrow server-side fix, but the restricted decision contract must also reach Control UI before this is merge-ready.
