## Summary

Keep open: current main still sends the dangerous Codex CLI resume node command straight to node dispatch, while this PR adds a focused approval gate and UI decision filtering with no blocking code-review findings. The remaining blocker is maintainer acceptance of the intentional fail-closed compatibility and availability impact for approval-less automation.

## What This Changes

The PR gates `codex.cli.session.resume` behind restricted plugin approval, propagates allowed approval decisions through gateway/UI handling, and adds focused regression tests.

## Best Possible Solution

Land the approval gate after maintainer acceptance of fail-closed resume behavior, with release-note treatment for unattended callers that now need an approval runtime or explicit operator decision.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility impact. The PR uses the existing plugin approval contract, reuses the gateway decision validator, and keeps the resume restriction local to the Codex node-invoke policy plus approval UI propagation.
