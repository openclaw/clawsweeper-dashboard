## Summary

Keep this PR open: current main and the latest release still forward `codex.cli.session.resume` directly to the node, while the PR contains unique approval-gating work. The patch looks directionally correct, but it still needs maintainer acceptance of the fail-closed compatibility and availability impact before merge.

## What This Changes

The PR adds a restricted plugin approval gate before Codex CLI session resume node dispatch and propagates allowed approval decisions through the gateway node-invoke policy and Control UI approval handling.

## Best Possible Solution

Land the approval gate only after maintainers accept the fail-closed shipped-behavior impact, refresh the branch against current main, and carry release-note/operator guidance for unattended resume callers.

## Reproduction Assessment

## Solution Assessment

Yes for the code boundary: the PR uses the existing plugin approval mechanism at the gateway-side node-invoke policy boundary and keeps Codex CLI spawn/listing behavior out of scope. The merge decision remains a maintainer call because the fail-closed behavior changes shipped unattended resume workflows.
