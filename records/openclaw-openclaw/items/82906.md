## Summary

Keep open: the server-side Codex resume approval gate is valuable and the PR supplies real Gateway/node proof, but the latest diff still makes restricted plugin approvals visible to the Control UI without teaching that UI to hide unavailable decisions.

## What This Changes

The PR gates Codex CLI session resume behind plugin approval, restricts resume decisions to allow-once or deny, forwards allowed decisions through the gateway node-invoke approval runtime, and adds focused regression tests.

## Best Possible Solution

Carry plugin approval allowedDecisions through the Control UI model and render only permitted actions, then land the Codex resume gate with targeted gateway, Codex, and UI regression coverage.

## Reproduction Assessment

Yes, as a source-level reproduction: current main dispatches codex.cli.session.resume directly to the node, and that node handler can run codex exec resume. I did not run a live current-main repro in this read-only review.

## Solution Assessment

No, not yet. Gating resume in the plugin node-invoke policy is the right narrow server-side fix, but the restricted decision contract also needs to be reflected in the Control UI before this is merge-ready.
