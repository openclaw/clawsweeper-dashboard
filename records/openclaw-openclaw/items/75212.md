## Summary

Keep open: the underlying local-model prompt-pressure problem is still valid, but this branch is not merge-ready because minimal tiering changes delegated/cron bootstrap semantics and hook-added bootstrap files can bypass the new tier policy.

## What This Changes

The PR adds an opt-in `agents.defaults.bootstrapTier: "minimal"`, doctor guidance for large loopback-local bootstrap context, schema metadata, docs, changelog, and targeted tests.

## Best Possible Solution

Land one coherent bootstrap-tier design that keeps `standard` backward compatible, makes `minimal` session-aware and hook-aware, and aligns doctor guidance, schema metadata, docs, changelog, and tests with the chosen #75189/#22439 direction.

## Reproduction Assessment

Yes. Static inspection of PR head `0390b6b86d4666665adc2c569d9ef5a9ce43c80e` reproduces both blockers: delegated/cron sessions hit the minimal branch before the session check, and hook-added extras are appended after the initial tier filter.

## Solution Assessment

No. The two-tier config knob and doctor warning are plausible, but this implementation should preserve existing delegated/background filtering and thread or reapply tier policy after bootstrap hooks before it can be considered the best fix.
