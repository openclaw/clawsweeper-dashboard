## Summary

Keep open. Current main still lacks the proposed `agents.defaults.bootstrapTier` and loopback local-model doctor warning, but the PR is not merge-ready because the minimal tier currently changes delegated/background bootstrap semantics and does not cover the bundled extra-bootstrap hook path; it also overlaps the still-open #22439 public config/API direction.

## What This Changes

The PR adds `agents.defaults.bootstrapTier` with `standard` and `minimal`, applies minimal bootstrap filtering, adds a loopback local-model bootstrap-size doctor warning, and updates config metadata, docs, changelog, and targeted tests.

## Best Possible Solution

Land one coherent, session-aware bootstrap-tier design: keep `standard` backward compatible, make `minimal` reduce ordinary bootstrap predictably, preserve or explicitly document delegated/background context behavior, thread tier semantics through bootstrap hooks, and align doctor guidance, docs, schema metadata, changelog, and tests with the chosen #22439/#75189 direction.

## Reproduction Assessment

Yes. Static inspection of the PR patch is enough to reproduce the review blockers: a subagent call with `bootstrapTier: "minimal"` hits the new minimal branch before the subagent/cron filter, and hook-added files still re-enter through the bundled hook without the tier argument.

## Solution Assessment

No. The config knob plus doctor warning are plausible, but this implementation is not the best fix until tier filtering is session-aware and hook-aware, and maintainers resolve whether this narrower API or the overlapping #22439 tier model should be the public contract.
