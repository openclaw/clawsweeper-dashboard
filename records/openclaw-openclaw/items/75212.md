## Summary

Keep open. Current main still has no `agents.defaults.bootstrapTier`, and the PR branch has two merge-blocking prompt-context regressions: minimal mode overrides the existing delegated/cron session allowlist, and hook-added bootstrap files can bypass the new tier filter.

## What This Changes

The PR adds `agents.defaults.bootstrapTier` with `standard` and `minimal`, wires it into bootstrap file selection, adds a loopback local-model doctor warning, and updates config docs, generated schema/help, changelog, and tests.

## Best Possible Solution

Land a single session-aware and hook-aware bootstrap-tier design: keep `standard` compatible by default, make `minimal` reduce ordinary-session bootstrap predictably, preserve or explicitly document delegated/background behavior, and reconcile the public config shape with #22439 before release.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: `filterBootstrapFilesForSession(mockFiles, "agent:default:subagent:task-1", "minimal")` returns the new minimal allowlist instead of the delegated allowlist, and the bundled hook still calls the filter without the tier.

## Solution Assessment

No. The direction is plausible, but the current implementation is not the best fix until the tier is threaded through hook mutation paths and the session-specific bootstrap contract is preserved or deliberately redesigned.
