## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks `agents.defaults.bootstrapTier` and the loopback-local bootstrap warning, while the PR has two concrete prompt-context correctness blockers and overlaps the broader open #22439 API direction.

## What This Changes

The PR adds an opt-in `agents.defaults.bootstrapTier: "minimal"`, a doctor warning for large loopback-local bootstrap context, generated config metadata, docs, changelog, and targeted tests.

## Best Possible Solution

Land one coherent bootstrap-tier design that keeps `standard` backward compatible, makes `minimal` session-aware and hook-aware, and aligns doctor guidance, schema metadata, docs, changelog, and tests with the chosen #75189/#22439 direction.

## Reproduction Assessment

Yes. Static inspection of the PR head reproduces both blockers: a subagent or cron session with `bootstrapTier: "minimal"` hits the new minimal branch first, and hook-added bootstrap files are appended after the initial tier filter without the tier argument.

## Solution Assessment

No. The config knob and doctor warning are plausible, but this implementation is not the best fix until tier filtering preserves delegated/background semantics and the hook path re-applies the same tier policy.
