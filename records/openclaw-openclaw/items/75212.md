## Summary

Keep open. Current main does not implement `agents.defaults.bootstrapTier`, and this PR is a plausible targeted fix for the local-model bootstrap-pressure report, but the branch is not merge-ready because the new tier filter changes delegated/background session semantics and hook-added bootstrap files can bypass the minimal tier.

## What This Changes

Adds opt-in `agents.defaults.bootstrapTier` support, minimal bootstrap filtering, a loopback-local doctor warning, config metadata, docs, changelog, and targeted tests.

## Best Possible Solution

Land one coherent bootstrap-tier design that keeps `standard` backward compatible, makes `minimal` session-aware and hook-aware, and aligns schema metadata, doctor guidance, docs, changelog, and tests with the chosen #75189/#22439 direction.

## Reproduction Assessment

Yes. Static inspection of the PR diff reproduces both patch-level regressions: minimal filtering runs before the subagent/cron branch, and hook-added bootstrap extras are appended after the initial tier-aware filter without passing the tier onward.

## Solution Assessment

No. The config knob and doctor warning are a reasonable direction, but this implementation should first preserve delegated/background filtering semantics and apply the same tier policy after bootstrap hooks.
