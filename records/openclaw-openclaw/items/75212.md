## Summary

Keep open. Current main does not implement `agents.defaults.bootstrapTier`, and this PR is a plausible path for the local-model bootstrap pressure reported in #75189, but the patch is not merge-ready because it regresses delegated/background bootstrap filtering and lets hook-added bootstrap files bypass the new minimal tier.

## What This Changes

The PR adds `agents.defaults.bootstrapTier` with `standard` and `minimal`, applies minimal bootstrap filtering, adds loopback local-model doctor guidance, and updates schema metadata, docs, changelog, and targeted tests.

## Best Possible Solution

Land one coherent, session-aware bootstrap-tier design: keep `standard` backward compatible, make `minimal` reduce ordinary bootstrap predictably, preserve or explicitly document delegated/background context behavior, thread tier semantics through bootstrap hooks, and align doctor guidance, docs, schema metadata, changelog, and tests with the chosen #22439/#75189 direction.

## Reproduction Assessment

Yes. Static inspection of PR head is enough to reproduce the review blockers: `bootstrapTier: "minimal"` returns before the subagent/cron branch, and the bundled bootstrap-extra-files hook re-filters without the tier after appending extras.

## Solution Assessment

No. The config knob and doctor warning are a reasonable direction, but this implementation is not the narrowest maintainable fix until tier filtering is both session-aware and hook-aware, and maintainers choose it over the broader #22439 API shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve delegated bootstrap filtering under minimal:** `src/agents/workspace.ts:691-692`
  - body: The new `minimal` branch returns before the existing subagent/cron session check. With `agents.defaults.bootstrapTier: "minimal"`, delegated and cron runs lose the current reduced-session allowlist that preserves `SOUL.md` role context, and they gain first-run `BOOTSTRAP.md`, even though the PR describes the tier as ordinary-session filtering.
  - confidence: 0.86
- **[P2] Thread bootstrapTier through bootstrap-extra-files:** `src/agents/bootstrap-files.ts:249-253`
  - body: The tier is applied only before `agent:bootstrap` hooks run. The bundled `bootstrap-extra-files` hook appends extras and calls `filterBootstrapFilesForSession(..., context.sessionKey)` without the tier, so an ordinary session configured for `minimal` can still receive all matched hook extras after the new filter has run.
  - confidence: 0.82
