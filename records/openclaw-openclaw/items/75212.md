## Summary

Keep open. Current main does not implement `agents.defaults.bootstrapTier` or the loopback-local doctor warning, and the PR is not merge-ready because the new tier filter changes delegated/background bootstrap semantics and misses hook-added bootstrap files.

## What This Changes

The PR adds an opt-in `agents.defaults.bootstrapTier: "minimal"`, loopback-local bootstrap pressure doctor guidance, config metadata, docs, changelog, and targeted tests.

## Best Possible Solution

Land one coherent bootstrap-tier design that keeps `standard` backward compatible, makes `minimal` session-aware and hook-aware, and aligns doctor guidance, schema metadata, docs, changelog, and tests with the chosen #75189/#22439 direction.

## Reproduction Assessment

Yes. Static inspection of the PR head reproduces both blockers: subagent/cron sessions hit the new `minimal` branch before session filtering, and hook-added files are appended after the initial tier filter without carrying the tier.

## Solution Assessment

No. The config knob and doctor warning are plausible, but this implementation is not the best fix until tier filtering preserves delegated/background semantics and the hook path enforces the same tier policy.
