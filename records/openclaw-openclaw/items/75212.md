## Summary

Keep this PR open, but it is not merge-ready. The branch addresses a real prompt-pressure problem for loopback local models, yet PR head still has two prompt-context correctness blockers and overlaps active bootstrap-tier work in #22439.

## What This Changes

The branch adds `agents.defaults.bootstrapTier` with `standard` and `minimal` modes, wires minimal bootstrap filtering into agent bootstrap resolution, adds a loopback local-model doctor warning, and updates config metadata, docs, changelog, and tests.

## Best Possible Solution

Land one coherent bootstrap-tier design that keeps `standard` as the compatible default, makes `minimal` reduce ordinary bootstrap predictably, explicitly preserves or documents delegated/background session behavior, threads tier state through bootstrap hooks, and aligns docs/tests around the chosen API.

## Reproduction Assessment

Yes. Static inspection of PR head is enough: `filterBootstrapFilesForSession(mockFiles, "agent:default:subagent:task-1", "minimal")` takes the new minimal branch before the subagent/cron branch, and the bundled `bootstrap-extra-files` hook re-filters without the tier after appending extras.

## Solution Assessment

No. The config knob and doctor warning are plausible, but this patch is not the best implementation until the tier semantics are reconciled with #22439 and the filtering is session-aware and hook-aware.

Overall correctness: patch is incorrect
