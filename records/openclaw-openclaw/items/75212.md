## Summary

Keep open. Current main still lacks the proposed bootstrap-tier and local-model doctor guidance, but the PR head has blocking prompt-context regressions around delegated sessions and bootstrap hooks, so it is not merge-ready or obsolete.

## What This Changes

The PR adds `agents.defaults.bootstrapTier` with a `standard` default and opt-in `minimal`, wires minimal bootstrap filtering into bootstrap resolution, adds a loopback local-model doctor warning, and updates generated config metadata, docs, changelog, and tests.

## Best Possible Solution

Land a single session-aware and hook-aware bootstrap-tier design: keep `standard` as the compatible default, make `minimal` reduce ordinary bootstrap predictably, preserve or explicitly document delegated/background session behavior, and align docs/tests around the bundled extra-bootstrap hook.

## Reproduction Assessment

Yes. Static inspection of PR head is enough: `filterBootstrapFilesForSession(mockFiles, "agent:default:subagent:task-1", "minimal")` returns the new minimal allowlist with `BOOTSTRAP.md` and without `SOUL.md`, and the bundled hook still re-filters extras without the tier.

## Solution Assessment

No. The config and doctor warning are plausible, but this patch is not the best implementation until minimal filtering is threaded through every bootstrap mutation path and does not unintentionally change subagent/cron context.
