## Summary

Keep open: the PR is not implemented on current main and it has two source-reproducible correctness blockers in the tier filtering path, plus unresolved product/API overlap with the broader open bootstrap-tier PR.

## What This Changes

The PR adds schema/docs/tests/runtime support for an opt-in `agents.defaults.bootstrapTier: "minimal"` and a doctor warning for loopback local models with large bootstrap context.

## Best Possible Solution

Land one maintainer-approved bootstrap-context control that preserves `standard` behavior, keeps delegated and hook paths tier-aware, and aligns doctor guidance, schema, docs, tests, and real proof with the constrained-local-model issue.

## Reproduction Assessment

Yes for the PR blockers: source inspection of PR head shows the minimal tier returns before delegated-session filtering and that hook-added bootstrap files are re-filtered without the tier. The underlying local-model tool-dispatch bug was not live-rerun in this read-only review.

## Solution Assessment

No, not as-is. The config knob and doctor warning are plausible, but the implementation must preserve delegated/background semantics, make the hook path tier-aware, and settle the public API overlap before merge.
