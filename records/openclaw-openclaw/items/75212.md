## Summary

Keep this PR open, but it is not merge-ready. Current main does not implement `bootstrapTier`, and the PR targets a real local-model prompt-pressure problem, but the patch changes delegated/cron bootstrap semantics unintentionally and leaves the bundled extra-bootstrap hook outside the new tier contract.

## What This Changes

The PR adds `agents.defaults.bootstrapTier` with `standard` and opt-in `minimal`, applies it to bootstrap file filtering, adds a loopback local-model doctor warning, and updates config schema, docs, changelog, and tests.

## Best Possible Solution

Land one session-aware and hook-aware bootstrap-tier design: keep `standard` as the compatible default, make `minimal` reduce ordinary bootstrap predictably, preserve or explicitly revise delegated/background context semantics, and update docs/tests around bundled extra-bootstrap hooks.

## Reproduction Assessment

Yes. Static inspection of PR head is enough to reproduce the introduced patch bugs: `filterBootstrapFilesForSession(..., "agent:default:subagent:task-1", "minimal")` returns the ordinary minimal allowlist, and the bundled hook still calls the filter without the tier.

## Solution Assessment

No. The config knob and doctor warning are plausible, but this implementation is not the narrowest maintainable fix until the tier is threaded through every bootstrap mutation path and the delegated/cron behavior is made intentional.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve delegated bootstrap filtering under minimal:** `src/agents/workspace.ts:691-692`
  - body: The new `minimal` branch returns before the existing subagent/cron session check. With `agents.defaults.bootstrapTier: "minimal"`, delegated and cron runs now lose `SOUL.md` and gain the first-run `BOOTSTRAP.md`, even though the PR describes the tier as ordinary-session filtering and the existing allowlist was added to preserve delegated role context.
  - confidence: 0.86
- **[P2] Thread bootstrapTier through bootstrap-extra-files:** `src/agents/bootstrap-files.ts:252`
  - body: The tier is applied only before `agent:bootstrap` hooks run. The bundled `bootstrap-extra-files` hook appends extras and calls `filterBootstrapFilesForSession(..., context.sessionKey)` without the tier, so an ordinary session configured for `minimal` can still receive all matched hook extras after the new filter has run.
  - confidence: 0.82
