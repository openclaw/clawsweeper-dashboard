## What This Changes

The branch changes the plugin registry, manifest contract, docs, and tests so installed plugins with declared manifest contracts can register agent tool-result middleware and trusted tool policies while bundled trusted policies keep precedence.

## Best Possible Solution

Land the manifest-gated SDK contract only after maintainer/security approval of the trust model; otherwise replace it with the stricter trust mechanism maintainers want.

## Reproduction Assessment

Yes for the current blocked behavior: current main rejects non-bundled trusted policy and middleware registrations, and the middleware loader only considers bundled owners. I did not run the CLI repro because this review is read-only.

## Solution Assessment

Conditionally yes. The manifest-gated shape fits the manifest-first plugin boundary and is cleaner than ad hoc config plumbing, but it is only the best fix if maintainers accept manifest declaration as sufficient trust; otherwise an operator allowlist or signed-official-plugin gate is safer.
