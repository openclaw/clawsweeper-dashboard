## Summary

This PR is still needed because current main and the latest release still reject installed-plugin middleware/trusted-policy registration, but it should not merge as-is: the new trusted-policy registration path is not covered by plugin-loader rollback, so a plugin that registers a policy and then fails can leave that policy active.

## What This Changes

The PR lets installed plugins declare agent tool-result middleware runtimes and trusted tool policy ids in manifest/catalog contracts, admits those registrations, orders bundled trusted policies first, and updates plugin SDK docs/tests.

## Best Possible Solution

Fix trusted-policy rollback coverage, then have maintainers explicitly approve the manifest/catalog trust model before landing this plugin SDK contract.
