## Summary

The report remains necessary: current main has new protections for stale Codex binding metadata and selected internal handoffs, but the discussion’s broader Z.AI-only continuation case is not covered by evidence proving that every automatic runtime event preserves one coherent provider/model route. The sidecar workaround is useful context, yet the no-sidecar reproduction prevents an implemented-on-main close.

## What This Changes

Prevent embedded automatic continuations from combining a session provider with a stale or unrelated model identifier, including both Codex-sidecar and no-sidecar cases.

## Best Possible Solution

Resolve every automatic continuation from one canonical selected ModelRef, validate that full provider/model pair before dispatch and snapshot persistence, and preserve the user-selected session route when an internal run reports transport or fallback metadata.

## Reproduction Assessment

Yes at source level: the report includes repeated logs and persisted-state evidence, and current persistence code can store providerUsed and modelUsed as separate fields. A high-confidence current-main runtime reproduction is still missing for the exact no-sidecar automatic-continuation trigger.

## Solution Assessment

No, stale-sidecar deletion or a sidecar-only guard is not the complete solution because the reporter reproduced the same invalid route without a sidecar. The narrow maintainable fix is coherent route selection and validation at the internal-continuation boundary, with regression coverage for both sidecar and Z.AI-only sessions.
