## Summary

The central failure remains actionable on current main: persisted override fields are still recomposed without validating an atomic provider/model route, and Codex binding continuity relies on resume overrides rather than a direct route-identity guard. The reporter’s successive theories are not all proven, but neither current source nor the latest release establishes that invalid cross-provider state is prevented or repaired.

## What This Changes

Prevent persisted session, fallback, snapshot, or Codex binding state from producing invalid cross-provider model routes during embedded continuations and model switches.

## Best Possible Solution

Define one atomic resolved route contract for session overrides, fallback state, snapshots, and Codex bindings; enforce it at writers, repair malformed persisted entries through the owning migration path, and add regressions for both sidecar and no-sidecar embedded continuations.

## Reproduction Assessment

Partly yes: current source deterministically reproduces the reported double-prefix interpretation from malformed persisted override fields. A high-confidence live current-main path that identifies which writer creates that state and reproduces the complete continuation failure is still missing.

## Solution Assessment

No, none of the single proposed fixes is yet proven best. Reader normalization alone misses writer and migration ownership, while unconditional sidecar deletion risks valid Codex continuity; the maintainable fix is an atomic route invariant across all persistence boundaries.
