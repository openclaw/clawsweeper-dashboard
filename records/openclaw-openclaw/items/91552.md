## Summary

Keep open: current main has partial safeguards for configured-primary fallback and preserving displayed session model state, but there is no closing PR or release proof, and the Codex sidecar plus transcript snapshot paths still persist route state that can affect embedded continuations.

## What This Changes

The issue reports stale or mixed provider/model route state causing embedded continuations to dispatch invalid provider/model pairs such as `zai/gpt-5.5`.

## Best Possible Solution

Prove the provider-switch embedded-continuation path live on current main, then make route persistence derive from one validated resolved selection while invalidating stale Codex sidecars/snapshots and preserving wrapper-provider semantics.
