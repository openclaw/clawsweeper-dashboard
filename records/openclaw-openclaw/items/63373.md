## Summary

Current main now strips the reported uppercase bracketed tool-call/result transcript blocks from shared user-facing reply sanitization and heartbeat reply payloads before channel delivery, with regression coverage for the exact marker family.

## What This Changes

The issue requests that internal transcript markers such as `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` never reach Telegram or other user-visible channel messages after transcript repair or timeouts.

## Best Possible Solution

Keep the shared reply and heartbeat sanitizer coverage on main; any future leak should be tracked only if it uses a distinct marker shape or bypasses these normalized reply paths.

## Reproduction Assessment

Yes for the reported sanitizer gap: current tests exercise the same uppercase `[TOOL_CALL]` hash-style payload family and related `[TOOL_RESULT]` blocks. The original timing race is not deterministically reproduced, but the observable leak shape is now covered.

## Solution Assessment

Yes. Sanitizing in shared user-facing reply normalization plus heartbeat payload handling is the narrow maintainable fix for this reported Telegram leak because Telegram routed replies pass through those boundaries before channel formatting.
