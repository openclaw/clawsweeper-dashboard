## Summary

Keep open: current main and the latest shipped release still expose `chat.inject` as a closed, assistant-only transcript injection RPC with no public user-role replay or caller idempotency field. The useful implementation path is already represented by the open PR at https://github.com/openclaw/openclaw/pull/81513, so this issue should remain the tracker until that PR lands or maintainers reject the protocol change.

## What This Changes

The issue requests additive `chat.inject` support for user-role synthetic transcript replay plus stable caller-supplied IDs or equivalent returned IDs for durable cross-references.

## Best Possible Solution

Review and land an additive Gateway protocol change like https://github.com/openclaw/openclaw/pull/81513, with generated clients, docs, parent-chain coverage, and clear idempotent replay semantics.

## Reproduction Assessment

Yes for the current limitation: source inspection shows the closed `chat.inject` schema and assistant-only append path on current main. A live Gateway probe was not necessary because the accepted wire contract and handler path are explicit in source.

## Solution Assessment

Yes: an additive Gateway protocol change is the maintainable direction, and the open implementation PR is the right place to finish it. Closing this issue before that PR merges would lose the tracking item for a still-missing current-main capability.
