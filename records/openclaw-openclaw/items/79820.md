## Summary

Keep open: current main still exposes `chat.inject` as a closed, assistant-only transcript append, and the linked implementation PR is open and unmerged, so the requested replay/idempotency contract is not on main.

## What This Changes

Request to extend Gateway `chat.inject` or add a companion API so callers can replay user-role synthetic turns with durable caller IDs and documented parent-chain semantics.

## Best Possible Solution

Review, finish, and land or replace the linked additive Gateway protocol PR so schema, handler behavior, generated clients, tests, and public docs consistently cover user-role replay, idempotent returns, and parent-chain semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection on current main shows the schema rejects extra fields and the handler always appends through the assistant transcript path. This is not a broken documented behavior; it is a missing Gateway replay capability.

Not applicable as a patch on this issue. The requested direction is plausible, but the maintainable solution is to review and land or replace the open additive protocol PR after the remaining docs/protocol concerns are resolved.
