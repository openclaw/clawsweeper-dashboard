## Summary

Keep open: current main still exposes `chat.inject` as an assistant-only, closed-schema transcript append, while the issue asks for new replay and idempotency semantics that are not implemented and need Gateway protocol design.

## What This Changes

Request an additive Gateway chat replay contract that accepts user-role synthetic turns, stable caller IDs or returned cross-reference IDs, and documented parent-chain behavior.

## Best Possible Solution

Design an additive Gateway replay API or carefully extended `chat.inject` contract that appends user and assistant synthetic turns through the transcript manager, returns durable IDs, and documents parent-chain semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows the schema is closed to `sessionKey`, `message`, and `label`, and the implementation always writes an assistant transcript message. This is not a broken documented behavior; it is a missing replay capability.

## Solution Assessment

Not applicable as a patch yet: the issue requests a new public Gateway contract, and the safest direction is an additive protocol design rather than ad hoc extra fields on the current closed schema.
