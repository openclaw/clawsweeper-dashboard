## Summary

Keep open: current main still exposes `chat.inject` as an assistant-only, closed-schema append, and the requested user-role replay/idempotency contract is only being handled by the still-open implementation PR at https://github.com/openclaw/openclaw/pull/81513.

## What This Changes

The issue asks to extend Gateway `chat.inject` so synthetic transcript replay can append user-role turns and expose stable idempotency/caller IDs with documented parent-chain semantics.

## Best Possible Solution

Land an additive Gateway replay contract, likely through the open implementation PR after review, that supports synthetic user and assistant appends, idempotent caller references, durable returned IDs, and parent-chain documentation.

## Reproduction Assessment

Yes for the current limitation: source inspection on current main shows `chat.inject` rejects fields outside `sessionKey`, `message`, and `label`, and the handler always writes through the assistant append path. This is a missing public protocol capability, not a broken documented behavior.

## Solution Assessment

Not applicable as a patch on this issue. The narrowest maintainable path is an additive Gateway protocol change with matching docs/tests, then closing this issue after the linked implementation PR merges or is replaced.
