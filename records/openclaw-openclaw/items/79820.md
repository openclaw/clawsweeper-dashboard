## Summary

Keep open: current main and the latest shipped release still expose `chat.inject` as a closed, assistant-only Gateway RPC, while the matching role/idempotency implementation remains open and unmerged in https://github.com/openclaw/openclaw/pull/81513.

## What This Changes

Requests an additive `chat.inject` contract for synthetic transcript replay with user-role turns, stable caller/idempotency IDs, and documented parent-chain behavior.

## Best Possible Solution

Review and land an additive Gateway protocol change like https://github.com/openclaw/openclaw/pull/81513 with generated clients, docs, parent-chain coverage, and clear idempotent replay semantics, then close this issue after merge.

## Reproduction Assessment

Yes. Current main source shows the closed `chat.inject` schema and assistant-only append path, so requests with `role` or caller idempotency fields would fail validation without needing a live Gateway probe.

## Solution Assessment

Yes. An additive Gateway protocol update is the maintainable direction, and the open implementation PR is the right place to finish it; closing this issue before that PR lands would lose the tracking item for a still-missing current-main capability.
