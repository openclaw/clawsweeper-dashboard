## Summary

Keep open: current main still exposes `chat.inject` as a closed, assistant-only transcript append, while this issue asks for new user-turn replay, caller-supplied correlation/idempotency semantics, and parent-chain documentation. The related closed role-param attempt and the open `originAgent` attribution PR do not cover the full requested replay and durable ID contract.

## What This Changes

The issue requests an additive Gateway transcript replay contract for `chat.inject` or a companion method that can replay user-role synthetic turns and preserve durable caller IDs.

## Best Possible Solution

Design an additive Gateway replay API or carefully extended `chat.inject` contract that appends user and assistant synthetic turns through the transcript manager, returns durable IDs, and documents parent-chain and trust semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows `chat.inject` has a closed schema and always writes assistant-role transcript entries. This is a missing public replay capability rather than a broken documented bug.

## Solution Assessment

Not applicable as a patch yet. The requested direction needs an additive Gateway protocol design, and a role-only extension would not cover durable caller IDs, metadata trust, or documented parent-chain behavior.
