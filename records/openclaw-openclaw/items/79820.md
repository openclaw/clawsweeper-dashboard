## Summary

Keep open: current main still exposes `chat.inject` as an assistant-only closed-schema transcript append, while user-role replay, stable caller IDs, and documented injected-turn parent-chain semantics remain new Gateway protocol work.

## What This Changes

Allow `chat.inject` or a companion API to replay synthetic user turns, expose durable caller/cross-reference IDs, and document parent-chain behavior for injected user-to-assistant pairs.

## Best Possible Solution

Design an additive Gateway replay contract that writes synthetic user and assistant turns through the transcript manager, returns durable IDs or cross-reference metadata, and documents replay/parent-chain semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows the schema is closed to `sessionKey`, `message`, and `label`, and the implementation always writes an assistant transcript message. This is a missing replay capability rather than a broken documented contract.

## Solution Assessment

Not applicable as a patch yet: the request needs a public Gateway protocol design, and the safer direction is an additive replay contract rather than ad hoc extra fields on the current assistant-note RPC.
