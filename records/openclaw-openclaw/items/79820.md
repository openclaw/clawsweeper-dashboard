## Summary

Keep open: current main still exposes `chat.inject` as a closed-schema, assistant-only transcript append, and the requested replay/idempotency semantics are unimplemented Gateway protocol work rather than a closeable bug or duplicate.

## What This Changes

The issue requests user-role synthetic turn replay, stable caller-supplied or returned message identifiers, and documented parent-chain behavior for injected turns.

## Best Possible Solution

Design an additive Gateway replay API or carefully extended `chat.inject` contract that appends synthetic user and assistant turns through the transcript manager, returns durable IDs, and documents parent-chain semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows a closed `chat.inject` schema and an assistant-only append path. This is not a broken documented behavior; it is a missing replay capability.

## Solution Assessment

Not applicable as a patch yet: the issue requests a new public Gateway contract. The safest direction is an additive protocol design rather than ad hoc extra fields on the current closed schema.
