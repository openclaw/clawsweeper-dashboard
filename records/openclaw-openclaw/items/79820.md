## Summary

Keep open: current main still exposes `chat.inject` as a closed-schema, assistant-only transcript append, while the requested user-turn replay and durable caller-ID semantics remain unimplemented and require Gateway protocol design.

## What This Changes

Request to extend `chat.inject` or add a companion replay API for user-role synthetic turns, stable caller-supplied IDs, and documented parent-chain behavior.

## Best Possible Solution

Design an additive Gateway replay contract that can append synthetic user and assistant turns through the transcript manager, return durable IDs, and document parent-chain semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows the schema is closed to `sessionKey`, `message`, and `label`, and the implementation always appends an assistant message. This is source-reproducible as a missing capability rather than a broken documented behavior.

## Solution Assessment

Not applicable as a patch yet: the request changes the public Gateway protocol and transcript trust model. The safest direction is an additive replay API or carefully versioned extension, not an ad hoc field expansion.
