## Summary

This should stay open: current main still exposes `chat.inject` as an assistant-only, closed-schema RPC, and the requested user-role replay plus stable idempotency contract is being handled by the open implementation PR at https://github.com/openclaw/openclaw/pull/81513.

## What This Changes

Request `chat.inject` support for synthetic user-role transcript replay plus stable caller-supplied or returned IDs for durable idempotent cross-references.
