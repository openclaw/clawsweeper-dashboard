## Summary

Keep open: current main still exposes `chat.inject` as an assistant-only, closed-schema transcript append, while the requested user-turn replay, stable caller IDs, metadata semantics, and parent-chain documentation are not implemented and require Gateway protocol design.

## What This Changes

The issue requests expanding `chat.inject` or adding a companion replay API for synthetic user turns, stable caller-provided IDs, and documented transcript parent-chain behavior.

## Best Possible Solution

Design an additive Gateway replay contract, or a carefully extended `chat.inject`, that appends synthetic user and assistant turns through the transcript manager, returns durable IDs, and documents parent-chain and idempotency semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection on current main shows the schema is closed to `sessionKey`, `message`, and `label`, and the implementation writes only assistant transcript entries. This is a missing replay capability rather than a broken documented behavior.

## Solution Assessment

Not applicable as a patch yet: the issue asks for a new public Gateway transcript contract. The maintainable path is an additive protocol design that explicitly handles role, idempotency, return IDs, trust boundaries, and parent-chain behavior.
