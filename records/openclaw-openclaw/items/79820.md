## Summary

Keep open: current `main` still exposes `chat.inject` as a closed, assistant-only transcript append. The related implementation PR at https://github.com/openclaw/openclaw/pull/81513 targets the requested user-role/idempotent replay contract, but it is still open and should be reviewed or landed before this issue closes.

## What This Changes

Add user-role replay and durable caller/idempotency identifiers for synthetic `chat.inject` transcript turns, or provide an equivalent companion replay API.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81513, or an equivalent additive Gateway contract, with schema, handler behavior, generated clients, tests, and public docs aligned around user-role injection, idempotent replay, returned IDs, and parent-chain semantics.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows the schema is closed to `sessionKey`, `message`, and `label`, and the handler always appends an assistant transcript message. This is a missing Gateway replay capability rather than a broken documented bug.

## Solution Assessment

Yes in direction, but not complete on `main`: the maintainable path is an additive protocol change like the open implementation PR, with docs and protocol review before merge.
