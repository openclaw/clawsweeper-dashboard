## Summary

Keep open. Current main still has the reported Linux service PATH inconsistency, and the issue is already paired with open closing PR #75440, so it should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue reports that Linux `openclaw gateway install --force` and doctor repair can preserve version-manager/package-manager PATH entries from an existing service unit, then doctor immediately flags the regenerated PATH as non-minimal.

## Best Possible Solution

Review and land PR #75440, or an equivalent patch, so preserved existing PATH additions are filtered by the same non-minimal policy that doctor audits while safe custom absolute PATH entries remain preserved.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a gateway install plan with a generated minimal PATH and `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, fnm, or pnpm entries, then audit the planned PATH and observe `gateway-path-nonminimal`. The issue discussion also includes an independent Ubuntu 24.04/npm confirmation on OpenClaw 2026.4.29.

## Solution Assessment

Yes. The narrow maintainable fix is to align install-plan preservation with the existing audit contract; relaxing doctor warnings would keep the repair path and audit policy inconsistent.
