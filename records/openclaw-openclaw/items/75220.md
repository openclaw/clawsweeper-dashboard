## Summary

Keep open. Current main still merges preserved existing service PATH additions into gateway install planning without applying the same non-minimal version-manager/package-manager filter that doctor later audits, and open PR #75440 is already the closing implementation candidate.

## What This Changes

The issue asks for Linux gateway install and doctor repair to stop writing user-systemd service PATH entries that doctor immediately flags as non-minimal.

## Best Possible Solution

Review and land PR #75440, or an equivalent patch, that applies the non-minimal service PATH policy to preserved existing PATH additions while continuing to preserve safe custom absolute entries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a gateway install plan with `existingEnvironment.PATH` containing stale version-manager/package-manager segments, then audit the planned PATH and observe the `gateway-path-nonminimal` issue; the comments also include an independent Ubuntu 24.04/npm confirmation.

## Solution Assessment

Yes at the implementation direction level. Aligning installer preservation with the existing audit contract is narrower and safer than relaxing doctor warnings, because it makes forced reinstall and doctor repair converge on the same accepted PATH.
