## Summary

Keep open. Current main still lets `gateway install --force` and doctor repair append preserved PATH segments from the existing service unit, while the service audit still reports stale version-manager/package-manager segments as `gateway-path-nonminimal`. The open closing PR #75440 is the right implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for Linux gateway service reinstall/repair to stop generating service PATH entries that OpenClaw doctor immediately flags as non-minimal.

## Best Possible Solution

Review and land PR #75440, or an equivalent current-main patch, that filters preserved existing PATH entries using the same non-minimal policy doctor applies while keeping safe custom absolute PATH additions and existing temp/proc/workspace hardening. Keep this issue open until the closing PR merges.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildGatewayInstallPlan` with a minimal `serviceEnvironment.PATH` plus an `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm/current`, `.fnm/current`, or pnpm entries, then audit the planned environment and observe `gateway-path-nonminimal`. The issue discussion also includes an independent Ubuntu 24.04/npm confirmation on OpenClaw 2026.4.29.

## Solution Assessment

Yes. The narrow maintainable solution is to align the preserved-PATH filter with the existing audit contract instead of relaxing doctor warnings or removing all custom PATH preservation.
