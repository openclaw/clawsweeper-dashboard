## Summary

Keep open. Current main still preserves existing service PATH additions during gateway install planning without applying the same version-manager/package-manager filter that doctor later uses, and an open closing PR (#75440) is already the implementation candidate, so this issue should remain open until that PR or an equivalent fix merges.

## What This Changes

Make `openclaw gateway install --force` and doctor repair generate a gateway service PATH that doctor accepts as minimal instead of preserving stale version-manager or package-manager PATH entries.

## Best Possible Solution

Review and land PR #75440, or an equivalent patch, that filters preserved existing PATH additions through the same non-minimal policy doctor audits while continuing to preserve safe custom absolute PATH entries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a gateway install plan with a generated minimal PATH plus `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, `.fnm`, or pnpm entries, then audit the planned PATH and observe `gateway-path-nonminimal`. The issue discussion also includes an independent Ubuntu 24.04/npm confirmation on OpenClaw 2026.4.29.

## Solution Assessment

Yes. The narrow maintainable solution is to align the preserved-PATH filter with the existing audit contract; relaxing doctor warnings would leave the service installer and repair path internally inconsistent.
