## Summary

## What This Changes

The issue asks for `openclaw gateway install --force` and doctor repair to stop creating or preserving service PATH entries that doctor immediately flags as non-minimal.

## Best Possible Solution

## Solution Assessment

Yes. Filtering preserved PATH additions to match the existing audit contract is the narrow maintainable fix; relaxing doctor would leave the repair path accepting paths it already considers non-minimal.
