## Summary

Keep open. Current main still preserves existing service PATH entries during gateway install and doctor repair without applying the audit's non-minimal PATH policy, and the open closing PR #75440 is the right implementation candidate rather than a reason to close this issue early.

## What This Changes

Make `openclaw gateway install --force` and doctor repair generate service PATHs that doctor accepts as minimal instead of preserving stale version-manager and package-manager segments.

## Best Possible Solution

Review and land PR #75440, or an equivalent narrow patch, that filters preserved existing PATH additions with the same non-minimal policy doctor applies while preserving safe custom absolute PATH additions and the existing temp/proc/workspace hardening.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildGatewayInstallPlan` with a generated minimal service PATH and an `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, fnm, or pnpm entries, then audit the planned PATH and observe `gateway-path-nonminimal`. The issue discussion also includes an independent Ubuntu 24.04/npm confirmation on OpenClaw 2026.4.29.

## Solution Assessment

Yes. The narrow maintainable fix is to align preserved-PATH filtering with the existing service audit predicate; relaxing doctor warnings or dropping all custom PATH preservation would either hide the contradiction or risk breaking legitimate custom service additions.
