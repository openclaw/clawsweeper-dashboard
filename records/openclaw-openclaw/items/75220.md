## Summary

Keep open. Current main still preserves stale service PATH additions that doctor audits as non-minimal, and open PR #75440 already references this issue with closing syntax as the implementation candidate.

## What This Changes

Linux gateway install and doctor repair should stop regenerating service PATH entries that the gateway service audit immediately flags as non-minimal.

## Best Possible Solution

Review and land PR #75440, or an equivalent patch, so preserved existing PATH additions are filtered by the same non-minimal policy that doctor audits while safe custom absolute PATH entries remain preserved.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a gateway install plan with `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, `.fnm`, or pnpm entries, then audit the planned PATH and observe `gateway-path-nonminimal`.

## Solution Assessment

Yes. Filtering preserved PATH additions to match the existing audit contract is the narrow maintainable fix; relaxing the doctor warning would keep installer repair and audit policy inconsistent.
