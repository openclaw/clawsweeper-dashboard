## Summary

Keep open. Current main still preserves existing service PATH segments during forced gateway install and doctor repair, while the audit still flags stale version-manager/package-manager segments as non-minimal. The open closing PR #75440 is the right implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for Linux `openclaw gateway install --force` and doctor repair to generate a service PATH that doctor accepts as minimal instead of preserving version-manager/package-manager entries it later warns about.

## Best Possible Solution

Review and land PR #75440, or an equivalent patch, that aligns preserved existing PATH filtering with the audit non-minimal policy while keeping explicit env-configured roots and safe custom absolute PATH additions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildGatewayInstallPlan` with a generated service PATH and an `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, `.fnm`, or pnpm entries, then audit the planned PATH and observe `gateway-path-nonminimal`. The issue comments also include an independent Ubuntu/npm confirmation.

## Solution Assessment

Yes. The narrow maintainable fix is to filter only preserved stale PATH additions using the same non-minimal policy doctor already enforces, rather than relaxing doctor warnings or dropping all custom PATH preservation.
