## Summary

Keep open. Current main still preserves existing gateway service PATH entries during install planning without applying the same non-minimal version-manager/package-manager policy that doctor later audits, and open PR #75440 is already the closing implementation candidate.

## What This Changes

The issue asks for Linux gateway install and doctor repair to stop regenerating service PATH entries that doctor immediately reports as non-minimal.

## Best Possible Solution

Review and land PR #75440, or an equivalent patch, that filters preserved existing PATH additions through the same non-minimal policy doctor audits while continuing to preserve safe custom absolute PATH entries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a gateway install plan with a generated Linux service PATH plus `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, `.fnm`, or pnpm entries, then audit that planned PATH and observe `gateway-path-nonminimal`. The issue discussion also includes an independent Ubuntu 24.04/npm confirmation on OpenClaw 2026.4.29.

## Solution Assessment

Yes. Aligning preserved-PATH filtering with the existing audit contract is the narrow maintainable fix; relaxing the doctor warning would leave installer output and repair diagnostics internally inconsistent.
