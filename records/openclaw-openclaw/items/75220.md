## Summary

Keep open. Current main still has the installer/audit mismatch, and the open closing PR #75440 is the active implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Linux `openclaw gateway install --force` and doctor repair stop generating or preserving service `PATH` entries that the doctor audit immediately flags as non-minimal.

## Best Possible Solution

Review and land PR #75440, or an equivalent narrow patch, so preserved existing service PATH additions are filtered through the same non-minimal policy the doctor audit uses while safe custom absolute PATH additions remain preserved.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a gateway install plan with generated minimal PATH plus `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, `.fnm`, or pnpm entries, then audit the planned PATH and observe `gateway-path-nonminimal`; the issue discussion also includes an independent Ubuntu 24.04/npm confirmation on OpenClaw 2026.4.29.

## Solution Assessment

Yes. Aligning installer preservation with the existing audit predicate is the narrow maintainable fix; relaxing doctor would leave the repair path and service hardening policy inconsistent.
