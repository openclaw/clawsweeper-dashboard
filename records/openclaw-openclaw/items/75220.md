## Summary

Keep open. Current main still preserves existing service PATH additions during gateway install planning without filtering the same version-manager and pnpm-like entries that doctor audits as non-minimal, and open PR #75440 already targets this issue with closing syntax.

## What This Changes

The report asks for gateway install and doctor repair to stop regenerating service PATH entries that doctor immediately flags as non-minimal.

## Best Possible Solution

Land PR #75440 or an equivalent patch that filters preserved existing PATH additions through the same non-minimal policy doctor audits while continuing to preserve safe custom absolute PATH entries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a gateway install plan with `existingEnvironment.PATH` containing stale `.volta`, `.asdf`, `.nvm`, `.fnm`, or pnpm entries, then audit the planned PATH and observe `gateway-path-nonminimal`.

## Solution Assessment

Yes. Aligning preserved-PATH filtering with the existing audit contract is the narrow maintainable fix; relaxing the doctor warning would leave installer repair and audit policy inconsistent.
