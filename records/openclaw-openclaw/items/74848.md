## Summary

Keep open: current main still does not repair trusted public WSS stale-pin mismatches, and open PR #75228 references this issue with closing syntax but remains unmerged and under review.

## What This Changes

The issue reports that the macOS app node loops with `cancelled` against a direct public WSS gateway while a CLI node in the same environment connects normally.

## Best Possible Solution

Review and land PR #75228 or an equivalent fix that keeps default pinning fail-closed, adds a schema-valid documented opt-in for system-trusted public-host stale-pin repair, and preserves untrusted/private-CA rejection.

## Reproduction Assessment

Yes for source-level reproduction: current main rejects trusted `wss://gateway.example.com` stale-pin repair while the TLS delegate cancels stored-pin mismatches. A live reproduction would require a macOS app install with a stale stored pin plus a rotated public certificate.

## Solution Assessment

Unclear until the TLS policy is approved. The open PR's default-off opt-in is safer than broad automatic repair, but it still needs config-contract follow-through and maintainer/security review before this issue can close.
