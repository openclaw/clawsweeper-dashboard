## Summary

Keep open. Current main still matches the reported macOS app node failure mode for public WSS gateways with stale stored TLS pins, and open PR #75228 already references this issue with closing syntax, so the issue should remain open until that or an equivalent reviewed fix lands.

## What This Changes

The issue reports that the macOS app remote node repeatedly disconnects with a cancelled WSS gateway connection while `openclaw node run --tls` works against the same gateway.

## Best Possible Solution

Review and land PR #75228 or an equivalent macOS fix that gives public ACME-style WSS gateways an explicit, documented stale-pin repair path while keeping self-signed, private-CA, certificate-unavailable, store-key-mismatched, and untrusted-chain paths fail-closed. If the opt-in config flag remains the chosen design, align it with schema/help/docs before release.

## Reproduction Assessment

Yes for source-level reproduction: current tests assert that a trusted public `wss://gateway.example.com` pin mismatch is not auto-repaired, and the TLS delegate cancels mismatched stored pins. A live reproduction needs the reporter's stale Keychain pin state or a controlled public certificate rotation.

## Solution Assessment

Unclear until maintainers approve the TLS trust boundary. PR #75228's explicit opt-in direction is safer than silent broad repair, but it still needs maintainer/security review and config/docs alignment before this issue should close.

## Security Review

Status: needs_attention

Summary: The issue and proposed fix path change TLS pin auto-repair policy for public WSS hosts, so maintainer/security approval is needed before closing.

Concerns:

- **[medium] Confirm public-host stale pin repair policy:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:204`
  - body: Current main only repairs stale pins for loopback and Tailscale Serve hosts. Expanding repair to public system-trusted hosts changes the pinning threat model because system trust may include admin, MDM, user-installed, or interception roots.
  - confidence: 0.9
- **[medium] Preserve fail-closed handling for untrusted chains:** `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayTLSPinning.swift:229`
  - body: The TLS delegate currently cancels mismatched and untrusted certificate challenges. Any fix should continue rejecting self-signed, private-CA, unavailable, store-key-mismatched, and otherwise untrusted certificate paths unless the operator explicitly opts into the intended trade-off.
  - confidence: 0.86
