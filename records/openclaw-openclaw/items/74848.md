## Summary

Keep open. Current main still matches the reported macOS direct-remote failure mode for public WSS gateways with a stale stored TLS pin: pin mismatches are cancelled, auto-repair is limited to loopback and Tailscale Serve hosts, and PR #75228 is open with closing syntax for this issue.

## What This Changes

The issue asks for the macOS app node in direct remote mode to recover from repeated `cancelled` disconnects on a public `wss://` gateway while `openclaw node run --tls` works in the same environment.

## Best Possible Solution

Review and land PR #75228 or an equivalent macOS fix that gives public system-trusted WSS gateways a documented stale-pin repair path while keeping self-signed, private-CA, store-key-mismatched, unavailable-certificate, and untrusted-chain cases fail-closed.

## Reproduction Assessment

Yes for source-level reproduction: current main has a test asserting that a trusted public `wss://gateway.example.com` pin mismatch is not auto-repaired, and the TLS delegate cancels mismatched stored pins. A live reproduction still requires a stale stored app pin plus a rotated public certificate or the reporter's original environment.

## Solution Assessment

Unclear until maintainers approve the TLS trust boundary. The linked PR's system-trusted repair direction is plausible and narrower than silent broad repair, but it should be reviewed as a security-sensitive policy change before this issue closes.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because the fix path changes when macOS app TLS pin mismatches may be auto-repaired instead of staying fail-closed.

Concerns:

- **[medium] Confirm public-host stale-pin repair policy:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:204`
  - body: Current main only repairs stale pins for loopback and Tailscale Serve hosts. Extending repair to public system-trusted WSS hosts changes the pinning threat model, especially for machines with admin, MDM, user-installed, or interception roots in system trust.
  - confidence: 0.9
- **[medium] Preserve fail-closed handling for untrusted chains:** `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayTLSPinning.swift:229`
  - body: The TLS delegate currently cancels mismatched and untrusted certificate challenges. Any fix should keep self-signed, private-CA, certificate-unavailable, store-key-mismatched, and otherwise untrusted paths rejected unless maintainers explicitly choose a different policy.
  - confidence: 0.86
