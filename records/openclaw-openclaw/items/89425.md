Keep open: current main and the latest release still carry the `speech-core` bundled-facade/package mismatch, and the open closing PR at https://github.com/openclaw/openclaw/pull/89899 remains the direct fix path.

## What This Changes

The issue reports a shipped npm packaging regression where Gateway message handling tries to load `speech-core/runtime-api.js` as a bundled extension sidecar that is absent from the package, causing replies to fail before provider execution.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/89899 if maintainers accept removing the unsupported sidecar contract; otherwise require a narrow sidecar-preserving compatibility fix before closing this issue.

## Reproduction Assessment

Yes. Current main and `v2026.6.8` still allow `speech-core/runtime-api.js` as an always-allowed runtime facade while no bundled `extensions/speech-core` sidecar exists; I did not run a fresh live Gateway repro.

## Solution Assessment

Yes, likely, subject to maintainer compatibility acceptance. The open PR's remove-the-invalid-allowlist-entry plus release-gate approach is narrower than a broad missing-surface fallback, but maintainers must decide whether to preserve the sidecar as compatibility.
