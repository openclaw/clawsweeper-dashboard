## Summary

Keep open. Current main still lacks the requested recipes page, so the PR is not obsolete, but the submitted docs are not merge-ready because the webhook example crosses the current prompt trust boundary and two anti-pattern rows no longer match current agent/channel behavior.

## What This Changes

This PR adds a new `docs/automation/system-events.md` recipes page, adds it to the Automation navigation, and cross-links it from `docs/cli/system.md`.

## Best Possible Solution

Keep the docs work, but revise it against current main so it clearly separates trusted operator CLI wake events from authenticated external webhooks, points external payload examples at `POST /hooks/wake` or explicit validation guidance, and updates the anti-pattern table to match current agent and channel semantics.

## Reproduction Assessment

Yes. The missing documentation surface is reproducible with `pnpm docs:list` and the current docs tree, and the review findings are reproducible by comparing the PR-added lines with the current CLI, hook, system-event, Slack, and Discord code paths.

## Solution Assessment

No, not as-is. A dedicated recipes page is the right direction, but the narrow maintainable solution must document the supported wake mechanisms with the correct trust boundary and current failure modes before merge.

  - confidence: 0.9
