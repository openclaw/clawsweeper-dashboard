## Summary

Keep open. Current main still lacks the requested system-event recipes page, and this PR is useful, but it should not merge as-is because the webhook recipe crosses the trusted system-event boundary and the anti-pattern table misstates current agent and channel behavior.

## What This Changes

The PR adds a `docs/automation/system-events.md` recipes page, links it from `docs/cli/system.md`, and adds an Automation navigation entry in `docs/docs.json`.

## Best Possible Solution

Revise and rebase the docs-only change so trusted local CLI wake recipes are separated from external webhook guidance, with webhooks routed to authenticated `/hooks/wake` or clearly documented validation boundaries.

## Reproduction Assessment

Yes. The missing docs surface and the PR blockers are source-reproducible by comparing the added docs with current `system event`, `/hooks/wake`, `agent`, Slack, and Discord behavior.

## Solution Assessment

No, not as-is. A dedicated recipes page is the right direction, but the maintainable version needs to document trusted CLI wakes separately from authenticated or validated external webhook input and align the anti-patterns with current code.
