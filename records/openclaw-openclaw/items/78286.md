## Summary

Keep open: the PR is a focused fix for a source-reproducible cron regression and appears correct, with sufficient terminal proof, but it still needs maintainer approval because it changes the cron tool authorization boundary.

## What This Changes

The branch allows self-remove-scoped cron tool runs to dispatch `status` and session-agent-scoped `list`, adds regression tests, and records the user-facing changelog fix.

## Best Possible Solution

Land the focused PR after maintainer acceptance of the read-only cron authorization boundary, preserving scoped `list` and mutation-deny coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main calls `assertCronSelfRemoveScope` before the action switch and the guard throws for any self-remove-scoped non-remove action; the current test also expects `list` to throw.

## Solution Assessment

Yes, with maintainer approval. The patch keeps mutation/remove-other paths denied and reuses the existing `agentId` filter for `list`; the only remaining judgment is whether global `cron.status` is acceptable in this scoped read context.
