## What This Changes

The issue requests Twilio AMD support for outbound voice-call conversations, including config knobs, outbound request parameters, AMD callback parsing, and a dynamic switch to voicemail-style notification when a machine answers.

## Best Possible Solution

Implement this inside the voice-call plugin with an explicit AMD config contract, Twilio request and callback parsing, a typed machine-answer event, manager transition tests, docs/schema updates, and live Twilio proof for human and voicemail paths.

## Reproduction Assessment

## Solution Assessment

Unclear on the final API shape, but the owner boundary is right. Voice-call should own Twilio AMD and the mode transition, while maintainers choose the public config defaults and lifecycle semantics before implementation.
