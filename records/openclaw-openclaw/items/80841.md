## Summary

Keep open: current main and the latest stable release still lack Twilio AMD config, outbound call parameters, `AnsweredBy` parsing, and a machine-answer lifecycle event. This is a real Voice Call plugin feature/API request that needs owner/product review before implementation.

## What This Changes

Add Twilio answering-machine detection for outbound Voice Call conversation calls and switch to a voicemail-style notify flow when Twilio reports a machine answer.

## Best Possible Solution

Add an owner-reviewed Twilio AMD path in the Voice Call plugin: typed outbound config, Twilio call parameters, verified AMD callback parsing, a machine-answer event, targeted notify-after-answer transition logic, tests, and docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because Twilio AMD is not an existing documented OpenClaw contract. Source inspection on current main confirms the requested surface is absent with high confidence.

## Solution Assessment

Unclear whether the exact suggested API is the best final shape. Twilio supports both blocking and asynchronous AMD, so the maintainable fix should deliberately choose additive config and callback semantics without changing existing notify/conversation behavior by default. ([twilio.com](https://www.twilio.com/docs/voice/answering-machine-detection?utm_source=openai))
