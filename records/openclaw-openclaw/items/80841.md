## Summary

Keep open. Current main still lacks Twilio AMD configuration, call-creation parameters, `AnsweredBy` parsing, and a machine-answer event or mode transition, so this is a valid voice-call feature request rather than cleanup-close material.

## What This Changes

Add Twilio outbound AMD configuration, pass AMD params on call creation, parse machine-answer callbacks, and switch conversation calls into a voicemail delivery flow when a machine is detected.

## Best Possible Solution

Implement this inside the voice-call plugin as an explicit Twilio AMD feature with schema, manifest/UI hints, docs, focused regression tests, and live Twilio proof before release.

## Reproduction Assessment

Yes, at source level: current main has no outbound AMD config, no Twilio AMD request params, no `AnsweredBy` parsing, and no machine-answer lifecycle event. I did not run a live Twilio voicemail call in this read-only review.

## Solution Assessment

Unclear until owner review: Twilio AMD belongs in the voice-call plugin, but the exact config names, async versus blocking AMD behavior, and machine-detected mode transition need a maintainer product decision.
