## Summary

The PR changes `extensions/mattermost/src/mattermost/monitor.ts` so non-allowlisted Mattermost group senders fall through to pending-history recording before returning without a bot response.

## Best Possible Solution

Port the history-only allowlist fallthrough to the current Mattermost nested ingress flow and add monitor-path regression coverage for both recorded context and no response dispatch.

## Reproduction Assessment

Yes. Source inspection on current main shows a Mattermost group message from a non-allowlisted sender under `groupPolicy: "allowlist"` reaches the immediate not-allowlisted return before pending-history recording can run.

## Solution Assessment

No, not as the current branch is written. The behavior fix is the right direction, but it must be ported from the old flat access-decision flow to current nested `ingress` and `commandAccess` handling.
