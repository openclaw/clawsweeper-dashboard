## Summary

Keep open. Current main still drops Mattermost non-allowlisted group senders before pending history can be recorded, and this PR is the active focused implementation candidate, but it now conflicts with the current ingress refactor and needs adaptation plus targeted coverage before merge.

## What This Changes

The PR changes the Mattermost monitor so non-allowlisted group senders under `groupPolicy: "allowlist"` are recorded into pending history before returning without a response.

## Best Possible Solution

Adapt this Mattermost-local history-only fallthrough to current main's ingress/commandAccess shape, add focused monitor coverage, and land it with https://github.com/openclaw/openclaw/issues/57607 once checks pass.

## Reproduction Assessment

Yes. The source path is high-confidence: with Mattermost `groupPolicy: "allowlist"` and a non-allowlisted group sender, current main reaches the immediate not-allowlisted return before pending-history recording can run.

## Solution Assessment

Yes, with current-main adaptation. A Mattermost-local fallthrough that records bounded pending history before a no-response return is the narrowest maintainable fix, provided it skips response and command execution for blocked senders.
