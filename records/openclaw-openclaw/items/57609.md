## Summary

Keep open. The Mattermost bug still exists on current main and the PR has useful real-world proof, but the branch is not merge-ready because it edits the pre-refactor access-flow names and GitHub reports it conflicting.

## What This Changes

The PR changes Mattermost group allowlist handling so non-allowlisted group messages are recorded as pending history before returning without a bot response.

## Best Possible Solution

Port the history-only allowlist fallthrough to the current Mattermost nested access flow, return before response dispatch, and add focused regression coverage without adding a new config option.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a Mattermost group message from a sender outside `groupAllowFrom` under `groupPolicy: "allowlist"` reaches the immediate not-allowlisted return before pending history can be recorded.

## Solution Assessment

No, not as patched. The desired history-only fallthrough is the narrow maintainable fix, but this branch must be ported from the old flat access-decision flow to current `ingress` and `commandAccess` handling.
