Keep open. Current main has partial mitigations for the original binary-only concern, but the requested progressive approval-history, pattern-learning, and user-confirmed rule proposal model is still absent and needs security/product design rather than cleanup closure.

## What This Changes

The issue requests progressive exec permissions that evolve from binary+args approvals into user-confirmed generalized rules based on approval history.

## Best Possible Solution

Define a maintainer-approved progressive exec-permission contract that preserves current exact-command and argPattern mitigations while adding audited approval history, safe rule proposals, explicit confirmation, schema/UI/docs support, and regression coverage.
