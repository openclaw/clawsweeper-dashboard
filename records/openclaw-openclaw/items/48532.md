## Summary

Keep open: current main has partial exec-approval hardening, but POSIX allow-always entries still use path-only allowlist semantics and the requested progressive pattern-learning model is not implemented.

## What This Changes

Request a generalized exec permission model that binds allow-always to binary+args and later proposes user-confirmed scoped patterns from approval history.

## Best Possible Solution

Define a first-class exec approval policy design that enforces cross-platform argv-aware rules, preserves existing path-only allowlists deliberately, and makes automatic pattern generalization an auditable opt-in flow.

## Reproduction Assessment

Yes, from source inspection: POSIX allow-always persistence records path-only entries for ordinary binaries, and POSIX matching ignores `argPattern` even when present. I did not run a live approval flow because this review was read-only.

## Solution Assessment

Unclear. The direction is valid security work, but the proposed automatic inference model is broader than a safe one-shot fix; maintainers should first settle the compatibility, precedence, review UI, and audit model.
