## Summary

Keep open. Current main has partial safeguards for the original over-broad exec trust concern, but the broader progressive permission model, approval-history learning, deny-before-allow policy, and user-confirmed generalized rule flow are still unimplemented security/product work.

## What This Changes

The issue requests progressive exec permissions that move from exact command approval toward user-confirmed inferred/generalized policy rules.

## Best Possible Solution

Keep the current exact/durable and argPattern safeguards, then have maintainers define or split the remaining policy design for approval logs, inferred rules, deny-before-allow precedence, dry-run validation, and explicit user-confirmed promotion.
