## Summary

Keep open: current main has several exec-approval hardenings, including exact-command durable trust and interpreter safeguards, but the requested cross-platform argument-scoped permissions plus progressive pattern inference are not fully implemented and need maintainer security/product design.

## What This Changes

The issue requests a generalized exec permission model that moves beyond binary-only allow-always entries toward exact binary+argument matching, approval-log pattern detection, and user-confirmed generalized rules.

## Best Possible Solution

Turn this into a maintainer-owned exec-permission design track: first decide the cross-platform exact-argument policy and compatibility story, then split pattern mining and rule-proposal UI into narrower follow-up issues or PRs.

## Reproduction Assessment

Source-level yes for the remaining broad-match behavior, but I did not perform a live run. Current main still ignores argPattern on non-Windows path matches, while the original interpreter example is partially mitigated by interpreter allow-always hardening and exact-command durable trust.

## Solution Assessment

Unclear as a single implementation path. The direction is valid, but the best maintainable solution is likely a staged design that separates exact argument matching from later approval-log pattern inference and generalized rule UX.
