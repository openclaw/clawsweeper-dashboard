## Summary

Keep open: current main has partial Phase 1 coverage through `argPattern` and exact durable command approvals, but the requested approval log, pattern detector, generalized scoped rules, and policy UX are still not implemented and need maintainer security/product design.

## What This Changes

The issue requests fine-grained exec approval permissions that preserve command arguments, learn repeated approval patterns, and propose scoped rules to reduce over-permissioning and approval fatigue.

## Best Possible Solution

Define a maintainer-approved progressive exec permission contract that covers exact durable approvals, argument/path rules, deny-before-allow precedence, approval-log/audit storage, pattern proposal UX, schema/protocol/UI/docs, and migration behavior across gateway and node paths.

## Reproduction Assessment

Yes, for the remaining issue scope by source inspection: current main has partial argPattern and exact durable approval behavior, but no approval-log clustering, pattern detector, generalized rule proposal UI, or progressive permission workflow. I did not run a live gateway repro because the missing surfaces are clear in source and docs.

## Solution Assessment

No implemented solution is present for the full request. The current partial `argPattern`/durable-command approach is useful, but the best solution needs maintainer-owned security and UX design before adding automatic pattern inference or broad policy generalization.
