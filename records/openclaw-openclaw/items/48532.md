## Summary

Keep open. Current main partially addresses the original binary-only concern with `argPattern`, exact-command durable approvals, and interpreter hardening, but it does not implement the requested progressive policy-learning/review/proposal model; the related open items are narrower and do not supersede this umbrella security design.

## What This Changes

Request to replace broad binary-level exec allow-always with progressive exact-command, argument-pattern, and user-confirmed pattern-generalization permissions.

## Best Possible Solution

Define a maintainer-approved exec permission contract that keeps the current exact-command and argPattern mitigations, then adds auditable approval history, safe rule-generalization proposals, explicit user confirmation, schema/UI/docs support, and focused regression coverage.

## Reproduction Assessment

Yes, for a source-level state check. Current main shows partial argument-aware and exact-command durable approval support, while the requested approval-log clustering and user-confirmed generalized rules are absent from source, docs, schema, and UI.

## Solution Assessment

Unclear as an implementation path. The direction is plausible, but the best fix needs maintainer security/product design before code because it changes core exec permission semantics and introduces new policy surfaces.
