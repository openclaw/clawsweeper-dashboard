## Summary

Keep open. Current main has meaningful partial mitigations for coarse exec allowlists, but it still lacks approval-history clustering, user-reviewed generalized rule proposals, and a settled progressive permission model; adjacent issues and the open policy PR do not cover the full remaining request.

## What This Changes

The issue requests a progressive exec permission model that moves allow-always approvals from binary-only trust toward exact argument matching, learned patterns, and explicit user-confirmed generalized rules.

## Best Possible Solution

Define a maintainer-approved progressive exec-permission contract that keeps the current exact/argPattern mitigations while adding audited approval history, safe rule proposal logic, explicit confirmation UX, schema/docs support, and regression coverage.

## Reproduction Assessment

Yes, for a source-level state check. Current main exposes argPattern, durable exact-command approval, and last-use metadata, while targeted source/docs/UI searches show no approval-history clustering or user-confirmed generalized rule flow.

## Solution Assessment

Unclear as an implementation path. The direction is plausible, but the maintainable fix needs security/product design first because it changes core exec permission semantics, persistence, and user-facing policy surfaces.
