## Summary

Keep open. Current main has solved much of the original Phase 1 binary-plus-arguments concern, but the approval-history logging, inferred/generalized rules, deny-before-allow policy, and user-confirmed promotion flow remain unimplemented security/product work.

## What This Changes

The issue asks OpenClaw exec approvals to progress from binary-only allow-always toward exact command arguments, approval-history pattern inference, and user-confirmed generalized permission rules.

## Best Possible Solution

Keep the current exact/argPattern behavior, then have maintainers define or split the remaining policy design for approval logs, inferred rules, deny-before-allow precedence, and explicit user-confirmed promotion.

## Reproduction Assessment

Not applicable as a feature/security-design issue. Source inspection gives a high-confidence state check: exact command and argPattern support exist, while approval-history inference and generalized rule confirmation do not.

## Solution Assessment

No complete solution is present yet. Closing as implemented would lose the remaining progressive-permission design, and an automated fix would be unsafe without maintainer security/product decisions.
