## Summary

Keep this PR open: the production change is directionally correct for the WebChat `HEARTBEAT_OK` leak, but this external PR only provides mock/unit-test proof and the added regression test does not actually exercise `showOk: true` through the mocked visibility resolver.

## What This Changes

This PR changes Gateway chat heartbeat suppression to hide heartbeat assistant stream output based only on the heartbeat run context and adds a server-chat unit test for `showOk: true`.

## Best Possible Solution

Land a narrow Gateway fix that keeps WebChat heartbeat ACK output hidden regardless of external-channel `showOk`, with a regression test that genuinely exercises the `showOk: true` path and contributor-supplied real behavior proof.

## Reproduction Assessment

Yes, from source inspection: current main resolves WebChat heartbeat visibility and returns `!showOk`, so a heartbeat run with `channels.defaults.heartbeat.showOk: true` can bypass live chat suppression. I did not run a live Gateway/WebChat reproduction because this review is read-only.

## Solution Assessment

Mostly yes for the production code: using the heartbeat run context instead of external-channel visibility is the narrow fix for the `HEARTBEAT_OK` stream leak. The test and merge evidence need tightening before this is the best merge-ready path.
