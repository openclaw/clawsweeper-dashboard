## What This Changes

The issue reports that upgrading from v2026.6.8 to v2026.6.9 leaves configured Groq audio STT unavailable after provider externalization, breaking Telegram and CLI transcription with a missing media provider error.

## Best Possible Solution

Keep Groq externalized, but make the upgrade, doctor, registry refresh, and Gateway paths converge so configured Groq STT either loads correctly after repair or fails visibly with actionable operator guidance without treating failed STT as valid content.

## Reproduction Assessment

Yes, source-reproducible: a configured Groq audio entry with Groq absent from the media provider registry reaches the missing-provider throw, and the reporter supplied live CLI/Gateway logs. I did not run a packaged Gateway/systemd repro in this read-only pass.
