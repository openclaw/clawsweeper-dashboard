## Summary

Keep open. Current main has the Groq external plugin catalog and doctor repair path, but the reported missing installed media provider still falls through to `Media provider not available: groq` and can leave the channel turn without a transcript-specific failure surfaced to the user.

## What This Changes

Issue reports that upgrading to v2026.6.9 breaks existing Groq audio transcription configs because the Groq media provider is now an external plugin that is not installed or clearly hinted during the failing path.

## Best Possible Solution

Keep this issue open and add a bounded repair that turns missing official external media providers into actionable install or `openclaw doctor --fix` guidance while preventing silent empty-transcript channel handling; do not add package-manager installs to Gateway startup.

## Reproduction Assessment

Yes, source-reproducible: with `tools.media.audio.models` selecting `groq` and no installed Groq plugin, the media registry lacks `groq`, `runProviderEntry` throws the reported provider-unavailable error, and channel processing records no transcript output. I did not run the packaged Telegram flow in this read-only review.

## Solution Assessment

No, current main is only a partial solution. Doctor can repair some missing configured plugin installs, but the runtime and CLI failure path still lacks the package-specific hint and channel-safe transcript failure behavior requested here.
