## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Add schema support for the new config key:** `src/config/types.gateway.ts:378-379`
  - body: `invokeTimeoutMs` is added only to the TypeScript config type. The runtime schema for `gateway.nodes` is strict and still allows only browser, pairing, allowCommands, and denyCommands, so a user who copies the PR body's `openclaw.json` example will hit an unrecognized-key config validation failure instead of changing the timeout.
  - confidence: 0.95
- **[P2] Send the resolved timeout to node clients:** `src/gateway/node-registry.ts:141`
  - body: The PR changes only the gateway wait fallback, but the `node.invoke.request` payload is still built with `timeoutMs: params.timeoutMs`. When no per-call timeout is supplied, Swift nodes receive no timeout and fall back to their own 30s default, so the configured gateway default would not reliably allow long-running node commands.
  - confidence: 0.9
- **[P2] Reject invalid configured timeout values:** `src/gateway/server.impl.ts:488-490`
  - body: Startup wiring accepts any numeric config value, including zero, negatives, non-integers, and non-finite numbers. Those values can make invokes time out immediately or violate the advertised millisecond budget; the config schema and runtime handoff should require a positive finite integer.
  - confidence: 0.86
