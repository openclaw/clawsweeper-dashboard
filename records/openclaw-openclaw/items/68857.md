- **[P1] Add schema support for the new config key:** `src/config/types.gateway.ts:378-379`
  - body: `invokeTimeoutMs` is added only to the TypeScript config type. The runtime `gateway.nodes` schema is strict and still rejects this key, so a user who copies the PR body's config example will fail validation instead of changing the timeout.
  - confidence: 0.95
- **[P2] Send the resolved timeout to node clients:** `src/gateway/node-registry.ts:143-144`
  - body: The PR changes the gateway wait fallback, but the `node.invoke.request` payload is still built from `params.timeoutMs`. When no per-call timeout is supplied, Swift nodes receive no timeout and keep their own 30s default, so the configured default will not reliably allow longer node commands.
  - confidence: 0.9
- **[P2] Reject invalid configured timeout values:** `src/gateway/server.impl.ts:488-490`
  - body: Startup wiring accepts any numeric config value, including zero, negatives, fractions, and non-finite numbers. The config schema and runtime handoff should require a positive finite integer so configured invokes do not time out immediately or violate the advertised millisecond budget.
  - confidence: 0.86
