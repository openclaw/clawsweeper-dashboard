## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Add runtime schema support for invokeTimeoutMs:** `src/config/types.gateway.ts:377-378`
  - body: The PR adds the public TypeScript field, but `gateway.nodes` is parsed by a strict zod object that still omits this key. A user following the advertised config would hit an unknown-property error before startup can copy the value.
  - confidence: 0.94
- **[P2] Send the resolved timeout to node hosts:** `src/gateway/node-registry.ts:140-143`
  - body: The fallback is computed only after `node.invoke.request` is sent, while the payload still uses `params.timeoutMs`. When callers omit a per-call timeout, node hosts keep their own 30000 ms default even if the gateway waits longer.
  - confidence: 0.91
- **[P2] Validate configured timeout values:** `src/gateway/server.impl.ts:488-490`
  - body: Startup copies any number into the registry, including `0`, negative, `NaN`, or infinite values. The schema and startup path should require a finite positive bounded integer so configured defaults cannot make invokes time out immediately or behave inconsistently.
  - confidence: 0.88
