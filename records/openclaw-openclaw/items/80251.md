## Summary

## What This Changes

The PR adds generated reset transcript rotation, clears reset compaction metadata, and adds reset/session path regression coverage.

## Best Possible Solution

Revise this PR or an equivalent replacement to rotate all generated reset transcript shapes, clear compaction lineage, preserve custom paths, and then provide branch-live reset proof with green checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a session-store entry with a generated old `sessionFile`, `compactionCount`, and `compactionCheckpoints`, then call `sessions.reset`; current main reuses the old file hint and copies compaction fields forward.

## Solution Assessment

No as written. The code direction is right for UUID/topic/custom cases and compaction cleanup, but the helper should also rotate timestamped fork transcript files instead of falling back to the stale old `sessionFile`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve fork transcript names on reset:** `src/config/sessions/paths.ts:240-260`
  - body: `resolveGeneratedSessionFileSuffix` recognizes bare UUID and `-topic-` generated files, but generated fork transcripts are written as `<timestamp>_<sessionId>.jsonl`. For those sessions this helper returns `undefined`, so reset falls back to the old `sessionFile` and keeps the new session attached to the prior fork transcript. Carry over the existing fork-pattern rewrite behavior before falling back to custom-path preservation.
