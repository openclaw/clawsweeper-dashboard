## Summary

Keep open: this PR targets a real reset bug, but the proposed generated-transcript classifier misses OpenClaw's timestamped fork transcript shape and the supplied real behavior proof is from an equivalent installed hotfix rather than this branch.

## What This Changes

The PR adds a session path helper to rotate generated reset transcript filenames, clears reset compaction fields, and adds path/reset regression tests.

## Best Possible Solution

Land a focused reset fix that rotates every generated transcript shape, clears reset compaction lineage, preserves custom paths, and includes branch-live real behavior proof.

## Reproduction Assessment

Yes from source inspection. Seed a session-store entry with a generated sessionFile plus compactionCount/compactionCheckpoints, then call sessions.reset; current main will preserve those fields through resolveSessionFilePath and the reset entry copy.

## Solution Assessment

No, not yet. The helper boundary is right, but the implementation should also rotate timestamped fork-generated transcript filenames already recognized by transcript and compaction utilities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Rotate forked generated transcript files too:** `src/config/sessions/paths.ts:240-241`
