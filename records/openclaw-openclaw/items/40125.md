## Summary

Keep open. Current main has preflight compaction, after-failure context-overflow guidance, on-demand `/context` diagnostics, and Control UI context pressure UI, but it still lacks the requested one-time proactive warning delivered into persistent chat channels with `/reset` or `/new` guidance before overflow.

## What This Changes

Warn persistent-chat users when a session approaches the active model context window and provide `/reset` or `/new` recovery guidance before overflow.

## Best Possible Solution

Add a generic context-pressure warning helper in the auto-reply/session pipeline that reuses current context usage data, records per-session warning level state, and lets persistent channels deliver concise `/reset` and `/new` recovery guidance through normal reply plumbing.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence current-main evidence that recovery, diagnostics, and Control UI notices exist, but proactive persistent-channel warning delivery does not.

## Solution Assessment

Unclear until maintainers choose the warning policy. The maintainable direction is a shared context-pressure warning surface, not Telegram-only logic or another provider failover path.
