## Summary

Keep open. Current main has overflow classification, preemptive compaction/recovery, a WebChat-only context notice, and `/context` diagnostics, but I did not find the requested proactive per-session warning delivered to Telegram or other persistent chat channels before the session reaches overflow.

## What This Changes

Request to warn users before persistent sessions reach model context limits, give `/reset` and `/new` recovery guidance, and avoid useless fallback to equal-or-smaller context models.

## Best Possible Solution

Add a generic context-pressure warning surface in the auto-reply/session pipeline that reuses fresh session usage and existing reset/new guidance, with channel-safe per-session threshold state and targeted tests for persistent chat delivery.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has recovery, diagnostics, and WebChat notice pieces but not the requested proactive persistent-channel warning.

## Solution Assessment

Unclear until maintainers choose the cross-channel warning policy. The maintainable direction is likely a generic session-pressure helper used by auto-reply/channel delivery, not Telegram-only logic or additional provider failover.
