## Summary

Keep open: current main has reactive context-overflow handling, compaction/retry paths, `/context` reporting, and a Control UI context notice, but it does not send the requested proactive warning into persistent chat surfaces before a provider failure.

## What This Changes

The issue requests proactive per-session context-pressure warnings with `/reset` or `/new` recovery guidance for long-running persistent chats.

## Best Possible Solution

Add a generic session-context pressure warning path that reuses fresh session token snapshots, sends at most one warning per threshold per session on deliverable chat channels, and keeps recovery copy aligned with `/new`, `/reset`, and `/compact`.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection confirms the missing behavior is proactive channel warning rather than a broken documented contract.

## Solution Assessment

Unclear. The direction is reasonable, but the proposal mixes warnings, failure copy, fallback policy, and optional reporting; maintainers should first settle the narrow warning semantics and thresholds.
