## Summary

Keep open: current main has improved context-overflow detection, precheck compaction, and post-failure `/reset`/`/new` guidance, but it does not implement the requested one-time proactive user warnings before a persistent session reaches the context limit.

## What This Changes

Request proactive context-window pressure warnings for persistent chat sessions, with `/reset` or `/new` recovery guidance and context-aware fallback behavior.

## Best Possible Solution

Add a channel-neutral session context-pressure notifier that uses fresh token snapshots, persists per-threshold dedupe state, and emits concise `/reset`/`/new` guidance only when the user can act on it.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main has recovery and compaction paths but lacks the requested proactive warning state and user notification.

## Solution Assessment

No, current main is not the complete solution for the requested UX. The maintainable path is to layer a small warning policy on existing token/session metadata rather than adding Telegram-specific handling or relying only on post-failure overflow messages.
