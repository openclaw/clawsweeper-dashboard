## Summary

Keep open. Current main has preflight compaction, opt-in compaction status notices, Control UI context pressure UI, and /context diagnostics, but I did not find a proactive per-session warning delivered to Telegram or other persistent chat channels with /reset or /new guidance before overflow.

## What This Changes

Request proactive context-limit warnings with /reset or /new recovery guidance for persistent chat sessions.

## Best Possible Solution

Add a generic context-pressure warning helper in the auto-reply/session pipeline that reuses existing session usage and reset/new guidance, persists per-session threshold state, and lets Telegram and other channels deliver it through normal reply plumbing.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has recovery, diagnostics, and Control UI-only notice pieces but not proactive persistent-channel warning delivery.

## Solution Assessment

Unclear until maintainers choose the warning policy. The maintainable direction is a generic session-pressure surface shared by channel delivery, not Telegram-only logic or provider failover changes.
