## Summary

Keep open. Current main has context-overflow precheck/recovery, context-overflow fallback suppression, `/context` diagnostics, and a Control UI context notice, but I did not find a generic proactive warning delivered into Telegram or other persistent chat channels before overflow.

## What This Changes

Request proactive context-pressure warnings and `/reset` or `/new` recovery guidance before long-lived persistent sessions hit model context overflow.

## Best Possible Solution

Add a generic context-pressure warning surface in the auto-reply/session pipeline that reuses fresh session usage and existing reset/new guidance, with channel-safe per-session threshold state and targeted persistent-channel coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has diagnostics, recovery, and a Control UI notice, but not channel-delivered proactive warnings.

## Solution Assessment

Unclear until maintainers choose the cross-channel warning policy. The maintainable direction is likely a generic session-pressure helper used by auto-reply/channel delivery, not Telegram-only logic or provider failover.
