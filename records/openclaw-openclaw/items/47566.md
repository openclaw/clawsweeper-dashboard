## What This Changes

The PR adds an `openclaw plugins init [directory]` CLI command and tests that scaffold `openclaw.plugin.json`, `index.ts`, and `package.json`.

## Best Possible Solution

Land a rebased plugin-init command that follows the canonical plugin id validator, emits current activation and entrypoint starter metadata, documents the public command, and includes redacted scaffold/install proof.

## Reproduction Assessment

Yes. Static PR-head inspection shows the scaffold rejects scoped ids, omits activation metadata, emits a non-canonical starter entrypoint, and adds no public docs for the new command.

## Solution Assessment

No. The feature direction is reasonable, but the patch should be rebased onto current plugin CLI structure and aligned with current plugin authoring contracts before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Allow scoped plugin IDs during scaffolding:** `src/cli/plugins-init.ts:30-31`
  - body: `plugins init` rejects every id containing `/`, so valid scoped ids such as `@scope/name` fail even though the install path accepts and tests that shape. Reuse the canonical plugin-id validator so scaffolded ids match installable ids.
  - confidence: 0.95
- **[P2] Emit explicit startup activation metadata:** `src/cli/plugins-init.ts:72-80`
  - body: The generated manifest omits `activation`, but current plugin docs say every plugin should choose `activation.onStartup` intentionally and omission no longer startup-loads. Generate an explicit activation block, likely `onStartup: false` by default or behind an option.
  - confidence: 0.88
- **[P3] Generate the current plugin entry shape:** `src/cli/plugins-init.ts:36-40`
  - body: The scaffold emits a bare default register function, while current entrypoint docs and the authoring checklist use `definePluginEntry` from `openclaw/plugin-sdk/plugin-entry`. New projects should start from the canonical entry helper rather than a compatibility shape.
  - confidence: 0.78
- **[P3] Document the plugins init command:** `src/cli/plugins-cli.ts:828-836`
  - body: This adds a new public `openclaw plugins init` subcommand without updating the CLI or plugin-authoring docs, so users cannot discover the command, its options, generated files, or intended install flow from the published reference.
  - confidence: 0.84
