The PR adds an `openclaw plugins init [directory]` command that scaffolds `openclaw.plugin.json`, `index.ts`, and `package.json` with unit coverage.

## Best Possible Solution

Land a rebased plugin-init command that reuses the canonical plugin-id contract, emits docs-aligned starter manifests, documents the public CLI surface, and includes redacted real behavior proof.

## Reproduction Assessment

Yes. Static inspection is enough: the PR rejects scoped ids that current main accepts and scaffolds manifests without the activation metadata current docs ask plugin authors to set intentionally.

## Solution Assessment

No. The feature direction is reasonable, but the patch should reuse the canonical validator, generate activation-aware manifests, and add user-facing docs before merge.
