## Summary

Keep open. Current main and the latest shipped release still parse managed browser evaluate input as a parenthesized expression, so declaration statements like `var`, `let`, and `const` remain unsupported without the reported IIFE workaround.

## What This Changes

Add support for statement-style JavaScript bodies such as `var`, `let`, and `const` declarations in browser `act`/CLI evaluate.

## Best Possible Solution

Add compatibility-preserving statement-body support in the browser plugin evaluate implementation with shared parsing/fallback logic and focused regression coverage for page and ref evaluate.

## Reproduction Assessment

Yes, at source level. Current main wraps evaluate input as a parenthesized expression in both managed Playwright paths, and the same wrapper rejects `var`, `let`, and `const` declaration bodies.

## Solution Assessment

Yes. The narrowest maintainable path is to extend the existing browser plugin evaluate implementation and tests, not add a separate `browser.exec` surface or core tool.
