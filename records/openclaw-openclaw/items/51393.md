## Summary

Keep open: current main and the latest release still parse browser evaluate input as a parenthesized expression, so var/let/const statement bodies remain unsupported. The related browser exec/evaluate request at https://github.com/openclaw/openclaw/issues/55598 was closed with this narrower behavior explicitly left as follow-up, and no open PR or canonical duplicate now owns it.

## What This Changes

Request browser act/evaluate to accept JavaScript statement or function bodies with declarations without requiring callers to wrap them in an IIFE.

## Best Possible Solution

Add compatibility-preserving statement/function-body support in the existing browser evaluate path, with focused tests and docs, while keeping current security gates and timeout/navigation guards.

## Reproduction Assessment

Yes. Source inspection and a direct parser check show the current eval("(" + fnBody + ")") wrapper rejects var/let/const statement bodies while the documented IIFE workaround remains expression-shaped.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing evaluate implementation to accept statement/function bodies without adding a separate browser exec command or weakening evaluate security defaults.
