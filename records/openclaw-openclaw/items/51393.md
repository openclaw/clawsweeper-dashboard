## Summary

Keep open: current main and the latest release still parse managed browser evaluate input as a parenthesized expression, so statement bodies with var/let/const still fail. The closed browser exec thread explicitly left this narrower request as follow-up, and the recent FSB prior-art comment supports a focused function-body style repair rather than closing this as solved.

## What This Changes

Request browser evaluate support for JavaScript statement/function-body input such as var, let, and const without requiring callers to wrap code in an IIFE.

## Best Possible Solution

Extend the existing browser evaluate path to accept statement/function-body input while preserving current expression and function compatibility, evaluateEnabled gating, timeout bounds, abort behavior, page/ref handling, batch behavior, and navigation/SSRF guards.

## Reproduction Assessment

Yes. Source inspection plus a direct parser check show current main rejects statement bodies because the managed evaluator wraps input as a parenthesized expression; I did not run a full browser E2E repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is compatibility-preserving statement/function-body support in the existing evaluate implementation, not a separate browser.exec command or a change to evaluate security policy.
