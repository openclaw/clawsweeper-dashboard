## Best Possible Solution

Land the narrow memory-core reset fix after maintainer acceptance of the sqlite-vec state-lifecycle change and required-check status, then let the linked issue close through the merged PR.

## Reproduction Assessment

## Solution Assessment

Yes, the patch is the best bounded fix I found: keep the vec0 table schema when the cached vector store is available, and reset cached vector readiness when that path fails. A narrower fix that only flips `vector.available` after dropping would avoid the prepare crash but would unnecessarily discard the reusable schema/dim state on the normal path.
