## Summary

Keep open. Current main does not implement this access-groups feature, and the latest PR head still has a concrete Discord permission correctness bug: channel-audience checks do not honor guild owners, so legitimate owners can be falsely blocked from DM authorization. The PR is also protected by the `maintainer` label, so it should stay in normal review/repair flow rather than cleanup closure.

## What This Changes

This PR adds top-level `accessGroups` config, Discord `discord.channelAudience` DM authorization, generic `message.senders` expansion helpers, channel integrations, docs, tests, generated baselines, and a changelog entry.

## Best Possible Solution

Keep the feature in review, make the Discord channel-audience permission helper owner-aware, add focused owner coverage, resolve the dirty branch/check state, and preserve the generic access-group boundary unless maintainer review asks for a narrower product shape.

## Reproduction Assessment

Yes. A focused unit test can mock a channel that denies `ViewChannel` to `@everyone`, a guild whose `owner_id` matches the sender, and a member with no granting roles; Discord should allow the owner while this helper currently returns false.

## Solution Assessment

No. The feature direction is maintainable, but the current permission implementation is not the best fix until it matches Discord owner/admin/overwrite semantics; the safer repair is an owner short circuit before role and overwrite calculations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor guild owners in channel audience checks:** `extensions/discord/src/send.permissions.ts:87-88`

Include the guild owner field in the permission calculation and add owner-path coverage.
