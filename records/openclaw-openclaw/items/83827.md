## Best Possible Solution

Land the narrow queued-abort contract if maintainers agree that accepted user requests should be durable once enqueued, with optional Mantis Telegram Desktop proof for extra delivery confidence.

## Solution Assessment

Yes. Carrying abort metadata only on the queued item, and only for `room_event` followups, is the narrow maintainable fix because queue admission has already accepted `user_request` turns.
