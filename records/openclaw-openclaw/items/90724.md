## Summary

Keep open: current main and v2026.6.9 still expose only coarse TUI busy/connection/footer state; internal model progress and byte diagnostics exist, but they are not wired into a live status-line liveness indicator.

## What This Changes

Request adding a live TUI status-line indicator showing whether the model connection is actively exchanging packets or progress so users can tell working from stalled or disconnected.

## Best Possible Solution

Define one stable per-run TUI liveness signal, likely based on semantic model/backend progress using existing diagnostics, and only expose raw packet or byte counters if maintainers explicitly want that contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks a packet, byte, or last-progress status-line indicator.

## Solution Assessment

Unclear. The literal packet-counter proposal is probably not the best permanent contract; a semantic last-progress or active-model-call signal surfaced through the TUI would be more maintainable.
