## Summary

Keep open: current main still has coarse TUI busy/connection states and watchdog notices, but no live status-line packet, byte, or last-model-progress indicator that would answer the reported liveness uncertainty.

## What This Changes

The issue requests a TUI live connection/liveness monitor, possibly packets in/out in the status line, so users can tell whether a model call is still active or stalled.

## Best Possible Solution

Define one stable per-run liveness signal for TUI display, likely based on last semantic model/backend progress rather than raw packet counts, and keep watchdog configuration work separate from this status-line UX.

## Reproduction Assessment

## Solution Assessment
