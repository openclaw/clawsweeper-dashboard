## Summary

Keep open: current main and v2026.6.8 provide coarse TUI run states plus a streaming watchdog notice, but they still do not expose a user-facing packet, byte, or last-model-progress monitor in the status line.

## What This Changes

Request to add a live TUI status-line monitor showing whether the model connection is still exchanging packets or otherwise making progress during long runs.

## Best Possible Solution

Define a compact provider-agnostic model-liveness signal, then surface it in the TUI status line alongside existing run states if maintainers want this UX.
