## What This Changes

The issue requests a live TUI connection/liveness monitor, possibly showing model packet or I/O activity so users can distinguish slow model work from a stalled or disconnected run.

## Best Possible Solution

Define a compact user-facing liveness signal, such as last model progress or diagnostic freshness, and expose it in the TUI status line only after maintainers choose the semantics.
