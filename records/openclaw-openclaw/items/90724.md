## Summary

Keep open: current main and v2026.6.1 provide coarse TUI activity states, elapsed waiting/streaming display, disconnect handling, watchdog warnings, and internal model-call byte/progress diagnostics, but they still do not expose the requested user-facing model connection or packet/last-byte monitor in the TUI status line.

## What This Changes

Request to add a live TUI connection/liveness indicator, possibly showing model packets or traffic activity, so users can tell whether a long-running model call is still active.

## Best Possible Solution

If maintainers want this, reuse the existing model-call progress/byte diagnostics as the backend signal and expose a compact TUI last-progress or liveness indicator rather than a raw provider-specific packet counter.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection confirms current main lacks a user-facing packet, byte, or last-model-progress monitor in the TUI status line.

## Solution Assessment

Unclear: the exact packet-count proposal may not be the best UI, because existing diagnostics already track safer model-call progress signals. The maintainable path is a maintainer UX decision about what liveness signal to surface and how much diagnostic detail belongs in the status line.
