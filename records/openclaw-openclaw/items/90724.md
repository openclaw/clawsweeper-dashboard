## Summary

Keep open: current main and v2026.6.9 still expose only coarse TUI run states plus a delayed streaming watchdog notice, not a live packet/byte/last-progress indicator for model activity.

## What This Changes

Request a TUI status-line liveness indicator, such as packets in/out or recent model traffic, so users can tell whether a long-running model turn is still active.

## Best Possible Solution

Define one stable per-run liveness/progress signal for TUI display, likely based on recent semantic model/backend progress rather than raw packet counts, and keep watchdog tuning as a separate adjacent issue.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does confirm that current main lacks the requested packet/byte/last-model-progress status-line indicator.

## Solution Assessment

Unclear. The literal packet-count proposal is probably not the best permanent contract; a stable semantic liveness/progress signal surfaced through the TUI would be a safer maintainable solution.
