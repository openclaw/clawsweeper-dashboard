## Summary

Keep open: current main and v2026.6.8 still show only coarse TUI activity/connection states plus watchdog notices, not the requested packet, byte, or last-model-progress indicator in the status line.

## What This Changes

## Best Possible Solution

Define a stable per-run liveness signal, likely last model/backend progress rather than raw packets, and render it compactly in the TUI status line after maintainers choose the backend contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a reproduced bug. Source inspection confirms current main lacks a user-facing packet, byte, or last-model-progress monitor in the TUI status line.

## Solution Assessment

Unclear: literal packet counts are probably not the best product surface. A maintainable solution should choose a stable liveness contract from backend events or diagnostics before rendering it in the TUI.
