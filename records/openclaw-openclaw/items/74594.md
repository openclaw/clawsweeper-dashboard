## Summary

Keep open for maintainer product and security review. Current main still lacks the requested optional skill capability metadata, status/JSON propagation, `openclaw skills inspect` command, and docs coverage; the related permission-manifest issues are useful prior art but not a settled replacement for this narrower inspect-only RFC.

## What This Changes

The issue requests optional skill capability metadata plus a CLI inspect command that displays declared tools, files, and exec access without changing runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the inspect-only capability vocabulary and warning model; if accepted, implement typed optional metadata parsing, status propagation, human and JSON inspect output, docs, and focused parser/formatter/CLI tests while preserving no-enforcement wording.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection on current main gives a high-confidence verification path: the requested schema field, status payload, inspect subcommand, and docs are absent.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about vocabulary, user-facing warnings, and overlap with the broader permission-manifest/security issues.
