## Summary

Keep open for maintainer product/security follow-up. Current main still lacks the requested optional skill capability metadata, status exposure, `openclaw skills inspect` command, and docs contract; the related open permission-manifest thread at https://github.com/openclaw/openclaw/issues/12219 is broader prior art rather than a settled replacement for this inspect-only RFC.

## What This Changes

The issue proposes optional skills-first capability declarations for tools/files/exec plus an `openclaw skills inspect` command with warning-only v0 behavior.

## Best Possible Solution

Keep this RFC open until maintainers decide whether to carve an inspect-only v0 out of the broader manifest work; if accepted, implement typed metadata parsing, status propagation, human and JSON inspect output, docs, and focused parser/formatter/CLI tests with explicit non-enforcement wording.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection on current main gives a high-confidence absence check for the requested schema field, status payload, inspect subcommand, and docs coverage.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the vocabulary and how to avoid implying enforcement.
