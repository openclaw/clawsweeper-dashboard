## Summary

Keep open for maintainer product/security follow-up. Current `main` still lacks the requested optional skill capability metadata, capability propagation, `openclaw skills inspect` command, and docs, while #12219 remains broader prior art rather than a settled replacement for this inspect-only RFC.

## What This Changes

The issue proposes an optional skills-first capability manifest plus an inspect command that displays declared tools/files/exec access without changing runtime behavior.

## Best Possible Solution

Keep the RFC open until maintainers choose the skill capability vocabulary and warning model; if accepted, implement typed optional metadata parsing, status propagation, human and JSON inspect output, docs, and focused parser/formatter/CLI tests.
