## Summary

Keep open. Current main and v2026.6.10 still lack the requested optional skill capability metadata and top-level installed-skill inspect command; https://github.com/openclaw/openclaw/pull/95220 is an open partial audit-only permission-metadata candidate for a related supply-chain issue, not a merged implementation of this inspect-first RFC.

## What This Changes

Request to add an optional skill capability manifest plus `openclaw skills inspect <skill-id>` output for declared tool, file, and exec access without runtime enforcement.

## Best Possible Solution

Maintainers should choose one versioned skill permission/capability schema and inspect/status wording, then coordinate implementation with the broader permission-manifest and supply-chain trackers.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 lack both a skill capability declaration schema and a top-level `openclaw skills inspect <skill-id>` command.

## Solution Assessment

Unclear as a final implementation path. The inspect-only slice is plausible, but maintainers still need to choose schema location, capability vocabulary, warning language, and how it relates to the broader permission/enforcement work.
