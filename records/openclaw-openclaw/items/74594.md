## Summary

Keep open for maintainer product and security review. Current main still lacks the requested inspectable skill capability declaration, and related #12219/#10827 overlap with the broader permission-manifest work without settling this narrower inspect-only v0.

## What This Changes

Request an optional skills-first capability manifest plus an `openclaw skills inspect` command that displays tools/files/exec declarations without changing runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the public capability vocabulary and consolidation path; if accepted, implement typed parsing, status propagation, human/JSON CLI output, docs, and focused tests with clear inspect-only warnings.

## Reproduction Assessment

Not applicable as a runtime bug. Static verification is high-confidence: current main lacks the metadata field, parser/status payload, CLI subcommand, and docs coverage requested here.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about core SKILL.md metadata versus ClawHub metadata or a shared security manifest contract.
