Keep open. Current main and the latest release still lack both the skill capability declaration schema and the `openclaw skills inspect <skill-id>` surface, and the related open security trackers are broader coordination items rather than safe superseding issues.

## What This Changes

Requests an inspect-only v0 skill capability manifest with tools/files/exec declarations and no runtime enforcement.

## Best Possible Solution

Adopt one maintainer-approved, versioned skill capability vocabulary that can power inspect output now and later feed consent, grants, enforcement, and audit work without fragmenting the skill ecosystem.
