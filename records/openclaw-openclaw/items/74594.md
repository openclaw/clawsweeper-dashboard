## Summary

Keep open for maintainer product/security follow-up. Current main still lacks a skills capability manifest shape and an `openclaw skills inspect` command, while the related permission/security issues are broader prior art rather than a clearly canonical replacement for this narrower inspect-only RFC.

## What This Changes

This issue requests an optional skills-first capability manifest plus a human-readable `openclaw skills inspect <skill-id>` command, without changing runtime enforcement in v0.

## Best Possible Solution

Keep this RFC open until maintainers settle the skills capability vocabulary and its relationship to the broader permission/security issues. If accepted, implement a typed schema, parser/status propagation, `skills inspect` rendering, docs, and focused tests with clear non-enforcement wording.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/RFC. Static inspection gives a high-confidence verification path: current main lacks the requested schema field, status propagation, CLI command, docs, and tests.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable path depends on maintainer agreement about public permission vocabulary and whether it should remain separate from or fold into #12219/#10827.
