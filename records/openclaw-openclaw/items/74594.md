## Summary

Keep open for maintainer product/security review. Current main and v2026.5.2 still lack the requested skill capability metadata, status payload, `openclaw skills inspect` command, and docs; related #12219 and #10827 are broader permission/security threads rather than a settled implementation of this inspect-only slice.

## What This Changes

Requests an inspect-only v0 skill capability manifest with optional tools/files/exec metadata plus `openclaw skills inspect`, without runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose a small skill capability vocabulary and warning model; then add typed optional metadata, status propagation, `skills inspect` human/JSON output, docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static source/docs inspection on current main and v2026.5.2 is high-confidence: there is no capability field, status payload, inspect subcommand, or docs entry for this RFC.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about vocabulary and warnings that do not imply enforcement.
