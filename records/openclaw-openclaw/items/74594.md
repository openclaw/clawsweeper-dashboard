The issue requests an optional skills-first capability manifest plus an `openclaw skills inspect <skill-id>` command that displays declared tools, file access, and exec expectations without changing runtime enforcement.

## Best Possible Solution

Define a narrow, explicitly non-enforcing skill capability metadata contract and inspection UX only after maintainers choose how it relates to the broader skill permission-manifest security roadmap.
