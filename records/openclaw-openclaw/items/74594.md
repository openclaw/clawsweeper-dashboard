## Summary

Keep open. Current main and the latest release still lack the requested skill capability declaration schema and top-level installed-skill capability inspect command; the related open trackers are broader and should be coordinated, not used to close this narrower inspect-first RFC.

## What This Changes

The issue requests an optional skill capability manifest plus an inspect-only CLI surface that shows declared tools, file access, and exec expectations without changing runtime enforcement.

## Best Possible Solution

Choose one maintainer-approved, versioned skill capability vocabulary and CLI inspection surface, then coordinate it with the broader manifest, grant, enforcement, and ClawHub security work before implementation.

## Reproduction Assessment

Yes, at source level: current skill metadata, CLI registration, docs, and the latest release lack a capability declaration schema and top-level `openclaw skills inspect <skill-id>` behavior. A failing runtime reproduction is not applicable because this is a feature RFC rather than broken existing behavior.

## Solution Assessment

Unclear as a final implementation path: the inspect-only v0 is a reasonable slice, but maintainers still need to choose the public capability taxonomy and warning semantics before code should land.
