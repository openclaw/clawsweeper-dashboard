## Summary

Keep open for maintainer product and security review. Current main still lacks the requested optional skill capability manifest, propagation, docs, and `openclaw skills inspect` command, while the related prior issues are broader permission/security manifest discussions rather than a clearly accepted narrow inspect-only design.

## What This Changes

The issue proposes an inspect-only v0 skill capability manifest: optional skill metadata for tools/files/exec plus an `openclaw skills inspect` command, without runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the capability vocabulary and its relationship to the broader permission-manifest/security work. If accepted, implement a typed optional declaration, propagate it through skill status and human/JSON CLI output, document the inspect-only boundary clearly, and add focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/RFC. Static inspection gives a high-confidence verification path because current main lacks the requested schema field, status propagation, CLI inspect command, and docs coverage.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the public permission vocabulary and whether it should fold into #12219/#10827.
