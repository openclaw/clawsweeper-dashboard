## Summary

Keep open for maintainer product and security review. Current main still lacks the requested optional skill capability declaration and `openclaw skills inspect` surface, while #12219/#10827 overlap with broader permission/security work but do not settle this narrower inspect-only vocabulary.

## What This Changes

Request to add an optional skills-first capabilities metadata block and an `openclaw skills inspect` command that renders declared tools/files/exec access without changing runtime behavior.

## Best Possible Solution

Keep the RFC open until maintainers decide whether this v0 vocabulary belongs in core SKILL.md metadata, ClawHub metadata, or a shared plugin/registry contract; if accepted, implement typed parsing, status propagation, human/JSON CLI output, inspect-only docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection gives a high-confidence verification path: current main lacks the schema field, status payload, `inspect` subcommand, and docs coverage requested here.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about overlap with broader permission-manifest and supply-chain security work.
