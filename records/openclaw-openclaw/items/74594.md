## Summary

Keep open for maintainer product/security follow-up. Current main still lacks the requested skill capability metadata, status exposure, `openclaw skills inspect` command, and docs contract; the related permission-manifest thread is broader prior art rather than a settled replacement for this inspect-only RFC.

## What This Changes

The issue requests an optional skill capability manifest and `openclaw skills inspect` command that make declared tools/files/exec expectations visible without runtime enforcement.

## Best Possible Solution

Keep this RFC open until maintainers decide whether to carve an inspect-only v0 out of the broader manifest work; if accepted, implement typed metadata parsing, status propagation, human and JSON inspect output, docs, and focused parser/formatter/CLI tests with explicit non-enforcement wording.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the capability vocabulary and how to avoid implying enforcement.
