## Summary

Keep open for maintainer product/security review. Current main still lacks the requested skill capability metadata, status exposure, `openclaw skills inspect` command, and docs coverage, while #12219 and #10827 remain broader prior-art threads rather than a settled replacement for this inspect-only RFC.

## What This Changes

The issue proposes optional skill capability metadata for tools/files/exec plus an `openclaw skills inspect` command that displays it without changing runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the v0 capability vocabulary and inspect UX; if accepted, implement typed metadata parsing, status propagation, human/JSON CLI output, docs warnings, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection on current main gives a high-confidence verification path: the requested schema field, status propagation, inspect subcommand, and docs are absent.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the public vocabulary and how to avoid implying enforcement.
