## Summary

Keep open for maintainer design and security review. Current main still lacks a typed skill capability manifest and an `openclaw skills inspect` command, and the related permission-manifest issues are broader prior art rather than a terminal duplicate for this narrower inspect-only RFC.

## What This Changes

The issue proposes an optional skills-first capability manifest and an `openclaw skills inspect` command that displays declared tools, file, and exec access without changing runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the vocabulary and canonical tracking path. If accepted, implement a backward-compatible typed metadata/parser extension, propagate declared capabilities through skill status, add human and JSON CLI rendering, document the inspect-only boundary, and test parser, formatter, and CLI behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/RFC. Static inspection gives a high-confidence verification path: current main lacks the requested schema field, status propagation, docs, tests, and `openclaw skills inspect` command.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the public permission vocabulary and its relationship to the broader open skill security-manifest work.
