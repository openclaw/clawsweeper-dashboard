## Summary

Keep open for maintainer product and security review. Current main and the latest tagged release still lack the requested skill capability declaration shape, status propagation, docs, and `openclaw skills inspect` command; the linked #12219 and #10827 threads are useful prior art, but they are broader permission/security proposals rather than a clearly accepted canonical replacement for this narrower inspect-only RFC.

## What This Changes

The issue requests an optional skills-first capability manifest and an `openclaw skills inspect <skill-id>` command that displays declared tool, file, and exec access without changing runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose whether this narrower inspect-only vocabulary should stand alone or fold into the broader skill permission-manifest work. If accepted, implement a typed optional declaration through skill metadata parsing, status/JSON output, human CLI rendering, and docs with explicit no-enforcement wording and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection is high confidence: current main lacks the requested schema field, status payload, `openclaw skills inspect` command, and docs coverage.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible small first step, but the best maintainable solution depends on maintainer agreement about the public permission vocabulary and its relationship to #12219/#10827.
