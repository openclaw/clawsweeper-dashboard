## Summary

Keep open for maintainer product/security review. Current main still lacks the requested skills capability manifest and `openclaw skills inspect` surface, while the linked prior issues are broader security/enforcement proposals rather than a terminal replacement for this inspect-only RFC.

## What This Changes

The issue proposes an optional skills-first capability manifest for tools, files, and exec plus an `openclaw skills inspect` command, without runtime enforcement in v0.

## Best Possible Solution

Keep the RFC open until maintainers choose the skill capability vocabulary and its relationship to the broader security-manifest threads. If accepted, implement it as a typed parser/schema, status propagation, human and JSON CLI rendering, docs with explicit inspect-only wording, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection gives a high-confidence verification path: current main lacks the requested schema field, status propagation, CLI command, docs, and tests.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about public permission vocabulary and whether it should fold into the broader open skill security work.
