## Summary

Keep open. Current main still lacks the requested optional skill capability metadata, capability propagation, and `openclaw skills inspect` command; the related permission-manifest threads are useful prior art but remain broader design discussions rather than a settled replacement for this inspect-only RFC.

## What This Changes

The issue requests an optional skills-first capability manifest and `openclaw skills inspect` command that displays tools/files/exec expectations without changing runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the canonical skill capability vocabulary and warning model; if accepted, implement typed optional metadata parsing, status propagation, human/JSON inspect output, docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection of current main gives a high-confidence verification path: the requested schema field, status propagation, inspect subcommand, docs, and tests are absent.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the manifest vocabulary and how to avoid implying enforcement.
