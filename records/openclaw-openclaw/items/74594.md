## Summary

Keep open. Current main and v2026.6.10 still lack the requested optional skill capability manifest and top-level installed-skill inspect command; https://github.com/openclaw/openclaw/pull/95220 is an overlapping metadata-visibility PR, but it does not fully own the inspect-first CLI slice and is still open.

## What This Changes

Request to add an inspect-only skill capability manifest and `openclaw skills inspect <skill-id>` command without changing runtime enforcement.

## Best Possible Solution

Choose one maintainer-approved, versioned audit-only skill capability vocabulary and inspection/status surface, with explicit declared-not-enforced wording, then coordinate it with the broader permission and enforcement trackers.

## Reproduction Assessment

Yes, at source level: current main lacks a skill capability metadata field, parser support, and the requested top-level installed-skill inspect command. The same gap is visible in the latest release tag.

## Solution Assessment

Unclear as a final implementation path: the inspect-only v0 is a plausible first slice, but maintainers still need to choose the public schema, command/output wording, and relationship to the overlapping permission-manifest PR.
