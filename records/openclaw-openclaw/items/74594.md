## Summary

Keep open for maintainer product and security follow-up. Current main still lacks the requested optional skill capability metadata, status propagation, `openclaw skills inspect` command, and docs, while #12219 remains a broader permission-manifest track rather than a settled replacement for this narrower inspect-only RFC.

## What This Changes

The issue requests an optional skills-first capability manifest plus one inspect command that displays declared tools, file, and exec access without changing runtime behavior.

## Best Possible Solution

Keep the RFC open until maintainers choose the capability vocabulary and warning model; if accepted, implement typed optional metadata parsing, status propagation, human/JSON inspect output, docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection on current main gives a high-confidence verification path: the requested schema field, status payload, inspect subcommand, and docs are absent.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about vocabulary, warning semantics, and whether this should remain separate from #12219.
