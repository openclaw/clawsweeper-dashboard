## Summary

Keep open for maintainer product and security design. Current main still lacks the requested optional skill capability declaration and `openclaw skills inspect` surface, while the related #12219 and #10827 threads are broader and do not clearly supersede this narrower inspect-only RFC.

## What This Changes

The issue proposes an optional skills-first capability manifest plus an `openclaw skills inspect` command that reports tools, files, and exec declarations without changing runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the shared skill capability vocabulary; if accepted, implement typed optional parsing, status propagation, human/JSON CLI inspect output, explicit inspect-only docs, and focused parser/CLI/formatter tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection provides a high-confidence verification path: current main lacks the metadata field, status payload, inspect subcommand, and docs for declared skill capabilities.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable path depends on maintainer agreement about overlap with broader permission-manifest and supply-chain security RFCs.
