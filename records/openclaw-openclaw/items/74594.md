## Summary

Keep open for maintainer product and security review. Current main still lacks the requested optional skill capability metadata, status propagation, `openclaw skills inspect` command, and docs coverage; related permission-manifest threads are useful prior art but do not replace this narrower inspect-only RFC.

## What This Changes

Request an inspect-only v0 for optional skill capability manifests that declare tools, files, and exec access without runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the minimal skills-first vocabulary and warning model; if accepted, implement typed optional metadata parsing, status/JSON propagation, human and JSON inspect output, docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection against current main shows the requested schema, status payload, inspect subcommand, and docs are absent.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about security vocabulary and whether this belongs in core skill metadata, ClawHub/plugin surfaces, or a broader permission-manifest track.
