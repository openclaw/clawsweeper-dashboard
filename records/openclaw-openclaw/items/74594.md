## Summary

Keep open. Current main still lacks a skill capability declaration schema and `openclaw skills inspect <skill-id>` surface; the related open trackers are broader and should be coordinated, but they do not safely supersede this narrower inspect-first RFC.

## What This Changes

The issue requests an optional skill capability manifest plus a human-readable inspect command, without changing runtime enforcement in the first version.

## Best Possible Solution

Adopt one maintainer-approved, versioned skill capability declaration vocabulary that can power inspect output now and later feed consent, grants, enforcement, and audit work without fragmenting the skill ecosystem.

## Reproduction Assessment

Yes at source level: current skill metadata, CLI registration, and docs show no capability declaration or top-level capability inspect command. A failing runtime reproduction is not applicable because this is a feature RFC rather than a broken existing contract.

## Solution Assessment

Unclear as an implementation path: the inspect-only v0 is a reasonable first slice, but maintainers must first approve the public capability taxonomy and coordinate it with the broader permission-manifest and enforcement trackers.
