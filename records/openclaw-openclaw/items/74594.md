## Summary

Keep open for maintainer product and security review. Current main still lacks optional skill capability declarations and an `openclaw skills inspect` surface, and the related #12219/#10827 threads overlap with broader permission/security design without resolving this narrower inspect-only RFC.

## What This Changes

RFC proposes an optional skills-first capability manifest plus one inspect-only CLI command for declared tools/files/exec access.

## Best Possible Solution

Keep the RFC open until maintainers choose the public capability vocabulary; if accepted, implement typed parsing, status propagation, human/JSON CLI output, inspect-only warnings, docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection gives a high-confidence verification path: current main lacks the schema field, status payload, `inspect` subcommand, and docs coverage requested here.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the permission vocabulary and its relationship to broader supply-chain security work.
