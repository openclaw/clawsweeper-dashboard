## Summary

Keep open. Current main and v2026.6.8 still lack skill capability metadata parsing and a top-level installed-skill capability inspect command; the remaining work is security/product API design that overlaps broader permission trackers.

## What This Changes

Request a skills-first, inspect-only capability manifest v0 plus `openclaw skills inspect <skill-id>` to display declared tools/files/exec access without runtime enforcement.

## Best Possible Solution

Choose one maintainer-approved, versioned skill capability vocabulary and CLI inspection surface, then coordinate it with broader manifest, grant, enforcement, and ClawHub security work before implementation.

## Reproduction Assessment

Yes, at source level: current skill metadata, CLI registration, docs, and the v2026.6.8 release lack a capability declaration schema and top-level `openclaw skills inspect <skill-id>` behavior.

## Solution Assessment

Unclear as a final implementation path: the inspect-only v0 is a reasonable slice, but maintainers still need to choose the public capability taxonomy, schema location, and warning semantics before code should land.
