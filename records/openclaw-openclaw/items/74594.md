## Summary

Keep open for maintainer design and security review. Current main still has skill metadata, status, CLI, and docs surfaces, but not a declared skill capability manifest or an `openclaw skills inspect` command; the related prior issues are useful context but are broader than this inspect-only RFC.

## What This Changes

The issue proposes an optional skills-first capability manifest and an `openclaw skills inspect` command that displays declared tools, file, and exec access without runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the capability vocabulary and decide how it relates to #12219 and #10827. If accepted, implement it as a typed skill metadata/parser extension, propagate declared capabilities through skill status, add human and JSON CLI rendering, document the inspect-only boundary, and cover parser/formatter/CLI behavior with focused tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/RFC. Static inspection gives a high-confidence verification path: current main lacks the requested schema field, status propagation, docs, tests, and `openclaw skills inspect` command.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the public permission vocabulary and whether it should fold into the broader open skill security-manifest work.
