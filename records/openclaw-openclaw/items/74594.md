## Summary

Keep open for maintainer product and security direction. Current main still has only load-time skill metadata and status/list/info/check surfaces; it does not implement an optional skill capability declaration, capability-aware status payload, or the proposed inspect-only command, and the linked prior issues are broader rather than a settled replacement for this narrower RFC.

## What This Changes

The issue requests an inspect-only v0 skill capability manifest plus a CLI command to show declared tools, file access, and exec expectations without changing runtime behavior.

## Best Possible Solution

Keep this RFC open until maintainers choose the v0 capability vocabulary and UX contract; if accepted, implement typed optional metadata parsing, status propagation, `skills inspect` human/JSON output, docs, and focused parser/formatter/CLI tests while preserving the existing untrusted-skill warning.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection on current main gives a high-confidence verification path: the requested schema field, status propagation, inspect subcommand, and docs are absent.

## Solution Assessment

Unclear pending maintainer decision. The inspect-only v0 is a plausible narrow first step, but the best maintainable implementation depends on agreed field names and wording that avoids implying enforcement.
