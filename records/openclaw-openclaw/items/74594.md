## Summary

Keep open for maintainer product/security review. Current main still lacks the requested optional skill capability manifest and `openclaw skills inspect` surface, and the related prior issues are broader permission/security proposals rather than a settled replacement for this narrow inspect-only RFC.

## What This Changes

The issue proposes optional skills-first capability metadata for tools/files/exec plus `openclaw skills inspect` output, without runtime enforcement in v0.

## Best Possible Solution

Keep the RFC open until maintainers choose the skill capability vocabulary and warning model; if accepted, implement typed optional metadata parsing, status propagation, human/JSON inspect output, docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection on current main gives a high-confidence verification path: the requested schema field, status propagation, inspect subcommand, and docs are absent.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about field names, scope, and no-enforcement wording.
