## Summary

Keep open for maintainer product and security review. Current main still lacks the requested optional skill capability declaration and `openclaw skills inspect` surface, and the related #12219/#10827 threads overlap with broader permission/security work without settling this narrower inspect-only vocabulary.

## What This Changes

Request an optional skills-first capability manifest in skill metadata plus an `openclaw skills inspect` command that displays declared tools/files/exec access without runtime enforcement.

## Best Possible Solution

Keep the RFC open until maintainers choose the public capability vocabulary and surface; if accepted, implement typed parsing, status propagation, human/JSON CLI output, inspect-only docs, and focused parser/formatter/CLI tests.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection gives a high-confidence verification path: current main lacks the schema field, status payload, `inspect` subcommand, and docs coverage requested here.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about overlap with broader permission-manifest and supply-chain security work.
