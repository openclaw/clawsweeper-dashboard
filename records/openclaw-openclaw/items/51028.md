## Summary

Keep open. Current main still lacks the requested semantic activity contract, activity reason, and parent/child rollup; the Sessions gateway contract and Control UI still order rows from raw `updatedAt`. The split sub-agent label slice has been closed as implemented, but that does not cover the central sorting request.

## What This Changes

Request sorting the Sessions panel by semantic last meaningful activity, with reason labels and parent/child rollup, instead of raw update/message time.

## Best Possible Solution

Add an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, and a concise reason; then sort/render the Sessions panel from that contract with defined child rollup semantics.

## Reproduction Assessment

Yes, at source level: current main has no meaningful/visible activity fields or reason label, and both gateway selection plus Control UI live updates sort by `updatedAt`. I did not run a live heartbeat-displacement scenario.

## Solution Assessment

Yes, the requested gateway-backed semantic activity contract is the maintainable direction. A UI-only sort change would not suppress heartbeat/internal noise, explain why a row moved, or roll child activity into parent rows.
