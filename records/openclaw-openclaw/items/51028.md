## Summary

Keep open. Current main still lacks the requested semantic activity timestamp, activity reason, and parent/child activity rollup; the gateway contract and Control UI continue to expose and sort session rows by raw `updatedAt`.

## What This Changes

The issue asks for the Sessions panel to sort by semantic last meaningful activity, show why a row moved, and roll child activity up to parent sessions instead of relying on raw message/update time.

## Best Possible Solution

Add an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, and a concise reason, then sort and render the Sessions panel from that contract with defined child rollup semantics.

## Reproduction Assessment

Yes, at source level: current main lacks the requested meaningful/visible activity fields and reason label, and both gateway selection plus Control UI live updates sort by raw `updatedAt`. I did not run a live heartbeat-displacement scenario.

## Solution Assessment

Yes, the gateway-backed semantic activity contract is the maintainable direction. A UI-only sort change would not suppress heartbeat/internal noise, explain why a row moved, or roll child activity into parent rows.
