## Summary

Keep open: current main and the latest release still sort session rows by raw `updatedAt`, while the existing session lifecycle already distinguishes human interaction from system/background writes. The remaining work is a product/API decision for the meaningful-activity projection, reason taxonomy, and parent/child rollup.

## What This Changes

Request to make the Control UI sessions panel sort by a canonical last-meaningful-activity timestamp, with visible reason labels and child-session rollup instead of raw last-message or update time.

## Best Possible Solution

Define a gateway-owned meaningful-activity projection for session rows, then sort/render the Control UI from that projection with focused coverage for heartbeat noise, visible reason labels, and child-session rollup.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.6` sort sessions by `updatedAt`, while lifecycle code preserves `lastInteractionAt` separately for system events.

## Solution Assessment

Unclear until maintainers choose the activity contract. The maintainable path is a gateway/protocol row projection plus UI sorting, not ad hoc inference inside the sessions table alone.
