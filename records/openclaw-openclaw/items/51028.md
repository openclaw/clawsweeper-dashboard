## Summary

Keep open: current main and the latest release still sort and limit Sessions rows by `updatedAt`, while the requested meaningful-activity timestamp, reason label, and parent/child rollup contract are not implemented. The remaining work needs maintainer product/API direction, especially after the latest comment raised write-amplification concerns with a persisted sort key.

## What This Changes

The issue requests that the Sessions panel order rows by last meaningful human-facing activity, show why a row moved, and roll relevant child-session activity up to parent sessions.

## Best Possible Solution

Define a gateway-owned activity projection with explicit reason codes and parent/child rollup semantics, preferably evaluating the read-time capped-tail approach before adding any persisted sort key, then have the Control UI sort and explain rows from that contract.

## Reproduction Assessment

Yes, source-reproducible: current main initializes and renders Sessions sorting through `updatedAt`, and the gateway row contract lacks `lastMeaningfulActivityAt` or an activity-reason field. I did not run a live UI scenario because this review is read-only.

## Solution Assessment

Unclear as specified: the user problem is valid, but a persisted `lastMeaningfulActivityAt` may not be the best implementation because of the write-amplification concern raised in the latest comment. The best path is a maintainer-approved activity taxonomy and projection strategy before code changes.
