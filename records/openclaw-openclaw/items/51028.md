## Summary

Keep open: current main still orders and filters Sessions rows by stored `updatedAt`, and the requested meaningful-activity projection, reason label, and parent/child rollup semantics are not implemented; the split label/readability issue was closed separately and does not cover this ordering work.

## What This Changes

Request to make the Sessions panel sort by meaningful session activity, show why a row moved, and roll child-session activity up to the parent instead of relying on raw message or `updatedAt` timestamps.

## Best Possible Solution

Define a gateway-owned activity projection with explicit reason codes and parent/child rollup semantics, then expose it through `sessions.list` for Control UI sorting without overloading `updatedAt`.

## Reproduction Assessment

Yes. Source inspection shows current main still sorts and filters Sessions rows by `updatedAt`, with no meaningful-activity timestamp or reason field; no live UI run was performed in this read-only review.

## Solution Assessment

Unclear. The user problem is valid, but the proposed persisted `lastMeaningfulActivityAt` may not be the best implementation after the write-amplification concern, so maintainers should decide the activity taxonomy and projection strategy first.
