## Summary

Keep open: current main and the latest release still expose and sort the Sessions panel by `updatedAt`, while the requested `lastMeaningfulActivityAt`/reason-label/child-rollup model is not implemented. The remaining work is a maintainer product/API decision, not a stale cleanup close.

## What This Changes

The issue requests that the Sessions panel sort by last meaningful activity, show why a row moved, and roll child-session activity up to the parent instead of relying on raw session update timestamps.

## Best Possible Solution

Define a gateway-owned activity projection with explicit timestamp fields, activity reason codes, and parent/child rollup semantics, then have the Control UI sort and explain rows from that contract.

## Reproduction Assessment

Yes, source-reproducible: current `sessions.list` and the Sessions view sort by `updatedAt`, and the row contract lacks `lastMeaningfulActivityAt` or an activity reason field. I did not run a live UI scenario because this review is read-only.

## Solution Assessment

No, not as an automated bug fix yet: the requested direction is sensible, but the best fix requires a maintainer-approved activity taxonomy, durable gateway projection, and UI contract before code changes.
