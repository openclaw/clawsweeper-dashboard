## Summary

Keep open: current main still exposes and orders Sessions rows by raw `updatedAt`, and the requested semantic activity timestamp, reason label, and parent-child rollup contract are still missing. The split-out readable-label work is already closed at https://github.com/openclaw/openclaw/issues/51030, but that does not cover this issue's ordering model.

## What This Changes

Request a Sessions panel and gateway session contract that sorts by last meaningful activity, suppresses heartbeat/background noise, rolls child activity up to parents, and explains the ordering reason in each row.

## Best Possible Solution

Define an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, concise reason, and parent-child rollup semantics, then update Sessions panel ordering and row copy to use that contract.

## Reproduction Assessment

Yes. Source inspection on current main shows the gateway/session row contracts expose only `updatedAt` for list ordering, and both gateway and Control UI live updates compare rows by that field.

## Solution Assessment

Unclear until maintainers settle the event taxonomy and rollup semantics. The maintainable direction is an additive gateway/protocol contract; a UI-only sort would not solve pagination or live-update ordering.
