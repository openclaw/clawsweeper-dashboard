## Summary

Keep open. Current main still lacks the requested semantic activity timestamp, activity reason, and parent-child activity rollup; the gateway contract and Control UI continue to expose and sort session rows by raw `updatedAt`.

## What This Changes

The issue requests a Sessions panel activity model that sorts by semantic `lastMeaningfulActivityAt`, suppresses heartbeat/internal churn, rolls child activity into parent sessions, and shows an activity reason.

## Best Possible Solution

Define an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, and a concise reason, then have the Sessions panel sort/render from that contract with defined child rollup semantics.

## Reproduction Assessment

Yes, at source level: current main lacks the requested meaningful/visible activity fields and reason label, and both gateway selection plus Control UI live updates sort by raw `updatedAt`. I did not run a live heartbeat-displacement scenario.

## Solution Assessment

Yes, the proposed gateway-backed semantic activity contract is the maintainable direction. A UI-only sort change would not classify event meaning, suppress heartbeat noise, explain row movement, or roll child activity into parents.
