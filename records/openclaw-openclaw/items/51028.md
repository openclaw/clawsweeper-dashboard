## Summary

Keep open. Current main still lacks the requested semantic activity contract, activity reason, and parent/child rollup; the Sessions gateway contract and Control UI still expose and sort from raw `updatedAt`, while the split sub-agent label concern is already handled separately.

## What This Changes

Requests Sessions panel ordering by semantic last meaningful activity, with visible reason labels and parent-child activity rollup instead of raw message/update timestamps.

## Best Possible Solution

Add an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, concise reason labels, and explicit child rollup semantics, then sort/render the Sessions panel from that contract.

## Reproduction Assessment

Yes, at source level: current main has no meaningful/visible activity fields or activity reason, and both gateway selection plus Control UI live updates sort by `updatedAt`. I did not run a live heartbeat-displacement scenario.

## Solution Assessment

Yes, the requested gateway-backed semantic activity contract is the maintainable direction. A UI-only sort change would not suppress heartbeat/internal noise, explain why a row moved, or roll child activity into parent rows.
