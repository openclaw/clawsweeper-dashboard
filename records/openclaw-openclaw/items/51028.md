## Summary

Keep open: current main still sorts and displays Sessions panel activity from raw `updatedAt`, while the requested semantic activity timestamp, reason label, and parent-child rollup are not implemented. The related raw-label concern was split out and closed separately, but that does not cover this ordering/activity model request.

## What This Changes

The issue requests a gateway-backed Sessions panel activity model that sorts rows by last meaningful activity, explains why a row moved, and rolls child activity up to the parent session.

## Best Possible Solution

Define an additive gateway session activity contract with raw event time, meaningful activity time, visible conversation time, concise reason, and parent-child rollup semantics, then update Sessions panel ordering and row copy to use that contract.

## Reproduction Assessment

Yes. Source inspection shows the gateway contract, list selection, websocket row updates, tests, and UI sorting/rendering still use `updatedAt`, while the requested semantic activity fields and reason label are absent.

## Solution Assessment

Yes, the maintainable direction is an additive gateway-backed activity contract rather than a UI-only sort. The exact event taxonomy and child rollup semantics still need maintainer product judgment before implementation.
