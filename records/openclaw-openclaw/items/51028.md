## Summary

Keep open. Current main still exposes and sorts the Sessions panel by raw session-row `updatedAt`, while the requested semantic activity fields, activity reason, and parent rollup sort key are not present.

## What This Changes

The issue requests sorting the Sessions panel by semantic last meaningful activity, excluding heartbeat/internal churn and rolling child activity up to parent sessions.

## Best Possible Solution

Add an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, and a concise reason; sort and render the Sessions panel from that contract, roll child activity into parent rows, and leave the already-split label work to #51030.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a session touched only by a heartbeat/background write can receive a newer `updatedAt` than a real conversation, and current `sessions.list` plus the Control UI sort by that raw `updatedAt`.

## Solution Assessment

Yes. The maintainable solution is an additive gateway/UI contract; a UI-only sort tweak would still lack the semantic timestamp, activity reason, and child rollup needed to distinguish meaningful activity from bookkeeping.
