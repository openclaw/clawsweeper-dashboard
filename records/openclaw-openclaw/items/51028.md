## Summary

Keep open: current main and v2026.6.10 still order and filter Sessions rows by stored `updatedAt`; the requested meaningful-activity projection, reason label, and parent/child rollup semantics remain unimplemented and need maintainer product/API direction.

## What This Changes

The issue asks for the Sessions panel to sort by meaningful user-visible activity, separate that from raw events, roll child activity up to parent sessions, and show why a row moved.

## Best Possible Solution

Define a gateway-owned activity projection with explicit reason codes and parent/child rollup semantics, then expose it through `sessions.list` so the Control UI can sort and explain rows without overloading `updatedAt`.

## Reproduction Assessment

Yes. Source inspection shows current main still sorts and filters Sessions rows by `updatedAt`, with no meaningful-activity timestamp or reason field in the session row shape; I did not run a live UI scenario because this review is read-only.

## Solution Assessment

Unclear as specified. The user problem is valid, but the proposed persisted `lastMeaningfulActivityAt` may not be the best implementation after the write-amplification concern; maintainers should choose the activity taxonomy and projection strategy first.
