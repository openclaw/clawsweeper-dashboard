## Summary

Keep open: current main still lacks a semantic Sessions activity contract, and the Sessions panel continues to sort, patch, and render rows from raw `updatedAt`. The related raw-label work was split to https://github.com/openclaw/openclaw/issues/51030 and closed, but that does not cover meaningful-activity timestamps, reason labels, or parent-child rollup ordering.

## What This Changes

The issue asks for the Sessions panel to sort by last meaningful activity, explain the activity reason in each row, and roll child-session activity into parent-session ordering.

## Best Possible Solution

Define an additive gateway session activity contract with raw event time, meaningful activity time, visible conversation time, concise reason, and parent-child rollup semantics, then update Sessions panel ordering and row copy to use it.

## Reproduction Assessment

Yes. Source inspection shows current main still uses `updatedAt` for gateway selection, websocket row patches, UI sorting, and row rendering while the requested semantic activity fields are absent.

## Solution Assessment

Yes, the maintainable direction is an additive gateway-backed activity contract rather than a UI-only sort. The exact event taxonomy and child rollup behavior still need maintainer product judgment before implementation.
