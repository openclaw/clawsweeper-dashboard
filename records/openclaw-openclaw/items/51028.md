## Summary

Keep open: current main and the latest release still sort session rows by `updatedAt`, while existing lifecycle docs/tests already distinguish human interaction timestamps from noisy row mutations. The remaining work is a product/API decision for a canonical meaningful-activity projection, reason label, and child rollup.

## What This Changes

Request to make the Control UI sessions panel order sessions by human-meaningful activity, with reason labels and parent/child rollup, instead of raw updated/message timestamps.

## Best Possible Solution

Define a gateway-owned meaningful-activity projection for session rows, likely using `lastInteractionAt` as one input, then sort/render the Control UI from that projection with focused tests for heartbeat noise and child-session rollup.

## Reproduction Assessment

Yes, from source inspection: current main and `v2026.6.6` sort session rows by `updatedAt`, while docs/tests distinguish `lastInteractionAt` from noisy row mutations. I did not run a browser scenario because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the activity contract. The maintainable path is a gateway/protocol row projection plus UI sorting, not ad hoc inference in the sessions table alone.
