## Summary

Keep open: current main and the latest release still sort and render Sessions rows from raw `updatedAt`, with no semantic activity timestamp, reason label, or parent-child rollup contract. The related raw-label issue at https://github.com/openclaw/openclaw/issues/51030 was closed as implemented, but that does not cover this issue's meaningful-activity ordering request.

## What This Changes

Request that the Sessions panel order sessions by semantic last meaningful activity, explain why rows moved, and roll child-session activity up to parents instead of relying on raw message or store-update timestamps.

## Best Possible Solution

Define an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, concise reason, and parent-child rollup semantics, then update Sessions panel ordering and row copy to use it.

## Reproduction Assessment

Yes. Source inspection on current main shows the gateway/session row contracts have only `updatedAt`, and the gateway plus UI sort and live-reorder rows by that field while the requested semantic activity fields are absent.

## Solution Assessment

Yes, the maintainable direction is an additive gateway-backed activity contract rather than a UI-only sort. The exact event taxonomy and child rollup behavior still need maintainer product judgment before implementation.
