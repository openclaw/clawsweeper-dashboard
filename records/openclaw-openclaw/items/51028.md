## Summary

Keep open: current main still uses raw session `updatedAt` for Sessions panel recency, filtering, live row ordering, and display. The related raw sub-agent label concern has been split to a closed issue, but the requested meaningful-activity timestamp, reason label, and parent-child rollup model are still missing.

## What This Changes

The issue requests a Sessions panel activity model that sorts by last meaningful activity, suppresses heartbeat/internal churn, rolls child activity up to parent sessions, and shows a concise reason label in each row.

## Best Possible Solution

Define an additive gateway session activity contract with raw event time, meaningful activity time, visible conversation time, concise reason, and parent-child rollup semantics, then update Sessions panel ordering and row display to use that contract.

## Reproduction Assessment

Yes. Source inspection shows the gateway contract, list selection, websocket row updates, tests, and UI rendering still use updatedAt while the requested semantic activity fields and reason label are absent.

## Solution Assessment

Yes directionally: the maintainable solution is an additive gateway-backed activity contract rather than client-side inference. The exact event taxonomy and child rollup semantics still need maintainer product judgment before implementation.
