## Summary

Keep open. The related sub-agent label slice was split into a closed implemented issue, but current main still lacks the semantic activity fields, activity reason, and parent rollup needed for this request, and the gateway plus Control UI still order session rows by raw `updatedAt`.

## What This Changes

The issue requests that the Sessions panel sort and explain sessions by semantic last meaningful activity, with child-session activity rolled up, instead of relying on raw session update time.

## Best Possible Solution

Add an additive gateway-backed session activity contract with raw event time, meaningful activity time, visible conversation time, and a concise reason; sort and render the Sessions panel from that contract, roll child activity into parent rows, and keep the already-split label work in the closed label issue.

## Reproduction Assessment

Yes, at source level: current main has no `lastMeaningfulActivityAt` or `lastVisibleConversationAt` fields, and both `sessions.list` plus live Control UI updates sort by `updatedAt`. I did not establish a live heartbeat displacement trace.

## Solution Assessment

Yes, the maintainable direction is an additive gateway/UI activity contract; a UI-only sort change would still lack the semantic timestamp, activity reason, and parent-child rollup requested here.
