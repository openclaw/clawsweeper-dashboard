- **[P1] Keep replayed user text out of trusted system events:** `src/gateway/server-pending-messages.ts:33-41`
  - body: `item.prompt` is channel/user input, but this builds an `enqueueSystemEvent` payload; current system events are rendered as trusted `System:` lines on the next turn. Replay through an untrusted user/queue path or quote it behind an explicit untrusted boundary so restart recovery cannot elevate user instructions.
  - confidence: 0.9

  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. The repository storage contract requires runtime queues/state in SQLite with doctor-owned migration, so this adds an unmigrated parallel source of truth for prompts and routing state.
  - confidence: 0.88
- **[P1] Rebase replay scheduling onto current gateway startup:** `src/gateway/server-startup.ts:193-197`
  - body: The branch schedules replay in `src/gateway/server-startup.ts`, but current main has moved post-ready startup work into `server-startup-post-attach.ts` and `server-runtime-services.ts`, and GitHub reports this branch as conflicting. The replay hook has to be rebuilt on the current startup lifecycle before review can validate it.
  - confidence: 0.86
- **[P2] Do not discard restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:110-116`
  - body: A restart that takes longer than five minutes silently drops the only persisted queue snapshot, which reintroduces the message-loss bug during manual downtime or repeated failed starts. Recovery should be durable until consumed or governed by a maintainer-approved expiry policy.
  - confidence: 0.82
