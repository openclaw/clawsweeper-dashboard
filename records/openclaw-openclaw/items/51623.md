  - body: `item.prompt` comes from queued channel user messages, but this replay path embeds it in `eventText` and sends it through `enqueueSystemEvent`; current main renders those entries as trusted `System:` lines on the next turn.   - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. The repository storage contract requires runtime queues/state to use SQLite with an owned schema or migration path, so this sidecar would create a second durable source of truth for upgrades and recovery.
  - confidence: 0.95

  - body: The fixed five-minute staleness check discards the persisted queue for any longer shutdown, slow reboot, or repeated failed restart. Since the feature is supposed to prevent restart message loss, recovery needs durable acknowledgement or a much safer owner-approved retention policy.
  - confidence: 0.91
