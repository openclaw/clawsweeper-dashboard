Full review comments:

- **[P1] Keep replayed prompts out of trusted system events:** `src/gateway/server-pending-messages.ts:32-33`
  - body: `item.prompt` is channel user text, but this formats it into an `enqueueSystemEvent` payload. Current system events are rendered as trusted `System:` lines after only nested marker sanitization, so a queued message can become higher-priority instructions after restart; replay through the normal untrusted user/queue path or quote it as untrusted content instead.
  - confidence: 0.93
- **[P1] Persist queue recovery state in SQLite:** `src/auto-reply/reply/queue/persist.ts:67`
  - body: This writes durable runtime queue state to `state/pending-messages.json`. OpenClaw runtime state now belongs in the shared SQLite store with a clear owner and doctor migration path, so this sidecar would create a second persistence model for queued messages and upgrades.
  - confidence: 0.9
- **[P2] Do not discard restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:110-112`
  - body: The hard five-minute cutoff silently drops the very messages the feature is meant to preserve whenever downtime, reboot, or repeated startup failures last longer than that window. Recovery should preserve pending queue state until it is drained or intentionally expired by an owner-reviewed policy.
  - confidence: 0.86
