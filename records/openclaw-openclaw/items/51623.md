## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep replayed user text out of trusted system events:** `src/gateway/server-pending-messages.ts:30-31`
  - body: `item.prompt` comes from channel users, but this code concatenates it into a queued system event. Current main renders system events as `System:` prompt lines, so after restart a crafted queued message can be promoted from user input into trusted system context; replay it through normal followup dispatch or an explicitly untrusted user-context block instead.
  - confidence: 0.91
- **[P1] Store queue recovery in SQLite, not a JSON sidecar:** `src/auto-reply/reply/queue/persist.ts:66`
  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. The repository storage contract requires runtime state, caches, and queues to use SQLite-backed stores with doctor/migration ownership, so this sidecar creates a parallel source of truth with no upgrade owner.
  - confidence: 0.9
- **[P2] Do not expire restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:107-109`
  - body: The loader discards persisted queues older than five minutes, so manual downtime, slow boot, or repeated failed restarts still lose the very messages this feature is meant to preserve. Durable queue state needs explicit processed/abandoned semantics instead of a short fixed TTL.
  - confidence: 0.83
