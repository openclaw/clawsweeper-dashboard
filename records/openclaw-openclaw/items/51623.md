Full review comments:

- **[P1] Move queue persistence out of JSON sidecar state:** `src/auto-reply/reply/queue/persist.ts:67`
  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. The repository state contract says runtime queues/caches belong in SQLite-backed stores, with old sidecars handled by doctor/migration code, so this creates a parallel source of truth with no upgrade owner.
  - confidence: 0.91
- **[P1] Replay queued text as untrusted user content:** `src/gateway/server-pending-messages.ts:32-44`
  - body: `item.prompt` comes from user/channel messages, but this path embeds it in a recovery summary and queues that summary as a system event. Current main renders system events as `System:` prompt lines, so a crafted queued message can be elevated after restart instead of being replayed through normal untrusted/user-message handling.
  - confidence: 0.9
- **[P2] Do not expire restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:110-116`
  - body: The loader drops all persisted queue entries older than five minutes, which means a slow update, manual downtime, or repeated restart failure still silently loses exactly the messages this PR is meant to preserve. Durable queue recovery should either restore until consumed or use an explicit product policy with visible handling.
  - confidence: 0.78
