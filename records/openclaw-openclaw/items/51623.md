Full review comments:

- **[P1] Move pending queue state out of the JSON sidecar:** `src/auto-reply/reply/queue/persist.ts:9`
  - body: This persists OpenClaw-owned runtime queue state to `state/pending-messages.json`. Current repository policy requires runtime state and queues to use SQLite with doctor/migration ownership; a JSON sidecar here creates a second, unmigrated source of truth for message recovery.
  - confidence: 0.91
- **[P1] Keep user prompts out of trusted system events:** `src/gateway/server-pending-messages.ts:32-33`
  - body: `item.prompt` is user/channel text, but this replay path embeds it into an event that current prompt assembly renders as trusted `System:` lines. A crafted queued message can therefore gain system-priority framing after restart; replay should preserve the text as untrusted user content or a clearly quoted untrusted block.
  - confidence: 0.93
- **[P2] Do not discard pending messages after five minutes:** `src/auto-reply/reply/queue/persist.ts:110-116`
  - body: The fixed five-minute expiry drops the very messages this feature is meant to preserve whenever downtime, manual restarts, or repeated failed starts exceed the TTL. That reintroduces silent message loss across a plausible restart window instead of letting recovery or operator policy decide staleness.
  - confidence: 0.84
