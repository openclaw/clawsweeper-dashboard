Full review comments:

- **[P1] Keep replayed prompts out of system events:** `src/gateway/server-pending-messages.ts:32-33`
  - body: `item.prompt` is user-controlled queued message text, but this path embeds it in a system event. Current main formats queued system events as `System:` lines, so a restarted user message gets a trusted-looking presentation instead of staying clearly untrusted user content.
  - confidence: 0.9
- **[P1] Store restart queue state in SQLite:** `src/auto-reply/reply/queue/persist.ts:67`
  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. The repository storage contract requires runtime queues/state to live in SQLite-backed stores, with legacy sidecars handled by doctor or migration code, so this sidecar creates a second persistence path with no upgrade owner.
  - confidence: 0.88
- **[P1] Rebase replay onto the current startup lifecycle:** `src/gateway/server-startup.ts:193-197`
  - body: The PR schedules replay from `src/gateway/server-startup.ts`, but that file no longer exists on current main and GitHub reports the branch as conflicting. Rebuild this integration in the current post-ready runtime service path before it can land.
  - confidence: 0.86
- **[P2] Do not drop restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:112-116`
  - body: The loader discards persisted queues older than five minutes, which reintroduces the same message-loss bug after manual downtime, slow reboot, or repeated failed restarts. Recovery should preserve pending user work until a deliberate migration, replay, or operator-visible discard path handles it.
  - confidence: 0.82
