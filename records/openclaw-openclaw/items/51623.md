  - body: This takes `item.prompt` from queued channel input and injects it through `enqueueSystemEvent`, but current system events are rendered as trusted `System:` prompt lines.   - body: The PR writes OpenClaw-owned queue/runtime state to `state/pending-messages.json`. Repository policy and the current replacement path keep this kind of persistent runtime queue state in SQLite with doctor-owned migration, so this sidecar would create a second persisted data model for the same recovery surface.
  - confidence: 0.91

  - body: Discarding persisted entries after five minutes means any longer outage loses the queued messages without replay, which reintroduces the message-loss behavior this PR is supposed to fix. Recovery should either preserve the queue until consumed or make expiration an explicit, tested product policy.
  - confidence: 0.86
