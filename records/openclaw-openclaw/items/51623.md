Full review comments:

- **[P1] Keep replayed message text untrusted:** `src/gateway/server-pending-messages.ts:32-33`
  - body: `item.prompt` is user-controlled channel text, but this code interpolates it into an event that current main renders as `System:` prompt lines. A queued message recovered after restart can therefore be elevated from user input into trusted system context; replay it through an untrusted prompt block or the normal inbound/followup path instead.
  - confidence: 0.91
- **[P1] Store pending queues in SQLite:** `src/auto-reply/reply/queue/persist.ts:67`
  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. Current repository policy requires runtime queues/state to use the shared SQLite store rather than JSON sidecars, so this would create a second persistence owner with no doctor/migration path.
  - confidence: 0.86
- **[P2] Do not expire recovered messages after five minutes:** `src/auto-reply/reply/queue/persist.ts:112-116`
  - body: The loader deletes the file before validation and then discards entries older than five minutes. A manual restart, slow update, or repeated failed restart can exceed that window, so this reintroduces the message loss the PR is meant to prevent.
  - confidence: 0.83
