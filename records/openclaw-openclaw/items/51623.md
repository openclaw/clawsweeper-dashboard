Full review comments:

- **[P1] Store queue recovery in SQLite-backed state:** `src/auto-reply/reply/queue/persist.ts:67`
  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. The repository storage contract says runtime queues/state should use SQLite-backed stores, with old sidecars handled by doctor/migration code, so this sidecar creates an unsupported parallel source of truth.
  - confidence: 0.9
- **[P1] Keep replayed user text untrusted:** `src/gateway/server-pending-messages.ts:32-33`
  - body: `item.prompt` is user/channel message text, but this code inserts it into `eventText` and sends it through `enqueueSystemEvent`; current main renders those entries as trusted `System:` lines, so a queued message can gain system-event authority after restart.
  - confidence: 0.88
- **[P1] Rebase replay onto the current startup path:** `src/gateway/server-startup.ts:193-197`
  - body: This schedules replay from `src/gateway/server-startup.ts`, but current main no longer has that module and recovery now runs through the post-ready runtime services path. The PR is reported as conflicting, so this hook cannot land as-is.
  - confidence: 0.86
- **[P2] Do not expire restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:110-116`
  - body: The hard five-minute TTL discards queued messages whenever downtime is longer than the fixed window, reintroducing the message loss this PR is meant to prevent for manual restarts, slow boots, or repeated failed restarts.
  - confidence: 0.82
