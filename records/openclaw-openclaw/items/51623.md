  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. Current policy requires runtime queues/state to live in SQLite-owned stores, with legacy sidecars handled only by doctor or migration code, so this creates a parallel source of truth with no upgrade owner.

  - body: `item.prompt` is user-originating queued message text, but the replay path interpolates it into `enqueueSystemEvent`; current main renders these entries as trusted `System:` lines on the next prompt.   - confidence: 0.88

  - confidence: 0.86

  - body: The fixed five-minute stale guard discards persisted followups after slow restarts, manual downtime, or repeated failed restarts, reintroducing the message loss this PR is trying to prevent. Recovery should use an explicit retention/ack policy rather than a short unconditional TTL.
  - confidence: 0.82
