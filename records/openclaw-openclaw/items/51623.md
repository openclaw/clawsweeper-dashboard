## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep replayed prompts out of trusted system events:** `src/gateway/server-pending-messages.ts:34`
  - body: `item.prompt` is user-controlled channel text, but this path wraps it into an event that later renders as trusted `System:` prompt context. Replay recovered messages through the normal inbound/follow-up path, or fence the content as explicitly untrusted data before it reaches the model.
  - confidence: 0.91
- **[P1] Use the shared state database for queued message state:** `src/auto-reply/reply/queue/persist.ts:62`
  - body: This writes OpenClaw-owned runtime queue state to a JSON sidecar under the state directory. Current repository policy requires queue/runtime state to live in SQLite, which also gives a safer place for schema, permissions, migration, and per-row replay acknowledgement.
  - confidence: 0.9
- **[P2] Delete pending queues only after replay succeeds:** `src/auto-reply/reply/queue/persist.ts:89`
  - body: `consumePersistedQueues` unlinks the file before `replayPersistedPendingMessages` has injected every entry. A crash or throw after this point loses the only saved copy, which defeats the restart-recovery goal.
  - confidence: 0.86
- **[P2] Do not expire restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:105`
  - body: A restart can take longer than five minutes during manual downtime, slow service startup, or repeated failures. Discarding the file at that age reintroduces silent message loss for exactly the recovery window this PR is trying to cover.
  - confidence: 0.82
