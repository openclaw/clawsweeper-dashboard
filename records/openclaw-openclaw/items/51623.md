## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Store recovered queue state in SQLite:** `src/auto-reply/reply/queue/persist.ts:67`
  - body: This writes OpenClaw-owned runtime queue state to `state/pending-messages.json`. The repository's storage contract puts runtime queues/state in SQLite with doctor-owned migration/cleanup, so this sidecar would create a parallel source of truth and upgrade debt; rebuild this on the shared state DB instead.
  - confidence: 0.9
- **[P1] Keep replayed user text out of trusted system events:** `src/gateway/server-pending-messages.ts:33-41`
  - body: `item.prompt` is user-originated channel text, but replay wraps it into a system event that is rendered as `System:` context on the next turn. Recovery should either redispatch through the normal user-message path or quote the content as explicitly untrusted so restart does not upgrade user instructions into system-priority prompt text.
  - confidence: 0.86
- **[P2] Do not discard queued messages after five minutes:** `src/auto-reply/reply/queue/persist.ts:110-116`
  - body: The hard five-minute age check can drop the same queued messages this PR is meant to preserve whenever downtime, slow reboot, or repeated failed restarts lasts longer than that window. Durable recovery should keep the accepted messages until replayed, migrated, or explicitly expired by a maintainer-approved policy.
  - confidence: 0.82
