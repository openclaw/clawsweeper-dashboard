## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not replay user prompts as trusted system events:** `src/gateway/server-pending-messages.ts:31`
  - body: `item.prompt` comes from channel users, but this replay path embeds it into an `enqueueSystemEvent` payload. Current system events are rendered as `System:` prompt lines, so a queued user message can gain trusted instruction priority after restart; restore it as untrusted user context or through the normal followup queue path instead.
  - confidence: 0.94
- **[P1] Use SQLite instead of a queue-state JSON sidecar:** `src/auto-reply/reply/queue/persist.ts:8`
  - body: `pending-messages.json` adds OpenClaw-owned runtime queue state as a JSON sidecar under the state directory. The repository policy requires runtime queues/state to live in SQLite or migrate through doctor, so this would create a second persistence contract and upgrade debt.
  - confidence: 0.9
- **[P2] Do not expire valid restart recovery after five minutes:** `src/auto-reply/reply/queue/persist.ts:106-111`
  - body: A fixed five-minute age cutoff means queued messages are still silently discarded after slow restarts, manual downtime, or repeated failed restarts. A recovery feature for message loss should not reintroduce loss based on a short hard-coded wall-clock TTL.
  - confidence: 0.84
