## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Store pending queues in SQLite, not a sidecar:** `src/auto-reply/reply/queue/persist.ts:9`
  - body: This adds `state/pending-messages.json` for OpenClaw-owned runtime queue state. Root policy requires runtime queues/state to live in SQLite/shared DB ownership rather than steady-state JSON sidecars, so this creates a second storage path with no doctor-owned migration boundary.
  - confidence: 0.95
- **[P1] Keep queued user text out of trusted system events:** `src/gateway/server-pending-messages.ts:32-33`
  - body: `item.prompt` is channel-user content, but the replay path embeds it in an `enqueueSystemEvent` payload. Current main renders those events as trusted `System:` prompt lines, so a queued user message can gain system-priority instruction weight after restart; replay it through normal untrusted follow-up/user context instead.
  - confidence: 0.94
- **[P2] Avoid expiring persisted restart messages after five minutes:** `src/auto-reply/reply/queue/persist.ts:111-112`
  - body: The hard `MAX_AGE_MS` discard means messages are still lost whenever restart downtime exceeds five minutes, such as manual maintenance or repeated failed starts. A restart-recovery feature should preserve queued messages until a durable delivery or explicit owner cleanup decision occurs.
  - confidence: 0.86
