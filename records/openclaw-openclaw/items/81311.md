## Summary

Keep open: current main and v2026.5.12 implement imsg typing indicators and read receipts, but automatic tapback acknowledgement and the canonical iMessage ack/capabilities config contract remain unresolved.

## What This Changes

The issue asks to wire the iMessage runtime to imsg typing, read receipts, tapback acknowledgements, and explicit capability/config handling.

## Best Possible Solution

Keep the shipped imsg typing/read implementation, then decide whether iMessage should add automatic tapback ack through the shared ackReaction contract or explicitly document read receipts as the supported acknowledgement path.

## Reproduction Assessment

Yes at source level for the remaining gap: current main calls imsg for typing/read and exposes manual tapbacks, but the iMessage inbound path does not wire automatic ack reactions or ackReactionScope. I did not run a live macOS imsg bridge reproduction.

## Solution Assessment

Unclear. The maintainable fix depends on a maintainer decision about whether iMessage should use the shared ackReaction/ackReactionScope contract, a channel-specific toggle, capabilities strings, or read receipts only with docs narrowed.
