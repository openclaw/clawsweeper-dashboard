## Summary

Keep open: current main and the latest release now cover iMessage typing indicators and read receipts, but automatic inbound tapback acknowledgment is still absent and the canonical config surface is unresolved.

## What This Changes

Request to wire iMessage imsg private API typing indicators, tapback acknowledgments, read receipts, and capability/config handling into the channel runtime.

## Best Possible Solution

Keep the shipped typing/read implementation, then add iMessage automatic ack tapbacks through the maintainer-chosen shared or iMessage-specific ack-reaction contract with focused schema/docs/tests.

## Reproduction Assessment

Yes for source-level reproduction of the remaining gap: current main calls imsg for typing/read, but the inbound monitor never sends an automatic tapback acknowledgment. I did not run a live Messages.app/imsg bridge session.

## Solution Assessment

Unclear: adding tapbacks is plausible, but the best implementation depends on whether maintainers want iMessage to use the shared ack-reaction contract, a schema addition, or an iMessage-specific toggle and emoji mapping.
