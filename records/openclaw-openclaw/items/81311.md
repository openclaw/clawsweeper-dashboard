## Summary

Keep open: current main covers iMessage typing indicators and read receipts, and tapbacks are available as manual message actions, but there is still no automatic inbound tapback acknowledgement path or settled iMessage ack config contract.

## What This Changes

The issue requests wiring iMessage runtime behavior to imsg typing, tapback acknowledgement, read receipts, and canonical capability/config toggles.

## Best Possible Solution

Keep the shipped typing/read behavior, then either add iMessage automatic ack tapbacks through the shared ackReaction contract or document read receipts as the supported iMessage acknowledgement path.

## Reproduction Assessment

Yes at source level: current main marks accepted chats read and wires typing callbacks, but the inbound iMessage path never creates an ack reaction handle or sends a tapback on receipt. I did not run a live macOS/imsg bridge reproduction.

## Solution Assessment

Unclear. Typing/read are already solved cleanly, but automatic tapbacks should either use the shared ackReaction/ackReactionScope contract or be intentionally left undocumented/unsupported in favor of read receipts.
