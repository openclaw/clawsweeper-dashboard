## Summary

Keep open. Current main and the latest release still drop the Discord attachment/sticker name before saving inbound media, and this issue is already paired with open PR #59783, so it should remain open until that PR lands or is replaced.

## What This Changes

Preserve Discord inbound attachment and sticker filenames in the saved inbound media path and agent-facing media context.

## Best Possible Solution

Review and land a Discord-owned patch that passes `attachment.filename` and sticker candidate filenames into `saveMediaBuffer`, updates the Discord media tests, and decides whether Discord should also expose a separate filename field for exact WhatsApp-style metadata parity.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: a Discord message with `attachment.filename` reaches `resolveMediaList`, but the filename is not forwarded to `saveMediaBuffer`, so the saved path remains UUID-based.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix for the saved-path loss because the shared media store already has the filename-preserving API. The only product detail to confirm is whether path preservation is enough or whether Discord should also add a distinct agent metadata field like WhatsApp `mediaFileName`.
