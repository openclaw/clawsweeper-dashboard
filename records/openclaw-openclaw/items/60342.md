- `impact:message-loss`: The issue is about inbound Telegram images/documents failing to download and being omitted from the agent message flow.

## Summary

Keep open. Current main still leaves Telegram Bot API file downloads on the strict guarded media path for env-proxy transports, so proxy-only or no-DNS environments can still skip images/files even while normal Telegram API calls use the env proxy successfully.

## What This Changes

The issue reports Telegram Bot API media/image downloads failing DNS resolution in VPN or no-DNS proxy environments while Telegram text API calls continue to work.

## Best Possible Solution

Adopt a maintainer-approved Telegram media policy that safely uses trusted env-proxy handling for Bot API file downloads when HTTP(S) proxy env applies, or document that this path requires working local DNS.

## Reproduction Assessment

Yes, source-level: configure Telegram with HTTP(S) env proxy and no usable local DNS, then media downloads still use strict guarded fetch and can resolve api.telegram.org locally. The issue discussion also includes sanitized logs showing media failures while normal sends succeed through EnvHttpProxyAgent.

## Solution Assessment

Unclear. The likely code path is narrow, but maintainers need to choose the SSRF/env-proxy contract; hardcoded public DNS or alternate-DNS retries would not be the maintainable fix.
