## Summary

Keep open. This is a focused bug fix: current main checks the mutating-tool warning branch before `messages.suppressToolErrors` even though the current config help says that setting suppresses user-facing tool-error warnings; the PR reorders the policy and adds targeted regression coverage, with terminal proof now supplied.

## What This Changes

The branch moves `messages.suppressToolErrors` ahead of mutating-tool warning generation in `buildEmbeddedRunPayloads` and updates the mutating-tool regression expectation.

## Best Possible Solution

Merge the focused policy reorder and regression after normal maintainer gates; leave broader configurable warning-delivery policy to the existing wider discussions if maintainers want more modes.

## Reproduction Assessment

## Solution Assessment

Yes. Reordering the existing suppression check before mutating-tool warning generation is the narrow maintainable fix, and the PR preserves the special `sessions_send` suppression behavior.
