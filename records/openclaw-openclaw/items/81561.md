## Summary

Keep this PR open for normal maintainer review and merge gates; the patch is a focused fix for a real payload-warning policy bug, includes sufficient terminal proof, and I found no blocking correctness issue.

## What This Changes

The branch reorders `messages.suppressToolErrors` ahead of mutating-tool warning generation in `buildEmbeddedRunPayloads` and updates the regression expectation.

## Best Possible Solution

Merge the focused suppression-order fix and regression test after normal maintainer gates; keep broader warning-delivery policy changes out of this PR.

## Reproduction Assessment

Yes, source-reproducible on current main: `write` is classified as mutating and the mutating warning branch returns before the global `messages.suppressToolErrors` check. The current regression test also encodes the incorrect visible-warning behavior for that configuration.

## Solution Assessment

Yes. Moving the existing suppression guard before mutating-tool warning generation is the narrowest maintainable fix, while preserving `sessions_send` and run-specific `suppressToolErrorWarnings` behavior.
