## Summary

Keep open: the code change is narrowly aligned with the reported command gate, but the PR still needs actual after-fix command proof and docs/manifest alignment for the new empty-agent fallback contract.

## What This Changes

The PR makes Active Memory treat an empty agent allowlist as the default `main` agent and adds a slash-command regression test.

## Best Possible Solution

Land the fallback only with docs/manifest wording aligned, explicit non-empty allowlists preserved, and real after-fix `/active-memory` output from a single-agent setup.

## Reproduction Assessment

Yes from source inspection: with Active Memory enabled, omitted `config.agents` becomes `[]`, `isEnabledForAgent` rejects `main`, and the command returns `Active Memory: off for this session.` before handling `status` or `on`. I did not run tests or a live command because this review is read-only.

## Solution Assessment

Mostly. Reusing the shared agent gate is the narrow code fix and preserves explicit non-empty allowlists, but the contract change needs docs/manifest alignment and real command proof before merge.
