## Summary

Keep this PR open: the source path supports the reported active-memory slash-command bug and the code change looks narrow, but the external PR still lacks after-fix proof from a real setup. There is also a small docs follow-up because the patch changes the meaning of an omitted or empty active-memory agent allowlist.

## What This Changes

The PR changes active-memory agent gating so missing or empty `config.agents` falls back to the default `main` agent, and adds a regression test for `/active-memory on/status`.

## Best Possible Solution

Land the narrow fallback with regression coverage, update the active-memory docs to describe the empty-agent default, and preserve fail-closed behavior for explicit non-empty allowlists.

## Reproduction Assessment

Yes. Source inspection on current `origin/main` shows omitted `config.agents` becomes `[]`, then `/active-memory` rejects `main` through `isEnabledForAgent` and returns the reported off response before processing the toggle.

## Solution Assessment

Yes, mostly. Reusing the shared `isEnabledForAgent` gate is the narrow maintainable fix for slash commands and hook gating, but the new empty-agent contract should be documented and proven in a real setup before merge.
