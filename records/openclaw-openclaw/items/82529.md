## Summary

Keep open: the PR plausibly fixes the source-reproducible active-memory slash-command short-circuit, but it still lacks contributor real behavior proof and changes the documented agent-targeting semantics without docs/manifest alignment.

## What This Changes

The PR changes active-memory agent gating to treat an omitted or empty `config.agents` list as `[main]` and adds a regression test for `/active-memory on/status`.

## Best Possible Solution

Land the fallback only if maintainers want empty active-memory targets to mean the default `main` agent, then align docs/manifest help, keep explicit non-empty allowlists fail-closed, and close the linked issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted `config.agents` becomes `[]`, then `/active-memory status|on|off` rejects `main` through `isEnabledForAgent` and returns the reported off response before the command action runs.

## Solution Assessment

Unclear until maintainers accept empty `config.agents` as default-main. If that semantics change is accepted, reusing `isEnabledForAgent` is narrow, but docs/manifest alignment and real behavior proof still need follow-through.
