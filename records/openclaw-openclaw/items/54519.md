## Summary

Keep open. Current main still lacks a shared per-account receive-only channel mode: schemas do not accept an account `readOnly` field, the existing `session.sendPolicy` path is session-scoped, and shared plus WhatsApp outbound paths still have no account-level deny gate; the related WhatsApp PR is useful context but is open and narrower than this cross-channel request.

## What This Changes

Request a per-account channel config option that blocks outbound delivery while continuing inbound processing.

## Best Possible Solution

Add a maintainer-approved shared channel/account outbound policy, document it, expose schema/UI metadata, and enforce it before shared delivery plus plugin-owned transport side effects.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap because current schemas do not accept the requested account field and the existing policy mechanism cannot target a channel account.

## Solution Assessment

No, current main is not the complete solution. `session.sendPolicy` is a useful partial guard, but the maintainable fix is a shared account-level outbound-deny contract with transport-level enforcement and coordination with the WhatsApp-specific PR.
