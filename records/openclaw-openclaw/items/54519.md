## Summary

Keep open. Current main and the latest release still lack the requested per-account receive-only channel mode: existing `sendPolicy` only suppresses automatic source delivery in session/reply paths, while shared outbound delivery and WhatsApp transport sends still have no account-level hard deny contract. The open WhatsApp PR is useful related work, but it is still open, WhatsApp-specific, and does not resolve the broader cross-channel policy request.

## What This Changes

Request a per-account receive-only channel configuration that preserves inbound processing while blocking all outbound delivery at the gateway or transport boundary.

## Best Possible Solution

Define an additive channel/account outbound policy in config, schema, docs, and UI metadata, then enforce it before shared outbound delivery and inside plugin-owned transport side effects.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken documented behavior. Source and release inspection give high confidence that current main lacks the requested account-level receive-only key and hard outbound gate.

## Solution Assessment

No: current `session.sendPolicy` is only a partial reply-delivery suppression mechanism, and the open WhatsApp PR is a channel-specific subset. The maintainable solution needs an explicit product contract plus enforcement at shared outbound and plugin transport boundaries.
