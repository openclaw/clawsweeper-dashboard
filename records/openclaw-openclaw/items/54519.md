## Summary

Keep open: current main and v2026.5.22 still do not expose a per-account receive-only channel mode, and the only related implementation is an open WhatsApp-only PR that does not settle the broader cross-channel config and safety contract.

## What This Changes

The issue requests a per-account channel configuration such as `readOnly: true` that still receives inbound messages while blocking all outbound channel delivery at a hard gateway or transport boundary.

## Best Possible Solution

Define an additive channel/account outbound policy with schema, docs, status metadata, and enforcement in shared outbound delivery plus channel-owned side-effect paths, then decide whether the WhatsApp PR is a plugin-specific first step or should wait for the shared contract.
