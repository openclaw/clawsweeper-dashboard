## Summary

Keep open. Current main still has only partial observe-style behavior through session `sendPolicy`; it does not provide the requested per-account, gateway/transport-level receive-only mode that blocks all outbound delivery while preserving inbound processing. The related WhatsApp PR is useful context, but it is still open, WhatsApp-specific, and does not resolve the broader cross-channel policy request.

## What This Changes

The issue requests a per-account channel config option that makes a channel inbound-only and hard-blocks outbound delivery regardless of agent behavior.

## Best Possible Solution

Define a shared channel/account outbound policy in config, schema, docs, and UI metadata, then enforce it before shared outbound delivery and in plugin-owned transport side effects.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap: the proposed account-level receive-only key is absent from current schemas, and existing sendPolicy cannot target a channel account or block every outbound transport side effect.

## Solution Assessment

No, current main is not the complete solution. `session.sendPolicy` is a useful partial guard for automatic source delivery, and the open WhatsApp PR may cover one channel, but the requested hard per-account outbound block needs a shared policy decision and enforcement point.
