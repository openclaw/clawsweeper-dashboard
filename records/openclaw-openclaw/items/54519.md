## Summary

Keep open. Current main and the latest release still lack the requested per-account receive-only channel mode: `sendPolicy` is only a partial session/reply suppression path, while shared outbound delivery and WhatsApp transport side effects do not have a hard account-level outbound deny contract.

## What This Changes

The issue requests a per-account read-only channel mode that receives inbound messages while blocking all outbound channel delivery at the gateway level.

## Best Possible Solution

Define an additive channel/account outbound policy in config, schema, metadata, docs, and UI surfaces, then enforce it before shared outbound delivery and inside channel-owned transport side effects.
