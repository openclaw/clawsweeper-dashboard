## Summary

Keep open. Current main still lacks the requested per-account receive-only channel mode: shared and WhatsApp config schemas have no `readOnly` account field, `session.sendPolicy` is session-scoped rather than account-scoped, and shared delivery plus WhatsApp transport paths still send without an account-level outbound-deny gate; #71382 is useful WhatsApp-specific context but remains open and does not cover the broader cross-channel request.

## What This Changes

Request a per-account channel config option that allows inbound processing while hard-blocking all outbound delivery and transport side effects.

## Best Possible Solution

Define a shared channel/account outbound policy in config, schema, docs, and UI metadata, then enforce it before shared outbound delivery and in plugin-owned transport side effects.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap: the proposed account config key is not accepted by current schemas, and the existing policy mechanism cannot target a channel account.

## Solution Assessment

No, current main is not the complete solution. `session.sendPolicy` is a useful partial guard and #71382 may cover the WhatsApp subset, but the maintainable fix is a maintainer-approved shared account outbound policy with transport-level enforcement.
