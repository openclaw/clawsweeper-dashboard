## Summary

Keep open. Current main still lacks a hard per-account read-only channel mode: shared channel config and WhatsApp config have no receive-only/outbound-deny field, gateway send/poll and shared outbound delivery still call outbound adapters directly, and WhatsApp transport side effects still have no account-level deny gate. The related #71382 PR is useful WhatsApp implementation context, but it is open and WhatsApp-scoped, so it does not resolve this broader cross-channel request.

## What This Changes

Request a per-account channel mode that continues receiving inbound messages while blocking all outbound delivery at the gateway or transport level.

## Best Possible Solution

Define a maintainer-approved channel/account outbound policy in shared config, schema/help, docs, and UI metadata, then enforce it before every outbound surface that can leave the gateway or a channel transport. Coordinate #71382 as the WhatsApp subset, but close this issue only after the generic policy lands or maintainers explicitly split the remaining scope into narrower tracked work.
