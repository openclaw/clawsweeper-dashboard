## Summary

Keep open: this is the active implementation path for WebChat bounded-history pagination, but it is not merge-ready because an additive protocol field still breaks Swift source compatibility and the new load-older control does not match its added CSS selectors; maintainer review is also still needed for the session-state semantics.

## What This Changes

The PR adds bounded cursor pagination to Gateway `chat.history`, WebChat older-message loading, generated protocol/docs updates, mock scenario data, and focused Gateway/UI tests.

## Best Possible Solution

Land the additive cursor implementation only after Swift source compatibility is preserved, load-older styling matches the rendered markup, and maintainers accept the transcript pagination semantics for `chat.history`.

## Reproduction Assessment

Yes. At source level, current WebChat requests a bounded 100-message history page and current `chat.history` has no `beforeSeq` or has-more signal, so older tool-heavy rows cannot be requested through the existing WebChat path.

## Solution Assessment

Yes with repairs. Bounded cursor pagination plus a WebChat load-older control is a better fix than raising the limit, but this branch needs the Swift default, selector fix, mergeability refresh, and maintainer session-state approval before it is the best landing version.
