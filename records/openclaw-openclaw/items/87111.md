## Summary

Keep open: this is an active maintainer-labeled PR with a useful WebChat/Gateway pagination direction, but it is not merge-ready because the generated Swift initializer remains source-incompatible and the load-older view still misses the stylesheet selectors; the public `chat.history` session-state semantics also need maintainer review.

## What This Changes

The PR adds Gateway/WebChat cursor pagination for chat history, including `beforeSeq` request support, response cursor metadata, a WebChat load-older control, docs, generated Swift model output, mock data, and regression coverage.

## Best Possible Solution

Land the pagination only after additive Swift source compatibility is preserved, the load-older styling matches the rendered markup, and maintainers explicitly accept the public `chat.history` cursor/session-state semantics.

## Reproduction Assessment

Yes. At source level, current main has bounded `chat.history` without a `beforeSeq` cursor or `hasMore` pagination signal, so WebChat cannot request older bounded pages.

## Solution Assessment

Yes with repairs. Bounded cursor pagination plus a WebChat load-older control is a better solution than raising history limits because it preserves payload caps, but this branch still needs the Swift default, selector fix, and maintainer session-state approval before it is the best landing version.
