## What This Changes

The PR strips matrix-js-sdk's MSC4222 `state_after` opt-in from guarded Matrix `/sync` requests, adds transport regression tests, and updates Matrix E2EE changelog entries.

## Best Possible Solution

Land this narrow Matrix transport mitigation after maintainer review so OpenClaw avoids the upstream `/sync` opt-in until `state_after` handling is safe for affected homeservers.

## Reproduction Assessment

Yes at source level. matrix-js-sdk 41.5.0 hard-codes the `/sync` opt-in and uses `state_after` for final room state, while current OpenClaw routes SDK sync requests through the guarded fetch unchanged; the missing piece is a live broken homeserver repro.

## Solution Assessment

Yes. The fetchFn boundary is the narrowest maintainable fix because OpenClaw already centralizes matrix-js-sdk HTTP there and the PR avoids adding config or changing unrelated Matrix behavior.
