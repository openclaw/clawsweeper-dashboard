## Summary

Current main now exposes the collapsed Telegram direct runtime-policy identity on the active sessions surface, with regression coverage and merged PR proof, so the central operator-visibility bug is implemented on main but not yet in the latest release.

## What This Changes

The issue asks OpenClaw to avoid hiding Telegram direct runtime work when main-DM routing stores the active row as `agent:main:main`.

## Best Possible Solution

Keep the current main implementation that preserves the persisted main session row while surfacing the runtime-policy Telegram direct alias; open a narrower follow-up only if `openclaw status` needs the same alias field.

## Reproduction Assessment

No live Telegram reproduction was rerun in this read-only review. Source inspection plus the current regression test give a clear reproduction shape for the fixed observable path: an active main-scoped Telegram direct row now emits its runtime-policy peer alias.

## Solution Assessment

Yes. The merged fix is a maintainable central sessions-surface change: it keeps session persistence canonical while exposing the collapsed runtime-policy identity operators need for debugging.
