## Summary

Keep open. Current main still exposes a browser `timeoutMs` input but drops it along the snapshot path before the host client, `/snapshot` plan/route, Chrome MCP snapshot call, and Playwright snapshot helpers; open PR #75702 already targets this issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Make `browser action=snapshot` honor a caller/default timeout and return page snapshots after successful open/navigate instead of consistently timing out.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the agent tool, proxy/client, `/snapshot` route, Chrome MCP calls, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for source-level triage. The issue gives precise `open` then `snapshot` steps, and current-main source inspection shows accepted `timeoutMs` is not propagated through several snapshot operations that can hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable fix; closing this issue before the linked implementation lands would be premature.
