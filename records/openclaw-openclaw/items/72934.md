## Summary

Keep open. Current `main` still does not propagate browser snapshot `timeoutMs` end-to-end, and https://github.com/openclaw/openclaw/pull/75702 is an open, unmerged closing PR for this issue.

## What This Changes

Browser snapshot should return page content after open/navigate instead of timing out on the snapshot action path.

## Best Possible Solution

Land or replace the open browser plugin fix so one normalized snapshot timeout reaches the action, client/proxy, `/snapshot` plan and route, Chrome MCP, Playwright helpers, and regression tests.

## Reproduction Assessment

Yes, for source-level triage. The reporter gives concrete `open` then `snapshot` steps, and current `main` independently shows accepted `timeoutMs` cannot reach several snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing timeout budget through the browser snapshot path is the narrow maintainable fix; closing this issue before the linked PR or an equivalent fix lands would be premature.
