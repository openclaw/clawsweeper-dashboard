## Summary

Keep open. Current main still drops `timeoutMs` before several browser snapshot layers, and open PR #75702 already uses closing syntax for this issue, so cleanup should wait for that PR to land or be replaced.

## What This Changes

The issue reports that `browser action=snapshot` times out after a successful browser `open`/navigation on macOS, even though the gateway and browser control surface are reachable.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the agent tool, proxy/client, `/snapshot` plan and route, Chrome MCP snapshot calls, Playwright helpers, tests, and changelog.

## Reproduction Assessment

Yes for triage. The issue gives a precise `open` then `snapshot` sequence, and current-main source inspection independently shows the accepted timeout cannot reach several snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable fix; closing this issue before the linked open PR lands would be premature.
