## Summary

Keep open. Current main still accepts a browser `timeoutMs` input but does not propagate it through the snapshot client, proxy/query plan, Chrome MCP snapshot call, or several Playwright snapshot helpers; open PR #75702 already targets this issue with closing syntax, so closing this issue before that PR lands would be premature.

## What This Changes

The issue reports that `browser action=snapshot` times out after successful browser `open` or `navigate` operations on macOS OpenClaw 2026.4.25.

## Best Possible Solution

The browser plugin should normalize one snapshot timeout budget and propagate it end-to-end through the agent action, proxy/client request, `/snapshot` route plan, Chrome MCP call, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for source-level triage. The issue gives precise `open` then `snapshot` steps, and current main independently shows the accepted `timeoutMs` cannot reach multiple snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable fix; closing this issue before linked PR #75702 lands would be premature.
