## Summary

Keep open. Current main still has a source-backed browser snapshot timeout propagation gap: callers can pass `timeoutMs`, and the gateway/node proxy can honor it when supplied, but the snapshot action, host client, `/snapshot` route, Chrome MCP snapshot call, and Playwright snapshot helpers do not consistently carry that timeout into the work that can hang.

## What This Changes

The issue requests fixing the browser snapshot action so it returns page content or an accessibility tree instead of timing out after successful browser open/navigate operations.

## Best Possible Solution

Repair the browser plugin snapshot timeout path so one normalized snapshot timeout is propagated from the agent tool through host/node proxy calls, `browserSnapshot`, `/snapshot`, Chrome MCP snapshot, Playwright AI/role/ARIA snapshot helpers, and snapshot label side paths, with focused regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes for triage: the issue provides a precise open-then-snapshot sequence, version, platform, gateway status, and repeated cross-URL failures. Current-main source inspection independently confirms that snapshot timeout budgets cannot reach several parts of the snapshot stack.

## Repair Work Prompt

Fix the browser snapshot timeout propagation bug reported at https://github.com/openclaw/openclaw/issues/72934. The browser tool already exposes top-level `timeoutMs`, and the gateway/node proxy can honor a supplied timeout, but snapshot currently drops it before `/snapshot` and before Chrome MCP / Playwright snapshot work. Keep the fix inside the browser plugin unless a generic seam is strictly necessary. Normalize a caller-provided snapshot timeout once, preserve current defaults when omitted, add proxy slack where the existing proxy pattern requires it, and propagate the timeout through `executeSnapshotAction`, `browserSnapshot`, node proxy calls, `/snapshot` query/plan handling, `takeChromeMcpSnapshot`, Playwright AI/role/ARIA snapshot helpers, and snapshot label/screenshot side paths that are part of the snapshot request. Add focused regression tests proving host snapshot and node-proxy snapshot receive the timeout, route handlers pass it to Chrome MCP and Playwright snapshot helpers, and existing-session behavior still rejects only the unsupported timeout overrides documented for act actions. Include a single-line `CHANGELOG.md` fix entry with `Thanks @wkomers-hash`; include @sylviazhang2006-design only if the implemented fix explicitly covers the related #72653 symptoms. Do not broaden this into a core gateway rewrite, browser profile migration, dependency update, or unrelated timeout-policy refactor.
