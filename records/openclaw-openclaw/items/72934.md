## Summary

Keep open. Current main still has a source-backed browser snapshot timeout propagation gap, and the report is recent, concrete, and not safely covered by a merged fix or a broader duplicate.

## What This Changes

The issue reports that browser open/navigate succeeds but browser snapshot consistently times out on OpenClaw 2026.4.25 for a macOS loopback gateway setup.

## Best Possible Solution

Repair the browser plugin snapshot timeout path so one normalized timeout budget is propagated from the agent tool through host and node proxy calls, `browserSnapshot`, `/snapshot`, Chrome MCP snapshot and label helpers, CDP/Playwright AI/role/ARIA snapshot helpers, with focused regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes for triage: the issue gives a concrete open-then-snapshot sequence, version, platform, gateway status, and repeated cross-URL failures. Current-main source inspection independently confirms that snapshot timeout budgets cannot reach several parts of the snapshot stack.

## Solution Assessment

Yes: a narrow browser plugin timeout propagation fix is the best maintainable solution. A broader gateway rewrite, browser profile migration, or duplicate close against the Linux all-actions timeout report would be larger and less precise than the evidence supports.

Keep the change inside the bundled browser plugin unless a truly generic seam is already present; do not add core owner-specific browser policy. Normalize top-level `timeoutMs` for `action: "snapshot"` and propagate it through `executeSnapshotAction`, node proxy calls, `browserSnapshot`, the `/snapshot` route/plan, Chrome MCP snapshot and label-side helper calls, Playwright AI/role/ARIA snapshot helpers, and CDP snapshot fallbacks where a timeout can bound hanging work. Preserve existing SSRF checks, profile behavior, labels/urls/refs fallback behavior, and current default timeouts when no timeout is supplied. Add focused regression tests proving host and node snapshot calls carry `timeoutMs`, the HTTP client uses it, the route passes it to Chrome MCP/Playwright/CDP snapshot helpers, and fallback/labels paths do not drop it.
