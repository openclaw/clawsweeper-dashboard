## Summary

Keep open. Current main still drops the browser snapshot timeout before it reaches the snapshot client, route plan, Chrome MCP snapshot call, and multiple Playwright snapshot helpers; open PR #75702 already uses closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Fix `browser action=snapshot` so it returns page content or an accessibility tree instead of timing out after successful `open` or `navigate` actions.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the tool action, client/proxy request, `/snapshot` plan and route, Chrome MCP calls, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for source-level triage. The report gives precise `open` then `snapshot` steps, and current-main source inspection independently shows accepted `timeoutMs` cannot reach several snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable fix; closing this issue before linked PR #75702 lands would be premature.
