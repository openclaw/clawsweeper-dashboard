## Summary

Keep open. Current main still accepts a browser `timeoutMs` input but does not carry it through the snapshot action, host client, `/snapshot` plan/route, Chrome MCP snapshot call, or several Playwright snapshot helpers, and open PR #75702 already uses closing syntax for this issue.

## What This Changes

Ensure browser `action=snapshot` returns page content reliably after open/navigate by honoring the caller snapshot timeout end to end.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the agent tool, proxy/client, `/snapshot` plan and route, Chrome MCP calls, Playwright helpers, regression tests, and changelog.
