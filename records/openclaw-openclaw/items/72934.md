## Summary

Keep open. Current main still drops browser snapshot `timeoutMs` before it reaches the snapshot client, route plan, Chrome MCP snapshot call, and several Playwright snapshot paths; https://github.com/openclaw/openclaw/pull/75702 remains the open, unmerged closing PR for this issue.

## What This Changes

Browser snapshot should return page content or an accessibility tree after a successful open/navigate instead of timing out.

## Best Possible Solution

Land, update, or replace the linked browser plugin fix so one normalized snapshot timeout reaches the action, client/proxy, `/snapshot` plan and route, Chrome MCP, Playwright helpers, and regression tests.

## Reproduction Assessment

Yes, for source-level triage. The reporter gives concrete open-then-snapshot steps, a later Browserbase/Linux reproduction confirms the timeout chain, and current main still shows accepted `timeoutMs` cannot reach several snapshot operations.

## Solution Assessment

Yes. Threading the existing timeout budget through the snapshot path is the narrow maintainable fix; closing this issue before the linked PR or an equivalent implementation lands would be premature.
