## Summary

Keep open. Current main still drops the browser snapshot timeout budget across the agent snapshot path, and open PR #75702 already references this issue with closing syntax, so the issue should stay open until that fix lands, is replaced, or is closed.

## What This Changes

The issue reports that browser open/navigate return a targetId but subsequent snapshot calls consistently time out on macOS OpenClaw 2026.4.25.

## Best Possible Solution

Thread one normalized snapshot timeout through the browser plugin agent tool, host client/proxy, `/snapshot` plan and route, Chrome MCP calls, Playwright snapshot helpers, focused regression tests, and changelog entry.

## Reproduction Assessment

Yes for triage. The issue gives precise `open` then `snapshot` steps, and current-main source inspection independently confirms the caller timeout cannot reach several snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot layers is the narrow maintainable direction; closing this issue before PR #75702 lands would be premature.
