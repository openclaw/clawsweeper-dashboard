## Summary

Keep open. Current main still drops the browser snapshot timeout before it reaches the snapshot client, route plan, Chrome MCP snapshot call, and several Playwright snapshot helpers; open PR #75702 already uses closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that browser `snapshot` consistently times out after successful browser `open` or navigation actions.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the agent tool, proxy/client, `/snapshot` plan and route, Chrome MCP calls, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for source-level triage. The issue gives precise `open` then `snapshot` steps, and current-main source inspection independently shows accepted `timeoutMs` cannot reach several snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable solution; closing this issue before linked PR #75702 lands would be premature.
