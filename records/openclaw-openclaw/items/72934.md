## Summary

Keep open. Current main still does not thread `timeoutMs` through the browser snapshot action, host client, `/snapshot` plan/route, Chrome MCP snapshot call, or Playwright snapshot helpers, and open PR #75702 already uses closing syntax for this issue.

## What This Changes

The issue requests that `browser action=snapshot` return page content after successful open/navigate operations instead of consistently timing out.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the agent action, proxy/client, `/snapshot` plan and route, Chrome MCP calls, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

## Solution Assessment
