## Summary

Keep open. Current main still drops the browser snapshot timeout before it reaches the host client, `/snapshot` plan/route, Chrome MCP snapshot call, and Playwright snapshot helpers, and open PR #75702 uses closing syntax for the same bug but remains unmerged.

## What This Changes

The issue reports that `browser action=snapshot` consistently times out after `open` or `navigate` succeeds and returns a target id.

## Best Possible Solution

Land or replace PR #75702 so one normalized snapshot timeout reaches the agent tool, proxy/client, `/snapshot` plan and route, Chrome MCP calls, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for triage. The issue gives a concrete `open` then `snapshot` sequence, and current source inspection independently shows that the accepted `timeoutMs` cannot reach multiple snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing timeout budget through the browser plugin snapshot path is the narrow maintainable fix; closing this issue before the linked PR lands would be premature.
