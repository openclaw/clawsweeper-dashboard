## Summary

Keep open. Current main still has a source-level browser snapshot timeout propagation gap, and open PR #75702 already targets this issue with closing syntax, so the issue should remain open until that PR lands, is replaced, or is closed.

## What This Changes

Make browser `snapshot` return page content/A11y data after a successful open or navigate instead of timing out consistently.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the tool action, host client, proxy, route plan, Chrome MCP snapshot calls, Playwright snapshot helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for source-level triage. The report gives a concrete `open` then `snapshot` sequence, and current main independently shows accepted browser `timeoutMs` cannot reach several snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser snapshot path is the narrow maintainable fix; closing before #75702 lands would be premature.
