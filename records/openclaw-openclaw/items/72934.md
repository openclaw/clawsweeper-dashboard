## Summary

Keep open. Current main still drops the accepted browser `timeoutMs` before the snapshot client/proxy, snapshot route plan, Chrome MCP snapshot call, and several Playwright snapshot helpers; open PR #75702 still targets the same issue with closing syntax, so this should stay open until that PR lands or is replaced.

## What This Changes

The report asks for browser snapshots to return page content or the accessibility tree instead of timing out after successful browser open/navigate calls.

## Best Possible Solution

Land or replace PR #75702 with a browser plugin fix that threads one normalized snapshot timeout through the agent action, client/proxy, snapshot plan and route, Chrome MCP, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for source-level triage: the issue gives concrete `open` then `snapshot targetId=...` steps, and current main still shows the accepted `timeoutMs` cannot reach several snapshot operations that may hang. I did not run a live Gateway/browser smoke in this read-only review.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable fix; closing this issue before PR #75702 lands would be premature.
