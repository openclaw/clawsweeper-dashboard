## Summary

Keep open. Current main at a0d5372 still does not propagate the browser snapshot timeout through the agent action, host client, snapshot plan, route, Chrome MCP snapshot helper, or Playwright snapshot helpers, and open PR #75702 already uses closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that `browser action=snapshot` times out after successful browser open/navigate operations on macOS OpenClaw 2026.4.25.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the agent tool, proxy/client, `/snapshot` plan and route, Chrome MCP calls, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for triage. The issue gives a precise `open` then `snapshot` sequence, and current-main source inspection independently shows the accepted `timeoutMs` cannot reach several snapshot operations that may hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable direction; closing this issue before the linked open PR lands would be premature.
