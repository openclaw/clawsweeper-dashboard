## Summary

Keep open. Current main at 2a22eb68aafd251b71778464373b8ce5cee096dc still drops the browser snapshot timeout before it reaches the snapshot client, route plan, Chrome MCP snapshot call, and Playwright snapshot helpers; open PR #75702 already uses closing syntax for this issue and remains unmerged.

## What This Changes

The issue asks for `browser action=snapshot` to return page content/accessibility data instead of timing out after `open` or `navigate` succeeds.

## Best Possible Solution

Review and land PR #75702, or an equivalent browser plugin fix, so one normalized snapshot timeout reaches the tool action, proxy/client, `/snapshot` route plan, Chrome MCP calls, Playwright helpers, regression tests, and changelog.

## Reproduction Assessment

Yes for source-level triage. The issue gives explicit `open` then `snapshot` steps, and current-main code shows accepted `timeoutMs` is not carried into multiple snapshot seams that can hang.

## Solution Assessment

Yes. Threading the existing `timeoutMs` through the browser plugin snapshot path is the narrow maintainable solution; closing this issue before PR #75702 lands would be premature.
