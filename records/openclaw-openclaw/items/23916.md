## Summary

Keep open: current main and the latest release still show the compact paired-device list without client purpose, platform/device family, activity/detail output, or editable notes, and the remaining scope needs maintainer product scoping.

## What This Changes

Request richer paired-device information in `openclaw devices list`, including client purpose, platform/activity detail, and optional user notes or names.

## Best Possible Solution

Adopt one coordinated device-identification UX across CLI, Gateway/protocol, UI if needed, and docs: show sanitized existing metadata, preserve `--json`, and route editable labels or notes through one persisted contract coordinated with https://github.com/openclaw/openclaw/issues/13870.

## Reproduction Assessment

Yes. At source level, current main deterministically renders paired devices without the requested client purpose, platform/device family, activity, detail, or note fields while storage carries some of that metadata.

## Solution Assessment

Unclear for the full request. Displaying sanitized existing metadata is a bounded first step, but editable names or notes need a shared persisted Gateway/CLI/UI contract rather than a CLI-only field.
