## Summary

Keep open: current main still renders a compact `openclaw devices list` paired-device table and does not expose the requested client purpose, OS/device family, activity/detail mode, or editable notes/name workflow. Related display and naming work remains open, so this is real feature/product scope rather than cleanup-closeable stale work.

## What This Changes

Show richer paired-device information in `openclaw devices list`, including device purpose/client metadata, host or OS details, activity/detail output, and user-settable notes or names.

## Best Possible Solution

Land or replace the partial display PRs with sanitized client/platform/activity output and docs, then resolve editable names or notes through the paired-device naming workflow.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high-confidence proof that the CLI table, Gateway methods, and docs still omit the requested metadata and note/name/detail surfaces.

## Solution Assessment

Partly yes: surfacing already-collected metadata in `openclaw devices list` is the narrow first slice. Editable notes, durable names, and any `--full` or detail mode need maintainer UX/API scoping with https://github.com/openclaw/openclaw/issues/13870 and the open display PRs.
