## What This Changes

The issue requests richer `openclaw devices list` output so users can identify paired devices by purpose, client, OS or user-agent style metadata, detail output, and optional notes.

## Best Possible Solution

Adopt one coordinated device-identification UX: show sanitized client, platform, stable ID, and activity metadata in the CLI, preserve `--json`, and route editable labels or notes through one shared Gateway/CLI/UI contract coordinated with https://github.com/openclaw/openclaw/issues/13870.

## Solution Assessment
