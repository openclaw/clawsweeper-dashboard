## Summary

Keep open: current `main` still shows a compact `openclaw devices list` table and docs, while the requested client purpose, platform/activity detail, `--full`/detail mode, and editable note/name workflow remain unimplemented. The display slice is partly covered by open PRs at https://github.com/openclaw/openclaw/pull/55973 and https://github.com/openclaw/openclaw/pull/57449, and durable human-friendly naming is still tracked by https://github.com/openclaw/openclaw/issues/13870.

## What This Changes

The issue requests richer paired-device metadata, detail output, and user notes or names in `openclaw devices list`.

## Best Possible Solution

Land or replace the partial display PRs with sanitized client/platform/activity output and docs, then resolve editable names or notes through the paired-device naming workflow.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current `main` gives high-confidence proof that the CLI table and docs still omit the requested metadata, note/name, and detail-mode surfaces.

## Solution Assessment

Partly yes: surfacing already-collected metadata in `openclaw devices list` is the narrow first slice. Editable notes, durable names, and any `--full` mode need maintainer UX/API scoping with the related naming issue before implementation.
