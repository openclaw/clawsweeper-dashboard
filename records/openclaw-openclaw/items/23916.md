## Summary

Keep open: current main still shows the compact `openclaw devices list` table, drops richer paired-device metadata in the CLI-local model, lacks note/name/detail methods, and the related display/naming work is still open rather than shipped.

## What This Changes

Request richer device purpose, platform/client/activity, and user note/name detail in `openclaw devices list` so paired devices are easier to identify.

## Best Possible Solution

Land or replace the open table-display PRs with sanitized client/platform/activity columns and docs, preserve `--json`, then resolve durable notes/names through the canonical paired-device naming workflow at https://github.com/openclaw/openclaw/issues/13870.

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence proof that current main still omits the requested metadata, note/name, and detail-mode surfaces.

## Solution Assessment
