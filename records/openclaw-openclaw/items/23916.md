## Summary

Keep open. Current main still does not surface the requested device purpose/client/platform/activity/note information in the default paired-device CLI table or docs, and related implementation work remains open rather than merged.

## What This Changes

Request to make `openclaw devices list` show clearer device-identifying metadata, including client purpose, host/browser/OS-style details, optional full output, and user-editable notes/names.

## Best Possible Solution

Land a focused device-list UX update that surfaces existing non-secret metadata, preserves `--json`, sanitizes device-controlled strings, updates CLI tests and docs, and coordinates editable names/notes with #13870.

## Reproduction Assessment

Yes for the observable CLI gap: source inspection shows the paired-device table still renders only Device/Roles/Scopes/Tokens/IP while storage and Gateway paths carry richer metadata. Editable notes and a full/detail mode are feature-design requests rather than a runtime bug reproduction.

## Solution Assessment

Unclear until an implementation PR lands. #57449 looks like a plausible base for display columns, but a complete solution should reconcile timestamp behavior with #55973 and names/notes with #13870.
