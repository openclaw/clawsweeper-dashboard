## Summary

Keep open: current main and v2026.6.5 still expose only the compact paired-device table, and the surviving related items do not cover the full client-purpose, platform, activity, and naming request.

## What This Changes

Show clearer paired-device identity and purpose information in `openclaw devices list`, potentially including client/platform details, activity timestamps, and user-friendly names or notes.

## Best Possible Solution

Define one sanitized device-identification presentation contract for CLI and UI, land client/platform/activity details through the canonical device-list path, and reuse the durable naming model tracked in https://github.com/openclaw/openclaw/issues/13870 rather than introducing a separate notes system.

## Reproduction Assessment

Yes at source level: current main’s paired table explicitly renders only five compact columns, directly demonstrating the missing identity and activity information without requiring a failing runtime condition.

## Solution Assessment

No, not as one undifferentiated implementation; safe metadata display is bounded, while timestamps and editable naming should be coordinated with their canonical open items to avoid competing semantics and storage models.
