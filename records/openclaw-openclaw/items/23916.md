## Summary

Keep open. Current main still has the device-identification UX gap: the CLI paired-device table remains compact, richer metadata is stored and returned but not displayed, docs do not describe a detail/full mode or notes, and related implementation work is still open/unmerged.

## What This Changes

Request to make `openclaw devices list` identify paired devices with richer client, OS/device, activity, and optional user note/name details.

## Best Possible Solution

Land a focused device-list UX update that surfaces existing non-secret metadata, preserves `--json`, sanitizes all device-controlled strings, updates CLI tests and docs, and coordinates editable names/notes with the existing related issue.

## Reproduction Assessment

Yes for the observable CLI gap: source shows the current paired-device table only renders Device/Roles/Scopes/Tokens/IP while storage and Gateway paths carry richer metadata. Editable notes and a full/detail mode are feature-design requests rather than a runtime bug reproduction.

## Solution Assessment

Unclear until an implementation PR lands. #57449 looks like a plausible base for display columns, but a complete solution should reconcile timestamp behavior with #55973 and names/notes with #13870.
