## Summary

Keep open. Current main still shows only the compact paired-device table, while the pairing and Gateway layers already carry richer metadata that is not exposed as client purpose, OS/device family, last activity/detail output, or editable notes/names. The linked work remains open or unmerged, so there is still actionable product/code work here.

## What This Changes

Add richer, user-understandable identifying details to `openclaw devices list`, including client purpose, OS/device information, last activity or detail output, and possibly editable notes or names.

## Best Possible Solution

Land or adapt the existing devices-list work so the CLI exposes stable, sanitized columns for device id/name, client, OS/device family, and activity while preserving `--json`; then coordinate any editable name/note workflow with #13870 and document the resulting commands in the public devices CLI page.
