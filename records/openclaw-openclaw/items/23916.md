## Summary

Keep open: current main still lacks the requested client purpose, OS/device-family, activity/detail, and editable note/name surfaces in `openclaw devices list`; related display and naming work is still open rather than shipped.

## What This Changes

The request asks for `openclaw devices list` to show understandable client/device metadata, support richer detail output, and allow user notes or names so paired devices are identifiable.

## Best Possible Solution

Land or replace the open display PRs with sanitized client/platform/activity columns and docs, then resolve durable editable names or notes through the paired-device naming workflow.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high-confidence proof that the CLI table and docs still omit the requested metadata, note/name, and detail-mode surfaces.

## Solution Assessment

Partly yes: surfacing already-collected metadata in the CLI table is the narrow first slice. Editable notes, durable names, and any full/detail mode still need maintainer UX/API scoping with the related naming issue.
