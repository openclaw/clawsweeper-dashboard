## What This Changes

Request to make `openclaw devices list` show clearer paired-device purpose, client/platform/activity details, and user-readable labels or notes.

## Best Possible Solution

Define one sanitized device-identification presentation contract for the CLI, coordinate timestamp semantics with https://github.com/openclaw/openclaw/pull/55973, and reuse the durable naming model tracked in https://github.com/openclaw/openclaw/issues/13870 rather than introducing a separate notes system.

## Reproduction Assessment

Yes, at source level: current main's paired table explicitly renders only the compact five-column view, while storage contains richer metadata that is not surfaced. This is a feature gap rather than a failing existing contract.

## Solution Assessment

No, not as one undifferentiated implementation. The safest path is to partition sanitized metadata display, timestamp semantics, and durable user-owned labels so the open trackers do not create competing storage or UI rules.
