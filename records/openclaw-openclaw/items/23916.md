The issue requests richer `openclaw devices list` output for paired devices, including client purpose, host/OS/user-agent style details, a fuller view, and user-editable notes or names.

## Best Possible Solution

Ship one coordinated device-list UX path: sanitized client/platform/activity columns in the runtime renderer, unchanged `--json`, documented output, and a separate durable operator-owned naming or note workflow coordinated with the existing naming issue.

## Solution Assessment
