## Summary

Keep open: current main still renders the compact paired-device list and does not expose the requested client purpose, platform/activity details, full/detail mode, or editable notes/name workflow. The related display PRs and naming issue remain open, so this is real feature/product work rather than cleanup-closeable stale work.

## What This Changes

Request to make `openclaw devices list` show clearer paired-device identity details such as client mode, OS/browser, host/user-agent style labels, activity/detail output, and user notes or names.

## Best Possible Solution

Land or replace the display PRs with sanitized client/platform/activity output and docs, then resolve editable labels or notes through the paired-device naming workflow tracked at https://github.com/openclaw/openclaw/issues/13870.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high-confidence proof that the CLI table and docs still omit the requested metadata and note/name surfaces.

## Solution Assessment

Partly yes: surfacing already-collected metadata in `openclaw devices list` is the narrow first slice. Editable notes, durable names, and any `--full` mode need maintainer UX/API scoping with the related naming issue before implementation.
