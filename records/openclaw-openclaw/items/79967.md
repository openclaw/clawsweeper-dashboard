## Summary

Keep open: the latest head still logs a fallback that never runs, and the supplied proof is only a mocked logger simulation rather than real cron/OpenClaw output.

## What This Changes

The PR adds a `model-fallback/decision` warning in cron model selection when an `agentTurn` payload model is rejected before returning the existing error.

## Best Possible Solution

Emit an accurate `model-fallback/decision` warning on the fail-closed cron rejection path, omit fallback wording unless a fallback actually runs, and add focused regression coverage for the log fields.

## Reproduction Assessment

Yes, source-reproducible: current main rejects a disallowed cron `payload.model` without emitting through `model-fallback/decision`, and the caller returns an error before any model fallback run. I did not establish a live cron reproduction in this read-only review.

## Solution Assessment

No. Adding a warning is the right direction, but this implementation should log a rejected/failed cron selection rather than claiming a fallback model was used.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Stop logging a fallback that never runs:** `src/cron/isolated-agent/model-selection.ts:149-151`
