## Summary

Keep open: current main still lacks a TUI plain-text/accessibility mode, the proposed config key is not accepted or documented, and the related implementation PR was closed unmerged.

## What This Changes

Request adds a documented TUI accessibility/plain-text config option to suppress emoji and Unicode glyphs for screen-reader-friendly terminal output.

## Best Possible Solution

Ship a default-off core TUI plain-text mode that is documented in config surfaces and applied consistently to tool summaries, selectors, status/waiting text, and TUI component glyphs.

## Reproduction Assessment

Yes, from source inspection of current main. The requested config key is absent, strict validation rejects it, and inspected TUI/tool paths still hard-code emoji or Unicode glyphs; this is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. A core default-off config flag is the right boundary because the affected config validation/docs and built-in TUI renderers are OpenClaw-owned; plugins cannot consistently replace these glyphs from outside core.
