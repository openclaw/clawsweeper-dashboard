## Summary

Keep open: current main still lacks the requested TUI plain-text accessibility mode, the config key is not accepted or documented, and the related implementation at https://github.com/openclaw/openclaw/pull/9685 was closed unmerged.

## What This Changes

Add a core TUI accessibility/plain-text config option that replaces emoji and Unicode glyphs with ASCII-friendly alternatives.

## Best Possible Solution

Ship a default-off core TUI plain-text mode, document it in config surfaces, and apply it consistently across tool summaries, selectors, status/waiting text, and component glyphs.

## Reproduction Assessment

Yes, for verifying the missing behavior: current-main source shows no accepted or documented config key, strict validation rejects unknown UI fields, and inspected render paths still emit emoji or Unicode glyphs. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes, with maintainer agreement on naming and replacement text. A default-off core config flag is the maintainable boundary because config validation, generated docs, and built-in TUI renderers are OpenClaw-owned.
