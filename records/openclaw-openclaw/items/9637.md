## Summary

Keep open: current main still lacks an accepted or documented TUI plain-text/accessibility mode, strict config validation would reject the proposed key, and TUI/tool render paths still emit emoji or Unicode glyphs unconditionally. The prior implementation attempt at https://github.com/openclaw/openclaw/pull/9685 is closed unmerged, so this remains actionable core TUI/config work.

## What This Changes

The issue requests a TUI accessibility config option to disable emoji and Unicode glyph output for screen-reader and plain-text terminal use.

## Best Possible Solution

Ship a documented core TUI plain-text mode read once from config, validated in schema/help/docs, and applied consistently across tool summaries, selectors, settings, waiting/status text, and relevant TUI component glyphs.

## Reproduction Assessment

Yes, for source-level verification: current main has no accepted or documented TUI plain-text config key, and inspected TUI/tool render paths still emit emoji or Unicode glyphs. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Yes in broad direction: a default-off core config flag is the maintainable boundary because OpenClaw owns config validation, public docs, and built-in TUI renderers. The option name and ASCII replacement set still need maintainer agreement before implementation.
