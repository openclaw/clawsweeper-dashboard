## Summary

Keep open: current main still lacks a TUI plain-text/accessibility mode, strict config validation would reject the proposed key, and the related implementation PR was closed unmerged.

## What This Changes

Request adds a TUI accessibility config option to render tool metadata, selectors, separators, and related terminal UI output without emoji or Unicode symbols.

## Best Possible Solution

Ship core TUI plain-text support with rich output unchanged by default and deterministic ASCII alternatives when the documented config flag is enabled.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested config key is absent from exported types and strict validation while tool summaries, selectors, status text, and tool cards still emit emoji or Unicode glyphs without an alternate branch.

## Solution Assessment

Yes. A core TUI config flag is the narrow maintainable direction because OpenClaw itself emits these glyphs in config-validated TUI/tool render paths that an external plugin cannot reliably override.
