## Summary

Keep open: current main and the latest release still lack an accepted or documented TUI plain-text/accessibility mode, strict config validation would reject the proposed key, and TUI/tool render paths still emit emoji or Unicode glyphs. The linked implementation attempt at https://github.com/openclaw/openclaw/pull/9685 is closed unmerged, so this remains actionable core TUI/config work.

## What This Changes

Add a documented TUI accessibility config option that replaces emojis and Unicode glyphs with ASCII/plain-text alternatives.

## Best Possible Solution

Ship a documented core TUI plain-text mode read once at startup and applied consistently across config validation/docs plus tool, selector, settings, waiting/status, and TUI component glyph renderers.

## Reproduction Assessment

Yes, for source-level verification: current main lacks an accepted or documented plain-text config key, and inspected TUI/tool render paths still emit emoji or Unicode glyphs. I did not run a live TUI or screen-reader session in this read-only review.

## Solution Assessment

Yes in broad direction: a default-off core config flag is the maintainable boundary because OpenClaw owns strict config validation, public docs, and built-in TUI renderers. The option name and replacement set should be decided before implementation.
