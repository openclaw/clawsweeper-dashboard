## Summary

Current main still has no plain-text TUI mode, the prior implementation closed unmerged after deprioritization, and no other item owns the work. Keep this core accessibility request open for a maintainer decision on the public control and replacement semantics.

## What This Changes

Add a user-selectable plain-text terminal UI mode that replaces decorative emoji and Unicode symbols with screen-reader-friendly ASCII or text.

## Best Possible Solution

Approve one TUI presentation-style contract resolved at startup, derive all owned theme and component symbols from it, document that arbitrary assistant content is unchanged, and verify the complete glyph inventory with focused unit and PTY-visible tests.

## Reproduction Assessment

Yes, at source level: current main unconditionally emits decorative emoji and Unicode across TUI-related render paths and has no plain-text control. A live screen-reader session was not run, but another user independently confirmed the reported usability problem.

## Solution Assessment

The accessibility goal is correct, but the issue's literal-by-literal proposal is not the best final shape. A centralized presentation-style contract is narrower and less drift-prone, while maintainers still need to select the public control and replacement vocabulary.
