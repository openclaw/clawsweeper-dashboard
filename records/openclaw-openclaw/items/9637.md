## What This Changes

Add a TUI accessibility mode that replaces decorative emoji and Unicode glyphs with screen-reader-friendly ASCII or text.

## Best Possible Solution

Implement one documented, startup-resolved plain-text rendering policy across all decorative TUI glyph sources, including verified pi-tui theme seams, with focused component tests and PTY proof plus a real screen-reader usability check when feasible.

## Reproduction Assessment

## Solution Assessment

The requested direction is sound, but the best implementation is one canonical TUI rendering policy rather than scattered character replacements. Maintainers still need to select the public config contract and fallback wording.
