## What This Changes

## Repair Work Prompt

Implement the TUI plain-text accessibility mode requested by https://github.com/openclaw/openclaw/issues/9637, using https://github.com/openclaw/openclaw/pull/9685 only as prior art. Add a documented `ui.accessibility.plainText` or equivalent boolean defaulting to false across OpenClawConfig, strict zod validation, generated config schema/help/labels, and public configuration docs. Load it at TUI startup and thread it through the TUI render path without calling config loading during render. When enabled, replace OpenClaw-owned TUI emoji/Unicode decorations with deterministic ASCII alternatives in tool display summaries/headers, chat/tool execution output, filterable/searchable/settings selectors, waiting/busy status text, and nearby TUI wrappers; preserve current rich output by default. Do not change channel reaction emoji behavior, provider/plugin message semantics, or unrelated UI/browser surfaces. Add or update focused tests for config validation/generated docs expectations and affected TUI/tool render components, and add a user-facing changelog entry because this is a new config feature.
