## Summary

Keep open: current main still lacks a plain-text TUI accessibility mode, and the related implementation at https://github.com/openclaw/openclaw/pull/9685 was closed unmerged. Current config/docs only expose branding UI fields while TUI render paths still hard-code emoji and Unicode glyphs.

## What This Changes

Request a TUI accessibility config option that renders plain ASCII instead of emoji and Unicode symbols.

## Best Possible Solution

Ship a default-off core TUI plain-text mode, document it in config surfaces, and apply it consistently across tool summaries, selectors, status/waiting text, and component glyphs.

## Reproduction Assessment

Yes, for verifying the missing mode: current-main source shows no accepted or documented config key, strict validation rejects unknown UI fields, and inspected render paths still emit emoji or Unicode glyphs. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Yes, broadly: a default-off core TUI config flag is the maintainable boundary because config validation, generated docs, and built-in TUI renderers are OpenClaw-owned. The exact key name and ASCII replacements still need maintainer agreement.
