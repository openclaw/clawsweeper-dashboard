## Summary

Keep open: current main still lacks a TUI plain-text/accessibility mode, strict config validation would reject the proposed key, and the related implementation PR was closed unmerged.

## What This Changes

The issue requests a documented TUI accessibility config option that replaces emojis and Unicode symbols with screen-reader-friendly ASCII output.

## Best Possible Solution

Ship core TUI plain-text support with rich output unchanged by default and deterministic ASCII alternatives when the documented config flag is enabled.

## Reproduction Assessment

Yes for current source behavior, though this is a feature request rather than a broken existing contract. Current main has no accepted config key and multiple OpenClaw-owned TUI render paths still emit emoji or Unicode glyphs unconditionally.

## Solution Assessment

Yes. A core TUI config flag is the narrow maintainable path because the affected config validation and TUI render code are OpenClaw-owned and cannot be reliably overridden by a plugin.
