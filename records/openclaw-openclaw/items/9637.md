## Summary

Keep open: current main still lacks a documented TUI plain-text accessibility mode, and the config, schema, docs, tool summaries, and TUI render paths still emit emoji or Unicode glyphs without an opt-out.

## What This Changes

The issue requests a TUI accessibility configuration option to disable emojis and Unicode symbols for screen-reader/plain-text use.

## Best Possible Solution

Ship a default-off core TUI plain-text mode that is documented in config surfaces and applied consistently to tool summaries, selectors, status/waiting text, and TUI component glyphs.

## Reproduction Assessment

Yes for current source behavior, although this is a feature request rather than a broken existing contract. The requested config key is absent, strict validation rejects it, and inspected TUI/tool paths still hard-code emoji or Unicode glyphs.

## Solution Assessment

Yes. A default-off core config flag is the narrow maintainable direction because the affected config/schema/docs and render paths are OpenClaw-owned, while dependency selection display can be handled through a wrapper or theme seam after checking the pi-tui API.
