## Summary

Keep open: current main still lacks the requested TUI plain-text accessibility mode, the config key is neither accepted nor documented, and the related implementation PR at https://github.com/openclaw/openclaw/pull/9685 was closed unmerged.

## What This Changes

Add a `ui.accessibility.plainText`-style option to render the TUI without emoji or Unicode symbols for screen-reader users.

## Best Possible Solution

Ship a default-off core TUI plain-text mode that is documented in config surfaces and applied consistently to tool summaries, selectors, status/waiting text, and component glyphs.

## Reproduction Assessment

Yes, for verifying the missing behavior: source inspection of current main shows the requested config key is absent, strict validation rejects it, and inspected TUI/tool paths still hard-code emoji or Unicode glyphs. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. A default-off core config flag is the right boundary because config validation, generated docs, and built-in TUI renderers are OpenClaw-owned and cannot be consistently replaced from a plugin.
