## Summary

Keep open: current main still lacks a documented plain-text TUI accessibility mode, the config schema would reject the proposed key, and the related implementation at https://github.com/openclaw/openclaw/pull/9685 was closed unmerged.

## What This Changes

The issue requests a TUI accessibility configuration option to render plain ASCII text instead of emoji and Unicode symbols.

## Best Possible Solution

Ship a default-off core TUI plain-text mode, document it in config surfaces, and apply it consistently across tool summaries, selectors, settings, status/waiting text, and component glyphs.

## Reproduction Assessment

Yes, for source verification: current main has no accepted or documented TUI plain-text config key, and inspected render paths still emit emoji or Unicode glyphs. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Yes, broadly: a default-off core TUI config flag is the maintainable boundary because config validation, generated docs, and built-in TUI renderers are OpenClaw-owned. The exact option name and ASCII replacements still need maintainer agreement.
