## Summary

Keep open: current main still lacks a plain-text TUI accessibility mode, the linked implementation PR was closed unmerged, and source inspection shows config/schema/docs plus TUI render paths still emit emoji or Unicode glyphs without a switch.

## What This Changes

Request adds a TUI accessibility setting to replace emoji and Unicode symbols with plain ASCII output.

## Best Possible Solution

Ship a default-off core TUI plain-text mode documented in config surfaces and applied consistently to tool summaries, TUI components, selectors, waiting/status text, and pi-tui wrapper seams with deterministic ASCII alternatives.

## Reproduction Assessment

Yes for the current source behavior, although this is a feature request rather than a broken existing contract. The requested config key is absent, strict validation rejects it, and inspected TUI/tool paths still hard-code emoji or Unicode glyphs.

## Solution Assessment

Yes. A default-off core config flag is the narrow maintainable route because the affected config/schema/docs and render paths are OpenClaw-owned, while the dependency selection glyph can be handled through a local wrapper/theme seam.
