## Summary

Keep open: current main still lacks the requested TUI plain-text accessibility mode, the config key is not accepted or documented, and the related implementation at https://github.com/openclaw/openclaw/pull/9685 was closed unmerged.

## What This Changes

Request a TUI accessibility config option that renders tool and selector output without emojis or Unicode glyphs.

## Best Possible Solution

Ship a default-off core TUI plain-text mode, document it in config surfaces, and apply it consistently to tool summaries, selectors, status/waiting text, and component glyphs.

## Reproduction Assessment

Yes, for verifying the missing behavior: source inspection of current main shows the requested config key is absent, strict validation rejects it, and inspected TUI/tool paths still hard-code emoji or Unicode glyphs. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes, with maintainer agreement on naming and replacement text. A default-off core config flag is the right boundary because config validation, generated docs, and built-in TUI renderers are OpenClaw-owned and cannot be consistently replaced from a plugin.
