## Summary

Keep open: current main still lacks a TUI plain-text/accessibility mode, and the config/schema/docs plus TUI/tool-display paths still emit emoji or Unicode glyphs without a switch.

## What This Changes

The issue requests a configurable TUI accessibility/plain-text mode to avoid emoji and Unicode glyph output for screen reader users.

## Best Possible Solution

Ship a default-off core TUI plain-text mode that is documented in config surfaces and applied consistently to tool summaries, selectors, status/waiting text, and TUI component glyphs.

## Reproduction Assessment

Yes for current source behavior, although this is a feature request rather than a broken existing contract. The requested config key is absent, strict validation rejects it, and inspected TUI/tool paths still hard-code emoji or Unicode glyphs.

## Solution Assessment

Yes. A default-off core config flag such as ui.accessibility.plainText is the narrow maintainable direction because the affected config/schema/docs and TUI render paths are OpenClaw-owned.
