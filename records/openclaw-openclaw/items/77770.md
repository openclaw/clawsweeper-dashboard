## What This Changes

TUI `/reset` should rotate generated transcript filenames to the new session id instead of reusing a stale generated basename.

## Best Possible Solution

Land a corrected reset-path fix that rotates OpenClaw-generated transcript basenames to the new session id, preserves explicit custom transcript paths, and includes focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows TUI `/reset` reaches Gateway `sessions.reset`, which creates a new session id while still resolving the new active transcript from the previous `sessionFile`.

## Solution Assessment

Yes. The maintainable fix belongs in Gateway reset transcript path selection with tests for generated and already-stale generated filenames, not in a new config option or TUI-only workaround.
