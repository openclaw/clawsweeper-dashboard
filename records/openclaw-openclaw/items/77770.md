## Summary

Keep open: current main still carries a persisted `sessionFile` into the new Gateway reset session, and the linked fixing PR is still open rather than merged.

## What This Changes

TUI `/reset` should rotate stale generated transcript filenames to the new session id instead of continuing to write to an old `.jsonl` file.

## Best Possible Solution

Land a corrected version of the linked reset-path fix, preserving explicit custom transcript paths while rotating OpenClaw-generated basenames and adding regression coverage for already-stale rows.

## Reproduction Assessment

Yes. Source inspection shows TUI `/reset` reaches `sessions.reset`, which mints a new session id while passing the previous `sessionFile` into a resolver that prefers that stored path.

## Solution Assessment

Yes, keeping this issue open behind the linked fixing PR is the right path. The maintainable fix belongs in Gateway reset transcript path selection, not in TUI command handling or a new config option.
