- `impact:session-state`: The issue can leave a fresh logical session associated with a stale generated transcript filename and mismatched transcript state.

## Summary

Keep open: current main and v2026.6.6 rotate generated reset paths only when the filename matches the stored previous session id, so a historically polluted row with `sessionId=B` and `sessionFile=A.jsonl` can still preserve `A.jsonl`; https://github.com/openclaw/openclaw/pull/80251 is the active but conflicting fix vehicle.

## What This Changes

The issue reports that TUI `/reset` can keep writing a fresh logical session to an older generated transcript filename after upgrades from earlier reset behavior.

## Best Possible Solution

Rebase, review, or replace https://github.com/openclaw/openclaw/pull/80251 with a current-main fix that treats drifted UUID-shaped generated transcript basenames as generated while preserving true custom transcript paths.

## Reproduction Assessment

Yes, at source level: seed a historical row with `sessionId=B` and `sessionFile=A.jsonl`, then `sessions.reset` cannot rewrite `A.jsonl` because the helper only matches `B`, and the resolver preserves the stale candidate.

## Solution Assessment

Yes for the repair direction, but not as a TUI-only workaround. The maintainable fix belongs at the gateway/session path boundary, where generated drifted UUID basenames can be rotated while explicit custom paths remain preserved.
