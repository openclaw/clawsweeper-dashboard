## Summary

Keep open: this PR targets a real plugin callback scoping bug, but the current patch can break valid plugin tools by cloning returned tools with object spread, and the proof is still fixture-test-only rather than real installed or bundled plugin runtime output.

## What This Changes

The PR wraps plugin tool factories plus `prepareArguments` and `execute` callbacks in the owning plugin runtime scope and adds fixture-based regression coverage in `src/plugins/tools.optional.test.ts`.

## Best Possible Solution

Preserve the original `AnyAgentTool` surface with a delegating callback wrapper, add regression coverage for prototype/non-enumerable/class-backed tools, and then provide real runtime proof or choose the proof-positive https://github.com/openclaw/openclaw/pull/87829 path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: shipped and current code call plugin tool factories and cached descriptor runtime callbacks without `withPluginRuntimePluginScope`. I did not run a live runtime repro because this was a read-only review.

## Solution Assessment

No, not as written. Wrapping at the shared materialization boundary is a reasonable fix direction, but the concrete spread clone should be replaced with a delegating wrapper that preserves the original tool's property surface.
