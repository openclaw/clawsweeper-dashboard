## Summary

Keep open. The PR is still an active draft Plugin SDK contract change, and review found one remaining composition regression: once retry metadata is present, the harness sends only the retry instruction and drops the merged revise reason from other hooks.

## What This Changes

The PR adds bounded `before_agent_finalize` retry metadata, hardens plugin run-context cleanup around terminal event handlers, wraps plugin cleanup hooks in timeouts, updates SDK docs/changelog, and adds focused contract tests.

## Best Possible Solution

Keep this as a focused generic Plugin SDK hardening slice, but preserve merged revise instructions when applying retry budgeting, add a regression test for mixed legacy revise plus bounded retry hooks, then proceed through public SDK contract review.

## Reproduction Assessment

Yes. A focused unit test with two `before_agent_finalize` hooks, one returning a legacy `reason` and another returning `retry.instruction`, would reproduce the issue: PR-head normalization returns only the retry instruction to the native relay.

## Solution Assessment

No. The generic host-hook boundary is the right place for the feature, but the retry normalization should combine the merged revise reason with the bounded retry instruction instead of replacing it.
