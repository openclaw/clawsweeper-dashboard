## Summary

Keep open: the split Plugin SDK lifecycle work is useful and exact-head checks are green, but the retry-budget path can drop an independent plain `revise` decision after another plugin's bounded retry is spent.

## What This Changes

This PR adds bounded `before_agent_finalize` retry metadata, hardens plugin run-context and cleanup lifecycles, adjusts Matrix live DM owner pinning, and updates docs, changelog, and tests.

## Best Possible Solution

Keep the split lifecycle PR, but preserve independent plain `revise` decisions when bounded retry candidates are exhausted and add focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: a merged `before_agent_finalize` result with retry candidates always takes the retry-budget branch, and once those candidates are spent it returns `continue`. That source path can suppress a separate plain `revise` reason from another plugin.

## Solution Assessment

No: the overall direction is maintainable, but retry normalization needs one more narrow fix so bounded retry exhaustion only suppresses the spent retry request, not unrelated revise guidance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve plain revise requests after retry budget exhaustion:** `src/agents/harness/lifecycle-hook-helpers.ts:182`
  - body: When a merged `before_agent_finalize` result contains any retry candidates, `normalizeBeforeAgentFinalizeResult` ignores the normal `reason` path once those candidates are over budget and returns `continue`. That means a second plugin that returned `{ action: "revise", reason }` without retry metadata can be silently dropped just because another plugin's bounded retry is spent, even though retry metadata is optional. Track whether a non-retry revise contributed, or otherwise keep unbounded revise reasons separate from spent retry guidance.
  - confidence: 0.86
