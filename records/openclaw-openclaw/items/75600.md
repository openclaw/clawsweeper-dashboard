## Summary

Keep open. Current main still lacks the bounded finalize-retry contract, and the latest PR head still has a P2 retry-merging bug: it keeps only the first valid retry candidate before budget evaluation, so a spent retry from one plugin can suppress a later plugin's still-valid retry request. Several earlier review comments appear fixed, but this blocker remains.

## What This Changes

The PR adds bounded `before_agent_finalize` retry metadata and budgeting, terminal run-context cleanup guards, bounded plugin cleanup timeouts, docs/changelog/API-baseline updates, lifecycle tests, and small Matrix/onboarding fixes.

## Best Possible Solution

Keep the focused Plugin SDK hardening direction, but preserve or evaluate all valid retry candidates until one with remaining budget is found. Add a focused multi-plugin test where the first retry key is exhausted and a later plugin's retry still gets honored, then rebase and rerun the hook/runtime contract checks.

## Reproduction Assessment

Yes. Register two `before_agent_finalize` hooks for one run, have both return valid retry metadata with distinct keys, exhaust the first hook's `maxAttempts`, and verify the second hook's retry is still honored instead of the merged result becoming `continue`.

## Solution Assessment

No, not as submitted. The seam placement is maintainable, but selecting one retry during merge before budget evaluation lets an exhausted plugin suppress another plugin's valid retry request; choosing after budget evaluation or carrying retry candidates is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Honor later retry requests after a spent retry:** `src/plugins/hooks.ts:341`
  - body: When multiple finalize hooks return retry metadata, this merge keeps the first valid retry and drops later candidates before the harness checks budget. Once that first retry key exceeds `maxAttempts`, normalization returns `continue`, so a later plugin with unused retry budget never gets its requested extra pass. Preserve retry candidates through merge or choose a retry after budget evaluation.
  - confidence: 0.87
