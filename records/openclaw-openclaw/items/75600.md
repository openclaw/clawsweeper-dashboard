## Summary

Keep open. Current main does not contain the bounded finalize-retry contract, and the latest PR head fixes the earlier reason-preservation concern, but it still has a narrow retry-normalization bug: malformed retry metadata can throw during hook merging/normalization and drop an otherwise valid revise decision.

## What This Changes

The PR adds bounded `before_agent_finalize` retry metadata, run-context cleanup hardening, bounded plugin cleanup timeouts, SDK docs/API-baseline/changelog updates, and focused lifecycle contract tests.

## Best Possible Solution

Keep this as a focused generic Plugin SDK hardening slice, but defensively normalize retry metadata at the hook boundary, add regression coverage for malformed retry payloads, then rerun the targeted SDK hook/cleanup tests and exact-head API/lint checks before merge.

## Reproduction Assessment

Yes. A focused unit test where a `before_agent_finalize` hook returns `{ action: "revise", reason: "keep going", retry: {} as never }` or a non-string `retry.instruction` reproduces the bug because PR head calls `.trim()` without a string guard and drops the revise result through fail-open error handling.

## Solution Assessment

No. The generic host-hook boundary is the right place for bounded finalize retry, but the current implementation should treat invalid retry metadata as absent while preserving the revise reason, and it should apply the same string guards in both hook merging and harness normalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard retry metadata before trimming it:** `src/plugins/hooks.ts:325`
  - body: Hook return values are runtime data, but `normalizeRetry` assumes `retry.instruction` is always a string. If a plugin returns malformed retry metadata such as `{ retry: {} }` or a non-string instruction alongside a valid revise reason, this throws during merge and the host fail-opens by discarding the revise decision instead of ignoring the bad retry fields. Check `typeof retry?.instruction === "string"` before trimming, and apply the same guard to optional retry string fields used during normalization.
  - confidence: 0.88
