## Summary

Keep open. Current main does not contain the bounded finalize retry and terminal run-context cleanup behavior, and the latest PR head still has a concrete pending-handler bookkeeping race after terminal cleanup timeout. Earlier review blockers around malformed retry metadata, preserving revise reasons, and preserving the expired terminal marker appear addressed on the latest head.

## What This Changes

The PR adds Plugin SDK `before_agent_finalize` retry metadata, bounded retry budgeting, terminal run-context cleanup hardening, bounded plugin cleanup timeouts, docs/API baseline updates, changelog text, and focused lifecycle contract tests.

## Best Possible Solution

Keep the focused Plugin SDK hardening direction, but guard pending handler map deletion by Set identity or a per-wait token, add a fake-timer regression for the timeout-plus-replacement case, then rebase and rerun the targeted Plugin SDK hook/runtime checks before merge.

## Reproduction Assessment

Yes. A focused fake-timer test can time out a first terminal handler, dispatch another terminal handler for the same run, let the first handler settle, and then assert the newer pending-handler Set is still tracked for later terminal waits.

## Solution Assessment

No. The overall retry and cleanup approach is the right generic SDK seam, but the current timeout cleanup is not the narrowest safe implementation because old handler finalizers can erase newer pending state for the same run id.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard pending handler cleanup by set identity:** `src/plugins/host-hook-runtime.ts:123`
  - body: On terminal timeout this deletes `pendingAgentEventHandlersByRunId` while the timed-out handler promises still hold their original `handlers` Set. When one of those old promises later settles, `trackAgentEventHandler` unconditionally deletes the run id, which can remove a newer Set installed by a repeated terminal event and make later cleanup waits miss active handlers. Guard that delete with an identity check, or avoid replacing/deleting the map entry on timeout.
  - confidence: 0.82
