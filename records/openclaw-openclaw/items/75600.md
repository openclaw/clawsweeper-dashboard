## Summary

Keep open. Current main does not contain this Plugin SDK retry/cleanup hardening, and the PR is a plausible focused follow-up to the merged host-hook foundation, but the latest head is not merge-ready because the timeout path can corrupt same-run pending handler tracking and GitHub reports the branch as dirty against main.

## What This Changes

The PR adds bounded `before_agent_finalize` retry metadata, per-run retry budgeting, terminal run-context cleanup hardening, bounded plugin cleanup timeouts, SDK docs/API-baseline/changelog updates, and lifecycle contract tests.

## Best Possible Solution

Keep this as a focused Plugin SDK hardening slice, but make pending handler tracking identity-safe across terminal cleanup timeouts, rebase it cleanly onto current main, then rerun the focused hook/runtime contract tests and SDK/lint checks before merge.

## Reproduction Assessment

Yes. A focused fake-timer contract can trigger a terminal handler timeout, install another same-run pending handler, release the first handler, and assert that the older finalizer cannot remove the newer pending-handler Set before the newer handler settles or times out.

## Solution Assessment

No. The overall retry and cleanup direction is the right generic SDK seam, but the current timeout implementation is not the narrowest safe version because it deletes per-run pending-handler state while old finalizers can still run; an identity-guarded cleanup path is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Keep pending handler tracking identity-safe:** `src/plugins/host-hook-runtime.ts:123`
  - body: `waitForTerminalEventHandlers` deletes the run's pending-handler entry on timeout while the timed-out handlers still have `.finally()` callbacks that close over the old Set. If another handler for the same run is tracked after the timeout, the old finalizer can delete that newer Set from `pendingAgentEventHandlersByRunId`, so later terminal cleanup may stop tracking active handlers. Guard the delete with a Set identity check, or avoid replacing/deleting the entry in a way old finalizers can clobber.
  - confidence: 0.82
