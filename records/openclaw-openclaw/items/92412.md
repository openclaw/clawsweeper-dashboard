## Summary

Current main still throws on unsupported programmatic thinking overrides, and an open closing PR now owns the narrow primary fix; keep the issue open until that PR has focused branch coverage and live spawn/task-state proof, then land or reject it.

## What This Changes

Fix `sessions_spawn` so unsupported programmatic thinking levels do not kill an already-accepted child run, while preserving clear rejection for user-entered one-shot CLI thinking levels.

## Best Possible Solution

Land the narrow primary fix with focused tests proving unsupported `thinkOverride` falls back while unsupported `thinkingOnce` still rejects, plus a live parallel spawn showing every accepted child reaches a terminal state; track ack metadata or catalog exposure separately if maintainers want those API additions.

## Reproduction Assessment

Yes at source level: current main deterministically routes programmatic `thinking` into `thinkOverride` and then throws for an unsupported level before fallback. The issue also supplies matching live task output, although this review did not independently run the multi-agent setup.

## Solution Assessment

Yes for the primary failure: distinguishing user-entered `thinkingOnce` from programmatic `thinkOverride` is the narrowest existing-boundary repair. The proposed ack and model-list additions are separate product/API decisions and should not block the focused fix.
