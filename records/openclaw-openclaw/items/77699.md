## Summary

Close as implemented: current main already skips scheduled heartbeat model/reply/send work when `HEARTBEAT.md` is effectively empty, and task-block heartbeats only run when tasks are due; the empty/comment-only skip shipped in `v2026.4.10` and is included in the latest release line.

## What This Changes

Request heartbeat to avoid model calls when `HEARTBEAT.md` has no actionable periodic tasks and resume when actionable heartbeat content exists.

## Best Possible Solution

Keep the shipped no-new-config preflight skip behavior; any future file-watcher or `every: auto` mode should be a separate product discussion only if users need more than avoiding empty heartbeat model calls.

## Reproduction Assessment

Not applicable as a feature request. Current main has source and regression-test evidence showing empty/comment-only `HEARTBEAT.md` interval ticks skip before reply/model/send calls, while actionable files still run.

## Solution Assessment

Yes. The current preflight gate is narrower than adding `every: auto`: it preserves the existing cadence contract, avoids wasted model calls for empty files, and still allows explicit wake/cron events and actionable heartbeat content to run.
