## Summary

Current main already solves the central cost problem: scheduled heartbeat preflight skips model/reply work for effectively empty HEARTBEAT.md files and due-only task mode skips when no task interval is due.

## What This Changes

The issue requests an automatic heartbeat behavior that avoids model invocations when HEARTBEAT.md has no actionable periodic work.

## Best Possible Solution

Keep the current preflight-based skip behavior and due-task gating; a separate file-watcher or every:auto mode would be a new product choice, not needed for the reported cost leakage.

## Reproduction Assessment

Not applicable: this is a feature/cost-control request rather than a broken existing contract. Source and tests provide a high-confidence current-main verification path for the implemented behavior.

## Solution Assessment

Yes. The implemented preflight skip is narrower than a new config mode because it avoids model calls whenever the heartbeat file is empty while preserving existing heartbeat cadence and wake/cron behavior.
