## Summary

Keep open. Current main does not contain this repotype setup, and the latest PR head still has two merge-blocking config semantics problems: the literal `#` required section will fail normal skill docs, and file-only skill rules cannot enforce the advertised missing-`SKILL.md` check.

## What This Changes

The PR adds a root `repotype.yaml`, a `schemas/skill.frontmatter.schema.json`, and YAML frontmatter for `skills/canvas/SKILL.md` to validate repository structure and skill metadata.

## Best Possible Solution

Revise the config to encode OpenClaw's actual skill/plugin structure and repotype's actual rule model: folder rules should enforce required `SKILL.md` files for supported skill directory shapes, file rules should validate frontmatter only on matching files, and any CI or package-script adoption should be a separate maintainer-approved step.

## Reproduction Assessment

Yes. Static reproduction is high-confidence from the latest PR head and repotype 0.1.5 source: normal skill files use H1 headings, while repotype checks exact H2-or-deeper section names and missing-file checks come from folder rules.

## Solution Assessment

No. The proposed direction is reasonable, but the current patch is not the best mergeable solution until the config removes the literal heading requirement and uses folder-level required-file enforcement for the skill layouts OpenClaw supports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Remove the literal heading requirement:** `repotype.yaml:21-30`
  - body: `requiredSections` is not a wildcard for "has any heading." Repotype extracts exact `##`-or-deeper section names, so `"#"` requires a literal `## #` section and will report normal skill files as missing that section. Remove these entries or replace them with real shared section names.
  - confidence: 0.96
- **[P2] Use folder rules to require skill files:** `repotype.yaml:15-25`
  - body: These `files` rules only validate `SKILL.md` files that already exist, so a new skill directory without `SKILL.md` will not trigger the advertised missing-skill check. Add `folders` rules with `requiredFiles: ["SKILL.md"]` for the supported top-level, grouped, and plugin-packaged skill directory shapes.
  - confidence: 0.94
