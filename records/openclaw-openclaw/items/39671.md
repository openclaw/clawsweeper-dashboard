## Summary

Keep open: current main does not contain the requested repotype enforcement, and the latest PR head still has concrete config-semantics blockers against repotype's rule model.

## What This Changes

The PR adds a root repotype.yaml, a SKILL.md frontmatter schema, and canvas skill frontmatter to validate repository structure for skills, plugins, packages, docs, and common repo files.

## Best Possible Solution

Revise repotype.yaml to encode OpenClaw's actual skill and plugin layouts with folder-level required-file rules, remove or replace the literal heading requirement, and leave CI enforcement as a separate maintainer decision.

## Reproduction Assessment

Yes. Source reproduction is high-confidence from latest PR head 6a1ecf38870fb0caf522e1d02fd7a995533db071 and repotype 0.1.5 source: requiredSections checks exact ## headings, while missing required files are enforced by folders[].requiredFiles rather than file globs.

## Solution Assessment

No. The requested direction is reasonable, but the current patch is not the best mergeable solution until it uses repotype's folder rule model and removes or replaces the literal "#" section requirement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Remove the literal heading requirement:** `repotype.yaml:20-30`
  - body: `requiredSections` is checked against exact `##`-or-deeper heading text, so `"#"` requires a literal `## #` section and does not prove that a skill has a title. This makes normal skill files report `missing_section`; remove these entries or require real shared section names.
  - confidence: 0.96
- **[P2] Use folder rules for required skill files:** `repotype.yaml:15-25`
  - body: These `files` rules only validate SKILL.md files that already exist. A new skill directory without SKILL.md is still accepted by the catch-all file rules, so the advertised missing-skill check does not run; add folder rules with `requiredFiles: ["SKILL.md"]` for the supported skill directory shapes.
  - confidence: 0.94
