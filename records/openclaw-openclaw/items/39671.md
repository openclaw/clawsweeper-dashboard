## Summary

Keep open; this PR is not ready to merge. The latest head still has source-reproducible rule-contract blockers, lacks contributor real behavior proof, and adds a new repository-lint policy surface that needs maintainer approval.

## What This Changes

The PR adds a root `repotype.yaml`, a `SKILL.md` frontmatter JSON Schema, and canvas skill frontmatter for repository structure validation.

## Best Possible Solution

Adopt repotype only after maintainer policy approval, with folder-level `requiredFiles` rules for skills/packages/plugins, coverage for grouped and plugin-declared skill roots, valid section rules or no section rule, and redacted validation output.

## Reproduction Assessment

Yes. Source inspection against PR head and repotype 0.1.5 shows normal `SKILL.md` files can fail the literal `## #` section requirement, and missing required files are not enforced by `files[]` globs.

## Solution Assessment

No. The direction may be useful, but the current config is not the narrowest maintainable solution until it follows repotype's folder-rule contract and maintainers approve the new lint policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Use folder rules for required files:** `repotype.yaml:15-42`
  - body: The `files[]` globs only validate paths that already exist. A new `skills/foo/` without `SKILL.md`, `packages/foo/` without `package.json`, or `extensions/foo/` without `openclaw.plugin.json` will not match these rules, so the advertised structure enforcement does not happen. Model those directories with `folders[].requiredFiles` instead.
  - confidence: 0.94
- **[P2] Remove the literal section requirements:** `repotype.yaml:20-30`
  - body: repotype extracts only `##`-or-deeper headings and reports missing sections as `## ${section}`, so `requiredSections: ["#"]` requires a literal `## #` section rather than checking for an H1. Drop this rule or require real section names that `SKILL.md` files actually contain.
  - confidence: 0.96
