## Summary

Keep open: current main does not have this repository-structure enforcement, but the PR is not mergeable as-is because the required-file checks are modeled as existing-file globs and the SKILL.md section rule requires a literal section that normal skills do not have.

## What This Changes

The branch adds a root `repotype.yaml`, a SKILL.md frontmatter JSON schema, and YAML frontmatter for `skills/canvas/SKILL.md` to introduce optional repository structure validation.

## Best Possible Solution

Adopt repository-structure linting only if maintainers want this policy, with folder-rule required-file checks, valid markdown section checks, OpenClaw path coverage, and recorded validation output.

## Reproduction Assessment

Yes for the PR defects: PR-head source plus upstream repotype source show the required-file and section-rule contract problems. I did not run `repotype validate` because this review kept the checkout read-only.

## Solution Assessment

No. The direction may be useful, but this implementation should use folder-level `requiredFiles`, remove or replace the literal section rules, and wait for maintainer policy approval plus real validation proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Use folder rules for required files:** `repotype.yaml:15-42`
  - body: The advertised missing-file checks will not fire because these `files[]` globs only apply to files that already exist. A new skill, package, or plugin directory missing `SKILL.md`, `package.json`, or `openclaw.plugin.json` never matches these rules; model those directories with `folders[].requiredFiles` instead.
  - confidence: 0.94
- **[P2] Remove the literal section requirements:** `repotype.yaml:20-30`
  - body: `requiredSections: ["#"]` does not check for an H1. repotype extracts only `##`-or-deeper section names and reports missing sections as `## ${section}`, so this asks for a literal `## #` section and will make normal SKILL.md files fail.
  - confidence: 0.96
