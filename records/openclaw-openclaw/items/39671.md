## Summary

Keep open. This PR is not implemented on current main and the latest patch still has two merge-blocking config semantics issues: the `requiredSections: ["#"]` rule would fail normal skill files, and the skill checks are file-only rules that cannot catch a skill directory missing `SKILL.md`.

## What This Changes

The PR adds a root `repotype.yaml`, a `schemas/skill.frontmatter.schema.json`, and frontmatter for `skills/canvas/SKILL.md` to opt into repository-structure and skill-metadata validation.

## Best Possible Solution

Revise the config to match repotype semantics and OpenClaw's skill/plugin contracts: use folder rules with `requiredFiles: ["SKILL.md"]` for top-level, grouped, and plugin-packaged skill directories, remove the literal `#` required section or replace it with real shared section names, and leave CI or package-script adoption as an explicit maintainer decision.

## Reproduction Assessment

Yes. Static reproduction is high-confidence from the PR diff and repotype 0.1.5 source: `requiredSections: ["#"]` checks for a literal `## #` section, and file-only globs cannot report a skill directory that lacks `SKILL.md`.

## Solution Assessment

No. The direction may be useful, but this patch is not the best mergeable solution until the config uses folder-level required-file enforcement and removes the literal heading requirement; a pinned invocation or explicit opt-in CI policy would be the safer adoption path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Remove the literal heading requirement:** `repotype.yaml:21`
  - body: `requiredSections` is not a wildcard for "has any heading." Repotype extracts exact `##`-or-deeper section names, so `"#"` requires a section rendered as `## #` and will report missing-section errors for normal skill files. Remove these entries from both skill rules or replace them with real shared section names.
  - confidence: 0.96
- **[P2] Use folder rules to require skill files:** `repotype.yaml:15-16`
  - body: These `files` rules only validate `SKILL.md` files that already exist, so a new skill directory without `SKILL.md` will not trigger the advertised missing-skill check. Add `folders` rules with `requiredFiles: ["SKILL.md"]` for the relevant top-level, grouped, and plugin-packaged skill directory patterns.
  - confidence: 0.93
