## Summary

Keep this PR open: the repo-linting direction is useful, but the latest head still has two merge-blocking config semantics issues in `repotype.yaml`, and current `main` does not already provide an equivalent repotype setup.

## What This Changes

The PR adds a root `repotype.yaml`, `schemas/skill.frontmatter.schema.json`, and frontmatter for `skills/canvas/SKILL.md` to validate repository structure and SKILL.md metadata.

## Best Possible Solution

Revise `repotype.yaml` to encode OpenClaw's actual skill/plugin layouts with folder-level required-file rules, keep file rules focused on frontmatter/schema checks, and leave CI or package-script adoption to a separate maintainer-approved step.

## Reproduction Assessment

Yes. Static reproduction is high-confidence from the latest PR head and repotype 0.1.5 source: `requiredSections` checks exact `##` headings, and missing required files come from folder rules rather than file globs.

## Solution Assessment

No. The requested direction is reasonable, but the current patch is not the best mergeable solution until it uses repotype's folder rule model and removes the literal `#` section requirement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Remove the literal heading requirement:** `repotype.yaml:20-30`
  - body: `requiredSections` is not a wildcard for “has any heading.” Repotype extracts exact `##`-or-deeper section names, so `"#"` requires a literal `## #` section and will report normal skill files as missing that section. Remove these entries or replace them with real shared section names.
  - confidence: 0.96
- **[P2] Use folder rules to require skill files:** `repotype.yaml:15-25`
  - body: These `files` rules only validate `SKILL.md` files that already exist, so a new skill directory without `SKILL.md` will not trigger the advertised missing-skill check. Add `folders` rules with `requiredFiles: ["SKILL.md"]` for supported top-level, grouped, and plugin-packaged skill directory shapes.
  - confidence: 0.94
