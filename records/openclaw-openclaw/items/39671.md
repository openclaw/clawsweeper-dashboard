## Summary

Keep open. The PR is not implemented on current main, but the latest head still has source-backed repotype contract bugs that make the proposed structure enforcement weaker than claimed, and the external PR still lacks real validation proof plus maintainer approval for adopting a new repo linter.

## What This Changes

Adds a root repotype configuration, a SKILL.md frontmatter JSON Schema, and canvas skill frontmatter for repository-structure validation.

## Best Possible Solution

Adopt repotype only if maintainers want this new repo policy surface, with folder-level required-file checks, no unsupported literal section rule, schema alignment with OpenClaw skill contracts, and redacted real validation output.

## Reproduction Assessment

Yes. The PR blockers are source-reproducible from repotype@0.1.5: `requiredSections` is matched against extracted `##` headings, and missing files are checked through folder `requiredFiles`, while this PR uses file globs and a literal `"#"` section.

## Solution Assessment

No. The direction may be useful, but this is not the best mergeable version while it cannot enforce missing `SKILL.md` files as claimed and adds an unsupported literal section requirement; a folder-rule based config plus maintainer approval is the safer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Remove the literal section requirements:** `repotype.yaml:20-30`
  - body: `repotype@0.1.5` extracts only `##`-or-deeper headings and reports missing sections as `## ${section}`, so `requiredSections: ["#"]` makes normal `SKILL.md` files fail with missing `## #` instead of checking for an H1. Drop this rule or require real section names.
  - confidence: 0.96
- **[P2] Require skill files with folder rules:** `repotype.yaml:15-25`
  - body: These `files` globs only run for existing `SKILL.md` files. A new `skills/foo/`, supported grouped skill directory, or plugin skill directory without `SKILL.md` will not hit this rule, because repotype emits missing-file diagnostics from `folders[].requiredFiles`; model the skill roots as folder rules before claiming missing skills are enforced.
  - confidence: 0.94
