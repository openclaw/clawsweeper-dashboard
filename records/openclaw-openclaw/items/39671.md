## Summary

Keep open: current main does not contain this repotype setup, and latest PR head still has source-reproducible repotype contract bugs that would make the config fail or miss its advertised required-file checks.

## What This Changes

The PR adds a root repotype.yaml, a SKILL.md frontmatter schema, and frontmatter for skills/canvas/SKILL.md to support optional repository structure validation.

## Best Possible Solution

Revise repotype.yaml to model OpenClaw's actual skill, plugin, and package layouts with folders[].requiredFiles, remove or replace the literal "#" section checks, and leave CI/tool adoption as a separate maintainer decision.

## Reproduction Assessment

Yes, source-reproducible. At PR head 6a1ecf38870fb0caf522e1d02fd7a995533db071, the config uses literal "#" requiredSections and files[] globs for required-file checks, while repotype@0.1.5 checks requiredSections against exact ## headings and missing files through folders[].requiredFiles.

## Solution Assessment

No. The requested direction may be useful, but the current patch is not the narrowest maintainable solution until required files are expressed as folder rules and the literal section checks are removed or replaced with real shared ## sections.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Remove the literal section requirement:** `repotype.yaml:20-21`
  - body: `requiredSections` is compared against exact `##`-or-deeper heading text in repotype@0.1.5, so `"#"` requires a literal `## #` section. Normal `SKILL.md` files have an H1 title plus real section names, so this will report `missing_section` for matched skills instead of validating that a title exists.
  - confidence: 0.96
- **[P2] Use folder rules to require SKILL.md:** `repotype.yaml:15-16`
  - body: This file rule only validates `SKILL.md` files that already exist. Missing files are enforced through `folders[].requiredFiles`, and this config has no folder rule for `skills/*`, so a new skill directory without `SKILL.md` is not caught by the advertised check.
  - confidence: 0.95
- **[P2] Require package and plugin files with folder rules:** `repotype.yaml:32-42`
  - body: The `package-json` and `extension-manifest` file globs also match only existing files. A new package or plugin directory missing `package.json` or `openclaw.plugin.json` can still be classified by the catch-all package/extension rules with `unmatchedFiles: allow`, so the package and plugin presence checks are inert.
  - confidence: 0.92
