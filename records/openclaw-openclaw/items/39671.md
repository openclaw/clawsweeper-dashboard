## Summary

Keep open. Current main still has no repotype config or skill frontmatter schema, and the latest PR head still has source-reproducible rule correctness blockers plus missing contributor real behavior proof; adopting a new repository-structure linter also needs maintainer policy judgment.

## What This Changes

This PR adds a root repotype configuration, a SKILL.md frontmatter JSON Schema, and canvas skill frontmatter to validate repository structure for skills, packages, plugins, docs, scripts, and related files.

## Best Possible Solution

Adopt repotype only if maintainers want the new policy, with folder-level requiredFiles rules for direct, grouped, and plugin skills, valid section requirements or no section rule, schema alignment with OpenClaw skill docs, and redacted validation output.

## Reproduction Assessment

Yes. The PR blockers are source-reproducible from the PR head and repotype@0.1.5: `requiredSections: ["#"]` cannot match repotype's ##-only section extraction, and file globs cannot enforce missing SKILL.md files.

## Solution Assessment

No. The direction may be useful, but this is not the best mergeable version until the rules match repotype's folder and section contracts and maintainers approve the new repo-lint policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Remove the literal section requirements:** `repotype.yaml:20-30`
  - body: repotype@0.1.5 extracts only `##`-or-deeper headings and reports missing sections as `## ${section}`, so `requiredSections: ["#"]` makes normal SKILL.md files fail with a missing `## #` section. Drop this rule or require real section names.
  - confidence: 0.96
- **[P2] Require missing skill files with folder rules:** `repotype.yaml:15-25`
  - body: These `files` globs only validate SKILL.md files that already exist. A new direct skill, supported grouped skill, or plugin skill directory without SKILL.md will not hit this rule; model the directories with `folders[].requiredFiles` instead.
  - confidence: 0.94
