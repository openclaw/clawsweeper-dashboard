## Summary

Keep this PR open: the feature direction matches the merged commentary-progress path, but the current patch re-emits cumulative Claude assistant text under fresh item ids, which can duplicate earlier commentary in visible progress drafts, and the external PR still lacks real Claude CLI/channel proof.

## What This Changes

The PR adds an optional Claude CLI JSONL parser callback for commentary before tool-use blocks, wires it through regular and live Claude CLI execution, and adds parser tests.

## Best Possible Solution

Emit only the unflushed commentary segment, or use a stable update identity matching the Codex preamble pattern, then add text-tool-text-tool coverage and real Claude CLI/channel proof before merge.

## Reproduction Assessment

Yes, by source inspection: a Claude stream with text A, tool_use, text B, tool_use will pass assistantText.trim() as A and then A+B, while the runner gives those events different item ids. The channel compositor appends those as separate progress lines.

## Solution Assessment

No. The requested direction is reasonable, but the patch should either emit only the suffix since the last flush or update a stable preamble item identity like the Codex projector.
