## Summary

Keep open: current main still has a source-level stale menu-bar activity path, and this PR targets the right macOS surface, but the fixed timeout can hide valid quiet work as idle and the PR still lacks required real macOS behavior proof.

## What This Changes

The PR adds a macOS `WorkActivityStore` watchdog, fixed-age stale activity eviction, all-activity clearing, timestamp refreshes, role recomputation, and new Swift tests.

## Best Possible Solution

Repair orphaned menu-bar activity through explicit terminal/disconnect/orphan handling or state-aware cleanup that preserves valid quiet jobs/tools, with deterministic clock-based coverage and real macOS overload proof.

## Reproduction Assessment

Yes, at source level: current main routes gateway job/tool events into `WorkActivityStore`, and a started or streaming job remains active until a terminal state arrives. I did not run a live macOS provider-overload setup in this read-only review.

## Solution Assessment

No. A fixed three-minute timeout is not the narrowest maintainable fix because it conflates orphaned terminal events with valid quiet active work; explicit orphan/disconnect handling or a state-aware stale UI would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve quiet active work during stale cleanup:** `apps/macos/Sources/OpenClaw/WorkActivityStore.swift:337-344`
  - body: The watchdog removes jobs and tools whenever `lastUpdate` is older than the fixed threshold, but that timestamp is only refreshed on streaming job updates and tool starts. A valid non-streaming model call or long-running tool with no progress events can pass this threshold, making the menu bar report idle while work is still active.
  - confidence: 0.9
- **[P2] Age stale test activity before expecting idle:** `apps/macos/Tests/OpenClawIPCTests/WorkActivityStoreTests.swift:171-173`
  - body: This test calls `evictStaleActivities()` immediately after a fresh `started` event and expects idle, but production eviction requires the activity to be older than `jobStaleThreshold`. As written, the test fails the intended behavior or pressures the implementation toward evicting fresh active work.
  - confidence: 0.88
