## Summary

Keep open for contributor and maintainer action: the PR targets a real macOS menubar stale-working symptom, but the fixed-age watchdog can show idle during legitimate quiet active work, one new test contradicts the 180-second threshold, and real macOS proof is missing.

## What This Changes

The PR adds a macOS WorkActivityStore watchdog that evicts stale job/tool UI activity after 180 seconds and adds Swift tests for watchdog startup, role refresh, eviction, and cleanup.

## Best Possible Solution

Use a bounded macOS activity recovery design that clears truly orphaned UI state without dropping quiet active work solely because no events arrived for a fixed wall-clock interval.

## Reproduction Assessment

Yes at source level: current main keeps a started/streaming job active until a terminal event is received, so a missing terminal event can leave the menubar in a working state. I did not establish a live provider-overload macOS reproduction in this read-only review.

## Solution Assessment

No, not as written. A watchdog may be part of the fix, but fixed-age eviction is not the best solution because it also removes valid quiet active work from the displayed state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve quiet active work before evicting it:** `apps/macos/Sources/OpenClaw/WorkActivityStore.swift:338`
  - body: This watchdog removes every job/tool whose last UI event is older than 180 seconds. A legitimate non-streaming model call or long shell tool can have no intermediate events for more than three minutes, so the menubar would report idle while work is still active. The stale cleanup needs a stronger signal than elapsed time alone, or coverage that proves quiet active work is preserved.
  - confidence: 0.89
- **[P2] Age the activity before expecting stale eviction:** `apps/macos/Tests/OpenClawIPCTests/WorkActivityStoreTests.swift:173`
  - body: This test starts a fresh job and immediately expects eviction to clear it, but the production code only evicts activities older than the 180-second threshold. As written, the assertion contradicts the intended threshold; age the activity through an injected clock or test hook before expecting idle.
  - confidence: 0.86
