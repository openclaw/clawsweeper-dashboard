## Summary

Keep open: the stale macOS activity problem is source-reproducible, but this branch still has a correctness blocker around fixed-timeout cleanup of quiet active work and lacks real macOS after-fix proof.

## What This Changes

The PR adds a macOS WorkActivityStore watchdog that clears activity display state after a fixed timeout, refreshes some activity timestamps, adds clearAllActivities(), and extends Swift tests.

## Best Possible Solution

Use state-aware orphan/disconnect handling or a stale display state that preserves quiet active work, backed by deterministic tests and real macOS overload proof.

## Reproduction Assessment

Yes, at source level: current main routes gateway job/tool events into WorkActivityStore and a started or streaming job remains active until a terminal event arrives. I did not establish a live macOS provider-overload reproduction in this read-only review.

## Solution Assessment

No. A fixed three-minute timeout is not the narrowest maintainable solution because it conflates lost terminal events with valid quiet active work; explicit orphan/disconnect handling or a state-aware stale display would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve quiet active work during stale cleanup:** `apps/macos/Sources/OpenClaw/WorkActivityStore.swift:337-344`
  - body: The watchdog removes jobs and tools whenever lastUpdate is older than the fixed threshold, but that timestamp is only refreshed on selected streaming and tool-start events. A valid non-streaming model call or long-running tool with no progress events can pass this threshold, making the menu bar report idle while work is still active.
  - confidence: 0.9
- **[P2] Age stale test activity before expecting idle:** `apps/macos/Tests/OpenClawIPCTests/WorkActivityStoreTests.swift:171-173`
  - body: This test calls evictStaleActivities() immediately after a fresh started event and expects idle, but production eviction requires the activity to be older than jobStaleThreshold. As written, the test fails the intended behavior or pressures the implementation toward evicting fresh active work.
  - confidence: 0.88
