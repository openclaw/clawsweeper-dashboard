## Summary

Keep open: the PR targets a real source-reproducible macOS menu-bar stale-activity problem, but the current patch can make valid quiet long-running work appear idle, includes a stale-eviction test whose expectation conflicts with the production threshold, and still lacks required real macOS after-fix proof.

## What This Changes

Adds a macOS WorkActivityStore watchdog that periodically evicts stale job/tool activity and expands Swift tests for watchdog startup, streaming refresh, role refresh, clear-all, and stale eviction behavior.

## Best Possible Solution

Recover orphaned menu-bar activity with an explicit gateway/disconnect/orphan signal or state-aware cleanup that preserves valid quiet work, backed by deterministic clock-based tests and real macOS overload proof.

## Reproduction Assessment

Yes, at source level. Current main routes agent job events into WorkActivityStore and keeps started/streaming activity active until a non-start state arrives, so a missing terminal event leaves the menu-bar activity active indefinitely; I did not run a live macOS overload setup in this read-only review.

## Solution Assessment

No. A fixed three-minute timeout is not the narrowest maintainable fix because it conflates lost terminal events with valid quiet work; explicit orphan/disconnect handling, gateway-side terminal-event repair, or a visible stale state would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve quiet active work during stale cleanup:** `apps/macos/Sources/OpenClaw/WorkActivityStore.swift:338-344`
  - body: The watchdog removes jobs/tools whenever `lastUpdate` is older than 180 seconds, but `lastUpdate` only refreshes on streaming job events and tool starts. A valid non-streaming model call or long tool invocation can run quietly past that threshold, so the menu can flip to idle while work is still active.
  - confidence: 0.9
- **[P2] Age stale-eviction test data before asserting idle:** `apps/macos/Tests/OpenClawIPCTests/WorkActivityStoreTests.swift:171-173`
  - body: This test calls `evictStaleActivities()` immediately after a fresh `started` event, but production only removes entries older than `jobStaleThreshold`. Use an injected clock or another test seam to make the activity genuinely stale before expecting idle.
  - confidence: 0.88
