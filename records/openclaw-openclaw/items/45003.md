## Summary

Keep open: current main still has no restore/re-index command and no approved archive-history contract for reset transcripts. The related restore and archive-read PRs remain open with product, proof, and safety questions, so this is not ready for cleanup closure.

## What This Changes

The issue requests a utility or product path to restore `.jsonl.reset.<timestamp>` session archives and rebuild session indexing after daily reset rotation hides prior history.

## Best Possible Solution

Choose one backward-compatible archive recovery contract: keep current history as the default, add explicit bounded archive/logical-history access with segment metadata, and add a mutating restore/re-index command only after maintainers approve identity, ownership, and retention semantics.

## Reproduction Assessment

Yes, at source level. Current main still creates `.jsonl.reset.<timestamp>` archives and can prune them, while `chat.history`, `sessions_history`, and slash commands provide no restore/re-index or archive-history access path.

## Solution Assessment

No for a standalone rename/re-index script as the durable repo-level solution. The safer path is an approved archive-read contract first, with mutating restore behavior added only after session identity, visibility, and retention semantics are settled.
