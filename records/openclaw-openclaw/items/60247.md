## Summary

Keep open. Current main still lacks proof that the reported Control UI CJK streaming parse failure is fixed: the browser client parses each Gateway WebSocket message as standalone JSON with no reassembly path, while the server broadcast path sends one complete JSON text frame and the recent chat-delta protocol still includes the full assistant message text in delta payloads.

## What This Changes

The issue requests reliable Control UI streaming display for long Chinese/CJK responses when UTF-8 multi-byte content is split across transport or stream boundaries.

## Best Possible Solution

Capture a raw failing Control UI Gateway frame and matching server/provider logs, or add a deterministic malformed-frame CJK regression, then fix the proven boundary while leaving provider failover work to the canonical provider-streaming issue.
