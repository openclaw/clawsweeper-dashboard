## What This Changes

The issue reports a subagent abort/retained session lock leak that leaves `.jsonl.lock` behind and blocks later messages to the session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95854 or an equivalent bounded dispose cleanup fix, then close this issue after merge; handle abort-threshold default tuning separately.

## Reproduction Assessment

Yes. At source level, start a retained `withSessionWriteLock()` operation that never settles, then call `dispose()`; current main can wait or return before releasing `heldLock`.

## Solution Assessment

Yes for the lock leak: a bounded retained-idle wait with force-release in final dispose is the narrow owner-boundary fix. Lowering `stuckSessionAbortMs` defaults is a separate product/default decision.
