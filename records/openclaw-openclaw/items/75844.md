Keep open: multiple recent reports still point to a real TUI hang, and current main shows relevant mitigations rather than live proof that the WSL2/macOS startup/input freezes and raw group-switch service repair behavior are fixed.

The issue reports a regression where `openclaw tui` freezes during Gateway connection or first input while service status/doctor mishandle a Docker group-switch wrapper.

## Best Possible Solution

Reproduce the TUI hang on current main, then split any confirmed service-wrapper regression into either executable-wrapper documentation or explicit raw group-switch support.

## Reproduction Assessment

No high-confidence current-main reproduction is available. The report and comments provide plausible WSL2/macOS paths, but the review did not run a live affected terminal or inspect the reporter's exact service unit.

## Solution Assessment

Unclear: the current readiness and `sessions.list` changes are the right area, but they are not proven to solve the observed freezes. The service-wrapper portion should be handled after maintainers decide whether raw group-switch units are supported.
