## Summary

Keep open: the current main OOM concern is real and the PR moves in the right direction, but the streamed parser still accepts arbitrary provider output keys into memory, and the external Google provider path still needs live proof or a maintainer waiver.

## What This Changes

The PR streams Gemini batch embedding JSONL output from the download response, adds a 16 MiB per-record guard, and adds loopback HTTP regression tests.

## Best Possible Solution

Merge only after the streamed parser bounds accepted output records to the requested batch, keeps the per-record guard, and has live Gemini proof or an explicit maintainer waiver for that proof gap and cap tradeoff.

## Solution Assessment

No, not as submitted. Streaming JSONL is the right layer for a finite batch output file, but the implementation must reject or ignore records outside the current batch before storing them, and maintainers still need live proof or a waiver for the Google provider path.
