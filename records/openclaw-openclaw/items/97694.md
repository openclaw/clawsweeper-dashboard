## Summary

Keep open: the current Google embedding-batch path really does buffer successful output with `res.text()`, but this PR is not merge-ready because the streamed parser can still retain unlimited unknown output IDs, lacks live Gemini proof or a maintainer waiver, and adds a 16 MiB per-record fail-closed compatibility tradeoff.

## What This Changes

The branch replaces Gemini batch output whole-body parsing with streaming JSONL parsing and adds loopback tests for large output files and oversized JSONL record rejection.

## Best Possible Solution

Merge only after the parser rejects or ignores records outside the requested group with regression coverage, live Gemini proof or an explicit waiver exists, and maintainers accept or adjust the 16 MiB output-record cap.

## Solution Assessment

No, not as submitted. Streaming JSONL is the right layer, but the implementation should bound accepted records to the requested batch and still needs live provider proof or a maintainer waiver.
