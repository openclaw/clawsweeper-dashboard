## Summary

Keep this PR open: current main still buffers the successful Gemini batch output file with `res.text()`, and the latest head now addresses the earlier unknown-id memory-growth finding. The remaining blockers are maintainer acceptance of the new per-record fail-closed cap and live Gemini/OpenClaw proof or an explicit waiver.

## What This Changes

The PR replaces Gemini batch output whole-file parsing with streamed JSONL parsing, caps each output record at 16 MiB, ignores output records for unknown request ids, and adds loopback regression tests.

## Best Possible Solution

Land the streamed parser with the requested-id guard after maintainers accept or tune the 16 MiB per-record cap and the PR has live Gemini proof or an explicit waiver.

## Solution Assessment

Yes for the code shape: streaming JSONL at the output-file boundary is narrower than a whole-body text cap and now ignores unknown ids before storage. Merge still needs live provider proof or waiver plus maintainer acceptance of the per-record cap.
