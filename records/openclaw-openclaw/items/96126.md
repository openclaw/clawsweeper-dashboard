## Summary

Keep open: current main still suppresses auto fallback model-selection differences in the status paths, and the linked fix PR is open rather than merged.

## What This Changes

The issue asks status output to distinguish the configured base model from an active fallback model and show why the fallback was selected.

## Best Possible Solution

Land or replace the linked PR with a focused fix that labels auto fallbacks separately from user pins, shows configured versus active model state in both CLI status and `session_status`, and preserves runtime-only alias silence.

## Reproduction Assessment

Yes, by source inspection: an auto fallback session can have `modelOverrideSource: "auto"` with a configured/selected model difference, while current status code only emits mismatch reasons for user-pinned overrides.

## Solution Assessment

Yes at the issue level: the best fix is a narrow status rendering and reason-population change across both `openclaw status` and `session_status`, not a new config option or broader routing change.
