## What This Changes

The issue requests that OpenClaw status output distinguish the configured base model from the active fallback model and show why the fallback was selected.

## Best Possible Solution

Resolve this by landing or replacing the linked focused status PR with a fix that covers both `openclaw status` and `session_status` while preserving runtime-only null-reason behavior.
