## Summary

Keep open: current main still suppresses auto-fallback model selection explanations in both the status summary path and the session_status renderer, and the linked open PR is the current candidate fix rather than a reason to close this issue.

## What This Changes

The issue asks OpenClaw status surfaces to distinguish the configured base model from the active fallback model and expose the fallback reason.

## Best Possible Solution

Resolve this by reviewing and landing a focused status fix, or a replacement, that updates both openclaw status and session_status fallback model rendering with regression coverage and real behavior proof.

## Reproduction Assessment

Yes. By source inspection, a session row with modelOverrideSource: "auto", configuredModel != selectedModel, and null modelSelectionReason is kept silent by current status summary/rendering code, while session_status only adds configured-primary text for user-pinned selections.

## Solution Assessment

Yes at the issue level. The maintainable fix belongs in the status summary and status-message rendering paths so the existing configured/selected model state is surfaced without adding new config or policy surface.
