## Summary

Keep open: current main and the latest release still suppress the auto-fallback model-selection reason in status output, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks for status output to distinguish the configured base model from the active fallback model and explain why fallback routing was selected.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/96151 or a narrow replacement that updates both `openclaw status` and `session_status` to show configured primary, active fallback, and fallback reason.

## Reproduction Assessment

Yes. By source inspection, an auto fallback session can have a different configured and selected model while current main leaves `modelSelectionReason` null and the renderer suppresses that row.

## Solution Assessment

Yes at the issue level. The narrow maintainable fix is to populate explicit auto-fallback status reasons and render configured-versus-active model state consistently across CLI status and `session_status`.
