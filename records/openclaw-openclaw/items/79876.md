The issue requests new Google plugin `webSearch` config fields to send Gemini `generationConfig.thinkingConfig` and the `url_context` tool with web-search requests.

If maintainers accept the feature, keep it plugin-local in `extensions/google`, add model-aware web-search reasoning and URL-context settings with explicit defaults, update manifest schema/UI hints/docs/changelog, and cover cache-key/request-body behavior in focused tests.
