## Summary

Keep open: current main and v2026.5.12 still expose only `apiKey`, `baseUrl`, and `model` for Gemini web search and still send `google_search` without `url_context` or `generationConfig.thinkingConfig`; this is a new provider-specific config/default-policy feature rather than a broken existing contract.

## What This Changes

Expose optional Gemini web-search config for thinking/reasoning and URL context in the bundled Google plugin.

## Best Possible Solution

If maintainers accept the feature, implement it inside `extensions/google` with model-aware thinking settings, an explicit URL-context default policy, cache-key coverage, schema/UI hints, docs, tests, and a changelog entry.
