## Summary

Keep open: current main still lacks the requested Gemini web-search thinking and URL-context config, and the remaining work is a new Google-plugin capability with default-policy decisions rather than a broken existing contract.

## What This Changes

The issue asks OpenClaw to expose optional `plugins.entries.google.config.webSearch` settings for Gemini thinking controls and `url_context` in Gemini web-search requests.

## Best Possible Solution

If maintainers accept the feature, implement it inside `extensions/google` with model-aware thinking settings, an explicit URL-context default decision, schema/UI/docs/changelog updates, cache-key coverage, and focused web-search provider tests.
