## Summary

Keep open: current main does not expose the requested Gemini web-search `thinkingConfig` or `url_context` controls, but this is a new provider-specific config/default-policy capability rather than a broken existing contract.

## What This Changes

The issue requests optional Google plugin `webSearch` config fields to add Gemini thinking controls and URL context to Gemini web-search `generateContent` requests.

## Best Possible Solution

If maintainers accept the feature, implement it inside `extensions/google` with model-aware thinking controls, an explicit URL-context default, schema/UI/docs/changelog updates, and cache-key plus request-body tests.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a current contract failure. Source inspection confirms current main only sends Google Search grounding and the strict Google plugin schema lacks the requested keys.

## Solution Assessment

Unclear as proposed: plugin-local implementation is the right boundary, but the exact public config should be model-aware and maintainer-approved rather than adding a budget-only knob and enabling URL context by default.
