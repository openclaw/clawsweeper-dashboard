## What This Changes

Expose optional Gemini web search config for thinking and URL context so the Google plugin can send `generationConfig.thinkingConfig` and `{ url_context: {} }` in grounding requests.

## Best Possible Solution

Add opt-in Google plugin `webSearch` fields, wire them into the Gemini search request/cache/docs/tests, and preserve current defaults unless maintainers explicitly choose a default behavior change.

## Solution Assessment

Implement opt-in Gemini web search support for thinking and URL context in the bundled Google plugin. Keep the boundary inside the Google plugin and shared public plugin contracts; do not move provider-specific behavior into core. Add typed/config parsing for `plugins.entries.google.config.webSearch` fields such as a numeric thinking budget or level and a boolean URL-context toggle, update `extensions/google/openclaw.plugin.json` schema/ui hints, and preserve current default behavior unless maintainers explicitly choose a default-on `url_context`.
