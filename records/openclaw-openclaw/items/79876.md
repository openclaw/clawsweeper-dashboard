## Summary

Keep open: current main and the latest release still expose Gemini web search with only the existing Google Search grounding request and config keys, while the requested Gemini API capabilities are real but need a maintainer decision on provider-local config shape and defaults.

## What This Changes

Requests new Gemini web search plugin config for thinking budget and URL context so GenerateContent can send `thinkingConfig` and `url_context` alongside `google_search`.

## Best Possible Solution

Add this as a Google-plugin-owned web-search enhancement only after maintainers choose the config shape/defaults, reusing existing Google thinking semantics where possible and updating schema, metadata, docs, tests, and live proof together.

## Reproduction Assessment

Not applicable as a feature/config request rather than a broken existing contract. Source inspection confirms current main lacks the requested config keys and request fields.

## Solution Assessment

Unclear as proposed: exposing raw `thinkingBudget` and defaulting URL context on may not be the best product shape. A safer solution would align web-search thinking with existing Google thinking-level/adaptive semantics and make the URL Context default an explicit maintainer decision.
