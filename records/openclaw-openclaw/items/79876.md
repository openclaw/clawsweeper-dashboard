## Risks / Open Questions

- Defaulting `url_context` to enabled would change retrieval behavior, latency, token/cost profile, and privacy expectations, so maintainers should choose the default explicitly.
- A raw `thinkingBudget`-only config would not be model-aware for current Gemini models; Gemini 3 should likely use `thinkingLevel` handling instead.
- Any implementation needs cache-key coverage for the new request-shape knobs to avoid reusing cached results across different thinking or URL-context settings.
