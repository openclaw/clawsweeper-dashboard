## Summary

Keep open. Source inspection confirms the linked bug on current main, and the PR is aimed at the right plugin-local guard, but the diff should refresh live config before the new eligibility check and the supplied proof does not exercise the actual command path.

## What This Changes

This PR makes `/active-memory status` include active-memory `config.agents` eligibility, adds a regression test, and records a changelog fix for https://github.com/openclaw/openclaw/issues/78986.

## Best Possible Solution

Keep the fix plugin-owned: refresh live config at the start of session status handling, use the same agent eligibility gate as runtime recall, cover non-targeted and live-config agent-scope cases, then merge with real `/active-memory status` proof.

## Reproduction Assessment

Yes from source inspection: docs require agent targeting, runtime recall enforces `isEnabledForAgent`, and current main's status action only checks the session toggle. I did not run a live gateway command in this read-only review.

## Solution Assessment

No, not quite: adding agent eligibility is the right direction, but the status command should refresh live active-memory config before reading `config.agents` so it stays aligned with runtime recall after config changes.
