## Summary

Keep open: current main lacks page-level status filtering for memory-wiki compile/search/lint, and the request is a new plugin feature plus config/tool-schema decision rather than conservative cleanup. I found no duplicate or merged PR covering the same `pageStatus` request.

## What This Changes

Request optional pageStatus filters for `openclaw wiki compile`, `wiki search`/`wiki_search`, `wiki lint`, and compiled wiki cache metadata.

## Best Possible Solution

Add this, if accepted, inside the `memory-wiki` plugin by normalizing a page lifecycle field into `WikiPageSummary`, threading an optional filter through compile/query/lint/CLI/gateway/tool surfaces, and documenting backward-compatible defaults.

## Solution Assessment
