## Summary

Close as superseded by #1210: current main already has a documented in-memory recent-image pruning policy, while this PR's write-time empty-payload approach would remove replayable image context and its test does not exercise production wiring.

## What This Changes

The PR adds a tool-result persistence transform that replaces image block base64 data with empty strings and adds a colocated test file that reimplements the helper.

## Best Possible Solution

Keep #1210 as the canonical media-reference/hydration design; a future PR should preserve recent replayable image data or store durable media references instead of blanking all persisted image blocks.

## Reproduction Assessment

Yes. Persist a tool-result image through the PR transform, then rebuild replay context: `sanitizeContentBlocksImages` turns `data: ""` into an omitted-empty-image text block, unlike current main's recent-turn preservation path.

## Solution Assessment

No. Write-time emptying is narrower than the open media-reference design but regresses follow-up image replay; the safer direction is durable media references with hydration, or pruning only older replay context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve replayable image context:** `src/agents/session-tool-result-guard-wrapper.ts:37`
  - body: The transform writes every persisted tool-result image back with `data: ""`. On the next context rebuild, `sanitizeContentBlocksImages` turns that into an omitted-empty-image text block, so follow-up turns lose screenshots that current main keeps for three completed turns. Store or hydrate a media reference, or preserve recent image data instead of blanking all images at write time.
  - confidence: 0.88
- **[P3] Test the production persistence path:** `src/agents/session-tool-result-guard-wrapper.test.ts:12`
  - body: The new test file defines its own copy of `stripBase64ImagesFromToolResult`, so it can still pass if production `guardSessionManager` stops applying the transform. Drive `guardSessionManager` or import production code instead.
  - confidence: 0.9
