## Summary

This PR is still necessary: current main incorrectly treats a durable `suppressed` result as visible delivery, while the patch aligns commitment, dedupe, and heartbeat-event state with the existing durable-send contract. The focused regression, live AWS Crabbox report, clean merge state, and green checks support ordinary maintainer landing; no actionable defect was found.

Likely related people: @vincentkoc (high confidence) introduced the current commitments/heartbeat and durable-send implementation; @joshavant is also relevant as an active maintainer-team contributor with the live verification context.

## What This Changes

The PR keeps due commitments and heartbeat dedupe state retryable after suppressed durable sends, emits a skipped heartbeat event with the suppression reason, and adds a suppressed-then-visible retry regression test.

## Best Possible Solution

Land this bounded change so only a visibly sent durable outcome advances commitment, dedupe, and heartbeat-event delivery state, while intentional model dismissal remains unchanged.
