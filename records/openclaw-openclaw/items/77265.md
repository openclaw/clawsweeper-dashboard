## Summary

Keep open: current main still has a Gateway CLI path where `agent --deliver` can omit strict delivery intent, inherit main-session best-effort delivery, and return an ok agent result after outbound media delivery fails.

## What This Changes

The issue asks for `openclaw agent --deliver` to deliver final `mediaUrl`/`mediaUrls` payloads to Telegram or surface an explicit delivery error.

## Best Possible Solution

Make explicit CLI Gateway `agent --deliver` requests strict by default while keeping UI/chat main-session RPC best-effort defaults, then add mocked regression coverage for media delivery failure propagation.

## Reproduction Assessment

Yes, for the silent-success class: source inspection shows Gateway-backed main-session CLI `--deliver` without explicit `bestEffortDeliver` reaches outbound delivery with best-effort enabled, so a mocked media send failure would be swallowed. The reporter's exact live Telegram account path was not run.

## Solution Assessment

Yes, the narrow maintainable fix is to set or preserve strict delivery intent for explicit CLI Gateway `--deliver` requests, not to add a public config flag or change Telegram media sending without further proof.
