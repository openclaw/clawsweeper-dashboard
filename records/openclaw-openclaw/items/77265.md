## Summary

Keep open: current main still has a source-backed path where Gateway-backed CLI `agent --deliver` runs for main sessions are converted to best-effort delivery, so outbound media send failures can return an ok agent result with `mediaUrl`/`mediaUrls` instead of surfacing a delivery error.

## What This Changes

The issue asks `openclaw agent --deliver` to either deliver returned Telegram media payloads or fail explicitly when media delivery does not happen.

## Best Possible Solution

Make explicit CLI `agent --deliver` Gateway requests strict by default while preserving best-effort behavior for chat/UI main-session RPCs, then add regression coverage for surfaced media delivery failures.

## Reproduction Assessment

Yes, for the silent-success failure mode from source inspection: a Gateway-backed main-session `openclaw agent --deliver --json` request without `bestEffortDeliver=false` is converted to best-effort, and outbound media delivery errors are swallowed. I did not reproduce the reporter's live Telegram account path.

## Solution Assessment

Yes: the narrow maintainable fix is to make explicit CLI delivery fail closed by default while keeping Gateway best-effort defaults for UI/chat callers that need session-only fallback.

## Repair Work Prompt

Fix the issue at https://github.com/openclaw/openclaw/issues/77265. Current main lets explicit CLI `openclaw agent --deliver` Gateway requests reach main-session delivery with `bestEffortDeliver` undefined, after which `src/gateway/server-methods/agent.ts` defaults it to true and outbound delivery errors can be swallowed. Make explicit CLI `--deliver` requests strict by default, likely by sending `bestEffortDeliver: false` from the CLI Gateway path when `opts.deliver` is true and no internal override is present. Preserve best-effort behavior for chat/UI Gateway main-session callers and for callers that explicitly request `bestEffortDeliver: true`. Add regression coverage showing a CLI Gateway delivery failure is surfaced, plus server-method coverage that explicit false is still honored and explicit true still downgrades.
