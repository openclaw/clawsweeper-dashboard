## Label Justifications

- `P2`: This bounded security-model proposal has no current runtime effect, but its trust semantics require correction before future consumers rely on it.
- `merge-risk: 🚨 security-boundary`: The default policy permits privileged actions for an `agent` source without proving that external content did not influence the turn.

## Summary

This draft should remain open but is not merge-ready: its default policy treats every `agent`-labeled action as trusted even though current OpenClaw turns can be steered by external content, and the new model has no provenance mechanism that can establish otherwise. Security-owner design approval is needed before this becomes foundational policy.

## What This Changes

The draft PR adds an inert Security Matrix policy model, evaluator, unit tests, documentation, and Vitest fast-path registration.

## Best Possible Solution

Carry explicit, composable provenance from context assembly into tool execution, then derive audit output from the canonical tool-policy, sandbox, and approval outcomes under an owner-approved precedence model.

## Reproduction Assessment

Yes at the model level: evaluating an `agent` source with the `exec` capability returns `allow`, while current security policy says external senders can steer a tool-enabled agent within granted permissions. There is no live runtime reproduction because the evaluator is intentionally unwired.

## Solution Assessment

No; a caller-supplied single source and parallel enforcement vocabulary are not a safe foundation for mixed-context turns or the existing security-control owners.
