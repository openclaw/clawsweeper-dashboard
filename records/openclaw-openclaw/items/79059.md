## Summary

Keep open: current main appears to have fixed the original user-visible delivery drop through queue fallback, but it does not implement this PR's remaining callGateway fallthrough, and that behavior changes an existing focused expectation that currently avoids `callGateway` for active requester wake failures.

## What This Changes

The PR removes an active-requester early return in subagent announcement delivery so the existing requester-agent `callGateway` handoff can run, and updates the focused test expectation.

## Best Possible Solution

Maintainers should decide whether active requester wake failures should try requester-agent `callGateway` before queue fallback; if yes, land this narrow diff after CI, otherwise close it as superseded by the current queue-fallback behavior.

## Reproduction Assessment

Yes for source-level reproduction of the PR's remaining behavior: current main still returns before `callGateway` in the active requester wake-fail path, and the focused test asserts `callGateway` is not called. The original user-visible drop is separately mitigated on current main by queue fallback.

## Solution Assessment

Unclear as a maintainer policy choice: the patch is narrow and avoids raw sends, but it changes current-main behavior from queue fallback to requester-agent handoff before fallback for active requesters. The safer path is an explicit maintainer decision on that contract before merging or closing.
