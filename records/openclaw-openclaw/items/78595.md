## Summary

Keep open: this PR is not implemented on current main, has protected `security` and `maintainer` labels, lacks required real behavior proof, and the latest head still has source-backed migration/data-loss blockers.

## What This Changes

The PR refactors runtime/session/transcript/cron/plugin/device/pairing/backup/UI/protocol state from file-backed stores into global and per-agent SQLite databases with migration, docs, tests, and generated API updates.

## Best Possible Solution

Keep the database-first direction in maintainer/security review, make all legacy imports conflict-aware and non-destructive, preserve recovery sources on malformed input, then require real upgrade, backup, startup, and channel/plugin proof before merge.

## Reproduction Assessment

Yes for the current PR blockers by source inspection: delayed doctor imports can overwrite or delete newer live SQLite state on the latest head. I did not run a live migration because this review is read-only.

## Solution Assessment

No. The storage direction may be maintainable, but this patch is not the best merge candidate while legacy imports can roll back runtime/trust state and real upgraded-run proof is absent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Merge transcript imports instead of replacing live rows:** `src/commands/doctor/state-migrations.ts:304-310`
  - body: `importLegacyTranscriptFileToSqlite` routes through `replaceSqliteSessionTranscriptEvents`, which deletes all SQLite transcript rows for the session before inserting legacy JSONL events. A delayed doctor run after upgraded runtime activity can roll a session back to stale history and then remove the source file.
  - confidence: 0.9
- **[P1] Preserve newer device-auth state during legacy import:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: This importer writes the entire legacy `device-auth.json` snapshot into the live SQLite store and then removes the file. If tokens were added, rotated, or revoked after upgrade, running doctor later can restore stale tokens or drop newer ones.
  - confidence: 0.9
- **[P1] Reject malformed cron stores before saving empty jobs:** `src/commands/doctor/legacy/cron-store.ts:129-134`
  - body: Unexpected parsed cron payloads are coerced to an empty jobs list. Because the doctor path can save that payload and remove the source, malformed legacy input can erase current SQLite cron jobs instead of preserving the file for operator repair.
  - confidence: 0.86
- **[P1] Preserve Telegram binding semantics during import:** `extensions/telegram/src/state-migrations.ts:143-153`
  - body: The migration defaults missing or unrecognized `targetKind` values to `subagent` and rebuilds each binding with only a subset of fields. Current runtime loading treats non-`subagent` as ACP and preserves timeout metadata, so migration can change routing and expiry semantics before deleting the source file.
  - confidence: 0.86
