## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve transcript agent scope for custom paths:** `src/agents/transcript/session-manager.ts:112-119`
  - body: Path-based transcript callers that cannot map a SQLite scope fall back to the default agent, so custom or non-main transcript layouts can write history under the wrong agent partition.
  - confidence: 0.88
- **[P1] Run legacy cron imports when jobs need no rewrite:** `src/commands/doctor-cron.ts:264-266`
  - body: The doctor cron path can return before importing `jobs-state.json` and cron run logs when `jobs.json` is already normalized, leaving legacy runtime state stranded after upgrade.
  - confidence: 0.86
- **[P1] Keep transcript appends atomic across processes:** `src/config/sessions/transcript-append.ts:142-166`
  - body: The transcript append queue only serializes one process while idempotency and parent selection happen before the SQLite insert, so concurrent gateway/CLI/worker writers can duplicate messages or create conflicting parent chains.
  - confidence: 0.86
- **[P1] Honor byte and line caps for recent reads:** `src/gateway/session-utils.fs.ts:299-303`
  - body: Recent SQLite transcript reads load the scoped message history before applying only the message count limit, so callers that pass byte or line caps can still force full-history scans.
  - confidence: 0.85
- **[P1] Snapshot SQLite databases before archiving state:** `src/infra/backup-create.ts:372-390`
  - body: Backing up live SQLite paths directly can capture an inconsistent main database, WAL, and SHM set while writes are active; create and verify database snapshots before adding them to the archive.
  - confidence: 0.84
