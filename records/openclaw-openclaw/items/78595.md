## What This Changes

The PR moves OpenClaw runtime/session/transcript/auth/plugin/channel/scheduler/cache state from JSON/JSONL and sidecars into global and per-agent SQLite databases, with migrations, SDK/API/docs/UI/test updates, and a Node runtime requirement bump.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore Node 22 support unless the release bump is approved:** `package.json:1810`
  - body: This PR raises the package engine and macOS runtime gate to Node 24 even though current main supports Node >=22.16.0. That makes existing supported Node 22 installs fail before they can run the storage migration, so the runtime bump needs to be reverted or split into an explicit release decision.
  - confidence: 0.9
- **[P1] Skip legacy imports when live SQLite state exists:** `src/commands/doctor/legacy/exec-approvals.ts:44-50`
  - body: This importer writes the legacy exec-approvals snapshot into the live SQLite KV row and then deletes the source. A delayed `doctor --fix` after normal SQLite operation can roll back newer approval decisions and make the stale state durable.
  - confidence: 0.88
- **[P1] Merge transcript imports instead of replacing rows:** `src/commands/doctor/state-migrations.ts:304-309`
  - body: Each legacy transcript file is imported through the replacement helper, which deletes all existing rows for the session before inserting that file's events. Real installs can have multiple files for the same session, so only the last import survives and earlier or newer SQLite rows are lost.
  - confidence: 0.9
- **[P2] Keep migration sources when repaired imports drop rows:** `src/commands/doctor-session-transcripts.ts:233-238`
  - body: The repair/import path can persist only the selected event list and then removes the original file. If the repair selection or import later proves incomplete, operators cannot correct the legacy source and retry recovery.
  - confidence: 0.78
