## Summary

Keep open: this is an active, protected, broad SQLite storage refactor with `security` and `maintainer` labels, an unmergeable head, missing real behavior proof, and still-reproducible P1/P2 patch review blockers in PR-head source.

## What This Changes

The PR moves OpenClaw runtime/session/transcript/cron/pairing/device/plugin/VFS state from JSON, JSONL, and sidecars into SQLite-backed global and per-agent stores with docs, migration, SDK, and cleanup changes.

## Best Possible Solution

Land the SQLite runtime-state architecture only after maintainer/security review resolves agent partitioning, migration completeness, transactional transcript writes, bounded reads, SQLite-consistent backups, integrity controls, and live upgrade/runtime proof.

## Reproduction Assessment

Yes for PR review blockers: direct PR-head source inspection shows the mis-scoping, migration, transaction, bounded-read, and backup paths. This is patch review evidence rather than a current-main bug reproduction.

## Solution Assessment

No. SQLite-first runtime state may be a good architecture, but this patch is not yet the safest landing shape while it can mis-scope agent data, skip configured migration inputs, weaken bounded reads, and archive live WAL-mode databases directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve configured store agent scope:** `src/config/sessions/store-backend.sqlite.ts:43-44`
  - body: When `session.store` uses a custom non-canonical path, this fallback writes rows under the default agent, so non-main agent session metadata can be persisted and queried from the wrong SQLite partition.
  - confidence: 0.92
- **[P1] Preserve transcript scope for custom paths:** `src/agents/transcript/session-manager.ts:112-119`
  - body: `SessionManager.open(sessionFile)` callers do not always pass an agent id, and non-canonical transcript paths fall back to the default agent, mixing non-main transcript events into the wrong partition.
  - confidence: 0.9
- **[P1] Import configured session.store paths in doctor:** `src/infra/state-migrations.ts:763-767`
  - body: Doctor migration detection only checks the default per-agent `sessions.json` path, so installs using a configured `session.store` can be skipped before runtime JSON support is removed.
  - confidence: 0.9
- **[P1] Make transcript appends cross-process atomic:** `src/config/sessions/transcript-append.ts:142-166`
  - body: The append queue is process-local and idempotency/parent selection happen before the final DB insert, so gateway, CLI, or worker writers can append duplicate idempotent messages or conflicting parent chains.
  - confidence: 0.87
- **[P1] Honor recent-read byte and line caps:** `src/gateway/session-utils.fs.ts:299-303`
  - body: Recent reads now load the full SQLite transcript and only apply `maxMessages`, ignoring `maxBytes` and `maxLines` bounds that callers use to avoid full-history latency and memory spikes.
  - confidence: 0.87
- **[P1] Snapshot SQLite databases before backup:** `src/infra/backup-create.ts:372-390`
  - body: The backup archive includes the live state directory directly; in WAL mode this can capture an inconsistent mix of database, `-wal`, and `-shm` files while writes are active.
  - confidence: 0.84
- **[P2] Raise the shared SQLite busy timeout:** `src/state/openclaw-state-db.ts:177-179`
  - body: The shared state database uses a 5 second busy timeout even though this PR moves gateway, CLI, cron, transcript, pairing, and artifact writes into SQLite paths that can contend during normal operation.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff moves sensitive runtime state into SQLite while leaving concrete partitioning, integrity, and backup-consistency concerns.

Concerns:

- **[high] Prevent agent partition mixing:** `src/config/sessions/store-backend.sqlite.ts:43`
  - body: Custom store and transcript paths can fall back to the default agent partition, risking exposure or mutation of one agent's session/transcript state through another agent's SQLite scope.
  - confidence: 0.9
- **[high] Use SQLite-consistent backups:** `src/infra/backup-create.ts:372`
  - body: Archiving the live state directory directly can produce an inconsistent SQLite backup under WAL-mode writes, undermining recovery integrity for state that now includes pairing/device/runtime data.
  - confidence: 0.84
- **[medium] Enable or compensate for relational integrity:** `src/state/openclaw-state-db.ts:177`
  - body: The state database defines related agent/session/transcript/VFS/artifact tables but does not enable `PRAGMA foreign_keys` or define ownership relationships, leaving orphaned sensitive runtime rows possible after deletes or failed migrations.
  - confidence: 0.76
