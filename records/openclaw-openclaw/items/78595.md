## Summary

Keep open: this is a broad, active SQLite runtime-state refactor with protected `security` and `maintainer` labels, and sampled current-head migration paths still show stale-overwrite and source-deletion risks.

## What This Changes

This branch moves runtime, session, transcript, auth, plugin, cache, and scheduler state from JSON/JSONL sidecars into global and per-agent SQLite databases with migration, docs, SDK, UI, and test updates.

## Best Possible Solution

Keep the database-first architecture direction, but make legacy imports idempotent or recency-safe, preserve unreadable sources for retry, land the correctness follow-up lane, and require populated-upgrade proof plus maintainer/security review before merge.

## Reproduction Assessment

Yes, by source inspection. Run the upgraded runtime so SQLite has newer device/auth or preference rows, leave the old JSON file present, then run `openclaw doctor --fix`; the sampled importers overwrite the SQLite row and remove the source.

## Solution Assessment

No, not as-is. SQLite is a maintainable target, but the migration boundary needs skip, merge, recency, and retry-preservation guards before this is the safest implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip stale device-auth imports when SQLite has state:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: This migration writes the legacy token snapshot into the same SQLite row that runtime token rotation and revocation mutate, then removes the source file. A delayed doctor run can restore stale or revoked device tokens permanently.
  - confidence: 0.9
- **[P2] Preserve unreadable TTS prefs for retry:** `src/commands/doctor/legacy/tts-prefs.ts:35-42`
  - body: For any read or parse error other than ENOENT, this path writes an empty prefs snapshot and deletes the legacy file. That can wipe current SQLite prefs and remove the only recoverable source instead of letting the operator fix and rerun migration.
  - confidence: 0.86
- **[P2] Guard Voice Wake routing imports against stale rollback:** `src/commands/doctor/legacy/voicewake-routing.ts:40-42`
  - body: The importer replaces the SQLite routing config from the legacy file and deletes the source. Since normal runtime updates use the same SQLite key, running doctor after route changes can roll wake routing back to stale settings.
  - confidence: 0.84
