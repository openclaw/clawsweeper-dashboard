## Summary

Keep open. This PR is an active, broad SQLite runtime-state refactor with protected `security` and `maintainer` labels; source inspection of current head still shows security-sensitive legacy imports that can overwrite newer SQLite state, and the current-head real behavior proof gate is failing.

## What This Changes

This PR moves runtime, session, auth, plugin, cron, cache, transcript, and artifact state from JSON/JSONL file stores into global and per-agent SQLite databases with migration, docs, SDK, UI, and test updates.

## Best Possible Solution

Keep the database-first direction, but make legacy imports idempotent or recency-safe, fold or land the correctness lane, and require current-head populated-upgrade proof plus maintainer/security review before merge.

## Reproduction Assessment

Yes. By source inspection, run the upgraded runtime long enough to write newer SQLite rows, leave legacy files present, then run `openclaw doctor --fix`; the sampled importers overwrite the SQLite rows and remove the legacy sources.

## Solution Assessment

No, not as-is. The SQLite storage boundary is a maintainable direction, but the migration layer needs skip, merge, or recency guards before this is the safest implementation.
