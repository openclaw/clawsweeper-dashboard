## Summary

Keep open: this is an open, unmerged XL storage refactor with protected security/maintainer labels; current main still uses file-backed runtime/session state, and PR head still has blocking migration/security defects plus no real behavior proof.

## What This Changes

This PR migrates runtime, session, plugin, channel, scheduler, and artifact state from JSON/JSONL sidecars into shared and per-agent SQLite databases, with docs, tests, protocol, UI, SDK, and migration updates.

## Best Possible Solution

Continue the database-first direction only after legacy imports become conflict-aware and retry-safe, invalid inputs preserve recovery sources, real upgrade/backup/channel proof is supplied, and maintainer/security-selected gates pass.

## Reproduction Assessment

Yes for the current review blockers by source inspection: several PR-head doctor/migration paths replace live SQLite rows with legacy snapshots and remove the legacy files. I did not run a live migration because this review was read-only.

## Solution Assessment

No. The SQLite architecture may be the right direction, but this patch is not the best merge candidate while delayed imports can roll back trust/runtime state and real upgraded-run proof is absent.
