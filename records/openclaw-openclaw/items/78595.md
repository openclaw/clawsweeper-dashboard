## Summary

Keep open: this is an active, protected, broad SQLite runtime-state refactor with security and maintainer labels, current-head migration/state correctness blockers, and insufficient current-head real behavior proof.

## What This Changes

The branch moves runtime state for sessions, transcripts, auth, plugin/channel state, caches, apps, migration, backup, docs, scripts, and tests from JSON/JSONL/file-backed stores into global and per-agent SQLite stores.

## Best Possible Solution

Keep the database-first direction under maintainer review, but require safe merge-or-skip legacy imports, normalized session row writes, current-head migration/runtime proof, and explicit decisions on which public storage seams land here versus follow-up PRs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the blocking findings: PR-head source inspection shows the stale import, empty-auth delete, raw-key patch, and transcript replacement paths; the broad refactor itself is not a single current-main bug reproduction.

## Solution Assessment

Is this the best way to solve the issue? Not yet. The SQLite direction may be correct, but the current branch is not merge-ready until migration safety, key normalization, security rollback hazards, and current-head proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Skip stale exec approvals before deleting legacy state:** `src/commands/doctor/legacy/exec-approvals.ts:34-35`
  - body: This import writes the legacy `exec-approvals.json` snapshot into the live SQLite approvals store and then removes the source file without checking whether SQLite already contains newer approval policy. A delayed doctor run can roll back allowlists/socket settings and make that stale policy durable.
  - confidence: 0.9
- **[P1] Reject unreadable auth state before deleting source:** `src/commands/doctor/legacy/auth-profile-state.ts:30-33`
  - body: `loadJsonFile(statePath)` can return `undefined` for malformed or unreadable legacy JSON; `coerceAuthProfileState` turns that into `{}`, and the save path deletes SQLite state before the source file is unlinked. This can silently wipe newer auth routing state and remove the recoverable legacy input.
  - confidence: 0.88
- **[P1] Normalize the key used for optimistic session patches:** `src/config/sessions/store.ts:116-120`
  - body: `patchSessionEntry` may read an existing row through the normalized-key fallback, but it then applies the compare-and-swap using the raw `options.sessionKey`. Non-canonical key inputs can conflict forever or create/update the wrong alias row instead of the canonical session row.
  - confidence: 0.86
- **[P1] Merge legacy transcript files instead of replacing rows:** `src/commands/doctor/state-migrations.ts:307-312`
  - body: Legacy import calls `replaceSqliteSessionTranscriptEvents`, whose implementation deletes all existing transcript rows for the session before inserting the current file. Real installs can have multiple legacy transcript files for the same session, so importing each file by replacement drops events imported from earlier files.
  - confidence: 0.87
