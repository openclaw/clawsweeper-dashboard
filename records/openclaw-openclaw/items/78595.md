## Summary

Keep open: this protected security/maintainer XL PR still has source-proven migration rollback and data-loss blockers, plus missing real upgraded-state proof before merge.

## What This Changes

The PR moves runtime, session, plugin, channel, scheduler, artifact, and migration state from JSON/JSONL sidecars into global and per-agent SQLite databases across CLI, SDK, docs, tests, macOS, and UI surfaces.

## Best Possible Solution

Continue the database-first direction only after legacy imports become conflict-aware and retry-safe, invalid inputs preserve recovery sources, real upgraded-run proof is supplied, and maintainer/security-selected gates pass.

## Reproduction Assessment

Yes for the current review blockers by source inspection: several doctor/migration paths on the latest PR head replace live SQLite rows with legacy snapshots and remove legacy files. I did not run a live migration because this review was read-only.

## Solution Assessment

No. The SQLite architecture may be the right direction, but this patch is not the best merge candidate while delayed imports can roll back trust/runtime state and real upgraded-run proof is absent.
