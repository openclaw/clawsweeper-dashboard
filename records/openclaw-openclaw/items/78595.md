## Summary

Keep open: protected `security` and `maintainer` labels apply, and latest PR head still has source-proven protocol and security migration blockers plus missing real behavior proof.

## What This Changes

The PR moves OpenClaw runtime state from file-backed stores into global and per-agent SQLite databases across sessions, transcripts, plugins, channels, schedulers, artifacts, SDK surfaces, docs, and UI protocol models.

## Best Possible Solution

Continue the SQLite refactor only after protocol compatibility is restored, legacy imports become non-destructive and recency-aware, real upgraded-run proof is supplied, and maintainer/security-selected gates pass.

## Reproduction Assessment

Yes, for the PR blockers by source inspection: latest head removes `configuredAgentsOnly` from the sessions schema while the Control UI still sends it, and delayed doctor imports can overwrite the same SQLite security state that runtime paths mutate. I did not run a live migration in this read-only review.

## Solution Assessment

No. SQLite may be the right architecture, but this patch is not a good merge candidate while it can reject normal sessions UI calls and roll security/runtime state backward during migration.
