## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip stale device-auth imports when SQLite has state:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: This migration writes the legacy device-auth snapshot into the same SQLite row that runtime token rotation and revocation mutate, then removes the source file. A delayed doctor run can restore stale or revoked device tokens permanently.
  - confidence: 0.9
- **[P2] Preserve unreadable TTS prefs for retry:** `src/commands/doctor/legacy/tts-prefs.ts:35-42`
  - body: For read or parse failures other than missing-file cases, the importer writes an empty prefs snapshot and removes the legacy file. That can wipe current SQLite prefs and delete the only recoverable source instead of letting the operator fix and rerun migration.
  - confidence: 0.86
- **[P2] Guard Voice Wake routing imports against stale rollback:** `src/commands/doctor/legacy/voicewake-routing.ts:40-42`
  - body: The importer replaces the SQLite routing config from the legacy file and deletes the source. Since normal runtime updates use the same SQLite key, running doctor after route changes can roll wake routing back to stale settings.
  - confidence: 0.84
