## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep macOS protocol version aligned with the server:** `apps/macos/Sources/OpenClawProtocol/GatewayModels.swift:5`
  - body: This generated macOS model advertises protocol 3, but the gateway server still requires protocol 4 and rejects handshakes whose advertised range excludes it. macOS clients built from this file will fail to connect to current gateways, so regenerate or restore the constant to 4.
  - confidence: 0.95
- **[P1] Re-allow configuredAgentsOnly in sessions.list params:** `src/gateway/protocol/schema/sessions.ts:64`
  - body: The Control UI still sends `configuredAgentsOnly` on every `sessions.list` request, but the schema no longer declares it while `additionalProperties` remains false. Normal session loads will be rejected unless the field is restored or all clients stop sending it.
  - confidence: 0.93
- **[P1] Preserve newer per-agent session rows during migration:** `src/infra/state-migrations.ts:2684-2687`
  - body: The global session migration always overwrites per-agent `session_entries` with `excluded` values and then drops the global table. If upgraded runtime traffic has already written newer per-agent rows before doctor/migrate runs, this permanently rolls session metadata backward; merge by recency or skip newer rows.
  - confidence: 0.91
- **[P1] Treat missing legacy pairing files as non-imports:** `src/infra/pairing-files.ts:96-107`
  - body: `readJsonIfExists` returns `null` for missing files, but these branches check `!== undefined`, so a partial legacy pairing state can overwrite the missing sibling SQLite key with `{}`. That can wipe current device/node pending or paired state during doctor import.
  - confidence: 0.9
- **[P1] Avoid replacing live device auth state from legacy files:** `src/infra/device-auth-store.ts:95-97`
  - body: `importLegacyDeviceAuthFileToSqlite` writes the entire legacy token store into SQLite and then removes the source file. If tokens were rotated or revoked in SQLite before doctor runs, this can permanently reintroduce stale credentials.
  - confidence: 0.89
- **[P1] Merge legacy pairing imports instead of replacing access state:** `src/pairing/pairing-store.ts:731-747`
  - body: The legacy channel pairing import replaces current SQLite requests and account allowlists from legacy JSON before deleting the files. A delayed doctor run can re-authorize revoked senders or remove newer approvals, so this needs merge or recency-aware behavior.
  - confidence: 0.88
- **[P1] Honor byte and line caps for recent transcript reads:** `src/gateway/session-transcript-readers.ts:283-287`
  - body: The recent transcript read path loads all SQLite transcript messages and slices only by `maxMessages`, ignoring the byte and line caps callers use to bound large transcript tails. Long sessions can now force full-history reads and memory spikes where a bounded tail was expected.
  - confidence: 0.88
- **[P2] Honor maxBytes in recent usage readers:** `src/gateway/session-transcript-readers.ts:732-753`
  - body: Both recent usage helpers discard `maxBytes` and load all usage events, so status, auto-reply, and token-estimation paths can parse an entire large session despite passing a byte cap. Apply bounded-tail semantics before extracting usage.
  - confidence: 0.86
