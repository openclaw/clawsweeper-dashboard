## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve Node 22 support or split the runtime bump:** `package.json:1836`
  - body: Current main still supports Node 22.16+, but this PR raises the package engine to Node 24 and removes the Node 22 compatibility lane inside a storage migration. That can block supported installs before users can run the migration unless maintainers explicitly approve and sequence the breaking runtime change.
  - confidence: 0.9
- **[P1] Merge legacy transcript imports instead of replacing rows:** `src/commands/doctor/state-migrations.ts:306-311`
  - body: `importLegacyTranscriptFileToSqlite` replaces all SQLite transcript events for the session on each source file. Real installs can have multiple legacy transcript/checkpoint files or newer SQLite rows for the same session, so a delayed doctor run can drop transcript history.
  - confidence: 0.92
- **[P1] Avoid clobbering live device-auth state during migration:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: This importer writes the legacy `device-auth.json` snapshot into active SQLite state and then removes the file without checking for newer runtime tokens. A delayed doctor run can restore revoked tokens or drop newly issued ones and make that rollback durable.
  - confidence: 0.9
- **[P1] Merge channel pairing imports with live state:** `src/commands/doctor/legacy/channel-pairing.ts:231-247`
  - body: The pairing importer replaces request and allowlist entries from legacy files before deleting those files. If SQLite already contains newer approvals or revocations, migration can re-add revoked senders or remove newly approved ones.
  - confidence: 0.86
- **[P1] Preserve existing plugin state on migration conflict:** `src/plugin-sdk/migration-runtime.ts:55-60`
  - body: The plugin migration helper resolves `plugin_state_entries` conflicts by overwriting the current row with the imported value. Because plugin state is live runtime state, delayed sidecar migration can replace newer plugin data with stale legacy data.
  - confidence: 0.85
