## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restore configuredAgentsOnly to sessions.list:** `src/gateway/protocol/schema/sessions.ts:49-64`
  - body: The Control UI still sends configuredAgentsOnly by default, but the PR-head params schema no longer declares it while rejecting additional properties. Normal session loads from the UI can fail validation instead of returning rows.
  - confidence: 0.94
- **[P1] Preserve live device tokens during import:** `src/infra/device-auth-store-legacy.ts:39-41`
  - body: This import writes the whole legacy device-auth.json payload over the live SQLite record and then removes the source file. A delayed doctor run after token rotation or revocation can permanently restore stale tokens.
  - confidence: 0.91
- **[P1] Merge pairing imports instead of replacing state:** `src/pairing/pairing-store.ts:732-747`
  - body: The migration replaces current SQLite pairing requests and account allowlists with legacy file contents before deleting those files. Delayed doctor runs can re-add revoked senders or drop newer approvals.
  - confidence: 0.89
- **[P2] Honor byte and line caps for recent transcript reads:** `src/gateway/session-transcript-readers.ts:283-287`
  - body: Recent transcript reads materialize all SQLite events and only apply maxMessages, ignoring maxBytes and maxLines. Large sessions can force full-history scans where callers requested a bounded tail.
  - confidence: 0.87
