Full review comments:

- **[P1] Merge legacy transcripts instead of replacing rows:** `src/commands/doctor/state-migrations.ts:304-309`
  - body: `importLegacyTranscriptFileToSqlite` imports legacy transcript files through a replace path that deletes existing rows for the session before inserting the current file. Real installs can have multiple legacy transcript files for the same session, so a later file can discard events imported from an earlier one.
  - confidence: 0.94
- **[P1] Preserve live device-auth tokens during import:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: The device-auth importer writes the legacy snapshot into the live SQLite token store and removes the source file. If the upgraded runtime already rotated or revoked tokens in SQLite, a later doctor run can restore stale credentials permanently.
  - confidence: 0.91
- **[P1] Merge channel pairing state instead of replacing it:** `src/commands/doctor/legacy/channel-pairing.ts:229-245`
  - body: The channel-pairing migration replaces current pairing requests and allowlists with legacy file contents before deleting the files. If SQLite already has newer approvals, revocations, or pending requests, a delayed doctor run can roll access control backward.
  - confidence: 0.89
- **[P1] Skip stale plugin-state upserts when rows already exist:** `src/plugin-sdk/migration-runtime.ts:55-60`
  - body: The shared plugin-state migration helper updates existing plugin-state rows on conflict, so delayed doctor runs can replace newer runtime plugin state with stale legacy snapshots. Several callers remove the source afterward, making the rollback durable.
  - confidence: 0.86
