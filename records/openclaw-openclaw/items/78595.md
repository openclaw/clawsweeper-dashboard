## Summary

Keep open: this is an active protected PR with `security` and `maintainer` labels, and current-head source inspection still shows security-sensitive migration paths that can overwrite newer SQLite state with stale legacy files.

## What This Changes

This PR moves runtime state from JSON/JSONL and sidecar stores into global and per-agent SQLite databases across sessions, transcripts, plugin/channel state, scheduler/task state, artifacts, SDK/docs/tests, and app/UI protocol surfaces.

## Best Possible Solution

Continue the database-first direction only after legacy imports are non-destructive and recency-aware, malformed inputs remain retryable, real upgraded-run proof is supplied, and maintainer/security-selected gates pass.

## Reproduction Assessment

Yes, for the current PR blockers by source inspection: delayed doctor imports can write stale legacy snapshots over live SQLite records and then delete the source. I did not run a live migration in this read-only review.

## Solution Assessment

No. SQLite may be the right architecture, but this patch is not the best merge candidate while migration conflicts can roll back trust, routing, plugin, and runtime state.
