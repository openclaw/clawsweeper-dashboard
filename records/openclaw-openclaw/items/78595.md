## Summary

Keep open: this is a protected `security`/`maintainer` XL storage refactor with a dirty merge state, missing real behavior proof, and source-proven migration paths that can overwrite newer SQLite runtime/security state with stale legacy files.

## What This Changes

The PR moves runtime, session, plugin, channel, scheduler, artifact, and migration state from JSON/JSONL sidecars into shared and per-agent SQLite databases across code, docs, tests, SDK, UI, and CLI surfaces.

## Best Possible Solution

Continue the database-first direction only after legacy imports become conflict-aware and retry-safe, invalid inputs preserve recovery sources, real upgraded-run proof is supplied, and maintainer/security-selected gates pass.

## Reproduction Assessment

Yes for the current review blockers by source inspection: several PR-head doctor/migration paths replace live SQLite rows with legacy snapshots and remove legacy files. I did not run a live migration because this review was read-only.

## Solution Assessment

No. The SQLite architecture may be the right direction, but this patch is not the best merge candidate while delayed imports can roll back trust/runtime state and real upgraded-run proof is absent.
