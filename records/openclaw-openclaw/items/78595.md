## What This Changes

The PR moves OpenClaw session/transcript and other runtime state into the shared SQLite state database, shifts legacy JSON/JSONL import to doctor, removes runtime JSON fallbacks, and updates docs/tests.

## Best Possible Solution

Fix the data-scope, append-locking, bounded-read, cleanup, and legacy-repair blockers with focused regression coverage, add redacted real doctor migration and SQLite runtime proof, then have session/runtime/security owners decide whether to land this branch or split it.

## Reproduction Assessment

## Solution Assessment

No. The SQLite and doctor-only migration direction may be maintainable, but this patch is not the best landing shape while it mis-scopes custom paths, drops cross-process append coordination, ignores recent-read bounds, leaks discarded snapshots, and lacks real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve configured store agent scope:** `src/config/sessions/store-backend.sqlite.ts:43`
  - body: When session.store uses a custom path that is not shaped like the canonical per-agent sessions.json path, this falls back to DEFAULT_AGENT_ID. After runtime JSON fallback is removed, non-main agent sessions can be persisted and queried under the main SQLite partition.
  - confidence: 0.92
- **[P1] Import configured session.store paths in doctor:** `src/infra/state-migrations.ts:767`
  - body: Doctor migration detection only checks the default per-agent sessions.json path. Installs that used a custom session.store path are skipped before runtime JSON support is removed, so their visible session history is not imported unless they recover it manually.
  - confidence: 0.9
- **[P1] Make transcript append decisions cross-process atomic:** `src/config/sessions/transcript-append.ts:142`
  - body: The queue is process-local, while idempotency and parent-link decisions are made before SQLite insert. A gateway and CLI or worker process can read the same stale event list and then both append, duplicating messages or splitting parent chains.
  - confidence: 0.88
- **[P1] Honor byte and line caps on recent reads:** `src/gateway/session-utils.fs.ts:287-290`
  - body: readSessionMessagesAsync loads the whole SQLite transcript before slicing maxMessages and ignores maxBytes/maxLines. Callers that pass byte caps to avoid full-history scans can now hit avoidable memory and latency spikes on long sessions.
  - confidence: 0.87
- **[P1] Preserve transcript agent scope for non-canonical paths:** `src/commands/doctor-session-transcripts.ts:84`
  - body: Doctor import maps any transcript outside the canonical agents/<id>/sessions shape to DEFAULT_AGENT_ID and then removes the source file. Custom non-main transcript locations can be migrated into the wrong agent partition and disappear from intended history.
  - confidence: 0.86
- **[P2] Delete discarded checkpoint snapshots from SQLite:** `src/gateway/session-compaction-checkpoints.ts:284-285`
  - body: captureCompactionCheckpointSnapshotAsync writes a full snapshot transcript into SQLite, but cleanupCompactionCheckpointSnapshot is a no-op. Failed or discarded compactions can leave unreferenced snapshot rows that accumulate over time.
  - confidence: 0.84
- **[P2] Repair migrated JSONL before deleting legacy sources:** `src/infra/state-migrations.ts:963`
  - body: The legacy state migration path imports JSONL events directly into SQLite and removes the source file, bypassing the doctor transcript repair helper. A broken prompt-rewrite branch can be preserved permanently once the JSONL is deleted.
  - confidence: 0.82
