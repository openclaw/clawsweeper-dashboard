## Summary

Keep open. Current-head source inspection still shows stale legacy import paths can overwrite SQLite state that runtime code now mutates directly, and the PR has protected `security` and `maintainer` labels plus a failing real-behavior-proof check.

## What This Changes

The branch refactors runtime/session/plugin/channel state from JSON, JSONL, and sidecar stores into global and per-agent SQLite databases with migrations, docs, apps, and tests.

## Best Possible Solution

Keep the database-first architecture direction, but harden all legacy import paths for idempotency/recency and require maintainer/security review plus exact-head populated-upgrade proof before merging.

## Reproduction Assessment

Yes by source inspection. The PR head still has legacy import paths that write stale snapshots into the same SQLite rows used by runtime mutation paths; I did not run a live migration in this read-only review.

## Solution Assessment

No for merging as-is. The SQLite storage direction is plausible, but the current patch is not the narrowest maintainable solution until stale-import rollback paths, security review, and current-head real upgrade proof are resolved.
