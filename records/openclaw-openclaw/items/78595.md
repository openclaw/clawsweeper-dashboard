## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Merge legacy imports with live state before deleting sources:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: `importLegacyDeviceAuthFileToSqlite` writes the legacy token snapshot into the active SQLite store and then removes the source file without checking for newer SQLite rows. A delayed `openclaw doctor --fix` after rotation or revocation can restore stale credentials and make that rollback durable.
  - confidence: 0.9
- **[P2] Restore the dreaming repair RPC:** `src/gateway/server-methods-list.ts:16`
  - body: The UI still exposes the Dreaming Artifacts repair action and calls `doctor.memory.repairDreamingArtifacts`, but this PR removes that gateway method from the advertised method list and handler. Triggering the action will now fail with an unsupported-method error instead of running the repair flow.
  - confidence: 0.87
- **[P2] Normalize patch keys before writing SQLite session rows:** `src/config/sessions/store.ts:107-108`
  - body: `patchSessionEntry` can read an existing normalized row through `getSessionEntry`, but it writes and conflict-checks the raw `options.sessionKey`. Mixed-case or whitespace variants can create alias rows or compare against the wrong key instead of updating the canonical session row.
  - confidence: 0.84
- **[P2] Get approval before dropping Node 22 support:** `package.json:1840`
  - body: The PR raises the package engine to Node 24 while current root policy still requires Node 22+ paths to keep working. That supported-runtime change should not land as an incidental part of the storage refactor without explicit maintainer approval.
  - confidence: 0.84
