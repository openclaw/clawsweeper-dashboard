## Summary

This PR should stay open. It is a very large protected-label SQLite runtime-state refactor, and the current branch still has security-sensitive migration blockers plus no after-fix real behavior proof from an upgraded runtime setup.

## What This Changes

The PR moves OpenClaw runtime/session/transcript/plugin/channel/scheduler/artifact state from scattered file-backed stores into shared and per-agent SQLite storage, with migrations, SDK/protocol/UI/docs/tests updates.

## Best Possible Solution

Keep the PR open for maintainer/security-led review after the branch preserves newer SQLite state during legacy imports, restores protocol compatibility, exposes needed transcript-reader seams, and provides real upgraded-run proof.

## Reproduction Assessment

Yes, for the PR blockers by source/discussion evidence: the migration paths write legacy snapshots into live SQLite-backed stores that runtime also mutates, so delayed doctor runs can roll state backward. A live upgraded-runtime repro was not established in this read-only review.

## Solution Assessment

No. SQLite may be the right architecture, but this branch is not merge-ready while legacy import paths can overwrite live security/runtime state and the PR lacks real after-fix upgrade proof.
