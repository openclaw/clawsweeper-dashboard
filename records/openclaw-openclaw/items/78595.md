## Summary

Keep open. This protected security/maintainer-labeled PR is not implemented on current main and still has source-backed migration/security blockers plus missing real behavior proof.

## What This Changes

The PR moves runtime, session, transcript, cron, plugin, device, pairing, backup, UI, and protocol state from file-backed stores into global and per-agent SQLite stores with doctor migrations, docs, tests, and generated schema/protocol updates.

## Best Possible Solution

Keep the database-first direction in maintainer/security review, make legacy imports conflict-aware and non-destructive, preserve retryable recovery files, and require real upgrade, backup, startup, and channel/plugin proof before merge.

## Reproduction Assessment

Yes for the current PR blockers by source inspection from the provided diff/comments: delayed doctor imports can replace newer live SQLite state or delete recovery files. I did not run a live migration because this review is read-only.

## Solution Assessment

No. The SQLite direction may be maintainable, but this patch is not the best merge candidate while legacy imports can roll back trust/runtime state and real upgraded-run proof is absent.
