## Summary

Keep open. This is an active, protected `security`/`maintainer` XL storage refactor, the PR body still says it is not merge-ready, and sampled current diff paths still show stale legacy imports overwriting live SQLite state.

## What This Changes

The PR moves runtime/session/auth/plugin/channel state from JSON/JSONL and sidecar files into global and per-agent SQLite stores, with doctor migrations, SDK/app/docs/test updates, and a Node 24 runtime bump.

## Best Possible Solution

Keep the database-first direction, but land it only after stale-safe migration semantics, protected security/runtime approval, and current-head real install/Testbox proof are in place.

## Reproduction Assessment

Yes, at source level. The sampled importers write legacy snapshots into live SQLite-backed auth/access/runtime stores and then remove the sources, so a delayed `doctor --fix` after runtime writes can roll newer state back.

## Solution Assessment

No, not as currently proposed for merge. SQLite as canonical runtime state is a maintainable direction, but the import boundary needs merge/skip/freshness behavior and the Node/runtime policy change needs explicit approval.
