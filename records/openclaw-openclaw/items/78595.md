## Summary

Keep open: the current PR head still has blocking session-store scoping, doctor migration, and transcript append atomicity regressions, and it lacks required real behavior proof for a broad protected migration.

## What This Changes

The PR moves session metadata and transcripts toward SQLite with doctor-only JSON/JSONL migration, removes runtime JSON fallbacks and cron session retention, adds shared state/VFS/tool-artifact scaffolding, and updates docs/tests.

## Best Possible Solution

Fix the custom-store scope, custom-store doctor import, and append atomicity issues, add focused regression coverage, then have session/runtime/security owners decide whether the broad refactor should land as one branch or split into smaller slices.

## Reproduction Assessment

Not applicable for the PR as a whole because this is a broad refactor, not a single bug report. The blocking findings are source-reproducible from custom `session.store` paths and cross-process transcript append decisions on the current PR head.

## Solution Assessment

No. The SQLite/doctor-only direction may be maintainable, but this patch is not the best landing shape while it mis-scopes custom stores, skips configured-store migration, races transcript append decisions, and lacks real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve configured store agent scope:** `src/config/sessions/store-backend.sqlite.ts:43`
  - body: When `session.store` points at a custom path not shaped like `.../agents/<id>/sessions/sessions.json`, this fallback stores that agent's SQLite rows under `main`. After runtime JSON stores are removed, non-main agent sessions can be mixed into the wrong partition and returned through the wrong agent scope.
  - confidence: 0.92
- **[P1] Import configured session.store paths in doctor:** `src/infra/state-migrations.ts:767`
  - body: Doctor only detects the default per-agent `sessions.json`, so users with a configured custom `session.store` are skipped by `openclaw doctor --fix`. Since this PR removes runtime JSON fallback, those sessions disappear from normal listings until manually imported.
  - confidence: 0.9
- **[P1] Make transcript append decisions cross-process atomic:** `src/config/sessions/transcript-append.ts:142`
  - body: `appendSessionTranscriptMessage` reads existing events and decides header, idempotency, and parent linkage under an in-process queue, then inserts through separate SQLite transactions. A gateway and CLI/worker process can both base their append on stale events, producing duplicate headers/messages or conflicting `parentId` chains.
  - confidence: 0.87
