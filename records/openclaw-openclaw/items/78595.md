## Summary

Keep open: this broad storage refactor has protected `security` and `maintainer` labels, lacks current-head real behavior proof, and current PR head still has recency-unsafe legacy imports that can overwrite newer SQLite runtime state.

## What This Changes

The PR moves runtime/session/auth/plugin state from JSON and JSONL file stores into global and per-agent SQLite databases with migration, docs, SDK, UI, and test updates.

## Best Possible Solution

Keep the database-first direction, but make legacy imports idempotent or recency-safe, preserve recoverable malformed sources for retry, and require protected maintainer/security review plus current-head populated-upgrade proof before merge.

## Reproduction Assessment

Yes, by source inspection: run the upgraded runtime long enough to write newer SQLite rows, leave legacy files present, then run `openclaw doctor --fix`; the sampled importers replace active SQLite rows from stale files and remove those sources.

## Solution Assessment

No, not as-is; the SQLite storage boundary is directionally maintainable, but delayed imports need merge, skip, or recency guards before this is the narrowest safe implementation.
