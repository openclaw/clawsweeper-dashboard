## Summary

Keep open: this protected storage/security refactor is not merge-ready because PR-head source still shows agent partitioning, migration, atomic-write, bounded-read, and live SQLite backup issues, and real behavior proof is missing.

## What This Changes

This PR moves OpenClaw runtime/session/transcript/cron/pairing/device/plugin/VFS state from JSON, JSONL, and sidecars into SQLite-backed stores with migrations, docs, SDK, and runtime updates.

## Best Possible Solution

Resolve the partitioning, migration, atomic-write, bounded-read, snapshot-backup, integrity, timeout, and shared-vs-per-agent design issues, then provide redacted live upgrade/runtime proof for maintainer and security review.

## Reproduction Assessment

Yes for PR review blockers: direct PR-head source inspection shows the affected paths and line-level regressions. I did not run a live current-main failure reproduction because this is patch review evidence rather than a reported runtime bug.

## Solution Assessment

No. SQLite-first runtime state may be a good direction, but this patch is not the safest landing shape while it can mis-scope agent data, skip legacy imports, weaken bounded reads, commit async/nested transactions unsafely, and archive live WAL databases directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve configured store agent scope:** `src/config/sessions/store-backend.sqlite.ts:43-44`
  - body: When a store path is not shaped like the canonical agent sessions directory, this fallback writes and reads the SQLite store under the default agent. Explicit/custom store paths can therefore mix non-main agent rows into the main partition.
  - confidence: 0.9
- **[P1] Preserve transcript scope for custom paths:** `src/agents/transcript/session-manager.ts:112-119`
  - body: Path-only transcript callers that do not already have a mapped SQLite scope fall back to the default agent for non-canonical paths. Custom/non-main transcript layouts can then read or write history under the wrong agent.
  - confidence: 0.9
- **[P1] Import configured session stores in doctor:** `src/infra/state-migrations.ts:763-767`
  - body: Doctor migration detection only looks at the legacy root and default target sessions.json path. A still-supported explicit/custom store path can be missed before runtime JSON fallback is removed, leaving history stranded after upgrade.
  - confidence: 0.86
- **[P1] Make transcript appends cross-process atomic:** `src/config/sessions/transcript-append.ts:142-166`
  - body: The in-memory queue only serializes one process, while idempotency and parent selection happen before the final SQLite insert. Concurrent gateway, CLI, or worker writers can duplicate idempotent messages or create conflicting parent chains.
  - confidence: 0.88
- **[P1] Honor recent-read byte and line caps:** `src/gateway/session-utils.fs.ts:299-303`
  - body: Recent reads now load all scoped SQLite messages and slice only by maxMessages. Callers that pass maxBytes or maxLines to bound memory and latency can still force full-history reads for large transcripts.
  - confidence: 0.87
- **[P1] Snapshot SQLite databases before backup:** `src/infra/backup-create.ts:372-390`
  - body: The archive is built from live source paths, so a WAL-mode SQLite state directory can be captured with an inconsistent main database, -wal, and -shm set while writes are active. Snapshot and verify databases before tarring them.
  - confidence: 0.85
- **[P1] Run legacy cron imports without job rewrites:** `src/commands/doctor-cron.ts:264-266`
  - body: When the cron jobs file needs no normalization and has no notify warnings, this early return skips the newly added sidecar/run-log imports. Doctor can prompt for repair but leave jobs-state.json and cron run logs unimported.
  - confidence: 0.86
- **[P2] Guard state transactions against async callbacks:** `src/state/openclaw-state-db.ts:192-193`
  - body: The generic transaction helper commits immediately after the callback returns, so a Promise-returning callback would commit before awaited work finishes. Make the helper sync-only with guardrails or provide an async-safe mutex/savepoint helper.
  - confidence: 0.8
- **[P2] Enable integrity for owned state rows:** `src/state/openclaw-state-db.ts:84-101`
  - body: The schema creates agent-owned tables without foreign keys and connection setup does not enable foreign_keys, so deletes or failed migrations can leave orphaned session, transcript, VFS, or artifact rows. Add real constraints or explicit integrity checks.
  - confidence: 0.78
- **[P2] Merge legacy cron state by recency:** `src/cron/store.ts:201-207`
  - body: Importing jobs-state.json replaces the SQLite cron state wholesale. If SQLite already has newer scheduler state and a stale sidecar remains, a later doctor run can roll pending runtime state backward.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff moves sensitive runtime state into SQLite while leaving concrete partitioning, integrity, transaction, and backup-consistency concerns.

Concerns:

- **[high] Prevent agent partition mixing:** `src/config/sessions/store-backend.sqlite.ts:43`
  - body: Custom store and transcript paths can fall back to the default agent partition, risking exposure or mutation of one agent's session/transcript state through another agent's SQLite scope.
  - confidence: 0.9
- **[high] Use SQLite-consistent backups:** `src/infra/backup-create.ts:390`
  - body: Archiving the live state directory directly can produce inconsistent SQLite backups under WAL-mode writes, undermining recovery integrity for state that now includes pairing, device, session, and runtime data.
  - confidence: 0.85
- **[medium] Make state transactions safe:** `src/state/openclaw-state-db.ts:192`
  - body: A generic transaction helper that can commit before an async callback completes risks partial or misordered writes in the new central state database.
  - confidence: 0.8
- **[medium] Enable or compensate for relational integrity:** `src/state/openclaw-state-db.ts:84`
  - body: Agent/session/transcript/VFS/artifact tables lack enforced ownership relationships, leaving orphaned sensitive rows possible after deletes, failed imports, or partial migrations.
  - confidence: 0.78
