## Summary

Keep open: this is a protected, unmergeable, broad storage-architecture PR, and the latest head still has source-backed P1 migration/data-loss blockers plus missing real behavior proof.

## What This Changes

The PR refactors runtime/session/transcript/cron/plugin/device/pairing/backup/UI/protocol state from file-backed stores into global and per-agent SQLite databases.

## Best Possible Solution

Keep the database-first direction in maintainer/security review, make legacy imports non-destructive and conflict-aware, preserve recovery files on malformed input, and require real upgrade, backup, startup, and channel/plugin proof before merge.

## Reproduction Assessment

Yes for the current PR blockers by source inspection: delayed doctor imports can replace newer live SQLite state or delete recovery files. I did not run a live migration because this review is read-only.

## Solution Assessment

No. The storage direction may be maintainable, but this patch is not the best merge candidate while legacy imports can roll back runtime/trust state and real upgraded-run proof is absent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Merge transcript imports instead of replacing live rows:** `src/commands/doctor/state-migrations.ts:304-310`
  - body: `importLegacyTranscriptFileToSqlite` still routes legacy JSONL through `replaceSqliteSessionTranscriptEvents`, which deletes the SQLite transcript rows for the session before inserting stale file-era events. A delayed doctor run after upgraded runtime activity can roll a session back and then remove the legacy source file.
  - confidence: 0.91
- **[P1] Preserve newer device-auth state during legacy import:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: This importer writes the entire legacy `device-auth.json` snapshot into the live SQLite-backed store and then removes the file. If tokens were added, rotated, or revoked after upgrade, running doctor later can restore stale tokens or drop newer ones.
  - confidence: 0.9
- **[P1] Reject malformed cron stores before saving empty jobs:** `src/commands/doctor/legacy/cron-store.ts:129-134`
  - body: Unexpected parsed cron payloads are coerced to an empty jobs list. Because the doctor path can save that payload and remove the source, malformed legacy input can erase current SQLite cron jobs instead of preserving the file for operator repair.
  - confidence: 0.86
- **[P1] Default missing Telegram target kind to ACP:** `extensions/telegram/src/state-migrations.ts:143-147`
  - body: The migration maps every missing or unrecognized `targetKind` to `subagent`, while the existing runtime sanitizer defaults missing target kind to ACP. Older binding files can therefore be reclassified during import and lose their original routing semantics before the source file is deleted.
  - confidence: 0.84
