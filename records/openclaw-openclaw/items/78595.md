## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve configured store agent scope:** `src/config/sessions/store-backend.sqlite.ts:43`
  - body: When session.store uses a custom path not shaped like the canonical per-agent sessions.json location, this falls back to main. After runtime JSON fallback is removed, non-main sessions can be persisted and queried under the wrong agent scope.
  - confidence: 0.92
- **[P1] Import configured session.store paths in doctor:** `src/infra/state-migrations.ts:767`
  - body: Doctor only checks the default per-agent sessions.json path, so configured custom session.store JSON files are not imported before runtime JSON support is removed. Those users lose visible session history unless they manually import it.
  - confidence: 0.9
- **[P1] Make transcript append decisions cross-process atomic:** `src/config/sessions/transcript-append.ts:142`
  - body: The queue is process-local, and the existing-event read plus idempotency and parent-link decisions happen before the SQLite insert. A gateway and CLI or worker process can both append from stale state, duplicating messages or splitting parent chains.
  - confidence: 0.88
- **[P1] Honor byte and line caps on recent reads:** `src/gateway/session-utils.fs.ts:287-289`
  - body: This loads every SQLite transcript event before slicing by maxMessages, ignoring maxBytes and maxLines that callers use to cap scans. Long sessions can again force full-history reads and memory or latency spikes.
  - confidence: 0.87
