The branch moves OpenClaw runtime/session/transcript/auth/plugin/cache state from JSON, JSONL, and sidecar files into global and per-agent SQLite stores with generated Kysely schemas, migrations, docs, UI/API renames, and broad tests.

## Best Possible Solution

Yes, by source inspection: run the upgraded runtime so SQLite has newer rows, leave the old JSON/JSONL files in place, then run `openclaw doctor --fix`; the sampled import paths overwrite live SQLite state from stale files and remove those source files.

## Solution Assessment
