## Summary

Keep open. Current `main` still has the Codex app-server source-reply mismatch, and #76663 is already an open unmerged fix PR that explicitly closes this issue.

## What This Changes

The issue requests that Codex app-server source replies force the `message` dynamic tool when delivery is `message_tool_only`, even under restrictive tool profiles.

## Best Possible Solution

Land or replace #76663 after review so Codex app-server forwards `forceMessageTool` for message-tool-only source replies and keeps coding-profile regression coverage.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence reproduction path: Codex source delivery can resolve to `message_tool_only` while `tools.profile: "coding"` removes `message` from the Codex app-server dynamic tool catalog.

## Solution Assessment

Yes. The narrow maintainable fix is to pass `forceMessageTool: params.sourceReplyDeliveryMode === "message_tool_only"` into the Codex app-server tool build, matching the embedded and follow-up runner paths.
