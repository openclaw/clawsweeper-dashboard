## Summary

Keep open: the PR targets a real Control UI leak, but the current diff removes the shared global error display for every non-config page and would hide non-chat errors that still only flow through `state.lastError`. It also needs real browser/UI behavior proof before merge.

## What This Changes

The branch removes the shared Control UI header `state.lastError` pill and adds jsdom regression tests asserting chat errors stay routed to the Chat view.

## Best Possible Solution

Separate chat-scoped transient errors from global/page errors, or add page-local rendering for every non-chat `state.lastError` writer before deleting the shared header pill.

## Reproduction Assessment

Yes from source inspection: current main renders `.page-meta .pill.danger` whenever `state.lastError` is set and also passes that same value to Chat. I did not run a browser session in this read-only review.

## Solution Assessment

No as written: deleting the shared pill is too broad because non-chat paths still rely on `state.lastError` for visible failures. The safer fix is to scope chat transient errors separately or preserve a non-chat error surface.
