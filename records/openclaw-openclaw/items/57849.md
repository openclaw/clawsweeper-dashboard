## Summary

Keep this PR open: it remains the active implementation tracker for the provenance-loss bug, but the current head is still not merge-ready because it lacks real behavior proof, is conflicting, and expands `before_compaction` transcript payloads across a hook surface that is not covered by the existing conversation-access gate.

## What This Changes

The PR adds transcript-backed message loading for plugin hook events so `provenance` and other persisted extension fields can appear in `agent_end`, `before_compaction`, and `before_reset` payloads.

## Best Possible Solution

Land a rebased, scoped fix that preserves provenance for hook payloads through the correct active transcript source, keeps compaction payload policy/counts coherent, preserves current reset-hook fallback behavior, and includes real plugin-hook proof.

## Reproduction Assessment

Yes, by source inspection: `sessions_send` creates inter-session `inputProvenance`, the persistence transform writes it onto user messages, and current `agent_end` still sends the in-memory `messagesSnapshot` instead of reading persisted transcript fields. I did not run a live multi-agent plugin setup in this read-only review.

## Solution Assessment

No, not as-is. Transcript-backed hook payloads are the right direction, but this branch needs a rebase, active transcript handling, coherent compaction counts, and a maintainer decision on whether `before_compaction` may carry raw transcript messages without the conversation-access gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390-393`
  - body: This sends full transcript-derived `messages` through `before_compaction`, but that hook is not covered by the `allowConversationAccess` gate used for raw conversation hooks such as `agent_end`. Gate this payload or keep full transcript messages out of ungated compaction events so non-bundled plugins do not get raw conversation content unexpectedly.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757-1759`
  - body: Current main can rotate to a successor transcript before `agent_end` runs after automatic compaction. Reading `params.sessionFile` here can load the pre-rotation transcript and omit the active post-compaction conversation, so use the active/used session file or an enriched run snapshot instead.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: This combines `messageCount` from in-memory compaction metrics with `messages` read from the full transcript. When history limiting trims the compaction input, consumers can see `messageCount !== messages.length`; derive both fields from the same source or expose explicit original/current counts.
  - confidence: 0.8
