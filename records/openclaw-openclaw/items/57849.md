## Summary

Keep open: the provenance bug still appears source-reproducible on current main, but this branch is not ready because it lacks real behavior proof, is outdated/conflicting, and broadens raw `before_compaction` message exposure without the existing conversation-access gate.

## What This Changes

The PR adds a transcript JSONL message reader and wires transcript-derived messages into `agent_end`, `before_compaction`, and `before_reset` hook payloads, plus utility tests and a small media formatting change.

## Best Possible Solution

Land a rebased, scoped fix that reads provenance-preserving messages from the correct active transcript source for gated hook surfaces, preserves current `before_reset` archive fallback, and adds focused tests plus real hook proof.

## Reproduction Assessment

Yes, by source inspection: `sessions_send` creates `inputProvenance`, the persistence transform writes it into transcript messages, and current `agent_end` still passes the in-memory `messagesSnapshot`. I did not run a live multi-agent plugin setup in this read-only review.

## Solution Assessment

No, not as-is. Reading transcript-backed messages is the right direction, but the PR needs to use the active transcript source, keep compaction payload policy/counts coherent, and rebase over current reset-hook behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390-393`
  - body: This sends full transcript-derived `messages` through `before_compaction`, but that hook is not covered by the `allowConversationAccess` gate used for raw conversation hooks such as `agent_end`. Non-bundled plugins can receive raw conversation content here without opting into raw conversation access, so gate this payload or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757`
  - body: Current main can rotate to a successor transcript before `agent_end` runs after automatic compaction. Reading `params.sessionFile` here can load the pre-rotation transcript and omit the active post-compaction conversation, so use the active/used session file or an enriched run snapshot instead.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: This combines `messageCount` from the in-memory compaction metrics with `messages` read from the full transcript. When history limiting trims the compaction input, consumers can see `messageCount !== messages.length`; derive both fields from the same source or expose explicit original/current counts.
  - confidence: 0.8
