## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.85

Full review comments:

- **[P1] Gate compaction transcript messages:** `src/agents/pi-embedded-runner/run.ts:388-393`
  - body: This adds transcript-derived messages to before_compaction, but before_compaction is not in the conversation-hook set protected by allowConversationAccess. Either gate this hook before sending raw transcript content or avoid adding full transcript messages to ungated compaction events.
  - confidence: 0.82
- **[P2] Use a scoped active transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757-1759`
  - body: Reading params.sessionFile after the run is an unbounded, fire-and-forget transcript read. After compaction rotation it can read the old file, and if another turn appends before the async read finishes it can include messages outside the completed run; use the active run-end transcript boundary or enrich the captured run snapshot instead.
  - confidence: 0.78
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: This combines messageCount from the in-memory compaction metrics with messages read from the full transcript. When history limiting trims compaction input, plugins can receive a count that does not match the payload, so derive both from one source or expose separate original/current counts.
  - confidence: 0.8
