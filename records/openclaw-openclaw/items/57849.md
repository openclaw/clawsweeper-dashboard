## Summary

Keep this PR open: the underlying provenance-loss bug is still present on current main, but the branch is not merge-ready because it lacks real behavior proof and the diff still widens raw transcript delivery through ungated compaction hooks.

## What This Changes

The PR adds transcript-backed message loading for plugin hook events so persisted fields such as `provenance` are passed to `agent_end`, `before_compaction`, and `before_reset` hook payloads.

## Best Possible Solution

Land a refreshed, scoped provenance fix that reads from the correct active transcript source, preserves reset-hook archived transcript behavior, keeps compaction payload policy/counts coherent, and includes real plugin-hook proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessions_send` creates `inputProvenance`, persistence adds it to transcript user messages, and current `agent_end` still sends the in-memory snapshot that may not carry transcript-only extension fields.

## Solution Assessment

No, not as-is. Transcript-backed hook payloads are the right direction, but this branch needs a rebase, active transcript handling, coherent compaction payload policy, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate compaction transcript messages:** `src/agents/pi-embedded-runner/run.ts:388-393`
  - body: This sends transcript-derived `messages` through `before_compaction`, but that hook is not covered by the `allowConversationAccess` gate used for raw conversation hooks such as `agent_end`. Gate this payload or avoid full transcript messages in ungated compaction events so non-bundled plugins do not receive raw conversation content unexpectedly.
  - confidence: 0.78
- **[P2] Use the active transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757-1759`
  - body: Reading `params.sessionFile` here can load the original transcript after current main rotates to a successor transcript during automatic compaction. Use the active session file or an enriched run snapshot so `agent_end` does not omit the post-compaction conversation.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: This combines `messageCount` from in-memory compaction metrics with `messages` read from the full transcript. When history limiting trims the compaction input, consumers can see inconsistent counts and payloads; derive both fields from the same source or expose explicit original/current counts.
  - confidence: 0.8
