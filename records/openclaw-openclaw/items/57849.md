## Summary

Keep open: the provenance gap remains on main, but this branch is not merge-ready because it sends raw transcript messages through ungated compaction hooks and needs current-main compaction/reset adaptation.

## What This Changes

The PR adds a JSONL transcript-message reader and wires transcript-derived messages into `agent_end`, `before_compaction`, and `before_reset` hook payloads with focused utility tests.

## Best Possible Solution

Refresh this into a narrow hook-provenance fix that reads the transcript source of truth only on allowed raw-conversation surfaces, uses the active rotated transcript, keeps compaction counts coherent, and leaves `before_reset` aligned with its current transcript-backed implementation.

## Reproduction Assessment

Yes. Send an inter-session message with `sessions_send`, verify `message.provenance.kind = "inter_session"` in the JSONL transcript, then inspect current main hook payloads where `agent_end` uses `messagesSnapshot` and compaction paths omit or use in-memory messages.

## Solution Assessment

No, not as-is. Reading the transcript is the right source for preserved provenance, but the PR needs a conversation-access policy decision for compaction payloads and current-main transcript-rotation handling before it is the safest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390`
  - body: This adds raw transcript `messages` to `before_compaction`, but that hook is not protected by `allowConversationAccess`; non-bundled plugins can register it without the raw-conversation opt-in required for `agent_end` and similar hooks. Gate this payload the same way or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1760`
  - body: On current main, automatic compaction can rotate to a successor transcript before `agent_end` fires. Reading `params.sessionFile` after that can load the parent transcript instead of the active `sessionFileUsed`, so plugins may receive stale or incomplete post-compaction messages.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:208`
  - body: This can combine `messageCount` from the truncated in-memory compaction input with `messages` from the full transcript. When history limiting trimmed the compaction input, consumers see `messageCount !== messages.length`, which breaks budgeting and analytics assumptions.
  - confidence: 0.8
