## Summary

Keep this PR open: it targets a real provenance gap in plugin hook payloads, but the current branch is dirty against main, still lacks real behavior proof, and the diff needs fixes around active transcript selection, compaction payload consistency, and raw conversation access policy before it is mergeable.

## What This Changes

The branch adds a transcript-reading helper and uses it to send JSONL transcript messages, including fields such as `provenance`, to `agent_end`, `before_compaction`, and `before_reset` plugin hooks.

## Best Possible Solution

Land a refreshed provenance fix that reads from the correct active transcript or an enriched run snapshot, preserves current reset archived-transcript behavior, and either gates or avoids full before_compaction transcript payloads while keeping counts coherent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sessions_send creates inputProvenance, persistence adds it to transcript user messages, and current agent_end still emits the in-memory snapshot that may lack transcript-only extension fields.

## Solution Assessment

No, not as-is. Transcript-backed hook payloads are the right direction, but this branch needs active transcript handling, coherent compaction policy/counts, a clean rebase, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate compaction transcript messages:** `src/agents/pi-embedded-runner/run.ts:388-393`
  - body: This sends transcript-derived `messages` through `before_compaction`, but that hook is not covered by the `allowConversationAccess` gate used for raw conversation hooks such as `agent_end`. Gate this payload or avoid full transcript messages in ungated compaction events so non-bundled plugins do not receive raw conversation content unexpectedly.
  - confidence: 0.78
- **[P2] Use the active transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757-1759`
  - body: Reading `params.sessionFile` here can load the original transcript after current main rotates to a successor transcript during automatic compaction. Use the active session file or an enriched run snapshot so `agent_end` does not omit the post-compaction conversation.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: This combines `messageCount` from in-memory compaction metrics with `messages` read from the full transcript. When history limiting trims the compaction input, consumers can see inconsistent counts and payloads; derive both fields from the same source or expose explicit original/current counts.
  - confidence: 0.8
