## Summary

Keep this PR open: current main still has a real provenance gap for `agent_end` and some compaction hook paths, but this branch is not merge-ready because it broadens `before_compaction` raw transcript delivery without the conversation-access gate and predates current transcript-rotation behavior.

## What This Changes

The PR adds a JSONL transcript-message reader and wires transcript-derived messages into `agent_end`, `before_compaction`, and `before_reset` hook payloads so custom fields such as `provenance` reach plugins.

## Best Possible Solution

Refresh this into a narrow hook-provenance fix that uses the active transcript source of truth only on allowed hook surfaces and keeps `before_compaction` metrics and raw-message policy coherent.

## Reproduction Assessment

Yes. Source inspection shows provenance is applied before transcript persistence, while current main still sends `agent_end` from `messagesSnapshot` and leaves some compaction hook paths without transcript messages; this review did not run a live setup because the checkout is read-only.

## Solution Assessment

No, not as-is. Reading the transcript is the right source for preserved provenance, but the PR needs a conversation-access policy decision for compaction payloads, active transcript handling, and count/message coherence before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390`
  - body: This adds raw transcript `messages` to `before_compaction`, but that hook is not protected by `allowConversationAccess`; non-bundled plugins can register it without the opt-in required for raw conversation hooks such as `agent_end`. Gate this payload the same way or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757`
  - body: On current main, automatic compaction can rotate to a successor transcript before `agent_end` fires. Reading `params.sessionFile` after that can load the parent transcript instead of the active session file, so plugins may receive stale or incomplete post-compaction messages.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:209-210`
  - body: This can combine `messageCount` from the truncated in-memory compaction input with `messages` from the full transcript. When history limiting trimmed the compaction input, consumers see `messageCount !== messages.length`, which breaks budgeting and analytics assumptions.
  - confidence: 0.8
