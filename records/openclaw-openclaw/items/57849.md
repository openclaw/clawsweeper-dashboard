## Summary

## What This Changes

The branch adds a shared JSONL transcript reader and rewires `agent_end`, `before_compaction`, and `before_reset` hook payloads to prefer transcript messages so provenance and custom fields reach plugins.

## Best Possible Solution

Land a refreshed provenance fix that preserves transcript-only fields for explicitly gated hook payloads, keeps current reset-hook archived-transcript behavior, and makes any `before_compaction` transcript access a deliberate maintainer-approved policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: inter-session provenance is added before persistence, current main writes it to the transcript, and `agent_end` still passes the in-memory `messagesSnapshot` to plugins.

## Solution Assessment

No as-is. Reading transcript-backed hook messages is directionally sound for provenance, but the branch needs current-main rebase, scoped transcript selection, and a clear access policy for `before_compaction`.

- **[P1] Gate compaction transcript messages:** `src/agents/pi-embedded-runner/run.ts:390-394`
  - body: `before_compaction` is not part of `CONVERSATION_HOOK_NAMES`, so non-bundled plugins can register it without `allowConversationAccess`; this patch now reads the transcript and passes preserved custom/provenance fields there. Please either put this hook behind the existing conversation-access gate or keep transcript messages out of `before_compaction` until maintainers approve that data surface.
  - confidence: 0.86
- **[P2] Use the scoped transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1755-1757`
  - body: This reads `params.sessionFile` after the run, but current main can rotate the active transcript during compaction and returns the actual `sessionFileUsed`. Use the finished run's active transcript or an enriched bounded snapshot so `agent_end` cannot receive stale or wrongly scoped messages.
  - confidence: 0.8
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:210-211`
  - body: This event combines `messageCount` from the in-memory compaction window with `messages` from the full transcript. In trimmed-history flows, plugins can see `messages.length` much larger than `messageCount`, which makes budgeting and attribution consumers inconsistent.
  - confidence: 0.84
