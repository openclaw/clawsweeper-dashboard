## Summary

Keep open: current main still has the hook provenance gap and this PR is the canonical implementation tracker, but the proposed patch has blocking correctness/security issues and still lacks real behavior proof.

## What This Changes

The branch adds a JSONL transcript message reader and switches several plugin hook payloads to transcript-derived messages, with utility tests and one unrelated media formatting change.

## Best Possible Solution

Rework the PR so hook payloads use a consistent active transcript source, preserve provenance, obey the existing raw-conversation access policy, and include focused regression coverage plus redacted real plugin-hook proof.

## Reproduction Assessment

Yes, by source inspection: current main attaches provenance before persistence, while `agent_end` still uses the in-memory snapshot and some compaction paths omit transcript messages. I did not run a live multi-agent plugin setup because this review is read-only.

## Solution Assessment

No, not as-is. Reading transcript messages is the right direction, but the patch needs to gate raw compaction payloads, use the active rotated transcript for `agent_end`, and keep compaction counts coherent with the message payload.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390`
  - body: This sends full transcript-derived `messages` through `before_compaction`, but that hook is not covered by the `allowConversationAccess` gate that protects raw conversation hooks such as `agent_end`. Non-bundled plugins can receive raw conversation content here even when they would be blocked from the gated hooks, so either gate this payload or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757`
  - body: Current main can rotate to a successor transcript before `agent_end` runs after automatic compaction. Reading `params.sessionFile` here can load the pre-rotation transcript and omit the active post-compaction conversation; use the active/used session file or an enriched run snapshot instead.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:209-210`
  - body: This combines `messageCount` from the in-memory compaction metrics with `messages` read from the full transcript. When history limiting trims the compaction input, consumers can see `messageCount !== messages.length`; derive both fields from the same source or expose explicit original/current counts.
  - confidence: 0.8
