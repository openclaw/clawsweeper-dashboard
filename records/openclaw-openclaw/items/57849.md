## Summary

Keep open: the underlying provenance bug is valid and this PR is relevant, but the current diff still broadens ungated raw before_compaction payloads, reads the pre-rotation transcript for agent_end, mixes compaction count/message sources, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a shared JSONL transcript message reader and switches agent_end, before_compaction, and before_reset hook payloads to transcript-derived messages, plus utility tests and one unrelated media formatting change.

## Best Possible Solution

Revise the PR to preserve provenance from a current, policy-gated hook message source with coherent compaction metrics, adapt it to current reset/rotation code, add focused regression tests, and attach redacted real plugin-hook proof.

## Reproduction Assessment

Yes, by source inspection: current main attaches provenance before persistence, while agent_end still uses the in-memory snapshot and some before_compaction paths omit transcript-backed messages. I did not run a live multi-agent plugin setup because this review is read-only.

## Solution Assessment

No, not as-is. Reading transcript-backed messages is the right direction, but the patch needs to use the active transcript, obey raw conversation access policy, and keep compaction counts coherent with the payload.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390`
  - body: This adds transcript-derived messages to before_compaction, but non-bundled plugins can register that hook without allowConversationAccess because before_compaction is outside CONVERSATION_HOOK_NAMES. That broadens raw transcript exposure; gate this hook or keep full messages behind the existing raw-conversation policy.
  - confidence: 0.78
- **[P2] Read the active transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757`
  - body: The hook reads params.sessionFile after the turn, but current main can rotate to a successor transcript before agent_end runs after compaction. When that happens, this can load the pre-rotation file and miss the active post-compaction conversation/provenance; use the active used session file or an enriched run snapshot.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:209-210`
  - body: This combines messageCount from the current compaction window with messages read from the full transcript. When history limiting trims the compaction input, consumers can see messageCount differ from messages.length; derive both fields from one source or expose separate original/current counts.
  - confidence: 0.8
