## Summary

Keep open. Current main still does not provide transcript-preserved provenance to `agent_end` and several `before_compaction` hook paths, so this PR is not obsolete; the branch is not merge-ready because it broadens raw transcript payloads through an ungated compaction hook and needs adaptation for current compaction rotation semantics.

## What This Changes

The PR adds a shared JSONL transcript-message reader and wires transcript-derived messages into `agent_end`, `before_compaction`, and `before_reset` plugin hook events, with focused utility tests.

## Best Possible Solution

Refresh the PR into a narrow hook-provenance fix that reads the transcript source of truth where raw message payloads are allowed, respects the active rotated session file on current main, keeps count and message semantics coherent, preserves existing `before_reset` behavior, and adds targeted hook-payload tests plus any needed docs/changelog entry.

## Reproduction Assessment

Yes. The underlying bug has a high-confidence reproduction path: send an inter-session message with `sessions_send`, observe provenance in the JSONL transcript, and inspect current main hook payloads where `agent_end` uses `messagesSnapshot` and compaction paths omit or use in-memory messages.

## Solution Assessment

No, not as-is. Reading the transcript is the right source for preserved provenance, but the proposed patch needs a privacy gate decision, current-main transcript-rotation handling, and coherent compaction metrics before it is the safest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390`
  - body: This adds raw transcript `messages` to `before_compaction`, but that hook is not protected by `allowConversationAccess`; non-bundled plugins can register it without the raw-conversation opt-in required for `agent_end` and similar hooks. Gate this payload the same way or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1759-1760`
  - body: On current main, automatic compaction can rotate to a successor transcript before `agent_end` fires. Reading `params.sessionFile` after that can load the parent transcript instead of the active `sessionFileUsed`, so plugins may receive stale or incomplete post-compaction messages. Use the active session file with the existing in-memory fallback.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:208`
  - body: This can combine `messageCount` from the truncated in-memory compaction input with `messages` from the full transcript. When history limiting trimmed the compaction input, consumers see `messageCount !== messages.length`, which breaks budgeting and analytics assumptions. Derive both fields from the same snapshot or expose separate original/current counts.
  - confidence: 0.8

## Security Review
