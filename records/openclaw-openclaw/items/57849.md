## Summary

Keep open: the canonical issue is already routed to this PR and current main still has the provenance gap, but the patch cannot merge as-is because it lacks real behavior proof and introduces raw transcript exposure, active-transcript, and compaction count problems.

## What This Changes

The PR adds a shared JSONL transcript message reader, wires transcript-derived messages into `agent_end`, `before_compaction`, and `before_reset` hook payloads, adds utility tests, and includes an unrelated media formatting change.

## Best Possible Solution

Rework the PR to preserve provenance from the correct active transcript source, keep or add focused regression tests, and either gate raw `before_compaction` transcript payloads or avoid widening that hook surface.

## Reproduction Assessment

Yes, by source inspection: `sessions_send` supplies `inputProvenance`, persistence applies it to transcript messages, and current `agent_end` still sends the in-memory `messagesSnapshot`. I did not run a live multi-agent plugin setup in this read-only review.

## Solution Assessment

No, not as-is. Reading transcript-backed messages is the right direction, but the patch needs to use the active transcript source, preserve current reset-hook behavior, and resolve the raw `before_compaction` access policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390-393`
  - body: This sends full transcript-derived `messages` through `before_compaction`, but that hook is not covered by the `allowConversationAccess` gate used for raw conversation hooks such as `agent_end`. Non-bundled plugins can receive raw conversation content here without opting into raw conversation access, so gate this payload or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757`
  - body: Current main can rotate to a successor transcript before `agent_end` runs after automatic compaction. Reading `params.sessionFile` here can load the pre-rotation transcript and omit the active post-compaction conversation, so use the active/used session file or an enriched run snapshot instead.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: This combines `messageCount` from the in-memory compaction metrics with `messages` read from the full transcript. When history limiting trims the compaction input, consumers can see `messageCount !== messages.length`; derive both fields from the same source or expose explicit original/current counts.
  - confidence: 0.8
