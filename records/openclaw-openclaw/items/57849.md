## What This Changes

The branch adds a JSONL transcript message reader and uses transcript-derived messages for `agent_end`, `before_compaction`, and `before_reset` hook events, with focused utility tests and a small media formatting change.

## Best Possible Solution

Refresh the PR around current main so hook payloads use a consistent active transcript source, obey conversation-access policy, and include focused regression coverage plus real plugin-hook proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: provenance is applied during persistence, while current main still sends `agent_end` from `messagesSnapshot` and some compaction paths without transcript-derived messages. This review did not run a live multi-agent plugin setup because the checkout is read-only.

## Solution Assessment

No, not as-is. Reading transcript messages is the right source for preserved fields, but this branch must first resolve raw-message gating, active transcript selection after rotation, and count/message coherence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390`
  - body: This adds transcript-derived `messages` to `before_compaction`, but that hook is not in the conversation-access gate, so non-bundled plugins can receive raw transcript content without the opt-in required for `agent_end` and other raw conversation hooks. Gate this payload or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757`
  - body: Current main can rotate to a successor transcript before `agent_end` fires after automatic compaction. Reading `params.sessionFile` here can load the pre-rotation transcript and omit the active post-compaction conversation, so use the active/used session file or an enriched run snapshot.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:209-210`
  - body: This combines `messageCount` from the in-memory compaction metrics with `messages` from the full transcript. When history limiting trims the compaction input, consumers can see `messageCount !== messages.length`; derive both from the same source or expose explicit original/current counts.
  - confidence: 0.8
