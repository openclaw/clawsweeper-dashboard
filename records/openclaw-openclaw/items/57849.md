## Summary

Keep open: the provenance bug is still real on current main, but this PR is not ready to merge because it broadens raw transcript delivery through an ungated hook, reads a stale transcript path for `agent_end`, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds a JSONL transcript message reader and wires `agent_end`, `before_compaction`, and `before_reset` hook payloads to transcript-derived messages, plus utility tests and an unrelated media formatting change.

## Best Possible Solution

Rework the PR so hook payloads preserve provenance from the correct active transcript source, obey the raw-conversation access policy, keep compaction counts coherent, and include focused regression coverage plus redacted real plugin-hook proof.

## Reproduction Assessment

Yes, by source inspection: current main applies provenance before persistence, while `agent_end` still uses the in-memory snapshot and compaction paths either omit transcript-derived messages or deliver them through an ungated surface. I did not run a live multi-agent plugin setup in this read-only review.

## Solution Assessment

No, not as-is. Reading transcript messages is the right direction, but the patch needs to use the active transcript, gate or avoid raw compaction payloads, and keep compaction count fields consistent with the message source.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate transcript messages on compaction hooks:** `src/agents/pi-embedded-runner/run.ts:390-393`
  - body: This sends full transcript-derived `messages` through `before_compaction`, but that hook is not covered by the `allowConversationAccess` gate used for raw conversation hooks such as `agent_end`. Non-bundled plugins can receive raw conversation content here without opting into raw conversation access, so gate this payload or keep full transcript messages out of ungated compaction events.
  - confidence: 0.78
- **[P2] Read the active rotated transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757`
  - body: Current main can rotate to a successor transcript before `agent_end` runs after automatic compaction. Reading `params.sessionFile` here can load the pre-rotation transcript and omit the active post-compaction conversation, so use the active/used session file or an enriched run snapshot instead.
  - confidence: 0.73
- **[P2] Keep compaction counts aligned with messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: This combines `messageCount` from the in-memory compaction metrics with `messages` read from the full transcript. When history limiting trims the compaction input, consumers can see `messageCount !== messages.length`; derive both fields from the same source or expose explicit original/current counts.
  - confidence: 0.8
