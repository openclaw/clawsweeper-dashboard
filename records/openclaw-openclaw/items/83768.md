## Summary

Keep open: the native Mistral cache-key direction is supported, but this PR puts the opt-in on a shared compat patch that also feeds OpenRouter/custom Mistral-like routes, and it enables cached-token accounting while bundled Mistral cache-read pricing remains zero.

## What This Changes

The PR adds `supportsPromptCacheKey: true` to the Mistral transport compat patch and updates focused Mistral compat tests.

## Best Possible Solution

Scope `supportsPromptCacheKey` to verified native Mistral routes, update bundled Mistral `cacheRead` pricing to the upstream 10% input-rate contract with tests, then add redacted runtime proof.

## Reproduction Assessment

Yes, from source inspection: current main lacks `supportsPromptCacheKey` on the Mistral patch, and the OpenAI-compatible transport only sends `prompt_cache_key` when that compat flag is true. I did not run a live Mistral request in this read-only review.

## Solution Assessment

No, not yet. The native Mistral opt-in is the right direction, but putting it on the shared patch is broader than the proven API contract and the cached-token pricing metadata must be updated with it.

- **[P1] Scope the cache-key opt-in to verified Mistral routes:** `extensions/mistral/api.ts:22`
  - body: `MISTRAL_MODEL_TRANSPORT_PATCH` is also returned for OpenRouter Mistral-style refs and custom Mistral-like routes. Adding `supportsPromptCacheKey` here makes those routes send a Mistral-native top-level field whenever `cacheRetention` is enabled, while the linked issue and upstream Mistral docs only prove the field for `api.mistral.ai`; OpenRouter documents a different caching path. Please split native Mistral support from the broader request-shape patch, or add provider proof/contracts before proxy-like routes inherit it.
  - confidence: 0.86
- **[P2] Update Mistral cache-read pricing with the opt-in:** `extensions/mistral/api.ts:22`
  - body: Once this flag lets Mistral responses surface `usage.prompt_tokens_details.cached_tokens`, OpenClaw will multiply those cached tokens by the bundled model's `cacheRead` rate. The Mistral catalog still sets `cacheRead: 0`, but Mistral bills cached tokens at 10% of the standard input price, so the patch would under-report cost for successful cache hits.
  - confidence: 0.91
