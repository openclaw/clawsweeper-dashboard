## Summary

Keep open: current main still lacks a supported plugin-side Gateway caller, but this branch exposes raw Gateway RPC dispatch with unresolved method policy and caller-identity concerns, and it still lacks required after-fix real behavior proof.

## What This Changes

The PR branch exports the internal Gateway dispatcher and adds `api.runtime.gateway.dispatchMethod()` to plugin runtime options and types.

## Best Possible Solution

Define a narrow documented plugin Gateway-call or interop seam with explicit method policy, plugin caller attribution, SDK types/docs, and focused tests, then rework or supersede this PR.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms the current API gap, but this is a new plugin runtime capability rather than broken existing behavior.

## Solution Assessment

No. Exposing arbitrary `api.runtime.gateway.dispatchMethod()` is broader than the maintainable boundary; the safer solution is a policy-limited, identity-aware plugin Gateway-call API.
