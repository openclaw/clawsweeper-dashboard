## Summary

Keep open, but the current patch is not ready: current main still lacks a plugin-side Gateway caller, while this PR exposes arbitrary in-process Gateway RPC dispatch without a method policy or plugin caller attribution and has no after-fix real behavior proof. The related API-design request remains open at https://github.com/openclaw/openclaw/issues/39000.

## What This Changes

The PR exports the internal Gateway dispatcher and adds `runtime.gateway.dispatchMethod()` to the plugin runtime types and Gateway plugin load options.

## Best Possible Solution

Define a narrow documented plugin Gateway-call or interop seam with explicit method policy, plugin caller attribution, SDK types/docs, and focused tests, then update or supersede this PR.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms the API gap on current main, but this is a new plugin runtime capability rather than broken existing behavior.

## Solution Assessment

No. Exposing the raw dispatcher is broader than the maintainable boundary; the safer direction is an identity-aware, policy-limited Gateway-call API.
