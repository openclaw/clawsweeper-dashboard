## Summary

Keep open: this addresses a real plugin API gap, but the branch exposes the raw admin-capable Gateway dispatcher to every plugin and does not include after-fix real behavior proof.

## What This Changes

The PR adds a `runtime.gateway.dispatchMethod()` plugin runtime surface by exporting the internal Gateway dispatcher and passing it through plugin runtime options.

## Best Possible Solution

Define a narrow documented plugin-to-Gateway call contract for https://github.com/openclaw/openclaw/issues/39000 with explicit method policy, plugin caller attribution, tests, and SDK docs, then revise or supersede this PR.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an additive plugin SDK feature. Source inspection does confirm current main lacks a generic plugin-side Gateway caller while the PR adds one.

## Solution Assessment

No. A raw `dispatchMethod(method, params)` surface wired to an admin-capable synthetic client is broader than the maintainable contract; the safer path is a policy-gated API with plugin identity propagation.
