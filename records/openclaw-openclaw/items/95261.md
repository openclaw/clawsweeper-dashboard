## Summary

Keep open: the PR addresses a real unbounded Volcengine TTS success-body read, but the current patch uses a diagnostic prefix reader that can make over-limit Seed Speech responses look like successful partial audio.

## What This Changes

The PR changes `extensions/volcengine/tts.ts` so Seed Speech and legacy Volcengine TTS success response bodies are read through `readResponseTextLimited` with a 16 MiB cap.

## Best Possible Solution

Keep the response-size cap, but read success payloads with an overflow-signaling reader and add focused Volcengine TTS regressions plus redacted real-path proof.

## Reproduction Assessment

Yes, source-reproducible: current main has unbounded `response.text()` reads, and PR head can be checked with a Seed Speech-style response whose 16 MiB prefix ends on complete success frames while more data follows. I did not run a live reproduction in this read-only pass.

## Solution Assessment

No. Bounding the response is the right direction, but `readResponseTextLimited` is a diagnostic prefix reader; the best fix is an overflow-throwing success-payload read with Seed and legacy overflow tests.
