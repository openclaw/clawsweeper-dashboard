## Summary

Keep open. The Volcengine OOM class is real, but this patch uses a diagnostic truncating reader on a success payload, so an over-limit Seed Speech stream can be parsed as valid prefix audio and returned as success instead of failing explicitly.

## What This Changes

This PR caps both Volcengine Seed Speech and legacy TTS success response text reads at 16 MiB via `readResponseTextLimited`.

## Best Possible Solution

Keep the response-size cap, but use an overflow-signaling reader so over-limit success bodies fail explicitly, then add Volcengine TTS regressions and real local-server proof through `volcengineTTS`.

## Reproduction Assessment

Yes from source inspection: a local Seed Speech-style response with complete line-delimited frames beyond 16 MiB can be truncated by `readResponseTextLimited` and then parsed as prefix audio. I did not run a live reproduction because this review is read-only.

## Solution Assessment

No. Bounding the response is the right direction, but the current helper is for diagnostics and does not signal overflow; use an overflow-throwing success-payload reader instead.
