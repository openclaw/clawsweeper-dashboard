## Summary

Keep open: the OOM class is real and the cap direction is useful, but this patch uses a diagnostic text reader that returns a truncated prefix, so Seed Speech can parse partial frames as successful audio and the posted proof does not exercise the modified Volcengine path.

## What This Changes

The branch changes `extensions/volcengine/tts.ts` to read Seed Speech and legacy Volcengine TTS success response bodies with `readResponseTextLimited` under a 16 MiB cap.

## Best Possible Solution

Keep the response-size cap, but use an overflow-signaling success-payload reader and add focused Volcengine TTS regressions plus local-server proof through `volcengineTTS`.

## Reproduction Assessment

Yes from source inspection: current main has unbounded success-body reads, and the PR version can be checked with a local Seed Speech-style response containing complete line-delimited frames beyond the cap. I did not run a live reproduction in this read-only review.

## Solution Assessment

No. Bounding the response is the right direction, but `readResponseTextLimited` is a diagnostic prefix reader; the maintainable fix is an overflow-throwing success-payload read with tests for Seed and legacy over-limit bodies.
