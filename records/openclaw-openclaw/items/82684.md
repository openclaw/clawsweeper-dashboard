## Summary

Keep open: this is a useful feature PR, but it is not mergeable as-is because source inspection found a session-runtime state carry-over bug, the posted proof is only unit tests, and the broader restore/snapshot semantics overlap an open canonical design thread.

## What This Changes

The PR adds a built-in `/resume` chat command, bounded session-history persistence on rollover, transcript restoration from history/archive files, help/registry/docs wiring, and targeted tests.

## Best Possible Solution

Land a maintainer-approved restore/resume design that either stores and restores all session-owned runtime metadata or deliberately clears incompatible fields, adds regression coverage, includes redacted real chat/provider proof, and stays coordinated with https://github.com/openclaw/openclaw/issues/13700 and https://github.com/openclaw/openclaw/pull/51889.

## Reproduction Assessment

Yes for the review finding. Source inspection shows the PR-head `/resume` store update preserves existing runtime fields, and current main uses active-session CLI bindings on the next CLI turn.

## Solution Assessment

No. A first-class resume command may be the right feature, but this implementation needs a clearer session-state switch contract and real behavior proof before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Clear stale CLI bindings when switching sessions:** `src/auto-reply/reply/commands-session-resume.ts:245-247`
  - body: When `/resume` updates the store it spreads `existing` and only overwrites the session id/file. Existing entries can still contain `cliSessionIds`, `cliSessionBindings`, and `claudeCliSessionId`, and the next CLI turn reads those bindings from the active session, so restoring an older transcript can immediately continue the outgoing backend conversation instead of the restored one. Clear or intentionally restore session-owned runtime bindings as part of the switch.
  - confidence: 0.9
