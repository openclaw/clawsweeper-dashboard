Full review comments:

- **[P1] Clear CLI bindings when switching sessions:** `src/auto-reply/reply/commands-session-resume.ts:203`
  - body: `/resume` switches the stored `sessionId` and `sessionFile`, but because this assignment spreads the existing entry first it keeps `cliSessionBindings`, `cliSessionIds`, and `claudeCliSessionId` from the session the user is leaving. The next CLI-backed turn reads those fields from the active entry and passes the stale backend session id into the run, so `/resume #2` can report that an older transcript was restored while the follow-up prompt continues the newer backend conversation. Clear all CLI bindings on resume, or store and restore the matched history entry's binding, and cover that boundary in the resume tests.
