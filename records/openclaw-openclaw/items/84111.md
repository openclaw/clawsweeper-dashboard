## Best Possible Solution

Keep the reset path fail-closed unless a real interactive prompt is available, leave non-interactive and auto-repair paths warning-only, and require real doctor-run proof before merge.

## Reproduction Assessment

## Solution Assessment

No as written. The maintainable fix is to reuse the fail-closed interactive-confirmation contract or make the fallback return false when no real prompt is available, then prove interactive accept/decline and non-interactive warning-only behavior.
