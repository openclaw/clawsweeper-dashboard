## Summary

Keep open: current main still skips the configured-plugin install repair whenever the source config is invalid, while the missing official external plugin validation path still tells operators to run doctor. The reporter's follow-up also rules out the older-version pre-stage workaround for Brave/Discord, so this remains a narrow upgrade-blocking doctor/plugin migration bug.

## What This Changes

The issue asks OpenClaw to unblock 2026.4.29-to-2026.5.12 migrations where stock Brave, Slack, or Discord config references become invalid after those plugins are externalized.

## Best Possible Solution

Teach doctor to run configured missing-official-plugin repair for narrowly recognized invalid-config cases, then revalidate, without adding a broad invalid-config bypass.

## Reproduction Assessment

Yes, at source level: a missing official external plugin reference makes sourceConfigValid false, and current doctor returns before the repair helper can install the configured plugin. I did not run the full live package-upgrade scenario in this read-only review.

## Solution Assessment

Yes: the best fix is the narrow doctor gate relaxation for known missing official external plugin validation failures. A broad plugins install bypass or registry-wide compatibility search would be larger follow-up work.
