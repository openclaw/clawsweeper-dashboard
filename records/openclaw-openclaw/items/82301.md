## Summary

Keep open: current main still has the doctor early return that skips configured-plugin install repair when source config is invalid, and https://github.com/openclaw/openclaw/pull/82361 is an open fixing PR with closing syntax for this issue.

## What This Changes

The issue reports that upgrading from 2026.4.29 to 2026.5.12 can leave missing externalized Brave/Slack/Discord plugin references unrecoverable because doctor skips its configured-plugin install repair when source config validation fails.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82361 if maintainers accept the narrow gate, while treating https://github.com/openclaw/openclaw/pull/82420 as an optional immediate catalog workaround rather than the general fix.

## Reproduction Assessment

Yes, at source level: current main returns before the repair helper when `sourceConfigValid` is false, and current validation produces that invalid state for a missing installable `brave` web-search provider. I did not run the full package-upgrade scenario locally in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to let doctor run this repair only for known repairable missing-plugin validation issues, as in https://github.com/openclaw/openclaw/pull/82361, rather than adding a broad invalid-config bypass.
