## Summary

Keep open: current main still returns from the configured-plugin doctor repair when `sourceConfigValid` is false, which matches the reported missing externalized plugin deadlock.

## What This Changes

The issue reports that upgrading to 2026.5.12 can leave configured externalized plugins missing while both `plugins install` and `doctor --fix` are blocked by config validation.

## Best Possible Solution

Fix doctor to run configured-plugin install repair for narrowly identified missing official external plugin validation failures, then revalidate while keeping arbitrary invalid configs fail-closed.

## Reproduction Assessment

Yes, at source level: a missing configured official external plugin makes `sourceConfigValid` false, and the current doctor contribution returns before the repair helper can run. I did not execute the live package-upgrade scenario in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to specialize the doctor gate for known missing official external plugin references, not add a broad invalid-config bypass or a new CLI flag first.
