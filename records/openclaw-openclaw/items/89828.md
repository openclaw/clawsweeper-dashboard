This change strips every resolved target and deletes those tests, so plugin-owned canonical IDs that start with the selected channel prefix can be truncated and delivered to the wrong target.
  - confidence: 0.94
- **[P1] Keep cron preflight on the fallback candidate chain:** `src/cron/isolated-agent/run.ts:643-649`
  - body: Current cron preflight walks `resolveCronPreflightCandidates`, switches to the first available fallback provider/model, and passes the remaining fallback chain to the executor. This patch preflights only the initially selected provider/model, so a cron job with a configured fallback can be skipped instead of running on the fallback when the primary is unavailable.
  - confidence: 0.92
- **[P2] Keep CLI session clearing wired through run metadata:** `src/cron/isolated-agent/run.ts:904-909`
  - body: Current main honors `clearCliSessionBinding` by clearing the stored CLI session before writing a new session id. Dropping that branch leaves stale CLI session bindings in cron state when the runner explicitly asks to clear them, which can mis-associate later cron turns with an old CLI session.
  - confidence: 0.86
- **[P2] Retire the isolated cron MCP runtime during disposal:** `src/cron/isolated-agent/run.ts:1151-1153`
  - body: The disposal path currently retires the session MCP runtime before dropping in-memory references; this patch only clears the agent run context. That leaves per-session MCP runtime resources alive after isolated cron completion, regressing the cleanup contract documented immediately above this function.
  - confidence: 0.84
