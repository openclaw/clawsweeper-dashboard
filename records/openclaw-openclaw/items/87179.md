## What This Changes

The branch adds a shared announce-send helper and bypasses the A2A announce decider for same-session, same-channel `sessions_send` replies, with regression coverage for direct delivery, different-channel decider use, and `ANNOUNCE_SKIP` suppression.

## Best Possible Solution

Land the bounded same-session same-channel direct-delivery rule after maintainer acceptance and normal checks, while keeping cross-session and different-channel flows on the existing announce-decider path.

The PR body also includes live Discord after-fix proof, but I did not run a live reproduction in this read-only review.

## Solution Assessment

Yes, assuming maintainers confirm the same-session same-channel boundary. The patch is narrower than adding a config knob, reuses the existing gateway send semantics, and preserves the decider for cross-session and different-channel flows.
