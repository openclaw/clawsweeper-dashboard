## Summary

Keep this PR open for maintainer review. The patch targets the reported Discord channel message-loss path with focused regression coverage and real Gateway/Discord proof, and I did not find a blocking code defect; the remaining concern is the intentionally channel-generic direct-delivery policy.

## What This Changes

The branch extracts announce delivery into a helper and sends same-session, same-channel `sessions_send` replies directly while preserving the announce decider for cross-session and different-channel flows.

## Best Possible Solution

Merge the bounded direct-delivery path after maintainers accept same-session same-channel replies as source-channel delivery rather than A2A announcements, with normal checks green.

## Reproduction Assessment

Yes. Current main clearly routes same-session channel replies through the announce decider and suppresses `ANNOUNCE_SKIP`; the linked report and PR proof show that this can leave Discord without the generated reply.

## Solution Assessment

Yes, if maintainers accept the policy boundary. The helper keeps the existing send semantics in one place and the bypass is narrow, but the channel-generic behavior needs explicit owner acceptance before merge.
