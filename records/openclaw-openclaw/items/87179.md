## Summary

No blocking findings. Current main and v2026.5.22 still send same-session channel replies through the announce decider, while this focused PR adds a narrow same-session same-channel direct-delivery path with live Discord proof; keep it open for maintainer confirmation of that delivery boundary and normal checks.

## What This Changes

The PR changes sessions_send A2A delivery so same-session same-channel source replies bypass the announce decider and send the latest assistant reply directly back to the source channel, with regression coverage for the bypass and preserved decider paths.

## Best Possible Solution

Land this focused bypass after maintainer confirmation that same-session same-channel source replies should bypass the announce decider; otherwise narrow the guard before merge.

## Reproduction Assessment

Yes from source, though I did not live-reproduce the before state. Current main and v2026.5.22 route same-session channel replies through runAgentStep and suppress delivery when the announce reply is ANNOUNCE_SKIP, matching the linked Discord message-loss report.

## Solution Assessment

Yes, with a maintainer boundary decision. The patch is narrow and preserves cross-session plus same-session different-channel decider behavior, but maintainers should confirm that same-session same-channel replies are source-channel replies rather than A2A announcements.
