## What This Changes

## Best Possible Solution

Land the bounded direct-delivery path after maintainers explicitly approve same-session same-channel source replies as normal channel replies, while keeping cross-session and different-channel flows on the announce-decider path.

## Reproduction Assessment

Yes at source level: current main always runs the announce-decider step before channel delivery, and an ANNOUNCE_SKIP result prevents the send call. The linked issue supplies the live Discord symptom, while this review did not run a live current-main reproduction.

## Solution Assessment

Yes, if maintainers accept the boundary: the PR changes the implicated A2A delivery path and reuses the existing send helper instead of adding Discord-specific policy. The safer alternative, if cross-channel semantics are not approved, is to narrow the bypass to the reported Discord source-channel case or require more sibling-surface proof.
