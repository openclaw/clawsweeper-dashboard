## What This Changes

The PR adds optional `appendContext` and `prompt` fields to plugin prompt-build hook results, wires them through part of prompt assembly, and updates hook documentation.

## Best Possible Solution

Land a rebased patch that defines a deliberate prompt replacement contract, applies it consistently across embedded, CLI, and Codex prompt assembly, removes localized docs edits, and includes focused tests plus real plugin-run proof.

Not applicable as a bug reproduction; this is a Plugin SDK capability change. Source inspection gives high confidence that current main supports additive append context but not full user-prompt replacement.

No. The direction is plausible, but this branch is incomplete because replacement is not wired through all prompt-build consumers and the public API semantics still need maintainer review.
