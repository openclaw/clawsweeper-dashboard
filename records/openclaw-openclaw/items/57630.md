## Summary

Keep open: the PR targets a useful Plugin SDK capability, but current main already has the additive `appendContext` portion while the remaining user-prompt replacement API is incomplete across runner paths, security-sensitive, conflicting, and still missing real behavior proof.

## What This Changes

This PR adds optional `appendContext` and `prompt` fields to prompt-build hook results, wires them into hook merging and embedded prompt assembly, and updates hook documentation.

## Best Possible Solution

Land a rebased or replacement patch that keeps the shipped additive context behavior, defines a deliberate user-prompt replacement contract, and applies it consistently across embedded, CLI, and Codex prompt assembly with focused tests, docs, and real plugin-run proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability PR. Source inspection gives high confidence that current main supports additive prompt context but not user-prompt replacement.

## Solution Assessment

No. The direction is plausible, but this branch is not the best current form because it only wires replacement into embedded prompt assembly, leaves CLI and Codex harness paths inconsistent, drops bootstrap-warning context, edits localized docs, and lacks real behavior proof.
