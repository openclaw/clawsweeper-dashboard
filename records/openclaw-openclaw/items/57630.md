## Summary

Keep open. The PR targets a real remaining Plugin SDK capability gap, but it is not ready to merge because real behavior proof is missing and the proposed public `prompt` field is not applied consistently across embedded, CLI, and Codex harness prompt assembly.

## What This Changes

The PR adds optional `appendContext` and `prompt` hook result fields, merges them through prompt-build hooks, applies prompt replacement in embedded runs, and updates hook documentation.

## Best Possible Solution

Land a rebased implementation that defines the prompt replacement contract and applies it consistently across embedded, CLI, and Codex prompt assembly with focused regression coverage, English docs, and real plugin-run proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability change. Source inspection gives high confidence that current main has additive `appendContext` support but no user-prompt replacement contract or runner flow.

## Solution Assessment

No. The direction is plausible, but this branch is incomplete because the public `prompt` field is not wired through all prompt-build consumers, can discard existing prompt wrapping, and edits localized docs that this repo does not maintain.
