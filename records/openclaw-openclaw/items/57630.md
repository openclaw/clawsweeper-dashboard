## Summary

Keep open. The remaining prompt-replacement capability is not implemented on current main, and this PR is still incomplete because it exposes a destructive prompt rewrite field without wiring every prompt-build consumer or providing real plugin-run proof.

## What This Changes

The PR adds optional `appendContext` and `prompt` hook-result fields, merges them through prompt-build hook results, applies replacement in the embedded runner, and updates hook docs.

## Best Possible Solution

Land a rebased patch that defines a deliberate prompt replacement contract, applies it consistently across embedded, CLI, and Codex prompt assembly, removes localized docs edits, and includes focused tests plus real plugin-run proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability change. Source inspection gives high confidence that current main supports additive prompt context but not full user-prompt replacement.

## Solution Assessment

No. The direction is plausible, but this branch is not the best complete solution yet because replacement is only partially wired and the public API/security semantics still need maintainer review.
