## Summary

Keep open. The direction is useful, but current main already has the additive `appendContext` part while the remaining `prompt` replacement API is incomplete on this branch, security-sensitive, conflicting, and still missing real after-fix plugin-run proof.

## What This Changes

The PR adds optional `appendContext` and `prompt` hook result fields, merges them through prompt-build hooks, applies them in the embedded runner, and updates hook documentation.

## Best Possible Solution

Land a rebased or replacement implementation that defines the prompt replacement contract deliberately and applies it consistently across embedded, CLI, and Codex prompt assembly with focused tests, English docs, and real plugin-run proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability PR. Source inspection gives high confidence that current main supports additive prompt context but not full user-prompt replacement.

## Solution Assessment

No. The proposed API direction is plausible, but this branch is not yet the maintainable fix because replacement is not honored in all prompt assembly paths, bootstrap warning handling is unsafe, generated localized docs are edited, and real behavior proof is missing.

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply replacement prompts in CLI runs:** `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:77`
  - body: `resolvePromptBuildHookResult` is also used by CLI run preparation, but the CLI caller still starts from `params.prompt` and only applies prepend/append context. A plugin using `prompt` for redaction or truncation would still send the original prompt in CLI-backed runs.
  - confidence: 0.88
- **[P2] Apply replacement prompts in Codex harness runs:** `src/plugins/types.ts:1924`
  - body: Publishing `prompt` on the public hook result lets Codex harness hooks return it, but `resolveAgentHarnessBeforePromptBuildResult` still builds from prepend context plus `params.prompt`. Codex-backed runs would silently ignore replacement or redaction plugins.
  - confidence: 0.84
- **[P2] Preserve bootstrap warnings when replacing prompts:** `src/agents/pi-embedded-runner/run/attempt.ts:1413`
  - body: This assignment overwrites the already-built effective prompt when a hook returns `prompt`. In bootstrap truncation cases, the model loses the warning that workspace context is partial before the request is sent.
  - confidence: 0.86
- **[P2] Treat empty appended context as absent:** `src/plugins/hooks.ts:212`
  - body: `concatOptionalTextSegments` returns `right ?? left` when either side is falsy, so a later hook returning `appendContext: ""` erases previously accumulated appended context instead of no-oping. Normalize empty strings before merging appended context.
  - confidence: 0.8
- **[P3] Drop generated localized docs edits:** `docs/zh-CN/tools/plugin.md:1052-1053`
  - body: The docs guide says localized docs under `docs/<locale>/**` are generated outside this repo. Keep the English source docs change here and let the i18n pipeline update `docs/zh-CN/**`.
  - confidence: 0.9

## Security Review

Status: needs_attention
