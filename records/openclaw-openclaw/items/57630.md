## Summary

Keep open. Current main already has `appendContext`, but it still lacks the `prompt` replacement hook capability that this PR is trying to add for plugin use cases like GuardClaw/EdgeClaw; the branch is not merge-ready because it is stale against the current hook/docs layout and its prompt replacement path misses current CLI and bootstrap-warning behavior.

## What This Changes

This PR adds optional `appendContext` and `prompt` fields to prompt-build hook results, merges and applies them in agent prompt assembly, and updates hook documentation.

## Best Possible Solution

Land a rebased or replacement patch that adds only `prompt?: string` to the current prompt hook contract, merges it with higher-priority-wins semantics, applies it consistently before wrapping in embedded and CLI prompt assembly, and updates current docs, tests, and changelog.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: a plugin cannot type-return a `prompt` replacement through the current hook result contract, and both embedded and CLI prompt assembly ignore any such value.

## Solution Assessment

No. The feature direction is narrow and compatible, but this PR is not the best current implementation because current main already has `appendContext`, the branch targets old files, and the replacement path would drop bootstrap warnings and be ignored by the CLI path.
