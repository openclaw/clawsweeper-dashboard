## Summary

Keep open: current main still lacks user-prompt replacement in the plugin hook contract, so the PR is not obsolete, but the branch is incomplete because replacement is only partially wired and is missing real plugin-run proof.

## What This Changes

The PR adds optional `appendContext` and `prompt` fields to plugin prompt-build hook results, wires them into the hook merger/resolver/embedded runner, and updates hook documentation.

## Best Possible Solution

Land a rebased or replacement patch that keeps the shipped `appendContext` behavior, defines a deliberate `prompt` replacement contract, and applies it consistently across embedded, CLI, and Codex prompt assembly with focused tests, docs, and real plugin-run proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability PR. Source inspection gives high confidence that current main supports additive context but not full user-prompt replacement.

## Solution Assessment

No. The direction is plausible, but this branch is not complete because replacement is not honored in CLI or Codex harness paths, bootstrap warnings can be dropped, localized docs are edited against policy, and real behavior proof is missing.
