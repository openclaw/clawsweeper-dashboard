## Summary

Keep open: current main has additive `appendContext` support but still lacks user-prompt replacement, and this PR is a meaningful Plugin SDK API proposal that remains conflicting, incomplete across prompt assembly paths, and missing real behavior proof.

## What This Changes

The PR adds optional `appendContext` and `prompt` fields to plugin prompt-build hook results, wires them into the embedded runner, and updates hook documentation.

## Best Possible Solution

Land a rebased or replacement implementation that keeps the shipped additive context behavior, defines a deliberate user-prompt replacement contract, and applies it consistently across embedded, CLI, and Codex prompt assembly with tests, docs, and real plugin-run proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability PR. Current-main source inspection gives high confidence that additive context exists but user-prompt replacement is not implemented.

## Solution Assessment

No. The direction is plausible, but this branch is stale and incomplete because replacement is not honored in CLI or Codex harness paths, bootstrap warnings can be dropped, localized docs are edited against policy, and real behavior proof is missing.
