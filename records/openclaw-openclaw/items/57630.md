## Summary

Keep open for maintainer and contributor follow-up. Current main already has additive `appendContext`, but it still lacks user-prompt replacement; this PR is the active implementation direction for https://github.com/openclaw/openclaw/issues/8176, but the branch is incomplete, conflicted, and has only test-plan proof.

## What This Changes

The PR adds `appendContext` and `prompt` hook-result fields, merges them through prompt-build hooks, applies them in the embedded runner, and updates hook docs.

## Best Possible Solution

Land a rebased or replacement patch that defines a deliberate prompt replacement contract and applies it consistently across embedded, CLI, and Codex prompt assembly with focused tests, docs, and real plugin-run proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability change. Source inspection gives high confidence that current main supports additive prompt context but not full user-prompt replacement.

## Solution Assessment

No. The direction is plausible, but this branch is not the best complete solution yet because replacement is only partially wired, localized docs are edited against policy, and contributor real behavior proof is missing.
