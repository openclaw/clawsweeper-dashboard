## What This Changes

The PR adds optional appendContext and prompt fields to prompt-build hook results, merges them, applies them in embedded prompt assembly, and updates hook documentation.

## Best Possible Solution

Land a rebased patch that adds a deliberate prompt replacement contract to current hook types, applies it consistently across embedded, CLI, and Codex prompt assembly while preserving bootstrap/system wrapping, and keeps https://github.com/openclaw/openclaw/issues/8176 open until the implementation merges.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best current implementation because it misses CLI prompt replacement, would drop bootstrap warnings in embedded runs, and targets stale contract/docs files.
