## Summary

Keep open. Current main still creates scheduled, startup catch-up, and manual cron task rows without `progressSummary`, while `/tasks` and `session_status` already depend on that field for active task detail.

## What This Changes

The issue requests coarse `progressSummary` updates for managed cron task runs so `/tasks` and `session_status` show live cron activity before terminal completion.

## Best Possible Solution

Add cron-owned progress seeding or coarse lifecycle updates for scheduled, startup catch-up, and manual cron runs while leaving the shared task runtime and status consumers unchanged.

## Reproduction Assessment

Yes, source-level. Scheduled, startup catch-up, and manual cron paths create running task rows without `progressSummary`, while active status detail is displayed only from that field.

## Solution Assessment

Yes. The narrowest maintainable fix is in the cron producer lifecycle because the existing task runtime, storage, and status-formatting contracts already support active progress text.
