## Summary

Keep open. Current main still creates scheduled, startup catch-up, and manual cron task rows without `progressSummary`, while the shared task lifecycle and `/tasks`/`session_status` status formatting already support active progress detail; I found no merged or open canonical fix that owns this cron-specific producer gap.

## What This Changes

Managed cron runs should populate live `progressSummary` so `/tasks` and `session_status` show coarse progress during execution.

## Best Possible Solution

Seed or update cron-owned progress in the scheduled, startup catch-up, and manual cron producer paths, with focused regression coverage and unchanged silent notification behavior.

## Reproduction Assessment

Yes, source-level. Follow scheduled `onTimer`, startup catch-up, or manual `cron.run` through the cron task-row creators and the active row is created without `progressSummary`, while active status formatting displays only that field as task detail.

## Solution Assessment

Yes. The narrowest maintainable fix is in the cron producer lifecycle because the shared registry contract, storage, `/tasks`, and `session_status` already support active progress text.
