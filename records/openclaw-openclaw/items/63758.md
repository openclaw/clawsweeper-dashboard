## Summary

Keep open. Current main still matches the reported cron producer gap: scheduled, startup catch-up, and manual cron task rows are created without `progressSummary`, while the shared task and `session_status` surfaces already know how to display active progress when producers provide it.

## What This Changes

Managed cron task runs should populate live `progressSummary` text so `/tasks` and `session_status` are useful while cron work is still running.

## Best Possible Solution

Add cron-owned progress updates in the cron producer lifecycle, preserving the current silent notification default and leaving shared task/status rendering generic.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction follows any due, startup catch-up, or manual cron run through `tryCreateCronTaskRun` or `tryCreateManualTaskRun` and observes that no active `progressSummary` is written before terminal completion.

## Solution Assessment

Yes. The narrowest maintainable fix is producer-side in cron because the task lifecycle and status surfaces already support progress for active tasks.
