## Summary

Keep open. Current main still has the reported cron producer gap: scheduled, startup catch-up, and manual cron task rows are created without live progress text even though the task lifecycle and status surfaces already support `progressSummary`.

## What This Changes

Managed cron task runs should publish coarse live `progressSummary` updates so `/tasks` and `session_status` show useful status while cron work is active.

## Best Possible Solution

Add cron-owned progress updates at the producer lifecycle: seed useful active progress when scheduled, startup catch-up, and manual cron task rows are created; update through the existing detached-task progress API at meaningful phases; preserve cron's silent notification default; and cover the active status behavior with focused cron/status tests plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction follows due, startup catch-up, or manual cron execution through `tryCreateCronTaskRun`/`tryCreateManualTaskRun` and observes that active task rows never receive `progressSummary`, while active status rendering only displays that field for detail.

## Solution Assessment

Yes. The narrowest maintainable fix is in the cron producer lifecycle, not the shared registry or status renderers, because the generic task lifecycle already supports progress and other producers already use that seam.
