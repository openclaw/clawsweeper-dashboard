Keep open. Current main and latest v2026.6.6 only partially handle same-PID session-lock cleanup, and an open linked PR still claims this issue with closing syntax while remaining conflicting.

## What This Changes

Clear orphaned session `.jsonl.lock` files owned by the current gateway PID during startup or restart so later session writes do not time out indefinitely.

## Best Possible Solution

Repair or replace https://github.com/BingqingLyu/openclaw/pull/980 for the narrow startup self-lock cleanup, and keep runtime heartbeat or long-held lock policy in the broader session-lock reliability track.
