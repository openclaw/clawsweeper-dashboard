## Best Possible Solution

Keep the shipped shared session-lock inspection in place; any remaining `.deleted.*`, `.reset.*`, or `.bak.*` transcript sidecar accumulation should be tracked separately because it is a different cleanup path.

## Solution Assessment

Yes. The shipped fix is narrow and maintainable because it reuses session-aware lock inspection for both startup cleanup and acquisition while preserving active in-process locks.
