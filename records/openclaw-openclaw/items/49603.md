## What This Changes

The issue reports that gateway session `.jsonl.lock` files owned by the current process can survive state loss or restart recovery, causing later session writes to time out.

## Best Possible Solution

Keep this open until maintainers either confirm the startup self-lock cleanup is fully superseded by current main and move remaining work to https://github.com/openclaw/openclaw/issues/86538, or land a narrow owner-reviewed fix with regression coverage and real gateway proof.

## Reproduction Assessment

## Solution Assessment

Unclear as an automated fix. A boot/acquire orphan-self cleanup is already present, but the remaining live-PID lock failures need maintainer direction on whether to solve this through run-aware ownership, delivery serialization, lock timing, or the canonical delivery issue.
