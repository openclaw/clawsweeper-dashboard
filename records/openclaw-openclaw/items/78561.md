## What This Changes

The issue asks OpenClaw to prevent misleading Discord mid-turn fragments when partial/preview output is disabled and to classify any progress output consistently.

## Best Possible Solution

Add a focused Discord runtime regression that distinguishes preview streaming, block delivery, and tool-progress messages, then align code and docs around the chosen final-only policy for `streaming.mode=off`.

## Reproduction Assessment

No high-confidence live reproduction is available from the issue alone. Source inspection identifies a plausible path where Discord preview streaming is off but core block delivery can still emit mid-turn messages depending on block-streaming configuration.

## Solution Assessment

Unclear. A narrow code fix may be possible, but maintainers should first decide whether Discord `streaming.mode=off` means preview-only off or all mid-turn block/progress output off when block streaming is separately configured.
