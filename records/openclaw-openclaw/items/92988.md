## What This Changes

Adds a memory-core startup sweep that scans the memory store directory for aged `*.sqlite.tmp-*` reindex files and removes matching SQLite temp groups from `MemoryIndexManager` construction.

## Best Possible Solution

Keep the startup cleanup direction, but make deletion race-safe, use the actual subsystem logger contract, handle sidecar-only leftovers, and add focused tests plus redacted real startup-cleanup proof before merge.

## Reproduction Assessment

Yes. Current main creates the temp SQLite DB before reindex work and only cleans it through in-process success/error paths, so a hard kill can leave orphaned temp files; I did not run a destructive live SIGKILL repro in this read-only review.

## Solution Assessment

No. A memory-core startup sweep is the right owner boundary, but this implementation needs a live-reindex guard, a logger type fix, sidecar-only cleanup, and real behavior proof before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Guard cleanup against active reindexes:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:301-302`
  - body: `runSafeReindex()` creates the temp DB before the rebuild work starts, so a large in-flight rebuild can run longer than 30 seconds. A second manager startup against the same store would classify that live temp DB as stale and remove it, making the active reindex fail; add a real ownership or live-file guard before deletion.
  - confidence: 0.9
- **[P2] Use the actual subsystem logger contract:** `extensions/memory-core/src/memory/manager.ts:393`
  - body: `cleanupOrphanedTempFiles` requires `info`, `warn`, and `error` to accept arbitrary `unknown[]`, but `createSubsystemLogger` exposes `(message: string, meta?: Record<string, unknown>)`. Passing `log` here fails prod and test type checks, so narrow the helper logger type to the real contract.
  - confidence: 0.97
- **[P3] Remove sidecar-only temp groups:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:295-298`
  - body: If startup sees only `*.sqlite.tmp-*-wal` or `-shm` after a crash between publishing the main DB and post-swap sidecar cleanup, `fs.stat` on the base path fails and this branch marks the group fresh forever. Cleanup needs to remove aged sidecar-only groups too.
  - confidence: 0.82
