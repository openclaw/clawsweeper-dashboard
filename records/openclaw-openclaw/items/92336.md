## Summary

The PR targets the wrong precedence layer: current main already resolves configured exec security as an agent default after one-turn directives and durable session overrides, while this patch copies that default into the durable override slot. That fixes the reported restart symptom for a new session but makes the configured value sticky in existing sessions and can prevent later config changes from taking effect; the linked bug should remain open while the runtime path that fails to supply `agentExecDefaults` is repaired. The supplied proof is unit-test-only and does not exercise the reported gateway restart plus exec workflow.

Likely related people
- Lu Wang — high confidence; introduced the current reply exec-override/session-state pipeline in `43b108896219a265beea4a2afcb19750940d4730`.
- Peter Steinberger — high confidence; repeated recent work on exec approval defaults and security semantics, including `c678ae7e7a53e76c4641577c1f3203ffc6ec0152` and `61718d2da55dd1755db6ab270710e9e1c122a521`.

## What This Changes

The PR initializes durable `SessionEntry.execSecurity` from `tools.exec.security` during session creation and adds the same two regression cases in a new focused test file and the existing session test file.

## Best Possible Solution

Keep `tools.exec.security` as the canonical configured default, keep `SessionEntry.execSecurity` only for explicit `/exec` session overrides, and repair the startup/reply path that fails to pass the configured agent exec defaults into `resolveReplyExecOverrides`; cover cold restart plus an existing-session config change without duplicating tests.

## Reproduction Assessment

Yes, at source level with high confidence: the linked report’s cold-start path and the missing effective default can be traced, but the PR does not provide an after-fix live gateway restart and exec attempt.

## Solution Assessment

No. The proposed assignment fixes the symptom by populating the higher-precedence persisted override field; the narrower maintainable fix is to restore configured-default propagation through the existing exec override resolver.
