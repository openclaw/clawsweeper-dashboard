## Summary

Keep open: this is a draft PR with a protected `maintainer` label, and the diff introduces a write-risk where `--diff` is accepted without `--dry-run` and then falls through to the normal write path. The PR body also has only an illustrative output shape, not observed after-fix real behavior proof.

## What This Changes

The PR adds an `openclaw path set --dry-run --diff` option, unified-diff formatting, docs/changelog updates, and handler tests for human and JSON dry-run output.

## Best Possible Solution

Land the oc-path diff feature only after `--diff` cannot trigger writes when `--dry-run` is omitted, real CLI proof is posted, and maintainers approve the new flag/output contract.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the review finding: the PR registers `--diff`, but the handler ignores it unless `dryRun` is true and otherwise reaches the existing `fs.writeFile` path.

## Solution Assessment

No. The feature direction is plausible, but the implementation should reject `--diff` without `--dry-run` or make `--diff` imply dry-run, with a regression test for that mode boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard `--diff` so it cannot write:** `extensions/oc-path/src/cli.ts:496`
  - body: Registering `--diff` makes `openclaw path set ... --diff` parse successfully, but `pathSetCommand` only uses `options.diff` inside the dry-run branch.
