## Summary

Keep open. The PR is useful and has real CLI proof, but current head still has a correctness blocker: the custom diff formatter can omit a final-newline-only byte change from `--dry-run --diff`, and a maintainer review has already requested that fix.

## What This Changes

The PR adds `openclaw path set --dry-run --diff` unified diff output, rejects `--diff` without `--dry-run`, and updates OC Path docs, tests, and changelog.

## Best Possible Solution

Land the additive dry-run diff preview after the formatter represents EOF newline changes accurately and a targeted OC Path regression covers that edge case.

## Reproduction Assessment

Yes for the review finding: source inspection shows `splitDiffLines()` drops the final empty element, and the maintainer review provides a concrete PR-head command where changed bytes lose a trailing newline while the diff shows no actual change. For the requested feature itself, reproduction is not a bug-style current-main failure; current main simply lacks the new `--diff` flag.

## Solution Assessment

No for the current patch: making `--diff` dry-run-only is the right shape, but the custom formatter must preserve byte-level trailing-newline changes. A proven unified-diff formatter or explicit line-terminator modeling with a no-newline marker would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve trailing-newline changes in diff output:** `extensions/oc-path/src/cli.ts:208-210`

In that case `--dry-run --diff` can emit a hunk with only context lines while `bytes` changed, so operators cannot actually review the addressed edit before writing.
