The additive OC Path diff preview is useful and the contributor supplied sufficient real behavior proof, but the current PR branch still has a formatter correctness issue where CRLF/LF-only byte changes can disappear from `--dry-run --diff`; the protected `maintainer` label also rules out cleanup close.

## What This Changes

The PR adds `openclaw path set --dry-run --diff` preview output, updates OC Path docs/tests/changelog, and stringifies workflow dispatch fields in the release-candidate checklist script.

## Best Possible Solution

Land the dry-run diff preview after the formatter represents line-terminator byte changes accurately and a focused OC Path regression covers that edge case.

## Reproduction Assessment

Yes for the review finding: source inspection shows `splitDiffLines()` normalizes CRLF and LF before hunk comparison, while Markdown `set` can rebuild edited files with LF. For the feature itself, current main simply lacks `--diff`, so this is not a bug-style current-main reproduction.

## Solution Assessment

No for the current patch. Keeping `--diff` dry-run-only is the right shape, but the formatter needs explicit line-terminator modeling or another proven diff path so byte-level newline changes are visible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve line-ending-only byte changes:** `extensions/oc-path/src/cli.ts:206-207`
  - body: `formatUnifiedDiff()` only checks raw equality before splitting with `/\r?\n/`, so CRLF and LF versions of the same text become identical during hunk generation. Markdown `set` can canonicalize edited CRLF files through the existing render/finalize path, which means `--dry-run --diff` can say there are no byte changes while `bytes` would still overwrite the file with different line endings.
  - confidence: 0.88
