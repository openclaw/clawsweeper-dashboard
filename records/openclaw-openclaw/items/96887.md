## What This Changes

Adds a per-run hasDelivered flag to the embedded message_tool_only after-tool hook and a regression test so subsequent delivered message.send source replies return terminate: true.

## Best Possible Solution

Implement or explicitly approve a message_tool_only lifecycle invariant that prevents repeated implicit current-source sends while preserving follow-up non-message tools, then prove it in Telegram and the Slack continuation path.

## Solution Assessment
