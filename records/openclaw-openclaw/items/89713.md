## Summary

This PR fixes a real plugin-registry policy-refresh bug, but the submitted guard is too broad: it treats every install record missing from `plugins[]` as rebuild-worthy, while the sibling registry snapshot path deliberately keeps vanished durable install records on the persisted fast path.

## What This Changes

The PR adds a policy-refresh completeness guard and regression test so persisted plugin indexes with install records but missing plugin rows rebuild from source.

## Best Possible Solution

Force source rebuild only when a recoverable install record is missing from `plugins[]`, while keeping vanished durable records on the policy-refresh fast path and adding regression coverage for both cases.

## Solution Assessment

No as submitted. The fix belongs in the policy-refresh guard, but it should reuse the existing recoverable-record invariant instead of treating every missing install record as stale.
