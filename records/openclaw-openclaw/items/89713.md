## Summary

Keep this PR open as a useful bug-fix candidate: current main still lets a policy refresh reuse a persisted index whose `installRecords` mention plugins missing from `plugins[]`, while the PR adds the missing guard and focused regression coverage with sufficient runtime proof.

## What This Changes

Adds a policy-refresh guard that rebuilds the persisted installed plugin index when durable install records lack corresponding plugin records, plus a regression test for that stale persisted state.

## Best Possible Solution

Land this narrow guard and regression test after normal exact-head CI and maintainer checks so policy refresh keeps installed plugin records and persisted plugin rows consistent.

## Reproduction Assessment

Yes. Source inspection on current main shows the policy-refresh fast path can skip rebuilding without checking that install records have matching plugin records, and the PR's test models that exact stale persisted state.

## Solution Assessment

Yes. The guard belongs in the policy-refresh fast-path decision because the existing fallback already rebuilds the installed plugin index from install records and manifests; changing only snapshot reads would not repair the persisted index during config mutation.
