## Summary

Keep this PR open: the current main policy-refresh fast path can preserve a persisted plugin view whose `installRecords` still contain managed plugins but whose `plugins[]` rows are missing, and this patch is a narrow guard plus regression test for that state. I found no actionable patch defect; the only maintainer choice is to land one canonical fix for the same bug cluster rather than merge overlapping open fixes.

## What This Changes

The PR adds a persisted-index completeness guard for `policy-changed` refreshes and a regression test showing that a persisted index with install records but missing plugin rows rebuilds from source.

## Best Possible Solution

Land one canonical fix for https://github.com/openclaw/openclaw/issues/89606 that keeps the policy-refresh fast path only when the persisted plugin rows are complete for durable install records; avoid merging overlapping PRs for the same invariant.

## Reproduction Assessment

Yes. Source inspection shows current main can reuse a `policy-changed` persisted index without checking that `installRecords` plugin ids are present in `plugins[]`; the PR body also supplies terminal proof for the same state shape.

## Solution Assessment

Yes. The proposed check is at the persisted installed-index owner boundary, preserves the existing policy-only fast path for complete indexes, and falls back to the established source rebuild path when the persisted view is incomplete.
