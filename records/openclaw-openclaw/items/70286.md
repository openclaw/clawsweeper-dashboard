## Summary

Keep open: current main still has the reported plugin-bound handled path returning no committed final response, and Telegram still has a no-delivery group silent-reply fallback gated on `!queuedFinal`. The only matching fix PR is closed unmerged, and search found no replacement canonical fix.

## What This Changes

The issue requests suppressing Telegram silent-reply fallback when a plugin claims an inbound message with `inbound_claim` and responds asynchronously.

## Best Possible Solution

Land a narrow fix that preserves an explicit async plugin-claim response signal through dispatch finalization and Telegram fallback handling, with regression coverage; defer timeout behavior to a separate follow-up.

## Reproduction Assessment

Yes, at source level. A targeted plugin-owned `inbound_claim` handled result returns `queuedFinal: false` with zero counts, and Telegram can enter its group no-delivery silent fallback when `!queuedFinal`.

## Solution Assessment

No, current main is not the best solution because it still reproduces the regression. The safer fix is an explicit committed async-response signal preserved through `src/auto-reply/dispatch.ts`, rather than a direct `queuedFinal` flip that the wrapper can erase.
