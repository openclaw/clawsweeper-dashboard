## Summary

Keep open: current main still has the plugin-owned handled claim returning no committed final response, and Telegram still has the group silent-reply fallback path gated by false `queuedFinal`; the linked fix PR is closed unmerged and no canonical open replacement was found.

## What This Changes

The issue reports that Telegram can emit a silent-reply fallback before an asynchronously responding plugin-owned `inbound_claim` handler sends its real response.

## Best Possible Solution

Add an internal committed-response signal for plugin-owned async claims and use it to suppress Telegram’s silent fallback while preserving existing missing-plugin, no-handler, declined, error, cancelled, and failed local-delivery behavior.

## Reproduction Assessment

Yes, at source level. A plugin-owned handled claim returns `queuedFinal: false` with zero local final counts, and Telegram’s group no-visible-delivery fallback is still gated on false `queuedFinal`.

## Solution Assessment

Yes, the maintainable fix is an explicit committed-response signal for plugin-owned async turns. A bare `queuedFinal: true` flip is not enough by itself because dispatch finalization currently ties `queuedFinal` to local final counts.
