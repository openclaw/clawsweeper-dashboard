## Summary

Keep open: current main still preserves the reported plugin-owned async claim versus Telegram silent-reply mismatch, and the linked fix PR was closed unmerged despite the source path still showing the failing signal.

## What This Changes

The issue reports that Telegram sends a silent-reply fallback before a plugin-owned asynchronous `inbound_claim` response arrives.

## Best Possible Solution

Add an internal committed external-response signal for plugin-owned async claims, preserve it through dispatch finalization, and use it to suppress Telegram’s silent fallback without changing missing-plugin, no-handler, declined, error, cancelled, or failed local-delivery behavior.

## Reproduction Assessment

Yes, at source level. A plugin-owned handled claim returns `queuedFinal: false` with zero local final counts, and Telegram still has a no-visible-delivery group fallback path gated by false `queuedFinal`.

## Solution Assessment

Yes. The narrow maintainable fix is an explicit committed-response signal rather than only flipping `queuedFinal`, because dispatch finalization currently ties `queuedFinal` to local final counts.
