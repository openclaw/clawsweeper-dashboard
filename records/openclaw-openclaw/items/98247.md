Full review comments:

- **[P2] Import the polyfill from this test directory:** `ui/src/array-prototype-polyfills.test.ts:6`
  - body: This test file lives in `ui/src`, so `../array-prototype-polyfills.ts` resolves to `ui/array-prototype-polyfills.ts`, which does not exist. The `beforeEach` import will fail before any polyfill behavior is covered; use the sibling `./array-prototype-polyfills.ts` path.
  - confidence: 0.97
- **[P2] Make the global polyfills match native edge cases:** `ui/src/array-prototype-polyfills.ts:52-67`
  - body: Because this module patches standard globals, the fallbacks need native-compatible semantics. Right now `toSpliced(1)` keeps the array instead of deleting through the end, and `with(-len - 1, value)` clamps to index 0 instead of throwing, so UI code or dependencies can get nonstandard behavior once these methods appear to exist.
  - confidence: 0.9
- **[P2] Restore native descriptors after prototype tests:** `ui/src/array-prototype-polyfills.test.ts:33-40`
  - body: The UI Vitest project runs with `isolate: false`, but these tests delete native Array/Object methods in `afterEach` and never restore the original descriptors. If this file runs before another UI test in the same worker, later tests can unexpectedly lose the native built-ins.
  - confidence: 0.88
