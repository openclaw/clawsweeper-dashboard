## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Port Mermaid handling to the active renderer:** `ui/src/ui/markdown.ts:260-262`
  - body: This change inserts Mermaid markup into the old `htmlEscapeRenderer.code` path, but current main renders fenced code through `markdown-it` (`md.renderer.rules.fence`) and streaming-safe markdown. As submitted, the branch is dirty and cannot be trusted to add Mermaid support to the active renderer without a real rebase and port.
  - confidence: 0.95
- **[P1] Remove the workspace-wide lodash override:** `package.json:1253`
  - body: A Control UI Mermaid feature should not add a root `lodash-es` override that changes dependency resolution for the whole workspace. Root policy treats dependency overrides and lockfiles as security-sensitive, so this needs removal or explicit maintainer approval with regenerated lockfile proof.
  - confidence: 0.9
- **[P2] Scope Mermaid placeholders to surfaces that render them:** `ui/src/ui/markdown.ts:261`
  - body: `toSanitizedMarkdownHtml` is shared by chat, Markdown sidebar, side results, skill cards, and other views, but the PR only schedules Mermaid rendering from the chat lifecycle. Any shared caller that receives a Mermaid fence would get an empty render slot with the source hidden under details instead of a rendered diagram or normal code block.
  - confidence: 0.86
- **[P2] Call Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:176-177`
  - body: Mermaid 11.13.0 documents that `render()` can return `bindFunctions` that must run after the SVG is inserted. The PR's local API type drops that return field and only assigns `innerHTML`, so Mermaid diagrams with tooltips or click bindings silently become static.
  - confidence: 0.82
- **[P2] Allow failed Mermaid blocks to retry:** `ui/src/ui/mermaid.ts:143-145`
  - body: The renderer only queries `.mermaid-block:not([data-mermaid-status])`, but load and render failures mark blocks as `error`. A transient chunk, network, or first-pass render failure therefore permanently skips the same DOM block even though the module promise is reset.
  - confidence: 0.8
