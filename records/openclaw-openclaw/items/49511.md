## Summary

Keep open, but do not merge as-is: current main still lacks built-in WebChat Mermaid rendering, and this is the canonical remaining PR, but the branch is stale against the hardened markdown-it renderer and has source-reproducible correctness and security blockers.

## What This Changes

The PR adds lazy-loaded Mermaid rendering, styling, tests, a UI dependency, and lockfile changes for Mermaid code fences in Control UI WebChat.

## Best Possible Solution

Land a narrower current-main implementation that preserves markdown-it, sanitizer/task-list/code-block contracts, scopes Mermaid rendering to consumers that schedule it, calls Mermaid's full render contract, and receives dependency/security approval.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The merge blockers are source-reproducible from the PR head and current main: the branch uses the older marked renderer, rewrites shared markdown without shared scheduling, and drops Mermaid bindFunctions.

## Solution Assessment

No. Mermaid rendering may be a valid feature, but this branch should be ported to the current markdown-it path and implemented as an explicitly scoped post-render pass before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port Mermaid support to markdown-it:** `ui/src/ui/markdown.ts:2`
  - body: Current main moved shared UI markdown rendering to markdown-it for parser hardening, task lists, sanitizer behavior, cache, and localized code-block copy labels. This branch still imports `marked` and calls `marked.parse`, so rebasing it as-is would replace that current renderer instead of extending it.
  - confidence: 0.94
- **[P2] Scope Mermaid rewriting to scheduled views:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared outside the chat transcript, but this branch converts every Mermaid fence into an empty `.mermaid-block__render` while only the chat lifecycle schedules rendering. Sidebar and other shared markdown consumers can show a blank panel with the source hidden under details.
  - confidence: 0.9
- **[P2] Let failed Mermaid blocks retry:** `ui/src/ui/mermaid.ts:149-150`
  - body: The renderer selects only blocks without `data-mermaid-status`, then marks import, streaming, parse, or render failures as `error`. A partial streamed diagram or transient chunk failure can permanently skip the same DOM node after the definition becomes valid.
  - confidence: 0.89
- **[P2] Call Mermaid bindFunctions after injection:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid's render contract can return `bindFunctions` that must run after the SVG is inserted so links, tooltips, and click handlers work. This branch's API type drops that property and only uses `svg`, so interactive diagrams silently degrade.
  - confidence: 0.86
- **[P2] Show an empty-definition fallback:** `ui/src/ui/mermaid.ts:178-180`
  - body: When the Mermaid source is empty, this branch marks the block as `error` but never writes fallback text into the render target. Users then see a blank panel with the source collapsed instead of an actionable render failure.
  - confidence: 0.82
