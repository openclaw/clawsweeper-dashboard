  - body: Current main no longer uses the marked `htmlEscapeRenderer.code` path that this branch modifies; it renders fenced code through `md.renderer.rules.fence`. Carrying this branch through conflict resolution as-is would drop the current markdown-it renderer behavior, including recent highlighting, task-list, sanitizer, and ReDoS-hardening work.
  - confidence: 0.94
- **[P2] Render Mermaid placeholders only where rendering is scheduled:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared outside WebChat, but this branch rewrites every Mermaid fence into an empty render placeholder while only scheduling Mermaid rendering from the chat lifecycle. Mermaid content in markdown sidebars or other shared previews can therefore show a blank panel instead of a diagram or normal code fence.
  - confidence: 0.88
- **[P2] Allow failed Mermaid blocks to retry:** `ui/src/ui/mermaid.ts:149-150`
  - body: The renderer only selects `.mermaid-block:not([data-mermaid-status])`, then marks load/render failures as `error`. A transient import failure or early render failure leaves the same block skipped on later scheduled passes until the DOM is rebuilt.
  - confidence: 0.87
- **[P2] Apply Mermaid bindFunctions after SVG insertion:** `ui/src/ui/mermaid.ts:185-188`
  - body: Mermaid's render contract can return `bindFunctions` that callers should invoke after inserting the SVG. This branch discards that callback, so Mermaid diagrams that rely on built-in click or tooltip bindings render as inert SVGs.
  - confidence: 0.82
