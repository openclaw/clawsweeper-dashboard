  - confidence: 0.88
- **[P2] Let transient Mermaid failures retry:** `ui/src/ui/mermaid.ts:150`
  - body: The renderer only selects `.mermaid-block:not([data-mermaid-status])`, then marks load, empty-definition, and render failures as `error`. A transient chunk failure or first pass over incomplete streamed content is therefore skipped forever until the DOM is rebuilt, so failed blocks need a retry path or a status that later updates can revisit.
  - confidence: 0.8

  - body: Mermaid 11.13.0's `RenderResult` can include `bindFunctions`, and upstream documents calling it after the SVG is inserted so diagram event listeners work. This PR narrows the local type to `{ svg }` and discards `bindFunctions`, so diagrams with interactions or tooltips render as static/broken output.
  - confidence: 0.78
