## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Retry Mermaid blocks after transient render failures:** `ui/src/ui/mermaid.ts:150`
  - body: The render pass only selects `.mermaid-block:not([data-mermaid-status])`, but load/render failures set `data-mermaid-status="error"`. A streamed incomplete Mermaid fence or temporary chunk/render failure is never revisited after the message completes or the import can retry, leaving a permanent broken diagram until the DOM is rebuilt.
  - confidence: 0.9
- **[P2] Scope Mermaid placeholders to rendered surfaces:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared by the Markdown sidebar and other previews, but this branch turns every Mermaid fence into an empty render placeholder while only the chat lifecycle calls `scheduleMermaidRender()`. Those shared views will show a blank panel with source hidden instead of a rendered diagram or a normal code fence.
  - confidence: 0.88
- **[P2] Run Mermaid bind functions after inserting SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11's `render()` result can include `bindFunctions`, and the upstream type docs show callers should run it after assigning the SVG into the target element. This implementation discards it, so diagrams with Mermaid click handlers or tooltips render as static SVG with interactions silently broken.
  - confidence: 0.83
