  - confidence: 0.9

  - body: Adding `mermaid` to the UI package does not justify pinning `lodash-es` for the entire workspace from the root override table. This changes dependency resolution outside the feature scope and needs removal or explicit maintainer approval before merge.
  - confidence: 0.91
- **[P2] Call Mermaid bindFunctions after SVG insertion:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid's render contract can return `bindFunctions` that callers should run after inserting the SVG. This code only uses `svg`, so Mermaid diagrams with tooltips or click bindings will render as static images with broken interactions.
  - confidence: 0.86
- **[P2] Show a fallback for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When the Mermaid fence body is empty, this branch marks the block as `error` and continues without calling `setMermaidRenderError()`. Users then see a blank collapsed panel instead of actionable fallback text, which is a regression from plain code-fence rendering.
  - confidence: 0.88
