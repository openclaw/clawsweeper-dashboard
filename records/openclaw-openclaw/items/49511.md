## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Retry Mermaid blocks after transient failures:** `ui/src/ui/mermaid.ts:150`
  - body: `scheduleMermaidRender()` runs while `chatStream` changes, so an incomplete streamed fence or transient Mermaid error can mark the block `data-mermaid-status="error"`. Because later passes only select blocks without any status, the completed diagram is skipped until the DOM is rebuilt; allow failed blocks to retry when the source changes or when a later render pass runs.
  - confidence: 0.88
- **[P2] Scope Mermaid rewriting to surfaces that render it:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared by non-chat previews such as the markdown sidebar, but this branch converts every Mermaid fence into an empty render placeholder while only the chat lifecycle schedules Mermaid rendering. Those surfaces will show blank panels with the source hidden unless the renderer is installed for every consumer or the rewrite is limited to WebChat.
  - confidence: 0.86
- **[P2] Apply Mermaid bind functions after inserting SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid's render contract returns `bindFunctions` and documents calling it after inserting the SVG. This code types and consumes only `{ svg }`, so diagrams that rely on Mermaid click handlers or tooltips render as static SVGs with their expected interactions silently missing.
  - confidence: 0.8
