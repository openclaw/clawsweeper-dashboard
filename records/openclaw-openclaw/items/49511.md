## Summary

Keep open: this is the canonical active implementation for WebChat Mermaid rendering, but the current branch is not merge-ready because it is stale against the current markdown-it renderer and introduces unresolved SVG/dependency security review concerns.

## What This Changes

The PR adds lazy-loaded Mermaid rendering for Control UI WebChat markdown fences, Mermaid-specific styles/tests, a new Mermaid dependency, and package/lockfile updates.

## Best Possible Solution

Land a focused current-main Mermaid implementation in the markdown-it fence renderer or a clearly scoped browser render pass, preserving existing sanitizer, task-list, code-block, i18n, and shared-markdown contracts.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The merge blockers are source-reproducible from current main and the PR head: main uses markdown-it, while the branch rewrites Mermaid fences in the old marked renderer and schedules rendering through only part of the chat lifecycle.

## Solution Assessment

No. Mermaid rendering is a plausible feature direction, but this branch is not the best merge shape until it is ported to current main's renderer and the Mermaid SVG/dependency boundary is accepted by maintainers.

  - confidence: 0.87

  - confidence: 0.89

  - body: Mermaid's render contract can return `bindFunctions` that callers must run after inserting the SVG for links, tooltips, and click handlers. This branch only uses `svg`, so interactive diagrams silently degrade.
  - confidence: 0.84
