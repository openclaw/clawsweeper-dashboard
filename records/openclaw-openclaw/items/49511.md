  - confidence: 0.86

  - body: Manual refresh keeps `chatManualRefreshInFlight` true while refreshed messages are applied, so `handleUpdated()` returns before this scheduler call and the later flag-clear update does not touch the checked chat fields. Refreshed transcripts with Mermaid fences can stay blank until an unrelated update or tab switch.
  - confidence: 0.83

  - body: The loader resets the module promise after an import failure, but this selector only revisits blocks without `data-mermaid-status`. Once a transient load or partial-definition render marks a block `error`, later scheduler passes skip the same diagram until the DOM is rebuilt.
  - confidence: 0.84

  - body: When a Mermaid fence has no definition, the branch marks the block `error` and continues without calling the error renderer. Users see an empty panel with the source collapsed, which regresses from the previous visible code fence for malformed or partial content.
  - confidence: 0.8
