  - body: The feature only needs a Control UI Mermaid dependency, but this line adds a root override that rewrites `lodash-es` resolution for the whole workspace. Root overrides are compatibility and supply-chain surface; remove it or include explicit maintainer-approved audit/proof for the repo-wide pin.
  - confidence: 0.88
- **[P2] Keep shared markdown consumers from showing blank Mermaid blocks:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is used outside chat, but this branch rewrites every Mermaid fence into a block that only `scheduleMermaidRender()` can fill. Markdown sidebar or other shared consumers can therefore display an empty render slot with source hidden unless the implementation is scoped to chat or all consumers schedule rendering.
  - confidence: 0.86
- **[P2] Retry errored Mermaid blocks on later render passes:** `ui/src/ui/mermaid.ts:150`
  - body: The renderer only selects `.mermaid-block:not([data-mermaid-status])`, then marks load/render failures as `error`. A transient chunk failure or streamed incomplete diagram can become permanently skipped even after the source completes or the import can retry.
  - confidence: 0.9
- **[P2] Run Mermaid bindFunctions after inserting the SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13.0 can return `bindFunctions` from `render()` for post-insertion diagram behavior, but this wrapper types and reads only `{ svg }`. Interactive Mermaid behavior such as click bindings or tooltips will silently be lost.
  - confidence: 0.82
- **[P3] Show fallback text for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When the code fence is empty, the branch marks the block as `error` and continues without calling the fallback helper. Users see a blank placeholder with the source collapsed instead of an actionable render-failed message.
  - confidence: 0.78
