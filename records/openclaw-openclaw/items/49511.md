## Summary

Keep this PR open because current main still does not render Mermaid fences in WebChat and the duplicate/related Mermaid attempts now point back to this PR as the canonical remaining implementation. The branch should not merge as-is: it is conflicting and stale behind the current markdown-it renderer and needs maintainer/security judgment for the Mermaid dependency and SVG rendering boundary.

## What This Changes

The PR adds built-in Mermaid rendering for WebChat code fences by adding a Mermaid dependency, markdown/code-fence output, render scheduling, chat styles, tests, and package/lockfile updates.

## Best Possible Solution

If maintainers want built-in Mermaid, land a narrow current-main implementation on the markdown-it fence renderer that preserves sanitizer/cache/i18n behavior, renders every surface that emits Mermaid blocks, and honors Mermaid's full render result contract.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The review blockers are source-reproducible by comparing the PR head's marked-based renderer and scheduler with current main's markdown-it renderer and shared markdown consumers.

## Solution Assessment

No, not as currently implemented. The safer path is a focused port onto current main's markdown-it renderer plus shared render scheduling, after maintainer/security approval of the dependency and SVG boundary.
