  - confidence: 0.9
- **[P2] Scope Mermaid placeholders to surfaces that render them:** `ui/src/ui/markdown.ts:261`
  - body: `toSanitizedMarkdownHtml` is shared by chat, Markdown sidebar, side results, skills, and other views, but the PR only schedules Mermaid rendering from the chat lifecycle. Shared callers that receive a Mermaid fence would get an empty render slot with the source hidden under details instead of a rendered diagram or normal code block.
  - confidence: 0.86
- **[P2] Call Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:176-177`
  - body: Mermaid 11.13.0's render result can include `bindFunctions` that must run after the SVG is inserted. The PR's local type drops that field and only assigns `innerHTML`, so diagrams with tooltips or click bindings silently become static.
  - confidence: 0.82
- **[P2] Allow failed Mermaid blocks to retry:** `ui/src/ui/mermaid.ts:143-145`
  - body: The renderer only queries `.mermaid-block:not([data-mermaid-status])`, while load and render failures mark blocks as `error`. A transient chunk, network, or first-pass render failure permanently skips the same DOM block even if a later render pass could succeed.
  - confidence: 0.8
