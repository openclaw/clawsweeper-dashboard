  - confidence: 0.93
- **[P1] Remove the workspace-wide lodash-es override:** `package.json:1253`
  - body: A Control UI Mermaid feature should not pin `lodash-es` for the whole workspace without explicit maintainer approval. This override changes dependency resolution outside the UI feature and can create install or upgrade drift for unrelated packages.
  - confidence: 0.88
- **[P2] Scope Mermaid placeholders to surfaces that render them:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is used by non-chat surfaces, but this PR rewrites every Mermaid fence into an empty render placeholder while only scheduling Mermaid rendering from the chat lifecycle. Non-chat markdown views can therefore show a blank block with source hidden under details.
  - confidence: 0.86
- **[P2] Call Mermaid bindFunctions after injecting the SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13.0 render results can include `bindFunctions`, and upstream documents calling them after inserting the SVG. This code only uses `svg`, so diagrams that rely on Mermaid-provided bindings can render with broken interactions.
  - confidence: 0.78
- **[P2] Show a fallback for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When the Mermaid source is empty, the code marks the block as error and continues without calling the fallback renderer. Users get an empty placeholder with the source collapsed instead of an actionable failure message.
  - confidence: 0.82
