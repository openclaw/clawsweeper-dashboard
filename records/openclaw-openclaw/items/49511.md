## Summary

Keep this PR open: current main still does not render Mermaid fences in Control UI WebChat, and the closed related Mermaid PR now points back here as the canonical implementation. The branch should not merge as-is because it is stale against the current markdown-it renderer/security boundary and still has source-reproducible rendering lifecycle bugs plus SVG/dependency security review needs.

## What This Changes

The PR adds lazy-loaded Mermaid rendering for Control UI WebChat markdown fences, Mermaid source/preview styling and tests, a Mermaid dependency, lockfile updates, a root lodash-es override, and chat lifecycle scheduling.

## Best Possible Solution

Land a focused current-main Mermaid implementation in the markdown-it fence renderer or a scoped browser render pass, preserving sanitizer, task-list, code-block, i18n, shared-markdown, and lifecycle contracts after maintainers accept the Mermaid/SVG boundary.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The merge blockers are source-reproducible by comparing current main's markdown-it renderer and lifecycle paths with the PR head's marked renderer, chat-only scheduler, and Mermaid render-error handling.

## Solution Assessment

No. Mermaid rendering is a reasonable feature direction, but this branch is not the best merge shape until it is ported to current main's renderer and the Mermaid SVG/dependency boundary is accepted by maintainers.
