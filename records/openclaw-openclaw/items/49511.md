  - confidence: 0.86
- **[P2] Keep shared markdown consumers from showing blank Mermaid blocks:** `ui/src/ui/markdown.ts:263-264`

Other views that call `toSanitizedMarkdownHtml()` can show an empty Mermaid box with the source hidden unless the feature is scoped or those consumers also render/fallback correctly.
  - confidence: 0.84
- **[P2] Run Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:185-187`
  - body: Mermaid 11.13.0 documents that `render()` can return `bindFunctions` that callers should run after inserting the SVG. This wrapper type drops that field, so diagrams with Mermaid-provided interactions such as click handlers or tooltips silently lose behavior.
  - confidence: 0.8
- **[P3] Show fallback text for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When the Mermaid fence is empty, the block is marked as error without writing any fallback text. Users then see a blank render area with the source collapsed, which regresses from the previous visible code-fence behavior for malformed output.
  - confidence: 0.76
