The PR proposes lazy-loaded Mermaid rendering for Control UI WebChat markdown fences, associated Mermaid styling/tests, and Mermaid package/lockfile updates.

## Best Possible Solution

Land a focused current-main Mermaid implementation in the markdown-it fence renderer or a scoped browser render pass, preserving sanitizer, task-list, code-block, i18n, shared-markdown, and lifecycle contracts.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The merge blockers are source-reproducible by comparing current main's markdown-it renderer and lifecycle paths with the PR head's marked renderer, chat-only scheduler, and Mermaid render-error handling.

## Solution Assessment

No. Mermaid rendering is a reasonable feature direction, but this branch is not the best merge shape until it is ported to current main's renderer and the Mermaid SVG/dependency boundary is accepted by maintainers.
