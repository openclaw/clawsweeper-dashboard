  - body: Current main renders markdown through markdown-it's `md.renderer.rules.fence` path, but this branch adds Mermaid handling to the old `marked` renderer. Rebuild the feature on the active renderer so it preserves current markdown safety, highlighting, i18n, streaming, and cache behavior.

  - body: This Control UI feature adds a root `lodash-es` override, which changes dependency resolution for the whole workspace. Keep the Mermaid dependency scoped to the UI package or get explicit maintainer approval for the global pin with proof that it is required.
  - confidence: 0.9
- **[P2] Render Mermaid only where the scheduler runs:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared by non-chat previews, but the PR turns every Mermaid fence into an empty render placeholder while only calling `scheduleMermaidRender()` from chat lifecycle. Either scope the rewrite to chat or schedule rendering for every consumer; otherwise sidebars and file previews can show blank Mermaid blocks.
  - confidence: 0.88
- **[P2] Let failed Mermaid blocks retry on later updates:** `ui/src/ui/mermaid.ts:150`
  - body: `renderMermaidBlocks()` selects only blocks without `data-mermaid-status`, then marks load/render failures as `error`. A transient chunk failure or a streamed diagram parsed before completion is skipped forever until the DOM is rebuilt, so retryable chat diagrams can stay broken.
  - confidence: 0.86
- **[P2] Call Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11's render result can include `bindFunctions` for diagram interactions, and upstream usage calls it after inserting the SVG. This code discards that callback, so valid Mermaid diagrams with tooltips or click bindings render as static output with broken interactions.
  - confidence: 0.82
