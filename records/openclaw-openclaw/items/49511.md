  - body: This branch's markdown renderer still imports `marked`, but current main moved WebChat markdown to markdown-it for the ReDoS/security fix and now has task-list, sanitizer, code-block, and CJK URL contracts there.   - body: `renderMermaidBlocks()` only visits blocks with no `data-mermaid-status`, but both module-load and render failures set the status to `error`. A streamed or temporarily failed diagram can therefore stay stuck on the fallback text after the source becomes valid.
  - confidence: 0.9
- **[P2] Render Mermaid blocks wherever the shared renderer emits them:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared outside the chat transcript, but the PR converts every Mermaid fence into an empty render slot while only the chat lifecycle calls `scheduleMermaidRender()`. Shared markdown panes can show blank diagram boxes instead of either rendered diagrams or normal code fences.
  - confidence: 0.88
- **[P2] Apply Mermaid bind functions after inserting SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid's render API can return `bindFunctions` for diagram interactions and its own type docs show callers invoking it after `innerHTML = svg`. This code discards that callback, so interactive Mermaid diagrams render as static SVGs with broken clicks or tooltips.
  - confidence: 0.84
