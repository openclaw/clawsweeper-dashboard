## Summary

Keep open: this is the canonical active PR for built-in Control UI/WebChat Mermaid rendering, but the branch is not currently mergeable because it is stale against main's markdown-it renderer and still has source-reproducible lifecycle, retry, and SVG/dependency review blockers.

## What This Changes

The PR adds lazy-loaded Mermaid rendering for Control UI WebChat markdown fences, Mermaid source/preview styling and tests, a Mermaid dependency, lockfile updates, and a root lodash-es override.

## Best Possible Solution

Land a focused current-main Mermaid implementation in the markdown-it fence renderer or a scoped browser render pass, preserving sanitizer, task-list, code-block, i18n, shared-markdown, and lifecycle contracts after maintainers accept the Mermaid/SVG boundary.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The merge blockers are source-reproducible by comparing current main's markdown-it renderer and shared markdown consumers with the PR head's marked renderer, chat-only scheduler, and Mermaid render-error state handling.

## Solution Assessment

No. Mermaid rendering is a reasonable feature direction, but this branch is not the best merge shape until it is ported to current main's renderer and the Mermaid SVG/dependency boundary is accepted by maintainers.
