## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Allow Mermaid blocks to retry after transient errors:** `ui/src/ui/mermaid.ts:149-151`
  - body: This selector permanently skips any block after a transient render failure or incomplete streamed diagram sets `data-mermaid-status="error"`, so the completed diagram can stay stuck until the DOM is rebuilt.
  - confidence: 0.88
- **[P2] Only rewrite Mermaid fences where rendering is scheduled:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared outside chat, but the PR converts every Mermaid fence into an empty render slot while only chat lifecycle updates call the Mermaid scheduler. Non-chat markdown views can therefore show a blank panel instead of a normal code fence or rendered diagram.
  - confidence: 0.86
- **[P2] Run Mermaid bindFunctions after inserting the SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13's render contract can return `bindFunctions` that must be called after SVG insertion; dropping it makes Mermaid interactions or tooltips render as static broken output.
  - confidence: 0.82
- **[P3] Show a fallback for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When the Mermaid definition is empty, this path marks the block as errored but leaves the render target blank, which regresses from ordinary code-fence rendering for malformed or partial output.
  - confidence: 0.78
