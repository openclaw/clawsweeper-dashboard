  - body: A Control UI Mermaid feature should not pin `lodash-es` for the whole workspace unless maintainers explicitly approve that dependency-resolution change. This override can alter unrelated package resolution, so remove it or document the approved supply-chain reason with regenerated lockfile proof.
  - confidence: 0.88
- **[P2] Render Mermaid wherever the shared renderer emits it:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is used by shared views such as markdown sidebars and side results, but this PR only schedules Mermaid rendering from the chat lifecycle. If the shared renderer emits `.mermaid-block` placeholders in those views, users get an empty render slot with source hidden under details; either scope Mermaid output to chat or schedule rendering for every consumer.
  - confidence: 0.86
- **[P2] Let failed Mermaid blocks retry on later updates:** `ui/src/ui/mermaid.ts:150`
  - body: The renderer only selects `.mermaid-block:not([data-mermaid-status])`, while load and render failures mark blocks as `error`. A transient chunk failure or initially invalid streamed diagram therefore leaves the same DOM node permanently skipped even though later updates may make rendering possible.
  - confidence: 0.84
- **[P2] Call Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:186-188`
  - body: Mermaid's render contract returns `bindFunctions` for diagram interactions after the SVG is inserted. This code narrows the API to `{ svg }` and never calls those bindings, so tooltips or click behavior from valid Mermaid diagrams silently break in chat.
  - confidence: 0.8
