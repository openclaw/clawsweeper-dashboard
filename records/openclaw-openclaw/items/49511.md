## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Port Mermaid handling to the markdown-it fence renderer:** `ui/src/ui/markdown.ts:241-264`
  - body: The PR head adds Mermaid rendering inside `htmlEscapeRenderer.code`, but current main renders code fences through `md.renderer.rules.fence`. Because GitHub reports this branch dirty, the feature needs to be rebased and ported to the current markdown-it path instead of the removed marked renderer.
  - confidence: 0.94
- **[P2] Scope Mermaid rewriting to rendered markdown surfaces:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is used outside chat, but this branch rewrites every Mermaid fence into an empty `.mermaid-block__render` placeholder while only the chat lifecycle schedules Mermaid rendering. Non-chat markdown panes would show blank diagram boxes unless the scheduler is shared or Mermaid rewriting is scoped to chat.
  - confidence: 0.88
- **[P2] Call Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:3-5`
  - body: Mermaid 11.13.0 documents that `render()` can return `bindFunctions` that must run after the SVG is inserted. This wrapper types `render()` as returning only `svg` and never calls `bindFunctions`, so Mermaid diagrams with tooltips or click handlers would silently lose interactions.
  - confidence: 0.9
- **[P2] Let failed Mermaid blocks retry on later renders:** `ui/src/ui/mermaid.ts:150`
  - body: The renderer only selects blocks without `data-mermaid-status`, then marks failed renders as `error`. A transient chunk load failure or an incomplete streamed Mermaid definition can therefore permanently skip the same DOM block on later updates instead of retrying when the input is complete.
  - confidence: 0.83
- **[P1] Remove the workspace-wide lodash-es override:** `package.json:1253`
  - body: This root override changes dependency resolution for the whole workspace even though the feature is local to Control UI and Mermaid already declares its lodash-es range. Dependency overrides are security-sensitive in this repo and need either explicit maintainer approval or removal before merge.
  - confidence: 0.86
