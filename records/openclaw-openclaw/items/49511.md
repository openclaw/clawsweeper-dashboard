Full review comments:

- **[P1] Port Mermaid rendering onto the current markdown-it renderer:** `ui/src/ui/markdown.ts:263-264`
  - body: Current main's `toSanitizedMarkdownHtml()` now uses `markdown-it` and the fence renderer includes current highlighting, copy-label, sanitizer, and link-trimming behavior. This branch adds Mermaid support in the older `marked` renderer path, so resolving the conflict by taking this block would lose current markdown behavior; rebase the feature onto the current `md.renderer.rules.fence` implementation instead.
  - confidence: 0.9
- **[P2] Scope Mermaid blocks to rendered surfaces:** `ui/src/ui/markdown.ts:263-264`
  - body: This changes the shared markdown renderer to emit `.mermaid-block` for every Mermaid fence, but rendering is only scheduled from the chat lifecycle. Non-chat consumers such as the markdown sidebar can insert an empty render slot with the source hidden under `<details>`, so either scope the rewrite to chat or schedule rendering wherever this shared renderer is used.
  - confidence: 0.86
- **[P2] Schedule Mermaid after manual chat refreshes:** `ui/src/ui/app-lifecycle.ts:124`
  - body: Manual refresh keeps `chatManualRefreshInFlight` true while `refreshChat()` replaces messages, so `handleUpdated()` returns before this new scheduler call. The later flag-clear update does not touch the chat fields checked here, leaving refreshed transcripts with blank Mermaid render slots until another unrelated chat update or tab switch.
  - confidence: 0.83
- **[P2] Retry blocks after transient Mermaid failures:** `ui/src/ui/mermaid.ts:149-150`
  - body: The loader resets `mermaidApiPromise` after an import failure, but this selector only revisits blocks with no `data-mermaid-status`. Once the catch path marks current blocks as `error`, a transient chunk, network, or streamed partial-definition failure leaves the same diagrams permanently skipped until the DOM is rebuilt.
  - confidence: 0.84
- **[P3] Show a fallback for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When a Mermaid fence has no definition, this branch marks the block as `error` and continues without calling `setMermaidRenderError()`. Users see an empty panel with the source collapsed, which regresses from the previous visible code fence for malformed or partial content.
  - confidence: 0.8
- **[P2] Bind Mermaid interactions after inserting SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13.0 documents that `render()` can return `bindFunctions` to run after the SVG is inserted. This code destructures only `svg`, so diagrams that rely on Mermaid-managed interactions or tooltips render as static SVG with bindings silently dropped.
  - confidence: 0.74
