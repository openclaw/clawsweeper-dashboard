Full review comments:

- **[P1] Port Mermaid rendering to markdown-it:** `ui/src/ui/markdown.ts:202-206`
  - body: Current main's user-controlled markdown path renders through markdown-it, but this branch calls marked.parse() and adds Mermaid to the old marked renderer. Merging it would drop the current ReDoS hardening plus sanitizer, cache, task-list, and i18n behavior, so the Mermaid branch needs to live in md.renderer.rules.fence.
  - confidence: 0.93
- **[P2] Render Mermaid wherever shared markdown emits it:** `ui/src/ui/markdown.ts:263-264`
  - body: toSanitizedMarkdownHtml() is shared outside chat, but this branch rewrites every Mermaid fence into an empty render target while only the chat lifecycle calls scheduleMermaidRender(). Non-chat markdown consumers can show blank diagram panels with the source hidden behind details.
  - confidence: 0.89
- **[P2] Schedule Mermaid after manual refreshes:** `ui/src/ui/app-lifecycle.ts:124`
  - body: handleUpdated() returns while chatManualRefreshInFlight is true; manual refresh replaces history under that flag and clears it on a later animation frame. The new scheduler call is skipped for refreshed Mermaid blocks, leaving them blank until an unrelated update or tab switch.
  - confidence: 0.86
- **[P2] Allow transient Mermaid failures to rerender:** `ui/src/ui/mermaid.ts:149-151`
  - body: The render pass only selects .mermaid-block:not([data-mermaid-status]), then marks incomplete streamed blocks, import failures, and render failures as error. Once the streamed fence completes or the import can retry, the same DOM node is never revisited.
  - confidence: 0.9
- **[P2] Call Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13.0's render result can include bindFunctions, documented as needing a call after the SVG is inserted to attach interactions. This code destructures only svg, so links, tooltips, and click handlers in valid diagrams silently degrade.
  - confidence: 0.84
- **[P2] Show fallback text for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: If the fenced Mermaid block is empty, this branch marks the block as error and continues without calling setMermaidRenderError. Users see an empty panel with collapsed source instead of actionable fallback text, which regresses from plain code-fence rendering.
  - confidence: 0.86
