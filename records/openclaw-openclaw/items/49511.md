Full review comments:

- **[P1] Port Mermaid output to the markdown-it fence renderer:** `ui/src/ui/markdown.ts:263-264`
  - body: Current main replaced the marked renderer with `md.renderer.rules.fence` for ReDoS hardening, highlighting, i18n copy labels, task lists, and sanitizer behavior. This PR still adds Mermaid output to the old `htmlEscapeRenderer.code` path, so carrying it forward during conflict resolution would drop current main behavior.
  - confidence: 0.93
- **[P2] Render Mermaid placeholders only where a renderer will run:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared by non-chat markdown views, but this PR globally emits empty `.mermaid-block__render` placeholders while only chat lifecycle updates call `scheduleMermaidRender()`. Mermaid content opened in those shared views can become a blank panel with the source hidden behind details.
  - confidence: 0.87
- **[P2] Retry or reset failed Mermaid blocks:** `ui/src/ui/mermaid.ts:149-150`
  - body: `renderMermaidBlocks()` only selects `.mermaid-block:not([data-mermaid-status])`, but the import and render failure paths mark current blocks as `error`. A transient chunk/import/render failure is therefore sticky for the same DOM until a full rebuild or reload, even though `loadMermaidApi()` is designed to retry.
  - confidence: 0.86
- **[P2] Run Mermaid bindFunctions after injecting the SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13.0 documents that `render()` returns optional `bindFunctions` to run after the SVG is inserted. This code only consumes `svg`, so diagrams with Mermaid-managed interactions, click handlers, or tooltips render as static output without their expected bindings.
  - confidence: 0.84
- **[P2] Avoid a root lodash-es override for a UI-only dependency:** `package.json:1253`
  - body: The new root `pnpm.overrides` entry forces `lodash-es` 4.18.1 across the whole workspace, not only Mermaid's UI dependency tree. Dependency overrides need explicit approval and can change non-UI package resolution, so this should be removed or deliberately accepted by maintainers.
  - confidence: 0.8
