## Summary

Keep open: this is a plausible Control UI feature that is not implemented on current main, but the current PR is not merge-ready because it is based on the old marked renderer, introduces a Mermaid/SVG dependency boundary, and has source-reproducible lifecycle/rendering bugs.

The PR adds lazy-loaded Mermaid rendering for Control UI WebChat markdown fences, Mermaid-specific chat styling/tests, a Mermaid dependency, lockfile updates, and a root lodash-es override.

## Best Possible Solution

Land a focused current-main implementation in the markdown-it fence renderer or a scoped post-render pass after maintainers accept the Mermaid/SVG dependency boundary, preserving sanitizer, task-list, shared markdown, lifecycle, and test contracts.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The merge blockers are source-reproducible from the PR head, current main's markdown-it renderer, shared markdown consumers, and Mermaid 11.13.0's render contract.

## Solution Assessment

No. Mermaid rendering may be a valid feature direction, but this branch is not the best merge shape until it is ported to current main and the SVG/dependency boundary is explicitly accepted.
