## Summary

Keep open: current main still lacks WebChat Mermaid rendering and this is the canonical open implementation, but the branch is conflicting and has source-reproducible renderer, lifecycle, and security blockers before it can merge.

## What This Changes

The PR adds lazy-loaded Mermaid rendering for Control UI WebChat markdown fences, Mermaid styles/tests, a Mermaid dependency, and package/lockfile updates.

## Best Possible Solution

Land a focused current-main Mermaid implementation in the markdown-it fence renderer or a clearly scoped browser render pass, preserving sanitizer, task-list, code-block, i18n, cache, and shared-markdown contracts.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The review blockers are source-reproducible from current main and the PR head: stale renderer path, incomplete render scheduling, non-retrying error state, and missing Mermaid bindFunctions handling.

## Solution Assessment

No. Mermaid rendering is a reasonable feature direction, but this branch is not the best merge shape until it is rebased or replaced on the current markdown-it renderer and the SVG/dependency boundary is accepted by maintainers.

  - body: Current main renders shared UI markdown through markdown-it, but this branch adds Mermaid handling to the older marked renderer path. Rebase the feature into `md.renderer.rules.fence` or a scoped adjacent pass so parser, sanitizer, task-list, cache, and i18n behavior stay intact.

  - confidence: 0.9
- **[P2] Schedule Mermaid rendering after manual refreshes:** `ui/src/ui/app-lifecycle.ts:93-94`
  - body: `handleUpdated()` returns while `chatManualRefreshInFlight` is true, so a manual refresh that replaces transcript HTML can insert Mermaid blocks without reaching the scheduler. Those diagrams stay blank until an unrelated chat update or tab switch occurs.
  - confidence: 0.87

  - body: The renderer only selects blocks without `data-mermaid-status`, then marks transient import, parse, streaming, or render failures as `error`. A partial streamed diagram or temporary chunk failure can permanently skip the same DOM node.
  - confidence: 0.89
- **[P2] Call Mermaid bindFunctions after SVG insertion:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid render results can include `bindFunctions` that callers must run after inserting the SVG. This branch only uses `svg`, so diagrams with Mermaid-provided interactions silently degrade.
  - confidence: 0.84
