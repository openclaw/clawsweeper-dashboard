Full review comments:

- **[P1] Rebuild Mermaid handling on the markdown-it renderer:** `ui/src/ui/markdown.ts:260`
  - body: This hunk adds Mermaid support to the old `htmlEscapeRenderer.code` path, but current main renders fences through `md.renderer.rules.fence`. Since GitHub reports the PR as conflicting, resolving this without moving the feature to the current markdown-it renderer can leave Mermaid rendering absent or broken.
  - confidence: 0.92
- **[P2] Scope Mermaid placeholders to views that render them:** `ui/src/ui/markdown.ts:260`
  - body: `toSanitizedMarkdownHtml()` is shared by sidebar, file-preview, reasoning, and other markdown views, but the PR only schedules `scheduleMermaidRender()` from the chat lifecycle. Those other consumers would receive an empty `.mermaid-block__render` with the source collapsed instead of either a rendered diagram or a normal code fence.
  - confidence: 0.87
- **[P2] Call Mermaid bindFunctions after inserting SVG:** `ui/src/ui/mermaid.ts:182-183`
  - body: Mermaid's `render()` result can include `bindFunctions` that must run after the SVG is inserted for diagram event handlers and interactions to work. The PR narrows the API to `{ svg }` and never calls the returned binder, so interactive Mermaid diagrams silently degrade.
  - confidence: 0.86
- **[P2] Show a visible fallback for empty Mermaid fences:** `ui/src/ui/mermaid.ts:174-175`
  - body: When the Mermaid source is empty, this branch marks the block as `error` and continues without calling `setMermaidRenderError()`. Users then see a blank placeholder with the source collapsed, which is worse than the current plain code-fence behavior for malformed or partial output.
  - confidence: 0.82
- **[P1] Remove the repo-wide lodash-es override:** `package.json:1250`
  - body: The PR adds a root `lodash-es` override for a Control UI feature, changing workspace-wide dependency resolution. OpenClaw treats dependency overrides as security and compatibility surface, so this should be removed or explicitly approved with lockfile and audit rationale before merge.
  - confidence: 0.88
