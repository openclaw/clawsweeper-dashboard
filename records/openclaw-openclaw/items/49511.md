  - body: Current main has replaced this marked renderer with markdown-it in `md.renderer.rules.fence`; adding Mermaid handling here cannot land cleanly and would miss the current renderer's safety and compatibility behavior. Rebuild the feature on the current fence renderer instead.
  - confidence: 0.94
- **[P1] Remove the root lodash-es override from the UI feature:** `package.json:1253`
  - body: This UI feature pins a transitive dependency for the whole workspace. Root overrides change package resolution outside Control UI and are a supply-chain surface, so this should be removed or carry explicit maintainer approval and proof.
  - confidence: 0.86
- **[P2] Run Mermaid bindFunctions after inserting the SVG:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid documents `render()` as returning `svg` and `bindFunctions`, with callers expected to run `bindFunctions?.(element)` after insertion. Dropping it means diagrams with tooltips or click bindings render as inert SVGs.
  - confidence: 0.88
- **[P2] Let transient Mermaid failures render again later:** `ui/src/ui/mermaid.ts:195`
  - body: A streamed or transiently invalid diagram is marked `data-mermaid-status="error"`, and the selector only visits blocks without any status. Once the message completes or the import recovers, that same block is skipped until the DOM is rebuilt.
  - confidence: 0.84
- **[P3] Show fallback text for empty Mermaid definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When Mermaid source is empty, this branch marks the block as error and continues without calling the error helper. Users see a blank render area while the source stays collapsed, which is worse than the current plain code fence.
  - confidence: 0.78
