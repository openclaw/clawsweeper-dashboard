Full review comments:

- **[P1] Port Mermaid fences to the current markdown-it renderer:** `ui/src/ui/markdown.ts:263-264`
  - body: Current main renders fences through `md.renderer.rules.fence`, but this branch still edits the old `marked` renderer and GitHub reports it as conflicting. Rebuild the Mermaid branch on the current renderer so it preserves the markdown-it sanitizer, task-list, code-block, and streaming behavior.
  - confidence: 0.93
- **[P1] Remove the workspace-wide lodash-es override:** `package.json:1253`
  - body: A Control UI Mermaid feature should not add a root `pnpm.overrides` pin for `lodash-es` without explicit maintainer supply-chain approval. This override changes dependency resolution across the whole workspace, so it should be removed or separately justified before merge.
  - confidence: 0.88
- **[P2] Call Mermaid bindFunctions after SVG insertion:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13.0 documents an optional `bindFunctions` return from `render()` that must be called after inserting the SVG. This code discards it, so diagrams that rely on Mermaid event bindings or tooltips will render as static SVGs.
  - confidence: 0.86
- **[P2] Do not leave non-chat Mermaid previews blank:** `ui/src/ui/markdown.ts:263-264`
  - body: The shared markdown renderer is used outside chat, but this branch converts every Mermaid fence into an empty deferred render block while only chat lifecycle updates schedule rendering. Non-chat markdown consumers need either normal code fences or their own Mermaid render pass.
  - confidence: 0.82
- **[P2] Allow failed Mermaid blocks to retry after streaming completes:** `ui/src/ui/mermaid.ts:150-151`
  - body: `renderMermaidBlocks()` only selects blocks without `data-mermaid-status`, so an incomplete streamed diagram that fails once is marked `error` and skipped on later chat updates. Transient render failures need a retry path or a status model that distinguishes final failures from incomplete streams.
  - confidence: 0.8
