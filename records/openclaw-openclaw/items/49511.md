- **[P1] Port Mermaid fences to the current markdown-it renderer:** `ui/src/ui/markdown.ts:263-264`
  - body: Current main no longer uses this `marked` renderer path; the shipped renderer is `md.renderer.rules.fence` in `ui/src/ui/markdown.ts`.   - confidence: 0.95
- **[P1] Remove the root lodash-es override:** `package.json:1253`
  - body: A Control UI Mermaid feature should not add a workspace-wide `pnpm.overrides` entry without explicit maintainer supply-chain approval. This changes dependency resolution outside the UI package and can affect unrelated packages during install or audit.
  - confidence: 0.9
- **[P2] Call Mermaid bindFunctions after SVG insertion:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid's 11.13.0 render contract can return `bindFunctions` that must run after the SVG is inserted for diagram event listeners. This code only uses `svg`, so interactive Mermaid diagrams silently lose their interactions.
  - confidence: 0.88
- **[P2] Render or preserve Mermaid fences outside chat:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared by markdown sidebars, file previews, skills, cron, and side-result views, but this PR converts every Mermaid fence into an empty render placeholder while only chat lifecycle calls `scheduleMermaidRender()`. Non-chat previews can show a blank box with the source collapsed.
  - confidence: 0.86
- **[P2] Allow transient Mermaid failures to retry:** `ui/src/ui/mermaid.ts:150`
  - body: The renderer only queries blocks without `data-mermaid-status`, then marks module-load and render failures as `error`. A transient import failure or first-pass render failure leaves the same DOM block permanently skipped until the message is recreated.
  - confidence: 0.78
