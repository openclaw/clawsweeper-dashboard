## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Port Mermaid fences to the current markdown-it renderer:** `ui/src/ui/markdown.ts:263-264`
  - body: Current main no longer uses this marked renderer path; fenced code blocks are rendered by `md.renderer.rules.fence` in `ui/src/ui/markdown.ts`. Landing Mermaid here would need a current-main port that preserves the existing code-block, task-list, sanitizer, i18n, and streaming markdown behavior.
  - confidence: 0.95
- **[P1] Remove the root lodash-es override:** `package.json:1253`
  - body: A Control UI Mermaid feature should not add a workspace-wide `pnpm.overrides` pin without explicit maintainer supply-chain approval. This changes dependency resolution outside the UI package and can affect unrelated workspaces during install or audit.
  - confidence: 0.91
- **[P2] Call Mermaid bindFunctions after SVG insertion:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13.0 can return `bindFunctions` from `render()`, and its own type/docs say to call it after inserting the SVG so diagram event listeners are attached. This code discards it, so interactive Mermaid diagrams silently become static.
  - confidence: 0.9
- **[P2] Render or preserve Mermaid fences outside chat:** `ui/src/ui/app-lifecycle.ts:124`
  - body: `toSanitizedMarkdownHtml()` is shared by sidebar, skill card, and side-result markdown views, but this PR only schedules Mermaid rendering from chat lifecycle updates. Those non-chat consumers would receive an empty render placeholder with the source hidden unless they also render Mermaid or keep a normal code fence.
  - confidence: 0.86
- **[P2] Allow transient Mermaid failures to rerender:** `ui/src/ui/mermaid.ts:149-150`
  - body: The renderer only selects `.mermaid-block:not([data-mermaid-status])` and marks failed blocks as `error`. A transient import/render failure or incomplete streamed definition can therefore leave the same diagram permanently skipped until the DOM is rebuilt.
  - confidence: 0.84
