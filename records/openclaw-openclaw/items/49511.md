## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Allow failed Mermaid blocks to render again:** `ui/src/ui/mermaid.ts:149-151`
  - body: This selector permanently excludes any block once a transient import/render failure or incomplete streamed diagram sets `data-mermaid-status="error"`, so later chat updates cannot render the completed diagram until the DOM is rebuilt.
  - confidence: 0.88
- **[P2] Scope Mermaid rewriting to rendered views:** `ui/src/ui/markdown.ts:263-264`
  - body: `toSanitizedMarkdownHtml()` is shared outside chat, but this branch converts every Mermaid fence into an empty render slot while only chat lifecycle updates call the Mermaid scheduler; sidebar and other markdown views can show blank boxes instead of code or diagrams.
  - confidence: 0.86
- **[P2] Call Mermaid bindFunctions after insertion:** `ui/src/ui/mermaid.ts:186-187`
  - body: Mermaid 11.13 can return `bindFunctions` that must run after the SVG is inserted; dropping it makes diagrams with Mermaid click handlers or tooltips render as static broken output.
  - confidence: 0.82
- **[P3] Show fallback text for empty definitions:** `ui/src/ui/mermaid.ts:178-180`
  - body: When the Mermaid definition is empty, this path marks the block as errored but leaves the render target blank, which regresses from ordinary code-fence rendering for malformed or partial output.
  - confidence: 0.78
