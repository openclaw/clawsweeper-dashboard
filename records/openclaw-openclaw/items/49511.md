## Summary

Keep open for maintainer/security review: the feature is valid and current main still lacks built-in WebChat Mermaid rendering, but this branch is stale against the current markdown-it renderer and still has source-reproducible lifecycle, retry, and SVG-boundary blockers.

## What This Changes

The branch adds lazy-loaded Mermaid rendering for Control UI WebChat markdown fences, related chat styling/tests, a Mermaid dependency, lockfile updates, and a root lodash-es override.

## Best Possible Solution

Land a focused current-main Mermaid implementation in the markdown-it fence renderer or a scoped post-render pass after maintainers accept the Mermaid/SVG boundary, preserving sanitizer, task-list, shared markdown, lifecycle, and test contracts.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The merge blockers are source-reproducible from the PR head, current main's markdown-it renderer, shared markdown consumers, and Mermaid 11.13.0's render contract.

## Solution Assessment

No. Mermaid rendering is a reasonable feature direction, but this branch is not the best merge shape until it is ported to current main and the SVG/dependency boundary is explicitly accepted.
