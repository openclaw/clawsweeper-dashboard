## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve request policy on Vertex video REST calls:** `extensions/google/video-generation-provider.ts:533`
  - body: The new `google-vertex` video fallback builds headers from `{}` and then calls the REST helper with only those headers. That drops `models.providers.google-vertex.request` controls such as extra headers, explicit proxy/TLS dispatcher policy, and `allowPrivateNetwork`, so enterprise Vertex video setups can fail or bypass intended transport controls. Route this path through the resolved Vertex HTTP request config, as the image path does, and pass the resulting policy into the guarded requests.
  - confidence: 0.91
- **[P1] Reuse the Vertex origin resolver for video REST:** `extensions/google/video-generation-provider.ts:68`
  - body: For default Vertex video URLs this constructs `https://${location}-aiplatform.googleapis.com`, but current main's resolver and `@google/genai` route `global` to the unprefixed host and `eu`/`us` to `aiplatform.<location>.rep.googleapis.com`. Use `resolveGoogleVertexBaseOrigin` here so video generation does not keep 404ing for those locations.
  - confidence: 0.9
