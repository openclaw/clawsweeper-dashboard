## Summary

Keep open: current main still has the reported source path where an implicit isolated cron announce route can be known unroutable before model execution, but the runner still executes the agent before dispatch returns a delivery-target error.

## What This Changes

The issue asks cron to fail or guard unroutable implicit isolated announce jobs before they repeatedly spend model tokens and fail delivery.

## Best Possible Solution

Add a narrow isolated cron runtime preflight that returns the existing delivery-target error before executeCronRun for permanently unresolved implicit announce routes, with regression coverage and no broad scheduler policy change.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: create an isolated agentTurn job with implicit announce/last and no previous route, observe resolveDeliveryTarget ok:false, and current runCronIsolatedAgentTurn still calls executeCronRun before dispatch returns the delivery-target error.

## Solution Assessment

Yes, the narrowest maintainable fix is to preflight the already-resolved delivery target before model execution for this permanent implicit-route failure. Add/update rejection and auto-disable behavior are broader policy choices and should stay separate unless maintainers request them.
