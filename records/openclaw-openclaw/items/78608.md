## Summary

Keep open: current main still has the reported source path where an implicit isolated cron announce target can be known-unroutable before the model run, yet the isolated runner proceeds to execute before surfacing a delivery-target error.

## What This Changes

The issue requests safer handling for isolated `agentTurn` cron jobs whose default announce delivery resolves to implicit `last` with no usable previous route.

## Best Possible Solution

Add a narrow runtime preflight that fails the implicit unresolved announce route before model execution, with regression coverage, while leaving broader add/update rejection or auto-disable policy to separate maintainer direction.

## Reproduction Assessment

Yes, from source inspection: default isolated announce delivery resolves implicit `last`, failed target resolution is available before execution, and current main still calls `executeCronRun` before the delivery-target failure is returned. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to preflight unresolved implicit announce delivery before `executeCronRun` for this shape. Rejecting jobs at add/update time or auto-disabling recurring jobs would be broader scheduling policy and should be separate unless maintainers request it.
