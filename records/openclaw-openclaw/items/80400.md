## What This Changes

The PR routes Telegram video media through `sendDocument` when `forceDocument` is set, skips video-dimension probing for that path, and updates tests, docs, CLI help, message-tool schema text, prompt snapshots, and changelog wording.

## Best Possible Solution

Land the focused `forceDocument` video routing fix after real Telegram proof is attached, and track automatic fallback or exit-code behavior separately if those are still reproducible.

## Reproduction Assessment

Yes for the routing bug by source inspection: current main routes `video/mp4` through `sendVideo` before the document fallback even when `forceDocument` is true. I did not live-reproduce the reporter's Telegram network failure or exit-code symptom.

## Solution Assessment

Yes for the code direction: making explicit `forceDocument` video sends bypass `sendVideo` is the narrow maintainable fix, and deferring ambiguous automatic fallback avoids duplicate-send risk. Merge should still wait for real Telegram proof because this is a transport-visible change.
