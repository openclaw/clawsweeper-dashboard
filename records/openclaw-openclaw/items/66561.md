## Summary

Keep open. Current main still lacks response-started or first-byte state in the embedded-run attempt result and failover path, so the traced post-start Codex SSE abort can still be normalized as `timeout`/408; the closed AbortError PR did not land and would not cover the full reported lifecycle gap.

## What This Changes

The issue asks OpenClaw to distinguish post-start Codex SSE aborts from provider-silent timeouts when embedded runs abort after upstream response bytes have begun.

## Best Possible Solution

Carry model-call lifecycle observations into embedded-run timeout and failover classification so post-start aborts are surfaced separately from no-response timeouts.

## Reproduction Assessment

Yes, source-level with strong external trace evidence. The issue provides correlated mitmproxy first-byte/client-close timing and OpenClaw timeout logs, and current main still lacks request-started or first-byte state at the failover boundary.

## Solution Assessment

Yes. The narrow maintainable fix is to carry model-call lifecycle observations from the diagnostic wrapper into the attempt result and failover/logging boundary; only changing AbortError message classification would miss the reported post-start abort case.
