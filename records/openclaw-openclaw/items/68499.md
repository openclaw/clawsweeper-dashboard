## Summary

Keep open: current main still has only one task failure bucket, and this PR remains the active implementation path, but the current patch has concrete P2 correctness regressions plus missing external real behavior proof.

## What This Changes

The PR adds recentFailures and historicalFailures task summary fields and updates tasks, status, flows, and targeted tests to show the split in CLI summaries.

## Best Possible Solution

Land a revised split that computes recent and historical counts from the same task records each command renders, keeps the flows table width aligned, and includes redacted real CLI proof.

## Reproduction Assessment

Yes, source-level. A cleanupAfter-expired failed task can remain in the inspected/listed records until maintenance pruning, while the PR’s summary counts use buildTaskStatusSnapshot, which filters that task out.

## Solution Assessment

No. The recent/historical split is useful, but this patch is not the best implementation until registry, tasks, and flows classify failures from the same record set they display and the flows header width is fixed.
