## Summary

Keep open: current main still has the old single failure bucket, but this PR needs revision because its new recent/historical counts can disagree with the task rows and aggregates it prints, and it still lacks required real CLI behavior proof.

## What This Changes

The PR adds recentFailures and historicalFailures task summary fields and updates tasks, status, flows, and related tests to show recent versus historical task failures.

## Best Possible Solution

Land a revised split that computes recent and historical failures from the same task record set each command renders, keeps the flows table aligned, and includes redacted real CLI proof.

## Reproduction Assessment

Yes, source-level. A cleanupAfter-expired failed task can remain in reconcileInspectableTasks/listTasksForFlowId until maintenance pruning, while the PR’s new summaries use buildTaskStatusSnapshot, which filters that task out.

## Solution Assessment

No. The recent/historical split is useful, but this patch is not the best implementation until registry, tasks, and flows classify failures from the same record set they display and the flows column width is fixed.
