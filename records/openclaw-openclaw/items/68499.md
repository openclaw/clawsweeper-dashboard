## Summary

Keep open: the PR addresses a real founder-visible summary gap, but the current patch still has concrete P2 correctness regressions where aggregate counts can disagree with the rows being rendered.

## What This Changes

The PR adds recent versus historical task failure counts to task/status/flow summaries and updates targeted task and command tests.

## Best Possible Solution

Land the split through a shared failure-recency classifier over the exact records each command renders, keep cleanup-aware filtering limited to status-card views, align the widened flow table header, and add a focused changelog entry with tests.

## Reproduction Assessment

Yes. Create a failed, timed_out, or lost task whose cleanupAfter is already in the past but still present before maintenance pruning; the PR can render the row while snapshot-derived summaries omit it from recent/historical failure pressure.

## Solution Assessment

No. The product direction is sound, but using buildTaskStatusSnapshot as the aggregate source for registry, task-list, and flow summaries is not the narrowest maintainable fix because that helper intentionally filters expired cleanup rows.

  - body: tasksListCommand renders every task in the filtered task array, but the new pressure headline comes from buildTaskStatusSnapshot, which excludes expired cleanup rows. That can show failed task rows while reporting no recent or historical failures, so the pressure line should classify the same list it prints.
  - confidence: 0.9

  - body: flows show reports recent/historical counts through taskSummary while the linked-task section still prints every listTasksForFlowId record. In the pre-prune window, visible failed linked tasks can be absent from the headline; make the summary and linked rows share the same basis.
  - confidence: 0.89

  - body: Rows now pad the task summary to TASKS_PAD, but the header still pads Tasks to 14 characters. This shifts the Goal header left of the row values in flows list; use the same width for the header as the data column.
