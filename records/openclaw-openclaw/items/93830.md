## Best Possible Solution

Land one canonical Matrix repair that forwards only explicit `isReasoning` payloads as `m.notice`, preserves raw/off-mode suppression and TTS exclusion, and includes live or maintainer-supplied Matrix proof.

## Reproduction Assessment

## Solution Assessment

Mostly yes: forwarding only explicit `isReasoning` payloads to a Matrix `m.notice` lane while keeping raw suppression is the narrow fix. The remaining question is proof and whether maintainers prefer this per-turn option or the broader channel-capability API shape.
