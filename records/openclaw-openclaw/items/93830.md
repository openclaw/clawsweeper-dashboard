## What This Changes

The PR adds a Matrix-specific reasoning opt-in to generic reply dispatch, delivers explicit reasoning payloads as Matrix `m.notice`, bypasses draft-preview editing for those blocks, and excludes reasoning from TTS.

## Best Possible Solution

Land one canonical Matrix repair that forwards only explicit `isReasoning` payloads as `m.notice`, preserves raw reasoning suppression and `/reasoning off`, excludes reasoning from TTS, and includes live Matrix proof or an explicit maintainer proof waiver.

## Reproduction Assessment

Yes by source inspection: current main persists `/reasoning on`, builds explicit `isReasoning` reply payloads, then suppresses them in generic dispatch and Matrix delivery before send. I did not run a live Matrix homeserver reproduction in this read-only review.

## Solution Assessment

Mostly yes for the code shape: forwarding only explicit `isReasoning` payloads for Matrix preserves the older raw-reasoning suppression guard and addresses the generic dispatch drop. It is not merge-ready until live Matrix proof is supplied or maintainers explicitly waive it and choose this internal option over the broader channel-capability design.
