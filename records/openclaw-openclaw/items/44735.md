## What This Changes

The PR adds a doctor plugin auto-enable guard so an enabled alternative Feishu-capable plugin prevents auto-enabling the built-in Feishu plugin, plus a regression test for openclaw-lark.

## Best Possible Solution

Rebase onto the split auto-enable modules and implement a generic effective channel-owner guard that respects allowlist, deny/disable, and `preferOver`, with focused Feishu/openclaw-lark regressions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with restrictive `plugins.allow`, `plugins.entries.openclaw-lark.enabled=true` is not enough to activate that plugin, but the PR guard still skips Feishu auto-enable.

## Solution Assessment

No. The direction is useful, but the submitted solution should check effective channel ownership rather than raw `entries.enabled`, and it needs to be adapted to current main's split auto-enable files.
