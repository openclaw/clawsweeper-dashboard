## Summary

Keep open: the Feishu/Lark conflict is a valid doctor repair bug, but this PR is stale against the current auto-enable split and its guard treats a raw enabled entry as active even when `plugins.allow` would block that plugin.

## What This Changes

The PR adds a plugin auto-enable guard and regression test so bundled `feishu` is not enabled when `openclaw-lark` is already enabled for the Feishu channel.

## Best Possible Solution

Port the alternative channel-owner check into the current shared auto-enable and doctor install-repair flows using effective activation/allowlist semantics, then add focused Feishu/Lark regression coverage and a changelog entry.

Yes. The source path is high-confidence: `channels.feishu` plus enabled `openclaw-lark` reaches channel auto-enable, while current main only avoids bundled fallback through `preferOver`; the PR's own test also captures the unrestricted case.

## Solution Assessment
