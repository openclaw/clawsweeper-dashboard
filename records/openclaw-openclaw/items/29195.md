## Summary

Keep open. Current main has documented the ACPX permission workaround and added better ACP task/result plumbing, but the central ACPX/Codex reliability gaps remain: the default still uses `approve-reads` plus `fail`, runtime startup does not preflight that noninteractive footgun, parent-to-own-child ACP visibility is still blocked by cross-agent visibility rules unless broader access is enabled, and stalled sessions still rely on long/default timeouts rather than a tight idle cleanup/reporting policy.

## What This Changes

Request to make ACPX/Codex noninteractive sessions reliable by surfacing permission/session failures, exposing parent-owned child progress/status, timing out stalled sessions, documenting permission configuration, and adding spawn-time health checks.

## Best Possible Solution

Keep this as the ACPX/Codex reliability umbrella unless maintainers split it into narrower issues. The desired end state is a maintainer-approved noninteractive ACPX permission profile or spawn preflight with actionable remediation, parent-owned ACP child status/history that does not require global visibility, and explicit stalled-session timeout/cleanup/reporting for ACP background tasks.
