## Summary

Keep open. Current main has meaningful mitigations for ACP/Codex failures, but the issue is not fully resolved: ACPX still defaults to restrictive non-interactive permissions, parent agents can still hit cross-agent gates before spawned-child visibility is considered, and the remaining default/preflight direction is security-sensitive.

## What This Changes

The issue asks OpenClaw to make non-interactive Codex ACP sessions reliable by surfacing failures, exposing child progress/status, cleaning up stalled sessions, documenting or preflighting permissions, and detecting incomplete work.

## Best Possible Solution

Keep the current docs/runtime/task-delivery mitigations, resolve the narrow parent-owned child visibility bug under #54878, and make an explicit maintainer decision on ACPX permission defaults or preflight warnings rather than silently escalating defaults.

## Reproduction Assessment

Yes for the remaining visibility portion: source inspection shows cross-agent checks run before spawned-child tree membership, matching #54878. I did not live-run ACP/Codex, and the permission/stall portions are partly mitigated by docs, task delivery, and turn timeout handling.

## Solution Assessment

No. Closing this broad issue now would hide remaining access-policy and permission-default decisions; the maintainable path is to keep it open or split it into the canonical visibility bug plus a maintainer-approved permission preflight/default policy item.
