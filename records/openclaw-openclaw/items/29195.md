## Summary

Keep open. Current main has useful mitigations and docs, but the issue is still not fully resolved: ACPX still defaults to restrictive non-interactive permissions, parent-owned cross-agent ACP children can still be denied by the shared visibility guard, and the remaining approve-all/preflight direction is security-sensitive.

## What This Changes

The issue asks OpenClaw to make ACP/Codex spawned sessions fail visibly, expose child-session progress to the spawning agent, clean up stalls, document ACPX permissions, and optionally preflight unsafe defaults.

## Best Possible Solution

Keep the current docs and runtime/task-delivery mitigations, land narrow fixes under #54878 and #44810, and make an explicit maintainer decision before adding an ACPX approve-all default or preflight remediation.

## Reproduction Assessment

Yes for the remaining source-backed failures: current main still checks cross-agent access before spawned-child lineage, and the ACP parent stream still emits the generic no-output interactive-wait notice. I did not live-run the original macOS Codex ACP setup.

## Solution Assessment

No for the approve-all default as stated: approve-all is manifest-declared dangerous, so the safer maintainable path is narrow lineage access, better no-output diagnostics, and maintainer-approved preflight/default policy.
