## Summary

Keep open. Current main has addressed the documentation and some status/error/timeout plumbing, but the core ACPX/Codex reliability umbrella is not fully resolved: the default permission mode still permits non-interactive write/exec failures, parent-owned cross-agent child visibility is still separately tracked, and the requested safer default/preflight needs maintainer security/product judgment.

## What This Changes

The issue requests ACPX/Codex reliability improvements for non-interactive permission failures, child session monitoring, stalled-session cleanup, completion/error reporting, and permission documentation.

## Best Possible Solution

Keep this as the canonical ACPX/Codex reliability umbrella until maintainers choose a safe permission profile or preflight, land scoped parent-owned child visibility, and replace manual stalled-process cleanup with automatic timeout/reporting behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the permission-default and child-visibility classes from current source/docs; unclear for the full 80-minute Codex stall because this review did not run live ACP/Codex processes.

## Solution Assessment

Is this the best way to solve the issue? No single default flip is the best fix: `approve-all` is security-sensitive, so the maintainable path is a maintainer-approved preflight/profile plus scoped visibility and cleanup/reporting changes.
