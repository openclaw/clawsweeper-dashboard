## Summary

Keep open. Current main has important ACP/Codex mitigations, including permission docs, task terminal notifications, and turn timeout/error handling, but it has not fully resolved the request: ACPX still defaults to `approve-reads`/`fail`, parent agents still hit cross-agent visibility gates for spawned ACP children, and the remaining default/preflight/build-validation direction needs maintainer product and security judgment.

## What This Changes

The issue asks for ACP/Codex sessions to fail visibly, be monitorable by spawning agents, clean up/report stalls, and document non-interactive permission configuration.

## Best Possible Solution

Keep the existing runtime/docs mitigations, land or resolve the narrow parent-owned ACP visibility work under #54878, and make an explicit maintainer decision on permission preflight/defaults instead of silently switching all ACP sessions to `approve-all`.

## Reproduction Assessment

Yes for the remaining visibility portion: source inspection shows cross-agent checks run before spawned-child tree visibility, matching the #54878 reproduction. I did not live-run ACP/Codex, and the permission/stall portions are now partly mitigated by docs, task delivery, and timeout handling.

## Solution Assessment

No. Closing this broad issue now would hide remaining access-policy and permission-default decisions; the safer path is to keep it open or explicitly split the remaining work into the canonical visibility issue plus a maintainer-approved preflight/default policy change.
