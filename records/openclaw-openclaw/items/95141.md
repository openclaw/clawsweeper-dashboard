## Summary

Keep open. Current main still fails the WebChat non-deliverable recovery-notice path, and the direct fix is an open, unmerged pull request rather than implemented code.

## What This Changes

The issue asks OpenClaw to stop silently losing an in-flight WebChat turn when a config-triggered gateway restart interrupts it, either by deferring restart, persisting/resuming the request, or showing a recovery notice.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/95154 as the scoped WebChat notice fix or choose a broader recovery design that also preserves/resumes interrupted WebChat turns.

## Reproduction Assessment

No live high-confidence current-main reproduction was run. Source inspection gives a medium-confidence path: WebChat is non-deliverable, so the unresumable recovery branch can mark the session failed without sending or persisting a visible notice.

## Solution Assessment

Unclear. The open PR is a narrow maintainable notice fix, but maintainers should decide whether the best final behavior also needs restart deferral or immediate user-message persistence.
