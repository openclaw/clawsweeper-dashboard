## Summary

Keep open. Current main and the latest release still have the WebChat non-deliverable restart-recovery notice gap, and the open linked PR is a candidate fix rather than merged implementation evidence.

## What This Changes

The issue reports that a config-triggered gateway restart can silently lose an in-flight WebChat dashboard turn, leave the session failed with an empty transcript, and show no recovery notice.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/95154 as the scoped WebChat notice fix or choose a broader recovery design that preserves or resumes the interrupted WebChat turn.

## Reproduction Assessment

Yes at source level, but not as a live current-main run: WebChat is non-deliverable, and the non-resumable recovery branch can mark the session failed without a WebChat-visible notice. The reporter's v2026.6.8 logs provide concrete live context for the same path.

## Solution Assessment

Unclear. The linked PR is a narrow maintainable fix for the missing WebChat notice, but maintainers should decide whether the best final behavior also needs restart deferral or immediate user-message persistence.
