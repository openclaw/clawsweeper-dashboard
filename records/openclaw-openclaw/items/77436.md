## Summary

Keep open: current main still shows the Docker host-path leak, and open PR #77446 explicitly targets and closes this regression, so this issue should wait for that PR to merge or be rejected.

## What This Changes

Fresh Docker setup on macOS can persist host `/Users/...` config/workspace paths into container runtime state, causing first agent replies to fail with `EACCES` after 2026.04.29.

## Best Possible Solution

Land or revise #77446 so Compose keeps host path variables only for bind mounts while exposing `/home/node/.openclaw` and `/home/node/.openclaw/workspace` inside gateway and CLI containers.

## Reproduction Assessment

Yes. By source inspection, Docker setup can export macOS host paths, Compose passes them into the containers, onboarding can persist them, and the first reply path attempts to mkdir the configured `/Users/...` workspace.

## Solution Assessment

Yes. The linked PR's container-side env override is the narrowest maintainable direction; closing this issue before that PR merges would be premature.
