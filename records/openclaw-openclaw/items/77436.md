## Summary

Keep this open: current main still has a source-reproducible Docker setup regression, and open PR #77446 already uses closing syntax for this issue, so the issue should stay open until that PR lands or is rejected.

## What This Changes

Fix Docker setup after 2026.04.29 so fresh container onboarding does not persist macOS host paths that later fail inside Linux Docker.

## Best Possible Solution

Review and land PR #77446, or a narrow replacement, so host bind-mount variables remain host paths while container runtime config and workspace resolution use /home/node/.openclaw paths.

## Reproduction Assessment

Yes by source inspection: setup writes host HOME-derived paths into .env, Compose imports that file into the containers, onboarding can persist the workspace, and the reply path mkdirs the configured path. I did not run a live Docker/Ollama Cloud repro in this read-only review.

## Solution Assessment

Yes: the narrow fix belongs at the Docker setup/Compose boundary by separating host mount sources from container-visible runtime paths. Provider-specific changes would not address the mkdir '/Users' failure source.
