## Summary

Keep open. Current main still awaits the Gateway-location select before it evaluates section-filtered configure work, so `openclaw configure --section model` can still block before Anthropic model/auth setup. The direct `openclaw models auth ...` commands are useful workarounds from the discussion, but they do not fix the configure wizard path.

## What This Changes

The issue asks for the configure wizard to stop hanging at Gateway selection so Anthropic auth can proceed, or for a direct bypass/manual auth path to be available.

## Best Possible Solution

Update `runConfigureWizard` so section-filtered model/auth setup can reach `promptAuthConfig` without first requiring Gateway-location selection when the requested sections do not need gateway, daemon, channels, or health work. Add a clear non-interactive TTY guard for interactive configure paths and keep direct Anthropic `models auth` commands documented as workarounds.

## Repair Work Prompt

Fix the configure wizard hang reported for section-filtered model/auth setup. On current main, `runConfigureWizard` asks `Where will the Gateway run?` before it handles `opts.sections`, so `openclaw configure --section model` can block before Anthropic auth setup. Update the wizard so section-filtered model/auth configuration can call `promptAuthConfig` without first requiring the gateway-location select when the requested sections do not need gateway, daemon, channels, or health work. Preserve existing behavior for full `openclaw configure` and for sections that genuinely need gateway mode. Add an early actionable non-interactive TTY guard for interactive configure paths that points Anthropic users to `openclaw models auth setup-token --provider anthropic` and `openclaw models auth paste-token --provider anthropic`. Add regression tests in `src/commands/configure.wizard.test.ts` proving `sections: ["model"]` reaches `promptAuthConfig` without consuming a gateway select, and keep or update tests for gateway/channel/daemon/health paths that should still request gateway mode. Consider related https://github.com/openclaw/openclaw/issues/57570 only insofar as the same configure/config interactive boundary is affected; do not turn this into a broad config preaction refactor. Update `docs/cli/configure.md` if behavior changes, keep direct `models auth` commands documented as workarounds, and add a user-facing changelog fix entry crediting `Thanks @LevityLeads`. Do not change provider auth semantics, token storage, or unrelated model selection behavior.
