## Summary

Keep open: current main still asks the Gateway run-mode question before honoring the model-only configure section, so the reported auth path can still be blocked despite the documented `models auth` workaround.

## What This Changes

The issue asks for `openclaw configure` and `openclaw configure --section model` to proceed past Gateway selection so Anthropic auth setup can continue.

## Best Possible Solution

Move section-aware model/auth runs ahead of Gateway run-mode selection, keep Gateway setup for full and gateway-related sections, and add regression coverage for both `configure` and `config` aliases.

## Reproduction Assessment

Yes. Source inspection shows `openclaw configure --section model` and the `openclaw config --section model` alias enter `runConfigureWizard`, which awaits Gateway run-mode selection before `promptAuthConfig`; no live Docker TTY run was performed, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes. The narrow maintainable fix is to honor section filters before the Gateway run-mode prompt and cover that ordering with tests; the `models auth` workaround is useful but does not fix the documented configure path.
