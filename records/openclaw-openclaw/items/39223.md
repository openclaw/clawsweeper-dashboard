## Summary

Keep open: current `main` and `v2026.5.12` still await the Gateway run-mode prompt before section-filtered model auth can run, so the documented `configure --section model` path remains blocked if that prompt is unresponsive. The separate `models auth setup-token` and `paste-token` commands are useful workarounds, but they do not fix the configure wizard contract.

## What This Changes

The issue requests that `openclaw configure --section model` avoid hanging at the Gateway selection step so users can reach Anthropic auth setup.

## Best Possible Solution

Honor section filters before the Gateway run-mode prompt: model-only configure should go directly to `promptAuthConfig`, while full wizard and gateway-related sections retain Gateway setup.

## Reproduction Assessment

Yes, by source inspection: `configure --section model` and `config --section model` both enter `runConfigureWizard`, which awaits the Gateway selection before `promptAuthConfig`. The Docker keypress hang itself was not live-run in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to skip or defer the Gateway run-mode prompt for section-filtered flows that do not request Gateway-related work, rather than adding a new bypass flag or relying on separate auth commands.
