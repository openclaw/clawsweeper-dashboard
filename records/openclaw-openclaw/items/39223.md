## Summary

Keep open: current main still asks the Gateway run-mode question before honoring the model-only configure section, so the documented model/auth wizard path can still be blocked despite the separate `models auth` workaround.

## What This Changes

The issue reports the configure wizard becoming unresponsive at the Gateway run-mode prompt before reaching the model/auth setup flow.

## Best Possible Solution

Honor section filters before Gateway run-mode selection, keep Gateway setup for full and gateway-related sections, and add regression coverage for both `configure` and `config` aliases.

## Reproduction Assessment

Yes. Source inspection shows `openclaw configure --section model` and `openclaw config --section model` enter `runConfigureWizard`, which awaits Gateway run-mode selection before `promptAuthConfig`; no live Docker TTY run was performed.

## Solution Assessment

Yes. The narrow maintainable fix is to honor model-only section filters before the Gateway run-mode prompt; the documented `models auth` workaround is useful but does not repair the guided configure path.
