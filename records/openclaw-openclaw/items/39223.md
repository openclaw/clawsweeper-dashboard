## Summary

Keep open. Current main still enters the gateway run-mode prompt before section-specific handling, and neither `configure --section model` nor the `config` alias has a bypass that reaches model auth first; the documented `models auth` commands are useful workarounds, not a fix for the reported configure path.

## What This Changes

The issue asks for the configure wizard to stop hanging at the Gateway selection step so Anthropic model/auth setup can proceed, or to provide a direct auth bypass/documented flow.

## Best Possible Solution

Make section planning decide whether gateway run-mode selection is needed. A model/auth-only configure run should reach `promptAuthConfig` without gateway probing or run-mode selection, while full configure and gateway/daemon/health/channel paths should keep the existing gateway setup behavior and get an early actionable error when no interactive TTY is available.

## Reproduction Assessment

Yes. The source-level reproduction is `openclaw configure --section model` or `openclaw config --section model`: both route into `runConfigureWizard`, which awaits the gateway run-mode `select` before it calls `promptAuthConfig`.

## Solution Assessment

No. The current direct `models auth` commands are a reasonable workaround, but the maintainable fix is to repair the configure wizard path itself and cover the ordering with focused tests.
