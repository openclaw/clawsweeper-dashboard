## Summary

Keep open: current main and the latest release still ask the Gateway run-mode question before the model-only configure path reaches provider auth, so the documented guided auth path can still be blocked despite the separate `models auth` workaround.

## What This Changes

The issue reports that the interactive configure wizard can hang at Gateway run-mode selection and block the model authentication flow.

## Best Possible Solution

Make section-filtered model/configure flows reach `promptAuthConfig` without first awaiting Gateway run-mode selection, while preserving Gateway setup in the full wizard and gateway-related sections.

## Reproduction Assessment

Yes for the central blocking path by source inspection: `configure --section model` and `config --section model` enter `runConfigureWizard`, which awaits the Gateway selection before `promptAuthConfig`. The exact Docker TTY keypress hang was not live-run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to honor section filters before the Gateway run-mode prompt for model-only setup; the `models auth` workaround is useful but does not fix the guided configure contract.
