## Summary

Keep open: current main still routes model-only configure runs through gateway run-mode selection before model auth, so the workaround commands discussed on the issue do not fix the reported configure path.

## What This Changes

The issue asks for `openclaw configure` and `openclaw config --section model` to stop blocking at gateway selection and let Anthropic auth setup proceed.

## Best Possible Solution

Plan configure sections before gateway run-mode selection so model/auth-only runs reach `promptAuthConfig` directly, while full configure and gateway/daemon/health paths keep gateway setup and get an actionable non-TTY failure.

## Reproduction Assessment

Yes. The source-level path is high-confidence: `openclaw configure --section model` and `openclaw config --section model` both enter `runConfigureWizard`, which awaits the gateway run-mode `select` before the model branch calls `promptAuthConfig`.

## Solution Assessment

Yes. The narrow maintainable fix is to repair configure wizard ordering and add regression coverage; relying on `models auth` commands is a useful workaround but leaves the documented configure path broken.
