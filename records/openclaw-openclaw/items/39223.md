## Summary

Keep open: current main still routes model-only configure through the Gateway run-mode prompt before model auth, so the workaround discussed on the issue does not repair the reported configure path.

## What This Changes

The issue reports `openclaw configure --section model` hanging at the Gateway selection prompt before Anthropic auth setup can proceed.

## Best Possible Solution

Plan selected configure sections before Gateway run-mode selection so model/auth-only runs reach `promptAuthConfig` directly, while full configure and gateway/daemon/health paths keep Gateway setup behavior and actionable non-TTY failure handling.

## Reproduction Assessment

Yes, by source inspection. `openclaw configure --section model` and `openclaw config --section model` both forward sections into `runConfigureWizard`, which awaits the Gateway run-mode select before calling `promptAuthConfig`.

## Solution Assessment

Yes. The narrow maintainable fix is to repair configure wizard ordering and add regression coverage; `models auth` commands are a useful workaround but leave the documented configure path broken.

## Repair Work Prompt

Fix the current `openclaw configure --section model` and `openclaw config --section model` path so it reaches `promptAuthConfig` without first requiring the Gateway run-mode select. Keep full `openclaw configure` behavior and keep Gateway run-mode selection for gateway, daemon, health, and any section combination that actually needs Gateway mode. Prefer deciding whether selected sections require Gateway setup before awaiting the run-mode prompt. Do not add a new CLI flag, do not reintroduce Anthropic browser OAuth, and do not change provider auth semantics. Add regression coverage in `src/commands/configure.wizard.test.ts` proving `sections: ["model"]` calls `promptAuthConfig` and does not call the top-level Gateway run-mode `select`. Update `docs/cli/configure.md`, `docs/cli/config.md`, and `CHANGELOG.md` if section semantics or user-facing behavior changes, crediting @LevityLeads for the bug report if adding a changelog entry.
