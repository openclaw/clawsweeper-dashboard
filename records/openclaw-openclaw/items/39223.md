## Summary

Keep open: current main still asks the Gateway run-mode question before honoring the model-only configure section, so the documented model/auth wizard path can still be blocked despite the separate `models auth` workaround.

## What This Changes

The issue reports that `openclaw configure` and `openclaw configure --section model` block at Gateway selection before reaching Anthropic auth.

## Best Possible Solution

Honor section filters before Gateway run-mode selection, keep Gateway setup for full and gateway-related flows, and add regression coverage for both `configure` and `config` aliases.

## Reproduction Assessment

Yes for the central blocking path: source inspection shows `openclaw configure --section model` and `openclaw config --section model` enter `runConfigureWizard`, which awaits Gateway run-mode selection before `promptAuthConfig`. No live Docker TTY run was performed, so the exact keypress hang remains unverified.

## Solution Assessment

Yes. The narrow maintainable fix is to make model-only section runs reach auth setup without first consuming the Gateway run-mode prompt; the documented `models auth` workaround is useful but does not repair the guided configure path.
