## What This Changes

The issue asks OpenClaw ACPX to stop forwarding unsupported timeout config options to non-Codex ACP agents during spawned ACP sessions.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81006; the final fix should drop `timeout` / `timeout_seconds` before the ACP wire for all backends while preserving valid model and reasoning controls.

Yes. Source inspection shows current main still maps timeout runtime options into `setConfigOption` and forwards them for non-Codex ACP commands, while the Claude ACP dependency rejects unknown config IDs.
