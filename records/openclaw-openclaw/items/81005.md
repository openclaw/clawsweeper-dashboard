## Summary

Keep open: current main still forwards `timeout` for non-Codex ACPX sessions, the latest shipped release matches the reported failing path, and the open linked fix PR should close this issue only after it lands.

## What This Changes

The issue reports that ACPX sends the runtime-only `timeout` config option to non-Codex ACP agents, causing Claude ACP spawns to fail with `ACP_TURN_FAILED`.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81006; the final fix should drop `timeout` and `timeout_seconds` before the ACP wire for all backends while preserving valid model and reasoning controls.

## Reproduction Assessment

Yes. Source inspection shows current main still maps `timeoutSeconds` into `setConfigOption` and forwards it for non-Codex ACPX commands, while the Claude ACP dependency rejects unknown config IDs; I did not run the live Claude setup myself.

## Solution Assessment

Yes. The linked PR's stated approach, hoisting the existing timeout skip and flipping the regression test, is the narrow maintainable fix; the separate configured-timeout plumbing mismatch can remain a follow-up.
