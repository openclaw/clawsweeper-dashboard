## Summary

Close as implemented on current main: the merged ACP Claude timeout fix now suppresses unsupported timeout config forwarding for Claude and filters timeout emission when advertised config keys exclude it, while preserving OpenClaw's own turn timeout.

## What This Changes

The issue reports that ACPX sends an unsupported `session/set_config_option` timeout key to Claude and other non-Codex ACP agents, causing `sessions_spawn` to fail with `ACP_TURN_FAILED`.

## Best Possible Solution

Keep the merged current-main implementation from https://github.com/openclaw/openclaw/pull/80812, let the next release carry it, and treat the still-open broader proposal as superseded by the shipped mainline fix path.

## Reproduction Assessment

Yes for the original shipped behavior: v2026.5.7 forwarded `timeout` to Claude ACP, and the dependency rejects unknown config IDs. Current main no longer has that failing Claude path by source inspection because the wrapper suppresses the key and the control plane omits timeout when advertised keys exclude it.

## Solution Assessment

Yes. The current-main fix is narrower than a blanket all-agent drop: it uses advertised config keys where available, keeps timeout forwarding for backends that explicitly advertise it, and adds a Claude-specific ACPX defense for the observed adapter.
