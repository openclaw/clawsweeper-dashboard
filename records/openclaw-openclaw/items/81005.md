## Summary

Keep open: current main still has the reported ACPX non-Codex timeout-forwarding path, and https://github.com/openclaw/openclaw/pull/81006 is an open linked fix that should resolve this issue only after it lands.

## What This Changes

The issue asks OpenClaw ACPX to stop forwarding unsupported `timeout` / `timeout_seconds` config options to non-Codex ACP agents during `sessions_spawn`.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81006; the final fix should drop `timeout` / `timeout_seconds` before the ACP wire for all backends while preserving valid model, mode, and reasoning controls.

## Reproduction Assessment

Yes. Source inspection shows current main maps timeout runtime options into `setConfigOption`, ACPX forwards that to `session/set_config_option`, and Claude ACP rejects unknown config IDs; I did not run the live Claude setup myself.

## Solution Assessment

Yes. The linked PR's stated approach is the narrow maintainable fix for this issue, while the separate configured-timeout plumbing mismatch can remain a follow-up.
