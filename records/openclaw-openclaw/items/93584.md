## What This Changes

The PR applies tool-use/tool-result pairing repair with aborted-turn dropping in agent compaction planning and safeguard paths, with regression tests for prune and no-drop planner cases.

## Best Possible Solution

Move the abort-drop pairing repair to the common built-in compaction LLM input boundary before `summarizeInStages()` and keep the existing prune/drop behavior as an implementation detail.

## Reproduction Assessment

## Solution Assessment

No; this is a plausible partial fix, but the best fix is to sanitize the common built-in LLM compaction input before `summarizeInStages()` rather than only selected planner outputs.
