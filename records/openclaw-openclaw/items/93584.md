## What This Changes

## Best Possible Solution

Move the transcript-pair repair to an unconditional built-in compaction LLM input boundary so every `summarizeInStages()` call, including no-prune `recentTurnsPreserve: 0`, receives repaired messages before the linked issue closes.

## Reproduction Assessment

Yes, source-reproducible: a built-in compaction run that stays under the prune gate and has `recentTurnsPreserve: 0` still flows raw messages through `splitPreservedRecentTurns()` and into `summarizeInStages()`. I did not run tests because this review is read-only.

## Solution Assessment

No, this is not yet the best fix because the repair remains on selected caller branches rather than the common built-in LLM input boundary. The safer bounded repair is to sanitize/repair once before any `summarizeInStages()` call and cover that with the no-prune regression.
