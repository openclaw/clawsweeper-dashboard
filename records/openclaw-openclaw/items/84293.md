## Summary

Keep open: current main still has source-backed JSON stdout pollution in the `nodes list --json` path, including a human summary line before the JSON payload and an internal nodes plugin-registration path that does not use the existing JSON-mode stderr guard.

## What This Changes

Ensure JSON-mode CLI commands such as `openclaw nodes list --json` produce only parseable JSON on stdout while routing plugin startup and human diagnostics to stderr.

## Best Possible Solution

Move all `nodes list --json` diagnostics and nodes plugin-registration startup output off stdout before the JSON payload, while preserving current human output for non-JSON commands and adding focused regression coverage.

## Reproduction Assessment

Yes. The reporter provided an exact v2026.5.18 command, and current source still writes non-JSON `nodes list` text before `writeJson` plus has an unguarded nodes plugin-registration path; I did not execute the Helm/container flow in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to make JSON-mode routing cover both nodes sub-CLI plugin registration and command-level human summaries, instead of patching only LCM logging or only one startup path.
