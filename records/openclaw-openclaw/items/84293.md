## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/84293. Current main already fixed plugin-provided primary registration in commit 2d6fd54ebdfe83d57cf157ba46f61867c71fe990, but `openclaw nodes list --json` still has a gap because `runCli` registers the primary/sub-CLI before the later guarded plugin-registration block, and `src/cli/nodes-cli/register.ts` registers plugin CLI commands under `nodes` during that phase. Keep the fix scoped: route plugin/startup diagnostics to stderr only during the early primary/sub-CLI registration window for real pre-terminator JSON output, restore `loggingState.forceConsoleToStderr` before parse/action execution, preserve `config set --json` parse-only behavior, and preserve pass-through `-- --json` behavior. Add focused regression tests that prove `nodes list --json`-style sub-CLI plugin registration sees stderr routing, parse sees the restored state, and pass-through JSON does not trigger routing. Add a one-line changelog fix entry because this changes user-visible CLI behavior. Do not add a new flag/config option, do not make plugins/core policy owner-specific, and do not route the JSON payload itself through `runtime.log`.
