## Summary

Keep open: current main still has a JSON-mode startup gap around the `nodes` sub-CLI plugin registration path, so the reporter's v2026.5.18 stdout pollution remains source-reproducible and needs a narrow fix with regression coverage.

## What This Changes

The issue asks for `openclaw nodes list --json` and other JSON CLI commands to keep stdout pure JSON even when plugins emit startup logs.

## Best Possible Solution

Route every JSON-mode CLI plugin/subcommand registration path through stderr before any JSON stdout is emitted, and add a regression test for `nodes list --json` with noisy plugin startup output.

## Reproduction Assessment

Yes, we have a high-confidence reproduction path from the reporter's exact command and current source showing a pre-preAction plugin-registration path that can write diagnostics before JSON stdout. I did not execute the Helm/container flow in this read-only sweep, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes, the best fix is to make JSON-mode startup routing cover plugin registration before any stdout JSON, rather than patching only LCM log lines. The fix should preserve normal human output for non-JSON commands.
