## Summary

Keep open: current main still sends live tool events to both run-scoped recipients and all session subscribers, the issue has the protected `maintainer` label, and the open linked fix PR is the right resolution path.

## What This Changes

The issue requests deduplicating live gateway websocket tool events so clients that already receive run-scoped `agent` tool frames do not also receive the session-scoped `session.tool` mirror.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/84823, preserving run-scoped delivery while sending the `session.tool` mirror only to session subscribers that did not already receive the run-scoped frame.

## Reproduction Assessment

Yes for source-level reproduction: Control UI clients can both advertise `tool-events` and subscribe to session events, and the current handler broadcasts the same tool activity to both recipient sets without dedupe. I did not rerun the live pidstat/websocket-log profile.

## Solution Assessment

Yes, the linked PR's recipient-set subtraction is the narrowest maintainable fix shape because it changes only recipient selection and preserves session-only late joiners. Maintainers should still review the compatibility edge where a client may be incorrectly registered as run-scoped.
