The issue asks OpenClaw to stop sending duplicate live tool websocket events to clients that receive both run-scoped `agent` events and session-scoped `session.tool` mirrors, while preserving `session.tool` for late session subscribers.

## Best Possible Solution

Keep the shipped gateway dedupe as canonical, have a maintainer close this issue and retire https://github.com/openclaw/openclaw/pull/84823 after accepting the shipped proof, and leave broader CPU-pressure follow-up to https://github.com/openclaw/openclaw/issues/84850.

## Solution Assessment

Yes, the current recipient-set subtraction is the narrow maintainable fix: it preserves `session.tool` for session-only late joiners while avoiding duplicate frames for run-scoped recipients. The remaining action is maintainer cleanup, not another code fix.
