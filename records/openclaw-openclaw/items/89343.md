## Summary

Keep this PR open, but it is not merge-ready: the reported hidden-reasoning leak is plausible and worth fixing, but the patch gates output on the model capability flag instead of the resolved request/session thinking state, so explicit-off reasoning-capable paths can still leak and some custom/default-false paths may lose legitimate reasoning output.

## What This Changes

The PR gates OpenAI-compatible streamed reasoning deltas in the direct completions provider and transport stream parser based on `model.reasoning`.

## Best Possible Solution

Pass a resolved reasoning-output-enabled state into both completions streaming parsers, keep strict tag scanning, add focused enabled/off/default/post-tool regression coverage, and require redacted after-fix live provider output before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: disabled thinking is represented by request/session options and provider wrappers while `model.reasoning` remains a capability flag. A reasoning-capable model with explicit thinking off can still append a leaked `reasoning_content` chunk through the patched transport parser.

## Solution Assessment

No. Gating at the parser output boundary is a plausible shape, but the gate must use resolved thinking state from the request/session path rather than `model.reasoning`; focused tests and real after-fix provider proof are still missing.
