## Summary

Keep this PR open, but it is not merge-ready: the reported hidden-reasoning leak is security-sensitive and still worth fixing, but the patch gates on model capability instead of the per-request reasoning setting and leaves a structured-content reasoning path outside the new guard.

## What This Changes

The PR adds `model.reasoning`-based gates in the OpenAI-compatible completions parsers to skip thinking deltas when reasoning is disabled.

## Best Possible Solution

Thread a single request-level reasoning visibility/off decision into both OpenAI-compatible parsers and apply it before every thinking append or queue path, with focused regression tests and redacted real affected-provider proof.

## Reproduction Assessment

Yes, from source inspection rather than a live provider run: request reasoning state and model capability are separate, and the PR gates on the capability flag. A typed structured-content reasoning delta also reaches `appendRoutedContentDelta` before the new guard.

## Solution Assessment

No. The direction is right, but the best fix is a single request-level reasoning-output decision applied before every thinking output path, plus tests and real affected-provider proof.

Full review comments:

- **[P1] Gate direct completions on request state:** `src/llm/providers/openai-completions.ts:389`
  - body: This guard still checks `model.reasoning`, which is the model capability flag. For a reasoning-capable model where the caller explicitly turns reasoning off, `model.reasoning` remains true, so `reasoning_content` will still be appended; use the resolved request/off state instead.
  - confidence: 0.93
- **[P1] Gate transport reasoning on request state:** `src/agents/openai-transport-stream.ts:2916`
  - body: `reasoningEnabled` is also derived from `model.reasoning` in the transport parser. `buildOpenAICompletionsParams` resolves request reasoning separately, so an explicit off request on a reasoning-capable model can still pass these thinking deltas through.
  - confidence: 0.92
- **[P1] Gate structured-content thinking before routing:** `src/agents/openai-transport-stream.ts:2912`
  - body: Typed `delta.content` reasoning is routed through `appendRoutedContentDelta` before the new guard runs. A provider chunk like `{ type: "reasoning", text: "..." }` still becomes thinking output even when reasoning should be disabled.
  - confidence: 0.9
