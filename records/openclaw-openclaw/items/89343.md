## Summary

Keep open: the revision fixes the obvious direct reasoning-field emission path, but the transport-aware parser still queues reasoning deltas during tool-call blocks before the new disabled-reasoning guard runs, and the external PR still has no after-fix real-provider proof.

## What This Changes

The PR gates OpenAI-compatible streamed reasoning fields on active reasoning effort in both the generic and transport-aware completions parsers.

## Best Possible Solution

Apply one boundary rule across both parsers: always detect mirrored reasoning for strict visible-text routing, suppress thinking output whenever reasoning is disabled including buffered tool-call paths, add focused regression tests, then include redacted real-provider proof.

## Reproduction Assessment

Yes. Source inspection shows current main unconditionally emits mirrored reasoning fields as thinking, and the revised patch still has a clear post-tool-call path where disabled reasoning is queued before the guard; I did not run a live MiMo request.

## Solution Assessment

No. This is a plausible partial fix, but the best fix must apply the disabled-reasoning boundary before both direct emission and buffered tool-call flushing, with tests and real-provider proof.
