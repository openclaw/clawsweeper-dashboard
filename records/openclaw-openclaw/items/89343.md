## Summary

Keep open: the PR targets a real hidden-reasoning leak class, but the patch is not merge-ready because it disables the mirror signal needed for strict tag partitioning and lacks real behavior proof.

## What This Changes

The branch adds a `reasoningEnabled` guard around `reasoning_content`/`reasoning`/`reasoning_text` extraction in `src/llm/providers/openai-completions.ts`.

## Best Possible Solution

Keep reasoning-field detection as a strict-routing signal in every OpenAI completions parser, suppress thinking-block emission when reasoning is disabled, and add focused regression plus redacted real-provider proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current tests already model `reasoning: false` with mirrored `reasoning_content`, and the proposed guard would keep that stream out of strict mode. I did not run a live MiMo request in this read-only review.

## Solution Assessment

No. The safer fix is to keep detecting mirrored reasoning for strict partitioning and only suppress the emitted thinking block when reasoning is disabled, with the transport-aware completions parser handled too.
