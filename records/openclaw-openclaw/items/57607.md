## Summary

Keep open. Current main still returns from the Mattermost `GROUP_POLICY_NOT_ALLOWLISTED` branch before the pending-history helper is defined or called, and #57609 is an open implementation candidate with closing syntax for this issue.

## What This Changes

The report asks Mattermost group allowlist handling to keep non-allowlisted group messages in bounded pending history while still blocking them from triggering responses.

## Best Possible Solution

Review and land #57609 or an equivalent Mattermost-only change that records bounded pending history for visible non-allowlisted group messages, then returns without responding, with a targeted regression test.

## Reproduction Assessment

Yes. The static path is high-confidence: configure Mattermost `groupPolicy: "allowlist"` with a non-allowlisted sender, and current main reaches the immediate not-allowlisted return before any pending-history recording can happen.

## Solution Assessment

Yes, if maintainers accept the context-visibility semantics. The narrow maintainable fix belongs in `extensions/mattermost/src/mattermost/monitor.ts` with existing history bounds and no core policy change.
