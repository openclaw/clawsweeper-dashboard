## Summary

Keep open. Current main still drops Mattermost group messages from non-allowlisted senders before the pending-history helper is defined or called, while the later mention/onchar drop paths do record pending history. There is also an open same-author implementation PR with closing syntax for this issue, so the maintainer path is to review, rebase, land, or close that PR rather than close this issue early.

## What This Changes

The issue requests that Mattermost `groupPolicy: "allowlist"` continue recording pending group history for non-allowlisted senders while still preventing those senders from triggering responses.

## Best Possible Solution

Review and either land #57609 or implement the equivalent directly in `extensions/mattermost/src/mattermost/monitor.ts`: for visible Mattermost group messages from non-allowlisted senders, record bounded pending history, then return before response generation or command handling. Add a focused regression test for `groupPolicy: "allowlist"` with a non-allowlisted sender and update docs if maintainers want to make the trigger-vs-context semantics explicit for Mattermost.

## Reproduction Assessment

Yes. The high-confidence reproduction path is visible in current main: configure Mattermost with `groupPolicy: "allowlist"`, send a group message from a non-allowlisted sender, and the monitor returns at `GROUP_POLICY_NOT_ALLOWLISTED` before `recordPendingHistory()` can run.

## Solution Assessment

Yes, with maintainer confirmation on privacy semantics. The narrow maintainable fix is to keep allowlist as a trigger gate while recording bounded in-memory pending history before returning, plus a regression test that proves the sender still cannot trigger a response.
