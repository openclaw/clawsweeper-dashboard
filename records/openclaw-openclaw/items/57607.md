## Summary

Keep open. Current main still returns early for Mattermost `groupPolicy: "allowlist"` messages from non-allowlisted group senders before pending history can be recorded, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/57609 is still open, unmerged, and conflicting.

## What This Changes

Mattermost `groupPolicy: "allowlist"` should store non-allowlisted group messages as pending history for later allowlisted prompts while still blocking those senders from triggering responses.

## Best Possible Solution

Update and land https://github.com/openclaw/openclaw/pull/57609, or an equivalent Mattermost-only fix, with regression coverage for bounded pending history without response dispatch.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: with Mattermost `groupPolicy: "allowlist"`, a non-allowlisted group sender reaches the immediate return before `recordPendingHistory` exists.

## Solution Assessment

Yes, the requested Mattermost-local history-only fallthrough is the narrow maintainable direction. The linked implementation needs to be ported to the current `ingress`/`commandAccess` handler and covered by a focused regression test.
