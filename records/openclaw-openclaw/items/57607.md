## Summary

Keep open. Current main and v2026.5.20 still return before Mattermost pending history is recorded for non-allowlisted group senders, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/57609 is still open and unmerged.

## What This Changes

Mattermost should record pending group history from non-allowlisted senders under `groupPolicy: "allowlist"` while still preventing those senders from triggering replies or commands.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/57609 or an equivalent Mattermost-only fix with regression coverage proving bounded history recording and no dispatch or command authorization for blocked senders.

## Reproduction Assessment

Yes. Source inspection on current main shows a Mattermost group message from a non-allowlisted sender reaches an immediate return before pending history can be recorded; I did not run a live Mattermost repro.

## Solution Assessment

Yes. The narrow maintainable solution is a Mattermost-local history-only fallthrough that records bounded pending history, then returns before command, tool, or outbound dispatch; the linked PR should be reviewed against the current nested ingress flow.
