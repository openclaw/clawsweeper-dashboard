## Label Justifications

- `P2`: The issue is a normal-priority Google Chat delivery bug with limited channel-specific blast radius.
- `impact:message-loss`: Media replies can be dropped instead of delivering the artifact link when upload fails under app auth.
- `impact:auth-provider`: The failure is tied to Google Chat service-account app authentication scopes versus user-auth media-upload requirements.

## Summary

Keep open: current main and the latest release still route Google Chat media through attachment upload under service-account app auth without a text-link fallback, and open PRs now own competing fixes that have not merged.

## What This Changes

Google Chat app-auth media replies should fall back to delivering the remote media URL as text when attachment upload is rejected for insufficient scopes.

## Best Possible Solution

Land one narrow Google Chat plugin fix that handles documented upload-scope failures for remote media across the outbound adapter, monitor reply delivery, and message action paths without adding config surface.

## Reproduction Assessment

Yes from source: service-account Google Chat media delivery reaches `uploadGoogleChatAttachment` without a scope-failure fallback, and monitor delivery logs then drops the media reply on failure. I did not run a live Google Chat workspace repro in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a plugin-local fallback for documented upload-scope failures on remote URLs across all Google Chat media delivery paths; the broader user-OAuth work is adjacent, not required for this bug.
