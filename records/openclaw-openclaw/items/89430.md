## Summary

Keep open: current main still sends Google Chat remote media through attachment upload under the plugin's app-auth flow, while the official API contract requires user auth for media.upload and current code has no text-link fallback for the URL.

## What This Changes

Request graceful Google Chat fallback from failed app-auth media upload to a text link for remote media URLs.

## Best Possible Solution

Add a Google Chat plugin-owned remote-URL fallback for auth/scope upload failures across the adapter, monitor reply, and message-action media paths while preserving upload behavior where it succeeds.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure Google Chat with service-account app auth, send remote media, and current main attempts attachments:upload without any text-link fallback.

## Solution Assessment

Yes. A narrow plugin-owned fallback on known auth/scope upload failures is the best repair shape; a broad config flag would add operator surface before proving it is needed.
