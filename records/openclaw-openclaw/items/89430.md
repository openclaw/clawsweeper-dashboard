## Summary

Keep open: current main and v2026.6.9 still upload Google Chat media under app auth without a text-link fallback, and linked open PRs remain unmerged implementation candidates.

## What This Changes

The issue requests a Google Chat plugin fallback that sends caption plus URL as text when app-auth remote media attachment upload fails with insufficient scope.

## Best Possible Solution

Review and land one canonical Google Chat plugin fix that covers action, outbound adapter, and monitor reply media paths with a narrow upload insufficient-scope fallback, then close this issue.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main uses chat.bot app auth and all three Google Chat media paths upload before sending without a text-link fallback. I did not run a live Google Chat workspace repro in this read-only review.

## Solution Assessment

Yes at the issue level: a plugin-local text-link fallback for the documented upload insufficient-scope case is the narrowest maintainable fix. The remaining work is choosing and landing one canonical open PR.
