## Summary

Keep open: current main and v2026.6.8 still send Google Chat media through attachment upload under app-auth `chat.bot` without a text-link fallback, and open linked PRs are candidate fixes that have not merged.

## What This Changes

The issue requests a Google Chat plugin fallback that delivers remote media URLs as text when app-auth attachment upload fails with insufficient scopes.

## Best Possible Solution

Land one narrow Google Chat plugin fix that covers message actions, outbound adapter delivery, and monitor reply delivery, falling back to caption/message plus URL only for remote upload insufficient-scope failures.

## Reproduction Assessment

Yes, at source level: current main uses `chat.bot` app auth and all three media paths upload before sending, with no text-link fallback. I did not run a live Google Chat workspace repro in this read-only review.

## Solution Assessment

Yes at the issue level: a plugin-local fallback for the documented upload insufficient-scope case is the narrowest fix. The broader delegated user-OAuth work remains separate in https://github.com/openclaw/openclaw/issues/9764.
