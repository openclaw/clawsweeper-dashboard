## Summary

Keep open: current main and the latest release still route Google Chat remote media through attachment upload under the service-account app-auth path, and two open closing PRs now own the proposed fallback work but have not merged.

## What This Changes

The issue requests a Google Chat plugin fallback that sends remote media URLs as text links when app-auth attachment upload fails with an auth/scope 403.

## Best Possible Solution

Land one narrow Google Chat plugin fix that handles only remote http(s) media plus known upload auth/scope failures across the outbound adapter, monitor reply delivery, and message actions, while preserving normal upload errors.

## Reproduction Assessment

Yes, at source level. Configure Google Chat with service-account app auth and send remote media; current main uses chat.bot, calls attachments:upload, and either propagates or logs upload errors without sending caption plus URL.

## Solution Assessment

Yes. A narrow plugin-owned fallback for remote URL media on known upload auth/scope failures is the best repair shape; a new config flag or broader OAuth feature is not needed for this issue.
