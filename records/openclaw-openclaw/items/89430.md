## Summary

Keep open: current main and latest release still route Google Chat media through attachment upload without a text-link fallback, and open linked PRs with closing syntax are active candidate fixes rather than merged provenance.

## What This Changes

The issue requests a Google Chat plugin fallback that delivers a remote media URL as text when app-auth attachment upload fails with insufficient scopes.

## Best Possible Solution

Land one narrow Google Chat plugin fix covering message actions, outbound adapter delivery, and monitor reply delivery, falling back to caption/message plus URL only for remote upload insufficient-scope failures.

## Reproduction Assessment

Yes, at source level: current main uses app-auth chat.bot and all three Google Chat media paths upload before sending, with no text-link fallback. I did not run a live Google Chat workspace repro in this read-only review.

## Solution Assessment

Yes at the issue level: a plugin-local fallback for the documented upload insufficient-scope case is the narrowest maintainable fix. The broader delegated user-OAuth work remains separate in https://github.com/openclaw/openclaw/issues/9764.
