## Summary

Keep open. Current main still routes normal runtime HTTP, including Ollama embedding `/api/embed` calls, through the managed proxy while clearing `NO_PROXY`; the related exact-origin SSRF change improves adjacent local-provider trust but does not provide the requested local-model proxy bypass.

## What This Changes

Requests a secure proxy bypass or allowlist so local Ollama embedding requests to loopback are not tunneled through an enabled managed proxy.

## Best Possible Solution

Define and test an owner-approved narrow local model-server proxy exception, likely across proxy lifecycle or the Ollama memory embedding path, without restoring a broad `NO_PROXY` loopback bypass.

## Reproduction Assessment

Yes for the routing premise: current source and e2e tests show loopback runtime HTTP still routes through the managed proxy even when `NO_PROXY` includes loopback, and Ollama embeddings call that guarded fetch path. No live Gluetun/Ollama 503 run was performed.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `proxy.allowLocal` would address this deployment, but it conflicts with the documented SSRF boundary; the safer path is a narrow, reviewed local model-server exception with regression coverage.
