## Summary

Keep open. Current main still deliberately routes normal runtime HTTP, including Ollama embedding fetches, through the managed proxy with only a narrow Gateway control-plane loopback bypass, so the requested Ollama/local-model exception is a security-sensitive feature/config decision rather than a cleanup-close candidate.

## What This Changes

Request a secure proxy bypass configuration path so local Ollama embedding traffic can reach loopback while managed proxy routing is enabled.

## Best Possible Solution

Define an owner-approved narrow local model-server proxy exception and document/test it, rather than enabling a general NO_PROXY-style loopback bypass.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows proxy.enabled clears NO_PROXY and sends Ollama /api/embed through guarded fetch and the managed proxy, so the routing premise is source-reproducible while the exact 503 depends on the reporter's proxy/container setup.

## Solution Assessment

Unclear. A general proxy.bypassList or proxy.allowLocal could solve this deployment, but the safer maintainable path is a narrow, owner-approved local model-server exception with SSRF/proxy regression coverage.
