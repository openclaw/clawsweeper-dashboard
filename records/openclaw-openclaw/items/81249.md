## Summary

Keep open. Current main intentionally clears proxy bypass env vars and routes normal runtime HTTP, including Ollama embedding fetches, through the managed proxy; only Gateway control-plane loopback has a scoped exception, so this needs security/product judgment rather than cleanup closure.

## What This Changes

Request a secure proxy bypass or allow-local config so loopback Ollama memory embeddings can avoid managed proxy routing without reopening broad SSRF bypasses.

## Best Possible Solution

Have network/security owners decide whether OpenClaw should support a narrowly scoped local model-server exception; if approved, implement it with schema, docs, proxy lifecycle, guarded-fetch, and Ollama/memory regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run against a Gluetun plus Ollama setup. Source inspection and existing e2e tests show current main clears bypass vars and routes normal loopback HTTP through the managed proxy, so the routing premise is source-reproducible.

## Solution Assessment

Unclear. A proxy.bypassList or proxy.allowLocal option is plausible, but the current design deliberately avoids general destination-based bypasses; the safer direction is an owner-approved, narrow local model-server exception if the product wants this behavior.
