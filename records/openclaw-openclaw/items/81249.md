## Summary

Close as implemented on current main: the Ollama embedding managed-proxy gap is fixed by https://github.com/openclaw/openclaw/pull/85707 at `fd2a9adbe6b0a38cd2eb348376e58d237d40df3c`. This is current-main-only evidence; no release tag contains the fix yet.

## What This Changes

The issue requests a secure way for host-local Ollama embeddings to avoid managed proxy routing under `proxy.enabled=true` without opening a broad `NO_PROXY` or local-network bypass.

## Best Possible Solution

Keep the merged exact-origin loopback Ollama implementation, release it, and avoid adding a general `proxy.bypass` or `NO_PROXY` allowlist.

## Reproduction Assessment

Yes for the original pre-fix problem: source and tests show local Ollama `/api/embed` traffic previously stayed on the managed proxy and now has a narrow direct path. I did not run a live Gluetun plus Ollama repro, and no current-main failing reproduction remains after the merged fix.

## Solution Assessment

Yes. The merged fix is narrower than the requested broad bypass knob: it bypasses only the exact configured loopback Ollama embedding origin after SSRF/DNS validation and still respects `proxy.loopbackMode`.
