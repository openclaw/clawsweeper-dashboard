## Summary

Keep open. Current main still intentionally routes normal runtime HTTP, including Ollama embedding requests, through the managed proxy and clears bypass env vars; adding a local-model bypass would be a new security-sensitive config/product decision, not a cleanup-close candidate.

## What This Changes

Request to add a secure native proxy bypass for local Ollama embedding endpoints when `proxy.enabled` is active.

## Best Possible Solution

Have network/security owners decide whether OpenClaw should support a narrow local model-server exception, then implement it with schema, docs, proxy lifecycle, guarded-fetch, and Ollama/memory regression coverage if approved.

## Reproduction Assessment

No live high-confidence deployment reproduction was run. Source and test inspection does give a high-confidence source reproduction path: enable the managed proxy, set loopback NO_PROXY values, and issue an Ollama `/api/embed` request through the guarded fetch path.

## Solution Assessment

Unclear. A `proxy.bypassList` or `proxy.allowLocal` direction is plausible, but the safer maintainable solution is an owner-approved, narrowly scoped local model-server exception rather than a general destination bypass.
