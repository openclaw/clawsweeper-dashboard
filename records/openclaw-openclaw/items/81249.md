## Summary

Keep open. Current main still intentionally routes normal runtime HTTP, including local Ollama embeddings, through the managed proxy and has no accepted proxy bypass config, so this is a security-sensitive feature/config decision rather than a cleanup-close candidate.

## What This Changes

Add a secure config-supported way for local Ollama or loopback embedding traffic to bypass managed proxy routing without reopening broad NO_PROXY SSRF bypasses.

## Best Possible Solution

Have network/security owners decide whether a narrowly scoped local model-server exception is acceptable; if approved, implement it with schema, docs, proxy lifecycle, guarded-fetch, Ollama/memory tests, and a live proxy/Ollama smoke.

## Reproduction Assessment

No live high-confidence reproduction was run against current main. Source inspection gives a clear reproduction premise: with managed proxy enabled, the Ollama `/api/embed` request goes through guarded fetch using the managed EnvHttpProxyAgent while bypass env is cleared.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `proxy.allowLocal` may solve the deployment but is not automatically the best design because current docs deliberately avoid a general local-network bypass; the safer path is an owner-approved narrow exception or an explicit decision to keep this in operator proxy policy.
