## Summary

Keep open. Current main still intentionally clears managed-proxy bypass variables and routes normal runtime HTTP, including Ollama embedding fetches, through the managed proxy path; the requested local Ollama exception is a security-sensitive feature/config decision rather than a cleanup-close candidate.

## What This Changes

The issue requests a secure OpenClaw proxy bypass for local Ollama embeddings when the managed proxy is enabled.

## Best Possible Solution

Have network/security owners decide whether OpenClaw should support a narrowly scoped local model-server exception, then implement the approved contract across proxy schema, guarded fetch or lifecycle behavior, docs, and Ollama or memory regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows the current path clears NO_PROXY and routes Ollama /api/embed fetches through the managed proxy when proxy.enabled is active, so the reported routing premise is source-reproducible.

## Solution Assessment

Unclear. A proxy.bypassList or proxy.allowLocal setting is plausible, but the current docs intentionally avoid destination-based bypasses; the safer direction is an owner-approved, narrowly scoped local model-server exception if the product wants this behavior.
