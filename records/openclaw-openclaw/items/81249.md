## Summary

Keep open. Current main still routes normal runtime HTTP through the managed proxy and clears NO_PROXY, while native Ollama memory embeddings call strict guarded fetch for `/api/embed` with no local model-server bypass. The linked merged PR is useful adjacent SSRF exact-origin work for custom/local model-provider requests, but it does not change this native Ollama embedding managed-proxy path.

## What This Changes

The issue asks for a secure way for local Ollama embeddings on loopback to avoid the managed HTTP proxy while proxy SSRF defenses remain enabled.

## Best Possible Solution

Define an owner-approved narrow local model-server exception for managed proxy mode, with SSRF/proxy regression tests and docs, rather than adding a broad NO_PROXY-style loopback bypass.

## Reproduction Assessment

Yes for the source-level routing premise: native Ollama embeddings call strict guarded fetch and managed proxy mode attaches an environment proxy dispatcher after clearing NO_PROXY. I did not run the reporter's Gluetun/Ollama setup, so the exact 503 is not live-reproduced.

## Solution Assessment

Unclear. A general proxy.bypassList or proxy.allowLocal would address the deployment, but the safer maintainable direction is a narrow, owner-approved local model-server exception that preserves SSRF guarantees.
