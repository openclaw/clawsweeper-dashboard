## Summary

Keep open. Current main intentionally routes normal runtime HTTP through the managed proxy and clears destination bypass lists, while Ollama memory embeddings use that guarded HTTP path; adding a local Ollama bypass would be a new SSRF-sensitive config/product decision rather than a strict bug fix.

## What This Changes

The issue requests a secure config path for local Ollama embedding traffic to bypass OpenClaw's managed HTTP proxy without disabling SSRF protections.

## Best Possible Solution

Define a narrow, reviewed local-provider proxy policy for memory embeddings that preserves managed-proxy SSRF guarantees, then document its interaction with operator proxy policy and Ollama setup.

## Reproduction Assessment

Yes for source-level reproduction: Ollama embeddings call guarded /api/embed HTTP, managed proxy mode clears NO_PROXY, and current tests assert normal loopback egress still routes through the proxy. I did not live-run the reported Gluetun plus Ollama setup.

## Solution Assessment

No for an automated bypass as proposed; a secure fix needs maintainer design because broad bypassList or allowLocal semantics would change the managed proxy trust boundary. A narrower provider/local-model policy is the safer direction if maintainers want this capability.
