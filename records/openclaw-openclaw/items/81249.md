## Summary

Keep open. Current main deliberately routes normal runtime loopback HTTP, including Ollama embedding fetches, through the managed proxy and exposes no proxy bypass schema, so this is security-sensitive feature/config work rather than a cleanup-close candidate.

## What This Changes

Request a native secure proxy bypass or allow-local mechanism so local Ollama embedding calls can reach loopback when `proxy.enabled` is true.

## Best Possible Solution

Have network/security owners approve a narrow local model-server exception contract, then implement it across config schema, proxy lifecycle or guarded fetch, Ollama behavior, docs, and regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows `proxy.enabled` clears `NO_PROXY` and sends Ollama `/api/embed` through the managed proxy, while the exact 503 depends on the reporter's proxy/container setup.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `proxy.allowLocal` could address this deployment, but the safer maintainable path is an owner-approved local model-server exception rather than a broad loopback bypass.
