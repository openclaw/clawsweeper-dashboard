## Summary

Keep open. Current main intentionally routes normal runtime HTTP, including Ollama embedding fetches, through the managed proxy and only exposes a Gateway control-plane loopback exception, so this is a security-sensitive feature/config decision rather than a cleanup-close candidate.

## What This Changes

Add a secure native proxy bypass for local Ollama memory-embedding loopback traffic when managed proxy mode is enabled.

## Best Possible Solution

Have network/security owners decide whether OpenClaw should support a narrowly scoped local model-server exception, then implement it with schema, docs, proxy lifecycle, guarded-fetch, and Ollama/memory regression coverage if approved.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection does show the current path clears `NO_PROXY` and routes Ollama `/api/embed` fetches through the managed proxy when `proxy.enabled` is active, so the routing premise is source-reproducible.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `proxy.allowLocal` is plausible but security-sensitive because current docs deliberately avoid a general local-network bypass; the safer direction is an owner-approved, narrowly scoped exception if the product wants this behavior.
