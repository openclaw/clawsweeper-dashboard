## Summary

Keep open. Current main still has no native Ollama/local-model proxy bypass, the docs intentionally keep general local-network bypasses out of the managed proxy contract, and the reporter retested v2026.5.20 with the failure still present; the newer browser/CDP symptom has an open targeted PR, but it does not solve the Ollama embedding path.

## What This Changes

The issue asks for a secure native proxy bypass or allowlist so local Ollama embeddings and loopback browser/CDP traffic are not tunneled through the managed proxy.

## Best Possible Solution

Define an owner-approved narrow local model-server proxy policy, or explicitly keep the current operator-proxy allowlist contract, while letting the separate browser CDP loopback fix proceed through https://github.com/openclaw/openclaw/pull/83255 if maintainers accept it.

## Reproduction Assessment

Yes for the source-level routing premise: current main clears NO_PROXY and sends Ollama /api/embed through guarded fetch while the managed proxy is active. A live Gluetun/Ollama reproduction would still be needed to prove the exact 503 timing and proxy-container behavior.

## Solution Assessment

Unclear. A general proxy.bypassList or allowLocal setting is not obviously the safest solution; the maintainable path is a narrow, maintainer-approved local model-server exception with SSRF/proxy regression coverage, while browser CDP remains a separate targeted PR.
