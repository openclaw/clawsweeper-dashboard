## Summary

Keep open. Current main intentionally routes normal runtime HTTP, including Ollama embedding fetches, through the managed proxy and only has a narrow Gateway control-plane loopback exception, so this is a security-sensitive feature/config request rather than a cleanup-close candidate.

## What This Changes

Request a secure OpenClaw proxy bypass for local Ollama embedding traffic when managed proxying is enabled.

## Best Possible Solution

Have network/security owners decide whether OpenClaw should support a narrowly scoped local model-server exception, then implement the approved contract across schema, docs, proxy lifecycle, guarded fetch, and Ollama/memory regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was run. Source and tests show that proxy.enabled clears NO_PROXY and routes normal Ollama /api/embed runtime HTTP through the managed proxy; the exact 503 depends on the reporter's proxy/container deployment.

## Solution Assessment

Unclear. A general proxy.bypassList or proxy.allowLocal could solve this deployment, but the safer maintainable path is an owner-approved narrow local model-server exception rather than a broad loopback bypass.
