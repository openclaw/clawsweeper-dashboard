## Summary

Keep open. Current main still intentionally routes normal runtime HTTP, including Ollama embedding calls, through the managed proxy while clearing destination bypass env vars, so the requested local Ollama exception is a security-sensitive feature/config decision rather than an implemented cleanup-close candidate.

## What This Changes

The issue requests a secure managed-proxy bypass for loopback Ollama embeddings when `proxy.enabled` is active.

## Best Possible Solution

Have network/security owners decide whether OpenClaw should support a narrowly scoped local model-server exception, then implement schema, docs, proxy lifecycle, guarded-fetch/Ollama coverage, and live proxy smoke proof if approved.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does show the current path clears bypass env vars and sends Ollama `/api/embed` through guarded fetch with the managed proxy active, so the routing premise is source-reproducible.

## Solution Assessment

Unclear. A `proxy.bypassList` or `proxy.allowLocal` could solve the operational problem, but the maintainable path needs an owner-approved, narrowly scoped exception because current docs intentionally reject general local-network bypasses.
