## Summary

Keep open. Current main still sends bundled Ollama memory embedding `/api/embed` requests through the managed proxy when `proxy.enabled=true`, and the active remaining fix is the open replacement PR at https://github.com/openclaw/openclaw/pull/85707 rather than code already merged to main.

## What This Changes

The issue asks for local Ollama memory embeddings to work with `proxy.enabled=true` without weakening the managed-proxy SSRF boundary.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85707 or an equivalent narrow exact configured-loopback Ollama embedding fix, while keeping a general local-network bypass out of the core proxy contract.

## Reproduction Assessment

Yes, source-reproducible: configure Ollama embeddings to `http://127.0.0.1:11434` with a managed proxy that cannot reach the host loopback service, and current main routes `/api/embed` through that proxy. I did not run the live Gluetun/Ollama setup here, but source, tests, docs, and reporter retests align.

## Solution Assessment

Partially. A broad `proxy.bypass` or `allowLocal` knob is not the safest solution; the narrow active PR direction that bypasses only the exact configured loopback Ollama embedding origin is the more maintainable path if maintainers accept the policy exception.
