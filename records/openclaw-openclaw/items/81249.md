## Summary

Keep open. Current main still routes Ollama memory embedding HTTP through the managed proxy, and the merged exact-origin SSRF work linked in the discussion covers custom/local model-provider requests rather than a local model-server proxy bypass.

## What This Changes

Issue requests an SSRF-safe way for local Ollama embedding endpoints on loopback to avoid the managed proxy when `proxy.enabled` is true.

## Best Possible Solution

Define an owner-approved, narrow local model-server exception for the managed proxy boundary, with SSRF/proxy tests and docs, instead of adding a general `NO_PROXY`-style bypass.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows `proxy.enabled` clears `NO_PROXY` and sends Ollama `/api/embed` through guarded fetch plus the active managed proxy, while the exact 503 depends on the reporter's proxy container topology.

## Solution Assessment

Unclear. The linked exact-origin SSRF change is not sufficient for the reported proxy-routing failure; the maintainable solution needs a narrow network/security-approved exception rather than a general local bypass list.
