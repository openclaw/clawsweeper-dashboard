## Summary

Keep open. Current main still intentionally routes normal runtime HTTP/WebSocket traffic, including local Ollama embedding calls, through the managed proxy with no general local bypass; the newer CDP failure appears partially covered by an open browser PR, but the Ollama/local-model proxy exception remains a security-sensitive product decision.

## What This Changes

Request a secure local proxy bypass so loopback Ollama embeddings, and related local runtime endpoints, can work when `proxy.enabled=true`.

## Best Possible Solution

Define owner-approved narrow exceptions only where OpenClaw treats local endpoints as trusted IPC, review or land the focused browser CDP PR separately, and require SSRF/proxy regression coverage plus live proxy/Ollama proof before changing the Ollama path.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection confirms the reported routing premise: managed proxy clears NO_PROXY and local Ollama embedding HTTP remains on the guarded proxy path.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `allowLocal` would solve the deployment but is too broad without an explicit security decision; the better shape is narrow endpoint-specific trust with tests and live proof.
