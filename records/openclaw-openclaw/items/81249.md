## Summary

Keep open. Current main still routes normal runtime HTTP, including Ollama embedding fetches, through the managed proxy and clears NO_PROXY; the merged exact-origin SSRF work in https://github.com/openclaw/openclaw/pull/80751 is adjacent but does not add the requested native local-Ollama proxy bypass.

## What This Changes

Request a secure OpenClaw config path to exempt local Ollama embedding traffic from managed proxy routing while preserving SSRF protections.

## Best Possible Solution

Define a security-reviewed, narrow local model-server exception or document the supported proxy-network topology, with tests proving metadata/link-local and unrelated loopback targets remain blocked.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced: current code clears NO_PROXY, routes guarded fetches through the active managed proxy, and the proxy E2E test codifies normal loopback traffic going through that proxy. The exact 503 requires the reporter's proxy/container topology.

## Solution Assessment

Unclear. A proxy.bypassList or proxy.allowLocal would address the deployment, but the maintainable path needs network/security approval and should be narrower than a general NO_PROXY-style loopback bypass.
