## Summary

Keep open. Current main still intentionally sends normal Ollama embedding HTTP through the managed proxy, and the reporter’s v2026.5.18 retest confirms the adjacent exact-origin SSRF fix did not solve this managed-proxy loopback case.

## What This Changes

The issue requests a secure proxy bypass path so local Ollama embedding traffic can reach loopback while OpenClaw's managed proxy and SSRF protections remain enabled.

## Best Possible Solution

Define an owner-approved, narrow local model-server exception with proxy and SSRF regression coverage instead of accepting general NO_PROXY-style destination bypasses.

## Reproduction Assessment

No live high-confidence repro was run. Source inspection shows managed proxy clears bypass env and Ollama `/api/embed` uses guarded fetch through the active managed proxy, while the exact timeout depends on the reporter's Gluetun/Ollama host setup.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `proxy.allowLocal` could solve this deployment, but the safer maintainable path is a narrow local model-server exception with explicit SSRF/proxy tests and docs.
