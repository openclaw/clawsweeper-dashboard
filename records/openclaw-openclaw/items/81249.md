## Summary

Keep open. Current main still has no local Ollama/proxy bypass path: managed proxy mode clears NO_PROXY, normal runtime fetches still use the managed proxy dispatcher, and Ollama embeddings call `/api/embed` through guarded fetch without a proxy-bypass contract. The merged exact-origin SSRF work is adjacent but does not solve this managed-proxy routing problem.

## What This Changes

Request a secure proxy bypass or allow-local path so local Ollama embedding calls on loopback do not route through the managed HTTP proxy.

## Best Possible Solution

Define, document, and test an owner-approved narrow local model-server proxy exception for Ollama embeddings that preserves the managed proxy SSRF boundary, rather than adding a general NO_PROXY-style bypass list.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a high-confidence routing path: `proxy.enabled` clears NO_PROXY and Ollama `/api/embed` uses guarded fetch, which uses the managed proxy dispatcher when the managed proxy is active; the concrete 503 depends on the reporter's proxy/container setup.

## Solution Assessment

Unclear. The requested `proxy.bypassList` or `allowLocal` direction would solve this deployment, but the safer maintainable solution is a narrow, reviewed local model-server exception with SSRF/proxy regression coverage.
