## Summary

Keep open. Current main still has no local-model proxy bypass: normal runtime HTTP to loopback is documented and tested to use the managed proxy, while the merged exact-origin SSRF work in https://github.com/openclaw/openclaw/pull/80751 addresses private-provider SSRF blocking rather than the proxy tunnel routing reported here.

## What This Changes

The issue asks for a secure way to let local Ollama embeddings on loopback avoid the managed proxy when proxy routing is enabled.

## Best Possible Solution

Define an owner-approved narrow local model-server proxy exception, with proxy/SSRF regression coverage and docs, instead of exposing a general NO_PROXY-style local-network bypass.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows proxy.enabled clears NO_PROXY and sends Ollama /api/embed through guarded fetch and the managed proxy, but the exact 503 requires a proxy setup that cannot reach the host loopback service.

## Solution Assessment

Unclear. A broad proxy.bypassList or allowLocal setting could address this deployment, but the safer maintainable direction is a narrow, reviewed local model-server exception that preserves the SSRF/proxy boundary.
