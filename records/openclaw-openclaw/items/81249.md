## Summary

Keep open. Current main intentionally routes normal runtime HTTP, including Ollama embedding requests, through the managed proxy and only provides a narrow Gateway control-plane loopback exception, so a local Ollama bypass is a security-sensitive feature/config decision.

## What This Changes

Request a secure proxy bypass for local Ollama embedding traffic when managed proxy mode is enabled and bypass environment variables are cleared.

## Best Possible Solution

Have network/security owners decide whether OpenClaw should support a narrowly scoped local model-server exception, then implement it with schema, docs, guarded-fetch/proxy lifecycle coverage, and a live proxy/Ollama smoke if approved.

## Reproduction Assessment

No, there is not a high-confidence live reproduction from this review. Source inspection does show current main clears bypass env, routes strict guarded fetch through the managed proxy, and sends Ollama `/api/embed` through that path when proxy mode is active.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `proxy.allowLocal` is plausible but security-sensitive; the safer direction is an owner-approved, narrow exception or documented operator proxy policy.
