## Summary

Keep open. Current main and the latest shipped release still intentionally clear proxy bypass env vars and route normal runtime HTTP, including Ollama embedding fetches, through the managed proxy; supporting a loopback/local-model exception would add a new security-sensitive config surface, not a cleanup close.

## What This Changes

Request a secure proxy bypass or allowlist config so local Ollama embedding traffic can avoid the managed proxy while preserving SSRF protections.

## Best Possible Solution

Have network/security owners decide whether to support a narrowly scoped local model-server exception; if approved, implement it across schema, docs, proxy lifecycle or guarded-fetch routing, Ollama/memory tests, and a live proxy/Ollama smoke.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection does show the reported path: managed proxy mode clears `NO_PROXY`, strict guarded fetch uses the configured env proxy, and Ollama embeddings call that guarded fetch path.

## Solution Assessment

Unclear. A general `proxy.bypassList` or `proxy.allowLocal` might solve the reported deployment, but the maintainable path needs an explicit security/product decision and likely a narrower local model-server exception than a broad destination bypass.
