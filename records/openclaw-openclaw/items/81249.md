## Summary

Keep open. The merged related work in https://github.com/openclaw/openclaw/pull/80751 added exact-origin SSRF trust for configured custom/local model provider fetches, but current main still clears NO_PROXY, has no proxy bypass config, and sends native Ollama embedding fetches through the managed proxy when it is active.

## What This Changes

The issue asks for a secure OpenClaw proxy bypass or whitelist so native Ollama embeddings on loopback can work while managed proxy routing remains enabled.

## Best Possible Solution

Have a network/security owner define a narrow local model-server proxy exception, with schema/docs/tests and live proxy/Ollama proof, instead of accepting a general NO_PROXY-style bypass.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows the reported route is still present: proxy.enabled clears NO_PROXY, native Ollama embeddings call /api/embed through guarded fetch, and strict managed proxy mode selects the env proxy dispatcher.

## Solution Assessment

Unclear. The merged exact-origin SSRF change is useful related work, but the requested managed-proxy bypass still needs a deliberate security/product decision and likely a narrower local-model exception than a general bypass list.
