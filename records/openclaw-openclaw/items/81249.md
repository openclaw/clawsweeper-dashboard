## Summary

Keep open. Current main still routes bundled Ollama memory embedding `/api/embed` requests through the managed proxy when `proxy.enabled=true`, and the remaining fix is the open replacement PR at https://github.com/openclaw/openclaw/pull/85707 rather than code already merged to main.

## What This Changes

The issue asks for local Ollama loopback embeddings to work under the managed proxy without reopening broad `NO_PROXY` SSRF bypasses.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85707 or an equivalent exact configured-loopback Ollama embedding exception under `proxy.loopbackMode`, without adding a broad `proxy.bypass` contract.

## Reproduction Assessment

Yes, source-reproducible: configure Ollama embeddings to `http://127.0.0.1:11434` with a managed proxy that cannot reach host loopback, and current main routes `/api/embed` through that proxy. I did not run the live Gluetun/Ollama setup here, but source, tests, docs, and reporter retests align.

## Solution Assessment

Partially. A broad `proxy.bypass` or `allowLocal` knob is not the best fix; the safer path is a narrow exact configured-loopback Ollama embedding exception with SSRF validation and `proxy.loopbackMode` coverage.
