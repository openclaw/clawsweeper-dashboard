## Summary

Keep open. Current main still routes Ollama embedding requests through guarded fetch and the managed proxy path, while the merged provider trust work and the open browser/CDP PR cover different paths; the reporter also retested the latest release and confirmed the Ollama failure remains.

## What This Changes

Request a secure way for local Ollama embedding or model-server traffic to avoid the managed proxy when `proxy.enabled=true` without reopening broad `NO_PROXY` bypasses.

## Best Possible Solution

Define an owner-approved, exact configured local embedding/model-server proxy exception with SSRF tests proving metadata hosts, other private hosts, different ports, and redirects do not bypass the boundary.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows `proxy.enabled` clears `NO_PROXY` and strict guarded Ollama embedding fetches can use the managed proxy, while the exact 503 depends on the reporter's proxy/container setup.

## Solution Assessment

Unclear. A general bypass list would be risky; the maintainable direction is a narrow exact-origin local embedding/model-server exception if maintainers approve that security contract.
