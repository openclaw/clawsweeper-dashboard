  - body: This adds audit data to the old `gateway-restart-intent.json` writer. Current main persists restart intents in `gateway_restart_intent`, so this must extend the SQLite-backed state instead of reintroducing file-backed runtime restart state.

  - body: The modified `/restart` scheduling call adds `triggerAudit` but omits current main's `sessionKey: sentinelPayload?.sessionKey` guard. Without that guard, coalesced restarts from another session can replace or drop the original restart continuation.
  - confidence: 0.88

  - body: The new writer emits raw `sender_id`, `session_key`, `delivery_context`, and `ws_conn_id` values. Those can identify users or routing context in shared support logs, so hash/remove them or get explicit maintainer approval for the local-log contract.
  - confidence: 0.84
- **[P2] Carry audit into the SIGTERM receiver path:** `src/cli/gateway-cli/run-loop.ts:466-470`
  - body: The SIGTERM audit line only reads `restartIntent?.audit`, but the slash-command supervisor path writes audit to the sentinel and passes it to `triggerOpenClawRestart`; a real launchctl/systemd SIGTERM can still log `source=external`. Persist or read the same audit context for the receiver side.
  - confidence: 0.9
- **[P2] Use distinct keys for trigger and lifecycle actions:** `src/cli/gateway-cli/run-loop.ts:466-471`

  - confidence: 0.86
