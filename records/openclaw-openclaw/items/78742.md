## Best Possible Solution

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction for the PR defect: the actual `/restart` path passes audit only to trigger helpers, while the signal handler logs audit only from `restartIntent?.audit`. I did not run a live gateway restart in this read-only review.

## Solution Assessment

No. The overall approach is maintainable, but this implementation must carry audit through the real signal boundary and avoid duplicate `action` keys before it is the right fix.
