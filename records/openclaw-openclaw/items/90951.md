## Summary

Keep open: current main and v2026.6.8 still exclude the sandbox setup/helper/Dockerfile chain from the npm package while doctor still tries to call those scripts; the remaining work is a maintainer package and setup UX decision, not an already-implemented cleanup.

## What This Changes

The issue asks maintainers to decide how package-only npm installs should obtain sandbox setup assets so `openclaw doctor --fix` can build missing sandbox images outside a source checkout.

## Best Possible Solution

Align package contents, doctor behavior, release checks, and docs around one supported npm-install sandbox setup path; if auto-build is desired, ship the full asset chain with tarball validation or embed the build logic.

## Reproduction Assessment

Yes. Source inspection shows the npm package allowlist omits the sandbox setup scripts, their sourced helper, and required Dockerfiles while doctor still calls those paths for missing default sandbox images.

## Solution Assessment

Unclear. Packaging the full asset chain and embedding a first-class build path are both plausible, while the current docs-only manual path is also an intentional contract that maintainers should either keep or replace.
