## Summary

Keep open: current main and v2026.6.9 still do not ship the sandbox setup scripts or their required helper/Dockerfile chain in the npm package, while doctor still offers to run those scripts for missing sandbox images. This remains a package/doctor UX product decision, not an implemented cleanup.

## What This Changes

The issue requests a supported way for npm-package installs to build sandbox images through doctor, either by shipping the setup assets, embedding build logic, or documenting the manual-only contract.

## Best Possible Solution

Choose one supported npm-install sandbox setup contract and align doctor repair behavior, package contents or embedded build logic, release tarball checks, and docs around that contract.

## Reproduction Assessment

Yes. Source inspection shows package installs omit the sandbox setup asset chain while doctor still offers to run those script paths for missing sandbox images; I did not run a packed npm install flow in this read-only review.

## Solution Assessment

Unclear. Shipping the full asset chain and embedding a first-class build path are both plausible, while the current docs-only manual path is also an intentional contract that maintainers should either keep or replace.
