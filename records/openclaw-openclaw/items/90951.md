## Summary

Keep open: current main and v2026.6.6 still do not ship the sandbox setup assets in the npm package, and `doctor --fix` still reaches source-checkout script paths for missing default sandbox images. The docs now document a manual npm-install build path, but the requested in-tool/package auto-build behavior still needs a maintainer product and release-surface decision.

## What This Changes

The issue asks for a supported package-install path so `openclaw doctor --fix` can build missing Docker sandbox images even though the sandbox setup scripts are omitted from the npm package.

## Best Possible Solution

Keep the issue open until maintainers choose one supported package-install model: package the complete sandbox build asset set, implement an embedded doctor build path, or keep npm installs manual-only and make the doctor message match that contract.

## Reproduction Assessment

Yes. Source inspection shows the npm package allowlist omits the sandbox build assets while `doctor-sandbox.ts` still maps missing default images to those script paths; I did not run a package install because this review is read-only.

## Solution Assessment

Unclear. Packaging the scripts can work only if the full helper and Dockerfile dependency chain is shipped, while an embedded build path or explicit manual-only behavior are also plausible maintainer choices.
