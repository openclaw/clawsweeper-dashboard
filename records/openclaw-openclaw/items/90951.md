## Summary

Keep open: current main and v2026.6.6 still package `doctor --fix` in a way that cannot auto-build sandbox images from an npm-only install, while current docs only provide a manual npm-install Docker build path. The remaining choice is a package/release/sandbox UX decision, not an already-implemented fix.

## What This Changes

The issue requests a maintainer decision for npm-package installs where `openclaw doctor --fix` cannot auto-build sandbox images because the sandbox setup assets are not shipped.

## Best Possible Solution

Maintainers should choose one package-safe sandbox image setup contract, then align doctor prompts, package allowlist/tarball checks, docs, and focused tests around that path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: npm package files omit the sandbox setup assets while doctor maps missing default images to those scripts and falls back to an unable-to-locate message; I did not run a package install because this review is read-only.

## Solution Assessment

Unclear. The issue lists viable directions, but shipping package assets, embedding the build path, or keeping npm installs manual-only each changes the release/sandbox UX and needs maintainer choice before implementation.
