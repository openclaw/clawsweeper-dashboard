## Summary

Keep open: current main and the latest release still do not let an npm-package install auto-build sandbox images through `doctor --fix`; the remaining choice is an explicit package/setup UX decision, not something already solved by the related symlink-launcher PR.

## What This Changes

The issue requests a package-install-safe way for `openclaw doctor --fix` to build missing Docker sandbox images when the setup scripts are not shipped in the npm package.

## Best Possible Solution

Keep this open for maintainers to choose one supported npm-install sandbox image path, then align `doctor`, package contents, focused tests, and public docs around that path.

## Reproduction Assessment

Yes. Source inspection shows npm-package installs lack the setup assets that `doctor-sandbox.ts` needs when a default sandbox image is missing; current docs also acknowledge those scripts are not in the npm package.

## Solution Assessment

Unclear. Shipping only the three scripts is incomplete because they depend on unshipped helper and Dockerfile assets, while an embedded build path or documented manual-only path needs maintainer product judgment.
