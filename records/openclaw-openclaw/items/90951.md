## Summary

Keep open: current main and v2026.6.8 still do not ship the sandbox setup assets in the npm package, while doctor still shells out to those script paths for missing default sandbox images. The docs now document the manual npm-install build path, so the remaining question is a maintainer product/release decision about whether package installs should gain doctor auto-build support.

## What This Changes

The issue asks maintainers to decide whether npm-package installs should be able to auto-build Docker sandbox images through doctor despite the setup scripts not being shipped.

## Best Possible Solution

Choose one supported package-install sandbox setup path and align doctor behavior, package allowlists, release/tarball checks, and docs around that path.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.8 shows the npm package file list omits the sandbox build assets while doctor still maps missing default images to those scripts; I did not run a package install because this review is read-only.

## Solution Assessment

Unclear. The proposed script-shipping option is only safe if the whole helper/Dockerfile dependency chain is packaged, while an embedded build path or manual-only policy needs maintainer product direction.
