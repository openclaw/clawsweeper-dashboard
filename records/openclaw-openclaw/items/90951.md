## Summary

Keep open: current main and the latest release still exclude the sandbox build script/helper/Dockerfile chain from the package while doctor still tries to call those scripts, and the docs only document manual npm-install builds. The remaining change is a maintainer package and setup UX decision, not an already-implemented cleanup.

## What This Changes

Make `openclaw doctor --fix` able to build missing sandbox images for npm or pnpm global package installs where the setup scripts are not currently shipped.

## Best Possible Solution

Align package contents, doctor prompts, and docs around one supported npm-install sandbox setup path; if auto-build is desired, ship the full asset chain with tarball validation or embed the build logic.

## Reproduction Assessment

Yes. Source inspection shows the npm package allowlist omits the setup scripts, their sourced helper, and their Dockerfiles while doctor still calls those paths for missing default sandbox images.

## Solution Assessment

Unclear. Packaging the full asset chain and embedding a first-class build path are both plausible, while the current docs-only manual path is also an intentional contract that maintainers should either keep or replace.
