## Summary

Keep open: current main and the latest release still exclude the sandbox setup script/helper/Dockerfile chain from the npm package while doctor still tries to run those scripts for missing default sandbox images. Because the docs now document manual npm-install builds, making this automatic is a maintainer package/release UX decision rather than a cleanup close.

## What This Changes

The issue asks for an npm-package-safe sandbox image build/repair path for `openclaw doctor --fix`, either by shipping the needed setup assets, embedding the build path, or explicitly keeping npm installs on manual builds.

## Best Possible Solution

Keep docs, doctor behavior, and package contents aligned around one supported npm-install sandbox setup path, with tarball-level validation for every required asset if maintainers choose auto-build support.

## Reproduction Assessment

Yes. Source inspection shows npm-package installs do not contain the helper scripts, sourced build helpers, or sandbox Dockerfiles that doctor tries to run when default sandbox images are missing.

## Solution Assessment

Unclear. Auto-building from npm packages could be solved by packaging the full asset chain or embedding a first-class build path, but the current docs-only contract is also intentional enough that maintainers should choose the supported UX.
