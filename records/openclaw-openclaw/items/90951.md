## Summary

Keep open: current main and the latest v2026.6.9 release still package the doctor flow without the sandbox setup scripts it tries to run, so the remaining question is a maintainer product decision about package-installed sandbox image setup rather than completed cleanup.

## What This Changes

The issue asks OpenClaw to support or explicitly decide against sandbox image auto-builds for npm-package installs where the setup scripts are not shipped.

## Best Possible Solution

Choose one supported npm-package sandbox setup contract and align doctor repair behavior, packaged assets or embedded build logic, release pack checks, and docs around that contract.

## Reproduction Assessment

Yes. Source inspection shows npm package installs lack the setup assets while doctor still tries to run those paths for missing default sandbox images; I did not run a packed npm install flow in this read-only review.

## Solution Assessment

Unclear. Packaging the full asset chain, embedding a build path, or intentionally keeping the current documented manual path are all plausible product choices that maintainers need to choose between.
