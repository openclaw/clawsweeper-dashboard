## Summary

Keep open: current main and the latest release still have the large inline WebChat attachment path, no client size preflight, and a whole-payload gateway base64 regex in the related parser path. An open fix PR already references this issue with closing syntax, so this issue should stay open until that PR lands, is replaced, or maintainers pick the overlapping gateway-validator fix path.

## What This Changes

WebChat should not fail with `Maximum call stack size exceeded` when users upload or send attachments larger than about 4 MB; it should either send them successfully or report a clear supported limit.

## Best Possible Solution

Review and land or replace one coherent fix that prevents large WebChat attachment stack overflows across picker, drop, paste, and gateway validation, then close this issue together with https://github.com/openclaw/openclaw/issues/80679 if the same root cause is resolved.
