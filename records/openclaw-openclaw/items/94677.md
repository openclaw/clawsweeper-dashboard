## Summary

Keep open: current main still has a source-backed path where new input can interrupt a wait, the subagent wait result is treated as killed, and killed termination suppresses normal completion announce handling. ## What This Changes

The issue asks OpenClaw to preserve or explicitly report non-persistent Monitor completion or cancellation when a new user turn arrives before the watched condition fires.

## Best Possible Solution

Maintainers should define the non-persistent Monitor lifecycle, then add focused proof that completion survives intervening turns or that intentional interruption emits a durable parent-visible terminal event.
