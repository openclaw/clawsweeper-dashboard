## Summary

## What This Changes

The issue reports that non-persistent Monitor tasks can be killed when a new user turn starts before their watched condition fires, causing the eventual completion notification to be lost.

## Best Possible Solution

Maintainers should define the non-persistent Monitor lifecycle explicitly, then either preserve completion through intervening user turns until condition or timeout, or deliver a visible terminal cancellation event and document that supersession is intentional.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection and the issue's two real instances make the killed/lost-completion path plausible, but the exact persistent:false Monitor path still needs a current-main live repro.
