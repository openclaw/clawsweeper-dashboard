## What This Changes

The issue asks OpenClaw to stop losing completion notifications from non-persistent Monitor tasks when a later user turn starts before the watched condition fires.

## Best Possible Solution

Define the non-persistent Monitor lifecycle explicitly, then either preserve completion through intervening user turns until condition or timeout, or deliver a visible terminal cancellation event and document the intentional cancellation behavior.
