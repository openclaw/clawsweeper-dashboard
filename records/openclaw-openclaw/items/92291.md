## What This Changes

Preserve an existing cron job’s `tz` and `staggerMs` when `cron edit --cron` changes only its expression.

## Best Possible Solution

Land the linked focused fix after confirming the real CLI/gateway path preserves omitted cron metadata, explicit overrides still win, and `at` or `every` edits continue replacing the schedule normally.

## Solution Assessment
