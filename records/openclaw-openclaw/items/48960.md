## Summary

Keep open: the underlying silent malformed-skill diagnostic gap still exists on current main, but this PR is not merge-ready because its diagnostics metrics and tests are wrong, it lacks real behavior proof, and it raises concrete logging-safety concerns.

## What This Changes

The PR adds structured skill-loading diagnostics and new tests around malformed `SKILL.md` handling in `src/agents/skills/workspace.ts`.

## Best Possible Solution

Use a shared loader/status diagnostics contract that reports skipped malformed skill candidates through CLI/status/UI surfaces, with redacted and bounded logging as supporting evidence rather than the only user-facing path.

## Reproduction Assessment

Yes. Source inspection shows malformed local `SKILL.md` candidates return `null` in `loadSingleSkillDirectory`, and the status builder only maps successfully loaded entries.

## Solution Assessment

No. The PR moves in a useful direction, but the mergeable solution should fix the metric/test defects, redact and bound logs, and align with the current loader/status diagnostic path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Log the raw pre-dedupe skill total:** `src/agents/skills/workspace.ts:597`
  - body: The PR computes `totalSkillsLoaded`, but the summary still emits `totalSkills: skillEntries.length`. Since `skillEntries` comes from the deduped map, `totalSkills` always matches `uniqueSkills`, leaving the duplicate-removal diagnostic misleading.
  - confidence: 0.95
- **[P2] Assert subsystem logger strings:** `src/agents/skills/workspace.test.ts:49-52`
  - body: These tests spy on object-shaped `console.warn`/`console.info` calls with a `message` field, but `createSubsystemLogger` formats console output as strings before writing. The assertions can fail or miss the real diagnostics even when logging works.
  - confidence: 0.92
- **[P2] Count skipped candidates by directory outcome:** `src/agents/skills/workspace.ts:477`
  - body: The discovery summary subtracts loaded `Skill` objects from candidate directory count. One directory can produce multiple skills, so `skipped` can become negative or misleading; track zero-skill candidate directories or parse failures separately.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff adds path-heavy warn/info diagnostics and per-candidate warnings that need redaction and bounding before merge.

Concerns:

- **[low] Redact path-heavy skill diagnostics:** `src/agents/skills/workspace.ts:441`
  - body: The new warnings and summary include `filePath` and `workspaceDir` values at warn/info level. In collected or shared logs, those paths can expose usernames, repository names, and internal directory layout.
  - confidence: 0.86
- **[low] Cap repeated malformed-skill warnings:** `src/agents/skills/workspace.ts:441`
  - body: A workspace with many malformed candidate directories can emit one warning per candidate on repeated loads. Aggregating or capping those warnings would preserve diagnostics without attacker-influenced log volume.
  - confidence: 0.78
