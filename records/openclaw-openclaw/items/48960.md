## Summary

Keep open: the no-diagnostics problem is real, but this PR still needs a current-loader refresh, corrected logger assertions, accurate summary metrics, current SKILL.md guidance, real behavior proof, and conservative logging/security choices before merge.

## What This Changes

The PR adds structured skill-loading diagnostics and tests for inaccessible roots, malformed SKILL.md files, frontmatter failures, discovery counts, and final load summaries.

## Best Possible Solution

Refresh the diagnostics on the current safe loader path, use YAML-frontmatter guidance, redact or compact path/error metadata, aggregate repeated malformed-skill warnings, and prove the behavior with a real skill-loading run plus focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows malformed or missing-description SKILL.md files return null in src/agents/skills/local-loader.ts and are filtered by src/agents/skills/workspace.ts without targeted parse-failure diagnostics; this was not live-run in the read-only review.

## Solution Assessment

No. The direction is useful, but this patch is not the best current solution because it targets stale loader code, asserts the wrong logger output shape, reports misleading metrics, gives stale format guidance, and logs sensitive/repeatable details without mitigation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Assert subsystem logs as formatted strings:** `src/agents/skills/workspace.test.ts:49-52`
  - body: createSubsystemLogger writes warn/info output as formatted strings through raw console sinks, and info-level output goes through console.log, so these object-shaped console.warn/console.info assertions miss or fail the diagnostics they intend to prove.
  - confidence: 0.94
- **[P2] Report the raw skill total:** `src/agents/skills/workspace.ts:597`
  - body: totalSkills is set from skillEntries.length after duplicate names have already been merged, while duplicatesRemoved is computed from raw source counts. With duplicate skill names, the summary can report totalSkills equal to uniqueSkills while also reporting removed duplicates.
  - confidence: 0.92
- **[P2] Use the frontmatter-based SKILL.md hint:** `src/agents/skills/workspace.ts:445`
  - body: The warning tells users to add ## Description and ## Usage sections, but the current loader and docs require YAML frontmatter with at least name and description. Following this hint can still leave the skill unloaded.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff adds no dependency or code-execution path, but the new diagnostics expose local path details and can amplify repeated malformed-skill warnings.

Concerns:

- **[low] Redact path and parser details in skill logs:** `src/agents/skills/workspace.ts:560`
  - body: The new metadata includes filePath, workspaceDir, and String(error). Subsystem metadata can reach file logs and JSON console output, so collected logs may disclose usernames, repository names, and local directory layout.
  - confidence: 0.88
- **[low] Cap repeated malformed-skill warnings:** `src/agents/skills/workspace.ts:441`
  - body: A workspace or extra skill root can contain many malformed SKILL.md files, and the patch emits a warning for each malformed candidate on each load. Aggregate or cap these diagnostics so bad input cannot flood logs.
  - confidence: 0.82
