## Summary

Keep open: the diagnostics goal is still useful, but this PR is not mergeable as-is because the new tests assert the wrong logger shape, the summary metrics remain misleading, real behavior proof is missing, and the new logs need path/noise hardening.

## What This Changes

Adds structured skill discovery/load diagnostics in `src/agents/skills/workspace.ts` and a new workspace loader test file for malformed `SKILL.md` logging.

## Best Possible Solution

Retarget diagnostics onto the current skill loader/status path, keep normal logs redacted and bounded, fix the metric/test assertions, and prove the malformed-skill flow with redacted real CLI or gateway output.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough: the new tests assert object-shaped logger calls that the subsystem logger does not emit, and the summary log uses the deduped entry count for `totalSkills`.

## Solution Assessment

No. The direction is useful, but the current patch is not the best fix until it is rebased onto the current loader, uses safe bounded diagnostics, and makes its tests match the real logger contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Assert against emitted log strings:** `src/agents/skills/workspace.test.ts:101-108`
  - body: `createSubsystemLogger` emits formatted strings through `console.warn` and `console.log`; this test spies on `console.info` and filters for `call[0]?.message`, so it never observes the summary log and fails even when the diagnostic is emitted. Use the repo's raw-console capture pattern or assert against the emitted string sink.
  - confidence: 0.95
- **[P2] Log the raw pre-dedup skill total:** `src/agents/skills/workspace.ts:597`
  - body: The PR computes `totalSkillsLoaded`, but the summary still logs `totalSkills: skillEntries.length`. Since `skillEntries` comes from `merged.values()`, `totalSkills` stays equal to `uniqueSkills` and contradicts `duplicatesRemoved` whenever duplicate skills were removed.
  - confidence: 0.92
- **[P2] Keep skipped counts tied to skipped candidates:** `src/agents/skills/workspace.ts:477`
  - body: `childDirs` counts directories while `loadedSkills` counts returned skills, so `skipped: childDirs.length - loadedSkills.length` can go negative or mix skipped directories with loaded-but-truncated skills. Track skipped/failed candidate directories separately, or rename and compute the metric from a single unit.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff adds diagnostics that can expose absolute local paths and amplify malformed-skill log volume unless redacted and bounded.

Concerns:

- **[low] Redact absolute paths in normal skill logs:** `src/agents/skills/workspace.ts:441`
  - body: The new warn/info diagnostics include `filePath`, `workspaceDir`, and raw error strings in normal logs. These can disclose usernames, repository names, and internal directory layout when logs are collected or shared.
  - confidence: 0.86
- **[low] Cap per-candidate parse-failure warnings:** `src/agents/skills/workspace.ts:440`
  - body: A malformed `SKILL.md` can trigger one warning per candidate directory on every load. Aggregate or cap repeated parse-failure warnings so a large or attacker-controlled skills tree cannot flood console or file logs.
  - confidence: 0.82
