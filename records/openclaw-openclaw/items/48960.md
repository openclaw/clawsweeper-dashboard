## Summary

Keep this PR open: the diagnostics target a real source-visible gap, but the patch is not merge-ready because the new tests do not observe the subsystem logger correctly, one summary metric is still misleading, real behavior proof is missing, and the new logging needs a small security/privacy pass.

## What This Changes

The PR adds structured skills-loader diagnostics and tests for malformed `SKILL.md` discovery failures.

## Best Possible Solution

Rebase onto current main and add sanitized, capped malformed-skill diagnostics in the current loader path, with logger-aware regression tests and redacted runtime output showing the warning/summary on a real malformed `SKILL.md`.

## Reproduction Assessment

Yes, source-based: current main drops malformed or incomplete `SKILL.md` files without a parse-failure diagnostic, as shown by `local-loader.ts` and the workspace loader. I did not run a live repro because this review is read-only.

## Solution Assessment

No. The direction is useful, but this patch still has broken logger assertions, a misleading `totalSkills` field, missing real behavior proof, and unresolved path/log-volume concerns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assert subsystem logs using the real logger sink:** `src/agents/skills/workspace.test.ts:49-52`
  - body: These tests spy on `console.warn`/`console.info` and expect an object with `message`, but subsystem logs are silent by default in Vitest and, when enabled, are formatted as strings; info logs go through `console.log`. As written, the new diagnostics tests fail or do not actually prove the emitted log fields.
  - confidence: 0.93
- **[P2] Report the raw skill count before dedupe:** `src/agents/skills/workspace.ts:597-599`
  - body: `totalSkills` is still `skillEntries.length`, which is built from `merged.values()`, so it always matches `uniqueSkills`. The summary should use the raw per-source total that `duplicatesRemoved` is already derived from, otherwise the diagnostic hides duplicate removal.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff adds new warn/info logs for local skill paths and parse errors without redaction or capping.

Concerns:

- **[low] Redact paths and parser details in new skill logs:** `src/agents/skills/workspace.ts:441`
  - body: The new parse-failure and summary logs include `filePath`, `workspaceDir`, and raw `String(error)` at warn/info level. Those values can expose usernames, workspace names, or internal layouts in collected logs, so they should be compacted/redacted or limited to debug output.
  - confidence: 0.84
- **[low] Cap repeated malformed-skill warnings:** `src/agents/skills/workspace.ts:440`
  - body: A workspace can contain many malformed skill directories, and this change emits a warning per failed candidate on each load. Add a per-root cap or aggregate summary so a large or shared workspace cannot flood logs.
  - confidence: 0.8
