## Summary

Keep open. Current main still has the silent malformed-skill loading path this PR is trying to expose, but the proposed patch is not merge-ready because its logger tests assert the wrong console shape, the summary total is internally inconsistent, and the new diagnostics need a maintainer decision on path redaction and warning aggregation.

## What This Changes

The PR adds structured diagnostics and tests for skill directory resolution, malformed `SKILL.md` handling, frontmatter failures, per-source discovery summaries, and overall skill-loading counts.

## Best Possible Solution

Refresh the diagnostics on current main by returning structured skip/load diagnostics from the workspace/local-loader path, logging compact or redacted path details at appropriate levels, aggregating repeated malformed-skill warnings, reporting raw versus unique counts accurately, and testing through the subsystem logger’s real formatted output path.

## Reproduction Assessment

Yes. A workspace skill directory containing a `SKILL.md` without valid frontmatter or without a usable `description` follows `loadSingleSkillDirectory()` to `null` and is omitted by `loadSkillEntries()` without a warning on current main; this was verified from source in read-only mode.

## Solution Assessment

No, not as currently written. The PR targets the right surface, but the best fix is a current-main refresh with accurate metrics, redacted/capped log output, and logger tests that capture formatted subsystem strings rather than object-shaped console calls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Assert subsystem logs as formatted console lines:** `src/agents/skills/workspace.test.ts:49-52`
  - body: `createSubsystemLogger` writes formatted strings to the console sink, and info-level messages use `console.log`, not `console.info`. These assertions look for object arguments with a `message` field, so the new tests can fail or miss the summary log even when diagnostics emit correctly.
  - confidence: 0.93
- **[P2] Report the raw skill total in the summary:** `src/agents/skills/workspace.ts:597`
  - body: `totalSkills` is set to `skillEntries.length` after duplicate skill names have already been merged, so it always matches `uniqueSkills`. Use the raw source-count sum instead, otherwise duplicate-removal diagnostics are internally inconsistent.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or code-execution paths, but it introduces security-relevant logging of local path/error details and repeatable warning volume from filesystem-controlled skill entries.

Concerns:

- **[low] Redact path and parser details in skill logs:** `src/agents/skills/workspace.ts:595`
  - body: The new warn/info metadata includes `filePath`, `workspaceDir`, and `String(error)`. Subsystem metadata can be written to file logs and JSON console output, so this can disclose usernames, repo names, and local directory layout in collected logs.
  - confidence: 0.88
- **[low] Cap repeated malformed-skill warnings:** `src/agents/skills/workspace.ts:440`
  - body: A workspace or extra skill root can contain many malformed `SKILL.md` files, and the patch emits a warning for each malformed candidate on each load. Aggregate or cap these diagnostics so bad input cannot flood logs.
  - confidence: 0.8
