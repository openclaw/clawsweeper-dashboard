Full review comments:

- **[P2] Log the raw pre-dedupe skill total:** `src/agents/skills/workspace.ts:597`
  - body: The summary computes `totalSkillsLoaded`, but still logs `totalSkills: skillEntries.length`. Because `skillEntries` is built from the deduped map, this always matches `uniqueSkills` and makes the duplicate-removal diagnostic misleading.
  - confidence: 0.95
- **[P2] Assert subsystem logger strings:** `src/agents/skills/workspace.test.ts:49-52`
  - body: These tests spy on `console.warn`/`console.info` and expect object arguments with a `message` field, but `createSubsystemLogger` formats subsystem console output as strings before writing. The assertions can fail or miss the real diagnostics even when logging works.
  - confidence: 0.92
- **[P2] Count skipped candidates by directory outcome:** `src/agents/skills/workspace.ts:477`
  - body: The discovery summary subtracts loaded `Skill` objects from candidate directory count. One directory can produce multiple skills, so `skipped` can become negative or misleading; track zero-skill candidate directories or parse failures separately.
  - confidence: 0.86
