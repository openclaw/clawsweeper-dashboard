## Summary

Keep open: the linked diagnostics problem is still source-reproducible on current main, but this PR is not merge-ready because the branch is stale against the current safe loader path, has blocking diagnostic/test bugs, lacks real behavior proof, and needs conservative logging changes before merge.

## What This Changes

The branch adds structured skill-loading diagnostics and tests for inaccessible roots, malformed `SKILL.md` files, frontmatter failures, discovery counts, and final load summaries.

## Best Possible Solution

Refresh the diagnostics on the current safe loader path, use the YAML-frontmatter skill contract in user guidance, redact or compact path/error metadata, aggregate repeated malformed-skill warnings, and test through the subsystem logger capture path.

## Reproduction Assessment

Yes. Source inspection on current main shows malformed or missing-description `SKILL.md` files return `null` in `src/agents/skills/local-loader.ts` and are filtered by `src/agents/skills/workspace.ts` without targeted parse-failure diagnostics; I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is useful, but this patch is not the best current solution because it targets an older loader path, asserts the wrong logger output shape, reports misleading summary metrics, gives stale format guidance, and logs sensitive/repeatable details without mitigation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Assert subsystem logs as formatted strings:** `src/agents/skills/workspace.test.ts:49-52`
  - body: `createSubsystemLogger` writes warning output as formatted strings, and info-level output uses `console.log`, so these object-shaped `console.warn` assertions do not prove the diagnostic was emitted and can fail even when the logger works.
  - confidence: 0.94
- **[P2] Report the raw skill total:** `src/agents/skills/workspace.ts:597`
  - body: `totalSkills` is set from `skillEntries.length` after duplicate names have already been merged, while `duplicatesRemoved` is computed from raw source counts. With duplicates, the summary can claim `totalSkills` equals `uniqueSkills` while also reporting removed duplicates.
  - confidence: 0.92
- **[P2] Use the frontmatter-based SKILL.md hint:** `src/agents/skills/workspace.ts:445`
  - body: The warning tells users to add `## Description` and `## Usage` sections, but current docs and loader behavior require YAML frontmatter with at least `name` and `description`. Following this hint can still leave the skill unloaded.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff adds no dependency or code-execution path, but the new diagnostics expose local path/error details and can amplify repeated malformed-skill warnings.

Concerns:

- **[low] Redact path and parser details in skill logs:** `src/agents/skills/workspace.ts:560`
  - body: The new metadata logs `filePath`, `workspaceDir`, and `String(error)`. Subsystem metadata can reach structured console/file logs, so collected logs may disclose usernames, repository names, local layout, and parser details.
  - confidence: 0.88

  - body: A workspace or extra skill root can contain many malformed `SKILL.md` files, and the patch emits a warning for each malformed candidate on each load. Aggregate or cap these diagnostics so bad input cannot flood logs.
  - confidence: 0.82
