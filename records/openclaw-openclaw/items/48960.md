## Summary

Keep this PR open, but it is not mergeable as-is: the diagnostics direction is useful and overlaps an open canonical diagnostics request, while the branch still has misleading metrics, brittle logger tests, missing real behavior proof, security concerns around path-heavy logs, and conflicts with current main.

## What This Changes

The PR adds structured skill-loading diagnostics and tests for malformed workspace `SKILL.md` handling in `src/agents/skills/workspace.ts`.

## Best Possible Solution

Implement a shared skill discovery diagnostics contract that records skipped malformed candidates once, redacts and bounds supporting logs, and surfaces the same diagnostics through status, CLI list/check/validate, and the Control UI.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: malformed or incomplete `SKILL.md` files return `null` in the local loader, and current status output only receives successfully loaded entries.

## Solution Assessment

No. The PR is useful source work, but the mergeable solution should fix the metric/test defects, redact and cap logs, and align with the broader shared diagnostics/status path rather than relying on path-heavy logs alone.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Log the raw pre-dedupe skill total:** `src/agents/skills/workspace.ts:597`
  - body: The summary computes `totalSkillsLoaded`, but still logs `totalSkills: skillEntries.length`. Because `skillEntries` is built from the deduped map, this always matches `uniqueSkills` and makes the duplicate-removal diagnostic misleading.
  - confidence: 0.95
- **[P2] Assert subsystem logger strings:** `src/agents/skills/workspace.test.ts:49-52`
  - body: These tests spy on `console.warn`/`console.info` and expect object arguments with a `message` field, but `createSubsystemLogger` formats subsystem console output as strings before writing. The assertions can fail or miss the real diagnostics even when logging works.
  - confidence: 0.92
- **[P2] Count skipped candidates by directory outcome:** `src/agents/skills/workspace.ts:477`
  - body: The discovery summary subtracts loaded `Skill` objects from candidate directory count. One directory can produce multiple skills, so `skipped` can become negative or misleading; track zero-skill candidate directories or parse failures separately.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff adds path-heavy warn/info diagnostics and per-candidate warnings that need redaction and bounding before merge.

Concerns:

- **[low] Redact path-heavy skill diagnostics:** `src/agents/skills/workspace.ts:441`
  - body: The new diagnostics include `filePath` and `workspaceDir` values at warn/info level. In collected or shared logs, those paths can expose usernames, repository names, and internal directory layout.
  - confidence: 0.86
- **[low] Cap repeated malformed-skill warnings:** `src/agents/skills/workspace.ts:441`
  - body: A workspace with many malformed candidate directories can emit one warning per candidate on repeated loads. Aggregating or capping those warnings would preserve diagnostics without attacker-influenced log volume.
  - confidence: 0.78
