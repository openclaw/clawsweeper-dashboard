## Summary

Keep this PR open. Current main still silently drops malformed or missing skill roots in the relevant loader paths, so the diagnostics goal remains valid, but the PR head is not merge-ready because its new tests assert the wrong logger shape, its summary metrics still misreport pre-deduplication totals, and the added logs need path/detail sanitization and volume controls.

## What This Changes

The PR adds skills-loading diagnostics and tests for silent failures, including missing/inaccessible skill roots, malformed `SKILL.md` files, frontmatter errors, per-source discovery summaries, and an overall skills-load summary in `src/agents/skills/workspace.ts`.

## Best Possible Solution

Refresh the diagnostics on current main by having the workspace/local-loader path return structured skip diagnostics, logging compact or redacted path details at appropriate levels, aggregating repeated parse failures, reporting raw versus unique counts accurately, and testing through the subsystem logger’s actual formatted output path.

## Reproduction Assessment

Yes. A workspace skill directory containing a `SKILL.md` without valid frontmatter or without a usable `description` follows `loadSingleSkillDirectory()` to `null` and is omitted by `loadSkillEntries()` without a warning; this is source-reproducible from the current loader paths, though no filesystem test was run because this review is read-only.

## Solution Assessment

No, not as currently written. The PR targets the right surface, but the best fix is a current-main rebase with accurate counts, sanitized/capped log output, and logger tests that capture formatted subsystem strings instead of object-shaped console calls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Assert subsystem logs as formatted console lines:** `src/agents/skills/workspace.test.ts:49-52`
  - body: `createSubsystemLogger` writes formatted strings to the console sink, and info-level messages go through `console.log`, not `console.info`. These assertions look for object arguments with a `message` field, so the new tests can fail or miss the summary log even when the diagnostics emit.
  - confidence: 0.93
- **[P2] Report the raw skill total in the summary:** `src/agents/skills/workspace.ts:595-599`
  - body: `totalSkills` is set to `skillEntries.length` after the merge map has already removed duplicates, so it always matches `uniqueSkills`. Use the raw source-count sum for `totalSkills`; otherwise duplicate removal diagnostics are internally inconsistent.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or code-execution paths, but it does introduce security-relevant logging of local path/error details and repeatable warning volume from filesystem-controlled skill entries.

Concerns:

- **[low] Redact path and parser details in skill logs:** `src/agents/skills/workspace.ts:595`
  - body: The new warn/info metadata includes `filePath`, `workspaceDir`, and `String(error)`. Subsystem metadata is written to file logs and JSON console output, so this can disclose usernames, repo names, and local directory layout in collected logs.
  - confidence: 0.88

  - confidence: 0.8
