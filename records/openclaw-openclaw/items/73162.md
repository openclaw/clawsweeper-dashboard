## Summary

This PR should stay open: current main and the latest release still have the finite Slack Socket Mode reconnect cap, while the PR is a narrow and correct fix for the linked Slack dead-connection bug.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited retries and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow reconnect-cap removal, or an equivalent maintainer replacement, and track reconnect-window message loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where recoverable Slack disconnect or start-failure events hit the 12-attempt guard, and the linked issue includes matching finite-retry logs plus after-mitigation `1/∞` logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because existing guards and log formatting already define non-positive attempts as unlimited, while non-recoverable Slack auth failures still fail fast.
