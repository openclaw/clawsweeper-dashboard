## Summary

Keep open. Current main still has the Slack Socket Mode 12-attempt fallback cap, this PR is the narrow fix for the linked Slack outage, and I found no blocking code or security finding in the diff.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited retries and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land or rebase this narrow policy change and changelog after normal CI and mergeability gates, while tracking reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated Slack disconnect or start-failure events hit the 12-attempt guard and throw, and the linked issue includes matching field logs plus an installed-policy mitigation showing the unlimited retry behavior.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix for the permanent cap-exhaustion failure because the current loop already treats zero as unlimited, keeps 30s-capped abortable backoff, and still fails fast for permanent Slack auth errors.
