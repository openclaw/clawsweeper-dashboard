## Summary

Keep open: the Slack reconnect fix is still needed on current main and the one-line runtime change is technically sound, but the PR is incomplete because it lacks the required user-facing changelog entry.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited retries by setting `SLACK_SOCKET_RECONNECT_POLICY.maxAttempts` to `0`.

## Best Possible Solution

Land the unlimited-retry policy with a single-line changelog entry, and track disconnected-status or provider-supervisor hardening separately if needed.

## Reproduction Assessment

Yes. Repeated Slack Socket Mode disconnect or start-failure events drive `reconnectAttempts` to the current 12-attempt guard, and #72808 includes field logs matching that retry/exhaustion path.

## Solution Assessment

Yes for the reconnect-cap bug: `maxAttempts: 0` is the narrowest maintainable runtime fix because the provider loop already defines zero as unlimited and permanent Slack auth failures still fail fast. The PR is not merge-ready until the changelog entry is added.
