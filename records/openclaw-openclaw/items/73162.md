## Summary

Keep open. Current main still caps Slack Socket Mode reconnect attempts at 12, while this PR is a narrow fix for the linked silent Slack disconnect path with no blocking code findings.

## What This Changes

This PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited attempts and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow reconnect-cap removal, or an equivalent maintainer replacement, then track reconnect-window message loss as a separate follow-up if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where recoverable Slack disconnect or start-failure events hit the 12-attempt guard, and https://github.com/openclaw/openclaw/issues/72808 includes matching finite-retry logs plus after-mitigation `1/∞` logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because existing guards and log formatting already define non-positive attempts as unlimited, while non-recoverable Slack auth failures still fail fast.
