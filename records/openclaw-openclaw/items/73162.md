## Summary

Keep open. Current main and the latest release still cap Slack Socket Mode reconnects at 12 attempts, while this PR is a narrow, source-backed fix with linked redacted runtime logs showing the unlimited-retry behavior after mitigation.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow cap removal, or an equivalent maintainer patch, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and linked field logs match that failure mode.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` uses the existing non-positive-as-unlimited guard and retry-text convention while leaving non-recoverable Slack auth errors on the fail-fast path.
