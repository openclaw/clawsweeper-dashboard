## Summary

Keep open. Current main still has Slack Socket Mode's 12-attempt reconnect cap, while this PR makes the existing zero-as-unlimited path the default; the linked field logs support the failure mode and I found no blocking correctness issue in the diff.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from `maxAttempts: 12` to `maxAttempts: 0` and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow Slack policy change, or an equivalent maintainer replacement, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack disconnect or start failures hit the 12-attempt guard and throw, and the linked issue logs show the finite retry behavior plus an unlimited-retry mitigation.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix because the loop already treats zero as unlimited, keeps abortable capped backoff, and still fast-fails permanent Slack auth errors.
