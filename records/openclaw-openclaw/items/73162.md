## Summary

Keep open. Current main and the latest release still retain the Slack Socket Mode 12-attempt cap, while this PR is a narrow and source-consistent fix for the linked permanent Slack disconnect failure.

## What This Changes

The PR changes Slack Socket Mode reconnects from 12 attempts to unlimited attempts by setting `SLACK_SOCKET_RECONNECT_POLICY.maxAttempts` to `0` and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Rebase and land this narrow Slack policy change, or an equivalent maintainer replacement, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack disconnect or start-failure events hit the 12-attempt guard and throw, and the linked issue has matching `retry 1/12` field logs plus a real mitigation log.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix for permanent cap exhaustion because the loop already treats zero as unlimited, keeps abortable capped backoff, and still fast-fails permanent Slack auth errors.
