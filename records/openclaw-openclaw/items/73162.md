## Summary

Keep open. Current main and the latest release still have the finite Slack Socket Mode reconnect cap, while this PR is a focused and correct fix for the linked Slack dead-connection bug.

## What This Changes

The PR changes Slack Socket Mode reconnects from a 12-attempt cap to unlimited retries and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow reconnect-cap removal, or an equivalent maintainer replacement, and track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack disconnect or start-failure events hit the 12-attempt guard and throw, and the linked issue includes matching field logs plus after-mitigation `1/∞` logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because existing guards and log formatting already define non-positive attempts as unlimited, while non-recoverable Slack auth failures still fail fast.
