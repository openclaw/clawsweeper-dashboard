## Summary

Keep this PR open: current main still has the finite Slack Socket Mode reconnect cap, and the proposed one-line policy change matches the existing zero-as-unlimited retry semantics while preserving permanent-auth-error fail-fast behavior.

## What This Changes

The PR changes Slack Socket Mode reconnects from 12 attempts to unlimited attempts and adds a changelog entry for the linked Slack disconnect bug.

## Best Possible Solution

Land the policy change and changelog after normal CI and mergeability checks, while tracking disconnected-status or provider-supervisor hardening separately if maintainers want that broader reliability work.

## Reproduction Assessment

Yes. Source inspection shows current main throws after 12 Slack socket reconnect attempts, and the linked field report/comments match that exhaustion path; I did not run a live Slack disconnect loop in this read-only pass.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix because the existing loop already treats zero as unlimited, uses abortable 30-second-capped backoff, and still fails fast for permanent Slack auth errors.
