## Summary

Keep open. Current main and v2026.5.12 still cap Slack Socket Mode reconnects at 12 attempts, while this PR is a narrow, source-aligned fix; the remaining blocker is branch conflict/maintainer refresh rather than cleanup closure.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from `maxAttempts: 12` to `maxAttempts: 0` and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Refresh and land this narrow policy change, or an equivalent maintainer patch, then track reconnect-window delivery loss and auth-error handling separately.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard, and linked field logs match that failure mode.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` uses the existing non-positive-as-unlimited guard and retry text convention while leaving non-recoverable Slack auth errors on the fail-fast path.
