## Summary

Keep open: this is a focused, still-relevant Slack bug-fix PR. Current main still has the finite retry-exhaustion path, the linked field logs corroborate it, and the proposed fix is narrow enough to review and land rather than close or replace with a separate repair branch.

## What This Changes

The branch changes Slack Socket Mode reconnect policy from a 12-attempt cap to unlimited retries and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow Slack policy change, or an equivalent maintainer replacement, then track gateway-level restart escalation and reconnect-window message loss separately if they remain reproducible.

## Reproduction Assessment

Yes. Current main has a source-visible path where recoverable Slack socket disconnects or start failures hit the 12-attempt guard and throw, and the linked issue includes matching finite-retry logs plus after-fix `1/∞` mitigation logs.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix because the loop already treats zero as unlimited, keeps abortable capped backoff, and still fast-fails permanent Slack auth errors; process-level escalation belongs to a separate follow-up.
