## Summary

Keep this PR open. The Slack reconnect cap bug is still present on current main, the PR is a narrow and mostly correct fix with real installed-runtime log proof, but it should also sync the Slack docs that still describe the old 12-attempt stop.

## What This Changes

The PR changes Slack Socket Mode reconnects from a 12-attempt cap to unlimited retries and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow cap removal with the Slack docs corrected to match unlimited recoverable retries, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and the linked issue includes matching field logs.

## Solution Assessment

Yes, with one docs follow-up. Setting `maxAttempts` to `0` is the narrow runtime fix because the consumer already treats non-positive values as unlimited while non-recoverable auth errors stay fail-fast.
