## Summary

Keep open: current main still has the Slack Socket Mode 12-attempt exhaustion path, and this focused PR addresses the linked outage; however the PR should update the Slack docs before merge because they still document the old 12-attempt stop behavior.

## What This Changes

This PR changes Slack Socket Mode reconnect attempts from 12 to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land the cap-removal patch after updating Slack docs to match unlimited recoverable retries, and track process-restart escalation or transient reconnect-window delivery loss separately.

## Reproduction Assessment

Yes. Current main has a source-visible path where recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and the linked outage discussion includes matching field logs.

## Solution Assessment

Mostly yes. Setting `maxAttempts` to `0` is the narrowest code change because the consumer already treats non-positive values as unlimited, but the merge-ready solution should also update the Slack docs that still describe the old finite cap.
