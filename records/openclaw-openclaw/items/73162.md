## Summary

Keep open: this is a focused Slack bug-fix PR with source and field-log support, but the current diff leaves public Slack docs saying Socket Mode stops after 12 attempts, so it needs a small docs update before merge.

## What This Changes

The PR changes Slack Socket Mode reconnects from a 12-attempt cap to unlimited attempts and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Update the Slack Socket Mode docs to match unlimited recoverable retries, land this cap removal, and keep process-level escalation or reconnect-window delivery loss in separate follow-up issues.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main throws after 12 recoverable Slack Socket Mode reconnect/start failures, and the linked issue contains matching field logs plus after-mitigation `1/∞` logs.

## Solution Assessment

Mostly yes. Setting `maxAttempts` to `0` uses the existing unlimited sentinel and preserves auth fail-fast behavior, but the submitted branch also needs the public Slack docs updated for the new retry semantics.
