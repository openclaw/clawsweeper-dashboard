Keep open. Current main still has the finite Slack Socket Mode reconnect cap, the linked issue has field logs matching that exhaustion path, and this PR is a narrow implementation path; before merge it still needs real after-fix behavior proof plus normal maintainer review.

## What This Changes

The PR changes Slack Socket Mode reconnect policy to `maxAttempts: 0` and adds a changelog entry for the linked Slack silent-disconnect fix.

## Best Possible Solution

Land this policy change or an equivalent current-main replacement after contributor proof and maintainer gates, while tracking transient message loss during reconnect windows separately.

## Reproduction Assessment

Yes. Source inspection shows repeated recoverable Slack disconnects or socket start failures can reach the twelve-attempt guard and throw, and the linked issue comments include matching field logs, though I did not run a live Slack disconnect loop.

## Solution Assessment

Yes. Setting the policy to `0` is the narrowest maintainable code change because the current loop already treats non-positive `maxAttempts` as unlimited and still fails fast for permanent Slack auth errors.
