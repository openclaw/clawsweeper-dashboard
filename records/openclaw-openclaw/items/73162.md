## Summary

Keep open. Current main and the latest release still have the 12-attempt Slack Socket Mode reconnect cap, while this focused PR removes that cap and has no blocking correctness finding; the remaining blockers are maintainer review, conflict resolution, and landing/proof-gate reconciliation.

## What This Changes

The PR changes Slack socket reconnect `maxAttempts` from 12 to 0 and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this reconnect-cap removal or an equivalent maintainer replacement, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack socket disconnects or start failures hit the 12-attempt guard and throw, and the linked issue includes matching field logs plus after-mitigation `1/∞` logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because existing guards and log formatting already define non-positive attempts as unlimited, while non-recoverable Slack auth failures still fail fast.
