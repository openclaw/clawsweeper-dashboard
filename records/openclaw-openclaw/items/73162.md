## Summary

Keep open: current main and the latest release still have the finite Slack reconnect cap, while this PR is a focused fix with source-backed behavior and real installed-mitigation logs.

## What This Changes

The PR changes Slack Socket Mode reconnect policy to `maxAttempts: 0` and adds a `CHANGELOG.md` fix entry for the Slack outage bug.

## Best Possible Solution

Land or rebase this narrow policy change after normal CI, while tracking reconnect-window delivery loss and zombie-WSS liveness separately.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack disconnects or start failures hit the 12-attempt guard and throw, and the linked issue includes matching field logs plus an installed unlimited-retry mitigation.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix for the permanent cap-exhaustion failure because the loop already treats zero as unlimited, keeps 30s-capped abortable backoff, and still fails fast for permanent Slack auth errors.
