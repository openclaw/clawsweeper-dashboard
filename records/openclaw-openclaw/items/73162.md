## Summary

Keep open. This is a focused Slack bug-fix PR whose central change is still needed on current main: Slack Socket Mode reconnects still stop after 12 recoverable failures, while the patch uses the existing non-positive/unlimited sentinel and preserves fail-fast handling for non-recoverable auth errors.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow cap removal or an equivalent maintainer patch, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and the linked issue includes matching field logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because the consumer already treats non-positive values as unlimited while non-recoverable auth errors stay on the fail-fast path.
