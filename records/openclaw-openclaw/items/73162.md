## Summary

Keep open: current main and the latest release still have the finite Slack Socket Mode reconnect cap, the linked user bug remains open, and this focused PR should proceed through maintainer review rather than cleanup closure.

## What This Changes

The branch changes Slack Socket Mode reconnects to use the existing unlimited-attempt sentinel and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow cap removal or an equivalent maintainer patch, then track reconnect-window message or slash-command loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and the linked issue includes matching field logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because the consumer already treats non-positive values as unlimited while keeping non-recoverable auth errors on the fail-fast path.
