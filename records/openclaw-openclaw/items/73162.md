## Summary

Keep open. Current main and the latest release still carry the Slack Socket Mode 12-attempt reconnect cap, while this focused PR matches the linked field logs and has no blocking code or security finding.

## What This Changes

This PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this cap removal or an equivalent maintainer patch, then handle transient reconnect-window message loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and the linked issue includes matching field logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because the consumer already treats non-positive values as unlimited while non-recoverable auth errors stay on the fail-fast path.
