## Summary

Keep open: current main and the latest release still ship the Slack Socket Mode 12-attempt cap, and this PR is a focused fix with source-visible reproduction plus linked field-log proof. The remaining issue is docs alignment and normal maintainer landing/rebase work.

## What This Changes

This PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this cap removal or an equivalent maintainer patch with docs aligned to unlimited retry behavior, then track reconnect-window delivery loss or process escalation separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and https://github.com/openclaw/openclaw/issues/72808 includes matching field logs.

## Solution Assessment

Mostly yes. Setting `maxAttempts` to `0` is the narrow functional fix because the consumer already treats non-positive values as unlimited while auth errors stay fail-fast; the Slack docs should be updated before landing.
