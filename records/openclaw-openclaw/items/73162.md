## Summary

Keep open: current main and v2026.5.7 still cap Slack Socket Mode reconnects at 12 attempts, while this focused PR removes that cap and has linked field-log proof. The patch looks correct, but it still needs maintainer review, rebase/conflict handling, and normal CI before merge.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog bullet for the linked Slack outage bug.

## Best Possible Solution

Land this narrow Slack policy change, or an equivalent maintainer replacement, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main still uses `maxAttempts: 12` and provider code throws after recoverable disconnect/start failures reach the cap, while the linked issue includes matching finite-retry logs and after-fix `1/∞` mitigation logs.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix because the loop already treats zero as unlimited, keeps abortable capped backoff, and still fast-fails permanent Slack auth errors.
