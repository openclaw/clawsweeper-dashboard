## Summary

Keep open: current main still has the Slack reconnect cap and linked field logs plus source inspection support the bug; this PR is the focused implementation candidate and I found no blocking correctness issue.

## What This Changes

This PR changes the Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow cap removal, or an equivalent maintainer patch, then track reconnect-window delivery loss and auth-error graceful-stop behavior in their separate Slack/gateway threads.

## Reproduction Assessment

Yes. Current main still has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and the linked issue includes matching field logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` uses the existing non-positive-as-unlimited guard and log convention while leaving auth fail-fast and broader reconnect-window hardening as separate work.
