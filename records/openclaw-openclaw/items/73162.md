## Summary

Keep open: current main still has the Slack Socket Mode 12-attempt reconnect cap, while this PR is a focused, proof-backed bug fix with no blocking correctness finding. The remaining action is maintainer review and landing validation, not cleanup closure.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage report.

## Best Possible Solution

Land this narrow cap removal, or an equivalent maintainer patch, then track reconnect-window delivery loss and auth-error graceful-stop behavior in their separate Slack/gateway threads.

## Reproduction Assessment

Yes, by source inspection and field logs: current main can hit the hardcoded 12-attempt guard after repeated recoverable Slack Socket Mode disconnects or start failures, and the linked issue includes matching retry logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` uses the existing non-positive-as-unlimited guard and log convention while leaving auth fail-fast and broader reconnect-window delivery hardening as separate work.
