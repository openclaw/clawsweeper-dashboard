## Summary

Keep open: the Slack reconnect-cap fix is focused and supported by current source plus field logs, but the PR should not land while the public Slack docs still state the old 12-attempt retry contract.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this reconnect-cap removal after aligning the Slack channel docs, then track reconnect-window delivery loss or process-level escalation in their separate issues.

## Reproduction Assessment

Yes, from source inspection and field logs. Current main reaches a 12-attempt guard on repeated recoverable Slack Socket Mode failures, and the linked issue includes redacted logs showing both finite `1/12` retries and after-mitigation `1/∞` retries.

## Solution Assessment

Mostly yes. The code change is the narrowest fix for permanent exhaustion, but the PR also needs the Slack channel docs updated so the published runtime contract matches unlimited retries.
