## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land or rebase this narrow Slack policy change after normal CI, while tracking reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. A high-confidence source path exists by driving repeated Slack Socket Mode disconnect or start-failure events until the 12-attempt guard throws, and the linked issue contains matching field logs plus a mitigation log.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix for permanent cap exhaustion because the current loop already treats zero as unlimited, keeps abortable capped backoff, and still fast-fails permanent Slack auth errors.
