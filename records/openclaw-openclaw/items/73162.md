## Summary

Keep open: current main and v2026.5.7 still ship the 12-attempt Slack Socket Mode cap, while source inspection and the linked field logs show recoverable reconnects can exhaust and leave Slack dead. The PR is a narrow valid fix awaiting maintainer review and normal landing validation.

## What This Changes

The branch changes Slack Socket Mode reconnects from a 12-attempt cap to unlimited retries and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow reconnect-cap removal, or an equivalent maintainer replacement, and track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack disconnect or start-failure events hit the 12-attempt guard and throw, and https://github.com/openclaw/openclaw/issues/72808 includes matching field logs plus after-mitigation `1/∞` logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because existing guards and log formatting already define non-positive attempts as unlimited, while non-recoverable Slack auth failures still fail fast.
