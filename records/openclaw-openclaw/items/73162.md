## Summary

Keep open. Current main still has the Slack Socket Mode reconnect cap that can permanently exhaust the channel, and this PR is the focused fix path for the linked Slack outage rather than a cleanup-close candidate.

## What This Changes

The PR changes Slack Socket Mode reconnect policy from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow cap removal, or an equivalent maintainer patch, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the 12-attempt guard and throw, and linked field logs match that failure mode.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` uses the existing non-positive-as-unlimited guard and infinity log convention while keeping non-recoverable Slack auth errors on the fail-fast path.
