## Summary

Keep open. Current main still has the Slack Socket Mode 12-attempt cap, while this PR is a focused, source-backed fix with sufficient linked runtime logs and no blocking review findings.

## What This Changes

The PR changes Slack Socket Mode reconnects from 12 attempts to unlimited and adds a changelog entry for the linked Slack outage bug.

## Best Possible Solution

Land this narrow cap removal or an equivalent maintainer patch, then track reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current main has a source-visible path where repeated recoverable Slack Socket Mode disconnects or start failures hit the hardcoded 12-attempt guard and throw, and the linked issue includes matching field logs.

## Solution Assessment

Yes. Setting `maxAttempts` to `0` is the narrowest maintainable fix because the consumer already treats non-positive values as unlimited while keeping non-recoverable auth errors on the fail-fast path.
