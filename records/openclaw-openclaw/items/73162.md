## What This Changes

The PR changes Slack Socket Mode reconnects from a 12-attempt cap to unlimited retries and adds a changelog entry for the linked lost-Slack-connection bug.

## Best Possible Solution

Rebase or otherwise resolve the dirty branch, land the policy change and changelog after CI, and track transient Slack not-ready delivery loss as separate reliability work if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows current main throws after twelve consecutive Slack Socket Mode reconnect attempts, and the linked field logs match that path; I did not run a live Slack disconnect loop in this read-only pass.

## Solution Assessment

Yes. Setting `maxAttempts: 0` is the narrowest maintainable fix because the existing loop already treats zero as unlimited, keeps abortable capped backoff, and still fails fast for permanent Slack auth errors.
