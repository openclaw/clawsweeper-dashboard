## Summary

Keep this issue open: current main and the latest release still prefer an attached gateway method registry before rebuilding from the live plugin registry, so the reported plugin RPC dispatch failure remains until the linked candidate PR lands.

## What This Changes

The issue reports that plugin-registered gateway RPC methods appear in runtime plugin inspection but `openclaw gateway call <method>` rejects them as unknown.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94154 or an equivalent dispatch-chokepoint fix that preserves attached-registry authorization while falling back to the live plugin registry for stale misses.

## Reproduction Assessment

Yes. Source inspection shows current main receives an attached startup registry in the WebSocket request path and selects it before rebuilding from the live plugin registry that `registerGatewayMethod` populates.

## Solution Assessment

Yes. The best fix path is in gateway dispatch: keep the attached registry when it owns the method, but fall back to a live registry rebuild when a stale attached snapshot misses a plugin-registered method.
