## Summary

Keep this issue open: current main and the latest release still use a cached gateway method registry when one is supplied, while plugin registration writes the live active registry. An open linked PR already carries a focused candidate fix, so this should stay open until one canonical fix merges.

## What This Changes

A plugin author reports that `openclaw gateway call <plugin-method>` cannot dispatch methods registered with `api.registerGatewayMethod`, even though runtime plugin inspection shows the method registered.

## Best Possible Solution

Land one canonical fix, likely https://github.com/openclaw/openclaw/pull/94154 or an equivalent branch, that removes the stale dispatch registry shortcut and proves a real plugin `gateway call` succeeds.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main passes a supplied startup `methodRegistry` into `handleGatewayRequest`, which then skips the live active plugin registry that `registerGatewayMethod` populates.

## Solution Assessment

Yes. The best fix path is in gateway dispatch: use the live request registry for plugin RPC handler lookup and keep a stale-registry regression test plus live CLI proof on the landing PR.
