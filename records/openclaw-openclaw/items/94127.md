## Summary

Keep this issue open: current main and the latest release still prefer a supplied Gateway method registry before rebuilding from the live plugin registry, so plugin-registered methods can still miss dispatch. The open linked PR https://github.com/openclaw/openclaw/pull/94154 is the canonical fix candidate and should resolve or be rejected before this issue closes.

## What This Changes

The issue reports that plugin-registered Gateway RPC methods shown by runtime inspect cannot be invoked through `openclaw gateway call` and return `unknown method`.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/94154 as the canonical dispatch fix, with end-user plugin install/restart/gateway-call proof before closing this issue.

## Reproduction Assessment

Yes. Source inspection shows the WebSocket path passes an attached method registry into `handleGatewayRequest`, which then prefers that snapshot over the live plugin registry that `registerGatewayMethod` updates; I did not run a live install/restart repro in this read-only review.

## Solution Assessment

Yes. The best fix belongs at the gateway dispatch chokepoint: use the attached registry when it owns the method and fall back to the live plugin registry for stale misses, which is the shape of the linked canonical PR.
