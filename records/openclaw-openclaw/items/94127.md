## Summary

Keep this issue open: current main and `v2026.6.8` still prefer a supplied gateway method registry snapshot before consulting the live plugin registry, while the linked canonical fix PR remains open and unmerged.

## What This Changes

The issue asks for plugin-registered gateway RPC methods to be reachable through `openclaw gateway call` when runtime inspection shows the method is registered.

## Best Possible Solution

Land one canonical gateway dispatch fix, likely https://github.com/openclaw/openclaw/pull/94154 or an equivalent branch, that preserves attached registries when they own the method and falls back to the live plugin registry for stale misses.

## Reproduction Assessment

Yes, at source level: external gateway calls pass a method-registry snapshot into `handleGatewayRequest`, and current main uses that snapshot before consulting the live plugin registry. I did not run the live plugin install/restart CLI path because this review is read-only.

## Solution Assessment

Yes: the narrow gateway dispatch fix is the best layer, as long as it keeps the attached registry authoritative when it owns the method and falls back to the live plugin registry only for stale misses.
