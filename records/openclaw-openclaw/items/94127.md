## Summary

Keep this issue open: current main and v2026.6.9 still select the caller-attached Gateway method registry before rebuilding from the live plugin registry, and the open linked PR is the active fix candidate with closing syntax.

## What This Changes

The issue reports that plugin methods registered with `api.registerGatewayMethod` appear in runtime inspection but fail through `openclaw gateway call` with `unknown method`.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/94154 as the canonical dispatcher fix, with end-user gateway-call proof or an explicit maintainer decision accepting chokepoint proof.

## Reproduction Assessment

Yes, at source level. Current main still passes an attached method registry into `handleGatewayRequest`, and that function still prefers the attached snapshot before rebuilding from the live plugin registry that `registerGatewayMethod` updates.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the Gateway dispatch chokepoint: preserve attached-registry dispatch when it owns the method and fall back to the live plugin registry for stale misses.
