## Summary

The patch adds a useful alias in `normalizeUsage`, yet it does not prove the reported derived prompt-token reduction for the described raw payload and misses streamed usage mapping.

## What This Changes

The PR adds MiniMax `prompt_cache_hit_tokens` to `UsageLike` and the `normalizeUsage` cache-read/OpenAI-style prompt-total detection chain in `src/agents/usage.ts`.

## Best Possible Solution

Land one canonical MiniMax usage-accounting fix that covers every raw usage ingress path, demonstrates the derived prompt/compaction behavior with focused tests, and includes redacted real MiniMax/OpenClaw proof.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows the alias is absent and stream chunks would drop a top-level cache-hit field, but the PR still lacks real MiniMax proof and a failing derived prompt-token path for the claimed compaction reduction.

## Solution Assessment

No. Adding the alias to `normalizeUsage` is a useful subset, but the best fix also needs stream ingress support, a regression that proves the prompt-token or compaction behavior changes, and real provider evidence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Prove the prompt-token reduction:** `src/agents/usage.ts:102-103`
  - body: For the raw MiniMax shape described by the PR, current main derives `prompt_tokens`; after this patch it derives `(prompt_tokens - prompt_cache_hit_tokens) + prompt_cache_hit_tokens`, which is the same prompt count. Please either fix the actual inflated path or add a before-failing/after-passing assertion around the real derived prompt-token or compaction behavior.
  - confidence: 0.86
- **[P2] Preserve cache hits in streamed usage:** `src/agents/usage.ts:90`
  - body: This adds `prompt_cache_hit_tokens` only to `normalizeUsage`, but OpenAI-compatible final stream chunks are converted first by `parseTransportChunkUsage()`, which only reads `prompt_tokens_details.cached_tokens`. A streamed MiniMax payload with the top-level field would still produce `cacheRead: 0`, leaving session and compaction accounting wrong on that ingress path.
  - confidence: 0.78
