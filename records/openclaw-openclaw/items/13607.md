## Summary

Keep open. Current main implements the core `agents.defaults.reasoningDefault` type/schema and main reply fallback, but Telegram and Feishu reasoning-preview helpers still bypass configured defaults, and the related fix PR is open and conflicting rather than merged.

## What This Changes

Request adds a global `agents.defaults.reasoningDefault` so new sessions inherit reasoning visibility across channels while preserving per-session `/reasoning` overrides.

## Best Possible Solution

Finish and land or supersede https://github.com/openclaw/openclaw/pull/71817 so channel preview helpers, `/reasoning` docs, and config help/label metadata all follow inline or session override, per-agent default, global default, then `off`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure `agents.defaults.reasoningDefault: "stream"` with no session override, then core helpers resolve `stream` while Telegram and Feishu preview helpers still return `off` or `false`.

## Solution Assessment

No for closing. The current core implementation is the right direction, but it is incomplete until channel preview resolution and user-facing docs/help metadata honor the same global-default precedence.
