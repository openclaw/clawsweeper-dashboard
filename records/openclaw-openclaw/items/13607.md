## Summary

Keep open. Current main has the core `agents.defaults.reasoningDefault` config, schema, reply fallback, and status plumbing, but Telegram and Feishu reasoning-preview paths still read only per-session state and otherwise fall back to off/false; the thinking docs also still omit the global default from the precedence description. Open PR #71817 is relevant partial channel work, not a merged superseding fix.

## What This Changes

Request a global `agents.defaults.reasoningDefault` setting so new sessions inherit reasoning visibility while `/reasoning` session overrides still win.

## Best Possible Solution

Finish the existing config feature by threading per-agent and global reasoning defaults through channel-specific preview resolution, updating the thinking docs and config help metadata, and landing or superseding #71817 before closing this issue.

## Reproduction Assessment

Yes. A source-level path is clear: configure `agents.defaults.reasoningDefault: "stream"` with no session override, then compare the core reply/status helpers that can resolve `stream` with Telegram and Feishu helpers that only inspect persisted session `reasoningLevel` and otherwise return off/false.

## Solution Assessment

No. Current main is the right core shape but not the complete cross-channel solution; the maintainable path is to amend or land #71817 with global fallback support and align the docs/help metadata afterward.
