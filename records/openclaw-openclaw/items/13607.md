## Summary

Keep open. Current main now has the core global reasoning default type/schema and main reply/status fallback, but the requested cross-channel behavior is still incomplete because Telegram and Feishu reasoning-preview helpers bypass configured defaults and the thinking/help metadata is not fully aligned.

## What This Changes

The issue requests a global `agents.defaults.reasoningDefault` setting so new sessions inherit a default reasoning visibility level while keeping per-session overrides.

## Best Possible Solution

Finish the feature by aligning channel preview helpers, `/reasoning` docs, and config help/label metadata with the existing precedence: inline or session override, per-agent default, global default, then `off`; land or supersede #71817 as part of that cleanup.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.reasoningDefault: "stream"` with no session override, then core helpers resolve `stream` while Telegram and Feishu preview helpers still return `off`/`false` from missing session state.

## Solution Assessment

No. The current core implementation is the right direction, but it is not the complete solution until channel preview resolution and user-facing docs/help metadata honor the same global-default precedence.
