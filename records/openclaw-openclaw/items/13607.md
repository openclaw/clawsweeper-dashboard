## Summary

Keep open. Current main now has the core `agents.defaults.reasoningDefault` config, schema, runtime fallback, status plumbing, and config docs, but the requested cross-channel behavior is not complete because Telegram and Feishu still resolve reasoning visibility only from per-session state and then fall back to off. The open #71817 PR is relevant partial channel work, not a superseding merged fix.

## What This Changes

The issue requests a global `agents.defaults.reasoningDefault` setting so new sessions inherit reasoning visibility across channels without per-session or per-agent setup.

## Best Possible Solution

Finish the feature by making every reasoning delivery path honor the same precedence: inline `/reasoning` and session override, then per-agent `agents.list[].reasoningDefault`, then global `agents.defaults.reasoningDefault`, then off/model fallback as appropriate. The remaining work should include Telegram/Feishu channel tests, a small audit of other channel-specific reasoning paths, and a docs update for `docs/tools/thinking.md` so the public precedence description matches current config behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure `agents.defaults.reasoningDefault: "stream"` with no session override, then compare the main reply path, which resolves the global default, with Telegram and Feishu helpers, which read only session-store reasoning state and otherwise return off/false.

## Solution Assessment

No, current main is the right core shape but not yet the complete solution requested by the issue. The safest path is to amend or land the relevant #71817 channel work with global fallback support, then close this after channel-specific bypasses and docs are aligned.
