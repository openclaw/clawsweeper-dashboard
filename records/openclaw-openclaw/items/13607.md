## Summary

Keep open. Current main has the core `agents.defaults.reasoningDefault` config, schema, main reply fallback, and status plumbing, but Telegram and Feishu still have channel-specific reasoning-preview paths that read only per-session state and otherwise fall back to off/false; `docs/tools/thinking.md` also still omits the global default from the public precedence description. The open #71817 PR is relevant partial channel work, not a merged superseding fix.

## What This Changes

The issue requests a global `agents.defaults.reasoningDefault` setting so new sessions inherit reasoning visibility across channels while per-session `/reasoning` overrides still win.

## Best Possible Solution

Finish the existing config feature by threading per-agent and global reasoning defaults into channel-specific reasoning resolution, audit other channel bypasses, and align the thinking docs with the final precedence.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure `agents.defaults.reasoningDefault: "stream"` with no session override, then compare the main reply/status helpers, which can resolve `stream`, with Telegram and Feishu helpers, which read only persisted session `reasoningLevel` and otherwise return off/false.

## Solution Assessment

No. Current main is the right core shape but not the complete cross-channel solution; the maintainable path is to amend or land #71817 with global fallback support, then close this after channel paths and docs agree.
