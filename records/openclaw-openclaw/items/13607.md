## Summary

Keep open. Current main has the core global `agents.defaults.reasoningDefault` type/schema and main reply/status fallback, but Telegram and Feishu reasoning-preview paths still bypass configured defaults and the thinking/help metadata is not fully aligned.

## What This Changes

The issue requests a global agent default for reasoning visibility so new sessions inherit a consistent `/reasoning` level while preserving per-session overrides.

## Best Possible Solution

Finish the feature by applying per-agent and global reasoning defaults in channel preview resolution, adding the missing config help/label metadata and thinking-doc precedence text, then land or supersede the related Telegram/Feishu PR before closing.

## Reproduction Assessment

Yes. A source-level path is clear: configure `agents.defaults.reasoningDefault: "stream"` with no session override, then compare the core reply/status helpers that resolve `stream` with Telegram and Feishu preview helpers that only inspect persisted session state and otherwise return off/false.

## Solution Assessment

No. Current main is the right core direction, but it is not the complete solution until channel preview fallbacks and user-facing docs/help metadata are aligned with the global default.
