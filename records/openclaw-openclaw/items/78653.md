## Summary

Keep open: this is a draft maintainer-labeled PR, and the patch still has a docs correctness blocker because the new on-demand AWS command implies multi-region capacity while current Crabbox only uses `capacity.regions` for Spot placement scoring.

## What This Changes

Updates Crabbox skill and CI docs with Blacksmith queued-outage cleanup guidance and changes the repo-owned Crabbox AWS fallback default from `beast` to `standard` with additional capacity regions.

## Best Possible Solution

Align the docs and skill with actual Crabbox behavior by either using Spot for regional scoring, adding Crabbox on-demand regional fallback first, or removing multi-region claims from on-demand fallback guidance.

## Reproduction Assessment

Yes for the review finding: the PR head documents `CRABBOX_CAPACITY_REGIONS` with `--market on-demand`, while current Crabbox source returns before region selection unless the market is `spot`. I did not perform a live AWS run.

## Solution Assessment

No. Reducing default `beast` usage is directionally source-backed, but the proposed on-demand multi-region fallback guidance is not correct until Crabbox supports that behavior or the docs stop claiming it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Fix the on-demand region guidance:** `docs/ci.md:586-587`
  - body: The new pressure-relief command sets `CRABBOX_CAPACITY_REGIONS` while also forcing `--market on-demand`, but current Crabbox only selects among `capacity.regions` in `chooseAWSRegion` when `Capacity.Market == "spot"`. Operators copying this during a regional on-demand quota/capacity problem will still launch in `.crabbox.yaml`'s `aws.region` (`eu-west-1`), so the documented multi-region fallback does not work. Either make the guidance use Spot for regional scoring, add Crabbox on-demand region fallback first, or remove the multi-region claim for on-demand.
  - confidence: 0.91
