## Summary

Keep this PR open for review: the cron/provider-schema direction is useful, but the patch currently normalizes shared runtime projection before collecting raw-schema diagnostics, which can hide unsupported schema branches from doctor and Codex dynamic-tool quarantine.

## What This Changes

The PR projects runtime cron tool schemas into provider-facing OpenAPI-friendly shapes, preserves raw validation for null clear sentinels, and refreshes Codex prompt snapshots.

## Best Possible Solution

Keep provider-facing schema normalization, but collect raw-schema diagnostics first and only return a normalized provider schema after unsupported runtime shapes are still quarantined and tested.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a schema with an `anyOf` string branch plus another branch containing `$dynamicRef` would be normalized to the string branch before diagnostics run, so the unsupported keyword would no longer be reported.

## Solution Assessment

No. The maintainable fix is to separate raw compatibility diagnostics from provider-facing schema projection instead of making the shared projection helper inspect only the normalized shape.
