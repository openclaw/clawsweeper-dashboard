Expose per-job model fallback overrides through `openclaw cron create/edit`.

## Best Possible Solution

Add a narrow CLI/docs change that maps `--fallbacks <list>` on cron create/edit to the existing agentTurn `payload.fallbacks` array without changing runtime fallback semantics.

Implement a narrow OpenClaw fix for https://github.com/openclaw/openclaw/issues/90302. Add `--fallbacks <list>` to `openclaw cron add`/`create` and `openclaw cron edit` for isolated/current/session agentTurn jobs, parsing comma- or whitespace-separated model refs consistently with `--tools`. On create, include `payload.fallbacks` only when the flag is explicitly provided; on edit, include `payload.fallbacks` in the agentTurn patch and allow it without `--message`, like `--model`/`--thinking`. Preserve existing systemEvent/main-session validation, delivery behavior, runtime fallback semantics, and Gateway protocol schemas. Document the CLI flag in `docs/automation/cron-jobs.md` and `docs/cli/cron.md`; do not edit CHANGELOG.md. Be explicit in tests about `--fallbacks ""`/empty-list behavior if supported, and do not solve null-clear/inherit semantics unless maintainers decide to fold in https://github.com/openclaw/openclaw/pull/67429.
