## Summary

Keep open: current `main` and the latest release still lack a `--fallbacks` flag on `openclaw cron add/create` and `openclaw cron edit`, while the cron runtime and Gateway payload contract already support per-job `payload.fallbacks`.

## What This Changes

Expose a `--fallbacks` CLI option on `openclaw cron create/add` and `openclaw cron edit` so users can set per-job model fallback overrides.

## Best Possible Solution

Add a narrow CLI/docs layer that maps `--fallbacks <list>` to existing agent-turn `payload.fallbacks` without changing Gateway or runtime fallback semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current `main` shows the CLI lacks `--fallbacks` while the Gateway payload and cron runtime already accept fallback arrays.

## Solution Assessment

Yes. The best solution is a narrow CLI and docs exposure of the existing payload field, avoiding runtime/protocol churn and treating clear/inherit semantics as separate work unless maintainers choose to include it.
