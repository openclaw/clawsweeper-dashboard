## Summary

Keep open: current main and the latest release still send manual cron completion through the same success-only `deleteAfterRun` gate, while the live closing PR is open and unmerged. This issue should remain attached to that PR instead of being closed or duplicated.

## What This Changes

The issue reports that manual `cron run` can silently consume and delete a `deleteAfterRun: true` one-shot cron job, losing the future scheduled fire.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/83933 as the canonical fix so manual verification cannot consume scheduled one-shot jobs.

## Reproduction Assessment

Yes. Source inspection shows manual completion calls `applyJobResult` without a manual discriminator, and an `ok` `at` job with `deleteAfterRun: true` is removed from the store.

## Solution Assessment

Unclear until maintainers accept the policy, but the linked PR's shared `applyJobResult` discriminator is the best current path. A duplicate fix PR would create churn while that closing PR remains open and proof-positive.
