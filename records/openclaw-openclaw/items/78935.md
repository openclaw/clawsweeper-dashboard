## Summary

Keep open: the report describes broken existing Nextcloud Talk reply behavior, and current source still has a source-reproducible path where a group-room incomplete-turn error can be produced but not delivered back to Nextcloud Talk under message-tool-only source suppression.

## What This Changes

The issue asks OpenClaw to fix a Nextcloud Talk regression where inbound messages reach the LLM but no reply is sent back to the Talk room.

## Best Possible Solution

Fix the shared source-suppressed delivery path so OpenClaw-generated runtime/incomplete-turn error payloads become visible through Nextcloud Talk while preserving message-tool-only privacy for ordinary assistant replies and preserving sendPolicy deny.

## Reproduction Assessment

Yes, at source level. A Nextcloud Talk group turn defaults to `message_tool_only`, the runner can return the reported incomplete-turn error payload, and dispatch skips final payloads lacking source-suppression delivery metadata before `sendMessageNextcloudTalk` can run.

## Solution Assessment

Yes, the maintainable direction is a narrow shared delivery fix rather than a Nextcloud-specific transport workaround. The exact patch still needs to preserve normal message-tool-only behavior and coordinate with https://github.com/openclaw/openclaw/issues/76980.
