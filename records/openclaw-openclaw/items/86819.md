## What This Changes

The issue asks OpenClaw to explain and handle a large gap between tracked `/context detail` prompt estimates and cached Codex gpt-5.5 context usage on fresh sessions.

## Best Possible Solution

Add Codex-aware `/context detail` attribution that separates last input, cache read/write, OpenClaw-tracked prompt, and confirmed native Codex overhead, then use only verified context occupancy for warnings or compaction decisions.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does confirm that current `/context detail` can show a generic residual from cached session tokens minus tracked prompt estimate, but the exact gpt-5.5 Codex OAuth baseline needs live provider proof.

## Solution Assessment

Unclear until Codex token-usage semantics are verified. The safer direction is not to subtract the residual blindly, but to attribute Codex native/cache components and prove whether they consume the active model window.
