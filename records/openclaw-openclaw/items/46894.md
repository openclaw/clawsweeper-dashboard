## What This Changes

The issue reports that Control UI `/model status` and `/model list` are interpreted as model IDs instead of informational `/model` subcommands.

## Best Possible Solution

Land or replace one canonical Control UI fix that routes `/model list` and `/model status` to the documented flows before model mutation, preserves normal model selection, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `/model status` and `/model list` reach `executeModel` as non-empty args, and the latest release tag has the same `sessions.patch` branch.

## Solution Assessment

Yes on the bug boundary, but not yet landed. The maintainable solution is a narrow Control UI executor change plus regression tests for both `/model status` and `/model list`, aligned with the shared directive handler and docs.
