## Summary

Keep open: current main and the latest release still route non-empty Control UI `/model` arguments through `sessions.patch`, while the docs define `/model list` and `/model status` as informational commands. The right next path is to resolve or replace the already-open fix PR at https://github.com/openclaw/openclaw/pull/46895 rather than closing this issue or queuing a duplicate repair.

## What This Changes

The issue asks for Control UI `/model status` and `/model list` to be handled as informational slash-command subcommands instead of being sent as literal model ids to `sessions.patch`.

## Best Possible Solution

Land a narrow Control UI executor fix that routes `status`, `list`, and any intended informational aliases through the existing model-info path, with regression tests proving those inputs do not call `sessions.patch`.

## Reproduction Assessment

Yes. Source inspection of current main shows `/model status` and `/model list` enter the non-empty argument branch and call `sessions.patch` with the argument as `model`, matching the reporter's logged `openai-codex/status` and `openai-codex/list` errors.

## Solution Assessment

Yes. The narrow maintainable fix is to classify the documented informational `/model` subcommands before the patch branch and add regression tests; the existing open PR targets the right files but needs conflict resolution or replacement.
