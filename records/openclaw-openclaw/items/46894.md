## Summary

Keep open. Current main still routes non-empty Control UI `/model` arguments through `sessions.patch`, so `/model status` and `/model list` still match the reported failure path; the linked implementation PRs are open/conflicting and should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks for Control UI `/model status` and `/model list` to be handled as slash-command subcommands instead of literal session model names.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` through the documented informational behavior, preserves normal model selection, and adds focused regression coverage proving those tokens do not call `sessions.patch`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and `executeModel` calls `sessions.patch` for every non-empty `/model` argument.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI local slash-command executor and tests, aligned with the existing docs and shared directive behavior, not as a new command surface or config option.
