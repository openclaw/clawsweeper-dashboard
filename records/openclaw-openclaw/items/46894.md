## Summary

Keep open: current main and the latest release still send non-empty Control UI `/model` arguments through `sessions.patch`, and the issue is paired with open/stale fix attempts that have not landed.

## What This Changes

The issue reports that Control UI `/model status` and `/model list` are parsed as model names and sent to `sessions.patch` instead of being handled as informational slash-command subcommands.

## Best Possible Solution

Land or replace the linked Control UI fix so `/model list` and `/model status` route before model mutation and match the documented/shared command semantics; close this issue only after that PR merges.

## Reproduction Assessment

Yes at source level. Current main parses `/model status` as a non-empty `model` command arg, and `executeModel` sends every non-empty arg to `sessions.patch`.

## Solution Assessment

Yes on the bug boundary, but the existing open fixes are not yet the best landing path. The maintainable fix should align Control UI with the shared/docs distinction between bare `/model`, `/model list`, and `/model status`.
