## Summary

Keep open. Current main still routes non-empty Control UI `/model` arguments through `sessions.patch`, so `/model status` and `/model list` remain source-reproducible failures; the issue is also paired with open implementation PRs that should be reviewed, revised, or replaced before this closes.

## What This Changes

Treat Control UI `/model status` and `/model list` as informational `/model` subcommands instead of passing them to `sessions.patch` as model names.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` through the documented informational behavior, preserves normal model selection, and adds focused regression coverage proving those tokens do not call `sessions.patch`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and `executeModel` calls `sessions.patch` for every non-empty `/model` argument.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI local slash-command executor and tests, aligned with the existing docs and shared directive behavior; the open PRs need revision or replacement before merge.
