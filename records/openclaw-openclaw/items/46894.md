## Summary

Keep open. ## What This Changes

The issue asks Control UI to treat `/model list` and `/model status` as informational slash-command modes instead of model ids passed to `sessions.patch`.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` to informational model flows before model mutation while preserving `/model <provider/model>` selection.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, dispatch preserves that value, and `executeModel` sends every non-empty `/model` argument to `sessions.patch`.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: handle documented `list` and `status` modes in the Control UI executor before the model mutation branch.
