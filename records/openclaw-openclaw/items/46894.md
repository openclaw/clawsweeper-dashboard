## Summary

Keep open. Current main still sends non-empty Control UI `/model` arguments to `sessions.patch`, so `/model status` and `/model list` still follow the reported failing path; the open implementation PRs at https://github.com/openclaw/openclaw/pull/46895 and https://github.com/openclaw/openclaw/pull/47069 should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks the Control UI to treat `/model status` and `/model list` as informational slash-command modes instead of model ids.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` through documented informational behavior, preserves normal model selection, and adds focused regression coverage proving those tokens do not call `sessions.patch`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `/model status` parses to `args: "status"`, the Control UI dispatch passes it unchanged, and `executeModel` calls `sessions.patch` for every non-empty argument.

## Solution Assessment

Yes. The maintainable fix is a narrow Control UI executor/test change aligned with the existing docs and shared directive handler, not a new command surface or config option.
