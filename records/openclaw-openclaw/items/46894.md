## Summary

Keep open. Current main and the latest release still route non-empty Control UI `/model` arguments through `sessions.patch`, so `/model status` and `/model list` remain source-reproducible against the documented slash-command contract; the linked implementation PRs are still open and should be reviewed, deduped, revised, or replaced before this issue closes.

## What This Changes

The issue asks for Control UI `/model status` and `/model list` to behave as informational slash-command modes instead of being submitted as model IDs.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented informational behavior, preserves normal model selection, and adds focused regression coverage plus redacted real Control UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, Control UI dispatch passes that string unchanged, and `executeModel` sends every non-empty `/model` argument to `sessions.patch`; I did not run a live Control UI session.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI local slash-command executor and tests, aligned with the shared directive behavior and public docs; the existing linked PRs should be revised, deduped, or replaced rather than closing this issue early.
