## Summary

Keep open. Current main still sends non-empty Control UI `/model` arguments through `sessions.patch`, so `/model status` and `/model list` remain source-reproducible failures against the documented slash-command contract; open linked fix PRs should be revised, deduped, or replaced before this issue closes.

## What This Changes

Fix the WebUI `/model status` and `/model list` slash-command handling so those tokens show informational/list output instead of being submitted as model ids.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented informational behavior, preserves normal model selection, and adds focused regression coverage plus real Control UI proof.

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, Control UI dispatch passes that string unchanged, and `executeModel` sends every non-empty `/model` arg to `sessions.patch`; I did not run a live Control UI session.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI local slash-command executor and tests, aligned with the existing shared directive behavior and public docs.
