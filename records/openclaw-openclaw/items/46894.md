## Summary

Keep open. Current main still routes Control UI `/model status` and `/model list` through the model-setting path even though docs and the shared directive handler define them as informational commands; linked PRs #46895 and #47069 already target this regression, so the issue should stay open until one lands or is replaced.

## What This Changes

The issue requests that Control UI treat `/model status` and `/model list` as informational slash-command subcommands instead of model names passed to `sessions.patch`.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `status`, `list`, and likely `info` through the existing model-info path, with regression coverage proving those inputs do not call `sessions.patch` while real model changes still work.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic path: `/model status` parses to command `model` with args `status`, app-chat forwards that arg unchanged, and `executeModel` calls `sessions.patch` for every non-empty arg.

## Solution Assessment

Yes. The narrow maintainable fix is in `ui/src/ui/chat/slash-command-executor.ts` with colocated tests, matching the existing docs and shared directive handler behavior.
