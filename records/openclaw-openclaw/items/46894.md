## Summary

Keep open. Current main still sends Control UI `/model status` and `/model list` through the model-setting path, while docs and the shared directive handler define them as informational commands; open PRs #46895 and #47069 already target this regression.

## What This Changes

The issue requests that Control UI `/model status` and `/model list` show model information instead of sending `status` or `list` to `sessions.patch` as model ids.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `status`, `list`, and likely `info` through the existing model-info path with regression tests proving those args do not call `sessions.patch`.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic path: `/model status` parses to command `model` with args `status`, app-chat forwards that arg unchanged, and `executeModel` calls `sessions.patch` for every non-empty arg.

## Solution Assessment

Yes. The narrow maintainable fix is in the Control UI slash-command executor with colocated tests; because linked PRs already target it, the issue should stay open until one lands or is replaced.
