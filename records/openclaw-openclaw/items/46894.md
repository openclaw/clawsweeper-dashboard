## Summary

Keep open. Current main and the latest release still source-reproduce the Control UI path where `/model status` and `/model list` are preserved as non-empty arguments and sent to `sessions.patch`; the open related pull requests are implementation candidates, not merged fixes.

## What This Changes

The issue reports that Control UI local `/model` slash commands treat `status` and `list` as model names and call `sessions.patch` instead of showing model information.

## Best Possible Solution

Land or replace one canonical Control UI fix that routes `/model list` and `/model status` to their documented informational flows before model mutation, preserves normal `/model <provider/model>` selection, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. `/model status` parses to `args: "status"`, Control UI preserves that argument, and `executeModel` sends every non-empty `/model` arg to `sessions.patch`; I did not run a live UI session.

## Solution Assessment

Yes. The requested boundary is the maintainable fix because docs and the shared directive path already define `list` and `status` as informational modes while `/model <provider/model>` remains the mutation path.
