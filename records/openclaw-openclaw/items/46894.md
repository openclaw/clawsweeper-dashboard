## Summary

Keep open. Current main still treats Control UI `/model` with any non-empty argument as a model change, while docs and the shared directive handler define `status` and `list` as informational model commands. Open PRs already target the exact regression, so the issue should remain open for maintainer review rather than cleanup closure.

## What This Changes

The issue requests that Control UI handle `/model status` and `/model list` as informational model commands instead of sending them to `sessions.patch` as model names.

## Best Possible Solution

Add a narrow Control UI executor guard that normalizes model arguments and routes documented informational subcommands such as `status` and `list` through the existing model-info path, with regression tests proving those inputs do not call `sessions.patch` while real `/model <provider/model>` changes still do.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: `/model status` parses as command `model` with args `status`, app-chat passes that to the local executor, and `executeModel` calls `sessions.patch` for any non-empty argument.

## Solution Assessment

Yes. The narrowest maintainable fix is in `ui/src/ui/chat/slash-command-executor.ts` plus its colocated tests; the shared directive handler and docs already describe the desired informational behavior.
