## Summary

Keep this open: current `main` and the latest release still route non-empty Control UI `/model` arguments such as `status` and `list` into `sessions.patch`, while docs and the gateway directive path treat those words as informational modes. The issue is also paired with open linked fix PRs, so cleanup should not close it or spawn a separate issue-level fix lane yet.

## What This Changes

The issue reports that WebUI `/model status` and `/model list` are interpreted as model names and sent to `sessions.patch` instead of showing model information.

## Best Possible Solution

Land a narrow Control UI executor fix that treats reserved model-info words such as `status`, `list`, and likely `info` as informational commands, with regression coverage proving they do not call `sessions.patch`.

## Reproduction Assessment

Yes. Source inspection shows current `executeModel` sends any non-empty `/model` argument, including `status` or `list`, to `sessions.patch`; I did not run the WebUI because this review is read-only and tests would create artifacts.

## Solution Assessment

Yes. The narrow maintainable solution is in the Control UI slash-command executor because `ui/src/ui/app-chat.ts` intercepts local commands before they reach the gateway, and the gateway sibling already has the desired `list`/`status` split.
