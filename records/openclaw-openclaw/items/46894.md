## What This Changes

The issue requests that Control UI treat `/model status` and `/model list` as informational slash-command modes instead of model-selection values.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented informational flows while preserving normal model selection, then close this issue after the linked fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, dispatch preserves that value, and `executeModel` sends every non-empty `/model` argument to `sessions.patch`.

## Solution Assessment

Yes, the requested direction is the right bug fix: Control UI should handle documented `list` and `status` modes before model mutation. The maintainable path is one canonical implementation aligned with the shared directive behavior, not multiple overlapping stale PRs.
