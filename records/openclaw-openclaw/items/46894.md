## Summary

Keep open. Current main still routes Control UI `/model status` and `/model list` through the model-setting path, while the docs and shared directive handler define those as informational model commands; open implementation PRs #46895 and #47069 already target this regression, so this issue should remain open until one lands or is replaced. https://github.com/openclaw/openclaw/issues/46894

## What This Changes

Treat Control UI `/model status` and `/model list` as informational slash-command subcommands instead of model IDs passed to `sessions.patch`.

## Best Possible Solution

Land or replace one narrow Control UI fix that normalizes model subcommand args and routes `status`, `list`, and possibly `info` through the existing model-info path, with regression tests proving those inputs do not call `sessions.patch` while real model changes still work.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic on current main: `/model status` parses to command `model` with args `status`, app-chat forwards that arg, and `executeModel` calls `sessions.patch` for every non-empty arg.

## Solution Assessment

Yes. The narrowest maintainable solution is a guard in `ui/src/ui/chat/slash-command-executor.ts` plus colocated tests; the shared directive handler and public docs already define the intended informational behavior.
