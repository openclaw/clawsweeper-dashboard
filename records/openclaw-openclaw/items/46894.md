## What This Changes

The issue requests that Control UI handle `/model status` and `/model list` as informational model slash-command modes instead of treating `status` or `list` as model ids.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented informational flows before model mutation while preserving `/model <provider/model>` selection.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: `/model status` parses to `args: "status"`, dispatch preserves it, and `executeModel` sends every non-empty `/model` arg to `sessions.patch`. I did not run a live Control UI session in this read-only sweep.

## Solution Assessment
