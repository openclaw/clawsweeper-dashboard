## Summary

Keep open. Current main still has the synchronous banner-config read that this PR targets, but the PR head is not mergeable because its cold-path cache stores `undefined` before the dynamic import can resolve and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR adds a `banner-config-lite.runtime.ts` dynamic-import boundary and rewrites CLI banner tagline-mode loading to use a process-wide lazy cache.

## Best Possible Solution

Land a corrected CLI banner-config fix that removes the heavy help-time import while preserving configured `cli.banner.taglineMode` values and `env`-scoped reads, with focused tests and a changelog entry.

## Reproduction Assessment

Yes for the PR regression. Source inspection plus the Node Promise timing check show the dynamic import callback cannot populate `resolved` before the cache assignment, so a cold call caches `undefined` for the process.

## Solution Assessment

No. The runtime-boundary direction is plausible, but this implementation should not cache the cold-path fallback as a completed value; use a real lightweight synchronous selector or an async cache that records the resolved result without regressing `env` semantics.
