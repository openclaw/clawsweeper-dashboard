## What This Changes

The PR moves CLI banner config loading behind a dynamic `banner-config-lite.runtime.ts` import and caches the banner tagline mode to avoid the `openclaw tui -h` help hang.

## Best Possible Solution

Land or replace this with a narrow CLI banner fix that removes the help-time heavy config import while preserving `cli.banner.taglineMode` and proving the help command exits in a real terminal.

## Reproduction Assessment

Yes for the PR regression: PR head source plus a Node dynamic-import timing probe show the fallback cache path is taken before callbacks can persist the resolved config. I did not execute the potentially hanging original command in this read-only review.

## Solution Assessment

No. The runtime-boundary direction is plausible, but this implementation caches the fallback before the promise result and needs a resolved-result cache or another lightweight reader that preserves config semantics.
