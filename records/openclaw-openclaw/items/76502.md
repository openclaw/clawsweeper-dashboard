## Summary

Keep open. Current main still has a broad backup payload policy that includes the resolved state directory and only filters installed plugin node_modules, so large generated state can still be archived; the related log-growth item does not cover backup payload or retention policy.

## What This Changes

The issue asks OpenClaw backups and upgrades to preserve critical user state without duplicating large generated data such as logs, browser caches, and workspaces.

## Best Possible Solution

Define and document a conservative backup payload policy that keeps config, credentials, auth profiles, session state, and necessary plugin metadata while excluding or pruning generated runtime data by default.

## Reproduction Assessment

Yes, partially from source: `resolveBackupPlanFromPaths` includes the state directory and `createBackupArchive` passes that tree to `tar.c`, with only `extensions/*/node_modules` filtered. I did not reproduce the legacy automatic `~/.openclaw_backup_YYYYMMDD` directory creation path on current main.

## Solution Assessment

Unclear pending maintainer policy. The safest solution is a documented durable-vs-generated state policy, not a one-off logs exclusion that could miss browser, workspace, memory, or future generated payloads.
