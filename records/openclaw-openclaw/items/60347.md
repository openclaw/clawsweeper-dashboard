## Summary

Keep open. Current main does not implement GPU backend selection for local embeddings, and this PR is still incomplete because the accepted config value would not be propagated through the gateway memory config resolver; the production-validation comment is useful context but does not include inspectable after-fix proof.

## What This Changes

Adds an optional `memorySearch.local.gpu` config field and attempts to pass it through to node-llama-cpp `getLlama()` for local embedding inference.

## Best Possible Solution

Refresh the branch so `memorySearch.local.gpu` is a complete public config contract: type/schema/help/docs, resolver propagation, node-llama wrapper typing, provider pass-through, and focused regression tests, while leaving broader Apple Silicon crash policy to https://github.com/openclaw/openclaw/issues/44202.

## Reproduction Assessment

Yes, for the PR bug. A source-level repro can configure `agents.defaults.memorySearch.local.gpu: "vulkan"`, call `resolveMemorySearchConfig()`, and observe that the resolved `local` object has no `gpu` field before provider creation.

## Solution Assessment

No, not as proposed. The direction is narrow and dependency-backed, but the patch must propagate the setting through the resolver and update the public config/docs surfaces before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread gpu through resolved memory settings:** `src/config/types.tools.ts:385`
  - body: Adding `memorySearch.local.gpu` to the accepted config and reading `options.local?.gpu` in the provider is not enough because gateway-created providers receive `resolveMemorySearchConfig().local`. That resolver currently builds `local` from only `modelPath`, `modelCacheDir`, and `contextSize`, so `gpu: "vulkan"` or `gpu: false` would be accepted by validation and then silently dropped before `getLlama()` is called.
  - confidence: 0.93
- **[P2] Document the public gpu config field:** `src/config/zod-schema.agent-runtime.ts:657-665`
  - body: The new runtime schema field is a user-facing config contract, but the PR does not add matching config help/labels or memory-config documentation for `memorySearch.local.gpu`. Schema/UI/docs consumers and users would still see only the existing local model/cache/context fields, so the branch should update those public surfaces and regenerate the config docs baseline from current main.
  - confidence: 0.86
