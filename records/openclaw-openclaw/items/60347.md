## Summary

Keep open. Current main still lacks a local-embedding GPU backend selector, but this PR is not merge-ready because the branch does not thread the option through the current resolver and active wrapper surfaces and still lacks inspectable real behavior proof.

## What This Changes

The PR adds an optional `memorySearch.local.gpu` config field, runtime validation, a `getLlama` GPU passthrough, ambient type updates, and a generated config-baseline checksum change.

## Best Possible Solution

Land a rebased implementation that exposes an upstream-supported backend selector through schema, help, labels, docs, resolver, provider option types, the active node-llama wrapper, and focused tests while preserving existing `contextSize` behavior.

## Reproduction Assessment

Yes for the PR defects by source inspection: current main shows the resolver and active wrapper paths where `gpu` must be threaded before it can affect local embeddings. No live GPU runtime proof artifact is available.

## Solution Assessment

No, not as currently proposed. The backend selector direction matches the upstream node-llama-cpp contract, but the branch needs a current-main rebase across resolver, active wrapper, public config surfaces, tests, and real proof.
