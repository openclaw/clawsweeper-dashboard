Keep open: current main has useful partial guardrails, including opt-in rolling loop detection, default-on post-compaction loop aborts, and the merged generic no-progress block, but it still does not implement the requested arbitrary-tool rate-limit cooldown, error-category stop policy, default hard-stop behavior, escalation counter, or user retry confirmation flow.

## What This Changes

Request hard platform-level circuit breakers for repeated failing tool calls, rate-limit cooldowns, error-category retry policy, escalation counters, and user override prompts.

## Best Possible Solution
