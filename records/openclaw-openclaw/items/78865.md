Keep open: current main has partial loop guardrails, but it still lacks the requested arbitrary-tool rate-limit cooldown, error-category policy, user retry confirmation, and cross-tool failure escalation behavior.

## Best Possible Solution

Define a maintainer-approved retry safety policy on top of the existing loop-detection, before-tool-call, diagnostics, and approval seams, then split or land focused implementations for cooldowns, classifications, and confirmation UX with tests and docs.
