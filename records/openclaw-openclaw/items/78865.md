## Summary

Keep open: current main has opt-in tool-loop detection and a default-on post-compaction guard, but the requested default hard stop, rate-limit cooldown, error-category handling, and user retry confirmation are not fully covered.

## What This Changes

Request platform-level circuit breakers that detect repeated failing tool calls, pause rate-limited or fatal tools, classify errors, and force user confirmation after repeated failures.

## Best Possible Solution

Define a maintainer-approved tool retry safety policy on top of the existing loop-detection and before-tool-call seams, including default behavior, 429/fatal classification, cooldown state, override UX, and focused regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for stronger default platform behavior. Source inspection does confirm the current rolling-history guard is opt-in and does not show arbitrary-tool rate-limit cooldown or user retry confirmation.

## Solution Assessment
