Keep open. The report is fresh and detailed, and current main only shows partial mitigations after v2026.4.29 rather than proof that the ARM64 Gateway CPU/event-loop stalls are fixed end to end.

## What This Changes

The issue reports ARM64 Linux Gateway CPU saturation and 37-40s event-loop stalls during embedded-run prep in v2026.4.29, with plugin runtime dependencies, system-prompt construction, stream setup, and cron amplification called out as suspected causes.

## Best Possible Solution

Keep this issue open as part of the Gateway CPU/per-run prep cluster, validate current main on the reported ARM64 profile, and close only after telemetry shows core-plugin-tools, system-prompt, and stream-setup no longer block the event loop for this configuration.
