Full review comments:

- **[P1] Avoid double-emitting native progress final replies:** `extensions/slack/src/monitor/message-handler/dispatch.ts:897-899`
  - body: In native-progress mode a progress stream can already exist, but the final payload is still delivered through `deliverNormally` below, which will emit `message_sent` via `deliverReplies`. Because `streamedFinalContent` is set here before that branch, the finalizer later emits a second success event for the same final answer when `stopSlackStream` succeeds. Only capture content for the actual text-stream path, or clear it after the native-progress normal delivery path, and add a regression test.
  - confidence: 0.93
- **[P2] Pass the session key into sent hook context:** `extensions/slack/src/message-sent-hook.ts:39-50`
  - body: `sessionKeyForInternalHooks` is forwarded from dispatch, but `buildSlackSentHookContext` does not pass it as `sessionKey` to `buildCanonicalSentMessageHookContext`. The mapper only copies `sessionKey` into the plugin event/context when it is on the canonical object, and the shared outbound emitter explicitly does that, so Slack hook consumers still cannot correlate `message_sent` with session-scoped hooks. Add `sessionKey: params.sessionKeyForInternalHooks` and assert it in the new hook tests.
  - confidence: 0.9
