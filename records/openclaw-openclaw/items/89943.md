Full review comments:

- **[P1] Avoid double-emitting for native progress replies:** `extensions/slack/src/monitor/message-handler/dispatch.ts:897-899`
  - body: In native-progress mode, the final payload is delivered through `deliverNormally`, which now emits `message_sent`, but this earlier assignment still records the same final text for the stream finalizer. When `stopSlackStream` later succeeds, the finalizer emits another `message_sent` for the same reply, so hook observers can double-count one delivered Slack response. Only record streamed final content for the partial native stream path, or clear it before the native-progress normal-send branch returns.
  - confidence: 0.88
- **[P2] Pass the session key into Slack sent-hook context:** `extensions/slack/src/message-sent-hook.ts:38-50`
  - body: `sessionKeyForInternalHooks` is available here, but the canonical context never receives it as `sessionKey`, so plugin `message_sent` handlers get an event/context that cannot correlate with the earlier `message_sending` context or `agent_end` for the same route. The shared outbound emitter explicitly preserves this field; the Slack helper should do the same and test it.
  - confidence: 0.86
