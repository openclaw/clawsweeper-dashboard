## Summary

Keep open. Current main still treats interactive onboarding provider/model setup as a single successful provider pass, and the fresh follow-up proposal confirms this is a new onboarding UX capability that needs maintainer scoping rather than conservative cleanup closure.

## What This Changes

The issue requests that `openclaw onboard` let users configure multiple providers/models in one interactive setup flow, choose defaults or role-specific models, and show a final summary.

## Best Possible Solution

Define a scoped onboarding provider stage that can repeat auth/custom-provider setup, then map selected model refs into the existing primary, fallback, and role-specific config fields with a clear final summary.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection confirms the current interactive provider setup exits after one successful provider path.

## Solution Assessment
