Keep open. Current main still exits interactive onboarding after one provider/default-model pass, and the remaining request is fresh product/UX work that overlaps open related issues without a maintainer consolidation signal.

## What This Changes

The issue requests an interactive `openclaw onboard` flow that can configure multiple providers/models, choose defaults or roles, persist them consistently, and show a final summary.

## Best Possible Solution

Define a scoped onboarding provider stage that repeats only interactive prompt-driven provider setup, then maps selected models into the existing primary, fallback, allowlist, and role-specific config contracts with docs and focused wizard tests.
