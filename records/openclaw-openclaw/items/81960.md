## Summary

Keep open. Current `main` and latest release still run classic onboarding through one successful provider/model setup pass, the older broad issue was closed in favor of this issue, and the only linked implementation PR is closed unmerged, so this remains the canonical product-scoping item.

## What This Changes

Issue requests extending interactive `openclaw onboard` so one setup run can configure multiple providers/models, choose defaults or roles, persist them, and show a final summary.

## Best Possible Solution

Keep this issue as the canonical CLI onboarding request, then have maintainers scope a bounded implementation that reuses existing provider auth/model-picker helpers, preserves non-interactive compatibility, localizes prompts, and decides which role-selection pieces belong in the first PR.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current main and v2026.6.8 still use a single-pass provider/model onboarding loop.

## Solution Assessment

Unclear until maintainers choose the product scope. The narrow repeat-loop proposal is plausible, but the maintainable solution should reuse existing provider auth/model-picker helpers, localize any prompt, and decide whether role assignment and final summary belong in the same change.
