## Summary

Keep open. Current main still runs interactive onboarding through one provider/default-model pass, and this fresh request needs maintainer UX scoping because it affects first-run prompts, default/fallback policy, and related open multi-model/provider work.

## What This Changes

Request to extend interactive `openclaw onboard` so one run can configure multiple providers/models, choose role/default models, persist them, and show a final summary.

## Best Possible Solution

Define a scoped interactive onboarding provider stage that can repeat prompt-driven provider setup and reuse existing primary, fallback, allowlist, and provider config contracts, while keeping Dashboard provider management separate.

## Solution Assessment

Unclear pending maintainer scoping. The proposal is a plausible first slice, but the maintainable fix should reuse existing allowlist/fallback helpers and current provider schema rather than only adding a repeat prompt.
