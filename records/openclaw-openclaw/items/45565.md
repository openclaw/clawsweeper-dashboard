## Summary

Keep open: current main and v2026.6.6 only provide suppression and cron-specific failure routing, not a supported way to route gateway lifecycle or internal warning categories to a dedicated channel.

## What This Changes

Request adds a messages-level config option to route gateway lifecycle, tool-error, and internal warning messages away from the active conversation channel.

## Best Possible Solution

Define one canonical warning-routing policy for lifecycle, tool-error, and system-notice messages, then implement it through shared warning delivery, schema, docs, and focused compatibility tests while preserving current default behavior.

## Reproduction Assessment

Yes at source level, but not as a live Discord replay. Current main lacks the requested config keys and still builds warning payloads that flow through the existing reply route.

## Solution Assessment

Unclear: a one-off lifecycleTarget key is plausible but likely too narrow. A maintainer-approved category taxonomy should come first so lifecycle, tool-error, cron, and system-notice behavior does not drift.
