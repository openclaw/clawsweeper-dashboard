## Summary

Keep open: current main still has suppression-only warning controls and no configured internal-warning route target; the remaining work is a user-facing config/product design, not cleanup.

## What This Changes

Request a messages config option to route gateway lifecycle and internal warning notifications to a dedicated channel, with possible catch-all, per-category, or suppress semantics.

## Best Possible Solution

Define a documented, validated warning-routing policy in core messages config with category defaults, route/suppress semantics, authorization/fallback behavior, and tests/docs for lifecycle and tool-error delivery.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows the proposed routing keys are absent and current warning payloads continue through normal reply delivery.

## Solution Assessment

Unclear for the exact key names. The maintainable direction is a shared category-aware core warning-routing policy, coordinated with existing tool-warning suppression work, rather than a Discord-specific special case.
