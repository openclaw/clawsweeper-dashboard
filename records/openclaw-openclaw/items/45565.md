## Summary

Keep open: current main still lacks configurable routing for gateway lifecycle or internal warning notifications. The adjacent `messages.suppressToolErrors` control only suppresses some non-mutating tool warnings, while warning payloads still use normal reply delivery, so this remains a product/config feature request rather than conservative cleanup.

## What This Changes

The issue requests a config option to route or suppress gateway lifecycle and internal warning messages away from the active conversation channel.

## Best Possible Solution

Add a generic, category-aware internal-warning delivery policy in core config with validated route targets, backwards-compatible active-channel fallback, docs/schema/help updates, and regression coverage for lifecycle, tool-error, and system-notice delivery.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main confirms the proposed routing keys are absent and warning payloads still use ordinary reply delivery.

## Solution Assessment

Unclear for the exact proposed key names because the public API has not been chosen. The maintainable direction is a shared category-aware routing policy, not a Discord-only special case or silent suppression by default.
