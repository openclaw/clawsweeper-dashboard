## What This Changes

The issue requests a config option for routing or suppressing gateway lifecycle and internal warning messages away from the active conversation channel.

Define one canonical, default-compatible warning-routing policy with category semantics, config/schema/docs alignment, and regression coverage across shared tool/lifecycle warnings plus cron-specific delivery.
