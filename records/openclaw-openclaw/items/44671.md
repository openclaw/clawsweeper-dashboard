## Summary

Keep open: this is a real Docker setup bug, but the PR branch is stale against current main and still has correctness/proof blockers. It edits the removed root `docker-setup.sh`, while the active `scripts/docker/setup.sh` still rejects whitespace and still needs safe YAML encoding for all generated mounts.

## What This Changes

The branch relaxes Docker mount path validation in the root `docker-setup.sh` and emits generated volume mounts as single-quoted YAML scalars.

## Best Possible Solution

Implement whitespace-path support in `scripts/docker/setup.sh` with one YAML-safe mount writer for generated extra mounts and the sandbox socket overlay, then cover the active path with focused Docker setup proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a value with a space and run `scripts/docker/setup.sh`; current main sends it to `validate_mount_path_value`, which fails with the reported whitespace error.

## Solution Assessment

No. The requested direction is right, but this PR edits the removed root entrypoint and does not safely encode every generated Compose mount after relaxing validation.
