## Summary

Keep open. The linked Docker whitespace bug still exists on current main, but this PR is not merge-ready because it changes the stale full root setup script instead of the active `scripts/docker/setup.sh`, and the proposed YAML quoting misses valid apostrophe paths plus the sandbox socket overlay after relaxing validation.

## What This Changes

The PR relaxes Docker setup mount path validation to allow whitespace and emits generated Compose volume mounts as single-quoted YAML scalars in `docker-setup.sh`.

## Best Possible Solution

Implement whitespace-path support in the active Docker setup script with one reusable YAML scalar writer that preserves spaces and backslashes while escaping embedded apostrophes, use it for config/workspace/home/extra mounts and the sandbox socket overlay, keep the root wrapper intact, add focused e2e coverage, and add the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a path containing a space and current main reaches `validate_mount_path_value`, whose `[[:space:]]` branch fails before setup proceeds.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it targets the stale root implementation and does not safely encode every generated Compose mount after relaxing validation.
