## Summary

Keep open: the Docker whitespace bug remains valid on current main, but this PR edits the removed root setup script and would leave the active Docker setup path broken. The patch also needs a shared YAML-safe mount writer before relaxing validation.

## What This Changes

The PR branch changes the legacy root `docker-setup.sh` to allow whitespace in mount path validation and emit single-quoted Compose volume mounts.

## Best Possible Solution

Port the whitespace-path support to `scripts/docker/setup.sh` with one reusable YAML scalar writer, cover config/workspace/extra/sandbox mounts in tests, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` pass through `validate_mount_path_value`, which rejects `[[:space:]]` before Compose generation.

## Solution Assessment

No. The requested direction is the right narrow bug fix, but this PR targets a removed root script and does not safely encode every current Compose mount surface after relaxing validation.
