## Summary

Keep open. The underlying Docker install bug is still present on current main and this PR is directionally useful, but it cannot merge as-is because current main moved the active setup logic to `scripts/docker/setup.sh` and the PR still has YAML quoting gaps for apostrophes and sandbox socket mounts.

## What This Changes

The PR relaxes Docker setup mount path validation to permit spaces and emits generated Compose volume mounts as single-quoted YAML scalars in `docker-setup.sh`.

## Best Possible Solution

Implement whitespace-path support in the active `scripts/docker/setup.sh` with a small reusable YAML scalar emitter that preserves spaces and backslashes while escaping apostrophes, use it for config/workspace/home/extra mounts and the sandbox socket overlay, keep the root wrapper intact, and add focused Docker setup e2e coverage for spaces and apostrophes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a path containing a space and current main reaches `validate_mount_path_value`, which fails with the existing whitespace check before setup can proceed.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it edits the stale root implementation path and does not safely encode every generated Compose mount after relaxing validation.
