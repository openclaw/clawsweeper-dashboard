## Summary

Keep open: the PR targets a real Docker setup bug, but the branch now edits the removed root setup script instead of the active `scripts/docker/setup.sh`, still needs safe YAML encoding for every generated mount, and lacks after-fix real behavior proof.

## What This Changes

The branch relaxes Docker mount path validation in the root `docker-setup.sh` and emits generated volume mounts as single-quoted YAML scalars.

## Best Possible Solution

Implement whitespace-path support in `scripts/docker/setup.sh` with one YAML-safe scalar writer for all generated mounts, including the sandbox socket overlay, and cover it with focused Docker setup proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a spaced `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` reaches `validate_mount_path_value` in `scripts/docker/setup.sh` and fails with `cannot contain whitespace`; I did not run the setup script because this review is read-only.

## Solution Assessment

No. The requested direction is right, but this branch edits the removed root entrypoint and does not safely encode every generated Compose mount after relaxing validation.

  - confidence: 0.87
