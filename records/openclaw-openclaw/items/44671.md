## Summary

Keep open. The underlying Docker whitespace-path bug still exists on current main, and this PR is useful source work, but it is not merge-ready because current main moved the active setup logic to scripts/docker/setup.sh and the proposed YAML quoting still breaks valid paths plus the sandbox socket overlay.

## What This Changes

The PR removes Docker setup whitespace rejection, relaxes mount validation, and quotes generated Compose volume mounts in the old root docker-setup.sh implementation.

## Best Possible Solution

Implement the whitespace-path fix in scripts/docker/setup.sh with a small reusable YAML scalar emitter that preserves spaces and backslashes while escaping apostrophes, use it for generated extra mounts and the sandbox socket overlay, and add focused Docker setup e2e coverage for config, workspace, extra mount, and socket paths with spaces and apostrophes.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, any OPENCLAW_CONFIG_DIR or OPENCLAW_WORKSPACE_DIR containing a space reaches validate_mount_path_value and fails on the existing whitespace check before setup can proceed.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it edits the old root implementation, leaves single-quoted YAML unescaped, and does not handle the sandbox socket overlay.
