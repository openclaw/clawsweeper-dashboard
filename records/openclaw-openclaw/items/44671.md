## Summary

Keep open: the linked Docker whitespace bug is valid on current main, but this PR is not mergeable as-is because it edits the removed root setup script and still has Compose YAML encoding gaps. It also needs real after-fix behavior proof before merge.

## What This Changes

The branch relaxes Docker mount path validation in `docker-setup.sh` and emits generated Compose volume mounts as single-quoted YAML scalars.

## Best Possible Solution

Port whitespace-path support to `scripts/docker/setup.sh` with one YAML-safe scalar writer for all generated mounts, include the sandbox socket overlay, and add focused Docker setup e2e coverage.

## Reproduction Assessment

Yes. A high-confidence source path exists: a spaced `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` reaches `validate_mount_path_value` in `scripts/docker/setup.sh` and fails with `cannot contain whitespace`.

## Solution Assessment

No. The requested direction is right, but this branch edits the stale root script and does not safely encode every generated Compose mount after relaxing validation.
