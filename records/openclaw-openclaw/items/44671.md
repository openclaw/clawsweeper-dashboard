## Summary

Keep open. Current main still rejects whitespace in Docker setup paths, and this PR is useful source work but not merge-ready because it targets the stale root implementation and leaves generated Compose YAML unsafe for some now-allowed values.

## What This Changes

The PR relaxes Docker setup mount validation to allow whitespace in config/workspace paths and emits generated Compose volume mounts as single-quoted YAML scalars.

## Best Possible Solution

Implement whitespace-path support in scripts/docker/setup.sh with one reusable YAML scalar writer used by extra mounts and the sandbox socket overlay, preserve the root wrapper, add focused e2e coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: set OPENCLAW_CONFIG_DIR or OPENCLAW_WORKSPACE_DIR to a path containing a space and current main reaches validate_mount_path_value, whose [[:space:]] branch fails before setup proceeds.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it patches the stale root implementation and does not safely encode every generated Compose mount after relaxing validation.
