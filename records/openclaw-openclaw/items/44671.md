## Summary

Keep open: the Docker whitespace-path bug is valid, but this PR is stale against current main, edits a root script that no longer exists, and still needs YAML-safe emission for all generated Compose mounts before it can merge.

## What This Changes

The PR relaxes Docker setup mount path validation to allow whitespace and emits generated Compose volume mounts as single-quoted YAML scalars in the legacy root `docker-setup.sh`.

## Best Possible Solution

Implement whitespace-path support in `scripts/docker/setup.sh` with one reusable YAML scalar writer, use it for config/workspace/home/extra mounts and the sandbox socket overlay, add focused e2e coverage, and include the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a path containing a space and current main fails in `validate_mount_path_value` before setup continues.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it targets the removed root script and does not safely encode every generated Compose mount after relaxing validation.
