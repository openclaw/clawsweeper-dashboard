## Summary

Keep open: the reported Docker whitespace bug is still present on current main, but this PR is not mergeable as-is because it edits the removed root setup script, misses safe YAML encoding surfaces, and lacks real after-fix proof.

## What This Changes

The PR removes whitespace rejection from Docker setup mount validation and writes generated volume mounts as single-quoted YAML scalars in the old root `docker-setup.sh`.

## Best Possible Solution

Implement whitespace-path support in `scripts/docker/setup.sh` with one YAML-safe mount writer for generated extra mounts and the sandbox socket overlay, plus focused regression coverage and real Docker setup proof for spaced config/workspace paths.

## Reproduction Assessment

Yes, from source inspection: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a value with a space and run `scripts/docker/setup.sh`; current main routes those values through `validate_mount_path_value`, which emits the reported whitespace error.

## Solution Assessment

No. The requested direction is correct, but this branch edits the removed root setup script and does not safely encode every generated Compose mount after relaxing validation.
