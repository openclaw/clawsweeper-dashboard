## Summary

Keep open. The PR targets a real Docker setup regression, but the proposed patch is not mergeable as-is against current main: the active implementation has moved under `scripts/docker/setup.sh`, current main still rejects whitespace there, and the PR's YAML quoting strategy remains incomplete for apostrophes and the sandbox socket overlay.

## What This Changes

The PR branch relaxes Docker setup mount-path validation in `docker-setup.sh` to allow spaces and emits generated Compose volume mounts as single-quoted YAML scalars.

## Best Possible Solution

Implement whitespace-path support in `scripts/docker/setup.sh` with a small reusable YAML scalar emitter that preserves spaces and backslashes while escaping apostrophes, use it for all generated Compose volume mounts including the sandbox socket overlay, keep the root wrapper intact, and add focused e2e coverage for spaces and apostrophes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a path containing a space and current main reaches `validate_mount_path_value`, which fails on `[[:space:]]`; I did not run the script because this review is constrained to read-only inspection.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it edits the stale root implementation path and does not safely encode every generated Compose mount after relaxing validation.
