## Summary

Keep open: the linked Docker whitespace bug is still valid on main, but this PR is not mergeable as-is because it misses the active setup entrypoint, has Compose encoding blockers, and lacks real behavior proof.

## What This Changes

The branch relaxes Docker mount path validation and quotes generated Compose volume mounts in the root `docker-setup.sh` script to allow spaces in config and workspace paths.

## Best Possible Solution

Update `scripts/docker/setup.sh` with one YAML-safe mount scalar helper used for all generated mounts, include sandbox socket encoding, add focused e2e coverage for spaced config/workspace paths, and verify a real Docker setup run.

## Reproduction Assessment

Yes. Source inspection shows a spaced `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` reaches `validate_mount_path_value` in `scripts/docker/setup.sh` and fails with `cannot contain whitespace`.

## Solution Assessment

No. The direction is right, but the branch edits the removed root script and needs a shared YAML-safe encoder plus coverage for every generated mount path before it is the maintainable fix.
