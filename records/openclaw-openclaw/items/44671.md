## Summary

Close as superseded: the Docker whitespace bug remains real, but it is already tracked by the canonical open issue while this stale branch targets a deleted root script and still has YAML encoding gaps.

## What This Changes

The PR relaxes Docker setup mount validation to allow whitespace in config/workspace paths and writes generated Compose volume mounts as single-quoted YAML scalars.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/44599 as the canonical tracker and land a fresh, focused fix in `scripts/docker/setup.sh` with YAML-safe mount encoding and Docker setup regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` still flow through `validate_mount_path_value`, which rejects `[[:space:]]` with the reported message.

## Solution Assessment

No. The requested direction is right, but this branch changes a removed script and does not safely encode single quotes or the sandbox socket mount after relaxing validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Port the fix to the active setup script:** `docker-setup.sh:144`
  - body: Current main no longer has the root `docker-setup.sh`; users and docs run `scripts/docker/setup.sh`. As written, this patch would not change the active Docker setup path after rebase, so the reported failure remains.
  - confidence: 0.96
- **[P2] Escape single quotes before YAML emission:** `docker-setup.sh:278-286`
  - body: The branch writes mounts as single-quoted YAML scalars while the relaxed validator still allows apostrophes. A valid path such as `/Users/O'Connor/.openclaw` would generate invalid Compose YAML unless internal single quotes are doubled or a shared encoder is used.
  - confidence: 0.94
- **[P2] Encode the sandbox socket mount too:** `docker-setup.sh:144`
  - body: Relaxing the shared path validator also admits `OPENCLAW_DOCKER_SOCKET` values with spaces, but the sandbox overlay still emits that mount as a plain scalar. Route it through the same YAML-safe writer before accepting whitespace.
  - confidence: 0.88
