## Summary

Current main still rejects the reported whitespace paths in the active Docker setup script, while this PR targets the removed root `docker-setup.sh` and still has YAML encoding gaps, so it remains useful but is not merge-ready.

## What This Changes

The PR relaxes Docker setup mount-path validation and emits generated volume mounts as single-quoted YAML scalars in the old root `docker-setup.sh`.

## Best Possible Solution

Implement whitespace-path support in `scripts/docker/setup.sh` with one YAML-safe mount writer for generated extra mounts and the sandbox socket overlay, plus focused Docker setup proof for config/workspace paths with spaces.

## Reproduction Assessment

Yes, from source inspection: set `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` to a value with a space and run `scripts/docker/setup.sh`; current main routes those values through `validate_mount_path_value`, which emits the reported whitespace error.

## Solution Assessment

No. The requested direction is correct, but this branch edits the removed root setup script and does not safely encode every generated Compose mount after relaxing validation.
