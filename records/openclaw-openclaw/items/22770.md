## Summary

Keep this issue open: current main and v2026.6.10 still lack an active TUI external-editor compose handoff, and the remaining work needs maintainer UX and security-boundary decisions before implementation.

## What This Changes

Request to add a TUI external-editor compose flow that opens the current draft in `$VISUAL`/`$EDITOR`, restores edited text, and leaves manual submission to the user.

## Best Possible Solution

Add one maintainer-approved core TUI handoff that reuses the suspension lifecycle, safely spawns the selected editor, uses a restrictive temporary draft file, reloads without submitting, and updates focused tests and public docs.

## Reproduction Assessment

Yes. Current-main source and release inspection show no active TUI external-editor trigger, draft tempfile handoff, `$VISUAL`/`$EDITOR` resolution, or edited-draft restore path.

## Solution Assessment

Yes in broad direction, but not as an automation-ready fix. A core TUI composer handoff is the right surface, while shortcut UX and the local process/tempfile security contract need maintainer approval first.
