## Summary

Keep open: current main still lacks the requested $VISUAL/$EDITOR compose flow, and the related open TUI issues cover Shift+Enter/newline and vi/vim input behavior rather than external-editor handoff.

## What This Changes

The issue requests a TUI compose mode that opens the current prompt in $VISUAL or $EDITOR, reads the saved content back into the composer, and leaves submission manual.

## Best Possible Solution

Implement a TUI-owned external editor compose helper after UX/security signoff, with no auto-submit, clear status/error handling, focused tests, and public TUI docs.

## Reproduction Assessment

Not applicable as a feature request; there is no broken current contract to reproduce. Current-main source and docs inspection shows the requested editor hook, launch/readback path, command substitute, and documentation are absent.

## Solution Assessment

Unclear until maintainers choose the shortcut or slash-command UX and approve local editor/temp-file handling. The narrow maintainable direction is a TUI composer helper using the existing suspend/resume seam, not a Gateway or plugin rewrite.
