## Summary

Keep open: current main still lacks the requested $VISUAL/$EDITOR compose handoff, and the related open TUI issues cover adjacent newline and vi-mode behavior rather than this external-editor workflow.

## What This Changes

The issue requests an external editor compose mode for the TUI so users can edit the current prompt in $VISUAL or $EDITOR and return the text to the composer without auto-submit.

## Best Possible Solution

Implement a TUI-owned external editor compose helper using the existing suspend/resume seam, with no auto-submit, clear status/error handling, focused tests, and public TUI docs after maintainer UX/security signoff.

## Reproduction Assessment

Not applicable as a feature request; there is no broken existing contract to reproduce. Current-main source and docs inspection shows the requested editor hook, launch/readback path, command substitute, and documentation are absent.

## Solution Assessment

Unclear until maintainers choose the shortcut or slash-command UX and approve local editor/temp-file handling. The narrow maintainable direction is a TUI composer helper using the existing suspend/resume seam, not a Gateway or plugin rewrite.
