## Summary

Keep this issue open: current main and the latest release still lack a working TUI external-editor compose handoff, and the remaining work needs maintainer UX and security-boundary decisions before implementation.

## What This Changes

Add a TUI compose mode that opens the current prompt in $VISUAL or $EDITOR, reloads the edited text into the composer, and leaves final submission manual.

## Best Possible Solution

Add one maintainer-approved core TUI handoff that reuses the suspension lifecycle, safely spawns the selected editor, uses a restrictive temporary draft file, reloads without submitting, and updates focused tests and public docs.

## Reproduction Assessment

Yes. The missing capability is source-reproducible because current main has no active TUI external-editor trigger, draft tempfile handoff, $VISUAL/$EDITOR resolution, or edited-draft restore path.

## Solution Assessment

Yes in broad direction, but not as an automation-ready fix. A core TUI composer handoff is the right surface, while shortcut UX and the local process/tempfile security contract need maintainer approval first.
