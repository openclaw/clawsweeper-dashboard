## Summary

Keep this issue open: current main and v2026.6.10 still do not provide a TUI external-editor compose handoff, and the related long-prompt/editor issues are overlapping or adjacent rather than substitutes. The remaining work is a feature with shortcut UX and local editor/tempfile security decisions, so it should stay in maintainer review rather than be auto-closed or queued blindly.

## What This Changes

The issue requests a TUI external-editor compose mode that opens `$VISUAL` or `$EDITOR` for long prompt editing and loads the edited text back without auto-submitting.

## Best Possible Solution

Add one maintainer-approved core TUI handoff that reuses the TUI suspension lifecycle, safely spawns the selected editor, writes a restrictive temporary prompt draft, reloads without submitting, and updates focused tests plus the public TUI guide.

## Reproduction Assessment

Yes. The missing capability is source-reproducible: current main has no active TUI external-editor trigger, draft tempfile handoff, $VISUAL/$EDITOR resolution, or edited-draft restore path, and v2026.6.10 has the same absence.

## Solution Assessment

Yes in broad direction, but not as an automation-ready fix. A core TUI composer handoff is the right surface, while shortcut UX and the local process/tempfile security contract need maintainer approval first.
