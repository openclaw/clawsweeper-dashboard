## Summary

Keep this issue open: current main and v2026.6.6 still lack the requested TUI external-editor compose flow, while the related multiline, vi-mode, and expanded-composer issues cover adjacent editing problems rather than this `$VISUAL`/`$EDITOR` handoff.

## What This Changes

Request to add a TUI compose mode that opens `$VISUAL` or `$EDITOR`, loads edited markdown back into the composer, and leaves submission manual.

## Best Possible Solution

Add one maintainer-approved core TUI handoff that reuses the suspension lifecycle, spawns the parsed editor command directly, manages a restrictive markdown tempfile, reloads the edited draft without sending, and documents the workflow.

## Reproduction Assessment

Yes for the capability gap: current main and v2026.6.6 have no external-editor trigger, draft tempfile handoff, or `$VISUAL`/`$EDITOR` compose path. This is source-verifiable missing functionality, not a broken-behavior bug reproduction.

## Solution Assessment

Yes in broad shape, but not implementation-ready without maintainer approval. The existing TUI suspension seam plus direct-spawn, secure-tempfile handling is the narrowest maintainable direction; the public trigger and active-state policy still need a decision.
