## Summary

Keep this issue open. Current main still lacks the requested external-editor compose flow: there is no Ctrl+E or equivalent TUI editor hook, no temp-file editor handoff wired into the composer, no slash-command substitute, and no public TUI documentation for $VISUAL/$EDITOR prompt editing. The related #55836 remains adjacent vi/vim input-mode work rather than a replacement for external editor handoff.

## What This Changes

Add a TUI compose mode that opens the current prompt in $VISUAL or $EDITOR, reads the edited markdown back into the composer, and leaves final submission to the user.

## Best Possible Solution

Implement this as a core TUI composer feature after maintainer signoff: suspend the TUI, write the current composer text to a securely created temporary markdown file, launch $VISUAL with $EDITOR fallback, wait for exit, read edited content back into the composer without auto-submitting, surface clear missing-editor/non-zero/read-write errors, and document the workflow in the public TUI guide. Keep #55836 separate for in-place vi/vim keybindings.
