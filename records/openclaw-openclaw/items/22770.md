## Summary

The requested compose handoff remains absent from current main and the latest release, while adjacent multiline and vi-mode issues do not replace it; keep it open for an explicit maintainer decision on shortcut UX and the sensitive local editor/temp-file boundary.

## What This Changes

Requests a TUI compose action that opens the current prompt in `$VISUAL` or `$EDITOR`, then restores edited text without submitting it.

## Best Possible Solution

Implement one dedicated external-editor helper and one explicit composer action that uses the existing TUI suspension lifecycle, private temporary Markdown storage, parsed direct process spawning, guaranteed cleanup/restoration, clear errors, buffer replacement, and no automatic submission.

## Solution Assessment

Yes in broad architecture, but not yet in final UX: a dedicated helper using the existing suspension seam is the narrowest maintainable solution, while maintainers still need to approve the trigger and security contract.
