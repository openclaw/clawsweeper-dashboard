## Summary

Keep open: current main still lacks the requested TUI $VISUAL/$EDITOR compose handoff, and the related open TUI issues cover adjacent newline and vi-mode behavior rather than this external-editor workflow.

## What This Changes

Add a TUI external-editor compose mode that opens current composer text in $VISUAL or $EDITOR and loads the edited result back without auto-submitting.

## Best Possible Solution

Implement a TUI-owned external-editor compose helper using the existing suspend/resume seam, with no auto-submit, clear error/status handling, focused tests, and public TUI docs after UX and local-process boundary signoff.
