## Summary

Keep this issue open: current main and v2026.6.6 still do not implement the requested TUI external-editor compose handoff, and the remaining work needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue requests a TUI compose mode that opens the current draft in `$VISUAL` or `$EDITOR`, then loads the edited text back into the composer without auto-submitting.

## Best Possible Solution

Add one maintainer-approved core TUI handoff that reuses the suspension lifecycle, parses the editor command safely, manages a restrictive markdown tempfile, reloads the edited draft without sending, and documents the chosen shortcut or command.

## Reproduction Assessment

Yes. The missing capability is source-reproducible: current main has no active TUI external-editor trigger, draft tempfile handoff, `$VISUAL`/`$EDITOR` resolution, or edited-draft restore path.

## Solution Assessment

Yes in broad direction, but not as an automation-ready fix. A core TUI composer handoff is the right surface, while the shortcut/command UX and local process/tempfile security contract need maintainer approval first.
