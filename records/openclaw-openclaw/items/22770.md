## Summary

Keep this issue open: neither current main nor v2026.6.6 provides external-editor composition, and the adjacent multiline and vi-mode requests do not replace it. The feature fits the TUI ergonomics direction, but maintainers must approve the public trigger and the security-sensitive process and temporary-file lifecycle.

## What This Changes

Add an external-editor compose flow that round-trips the current TUI draft through `$VISUAL` or `$EDITOR` without automatically submitting it.

## Best Possible Solution

Add one core TUI handoff built on the existing suspension lifecycle, using direct argument-vector spawning and a securely managed Markdown tempfile, after maintainers approve the trigger and active-state semantics.

## Reproduction Assessment

Yes for the capability gap: current main and v2026.6.6 have no external-editor trigger or draft handoff path. This is source-verifiable missing functionality, not a broken-behavior bug reproduction.

## Solution Assessment

Yes in broad shape, but it is not implementation-ready: the existing suspension seam plus Codex-style direct spawning and managed tempfile is the narrowest maintainable approach, while the public binding and active-state policy still need approval.
