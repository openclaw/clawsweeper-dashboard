## Summary

Keep open: current main and v2026.6.10 still do not provide a TUI external-editor compose handoff, and the related long-prompt/editor issues cover adjacent editing surfaces rather than this $VISUAL/$EDITOR workflow. The remaining work is a feature with shortcut UX plus local process/tempfile security decisions, so it should stay in maintainer review instead of being auto-closed or queued blindly.

## What This Changes

The issue requests a TUI external-editor compose mode that launches $VISUAL or $EDITOR for multiline prompt drafting and loads the edited content back into the composer without auto-submitting.

## Best Possible Solution

Add one maintainer-approved core TUI handoff that reuses the TUI suspension lifecycle, safely spawns the selected editor, writes a restrictive temporary prompt draft, reloads without submitting, and updates focused tests plus the public TUI guide.

## Reproduction Assessment

Yes. The missing capability is source-reproducible: current main has no active TUI external-editor trigger, draft tempfile handoff, $VISUAL/$EDITOR resolution, or edited-draft restore path, and v2026.6.10 has the same absence.

## Solution Assessment

Unclear until maintainers settle the exact UX and security contract. A core TUI composer handoff is the right surface, while shortcut choice and local process/tempfile handling need explicit approval first.
