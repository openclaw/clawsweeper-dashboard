## Summary

Keep this open. Current main and the latest release still do not implement the requested TUI external-editor compose flow, and the remaining work needs maintainer UX plus local-process/temp-file security signoff before automation should attempt it.

## What This Changes

The issue requests a TUI compose mode that opens `$VISUAL` or `$EDITOR` on a temporary markdown prompt, then returns the edited text to the composer without auto-submitting.

## Best Possible Solution

Keep this issue open until maintainers approve the shortcut or command UX and security boundary, then implement a TUI-owned flow that suspends the TUI, writes a private markdown temp file, launches `$VISUAL` then `$EDITOR`, restores edited text without sending, surfaces clear errors, and updates TUI docs/tests.

## Reproduction Assessment

Yes. Source inspection on current main shows no TUI keybinding, slash command, or handler that launches `$VISUAL`/`$EDITOR` and reloads edited content into the composer.

## Solution Assessment

Unclear until maintainers choose the UX and security boundary. The likely maintainable path is a TUI-owned implementation that reuses the existing suspension seam and keeps submission manual.
