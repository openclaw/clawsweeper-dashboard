## Summary

Keep open: current main and v2026.6.10 still lack a working TUI external-editor compose handoff, while the related long-prompt/editor issues cover adjacent surfaces rather than this $VISUAL/$EDITOR workflow.

## What This Changes

The issue requests a TUI compose mode that opens $VISUAL or $EDITOR on the current prompt text, reloads edited content into the composer, and does not auto-submit it.

## Best Possible Solution

Add one maintainer-approved core TUI handoff that reuses the TUI suspension lifecycle, safely spawns the selected editor, writes a restrictive temporary prompt draft, reloads without submitting, and updates focused tests plus the public TUI guide.

## Reproduction Assessment

Yes. The missing capability is source-reproducible: current main and v2026.6.10 have no active TUI external-editor trigger, $VISUAL/$EDITOR launch path, prompt tempfile handoff, or edited-draft restore path.

## Solution Assessment

Unclear until maintainers settle the shortcut or slash-command UX and the local process/tempfile security contract. A core TUI composer handoff appears to be the right surface, but the exact boundary needs approval first.
