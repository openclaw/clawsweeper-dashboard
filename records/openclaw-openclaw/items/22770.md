## Summary

Keep open: current main and the latest release still lack the requested $VISUAL/$EDITOR compose handoff, and the related open TUI issues cover adjacent newline or vi-mode behavior rather than this external-editor flow.

## What This Changes

The issue requests a TUI compose mode that opens $VISUAL/$EDITOR on the current composer text, reads the saved prompt back into the composer, and does not auto-submit it.

## Best Possible Solution

Implement a TUI-owned external-editor compose helper using the existing suspend/resume seam, with no auto-submit, clear status/error handling, focused tests, and public TUI documentation after UX and local-process signoff.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken current contract. Current-main and latest-release source/docs inspection show there is no external-editor hook, command substitute, launch/readback path, or public documentation to exercise.

## Solution Assessment
