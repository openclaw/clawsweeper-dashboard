## Summary

Current main still has the fixed 500px `.log-stream` cap that the linked bug reports, and this PR is a narrow CSS-only change that addresses that behavior. It should stay open for normal maintainer review, a current CI rerun/rebase, and possible merge rather than being closed by cleanup.

## What This Changes

The PR updates the Control UI log stream CSS from a fixed 500px maximum height to a viewport-relative maximum height with a 200px minimum.

## Best Possible Solution

Land this small Control UI stylesheet fix, or an equivalent that sizes the Logs scroll area from available viewport height, after current required checks are green. A follow-up can replace the hardcoded offset with a layout variable or flex-based sizing if the page chrome changes later.

## Reproduction Assessment

Yes. Current main still caps `.log-stream` at 500px, so opening the Control UI Logs tab in a browser window taller than about 1000px with many log entries reproduces the unused vertical space described in #53881.

## Solution Assessment

Yes. For the reported behavior, changing the scroll container's maximum height to a viewport-relative value is the narrowest maintainable fix; the only non-blocking refinement would be documenting or deriving the 280px offset.
