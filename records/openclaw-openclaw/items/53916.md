## Summary

Keep open: current main still has the fixed log-stream cap and this PR is a plausible narrow fix, but the external PR still lacks after-fix real behavior proof, so it should not merge yet.

## What This Changes

This PR replaces `.log-stream`'s fixed `max-height: 500px` with `calc(100vh - 280px)`, adds `min-height: 200px`, and adds a changelog entry.

## Best Possible Solution

Land this narrow CSS and changelog fix after visible after-fix proof is added, then let the linked Logs-height bug close through the merge.

## Reproduction Assessment

Yes, by source inspection. Current main routes the Logs tab to `renderLogs`, whose `.log-stream` container is capped at `max-height: 500px`; a tall viewport with enough rows will leave unused space, though I did not run a browser E2E in this read-only review.

## Solution Assessment

Yes, once proof is supplied. The CSS-only viewport-relative max height with a minimum floor is the narrowest fix for the fixed-height layout bug; a named layout variable can be a later cleanup rather than a blocker.
