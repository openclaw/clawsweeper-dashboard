## What This Changes

The issue reports that Telegram selected-text Quote/Quote & Reply disappears for assistant messages delivered with `channels.telegram.richMessages = true`, while plain/default delivery preserves it.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/95990; close this issue only after the fix merges or maintainers explicitly document the rich-mode tradeoff.

Yes at source level, though I did not run a live Telegram client repro in this read-only review. Current main routes `richMessages: true` durable text through `sendRichMessage`, while the plain/default route uses `sendMessage`, and the linked PR has Mantis baseline evidence for the visible Telegram behavior.
