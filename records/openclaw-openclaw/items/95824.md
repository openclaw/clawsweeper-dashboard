## Best Possible Solution

Land after the endpoint migration either preserves/migrates shipped DeepInfra video override configs or explicitly accepts and documents the compatibility break with focused upgrade tests.

## Reproduction Assessment

Yes, at source level: current main honors DeepInfra video `nativeBaseUrl` and `/v1/inference` `baseUrl` overrides, while this branch resolves only `baseUrl` and posts to `${baseUrl}/videos`. I did not run a live upgraded config in this read-only review.

## Solution Assessment

No, not yet. The endpoint move and DeepSeek thinking-format tag are in the right plugin-owned layer, but the best fix needs an upgrade-safe decision for shipped video override configs.
