## Summary

Keep open for maintainer review: the patch appears correct and well-proven, but the DeepInfra video endpoint move and removed guidance options are compatibility-sensitive for custom proxy setups and existing provider-option usage.

## What This Changes

The PR updates the bundled DeepInfra provider to use live video-gen discovery and the OpenAI-compatible async video endpoint, tags DeepInfra DeepSeek-family models with the DeepSeek thinking format, removes unsupported video guidance options, and updates docs/tests.

## Best Possible Solution

Land the provider-local fix after a maintainer explicitly accepts `/v1/openai/videos` as the DeepInfra video contract and carries release-note context for custom proxies and removed guidance options.

## Reproduction Assessment

Yes at source level: current main still posts DeepInfra video requests to native `/v1/inference/<model>` and does not tag DeepInfra DeepSeek-family rows as `thinkingFormat: "deepseek"`. I did not rerun a live current-main failure in this read-only review, but the PR provides inspected after-fix live proof.

## Solution Assessment

Yes for default DeepInfra behavior: the patch stays in the provider plugin, uses the shared async polling helper, and feeds existing core DeepSeek parsing through model compat metadata. The safer alternative only applies if maintainers require backward compatibility for custom native-video proxies or the removed guidance option.
