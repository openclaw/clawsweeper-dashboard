## Summary

## What This Changes

The PR switches DeepInfra video generation from the native inference endpoint to `/v1/openai/videos`, adds async submit/poll handling, and tags DeepInfra DeepSeek-family chat models with `thinkingFormat: "deepseek"`.

## Best Possible Solution

Land the endpoint and DeepSeek-family fixes after the DeepInfra video payload matches the upstream OpenAPI schema and the advertised video provider options match what runtime actually sends.

## Reproduction Assessment

## Solution Assessment

No, not yet. The endpoint switch and DeepSeek thinking-format tagging are the right owner-boundary shape, but the video payload and capability surfaces need to match the upstream contract before this is the best fix.
