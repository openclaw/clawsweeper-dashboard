## Summary

Keep open: current main still appends the reminder note into reply payload text, so this PR is not obsolete, but the branch should not merge as-is because the sanitizer can strip legitimate model-authored text and contributor real behavior proof is missing.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it before typing/final delivery, and updates reminder-guard tests to expect clean reply text.

## Best Possible Solution

Keep the note out of delivered chat by never adding it to final delivery payloads, or by marking the guard-mutated payload and stripping only that mutation, then validate with focused tests and redacted live channel proof.

## Reproduction Assessment

Yes, at source level: current main appends the hardcoded note when reminder-like reply text has no successful cron add and no same-session active cron coverage. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every matching suffix from every non-error text payload is broader than guard-injected text; the safer path is to track the modified payload or avoid putting the note into deliverable payloads.
