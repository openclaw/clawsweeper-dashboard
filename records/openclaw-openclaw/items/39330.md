## Summary

Keep open: current main still appends the unscheduled-reminder note, so the PR is not obsolete, but the proposed sanitizer is too broad and contributor-side real behavior proof is still missing.

## What This Changes

The PR adds a helper to strip the hardcoded unscheduled-reminder note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean replies.

## Best Possible Solution

Choose one reminder-guard UX, then keep the note out of delivered chat by avoiding deliverable annotation injection or stripping only the exact guard-mutated payload, with focused regression coverage and real channel proof.

## Reproduction Assessment

Yes by source inspection: current main has a clear path where a reminder-like reply with successfulCronAdds equal to 0 and no same-session active cron is passed through appendUnscheduledReminderNote. Current focused tests also encode the visible-note behavior.

## Solution Assessment

No. The PR fixes the common leak, but applying the stripper to every non-error text payload is broader than the guard-injected mutation; a safer implementation should avoid adding the note to deliverable payloads or track and strip only the mutated payload.
