## Summary

Keep open: current main still delivers the reminder guard note on the affected path, and this PR targets a real user-visible auto-reply problem, but it is not merge-ready because the sanitizer can remove legitimate reply text and the contributor proof is mock-only.

## What This Changes

The branch adds a helper to strip the unscheduled reminder note from auto-reply payloads before typing/final delivery and changes reminder-guard tests to expect clean reply text.

## Best Possible Solution

Land one selected reminder-guard design that keeps this note out of delivered chat without deleting legitimate content or losing the intended no-reminder signal, backed by focused regression tests and redacted real channel proof.

## Reproduction Assessment

Yes, at source level: current main appends the note when reminder-like text has zero successful cron adds and no same-session cron coverage, and focused tests assert that visible note. I did not run a live Discord or Telegram repro in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but broad post-processing is not the narrowest maintainable solution; the safer path is to keep deliverable payloads free of internal annotations or strip only text known to be guard-injected.
