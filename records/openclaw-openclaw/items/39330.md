## Summary

Keep open. Current main still exposes the internally injected unscheduled-reminder note on the final reply path, so the PR addresses a real user-visible bug, but the proposed sanitizer is too broad because it can remove legitimate model-authored text that happens to end with the same note.

## What This Changes

The PR adds a helper to strip the unscheduled reminder note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean user-visible replies.

## Best Possible Solution

Revise the PR so OpenClaw never delivers the internally injected UNSCHEDULED_REMINDER_NOTE while preserving legitimate model-authored text. The maintainable path is to tag or otherwise track the payload modified by appendUnscheduledReminderNote, or refactor the guard to record the unbacked-reminder condition without adding internal text to delivery payloads.

## Reproduction Assessment

Yes. Current main has a source-level and test-level reproduction: a runReplyAgent reply such as "I'll remind you tomorrow morning." with successfulCronAdds set to 0 flows through appendUnscheduledReminderNote and current tests assert the note appears in the returned text.

## Solution Assessment

No. The PR's after-the-fact global suffix stripping fixes the common leak but is not the narrowest safe solution because it cannot distinguish internally injected text from a legitimate model response containing the same sentence.
