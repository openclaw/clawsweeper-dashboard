## Summary

Keep open: current main still appends the reminder-guard note, so this PR is not obsolete, but the branch still has a P2 correctness issue and only mock/unit proof for a user-visible delivery change.

## What This Changes

The PR adds a helper to strip the hardcoded unscheduled-reminder note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean replies.

## Best Possible Solution

Choose one coherent reminder-guard product path, then keep the note out of delivered chat without stripping unrelated model-authored text and prove the result in a real channel or gateway run.

## Reproduction Assessment

Yes, by source inspection: current main has a clear path where reminder-like reply text with `successfulCronAdds === 0` and no same-session active cron is passed through `appendUnscheduledReminderNote`. I did not run a live channel repro in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every non-error text payload is broader than the guard-injected mutation; a safer fix should avoid adding the note to deliverable payloads or strip only the exact payload the guard mutated.
