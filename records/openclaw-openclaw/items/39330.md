## Summary

Keep open: current main still delivers the unscheduled-reminder guard note, but this PR is not ready to merge because its sanitizer can delete legitimate model-authored text and the external contributor has not provided real delivery-path proof.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it to reply payload text before typing/final delivery, and updates reminder-guard tests to expect clean visible replies.

## Best Possible Solution

Keep guard-generated text out of agent-authored delivery payloads without deleting legitimate content, then align regression tests and redacted real channel proof with the chosen reminder-guard UX.

## Reproduction Assessment

Yes. Source inspection shows current main appends the guard note when reminder-like text has no current-turn or same-session cron coverage, and focused tests still assert the visible appended note.

## Solution Assessment

No. The PR fixes the common leak, but the broad sanitizer can strip user-authored content; a safer fix would avoid injecting the note into delivery payloads or track exactly which payload the guard modified.
