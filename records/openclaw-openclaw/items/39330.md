## Summary

Keep open: current main still appends the user-visible unscheduled-reminder note, and this PR is a plausible fix path, but the branch still needs real behavior proof and the sanitizer is broader than the guard-injected mutation it is trying to remove.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it to auto-reply payloads before typing/final delivery, and updates focused reminder-guard tests to expect clean reply text.

## Best Possible Solution

Choose one reminder-guard UX and land a narrow implementation that prevents the internal note from reaching delivered chat while preserving legitimate model-authored text, with focused regression tests and redacted real channel proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `runReplyAgent` appends the note when a reminder-like reply has `successfulCronAdds` set to zero and no same-session cron coverage, and focused tests assert the visible note; no live channel repro was run in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every non-error payload is broader than the guard mutation; a safer implementation should avoid adding the annotation to deliverable payloads or strip only the guard-mutated payload after maintainers choose the UX.
