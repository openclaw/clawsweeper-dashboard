## Summary

Keep open. Current main still appends the unscheduled-reminder note, and the PR is a focused attempt to address a real user-visible problem, but the current patch can strip legitimate payload text and still lacks after-fix proof from a real channel or gateway run.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it before typing/final delivery, and updates focused runReplyAgent reminder-guard tests to expect clean reply text.

## Best Possible Solution

Keep the note out of delivered chat only for text the guard actually injected, preferably by avoiding deliverable-payload mutation or by tracking the modified payload, then validate with focused tests and redacted live channel proof.

## Reproduction Assessment

Yes, at source level: current main appends the hardcoded note when reminder-like reply text has no successful cron add and no same-session active cron coverage. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every matching suffix from every non-error text payload is broader than guard-injected text; the safer path is to track the payload modified by the guard or avoid adding the note to deliverable payloads.
