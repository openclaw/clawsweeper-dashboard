## Summary

Keep open. Current main and the latest release still append the unscheduled-reminder note to delivered reply text, but this branch is not mergeable as-is because it can strip legitimate model-authored text and still lacks real channel/gateway proof.

## What This Changes

The PR adds a helper that strips the hardcoded unscheduled-reminder note from reply payload text before typing/final delivery and updates focused reminder-guard tests to expect clean visible replies.

## Best Possible Solution

Choose one reminder-guard UX and implement it without deleting model-authored text: avoid adding the note to delivery payloads, tag and strip only guard-modified payloads, or adopt a complete config/annotation path with regression and live proof.

## Reproduction Assessment

Yes, source-reproducible. Current main appends the note when reminder-like text has no successful cron add or same-session cron coverage, and focused tests still assert the appended visible text; I did not run a live channel reproduction in this read-only pass.

## Solution Assessment

No. The PR fixes the common leak, but broad post-hoc stripping is not the narrowest maintainable solution because it can remove legitimate suffix text that the guard did not inject.
