## Summary

Keep open. Current main still sends the unscheduled-reminder note through the visible reply path, but this PR is not merge-ready because the sanitizer can remove legitimate model-authored text, the branch is conflicting, and the external PR only has mocked test proof.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it before typing/final reply delivery, and updates `runReplyAgent` reminder-guard tests to expect clean user-visible text.

## Best Possible Solution

Choose one reminder-guard UX, then prevent only OpenClaw-injected guard text from reaching delivered replies while preserving legitimate model-authored text, with regression coverage and redacted runtime proof.

## Reproduction Assessment

Yes, from source inspection. Current main appends the guard note for unbacked reminder-like replies and then uses the guarded payloads for typing and final delivery; current tests assert the note in no-cron cases.

## Solution Assessment

No. The PR fixes the common leak but is broader than necessary because it strips any payload ending with the exact note; the safer solution is to track guard-injected text or keep the guard note out of delivery payloads, after maintainers choose the UX direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Restrict stripping to guard-injected note text:** `src/auto-reply/reply/agent-runner.ts:535-541`
