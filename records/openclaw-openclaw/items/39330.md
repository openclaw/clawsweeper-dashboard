## Summary

Keep open: current main still delivers the unscheduled-reminder note on the focused auto-reply path, but this PR is not merge-ready because the sanitizer can remove legitimate user-visible text and the after-fix proof is mock-only.

## What This Changes

The PR adds a helper to strip the unscheduled-reminder note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean replies.

## Best Possible Solution

Settle the reminder-guard delivery UX, then land a path that either keeps the note out of deliverable payloads entirely or strips only guard-owned text, with focused tests and live channel proof.

## Reproduction Assessment

Yes for the focused source path: current main appends and returns the note for unbacked reminder-style replies, and current tests still assert that delivered text includes it. I did not run a live Discord or Telegram reproduction in this read-only review.

## Solution Assessment

No: stripping the suffix from every text payload is broader than the guard-owned mutation and can delete legitimate reply text. A safer solution should tag or avoid the injected note rather than sanitize unrelated payloads.
