## Summary

Keep open: current main still has the user-visible reminder-note path, and this PR targets the right surface, but it is not merge-ready because the sanitizer can remove legitimate reply text and the PR only provides mocked test proof.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it to auto-reply payloads before typing/final delivery, and updates reminder-guard tests to expect clean visible text.

## Best Possible Solution

Land one chosen reminder-guard design that keeps internal note text out of delivered chat without deleting legitimate user-visible content, with focused regression tests and redacted real channel proof.

## Reproduction Assessment

Yes, at source level: current main appends the note when a reminder-like reply has zero successful cron adds and no same-session cron coverage, and focused tests assert the visible note. I did not run a live Discord or Telegram repro in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every non-error payload is broader than the guard mutation; the safer fix should avoid adding the annotation to deliverable payloads or strip only payloads this guard actually mutated after maintainers choose the UX.
