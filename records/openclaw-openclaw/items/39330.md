## What This Changes

The PR adds a sanitizer that strips the unscheduled-reminder guard note from reply payload text before typing/final delivery and updates focused reminder-guard expectations.

## Best Possible Solution

Keep guard-generated text out of delivered chat payloads without deleting legitimate content, then align focused tests and redacted real channel proof with the chosen reminder-guard UX.

## Reproduction Assessment

Yes. Source inspection and focused current-main tests show unbacked reminder-like replies still flow through `appendUnscheduledReminderNote` and are delivered with the hardcoded note.

## Solution Assessment

No. The PR fixes the common leak, but stripping every matching suffix from delivered payloads is broader than the guard-injected text and should be narrowed or replaced by a design that never injects the note into delivery payloads.
