## Summary

Keep open: current main still delivers the appended reminder-guard note, so this PR is not obsolete, but it needs changes and real delivery-path proof before merge.

## What This Changes

The PR adds a `stripUnscheduledReminderNote` helper, applies it to reply payloads before typing/final delivery, and updates reminder-guard tests to expect clean visible text.

## Best Possible Solution

Keep OpenClaw-injected reminder-guard text out of user-visible delivery while preserving legitimate model-authored text, with focused regression coverage and redacted delivery-path proof.

## Reproduction Assessment

Yes, from source inspection: current main appends the guard note when no cron backs a reminder commitment and then uses those guarded payloads for typing and final delivery. I did not run tests because this review is read-only.

## Solution Assessment

No. The PR fixes the common leak, but the sanitizer is broader than the injected-text path and can delete legitimate visible text that happens to end with the same note.
