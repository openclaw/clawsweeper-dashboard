This PR should stay open but is not merge-ready: the reported note leak is source-reproducible on current main and the latest release, but the patch can delete legitimate reply text, lacks real transport proof, and overlaps an active reminder-guard PR cluster.

## What This Changes

The PR adds a helper to strip the unscheduled-reminder note from auto-reply text before typing/final delivery and updates reminder-guard tests to expect clean delivered text.

## Best Possible Solution

Settle one canonical reminder-guard delivery path that keeps user-authored content intact, either by preventing guard annotations from entering deliverable payloads or by tagging only guard-injected text for removal, with quote coverage and real transport proof.

## Reproduction Assessment

Yes for the focused source path: current main and v2026.5.18 append and deliver the note for unbacked reminder-style replies, and focused tests assert that behavior. I did not run a live Discord or Telegram reproduction in this read-only review.

## Solution Assessment

No. A broad post-processing sanitizer is not the safest fix because it can remove legitimate quoted content; the better path is to avoid injecting deliverable guard text or remove only guard-owned text with explicit coverage.
