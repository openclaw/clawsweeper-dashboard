## Summary

Keep open: current main still has the user-visible reminder note path, but this PR’s sanitizer is too broad and after-fix real behavior proof is still missing, so it is not ready to merge or close as obsolete.

## What This Changes

The PR adds a helper to strip the unscheduled-reminder note from reply payload text before typing/final delivery and updates focused reminder-guard tests to expect clean returned text.

## Best Possible Solution

Select one reminder-guard UX and implement it so internally generated notes stay out of delivered chat without deleting legitimate text, backed by focused regression coverage and live channel proof.

## Reproduction Assessment

Yes, by source inspection on current main: runReplyAgent appends the hardcoded note when reminder-like text has no successful cron add and no same-session active cron. Focused current tests also assert the visible-note path, though I did not run tests in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every matching suffix after payload construction is broader than guard-injected text; a safer path should track the payload the guard changed or avoid injecting the note into delivery payloads at all.
