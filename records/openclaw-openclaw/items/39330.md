## Summary

Current main still appends the unscheduled-reminder note into returned auto-reply payloads, so the PR is not obsolete; it should stay open, but the current strip-all-text implementation still has a concrete message-delivery correctness issue and lacks real channel proof.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it to auto-reply payloads before typing/final delivery, and changes reminder-guard tests to expect clean delivered text.

## Best Possible Solution

Settle one canonical reminder-guard delivery path that keeps legitimate user-visible text intact, either by preventing guard annotations from entering deliverable payloads or by tagging only guard-injected text for suppression, with focused tests and real transport proof.

## Reproduction Assessment

Yes for the focused source path: current main appends and returns the note for unbacked reminder-style replies, and the focused test expectations still assert that delivered text includes it. I did not run a live Discord or Telegram reproduction in this read-only review.

## Solution Assessment

No, not as-is: stripping the exact note from every non-error text payload is too broad because it can delete legitimate quoted or model-authored content. The safer fix is to suppress only guard-injected text or redesign the guard so the internal annotation never enters deliverable payloads.
