## Summary

Keep open: current main still routes the reminder guard note into user-visible reply payloads, but this PR is not merge-ready because it can strip legitimate model-authored text and the external proof is mocked tests only.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it to reply payloads before typing and final delivery, and updates reminder-guard tests to expect clean visible reply text.

## Best Possible Solution

Prevent only OpenClaw-injected reminder guard text from reaching delivery, preserve legitimate model-authored or quoted matching content, add regression coverage for that provenance boundary, and collect redacted real behavior proof.

## Reproduction Assessment

Yes, from source inspection. Current main appends the note for unbacked reminder commitments, sends typing from `guardedReplyPayloads`, initializes final delivery from that same list, and current tests assert the note remains visible in no-cron cases.

## Solution Assessment

No, not as submitted. The useful direction is plausible, but this patch strips every matching payload suffix instead of proving the text was injected by OpenClaw, so a legitimate model-authored quote can be deleted.
