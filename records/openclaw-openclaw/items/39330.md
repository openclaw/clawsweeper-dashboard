## Summary

Keep open. Current main still sends the internally appended unscheduled-reminder note on the reply payload path, so the PR addresses a real user-visible bug, but the proposed sanitizer is too broad because it can delete legitimate model-authored text that happens to end with the same sentence. The related open #47663 is adjacent config work, not a superseding fix for this PR’s default-delivery leak.

## What This Changes

The PR adds a helper that strips the unscheduled reminder note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean user-visible replies.

## Best Possible Solution

Revise the PR so OpenClaw never delivers the internally injected UNSCHEDULED_REMINDER_NOTE while preserving legitimate model-authored text. The maintainable path is to track which payload appendUnscheduledReminderNote modified, or refactor the guard so it records the unbacked-reminder condition without adding internal text to delivery payloads; keep #47663’s opt-out config discussion separate.

## Reproduction Assessment

Yes. Current main has a source-level and test-level reproduction: a runReplyAgent reply such as "I'll remind you tomorrow morning." with successfulCronAdds set to 0 flows through appendUnscheduledReminderNote, and current tests assert the note appears in the returned text.

## Solution Assessment

No. The PR’s after-the-fact suffix stripping fixes the common leak but is not the narrowest safe solution because it cannot distinguish internally injected text from a legitimate model response containing the same sentence.
