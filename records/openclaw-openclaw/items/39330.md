## Summary

Keep open: current main still exposes the unscheduled-reminder guard note on the delivery path, but this branch should not merge as-is because the sanitizer can remove legitimate model-authored text and the external PR still has only mocked test proof.

## What This Changes

The PR adds a helper to strip the unscheduled reminder note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean visible text.

## Best Possible Solution

Prevent only OpenClaw-injected reminder-guard text from reaching user-visible delivery, preserve legitimate matching user/model content, add focused regression coverage, and collect redacted delivery-path proof after the UX direction is chosen.

## Reproduction Assessment

Yes, from source inspection. Current main appends the note for unbacked reminder commitments, then uses `guardedReplyPayloads` for typing and final payloads, while current tests assert the note is visible in no-cron cases.

## Solution Assessment

No. The PR fixes the common leak but is not the narrowest maintainable fix because it strips every matching non-error text payload suffix instead of tying removal to text that OpenClaw itself injected.
