## Summary

Keep open: the PR targets a real current-main leak path, but the latest patch still strips any payload that happens to end with the guard note, even when OpenClaw did not append it, and the external contributor proof is only mocked Vitest coverage.

## What This Changes

The PR adds a helper to strip the unscheduled reminder note before typing/final delivery and updates reminder-guard tests to expect clean reply text.

## Best Possible Solution

Prevent only OpenClaw-injected reminder guard text from reaching final delivery, preserve legitimate model-authored or quoted matching text, add regression coverage for that provenance boundary, and collect redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection of current main shows `runReplyAgent` appends the guard note and then uses the guarded payloads for typing and final delivery; existing tests assert the visible note in the no-cron cases.

## Solution Assessment

No. The PR's sanitizer fixes the common leak but is not the narrowest maintainable fix because it can remove matching model-authored content; the removal should be tied to the payload text OpenClaw itself appended or the note should be kept out of delivery payloads in the first place.

## Review Findings

Overall correctness: patch is incorrect
