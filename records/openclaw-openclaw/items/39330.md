## Summary

Keep open: current main still appends the reminder guard note to delivered reply payloads, but this PR is not merge-ready because it strips too broadly and only has mock/unit proof.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it to auto-reply payload text before final delivery, and updates reminder-guard tests to expect clean user-visible text.

## Best Possible Solution

Keep the guard note out of delivered chat by never adding it to final payloads, or by stripping only the exact payload mutation introduced by the guard, then prove the delivered channel output with focused regression tests and redacted live proof.

## Reproduction Assessment

Yes, by source inspection. Current main has a clear path where reminder-like reply text with `successfulCronAdds === 0` and no same-session active cron is passed through `appendUnscheduledReminderNote`; I did not run a live channel repro in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every non-error text payload is broader than the guard-injected mutation; the safer path is to avoid adding the note to deliverable payloads or strip only the exact mutated payload.
