## What This Changes

The PR adds a helper that strips the hardcoded unscheduled-reminder note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean visible text.

## Best Possible Solution

Prevent only OpenClaw-injected reminder-guard text from reaching user-visible delivery, preserve legitimate matching content, add focused regression coverage, and collect redacted delivery-path proof after the UX direction is chosen.

## Reproduction Assessment

Yes, from source inspection. Current main appends the note for unbacked reminder commitments, uses the guarded payloads for typing and final delivery, and current tests assert the note is present in the affected cases.

## Solution Assessment

No. The PR fixes the common leak, but the sanitizer is broader than the injected text path and can remove legitimate visible content that happens to end with the same note string.
