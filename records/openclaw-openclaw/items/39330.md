## Summary

Keep open. Current main still appends the reminder-guard note, so the PR is not obsolete, but the proposed sanitizer can remove legitimate reply text and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a helper to strip the unscheduled reminder note from auto-reply payload text before typing/final delivery and updates runReplyAgent reminder-guard tests to expect clean visible text.

## Best Possible Solution

Choose one reminder-guard UX and land a narrow implementation that prevents internal note text from reaching delivered chat while preserving legitimate model-authored content, with focused regression tests and redacted real channel proof.

## Reproduction Assessment

Yes, at source level: current main appends the note when a reminder-like reply has successfulCronAdds set to zero and no same-session cron coverage, and focused tests assert the visible note. No live Discord or Telegram repro was run in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but stripping every non-error payload is broader than the guard mutation; a safer implementation should avoid adding the annotation to deliverable payloads or strip only payloads this guard actually mutated after maintainers choose the UX.
