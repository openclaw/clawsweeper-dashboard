## Summary

Current main still has the user-visible reminder-note leak, so this PR is not obsolete; however the patch is not ready because it can strip legitimate model-authored text and lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, applies it to non-error text reply payloads before typing and final delivery, and updates reminder-guard tests to expect clean user-visible text.

## Best Possible Solution

Prevent only OpenClaw-injected reminder guard text from reaching delivery, preserve legitimate model-authored or quoted matching content, add regression coverage for that provenance boundary, and collect redacted real behavior proof.

## Reproduction Assessment

Yes, from source inspection. Current main appends the note for unbacked reminder commitments and then uses `guardedReplyPayloads` for typing and final payloads, while existing tests assert the note is visible in the no-cron cases.

## Solution Assessment

No. The PR fixes the common leak but is not the narrowest maintainable solution because it strips every matching payload suffix instead of tying removal to text that OpenClaw itself injected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Restrict stripping to guard-injected note text:** `src/auto-reply/reply/agent-runner.ts:532-540`
  - body: The sanitizer runs on every non-error text payload after `guardedReplyPayloads` is built. If the guard did not append anything, a legitimate model response that ends with the same `\n\nNote: ...` text is still stripped, so quoted or model-authored content can be deleted. Track which payload was modified by the guard, or avoid injecting the note into delivery payloads, so only OpenClaw-injected text is removed.
