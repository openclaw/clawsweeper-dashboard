## Summary

Keep open: current main still returns the unscheduled-reminder guard note on the final reply path, and this PR addresses the right surface, but the proposed sanitizer can also remove legitimate model-authored text and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR adds a helper to strip the unscheduled-reminder guard note from auto-reply payload text before delivery and updates reminder-guard tests to expect clean replies.

## Best Possible Solution

Keep the fix focused on the final auto-reply delivery path: remove only the guard text that OpenClaw itself injected, preserve model-authored content, add regression coverage for that provenance boundary, and leave the separate config-flag discussion to #47663.

## Reproduction Assessment

Yes. Source inspection and existing tests show the path: `runReplyAgent` appends the guard note when no cron add succeeds, then returns `guardedReplyPayloads` as the final payloads.

## Solution Assessment

No. The PR's suffix-stripping approach fixes the common leak, but it is not the narrowest maintainable solution unless stripping is tied to text that OpenClaw actually appended.
