## What This Changes

The branch adds `stripUnscheduledReminderNote`, applies it to auto-reply payloads before typing/final delivery, and updates focused reminder-guard tests to expect clean reply text.

## Best Possible Solution

Choose one reminder-guard UX, then keep the note out of delivered chat by avoiding deliverable annotation injection or stripping only the exact guard-mutated payload, with focused regression coverage and real channel proof.

## Reproduction Assessment

Yes by source inspection: current main still appends the hardcoded note when a reminder-like reply has zero successful cron adds and no same-session active cron coverage. Focused current-main tests also encode that visible-note behavior; no live channel repro was run in this read-only review.

## Solution Assessment

No. The PR fixes the common leak, but applying the stripper to every non-error text payload is broader than the guard-injected mutation; a safer implementation should avoid adding the note to deliverable payloads or track and strip only the mutated payload.
