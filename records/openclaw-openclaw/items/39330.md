## Summary

Keep open: current main still routes the appended reminder guard note into final reply payloads, and this PR is useful but not merge-safe as written. The follow-up commit fixed the earlier `didStrip` edge case, but the remaining sanitizer provenance bug and missing changelog entry still need repair before merge.

## What This Changes

The PR adds `stripUnscheduledReminderNote`, sends typing/final delivery through sanitized reply payloads, and updates reminder-guard tests to expect clean user-visible text.

## Best Possible Solution

Revise the fix so OpenClaw never delivers the internally injected reminder note while preserving legitimate model-authored text, preferably by tracking the payload modified by the guard or by avoiding delivery-payload injection entirely; keep the separate config discussion in #47663 out of this PR.

## Reproduction Assessment

Yes. Source inspection and current tests show a high-confidence path: `runReplyAgent` receives a reminder-like reply with `successfulCronAdds: 0`, appends the guard note, and returns `guardedReplyPayloads` as final payloads.

## Solution Assessment

No. The PR's after-the-fact suffix stripping fixes the common leak, but it is not the narrowest maintainable solution because it cannot prove the stripped text was internally appended rather than model-authored.
