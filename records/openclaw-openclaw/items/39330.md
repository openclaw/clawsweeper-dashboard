## Summary

Keep open: current main still delivers the reminder-guard note, so the PR targets a real bug, but it is not merge-ready because the sanitizer can remove legitimate user-visible text and the external PR still lacks real delivery-path proof.

## What This Changes

The PR adds a helper to strip the unscheduled-reminder guard note from reply payload text before typing/final delivery and updates reminder-guard tests to expect clean visible replies.

## Best Possible Solution

Keep OpenClaw-injected reminder-guard text out of delivered replies while preserving legitimate model-authored text, with focused regression coverage and redacted real delivery-path proof.

## Reproduction Assessment

Yes, from source inspection: current main appends the guard note when no cron backs a reminder commitment and then uses those guarded payloads for typing and final delivery. I did not run tests because this review is read-only.

## Solution Assessment

No. The PR fixes the common leak, but the sanitizer is broader than the injected-note path and can delete legitimate visible text that happens to end with the same note; track the payload modified by the guard or avoid injecting the note into delivery payloads.
