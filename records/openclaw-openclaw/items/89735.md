## What This Changes

The issue requests that `openclaw status` render token counts below 1000 as plain integers instead of misleading fractional `k` values.

## Best Possible Solution

Align the status formatter with the canonical token-count formatter at the CLI status boundary, then cover sub-1000 status and cache lines with focused tests.

## Reproduction Assessment

Yes from source: the current formatter converts `999` to `(999 / 1000).toFixed(1) + 'k'`, which is `1.0k`. I did not run a command-level repro because this review was read-only.

## Solution Assessment

Yes: the narrowest maintainable fix is to reuse or mirror the existing sub-1000 integer behavior from the canonical token formatter while leaving >=1000 status output unchanged.
