## Summary

Keep open. Current main still has the Feishu topic follow-up gap, but this PR should not merge as-is because the branch references a block-scoped bypass flag from broadcast dispatch, and the new default mention-bypass policy needs maintainer approval plus docs/changelog coverage.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option, active-topic tracking, and tests so topic-scoped group thread replies can bypass the @-mention gate after the bot is invited to a topic.

## Best Possible Solution

Land a Feishu-owned implementation after hoisting or recomputing the bypass state, documenting and changelogging the config, and choosing the approved default policy for thread mention bypass.

## Reproduction Assessment

Yes. Current main can be source-reproduced with `groupSessionScope: "group_topic"`, `replyInThread: "enabled"`, and `requireMention: true`: an unmentioned thread reply reaches the existing mention gate and returns before topic routing dispatches.

## Solution Assessment

No for the current patch. The active-topic model is plausible, but the implementation has a block-scope error in broadcast dispatch, lacks docs/changelog for a new config key, and needs maintainer agreement on the default mention-bypass policy.
