## Summary

Keep open: current main still lacks a Feishu thread-specific mention policy, and this PR addresses a real source-level gap, but the branch is not merge-ready because the PR head references follow-up state outside its scope and leaves generated/user config surfaces incomplete.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option and active-topic tracking so topic-scoped group thread replies can bypass repeated bot @mentions.

## Best Possible Solution

Land one Feishu-owned thread mention policy that preserves group admission and top-level `requireMention`, carries follow-up state through the current ingress/broadcast path, updates generated config metadata and docs, and reconciles the API with https://github.com/openclaw/openclaw/pull/76020.

## Reproduction Assessment

Yes, at source level. Current main resolves `requireMention`, passes it into group sender activation ingress, and returns before dispatch for unmentioned group messages, so topic-thread follow-ups still hit the same mention gate.

## Solution Assessment

No, not as-is. The feature direction is plausible, but this branch needs the scoped-state fix, generated config metadata/docs, and a maintainer decision against the overlapping `topicRequireMention` design.
