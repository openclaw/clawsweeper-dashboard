## Summary

Keep open: current main still lacks the Feishu thread follow-up mention policy, and this PR has useful live proof, but the branch still has concrete blockers plus overlapping Feishu policy work in https://github.com/openclaw/openclaw/pull/76020.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option and active-topic tracking so topic-scoped group thread replies can bypass repeated bot @mentions.

## Best Possible Solution

Land one Feishu-owned thread mention policy that preserves group admission and top-level `requireMention`, carries follow-up state through the ingress and broadcast paths, and updates generated metadata plus operator docs.

## Reproduction Assessment

Yes, at source level. Current main resolves `requireMention`, passes it into Feishu group sender activation ingress, and returns before dispatch for unmentioned group messages, so topic-thread follow-ups still hit the same mention gate.

## Solution Assessment

No, not as-is. The feature direction is plausible, but this branch needs the scoped-state fix, generated config metadata/docs, and a maintainer decision against the overlapping `topicRequireMention` design.
