## Summary

Keep open: the Feishu thread-follow-up feature is still useful and not implemented on current main, but this PR head has a blocking scoped-variable bug in the broadcast path and still needs user-facing Feishu docs for the new config option.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option, active-topic tracking, and tests so topic-scoped group thread replies can bypass repeated @mentions after the bot was invited.

## Best Possible Solution

Land a repaired Feishu implementation that preserves group admission rules, scopes active topics per account/chat/topic, bounds the cache, carries the bypass state safely through broadcast dispatch, and documents the config surface.

## Reproduction Assessment

Yes, at source level. PR head declares `skipMentionForThread` inside the group block and references it later in broadcast `activeAgentId`, while current main still drops unmentioned Feishu group messages before dispatch.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable form until the bypass state is hoisted, broadcast regression coverage is added, and the Feishu docs describe `threadFollowUp`.
