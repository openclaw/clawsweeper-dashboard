The branch adds a Feishu `threadFollowUp` config option, active-topic tracking, mention-gate bypass logic, and focused tests for topic-scoped Feishu thread replies.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No, not as-is. The active-topic approach may be a valid product choice, but this branch must first fix the out-of-scope broadcast reference and update the public config surfaces, and maintainers need to pick the canonical config shape.
