## Summary

Keep open. This PR remains the canonical Feishu thread-follow-up implementation path, but the current head still has a merge-blocking broadcast-scope bug and needs public docs for the new config option.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option, active-topic tracking, and tests so topic-scoped group thread replies can bypass repeat @mentions after the bot was invited.

## Best Possible Solution

Land a repaired Feishu-owned implementation that preserves group admission rules, scopes active topics per account, bounds the cache, carries the thread-follow-up bypass safely through broadcast dispatch, and documents the new config surface.

## Reproduction Assessment

Yes, at source level. Current main still drops unmentioned Feishu group messages at the mention gate, and the PR-specific blocker is visible because `skipMentionForThread` is read in broadcast dispatch outside the block where it is declared.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable form until the bypass state is hoisted, broadcast follow-up coverage is added, and the Feishu docs describe the new option.
