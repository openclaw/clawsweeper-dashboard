## Summary

Keep open: current main still lacks a Feishu thread-follow-up mention bypass, and this PR still has a blocking out-of-scope `skipMentionForThread` reference plus an undocumented public config key.

## What This Changes

This PR adds a Feishu `threadFollowUp` config option, active-topic tracking, and tests so topic-scoped thread replies can bypass repeated bot mentions.

## Best Possible Solution

Land a conservative Feishu-owned `threadFollowUp` implementation after fixing the scoping bug, documenting the public config contract, and validating focused Feishu tests.

## Reproduction Assessment

Yes for the PR blocker: the PR head declares `skipMentionForThread` inside the group block and reads it later from broadcast dispatch. The original feature gap is also source-supported on current main by the existing mention gate and absence of a thread-follow-up config.

## Solution Assessment

No. The `threadFollowUp` direction is plausible, but this exact patch is not ready until the flag is hoisted into shared scope and the new user-facing config is documented.
