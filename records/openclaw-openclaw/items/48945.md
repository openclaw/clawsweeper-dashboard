## Summary

Keep open: this PR is still the canonical Feishu thread-follow-up implementation candidate, but the current head is not mergeable because the new `skipMentionForThread` state is read outside its block scope in broadcast dispatch, and the new user-facing config is not documented.

## What This Changes

The PR adds Feishu `threadFollowUp` config, active topic tracking, and tests so topic-scoped group thread replies can bypass repeated @mentions after activation.

## Best Possible Solution

Land a repaired Feishu implementation that preserves group admission rules, scopes active topics per account/chat/topic, bounds the cache, carries the bypass state safely through broadcast dispatch, and documents the config surface.

## Reproduction Assessment

Yes, at source level. Current main still drops unmentioned Feishu group messages before dispatch, and PR head reads a block-scoped `skipMentionForThread` in broadcast active-agent selection.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable form until the bypass state is hoisted, broadcast regression coverage is added, and the Feishu docs describe `threadFollowUp`.
