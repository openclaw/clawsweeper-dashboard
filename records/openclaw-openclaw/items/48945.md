## Summary

Keep open: current main still lacks a Feishu topic-thread mention override, but this PR is not mergeable as-is because PR head has a scoped-variable build blocker, incomplete config/docs/generated surfaces, and an unresolved design choice against https://github.com/openclaw/openclaw/pull/76020.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option, in-memory active-topic tracking, and tests so unmentioned replies in topic-scoped group threads can dispatch after bot activation.

## Best Possible Solution

Pick one Feishu-owned topic-thread mention policy, implement it through the policy/dispatch path including broadcast, regenerate config metadata and docs, and validate focused Feishu tests plus changed checks.

## Reproduction Assessment

Yes, at source level. Current main sends unmentioned Feishu group messages through the same `requireMention` gate with no topic/thread policy input, so topic-thread follow-ups still return before dispatch.

## Solution Assessment

No, not as-is. The active-topic approach could solve the usability gap, but this PR currently has a build-scope blocker, incomplete public config surfaces, and a product/design conflict with https://github.com/openclaw/openclaw/pull/76020.
