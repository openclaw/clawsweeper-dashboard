## Summary

Keep open. Current main still lacks a Feishu topic-thread follow-up mention bypass, and this PR includes useful live Feishu proof, but PR head is not merge-ready because it has a broadcast-scope build blocker, missing public config/docs surfaces, and an unresolved product choice against the overlapping `topicRequireMention` design.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option, in-memory active-topic tracking, and tests so topic-scoped group thread replies can bypass repeated bot @mentions.

## Best Possible Solution

Land one Feishu-owned topic reply mention policy that preserves group admission and top-level mention gating, carries follow-up state through normal and broadcast dispatch, and exposes the chosen config consistently in schema, generated metadata, and docs.

## Reproduction Assessment

Yes, at source level. Current main resolves Feishu group `requireMention`, passes `ctx.mentionedBot` into ingress, and returns before dispatch for unmentioned group messages, so topic-thread follow-ups still hit the same mention gate.

## Solution Assessment

No, not as-is. The feature direction is plausible, but PR head has a scoped-variable build blocker, missing docs/generated config surfaces, and an unresolved product choice against the overlapping `topicRequireMention` design.
