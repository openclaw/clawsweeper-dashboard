## Summary

Keep open. Current main still lacks the Feishu thread-follow-up mention bypass, and the contributor provided useful live Feishu verification, but the branch has a blocking source bug in the broadcast path and needs to be adapted to the current Feishu ingress flow before merge.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option, active-topic tracking, and tests so topic-scoped group thread replies can skip repeat @mentions after the bot is invited.

## Best Possible Solution

Land a current-main implementation that preserves existing group admission rules, keeps the active-topic cache per account with a bounded TTL, makes broadcast follow-ups dispatch through the active agent, and documents the new Feishu config.

## Reproduction Assessment

Yes. The PR-specific blocker is source-reproducible from the diff because `skipMentionForThread` is block-scoped and then read in broadcast dispatch; the original follow-up gap is also source-visible on current main because unmentioned group messages still return at the activation ingress gate.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best mergeable form until the bypass state is carried through the current ingress path, broadcast coverage is added, and the new config is documented.
