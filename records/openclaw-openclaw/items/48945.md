## Summary

Keep open. ## What This Changes

The PR adds Feishu `threadFollowUp` config plus active-topic tracking so topic-scoped thread replies can bypass repeated @-mentions.

## Best Possible Solution

Pick one Feishu-owned topic-thread mention policy, implement it through config schema, policy, dispatch including broadcast, generated config metadata, docs, and focused tests, then land the canonical PR.

## Reproduction Assessment

Yes, at source level. Current main has thread/topic context before dispatch, but Feishu group messages still pass through one `requireMention` gate and unmentioned thread replies return before dispatch.

## Solution Assessment

No, not as-is. The active-topic approach could solve the usability gap, but this PR head has a broadcast-scope build blocker, missing public config surface updates, and a competing open config-shape proposal that needs maintainer choice.
