## Summary

Keep open: current main still lacks the Feishu thread-follow-up mention bypass, and the PR remains useful, but the latest branch has a source-reproducible broadcast-path bug that should block merge until repaired.

## What This Changes

The PR adds a Feishu `threadFollowUp` config option, active-topic tracking, and tests so topic-scoped group thread replies can bypass repeat @mentions after the bot was invited.

## Best Possible Solution

Land a current-main Feishu implementation that preserves group admission rules, scopes active topics per account, bounds the cache, carries the thread-follow-up bypass through broadcast dispatch, and documents the new config surface.

## Reproduction Assessment

Yes, at source level. Current main still returns unmentioned Feishu group messages at the mention gate, and the PR-specific broadcast blocker is visible because `skipMentionForThread` is read outside the block where it is declared.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable form until the bypass state is hoisted, broadcast follow-up coverage is added, and the Feishu docs describe the new option.
