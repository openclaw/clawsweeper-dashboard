## Summary

Keep open: current main does not already expose `ctx.agentId`, and the PR is a narrow, well-proven implementation, but it adds public Plugin SDK command-context API that needs maintainer acceptance and a small docs follow-through before merge.

## What This Changes

The PR adds optional `agentId` to plugin command context, populates it from existing bound-agent resolution, and adds command tests for agent-scoped and explicit-agent paths.

## Best Possible Solution

If maintainers accept the SDK direction, land the resolver-based plumbing with a short command-author docs note; otherwise close this API expansion and keep plugin authors on existing session/runtime context surfaces.

## Reproduction Assessment

Not applicable as a bug repro; this is an additive Plugin SDK feature rather than a broken documented current contract. Source inspection confirms current main resolves command agent authority for `runtimeContext` but does not expose top-level `ctx.agentId`.

## Solution Assessment

Yes for the code shape, subject to maintainer product acceptance: the PR reuses `resolveBoundAgentIdForSession` and keeps exposed metadata aligned with runtime authority. The safer pre-merge path is to approve the SDK contract explicitly and document the optional field.
