## What This Changes

The PR adds optional `PluginCommandContext.agentId`, populates it from existing bound-agent resolution in `executePluginCommand`, and adds command tests for agent-scoped and explicit-agent paths.

## Best Possible Solution

If maintainers accept the SDK contract, land the narrow plumbing with command-author docs aligned; otherwise leave command authors on existing session/runtime helper surfaces.

## Reproduction Assessment

## Solution Assessment

Yes for the code shape if maintainers want the API: it reuses the existing resolver and keeps runtime authority aligned with the exposed value. The safer pre-merge state is explicit maintainer acceptance plus a docs note for command authors.
