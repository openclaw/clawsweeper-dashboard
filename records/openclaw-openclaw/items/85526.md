## Summary

Keep open: current main has cron timeout/error recording and Codex app-server terminal-idle diagnostics, but no declared-artifact reconciliation path that can verify an already-created external artifact and downgrade the cron failure to success or partial success.

## What This Changes

This issue requests cron-side reconciliation and clearer diagnostics when a scheduled Codex app-server turn performs an external side effect but times out before final assistant or artifact delivery.

## Best Possible Solution

Define an opt-in cron completion-artifact contract that lets jobs declare verifiable external artifacts, records verified partial/success outcomes without masking ordinary failures, and surfaces Codex app-server terminal-idle diagnostics distinctly from pre-side-effect timeouts.

## Reproduction Assessment

No live high-confidence end-to-end reproduction was run. Source inspection does show the current-main state transition: timeout/error paths record failure and task timeout state without any artifact verifier that could reconcile an already-created GitHub PR.

## Solution Assessment

Unclear: the requested reconciliation hook is plausible, but the durable artifact declaration and trust model need maintainer design. A safe first step is to propagate Codex terminal-idle timeout diagnostics through cron, then add an explicit opt-in artifact verifier contract.
