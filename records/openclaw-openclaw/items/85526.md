## Summary

Keep open: current main still records cron timeout/error outcomes from the agent run itself and has no declared-artifact reconciliation path that could verify an already-created external PR and convert the run to success or partial success.

## What This Changes

The issue asks for scheduled Codex app-server cron runs to distinguish post-side-effect terminal-idle timeouts from pre-side-effect failures and optionally reconcile declared completion artifacts.

## Best Possible Solution

Define an opt-in cron completion-artifact contract that lets jobs declare verifiable external artifacts, records verified partial or success outcomes without hiding ordinary failures, and surfaces Codex terminal-idle timeout diagnostics separately from pre-side-effect timeouts.

## Reproduction Assessment

No live high-confidence end-to-end reproduction was run. Source inspection does show the current-main state transition: timeout paths record error/failure state without any declared-artifact verifier that could reconcile an already-created GitHub PR.

## Solution Assessment

Unclear: the requested reconciliation hook is plausible, but the artifact declaration, verifier trust model, and partial-success state need maintainer design. A safe path is to improve terminal-idle diagnostics and add an explicit opt-in artifact contract rather than inferring side effects from arbitrary prompts.
