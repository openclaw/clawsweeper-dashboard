## Summary

Keep open: current main still converts the cron timeout path into an error/failure state and has no declared external-artifact reconciliation contract. Related open issues cover adjacent timeout/completion reliability problems, but none owns this exact scheduled-side-effect-then-timeout artifact verifier gap.

## What This Changes

The issue requests cron support for reconciling scheduled Codex runs that create an external artifact but time out before final assistant delivery, or at least diagnostics that distinguish that state.

## Best Possible Solution

Define an opt-in cron completion-artifact contract that can verify declared external artifacts after a timeout, record verified partial/success outcomes without hiding real failures, and improve terminal-idle timeout diagnostics.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the current-main path where cron timeout/error states are recorded without any external artifact verifier that could reconcile an already-created GitHub PR.

## Solution Assessment

Unclear: the reconciliation hook is plausible, but the durable artifact declaration, trust model, and state semantics need maintainer design. A safe first slice may be clearer terminal-idle diagnostics before adding an opt-in artifact verifier contract.
