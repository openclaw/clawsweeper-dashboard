- `impact:message-loss`: The reported failure is that final assistant/artifact delivery is lost or reported failed after the external side effect already happened.

## Summary

Keep open: current main has better Codex app-server side-effect timeout diagnostics, but cron still records timeout/error outcomes without a declared-artifact reconciliation path that can verify an already-created external artifact and mark the run successful or partial.

## What This Changes

Request cron support for reconciling side-effect-created completion artifacts after an agent-turn timeout, plus clearer timeout diagnostics for that case.

## Best Possible Solution

Define an opt-in cron completion-artifact contract that lets scheduled jobs declare verifiable external artifacts, records verified partial/success outcomes without masking ordinary failures, and keeps the improved Codex terminal-idle diagnostics for unreconciled timeouts.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence current-main path: cron timeout/error finalization and failure alerts have no declared-artifact verifier, while Codex app-server only provides side-effect-aware diagnostics.

## Solution Assessment

Unclear: the reconciliation hook is plausible, but the durable artifact declaration, verifier trust model, and idempotency behavior need maintainer design before implementation.
