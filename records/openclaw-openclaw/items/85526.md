## Summary

Keep open: current main includes the adjacent Codex missing-turn timeout handling, but cron still has no declared external-artifact reconciliation path and still records the timeout outcome from cron state rather than verifying an already-created artifact.

## What This Changes

The issue asks cron agent turns to distinguish post-side-effect timeouts from pre-side-effect failures and optionally reconcile declared external artifacts such as created pull requests.

## Best Possible Solution

Define an opt-in cron completion-artifact contract that can verify declared external artifacts after replay-unsafe timeouts and record a clear verified partial/success outcome without hiding real failures.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the current-main path where cron timeout/error state is recorded without an external artifact verifier that could reconcile an already-created PR.

## Solution Assessment

Unclear for the exact hook shape: artifact reconciliation is plausible, but durable artifact declarations, trust model, idempotency, and cron state semantics need maintainer design. A safer incremental path is to keep the current replay-unsafe timeout warning and add an explicit opt-in verifier contract.
