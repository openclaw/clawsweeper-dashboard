## Summary

Keep open: current main has the Codex missing-terminal-turn recovery from https://github.com/openclaw/openclaw/pull/85107, but cron still has no declared external-artifact verifier or partial/success outcome that can reconcile a timed-out scheduled run after a side effect already created the expected artifact.

## What This Changes

The issue requests cron post-timeout reconciliation or clearer diagnostics when a scheduled Codex app-server turn performs an external side effect, then times out before final assistant artifact delivery.

## Best Possible Solution

Define an opt-in cron completion-artifact contract that lets scheduled jobs declare verifiable external artifacts, records verified partial or successful outcomes without hiding ordinary failures, and keeps the newer Codex replay-unsafe timeout diagnostics for missing terminal events.

## Reproduction Assessment

No live high-confidence current-main repro was run. Source inspection gives a medium-confidence reproduction shape: an isolated agentTurn timeout returns an error, cron/task state follows that error path, and no current field or hook can declare and verify an external artifact after the timeout.

## Solution Assessment

Unclear as a final implementation path. The requested reconciliation hook is plausible, but the durable artifact declaration, verifier trust model, partial-success status semantics, and failure-alert behavior need maintainer design before code should change.
