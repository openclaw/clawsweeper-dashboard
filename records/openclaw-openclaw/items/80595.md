## Summary

Keep open: current main still hard-codes the generated Linux `gateway.systemd.env` path and lacks a supported config option to use the state-dir `.env` directly. The related closed reports made the generated-file path safer, but they do not implement this requested opt-in mode.

## What This Changes

Request an opt-in Linux systemd mode/config to use `~/.openclaw/.env` or a custom file as the gateway service `EnvironmentFile` instead of generating `gateway.systemd.env`.

## Best Possible Solution

Choose and document an explicit Linux systemd env-file config contract, validate systemd-compatible dotenv contents, and update gateway install, doctor repair, and regression tests to preserve that contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection shows current main still renders `gateway.systemd.env` and has no `gateway.systemd` config key, so the requested mode is absent.

## Solution Assessment

Unclear until maintainers choose the supported contract. Direct `EnvironmentFile` support is plausible, but the maintainable solution needs schema, validation, docs, and doctor/install behavior rather than local unit edits.
