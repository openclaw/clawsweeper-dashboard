## Summary

Keep open: current main still implements only the generated `gateway.systemd.env` model for Linux systemd installs, while this request asks for a new explicit operator mode to use the state-dir `.env` directly. The related closed reports made the generated-file path safer, but none adds the requested config contract.

## What This Changes

The issue requests an opt-in Linux systemd configuration mode that uses the state-dir `.env` as the gateway service `EnvironmentFile` or disables generation of `gateway.systemd.env`.

## Best Possible Solution

Add a deliberate Linux systemd config contract with validation for systemd-compatible `.env` values and clear install/doctor preservation semantics, or document that the generated env file remains the only supported model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection and existing tests show current main still renders the generated `gateway.systemd.env` behavior the issue describes.

## Solution Assessment

Unclear until maintainers choose the supported contract. A direct `EnvironmentFile` option is plausible, but the maintainable solution needs schema, validation, docs, and doctor/install behavior rather than an ad hoc unit override.
