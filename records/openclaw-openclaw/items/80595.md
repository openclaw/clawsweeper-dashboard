## Summary

Keep open: current main and the latest release still use the generated Linux systemd gateway environment-file model, and there is no supported config surface for using the state-dir `.env` directly. The related closed reports improved generated-file safety and update/doctor behavior, but this remains a new opt-in service environment contract that needs maintainer product and security judgment.

## What This Changes

Request a supported Linux systemd configuration mode that uses the state-dir `.env` directly, or disables generation of `gateway.systemd.env`.

## Best Possible Solution

Define an explicit Linux systemd service-env contract with schema, systemd-compatible env-file validation, install/doctor preservation semantics, docs, and tests; otherwise document generated `gateway.systemd.env` as the only supported model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection confirms current main still generates `gateway.systemd.env` and lacks a supported direct state-dir `.env` mode.

## Solution Assessment

Unclear until maintainers choose the contract. A direct `EnvironmentFile` option is plausible, but the maintainable solution needs schema, validation, docs, and install/doctor behavior rather than an unsupported unit edit.
