## Summary

Keep open: current main and v2026.6.6 still use the generated Linux systemd `gateway.systemd.env` path, and the requested direct state-dir `.env`/custom `EnvironmentFile` mode is absent and security-sensitive.

## What This Changes

The issue requests an explicit Linux systemd gateway option to use the state-dir `.env` or a custom env file directly and stop regenerating `gateway.systemd.env` in that mode.

## Best Possible Solution

Decide whether OpenClaw should support this opt-in systemd env-file contract; if yes, add schema/help/docs, installer and doctor preservation, validation, upgrade coverage, and keep `gateway.systemd.env` as the default.

## Reproduction Assessment

Not applicable as a bug reproduction: source and release inspection show current behavior intentionally generates and references `gateway.systemd.env`. The issue asks for a new opt-in mode.

## Solution Assessment

Unclear until maintainers choose the service-environment contract. Direct `.env` mode is plausible, but it is only the best fix if it preserves the current generated-file safety checks and upgrade behavior.
