## Summary

Keep open: current main still implements the generated `gateway.systemd.env` model and does not provide a supported Linux systemd option to use the state-dir `.env` or a custom `EnvironmentFile` directly. This remains a feature/config request that needs maintainer product and security judgment.

## What This Changes

Request support for a Linux systemd gateway option to use the state-dir `.env` or a custom file directly as `EnvironmentFile` and skip generating `gateway.systemd.env`.

## Best Possible Solution

Add a documented `gateway.systemd` environment-file contract with validation and install/doctor preservation semantics, or explicitly document the generated `gateway.systemd.env` file as the only supported Linux systemd model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection provides high-confidence evidence that current main still uses the generated env-file model described in the issue.

## Solution Assessment

Unclear until maintainers choose the supported contract. A direct `EnvironmentFile` option is plausible, but the maintainable solution needs schema, validation, docs, and doctor/install behavior rather than an ad hoc unit override.
