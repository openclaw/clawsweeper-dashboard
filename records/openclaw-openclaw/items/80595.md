## Summary

Keep open: current main still implements Linux systemd installs through a generated `gateway.systemd.env` file and does not expose a supported config mode for using the state-dir `.env` directly as the unit `EnvironmentFile`. The related closed reports improved the generated-file path, but they do not resolve this requested opt-in operator mode.

## What This Changes

Request a supported Linux systemd configuration mode that uses the state-dir `.env` directly as the gateway unit `EnvironmentFile` or disables generated `gateway.systemd.env` creation.

## Best Possible Solution

Add an explicit Linux systemd env-file config contract with schema/types/docs, systemd-compatible `.env` validation, and clear install/doctor preservation semantics; alternatively document that the generated env file remains the only supported model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still renders `gateway.systemd.env` and lacks the requested supported direct `.env` mode.

## Solution Assessment

Unclear until maintainers choose the supported contract. A direct `EnvironmentFile` option is plausible, but the maintainable fix needs schema, validation, docs, and doctor/install behavior rather than an unsupported local unit override.
