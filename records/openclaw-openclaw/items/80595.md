## Summary

Keep open: current main and v2026.6.6 still generate and reference `gateway.systemd.env`, while the requested direct state-dir `.env` or custom `EnvironmentFile` mode is absent and security-sensitive.

## What This Changes

Request an opt-in Linux systemd mode that uses the state-dir `.env` or a custom `EnvironmentFile` directly instead of generating `gateway.systemd.env`.

## Best Possible Solution

Maintainers should decide whether to support this explicit systemd env-file contract; if yes, keep generated `gateway.systemd.env` as the default and add schema, docs, installer/doctor preservation, validation, and upgrade tests for the opt-in path.

## Reproduction Assessment

Not applicable as a bug reproduction: source and release inspection show current behavior intentionally generates and references `gateway.systemd.env`. The issue asks for a new opt-in mode.

## Solution Assessment

Unclear until maintainers choose the service-environment contract. Direct `.env` mode is plausible, but it is only the best fix if it preserves the current generated-file safety checks and upgrade behavior.
