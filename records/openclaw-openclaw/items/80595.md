## Summary

Keep open: current main and v2026.6.6 still use the generated `gateway.systemd.env` path and do not expose a supported `gateway.systemd.environmentFile` or disable-generation mode. The request remains a new opt-in service-environment contract with security and product implications, not conservative cleanup closure.

## What This Changes

Add an explicit Linux systemd mode or config option to use the state-dir `.env` directly as the gateway `EnvironmentFile`, or to disable generated `gateway.systemd.env` creation.

## Best Possible Solution

Decide whether OpenClaw should support this opt-in systemd environment contract; if yes, add schema/help/docs, install and doctor preservation behavior, equivalent validation, and upgrade tests while keeping generated `gateway.systemd.env` as the default.

## Reproduction Assessment

Not applicable as a bug reproduction: source, tests, and latest-release inspection show the current intended behavior still generates and references `gateway.systemd.env`. This issue asks for a new opt-in mode rather than reporting a current contract failure.

## Solution Assessment

Unclear until maintainers choose the service-environment contract. Direct state-dir `.env` mode is plausible, but it must preserve the generated-file path's validation, security boundary, doctor behavior, and upgrade expectations.
