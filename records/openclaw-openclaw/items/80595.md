## Summary

Keep open: current main and the latest release still render Linux systemd installs through the generated `gateway.systemd.env` path, while this issue requests a new opt-in service-environment mode. Because that would change credential/env handling, this is a maintainer product and security decision rather than cleanup closure.

## What This Changes

The issue requests an explicit Linux systemd gateway configuration mode that uses the state-dir `.env` as the service `EnvironmentFile` or disables `gateway.systemd.env` generation.

## Best Possible Solution

Decide whether OpenClaw should support this opt-in systemd environment contract; if yes, add it with schema/help/docs, install and doctor preservation behavior, explicit validation, and upgrade coverage while keeping `gateway.systemd.env` as the default.

## Reproduction Assessment

Not applicable as a bug reproduction: source, tests, and release inspection show the current intended behavior still generates and references `gateway.systemd.env`. The issue asks for a new opt-in mode rather than reporting a current contract failure.

## Solution Assessment

Unclear until maintainers choose the service-environment contract. The requested direct `.env` mode is plausible, but the current generated-file default is safer unless an opt-in mode preserves validation and upgrade behavior.
