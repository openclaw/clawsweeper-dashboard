## Summary

Keep open: current main and `v2026.5.12` still use the generated `gateway.systemd.env` path for Linux systemd gateway installs, and there is no supported config surface for opting into the state-dir `.env` as the unit `EnvironmentFile`. The related closed reports made the generated-file path safer, but they do not implement this requested operator mode.

## What This Changes

The issue requests an explicit Linux systemd gateway option to use the state-dir `.env` directly, or to disable generation of `gateway.systemd.env`.

## Best Possible Solution

Add a deliberate Linux systemd config contract with validation and clear install/doctor preservation semantics, or document that the generated env file remains the only supported model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source and release inspection show current main still renders the generated `gateway.systemd.env` behavior described in the issue.

## Solution Assessment

Unclear until maintainers choose the supported contract. A direct `EnvironmentFile` option is plausible, but the maintainable solution needs schema, validation, docs, and doctor/install behavior rather than an ad hoc unit override.
