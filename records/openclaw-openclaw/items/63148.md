## Summary

Keep open. The PR targets a real Feishu DM-topic isolation gap, but the current branch still introduces topic-group queue and DM-topic delivery regressions, and the external-contributor real behavior proof gate is still missing.

## What This Changes

The PR scopes Feishu DM messages carrying root_id or thread_id to topic-specific peers, thread replies, parent routing, per-topic queue/debounce keys, and focused tests.

## Best Possible Solution

Land or replace a focused Feishu plugin change that isolates topic-bearing DMs, preserves group/topic_group queue semantics, adds a DM-topic-safe delivery/parser path, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes, source-level. Current main carries root_id/thread_id into FeishuMessageContext, but direct messages still route to the flat sender peer with replyInThread disabled and a chat-scoped queue.

## Solution Assessment

No, not yet. The Feishu plugin is the right surface, but the branch should preserve topic_group group behavior, fix outbound delivery for DM-topic conversation IDs, and settle the default-on versus opt-in product shape before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Treat topic_group chats as group queues:** `extensions/feishu/src/sequential-key.ts:19-21`
  - body: The new queue branch only treats chat_type === "group" as grouped. Current Feishu code classifies "topic_group" as a group chat, so topic_group events with root_id or thread_id now take the DM-style :topic: queue key and can run outside the existing group queue semantics. Use the shared classifier or include topic_group, with coverage.
  - confidence: 0.9
- **[P2] Route DM-topic IDs before direct user IDs:** `extensions/feishu/src/bot.ts:711-715`
  - body: This encodes DM topics as ou_*:topic:<id>, but Feishu delivery resolves direct IDs before parsing topic IDs and accepts any ou_ prefix. Sends or subagent replies addressed to the topic session can therefore target user:ou_*:topic:<id> instead of the original chat/thread. Add a DM-topic-safe encoding or parser path with tests.
  - confidence: 0.82
