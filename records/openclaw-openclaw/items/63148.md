## Summary

## What This Changes

The PR scopes Feishu DM messages with root_id/thread_id to topic-specific sessions, threaded replies, parent routing, per-topic sequential/debounce keys, and adds focused Feishu tests.

## Best Possible Solution

Land or replace a focused Feishu plugin change that adds opt-in DM topic isolation and threaded replies, fixes parser and queue edge cases, preserves upgrade defaults, and includes redacted live Feishu proof.

Yes, source-level: current main parses Feishu root_id/thread_id into the message context, but direct messages still route to ctx.senderOpenId with parentPeer null, replyInThread false, and a chat-scoped sequential key. I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment
