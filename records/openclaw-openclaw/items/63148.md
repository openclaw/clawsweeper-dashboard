Full review comments:

- **[P1] Gate DM topic scoping behind an explicit mode:** `extensions/feishu/src/bot.ts:707-724`
  - body: The linked request asked for optional DM topic scoping, but this branch enables it for every DM carrying `root_id` or `thread_id`. That changes existing Feishu users' session continuity and reply behavior on upgrade, so keep the current flat DM behavior by default and only opt into topic sessions through an explicit, documented setting or maintainer-approved migration.
  - confidence: 0.88
- **[P1] Preserve a real delivery target for DM topic sessions:** `extensions/feishu/src/bot.ts:713-717`
  - body: Building DM topic sessions with the existing `group_topic` conversation-id shape creates IDs like `ou_xxx:topic:om_xxx`. Current Feishu outbound target normalization tries direct IDs first and accepts values starting with `ou_`, so message-tool/outbound sends from that session can become `user:ou_xxx:topic:om_xxx` instead of targeting the real Feishu chat plus thread. Store or resolve the actual DM chat/thread target before using synthetic topic IDs as session keys.
  - confidence: 0.86
- **[P2] Treat topic_group as group chat for queue keys:** `extensions/feishu/src/sequential-key.ts:19-24`
  - body: `topic_group` is a group chat type in this plugin, but this check only excludes literal `group`, so native Feishu topic groups with `thread_id` take the new DM topic queue path. That contradicts the PR's claim that group behavior is untouched and can change ordering/parallelism for group topic messages; use the existing group-chat classifier or explicitly exclude `topic_group`.
  - confidence: 0.82
