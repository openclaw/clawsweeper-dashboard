## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep logout safety checks on timeout:** `extensions/whatsapp/src/auth-store.ts:304-305`
  - body: When the auth barrier times out, this branch skips `shouldClearOnLogout` and then recursively removes `resolvedAuthDir`. Because `authDir` can be caller-provided and the PR head does not constrain it before this delete, a timeout can turn logout into deletion of an unrelated directory instead of only a managed WhatsApp auth directory.
  - confidence: 0.94
- **[P1] Run runtime checks when linked is unknown:** `extensions/whatsapp/src/status-issues.ts:72-73`
  - body: Timeout snapshots intentionally omit `linked`, but this early return exits before the stale/reconnecting/conflict/logged-out/disconnected checks. That can make `channels status` miss active WhatsApp runtime failures during the exact auth-stabilizing window this PR is trying to handle.
  - confidence: 0.91
- **[P1] Pass account context through heartbeat auth deps:** `extensions/whatsapp/src/heartbeat.ts:9`
  - body: The new public `readWebAuthExistsForDecision` dependency is called with no account or authDir even though the caller has resolved the account. Account-aware or multi-account implementations cannot know which auth directory to inspect, so they can report readiness for the wrong WhatsApp account.
  - confidence: 0.88
