Full review comments:

- **[P1] Keep logout safety checks on timeout:** `extensions/whatsapp/src/auth-store.ts:304-305`
  - body: When the auth barrier times out, `forceClear` skips `shouldClearOnLogout`, and this branch then reaches recursive `fs.rm` for non-legacy auth dirs. A caller-provided custom authDir with unrelated contents can be deleted during a timeout, and the current-main managed-directory/symlink protections must be preserved when this branch is rebased.
  - confidence: 0.95
- **[P1] Run runtime checks when linked is unknown:** `extensions/whatsapp/src/status-issues.ts:72-73`
  - body: The PR intentionally omits `linked` on timeout snapshots, but this early return exits before stale, reconnecting, conflict, logged-out, and disconnected checks. That can make `channels status` look clean while the WhatsApp runtime is actively failing during an auth-timeout window.
  - confidence: 0.9
- **[P1] Pass account context through heartbeat auth deps:** `extensions/whatsapp/src/heartbeat.ts:9`
  - body: The new heartbeat auth-decision dependency is called with no account or authDir, so account-aware implementations cannot distinguish non-default WhatsApp accounts and may report readiness from the wrong auth state. Either keep this helper plugin-local or make the public SDK callback receive the needed context before exporting it.
  - confidence: 0.86
