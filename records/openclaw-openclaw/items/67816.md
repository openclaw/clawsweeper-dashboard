  - body: On an auth-barrier timeout this branch sets `forceClear` and skips `shouldClearOnLogout`, then recursively removes `resolvedAuthDir`.   - body: Timeout status snapshots now omit `linked`, but this branch returns immediately when `linked` is unknown. That suppresses stale, reconnecting, conflict, logged-out, and disconnected runtime diagnostics during the same auth-stabilizing window the PR is trying to make safer.
  - confidence: 0.9
- **[P2] Pass authDir through heartbeat decision deps:** `extensions/whatsapp/src/heartbeat.ts:8-9`
  - body: The new heartbeat helper calls `deps.readWebAuthExistsForDecision()` with no account context, even though `checkWhatsAppHeartbeatReady` has already resolved `account.authDir`. Account-aware dependencies can therefore report readiness for the wrong WhatsApp account, and the exported SDK callback shape bakes in that missing context.
  - confidence: 0.86
