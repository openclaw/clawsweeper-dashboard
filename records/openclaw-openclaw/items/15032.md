## What This Changes

The issue requests a per-call tool allow/deny mechanism for sub-agents spawned through `sessions_spawn` so child agents can run with least-privilege tool access.

## Best Possible Solution

Keep this issue as the canonical tracker and close it only after a maintainer-approved per-spawn tool-policy API lands with source, tests, docs, and security semantics.

## Solution Assessment
