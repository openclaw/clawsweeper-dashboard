## What This Changes

The issue requests an enforced per-spawn tool restriction API for `sessions_spawn` so child agents can run with narrower tool access than their parent.

## Best Possible Solution

Keep this issue as the canonical tracker, then land or replace https://github.com/openclaw/openclaw/pull/78441 after maintainers accept the API, persistence, upgrade, and security-boundary contract.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the `sessions_spawn` schema, native spawn params, and child-session policy path have no per-call tool restriction field.

## Solution Assessment
