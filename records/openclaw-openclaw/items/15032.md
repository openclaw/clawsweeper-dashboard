## Summary

Keep open: current main still lacks a per-spawn enforced tool policy for `sessions_spawn`, and the active implementation candidate is open and narrower than the original allow/deny security-boundary request.

## What This Changes

Request adding a per-spawn tool policy to `sessions_spawn` so child sub-agents can be constrained to an allowlist or denylist.

## Best Possible Solution

Land one reviewed additive `sessions_spawn` tool-policy API with enforced final narrowing, documented precedence, focused coverage, fail-closed unsupported runtime behavior, and docs at https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that the schema, native params, spawn handoff, resolver, docs, and Gateway request schema do not expose a per-spawn tool-policy path.

## Solution Assessment

Unclear until maintainers approve the final API shape. The open native `toolsAllow` PR is a useful subset, but the full request still needs decisions on deny semantics, no-tools mode, unsupported runtimes, persistence, and policy precedence.
