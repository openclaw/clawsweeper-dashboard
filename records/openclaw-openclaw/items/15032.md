## Summary

Keep open: current main still does not expose an enforced per-spawn tool restriction on `sessions_spawn`, and the active related PR only covers a narrower native allowlist path rather than the full allow/deny security-boundary request.

## What This Changes

The issue requests an additive per-spawn tool restriction API for `sessions_spawn` so child sub-agents can run with a narrower enforced tool set than their parent.

## Best Possible Solution

Land one maintainer-approved additive `sessions_spawn` tool-policy contract that only narrows child capabilities, documents precedence, fails closed for unsupported runtimes, and updates focused tests plus subagents docs.

## Reproduction Assessment

Not applicable: this is a feature/security-boundary request, not a bug reproduction. Source inspection gives high-confidence proof that current main lacks the public schema, spawn-param carrier, Gateway field, and policy merge path for per-spawn restrictions.

## Solution Assessment

Unclear until maintainers approve the final public API and precedence rules. The narrowest maintainable direction is to reuse the existing embedded-runner allowlist where valid while defining deny/no-tools semantics and fail-closed unsupported-runtime behavior.
