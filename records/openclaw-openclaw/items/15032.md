## What This Changes

The issue requests a per-call tool restriction parameter for `sessions_spawn` so spawned sub-agents can receive an enforced narrower tool set than the parent.

## Best Possible Solution

Keep this issue open as the canonical core API/security gap, then either land https://github.com/openclaw/openclaw/pull/78441 after maintainer product/security review or replace it with a narrower additive spawn-policy API that threads through schema, session metadata, resolver, tests, and docs.
