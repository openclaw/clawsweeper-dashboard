## Summary

Keep open: current main still lacks an enforced per-spawn tool restriction on `sessions_spawn`, and the related implementation candidate is open but unmerged, so this remains the canonical API/security feature request.

## What This Changes

Add a spawn-time tool policy to `sessions_spawn` so child subagents can be given a narrower enforced tool set.

## Best Possible Solution

Land one additive `sessions_spawn` tool-policy API with enforced narrowing, fail-closed unsupported runtimes, focused regression coverage, docs, changelog, and clear precedence against existing global/subagent policies.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that the public schema, native params, spawn handoff, resolver, and docs do not expose a per-spawn tool-policy path.

## Solution Assessment

Unclear until maintainers settle the API shape. Reusing existing embedded `toolsAllow` machinery looks maintainable for native runs, but deny semantics, no-tools mode, and unsupported runtime behavior still need explicit product/security approval.
