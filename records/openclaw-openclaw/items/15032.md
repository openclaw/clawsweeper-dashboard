## Summary

The central request is still missing on current main: `sessions_spawn` has no per-spawn allow/deny input, the native spawn params cannot carry one, and child sessions only inherit/globalize tool policy. An open PR is a plausible implementation path, but it is unmerged and currently conflicting, so this issue should stay open.

## What This Changes

The issue requests a per-call tool allow/deny restriction for sub-agents spawned with `sessions_spawn`.

## Best Possible Solution

Keep this issue as the canonical tracker and have maintainers review or revise the open `toolsAllow` PR into the accepted per-spawn tool-policy contract, including docs and tests for native child enforcement and unsupported-runtime behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current `sessions_spawn` has no per-spawn allow/deny field, `SpawnSubagentParams` cannot carry one, and the native spawn call forwards no caller-selected child policy.

## Solution Assessment

Unclear pending maintainer API/security approval. A narrow additive per-spawn policy parameter that persists an immutable child restriction is the right core direction, but maintainers still need to choose the shipped field shape and unsupported-runtime semantics.
