## Summary

Keep open: current main still lacks a per-spawn tool policy path for `sessions_spawn`; only global subagent policy and role/depth restrictions are wired. The closed unmerged PR #23506 and related open items remain useful context, but they do not ship this core API/security capability.

## What This Changes

The issue requests an additive per-spawn tool policy for `sessions_spawn` so each spawned sub-agent can be restricted to an allowlist or denylist independent of global subagent tool policy.

## Best Possible Solution

Keep this open as the canonical `sessions_spawn` capability-isolation request, then add a documented per-spawn policy contract with schema, spawn params, immutable child-session metadata, resolver merge logic, focused tests, and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the `sessions_spawn` schema, spawn params, child metadata, and subagent policy resolver have no per-spawn tool policy path.

## Solution Assessment

Unclear until maintainers settle the API shape. The narrow maintainable direction is an additive `sessions_spawn` policy contract that reconciles allow/deny with `tools: "none"` and preserves current behavior when omitted.
