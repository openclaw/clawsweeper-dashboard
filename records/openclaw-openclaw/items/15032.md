## Summary

Keep open: current main still has no trusted per-call tool policy on `sessions_spawn`; it only carries inherited/global sub-agent policy. The open related work at https://github.com/openclaw/openclaw/pull/78441 is useful but narrower and unmerged, so this issue remains the canonical product/security API gap.

## What This Changes

Request `sessions_spawn` to accept per-call tool allow/deny restrictions so spawned sub-agents can run with enforced least-privilege tool access.

## Best Possible Solution

Land one maintainer-approved additive `sessions_spawn` tool-policy contract that only narrows child capabilities, composes with inherited/global/sandbox policies, fails closed for unsupported runtimes, and updates focused tests plus https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a bug reproduction. Source inspection gives high-confidence proof that current main lacks the public schema, spawn-param carrier, session metadata, and policy merge path for per-spawn restrictions.

## Solution Assessment

Unclear until maintainers approve the final public API and precedence rules. The narrow maintainable direction is to reuse existing tool-policy/embedded-runner filtering where valid while defining deny/no-tools semantics and fail-closed unsupported-runtime behavior.
