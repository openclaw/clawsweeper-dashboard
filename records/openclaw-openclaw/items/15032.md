## Summary

Keep open: current main still lacks an enforced per-spawn tool restriction contract for `sessions_spawn`, and the current related implementation is an open PR rather than shipped behavior. The Claw Drive comment and the closed unmerged implementation attempt remain useful context, but this still needs maintainer product and security review before the issue can close.

## What This Changes

The issue requests a per-call tool allow/deny restriction for spawned sub-agents so untrusted-content workflows can isolate child-agent capabilities.

## Best Possible Solution

Review and land or revise one additive `sessions_spawn` policy contract that only narrows child capabilities, composes with existing inherited/global/sandbox policy, defines unsupported-runtime behavior, and updates focused tests plus https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a broken existing behavior. Source inspection gives high confidence that current main lacks the schema field, spawn-param carrier, persisted policy, and resolver input needed for per-spawn restrictions.

## Solution Assessment

Unclear until maintainers approve the public API and policy precedence. The maintainable direction is to reuse the existing tool-policy pipeline with additive allow/deny or no-tools semantics that can only reduce a child session's capabilities.
