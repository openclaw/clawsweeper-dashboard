## Summary

Keep open: current main still does not expose a per-spawn `sessions_spawn` tool allow/deny/no-tools contract or a durable policy carrier, and the related implementation PR at https://github.com/openclaw/openclaw/pull/78441 is still open and narrower than the original security-boundary request.

## What This Changes

The issue requests per-spawn tool restrictions for sub-agents so each child session can run with a narrowed tool set for least-privilege orchestration.

## Best Possible Solution

Keep this as the canonical tracker and land one maintainer-approved additive `sessions_spawn` policy contract that only narrows child capabilities, composes with existing policy layers, defines unsupported runtimes clearly, and ships with focused tests plus docs.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a broken existing behavior. Source inspection gives high confidence that current main lacks the schema field, spawn-param carrier, durable policy metadata, and resolver input needed for the requested contract.

## Solution Assessment

Unclear until maintainers approve the public API and policy precedence. The maintainable direction is to reuse the existing embedded-runner/tool-policy pipeline with additive narrowing semantics that cannot expand child capabilities.
