## Summary

Current main still lacks a public per-spawn tool restriction for `sessions_spawn`; a clean, proof-positive open PR now owns the implementation, so the issue remains necessary until that PR is merged, replaced, or rejected.

## What This Changes

Add per-spawn tool restrictions so each `sessions_spawn` call can enforce a narrower sub-agent tool set.

## Best Possible Solution

Resolve the linked PR with one additive spawn-policy contract that reuses existing runtime filtering, persists restrictions for native child sessions, composes without widening inherited policy, and clearly rejects unsupported enforcement paths.

## Reproduction Assessment

Yes at source level: current main’s `sessions_spawn` schema and spawn parameter type lack a caller-selected restriction, while the embedded runner already contains the lower-level enforcement primitive.

## Solution Assessment

The linked PR’s reuse of `toolsAllow` is the narrowest maintainable direction, but maintainers still need to decide the public allow/deny semantics and cross-runtime fail-closed contract.
