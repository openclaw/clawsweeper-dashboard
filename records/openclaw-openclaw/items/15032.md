## Summary

Keep open: current main still only supports global or inherited sub-agent tool policy and does not expose a trusted per-spawn allow/deny policy. The active related PR is narrower native allowlist plumbing, so this still needs maintainer product and security review.

## What This Changes

Requests an additive per-call tool policy for `sessions_spawn` so spawned sub-agents can run with least-privilege tool access for untrusted-content workflows.

## Best Possible Solution

Land one maintainer-approved additive `sessions_spawn` tool-policy contract that only narrows child capabilities, documents precedence, fails closed for unsupported runtimes, and updates focused tests plus subagents docs.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a bug reproduction. Source inspection gives high-confidence proof that current main lacks the public schema, spawn-param carrier, Gateway field, and policy merge path for per-spawn restrictions.

## Solution Assessment

Unclear until maintainers approve the final public API and precedence rules. The narrowest maintainable direction is to reuse the existing embedded-runner allowlist where valid while defining deny/no-tools semantics and fail-closed unsupported-runtime behavior.
