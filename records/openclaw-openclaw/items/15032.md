## Summary

Keep open: current main and the latest release still do not expose per-spawn tool restrictions on `sessions_spawn`, and the open native `toolsAllow` implementation is unmerged, conflicting, and still needs maintainer API/security review.

## What This Changes

Add per-spawn tool restrictions to `sessions_spawn` so spawned child agents can be constrained to a narrower allow or deny tool set.

## Best Possible Solution

Keep this issue open as the canonical per-spawn `sessions_spawn` tool-policy tracker, and resolve it by landing or replacing https://github.com/openclaw/openclaw/pull/78441 after maintainers approve the public API, precedence, persistence, and unsupported-runtime semantics.

## Reproduction Assessment

Yes, as a source-level reproduction: current main's `sessions_spawn` schema, native spawn params, and child-session policy path have no per-call tool restriction field. I did not run a live agent flow because the absence is directly visible in the current API surface.

## Solution Assessment

Unclear for the final API shape. The current open `toolsAllow` PR is a plausible implementation path, but maintainers still need to choose the shipped contract across allow-only, deny semantics, tool-less mode, plugin/MCP names, and unsupported runtimes.
