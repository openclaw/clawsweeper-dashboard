## What This Changes

Request per-spawn allow/deny tool restrictions for `sessions_spawn` sub-agents so untrusted-content workflows can isolate child capabilities.

## Best Possible Solution

Land a maintainer-approved additive spawn-time policy API that reuses existing runtime `toolsAllow` enforcement, documents precedence, covers unsupported runtimes, and adds focused schema/session/policy tests before closing this issue.

## Reproduction Assessment

Yes for the missing capability: current-main source shows the `sessions_spawn` schema, spawn params, session metadata, and policy resolver do not carry a per-spawn tool policy. I did not run a live agent because read-only source inspection is enough to prove the API surface is absent.

## Solution Assessment

Unclear on the final API name, but the best direction is an additive policy-narrowing path that reuses existing runtime `toolsAllow` enforcement instead of adding a second filter. The open PR is the current candidate and needs maintainer product/security approval before this issue can close.
