## Summary

Keep open: current main still does not expose a per-spawn enforced tool-policy parameter for `sessions_spawn`, and the active implementation candidate is still open and narrower than the original allow/deny security-boundary request.

## What This Changes

Request adds a per-spawn `sessions_spawn` tool policy, such as allow/deny lists, so child agents can be launched with least-privilege tool access.

## Best Possible Solution

Land one reviewed additive `sessions_spawn` tool-policy API with enforced final narrowing, documented precedence, focused coverage, fail-closed unsupported runtime behavior, and docs at https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable: this is a feature/security-boundary request, not a bug reproduction. Source inspection on current main gives high-confidence proof that the schema, native params, spawn handoff, policy resolver, docs, and Gateway request schema do not expose a per-spawn tool-policy path.

## Solution Assessment

Unclear until maintainers approve the final API shape. The narrowest maintainable path is one additive policy parameter that reuses existing embedded allowlist machinery where possible while defining deny/no-tools semantics and fail-closed unsupported runtimes.
