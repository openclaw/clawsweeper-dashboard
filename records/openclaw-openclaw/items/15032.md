## Summary

Current main contains the internal runtime allowlist mechanism but still does not expose per-spawn tool restrictions through `sessions_spawn`; the clean, proof-positive implementation at https://github.com/openclaw/openclaw/pull/78441 remains open, so this issue is still necessary until that PR is merged, replaced, or rejected.

## What This Changes

Requests per-spawn sub-agent tool restrictions so callers can enforce narrower capabilities for individual `sessions_spawn` invocations.

## Best Possible Solution

Land one additive allowlist-first `sessions_spawn` API through the existing runtime tool-policy path, persist it for child-session turns, reject unsupported runtimes rather than silently ignoring it, and document its composition with global policy.

## Reproduction Assessment

Yes. Current main’s public `sessions_spawn` schema has no per-call tool-policy field even though the embedded runner already has an internal `toolsAllow` filter, making the missing control-plane path directly source-reproducible.

## Solution Assessment

Yes in direction: an allowlist-first additive API reusing the existing runtime filter is the narrowest maintainable solution. Maintainers still need to settle naming, denylist composition, and unsupported-runtime behavior before landing.
