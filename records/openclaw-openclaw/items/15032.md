## Summary

Keep open: current `main` still lacks a shipped per-spawn tool restriction contract for `sessions_spawn`, while the related `toolsAllow` implementation is still an open PR rather than merged behavior. The Claw Drive comment and the closed unmerged implementation attempt remain useful context, but this needs maintainer product and security review before closure.

## What This Changes

Request an enforced per-spawn tool policy for `sessions_spawn` so spawned sub-agents can receive a narrower tool set than the parent.

## Best Possible Solution

Review and land or revise one additive `sessions_spawn` policy contract that only narrows child capabilities, composes with existing inherited/global/sandbox policy, defines unsupported-runtime behavior, and updates focused tests plus https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a broken existing behavior. Source inspection gives high confidence that current `main` lacks the schema field, spawn-param carrier, persisted per-spawn policy, and resolver input needed for the requested contract.

## Solution Assessment

Unclear until maintainers approve the public API and policy precedence. The maintainable direction is to reuse the existing tool-policy pipeline with additive narrowing semantics that cannot expand a child session's capabilities.
