## Summary

Keep open. Current main and the latest release still do not expose a caller-supplied per-spawn tool restriction on `sessions_spawn`; the existing implementation only carries already-effective parent restrictions forward, while the likely implementation path is still the open, unmerged https://github.com/openclaw/openclaw/pull/78441.

## What This Changes

The issue requests an enforced per-spawn tool policy for `sessions_spawn` so sub-agents can be launched with narrower tool access than their parent.

## Best Possible Solution

Keep this issue as the canonical core security/API gap until maintainers either land https://github.com/openclaw/openclaw/pull/78441 or choose a different per-spawn policy contract that is documented, persisted, and covered by focused tests.

## Reproduction Assessment

Yes. This is source-reproducible because current main and `v2026.6.6` expose no model-facing `sessions_spawn` field or native spawn parameter that carries a caller-supplied per-spawn tool restriction.

## Solution Assessment

Unclear until maintainers choose the public contract. A runtime-enforced per-spawn policy belongs in core `sessions_spawn`, but the allow-only `toolsAllow` shape, allow/deny object shape, ACP behavior, and persistence semantics still need maintainer/security acceptance.
