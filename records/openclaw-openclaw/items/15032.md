## Summary

Keep open: current `main` still has no shipped per-spawn tool restriction contract for `sessions_spawn`. The Claw Drive comment, closed unmerged implementation at https://github.com/openclaw/openclaw/pull/23506, and open narrower `toolsAllow` implementation at https://github.com/openclaw/openclaw/pull/78441 are useful context, but this still needs maintainer product and security review before closure.

## What This Changes

The issue requests an enforced per-call tool allow/deny policy for sub-agents spawned with `sessions_spawn`, primarily for least-privilege isolation around untrusted content.

## Best Possible Solution

Keep this issue as the canonical security/API tracker and land one additive `sessions_spawn` policy contract that only narrows child capabilities, composes with global/agent/sandbox/inherited policies, defines unsupported-runtime behavior, and updates focused tests plus https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a broken existing behavior. Source inspection gives high confidence that current `main` lacks the schema field, spawn-param carrier, persisted per-spawn policy, and resolver input needed for the requested contract.

## Solution Assessment

Unclear until maintainers approve the public API and policy precedence. The maintainable direction is to reuse the existing tool-policy/embedded-runner pipeline with additive narrowing semantics that cannot expand a child session's capabilities.
