## What This Changes

The issue requests per-spawn `sessions_spawn` tool restrictions so child agents can be constrained to a narrower enforced tool set than their parent.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/78441 or a maintainer-approved equivalent, then close this issue once the per-spawn tool policy is on main.

## Reproduction Assessment

Yes by source inspection: current main has no `sessions_spawn` per-call tool policy parameter or spawn-param carrier, so the requested call shape cannot be enforced. I did not run a live agent scenario because the missing schema and forwarding path are sufficient to prove the feature gap.

## Solution Assessment

Yes in direction, but not complete yet. The best current path is a maintainer-approved additive spawn-time narrowing API that reuses the existing tool-policy filtering and persists the child policy for later turns; the open PR is the active candidate for that path.
