## Summary

Keep open: current `main` still does not expose an enforced per-spawn tool restriction contract on `sessions_spawn`. The Claw Drive use case, closed unmerged implementation at https://github.com/openclaw/openclaw/pull/23506, and open narrower `toolsAllow` PR at https://github.com/openclaw/openclaw/pull/78441 are useful implementation context, but the canonical API/security decision has not shipped.

## What This Changes

The issue requests an additive per-spawn tool policy for `sessions_spawn` so child agents can be restricted to a narrower tool set for least-privilege workflows.

## Best Possible Solution

Keep this issue as the canonical security/API tracker and land one maintainer-approved additive `sessions_spawn` policy contract that only narrows child capabilities, composes with existing policy layers, documents unsupported runtimes, and has focused tests plus docs.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a broken existing behavior. Source inspection gives high confidence that current `main` lacks the schema field, spawn-param carrier, durable policy metadata, and resolver input needed for the requested contract.

## Solution Assessment

Unclear until maintainers approve the public API and policy precedence. The maintainable direction is to reuse the existing embedded-runner/tool-policy pipeline with additive narrowing semantics that cannot expand child capabilities.
