## Summary

Keep open. Current main still lacks an enforced per-spawn tool restriction API for `sessions_spawn`; the only implemented controls are global sub-agent tool policy plus depth/role restrictions, and the linked implementation PR is closed unmerged.

## What This Changes

Add an enforced per-spawn tool policy parameter to `sessions_spawn` so each child session can be narrowed independently of global sub-agent policy.

## Best Possible Solution

Implement a narrow additive `sessions_spawn` policy that is parsed by the tool schema, stored immutably on child sessions, enforced by `resolveSubagentToolPolicyForSession`, tested, and documented with semantics aligned to #23906.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: current main has no schema field, spawn parameter, session metadata field, or policy resolver input for per-spawn tool restrictions, so passing `tools` or `denyTools` to `sessions_spawn` cannot enforce the requested isolation.

## Solution Assessment

Unclear on the exact spelling, but the best direction is a core additive API rather than prompt-only instructions or a plugin workaround. Maintainers should reconcile this issue's allow/deny object with #23906's `tools: "none"` use case before implementation.
