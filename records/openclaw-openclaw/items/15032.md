The issue asks for per-call tool allow/deny restrictions on `sessions_spawn` sub-agents to enforce least-privilege isolation for workflows such as web-search pipelines.

## Best Possible Solution

Keep this as the canonical tracker and land one maintainer-approved additive `sessions_spawn` policy contract that only narrows child capabilities, composes with existing policy layers, and ships with focused tests plus docs.

Unclear until maintainers approve the public API and policy precedence. The maintainable direction is an additive narrowing layer that reuses existing embedded-runner allowlist plumbing where applicable without widening child capabilities.
