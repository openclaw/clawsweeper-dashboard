## What This Changes

The issue requests a per-spawn allow/deny or tool-list parameter so child agents can be given a narrower enforced tool set than their parent.

## Best Possible Solution

Keep this issue as the canonical tracker and land one maintainer-approved additive `sessions_spawn` policy contract that only narrows child capabilities, composes with existing policy layers, and ships with focused tests plus docs.

## Solution Assessment
