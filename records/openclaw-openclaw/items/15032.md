## What This Changes

The issue asks for `sessions_spawn` to accept per-call tool restrictions so child agents can run with a narrower enforced tool set than the parent.

## Best Possible Solution

Land one additive maintainer-approved `sessions_spawn` tool-policy contract that only narrows child capabilities, composes with existing inherited/global/sandbox policy, defines unsupported-runtime behavior, and updates tests plus https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable: this is a feature and security-boundary request, not a broken existing behavior. Source inspection shows current main lacks the schema field, spawn-param carrier, persisted policy, and resolver input needed for per-spawn restrictions.

## Solution Assessment

Unclear until maintainers approve the public API and policy precedence. The maintainable direction is to reuse the existing tool-policy pipeline while defining additive allow/deny or no-tools semantics that can only reduce a child session's capabilities.
