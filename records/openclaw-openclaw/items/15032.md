## Summary

Keep open. Current main and the latest release still expose only global or inherited sub-agent tool policy, while the remaining work is now best handled by reviewing the open implementation candidate at https://github.com/openclaw/openclaw/pull/78441.

## What This Changes

Request adding a per-spawn tool policy to `sessions_spawn` so each spawned child agent can be restricted to an allow/deny subset of tools.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers land or explicitly reject an additive `sessions_spawn` per-spawn tool policy, preferably by reviewing and narrowing https://github.com/openclaw/openclaw/pull/78441 or an equivalent implementation with docs and focused regression proof.
