## Summary

Current main and the latest release still lack a public per-spawn tool restriction for `sessions_spawn`. The clean, proof-positive linked implementation at https://github.com/openclaw/openclaw/pull/78441 now owns this work, so the paired issue should remain open until that PR lands or maintainers reject its API direction.

## What This Changes

Add an additive per-spawn tool restriction to `sessions_spawn` so a native child agent can receive an enforced subset of its parent’s tools.

## Best Possible Solution

Land one additive native-spawn allowlist contract through the canonical stored-capability and runtime enforcement path, preserving explicit empty lists, persisting restrictions for later child turns, rejecting unsupported runtimes, and leaving omitted calls unchanged.

## Reproduction Assessment

Yes, at source level: current main’s public `sessions_spawn` schema lacks a per-call tool restriction, and its policy resolver consumes stored capabilities plus global configuration. This is a clearly verifiable missing capability rather than a failing existing contract.

## Solution Assessment

Yes in direction: extending the existing `toolsAllow` enforcement and canonical capability-store path is narrower and safer than adding a parallel policy engine. Maintainers still need to approve the public name, composition semantics, persistence, and unsupported-runtime behavior before landing.
