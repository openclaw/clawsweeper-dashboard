## What This Changes

The issue requests an enforced per-spawn tool restriction parameter for `sessions_spawn` so spawned sub-agents can run with a narrower tool set than the parent.

## Best Possible Solution

Keep this issue open as the canonical `sessions_spawn` per-spawn tool-restriction gap, then land or revise the active `toolsAllow` implementation after maintainer API, compatibility, and security-boundary approval.

## Reproduction Assessment

Yes, as a source-level feature gap: current main has no model-facing or native `sessions_spawn` parameter that restricts tools per spawn. The existing inherited allow/deny lists and `tools.subagents.tools` policy are global or session-derived rather than per-call.

## Solution Assessment

Yes for issue state: keeping this open is the best cleanup outcome because current main and v2026.6.6 do not implement the request and the related PR is open but conflicting. The implementation should reuse the existing embedded-runner allowlist and session-policy pipeline rather than adding a separate prompt-only restriction.
