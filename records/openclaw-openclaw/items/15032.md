## Summary

Keep open: current main and v2026.6.6 still do not expose a per-spawn tool restriction on `sessions_spawn`, while the likely implementation path is an open PR that still needs maintainer acceptance of the public API and security-boundary semantics.

## What This Changes

The issue requests a per-call tool allow or deny policy for spawned sub-agents so child sessions can be isolated from tools they should not use.

## Best Possible Solution

Keep this issue open as the canonical user/security request until maintainers land https://github.com/openclaw/openclaw/pull/78441 or choose a different per-spawn tool-policy contract.

## Reproduction Assessment

Yes, at source level: current main and the latest release expose no model-facing `sessions_spawn` per-spawn tool restriction, and the direct spawn path has no policy carrier from tool input. This is a missing feature/security boundary rather than a broken existing contract.

## Solution Assessment

Yes at the direction level: an additive public spawn-time policy that narrows and persists child capabilities is the maintainable solution. The exact API shape still needs maintainer approval because allowlist-only and allow/deny-object semantics have different compatibility and security tradeoffs.
