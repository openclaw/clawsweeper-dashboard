## Summary

Keep open: current main still lacks an enforced per-spawn tool policy for `sessions_spawn`, and the active related PR is a narrower allowlist implementation rather than a merged replacement for the broader allow/deny security-boundary request.

## What This Changes

Add a per-call tool policy to `sessions_spawn` so spawned sub-agents can be restricted to specific allowed tools or denied tools.

## Best Possible Solution

Land one reviewed additive `sessions_spawn` tool-policy contract that only narrows child capabilities, documents precedence and unsupported-runtime behavior, and updates focused tests plus the subagents docs.

## Reproduction Assessment

Not applicable: this is a feature/security-boundary request, not a bug reproduction. Source inspection on current main shows the public schema, spawn-param carrier, session metadata, and policy resolver still lack per-spawn restrictions.

## Solution Assessment

Unclear until maintainers approve the final API and precedence rules. The maintainable direction is an additive narrowing-only policy where deny wins and unsupported runtimes fail closed.
