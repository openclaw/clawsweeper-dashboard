## Summary

Keep open: current main still does not expose an enforced per-spawn tool policy for `sessions_spawn`, and the active related PR is open and narrower than the original allow/deny security-boundary request.

## What This Changes

Request an optional per-call tool restriction API for spawned sub-agents so each child session can run with least-privilege tool access.

## Best Possible Solution

Land one reviewed additive `sessions_spawn` tool-policy contract that enforces final child capability narrowing, documents precedence, fails closed on unsupported runtimes, and updates focused tests plus https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Not applicable: this is a feature/security-boundary request, not a bug reproduction. Source inspection gives high-confidence proof that current main lacks the public schema, spawn-param carrier, Gateway field, and policy merge path for per-spawn tool restrictions.

## Solution Assessment

Unclear until maintainers approve the final API shape. The narrowest maintainable direction is to reuse the existing embedded-runner allowlist primitive where possible while defining deny/no-tools semantics and fail-closed unsupported runtime behavior.
