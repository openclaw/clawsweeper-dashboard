## Summary

## What This Changes

The issue requests an additive per-spawn tool restriction API for `sessions_spawn`, so spawned sub-agents can receive a narrower tool set than the parent.

## Best Possible Solution

Keep this as the canonical tracker, and review the active implementation candidate against a maintainer-approved API that defines allow, deny or no-tools semantics, ACP/native behavior, plugin/MCP resolution, and deny-wins precedence before closing the issue.
