## Summary

Keep open: current `main` still lacks a per-spawn tool policy path for `sessions_spawn`; only global subagent policy plus stored role/depth restrictions are wired, and the closed prior implementation PR did not merge.

## What This Changes

The issue requests a per-spawn tool policy for `sessions_spawn` so callers can hard-limit a child sub-agent’s tools for least-privilege workflows.

## Best Possible Solution

Keep this as the canonical `sessions_spawn` capability-isolation request and implement an additive core policy contract that threads through schema parsing, spawn params, immutable child-session metadata, policy resolution, tests, and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current `main` high-confidently shows no per-spawn policy carrier in the tool schema, spawn params, child metadata, session schema, resolver, or docs.

## Solution Assessment

Unclear until maintainers settle the API shape. The narrow maintainable direction is an additive `sessions_spawn` policy parameter that preserves default behavior and composes predictably with global deny-wins policy.
