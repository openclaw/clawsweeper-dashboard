## Summary

Keep open: current main and the latest shipped release still lack a per-spawn `sessions_spawn` tool allow/deny/no-tools contract, while the related implementation at https://github.com/openclaw/openclaw/pull/78441 is still open and requires product/security approval before it can close this issue.

## What This Changes

Request adds a per-call `sessions_spawn` tool policy so spawned sub-agents can run with least-privilege tools for untrusted-content workflows.

## Best Possible Solution

Keep this as the canonical tracker and land one maintainer-approved additive `sessions_spawn` policy contract that only narrows child capabilities, composes with existing policy layers, defines unsupported runtimes clearly, and ships with focused tests plus docs.

## Reproduction Assessment

## Solution Assessment
