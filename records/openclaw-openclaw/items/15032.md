## Summary

Keep open: current main and v2026.6.10 still lack a public per-spawn tool restriction for `sessions_spawn`, and the active implementation candidate is open rather than merged. This remains the canonical security/API tracker until maintainers accept and land the API or choose a different boundary.

## What This Changes

Issue requests a per-spawn tool restriction API for `sessions_spawn` so child agents can run with narrower tool access than their parent.

## Best Possible Solution

Keep this issue as the canonical tracker, then land or replace https://github.com/openclaw/openclaw/pull/78441 after maintainers accept the API, persistence, compatibility, and security-boundary contract.

## Reproduction Assessment

Yes. Source inspection on current main shows the `sessions_spawn` schema, native spawn params, child-session patch, and subagent policy resolver have no caller-selected per-spawn tool restriction field.

## Solution Assessment

Unclear for the final API shape. The open `toolsAllow` PR is a plausible implementation, but maintainers still need to choose and approve the shipped contract for allow/deny semantics, empty-list behavior, persistence, and unsupported runtimes.
