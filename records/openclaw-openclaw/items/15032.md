## Summary

Keep open. Current main and the latest release still do not expose a per-spawn tool restriction on `sessions_spawn`, and the open implementation candidate has not merged, so the security/API gap remains.

## What This Changes

The issue requests an additive `sessions_spawn` option to restrict or deny tools for each spawned sub-agent.

## Best Possible Solution

Keep this issue as the canonical tracker and land a maintainer-approved additive `sessions_spawn` tool-policy API with immutable child-session metadata, policy tests, live proof, and docs.

## Reproduction Assessment

Yes for source proof, though this is a feature request rather than a broken existing contract. Current main has no model-facing schema field, spawn parameter, or child-session policy carrier for per-spawn tool restrictions.

## Solution Assessment

Yes, keeping this open is the right cleanup outcome. The implementation should be a core additive `sessions_spawn` API because the requested control cannot be supplied by a plugin or global config alone.
