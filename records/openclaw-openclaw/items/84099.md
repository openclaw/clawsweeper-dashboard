Keep open. Current main has partial readable/export/search paths, but it does not provide the requested full, date-organized Markdown archive alongside the JSONL session store; privacy, retention, and file-layout choices make this a product feature rather than an automatic bug fix.

## What This Changes

Requests a human-readable, date-organized Markdown archive of chat/session transcripts alongside existing JSONL session storage.

## Best Possible Solution

Define an opt-in, documented transcript archive/export design that reuses the canonical JSONL reader and shared sanitization, keeps JSONL as the source of truth, and specifies file layout, retention, and privacy behavior.

Not applicable; this is a feature request, not a broken existing contract. Source and docs show the current behavior is JSONL storage plus partial export/search helpers.

## Solution Assessment
