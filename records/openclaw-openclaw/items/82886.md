## Summary

Keep open: current main and the latest release still drop Slack block-action thread context when the thread timestamp is only on the message payload, Slack has no heartbeat typing/status adapter, and an open linked PR already references this issue as the fix path.

## What This Changes

Slack interactive reply button/select clicks should show native assistant status/typing for the follow-up turn in the relevant Slack thread.

## Best Possible Solution

Review and land the focused Slack plugin fix in https://github.com/openclaw/openclaw/pull/82895 if its implementation and proof are acceptable, reconcile the overlap with https://github.com/openclaw/openclaw/pull/76185, then close this issue after merge.

## Reproduction Assessment

Yes from source, though not from a live Slack workspace in this review: a block action with `message.thread_ts` and no `container.thread_ts` parses with no `threadTs`, and the queued heartbeat follow-up then has no thread target for Slack status/typing.

## Solution Assessment

Yes: the narrow maintainable solution is to keep the fix in the Slack plugin boundary by preserving message-level thread timestamps and adding Slack heartbeat typing/status support for thread-targeted follow-ups.
