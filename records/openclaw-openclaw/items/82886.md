## What This Changes

Slack interactive button/select follow-up turns should show native Slack assistant status/typing in the relevant thread.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82895, or an equivalent Slack-plugin fix that preserves `message.thread_ts` into session/delivery context and adds Slack heartbeat set/clear status support, then close this issue after merge.

## Reproduction Assessment

Yes, source-reproducible: a Slack block action with `message.thread_ts` and no `container.thread_ts` parses `threadTs` as undefined on current main, then stores that missing value in the queued event delivery context. I did not run a live Slack workspace reproduction.

## Solution Assessment

Yes: the narrow maintainable fix belongs in the Slack plugin boundary by preserving the thread timestamp and implementing the existing heartbeat typing adapter contract. The issue should remain open while the linked draft PR is reviewed and proven.
