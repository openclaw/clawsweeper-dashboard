## Summary

Keep open: current main still loses Slack interaction thread context when the thread timestamp is only on the message payload, and the linked draft fix PR at https://github.com/openclaw/openclaw/pull/82895 is the right closure path once it is reviewed and landed.

## What This Changes

Slack interactive reply follow-up turns should show native Slack assistant thread status/typing like ordinary Slack message turns.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82895 or an equivalent Slack-plugin fix that preserves `message.thread_ts` and implements heartbeat status updates, then close this issue after merge.

## Reproduction Assessment

Yes from source inspection, but not from a live Slack workspace: a block-action payload with `message.thread_ts` and no `container.thread_ts` loses the thread before the queued heartbeat follow-up can target Slack status.

## Solution Assessment

Yes: the narrow maintainable fix is in the Slack plugin boundary, preserving the thread timestamp and using the existing heartbeat adapter contract to call Slack assistant thread status APIs.
