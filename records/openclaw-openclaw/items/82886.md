## Summary

Keep open: current main still drops Slack block-action thread context when the thread timestamp is only on the message payload, and Slack does not currently expose a heartbeat typing/status adapter for system-event follow-up turns.

## What This Changes

Slack interactive reply button/select clicks should preserve thread context and show Slack assistant typing/status for the follow-up agent turn.

## Best Possible Solution

Keep the fix in the Slack plugin boundary: preserve interaction thread timestamps from Slack payloads and add Slack heartbeat typing/status support without changing core heartbeat stale-thread policy.

## Reproduction Assessment

Yes from source, but not from a live Slack workspace: a block action with `message.thread_ts` and no `container.thread_ts` loses the thread before the queued heartbeat follow-up, so the status path has no thread target.

## Solution Assessment

Yes: the narrow maintainable fix is to keep the thread fact in the Slack interaction path and teach the Slack channel heartbeat adapter to set/clear assistant status for thread-targeted follow-ups.
