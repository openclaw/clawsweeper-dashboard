Slack interactive reply button/select follow-up turns should preserve Slack thread context and show native Slack assistant status/typing in the relevant thread.

## Best Possible Solution

## Solution Assessment

Yes, the current linked fix direction is the narrow maintainable solution: preserve the Slack payload's thread timestamp in the interaction path and implement Slack's existing heartbeat typing adapter contract. The issue should stay open until the linked PR lands or is replaced.
