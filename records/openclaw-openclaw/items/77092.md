- **[P2] Clear context engines on fresh active loads:** `src/plugins/loader.ts:1488`
  - body: The cache-hit path now restores context engines, but fresh active reloads still use `clearActivatedPluginRuntimeState()`, which clears commands, compaction, and memory state without clearing plugin-owned context engines. If a context-engine plugin is disabled, removed, or loaded under a cache miss, the old engine can remain globally registered and still satisfy `plugins.slots.contextEngine`.
  - confidence: 0.78

  - body: The new timeout tests return a promise that never resolves and only check `signal.aborted` before the timeout fires. Since `fetchWithTimeout` just passes the signal into fetch and awaits it, advancing fake timers aborts the signal but the mock never rejects, so the test can hang until Vitest times out.
  - confidence: 0.93

  - body: Adding `channels.telegram.network.httpTimeoutMs` to the schema/type without updating generated channel config metadata and Telegram UI hints leaves Control UI/config surfaces blind to the new key and is likely to fail config metadata drift checks. Regenerate/update those surfaces, or keep this unrelated option out of this PR.
  - confidence: 0.82

  - body: This branch contains user-facing fixes for the context-engine fallback and other surfaces but does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing fix, feat, and perf changes before merge.
