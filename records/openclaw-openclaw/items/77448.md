## Summary

Keep open: current main and v2026.6.10 still handle explicit `MEDIA:` directives and structured media fields, but they do not implement the requested channel-neutral, success-gated guard for generic local-path prose, parsed metadata, and sanitized attachment-failure behavior.

## What This Changes

Add a channel-neutral outbound guard that turns validated generated artifact references into attachments and strips raw path/directive text only after delivery outcome is known.

## Best Possible Solution

Define a core-owned outbound artifact delivery contract that reuses existing media access policy, supports explicit and structured artifact sources plus safe path prose, and suppresses raw directives only after attachment delivery succeeds or fails cleanly.
