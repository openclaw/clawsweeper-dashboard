## Summary

Keep open: current main and the latest release still handle explicit `MEDIA:` or structured media inputs, but they do not implement the requested success-gated, channel-neutral artifact guard for generic path prose and sanitized attachment-failure behavior.

## What This Changes

The issue requests a channel-neutral guard that validates generated artifact references, delivers them as attachments, and strips unsafe raw path/directive text only after delivery succeeds.

## Best Possible Solution

Define a core-owned outbound artifact delivery contract that reuses existing media access policy, handles explicit and structured artifact sources plus safe path prose, and only suppresses raw directives after successful attachment delivery.

## Reproduction Assessment

Yes, at source level: current main handles explicit `MEDIA:` and structured media fields but leaves ordinary local-path prose as text. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear. A channel-neutral final-mile guard is plausible, but maintainers still need to choose the hook location, trusted artifact scopes, and sanitized failure UX before implementation.
