## Summary

Keep open: current main and v2026.6.10 support explicit `MEDIA:` directives and structured media fields, but they still do not provide the requested channel-neutral, success-gated guard for generic local-path prose and sanitized attachment-failure behavior.

## What This Changes

The issue requests a channel-neutral outbound artifact delivery guard that converts validated generated file references into real attachments and sanitizes final chat text based on delivery success.

## Best Possible Solution

Define a core-owned outbound artifact delivery contract that reuses existing media access policy, handles explicit and structured artifact sources plus safe path prose, and suppresses raw directives only after successful attachment delivery.

## Reproduction Assessment

Yes, at source level: current main handles explicit `MEDIA:` and structured media fields but leaves ordinary local-path prose as text. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear. A channel-neutral final-mile guard is plausible, but maintainers still need to choose the hook location, trusted artifact scopes, and sanitized failure UX before implementation.
