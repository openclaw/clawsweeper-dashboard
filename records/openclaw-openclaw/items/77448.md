## Summary

Keep open: current main supports documented `MEDIA:` directives, structured media payloads, media staging, and outbound delivery, but it does not implement the broader channel-neutral raw-path artifact guard with send-success-gated stripping requested here. The related open PR is adjacent transcript hygiene work and explicitly excludes outbound delivery.

## What This Changes

Request to add a channel-neutral final-mile guard that detects generated artifact references, validates allowed roots, delivers attachments, and strips or replaces unsafe raw path text based on send outcome.

## Best Possible Solution

Define an additive core outbound artifact-normalization policy around outbound payload planning and delivery, reuse existing media staging/access checks, and cover raw text, structured blocks, metadata media, success stripping, and failure fallback in synthetic-channel tests.

## Reproduction Assessment

Not applicable as a feature/design request rather than a bug with an already-defined expected behavior. Source inspection shows current main handles documented `MEDIA:` and structured media paths, but not generic `File:`/localized raw-path final prose or send-success-gated stripping.

## Solution Assessment

Unclear: the requested direction is plausible, but the best implementation depends on maintainer decisions about hook location, path grammar, allowed roots, and failure payload behavior. A safer path is an additive core helper that reuses existing `ReplyPayload.mediaUrls` staging and `deliverOutboundPayloads` delivery semantics.
