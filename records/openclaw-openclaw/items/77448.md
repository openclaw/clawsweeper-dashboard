## Summary

Keep open: current main handles documented `MEDIA:` directives and structured payload media, but the requested generic final-mile guard for raw path claims, configurable roots, success-gated stripping, and failure payloads is still a product/security design feature rather than implemented behavior.

## What This Changes

The issue requests a channel-neutral outbound artifact delivery guard that converts validated generated artifact references into real attachments and removes unsafe raw path/directive prose only after successful delivery.

## Best Possible Solution

Keep this as a design issue; if accepted, add a small centralized outbound artifact-normalization layer that reuses existing media staging/access checks and defines a narrow grammar, allowed roots, send-success stripping, and failure fallback semantics before channel-specific patches.
