## Summary

Keep open: current main handles explicit `MEDIA:` directives and known payload media, but it still does not provide the requested generic final-mile artifact guard for arbitrary local path prose, configurable generated-artifact roots, send-success-gated stripping, and safe delivery-failure fallback.

## What This Changes

Request to add a channel-neutral outbound guard that converts validated generated file/media references into real channel attachments and removes unsafe raw delivery text only after successful send.

## Best Possible Solution

Keep this as a design issue until maintainers define a central outbound artifact-normalization policy with a narrow grammar, explicit trust roots, reuse of existing media staging/access checks, send-success-dependent text stripping, and observable failure semantics.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a bug against an already-defined OpenClaw contract. Source inspection gives a high-confidence gap analysis because documented `MEDIA:` and explicit payload media are handled, while arbitrary raw path claims are not.

## Solution Assessment

Unclear: the requested direction is plausible, but the exact grammar, allowed roots, hook location, and failure payload behavior need maintainer product/security decisions. The safer path is an additive central helper that reuses existing media staging and delivery outcomes rather than channel-specific patches.
