## Summary

Keep open: current main handles documented `MEDIA:` directives and explicit payload media, but it does not implement the requested generic final-mile guard for arbitrary raw path claims, configurable artifact roots, success-gated stripping, and safe failure payloads. This is a product/security design feature, not a stale or already-implemented bug fix.

## What This Changes

Add a channel-neutral guard that converts validated generated artifact references into delivered attachments while preventing unsafe raw path or directive text from leaking into final chat.

## Best Possible Solution

Keep this as a design issue until maintainers define a central outbound artifact-normalization policy with a narrow grammar, explicit trust roots, reuse of existing media staging/access checks, send-success-dependent text stripping, and observable failure semantics.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a bug against an already-defined OpenClaw contract. Source inspection gives a high-confidence gap analysis because documented `MEDIA:` and explicit payload media are handled, while arbitrary raw path claims are not.

## Solution Assessment

Unclear: the requested direction is plausible, but the exact grammar, allowed roots, hook location, and failure payload behavior need maintainer product/security decisions. The safer path is an additive central helper that reuses existing media staging and delivery outcomes instead of channel-specific patches.
