## Summary

Keep open: current main handles documented `MEDIA:` directives and structured tool media, but it does not implement the broader final-mile guard for raw `File:`/localized local-path claims, configurable artifact roots, or send-success-gated stripping. The linked #77445 is adjacent transcript hygiene work and explicitly excludes outbound delivery.

## What This Changes

Add a channel-neutral guard that validates generated artifact references in final assistant output, sends them as attachments, and strips or replaces raw path/directive text based on send success.

## Best Possible Solution

Define an additive channel-neutral outbound artifact normalization layer around the outbound plan/delivery boundary that reuses existing media staging, documents the safe path grammar and roots, and adds synthetic-channel success/failure coverage.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a report of broken documented behavior. Source inspection shows current main's defined contract covers `MEDIA:` and structured media paths, not generic raw local-path claims.

## Solution Assessment

Unclear: the requested direction is plausible, but the best implementation depends on maintainer choices for hook location, path grammar, allowed roots, and failure payloads. A narrow additive helper that feeds existing media staging/delivery is safer than channel-specific patches.
