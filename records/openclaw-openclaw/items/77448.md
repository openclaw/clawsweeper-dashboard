## Summary

Keep open: current main handles documented `MEDIA:` directives, explicit `mediaUrls`, and some structured tool-result media, but it does not implement the broader channel-neutral raw-path/final-mile guard requested here. The same-author related PR is still open and explicitly excludes outbound delivery, so it is adjacent context rather than a superseding fix.

## What This Changes

Add a channel-neutral outbound guard that validates generated artifact references, sends them as attachments, strips raw path/directive text only after send success, and defines failure fallback behavior.

## Best Possible Solution

Define an additive outbound artifact-normalization layer near reply/outbound delivery that reuses existing media staging, path policy, and channel adapters, with synthetic-channel tests for success, failure, and raw-prose stripping.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a bug with already-defined expected behavior. Source inspection shows current main covers documented `MEDIA:` and structured media paths but not the broader raw-path guard requested here.

## Solution Assessment

Unclear: the requested direction is plausible, but the best implementation depends on maintainer decisions about hook location, path grammar, allowed roots, and failure payload behavior. A narrow additive helper that reuses existing `ReplyPayload` media staging is the safer direction than a channel-specific patch.
