## Summary

Keep open: current main has partial outbound media handling, but it does not implement the requested generic final-mile artifact guard for local-path claims, structured text blocks, metadata sources, and sanitized failure fallback. The related open bugs support the same problem class, while the linked PRs do not supersede this broader design issue.

## What This Changes

The issue requests a channel-neutral outbound artifact guard that parses generated-file references from final assistant output, validates them against trusted roots, sends them as attachments, and sanitizes final chat text based on delivery success.

## Best Possible Solution

Define a core-owned outbound artifact contract that reuses existing media access policy, covers final text plus structured payload sources, and sends a sanitized failure response when attachment delivery cannot be proven.

## Reproduction Assessment

Yes, from source inspection for the missing behavior: current main handles explicit `MEDIA:` directives and media fields, but ordinary local-path claims and structured text blocks are not guarded as requested. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear. The requested final-mile guard is plausible and aligned with the problem, but the hook location, scan sources, allowed roots, and failure payload need maintainer design review before implementation.
