## Summary

Keep open: current main already supports documented `MEDIA:` directives and structured media payloads, but this issue asks for a broader channel-neutral final-mile artifact guard for raw path claims, configurable roots, send-success-gated stripping, and failure semantics. The related open PR is adjacent transcript hygiene work and explicitly excludes outbound delivery.

## What This Changes

Add a channel-neutral outbound artifact guard that validates generated local file/media references, sends them through channel attachments, and removes unsafe raw path/directive prose only after delivery succeeds.

## Best Possible Solution

Keep this as a design issue and, if accepted, add a small generic outbound artifact-normalization layer that reuses existing media staging/delivery while defining safe parsing and failure semantics before any channel-specific patches.

## Reproduction Assessment

Not applicable as a feature/design request, not a bug with an already-defined expected behavior. Source inspection confirms current main handles documented `MEDIA:` and structured media paths but not the broader raw-path guard requested here.

## Solution Assessment

Unclear: the requested direction is plausible, but the best implementation depends on maintainer decisions about hook location, path grammar, allowed roots, and failure payload behavior. A safer path is an additive core helper with synthetic-channel coverage that reuses existing `ReplyPayload.mediaUrls` staging and `deliverOutboundPayloads`.
