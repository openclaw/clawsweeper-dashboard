## Summary

Keep open: current main supports explicit `MEDIA:` directives and structured media fields, and docs now steer tools toward structured attachments, but it still does not implement the requested generic success-gated artifact guard for ordinary local-path prose, structured assistant sources, metadata, and sanitized delivery-failure behavior. This remains useful work, but it needs maintainer product/security review before implementation.

## What This Changes

The issue requests a channel-neutral outbound guard that validates generated-artifact references in assistant output, sends them as channel attachments, and sanitizes raw path/directive text based on delivery success.

## Best Possible Solution

Define a core-owned outbound artifact delivery contract that reuses existing media access policy, covers final text plus structured payload sources, and sends a sanitized failure response when attachment delivery cannot be proven.

## Reproduction Assessment

Yes, from source inspection: current main handles explicit `MEDIA:` directives and media fields, while ordinary local-path claims and structured/metadata sources are not guarded as requested. A live channel reproduction is still missing.

## Solution Assessment

Unclear. A channel-neutral final-mile guard is plausible and better than a channel-specific patch, but the hook location, source coverage, trusted-root policy, and failure payload need maintainer design review first.
