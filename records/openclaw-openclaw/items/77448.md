## Summary

Keep open: current main has partial `MEDIA:` and `mediaUrl` handling, but it does not implement the requested channel-neutral final-mile artifact guard for ordinary local-path claims, structured text or metadata sources, and success-dependent sanitized failure behavior. Related open and closed items cover adjacent channel-specific symptoms but do not supersede this broader design issue.

## What This Changes

The issue requests a channel-neutral guard that turns validated generated artifact references in final replies into real attachments and removes unsafe raw paths or directives only after send success.

## Best Possible Solution

Define a core-owned outbound artifact delivery contract that reuses existing media access policy, covers final text plus structured payload sources, and sends a sanitized failure response when attachment delivery cannot be proven.

## Reproduction Assessment

Yes, from source inspection: current main handles explicit `MEDIA:` directives and media fields, while ordinary local-path claims and structured/metadata sources are not guarded as requested. A live channel reproduction is still missing.

## Solution Assessment

Unclear. The requested final-mile guard is plausible and better than channel-specific patches, but the hook location, scan sources, trusted roots, and sanitized failure payload need maintainer design review first.
