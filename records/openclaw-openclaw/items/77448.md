## Summary

Keep open: current main and v2026.6.8 support structured media fields and legacy `MEDIA:` parsing, but they still do not provide the requested generic success-gated artifact guard for ordinary local-path prose, structured/metadata sources, and sanitized delivery-failure behavior.

## What This Changes

Request a channel-neutral outbound artifact delivery guard that validates generated file references, sends them as attachments, and strips or replaces raw path/directive text only after delivery succeeds.

## Best Possible Solution

Define a core-owned outbound artifact delivery contract that reuses existing media access policy, covers final text plus structured payload sources, and sends a sanitized failure response when attachment delivery cannot be proven.

## Reproduction Assessment

Yes, at source level: current main parses explicit `MEDIA:` directives and structured media fields but keeps ordinary local-path prose as text. A live channel reproduction is still missing.

## Solution Assessment

Unclear. A channel-neutral final-mile guard is plausible, but the hook location, trusted-root policy, structured-source coverage, and sanitized failure payload need maintainer product/security review first.
