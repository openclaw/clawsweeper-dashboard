## Summary

Keep open: current main and v2026.6.10 still lose keyword/BM25 memory recall when FTS is unavailable, and the open related PRs cover diagnostic visibility or adjacent missing-`node:sqlite` fallback rather than a merged fix for this FTS5-unavailable path.

## What This Changes

The issue reports that OpenClaw memory search loses keyword/BM25 fallback when the active Node `node:sqlite` build lacks FTS5, degrading hybrid search to vector-only behavior.

## Best Possible Solution

Keep this issue open as the canonical FTS5-unavailable memory recall tracker until maintainers land a recall-restoring implementation or explicitly decide that documentation/doctor diagnostics are the supported degraded behavior.

## Reproduction Assessment

Yes at source level. Current main and v2026.6.10 still gate keyword/BM25 search on `this.fts.available`, so a Node runtime where `CREATE VIRTUAL TABLE ... USING fts5` fails loses keyword recall.

## Solution Assessment

No complete fix is on main. The diagnostic PR is a useful partial mitigation, and the missing-`node:sqlite` fallback PR is adjacent, but this FTS5-unavailable path still needs a maintainer-approved recall or backend strategy.
