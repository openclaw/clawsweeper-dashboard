## Summary

Keep open: current main and the latest release still skip keyword/BM25 recall whenever FTS5 is unavailable, so hybrid memory search degrades to vector-only or empty FTS-only results. The open diagnostic PR improves visibility but does not restore keyword fallback, and the durable fix still needs maintainer direction.

## What This Changes

Restore usable keyword/BM25 memory recall, or at least actionable diagnostics, when Node's built-in SQLite lacks FTS5.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers choose and land a narrow memory-core path that either restores keyword recall on FTS-missing Node builds or explicitly documents and surfaces the degraded mode.

## Reproduction Assessment

Yes. Source inspection shows that FTS5 setup failure sets `fts.available` false and current search paths then return empty FTS-only results or vector-only hybrid results; the issue comments add matching runtime logs across macOS, Linux, and Windows builds.

## Solution Assessment

Unclear. The diagnostic PR is useful but only partial; restoring the advertised keyword recall needs a maintainer-approved backend or fallback strategy rather than another silent degradation path.
