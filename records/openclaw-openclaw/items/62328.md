## What This Changes

The issue reports that memory search loses keyword/BM25 fallback on Node builds whose node:sqlite lacks FTS5 and asks for a runtime fallback, clearer documentation, or doctor visibility.

## Best Possible Solution

Keep this as the canonical FTS5 tracker until maintainers land a backend/fallback strategy that preserves keyword recall, hybrid ranking, and source filters, with the doctor PR treated as a diagnostic slice only.

Yes, at source/report level: when FTS5 table creation fails, current main records FTS unavailable and hybrid search skips keyword results. I did not reproduce the failure live because this checkout's Node v24.15.0 has FTS5 enabled.

## Solution Assessment

No current-main solution is sufficient. The diagnostic PR is useful visibility work, but restoring recall needs a maintainer-approved fallback/backend/docs path rather than closing the runtime issue.
