## Summary

Keep open: current main and the latest release still drop keyword/BM25 memory recall when FTS5 is unavailable, while the linked open PRs cover diagnostic visibility or adjacent missing-`node:sqlite` fallback rather than a merged recall fix.

## What This Changes

The issue reports that missing FTS5 support in Node's built-in SQLite breaks OpenClaw memory keyword/BM25 fallback, degrading hybrid memory search to vector-only or no keyword recall.

## Best Possible Solution

Keep this issue open as the canonical FTS5-unavailable memory recall tracker until maintainers land a recall-restoring implementation or explicitly choose diagnostic-only degraded behavior.

## Reproduction Assessment

Yes at source level. Current main still gates keyword/BM25 search on `this.fts.available`, so a Node runtime where `CREATE VIRTUAL TABLE ... USING fts5` fails loses keyword recall.

## Solution Assessment

No complete fix is on main. The existing LIKE fallback is useful for FTS `MATCH` failures after an FTS table exists, but the missing-module path still needs a maintainer-approved recall or backend strategy.
