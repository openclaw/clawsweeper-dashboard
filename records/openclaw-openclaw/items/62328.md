## Summary

Keep open. Current main still depends on `node:sqlite` FTS5 for BM25 keyword recall and only degrades hybrid search to vector-only when FTS is unavailable; the diagnostic gap is actively covered by open PR #62338, which should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks OpenClaw to handle Node built-in SQLite builds without FTS5 so hybrid memory search does not silently lose its keyword/BM25 fallback, via a fallback driver, documentation, or a more visible doctor warning.

## Best Possible Solution

Keep this open while #62338, or a replacement, is reviewed. The best end state is a narrow diagnostic/docs fix that surfaces FTS unavailability in doctor plus a separate maintainer decision on whether OpenClaw should support an alternate SQLite runtime or driver to restore keyword fallback on runtimes without FTS5.

## Reproduction Assessment

Yes. The issue gives concrete runtime environments, and current main has a direct reproduction path: a Node `node:sqlite` build that cannot create an FTS5 virtual table makes `ensureMemoryIndexSchema` record FTS unavailable, after which hybrid search skips keyword results and returns vector-only results.

## Solution Assessment

Unclear. The open doctor-warning PR is a reasonable narrow diagnostic repair, but it is not the full behavioral fix if maintainers want exact keyword/BM25 recall to work on Node builds without FTS5; that would require a supported SQLite runtime/driver strategy or a separate fallback design.
