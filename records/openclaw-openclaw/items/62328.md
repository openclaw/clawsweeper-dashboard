## Summary

Keep open: current main still degrades hybrid memory search to vector-only when `node:sqlite` lacks FTS5, and the active doctor-warning follow-up remains open as #62338 rather than merged.

## What This Changes

The issue asks OpenClaw to restore or visibly diagnose missing FTS5 support in `node:sqlite` so BM25 keyword fallback does not disappear silently from hybrid memory search.

## Best Possible Solution

Keep this open while #62338, or a replacement, is reviewed; the desired end state is visible doctor/docs guidance plus a deliberate maintainer decision on whether to restore keyword recall through an alternate SQLite runtime or supported Node requirement.

## Reproduction Assessment

Yes. A `node:sqlite` runtime that cannot create an FTS5 virtual table makes `ensureMemoryIndexSchema` record FTS unavailable, after which hybrid search skips keyword results and returns vector-only results.

## Solution Assessment

Unclear. A doctor warning is a good narrow diagnostic fix, but it does not restore BM25 keyword recall on runtimes whose built-in SQLite lacks FTS5.
