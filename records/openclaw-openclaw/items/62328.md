## Summary

Keep open: current main still requires FTS5 for builtin keyword/BM25 recall and degrades hybrid memory search to vector-only when FTS is unavailable; the linked diagnostic PR remains open and does not restore keyword recall.

## What This Changes

The issue reports that builtin memory search loses FTS/BM25 keyword fallback on Node builds whose `node:sqlite` lacks FTS5 or blocks extension loading.

## Best Possible Solution

Land or replace #62338 for visible diagnostics, then choose a maintained recall strategy such as a supported-runtime requirement, non-FTS keyword fallback over chunks, or an alternate SQLite implementation.

## Reproduction Assessment

Yes. On a Node runtime whose `node:sqlite` fails `CREATE VIRTUAL TABLE ... USING fts5`, current source records FTS unavailable and hybrid search skips keyword results.

## Solution Assessment

Unclear. The doctor warning path is a good narrow diagnostic step, but restoring BM25 recall needs a maintainer-chosen SQLite/runtime or non-FTS fallback strategy.
