## Summary

Keep open: current main still relies on SQLite FTS5 for builtin keyword/BM25 recall, and when `node:sqlite` lacks FTS5 the hybrid path degrades to vector-only while the doctor status path still does not surface FTS state.

## What This Changes

Restore or visibly diagnose memory search keyword/BM25 fallback when `node:sqlite` lacks FTS5.

## Best Possible Solution

Keep this open until a doctor/docs diagnostic lands and maintainers choose the supported strategy for restoring lexical recall or declaring the runtime requirement.

## Reproduction Assessment

Yes. A runtime where `node:sqlite` fails `CREATE VIRTUAL TABLE ... USING fts5` drives `ensureMemoryIndexSchema` to `ftsAvailable: false`, after which hybrid search bypasses keyword lookup and returns vector-only results.

## Solution Assessment

Unclear. The diagnostic PR is a narrow useful step, but restoring exact-token recall needs a maintainer choice among a supported Node runtime requirement, a chunks-table LIKE fallback, or an alternate SQLite implementation.
