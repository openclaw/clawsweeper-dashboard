## Summary

Keep open: current main still drops keyword/BM25 recall whenever FTS is unavailable, and the linked diagnostic PR is open and conflicting rather than a merged fix. The issue remains the best tracker for the memory-core backend or fallback decision.

## What This Changes

The issue reports that memory_search degrades when Node's built-in SQLite cannot create FTS5 tables, leaving hybrid search without its keyword/BM25 fallback.

## Best Possible Solution

Land a maintainer-approved memory-core fix that preserves lexical recall when FTS5 is unavailable, likely by reviewing the broader SQLite backend replacement or designing a bounded text-scan fallback, with doctor/docs aligned to the chosen behavior.

## Reproduction Assessment

Yes from source: current main returns no keyword results when `this.fts.available` is false, matching the reported missing-FTS5 degradation. The exact runtime failure was not reproduced locally because this checkout runs on Node v24.16.0 with FTS5 available.

## Solution Assessment

No complete fix is on main yet. The doctor warning PR is useful but not sufficient, while a backend replacement or bounded scan fallback needs maintainer approval for dependency, storage, performance, and upgrade behavior.
