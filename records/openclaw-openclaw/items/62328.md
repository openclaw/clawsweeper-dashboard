## Summary

Keep open: current main still depends on `node:sqlite` FTS5 for builtin keyword/BM25 recall, still degrades hybrid search to vector-only when FTS is unavailable, and the linked doctor-warning PR is open and unmerged rather than a shipped fix.

## What This Changes

Reports that builtin memory hybrid search loses keyword/BM25 recall when `node:sqlite` lacks FTS5, and asks for a fallback, clearer doctor warning, or documented runtime guidance.

## Best Possible Solution

Keep this issue as the canonical tracker until https://github.com/openclaw/openclaw/pull/62338 or a replacement lands for diagnostics, then make a maintainer decision on restoring keyword recall through a supported runtime requirement, alternate SQLite implementation, or documented limitation.

## Reproduction Assessment

Yes by source and reported dependency behavior, though this local Node 24.14.1 runtime has FTS5 enabled. On a Node build where `CREATE VIRTUAL TABLE ... USING fts5` fails, current main records FTS unavailable and hybrid search skips keyword/BM25 results.

## Solution Assessment

Unclear. The open doctor-warning path is a useful narrow diagnostic improvement, but restoring keyword/BM25 recall on affected Node builds needs a deliberate maintainer choice about supported runtimes, alternate SQLite bindings, or fallback behavior.
