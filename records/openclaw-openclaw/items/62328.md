## Summary

Keep open: current main still disables builtin keyword/BM25 recall when FTS5 is unavailable, and the open diagnostic PR #62338 is unmerged and does not restore keyword recall.

## What This Changes

The issue asks OpenClaw to restore or visibly diagnose builtin memory keyword/BM25 fallback when the active node:sqlite build lacks FTS5.

## Best Possible Solution

Keep this open until #62338 or a replacement lands for diagnostics, then decide whether keyword recall should be restored through a supported-runtime requirement, LIKE fallback, or alternate SQLite implementation.

## Reproduction Assessment

Yes. A node:sqlite runtime that fails CREATE VIRTUAL TABLE ... USING fts5 makes ensureMemoryIndexSchema mark FTS unavailable, after which current hybrid search skips keyword results and returns vector-only results.

## Solution Assessment

Unclear. The open doctor warning is a good narrow diagnostic step, but restoring BM25 recall requires a maintainer choice among runtime requirements, a non-FTS lexical fallback, or a different SQLite implementation.
