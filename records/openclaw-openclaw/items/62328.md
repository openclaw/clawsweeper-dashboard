## Summary

Keep open: current main still requires SQLite FTS5 for keyword/BM25 memory search and degrades hybrid search to vector-only when FTS is unavailable; the linked open PR only covers doctor visibility and has not merged.

## What This Changes

The issue reports that `node:sqlite` builds without FTS5 break the keyword/BM25 side of hybrid memory search and asks for a runtime fallback, documentation, or louder diagnostics.

## Best Possible Solution

Keep this as the canonical FTS5/runtime tracker, land or replace the diagnostic PR for doctor visibility, and separately decide how builtin memory should preserve exact-token recall on supported runtimes without FTS5.

## Reproduction Assessment

Yes, source-reproducible for affected runtimes: when `CREATE VIRTUAL TABLE ... USING fts5` throws, current main records FTS unavailable and hybrid search skips keyword results. This checkout's Node has FTS5 enabled, so I did not directly reproduce the platform build failure locally.

## Solution Assessment

No current-main solution is sufficient. The linked diagnostic PR is a good narrow visibility change, but the underlying recall degradation still needs a maintainer decision on runtime support or fallback strategy.
