## Summary

Keep open: current main still treats missing SQLite FTS5 as an unavailable keyword index and hybrid memory search still degrades to vector-only on that path, while the linked doctor visibility PR remains open rather than merged.

## What This Changes

The issue requests restored or clearly surfaced keyword/BM25 memory recall when `node:sqlite` lacks FTS5 support.

## Best Possible Solution

Keep this open until maintainers either land a doctor/docs diagnostic slice and decide the supported runtime requirement, or implement a keyword fallback/SQLite runtime strategy that restores lexical recall when FTS5 is unavailable.

## Reproduction Assessment

Yes, by using a `node:sqlite` build where `CREATE VIRTUAL TABLE ... USING fts5` throws `no such module: fts5`; source inspection shows that condition records FTS as unavailable and hybrid search then skips keyword results. I did not reproduce it on this host because Node v24.14.1 has FTS5 enabled.

## Solution Assessment

Unclear. The open doctor warning path is a narrow visibility improvement, but the full issue needs maintainer choice between a supported Node/runtime requirement, a non-FTS keyword fallback, or an alternate SQLite implementation.
