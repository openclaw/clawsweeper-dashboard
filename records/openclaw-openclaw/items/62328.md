## Summary

Keep open: current main still uses an FTS5-only keyword index, degrades hybrid memory search to vector-only when FTS is unavailable, and the related doctor/text-fallback PRs are still open rather than merged.

## What This Changes

Request to restore or clearly surface builtin memory keyword/BM25 recall when `node:sqlite` lacks FTS5, with possible fallback, doctor, or docs changes.

## Best Possible Solution

Keep this issue open while maintainers choose and land a supported fix that restores keyword recall on FTS-less `node:sqlite` runtimes or clearly surfaces and documents the supported runtime requirement.

## Reproduction Assessment

Yes. A runtime that fails `CREATE VIRTUAL TABLE ... USING fts5` makes current main record FTS as unavailable and skip keyword results; this runner's Node v24.14.1 has FTS5, so the failure was verified from source and issue-provided Node 22/23 repros rather than locally reproduced.

## Solution Assessment

Unclear. https://github.com/openclaw/openclaw/pull/62338 is a good narrow diagnostic step, but restoring keyword recall likely needs a deliberate fallback or SQLite-runtime decision, with https://github.com/openclaw/openclaw/pull/65435 now covering one possible fallback direction.
