## Summary

Keep open: current main still depends on SQLite FTS5 for builtin keyword/BM25 recall, degrades hybrid memory search to vector-only when FTS is unavailable, and the doctor-warning follow-up remains open rather than merged.

## What This Changes

The issue asks OpenClaw to handle or visibly diagnose missing SQLite FTS5 support in node:sqlite so hybrid memory search does not silently lose keyword fallback.

## Best Possible Solution

Keep this open while #62338, or a replacement, is reviewed; the desired end state is visible doctor/docs guidance plus a maintainer decision on whether to restore keyword recall through a supported runtime requirement, LIKE fallback, or alternate SQLite implementation.

## Reproduction Assessment

Yes. A `node:sqlite` runtime that fails `CREATE VIRTUAL TABLE ... USING fts5` makes `ensureMemoryIndexSchema` record FTS unavailable, and current hybrid search then skips keyword results and returns vector-only results.

## Solution Assessment

Unclear. The open doctor warning is a good narrow diagnostic step, but restoring BM25 recall on affected runtimes needs a deliberate SQLite runtime or fallback strategy.
