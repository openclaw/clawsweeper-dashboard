## Summary

Keep open: current main still degrades hybrid memory search to vector-only when FTS5 is unavailable, and the doctor diagnostic gap is still only covered by open PR #62338 rather than merged code.

## What This Changes

Request to restore or clearly diagnose keyword/BM25 memory search when Node's `node:sqlite` lacks FTS5, with suggested driver fallback, docs, or doctor warning.

## Best Possible Solution

Keep this issue open while #62338, or a replacement, is reviewed; the end state should at least surface FTS unavailability in doctor, with a separate maintainer decision on whether to support an alternate SQLite runtime for restored keyword recall.

## Reproduction Assessment

Yes. A Node `node:sqlite` build that cannot create an FTS5 virtual table makes `ensureMemoryIndexSchema` record FTS unavailable, after which current hybrid search skips keyword results and returns vector-only results.

## Solution Assessment

Unclear. The open doctor-warning PR is a narrow maintainable diagnostic fix, but it is not the full behavioral fix if maintainers want keyword/BM25 recall to work on runtimes without FTS5.
