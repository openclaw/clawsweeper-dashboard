## Summary

Keep open: current main still requires SQLite FTS5 for builtin keyword/BM25 recall, degrades hybrid search to vector-only when FTS is unavailable, and the linked doctor-visibility PR is still open rather than merged. Recent issue comments also report the same failure on supported Node/OpenClaw combinations, so this is not stale cleanup.

## What This Changes

The issue reports that OpenClaw hybrid memory search loses keyword/BM25 recall when the runtime `node:sqlite` build lacks SQLite FTS5 support.

## Best Possible Solution

Keep this issue as the canonical FTS5/node:sqlite tracker, land or replace the scoped doctor/docs diagnostic without closing it prematurely, then make a maintainer decision on the supported runtime, fallback, or SQLite driver strategy for restoring keyword recall.

## Reproduction Assessment

Yes. On a `node:sqlite` runtime where `CREATE VIRTUAL TABLE ... USING fts5` fails, current source records FTS unavailable and the hybrid path skips keyword results, leaving vector-only search.

## Solution Assessment

Unclear. The open doctor-warning PR is a good narrow diagnostic step, but the full issue needs a maintainer choice between documenting the runtime requirement, adding a fallback, or changing the SQLite implementation strategy.
