## Summary

Keep open: current main still relies on SQLite FTS5 for builtin keyword/BM25 recall and degrades hybrid memory search to vector-only when FTS is unavailable. The linked doctor-warning PR is still open and unmerged, and it would improve visibility rather than restore keyword recall on affected Node runtimes.

## What This Changes

Restore or visibly diagnose builtin memory keyword/BM25 search when `node:sqlite` lacks FTS5 so hybrid search does not silently become vector-only.

## Best Possible Solution

Keep this issue open while #62338 or a replacement is reviewed; the end state should include visible doctor/docs guidance plus a maintainer decision on whether to restore keyword recall through a supported runtime requirement, a non-FTS fallback, or an alternate SQLite implementation.

## Reproduction Assessment

Yes. On a `node:sqlite` runtime where `CREATE VIRTUAL TABLE ... USING fts5` throws, current source marks FTS unavailable and hybrid search skips keyword results, matching the issue’s reported `no such module: fts5` path.

## Solution Assessment

Unclear. The open doctor-warning PR is a good narrow diagnostic step, but restoring BM25 keyword recall on affected runtimes needs a deliberate maintainer choice between runtime support policy, fallback search, or a different SQLite driver.
