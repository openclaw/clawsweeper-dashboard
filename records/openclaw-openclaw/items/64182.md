## Summary

## Best Possible Solution

Keep the shipped shared `/verbose` group policy and close this stale conflicting branch; any desire for automatic command-only start-status bubbles should be raised as a fresh product/security decision.

## Reproduction Assessment

Yes for the historical central symptom: the old dispatch gate suppressed group text tool summaries, and current tests now cover the fixed group verbose paths. The exact leftover command-only start-status behavior is not a current-main bug reproduction; it is a product policy choice.

## Solution Assessment

No. Current main's shared `/verbose` gate is the better maintainable solution because it uses one channel-agnostic visibility policy, preserves quiet defaults, and avoids adding a separate authorized-command progress bypass.
