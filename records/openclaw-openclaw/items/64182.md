## Summary

Keep open. Current main still suppresses the requested authorized non-forum group command tool updates, while this PR remains useful but stale against main's newer shared verbose-progress gate and would drop Slack non-direct suppression if merged as-is.

## What This Changes

The PR adds mention-aware control-command detection and changes reply dispatch so authorized non-forum group text-command sessions can receive tool summaries and tool-start statuses, with regression tests for authorized, unauthorized, mention-prefixed, and path-like slash inputs.

## Best Possible Solution

Update the PR against current main by adding the authorized-group-command exception at the shared verbose-progress decision point, keeping Slack non-direct suppression, ordinary group-chatter suppression, unauthorized suppression, and forum-topic behavior intact, with regression coverage for both the new command path and the existing Slack guard.

## Reproduction Assessment

Yes. A focused dispatch test with `Surface`/`Provider` set to `slack`, `ChatType: "channel"`, verbose enabled, and tool/progress callbacks reproduces the regression risk: current main expects no tool result, while the PR predicate would permit summaries/statuses if merged without the Slack guard.

## Solution Assessment

No. The provider-agnostic `CommandAuthorized` plus control-command direction is the right shape, but this patch should be folded into current main's shared verbose-progress predicate instead of replacing the older group/forum check.
