## Summary

Keep open: current main and the latest shipped release still take the ack-only path for bare chat `/reset`, but that behavior is now documented and covered by tests, so restoring the prior startup-turn UX needs maintainer product direction rather than an automatic cleanup close.

## What This Changes

The issue asks to restore or opt into a Telegram DM bare `/reset` startup agent turn after the command currently only sends the reset acknowledgment.

## Best Possible Solution

Decide the product contract for bare channel resets; if startup turns should return, add an explicit compatible path or config and update the reset docs plus Telegram/native/text reset tests together.

## Reproduction Assessment

Yes, source-reproducible: Telegram native `/reset` reaches the shared reply command path, which returns `shouldContinue: false` with the ack before prepared reply/model execution. Existing tests already assert the same no-agent-call behavior.

## Solution Assessment

Unclear. The suggested fallthrough is mechanically plausible, but it would contradict current docs and tests that make bare chat resets ack-only; an explicit opt-in or maintainer-approved contract change is safer.
