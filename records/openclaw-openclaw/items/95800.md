## Summary

Keep open. Current main still has a source-backed no-feedback gap for Telegram approval requests when no approver route is configured, and the open approval-delivery PR only partially overlaps this report rather than safely superseding it.

## What This Changes

Request that Telegram native approvals explain missing `execApprovals`/`ownerAllowFrom` configuration and guide the operator instead of silently timing out or returning an unexplained denial.

## Best Possible Solution

Keep the security gate, but add an explicit no-route/no-approver explanation to the originating Telegram/operator path and coordinate any broader auto-routing behavior with approval owners.

## Reproduction Assessment

Yes at source level, but not from a live Telegram run. The code shows Telegram native approval availability is disabled without resolved approvers, and the shared no-route response lacks Telegram setup guidance.

## Solution Assessment

Unclear for the full request. Surfacing explicit no-route feedback is the right narrow fix; automatic rerouting to Web/terminal and doctor policy should be decided by approval owners.
