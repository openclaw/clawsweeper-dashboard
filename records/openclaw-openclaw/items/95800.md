## What This Changes

Telegram-originated native/plugin approval failures should explain when Telegram approval delivery is disabled because no approvers are configured.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95973 or an equivalent focused fix that appends Telegram setup guidance to plugin approval no-route and timeout failures while preserving the approver gate.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main can keep or fail a Telegram-originated plugin approval with no delivery route and returns only generic no-route or timeout text. The issue also includes three v2026.6.9 log examples of `plugin.approval.waitDecision` taking about 120 seconds before denial.

## Solution Assessment

Yes for the central bug: surfacing the existing Telegram setup guidance in the agent-facing plugin approval failure path is the narrowest maintainable fix. Auto-routing or doctor/docs hints can be follow-up work if maintainers want them after the silent-failure path is fixed.
