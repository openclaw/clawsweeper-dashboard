## Summary

Keep open: current main and the latest release still route MiniMax Anthropic-message replay through strict tool-call ID sanitization, so the central bug remains. This PR is the canonical narrow implementation candidate, but it needs a branch refresh and inspectable live MiniMax proof before merge.

## What This Changes

The PR adds a MiniMax provider replay-policy wrapper that disables tool-call ID sanitization for `anthropic-messages` while preserving strict sanitization for OpenAI-compatible MiniMax paths, plus a regression-test expectation update.

## Best Possible Solution

Land this provider-owned override or an equivalent shared replay-family option after refreshing the branch and adding inspectable MiniMax Anthropic-compatible tool-call proof that succeeds without error 2013 while OpenAI-compatible MiniMax paths remain sanitized.

## Reproduction Assessment

Yes from source inspection, but not from a live run in this review: current main and `v2026.6.10` route MiniMax `anthropic-messages` through the shared hybrid replay policy that sets strict tool-call ID sanitization, matching the linked error-2013 reports.

## Solution Assessment

Yes, likely: the MiniMax plugin owns provider-specific replay behavior and `extensions/AGENTS.md` allows provider-local `buildReplayPolicy` hooks that reuse shared family helpers. A shared factory option would be a cleaner future cleanup if more providers need the same knob, but it is not required for this narrow fix.
