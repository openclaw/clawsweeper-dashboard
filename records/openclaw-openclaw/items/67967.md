## Summary

Keep open. Current main still routes MiniMax through the generic hybrid Anthropic/OpenAI replay policy, so the Anthropic-compatible MiniMax surface still sanitizes provider-native tool-call IDs; the PR remains a narrow provider-owned fix with no concrete review findings from the diff/security pass.

## What This Changes

This PR adds a MiniMax replay-policy wrapper that disables tool-call ID sanitization for `anthropic-messages` while preserving strict sanitization for OpenAI-compatible MiniMax paths, and updates the MiniMax provider hook test expectation.

## Best Possible Solution

Land this PR or an equivalent MiniMax-owned replay-policy hook so MiniMax Anthropic-compatible requests echo provider-native tool-call IDs verbatim, while OpenAI-compatible MiniMax requests keep strict ID sanitization. A shared factory option can remain a later cleanup if more providers need the same knob.

## Reproduction Assessment

Yes. Current-main code inspection gives a high-confidence unit-level reproduction: MiniMax's registered `buildReplayPolicy` for `modelApi: "anthropic-messages"` comes from the shared hybrid helper and currently returns `sanitizeToolCallIds: true`. Live MiniMax API reproduction was not run in this read-only pass.

## Solution Assessment

Yes. The proposed provider-local wrapper uses the existing plugin replay-policy seam and scopes the behavior change to MiniMax `anthropic-messages`, while preserving the existing OpenAI-compatible policy. The retained `toolCallIdMode` value is not a current correctness blocker because downstream sanitization paths also require `sanitizeToolCallIds` to be true.
