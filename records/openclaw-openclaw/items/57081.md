## Summary

Keep open. The docs addition remains useful because current main still lacks the install-drift `browser.request` troubleshooting signature, but it should not merge as-is: maintainer discussion and current code show `browser.request` can also be absent when the browser plugin is filtered by config, so the troubleshooting needs branch-specific recovery before this PR can close the linked issue.

## What This Changes

The PR adds one troubleshooting bullet to the gateway and help docs for `GatewayClientRequestError: unknown method: browser.request` paired with CLI/Gateway version mismatch, recommending Mac app restart/update or install-state repair.

## Best Possible Solution

Refresh the PR against current main, keep the install-drift signature, add a sibling plugin-filter signature in both troubleshooting pages, and make the recovery steps branch-specific: config plus Gateway restart for plugin filtering, install repair only for CLI/Gateway drift.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the docs gap: current main can be verified by reading the target docs and the browser/plugin activation source. A live macOS mixed-install reproduction was not needed for this docs review because the discussion and code establish both missing-method branches.

## Solution Assessment

Is this the best way to solve the issue? No, not as-is. The drift wording is useful, but the narrow maintainable fix is the maintainer-requested expanded troubleshooting entry that covers both install drift and plugin filtering without steering every missing-method case to `gateway install --force`.
