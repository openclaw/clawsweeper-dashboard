## Summary

Keep open: the PR targets the right process-tree bug, but as submitted it changes the shared Unix default without preserving group cleanup for existing detached spawn callers that still omit `detached: true`; the supplied proof is also unit/build output only.

## What This Changes

The PR changes Unix process-tree termination to default to direct PID signals, adds explicit detached group-kill opt-in for one timeout caller, and updates process-kill tests.

## Best Possible Solution

Keep the safe direct-PID default only after auditing detached spawn sites and passing `detached: true` where the child is known to be a process-group leader, or choose a helper-side group-leader check with equivalent regression coverage.
