## What This Changes

The PR adds `deepseek-v4` to Microsoft Foundry’s Responses-default heuristic and adds unit assertions for DeepSeek-V4-Pro and DeepSeek-V4-Flash.

## Best Possible Solution

Only change the Foundry default after maintainer-visible provider proof confirms the exact DeepSeek V4 API surface; otherwise keep DeepSeek V4 on Chat Completions and address any separate reasoning payload bug in the shared DeepSeek wrapper path.

## Reproduction Assessment

## Solution Assessment
