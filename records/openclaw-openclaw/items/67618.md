## Summary

Keep open. Current main still emits the TUI pairing-required hint without the concrete `openclaw devices approve <requestId>` command, and PR #73208 is still open and unmerged with closing syntax for this issue.

## What This Changes

Update the TUI pairing-required error message so users see the approval command after listing pending pairing requests.

## Best Possible Solution

Review and land or close PR #73208; the end state should be a TUI disconnect hint that names both `openclaw devices list` and `openclaw devices approve <requestId>`, using the existing parser when an exact request ID is available.

## Reproduction Assessment

Yes. Calling `resolveGatewayDisconnectState("gateway closed (1008): pairing required")` deterministically returns a hint with only `openclaw devices list`, and the current test only asserts that list guidance.

## Solution Assessment
