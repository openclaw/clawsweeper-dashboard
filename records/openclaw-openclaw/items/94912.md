## Summary

Keep open: the fix direction is valid, but this PR still acknowledges every signed JSON schema mismatch with HTTP 200/no dispatch, including malformed message-shaped `Create`/`Note` payloads that current main and the latest release reject with 400. External real behavior proof is also still mock/test-only rather than a redacted patched webhook-server or live Nextcloud Talk run.

## What This Changes

The PR changes Nextcloud Talk webhook decoding to parse JSON before schema validation, return 400 for malformed JSON, return 200/no dispatch for schema mismatches, and update replay tests for those paths.

## Best Possible Solution

Keep JSON-first parsing, but classify signed payloads so true non-message Activity Streams events return 200/no dispatch while malformed JSON and malformed message-shaped `Create`/`Note` payloads still return 400.

## Reproduction Assessment

Yes. The underlying bug is source-reproducible because signed non-message Talk events fail the strict current schema and return 400, and the PR regression is source-reproducible because the updated fixture changes a malformed `Create`/`Note` payload to 200.

## Solution Assessment

No. The handler layer and JSON-first split are the right direction, but this is not the best fix until schema misses are classified so malformed supported-message payloads stay on the existing 400 path.
