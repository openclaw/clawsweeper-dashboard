## Summary

Keep open: this PR targets a real Nextcloud Talk webhook bug, but the current branch is not merge-ready because it acknowledges every signed JSON schema miss, including malformed message-shaped Note payloads, and the contributor proof is still only Vitest/replay output.

## What This Changes

The branch changes Nextcloud Talk webhook decoding to parse JSON before schema validation, return 200/no dispatch on schema mismatches, keep malformed JSON on 400, and update replay tests.

## Best Possible Solution

Keep the JSON-first parse, add a narrow classifier that returns 200 only for valid non-message Activity Streams events, and preserve 400 for malformed JSON or malformed message-shaped Note payloads.

## Reproduction Assessment

Yes. Current main is source-reproducible because signed non-Note JSON fails the strict schema and returns 400, and the PR regression is source-reproducible because a signed Create/Note payload with empty target.id is changed from 400 to 200.

## Solution Assessment

No. JSON-first parsing is the right layer, but this branch is not the best fix until schema misses are classified so malformed supported-message payloads stay on the existing 400 path.
