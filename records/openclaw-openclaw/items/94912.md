## Summary

Keep open: this is a candidate fix for a real Nextcloud Talk webhook bug, but the current branch acknowledges every parsed-JSON schema miss, including malformed signed Note-shaped payloads, with HTTP 200/no dispatch. The PR also still has only Vitest/replay evidence, not real webhook-server or live Nextcloud after-fix proof.

## What This Changes

The PR changes Nextcloud Talk webhook decoding to parse JSON before schema validation, acknowledge schema mismatches with 200/no dispatch, keep malformed JSON on 400, and update replay tests.

## Best Possible Solution

Keep JSON-first parsing, but classify signed payloads so true non-message Activity Streams events return 200/no dispatch while malformed JSON and malformed message-shaped Note payloads still return 400, with real webhook-server or live Nextcloud proof.

## Reproduction Assessment

Yes. The underlying bug is source-reproducible because signed non-message Talk events fail the strict current schema and return 400, and the PR regression is source-reproducible because the updated fixture changes a malformed Create/Note payload to 200.

## Solution Assessment

No. JSON-first parsing in the handler is the right layer, but this is not the best fix until schema misses are classified so malformed supported-message payloads stay on the existing 400 path.
