## Summary

Keep open: the PR still fixes the right Nextcloud Talk webhook boundary, but the current branch classifies every parsed JSON schema miss as ignored, including malformed Note-shaped messages that should keep the existing bad-request signal.

## What This Changes

The branch changes Nextcloud Talk webhook decoding to parse JSON before schema validation, return 400 for malformed JSON, and return 200/no dispatch for schema mismatches.

## Best Possible Solution

Keep the JSON-first split, classify true non-message Activity Streams events separately from malformed supported message payloads, and require real signed-webhook proof before merge.

## Reproduction Assessment

Yes, source-level. Current main maps signed schema misses to 400 after signature verification, and PR head maps every parsed JSON schema miss to 200/no dispatch.

## Solution Assessment

No. JSON-first parsing in the monitor is the right layer, but this branch is too broad because it treats malformed message-shaped payloads the same as true non-message events.
