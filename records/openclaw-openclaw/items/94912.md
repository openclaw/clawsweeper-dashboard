## Summary

Keep this PR open, but do not treat it as merge-ready: it is aimed at the right Nextcloud Talk webhook bug, yet the current classifier acknowledges every parsed schema mismatch, including malformed message-shaped Note payloads that current main still rejects as invalid.

## What This Changes

The PR changes Nextcloud Talk webhook decoding so malformed JSON still returns 400, while valid JSON that fails the chat-message schema is acknowledged as ignored.

## Best Possible Solution

Keep JSON-first parsing, but classify signed payloads so true non-message Activity Streams events return 200/no dispatch while malformed JSON and malformed message-shaped Note payloads still return 400, backed by real webhook-server or live Nextcloud proof.

## Reproduction Assessment

Yes. Source inspection shows current main returns 400 for strict schema misses, and the PR regression is source-reproducible because a malformed Create/Note fixture with an empty target id is changed to expect 200.

## Solution Assessment

No. JSON-first parsing in the webhook handler is the right layer, but this is not the best fix until schema misses distinguish true non-message events from malformed supported-message payloads.
