## Summary

## What This Changes

The PR changes the Nextcloud Talk webhook decoder so any strict webhook schema parse miss is acknowledged as ignored instead of returned as an invalid payload.

## Best Possible Solution

Use a two-stage webhook decoder that keeps malformed JSON and malformed message-shaped payloads on 400, returns 200/no dispatch for valid non-message Activity Streams events, and backs it with signed webhook tests plus redacted real-path proof.

## Solution Assessment

No. The handler is the right layer, but this implementation collapses malformed JSON, malformed message-shaped payloads, and valid non-message events into the same 200/no-dispatch result.
