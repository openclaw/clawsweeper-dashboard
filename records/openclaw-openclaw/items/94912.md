## Summary

Keep open: the PR targets the right Nextcloud Talk bug, but the patch acknowledges every strict-schema parse miss, including malformed signed message payloads that current main and existing tests still treat as HTTP 400. A parallel open PR exists for the same issue, but it still has unresolved classifier/proof concerns, so it is not a safe superseding target for closing this PR.

## What This Changes

The PR changes the Nextcloud Talk webhook decoder so a strict payload-schema parse failure returns the ignore path instead of writing a 400 error.

## Best Possible Solution

Keep the monitor-level fix, but split JSON parsing from strict message-schema validation so valid non-message Talk events return 200/no dispatch while malformed JSON or malformed message-shaped payloads still return 400.

## Reproduction Assessment

Yes. Source inspection shows current main returns 400 for signed schema misses, and the adjacent replay test covers a malformed message-shaped payload that should stay on that 400 path.

## Solution Assessment

No. Silently acknowledging non-message Talk events is the right direction, but this patch is too broad because it cannot distinguish valid ignored events from malformed JSON or malformed chat-message payloads.
