## Summary

Keep open: the PR targets the right Nextcloud Talk bug, but the current branch acknowledges every strict parse miss with 200/no dispatch, which loses the existing 400 signal for malformed JSON and malformed Note-shaped message payloads.

## What This Changes

The PR changes the Nextcloud Talk webhook decoder so failed strict payload parses are treated as ignored webhook events instead of invalid payloads.

## Best Possible Solution

Use a two-stage webhook decoder that keeps malformed JSON and malformed message-shaped payloads on 400, returns 200/no dispatch for valid non-message Activity Streams events, and backs it with signed webhook tests plus redacted real-path proof.

## Reproduction Assessment

Yes. Source inspection shows current main sends signed schema misses through the strict Note schema and returns 400 before any ignore branch can run; this read-only review did not run a live Nextcloud instance or local tests.

## Solution Assessment

No. The handler is the right layer, but this implementation collapses malformed JSON, malformed message-shaped payloads, and valid non-message events into the same 200/no-dispatch result.
