## Summary

Keep open: the linked Nextcloud Talk bug is real, but this PR is too broad because it acknowledges every schema parse miss, including malformed JSON and malformed message-shaped payloads, as 200/no dispatch and it does not include after-fix real behavior proof.

## What This Changes

The branch changes the Nextcloud Talk webhook decoder so payloads that fail the strict message schema are acknowledged as ignored instead of returning `400 Invalid payload format`.

## Best Possible Solution

Use a two-stage webhook decoder that keeps malformed JSON and malformed message-shaped payloads on 400, returns 200/no dispatch for valid non-message Talk ActivityStreams events, and backs it with signed webhook tests plus redacted real-path proof.

## Reproduction Assessment

Yes, source-level: current main sends signed schema misses through the strict Note schema and returns 400 before any ignore branch can run. I did not run a live Nextcloud instance in this read-only PR review.

## Solution Assessment

No. The handler is the right layer, but this implementation collapses malformed JSON, malformed message-shaped payloads, and valid non-message events into the same 200/no-dispatch result.
