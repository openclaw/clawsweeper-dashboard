## What This Changes

The PR changes Nextcloud Talk webhook decoding to JSON-parse first, keep malformed JSON on HTTP 400, and acknowledge schema-mismatch payloads as ignored events.

## Best Possible Solution

Keep JSON-first parsing, but classify schema misses so only true non-message Activity Streams events are acknowledged while malformed supported-message payloads remain HTTP 400.

## Solution Assessment

No. The handler layer and JSON-first parse are right, but the current implementation is too broad because it treats malformed supported-message payloads as non-message events.
