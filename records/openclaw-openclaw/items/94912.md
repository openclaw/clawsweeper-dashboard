## Summary

Keep open: the branch still turns every parsed JSON schema miss into an ignored webhook, so malformed Note-shaped message payloads can be acknowledged with 200/no dispatch, and the contributor proof remains Vitest/replay-only rather than real signed-webhook or live Talk output.

## What This Changes

The branch changes Nextcloud Talk webhook decoding to parse JSON before schema validation, preserve 400 for malformed JSON, return 200/no dispatch for schema mismatches, and update replay tests.

## Best Possible Solution

Keep the JSON-first split, classify signed ActivityStreams payloads so only true non-message events return 200/no dispatch, preserve 400 for malformed JSON and malformed Note-shaped messages, and require real signed-webhook proof before merge.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.9 parse signed bodies through the strict message schema and return 400 on schema miss; the PR-head fixture shows a signed Create/Note with empty target.id now expected to return 200.

## Solution Assessment

No. JSON-first parsing in the monitor is the right layer, but this branch is too broad because every schema mismatch becomes ignore instead of distinguishing true non-message events from malformed message-shaped payloads.
