## Summary

Keep open: current main and the latest release still have the reported Nextcloud Talk 400 behavior, so this PR remains a useful candidate, but the branch is not merge-ready because it acknowledges every signed schema mismatch, including malformed message-shaped payloads, and still lacks real webhook-server or live Nextcloud proof.

## What This Changes

The PR changes Nextcloud Talk webhook decoding to parse JSON before schema validation, acknowledge signed schema mismatches with 200/no dispatch, keep malformed JSON on 400, and update replay tests.

## Best Possible Solution

Keep JSON-first parsing, but classify signed payloads so true non-message Activity Streams events return 200/no dispatch while malformed JSON and malformed message-shaped Note payloads still return 400.

## Reproduction Assessment

Yes. Current main sends any signed payload that fails the strict Nextcloud Talk message schema through the 400 Invalid payload format path, and the PR-head fixture changes a malformed Note-shaped payload to expect 200.

## Solution Assessment

No. JSON-first parsing in the handler is the right layer, but this is not the best fix until schema misses are classified so malformed supported-message payloads stay on the existing 400 path.
