## Summary

Keep open: this PR targets the right Nextcloud Talk reliability bug, but the current patch still acknowledges every signed JSON schema miss with HTTP 200, including malformed message-shaped `Create`/`Note` payloads that current main rejects with 400, and the contributor proof remains test-harness output rather than real after-fix webhook behavior.

## What This Changes

The PR splits Nextcloud Talk webhook decoding into JSON parsing before schema validation, returns 400 for malformed JSON, returns 200/no dispatch for schema mismatches, and updates replay tests for those paths.

## Best Possible Solution

Keep JSON-first parsing, but classify signed payloads so true non-message Activity Streams events return 200/no dispatch while malformed JSON and malformed message-shaped `Note` payloads still return 400.

## Reproduction Assessment

Yes. The underlying bug is source-reproducible because signed non-message Talk events fail the strict current schema and return 400, and the PR regression is source-reproducible because the updated fixture changes a malformed `Create`/`Note` payload to 200.

## Solution Assessment

No. The handler layer and JSON-first split are the right direction, but this is not the best fix until schema misses are classified so malformed supported-message payloads stay on the existing 400 path.
