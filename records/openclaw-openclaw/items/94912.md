## Summary

Keep open: the latest head fixes the malformed-JSON part of the earlier review, but it still treats every signed schema mismatch as a non-message event, including malformed Note-shaped message payloads that current main and the existing replay test reject with HTTP 400. The PR is a meaningful candidate for the linked Nextcloud Talk bug, so it should be repaired rather than closed as stale or superseded.

## What This Changes

The PR changes the Nextcloud Talk webhook decoder and replay tests to parse JSON before schema validation, return 400 for unparseable JSON, and return 200/no dispatch for schema-mismatch payloads.

## Best Possible Solution

Use a two-stage decoder that returns 200/no dispatch only for valid non-message Activity Streams events, while preserving HTTP 400 for malformed JSON and malformed message-shaped Note payloads.

## Reproduction Assessment

Yes: source inspection gives a high-confidence path. Current main rejects a signed `Create`/`Note` payload with an empty `target.id` as HTTP 400, while PR head changes that same malformed message-shaped case to HTTP 200/no dispatch.

## Solution Assessment

No: the handler is the right layer and JSON-first parsing is the right direction, but the current implementation is too broad because it treats malformed supported-message payloads as non-message events. A safer fix classifies message-shaped payloads before deciding ignore versus invalid.
