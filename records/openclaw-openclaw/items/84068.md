## Label Justifications

- `P1`: The in-product update path is broken for recommended Linux systemd-supervised installs on a shipped release, even though a manual shell update workaround exists.

## Summary

Keep open: current main and the latest shipped tag still launch the managed update helper as a plain detached child, which remains vulnerable to systemd cgroup termination during the supervised gateway restart. Two open PRs already target this issue, so the issue should stay open until one fix is reviewed, landed, and verified.

## What This Changes

The issue asks OpenClaw to make Control UI/package update handoffs survive systemd-supervised gateway restarts instead of leaving the updater helper killed inside openclaw-gateway.service.

## Best Possible Solution

Land one maintained fix that launches systemd managed-service update helpers outside openclaw-gateway.service, preserves non-systemd behavior, adds focused regression coverage, and includes live Linux user-systemd upgrade proof before closing this issue.

## Reproduction Assessment

Yes. The reporter provides concrete Linux user-systemd upgrade steps and current source still follows the same plain detached-helper plus supervised-restart path; I did not run a live systemd upgrade in this read-only review.

## Solution Assessment

Yes. Moving only the systemd handoff helper into a transient user scope or equivalent supervisor-owned boundary is the narrow maintainable fix, with one linked PR selected and verified before merge.
