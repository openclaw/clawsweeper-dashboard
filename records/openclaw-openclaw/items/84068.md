## What This Changes

The issue reports that Control UI updates fail for systemd-supervised Linux installs because the managed update handoff helper is killed with the gateway unit before it runs.

## Best Possible Solution

Review, repair if needed, and land the linked systemd handoff PR or a narrow replacement that launches the helper outside openclaw-gateway.service while preserving sentinel metadata and existing non-systemd behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Linux user-systemd steps and artifacts, and current main still launches the helper inside the supervised gateway context before scheduling the restart; I did not run a live systemd upgrade in this read-only review.

## Solution Assessment

Yes. Escaping the helper into a transient user-systemd scope or service is the right boundary for the existing managed-service handoff contract, with focused tests and live systemd proof before merge.
