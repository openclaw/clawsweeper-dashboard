## Label Justifications

- `P1`: The in-product update workflow is broken for recommended Linux systemd-supervised installs, leaving users on the old version unless they use a manual shell upgrade.

## Summary

Keep open: current main and v2026.5.20 still launch the managed update helper as a plain detached child, which does not escape a systemd service cgroup, and two open closing PRs are still the active fix path.

## What This Changes

The issue reports that Control UI managed updates fail under user systemd supervision because the update handoff helper is killed before it can run.

## Best Possible Solution

Land one narrow systemd-specific handoff fix that launches the helper outside `openclaw-gateway.service`, preserves existing non-systemd behavior, and proves a live Linux user-systemd update.

## Reproduction Assessment

Yes. The reporter provided a concrete Linux user-systemd upgrade failure, and current source shows the helper is only a detached child while the generated systemd unit uses `KillMode=control-group`; I did not run a live systemd upgrade in this read-only review.

## Solution Assessment

Yes. A systemd-specific transient scope or service handoff is the narrow maintainable direction, but this issue should remain open until one linked PR merges and live systemd proof confirms the final version and sentinel state.
