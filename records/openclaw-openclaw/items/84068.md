## Label Justifications

- `P1`: The in-product update path is broken for recommended Linux systemd-supervised installs, leaving users unable to update from the Control UI without a manual shell workaround.

## Summary

Keep open: current main still launches the managed update helper as a plain detached child, while generated systemd units use control-group killing, and https://github.com/openclaw/openclaw/pull/84151 is the active closing fix path. This issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Fix systemd-supervised Control UI updates so the managed update handoff helper survives the gateway service restart and reports clearer failure state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84151, or an equivalent scoped systemd handoff, then close this issue from the merged commit and shipped release proof.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: current main detached-spawns the helper, then restarts a systemd unit configured with `KillMode=control-group`. I did not run a live systemd upgrade in this read-only review.

## Solution Assessment

Yes, the best fix shape is to launch the systemd handoff helper in a transient user scope or equivalent supervisor-owned unit before stopping the gateway. The open linked PR is the current concrete implementation path.
