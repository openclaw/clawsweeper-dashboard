- `P1`: The in-product update path is broken for recommended Linux systemd-supervised installs on the current shipped release, although a manual shell update workaround exists.

## Summary

Keep open: current `main` and the latest shipped tag still launch the managed update helper as a plain detached child before the supervised gateway exits, and the linked fixing PR is open rather than merged.

## What This Changes

The issue reports that Control UI updates are a no-op on Linux systemd-supervised installs because the managed update handoff helper is killed with the gateway unit cgroup before it can run.

## Best Possible Solution

Keep this issue open until the linked PR or a replacement launches the systemd handoff outside `openclaw-gateway.service`, preserves the existing sentinel contract, and includes redacted Linux user-systemd update proof.

## Reproduction Assessment

Yes for source-level reproduction: the reporter provided concrete Linux user-systemd steps and current source still launches the helper from inside the supervised gateway before the service restart. I did not run a live systemd upgrade in this read-only review.

## Solution Assessment

Yes, the maintainable direction is to keep the managed-service update contract while making the systemd handoff escape the gateway service cgroup and report a clear never-ran failure when the helper does not start.
