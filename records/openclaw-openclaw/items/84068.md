## Summary

Keep open: current `main` and the latest shipped tag still launch the managed update helper as a plain detached child before restarting the supervised gateway, which matches the reported Linux user-systemd failure. The closing implementation path is https://github.com/openclaw/openclaw/pull/84151, but that PR is still open, so this issue should remain open until the fix lands.

## What This Changes

The issue asks OpenClaw to make control-plane managed updates work under Linux user-systemd by launching the handoff helper outside the gateway unit cgroup and surfacing a clearer never-ran diagnostic.

## Best Possible Solution

Land a focused systemd handoff fix that runs the helper outside `openclaw-gateway.service`, preserves the existing control-plane update contract, and proves the version change, helper log, healthy restart, and final sentinel on a real Linux user-systemd install.

## Reproduction Assessment

Yes for source-level reproduction: current `main` launches a detached helper from inside the supervised gateway and then schedules a unit restart, while systemd's default control-group kill behavior can kill that helper. I did not run a live Linux user-systemd upgrade in this read-only review.

## Solution Assessment

Yes, escaping the helper into a transient user-systemd unit or service is the right fix direction for the documented control-plane update contract. The issue should stay open while the linked PR is repaired/proved and either landed or closed.
