## What This Changes

The PR changes Gateway chat display projection to drop a provenanced stream-error sentinel assistant row when a later visible assistant reply exists before the next user turn, and adds regression coverage for that display path.

## Best Possible Solution

Land the narrow display-projection fix after maintainer acceptance and exact-head gates, keeping the persisted replay sentinel unchanged while suppressing only provenanced synthetic display rows with later visible assistant content.

## Solution Assessment

Yes. Gateway display projection is the right layer because replay and session repair need to retain the persisted sentinel, and this PR fixes the projection ordering without changing persistence or provider replay.
