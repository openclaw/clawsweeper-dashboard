## Summary

Keep open: current main still has the message warmup and channel execution-mode gaps this PR targets, but the submitted patch should not merge until the new execution-mode wrappers satisfy the adapter return contract and the failing type/check gates are cleared.

## What This Changes

The PR skips eager model-context warmup for `openclaw message`, forwards Discord/Telegram message action execution modes, reuses manifest registry metadata during config defaulting, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the PR direction, but make the wrapper callbacks return a concrete channel policy value, rerun the focused tests and build/type gates, then choose this broader PR over the older warmup-only overlap if it stays green.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the issue path: `message` is not in the eager-warmup skip set, and the message runner defaults channel actions to local unless the wrapper exposes `resolveExecutionMode`.

## Solution Assessment

No as submitted. The warmup skip and wrapper forwarding are the narrow maintainable approach, but the implementation must preserve the adapter's concrete return type before it is mergeable.
