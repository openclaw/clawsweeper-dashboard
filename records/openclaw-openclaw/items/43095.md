## Summary

Keep open. Current main still returns a bare `status: "timeout"` from `sessions_send` after the target run has already been accepted, and the overlapping fixes remain open rather than merged.

## What This Changes

The issue asks for `sessions_send` to distinguish target delivery/acceptance from reply-wait timeouts so delivered messages do not look like routing failures.

## Best Possible Solution

Land one focused contract change, likely by reviewing and consolidating #58133 and #68196, that preserves accepted/pending delivery state separately from reply-wait completion and adds regression coverage for accepted-then-timeout behavior.

## Reproduction Assessment

Yes, at source level. Mock `agent` as accepted and `agent.wait` as `status: "timeout"`; current `sessions_send` then returns a bare timeout even though the target run was already accepted.

## Solution Assessment

No for current main. The narrow maintainable fix is an additive accepted/pending or delivery-state response for the accepted-then-wait-timeout path, with richer model-error diagnostics handled separately if needed.
