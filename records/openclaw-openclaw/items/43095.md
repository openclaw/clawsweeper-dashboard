## Summary

Close as implemented on current main: the accepted-then-soft-wait-timeout path now returns `status: "accepted"` with pending announce delivery instead of a bare timeout, while terminal run timeouts still return timeout with error context.

## What This Changes

The issue asks for `sessions_send` to distinguish target-session delivery/acceptance from reply-wait timeout or delivery failure.

## Best Possible Solution

Keep the current accepted/pending delivery contract for soft wait-window timeouts and leave richer target model-error diagnostics to the separate timeout-reason follow-up.

## Reproduction Assessment

No. The original accepted-run plus soft `agent.wait` timeout path no longer reproduces on current main: source and regression coverage show it returns `accepted` with pending delivery instead of a bare timeout.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it reuses the existing accepted/pending delivery result for non-terminal wait-window timeouts while preserving hard terminal timeouts as actual timeout results.
