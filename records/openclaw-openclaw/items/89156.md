## Summary

Keep open. The PR is a plausible candidate for the linked Feishu abort-race bug, but the current patch detaches recovery from the channel lifecycle and still lacks real Feishu/Lark restart proof, so it is not merge-ready.

## What This Changes

The PR changes Feishu monitor startup to schedule bot identity recovery even when the startup abort signal is already set, and adds a focused Vitest regression.

## Best Possible Solution

Keep the candidate PR open, but require a lifecycle-owned identity recovery fix and real Feishu/Lark restart proof before merge.

## Solution Assessment

No. The PR fixes skipped scheduling, but passing no abort signal creates a detached retry instead of tying recovery to the active account lifecycle.
