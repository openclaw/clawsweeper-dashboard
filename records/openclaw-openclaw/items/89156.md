## Summary

Keep open. The linked Feishu race remains real, but this PR detaches the recovery worker from the channel lifecycle and still lacks live Feishu/Lark proof, so it is not merge-ready.

## What This Changes

The PR changes Feishu monitor startup to call bot identity recovery whenever `botOpenId` is missing and to pass no abort signal if the startup signal is already aborted, plus adds a focused regression test.

## Best Possible Solution

Keep bot identity recovery lifecycle-owned: active unknown identities should retry, while stale aborted starts must not keep probing or overwrite identity state after stop/reload.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.8` skip `startBotIdentityRecovery` when `botOpenId` is missing and the lifecycle signal is already aborted. I did not establish a live Feishu/Lark tenant reproduction.

## Solution Assessment

No. The PR fixes the skipped scheduling call, but dropping the abort signal makes the retry worker unowned; a lifecycle-owned, replaceable, or guarded recovery path is safer.
