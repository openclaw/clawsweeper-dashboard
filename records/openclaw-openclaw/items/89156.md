- `merge-risk: 🚨 message-delivery`: An uncancelled or stale identity recovery can overwrite Feishu bot identity state and keep mention-gated message handling broken or inconsistent.

## Summary

Keep open. The linked Feishu race is still present on current main and in the latest release, but this PR is not merge-ready because it fixes the skipped retry by detaching identity recovery from the account lifecycle and only provides source-level/mock proof.

## What This Changes

The PR changes Feishu monitor startup to schedule bot identity recovery when `botOpenId` is missing even if the supplied abort signal is already aborted, and adds a focused regression test.

## Best Possible Solution

Keep bot identity recovery lifecycle-owned: an active unknown identity should retry, while stale aborted starts must not keep probing or overwrite identity state after stop/reload.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.6 skip `startBotIdentityRecovery` when `botOpenId` is missing and the lifecycle signal is already aborted. I did not establish a live Feishu/Lark tenant reproduction.

## Solution Assessment

No. The PR fixes the skipped call, but dropping the abort signal makes the retry worker unowned; the safer fix is a cancellable or replaceable recovery path tied to the active account lifecycle.
