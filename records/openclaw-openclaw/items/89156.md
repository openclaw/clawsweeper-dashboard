## Summary

Keep open. The PR targets a source-backed Feishu abort race, but the current patch detaches bot identity recovery from the channel lifecycle and still has only mocked/source-level proof, so it is not merge-ready.

## What This Changes

This PR changes Feishu monitor startup to schedule bot identity recovery whenever `botOpenId` is missing, passing no abort signal when the supplied lifecycle signal is already aborted, and adds a focused Vitest regression.

## Best Possible Solution

Keep bot identity recovery lifecycle-owned: retry unknown identity for the active account lifecycle without letting stale aborted starts keep probing or overwrite identity after stop/reload.

## Reproduction Assessment

Yes, at source level: current main skips `startBotIdentityRecovery` when identity is unknown and the lifecycle signal is already aborted. I did not establish a live Feishu/Lark tenant reproduction.

## Solution Assessment

No. The PR fixes the skipped call, but dropping the abort signal makes recovery unowned; the safer fix is a lifecycle-owned or guarded recovery worker plus real Feishu proof.
