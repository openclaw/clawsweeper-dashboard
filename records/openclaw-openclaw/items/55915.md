## Summary

Keep open. Current main still has direct LAN/tailnet/manual mobile connectivity, QR/setup-code pairing, Tailscale Serve/Funnel guidance, Talk-specific `gateway-relay`, and iOS APNs push relay, but not the requested cloud account login, Gateway registration/heartbeat, mobile Gateway listing, or transparent remote mobile relay transport.

## What This Changes

The issue requests cloud-account login, Gateway registration heartbeat, mobile Gateway listing, and a WebSocket relay for remote mobile Gateway access.

## Best Possible Solution

Keep the design open until maintainers choose either a coherent first-party cloud relay across CLI, config/schema, Gateway, mobile apps, backend protocol, docs, and security tests, or an explicit plugin/API seam if core should not own this transport.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, and search checks provide a high-confidence verification path that the requested commands, config, heartbeat registration, mobile Gateway listing, and transparent relay transport are absent on current main.

Unclear. The proposal is coherent, but the safest path is maintainer product and security design first because it adds cloud accounts, stored tokens, backend registration, and remote Gateway transport, and should be coordinated with https://github.com/openclaw/openclaw/issues/55914.
