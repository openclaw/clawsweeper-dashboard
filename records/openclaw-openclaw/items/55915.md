## Summary

Keep open. Current main still has direct Gateway/mobile connectivity, setup-code QR pairing, Tailscale/Serve guidance, Talk-specific gateway-relay sessions, and iOS APNs push relay, but not the requested cloud account login, cloud Gateway registration/heartbeat, mobile Gateway listing, or transparent relay transport for remote mobile access.

## What This Changes

Add cloud relay support with OAuth login, Gateway registration/heartbeat, and a transparent WebSocket relay for mobile access outside LAN or Tailscale reachability.

## Best Possible Solution

Keep this open for maintainer design, ending in either a coherent first-party cloud relay across CLI, config/schema, Gateway, mobile apps, backend protocol, docs, and security tests, or an explicit plugin/API seam if core should not own the relay.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main provide a high-confidence verification path showing the requested commands, config, heartbeat registration, mobile Gateway listing, and transparent relay transport are absent.

## Solution Assessment

Unclear. The proposal is coherent, but the safest path is maintainer product and security design first because it adds cloud accounts, stored tokens, backend registration, and remote Gateway transport, and should be coordinated with https://github.com/openclaw/openclaw/issues/55914.
