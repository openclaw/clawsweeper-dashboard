## Summary

Keep open. Current main still has QR/setup-code pairing, direct LAN/Tailscale/manual mobile connectivity, Talk-specific gateway relay sessions, and iOS APNs push relay, but not the requested cloud account login, Gateway registration/heartbeat, mobile Gateway listing, or transparent remote mobile relay transport.

## What This Changes

Add opt-in cloud relay support with OAuth login, Gateway registration and heartbeat, mobile Gateway listing, and transparent remote mobile WebSocket relay access.

## Best Possible Solution

Keep this open until maintainers choose a coherent first-party cloud relay design across CLI, config/schema, Gateway, mobile apps, backend protocol, docs, and security tests, or define an explicit plugin/API seam if core should not own the transport.
