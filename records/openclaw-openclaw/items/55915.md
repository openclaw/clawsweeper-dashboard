## Summary

Keep open. Current main still supports direct LAN/Tailscale/manual mobile Gateway connectivity, QR/setup-code pairing, Talk-specific Gateway relay sessions, and iOS APNs push relay, but not the requested cloud account login, Gateway registration/heartbeat, mobile Gateway listing, or transparent remote mobile relay transport.

## What This Changes

Request to add opt-in cloud relay support with `openclaw login`, cloud auth/config, Gateway registration/heartbeat, and relayed mobile WebSocket access.

## Best Possible Solution

Keep this open until maintainers choose a coherent first-party cloud relay design across CLI, config/schema, Gateway, mobile apps, backend protocol, docs, and security tests, or define an explicit plugin/API seam if core should not own the transport.

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, and search checks give a high-confidence verification path that the requested commands, config, heartbeat registration, Gateway listing, and transparent relay transport are absent on current main.

Unclear. The proposal is coherent, but the safest path is maintainer product and security design first because it adds cloud accounts, stored tokens, backend registration, and remote Gateway transport coordinated with https://github.com/openclaw/openclaw/issues/55914.
