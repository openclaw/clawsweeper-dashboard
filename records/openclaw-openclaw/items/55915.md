## Summary

Keep open. Current main still has direct mobile-to-Gateway connection paths, setup codes, Tailscale/Serve guidance, Talk gateway-relay sessions, and iOS APNs push relay support, but it does not implement the requested cloud account login, cloud config block, Gateway registration/heartbeat, mobile cloud Gateway listing, or transparent Socket.IO/WebSocket relay for mobile connectivity.

## What This Changes

Request cloud relay support with `openclaw login`, OAuth token storage, Gateway registration/heartbeat, and a relayed mobile WebSocket transport gated by gateway cloud config.

## Best Possible Solution

Keep this open for maintainer design review, ending in either a first-party cloud relay design across CLI, Gateway config/schema, mobile apps, backend protocol, docs, and security tests, or a documented plugin/API seam that can own an optional relay without bypassing device approval.

Not applicable as a bug reproduction; this is a feature request. Current-main source and docs inspection provides a high-confidence verification path showing the requested commands, config, heartbeat registration, mobile gateway listing, and transparent relay transport are absent.
