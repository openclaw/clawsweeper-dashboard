## Summary

Keep open. Current main still provides direct LAN/tailnet/manual mobile connectivity, QR/setup-code pairing, Tailscale Serve/Funnel guidance, Talk-specific `gateway-relay`, and iOS APNs push relay, but not the requested cloud account login, Gateway registration/heartbeat, mobile Gateway listing, or transparent remote mobile relay transport.

## What This Changes

Request adds opt-in cloud relay support with `openclaw login`, OAuth token storage, Gateway registration/heartbeat, mobile Gateway listing, and relayed WebSocket transport behind new cloud config.

## Best Possible Solution

Keep the design open until maintainers choose either a coherent first-party cloud relay across CLI, config/schema, Gateway, mobile apps, backend protocol, docs, and security tests, or an explicit plugin/API seam if core should not own this transport.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main provide a high-confidence verification path showing the requested commands, config, heartbeat registration, mobile Gateway listing, and transparent relay transport are absent.

## Solution Assessment

Unclear. The proposal is coherent, but the safest path is maintainer product and security design first because it adds cloud accounts, stored tokens, backend registration, and remote Gateway transport, and should be coordinated with https://github.com/openclaw/openclaw/issues/55914.
