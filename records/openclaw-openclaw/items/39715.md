## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve session state during the drain window:** `src/process/supervisor/supervisor.ts:234-242`
  - body: By finalizing the supervisor record and deleting it from `active` before the new `setImmediate`, there is a full event-loop turn where `process kill`/`remove` sees the supervisor record as exited while the `ProcessSession` is still running. That path falls back to `killProcessTree()` and `markExited(..., "failed")`, so a request in this window can move a normally completed session to finished as SIGKILL before `managedRun.wait()` snapshots the drained stdout. Keep this draining state out of the fallback kill path or make kill/remove treat it as already completing.
  - confidence: 0.87
