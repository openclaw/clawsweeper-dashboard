## Summary

Keep this PR open. Current main still resolves the supervisor child wait path and snapshots stdout/stderr without an event-loop drain, while this PR is a narrow implementation of the linked background-output loss fix and the earlier cancel/finalize races raised in review appear addressed in the latest diff.

## What This Changes

This PR updates the process supervisor wait lifecycle to guard post-settle cancellation, finalize active run state before a one-turn stdout/stderr drain, and adds focused background exec stdout/stderr regression tests.

## Best Possible Solution

Land or adapt the narrow supervisor-level drain so stdout/stderr callbacks queued with process close are processed before RunExit and finished-session aggregate snapshots, while preserving the settled/cancel/finalize safeguards already added. Keep the regression coverage focused on backgrounded stdout, stderr, and larger exit-time output.

## Reproduction Assessment

Yes. The linked report gives a concrete sandbox exec path where output is lost after yieldMs backgrounding unless redirected with 2>&1, and current main still has the immediate wait-to-snapshot sequence that makes that failure plausible.

## Solution Assessment

Yes, with maintainer validation. The latest PR moves the drain to the supervisor after settlement instead of the child adapter, which is the narrowest place to protect the stdout/stderr snapshot while avoiding the cancellation races called out in earlier review.
