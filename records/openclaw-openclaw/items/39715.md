## Summary

Keep open, but not merge-ready: this is a focused bug-fix PR, yet the current head still creates a one-turn split between supervisor state and background-session state, and it lacks real sandbox proof that current main still drops stdout after the shipped close/drain behavior.

## What This Changes

The PR adds a post-wait `setImmediate` drain in supervisor completion, hardens cancellation after settle, and adds background exec stdout regression tests.

## Best Possible Solution

Prove the remaining stdout-loss path with a redacted live sandbox run, then drain without splitting supervisor and session registry state, either by yielding before process-visible finalization or by adding a completing state that process tooling handles explicitly.

## Reproduction Assessment

Yes for the PR-introduced session-state gap by source inspection: the PR finalizes supervisor state before `managedRun.wait()` resolves and before `runExecProcess()` marks the session exited. No live Bun/Docker sandbox reproduction of the original stdout-loss report is present.

## Solution Assessment

No: the drain idea is narrow, but this implementation publishes terminal supervisor state before the final session snapshot is ready. A safer fix keeps completion state coherent until the final `RunExit` has been handed to the exec runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep the run active until the final session is published:** `src/process/supervisor/supervisor.ts:229-242`
  - body: The new drain yield finalizes `getRecord(runId)` and removes the run from `active` before `managedRun.wait()` resolves. During that full event-loop turn, `process remove`/`kill` can still see a running background session, but `cancelManagedSession()` returns false because the supervisor record is already `exited`; `remove` can then fallback-kill/delete the session, and the wait continuation can publish it again. Keep an explicit completing state or defer process-visible finalization until the final `RunExit` is ready.
