## Summary

Keep this PR open. Current main still snapshots child stdout/stderr and finished background-session aggregates without an event-loop drain turn, while the PR remains a focused implementation candidate for the stdout/stderr data-loss half of the linked sandbox exec report. I did not find a definite blocking bug in the latest diff, but the process lifecycle timing change still deserves maintainer review before merge.

## What This Changes

This PR updates the process supervisor to guard cancellation/finalization around a post-wait `setImmediate` drain before stdout/stderr snapshots, and adds focused background exec stdout/stderr regression tests.

## Best Possible Solution

Land or adapt the narrow supervisor-level drain so stdout/stderr callbacks queued with process close are processed before `RunExit` and finished-session aggregate snapshots, while preserving the settled/cancel/finalize safeguards already discussed on the PR. Keep the regression tests focused on backgrounded stdout, stderr, and larger exit-time output.
