## What This Changes

This PR adds a supervisor-level event-loop drain before building `RunExit`, guards cancellation/finalization across that drain, adds background stdout/stderr regression coverage, and records the fix in the changelog.

## Best Possible Solution

Merge the narrow supervisor drain/hardening after exact-head CI completes green, preserving the regression tests and changelog entry.

Yes, based on the latest head. Moving the drain to the supervisor after settlement and before `RunExit` construction is the narrowest maintainable fix, and the latest fake-timer update keeps `setImmediate` real from the start instead of dropping the queued drain.
