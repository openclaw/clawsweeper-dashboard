## Summary

Keep open: current main still skips OpenClaw memory flush/preflight compaction for CLI runtimes, `claude-cli` is explicitly backend-owned for compaction, and the usage footer has only a raw context percentage/meter rather than backend-aware or staged status. Related issues cover adjacent reset, status, and context-budget problems, but none fully owns this focused native CLI handoff plus status-clarity request.

## What This Changes

The issue requests a supported native CLI-backend pre-compaction handoff/memory-flush hook and clearer backend-aware context status labels.

## Best Possible Solution

Maintain this as a focused maintainer design item: define the native CLI pre-transition hook contract and then update the shared usage/status surface to distinguish OpenClaw-managed compaction from backend-owned compaction.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level for the core gap: current main skips memoryFlush/preflight compaction for CLI runtimes and renders context usage without backend/stage metadata; I did not live-reproduce the exact scary `100%` channel display on a claude-cli deployment.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the native CLI handoff contract; the narrow maintainable direction is to design a CLI-aware pre-transition hook rather than remove the existing CLI gates wholesale.
