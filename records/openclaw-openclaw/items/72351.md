## Summary

Keep open. The runtime change is the right narrow fix for the TUI zero-history path, but it makes `--history-limit 0` a supported public mode without the required TUI docs and `CHANGELOG.md` Fixes entry.

## What This Changes

The PR adds a TUI `historyLimit === 0` branch that skips backend history loading, clears local chat state, renders the session marker, and adds a regression test.

## Best Possible Solution

Merge the TUI-side zero guard after adding a public docs note that `--history-limit 0` skips initial history loading and a `CHANGELOG.md` Fixes entry; keep the gateway `chat.history` RPC contract limited to positive history requests.

## Reproduction Assessment

Yes. Current main parses `--history-limit 0`, forwards it as `limit: 0`, and the `chat.history` schema rejects limits below 1; the PR test covers the desired no-backend-call path.

## Solution Assessment

Yes for the runtime fix. Handling zero in the TUI before calling `chat.history` is narrower than relaxing the gateway protocol for all clients, but the PR still needs docs and changelog coverage before merge.
