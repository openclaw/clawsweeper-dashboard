## Summary

Keep open. The PR fixes a real TUI zero-history bug with a narrow runtime guard and regression coverage, but it still needs public docs and changelog coverage for the newly supported `--history-limit 0` behavior before merge.

## What This Changes

The PR adds a TUI `historyLimit === 0` branch that skips backend history loading and adds a unit test for the no-history path.

## Best Possible Solution

Merge the TUI-side zero-history guard after adding a public docs note and a `CHANGELOG.md` Fixes entry, while keeping the gateway `chat.history` contract positive-only.

Yes. Current main parses `--history-limit 0`, forwards `limit: 0` into TUI history loading, and the gateway `chat.history` schema rejects limits below 1; the PR test covers the desired no-backend-call path.

## Solution Assessment

Yes for the runtime fix. Handling zero in the TUI is narrower than relaxing the gateway protocol for all clients, but the PR is incomplete until docs and changelog are aligned.
