## Summary

Keep this PR open, but not merge-ready: the branch is a focused attempt at zero-history TUI behavior, yet it only handles the lower-level session-action path while current main and the latest release still reject `--history-limit 0` at the CLI, and the stale branch misses current session-state and package-renaming changes.

## What This Changes

The PR adds a TUI `loadHistory` early return for `historyLimit === 0` and a regression test that expects no gateway history call.

## Best Possible Solution

Either support zero-history mode end to end at the CLI and TUI session boundary while preserving current session side effects, or keep the positive-integer public contract and avoid merging a lower-level-only guard.

## Reproduction Assessment
