## Summary

Keep open. Current main still forwards `--history-limit 0` into the TUI history-loading path, while the PR’s runtime change is narrow and matches the gateway contract; the remaining blocker is the missing docs and changelog for a user-facing TUI behavior change.

## What This Changes

The PR adds a TUI `historyLimit === 0` branch that clears local history state, skips backend history loading, refreshes session info, renders the session marker, and adds regression coverage.

## Best Possible Solution

Merge the TUI-side zero guard after adding a public docs note that `--history-limit 0` skips initial history loading and a `CHANGELOG.md` Fixes entry; keep the gateway `chat.history` RPC contract limited to positive history requests.

## Reproduction Assessment

Yes. Current main parses `--history-limit 0`, forwards it as `limit: 0`, and the `chat.history` schema rejects limits below 1; the PR test covers the desired no-backend-call path.

## Solution Assessment

Yes for the runtime fix. Handling zero in the TUI before calling `chat.history` is narrower than relaxing the gateway protocol for all clients, but the PR should add docs and changelog before merge.
