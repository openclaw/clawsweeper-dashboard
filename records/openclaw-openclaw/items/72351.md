## What This Changes

## Best Possible Solution

Either intentionally support zero-history mode end to end with CLI parsing, docs/tests, and current TUI session side effects, or keep the positive-integer contract and close or replace this stale branch.

## Reproduction Assessment

The PR body's public CLI command is not reproduced end to end because current `registerTuiCli` rejects zero before `runTui`.

## Solution Assessment

No, not yet. The lower-level guard is plausible, but it is not the best current-base fix unless maintainers accept zero as a public no-history mode, update CLI validation, and preserve current session-state side effects.
