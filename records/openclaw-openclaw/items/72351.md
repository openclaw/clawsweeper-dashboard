## Summary

## What This Changes

The PR adds a TUI `historyLimit === 0` branch that clears local chat state, skips gateway history loading, and adds a regression test.

## Best Possible Solution

Land a rebased TUI-side zero-history guard that skips `chat.history` while preserving session remembering, uses current `@earendil-works/pi-tui` test types, keeps regression coverage, and includes redacted after-fix terminal proof.

## Reproduction Assessment

Yes. Source inspection shows the CLI preserves `0`, the TUI forwards it as `limit: 0`, and the gateway schema rejects `chat.history` limits below `1`; I did not run the TUI because this review is read-only.

## Solution Assessment

No, not as written. Handling zero before the gateway call is the right boundary, but the patch needs to preserve current session remembering and use the current pi-tui package name before it is the narrow maintainable fix.
