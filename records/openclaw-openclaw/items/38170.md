## Summary

Keep open: current main still lacks the full Telegram file-id placeholder and download action behavior, but the PR is stale against the plugin migration and has blocking security/compatibility issues.

## What This Changes

The PR embeds Telegram file IDs in media placeholders, adds a Telegram download-file message action, and moves inbound file-id lookup into a shared helper.

## Best Possible Solution

Replace the stale branch with a Telegram plugin-surface implementation that defines a parser-safe file_id history contract and routes downloads through the existing guarded media downloader, with focused plugin tests and docs/changelog if maintainers accept the feature.

## Reproduction Assessment

Yes. Source inspection is enough: current main does not advertise or dispatch Telegram download-file, and its placeholder helpers still emit bare media tags except for a limited sidecar fallback.

## Solution Assessment

No. The direction may be useful, but this PR is not the best implementation because it targets removed legacy files and adds direct credentialed downloads outside the guarded Telegram media path.

  - body: The added discovery/dispatch lives in the legacy channel-action shim, but current main owns Telegram message actions in `extensions/telegram/src/channel-actions.ts` and this legacy path is absent. Rebase this onto the plugin surface so `download-file` is actually advertised and handled.

  - confidence: 0.91
- **[P2] Preserve attribute-bearing audio detection:** `src/telegram/bot-message-context.ts:470`
  - body: Switching placeholders to forms like `<media:audio file_id="...">` also requires updating the shared inbound-audio parser, which currently recognizes only the bare audio placeholder when media type facts are absent. Without that, TTS/inbound-audio-conditioned dispatch can regress.
  - confidence: 0.87
- **[P3] Avoid deterministic temporary-path overwrites:** `src/agents/tools/telegram-actions.ts:500`
  - body: The output path uses only the first 16 characters of `fileId`, so distinct files with the same prefix can overwrite each other. Use the media store or a collision-resistant sanitized name.
  - confidence: 0.79
