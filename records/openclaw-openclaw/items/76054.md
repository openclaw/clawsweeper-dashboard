## Summary

Keep open. Current main still has only the defaults-level resolver, and the latest PR is useful but not merge-ready because its new per-agent lookup uses raw ID equality and the user-facing docs/changelog are still not aligned.

## What This Changes

The PR adds optional `agents.list[].contextInjection`, threads agent IDs into embedded run and compaction context-injection resolution, updates generated config metadata, and adds resolver tests.

## Best Possible Solution

Land the feature after reusing normalized per-agent ID lookup semantics and aligning generated metadata, public docs, tests, and the changelog.

## Reproduction Assessment

Yes. Configure `{ id: "Strict Agent", contextInjection: "always" }` with defaults set to `"continuation-skip"`; the runtime normalized `strict-agent` ID will miss the PR's raw equality lookup and fall back to defaults.

## Solution Assessment

No, not yet. The config/schema direction is the narrow maintainable fix, but the resolver should use the existing normalized per-agent lookup behavior and the public docs/changelog need to match the new user-facing option.
