Keep open. Current main still supports only `agents.defaults.contextInjection`, and this PR is the right narrow feature direction, but the latest supplied diff still bypasses normalized agent-id lookup and misses the public agent config docs update before merge.

## What This Changes

The PR adds `agents.list[].contextInjection` as a per-agent override with fallback to `agents.defaults.contextInjection` and then the code default.

## Best Possible Solution

Land the per-agent override after routing lookup through the existing normalized agent-config path and documenting the per-agent key plus precedence in the public agent configuration docs.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction: current main cannot honor any per-agent override, and the PR can miss an override when the configured id needs the same normalization used by existing per-agent fields.

## Solution Assessment

No, not as written. The schema/runtime direction is narrow and maintainable, but the implementation should reuse normalized per-agent resolution and ship matching public docs before merge.
