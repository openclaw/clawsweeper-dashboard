## Summary

Current shipped code still has a concrete false-positive path: durable delivery may return `suppressed` with no outbound result, but heartbeat treats that as success and marks due commitments sent. The issue should remain open for a narrow result-handling fix rather than the broader active-session polling and metrics proposal.

## What This Changes

Fix inferred commitments that transition to sent even though the targeted chat receives no outbound message.

## Best Possible Solution

Handle the durable-send result as a closed union: mark commitments sent only for `status: "sent"`, retain existing explicit model-dismissal behavior, and keep suppressed deliveries eligible for bounded retry or expiry with focused regression coverage.

## Reproduction Assessment

Yes at source level. Mocking the durable sender to return `status: "suppressed"` on current main should show the commitment incorrectly becoming sent with no outbound result; a live Telegram reproduction has not been run.

## Solution Assessment

No for the full proposed redesign. The best fix is the narrow exhaustive result handling in the existing heartbeat delivery owner; active-session idle polling and delivery-health metrics are separate product enhancements.
