## Summary

Keep open: current main no longer supports the original clipping root cause, but the keyboard auto-scroll bug still appears source-reproducible and this PR contains unique remaining work. It is not ready to merge as-is because the branch is conflicting, the wrapper/layout portion is obsolete unless reverified, and the external PR still needs clear real WebChat proof of the after-fix keyboard-scroll behavior.

## What This Changes

The PR wraps the WebChat input, moves slash-menu rendering outside the input container, and calls `scrollIntoView` after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land or replace this with a rebased narrow patch that scrolls the active slash-menu option within its own listbox, keeps the current no-clipping layout, and adds focused coverage plus real browser proof.

## Reproduction Assessment

Yes, source-level: current main has a scrollable slash menu, and ArrowUp/ArrowDown update the active index without scrolling the active option into view. The original clipping symptom is not supported by current main because `.agent-chat__input` no longer has `overflow: hidden`.

## Solution Assessment

No, not as-is: the auto-scroll direction is useful, but the lookup should be scoped to the current composer/listbox and the obsolete wrapper layout changes should be dropped unless clipping is reverified.
