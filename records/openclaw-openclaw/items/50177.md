## Summary

Keep open. Current main has already made the overflow-clipping portion stale, but it still lacks keyboard auto-scroll for the active slash-menu item, so this PR contains useful remaining WebChat work and should not be closed as implemented.

## What This Changes

The PR wraps the WebChat composer input so the slash menu renders outside the input container, adjusts desktop/mobile CSS selectors, and calls `scrollIntoView` after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land a rebased minimal WebChat fix that keeps the active slash-menu option visible in both command and argument modes, scopes the lookup from the current composer/root, adds focused test coverage, and only keeps wrapper/layout changes if current main still needs them.

## Reproduction Assessment

Yes. The PR body gives a browser path, and current source review confirms the remaining failure path: ArrowUp/ArrowDown changes the active slash-menu option without focusing it or scrolling it into view.

## Solution Assessment

No, not as-is. The auto-scroll direction is maintainable and matches an existing command-palette pattern, but the final patch should scope the lookup to the active composer, include the required changelog entry, and avoid stale wrapper churn unless the clipping still reproduces.
