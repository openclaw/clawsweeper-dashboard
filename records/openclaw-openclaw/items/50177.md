## Summary

Keep this PR open: current main no longer reproduces the overflow-clipping part, but the keyboard auto-scroll gap is still present by source inspection and the PR contains a useful, narrow direction that needs a rebased/scoped repair.

## What This Changes

The PR wraps the WebChat composer so the slash menu renders outside the input container and adds `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased current-main patch that adds scoped active-option auto-scroll with focused WebChat coverage, preserving the existing composer layout unless overflow clipping still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows ArrowUp/ArrowDown update the active slash-menu index while the menu is scrollable, but there is no code path that scrolls the active option into view; the original clipping sub-claim is not reproduced because `.agent-chat__input` lacks `overflow: hidden`.

## Solution Assessment

No, not as-is. The auto-scroll behavior is the right fix direction, but it should be scoped to the current composer/listbox and rebased onto the current WebChat structure instead of carrying stale wrapper/layout churn.
