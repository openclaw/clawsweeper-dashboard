## Summary

Keep open: current main no longer reproduces the overflow-clipping half, but the keyboard auto-scroll gap is still present and the PR contains useful behavior that is not implemented on main. The branch needs a focused repair before merge because the active-item lookup is globally scoped and the wrapper/layout part appears stale against current WebChat CSS.

## What This Changes

The PR wraps the WebChat composer slash menu outside the input container and adds `scrollIntoView` after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land a rebased current-main patch that adds scoped active-option auto-scroll with focused WebChat coverage, while keeping the existing composer layout unless overflow clipping still reproduces.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue: source inspection shows ArrowUp/ArrowDown update the active slash-menu index while the menu is scrollable, with no `scrollIntoView` path. The original overflow-clipping sub-claim is not reproducible from current source because `.agent-chat__input` does not set `overflow: hidden`.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the active-item lookup should be scoped to the current composer/listbox and the stale wrapper/layout churn should be dropped unless clipping is reverified.
