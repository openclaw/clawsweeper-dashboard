## Summary

Keep open. The overflow-clipping half is already fixed on current main, but the keyboard auto-scroll part is still a real, source-reproducible WebChat bug and the PR contains useful work that needs a focused repair/rebase.

## What This Changes

The PR wraps the WebChat composer input so the slash menu renders outside the input container and calls scrollIntoView after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land a rebased patch that keeps the current composer layout, scopes active-option scrolling to the current listbox/composer, and adds focused WebChat coverage for keyboard auto-scroll.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue by source inspection: current main has a scrollable slash menu and ArrowUp/ArrowDown only update the active index. The original overflow-clipping claim no longer reproduces from current source because .agent-chat__input no longer sets overflow: hidden.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the DOM lookup should be scoped to the current composer/listbox and the stale wrapper layout changes should be dropped unless clipping is reverified on main.
