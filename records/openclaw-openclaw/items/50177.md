## Summary

Keep open: the overflow-clipping half is already fixed on current main and shipped, but current main still lacks keyboard auto-scroll for the scrollable slash menu, so this PR contains useful remaining work that should be rebased and narrowed.

## What This Changes

The PR adds an input wrapper for WebChat slash-menu positioning and calls `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased WebChat patch that adds scoped active-option auto-scroll with focused coverage, while keeping the current composer layout unless clipping reproduces again.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue by source inspection: the menu is scrollable and ArrowUp/ArrowDown update the active option without scrolling it into view. The original overflow-clipping sub-issue is not reproducible from current source because `.agent-chat__input` no longer clips overflow.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the lookup should be scoped to the current composer or listbox and the stale wrapper layout changes should be dropped unless clipping is reverified on main.
