## Summary

Keep open. Current main already removed the original overflow-clipping condition, but the keyboard auto-scroll behavior remains missing, and this PR still points at a real WebChat usability bug that is better repaired on top of current main than closed.

## What This Changes

The PR wraps the WebChat composer so the slash menu renders outside the input container and calls `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased or replacement WebChat patch that adds scoped active-option auto-scroll with focused coverage, while preserving the current composer structure and only changing wrapper/layout CSS if clipping still reproduces.

## Reproduction Assessment

Yes, by source inspection on current main: ArrowUp/ArrowDown update `vs.slashMenuIndex` and rerender, but no code scrolls the newly active option inside the `.slash-menu` scroll area. The original overflow-clipping sub-bug is not reproduced on current main because `.agent-chat__input` no longer has `overflow: hidden`.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the helper should be scoped to the current composer/root and the patch should be rebased or replaced on current main with focused tests.
