## Summary

Keep open: the clipping half is already fixed on current main, but the keyboard auto-scroll behavior is still missing. The PR has a useful narrow fix, but it should be rebased into a smaller patch that scopes the active-option lookup and avoids the obsolete wrapper/layout churn.

## What This Changes

The PR moves WebChat slash-menu rendering into an input wrapper, retargets related CSS margins, and adds `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased WebChat patch that adds scoped active-option auto-scroll with focused coverage, while keeping the current composer layout unless clipping is reproduced again.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue by source inspection: the menu is scrollable and ArrowUp/ArrowDown update the active index without any scroll path. The original overflow-clipping sub-issue is not reproducible from current source because `.agent-chat__input` no longer sets `overflow: hidden`.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the active item lookup should be scoped to the current composer/listbox and the stale wrapper layout changes should be dropped unless clipping is reverified on main.
