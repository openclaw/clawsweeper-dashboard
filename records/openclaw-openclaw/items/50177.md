## Summary

Keep this PR open: the original overflow-clipping part is already fixed on current main, but current WebChat still has a source-proven keyboard-scroll gap for the scrollable slash menu, and the PR contains useful but not yet ideal work for that behavior.

## What This Changes

This PR moves the WebChat slash menu outside the input container and calls `scrollIntoView` after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land a rebased patch that adds scoped active-option auto-scroll with focused WebChat coverage, while keeping the current composer layout unless clipping is reverified on main.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue by source inspection: the menu is scrollable and ArrowUp/ArrowDown update the active index without any scroll path. The original overflow-clipping sub-claim is not reproducible from current source because `.agent-chat__input` no longer sets `overflow: hidden`.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the active-item lookup should be scoped to the current composer or listbox, and the stale wrapper/layout churn should be dropped unless clipping is reverified.
