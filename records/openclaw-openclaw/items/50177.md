## Summary

Keep open: the PR still targets a real remaining WebChat keyboard-scroll gap, but it is not ready to merge because the proof is insufficient, the branch is dirty/conflicting, and the patch should be narrowed/scoped before landing.

## What This Changes

The PR changes WebChat composer layout and slash-menu keyboard handling so the menu renders outside the input container and the active menu item scrolls into view during ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land or replace this with a rebased, narrow WebChat fix that scrolls the active slash-menu option within the current composer/listbox, adds focused regression coverage, and avoids wrapper layout churn unless clipping is reverified.

## Reproduction Assessment

Yes, source-level: current main has a scrollable slash menu, and ArrowUp/ArrowDown update the active index without scrolling the active option into view. The original overflow-clipping symptom is not supported by current main because `.agent-chat__input` no longer has `overflow: hidden`.

## Solution Assessment

No, not as-is: the auto-scroll direction is useful, but the lookup should be scoped to the current composer/listbox and the obsolete wrapper layout changes should be dropped unless clipping is reverified.
