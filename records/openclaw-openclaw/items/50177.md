## Summary

Keep open: current main already shipped the overflow-clipping part, but the keyboard auto-scroll fix is still absent on main and this external PR still needs stronger real behavior proof plus a rebase/narrowing pass before merge.

## What This Changes

The PR wraps the WebChat composer input, moves the slash menu outside it, and calls `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased, narrow WebChat fix that scopes active-option auto-scroll to the current slash menu/listbox and drops stale wrapper layout changes unless clipping reproduces again on main.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection gives a clear reproduction path for the keyboard-scroll sub-issue: the menu is scrollable and ArrowUp/ArrowDown move the active option without scrolling it into view. The original overflow-clipping sub-issue is no longer reproducible from current source because `.agent-chat__input` no longer clips overflow.

## Solution Assessment

Is this the best way to solve the issue? No, not as-is: the auto-scroll direction is useful, but the lookup should be scoped to the current composer/listbox and the obsolete wrapper layout changes should be dropped unless clipping is reverified on main.
