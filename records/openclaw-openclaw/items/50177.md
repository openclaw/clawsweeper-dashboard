## Summary

Keep open: current main still supports the keyboard auto-scroll problem in source, but the clipping root cause described by the PR is no longer present, so this needs a rebased/narrower patch plus clearer real behavior proof before merge.

## What This Changes

The PR wraps the WebChat input to move the slash menu outside the input container and calls `scrollIntoView` after ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land or replace this with a rebased narrow patch that scrolls the active slash-menu option within the current composer/listbox, preserves the current no-clipping layout, and adds focused coverage plus real WebChat proof.

## Reproduction Assessment

Yes, source-level: current main has a scrollable slash menu and ArrowUp/ArrowDown handlers only update the active index without scrolling the active option into view. The original clipping symptom is not supported by current main because `.agent-chat__input` no longer has the claimed `overflow: hidden` rule.

## Solution Assessment

No, not as-is: auto-scroll is the right narrow behavior, but the lookup should be scoped to the current composer/listbox and the stale wrapper layout changes should be dropped unless clipping is reverified.
