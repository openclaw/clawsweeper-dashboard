## Summary

Keep open: current main already shipped the overflow-clipping part, but it still lacks keyboard auto-scroll for the scrollable slash menu, so this PR contains useful remaining behavior. It is not merge-ready as-is because the branch is conflicting/stale, the wrapper layout change is now obsolete, the active-item lookup is still document-scoped, and the external real-behavior proof is insufficient.

## What This Changes

The branch changes WebChat CSS and chat view rendering to wrap the composer, render the slash menu outside the composer container, and call `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land or replace this with a rebased, narrow WebChat fix that scrolls the active slash-menu option within the current composer/listbox, adds focused regression coverage, and leaves the already-shipped clipping fix alone unless it reproduces again on main.

## Reproduction Assessment

Yes for the remaining keyboard-scroll sub-issue from source: `.slash-menu` is scrollable and ArrowUp/ArrowDown rerender the active item without scrolling it into view. The original overflow-clipping sub-issue is not reproducible from current source because `.agent-chat__input` no longer hides overflow.

## Solution Assessment

No, not as-is: the auto-scroll direction is useful, but the lookup should be scoped to the current composer/listbox and the obsolete wrapper layout changes should be dropped unless clipping is reverified on main.
