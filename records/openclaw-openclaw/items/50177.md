## Summary

Keep open. Current main has made the overflow-clipping part mostly stale, but it still lacks the keyboard auto-scroll behavior this PR targets, so the useful remaining work should be rebased and narrowed rather than closed.

## What This Changes

The PR wraps the WebChat composer input so the slash menu renders outside the input container, adjusts related CSS/mobile selectors, and calls `scrollIntoView` for the active slash-menu item after ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased minimal WebChat fix that keeps the active slash-menu item visible in both command and argument modes, scoped to the current composer/root, and retain wrapper/layout changes only if overflow clipping still reproduces on current main.

## Reproduction Assessment

Yes. The PR body gives a browser path, and source review confirms the remaining issue: ArrowUp/ArrowDown changes the active slash-menu option without focusing or scrolling that option into view.

## Solution Assessment

No, not as-is. The auto-scroll approach matches an existing UI pattern, but the wrapper portion looks stale against current main and the final patch should be rebased, scoped to the current chat instance, and covered by a focused UI test or explicit browser proof.
