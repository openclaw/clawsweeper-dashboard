## Summary

Keep this PR open: the keyboard auto-scroll behavior is still a plausible WebChat bug, but the clipping half is already gone on current main and the branch needs real behavior proof plus a rebase before merge.

## What This Changes

The PR adds a WebChat input wrapper to unclip the slash menu and calls `scrollIntoView` after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land or replace this with a rebased narrow patch that scrolls the active slash-menu option within the current composer/listbox, preserves the current no-clipping layout, and adds focused coverage plus real browser proof.

## Reproduction Assessment

Yes, source-level: current main has a scrollable slash menu and ArrowUp/ArrowDown handlers only update the active index without scrolling the active option into view. The original clipping symptom is not supported by current main because `.agent-chat__input` no longer has the claimed `overflow: hidden` rule.

## Solution Assessment

No, not as-is: active-option auto-scroll is the right behavior, but the lookup should be scoped to the current composer/listbox and the stale wrapper layout changes should be dropped unless clipping is freshly reverified.
