## Summary

Keep open. Current main already shipped the overflow-clipping fix, but the keyboard auto-scroll gap remains source-reproducible, so this PR still contains useful WebChat behavior that should be landed as a narrower scoped patch.

## What This Changes

This PR wraps the WebChat composer input to move the slash menu outside the input container and calls `scrollIntoView` after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land a small WebChat patch that adds scoped active-option auto-scroll plus regression coverage, while leaving the shipped no-clipping composer layout unless a new clipping repro appears.

## Reproduction Assessment

Yes, source-reproducible for the keyboard-scroll sub-issue: current main has a scrollable slash menu and ArrowUp/ArrowDown update the active index without scrolling the active option into view. The original clipping sub-issue is not reproducible from current source because `.agent-chat__input` no longer clips overflow.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the DOM lookup should be scoped to the current composer or listbox, and the stale wrapper layout change should be dropped unless clipping reproduces again.

## Review Findings

Overall correctness: patch is correct
