## Summary

Keep open. Current main no longer shows the original overflow-clipping root cause, but it still lacks keyboard auto-scroll for the scrollable WebChat slash menu; this PR points in the right direction but is stale/conflicting and still scopes the active-item lookup from `document`.

## What This Changes

This PR wraps the WebChat composer so the slash menu renders outside the input container and calls `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased fix or replacement patch on current main that adds scoped active-option auto-scroll with focused coverage, preserving the existing composer structure unless overflow clipping still reproduces.

## Reproduction Assessment

Yes, source-reproducible on current main: ArrowUp/ArrowDown change `vs.slashMenuIndex` and rerender the menu, but there is no `scrollIntoView` path for the active option inside the `.slash-menu` scroll area. The original clipping sub-bug is not reproduced by current source because `.agent-chat__input` no longer has `overflow: hidden`.

## Solution Assessment

No, not as-is. The auto-scroll behavior is the right direction, but it should be rebased onto current main, scoped to the current composer/listbox, and covered by focused tests instead of carrying stale wrapper/layout churn unless clipping still reproduces.
