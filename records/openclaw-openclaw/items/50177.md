## Summary

Keep open: the keyboard auto-scroll problem is still source-reproducible on current main and this PR has useful work, but it should not merge as-is because the clipping half is already shipped fixed, the branch is stale against current WebChat composer code, and the attached proof does not visibly show the slash menu or keyboard scrolling.

## What This Changes

The PR wraps the WebChat input, moves the slash menu outside that input, and calls `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land a rebased, narrow WebChat fix that adds scoped active-option auto-scroll with focused coverage, while keeping the current composer layout unless clipping reproduces again on main.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue by source inspection: the menu is scrollable and ArrowUp/ArrowDown update the active option without scrolling it into view. The original overflow-clipping sub-issue is not reproducible from current source because `.agent-chat__input` no longer clips overflow.

## Solution Assessment

No, not as-is. The auto-scroll direction is right, but the DOM lookup should be scoped to the current composer or listbox and the obsolete wrapper layout changes should be dropped unless clipping is reverified on main.
