## Summary

Keep open: the overflow-clipping part is already fixed on current main, but the keyboard auto-scroll part remains a plausible WebChat bug and this PR is not old enough for a mostly-implemented cleanup close. The branch still needs contributor real behavior proof and a maintainer decision on rebasing/narrowing the fix.

## What This Changes

The PR adds a WebChat input wrapper, moves slash-menu rendering outside `.agent-chat__input`, and calls `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land or replace this with a rebased narrow WebChat patch that scrolls the active slash-menu option within its own listbox, keeps the already-shipped no-clipping layout, and adds focused coverage plus real browser proof.

## Reproduction Assessment

Yes, source-level: current main has a scrollable slash menu, and ArrowUp/ArrowDown update the active index without scrolling the active option into view. The original clipping symptom is not supported by current main because `.agent-chat__input` no longer has `overflow: hidden`.

## Solution Assessment

No, not as-is: the auto-scroll direction is useful, but the lookup should be scoped to the current composer/listbox and the obsolete wrapper layout changes should be dropped unless clipping is reverified.
