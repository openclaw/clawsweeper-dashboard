## Summary

Keep open: current main already fixed the overflow-clipping half, but the keyboard auto-scroll gap is still visible from source and the PR contains unique useful work that is not old enough for stale-branch cleanup.

## What This Changes

The PR wraps the WebChat composer, renders the slash menu outside the input container, and calls `scrollIntoView` after slash-menu ArrowUp/ArrowDown navigation.

## Best Possible Solution

Land or replace this with a rebased, narrow WebChat fix that scrolls the active slash-menu option within the current composer/listbox, adds focused regression coverage, and leaves the shipped clipping fix alone unless it reproduces again.

## Reproduction Assessment

Yes, source-level: current main has a scrollable `.slash-menu`, and ArrowUp/ArrowDown update the active index without scrolling the active option into view. The original overflow-clipping sub-issue is not reproducible from current source because `.agent-chat__input` no longer hides overflow.

## Solution Assessment

No, not as-is: the auto-scroll direction is useful, but the lookup should be scoped to the current composer/listbox and the obsolete wrapper layout changes should be dropped unless clipping is reverified on main.
