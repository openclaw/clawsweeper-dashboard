## Summary

Keep open: current main already lacks the overflow clipping root cause, but the remaining keyboard-scroll behavior is still absent. The PR’s useful direction is valid, though the branch should be rebased to keep only the needed auto-scroll fix with scoped lookup and coverage.

## What This Changes

The PR moves the WebChat slash menu outside the input container and adds `scrollIntoView` after ArrowUp/ArrowDown slash-menu navigation.

## Best Possible Solution

Land a rebased patch that adds scoped active-option auto-scroll with focused WebChat coverage, while keeping the current composer layout unless clipping is reverified on main.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue by source inspection: the menu is scrollable and ArrowUp/ArrowDown update the active index without any scroll path. The original overflow-clipping sub-claim is not reproducible from current source because `.agent-chat__input` no longer sets `overflow: hidden`.

## Solution Assessment

No, not as-is. The auto-scroll fix is the right direction, but the active item lookup should be scoped to the current composer/listbox and the stale wrapper layout changes should be dropped unless clipping is reproduced again.
