## Summary

Keep open: current main has already shipped the original clipping fix, but the keyboard auto-scroll problem still exists by source inspection. This PR adds useful scroll behavior, but the branch is stale and the new active-item lookup is globally scoped, so it needs a focused repair before merge.

## What This Changes

The PR changes WebChat slash-menu layout and keyboard navigation so the menu renders outside the input container and ArrowUp/ArrowDown scroll the active item into view.

## Best Possible Solution

Land a rebased WebChat patch that adds scoped active-option auto-scroll with focused coverage, while keeping current composer layout unless clipping reproduces again on current main.

## Reproduction Assessment

Yes for the keyboard-scroll sub-issue by source inspection: the menu is scrollable and ArrowUp/ArrowDown update the active option without scrolling it into view. The original overflow-clipping sub-issue is not reproducible from current source because `.agent-chat__input` no longer clips overflow.

## Solution Assessment

No, not as-is. The auto-scroll behavior is the right direction, but the active option lookup should be scoped to the current composer or listbox, and the stale wrapper layout churn should be dropped unless clipping is reverified on current main.
