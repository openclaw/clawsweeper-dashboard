## Label Justifications

- `P2`: This is a normal-priority backup reliability fix with limited blast radius but real impact for scheduled backups on live installs.
- `merge-risk: 🚨 availability`: The proposed Pack error listener can swallow ENOENT/ENOTDIR without completing node-tar's failed job, causing backup creation to stall.
- `merge-risk: 🚨 compatibility`: The proposed archive-time skip reporting can publish backups whose embedded manifest does not match the payloads actually written.

## Summary

Keep open. The ENOENT deletion race is still a real backup reliability problem, but this PR is merge-conflicting against current main, lacks real behavior proof, and its Pack error handling can hang instead of completing the backup.

## What This Changes

The PR replaces backup archive creation with a tar.Pack helper that tries to swallow ENOENT/ENOTDIR during archiving and report skipped paths in the backup result.

## Best Possible Solution

Rebase onto current main and implement per-entry ENOENT handling that completes tar work and keeps manifest/verify metadata coherent, then prove the live deleted-transcript race.

## Reproduction Assessment

Yes, source-reproducible. Current main still passes source roots to node-tar, node-tar lstat happens before filter evaluation, and active session or queue files can disappear while tar walks the tree; I did not run a live race in this read-only pass.

## Solution Assessment

No. The PR targets the right failure mode, but the current Pack listener can strand tar work and the skipped-file metadata is discovered after the archive manifest is written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:155-161`
  - body: When Pack emits ENOENT/ENOTDIR during lstat, this listener records the path and returns. node-tar leaves the failed job in its queue and only closes once the queue is empty, so backup create can hang instead of completing with a skipped entry.
  - confidence: 0.9
- **[P2] Keep manifest metadata aligned with skipped paths:** `src/infra/backup-create.ts:413-420`
  - body: The archive manifest is written before archive-time skips are known, then result.skipped is amended afterward. If a planned payload is omitted during packing, the embedded manifest can still declare it present and backup verify can reject the archive.
  - confidence: 0.84
- **[P3] Model Pack skips without failing the stream:** `src/commands/backup.test-support.ts:41-47`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. A mocked ENOENT skip would still reject through stream.on("error"), so tests cannot cover the successful skip behavior this helper relies on.
  - confidence: 0.82
