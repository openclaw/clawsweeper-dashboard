## Summary

Keep open. The PR targets a real backup race, but the proposed patch still has blocking correctness issues and lacks after-fix real behavior proof; the merged volatile-file work on current main is adjacent but does not fully replace this deleted-before-lstat fix.

## What This Changes

The PR replaces backup archive creation with a `tar.Pack` helper that records missing-path skips and extends the backup tar test mock.

## Best Possible Solution

Implement one coherent backup archiver fix that treats expected per-entry ENOENT/ENOTDIR as non-fatal without hanging, keeps manifest and verification metadata accurate, and proves the real backup path.

## Reproduction Assessment

Yes, source-reproducible: current main feeds state roots into `tar.c`, node-tar v7.5.15 emits lstat errors before `filter`, and session maintenance can rename transcript files during cleanup. I did not run a live race.

## Solution Assessment

No. The PR aims at the right failure mode, but swallowing Pack errors can leave queued jobs unresolved and skipped paths are added after the manifest is already written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:155-161`
  - body: When node-tar emits ENOENT/ENOTDIR during `lstat`, this listener only records the path and returns. Pack leaves the failed job in its queue and only ends when the queue is empty, so the output stream may never close and `backup create` can hang instead of completing with a skipped path.
  - confidence: 0.88
- **[P2] Keep manifest metadata aligned with skipped paths:** `src/infra/backup-create.ts:415-421`
  - body: `manifest.json` is written before archive-time skips are known, then skipped paths are appended only to `result.skipped` after packing. If an asset or payload is omitted during packing, the embedded manifest still declares it present, leaving verify/restore metadata inconsistent with the created archive.
  - confidence: 0.8
- **[P3] Model Pack errors without failing the stream:** `src/commands/backup.test-support.ts:41-45`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. An intended ENOENT skip would still reject through `stream.on("error")`, so regression tests cannot exercise the behavior the production helper depends on.
  - confidence: 0.82
