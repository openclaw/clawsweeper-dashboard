## Summary

Keep open. The PR targets a real backup ENOENT race, but it is not merge-ready: current main only has adjacent volatile-file/EOF hardening from https://github.com/openclaw/openclaw/pull/72251, this patch can hang on skipped Pack errors and leaves archive metadata inconsistent, and no after-fix real backup proof is present.

## What This Changes

The PR replaces `tar.c` backup archive creation with a `tar.Pack` helper that records ENOENT/ENOTDIR paths as skipped and extends the backup tar mock.

## Best Possible Solution

Use current main's volatile filtering as the baseline, then add a correct per-entry ENOENT/ENOTDIR handling path that cannot hang, keeps manifest and verification semantics accurate, includes focused regression coverage, and has real backup proof.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Session cleanup can rename a transcript while backup is walking state, and tar v7.5.15 emits lstat ENOENT before OpenClaw's filter can run.

## Solution Assessment

No. The PR targets the right failure mode, but returning from the Pack error listener does not complete the queued Pack job, and appending skipped paths after manifest creation leaves metadata and verification semantics inconsistent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:154-160`
  - body: When node-tar emits ENOENT/ENOTDIR from `lstat`, returning from this listener only suppresses the error; it does not remove the queued Pack job or advance Pack's accounting. The output stream may never close, so `backup create` can hang instead of completing with a skipped path.
  - confidence: 0.88
- **[P2] Rebuild the manifest after archive-time skips:** `src/infra/backup-create.ts:414-421`
  - body: The manifest is written before archive-time skips are discovered, then `result.skipped` is updated afterward. That can leave the archive manifest declaring payloads that were omitted, breaking verification/restore metadata for backups that otherwise appear successful.
  - confidence: 0.8
- **[P3] Separate Pack errors from stream errors in the mock:** `src/commands/backup.test-support.ts:41-45`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. That makes the intended ENOENT skip reject through `stream.on("error")`, so tests cannot exercise the production behavior this PR depends on.
  - confidence: 0.82
