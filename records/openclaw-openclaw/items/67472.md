## Summary

Keep open. The PR targets a real, source-reproducible `backup create` ENOENT race, and the merged related work at https://github.com/openclaw/openclaw/pull/72251 usefully reduced live-backup volatility, but it is complementary and does not fully cover deleted-before-lstat ENOENT; this branch still has blocking implementation issues and no after-fix real behavior proof.

## What This Changes

The PR replaces `tar.c` backup creation with a `tar.Pack`-based archiver that records ENOENT/ENOTDIR skips and updates the backup tar test mock.

## Best Possible Solution

Rework or replace the archiver fix so ENOENT/ENOTDIR during tar walks are handled per entry without hanging, manifest/verify metadata stay coherent, and real proof covers a session transcript removed during archive creation.

## Reproduction Assessment

Yes, source-reproducible: current main still archives planned roots through `tar.c`, node-tar stats before invoking the filter, and session maintenance can rename transcript files during cleanup. I did not run a live race.

## Solution Assessment

No. The PR targets the right failure mode, but swallowing Pack errors can leave jobs unresolved and skipped paths are added only after the manifest has already been written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:155-161`
  - body: When `Pack` emits ENOENT/ENOTDIR during `lstat`, this listener records the path and returns. node-tar keeps the failed job queued and only ends once the queue is empty, so `backup create` can hang or repeatedly retry the same missing path instead of completing with a skip.
  - confidence: 0.88
- **[P2] Keep manifest metadata aligned with skipped paths:** `src/infra/backup-create.ts:413-420`
  - body: `manifest.json` is written before archive-time skips are known, then `result.skipped` is amended only after packing. If a planned asset or payload is omitted during packing, the embedded manifest can still declare it present, leaving verify/restore metadata inconsistent with the published archive.
  - confidence: 0.8
- **[P3] Model Pack skips without failing the stream:** `src/commands/backup.test-support.ts:41-45`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. A mocked ENOENT skip would still reject through `stream.on("error")`, so regression tests cannot exercise the behavior the helper relies on.
  - confidence: 0.82
