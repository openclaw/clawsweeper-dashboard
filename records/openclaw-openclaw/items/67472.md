## Summary

Keep open. The PR targets a real `openclaw backup create` race from https://github.com/openclaw/openclaw/issues/67417, but the proposed Pack error handling can hang on skipped entries, the archive manifest can become inconsistent, the branch lacks real behavior proof, and the later merged live-backup work at https://github.com/openclaw/openclaw/pull/72251 is adjacent rather than a complete replacement for the deleted-before-lstat ENOENT path.

## What This Changes

The branch replaces backup archive creation with a `tar.Pack` helper that records ENOENT/ENOTDIR paths as skipped and extends the backup test tar mock.

## Best Possible Solution

Use current main's volatile filtering as the baseline, then handle expected per-entry ENOENT without hanging, keep manifest and verification semantics accurate, add focused regression coverage, and require real backup proof.

## Reproduction Assessment

Yes, from source inspection. The linked issue gives a concrete race, session cleanup can rename transcript files, and tar 7.5.15 emits lstat ENOENT before OpenClaw's current filter can run.

## Solution Assessment

No. The PR targets the right failure mode, but swallowing Pack errors does not complete the queued Pack job and appending skips after manifest creation leaves metadata and verification semantics inconsistent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:154-160`
  - body: When node-tar emits an ENOENT/ENOTDIR error for a filesystem entry, returning from this listener does not remove the queued Pack job or decrement Pack's job accounting. The output stream may never close, so `backup create` can hang instead of succeeding with a skipped path.
  - confidence: 0.88
- **[P2] Keep skipped payloads out of the manifest:** `src/infra/backup-create.ts:414-421`
  - body: The manifest is written before archive-time skips are discovered, then `result.skipped` is updated afterward. That can leave the embedded manifest declaring payloads that were omitted, which breaks `--verify` semantics and makes restore metadata misleading.
  - confidence: 0.8
- **[P3] Separate Pack errors from stream errors in the mock:** `src/commands/backup.test-support.ts:41-45`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. That makes the intended ENOENT skip reject through `stream.on("error")`, so tests cannot exercise the production behavior this PR depends on.
  - confidence: 0.82
