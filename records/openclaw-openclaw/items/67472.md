## Summary

Keep open: the linked backup race remains source-reproducible on current main, and this PR still has merge-blocking correctness issues plus missing external-contributor real behavior proof.

## What This Changes

The PR replaces `backup create` archive writing with a `tar.Pack` helper that skips ENOENT/ENOTDIR paths and extends the backup tar mock.

## Best Possible Solution

Implement one coherent backup archiver fix that treats expected per-entry ENOENT/ENOTDIR as non-fatal without hanging, keeps manifest and verify metadata accurate, and proves the real backup path.

## Reproduction Assessment

Yes, source-reproducible: session maintenance can rename a transcript while `backup create` is walking state, and tar@7.5.15 emits lstat ENOENT before the current filter can skip the entry. I did not run a live race in this read-only pass.

## Solution Assessment

No. The PR targets the right failure mode, but swallowing Pack errors can leave queued jobs stuck and skipped paths are added after the manifest is already written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:155-161`
  - body: When node-tar emits ENOENT/ENOTDIR during `lstat`, returning from this listener only suppresses the error. The Pack job remains queued, so the output stream may never close and `backup create` can hang instead of completing with a skipped path.
  - confidence: 0.88
- **[P2] Keep manifest metadata consistent with skipped paths:** `src/infra/backup-create.ts:415-421`
  - body: Skipped paths are appended after `manifest.json` has already been serialized and included in the archive. If a planned asset or payload is omitted at archive time, verification/restore metadata does not reflect what was actually archived.
  - confidence: 0.8
- **[P3] Model Pack errors without also failing the stream:** `src/commands/backup.test-support.ts:41-45`
  - body: The mock calls Pack error listeners and then emits the same error on the output stream. That makes an intended ENOENT skip reject through `stream.on("error")`, so tests cannot exercise the production behavior this PR depends on.
  - confidence: 0.82
