## Summary

Keep open. The PR targets a real source-reproducible backup ENOENT race, but it is currently conflicting with main, lacks after-fix real behavior proof, and introduces blocking archiver correctness risks around Pack error completion and manifest consistency.

## What This Changes

The PR replaces backup archive creation with a tar.Pack helper that catches ENOENT/ENOTDIR during packing, records skipped paths in the backup summary, and extends the backup test tar mock.

## Best Possible Solution

Rebase or replace the branch with an archive fix that treats expected per-entry ENOENT/ENOTDIR as non-fatal while completing tar work, keeping manifest and verify metadata coherent, preserving current main’s volatile-file filtering, and proving the deleted-transcript race in a real backup run.

## Reproduction Assessment

Yes, source-reproducible. Current main passes source roots to node-tar, node-tar lstat happens before the filter, and session cleanup can rename transcript files while backup walks the state tree; I did not run a live race in this read-only pass.

## Solution Assessment

No. The PR targets the right failure mode, but the current Pack error listener can leave tar work unresolved and the skipped-file result is recorded after the archive manifest is written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:155-161`
  - body: When Pack emits ENOENT/ENOTDIR during lstat, this listener records the path and returns. node-tar leaves the failed job in its queue and only closes once the queue is empty and jobs are zero, so backup create can hang instead of completing with a skipped entry.
  - confidence: 0.9
- **[P2] Keep manifest metadata aligned with skipped paths:** `src/infra/backup-create.ts:413-420`
  - body: The archive manifest is written before archive-time skips are known, then result.skipped is amended afterward. If a planned payload is omitted during packing, the embedded manifest can still declare it present or omit the skip record, so verify/restore tooling can see an inconsistent archive.
  - confidence: 0.84
- **[P3] Model Pack skips without failing the stream:** `src/commands/backup.test-support.ts:41-47`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. A mocked ENOENT skip would still reject through stream.on("error"), so tests cannot cover the successful skip behavior this helper relies on.
  - confidence: 0.82
