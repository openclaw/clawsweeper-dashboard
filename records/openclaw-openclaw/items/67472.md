## Summary

Keep open. ## What This Changes

The PR replaces `tar.c` backup archive creation with a `tar.Pack` helper that swallows ENOENT/ENOTDIR during packing and records skipped paths in the backup result, plus a matching tar mock update.

## Best Possible Solution

Rebase or replace the branch with a narrow archive fix that treats expected per-entry ENOENT/ENOTDIR as non-fatal without stalling tar work, keeps manifest and verify metadata coherent, preserves current volatile filtering, and proves the deleted-transcript race in a real backup run.

## Reproduction Assessment

Yes, source-reproducible. Current main passes source roots to node-tar, node-tar emits lstat ENOENT before the filter can run, and session cleanup can rename transcript files while tar walks the tree; I did not run a live race in this read-only pass.

## Solution Assessment

No. The PR targets the right failure mode, but the current Pack error listener can leave tar work unresolved and the skipped-file result is recorded after the archive manifest is written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:155-161`
  - body: When Pack emits ENOENT/ENOTDIR during lstat, this listener records the path and returns. node-tar leaves the failed job in its queue and only closes once the queue is empty and jobs are zero, so backup create can hang instead of completing with a skipped entry.
  - confidence: 0.9
- **[P2] Keep manifest metadata aligned with skipped paths:** `src/infra/backup-create.ts:415-421`
  - body: The archive manifest is written before archive-time skips are known, then `result.skipped` is amended afterward. If a planned payload is omitted during packing, the embedded manifest can still declare it present or omit the skip record, so verify/restore tooling can see an inconsistent archive.
  - confidence: 0.84
- **[P3] Model Pack skips without failing the stream:** `src/commands/backup.test-support.ts:41-46`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. A mocked ENOENT skip would still reject through `stream.on("error")`, so tests cannot cover the successful skip behavior this helper relies on.
  - confidence: 0.82
