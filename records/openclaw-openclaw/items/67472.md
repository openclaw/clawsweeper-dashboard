## Summary

Keep open. The PR targets a valid backup race, but the proposed Pack error handling can leave node-tar jobs unresolved, the archive manifest can become inconsistent with skipped paths, and the external PR still lacks after-fix real backup proof; merged live-backup hardening in https://github.com/openclaw/openclaw/pull/72251 is related but does not fully obsolete the deleted-before-lstat ENOENT case.

## What This Changes

The PR replaces backup archive creation with a tar.Pack-based helper that records ENOENT/ENOTDIR skips and adds a Pack mock for backup command tests.

## Best Possible Solution

Rework or replace the branch with a rebased archiver fix that treats expected per-entry ENOENT/ENOTDIR as non-fatal while completing tar work, keeps manifest and verify metadata coherent, and proves the real deleted-transcript race.

## Reproduction Assessment

Yes, source-reproducible. Current main still passes source roots to node-tar, node-tar lstat happens before the filter, and session maintenance can rename transcript files while backup walks the tree; I did not run a live race in this read-only pass.

## Solution Assessment

No. The PR targets the right failure mode, but the Pack error listener can leave work unresolved and the skipped-file result is recorded after the archive manifest is written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:155-161`
  - body: When Pack emits ENOENT/ENOTDIR during lstat, this listener records the path and returns. node-tar leaves the failed job in its queue and only closes once the queue is empty, so backup create can hang instead of completing with a skipped entry.
  - confidence: 0.9
- **[P2] Keep manifest metadata aligned with skipped paths:** `src/infra/backup-create.ts:413-420`
  - body: The archive manifest is written before archive-time skips are known, then result.skipped is amended afterward. If a planned payload is omitted during packing, the embedded manifest can still declare it present and backup verify can reject the archive.
  - confidence: 0.84
- **[P3] Model Pack skips without failing the stream:** `src/commands/backup.test-support.ts:41-47`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. A mocked ENOENT skip would still reject through stream.on("error"), so tests cannot cover the successful skip behavior this helper relies on.
  - confidence: 0.82
