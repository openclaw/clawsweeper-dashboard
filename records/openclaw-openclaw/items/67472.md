## Summary

Keep open. The PR addresses a real backup race, but current main has not implemented the deleted-before-lstat ENOENT fix and this branch still has blocking correctness issues plus missing real behavior proof.

## What This Changes

The PR replaces backup `tar.c` archive creation with a `tar.Pack` helper that records ENOENT/ENOTDIR paths as skipped and extends the backup tar test mock.

## Best Possible Solution

Keep current main's volatile filtering as the baseline, then add a correct per-entry ENOENT/ENOTDIR handling path that cannot hang, preserves manifest and verification semantics, and has focused regression plus real backup proof.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Session cleanup can rename a transcript while `backup create` walks state, and tar 7.5.15 emits lstat ENOENT before OpenClaw's current filter can run.

## Solution Assessment

No. The PR targets the right failure mode, but returning from the Pack error listener does not complete the queued Pack job, and appending skipped paths after manifest creation leaves metadata and verification semantics inconsistent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Complete skipped Pack jobs before resolving:** `src/infra/backup-create.ts:154-160`
  - body: When node-tar emits ENOENT/ENOTDIR from `lstat`, returning from this listener only suppresses the error; it does not remove or complete the queued Pack job. The output stream may never close, so `backup create` can hang instead of completing with a skipped path.
  - confidence: 0.88
- **[P2] Rebuild the manifest after archive-time skips:** `src/infra/backup-create.ts:412-420`
  - body: The manifest is written before archive-time skips are discovered, then `result.skipped` is updated afterward. If a planned asset or skipped path is omitted from the archive, the embedded manifest can still declare it present, breaking verify/restore metadata for an otherwise published backup.
  - confidence: 0.8
- **[P3] Separate Pack errors from stream errors in the mock:** `src/commands/backup.test-support.ts:41-47`
  - body: The mock invokes Pack error listeners and then emits the same error on the output stream. That makes the intended ENOENT skip reject through `stream.on("error")`, so tests cannot exercise the production behavior this PR depends on.
  - confidence: 0.82
