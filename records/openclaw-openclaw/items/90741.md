## Summary

Keep this PR open: the reconciled models-config cache work is useful and well-proven in several seams, but the current head introduces a blocking stale persisted-catalog cache path because the source fingerprint no longer reflects root models.json or generated plugin-catalog drift before cached catalog rows are read.

## What This Changes

This PR replaces mtime-based models-config readiness checks with content/outcome fingerprints, adds provider/model-scoped short-circuit caching, reads canonical SQLite auth state for cache keys, and guards persisted catalog cache use when auth source fingerprints are uncacheable.

## Best Possible Solution

Preserve the PR's canonical SQLite auth and scoped-cache improvements, but make persisted catalog cache keys include post-ensure root models.json and generated plugin-catalog outcome freshness before cached rows can be reused.
