## Best Possible Solution

## Reproduction Assessment

Yes, at source level: corrupt or unreadable auth state makes buildModelsJsonSourceFingerprint return the same `uncacheable:${agentDir}` value, and two cache consumers can read persisted catalog entries with that key before discovery. I did not run tests because this review is read-only.

## Solution Assessment

No: the central ensureOpenClawModelsJson path is a strong fail-closed design, but the source-fingerprint shim is incomplete because it lets persisted catalog caches bypass the same uncacheable-auth contract. The narrow fix is to expose a cache-bypass or per-call noncacheable key to both consumers.
