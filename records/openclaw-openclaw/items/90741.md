## Summary

Keep this PR open, but it is not merge-ready: the cache rewrite drops generated plugin catalog sidecar validation from the warm hit path, and the PR still lacks real behavior proof for the runtime cache change.

## What This Changes

The branch rewrites `ensureOpenClawModelsJson` caching to use SQLite auth-profile content fingerprints, root `models.json` content outcomes, and provider-scoped `targetProvider`/`targetModel` short-circuiting.

## Best Possible Solution

Keep the SQLite auth fingerprint and provider-scoped cache direction, but restore validation for every generated artifact the function owns before warm cache returns and use a no-create/read-only auth-store fingerprint path when no DB is already needed.

## Reproduction Assessment

Yes from source inspection: warm the cache with a generated plugin catalog sidecar, mutate or delete that sidecar without changing root `models.json`, then call `ensureOpenClawModelsJson` again; the PR's hit path checks only root `models.json`, while current main included plugin catalog sidecar mtimes in the cache key.

## Solution Assessment

No; the overall performance/cache direction is plausible, but the current implementation is too narrow because it validates only root `models.json` and turns an auth-store fingerprint read into a database-creating operation.
