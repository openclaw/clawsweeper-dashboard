## Summary

Keep open. Current main mitigates transcript loss by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: the idle composer toolbar still shows a plus-icon New session button beside send/export and wires it directly to session creation without confirmation, relocation, clearer icon, or an approved hide setting.

## What This Changes

Request to reduce accidental Control UI New session activation by adding confirmation, changing or moving the plus icon, or adding a hide setting.

## Best Possible Solution

Keep the fresh-session creation behavior, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and aligns with the header/session-switcher direction if PR #61423 is accepted.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and its click path calls `createChatSession` directly; the unit test asserts one click invokes `onNewSession`.

## Solution Assessment

Unclear. A confirmation dialog would be a narrow guard, but the issue intentionally offers several UX alternatives, and the open header-modal PR means maintainers should choose the final placement/icon/config direction before implementation.
