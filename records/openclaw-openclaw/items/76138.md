## Summary

Keep open. Current main mitigates the older transcript-loss problem by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: the idle composer toolbar still shows a plus-icon New session button beside the send controls, and one click still creates a session without confirmation, relocation, clearer iconography, or a hide setting.

## What This Changes

The issue requests confirmation, clearer or relocated iconography, or a setting for the Control UI New Session button in the composer toolbar.

## Best Possible Solution

Keep the fresh-session behavior, then choose one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and coordinates with the open header/session-modal work.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and its click path directly calls createChatSession; the focused unit test asserts that one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog would be a narrow guard, but choosing between confirmation, icon/placement changes, and a hide setting should be reconciled with #61423 before implementation.
