## Summary

Keep open. Current main mitigates transcript loss by creating and switching to a fresh dashboard session, but the idle composer still shows a plus-icon New session button beside the send controls and wires it directly to session creation without confirmation, relocation, clearer icon, or a hide setting.

## What This Changes

Request to reduce accidental Control UI New session activation by adding confirmation, changing or relocating the send-adjacent plus button, or adding a hide setting.

## Best Possible Solution

Keep the fresh-session creation path, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and aligns with the header/session-switcher direction in #61423.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and the click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be reconciled with #61423.
