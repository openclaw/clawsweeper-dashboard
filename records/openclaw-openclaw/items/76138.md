## Summary

Keep open. Current main fixed the most destructive reset behavior by creating and switching to a preserved dashboard session, but the reported Control UI affordance remains: the composer toolbar still shows a plus-labelled New session button beside Send, without a confirmation, clearer icon/placement, or hide setting; related PR #61423 is open and does not clearly settle the existing toolbar button.

## What This Changes

Requests a safer Control UI New Session affordance by adding confirmation, changing or relocating the plus icon, or adding a hide setting.

## Best Possible Solution

Keep the create-and-switch session behavior, then make one maintainer-approved Control UI change that removes the attachment-like send-adjacent plus affordance or gates it without creating duplicate session-entry confusion.

## Solution Assessment
