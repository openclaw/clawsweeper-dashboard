## Summary

Keep open. Current main mitigates transcript loss by creating a fresh dashboard session, but the reported Control UI affordance remains: the composer toolbar still shows a plus-icon New session button beside send/export controls and one click creates/switches sessions without confirmation, relocation, clearer icon, or a hide setting. Open #61423 overlaps with the header/modal direction and should be reviewed before choosing a concrete fix.

## What This Changes

Request asks Control UI to prevent accidental New Session clicks by adding confirmation, changing or relocating the plus icon, or adding a hide setting.

## Best Possible Solution

Keep the fresh dashboard-session implementation, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and aligns with any accepted header/session-switcher work.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and its click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be reconciled with #61423.
