## Summary

Keep open. Current main preserves the old transcript by creating a separate dashboard session, but the reported Control UI affordance remains: the plus-icon New session button still sits in the composer toolbar and a single click directly creates a new session with no confirmation, alternate icon, relocation, or hide setting.

## What This Changes

The issue asks for the Control UI New session toolbar button to be made less misclick-prone through confirmation, clearer icon or placement, or a hide setting.

## Best Possible Solution

Keep the fresh-session preservation behavior, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and aligns with any accepted header/session-switcher work.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and its click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, icon or placement changes, and a hide setting should be reconciled with the open header/session-creation work in #61423.
