## Summary

Keep open. Current main mitigates the transcript-loss part by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: the idle composer toolbar still shows a plus-icon New session button beside the send controls and wires it directly to session creation without confirmation, relocation, clearer icon, or a hide setting.

## What This Changes

Requests a Control UI UX change that makes the New Session toolbar action less accidental, less attachment-like, or configurable.

## Best Possible Solution

Keep the fresh-session creation implementation, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and aligns with any accepted header/session-switcher work.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and its click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be reconciled with #61423.
