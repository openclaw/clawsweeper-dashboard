## Summary

Keep open. Current main preserves the previous transcript by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: a plus-labelled New session button still sits in the composer toolbar next to the send controls, with no confirmation, clearer icon/placement, or hide setting.

## What This Changes

Request to prevent accidental Control UI New Session clicks by adding confirmation, changing or relocating the plus button, or adding a hide setting.

## Best Possible Solution

Keep fresh-session creation discoverable while removing the send-adjacent plus ambiguity in one maintainer-approved Control UI design, aligned with #61423 if that header modal continues.

## Reproduction Assessment

Yes. Source inspection on current main shows the composer toolbar renders a plus-labelled New session button next to send controls and clicking it calls createChatSession directly; the focused run-controls test asserts that immediate click path.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, a clearer icon or placement, and a hide setting should be reconciled with the open header-session modal work in #61423.
