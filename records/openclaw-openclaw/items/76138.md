## Summary

Keep open. Current main fixed the worst data-loss behavior by creating and switching to a fresh dashboard session, but the remaining UX request is still unresolved: the composer toolbar still shows a plus-labelled New session button beside Send with no confirmation, clearer icon/placement, or hide setting, and open PR #61423 only partially overlaps that direction.

## What This Changes

Request to make the Control UI send-adjacent New Session + button less accidental by adding confirmation, changing or relocating the icon, or adding a hide setting.

## Best Possible Solution

Keep the non-destructive create-and-switch behavior, then choose one consistent Control UI affordance that makes New Session discoverable without looking like an attachment or sitting as a risky send-adjacent action.

## Reproduction Assessment

Yes for the observable current behavior. Source inspection on current main shows the plus-labelled New session toolbar button still renders beside the send controls and calls createChatSession without a confirmation or config gate.

## Solution Assessment

Unclear. Current main is the right narrow fix for transcript preservation, but the requested confirmation, icon/placement, or hide setting needs maintainer UX direction and should be reconciled with #61423.
