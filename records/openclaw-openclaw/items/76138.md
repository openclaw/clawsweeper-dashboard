## Summary

Keep open. Current main has mitigated the transcript-wipe side by creating and switching to a fresh dashboard session, but the composer toolbar still exposes a plus-labelled New session button next to the send controls and wires it directly to session creation without confirmation, relocation, icon change, or a hide setting; open PR #61423 is related but does not settle this toolbar affordance in the provided context.

## What This Changes

Request to reduce accidental Control UI New Session activation by adding confirmation, changing or relocating the plus icon, or adding a hide setting.

## Best Possible Solution

Keep the fresh-session create-and-switch behavior, then make one maintainer-approved Control UI affordance change that removes the send-adjacent plus ambiguity and coordinates with the header modal work.

## Reproduction Assessment

Yes. Source inspection on current main shows the plus-labelled New session button still renders in the composer toolbar and calls createChatSession directly, while the config schema has no hide setting for that control.

## Solution Assessment

Unclear. Confirmation is the narrowest code change, but the issue also asks for icon, placement, or config changes and related PR #61423 overlaps the session-creation UI, so maintainers should choose the desired UX before implementation.
