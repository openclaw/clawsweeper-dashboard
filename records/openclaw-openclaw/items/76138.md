## Summary

Keep open. Current main mitigates transcript loss by creating and switching to a fresh dashboard session, but the Control UI still renders a plus-icon New session button in the composer toolbar beside the send controls and invokes session creation without confirmation, relocation, a clearer icon, or a hide setting; related #61423 is still open and does not close this issue.

## What This Changes

The issue requests a safer Control UI New Session affordance through confirmation, clearer icon or placement, or an optional hide setting.

## Best Possible Solution

Keep the non-destructive fresh-session behavior, then make one approved Control UI change that removes the composer-toolbar plus ambiguity and aligns with the accepted header or session-selector pattern.

## Reproduction Assessment

Yes. Source inspection on current main shows renderChatRunControls still renders button[title="New session"] with icons.plus and the click path calls createChatSession through renderChat.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but the issue presents several UX alternatives, so the best fix should be selected by maintainers and reconciled with #61423.
