## Summary

Current main solves the reported Control UI session-loss hazard: the toolbar New Session action now creates and switches to a fresh dashboard session instead of directly resetting the active transcript, and button-triggered reset overrides are confirmation-gated.

## What This Changes

Request to prevent accidental Control UI New Session toolbar clicks from losing the active conversation by adding confirmation, changing or moving the button, or making it configurable.

## Best Possible Solution

Keep the current create-and-switch New Chat behavior; any future icon or placement preference can be handled as a separate UI polish item rather than a session-loss bug.

## Reproduction Assessment

Yes. The relevant path is the Control UI toolbar New Session button, and current-main source plus regression tests show it creates/switches to a new dashboard session while preserving composer state instead of resetting the active transcript.

## Solution Assessment

Yes. Replacing the destructive one-click reset with a safe create-and-switch flow is a narrower and better product fix than hiding the button; the remaining plus-icon preference is no longer tied to data loss.
