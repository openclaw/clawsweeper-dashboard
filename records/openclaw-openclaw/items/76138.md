## Summary

Keep open. Current main mitigates the destructive transcript-loss concern by creating a fresh dashboard session and preserving the current composer, but the reported Control UI affordance remains: a plus-icon New session button still sits in the composer toolbar beside send controls and invokes session creation on a single click with no confirmation, icon/placement change, or hide setting.

## What This Changes

Request to prevent accidental Control UI New session clicks by adding confirmation, changing or relocating the plus button, or adding a hide setting.

## Best Possible Solution

Keep the fresh-session creation behavior, then make one maintainer-approved Control UI affordance change that removes the send-adjacent plus ambiguity and aligns with any accepted header/session-selector work.

## Reproduction Assessment

Yes. Source inspection on current main shows the composer toolbar renders a plus-labelled New session button and its click path calls createChatSession directly; the run-controls test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but selecting confirmation, clearer icon/placement, or a hide setting should be reconciled with the open header-modal/session-selector work in #61423.
