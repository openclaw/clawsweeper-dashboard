## Summary

Keep open. Current main mitigates the older destructive reset by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: a plus-labelled New session button is still in the composer toolbar beside the send/export controls, wired directly to session creation with no confirmation, relocation, clearer icon, or hide setting.

## What This Changes

Request to reduce accidental Control UI New Session clicks by adding confirmation, changing or relocating the plus icon, or adding a hide/config option.

## Best Possible Solution

Keep the create-and-switch session flow, then make one maintainer-approved Control UI affordance change that removes the attachment-like send-adjacent ambiguity and is reconciled with #61423.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders a plus-labelled New session button and the click path calls createChatSession directly; the run-controls test asserts the click immediately invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be coordinated with #61423.
