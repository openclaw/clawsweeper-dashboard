## Summary

Keep open. Current main mitigates in-place transcript loss by creating a fresh dashboard session, but the Control UI still exposes a send-adjacent plus-icon New session toolbar button that immediately creates/switches sessions without confirmation, relocation, clearer icon, or a hide setting; open #61423 overlaps but does not currently close this issue or remove the toolbar affordance.

## What This Changes

Request to reduce accidental Control UI New Session activation by adding confirmation, changing or relocating the plus icon, or adding a hide setting.

## Best Possible Solution

Keep the fresh-session creation behavior, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and aligns the toolbar/header session-entry design with #61423.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and its click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation guard would be narrow, but current main already mitigates transcript loss by creating a separate dashboard session, so choosing confirmation, icon/placement, or config needs UI/product direction and coordination with #61423.
