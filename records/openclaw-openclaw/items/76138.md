## Summary

Keep open. Current main still renders the composer-toolbar plus icon as New session beside the send controls and wires one click directly to fresh-session creation without confirmation, relocation, clearer icon, or a hide setting; the transcript-loss part is mitigated, but the reported affordance remains.

## What This Changes

Request to reduce accidental Control UI New session activations by adding confirmation, changing or relocating the plus icon, or exposing a hide setting.

## Best Possible Solution

Keep the sessions.create fresh-session behavior, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity while leaving broader reset-command policy to https://github.com/openclaw/openclaw/issues/45564.

## Reproduction Assessment

Yes. Source inspection on current main shows the composer toolbar still renders the plus-labelled New session button, places it beside send controls, and routes one click directly to createChatSession; I did not need a browser run to verify that code path.

## Solution Assessment

Unclear. A confirmation dialog would be narrow, but choosing between confirmation, icon/placement changes, and a hide preference is a UI/product decision rather than a mechanical bug fix.
