## Summary

Keep open. Current main preserves the prior transcript by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: the idle composer toolbar still shows a plus-icon New session button beside the send controls and wires it directly to session creation without confirmation, relocation, clearer icon, or a hide setting.

## What This Changes

Request to prevent accidental Control UI New Session clicks by adding confirmation, changing or relocating the plus button, or adding a hide/config option.

## Best Possible Solution

Keep the fresh-session behavior, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and coordinates with #61423's header/modal direction.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders the plus-labelled New session button and its click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be reconciled with #61423.
