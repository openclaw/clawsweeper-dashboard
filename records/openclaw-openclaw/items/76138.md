## Summary

Keep open. Current main has mitigated the older transcript-loss behavior by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: an idle composer toolbar still shows a plus-labelled New session button beside the send controls, wired directly to session creation without confirmation, relocation, clearer iconography, or a hide setting.

## What This Changes

Request to reduce accidental Control UI session creation by confirming, changing or moving the plus button, or adding a hide setting.

## Best Possible Solution

Keep the shipped create-and-switch session behavior, then make one maintainer-approved Control UI change that removes the attachment-like send-adjacent ambiguity and coordinates with #61423.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar still renders a plus-labelled New session button and the click path calls createChatSession directly; the run-controls test also asserts the immediate onNewSession click behavior.

## Solution Assessment

Unclear. A confirmation dialog would be a narrow guard, but choosing among confirmation, clearer icon or placement, and a hide setting is a Control UI product decision that should be reconciled with #61423.
