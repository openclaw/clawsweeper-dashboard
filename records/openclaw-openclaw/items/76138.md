## Summary

Keep open. Current main mitigates the older transcript-loss behavior by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: the idle composer toolbar still shows a plus-icon New session button beside the send controls and wires one click directly to session creation without confirmation, relocation, clearer icon, or a hide setting.

## What This Changes

The issue requests a safer Control UI New session affordance by adding confirmation, changing or moving the plus icon, or making the button configurable.

## Best Possible Solution

Keep the fresh dashboard-session creation behavior, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and coordinates cleanly with #61423.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders a plus-labelled New session button and the click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing confirmation versus icon/placement/config should be reconciled with the open header-modal work in #61423.
