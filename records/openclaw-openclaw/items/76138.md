## Summary

Keep open. Current main mitigates the destructive transcript-wipe behavior by creating and switching to a fresh dashboard session, but the requested Control UI affordance remains unresolved: the composer toolbar still shows a plus-labelled New session button beside the send controls, wired directly to session creation with no confirmation, relocation, clearer icon, or hide setting; the discussion comments reinforce the icon-confusion problem, and open PR #61423 does not yet settle this toolbar button.

## What This Changes

Request a safer Control UI New Session affordance by adding confirmation, changing or relocating the plus icon, or adding a hide setting.

## Best Possible Solution

Keep the shipped create-and-switch session behavior, then make one maintainer-approved Control UI change that removes the attachment-like send-adjacent ambiguity instead of relying only on user CSS workarounds.

## Reproduction Assessment

Yes. Source inspection on current main shows the plus-labelled New session button still renders in the composer toolbar and calls createChatSession directly, with no hide-config or confirmation gate on that path.

## Solution Assessment

Unclear. The current implementation is the right narrow mitigation for data preservation, but choosing between confirmation, clearer icon/placement, or a config setting is a UI/product decision that should be reconciled with #61423.
