## Summary

Keep open. Current main fixed the worst data-loss behavior by creating and switching to a preserved dashboard session, but it has not resolved the requested Control UI affordance: the composer toolbar still shows a plus-labelled New session button beside Send, with no hide setting, icon/placement change, or confirmation on that direct create-session path.

## What This Changes

The issue asks to make the Control UI toolbar New Session plus button less misclick-prone by adding confirmation, changing or moving the icon, or adding a hide setting.

## Best Possible Solution

Keep the current create-and-switch preservation behavior, then make one product-scoped Control UI affordance change that prevents the composer plus button from reading like an attachment or one-click destructive action.

## Reproduction Assessment

Yes. Source inspection on current main shows the plus-labelled New session button still renders in the composer toolbar and directly calls createChatSession, while no hide-New-session config key exists.

## Solution Assessment

Unclear. The current implementation is a good narrow fix for preserving old sessions, but choosing confirmation, icon/placement, or configurability is a maintainer UI/product decision and should be reconciled with #61423.
