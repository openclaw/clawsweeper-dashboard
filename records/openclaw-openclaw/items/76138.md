## Summary

Keep open. Current main mitigates the transcript-loss part by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: the composer still shows a plus-icon New session button beside the send controls and one click creates a session without confirmation, relocation, clearer icon, or a hide setting. The follow-up comments add useful context about icon confusion, and #61423 is relevant but still open and not a complete settled fix.

## What This Changes

Request to make the Control UI New Session affordance less misclick-prone by adding confirmation, changing or moving the icon, or offering a hide setting.

## Best Possible Solution

Adopt one maintainer-approved Control UI UX change that removes the send-adjacent plus ambiguity while preserving fresh-session creation and coordinating with #61423.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders a plus-labelled New session button and a single click calls createChatSession directly; the unit test also asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be reconciled with #61423.
