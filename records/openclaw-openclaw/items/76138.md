## Summary

Keep open. Current main preserves old transcripts by creating a fresh dashboard session, but the reported Control UI affordance remains: the idle composer toolbar still exposes a send-adjacent plus-icon New session button with no confirmation, relocation, clearer icon, or hide setting.

## What This Changes

The issue requests a safer Control UI New Session affordance by adding confirmation, changing or relocating the plus icon, or adding a hide option.

## Best Possible Solution

Keep the fresh-session implementation, then make one maintainer-approved Control UI change that removes the ambiguous send-adjacent plus affordance and aligns with any accepted header/session-selector work.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders `button[title="New session"]` beside the send controls, and the unit test asserts one click invokes `onNewSession`.

## Solution Assessment

Unclear. A confirmation dialog would be a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting needs maintainer UI direction and coordination with #61423.
