## Summary

Keep open. Current main mitigates the original transcript-loss concern by creating a separate dashboard session, but the reported Control UI affordance remains: the idle composer toolbar still shows a plus-icon New session button beside the send controls and invokes session creation without confirmation, relocation, a clearer icon, or a hide setting.

## What This Changes

The issue asks to reduce accidental New Session activation in the Control UI by adding confirmation, changing or relocating the plus button, or providing a hide setting.

## Best Possible Solution

Keep the fresh-session implementation, then make one maintainer-approved Control UI change that removes the send-adjacent plus ambiguity and coordinates with #61423.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders a plus-labelled New session button and its click path calls createChatSession directly; the unit test asserts one click invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be reconciled with #61423.
