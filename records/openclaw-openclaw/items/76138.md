## Summary

Keep open. Current main preserves the previous transcript by creating and switching to a fresh dashboard session, but the reported Control UI affordance remains: the composer toolbar still shows a plus-labelled New session button next to the send controls, wired directly to session creation with no confirmation, relocation, clearer icon, or hide setting.

## What This Changes

Request to make the Control UI New Session toolbar button less accidental and less attachment-like by adding confirmation, changing or relocating the icon, or adding a hide configuration option.

## Best Possible Solution

Keep the shipped create-and-switch session behavior, then make one maintainer-approved Control UI change that removes the attachment-like send-adjacent ambiguity and aligns with any accepted header-session work.

## Reproduction Assessment

Yes. Source inspection on current main shows the idle composer toolbar renders a plus-labelled New session button and the click path calls createChatSession directly; the run-controls test also asserts the click immediately invokes onNewSession.

## Solution Assessment

Unclear. A confirmation dialog is a narrow guard, but choosing between confirmation, clearer icon or placement, and a hide setting is a UI/product decision that should be reconciled with #61423.
