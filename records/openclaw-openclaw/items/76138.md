## Summary

Keep open. Current main fixed the most destructive behavior by creating and switching to a preserved dashboard session, but the requested Control UI affordance remains unresolved: the composer toolbar still has a plus-labelled New session button beside the send controls, with no confirmation, relocation, clearer icon, or hide setting; open PR #61423 overlaps part of the header/modal direction but is not merged and does not clearly settle the existing toolbar button.

## What This Changes

Request to make the Control UI New Session toolbar action safer or clearer by adding confirmation, changing or relocating the plus icon, or adding a hide setting.

## Best Possible Solution

Keep the create-and-switch session behavior, then resolve the remaining Control UI affordance in one product-scoped UI change that makes session creation discoverable without looking like an attachment or destructive send-adjacent action.

## Reproduction Assessment

Yes. Source inspection on current main shows the plus-labelled New session button still renders in the composer toolbar and calls `createChatSession` directly without a confirmation or hide-config gate.

## Solution Assessment

Unclear. Current main is the right narrow fix for data preservation, but choosing between confirmation, clearer icon/placement, or a config hide option is a maintainer UI/product decision that should be reconciled with PR #61423.
