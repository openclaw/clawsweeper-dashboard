## Summary

Keep open. Current main fixed the destructive transcript-reset part by creating and switching to a preserved dashboard session, but the remaining Control UI affordance is still present: the composer toolbar renders a plus-labelled New session button beside Send with no confirmation, no relocation or clearer icon, and no hide setting; #61423 overlaps the header/modal direction but is open and does not settle the existing toolbar button.

## What This Changes

Request to prevent accidental Control UI New Session activation by adding confirmation, changing or relocating the plus icon, or adding a hide setting.

## Best Possible Solution

Adopt one Control UI pattern that removes the send-adjacent ambiguous plus action while preserving quick session creation and the shipped create-and-switch session behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the plus-labelled New session button still renders in the composer toolbar and calls createChatSession directly, while config inspection shows no hide setting.

## Solution Assessment

Unclear. The shipped create-and-switch implementation is the right data-preservation base, but choosing confirmation, icon/placement, or configurability is a maintainer UI/product decision rather than an obvious one-line fix.
