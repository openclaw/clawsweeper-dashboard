## Summary

Keep this PR open, but it needs changes before merge: the main fix is aimed at the right Gateway sessions bug, yet the new preservation helper uses a raw profile-id prefix comparison and would still drop provider-auth aliases that are compatible in the existing auth contract.

## What This Changes

The PR changes Gateway `sessions.patch` model handling to preserve same-provider session auth overrides and adds regression tests for same-provider preservation and cross-provider cleanup.

## Best Possible Solution

Use the existing alias-aware auth-profile compatibility contract when deciding whether to preserve an auth override, while keeping cross-provider cleanup and adding an alias regression case.

## Reproduction Assessment

Yes. Source inspection shows Dashboard model switches send only `key` and `model`, current main routes that to `applyModelOverrideToSessionEntry` without preservation, and the helper deletes the stored auth override.

## Solution Assessment

No. The PR is close, but the raw prefix comparison is not the best fix because runtime auth compatibility already resolves provider-auth aliases through `resolveProviderIdForAuth`.
