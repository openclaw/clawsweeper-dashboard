## Summary

Keep open: the PR fixes the main Dashboard `sessions.patch` auth-profile clearing path, but its preservation helper still compares raw provider prefixes, so valid provider-auth alias profiles remain broken.

## What This Changes

The PR updates gateway session patching and tests to preserve auth profile overrides across same-provider model changes, and also changes a Mantis Telegram proof evidence test expectation.

## Best Possible Solution

Use the existing alias-aware auth-provider contract when deciding preservation, keep cross-provider cleanup, and add a regression for a provider-auth alias case.

## Reproduction Assessment

Yes. Source inspection shows the Dashboard sends only `key` and `model`, and current main then clears the stored auth override through `applyModelOverrideToSessionEntry`; the remaining PR bug is also source-reproducible with a `byteplus-plan` model and `byteplus:*` auth profile.

## Solution Assessment

No. The PR is directionally right, but the preservation decision should normalize both the selected provider and auth-profile provider prefix through `resolveProviderIdForAuth` or an existing compatibility helper.
