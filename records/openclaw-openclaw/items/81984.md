## What This Changes

The issue reports that `/stop` followed by `/new` can silently leave the prior session transcript unarchived and invisible to downstream ingestion.

## Best Possible Solution

Yes in direction: surfacing reset/new archive failures is the narrow maintainable fix, while changing `/stop` into a transcript finalizer would conflict with the documented command contract.
