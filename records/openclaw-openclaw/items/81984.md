## Summary

Keep open: current main still swallows transcript archive rename failures during `/new` rollover, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks for `/stop` followed by `/new` to durably archive the prior transcript or fail visibly instead of reporting success with missing archive content.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82081 or an equivalent narrow fix that surfaces reset/new archive failures while preserving documented `/stop` abort-only behavior.

## Reproduction Assessment

Yes at source level, but not as a live gateway run: if `archiveFileOnDisk` throws during `/new` rollover, `archiveSessionTranscriptsDetailed` catches and ignores it, so the new session can continue without surfacing the failed archive.

## Solution Assessment

Yes in direction: surfacing reset/new archive failures is the narrow maintainable fix, while changing `/stop` into a transcript finalizer would fight the documented command contract. The linked PR is the active candidate and should be reviewed or replaced before this issue closes.
