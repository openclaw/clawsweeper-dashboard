## Summary

Keep open: current main and the latest release still swallow transcript archive rename failures during `/new` rollover, and the linked fix PR is open rather than merged.

## What This Changes

The issue reports that `/stop` followed by `/new` can return success while the previous session transcript archive fails silently, leaving downstream ingestion with missing session content.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82081, or an equivalent narrow fix that surfaces `/new` rollover archive failures while preserving `/stop` abort-only behavior.

## Reproduction Assessment

Yes, at source level: `archiveFileOnDisk` errors are caught and ignored by `archiveSessionTranscriptsDetailed`, and `/new` calls that helper for the previous session without a failure callback. I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes in direction: the active linked PR is the narrow maintainable fix because it surfaces `/new` archive failures without changing `/stop` into a transcript finalizer. The issue should remain open until that PR or an equivalent replacement lands.
