## Summary

Keep open: current main still lets `/new` continue after transcript archive rename failures are caught as best-effort, and the linked open PR is the active implementation path rather than a reason to close this issue early.

## What This Changes

The issue requests that `/stop` followed by `/new` durably archive the prior transcript or surface archive failures instead of silently reporting success.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82081 or an equivalent narrow fix that surfaces reset/new archive failures while keeping `/stop` abort-only.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the failure path clearly: archive rename errors are caught and ignored, and `/new` uses that helper during previous-session rollover.

## Solution Assessment

Yes in direction: making the reset/new archive operation fail-loud or at least log a structured warning is the narrow maintainable fix. Changing `/stop` into a transcript-finalization command or requiring a new journal line should not be necessary for the core data-loss bug.
