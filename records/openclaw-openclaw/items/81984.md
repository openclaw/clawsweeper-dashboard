## Summary

Current main still has the reported silent archive-failure path: `/new` archives the previous session through a helper that swallows rename/archive failures. The linked open PR is the active closing implementation, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `/stop` followed by `/new` can silently lose the previous session transcript when archive persistence fails.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82081 or an equivalent narrow fix that surfaces previous-session archive failures during `/new` rotation while keeping `/stop` abort-only.

## Reproduction Assessment

Yes at source level, but not as a live end-to-end run in this review. Current main calls `archiveSessionTranscriptsDetailed` during `/new` rollover, and that helper catches archive/rename failures without reporting them.

## Solution Assessment

Yes in direction: the narrow maintainable fix is to surface archive failures from the reset/new archive operation while preserving `/stop` as an abort command. The linked PR is the active implementation path and should be reviewed or replaced by an equivalent patch.
