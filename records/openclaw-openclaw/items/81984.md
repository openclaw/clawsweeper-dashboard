## Summary

Keep open: current main and the latest release still swallow archive failures during `/new` rollover, and the linked fixing PR is open rather than merged.

## What This Changes

The issue reports that `/stop` followed by `/new` can silently leave the prior session transcript unarchived, causing lost transcript content for downstream ingestion.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82081 or an equivalent narrow fix that surfaces `/new` archive failures while preserving the documented `/stop` abort-only behavior.

## Reproduction Assessment

Yes at source level: if `archiveFileOnDisk` throws during `/new` rollover, `archiveSessionTranscriptsDetailed` catches and ignores it, so the new session can continue without surfacing the failed archive.

## Solution Assessment

Yes in direction: surfacing reset/new archive failures is the narrow maintainable fix, while changing `/stop` into a transcript finalizer would fight the documented command contract.
