## Summary

Keep open: current main still swallows per-transcript archive failures during `/new` rollover, and the linked open PR is the active fix path rather than a reason to close this issue before merge.

## What This Changes

The issue asks for `/stop` followed by `/new` to preserve or fail-loudly archive the stopped session transcript instead of reporting success after a silent archive failure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82081 or an equivalent narrow fix that makes `/new` rollover archive failures visible while preserving the documented `/stop` abort behavior.

## Reproduction Assessment

No live reproduction was established in this read-only review. Source inspection gives a high-confidence path: force `archiveFileOnDisk` or `fs.renameSync` to throw during `/new` rollover and current main catches the failure without surfacing it.

## Solution Assessment

Yes in direction: surfacing or propagating previous-session archive failures from the `/new` rollover path is the narrow maintainable fix. Changing `/stop` into a transcript-finalization command is broader than the documented command contract requires.
