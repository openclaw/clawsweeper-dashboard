## Summary

Keep open: current main still swallows transcript archive failures during `/new` rollover, and the linked open PR is the active closing path rather than a reason to close the issue before merge.

## What This Changes

The issue reports that a `/stop` followed by `/new` can return success while the prior session transcript is not durably archived for downstream ingestion.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82081 or an equivalent narrow fix that surfaces `/new` archive failures while preserving `/stop` as abort-only.

## Reproduction Assessment

Yes, at source level: current main catches and ignores archive rename failures, and `/new` uses that helper during previous-session rollover. I did not run a live gateway or Telegram reproduction in this read-only review.

## Solution Assessment

Yes in direction: surfacing archive failures from the reset/new archive path is the narrow maintainable fix. Making `/stop` a transcript-finalization command or adding a new journal contract is broader than needed for the core bug.
