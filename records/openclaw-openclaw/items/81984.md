## Summary

Keep open: current main still silently ignores archive rename failures during `/new` rollover, and the linked fix PR is open rather than merged.

## What This Changes

The issue reports that `/stop` followed by `/new` can silently fail to persist the previous session transcript archive.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82081, or an equivalent narrow fix that surfaces `/new` rollover archive failures while keeping `/stop` abort-only.

## Reproduction Assessment

Yes at source level, but not as a live gateway run. If `archiveFileOnDisk` throws during `/new` rollover, `archiveSessionTranscriptsDetailed` catches and ignores it, so the new session can continue without surfacing the failed archive.

## Solution Assessment

Yes in direction: surfacing reset/new archive failures is the narrow maintainable fix, while changing `/stop` into a transcript finalizer would fight the documented command contract. The open linked PR is the active candidate but still needs normal review/proof before this issue closes.
