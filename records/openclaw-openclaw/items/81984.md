## Summary

Keep open: current main still swallows transcript archive failures during reset/new rotation, and an open linked PR is already the active fix path, so closing this issue before that PR lands would be premature.

## What This Changes

The issue reports that `/stop` followed by `/new` can lose or fail to archive the prior transcript without surfacing an error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82081 or an equivalent narrow fix that surfaces archive failures during `/new` or reset rollover while preserving `/stop` as abort-only behavior.

## Reproduction Assessment

No, not as a live gateway reproduction. Source inspection on current main shows `/new` archives the previous session through a helper that suppresses rename/archive errors, so the failure mode is source-reproducible.

## Solution Assessment

Yes, the linked reset-side hook/logging direction is the narrow maintainable fix. Changing `/stop` into an archive/finalize command would conflict with the documented abort-only command contract.
