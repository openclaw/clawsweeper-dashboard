The issue asks OpenClaw to support provider-official subscription or OAuth-backed model auth in normal setup flows and distinguish those paths from API-credit billing.

## Best Possible Solution

Use the existing provider plugin/auth-choice architecture for provider-sanctioned OAuth or subscription paths, keep unofficial/community integrations outside core or behind clear warnings, and split concrete provider work into owning plugin issues or PRs with docs and setup-status labels.
