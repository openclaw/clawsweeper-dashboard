## Summary

Keep open: current main still evaluates the global circuit breaker from the current tool signature's no-progress streak, while the docs describe a global no-progress breaker; the linked open PR is the right fix path but has not merged yet.

## What This Changes

Request that `globalCircuitBreakerThreshold` count stable no-progress outcomes across the session/run instead of resetting by tool type or tool signature.

## Best Possible Solution

Review and land the linked session/run-scoped aggregate breaker PR, keeping the existing per-pattern detectors and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: record stable no-progress outcomes under one or more tool signatures, then attempt a different tool and current main reads only that different signature's streak for the global breaker.

## Solution Assessment

Yes. The narrow maintainable fix is to make the existing documented breaker aggregate stable no-progress outcomes across the active session/run without adding a new config knob.
