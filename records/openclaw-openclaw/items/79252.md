## Summary

Keep open: current main still computes the global circuit breaker from a same-tool/same-args no-progress streak, while the docs describe a global/any-run breaker, so the cross-tool evasion report remains valid and narrow.

## What This Changes

Make `globalCircuitBreakerThreshold` count no-progress tool outcomes across the session or run rather than independently per tool type.

## Best Possible Solution

Add a session/run-scoped aggregate no-progress breaker in the tool-loop detector, keep the existing per-pattern detectors, and cover cross-tool switching with unit and before-tool hook tests.

## Reproduction Assessment

Yes, from current-main source inspection: construct a session history with stable no-progress outcomes split across multiple tool names below each tool's own threshold, and the current detector will only count the current tool+args streak. A targeted test should use ordered thresholds such as warning 2, critical 4, global 5 to avoid schema ordering ambiguity.

## Solution Assessment

Yes, the narrow maintainable fix is to make the existing documented breaker aggregate no-progress across the session or run without adding a new config knob. The implementation should stay in `src/agents/tool-loop-detection.ts` plus the before-tool hook regression tests.
