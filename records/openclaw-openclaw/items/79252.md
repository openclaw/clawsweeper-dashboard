## Summary

Keep open. Current main still scopes the “global” breaker to the current tool+args/result signature rather than a session-wide no-progress total, and the related open report at https://github.com/openclaw/openclaw/issues/64500 only covers pair/ping-pong enforcement rather than this broader multi-tool chain.

## What This Changes

The issue requests that `globalCircuitBreakerThreshold` count no-progress calls across all tools in a session/run and block further tool calls after the threshold.

## Best Possible Solution

Implement run/session-scoped global no-progress accounting in the existing loop-detection path, preserving the current config/defaults and adding regression coverage for both multi-tool rotation and related pair-loop cases.

## Reproduction Assessment

Yes, at source level. A focused reproduction is a run-scoped history with stable no-progress outcomes spread across multiple tool names, where each per-signature streak remains below the threshold while the total exceeds it.

## Solution Assessment

Yes, if the fix stays inside the existing loop-detection contract. The narrowest maintainable solution is to add session/run-scoped global no-progress enforcement without adding a new config option; the pair-only work in https://github.com/openclaw/openclaw/issues/64500 would not cover the full rotating-tool chain by itself.
