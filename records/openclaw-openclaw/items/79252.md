## Summary

Keep open: current main still computes the documented global breaker from the current tool+args no-progress streak, and an open closing PR is the right path to resolve this rather than closing the issue before merge.

## What This Changes

Request that `globalCircuitBreakerThreshold` count stable no-progress outcomes across all tools in a session/run instead of resetting by tool signature.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79267 or an equivalent session/run-scoped aggregate breaker with detector and before-tool-call regression coverage, then close this issue.

## Reproduction Assessment

Yes. Source inspection on current main shows `detectToolCallLoop` computes the breaker from `getNoProgressStreak(history, toolName, currentHash)`, so a history split across different tool signatures stays below the breaker for the next tool.

## Solution Assessment

Yes. The narrow maintainable fix is to make the existing documented breaker aggregate stable no-progress outcomes across the session/run while preserving the existing per-pattern detectors and thresholds.
