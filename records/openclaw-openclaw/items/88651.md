## Summary

Keep open: current main and the latest release still have the unbounded steer-rate map, but this PR is not merge-ready because the FIFO cap can evict active throttle entries and the external proof is prose-only.

## What This Changes

The PR adds a 1024-entry size cap and a 60-second stale sweep to the in-memory subagent steer rate-limit map in `src/agents/subagent-control.ts`.

## Best Possible Solution

Prune or bound the map without deleting entries younger than `STEER_RATE_LIMIT_MS`, add focused regression coverage through a testable helper, and require redacted after-fix Gateway proof before merge.

## Reproduction Assessment

Yes from source inspection: current main writes unique controller/child keys into a module-level map and never deletes them. I did not run a long-duration Gateway reproduction.

## Solution Assessment

No: stale cleanup is the right direction, but the fixed FIFO cap can delete active in-window throttle entries. The safer fix is to prune entries old enough to be irrelevant to the 2-second throttle and preserve active keys.
