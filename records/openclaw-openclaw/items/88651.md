## Summary

Keep this PR open: the underlying unbounded `steerRateLimit` map still exists on current `main` and `v2026.6.9`, but this patch is not merge-ready because the hard FIFO cap can remove active throttle entries and the external proof is prose-only.

## What This Changes

The PR adds a 1024-entry cap, a 60-second stale sweep, and FIFO eviction to the in-memory subagent steer rate-limit map.

## Best Possible Solution

Bound only entries that are older than the steer throttle window, add focused limiter coverage through a testable helper or injected clock, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes for the source-level growth path: current `main` and `v2026.6.9` store accepted steer keys in a module-level map without deleting them. I did not reproduce a long-running Gateway OOM.

## Solution Assessment

No; stale cleanup is the right direction, but the hard FIFO cap can remove active in-window throttle entries. A safer fix prunes only entries old enough to be irrelevant to the two-second limiter window.
