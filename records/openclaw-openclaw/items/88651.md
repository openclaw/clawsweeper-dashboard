## Summary

Keep open: current main and the latest release still have the unbounded in-memory steer rate-limit map, but this PR is not merge-ready because the FIFO cap can evict active throttle entries and the after-fix proof is prose-only.

## What This Changes

The PR adds a 1024-entry cap and a 60-second stale sweep to the in-memory subagent steer rate-limit map.

## Best Possible Solution

Bound the map by pruning entries that are older than the steer throttle window, preserve every in-window key, add focused coverage through a testable helper, and require redacted after-fix Gateway proof before merge.

## Reproduction Assessment

Yes from source inspection: current main writes unique controller/child keys into a module-level map and does not delete them. I did not run a long-duration Gateway reproduction.

## Solution Assessment

No: stale cleanup is the right direction, but the fixed FIFO cap can delete active in-window throttle entries. The safer fix is to prune only entries old enough to be irrelevant to the 2-second throttle.
