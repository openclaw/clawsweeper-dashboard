## What This Changes

This PR caps skill download installer responses at 256 MiB, rejects oversized advertised responses, enforces the same cap while streaming, and updates the focused regression test and skills docs.

## Best Possible Solution

Merge this narrow cap, regression test, and docs change after real behavior proof or maintainer override, then close https://github.com/openclaw/openclaw/issues/81817 when the fix lands.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence reproduction path: mock installDownloadSpec with Content-Length 268435457 and extract: false, and the current downloadFile path pipes the response to staging without a preflight or streaming byte cap.

## Solution Assessment

Yes for the code approach. The PR follows the adjacent marketplace 256 MiB cap pattern by rejecting oversized advertised lengths and counting streamed bytes before the staging file can exceed the limit; the remaining blocker is proof, not design.
