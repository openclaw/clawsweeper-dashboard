## Summary

Keep open: this is a focused fix for a source-reproducible unbounded skill download write, and I found no blocking code findings, but the external PR only provides mocked test proof so it needs after-fix real behavior proof before merge.

## What This Changes

The PR updates the skill download installer, its regression tests, and the skills reference to reject advertised or streamed download responses over 256 MiB.

## Best Possible Solution

Keep the focused installer cap, ask for real behavior proof, then land this PR so https://github.com/openclaw/openclaw/issues/81817 closes after merge.

## Reproduction Assessment

Yes, source-reproducible: current main's `downloadFile` pipes `response.body` directly to the staging file after only `ok/body` checks, so the linked oversized `Content-Length` mock would not be rejected before writing. I did not run the regression because this review is read-only.

## Solution Assessment

Yes, the PR puts the cap at the narrow download boundary and enforces both advertised and streamed sizes before the final copy/extract path. The remaining blocker is proof quality, not a code-shape problem.
