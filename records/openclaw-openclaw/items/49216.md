## Summary

Keep open. Current main still has the deleted-archive cleanup gap when reset archive retention is disabled, and this PR targets the right maintenance path, but it needs a changelog entry and a stronger regression test before merge.

## What This Changes

The PR updates session archive timestamp parsing and enforced session-store maintenance so expired deleted/reset archive artifacts are pruned more reliably, with focused tests.

## Best Possible Solution

Land a rebased focused fix that cleans deleted archives independently of reset archive retention, preserves the reset cleanup opt-out, covers legacy timestamp parsing, and includes a changelog entry.

Yes. From source, enforced maintenance with `resetArchiveRetention: false`, no pruned live sessions, and an expired `.deleted.*` artifact leaves `archivedDirs` empty and `resetArchiveRetentionMs` null, so current main skips deleted-archive cleanup.

## Solution Assessment

Yes for the code direction. Running deleted archive cleanup against the store directory during enforced maintenance is the narrow fix; the PR should add the changelog and make the regression test exercise the disabled reset-retention path.
