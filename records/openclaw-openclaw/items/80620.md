## Summary

Keep open. The CJK fix direction is right, but this PR currently changes dreaming dedupe so distinct same-path snippets that produce empty token sets, such as Cyrillic, Arabic, emoji-only, or punctuation-only text, can be treated as identical and one can be dropped.

## What This Changes

The branch extracts memory-core's CJK-aware MMR tokenizer into a shared helper, routes dreaming dedupe through it, and adds focused CJK dedupe regression tests.

## Best Possible Solution

Keep the shared CJK tokenizer extraction, but add a dreaming-specific similarity wrapper that preserves the old exact normalized string fallback when both sides tokenize to empty sets.

## Reproduction Assessment

Yes. Source inspection shows current main uses exact-string fallback for empty-token dreaming dedupe, while the PR call path uses a helper that returns 1 for any two empty token sets.

## Solution Assessment

No as currently patched. The shared tokenizer is the right direction for the CJK bug, but dreaming dedupe needs a small adapter so non-tokenized content does not regress into false merges.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve exact-match fallback for empty-token snippets:** `extensions/memory-core/src/dreaming-phases.ts:1360`
  - body: `textSimilarity` returns `1` whenever both token sets are empty. After routing dreaming dedupe through it, two distinct same-path snippets in scripts this tokenizer does not cover, or emoji/punctuation-only snippets, satisfy normal dedupe thresholds and one is dropped. Keep the CJK-aware helper, but wrap this call so empty-token pairs only merge when the normalized full strings match, as current main does.
