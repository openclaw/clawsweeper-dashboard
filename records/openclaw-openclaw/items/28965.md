## Summary

Keep open. Current main still has targeted model status/list/set/scan/auth commands and per-session patch/reset behavior, but it does not implement the proposed first-class `openclaw models reconcile` workflow or a scoped `openclaw sessions model-reset` operator command. The related open item #59750 covers a narrower resumed-session strategy question, and the closed #41736 does not supersede this broader reconcile/reset proposal.

## What This Changes

Request to add an official model configuration reconcile command, a scoped session model override reset command, and clearer model-source observability.

## Best Possible Solution

Define the canonical model/config reconciliation contract first, then implement documented operator commands for `openclaw models reconcile` and `openclaw sessions model-reset` with dry-run and JSON output, scoped target selection, tests for preservation vs reset semantics, and docs that explain model-source precedence.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence verification path for the requested feature gap: source and docs searches on current main show the requested commands are absent while existing reset/patch behavior remains narrower.

## Solution Assessment

Unclear until maintainers define the contract. The requested direction is plausible, but a safe implementation needs explicit rules for which sources are authoritative and which session/auth fields may be reset or preserved.
