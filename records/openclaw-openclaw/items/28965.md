## Summary

Keep open. Current main documents and implements model status/list/set/scan plus per-session patch/reset semantics, but it still does not provide the requested first-class `openclaw models reconcile` command or scoped `openclaw sessions model-reset` workflow. The remaining work is a broader model/config contract decision, not a conservative cleanup close.

## What This Changes

The issue proposes a single-source model configuration reconcile command, a scoped session model-override reset command, and clearer model-source observability for operators changing defaults.

## Best Possible Solution

Define the model configuration reconciliation contract first, then add documented CLI workflows for reconciling model config and explicitly clearing scoped session model/provider/auth overrides with dry-run and JSON reporting.
