## Summary

Keep open: current `origin/main` and `v2026.6.5` still do not provide `openclaw models reconcile` or `openclaw sessions model-reset`. Current code covers narrower pieces, such as `models set` ensuring the selected default exists in `agents.defaults.models` and `/reset` clearing auto-sourced runtime state, but explicit user/legacy session model choices are still intentionally preserved and the open resolver issue does not supersede the requested CLI reconciliation/reset design.

## What This Changes

The issue proposes a core model-configuration reconciliation command, a bulk session model/auth override reset command, and clearer status reporting for model-source precedence.

## Best Possible Solution

Use the shared resolver issue as the foundation, then design an explicit dry-run reconciliation report and scoped reset flow that preserves intentional user choices by default.

## Reproduction Assessment

Not applicable: this is an enhancement umbrella, not one narrowly failing behavior. Source inspection does give high-confidence proof that the proposed commands are absent, while related open issues provide concrete reproductions for individual override and display problems.

## Solution Assessment

Unclear: a shared resolver is the best foundation, but the proposed reconcile and bulk reset behavior needs maintainer decisions around preservation, dry-run reporting, scope filters, and mutation safety. The current issue should remain open as that product/API design container rather than being folded into the narrower resolver work.
