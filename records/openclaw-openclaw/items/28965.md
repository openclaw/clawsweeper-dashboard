## Summary

Current source and the latest release still lack the proposed reconcile and bulk reset commands; narrower normalization, reset preservation, and display fixes exist, but the maintainer-owned resolver follow-up explicitly leaves the operational reconcile/reset design unresolved.

## What This Changes

Add model-configuration reconciliation, scoped clearing of persisted session model/auth overrides, and clearer reporting of the source of the effective model.

## Best Possible Solution

Land the shared selected/runtime/display/auth resolver first, then split follow-up work into doctor-owned canonical config repair, a non-destructive dry-run reconciliation report, and an explicitly scoped source-aware session reset command.
