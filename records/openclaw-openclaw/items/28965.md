## Summary

Keep open. Current main has partial safeguards and observability for model selection, but the requested official reconcile/reset workflow is still a broad enhancement that needs maintainer product design rather than conservative cleanup closure.

## What This Changes

The issue proposes a single-source model configuration reconcile command, an explicit session model override reset command, and clearer model-source observability.

## Best Possible Solution

Define the intended model precedence and repair UX first, then implement a narrow set of CLI/status changes that reconcile defaults, agent overrides, allowlists, and session overrides without hiding explicit user selections.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a proposal for new CLI workflows and observability. Source inspection does confirm the requested commands are not present and the linked narrower reports cover concrete current behaviors.

## Solution Assessment

Unclear as an implementation plan. A single reconcile/reset command family may be the right direction, but maintainers need to decide which layers are authoritative and which explicit overrides should be preserved, warned about, or reset.
