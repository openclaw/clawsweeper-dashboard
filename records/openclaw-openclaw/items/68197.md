## What This Changes

The PR adds a LINE group `requireMentionForNonText` config option, uses it to gate non-text group messages under `requireMention`, and adds LINE handler regression tests.

## Best Possible Solution

Rebase the opt-in onto the current LINE ingress path, regenerate bundled config metadata and config docs baseline, add redacted real LINE proof, and land only if maintainers accept the config contract.

## Reproduction Assessment

Yes from source, but not from a live LINE run here. Current main sets LINE mention detectability to text-only, the shared gate allows `canDetectMention: false`, and the existing LINE test records that an image group message is processed under `requireMention`.

## Solution Assessment

Mostly yes: a compatibility-preserving opt-in is a narrow maintainable direction for deployments that want stricter LINE group media gating. The proposed patch still needs generated config follow-through, a rebase onto the current ingress path, and real behavior proof before it is the best merge candidate.
