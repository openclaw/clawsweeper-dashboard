## Summary

Keep open. Current main still derives the Mattermost interaction signing secret from the bot token with a public HMAC constant, and the linked security fix PR remains open with unresolved design/proof concerns.

## What This Changes

Report asks OpenClaw to stop deriving Mattermost interaction signing secrets from the bot token with a public HMAC constant.

## Best Possible Solution

Land or replace the linked security PR with a durable per-account Mattermost interaction signing secret independent of the bot token, with aligned schema/types/docs/tests and redacted real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a Mattermost bot token and known interaction context, compute the documented derived secret and `_token`, and current main's `verifyInteractionToken` will accept it before the handler proceeds.

## Solution Assessment

Yes on direction, but not resolved on main. Separating interaction signing from the Mattermost API bot token is the maintainable fix; the exact durable persistence, migration, and docs contract should be resolved in the linked PR or a reviewed successor.
