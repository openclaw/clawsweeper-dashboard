## Summary

Keep open. Current main and v2026.5.12 still derive the Mattermost interaction signing secret from the bot token with a public HMAC constant, while the paired fix PR remains open and unmerged with security-design and proof concerns still unresolved.

## What This Changes

The issue requests replacing the Mattermost bot-token-derived interaction HMAC secret with a durable independent per-account signing secret.

## Best Possible Solution

Land a reviewed security fix that separates Mattermost interaction signing from the bot token with a durable per-account secret, aligned schema/types/docs/tests, and migration or rotation guidance.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with a Mattermost bot token and known interaction context, compute the documented derived secret and `_token` HMAC, and current main's `verifyInteractionToken` will accept it before the handler proceeds.

## Solution Assessment

Yes on direction, but not resolved on main. Separating interaction signing from the Mattermost API bot token is the right fix; the exact durable persistence, migration, and docs contract should be resolved in the linked PR or a reviewed successor.
