## Summary

Keep open. Current main still derives Mattermost interaction signing secrets from the bot token with a public hardcoded HMAC key, the latest release still contains the same behavior, and the provided context shows the closing fix PR is open and unmerged, so this issue remains the active security tracker.

## What This Changes

Replace Mattermost interaction token signing that derives from the bot token via a hardcoded HMAC key with a separate per-account signing secret.

## Best Possible Solution

Keep this issue open as the Mattermost security tracker until PR #64546 or a reviewed successor lands a SecretRef-aware, persistent per-account interaction signing secret, updates schema/types/docs/tests, and ships in a release.

## Reproduction Assessment

Yes. The source, tests, and docs provide a high-confidence reproduction path: given a bot token and interaction context, the current code deterministically derives the same signing secret and verifies a matching `_token` without any additional random per-account secret.

## Solution Assessment

Yes, the requested direction is the maintainable fix: the Mattermost plugin should use a persisted signing secret separate from the bot token, while preserving existing callback URL and source-IP controls as defense in depth.
