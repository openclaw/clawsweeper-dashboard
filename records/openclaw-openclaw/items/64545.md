## Label Justifications

- `P1`: A bundled Mattermost callback security boundary can accept forged interaction tokens when an attacker has the bot token and target interaction context.
- `impact:security`: The issue concerns authentication and integrity for Mattermost button callback requests that can reach system-event dispatch.

## Summary

Keep open. Current main and latest release v2026.6.6 still derive the Mattermost interaction signing root from the bot token with a public HMAC key, and the linked fixing PR is open with maintainer security/product review still needed.

## What This Changes

Fix Mattermost interaction callback signing so `_token` values cannot be forged from the bot token and a public HMAC derivation key.

## Best Possible Solution

Land the linked PR or a maintainer-approved replacement that uses a durable separate per-account Mattermost interaction secret with docs, tests, and fresh-install plus upgrade proof.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 both derive the Mattermost interaction secret from the bot token with a public HMAC key, and current tests assert deterministic secret derivation.

## Solution Assessment

Yes on direction: a separate per-account interaction secret is the right security shape. The exact persistence, config, docs, and upgrade behavior still needs maintainer security/product review.
