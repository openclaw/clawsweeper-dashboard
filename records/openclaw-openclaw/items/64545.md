## Label Justifications

- `P1`: A forgeable Mattermost callback token can trigger channel action flows when bot-token-derived signing is compromised.
- `impact:security`: The issue concerns the authorization boundary and credential-derived signing secret for Mattermost interaction callbacks.

## Summary

Current main and v2026.6.10 still derive the Mattermost interaction signing secret from the bot token with a public HMAC key; the linked fix PR is open, so this security issue is still necessary.

## What This Changes

The issue reports that Mattermost interaction `_token` values are forgeable because signing derives a secret from the bot token using a hardcoded public HMAC key.

## Best Possible Solution

Land a maintainer-approved version of https://github.com/openclaw/openclaw/pull/64546, or a replacement, that uses a durable per-account Mattermost interaction secret with setup/doctor/docs coverage and old-forgery rejection tests.

## Reproduction Assessment

Yes, source-level. Current main derives the Mattermost interaction secret from the bot token with a public HMAC key and verifies callback `_token` values against that derived secret; I did not run a live Mattermost server in this read-only review.

## Solution Assessment

Yes on direction, not yet on an accepted implementation. The narrow maintainable fix is to stop using the bot token as the signing root and use a durable per-account interaction secret with explicit upgrade/setup behavior.
