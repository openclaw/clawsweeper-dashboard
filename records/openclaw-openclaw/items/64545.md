## Label Justifications

- `P1`: The issue tracks a security-sensitive Mattermost interaction signing bypass that remains present in current main and the latest release, with an open fix PR pending review.
- `impact:security`: The report concerns callback token forgery and authorization of Mattermost button interactions using a bot-token-derived signing secret.

## Summary

Keep open. Current main and the latest shipped tag still derive the Mattermost interaction signing secret from the bot token with a public HMAC constant, and the linked closing PR is still open and unmerged.

## What This Changes

The issue asks OpenClaw to stop deriving Mattermost interaction callback HMAC tokens from the bot token with a public constant and use an independent per-account signing secret.

## Best Possible Solution

Land or replace the linked PR with a durable per-account Mattermost interaction signing secret independent of the bot token, plus aligned schema, docs, tests, migration guidance, and redacted real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a Mattermost bot token and known interaction context, compute the documented derived secret and `_token`, and current main's verifier will accept it before the handler proceeds.

## Solution Assessment

Yes on direction, but it is not resolved on main. Separating interaction signing from the Mattermost API bot token is the maintainable fix; persistence, migration, and docs need maintainer-reviewed design in the linked PR or a replacement.
