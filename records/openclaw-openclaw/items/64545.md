## Summary

Keep open. Current main still derives Mattermost interaction signing secrets from the bot token with the public hardcoded HMAC key, the latest tagged release contains the same behavior, and the linked closing PR remains open and unmerged in the provided GitHub context.

## What This Changes

Mattermost interaction token signing should stop deriving a forgeable signing secret from the bot token with a public hardcoded HMAC key.

## Best Possible Solution

Keep this issue open as the Mattermost security tracker until PR #64546 or a reviewed successor lands a persistent per-account interaction signing secret that is separate from the bot token, updates schema/types/docs/tests, and ships in a release.

## Reproduction Assessment

Yes. The current source and tests provide a high-confidence reproduction path: with a bot token and known interaction context, the deterministic derivation and token verification can be recomputed without any separate random per-account secret.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: Mattermost should use a persisted signing secret independent of the API bot token, while retaining source-IP and post/channel validation as defense in depth.
