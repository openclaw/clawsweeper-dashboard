## Label Justifications

- `P1`: This is a concrete security-sensitive Mattermost interaction signing bypass in a shipped channel path, but it is scoped to Mattermost and needs design/proof before release.
- `impact:security`: The issue concerns a forgeable HMAC authorization token that can cross a Mattermost interaction security boundary.

## Summary

Keep open. Current main and the latest shipped tag still derive the Mattermost interaction signing secret from the bot token with a public HMAC constant, and the Fixes-linked security PR remains open with unresolved design/proof concerns.

## What This Changes

The issue requests replacing Mattermost button interaction signing so `_token` values cannot be forged from the bot token plus a public HMAC derivation key.

## Best Possible Solution

Land or replace the linked PR with a reviewed Mattermost fix that uses a durable per-account interaction signing secret independent of the bot token, with schema/docs/tests and redacted real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection shows that a caller with the Mattermost bot token and known interaction context can recompute the same derived secret and `_token`; a live Mattermost repro would still need a valid post/channel and bot token.

## Solution Assessment

Yes on direction, but not resolved on main. Separating interaction signing from the Mattermost API bot token is the right fix, with the durable persistence and migration contract settled in the linked PR or a reviewed replacement.
