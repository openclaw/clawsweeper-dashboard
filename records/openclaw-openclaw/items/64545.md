## Summary

Keep open. Current main and the latest release still derive the Mattermost interaction signing secret from the bot token with a public constant, and the linked fix PR at https://github.com/openclaw/openclaw/pull/64546 is still open rather than merged.

## What This Changes

Replace Mattermost's bot-token-derived interaction HMAC secret with a durable per-account signing secret so button callback tokens are not forgeable from the bot token.

## Best Possible Solution

Keep this issue open until the linked PR or a reviewed successor lands a durable per-account Mattermost interaction signing secret independent of the bot token, with schema/types/docs/tests and migration guidance updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with a Mattermost bot token and known interaction context, compute the documented derived secret and token HMAC, and verifyInteractionToken will accept it on current main.

## Solution Assessment

Yes. Separating interaction signing from the Mattermost API bot token is the right security direction, but the persistence and migration details need maintainer security review before merge.
