## Summary

Keep open. Current main still derives the Mattermost interaction signing secret from the bot token with a public hardcoded HMAC key, the documented external-token flow still publishes that derivation, and open PR #64546 is an unmerged implementation candidate rather than a reason to close the tracker early.

## What This Changes

The issue asks Mattermost to replace a bot-token-derived, hardcoded-key interaction HMAC secret with a separate per-account signing secret.

## Best Possible Solution

Keep this issue open until PR #64546 or a reviewed successor lands a persistent per-account Mattermost interaction signing secret independent of the bot token, updates schema/types/docs/tests, and ships with clear migration guidance.

## Reproduction Assessment

Yes. Current main provides a high-confidence source-level reproduction: with a bot token and known signed context, the public derivation and token HMAC can be recomputed and will pass the same `verifyInteractionToken` path.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: use a persisted per-account random signing secret separate from the Mattermost API bot token, while retaining source-IP, post/channel, and action validation as defense in depth.
