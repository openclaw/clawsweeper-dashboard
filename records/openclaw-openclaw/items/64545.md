## Label Justifications

- `P1`: The issue tracks a security-sensitive Mattermost interaction signing bypass that remains present in current main and the latest release, with an open fix PR pending review.
- `impact:security`: The report concerns callback token forgery and authorization of Mattermost button interactions using a bot-token-derived signing secret.

## Summary

Keep open. Current main and latest shipped release still derive Mattermost interaction signing secrets from the bot token with a public HMAC constant, and the linked closing PR is open and unmerged.

## What This Changes

The issue reports forgeable Mattermost interaction `_token` values because the signing secret is derived from the Mattermost bot token with a hardcoded public HMAC key.

## Best Possible Solution

Land or replace the linked PR with a maintainer-reviewed durable per-account Mattermost interaction signing secret independent of the bot token, plus schema, docs, tests, migration guidance, and real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a Mattermost bot token and known interaction context, compute the documented derived secret and `_token`, and current main's verifier will accept it before the handler proceeds.

## Solution Assessment

Yes on direction, but it is not resolved on main. Separating interaction signing from the Mattermost API bot token is the maintainable fix; persistence, migration, rotation, and docs need maintainer-reviewed security design.
