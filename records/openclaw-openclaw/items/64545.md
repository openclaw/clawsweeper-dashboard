## Label Justifications

- `P1`: A forged Mattermost interaction token can cross a channel authorization boundary and trigger agent action flows when the bot-token-derived signing root is exposed.
- `impact:security`: The issue is about credential-derived HMAC signing and the authorization boundary for Mattermost interaction callbacks.

## Summary

Keep open: current main and the latest stable release still derive the Mattermost interaction signing secret from the bot token with a public HMAC key, and the same-repository candidate fix at https://github.com/openclaw/openclaw/pull/64546 is open, conflicting, and awaiting maintainer/security decisions.

## What This Changes

The issue asks OpenClaw to stop deriving Mattermost interaction callback tokens from a public HMAC key plus bot token and move to a durable per-account signing secret.

## Best Possible Solution

Land a refreshed version of https://github.com/openclaw/openclaw/pull/64546, or a narrow replacement, that removes bot-token-derived signing and provides durable setup, docs, SecretRef or doctor coverage, and old-forgery rejection tests.

## Reproduction Assessment

Yes. Source inspection of current main shows the interaction secret is deterministically derived from the bot token with a public HMAC key, then used to verify callback _token values; I did not run a live Mattermost server in this read-only review.

## Solution Assessment

Yes on direction, but not yet as a completed repository state. The maintainable fix is to stop using the bot token as the signing root and provide a durable per-account interaction secret with explicit setup, docs, and upgrade behavior.
