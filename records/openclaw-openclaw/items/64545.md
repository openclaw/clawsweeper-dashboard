## Summary

Keep open. Current main and v2026.5.28 still derive the Mattermost interaction signing secret from the bot token using a public HMAC constant, and the linked unmerged fix PR should be reviewed, landed, replaced, or explicitly rejected before this issue closes.

## What This Changes

The issue reports that Mattermost button interaction tokens can be forged because the signing secret is deterministically derived from the bot token using a public HMAC key.

## Best Possible Solution

Land a security-reviewed Mattermost change that replaces bot-token-derived interaction signing with a per-account secret or credential path, updates docs and tests, and handles existing buttons or external scripts deliberately.

## Reproduction Assessment

Yes for source reproduction: current code and docs expose the deterministic derivation from bot token plus a public HMAC key, and the handler verifies forged-compatible tokens through that same path. I did not run a live Mattermost exploit flow in this read-only review.

## Solution Assessment

Unclear until maintainer security review picks the upgrade shape: a per-account random secret is the right direction, but the current docs make bot-token-derived signing a documented external integration contract that needs an explicit migration or compatibility decision.
