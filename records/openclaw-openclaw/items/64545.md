## Label Justifications

- `P1`: A bundled Mattermost callback authorization primitive can be forged by someone with the bot token and enough interaction context to target a valid button.
- `impact:security`: The issue concerns callback authentication and integrity for Mattermost button requests that can reach agent event dispatch.

## Summary

Keep open. Current main and v2026.6.6 still derive the Mattermost interaction signing root from the bot token with a public HMAC key, and the linked fixing PR is open and unmerged with security/product review still required.

## What This Changes

The issue requests replacing Mattermost button interaction tokens signed from a bot-token-derived public HMAC root with a separate durable per-account signing secret.

## Best Possible Solution

Land the linked PR or a maintainer-approved replacement that uses a separate stable per-account Mattermost interaction secret, updates docs/tests, and proves fresh-install plus upgrade behavior.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 both derive the Mattermost interaction secret from the bot token with a public HMAC key, and current tests assert deterministic secret derivation.

## Solution Assessment

Yes on direction: a separate durable per-account interaction secret is the right security shape. The exact config or credential-store contract, docs, and upgrade path still need maintainer security/product review.
