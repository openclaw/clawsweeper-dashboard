## Summary

Keep open. Current main and v2026.5.12 still derive the Mattermost interaction signing secret from the bot token with a public constant, and the linked fix PR at https://github.com/openclaw/openclaw/pull/64546 is still open rather than merged.

## What This Changes

The issue reports that Mattermost interaction tokens are derived from the bot token using a public HMAC key, allowing token forgery if the bot token is known.

## Best Possible Solution

Land a reviewed security fix that separates Mattermost interaction signing from the bot token with a durable per-account secret, plus aligned schema/types/docs/tests and migration or rotation guidance.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with a Mattermost bot token and known interaction context, compute the documented derived secret and token HMAC, and current main's `verifyInteractionToken` will accept it before the handler proceeds.

## Solution Assessment

Yes on direction, unclear on final implementation. Separating interaction signing from the Mattermost API bot token is the right fix, but the exact durable secret persistence, migration, and docs contract should be resolved in the linked PR or a reviewed successor.
