## Label Justifications

- `P1`: A bundled Mattermost callback authorization primitive can be forged by someone with the bot token and enough interaction context to target a valid button.
- `impact:security`: The issue concerns callback authentication and integrity for Mattermost button requests that can reach agent system-event dispatch.

## Summary

Keep open. Current main and latest release still derive the Mattermost interaction signing root from the bot token with a public HMAC key, and the linked fix PR is open and unmerged; the remaining decision is maintainer security/product review of durable per-account secret storage and the external-script contract.

## What This Changes

The issue reports that Mattermost button callback tokens can be forged because the interaction signing secret is deterministically derived from the bot token using a public HMAC key.

## Best Possible Solution

Land the linked PR or a maintainer-approved replacement that uses a separate durable per-account Mattermost interaction secret, updates docs/tests, and proves fresh-install plus upgrade behavior before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 both derive the Mattermost interaction secret from the bot token with a public HMAC key, and current tests assert deterministic secret derivation.

## Solution Assessment

Yes on direction: a separate durable per-account interaction secret is the right security shape. The exact config or credential-store contract, docs, and upgrade path still need maintainer security/product review.
