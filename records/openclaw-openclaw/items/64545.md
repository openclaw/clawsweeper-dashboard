## Label Justifications

- `P1`: A forgeable Mattermost callback token can trigger action-gated channel events when an attacker has bot-token access.
- `impact:security`: The issue concerns the authorization boundary and signing-secret integrity for Mattermost interaction callbacks.

## Summary

Keep this open: current main and v2026.6.10 still derive the Mattermost interaction signing secret from the bot token with a public HMAC key, and the same-repository fix PR is open, conflicting, and still needs security/upgrade review.

## What This Changes

The issue reports that Mattermost interaction callback tokens are forgeable when the bot token is known because the signing secret is deterministically derived with a public HMAC key.

## Best Possible Solution

Land a maintainer-approved Mattermost security fix that stops bot-token-derived signing and uses a durable per-account interaction secret or explicit fail-closed setup with upgrade, docs, and old-forgery rejection proof.

## Reproduction Assessment

Yes, for source-level reproduction: current main still derives the Mattermost interaction signing secret from the bot token with a public HMAC key and verifies callback `_token` values against that derived secret. I did not run a live Mattermost exploit in this read-only review.

## Solution Assessment

The requested direction is right, but the active implementation path is not yet the best accepted fix. The durable-secret versus explicit fail-closed policy, SecretRef/setup/docs coverage, and upgrade behavior need maintainer security review before this issue can close.
