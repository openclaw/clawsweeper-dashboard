## Label Justifications

- `P1`: A bundled Mattermost channel callback can accept forged interaction tokens when the attacker has the bot token and target interaction context.
- `impact:security`: The issue concerns authentication and integrity for Mattermost button callback requests that can reach system-event dispatch.

## Summary

Keep open. Current main and latest release v2026.6.6 still derive the Mattermost interaction signing secret from the bot token using a public HMAC key, and the linked fix PR at https://github.com/openclaw/openclaw/pull/64546 is open and unmerged. The reporter's June 3 follow-up is consistent with current source and shipped code.

## What This Changes

The issue asks OpenClaw to stop deriving Mattermost button interaction signing secrets from the bot token with a public HMAC derivation key.

## Best Possible Solution

Land the linked PR or a maintainer-approved replacement that uses a separate per-account Mattermost interaction secret, updates docs/tests, and covers fresh-install plus upgrade behavior before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 both derive the Mattermost interaction secret from the bot token with a public HMAC key, and current tests assert deterministic secret derivation.

## Solution Assessment

Yes on direction: a separate per-account interaction secret is the right security shape. The exact persistence, config, docs, and upgrade behavior still needs maintainer security/product review.
