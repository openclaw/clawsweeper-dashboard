## Summary

Keep open: this is a useful scoped fix for the linked GitHub CLI HOME mismatch, with sufficient terminal proof, but the proposed helper does not fully match `gh`'s documented config-dir precedence on macOS and the user-visible doctor change lacks a required changelog entry.

## What This Changes

The PR adds a GitHub CLI config-dir mismatch detector, wires its hint into `openclaw doctor`, adds tests, and documents `GH_CONFIG_DIR` troubleshooting in the GitHub skill.

## Best Possible Solution

Merge a corrected version that follows `gh`'s documented lookup order on every platform, preserves the pure no-credential-copy diagnostic boundary, and includes a changelog entry.

## Reproduction Assessment

Yes: the linked Linux issue gives a concrete `env -u GH_CONFIG_DIR HOME=/root/.openclaw/agents/main/agent/codex-home/home gh auth status` failure while root auth exists, and current main's doctor flow only checks skill requirements. I did not run a live systemd gateway reproduction in this read-only review.

## Solution Assessment

No: the pure helper and doctor hint are the right narrow direction, but the implementation must honor `gh`'s documented `XDG_CONFIG_HOME` precedence on macOS and add the required changelog entry before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Honor XDG_CONFIG_HOME on macOS:** `src/agents/skills/gh-config-discovery.ts:78-83`
