## What This Changes

The PR adds a pure GitHub CLI config-dir mismatch detector, wires a GitHub CLI note into `openclaw doctor`, adds helper/doctor tests, and documents `GH_CONFIG_DIR` troubleshooting in the bundled GitHub skill.

## Best Possible Solution

## Reproduction Assessment

Yes: the linked Linux report gives a concrete `env -u GH_CONFIG_DIR HOME=/root/.openclaw/agents/main/agent/codex-home/home gh auth status` failure while `/root/.config/gh/hosts.yml` exists, and current main has no matching doctor discovery path. I did not run a live systemd gateway reproduction during this read-only review.

## Solution Assessment

No: the helper-and-doctor-note direction is the narrow maintainable fix, but the proposed implementation must honor `gh`'s documented `XDG_CONFIG_HOME` precedence across platform branches and add the required changelog entry before merge.

- **[P2] Honor XDG_CONFIG_HOME before platform defaults:** `src/agents/skills/gh-config-discovery.ts:67-83`
  - body: `gh help environment` lists `$XDG_CONFIG_HOME/gh` before both the Windows `%AppData%` fallback and `$HOME/.config/gh`, but this helper enters the `win32` and `darwin` branches before checking XDG.
