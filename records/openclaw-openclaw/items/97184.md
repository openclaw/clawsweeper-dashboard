## Summary

Keep open: current main and v2026.6.10 still resolve tool status emojis from static category/core-tool metadata, while plugin manifest and SDK tool metadata do not expose an emoji/icon contract. The request fits plugin API evolution, but needs maintainer product direction before implementation.

## What This Changes

Request to let plugin-owned tools declare custom emoji or icon metadata for Discord status reactions and similar channel status displays.

## Best Possible Solution

Define one generic plugin-owned tool display metadata contract, then wire supported status/progress displays to consume it without hardcoding plugin tool names in Discord.

## Reproduction Assessment

Yes at source level: current main lacks an emoji/icon field in plugin tool metadata and status reactions only receive the tool name. No live Discord repro was run because this is a missing public API surface rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public API shape. A generic plugin tool display metadata contract is likely better than a Discord-specific table or per-plugin hardcoding.
