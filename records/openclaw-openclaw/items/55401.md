## Summary

Keep open: current main and the latest release still expose only root-scoped plugin configuration, while production comments show the generic per-agent config/scoping API is still needed beyond the narrower memory-wiki tracker.

## What This Changes

Request per-agent plugin configuration overrides so multi-agent gateways can route plugin settings, credentials, memory stores, and plugin/tool exposure by agent.

## Best Possible Solution

Define a backward-compatible agent-scoped plugin effective-config contract covering precedence, validation, SecretRef-safe merge semantics, lifecycle/reload behavior, SDK/runtime exposure, hook/tool filtering, and docs while preserving root plugin defaults.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence in the current limitation: strict agent entries do not accept plugins, and plugin config resolution reads only the top-level plugins.entries map.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but maintainers need to choose the durable contract across schema, validation, SDK/runtime behavior, plugin lifecycle, tool filtering, and SecretRef handling.
