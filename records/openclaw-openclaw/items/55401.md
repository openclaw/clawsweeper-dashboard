## Summary

Keep open. Current main still has root-scoped plugin configuration only, strict agent entries reject the requested `agents.list[].plugins` override shape, and related reports either cover narrower plugin scoping or plugin-specific memory isolation rather than replacing this generic config/API design request.

## What This Changes

Add a documented per-agent plugin configuration override mechanism for multi-agent setups so individual agents can use different plugin config such as separate Mem0 credentials or user IDs.

## Best Possible Solution

Define a backward-compatible agent-scoped plugin effective-config contract with clear precedence, validation, SecretRef-safe merge semantics, lifecycle/reload behavior, and SDK/runtime exposure while preserving root defaults.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence in the current behavior: agent entries are strict without `plugins`, and plugin config resolution reads only the top-level `plugins.entries` map.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but maintainers need to choose whether the durable solution is generic agent-level config, plugin-owned per-agent maps, agent plugin allow/deny scoping, or a combination with explicit precedence and secret handling.
