## Summary

Keep open. Current main still exposes plugin config only at the root plugin-entry layer, while agent entries support per-agent tools, sandbox, TTS, skills, and runtime settings but not per-agent plugin config payload overrides; the closed #62700 scoping request does not cover separate plugin credentials or config objects per agent.

## What This Changes

Request per-agent plugin configuration overrides so multi-agent deployments can merge agent-specific plugin settings such as Mem0 API keys and user IDs over global plugin config.

## Best Possible Solution

Define a documented agent-scoped plugin override contract that validates and safely merges per-agent plugin config with root plugin entries, then exposes the effective request-time config through supported SDK/runtime seams.

## Reproduction Assessment

Not applicable as a feature request, but the source-level contract check is high confidence: current main has no agents.list[].plugins field and resolves plugin config only from root plugins.entries.<id>.config.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but the best fix needs maintainer design for precedence, secrets, reloads, hook behavior, and SDK compatibility.
