- `impact:session-state`: The request is about preventing memory and plugin-owned state from being shared or mis-associated across agents in one gateway.

## Summary

Keep open. Current main and the latest release still expose only root-scoped plugin config, strict agent entries still have no `plugins` override shape, and related items cover narrower memory-wiki, dreaming, or prompt-supplement scoping work rather than replacing this generic plugin configuration contract.

## What This Changes

Request per-agent plugin configuration overrides so multi-agent setups can use different plugin config, credentials, and memory/project routing per agent.

## Best Possible Solution

Define a documented, backward-compatible agent-scoped plugin effective-config contract with precedence, validation, SecretRef-safe merge semantics, lifecycle/reload behavior, and SDK/runtime exposure while preserving root defaults.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence in the current limitation: agent entries are strict without `plugins`, and plugin config resolution reads only the top-level `plugins.entries` map.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but maintainers need to choose the durable contract across config schema, SDK/runtime behavior, plugin lifecycle scoping, and secret handling.
