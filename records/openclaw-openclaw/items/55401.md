## Summary

Keep open: current main and the latest release still have only global plugin config, while the issue and follow-up comments describe a real multi-agent isolation and credential-routing gap. This is not safe cleanup closure because solving it needs a core/plugin-SDK product and security decision, not a narrow bug fix.

## What This Changes

Request per-agent plugin configuration overrides so each agent can override or merge global plugin settings in multi-agent setups.

## Best Possible Solution

Define a narrow maintainer-approved agent-scoped plugin contract with explicit precedence, lifecycle, SecretRef/security, docs, and tests; keep plugin-owned per-agent maps or path templates as the preferred path when they can satisfy a specific plugin without new core API.

## Reproduction Assessment

Yes, at source level. Current main's strict AgentEntrySchema and AgentConfig omit plugins, so attempts to set agents.list[].plugins remain rejected; I did not run a live gateway because this was a read-only source and release inspection.

## Solution Assessment

Unclear. The proposed deep-merge override is plausible, but the best solution needs maintainer decisions on precedence, lifecycle scoping, SecretRef handling, and when plugin-owned per-agent config is enough.
