## Summary

Keep open. Current main still accepts plugin config only under root `plugins.entries.<id>.config`, agent entries are strict, and plugin config resolution lacks an agent dimension; the related #62700 scoping request is not a substitute for per-agent config payload overrides.

## What This Changes

Request per-agent plugin configuration overrides so multi-agent deployments can deep-merge agent-specific plugin settings with global plugin config.

## Best Possible Solution

Define a core agent-scoped plugin override contract that validates and safely merges per-agent plugin config with global entries, exposes the effective request-time config through documented SDK/runtime seams, and updates docs/tests for plugin authors and operators.

## Reproduction Assessment

Not applicable as a feature request, but the source-level config-contract check is high confidence. Current main still rejects `agents.list[].plugins` through the strict agent schema and resolves plugin config only from root `plugins.entries.<id>.config`.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but the best solution needs maintainer design because it changes public config, plugin SDK/runtime semantics, and secret-bearing per-agent credentials for arbitrary plugins.
