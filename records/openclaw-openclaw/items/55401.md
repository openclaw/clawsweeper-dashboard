## Summary

Keep open. Current main still has only root-scoped plugin config, strict agent entries still reject an `agents.list[].plugins` override shape, and the related open memory issues or memory-wiki PR cover narrower plugin-specific isolation rather than the generic per-agent plugin config contract requested here.

## What This Changes

The issue requests agent-level plugin configuration overrides so different agents in one OpenClaw instance can use different plugin settings, secrets, accounts, and projects.

## Best Possible Solution

Define a documented, backward-compatible agent-scoped plugin effective-config contract, including precedence, validation, SecretRef-safe merge semantics, lifecycle/reload behavior, and SDK exposure while preserving root defaults.

## Reproduction Assessment

Not applicable as a feature request. The current-main source check is high confidence: strict agent entries reject `plugins`, and runtime plugin config resolution still reads only root `plugins.entries.<id>.config`.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but maintainers need to choose whether the generic contract should be agent-level config overrides, plugin-owned per-agent maps, per-agent plugin scoping, or some combination with clear precedence and secret handling.
