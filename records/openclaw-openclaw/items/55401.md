## Summary

Keep open. Current main still has only root-scoped plugin config, strict agent entries reject a `plugins` key, and the open memory-wiki thread plus PR cover narrower vault isolation rather than arbitrary per-agent plugin configuration overrides.

## What This Changes

Request per-agent plugin config overrides so different agents can use different plugin settings such as Mem0 API keys and user ids in one OpenClaw instance.

## Best Possible Solution

Define a documented core contract for agent-scoped plugin config overrides, including validation and safe merge semantics, then expose the effective config through stable plugin SDK/runtime seams while preserving current root defaults.

## Reproduction Assessment

Not applicable as a feature request, but the source-level contract check is high confidence: current main still rejects agent-level plugin config and resolves plugin config only from root `plugins.entries`.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but maintainers need to choose precedence, secret handling, validation, and whether generic per-agent overrides should coexist with narrower plugin-owned templating/scoping fixes.
