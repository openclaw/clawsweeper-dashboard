## Summary

Keep open. Current main still has root-scoped plugin config only, strict agent entries still reject an `agents.list[].plugins` shape, and the related open memory RFC or plugin-specific workarounds do not cover arbitrary per-agent plugin configuration overrides.

## What This Changes

Request per-agent plugin configuration overrides so different agents in one OpenClaw instance can use different plugin config such as API keys, user IDs, slots, or allowlists.

## Best Possible Solution

Define a documented, backward-compatible agent-scoped plugin config contract, including validation and safe merge semantics, then expose the effective config through stable plugin SDK/runtime seams while preserving root defaults.

## Reproduction Assessment

Not applicable as a feature request. The current-main source check is high confidence: strict agent entries reject `plugins`, and runtime plugin config resolution still reads only root `plugins.entries.<id>.config`.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but maintainers need to choose precedence, secret handling, validation, and how generic overrides should coexist with narrower plugin-owned scoping fixes.
