## Summary

Keep open: current main still has only root-scoped plugin configuration, and the issue now has additional production context showing both credential/account separation and per-agent tool/plugin scoping needs that require a maintainer API and security design rather than cleanup closure.

## What This Changes

The issue requests a generic per-agent plugin configuration override mechanism so multi-agent deployments can give each agent distinct plugin settings, credentials, and backend resources.

## Best Possible Solution

Define a backward-compatible agent-scoped plugin effective-config contract that covers precedence, validation, SecretRef-safe merge semantics, lifecycle/reload behavior, SDK/runtime exposure, and plugin filtering while preserving root plugin defaults.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence in the current limitation: strict agent entries do not accept plugins, and plugin config resolution reads only the top-level plugins.entries map.

## Solution Assessment

Unclear. The proposed deep-merge shape is plausible, but maintainers need to choose the durable contract across config schema, SDK/runtime behavior, plugin lifecycle scoping, tool filtering, and secret handling.
