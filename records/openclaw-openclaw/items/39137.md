## Best Possible Solution

Land the quoted single dependency specifier once required CI is green, preserving current main's newer `pytest -q -c skills/pyproject.toml skills` hook entry.

## Solution Assessment

Yes. Quoting the combined pytest version range as one YAML string matches pre-commit's string-array contract and is the narrowest maintainable fix for this hook.
