## What This Changes

The PR changes the `skills-python-tests` pre-commit hook to pass pytest as a single quoted `pytest>=8,<9` dependency string.

## Best Possible Solution

Land the quoted single-element pytest dependency once the required CI gate is green, while preserving current main's `pytest -q -c skills/pyproject.toml skills` hook entry.

## Reproduction Assessment

Yes, source-level with high confidence. Current main parses the hook dependency as `["pytest>=8", "<9"]`, and Python packaging rejects the standalone `<9` requirement that pre-commit would pass through to pip.

## Solution Assessment

Yes. Quoting the combined pytest version range as one YAML string matches pre-commit's array-of-strings contract and is the narrowest maintainable fix for this hook.
