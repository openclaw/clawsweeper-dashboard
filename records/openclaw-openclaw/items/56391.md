## Summary

Keep open. The linked blank-tool-name bug is real and current main still preserves unresolved blank names, but this PR's literal `_blank` sentinel introduces dispatch, stream-recovery, and replay-sanitization regressions.

## What This Changes

The PR changes embedded runner tool-call normalization to replace blank or whitespace-only tool names with `_blank` and treats that sentinel as incomplete in unknown-tool loop classification.

## Best Possible Solution

Fix blank-name loops with an internal non-dispatchable invalid-name path or direct loop-guard rewrite that preserves id-based recovery, keeps replay pruning intact, and adds focused regression tests plus the required user-facing changelog entry.

## Reproduction Assessment

Yes. The linked issue provides provider logs for the user-visible blank-name loop, and the PR regressions can be reproduced with source-level tests for blank-first/id-later streaming, `_blank` allowlist collision, and no-allowlist replay pruning.

## Solution Assessment

No. A normal `_blank` string is not the narrowest safe fix because OpenClaw allows arbitrary trimmed tool names and pi-agent-core dispatches by exact tool name.

  - confidence: 0.91
- **[P2] Prevent `_blank` from reaching dispatch:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: The sentinel is a normal literal tool name, while tool allowlists accept arbitrary trimmed tool and client-tool names. If `_blank` is registered, malformed blank model output can dispatch that real tool instead of failing closed.
  - confidence: 0.89
- **[P2] Keep blank replay calls pruned:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Replay normalization accepts any non-empty normalized name when no allowlist is available. With this fallback, persisted assistant tool calls with blank names can be rewritten to `_blank` and replayed instead of being dropped as malformed.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The patch turns malformed blank model output into a dispatchable literal name, creating unintended tool execution if `_blank` is registered.

Concerns:

- **[medium] Literal sentinel can dispatch a real tool:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Plugin and client tool names are accepted as trimmed strings and pi-agent-core dispatch is exact-name based. A registered `_blank` tool would be invoked for malformed blank tool calls under this patch, expanding the impact from a not-found error to unintended tool execution.
