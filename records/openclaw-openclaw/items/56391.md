## Summary

Keep open: the linked blank-tool-name loop is a real agent bug and current main does not already implement this PR's intended fix, but this patch should not merge because the literal sentinel creates dispatch, streaming recovery, and replay regressions.

## What This Changes

The PR maps blank embedded-runner tool-call names to a `_blank` sentinel, treats that sentinel as incomplete in loop classification, and updates stream-normalization tests.

## Best Possible Solution

Handle blank tool names as invalid, non-dispatchable malformed model output while preserving ID-based recovery and replay pruning, then cover streaming, allowlist collision, and replay with focused regressions.

## Reproduction Assessment

Yes. The linked issue gives concrete provider logs for the user-visible blank-name loop, and the PR regressions are source-reproducible through current stream mutation, allowlist collection, dispatch normalization, and replay sanitization.

## Solution Assessment

No. Replacing blanks with a normal `_blank` string is not the narrowest safe fix because it makes malformed model output look like a callable tool name and can block later ID-based recovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve ID recovery for blank streamed calls:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Writing `_blank` into the mutable stream block means a blank-name delta with a usable or later-arriving id is no longer blank on the next normalization pass, so `inferToolNameFromToolCallId(...)` is skipped and a recoverable call can become `Tool _blank not found`.
  - confidence: 0.91
- **[P2] Prevent `_blank` from reaching dispatch:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: The fallback is a normal tool name. Since run, client, and plugin allowlists accept arbitrary non-empty trimmed names, a registered `_blank` tool would be executed for malformed blank model output instead of failing closed.
  - confidence: 0.88
- **[P2] Keep blank replay calls pruned:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: `resolveReplayToolCallName` accepts any non-empty normalized name when no allowlist is present, so this changes persisted blank assistant tool calls from pruned malformed replay into `_blank` replay blocks that can fail provider validation again.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete tool-dispatch safety concern by representing malformed blank model output as a literal tool name.

Concerns:

- **[medium] Literal sentinel can dispatch a real tool:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Plugin and client tool names are accepted as trimmed strings and dispatch is exact-name based. A registered `_blank` tool would be invoked for malformed blank tool calls under this patch, expanding the impact from a not-found error to unintended tool execution.
  - confidence: 0.88
