## Summary

Keep this PR open, but do not merge it as-is: current main still lacks the requested blank-name loop fix, while the literal `_blank` fallback introduces concrete stream recovery, dispatch, replay, and security regressions.

## What This Changes

The PR changes embedded runner tool-call normalization to convert blank or whitespace-only tool names to `_blank`, treats that sentinel as incomplete for unknown-tool loop classification, and updates related stream-normalization tests.

## Best Possible Solution

Revise blank-name handling so malformed blank tool calls stop retrying without becoming a callable tool name; preserve id-based recovery, keep replay pruning, and add focused coverage for blank-first streaming, `_blank` collisions, and no-allowlist replay.

## Reproduction Assessment

Yes. The linked issue gives provider logs for the user-visible blank-name loop, and the PR regressions have deterministic source-level paths through current stream normalization, allowlist, dispatch, and replay sanitizer code.

## Solution Assessment

No. The bug fix is valid, but a normal `_blank` string is not the safest or narrowest solution because it mutates recoverable stream state and remains a legal registered tool name.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve ID recovery for blank streamed calls:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Writing `_blank` into the mutable stream block means a blank-name delta with a usable or later-arriving id is no longer blank on the next normalization pass, so `inferToolNameFromToolCallId(...)` is skipped and a recoverable call can become `Tool _blank not found`.
  - confidence: 0.91
- **[P2] Prevent `_blank` from reaching dispatch:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: The fallback is a normal tool name. Since run, client, and plugin allowlists accept arbitrary non-empty trimmed names, a registered `_blank` tool would be executed for malformed blank model output instead of failing closed.
  - confidence: 0.88
- **[P2] Keep blank replay calls pruned:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: `resolveReplayToolCallName` accepts any non-empty normalized name when no allowlist is present, so this changes persisted blank assistant tool calls from pruned malformed replay into `_blank` replay blocks that can fail provider validation again.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch changes malformed model output into a dispatchable literal name, creating an unintended tool-execution path if `_blank` is registered.

Concerns:

- **[medium] Literal sentinel can dispatch a real tool:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Plugin and client tool names are accepted as trimmed strings and dispatch is exact-name based. A registered `_blank` tool would be invoked for malformed blank tool calls under this patch, expanding the impact from a not-found error to unintended tool execution.
