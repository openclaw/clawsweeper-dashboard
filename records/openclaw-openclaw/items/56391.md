## Summary

Keep open. The linked blank-tool-name loop is still a valid agents bug, but this PR's literal `_blank` fallback is not safe to merge because it mutates malformed streamed calls into a normal tool name, can collide with real allowlisted tools, and changes replay sanitization semantics.

## What This Changes

The PR changes embedded-runner tool-call normalization to replace blank or whitespace-only tool names with `_blank`, treats that sentinel as incomplete for loop classification, and updates related stream-normalization tests.

## Best Possible Solution

Handle blank tool names as invalid malformed model output that cannot dispatch, while preserving ID-based recovery and keeping malformed replay blocks pruned, then add focused regression coverage for those paths.

## Reproduction Assessment

Yes. The PR regressions have a high-confidence source reproduction through the sentinel fallback, in-place stream mutation, OpenAI-compatible `currentBlock` reuse, allowlist collection, and replay sanitizer paths.

## Solution Assessment

No. A literal `_blank` sentinel is not the best fix because it is treated like a normal tool name; a safer approach should keep the invalid state separate from dispatch while still giving the model a clear failure signal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve ID recovery for blank streamed calls:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Writing `_blank` into the mutable stream block means a blank-name delta with a usable or later-arriving id is no longer blank on the next normalization pass, so `inferToolNameFromToolCallId(...)` is skipped and a recoverable call can become `Tool _blank not found`.
  - confidence: 0.91
- **[P2] Keep blank sentinels out of dispatch:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: The fallback is a normal tool name. Since run, client, and plugin allowlists accept arbitrary non-empty trimmed names, a registered `_blank` tool would be executed for malformed blank model output instead of failing closed.
  - confidence: 0.88
- **[P2] Keep blank replay calls pruned:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: `resolveReplayToolCallName` accepts any non-empty normalized name when no allowlist is present, so this changes persisted blank assistant tool calls from pruned malformed replay into `_blank` replay blocks that can fail provider validation again.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces a dispatch-safety concern because `_blank` is a normal allowlisted tool name, not a reserved invalid state.

Concerns:

- **[medium] Literal sentinel can dispatch a real tool:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Plugin and client tool names are accepted as trimmed strings and dispatch is exact-name based. A registered `_blank` tool would be invoked for malformed blank tool calls under this patch, expanding the impact from a not-found error to unintended tool execution.
  - confidence: 0.88
