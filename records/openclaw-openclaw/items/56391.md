## Summary

Keep open: current main still preserves blank tool names when ID recovery fails, so the linked user-visible loop is not implemented away, but this PR's `_blank` sentinel approach introduces dispatch, streaming, and replay regressions that should block merge.

## What This Changes

This PR changes embedded-runner tool-call normalization to convert blank tool names to `_blank`, treats `_blank` as incomplete for loop classification, and updates stream-normalization tests.

## Best Possible Solution

Handle blank tool names as malformed, non-dispatchable model output while preserving ID-based recovery, keeping blank replay calls pruned, and covering the stream/replay/allowlist cases with focused tests.

## Reproduction Assessment

Yes. The linked issue has concrete NVIDIA/Kimi logs, and source inspection shows current main keeps blank names when ID recovery fails; the PR regressions are source-reproducible through stream mutation, replay sanitization, and allowlist collection.

## Solution Assessment

No. Replacing blanks with a normal `_blank` string is not the narrowest safe fix because that value can be persisted, retried, replayed, and dispatched like a real tool name.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve ID recovery for blank streamed calls:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Writing `_blank` into the mutable stream block means a blank-name delta with a usable or later-arriving id is no longer blank on the next normalization pass, so `inferToolNameFromToolCallId(...)` is skipped and a recoverable call can become `Tool _blank not found`.
  - confidence: 0.91
- **[P2] Prevent `_blank` from reaching dispatch:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: The fallback is a normal tool name. Since local, client, and plugin allowlists accept arbitrary non-empty trimmed names, a registered `_blank` tool would be executed for malformed blank model output instead of failing closed.
  - confidence: 0.88
- **[P2] Keep blank replay calls pruned:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: `resolveReplayToolCallName` accepts any non-empty normalized name when no allowlist is present, so this changes persisted blank assistant tool calls from pruned malformed replay into `_blank` replay blocks that can fail provider validation again.
  - confidence: 0.82
