## What This Changes

The PR adds a full-CLI shutdown step that cancels active deferred context-engine turn-maintenance runs, clears their lanes, and threads cancellation into maintenance calls.

## Best Possible Solution

Keep the scoped CLI teardown idea, but preserve existing `maintain()` params or add an explicit compatibility path, document any intentional SDK cancellation contract, and require a real terminal proof showing a short-lived `agent --local` run emits its result and exits.

## Reproduction Assessment

## Solution Assessment

No, not yet. The CLI teardown boundary is reasonable, but passing a new top-level `abortSignal` through the public `maintain()` hook is not the safest compatibility shape; keep it inside `runtimeContext` or add explicit compat/docs before merge.
