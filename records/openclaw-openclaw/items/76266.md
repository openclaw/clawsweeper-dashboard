## Summary

Close as cannot reproduce: supported Node runtimes emit FileHandle-GC warnings rather than an uncaughtException, and the PR suppresses an unverified message while missing the active packaged CLI entry path on current main.

## What This Changes

The PR adds a FileHandle-GC message predicate to `isBenignUncaughtExceptionError` and calls it from `src/index.ts` before exiting on uncaught exceptions.

## Best Possible Solution

Leave this closed unless a supported Node version produces a real uncaughtException; then fix the leaked FileHandle source or add a tested suppression for the actual emitted error shape in the active CLI handler.

## Reproduction Assessment

Yes. A high-confidence negative reproduction path exists: forcing GC of an unclosed FileHandle on supported Node 22.22.2 and 24.14.1 produced warning events and `sawUncaught=false`, so the reported gateway-fatal path did not reproduce.

## Solution Assessment

No. The PR suppresses an unverified exact message and only patches `src/index.ts`; the maintainable fix would be to close the originating FileHandle leak or handle a reproduced supported-runtime exception shape in the active gateway entry path.
