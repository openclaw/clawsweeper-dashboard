## Summary

Close as cannot reproduce. Supported Node runtimes emit DEP0137 warnings, not an uncaughtException, for a garbage-collected unclosed FileHandle, so the proposed suppression targets an unverified crash shape rather than a reproduced gateway failure.

## What This Changes

The PR adds a FileHandle-GC message predicate to the gateway uncaughtException handler so matching errors log a warning instead of exiting.

## Best Possible Solution

Leave FileHandle GC out of the benign uncaught-exception allowlist unless a supported Node version produces an actual uncaughtException; fix any real leaked FileHandle at its source or add a tested predicate for the reproduced shape.

## Reproduction Assessment

Yes, there is a high-confidence negative reproduction path: forcing GC of an unclosed FileHandle on Node 22.22.2 and 24.14.1 emitted warnings and no uncaughtException. The reported fatal gateway path does not reproduce on supported runtimes checked against current main.

## Solution Assessment

No. Suppressing an exact FileHandle message is not the best fix without a reproduced uncaughtException; a narrower maintainable fix would either close the leaking FileHandle or handle the actual emitted error shape with a test.
