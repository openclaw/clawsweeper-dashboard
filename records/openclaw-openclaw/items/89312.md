## Summary

Keep open: this is a protected maintainer draft and the branch is currently conflicting, and the patch is not yet the best descriptor-hardening fix because it snapshots client tool names but still rereads raw descriptors for admission and registration.

## What This Changes

The PR adds guarded client-tool descriptor reads, shared name collection, and regressions so unreadable descriptors are skipped while readable client tools remain available.

## Best Possible Solution

Use the snapshot-once shape from https://github.com/openclaw/openclaw/pull/89699 or update this branch to pass one descriptor snapshot through reservation, Tool Search, conflicts, allowlists, and registration before undrafting or merging.

## Reproduction Assessment

Yes. Source inspection shows current main directly reads `tool.function` and `tool.function.name` across conflict, reservation, allowlist, Tool Search, and conversion paths, so throwing or stateful accessors reproduce the crash/divergence path.

## Solution Assessment

No. This PR is a useful partial guard, but the best fix is to snapshot each client descriptor once per attempt and reuse that snapshot across every admission and registration consumer.
