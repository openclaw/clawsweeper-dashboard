## Summary

Keep open for maintainer handling: this protected maintainer draft targets a real source-reproducible agent bug, but the branch is currently conflicting with main and the patch still snapshots the same client descriptor multiple times. The overlapping https://github.com/openclaw/openclaw/pull/89699 appears to carry the stronger one-snapshot shape, so this branch should not be cleanup-closed automatically.

## What This Changes

The PR adds guarded client tool descriptor reads and regression tests so malformed client tools are skipped or degraded while healthy sibling tools remain available to agent runtime consumers.

## Best Possible Solution

Use a single per-attempt client descriptor snapshot shared by reservation, Tool Search, allowlists, conflict checks, and conversion, then let maintainers choose whether this branch should be rebased to that shape or superseded by https://github.com/openclaw/openclaw/pull/89699.

## Reproduction Assessment

Yes. Source inspection of current main shows direct `tool.function` and `tool.function.name` reads in bundle reservation, Tool Search allowlists, conflict checks, and conversion, so throwing or stateful descriptor accessors can reproduce the crash/divergence path without a new product mode.

## Solution Assessment

No. This is a plausible guard, but it is not the best fix because it exposes a names-only collector and later rereads descriptors during conversion; the maintainable shape is to snapshot the full descriptor once per attempt and pass that snapshot through every consumer.
