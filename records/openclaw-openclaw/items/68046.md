## Label Justifications

## Summary

Keep open. The linked gateway bug is valid and current main/latest release still strip non-text MCP content fields, but this PR still needs inspectable real HTTP MCP proof and stricter SDK-shape validation before merge.

## What This Changes

The PR changes the MCP HTTP tools/call content normalizer to pass through typed text, image, audio, resource, and resource_link blocks and adds helper-level tests for those variants plus fallback cases.

## Best Possible Solution

Preserve only SDK-valid MCP content blocks in the gateway normalizer, fall back to text for malformed or unknown entries, add HTTP tools/call regression coverage, and require redacted real-call proof before merge.

## Reproduction Assessment

Yes, at source level: a loopback tool returning an image block reaches HTTP MCP tools/call and current main drops data and mimeType in normalizeToolCallContent. I did not run a live request in this read-only review.

## Solution Assessment

No. The PR fixes the right boundary for valid multimodal blocks, but the best implementation should validate the SDK content-block shapes before pass-through and prove the HTTP endpoint path.
