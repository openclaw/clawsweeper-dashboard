## Summary

Keep open. Current `main` still matches the report: a fresh `bootstrap` leaves `repairStatus` as `repaired`, but `repairLaunchAgentBootstrap` immediately runs `launchctl kickstart -k`, and the existing unit test still asserts that sequence.

## What This Changes

The issue requests that macOS LaunchAgent bootstrap repair skip `kickstart -k` after a successful fresh bootstrap while preserving the bounce for already-loaded services.

## Best Possible Solution

Gate `kickstart -k` to the already-loaded bootstrap branch, return `repaired` immediately after a fresh bootstrap, and cover both branches with regression tests plus a changelog entry.

## Reproduction Assessment

Yes. The high-confidence source reproduction is the current mocked fresh-bootstrap path in `src/daemon/launchd.test.ts`, which returns `repaired` and records a later `kickstart -k`; the reporter also supplied live macOS logs for the same sequence.

## Solution Assessment

Yes. The requested fix is the narrowest maintainable path because it preserves the existing bounce when bootstrap reports an already-loaded job while avoiding a needless SIGTERM of a freshly loaded RunAtLoad agent.
