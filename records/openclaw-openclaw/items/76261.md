## Summary

Keep open: current `main` still unconditionally runs `launchctl kickstart -k` after a successful LaunchAgent bootstrap repair, and nearby install-path comments/tests support that this is the wrong control flow for a freshly bootstrapped service.

## What This Changes

The issue requests changing macOS LaunchAgent bootstrap repair so a freshly bootstrapped gateway is not immediately kickstarted with `-k`, while preserving the bounce for an already-loaded service.

## Best Possible Solution

Gate `kickstart -k` on the `already-loaded` repair status, return success directly after a clean bootstrap, and add focused launchd tests for both branches.

## Reproduction Assessment

Yes. The current source and unit test expectations reproduce the core control-flow bug without a live macOS host: a successful bootstrap repair is still followed by `kickstart -k`.

## Solution Assessment

Yes. The status-gated kickstart is the narrowest maintainable solution because it preserves the already-loaded bounce while avoiding a needless kill of a newly spawned process.

## Repair Work Prompt

Fix the macOS LaunchAgent repair race reported in openclaw/openclaw#76261. In `src/daemon/launchd.ts`, update `repairLaunchAgentBootstrap` so `launchctl kickstart -k` is only invoked when `bootstrap` reports the service is already loaded and `repairStatus` is `"already-loaded"`; when bootstrap succeeds and a fresh process is spawned, return `{ ok: true, status: "repaired" }` without kickstarting. Preserve existing bootstrap error handling, `enable`, and already-loaded bounce semantics. Add/update focused tests in `src/daemon/launchd.test.ts` so the bootstrap-success path asserts no kickstart call and the already-loaded paths still assert `kickstart -k`. Check whether `src/commands/doctor-gateway-daemon-flow.test.ts` or `src/cli/daemon-cli/launchd-recovery.test.ts` need expectation updates. Add a user-facing `CHANGELOG.md` fix entry unless maintainers decide this is internal-only. Do not change runtime-deps lock semantics or broaden LaunchAgent restart behavior beyond this repair path.
