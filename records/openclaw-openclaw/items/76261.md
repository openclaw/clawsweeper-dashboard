## Summary

Keep open. Current `main` still matches the report: a fresh `bootstrap` leaves `repairStatus` as `repaired` but the helper immediately runs `launchctl kickstart -k`, and the unit test still asserts that sequence.

## What This Changes

The issue requests avoiding `launchctl kickstart -k` after a successful fresh macOS LaunchAgent bootstrap while preserving the bounce for already-loaded services.

## Best Possible Solution

Gate `kickstart -k` to the `already-loaded` branch, leave fresh bootstrap as a successful `repaired` result, and cover both branches with regression tests.

## Reproduction Assessment

Yes. The high-confidence source reproduction is the current mocked fresh-bootstrap test path in `src/daemon/launchd.test.ts`, which returns `repaired` and records a later `kickstart -k`; the reporter also supplied live macOS logs showing the same sequence.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable change because it preserves the existing bounce when bootstrap reports an already-loaded job while avoiding a needless SIGTERM of a freshly loaded RunAtLoad agent.
