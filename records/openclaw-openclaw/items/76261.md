## Summary

Keep open. Current main still unconditionally runs `launchctl kickstart -k` after a successful `bootstrap` in `repairLaunchAgentBootstrap`, and the existing unit test asserts that fresh-bootstrap sequence; the reporter's macOS logs match the source path.

## What This Changes

Skip macOS LaunchAgent `kickstart -k` after a fresh bootstrap repair while preserving the bounce for already-loaded jobs.

## Best Possible Solution

Gate `kickstart -k` to the already-loaded bootstrap branch, return `repaired` immediately after a fresh bootstrap, and cover both branches with regression tests plus a changelog entry.

## Reproduction Assessment

Yes. The current mocked fresh-bootstrap path returns `repaired` while recording a later `kickstart -k`, and the reporter supplied live macOS logs for the same sequence.

## Solution Assessment

Yes. The requested branch gate is the narrowest maintainable fix because it preserves the existing bounce for already-loaded jobs while avoiding a needless SIGTERM of a freshly loaded RunAtLoad agent.
