## What This Changes

The branch wraps the legacy `/tools/invoke` action-schema inspection in a non-throwing helper and adds HTTP regression coverage for a throwing schema `properties` getter.

## Best Possible Solution

Land the focused shared `/tools/invoke` schema guard and regression after maintainer review and draft resolution.

## Reproduction Assessment

Yes, source-level. Current main reads `schemaObj.properties` during the optional legacy action merge before hook/tool execution, so an unreadable getter can fail the invoke path; I did not execute tests in this read-only review.

## Solution Assessment

Yes. Guarding only the optional legacy action schema probe in the shared helper is the narrowest maintainable fix because HTTP and RPC share that helper, and unreadable schemas should not receive a synthetic `action`.
