## Reproduction Assessment

Yes, source-level: current main reads `schemaObj.properties` during the optional action probe before hook/tool execution, so a throwing accessor can turn an otherwise allowed invoke into a tool error. I did not run tests because this review is read-only.

## Solution Assessment

Yes. Guarding only the optional schema-inspection helper is narrower than changing tool schema contracts or wrapping the entire invoke path differently, and it matches the existing best-effort treatment of unreadable Gateway tool schema fields.
