## What This Changes

The PR wraps the Gateway `/tools/invoke` legacy top-level `action` schema probe in a best-effort helper and adds an HTTP regression test for a throwing `properties` getter.

## Best Possible Solution

Land the focused guard and regression after draft resolution and maintainer review, keeping the legacy HTTP `action` probe best-effort and non-fatal when schema inspection is unreadable.

## Reproduction Assessment

Yes, at source level: current main reads `schemaObj.properties` during `/tools/invoke` argument preparation, so a throwing getter can fail the invocation before the hook or tool sees unchanged args. I did not execute tests because this review is read-only.

## Solution Assessment

Yes. Guarding only the optional schema-inspection helper is the narrowest maintainable fix; changing the tool contract or swallowing all invoke errors would be broader and less precise.
