## Summary

Keep open. The PR addresses a real missing path on current main, but current head still mirrors only text payloads, so media-only BlueBubbles cron deliveries can be sent without being written to the target transcript.

## What This Changes

This PR adds BlueBubbles-only isolated cron delivery mirroring through outbound route resolution, a new focused regression test file, and an active-version changelog entry.

## Best Possible Solution

Keep the BlueBubbles-only route-aware mirror, but build mirror text and media from the shared outbound payload plan projection, pass mediaUrls into the DeliveryMirror, add a media-only regression, and then land after focused test plus changed-gate validation.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron delivery with a payload like `{ mediaUrl: "https://example.test/photo.jpg" }` passes delivery filtering, but the PR helper returns undefined because it only joins payload.text, so no target transcript mirror is sent.

## Solution Assessment

No. The route-aware BlueBubbles seam is the right narrow approach, but duplicating text-only mirror projection is not the best fix when the existing outbound payload mirror projection already preserves text and media consistently.
