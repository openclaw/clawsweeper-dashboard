## What This Changes

The PR adds BlueBubbles-only transcript mirroring for isolated cron deliveries, route-shape regression tests, and active-version changelog entries.

## Best Possible Solution

Land the route-aware BlueBubbles cron mirror after changing the mirror payload projection to use the shared outbound plan/projection semantics for text and media, add a media-only regression case, and keep a single active changelog entry for the user-facing fix.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron payload such as `{ mediaUrl: "https://example.test/photo.jpg" }` passes current delivery-content filtering, but the PR helper returns undefined because it only projects `payload.text`, so the delivery can send without a target transcript mirror.

## Solution Assessment

No. The BlueBubbles-only route seam is the right narrow direction, but hand-rolling a text-only mirror projection is not the best fix when current outbound code already has a shared projection that preserves media URLs and idempotency semantics.
