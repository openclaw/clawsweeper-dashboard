## Summary

Keep open. The reported BlueBubbles cron transcript gap appears valid, but this patch is not merge-ready because it can mirror into a different session key than later BlueBubbles inbound replies use, and it is missing the required changelog entry for a user-facing bug fix.

## What This Changes

The branch adds BlueBubbles-only transcript mirror construction to isolated cron direct delivery and adds a new regression test file for group, DM, fallback, and multi-payload cases.

## Best Possible Solution

Keep this PR open, repair the BlueBubbles mirror route so the appended assistant turn lands in the same transcript key used by later inbound BlueBubbles replies, add the required changelog entry, then run focused cron and BlueBubbles routing tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: for a BlueBubbles `chat_guid:` target, the PR routes the mirror with the raw outbound `to` string, while current inbound BlueBubbles replies route with normalized chat GUID or sender-handle peer ids, producing different session keys.

## Solution Assessment

No as submitted. The direction is reasonable, but the maintainable fix should reuse the plugin-owned BlueBubbles target/session routing contract or an equivalent shared normalization helper instead of duplicating partial parsing in core.
