## Summary

Keep open. The PR is aimed at a real cron transcript gap, and its route-aware BlueBubbles direction is sound, but the current head still builds the mirror from text only, so media-only BlueBubbles cron deliveries can be sent without being mirrored into the target transcript.

## What This Changes

The branch adds BlueBubbles-only isolated cron delivery mirroring through outbound route resolution, focused text/DM/group regression tests, and a changelog entry.

## Best Possible Solution

Keep the BlueBubbles outbound-route seam, but build the mirror content from the shared outbound payload plan/projection so text and media are both normalized into the target transcript, then add a media-only regression in the new cron mirror test file.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron delivery with `deliveryPayloads: [{ mediaUrl: "https://example.test/photo.jpg" }]` passes current content filtering, but the PR helper returns `undefined` before adding `mirror`, so the delivered media is absent from the target transcript.

## Solution Assessment

No. The route-aware mirror is the right narrow shape, but this implementation duplicates text-only payload handling instead of reusing the existing text/media mirror projection already used by outbound sends.
