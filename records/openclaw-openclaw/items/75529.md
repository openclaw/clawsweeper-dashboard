## Summary

Keep open. The PR targets a real current-main gap in BlueBubbles cron transcript mirroring, but the proposed helper only mirrors text payloads, so media-only BlueBubbles cron deliveries would still be delivered without any target transcript entry. That is a narrow repair on this PR branch before normal maintainer review and exact-head CI gating.

## What This Changes

This PR adds a BlueBubbles-only cron delivery mirror path through the outbound session route, focused group/DM regression coverage, and a changelog entry.

## Best Possible Solution

Keep the route-backed BlueBubbles approach, but build the mirror from the same text/media projection used by outbound mirrors so text, media-only, and silent-text-plus-media cron deliveries all append to the normalized BlueBubbles target transcript. Then land after the focused regression test and exact-head checks are green or explicitly cleared.

## Reproduction Assessment

Yes. Current main statically reproduces the original gap because direct cron delivery omits params.mirror while transcript append only runs when mirror is present; the PR-specific bug reproduces with a BlueBubbles cron payload that has mediaUrl/mediaUrls and no text.

## Solution Assessment

No. Reusing the BlueBubbles outbound session route is the right maintainable direction, but the submitted implementation is incomplete because it ignores the existing mediaUrls mirror path and still skips media-only deliveries.

## Review Findings
