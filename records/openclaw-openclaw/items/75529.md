## Summary

Keep open. The PR addresses a real current-main BlueBubbles cron transcript gap and uses the right outbound-route seam, but the latest head still builds the mirror from text only, so media-only BlueBubbles cron deliveries would still be sent without being appended to the target transcript.

## What This Changes

The PR adds BlueBubbles-only isolated cron delivery mirroring through the outbound session route, focused regression tests, and a changelog entry.

## Best Possible Solution

Keep the plugin-aware BlueBubbles outbound-route approach, but build the mirror from the existing outbound payload plan/projection so text, mediaUrl/mediaUrls, media-only, and silent-text-plus-media cron deliveries all append to the normalized target transcript.

## Reproduction Assessment

Yes. Current main statically reproduces the original gap because direct cron delivery never passes a mirror, and the PR-specific regression is reproduced by a BlueBubbles cron payload that has mediaUrl or mediaUrls but no text.

## Solution Assessment

No. The route seam is the narrow maintainable direction, but the submitted implementation is incomplete because it uses ad hoc text-only mirror construction instead of the existing text/media payload projection.
