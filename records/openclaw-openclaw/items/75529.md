## Summary

Keep open. Current main still lacks the BlueBubbles isolated-cron transcript mirror, so the PR is not obsolete, but the PR head still has a merge-blocking media regression and a duplicate active changelog entry.

## What This Changes

The PR adds BlueBubbles-only isolated cron delivery transcript mirroring through outbound session route resolution, focused route-shape regression tests, and active-version changelog entries.

## Best Possible Solution

Land the BlueBubbles-only cron mirror after it uses the existing media-aware outbound/mirror projection semantics, preserves DM/group route resolution, adds a media-only regression case, and keeps one active changelog bullet for the fix.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron payload such as `{ mediaUrl: "https://example.test/photo.jpg" }` is sendable on current main, but the PR helper returns `undefined` for the mirror because it only reads `payload.text`; a text-plus-media payload would mirror only the text.

## Solution Assessment

No. The route-aware BlueBubbles-only seam is the right narrow direction, but the PR should not hand-roll a text-only mirror projection when the existing outbound mirror path already supports media URLs and media-only transcript text.
