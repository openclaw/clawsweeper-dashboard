## Summary

Keep open. Current main still lacks BlueBubbles cron transcript mirroring, but this PR head remains incorrect because the new mirror is text-only, can stamp transcript entries during partial best-effort delivery, and duplicates the changelog entry.

## What This Changes

The PR adds BlueBubbles-only isolated cron delivery transcript mirroring via outbound route resolution, a new regression test file, and active-version changelog entries.

## Best Possible Solution

Land route-aware BlueBubbles cron mirroring using the shared outbound payload projection and delivery-success semantics, preserving media and idempotent route context, with one focused regression suite and one changelog bullet.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron payload such as { mediaUrl: "https://example.test/photo.jpg" } passes current delivery-content filtering, but the PR helper returns undefined because it only projects payload.text; partial bestEffort delivery can also be reasoned from the existing deliverOutboundPayloads mirror append point.

## Solution Assessment

No as-is. The BlueBubbles-only outbound route seam is the right narrow direction, but the mirror should reuse shared payload projection and avoid appending a full mirror before cron knows bestEffort delivery fully succeeded.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Mirror media-bearing BlueBubbles cron payloads:** `src/cron/isolated-agent/delivery-dispatch.ts:138-144`
  - body: The helper derives the mirror only from `payload.text` and returns before route resolution when that text is empty. Current delivery filtering still allows `mediaUrl` and `mediaUrls`, so media-only BlueBubbles cron deliveries can send successfully without any target transcript mirror, and mixed text/media deliveries lose their media in the mirror. Reuse the shared outbound payload mirror projection and add a media-only regression.
  - confidence: 0.93
- **[P2] Gate mirrors on complete bestEffort delivery:** `src/cron/isolated-agent/delivery-dispatch.ts:726`
  - body: Passing `bluebubblesMirror` directly into `deliverOutboundPayloads` lets the outbound helper append the mirror whenever any payload succeeds. For `deliveryBestEffort`, cron still treats partial failure as undelivered and retries with the same idempotency key, so a partial send can stamp an incomplete or over-broad transcript entry that the later full retry cannot correct.
  - confidence: 0.88
- **[P3] Remove the duplicate changelog entry:** `CHANGELOG.md:153-154`
  - body: The active changelog now contains two bullets for the same BlueBubbles/cron fix, one without the PR reference and one with it. Keep one entry so release notes do not duplicate this user-facing change.
  - confidence: 0.97
