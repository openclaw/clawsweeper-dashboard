## Summary

Keep this PR open, but not merge-ready. The BlueBubbles route-based mirror is the right narrow direction for the reported cron transcript gap, but the latest head still builds the mirror from text only, so media-only and silent-text-plus-media BlueBubbles cron deliveries would still send without appending anything to the target transcript.

## What This Changes

This PR adds BlueBubbles-only isolated cron delivery transcript mirroring through the outbound session route, focused regression tests, and a changelog entry.

## Best Possible Solution

Keep the plugin-aware BlueBubbles outbound route approach, but build the mirror from the existing outbound payload projection so text, mediaUrl/mediaUrls, media-only, and silent-text-plus-media cron deliveries all append to the normalized target transcript before merge.

## Reproduction Assessment

Yes. Current main statically reproduces the original gap because direct cron delivery never passes mirror, and the PR-specific bug reproduces with a BlueBubbles cron payload that has mediaUrl or mediaUrls but no text.

## Solution Assessment

No. The route seam is the narrow maintainable solution, but the submitted implementation is incomplete because it uses ad hoc text-only mirror construction instead of the existing text/media payload projection.
