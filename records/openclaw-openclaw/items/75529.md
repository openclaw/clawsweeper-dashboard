## Summary

Keep open. The PR is aimed at a real current-main gap and uses the right BlueBubbles outbound-route seam, but the submitted helper still builds the transcript mirror from text only, so media-only BlueBubbles cron deliveries can be sent without being mirrored into the target transcript.

## What This Changes

The PR adds BlueBubbles-only isolated cron delivery mirroring through outbound session route resolution, focused regression tests, and a changelog entry.

## Best Possible Solution

Keep the BlueBubbles outbound-route seam, but derive the mirror from the existing outbound payload plan/projection so text, `mediaUrl`, `mediaUrls`, media-only payloads, and silent text with media all append normalized target transcript content.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron payload with `mediaUrl` or `mediaUrls` and no text passes current delivery filtering, while the PR helper returns `undefined` before setting `mirror`, so the delivered media is not appended to the target transcript.

## Solution Assessment

No. The route-aware mirror is the narrow maintainable direction, but the submitted implementation is incomplete because it duplicates text-only payload handling instead of using the shared text/media mirror projection already used by outbound send surfaces.
