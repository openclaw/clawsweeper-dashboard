## Summary

Keep this PR open, but it still needs changes before merge. The route-aware BlueBubbles cron mirror is the right narrow boundary, but the current patch misses media-only deliveries, can mirror failed best-effort payloads, and duplicates the changelog entry.

## What This Changes

This PR adds a BlueBubbles-only isolated cron transcript mirror through outbound session routing, a focused regression test file, and active changelog entries.

## Best Possible Solution

Land the BlueBubbles-only cron target-transcript mirror after deriving mirror text and media from the shared outbound payload projection, aligning transcript append with cron all-success delivery semantics, and keeping one changelog entry.

## Reproduction Assessment

Yes. Source inspection against PR head gives a high-confidence path: media-only BlueBubbles cron payloads pass current content filtering but the PR helper returns no mirror, and best-effort partial success can append a full-batch mirror before cron decides the batch was delivered.

## Solution Assessment

No. The BlueBubbles route seam is the right narrow boundary, but the patch should reuse the shared outbound payload projection and mirror only content that matches cron’s delivered semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve media in cron transcript mirrors:** `src/cron/isolated-agent/delivery-dispatch.ts:138-144`
  - body: The helper builds the mirror only from `payload.text` and returns before route resolution when that text is empty. Current delivery filtering accepts `mediaUrl` and `mediaUrls`, so media-only BlueBubbles cron deliveries can still send without any target transcript mirror, and mixed text+media deliveries lose media context. Reuse the shared outbound payload mirror projection and add a media-only regression.
  - confidence: 0.94
- **[P2] Do not mirror failed best-effort payloads:** `src/cron/isolated-agent/delivery-dispatch.ts:726`
  - body: This passes one full-batch mirror into `deliverOutboundPayloads` even when cron is running best-effort delivery. That helper appends the mirror whenever any result succeeds, while cron intentionally treats partial failures as not delivered, so a multi-payload BlueBubbles cron run can record failed payload text/media in the target transcript. Gate the mirror on all-success semantics or mirror only the successfully delivered projection.
  - confidence: 0.88
- **[P3] Remove the duplicate changelog entry:** `CHANGELOG.md:153-154`
  - body: The active changelog now has two adjacent bullets for the same BlueBubbles cron fix, one with this PR reference and one without. Keep a single user-facing entry so release notes do not duplicate the change.
  - confidence: 0.98
