## Summary

Keep this PR open with changes requested. The BlueBubbles route-backed direction is right, but the latest head still builds the transcript mirror from text only, so media-only BlueBubbles cron deliveries remain delivered externally without being mirrored into the target transcript.

## What This Changes

This PR adds a BlueBubbles-only isolated cron delivery transcript mirror through the outbound session route, focused DM/group regression tests, and a changelog entry.

## Best Possible Solution

Keep the plugin-aware BlueBubbles route approach, but build the mirror from the existing outbound payload projection so text, mediaUrl/mediaUrls, media-only, and silent-text-plus-media cron deliveries all append to the normalized target transcript before the PR lands.

## Reproduction Assessment

Yes. Current main statically reproduces the original gap because dispatchCronDelivery never passes mirror while deliver.ts only appends transcripts when mirror exists; the PR-specific bug reproduces with a BlueBubbles cron payload that has mediaUrl/mediaUrls and no text.

## Solution Assessment

No. Reusing the BlueBubbles outbound session route is the narrow maintainable solution, but the submitted implementation is incomplete because it uses ad hoc text-only mirror construction instead of the existing text/media payload projection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Mirror media-only BlueBubbles cron payloads:** `src/cron/isolated-agent/delivery-dispatch.ts:138-143`

A BlueBubbles cron delivery with `mediaUrl` or `mediaUrls` still passes delivery content checks and can be sent, but this patch omits `mirror`, so the target transcript remains missing the delivered media. Build the mirror from the existing outbound payload projection, including `mediaUrls`, and add a media-only regression.
