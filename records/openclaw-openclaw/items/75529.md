## Summary

Keep open. The PR targets a real current-main gap in isolated BlueBubbles cron delivery mirroring, but the re-review head still drops media-bearing payloads from the mirror path and still has duplicate active changelog bullets.

## What This Changes

The branch adds a BlueBubbles-only cron delivery transcript mirror through outbound session routing, focused DM/group route-shape regression tests, and active changelog entries.

## Best Possible Solution

Keep the BlueBubbles-only route-aware mirror direction, but build the mirror payload from the same outbound payload plan/projection semantics used by message sends, add a media-only regression case, and leave one active changelog entry for the fix.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron payload such as `{ mediaUrl: "https://example.test/photo.jpg" }` passes current delivery-content filtering, but the PR helper returns undefined because it only projects `payload.text`.

## Solution Assessment

No. The route-aware BlueBubbles seam is the right narrow boundary, but the mirror projection should reuse or match the existing outbound payload projection so text, media URLs, and idempotency behavior stay consistent with normal sends.
