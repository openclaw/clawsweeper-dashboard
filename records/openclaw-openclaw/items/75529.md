## Summary

Keep open. The route-aware BlueBubbles cron mirror is the right narrow direction, but the latest PR head still mirrors only text payloads; media-only BlueBubbles cron deliveries can still be sent without being appended to the target transcript, which leaves the reported context gap partially unfixed.

## What This Changes

The PR adds BlueBubbles-only isolated cron delivery transcript mirroring through outbound route resolution, focused text/DM/group regression tests, and a changelog entry.

## Best Possible Solution

Keep the BlueBubbles-only route-aware mirror, but build the mirror content from the shared outbound payload mirror projection so text and media are normalized into the same target transcript path, then land after focused regression and changed-gate validation.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron delivery with `deliveryPayloads: [{ mediaUrl: "https://example.test/photo.jpg" }]` passes current delivery filtering, but the PR helper returns `undefined` before adding `mirror`, so the delivered media is absent from the target transcript.

## Solution Assessment

No. The route-aware BlueBubbles seam is the right narrow shape, but the current implementation duplicates text-only mirror handling instead of reusing the existing text/media outbound mirror projection.
