Keep open. Current main still lacks the BlueBubbles cron transcript mirror, and this PR is the right narrow surface, but the latest head is not merge-ready because it still drops valid media-only BlueBubbles cron deliveries from the mirrored transcript and duplicates the changelog entry.

## What This Changes

The PR adds BlueBubbles-only isolated cron delivery transcript mirroring via outbound route resolution, a new BlueBubbles cron mirror regression test file, and two active-version changelog bullets for the same change.

## Best Possible Solution

Land a BlueBubbles-only cron mirror that reuses the shared outbound payload mirror projection so text, media, idempotency, and route semantics stay aligned with the message tool, with one focused regression file and a single active-version changelog entry.

## Reproduction Assessment

Yes. A BlueBubbles isolated cron delivery with a payload such as { mediaUrl: "https://example.test/photo.jpg" } passes delivery content filtering, but the PR helper returns undefined because it only builds mirror text from payload.text.

## Solution Assessment

No. The route-aware BlueBubbles seam is the right narrow direction, but hand-rolling a text-only mirror projection is not the best fix when the existing outbound mirror projection already preserves media URLs consistently.
