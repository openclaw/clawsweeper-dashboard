## Summary

Keep this PR open. The underlying BlueBubbles cron transcript gap is valid on current main, but this patch is not merge-ready because it mirrors only text, can record failed best-effort payloads as if delivered, and duplicates the active changelog entry.

## What This Changes

The PR adds BlueBubbles-specific transcript mirroring to isolated cron direct delivery, a focused BlueBubbles mirror regression test file, and two changelog bullets for the same fix.

## Best Possible Solution

Land the BlueBubbles route-aware cron mirror after reusing the shared outbound payload mirror projection, preserving media context, aligning transcript append with successful delivery semantics, and keeping one changelog bullet.

## Reproduction Assessment

Yes. A focused unit test can dispatch a BlueBubbles cron payload containing only mediaUrl/mediaUrls and observe that this PR supplies no mirror; another can make one best-effort payload fail and confirm the batch mirror records content that cron marks undelivered.

## Solution Assessment

No. The BlueBubbles-only outbound route seam is a reasonable narrow boundary, but the implementation should not hand-roll text-only full-batch mirror content when shared media-aware projection and all-success delivery semantics already exist.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Mirror media payloads in cron transcripts:** `src/cron/isolated-agent/delivery-dispatch.ts:138-144`
  - body: The helper builds the mirror only from `payload.text` and returns before route resolution when that text is empty. Current delivery filtering accepts `mediaUrl` and `mediaUrls`, so media-only BlueBubbles cron deliveries can still send without any target transcript mirror, and mixed text+media deliveries lose the media context. Reuse the shared outbound payload mirror projection and add a media-only regression.
  - confidence: 0.94
- **[P2] Mirror only successfully delivered best-effort payloads:** `src/cron/isolated-agent/delivery-dispatch.ts:723`
  - body: This passes one full-batch mirror into `deliverOutboundPayloads` even when cron is using best-effort delivery. That helper appends the mirror whenever any result succeeds, while cron marks partial failures as undelivered for retry, so a multi-payload run can record failed payload text/media in the target transcript. Gate mirroring on all-success semantics or mirror only the successful projection.
  - confidence: 0.9
- **[P3] Remove the duplicate changelog entry:** `CHANGELOG.md:153-154`
  - body: The active changelog now has two adjacent bullets for the same BlueBubbles/cron fix, one with this PR reference and one without. Keep a single user-facing entry so release notes do not duplicate the change.
  - confidence: 0.98
