## Summary

Keep open. Current main still builds the Webchat session selector from stored session rows plus the active session key, so configured agents with no stored session remain absent; related implementation work is open but unmerged.

## What This Changes

Show every configured Webchat agent in the session dropdown, including agents that do not yet have stored sessions.

## Best Possible Solution

Land #48671 or an equivalent focused Control UI patch that merges `state.agentsList.agents` into `resolveSessionOptionGroups`, creates default per-agent main-session options using the configured main key, dedupes against stored and active session keys, preserves global/unknown/cron filtering, and adds a regression test where an agent exists only in `agents.list`.

## Reproduction Assessment

Yes. A source-level reproduction is `agentsList` containing `alpha` and `beta` while `sessions.sessions` contains only `agent:alpha:main`; current `resolveSessionOptionGroups` has no path that adds `beta`, so the dropdown omits it.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI resolver because `agents.list` is already loaded and the gateway already exposes configured agents; changing `sessions.list` would blur the store-backed session API contract.
