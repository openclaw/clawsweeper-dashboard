## Summary

Current main solves the reported Webchat agent-selection problem with a dedicated agent filter before the scoped session picker, and configured agents without stored sessions now switch to their main session. The proof commit is on current main only and is not contained in the latest release tag checked here.

## What This Changes

The issue requests that Webchat expose all configured agents, including agents without existing stored sessions, instead of only sessions returned by `sessions.list`.

## Best Possible Solution

Keep the current-main agent-filter design as the canonical solution and let the next release carry it, rather than reopening the older synthetic-session-option patches.

## Reproduction Assessment

No against current main. Source inspection shows configured agents are included through the agent filter, and the regression test covers switching to beta's main session when beta has no stored session row.

## Solution Assessment

Yes. Separating agent selection from per-agent session selection is the narrower maintainable fix, and it still solves the observable problem of selecting configured agents that have no existing sessions.
