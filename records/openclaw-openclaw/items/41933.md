## Summary

Close as implemented. Current main solves the observable Webchat problem with a dedicated agent filter before the session picker, and that shipped in v2026.5.20 with tests and docs covering configured agents that have no stored session row.

## What This Changes

The issue asks for the Webchat session dropdown to include configured agents that do not yet have persisted sessions.

## Best Possible Solution

Keep the shipped agent-filter implementation as the canonical Control UI behavior and close older duplicate implementation attempts rather than merging synthetic session-picker rows.

## Reproduction Assessment

No against current main. The old source-level scenario is now covered by `ui/src/ui/views/chat.test.ts:1658`, where beta is configured but absent from stored sessions and selecting beta switches to `agent:beta:main`.

## Solution Assessment

Yes. The current implementation separates agent selection from per-agent session selection, which is narrower than synthesizing configured agents into stored session rows and is documented as the intended Control UI behavior.
