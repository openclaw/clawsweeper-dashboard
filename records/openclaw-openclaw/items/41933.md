## Summary

Close as implemented. Current main solves the user-facing problem with a dedicated agent filter backed by the configured agent list before the scoped session picker; selecting an agent with no stored rows falls back to that agent's main session, with tests/docs and release provenance.

## What This Changes

The issue requests that Webchat let users select all configured agents, including agents with no existing persisted session rows.

## Best Possible Solution

Keep the shipped agent-filter UX as the canonical solution: configured agents are selected first, and the session picker remains scoped to the selected agent's sessions.

## Reproduction Assessment

No against current main for the user-facing failure. A configured `beta` agent with only an `alpha` session is selectable through the agent filter, and the regression test verifies switching to `agent:beta:main`.

## Solution Assessment

Yes. Separating agent selection from per-agent session selection is narrower and more maintainable than synthesizing cross-agent session rows inside the session picker.
