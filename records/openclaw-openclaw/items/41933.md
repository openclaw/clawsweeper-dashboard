## Summary

Current main already solves the reported Webchat multi-agent selection problem with an agent filter before the scoped session picker, backed by configured agents and covered by regression tests; the implementation is shipped in v2026.5.12.

## What This Changes

The issue asks Webchat to show configured agents that do not yet have stored sessions, instead of deriving selectable agents only from session rows.

## Best Possible Solution

Keep the shipped agent-filter design as the canonical solution: configured agents are selected in the agent filter, while the session picker remains scoped to the selected agent's sessions.

## Reproduction Assessment

No against current main. The current source and regression test cover an alpha/beta `agentsList` where beta has no stored session row and selecting beta falls back to `agent:beta:main`.

## Solution Assessment

Yes. Separating agent selection from per-agent session selection is now the narrow maintainable solution, and it is documented as the intended Control UI behavior.
