## Summary

Close as implemented. Current main solves the user-facing problem with a documented agent filter before the scoped session picker, falls back to an agent main session when no stored row exists, and the fix is included in a shipped stable release.

## What This Changes

The issue asks Webchat to let users select every configured agent even when some agents do not yet have persisted sessions.

## Best Possible Solution

Keep the current agent-filter-plus-scoped-session-picker design as the canonical fix, rather than adding synthetic cross-agent entries back into the session picker.

## Reproduction Assessment

No against current main. Source and regression coverage show a configured beta agent remains selectable through the agent filter and switches to `agent:beta:main` even when beta has no stored session row.

## Solution Assessment

Yes. The current implementation is the narrower maintainable solution because it separates agent selection from per-agent session selection and documents that UX explicitly.
