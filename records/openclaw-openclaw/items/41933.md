## Summary

Close as implemented on current main. Webchat now exposes a configured-agent filter before the scoped session picker, and selecting an agent with no stored sessions falls back to that agent's main session; the proof commit is on main only and is not in v2026.5.3.

## What This Changes

The requested behavior is for Webchat to let users select every configured agent, including agents that do not yet have stored sessions.

## Best Possible Solution

Keep the current two-select Control UI behavior: configured agents come from `agents.list`, while the session picker remains scoped to the selected agent's sessions.

## Reproduction Assessment

No, not against current main. A source-level check with `agentsList` containing alpha and beta while sessions contain only alpha now has a beta option in the agent filter, and selecting it switches to `agent:beta:main`.

## Solution Assessment

Yes. The current implementation is the narrow maintainable shape: use `agents.list` for agent selection and keep `sessions.list` store-backed for actual session rows.
