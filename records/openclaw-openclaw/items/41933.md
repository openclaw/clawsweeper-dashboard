## Summary

Close as implemented. Current main solves the reported Webchat agent-selection problem with a dedicated Control UI agent filter before the scoped session picker, and the fallback-to-main behavior for configured agents without stored sessions has shipped.

## What This Changes

The issue asks Control UI Webchat to show configured agents that do not yet have stored session rows.

## Best Possible Solution

Keep the shipped agent-filter implementation as the canonical Control UI solution and leave the session picker scoped to the selected agent.

## Reproduction Assessment

No against current main. The historical source-level repro now routes through the agent filter, and the focused test covers switching to a configured agent with no stored session row.

## Solution Assessment

Yes. Separating agent selection from per-agent session selection is cleaner than synthesizing every configured agent into the session picker itself, and it is documented and tested.
