## Summary

Close as implemented. Current main and the shipped v2026.5.22 release solve the user-visible problem with a dedicated chat agent filter that lists configured agents and falls back to the selected agent's main session when no stored session row exists.

## What This Changes

The issue requests that Webchat let users select every configured agent, including agents with no stored session row, instead of deriving choices only from `sessions.list`.

## Best Possible Solution

Keep the shipped agent-filter design as the canonical solution and treat the older session-dropdown-only PR attempts as obsolete duplicates.

## Reproduction Assessment

No against current main. Source inspection and regression coverage show the chat agent filter includes configured agents and switches a configured agent with no stored row to `agent:beta:main`.

## Solution Assessment

Yes. The current implementation keeps `sessions.list` store-backed, separates agent selection from per-agent session selection, and documents the fallback behavior.
