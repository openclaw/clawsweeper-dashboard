## Summary

Close as implemented. Current main solves the user problem with an agent filter before the scoped session picker, falls back to the selected agent's main session when no stored row exists, and the fix shipped in v2026.5.4.

## What This Changes

The issue requests that Webchat show all configured agents in the chat selection UI instead of only agents with stored sessions.

## Best Possible Solution

Keep the shipped agent-filter design as the canonical solution because it separates agent selection from per-agent session selection while preserving scoped session behavior.

## Reproduction Assessment

No against current main. The current source includes all configured agents through the agent filter, and the regression test covers switching to beta's main session when beta has no stored session row.

## Solution Assessment

Yes. The shipped solution is narrower and more maintainable than synthesizing “new” entries inside the session picker because it keeps agent choice and session choice as separate controls.
