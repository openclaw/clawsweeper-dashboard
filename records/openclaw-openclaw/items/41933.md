## What This Changes

The issue asks the Control UI Webchat selector to show every configured agent, including agents that do not yet have persisted session rows.

Keep the shipped agent-filter plus scoped session-picker design as the canonical solution, and treat older duplicate PR attempts as superseded by current main.

No failing reproduction remains against current main. Source and tests now cover selecting a configured beta agent when sessions only contain alpha, with fallback to agent:beta:main.

## Solution Assessment

Yes. The shipped agent-filter design is more maintainable than adding synthetic rows directly to the session picker because it separates agent selection from per-agent session selection.
