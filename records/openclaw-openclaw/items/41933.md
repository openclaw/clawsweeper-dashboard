## Summary

Current main now solves the user-visible problem with a dedicated Webchat agent filter before the scoped session picker, so configured agents without stored session rows are selectable and fall back to their main session.

## What This Changes

The issue requests that Webchat show all configured agents in the chat session selection flow, not only agents that already have persisted sessions.

## Best Possible Solution

Keep the current split agent-filter/session-picker design as the canonical solution and retire the older unmerged dropdown-only attempts as obsolete.

## Reproduction Assessment

No against current main. The source path and regression test cover `agentsList` containing alpha and beta while only alpha has a stored session, and switching the agent filter to beta targets `agent:beta:main`.

## Solution Assessment

Yes. The current implementation is the cleaner maintainable fix because it separates agent selection from per-agent session selection while preserving fallback behavior for configured agents with no sessions.
