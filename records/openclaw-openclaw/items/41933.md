## Summary

Keep open. Current main still builds the Webchat session dropdown from stored session rows plus the active key, so configured agents with no stored session row remain omitted, and the related implementation work is still open/unmerged.

## What This Changes

The issue requests that Webchat’s session dropdown include every configured agent, including agents that do not yet have a stored session.

## Best Possible Solution

Land #48671 or an equivalent focused patch that merges configured agents into the session option groups, dedupes against stored/active session keys, preserves existing filtering and labels, and adds a configured-agent-without-session regression test.

## Reproduction Assessment

Yes. A source-level reproduction is to pass `agentsList` with `alpha` and `beta` while `sessions.sessions` contains only `agent:alpha:main`; current main has no code path that adds `beta` to the dropdown options.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in `resolveSessionOptionGroups`, because `agents.list` is already loaded by Control UI and `sessions.list` is intentionally store-backed.
