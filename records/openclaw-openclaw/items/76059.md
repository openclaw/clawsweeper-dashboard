## Summary

Current main and shipped releases after the reporter's 2026.4.24 build no longer use the old aggregate prompt-token formula for the Web UI message metadata context percentage; the footer now bases the percentage on the largest single assistant call in the rendered group and has focused regression coverage.

## What This Changes

The issue requests correcting the Web UI message metadata context percentage so long sessions do not report 100% context from accumulated per-call cache-read tokens.

## Best Possible Solution

Keep the shipped v2026.4.25+ implementation and have affected users upgrade from 2026.4.24 or older.

## Reproduction Assessment

Yes. The v2026.4.24 source has the summed `input + cacheRead + cacheWrite` path described by the report, while current main removes that formula and includes a focused grouped-call regression test.

## Solution Assessment

Yes. The current implementation is a maintainable fix because it preserves aggregate token/cache display while deriving the context badge from a single prompt snapshot instead of lifetime accumulated cache reads.
