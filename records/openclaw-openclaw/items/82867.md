## Summary

Keep this PR open: the doctor diagnostic direction is useful and the contributor provided sufficient before/after proof, but the latest diff still misses supported configured session-store layouts. The protected `maintainer` label also means this should stay in explicit maintainer handling rather than cleanup close.

## What This Changes

The PR adds a read-only `openclaw doctor` health contribution and tests that scan cached session snapshot metadata for stale bundled-skill runtime paths.

## Best Possible Solution

Keep the new doctor diagnostic, but pass `ctx.cfg` into the contribution and use `resolveAllAgentSessionStoreTargets` or equivalent cfg-aware discovery before scanning stores.

## Reproduction Assessment

Yes. The linked proof artifacts show the current-main absence and after-PR stale-path detection, and the review blocker is source-reproducible because the scanner enumerates only default state-dir stores despite the documented `session.store` contract.

## Solution Assessment

No, not yet. The metadata scanner is the right shape, but the store discovery is narrower than OpenClaw's supported session-store contract; reusing the cfg-aware target resolver is the safer fix.
