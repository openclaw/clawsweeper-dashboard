## Summary

Keep open: current main and the latest release still only count structured `cron` tool adds as successful same-turn reminder scheduling, so a successful `openclaw cron add` through `exec`/`bash` can still leave `successfulCronAdds` at zero and trigger the contradictory note. The remaining work is already owned by the open linked fix PR at https://github.com/openclaw/openclaw/pull/86064, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

This issue reports that Telegram reminder replies can get the false `I did not schedule a reminder` note after a successful `openclaw cron add --at` reminder.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/86064 or an equivalent focused fix that treats successful `exec`/`bash` `openclaw cron add` calls as same-turn cron scheduling, with regression coverage for positive and bypass cases.

## Reproduction Assessment

Yes, we have a high-confidence source reproduction path: an agent can run `openclaw cron add` through `exec`/`bash`, but current main increments `successfulCronAdds` only for the structured `cron` tool, so the reply guard can still append the note when the assistant confirms the reminder.

## Solution Assessment

Yes, the best path is the narrow handler/test fix already represented by https://github.com/openclaw/openclaw/pull/86064; closing this issue before that PR or an equivalent change lands would drop a valid current-main bug.
