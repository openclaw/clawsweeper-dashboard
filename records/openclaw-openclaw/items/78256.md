Keep open. The issue reports that an approved subagent exec command can return stdout/stderr from an unrelated context instead of the approved command result.

## Best Possible Solution

Land a focused exec approval followup fix that threads the approved command through gateway and node followup targets, makes the exact completion details authoritative for successful followups, and adds concurrent/out-of-order approved-run regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established for the intermittent SSH case. Source inspection does show the successful approval followup still lacks approved-command binding and a stale-output guard, which matches the closed direct fix's diagnosis.

## Solution Assessment

Yes, the narrow maintainable solution is to bind the approved command through the existing followup target/prompt path and cover out-of-order completions. The adjacent truncation PR is useful but does not solve this command/result association issue.
