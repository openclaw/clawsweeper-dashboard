## What This Changes

The issue requests reliable binding between an approved subagent exec command and the stdout/stderr returned after approval.

## Best Possible Solution

Bind the approved command plus approval/run/session identity through gateway/node followup targets and the successful followup prompt, make exact completion details authoritative, and add concurrent out-of-order completion coverage.

No high-confidence live reproduction was established for the intermittent SSH case. Source inspection does show current main still lacks approved-command binding in successful exec approval followups, matching the closed direct fix's diagnosis.

## Solution Assessment
