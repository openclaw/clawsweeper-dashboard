## Summary

Current main already routes explicit Telegram cron announce targets, including bare numeric DM IDs, through the direct outbound adapter and marks delivery only after a real send result, so the v2026.5.4 symptom is addressed on main.

## What This Changes

Cron announce delivery to Telegram DM targets was reported to complete successfully while no message reached the DM target.

## Best Possible Solution

Keep the current direct outbound cron announce implementation and include it in the next release; handle the linked `delivery.mode: none` and Telegram group-routing reports on their separate issue tracks.

## Reproduction Assessment

No current-main failing reproduction remains from source inspection: the explicit Telegram DM target is resolved and sent through the direct outbound adapter, with regression coverage for a bare numeric target. I did not run a live Telegram DM because this read-only review cannot use private bot credentials.

## Solution Assessment

Yes. Routing cron announce output through the existing durable outbound adapter is the narrow maintainable solution because it reuses channel send semantics and makes `delivered=true` depend on a real send result.
