Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not fall back after rejecting a wrapper:** `extensions/acpx/src/runtime.ts:711-713`
  - body: The catch block returns the original command even when `applyWindowsSpawnProgramPolicy({ allowShellFallback: false })` rejected an unresolved wrapper. That hands the wrapper back to acpx, whose current spawn options use `shell: true` for `.cmd`/`.bat`, so the intended fail-closed no-shell policy is silently bypassed.
  - confidence: 0.9
- **[P1] Preserve acpx built-in launch resolution:** `extensions/acpx/src/runtime.ts:745`
  - body: Wrapping the registry before `BaseAcpxRuntime` sees it rewrites the exact built-in claude/codex command strings. In acpx 0.10.0 those exact strings are how `resolveBuiltInAgentLaunch` chooses the installed adapter package or npm CLI via `process.execPath`; after this rewrite, Windows built-ins can fall back to package-manager execution instead of the bundled adapter path.
  - confidence: 0.87
