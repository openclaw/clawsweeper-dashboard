## Summary

Keep open. The linked Feishu bug is source-reproducible and this PR targets the right card-header paths, but it is not merge-ready because the new shared helper import pulls the full outbound adapter into reply-dispatcher, the extractor splits valid emoji clusters, and the supplied proof is only grep plus unit-test output rather than a real Feishu card run.

## What This Changes

The PR adds an `extractEmojiOnly` helper in the Feishu outbound module and uses it to strip non-emoji text from identity emoji values before building Feishu card headers.

## Best Possible Solution

Move the sanitizer into a small Feishu-local identity/header helper used by outbound and reply-dispatcher, preserve emoji clusters with focused tests, and add redacted real Feishu card proof before merge.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.6 shows three Feishu card-header paths concatenate raw `identity.emoji`, matching the linked issue; I did not run a live Feishu send in this read-only review.

## Solution Assessment

No. Filtering all card header paths is the right direction, but this branch couples reply-dispatcher to outbound and corrupts valid emoji clusters; a shared, cluster-aware Feishu helper with tests is the safer fix.
