## Summary

Keep open: the direction is useful and exact-head checks are green, but source review found a scoped-policy regression where some non-native channels can stop enforcing required mentions after configured patterns are denied.

## What This Changes

The PR adds scoped mention-pattern policy controls across config, SDK docs, channel runtimes, generated baselines, tests, and a few rebase-related cleanup changes.

## Best Possible Solution

Fix the canDetectMention interaction for channels without native mention metadata, add focused regression coverage, then proceed through normal maintainer review with the existing changelog entry.

## Reproduction Assessment

Yes for the review finding from source: configure a Signal group that requires mentions, has configured mention patterns, and has a global or provider policy denial; the PR empties mentionRegexes, making canDetectMention false so the missing-mention drop is bypassed.

## Solution Assessment

No, not yet; the shared resolver shape is maintainable, but callers must separate pattern contribution to wasMentioned from whether mention gating should still be enforceable when policy disables configured regexes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve mention gating when scoped policy disables regexes:** `extensions/signal/src/monitor/event-handler.ts:686-694`
  - body: When the new policy denies configured patterns, this block replaces the regex list with `[]`, but the later gate still computes `canDetectMention` from `mentionRegexes.length` and only drops missing mentions when that is true. In a Signal group with `requireMention` enabled and `mentionPatternsMode: "deny"` or a matching `denyIn`, an unmentioned message will now pass instead of being dropped; the same shape should be checked in iMessage and BlueBubbles. Keep regex matches from contributing to `wasMentioned`, but do not turn off the required-mention gate for scopes that deliberately deny configured patterns.
  - confidence: 0.86
