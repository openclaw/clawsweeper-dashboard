## What This Changes

The PR adds scoped mention-pattern policy controls across config, SDK runtime helpers, bundled channel runtimes, docs, generated metadata, changelog, and tests.

## Best Possible Solution

Land this only after every configured-pattern consumer, including qa-channel, uses the shared policy resolver and exact-head redacted real behavior proof covers the affected visible channel path.

## Reproduction Assessment

Yes for the review finding: PR-head source inspection shows deny mode disables configured regexes, while qa-channel still builds and evaluates those regexes directly with canDetectMention true. I did not run tests because this was a read-only review.

## Solution Assessment

No: the shared resolver is the right direction, but the rollout remains incomplete while qa-channel bypasses it. The narrow fix is to call resolveMentionPatternsEnabled with the routed agent and stable qa-channel conversation id before building regexes, then add deny-mode coverage.
