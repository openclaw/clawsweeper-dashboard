## Summary

Keep open: the latest PR head still has a source-reproducible Nextcloud Talk blocker where disabled configured mention patterns make `requireMention` rooms skip messages because the custom gate does not consider whether mention detection is possible.

## What This Changes

The PR adds scoped mention-pattern policy config across global, agent, provider, and conversation scopes, wires it through bundled channel runtimes, updates SDK/config/docs/changelog surfaces, and includes related tests plus post-rebase cleanup fixes.

## Best Possible Solution

Keep the scoped policy design, but make Nextcloud Talk use shared mention decision semantics or pass detection capability into its custom gate with regression coverage.

## Reproduction Assessment

Yes for the PR finding by source inspection: a Nextcloud Talk group with `requireMention` and denied configured patterns gets `mentionRegexes=[]`, `wasMentioned=false`, and the custom gate skips without checking whether mention detection is possible. I did not run tests because this was a read-only review.

## Solution Assessment

No, not yet. The shared resolver and config shape are reasonable, but the Nextcloud Talk integration needs the same `canDetectMention` behavior as the shared mention gate before merge.
