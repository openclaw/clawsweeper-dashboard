## Summary

Keep open: this PR adds a useful scoped mention-pattern policy, but the latest head still needs a source-level fix before merge because deny-scoped configured patterns can make pattern-only required-mention channels treat mention detection as unavailable.

## What This Changes

Adds scoped mention-pattern policy config/helpers and wires configured regex mention allow/deny behavior through channel runtimes, docs, generated metadata, and tests.

## Best Possible Solution

Keep the shared resolver and config surface, but fix pattern-only channel callers so policy-disabled regexes cannot count as mention evidence while still preserving required-mention drops when configured patterns exist.

## Reproduction Assessment

Yes, source-reproducible: configure a Signal group with requireMention enabled, configured mentionPatterns, and a provider/global deny policy; the PR empties mentionRegexes and makes canDetectMention false, so an unmentioned message is not dropped.

## Solution Assessment

No, not yet; the shared policy resolver is a reasonable direction, but pattern-only handlers need a separate detection-capability signal from the policy-filtered regex evidence.
