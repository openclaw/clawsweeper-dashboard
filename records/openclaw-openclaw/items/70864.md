## Summary

Keep open; current main does not have the scoped mention-pattern policy, and the latest PR head still has a concrete Nextcloud Talk regression. Prior review follow-ups fixed several scoping and docs issues, but this blocker remains before merge.

## What This Changes

The PR adds global, per-agent, provider, and conversation-scoped mention-pattern policy config, wires it through bundled channel runtimes and SDK helpers, and updates docs, generated metadata, changelog, and focused tests.

## Best Possible Solution

Keep the scoped policy architecture, but make Nextcloud Talk use the shared mention decision helper or pass an explicit detection-capability flag, then add a regression test for denied configured patterns in requireMention rooms.

## Reproduction Assessment

Yes for the PR finding. On head 05ebaea3c1236c2cad48447f03b3fdbacc1d345b, a requireMention Nextcloud Talk group with configured mention patterns denied gets mentionRegexes=[], wasMentioned=false, and the custom gate skips every non-command message.

## Solution Assessment

No, not yet. The shared resolver and config shape are reasonable, but the Nextcloud Talk integration needs the same canDetectMention semantics as the shared mention gate before this is merge-ready.
