## Summary

Keep this PR open: current main does not already contain the scoped mention-pattern policy, but the PR still has a source-reproducible Nextcloud Talk regression and the latest head is dirty/unmergeable.

## What This Changes

The PR adds scoped mention-pattern policy config, SDK helpers, channel runtime wiring, docs, generated metadata, and tests for global, agent, provider, and conversation-level regex mention controls.

## Best Possible Solution

Keep the shared policy direction, but make Nextcloud Talk preserve detection-capability semantics when configured patterns are disabled and add a regression test for a requireMention room with provider deny policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Nextcloud Talk group with requireMention enabled and configured mentionPatterns plus a provider deny policy gets an empty regex list, wasMentioned=false, and the custom gate drops every non-command message.

## Solution Assessment

No, not yet. The shared resolver is a reasonable implementation direction, but Nextcloud Talk should use the shared mention decision or add a canDetectMention input so disabled regex evidence does not create an unreachable requireMention room.
