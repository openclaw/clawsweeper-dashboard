## Summary

Keep open: the latest head fixes the earlier qa-channel policy bypass, but it still leaves qa-channel reporting mention detection as available after scoped policy disables all configured regexes. That causes `requireMention` qa-channel groups to skip messages instead of following the documented no-detection fallback; security review did not find a concrete supply-chain or secrets regression.

## What This Changes

The PR adds scoped mention-pattern policy controls for global, agent, provider, and conversation-level regex mention gating across channel runtimes, SDK exports, config schemas, docs, generated metadata, changelog, and tests.

## Best Possible Solution

Keep the shared resolver rollout, but align qa-channel activation facts with policy-enabled regex availability before merge.

## Reproduction Assessment

Yes for the review finding: source inspection shows qa-channel sets regexes to an empty array when policy disables patterns, then still passes `canDetectMention: true` into the shared gate. I did not execute tests because this was a read-only review.

## Solution Assessment

No: the shared resolver is the right direction, but qa-channel should not claim mention detection is available when policy-disabled regexes leave it with no native or configured mention signal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Report qa-channel detection only when regexes remain enabled:** `extensions/qa-channel/src/inbound.ts:102-111`
  - body: When scoped policy disables configured patterns, this branch sets `mentionRegexes` to `[]` but the later ingress call still passes `canDetectMention: true`. The shared gate skips `requireMention` groups whenever detection is claimed and `wasMentioned` is false, so a qa-channel group in a denied or unlisted scope drops every normal message instead of using the documented no-detection fallback. Derive `canDetectMention` from the policy-enabled regex list, or omit mention facts when no qa-channel mention signal exists, and update the disabled-policy test accordingly.
