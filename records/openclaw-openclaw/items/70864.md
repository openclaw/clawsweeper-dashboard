Full review comments:

- **[P2] Restore browser action timeout slack:** `extensions/browser/src/cli/browser-cli-actions-input/shared.ts:40`
  - body: The branch passes the inner browser action timeout as the outer HTTP request timeout. Waits, downloads, uploads, and dialog hooks can legitimately take the full requested timeout, so the client may time out before the browser operation completes; current main added slack and longer hook defaults to avoid that.
  - confidence: 0.93
- **[P2] Reject conflicting dialog actions:** `extensions/browser/src/cli/browser-cli-actions-input/register.files-downloads.ts:184`
  - body: When users pass both `--accept` and `--dismiss`, this expression silently chooses accept and arms the hook. Current main rejects that conflicting input before making the browser request, which prevents an accidental accept of a dialog the caller intended to dismiss.
  - confidence: 0.91
- **[P2] Preserve lazy browser option parsing:** `extensions/browser/src/cli/browser-cli.ts:152`
  - body: Using `commandPath[1]` treats browser option values as the lazy subcommand, so an invocation like `browser --browser-profile status start` registers the status group instead of the requested start command. Current main has a parser that skips root/browser option values and keeps nested command parent options intact.
  - confidence: 0.88
