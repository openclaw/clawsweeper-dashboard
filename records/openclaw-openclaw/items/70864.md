## Summary

Keep open: the feature is useful and the exact-head terminal proof is now present, but latest head still leaves qa-channel outside the scoped mention-pattern policy, so deny mode can be bypassed there.

## What This Changes

The PR adds scoped mention-pattern allow/deny config and SDK helpers, wires them through many bundled channel runtimes, updates docs/generated metadata, and refreshes related tests/baselines.

## Best Possible Solution

Finish the policy rollout by applying the shared resolver to every configured mention consumer, including qa-channel, with focused deny-mode coverage and final channel/CI proof before landing.

## Reproduction Assessment

Not applicable as a current-main user bug reproduction because this is a feature/config PR. For the review finding, yes: latest-head source inspection shows qa-channel still builds configured regex mentions directly while the new helper defines deny mode as disabling them.

## Solution Assessment

No: the shared resolver is the right direction, but the rollout is incomplete while qa-channel bypasses it. The narrow fix is to reuse resolveMentionPatternsEnabled before qa-channel builds regexes and cover deny mode in qa-channel tests.
