## Summary

Keep open: the latest PR head still has a source-level mention-gating regression in Signal when scoped policy disables configured regex matches, so the patch is not ready to merge.

## What This Changes

The PR adds scoped mention-pattern policy config and SDK/docs/schema support, wires it through bundled channel runtimes, and includes related tests, generated baselines, changelog, and rebase cleanup hunks.

## Best Possible Solution

Keep the new policy surface, but fix callers so disabled configured regexes no longer count as mention evidence without turning required-mention groups into allow-all groups.

## Reproduction Assessment

Yes, source-reproducible: configure a Signal group with requireMention true, a configured mention pattern, and provider/global mentionPatterns mode deny; the PR empties mentionRegexes and then makes canDetectMention false, bypassing the missing-mention drop.

## Solution Assessment

No, not yet; the shared resolver shape is maintainable, but channel callers need to separate whether configured patterns count as wasMentioned from whether mention gating remains enforceable.
