## Summary

Keep open. This PR is aimed at a real shared-session heartbeat model bleed bug and moves in the right direction, but it is not merge-ready: the preserve path can create an invalid provider/model pair for legacy model-only session entries, and the changelog entry will not pass the repository PR changelog gates.

## What This Changes

The PR adds a `preserveRuntimeModel` option to session-store persistence, wires it for heartbeat runs, adds focused session-store tests, and adds a changelog entry for the heartbeat model override bleed fix.

## Best Possible Solution

Land a narrow corrected patch that keeps heartbeat model overrides scoped to heartbeat runs, preserves only valid pre-existing runtime model identity, covers legacy model-only session entries, leaves non-heartbeat persistence unchanged, and satisfies the active changelog gate.

## Reproduction Assessment

Yes. The linked issue gives a concrete heartbeat-model-vs-primary-model setup and status before/after; at code level, current main reproduces the stale metadata path because `updateSessionStoreAfterAgentRun` persists the completed heartbeat run model into the shared session entry.

## Solution Assessment

No, not as currently written. The proposed direction is maintainable, but the preserve branch should not combine a preserved legacy `entry.model` with the heartbeat `providerUsed`, and the changelog line must match the PR gate contract before merge.
