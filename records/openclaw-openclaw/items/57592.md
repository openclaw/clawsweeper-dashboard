## What This Changes

The issue requests inline image rendering in `openclaw tui` for image-capable terminals such as iTerm2, Kitty, and Sixel terminals, with fallback placeholder behavior and an optional config setting.

## Best Possible Solution

Review, land, or replace the open draft with a TUI-owned inline media renderer that preserves placeholder fallback, documents any config contract, handles local media safely, and proves real behavior in at least one image-capable terminal.

## Reproduction Assessment

## Solution Assessment

Unclear. Inline terminal rendering is plausible, but maintainers still need to settle supported protocols, config shape, fallback behavior, local media safety, and proof expectations through the open draft or a replacement.
