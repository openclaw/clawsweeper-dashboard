## Summary

Keep open. Current main and the latest release still render TUI image tool-result content as bracketed placeholder text, while the matching implementation path is an open draft PR that still needs maintainer decisions and real terminal proof.

## What This Changes

The issue requests inline image rendering in the TUI for image-capable terminals, with graceful placeholder fallback and a possible `tui.inlineImages` configuration mode.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/49022 with a TUI-owned inline media renderer that preserves placeholder fallback, constrains local media reads, documents any accepted setting, and proves behavior in a real image-capable terminal.

## Reproduction Assessment

Yes, source-level: current `ToolExecutionComponent` converts image content entries to bracketed placeholder text. I did not run a live image-capable terminal repro in this read-only review.

## Solution Assessment

Unclear. Inline terminal rendering is plausible, but maintainers still need to settle protocol support, config shape, fallback behavior, local media safety, and proof expectations through the open draft or a replacement.
