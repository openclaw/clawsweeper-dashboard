## Summary

Keep open. Current main still converts TUI image content to text placeholders and has no inline terminal-image renderer or documented inline-image setting, while the only active implementation path is the open draft at https://github.com/openclaw/openclaw/pull/49022.

## What This Changes

The issue requests inline image rendering in the TUI for iTerm2, Kitty, and Sixel-capable terminals, with graceful placeholder fallback and an auto/disable config option.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/49022 with a TUI-owned inline media renderer that keeps placeholder fallback, defines the supported protocol/config contract, handles local media safely, and adds focused tests and docs.

## Solution Assessment
