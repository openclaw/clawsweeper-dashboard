## Summary

Keep open. Current main and v2026.5.22 still render TUI image tool-result content as bracketed placeholder text, while the related implementation remains an open draft PR with proof and product/security-boundary decisions still unresolved.

## What This Changes

Requested inline image rendering in `openclaw tui` for image-capable terminals, with graceful placeholder fallback and an optional auto/on/off config flag.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/49022 with a TUI-owned inline media renderer that preserves placeholder fallback, constrains local media reads, documents any accepted config, and proves behavior in at least one real image-capable terminal.

## Reproduction Assessment

Yes, source-level: current `ToolExecutionComponent` converts image content entries to bracketed placeholder text on main and in `v2026.5.22`. I did not run a live image-capable terminal repro in this read-only review.

## Solution Assessment

Unclear. Inline terminal rendering is a plausible direction, but maintainers still need to choose protocol scope, config shape, fallback behavior, and local media safety through the open draft or a replacement.
