## What This Changes

The issue requests inline image rendering in the TUI for image-capable terminals, with graceful placeholder fallback and an optional auto/on/off configuration.

## Best Possible Solution

Review, repair, land, or replace the open draft with a TUI-owned inline media renderer that preserves placeholder fallback, constrains local media reads, documents any accepted config, and proves behavior in at least one real image-capable terminal.

Yes, source-level: current `ToolExecutionComponent` converts image content entries to bracketed placeholder text. I did not run a live image-capable terminal repro in this read-only review.

## Solution Assessment

Unclear. Inline terminal rendering is plausible, but maintainers still need to choose supported protocols, config shape, fallback behavior, local media safety, and proof expectations through the open draft or a replacement.
