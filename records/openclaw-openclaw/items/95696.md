## Summary

Keep open: current main and the latest release still detect a pending live `/model` switch but feed the run-start provider/model/auth tuple into timeout and overflow compaction when the active attempt cannot restart cleanly. The issue already has open linked fix candidates, so cleanup should wait for maintainer review of those PRs rather than closing or queuing another repair.

## What This Changes

The issue reports that auto-compaction can lose a live `/model` override and route subsequent API calls to the previous provider/model.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/95713 or an equivalent narrow run-loop fix that carries the already-resolved live model selection and matching auth profile into both compaction paths, with regression coverage for timeout and overflow compaction.

## Reproduction Assessment

Yes. Source inspection shows current main can detect a pending live `/model` selection, but if the active attempt cannot restart cleanly, both timeout and overflow compaction still use the run-start provider/model/auth tuple.

## Solution Assessment

Yes. The best fix is a narrow embedded-runner compaction-context repair that reuses the already-prepared live selection; adding a new lookup or broad model-selection policy change would be wider than the observed failure.
