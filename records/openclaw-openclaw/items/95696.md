## Label Justifications

- `P1`: The bug can silently route compaction and subsequent work through the wrong model/provider after a user-selected `/model` switch.
- `impact:session-state`: The issue is about session model override state diverging from the model used by compaction.

## Summary

Keep open: current main still has the stale compaction routing path, and the strongest fix is still an open PR rather than merged or shipped.

## What This Changes

The issue reports that after a session-level `/model` switch, auto-compaction can use an older provider/model while the UI still shows the most recent model.

## Best Possible Solution

Review and land or explicitly reject the focused embedded-runner compaction routing fix so auto-compaction uses the same live provider/model/auth selection as the active session.

## Reproduction Assessment

Yes at source level: current main reads a pending live model selection but still passes the attempt-local provider/model/auth profile into both compaction context builders. I did not run a live multi-provider compaction session in this read-only review.

## Solution Assessment

Yes: the narrow runner-side repair that threads the already-resolved live selection into timeout and overflow compaction is the best fit. A model-selection-only fix does not address the proven stale compaction caller inputs.
