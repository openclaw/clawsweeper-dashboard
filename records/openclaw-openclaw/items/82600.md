## Summary

Keep this PR open, but it should not merge as-is. The linked status bug is plausible, but the patch changes registry loading while current `status --json` explicitly disables channel summaries, and the diff also adds user-visible stdout debug output.

## What This Changes

The branch wraps CLI plugin registry loading with plugin auto-enable and prints channel plugin IDs while building channel summaries.

## Best Possible Solution

Decide the `status --json` channel-summary contract, then fix the actual status summary path to include configured auto-enabled channels without stdout debug output and with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current `status --json` passes `includeChannelSummary: false`, and its test asserts this behavior. I did not run a live command in this read-only review.

## Solution Assessment

No. The proposed registry-loader change does not reach the current fast JSON channel-summary path, and the added debug print would corrupt or noisy-up command output; the fix should target status summary resolution directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - confidence: 0.9
