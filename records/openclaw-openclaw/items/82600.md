## Summary

Keep open for maintainer follow-up, but this PR should not merge as-is: it does not fix the linked `openclaw status --json` path, it adds stdout debug output, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR attempts to apply plugin auto-enable while loading the CLI plugin registry and adds a channel-summary debug log of loaded channel plugin IDs.

## Best Possible Solution

Define the `status --json` channel-summary contract, then fix the actual status path without stdout debug output and add regression coverage plus redacted real CLI proof.

## Reproduction Assessment

Yes, source-reproducible: current main's `status --json` fast path sets `includeChannelSummary: false`, and its test asserts the channel summary path is bypassed. I did not run a live Telegram setup.

## Solution Assessment

No. The proposed registry-loader change does not reach the current `status --json` summary path, and the debug stdout line would break or pollute status output; the fix should target the status summary/fast JSON contract directly.
