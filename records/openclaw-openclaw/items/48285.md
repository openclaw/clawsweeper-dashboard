## Summary

Keep this PR open for maintainer handling, but do not merge it as-is. The branch tries to restore loopback internal-service connectivity, but the diff relaxes the node-role device-identity gate while current main routes the named internal RPC callers through the gateway-client/backend operator path instead.

## What This Changes

The PR adds a loopback authenticated node-role device-identity bypass, an auth.mode=none backend self-pairing skip, and unit coverage for those paths.

## Best Possible Solution

Keep node device identity enforcement intact; if a loopback internal-service failure remains, fix the gateway-client/backend auth or pairing path with targeted cron, sessions_spawn, and ACP coverage.
