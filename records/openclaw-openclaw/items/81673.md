## Summary

Keep open: current main still treats Windows and Linux companion apps as planned work, while macOS is the only platform with an established first-party appcast and packaged companion release path. Related open desktop-app PRs are useful implementation candidates, but they do not yet provide the official Windows/Linux download artifacts, release metadata, or downstream manifest requested here.

## What This Changes

The issue asks OpenClaw to define and publish official Windows and Linux companion app packaging, download metadata, and platform support docs.

## Best Possible Solution

Keep this as the release/download acceptance tracker, link it with the broader Linux/Windows companion-app work, and implement official packaging, manifest or stable URLs, and platform docs after maintainers choose the supported Windows and Linux companion targets.
