## Summary

Keep open: this is a focused candidate fix for the still-open sidebar part of the provider quota regression. Current main still suppresses the existing quota pill when the sidebar calls the session switcher, while this PR changes that gate, adds the missing sidebar quota grid row, and adds expanded/collapsed regression coverage.

## What This Changes

The branch changes Control UI sidebar session-switcher rendering and CSS so the provider usage quota pill appears when expanded, stays hidden when collapsed, and is covered by regression tests.

## Best Possible Solution

Land the narrow sidebar render/layout fix after maintainer review and check gating, while keeping the existing provider-agnostic quota data path unchanged.

## Reproduction Assessment

Yes, at source level: current main passes the sidebar through `sessionSwitcherOnly: true`, and that path suppresses `renderChatQuotaPill`. The PR body also includes before/after live render output for the same renderer path.

## Solution Assessment

Yes, for the sidebar-specific regression: gating the informational quota pill on compact space preserves the intended model-selector suppression and reuses the existing quota data/render path. The older unconditional guard removal was too broad because it did not handle collapsed sidebar space or the missing quota grid area.
