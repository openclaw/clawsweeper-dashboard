## Summary

Keep open: current main still accepts and forwards `memorySearch.outputDimensionality`, but the built-in local GGUF provider ignores it and the local provider cache identity omits the configured dimension. The related OpenAI-compatible dimensions issues are closed as shipped, but they do not cover this local GGUF path.

## What This Changes

Request to make `memorySearch.outputDimensionality` truncate local GGUF embedding vectors, trigger reindexing when the dimension changes, and optionally add query-instruction support.

## Best Possible Solution

Implement local GGUF truncation through the existing `outputDimensionality` contract, include the dimension in local provider cache/reindex identity, add focused tests and docs, and leave query-instruction support as a separate follow-up.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create the local provider with `outputDimensionality` and a mocked `node-llama-cpp` vector; current main never reads that option and returns the full normalized vector.

## Solution Assessment

Yes for the main requested fix. Reusing the existing `outputDimensionality` field is the narrowest maintainable solution; adding `queryInstruction` would be a separate public config decision.
