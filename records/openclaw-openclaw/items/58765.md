## Summary

Keep open: current `main` and the latest release still do not make local GGUF embeddings honor `outputDimensionality`, while the earlier OpenAI-compatible dimensions issue is closed as implemented and explicitly leaves this local-provider follow-up separate.

## What This Changes

The issue asks OpenClaw to let local GGUF embedding models truncate output vectors via the existing `memorySearch.outputDimensionality` setting and rebuild the memory index when that dimension changes.

## Best Possible Solution

Implement local GGUF dimension support in the existing local embedding path by preserving `outputDimensionality`, slicing before normalization, and including the dimension in cache/reindex identity.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the local GGUF path receives `outputDimensionality` but drops or ignores it before embedding and cache identity, so configured local vectors remain full length.

## Solution Assessment

Yes. Reusing `memorySearch.outputDimensionality` in the local provider path is narrower and more maintainable than adding a parallel `truncateDimensions` config key.
