## Label Justifications

- `P2`: The issue affects memory-search storage and retrieval performance for local embedding users, but it is not an urgent runtime outage.
- `impact:session-state`: The requested behavior changes persisted memory embedding vectors and reindex identity, which affects memory/session recall state.

## Summary

Keep open: current main and v2026.6.8 still carry `outputDimensionality` to the local GGUF path without applying it to returned vectors or local index identity, so the central memory-storage and reindexing request remains valid.

## What This Changes

The issue requests that `memorySearch.outputDimensionality` truncate vectors produced by local GGUF embedding models, with optional query-instruction support as a bonus.

## Best Possible Solution

Implement local GGUF dimension support through the existing `outputDimensionality` setting, slice local query and batch vectors before normalization/storage, and include the effective dimension in local provider/index identity.

## Reproduction Assessment

Yes. Current source shows the local GGUF path receives `outputDimensionality`, but the worker-backed local provider returns normalized full-length vectors and the local provider identity does not include the dimension.

## Solution Assessment

Yes for the dimensionality portion. Reusing the existing `memorySearch.outputDimensionality` field is narrower than adding `memorySearch.local.truncateDimensions`, while query-instruction support should be tracked separately.
