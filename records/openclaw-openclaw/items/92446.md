## Summary

The revised branch correctly routes file-backed requests through the existing edit capability, but it is not merge-ready because it silently discards `--count`, currently fails lint, and supplies only mocked provider evidence.

## What This Changes

The PR adds a repeatable `--file` option to `image generate` and routes file-backed invocations through the existing `image.edit` capability.

## Best Possible Solution

Keep file-backed requests on the canonical edit capability, reject `--count` when `--file` is present, rename the shadowing variable, add focused regression coverage, and provide redacted real-provider terminal proof.

## Reproduction Assessment

Yes at source level: current main exposes `--file` on `image edit` but not `image generate`, and the branch clearly reproduces the new mode-switch behavior and silent `--count` discard.

## Solution Assessment

No in its current form. Delegating to the edit capability is the correct ownership boundary, but the command must reject unsupported option combinations rather than silently changing their meaning.
