## Summary

Keep open: current main and the latest release still document the embedded Lobster bridge as missing, and the same-author implementation PR is open with unresolved provider-contract, security, and live-proof questions.

## What This Changes

The issue requests host-provided in-process Lobster `llm.invoke` adapters so embedded workflows can call LLM tasks without Gateway URL or bearer-token credentials.

## Best Possible Solution

Resolve the open implementation PR or a replacement by landing a maintainer-approved embedded Lobster LLM bridge that preserves `llm-task` policy, avoids Gateway credential env exposure, documents the provider contract, and includes redacted live invocation proof.

## Reproduction Assessment

Yes. Source and dependency inspection show current main does not populate `ctx.llmAdapters`/`ctx.registry`, while Lobster's in-process `llm.invoke` path depends on those fields and the docs still say the embedded bridge is missing.

## Solution Assessment

Unclear pending maintainer direction. The in-process adapter direction is safer than credential env injection, but provider naming, registry exposure, and preservation of `llm-task` auth/model policy need a maintainer-approved contract.
