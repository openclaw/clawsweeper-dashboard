## Summary

Keep open: current main and the latest release still lack the embedded Lobster `ctx.llmAdapters` bridge, and the open paired implementation pull request still needs maintainer product/security decisions before it can resolve this issue.

## What This Changes

Add host-provided in-process LLM adapters to embedded Lobster so workflows can use `llm.invoke` without a Gateway URL or bearer token.

## Best Possible Solution

Land or reject a maintainer-approved embedded Lobster LLM adapter contract that preserves model/auth policy, avoids provider-name surprises, and never places Gateway operator credentials in workflow env.

## Reproduction Assessment

Yes. Source and dependency inspection show current main does not populate `ctx.llmAdapters`, while Lobster’s `llm.invoke` only uses the in-process path when that context field is provided.

## Solution Assessment

Unclear. The adapter direction is plausible and safer than credential env injection, but the best fix requires maintainer agreement on provider naming, policy preservation, and live-proof expectations.
