## Summary

Keep open: current main still has a source-level path where a trailing cache-TTL custom entry can hide the canonical assistant tail from embedded CLI gap-fill dedupe and allow a duplicate assistant mirror to be appended.

## What This Changes

The issue asks OpenClaw to stop persisting duplicate CLI assistant mirror turns after cache-TTL events so repeated CLI prompts do not poison session context.

## Best Possible Solution

Keep the canonical model assistant turn as the model-facing transcript entry, treat cache-TTL custom rows as non-conversational metadata during embedded gap-fill dedupe, and add focused regression coverage for assistant-plus-cache-TTL tails without globally dropping legitimate CLI backend transcript turns.

## Reproduction Assessment

Yes, at source level: an embedded turn can write the canonical assistant, append `openclaw.cache-ttl`, then have gap-fill miss the preceding assistant because the tail reader returns undefined on the custom row. I did not run a live model loop in this read-only review.

## Solution Assessment

Yes, but narrower than filtering all `api: "cli"` rows: the maintainable fix is to skip metadata-only custom rows for this tail dedupe while still stopping at real user/tool tails and preserving true CLI transcript persistence.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/83427. In current main, embedded CLI gap-fill dedupes against `readTailAssistantTextFromSessionTranscript`, but that reader returns undefined when the physical transcript tail is a `type: "custom"` `openclaw.cache-ttl` entry, so it can append a duplicate CLI assistant mirror after the canonical embedded assistant turn. Patch the narrow transcript/gap-fill boundary so cache-TTL and other metadata-only custom rows do not block dedupe against the immediately preceding assistant, while preserving the existing behavior that a real user/tool tail starts a new turn and preserving real CLI backend transcript persistence. Add regression coverage for assistant -> cache-ttl custom -> embedded gap-fill with identical text, and keep the existing repeated-reply-after-user-tail test passing. Do not globally filter all `api: "cli"` transcript rows and do not change cache-TTL append semantics unless the focused fix proves that is necessary. Changelog is expected because this fixes user-visible CLI/session behavior.
