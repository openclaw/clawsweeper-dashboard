## Summary

Keep open. Current main still has the source-backed cache-ttl/custom-tail path that defeats embedded CLI assistant gap-fill and can persist a duplicate `api: "cli"` assistant row into session history.

## What This Changes

The issue reports duplicate persisted assistant turns after cache-TTL metadata in CLI multi-turn sessions, causing repeated context and model output repetition.

## Best Possible Solution

Teach embedded CLI gap-fill to dedupe across trailing cache-ttl/custom metadata while treating real user turns as boundaries and preserving existing transcript-only mirror semantics.

## Reproduction Assessment

Yes, at source level: cache-ttl appends a custom row after the canonical embedded assistant turn, then gap-fill reads only the physical tail and appends `api: "cli"` when that tail is not an assistant. I did not execute a live multi-turn model session in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to make embedded assistant gap-fill skip trailing metadata such as `openclaw.cache-ttl` without broadly filtering all CLI transcript rows from replay snapshots.
