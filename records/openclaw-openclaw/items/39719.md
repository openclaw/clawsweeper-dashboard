## Summary

Keep open. Current main still has the reported stale skillsSnapshot invalidation gap: persisted snapshots can be reused after a workspace move because reuse only compares an in-memory version and agent skill filter, while neither the runtime nor persisted snapshot records workspace/source-root provenance.

## What This Changes

The issue requests that cached skill paths in persisted sessions be re-resolved or invalidated when the configured workspace moves, so old iCloud SKILL.md paths do not survive in sessions.json.

## Best Possible Solution

Add durable workspace/source-root-aware skillsSnapshot invalidation: store a normalized workspace/source-root identity or stable inputs hash with SkillSnapshot and SessionSkillSnapshot, compare it before every reuse, and rebuild snapshots when the configured workspace or resolved skill roots change.

## Reproduction Assessment

Yes. Persist a session skillsSnapshot with version 0 and an old workspace SKILL.md path, change openclaw.json to a new workspace, restart so the in-memory refresh version is 0, then send on the same session; current main will reuse the stale snapshot when the skill filter still matches.

## Solution Assessment

Yes. The best fix is a generic snapshot input/provenance check rather than an iCloud-specific sessions.json migration, because the same mechanism covers workspace moves and the related post-restart config invalidation gap.

## Repair Work Prompt

Fix stale skillsSnapshot reuse after workspace/source-root changes. Current main reuses persisted session snapshots based only on in-memory version and skillFilter, so moving a workspace from an old iCloud path to a new local path can leave old SKILL.md locations in sessions.json and in the next prompt. Implement a generic durable invalidation mechanism, not an iCloud-specific sed/path rewrite: store a normalized workspace/source-root identity or stable inputs hash with SkillSnapshot and SessionSkillSnapshot, compute it from the current workspace and resolved skill roots/skills config inputs, and refresh snapshots whenever the stored identity/hash differs. Cover related #73765 where skills.entries.* changes after restart can leave stale snapshots. Keep backward compatibility for existing snapshots with no provenance by rebuilding them. Likely files: src/agents/agent-command.ts, src/agents/skills/types.ts, src/config/sessions/types.ts, src/agents/skills/workspace.ts, src/agents/skills/refresh-state.ts, src/gateway/config-reload.ts, plus focused tests. Add or update a user-facing CHANGELOG.md entry because this changes visible skill prompt behavior. Do not broaden into the doctor diagnostics feature from #47534 unless required for the core fix.
