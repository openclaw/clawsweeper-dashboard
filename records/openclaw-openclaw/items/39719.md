## Summary

Keep open. Current main still reuses persisted skill snapshots based only on the in-memory snapshot version and agent skill filter, while the persisted snapshot has no workspace or source-root provenance, so a workspace move can leave old SKILL.md paths in the prompt.

## What This Changes

The issue asks OpenClaw to re-resolve cached user skill paths after a workspace migration so `sessions.json` does not retain stale iCloud SKILL.md locations.

## Best Possible Solution

Persist and compare stable skill snapshot inputs, including normalized workspace/source roots and skill config/filter, before reusing a session `skillsSnapshot`; rebuild when those inputs change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: persist a version-0 snapshot prompt with an old workspace SKILL.md location, change the configured workspace, restart so the in-memory version is 0, and send on the same session; current main will reuse the prompt because version and filter still match.

## Solution Assessment

Yes. The narrow maintainable fix is durable snapshot-input invalidation, not an iCloud-specific path rewrite; that also covers the related stale-cache symptom in #73765.

## Repair Work Prompt

Fix the stale `skillsSnapshot` invalidation bug for this issue and related #73765. Current main reuses `sessionEntry.skillsSnapshot` in `src/agents/agent-command.ts` when only version and skill filter match; persisted snapshots in `src/config/sessions/types.ts` do not record workspace/source-root/config inputs, so workspace migration can leave old SKILL.md paths in the prompt. Add durable snapshot provenance or an inputs hash to `SkillSnapshot`/`SessionSkillSnapshot`, computed from normalized workspace identity, resolved skill roots/enumeration, effective skill config, and agent skill filter. Refresh/rebuild when the computed inputs differ, and treat legacy snapshots without provenance as stale when current inputs are available. Do not special-case iCloud and do not rewrite paths in `sessions.json` as the primary fix. Add regression coverage for workspace migration; include the enabled:false/restart case from #73765 if the same implementation covers it. Add a user-facing changelog entry for the bug fix.
