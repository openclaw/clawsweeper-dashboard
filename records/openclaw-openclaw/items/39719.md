## Summary

Keep open. Current main still reuses persisted `skillsSnapshot` entries when the in-memory version and agent skill filter match, while the persisted snapshot has no workspace or source-root provenance; a workspace migration can therefore keep stale SKILL.md locations in existing session prompts.

## What This Changes

The issue asks for session skill snapshots to be re-resolved or invalidated when an agent workspace moves so persisted SKILL.md paths no longer point at the old iCloud workspace.

## Best Possible Solution

Persist stable skill snapshot input provenance, then rebuild cached session snapshots when normalized workspace/source roots, skill config, filter, or relevant eligibility inputs change.

## Reproduction Assessment

Yes, source-reproducible: persist a version-0 session `skillsSnapshot` with an old workspace SKILL.md path, change the configured workspace, restart so in-memory versions are again 0, then reuse the same session. Current main keeps the old prompt because the version and filter still match.

## Solution Assessment

Yes. Durable snapshot-input invalidation is the narrow maintainable fix; it avoids iCloud-specific rewriting while covering workspace moves and the related stale-cache family.
