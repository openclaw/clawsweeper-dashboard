## Summary

Keep open. Current main still reuses persisted skill snapshots using only the in-memory version and agent skill filter, while the persisted snapshot has no workspace/source-root provenance; a workspace move can still leave stale SKILL.md paths in existing session prompts. The related #73765 is the same cache-invalidation family, but this issue carries the workspace-migration symptom the fix must explicitly cover.

## What This Changes

Invalidate or re-resolve persisted session skill snapshots after workspace migration so sessions.json no longer advertises old iCloud SKILL.md paths.

## Best Possible Solution

Persist and compare stable skill snapshot inputs, including normalized workspace/source roots and skill config/filter, then rebuild cached session snapshots only when those inputs change.

## Reproduction Assessment

Yes, by source inspection: persist a version-0 session skillsSnapshot with an old workspace SKILL.md path, change the configured workspace, restart so in-memory versions return to 0, then reuse the same session. Current main keeps the old prompt because version and filter still match.

## Solution Assessment

Yes. Durable snapshot-input invalidation is the narrow maintainable fix; it avoids iCloud-specific rewriting and should also cover the related stale-cache symptom in #73765.
