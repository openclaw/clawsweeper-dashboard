## Summary

Keep open. Current main still reuses persisted skill snapshots based on version and agent skill filter only, while the saved snapshot has no workspace/source-root provenance, so a workspace move can leave stale SKILL.md locations in the prompt.

## What This Changes

The issue asks OpenClaw to rebuild or re-resolve session skill snapshots after the configured workspace moves so old iCloud skill paths are not reused.

## Best Possible Solution

Persist and compare stable skill snapshot inputs, including normalized workspace/source roots and skill config/filter, before reusing a session skillsSnapshot; rebuild when those inputs change.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: persist a version-0 snapshot prompt with an old workspace SKILL.md location, change the configured workspace, restart so the in-memory version is 0, and send on the same session. Current main will reuse the prompt because version and filter still match.

## Solution Assessment

Yes. The narrow maintainable fix is durable snapshot-input invalidation, not an iCloud-specific path rewrite; that same fix should cover the related stale-cache symptom in #73765.
