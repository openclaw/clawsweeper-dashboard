## Summary

Keep this issue open. Current main still has no WhatsApp `silentErrorReplies`, `suppressAgentErrors`, `errorPolicy`, `errorCooldownMs`, or custom run-failure message config, and the generic direct-chat agent-run failure path remains observable in source and tests. The same author has the open paired implementation candidate #59801, so cleanup should not close this issue before that work lands or is replaced.

## What This Changes

The issue requests a WhatsApp channel option to suppress or customize generic agent-run failure replies, similar to Telegram silent error handling.

## Best Possible Solution

Keep the issue open and resolve it through a WhatsApp-owned config/runtime change: typed root/account error policy fields, strict schema and metadata/docs alignment, runtime suppression or cooldown handling for direct agent-run failures, and focused WhatsApp tests. #59801 is the current candidate to review, rebase, revise, or replace.

Yes. Source and tests provide a high-confidence reproduction path: a WhatsApp direct run failure still resolves to the generic `/new`-style text, and the strict WhatsApp config schema does not accept the requested suppression keys.

## Solution Assessment
