## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the Google Vertex ADC credential marker:** `src/agents/model-auth.ts:345-349`
  - body: Returning oauth mode without an apiKey drops the credential marker that current runtime auth expects. Non-AWS auth modes without a key are rejected before the Google Vertex transport can prepare ADC headers, so ADC-backed users can fail as unauthenticated.
  - confidence: 0.88
- **[P2] Recheck JPEG size after HEIC conversion:** `src/gateway/chat-attachments.ts:103-104`
  - body: This conversion validates only the original HEIC/HEIF payload. HEIC-to-JPEG can expand the bytes, so the branch can return an oversized inline JPEG to the prompt path instead of enforcing the image cap after conversion.
  - confidence: 0.86
- **[P2] Validate denyCommands against the full catalog:** `src/cli/config-cli.ts:354-363`
  - body: resolveNodeCommandAllowlist returns the effective allowlist, not every valid node command. Using it as the known-command catalog reports valid dangerous or capability-gated IDs such as camera.snap or screen.record as unknown.
  - confidence: 0.89
- **[P3] Assert the exact JS placeholder:** `src/auto-reply/reply/export-html/template.security.test.ts:89`
  - body: The new test checks toContain("JS"), which still passes when only MARKED_JS or HIGHLIGHT_JS is present. Match the actual placeholder so the regression test fails if the main script placeholder disappears.
  - confidence: 0.82
