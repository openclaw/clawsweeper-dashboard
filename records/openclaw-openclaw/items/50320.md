## Summary

Keep this PR open. Current main has superseded the export-template part, and the linked Google Vertex issue is already closed, but the PR still tracks two unresolved open regressions: current-run HEIC/HEIF prompt images can still reach the model unchanged, and config set still lacks a targeted gateway.nodes.denyCommands warning path. The proposed branch is not merge-ready because its auth, HEIC, and denyCommands changes introduce concrete correctness/security-review issues.

## What This Changes

This PR adds regression fixes and tests for export-session template placeholder replacement, HEIC/HEIF chat attachment normalization, Google Vertex ADC sentinel handling, and gateway.nodes.denyCommands config-set warnings.

## Best Possible Solution

Keep the useful remaining work but rebase or replace the branch narrowly: preserve current main's export placeholder implementation, normalize HEIC/HEIF inside parseMessageWithAttachments before parsed.images is returned, revalidate converted JPEG size against the existing limits, add config-set denyCommands warnings using a full command catalog, and align any Google Vertex ADC change with the current auth marker/runtime contract.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the remaining HEIC issue: chat.send calls parseMessageWithAttachments, and small inline image attachments are returned as original data plus finalMime. The denyCommands gap is also source-verifiable because config set writes nextConfig without a denyCommands-specific warning path.

## Solution Assessment

No. The PR is pointed at real regressions, but the current branch is not the best fix as-is because it drops Google Vertex ADC credential material, omits a post-conversion HEIC size check, and validates denyCommands against an effective allowlist rather than the full command catalog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep Google Vertex ADC usable:** `src/agents/model-auth.ts:345-349`
  - body: Returning only oauth mode here makes auth resolution report success while downstream non-AWS runtime paths that call requireApiKey still throw because apiKey is undefined. Preserve a runtime-supported ADC marker or update the transport contract instead of dropping the credential material in this resolver.
  - confidence: 0.87
- **[P2] Recheck converted HEIC image size:** `src/gateway/chat-attachments.ts:103`
  - body: This returns the JPEG produced by convertHeicToJpeg without checking its byte length. HEIC to JPEG conversion can expand the payload beyond the configured image budget, so oversized prompt images can still reach the model; mirror the converted-size guard used by normalizeInputImage before returning the image.
  - confidence: 0.9
- **[P2] Validate denyCommands against the full command catalog:** `src/cli/config-cli.ts:354-363`
  - body: resolveNodeCommandAllowlist is the effective allowlist for a platform/config, not the full set of valid node command IDs. Using it as the known-command source flags supported dangerous commands such as camera.snap or screen.record as unknown unless they are also allowed, which makes the new hardening warning misleading.
  - confidence: 0.88
- **[P3] Assert the exact JS placeholder:** `src/auto-reply/reply/export-html/template.security.test.ts:89`
  - body: The new test checks toContain("JS"), which still passes when only MARKED_JS or HIGHLIGHT_JS is present. Assert the actual {{JS}} placeholder form so the test fails if the main script slot is removed or reformatted incorrectly.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff touches provider auth and node command hardening, and both areas have concrete behavioral security concerns that should be resolved before merge.

Concerns:

- **[medium] ADC auth can become a false success:** `src/agents/model-auth.ts:345`
  - body: The PR changes Google Vertex ADC resolution to return oauth mode without apiKey, but existing runtime callers still require usable credential material for non-aws-sdk paths. That can turn a configured provider into a later runtime auth failure while status/resolution appears successful.
  - confidence: 0.86
- **[medium] denyCommands warnings can mislead hardening:** `src/cli/config-cli.ts:354`
  - body: The proposed config warning derives known commands from the effective allowlist, so valid dangerous commands may be reported as unknown when operators intentionally deny them. That undermines the hardening guidance for node command policy.
  - confidence: 0.88
