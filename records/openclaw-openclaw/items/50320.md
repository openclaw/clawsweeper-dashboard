## Summary

Keep open, but this PR is not merge-ready. It targets real linked regressions, and current main has already replaced the export-template and Google Vertex ADC designs, while this branch still introduces concrete auth, attachment-size, and node-hardening warning regressions.

## What This Changes

The PR changes export-session HTML placeholder injection, current-run HEIC/HEIF attachment handling, Google Vertex ADC auth resolution, and config-set warnings for gateway node deny commands, with tests for each area.

## Best Possible Solution

Split the remaining work into focused current-main fixes for #50081 and #50011, preserving the shipped export placeholder and Google Vertex ADC marker designs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the PR-introduced regressions: oauth/no-apiKey fails the current auth controller, converted HEIC images skip the existing post-conversion size guard, and denyCommands warnings use an effective allowlist instead of the full known-command catalog.

## Solution Assessment

No. The requested bugs are valid, but this PR is not the best fix because it mixes stale implementations with newer current-main designs; the maintainable path is narrower current-main repairs for the remaining two open behaviors.
