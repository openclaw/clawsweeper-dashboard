## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the Google Vertex ADC credential marker:** `src/agents/model-auth.ts:345-349`
  - body: Returning `oauth` without an `apiKey` drops the credential marker that current runtime auth and the Google transport use for ADC. Non-`aws-sdk` modes without a key are rejected before the provider transport can run, so ADC-backed Google Vertex calls can fail as unauthenticated.
  - confidence: 0.88
- **[P2] Check converted HEIC payload size before returning it:** `src/gateway/chat-attachments.ts:103`
  - body: This converts HEIC/HEIF data to JPEG and immediately returns it to the prompt path without checking the converted byte length. HEIC-to-JPEG can expand the payload, so this should mirror the existing post-conversion image size guard before sending it to a model.
  - confidence: 0.9
- **[P2] Validate denyCommands against the full command vocabulary:** `src/cli/config-cli.ts:354-363`
  - body: `resolveNodeCommandAllowlist` is the effective allowlist, not the complete set of valid node command IDs. Using it here reports valid dangerous or capability-gated commands such as `camera.snap`, `screen.record`, or talk commands as unknown when operators try to deny them.
  - confidence: 0.89
- **[P3] Assert the exact JS placeholder in the regression test:** `src/auto-reply/reply/export-html/template.security.test.ts:89`
  - body: `toContain("JS")` still passes when only `MARKED_JS` or `HIGHLIGHT_JS` is present. Match the actual `{{JS}}` placeholder form so the test fails if the main script placeholder disappears.
  - confidence: 0.82
