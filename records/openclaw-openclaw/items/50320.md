## Summary

Keep open. The PR targets real linked regressions and current main only partially supersedes it, but the branch still has concrete Google Vertex ADC, HEIC size-budget, and denyCommands validation defects before it can merge.

## What This Changes

The PR changes export-session HTML placeholder replacement, chat attachment HEIC/HEIF handling, Google Vertex ADC auth resolution, and config-set warnings for gateway.nodes.denyCommands, with regression tests.

## Best Possible Solution

Rebase or replace the branch narrowly: keep current main's export placeholder and Google ADC marker/transport design, add HEIC/HEIF normalization with post-conversion size validation, and add config-set denyCommands warnings using the same full known-command semantics as the security audit.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: chat.send calls parseMessageWithAttachments for current-run images, and the PR-introduced ADC and denyCommands defects follow directly from the current auth-controller and node-policy contracts.

## Solution Assessment

No. The PR targets valid regressions, but this patch is not the best fix because it drops required ADC credential material, skips converted-image size validation, and validates denyCommands against an effective allowlist instead of the full command catalog.

  - body: Returning oauth mode without apiKey makes auth resolution look successful, but the embedded runner still rejects non-aws-sdk modes with no credential material and the current Google Vertex transport expects the ADC marker path.   - confidence: 0.89
