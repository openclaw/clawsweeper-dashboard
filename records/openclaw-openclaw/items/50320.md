## Summary

Keep open. The PR targets real linked regressions, but current main has already solved the export placeholder and Google Vertex marker pieces differently while this branch still introduces blocking auth, attachment-size, and denyCommands catalog regressions.

## What This Changes

This PR updates export-session HTML injection, chat attachment HEIC/HEIF handling, Google Vertex ADC auth resolution, and config-set warnings for gateway.nodes.denyCommands, with regression tests.

## Best Possible Solution

Replace or rebase into narrower fixes: keep main's placeholder and Google Vertex marker/transport design, add current-run HEIC normalization with converted-size checks, and validate config-set denyCommands using the full known command catalog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR returns oauth without apiKey for Google Vertex ADC while current runtime rejects that shape, current main still hands inline HEIC prompt images through unchanged, and the PR's denyCommands warning source excludes valid dangerous commands.

## Solution Assessment

No. The PR addresses valid bugs, but this is not the best merge shape because it regresses the current auth contract, skips converted-image size validation, and validates denyCommands against an effective allowlist rather than the full known command catalog.
