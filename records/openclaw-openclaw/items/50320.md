## Summary

Keep open. The PR targets real linked regressions, but it is not safe to merge as-is: current main already solved the export placeholder and Google Vertex ADC surfaces differently, while the PR branch still introduces blocking auth, attachment-size, and denyCommands validation problems.

## What This Changes

This PR changes export-session HTML placeholder replacement, chat attachment HEIC/HEIF handling, Google Vertex ADC auth mode resolution, and config-set warnings for gateway.nodes.denyCommands, with regression tests for each area.

## Best Possible Solution

Rebase or replace the branch narrowly: keep current main's export placeholder and Google Vertex ADC marker/transport design, add HEIC/HEIF normalization in parseMessageWithAttachments with a post-conversion size check, and add config-set denyCommands warnings using the same full known-command semantics as the security audit.

## Reproduction Assessment

Yes. The remaining HEIC issue is source-reproducible through chat.send -> parseMessageWithAttachments -> parsed.images, and the PR-introduced ADC and denyCommands defects are reproducible from the changed resolver/warning logic against the current runtime and audit contracts.

## Solution Assessment

No. The PR addresses valid problems, but it is not the best fix as-is because it drops credential material for a non-AWS auth path, skips converted-image size validation, and validates denyCommands against an effective allowlist instead of the full command catalog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep Google Vertex ADC usable:** `src/agents/model-auth.ts:345-349`
  - body: Returning oauth mode without apiKey makes auth resolution look successful, but the embedded runner still throws for non-aws-sdk modes without credential material. Preserve the current ADC marker/transport contract or update the runtime contract before dropping the marker.
  - confidence: 0.88
- **[P2] Recheck converted HEIC image size:** `src/gateway/chat-attachments.ts:103`
  - body: This returns the JPEG produced by HEIC conversion without checking its byte length. HEIC-to-JPEG conversion can expand the payload beyond the image budget, so mirror the converted-size guard used by normalizeInputImage before returning the prompt image.
  - confidence: 0.9
- **[P2] Validate denyCommands against the full catalog:** `src/cli/config-cli.ts:354-363`
  - body: resolveNodeCommandAllowlist is the effective allowlist, not the full set of valid node command IDs. This flags supported dangerous commands such as camera.snap or screen.record as unknown unless they are also allowed, which makes the hardening warning misleading.
  - confidence: 0.89
- **[P3] Assert the exact JS placeholder:** `src/auto-reply/reply/export-html/template.security.test.ts:89`
  - body: The new test uses toContain("JS"), which still passes when only MARKED_JS or HIGHLIGHT_JS exists. Match the actual {{JS}} placeholder form so the test fails if the main script slot disappears.
  - confidence: 0.82
