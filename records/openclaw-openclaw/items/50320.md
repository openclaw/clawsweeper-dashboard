## Summary

Keep open. The PR targets real regressions and two linked issues still need fixes on current main, but this branch is not mergeable as-is because it introduces Google Vertex ADC auth breakage, omits post-conversion image size enforcement, and validates denyCommands against the effective allowlist instead of the full command catalog.

## What This Changes

The PR changes export-session placeholder injection, HEIC/HEIF prompt-image normalization, Google Vertex ADC auth handling, and gateway.nodes.denyCommands config-set warnings, with regression tests.

## Best Possible Solution

Rebase or replace the branch narrowly: keep main's export placeholder and Google Vertex marker/transport design, normalize current-run HEIC/HEIF prompt images with post-conversion size checks, and add config-set denyCommands warnings using the same full known-command semantics as the security audit.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main still returns inline prompt images without HEIC/HEIF conversion, and the PR-introduced ADC and denyCommands regressions follow directly from the current auth-controller and node-policy contracts.

## Solution Assessment

No. The PR addresses valid bugs, but the proposed implementation is not the narrowest maintainable fix because it drops required ADC credential material, skips converted-image size validation, and validates denyCommands against an effective allowlist rather than the full command catalog.

## Review Findings

Full review comments:

- **[P1] Keep Google Vertex ADC credential material:** `src/agents/model-auth.ts:345-349`
  - body: Returning oauth mode without apiKey drops the current GCP Vertex marker, but auth-controller rejects non-aws-sdk auth with no apiKey before the Google transport can resolve ADC. Preserve the existing marker/prepare-runtime contract or change both sides together.
  - confidence: 0.88
- **[P2] Recheck converted HEIC image size:** `src/gateway/chat-attachments.ts:103`
  - body: This returns the JPEG produced by HEIC conversion without checking its byte length. HEIC-to-JPEG conversion can expand the payload beyond the image budget, so mirror the converted-size guard used by the existing media normalization path before returning the prompt image.
  - confidence: 0.9
- **[P2] Validate denyCommands against the full catalog:** `src/cli/config-cli.ts:354-363`
  - body: resolveNodeCommandAllowlist is the effective allowlist, not every valid node command. This flags supported dangerous commands such as camera.snap or screen.record as unknown unless they are also allowed, which makes the hardening warning misleading.
  - confidence: 0.89
- **[P3] Assert the exact JS placeholder:** `src/auto-reply/reply/export-html/template.security.test.ts:89`
  - body: The new test uses toContain("JS"), which still passes when only MARKED_JS or HIGHLIGHT_JS exists. Match the actual {{JS}} placeholder form so the test fails if the main script slot disappears.
  - confidence: 0.82
