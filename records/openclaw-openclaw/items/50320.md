## Summary

Keep open. The PR addresses real regressions, but current main does not yet include the remaining HEIC current-run normalization or config-set denyCommands warning, and the branch has merge-blocking defects in Google Vertex ADC auth, converted-image size validation, and denyCommands catalog validation.

## What This Changes

The PR changes export-session HTML placeholder replacement, chat attachment HEIC/HEIF handling, Google Vertex ADC auth mode resolution, and config-set warnings for gateway.nodes.denyCommands, with regression tests for each area.

## Best Possible Solution

Rebase or replace the branch narrowly: keep current main's export placeholder and Google ADC marker/transport design, add HEIC/HEIF normalization in parseMessageWithAttachments with post-conversion size validation, and add config-set denyCommands warnings using the same full known-command semantics as the security audit.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: chat.send calls parseMessageWithAttachments for current-run images, and the PR-introduced ADC and denyCommands defects follow directly from the current auth-controller and node-policy contracts.

## Solution Assessment

No. The PR targets valid regressions, but the current patch is not the best fix because it drops required ADC credential material, skips converted-image size validation, and validates denyCommands against an effective allowlist instead of the full command catalog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep Google Vertex ADC usable:** `src/agents/model-auth.ts:345-349`
  - body: Returning oauth mode without apiKey makes auth resolution look successful, but the embedded runner still throws for non-aws-sdk modes with no credential material. Preserve the current ADC marker/transport contract or update the runtime auth contract before dropping the marker.
  - confidence: 0.88
- **[P2] Recheck converted HEIC image size:** `src/gateway/chat-attachments.ts:103`
  - body: This returns the JPEG produced by HEIC conversion without checking its byte length. HEIC-to-JPEG conversion can expand the payload beyond the image budget, so mirror the converted-size guard used by normalizeInputImage before returning the prompt image.
  - confidence: 0.9
- **[P2] Validate denyCommands against the full catalog:** `src/cli/config-cli.ts:354-363`
  - body: resolveNodeCommandAllowlist is the effective allowlist, not every valid node command. This flags supported dangerous commands such as camera.snap or screen.record as unknown unless they are also allowed, which makes the hardening warning misleading.
  - confidence: 0.89
- **[P3] Assert the exact JS placeholder:** `src/auto-reply/reply/export-html/template.security.test.ts:89`
  - body: The new test uses toContain("JS"), which still passes when only MARKED_JS or HIGHLIGHT_JS exists. Match the actual {{JS}} placeholder form so the test fails if the main script slot disappears.
  - confidence: 0.82
