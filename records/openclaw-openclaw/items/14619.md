## Summary

Keep open. Current main still renders the human-readable Tooling prompt text while also sending structured `customTools`, and there is still no scoped config option to suppress only that text.

## What This Changes

Request adds a scoped agent/default config option to omit human-readable tool-list text from the system prompt while preserving structured tool schemas.

## Best Possible Solution

Add an opt-in agent/default flag that suppresses only Tooling bullet text, then update prompt construction, config schema/help/docs, context reporting, and focused regression coverage.

## Reproduction Assessment

Yes for the current behavior by source inspection: current main still builds Tooling bullets from effective tools and separately sends those tools as structured `customTools`. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. A scoped opt-in flag is the narrowest maintainable direction; `systemPromptOverride` replaces the whole prompt and `localModelLean` changes tool availability instead of preserving schemas.
