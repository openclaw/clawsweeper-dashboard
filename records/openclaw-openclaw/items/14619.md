## Summary

Keep open. Current main still renders the human-readable Tooling list and still sends callable tool schemas, while the requested scoped config option is not present.

## What This Changes

Add an opt-in agent default to omit the human-readable Tooling list from generated system prompts while keeping tool schemas available.

## Best Possible Solution

Keep this tracked until OpenClaw has an opt-in flag that suppresses only Tooling bullet text while preserving callable `customTools`, policy guidance, docs, schema/help, and regression coverage.

## Reproduction Assessment

Yes for the reported current behavior by source inspection: current main builds Tooling bullets from effective tool names and separately sends those tools as structured `customTools`. This is a feature request, not a broken existing contract.

## Solution Assessment

Yes, the requested scoped opt-in setting is the narrow maintainable direction; `systemPromptOverride` is too broad, and `localModelLean` removes tools rather than preserving schemas while hiding only the text list.
