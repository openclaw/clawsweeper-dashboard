## Summary

Keep open. Current main still builds and emits the in-band Tooling list from effective tool names, still sends callable tool definitions with descriptions and schemas separately, and still has no scoped config option for suppressing only that prompt text.

## What This Changes

Request an opt-in agent/default setting to omit the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped setting that suppresses only the in-band Tooling bullet list while preserving customTools, prompt-cache stability, diagnostics, config schema/help, docs, and focused regression coverage.

## Reproduction Assessment

Yes for the current implementation state, not as a bug repro. Source inspection shows effective tool names still render into Tooling while structured customTools carry descriptions and parameters, and targeted config search shows no scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup close. The maintainable path is an opt-in prompt-text toggle or an explicit maintainer decision to make the broader tool-overhead tracker canonical for this subtask.
