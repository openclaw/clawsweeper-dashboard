## Summary

Keep open. Current main still emits the human-readable Tooling list from effective tool names and still sends callable tool definitions separately; the new Tool Search surface can compact large catalogs, but it is broader, opt-in, PI-specific behavior and not the requested scoped option to suppress only the prompt text.

## What This Changes

Request to add an agent/default config option that suppresses the human-readable Tooling list in the system prompt while preserving tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only the Tooling bullet list while preserving customTools, context diagnostics, config schema/help/docs, and tests, or explicitly make the broader tool-overhead tracker canonical for this subtask.

## Reproduction Assessment

Yes for the current implementation state. Source inspection shows effective tool names are rendered into Tooling while structured customTools still carry descriptions and parameters, and targeted config search shows no scoped suppression option.

## Solution Assessment

Yes in direction, but it needs maintainer product judgment. The current Tool Search feature is useful overlap for large catalogs, but it changes direct tool exposure instead of providing the narrower requested Tooling-text toggle.
