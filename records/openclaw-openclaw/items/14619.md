## Summary

Keep open. Current main still builds and emits the in-band Tooling list, still sends structured tool definitions separately, and still lacks a scoped config option to suppress only that prompt text; the broader open tool-schema overhead tracker discusses this work but has not superseded this narrower request.

## What This Changes

The issue requests a scoped configuration option to suppress the human-readable Tooling list in the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only the in-band Tooling bullet list, or explicitly fold that acceptance criterion into the broader tool-overhead tracker with matching prompt, config, docs, diagnostics, and tests.

## Reproduction Assessment

Yes for a source-level current-state check, not as a bug repro. Current main still renders effective tool names and summaries into Tooling, still sends structured customTools separately, and lacks a scoped suppression option.

## Solution Assessment

Yes in direction, but not as automatic cleanup. A scoped option is narrower than systemPromptOverride, but maintainers should choose the public config surface and coordinate it with Tool Search and the broader tool-overhead tracker.
