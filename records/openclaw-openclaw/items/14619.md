## Summary

Keep open. Current main still builds and emits the human-readable Tooling list, still passes structured tool definitions separately as customTools, and still has no scoped setting to suppress only the prompt text; the broader tool-overhead tracker references this work but has not made this narrower request obsolete.

## What This Changes

Request to add a scoped agent/default option that omits the human-readable Tooling list from the system prompt while still sending callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only the in-band Tooling bullet list while preserving callable schemas, or explicitly fold that acceptance criterion into https://github.com/openclaw/openclaw/issues/14785 with matching prompt, config, docs, diagnostics, and tests.

## Reproduction Assessment

Yes for a source-level current-state check, not as a bug repro. Current main still renders effective tool summaries into Tooling, still sends structured customTools separately, and lacks a scoped suppression option.

## Solution Assessment

Yes in direction, but not as automatic cleanup. A scoped option is narrower than systemPromptOverride, while maintainers should decide the public config surface and sequencing with https://github.com/openclaw/openclaw/issues/14785.
