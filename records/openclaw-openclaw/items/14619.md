## Summary

Keep open. Current main still builds and emits prompt-side Tooling lines from effective tools, still passes structured tool definitions separately as customTools, and no scoped skip option exists; broader token-overhead trackers remain related but do not supersede this narrower request.

## What This Changes

Request an opt-in configuration setting to omit the human-readable Tooling list from the system prompt while still sending callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, with aligned config schema, docs, context reporting, and focused tests.

## Reproduction Assessment

Yes for a source-level current-main gap check, not as a bug reproduction. Current main still emits Tooling lines, sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as automated cleanup. A scoped opt-in option is narrower than systemPromptOverride and distinct from broader schema reduction, while maintainers still need to decide the public config surface and coordination with related token-overhead work.
