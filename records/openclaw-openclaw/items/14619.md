## Summary

Keep open. Current main still emits prompt-side Tooling lines from effective tools, still sends structured tool definitions separately, and still has no scoped config option to suppress only the human-readable tool list; the broader token-overhead trackers do not clearly supersede this narrower request.

## What This Changes

Request to add a scoped agent/default config option that omits the human-readable Tooling list from the system prompt while keeping callable tool schemas available.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, with aligned config schema, docs, context reporting, and focused tests.

## Reproduction Assessment

Yes for a source-level current-main gap check, not as a bug reproduction. Current main still emits Tooling lines, sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup. A scoped opt-in option is narrower than systemPromptOverride and distinct from broader schema reduction, while maintainers still need to decide the public config surface and coordination with related token-overhead work.
