## Summary

Keep open. Current main still builds and emits the human-readable Tooling list from effective tool names while also registering callable tool definitions, and neither current main nor v2026.6.10 has the requested scoped config option; the broader https://github.com/openclaw/openclaw/issues/14785 overlaps but does not replace this narrower prompt-text request.

## What This Changes

The issue requests a scoped agent/default option to omit the human-readable tool list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Decide the product direction first, then either add a narrow prompt-surface option that suppresses only the Tooling list while preserving customTools, schema diagnostics, docs, and tests, or explicitly route users to Tool Search/localModelLean as the supported token-pressure path.

## Reproduction Assessment

Yes for the source-level behavior: current main still renders toolNames into the Tooling prompt text and separately registers callable customTools. I did not run a live agent turn because this review is read-only.

## Solution Assessment

Unclear: the proposed scoped config option is plausible, but OpenClaw's config-surface bar is high and existing Tool Search/localModelLean cover adjacent token-pressure cases without solving this exact prompt-text duplication.
