## Summary

Keep open. Current main still emits a human-readable Tooling list from effective tool names, still sends callable tool schemas separately, and still lacks a scoped config option for suppressing only that prompt text; the broader open token-overhead tracker does not supersede this narrower request.

## What This Changes

Add an agent/default config option to omit the human-readable Tooling list from the generated system prompt while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only the Tooling bullet list while preserving customTools, context diagnostics, config schema/help/docs, and regression coverage, or explicitly make the broader tracker canonical for this subtask.

## Reproduction Assessment

Yes. Source inspection on current main shows effective tool names are rendered into the Tooling section while structured customTools are still sent separately, and no scoped suppression option exists; this is a feature/config request rather than a broken existing contract.

## Solution Assessment

Yes in direction, but it needs maintainer product judgment. A narrow opt-in setting is more maintainable than systemPromptOverride or localModelLean because it preserves callable schemas and removes only the duplicated prompt text.
