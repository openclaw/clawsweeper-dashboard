## Summary

Keep open. Current main still emits the human-readable Tooling list in generated system prompts, still sends callable tool schemas separately through customTools, and still has no scoped skipToolListText/toolListText config option. The broader open #14785 schema-overhead tracker and the closed duplicate #73786 do not replace this narrower tool-list-text optimization.

## What This Changes

Request to add an opt-in config setting that omits the human-readable tool list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add a resolved agent/default config flag, with per-agent parity if it follows existing scoping cleanly, that suppresses only the per-tool Tooling bullets. Keep the callable schema path intact, update context reporting/docs/schema/help/tests, and leave dynamic discovery or schema compaction to the broader trackers.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: buildAgentSystemPrompt with toolNames still emits Tooling bullets, while splitSdkTools still sends the effective tools as customTools; existing tests also assert the current listing behavior.

## Solution Assessment

Yes. The scoped opt-in flag is the narrowest maintainable solution for this issue; systemPromptOverride is too broad, and #14785 should remain the broader schema-reduction design track.
