## Summary

Keep open. Current main still emits prompt-side Tooling lines, still sends model-facing tool definitions separately, and still has no scoped config option to suppress only the human-readable tool list; the existing Tool Search, Code Mode, local-model lean mode, and systemPromptOverride surfaces are broader opt-in alternatives, not this requested knob.

## What This Changes

Request a scoped agent/default config option to omit human-readable Tooling list text from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, with aligned config schema, docs, diagnostics, and focused tests.

## Reproduction Assessment

Yes for a source-level current-main check, not as a bug reproduction. Current main still emits Tooling lines, sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup. A scoped option is narrower than systemPromptOverride and distinct from broader schema reduction, while maintainers still need to decide the public config surface and coordination with related token-overhead work.
