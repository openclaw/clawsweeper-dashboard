## Summary

Keep open. Current main and the latest release still render human-readable Tooling text from active tool names while separately sending callable tool definitions, and no scoped config option exists to suppress only that prompt text.

## What This Changes

The issue requests a scoped agent/default config option to suppress the human-readable tool list text in the system prompt while still sending callable tool schemas.

## Best Possible Solution

Keep this issue open until maintainers choose a scoped prompt/config contract that suppresses only the human-readable Tooling list, preserves callable schemas, and updates config schema/help, docs, context reporting, and focused tests together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the condition: visible tool names are rendered into Tooling text while the same effective tools are sent as callable customTools with descriptions and parameters; no live run was needed for this source-level check.

## Solution Assessment

Unclear for the proposed config name and placement. A scoped option is plausible, but the best permanent solution needs maintainer product direction because OpenClaw has a high bar for new config surfaces and this overlaps existing Tool Search/local-model mitigation surfaces.
