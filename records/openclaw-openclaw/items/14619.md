## Summary

Keep open. Current main still builds and emits prompt-side Tooling lines, still sends separate structured ToolDefinition payloads, and still has no scoped config option to suppress only the human-readable tool list; the related Tool Search and broader tool-overhead work are adjacent but not equivalent to this narrower request.

## What This Changes

Request adds a config option to omit only the human-readable tool list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, or explicitly fold that acceptance criterion into https://github.com/openclaw/openclaw/issues/14785 with matching prompt, config, docs, diagnostics, and tests.

## Reproduction Assessment

Yes for a source-level current-main check, not as a bug repro. Current main still emits Tooling lines, still sends structured customTools separately, and lacks a scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup. A scoped option is narrower than systemPromptOverride and different from experimental Tool Search, while maintainers still need to decide the public config surface and coordinate broader tool-overhead work.
