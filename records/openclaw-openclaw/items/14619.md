## Summary

Keep open. Current main still builds and emits a human-readable Tooling list from effective tool names, still sends callable tool schemas separately through customTools, and still has no scoped config option that suppresses only that prompt text; the broader open tool-schema tracker does not supersede this narrower request, and the bundled token-flags issue was closed in favor of this issue.

## What This Changes

The issue requests an agent/default config option to omit the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only the Tooling bullet list while preserving customTools, prompt guidance, context diagnostics, config schema/help/docs, and tests, or explicitly fold this subtask into https://github.com/openclaw/openclaw/issues/14785.

## Reproduction Assessment

Yes. Source inspection on current main shows effective tool names are rendered into the Tooling section while structured customTools are sent separately, and targeted config search shows no scoped suppression option; this is a feature/config request rather than a broken existing contract.

## Solution Assessment

Yes in direction, but it needs maintainer product judgment. A narrow opt-in setting is more maintainable than using systemPromptOverride or localModelLean because it preserves callable schemas and removes only the duplicated prompt text.
