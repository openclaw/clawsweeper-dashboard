## Summary

Keep open. Current main still renders the effective tool names and summaries into the in-band Tooling section, still sends structured tool definitions separately, and still lacks a scoped agent/default option for suppressing only that prompt text.

## What This Changes

Add a scoped agent/default option to omit the human-readable Tooling list from the system prompt while still sending callable tool schemas.

## Best Possible Solution

Add an opt-in scoped setting that suppresses only the in-band Tooling bullet list while preserving customTools, prompt-cache stability, diagnostics, config schema/help, docs, and focused regression coverage.

## Reproduction Assessment

Yes for the current implementation state, not as a bug repro. Source inspection shows effective tool names are rendered into Tooling while structured customTools still carry descriptions and parameters, and targeted config search shows no scoped suppression option.

## Solution Assessment

Yes in direction, but this needs maintainer sequencing. The narrow maintainable path is an opt-in prompt-text toggle or an explicit decision to make the broader tool-overhead tracker canonical for this subtask.
