## Summary

Keep open. Current main and the latest tagged release still build prompt-side Tooling lines from the effective tools and still send structured tool definitions separately; the requested scoped suppression option is not present, though current `/context` accounting no longer appears to report those prompt lines as a separate nonzero tool-list bucket.

## What This Changes

Request to add an opt-in agent/default setting that omits the human-readable Tooling tool list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only prompt-side Tooling list lines while preserving structured `customTools`, with aligned config schema, docs, context reporting, and focused prompt/session tests.

## Reproduction Assessment

Yes, as a source-level current-main gap check rather than a bug reproduction. The prompt builder still emits Tooling lines from `toolNames`, the run path still sends structured `customTools`, and no scoped suppression config exists.

## Solution Assessment

Yes in direction, but it needs maintainer product review before implementation. A scoped opt-in setting is narrower than full prompt replacement and distinct from broader schema-reduction work, while existing `localModelLean` and Tool Search mitigations do not suppress this prompt text generally.
