## Summary

Keep open. Current main still duplicates tool information between the Tooling prompt text and structured `customTools` schemas, and no scoped config option exists to suppress only the prompt text.

## What This Changes

Request to add a scoped agent/default config option that omits the human-readable Tooling list from the system prompt while still sending callable tool schemas.

## Best Possible Solution

Add an opt-in agent/default flag that suppresses only per-tool Tooling bullets while preserving structured tool schemas, then update config schema/help/docs/context reporting and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `buildAgentSystemPrompt` renders Tooling bullets from `toolNames`, `buildEmbeddedSystemPrompt` passes effective tools into it, and the run path still sends those tools as `customTools`.

## Solution Assessment

Yes. A scoped opt-in flag is the narrowest maintainable direction; `systemPromptOverride` replaces the whole prompt and `localModelLean` removes tools instead of preserving callable schemas.
