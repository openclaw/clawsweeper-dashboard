## Summary

Keep open. Current main still emits the human-readable Tooling list, still sends structured tool definitions separately, and still lacks a scoped config option to suppress only the prompt text; the broader tool-schema overhead tracker references this work but has not replaced this narrower request.

## What This Changes

Request an opt-in agent/default config option to omit the Tooling list text from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only the in-band Tooling bullet list, or explicitly fold that acceptance criterion into the broader tool-overhead tracker with matching prompt, config, docs, diagnostics, and tests.

## Reproduction Assessment

Yes for a source-level current-state check, not as a bug repro. Current main still renders effective tool summaries into Tooling, still sends structured customTools separately, and lacks a scoped suppression option.

## Solution Assessment

Yes in direction, but not as automatic cleanup. A scoped option is narrower than systemPromptOverride, while maintainers should decide the public config surface and coordinate it with Tool Search, code mode, and https://github.com/openclaw/openclaw/issues/14785.
