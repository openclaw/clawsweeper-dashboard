## Summary

The current source and latest release still show separate prompt-side tool listings and callable tool definitions with no narrow suppression option, so the request remains necessary. The implementation boundary is identifiable, but the repository’s high bar for new config and possible model tool-use quality tradeoffs make this a maintainer product decision rather than an automatic fix.

## What This Changes

Request a scoped option to omit the human-readable tool list from agent system prompts while retaining callable tool schemas.

## Best Possible Solution

Add one narrowly scoped, opt-in agent-default control at the canonical prompt-assembly boundary that omits only prompt-side tool listings, preserves normal callable tool definitions, updates context reporting and public docs, and is validated across representative providers before any default change.

## Reproduction Assessment

Yes, at source level: current main builds human-readable tool lines into the system prompt while separately preparing callable tool definitions, and the same behavior is present in v2026.6.5. This is a feature request rather than a failing existing contract, so no runtime failure reproduction applies.

## Solution Assessment

Unclear. A narrow opt-in is more maintainable than code mode or full prompt replacement, but maintainers should approve the config shape and require real-model tool-use comparisons before deciding that schemas alone are sufficient guidance.
