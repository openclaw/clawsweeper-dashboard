## Summary

Keep open. Current main still renders the human-readable Tooling list while also sending callable tool schemas, and there is still no scoped config option to suppress only that prompt text; related broader token-overhead work has not landed or made this request obsolete.

## What This Changes

Request adding an agent/default config option to omit human-readable tool-list text from the system prompt while keeping tool schemas available for function calling.

## Best Possible Solution

Add an opt-in scoped agent/default setting that removes only the rendered Tooling list while preserving `customTools`, and update prompt construction, config schema/help/docs, context diagnostics, and focused tests.

## Reproduction Assessment

Yes. Current-main source inspection shows effective tool names are rendered into `## Tooling` while structured `customTools` are still sent separately; this is a feature/config request rather than a broken existing contract.

## Solution Assessment

Yes in direction, but it needs maintainer coordination. A scoped opt-in is narrower than `systemPromptOverride` or `localModelLean`, while the exact naming/defaults should align with the broader tool-schema optimization work.
