## Summary

Keep open. Current main still emits the human-readable Tooling prompt text from runtime tool names, still sends the same tools as structured `customTools`, and still has no scoped config option to suppress only that text.

## What This Changes

Requests a scoped agent/default option to omit the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in agent/default flag that suppresses only the Tooling bullet text while preserving `customTools`, then update config schema/help/docs, context reporting, and focused tests.

## Reproduction Assessment

Yes for the reported current behavior by source inspection: current main builds Tooling bullets from effective tools and separately sends those tools as structured `customTools`. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. A scoped opt-in flag is narrower than `systemPromptOverride` and preserves callable schemas unlike `localModelLean`, which changes actual tool availability.
