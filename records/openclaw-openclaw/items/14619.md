## Summary

Keep open. Current main and the latest shipped release still render prompt-side Tooling lines from effective tools, still send structured tool definitions separately, and still do not expose a scoped option to suppress only the human-readable tool list.

## What This Changes

Request adds an opt-in agent/default config option to omit the human-readable tool list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, with aligned config schema, docs, context reporting, and focused tests.

## Reproduction Assessment

Yes for a source-level current-main gap check, not as a bug reproduction. Current main still emits Tooling lines, sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as automated cleanup. A scoped opt-in option is narrower than systemPromptOverride and distinct from broader schema reduction, while maintainers still need to decide the public config surface.
