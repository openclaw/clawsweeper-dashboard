## Summary

Keep open. Current main still has the exact duplicated surfaces described here: prompt-side Tooling text is generated from runtime tools, structured tool definitions are still sent separately, and there is no scoped config option to suppress only the human-readable list; the broader open tool-overhead tracker is related but does not replace this narrower request.

## What This Changes

Request an agent/default config option to omit the human-readable Tooling list from the system prompt while keeping callable tool schemas available.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, or explicitly fold that acceptance criterion into the broader tool-overhead tracker with matching config, docs, diagnostics, and tests.

## Reproduction Assessment

Yes for a source-level current-main check, not as a bug repro. Current main still emits Tooling lines, still sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup. A scoped option is narrower than systemPromptOverride and different from experimental Tool Search, while maintainers still need to decide the public config surface and coordinate broader tool-overhead work.
