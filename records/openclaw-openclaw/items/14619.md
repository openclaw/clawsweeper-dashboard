## Summary

Keep open. Current main still emits the human-readable Tooling list from effective tool names and still sends structured tool definitions separately; the new Tool Search mode is useful related work, but it is an opt-in broader discovery mode, not the requested scoped option to suppress only the Tooling text.

## What This Changes

Request to add a scoped config option that omits the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only the Tooling bullet list while preserving customTools, prompt behavioral guidance, context diagnostics, generated config schema/help/docs, and focused prompt/tool-call regression coverage, or explicitly fold this exact subtask into the broader open tool-overhead tracker after maintainer review.

## Reproduction Assessment

Yes for the current implementation state. Source inspection shows effective tool names are still rendered in the Tooling section while structured customTools are sent separately, and targeted config search shows no scoped suppression option; this is a feature/config request rather than a broken existing contract.

## Solution Assessment

Yes in direction, but not ready for automation. A narrow opt-in setting is cleaner than systemPromptOverride and narrower than Tool Search, but maintainers need to decide defaults and provider/tool-selection risk before implementation.
