## Summary

Keep open. Current main still builds and emits prompt-side Tooling lines, still sends model-facing tool definitions separately, and still lacks a scoped config option to suppress only the human-readable tool list; existing Tool Search/code mode mitigations are broader opt-in surfaces, not this requested knob.

## What This Changes

Request an opt-in agent/default setting that omits the human-readable system-prompt tool list while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, with aligned config schema, docs, diagnostics, and focused tests.

## Reproduction Assessment

Yes for a source-level current-main check, not as a bug reproduction. Current main still emits Tooling lines, still sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup. A scoped option is narrower than systemPromptOverride and distinct from broader schema reduction, while maintainers still need to decide the public config surface and coordination with related token-overhead work.
