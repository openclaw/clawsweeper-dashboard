- `impact:session-state`: The duplicated tool catalog consumes the model context assembled for each affected agent session.

## Summary

Current main and the latest release still expose the effective tool catalog twice: concise names/descriptions in the OpenClaw-owned system prompt and callable JSON definitions in the model tool payload. Tool Search and local-model lean mode reduce the direct catalog but do not provide the requested tool-list-text suppression, so the issue remains useful; however, OpenClaw’s high bar for new config surfaces and the model-reliability tradeoff require a maintainer product decision rather than an autonomous fix.

## What This Changes

Requests a scoped option to omit the human-readable tool list from agent system prompts while retaining callable tool schemas.

## Best Possible Solution

Adopt one canonical prompt-overhead policy that can omit redundant tool prose when model/runtime evidence shows it is safe, while preserving Tool Search instructions, context diagnostics, and reliable tool invocation; expose a config knob only if automatic capability policy cannot cover the needed operator control.

## Reproduction Assessment

Yes at source level: current main builds a human-readable tool list and separately supplies callable tool definitions. The duplicate surfaces are also documented by `/context`; this read-only review did not run a live token benchmark.

## Solution Assessment

Unclear. Scoped suppression is maintainable in isolation, but the best permanent design depends on whether maintainers want explicit operator control, automatic model/provider capability policy, or integration with the existing local-model lean and Tool Search strategy.
