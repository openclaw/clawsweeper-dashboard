## Summary

Keep open. Current main still emits the human-readable Tooling section, still sends structured tool definitions separately, and still lacks a scoped config option to suppress only the prompt-side tool list; the broader tool-schema tracker references this work but has not superseded this narrower request.

## What This Changes

Add an opt-in agent/default setting to omit the system-prompt tool list text while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in setting for prompt-side Tooling text, or explicitly fold that acceptance criterion into https://github.com/openclaw/openclaw/issues/14785 with matching prompt, config, docs, diagnostics, and tests.

## Reproduction Assessment

Yes for a source-level current-state check, not as a bug repro. Current main still emits Tooling summaries, sends structured customTools separately, and has no scoped suppression option.

## Solution Assessment

Yes in direction, but not as automatic cleanup. A scoped option is narrower than systemPromptOverride, while maintainers still need to decide the public config surface and coordinate it with the broader tool-overhead tracker.
