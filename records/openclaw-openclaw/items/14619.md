## Summary

Keep open. Current main and the latest release still render prompt-side Tooling lines from effective tools while sending structured tool definitions separately, and there is still no scoped config option to suppress only that prompt text.

## What This Changes

Request adds an opt-in agent/default setting to omit the human-readable tool list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only prompt-side Tooling list lines while preserving structured customTools, with aligned config schema, docs, context reporting, and focused prompt/session tests.

## Reproduction Assessment

Yes, as a source-level current-main gap check rather than a bug reproduction. The prompt builder still emits Tooling lines from toolNames, the run path still sends structured customTools, and no scoped suppression config exists.

## Solution Assessment

Yes in direction, but it needs maintainer product review before implementation. A scoped opt-in setting is narrower than full prompt replacement and distinct from broader schema-reduction work, while existing token-saving surfaces do not generally suppress this prompt text.
