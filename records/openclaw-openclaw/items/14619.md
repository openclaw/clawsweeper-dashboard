## Summary

Keep open. Current main still builds and emits the human-readable Tooling section from the effective tool set, still sends callable tool definitions separately, and has no scoped config option to suppress only that prompt text; the broader token-overhead tracker does not supersede this specific request.

## What This Changes

Request to add a scoped agent/default config option that omits the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in setting that suppresses only the Tooling bullet list, keeps customTools intact, updates config schema/help/docs/context reporting, and adds focused regression coverage.

## Reproduction Assessment

Yes for the current implementation state by source inspection: main still builds Tooling prompt bullets from effective tool names and separately passes structured customTools. This is a feature/config request, not a broken existing contract.

## Solution Assessment

Yes in direction: a scoped prompt-text suppression is narrower than replacing the whole system prompt, and safer than localModelLean because it should not remove callable tools. The exact user-facing config contract still needs maintainer judgment.
