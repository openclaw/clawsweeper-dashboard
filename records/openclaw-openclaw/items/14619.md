## Summary

Keep open. Current main still renders effective tools into the in-band Tooling section and still sends callable tool definitions separately, while no scoped `skipToolListText` or equivalent agent/default setting exists.

## What This Changes

The issue requests an agent/default configuration option to omit only the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped setting that suppresses only the in-band Tooling bullet list while preserving `customTools`, diagnostics, docs/config schema, and prompt-cache stability, or explicitly make the broader tool-overhead tracker canonical for this subtask.

## Reproduction Assessment

Yes for the current implementation state. Source inspection shows effective tool names are rendered into Tooling while structured `customTools` still carry descriptions and parameters, and targeted config search shows no scoped suppression option.

## Solution Assessment

Yes in direction, but it needs maintainer product sequencing. The narrow maintainable solution is a scoped opt-in prompt-text toggle; Tool Search is useful overlap for large catalogs but changes direct tool exposure instead of solving this exact duplication.
