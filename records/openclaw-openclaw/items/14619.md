## Summary

Keep open. Current main still renders the Tooling section from effective tool names and separately sends callable tool definitions, and there is still no scoped config option to suppress only that prompt text.

## What This Changes

The issue requests an agent/default setting that omits the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in agent/default setting that suppresses only the Tooling bullet list while preserving callable customTools, policy guidance, schema/help/docs, context reporting, and regression tests, or explicitly make the broader token-overhead tracker canonical.

## Reproduction Assessment

Yes for the current implementation state by source inspection: current main builds Tooling bullets from effective tool names and separately sends structured customTools. This is a feature/config request, not a broken existing contract.

## Solution Assessment

Yes in direction but not yet in product details: a scoped opt-in setting is narrower than `systemPromptOverride`, and `localModelLean` removes tools rather than preserving schemas while hiding only the text list. Maintainers still need to choose the exact config semantics before implementation.
