## Summary

Keep open. Current main still has the duplicated surfaces described here: a human-readable Tooling section is assembled from effective tool names, callable tool definitions are still passed separately, and no scoped option exists to suppress only the prompt text.

## What This Changes

Add an agent/default config option to omit the human-readable tool list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only the Tooling bullet list while preserving callable customTools, context reporting, config schema/help/docs, and regression coverage, or explicitly make the broader token-overhead tracker canonical for this subtask.

## Reproduction Assessment

Yes for the current implementation state by source inspection: current main builds Tooling bullets from effective tool names and separately sends structured customTools. This is a feature/config request, not a broken existing contract.

## Solution Assessment

Yes in direction, but not ready for automation: a scoped opt-in setting is narrower than `systemPromptOverride`, while `localModelLean` removes tools instead of keeping schemas and hiding only the text list.
