## Summary

Keep open. Current main and v2026.6.10 still emit human-readable Tooling lines from effective tool names while also registering callable tool schemas separately, and there is no scoped skipToolListText/toolListText config option; the broader https://github.com/openclaw/openclaw/issues/14785 overlaps but does not replace this narrower prompt-text request.

## What This Changes

Request to add an agent/default option that suppresses the human-readable Tooling tool-list text in the system prompt while preserving callable API tool schemas.

## Best Possible Solution

Decide the product direction, then either add a narrow opt-in prompt-surface setting that suppresses only the Tooling list while preserving customTools, diagnostics, docs, and tests, or explicitly route users to Tool Search/localModelLean as the supported token-pressure path.

## Reproduction Assessment

Yes for the current behavior: source inspection shows toolNames are rendered into Tooling prompt text and the same effective tools are registered as callable customTools. I did not run a live agent turn because this review is read-only.

## Solution Assessment

Unclear: the proposed scoped config option is plausible, but OpenClaw's config-surface bar is high and existing Tool Search/localModelLean cover adjacent token-pressure cases without solving this exact prompt-text duplication.
