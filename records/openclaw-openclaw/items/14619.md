Keep open. ## What This Changes

The issue requests an opt-in agent/default config setting to omit the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, with aligned config schema, docs, diagnostics, and focused tests.

## Reproduction Assessment

Yes for a source-level current-main check, not as a bug reproduction. Current main still emits Tooling lines, still sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup. A scoped option is narrower than systemPromptOverride and different from Tool Search/code mode, while maintainers still need to decide the public config surface and coordination with broader tool-overhead work.
