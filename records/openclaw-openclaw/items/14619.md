## Summary

Keep open. Current main still renders effective tool names into the in-band Tooling section and still sends callable tool definitions separately; Tool Search overlaps with large-catalog compaction, but it is not the requested scoped option to suppress only the prompt text.

## What This Changes

The issue requests a scoped agent/default option to omit the human-readable Tooling list from the system prompt while still sending callable tool schemas.

## Best Possible Solution

Add an opt-in scoped setting that suppresses only the in-band Tooling bullet list while preserving customTools, diagnostics, config schema/help/docs, and prompt-cache stability, or explicitly make the broader tool-overhead tracker canonical for this subtask.

## Reproduction Assessment

Yes for the current implementation state. Source inspection and existing tests show effective tool names are rendered into the Tooling prompt text while structured customTools still carry descriptions and parameters, and targeted config search shows no scoped suppression option.

## Solution Assessment

Yes in direction, but it needs maintainer product judgment. The narrow maintainable solution is a scoped opt-in prompt-text toggle; Tool Search is useful overlap for large catalogs but changes direct tool exposure instead of solving this exact duplication.
