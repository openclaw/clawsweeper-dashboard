## Summary

Keep open. Current main still emits prompt-side Tooling lines, sends structured ToolDefinition schemas separately, and lacks a scoped config option to suppress only the human-readable tool list; Tool Search and the broader tool-overhead tracker are adjacent but not a replacement for this narrower request.

## What This Changes

Request an opt-in config setting to omit the human-readable Tooling list from the system prompt while preserving structured tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default setting that suppresses only prompt-side Tooling list lines while preserving callable schemas, or explicitly fold that acceptance criterion into the broader tool-overhead tracker with matching config, docs, diagnostics, and tests.

## Reproduction Assessment

Yes for a source-level current-main check, not as a bug repro. Current main still emits Tooling lines, still sends structured customTools separately, and lacks the requested scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automated cleanup. A scoped option is narrower than systemPromptOverride and different from experimental Tool Search, while maintainers still need to decide the public config surface and coordinate the broader tool-overhead work.
