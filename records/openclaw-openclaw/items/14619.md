## Summary

Keep open. Current main still builds a human-readable Tooling list from the effective tools, still sends those tools separately as structured `customTools`, and still has no scoped config option to suppress only the prompt text.

## What This Changes

The issue requests an opt-in agent/default setting that omits the human-readable tool list from the system prompt while still sending callable tool schemas.

## Best Possible Solution

Add a scoped opt-in agent/default flag that suppresses only the per-tool Tooling bullets while preserving structured tool schemas, then update config schema/help/docs/context reporting and focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `buildAgentSystemPrompt` emits Tooling bullets from `toolNames`, the embedded runner passes effective tool names into it, and the same tools are still sent as `customTools`.

## Solution Assessment

Yes. A scoped opt-in flag is the narrowest maintainable direction; `systemPromptOverride` is too broad, `localModelLean` changes actual tool availability, and #14785 should remain the broader schema-overhead track.
