## Summary

Keep open. Current main still emits the in-band Tooling list from effective tool names, still sends structured customTools with descriptions and schemas, and still has no scoped config option to suppress only the prompt text. The broader tool-overhead tracker remains relevant, but the live discussion explicitly preserves this issue as the narrower tool-list-text request unless maintainers choose otherwise.

## What This Changes

The issue requests an agent/default config option to omit the human-readable Tooling list from the system prompt while preserving callable tool schemas.

## Best Possible Solution

Add an opt-in scoped agent/default setting that suppresses only the in-band Tooling bullet list, or explicitly fold that subtask into the broader canonical tracker with matching acceptance criteria.

## Reproduction Assessment

Yes for a source-level current-state check, not as a bug repro. Current main still renders effective tool names and summaries into Tooling, still sends structured customTools separately, and lacks a scoped suppression option.

## Solution Assessment

Yes in direction, but not as an automatic cleanup fix. The maintainable solution needs a public config contract plus prompt, diagnostics, docs, and tests coordinated with Tool Search and the broader overhead tracker.
