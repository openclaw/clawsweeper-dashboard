## Summary

Keep open. Current main still lacks a `workspaceLifecycle` sandbox config/schema/runtime/docs path, and the linked implementation at https://github.com/openclaw/openclaw/pull/73738 remains open and unmerged, so this issue should track that PR through maintainer/secops review.

## What This Changes

Request an opt-in ephemeral or tmpfs-backed sandbox workspace lifecycle so isolated sessions can start from a clean per-run filesystem instead of reusing per-scope workspace state.

## Best Possible Solution

Land or close the linked PR after maintainer/secops review confirms a persistent default, run-owned ephemeral paths, bounded cleanup, aligned schema/docs/tests, and clear wording around tmpfs versus disposable lifecycle.

## Reproduction Assessment

Yes for the current feature gap by source inspection: current main has no `workspaceLifecycle` schema/type/runtime/docs support, and sandbox workspaces are still stable per scope. This is not a broken existing contract; it is a requested new lifecycle mode.

## Solution Assessment

Yes for keeping this issue open while https://github.com/openclaw/openclaw/pull/73738 is reviewed. The narrow maintainable direction is an opt-in lifecycle setting with persistent defaults and secops-approved cleanup boundaries, not a separate cleanup-only workaround.
