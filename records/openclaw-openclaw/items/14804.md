## Summary

Keep open. Current main still lacks first-class ephemeral sandbox workspace lifecycle support, and open PR #73738 uses closing syntax for this issue, so the safe path is maintainer/secops review of that PR rather than cleanup.

## What This Changes

Add an opt-in ephemeral or tmpfs-backed sandbox workspace lifecycle for isolated sessions so each run starts clean and does not persist workspace changes.

## Best Possible Solution

Review and either land or close PR #73738 after confirming an opt-in lifecycle contract, persistent defaults, bounded cleanup of run-owned paths only, aligned schema/docs/tests, and a clear tmpfs scope decision.

## Reproduction Assessment

Yes for the current-main feature gap by source inspection: the config type and strict schema have no workspaceLifecycle key, and sandbox workspace resolution still uses stable per-scope directories.

## Solution Assessment

Unclear until PR #73738 is reviewed. The maintainable direction is an opt-in lifecycle setting with persistent defaults and tightly bounded cleanup semantics; the safer next step is review of the linked PR, not a separate repair.
