## Summary

Keep open. Current main still lacks first-class ephemeral sandbox workspace lifecycle support, and open PR #73738 already references this issue with closing syntax, so cleanup should wait for maintainer/secops review and merge outcome.

## What This Changes

Request an opt-in ephemeral or tmpfs-backed sandbox workspace mode so isolated sessions start from disposable state and clean up after each run.

## Best Possible Solution

Add an opt-in sandbox workspace lifecycle contract with persistent as the default, run-owned ephemeral paths, bounded cleanup, aligned schema/docs/tests, and explicit tmpfs scope wording.

## Reproduction Assessment

Yes for the current-main feature gap by source inspection: the config types/schema reject a workspaceLifecycle key, and sandbox workspace resolution still derives stable scope-based directories. No live run was needed to show the missing user-facing config option.

## Solution Assessment

Unclear until PR #73738 receives maintainer/secops review. The maintainable direction is an opt-in lifecycle setting with persistent defaults and tightly bounded cleanup, not a separate cleanup-only workaround.
