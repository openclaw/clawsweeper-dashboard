## Summary

Keep open. Current main still lacks first-class ephemeral or tmpfs-backed sandbox workspace lifecycle support, and open PR #73738 directly references this issue with closing syntax, so the issue should stay open pending maintainer/secops review of that PR.

## What This Changes

The issue requests opt-in ephemeral or tmpfs-backed sandbox workspaces so isolated sessions do not reuse persistent per-scope workspace state.

## Best Possible Solution

Review and either land or close PR #73738 after confirming an opt-in lifecycle contract, persistent backwards-compatible defaults, cleanup constrained to run-owned ephemeral paths, aligned config/schema/docs/tests, and an explicit decision on tmpfs scope.

## Reproduction Assessment

Yes for the current-main feature gap. The focused source check is that workspaceLifecycle is absent from the config type and strict schema while sandbox workspace resolution still derives stable per-scope directories that preserve existing contents.

## Solution Assessment

Unclear until PR #73738 is reviewed. The maintainable direction is an opt-in lifecycle setting with persistent defaults and tightly bounded cleanup semantics; the safer next step is review of the linked PR, not a separate repair.
