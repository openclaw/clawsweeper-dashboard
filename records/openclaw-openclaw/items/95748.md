## Summary

Keep open: the PR targets a real cron/session failure, but the patch is not merge-ready because it retries a session-takeover safety error broadly and removes the empty-compaction boundary that current main added to prevent the closed cron compaction loop regression.

## What This Changes

The PR changes empty-conversation compaction from writing a boundary to returning cancel, adds a bounded cron retry for EmbeddedAttemptSessionTakeoverError, updates focused compaction tests, and adds two repro/proof files under scripts/repro.

## Best Possible Solution

Preserve the session-fence fail-fast boundary, handle zero-real-message compaction through an owned no-op/boundary or preparation-level skip that still blocks re-triggering, add focused regression coverage, and require redacted real cron/runtime proof before merge.
