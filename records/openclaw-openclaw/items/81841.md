## Summary

## What This Changes

This PR changes the embedded runner so signed-thinking/reasoning-only turns do not enter the silent `NO_REPLY` path and zero-output `stopReason=error` retries receive `EMPTY_RESPONSE_RETRY_INSTRUCTION`, with focused regression tests.

## Best Possible Solution

Land a narrow runner fix that retries signed-thinking-only failures with model-visible feedback, preserves intentional `NO_REPLY` silence, updates the silent-reply docs if the contract changes, and includes real runtime proof.

## Reproduction Assessment

Yes, at source level. Current main’s mocked runner path and helper tests show reasoning-only turns can be treated as silent when `allowEmptyAssistantReplyAsSilent` is true, and the empty-error branch retries without injecting a continuation instruction; I did not run tests in this read-only review.

## Solution Assessment

Mostly yes for the runner mechanics: the PR reuses the existing retry-instruction path and keeps the side-effect guard on empty-error retries. It is not merge-ready until real behavior proof is added and the silent-reply documentation is aligned with the new policy.
