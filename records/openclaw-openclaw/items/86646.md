## Best Possible Solution

Block stream fallback whenever a renderable terminal payload was skipped as a duplicate, add focused regression coverage for final and aborted stale-stream cases, then provide redacted WebChat or Gateway proof of the race after the patch.

## Solution Assessment

No; the tail-signature idea is the right layer for this controller race, but the implementation should not let a duplicate terminal payload fall through to stream fallback.
