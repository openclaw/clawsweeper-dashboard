## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Check the tail role before suppressing final messages:** `ui/src/ui/controllers/chat.ts:1191-1194`
  - body: `isTailDuplicate` only compares `content`. After a normal send, the tail can be the user's own message; if the assistant final echoes that same content, this treats the assistant reply as a duplicate and skips the append, so with no current stream the reply disappears. Require an assistant-tail/signature match before deduping terminal payloads.
  - confidence: 0.92
- **[P1] Handle duplicate terminals without materializing stream text:** `ui/src/ui/controllers/chat.ts:1267`
  - body: When `isTailDuplicate(...)` is true, the append branch is skipped and execution enters the unchanged `else`, which calls `materializeVisibleAssistantStreamMessages`. That still appends whatever `chatStream` holds, so a duplicate final/aborted payload can materialize stale or partial text instead of being treated as already visible. Split the duplicate case from the no-message fallback and cover both final and aborted stale-stream cases.
  - confidence: 0.95
