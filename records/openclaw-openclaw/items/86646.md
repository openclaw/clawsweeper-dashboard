## Summary

## What This Changes

The PR adds a `messageDisplaySignature`-based tail duplicate guard to WebChat `handleChatEvent` assistant final/aborted append paths and adds controller tests for those guarded cases.

## Best Possible Solution

Keep the tail-dedup direction, but only suppress finals when there is a stronger same-transcript/history-catch-up signal and prove the result in a real WebChat or gateway run.

## Reproduction Assessment

Yes for the review finding: the PR merge ref makes an other-run final matching the assistant tail skip the append, and the added test asserts that behavior. The original intermittent WebChat race is described in the linked issue but was not live-reproduced here.

## Solution Assessment

No. The controller is the right surface, but display-text-only dedupe is too broad; the safer fix is to dedupe only when the final event is proven to be the same transcript/history entry and to preserve legitimate repeated same-text messages.
