## Summary

Keep open: this is a focused WebChat bug fix in the right area, but the latest branch still allows stale stream fallback after skipping a duplicate visible final/aborted payload, and the external PR lacks real WebChat/gateway proof.

## What This Changes

The PR adds a WebChat controller tail-duplicate guard for same-run final, aborted, and stream-fallback assistant messages plus focused unit tests for duplicate and cross-run cases.

## Best Possible Solution

Restructure terminal handling so visible final/aborted payloads are considered consumed even when duplicate-skipped, use stream fallback only when no renderable terminal payload exists, add the divergent-stream regression, then supply real WebChat/gateway proof.

- **[P2] Keep duplicate finals out of the stream fallback:** `ui/src/ui/controllers/chat.ts:787-798`
  - body: When `finalMessage` is visible but already matches the chat tail, this condition becomes false and execution falls into the `chatStream` fallback.
