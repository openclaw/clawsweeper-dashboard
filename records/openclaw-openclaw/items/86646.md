## Summary

Keep this PR open, but not merge-ready: the tail-duplicate guard is aimed at the right WebChat race, yet the current branch can still fall through to append stale partial stream text after skipping a duplicate visible terminal payload, and the supplied proof is source/unit-only rather than a real WebChat/gateway run.

## What This Changes

The PR adds a WebChat controller tail-duplicate guard for same-run final, aborted, and stream-fallback assistant messages, plus focused controller tests.

## Best Possible Solution

Restructure terminal handling so visible final/aborted payloads are treated as consumed even when duplicate-skipped, use stream fallback only when no renderable terminal payload exists, add the divergent-stream regression, then supply real WebChat/gateway proof.

## Reproduction Assessment

Yes from source inspection: on PR head, chatMessages can already contain the full final while chatStream holds older partial text, and the duplicate final skip can fall into the stream fallback and append that partial.

## Solution Assessment

No: the tail-duplicate approach is narrow, but the implementation should separate visible terminal payload handling from missing-payload stream fallback before merge.

- **[P2] Keep duplicate finals out of the stream fallback:** `ui/src/ui/controllers/chat.ts:787-798`
