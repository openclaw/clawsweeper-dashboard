## What This Changes

The PR adds a WebChat `handleChatEvent` tail-duplicate helper, guards assistant final/aborted/stream-fallback appends, and adds controller tests for those paths.

## Best Possible Solution

Land a controller-layer dedup fix only after redacted real WebChat proof confirms the duplicate race is gone and maintainers accept tail-signature coalescing for this append path.

## Solution Assessment

Is this the best way to solve the issue? Mostly yes for the controller layer: a tail duplicate check is narrow and local, but merge should wait for real behavior proof and maintainer acceptance of identical-message coalescing.
