## Summary

Keep open: current main has related WebChat stream reconciliation, but it does not fully cover the history/final duplicate race, and this PR's current guard can suppress legitimate terminal replies or still materialize stale stream text.

## What This Changes

The PR adds a WebChat `handleChatEvent` tail-content duplicate guard for final/aborted terminal events plus controller tests for duplicate and cross-run cases.

## Best Possible Solution

Limit dedupe to the active-run history race with assistant-tail or persisted-replacement proof, handle duplicate terminals without stream fallback, and require refreshed WebChat/Gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection shows a selected-session final/aborted event or a user-tail echo with matching content can hit the new duplicate guard without being the history-load race; a duplicate terminal with stale `chatStream` can also fall through to stream materialization.

## Solution Assessment

No. The controller-local guard is a plausible direction, but the best fix must scope dedupe to proven active-run persisted replacements and handle duplicate terminals as fully handled instead of using the generic fallback branch.
