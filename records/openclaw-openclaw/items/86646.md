## Summary

Keep open: this PR still targets the open WebChat final-event/history race and is not fully superseded by current main or the merged stream-reconciliation work, but the current patch is not merge-ready because the new duplicate guard compares only `content` and can drop a legitimate assistant echo when the user tail has the same content.

## What This Changes

The branch adds a WebChat controller tail-duplicate guard for same-run final/aborted terminal payloads plus focused tests for duplicate terminal and stale stream fallback cases.

## Best Possible Solution

Keep the focused WebChat controller fix direction, but make duplicate detection assistant-tail and role-aware, add the user-echo regression case, and provide redacted live WebChat proof before merge.

## Reproduction Assessment

Yes from source for the patch defect: a `ChatState` ending in a user message with the same `content` as the assistant final will satisfy `isTailDuplicate` and skip the assistant append. The original intermittent browser race remains source-supported but was not live-reproduced here.

## Solution Assessment

No. The controller is a plausible repair boundary, but the current helper is too broad; the best fix is a role-aware or assistant-tail-only guard aligned with the existing stream-reconciliation helpers.
