## Summary

Keep this PR open: the patch looks narrowly scoped and I found no concrete line-level correctness defect, but it intentionally changes extension registration from fail-fast to per-tool quarantine and the PR is draft with a protected maintainer label, so it needs human maintainer policy review before merge.

## What This Changes

The PR snapshots extension tool descriptors during registration, skips unreadable registered tools with diagnostics, propagates those diagnostics through path-based and inline extension loading, and adds focused regression coverage.

## Best Possible Solution

Land this only after maintainers confirm that per-tool quarantine with visible diagnostics is the desired extension registration contract.

## Reproduction Assessment

## Solution Assessment

Yes, with a policy caveat. Snapshotting at registration is the narrow owner-boundary fix because downstream registry, prompt, and wrapper paths read tool fields later; the unresolved question is whether maintainers want partial extension loading instead of fail-fast behavior.
