## Summary

Keep this PR open, but it is not merge-ready: the performance direction is useful, yet the branch still has correctness problems around hook-mutated bootstrap context, auth-bound model registries, and current-main auth-profile sidecar behavior, plus the required real behavior proof is missing.

## What This Changes

This PR adds bootstrap-context and PI model-discovery caches, option-sensitive auth-profile store cache keys, cache disable environment variables, and focused model-discovery cache tests.

## Best Possible Solution

Rebase on current main, preserve current auth-profile option semantics, and narrow the caches so every hook-produced input and authStorage-bound registry is either fully scoped or left uncached, with regression tests plus real runtime proof.

## Reproduction Assessment

Yes, by source inspection. The PR returns cached bootstrap context before hook mutation and caches PI registries without authStorage identity; I did not execute a live repro because this review is read-only.

## Solution Assessment

No. The proposed cache layers are broader than the safe ownership boundaries; the maintainable path is to scope the keys to all mutable inputs or keep caching at the existing prepared-store layer.

- **[P1] Preserve legacy OAuth sidecar load options:** `src/agents/auth-profiles/store.ts:511`
  - body: The branch version calls `loadPersistedAuthProfileStore(agentDir)` without current main's `resolvePersistedLoadOptions(options)`, so default runtime loads would stop resolving legacy OAuth sidecars and option-specific reads can use the wrong credential shape.   - confidence: 0.87
- **[P2] Fingerprint hook-added bootstrap files before cache hits:** `src/agents/bootstrap-files.ts:436-438`
  - body: The cache lookup returns before `resolveBootstrapFilesFromRaw` runs `applyBootstrapHookOverrides`. Hooks like `bootstrap-extra-files` read workspace globs during `agent:bootstrap`, so edits to those extra files can keep serving stale cached context until a raw bootstrap file changes.
  - confidence: 0.92
- **[P2] Scope model discovery cache to the auth storage:** `src/agents/pi-model-discovery.ts:70-86`
  - body: `discoverModels` builds the PI registry with the caller's `authStorage`, but the new key only fingerprints files and config. A later call with a different in-memory or runtime auth storage can reuse a registry bound to old credentials.
  - confidence: 0.9
