## Summary

Keep open: the rebased follow-up is relevant, but the new PI discovery cache can cross auth contexts, and the external contributor has only provided test output rather than real behavior proof.

## What This Changes

This PR adds bootstrap-context and PI model-discovery caches, makes auth-profile store cache keys option-sensitive, adds a tool-descriptor cache kill switch, and updates focused tests.

## Best Possible Solution

Keep the cache at the layer that owns both authStorage and modelRegistry, or key any lower-level cache by the actual auth context and runtime-auth bypass rules, then add scoped external-CLI key coverage plus redacted real runtime proof.

## Reproduction Assessment

Yes for the blocking PR finding from source: discoverModels accepts authStorage, but the PR cache key ignores that argument while current main has runtime-auth cache bypass paths above it. I did not execute the PR branch.

## Solution Assessment

No; the direct discoverModels cache is not the safest layer because it loses the caller's auth context. A safer implementation would reuse or extend discoverCachedPiStores, or make the lower cache authStorage-aware and disabled for runtime/plugin auth contexts.
