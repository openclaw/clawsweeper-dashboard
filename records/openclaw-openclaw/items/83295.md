## Summary

Keep open: this PR still has cache-boundary correctness issues in bootstrap context and auth-backed model discovery, and the external PR proof gate is not met.

## What This Changes

The PR adds runtime caches and cache kill switches around agent bootstrap context, PI model discovery, auth-profile store loading, and plugin tool descriptors.

## Best Possible Solution

Require the branch to cache only fully fingerprinted bootstrap inputs or bypass final-context caching for mutable hooks, keep PI discovery caching at a layer that owns authStorage and modelRegistry together, include scoped external CLI filters in auth-store cache keys, and add redacted real runtime proof.

## Reproduction Assessment

Yes for the PR review findings from source inspection: current main defines mutable bootstrap hooks and auth-bound PI registry construction, while the PR cache lookups can bypass those changing inputs. I did not execute the PR branch.

## Solution Assessment

No; the current PR caches below boundaries that own hook-mutated bootstrap files and authStorage semantics. A safer implementation would cache only fully fingerprinted inputs or move registry caching to the prepared store layer that owns both auth and model registry state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Key PI registry cache by auth context:** `src/agents/pi-model-discovery.ts:342-345`
  - body: discoverModels accepts caller-provided authStorage, and createOpenClawModelRegistry builds a registry bound to that object. This cache key only uses agent/config/file fingerprints, so a later call with another in-memory or runtime auth store can reuse a registry tied to stale credentials.
  - confidence: 0.9
- **[P2] Fingerprint hook-added files before reusing context:** `src/agents/bootstrap-files.ts:436-438`
  - body: The cache hit returns before resolveBootstrapFilesFromRaw runs applyBootstrapHookOverrides. Because bootstrap-extra-files reads configured globs and file contents during agent:bootstrap, editing a matched extra file can keep serving the old context until another raw bootstrap input changes.
  - confidence: 0.9
- **[P2] Include scoped external CLI filters in the auth-store key:** `src/agents/auth-profiles/store.ts:77-83`
  - body: buildAuthStoreCacheKey records externalCli.mode but not externalCli.providerIds/profileIds. Callers using externalCliDiscoveryForProviderAuth or externalCliDiscoveryForProviders can still share a cache entry across different scoped CLI overlays.
  - confidence: 0.82
