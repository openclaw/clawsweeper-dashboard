## Summary

Keep this open: current main still strips structured reasoning and tool scaffolding, but untagged plain-English deliberation like the reported Kimi output is preserved as visible assistant prose and can flow into Discord delivery.

## What This Changes

The issue requests stripping Kimi's untagged internal deliberation from Discord-visible assistant replies.

## Best Possible Solution

Add a narrow, tested guard for high-confidence internal-deliberation prefixes in the provider or delivery sanitizer while preserving normal prose and existing structured reasoning behavior.

## Reproduction Assessment

No, not for a live Kimi run because the reported model emission is intermittent. Source inspection does show that the exact untagged text shape would pass through the current visible-text and Discord delivery path.

## Solution Assessment

Unclear as proposed: provider-agnostic semantic stripping is risky because the text has no structured marker. A narrow, regression-tested pattern set or provider-owned handling would be more maintainable.
