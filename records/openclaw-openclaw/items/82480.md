## Summary

This should stay open: current main strips structured reasoning and tool scaffolding, but untagged plain-English deliberation like the reported Kimi output is still treated as visible prose on the Discord delivery path.

## What This Changes

The issue asks OpenClaw to prevent Kimi Coding from leaking untagged internal deliberation into visible Discord assistant replies.

## Best Possible Solution

Add a narrowly scoped, regression-tested guard for high-confidence internal-deliberation prefixes, preferably at the provider or delivery boundary that can avoid false positives while preserving normal prose.

## Reproduction Assessment

No, not for the intermittent live Kimi emission. Source inspection does show that the reported untagged text shape would flow through current-main visible-text extraction and Discord delivery.

## Solution Assessment

Unclear as proposed: provider-agnostic plain-English stripping is risky without structured markers. A narrower provider-owned or high-confidence prefix guard with focused tests would be safer.
