## Summary

This should stay open: current main has structured reasoning/tag stripping, but plain unphased prose is still treated as visible assistant text, so the reported Kimi leak is not already fixed. The proposed provider-agnostic semantic sanitizer needs maintainer judgment because it can false-positive legitimate prose.

## What This Changes

The issue requests preventing Kimi-deliberation prose from being delivered before the final Discord reply.

## Best Possible Solution

Add a narrowly scoped guard with regression coverage for high-confidence internal-deliberation prefixes, or handle this in the Kimi/provider runtime, while preserving normal prose and existing structured reasoning behavior.

## Reproduction Assessment

No for a live Kimi run: I did not reproduce the intermittent model emission. Source-level reproduction is clear for the extraction path, because unphased text blocks are returned as visible text and the reported leak has no structured markers.

## Solution Assessment

Unclear as proposed: provider-agnostic natural-language stripping is risky without owner guidance. A narrow, tested sanitizer or provider-specific handling would be safer than broad semantic filtering.
