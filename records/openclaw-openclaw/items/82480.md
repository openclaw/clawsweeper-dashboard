## Summary

Keep this open: current main strips structured reasoning/tool scaffolding, but it still treats untagged plain-English deliberation as visible assistant prose, so the reported Kimi leak is not implemented/fixed. The remaining decision is where and how narrowly to sanitize model-specific natural-language monologue without false-positive removal of legitimate replies.

## What This Changes

The issue asks OpenClaw to prevent Kimi Anthropic-compatible replies from delivering plain-text internal deliberation such as reply-decision monologue before the final user-facing answer.

## Best Possible Solution

Add narrow, regression-tested handling for high-confidence internal-deliberation prefixes, either in the Kimi/provider runtime or the canonical delivery sanitizer, while preserving ordinary prose and existing structured reasoning behavior.

## Reproduction Assessment

No for a high-confidence live Kimi reproduction: the report describes an intermittent model emission and I did not run Kimi. Source-level reproduction is clear once that untagged text is present, because current extraction returns unphased text as visible output.

## Solution Assessment

Unclear as proposed: provider-agnostic semantic stripping is risky without owner guidance. A narrow prefix-based sanitizer with focused tests, or provider-local Kimi handling, is safer than broad natural-language filtering.
