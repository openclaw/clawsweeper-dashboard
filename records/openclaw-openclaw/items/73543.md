## Summary

Keep open. Current main still has the reported allowlist/realpath mismatch, and this PR is a useful focused fix, but it needs a changelog entry plus maintainer signoff on the linked trust-model scope before merge.

## What This Changes

This PR adds a `resolvedRealPath` fallback for path-shaped exec allowlist matching and regression tests for symlinked executable resolutions.

## Best Possible Solution

Land this focused matcher fix after adding the changelog entry and either narrowing the closing linkage or explicitly preserving follow-up tracking for safe-bin, skill auto-allow, persistence, and audit metadata realpath behavior.

## Reproduction Assessment

Yes. A source-level reproduction is an `ExecutableResolution` with `resolvedPath=/opt/homebrew/bin/rg`, `resolvedRealPath=/opt/homebrew/Cellar/ripgrep/14.1.1/bin/rg`, and an allowlist entry for the real path; current main matches only the symlink path while approved execution later pins the realpath.

## Solution Assessment

Yes for the narrow allowlist-miss regression, but unclear as the complete solution for the linked issue. The safer maintainer path is to merge this as a focused matcher fix only after deciding whether symlink approvals and adjacent trust surfaces stay as follow-up work.
