## Summary

Keep open. Current main still matches path-shaped exec allowlist entries only against `resolvedPath`, while approved execution pins to `resolvedRealPath`; this PR is a focused fix for that mismatch but still needs maintainer sign-off on security-sensitive approval identity semantics and linked-issue scope before merge.

## What This Changes

The PR adds a `resolvedRealPath` fallback to path-shaped exec allowlist matching and adds colocated symlink/realpath regression tests.

## Best Possible Solution

Land this narrow matcher fix with its regression tests and a changelog entry if maintainers accept the compatibility semantics, but keep #45595 open or split explicit follow-ups for safe-bin trust, skill auto-allow, Allow Always persistence, approval metadata, and doctor hints.

## Reproduction Assessment

Yes. A high-confidence reproduction is an `ExecutableResolution` with `resolvedPath=/opt/homebrew/bin/rg`, `resolvedRealPath=/opt/homebrew/Cellar/ripgrep/14.1.1/bin/rg`, and an allowlist entry for the real path; current main checks only the symlink path while approved execution pins the realpath.

## Solution Assessment

Yes for the narrow allowlist-miss regression, because the matcher already receives both paths and the PR adds a small fallback plus focused tests. It is unclear as the complete solution for #45595 because other trust and persistence surfaces still use non-canonical path identity.
