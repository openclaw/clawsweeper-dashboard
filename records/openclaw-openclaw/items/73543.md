## Summary

Keep open. Current main still has the reported realpath allowlist mismatch, and this PR is a focused implementation candidate, but it still needs maintainer handling because it changes security-sensitive exec approval identity semantics, uses closing syntax for broader #45595, and lacks the required changelog entry.

## What This Changes

The PR adds a `resolvedRealPath` fallback for path-shaped exec allowlist matching and adds symlink/realpath regression tests.

## Best Possible Solution

Land this matcher fix with regression tests, a required changelog entry, and an explicit maintainer decision on symlink compatibility; keep #45595 open or split follow-ups unless the broader canonical identity surfaces are covered.

## Reproduction Assessment

Yes. A high-confidence reproduction is an `ExecutableResolution` with `resolvedPath=/opt/homebrew/bin/rg`, `resolvedRealPath=/opt/homebrew/Cellar/ripgrep/14.1.1/bin/rg`, and an allowlist entry for the real path; current main checks only the symlink path while approved execution pins the realpath.

## Solution Assessment

Yes for the narrow allowlist-miss regression, but unclear as the complete solution for #45595. The safer path is to merge this as a focused matcher fix only after deciding whether durable approvals and adjacent trust surfaces should canonicalize to the real executable identity.
