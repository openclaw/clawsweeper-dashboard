## Summary

Keep open. Current main still has the symlink/realpath allowlist mismatch and this PR is a useful focused fix, but automerge should pause because the user-facing fix lacks a changelog entry and the linked #45595 scope is broader than this matcher-only patch.

## What This Changes

The PR adds a `resolvedRealPath` fallback for path-shaped exec allowlist matching and adds symlink/realpath regression tests.

## Best Possible Solution

Land the focused matcher/test fix after adding the changelog entry and deciding whether #45595 should stay open or be split for the remaining executable identity surfaces.

## Reproduction Assessment

Yes. The source-level reproduction is an `ExecutableResolution` with `resolvedPath` set to a Homebrew-style symlink, `resolvedRealPath` set to the Cellar binary, and an allowlist entry for the real path; current main checks only the symlink while execution pins the realpath.

## Solution Assessment

Partly. The matcher fallback is the narrow maintainable fix for the allowlist-miss symptom, but treating it as the full resolution of #45595 is not best until the remaining trust surfaces are kept open or split.
