## Summary

Keep open: current main and v2026.6.8 still have the QMD-scoped mcporter env leak, but this PR is not merge-ready because the patch drops user mcporter XDG discovery and leaves qmd config/cache env half-scoped.

## What This Changes

The PR adds a separate mcporter spawn environment that removes XDG_CONFIG_HOME and XDG_CACHE_HOME from QMD memory manager mcporter calls, and updates the corresponding test expectation.

## Best Possible Solution

Use a target-specific env split that preserves user mcporter discovery, prevents QMD-scoped XDG from shadowing mcporter config, and passes a paired QMD config/cache env only to the qmd MCP child.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.8 pass QMD-scoped XDG env into mcporter, and mcporter v0.10.2 uses XDG_CONFIG_HOME for config discovery. I did not run a live gateway plus mcporter/qmd scenario in this read-only review.

## Solution Assessment

No. Splitting mcporter from QMD-scoped XDG is the right surface, but this implementation should preserve user mcporter discovery env and avoid a half-paired QMD child env.
