## Label Justifications

- `P1`: The issue reports a current regression where iOS/node attachment media is lost from session history, breaking follow-up agent context for affected users.
- `impact:data-loss`: The transcript loses persisted media reference metadata even though the attachment was saved, making history replay lose user-provided media.
- `impact:session-state`: The bug corrupts session transcript state by omitting media fields needed for multi-turn context.

## Summary

Keep open. Current main and the latest tagged release still drop iOS share/node `agent.request` offloaded attachment refs before transcript media persistence, and the direct fix PR is closed unmerged.

## What This Changes

The issue requests preserving parsed `offloadedRefs` from iOS share/node `agent.request` attachments into session transcript `MediaPath`/`MediaPaths` fields.

## Best Possible Solution

Land a narrow gateway fix that carries parsed offloaded image refs through the node `agent.request` path using the existing `chat.send` media persistence/transcript rewrite contract, with regression coverage and no raw JSONL transcript appends.

## Reproduction Assessment

Yes, at source level. The iOS share path sends `node.event`/`agent.request`, `parseMessageWithAttachments` returns `offloadedRefs`, and current `server-node-events.ts` drops those refs on the success path before transcript media persistence.

## Solution Assessment

Yes. Reusing or extracting the existing `chat.send` media persistence and transcript rewrite helpers is the narrow maintainable solution; raw transcript appends would risk the session transcript contract.
