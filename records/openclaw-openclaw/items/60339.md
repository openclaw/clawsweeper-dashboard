## Summary

Keep open. Current main still has the reported gateway bug: the iOS share/node `agent.request` path parses and offloads large images but drops `parsed.offloadedRefs` before transcript media fields can be persisted, and the direct fix PR is closed unmerged.

## What This Changes

The issue asks for iOS share/node ingress to persist offloaded attachment refs into transcript `MediaPath`/`MediaPaths` fields.

## Best Possible Solution

Share the existing `chat.send` media persistence/transcript rewrite contract with the node `agent.request` path, ideally through a small gateway helper, and add regression coverage for offloaded refs becoming transcript media fields.

## Reproduction Assessment

Yes. A high-confidence source reproduction follows an iOS share or node `agent.request` with a large image through `parseMessageWithAttachments`: current main returns `offloadedRefs`, then `server-node-events.ts` drops them before transcript media persistence.

## Solution Assessment

Yes. Reusing or extracting the existing `chat.send` media persistence and transcript rewrite helpers is the narrow maintainable fix; duplicating raw transcript writes would be riskier for the session transcript contract.
