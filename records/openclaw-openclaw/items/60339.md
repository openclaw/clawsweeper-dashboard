The issue requests preserving offloaded attachment metadata from the iOS share/node `agent.request` path into session transcript `MediaPath`/`MediaPaths` fields.

## Best Possible Solution

Land or replace a narrow fix that routes `agent.request` parsed media through the existing user-turn transcript persistence path, preserves `imageOrder`, and adds disk-backed regression coverage before closing this issue.

## Solution Assessment

No completed solution is on main yet. The best fix is the durable user-turn transcript path with ordering-preserving media handling; the current linked PR needs rework or replacement before it can close this issue.
