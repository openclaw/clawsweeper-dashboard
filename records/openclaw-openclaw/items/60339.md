## Summary

Keep open. Current main still drops `parsed.offloadedRefs` on the iOS share/node `agent.request` path, and #74625 is an open, unmerged closing PR for this exact transcript persistence gap.

## What This Changes

The issue requests that large image attachments sent through iOS share or node ingress preserve offloaded media refs in transcript `MediaPath`/`MediaPaths` fields.

## Best Possible Solution

Land #74625 or an equivalent narrow gateway change that reuses the existing `chat.send` media persistence and user-turn rewrite contract from the node `agent.request` path, with a node-events regression test proving offloaded refs become ordered transcript media fields.

## Reproduction Assessment

Yes. The high-confidence static reproduction is to follow an iOS share or node `agent.request` with a large image through `parseMessageWithAttachments`: current main returns `offloadedRefs`, then `server-node-events.ts` drops them before dispatch/transcript rewrite.

## Solution Assessment

Yes. The proposed direction in #74625 is the narrowest maintainable fix because it reuses the already-tested `chat.send` transcript media contract instead of adding a separate node-only persistence implementation.
