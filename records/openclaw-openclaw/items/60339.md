## Summary

Keep open. Current main still has the reported source path: iOS share/node `agent.request` parses large image attachments into `offloadedRefs`, but `server-node-events.ts` only forwards `message`, `images`, and `imageOrder` to the agent run and never mirrors the `chat.send` transcript media persistence contract.

## What This Changes

The issue asks OpenClaw to preserve offloaded attachment refs as transcript media fields for iOS share and node `agent.request` ingress.

## Best Possible Solution

Share or extract the existing `chat.send` media persistence/transcript rewrite contract for the node `agent.request` path, then add a regression test proving offloaded refs become transcript `MediaPath`/`MediaPaths` without raw JSONL writes.

## Reproduction Assessment

Yes. A high-confidence source reproduction follows an iOS share or node `agent.request` with a large image through `parseMessageWithAttachments`: current main returns `offloadedRefs`, then `server-node-events.ts` drops them before transcript media persistence.

## Solution Assessment

Yes. Reusing the existing `chat.send` media persistence and transcript rewrite helpers is the narrow maintainable fix; duplicating raw transcript writes would violate the scoped gateway transcript guidance.
