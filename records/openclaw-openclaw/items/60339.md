## Summary

Keep open. Current main still drops offloaded media metadata on the iOS share/node `agent.request` path: the parser returns `offloadedRefs`, but `server-node-events.ts` only forwards `message`, `images`, and `imageOrder`, and only reads those refs for over-limit cleanup. The direct closing PR #74625 is now closed unmerged, so this remains a narrow source-level bug rather than a closeable duplicate.

## What This Changes

The issue reports that offloaded media refs from iOS share/node ingress are not written into session transcripts.

## Best Possible Solution

Share the existing `chat.send` media persistence/transcript rewrite contract with the node `agent.request` path and add a focused node-events regression test proving offloaded refs become `MediaPath`/`MediaPaths`.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to follow an iOS share or node `agent.request` with a large image through `parseMessageWithAttachments`: current main returns `offloadedRefs`, then `server-node-events.ts` drops them before dispatch/transcript media rewrite.

## Solution Assessment

Yes. Reusing `chat.send`'s existing persistence and transcript rewrite helpers is the narrowest maintainable direction; a separate node-only transcript implementation would duplicate already-covered gateway behavior.
