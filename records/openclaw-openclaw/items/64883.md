## Summary

Keep open. Current main still treats `exec`/`bash` as mutating by name and only allows the single-action planning retry bypass for read/search/list-style tools, so the reported dummy shell/no-op evasion remains unresolved and untested.

## What This Changes

The issue asks for strict-agentic planning-only detection to distinguish real progress from dummy/no-op tool calls so a lone `sleep`-style call cannot bypass retry and blocked-exit behavior.

## Best Possible Solution

Add a conservative structural progress classifier for strict-agentic planning retry that treats well-defined no-op shell/result cases as non-progress while preserving replay-unsafe handling for real shell, messaging, cron, write/edit, and session-spawn side effects.

## Reproduction Assessment

Yes. A high-confidence unit repro can construct a strict-agentic OpenAI/Codex GPT-5 attempt with one `exec`/`bash` tool meta, planning prose, and current replay metadata; current source returns no retry because shell is excluded from the safe bypass and marked potentially side-effectful.

## Solution Assessment

No for a broad tool-name whitelist that includes shell commands. The safer solution is command/result-aware classification with regression coverage for the dummy shell case and negatives for real side effects.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/64883. Current main only lets the strict-agentic single-action planning retry bypass run for read/search-style tools, while `exec`/`bash` are always treated as mutating, so a dummy/no-op shell command cannot be classified as non-progress. Add a conservative structural progress classifier for the planning-only retry path that catches well-defined no-op shell calls such as `sleep 0.1` with no meaningful output, but does not make arbitrary shell commands retry-safe by tool name. Prefer threading enough command/result metadata into `resolvePlanningOnlyRetryInstruction` or adjacent replay metadata rather than broad prompt rewrites. Add regression coverage proving a single dummy shell call plus planning prose retries and then reaches the strict-agentic blocked exit, and preserve existing tests for real side-effectful tools, unclassified tools, messaging, cron, writes, sessions, and multi-tool progress. Add a single-line user-facing CHANGELOG.md fix entry in the active version with `Thanks @100yenadmin`. Do not change bundled provider defaults or make `exec`/`bash` generally replay-safe.
