## Summary

Keep open: current main still has the broad Windows skips, and this PR moves the tests in a useful direction, but the new capability probe can leave temp files behind when symlink creation fails.

## What This Changes

The PR changes Telegram token tests to use a runtime file-symlink capability check instead of skipping the symlink-token tests on all Windows runs.

## Best Possible Solution

Keep the capability-based skip, but make the probe use a unique temp directory with guaranteed finally cleanup before merge.

## Reproduction Assessment

Yes for the review finding: source inspection shows the probe writes tempFile before fs.symlinkSync and returns from catch without cleanup if symlink creation throws. I did not execute the failure path because this review is read-only.

## Solution Assessment

Mostly yes: dynamic capability gating is the right layer for this test-only Windows skip, but the probe should follow the existing mkdtempSync/finally cleanup pattern before merge.
