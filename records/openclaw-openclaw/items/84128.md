## What This Changes

The PR normalizes dynamic plugin/channel config-schema property maps to null-prototype records, writes extension ids with safe data-property semantics, and adds a regression test for `__proto__`, `constructor`, and `prototype` ids.

## Best Possible Solution

Land the hardening after maintainer acceptance of the null-prototype schema-map contract, keeping the focused regression test and avoiding new config, default, or migration surfaces.

## Solution Assessment

Yes, this is the narrow maintainable fix for the reported schema-map issue: normalize only the dynamic extension-id maps and write entries as data properties. The safer alternative would be broader schema-object sanitization, but that would expand the contract beyond the observed bug.
