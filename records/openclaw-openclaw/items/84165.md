## Summary

Keep open for maintainer review; I found no blocking correctness issue. Current main still writes gateway-injected assistant media into the transcript from the active-run delivery callback, while this PR moves that persistence to post-dispatch and preserves the existing first-successful media append behavior with focused coverage.

## What This Changes

The PR defers WebChat gateway-injected assistant media transcript writes during active agent runs, flushes deferred media after dispatch in delivery order, and adds a gateway regression test for invalid-first/valid-later media payloads.

## Best Possible Solution

Land this bounded gateway change once current-head checks pass, keeping Pi as owner of normal agent turns while persisting WebChat-managed media supplements only after the active run drains.
