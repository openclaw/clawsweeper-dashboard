The issue reports that opencode-go model contextWindow metadata is not loaded into runtime context lookup, so large-context models fall back to 200K and compact too early.

## Best Possible Solution

Land a provider-owned fix through https://github.com/openclaw/openclaw/pull/92955 or another maintainer-chosen linked branch so synchronous opencode-go context lookup gets the catalog values, then let the merged PR close this issue.
