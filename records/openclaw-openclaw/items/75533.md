## Summary

Keep open for normal maintainer merge. Current main still has the Active Memory classifier gap for Telegram DM topic sessions, and the PR’s scoped suffix normalization addresses that path without falling back from unknown session formats to direct chats.

## What This Changes

The PR strips generic thread suffixes before Active Memory chat-type classification, adds Telegram DM topic regression coverage plus a negative unknown-threaded-shape test, and records the fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent scoped change after normal maintainer review, keeping the normalization local to Active Memory classification and leaving broader Active Memory reliability or session-routing work to separate reports.

## Reproduction Assessment

Yes. Static reproduction is high confidence: Telegram produces agent:main:main:thread:{chatId}:{topicId}, current main’s classifier does not strip the suffix before its three-part main-session check, and the hook exits before recall when that type is not allowed.

## Solution Assessment

Yes. Reusing parseThreadSessionSuffix before chat-type classification is the narrowest maintainable fix; it avoids treating every unknown provider/channel session key as direct.
