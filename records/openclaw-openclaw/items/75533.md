## Summary

Keep open. Current main still misses the reported threaded Telegram DM key shape, while this PR is a narrow runtime fix with regression coverage and a changelog entry; I did not find a blocking review finding.

## What This Changes

The PR strips generic `:thread:` suffixes before Active Memory chat-type classification, adds Telegram DM topic regression coverage plus a negative unknown-threaded-shape case, and records the fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent scoped change once required checks are green, keeping suffix normalization local to Active Memory classification and preserving the fail-closed behavior for unknown threaded session formats.

## Reproduction Assessment

Yes. Current main has a high-confidence static reproduction path: Telegram emits `agent:main:main:thread:{chatId}:{topicId}`, the Active Memory classifier does not strip the suffix before its exact main-session check, and the hook exits before recall when the chat type is not allowed.

## Solution Assessment

Yes. Reusing `parseThreadSessionSuffix()` inside `resolveChatType()` is the narrowest maintainable fix I found; it avoids a broad fallback that would classify unknown provider/session formats as direct.
