## What This Changes

The PR tokenizes shell-wrapped gateway service command strings so `gateway` and `--port` checks work for macOS LaunchAgent `/bin/zsh -lc` wrappers, with focused regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow tokenized audit fix, close the linked issue, and treat the older overlapping regex-based PR as superseded.

## Reproduction Assessment

Yes. Current main reads the LaunchAgent command into the audit unchanged, while the audit only accepts a standalone `gateway` argv element, so `/bin/zsh -lc "... gateway --port 18890"` is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. Tokenizing shell command fragments before the existing exact-token checks is a narrow maintainable fix, and the added `--gateway-url` negative test avoids the broad substring problem seen in the overlapping earlier PR.
