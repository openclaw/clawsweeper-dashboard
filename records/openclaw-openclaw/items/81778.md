## Summary

Keep this PR open, but it needs a narrow parsing fix before merge: current main still has the reported false `gateway-command-missing` path, and the PR is useful, but its token expansion now trusts whitespace inside any argv element instead of only shell `-c` payloads.

## What This Changes

The PR changes gateway service audit parsing and tests so shell-wrapped LaunchAgent commands containing `gateway --port` can be recognized.

## Best Possible Solution

Land a narrow service-audit parser fix that expands only recognized shell `-c` payloads, preserves exact argv semantics for normal `ProgramArguments`, and then closes the linked issue after merge.

## Reproduction Assessment

Yes. Current main passes LaunchAgent `ProgramArguments` into `auditGatewayServiceConfig`, and `hasGatewaySubcommand` only checks exact top-level `gateway` argv elements, so the reported `/bin/zsh -lc` wrapper is source-reproducible.

## Solution Assessment

No. The PR is in the right service-audit layer, but the implementation should expand only recognized shell command payloads instead of every whitespace-bearing argv string.
