## Summary

Keep open: the linked gateway status bug still exists on current main and in the latest release, but this PR is not merge-ready because its broad shell-token expansion can hide malformed service commands when prelude text looks like gateway argv.

## What This Changes

The PR changes gateway service-audit parsing to expand shell-wrapped command tokens for gateway subcommand and port detection, with regression tests for macOS LaunchAgent wrapper shapes.

## Best Possible Solution

Constrain shell-wrapper support to recognized shell `-c` payload command segments, validate the actual OpenClaw gateway argv, and keep positive plus negative regression coverage.

## Reproduction Assessment

Yes from source: current `main` stores `/bin/zsh -lc ... gateway --port ...` as a shell payload string, while the audit only accepts top-level `gateway` and top-level `--port` tokens. I did not run tests because this review is read-only.

## Solution Assessment

No. The owner boundary is right, but the current implementation is too broad; the best fix should recognize only the shell command segment that executes OpenClaw gateway, not every token in the wrapper.
