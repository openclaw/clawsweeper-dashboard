## What This Changes

The PR expands gateway service-audit parsing and tests so shell-wrapped LaunchAgent ProgramArguments can expose `gateway` and `--port` tokens.

## Best Possible Solution

Land a narrow parser that inspects recognized shell `-c` command segments, validates the actual OpenClaw gateway argv, and keeps positive plus negative regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main scans only top-level ProgramArguments for `gateway` and `--port`, while the reported LaunchAgent keeps those tokens inside one `/bin/zsh -lc` payload.

## Solution Assessment

No. The owner boundary is right, but the current implementation is too broad; the best fix should recognize only the shell command segment that executes OpenClaw gateway, not every token in the wrapper.
