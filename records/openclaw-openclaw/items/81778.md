## What This Changes

The PR tokenizes shell-wrapped service command fragments in `src/daemon/service-audit.ts` so `gateway` and `--port` can be detected inside LaunchAgent shell payloads, with focused regression tests.

## Best Possible Solution

Land this token-bound audit/test fix if maintainers choose this branch, let it close https://github.com/openclaw/openclaw/issues/81751, and close https://github.com/openclaw/openclaw/pull/81755 as the superseded duplicate.

## Solution Assessment

Yes. Tokenizing shell command fragments and reusing those tokens for port parsing is a narrow maintainable fix, and it is safer than the broad regex approach in the parallel PR.
