## Summary

Keep open: this PR targets the canonical macOS LaunchAgent service-audit bug, but the proposed shell-token expansion is too broad and can suppress diagnostics for malformed shell wrappers before merge.

## What This Changes

The PR expands daemon service-audit parsing to tokenize shell-wrapped service command strings for gateway subcommand and port detection and adds focused regression tests.

## Best Possible Solution

Constrain shell-wrapper support to recognized shell -c payloads, parse command segments, validate the actual OpenClaw gateway argv, and add negative coverage for prelude-only gateway/--port tokens.

## Reproduction Assessment

Yes. Current-main source reproduces the linked issue because shell-wrapped ProgramArguments keep gateway and --port inside one /bin/zsh -lc payload, while main only scans top-level argv; the PR body also includes terminal output from a real macOS LaunchAgent probe after the patch.

## Solution Assessment

No. The owner boundary is right, but the implementation is too broad; the best fix should recognize only the shell command segment that executes OpenClaw gateway, not every token in the shell wrapper.
