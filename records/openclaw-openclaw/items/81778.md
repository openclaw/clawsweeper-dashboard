## Summary

Keep open: this is a focused bug-fix PR for a source-reproducible gateway status false warning, with no blocking review findings and sufficient terminal proof from a real macOS LaunchAgent setup.

## What This Changes

The PR tokenizes shell-wrapped service command fragments in `src/daemon/service-audit.ts` so `gateway` and `--port` can be detected inside `/bin/zsh -lc ...` LaunchAgent commands, with focused regression tests.

## Best Possible Solution

Land this narrower tokenized service-audit fix or an equivalent implementation, then close the linked issue and supersede the older overlapping PR.

## Reproduction Assessment

Yes. Current main reads LaunchAgent `ProgramArguments` into the status audit unchanged, while the audit only checks exact argv elements for `gateway`; a `/bin/zsh -lc "... gateway --port ..."` wrapper is therefore source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. Tokenizing command fragments before the existing exact-token checks is a narrow maintainable fix, and it avoids the broad substring/word-boundary false positive in https://github.com/openclaw/openclaw/pull/81755.
