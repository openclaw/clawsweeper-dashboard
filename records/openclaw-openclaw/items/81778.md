## Best Possible Solution

Constrain shell-wrapper support to recognized shell `-c` payload command segments, validate the actual OpenClaw gateway argv, keep positive and negative regression coverage, and refresh the branch before merge.

## Reproduction Assessment

## Solution Assessment

No. The owner boundary is right, but the current implementation is too broad; the best fix should recognize only the shell command segment that executes OpenClaw gateway, not every token in the wrapper.
