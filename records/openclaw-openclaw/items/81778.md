## What This Changes

The PR adds shell command token expansion to gateway service audit so shell-wrapped LaunchAgent commands can satisfy `gateway` subcommand and `--port` checks, with regression tests.

## Best Possible Solution

Land this tokenized service-audit fix or an equivalent narrow implementation, then close the linked issue and treat the older overlapping PR as superseded.

## Solution Assessment

Yes. Tokenizing shell command fragments before the existing exact-token checks is a narrow maintainable fix, and it avoids the broad substring matching problem seen in the overlapping earlier PR.
