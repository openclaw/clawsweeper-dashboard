The PR removes the forwarded Gateway `agent` timeout, adds an accepted-ownership fallback path in `agentCliCommand`, and extends the gateway CLI command tests with mocked accepted-ownership coverage.

## Best Possible Solution

Land a current-main fix that separates CLI wait budget from Gateway-owned run timeout, preserves accepted-run ownership without mocked-only imports, and updates regression tests plus user-facing docs/changelog.
