## Summary

Keep open: this PR still needs review rather than cleanup closure because the branch imports a production guard that does not exist, lacks inspectable real behavior proof, and changes the documented `--timeout` Gateway run contract while a related fresh-session fallback fix already landed in https://github.com/openclaw/openclaw/pull/74543.

## What This Changes

The PR removes the Gateway `agent` request timeout parameter, adds an accepted-ownership catch path in `agentCliCommand`, and expands unit tests around client-side timeout behavior.

## Best Possible Solution
