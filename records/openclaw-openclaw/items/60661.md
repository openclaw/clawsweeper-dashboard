## Summary

Keep open: current main still has the timeout-boundary behavior this PR is trying to address, but the PR is not mergeable because it imports a mocked-only accepted-ownership guard and lacks inspectable real behavior proof.

## What This Changes

The PR removes the CLI timeout from Gateway `agent` params, adds an accepted-ownership fallback branch, and updates gateway CLI tests around client-side timeout behavior.

## Best Possible Solution

Land a current-main repair that separates CLI wait budget from Gateway-owned run timeout only after choosing and documenting the `--timeout` contract, with production accepted-run handoff plumbing and regression tests.

## Reproduction Assessment

Yes. Static source inspection is high-confidence: current main forwards `timeout` and the server consumes it, while the PR head imports a guard that production `src/gateway/call.ts` does not export; I did not run live Gateway because this review is read-only.

## Solution Assessment

No. The timeout-boundary direction is plausible, but this patch adds a mocked-only accepted-ownership path and leaves CLI timeout semantics unresolved; production call/error plumbing or a simpler catch removal plus aligned docs/tests would be safer.
