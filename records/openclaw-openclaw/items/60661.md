## Summary

Keep this PR open for maintainer judgment: current main does not implement the client-side wait-budget behavior, but the branch is not merge-ready because it adds a production-missing accepted-ownership guard and changes the documented Gateway run-timeout contract without upgrade proof.

## What This Changes

This PR removes the `timeout` field from Gateway `agent` requests, adds an accepted-ownership fallback catch path, and updates gateway CLI command tests around client-side timeout handling.

## Best Possible Solution

Preserve the shipped server-run timeout by default; if maintainers want a client wait-budget mode, add an explicit documented CLI/protocol contract with tests, upgrade notes, and redacted live CLI proof.

## Reproduction Assessment

Yes for source-level PR review: the missing export is visible from the PR diff and current `src/gateway/call.ts`, and the timeout-contract change is visible from current CLI, schema, docs, and server code. I did not run live Gateway because this review is read-only.

## Solution Assessment

No. The branch is not the best solution as-is because it removes a documented server-run timeout contract and adds mocked-only accepted-ownership handling; a separate explicit wait-budget contract or structured ownership metadata would be safer.
