## Summary

Keep open: this external PR still has a concrete build/runtime blocker, and its central timeout-ownership change is not implemented on current main. The related merged fix addressed Gateway timeout fallback session locking differently, without making `openclaw agent --timeout` client-only.

## What This Changes

The PR removes the Gateway `agent` request `timeout` parameter, adds an accepted-ownership catch path in `agentCliCommand`, and expands Gateway CLI timeout tests.

## Best Possible Solution

Decide the timeout ownership contract first; either keep the current server-run timeout plus the fresh fallback sessions from https://github.com/openclaw/openclaw/pull/74543, or implement a documented client wait-budget path with production accepted-ownership plumbing and proof.

## Reproduction Assessment

Yes for source-level review: the PR imports `isGatewayAcceptedOwnershipError`, and current `src/gateway/call.ts` has no export by that name. I did not run live Gateway because this is a read-only review.

## Solution Assessment

No. The direction may be acceptable after a maintainer contract decision, but this branch is not safe as-is because it adds mocked-only accepted-ownership handling and removes the documented server-run timeout input.
