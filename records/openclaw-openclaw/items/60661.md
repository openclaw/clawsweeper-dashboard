## Summary

Keep open: current main has not implemented the requested client-side timeout boundary, but this branch should not merge as-is because it imports a non-existent accepted-ownership guard, lacks inspectable real behavior proof, and needs alignment with the current Gateway timeout fallback contract.

## What This Changes

The branch removes `timeout: timeoutSeconds` from Gateway `agent` params, adds an accepted-ownership catch path in `agentCliCommand`, and adds unit coverage that `--timeout` remains the `callGateway` client wait budget.

## Best Possible Solution

Land a current-main patch that separates the CLI wait budget from Gateway run ownership only if that is the intended contract, with real production error plumbing, aligned docs/protocol tests, and inspectable live proof.

## Reproduction Assessment

Yes for source-level review: current main still sends `timeout` in Gateway `agent` params and the Gateway server consumes it as run timeout, while the PR head imports an accepted-ownership guard absent from production `call.ts`. I did not have inspectable live CLI proof for the after-fix behavior.

## Solution Assessment

No. The timeout-boundary direction may be valid, but this patch adds mocked-only accepted-ownership plumbing and needs docs/protocol alignment with the current fresh-session fallback before it is the narrow maintainable fix.
