## Summary

Keep open: this PR is focused on a real Gateway CLI timeout concern, but it is not merge-ready because it imports a non-existent production export, changes the documented `--timeout` run-timeout contract, lacks inspectable real behavior proof, and overlaps the safer related discussion in https://github.com/openclaw/openclaw/pull/56796.

## What This Changes

The branch removes the forwarded Gateway `agent` timeout from `agentViaGatewayCommand`, adds accepted-ownership fallback handling, and expands command tests around client-side timeout behavior.

## Best Possible Solution

Preserve the shipped server-run timeout contract by default; if maintainers want a client wait-budget mode, add an explicit documented CLI/protocol path with focused tests and redacted live CLI proof.

Yes for source-level PR review: the missing export is visible from the PR diff and current `src/gateway/call.ts`, and the timeout-contract change is visible from current CLI, schema, docs, and server code. I did not run live Gateway because this review is read-only.

## Solution Assessment

No. The branch is not the best current solution as-is because it removes a documented server-run timeout contract and adds mocked-only accepted-ownership handling; a separate explicit wait-budget contract or structured ownership metadata would be safer.
