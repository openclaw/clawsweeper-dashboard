## Summary

Keep open: the timeout behavior is not implemented on current main, but this branch is not mergeable as-is because it imports an accepted-ownership guard that production code does not export, lacks inspectable real behavior proof, and changes the documented Gateway `--timeout` contract while current main already has the narrower fresh-session fallback fix.

## What This Changes

This PR removes the forwarded `timeout` field from Gateway `agent` requests, adds an accepted-ownership catch path in `agentCliCommand`, and updates gateway CLI tests for client-side timeout behavior.

## Best Possible Solution

If maintainers want client-side wait-budget semantics, rework this on current main with real accepted-run ownership plumbing in `src/gateway/call.ts`/Gateway client handling, aligned docs/protocol behavior, preserved fresh-session fallback coverage, and redacted live proof.
