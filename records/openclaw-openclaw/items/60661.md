## Summary

Keep open for maintainer judgment: the session-lock part was superseded by the merged fresh-session fallback, but the PR still proposes a distinct client-side-only timeout contract and currently has a build-breaking missing export plus no inspectable real behavior proof.

## What This Changes

The PR removes `timeout` from Gateway `agent` request params, adds an accepted-ownership catch path, and updates Gateway CLI tests for client-side timeout behavior.

## Best Possible Solution

If maintainers want client-side wait-budget semantics, rework the CLI/Gateway contract explicitly across call plumbing, protocol/docs, and tests; otherwise keep the current fresh-session timeout fallback and reject this branch's timeout removal.
