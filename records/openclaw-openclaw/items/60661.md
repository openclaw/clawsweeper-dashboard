## Summary

Keep open. This PR still contains a blocking missing-export bug, lacks inspectable real behavior proof, and proposes a distinct Gateway `--timeout` ownership change while current main and the latest release keep `--timeout` as the Gateway agent-run timeout with the merged fresh-session fallback from https://github.com/openclaw/openclaw/pull/74543.

## What This Changes

The branch removes `--timeout` from Gateway `agent` params, adds an accepted-ownership catch path in `agentCliCommand`, and updates gateway CLI unit tests to assert timeout stays client-side.

## Best Possible Solution

Decide the Gateway `openclaw agent --timeout` contract first; if client-side wait-budget semantics are desired, rework this on current main with real accepted-run call/error plumbing, docs/protocol alignment, and tests that preserve the merged fresh-session fallback behavior.

## Reproduction Assessment

Yes for source-level checks: current main forwards `--timeout` into Gateway `agent` params and the server consumes it as run timeout, while the PR head imports a guard that production `src/gateway/call.ts` does not export. I did not run live Gateway in this read-only review, and the PR lacks inspectable proof artifacts.

## Solution Assessment

No. The timeout-boundary direction may be worth discussing, but this patch adds a mocked-only accepted-ownership branch, is stale against the merged fallback fix, and changes a documented Gateway agent timeout contract without aligned production plumbing or docs.
