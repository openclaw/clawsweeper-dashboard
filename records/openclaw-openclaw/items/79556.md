## Summary

Keep open: the PR targets two valid Feishu bugs, but the current patch is not mergeable because it introduces a broken internal import and the `GroupId` addition does not feed the current group/session/tool-policy resolution path.

## What This Changes

The branch updates `extensions/feishu/src/bot.ts` to set `CommandSource: "text"` for slash-looking Feishu messages and pass `GroupId` for group messages, while changing one conversation-runtime import.

## Best Possible Solution

Restore the public SDK import, fix Feishu group identity through the canonical session/group-resolution path while preserving sender identity, add focused regression coverage and a changelog entry, then require redacted real Feishu proof before merge.

## Reproduction Assessment

Yes, at source level: a Feishu group message still has `From=feishu:ou_*`, and the current resolver/session/tool-policy path derives the group id from `From`. I did not run a live Feishu workspace scenario for this PR.

## Solution Assessment

No. Setting `CommandSource` is plausible for text command delivery, but adding `GroupId` is not sufficient because current main does not consume it for group identity, and the import change breaks the plugin boundary.
