## Best Possible Solution

Yes, at source level: Feishu group contexts keep `From=feishu:ou_*`, and the current resolver/session/tool-policy path derives group identity from that resolver output. I did not run a live Feishu workspace scenario.

## Solution Assessment

No. Setting `CommandSource` is plausible for text command delivery, but adding `GroupId` is not the best fix because current main does not consume it for group identity; the fix should update the canonical Feishu group-resolution/session metadata path.
