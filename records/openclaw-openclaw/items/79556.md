## What This Changes

The branch adds Feishu inbound `CommandSource: "text"` for slash-looking messages, adds `GroupId: ctx.chatId` for group messages, and asserts those context fields in Feishu tests.

## Best Possible Solution

Keep main’s resolver-based group identity fix, and land a narrow Feishu command-authorization fix with real Feishu proof for slash commands such as `/compact`, `/new`, and `/stop`.

## Reproduction Assessment

Yes, source-level: current main can still carry an admitted Feishu group slash command into `CommandAuthorized=false`, and the reply path then drops unauthorized whole-message control commands without a reply. I did not run a live Feishu workspace reproduction.

## Solution Assessment

No. The group-id fix belongs in the resolver/session path already merged, and the slash-command fix needs to address Feishu command authorization rather than only setting `CommandSource`.
