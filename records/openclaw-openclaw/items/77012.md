## Summary

Keep open: current main still has the turn-scoped, index-based Codex app-server mirror keying and active-branch reader implicated by the report, and an open closing PR already targets this issue, so it should be resolved by reviewing and landing or rejecting that PR rather than closing the issue early.

## What This Changes

The issue reports a WebChat/Codex app-server regression where mirrored session JSONL entries can create a new active branch each turn, causing previous assistant replies to disappear after refresh.

## Best Possible Solution

Review the open stable-idempotency fix and branch-preservation regression coverage, then close this issue only after the accepted fix is merged and shipped or the PR is explicitly rejected with an alternate plan.

## Reproduction Assessment

Yes for a focused source-level reproduction: two Codex app-server mirror passes with changed turn-scoped keys can exercise the unstable dedupe boundary, then the active-tail reader shows why a sibling branch disappears. I did not live-run WebChat against current main in this read-only review.

## Solution Assessment

Yes as a direction: stabilizing the Codex app-server mirror identity and adding a two-turn branch-preservation regression is narrower than changing generic `chat.history` branch traversal. The issue should stay open while the existing closing PR is reviewed.
