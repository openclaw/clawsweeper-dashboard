## Summary

Keep open: current main still has the volatile Codex app-server mirror idempotency path, and an open closing PR is already tracking the narrow fix, so this should not be cleaned up before that work lands.

## What This Changes

The issue reports a WebChat/Codex app-server regression where mirrored transcript writes can orphan previous turns so refresh-visible history only shows the latest exchange.

## Best Possible Solution

Land #77046 or an equivalent narrow Codex app-server mirror fix with #77051-style two-turn active-branch regression coverage, then close this as implemented after the fix merges.

## Reproduction Assessment

Yes, at source level: current main still uses `codex-app-server:<threadId>:<turnId>:<role>:<index>` dedupe keys, and the active-tail reader would hide older sibling branches created by a missed re-mirror. I did not run a live WebChat session in this read-only review.

## Solution Assessment

Yes in direction: stabilizing the Codex app-server mirror identity and adding a two-turn transcript branch regression is the narrow maintainable fix. Changing generic history selection or adding a config knob would be broader than the demonstrated failure path.
