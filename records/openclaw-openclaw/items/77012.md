## Summary

Keep open: current main still has the volatile Codex app-server mirror key path that can re-append already mirrored messages, and open PR #77046 now tracks the focused fix with closing syntax, with #77051 adding the suggested regression coverage.

## What This Changes

WebChat v2026.5.2 is reported to overwrite or orphan prior Codex app-server mirrored transcript turns because mirror idempotency changes every turn.

## Best Possible Solution

Review and land PR #77046 or an equivalent narrow fix, with #77051's two-turn active-branch coverage folded in or kept as complementary regression proof.

## Reproduction Assessment

Yes, source-level evidence gives a high-confidence reproduction path: repeated Codex app-server mirror writes with different turn-scoped keys can bypass dedupe, append from the current leaf, and then chat.history follows only the newest active branch. The live WebChat path was not executed in this read-only review.

## Solution Assessment

Yes, the best next step is to review and land the already-open closing PR or an equivalent fix, rather than close the issue early or spawn duplicate repair work. The fix should stabilize mirror idempotency and keep regression coverage focused on two consecutive mirrored turns.
