Full review comments:

- **[P1] Propagate agentId on local global events:** `src/tui/embedded-backend.ts:354`
  - body: `sendChat` now records the selected `agentId` on each local run, but the local `chat` and `chat.side_result` emitters still publish only `sessionKey: "global"`. The new TUI filter rejects global events without `agentId` when the selected agent is not the default, so `openclaw tui --local`, `/agent work`, then a global send or `/btw` can execute while dropping visible delta/final/abort/BTW output. Include `agentId: run.agentId` on those local event payloads and cover the non-default selected-global path.
  - confidence: 0.95
- **[P1] Keep goal updates out of stale turn persistence:** `src/agents/tools/goal-tools.ts:131-136`
  - body: `update_goal` writes the completed/blocked status directly to the session store, but the agent turn still holds the older `sessionStore[sessionKey]` working copy. Later run persistence merges that stale entry back with normal spread semantics and can overwrite `goal`, so model-driven completion/blocking can be lost before the run finishes. Refresh the live session entry after the tool write or make later persistence omit stale `goal` fields.
  - confidence: 0.9
