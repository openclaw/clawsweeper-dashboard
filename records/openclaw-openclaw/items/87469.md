Full review comments:

- **[P1] Add display metadata for the goal tools:** `src/agents/tools/goal-tools.ts:71`
  - body: `scripts/tool-display.ts` scans runtime tool sources and fails when a tool name is missing from `TOOL_DISPLAY_CONFIG`. This new `get_goal` tool, along with `create_goal` and `update_goal`, is registered without updating `src/agents/tool-display-config.ts` or the generated OpenClawKit `tool-display.json`, so `pnpm tool-display:check` will fail and clients will fall back for these tools.
  - confidence: 0.95
- **[P2] Persist the first fresh baseline for model goal reads:** `src/agents/tools/goal-tools.ts:78`
  - body: Calling `getSessionGoal` with `persist: false` also forces `adoptFreshBaseline: false`. If a model creates a budgeted goal before a fresh token snapshot exists, later `get_goal` reads keep reporting the stale baseline and `tokensUsed` never advances, so the model-visible budget does not become effective unless a user runs `/goal` manually.
  - confidence: 0.88
- **[P2] Freeze accounting after a goal completes:** `src/config/sessions/goals.ts:189`
  - body: `accountGoalUsage` recomputes `tokensUsed` from the current session total even when the goal is already `complete`. If the session continues after completion, the achieved goal's displayed usage keeps growing with unrelated turns, so the final footer/status accounting is no longer the usage at completion.
  - confidence: 0.87
