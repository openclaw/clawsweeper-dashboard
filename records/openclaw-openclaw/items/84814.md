- `merge-risk: 🚨 compatibility`: The new classifier can change existing before_agent_run hook-block behavior when a local block message contains auth-like text.
- `merge-risk: 🚨 auth-provider`: The diff changes auth/provider error classification into model fallback routing, and the ordering currently overmatches a local hook-block path.

## Summary

Keep open: the classifier seam is the right place for the reported fallback bug, but the new provider-error classification runs before preserving local hook-block results, so auth-like before_agent_run block messages can be retried through model fallback instead of surfacing the intended block.

## What This Changes

The PR adds CE-011/provider-denial matching, classifies embedded Pi isError payloads through the shared failover classifier, and adds focused regression coverage plus a changelog entry.

## Best Possible Solution

Preserve hook-block/local-block semantics before provider-denial classification, then merge the narrow CE-011 fallback fix with regression coverage for both paths.

## Reproduction Assessment

Yes. Source inspection shows current main returns null for the reported non-GPT CE-011 isError payload, while the PR head classifies that payload; the review finding is also source-reproducible with a hook_block isError payload whose replacement text matches an auth pattern.

## Solution Assessment

No as currently ordered. Reusing the shared failover classifier is maintainable, but local hook_block results need to be excluded before provider-error matching so existing plugin gate semantics stay intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve hook-block results before provider classification:** `src/agents/pi-embedded-runner/result-fallback-classifier.ts:105`
  - body: `before_agent_run` blocks are returned as `isError` payloads with `meta.error.kind: "hook_block"`, and the hook's user-facing replacement text is arbitrary. With this classification before the hook-block/null path, a block message such as "Access denied by policy" matches the auth failover patterns, so OpenClaw will try fallback models and likely surface a fallback failure instead of the intended local block message. Keep hook_block results returning null before calling `classifyFailoverReason`, and add a regression with an auth-like block message.
  - confidence: 0.9
