Keep this PR open, but it still needs changes before merge. The valid glossary cleanup is mixed with prompt/glossary terminology conflicts, and the supplied proof is internally inconsistent with the JSON shape and latest diff.

This PR edits the zh-CN and zh-TW docs i18n glossary JSON files to replace selected English identity targets with localized Chinese terms.

## Best Possible Solution

Keep the low-risk glossary cleanup where it matches the existing i18n contract, but preserve prompt-protected terms unless maintainers intentionally update the prompt, glossary, and checks together.

## Reproduction Assessment

Yes. Source inspection on current main plus the latest PR diff reproduces the conflicting zh-CN `Gateway` and zh-TW `Skills` terminology rules without needing a full docs build.

## Solution Assessment

No. The `Status` and some zh-TW cleanup are narrow, but changing prompt-protected terms only in the glossary is not the narrowest maintainable fix; the safer path is to align the changed glossary entries with the existing prompt contract.
