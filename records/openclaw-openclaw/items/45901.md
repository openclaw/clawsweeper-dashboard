## Summary

## What This Changes

The PR adds private-mode managed session-directory creation and symlink/identity checks across setup, onboarding, agent creation, session store, transcript, and lock paths.

## Best Possible Solution

Resolve managed-versus-custom ownership together with the session target, carry that closed fact into lock/store/transcript writes, and apply one shared private-directory primitive only to explicitly managed targets.

## Reproduction Assessment

Yes at source level: current main creates session store and transcript directories through ordinary recursive mkdir, so POSIX umask can create 0755 directories. I did not run a filesystem repro in this read-only review, and the PR's full compatibility matrix remains unproven.

## Solution Assessment

No in its current form; the privacy goal is correct, but the patch should carry managed/custom provenance from target resolution instead of reloading ambient config inside persistence helpers.
