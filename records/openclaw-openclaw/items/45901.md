## Summary

This PR should stay open: current main still has the managed-session directory permission gap, but this branch is stale/dirty and would regress current session write-lock defaults if merged as-is.

## What This Changes

This PR adds managed-session directory hardening helpers and wires setup, onboarding, agent creation, session store, transcript, probe, and lock paths to create managed session directories with private permissions while preserving plain mkdir behavior for custom paths.

## Best Possible Solution

Land a rebased or replacement implementation that keeps private managed-session directory hardening while preserving current lock timeout, non-reentrant locking, custom-path mkdir behavior, and session-store writer semantics.

## Reproduction Assessment

Yes. Source inspection shows current main still creates managed sessions directories through bare recursive mkdir in setup, store, transcript, and lock paths; on POSIX hosts with a normal umask that can create 0755 directories.

## Solution Assessment

No. The hardening direction is appropriate, but this PR is not the best merge vehicle until it is rebased or replaced without regressing current lock defaults and with real behavior proof.
