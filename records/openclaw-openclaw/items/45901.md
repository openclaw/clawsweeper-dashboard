## Summary

Keep open: current main still has the managed-session directory permission gap, but this PR is not merge-ready because the final head still classifies managed/custom paths from ambient on-disk config, lacks inspectable real behavior proof, and is dirty against current main.

## What This Changes

This PR adds private managed session-directory helpers and routes setup, onboarding, agent creation, session-store writes, transcript writes, locks, and tests through that hardening path.

## Best Possible Solution

Resolve managed-versus-custom ownership when session targets are selected, pass that provenance into lock/store/transcript/bootstrap writes, and harden only explicitly managed targets with upgrade proof.

## Reproduction Assessment

Yes at source level: current main uses ordinary recursive mkdir for managed session store and bootstrap paths, so POSIX umask can yield 0755 rather than guaranteed 0700. I did not run live mode checks in this read-only review.

## Solution Assessment

No in its current form. The privacy objective is right, but managed/custom ownership should be resolved once with the session target and carried forward instead of rediscovered from ambient config inside persistence helpers.
