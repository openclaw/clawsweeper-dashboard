## Summary

Keep open: the PR addresses a real sensitive-session permission gap, and many earlier symlink/custom-path review points were worked through, but the final head still classifies paths by reloading ambient config and lacks inspectable real behavior proof.

## What This Changes

The PR adds private-permission creation and symlink checks for managed session directories across setup, onboarding, agent creation, session store, transcript, lock, and Pi runner paths.

## Best Possible Solution

Resolve managed-versus-custom ownership with the session target, carry that closed fact through lock/store/transcript writes, and apply one shared private-directory primitive only to explicitly managed targets.

## Reproduction Assessment

Yes at source level: current main uses ordinary recursive mkdir for managed session directories, so POSIX modes depend on umask. The PR's remaining bug is also source-reproducible because final-head classification reloads ambient config while callers can pass explicit or in-memory store paths.

## Solution Assessment

No in its current form: the privacy objective is correct, but managed/custom ownership should be resolved once with the session target and carried forward. Reloading ambient config in the low-level classifier is not the best long-term fix.
