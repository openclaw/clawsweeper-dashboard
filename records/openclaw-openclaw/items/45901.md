## Summary

The privacy gap remains on current `main`, so this PR is still necessary, but the final patch is not merge-ready: low-level writes decide whether a directory is managed by reloading ambient configuration, which can differ from the explicit store path or in-memory config used by the caller. Likely related people: @shakkernerd is the strongest current-history routing candidate for the session-store area, with medium confidence.

## What This Changes

The PR adds private permissions for managed session directories and introduces path classification, symlink protection, and identity checks across setup, session stores, transcripts, locks, and agent creation.

## Best Possible Solution

Resolve managed-versus-custom ownership together with the session target, carry that closed fact into lock, store, and transcript writes, and apply one shared private-directory primitive only to explicitly managed targets.

## Reproduction Assessment

Yes at source level: current `main` creates managed session directories through ordinary recursive directory creation, so POSIX umask can yield `0755` rather than guaranteed `0700`. The PR's complete compatibility matrix has not been demonstrated in a real setup.

## Solution Assessment

No in its current form; the privacy objective is correct, but managed ownership should be resolved once and carried forward instead of being rediscovered from ambient configuration inside persistence paths.
