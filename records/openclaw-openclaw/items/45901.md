## Summary

Keep this PR open because current main still leaves managed session directory modes to ordinary recursive mkdir, but the branch is not merge-ready: the final head still classifies managed paths by reloading ambient on-disk config and lacks inspectable real behavior proof.

## What This Changes

The branch adds private-mode creation and symlink/managed-path classification for managed session directories across setup, onboarding, agent creation, session stores, transcripts, and locks.

## Best Possible Solution

Resolve managed-versus-custom ownership together with the session target, carry that closed fact into lock, store, and transcript writes, and apply one shared private-directory primitive only to explicitly managed targets.

## Reproduction Assessment

Yes at source level: current main still creates managed session-store directories with ordinary recursive mkdir, so POSIX modes remain umask-dependent instead of guaranteed private. I did not run a live filesystem repro in this read-only review.

## Solution Assessment

No in its current form; the privacy objective is right, but the implementation should carry managed/custom ownership from target resolution instead of rediscovering it from ambient config inside persistence helpers.
