## Summary

Keep open: the managed-session directory hardening is still useful on current main, but this external PR is not merge-ready because it lacks real behavior proof and the stale head regresses current session-write-lock defaults.

## What This Changes

The PR adds private-mode managed session directory helpers and routes setup, onboarding, agent creation, session store, transcript, model probe, embedded runner, and lock paths through them while preserving plain mkdir behavior for custom store roots.

## Best Possible Solution

Land a rebased or replacement implementation that hardens only managed session directories while preserving current lock timeout, non-reentrant locking, writer-queue behavior, and custom-path mkdir behavior.

## Reproduction Assessment

Yes. Source inspection of current main shows setup, agent creation, store writes, transcript header creation, model probes, and session lock creation still use bare recursive mkdir for managed sessions directories, which can create 0755 directories under a normal POSIX umask.

## Solution Assessment

No. The hardening direction is appropriate, but this PR head is stale and not the best merge vehicle until it is rebased or replaced without regressing current session lock defaults and with real behavior proof.
