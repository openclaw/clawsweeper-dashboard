## What This Changes

The PR adds private-permission session directory helpers and routes setup, onboarding, agent creation, session store, transcript, and lock paths through managed/custom directory creation with tests.

## Best Possible Solution

Resolve managed-versus-custom ownership with the session target, pass that closed fact through lock, store, transcript, setup, and agent creation paths, and apply one shared private-directory primitive only to explicitly managed targets.

## Reproduction Assessment

Yes at source level: current main creates managed session store and transcript directories with ordinary recursive mkdir, so POSIX modes are umask-dependent rather than guaranteed private. I did not run a live before/after proof in this read-only review.

## Solution Assessment

No in its current form; the privacy objective is correct, but rediscovering managed ownership from ambient config is the wrong layer. The safer fix is to carry managed/custom provenance from target resolution into the low-level write helpers.
