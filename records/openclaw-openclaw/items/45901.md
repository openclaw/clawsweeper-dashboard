## Summary

Keep this PR open because current main still has the underlying managed-session-directory permission gap, but the branch is not merge-ready: the PR still chooses managed/custom hardening policy from ambient on-disk config, is currently conflicting, and lacks inspectable final-head real behavior proof.

## What This Changes

The PR adds private-permission session directory helpers and routes setup, onboarding, agent creation, session store, transcript, lock, and Pi runner session directory creation through managed/custom directory handling.

## Best Possible Solution

Pass managed/custom ownership from session target resolution through store, transcript, lock, setup, and agent creation, then apply private-directory hardening only to explicitly managed targets.

## Reproduction Assessment

Yes at source level: current main still creates managed session store and transcript directories with ordinary recursive mkdir, so POSIX modes are umask-dependent rather than guaranteed private. I did not run a live before/after proof because this review is read-only.

## Solution Assessment

No in its current form: the privacy objective is correct, but low-level writers rediscover managed ownership from ambient config. The safer fix is to carry a closed managed/custom provenance value from target resolution into the write helpers.
