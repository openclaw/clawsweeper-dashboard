## Summary

Keep open: current main still has the managed session-directory privacy gap, and this PR is useful but not merge-ready because its head regresses current session write-lock contracts and lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds private permission hardening for OpenClaw-managed session directories while trying to preserve plain mkdir behavior for caller-supplied custom session paths.

## Best Possible Solution

Land a rebased or replacement PR that ports private managed-session directory creation onto current main while preserving custom-path mkdir behavior, current session write-lock defaults, non-reentrant locking, and the session-store writer queue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, create a fresh managed sessions directory through setup, session-store save, transcript header, model probe, or lock acquisition on a POSIX host with a normal umask and inspect the directory mode; this review did not execute the filesystem repro because it is read-only.

## Solution Assessment

No. The hardening direction is appropriate, but this branch is not the best merge vehicle until it is rebased or replaced without regressing current lock timeout, non-reentrant lock, and store-writer contracts.
