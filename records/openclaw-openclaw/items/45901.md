## Summary

Keep open: this is still useful security hardening and current main still creates managed session directories with plain recursive mkdir, but the PR is not merge-ready because permission policy is still selected from ambient config and final-head real behavior proof is missing.

## What This Changes

The PR adds private-permission helpers for managed session directories and routes setup, onboarding, agent creation, session store, transcript, lock, and Pi runner paths through them while trying to preserve custom paths.

## Best Possible Solution

Thread an explicit managed/custom ownership decision from config resolution into the shared helpers, preserve custom paths on plain mkdir, harden only managed session directories, refresh the branch, and add redacted terminal or log proof for fresh, upgrade, custom, and symlink cases.

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? No. The hardening direction is valid, but the implementation should not decide permission policy from ambient config inside shared path helpers when callers already resolved a concrete store path.
