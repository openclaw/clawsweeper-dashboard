## Summary

Keep open: the private session-directory hardening is still useful because current main and the latest release still create central session store/transcript directories with plain recursive mkdir, but this PR is not merge-ready because the patch reclassifies filesystem ownership from ambient config, lacks final-head real behavior proof, and needs compatibility/security policy review.

## What This Changes

This PR adds private-permission helpers for managed session directories and routes setup, onboarding, agent creation, session store, transcript, lock, and Pi runner paths through them while preserving plain mkdir for custom store parents.

## Best Possible Solution

Resolve managed-versus-custom ownership when the session target is produced, pass that closed provenance through store, transcript, lock, setup, and agent creation paths, keep custom directories on plain mkdir unless maintainers explicitly approve stricter policy, and add final-head terminal or log proof.

## Reproduction Assessment

Yes. Current main and v2026.6.6 create session store and transcript parents with ordinary recursive mkdir, so a POSIX setup can produce umask-dependent directory modes instead of guaranteed 0700; I did not run a live repro in this read-only review.

## Solution Assessment

No. The privacy goal is appropriate for core session storage, but reloading ambient config inside low-level path classification is not the best fix; ownership should be carried from the target resolver that chose the store path.
