## Best Possible Solution

Thread an explicit managed/custom session-store ownership decision from config resolution into the shared helpers, keep custom paths on plain mkdir, harden only managed session directories, refresh the branch, and add redacted terminal/log proof for fresh, upgrade, custom-path, and symlink cases.

## Solution Assessment

Is this the best way to solve the issue? No. The private-directory hardening direction is valid, but the shared helpers should not infer managed/custom policy by reloading ambient config after callers already resolved a concrete store path.
