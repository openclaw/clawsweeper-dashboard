## Summary

Keep open. Current main still has the reported Claude Code OAuth alias gap, but this PR is not correct as-is because it uses stale PI imports, exposes raw filesystem discovery tools outside OpenClaw guard/policy surfaces, and still lacks after-fix real behavior proof.

## What This Changes

The PR maps Claude Code OAuth tool names such as `Glob` to local names and appends `find`/`grep`/`ls` pi-coding-agent tools to the default OpenClaw coding tool set.

## Best Possible Solution

Land a revised implementation that keeps the OAuth alias narrow and exposes discovery tools only through current PI factories, OpenClaw workspace/sandbox guards, catalog/docs policy updates, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `Glob` cannot resolve to local `find`, and the default OpenClaw coding tool set does not register `find`, `grep`, or `ls`.

## Solution Assessment

No. The alias direction is plausible, but this PR is not the best solution because it imports stale/raw tools and bypasses OpenClaw filesystem guard and policy catalog surfaces.
