## Summary

Keep this PR open, but it should not merge as-is: current main still lacks the reported OAuth `Glob` dispatch fix, while the branch uses the wrong discovery-tool API shape, bypasses OpenClaw filesystem guard/policy layers, and lacks external after-fix behavior proof.

## What This Changes

This PR adds Claude Code OAuth tool-name aliases for `Glob`/`LS` and appends `find`/`grep`/`ls` discovery tools to the default agent tool set.

## Best Possible Solution

Land a revised implementation that keeps the OAuth alias mapping small and exposes discovery tools only through OpenClaw-owned guarded factories, catalog/profile/docs updates, and focused regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: with an Anthropic OAuth token, an inbound `tool_use.name === "Glob"` reaches `fromClaudeCodeName()`, current main has no `Glob` to `find` alias, and the runtime tool list does not register `find`/`grep`/`ls`. I did not run a live OAuth session in this read-only pass.

## Solution Assessment

No. The submitted patch is not the best solution because it imports non-exported tool constants and exposes new filesystem discovery tools outside OpenClaw's guard and policy catalog layers; the safer fix is a guarded, catalog-aligned wrapper path with tests.
