## Summary

This PR addresses a real source-reproducible OAuth tool dispatch bug, but the current patch is not mergeable: it uses stale PI imports, exposes raw filesystem discovery tools outside OpenClaw guards and policy cataloging, and still lacks contributor real behavior proof.

## What This Changes

The branch adds Claude Code OAuth tool-name aliases in the Anthropic transport and appends upstream find/grep/ls tools to the default coding tool set.

## Best Possible Solution

Land a revised implementation that keeps the OAuth alias narrow and exposes discovery tools only through current PI factories, OpenClaw workspace/sandbox guards, catalog/profile policy updates, docs, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main advertises Claude Code-style OAuth tool names but maps inbound calls only by same-name matching, while the default coding tools do not register local `find`/`grep`/`ls`.

## Solution Assessment

No. The alias direction is plausible, but registering raw upstream discovery tools is stale and bypasses OpenClaw guard, sandbox, and policy surfaces.
