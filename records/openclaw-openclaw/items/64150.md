## Summary

Keep this PR open, but the current patch should not merge: the reported OAuth `Glob` dispatch problem is source-reproducible on current main, while the branch uses stale/raw PI discovery tools and skips OpenClaw filesystem guard and policy catalog layers.

## What This Changes

The PR adds Claude Code OAuth aliases for `Glob`/`LS` and appends pi-coding-agent `find`/`grep`/`ls` discovery tools to the default OpenClaw coding tool set.

## Best Possible Solution

Land a revised implementation that keeps the OAuth alias narrow and exposes any discovery tools only through factory-created, workspace/sandbox-guarded tools with catalog/docs/profile updates and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `fromClaudeCodeName("Glob", tools)` only does same-name lookup, while `createOpenClawCodingTools()` does not register local `find`/`grep`/`ls` discovery tools for that streamed name to dispatch.

## Solution Assessment

No. The alias direction addresses the right failure mode, but the submitted patch is not the best solution because it uses stale/raw upstream tool objects and bypasses OpenClaw guard and catalog policy surfaces.
