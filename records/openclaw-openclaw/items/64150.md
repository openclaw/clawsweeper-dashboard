## Summary

Keep open: the underlying OAuth tool-name dispatch bug is still source-reproducible on current main, but this PR is not currently safe to merge because it imports non-exported discovery-tool constants and exposes new filesystem discovery tools outside OpenClaw's guard and policy catalog paths.

## What This Changes

The PR adds Anthropic OAuth Claude Code tool-name aliases and appends upstream `find`/`grep`/`ls` discovery tools to the OpenClaw coding tool set.

## Best Possible Solution

Land a revised implementation that keeps the OAuth aliasing small and exposes discovery tools only through guarded factory-created wrappers, aligned catalog/profile/docs entries, and focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source path exists: with an Anthropic OAuth token, inbound `tool_use.name === "Glob"` reaches `fromClaudeCodeName()`, current main has no `Glob` to `find` alias, and the default coding tool set does not register `find`.

## Solution Assessment

No. The alias direction is reasonable, but appending raw discovery tools is not the narrow maintainable fix; the safer path is guarded factory wrappers plus catalog/docs/policy alignment.
