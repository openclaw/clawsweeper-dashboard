## Summary

Keep open: the PR targets a real source-reproducible OAuth tool-dispatch bug, but the branch lacks real behavior proof and the proposed raw discovery-tool registration bypasses current workspace/sandbox guard and tool-policy catalog surfaces.

## What This Changes

The PR adds Claude Code OAuth tool-name aliases in the Anthropic transport and appends upstream find/grep/ls tools to the default OpenClaw coding tool set.

## Best Possible Solution

Land a revised implementation that keeps the OAuth alias narrow and adds discovery tools only through current PI factories, OpenClaw workspace/sandbox guards, catalog/profile policy updates, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows OAuth `Glob` resolves only by same-name matching, while the default coding tool factory provides no `find` tool, so the reported dispatch failure has a clear current-main path.

## Solution Assessment

No. The alias direction is plausible, but the proposed tool registration is stale and bypasses guard and policy surfaces; the maintainable fix should use current factories plus OpenClaw wrappers and catalog coverage.
