## Summary

Keep open: the PR adds a useful contributor-facing Control UI i18n report and is not obsolete on current main, but the latest head still has a source-visible P2 bug in scoped surface matching.

## What This Changes

Adds `pnpm ui:i18n:report`, a read-only Control UI i18n baseline report over raw-copy and locale metadata, plus focused helper tests.

## Best Possible Solution

Keep the read-only baseline report, but tokenize the requested surface once and reuse that token set for both raw-copy and fallback-key filtering with regression coverage.

## Reproduction Assessment

Not applicable as a bug report because this PR adds a new contributor-facing CLI feature. The remaining defect is source-reproducible: `--surface chat-queue` normalizes to one string while path and key inputs are split into separate tokens.

## Solution Assessment

No, not yet: the report-over-baseline approach is a narrow fit, but the scope matcher should use the same token model for user input, paths, and translation keys.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
