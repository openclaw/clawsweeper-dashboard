## Summary

Keep open for maintainer review. The Google schema hardening is useful and current main still has the direct unreadable-schema exposure, but this draft should not merge as-is because constrained Google toolChoice can be silently dropped after schema skipping; the maintainer-labeled draft also overlaps newer related Google schema work at https://github.com/openclaw/openclaw/pull/90406.

## What This Changes

The branch adds Google tool-schema snapshot/skipping logic across the shared Google provider, Google transport, and realtime voice paths, with focused tests for unreadable, cyclic, and all-skipped declarations.

## Best Possible Solution

Land one Google schema-hardening path that snapshots/skips optional unreadable schemas but fails locally when a forced or pinned toolChoice can no longer be honored, with maintainer acceptance of the skip policy and focused regression coverage.

## Reproduction Assessment

Yes. Current main directly maps `tool.parameters` into Google declarations in the shared provider, transport, and realtime paths; a throwing accessor or cyclic schema can fail request construction or later serialization, and the PR tests model those cases.

## Solution Assessment

No, not as submitted. The cross-path snapshotting is the right owner boundary, but the best fix must preserve forced/pinned toolChoice semantics by failing closed when the requested declaration is skipped.
