## Summary

Keep open. Current main has packaged docs, prompt-level local-doc guidance, and config schema lookup, but it still does not provide the requested first-class local-first docs search/get/API/version retrieval surface for agents; the related docs-map and live-docs-versioning issues are complementary, not replacements.

## What This Changes

The issue asks OpenClaw to ship version-matched bundled documentation and expose native local-first docs retrieval to agents for onboarding and setup guidance.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, use schema/generated references before live fallback, and document installed-version versus latest-doc behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection gives high confidence that packaged docs and prompt guidance exist, while native local-first docs retrieval methods do not.

## Solution Assessment

Unclear until maintainers choose the API and bundle/index format. The local-first direction is appropriate, but prompt-only guidance plus live openclaw docs search is not the complete requested solution.
