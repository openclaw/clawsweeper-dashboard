## Summary

Keep open. Current main has useful groundwork: docs are packaged, agents are pointed at local docs when available, and `config.schema.lookup` covers config-field lookup, but there is still no first-class local-first docs search/get/API/changed-since retrieval surface for agents; the related open issues are complementary rather than superseding this request.

## What This Changes

Request to ship version-matched bundled docs with releases and expose local-first docs retrieval to agents for onboarding and setup guidance.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, use schema/generated references before live fallback, and document installed-version versus latest-doc behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection gives high confidence that packaged docs and prompt guidance exist, but native local-first docs retrieval methods do not.

## Solution Assessment

Unclear until maintainers choose the API and bundle/index format. The local-first direction is appropriate, but prompt-only guidance plus live `openclaw docs` search is not the complete solution requested here.
