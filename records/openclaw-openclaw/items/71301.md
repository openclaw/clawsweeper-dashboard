## Summary

Keep open. Current main partially satisfies the request by shipping docs and injecting local-doc guidance into the agent prompt, but it still lacks a first-class local-first docs search/get/API/version retrieval contract for agents.

## What This Changes

The issue requests bundled version-matched OpenClaw docs plus a native local-first docs retrieval surface for agent-guided onboarding.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, and document installed-version versus latest-doc fallback semantics.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection against current main verifies the missing surface: packaged docs and prompt guidance exist, but native local-first docs retrieval methods do not.

## Solution Assessment

Unclear until maintainers choose the API, bundle/index format, and release fallback model. The local-first direction is appropriate, but prompt-only guidance plus live docs search is not the complete solution requested here.
