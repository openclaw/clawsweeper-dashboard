## Summary

Keep open. Current main has partial groundwork, but it still does not provide the native version-coupled local-first docs retrieval layer requested; the related open live-doc drift report reinforces the product problem rather than superseding it.

## What This Changes

The issue requests a version-matched bundled OpenClaw docs bundle plus native local-first docs retrieval tools for agents during onboarding.

## Best Possible Solution

Define a maintained versioned docs manifest and local search/get index shipped with releases, expose local-first docs retrieval through a deliberate Gateway or agent contract, keep live Mintlify as an explicit latest-docs fallback, and test installed-version versus latest-version behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. The high-confidence verification path is source inspection: current main can locate packaged docs and guide agents to them, but exposes no native local-first docs search/get/API/changed-since retrieval surface.

## Solution Assessment

Unclear until maintainers choose the retrieval contract and bundle/index format. The local-first direction is appropriate, but prompt guidance plus live openclaw docs search is not the complete solution requested here.
