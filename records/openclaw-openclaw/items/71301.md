## Summary

Keep open. Current main has made useful progress by packaging docs, resolving a local docs path for agents, and exposing `config.schema.lookup`, but it still does not provide a native local-first docs retrieval surface such as docs search/get/API/changed-since for agents.

## What This Changes

Request a version-matched local documentation bundle plus first-class local-first docs retrieval for agents during onboarding.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, and keep live Mintlify docs as an explicit latest-docs fallback.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main has local docs guidance and config field lookup, but not native local-first docs retrieval methods.

## Solution Assessment

Unclear until maintainers choose the API and bundle/index format. The local-first direction is appropriate, but prompt-only guidance plus live `openclaw docs` search is not the complete solution requested here.
