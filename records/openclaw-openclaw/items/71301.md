## Summary

Keep open. Current main has useful groundwork: docs are packaged, the agent prompt points at local docs when available, and `config.schema.lookup` exists, but there is still no first-class local-first docs search/get/API/changed-since retrieval surface for agents; the related open issues cover complementary docs drift and docs-map work rather than replacing this request.

## What This Changes

The issue requests version-matched bundled documentation with releases and a native local-first docs retrieval layer for agent-guided onboarding and config/API questions.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, keep live Mintlify as an explicit latest-docs fallback, and cover installed-version versus latest-version behavior in tests and docs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection against current main gives high confidence that packaged docs and prompt guidance exist, but native local-first docs retrieval methods do not.

## Solution Assessment

Unclear until maintainers choose the API and bundle/index format. The local-first direction is appropriate, but prompt-only guidance plus live `openclaw docs` search is not the complete solution requested here.
