## Summary

Keep open. Current main has partial groundwork, including packaged docs, local docs path resolution, prompt guidance, and `config.schema.lookup`, but it still lacks the requested native local-first docs retrieval surface for agents. The reporter follow-up and related open docs-drift report support the product need rather than making this implemented or duplicated.

## What This Changes

Request to ship version-matched bundled docs with each release and expose local-first docs search/get/schema/API/changed-since retrieval to agents for onboarding.

## Best Possible Solution

Define a versioned docs manifest and search index shipped with releases, expose maintained local-first docs search/get/schema/API lookup through an intentional Gateway or agent contract, keep live Mintlify as an explicit latest-docs fallback, and document/test installed-version versus latest-version behavior.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature/API request. Source inspection gives a high-confidence verification path: current main exposes packaged docs and config schema lookup, but no native docs retrieval methods or local-first CLI search.

## Solution Assessment

Unclear until maintainers choose the retrieval contract and bundle/index format. The local-first direction is appropriate, but current prompt-only guidance and live `openclaw docs` search are not the complete solution requested here.
