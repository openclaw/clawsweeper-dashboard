## Summary

Keep open. Current main has useful groundwork by shipping the docs directory, resolving a local docs path, steering agents to local docs in the prompt, and exposing `config.schema.lookup`, but it still lacks the requested native local-first docs retrieval surface for agents. The related live-doc drift report #48920 supports the product problem rather than superseding this request.

## What This Changes

Request to ship version-matched local documentation with each release and expose local-first docs retrieval methods to agents for setup, config, API, and onboarding guidance.

## Best Possible Solution

Define a versioned docs manifest and local search/get index shipped with releases, expose maintained local-first docs retrieval through a deliberate Gateway or agent contract, keep live Mintlify as an explicit latest-docs fallback, and document/test installed-version versus latest-version behavior.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature/API request. The high-confidence verification path is source inspection: current main exposes packaged docs, local path guidance, and config schema lookup, but no native local-first docs search/get/API/changed-since methods for agents.

## Solution Assessment

Unclear until maintainers choose the retrieval contract and bundle/index format. The local-first direction is appropriate, but current prompt-only guidance plus live `openclaw docs` search is not the complete solution requested here.
