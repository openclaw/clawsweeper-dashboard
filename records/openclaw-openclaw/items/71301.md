## Summary

Keep open. Current main now ships the docs tree and gives agents prompt guidance for a local docs path, but the requested native, version-matched docs retrieval surface for agents is still missing; the related open items cover narrower docs-drift, docs-map, and CLI docs-search failures rather than superseding this broader product/API request.

## What This Changes

Request to ship version-matched bundled documentation with releases and expose local-first docs search/get/schema/API retrieval to agents for guided onboarding.

## Best Possible Solution

Define a versioned docs manifest and local search/get index shipped with releases, expose it through a stable local-first agent or Gateway retrieval contract, and keep live Mintlify search as an explicit latest-docs fallback.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection gives high confidence in the current-state gap: docs are packaged and referenced in prompts, but agents still lack a native local-first docs retrieval surface.

## Solution Assessment

Unclear until maintainers choose the public API and bundle/index format. The requested local-first direction is appropriate, but the maintainable fix should be a stable agent/Gateway contract rather than ad hoc shelling out or grep guidance.
