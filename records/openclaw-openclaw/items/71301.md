## Summary

Keep open. Current main and v2026.6.6 cover packaged docs plus local-docs prompt guidance, but they still do not provide the requested first-class local docs search/get/schema retrieval surface for agents.

## What This Changes

The issue asks OpenClaw to ship release-versioned docs with installs and expose local-first docs retrieval to agents for guided onboarding.

## Best Possible Solution

Define a narrow local-first agent docs retrieval contract that searches packaged docs and generated schema/help first, falls back to live docs only by policy, and keeps docs-map and live-docs drift tracked by their narrower issues.

## Reproduction Assessment

Not applicable as a runtime bug. Current-state source verification is high confidence: packaged docs and local-docs prompt guidance exist, but no first-class agent docs retrieval tool or skill exists on current main.

## Solution Assessment

Unclear until maintainers choose the permanent retrieval boundary. Prompt guidance is a useful partial fix, but it still relies on model file reads or live CLI search rather than a native local-first docs API.
