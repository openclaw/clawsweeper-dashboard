## Summary

Keep open. Current main has useful groundwork: packaged docs, a local docs path in the agent prompt, and `config.schema.lookup`, but it still lacks native local-first docs search/get/API/changed-since retrieval for agents; the `openclaw docs` command remains live-docs search, and related #48920 supports the version-drift problem.

## What This Changes

Request to ship version-matched local docs with releases and expose native local-first docs retrieval surfaces for agent-guided onboarding.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent/Gateway retrieval contract, keep live Mintlify as an explicit latest-docs fallback, and cover installed-version versus latest-version behavior in tests and docs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main has local docs guidance and config field lookup, but not native local-first docs retrieval methods.

## Solution Assessment

Unclear until maintainers choose the API and bundle/index format. The local-first direction is appropriate, but prompt-only guidance plus live `openclaw docs` search is not the complete solution requested here.
