## Summary

Keep open. Current main has useful groundwork by packaging docs, resolving a local docs path, adding prompt guidance, and exposing `config.schema.lookup`, but it still lacks the requested native local-first docs retrieval surface for agents. The related live-doc drift report #48920 supports the product need, but it is complementary rather than a duplicate or implementation path.

## What This Changes

The issue asks OpenClaw to ship release-matched bundled docs and expose local-first docs search/get/schema/API retrieval to agents for guided onboarding.

## Best Possible Solution

Define a versioned docs manifest and search index shipped with releases, expose maintained local-first docs search/get/schema/API lookup through a deliberate Gateway or agent contract, keep live Mintlify as an explicit latest-docs fallback, and document/test installed-version versus latest-version behavior.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature/API request. Source inspection gives a high-confidence verification path: current main exposes packaged docs and config schema lookup, but no native docs retrieval methods or local-first CLI search.

## Solution Assessment

Unclear until maintainers choose the retrieval contract and bundle/index format. The local-first direction is appropriate, but current prompt-only guidance and live `openclaw docs` search are not the complete solution requested here.
