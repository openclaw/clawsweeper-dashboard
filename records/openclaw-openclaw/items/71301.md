## Summary

Keep open. Current main and the latest release partially satisfy the bundled-docs half, but the requested first-class local-first docs search/get/API/version retrieval surface for agents is still missing; the related docs-map and live-docs-versioning issues are complementary, not replacements.

## What This Changes

Request to ship version-matched bundled docs and expose native local-first docs retrieval for agent-guided onboarding.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, and document fallback to schema/help and live docs.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection against current main shows the missing surface clearly: bundled docs, prompt guidance, config schema lookup, and live CLI search exist, but native local-first docs retrieval does not.

## Solution Assessment

Unclear until maintainers choose the retrieval contract, bundle/index format, and release fallback model. The local-first direction is appropriate, but the current path-only and prompt-only implementation is not the complete solution requested here.
