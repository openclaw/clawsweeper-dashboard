## Summary

Keep open. Current main has packaged docs, prompt-level local-doc guidance, `config.schema.lookup`, and a live `openclaw docs` CLI search, but it still does not expose the requested first-class local-first docs search/get/API/version retrieval surface for agents.

## What This Changes

The issue requests version-matched bundled OpenClaw docs plus a native local-first docs retrieval layer for agent-guided onboarding.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, use schema/help for config-specific answers, and document installed-version versus live-doc fallback semantics.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection gives high confidence that docs are packaged and prompt guidance exists, but native local-first docs retrieval methods do not.

## Solution Assessment

Unclear until maintainers choose the API, bundle/index format, and release fallback model. The local-first direction is appropriate, but prompt-only guidance plus live docs search is not the complete solution requested here.
