## Summary

Keep open. Current main has useful groundwork, but the central local-first native docs retrieval capability is still missing; the related live-doc drift report and the reporter's downstream scraped-wiki experience support the product problem rather than superseding it.

## What This Changes

Ship a version-matched local OpenClaw docs bundle with releases and expose native local-first docs search/get/schema/API/change retrieval for agent-guided onboarding.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first retrieval contract through the agent or Gateway surface, keep live Mintlify as an explicit latest-docs fallback, and document/test installed-version versus latest-version behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. The high-confidence verification path is current-main source/docs inspection: packaged docs, path guidance, and config schema lookup exist, but native local-first docs search/get/API/changed-since methods do not.

## Solution Assessment

Unclear until maintainers choose the retrieval contract and bundle/index format. The local-first direction is appropriate, but current prompt-only guidance plus live openclaw docs search is not the complete solution requested here.
