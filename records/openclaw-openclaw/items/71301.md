## Summary

Keep open. Current main has useful groundwork: docs are included in the package, agents get a local docs path in the prompt, and config field lookup exists. The central request is still missing because there is no native local-first docs retrieval API or Gateway/tool action for docs search/get/API lookup/changed-since, and the CLI docs search remains live-doc backed.

## What This Changes

The issue requests version-matched bundled OpenClaw documentation plus native local-first docs retrieval tools for agent-guided onboarding.

## Best Possible Solution

Ship a versioned local docs bundle/search manifest with releases, expose stable local-first docs search/get/reference lookup through a maintained Gateway/agent contract, keep live Mintlify as an explicit latest-docs fallback, and test installed-version behavior against docs drift cases like #48920.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Source inspection gives a high-confidence verification path for the current gap across package docs inclusion, docs path resolution, prompt guidance, Gateway tool actions, Gateway RPC methods, and the live-backed CLI docs command.

## Solution Assessment

Unclear because there is no proposed patch and the API contract is still open. The local-first direction is appropriate, but the current prompt-only guidance and live `openclaw docs` command are not a complete native retrieval solution.
