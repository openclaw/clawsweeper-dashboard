## Summary

Keep open. Current main and the latest release already package docs and give agents prompt-level local-doc guidance plus `config.schema.lookup`, but they still lack the requested first-class local-first docs retrieval surface or bundled OpenClaw docs skill; the related docs-map and live-docs-versioning issues are complementary, not replacements.

## What This Changes

Request to ship version-matched docs with each OpenClaw release and expose local-first docs retrieval to agents for onboarding, config, API, and setup questions.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, use schema/generated references before live fallback, and document installed-version versus latest-doc behavior.

## Reproduction Assessment

Not applicable as a runtime bug because this is a feature/API request. Current source inspection gives high confidence that packaged docs and prompt guidance exist, while native local-first docs search/get/API/version retrieval does not.

## Solution Assessment

Unclear until maintainers choose the API and bundle/index format. The local-first direction is appropriate, but prompt-only guidance plus live `openclaw docs` search is not the complete solution requested here.
