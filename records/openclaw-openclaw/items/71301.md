## Summary

Keep open. Current main has important groundwork, including packaged docs, prompt guidance to read local docs first, and `config.schema.lookup`, but it still lacks a first-class local-first docs retrieval surface for agents; the related open issues cover narrower docs-map, live-doc drift, and docs-command failures rather than superseding this broader product/API request.

## What This Changes

Request to ship version-matched docs with releases and expose local-first docs search/get/schema/API retrieval to agents for onboarding.

## Best Possible Solution

Define a versioned docs manifest plus local search/get index shipped with releases, expose a maintained local-first agent or Gateway retrieval contract, keep live Mintlify as an explicit latest-docs fallback, and cover installed-version versus latest-version behavior in tests and docs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature/API request. Current-state verification is high-confidence: source inspection shows bundled docs and prompt guidance, but no native local-first docs retrieval tool, skill, or Gateway action.

## Solution Assessment

Unclear until maintainers choose the API and bundle/index format. The requested local-first direction is appropriate, but the current prompt-only guidance plus live `openclaw docs` search is not the complete solution.
