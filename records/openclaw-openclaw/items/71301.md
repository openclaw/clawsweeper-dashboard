## Summary

Keep open. Current main and v2026.6.8 have bundled docs plus agent prompt guidance to prefer local docs, but the requested first-class local-first agent docs retrieval surface is still missing and needs maintainer API/product direction.

## What This Changes

Request to ship version-matched OpenClaw docs with releases and expose local-first docs retrieval to agents for setup, config, API, and migration guidance.

## Best Possible Solution

Define the maintainer-approved docs retrieval contract, then implement a local-first agent-visible search/get/schema path that reuses bundled docs and existing `config.schema.lookup` without adding an unnecessary core skill when a plugin or MCP bundle is the right boundary.

## Reproduction Assessment

Not applicable as a bug. Source inspection confirms the current state: local docs and prompt guidance exist, but no first-class local-first docs search/get/schema retrieval tool is registered.

## Solution Assessment

Unclear. Current prompt guidance is a helpful partial mitigation, but the best permanent fix requires maintainers to choose the native retrieval/API ownership boundary before implementation.
