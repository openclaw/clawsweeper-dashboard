The PR adds a local archive-transcript regex and a Dreaming ingestion skip for `.jsonl.reset.*` and `.jsonl.deleted.*` files while leaving the broader session-file listing path unchanged.

## Best Possible Solution

Use the SDK-exported archive classifier at the Dreaming ingestion boundary, add focused corpus-level proof or regression coverage for fresh ordinary reset/deleted archives, preserve memory_search archive indexing, and keep the broader cron-descended issue open.
