## Summary

Keep open: current main still lets Dreaming consider usage-counted reset/deleted archives, while this PR adds a Dreaming-local archive skip using the exported session artifact classifier and keeps the broader cron-parentage work out of scope. I found no blocking patch defect after the legacy runtime fallback was removed.

## What This Changes

The PR filters session archive artifacts out of Dreaming session-corpus ingestion, records skipped generated Dreaming/cron transcripts as zero ingested lines, and adds corpus regression coverage for ordinary reset archives.

## Best Possible Solution

Land the narrow Dreaming-only archive exclusion with the focused regression test, keep memory_search archive indexing unchanged, and leave cron-descended subagent parentage tracked in the linked issue.

## Reproduction Assessment

Yes. Source inspection on current main shows Dreaming uses listSessionFilesForAgent, which includes reset/deleted usage-counted archives, and only skips checkpoint transcript paths before ingestion.

## Solution Assessment

Yes. A Dreaming-local filter using the exported archive classifier is the best narrow fix because changing listSessionFilesForAgent or buildSessionEntry would also change memory_search and usage-counted archive behavior.
