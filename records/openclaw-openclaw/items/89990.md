## What This Changes

The PR changes web login provider discovery to skip unreadable gateway method and descriptor rows and adds a regression test for poisoned descriptor metadata before a healthy provider.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main and `v2026.6.6` map `descriptor.name` directly during web-login provider discovery, so a throwing descriptor getter can abort the search before a later healthy provider is reached.

## Solution Assessment

Yes for the narrow web-login handler, but likely not the best final landing shape if maintainers want all gateway descriptor consumers hardened together. The broader PR at https://github.com/openclaw/openclaw/pull/90054 appears to cover the sibling method catalog paths too.
