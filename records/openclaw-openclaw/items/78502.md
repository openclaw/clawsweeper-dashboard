Keep open; the latest discussion adds current-release confirmations after the adjacent Gemini fixes, and current main still routes main sessions through the direct Google SSE transport plus the shared idle watchdog without a proven fix for this heavy-context hang.

## What This Changes

Reports that Google Gemini 3.1/3/2.5 models stall in full main-session OpenClaw runs until idle timeout, while direct API calls and isolated/light subagents succeed.

## Best Possible Solution

Create a sanitized main-session-equivalent request/SSE fixture or live Gemini repro, then land the smallest Google provider or replay/liveness fix with regression coverage for heavy tool-call replay.

Yes, the comments now provide a high-confidence current-release reproduction path: heavy main-session or heavy subagent Gemini runs with large context/tool replay hang at model_call:started while direct Google streaming calls succeed. This review did not independently run the live Gemini call.

## Solution Assessment
