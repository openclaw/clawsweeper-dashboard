## Best Possible Solution

Use one Dreaming agent scope for selector display, initial refresh, manual refresh, status/diary reads, action writes, gateway handlers, and regression tests, then add redacted live multi-agent UI proof.

Yes, source-reproducible. Current main has no Dreaming agent selector and sends empty status/diary params, while the PR head still has clear source paths where selected-agent reads and writes diverge.

## Solution Assessment

No. Adding a selector is the right direction, but this patch needs one selected-agent scope across reads, writes, gateway handlers, and visible proof before it is the maintainable fix.
