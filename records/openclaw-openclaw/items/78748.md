## Summary

Keep open. The PR adds the visible Dreaming selector, but the selected agent is only applied through chat-session switching; Dreaming status and diary RPCs still load default or aggregate data, and the external proof is still test-only.

## What This Changes

The PR exports chat agent-filter helpers, adds an agent dropdown to the Dreaming view, wires it to chat-session switching, and adds a view-level selector test.

## Best Possible Solution

Thread selected agent identity through Dreaming UI state, controller requests, and doctor.memory handlers, reload Dreaming data after selection changes, and keep default fallback behavior backward-compatible.

Yes. Source inspection on current main and the PR head shows the visible selector path, the unscoped Dreaming controller calls, and the default/aggregate gateway handlers without needing a generated checkout artifact.

## Solution Assessment

No. The selector-only patch is incomplete because it switches chat sessions but does not route selected-agent identity into Dreaming RPCs or reload Dreaming data; the maintainable fix needs UI, controller, and gateway plumbing together.
