## Summary

Keep this PR open but not merge-ready. The selector direction matches the linked bug, but the latest head still leaves Dreaming reads scoped to the default agent and Dreaming actions can use stale or null selected-agent state; the supplied proof is test-only.

## What This Changes

The branch adds a Dreaming-tab agent selector, exports chat agent-selection helpers, adds selected Dreaming state, threads agentId into Dreaming action RPCs, and adds focused UI test coverage.

## Best Possible Solution

Use one normalized Dreaming agent scope across selector display, initial load, refreshes, status, diary, action RPCs, and gateway handlers, with default/non-default tests and redacted live multi-agent Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows current main lacks a Dreaming selector, and PR head still has selected-agent paths where status/diary reads send empty params and gateway read handlers resolve the default agent.

## Solution Assessment

No, not yet. Adding the selector is the right direction, but the patch needs a single selected-agent contract across UI state, controller requests, gateway read/action handlers, and tests before it is the maintainable fix.
