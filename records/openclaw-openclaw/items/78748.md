## Summary

Needs changes before merge: the selector and selected-agent gateway scoping are the right direction, but the PR still has a source-reproducible in-flight switch race that can show one agent's Dreaming state under another selected agent.

## What This Changes

The PR adds a Control UI Dreaming tab agent selector, threads selectedAgentId through Dreaming UI/controller/gateway requests, and adds regression coverage.

## Best Possible Solution

Keep the selector and selected-agent gateway scoping, but make Dreaming status and diary loads keyed, forced, cancellable, or stale-response-safe before merge.

## Reproduction Assessment

Yes by source inspection: start a Dreaming status or diary load, switch the selector before it completes, and the selected-agent reload can return early while the old response still writes state.

## Solution Assessment

No, not yet. The selector and selected-agent gateway scoping are the right fix direction, but the UI/controller needs keyed or stale-response-safe reloads before this is merge-ready.
