## Summary

This PR is useful but not merge-ready: the normal selected-agent path is covered, while source inspection still shows a selected-agent stale-response race when a Dreaming status or diary request is already in flight.

## What This Changes

The PR adds a Control UI Dreaming tab agent selector and scopes Dreaming status, diary, and Dreaming actions to the selected agent.

## Best Possible Solution

Keep the selector and selected-agent gateway scoping, but make Dreaming status and diary loads keyed, forced, or stale-response-safe before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start a Dreaming status or diary load, switch the Dreaming selector before it completes, and the selected-agent load can return early while the old response still writes state.

## Solution Assessment

No. The selector and `agentId` payload direction is right, but the PR needs keyed or stale-safe selected-agent loading before it is the best fix.
