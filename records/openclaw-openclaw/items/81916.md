## Best Possible Solution

Land a narrowly bounded estimator fix that treats post-usage compaction markers as stale evidence for output and tail pressure, preserves fresh latest-usage tail pressure, adds the missing regression, and trims or repairs unrelated formatting churn.

## Reproduction Assessment

Yes for a source-level path: a transcript with a post-usage compaction marker, moderate stale prompt usage, and large stale output or tail will not set hasStaleUsageSnapshot, so the stale output/tail can still force preflight compaction. I did not execute the test suite in this read-only pass.

## Solution Assessment

No. The marker-gated direction is right, but stale output and pre-marker tail removal should not depend on the prompt-disparity heuristic once a post-usage compaction marker proves the usage snapshot predates compaction.
