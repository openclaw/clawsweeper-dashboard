## Summary

Keep open: the branch is aimed at a real session-state regression, but the latest patch still ties marker-proven stale tail and output removal to a prompt-disparity heuristic, so a moderate stale usage snapshot after compaction can still force another early preflight compaction. Current-head real behavior proof is also still insufficient because the PR body proof names an older head SHA.

## What This Changes

The branch modifies preflight compaction transcript estimation to detect post-usage compaction markers, bound stale usage snapshots, narrow tail-byte pressure after markers, and add focused regression tests.

## Best Possible Solution

Land a marker-aware estimator that treats a post-usage compaction marker as sufficient stale evidence for output and pre-marker tail removal, while keeping no-marker latest usage plus tail pressure conservative.

## Reproduction Assessment

Yes, source-level. A transcript with latest usage followed by a compaction marker, moderate prompt usage, large stale output or pre-marker tail, and a small active replay still retains stale output/tail unless the prompt-disparity heuristic trips.

## Solution Assessment

No. The marker-gated direction is right, but marker-proven stale output and pre-marker tail should be dropped independently of the prompt-disparity threshold, with a focused regression and current-head proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Drop marker-proven stale output and tail pressure:** `src/auto-reply/reply/agent-runner-memory.ts:610-624`
  - body: When a compaction marker follows the latest usage record, that usage snapshot predates compaction even if raw prompt tokens are not greater than estimated replay * 2 + 10000. Because hasStaleUsageSnapshot gates both post-marker tail narrowing and output-token removal, a moderate stale prompt with large stale output or pre-marker tail can still trigger another early preflight compaction after compaction. Treat the post-usage marker as stale evidence for output and pre-marker tail pressure, and add that regression.
  - confidence: 0.87
