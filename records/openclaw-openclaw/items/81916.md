## What This Changes

The PR adds compaction-marker-aware bounding for stale transcript usage estimates plus regression coverage for stale and fresh preflight compaction cases.

## Best Possible Solution

Land a corrected estimator that treats prompt and output from known-stale usage snapshots consistently while preserving fresh latest-usage and post-usage-tail pressure, with focused regression coverage.

No. The marker-gated prompt cap is the right direction, but the fix should also stop stale output from contributing to projected active context while preserving fresh output-token pressure.
