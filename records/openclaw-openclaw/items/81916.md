## Summary

Keep open: this PR is the focused fix for stale transcript-usage compaction loops, but it still undercounts post-compaction context because the marker-aware byte slice excludes the structured compaction summary that the harness replays to the model.

## What This Changes

The PR changes preflight compaction transcript usage scanning to treat structured post-usage compaction records as stale-usage boundaries and adds stale-marker, moderate-output, and phrase-echo regression tests.

## Best Possible Solution

Keep the structured-marker approach, but include the compaction record's model-visible summary in the active post-marker estimate and add a large-summary regression before merge.

## Reproduction Assessment

Yes, source-level. Current main still lets the last nonzero transcript usage plus trailing bytes drive preflight compaction, and the PR-specific blocker is source-reproducible by comparing the marker byte slice with the model-visible compaction summary path.

## Solution Assessment

No. The structured marker boundary is the right layer, but this implementation is not the best fix until it counts the active compaction summary instead of slicing it out of the post-marker estimate.
