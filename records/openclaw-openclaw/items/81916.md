The PR adds compaction-marker-aware bounding for stale transcript prompt usage during preflight compaction and adds focused regressions for stale usage and fresh tail pressure.

## Best Possible Solution

Land a corrected estimator that treats prompt and output from a known-stale usage snapshot consistently, while preserving fresh latest-usage and post-usage-tail protections with targeted regressions.

## Solution Assessment

No. The marker-gated prompt cap is the right direction, but the same known-stale snapshot's output tokens also need to be excluded or bounded so stale output cannot keep the preflight loop alive.
