## Summary

Keep open: the PR is a focused candidate for the stale transcript-usage compaction bug, but it drops the structured compaction summary from the post-marker token estimate, which can under-compact an active session with a large summary.

## What This Changes

The PR changes `agent-runner-memory` to treat structured post-usage compaction records as a freshness boundary for transcript usage and adds preflight compaction regressions.

## Best Possible Solution

Keep the structured-marker approach, but include the compaction summary in the active post-marker estimate and add a regression with a large summary before merge.

## Reproduction Assessment

Yes. Source inspection shows current main still lets stale transcript usage plus trailing bytes drive the preflight gate after compaction, and the new review finding is source-reproducible because active compaction summaries are replayed but excluded by the PR's post-marker byte slice.

## Solution Assessment

No. The marker boundary is the right layer, but the implementation must count the structured compaction summary before it is the best fix.
