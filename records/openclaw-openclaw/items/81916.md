## Summary

Keep open: the latest PR still has a P2 estimator bug. It detects a stale post-compaction usage snapshot for prompt bounding, but still lets output tokens from that same stale snapshot inflate the projected preflight context and potentially keep compaction looping. The contributor's real behavior proof is sufficient for the covered path, so the remaining blocker is a narrow correctness repair.

## What This Changes

The PR changes preflight compaction transcript usage estimation to detect post-usage compaction markers, bound stale prompt usage against active replay, and add regression coverage for stale transcript usage.

## Best Possible Solution

Land a corrected estimator that treats prompt and output from known-stale usage snapshots consistently while preserving fresh latest-usage and post-usage-tail protections, with a focused high-stale-output regression.

## Reproduction Assessment

Yes for a source-level PR regression: a transcript with a high-output usage record before a compaction marker and a small active replay follows the stale-marker branch, but the stale output still contributes to `resolveEffectivePromptTokens`. I did not run the live Discord/Pi loop.

## Solution Assessment

No. The marker-gated prompt cap is the right direction, but the fix should also stop output from the same known-stale usage snapshot from contributing to projected active context while preserving fresh output-token pressure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Drop stale output with stale usage:** `src/auto-reply/reply/agent-runner-memory.ts:527-532`
  - body: When `hasPostUsageCompactionMarker` is true, this branch caps only the raw prompt tokens. The same stale snapshot's `outputTokens` is still returned and later added by `resolveEffectivePromptTokens`, so a large pre-compaction completion can still force another preflight compaction even though the active replay is small. Treat the stale snapshot consistently and add a high-stale-output regression.
