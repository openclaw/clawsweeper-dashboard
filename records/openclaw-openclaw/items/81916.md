## Summary

Keep open: the stale-usage bound is aimed at a real regression, but this patch can under-count legitimate latest usage plus post-usage tool-output pressure and skip required preflight compaction.

## What This Changes

The branch clamps raw transcript prompt-usage snapshots against recent replay estimates and adds a regression test for stale pre-compaction usage.

## Best Possible Solution

Keep the stale-usage bound, but distinguish stale pre-compaction snapshots from valid latest usage plus post-usage tail pressure and cover both cases in tests.

## Reproduction Assessment

Yes for the source-level PR regression: current main has a combined-tail fixture with 86k latest usage plus a 36 KB tool tail that should compact, and this clamp can reduce that floor below the guard threshold. I did not run the test because this pass is read-only.

## Solution Assessment

No. Bounding stale transcript usage is the right direction, but this implementation over-bounds legitimate latest usage; the safer fix should preserve the existing tail-pressure floor while capping only stale snapshots.
