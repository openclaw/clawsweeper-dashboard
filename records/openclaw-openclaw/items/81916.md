## Summary

Keep open: the stale-usage fix is pointed at a real source-backed bug, but the new bound is too broad and can undercount legitimate latest usage plus post-usage tail pressure.

## What This Changes

The PR bounds transcript prompt-usage snapshots against a recent replay estimate and adds a preflight compaction regression test for stale usage after compaction.

## Best Possible Solution

Keep the stale-usage cap, but make it distinguish stale pre-compaction snapshots from valid latest usage plus tail pressure, with regression coverage for both cases.

## Reproduction Assessment

## Solution Assessment

No. Bounding stale transcript usage is the right direction, but this implementation over-bounds legitimate latest usage plus post-usage tail pressure.
