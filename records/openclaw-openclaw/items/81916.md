## Summary

Keep open: this is still the focused candidate fix for stale transcript-usage preflight compaction loops, but the current patch can undercount active post-compaction context by excluding the persisted compaction summary from the marker-aware estimate.

## What This Changes

The PR changes preflight compaction transcript scanning to treat structured post-usage compaction records as stale-usage boundaries and adds regression tests for stale transcript usage after compaction.

## Best Possible Solution

Keep the structured-marker approach, but include the structured compaction summary in the active post-marker estimate and add a large-summary regression before merge.

## Reproduction Assessment

Yes. Current main still lets the last nonzero transcript usage plus trailing bytes drive preflight compaction, and the PR-specific blocker is source-reproducible by comparing the marker byte slice with the model-visible compaction summary path.

## Solution Assessment

No. The structured marker boundary is the right layer, but this implementation is not the best fix until it counts the active compaction summary instead of slicing it out of the post-marker estimate.
