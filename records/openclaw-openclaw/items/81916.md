## Summary

Keep open: the stale-usage fix is useful, but this patch can also clamp a valid latest usage snapshot and skip required preflight compaction when post-usage tail pressure should still count.

## What This Changes

The PR clamps transcript prompt-usage estimates to a recent replay estimate when usage appears stale and adds a regression test for stale usage after compaction.

## Best Possible Solution

Keep the stale-usage bound, but make it distinguish stale pre-compaction snapshots from valid latest usage plus post-usage tail pressure, with regression coverage for both cases.

## Reproduction Assessment

Yes for the source-level PR regression: current main has a combined-tail test with 86k latest usage plus a 36 KB tool tail that expects compaction, and this clamp can reduce that floor below the threshold. The original production Discord/Pi loop was not live-rerun.

## Solution Assessment

No. Bounding stale transcript usage is the right direction, but this implementation over-bounds legitimate latest usage; the safer fix should preserve the current tail-pressure floor while capping only stale snapshots.
