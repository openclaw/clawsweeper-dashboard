## Summary

Keep open: current main shows the reported stderr severity path, but this PR still needs a narrower classifier and after-fix real macOS iMessage runtime proof before merge.

## What This Changes

The PR adds an iMessage stderr classifier that routes matching Apple framework stderr lines through `runtime.log()` and adds a changelog entry for the severity change.

## Best Possible Solution

Use a small tested classifier for concrete known-benign Apple AddressBook reconciliation lines, keep unknown stderr at ERROR, and require redacted real macOS runtime evidence before merge.

## Reproduction Assessment

Yes from source inspection: current main routes every non-empty `imsg rpc` child stderr line to `runtime.error()`, so the linked ABGroup stderr sample would be logged at ERROR. A live reproduction still requires a macOS host with the iMessage channel enabled and the AddressBook flush occurring.

## Solution Assessment

No: the direction is right, but this implementation uses a broad token regex and lacks focused classifier coverage. The safer fix is a narrow benign-phrase allowlist with positive and negative stderr examples.
