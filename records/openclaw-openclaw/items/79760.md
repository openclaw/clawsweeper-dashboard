## Summary

Keep open: the linked bug is source-reproducible on current main, but this PR needs a narrower stderr classifier and after-fix real behavior proof before merge.

## What This Changes

The PR adds an iMessage RPC stderr regex classifier that routes Apple-framework-looking stderr lines to `runtime.log()` and adds a changelog entry.

## Best Possible Solution

Use a small tested classifier that matches only documented benign Apple reconciliation or annotation lines and leaves unknown framework stderr at ERROR.

## Reproduction Assessment

Yes from source, but not from a live run: current main routes every non-empty `imsg rpc` stderr line to `runtime.error()`, so the linked ABGroup sample would be emitted at ERROR. A live proof still requires a macOS iMessage host that emits the Apple framework line.

## Solution Assessment

No: the direction is right, but the proposed allowlist is broader than the reported benign patterns. The safer fix is to match the known reconciliation/annotation text and add coverage for both benign and non-benign stderr examples.
