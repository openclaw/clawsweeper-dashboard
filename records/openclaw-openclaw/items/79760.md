## Summary

Keep open: this PR targets a real iMessage logging bug, but it still needs real macOS imsg/OpenClaw behavior proof and the proposed allowlist is too broad to merge safely.

## What This Changes

The PR adds an iMessage RPC stderr regex classifier and changelog entry to route matching Apple-framework stderr lines through `runtime.log()` instead of `runtime.error()`.

## Best Possible Solution

Use a small tested classifier that matches only documented benign AddressBook reconciliation lines and leaves unknown stderr at ERROR, then require redacted real macOS runtime proof before merge.

## Reproduction Assessment

Yes from source, but not from a live run: current main routes every non-empty `imsg rpc` stderr line through `runtime.error()`, so the linked ABGroup sample would log at ERROR. A live reproduction still requires a macOS host with the iMessage channel enabled and the Apple AddressBook flush occurring.

## Solution Assessment

No: the direction is right, but this implementation downgrades any stderr line containing broad Apple framework tokens. The safer fix is a narrow benign-phrase classifier with coverage for both benign and non-benign stderr examples.
