## What This Changes

The PR adds global and per-agent `toolsBySender` config, channel-scoped sender keys, and sender-specific filtering through core, plugin, and bundled tool resolution.

## Best Possible Solution

Keep the PR open for maintainer review; if the direction is accepted, finish canonical channel matching, generated config docs, changelog, focused regression tests, and documented no-sender wildcard behavior before merge.

## Reproduction Assessment

Yes. A static reproduction is `toolsBySender: { "channel:msteams:alice": { deny: ["exec"] } }` with `messageProvider: "teams"` and `senderId: "alice"`; the PR composes `teams:alice` while OpenClaw's normal channel path canonicalizes `teams` to `msteams`.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the maintainable solution should reuse OpenClaw's channel normalization and complete the required config docs, changelog, and targeted tests before this security-sensitive boundary changes.
