## Summary

Keep open. Current main and v2026.6.10 already cover the implicit Linux no-display fallback and actionable explicit-headed text, but they still do not provide the requested machine-readable no-display result/remediation contract across the browser route, gateway, and client surfaces.

## What This Changes

Requests managed browser launches on no-display hosts to degrade with a structured machine-readable result or a policy-controlled headless fallback.

## Best Possible Solution

Add a narrow typed no-display unavailable result for explicit headed no-display launches across the browser route/gateway/client/tool surfaces, while preserving the existing implicit fallback unless maintainers approve broader policy-controlled fallback.

## Reproduction Assessment

Yes. Source inspection shows explicit headed local managed launches on Linux without DISPLAY/WAYLAND_DISPLAY throw an actionable BrowserProfileUnavailableError, then route/gateway/client helpers expose message-only errors rather than the requested structured reason/remediation object.

## Solution Assessment

No. Current main is only a partial solution; the maintainable next step is a maintainer-approved typed browser error/result contract rather than silently broadening fallback behavior.
