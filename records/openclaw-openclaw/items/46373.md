## Summary

Keep this PR open, but it still needs changes before merge: current main does not yet solve either requested bug, while this branch adds an over-broad tool_use classifier and a locale-loading race that should be repaired first.

## What This Changes

The PR adds awaitable Control UI locale startup readiness, gates initial app rendering on locale loading, and rewrites tool_use/tool_result mismatch errors to suggest `/new` with focused tests.

## Best Possible Solution

Land a rebased focused update that serializes locale precedence, narrows the mismatch classifier to real tool_result/tool_use pairing failures, updates the current sanitizer module, adds false-positive coverage, and includes the required changelog entry.

## Reproduction Assessment

Yes. The remaining current-main gaps are reproducible by source inspection, and the PR-introduced defects are visible from the diff: generic `tool_use_id` text matches the classifier, and concurrent locale promises can resolve in either order.

## Solution Assessment

No. The direction is good, but this implementation should not merge until settings locale is applied after startup readiness, the mismatch regex requires pairing context, and sanitizer handling is rebased onto the current split module.
