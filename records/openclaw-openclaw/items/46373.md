## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P2] Narrow the tool_use_id mismatch matcher:** `src/agents/pi-embedded-helpers/errors.ts:838-839`
  - body: The new regex treats any error containing `tool_use_id` as conversation-history corruption. Generic validation errors that merely mention the field would tell users to abandon the session with `/new`; require actual `tool_result`/corresponding `tool_use` mismatch context before rewriting.
  - confidence: 0.91
- **[P2] Apply the settings locale after startup hydration:** `ui/src/ui/app.ts:119-126`
  - body: The constructor starts `i18n.ready` and `i18n.setLocale(this.settings.locale)` concurrently. If the stored or navigator locale and settings locale differ, the slower async load can overwrite the settings value, so the initial render can use the wrong language.
  - confidence: 0.86
- **[P2] Retry topbar observation after the delayed render:** `ui/src/ui/app.ts:726-728`
  - body: Returning `nothing` for the first update lets `firstUpdated()` run before `.topbar` exists. Current lifecycle code calls `observeTopbar()` only once, so the ResizeObserver and `--topbar-height` setup can be skipped after `localeReady` renders the real UI.
  - confidence: 0.88
