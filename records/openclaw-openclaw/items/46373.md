## What This Changes

The branch combines a Control UI i18n startup render gate with agent error formatting for tool_use/tool_result mismatch messages and related tests.

## Best Possible Solution

Rework the branch into a narrow i18n startup fix that serializes locale selection, preserves topbar setup after the first real render, and avoids adding broad tool-mismatch copy unless a narrowly scoped classifier remains necessary.

## Reproduction Assessment

Yes, at source level for the PR defects: the diff shows the broad matcher, concurrent locale setters, and delayed empty render while current lifecycle observes .topbar only once. I did not run live browser or provider repro in this read-only review.

## Solution Assessment

No. The i18n direction is useful, but this patch can misclassify provider errors, race locale sources, and skip topbar setup after the delayed first render; the safer path is a narrowed i18n repair plus current transcript repair for the tool_result loop.
