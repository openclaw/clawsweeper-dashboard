  - body: The added regex treats any `tool_use_id` substring as transcript corruption, so unrelated provider validation errors such as missing or invalid fields would be rewritten to `/new` guidance. Require actual `tool_result`/corresponding `tool_use` mismatch context before rewriting.

  - body: The constructor starts `i18n.ready` and `i18n.setLocale(this.settings.locale)` in the same `Promise.all`. If the stored or navigator locale and settings locale differ, whichever async load settles last wins, so startup can render in the wrong locale.
  - confidence: 0.86
- **[P2] Retry topbar observation after the delayed render:** `ui/src/ui/app.ts:729-730`
  - body: Returning `nothing` before `localeReady` lets Lit run `firstUpdated()` while `.topbar` is absent. Because the lifecycle calls `observeTopbar()` once, the ResizeObserver and `--topbar-height` setup can be skipped for the real UI.
