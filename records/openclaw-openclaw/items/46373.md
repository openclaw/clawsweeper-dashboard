## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retry topbar observation after locale-ready rendering:** `ui/src/ui/app.ts:727-728`
  - body: Returning `nothing` on the first render means Lit still runs `firstUpdated()`, but `handleFirstUpdated()` calls `observeTopbar()` only once. At that point there is no `.topbar`, so the observer and `--topbar-height` setup are skipped permanently when `localeReady` flips and the real shell appears.
  - confidence: 0.9
- **[P2] Apply the settings locale after startup readiness:** `ui/src/ui/app.ts:121-124`
  - body: The constructor starts `i18n.ready` and `i18n.setLocale(this.settings.locale)` concurrently, so when persisted/navigator locale and settings locale differ, whichever async translation load resolves last wins. The app settings locale should be applied after startup readiness, or `setLocale` needs last-writer protection.
  - confidence: 0.86
- **[P2] Tighten the mismatch regex around tool_result context:** `src/agents/pi-embedded-helpers/errors.ts:835-837`
  - body: The new `tool_use_id` regex branch matches any provider validation error mentioning that field, including required/type/empty-field errors that are not conversation-history corruption. That would tell users to start `/new` for unrelated request-shape bugs.
  - confidence: 0.9
