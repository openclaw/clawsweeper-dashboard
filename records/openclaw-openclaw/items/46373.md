## Summary

Keep open: current main does not fully contain the Control UI locale-before-render change, but this PR is not mergeable as-is because it lacks real behavior proof and introduces correctness bugs in both the locale gating and mismatch-error detection paths.

## What This Changes

The PR combines a Control UI i18n startup render gate with an agent error formatter/sanitizer fallback for tool_use/tool_result mismatch errors and related tests.

## Best Possible Solution

Update or replace the branch with a narrow fix that serializes startup/settings locale selection, preserves lifecycle observers after delayed first render, avoids broad error regex matches, and includes real runtime proof.

## Reproduction Assessment

Yes, at source level. The PR diff deterministically shows the bare regex match, concurrent locale setters, and blank first render before `firstUpdated()` observes `.topbar`; I did not run tests or a browser because this review is read-only.

## Solution Assessment

No. The tool-mismatch user problem is better handled by the transcript repair already on main, and the i18n render gate needs a safer lifecycle-aware implementation before this branch can land.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Narrow the tool_use_id mismatch regex:** `src/agents/pi-embedded-helpers/errors.ts:838-839`
  - body: The new first regex branch matches any error containing `tool_use_id`, so ordinary validation errors such as a missing or invalid field would be reported as conversation history corruption and tell users to start `/new`. Require `tool_result` or mismatch context before returning that recovery copy.
  - confidence: 0.91
- **[P2] Apply the settings locale after startup hydration:** `ui/src/ui/app.ts:119-126`
  - body: The constructor starts `i18n.ready` and `i18n.setLocale(this.settings.locale)` concurrently. If the persisted i18n locale and settings locale differ, whichever async translation load finishes last wins, so the first rendered locale can ignore the settings value.
  - confidence: 0.86
- **[P2] Recreate topbar observation after the delayed render:** `ui/src/ui/app.ts:726-728`
  - body: Returning `nothing` for the first update lets `firstUpdated()` run before `.topbar` exists. `observeTopbar()` then returns and is never retried, so the topbar ResizeObserver and `--topbar-height` setup are skipped after `localeReady` renders the real UI.
  - confidence: 0.88
