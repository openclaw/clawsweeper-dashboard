## What This Changes

This PR adds a Control UI locale readiness gate before initial render and rewrites tool_use/tool_result mismatch errors to suggest `/new`, with focused tests.

## Best Possible Solution

Rebase or split this into a narrow i18n startup fix that serializes locale readiness and preserves topbar setup, while handling tool-mismatch copy through the canonical formatter/repair work.

## Reproduction Assessment

Yes, at source level for the PR defects: the diff shows the broad matcher, concurrent locale setters, and delayed empty render while current main observes `.topbar` only once. The real browser/provider flows still need after-fix behavior proof.

## Solution Assessment

No. The intended fixes are useful, but this patch can misclassify provider validation errors, race locale sources, and skip topbar setup after delayed first render; a rebased/split fix is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Narrow the tool_use_id mismatch matcher:** `src/agents/pi-embedded-helpers/errors.ts:835-836`
  - body: The new regex treats any `tool_use_id` substring as transcript corruption, so unrelated provider validation errors such as missing or invalid fields would be rewritten to `/new` guidance. Require actual `tool_result`/corresponding `tool_use` mismatch context before rewriting.
  - confidence: 0.91
- **[P2] Apply the settings locale after startup hydration:** `ui/src/ui/app.ts:119-126`
  - body: The constructor starts `i18n.ready` and `i18n.setLocale(this.settings.locale)` in the same `Promise.all`. If the stored or navigator locale and settings locale differ, whichever async locale load settles last wins, so startup can render in the wrong locale.
  - confidence: 0.86
- **[P2] Retry topbar observation after the delayed render:** `ui/src/ui/app.ts:726-728`
  - body: Returning `nothing` before `localeReady` lets Lit run `firstUpdated()` while `.topbar` is absent. Because the current lifecycle calls `observeTopbar()` once and that helper returns on a missing topbar, the real UI can miss its ResizeObserver and `--topbar-height` setup.
  - confidence: 0.88
