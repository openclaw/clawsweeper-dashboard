## Summary

Keep open: current main and latest release still accept partially numeric `--limit` values, and the linked fixing PR is open with closing syntax, so this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue asks CLI ClawHub plugin and skill search commands to reject non-positive, decimal, or partially numeric `--limit` values before making a search request.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84480 or an equivalent strict positive-integer parser with focused CLI coverage, then close this issue from the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows both command registrations use `Number.parseInt`, and the downstream search paths either forward parsed partial positives or default invalid non-positive values instead of failing fast.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `--limit` as a whole positive integer in the CLI parser before any ClawHub call, with regression coverage for plugin and skill search parsing.
