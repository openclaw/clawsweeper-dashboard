## What This Changes

The issue requests strict positive-integer validation for `openclaw plugins search --limit` and `openclaw skills search --limit` so partially numeric, decimal, zero, and negative values fail before ClawHub search.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84480 if it still passes maintainer review, then close this issue from the merged fix.

## Reproduction Assessment

Yes, from source inspection: both search commands use `Number.parseInt` for `--limit`, and the downstream code either forwards truthy partial values or defaults non-positive values instead of rejecting them. I did not run the full CLI because this review is read-only.

## Solution Assessment

Yes: the narrow maintainable fix is to fail fast in the Commander option parser with a shared strict positive-integer helper and focused CLI tests. The linked open PR appears to target that path, so this issue should remain open until that PR merges or is replaced.
