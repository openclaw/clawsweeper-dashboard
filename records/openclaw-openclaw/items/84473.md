## Summary

Keep open: current main still parses both search `--limit` options with `Number.parseInt`, latest release `v2026.5.20` has the same permissive behavior, and the linked fixing PR is still open with closing syntax.

## What This Changes

Require `openclaw plugins search` and `openclaw skills search` to reject partially numeric, decimal, zero, and negative `--limit` values before calling ClawHub.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84480, or an equivalent shared strict positive-integer parser with focused plugin and skills CLI coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: both current command registrations use `Number.parseInt`, and Node confirms partially numeric strings parse to positive integers. I did not run the full CLI path in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `--limit` as a whole positive integer in the CLI parser before any ClawHub request, with regression coverage for both plugin and skill search commands.
