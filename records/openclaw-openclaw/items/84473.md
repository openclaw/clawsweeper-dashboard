## Summary

Keep open: current main still accepts partially numeric `--limit` values for both plugin and skill search, and the open linked fixing PR uses closing syntax for this issue, so this should close only after that PR or an equivalent fix lands.

## What This Changes

The issue asks `openclaw plugins search` and `openclaw skills search` to reject non-whole or non-positive `--limit` values before calling ClawHub.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84480 or an equivalent strict positive-integer parser with focused CLI coverage, then close this issue from the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows both affected command registrations use `Number.parseInt`, and the downstream search paths either forward parsed partial positives or default invalid non-positive values instead of failing fast.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `--limit` as a whole positive integer in the Commander option parser before either ClawHub search action runs.
