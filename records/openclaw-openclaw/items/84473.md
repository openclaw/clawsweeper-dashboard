## Summary

Keep open: current main and latest release still accept partially numeric `--limit` values, and an open linked PR already targets the fix with closing syntax.

## What This Changes

The issue asks for `plugins search --limit` and `skills search --limit` to reject non-positive, decimal, and partially numeric values before any ClawHub search call.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84480 or an equivalent strict positive-integer parser with focused CLI coverage, then close this issue from the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows both affected command registrations use `Number.parseInt`, and the downstream search paths either forward parsed partial positives or default invalid non-positive values instead of failing fast.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `--limit` as a whole positive integer in the CLI parser before any ClawHub call, with regression coverage for plugin and skill search parsing.
