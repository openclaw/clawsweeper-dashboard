## What This Changes

The issue requests strict positive whole-integer validation for `--limit` in both ClawHub CLI search commands instead of partial `Number.parseInt` coercion.

## Best Possible Solution

Review and land the existing strict positive-integer parser PR, then close this issue once the fix is merged and shipped.

## Reproduction Assessment

Yes. Source inspection on current main shows both command registrations use `Number.parseInt`, and the reporter supplied real CLI output for `1abc`; I did not run the full CLI in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is a Commander option parser that rejects incomplete, decimal, and non-positive integer strings before dispatching to ClawHub, which the open fixing PR already targets.
