## Summary

Keep open: current main and the latest release still use partial-number parsing for both ClawHub search limit options, and the issue is now paired with an open closing PR that should be reviewed or landed before the issue closes.

## What This Changes

The issue asks for `openclaw plugins search --limit` and `openclaw skills search --limit` to reject non-positive, decimal, and partially numeric values before calling ClawHub.

## Best Possible Solution

Review and land `https://github.com/openclaw/openclaw/pull/84480` if it remains correct, then close this issue as implemented once the fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows both command registrations use `Number.parseInt`, with downstream search paths forwarding or defaulting parsed values rather than rejecting invalid input.

## Solution Assessment

Yes. A strict positive-integer Commander option parser shared by both search commands is the narrow maintainable fix, and the open paired PR targets that path.
