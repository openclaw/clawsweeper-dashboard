## Summary

Keep open. Current main still has the reported GitHub Copilot Claude Opus 1m metadata and thinking-profile gap, and the issue is already paired with open PR #72825 plus overlapping PR #72829, so the next step is maintainer reconciliation rather than closure or another repair branch.

## What This Changes

The issue asks OpenClaw's GitHub Copilot plugin to treat Claude Opus 4.6/4.7 1m variants as 1,000,000-token models and allow xhigh reasoning for claude-opus-4.7-1m-internal.

## Best Possible Solution

Land one narrow GitHub Copilot plugin fix, likely PR #72825 or a reconciled equivalent, that keeps dynamic model resolution, default/static catalog metadata, max-output metadata, context/status behavior, and thinking-profile support consistent for the Claude Opus 1m ids; treat live /models discovery as a separate follow-up decision.

## Reproduction Assessment

Yes. On current main, selecting or configuring github-copilot/claude-opus-4.7-1m-internal hits the synthetic Copilot model fallback with 128k/8192 metadata, while the provider thinking profile advertises only off/minimal/low/medium/high because the xhigh allowlist lacks that id.

## Solution Assessment

Yes for the narrow direction. A provider-local capability resolver shared by dynamic resolution and default/catalog generation, plus an xhigh profile entry for the Copilot-exposed Claude 4.7 1m id, is the smallest maintainable fix before any broader live-discovery work.
