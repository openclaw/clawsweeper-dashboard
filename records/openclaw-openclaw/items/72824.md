The issue asks the GitHub Copilot provider to use 1M/64K capability metadata for Claude Opus 1m variants and allow `xhigh` for `claude-opus-4.7-1m-internal`.

## Best Possible Solution

Land one canonical Copilot-provider fix that maps `/models` reasoning efforts into supported thinking levels, keeps active runtime and bundled provider-policy profiles in sync, and preserves correct Claude 1m static/offline fallback metadata.

Yes, source-reproducible with high confidence. Current main's Copilot thinking profile omits `claude-opus-4.7-1m-internal` from `xhigh`, and the directive path rejects explicit unsupported levels through the reported error path.

## Solution Assessment
