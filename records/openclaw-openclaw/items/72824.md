## Summary

Keep open: current main still only partially addresses this Copilot capability bug. ## What This Changes

The issue asks for GitHub Copilot Claude Opus 1m variants to use the upstream 1M/64K limits and for `claude-opus-4.7-1m-internal` to allow `xhigh` thinking.

## Best Possible Solution

Land one canonical GitHub Copilot provider fix that uses live `/models` capability data for context/output/thinking when available, preserves correct static/offline fallback metadata for the known Claude 1m ids, and keeps active runtime and bundled provider-policy thinking profiles in sync.
