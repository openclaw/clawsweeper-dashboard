## What This Changes

The issue asks OpenClaw's GitHub Copilot plugin to use Copilot-advertised 1M context/64K output metadata for Claude Opus 4.6/4.7 1m variants and to allow xhigh thinking for claude-opus-4.7-1m-internal.

## Best Possible Solution

Land one provider-owned GitHub Copilot capability fix, likely #72825 or a reconciled canonical equivalent, that keeps dynamic resolution, default/static catalog metadata, max-output metadata, status context lookup, and thinking profiles consistent for the Claude Opus 1m ids; treat live /models discovery as a separate follow-up decision.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: select github-copilot/claude-opus-4.7-1m-internal, and the current Copilot fallback/default metadata still resolves to 128k/8192 while the provider thinking profile does not advertise xhigh for that id.

## Solution Assessment
