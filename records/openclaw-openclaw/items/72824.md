## What This Changes

The issue requests that GitHub Copilot Claude Opus 1m variants use their advertised 1M/64K capabilities and that `claude-opus-4.7-1m-internal` allow `xhigh` thinking.

## Best Possible Solution

Land one provider-local GitHub Copilot fix, likely #72825 or a reconciled equivalent, that keeps dynamic resolution, seeded defaults, max-output metadata, context/status behavior, and thinking-profile support aligned for the Claude Opus 1m ids.

## Reproduction Assessment

Yes. ## Solution Assessment
