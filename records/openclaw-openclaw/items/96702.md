## What This Changes

The PR adds an argv-aware guard in `registerNodesCli` so built-in `nodes` subcommands skip plugin CLI registration while plugin-provided node subcommands and bare `nodes` still register lazily, and updates the plugin-registration tests.

## Best Possible Solution

Land the guarded nodes plugin-registration path after redacted real CLI proof shows built-in `nodes` commands avoid plugin initialization and plugin-owned node subcommands still resolve.

## Solution Assessment
