## Summary

Keep open for maintainer review: the PR is a focused candidate fix for the linked nodes CLI startup regression, and the new copied terminal proof is sufficient for the CLI behavior, but merging changes when nested plugin CLI registrars can replace built-in `nodes` command names.

## What This Changes

The PR adds an argv-based guard in `registerNodesCli` so built-in `openclaw nodes` subcommands skip plugin CLI/runtime registration while unknown plugin-provided node subcommands still trigger lazy plugin registration.

## Best Possible Solution

Land a narrow guard that preserves the fast built-in `nodes` path and explicitly settles whether plugin-owned node commands may overlap built-in node command names.

## Reproduction Assessment

Yes. Source inspection shows current main unconditionally enters plugin CLI registration inside `registerNodesCli`, and the linked issue plus PR proof show the observable slow/plugin-load path for `nodes status/list`.

## Solution Assessment

Yes, this is the right layer for the regression because the unconditional load is in `registerNodesCli`; the only remaining solution-fit question is whether built-in node command names are reserved from plugin-owned nested commands.
