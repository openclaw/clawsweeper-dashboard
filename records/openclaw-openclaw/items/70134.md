## Summary

Keep open. Current main still has the mtime-based ready-cache behavior, and this PR is a useful implementation candidate, but the current patch is not merge-ready because it misses active auth semantics that can affect provider discovery and generated models output.

## What This Changes

The PR replaces the models-config ready-cache auth-profile mtime input with a semantic auth-profile fingerprint and adds focused tests for auth-profile rewrites, SecretRefs, OAuth fields, parse/read errors, and models.json invalidation.

## Best Possible Solution

Land a semantic auth-profile fingerprint that mirrors the normalized credential semantics used by the persisted auth loader and provider discovery, preserves current main's existing fingerprint inputs, hashes discovery-sensitive secret material without plaintext exposure, and documents the user-visible cache behavior in the changelog.

## Reproduction Assessment

Yes. Current main has a direct source-level reproduction: rewrite auth-profiles.json with equivalent auth semantics but a new mtime, then ensureOpenClawModelsJson builds a different authProfilesMtimeMs-based key and reruns planning instead of hitting the ready cache.

## Solution Assessment

No. The semantic fingerprint direction is the right maintainable fix, but this patch must include OAuth access-token semantics that drive discovery and token SecretRef alias normalization before it can safely replace the mtime component.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Hash OAuth access tokens that drive discovery:** `src/agents/models-config.auth-fingerprint.ts:191-192`
  - body: The OAuth fingerprint records only access/refresh presence. Current main passes cred.access as discoveryApiKey during provider discovery, so rotating from one non-empty token to another can keep this ready-cache key warm while the discovered catalog or entitlements differ. Include a non-plaintext hash of the discovery-sensitive access token, or an equivalent semantic signal.
  - confidence: 0.88
- **[P2] Normalize token SecretRef aliases before fingerprinting:** `src/agents/models-config.auth-fingerprint.ts:177-180`
  - body: The persisted auth loader treats a SecretRef object stored in token as tokenRef, and the resolver uses tokenRef as an active credential. This branch fingerprints token profiles from credential.tokenRef plus string credential.token, so a legacy token SecretRef object can change the effective credential without changing the fingerprint. Normalize token inputs the same way the loader does.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/models-config.ts:10`
  - body: This is a user-facing models cache fix with visible latency/cache behavior, but the PR does not update CHANGELOG.md. Add a single-line active-version entry with contributor credit so release notes and credit checks stay aligned with repo policy.
  - confidence: 0.91
