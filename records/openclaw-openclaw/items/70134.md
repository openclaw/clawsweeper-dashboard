## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Hash OAuth access tokens that drive discovery:** `src/agents/models-config.auth-fingerprint.ts:208-209`
  - body: The OAuth branch records only access/refresh presence, but current provider discovery uses the actual OAuth access token as discoveryApiKey. Rotating from one non-empty token to another can keep the ready-cache key warm while discovered catalogs or entitlements differ, leaving models.json stale.
  - confidence: 0.88
- **[P2] Normalize token SecretRef aliases before fingerprinting:** `src/agents/models-config.auth-fingerprint.ts:193-198`
  - body: Persisted auth loading treats a SecretRef object stored in token as tokenRef, and the resolver uses tokenRef as an active credential. This branch fingerprints token profiles from credential.tokenRef plus string credential.token, so a legacy token SecretRef object can change the effective credential without changing the fingerprint.
  - confidence: 0.84
