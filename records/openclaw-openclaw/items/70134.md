## Summary

Keep this PR open. Current main and the latest release still use `auth-profiles.json` mtime in the models-config ready-cache fingerprint, so the branch is not obsolete; however, the patch needs maintainer review because its semantic OAuth token treatment can leave discovery-sensitive auth changes out of the cache key and the branch is stale against newer current-main fingerprint inputs.

## What This Changes

The PR replaces the models-config ready-cache auth-profile mtime input with a semantic auth-profile fingerprint and adds focused tests for cache invalidation across auth rewrites, SecretRefs, OAuth fields, parse/read errors, and `models.json` changes.

## Best Possible Solution

Land a semantic auth-profile fingerprint that follows the same normalized credential semantics used by provider discovery, preserves profile order, includes every current-main fingerprint dimension, and hashes any discovery-sensitive secret material without exposing plaintext.

## Reproduction Assessment

Yes. The source path is direct: current main puts `authProfilesMtimeMs` into the ready-cache fingerprint, and `ensureOpenClawModelsJson` returns early on matching cache keys; a no-op rewrite that changes only `auth-profiles.json` mtime is enough to force a cache miss and replanning.

## Solution Assessment

No, not in the current form. The semantic fingerprint direction is the right fix, but this patch should include OAuth auth material that can affect discovery and be rebased onto current main without dropping the newer fingerprint inputs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Hash OAuth tokens that drive discovery:** `src/agents/models-config.auth-fingerprint.ts:208-209`
  - body: The OAuth fingerprint records only whether `access` and `refresh` are present. Current provider discovery receives `cred.access` as `discoveryApiKey`, so rotating to a different non-empty token with the same identity and expiry bucket can keep the ready-cache key warm while discovered models or entitlements change. Include a non-plaintext hash of the discovery-sensitive token material, or another equivalent semantic signal, before relying on this cache key.
  - confidence: 0.87
