## Summary

Keep open. Current main still has the reported logger startup failure path, and the latest shipped tag has the same raw `logging.file` handling; the linked fix PR is closed unmerged, so there is still source work to do.

## What This Changes

The issue requests restoring leading-tilde expansion for `logging.file` so gateway startup does not crash when the configured log path begins with `~`.

## Best Possible Solution

Land a focused logger-boundary fix that expands leading `~` before rolling-path detection and directory creation, with regression coverage for ordinary and rolling `logging.file` paths; leave any larger config-path audit as a separate follow-up.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: `readLoggingConfig()` returns the raw logging object, `resolveSettings()` copies `cfg.file`, and `buildLogger()` creates the parent directory for a non-expanded `resolveActiveLogFile(settings.file)`.

## Solution Assessment

Yes. Reusing the existing `expandHomePrefix()` helper at the logger filesystem boundary is the narrowest maintainable solution for this startup crash; rewriting config loading or auditing every path-like setting is not required to stop this failure.

## Repair Work Prompt

Fix openclaw/openclaw#73587. Current main still passes `logging.file` values such as `~/.openclaw/logs/gateway.log` through `src/logging/logger.ts` without leading-tilde expansion before `fs.mkdirSync(path.dirname(activeFile))`, causing gateway startup failures. Keep the repair focused on the logger boundary: reuse `expandHomePrefix` from `src/infra/home-dir.ts` in or before `resolveActiveLogFile()`, preserve rolling log filename behavior, add targeted regression tests for non-rolling and rolling `logging.file` values beginning with `~`, and add a user-facing `CHANGELOG.md` fix entry crediting the reporter/source author as appropriate. Treat closed PR #73614 as useful source context, but do not broaden into a full config-path audit unless maintainers ask. Do not change dependency patches, release metadata, or unrelated config path handling.
