## Summary

Keep open: current main does not implement Signal live tool-progress via an edit-free send/delete lifecycle, and the request is a new Signal plugin capability that needs maintainer UX/API judgment rather than cleanup closure.

## What This Changes

Request adding Signal channel live tool-call progress using send-and-delete progress messages, either as a Signal-owned opt-in feature or a broader pipeline abstraction.

## Best Possible Solution

If maintainers accept the UX, implement this in the Signal plugin as an opt-in progress lifecycle using existing reply progress callbacks plus a Signal delete helper, with docs, config schema, tests, and changelog aligned.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current Signal wiring has typing and normal reply delivery but no edit-free tool-progress message lifecycle to reproduce as a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable path appears to be a Signal-owned opt-in implementation, while a shared tombstone preview abstraction should wait for evidence that multiple channels need it.
