## Summary

Keep open: current main does not implement Signal live tool-call progress via an edit-free send/delete lifecycle, and the request is a new Signal plugin capability that needs maintainer UX/API judgment rather than cleanup closure.

## What This Changes

Add opt-in Signal live tool-call progress by sending temporary progress messages during tool-heavy turns and deleting them when the final reply is queued.

## Best Possible Solution

If maintainers accept the UX, implement this in the Signal plugin as an opt-in progress lifecycle using existing reply progress callbacks plus a Signal delete helper, with config schema, docs, tests, and changelog aligned; reserve a shared tombstone abstraction for later multi-channel evidence.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current Signal wiring has typing and normal reply delivery but no edit-free tool-progress message lifecycle to reproduce as a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable path appears to be a Signal-owned opt-in implementation, while a shared tombstone preview abstraction should wait for evidence that multiple edit-free channels need it.
