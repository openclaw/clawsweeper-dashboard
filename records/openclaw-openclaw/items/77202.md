## Summary

Keep open. Current main does not implement Signal live tool-progress; adding send-and-delete progress would be a new Signal plugin capability/config with notification and deletion semantics that need maintainer product judgment.

## What This Changes

The issue requests live Signal tool-call progress by sending temporary progress messages and deleting them when the final reply is queued.

## Best Possible Solution

Keep the issue open for maintainer review; if accepted, implement a plugin-local Signal progress mode using existing reply callbacks, document the config, and validate remote deletion live before considering any shared pipeline abstraction.

## Reproduction Assessment

Not applicable as a bug reproduction: the issue requests a new Signal progress capability. Source and docs confirm current main only exposes Signal typing/block streaming, not live tool-progress previews.

## Solution Assessment

Unclear until maintainer review. The channel-side opt-in is the narrowest promising direction if the UX trade-off is accepted, while the shared pipeline abstraction is broader than needed for this Signal-only request.
