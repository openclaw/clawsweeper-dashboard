## Summary

Keep open: current main still lacks a Signal edit-free tool-progress lifecycle, and the reporter's v2 coalescing update reduces notification noise but leaves a maintainer UX/config decision.

## What This Changes

Request adds live Signal tool-call progress using send/delete tombstone messages, optionally coalesced, instead of relying on editable draft previews.

## Best Possible Solution

If maintainers accept the UX, implement this as a Signal-owned opt-in rolling progress lifecycle with a delete helper, callback injection, config/schema/docs/tests, and defer a shared tombstone abstraction until another channel needs it.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current Signal turn handling has typing and normal reply delivery, but no progress tombstone lifecycle to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable direction appears to be a Signal-owned opt-in rolling send/delete implementation rather than a shared pipeline tombstone abstraction right now.
