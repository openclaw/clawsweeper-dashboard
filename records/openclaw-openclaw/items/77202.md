## Summary

Keep open: current main still lacks Signal live tool-progress via an edit-free send/delete lifecycle, and this is a new Signal plugin capability that needs maintainer UX/API judgment before implementation.

## What This Changes

Request to add Signal live tool-call progress using per-event status messages that are deleted when the final reply is queued.

## Best Possible Solution

If maintainers accept the UX trade-off, implement this in the Signal plugin as an opt-in progress lifecycle with a delete helper, config/schema/docs, focused extension tests, and a changelog entry.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current Signal wiring has typing and normal reply delivery but no edit-free tool-progress message lifecycle to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable path appears to be a Signal-owned opt-in implementation, while a shared tombstone preview abstraction should wait for evidence that multiple edit-free channels need it.
