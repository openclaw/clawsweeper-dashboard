## Summary

Keep open: current main still lacks Signal live tool-progress via an edit-free send/delete lifecycle, and this is a new Signal plugin capability that needs maintainer UX/API judgment before implementation.

## What This Changes

Request to add Signal live tool-call progress by sending temporary progress messages and deleting them when the final reply is queued.

## Best Possible Solution

If maintainers accept the UX, implement this as a Signal-owned opt-in progress lifecycle using existing reply progress callbacks plus a Signal delete helper, and only generalize the pipeline after another edit-free channel needs the same pattern.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection confirms current Signal dispatch supports typing and normal reply delivery but not the requested send/delete tool-progress lifecycle.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable path appears to be a Signal-owned opt-in implementation; a shared tombstone preview abstraction should wait for evidence that multiple channels need it.
