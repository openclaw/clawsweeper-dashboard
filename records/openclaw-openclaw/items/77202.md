## Summary

Keep open: current main still lacks a Signal edit-free progress send/delete lifecycle, and the reporter’s v2 follow-up usefully narrows the notification-noise concern without removing the need for maintainer UX/API choice.

## What This Changes

Request to add Signal live tool-call progress using an edit-free send, track, and delete pattern, likely behind a Signal progress config knob.

## Best Possible Solution

If accepted, implement the feature in the Signal plugin as an opt-in rolling progress lifecycle using existing reply callbacks plus a Signal remoteDelete helper, with schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has normal Signal reply delivery and typing indicators, but no Signal progress send/delete lifecycle, config knob, or docs support for this behavior.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable direction appears to be a Signal-owned opt-in implementation; a shared tombstone preview abstraction should wait until another edit-free channel needs the same seam.
