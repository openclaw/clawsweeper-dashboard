## Summary

Keep open: current main does not implement Signal live tool-call progress with an edit-free send/delete lifecycle, and the request is a new Signal plugin UX/config capability rather than cleanup-close material.

## What This Changes

Add opt-in live tool-call progress for the Signal channel by sending standalone progress messages and deleting them when the final reply is queued.

## Best Possible Solution

If maintainers accept the UX, implement this inside the Signal plugin as an opt-in progress lifecycle with a delete helper, config/schema/docs coverage, focused tests, and a changelog entry.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current Signal delivery has typing/read-receipt/reply handling but no edit-free tool-progress message lifecycle to reproduce as a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable direction appears to be a Signal-owned opt-in implementation; a shared tombstone preview abstraction should wait until another edit-free channel needs the same behavior.
