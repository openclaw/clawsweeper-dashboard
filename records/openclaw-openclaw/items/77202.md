## Summary

Keep open: current main still lacks a Signal send/delete tool-progress lifecycle, and the v2 milestone/coalescing update makes the proposal more concrete, but this remains a new Signal plugin UX/config feature that needs maintainer product and API judgment.

## What This Changes

The request asks to add opt-in Signal live tool-call progress using send/delete tombstone messages, with a later v2 patch coalescing milestones into one rolling progress message.

## Best Possible Solution

If maintainers accept the UX, implement a Signal-owned opt-in milestone/coalesced send/delete progress lifecycle with a local delete helper, config schema/docs, regression tests, and changelog; reserve a shared tombstone pipeline abstraction until another channel needs it.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection on current main confirms Signal has typing and final reply delivery but no edit-free live tool-progress lifecycle.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The narrowest maintainable direction appears to be a Signal-owned opt-in v2-style milestone/coalesced send/delete implementation, while a shared tombstone abstraction should wait for more channel demand.
