## Summary

Keep open: current main still lacks a Signal send/delete live tool-progress lifecycle, and adding it is a new Signal UX/API feature with config, notification-noise, deletion, and native/container support decisions for maintainers.

## What This Changes

Request to add live Signal tool-call progress using coalesced send-and-delete messages so long tool-heavy turns show progress before the final reply.

## Best Possible Solution

If maintainers accept the UX, implement an opt-in coalesced Signal plugin progress feature with delete helpers, aligned config/docs/tests, and live native/container proof.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that Signal supports replies and typing indicators, but has no send/delete tool-progress lifecycle to reproduce as a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The reporter's v2 coalesced Signal-side pattern looks like the narrowest implementation if accepted, while a shared tombstone preview abstraction should wait for broader edit-free channel demand.
