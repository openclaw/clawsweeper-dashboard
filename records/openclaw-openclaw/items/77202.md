## Summary

Keep open: current main still lacks Signal live tool-call progress via an edit-free send/delete lifecycle, and the reporter's v2 rolling-message patch context is useful but still needs maintainer UX/API choice before implementation.

## What This Changes

Add opt-in live Signal tool-call progress by sending temporary progress messages and deleting or replacing them when the final reply is queued.

## Best Possible Solution

Implement this in extensions/signal as an opt-in coalesced milestone progress lifecycle with a Signal delete helper, docs/config alignment, tests, and changelog; defer a shared tombstone-preview abstraction until another edit-free channel needs it.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms Signal currently has typing indicators and normal reply delivery, but no live tool-progress send/delete lifecycle to reproduce as broken documented behavior.

## Solution Assessment

Unclear until maintainers choose the UX and API shape. The narrowest maintainable path appears to be a Signal-owned opt-in implementation, while a shared tombstone pipeline abstraction should wait for evidence that multiple channels need it.
