## Summary

Keep open: current main still lacks a Signal send/delete tool-progress lifecycle, and the request is a new Signal plugin capability with config and UX tradeoffs rather than cleanup closure.

## What This Changes

Add Signal live tool-call progress by sending temporary progress messages during tool-heavy turns and deleting them when the final reply is queued.

## Best Possible Solution

Implement a Signal-owned, opt-in progress lifecycle after maintainer UX approval, with a Signal remoteDelete helper, callback wiring, config/docs/schema updates, tests, and changelog.

## Reproduction Assessment

Not applicable as a feature request. Source inspection verifies the current absence of Signal progress callbacks and a delete lifecycle, but this is not a broken existing Signal contract.

## Solution Assessment

Unclear until maintainers choose the UX and API shape. The narrowest maintainable direction appears to be a Signal-owned opt-in implementation, while a shared tombstone preview abstraction should wait for multi-channel demand.
