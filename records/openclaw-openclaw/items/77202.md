## Summary

Keep open: current main still lacks Signal live tool-call progress using an edit-free send/delete lifecycle, and the newest v2 proposal reduces notification noise but still needs maintainer UX and config-shape judgment.

## What This Changes

Add optional Signal live tool-call progress by sending temporary progress messages and deleting or replacing them when the real reply is queued.

## Best Possible Solution

Implement an opt-in Signal-owned progress lifecycle if maintainers accept the UX, using existing reply progress callbacks plus a delete helper, with schema/docs/tests/changelog aligned.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current Signal supports typing and normal reply delivery but has no temporary send/delete tool-progress lifecycle to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the UX and API shape. The narrowest maintainable direction appears to be a Signal plugin opt-in, while a shared tombstone preview abstraction should wait until another edit-free channel needs the same behavior.
