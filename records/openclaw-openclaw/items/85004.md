- **[P1] Preserve plugin policy during migration discovery:** `src/plugins/manifest-contract-eligibility.ts:26-27`
  - body: The new allowDisabled branch returns true for any non-bundled migration-provider plugin before isInstalledPluginEnabled runs. That makes contract resolution treat disabled or policy-blocked installed plugins as eligible owners, which can widen the required plugin set and bypass the manifest contract layer's normal policy boundary. Keep policy filtering in place and make any intended external-provider exception explicit.
  - confidence: 0.88
- **[P1] Keep bundled compat scoped to bundled providers:** `src/plugins/migration-provider-runtime.ts:67`
  - body: Passing resolution.pluginIds into resolveMigrationProviderConfig feeds installed/external ids into withBundledPluginEnablementCompat. In the plugins.enabled:false case this rewrites the config to enable plugins and adds enabled entries for those installed providers before loadOpenClawPlugins, so migration commands can import disabled native plugin code. Keep this argument scoped to resolution.bundledCompatPluginIds or add an explicit external opt-in policy.
  - confidence: 0.92
