## Summary

Keep this PR open. Current main still has the queued-RPC window: device remove, token rotate, and token revoke respond before the microtask disconnect, while the per-request dispatch path only rechecks shared gateway auth. The PR is a narrow functional fix, but it is missing the required changelog entry for a user-visible/security gateway fix.

## What This Changes

The PR adds synchronous invalidation of affected device-token WebSocket clients before device remove/rotate/revoke success responses, adds a dispatch guard for invalidated clients, and extends gateway handler/request-context tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding the required Unreleased changelog entry and confirming the touched gateway tests/checks are green.
