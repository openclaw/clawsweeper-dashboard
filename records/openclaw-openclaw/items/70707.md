## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Recheck invalidation at final dispatch:** `src/gateway/server/ws-connection/message-handler.ts:1799-1807`
  - body: This guard runs before the async IIFE that imports and calls handleGatewayRequest. A pipelined request can pass this early check, then another request can rotate or revoke the token and mark the client invalidated before the first request resumes; because there is no second check inside the IIFE, that first request still reaches the handler with the revoked device-token client. Add the same invalidation close path immediately before handleGatewayRequest and cover that interleaving.
  - confidence: 0.86
- **[P3] Drop stale canvas fields from the security fix:** `src/gateway/server/ws-types.ts:17-19`
  - body: The PR adds canvasHostUrl, canvasCapability, and canvasCapabilityExpiresAtMs to GatewayWsClient even though current protocol docs say that deprecated compatibility path is unsupported. Keep this token-invalidation fix scoped and remove the stale type affordance.
  - confidence: 0.78
