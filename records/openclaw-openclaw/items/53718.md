## Summary

Keep open: current main still lacks the resolver behavior, the linked bug remains open, and this PR is the canonical open fix path after overlapping alternatives were closed. It needs contributor real-behavior proof and branch/review cleanup before merge, not closure.

## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to shared home-relative path resolution and extends resolver tests for env placeholder edge cases.

## Best Possible Solution

Land one focused shared resolver update that expands supported env placeholders before filesystem resolution, preserves unresolved/empty placeholder passthrough, covers both resolver helpers and the ClawHub install path, then close the linked bug.

## Reproduction Assessment

Yes, by source inspection on current main: `resolveUserPath` delegates to `resolveHomeRelativePath`, which expands only leading `~` and otherwise calls `path.resolve(trimmed)`. A configured `${XDG_CONFIG_HOME}/workspace` therefore remains literal before the skill install target is built.

## Solution Assessment

Yes, the shared resolver is the narrowest maintainable place to fix this because config, workspace, and skill-install paths already flow through it. The PR's current unknown/empty-placeholder passthrough semantics are safer than silently collapsing missing variables, but the branch still needs proof and cleanup before merge.
