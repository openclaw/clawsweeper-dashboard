## Summary

Keep open: current main still leaves environment placeholders literal in the shared user-path resolver, and this PR targets the right bug with focused resolver tests, but it needs external contributor real behavior proof and a branch refresh before merge.

## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to shared home-dir path resolution and expands resolver test coverage for braced, bare, unknown, inherited, empty, and tilde-valued env placeholders.

## Best Possible Solution

Land this PR or a maintainer-picked equivalent that expands supported env placeholders in the shared resolver, preserves unresolved-placeholder passthrough, verifies the real ClawHub install path, and then closes https://github.com/openclaw/openclaw/issues/53628.

## Reproduction Assessment

Yes, by source inspection. A configured `${XDG_CONFIG_HOME}/workspace` reaches `resolveUserPath` and then `path.resolve(trimmed)` without env substitution before ClawHub builds the workspace `skills/<slug>` target.

## Solution Assessment

Yes, with proof and branch cleanup before merge. Fixing the shared resolver is the narrowest maintainable solution because workspace and ClawHub install paths already use that helper; a one-off ClawHub output patch would leave other configured paths inconsistent.
