## Summary

Keep open: current main still lacks environment-placeholder expansion in the shared path resolver, and this PR targets the right helper for the linked ClawHub skill-install bug. Merge should wait for real behavior proof from the external contributor plus normal maintainer cleanup.

## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to `src/infra/home-dir.ts` path resolvers and adds resolver tests for known, unknown, inherited, empty, and tilde-valued placeholders.

## Best Possible Solution

Land one focused shared resolver fix that expands supported env placeholders before filesystem resolution, preserves tilde and unresolved-placeholder behavior, verifies the Docker/ClawHub install path, then close the linked bug and keep the superseded duplicate closed.

Yes. Source inspection on current main shows a configured `${XDG_CONFIG_HOME}/workspace` string reaches `resolveUserPath` and then `path.resolve(trimmed)` without env substitution before ClawHub appends `skills/<slug>`.

## Solution Assessment

Yes, with proof and cleanup before merge. Fixing the shared resolver is the narrowest maintainable path because workspace and ClawHub skill install targets already use that helper; a one-off ClawHub display or install patch would leave other configured paths inconsistent.
