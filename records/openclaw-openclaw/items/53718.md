Keep open: current main and the latest release still leave env placeholders literal in shared path resolution, while https://github.com/openclaw/openclaw/issues/53628 remains open and https://github.com/openclaw/openclaw/pull/58281 was closed as superseded in favor of this PR. ## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to shared home-relative path resolution and adds resolver tests for braced, bare, unknown, inherited, empty, and tilde-valued placeholders.

## Best Possible Solution

Land this PR or a maintainer-picked equivalent that expands supported env placeholders in the shared resolver, preserves unresolved-placeholder passthrough, proves the real ClawHub install path, and then closes https://github.com/openclaw/openclaw/issues/53628.

Yes, at source level. A configured `${XDG_CONFIG_HOME}/workspace` reaches `resolveUserPath`, then `resolveHomeRelativePath`, and the non-tilde branch calls `path.resolve(trimmed)` without env substitution; I did not run the full Docker install in this read-only review.

## Solution Assessment

Yes, once proof and branch cleanup are added. Fixing the shared resolver is the narrow maintainable solution because workspace and skill-install paths already depend on it; a ClawHub-only output patch would leave other configured paths inconsistent.
