## Summary

Keep open. Current main still treats environment placeholders as literal path text on the shared resolver path used by ClawHub skill installs, while the latest PR head addresses the reported gap and prior review edge cases; because it overlaps open PR #58281 and is not currently mergeable against main, this needs maintainer selection and rebase rather than closure.

## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to `resolveHomeRelativePath` and `resolveOsHomeRelativePath` in `src/infra/home-dir.ts` and adds focused resolver tests for placeholder, inherited-key, empty-value, and tilde-valued cases.

## Best Possible Solution

Land one focused shared path-resolver update that expands supported env placeholders before filesystem resolution, preserves deliberate passthrough for unresolved or unsafe placeholders, covers the active ClawHub workspace install path, then close #53628 and retire the unchosen overlapping PR.

## Reproduction Assessment

Yes. Current main has a high-confidence code reproduction: a configured workspace containing `${XDG_CONFIG_HOME}` flows through `resolveUserPath` into `resolveHomeRelativePath`, which only expands `~`, then ClawHub installs under the resulting literal workspace string.

## Solution Assessment

Yes.
