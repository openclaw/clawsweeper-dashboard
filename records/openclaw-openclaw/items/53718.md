## Summary

Keep open. Current main still leaves environment placeholders literal in the shared user-path resolver, and this PR is useful source work for the linked ClawHub install bug, but it needs small review cleanup and coordination with the overlapping open resolver PR.

## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to `resolveHomeRelativePath` and `resolveOsHomeRelativePath`, plus tests for known, unknown, inherited, empty, OS-home, and tilde-valued environment placeholders.

## Best Possible Solution

Land one focused shared resolver update that expands supported env placeholders before filesystem resolution, preserves passthrough for unresolved or unsafe placeholders, covers the ClawHub workspace install path, adds the changelog entry, then close #53628 and retire the unchosen overlapping PR.

Yes. Current main has a high-confidence code reproduction: a configured workspace containing `${XDG_CONFIG_HOME}` flows through `resolveUserPath` into `resolveHomeRelativePath`, which only expands `~`, so the skill install target keeps the literal placeholder.

## Solution Assessment

Yes. Fixing the shared resolver is the right boundary for this bug, and this PR's current semantics are safer than collapsing unresolved placeholders; the remaining cleanup is changelog/comment polish plus maintainer selection against #58281.
