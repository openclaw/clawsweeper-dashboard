## Summary

Keep open: the linked CLI lifecycle bug is valid, but this PR should not merge as-is because its early ownership gate uses `cliCommandCatalog` as the built-in root source and would reject built-in roots such as `infer` and `capability`.

## What This Changes

The PR adds an early CLI root ownership preflight before proxy/plugin startup, narrows plugin CLI runtime loading for unowned primaries, adds regression tests, and records a changelog fix.

## Best Possible Solution

Keep the early unknown-root fast-fail, but derive built-in ownership from the actual sub-CLI/core command descriptors or make the catalog authoritative and covered before rejecting.

## Reproduction Assessment

Yes. Source inspection shows `openclaw infer ...` is a built-in sub-CLI and proxy-eligible today, while the PR would reject it because it is absent from `cliCommandCatalog` and has no plugin manifest owner.

## Solution Assessment

No. The fast-fail direction is right, but the proposed catalog-only built-in ownership test is not the narrowest maintainable fix because current built-in roots are owned by sub-CLI descriptors and registration helpers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the actual built-in CLI root set:** `src/cli/run-main.ts:266`

Please derive built-in ownership from the sub-CLI/core descriptors or make the catalog complete and covered.
