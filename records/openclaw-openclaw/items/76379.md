## Summary

Keep open: this PR targets a real current-main CLI bug, but the current patch needs a compatibility repair because it rejects plugin CLI roots that are registered through the documented `api.registerCli` command metadata path unless they also have manifest activation ownership.

## What This Changes

The PR adds a pre-proxy unknown-root check for CLI commands, narrows plugin CLI loading when no manifest owner is found, updates CLI/plugin regression tests, and adds a changelog entry for `openclaw foo`.

## Best Possible Solution

Land a narrow CLI fix that rejects truly unknown roots before proxy/runtime plugin loading while preserving documented plugin CLI command ownership through metadata, descriptors, or a compatible non-broad fallback, then close the linked issue after merge verification.

## Reproduction Assessment

Yes, at source level. The linked issue has concrete `openclaw foo` process-group logs, and current main still shows unknown roots entering proxy startup plus broad plugin CLI loading; I did not run a live smoke in this read-only review.

## Solution Assessment

No, not as written. The fast-fail direction is right, but manifest-only ownership would break documented plugin CLI command roots; the safer fix should preserve those roots while still avoiding broad runtime plugin loading for truly unknown commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve runtime CLI command roots:** `src/plugins/cli-registry-loader.ts:100-105`
  - body: This new empty-registry path means any primary command without a manifest activation or command-alias owner disappears before runtime CLI registration. Current docs say `api.registerCli(..., { commands: [...] })` declares explicit command roots and that missing activation metadata should not change correctness, and current tests preserve the full-load fallback for that case. Please keep the `openclaw foo` fast-fail, but still allow plugin CLI roots that are owned by CLI metadata/runtime registrations or update the public contract with migration coverage.
