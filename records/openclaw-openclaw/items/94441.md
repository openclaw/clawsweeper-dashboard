The branch makes explicit missing local gateway exec workdirs fail before command validation/execution, preserves implicit/default workdir fallback, and adds a regression test.

## Best Possible Solution

Land one canonical exec fix that prevents explicit invalid workdirs from executing in a fallback directory, either by accepting this fail-fast behavior or by choosing a tilde-expansion alternative with equivalent regression coverage.

## Solution Assessment

Yes, this is an acceptable maintainable fix if maintainers want fail-fast semantics: it removes fallback execution for explicit invalid local workdirs without adding shell expansion. The safer remaining decision is whether this should be the canonical behavior instead of leading-tilde expansion.
