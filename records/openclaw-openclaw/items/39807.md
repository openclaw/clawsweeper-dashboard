## Summary

Keep open. Current main and the latest release still leave inline `models.providers.<id>.apiKey` credentials outside profile-backed billing cooldown tracking, and an open closing PR is the active fix path.

## What This Changes

Apply billing cooldown/backoff to inline provider API keys so 402 billing failures stop immediate retries and allow fallback routing.

## Best Possible Solution
