## Summary

Keep open. Current main still resolves inline provider API keys without a cooldown identity, skips failure marking when no auth profile id exists, and the closing fix at https://github.com/openclaw/openclaw/pull/80482 is still open rather than merged.

## What This Changes

The issue asks OpenClaw to apply billing cooldown/backoff to providers configured only with inline `models.providers.<id>.apiKey` credentials so 402 errors do not retry indefinitely.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80482 or an equivalent provider-level inline API-key cooldown fix, including literal inline keys and SecretRef-backed inline keys, then close from merged main or release proof.

## Reproduction Assessment

Yes, source-level. Configure `models.providers.<id>.apiKey` without an auth profile, trigger a billing-classified failure, and current main resolves no `profileId`, clears explicit-config profile candidates, then skips failure marking because `profileId` is absent.

## Solution Assessment

Yes for triage. Keeping this issue open while the linked PR is reviewed, updated, or replaced is the maintainable path because current main still lacks inline-key cooldown storage and skip logic.
