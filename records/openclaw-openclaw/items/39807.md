## Summary

Keep open. Current main still leaves inline `models.providers.<id>.apiKey` credentials outside the billing cooldown path, and https://github.com/openclaw/openclaw/pull/80482 is the active unmerged fix path with a recorded review blocker.

## What This Changes

Billing failures from providers configured only with inline `models.providers.<id>.apiKey` should trigger cooldown/backoff instead of being retried indefinitely.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80482 with a stable provider-level cooldown identity for inline API keys, including literal and configured env SecretRef cases, then close this issue from merged implementation proof.

## Reproduction Assessment

Yes, source-level. Configure a provider only with `models.providers.<id>.apiKey`: current main resolves that key without a profile id, skips profile candidates for explicit config API-key auth, and records billing failures only through profile-backed state.

## Solution Assessment

Yes for routing. Keeping this issue open while the linked PR is reviewed, fixed, or replaced is the narrow maintainable path; closing before merge would leave the reported retry/backoff bug unresolved on main.
