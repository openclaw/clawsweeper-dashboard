Keep open: current main supports Anthropic API key, Claude CLI reuse, and manual setup-token paths, but it still does not provide the requested one-click Anthropic browser OAuth onboarding or automatic Pro/API account-type setup. ## What This Changes

Request one-click Anthropic authentication that opens a browser, signs in with Anthropic, detects account type, and configures OpenClaw automatically.

## Best Possible Solution

Add any approved Anthropic browser auth as an Anthropic-provider-owned method that preserves API-key, Claude CLI, and setup-token paths; keep runtime token-prefix handling separate from the product decision for one-click onboarding.
