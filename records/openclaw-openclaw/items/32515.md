## Summary

Keep open: current main still lacks the requested one-click Anthropic browser OAuth onboarding and automatic account-type setup. The existing Anthropic surfaces cover Claude CLI reuse, manual setup-token/token entry, and API-key setup; the commenter’s prefix-detection suggestion is useful but narrower than the requested UX, and the related OAuth/header-regression threads do not resolve this onboarding feature.

## What This Changes

Request a one-click Anthropic sign-in flow from onboarding that opens a browser, detects subscription/API account type, and configures auth automatically.

## Best Possible Solution

Confirm Anthropic’s supported auth contract, then add any approved flow as an Anthropic-provider-owned auth method using the existing OAuth hooks while preserving API-key, Claude CLI, and setup-token paths.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection on current main shows available Anthropic auth routes but no failing current-main path to reproduce.

## Solution Assessment

Unclear as a direct implementation request because the full browser flow depends on Anthropic-supported endpoint, scope, refresh, and account-type semantics. The maintainable path is a provider-owned auth method after product/security confirmation, with prefix detection handled separately if maintainers want that narrower UX.
