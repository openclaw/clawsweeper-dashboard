## Summary

Keep open: current main has improved Anthropic auth through API-key, Claude CLI, setup-token, and token-runtime handling, but it still does not provide the requested one-click Anthropic browser OAuth flow with account-type detection.

## What This Changes

The issue requests replacing Anthropic setup with a one-click browser OAuth flow that detects account type and configures credentials automatically.

## Best Possible Solution

Confirm Anthropic's supported OAuth/setup contract, then implement any approved browser sign-in as an Anthropic-provider-owned auth method with docs while preserving API-key and Claude CLI paths.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection on current main shows Anthropic auth methods are Claude CLI, setup-token, and API key rather than a browser OAuth flow.

## Solution Assessment

Unclear as a direct implementation request because the full browser flow depends on Anthropic-supported endpoint, scope, refresh, and account-type semantics. The maintainable direction is provider-owned auth work in extensions/anthropic, with prefix-based paste autodetection treated as a narrower follow-up if maintainers want it.
