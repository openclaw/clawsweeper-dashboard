## Summary

Keep open: current main has improved Anthropic auth via API-key, Claude CLI, and setup-token/token paths, but it does not implement the requested one-click Anthropic browser OAuth flow with account-type detection.

## What This Changes

Request a one-click Anthropic onboarding flow that opens a browser, signs in with Anthropic, detects API-key versus OAuth/subscription account type, and configures OpenClaw automatically.

## Best Possible Solution

Add an Anthropic-provider-owned browser auth method only after confirming the supported Anthropic OAuth/setup contract; treat prefix-based paste/autodetection as a narrower follow-up if it can be shipped independently.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection verifies current main lacks an Anthropic browser OAuth auth method while retaining API key, Claude CLI, and setup-token paths.

## Solution Assessment

Unclear as a direct implementation request because the full browser flow depends on Anthropic-supported endpoint and scope semantics. The maintainable direction is provider-owned auth work in `extensions/anthropic`, with a smaller prefix-detection improvement considered separately.
