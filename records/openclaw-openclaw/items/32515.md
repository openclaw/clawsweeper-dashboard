## Summary

Keep open: current main still lacks one-click Anthropic browser OAuth onboarding and automatic Pro/API account-type configuration. Current Anthropic setup is API key, Claude CLI reuse, or setup-token/manual token entry; the related closed bug only covered runtime OAuth/header handling, so this remains a provider/auth product decision rather than conservative cleanup.

## What This Changes

The issue requests a one-click Anthropic browser OAuth onboarding flow that detects account type and configures Anthropic auth automatically.

## Best Possible Solution

Add an approved Anthropic-provider-owned browser auth method only after maintainer and provider-contract confirmation, keep the existing API-key, Claude CLI, and setup-token paths, and treat token-prefix UX as a separate bounded improvement if desired.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection verifies the current auth routes, but there is no failing current-main path to reproduce.

## Solution Assessment

Unclear as proposed because replacing the whole setup depends on Anthropic-supported browser OAuth, account semantics, scopes, and refresh behavior. The maintainable path is an additive provider-owned auth method after product/security approval, not a broad replacement of current auth paths.
