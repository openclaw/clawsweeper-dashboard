## Summary

Keep open: current main and v2026.5.22 still offer Anthropic API-key, Claude CLI reuse, and manual setup-token paths, but not the requested first-party Anthropic browser OAuth onboarding flow. Existing OAuth/provider-auth infrastructure is present, so the remaining work is Anthropic-specific product and credential-security design rather than cleanup.

## What This Changes

The issue requests one-click Anthropic browser OAuth onboarding that automatically configures Anthropic account type and credentials without manual keys, setup tokens, or environment files.

## Best Possible Solution

Keep this open as the Anthropic auth UX umbrella and only add an optional Anthropic-owned browser or device-code auth method after provider-contract and credential-security review, while preserving existing API-key, Claude CLI, and setup-token routes.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and v2026.5.22 release inspection show the requested Anthropic browser sign-in option is absent while existing manual and CLI routes remain available.

## Solution Assessment

Unclear as proposed: replacing the entire Anthropic setup flow is not the narrowest maintainable path. The safer solution is an optional provider-owned OAuth or device-code method with explicit security review and no removal of existing supported auth routes.
