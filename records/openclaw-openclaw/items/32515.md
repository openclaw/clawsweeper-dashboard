## Summary

Keep open: current main still does not provide the requested one-click Anthropic browser OAuth onboarding flow or automatic account-type setup. The current product surface supports API-key setup, Claude CLI reuse, and manual setup-token/token compatibility, while the related OAuth regression and stale PR do not resolve the requested onboarding UX.

## What This Changes

The issue requests a one-click Anthropic authentication flow where onboarding opens a browser sign-in and automatically configures API-key versus subscription-token usage.

## Best Possible Solution

Confirm Anthropic’s supported auth contract, then add any approved flow as an Anthropic-provider-owned auth method with docs and focused tests while preserving API-key, Claude CLI, and setup-token compatibility.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection on current main shows the available Anthropic auth routes but no failing current-main path to reproduce.

## Solution Assessment

Unclear as a direct implementation request because the full browser flow depends on Anthropic-supported endpoint, scope, refresh, and account-type semantics. The narrow maintainable path is a provider-owned auth method after maintainer/product confirmation, with pasted-token detection split only if maintainers choose that smaller UX.
