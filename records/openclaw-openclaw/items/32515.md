## Summary

Keep open: current main supports Anthropic API-key, Claude CLI reuse, and manual setup-token/token paths, but not the requested one-click Anthropic browser OAuth sign-in. The remaining work is provider-auth product design plus credential/callback security review, especially with related PR https://github.com/openclaw/openclaw/pull/74990 still open as a partial Claude-subscription onboarding direction.

## What This Changes

The issue requests one-click Anthropic browser OAuth onboarding that detects account type and configures OpenClaw without manual keys, tokens, or config editing.

## Best Possible Solution

Keep this issue open as the Anthropic auth UX umbrella, review or replace the related subscription-onboarding PR if that direction is desired, and only ship an optional Anthropic provider-owned browser auth method after the provider contract and callback/token-storage security model are clear.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source and docs inspection show the requested Anthropic browser sign-in option is absent from current main.

## Solution Assessment

Unclear as proposed; replacing the entire Anthropic setup flow is not the narrowest maintainable solution. The safer direction is an optional Anthropic-owned auth method or subscription onboarding flow that preserves existing API-key, Claude CLI, and setup-token routes.
