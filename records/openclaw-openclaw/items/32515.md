## Summary

Keep open. Current main still offers Anthropic authentication through Claude CLI reuse, manual setup-token paste, or API key setup; it does not provide a one-click browser OAuth flow that detects account type and configures the profile automatically. The related closed runtime OAuth issue and the open Anthropic beta-header PR are adjacent, but neither implements this onboarding feature.

## What This Changes

The issue requests one-click Anthropic onboarding via browser OAuth with automatic Pro-vs-API account detection, with token-prefix auto-detection suggested as a smaller fallback.

## Best Possible Solution

Keep the issue open until maintainers decide the product direction for Anthropic sign-in. If Anthropic supports a sanctioned browser flow, implement it inside the Anthropic plugin as a provider-owned auth method using the existing OAuth helpers; otherwise document the vendor limitation and consider a separate, narrow API-key/setup-token prefix-routing improvement.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. There is a high-confidence current-main verification path by inspecting the Anthropic manifest, runtime auth methods, setup-token validator, and onboarding docs, all of which still lack the requested browser OAuth flow.

## Solution Assessment

Unclear for the full requested OAuth flow until Anthropic's sanctioned OAuth, scopes, refresh, and account-type contract are confirmed. The maintainable repo-side direction is provider-owned work in `extensions/anthropic` using the existing generic OAuth seams, with prefix auto-detection treated as a separate smaller improvement if needed.
