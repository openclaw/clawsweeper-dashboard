## Summary

Keep open. Current main and the latest release still expose Anthropic auth as Claude CLI reuse, manual setup-token, or API key, not a one-click browser OAuth flow with account-type detection; the narrower API-key prefix problem is separately tracked by #72121.

## What This Changes

The issue requests one-click Anthropic onboarding through browser OAuth with automatic Pro-vs-API account detection, with token-prefix detection suggested as a smaller fallback.

## Best Possible Solution

Keep this open until maintainers decide the Anthropic sign-in direction; if Anthropic supports a sanctioned browser flow, implement it inside the Anthropic plugin using the existing provider OAuth helpers and leave the API-key prefix fallback to the narrower related work.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main verification is high-confidence by source/docs inspection: Anthropic auth choices remain Claude CLI, setup-token, and API key, with no browser OAuth sign-in flow.

## Solution Assessment

Unclear. Provider-owned OAuth helper surfaces exist, but the full requested flow depends on Anthropic-sanctioned OAuth, scope/refresh semantics, and account-type detection; the safer fallback is the separate #72121 API-key routing work.
