## Summary

The requested UX is not fully present on current main, but OpenClaw already deliberately separates Anthropic API-key, Claude CLI reuse, and setup-token paths; exposing direct browser OAuth remains a provider-contract and security-sensitive product decision rather than a narrow missing detector.

## What This Changes

Add a one-click Anthropic browser sign-in during onboarding that detects subscription versus API access and configures credentials automatically.

## Best Possible Solution

Keep the explicit Claude CLI, setup-token, and API-key modes, streamline their onboarding copy and execution, and expose direct browser OAuth only after Anthropic’s third-party client and scopes contract plus the resulting security and billing UX are explicitly approved.

## Reproduction Assessment

Not applicable: this issue requests a new authentication experience rather than reporting broken behavior under an existing documented contract.

## Solution Assessment

No, not in the proposed automatic form. Prefix detection is already a transport detail, while the user-facing solution must preserve explicit subscription, API, and billing choices and follow Anthropic’s approved third-party authorization contract.
