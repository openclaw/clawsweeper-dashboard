## What This Changes

The issue requests a browser-based Anthropic sign-in flow that automatically configures OpenClaw without users choosing between API keys, OAuth tokens, environment files, or manual auth profiles.

Maintainers should define the Anthropic auth product contract first: either implement an upstream-supported browser OAuth/Claude-login path with secure credential storage and entitlement routing, or improve the existing setup-token/API-key flow with explicit docs and safe token-type detection.
