## Summary

Keep open: current main still lacks the requested one-click Anthropic browser OAuth onboarding and account-type auto-configuration. The existing Anthropic paths are API key, Claude CLI reuse, and setup-token/manual token entry; the commenter’s token-prefix idea is a useful smaller improvement, but it does not implement the requested onboarding flow and the related closed OAuth-header regression is a different runtime bug.

## What This Changes

Request one-click `openclaw onboard` Anthropic browser sign-in that detects subscription versus API-credit auth and configures credentials automatically.

## Best Possible Solution

Confirm Anthropic’s supported auth contract, then add any approved flow as an Anthropic-provider-owned auth method using the existing OAuth context while preserving API-key, Claude CLI, and setup-token paths.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection verifies the available current-main auth routes, but there is no failing current-main path to reproduce.

## Solution Assessment

Unclear as a direct “replace the entire auth setup” implementation because the full flow depends on Anthropic-supported endpoint, scope, refresh, and account-type semantics. The maintainable path is a provider-owned auth method after product/security confirmation, with prefix detection handled separately if maintainers want that smaller UX improvement.
