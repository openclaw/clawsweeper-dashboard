## Summary

Keep open: current main and v2026.6.8 still expose explicit Anthropic API-key, setup-token, and Claude CLI auth choices, while the requested one-click Anthropic browser OAuth onboarding is not wired into onboarding and remains a product/security decision.

## What This Changes

Request Anthropic onboarding that opens a browser, signs in with Anthropic, detects account or token mode, and configures OpenClaw without manual token or env setup.

## Best Possible Solution

Add an explicit Anthropic plugin-owned OAuth onboarding path only if maintainers approve its consent, scope, storage, refresh, billing, docs, and upgrade contract; keep API-key and Claude CLI choices explicit.

## Reproduction Assessment

Not applicable: this is a feature request for a new Anthropic authentication experience, not a bug in an existing contract. Source inspection confirms current main does not expose the requested browser OAuth onboarding path.

## Solution Assessment

No, not as a blanket replacement for all Anthropic auth setup. The maintainable path is an additive provider-owned OAuth option after the security, consent, billing, and upgrade contract is approved.
