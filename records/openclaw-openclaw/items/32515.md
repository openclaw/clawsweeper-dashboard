## Summary

Current main and v2026.6.5 support Anthropic API keys, setup-tokens, existing OAuth/token profiles, and Claude CLI credential reuse, but they still do not provide the requested one-click browser sign-in or automatic Pro-versus-API account setup. The request remains necessary and aligns with first-run UX goals, but direct OAuth client ownership, entitlement routing, and credential security require maintainer and upstream-contract decisions.

## What This Changes

Requests a one-click Anthropic browser OAuth sign-in that detects whether the user has a Claude subscription or API credits and configures authentication automatically.

## Best Possible Solution

Design a provider-owned one-click Anthropic flow that either invokes and imports the sanctioned Claude CLI login result or uses an explicitly supported Anthropic third-party OAuth contract, while preserving clear API-key billing and secure credential storage.

## Reproduction Assessment

Not applicable: this is a request for a new browser-based authentication capability. Current source, docs, and v2026.6.5 consistently show that direct Anthropic browser sign-in is absent rather than broken.

## Solution Assessment

Unclear: the one-click UX is a good target, but prefix-based auto-detection and an independently implemented OAuth flow are not yet proven to be the safest supported solution. The narrowest maintainable path is likely Claude CLI-mediated login unless Anthropic confirms a third-party OAuth contract.
