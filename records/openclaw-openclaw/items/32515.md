## What This Changes

The issue requests a one-click Anthropic browser OAuth onboarding flow that avoids manual API key or token setup.

## Best Possible Solution

Keep this issue open as the Anthropic auth UX umbrella, review or replace the related subscription-onboarding PR if that direction is desired, and only ship an optional Anthropic provider-owned browser auth method after the provider contract and callback/token-storage security model are clear.
