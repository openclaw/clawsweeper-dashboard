## Summary

Keep open: current main still does not provide one-click Anthropic browser OAuth onboarding or automatic account-type configuration. The available Anthropic setup paths are API key, Claude CLI reuse, and setup-token/manual token entry, so this remains a product/provider-contract decision rather than conservative cleanup.

## What This Changes

The request asks OpenClaw to add a one-click Anthropic OAuth onboarding flow that opens a browser, detects Pro/API account type, and configures auth automatically.

## Best Possible Solution

Add any approved Anthropic browser auth as an Anthropic-provider-owned method using the existing provider OAuth context, preserve the current API-key, Claude CLI, and setup-token paths, and handle token-prefix detection separately if maintainers want that smaller UX improvement.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection verifies the current auth routes, but there is no failing current-main path to reproduce.

## Solution Assessment

Unclear as proposed because replacing the entire auth setup depends on Anthropic-supported OAuth, scope, refresh, and account-type semantics. The maintainable path is a provider-owned auth method after product/security confirmation, with any prefix detection handled as a bounded follow-up if approved.
