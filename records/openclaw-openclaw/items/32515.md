## Summary

Keep open: current main and the latest shipped release still expose Anthropic Claude CLI reuse, setup-token/manual token, and API-key auth, but not one-click Anthropic browser sign-in with account-type auto-configuration. The related OAuth-header/runtime work and the closed Anthropic OAuth-login PRs are useful context, but they do not supersede this onboarding/product request.

## What This Changes

Request one-click Anthropic browser OAuth onboarding that automatically configures Pro/subscription versus API-key style Anthropic access.

## Best Possible Solution

Confirm Anthropic’s supported auth contract, then add any approved flow as an Anthropic-provider-owned auth method and wizard choice using the existing OAuth context while preserving API-key, Claude CLI, and setup-token paths.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken current behavior report. Source and release inspection verify the available Anthropic auth routes, but there is no failing current-main path to reproduce.

## Solution Assessment

Unclear as stated: replacing the whole setup with a single Anthropic OAuth flow depends on provider-supported endpoint, scope, refresh, security, and account-type semantics. The maintainable direction is a provider-owned auth method after product/security review, with token-prefix detection treated as a narrower separate UX improvement if desired.
