## What This Changes

The issue requests an OpenClaw Anthropic browser OAuth onboarding flow that detects subscription versus API-credit auth and configures credentials automatically.

## Best Possible Solution

Add any approved Anthropic browser auth as an optional Anthropic-provider-owned auth method, preserving API-key, Claude CLI, and setup-token paths.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source, docs, and release inspection give high confidence that the requested Anthropic browser OAuth setup is not currently available.

## Solution Assessment

Unclear as proposed; replacing the entire Anthropic setup flow is not the safest maintainable solution without provider-contract and auth-security approval. The better path is an optional provider-owned auth method that preserves existing auth routes.
