## Summary

Keep open: current `main` still offers Anthropic Claude CLI reuse, setup-token, and API-key setup, but not the requested one-click Anthropic browser OAuth flow or automatic account-type setup. The related closed OAuth regression covered runtime use of existing tokens, not this onboarding feature.

## What This Changes

The issue requests a one-click Anthropic OAuth browser sign-in flow that automatically detects and configures account type.

## Best Possible Solution

Confirm Anthropic's supported auth contract, then add any approved sign-in as an Anthropic-provider-owned auth method with docs and focused tests while preserving API-key, Claude CLI, and setup-token compatibility.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection on current `main` shows no Anthropic browser OAuth flow to reproduce, only Claude CLI, setup-token, and API-key setup paths.

## Solution Assessment

Unclear as a direct implementation request because the full browser flow depends on Anthropic-supported endpoint, scope, refresh, and account-type semantics. The maintainable direction is provider-owned Anthropic auth work, with pasted-token prefix detection as a narrower follow-up only if maintainers want it.
