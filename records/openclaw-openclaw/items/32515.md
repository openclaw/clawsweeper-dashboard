## Summary

Keep open: current main still lacks the requested one-click Anthropic browser OAuth/account-type onboarding flow. The implemented Anthropic paths are Claude CLI reuse, manual setup-token, and API key, while the related closed OAuth regression only covered runtime handling of existing tokens.

## What This Changes

The issue requests a one-click Anthropic browser OAuth onboarding flow that detects Pro/subscription versus API-key credentials and configures OpenClaw automatically.

## Best Possible Solution

Confirm Anthropic’s supported auth contract, then implement any approved sign-in as an Anthropic-provider-owned auth method with docs and focused tests while preserving API-key, Claude CLI, and setup-token compatibility.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection on current main shows supported Anthropic onboarding routes, but no failing current-main path to reproduce.

## Solution Assessment

Unclear as a direct implementation request because the full browser flow depends on Anthropic-supported endpoint, scope, refresh, and account-type semantics. The maintainable path is provider-owned Anthropic auth work after maintainer/product confirmation, with pasted-token prefix detection handled as a smaller follow-up only if desired.
