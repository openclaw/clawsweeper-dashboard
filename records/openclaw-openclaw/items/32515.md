## Summary

Keep open. Current main still has no one-click Anthropic browser OAuth onboarding path; the Anthropic plugin exposes CLI reuse, setup-token paste, and API-key auth, while the remaining request needs maintainer/product confirmation around Anthropic OAuth scope and account-type semantics.

## What This Changes

The issue requests replacing manual Anthropic API-key/setup-token setup with a browser sign-in flow that detects subscription versus API-key account type and configures auth automatically.

## Best Possible Solution

Keep this issue open for maintainer product review. If accepted, implement an Anthropic-owned browser auth method in the Anthropic plugin using the existing provider OAuth seams, persist credentials safely, choose subscription/API defaults explicitly, and update onboarding docs and tests; otherwise split the API-key prefix fallback to the narrower tracking issue and mark the browser flow blocked on Anthropic support.
