## Summary

Keep this PR open, but it still needs changes before merge: the skip is directionally useful for ordinary owning context engines, but as written it bypasses the explicit `promptAuthority: "preassembly_may_overflow"` contract.

## What This Changes

The PR skips the pre-prompt overflow precheck whenever the active context engine owns compaction and adds a run-loop overflow-retry regression test.

## Best Possible Solution

Skip the lossy pre-prompt estimate for ordinary owning engines, but still run the precheck when the engine explicitly returns `promptAuthority: "preassembly_may_overflow"`, with a focused regression for that combination.

## Reproduction Assessment

Yes. The PR-relevant blocker is reproducible by source inspection: an owning context engine returning `promptAuthority: "preassembly_may_overflow"` now hits the new skip before `shouldPreemptivelyCompactBeforePrompt`, bypassing `unwindowedMessages`.

## Solution Assessment

No as written. The narrow maintainable fix is to skip the estimated precheck for ordinary owning engines while preserving the explicit preassembly-overflow authority branch.
