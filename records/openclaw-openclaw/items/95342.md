- **[P1] Preserve the documented owning-engine precheck contract:** `src/agents/embedded-agent-runner/run/attempt.ts:4584-4587`
  - body: For `ownsCompaction` engines with the default `promptAuthority: "assembled"`, this condition skips `shouldPreemptivelyCompactBeforePrompt` entirely. Current types and docs say the runner still checks the assembled prompt estimate and can route predicted overflow through the active engine's `compact()` before provider submission, so existing plugins relying on that safeguard would only discover overflow at the provider call. Keep an assembled-prompt or engine-token budget gate here, or make the new skip behavior an explicit maintainer-approved contract with docs and tests.
  - confidence: 0.86
- **[P2] Cover the owning preassembly branch at attempt level:** `src/agents/embedded-agent-runner/run.overflow-compaction.loop.test.ts:845-848`
  - body: This test stubs `runEmbeddedAttempt` to return a precheck error, so it would still pass if the real attempt-level guard skipped every `ownsCompaction` engine. Add a focused attempt-level regression with `ownsCompaction: true` and `promptAuthority: "preassembly_may_overflow"` that proves the real precheck still receives `unwindowedMessages` or blocks submission.
  - confidence: 0.9
