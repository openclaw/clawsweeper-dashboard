## Summary

Keep this PR open, but do not merge as-is: the bug fix is plausible for ordinary `ownsCompaction` engines, but the unconditional skip drops an existing context-engine precheck contract for engines that explicitly request preassembly overflow authority.

## What This Changes

The PR changes `runEmbeddedAttempt` to skip the pre-prompt overflow precheck when the active context engine advertises `ownsCompaction`, and adds a small overflow-retry test for that path.

## Best Possible Solution

Keep the ownsCompaction skip for normal assembled prompts, but preserve the `preassembly_may_overflow` precheck path and cover that combination with a focused attempt test.

## Reproduction Assessment

Yes for the review finding via source inspection: an `ownsCompaction` engine that returns `promptAuthority: "preassembly_may_overflow"` now hits the new skip before `shouldPreemptivelyCompactBeforePrompt`, so the unwindowed precheck path is bypassed. I did not run a live session in this read-only review.

## Solution Assessment

No as written. The narrow maintainable fix is to skip the lossy precheck for ordinary owning-engine assembled prompts while preserving the explicit preassembly-overflow authority branch.
