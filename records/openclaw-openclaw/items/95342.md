## Summary

Keep this PR open, but not merge-ready: the fix targets a real ownsCompaction false-positive precheck problem, yet the current patch would bypass the existing explicit `promptAuthority: "preassembly_may_overflow"` contract.

## What This Changes

The PR skips the pre-prompt overflow precheck when an active context engine declares `ownsCompaction: true` and adds an overflow-retry regression test.

## Best Possible Solution

Skip the lossy estimate for ordinary owning-engine assembled prompts, but still run the precheck when the engine explicitly returns `promptAuthority: "preassembly_may_overflow"` and cover that combination with a focused regression.

## Reproduction Assessment

Yes. Source inspection reproduces the PR defect: with `ownsCompaction` true and `promptAuthority: "preassembly_may_overflow"`, the new skip branch runs before `unwindowedMessages` can reach the precheck.

## Solution Assessment

No as written. The maintainable fix is to skip the lossy estimate only for ordinary owning-engine assembled prompts while preserving the explicit preassembly-overflow authority branch.
