## Summary

Keep open: the runtime fix is plausible and the reported real logs are useful, but this compatibility-sensitive precheck change still lacks attempt-level coverage for the `ownsCompaction` plus `preassembly_may_overflow` contract. The added opt-in test mocks `runEmbeddedAttempt`, so it cannot prove the real branch still runs the precheck for owning engines that explicitly opt into preassembly overflow gating.

## What This Changes

The PR changes `runEmbeddedAttempt` to skip the pre-prompt overflow precheck for context engines that own compaction unless they opt into `preassembly_may_overflow`, and adds overflow run-loop tests.

## Best Possible Solution

Land the runtime skip only after focused attempt-level coverage proves owning engines skip the default precheck but still run the precheck with unwindowed messages when `promptAuthority` is `preassembly_may_overflow`.
