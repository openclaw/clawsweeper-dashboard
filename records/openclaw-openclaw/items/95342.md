## Summary

Keep open: current main still runs the pre-prompt precheck for owning context engines, so the reported fix remains useful, but this branch would also bypass the existing `promptAuthority: "preassembly_may_overflow"` guard that plugin authors can explicitly opt into.

## What This Changes

The PR adds an `ownsCompaction`-based skip/log around `runEmbeddedAttempt`’s pre-prompt overflow precheck and a run-loop regression test for model-overflow retry.

## Best Possible Solution

Skip the lossy estimate for ordinary owning-engine assembled prompts, but still run the precheck when the engine explicitly returns `promptAuthority: "preassembly_may_overflow"` and cover that combination with a focused regression.

## Reproduction Assessment

Yes. Source inspection reproduces the patch defect: with `ownsCompaction` true and `promptAuthority: "preassembly_may_overflow"`, the new skip branch runs before `unwindowedMessages` can reach the precheck.

## Solution Assessment

No as written. The maintainable fix is to skip the lossy estimate only for ordinary owning-engine assembled prompts while preserving the explicit preassembly-overflow authority branch.
