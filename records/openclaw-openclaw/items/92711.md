## Summary

## What This Changes

The PR changes the embedded attempt session-lock controller so post-prompt reacquire verifies the fence without taking a retained write lock, factors fence capture into a helper, and updates two lock-lifecycle test expectations.

## Best Possible Solution

Keep the short-lived post-prompt write-lock direction, but only advance trusted fence state from held-lock or explicitly published owned-write paths, add the missing second-release regression, and require redacted queued follow-up proof before merge.

## Solution Assessment

No. The per-write post-prompt locking direction is maintainable, but the current PR also refreshes the fence from an unowned no-held-lock state; the safer fix keeps fence advancement tied to lock ownership or explicit owned-write publication.
