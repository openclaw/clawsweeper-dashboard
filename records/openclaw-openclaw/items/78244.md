## Summary

Keep open: the PR targets a valid doctor self-warning bug, but the early return still leaves configured-channel configs with an unset group visible-reply policy, which current doctor warning code still treats as default `message_tool` when the message tool is unavailable.

## What This Changes

This PR adds message-tool policy checks to the doctor legacy visible-reply normalizer so `doctor --fix` skips writing `messages.groupChat.visibleReplies: "message_tool"` when the message tool is unavailable.

## Best Possible Solution

When `message` is unavailable, doctor repair should persist `messages.groupChat.visibleReplies: "automatic"` or otherwise make the default/warning contract consistent, with regression coverage, changelog, and real doctor output.

## Reproduction Assessment

Yes, source-level. Current main writes `message_tool` during doctor repair, and the warning helper warns for both explicit and unset configured-channel defaults when the message tool is unavailable; I did not run the mutating doctor commands in this read-only review.

## Solution Assessment

No. Checking agent-aware policy is the right direction, but returning early leaves the value unset, so the existing warning path can still warn after `doctor --fix`; the narrower fix is to write `automatic` when tool-only delivery is unavailable.
