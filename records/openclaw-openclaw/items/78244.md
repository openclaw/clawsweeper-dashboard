## Summary

Keep open: the patch still leaves the visible-reply setting unset when the message tool is unavailable, but the current warning path treats an unset configured-channel group policy as the default `message_tool`, so the next `doctor` run can still warn after `doctor --fix`. The external PR also still lacks after-fix real behavior proof.

## What This Changes

The PR adds message-tool policy checks to the doctor legacy config normalizer and skips writing `messages.groupChat.visibleReplies: "message_tool"` when the resolved agent policy does not allow the message tool.

## Best Possible Solution

Share the message-tool availability decision with the warning path, persist `messages.groupChat.visibleReplies: "automatic"` when tool-only delivery is unavailable or update the default/warning contract consistently, keep `"message_tool"` when available, and add focused regression coverage plus redacted real doctor output.

## Reproduction Assessment

Yes, source-level. Current `main` writes `message_tool` during doctor compatibility repair, and the warning helper treats unset configured-channel group visible replies as default `message_tool` when the message tool is unavailable; I did not run mutating doctor commands in this read-only review.

## Solution Assessment

No. Reusing the policy check is the right direction, but returning the original config leaves the value unset and does not satisfy the no-follow-up-warning contract; the safer fix is to persist `automatic` or change repair and warning semantics together.
