## Summary

Keep open: current main does not include the proposed scanner rules, but this branch still has security-sensitive scanner accuracy defects and lacks after-fix real behavior proof.

## What This Changes

The branch adds four regex-based skill scanner rules for network listeners, prototype pollution, symlink calls, and credential-file reads, plus unit cases for those detections.

## Best Possible Solution

Revise the scanner rules so they detect real listener, symlink, credential-read, and prototype-mutation patterns with scoped context plus regression tests, or consolidate that work into a broader reviewed scanner-hardening PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `scanSource` checks line-rule context against the full source and source-rule pattern/context independently, while the proposed prototype regex only covers bracket notation.

## Solution Assessment

No. The core scanner is the right surface, but the proposed source-wide regexes and overbroad/incomplete prototype rule are not the narrowest maintainable security fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict prototype detection to mutations:** `src/security/skill-scanner.ts:188`
  - body: The new critical rule matches any `__proto__` or `constructor["prototype"]` access, including read-only checks. Because critical scanner findings can block installs, require an assignment or mutation shape before emitting this rule.
  - confidence: 0.89
- **[P2] Cover direct prototype-pollution writes:** `src/security/skill-scanner.ts:188`
  - body: The prototype-pollution pattern catches bracket notation but misses common direct writes such as `obj.__proto__.polluted = true` and `Object.setPrototypeOf(obj, payload)`. Broaden the mutation patterns or narrow the rule scope and tests.
  - confidence: 0.82
- **[P2] Tie credential paths to file-read arguments:** `src/security/skill-scanner.ts:233-234`
  - body: `SOURCE_RULES` checks the credential path and `readFile` context independently across the whole file, so a help string mentioning `.env` plus an unrelated `readFileSync("README.md")` can warn. Match the credential path inside the file-read expression.
  - confidence: 0.9
- **[P2] Constrain network-listener context to real APIs:** `src/security/skill-scanner.ts:181-182`
  - body: The network-listener context regex matches `http`, `https`, `net`, `http2`, or `tls` anywhere in the source, including URL strings or comments. Require a Node import/binding or a qualified call like `http.createServer`.
  - confidence: 0.86
- **[P3] Tie symlink detection to fs calls:** `src/security/skill-scanner.ts:194-195`
  - body: The symlink rule has the same source-wide context problem: any `fs` or `node:fs` token elsewhere enables warnings for local functions named `readlink` or `symlink`. Require a real fs import/binding or a qualified fs call before reporting it.
  - confidence: 0.78
