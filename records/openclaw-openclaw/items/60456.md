## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Restrict prototype detection to mutations:** `src/security/skill-scanner.ts:184`
  - body: The new `prototype-pollution` pattern flags any `__proto__` or `constructor["prototype"]` access as `critical`, including read-only guard code. Since critical scan findings block installs by default, benign defensive checks can now stop skill installs; match assignment/mutation shapes instead and add a negative test.
  - confidence: 0.88
- **[P2] Cover direct prototype-pollution writes:** `src/security/skill-scanner.ts:184`
  - body: The rule claims to catch prototype-pollution vectors, but `obj.__proto__.polluted = true` bypasses the added pattern entirely. Add dot-notation write coverage, and preferably `Object.setPrototypeOf` coverage if that is in scope, with regression tests.
  - confidence: 0.84
- **[P2] Match credential paths on readFile arguments:** `src/security/skill-scanner.ts:229-230`
  - body: `credential-access` currently matches a credential path anywhere in the source plus any unrelated `readFile*` usage, so a help string mentioning `~/.ssh/config` and a separate `readFileSync("README.md")` warning are enough to flag. Tie the credential path to the `readFile` argument or at least the same expression/line.
  - confidence: 0.86
- **[P2] Constrain listener context to Node server APIs:** `src/security/skill-scanner.ts:177-178`
  - body: The listener rule uses a source-wide `http|https|net|http2|tls` context guard, so a local `createServer()` helper can be warned merely because the file contains an unrelated URL or token. Match qualified/imported Node server APIs instead of broad token presence.
  - confidence: 0.82
- **[P3] Tie symlink detection to fs calls:** `src/security/skill-scanner.ts:190-191`
  - body: The symlink rule also uses source-wide `fs` context, so an unrelated `readlink()` or `symlink()` helper can warn when the file merely mentions `fs` or `node:fs` elsewhere. Narrow it to `fs.readlink*`/`fs.symlink*` or imported aliases that actually come from the fs module.
  - confidence: 0.74
