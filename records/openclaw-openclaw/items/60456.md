## Summary

Keep open. Current main still lacks these four proposed scanner rules, so this PR is not obsolete; however, the diff should not merge as-is because several new regex rules are source-wide or overbroad in ways that can create install-time false positives, and the prototype-pollution coverage still misses common write forms. The existing PR discussion usefully identifies the same scanner-quality concerns, and related open security PRs overlap but do not yet supersede this work.

## What This Changes

This PR adds four regex-based skill scanner rules for network listeners, prototype pollution, symlink calls, and credential-file access, plus positive and negative tests in `src/security/skill-scanner.test.ts`.

## Best Possible Solution

Tighten this as a focused scanner-hardening PR: bind network and symlink detections to real Node API imports or qualified calls, match credential paths inside the actual file-read argument or expression, make prototype-pollution detection mutation-oriented while covering common dot-notation and `Object.setPrototypeOf` forms, and add regression tests for each false-positive and false-negative shape before merge.

## Reproduction Assessment

Yes. The problematic behavior follows directly from the current scanner: `requiresContext` is evaluated against the whole source for line rules, and source rules independently match their pattern and context across the whole file, so small snippets with unrelated `https`/`readFile` context reproduce the proposed false positives without needing a full install run.

## Solution Assessment

No. The intended hardening belongs in the existing core scanner, but the current regex-only implementation is not the best merge shape because it does not tie detections closely enough to the dangerous API call or mutation being reported.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Restrict prototype detection to mutations:** `src/security/skill-scanner.ts:187`
  - body: This new rule reports any bracket access to `__proto__` or `constructor["prototype"]` as `critical`. Since critical scanner findings block gateway-backed skill installs by default, benign read-only inspection like `if (obj["__proto__"])` would now fail closed; require an assignment/mutation shape before emitting this critical finding.
  - confidence: 0.89
- **[P2] Cover direct prototype-pollution writes:** `src/security/skill-scanner.ts:187`
  - body: The new prototype-pollution rule only matches bracket notation, so direct writes such as `obj.__proto__.polluted = true` or `Object.setPrototypeOf(obj, payload)` still bypass the scanner while the rule claims to cover prototype-pollution vectors. Broaden the mutation patterns or narrow the rule name/scope and tests.
  - confidence: 0.82
- **[P2] Tie credential paths to file-read arguments:** `src/security/skill-scanner.ts:233-234`
  - body: `SOURCE_RULES` test `pattern` and `requiresContext` independently across the whole file, so this warns when a credential path appears in documentation text and an unrelated `readFileSync("README.md")` exists elsewhere. Match credential paths in the actual `readFile`/`readFileSync` argument or at least the same expression.
  - confidence: 0.9
- **[P2] Constrain network listener context to real APIs:** `src/security/skill-scanner.ts:180-181`
  - body: The `network-listener` context regex matches any `http`, `https`, `net`, `http2`, or `tls` token in the whole source, including URL strings and comments, so a local `createServer()` helper can be warned as a network listener. Require an import/require binding or a qualified call like `http.createServer`.
  - confidence: 0.86
- **[P3] Tie symlink detection to fs calls:** `src/security/skill-scanner.ts:193-194`
  - body: The symlink rule has the same source-wide context problem: any `fs` or `node:fs` token elsewhere in the file enables warnings for local functions named `readlink()` or `symlink()`. Require a real fs import/binding or qualified fs call before reporting this rule.
  - confidence: 0.78
