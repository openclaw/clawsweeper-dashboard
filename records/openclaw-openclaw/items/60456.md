  - body: The critical prototype-pollution rule matches any `__proto__` or `constructor["prototype"]` access, including read-only checks. Critical findings can block plugin or skill installs, so require an assignment or mutation shape before reporting.
  - confidence: 0.89
- **[P2] Cover direct prototype-pollution writes:** `src/security/skill-scanner.ts:184`
  - body: The pattern catches bracket notation but misses common direct writes such as `obj.__proto__.polluted = true` and `Object.setPrototypeOf(obj, payload)`, leaving obvious bypasses in the rule it adds.
  - confidence: 0.83
- **[P2] Match credential paths on readFile arguments:** `src/security/skill-scanner.ts:229-230`
  - body: This source rule checks credential paths and `readFile` context independently across the whole file, so a string mentioning `.env` plus an unrelated `readFileSync("README.md")` can warn. Tie the path to the file-read argument or same expression.
  - confidence: 0.9
- **[P2] Constrain listener context to Node server APIs:** `src/security/skill-scanner.ts:177-178`
  - body: The `network-listener` context regex matches `http`, `https`, `net`, `http2`, or `tls` anywhere in the source, including URL strings or comments. Require a real Node import/binding or a qualified call like `http.createServer`.
  - confidence: 0.86
- **[P3] Tie symlink detection to fs calls:** `src/security/skill-scanner.ts:190-191`
  - body: The symlink rule has the same source-wide context issue: any `fs` or `node:fs` token elsewhere enables warnings for local functions named `readlink` or `symlink`. Require a real fs binding or qualified fs call before reporting it.
  - confidence: 0.78
