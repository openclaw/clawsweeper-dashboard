- **[P1] Restrict prototype detection to mutations:** `src/security/skill-scanner.ts:184`
  - body: The new critical rule matches any `__proto__` or `constructor["prototype"]` access, including read-only defensive checks. Because critical scanner findings block installs by default, require an assignment or mutation shape before emitting this rule.
  - confidence: 0.9
- **[P2] Cover direct prototype-pollution writes:** `src/security/skill-scanner.ts:184`
  - body: The pattern catches bracket notation but misses common direct mutations such as `obj.__proto__.polluted = true` and `Object.setPrototypeOf(obj, payload)`. Broaden the mutation patterns or narrow the advertised rule scope and tests.
  - confidence: 0.84
- **[P2] Match credential paths on file-read calls:** `src/security/skill-scanner.ts:229-230`
  - body: The source rule checks credential paths and `readFile` context independently across the whole file, so a help string mentioning `.env` plus an unrelated `readFileSync("README.md")` can warn. Tie the credential path to the file-read argument or same expression.
  - confidence: 0.9
- **[P2] Constrain listener detection to Node server APIs:** `src/security/skill-scanner.ts:177-178`
  - body: The context guard matches `http`, `https`, `net`, `http2`, or `tls` anywhere in the source, including comments or URL strings. Require a real Node import/binding or a qualified server call such as `http.createServer` before warning.
  - confidence: 0.86
- **[P3] Tie symlink detection to fs calls:** `src/security/skill-scanner.ts:190-191`
  - body: The symlink rule has the same source-wide context issue: any `fs` or `node:fs` token elsewhere enables warnings for local functions named `readlink` or `symlink`. Require a real fs binding or qualified fs call before reporting it.
  - confidence: 0.78
