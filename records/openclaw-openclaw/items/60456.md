## Summary

Keep open. Current main does not implement these four scanner rules, but this PR is not merge-ready because the proposed source-wide regex/context checks introduce scanner false positives and miss common prototype-pollution writes; related open security PRs overlap but do not fully supersede this work.

## What This Changes

The PR adds four regex-based skill scanner rules for network listeners, prototype pollution, symlink calls, and credential-file access, with positive and negative scanner tests.

## Best Possible Solution

Revise the core scanner rules so they detect real listener, symlink, credential-read, and prototype-mutation patterns with scoped context plus regression tests for both false positives and bypasses.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: scanSource applies line-rule context to the full source and source-rule pattern/context independently across the source, so small snippets with unrelated http/readFile/fs tokens reproduce the false positives while dot-notation __proto__ writes reproduce the false negative.

## Solution Assessment

No. The core scanner is the right surface, but this PR's source-wide regex approach is not the narrowest maintainable security fix; the rules should be tied to real imports, call arguments, or mutation expressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Restrict prototype detection to mutations:** `src/security/skill-scanner.ts:188`
  - body: The new critical rule matches any __proto__ or constructor["prototype"] access, including read-only defensive checks. Since critical scanner findings block installs by default, require an assignment or other mutation shape before emitting this rule.
  - confidence: 0.89
- **[P2] Cover direct prototype-pollution writes:** `src/security/skill-scanner.ts:188`
  - body: The prototype-pollution pattern catches bracket notation but misses common direct writes such as obj.__proto__.polluted = true and Object.setPrototypeOf(obj, payload). Broaden the mutation patterns or narrow the rule scope and tests.
  - confidence: 0.82
- **[P2] Tie credential paths to file-read arguments:** `src/security/skill-scanner.ts:233-234`
  - body: SOURCE_RULES checks the credential path and readFile context independently across the whole file, so a help string mentioning .env plus an unrelated readFileSync("README.md") can warn. Match the credential path inside the readFile/readFileSync expression.
  - confidence: 0.9
- **[P2] Constrain network listener context to real APIs:** `src/security/skill-scanner.ts:181-182`
  - body: The network-listener context regex matches http, https, net, http2, or tls anywhere in the source, including URL strings or comments, so a local createServer helper can be flagged. Require a Node import/binding or a qualified call like http.createServer.
  - confidence: 0.86
- **[P3] Tie symlink detection to fs calls:** `src/security/skill-scanner.ts:194-195`
  - body: The symlink rule has the same source-wide context problem: any fs or node:fs token elsewhere enables warnings for local functions named readlink or symlink. Require a real fs import/binding or a qualified fs call before reporting this rule.
  - confidence: 0.78
