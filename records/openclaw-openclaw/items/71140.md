## Summary

Keep open. Current main still lacks shared reaction directive parsing and Telegram delivery/adapter execution for structured reactions; the media/GIF path is largely supported, and the related source PR at https://github.com/openclaw/openclaw/pull/71141 is closed unmerged.

## What This Changes

The issue requests Telegram outbound replies to consume reaction and media directives as native Telegram reactions/media instead of leaking raw text or rejecting reaction-only replies.

## Best Possible Solution

Land a focused shared parser/outbound-normalization plus Telegram plugin fix that parses reaction directives into structured channel data, permits reaction-only replies with a valid reply target, and invokes the existing Telegram reaction primitive while preserving current media behavior.

## Reproduction Assessment

Yes, at source level. Current main leaves `[[react_to_current:...]]` unrecognized by shared directive parsing, emits no structured Telegram reaction data, and rejects no-text/no-media Telegram replies before reaction handling can occur.

## Solution Assessment

Yes. The maintainable fix is shared directive parsing and outbound normalization plus Telegram delivery/adapter wiring; a Telegram-only text scrub would hide the leak without preserving reaction intent.
