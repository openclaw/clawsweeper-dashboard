## Summary

Keep open. The media/GIF part is already covered by current parsing and Telegram send paths, but the reaction directive path is still missing on current main: reaction tags are not parsed, no Telegram reaction channel data is produced, and Telegram delivery/adapter code does not execute reaction-only payloads. The linked implementation PR was closed unmerged, and duplicate searches did not find another open canonical fix.

## What This Changes

The issue requests Telegram outbound replies to consume reaction/media directives and apply native Telegram reactions or GIF/media sends instead of leaking or dropping directive text.

## Best Possible Solution

Land a focused replacement fix that parses reaction directives into structured outbound data, permits reaction-only Telegram replies with a valid target, and invokes the existing Telegram reaction primitive without changing media/GIF behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `[[react_to_current:...]]` is not recognized by the shared parser, no structured Telegram reaction data is emitted, and reaction-only replies still hit the text/media rejection path.

## Solution Assessment

Yes. The shared parser/outbound-normalization plus Telegram delivery/adapter wiring described in the issue is the narrow maintainable fix; a Telegram-only string special case would leave outbound directive handling inconsistent.
