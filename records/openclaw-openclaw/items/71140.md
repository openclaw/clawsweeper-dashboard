The issue requests Telegram outbound delivery to consume reaction and media directives as native Telegram reactions/GIFs instead of leaking or dropping directive text.

## Best Possible Solution

Land a focused reaction-directive fix that parses reaction intent into structured outbound data, permits reaction-only Telegram replies with a valid target, and invokes the existing Telegram reaction primitive while preserving current MEDIA/GIF behavior.

## Solution Assessment

Yes. The maintainable fix is shared parser/outbound-normalization plus Telegram delivery/adapter wiring; a Telegram-only text scrub would hide the leak without preserving reaction intent consistently.
