Keep open: current main still lacks the reaction-directive parsing and Telegram delivery/adapter wiring needed to turn `[[react_to_current:...]]` into a native Telegram reaction, while the media/GIF side is already handled.

Land a focused shared directive-normalization and Telegram delivery/adapter fix that strips reaction tags, carries structured reaction intent, permits reaction-only replies with a valid target, and uses the existing Telegram reaction primitive.
