## Summary

Keep this open. Current main still supports Telegram media/GIF delivery, but it does not implement the requested reaction-directive path; the issue is also paired with open draft PR #71141, so the issue should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

Make Telegram outbound reply normalization consume reaction directives and deliver them as native Telegram reactions while preserving MEDIA/GIF handling.

## Best Possible Solution

Review, land, or replace PR #71141 with a focused fix that extends the shared directive parser and outbound normalization, permits reaction-only Telegram replies when a reply target exists, and invokes the existing native Telegram reaction primitive without regressing current media/GIF behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: [[react_to_current:...]] is not parsed or stripped, no channelData.telegram.reaction is produced, and Telegram delivery rejects no-text/no-media replies before any reaction handling could run.

## Solution Assessment

Yes. The narrow maintainable solution is shared parser/normalization support plus Telegram delivery/adapter wiring; a Telegram-only string special case would leave outbound reply behavior inconsistent across delivery paths.
