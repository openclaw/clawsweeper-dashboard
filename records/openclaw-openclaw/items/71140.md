## Summary

Keep open. Current main still has Telegram media/GIF delivery, but the reaction directive path remains missing: `[[react...]]` is not parsed into structured outbound data, `channelData.telegram.reaction` is not carried or executed, and reaction-only Telegram replies can still be rejected before a native reaction is applied. Open draft PR #71141 is the active closing implementation candidate, so this issue should wait for that PR to be reviewed, landed, or replaced.

## What This Changes

The issue requests Telegram outbound reply normalization and delivery changes so reaction directives become native Telegram reactions and media directives are consumed as media rather than leaked as chat text.

## Best Possible Solution

Review, land, or replace PR #71141 with a focused implementation that extends shared directive parsing and outbound normalization, permits reaction-only Telegram replies when a valid reply target exists, and invokes the existing Telegram reaction primitive while preserving current media/GIF behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `[[react_to_current:...]]` is not recognized by the shared directive parser, no structured reaction data is emitted, and Telegram delivery still rejects reaction-only replies before channel data can be used.

## Solution Assessment

Yes. The shared parser/outbound-normalization plus Telegram delivery/adapter wiring described here is the narrow maintainable fix; a Telegram-only string special case would leave outbound behavior inconsistent across delivery lanes.
