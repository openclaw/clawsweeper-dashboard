## Summary

Keep open. Current main and latest release still support Telegram media/GIF delivery and native reaction primitives, but the reported reaction directive path remains missing: `[[react_to_current:...]]` is not parsed, not preserved as Telegram channel data, and not executed by Telegram reply or payload delivery; the linked fix PR is closed unmerged and no canonical replacement is open.

## What This Changes

The issue asks OpenClaw to turn Telegram outbound reaction/media directives into native reactions or GIF/media sends instead of leaking or dropping directive text.

## Best Possible Solution

Implement the reaction half through shared directive normalization plus Telegram delivery/adapter plumbing, while preserving the already-working `MEDIA:` extraction and GIF/photo send paths.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `[[react_to_current:...]]` is not parsed, no reaction metadata is preserved, and Telegram delivery rejects reaction-only replies. I did not run live Telegram because this review is read-only.

## Solution Assessment

Yes, the best fix is the narrow source path already outlined here: shared directive parsing plus Telegram reply/outbound adapter execution through the existing reaction primitive. A Telegram-only string special case would leave outbound projections and tests drifting.
