## Summary

Keep open. Current main still parses MEDIA lines but not `[[react...]]` directives, does not synthesize Telegram reaction channel data, and the Telegram delivery/adapter paths still treat reaction-only payloads as no text/media; the related source PR at https://github.com/openclaw/openclaw/pull/71141 is closed and unmerged.

## What This Changes

The issue requests Telegram outbound replies to consume reaction/media directives as native Telegram reactions and GIF/media sends instead of leaking or dropping directive text.

## Best Possible Solution

Land a focused fix that extends shared directive parsing/outbound normalization for reaction metadata, allows reaction-only Telegram replies with a valid reply target, and uses the existing Telegram reaction primitive while preserving current media behavior.

## Reproduction Assessment

Yes, at source level. Current main leaves `[[react_to_current:...]]` unrecognized by shared directive parsing, emits no structured Telegram reaction data, and still rejects no-text/no-media Telegram replies before any reaction branch can run.

## Solution Assessment

Yes. The maintainable fix is shared parser/outbound-normalization plus Telegram delivery/adapter wiring; a Telegram-only text scrub would hide the leak without preserving reaction intent consistently.
