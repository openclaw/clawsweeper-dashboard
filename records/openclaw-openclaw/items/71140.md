## Summary

Keep open: current main and v2026.6.8 still lack the Telegram reaction-directive path, so reaction-only replies can still be dropped and `[[react...]]` tags can still leak as visible text; the media side is already better covered by shared MEDIA parsing and Telegram media send paths.

## What This Changes

This issue requests fixing Telegram outbound reply normalization so reaction directives become native Telegram reactions and media directives do not leak as chat text.

## Best Possible Solution

Land a narrow shared outbound plus Telegram plugin fix that parses reaction directives into structured channel data, allows reaction-only replies with a valid target, invokes the existing Telegram reaction helper, and preserves existing MEDIA/GIF behavior.

## Reproduction Assessment

Yes, at source level. Current main lacks reaction directive parsing, does not create `channelData.telegram.reaction`, and rejects empty text/media Telegram replies before any reaction-only delivery could happen.

## Solution Assessment

Yes. The best fix is the shared parser plus Telegram adapter/delivery path described here, not a Telegram-only string special case or a new config option.
