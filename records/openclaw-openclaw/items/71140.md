## Summary

Keep open. Current main and the latest release still lack the reaction directive path: shared directive parsing does not recognize `[[react...]]`, outbound normalization does not produce Telegram reaction channel data, and Telegram delivery/adapter paths do not execute reaction-only payloads. The media/GIF side is partly supported already, with a separate open media-ordering PR, but the native reaction leak/drop remains a distinct bug.

## What This Changes

The issue requests Telegram outbound replies to consume reaction and media directives as native Telegram reactions/GIFs instead of leaking or dropping directive text.

## Best Possible Solution

Land a focused fix that parses reaction directives into structured outbound data, permits reaction-only Telegram replies with a valid target, and invokes the existing Telegram reaction primitive while preserving current MEDIA/GIF handling.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `[[react_to_current:...]]` is not recognized by the shared parser, no structured Telegram reaction data is emitted, and reaction-only replies still hit the text/media rejection path.

## Solution Assessment

Yes. The maintainable fix is the shared parser/outbound-normalization plus Telegram delivery/adapter wiring described in the issue; a Telegram-only string special case would leave directive handling inconsistent.
