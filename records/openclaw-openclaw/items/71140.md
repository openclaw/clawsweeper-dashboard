## Summary

Keep open. Current main still has the reaction-directive gap: shared directive parsing handles reply/audio/media paths but not `[[react...]]`, and Telegram delivery/adapter code still has no structured reaction branch. The closed, unmerged source PR is useful context, but no open canonical fix currently owns the remaining reaction work.

## What This Changes

The issue requests Telegram outbound replies to consume reaction directives and apply native Telegram reactions instead of leaking or dropping directive text.

## Best Possible Solution

Land a focused fix that parses reaction directives into structured outbound data, permits reaction-only Telegram replies with a valid reply target, and invokes the existing Telegram reaction primitive while preserving current MEDIA/GIF handling.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `[[react_to_current:...]]` is not recognized by the shared parser, no structured Telegram reaction data is emitted, and reaction-only replies still hit the text/media rejection path.

## Solution Assessment

Yes. The maintainable fix is shared parser/outbound-normalization plus Telegram delivery/adapter wiring; a Telegram-only string special case would leave directive handling inconsistent.
