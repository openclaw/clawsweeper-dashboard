## Summary

Keep open: current main and the latest release still parse MEDIA directives but do not parse or execute Telegram reaction directives, so reaction-only replies can still be dropped and reaction tags can still leak as text.

## What This Changes

The issue requests that Telegram outbound replies consume reaction and media directives and apply native Telegram reactions/GIF or photo sends instead of leaking or dropping control text.

## Best Possible Solution

Land a narrow shared outbound plus Telegram plugin fix that parses reaction directives into structured channel data, allows reaction-only replies with a valid target, invokes the existing Telegram reaction primitive, and preserves the existing MEDIA/GIF behavior.

## Reproduction Assessment

Yes, at source level. Current main lacks reaction directive parsing, does not create `channelData.telegram.reaction`, and rejects empty text/media replies before any reaction-only Telegram delivery can occur.

## Solution Assessment

Yes. The best fix is the shared parser plus Telegram adapter/delivery path described in the issue, not a Telegram-only string special case or a new config option.
