## Summary

Keep open. Current main still normalizes `MEDIA:` into outbound media and Telegram has native GIF/photo and reaction primitives, but the reaction directive path remains missing: `[[react...]]` is not parsed, not preserved in Telegram channel data, and not executed by Telegram reply or payload delivery.

## What This Changes

Telegram outbound replies should consume reaction/media directives and apply native Telegram reactions or GIF/photo sends instead of leaking or dropping control text.

## Best Possible Solution

Implement the narrow shared directive parser and Telegram delivery/adapter plumbing for reactions while preserving the already-working media/GIF normalization path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence source reproduction path: `[[react_to_current:...]]` is not parsed, reaction metadata is not preserved into `channelData.telegram`, and Telegram reply/payload delivery has no reaction execution path. I did not run a live Telegram repro or tests because this review is read-only.

## Solution Assessment

Yes, the best fix is the narrow source path already outlined in the issue: shared directive parsing plus Telegram reply/outbound adapter execution through the existing reaction primitive. A Telegram-only string special case would leave other outbound projections and tests drifting.
