## Summary

Keep open. Current main still has the gateway lifecycle early return that suppresses `emitChatFinal` when `peek()` found a chat link but `shift()` returns `undefined`, and the focused fix candidate is closed without merge.

## What This Changes

Ensure gateway terminal lifecycle handling still emits a final chat event when the chat-run registry entry drifts between `peek()` and `shift()`.

## Best Possible Solution

Patch the gateway finalizer to emit exactly one terminal chat event via the already-resolved fallback identifiers when `shift()` misses, with a focused regression test and changelog entry.

## Reproduction Assessment

Yes. A focused unit test can add a chat link, stream assistant text, force `registry.shift(evt.runId)` to return `undefined` after `peek()` succeeds, and assert current main skips the terminal `chat` broadcast.

## Solution Assessment

Yes. Fixing the producer-side lifecycle finalizer is the narrowest maintainable solution because it restores the missing terminal payload at the source instead of adding downstream UI recovery for a suppressed event.
