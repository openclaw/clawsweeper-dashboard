## Summary

Keep open. Checked-out main still has the terminal lifecycle early return that skips `emitChatFinal` when `peek()` succeeds but `shift()` misses, the registry contract permits that miss, and the focused fix PR is closed without merge.

## What This Changes

The issue asks the gateway chat lifecycle finalizer to emit a terminal chat event via resolved session/run fallback when the registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Patch the gateway finalizer to emit exactly one terminal chat event through the existing fallback identifiers when `shift()` misses, then lock it with a focused regression test and changelog entry.

## Reproduction Assessment

Yes. A focused unit test can register a chat link, stream assistant text, make `peek(evt.runId)` return that link while the subsequent `shift(evt.runId)` returns undefined, and assert current main skips the terminal `chat` broadcast.

## Solution Assessment

Yes. Fixing the producer-side finalizer is the narrowest maintainable solution because it restores the missing event at the source instead of adding downstream UI recovery for a suppressed terminal payload.
