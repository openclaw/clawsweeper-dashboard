## Summary

Keep open. Current main still has the terminal lifecycle branch that returns after a `peek()` hit followed by a `shift()` miss, and the only focused implementation candidate in the provided GitHub context, #74678, was closed without merge.

## What This Changes

Ensure gateway terminal lifecycle events still emit a final chat event when a chat-run registry entry disappears between `peek()` and `shift()` after streamed output.

## Best Possible Solution

Patch the gateway finalizer to fall back to the already resolved session/run identifiers on a `shift()` miss, add a focused regression test, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the handler branch: a focused unit can make `peek(evt.runId)` return a link and `shift(evt.runId)` return undefined, then assert current main skips the terminal chat payload. Live reproduction remains intermittent without a controlled registry-drift trigger.

## Solution Assessment

Yes. Fixing the server-side lifecycle finalizer is the narrowest maintainable solution because clients already retire active runs when they receive terminal chat payloads.
