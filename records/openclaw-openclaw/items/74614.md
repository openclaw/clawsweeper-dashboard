## Summary

Keep open. Current main still has the terminal lifecycle branch that returns after a truthy chat registry `peek()` but an undefined `shift()`, before emitting the terminal chat payload; the only linked implementation PR was closed unmerged.

## What This Changes

The issue asks the gateway chat lifecycle finalizer to emit the terminal `chat` event via resolved session/run fallbacks when the chat registry entry drifts between `peek()` and `shift()`.

## Best Possible Solution

Patch `finalizeLifecycleEvent` to fall back to the already resolved session/run identifiers on a shift miss, preserve terminal cleanup, add focused regression coverage, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the source-level failure branch: a focused test can add a chat link, force `registry.shift(evt.runId)` to return `undefined`, emit a terminal lifecycle event, and observe current main skipping the final chat payload. The intermittent live timing trigger was not reproduced in this read-only review.

## Solution Assessment

Yes. Fixing `finalizeLifecycleEvent` is the narrowest maintainable solution because existing clients already retire visible runs when they receive the terminal chat payload.
