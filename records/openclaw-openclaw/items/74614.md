Keep open. Current main still has the gateway terminal lifecycle early return that suppresses `emitChatFinal` when `peek()` succeeds but `shift()` misses, the registry contract permits that miss, and the linked focused fix PR is closed without merge.

## What This Changes

The issue requests that gateway chat lifecycle finalization still emit a terminal chat event when a registry `peek()` succeeds but the later `shift()` misses.

## Best Possible Solution

Patch the gateway finalizer to emit exactly one terminal chat event through the existing fallback identifiers when `shift()` misses, then add a focused regression test and changelog entry.

Yes. A focused unit test can register a chat link, make `peek(evt.runId)` return it while `shift(evt.runId)` returns undefined, and assert current main skips the terminal chat broadcast.

## Solution Assessment

Yes. Fixing the producer-side finalizer is the narrowest maintainable solution because the downstream UI already retires the active run correctly when it receives the terminal payload.

## Repair Work Prompt

Fix the gateway chat lifecycle finalizer for https://github.com/openclaw/openclaw/issues/74614. On current main, `finalizeLifecycleEvent` in `src/gateway/server-chat.ts` returns early when `chatRunState.registry.peek(evt.runId)` is truthy but `registry.shift(evt.runId)` returns undefined, suppressing the terminal `chat` final/error payload after streamed output. Change only the gateway finalization path so that a shift miss falls back to the already-resolved `sessionKey` and client/event run id and emits exactly one terminal chat event, preserving the existing abort, `skipChatErrorFinal`, context cleanup, and persistence behavior. Add a focused regression test in `src/gateway/server-chat.agent-events.test.ts` that forces `peek()` truthy and `shift()` undefined and asserts the final chat payload is still broadcast with the client run id. Add a user-facing `CHANGELOG.md` fix entry. Do not add UI-side recovery as the primary fix and do not change unrelated lifecycle semantics.
