## Summary

Keep open. Current main still contains the gateway terminal lifecycle branch that returns before `emitChatFinal` when a resolved chat link is followed by a missing registry shift, and the focused implementation PR is closed unmerged.

## What This Changes

The issue asks the gateway to emit a terminal chat event via resolved fallback identifiers when chat-run registry finalization misses after streamed output.

## Best Possible Solution

Patch the gateway finalizer to fall back to the already resolved session/run identifiers on a shift miss, continue terminal cleanup, add a focused regression test, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the handler branch: a focused unit can inject or spy a registry where `peek(evt.runId)` returns a link and `shift(evt.runId)` returns `undefined`, then assert current main skips the terminal chat broadcast. A live timing reproduction remains unclear with the stock synchronous registry.

## Solution Assessment

Yes. Fixing the producer-side gateway finalizer is the narrowest maintainable solution; a UI workaround would leave the suppressed terminal event broken for other websocket clients.
