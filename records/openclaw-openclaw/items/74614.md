## Summary

Keep open. Current main still has the exact terminal lifecycle branch where a truthy chat registry `peek()` followed by an undefined `shift()` clears context and returns before emitting the terminal chat event; the only linked fix PR was closed without merging.

## What This Changes

The issue asks the gateway chat lifecycle finalizer to emit a fallback terminal `chat` event when the chat run registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Patch `finalizeLifecycleEvent` to fall back to the already resolved session/run identifiers on a shift miss, preserve terminal cleanup, add focused regression coverage, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes for a source-level reproduction: add a chat registry entry, force `registry.shift(evt.runId)` to return `undefined` after `peek()` succeeds, emit a lifecycle `end`, and current main skips the final chat payload. I did not establish a live timing reproduction for the intermittent registry drift trigger.

## Solution Assessment

Yes. The narrow maintainable fix is in the gateway finalizer fallback path; broader TUI content-complete UX issues should stay separate from this terminal-event suppression bug.
