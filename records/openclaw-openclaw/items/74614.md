## Summary

Keep open. Current main still contains the reported gateway lifecycle path that clears the agent run context and returns when `registry.peek()` saw a chat link but the later `registry.shift()` misses, so the final `chat` event can still be suppressed. The latest release tag has the same branch, and the focused related fix PR is closed without merge, so no close reason applies.

## What This Changes

The issue requests that terminal gateway lifecycle handling emit exactly one final chat event using resolved session/run fallback data when the chat run registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Update `src/gateway/server-chat.ts` so terminal lifecycle handling falls back to the already resolved `sessionKey` and client run id when `registry.shift(evt.runId)` misses, still preserving abort and `skipChatErrorFinal` behavior, then add a focused regression test and changelog fix entry.

## Reproduction Assessment

Yes. A focused test can use the existing `server-chat.agent-events` harness, arrange a chat registry link for `peek()`, make the subsequent `shift()` return `undefined`, emit a terminal lifecycle event, and assert current main emits no final `chat` payload.

## Solution Assessment

Yes. Fixing the producer-side gateway finalization path is the narrowest maintainable solution because downstream UI clients cannot reliably recover from a final event that was never broadcast.

## Repair Work Prompt

Fix the gateway chat terminal finalization bug tracked by https://github.com/openclaw/openclaw/issues/74614. Current `src/gateway/server-chat.ts` can return before `emitChatFinal(...)` when `chatRunState.registry.peek(evt.runId)` found a chat link but `chatRunState.registry.shift(evt.runId)` returns `undefined`. Keep the fix scoped to gateway chat finalization: when a non-aborted terminal lifecycle event has a resolved `sessionKey`, emit exactly one final chat event using the shifted entry when present, otherwise fall back to the already resolved `sessionKey` and client run id. Add a regression test in `src/gateway/server-chat.agent-events.test.ts` that forces a truthy `peek()` and undefined `shift()` during a terminal lifecycle event and asserts a final chat payload is broadcast with the client run id.
