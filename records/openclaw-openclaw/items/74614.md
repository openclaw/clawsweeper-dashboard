## Summary

Keep open. Current main still has the reported gateway terminal-lifecycle early return, the registry contract allows a shift miss, and the linked focused fix PR is closed without merge.

## What This Changes

Terminal chat lifecycle handling should emit the final chat event via the resolved session/run fallback when a registry entry exists at peek time but is missing at shift time.

## Best Possible Solution

Update the gateway finalizer to emit exactly one terminal chat event using the already-resolved fallback session/run identifiers when `shift()` misses, then cover that branch with a focused agent-event regression test and a user-facing changelog fix entry.

## Reproduction Assessment

Yes. A focused unit test can register a chat run link, make `peek(evt.runId)` return it while the subsequent `shift(evt.runId)` returns undefined, and assert that current main skips the terminal `chat` broadcast.

## Solution Assessment

Yes. Fixing the producer-side gateway finalizer is the narrowest maintainable solution because existing Control UI/TUI clients already retire active state when they receive the final chat event.
