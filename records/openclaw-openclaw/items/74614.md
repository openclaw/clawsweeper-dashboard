## Summary

Keep open. Current main and the latest release still contain the reported gateway terminal-lifecycle early return, the registry API can legitimately return `undefined` from `shift()`, and Control UI/TUI state retirement depends on receiving the final chat event. The related focused fix PR is closed without merge, so there is no implemented or superseding fix to close against.

## What This Changes

Ensure gateway terminal lifecycle handling emits a final chat event when a chat run registry shift misses after streamed assistant output.

## Best Possible Solution

Update `src/gateway/server-chat.ts` so terminal lifecycle handling emits exactly one final chat event using the already-resolved session/run fallback when `registry.shift(evt.runId)` misses, then cover that race in `src/gateway/server-chat.agent-events.test.ts` and add a user-facing changelog fix entry.

## Reproduction Assessment

Yes. A focused unit test can register a chat link, deliver streamed assistant text, make `chatRunState.registry.peek(evt.runId)` return that link while the subsequent `shift(evt.runId)` returns `undefined`, then assert current main skips the terminal `chat` broadcast.

## Solution Assessment

Yes. The producer-side fallback is the narrowest maintainable fix because it repairs the dropped gateway event at the source instead of adding another downstream UI recovery path after the final event has already been suppressed.
