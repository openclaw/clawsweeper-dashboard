## Summary

Keep open. Current main still has the terminal lifecycle branch that returns after a truthy chat registry `peek()` but undefined `shift()`, so the reported final chat event suppression is still a valid narrow gateway bug; the only linked implementation PR was closed without merge.

## What This Changes

The issue asks for gateway chat lifecycle finalization to emit the terminal chat event using resolved session/run fallbacks when the chat registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Patch the gateway finalizer to fall back to the already resolved session/run identifiers on a shift miss, preserve cleanup, add a focused regression test, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the source-level handler branch: a test can add a chat link, force `registry.shift(evt.runId)` to return `undefined`, emit a terminal lifecycle event, and observe current main skipping the final chat payload. I did not establish a live timing reproduction for the intermittent registry drift trigger.

## Solution Assessment

Yes. Fixing `finalizeLifecycleEvent` is the narrowest maintainable solution because clients already retire visible runs when they receive the terminal chat payload.
