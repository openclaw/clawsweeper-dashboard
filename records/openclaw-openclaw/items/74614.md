## Summary

Keep open. Current `main` and latest release still have the reported gateway branch where a `peek()` hit followed by a `shift()` miss returns before emitting the terminal chat payload, and the exact linked fix PR was closed unmerged.

## What This Changes

The issue reports that gateway chat lifecycle finalization can skip the final chat event when `registry.peek()` finds a run but `registry.shift()` later returns `undefined`.

## Best Possible Solution

Patch `finalizeLifecycleEvent` to fall back to the already resolved session/run identifiers on a `shift()` miss, preserve terminal cleanup, and add focused regression coverage.

## Reproduction Assessment

Yes for a source-level reproduction: make `peek()` return a chat link, force `registry.shift(evt.runId)` to return `undefined`, then emit lifecycle `end` or `error`; current code returns before the final chat payload. I did not establish a live timing reproduction for the intermittent race trigger.

## Solution Assessment

Yes. The narrow maintainable fix is in the gateway finalizer fallback path; the broader embedded-timeout backstop tracked by https://github.com/openclaw/openclaw/issues/42011 is related but does not remove this gateway branch.
