## Summary

Keep open. Current main still contains the gateway terminal lifecycle branch that returns before emitting a terminal chat payload when `peek()` finds a chat link but `shift()` misses, and the focused PR for this exact edge case was closed without merge.

## What This Changes

The issue asks the gateway chat lifecycle finalizer to fall back to resolved session/run identifiers and still emit a terminal chat event when the registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Patch the gateway finalizer to fall back to the already resolved session/run identifiers on a shift miss, keep terminal cleanup intact, add a focused regression test, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the handler branch: a focused unit can make `peek(evt.runId)` return a link and `shift(evt.runId)` return undefined, then assert current main skips the terminal chat payload. Live timing reproduction remains unclear without a controlled registry drift trigger.

## Solution Assessment

Yes. Fixing the server-side lifecycle finalizer is the narrowest maintainable solution because clients already retire runs when they receive the terminal chat payload.
