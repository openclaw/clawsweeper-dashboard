## Summary

Keep open. Current main still contains the gateway terminal lifecycle early return that suppresses `emitChatFinal` when `peek()` succeeds but `shift()` misses, the registry contract permits that miss, and the linked focused fix PR is closed without merge.

## What This Changes

The issue requests that gateway chat terminal lifecycle handling still emit a final chat event when a visible chat-run registry entry disappears before `shift()` consumes it.

## Best Possible Solution

Patch the gateway finalizer to emit exactly one terminal chat event through the existing fallback identifiers when `shift()` misses, then add a focused regression test and changelog entry.

## Reproduction Assessment

Yes. A focused unit test can register a chat link, make `peek(evt.runId)` return it while `shift(evt.runId)` returns undefined, and assert current main skips the terminal chat broadcast.

## Solution Assessment

Yes. Fixing the producer-side finalizer is the narrowest maintainable solution because downstream clients already retire the active run correctly when they receive the terminal payload.
