## Summary

Keep open. Current main still has the reported gateway lifecycle branch that clears run context and returns when `peek()` found a chat link but `shift()` misses, before a terminal `chat` final event can be emitted; latest release `v2026.4.29` contains the same branch, and the focused related fix PR is closed without merge.

## What This Changes

The issue asks for gateway terminal lifecycle handling to emit the final chat event via resolved session/run fallback when the chat run registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Update the gateway lifecycle finalization path to emit exactly one final chat event using the already-resolved session/run fallback when `registry.shift(evt.runId)` misses, keep aborted/error-skip behavior intact, and cover the path with a focused regression test plus a user-facing changelog fix entry.

## Reproduction Assessment

Yes. A focused unit test can add a chat registry entry so `peek(evt.runId)` is truthy, stub the subsequent `registry.shift(evt.runId)` to return `undefined`, send a terminal lifecycle event, and observe that current main returns before broadcasting a final `chat` event.

## Solution Assessment

Yes. Fixing the producer-side gateway finalization branch is the narrowest maintainable solution because it restores the missing terminal event at the source instead of adding another downstream UI recovery path after suppression.
