## Summary

Keep open. Current main still has the terminal lifecycle early return that can suppress the final `chat` event after a `peek()`/`shift()` registry drift, the registry contract allows the miss, and the focused fix PR is closed unmerged.

## What This Changes

A terminal gateway chat lifecycle event should emit a final chat event via fallback identifiers when a chat run registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Patch the gateway finalizer to emit exactly one terminal chat event through the existing fallback identifiers when `shift()` misses, then add a focused regression test and changelog entry.

## Reproduction Assessment

Yes. A focused unit test can add a chat run link, stub `chatRunState.registry.shift()` to return `undefined` after `peek()` succeeds, send a terminal lifecycle event, and assert current main skips the final `chat` broadcast.

## Solution Assessment

Yes. Fixing the producer-side finalizer is the narrowest maintainable solution because downstream UI recovery would only mask a suppressed terminal payload.
