## Summary

Keep open. Current main still has the gateway terminal lifecycle early return that can suppress `emitChatFinal` after a truthy chat-link lookup, and the focused fix PR is closed without merge.

## What This Changes

The issue asks the gateway to emit a terminal chat event via resolved fallback identifiers when chat-run registry finalization misses after streamed output.

## Best Possible Solution

Patch the gateway finalizer to fall back to the already resolved session/run identifiers on a shift miss, continue terminal cleanup, add a focused regression test, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes. A focused unit test can inject or spy a registry where `peek(evt.runId)` returns a link and `shift(evt.runId)` returns `undefined`, then assert current main skips the terminal chat broadcast.

## Solution Assessment

Yes. Fixing the producer-side finalizer is the narrowest maintainable path because the Control UI already retires the visible run when it receives the terminal chat payload.
