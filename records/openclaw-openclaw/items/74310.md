## Summary

Keep this open. Current main still normalizes `kimi-coding` to `kimi`, and open PR #74315 is already linked as the candidate fix, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to stop rewriting explicit `kimi-coding/...` model refs to `kimi/...` so custom Kimi Coding provider namespaces resolve directly.

## Best Possible Solution

Review and land or replace #74315 with a fix that preserves exact configured `kimi-coding` provider ids while retaining legacy `kimi-code` compatibility and focused resolver/status tests.

## Reproduction Assessment

Yes. Configure `models.providers["kimi-coding"]` plus a selected `kimi-coding/k2p5` model; current `normalizeModelRef` normalizes the provider to `kimi` before the configured provider catalog can resolve the exact namespace.

## Solution Assessment

Unclear. Removing only the `kimi-coding` alias is a plausible narrow fix, but maintainers should confirm whether exact-provider-first lookup is safer for backwards compatibility across Kimi docs, plugin metadata, and persisted refs.
