## Summary

Keep this open. Current main still normalizes `kimi-coding` to `kimi`, the `/status` selected-model path resolves from an agent-only config that cannot see the exact `models.providers["kimi-coding"]` entry, and the issue is already paired with open closing PR #74315.

## What This Changes

Preserve provider-namespaced model refs such as `kimi-coding/k2p5` instead of collapsing the configured provider namespace to `kimi`.

## Best Possible Solution

Resolve the provider contract explicitly: exact configured provider ids such as `kimi-coding` should be preserved where users define them, while existing `kimi` and `kimi-code` configurations keep working with focused tests for `/status`, model selection, and persisted overrides.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure `agents.defaults.model.primary` as `kimi-coding/k2p5` with `models.providers["kimi-coding"]`, then `/status` resolves the selected model through `selectionConfig` and `normalizeProviderId`, which currently rewrites it to `kimi/k2p5`.

## Solution Assessment

Unclear. Removing `kimi-coding` from the alias chain is a plausible narrow fix, but an exact-configured-provider-first resolver may be safer unless maintainers confirm `kimi-coding` should be distinct across all affected selection paths.
