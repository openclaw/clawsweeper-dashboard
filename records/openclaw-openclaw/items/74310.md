## Summary

Keep this open. Current main still collapses `kimi-coding` to `kimi`, and the issue is paired with open closing PR #74315, so the right next step is to review or replace that PR rather than close the issue early.

## What This Changes

The issue asks OpenClaw to preserve `kimi-coding` as a distinct provider namespace when resolving provider-qualified model refs such as `kimi-coding/k2p5`.

## Best Possible Solution

Preserve exact configured provider ids for explicit `models.providers["kimi-coding"]` entries while retaining legacy `kimi-code` compatibility, with focused tests for resolver behavior and `/status` labels.

## Reproduction Assessment

Yes. Configure `agents.defaults.model.primary` as `kimi-coding/k2p5` with a `models.providers["kimi-coding"]` catalog entry; current source normalizes that provider to `kimi` before the selected-model path can resolve it exactly.

## Solution Assessment

Unclear. PR #74315 is a plausible minimal fix, but maintainers should decide whether a config-aware exact-provider-first resolver is safer than only removing `kimi-coding` from the alias chain.
