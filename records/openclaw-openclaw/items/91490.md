## Summary

Keep open: the PR targets a valid Windows Claude CLI supervisor bug, but the proposed `shell: true` path is not merge-safe because it routes supervisor argv through `cmd.exe` despite existing shell-injection hardening in the sibling process helper.

## What This Changes

The PR modifies `src/process/supervisor/adapters/child.ts` to resolve `claude` as a Windows `.cmd` shim and enable shell mode for resolved `.cmd`/`.bat` commands.

## Best Possible Solution

Keep this PR open, replace generic shell mode with a shared safe Windows batch invocation or direct entrypoint resolution for supervisor child spawns, add focused Windows unit coverage, and require redacted Windows proof after the patch.

## Reproduction Assessment

Yes, from source inspection. The default Claude CLI backend uses `command: "claude"`, one-shot and live CLI paths pass it to the supervisor child adapter, and current main does not resolve or wrap `claude.cmd` there; no live Windows repro was run in this read-only review.

## Solution Assessment

No. Adding Windows batch handling for Claude CLI is the right direction, but `shell: true` is not the best fix because current process code already uses an explicit cmd-wrapper and documents why argv-based shell execution should stay disabled.
