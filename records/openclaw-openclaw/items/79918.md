## Summary

Keep open: current main still routes realtime voice-call WebSocket upgrades with a raw prefix match, and the active linked fix PR should be reviewed or landed before this issue is closed.

## What This Changes

The issue asks for voice-call realtime WebSocket upgrades to reject sibling paths that merely share the configured `realtime.streamPath` prefix.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused fix that limits realtime upgrades to the exact configured stream path and slash-delimited child paths with regression coverage.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so a sibling path such as `/voice/stream/realtime-extra/token` is classified as realtime for the configured `/voice/stream/realtime` route. I did not run a live server because this was a read-only review.

## Solution Assessment

Yes: the linked PR's exact-or-slash-child helper is the narrow owner-boundary fix and keeps token format, TwiML generation, and provider behavior unchanged. The issue should stay open until that PR or an equivalent patch lands.
