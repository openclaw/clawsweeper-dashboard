## Summary

Keep open: current main and the latest release still route realtime voice-call WebSocket upgrades with a raw prefix match, and the open linked closing PR should be reviewed or landed before this issue is closed.

## What This Changes

The issue requests tightening voice-call realtime WebSocket upgrade matching so sibling paths like `/voice/stream/realtime-extra/<token>` are rejected.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused patch with regression coverage for exact, slash-child, sibling, and root stream-path cases.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce at classifier level: `/voice/stream/realtime-extra/token`.startsWith(`/voice/stream/realtime`) is true inside `isRealtimeWebSocketUpgrade`.

## Solution Assessment

Yes. Exact-or-slash-child matching is the narrow maintainable fix because valid generated realtime stream URLs append the token as a slash-delimited child of the configured route.
