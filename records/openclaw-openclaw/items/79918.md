## Summary

## What This Changes

The issue asks for voice-call realtime WebSocket upgrades to match only the configured stream path exactly or slash-delimited child paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused fix that preserves valid token child paths while rejecting sibling route prefixes.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for `/voice/stream/realtime`. I did not run a live server in this read-only review.

## Solution Assessment

Yes: exact-or-slash-child matching is the narrow maintainable fix because realtime stream tokens are generated as children of the configured route. The issue should stay open until the linked PR or equivalent patch lands.
