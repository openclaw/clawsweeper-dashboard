## Summary

Keep open: current main still uses raw prefix matching for realtime voice-call WebSocket upgrades, and the same-author closing PR is open rather than merged.

## What This Changes

The issue asks to restrict voice-call realtime WebSocket upgrades to the configured stream path or slash-delimited child routes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused fix with regression coverage for sibling and root stream paths.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for the configured `/voice/stream/realtime` route. I did not run a live server in this read-only review.

## Solution Assessment

Yes: exact-or-slash-child matching is the narrow maintainable fix because TwiML appends the stream token as a slash-delimited child. The issue should stay open until the linked PR or an equivalent patch lands.
