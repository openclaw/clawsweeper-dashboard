## Summary

Keep open: current main still routes realtime voice-call WebSocket upgrades with a raw prefix match, and https://github.com/openclaw/openclaw/pull/79919 is the active closing fix PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks for voice-call realtime WebSocket upgrades to reject sibling path prefixes and accept only the exact configured stream path or slash-delimited child paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919, or an equivalent focused patch, so realtime upgrades match only the configured stream path and slash-delimited token children with regression coverage.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for the configured `/voice/stream/realtime` route. I did not run a live server because this was a read-only review.

## Solution Assessment

Yes: exact-or-slash-child route matching is the narrow owner-boundary fix and preserves the intended token child route. The issue should stay open until the linked PR or an equivalent patch lands.
