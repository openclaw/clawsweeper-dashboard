## Summary

Keep open: current main and the latest release still use raw prefix matching for realtime voice-call WebSocket upgrades, and the linked closing PR remains open rather than merged.

## What This Changes

Restrict voice-call realtime WebSocket upgrade routing to the configured stream path exactly or slash-delimited child paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused patch with regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)` for realtime upgrade routing, so `/voice/stream/realtime-extra/token` matches the configured `/voice/stream/realtime` route. I did not run a live server in this read-only review.

## Solution Assessment

Yes: exact-or-slash-child matching is the narrow maintainable fix because generated realtime URLs append the token as a slash-delimited child. The issue should stay open until the linked PR or equivalent patch lands.
