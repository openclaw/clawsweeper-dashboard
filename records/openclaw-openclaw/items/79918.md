## Summary

Keep open: current main still routes realtime voice-call WebSocket upgrades with a raw prefix match, and the linked closing PR is open rather than merged.

## What This Changes

The issue requests tightening voice-call realtime WebSocket upgrade routing so only the configured stream path and slash-delimited children reach the realtime handler.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919, or an equivalent focused fix, so realtime upgrades require an exact configured stream path or a slash-delimited child path with regression coverage.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for a configured `/voice/stream/realtime` path. I did not run a live server because this was a read-only review.

## Solution Assessment

Yes: the linked PR's exact-or-slash-delimited route check is the narrow owner-boundary fix and preserves token format, TwiML generation, provider behavior, and root stream-path child routing.
