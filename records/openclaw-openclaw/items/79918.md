## Summary

Keep open: current main still routes realtime voice-call WebSocket upgrades with a raw prefix match, and the linked closing PR remains open rather than merged.

## What This Changes

Realtime voice-call WebSocket upgrades should reject sibling paths that merely share the configured `realtime.streamPath` prefix.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused patch with regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for the configured `/voice/stream/realtime` route. I did not run a live server because this review is read-only.

## Solution Assessment

Yes: exact-or-slash-child matching is the narrow owner-boundary fix and preserves the existing token child route. This issue should stay open until https://github.com/openclaw/openclaw/pull/79919 or an equivalent patch merges.
