## Summary

Keep open: current main still uses raw prefix matching for the voice-call realtime WebSocket upgrade route, and the linked open closing PR is the active fix path.

## What This Changes

The issue requests that voice-call realtime WebSocket upgrades match only the configured stream path exactly or slash-delimited child paths, not sibling prefixes.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused route-boundary fix with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for the configured `/voice/stream/realtime` route.

## Solution Assessment

Yes. Exact-or-slash-child matching is the narrow owner-boundary fix because generated realtime TwiML uses the configured path plus a slash-delimited token child.
