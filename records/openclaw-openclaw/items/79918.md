## Summary

Keep open: current main still routes realtime voice-call WebSocket upgrades with a raw prefix match, and an open linked fix PR already targets the bug. The issue should remain open until https://github.com/openclaw/openclaw/pull/79919 or an equivalent patch lands.

## What This Changes

Realtime voice-call WebSocket upgrades should reject sibling paths that merely share the configured `realtime.streamPath` prefix.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919, or an equivalent focused patch, so realtime upgrades match only the exact configured stream path or slash-delimited child paths with regression coverage.

## Reproduction Assessment

Yes. At source level, current main checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for the `/voice/stream/realtime` pattern; I did not run a live server in this read-only review.

## Solution Assessment

Yes. The exact-or-slash-child route check in the linked PR is the narrow maintainable fix because it preserves generated token child routes while rejecting sibling prefixes.
