## Summary

Keep open: current main still routes realtime voice-call WebSocket upgrades with a raw prefix match, and https://github.com/openclaw/openclaw/pull/79919 is an open closing PR rather than a merged fix.

## What This Changes

Tighten voice-call realtime WebSocket upgrade routing so sibling prefixes such as `/voice/stream/realtime-extra/<token>` no longer reach the realtime handler.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919, or an equivalent focused plugin fix, with regression coverage for valid child paths, sibling paths, and root stream-path child routing.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for the configured `/voice/stream/realtime` route. I did not run a live server in this read-only review.

## Solution Assessment

Yes: exact-or-slash-child path matching in the voice-call plugin is the narrow maintainable fix, as long as root `realtime.streamPath=/` child routes such as `/<token>` remain accepted.
