## Summary

Keep open: current main still uses raw prefix matching for realtime voice-call WebSocket upgrades, and the linked closing PR is open rather than merged.

## What This Changes

The issue requests that voice-call realtime WebSocket upgrades only match the configured stream path exactly or as slash-delimited child paths.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused fix with regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the route classification: with pattern `/voice/stream/realtime`, the current `pathname.startsWith(pattern)` check accepts `/voice/stream/realtime-extra/token`.

## Solution Assessment

Yes. Exact-or-slash-child matching is the narrow maintainable fix because generated realtime stream URLs append the token as a child path and do not need sibling-prefix acceptance.
