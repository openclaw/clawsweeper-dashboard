## Summary

Keep open: current main still accepts sibling realtime voice-call WebSocket paths through a raw prefix match, and an open closing PR already contains the focused fix path.

## What This Changes

Reject sibling-prefix realtime WebSocket upgrade paths and accept only the configured realtime stream path or slash-delimited child paths.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused patch with regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: current main checks `pathname.startsWith(pattern)`, so a sibling path like `/voice/stream/realtime-extra/token` is classified as realtime for `/voice/stream/realtime`. I did not run a live server because this was a read-only review.

## Solution Assessment

Yes: exact-or-slash-child matching is the narrow maintainable fix, and the linked PR also preserves root stream-path child routes. The issue should remain open until that PR or an equivalent patch lands.
