## Summary

Keep open: current main still routes voice-call realtime WebSocket upgrades with a raw prefix predicate, and the linked closing PR is still open rather than merged.

## What This Changes

The issue asks to restrict voice-call realtime WebSocket upgrades to the configured stream path or slash-delimited child paths instead of accepting sibling prefixes.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused fix with regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: current main calls `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for `/voice/stream/realtime`. I did not run a live server in this read-only review.

## Solution Assessment

Yes: exact-or-slash-delimited route matching is the narrow maintainable fix and preserves token child paths. The linked PR implements that direction, so the issue should stay open until it lands.
