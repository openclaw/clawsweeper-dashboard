## Summary

Keep open: current main still routes realtime voice-call WebSocket upgrades with a raw prefix match, and the open linked PR specifically closes this issue with a focused fix.

## What This Changes

The issue asks for voice-call realtime WebSocket upgrade routing to reject sibling paths that only share the configured `realtime.streamPath` prefix.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79919 or an equivalent focused patch that limits realtime upgrades to the exact configured stream path and slash-delimited child paths with regression coverage.

## Reproduction Assessment

Yes, at source level: current main still checks `pathname.startsWith(pattern)`, so `/voice/stream/realtime-extra/token` is classified as realtime for the configured `/voice/stream/realtime` route. I did not run a live server because this was a read-only review.

## Solution Assessment

Yes: exact-or-slash-child matching is the narrow maintainable fix because generated realtime TwiML appends the stream token as a slash-delimited child path. The issue should stay open until the linked PR or an equivalent patch lands.
