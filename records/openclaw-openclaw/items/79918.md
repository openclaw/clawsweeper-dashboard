## Summary

Keep open: current main and the latest release still route realtime voice-call WebSocket upgrades with a raw prefix match, and the linked open fix PR should be reviewed or closed before this issue is resolved.

## What This Changes

The issue asks for realtime voice WebSocket upgrades to accept only the configured stream path exactly or slash-delimited child paths, not sibling prefixes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79919, or an equivalent focused patch, with regression coverage for sibling prefixes and valid child paths.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: `pathname.startsWith(pattern)` classifies `/voice/stream/realtime-extra/token` as matching `/voice/stream/realtime`.

## Solution Assessment

Yes. Exact-or-slash-child matching is the narrow maintainable fix because it preserves generated token child URLs while rejecting sibling path prefixes.
