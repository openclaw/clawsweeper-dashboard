Full review comments:

- **[P1] Preserve recent tool results before pruning:** `src/agents/agent-hooks/compaction-safeguard.ts:1033-1035`
  - body: `preCompactionPrune()` runs before `splitPreservedRecentTurns()`, but that splitter later preserves tool results associated with recent assistant tool calls. A recent turn with several tool results can have older associated results outside the last `recentTurnsPreserve * 2` message slots, so this pass may replace content that is supposed to remain verbatim after compaction. Split first, or have pruning honor the preserved index/tool-call set.
  - confidence: 0.89
- **[P1] Wire or remove the multi-pass compaction path:** `src/agents/multi-pass-compaction.ts:119-121`
  - body: The new `multiPassCompact()` helper is tested in isolation but current manual, timeout, overflow, and CLI compaction entry points still call the existing single-pass `activeSession.compact()` or `contextEngine.compact()` paths. As merged, the central multi-pass feature would not run for users, so route the production owners through it or keep this PR to the narrower pruning/prompt changes.
  - confidence: 0.92
- **[P1] Keep the cut point inside the entry range:** `packages/agent-core/src/harness/compaction/compaction.ts:464`
  - body: This floor can move `cutIndex` to `startIndex + 1` even when `startIndex` is the only valid retained entry. `prepareCompaction()` then reads a missing first-kept entry and returns an invalid-session error instead of writing a valid boundary or no-oping, so clamp within the available range and add a single-entry/boundary-start regression test.
