Keep this open: current main still does not contain the `tool-defaults` runtime-pair QA lane, while the linked open parity PR and latest discussion scope the remaining work to Codex-aware mock/searchable dynamic-tool fidelity rather than a proven user-facing Codex runtime dropout.

## What This Changes

The issue asks the QA runtime tool-defaults harness to separate Codex-native workspace tools from OpenClaw dynamic-tool parity and to model searchable/deferred Codex dynamic tools honestly.

## Best Possible Solution

Land the parity harness through https://github.com/openclaw/openclaw/pull/80323 or a focused follow-up that keeps Codex-native workspace rows separate and adds honest mock coverage for searchable/deferred OpenClaw dynamic integration tools.

## Reproduction Assessment

No high-confidence current-main reproduction path is available because current main lacks the `--runtime-pair`/`--runtime-suite tool-defaults` lane and `qa/scenarios/runtime/tools/`. ## Solution Assessment

Yes, the corrected direction is the maintainable one: split native Codex workspace rows from OpenClaw dynamic integration rows and model searchable/deferred tools in the harness.
