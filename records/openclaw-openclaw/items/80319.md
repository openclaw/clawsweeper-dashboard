## What This Changes

## Best Possible Solution

Resolve this through the open runtime-parity PR or a focused follow-up by keeping Codex-native workspace rows out of OpenClaw dynamic parity and adding honest mock coverage for searchable/deferred OpenClaw dynamic integration tools.

No high-confidence current-main failing command is available because current main lacks `--runtime-pair`, `--runtime-suite tool-defaults`, and `qa/scenarios/runtime/tools/`. ## Solution Assessment

Yes in direction: the maintainable fix is to split Codex-native workspace rows from OpenClaw dynamic integration rows and model searchable/deferred tools in the harness. The exact mock-mode strategy should be reviewed through the active parity PR or a narrow maintainer-directed follow-up.
