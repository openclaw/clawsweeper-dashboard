Keep open: current main still rejects old desktop clients with structured protocol-mismatch details, but the macOS app only surfaces a generic update-the-app/gateway message and does not force an immediate Sparkle/appcast check or show an actionable desktop update prompt after a Gateway npm update.

## Best Possible Solution

Add a desktop-owned update-required recovery flow that detects protocol mismatch/version skew, triggers an immediate Sparkle check or clear download action, and keeps Gateway protocol negotiation strict.
