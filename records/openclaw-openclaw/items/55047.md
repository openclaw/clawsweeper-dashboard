## Summary

Keep open. Current main still has a credible Control UI blank-screen path matching the report: documented Control UI sections are supposed to show channels, sessions, cron, skills, nodes, exec approvals, and config editing, but malformed persisted cron rows can still flow into shared render and Cron UI consumers that dereference `job.payload` without a shape guard. The related focused PR #54552 was closed unmerged, so there is no implemented-on-main fix to close against.

## What This Changes

The issue reports that Control UI left-menu sections such as channels, instances, sessions, agents, and skills now render blank pages with only the page title instead of editable options.

## Best Possible Solution

Make Cron job rows safe at both boundaries: normalize, reject, filter, or annotate malformed persisted cron jobs before `cron.list` exposes them, and guard shared render plus Cron edit/list views so a bad row cannot blank unrelated Control UI sections. Add focused regression tests covering missing/invalid payloads through the service/gateway list path and the Control UI render/edit consumers.
