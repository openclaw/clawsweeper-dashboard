## Summary

Keep open: the PR targets a real source-visible gap for non-local embedding failures, but the proposed helper nulls explicitly configured remote providers and lets existing FTS/keyword fallback paths bypass the documented fail-closed contract.

## What This Changes

The PR adds a memory-core helper that nulls any embedding provider after persistent embedding failures and routes batch, structured-batch, and query failures through it.

## Best Possible Solution

Revise the patch so persistent non-local failures only degrade optional/default providers or activate a configured fallback; explicit remote providers without fallback should keep failing closed, with focused tests and redacted real runtime proof.

No high-confidence live reproduction was run. Source inspection shows the reported gap and the PR regression path: explicit remote providers are required, the patch nulls the provider, and existing catch paths can then return FTS/keyword fallback.

## Solution Assessment
