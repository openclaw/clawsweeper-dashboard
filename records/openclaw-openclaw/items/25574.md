## Summary

Keep open. This is a MEMBER-authored bug with an open closing PR, and current main plus v2026.5.22 still log every non-empty config-warning payload without content-sensitive dedupe.

## What This Changes

The issue requests deduping repeated config validation warnings so unchanged warnings are logged once and re-logged only when they change, disappear, or reappear.

## Best Possible Solution

Land a focused fix through https://github.com/openclaw/openclaw/pull/38295 or a split replacement that fingerprints config warning payloads per config path, resets when warnings clear or change, and covers load, write, reload, and repeated CLI invocation paths.

## Reproduction Assessment

Yes, source-level. A disabled plugin entry with retained config still produces a validation warning, and current load/write config IO logs the warning whenever it is present; I did not run a live reload loop in this read-only review.

## Solution Assessment

Yes for the core bug. Content-sensitive dedupe in the config IO/reload logging path is the narrow maintainable fix, with the open linked PR as the closest implementation path unless maintainers want a smaller split.
