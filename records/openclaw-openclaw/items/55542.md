## Summary

Keep open: the dependency swap is not safe to merge as-is because current main moved active image processing into media-understanding-core, while this branch changes the old root image path and still has format/backcompat regressions plus missing real behavior proof.

## What This Changes

This PR replaces root `sharp` usage with `@napi-rs/image` across image operations, related tests, install docs, package metadata, and the lockfile.

## Best Possible Solution

If maintainers approve the native dependency swap, migrate the media-understanding-core runtime and plugin-local dependency/ownership docs while preserving format, alpha, backend override compatibility, and installer docs.

## Reproduction Assessment

Yes for the review defects by source inspection: current main delegates active image ops to media-understanding-core while the PR changes root `src/media/image-ops.ts`; the branch also encodes EXIF normalization with `.jpeg(90)` and rewrites the backend override. I did not run tests in this read-only review.

## Solution Assessment

No, not as-is. The `@napi-rs/image@1.12.0` resize object overload is valid, but the maintainable path is a plugin-owned migration after dependency approval, with format and config compatibility preserved.

  - confidence: 0.87
