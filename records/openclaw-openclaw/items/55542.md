- **[P1] Move the image dependency swap into the media plugin:** `package.json:862`
  - body: Current main loads active image operations from media-understanding-core, and Sharp is owned there as a native parser dependency.   - body: @napi-rs/image documents `resize(width, height?, filterType?)`, but this passes a Sharp-style object.   - confidence: 0.94

  - body: The new non-sips path always returns `transformer.rotate().jpeg(90)`, so PNG/WebP inputs can become lossy JPEG and lose alpha even though the helper only promises orientation normalization. Preserve the source format or make the conversion an explicit API contract with caller updates.

  - body: Changing the non-sips override from `OPENCLAW_IMAGE_BACKEND=sharp` to only `napi-rs` means existing Bun/macOS installs using the old value can silently fall back to sips after upgrade. Keep `sharp` as a compatibility alias or add an explicit migration path.
  - confidence: 0.89
