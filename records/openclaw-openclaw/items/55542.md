## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve input format during EXIF normalization:** `src/media/image-ops.ts:299`
  - body: `normalizeExifOrientation` now encodes every non-sips input with `jpeg(90)`. The current sharp-backed runtime uses `rotate().toBuffer()`, so a PNG/WebP caller asking only for orientation normalization can receive lossy JPEG bytes and lose alpha; preserve the input format or make the contract explicit.
  - confidence: 0.95
- **[P2] Keep the legacy backend override working:** `src/media/image-ops.ts:28`
  - body: Changing the Bun/macOS non-sips override from `OPENCLAW_IMAGE_BACKEND=sharp` to only `napi-rs` makes existing deployments with the documented `sharp` override fall back to the sips path after upgrade. Keep `sharp` as a compatibility alias or add a deliberate migration path.
  - confidence: 0.9
- **[P2] Keep sharp build policy until the graph is sharp-free:** `package.json:964-965`
  - body: The PR removes sharp build/install guardrails while current main still uses sharp in media-understanding-core, still has sharp in the lockfile, and still documents/passes `SHARP_IGNORE_GLOBAL_LIBVIPS`. Dropping that policy before migrating the plugin dependency can break installs.
  - confidence: 0.88
