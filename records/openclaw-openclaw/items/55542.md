## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve source format during EXIF normalization:** `src/media/image-ops.ts:299`
  - body: This PR's non-sips path returns `transformer.rotate().jpeg(90)` for every input. Current sharp-backed behavior uses `rotate().toBuffer()`, so PNG/WebP callers can lose their original format and alpha when they only requested orientation normalization.
  - confidence: 0.9
- **[P2] Keep the existing sharp backend override as an alias:** `src/media/image-ops.ts:28`
  - body: Changing the Bun/macOS override from `OPENCLAW_IMAGE_BACKEND=sharp` to only `napi-rs` makes existing env config fall back to sips after upgrade. Keep `sharp` as a compatibility alias, or add an explicit migration path, while changing the backend.
  - confidence: 0.86
- **[P2] Migrate the current runtime before removing sharp policy:** `package.json:964`
  - body: The diff removes sharp from the approved native build list, but current main's image operations are now served by media-understanding-core, whose package still declares and imports sharp. A rebased version needs to migrate that runtime and package/build/docs policy together or leave sharp approved.
  - confidence: 0.84
