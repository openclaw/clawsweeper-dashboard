## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve source format during EXIF normalization:** `src/media/image-ops.ts:299`
  - body: `normalizeExifOrientation` now returns `transformer.rotate().jpeg(90)` for every non-sips input. The previous sharp-backed behavior returned `rotate().toBuffer()`, so PNG/WebP callers can unexpectedly receive lossy JPEG bytes and lose alpha when they only requested orientation normalization.
  - confidence: 0.95
- **[P2] Keep the legacy image backend override working:** `src/media/image-ops.ts:28`
  - body: Changing the Bun/macOS non-sips override from `OPENCLAW_IMAGE_BACKEND=sharp` to only `napi-rs` makes existing deployments with that env value silently select sips after upgrade. Keep `sharp` as a compatibility alias or provide an explicit migration path.
  - confidence: 0.9
- **[P2] Migrate the plugin-owned runtime before dropping sharp:** `package.json:964`
  - body: Current main loads image ops from `media-understanding-core`, and that plugin package/runtime still depends on sharp. Removing sharp build/install policy in this branch before migrating that runtime can break installs or leave the shipped graph inconsistent.
  - confidence: 0.88
