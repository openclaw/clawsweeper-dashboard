## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve source format during EXIF normalization:** `src/media/image-ops.ts:299`
  - body: `normalizeExifOrientation` now returns `transformer.rotate().jpeg(90)` for every non-sips input. The sharp-backed behavior this replaces returns `rotate().toBuffer()`, so PNG/WebP callers can unexpectedly receive lossy JPEG bytes and lose alpha when they only requested orientation normalization.
  - confidence: 0.9
- **[P2] Keep the legacy backend override working:** `src/media/image-ops.ts:28`
  - body: Changing the non-sips override from `OPENCLAW_IMAGE_BACKEND=sharp` to only `napi-rs` makes existing Bun/macOS deployments with the old env value silently select the sips path after upgrade. Keep `sharp` as a compatibility alias or provide an explicit migration path.
  - confidence: 0.86
- **[P2] Keep sharp build policy until the runtime moves:** `package.json:957-964`
  - body: The branch removes sharp from the package build allow-list, but current main now loads image processing from `media-understanding-core`, whose package still imports sharp. A rebased version needs to migrate that runtime and package first, or keep the sharp guardrails so installs do not break.
  - confidence: 0.84
