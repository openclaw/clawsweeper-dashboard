## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve source format during EXIF normalization:** `src/media/image-ops.ts:299`
  - body: `normalizeExifOrientation` now returns `transformer.rotate().jpeg(90)` for every non-sips input. The existing sharp-backed behavior returns `rotate().toBuffer()`, so PNG/WebP callers can unexpectedly receive lossy JPEG bytes and lose alpha when they only requested orientation normalization.
  - confidence: 0.95
- **[P2] Keep the existing sharp backend override:** `src/media/image-ops.ts:28`
  - body: Changing the Bun/macOS non-sips bypass from `OPENCLAW_IMAGE_BACKEND=sharp` to only `napi-rs` makes existing deployments with that env value silently select sips after upgrade. Keep `sharp` as a compatibility alias or provide an explicit migration path.
  - confidence: 0.9
- **[P2] Keep sharp build policy until no shipped graph uses sharp:** `package.json:963-964`
  - body: The PR removes sharp from the built-dependency allowlist and install docs, but current main and v2026.5.7 still have a sharp-backed media-understanding-core runtime. Dropping those guardrails before migrating that graph can break installs.
  - confidence: 0.88
