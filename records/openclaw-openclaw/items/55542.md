## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve source format during EXIF normalization:** `src/media/image-ops.ts:299`
  - body: normalizeExifOrientation now returns transformer.rotate().jpeg(90) for every non-sips input. The sharp-backed behavior this replaces returns rotate().toBuffer(), so PNG/WebP callers can unexpectedly receive lossy JPEG bytes and lose alpha when they only asked for orientation normalization.
  - confidence: 0.91
- **[P2] Keep the legacy backend override working:** `src/media/image-ops.ts:28`
  - body: Changing the non-sips override from OPENCLAW_IMAGE_BACKEND=sharp to only napi-rs makes existing Bun/macOS deployments with the old env value silently select the sips path after upgrade. Keep sharp as a compatibility alias or provide an explicit migration path.
  - confidence: 0.86
- **[P2] Move the runtime dependency swap into the media plugin:** `package.json:862`
  - body: Adding @napi-rs/image at the root package does not migrate current main's image runtime, which is loaded from media-understanding-core and still imports and declares sharp. A rebased dependency swap needs to update that plugin-owned runtime/dependency or keep sharp policy in place.
  - confidence: 0.84
