## Summary

Keep open: this PR targets a real `gateway:watch` bundled-hook gap, but it is not merge-ready on current main because runtime postbuild is now phase-wrapped and bundled-hook discovery still misses the copied `dist/bundled` tree.

## What This Changes

The PR adds runtime-postbuild code and tests to copy bundled hook `HOOK.md` manifests into `dist/bundled/<hook>/`, plus a changelog entry.

## Best Possible Solution

Land an equivalent current-main fix that copies bundled hook manifests as a named runtime-postbuild phase and makes bundled-hook discovery prefer the built `dist/bundled` directory before source fallback.

## Reproduction Assessment

Yes. The static path is high-confidence: `gatewayWatch` skips `copy-hook-metadata`, runtime postbuild currently lacks the hook metadata copy, hook loading requires `HOOK.md`, and the resolver does not select the built `dist/bundled` handler directory.

## Solution Assessment

No, not as submitted. The direction is narrow and maintainable, but the patch needs to be rebased into the current phase wrapper and must align bundled-hook discovery with the copied manifest location.
