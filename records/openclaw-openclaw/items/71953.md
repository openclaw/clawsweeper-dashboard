## What This Changes

The PR adds a runtime-postbuild helper and tests to copy bundled hook `HOOK.md` files into `dist/bundled/<hook>/`, plus a changelog entry.

## Best Possible Solution

Land a rebased fix that copies manifests as a named runtime-postbuild phase, updates bundled-hook discovery to prefer `dist/bundled`, and proves `gateway:watch` can load a bundled hook.

## Reproduction Assessment

Yes, by source inspection: `gatewayWatch` runs runtime-postbuild but skips `copy-hook-metadata`, hook handlers are emitted under `dist/bundled`, current discovery does not probe that directory, and hook loading requires `HOOK.md` beside a handler.

## Solution Assessment

No. The PR's direction is narrow, but the submitted patch must be rebased into the current runtime-postbuild phase structure and paired with resolver coverage for `dist/bundled` before it fully fixes the dev gateway path.
