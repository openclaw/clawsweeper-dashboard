## Summary

Keep open: the underlying gateway-watch bundled-hook bug is source-reproducible on current main, but the submitted patch is not ready because it is stale against the current runtime-postbuild phase structure, does not make discovery check the copied `dist/bundled` path, and still lacks artifact-backed real behavior proof.

## What This Changes

The PR adds a runtime-postbuild helper to copy bundled hook `HOOK.md` files, wires it into runtime postbuild, adds tests, and adds a changelog entry.

## Best Possible Solution

Land a rebased fix that adds a named runtime-postbuild hook metadata phase, aligns bundled-hook resolution with `dist/bundled`, and includes real `gateway:watch` proof that a bundled hook loads.

## Reproduction Assessment

Yes, by source inspection: `gatewayWatch` runs runtime-postbuild but skips `copy-hook-metadata`, handlers are emitted under `dist/bundled`, and hook loading requires `HOOK.md` beside a handler. I did not execute the writing watch flow in this read-only review.

## Solution Assessment

No. Copying manifests during runtime postbuild is the right direction, but the current patch needs to be rebased into the `runPhase(...)` flow and paired with resolver coverage for `dist/bundled`.
