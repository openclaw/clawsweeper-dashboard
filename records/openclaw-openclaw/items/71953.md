## What This Changes

The PR adds a runtime-postbuild helper, tests, and a changelog entry to copy bundled hook `HOOK.md` manifests into `dist/bundled/<hook>/`.

## Best Possible Solution

Land a rebased fix that copies bundled hook manifests as a named runtime-postbuild phase, updates bundled-hook discovery to prefer `dist/bundled`, and proves `gateway:watch` can load a bundled hook.

Yes, at source level: `gatewayWatch` skips `copy-hook-metadata`, tsdown emits handlers under `dist/bundled`, current discovery does not probe that directory, and hook loading requires `HOOK.md` beside a handler. I did not run the writing dev-gateway build in this read-only review.

## Solution Assessment

No. The PR's direction is narrow and useful, but the submitted patch needs to be rebased into the current runtime-postbuild phase structure and paired with resolver/test coverage for `dist/bundled`.
