## Summary

Keep open: the reported `gateway:watch` bundled-hook failure is valid by source inspection, but this PR is not mergeable as submitted because it lacks real behavior proof and still misses current-main runtime-postbuild/discovery alignment.

## What This Changes

The PR adds a runtime-postbuild helper and tests to copy bundled hook `HOOK.md` manifests into `dist/bundled/<hook>/`, plus a changelog entry.

## Best Possible Solution

Land a current-main fix that copies bundled hook manifests during runtime postbuild, updates bundled-hook discovery to prefer `dist/bundled`, and proves `gateway:watch` can load a bundled hook.

## Reproduction Assessment

Yes, by source inspection: `gatewayWatch` runs runtime-postbuild but skips `copy-hook-metadata`, tsdown emits hook handlers under `dist/bundled`, discovery does not probe that directory, and hook loading requires `HOOK.md` beside a handler. I did not run a writing build in this read-only review.

## Solution Assessment

No. The direction is narrow, but the submitted diff should be rebased as a named runtime-postbuild phase and paired with resolver coverage for `dist/bundled` before it can solve `gateway:watch`.
