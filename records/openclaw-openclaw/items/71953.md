## Summary

Keep this PR open. Current main still lacks bundled hook metadata copying in the runtime postbuild path used by `gateway:watch`, so the reported bug is real, but the proposed patch is not mergeable as-is because it targets the older direct-call postbuild structure and does not make bundled-hook discovery select the copied `dist/bundled` directory.

## What This Changes

This PR adds a runtime postbuild helper to copy bundled hook `HOOK.md` files into `dist/bundled`, invokes it during `runRuntimePostBuild`, adds runtime-postbuild tests, and adds a changelog entry.

## Best Possible Solution

Land an equivalent current-main fix that copies bundled hook manifests during `runRuntimePostBuild` through the existing phase wrapper, updates bundled-hook discovery to prefer the built directory containing both `HOOK.md` and `handler.js`, and covers the resolver plus postbuild behavior with focused tests and a current changelog entry.

## Reproduction Assessment

Yes. The high-confidence static path is: `gatewayWatch` skips the standalone metadata copier, runtime postbuild currently does not copy bundled hook manifests, hook discovery returns null without `HOOK.md`, and the resolver does not select `dist/bundled` where the built hook handlers live.

## Solution Assessment

No, not as this PR currently stands. The direction is narrow and maintainable, but the patch must be rebased into the current phase-wrapped postbuild flow and must reconcile bundled-hook discovery so the copied manifests are actually loaded.
