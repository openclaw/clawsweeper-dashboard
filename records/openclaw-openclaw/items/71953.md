## What This Changes

The PR adds runtime-postbuild copying of bundled hook `HOOK.md` files into `dist/bundled`, unit tests, and a changelog entry for `gateway:watch`.

## Best Possible Solution

Rebase the fix into the current named-phase runtime postbuild flow, make bundled-hook discovery prefer `dist/bundled` before source fallback, cover that path with tests, and post redacted `gateway:watch` proof.

## Reproduction Assessment

Yes, at source level: `gatewayWatch` skips `copy-hook-metadata`, current runtime postbuild has no bundled-hook metadata phase, built handlers emit under `dist/bundled`, and hook loading requires `HOOK.md` with the handler. I did not run the writing dev-gateway build in this read-only review.

## Solution Assessment

No. Copying manifests during runtime postbuild is the right direction, but the submitted patch needs a current-main rebase, named phase wiring, and resolver/test coverage for the `dist/bundled` directory it writes.
