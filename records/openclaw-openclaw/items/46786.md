## Summary

Keep open: current main still has the reported elevated exec routing chain, and changing it would alter a sandbox-to-gateway security contract that current docs and tests now encode.

## What This Changes

The issue asks for elevated exec to remain sandbox-default unless an exec call explicitly requests elevation, including changing implicit elevated defaults and `/elevated ask` semantics.

## Best Possible Solution

Maintainers should choose the elevated exec contract, then align default resolution, per-call/session semantics, prompt/status wording, docs, and regression coverage in one coordinated change.

## Reproduction Assessment

Yes, by source inspection: an allowed elevated session without an explicit directive resolves to `on`, omitted `params.elevated` inherits that level, and target resolution forces gateway. I did not run live Docker proof in this read-only review.

## Solution Assessment

Unclear. The reporter’s sandbox-default direction is plausibly safer, but it conflicts with current documented and tested session-elevated behavior, so the maintainable fix first needs maintainer security/product approval.
