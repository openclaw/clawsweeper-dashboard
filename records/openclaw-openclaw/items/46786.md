## Summary

Keep open. Current main still has the reported source-level routing chain, but the surrounding docs and tests now describe that behavior as the current elevated-exec contract, so changing it needs maintainer security/product approval rather than cleanup closure.

## What This Changes

Request that enabling `tools.elevated` preserve sandbox exec by default and make elevation opt-in per exec call instead of routing allowed elevated sessions' ordinary exec calls to the gateway.

## Best Possible Solution

Maintainers should decide the elevated exec contract; if sandbox-default opt-in elevation is desired, update default resolution, exec inheritance, prompt/status copy, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure an allowed sandboxed elevated session without `elevatedDefault`, call `exec` without `elevated`, and the default `on` level makes elevated routing resolve to `gateway`.

## Solution Assessment

Unclear. The requested default-off or ask-as-opt-in behavior is plausibly safer, but it changes the currently documented and tested elevated contract, so maintainer approval should come first.
