## Summary

Keep open: current main and the latest release still resolve an allowed elevated session with no explicit default to `on`, which makes default exec calls elevated and routes them to the gateway. The narrow default fix is still missing, and the broader `/elevated ask` semantics need maintainer product/security judgment.

## What This Changes

The issue reports that enabling `tools.elevated.enabled` changes default non-elevated exec routing from sandbox to gateway.

## Best Possible Solution

Change the implicit allowed elevated default to `off` with focused regression coverage, and handle any new `/elevated ask` meaning as a separate explicit maintainer decision.

## Reproduction Assessment

Yes by source inspection. Allowed elevated sessions still fall back to `on`, and that active elevated default makes default exec routing resolve to gateway instead of sandbox.

## Solution Assessment

No, the current implementation is not the best end state for safe defaults. The narrow fallback fix is maintainable, but redefining `/elevated ask` should be handled separately because current docs define it as an alias for `on`.
