## Summary

Current main still has the reported source-level routing chain: allowed elevated sessions fall back to active elevated mode, omitted exec calls inherit that state, and elevated auto target resolution routes to the gateway. Keep this open because changing it now requires a maintainer security/product decision: current docs and tests describe the host-exec default as intentional behavior.

## What This Changes

The issue asks for elevated exec to preserve sandbox routing for ordinary exec calls and make host/gateway execution opt-in per call or explicit session setting.

## Best Possible Solution

Maintainers should explicitly decide the elevated exec contract, then align default resolution, exec inheritance, prompt/status wording, docs, and regression coverage around either host-default elevated sessions or sandbox-default opt-in elevation.

Yes. Source inspection on current main gives a high-confidence path: enable elevated tools for an allowed sender, start a session without an explicit elevated directive, call exec without params.elevated, and the inherited default elevated state routes through gateway.
