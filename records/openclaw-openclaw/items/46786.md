## Summary

Keep open: current main and v2026.6.6 still default allowed elevated sessions with no explicit elevated default to `on`, and active elevated exec still routes to the gateway host. The earlier focused fix was closed unmerged, so no implemented or viable superseding path exists yet.

## What This Changes

The issue requests that enabling elevated exec not make ordinary non-elevated exec calls leave the sandbox by default.

## Best Possible Solution

Keep the issue open, make the implicit allowed elevated fallback safe by default with focused regression coverage, and handle any new `/elevated ask` semantics as a separate explicit maintainer product/security decision.

## Reproduction Assessment

Yes. Source inspection shows allowed elevated sessions still fall back to `on`, and active elevated exec routes to `gateway` unless the selected target is `node`.

## Solution Assessment

No. The current implementation is not the best safe-default behavior; the narrow fallback fix is maintainable, while changing `/elevated ask` semantics needs maintainer product/security judgment.
