## Summary

Keep open. This is a current architecture/design issue with concrete cost observations and useful community input, and current main still provides lower-level subagent isolation, fork, history, compaction, usage, depth, and concurrency controls rather than a decided tiered conversational coordination or budget-reservation design.

## What This Changes

The issue asks maintainers to decide how OpenClaw should expose or document lower-cost conversational multi-agent coordination without losing cross-agent emergent information.

## Best Possible Solution

Keep the thread open until maintainers choose the boundary: a generic coordination/cost-governance primitive, context-engine or plugin hooks for tiered delivery and budget reservation, or explicit docs saying these policies belong in scenario-specific orchestrators.

## Reproduction Assessment

No, not as a high-confidence repo-local reproduction. The issue gives plausible manual steps and comments add concrete observations, but there is no replayable runner, token trace, or deterministic fixture in current main for the reported conversational MAS scenario.

## Solution Assessment
