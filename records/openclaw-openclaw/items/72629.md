## Summary

Keep open. This is an active architecture/design issue with concrete cost reports and community input, and current main still exposes lower-level subagent, session-history, context-engine, usage, and compaction seams rather than a decided conversational coordination policy for tiered delivery, saliency routing, shared summaries, or pessimistic budget reservation.

## What This Changes

The issue asks for maintainer design input on reducing token-cost growth in conversational multi-agent coordination without losing emergent cross-agent information.

## Best Possible Solution

Keep the thread open until maintainers choose a concrete boundary: a generic coordination/cost-governance primitive, context-engine or plugin hooks for tiered delivery and budget reservation, or docs saying these policies belong in scenario-specific orchestrators.
