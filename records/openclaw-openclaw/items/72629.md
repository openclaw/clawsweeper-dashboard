## Summary

Keep open. This is a recent architecture/design issue with concrete cost observations and community patterns, and current main still provides lower-level subagent, session-history, context-engine, compaction, and usage primitives rather than a decided conversational coordination policy for tiered delivery, saliency routing, shared summaries, or pessimistic budget reservation.

## What This Changes

Requests design input on how OpenClaw should scale conversational multi-agent coordination without every agent rereading the full shared transcript each turn.

## Best Possible Solution

Keep the thread open until maintainers choose a concrete boundary for coordination cost governance: a generic primitive, context-engine/plugin hooks, or explicit docs that these policies belong in scenario harnesses.

No, not as a high-confidence repo-local reproduction. The issue and comments give concrete manual scenarios and cost observations, but current main does not include the runner, token traces, fixtures, or expected measurements needed for deterministic replay.

## Solution Assessment

Not applicable as a patch because no implementation is proposed. Current main has useful isolated/fork subagent modes, bounded history recall, context-engine assembly/compaction hooks, and usage reporting, but the best fix depends on maintainer architecture judgment.
