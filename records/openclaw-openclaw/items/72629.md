## Summary

Keep open. This is an active architecture/design thread with concrete cost reports and useful community input, and current main still exposes lower-level subagent, session-history, context-engine, compaction, usage, and cache controls rather than a decided conversational coordination scaling policy.

## What This Changes

The issue asks how OpenClaw should handle token-cost scaling for full-transcript conversational multi-agent coordination.

## Best Possible Solution

Keep this open until maintainers choose the boundary: a generic coordination/cost-governance primitive, context-engine or plugin hooks for tiered delivery and budget reservation, or explicit docs saying these policies belong in scenario-specific orchestrators.

No, not as a high-confidence repo-local reproduction. The issue gives plausible manual steps and comments add concrete observations, but the 5-agent runner, contexts, token trace, and fixture data needed for deterministic replay are not present in current main.

## Solution Assessment

Not applicable as a patch because no implementation is proposed. Existing main has useful isolation, fork, history, compaction, usage, cache, and context-engine seams, but the maintainable solution depends on a product/architecture decision about where coordination policy belongs.
