## Summary

Keep open. This is a recent architecture/design issue with concrete cost observations and community input, and current main still exposes lower-level subagent, session-history, context-engine, usage, and compaction primitives rather than a settled conversational coordination policy for tiered delivery, saliency routing, shared summaries, or pessimistic budget reservation.

## What This Changes

Requests design input on how OpenClaw should scale conversational multi-agent coordination without every agent rereading the full shared transcript each turn.

## Best Possible Solution

Keep the thread open until maintainers choose whether coordination cost governance belongs in a core primitive, context-engine/plugin extension point, or scenario harness/docs, then scope a narrow follow-up.

## Reproduction Assessment

No, not as a high-confidence repo-local reproduction. The issue and comments describe manual scenarios and cost observations, but current main does not include the runner, fixtures, token traces, or expected measurements needed for deterministic replay.

## Solution Assessment

Not applicable as a patch because no implementation is proposed. Current main has useful isolated/fork subagent modes, bounded session recall, context-engine assembly/compaction hooks, usage reporting, and a narrower prompt-authority fix, but the best fix depends on maintainer architecture judgment.
