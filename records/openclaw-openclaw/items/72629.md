## Summary

Keep open. This is a current maintainer design question with concrete cost observations and useful community patterns; current main has isolated/forked sub-agents, bounded session recall, context-engine compaction hooks, usage reporting, and fan-out guards, but no settled OpenClaw policy for tiered conversational context delivery, shared decision summaries, saliency routing, or pessimistic multi-agent budget reservation.

## What This Changes

The issue requests design input on how OpenClaw should handle token-cost scaling for conversational multi-agent coordination while preserving emergent cross-agent context.

## Best Possible Solution

Keep the design thread open until maintainers choose the boundary and success criteria, then pursue either a documented scenario-harness pattern or a generic, versioned coordination/cost-governance seam that can support tiered delivery, shared summaries, and budget reservation without baking one-off orchestration policy into core.
