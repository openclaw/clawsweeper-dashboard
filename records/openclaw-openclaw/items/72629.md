## Summary

Keep open. This is a current design/architecture issue with concrete cost reports and community input, and current main still exposes lower-level subagent, history, usage, compaction, prompt-cache, and context-engine seams rather than a decided conversational coordination policy such as tiered delivery, saliency routing, shared decision summaries, or pessimistic budget reservation.

## What This Changes

Request design input on how OpenClaw should handle token-cost scaling for conversational multi-agent coordination while preserving emergent cross-agent information sharing.

## Best Possible Solution

Keep the thread open until maintainers choose a concrete boundary: a generic coordination/cost-governance primitive, context-engine or plugin hooks for tiered delivery and budget reservation, or explicit docs saying these policies belong in scenario-specific orchestrators.

## Reproduction Assessment

No, not as a high-confidence repo-local reproduction. The issue gives plausible manual steps and comments add concrete observations, but current main does not include the 5-agent/3-round runner, token trace, or fixture needed to replay the reported conversational MAS shape deterministically.

## Solution Assessment

Not applicable as a patch because no implementation is proposed. Current main has useful isolation, fork, compaction, usage, prompt-cache, depth, and concurrency guardrails, but the best fix depends on a maintainer architecture decision about where coordination policy should live.
