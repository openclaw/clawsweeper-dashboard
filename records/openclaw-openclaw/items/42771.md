## Summary

Keep open. Current main partially solves brittle Docker Chromium autodetection through https://github.com/openclaw/openclaw/pull/80113, but it has not adopted the requested `/opt/playwright` default or PATH symlink, and this PR still has source-level Dockerfile blockers, insufficient final-head proof, conflicts, and an open linked issue that needs maintainer judgment.

## What This Changes

This PR changes the Docker image to store Playwright browsers under `/opt/playwright`, documents that path, and adds a `/usr/local/bin/chromium` symlink after optional browser install.

## Best Possible Solution

Refresh or replace the PR so the Docker browser path contract, node ownership, Linux x64/arm64 executable handling, docs, tests, and real Docker proof all agree before deciding whether `/opt/playwright` should become the default.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the PR head leaves browser-disabled `/opt/playwright` unwritable for `node`, targets the non-x64 Playwright layout for its symlink, and deletes a BuildKit cache mount. I did not run a live Docker build in this read-only review.

## Solution Assessment

No. Moving the Docker browser store may be acceptable, but this patch is not the best mergeable form until ownership, Linux x64 layout support, BuildKit cache behavior, branch freshness, and real Docker proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Make `/opt/playwright` writable during setup:** `Dockerfile:164-165`
  - body: When `OPENCLAW_INSTALL_BROWSER` is unset, this setup layer leaves `/opt/playwright` owned by `root:root` with mode `775`. The runtime `node` user is not in the root group, so node-user Playwright installs into the documented default path can fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point the Chromium symlink at the x64 layout:** `Dockerfile:176`
  - body: Playwright 1.60.0 uses `chrome-linux64/chrome` for Linux x64 and `chrome-linux/chrome` for arm64. This symlink only matches the arm64 layout, so normal x64 Docker builds can leave `/usr/local/bin/chromium` dangling.
  - confidence: 0.9
- **[P3] Preserve the mounted apt lists cache:** `Dockerfile:175`
  - body: This `RUN` mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` clears the named cache for later builds while not reducing the committed image layer size.
  - confidence: 0.84
