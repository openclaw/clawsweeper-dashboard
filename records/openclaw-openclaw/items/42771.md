## Summary

Keep open: current main partially overlaps the brittle Playwright path problem via merged Docker Chromium autodetection, but this PR still proposes a distinct Docker persistence/PATH contract and its current head has concrete Dockerfile regressions plus insufficient contributor proof.

## What This Changes

This PR changes the Docker image to use `PLAYWRIGHT_BROWSERS_PATH=/opt/playwright`, adds a `/usr/local/bin/chromium` symlink during optional browser preinstall, and updates Docker/browser docs to describe the new path.

## Best Possible Solution

Decide the Docker browser path and PATH-symlink contract explicitly, then refresh or replace this branch with ownership-safe Dockerfile changes, both Playwright Linux layouts, aligned docs/tests, and real Docker proof.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces the merge-blocking regressions: browser-disabled builds leave `/opt/playwright` unwritable for the final `node` user, and browser-enabled x64 builds can create a symlink to the wrong Playwright layout.

## Solution Assessment

No. Moving the Docker browser path may be valid, but this branch is not the narrowest maintainable solution until it preserves runtime ownership, both Playwright Linux layouts, and BuildKit cache behavior while maintainers decide the packaging contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assign `/opt/playwright` ownership during setup:** `Dockerfile:164-165`
  - body: This setup layer creates the documented default browser path as `root` and only applies mode `775`. In builds without `OPENCLAW_INSTALL_BROWSER`, the final `node` user is neither owner nor in the root group, so runtime Playwright installs into `/opt/playwright` fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point `chromium` at the Linux x64 layout too:** `Dockerfile:176`
  - body: The symlink only targets `chromium-*/chrome-linux/chrome`, but the pinned Playwright Linux x64 layout uses `chrome-linux64/chrome`. Browser-enabled x64 images can therefore install Chromium successfully while exposing a dangling `/usr/local/bin/chromium`.
  - confidence: 0.9
- **[P3] Stop deleting the mounted apt lists cache:** `Dockerfile:175`
  - body: This `RUN` mounts `/var/lib/apt` as a BuildKit cache, so `rm -rf /var/lib/apt/lists/*` deletes the shared cache contents used by later builds while providing no image-size reduction for cache-mounted data.
  - confidence: 0.84
