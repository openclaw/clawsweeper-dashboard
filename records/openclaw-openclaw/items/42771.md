## Summary

Keep open for maintainer path selection. Current main solved the brittle Playwright cache autodetection path in Docker, but it did not implement this PR's requested `/opt/playwright` Docker contract or PATH symlink, and the branch still has concrete Dockerfile regressions plus stale proof.

## What This Changes

The PR changes Docker Playwright browser storage from `/home/node/.cache/ms-playwright` to `/opt/playwright`, adds a `chromium` symlink, and updates Docker/browser docs to describe the new path.

## Best Possible Solution

Decide whether Docker should keep the shipped node-cache autodetection contract or intentionally migrate to `/opt/playwright`; if maintainers choose the migration, refresh or replace this branch with ownership-safe Dockerfile changes, docs, regression coverage, and current Docker proof.

## Reproduction Assessment

Yes for the PR regressions by source inspection: build this PR without `OPENCLAW_INSTALL_BROWSER` and run a node-user Playwright install into `/opt/playwright`, or build with the browser arg and exercise `/usr/local/bin/chromium` against a `chrome-linux64` cache layout. I did not run Docker in this read-only review.

## Solution Assessment

No. Moving the Docker browser cache may be valid, but this diff is not the narrowest maintainable implementation because it regresses runtime ownership, hard-codes one Playwright layout, mutates a BuildKit cache, and needs maintainer agreement on the path contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Make `/opt/playwright` writable for the runtime user:** `Dockerfile:164-165`
  - body: The setup layer creates `$PLAYWRIGHT_BROWSERS_PATH` as `root:root` and only applies mode `775`. In the default image where `OPENCLAW_INSTALL_BROWSER` is unset, the later `node` user cannot install browsers into the newly documented `/opt/playwright` path, so runtime installs fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point `chromium` at both Playwright Linux layouts:** `Dockerfile:176`
  - body: The symlink hard-codes `chromium-*/chrome-linux/chrome`, but the Playwright-managed Linux Chromium layout used by current images can be `chrome-linux64/chrome`. A browser-enabled image can therefore expose a dangling `/usr/local/bin/chromium` command even after Chromium was installed.
  - confidence: 0.9
- **[P3] Stop deleting the mounted apt lists cache:** `Dockerfile:175`
  - body: This `RUN` mounts `/var/lib/apt` as a BuildKit cache, so `rm -rf /var/lib/apt/lists/*` deletes the shared cache contents used by later builds while providing no image-size reduction for cache-mounted data.
  - confidence: 0.84
