## Summary

Keep open: the Docker direction is useful, but the PR is still blocked by concrete Dockerfile correctness issues and is stale against current browser-control docs; current main has not implemented the `/opt/playwright` default or Chromium PATH symlink.

## What This Changes

The PR changes Docker Playwright browser storage to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser persistence docs.

## Best Possible Solution

Land a corrected Docker/browser update that uses `/opt/playwright` consistently, keeps it writable for the runtime `node` user in both build modes, preserves BuildKit apt caches, updates the current docs surface, and adds focused Dockerfile assertions.

## Reproduction Assessment

Yes. From the PR diff, the default build creates `/opt/playwright` as root before the image later runs as `node`, and the browser-enabled build deletes a cache-mounted apt metadata directory; I did not run Docker builds in this read-only review.

## Solution Assessment

No. Moving Docker browser storage to `/opt/playwright` is a maintainable direction, but this patch needs ownership, cache-preservation, current-docs, and test-coverage fixes before it is the narrowest safe solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assign /opt/playwright to node during setup:** `Dockerfile:165-166`
  - body: The default build path creates `$PLAYWRIGHT_BROWSERS_PATH` as root with mode `775`, and no later `chown` runs when `OPENCLAW_INSTALL_BROWSER` is unset. Since the image later runs as `node`, documented runtime installs into the new default path can fail with `EACCES`.
  - confidence: 0.93
- **[P2] Leave the BuildKit apt lists cache intact:** `Dockerfile:176`
  - body: This step mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` clears cached apt metadata for later builds while not reducing the committed image layer. Remove the cleanup from this cache-mounted step.
  - confidence: 0.88
- **[P3] Update the current browser-control docs too:** `docs/install/docker.md:384-386`
  - body: The PR updates Docker persistence docs for `/opt/playwright`, but current main also documents Docker browser installs in `docs/tools/browser-control.md` with the old hidden-cache path. Leaving that page unchanged would give users two conflicting persistence paths after merge.
  - confidence: 0.78
