## Summary

Keep open. Current main only partially overlaps the PR by detecting Playwright-managed Chromium from the existing Docker cache; it does not move the Docker browser store to `/opt/playwright` or add a PATH symlink, and this PR still has Dockerfile correctness blockers plus insufficient real Docker proof.

## What This Changes

This PR changes the Docker image to use `PLAYWRIGHT_BROWSERS_PATH=/opt/playwright`, adds a `/usr/local/bin/chromium` symlink during optional browser preinstall, and updates Docker/browser docs to mount `/opt/playwright`.

## Best Possible Solution

Decide the Docker browser path and PATH-symlink contract, then refresh or replace this branch with ownership-safe Dockerfile changes, both Playwright Linux layouts, aligned docs/tests, and real Docker proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the browser-disabled build leaves `/opt/playwright` unwritable for the final `node` user, and the browser-enabled symlink only targets `chrome-linux/chrome` while current main covers `chrome-linux64` too.

## Solution Assessment

No. Moving the Docker browser path may be valid, but this patch is not the narrowest maintainable solution until ownership, Playwright layout support, BuildKit cache behavior, and the broader packaging contract are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assign `/opt/playwright` ownership during setup:** `Dockerfile:164-165`
  - body: This setup layer creates the documented default browser path as `root` and only applies mode `775`. In builds without `OPENCLAW_INSTALL_BROWSER`, the final `node` user is neither owner nor in the root group, so runtime Playwright installs into `/opt/playwright` fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point `chromium` at the Linux x64 layout too:** `Dockerfile:176`
  - body: The symlink only targets `chromium-*/chrome-linux/chrome`, but the current detector and tests cover the Linux x64 Playwright layout `chrome-linux64/chrome`. Browser-enabled x64 images can install Chromium successfully while exposing a dangling `/usr/local/bin/chromium`.
  - confidence: 0.9
- **[P3] Stop deleting the mounted apt lists cache:** `Dockerfile:175`
  - body: This `RUN` mounts `/var/lib/apt` as a BuildKit cache, so `rm -rf /var/lib/apt/lists/*` deletes the shared cache contents used by later builds while providing no image-size reduction for cache-mounted data.
  - confidence: 0.84
