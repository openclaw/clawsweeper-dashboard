## Summary

Keep open. Current main and the latest release partially address brittle Docker Chromium discovery, but they still keep the Docker browser cache under `/home/node/.cache/ms-playwright` and do not add a stable `chromium` command; this PR is not obsolete, but it is not merge-ready because it changes a Docker default path with unresolved compatibility, ownership, symlink-layout, proof, and conflict blockers.

## What This Changes

This PR moves Docker’s Playwright browser path to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser docs for the new persistence path.

## Best Possible Solution

Refresh or replace this PR with a maintainer-approved Docker browser path contract, compatibility or migration guidance, layout-aware symlink handling, updated docs/tests, and fresh Docker proof for browser-enabled, browser-disabled, and upgrade scenarios.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve existing browser-cache mounts:** `Dockerfile:162`
  - body: Changing Docker’s exported `PLAYWRIGHT_BROWSERS_PATH` from `/home/node/.cache/ms-playwright` to `/opt/playwright` drops the path existing users are already told to persist. Existing cached browsers can disappear on upgrade unless the PR keeps a compatible fallback or includes tested migration guidance.
  - confidence: 0.87
- **[P1] Make `/opt/playwright` writable for runtime installs:** `Dockerfile:164-165`
  - body: When `OPENCLAW_INSTALL_BROWSER` is unset, this directory is created by root with mode 775 and no later `chown` runs before `USER node`. The documented runtime install command can then fail with `EACCES` when Playwright writes into `/opt/playwright` as the `node` user.
  - confidence: 0.92
- **[P1] Point the symlink at the x64 Playwright executable:** `Dockerfile:176`
  - body: The repo pins Playwright 1.60.0, whose Linux x64 Chromium executable lives under `chrome-linux64/chrome`; this symlink only targets `chrome-linux/chrome`. On the normal x64 Docker image, `chromium` can become a dangling command after a successful browser install.
  - confidence: 0.91
- **[P3] Stop clearing the BuildKit apt-lists cache:** `Dockerfile:174-175`
  - body: `/var/lib/apt` is mounted as a BuildKit cache in this `RUN`, so deleting `/var/lib/apt/lists/*` wipes the reusable apt-lists cache while providing no image-size benefit from that cache mount. Remove the cleanup from the cache-mounted block.
  - confidence: 0.84
