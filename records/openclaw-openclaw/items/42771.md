## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve existing browser-cache mounts:** `Dockerfile:162`
  - body: Changing Docker's exported `PLAYWRIGHT_BROWSERS_PATH` from `/home/node/.cache/ms-playwright` to `/opt/playwright` drops the path existing users are already told to persist via `/home/node`. Existing cached browsers disappear on upgrade unless the PR keeps a compatible fallback or includes tested migration guidance.
  - confidence: 0.86
- **[P1] Make `/opt/playwright` writable in browser-disabled images:** `Dockerfile:164-165`
  - body: When `OPENCLAW_INSTALL_BROWSER` is unset, this directory is created by root with mode 775 and no later `chown` runs before `USER node`. The documented runtime install command will then fail with `EACCES` when Playwright tries to write into `/opt/playwright` as the `node` user.
  - confidence: 0.92
- **[P1] Link Chromium using the Playwright 1.60 Linux x64 path:** `Dockerfile:176`
  - body: The repo pins `playwright-core` 1.60.0, whose Linux x64 Chromium executable lives under `chrome-linux64/chrome`; this symlink only targets `chrome-linux/chrome`. On the normal x64 Docker image, `chromium` can become a dangling command even after a successful browser install.
  - confidence: 0.9
- **[P3] Stop clearing the BuildKit apt-lists cache:** `Dockerfile:174-175`
  - body: `/var/lib/apt` is mounted as a BuildKit cache in this `RUN`, so deleting `/var/lib/apt/lists/*` wipes the reusable apt-lists cache while providing no image-size benefit from that cache mount. Remove the cleanup from the cache-mounted block.
  - confidence: 0.84

## Risks / Open Questions

- Changing the Docker default from `/home/node/.cache/ms-playwright` to `/opt/playwright` can strand existing persisted home-volume or cache-mount setups unless maintainers intentionally accept and document the migration.
