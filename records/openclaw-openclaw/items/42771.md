## What This Changes

This PR changes Docker Playwright browser storage to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser docs for the new persistence path.

## Best Possible Solution

Refresh or replace the PR so the Docker browser path contract, node ownership, Linux x64/arm64 executable handling, docs, tests, and real Docker proof all agree before deciding whether `/opt/playwright` should become the default.

## Reproduction Assessment

Yes for the PR review findings via source inspection. The PR diff leaves `/opt/playwright` unwritable for the default `node` user in browser-disabled builds, symlinks only the non-x64 Playwright layout, and deletes a BuildKit cache mount; I did not run a live Docker build in this read-only review.

## Solution Assessment

No. Moving the path may be acceptable, but this patch is not the narrowest maintainable solution until ownership, Linux x64 layout support, BuildKit cache behavior, docs/tests, and real Docker proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Make /opt/playwright writable by node before runtime installs:** `Dockerfile:164-165`
  - body: This setup layer creates `/opt/playwright` as `root:root` and only grants group write. In browser-disabled images there is no later `chown` before the image runs as `node`, so runtime Playwright installs into the documented default path can fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point chromium symlink at the Linux x64 layout:** `Dockerfile:176`
  - body: Playwright 1.60 stores Linux x64 Chromium under `chrome-linux64/chrome`, but this symlink only targets `chrome-linux/chrome`. Normal x64 Docker builds can install Chromium successfully while leaving `/usr/local/bin/chromium` dangling.
  - confidence: 0.9
- **[P3] Preserve the mounted apt lists cache:** `Dockerfile:175`
  - body: This `RUN` mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` clears the shared cache used by later builds while providing no image-size benefit for cache-mounted data.
  - confidence: 0.84
