## Summary

Keep open: current main has not implemented the `/opt/playwright` Docker browser path, and the PR is directionally useful for the linked Docker browser report but is not ready to merge because the proposed Dockerfile can leave runtime installs unwritable and create a broken Chromium shim on linux-x64.

## What This Changes

The PR changes Docker Playwright browser persistence from `/home/node/.cache/ms-playwright` to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser docs.

## Best Possible Solution

Refresh this PR or a narrow replacement so Docker creates `/opt/playwright` as node-owned, preserves ownership after optional preinstall, derives or glob-matches the current Playwright Chromium executable path, avoids deleting cache-mounted apt lists, and updates current docs plus Dockerfile tests and changelog.

## Reproduction Assessment

Yes, source-level. The PR's default build path creates `/opt/playwright` as root-owned before `USER node`, and current Playwright writes install metadata under that registry path; the PR's Chromium symlink also targets `chrome-linux/chrome` while current linux-x64 Playwright uses `chrome-linux64/chrome`.

## Solution Assessment

No, not as-is. Moving the browser path can be a maintainable Docker cleanup, but the patch needs ownership fixes, a version/platform-safe Chromium shim, cache-mount-safe apt handling, and current docs/tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Create `/opt/playwright` as node-owned:** `Dockerfile:164-165`
  - body: This setup creates the new default browser path as root-owned and only changes its mode. In the default build where `OPENCLAW_INSTALL_BROWSER` is unset, there is no later ownership fix before the image switches to `USER node`, so runtime `playwright-core/cli.js install chromium` cannot write Playwright's registry metadata under `/opt/playwright`. Create it with `node:node` ownership here.
  - confidence: 0.92
- **[P1] Use the current Playwright Chromium executable path:** `Dockerfile:176`
  - body: The symlink points at `chromium-*/chrome-linux/chrome`, but current `playwright-core` resolves Chromium to `chrome-linux64/chrome` on linux-x64. Since `ln -s` succeeds even when the source is missing, standard amd64 Docker builds can publish a dangling `/usr/local/bin/chromium` shim. Resolve the path through Playwright or match the current platform-specific layout.
  - confidence: 0.9
- **[P2] Stop clearing the apt lists cache mount:** `Dockerfile:174-175`
  - body: This step mounts `/var/lib/apt` as a BuildKit cache, so removing `/var/lib/apt/lists/*` clears the shared cache and does not reduce the committed image layer. Drop this cleanup from the cache-mounted install block so later builds can reuse the apt lists cache.
  - confidence: 0.84
- **[P3] Update the current browser persistence docs:** `docs/tools/browser.md:411-413`
  - body: Current main still has Docker Playwright persistence guidance in `docs/tools/browser-control.md` and the FAQ. This PR updates the older `docs/tools/browser.md` copy, so after rebasing users would still see the old cache-path guidance unless the current public docs are updated too.
  - confidence: 0.79
