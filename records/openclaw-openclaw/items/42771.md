## Summary

Keep open. Current main still uses the hidden Playwright cache path and current docs still tell Docker users to persist that path, so the PR's core goal is not implemented; however the PR branch is not merge-ready because the default `/opt/playwright` directory is not owned by `node` when browser preinstall is disabled, and the browser install step deletes a BuildKit apt metadata cache.

## What This Changes

The PR sets Docker `PLAYWRIGHT_BROWSERS_PATH` to `/opt/playwright`, pre-creates that directory, chowns preinstalled browser files, adds a `/usr/local/bin/chromium` symlink, and updates Docker browser persistence docs.

## Best Possible Solution

Land a corrected Docker/browser change that sets a stable `/opt/playwright` default, creates it with `node:node` ownership for the default build, recursively reassigns browser files after any root-run preinstall, preserves BuildKit apt caches, creates the chromium symlink only after a successful install, and updates the current Docker/browser docs with focused Dockerfile assertions.

## Reproduction Assessment

Yes. Apply the PR and build without `OPENCLAW_INSTALL_BROWSER`, then run the bundled Playwright install command as the image's `node` user; the PR-created `/opt/playwright` is root-owned and not writable by that runtime user.

## Solution Assessment

No. Moving Docker browser storage to `/opt/playwright` is a maintainable direction, but this exact patch needs ownership, cache, docs, and test fixes before it is the narrowest safe solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assign /opt/playwright to node during setup:** `Dockerfile:165-166`
  - body: The default build path creates `/opt/playwright` as root with mode `775`, and no later `chown` runs when `OPENCLAW_INSTALL_BROWSER` is unset. Since the image later runs as `node`, documented runtime installs into the new default browser path can fail with `EACCES`.
  - confidence: 0.93
- **[P2] Leave the BuildKit apt lists cache intact:** `Dockerfile:176`
  - body: This `RUN` mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` clears the named cache for later builds while not reducing the committed image layer. Remove the cleanup from this cache-mounted step.
  - confidence: 0.88
