## Summary

Keep open. Current main partially overlaps by detecting Playwright-managed Chromium from the existing Docker cache, but it still does not move Docker browser storage to `/opt/playwright` or add a PATH-level Chromium command, and this PR head still has blocking Dockerfile issues plus insufficient real behavior proof.

## What This Changes

The PR changes Docker Playwright browser storage to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser docs to describe the new path.

## Best Possible Solution

A maintainer should decide the Docker browser path and PATH-symlink contract, then refresh or replace this branch with ownership-safe Dockerfile changes, aligned docs/tests, preserved cache autodetection, and real Docker proof.

## Reproduction Assessment

Yes for the PR review findings via source inspection. The PR head creates the default `/opt/playwright` path as root before the final `USER node`, deletes a BuildKit cache mount, and symlinks only the non-x64 Playwright layout; I did not run a live Docker build in this read-only review.

## Solution Assessment

No. Moving the path may be valid, but this patch is not the narrowest maintainable solution until ownership, Linux x64 layout support, BuildKit cache behavior, conflicts, and real Docker proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assign `/opt/playwright` to node before runtime installs:** `Dockerfile:164-165`
  - body: This setup layer creates the documented default browser path as root-owned and only applies mode `775`. In builds without `OPENCLAW_INSTALL_BROWSER`, the final `node` user has no later `chown`, so runtime Playwright installs into `/opt/playwright` fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point `chromium` at the Linux x64 layout too:** `Dockerfile:176`
  - body: The symlink only targets `chromium-*/chrome-linux/chrome`, but Playwright 1.60 maps Linux x64 Chromium to `chrome-linux64/chrome`. Browser-enabled x64 images can install Chromium successfully while exposing a dangling `/usr/local/bin/chromium`.
  - confidence: 0.9
- **[P3] Stop deleting the mounted apt lists cache:** `Dockerfile:175`
  - body: This step mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` clears the shared cache used by later builds while providing no image-size benefit for cache-mounted data.
  - confidence: 0.84
