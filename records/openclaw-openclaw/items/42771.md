## Summary

Keep open. The proposed Docker path migration still has concrete Dockerfile regressions, and current main only partially overlaps by autodetecting the Playwright cache without moving browser files or adding a PATH symlink.

## What This Changes

The branch moves Docker Playwright browser installs and docs to `/opt/playwright` and adds a `/usr/local/bin/chromium` symlink.

## Best Possible Solution

Maintainers should choose whether the Docker contract should migrate to `/opt/playwright` plus a PATH command or keep the current cache-autodetection design, then refresh this PR or replace it with Dockerfile contract coverage and real Docker proof.

## Reproduction Assessment

Yes for the PR regressions by source inspection: build the PR head without `OPENCLAW_INSTALL_BROWSER` and run a `node`-user browser install into `/opt/playwright`, or build with browser preinstall and exercise `/usr/local/bin/chromium` against a `chrome-linux64` Playwright layout. I did not run a Docker build in this read-only review.

## Solution Assessment

No. The direction may be valid, but the current diff is not mergeable until ownership, symlink layout, and BuildKit cache handling are fixed and maintainers decide the Docker path migration policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Make /opt/playwright owned by node in the default image:** `Dockerfile:164-165`
  - body: This setup layer creates `/opt/playwright` as root and only applies mode `775`. When `OPENCLAW_INSTALL_BROWSER` is unset there is no later `chown`, so the final `node` user cannot install browsers into the documented default path.
  - confidence: 0.95
- **[P2] Link chromium to the actual Playwright Linux path:** `Dockerfile:176`
  - body: The symlink hard-codes `chromium-*/chrome-linux/chrome`, but current Playwright-managed Linux Chromium can live under `chrome-linux64/chrome`. A browser-enabled image can therefore build with a dangling `/usr/local/bin/chromium` command.
  - confidence: 0.9
- **[P3] Do not delete the mounted apt lists cache:** `Dockerfile:175`
  - body: `/var/lib/apt` is mounted as a BuildKit cache in this `RUN`, so deleting `/var/lib/apt/lists/*` mutates the shared cache and provides no image-size benefit. This regresses rebuild caching for browser-enabled Docker builds.
  - confidence: 0.84
