## Summary

Keep open: current `main` still uses the old Docker Playwright cache path, and the PR still has a source-level runtime permission regression plus an apt cache regression that should be fixed before merge.

## What This Changes

The PR changes the Docker image to use `/opt/playwright` for Playwright browsers, adds a `chromium` symlink, and updates Docker/browser docs.

## Best Possible Solution

Keep the PR open, fix the Dockerfile ownership and apt-cache issues, update the current Docker/browser docs and source tests, then review it together with #42819.

## Reproduction Assessment

Yes for the PR-introduced blocker: source inspection shows the default build path would create `/opt/playwright` as root-owned before the image switches to `USER node`, so the documented runtime install path cannot write there. I did not run a Docker build in this read-only sweep.

## Solution Assessment

No as-is. The `/opt/playwright` direction may solve #42819, but the current diff needs a node-owned setup directory and should not delete apt lists from the BuildKit cache mount.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Chown `/opt/playwright` during setup:** `Dockerfile:164-165`
  - body: In the default image build where `OPENCLAW_INSTALL_BROWSER` is unset, only this setup layer runs. `mkdir` creates `/opt/playwright` as `root:root`, and mode `775` still does not let the later `USER node` write there, so the documented runtime `playwright-core ... install` flow will fail with `EACCES`. Create the directory with node ownership or chown it in this setup step.
  - confidence: 0.93
- **[P3] Remove apt-list deletion from the cache mount:** `Dockerfile:174-175`
  - body: This step mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` wipes the named cache rather than shrinking the image layer. That regresses subsequent cached builds and makes the cleanup comment misleading; drop the deletion inside the cache-mounted RUN.
  - confidence: 0.84
