## Summary

This PR should stay open because current main has not implemented the requested Docker browser-path change, and the proposed branch still has concrete Docker/runtime and docs issues to repair before merge.

## What This Changes

The PR moves Docker Playwright browsers to `/opt/playwright`, adds a `chromium` symlink, and updates Docker/browser docs for the new persistence path.

## Best Possible Solution

Rebase the PR, make `/opt/playwright` writable for `node` in both default and browser-preinstall builds, preserve BuildKit apt caches, update the current Docker browser docs surface, and then land it with Docker validation.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the patch defects: build the image without `OPENCLAW_INSTALL_BROWSER`, then run the bundled Playwright install as the runtime `node` user with `PLAYWRIGHT_BROWSERS_PATH=/opt/playwright`. I did not execute the Docker build in this read-only review.

## Solution Assessment

No, not as currently written. Moving Docker browser files to `/opt/playwright` is a reasonable direction, but the implementation needs ownership, cache, and current-docs fixes before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Make `/opt/playwright` writable for runtime installs:** `Dockerfile:164-165`
  - body: This setup creates `/opt/playwright` as root-owned and only changes its mode to `775`. In the default build path, `OPENCLAW_INSTALL_BROWSER` is unset, so the later recursive `chown` never runs; after the image switches to `USER node`, runtime installs such as the documented bundled Playwright CLI cannot write to the new default path.
  - confidence: 0.93
- **[P2] Update the current browser-control docs page:** `docs/tools/browser.md:411-412`
  - body: On current main, the Docker Playwright install guidance lives in `docs/tools/browser-control.md`, which still tells users to persist `/home/node/.cache/ms-playwright`. Updating the older browser page from this stale base leaves the active docs surface inconsistent with the Dockerfile change after rebase or merge.
  - confidence: 0.76
- **[P3] Preserve the BuildKit apt lists cache:** `Dockerfile:174-175`
  - body: This install block mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` writes into the named cache and forces later browser-enabled builds to repopulate apt lists. Cache-mount contents are not committed to the image layer, so this cleanup does not reduce final image size.
  - confidence: 0.82
