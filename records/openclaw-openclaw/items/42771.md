## Summary

Keep open for maintainer review: current main only solves part of the underlying Docker browser problem through Playwright cache autodetection, while this PR still proposes a broader `/opt/playwright` and PATH-contract change with concrete Dockerfile regressions and insufficient current proof.

## What This Changes

The PR moves Docker Playwright browser storage to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser docs to describe the new path.

## Best Possible Solution

Decide whether Docker should intentionally migrate from the shipped node-cache autodetection contract to `/opt/playwright`; if yes, refresh or replace this branch with ownership-safe Dockerfile changes, symlink layout handling, docs, focused tests, and current Docker proof.

## Reproduction Assessment

Yes. The PR regressions are reproducible by source inspection: build this branch without `OPENCLAW_INSTALL_BROWSER` and run a node-user Playwright install into `/opt/playwright`, or build with browser preinstall and exercise `/usr/local/bin/chromium` against a Playwright `chrome-linux64` layout.

## Solution Assessment

No. Moving the Docker browser cache may be valid, but this diff is not the narrowest maintainable solution because it changes the Docker persistence contract while regressing runtime ownership, hard-coding one Playwright Linux layout, and mutating a BuildKit apt cache.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Make `/opt/playwright` writable for the runtime user:** `Dockerfile:164-165`
  - body: This setup layer runs as root and leaves `$PLAYWRIGHT_BROWSERS_PATH` owned by `root:root` with mode `775`. In the default image where `OPENCLAW_INSTALL_BROWSER` is unset, the later `node` user cannot install browsers into the newly documented `/opt/playwright` path, so runtime installs fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point `chromium` at both Playwright Linux layouts:** `Dockerfile:176`
  - body: The symlink only targets `chromium-*/chrome-linux/chrome`, but the pinned Playwright Linux x64 layout uses `chrome-linux64/chrome`. Browser-enabled images can therefore expose a dangling `/usr/local/bin/chromium` command even after Chromium was installed.
  - confidence: 0.9
- **[P3] Stop deleting the mounted apt lists cache:** `Dockerfile:175`
  - body: This `RUN` mounts `/var/lib/apt` as a BuildKit cache, so `rm -rf /var/lib/apt/lists/*` deletes the shared cache contents used by later builds while providing no image-size reduction for cache-mounted data.
  - confidence: 0.84
