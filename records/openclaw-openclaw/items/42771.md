## Summary

Keep open: current main only partially overlaps by detecting Playwright-managed Chromium, while this PR still has Dockerfile correctness findings and needs maintainer review before changing the Docker browser path/PATH contract.

## What This Changes

The PR changes the Docker image Playwright browser path to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser docs for the new persistence path.

## Best Possible Solution

A maintainer should decide the Docker path and PATH-symlink contract, then refresh or replace this branch with ownership-safe Dockerfile changes, aligned docs/tests, preserved cache autodetection, and real Docker proof.

## Reproduction Assessment

Yes for the PR review findings via source inspection. The PR head leaves the default `/opt/playwright` setup path unwritable for the default `node` user in browser-disabled builds, deletes a BuildKit cache mount, and symlinks only the non-x64 Playwright layout; I did not run a live Docker build.

## Solution Assessment

No. Moving the path may be valid, but this patch is not the narrowest maintainable solution until ownership, Linux x64 layout support, BuildKit cache behavior, conflicts, and real Docker proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assign `/opt/playwright` to node before runtime installs:** `Dockerfile:164-165`
  - body: The setup layer creates the documented default browser path as `root:root` with mode `775`. In builds without `OPENCLAW_INSTALL_BROWSER`, no later `chown` runs before the image switches to `USER node`, so runtime Playwright installs into `/opt/playwright` fail with `EACCES`.
  - confidence: 0.95
- **[P2] Point `chromium` at the Linux x64 layout too:** `Dockerfile:176`
  - body: The symlink targets `chromium-*/chrome-linux/chrome`, but Playwright 1.60 maps Linux x64 Chromium to `chrome-linux64/chrome`. Browser-enabled x64 images can install Chromium successfully while exposing a dangling `/usr/local/bin/chromium`.
  - confidence: 0.9
- **[P3] Stop deleting the mounted apt lists cache:** `Dockerfile:175`
  - body: This step mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` clears the shared cache used by later builds while providing no image-size benefit for cache-mounted data.
  - confidence: 0.84
