## Summary

Keep this PR open but not merge-ready: current main only partially overlaps by detecting Playwright-managed Chromium, while this branch still proposes a distinct Docker default path and PATH symlink and introduces concrete Dockerfile regressions.

## What This Changes

The PR changes the Docker image Playwright browser path to `/opt/playwright`, adds a `/usr/local/bin/chromium` symlink, and updates Docker browser persistence docs.

## Best Possible Solution

Decide the Docker browser storage and PATH contract, then refresh the branch with node-writable path setup, Linux-layout-aware executable handling, preserved BuildKit caches, aligned docs/tests, and real Docker proof.

## Reproduction Assessment

Yes for the PR review findings via source inspection. The PR head leaves the default `/opt/playwright` setup path unwritable for the default `node` user in browser-disabled builds, deletes a BuildKit cache mount, and symlinks only the non-x64 Playwright layout; I did not run a live Docker build.

## Solution Assessment

No. Moving the path may be acceptable, but this patch is not the narrowest maintainable solution until ownership, Linux x64 layout support, BuildKit cache behavior, conflicts, and real Docker proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Assign `/opt/playwright` to node during setup:** `Dockerfile:164-165`
  - body: The directory is created as root:root with mode 775, but the image later runs as `node`. In builds without `OPENCLAW_INSTALL_BROWSER`, no later `chown` runs, so runtime Playwright installs into `/opt/playwright` fail with `EACCES`.
  - confidence: 0.95
- **[P2] Use the Playwright x64 Chromium layout:** `Dockerfile:176`
  - body: Playwright 1.60 maps Linux x64 Chromium to `chrome-linux64/chrome`, while this symlink targets only `chrome-linux/chrome`. Browser-enabled x64 images can install Chromium successfully while exposing a dangling `/usr/local/bin/chromium`.
  - confidence: 0.9
- **[P3] Preserve the mounted apt lists cache:** `Dockerfile:175`
  - body: This step mounts `/var/lib/apt` as a BuildKit cache, so deleting `/var/lib/apt/lists/*` clears the shared cache used by later builds while providing no image-size benefit for cache-mounted data.
  - confidence: 0.84
