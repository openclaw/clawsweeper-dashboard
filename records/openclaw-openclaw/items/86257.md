## Summary

Keep open. The patch is focused, but it only broadens the outer module-not-found predicate; the direct-miss guard still rejects Bun's raw-specifier `Cannot find module './dist/warning-filter.js' from ...` message, so the linked launcher crash loop would remain and the proof is Node/Vitest-only rather than real Bun behavior proof.

## What This Changes

The PR changes `openclaw.mjs` so `isModuleNotFoundError` also recognizes `Cannot find module` and `Cannot find package` text when an error lacks `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Update `isDirectModuleNotFoundError` to recognize the raw requested specifier in Bun's message while preserving Node URL and absolute-path checks, then validate with a Bun packaged-launcher smoke plus the launcher E2E regression.

## Reproduction Assessment

Yes for source-level reproduction: current `main` and `v2026.5.22` reject Bun's raw-specifier message shape, and a focused probe of the PR's proposed logic still returns `direct: false`. I did not run a live Bun install in this read-only review.

## Solution Assessment

No. Widening only `isModuleNotFoundError` is not enough because the actual swallow/rethrow decision is still made by `isDirectModuleNotFoundError`, which needs raw-specifier matching or equivalent Bun fields.
