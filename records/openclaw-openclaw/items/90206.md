## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Propagate fatal errors before partial-summary fallback:** `src/agents/compaction.ts:274-275`
  - body: `shouldFallbackOnError` only sees the error thrown out of `summarizeChunks`. When any chunk has already succeeded, `summarizeChunks` replaces a later provider failure with a new `partial summarization failure` error plus `partialSummary`, without preserving the original AWS credential error as a cause. That lets Bedrock compaction accept a partial summary after the credential chain fails, which is the history-preservation failure this PR is trying to prevent; attach/carry the original error or call the veto before wrapping it, with a regression for a later-chunk credential failure.
  - confidence: 0.86
- **[P2] Classify the production Bedrock error shape:** `src/agents/agent-hooks/compaction-safeguard.ts:390-392`
  - body: The structural provider-error check walks `Error.cause`, but production Bedrock summarization catches SDK exceptions, formats only `error.message` into `output.errorMessage`, and agent-core then wraps that string in a `CompactionError`. A terminal `ProviderError` with `tryNextLink: false` but no credential text will not reach this helper as a cause-bearing object, so the new regression mock does not cover the real path; preserve the cause through Bedrock/agent-core or classify/test the formatted production error.
  - confidence: 0.8
