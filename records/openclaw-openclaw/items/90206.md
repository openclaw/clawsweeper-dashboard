## What This Changes

The PR adds optional `aws-sdk` provenance to model-registry request-auth results and lets the compaction safeguard proceed for that explicit mode without static request credentials.

## Best Possible Solution

Land the explicit registry-to-safeguard auth provenance fix after maintainer review and normal checks, leaving AWS credential resolution and request signing owned by the Bedrock runtime.

## Reproduction Assessment

## Solution Assessment

Yes, this is the best narrow fix I found: it preserves explicit `aws-sdk` provenance at the registry boundary instead of rediscovering provider state in the safeguard, injecting a fake key, or changing Bedrock request signing.
