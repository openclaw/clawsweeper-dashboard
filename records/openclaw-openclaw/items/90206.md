## Summary

Keep open: the PR is aimed at the right Bedrock `aws-sdk` compaction/auth boundary, but the patch still lets a credential failure after partial chunk progress fall through as fallbackable, and the supplied proof still does not exercise a live AWS SDK credential-chain Bedrock compaction path.

The PR adds `authMode: "aws-sdk"` request-auth provenance, lets compaction proceed without static credentials for that mode, and adds a summarization fallback veto for AWS SDK credential failures.

## Best Possible Solution

Fix fatal AWS SDK credential propagation before partial-summary wrapping, then land the narrow registry-to-safeguard provenance fix after redacted live Bedrock compaction proof or explicit maintainer acceptance of the remaining provider-proof gap.
