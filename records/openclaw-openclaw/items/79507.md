## Summary

Keep open. Current main still lacks a provenance-backed filter for runtime-only cron delivery turns in memory session export, and the linked open PR with closing syntax is the active fix path; this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue requests that memory-core dreaming stop promoting runtime-generated cron delivery reports and other one-way system announcements into MEMORY.md.

## Best Possible Solution

Land or replace the linked provenance-based fix so runtime-only event turns and their immediate assistant delivery reports are excluded from session corpus export without suppressing user-typed marker text.

## Reproduction Assessment

Yes, at source level: current main creates a visible runtime-only event prompt and `buildSessionEntry()` has no provenance-backed filter for that prompt or the following assistant report before dreaming records session-corpus recall signals. I did not run a live overnight dreaming sweep in this read-only review.

## Solution Assessment

Yes, the linked PR's provenance-backed session export filter is the narrow maintainable direction because it avoids text-only spoofing while keeping generated delivery reports out of memory ingestion. The issue should stay open until that PR or an equivalent fix lands.
