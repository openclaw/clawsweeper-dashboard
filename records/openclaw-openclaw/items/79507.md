## Summary

Keep open: current main has partial cron-session filtering, but the specific runtime-only cron delivery-report path is still only addressed by the linked open fix PR, which uses closing syntax and should be reviewed before this issue is closed.

## What This Changes

The issue asks memory-core dreaming to stop promoting cron delivery reports and one-way system announcements into durable memory.

## Best Possible Solution

Land a spoofing-safe runtime-only provenance filter so cron delivery reports and other internal one-way announcements stay out of dreaming/session-corpus promotion while real user text remains searchable.

## Reproduction Assessment

Yes, at source level. Current main turns blank transcript runtime events into a visible marker prompt, and the memory session reader lacks internal runtime-only provenance handling for that marker plus the immediate assistant delivery report.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: filter using internal provenance instead of the visible marker text, which avoids the spoofing issue already called out in current session-reader comments.
