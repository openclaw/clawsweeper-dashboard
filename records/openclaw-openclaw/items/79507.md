## Summary

Current main already excludes isolated cron-run transcripts from dreaming session ingestion, so the reported v2026.5.7 pollution path is solved on main but not yet proven in a shipped release.

## What This Changes

Dreaming should avoid promoting cron delivery reports and other one-way system announcements into durable memory.

## Best Possible Solution

Keep the current main transcript-source filtering so cron-run artifacts never enter the dreaming session corpus, then ship it in the next release.

## Reproduction Assessment

No current-main repro remains: the described cron announce path is covered by source filtering and a regression test that leaves the dreaming session corpus absent for a cron-run transcript. The report is still plausible for v2026.5.7 because the proof commit is not contained in that release tag.

## Solution Assessment

Yes. Filtering by cron-run transcript provenance before session-corpus ingestion is the narrow maintainable fix and avoids adding a new config knob or broad ranking-policy change.
