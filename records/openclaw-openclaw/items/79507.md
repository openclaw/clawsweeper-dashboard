## Summary

Keep open: current main still has a source-visible gap for runtime-only cron delivery reports, and the issue is already paired with an open closing PR that should either land or be superseded before the issue closes.

## What This Changes

The issue asks for memory-core dreaming promotion to exclude cron delivery reports and other one-way runtime announcements from durable memory promotion.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79547 or a narrow replacement that records internal runtime-only provenance and filters those turns at session export without relying on spoofable visible text.

## Reproduction Assessment

Yes, at source level: current main creates a visible runtime-only marker prompt, exports user/assistant transcript lines without a runtime-only provenance filter, and memory-core ingests that export into short-term recall signals. I did not run a live cron/dreaming sweep because this was a read-only review.

## Solution Assessment

Yes, the linked PR's provenance-backed session export filter is the narrowest maintainable direction because it avoids broad text matching while preserving real user text that happens to look like a runtime marker.
