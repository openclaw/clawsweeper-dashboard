## Summary

Keep open: the reported memory-core dreaming bug is still a valid current-main source-level gap, and an open linked PR already references this issue with closing syntax, so the issue should close only after that fix lands.

## What This Changes

The issue asks memory-core dreaming to exclude runtime cron delivery reports and other one-way system announcements from short-term promotion into durable memory.

## Best Possible Solution

Land a provenance-backed filter that marks runtime-only event turns at transcript persistence time and excludes those turns plus their immediate assistant delivery reports from memory session-corpus export.

## Reproduction Assessment

Yes, at source level: current main creates a visible runtime-only event marker and buildSessionEntry lacks a provenance-backed filter for that marker and the following assistant delivery report. I did not run a live dreaming cycle in this read-only review.

## Solution Assessment

Yes, the linked PR's provenance-backed approach is the narrowest maintainable fix because it avoids relying on spoofable user-visible marker text and keeps the filtering at the session export boundary.
