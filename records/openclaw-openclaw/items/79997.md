## Summary

Keep open. This PR targets a real current-main TUI configurability gap, but it is not merge-ready because the external PR still lacks after-fix real behavior proof and the CLI resume watchdog default change conflicts with an existing backend merge test expectation.

## What This Changes

This PR adds `cli.tui.streamingWatchdogMs` config/schema/docs/tests, wires it into TUI event handlers, raises the TUI streaming watchdog default to 10 minutes, and raises CLI fresh/resume watchdog max defaults to 60 minutes.

## Best Possible Solution

Land a maintainer-approved version that exposes a validated TUI watchdog setting, keeps code/tests/docs/changelog aligned, and deliberately accepts or narrows the larger CLI default increases.

## Reproduction Assessment

Yes, from source inspection. Current main has an internal `streamingWatchdogMs` handler option, but `runTui` does not pass it and the strict config schema only exposes `cli.banner`, so normal TUI sessions fall back to the 30-second default.

## Solution Assessment

No, not merge-ready as written. The TUI config wiring is a plausible direction, but the stale backend default test must be fixed, real behavior proof is missing, and the new default durations need maintainer agreement.
