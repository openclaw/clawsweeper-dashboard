## Summary

## What This Changes

The PR changes macOS PortGuardian local-mode process recognition for launchd-managed `node .../openclaw/dist/index.js gateway` commands and adds Swift unit tests for expected and unexpected command shapes.

## Best Possible Solution

Use a token-aware matcher for known OpenClaw entrypoints plus a standalone `gateway` argument, add a negative substring regression test, and require redacted macOS launchd sweep proof before merge.

## Reproduction Assessment

Yes at source level: current `main` rejects the reported Node dist Gateway command shape, and the same `sweep(mode: .local)` path terminates unexpected local listeners. I did not run a live macOS launchd cycle in this read-only review.

## Solution Assessment

No as written. PortGuardian is the right layer, but the matcher should parse argv-style tokens and require a standalone `gateway` argument before preserving the listener.
