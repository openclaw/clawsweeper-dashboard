## What This Changes

The PR changes macOS PortGuardian local-mode matching to preserve Node-launched `openclaw/dist/index.js gateway` listeners and adds Swift unit coverage for that matcher.

## Best Possible Solution

Use a token-aware matcher for known OpenClaw entrypoints plus a standalone `gateway` argument, keep focused positive and negative Swift coverage, and require redacted macOS launchd sweep proof before merge.

## Solution Assessment

No as written. PortGuardian is the right layer, but the current matcher should follow argv-token structure rather than broad substring checks before it controls whether a listener is preserved.
