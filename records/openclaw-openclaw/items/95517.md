Keep this PR open: PortGuardian is the right surface, but the patch still uses a broad substring allowlist that can preserve an unexpected listener on the Gateway port, and the supplied proof is build/unit-test output rather than a live launchd sweep.

## What This Changes

The PR adds a macOS PortGuardian local-mode allowlist for command lines containing `openclaw`, `dist/index.js`, and `gateway`, plus Swift tests for expected and unexpected process shapes.

## Best Possible Solution

Use a token-aware matcher for known OpenClaw entrypoints plus a standalone `gateway` argument, add a negative substring regression test, and require redacted macOS launchd sweep proof before merge.

## Reproduction Assessment

Yes at source level: current main rejects the linked issue's `node .../openclaw/dist/index.js gateway --port 18789` shape in `isExpected`, and local `sweep` kills unexpected listeners. I did not run a live macOS launchd cycle.

## Solution Assessment

No as written. PortGuardian is the right layer, but the matcher should parse argv-style tokens and require standalone `gateway` before preserving a listener.
