## Summary

Keep open: current main and the latest shipped tag still swallow errors from the subagent completion wait path, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks for subagent completion finalization errors to be logged with run/session context and to trigger recovery instead of being silently ignored.

## Best Possible Solution

Update and land https://github.com/openclaw/openclaw/pull/75462, or an equivalent narrow fix, so completion failure is logged and retried or represented as a first-class failed completion path with targeted regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: make `agent.wait` return a terminal result and make `completeSubagentRun` reject after `entry.endedAt` is persisted, and current main catches that failure with no log or recovery path.

## Solution Assessment

Partially. Logging and recovery are the right direction, but the implementation should not rely only on generic orphan recovery after `endedAt` is set because that recovery path skips normal already-ended records.
