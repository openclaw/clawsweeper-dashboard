## Summary

Keep open: current main and v2026.6.8 still scope `OPENCLAW_NO_AUTO_UPDATE` to configured startup auto-update applies, while explicit supervised Gateway `update.run` can still create the launchd managed-service handoff and schedule an `update.run` restart.

## What This Changes

Request to document and make supervised macOS Gateway `update.run`/`gateway-update` restarts honor `OPENCLAW_NO_AUTO_UPDATE` before creating launchd restart handoffs.

## Best Possible Solution

Maintainers should define the kill-switch contract, then either gate Gateway-originated managed `update.run` handoffs on `OPENCLAW_NO_AUTO_UPDATE` and document it, or document explicit `update.run` as outside the kill-switch and keep launchd handoff reliability under the broader update-run tracker.

## Reproduction Assessment

Yes for source-level reproduction: current main still lets supervised global/git `update.run` start a managed handoff and schedule an `update.run` restart without checking `OPENCLAW_NO_AUTO_UPDATE`. No fresh live macOS launchd repro was run in this read-only review.

## Solution Assessment

Unclear: reusing `OPENCLAW_NO_AUTO_UPDATE` is a small implementation shape, but it changes the contract for explicit control-plane update requests. The best next step is a maintainer decision on whether the kill-switch blocks only configured automatic applies or all Gateway-originated package update restarts.
