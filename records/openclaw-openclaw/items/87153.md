## Summary

## What This Changes

Adds macOS helpers to select a node-scoped WebChat default in remote mode, routes the WebChat picker/window entry points through them, and adds Swift tests for remote/local default selection.

## Best Possible Solution

Land a version that keeps local mode unchanged, defaults remote WebChat to the actual paired node session, isolates identity state in tests, and includes redacted live two-Mac proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main resolves WebChat defaults through the shared main session, while gateway node events and node registry identity use node-scoped keys. I did not run the live two-Mac remote topology.

## Solution Assessment

Not yet. The runtime direction is plausible and narrow, but the PR should isolate test state and provide real remote two-Mac picker/message proof before it is the best mergeable fix.
