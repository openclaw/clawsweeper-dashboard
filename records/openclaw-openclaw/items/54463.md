## Summary

Keep open. The stale docs mismatch was corrected, but current main still has no OpenClaw-owned QMD traversal/version contract, no resolved collection metadata for exclusions or no-follow behavior, and docs still describe workspace-visible temp repos as a live workaround state; the adjacent open pull request is not a landed or sufficient canonical fix.

## What This Changes

QMD-backed memory indexing should avoid symlink-loop recursion in workspace-visible temporary monorepos or provide a documented first-class exclusion/traversal contract.

## Best Possible Solution

Confirm the QMD version where traversal became cycle-safe/no-follow, then add an OpenClaw-side contract through version diagnostics/docs or a pre-update guard so broad QMD workspace indexing cannot regress on symlink-loop temp repos.

## Reproduction Assessment

Yes, with a version caveat. The issue comments give a concrete symlink-loop fixture and current OpenClaw still delegates plain collections to qmd update, but current upstream QMD source appears to disable symlink following, so the remaining repro depends on the installed QMD version/contract.

## Solution Assessment

Unclear. The best fix needs maintainer choice between QMD minimum-version/probe/docs, OpenClaw pre-update guarding, or explicit QMD exclusion semantics; the related open pull request does not settle that contract.
