## Summary

Keep this open. The stale docs-only part was already corrected, but current main still has no OpenClaw-side QMD traversal guard, no typed exclusion/no-follow setting, no minimum safe-QMD version contract, and no regression showing symlink-loop temp monorepos cannot break QMD indexing.

## What This Changes

The issue requests QMD memory indexing robustness so workspace-visible temp monorepos with symlink loops cannot recurse until `ENAMETOOLONG`, and so docs match runtime symlink behavior.

## Best Possible Solution

Keep this as an active QMD traversal robustness item. Add the controlled symlink-loop temp-monorepo repro as regression coverage, then either require/detect a cycle-safe QMD scanner or add an OpenClaw-side pre-update no-follow/exclusion/realpath-cycle guard before `qmd update`; update QMD docs once runtime behavior is fixed.
