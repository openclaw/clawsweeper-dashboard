## Summary

Keep open. Current main and the latest release still document the workspace-visible QMD temp-repo symlink case as a live limitation, and OpenClaw still hands plain path/pattern collections to `qmd update` without enforcing or diagnosing a cycle-safe scanner contract.

## What This Changes

The issue asks for QMD-backed memory indexing to avoid breaking on symlink loops in workspace-visible temporary monorepos, with docs and runtime behavior agreeing on the traversal contract.

## Best Possible Solution

Keep this as an active QMD traversal robustness item: add the controlled symlink-loop repro as regression coverage, then define one maintained scanner contract through QMD version enforcement/diagnostics, upstream traversal, or pre-update exclusion plumbing before removing the docs workaround.
