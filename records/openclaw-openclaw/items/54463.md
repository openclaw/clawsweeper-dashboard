## Summary

Keep open. Current main and the latest release still document the workspace-visible temp-repo `ENAMETOOLONG` case as a QMD runtime limitation, while OpenClaw still delegates indexing to external `qmd update` with path/pattern collections and no QMD version floor or traversal contract enforcement.

## What This Changes

The issue asks OpenClaw to prevent QMD-backed memory indexing from recursing through symlink-looping workspace-visible temporary monorepos and breaking indexing with `ENAMETOOLONG`.

## Best Possible Solution

Keep this as the canonical QMD traversal robustness item, verify supported QMD version behavior with the symlink-loop fixture, then either enforce/document a safe QMD version floor or add an owner-approved OpenClaw pre-update pruning/exclusion path with tests and docs aligned.

## Solution Assessment

Unclear. The narrowest maintainable solution may be a QMD version floor plus docs/doctor guidance, OpenClaw-side traversal hardening, or explicit QMD ignore plumbing; maintainers need to choose the owner boundary before implementation.
