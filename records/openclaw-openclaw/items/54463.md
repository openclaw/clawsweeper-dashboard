## Summary

Keep this open. Current main and the latest shipped OpenClaw release still lack an OpenClaw-owned QMD version, traversal, or pre-update guard contract that proves workspace-visible symlink loops cannot break QMD indexing, even though latest upstream QMD source now appears to avoid following symlinks.

## What This Changes

The issue reports that QMD-backed memory indexing can recurse through workspace-visible temporary monorepo symlink loops and fail with ENAMETOOLONG.

## Best Possible Solution

Define the supported QMD scanner contract, then either require/diagnose a safe QMD version or add an OpenClaw-side pre-update traversal guard, with docs and a symlink-loop regression fixture matching that choice.

## Reproduction Assessment

Unclear for current main with latest QMD, but source-level evidence and issue comments give a credible reproduction for the affected traversal contract. Latest upstream QMD source now appears safe, so the remaining OpenClaw repro is the missing enforced version or pre-update guard contract.

## Solution Assessment

Unclear until maintainers choose the boundary. A maintainable fix should either rely on a proven safe QMD scanner version with diagnostics/docs or add OpenClaw-side cycle-safe traversal/exclusion before qmd update can fail.
