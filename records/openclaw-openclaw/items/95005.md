## Summary

Keep open: current main and the latest release still lack the requested `needed` sandbox mode, and this PR is the active candidate for the linked issue, but it is not merge-ready because the new mode is wired for embedded exec while the Codex app-server native shell/file surface can still be treated as host-capable. ## What This Changes

Adds `agents.defaults.sandbox.mode: "needed"`, run-versus-tool sandbox activation, lazy embedded exec sandbox resolution, sandbox explain/audit/docs updates, and focused tests.

## Best Possible Solution

Before merge, make `needed` a single fail-closed sandbox activation contract across embedded exec and Codex native tools, or explicitly disable/document unsupported native surfaces, with proof for both chat-only success and sandbox-bound exec failure.

## Solution Assessment

No. The PR is a plausible embedded-exec implementation, but it is not the best complete fix until the Codex native shell/file path either lazily resolves the sandbox or fails closed under the new mode.
