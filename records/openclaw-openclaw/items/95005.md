## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No: adding lazy sandboxing only to embedded exec is incomplete because Codex app-server native shell/file/MCP execution is another sandbox-bound surface. The safer fix is to route `needed` through that decision path or disable native execution fail-closed under `needed`.
