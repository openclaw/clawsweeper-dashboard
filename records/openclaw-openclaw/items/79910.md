## Summary

Keep open: the underlying session-store recovery bug is real and related to https://github.com/openclaw/openclaw/issues/77783, but this PR currently contradicts its own recovery policy and lacks after-fix real behavior proof.

## What This Changes

Adds load-time session-store recovery from `.bak` and `.tmp` artifacts, plus regression tests and a changelog entry.

## Best Possible Solution

Revise this into a narrow recovery-only patch that recovers only after an existing primary store actually fails to load, preserves intentional empty stores, matches the real temp artifact contract, and includes real setup proof.

## Reproduction Assessment

Yes. Source inspection shows current main returns an empty store on final read/parse failure, and https://github.com/openclaw/openclaw/issues/77783 gives a concrete 0-byte `sessions.json` incident.

## Solution Assessment

No. The recovery direction is plausible, but this implementation also resurrects stale backups for missing or intentionally empty stores and has no after-fix real behavior proof.
