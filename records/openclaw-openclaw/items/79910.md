## Summary

Keep open. This is a useful sessions recovery fix, but the branch still writes recovered backup/tmp JSON back to `sessions.json` before the current migration and validation pipeline runs, so a mixed recovered artifact can repersist malformed session rows that current main strips on load.

## What This Changes

The PR adds load-time `sessions.json` recovery from validated `.bak` and stale `.tmp` artifacts, self-heals the primary file, adds recovery tests, and edits `CHANGELOG.md`.

## Best Possible Solution

Keep the recovery approach, but serialize the migrated and normalized recovered store during self-heal and add a regression test for mixed valid plus malformed backup/tmp artifacts.

## Reproduction Assessment

Yes. Current main is source-reproducible: a final `sessions.json` read/parse failure is swallowed and the loader proceeds with an empty store, while the PR body includes after-fix logs for stale tmp recovery.

## Solution Assessment

No, not yet merge-ready. Backup/tmp recovery is the right narrow direction, but the self-heal write should run after the existing migration and normalization pipeline or serialize its normalized result.
