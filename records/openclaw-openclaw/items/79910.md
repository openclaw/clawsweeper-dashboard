## Summary

Keep open: this is a plausible session-store recovery bug fix with sufficient after-fix logs, but the current branch still has merge-blocking recovery-safety and test-coverage defects.

## What This Changes

The PR adds load-time recovery for corrupted session stores from `.bak` and `.tmp` artifacts, self-healing writeback, recovery tests, and a changelog entry.

## Best Possible Solution

Keep the loader-boundary recovery approach, but validate every recovery candidate before reading, reject in-flight temp files, and align fs-safe fixtures with the pinned temp prefix before merge.

## Reproduction Assessment

Yes, source-reproducible: current main catches a zero-byte or malformed `sessions.json` read/parse failure and returns the initialized empty store, and the linked report provides a concrete zero-byte incident. I did not run a mutating repro because this review is read-only.

## Solution Assessment

No, not yet. Load-time recovery is the right narrow boundary, but the current PR still trusts backup and fresh temp artifacts too broadly and the fs-safe test fixture does not cover the pinned writer path.
