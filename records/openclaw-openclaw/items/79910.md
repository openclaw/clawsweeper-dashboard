## Summary

Keep open: this PR is a plausible fix for the linked zero-byte session-store bug, but the current head still reads unvalidated backup artifacts, accepts fresh temp artifacts, and has a fs-safe fixture mismatch that should block merge.

## What This Changes

The branch adds load-time session-store recovery from `.bak` and `.tmp` artifacts, synchronous self-heal, recovery tests, and a changelog entry.

## Best Possible Solution

Keep recovery at the `loadSessionStore` boundary, validate every recovery artifact before reading, reject in-flight temp files, align tests with the pinned fs-safe temp contract, and leave the save hot path untouched.

## Reproduction Assessment

Yes, source-reproducible: current main returns an empty store after a zero-byte or malformed `sessions.json` read failure, and the linked report provides a concrete Linux incident. I did not run a mutating repro because this review is read-only and `pnpm` is unavailable in this checkout.

## Solution Assessment

No. The loader-boundary recovery direction is narrow and maintainable, but this PR is not mergeable until recovery candidates are validated/freshness-gated and the fs-safe test fixture matches the pinned writer contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Validate backup candidates before reading:** `src/config/sessions/store-load.ts:182`
  - body: `sessions.json.bak` is still read and parsed before the link and hardlink checks used for temp candidates. Since this imports private session metadata as trusted recovery state, run the same `lstatSync`/regular-file/no-symlink/no-hardlink gate before opening the backup file.
  - confidence: 0.93
- **[P2] Reject fresh temp recovery candidates:** `src/config/sessions/store-load.ts:218`
  - body: Temp files are accepted solely by name and file shape, then read during recovery. If a writer is still active and the primary read transiently fails, this can import an in-flight temp despite the PR's stated freshness mitigation; add an age guard and cover the fresh-only case.
  - confidence: 0.88
- **[P2] Use the pinned fs-safe temp name in tests:** `src/config/sessions/store-load.recovery.test.ts:87`
  - body: These fixtures create `.sessions.json.<pid>.<uuid>.tmp`, but the pinned `@openclaw/fs-safe` writer creates `.fs-safe-replace.<pid>.<uuid>.tmp` by default and the loader regex matches that prefix. Update the fixtures so the test actually exercises the intended fs-safe recovery path.
  - confidence: 0.96
