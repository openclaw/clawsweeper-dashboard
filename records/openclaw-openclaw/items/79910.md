## Summary

Keep open: this is a focused and useful session-store recovery PR, and the prior backup validation plus fs-safe fixture issues are addressed, but the latest head still reads fresh temp artifacts even though the stated mitigation is to ignore them to avoid racing active writers.

## What This Changes

The PR adds session-store load-time recovery from `.bak` and residual temp files, self-heals the main store, adds recovery tests, and records a changelog entry.

## Best Possible Solution

Keep the load-time recovery approach, but add the missing freshness gate for temp candidates and cover it with a focused regression test before merge.

## Reproduction Assessment

Yes, source-reproducible: current main returns an empty store after a final `sessions.json` read/parse failure and has no recovery path. The linked report provides a concrete zero-byte incident; I did not run a mutating repro in this read-only review.

## Solution Assessment

No, not yet. Load-time recovery from validated local artifacts is the narrow maintainable fix, but this PR should skip fresh temp files before reading them so it does not race active writers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
