## Best Possible Solution

Keep the load-time artifact recovery, but gate it to actual primary read/parse failures and preserve valid reset-store semantics with a `[]` regression test.

No, not yet. Validated load-time recovery from local artifacts is the right direction, but the current gate should not recover from stale artifacts after the primary file parsed successfully as a valid reset/non-object shape like `[]`.
