## Summary

Keep this PR open for normal maintainer review. Current main still returns an empty session store when `sessions.json` is zero-byte or malformed, and this latest branch addresses the earlier blockers by removing `.bak` promotion, avoiding load-time primary rewrites, and reading stale temp artifacts through an opened file descriptor.

## What This Changes

The PR adds load-time in-memory recovery of a corrupt `sessions.json` from stale session-store temp artifacts, shares the temp staleness constant, and adds focused recovery regression tests.

## Best Possible Solution

Land the narrowed stale-temp recovery if maintainers accept that recovery source, and keep any rebuild/doctor remediation as a separate follow-up for the linked issue.

## Reproduction Assessment

Yes. Source inspection of current main shows zero-byte or malformed `sessions.json` falls through as an empty store, and the PR body supplies after-fix live loader output for the stale-temp recovery path.

## Solution Assessment

Yes for the narrowed PR scope. Recovering only from stale writer-owned temp artifacts avoids inventing a `.bak` contract and avoids the prior load-time writeback race; broader rebuild and doctor behavior can be handled separately.
