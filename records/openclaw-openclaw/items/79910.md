## Summary

Keep open: this is a useful recovery PR with sufficient runtime proof, but the current patch still accepts generic JSON objects as trusted recovery stores, so unrelated `.bak` or `.fs-safe-replace` artifacts can mask later valid candidates or self-heal bad session-store shape.

## What This Changes

The PR adds load-time session-store recovery from `.bak` and stale temp artifacts, synchronous self-heal, focused recovery tests, and a changelog entry.

## Best Possible Solution

Land a recovery-only fix that validates `.bak` and temp candidates as session-entry-shaped stores, skips unrelated artifacts, preserves the existing save hot path, and keeps the focused recovery tests green.

## Reproduction Assessment

Yes, source-reproducible: current main catches zero-byte or malformed `sessions.json` reads and returns an empty store with no recovery path. I did not run a mutating repro because this review was read-only, but the linked incident and PR proof cover the runtime scenario.

## Solution Assessment

No, not yet. Load-time recovery from existing local artifacts is the right narrow boundary, but the implementation must validate recovered contents and continue past unrelated candidates before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Validate backup contents before stopping recovery:** `src/config/sessions/store-load.ts:186-187`
  - body: `isSessionStoreRecord` only checks that the parsed backup is a top-level object. A stale `sessions.json.bak` like `{ "version": 1 }` makes `store` nonempty, prevents scanning a later valid temp artifact, and can be written back as the recovered store; require session-entry-shaped contents before accepting the backup.
  - confidence: 0.9
- **[P2] Continue past unrelated temp artifacts:** `src/config/sessions/store-load.ts:232-241`
  - body: The `.fs-safe-replace` temp name is generic for any atomic replace in this directory, but this loop accepts any JSON object and breaks. An unrelated `{}` or `{ "version": 1 }` candidate can mask a later valid session-store artifact, so validate candidate contents and keep scanning when they are empty or unrelated.
  - confidence: 0.93
- **[P3] Format store-load.ts with oxfmt:** `src/config/sessions/store-load.ts:176-183`
  - body: The recovery condition and one-line backup guard do not match the repository's formatting style, so the changed-format check is likely to fail until this file is formatted with the repo wrapper.
  - confidence: 0.86
