Keep this PR open: the underlying bug is real and the recovery direction is useful, but the current patch still has blocking correctness and validation issues on the latest head.

## What This Changes

The PR adds load-time recovery of corrupted `sessions.json` from `.bak` or stale temporary artifacts, self-heals the primary store, adds recovery tests, and edits the changelog.

## Best Possible Solution

Keep loader-level recovery, but accept and self-heal only candidates that survive the existing session-store migration and normalization pipeline, with focused regression coverage.

## Reproduction Assessment

Yes by source inspection for the underlying bug. On current main, a zero-byte or malformed `sessions.json` follows the final read/parse failure path and returns the initialized empty store without trying `.bak` or `.tmp` recovery.

## Solution Assessment

No, not yet. The loader boundary is appropriate, but the PR must normalize and validate recovery candidates before choosing a source or writing the self-healed primary file.
