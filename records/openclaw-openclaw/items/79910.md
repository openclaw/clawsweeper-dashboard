## Summary

Keep this PR open: it is a focused fix for a real session-store recovery bug and has sufficient runtime proof, but the current head still has merge-blocking loader, security, and test issues.

## What This Changes

The PR adds load-time recovery for a corrupted `sessions.json` from `.bak` or temp artifacts, self-heals the store, adds recovery tests, and updates the changelog.

## Best Possible Solution

Keep the session-loader recovery approach, but validate every recovery artifact before reading, reject fresh/in-flight temps, and align the recovery tests with the pinned fs-safe temp naming before merge.

## Reproduction Assessment

Yes, source-reproducible: current main returns an empty store after the final `sessions.json` read/parse failure, and the linked bug reports a concrete zero-byte Linux incident. I did not run a mutating repro because this review must keep the checkout read-only.

## Solution Assessment

No, not yet. Load-time recovery is the narrow maintainable boundary, but this PR is not mergeable until backup reads are pre-validated, fresh temp artifacts are excluded, and the fs-safe tests match the actual writer contract.
