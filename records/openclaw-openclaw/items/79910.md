## What This Changes

This PR teaches `loadSessionStore` to recover a corrupted primary session store from validated `.bak` or stale session-store `.tmp` artifacts, then self-heal the primary file and cover the behavior with regression tests.

## Best Possible Solution

Land the bounded load-path recovery only after maintainers accept validated `.bak`/stale-temp promotion as the desired corrupted-store startup behavior and required checks remain green.

## Reproduction Assessment

Yes. Current main source shows malformed or zero-byte primary reads fall through to an empty store, and the PR body plus tests exercise real temp files and the actual `loadSessionStore` recovery path.

## Solution Assessment

Yes, with a merge-policy caveat. The fix is in the session-store load owner, reuses the existing temp artifact matcher and stale window, validates candidates, and refuses recovery on primary read errors; the remaining question is whether maintainers want this automatic promotion policy.
