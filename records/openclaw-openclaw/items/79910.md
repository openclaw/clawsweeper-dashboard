## Summary

Keep this PR open: the underlying session-store recovery bug is valid and the new real-behavior proof is sufficient, but the patch still has blocking correctness/security issues in the recovery implementation and tests.

## What This Changes

The PR adds load-time recovery for corrupted `sessions.json` from `.bak` and temporary artifacts, adds recovery tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Keep the load-time recovery direction, but align the temp recovery tests with the pinned writer contract and apply the same lstat-based rejection to `.bak` before merge.

## Reproduction Assessment

Yes, from source inspection: current main returns an empty store after the final `sessions.json` read/parse failure, and the linked report gives a concrete zero-byte incident. I did not run tests because this was a read-only checkout review.

## Solution Assessment

No. Load-time recovery is a reasonable narrow fix, but this patch is not the best mergeable form until it proves the real fs-safe temp path and closes the `.bak` symlink gap.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Match the fs-safe temp fixture to the pinned writer:** `src/config/sessions/store-load.recovery.test.ts:87`
  - body: These fs-safe recovery tests create `.sessions.json.<pid>.<uuid>.tmp`, but the implementation only accepts the pinned writer's `.fs-safe-replace.<pid>.<uuid>.tmp` files. The fs-safe recovery test will not exercise the implemented candidate path, and the preference test can fall through to the legacy temp instead of proving fs-safe recovery.
  - confidence: 0.97
- **[P2] Reject symlinked backup candidates before reading:** `src/config/sessions/store-load.ts:182`
  - body: The new tmp scan rejects symlinks with `lstatSync`, but the `.bak` candidate is read directly with `fs.readFileSync`. Since this PR explicitly broadens recovery to sibling artifacts and claims symlink hardening, apply the same lstat/non-file/hardlink rejection to `sessions.json.bak` before reading it.
  - confidence: 0.9
