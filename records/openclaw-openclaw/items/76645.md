## Summary

Keep open. The OpenAI-compatible max-token fallback addresses a real current-main gap and overlaps the useful part of related PR #66146, but the QMD hunk disables the documented `sync.onSessionStart` behavior instead of preserving session-start freshness while fixing the reader/writer race.

## What This Changes

This PR forwards model `maxTokens` into OpenAI-compatible stream payloads when request options omit `maxTokens`, and disables QMD session-start sync from first-search warmup.

## Best Possible Solution

Land or split the OpenAI max-token fallback with regression coverage, and rework the QMD race fix so session-start freshness remains supported unless maintainers intentionally change the config contract.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main omits `model.maxTokens` in both OpenAI-compatible builders when options are absent, and the proposed QMD hunk would make documented `onSessionStart` behavior a no-op. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The max-token fallback is a narrow fix, but disabling QMD session-start sync is not the best way to fix a race; the update should be serialized, deferred, or otherwise made reader-safe while preserving the documented freshness hook.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve QMD session-start sync behavior:** `extensions/memory-core/src/memory/qmd-manager.ts:1587-1594`
  - body: After `sessionWarm.add(key)`, this hunk leaves QMD's `sync.onSessionStart` path with no update at all. That contradicts the documented default that session start triggers an index sync for early-turn freshness and breaks the existing QMD tests that expect one non-blocking `qmd update` for a fresh session. Please fix the reader/writer race without making the configured session-start hook a no-op, or change the documented contract with maintainer signoff.
  - confidence: 0.9
