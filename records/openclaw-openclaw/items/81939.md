## Summary

Keep open. The PR is a plausible narrow fix for the bare reset/new regression, but it is not merge-ready because current e2e/helper tests and public docs still assert the acknowledgement-only behavior that this patch removes.

## What This Changes

The PR changes non-ACP bare `/new` and `/reset` to fall through to the model/startup path after reset hooks unless a hook already routed a reply, and updates reset-hook unit expectations.

## Best Possible Solution

Restore bare reset/new fallthrough through the existing startup prompt path, keep hook-routed replies, ACP handling, reset tails, and empty-provider-call protection intact, and align e2e/helper tests plus docs with that contract.

## Solution Assessment

No, not merge-ready yet. Returning `null` for non-routed bare resets is the narrow code direction, but the PR must update stale e2e/helper/docs expectations and preserve the empty-provider-call guard.
