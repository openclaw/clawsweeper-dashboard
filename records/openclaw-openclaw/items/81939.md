## Summary

Keep open. The core code direction is plausible for the linked regression, but the branch is not merge-ready because higher-level e2e/helper coverage and public docs still assert acknowledgement-only bare reset behavior with no model call.

## What This Changes

The PR changes bare non-ACP `/new` and `/reset` to fall through after reset hooks unless a hook routed its own reply, and updates reset-hook unit expectations.

## Best Possible Solution

Restore bare reset/new fallthrough through the existing startup prompt path, keep hook-routed replies terminal and empty-input provider errors covered, and align docs plus e2e/helper tests with that contract.
