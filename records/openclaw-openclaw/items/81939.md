## Summary

Keep open. The core fallthrough direction matches the reported regression and supplied live Gateway proof, but the branch is not merge-ready because it leaves higher-level reset coverage and public docs asserting the old acknowledgement-only contract.

## What This Changes

The PR changes bare non-ACP `/new` and `/reset` to fall through to the normal model/startup path after reset hooks unless a hook already routed its own reply, and updates reset-hook unit expectations.

## Best Possible Solution

Restore bare reset/new fallthrough through the existing startup prompt path, keep hook-routed replies terminal, preserve the empty-input provider guard with coverage, and align docs plus tests with that contract.

## Reproduction Assessment

Yes, source-level: current main returns a terminal command result for bare `/new` and `/reset`, and the reply pipeline stops before the existing bare-session reset startup prompt path can run. The PR body also includes copied live Gateway output showing the after-fix `/new` path returning a model/startup response.

## Solution Assessment

No, not merge-ready yet. Returning `null` for non-routed bare resets is the narrow code direction, but this branch also needs the stale e2e/helper/docs contract updated and the empty-provider-call guard preserved.
