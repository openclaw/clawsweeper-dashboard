## Summary

Keep open. The PR targets a real source-reproducible regression and includes acceptable real Gateway proof, but it is not merge-ready because higher-level trigger coverage and public docs still assert the canned no-model acknowledgement contract that this branch removes.

## What This Changes

The PR changes bare non-ACP `/new` and `/reset` to fall through after reset hooks unless a hook routed its own reply, and updates reset-hook unit expectations.

## Best Possible Solution

Restore bare reset/new fallthrough through the existing startup prompt path, keep hook-routed replies terminal and empty-input provider errors covered, and align docs plus tests with that contract.

## Reproduction Assessment

Yes, source-level: current main returns `shouldContinue: false` for bare `/new` and `/reset`, and the reply pipeline stops on that result before the existing bare-session-reset prompt path can run. I did not run a live channel reproduction in this read-only review.

No, not merge-ready yet. Returning `null` for non-routed bare resets is the narrow code direction for restoring startup/model fallthrough, but the PR must update the stale e2e/helper/docs contract and keep the empty-provider-call guard covered.
