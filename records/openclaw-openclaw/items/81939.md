## Summary

Keep open. The code change is the right narrow direction for the reported regression, but this PR is not merge-ready because current trigger e2e/helper coverage and user docs still assert the opposite no-model acknowledgement behavior.

## What This Changes

The branch makes bare `/new` and `/reset` fall through to the model/startup path after reset hooks unless a hook routed its own reply, and updates the focused reset-hook unit expectations.

## Best Possible Solution

Finish this PR by aligning the trigger e2e/helper coverage and user docs with restored bare reset startup/model fallthrough while keeping hook-routed replies and ACP resets terminal.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: bare `/new` or `/reset` enters `maybeHandleResetCommand`, returns a terminal canned acknowledgement, and never reaches `runPreparedReply`'s bare-session-reset startup path; I did not run tests because this review is read-only.

## Solution Assessment

No, not merge-ready yet. Returning `null` for non-routed bare resets is the narrow fix direction, but the PR also needs to update the e2e/helper expectations and docs that still encode the no-model acknowledgement contract.
