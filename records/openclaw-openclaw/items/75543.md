## Summary

Keep this PR open, but do not merge it as written. The linked config writer bug is real and narrow, but the proposed no-op branches break the existing return contract, miss key-order-stable semantic comparisons, and need the required user-facing changelog entry.

## What This Changes

The PR adds early no-op returns in `src/config/io.ts` for byte-identical or volatile-meta-only config writes and adds regression coverage in `src/config/io.write-config.test.ts`.

## Best Possible Solution

Land this as a narrow config writer guard that returns the full persisted result shape, compares stable canonical config content after removing only volatile metadata, preserves audit/backup/suspicious-write behavior for real changes, and records the user-facing fix in the changelog.
