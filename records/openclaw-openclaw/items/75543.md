## Summary

Keep this PR open, but do not merge it unchanged. Current main still lacks a writer-level no-op path for stable config writes, and the PR targets the right boundary, but the proposed diff breaks the writer return contract, misses key-order-insensitive semantic no-ops, and lacks the required user-facing changelog entry.

## What This Changes

The PR adds early no-op returns in `src/config/io.ts` for byte-identical or volatile-meta-only config writes and adds a regression test in `src/config/io.write-config.test.ts`.

## Best Possible Solution

Land this as a narrow writer-boundary guard that returns the full persisted result shape, compares stable canonical config content after removing only volatile metadata, preserves audit/backup/suspicious-write behavior for real changes, and documents the user-facing fix in the changelog.

## Solution Assessment

No. The direction is appropriate, but this implementation is not the best mergeable form because it omits `persistedConfig`, uses insertion-order JSON for the semantic comparison, and misses the required changelog entry.
