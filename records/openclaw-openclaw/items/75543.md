## Summary

Keep this PR open, but do not merge it as written: it targets a real writer-level no-op bug, yet the patch breaks the `writeConfigFile` return contract, performs an incomplete semantic comparison, and lacks the required changelog entry.

## What This Changes

The PR adds no-op early returns to `src/config/io.ts` for byte-identical or volatile-meta-only config writes and adds a regression test in `src/config/io.write-config.test.ts`.

## Best Possible Solution

Land a narrow writer-boundary guard that returns the full persisted result shape, compares stable canonical config content after removing only volatile metadata, preserves real-write audit/backup/suspicious-write behavior, and documents the user-facing fix.

## Reproduction Assessment

Yes. A focused reproduction can call `createConfigIO().writeConfigFile` twice with the same stable config and observe current `main` continue into writer side effects; the PR's return-contract break is also visible from the declared return type and diff.

## Solution Assessment

No. The writer-boundary direction is appropriate, but this implementation is not the best mergeable solution until it returns `persistedConfig`, uses canonical JSON5-aware stable comparison, and adds the required changelog entry.
