## Summary

Current main already preserves non-overlapping keyword hits: hybrid merging keeps them as independent results with their text scores, and search selection now relaxes the weighted threshold for lexically relevant keyword candidates when strict hybrid results are empty. This directly addresses the v2026.6.5 symptom without incorrectly assigning keyword evidence to a different vector chunk.

## What This Changes

Fix hybrid memory retrieval so keyword evidence remains effective when keyword and vector searches return different chunk IDs.

## Best Possible Solution

Keep exact chunk identity and the current modality-aware selection behavior; ship that implementation rather than blending keyword scores into unrelated vector chunks.

## Reproduction Assessment

No, not on current main: keyword-only chunks retain textScore, and a focused regression test verifies they survive the weighted minScore case described by the report. The report remains credible for the older v2026.6.5 release.

## Solution Assessment

No, the proposed relaxed matching or score blending is not the best fix because it could associate evidence from different chunks. Current main uses the narrower solution: preserve keyword-only candidates while keeping exact chunk identity.
