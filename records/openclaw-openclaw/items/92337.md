## Summary

Keep open: current `main` and the latest release still short-circuit to strict hybrid results when any merged row meets `minScore`, so keyword-only FTS hits below the weighted threshold can still be dropped. The issue is also paired with open closing PRs, so it should stay open until one corrected implementation lands.

## What This Changes

The issue reports that hybrid memory search loses keyword contribution when keyword and vector result chunk IDs do not overlap.

## Best Possible Solution

Land one canonical manager-level filtering fix that keeps current keyword-backed fallback semantics and additionally preserves keyword-only hybrid hits beside strict vector results.

## Reproduction Assessment

Yes, at source level. `MemoryIndexManager.search` returns only strict merged rows whenever any row reaches `minScore`, which excludes keyword-only merged rows below the weighted threshold even though they carry `textScore`.

## Solution Assessment

Partly. The symptom is real, but the best fix is at the manager selection boundary because `mergeHybridResults` already unions non-overlapping IDs; changing chunk identity matching would be a broader and riskier path.
