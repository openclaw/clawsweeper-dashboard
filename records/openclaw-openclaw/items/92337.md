## Label Justifications

- `P2`: The defect degrades hybrid memory retrieval for valid lexical matches, but vector retrieval remains operational and the blast radius is limited to result relevance.
- `impact:other`: Valid memory-search matches can be suppressed from the final result set, reducing retrieval quality without fitting a more specific owned impact category.

## Summary

Current main still suppresses valid keyword-only hybrid matches when another result clears the strict threshold. The issue should remain open because a focused closing implementation is active at https://github.com/openclaw/openclaw/pull/92357; the separate https://github.com/openclaw/openclaw/pull/92341 addresses complementary keyword-query planning behavior.

## What This Changes

Preserve keyword-only hybrid memory matches through final result selection when vector and keyword chunk identifiers do not overlap.

## Best Possible Solution

Keep the union merge unchanged and repair final hybrid selection with a bounded lexical fallback that retains non-overlapping keyword matches while preserving deterministic ranking, deduplication, `maxResults`, and explicit minimum-score behavior; cover the mixed passing-vector plus keyword-only case with a regression test.

## Reproduction Assessment

Yes, from current-main source with high confidence: provide one merged vector result above `minScore` and one non-overlapping keyword-only result below its weighted threshold, and the early return at `extensions/memory-core/src/memory/manager.ts:817` omits the lexical result. This read-only review did not run a live Ollama/CJK environment.

## Solution Assessment

Partly: correcting final selection is the narrowest maintainable boundary, but the linked PR should explicitly prove that retaining below-threshold keyword-only rows is a bounded lexical fallback and does not unintentionally nullify configured threshold semantics.
