## Summary

Keep open: current main and the latest release still short-circuit hybrid memory results when any strict vector result exists, and two open closing PRs already own the repair path rather than making this issue ready for cleanup close.

## What This Changes

The issue asks memory-core hybrid search to preserve keyword-only FTS relevance when keyword and vector chunk IDs do not overlap.

## Best Possible Solution

Land one corrected, focused PR that preserves exact-ID hybrid merge semantics, keeps the existing relaxed keyword fallback and MMR ordering, and adds regression coverage for keyword-only hybrid hits; leave the separate CJK LIKE fallback tracked by https://github.com/openclaw/openclaw/issues/92728 unless maintainers intentionally fold it in.

## Reproduction Assessment

Yes at source level: construct a hybrid search where one vector result passes minScore and a keyword-only FTS result scores below minScore after text weighting, and current main returns only the strict vector result. I did not run a live Ollama-backed CJK memory_search scenario in this read-only review.

## Solution Assessment

Yes on the requested outcome, but the best fix is narrower than the issue title suggests: extend the manager filtering boundary to retain keyword-only hits without changing unrelated CJK term fallback or hybrid merge identity semantics.
