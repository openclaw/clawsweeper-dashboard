## Summary

Keep open: current main and the latest release still return only strict hybrid results when any vector result passes minScore, so keyword-only FTS hits can still be dropped. The active fix path is already represented by open closing PRs, especially https://github.com/openclaw/openclaw/pull/92524; https://github.com/openclaw/openclaw/issues/92728 separately tracks the broader CJK LIKE fallback behavior.

## What This Changes

The issue reports that hybrid memory search drops keyword-only text relevance when keyword and vector chunks do not overlap under hybrid scoring.

## Best Possible Solution

Land a focused manager-level fix that preserves keyword-only FTS hits alongside strict vector results, while leaving the separate multi-character CJK LIKE fallback behavior to https://github.com/openclaw/openclaw/issues/92728.

## Reproduction Assessment

Yes. From current source, keyword-only merged results can carry textScore but be filtered out when any vector result already passes minScore; I did not run the live Ollama/CJK setup in this read-only review.

## Solution Assessment

Yes for the current direction. The narrowest maintainable fix is at the manager filtering boundary because mergeHybridResults already preserves non-overlapping keyword entries with textScore; the active PR still needs normal review and landing.
