## Summary

Current main and v2026.6.5 already implement the requested non-overlap behavior: hybrid results are unioned by chunk ID, keyword-only chunks retain their text score, and the caller preserves FTS matches when weighting would otherwise leave no result above minScore.

## What This Changes

Ensure keyword search results remain effective in hybrid memory search when their chunk IDs do not overlap vector results.

## Best Possible Solution

Keep the existing exact-ID union and FTS-preservation fallback; investigate the separate CJK query-planning path through the linked PR rather than transferring scores between distinct chunks.

## Reproduction Assessment

No. The v2026.6.5 and current-main source paths retain non-overlapping keyword results, and the shipped focused test verifies their text score and weighted score.

## Solution Assessment

Yes. Exact-ID union plus a narrow fallback for otherwise-empty weighted results preserves lexical relevance without incorrectly assigning keyword relevance to a different vector chunk.
