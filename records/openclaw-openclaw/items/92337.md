## Summary

Keep open: current `main` and `v2026.6.6` still return only strict hybrid rows when any result passes `minScore`, so keyword-only FTS hits below the weighted threshold remain dropped; the narrow closing path is the open proof-supplied PR at https://github.com/openclaw/openclaw/pull/92524.

## What This Changes

The issue reports that hybrid memory search drops or underweights keyword-only FTS hits when keyword and vector result chunk IDs do not overlap.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92524 or an equivalent narrow manager-boundary fix, then close this issue while keeping the separate CJK LIKE fallback work in https://github.com/openclaw/openclaw/issues/92728 distinct.

## Reproduction Assessment

Yes, at source level. Current main's strict-result early return drops non-overlapping keyword-only FTS hits below `minScore`, and the issue plus linked PR proof give a concrete CJK/trigram memory-search scenario.

## Solution Assessment

Yes for the narrow candidate path. The best fix is at the manager selection boundary because `mergeHybridResults` already unions keyword-only rows; path/line score blending is broader and should not be required for this issue.
