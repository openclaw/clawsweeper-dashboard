## Summary

Keep this open: current main and the latest release still have the mixed strict-result early return that can drop non-overlapping keyword-only FTS hits below `minScore`, and https://github.com/openclaw/openclaw/pull/92524 is the active targeted fix candidate.

## What This Changes

The issue reports that hybrid memory search drops keyword-only FTS hits below `minScore` when vector and keyword chunk IDs do not overlap, so keyword recall does not survive into final results.

## Best Possible Solution

Review and land the targeted manager-boundary fix in https://github.com/openclaw/openclaw/pull/92524, then close this issue after that fix reaches main.

## Reproduction Assessment

Yes at source level. Current main returns only strict merged hybrid rows whenever any row meets `minScore`, so a lower-scored keyword-only FTS row created by `mergeHybridResults` is dropped without needing a live Ollama setup to see the control flow.

## Solution Assessment

Yes. The narrowest maintainable solution is at the manager selection boundary because `mergeHybridResults` already unions non-overlapping keyword hits; https://github.com/openclaw/openclaw/pull/92524 follows that shape.
