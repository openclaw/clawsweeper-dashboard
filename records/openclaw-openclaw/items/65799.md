## Summary

Keep open. Current main and v2026.5.7 still make Feishu WebSocket inherit ambient HTTP(S) proxy env, and the open implementation PR at https://github.com/openclaw/openclaw/pull/65802 still references this issue with closing syntax but has not merged.

## What This Changes

Request that Feishu WebSocket default to direct connection instead of ambient HTTP(S) proxy inheritance, with an explicit proxy opt-in for operators who need it.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/65802, or a replacement, after Feishu-owned proxy-policy docs, test cleanup, and the env-vs-config decision are resolved.

## Reproduction Assessment

Yes, at source level. Setting HTTP_PROXY or HTTPS_PROXY and calling createFeishuWSClient() currently resolves an ambient proxy agent and passes it into Lark.WSClient; I did not run a live Feishu gateway in this read-only review.

## Solution Assessment

Unclear as submitted. The Feishu-local opt-in gate is the narrow code path, but maintainers still need to settle env-only versus config-based proxy policy and ensure the open PR includes docs, cleanup, and proof.
