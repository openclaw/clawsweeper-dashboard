## Summary

Keep open: current main and the latest release still inherit ambient proxy settings for Feishu WebSocket, and the open linked PR owns the proposed direct-by-default fix.

## What This Changes

The issue asks Feishu WebSocket to bypass ambient proxy environment variables by default and require explicit proxy opt-in for users who need it.

## Best Possible Solution

Keep this issue open and resolve it through https://github.com/openclaw/openclaw/pull/90324 or an explicit maintainer rejection of that proxy-default direction.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 both unconditionally resolve an ambient proxy agent for Feishu WebSocket creation when proxy env is configured, and the Lark SDK passes that agent into the underlying WebSocket transport. I did not run a live Feishu/proxy gateway scenario in this read-only review.

## Solution Assessment

Unclear as a product decision: direct-by-default is plausible and aligned with safe defaults, but the explicit opt-in env changes proxy/egress behavior and is already under review in https://github.com/openclaw/openclaw/pull/90324.
