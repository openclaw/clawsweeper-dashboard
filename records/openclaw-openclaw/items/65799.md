## Summary

Keep open. Current main and v2026.5.7 still make Feishu WebSocket inherit ambient HTTP(S) proxy env, and the open closing PR is the implementation candidate rather than a reason to close before merge.

## What This Changes

Request that Feishu WebSocket default to a direct connection instead of ambient proxy env, with explicit opt-in for proxy use.

## Best Possible Solution

Land the Feishu-plugin-local opt-in through https://github.com/openclaw/openclaw/pull/65802 or a replacement, with test env cleanup and user-visible docs/changelog for proxy-dependent deployments.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Current main can be source-reproduced by setting HTTP(S) proxy env and calling createFeishuWSClient(), which the current tests still assert creates a proxy agent.

## Solution Assessment

Is this the best way to solve the issue? Yes, with maintainer completion. A Feishu-owned opt-in gate is the narrow fix, but it should land through the open PR or a replacement after test cleanup, docs/changelog coverage, and proxy-policy review.
