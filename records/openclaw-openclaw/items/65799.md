## Label Justifications

- `P1`: Feishu inbound messages can stop arriving for users with inherited proxy env, which is a broken channel workflow affecting real users.
- `impact:message-loss`: The report says real Feishu inbound messages stop arriving while the account can still appear healthy.

## Summary

Keep open: current main and the latest local release tag still route Feishu WebSocket creation through ambient proxy resolution, and the same-author closing PR remains open and unmerged.

## What This Changes

Request Feishu WebSocket connections to default to direct routing instead of inheriting ambient HTTP(S)/SOCKS proxy environment variables, with any proxy use made explicit.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/65802 with a Feishu-owned default-direct WebSocket proxy fix, documented proxy opt-in behavior, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Setting ambient proxy env before `createFeishuWSClient()` follows `getWsProxyAgent()` to `resolveAmbientNodeProxyAgent()` and passes the resulting agent into the Lark WSClient; I did not run a live Feishu gateway.

## Solution Assessment

Yes for the default-direct direction, but the final opt-in surface still needs maintainer judgment. A Feishu-local gate around `getWsProxyAgent()` is the narrow code path, with docs/tests needed for intentional proxy routing.
