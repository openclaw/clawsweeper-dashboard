## Summary

Keep open. Current main and v2026.5.12 still route Feishu WebSocket creation through ambient proxy resolution, and the open closing PR at https://github.com/openclaw/openclaw/pull/65802 should be reviewed, rebased, or replaced before this issue is closed.

## What This Changes

Request to make Feishu WebSocket skip ambient HTTP(S)/SOCKS proxy environment by default and require an explicit opt-in for proxy use.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/65802 with a Feishu-owned default-direct WebSocket proxy fix, documented opt-in behavior, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Setting HTTP_PROXY or HTTPS_PROXY before createFeishuWSClient() follows getWsProxyAgent() to resolveAmbientNodeProxyAgent() and passes an agent into the Lark WSClient; I did not run a live Feishu gateway.

## Solution Assessment

Unclear as a final merge path. A Feishu-local default-direct gate is the narrow code change, but maintainers still need to settle the opt-in surface and document how this interacts with intentional proxy routing.
