## Summary

Keep open: current main and the latest release still inherit ambient proxy settings for Feishu WebSocket, and an open same-repo PR already targets this issue with a closing reference. The issue should stay open until that PR lands or maintainers reject the proxy-default direction.

## What This Changes

Request Feishu WebSocket to avoid ambient HTTP(S)/SOCKS proxy environment variables by default and use an explicit opt-in for proxy routing.

## Best Possible Solution

Finish review of the linked Feishu WebSocket proxy PR, including the direct-by-default policy and opt-in documentation/proof, then close this issue after the accepted fix reaches main.

## Reproduction Assessment

Yes. Current main source shows Feishu always resolves an ambient Node proxy agent before constructing the WS client, and the pinned Lark SDK passes that agent into the WebSocket transport; I did not run a live credentialed Feishu connection in this read-only review.

## Solution Assessment

Unclear as a final product contract. Direct-by-default with explicit proxy opt-in is a narrow owner-boundary fix, but the new env surface and egress-default change need maintainer acceptance through the linked PR.
