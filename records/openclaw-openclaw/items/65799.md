## Summary

Keep open. Current main and the latest release still route Feishu WebSocket creation through ambient proxy resolution, and https://github.com/openclaw/openclaw/pull/65802 remains an open closing PR that should be reviewed, rebased, or replaced rather than closing this issue early.

## What This Changes

The issue requests that Feishu WebSocket stop inheriting ambient proxy environment variables by default and use an explicit opt-in for proxy routing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/65802, or a rebased replacement, after settling the Feishu proxy opt-in surface and documenting the operator-visible behavior.

## Reproduction Assessment

Yes, at source level. Setting HTTP_PROXY or HTTPS_PROXY before createFeishuWSClient() follows getWsProxyAgent() to resolveAmbientNodeProxyAgent() and passes an agent into Lark WSClient; I did not run a live Feishu gateway.

## Solution Assessment

Unclear as a final merge path. A Feishu-local default-direct gate is the narrow code change, but maintainers still need to settle env-only versus config-backed opt-in and preserve the managed proxy contract.
