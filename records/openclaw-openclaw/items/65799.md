## Summary

Keep open: current main and the latest release still inherit ambient proxy settings for Feishu WebSocket, and https://github.com/openclaw/openclaw/pull/90324 is an open closing-reference fix candidate. ## What This Changes

The issue asks for Feishu WebSocket to stop inheriting process HTTP(S)/SOCKS proxy environment variables by default, with an explicit proxy opt-in for deployments that need it.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/90324 with explicit Feishu proxy-default, documentation, and proof decisions; close this issue only after the accepted path reaches main.

## Reproduction Assessment

Yes. Current main source shows Feishu always resolves an ambient Node proxy agent before constructing the WS client, current tests expect proxy env to attach an agent, and the pinned Lark SDK forwards that agent to the WebSocket transport; I did not run a live credentialed Feishu/proxy setup.

## Solution Assessment

Unclear as a final product contract. Direct-by-default with explicit proxy opt-in is a narrow owner-boundary fix, but the new env surface and egress-default change need maintainer acceptance through the linked PR.
