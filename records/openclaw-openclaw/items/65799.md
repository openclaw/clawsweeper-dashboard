## Summary

Keep open. Current main and latest shipped release still route Feishu WebSocket creation through ambient proxy resolution, and the open closing PR should be reviewed, rebased, or replaced before this issue is closed.

## What This Changes

The issue asks Feishu WebSocket connections to stop inheriting ambient proxy environment variables by default and use an explicit opt-in for proxy routing.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/65802 with a Feishu-owned default-direct WebSocket proxy fix, explicit opt-in policy, docs, and focused regression coverage.

## Solution Assessment

Unclear as a final issue resolution until the linked PR is refreshed and maintainers settle the opt-in surface. A Feishu-local default-direct gate is the narrow code path, but the docs and managed-proxy policy need to be explicit.
