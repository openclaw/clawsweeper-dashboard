## Summary

Keep this issue open: current `main` and the latest release still route Feishu WebSocket construction through ambient proxy resolution, and https://github.com/openclaw/openclaw/pull/90324 is the active open fix candidate.

## What This Changes

The issue asks Feishu WebSocket connections to bypass ambient proxy environment variables by default and use proxy routing only behind an explicit opt-in.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/90324; once the chosen Feishu proxy default reaches `main`, close this issue against that implementation.

## Reproduction Assessment

Yes, source-reproducible: current `main` always resolves an ambient Node proxy agent before constructing the Feishu WS client, and the Lark SDK passes that agent into the WebSocket transport. I did not run a live Feishu app credential flow in this read-only review.

## Solution Assessment

Likely yes, pending maintainer approval: direct-by-default Feishu WS with an explicit proxy opt-in is the narrow owner-boundary fix for this bug. The opt-in env var and egress-default change still need product/security acceptance before the linked PR lands.
