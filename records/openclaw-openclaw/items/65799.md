## Summary

Keep open: current main and the latest release still attach an ambient proxy agent to Feishu WebSocket when proxy env is present, and https://github.com/openclaw/openclaw/pull/90324 is the active open fix candidate with closing syntax.

## What This Changes

Request to make Feishu WebSocket bypass ambient proxy environment variables by default and use proxy routing only behind an explicit opt-in.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/90324 with an explicit Feishu WebSocket egress policy, then keep broader HTTP and mixed-proxy policy work on the adjacent issues.

## Reproduction Assessment

Yes, at source level. Current main unconditionally resolves an ambient Node proxy agent before constructing the Feishu WebSocket client, the inbound monitor path uses that client, and the pinned Lark SDK forwards the agent to the WebSocket transport.

## Solution Assessment

Unclear as a final product contract. Direct-by-default with explicit proxy opt-in is a narrow Feishu-owned implementation, but the egress default and new env surface still need maintainer review in the linked PR.
