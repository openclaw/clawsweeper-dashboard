## Summary

Keep open. Current main still makes Feishu WebSocket inherit ambient HTTP(S) proxy environment, and the linked implementation PR #65802 closes this issue but remains open and unmerged.

## What This Changes

The issue requests changing Feishu WebSocket setup so ambient HTTP(S)/SOCKS proxy env is ignored by default and used only behind an explicit opt-in.

## Best Possible Solution

Review and land PR #65802 after its PR-level blockers are addressed, or land an equivalent Feishu-plugin-local opt-in change with tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Current main can be reproduced from source by setting `HTTP_PROXY` or `HTTPS_PROXY` and calling `createFeishuWSClient()`, which currently creates and passes a proxy agent; the reporter’s live notes cover the broken ambient proxy failure mode.

## Solution Assessment

Yes. The narrowest maintainable solution is to gate Feishu WebSocket proxy-agent construction behind a Feishu-specific opt-in while leaving the shared ambient proxy helper available for transports that intentionally inherit process proxy settings.
