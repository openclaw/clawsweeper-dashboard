## Summary

Keep open. Current main still routes Feishu WebSocket creation through ambient proxy-agent resolution, and the issue has an open closing implementation PR that should be reviewed, rebased, or replaced before closure.

## What This Changes

The issue requests that Feishu WebSocket connections stop inheriting ambient HTTP(S)/SOCKS proxy environment variables by default, with an explicit opt-in for users who need proxy routing.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/65802 with a Feishu-owned default-direct WebSocket proxy fix, explicit opt-in behavior, docs, and focused regression coverage.

Yes, at source level. Setting HTTP(S) proxy env before `createFeishuWSClient()` follows `getWsProxyAgent()` to `resolveAmbientNodeProxyAgent()` and passes an agent into Lark `WSClient`; I did not run a live Feishu gateway.

## Solution Assessment

Unclear as a final merge path. A Feishu-local default-direct gate is narrow, but maintainers still need to settle the opt-in surface and document interaction with intentional or managed proxy routing.
