## Summary

Keep open. Current main and v2026.5.7 still make Feishu WebSocket inherit ambient proxy env, while https://github.com/openclaw/openclaw/pull/65802 is still open, conflicted, and already linked as the implementation candidate.

## What This Changes

The issue requests that Feishu WebSocket avoid ambient HTTP(S)/SOCKS proxy env by default, with an explicit proxy opt-in for deployments that need it.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/65802 or a replacement after resolving test env cleanup, docs/changelog coverage, and the env-only versus config-flag policy question.

## Reproduction Assessment

Yes. Current main can be source-reproduced by setting HTTP(S) proxy env and calling createFeishuWSClient(), and the current tests still assert that this creates a proxy agent.

## Solution Assessment

Yes, with maintainer completion. The Feishu-owned opt-in gate is the narrow code path, but the open PR still needs cleanup, documentation, and a decision on whether the escape hatch should be env-only or config-backed.
