## Summary

Keep open: current `main` and `v2026.5.12` still raw-scan identified script files for all-caps `$VAR` tokens without string/comment context, so benign Python literals such as `$SIM` can still be rejected outside the now-skipped gateway full/off path.

## What This Changes

The issue reports that exec script preflight rejects Python files containing uppercase dollar tokens such as `$SIM` inside benign string literals.

## Best Possible Solution

Implement context-aware script preflight so benign Python strings, comments, and help text are ignored while real executable-source `$VAR` leakage remains blocked.

## Reproduction Assessment

Yes, by source inspection: direct Python script commands are routed into file validation, and uppercase `$SIM` inside a string literal would still match the raw content regex. The original gateway full/off path is now skipped, so the remaining current-main repro surface is approval-mode script preflight.

## Solution Assessment

Unclear without a current patch. A lexer/AST or execution-context-aware change with regression tests is safer than a broad string skip that might weaken the shell-bleed guard.
