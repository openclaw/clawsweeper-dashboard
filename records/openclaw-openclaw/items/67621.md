## Summary

Keep open: current main still raw-scans Python/Node script content for all-caps `$VAR` tokens without string, docstring, comment, or help-text context; only the exact gateway `security=full` plus `ask=off` path is now skipped. The linked fix PRs are closed unmerged and no canonical open replacement specifically owns the remaining approval-mode false positive.

## What This Changes

The issue reports that exec script preflight rejects benign `$VAR`-shaped text inside Python string literals as shell variable injection.

## Best Possible Solution

Implement a language-aware script preflight detector that ignores benign Python string, docstring, comment, and help-text occurrences while preserving rejection of executable-source shell-variable leakage.

## Reproduction Assessment

Yes for active approval-mode preflight: source inspection shows a direct `python bad.py` target whose file contains uppercase `$SIM` in a string literal reaches the raw regex and throws. The exact gateway `security=full` plus `ask=off` path is now skipped, so the current reproducible surface is narrower than the original report.

## Solution Assessment

Unclear. The requested direction is correct, but the safest implementation should be a lexer/AST or execution-context-aware detector with negative coverage, not a broad regex skip that could hide real source leakage.
