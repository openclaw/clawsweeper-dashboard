## Summary

Keep open: current main still applies the all-caps `$VAR` regex to raw Python/Node script contents in preflight-enabled modes, with no string/docstring/comment context check. The exact gateway `security=full` plus `ask=off` path now skips this layer, but the underlying scanner remains false-positive prone and the linked fixes are closed unmerged.

## What This Changes

Request context-aware exec script preflight so benign Python string, docstring, argparse help, and currency `$VAR` text is not treated as shell-variable injection.

## Best Possible Solution

End state should be a context-aware script preflight detector that ignores benign Python/Node strings, docstrings, comments, and help text while still rejecting executable-source shell-variable leakage, backed by regression tests and a changelog entry if behavior changes for users.

## Reproduction Assessment

Yes. In a preflight-enabled mode such as gateway `security=full` with `ask=on-miss`, a direct `python bad.py` whose file contains an uppercase `$SIM` token in a string literal will match the raw regex and throw before literal context is considered; the gateway `full/off` path now skips preflight.

## Solution Assessment

Unclear. The requested direction is correct, but the exact lexer, AST, or execution-context boundary is a security-sensitive policy choice and should be chosen deliberately rather than by broad string skipping.
