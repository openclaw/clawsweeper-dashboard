## Summary

Keep open. Current main still lacks the host-mediated `sendSessionAttachment` SDK surface, and this PR is the active implementation slice, but the latest head still has a route-selection bug in normalized session lookup that can hide an older routable alias behind a newer non-routable record.

## What This Changes

This PR adds a bundled-plugin `sendSessionAttachment` SDK API, host-side local-file validation and outbound delivery routing, Telegram/Slack attachment hints, Telegram formatting helper exports, focused tests, API baseline hashes, and a changelog entry.

## Best Possible Solution

Land the additive bundled-plugin attachment seam after making every alias-resolution path prefer routable delivery contexts before freshness, while preserving the direct routable fast path and adding the inverse mixed-case alias regression.

## Reproduction Assessment

Yes. On the PR head, use a session store with a newer non-routable normalized/canonical entry and an older mixed-case alias for the same normalized key that has `deliveryContext`; the normalized index picks the newer non-routable entry and attachment delivery can miss the active route.

## Solution Assessment

No. The overall host-mediated seam is the right narrow direction, but the normalized index must rank routable entries before freshness, not only after a candidate has already survived the index lookup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Prefer routable aliases in the normalized index:** `src/config/sessions/delivery-info.ts:137-138`
  - body: `buildFreshestSessionEntryIndex` keeps only the newest entry for each normalized key, so when direct candidates are absent or non-routable, a newer non-routable record can hide an older mixed-case alias that has a valid `deliveryContext`. That makes `extractDeliveryInfo` return no active route even though the store has one; rank routable entries before freshness when building this index and add the inverse alias regression.
  - confidence: 0.83
