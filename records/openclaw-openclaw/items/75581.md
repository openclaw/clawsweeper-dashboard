## Summary

Keep open: the split attachment seam is useful and most earlier review feedback appears addressed, but the PR still introduces a Telegram chunking regression in the default Markdown path when newline chunking is enabled.

## What This Changes

The PR adds a bundled-only Plugin SDK `sendSessionAttachment` host hook with session route lookup, file validation, Telegram/Slack delivery hints, gateway send-option plumbing, docs, generated Swift models, and regression tests.

## Best Possible Solution

Land the host-mediated attachment seam after fixing Telegram chunk planning so explicit HTML captions use HTML-safe splitting without weakening the existing Markdown-aware default path.

## Reproduction Assessment

Yes for the review finding: source inspection shows Telegram now advertises `chunkerMode: "text"`, and the outbound planner routes newline mode through plain paragraph splitting unless the mode is `markdown`. A focused regression test should cover a long fenced Markdown Telegram message with newline chunking enabled.

## Solution Assessment

No: the host-mediated seam is directionally sound, but changing Telegram's adapter-wide chunker mode is not the best fix for HTML captions because it regresses default Markdown chunk planning. A safer solution is to keep Markdown-aware planning for normal sends while selecting HTML-safe chunking only when `formatting.parseMode === "HTML"`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve Markdown-aware newline chunking:** `extensions/telegram/src/outbound-base.ts:13`
  - body: This changes Telegram's `chunkerMode` to `text` for every send. In newline mode the shared planner only calls `chunkMarkdownTextWithMode` when the adapter mode is `markdown`; with `text` it pre-splits through `chunkByParagraph`, whose long-paragraph fallback uses plain `chunkText`. Default Telegram Markdown messages can therefore be split inside fenced code/Markdown before the Markdown chunker sees the whole block, regressing the existing non-HTML path. Please keep Markdown-aware planning for normal sends and make the HTML behavior context-specific.
  - confidence: 0.86
