## Summary

Keep open. Current main does not contain the proposed session-attachment SDK seam, and this PR is a fresh replacement for closed predecessor stack work rather than obsolete cleanup; however, the proposed public API still has an ignored-hints contract bug and is missing the required changelog entry.

## What This Changes

Adds `OpenClawPluginApi.sendSessionAttachment`, bundled-plugin host validation/routing for session file attachments, attachment hint types, Telegram parse-mode/helper exports, focused tests, and SDK API baseline updates.

## Best Possible Solution

Keep the PR open, resolve the SDK hint contract before readiness by either wiring every exposed hint through a real outbound/channel contract or removing unsupported fields from this slice, add the required Unreleased changelog entry, then validate the focused plugin SDK and Telegram lanes before coordinating with the docs split.

## Reproduction Assessment

Yes for the review findings: the ignored-hints issue is reproducible by diff/source inspection because the PR accepts fields that are never forwarded to outbound delivery, and the changelog gap is reproducible by searching `CHANGELOG.md`. For the feature itself, current main has no equivalent `sendSessionAttachment` API.

## Solution Assessment

No, not as currently written. The host-mediated attachment seam is the right general boundary, but the public SDK surface should only accept behavior it can deliver, and the user-facing API addition needs release notes before merge.
