## Summary

Keep open. Current main does not contain the requested host-mediated session attachment SDK seam, and the GitHub context shows this is a fresh split from the closed predecessor stack rather than obsolete cleanup. The PR is still draft and the current patch exposes public channel hint fields that are not delivered, so it needs API-owner resolution plus the required changelog entry before merge.

## What This Changes

The PR adds `OpenClawPluginApi.sendSessionAttachment`, host-side session file validation and outbound delivery, SDK exports/baseline updates, Telegram formatting helper exports, and focused plugin SDK/Telegram tests.

## Best Possible Solution

Keep the host-mediated attachment seam as a focused SDK addition, but make the public params match delivered behavior: either wire each advertised hint through a real outbound/channel contract or remove/defer unsupported hints, add the Unreleased changelog entry, then validate the focused plugin SDK and Telegram lanes before coordinating with the docs split.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head shows the ignored hint fields are accepted and resolved but not forwarded to outbound delivery, and `CHANGELOG.md` lacks a session attachment entry. For the feature request itself, current main has no equivalent `sendSessionAttachment` API.

## Solution Assessment

No, not as currently written. The host-mediated attachment boundary is the right maintainable direction, but a public SDK method should not accept no-op channel hints, and the user-facing SDK change needs release notes before merge.
