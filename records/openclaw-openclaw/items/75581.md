## Summary

Keep open. Current main does not have the requested `sendSessionAttachment` SDK seam, and this PR is the active implementation slice, but the latest head still has a routable-alias selection bug in session delivery lookup that can make attachment delivery miss an existing route.

## What This Changes

This PR adds a bundled-plugin `sendSessionAttachment` SDK API with host-side local-file validation, active-session delivery routing, Telegram/Slack attachment hints, Telegram formatting exports, tests, API baseline updates, and a changelog entry.

## Best Possible Solution

Keep the additive bundled-plugin attachment seam, but update the normalized session-entry lookup so every alias path ranks routable delivery contexts before freshness, add the inverse mixed-case alias regression, then rebase and run the targeted SDK, session, Telegram, lint, and API checks before merge.

## Reproduction Assessment

Yes. On the PR head, use a store where the queried/canonical normalized key is missing or newer but non-routable, while an older mixed-case key with the same normalized value has `deliveryContext`; the normalized index hides the routable alias and attachment delivery can report no active route.

## Solution Assessment

No. The overall host-mediated seam is the right narrow direction, but the current lookup implementation is not the best fix until the normalized index preserves routable aliases before comparing `updatedAt`.
