## Summary

Keep open. The proposed SDK seam is still absent from current main and the latest PR head addresses the earlier attachment-routing review feedback, but the patch is not merge-ready because it leaves Gateway protocol generated clients stale and introduces an eager registry-to-outbound import path that is already reflected in exact-head CI failures.

## What This Changes

The PR adds Plugin SDK `sendSessionAttachment` support with host-side session-route attachment delivery, validation and delivery hardening, channel hints, Telegram formatting exports, Gateway send option plumbing, tests, API baseline updates, and a changelog entry.

## Best Possible Solution

Keep the additive bundled-plugin attachment seam, repair the generated Gateway protocol artifacts and lazy-load the outbound channel resolver to preserve plugin registry topology, then rerun the targeted protocol, import-cycle, SDK, session attachment, Telegram, lint, and diff checks before maintainer review.

## Reproduction Assessment

Yes. The merge blockers reproduce from source inspection plus exact-head check metadata: the PR head schema contains new `SendParams` fields while generated Swift models do not, and the new static import chain explains the failing import-cycle/architecture checks.

## Solution Assessment

No. The feature direction is the right narrow host-mediated seam, but this exact patch is not the best mergeable form until generated protocol clients are synchronized and outbound resolution stays lazy behind the attachment call path.
