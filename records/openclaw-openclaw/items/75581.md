## Summary

Keep open: the PR adds a useful host-mediated attachment seam, but the current diff still regresses Telegram Markdown chunk planning when newline chunking is enabled.

## What This Changes

The PR adds a bundled-only Plugin SDK `sendSessionAttachment` host hook with session-route validation, file guards, Telegram/Slack delivery hints, gateway send-option plumbing, docs, generated Swift models, and tests.

## Best Possible Solution

Land the attachment seam after preserving Markdown-aware Telegram chunk planning for normal sends while using HTML-safe chunking only for explicit HTML captions.

## Reproduction Assessment

Yes: source inspection shows the PR changes Telegram to `chunkerMode: "text"`, and the outbound planner pre-splits newline mode through `chunkByParagraph` unless the mode is `markdown`. A focused regression should send a long fenced Markdown Telegram message with newline chunking enabled.

## Solution Assessment

No: the attachment seam is directionally sound, but changing Telegram's adapter-wide chunker mode is not the narrowest fix. Keep Markdown-aware planning for normal sends and select HTML-safe splitting only when `formatting.parseMode === "HTML"`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve Markdown-aware newline chunking:** `extensions/telegram/src/outbound-base.ts:13`
  - body: This changes Telegram's `chunkerMode` to `text` for every send. Default Telegram Markdown messages can therefore be split inside fenced code or Markdown before the Markdown chunker can protect the block.
