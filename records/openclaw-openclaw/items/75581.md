## What This Changes

The PR adds a public `OpenClawPluginApi.sendSessionAttachment` API, host-side session attachment validation and delivery routing, delivery-context lookup hardening, Telegram formatting exports and parse-mode plumbing, and focused contract/API baseline coverage.

## Best Possible Solution

Keep the PR open, fix the attachment helper so failed or empty delivery results return `ok: false`, add the required Unreleased changelog bullet, and then run the focused session-attachment, Telegram API, Plugin SDK API, type, lint, and diff checks before review continues.

## Reproduction Assessment

Yes. The failure path is reproducible by having the mocked `sendMessage` resolve without a concrete `result`, matching current best-effort delivery behavior when all payload sends fail; the helper currently still returns `ok: true`.

## Solution Assessment

No. The overall SDK seam direction is reasonable, but the implementation should preserve delivery failure semantics by removing best-effort for this API or explicitly checking the returned delivery result before reporting success.

## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P2] Return an error when delivery produces no result:** `src/plugins/host-hook-workflow.ts:174-179`
