## Summary

Keep open. Current main lacks the requested SDK API and the PR is still useful, but the patch has a P2 Telegram adapter bug in the explicit HTML parse-mode path that should be fixed before merge.

## What This Changes

The PR adds bundled-plugin `sendSessionAttachment` support with host-side file validation, session route lookup, Telegram/Slack delivery hints, gateway send-option plumbing, generated client updates, tests, and changelog coverage.

## Best Possible Solution

Land the split seam after aligning the public Telegram outbound adapter's parse-mode-aware chunking with the base adapter and rerunning the targeted PR checks.

## Reproduction Assessment

Yes. The high-confidence path is static: `deliverOutboundPayloads` passes `formatting` into an adapter chunker before `sendText`, while PR head's exported `telegramOutbound` accepts HTML parse mode but still declares a markdown chunker.

## Solution Assessment

No. The host-mediated attachment design is appropriately narrow, but the proposed patch should also fix the public Telegram adapter's explicit-HTML chunking path before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Align HTML parsing and chunking:** `extensions/telegram/src/outbound-adapter.ts:177`
  - body: This branch sets `textMode: "html"` when `formatting.parseMode` is present, but the exported `telegramOutbound` adapter still advertises `markdownToTelegramHtmlChunks` with `chunkerMode: "markdown"`. The shared delivery path invokes the adapter chunker with the formatting context before calling `sendText`, so long explicit-HTML text through this public adapter can be markdown-rendered/escaped and then delivered as HTML. Mirror the HTML-aware chunker used by `outbound-base` here or avoid exposing HTML parse mode on this adapter.
  - confidence: 0.86
