## Summary

Keep open. Current main does not implement the host-mediated session attachment seam, and this draft PR is the active fresh-main slice replacing the closed consolidated predecessors, but the latest head still has a blocking caption-format bug for Telegram attachment captions.

## What This Changes

The PR adds `OpenClawPluginApi.sendSessionAttachment`, bundled-plugin file validation and session delivery routing, Telegram/Slack attachment hints, outbound parse-mode plumbing, Telegram formatting helper exports, API baseline/changelog updates, and focused tests.

## Best Possible Solution

Keep the focused host-mediated attachment seam, but make the caption-format contract match delivered behavior before merge: plain captions should be escaped or carried as true plain text, MarkdownV2 should be consistently honored, and focused SDK/Telegram tests should lock the behavior.

## Reproduction Assessment

Yes. A bundled plugin sending a Telegram session attachment with `captionFormat: "plain"` and caption text such as `1 < 2 & 3` follows a source-visible path where the host omits `parseMode`, then Telegram sends the caption through the HTML text mode.

## Solution Assessment

No, not as currently written. The seam is a maintainable direction, but exposing `plain` while delivering it through Telegram's trusted HTML path is not the narrowest safe public contract; escaping or an explicit plain-text delivery mode is safer.
