## Summary

Keep open. Current main does not implement `sendSessionAttachment`, and the latest PR head is still the active implementation slice, but the patch has a blocking correctness issue: it can report successful attachment delivery after best-effort outbound delivery swallowed the actual send failure.

## What This Changes

This PR adds bundled-plugin `OpenClawPluginApi.sendSessionAttachment`, host-side attachment validation and session-route lookup, Telegram/Slack attachment hints, Telegram formatting exports, outbound parse-mode plumbing, contract tests, API baseline hashes, and a changelog entry.

## Best Possible Solution

Keep the focused host-mediated attachment seam, but make the result contract match real delivery: return `{ ok: false }` when best-effort delivery produces no concrete send result or let attachment delivery errors propagate, then lock that behavior with a focused contract test before maintainer review continues.

## Reproduction Assessment

Yes. On the PR head, a mocked or real best-effort outbound failure can leave `sendMessage` with `result: undefined`, while `sendPluginSessionAttachment` still returns `ok: true` and the validated file count.

## Solution Assessment

No, not as currently written. The seam direction is maintainable and the plain-caption fix is present, but returning success after a swallowed delivery failure is not the narrowest reliable contract for plugin authors.

## Review Findings
