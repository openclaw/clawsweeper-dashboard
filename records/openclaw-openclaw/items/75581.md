## Summary

Keep open. Current main still lacks the requested host-mediated attachment SDK surface, and the PR is the active implementation slice, but the latest head still has two blocking contract issues: Telegram `MarkdownV2` is advertised without true Telegram MarkdownV2 delivery, and delivery route lookup can stop on a stale alias before checking the canonical session key.

## What This Changes

This PR adds bundled-plugin `sendSessionAttachment`, host-side attachment validation and session-route lookup, Telegram/Slack delivery hints, outbound formatting plumbing, Telegram formatting exports, focused tests, API baseline hashes, and a changelog entry.

## Best Possible Solution

Keep the focused host-mediated attachment seam, but repair the PR head so the public parse-mode contract matches actual Telegram delivery and route lookup prefers a routable canonical entry over stale aliases. After that, keep the bundled-only gating, validation limits, changelog entry, and focused contract coverage.

## Reproduction Assessment

Yes. The `MarkdownV2` mismatch is reproducible by sending a Telegram attachment with `channelHints.telegram.parseMode: "MarkdownV2"`: the PR maps it to `textMode: "markdown"`, but `sendMessageTelegram` sends `parse_mode: "HTML"`. The route lookup issue is reproducible with one store containing a stale `main` alias entry and a routable `agent:main:main` entry; lookup returns the alias before checking the canonical key.

## Solution Assessment

No, not as currently written. The seam direction is maintainable and the earlier best-effort result bug is fixed, but the PR needs narrow contract repairs before this is the safest additive SDK surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor or remove the MarkdownV2 parse-mode hint:** `src/plugins/host-hooks.ts:160`
  - body: The new attachment API advertises `telegram.parseMode: "MarkdownV2"`, but the Telegram adapters only convert that to `textMode: "markdown"`; the sender still renders to HTML and sends `parse_mode: "HTML"`. Plugins that pass pre-escaped MarkdownV2 captions will not get Telegram MarkdownV2 delivery, so either implement true MarkdownV2 pass-through or remove/rename this option before exposing it.
  - confidence: 0.88
- **[P2] Continue past stale alias session entries:** `src/config/sessions/delivery-info.ts:102-104`
  - body: `findSessionEntryInStore` returns the first candidate entry it finds, so a stale alias such as `main` can prevent lookup from ever checking the canonical key that follows it. If the alias lacks a routable delivery context but the canonical entry has one, attachment sends fail with `session has no active delivery route`; keep scanning candidate keys until a routable entry is found or rank routable entries first.
  - confidence: 0.84
