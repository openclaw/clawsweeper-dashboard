## Summary

Keep open: the branch still has a concrete Telegram outbound regression. The latest fix makes the Telegram chunker HTML-aware, but leaves the adapter-wide `chunkerMode` as `text`, so newline-mode planning can pre-split default Markdown Telegram messages before Markdown-aware chunking runs.

## What This Changes

The PR branch adds a bundled-only `sendSessionAttachment` Plugin SDK host hook with host-side attachment validation, session route lookup, Telegram/Slack delivery hints, gateway send-option plumbing, docs, generated Swift models, and regression tests.

## Best Possible Solution

Land the attachment seam only after making Telegram chunk planning context-aware so default Markdown sends retain Markdown-aware newline splitting and explicit HTML captions use HTML-safe chunks.

## Reproduction Assessment

Yes for the review finding: source inspection shows `chunkerMode: "text"` routes newline-mode planning through plain paragraph pre-splitting before the Telegram chunker runs. A focused regression should plan a long fenced Markdown Telegram message with newline chunking enabled and no explicit HTML parse mode.

## Solution Assessment

No: the host-mediated seam is directionally coherent, but the Telegram chunking change is not the narrowest safe fix. The safer repair is to preserve Markdown-aware planning for normal Telegram sends and select HTML-safe planning only when `formatting.parseMode === "HTML"`.
