## Summary

Keep open. Current main still lacks the host-mediated session attachment SDK seam, so this draft PR is the active fresh-main replacement rather than obsolete cleanup; however the patch has blocking SDK behavior issues around ignored channel hints and attachment delivery success reporting, plus a required changelog gap.

## What This Changes

The PR adds `OpenClawPluginApi.sendSessionAttachment`, host-side session file validation and delivery routing, SDK exports/API baseline updates, Telegram formatting helper exports, and focused plugin SDK/Telegram tests.

## Best Possible Solution

Keep the focused attachment seam, but make the public SDK params match real delivery behavior, preserve failure semantics for attachment sends, add the required Unreleased changelog entry, and then revalidate the targeted plugin SDK, Telegram, API baseline, type, lint, and diff checks before review continues.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head shows ignored hint fields, forced best-effort delivery with unconditional success reporting, and no `CHANGELOG.md` update. For the feature itself, current main has no equivalent `sendSessionAttachment` API to exercise.

## Solution Assessment

No, not as currently written. The host-mediated attachment boundary is a maintainable direction, but the public API should not accept no-op hints, and the helper should not report successful attachment delivery when the underlying best-effort send produced no result.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Remove or wire ignored attachment hints:** `src/plugins/host-hooks.ts:164-170`
  - body: The new public `PluginAttachmentChannelHints` type accepts Discord `ephemeral`/`suppressEmbeds` and Slack `unfurlLinks`, but `sendPluginSessionAttachment` only forwards thread, media, force-document, parse-mode, and silent options into `sendMessage`. SDK callers can receive `{ ok: true }` while those hints are ignored, so either plumb them through supported outbound contracts or remove/defer them from this slice.
  - confidence: 0.87
- **[P2] Report failed best-effort attachment sends:** `src/plugins/host-hook-workflow.ts:174-178`
  - body: `sendPluginSessionAttachment` forces `bestEffort: true`, but outbound delivery suppresses per-payload send errors in that mode and `sendMessage` can return without a concrete delivery result. The helper then returns `{ ok: true }`, so plugins can be told an attachment was delivered after all sends failed; remove best-effort here or check for a concrete result before reporting success.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/plugins/types.ts:2512-2515`
  - body: This adds a public `feat(plugin-sdk)` API method and Telegram API exports, but the PR head has no `CHANGELOG.md` update. OpenClaw policy requires a single-line Unreleased entry for user-facing SDK changes before merge.
  - confidence: 0.9
