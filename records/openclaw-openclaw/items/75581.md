## Summary

Keep open. Current main still lacks the requested Plugin SDK attachment API, and this PR is the active implementation slice, but the latest head still has a normalized session-route lookup bug that can hide an older routable alias behind a newer non-routable entry.

## What This Changes

The PR adds `OpenClawPluginApi.sendSessionAttachment` for bundled-plugin session attachments with host validation/routing, channel delivery hints, Telegram formatting exports, tests, API baseline updates, and a changelog entry.

## Best Possible Solution

Keep the additive bundled-plugin attachment seam, but fix normalized delivery-route lookup so routable entries always outrank non-routable freshness in every alias path, add the inverse mixed-case regression, then rebase and run the targeted SDK/session/Telegram/lint/API checks before merge.

## Reproduction Assessment

Yes. On the PR head, use a session store where the queried or canonical normalized key is absent or newer but non-routable, while an older mixed-case key with the same normalized value has `deliveryContext`; the normalized fallback can return the non-routable record and attachment delivery reports no active route.

## Solution Assessment

No. The overall host-mediated attachment seam is the right narrow direction, but the current normalized index is not the best implementation until it preserves routable aliases before comparing `updatedAt`.
