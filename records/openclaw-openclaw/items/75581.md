## Summary

Keep open. Current main still lacks the session-attachment SDK surface, and this PR is the active implementation slice, but the latest head still has a routable-alias selection edge case that can make attachment delivery report no active route even when a valid alias entry exists.

## What This Changes

This PR adds a bundled-plugin `sendSessionAttachment` SDK seam, host-side file validation and session-route lookup, Telegram/Slack attachment hints, Telegram parse-mode plumbing/helper exports, tests, API baseline hash updates, and a changelog entry.

## Best Possible Solution

Keep the additive bundled-plugin attachment seam, but make normalized session-entry lookup rank routable records before freshness across all aliases, then add the inverse alias regression and rerun the focused session/plugin/Telegram checks plus API and lint gates.

## Reproduction Assessment

Yes. On the PR head, create a session store where the queried canonical key is newer but lacks route metadata while an older mixed-case alias for the same normalized key has `deliveryContext`; `extractDeliveryInfo(canonicalKey)` can miss the routable alias because the normalized index stores only the freshest entry.

## Solution Assessment

No, not as currently written. The overall seam is the right narrow direction, but the route lookup should apply the same routability-before-freshness rule when building the normalized alias index, not only after a candidate is surfaced.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Prefer routable aliases in the normalized index:** `src/config/sessions/delivery-info.ts:128-129`
  - body: `buildFreshestSessionEntryIndex` stores only the newest entry for each normalized key, so a fresh non-routable canonical record can hide an older mixed-case alias that has a valid `deliveryContext`. In that case `extractDeliveryInfo(canonicalKey)` returns no route and `sendSessionAttachment` reports no active delivery route even though the store has one; rank routable entries before freshness when building this index.
  - confidence: 0.78
