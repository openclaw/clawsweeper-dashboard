## Summary

Keep open. Current main still lacks the host-mediated session attachment SDK seam, and this draft PR is the active fresh-main slice replacing the closed consolidated stack. The latest head fixed the earlier changelog and unsupported-hints concerns, but source inspection found a remaining public SDK formatting mismatch for plain Telegram attachment captions that should block merge until corrected.

## What This Changes

The PR adds `OpenClawPluginApi.sendSessionAttachment`, host-side bundled-plugin file validation and session delivery routing, Telegram/Slack attachment hints, Telegram formatting helper exports, API baseline/changelog updates, and focused SDK/Telegram tests.

## Best Possible Solution

Keep the focused host-mediated attachment seam, but make the public caption-format contract match delivered behavior before merge: plain captions should be escaped or carried as true plain text, MarkdownV2 should either be real Telegram MarkdownV2 or be renamed/deferred, and focused Telegram/SDK contract tests should cover those cases.

## Reproduction Assessment

Yes for the review finding: source inspection shows a `captionFormat: "plain"` call resolves no parse mode, reaches Telegram as default HTML mode, and Telegram media captions are sent with `parse_mode: "HTML"`. A Telegram attachment caption containing plain `<`/`&` text is the concrete failing path.

## Solution Assessment

No, not as currently written. The host-mediated boundary is a maintainable direction, but the public SDK should not accept a plain caption format that is delivered through Telegram's trusted HTML path without escaping or an explicit plain-mode contract.

## Review Findings
