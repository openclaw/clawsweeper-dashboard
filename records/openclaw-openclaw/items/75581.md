## Summary

Keep open. Current main still lacks this Plugin SDK attachment seam, and the latest PR head appears to have fixed the earlier Telegram plain-caption bug, but the branch still has a blocking correctness issue: attachment delivery can fail under best-effort sending while `sendSessionAttachment` reports success to the plugin.

## What This Changes

This PR adds bundled-plugin `sendSessionAttachment` support, host-side attachment validation and session-route lookup, Telegram/Slack delivery hints, outbound parse-mode plumbing, Telegram formatting helper exports, API baseline/changelog updates, and focused tests.

## Best Possible Solution

Keep the focused host-mediated attachment seam, but make the result contract match real delivery: either let attachment delivery errors throw through this API or treat an empty/failed `sendMessage` result as `{ ok: false }`, then lock that behavior with a focused contract test before maintainer review continues.

## Reproduction Assessment

Yes. On the PR head, a channel/media failure that `deliverOutboundPayloads` catches under `bestEffort: true` leaves `sendMessage` with no concrete delivery result, while `sendPluginSessionAttachment` still returns `ok: true` and the validated file count.

## Solution Assessment

No, not as currently written. The seam direction is maintainable, and the caption fix is good, but returning success after best-effort attachment failure is not the narrowest reliable contract for plugin authors.
