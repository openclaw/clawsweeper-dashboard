## Summary

Keep open: current main still has the Slack reasoning suppression gap outside native streaming, and the linked fix PR is open and unmerged, so this issue should remain open until that implementation lands or is replaced.

## What This Changes

The issue asks Slack to suppress Claude reasoning/thinking payloads on normal delivery, streaming fallback, and slash-command reply paths, not only on native streaming.

## Best Possible Solution

Review and land the linked Slack plugin fix, or replace it with an equivalent Slack-owned suppression boundary plus regression tests for normal replies, fallback delivery, and slash replies.

## Reproduction Assessment

Yes, by source inspection: send a Slack reply payload with `isReasoning: true` while native streaming is disabled, fallback delivery is used, or slash-command replies run, and current main has no suppression check on those paths. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to suppress reasoning payloads inside Slack-owned normal/fallback/slash delivery boundaries and add regression coverage there. Because the linked PR is open and uses closing syntax, this issue should stay open while that PR is reviewed or replaced.
