- `P1`: The report describes internal reasoning text reaching real Slack channels for Claude/Bedrock users, which is an urgent channel delivery regression.

## Summary

Keep open: current main still suppresses Slack reasoning payloads only in native streaming, while normal delivery, stream fallback through `deliverReplies`, and slash-command replies can send `isReasoning` payloads. Two open linked PRs already target this issue, so this issue should remain open until one lands or maintainers choose a replacement fix.

## What This Changes

The issue reports that Slack non-streaming, fallback, and slash-command delivery can expose Claude `isReasoning` thinking payloads to users.

## Best Possible Solution

Review and land one of the linked Slack-plugin fixes, or an equivalent maintainer patch, that filters `isReasoning` before every Slack send path and adds regression coverage for normal delivery, fallback delivery, and slash replies.

## Reproduction Assessment

Yes. By source inspection, send a Slack `ReplyPayload` with `isReasoning: true` while native streaming is disabled or via slash replies, and the current helper path has no suppression check before sending.

## Solution Assessment

Yes. A narrow Slack delivery-boundary filter with focused regression tests is the best fix; changing model reasoning generation or adding config would be broader than needed.
