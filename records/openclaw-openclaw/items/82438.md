## Summary

Keep open: this is an active linked fix with sufficient proof, but the latest diff has a narrow correctness bug that should be repaired before merge. The protected `maintainer` label also means cleanup close is not appropriate.

## What This Changes

The PR changes config validation so a missing official external `memory-lancedb` memory slot warns with an install hint, adds regression coverage, and updates the Memory LanceDB docs to show the install step.

## Best Possible Solution

Narrow the warning-only path to official external plugins that can own the memory slot, keep wrong slot IDs fatal, and then land the validator/docs fix through normal maintainer review.

## Reproduction Assessment

Yes. Current-main source shows explicit missing `plugins.slots.memory` values become hard validation issues, and the linked issue plus PR body give a concrete `memory-lancedb` before/after validator path.

## Solution Assessment

No. The intended `memory-lancedb` behavior is narrow and maintainable, but the diff uses official catalog membership instead of memory-slot capability, so it also softens invalid non-memory slot ids.

## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P2] Keep non-memory official plugins fatal:** `src/config/validation.ts:1557-1562`
  - body: `formatMissingOfficialExternalPluginWarning(memorySlot)` returns a warning for any cataloged official external plugin, not only memory plugins. Please restrict this path to official memory-capable plugins, or add catalog capability metadata, so wrong slot ids still fail validation.
