## Summary

Keep this PR open. It targets a valid config-startup bug and has sufficient real behavior proof, but the current branch still has a blocking validator regression: blocked official external memory-slot candidates can be downgraded to warnings.

## What This Changes

The PR changes config validation so a missing official external memory slot warns with an install hint, adds regression coverage, updates the memory-lancedb docs, and adjusts one Codex extension test assertion.

## Best Possible Solution

Land a narrow validator and docs fix that warns only for truly missing cataloged `memory-lancedb` installs while preserving hard failures for unknown or blocked memory-slot IDs.

## Reproduction Assessment

Yes. The current-main source path is clear: `validateConfigObjectWithPlugins()` with an explicit missing `plugins.slots.memory = "memory-lancedb"` and an empty manifest registry reaches the hard missing-plugin issue path.

## Solution Assessment

No. The warning direction is the right narrow fix, but the branch should only downgrade the no-diagnostic missing official install case and must keep blocked official memory-slot diagnostics as validation issues.
