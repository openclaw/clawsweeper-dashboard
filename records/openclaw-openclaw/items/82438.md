## Summary

## What This Changes

The PR changes config validation to warn with the official install hint for an uninstalled cataloged `memory-lancedb` memory slot, adds regression coverage, updates the memory-lancedb docs, and fixes doctor test fixtures.

## Best Possible Solution

Land a narrow validator and docs fix that soft-warns only for a truly missing official `memory-lancedb` install while preserving hard failures for unknown or blocked memory-slot IDs.

## Reproduction Assessment

Yes. Current-main source shows an explicit missing `plugins.slots.memory` reaches the hard missing-plugin issue path, and the PR body provides before/after live validator output for `memory-lancedb` with an empty registry.

## Solution Assessment

No. The warning direction is the right narrow fix, but this branch should only downgrade the no-diagnostic missing official install case and must keep blocked official memory-slot diagnostics as validation issues.
