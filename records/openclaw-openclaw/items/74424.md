## Summary

Keep open: current main still accepts a restart `continuationMessage` but can return a successful coalesced result after the restart signal has already been emitted, before the continuation-writing hook can run. The exact attempted fix and the adjacent continuation PR are both closed unmerged, so no current PR owns the remaining narrow bug.

## What This Changes

Ensure a gateway restart `continuationMessage` is not silently lost when the restart request coalesces with an already emitted or in-flight gateway restart.

## Best Possible Solution

Fix the restart tool and scheduler contract so an accepted continuation is queued exactly once for the correct pending restart, or the tool response explicitly says the continuation was not queued.

## Reproduction Assessment

Yes, source-reproducible. With an emitted but unconsumed restart signal, calling `gateway.restart` with `continuationMessage` reaches the early coalesced return before the hook that writes the continuation sentinel can run.

## Solution Assessment

Yes, the best direction is a narrow gateway restart/tool contract fix, not broad task checkpointing. The implementation should preserve same-session routing and exactly-once delivery, or return an explicit non-queued result.

## Repair Work Prompt

Fix this issue: `gateway.restart` with `continuationMessage` can return `ok` and `coalesced: true` after a restart signal has already been emitted, but the continuation-writing hook never runs, so startup cannot enqueue the promised continuation. Keep the fix narrow to the gateway restart/tool contract: either queue the continuation exactly once for the correct same-session pending restart, or make the tool response explicitly report that it was not queued. Do not implement the broader checkpoint/auto-resume RFC, do not overwrite a global restart sentinel across sessions, and preserve existing durable session-delivery semantics. Add regression coverage for the already-emitted/unconsumed coalescing branch and the gateway tool response/continuation behavior. Add a single active changelog Fixes entry because this is user-visible restart workflow behavior.
