## Summary

Keep open: current main still has the reported control-flow gap, because the in-flight restart coalescing branch returns success before the gateway tool's continuation-writing hook can run, and the related implementation PR is still open.

## What This Changes

The issue asks for gateway restart continuations to be queued or explicitly reported as not queued when a restart request with `continuationMessage` coalesces with an in-flight restart.

## Best Possible Solution

Land a narrow restart-boundary change that persists or merges the accepted continuation into the pending or in-flight sentinel, or changes the tool result so non-delivery is explicit.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to leave a SIGUSR1 restart emitted but unconsumed, then call the gateway tool restart action with `continuationMessage`; current main returns from the in-flight coalescing branch before the tool's `beforeEmit` hook can write the sentinel.

## Solution Assessment

Yes. The issue's proposed boundary is the narrow maintainable direction: preserve the accepted one-shot continuation or explicitly report that it was not queued, while leaving broader checkpoint and auto-resume work to separate trackers.
