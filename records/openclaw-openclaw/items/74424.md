## Summary

Keep open: current main still accepts `continuationMessage` for `gateway.restart`, but an already-emitted restart can return a successful coalesced result before the continuation-writing hook is registered or run. The linked attempted fixes are closed unmerged, and the broader restart-continuation RFCs do not own this narrow contract bug.

## What This Changes

The issue requests that gateway restart continuations survive coalesced in-flight restarts, or explicitly report when the continuation was not queued.

## Best Possible Solution

Fix the restart tool/scheduler contract so an accepted continuation is queued exactly once for the correct pending restart, or the tool returns an explicit non-queued result.

## Reproduction Assessment

Yes, source-reproducible. With an emitted but unconsumed restart signal, `gateway.restart` with `continuationMessage` reaches the early coalesced return before `beforeEmit` can write the sentinel.

## Solution Assessment

Yes, the best path is a narrow gateway restart contract fix with session-safe continuation handling. A broad task checkpoint system would be useful separately, but it is not the minimal fix for this accepted one-shot continuation.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74424 on current main. The bug is that `gateway.restart` accepts `continuationMessage`, but if `scheduleGatewaySigusr1Restart()` hits the already-emitted `hasUnconsumedRestartSignal()` coalesced branch, the gateway tool's `emitHooks.beforeEmit` never runs, so the continuation is not persisted and startup cannot deliver it. Keep the fix narrow: preserve the existing one-shot restart continuation contract, add regression coverage for the already-emitted/unconsumed branch, and do not implement the broader checkpoint/auto-resume RFC. Avoid an unsafe global sentinel rewrite that can mix sessions; any merge/rewrite path must validate the same session/routing or support multiple continuations safely. If delivery cannot be guaranteed, return an explicit non-queued result instead of normal success. Likely files: `src/infra/restart.ts`, `src/agents/tools/gateway-tool.ts`, `src/infra/restart-sentinel.ts`, `src/infra/infra-runtime.test.ts`, `src/agents/tools/gateway-tool.test.ts`, `src/gateway/server-restart-sentinel.test.ts`, and `CHANGELOG.md` if the user-facing behavior changes. Validate with the focused Vitest wrapper and format check; run changed checks in Testbox before handoff if runtime files are changed.
