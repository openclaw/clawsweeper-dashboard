## Summary

Keep open: current main still has the reported in-flight coalescing path where the gateway tool accepts a continuation but `scheduleGatewaySigusr1Restart()` returns before the sentinel-writing hook can run. The linked open PR #73812 remains the implementation candidate, while the narrower #74443 was closed unmerged.

## What This Changes

The issue reports that a gateway restart request with `continuationMessage` can coalesce with an in-flight restart and return success while dropping the continuation.

## Best Possible Solution

Keep this issue open until #73812 or an equivalent narrow fix lands. The durable end state should persist or merge the accepted continuation for the in-flight coalesced restart branch, or return an explicit not-queued result, with regression coverage for that branch.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to leave a SIGUSR1 restart emitted but unconsumed, then call the gateway restart tool with `continuationMessage`; current main returns from the unconsumed coalescing branch before the tool's sentinel-writing hook can run.

## Solution Assessment

No. Current main is not the best way to solve the issue because it can acknowledge a coalesced restart while dropping the accepted continuation; the narrow maintainable fix belongs at the gateway-tool/restart-scheduler boundary rather than broad task checkpointing.
