## Summary

Keep open: current main still has the emitted-but-unconsumed coalescing gap, and the linked fix work is not merged proof that the continuation contract is repaired.

## What This Changes

The issue asks OpenClaw to preserve or explicitly reject a `gateway` restart `continuationMessage` when the restart request coalesces with an already in-flight gateway restart.

## Best Possible Solution

Land a narrow restart-boundary fix that either merges an accepted continuation into the pending or in-flight restart sentinel, or returns an explicit non-queued result, with regression coverage for the gateway tool entering the unconsumed in-flight coalescing branch.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to leave a SIGUSR1 restart emitted but unconsumed, then call the gateway tool restart action with `continuationMessage`; current main returns from `hasUnconsumedRestartSignal()` before the tool's `beforeEmit` hook can persist the sentinel.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve the accepted one-shot continuation or explicitly report non-delivery; the broader checkpoint/resume RFC is not required for this specific contract bug.
