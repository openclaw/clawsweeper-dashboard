## Summary

Keep open: current main still has the reported in-flight coalescing gap, and the related open PR is implementation context rather than merged proof that this is fixed.

## What This Changes

A gateway restart tool call with `continuationMessage` should durably queue or explicitly reject that continuation when it coalesces with an already in-flight restart.

## Best Possible Solution

Land a narrow restart-boundary fix that preserves or merges accepted continuations across in-flight coalescing, or returns an explicit non-queued result, with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to leave a SIGUSR1 restart emitted but unconsumed, then call the gateway tool restart action with `continuationMessage`; current main returns before the tool’s `beforeEmit` hook can persist the sentinel.

## Solution Assessment

Yes. The requested boundary is the maintainable fix: preserve the accepted one-shot continuation or explicitly report non-delivery, while leaving broader restart checkpoint/resume work to separate trackers.
