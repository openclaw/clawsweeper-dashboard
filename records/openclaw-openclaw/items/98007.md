## Summary

Keep open: this PR targets a real released update bug, but it is not merge-ready because the new classifier treats dual-format native package directories as bundle payloads and bypasses the existing native package validation path.

## What This Changes

The PR adds a payload-kind classifier and uses it to skip npm-specific update payload validation for bundle-format plugin directories.

## Best Possible Solution

Align post-update payload checks with the existing plugin install contract: skip npm checks for true bundle payloads while keeping native package validation for dual-format package directories.

## Reproduction Assessment

Yes. Current main's tracked-plugin payload checks reach `missing-package-json` for an existing bundle install directory without top-level `package.json`, and the PR regression is source-reproducible for a dual-format directory with both package metadata and bundle markers.

## Solution Assessment

No. The PR touches the right update-cli validation path, but the best fix must preserve native-first dual-format package validation instead of treating every bundle-marker directory as safe to skip npm checks.
