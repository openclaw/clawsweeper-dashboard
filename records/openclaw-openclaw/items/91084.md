## Summary

Keep open: current main and v2026.6.1 still let `agents.defaults.imageModel` block native-vision skip, despite docs saying it is fallback-only for primary models that cannot accept images.

## What This Changes

The issue reports redundant media-understanding image descriptions when `agents.defaults.imageModel` is configured alongside a vision-capable primary model.

## Best Possible Solution

Fix the media-understanding runner so `agents.defaults.imageModel` remains fallback-only after native vision detection, while preserving explicit `tools.media.image.models` overrides and MiniMax VLM special handling.

## Reproduction Assessment

Yes, source-level: current `runCapability` skips the native-vision check when `agents.defaults.imageModel` is present, then `resolveAutoEntries` selects that image model and successful outputs are formatted into `ctx.Body`. I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

Yes, with one refinement: the fix should not blindly remove all explicit-config handling, because explicit `tools.media.image.models` should still be able to force media understanding. The narrow maintainable fix is to treat `agents.defaults.imageModel` as fallback-only while keeping explicit per-media model config as an override.
