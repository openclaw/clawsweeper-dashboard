## Summary

Keep open: current main still routes Codex app-server ChatGPT token refresh requests through a path where upstream Codex waits only 10 seconds and emits the exact reported timeout text, while OpenClaw's own OAuth refresh work can legitimately take longer.

## What This Changes

The issue reports valid OpenAI/Codex OAuth sessions where probes succeed after slightly more than 10 seconds but cron, heartbeat, and embedded Codex app-server runs fail with `auth refresh request timed out after 10s`.

## Best Possible Solution

Coordinate or consume a Codex app-server fix that widens the external auth refresh wait beyond normal OpenClaw OAuth refresh latency, then add OpenClaw regression coverage for a slow-but-successful ChatGPT auth refresh request.

## Reproduction Assessment

Yes from source, not live: Codex app-server current source hard-codes a 10-second external auth refresh wait and emits the exact reported timeout error. A live repro would need real OpenAI/Codex OAuth credentials and a refresh path that runs longer than 10 seconds.

## Solution Assessment

No to closing or adding a config knob as the first move; the narrowest maintainable fix is to align the Codex app-server external-auth wait with OpenClaw's OAuth refresh envelope and keep diagnostics clear. The exact landing path needs maintainer/dependency-owner judgment because the timeout lives in @openai/codex.
