## Summary

Keep open: current main and the latest v2026.6.8 release still route Codex app-server `account/chatgptAuthTokens/refresh` callbacks into a forced OpenClaw OAuth refresh, while current upstream Codex still cancels that external-auth callback after 10 seconds. The open candidate PR needs maintainer review because its current review flags a 401-recovery risk, so it is not a safe reason to close this issue yet.

## What This Changes

The issue reports valid OpenAI/Codex OAuth profiles timing out during Codex app-server cron, heartbeat, and embedded-run refresh callbacks because the callback deadline is 10 seconds while OpenClaw refresh can take longer.

## Best Possible Solution

Keep this issue open until maintainers land and prove a reason-aware Codex auth-bridge mitigation or upstream timeout change that avoids the 10-second callback failure without breaking 401 recovery.

## Reproduction Assessment

Yes, at source level: OpenClaw still forces a Codex app-server OAuth refresh for `account/chatgptAuthTokens/refresh`, and upstream Codex still cancels that external-auth RPC after 10 seconds. I did not run a live OAuth account repro in this read-only pass.

## Solution Assessment

No, treating the 120s OpenClaw OAuth constant as a complete fix is not the best solution because upstream Codex still has the 10-second external-auth callback deadline. The best fix needs maintainer-approved bridge semantics plus regression and live app-server proof.
