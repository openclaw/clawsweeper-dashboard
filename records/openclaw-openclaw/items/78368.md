The issue requests adding a `SILENT_REPLY_TOKEN` duplicate-suppression guard to the `expectsCompletionMessage` subagent announce prompt to prevent duplicate user-visible completion messages.

## Best Possible Solution

Land or replace the linked narrow PR that adds the silent duplicate guard to the completion-mode announce instruction and regression coverage; keep structural fingerprint suppression as a separate follow-up if maintainers want it.

## Reproduction Assessment

Yes, source-reproducible: with a top-level requester and `expectsCompletionMessage: true`, current main builds a completion announce prompt that lacks the silent duplicate guard while adjacent branches include it. I did not run a live agent/channel scenario in this read-only pass.

## Solution Assessment
