## Best Possible Solution

Land a scoped UI fix that shortens only technical fallback identifiers, preserves the existing displayName/label layout, includes real Sessions table/details proof, and keeps the broader naming-precedence issue open unless fully addressed.

## Reproduction Assessment

## Solution Assessment

No, not as written. The safer fix is to shorten the technical fallback without promoting displayName into the primary key cell, and to keep the broader linked naming-precedence issue open unless it is fully implemented.
