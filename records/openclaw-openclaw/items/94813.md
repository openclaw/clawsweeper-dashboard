## Summary

Keep open: the readability problem is still present on current main and the PR is useful, but the patch still changes existing displayName layout beyond the stated long-ID fallback and the supplied screenshots do not prove the real Sessions table/details behavior.

## What This Changes

The PR truncates long direct-chat identifiers in the shared session display fallback, uses that fallback in the Control UI Sessions table/details, changes key-cell hover titles to the raw session key, and adds focused UI tests.

## Best Possible Solution

Land a scoped UI fix that truncates only technical fallback direct identifiers, preserves existing label/displayName layout, keeps the broader linked issue open unless naming precedence is fully implemented, and includes real Sessions table/details proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 keep full direct identifiers in fallback labels and raw Sessions fallback titles. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No, not as written. The safer fix is a key-only technical fallback or localized Sessions display helper, because passing the whole row into resolveSessionDisplayName changes existing label/displayName layout and broadens the helper-wide display surface.
