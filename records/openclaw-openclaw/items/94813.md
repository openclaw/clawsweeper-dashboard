## Summary

Keep this PR open: the scoped truncation idea is useful, but the current helper is wired into every fallback Sessions row and can regress non-direct labels such as subagent or cron sessions by rendering duplicated typed prefixes.

## What This Changes

The branch adds a Sessions-view formatter for long direct-chat session keys, wires it into the Sessions table and details title, and adds session display regression tests.

## Best Possible Solution

Limit the formatter to direct-chat fallback keys or otherwise preserve exact non-direct fallback output, add exact regression coverage for non-direct rows, and then supply redacted real UI proof for the long-key Sessions table/details behavior.

## Reproduction Assessment

Yes, at source level: current main renders fallback Sessions rows and details from the raw `row.key`, and the PR's helper can be traced to broader non-direct formatting. I did not run a browser session in this read-only review.

## Solution Assessment

No, not at this head: localizing truncation outside `parseSessionKey` is the right direction, but applying the helper to all fallback keys makes the fix broader than the reported direct-chat overflow path.
