## What This Changes

The PR truncates long direct-chat identifier fallbacks and uses `resolveSessionDisplayName` for the Control UI Sessions table and details title, with helper and renderer tests.

## Best Possible Solution

Use the shared helper for visible fallback labels, suppress duplicate secondary displayName when it supplies the primary label, preserve raw-key hover access, and leave the broader naming policy to https://github.com/openclaw/openclaw/issues/61213.

## Reproduction Assessment

Yes, source-level. On PR head, `keyCellTitle` can resolve to `row.displayName` while `showDisplayName` remains true, so displayName-only rows duplicate in the table and details panel; real browser rendering was not run.

## Solution Assessment

No. The display-layer approach is an acceptable narrow mitigation, but this patch still needs primary/secondary label separation and real dashboard proof before it is the best merge shape.
