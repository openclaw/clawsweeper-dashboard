## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid rendering display names twice:** `ui/src/ui/views/sessions.ts:928-932`
  - body: `keyCellTitle` now comes from `resolveSessionDisplayName`, which returns `row.displayName` when there is no label. Because `showDisplayName` still stays true for that same row, displayName-only sessions render the same name as the link text and again in `.session-key-display-name` (and likewise in the details title/subtitle). Split the primary label from the secondary displayName check, or hide the subtitle when it matches the resolved primary label.
  - confidence: 0.9
- **[P2] Preserve the raw key in the hover title:** `ui/src/ui/views/sessions.ts:843`
  - body: This value is used as the `<div title>`, but the PR also truncates long direct identifiers before returning it. That removes the current hover path for seeing/copying the full `agent:...:direct:...` key, unlike the sidebar pattern that keeps both label and raw key. Keep a separate visible label and title such as `${label} · ${row.key}`.
  - confidence: 0.86
