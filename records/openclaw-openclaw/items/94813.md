## Summary

Keep open. The PR addresses a real source-reproducible Control UI readability problem, but the patch still changes the existing displayName layout, the supplied screenshots do not prove the target Sessions table/details behavior, and the linked issue is broader than technical-ID truncation.

## What This Changes

The PR truncates long direct-chat session-key identifiers in Control UI display labels, uses the shared session display resolver in the Sessions table/details fallback path, preserves raw keys in hover titles, and adds UI regression tests.

## Best Possible Solution

Land a scoped UI fallback fix that shortens long technical direct IDs while preserving existing displayName layout and keeping the broader naming-precedence issue open unless fully solved.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 return the full direct-chat identifier from `parseSessionKey`, and Sessions rows currently render raw keys when no identity label exists.

## Solution Assessment

No, not as written: the safer fix is a technical-key fallback formatter for the Sessions table/details path, not passing the full row into `resolveSessionDisplayName` and changing label/displayName layout.
