## Summary

Keep open. Current main does not contain the ExecuTorch plugin or macOS Talk STT backend, so the PR is not obsolete, but it has blocking security and architecture issues plus insufficient real behavior proof for an external PR.

## What This Changes

The PR adds an optional ExecuTorch Parakeet-TDT STT backend for macOS Talk Mode, a bundled executorch model-plugin runtime, transcript-only Talk Mode rendering, backend-aware permissions, and related docs/tests.

## Best Possible Solution

Use the existing media-understanding provider contract, add a pinned and integrity-verified native-runtime distribution path, then provide real macOS setup/transcribe/Talk Mode proof before merge.

## Reproduction Assessment

Not applicable for the central feature: current main does not have this optional ExecuTorch STT backend. Source inspection confirms the PR adds new runtime/config paths, but I did not reproduce the macOS crash or native STT path in this read-only review.

## Solution Assessment

No. The current PR direction needs changes before merge because it downloads and loads an unverified native runtime and adds a parallel media-provider seam instead of using current main's canonical media-understanding provider contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Verify native runtime downloads before loading them:** `extensions/executorch/src/cli.ts:194-198`
  - body: `setup` downloads `libparakeet_tdt_runtime.dylib` from Hugging Face and the addon later loads that library with `dlopen`, but the code does not pin a revision or verify a checksum/signature. A changed or compromised external artifact would become native code execution inside OpenClaw after users run setup, so the runtime binary needs a pinned source plus integrity verification before it is ever loaded.
  - confidence: 0.92
- **[P2] Use the existing media provider registration seam:** `src/plugins/types.ts:1758-1760`
  - body: Current main already has `registerMediaUnderstandingProvider` and builds the media registry from plugin capability providers. Adding a separate optional `registerMediaProvider` path means ExecuTorch has to be manually threaded through each caller and can be missed by generic media-understanding paths; register this provider through the existing media-understanding provider contract instead.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR adds a path that downloads a native runtime from Hugging Face and later loads it in-process without source pinning or integrity verification.

Concerns:

- **[high] Unverified native runtime download:** `extensions/executorch/src/cli.ts:194`
  - body: `openclaw executorch setup` downloads runtime candidates from an external Hugging Face repository and only checks that files exist before the addon can load the shared library. This creates a supply-chain code-execution risk unless the binary source is pinned and verified with hashes/signatures before loading.
  - confidence: 0.92
