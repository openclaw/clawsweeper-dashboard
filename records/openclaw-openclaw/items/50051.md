## What This Changes

The PR adds a bundled ExecuTorch Parakeet-TDT STT plugin/runtime, wires local STT into macOS Talk Mode and media understanding, strips Talk Mode prompt boilerplate from chat display, adjusts Talk defaults/permissions, and documents the new STT/plugin surfaces.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Pin downloaded native runtime artifacts:** `extensions/executorch/src/cli.ts:196`
  - body: `executorch setup` downloads the runtime dylib/model files from Hugging Face by repo and filename, and the plugin later loads that dylib in-process. ## Security Review

Status: needs_attention

Summary: The PR introduces in-process native runtime execution and downloads model/runtime artifacts without pinned provenance or digest verification.

Concerns:

- **[high] Unpinned native runtime download:** `extensions/executorch/src/cli.ts:196`
  - body: The setup path shells out to `hf download` for runtime/model files and the model plugin identifies a mutable Hugging Face repository, but the PR does not pin a revision or verify checksums before loading the downloaded dylib into the OpenClaw process.
  - confidence: 0.88
- **[medium] Runtime dylib is modified after download:** `extensions/executorch/src/runtime-library.ts:81`
  - body: `ensureRuntimeLibraryLoadable` may rewrite the downloaded dylib with `install_name_tool`; any artifact verification design must account for this mutation so verification is not bypassed or invalidated silently.
  - confidence: 0.78
