## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the proposed ExecuTorch/Parakeet Talk Mode backend, transcript-only Talk display, and notification .app guard, while the PR branch still has blocking docs and native-artifact supply-chain issues that require maintainer/security review rather than conservative cleanup closure.

## What This Changes

The PR adds an optional bundled ExecuTorch Parakeet-TDT STT plugin/runtime, wires it into macOS Talk Mode, hides Talk Mode prompt boilerplate in chat display, and guards macOS notification permission checks outside app bundles.

## Best Possible Solution

Split the work into smaller maintainer-owned changes: land the narrow PermissionManager guard and transcript-only display if desired, decide whether macOS Talk needs a backend selector in the app, reuse or deliberately extend the existing media-understanding provider contract, and only consider a bundled ExecuTorch plugin after pinned artifact provenance and native-code security review are in place.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature PR. The current-main gap is reproducible by source inspection and repo search; the PR's docs and supply-chain issues are reproducible from the PR head files without live macOS STT hardware.

## Solution Assessment

No. The direction may be useful, but this PR is too broad and currently mixes app behavior, public plugin seams, native runtime execution, docs, and generated config in one branch; the safer solution is to split it and harden artifact provenance before any native plugin merge.

## Security Review

Status: needs_attention

Summary: The PR adds native code execution and downloads a runtime dylib/model artifacts without pinned provenance or digest verification.

Concerns:

  - body: The setup path shells out to `hf download` for runtime/model files and the model plugin identifies a mutable Hugging Face repository, but the PR does not pin a revision or verify checksums before loading the downloaded dylib into the OpenClaw process.
