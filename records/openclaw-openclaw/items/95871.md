## What This Changes

The issue asks OpenClaw to preserve the native Codex missing `web_search` tool-result / incomplete-turn cause in cron-visible diagnostics instead of reporting only a timeout-shaped cron failure.

## Best Possible Solution

Add a focused cron/agent boundary fix so timeout cleanup can preserve or prefer the underlying incomplete-turn / `missing_tool_result` cause when the embedded run has already recorded it, with regression coverage for the timeout-vs-core-result race.

## Reproduction Assessment
