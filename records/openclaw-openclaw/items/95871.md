## Summary

Keep open: current main still has a source-visible cron timeout/core-result race that can return the timeout-shaped cron outcome before the embedded Codex incomplete-turn or missing-tool-result result becomes cron-visible; related PR #95943 improves timeout row attribution but explicitly does not fix the underlying `missing_tool_result` cause preservation.

## What This Changes

The issue asks OpenClaw to preserve native Codex `missing_tool_result` or incomplete-turn causes in cron-visible diagnostics instead of surfacing only a timeout-shaped cron failure.

## Best Possible Solution

Teach the cron timeout/cancel outcome to preserve or prefer embedded incomplete-turn and `missing_tool_result` diagnostics when the Codex run has already recorded them, while retaining the watchdog timeout cleanup semantics.

## Reproduction Assessment

No live deterministic current-main reproduction was established. Source inspection gives a medium-confidence reproduction path: the cron timeout `Promise.race` can return the timeout wrapper before the embedded Codex missing-tool-result result becomes the cron result.

## Solution Assessment

Yes, preserving the underlying Codex failure cause is the right direction. The maintainable repair should be a narrow cron/agent handoff fix rather than a new timeout knob or a Codex-only workaround.
