## What This Changes

The PR moves MiniMax `current_interval_usage_count` and `current_weekly_usage_count` aliases from remaining-quota parsing to consumed-usage parsing and updates MiniMax usage tests to expect inverted percentages.

## Best Possible Solution

Settle the MiniMax endpoint contract on https://github.com/openclaw/openclaw/issues/81156, then land an endpoint- or record-shape-specific parser update only if same-account live API/dashboard proof shows current remaining-count handling is wrong for the endpoint OpenClaw polls.

## Solution Assessment

No. The PR is a plausible parser change, but not the best proven fix because current docs/tests and linked live evidence support remaining-count handling while the broader endpoint semantics remain open on https://github.com/openclaw/openclaw/issues/81156.
