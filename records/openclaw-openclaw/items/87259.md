## Summary

This PR should stay open for review, but the current patch is not merge-ready: it reverses shipped MiniMax quota semantics that current source, tests, docs, upstream discussion, and a live OpenClaw report all treat as remaining quota, and it has no real MiniMax behavior proof showing the endpoint contract changed.

## What This Changes

The PR reclassifies four MiniMax `current_*_usage_count` aliases from remaining-count parsing to used-count parsing and updates MiniMax usage tests to expect higher `usedPercent` values.

## Best Possible Solution

Preserve the shipped remaining-count handling, and only add endpoint-specific parsing if redacted live MiniMax evidence proves a different endpoint returns consumed counts.

## Reproduction Assessment

## Solution Assessment
