## What This Changes

The PR adds ClawHub artifact-unavailable handling and npm fallback for official ClawHub plugin install/update paths, updates tests and changelog, refreshes Codex prompt snapshots, and fixes the browser image-ops facade import.

## Best Possible Solution

Keep npm fallback for a machine-readable benign artifact-unavailable state, fail closed on ClawHub auth, moderation, security, and server failures, and require redacted live install/update proof before merge.

Yes. Source inspection of the PR head shows any non-special `ClawHubRequestError` during artifact download becomes `artifact_unavailable`, and update/onboarding/doctor fallback predicates then allow npm fallback; I did not run tests because this review is read-only.

## Solution Assessment

No. The broad fallback is not the best solution because it mixes benign artifact rollout gaps with auth, throttling, server, and moderation failures; a narrower ClawHub error contract is the safer repair.
