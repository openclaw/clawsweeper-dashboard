## Summary

Current main and the latest release still use stdin-backed `ffprobe pipe:0` for Telegram video dimensions, so this recent source-backed report should stay open as a narrow Telegram/media repair candidate.

## What This Changes

The issue requests fixing Telegram `sendVideo` aspect-ratio distortion for large videos by avoiding failed stdin-based video-dimension probing.

## Best Possible Solution

Change the shared video-dimension helper to probe a private seekable temp file with cleanup, while preserving the current no-crash fallback and Telegram-only `width`/`height` behavior.

## Reproduction Assessment

No live reproduction was completed, but current source shows the exact `pipe:0` probe and fallback path that would omit dimensions after a probe failure. The reporter supplied concrete size thresholds and observed Telegram playback distortion.

## Solution Assessment

Yes. A seekable temp-file probe is the narrowest maintainable fix because it changes how dimensions are obtained while keeping the existing Telegram send contract and no-crash fallback.
