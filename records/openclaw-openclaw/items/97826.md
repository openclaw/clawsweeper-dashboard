## Summary

Current main and the latest release still probe Telegram video dimensions through `ffprobe pipe:0`; the report is recent, source-backed, and narrow enough for a focused repair rather than cleanup closure.

## What This Changes

Large Telegram videos can be sent without `width` and `height` when stdin-based ffprobe probing fails, causing distorted playback in Telegram.

## Best Possible Solution

Keep this open and change the shared video-dimension helper to probe a private seekable temp file with cleanup, while preserving the existing no-crash fallback and Telegram-only width/height behavior.

## Reproduction Assessment

No live reproduction was completed because `ffprobe`/`ffmpeg` are unavailable in this review environment, but current source shows the exact `pipe:0` probe and fallback path that would omit dimensions after a probe failure.

## Solution Assessment

Yes, the reported seekable temp-file probe is the narrowest likely fix because it keeps the current Telegram send contract and only changes how the shared helper obtains dimensions before deciding whether to include `width` and `height`.

## Repair Work Prompt

Fix the Telegram large-video aspect-ratio bug reported at https://github.com/openclaw/openclaw/issues/97826. Current `probeVideoDimensions` in `src/media/video-dimensions.ts` probes `ffprobe` through `pipe:0`; when that fails for large MP4 buffers it returns `undefined`, and the Telegram direct and reply send paths omit `width`/`height` for `sendVideo`. Change the helper to materialize the buffer to a private seekable temp file, probe that path with ffprobe, and clean it up reliably. Preserve the existing behavior that ffprobe absence or malformed output does not crash sends; do not add new config/env knobs, do not force videos to documents, and do not add thumbnails unless a maintainer explicitly asks. Update focused tests for the new file-path probe and fallback cleanup behavior, plus keep Telegram direct/reply tests proving dimensions are passed when probing succeeds. No CHANGELOG.md edit for a normal fix PR.
