## Summary

Keep open. Current main has a documented system-prompt cache boundary and related hardening, but it does not implement this PR's full dynamic-tail/runtime split; the PR still contains potentially useful performance work. It is not mergeable as-is because the branch is stale against current main and its final-system-section placement of `extraSystemPrompt` still raises a concrete prompt trust-boundary concern.

## What This Changes

This PR reorders `src/agents/system-prompt.ts` for a larger prompt-cache-stable prefix, adds dynamic runtime-tail handling, moves volatile prompt sections, sanitizes context file headers, gates message hints, and updates system-prompt tests.

## Best Possible Solution

The maintainable end state is a rebased prompt-cache ordering change that preserves current main's user-role untrusted metadata boundary, keeps the context-header sanitization and message-hint gating, and proves cache behavior with focused prompt-composition/cache-boundary tests before a Testbox changed gate.

## Reproduction Assessment

Not applicable as a performance/refactor PR rather than a user bug. The cache claims can be reproduced by rendering paired prompts with changed runtime/context inputs and measuring the first differing byte; the security concern is source-reproducible by tracing `extraSystemPrompt` composition into the final system-prompt section on the PR head.

## Solution Assessment

No, not as-is. The volatility-ordering direction is reasonable, but the patch needs to be rebased over current main and redesigned so channel/user-sourced group metadata remains outside final trusted system text or is followed by a trusted delimiter/final rules section.
