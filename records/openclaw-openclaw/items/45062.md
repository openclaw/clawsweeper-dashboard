## Summary

Keep open. Current main has partially overlapping cache-boundary work, but this PR still needs a rebase onto that design, contributor real-behavior proof, and a security fix before it should merge.

## What This Changes

The branch restructures `src/agents/system-prompt.ts` and its tests to move volatile runtime, skills, memory, tool, and group-context sections later for prompt KV-cache reuse.

## Best Possible Solution

Rebase the performance idea onto the current cache-boundary implementation, keep untrusted conversation context neutralized and not final without a guardrail, and attach measured prompt-prefix proof from a real run.

## Reproduction Assessment

Yes for the review finding: source inspection traces `extraSystemPrompt` from inbound/group context assembly to the PR's final system-prompt insertion. No separate bug reproduction is applicable because the PR is a performance refactor.

## Solution Assessment

No. The stable-prefix goal is valid, but this branch should be adapted to main's existing cache-boundary design and should not make untrusted group/inbound context the final system prompt section without mitigation.

- **[P1] Keep untrusted group context out of the final system tail:** `src/agents/system-prompt.ts:849`
