## What This Changes

Update `openclaw status --deep` so mature workspaces with no pending `BOOTSTRAP.md` are described as not actively bootstrapping rather than as having no bootstrap files.

## Best Possible Solution

Land or replace the existing status wording fix so the Agents row reports the pending `BOOTSTRAP.md` count as `0 bootstrapping`/`N bootstrapping`, with regression coverage, then close this issue.
