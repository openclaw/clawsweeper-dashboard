## Summary

Keep open: the OOM hardening is useful and current main still has the unbounded provider transport paths, but the patch needs changes before merge because the SSE cap is enforced before complete events are drained and the new overflow paths do not deterministically cancel the guarded upstream stream.

## What This Changes

Adds file-local byte caps to provider transport SSE sanitization and streaming JSON-to-SSE synthesis, with tests for oversized delimiter-free SSE and streaming JSON bodies.

## Best Possible Solution

Keep the caps, but drain complete SSE events before capping only the unterminated tail and cancel the upstream managed reader when either overflow guard fires.

## Reproduction Assessment

Yes. Current main's provider transport source shows delimiter-free SSE and streaming JSON bodies can accumulate without a cap, and the contributor provided terminal proof against a real node:http server; I did not run tests in this read-only review.

## Solution Assessment

No. The cap is the right mitigation, but the best narrow fix is to drain complete SSE events before enforcing the tail cap and to cancel the upstream managed response on overflow.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Drain complete events before enforcing the SSE cap:** `src/agents/provider-transport-fetch.ts:175`
  - body: This check runs immediately after appending the decoded transport chunk, before the loop below removes complete `\n\n`-delimited events. A single valid network read containing more than 64 KiB of many small complete SSE events will now throw; process complete events first and cap only the remaining unterminated tail.
  - confidence: 0.91
- **[P2] Cancel the upstream stream when an overflow guard fires:** `src/agents/provider-transport-fetch.ts:138-140`
  - body: The new overflow errors are caught by the wrapper and surfaced with `controller.error(error)`, but the source reader is not canceled. Because this wrapper sits on top of `buildManagedResponse`, a local overflow can skip the deterministic cancel/finalize path that releases the guarded fetch and local-service lease; cancel the reader before erroring and cover it in the overflow tests.
  - confidence: 0.84
