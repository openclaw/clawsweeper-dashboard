Keep open. Current main covers part of the recovery problem with a newer diagnostic reset path, but it does not fully replace the PR’s queued busy receipt, aborted-run payload suppression, or reset audit metadata work; the PR also still has correctness and audit-integrity blockers before merge.

## What This Changes

The PR branch adds embedded-run watchdog lane recovery, aborted-run assistant payload suppression, queued busy receipts, sessions.reset audit metadata/protocol fields, and focused tests.

## Best Possible Solution

Replace or split the PR into narrower changes: keep recovery aligned with the current abort-and-drain diagnostic model, add aborted payload suppression against the current payload/directive logic if still needed, route queued feedback through the canonical busy-feedback issues, and only add reset audit metadata through documented protocol semantics with server-derived attribution.

## Reproduction Assessment

Yes for the PR blockers by code inspection: the watchdog release path clears active-run state while the attempt is still known to be streaming, and the audit fields are accepted from RPC params. For the requested features, current-main source inspection shows absence rather than a runtime failure; no tests were run because this review was read-only.

## Solution Assessment

No. The PR is not the best current solution because main now has a safer generic recovery path, while this branch mixes unrelated runtime, UX, payload, and protocol changes and introduces a force-clear race plus spoofable audit attribution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep the run active until it actually stops:** `src/agents/pi-embedded-runner/run/attempt.ts:1254-1262`
  - body: The watchdog release timer fires while the timed-out run is still known to be streaming, then resets the session lane and clears ACTIVE_EMBEDDED_RUNS. That lets a later same-session run observe the session as idle before the old attempt has stopped emitting or writing transcript state. Keep the active-run marker until abort/drain confirms termination, or gate any force-clear on a confirmed stop.
  - confidence: 0.88
- **[P2] Derive reset audit actor server-side:** `src/gateway/server-methods/sessions.ts:307-312`
  - body: sessions.reset accepts triggeredBy and triggerSource from RPC params and persists them as audit attribution. Any caller allowed to reset a session can spoof who or what caused the reset, so authoritative actor/source fields should come from the authenticated request context or be stored separately as caller-supplied hints.
  - confidence: 0.78
