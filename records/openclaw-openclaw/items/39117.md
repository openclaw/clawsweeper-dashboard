## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep active run marked until it stops:** `src/agents/pi-embedded-runner/run/attempt.ts:1258-1262`
  - body: The watchdog callback force-clears the session lane and active-run entry while the timed-out run may still be streaming. A same-session followup can then start before the old attempt has stopped emitting or writing transcript state, so keep the active marker until abort/drain confirms termination or gate force-clear on a confirmed stop.
  - confidence: 0.88
- **[P2] Derive reset audit actor server-side:** `src/gateway/server-methods/sessions.ts:305-312`
  - body: sessions.reset accepts triggerSource and triggeredBy from RPC params and persists them as audit attribution. Any caller allowed to reset a session can spoof who or what caused the reset, so authoritative actor/source fields should come from authenticated request context or be stored separately as caller-supplied hints.
  - confidence: 0.78
