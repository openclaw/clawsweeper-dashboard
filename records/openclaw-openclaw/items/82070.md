## What This Changes

The issue reports a severe v2026.5.12 regression where CLI commands and completion cache generation became slow after Codex was installed, with follow-up evidence of first-turn gateway event-loop stalls.

## Best Possible Solution

Profile a packaged current-main install with Codex enabled, then remove unnecessary plugin/runtime loading from health, completion-cache, and first-turn gateway startup paths while preserving plugin-owned behavior when commands actually need it.

No, not at high confidence. The report and comments include concrete timing output, and current source/fixture evidence shows related slow paths, but this read-only review did not run a packaged current-main Codex install or capture a CPU profile.
