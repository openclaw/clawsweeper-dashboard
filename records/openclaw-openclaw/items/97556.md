## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep presentation schema in the shared message tool:** `extensions/telegram/src/message-tool-schema.ts:94`
  - body: This adds a Telegram-specific `presentation` property even though the shared message tool already defines the portable presentation contract. Channel extra properties are spread after the generic send schema, so when inline buttons are enabled this replaces the broader schema that supports text/context/select blocks, callback actions, `value`, styles, and other existing fields; please remove this contribution and rely on the shared schema.
  - confidence: 0.92
- **[P2] Keep Telegram send runtime lazy:** `extensions/telegram/src/channel.ts:46`
  - body: `channel.ts` is a hot channel entrypoint, but this import pulls in `agent-tools-location.ts`, which imports `./send.js` at module load time. Current main keeps the full send runtime behind `send-runtime.ts`; move the send import behind the tool execution path or use the existing lazy loader so plugin discovery does not eagerly load the delivery runtime.
  - confidence: 0.86
- **[P2] Validate horizontal accuracy before sending:** `extensions/telegram/src/send.ts:2065-2067`
  - body: `horizontalAccuracy` is advertised as 0-1500 in the schema and Telegram's API uses that range, but the runtime forwards any parsed number after validating only latitude and longitude. Add the same runtime guard for `horizontalAccuracy` so CLI/tool calls fail locally with a clear error instead of sending an invalid Bot API request.
  - confidence: 0.78
