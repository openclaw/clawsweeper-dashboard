## Summary

Keep this PR open. Current main still returns the raw bootstrap file name in context stats, so hook-injected bootstrap files with a path but missing or empty name can still surface as an undefined label in `/context` output. The PR remains a focused, correct-looking implementation candidate for #47941, with only a small test-strengthening question before normal maintainer review.

## What This Changes

This PR updates `buildBootstrapInjectionStats` to derive a display name from `file.name`, normalized `basename(file.path)`, or `"(virtual file)"`, and adds a regression test for missing and empty bootstrap file names.

## Best Possible Solution

Land this PR or an equivalent small patch in `src/agents/bootstrap-budget.ts` that normalizes the display label from a nonempty name, normalized path basename, then a placeholder, with regression coverage for missing and empty names. Once merged, #47941 can close as implemented.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: a hook-provided bootstrap file with a valid `path` but `name` missing or empty passes through sanitization, current main returns `file.name` in the stat, and `/context` interpolates that raw stat name.

## Solution Assessment

Yes. The PR fixes the display-layer failure at the narrow stats boundary that feeds all context-report consumers, without broadening the bootstrap-file contract or adding core hook migration behavior. The safer refinement is only to strengthen the regression test assertions, not to move the fix elsewhere.
