## Summary

Keep this PR open: the feature is plausible, but the patch misses one Slack routing path, and the external PR still needs after-fix real Slack behavior proof before merge.

## What This Changes

The PR adds a Slack per-channel `replyToMode` config/schema/docs surface and uses it ahead of account or chat-type reply mode fallback.

## Best Possible Solution

Keep the per-channel override in the Slack plugin config if maintainers want the feature, but carry the resolved channel config through every routing resolution and prove it in a real Slack channel.

## Reproduction Assessment

Yes for the review finding: source inspection shows the regex-mention reroute calls `resolveSlackRoutingContext` without the new `channelConfig` parameter. A live Slack reproduction was not run in this read-only review.

## Solution Assessment

No. The proposed approach is close, but the best fix also passes `channelConfig` into the reroute path and adds coverage for a mention-pattern channel with per-channel `replyToMode`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Pass channel config through the Slack reroute:** `extensions/slack/src/monitor/message-handler/prepare.ts:558`
  - body: The added `channelConfig` only reaches the initial routing resolution. Pass the same `channelConfig` into the reroute and cover that mention-pattern path.
