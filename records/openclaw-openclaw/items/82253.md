## Summary

Keep open: this is a new Slack config feature rather than an already-implemented cleanup item, and the current PR still drops the per-channel override on the mention-pattern reroute path while also lacking real Slack behavior proof.

## What This Changes

The PR adds `replyToMode` to Slack per-channel config/schema/types/docs and uses the resolved channel setting ahead of account or chat-type fallback for initial Slack routing.

## Best Possible Solution

Carry the resolved Slack channel config through every routing resolution, add coverage for the mention-pattern reroute path, and merge only after maintainers accept the new config surface and real Slack proof is added.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR diff and current routing path shows the first routing call gets `channelConfig` but the later regex-mention reroute still omits it. A live Slack reproduction was not run in this read-only review.

## Solution Assessment

No. The proposed direction is close, but the complete solution should pass `channelConfig` into the reroute path and add regression coverage for a mention-pattern channel with per-channel `replyToMode`.
