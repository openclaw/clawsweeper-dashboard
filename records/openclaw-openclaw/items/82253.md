## Summary

Keep open: the update fixes the earlier regex-reroute gap and supplies acceptable terminal proof, but the new Slack per-channel reply mode still does not reach message-tool auto-threading, so the advertised override is incomplete.

## What This Changes

The PR adds Slack per-channel `replyToMode` schema, types, docs, generated metadata, tests, and inbound routing support, including the regex mention reroute path.

## Best Possible Solution

Carry the effective per-channel mode into every Slack reply-threading consumer, especially the message-tool threading context, with focused regression coverage.

## Reproduction Assessment

Yes for the review finding: configure a Slack channel override of `replyToMode: "off"` with account/channel fallback `all`, then invoke a same-channel Slack message tool from that turn. Source inspection shows the tool context will still use the account/chat-type mode.

## Solution Assessment

No. The inbound routing fix is the right shape, but the implementation is incomplete until Slack message-tool auto-threading honors the same channel override.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Apply channel reply mode to message tools:** `src/config/types.slack.ts:38`
