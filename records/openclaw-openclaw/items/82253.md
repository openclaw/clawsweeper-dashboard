## Summary

Keep open: current main does not have per-channel Slack `replyToMode`, and this PR fixes the initial preparation/reroute path, but it still leaves a concrete propagation gap for Slack tool/follow-up reply paths.

## What This Changes

Adds a Slack per-channel `replyToMode` config/schema/docs surface and uses resolved channel config when preparing Slack routing, including regex-mention reroutes.

## Best Possible Solution

Merge a version that treats the resolved per-channel Slack `replyToMode` as the mode for prepared replies, Slack tool context, and queued reply delivery, with schema/docs/generated metadata aligned.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR resolves the override for preparation but the unchanged Slack tool/follow-up context still re-resolves account-level mode. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No as submitted. The PR is close, but the maintainable solution should carry the resolved reply mode into the turn context and downstream Slack reply helpers so every Slack reply path follows the per-channel override.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Carry reply mode into Slack turn context:** `extensions/slack/src/monitor/message-handler/prepare-routing.ts:182`
  - body: The PR resolves `channelConfig?.replyToMode` only inside preparation. Carry the resolved mode into the turn context and have the Slack threading/follow-up paths prefer it.
