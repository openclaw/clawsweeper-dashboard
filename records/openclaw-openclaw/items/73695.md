## Summary

Keep open: current main still lacks WhatsApp live commentary, and this PR is a useful revival of closed PR #57484, but the proposed bounded delivered-commentary tracking can duplicate already-delivered long commentary once the stored snapshot truncates.

## What This Changes

The PR adds opt-in WhatsApp live assistant commentary delivery, wiring callback support through the embedded runner/subscriber, WhatsApp delivery, config/schema/docs/changelog, and targeted tests.

## Best Possible Solution

Keep the opt-in WhatsApp-only design, but make delivered-commentary tracking remember or verify enough bounded metadata for long segments to avoid duplicate delivery without dropping changed text, then run the focused agent/WhatsApp tests and changed gate.

## Reproduction Assessment

Yes. On the PR diff, deliver a commentary segment longer than 8,000 characters, then append a suffix: `resolveAssistantCommentaryDeltaText` slices from the stored 8,000-character prefix instead of the recorded delivered length, so the already-delivered tail is sent again.

## Solution Assessment

No as-is. The callback architecture and current `message_tool_only` gate are maintainable, but the long-segment dedupe needs stronger bounded verification before this is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve long delivered commentary without resending it:** `src/agents/pi-embedded-commentary.ts:43-46`
  - body: `rememberDeliveredCommentarySegment` stores only the first 8,000 characters while recording the full delivered length, but this helper ignores any recorded length greater than the stored prefix. If a 15,000-character commentary segment later grows, the next delta starts at character 8,000 and resends already-delivered text before the new suffix, defeating the live/final dedupe guarantee. Store enough bounded verification metadata, such as a suffix/hash, to skip already-delivered text without dropping changed text.
  - confidence: 0.86
