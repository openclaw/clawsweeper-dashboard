## Summary

Keep open. Current main still suppresses assistant commentary-phase text for WhatsApp and has no `commentaryDelivery` config, so the PR is not obsolete; however, the proposed patch still has a concrete long-commentary dedupe bug that can resend already-delivered text after the stored 8,000-character prefix.

## What This Changes

The PR adds opt-in WhatsApp live assistant commentary delivery with channel/account config, embedded runner and subscribe callback plumbing, text-only WhatsApp delivery filtering, final-reply dedupe, abort/timeout handling, docs, changelog, tests, and Plugin SDK reply-runtime exports.

## Best Possible Solution

Land a corrected version of the PR that keeps the default-off, WhatsApp-only, text-only, source-policy-aware design, fixes the long-segment delivered-commentary math, adds focused regression coverage for that case, and then passes the targeted agent/WhatsApp tests plus the appropriate changed gate before merge.

## Reproduction Assessment

Yes. The current-main gap is provable from source by the commentary-phase early return, and the PR regression is reproducible at helper level with a delivered segment longer than the stored prefix followed by a later current text that extends the same segment.

## Solution Assessment

No as-is. The proposed architecture is a reasonable narrow seam for opt-in WhatsApp live commentary, but the dedupe helper must use the recorded delivered length for true extensions of a truncated stored prefix before this is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor delivered lengths for extended commentary:** `src/agents/pi-embedded-commentary.ts:44-46`
  - body: `rememberDeliveredCommentarySegment` stores only the first 8,000 characters of a delivered segment while recording the full delivered length. When a later update extends that same long segment, this check rejects the recorded length because it is greater than the stored prefix length, so the delta starts at 8,000 and resends text the user already received. Trust the recorded length when the current text still starts with the stored prefix and extends beyond that delivered length, and add a long-commentary regression test.
  - confidence: 0.88
