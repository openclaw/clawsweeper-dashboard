## Summary

Keep open, but do not merge as-is. The PR addresses a real static attribution gap, yet the proposed wrapper mutates payloads for every provider when `user` is configured, while the adjacent payload-patch pattern is API-family guarded and covered by unsupported-provider tests.

## What This Changes

The PR adds a Pi embedded runner wrapper that normalizes `ctx.effectiveExtraParams.user` and injects it into outgoing provider payloads.

## Best Possible Solution

Land a narrow static `params.user` passthrough only for supported OpenAI/OpenRouter-compatible API families, with regression tests and a changelog entry; keep dynamic trace/session metadata design in #63901.

## Reproduction Assessment

Yes. A focused unit case can configure `agents.defaults.params.user`, run the PR wrapper with an `anthropic-messages` model, and observe the added top-level `user` key; I did not execute it because this review is read-only.

## Solution Assessment

No. The requested capability is reasonable, but this implementation is not the best fix because it applies globally; the maintainable path is an API-family guard plus supported and unsupported payload tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate user payload injection by supported API:** `src/agents/pi-embedded-runner/extra-params.ts:329-348`
  - body: With a global `agents.defaults.params.user`, this wrapper patches every provider payload. Non-OpenAI-family routes such as `anthropic-messages` and native Gemini payloads will receive an unexpected top-level `user` key, which can leak an attribution identifier and cause provider request rejection. Return the underlying stream unchanged unless the model API supports this field.
  - confidence: 0.9

  - body: This is a user-facing feature because it makes a new provider request-body parameter work from agent config, but the PR only changes runtime code. Add a single-line `CHANGELOG.md` entry under the active version before merge.
  - confidence: 0.78
