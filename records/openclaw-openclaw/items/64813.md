## Summary

Keep open. The static attribution feature is useful, but this PR is not mergeable as written because it injects configured `user` values into every provider payload, has no focused payload coverage, and lacks contributor real behavior proof.

## What This Changes

The PR adds a stream wrapper that normalizes `params.user` and injects it into provider request payloads through `streamWithPayloadPatch`.

## Best Possible Solution

Land a narrow static `params.user` passthrough only for explicitly compatible request APIs, with supported and unsupported payload-shape tests; leave dynamic session metadata design in https://github.com/openclaw/openclaw/issues/63901.

## Reproduction Assessment

Yes for the PR defect by source inspection: the added wrapper is applied whenever `effectiveExtraParams.user` is present, and `streamWithPayloadPatch` mutates any object payload. I did not run tests because this review is read-only.

## Solution Assessment

No. The requested static attribution capability is reasonable, but this implementation needs a supported-API guard, focused payload tests, and contributor real behavior proof before it is mergeable.
