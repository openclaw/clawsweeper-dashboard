## What This Changes

The PR restores QR/setup-code bootstrap handoff by returning a primary node token plus a bounded operator token, updates Android/iOS handoff scope filtering, and refreshes gateway docs and focused tests.

## Best Possible Solution

Land the bounded QR bootstrap handoff only after the restored-token retry path also re-emits the operator handoff and maintainers explicitly accept the setup-code operator-token trust model.

## Reproduction Assessment

Yes, source-reproducible. Current main defines setup-code bootstrap as node-only and the native clients only persist operator handoff tokens from trusted bootstrap `hello-ok.auth.deviceTokens`.

## Solution Assessment

Not yet. Restoring the bounded handoff matches the stated native contract, but the patch needs the retry-path fix and explicit maintainer acceptance of the QR operator-token boundary before it is the best merge path.
