## Summary

Keep open. The PR is a narrow task-audit behavior change, but it still leaves operator docs and the changelog out of sync with the new warning-only `lost` severity policy.

## What This Changes

This PR changes task audit so all `lost` findings are warnings, adds a 1-second tolerance for `startedAt < createdAt`, and updates the colocated audit tests.

## Best Possible Solution

Land one coherent audit-policy update where the code, tests, public task docs, CLI docs, and changelog all describe the same `lost` severity contract.

## Reproduction Assessment

Yes. A direct `listTaskAuditFindings` input with a `lost` task whose `cleanupAfter` is expired or missing reproduces the severity change, and a raw task with `startedAt` slightly before `createdAt` exercises the new timestamp-tolerance branch.

## Solution Assessment

No, not yet. The code change is narrow, but the PR should either update docs and changelog to match warning-only lost findings or preserve the existing expired-lost error contract.
