## Summary

Keep this PR open: it still changes expired or unstamped `lost` task audit findings from documented errors into warnings, while current code, tests, docs, and the related merged work in https://github.com/openclaw/openclaw/pull/71871 preserve that warn/error contract. It also still lacks after-fix real behavior proof.

## What This Changes

The PR adds a 1-second tolerance for small `startedAt`/`createdAt` skew and changes all `lost` task audit findings to warnings, with colocated test updates.

## Best Possible Solution

Land one coherent task-audit policy update: either keep expired/unstamped lost tasks as errors and limit this PR to timestamp-race tolerance, or update code, tests, docs, and real proof together after maintainer agreement on warning-only lost findings.

## Reproduction Assessment

Yes, from source inspection. Direct `listTaskAuditFindings` inputs exercise both changed paths: an expired or unstamped `lost` task shows the severity change, and a task with `startedAt` slightly before `createdAt` exercises the tolerance branch.

## Solution Assessment

No, not as proposed. The timestamp tolerance is narrow, but unconditional warning severity for all `lost` tasks conflicts with the current public contract unless the docs and operator policy change with it.
