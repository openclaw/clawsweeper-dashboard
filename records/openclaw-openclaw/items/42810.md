## Summary

Keep open: current main does not have ACP audit logging, and this external XL security feature still has blocking correctness/security issues plus missing real behavior proof; it needs maintainer design review rather than cleanup closure.

## What This Changes

The PR adds an ACP control-plane audit module with file/null loggers, session init/close integration, tests, and small config/schema edits.

## Best Possible Solution

Design a smaller opt-in ACP audit path with explicit config, private storage, deterministic retention/query semantics, best-effort sink failure policy, focused tests, and redacted live proof.

## Reproduction Assessment

Yes for the PR review blockers: source inspection of PR head shows normal ACP construction would use a null sink, initializeSession would drop current runtimeOptions, and FileAuditLogger has storage, buffering, query, and prune gaps. I did not run tests or a live ACP session in this read-only review.

## Solution Assessment

No. Audit logging is a reasonable direction, but this branch is not the best mergeable shape until production wiring, explicit security policy, hardened storage defaults, lifecycle isolation, and real behavior proof are in place.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire audit logging into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: DEFAULT_DEPS installs createNullAuditLogger(), and the production singleton still constructs new AcpSessionManager() without custom deps. Normal ACP init/close calls therefore audit to a no-op sink, so the advertised security trail is never persisted unless a custom caller injects a real logger.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252-266`
  - body: This rebuilds initialRuntimeOptions from only { cwd: input.cwd } and passes only cwd into ensureSession. Current main forwards input.runtimeOptions values such as model and thinking, so ACP spawns would silently lose caller-selected runtime options.
  - confidence: 0.95
- **[P2] Keep audit sink failures from aborting sessions:** `src/acp/control-plane/manager.core.ts:347-358`
  - body: The lifecycle finally blocks await auditLogger.log(). If an injected sink rejects, a successful init/close can fail or an original control-plane error can be replaced by the audit error; audit emission should be best-effort or explicitly policy-controlled.
  - confidence: 0.9
- **[P2] Create audit files with private permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink creates directories and appends JSONL records with default filesystem modes. Records can include actor identifiers, client IPs, session keys, and error text, so the sink should create directories/files with explicit private permissions.
  - confidence: 0.89
- **[P2] Await storage setup before first flush:** `src/acp/control-plane/audit/audit-logger.file.ts:61`
  - body: Directory creation is kicked off asynchronously in the constructor, but an immediate flush or close can reach appendFile before mkdir completes and fail with ENOENT on a fresh storage path.
  - confidence: 0.84
- **[P2] Cap buffered entries after repeated flush failures:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: When appendFile fails, flush() requeues the whole batch with unshift() while log() continues pushing new entries. Sustained disk or permission failures can grow memory far beyond the advertised maxBufferSize.
  - confidence: 0.88
- **[P2] Schedule retention pruning for file loggers:** `src/acp/control-plane/audit/audit-logger.file.ts:48-56`
  - body: retentionDays is configured and prune() exists, but the constructor only schedules periodic flushes. Production file loggers therefore do not enforce the advertised 90-day retention policy unless an external caller invokes pruning.
  - confidence: 0.86
- **[P2] Synchronize queries with in-flight flushes:** `src/acp/control-plane/audit/audit-logger.file.ts:225-239`
  - body: flush() removes entries from this.buffer before appendFile completes, while query() only scans the buffer and completed files. A query during that write window can miss entries, and a query racing a flush can duplicate entries across memory and disk.
  - confidence: 0.86
- **[P2] Avoid success records for no-op closeSession calls:** `src/acp/control-plane/manager.core.ts:1219-1233`
  - body: The finally block always emits SESSION_CLOSE, but closeSession can return early with requireAcpSession: false when the session is already missing or stale. Cleanup flows would then record successful closes for sessions that did not exist or were already gone.
  - confidence: 0.82
- **[P2] Return pruned entry counts instead of file counts:** `src/acp/control-plane/audit/audit-logger.file.ts:289`
  - body: The interface says prune() returns the number of entries pruned, but this increments once per deleted JSONL file. Any caller using the count for audit reporting or compliance metrics would get a misleading value.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive audit surface, but the audit trail is inert on normal ACP paths and the file sink lacks explicit storage, retention, and failure-policy hardening.

Concerns:

- **[medium] Audit trail is inert on normal ACP paths:** `src/acp/control-plane/manager.types.ts:149`
  - body: DEFAULT_DEPS uses NullAuditLogger and the production manager singleton does not inject a real sink, so the new security audit trail would not be collected for normal users.
  - confidence: 0.95
- **[medium] Audit records may be stored too broadly:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: FileAuditLogger relies on default mkdir/appendFile modes for records that can include actor identifiers, client IPs, session keys, and error text; private directory and file modes should be explicit.
  - confidence: 0.89
- **[low] Sink failures can affect control-plane availability:** `src/acp/control-plane/manager.core.ts:347`
  - body: Lifecycle paths await the audit sink in finally blocks, so a rejecting injected logger can turn otherwise successful session operations into failures or mask the original error.
  - confidence: 0.86
- **[medium] Audit buffering and retention are not hardened:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: Repeated flush failures can grow the in-memory buffer beyond the advertised cap, and retention pruning is not scheduled for production file loggers.
  - confidence: 0.84
