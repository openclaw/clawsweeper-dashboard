## Summary

Keep open: the audit direction is useful, but this PR is not merge-ready because normal ACP still defaults to a null audit sink, current initializeSession runtime-option behavior regresses, and the file sink needs security/storage policy plus real behavior proof.

## What This Changes

The branch adds ACP audit event types, file/null audit loggers and tests, injects session init/close audit calls into AcpSessionManager, and includes small config comment/schema reorder changes.

## Best Possible Solution

Land a smaller ACP audit design with explicit opt-in configuration, private storage and retention semantics, best-effort sink isolation, and regression coverage that preserves current runtime-option behavior.

## Reproduction Assessment

Yes. Source inspection of current main and the PR head is enough to reproduce the merge blockers: normal ACP uses a null sink, initializeSession drops runtimeOptions, and the file sink has unsafe audit semantics; I did not run tests because this was a read-only review.

## Solution Assessment

No. The audit capability is a reasonable direction, but this branch is not the best mergeable shape because it adds a security-sensitive audit surface without production wiring, explicit policy, safe storage defaults, or behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire audit logging into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` installs `createNullAuditLogger()`, and the production singleton still constructs `new AcpSessionManager()` without custom deps. Normal ACP init/close calls therefore audit to a no-op sink, so the advertised security trail is never persisted unless a test or custom caller manually injects a real logger.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252-266`
  - body: This rebuilds `initialRuntimeOptions` from only `{ cwd: input.cwd }` and passes only `cwd` into `ensureSession`, dropping `input.runtimeOptions` values such as `model` and `thinking` that current main forwards and tests. ACP sessions initialized with those options would silently lose them.
  - confidence: 0.95
- **[P2] Isolate audit sink failures from session lifecycle:** `src/acp/control-plane/manager.core.ts:347`
  - body: The lifecycle `finally` blocks await `auditLogger.log()`. If an injected audit sink rejects, a successful init/close can fail or an original control-plane error can be replaced by the audit error; audit emission should be best-effort or isolated from lifecycle results.
  - confidence: 0.9
- **[P2] Create audit logs with private permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink creates the audit directory and appends JSONL records with default filesystem modes. These records can include actor identifiers, client IPs, session keys, and error text, so the sink should create directories/files with explicit private permissions.
  - confidence: 0.89
- **[P2] Cap buffered entries when flushes keep failing:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: When `appendFile` fails, `flush()` requeues the batch with `unshift()` while `log()` continues pushing new entries. Sustained disk or permission failures can grow memory far beyond the advertised `maxBufferSize` instead of shedding or rejecting entries predictably.
  - confidence: 0.88
- **[P2] Schedule retention pruning for production loggers:** `src/acp/control-plane/audit/audit-logger.file.ts:47-56`
  - body: `retentionDays` is configured and `prune()` exists, but the constructor only schedules periodic flushes. Production file loggers therefore do not enforce the advertised 90-day retention policy unless some external caller remembers to invoke pruning.
  - confidence: 0.86
- **[P2] Keep in-flight flush entries visible to queries:** `src/acp/control-plane/audit/audit-logger.file.ts:225-239`
  - body: `flush()` removes entries from `this.buffer` before awaiting `appendFile`, and `query()` only scans the buffer and completed files. A query during that write window can miss entries that are neither buffered nor on disk, which is a correctness problem for audit lookups.
  - confidence: 0.86
- **[P2] Avoid success audit records for no-op closeSession calls:** `src/acp/control-plane/manager.core.ts:1219`
  - body: The `finally` audit block runs even after `closeSession({ requireAcpSession: false })` returns early for a missing or stale session. Cleanup flows would then record a successful `SESSION_CLOSE` for a session that did not exist or was already gone.
  - confidence: 0.82
- **[P3] Align prune counts with the logger contract:** `src/acp/control-plane/audit/audit-logger.file.ts:288-289`
  - body: `IAuditLogger.prune()` documents an entry count, but this implementation increments once per deleted file. Any caller using the return value for audit metrics or compliance reporting will get a misleading count.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds security-sensitive audit logging, but normal ACP still uses a null sink and the file sink needs private storage, bounded failure behavior, and explicit policy before merge.

Concerns:

- **[medium] Audit trail is inert on normal ACP paths:** `src/acp/control-plane/manager.types.ts:149`
  - body: The default ACP manager uses `NullAuditLogger`, and the production singleton does not inject a real sink, so the new security audit trail is not collected for normal users.
  - confidence: 0.95
- **[medium] Audit records may be stored too broadly:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: `FileAuditLogger` creates directories and appends JSONL through default filesystem modes even though entries may contain actor identifiers, client IPs, session keys, and error text.
  - confidence: 0.89
- **[low] Sink failures can affect control-plane availability:** `src/acp/control-plane/manager.core.ts:347`
  - body: Lifecycle paths await the audit sink in `finally`; a rejecting injected logger can turn otherwise successful session operations into failures or mask the original control-plane error.
  - confidence: 0.86
- **[medium] Audit buffering and retention are not hardened:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: Repeated flush failures can grow the in-memory buffer beyond the advertised cap, and retention pruning is not scheduled for production file loggers.
  - confidence: 0.84
