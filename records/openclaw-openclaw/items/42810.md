## Summary

Keep open: current main does not provide this ACP audit capability, but this branch is not mergeable as-is because normal ACP paths still use a null sink, initializeSession regresses runtime options, and the security-sensitive file logger lacks hardened storage, buffering, query, and proof semantics.

## What This Changes

Adds a new ACP audit module with file/null loggers, session init/close audit calls, tests, and two small unrelated config/schema comment/order edits.

## Best Possible Solution

Land a smaller opt-in ACP audit design with explicit configuration, private storage defaults, defined query/retention/failure semantics, regression coverage, and redacted real-run proof.

## Reproduction Assessment

Yes for the PR review blockers: PR-head source inspection shows the normal manager uses NullAuditLogger, initializeSession drops current runtimeOptions, and FileAuditLogger has storage, buffering, query, and prune gaps. No runtime tests were run because this was a read-only review.

## Solution Assessment

No. Audit logging is a reasonable direction, but this branch is not the best mergeable shape until production wiring, explicit security policy, hardened storage defaults, lifecycle isolation, and real behavior proof are in place.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire audit logging into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` installs `createNullAuditLogger()`, and the production singleton still constructs `new AcpSessionManager()` without custom deps. Normal ACP init/close paths therefore write to a no-op sink, so the advertised security audit trail is never persisted unless a custom caller injects a real logger.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252-266`
  - body: This rebuilds `initialRuntimeOptions` from only `{ cwd: input.cwd }` and calls `ensureSession` without model/thinking. Current main forwards `input.runtimeOptions` and has tests for that contract, so ACP spawns would silently lose caller-selected runtime options.
  - confidence: 0.95
- **[P2] Isolate audit sink failures from session lifecycle:** `src/acp/control-plane/manager.core.ts:347-358`
  - body: The lifecycle `finally` blocks await `auditLogger.log()`. If an injected sink rejects, a successful init/close can fail or an original control-plane error can be replaced by the audit error; audit emission should be best-effort or explicitly policy-controlled.
  - confidence: 0.9
- **[P2] Create audit files with private permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink creates directories and appends JSONL records with filesystem defaults. Records can include actor identifiers, client IPs, session keys, and error text, so directories/files should be created with explicit private permissions.
  - confidence: 0.89
- **[P2] Cap buffered entries after repeated flush failures:** `src/acp/control-plane/audit/audit-logger.file.ts:243-246`
  - body: When `appendFile` fails, `flush()` requeues the whole batch with `unshift()` while `log()` continues pushing new entries. Sustained disk or permission failures can grow memory far beyond the advertised `maxBufferSize`.
  - confidence: 0.88
- **[P2] Synchronize queries with in-flight flushes:** `src/acp/control-plane/audit/audit-logger.file.ts:225-239`
  - body: `flush()` removes entries from `this.buffer` before `appendFile` completes, while `query()` only scans the buffer and completed files. A query during that write window can miss entries, and a query racing a flush can duplicate entries across memory and disk.
  - confidence: 0.86
- **[P2] Return pruned entry counts, not file counts:** `src/acp/control-plane/audit/audit-logger.file.ts:266-297`
  - body: The interface says `prune()` returns the number of entries pruned, but the implementation increments once per deleted daily file. Any metrics or compliance reporting built on that value would undercount removals.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: Needs attention because the patch adds a security audit trail but leaves normal ACP paths no-op and file sink storage/failure semantics underdefined.

Concerns:

- **[medium] Audit trail is inert on normal ACP paths:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` uses NullAuditLogger and the production manager singleton does not inject a real sink, so the new security audit trail would not be collected for normal users.
  - confidence: 0.95
- **[medium] Audit records may be stored too broadly:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: FileAuditLogger relies on default mkdir/appendFile permissions for records that can include actor identifiers, client IPs, session keys, and error text; private directory and file modes should be explicit.
  - confidence: 0.89
- **[medium] Flush failure can grow memory without a cap:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: Failed flushes requeue all pending entries and new log calls keep appending, so a disk or permission outage can turn audit logging into unbounded process memory growth.
  - confidence: 0.88
