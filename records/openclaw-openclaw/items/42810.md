## Summary

Keep open. Current main still has no ACP control-plane audit logger or audit dependency seam, so the requested capability is not implemented, but this PR is not merge-ready because the proposed audit path is ineffective by default and has reliability/storage issues that are especially important for a security audit feature. The existing Greptile and ClawSweeper review comments are useful context and align with the remaining file-logger and default-sink concerns.

## What This Changes

The PR adds an ACP control-plane audit module with file/null loggers, audit types/utilities/tests, AcpSessionManager audit calls for session initialization and close, plus small incidental config comment/schema ordering changes.

## Best Possible Solution

Keep the work open, but require a rebased and narrower ACP-owned audit implementation that writes to the OpenClaw state/log directory with restrictive permissions, treats audit emission as best-effort, bounds failure behavior, defines retention/query semantics precisely, and covers session init, close, cancel, run, and runtime-control events with focused tests.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The review has a high-confidence verification path: current-main source inspection shows no ACP audit module or audit dependency seam, while the PR diff and review discussion show the proposed implementation remains incomplete.

## Solution Assessment

No. The requested direction is plausible, but this branch is not the best mergeable form because normal ACP paths still default to a null sink and the file logger is not robust enough for a security audit trail.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Replace the default audit logger with a real sink:** `src/acp/control-plane/manager.types.ts:146`
  - body: DEFAULT_DEPS binds auditLogger to createNullAuditLogger(), and the production singleton still constructs AcpSessionManager with default dependencies. Normal ACP initialize/close paths therefore call the new audit seam but persist no audit trail unless a custom test/dependency injection path supplies a logger.
  - confidence: 0.96
- **[P2] Make audit emission best-effort:** `src/acp/control-plane/manager.core.ts:347`
  - body: initializeSession and closeSession await auditLogger.log() inside lifecycle cleanup paths. If the audit sink rejects, that rejection can override a successful lifecycle operation or mask the original failure, so an unhealthy audit sink can break ACP session management.
  - confidence: 0.88
- **[P2] Enforce a hard cap after flush failures:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: flush() requeues failed batches with unshift(), while log() continues accepting entries and only schedules more asynchronous flushes. A sustained disk or permission failure can grow memory without bound instead of honoring the advertised buffer limit.
  - confidence: 0.9
- **[P2] Store audit logs with private state-directory permissions:** `src/acp/control-plane/audit/audit.types.ts:163`
  - body: The file sink defaults to a relative .audit directory and does not follow the existing config audit helper's state-dir, 0700 directory, and 0600 file pattern. Actor/session audit records should be treated as sensitive operational data.
  - confidence: 0.84
