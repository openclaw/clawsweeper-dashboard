## Summary

Keep open: the ACP audit logging direction is useful, but this PR still has blocking correctness and security issues, and it lacks real behavior proof from an actual ACP/control-plane run.

## What This Changes

The PR adds ACP audit types, file/null audit loggers and tests, injects audit hooks into AcpSessionManager session init/close, and makes two small config/schema edits.

## Best Possible Solution

Land a smaller ACP audit design with explicit configuration, private storage, retention and sink-failure policy, and focused regression coverage while preserving current session runtime-option behavior.

## Reproduction Assessment

Yes. Source inspection of current main plus the PR head shows the null default sink, dropped initializeSession runtime options, and unsafe file sink semantics; I did not run tests because this was a read-only review.

## Solution Assessment

No. The audit direction is useful, but this branch is not the best mergeable shape because it is inert on normal ACP paths, regresses established runtime-option behavior, and needs audit policy decisions first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire audit logging into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` installs `createNullAuditLogger()`, and the production singleton still constructs `new AcpSessionManager()` with no deps. Normal users therefore get session init/close calls that audit to a no-op sink, so the advertised security audit trail is never persisted.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252-266`
  - body: This rebuilds `initialRuntimeOptions` from `{ cwd: input.cwd }` and only passes `cwd` to `ensureSession`, dropping `input.runtimeOptions` values such as model and thinking that current main persists and forwards.
  - confidence: 0.95
- **[P2] Isolate audit sink failures from session lifecycle:** `src/acp/control-plane/manager.core.ts:347`
  - body: The lifecycle `finally` blocks await `auditLogger.log()`. If an injected audit sink rejects, a successful init/close can fail or an original control-plane error can be replaced by the audit error; audit emission should be best-effort or isolated.
  - confidence: 0.9
- **[P2] Create audit logs with private permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink creates the audit directory and appends JSONL records with default modes. These records can include actor data, client IPs, session keys, and error text, so the sink should create the directory/files with private permissions.
  - confidence: 0.89
- **[P2] Cap buffered entries when flushes keep failing:** `src/acp/control-plane/audit/audit-logger.file.ts:97-100`
  - body: When `appendFile` fails, `flush()` requeues the batch with `unshift()` while `log()` continues pushing new entries. Sustained disk or permission failures can grow memory far beyond the advertised `maxBufferSize`.
  - confidence: 0.88
- **[P2] Schedule retention pruning:** `src/acp/control-plane/audit/audit-logger.file.ts:48-56`
  - body: `retentionDays` is configured and `prune()` exists, but the constructor only schedules periodic flushes. Production file loggers therefore do not enforce the advertised 90-day retention policy.
  - confidence: 0.86
- **[P3] Align prune counts with the logger contract:** `src/acp/control-plane/audit/audit-logger.file.ts:288-289`
  - body: `IAuditLogger.prune()` documents an entry count, but this implementation increments once per deleted file. Any caller using the return value for audit metrics or compliance reporting will get a misleading count.
  - confidence: 0.82
