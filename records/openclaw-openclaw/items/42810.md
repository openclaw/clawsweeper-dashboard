## Summary

Keep this PR open for maintainer review: current main still lacks ACP audit logging, but the branch is a stale XL security feature and the latest diff has blockers that would make the audit path inert and regress existing ACP session/task behavior.

## What This Changes

The PR adds ACP control-plane audit types, file/null logger implementations, query/prune tests, and session init/close audit hooks in `AcpSessionManager`, with small unrelated config-schema/comment edits.

## Best Possible Solution

Land a smaller ACP audit design with explicit config, private storage, retention/failure policy, and focused tests while preserving current runtime-options and task lifecycle behavior.

## Reproduction Assessment

Yes for the PR blockers: source inspection of current main plus the PR diff shows the inert default sink, dropped runtime options, missing ACP task bookkeeping, and unsafe audit-file storage. I did not run tests because this was a read-only review and `pnpm` was unavailable.

## Solution Assessment

No. The audit direction is useful, but this branch is not the best mergeable shape because it is inert on normal ACP paths, regresses established session/task behavior, and needs audit policy decisions first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire the audit logger into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` installs `createNullAuditLogger()`, and the normal singleton manager is constructed with no deps, so session init/close calls record to a no-op sink unless a test manually injects a file logger.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252`
  - body: This rebuilds `initialRuntimeOptions` from `{ cwd: input.cwd }` only, dropping `input.runtimeOptions`. Current main forwards model/thinking/cwd into `runtime.ensureSession` and persists them, so this would regress ACP model/thinking selection.
  - confidence: 0.95
- **[P2] Keep ACP task bookkeeping in runTurn:** `src/acp/control-plane/manager.core.ts:655-657`
  - body: The PR starts the turn loop without the current `taskContext` and `createBackgroundTaskRecord` path. Current docs and code require parented ACP prompt runs to create and update task ledger records.
  - confidence: 0.91
- **[P2] Do not let audit sink failures break sessions:** `src/acp/control-plane/manager.core.ts:347`
  - body: `initializeSession` and `closeSession` await `auditLogger.log()` from lifecycle cleanup paths. If an injected logger rejects, a successful session operation can fail or the original control-plane error can be replaced by an audit-sink error.
  - confidence: 0.9
- **[P2] Create audit logs with private filesystem permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink uses default directory/file modes for audit entries that may include actor data, client IPs, session keys, and error text. Create the audit directory and JSONL files with private permissions.
  - confidence: 0.88
- **[P2] Cap the buffer when flushes keep failing:** `src/acp/control-plane/audit/audit-logger.file.ts:97`
  - body: `log()` keeps pushing entries while failed `flush()` calls requeue the whole batch with `unshift()`, so sustained disk or permission failures can grow memory far beyond `maxBufferSize`.
  - confidence: 0.87
- **[P2] Make retentionDays actually enforce retention:** `src/acp/control-plane/audit/audit-logger.file.ts:47-57`
  - body: The constructor schedules periodic flushes but never schedules `prune()`, so `retentionDays` has no production effect despite the advertised automatic 90-day retention.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff adds a security-sensitive audit feature, but the normal ACP path remains no-op and the file sink does not enforce private log storage.

Concerns:

- **[medium] Audit control is inert on normal ACP paths:** `src/acp/control-plane/manager.types.ts:149`
  - body: The default ACP manager uses `NullAuditLogger`, and no production construction path injects `FileAuditLogger`, so the new security audit trail is not collected for normal users.
  - confidence: 0.94
- **[medium] Audit records may be stored with broad permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: `FileAuditLogger` writes JSONL audit records through default filesystem modes. Because entries can contain actor identifiers, client IPs, session keys, and error text, the sink should create storage with private permissions.
  - confidence: 0.88
