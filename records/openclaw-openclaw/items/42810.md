## Summary

Keep open: ACP audit logging is a plausible security feature, but this PR is not mergeable as-is because normal ACP startup still uses a no-op audit sink, session initialization would drop current runtime options, real behavior proof is missing, and the file audit sink has unresolved security and correctness gaps.

## What This Changes

The branch adds file/null ACP audit loggers, audit types/utilities/tests, and session init/close audit calls in the ACP control-plane manager, with small adjacent config comment/schema-order drive-bys.

## Best Possible Solution

Land a smaller opt-in ACP audit design with explicit OpenClaw configuration, private storage defaults, defined retention/query/failure semantics, focused regression tests, and redacted real-run proof.

## Reproduction Assessment

Yes for the PR review blockers: source inspection of PR head shows the default no-op sink, dropped runtimeOptions, awaited audit sink, and file logger storage/buffering/query/prune gaps. No runtime test was run in this read-only review.

## Solution Assessment

No. Audit logging is a reasonable direction, but this branch is not the narrowest maintainable solution until production wiring, explicit security policy, hardened file semantics, lifecycle isolation, and real behavior proof are in place.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire audit logging into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` installs `createNullAuditLogger()`, and the production singleton still creates `new AcpSessionManager()` without custom deps. Normal ACP init/close paths therefore write to a no-op sink, so the advertised security audit trail is never persisted unless a custom caller injects a real logger.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252`
  - body: This rebuilds `initialRuntimeOptions` from only `{ cwd: input.cwd }`, dropping caller-supplied model and thinking values. Current main forwards those options to `ensureSession` and has tests for that contract, so existing ACP spawns would silently lose selected runtime options.
  - confidence: 0.95
- **[P2] Isolate audit sink failures from session lifecycle:** `src/acp/control-plane/manager.core.ts:347-358`
  - body: The lifecycle `finally` block awaits `auditLogger.log()`. If an injected sink rejects, a successful init can fail or an original control-plane error can be replaced by the audit error; audit emission should be best-effort or explicitly policy-controlled.
  - confidence: 0.9
- **[P2] Create audit files with private permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink creates directories and appends JSONL records with filesystem defaults. Records can include actor identifiers, client IPs, session keys, and error text, so directories/files should be created with explicit private permissions.
  - confidence: 0.89
- **[P2] Cap buffered entries after repeated flush failures:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: When `appendFile` fails, `flush()` requeues the whole batch with `unshift()` while `log()` continues pushing new entries. Sustained disk or permission failures can grow memory far beyond the advertised max buffer size.
  - confidence: 0.88
- **[P2] Synchronize and dedupe query reads:** `src/acp/control-plane/audit/audit-logger.file.ts:225-239`
  - body: `flush()` removes entries from `this.buffer` before `appendFile` completes, while `query()` scans the current buffer and completed files without an in-flight set or dedupe by id. Queries can miss or duplicate events during normal periodic flushing.
  - confidence: 0.86
- **[P2] Return pruned entry counts, not file counts:** `src/acp/control-plane/audit/audit-logger.file.ts:287-289`
  - body: The interface says `prune()` returns the number of entries pruned, but this implementation increments once per deleted daily file. Metrics or compliance reporting built on that value would undercount removals.
  - confidence: 0.83
- **[P2] Enforce the advertised retention policy:** `src/acp/control-plane/audit/audit-logger.file.ts:47-57`
  - body: The constructor starts a periodic flush loop but never schedules `prune()`, so `retentionDays` is not enforced for production file loggers. Operators could accumulate audit files indefinitely despite the documented automatic pruning behavior.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention because the patch adds a security-sensitive audit trail while normal ACP paths remain no-op and the file sink has unresolved confidentiality and reliability semantics.

Concerns:

- **[medium] Audit trail is inert on normal ACP paths:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` uses NullAuditLogger and the production manager singleton does not inject a real sink, so the new security audit trail would not be collected for normal users.
  - confidence: 0.95
- **[medium] Audit records may be stored too broadly:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: FileAuditLogger relies on default mkdir/appendFile permissions for records that can include actor identifiers, client IPs, session keys, and error text; private directory and file modes should be explicit.
  - confidence: 0.89
- **[medium] Audit sink failure can break control-plane lifecycle:** `src/acp/control-plane/manager.core.ts:347`
  - body: Awaiting the audit sink in lifecycle finally blocks lets a logging/storage failure override session init or close behavior, which is risky for security telemetry unless that fail-closed policy is explicit and tested.
  - confidence: 0.86
