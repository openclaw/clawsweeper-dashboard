## Summary

Keep open: current main does not already implement ACP audit logging, and this PR still contains unique feature work, but the patch is not mergeable as-is because it changes security-sensitive ACP lifecycle behavior without production wiring, explicit policy, safe sink semantics, or real behavior proof.

## What This Changes

The branch adds ACP audit event types, file/null audit loggers and tests, injects session init/close audit calls into AcpSessionManager, and includes small unrelated config/schema churn.

## Best Possible Solution

Land a smaller opt-in ACP audit design with private storage, explicit retention and failure semantics, production wiring, regression tests, and redacted proof from a real ACP run.

## Reproduction Assessment

Yes. Source inspection of current main and the PR head reproduces the merge blockers: normal ACP uses a null sink, initializeSession drops runtimeOptions, and the file sink has unsafe storage, buffering, and retention semantics; I did not run tests because this was a read-only review.

## Solution Assessment

No. Audit logging is a reasonable direction, but this branch is not the best mergeable shape until production wiring, explicit security policy, safe storage defaults, and real behavior proof are in place.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire audit logging into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: DEFAULT_DEPS installs createNullAuditLogger(), and the production singleton still constructs new AcpSessionManager() without custom deps. Normal ACP init/close calls therefore audit to a no-op sink, so the advertised security trail is never persisted unless a custom caller injects a real logger.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252-266`
  - body: This rebuilds initialRuntimeOptions from only { cwd: input.cwd } and passes only cwd into ensureSession, dropping input.runtimeOptions values such as model and thinking that current main forwards and tests.
  - confidence: 0.95
- **[P2] Isolate audit sink failures from session lifecycle:** `src/acp/control-plane/manager.core.ts:347`
  - body: The lifecycle finally blocks await auditLogger.log(). If an injected sink rejects, a successful init/close can fail or an original control-plane error can be replaced by the audit error; audit emission should be best-effort or explicitly policy-controlled.
  - confidence: 0.9
- **[P2] Create audit logs with private permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink creates the audit directory and appends JSONL records with default filesystem modes. Entries may contain actor identifiers, client IPs, session keys, and error text, so the sink should create directories/files with explicit private permissions.
  - confidence: 0.89
- **[P2] Await storage setup before first flush:** `src/acp/control-plane/audit/audit-logger.file.ts:60-63`
  - body: Directory creation is kicked off asynchronously in the constructor, but an immediate flush or close can reach appendFile before mkdir completes and fail with ENOENT on a fresh storage path.
  - confidence: 0.84
- **[P2] Cap buffered entries when flushes keep failing:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: When appendFile fails, flush() requeues the whole batch with unshift() while log() continues pushing new entries. Sustained disk or permission failures can grow memory far beyond the advertised maxBufferSize.
  - confidence: 0.88
- **[P2] Schedule retention pruning for production loggers:** `src/acp/control-plane/audit/audit-logger.file.ts:47-56`
  - body: retentionDays is configured and prune() exists, but the constructor only schedules periodic flushes. Production file loggers therefore do not enforce the advertised 90-day retention policy unless an external caller remembers to invoke pruning.
  - confidence: 0.86
- **[P2] Synchronize queries with in-flight flushes:** `src/acp/control-plane/audit/audit-logger.file.ts:225-239`
  - body: flush() removes entries from this.buffer before appendFile completes, while query() only scans the buffer and completed files. A query during that write window can miss entries; a query racing a flush can also duplicate entries across memory and disk.
  - confidence: 0.86
- **[P2] Avoid success records for no-op closeSession calls:** `src/acp/control-plane/manager.core.ts:1219`
  - body: The closeSession finally block always emits SESSION_CLOSE, even after requireAcpSession: false returns early for a missing or stale session. Cleanup flows would then record successful closes for sessions that did not exist or were already gone.
  - confidence: 0.82
- **[P3] Align prune counts with the logger contract:** `src/acp/control-plane/audit/audit-logger.file.ts:289`
  - body: IAuditLogger.prune() documents an entry count, but this implementation increments once per deleted file. Any caller using the return value for audit metrics or compliance reporting will get a misleading count.
  - confidence: 0.82
- **[P3] Start the flush timer only when file auditing is enabled:** `src/acp/control-plane/audit/audit-logger.file.ts:48-56`
  - body: The constructor starts setInterval before checking config.enabled. A FileAuditLogger constructed with enabled: false still schedules periodic no-op callbacks, which contradicts the zero-overhead disabled-mode claim.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive audit surface, but the audit trail is inert on normal ACP paths and the file sink lacks explicit storage, retention, and failure-policy hardening.

Concerns:

- **[medium] Audit trail is inert on normal ACP paths:** `src/acp/control-plane/manager.types.ts:149`
  - body: DEFAULT_DEPS uses NullAuditLogger and the production manager singleton does not inject a real sink, so the new security audit trail would not be collected for normal users.
  - confidence: 0.95
- **[medium] Audit records may be stored too broadly:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: FileAuditLogger relies on default mkdir/appendFile modes for records that can include actor identifiers, client IPs, session keys, and error text; private directory and file modes should be explicit.
  - confidence: 0.89
- **[low] Sink failures can affect control-plane availability:** `src/acp/control-plane/manager.core.ts:347`
  - body: Lifecycle paths await the audit sink in finally blocks, so a rejecting injected logger can turn otherwise successful session operations into failures or mask the original error.
  - confidence: 0.86
- **[medium] Audit buffering and retention are not hardened:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: Repeated flush failures can grow the in-memory buffer beyond the advertised cap, and retention pruning is not scheduled for production file loggers.
  - confidence: 0.84
