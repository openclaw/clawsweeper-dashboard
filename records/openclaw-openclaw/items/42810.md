## Summary

Keep open. The PR is useful in direction, but it is not merge-ready against current main: the normal ACP manager path still defaults to a null audit sink, audit writes can change lifecycle behavior, and the branch is stale enough to drop current ACP runtime-option and reset/discard behavior.

## What This Changes

The PR adds a new ACP control-plane audit module with file and null loggers, tests, and optional `AcpSessionManager` logging for session init/close, plus two unrelated small config-file edits.

## Best Possible Solution

Design ACP audit logging as a real, configured control-plane feature: wire an enabled sink through the normal Gateway/ACP construction path, keep audit emission best-effort, capture actor/device provenance from existing gateway context, preserve all current ACP runtime option and reset semantics, document the config surface, and cover success/failure/disabled/concurrency cases with focused tests.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: compare the PR head files at `835c188dcc13407f4bc104f0fea23f973185800c` with current main at `d647ba1c6fb69c17debfa7fd5b3e69344bbfcba0`; the dropped runtime-option and discard-state code paths are directly visible without needing a runtime test.

## Solution Assessment

No. The proposed patch is not the best way to solve the issue yet because it adds audit types and calls but does not enable a real sink on normal paths, lets audit sink errors affect lifecycle calls, and regresses current ACP behavior; a rebase plus a narrower config-backed audit integration would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve ACP spawn runtime options:** `src/acp/control-plane/manager.core.ts:252-266`
  - body: Current main accepts `input.runtimeOptions` and forwards model/thinking into `runtime.ensureSession`, but this branch rebuilds `initialRuntimeOptions` from only `cwd`. Merging this would make ACP spawns ignore current runtime overrides such as model and thinking, breaking the documented `/acp model` and related controls.
  - confidence: 0.92
- **[P1] Keep discardPersistentState close handling:** `src/acp/control-plane/manager.core.ts:1166-1169`
  - body: The close wrapper drops current-main handling for `discardPersistentState`: it no longer skips runtime close for pending identities, passes the discard flag to the runtime, prepares a fresh session, or discards persisted runtime state. Reset/cleanup flows that depend on that flag can leave stale backend state behind after this merge.
  - confidence: 0.88
- **[P1] Wire the audit logger into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: The new default dependency is still `NullAuditLogger`, and the singleton manager is constructed without injecting a file logger or config-derived audit sink. As a result, normal ACP `initializeSession` and `closeSession` calls still produce no persisted audit trail, so the advertised security feature is effectively disabled outside tests or manual dependency injection.
  - confidence: 0.9
- **[P2] Make audit emission best-effort:** `src/acp/control-plane/manager.core.ts:347-358`
  - body: Both lifecycle methods await `this.auditLogger.log(...)` from a `finally` block. Any real sink that rejects can turn an otherwise successful session init/close into a control-plane failure or mask the original error, so audit logging should catch/log sink failures instead of changing lifecycle semantics.
  - confidence: 0.86
