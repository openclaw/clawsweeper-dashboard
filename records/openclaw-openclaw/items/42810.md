## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire the audit logger into normal ACP startup:** `src/acp/control-plane/manager.types.ts:149`
  - body: `DEFAULT_DEPS` still installs `createNullAuditLogger()`, and the singleton manager is constructed with no deps, so normal ACP init/close paths record to the no-op sink. The file logger is only reachable through manual injection, so the advertised audit trail is never persisted in production.
  - confidence: 0.96
- **[P1] Preserve initializeSession runtime options:** `src/acp/control-plane/manager.core.ts:252`
  - body: This rebuilds `initialRuntimeOptions` from `{ cwd: input.cwd }` only, dropping `input.runtimeOptions`. Current main forwards model/thinking/cwd into `runtime.ensureSession` and persists them, so merging this would regress ACP model/thinking selection and the existing manager tests around those options.
  - confidence: 0.95
- **[P2] Keep ACP task bookkeeping in runTurn:** `src/acp/control-plane/manager.core.ts:655-657`
  - body: The PR head starts the turn loop without the current `taskContext` and `createBackgroundTaskRecord` path. Current docs/tests require parented ACP prompt runs to create task ledger records, so this stale body would make ACP background runs disappear from task status/audit.
  - confidence: 0.91
- **[P2] Do not let audit sink failures break sessions:** `src/acp/control-plane/manager.core.ts:347`
  - body: `initializeSession` and `closeSession` await `auditLogger.log()` from `finally` blocks. If an injected logger rejects, successful lifecycle calls can fail or the original control-plane error can be replaced by an audit-sink error; audit emission needs an explicit best-effort wrapper or a documented fail-closed policy.
  - confidence: 0.9
- **[P2] Create audit logs with private filesystem permissions:** `src/acp/control-plane/audit/audit-logger.file.ts:239`
  - body: The file sink uses default `mkdir`/`appendFile` permissions for audit entries that can include actor data, client IPs, session keys, and error text. On permissive umasks these JSONL logs may be readable beyond the operator; use private directory/file modes consistent with existing config audit storage.
  - confidence: 0.88
- **[P2] Cap the buffer when flushes keep failing:** `src/acp/control-plane/audit/audit-logger.file.ts:97`
  - body: `log()` pushes entries unconditionally, while failed `flush()` calls requeue the whole batch with `unshift()`. A sustained disk or permission failure can grow memory far beyond `maxBufferSize`, so the file sink needs a hard cap/backpressure policy and coverage.
  - confidence: 0.87
- **[P2] Make retentionDays actually enforce retention:** `src/acp/control-plane/audit/audit-logger.file.ts:47-57`
  - body: The constructor schedules periodic flushes but never schedules `prune()`, so `retentionDays` has no production effect despite the advertised automatic 90-day retention. Either wire pruning into the lifecycle or remove the retention claim/config until it is real.
  - confidence: 0.86

## Security Review
