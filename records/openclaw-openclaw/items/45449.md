## Summary

Keep open and request changes: current main still lacks the requested Discord timing support, and the branch changes the removed `src/discord` runtime instead of the active `extensions/discord` plugin, so it would not change shipped Discord behavior.

## What This Changes

The PR adds Discord `ackReactionTiming` config/docs/tests and a runtime branch to defer ack reactions until `onAgentRunStart` when configured.

## Best Possible Solution

Implement Discord-scoped `ackReactionTiming` in the active `extensions/discord` plugin, preserve `received` as the default, defer the lifecycle only from `onAgentRunStart`, and update docs, generated config surfaces, tests, and changelog together.

## Solution Assessment
