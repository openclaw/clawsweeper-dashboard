## Summary

Keep this PR open: current main and the latest release still ship the stale nested Docker image package path, while the PR is a focused fix with sufficient terminal proof and no blocking correctness findings. The only maintainer-visible caveat is compatibility for downstream images that were relying on the private stale `/app/node_modules/openclaw` or `.bin` shim path.

## What This Changes

The PR changes the Docker build to remove a stale nested `openclaw` package, its `.bin` shim, and the pnpm virtual-store copy from the `runtime-assets` stage before the final runtime image copy.

## Best Possible Solution

Keep the cleanup in the Docker image staging path, and treat the stale nested package path as unsupported private image debris while preserving the documented `/usr/local/bin/openclaw` launcher.

## Reproduction Assessment

Yes from source and reporter/PR Docker commands, but I did not run Docker in this read-only review. Current main copies `/app/node_modules` from `runtime-assets`, and current extension manifests plus `pnpm-lock.yaml` include the stale `openclaw@2026.5.28` dependency.

## Solution Assessment

Yes. Pruning the stale package and shim in `runtime-assets` is the narrowest Docker-image fix because it removes the artifact before the final `COPY` without changing extension package contracts or plugin build behavior.
