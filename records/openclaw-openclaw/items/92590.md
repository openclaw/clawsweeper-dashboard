## Summary

Keep open: the PR mitigates the top-level stale CLI shim, but it does not fully remove the stale OpenClaw package from the Docker image because pnpm can still leave the real package content in the virtual store after the final node_modules copy.

## What This Changes

The PR adds a Dockerfile final-stage cleanup that removes `/app/node_modules/openclaw` and `/app/node_modules/.bin/openclaw` from the runtime image.

## Best Possible Solution

Prune the stale package in the runtime-assets stage before copying node_modules into the final image, and verify the top-level link, `.bin` shim, and pnpm virtual-store package are all gone.

## Reproduction Assessment

Yes, source-reproducible. Current main copies `node_modules` from `runtime-assets`, bundled plugin manifests still pin `openclaw` to `2026.5.28`, and the linked issue provides a Docker build/run repro for the stale package and shim.

## Solution Assessment

No. The PR is a useful mitigation, but deleting only the top-level link and shim after the final copy is too narrow because pnpm package content can remain in the virtual store.
