## What This Changes

The PR adds an in-memory pending catch-up deferral Set to cron service state, populates it when startup deferrals are assigned, centralizes the future-slot repair exemption in maintenance recompute, and adds regression coverage for read-RPC preservation plus expiry.

## Best Possible Solution

Land the state-owned deferral exemption after real behavior proof is added or maintainer validation confirms the branch, then close https://github.com/openclaw/openclaw/issues/93935 through the merged fix.

## Reproduction Assessment

Yes. Current main shows startup can create a one-shot skip set, then read/list/status and timer finalization can call maintenance recompute without that exemption before the staggered slot; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Moving pending deferrals into CronServiceState and enforcing the exemption inside recomputeNextRunsForMaintenance is the best boundary because all read, timer, and manual-run maintenance callers share that helper.
