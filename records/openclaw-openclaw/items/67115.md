## Summary

Keep open. Current main still reproduces the false no-GUI path, and the PR is the right narrow code direction, but it is not merge-ready without the required changelog entry and focused regression coverage.

## What This Changes

The PR changes `resolveBrowserOpenCommand()` so the SSH/no-display guard skips Darwin and documents stale SSH variables on macOS.

## Best Possible Solution

Land this PR or an equivalent current-main patch with Darwin and Linux regression tests, preserving Windows URL-handler hardening and adding a one-line changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to evaluate `resolveBrowserOpenCommand()` on Darwin with `SSH_CLIENT` or `SSH_TTY` set and `DISPLAY`/`WAYLAND_DISPLAY` unset; current main returns `ssh-no-display` before reaching the Darwin `open` branch.

## Solution Assessment

Mostly yes. Excluding Darwin from the SSH/no-display guard is the narrowest maintainable code fix, but the mergeable solution should include regression tests and release-note coverage.
