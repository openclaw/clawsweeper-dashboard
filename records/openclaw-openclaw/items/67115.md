## Summary

Keep open. Current main still reproduces the macOS false no-GUI path, and the PR is the right narrow direction, but it still lacks the required user-facing changelog and focused regression coverage before merge.

## What This Changes

The PR narrows browser-open SSH/no-display detection so macOS can still use the system `open` command when SSH environment variables are present.

## Best Possible Solution

Land this PR or an equivalent current-main patch with Darwin and Linux regression tests, preserved Windows browser-open hardening, and a single-line user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to evaluate `resolveBrowserOpenCommand()` on Darwin with `SSH_CLIENT` or `SSH_TTY` set and `DISPLAY`/`WAYLAND_DISPLAY` unset; current main returns `ssh-no-display` before the Darwin `open` branch.

## Solution Assessment

Mostly yes. Excluding Darwin from the SSH/no-display guard is the narrowest code fix for the false negative, but the mergeable solution should add regression coverage and release-note coverage.
