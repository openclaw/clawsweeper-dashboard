## Summary

Keep open. Current main still short-circuits macOS dashboard browser opening when SSH environment variables are present without a display, and this PR is a narrow valid fix direction, but the submitted patch is not merge-ready because the user-facing fix lacks the required changelog entry and focused regression coverage.

## What This Changes

The PR changes `src/infra/browser-open.ts` so the SSH/no-display guard no longer applies on Darwin and adds a short explanatory comment for lingering SSH variables.

## Best Possible Solution

Land this PR or an equivalent current-main patch that excludes Darwin from the SSH/no-display guard, adds focused Darwin and Linux regression tests, preserves existing Windows browser-open hardening, and includes a single-line user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to evaluate `resolveBrowserOpenCommand()` on Darwin with `SSH_CLIENT` or `SSH_TTY` set and `DISPLAY`/`WAYLAND_DISPLAY` unset; current main returns `ssh-no-display` before the Darwin `open` branch.

## Solution Assessment

Mostly yes. Excluding Darwin from the SSH/no-display guard is the narrowest code fix for the false negative, but the mergeable solution should add regression coverage and a changelog entry.
