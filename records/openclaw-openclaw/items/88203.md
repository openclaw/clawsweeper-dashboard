## Best Possible Solution

Land this branch after maintainer acceptance of the fail-closed compatibility shift and normal current-head gates, keeping the shared classifier as the single stale-approval path.

## Reproduction Assessment

## Solution Assessment

Yes, this is the best bounded fix for the PR scope: it extends the existing shared classifier and applies it to sibling waitDecision consumers instead of adding separate one-off handlers. A structured gateway error reason would be a reasonable follow-up, but not a blocker for this repair.
