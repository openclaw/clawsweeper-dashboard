## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Recompute runtime cwd before using TUI fallbacks:** `src/tui/tui.ts:523`
  - body: `fallbackLaunchCwd` keeps the launch directory whenever it existed at TUI startup. If that directory is deleted later, the runtime paths this PR is meant to cover still pass the stale deleted path into autocomplete and auth-login spawn instead of falling back to the wrapper dir, so `openclaw tui` can still fail after startup. Resolve the cwd at the point of use or validate the captured path before using it.
  - confidence: 0.82
- **[P2] Avoid using the Node install dir as OpenClaw home:** `src/infra/home-dir.ts:93`
  - body: When no home source exists and cwd is gone, this returns `path.dirname(process.execPath)`, so `resolveStateDir()` derives `.openclaw` under the Node/Bun install directory. Those directories are commonly shared or non-writable and hold credential-adjacent state, so this should fail clearly or require an explicit home/state override instead of silently choosing the executable directory.
  - confidence: 0.8
