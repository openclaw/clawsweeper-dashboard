## Summary

Keep this PR open, but not merge-ready: the caller-specific nullable-cwd direction is good and the TUI/local-shell rework addresses prior review, yet the required-home fallback still redirects OpenClaw state/config/credentials roots to the Node executable directory and the external proof remains a stubbed `process.cwd()` probe rather than an actual deleted-cwd CLI/TUI run.

## What This Changes

The PR adds nullable cwd helpers and updates CLI dotenv loading, gateway dispatch dotenv loading, PATH bootstrap, home-dir fallback, `is-main`, TUI cwd handling, local-shell execution, tests, and CI test planning for deleted-current-working-directory cases.

## Best Possible Solution

Land one caller-specific nullable-cwd fix that skips workspace/env and project-local PATH lookups, refuses operator commands when cwd is gone, revalidates TUI runtime cwd, and handles the no-home/no-cwd state-root case by clear failure or an explicit configured state root instead of an implicit executable-directory fallback.
