## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Do not run deleted-cwd local shell commands from home:** `src/tui/tui-local-shell.ts:39`
  - body: When `process.cwd()` fails, this makes `!cmd` run from `os.homedir()`. The existing approval warns that commands run on the user's machine; it does not ask to retarget relative commands into HOME after the project cwd disappears, so commands like `!rm ./file` can affect a different directory than intended. Prefer a controlled error or an explicit cwd choice.
  - confidence: 0.86
- **[P1] Revalidate runtime cwd before TUI spawns:** `src/tui/tui.ts:540`
  - body: `fallbackLaunchCwd` captures the launch cwd once and later feeds auth-spawn and autocomplete paths. If that directory is deleted after TUI startup, the child spawn still receives a stale cwd and can fail with `ENOENT`; re-check at use time or fall back only after confirming the saved cwd still exists.
  - confidence: 0.84
- **[P2] Avoid using the executable directory as OpenClaw home:** `src/infra/home-dir.ts:93`
  - body: When every home source and cwd are unavailable, this falls back to `path.dirname(process.execPath)`. `resolveStateDir()` derives mutable state and credential-adjacent paths from required home, so the fix can silently move OpenClaw state under a Node/Bun install directory that may be shared, non-writable, or outside the intended state root; fail clearly or require an explicit state/home path instead.
  - confidence: 0.84
