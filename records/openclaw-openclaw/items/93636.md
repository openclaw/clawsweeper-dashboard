## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not run deleted-cwd local shell commands from home:** `src/tui/tui-local-shell.ts:39`
  - body: When the TUI process cwd disappears, this fallback makes `!cmd` run from `os.homedir()` instead of failing or asking for an explicit cwd. That can redirect relative operator shell commands from the vanished project into the user's home directory, which is a command-execution boundary change rather than a safe recovery.
  - confidence: 0.86
- **[P1] Revalidate the TUI cwd before auth spawns use it:** `src/tui/tui.ts:1206`
  - body: `fallbackLaunchCwd` is captured once during startup and reused later for auth spawns. If the cwd exists at launch but is deleted before the auth flow, Node spawn receives a missing cwd and emits ENOENT, so the runtime deleted-cwd path is still not handled at the point of use.
  - confidence: 0.84
- **[P2] Avoid using the executable directory as OpenClaw home:** `src/infra/home-dir.ts:93-94`
  - body: When every home source and cwd are unavailable, this routes required-home resolution through `path.dirname(process.execPath)`. `resolveStateDir()` derives mutable state and credential-adjacent paths from required home, so the fix can silently move OpenClaw state under a Node/Bun install directory that may be shared or non-writable; fail clearly or require an explicit operator-controlled state/home path instead.
  - confidence: 0.84
