## Summary

## What This Changes

## Best Possible Solution

Land one consolidated caller-specific nullable-cwd fix that refuses or requires an explicit configured state root for the no-home/no-cwd state path, preserves the TUI local-shell refusal, and includes redacted real deleted-cwd terminal proof.

## Solution Assessment

No, not as submitted. Caller-specific nullable cwd handling is the right direction, but required-home should fail clearly or use a maintainer-approved explicit state root instead of `path.dirname(process.execPath)`.
