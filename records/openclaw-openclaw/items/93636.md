## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Revalidate the TUI cwd before auth spawns use it:** `src/tui/tui.ts:1206`
  - body: `runTui` captures `fallbackLaunchCwd` once and later passes it into the auth-login spawn. If the cwd is deleted after TUI startup, the spawn still uses a now-missing directory instead of recomputing the caller-specific fallback, leaving a reported runtime TUI path able to fail.
  - confidence: 0.82
- **[P2] Avoid using the executable dir as OpenClaw home:** `src/infra/home-dir.ts:93`
  - body: When every home source is unavailable and cwd is gone, this makes required-home resolution use `path.dirname(process.execPath)`. `resolveStateDir()` derives mutable state from `resolveRequiredHomeDir()`, so state and credential-adjacent paths can silently move under a Node/Bun install directory that may be shared or non-writable; fail clearly or use an explicit operator-controlled fallback instead.
  - confidence: 0.84
