## Summary

Keep open. Current main has useful adjacent memory work, especially Active Memory, but it still does not provide a default deterministic project/task/focus recall contract for explicit `openclaw agent --session-id` workflows, and the OpenAI prompt-cache/session-id path still forwards raw session ids without local normalization or validation.

## What This Changes

The issue requests stronger default task and memory recall across normal and fresh agent conversations, including durable project/task capture, mutation-aware open-task retrieval, current-focus preference over stale memory, and clearer long session-id/provider cache-key errors.

## Best Possible Solution

Keep this open until maintainers define and test the default memory/task contract: add a repo-contained live or deterministic memory/task recall gate, document the canonical project/task/focus workspace state if OpenClaw owns one, make recall prefer current canonical state over stale semantic hits where such state exists, define open-task retrieval semantics, and validate or normalize provider prompt-cache/session identifiers before OpenAI transport.

## Reproduction Assessment

Yes, at the issue level: the report gives concrete CLI steps, profiles, datasets, and a live gate command. This review did not replay the live model/embedding harness, but current-main code inspection confirms the missing default contract and cache-key validation path remain plausible.

## Solution Assessment

No, current main is not yet the best/full solution. Active Memory is valuable adjacent work, but it is optional and gated; the safer end state is an explicit contract plus tests for task/focus recall and provider key validation rather than relying only on model-followed memory-search guidance.
