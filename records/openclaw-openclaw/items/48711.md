## What This Changes

The issue asks OpenClaw to reliably capture and recall project priorities, explicit memories, task mutations, current focus, and long-session cache-key errors across same-session and fresh-session conversations.

## Best Possible Solution

Define and test a default memory/task/current-focus contract, make current canonical state outrank stale semantic recall, add prompt-cache-key validation or normalization, and land the narrower open fixes only for their specific subproblems.
