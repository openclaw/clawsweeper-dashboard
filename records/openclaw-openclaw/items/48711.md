## Summary

Keep open: current main and the latest stable release have partial memory-search and prompt-cache fixes, but the reported default task/project memory reliability, fresh-session current-state precedence, and open-task recall behavior are still not fully implemented or live-proven. The open #11955 umbrella overlaps the product direction, and #80399 covers one closed retrieval sub-symptom, but neither replaces this concrete reliability report.

## What This Changes

The issue asks OpenClaw to reliably capture and recall project, task, and memory state across same-session, fresh-session, mutation, and natural-language recall workflows.

## Best Possible Solution

Split the work into focused maintainer-scoped tracks for canonical task/project state capture, current-state-over-stale-memory recall, open-task semantics, and live regression proof while preserving the shipped prompt-cache clamp and improved hybrid search.

## Reproduction Assessment

No high-confidence current-main live reproduction was established; source inspection does show default memory indexes only MEMORY.md/memory/*.md, session memory and active memory are opt-in, and PROJECTS.md/TASKS.md open-task behavior is not a guaranteed default contract.

## Solution Assessment

No as a one-shot fix: the best path is a scoped design and validation split that reuses memory-core and active-memory surfaces rather than adding broad default behavior in one automated PR.
