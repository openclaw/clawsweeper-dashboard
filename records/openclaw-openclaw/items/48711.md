## Summary

Keep open. Current main has useful adjacent memory improvements, including parallel hybrid search and optional Active Memory, but it still does not close the default project/task/focus recall contract or the long-session-id prompt-cache validation problem reported here.

## What This Changes

The issue requests reliable capture and recall of project, task, priority, focus, and explicit memory state across same-session, fresh-session, mutation, and drift workflows.

## Best Possible Solution

Define and test a default memory/task contract, decide whether OpenClaw owns canonical project/task/focus state, make current state outrank stale recall, and add local prompt-cache key validation or normalization for OpenAI transport.

## Reproduction Assessment

Yes at source/report level, but not as a replayed current-main live failure. The issue provides concrete live-gate commands and current source still shows the missing default task/focus contract plus raw sessionId prompt_cache_key forwarding.

## Solution Assessment

No current implementation is a complete solution. Active Memory and hybrid search improvements are useful, but the maintainable fix needs an explicit product contract and targeted provider validation rather than only prompt-followed recall guidance.
