## Summary

Keep open. Current main has useful adjacent memory work, especially Active Memory, but it still does not define or test the default deterministic project/task/focus recall contract reported here, and the OpenAI prompt-cache/session-id path still forwards session IDs without local provider-limit validation.

## What This Changes

The issue asks OpenClaw to make project priorities, task state, current focus, and explicit memory recall reliable across same-session, fresh-session, and mutated-state conversations.

## Best Possible Solution

Define and test the default memory/task contract: document any canonical project/task/focus workspace state OpenClaw owns, make current canonical state take precedence over stale semantic hits, define open-task recall semantics, add a deterministic or live-gated regression path, and validate or normalize provider prompt-cache session identifiers before OpenAI transport.

## Reproduction Assessment

Yes at the report level, but not as a replayed local failure in this review. The issue provides concrete CLI/live-gate steps, and current source still shows the missing default task/focus contract plus raw prompt-cache-key path.

## Solution Assessment

No, current main is not yet the best/full solution. Active Memory is valuable adjacent work, but it is optional and scoped; the maintainable fix needs an explicit contract and tests rather than only model-followed recall guidance.
