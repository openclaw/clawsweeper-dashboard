## Summary

Keep open. Current main has useful adjacent memory-search and Active Memory work, but it still does not define or enforce the default project/task/current-focus recall contract reported here, and the OpenAI transport still forwards session ids as prompt cache keys without local provider-limit validation.

## What This Changes

The issue asks OpenClaw to make conversation-seeded memory, project priorities, open tasks, mutations, fresh-session recall, and long-session-id errors reliable by default.

## Best Possible Solution

Define and test a default memory/task/focus contract, make current canonical state outrank stale semantic recall, and add local prompt-cache key validation or normalization for OpenAI-compatible transports.

## Reproduction Assessment

Yes at source/report level, but not as a replayed current-main live failure. The issue gives concrete live-gate commands and current source still shows the missing default task/focus contract plus raw sessionId prompt_cache_key forwarding.

## Solution Assessment

No current implementation is a complete solution. The safest path is maintainer design for the default memory/task contract plus a targeted transport validation fix, not only more prompt guidance or optional Active Memory configuration.
