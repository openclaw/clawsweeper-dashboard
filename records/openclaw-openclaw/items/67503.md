## Summary

Keep open. Current main still has the Slack and Microsoft Teams truncated-preview system-event path, Mattermost is already fixed, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to stop putting truncated inbound message-body previews into channel system-event headers when the full body is already delivered separately.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/67761 or an equivalent narrow fix that makes Slack and Microsoft Teams enqueue label-only events while preserving full inbound content in BodyForAgent/RawBody.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters, and current main will enqueue a truncated preview while delivering the full body separately.

## Solution Assessment

Yes for the code direction. Removing message content from notification-only system events is the narrowest maintainable fix because full content already belongs in BodyForAgent/RawBody; closure should wait for the linked PR or an equivalent fix to merge.
