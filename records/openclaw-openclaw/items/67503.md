## Summary

Keep open. Current main and the latest release still put 160-character Slack and Microsoft Teams inbound-body previews into model-visible system events while the full body is delivered separately; the linked closing PR remains open rather than merged.

## What This Changes

The issue asks to remove duplicated truncated inbound message previews from Slack, Microsoft Teams, and Mattermost system-event notifications.

## Best Possible Solution

Land or replace the linked narrow fix so Slack and Microsoft Teams enqueue label-only system events, keep full content in `BodyForAgent`/`RawBody`, and preserve the existing Mattermost behavior.

## Reproduction Assessment

Yes. Send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters on current main; the handler source will enqueue a truncated preview while delivering the full body separately.

## Solution Assessment

Yes. Removing message content from notification-only system events is the narrowest maintainable fix; closure should wait for https://github.com/openclaw/openclaw/pull/67761 or an equivalent patch to merge.
