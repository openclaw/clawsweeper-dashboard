## Summary

Keep open. Current main and the latest stable release still enqueue Slack and Microsoft Teams 160-character inbound-body previews into model-visible system events while separately delivering the full body, Mattermost has already moved to the desired behavior, and https://github.com/openclaw/openclaw/pull/67761 is still the open implementation path for this issue.

## What This Changes

The issue asks to remove duplicated truncated inbound message previews from Slack, Microsoft Teams, and Mattermost system-event headers so only the normal inbound body carries message content.

## Best Possible Solution

Review and land an updated, mergeable version of https://github.com/openclaw/openclaw/pull/67761 or an equivalent narrow fix that makes Slack and Microsoft Teams enqueue label-only events, preserves Mattermost’s current behavior, and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters and current main will enqueue a truncated preview while delivering the full body separately.

## Solution Assessment

Yes. Removing message content from notification-only system events is the narrowest maintainable fix because full content delivery already belongs in the channel payload fields.
