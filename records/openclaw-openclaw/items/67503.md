## Summary

Keep open. Current main and the latest release still enqueue truncated Slack and Microsoft Teams message-body previews into model-visible system events, while the open closing PR remains the active fix path.

## What This Changes

The issue asks OpenClaw to remove duplicated, truncated inbound message content from channel system-event headers while preserving full delivery through the normal inbound body fields.

## Best Possible Solution

Land or replace the existing fix path so Slack and Microsoft Teams enqueue only notification labels, keep full inbound content in BodyForAgent/RawBody, preserve the current Mattermost behavior, and add focused regression coverage.

## Reproduction Assessment

Yes. By source inspection, any Slack or Microsoft Teams inbound rawBody longer than 160 characters is truncated into the queued system-event preview while the full rawBody remains separately available to the agent.

## Solution Assessment

Yes. Removing message content from notification-only system events while preserving full channel payload delivery is the narrow maintainable fix; closure should wait for the linked PR or an equivalent replacement to merge.
