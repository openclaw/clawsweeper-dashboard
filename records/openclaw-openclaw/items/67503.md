## Summary

Keep open. Current main still has the reported Slack and Microsoft Teams truncated-preview system events, and the issue is already paired with open PR #67761 using closing intent; Mattermost appears fixed on main, but the central remaining behavior is not resolved.

## What This Changes

Remove duplicated truncated inbound-message body previews from Slack, Microsoft Teams, and Mattermost system-event headers so the model sees the full inbound body only through the normal message context.

## Best Possible Solution

Review, rebase, land, close, or replace PR #67761. The desired end state is label-only Slack and Microsoft Teams notification system events, full inbound content only in `BodyForAgent`/`RawBody`, the current Mattermost no-system-event behavior preserved, and focused regression coverage for long inbound messages.

## Reproduction Assessment

Yes. Send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters; current main enqueues a truncated system-event preview while separately placing the full content in the normal agent body fields.

## Solution Assessment

Yes. Removing message-body previews from notification-only system events is the narrowest maintainable fix; the safer completion path is to apply it to the remaining Slack and Microsoft Teams handlers and keep Mattermost’s current regression-tested behavior.
