## Summary

Keep this issue open. Current main still has the reported Slack and Microsoft Teams behavior: both handlers enqueue a 160-character body preview into model-visible system-event text while separately preserving the full inbound body, and the open paired PR remains the intended implementation path.

## What This Changes

The issue requests removing duplicated 160-character inbound message previews from model-visible system-event headers while keeping full message bodies in the normal inbound turn.

## Best Possible Solution

Review and update the existing fix path so Slack and Microsoft Teams enqueue only notification labels, keep full inbound content in `BodyForAgent`/`RawBody`, preserve the current Mattermost behavior, and add focused channel regression tests before closing after merge.

## Reproduction Assessment

Yes. The current-main source gives a high-confidence reproduction path: send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters and the handler will enqueue a truncated preview while separately placing the full text in the normal inbound context.

## Solution Assessment

Yes. Removing message-body previews from notification-only system events is the narrowest maintainable fix; the safer completion path is to review, rebase, land, close, or replace PR #67761 rather than opening a second automated repair lane.
