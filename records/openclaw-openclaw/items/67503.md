## Summary

Keep open. Current main still puts truncated Slack and Microsoft Teams inbound message previews into model-visible system events, while the full body is separately preserved and the open closing PR remains unresolved.

## What This Changes

The issue requests removing duplicated 160-character inbound message previews from Slack, Microsoft Teams, and Mattermost system-event labels so only the full channel payload carries message content.

## Best Possible Solution

Review, refresh, and land https://github.com/openclaw/openclaw/pull/67761 or an equivalent narrow replacement that removes message content from Slack and Teams system events while preserving full inbound content in `BodyForAgent`/`RawBody`.

## Reproduction Assessment

Yes, by source inspection. A Slack or Microsoft Teams inbound body longer than 160 characters will enqueue a truncated system-event preview while the full body remains separately available in `BodyForAgent`/`RawBody`.

## Solution Assessment

Yes. Removing content from notification-only system events while keeping full channel payload delivery is the narrow maintainable fix; closure should wait for the linked PR or an equivalent replacement to merge.
