## Summary

Keep open. Current main still has the reported Slack and Microsoft Teams truncated-preview system events, Mattermost already has the desired regular-message behavior, and open PR #67761 is the paired implementation candidate that should be reviewed before this issue closes.

## What This Changes

Remove duplicated 160-character inbound message previews from Slack, Microsoft Teams, and Mattermost system-event headers so full message content is delivered only through the normal inbound body.

## Best Possible Solution

Land an updated version of PR #67761, or an equivalent narrow fix, so Slack and Microsoft Teams enqueue label-only notification events while Mattermost’s current no-system-event behavior remains covered.

## Reproduction Assessment

Yes. From source inspection, sending or simulating a Slack or Microsoft Teams inbound body longer than 160 characters will enqueue a truncated system-event preview while the full body is carried separately in the inbound context.

## Solution Assessment

Yes. Removing message content from notification-only system events is the narrow maintainable solution because full content delivery already belongs in `BodyForAgent` and `RawBody`.
