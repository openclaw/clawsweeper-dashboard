## What This Changes

The issue requests removing duplicated 160-character inbound message previews from system-event notifications while preserving full channel body delivery.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/67761 or replace it with an equivalent narrow Slack/Teams fix that enqueues label-only system events and keeps full inbound content in `BodyForAgent`/`RawBody`.

Yes, by source inspection. Send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters and current main will enqueue a truncated preview while separately preserving the full body in the channel payload.

## Solution Assessment

Yes for the requested code direction. Removing message content from notification-only system events while retaining full channel payload content is the narrow maintainable fix; closure should wait for the linked PR or equivalent replacement to merge.
