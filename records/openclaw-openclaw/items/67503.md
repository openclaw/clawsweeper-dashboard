## Summary

Keep open. Current main still has the Slack and Microsoft Teams truncated-preview system-event paths, and the issue is already paired with open closing PR https://github.com/openclaw/openclaw/pull/67761 rather than a merged fix.

## What This Changes

Remove inbound message body previews from Slack, Microsoft Teams, and Mattermost system-event notification labels so the model does not see a duplicated truncated body.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/67761 with the narrow Slack/Teams label-only system-event fix and focused regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes, by source inspection. Send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters and current main will enqueue a truncated preview while separately preserving the full body in the channel payload.

## Solution Assessment

Yes for the code direction. Removing message content from notification-only system events while retaining full content in `BodyForAgent`/`RawBody` is the narrowest maintainable fix, but closure should wait for the linked PR or equivalent replacement to merge.
