## Summary

Keep open. Current main still has the reported Slack and Microsoft Teams truncated-preview system events, Mattermost has already moved to the desired behavior, and open PR #67761 is the paired implementation candidate that should be reviewed before this issue closes.

## What This Changes

The issue asks for inbound channel notification system events to omit duplicated 160-character message-body previews while preserving the full message body in the normal inbound context.

## Best Possible Solution

Land an updated version of #67761, or an equivalent narrow fix, so Slack and Microsoft Teams enqueue label-only notification events while Mattermost’s current no-system-event behavior remains covered.

## Reproduction Assessment

Yes. Send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters; current main enqueues a truncated system-event preview while separately delivering the full content through the normal inbound body fields.

## Solution Assessment

Yes. Removing message content from notification-only system events is the narrowest maintainable solution because full content delivery already belongs in `BodyForAgent` and `RawBody`.
