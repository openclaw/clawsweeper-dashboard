## Summary

Keep open. Current main and v2026.5.18 still enqueue 160-character Slack and Microsoft Teams message previews into model-visible system events while separately preserving the full message body, and the open closing PR remains unresolved.

## What This Changes

The issue requests removing duplicated truncated inbound message content from Slack, Microsoft Teams, and Mattermost system-event headers while preserving full message delivery.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/67761 so Slack and Microsoft Teams enqueue only notification labels, preserve full BodyForAgent/RawBody delivery, keep the Mattermost fix, and add focused regression proof before closing.

Yes, by source inspection. A Slack or Microsoft Teams inbound body longer than 160 characters is normalized and sliced into queued system-event text, and that event is later formatted as a model-visible `System:` line while the full body remains separate.

## Solution Assessment

Yes. Removing message content from notification-only system events while preserving full channel payload delivery is the narrow maintainable fix; current main has only completed that boundary for Mattermost.
