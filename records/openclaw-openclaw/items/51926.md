## What This Changes

The PR changes the agents `nodes` tool `describe` branch to treat `node` as optional and pass `allowDefault=true` to `resolveNodeId` before calling `node.describe`.

## Best Possible Solution

Define omitted-node `describe` semantics explicitly, implement that path with node-agnostic default or discovery/list behavior, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flat schema, then the `describe` branch requires `node` and deterministically throws `node required` before gateway dispatch.

## Solution Assessment

No. Making `node` optional for read-only `describe` is plausible, but reusing the canvas-biased default resolver is not the best fix until maintainers define describe-specific omitted-node semantics.
