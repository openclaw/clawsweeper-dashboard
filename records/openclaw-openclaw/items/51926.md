## Summary

Keep open. The PR targets a real agents `nodes` tool schema/runtime mismatch, but the proposed omitted-node `describe` path reuses the existing canvas-biased default-node picker and still needs maintainer semantics, regression coverage, and real behavior proof before merge.

## What This Changes

The PR changes the agents `nodes` tool `describe` action to read `node` as optional and call `resolveNodeId(..., true)` before `node.describe`.

## Best Possible Solution

Define an intentional omitted-node `describe` contract, use a node-agnostic default or discovery/list behavior for that contract, preserve explicit-node requirements for targeted actions, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flattened schema, then the `describe` branch calls `readStringParam(params, "node", { required: true })`, which deterministically throws `node required`.

## Solution Assessment

No, not yet. Making `node` optional for read-only `describe` is the right boundary, but `resolveNodeId(..., true)` inherits canvas-specific default selection rather than a describe-specific node-agnostic or discovery contract.
