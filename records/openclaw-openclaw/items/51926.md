## What This Changes

The PR changes `nodes` `action: "describe"` so an omitted `node` is read as optional and resolved through the existing default-node resolver before calling `node.describe`.

## Best Possible Solution

Confirm the intended omitted-node `describe` behavior, add focused regression coverage, and merge this PR or a maintained replacement with redacted real runtime proof.

## Reproduction Assessment

Yes, by source inspection. Current `origin/main` allows omitted `node` in the `nodes` tool schema, then the `describe` branch requires it before default-node resolution can run.

## Solution Assessment

Unclear as a product decision. The patch is the narrowest fix if omitted `describe` should mean default node, but maintainers still need to choose between default-node and discovery/list semantics for multi-node setups.
