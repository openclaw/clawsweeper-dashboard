## Summary

Keep open: current main still rejects omitted-node `describe`, while this active PR is the focused same-author fix for open issue #51903. The source change appears correct, but it should get maintainer acceptance and a focused regression test before merge.

## What This Changes

The branch changes `src/agents/tools/nodes-tool.ts` so `describe` reads `node` as optional and calls `resolveNodeId` with default fallback enabled.

## Best Possible Solution

Add a focused `nodes-tool.test.ts` omitted-node `describe` case and merge this narrow fix if maintainers accept using the existing default-node picker for read-only describe.

## Reproduction Assessment

Yes. Source inspection on current main shows `{ "action": "describe" }` reaches `readStringParam(..., { required: true })`, which throws `node required` before the resolver can choose a default node.

## Solution Assessment

Yes, with regression coverage. The proposed source change is the narrowest maintainable fix because it reuses `resolveNodeId(..., true)` and leaves other explicit-node actions unchanged.
