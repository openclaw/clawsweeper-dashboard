## Summary

Keep open. The PR is a focused attempt to fix a real schema/runtime mismatch, but it is not merge-ready because the omitted-node path reuses a canvas-biased default resolver and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR makes the agents `nodes` tool `describe` action read `node` as optional and resolve through the existing default-node fallback.

## Best Possible Solution

Define an intentional omitted-node `describe` contract, use a node-agnostic default or discovery/list path for that contract, preserve explicit-node requirements for targeted actions, and cover it with focused regression tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Current main accepts omitted `node` in the flattened schema, then the `describe` branch calls `readStringParam(params, "node", { required: true })`, so `{ "action": "describe" }` deterministically throws `node required`.

## Solution Assessment

No, not yet. Removing the required read is the right boundary, but `resolveNodeId(..., true)` inherits canvas-specific default selection rather than a describe-specific node-agnostic or discovery contract.
