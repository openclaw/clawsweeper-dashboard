## What This Changes

The PR changes the agents `nodes` tool `describe` action to read `node` as optional and call `resolveNodeId(gatewayOpts, node, true)` before `node.describe`.

## Best Possible Solution

Define the omitted-node `describe` contract explicitly, use node-agnostic default or discovery/list behavior for that contract, preserve explicit-node requirements for targeted actions, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flattened schema, then the `describe` branch calls `readStringParam(params, "node", { required: true })`, which throws `node required`.

## Solution Assessment

No, not yet. Making `node` optional for read-only `describe` is the right boundary, but `resolveNodeId(..., true)` inherits canvas-specific default selection instead of a describe-specific node-agnostic or discovery contract.
