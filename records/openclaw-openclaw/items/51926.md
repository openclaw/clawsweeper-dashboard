Keep open. The PR targets a valid current-main agents tool bug, but the proposed fix reuses canvas-biased default-node selection for generic `node.describe` and still lacks after-fix real behavior proof, so it should not merge as-is.

## What This Changes

The PR changes the agents `nodes` tool `describe` branch to make `node` optional and call `resolveNodeId` with default resolution enabled.

## Best Possible Solution

Define the omitted-node `describe` contract explicitly, use node-agnostic default or discovery/list semantics for that contract, preserve explicit-node requirements for targeted actions, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flattened schema, then the `describe` branch calls `readStringParam(params, "node", { required: true })`, which deterministically throws `node required`.

## Solution Assessment

No, not yet. Making `node` optional for read-only `describe` is the right boundary, but `resolveNodeId(..., true)` inherits canvas-specific default selection rather than a describe-specific node-agnostic or discovery contract.
