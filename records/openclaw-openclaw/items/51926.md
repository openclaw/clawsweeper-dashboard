## Summary

## What This Changes

This PR changes `src/agents/tools/nodes-tool.ts` so `describe` reads `node` as optional and calls `resolveNodeId` with default-node fallback enabled.

## Best Possible Solution

Merge or replace this narrow resolver change after adding focused omitted-node `describe` coverage and real behavior proof, while preserving explicit-node requirements for targeted node actions.

## Reproduction Assessment

Yes, at source level. On current main, `{ "action": "describe" }` can pass the flattened schema but reaches `readStringParam(params, "node", { required: true })`, which throws `node required` before default-node resolution can run.

## Solution Assessment

Mostly yes if maintainers accept default/current-node semantics: reusing `resolveNodeId(..., true)` is the narrowest maintainable code change. If the intended behavior is discovery/list output, this PR should be replaced with that explicit design.
