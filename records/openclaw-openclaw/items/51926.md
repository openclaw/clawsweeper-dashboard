## Summary

Keep open. Current main still rejects `nodes` `describe` without `node`, while this PR narrowly changes that branch to use the existing default-node resolver; it is also paired with open issue #51903, so the right path is maintainer review and a focused regression test before merge rather than cleanup closure.

## What This Changes

The PR changes `src/agents/tools/nodes-tool.ts` so the `describe` action reads `node` as optional and calls `resolveNodeId(..., true)` to use the existing default-node fallback.

## Best Possible Solution

Add a focused `src/agents/tools/nodes-tool.test.ts` case that asserts omitted-node `describe` calls `resolveNodeId({}, undefined, true)` and forwards to `node.describe`, then land the narrow source change if maintainers accept read-only default-node selection.

## Reproduction Assessment

Yes. On current main, call the nodes tool with `{ "action": "describe" }` and no `node`; the `describe` branch reaches `readStringParam(..., { required: true })` and throws `node required` before the resolver can default.

## Solution Assessment

Yes, with regression coverage. The proposed source change is the narrowest maintainable fix because it reuses the existing opt-in default resolver and leaves explicit-node requirements for other targeted actions unchanged.
