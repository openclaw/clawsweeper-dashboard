## Summary

Keep open: the reported bug is valid and still present on current main, but this PR still needs revision because it reuses a canvas-oriented default picker for `node.describe` and lacks after-fix real behavior proof.

## What This Changes

This PR makes `nodes` action `describe` read `node` as optional and calls `resolveNodeId(..., true)` before dispatching `node.describe`.

## Best Possible Solution

Define the omitted-node `describe` contract explicitly, use a describe-specific node-agnostic default or discovery/list response, preserve explicit-node requirements for targeted actions, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flattened schema, then the `describe` branch calls `readStringParam(params, "node", { required: true })`, which deterministically throws `node required`.

## Solution Assessment

No, not yet. Making `node` optional for read-only `describe` is the right boundary, but reusing the canvas-biased default resolver is not a describe-specific or discovery-safe contract.
