## Summary

Keep open: this PR targets a real current/release mismatch, but the patch reuses a canvas-biased default resolver for a general `describe` read, so omitted-node `describe` can still fail for valid non-canvas nodes or choose the wrong node in mixed setups.

## What This Changes

The PR makes the `nodes` tool's `describe` action read `node` as optional and call shared default-node resolution before dispatching `node.describe`.

## Best Possible Solution

Define omitted-node `describe` semantics explicitly for general node discovery or describe-owned default selection, then add focused regression coverage and live gateway proof.

## Reproduction Assessment

Yes, by source inspection. Current main and `v2026.6.6` make `node` optional in the flat schema, then the `describe` branch requires `node` and throws before gateway dispatch.

## Solution Assessment

No. The patch is a plausible small fix, but not the best proven fix because it reuses a canvas-biased resolver for general `node.describe`; the intended default/list behavior needs maintainer confirmation.
