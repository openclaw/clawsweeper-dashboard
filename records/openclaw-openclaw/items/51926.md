The PR changes the agents `nodes` tool `describe` branch to read `node` as optional and call `resolveNodeId` with default-node resolution enabled.

## Best Possible Solution

Define intentional omitted-node `describe` semantics, preferably a node-agnostic default or discovery/list behavior, then add focused regression coverage for omitted `node` including a non-canvas node case.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main accepts omitted `node` in the flat schema, then the `describe` branch reads `node` as required and deterministically throws `node required`.

## Solution Assessment

No, not yet. The PR removes the immediate required-parameter read, but `resolveNodeId(..., true)` inherits canvas-specific default selection, so the durable fix needs describe-specific node-agnostic semantics and regression coverage.
