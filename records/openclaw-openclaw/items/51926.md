## Summary

Keep open: current main still rejects omitted-node `describe`, but this PR routes the omitted case through a canvas-oriented default picker and has no real after-fix behavior proof, so it is not merge-ready as-is.

## What This Changes

The PR makes the agents `nodes` tool treat `node` as optional for `action: "describe"` and pass the call through the existing default-node fallback before `node.describe`.

## Best Possible Solution

Define the omitted-node `describe` contract explicitly, use either node-list discovery or a describe-specific node-agnostic default, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main exposes optional `node` in the flat tool schema, then the `describe` branch immediately calls `readStringParam` with `required: true`, producing `node required` before resolution.

## Solution Assessment

No, not yet: the PR removes the immediate required-param failure but reuses a canvas-oriented default picker for a metadata/read action; a describe-specific default or discovery behavior is safer.
