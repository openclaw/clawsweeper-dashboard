## What This Changes

The PR changes the agents `nodes` tool `describe` case to treat `node` as optional and call `resolveNodeId(..., true)` before dispatching `node.describe`.

## Best Possible Solution

Define omitted-node `describe` as a node-agnostic read/discovery contract, use a defaulting path that matches that contract, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes by source inspection. Current main accepts omitted `node` in the flat tool schema, then the `describe` branch requires `node` and would throw `node required` before dispatching to `node.describe`.

## Solution Assessment

No, not yet. Making `node` optional for the read-only `describe` action is the right repair boundary, but the proposed implementation reuses a canvas-specific default picker instead of a describe-specific node-agnostic or discovery contract.
