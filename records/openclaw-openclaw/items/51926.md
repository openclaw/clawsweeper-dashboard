## Summary

Keep open: current main still has the schema/runtime mismatch, and this PR is not merge-ready because it reuses the canvas-biased default resolver for generic `node.describe` and lacks real behavior proof.

## What This Changes

The PR makes the agents `nodes` tool `describe` action read `node` as optional and resolve omitted nodes through the existing default-node path before calling `node.describe`.

## Best Possible Solution

Define omitted-node `describe` semantics explicitly, use node-agnostic default or discovery/list behavior for that contract, preserve explicit-node requirements for targeted actions, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flat schema, then the `describe` branch requires `node` and deterministically throws `node required` before gateway dispatch.

## Solution Assessment

No, not yet. Making `node` optional for read-only `describe` is the right boundary, but reusing `resolveNodeId(..., true)` inherits canvas-specific default selection rather than a describe-specific node-agnostic or discovery contract.
