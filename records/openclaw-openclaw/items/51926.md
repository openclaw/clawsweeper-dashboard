## Summary

Keep open. Current main still has the reported schema/runtime mismatch, and the PR is an active same-author implementation path, but the proposed fallback is not ready to merge because it uses the canvas-biased default-node resolver and lacks after-fix real behavior proof.

## What This Changes

The PR changes `src/agents/tools/nodes-tool.ts` so `describe` reads `node` as optional and calls `resolveNodeId` with default-node fallback enabled.

## Best Possible Solution

Replace or revise the PR so omitted-node `describe` uses an intentional node-agnostic discovery/default contract, preserves explicit-node requirements for targeted actions, and adds focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main permits omitted `node` in the flat schema but the `describe` branch requires it, so the call deterministically throws `node required` before any default resolver can run.

## Solution Assessment

No, not yet. Reusing `resolveNodeId(..., true)` is narrow, but that resolver is canvas-biased, so `describe` should use a node-agnostic default or an explicit discovery/list behavior instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Use a node-agnostic default for describe:** `src/agents/tools/nodes-tool.ts:193`
  - body: `resolveNodeId(..., true)` uses the existing default picker, which filters candidates through `capability: "canvas"`.
