## What This Changes

The issue reports that a mixed MCP `streamable-http` config with both a spawned command and URL times out instead of connecting or failing clearly.

## Best Possible Solution

Review and land the docs-aligned resolver fix that rejects mixed `command` plus `url` with a clear diagnostic, or explicitly track separate product work if OpenClaw should support launching a child process and then connecting to its HTTP endpoint.

## Reproduction Assessment

## Solution Assessment

Yes, the best current fix is resolver-level rejection because that layer sees both raw config fields before SDK transport construction and the docs already define mixed entries as rejected. The linked PR still needs normal maintainer proof and compatibility review before it can close the issue.
