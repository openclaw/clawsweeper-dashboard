## Summary

Keep open: current main still resolves mixed command+url MCP server configs through stdio before streamable-http is considered, matching the reported 30-second timeout path. PR #72515 is open and uses closing syntax, so this issue should remain open until that PR or a maintainer-selected alternative lands.

## What This Changes

Request fixes MCP streamable-http configs that include both a spawned command and URL so OpenClaw no longer times out after silently choosing stdio.

## Best Possible Solution

Keep the issue open and resolve the product boundary: either land PR #72515 to reject mixed command+url configs with a clear diagnostic matching the docs, or design explicit managed HTTP child-process lifecycle support if OpenClaw should own that mode.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a config with non-empty command, non-empty url, and transport "streamable-http" resolves as stdio on current main and can hit the existing MCP connection timeout path.

## Solution Assessment

Unclear. Rejecting mixed command+url matches current docs and is the narrowest maintainable fix, but if maintainers want OpenClaw to manage spawned HTTP child processes then the best solution is a broader lifecycle design rather than only changing resolution.
