## Summary

Keep open: current main still resolves a mixed non-empty `command` plus `url` MCP server as stdio before the HTTP branch, and the linked closing PR #72515 is still open and unmerged.

## What This Changes

The issue requests fixing MCP `streamable-http` configs that include both a spawned command and URL so OpenClaw does not silently fall back to stdio and time out.

## Best Possible Solution

Review and land or revise PR #72515 so mixed `command` plus `url` configs fail fast with redacted diagnostics and focused resolver tests, then close this issue after the fix merges.

## Reproduction Assessment

Yes. A config with non-empty `command`, non-empty `url`, and `transport: "streamable-http"` currently follows the stdio-first resolver path and can then hit the existing 30000ms connection timeout.

## Solution Assessment

Yes, early rejection is the narrowest maintainable fix for the documented config contract. Supporting spawned HTTP MCP servers would be a separate runtime design, not a small resolver bug fix.
