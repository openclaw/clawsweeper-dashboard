## What This Changes

The issue reports an MCP streamable-http server startup timeout when a server config includes both a spawned command and an HTTP URL.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/72515; only close this issue after the linked resolver fix lands or maintainers explicitly reject the mixed-config behavior change.

## Reproduction Assessment

Yes, source-level: current main resolves a non-empty `command` before considering `url` or `transport`, so the reported mixed streamable-http config takes the stdio path and can hit the 30-second connect timeout. I did not run the external OpenSpace setup.

## Solution Assessment
