## Summary

Keep open: current main still has the mixed `command` plus non-empty `url` resolver bug, and the linked closing PR is open, unmerged, and cleanly targets this issue.

## What This Changes

The issue reports a confusing 30000 ms MCP timeout when a server config combines a spawned command with `transport: "streamable-http"` and an HTTP URL.

## Best Possible Solution

Review and land or revise #72515 so mixed `command` plus `url` configs fail fast with redacted diagnostics and focused resolver coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a config with non-empty `command`, non-empty `url`, and `transport: "streamable-http"` resolves as stdio first and can reach the existing 30000 ms connect timeout path.

## Solution Assessment

Yes. Early rejection matches the documented config contract and is narrower than adding spawned HTTP subprocess lifecycle support, which would be a separate feature.
