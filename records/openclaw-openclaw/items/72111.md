## Summary

Keep open: current main still resolves a mixed MCP server config with `command`, `url`, and `transport: "streamable-http"` as stdio before the HTTP branch can run, which matches the reported timeout path. Open PR #72515 has closing syntax and is the active implementation candidate, so the issue should remain open until that PR or a maintainer-selected alternative lands.

## What This Changes

The issue asks OpenClaw's MCP streamable-http config path to stop timing out when a server entry includes a spawned command plus an HTTP MCP URL and transport hint.

## Best Possible Solution

Review and land #72515, or revise it into the maintainer-preferred equivalent, so ambiguous mixed MCP configs fail early with a clear diagnostic and regression coverage. If managed spawned streamable-http servers are desired, that should be designed as a separate lifecycle feature rather than preserving silent stdio fallback.

## Reproduction Assessment

Yes. Source inspection on current main is enough: a config with non-empty `command`, non-empty `url`, and `transport: "streamable-http"` resolves as stdio and then reaches the existing MCP connection timeout path.

## Solution Assessment

Yes for the documented contract. Rejecting ambiguous mixed `command` plus `url` configs is the narrowest maintainable fix; supporting spawned HTTP servers would require a separate lifecycle design rather than transport inference.
