## Summary

Keep open: current main still resolves a mixed non-empty `command` plus `url` MCP server as stdio before HTTP, while the linked closing PR #72515 is open and unmerged.

## What This Changes

Report asks OpenClaw to avoid a 30000ms timeout when an MCP `streamable-http` config also includes a spawned server command.

## Best Possible Solution

Review and land or revise PR #72515 so ambiguous `command` plus `url` configs fail fast with redacted diagnostics and tests aligned to the documented MCP config contract.

## Reproduction Assessment

Yes. A non-empty `command`, non-empty `url`, and `transport: "streamable-http"` follows the current resolver's stdio-first branch, then can hit the existing 30000ms MCP connection timeout path.

## Solution Assessment

Unclear until maintainers confirm product intent. If OpenClaw does not intend to manage spawned HTTP child processes, early rejection is the narrowest maintainable fix; managed HTTP spawning would be a separate runtime design.
