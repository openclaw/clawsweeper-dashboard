## Summary

Keep open: current main still takes the stdio resolver path for a mixed non-empty `command` plus non-empty `url` MCP server, which can lead to the reported 30000ms timeout, and #72515 is an open unmerged PR that references this issue with closing syntax.

## What This Changes

The issue reports that an MCP `streamable-http` server configured with both `command`/`args` and `url` starts listening but OpenClaw times out while connecting.

## Best Possible Solution

Review and land or revise #72515 so mixed `command` plus `url` configs fail fast with redacted diagnostics and focused resolver coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a config with non-empty `command`, non-empty `url`, and `transport: "streamable-http"` resolves as stdio first and can reach the existing 30000ms connect timeout path.

## Solution Assessment

Yes. Early rejection is the narrowest maintainable fix under the documented config contract; supporting spawned HTTP subprocesses would be a separate runtime feature.
