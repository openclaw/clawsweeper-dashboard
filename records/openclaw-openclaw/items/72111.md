## Summary

Keep open: current main still resolves mixed non-empty `command` plus `url` MCP configs through the stdio path before HTTP handling, and open PR #72515 is the active closing candidate rather than grounds to close this issue before merge.

## What This Changes

The issue asks OpenClaw to handle a streamable-http MCP server config that also supplies a spawned command without ending in a 30000ms connection timeout.

## Best Possible Solution

Review and land or revise #72515 so mixed `command` plus `url` configs fail fast with a redacted diagnostic and focused resolver tests, then close this issue after the fix merges.

## Reproduction Assessment

Yes. A config with non-empty `command`, non-empty `url`, and `transport: "streamable-http"` follows the stdio-first resolver path on current main and can reach the existing 30000ms connection timeout.

## Solution Assessment

Yes, for the bug as currently understood: early rejection is the narrowest maintainable fix and matches the documented config contract. Managed spawned HTTP servers would be a separate feature decision.
