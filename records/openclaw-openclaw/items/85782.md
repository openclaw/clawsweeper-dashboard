## What This Changes

The issue asks for terminal TUI lifecycle/provider failures to render a visible run error, clear the active run, dismiss stale watchdog text, and unblock the next prompt.

## Best Possible Solution

Land a focused TUI handler and regression-test change that routes terminal lifecycle errors for the active run through the same visible error and cleanup semantics as chat `state: "error"`, while preserving retryable lifecycle fallback behavior.

## Reproduction Assessment

Yes. From source inspection, send an active-run lifecycle event with `phase: "error"` without a terminal chat error; current main updates the footer status but leaves the transcript and active-run cleanup path untouched.

## Solution Assessment

Yes. The narrow maintainable fix is to treat terminal lifecycle errors for the active TUI run like terminal chat errors, with focused coverage for visible error rendering, watchdog dismissal, active-run clearing, and retryable lifecycle errors.
