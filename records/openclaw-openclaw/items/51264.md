## Summary

Keep open: current main still has a source-reproducible scoped `sessions.resolve` path that loads the combined all-agent store before applying `agentId`, and the issue is already paired with an open fixing PR that uses closing syntax.

## What This Changes

The issue requests lazy/scoped Gateway session-store loading so hook/session resolution does not eagerly load every agent's sessions.json and trigger OOM under multi-agent load.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/79474; after it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes at source level: current main still routes `sessions.resolve` `sessionId` and `label` lookups through the all-agent combined loader even when `agentId` is present. I did not live-reproduce the OOM because the crash requires a high-memory multi-agent workload.

## Solution Assessment

Yes, the narrow maintainable fix is to pass the existing `agentId` parameter into the combined-store loader for scoped `sessionId` and `label` resolution while preserving unscoped cross-agent lookup behavior.
