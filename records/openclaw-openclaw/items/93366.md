## Summary

Keep open: current main and the latest release still lack the requested compact cron-list mode, the referenced implementation PR was closed unmerged, and duplicate search did not find an open canonical replacement. The remaining decision is maintainer acceptance of the new cron tool/protocol parameter shape.

## What This Changes

Add an optional `compact: true` parameter to cron list so agents can receive lightweight job summaries instead of full cron job objects.

## Best Possible Solution

Have maintainers accept the cron-list API shape, then implement an opt-in compact projection through protocol, gateway, service, and agent tool layers while preserving full-list defaults.

## Reproduction Assessment

Partly: source inspection gives high confidence that current `cron(action="list")` returns full job pages without compact projection. I did not live-create 30 jobs to prove the gateway truncation threshold in this read-only review.

## Solution Assessment

Unclear until maintainers accept the API shape. The requested opt-in compact mode is narrow and backward-compatible, while existing pagination/filter support alone does not remove verbose payload and delivery fields from each listed job.
