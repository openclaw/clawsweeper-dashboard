## Summary

Keep open: current main still contains an agent-facing bundled skill helper that can create live SKILL.md files directly, while the current Skill Workshop prompt and docs require generated durable skill work to go through the proposal workflow.

## What This Changes

The issue asks OpenClaw to align bundled skill-creator scaffolding with Skill Workshop so durable skill creation does not bypass proposal review, scanning, hashes, approval, and rollback metadata.

## Best Possible Solution

Make Skill Workshop the only agent-facing path for durable skill creation, then retire or clearly quarantine init_skill.py behind a documented non-agent fallback with tests.

## Reproduction Assessment

Yes, source-level. Current main shows init_skill.py writing SKILL.md directly while the current prompt and docs require generated durable skill work to go through skill_workshop; I did not execute the helper because this review is read-only and that path writes files.

## Solution Assessment

Unclear as a final implementation. The current direct initializer is not the best shape, but maintainers should choose whether to remove it, convert it to a CLI proposal helper, or retain a narrow non-agent fallback.
