## Summary

Keep open: current main and the latest release still contain the bundled skill-creator initializer that writes a live SKILL.md directly, while Skill Workshop prompt, docs, and service paths still define the proposal-first governed write path.

## What This Changes

The issue asks for skill-creator skill file creation to stop bypassing Skill Workshop's proposal, scan, approval, hash, and rollback workflow.

## Best Possible Solution

Make Skill Workshop the only agent-facing durable skill creation path, then either retire init_skill.py or quarantine it as a documented non-agent fallback with focused tests.

## Reproduction Assessment

Yes, source-level. Current main shows init_skill.py writing SKILL.md directly while the current prompt, docs, and service define skill_workshop as the governed proposal path; I did not execute the helper because this review is read-only and that path writes files.

## Solution Assessment

Unclear as a final implementation. Keeping the direct initializer is not the best shape, but maintainers should choose whether to remove it, route it through the existing workshop CLI/service, or keep a narrow non-agent fallback.
