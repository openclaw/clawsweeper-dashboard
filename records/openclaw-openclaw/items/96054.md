## Summary

Keep open: current main and the latest release still contain the skill-creator initializer that writes live SKILL.md files directly, while Skill Workshop prompt, docs, service, and tests define the governed proposal-first path.

## What This Changes

The issue asks OpenClaw to route bundled skill creation through Skill Workshop proposals instead of letting skill-creator's initializer write live SKILL.md files directly.

## Best Possible Solution

Make Skill Workshop the single agent-facing path for durable skill creation, then remove init_skill.py from agent-routed workflows or clearly quarantine it as a documented non-agent fallback with tests.

## Reproduction Assessment

Yes, source-level: current main shows init_skill.py writing SKILL.md directly, while the prompt, docs, service, and tests define skill_workshop as the governed durable skill path. I did not execute the helper because this review is read-only and that path writes files.

## Solution Assessment

Unclear as a final implementation. The current direct initializer is not the best governed shape, but maintainers should choose whether to remove it, convert it into a Skill Workshop proposal helper, or retain a narrow manual-only fallback.
