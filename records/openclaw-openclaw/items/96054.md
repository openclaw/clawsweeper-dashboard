## Summary

Keep open: current main still has the reported Skill Workshop policy mismatch, and v2026.6.10 still ships the same direct skill-creator scaffold path. This remains a maintainer product/security decision because the fix should define whether init_skill.py is removed, converted into a proposal helper, or retained only as a documented non-agent fallback.

## What This Changes

The issue asks OpenClaw to stop skill-creator from writing live SKILL.md files directly and route durable skill creation through Skill Workshop proposals or an explicit fallback.

## Best Possible Solution

Make Skill Workshop the only agent-facing path for durable skill creation, then remove skill-creator's direct live-write scaffold or clearly quarantine it as a documented non-agent fallback with tests.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the Skill Workshop prompt/docs require generated durable skill work to use skill_workshop, while init_skill.py writes live SKILL.md files directly; I did not execute the helper because this read-only review cannot run a file-writing path.

## Solution Assessment

Unclear as a final implementation. The current direct initializer is not the best shape, but maintainers should choose whether to remove it, convert it into a proposal helper, or retain a narrow non-agent fallback.
