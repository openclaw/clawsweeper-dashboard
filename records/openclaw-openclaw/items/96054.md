## Summary

Keep open: current main still has the direct skill-creator scaffold path while Skill Workshop remains the documented and agent-prompted governed path for generated durable skill work. The remaining work is a security/product boundary decision, not a cleanup close.

## What This Changes

The issue asks OpenClaw to stop letting the bundled skill-creator helper create live SKILL.md files outside the Skill Workshop proposal workflow.

## Best Possible Solution

Make Skill Workshop the only agent-facing durable skill authoring path, then retire init_skill.py or clearly constrain it as a documented non-agent fallback with focused tests.

## Reproduction Assessment

Yes. Source inspection shows init_skill.py writes a live SKILL.md directly while the prompt, service, tests, and docs route generated durable skill work through Skill Workshop proposals; I did not execute the helper because this review is read-only and that path writes files.

## Solution Assessment

Unclear as a final implementation. Routing generated skill creation through Skill Workshop is the right boundary, but maintainers need to choose whether to remove the initializer, convert it to a proposal helper, or retain a narrow non-agent fallback.
