## Summary

Keep open: current main still contains a direct `skill-creator` live `SKILL.md` writer alongside the Skill Workshop rule requiring generated skill work to go through proposals, and no canonical item fully owns that conflict.

## What This Changes

The issue asks OpenClaw to route bundled skill creation through the Skill Workshop proposal workflow instead of allowing `skill-creator` to write live skill files directly.

## Best Possible Solution

Make `skill-creator` workshop-aware for generated agent skill work, while either removing the direct initializer or clearly constraining it to documented manual/offline scaffolding.

## Reproduction Assessment

Yes. Source inspection on current `origin/main` shows both the direct `SKILL.md` writer and the workshop-only generated-skill rule; I did not run the mutating initializer because this review is read-only.

## Solution Assessment

Unclear. A workshop-first direction matches the current prompt and docs, but maintainers need to decide whether the direct initializer should disappear or remain as a non-agent fallback.
