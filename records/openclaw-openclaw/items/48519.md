The issue requests durable provenance metadata that follows user/tool-derived data into persistent state and remains available when future sessions read it.

## Best Possible Solution

Define a versioned framework provenance contract that covers tool outputs, durable writes, and read-time policy across memory, files, scheduled tasks, preferences, transcript hooks, and plugin SDK surfaces.
