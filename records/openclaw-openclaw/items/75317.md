## Summary

Keep open. Current main already has the retry-classifier subset from #76311, but source inspection shows query embeddings still use the timeout-only helper and `memory status` still reports raw scan issues without comparing indexed versus scanned source counts; this PR addresses remaining behavior and I found no blocking correctness or security issue.

This PR broadens memory-core embedding retry classification, routes vector query embeddings through the bounded retry loop, warns when builtin index coverage lags scanned files, updates tests, and adds a changelog entry.
