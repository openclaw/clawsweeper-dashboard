## What This Changes

The PR changes TUI optimistic submit handling so pending user messages are keyed by generated run id, preserved across same-session history reloads, reconciled when history catches up, and cleared on abort/session switch.

## Best Possible Solution

Keep the run-keyed pending-row design, but make successful Esc abort drop the pending user row for the aborted run and cover that with a real `ChatLog` regression before merge.

## Reproduction Assessment

## Solution Assessment
