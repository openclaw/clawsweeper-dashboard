Full review comments:

- **[P2] Avoid overwriting new editor input after abort:** `src/tui/tui.ts:1410`
  - body: `editor.onEscape` restores the aborted draft only after the async `abortActive()` promise resolves. Since normal submit clears the editor before `sendMessage()` runs, a slow abort can overwrite a new draft typed after Esc with the old prompt; restore only when the editor is still empty or otherwise known unchanged.
  - confidence: 0.82
- **[P2] Keep libopus error validation:** `extensions/discord/src/voice/receive-recovery.ts:96`
  - body: libopus-wasm 0.2.0 exports both `OpusErrorCode` and `isOpusError`, but this branch replaces that guard with a raw `code === -4` check. That makes any object with `{ code: -4, operation: "decode" }` look like Opus decode noise, so keep the dependency guard or remove this unrelated Discord change.
  - confidence: 0.9
