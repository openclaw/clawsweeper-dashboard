The branch changes TUI send/abort handling to track pending user prompts by generated run id, preserve and reconcile them across history reloads, and restore/drop an unregistered draft on Esc abort, with related tests and a changelog entry.

## Best Possible Solution

Keep the TUI-local pending-draft design, but route every successful pending-run abort through one shared cleanup path that drops the pending row, clears the draft, and preserves current main's abort ordering after rebase.
