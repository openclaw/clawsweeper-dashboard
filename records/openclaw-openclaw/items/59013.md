## What This Changes

The PR changes `src/agents/session-write-lock.ts` to retry Windows `EPERM` from lock creation and adds mocked regression tests in `src/agents/session-write-lock.test.ts`.

## Best Possible Solution

Handle Windows lock-contention `EPERM` at the active fs-safe/file-lock boundary or a narrow wrapper around `SESSION_LOCKS.acquire`, preserve fast failure for real permission errors, add focused regression coverage, and collect redacted Windows runtime proof.

## Solution Assessment

No, not as-is. The retry direction is reasonable, but the maintainable fix belongs at the active fs-safe/file-lock boundary current main actually uses, not the removed inline loop.
