## Summary

Keep open. Current main and the latest release still route eligible Feishu streamed finals through interactive CardKit messages with no searchable fallback or documented resolution, while the only direct fallback PR closed unmerged over live-search proof and product tradeoff gaps.

## What This Changes

The issue asks OpenClaw to make Feishu streaming-card replies searchable in Feishu conversation search, or to document the limitation/product tradeoff.

## Best Possible Solution

Keep this open as the Feishu searchability/product decision: verify the live tenant search behavior, then either document the CardKit limitation or land an owner-approved fallback/default policy with tests for default-off and opted-in delivery.

## Solution Assessment

Unclear. A default-off `post` fallback is a plausible narrow mitigation and restoring native text bubbles is tracked separately, but maintainers need to choose the Feishu UX/searchability tradeoff before implementation.
