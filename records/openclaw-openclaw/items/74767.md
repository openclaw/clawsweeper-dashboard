## Summary

Keep open. Current main and the latest release still send Feishu streamed final replies as interactive CardKit messages without a searchable text/post fallback, the direct fallback PR was closed unmerged, and the remaining fix needs Feishu owner product direction plus live tenant search proof.

## What This Changes

The issue requests that Feishu streaming-card reply content become searchable in Feishu conversation search, or that OpenClaw document or change the delivery mode when that cannot be guaranteed.

## Best Possible Solution

Feishu owners should choose the product policy for searchable streamed replies, then align dispatcher behavior, config/docs, and tests around that choice with redacted live tenant search proof or an explicit proof override.

## Reproduction Assessment

No, not at high confidence end to end. Source inspection proves streamed finals are emitted as interactive cards with no text/post fallback, but a live Feishu tenant search failure and search-hit proof are still missing.

## Solution Assessment

Unclear pending product direction. A default-off `post` fallback is a narrow implementation, while restoring stricter `renderMode:auto` behavior is another plausible path; maintainers need to choose the desired Feishu UX tradeoff first.
