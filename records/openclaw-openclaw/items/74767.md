## Summary

Keep open. Current main still completes Feishu streaming replies as CardKit interactive-card updates with no searchable plain-text delivery, and the linked open PR uses closing syntax, so the issue should remain open until that PR lands or is deliberately replaced.

## What This Changes

Make Feishu streaming-card replies searchable in Feishu conversation search, likely through a final plain-text fallback or documented limitation.

## Best Possible Solution

Add a Feishu-owned opt-in searchable fallback, or explicitly document the limitation if maintainers reject the visible duplicate-message tradeoff, while preserving current streaming defaults.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and source inspection of current main confirms the OpenClaw precondition: final streamed content remains in a CardKit interactive-card update with no normal text fallback, though I did not run a live Feishu search.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing and the workaround has a visible duplicate-message tradeoff.
