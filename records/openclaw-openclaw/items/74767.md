## Summary

Keep open. Current main and the latest release still route Feishu streamed final replies through `interactive` CardKit messages with no searchable `post` or text fallback, while the focused fallback PR at https://github.com/openclaw/openclaw/pull/74808 closed unmerged because live tenant search proof and product acceptance of duplicate delivery were missing.

## What This Changes

Request that Feishu streaming-card replies remain discoverable through Feishu conversation search, either by adding a searchable fallback, changing delivery shape, or documenting the limitation.

## Best Possible Solution

Keep the issue open until Feishu owners choose and prove either a default-off searchable fallback, a card/text routing change for overlapping plain replies, or explicit documentation that streaming cards are not guaranteed to be searchable.

## Reproduction Assessment

No high-confidence end-to-end Feishu tenant reproduction was established here. Source inspection does show the OpenClaw half: current main sends streamed finals as Feishu `interactive` CardKit messages and has no fallback `post`/text delivery.

## Solution Assessment

Unclear. A default-off fallback is a plausible narrow mitigation, but it is only the best fix if live tenant proof confirms the fallback is searchable and Feishu owners accept the duplicate-message and quota tradeoff.
