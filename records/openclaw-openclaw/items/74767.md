## Summary

Keep open. Current main still sends completed Feishu streaming replies as CardKit interactive-card messages without a searchable plain-text fallback, and PR #74808 remains open with closing syntax for this issue.

## What This Changes

Requests that Feishu streaming-card replies remain searchable in Feishu conversation search, likely via a final plain-text fallback or documented limitation.

## Best Possible Solution

Add a Feishu-owned opt-in searchable fallback that sends completed streaming text as plain text with docs about the duplicate-message tradeoff, then let the linked PR close the issue when merged.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the source precondition: completed card-eligible replies stay in CardKit interactive-card delivery and do not also send a plain-text message.

## Solution Assessment

Yes. Because the indexing limitation is Feishu-specific and a fallback creates visible duplicates, an opt-in Feishu plugin setting plus docs is the narrowest maintainable direction; PR #74808 is already the implementation candidate.
