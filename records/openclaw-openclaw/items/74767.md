## Summary

Keep open. Current main still sends completed Feishu streaming replies as CardKit interactive-card messages without a searchable plain-text fallback, and #74808 remains open with closing syntax for this issue.

## What This Changes

The issue asks for Feishu streaming-card replies to be searchable in Feishu conversation search, or for OpenClaw to provide/document a fallback for the CardKit indexing limitation.

## Best Possible Solution

Review and land or deliberately replace #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the key precondition: completed streaming replies remain interactive CardKit messages while tests assert no normal text fallback is sent.

## Solution Assessment

Yes as a direction. A Feishu-specific opt-in fallback is the narrowest maintainable solution because the limitation is tied to Feishu interactive-card indexing and has a visible duplicate-message tradeoff.
