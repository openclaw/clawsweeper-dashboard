## Summary

Keep open. Current main still routes card-eligible Feishu streaming finals through CardKit interactive-card delivery with no searchable plain-text fallback, and the open #74808 PR is still the active closing implementation path.

## What This Changes

Feishu streaming-card replies should become searchable in Feishu conversation search or gain a documented searchable fallback when streaming is enabled.

## Best Possible Solution

Review and land or deliberately replace #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the duplicate-message tradeoff.

## Reproduction Assessment

Yes, at source level. The issue gives concrete Feishu UI steps, and current main still meets the key precondition: completed streaming replies remain CardKit interactive messages while tests assert no normal text fallback is sent.

## Solution Assessment

Yes as a direction. A Feishu-specific opt-in fallback is the narrowest maintainable approach because Feishu card indexing is channel-specific and an always-on plain-text fallback would visibly duplicate replies.
