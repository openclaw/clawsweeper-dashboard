## Summary

Keep open. Current main still delivers Feishu streaming-card replies as CardKit interactive-card updates without a plain-text searchable fallback, and the linked implementation PR #74808 is open and unmerged with closing syntax for this issue, so the issue should stay open until that PR is reviewed, landed, or intentionally replaced.

## What This Changes

Make Feishu streaming-card replies searchable in Feishu conversation search, likely through a final plain-text fallback or documented platform limitation.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned, opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff. Keep the behavior in the Feishu plugin unless another channel shows the same missing delivery contract.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main matches the required precondition: final streaming replies are delivered through CardKit interactive cards while tests assert no normal text fallback is sent.

## Solution Assessment

Yes as a direction. A Feishu-specific opt-in fallback with documentation is the narrowest maintainable fix because the search limitation is tied to Feishu interactive-card indexing rather than a shared OpenClaw reply contract.
