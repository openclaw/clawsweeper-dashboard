## Summary

Keep open. Current main still delivers eligible Feishu streaming replies as CardKit interactive-card updates without a searchable plain-text fallback, and open PR #74808 explicitly targets this issue but has not merged.

## What This Changes

Make Feishu streaming CardKit replies searchable in Feishu conversation search, or document and mitigate the indexing limitation.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the key precondition: completed streaming replies remain interactive CardKit messages while tests assert no normal text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing rather than a shared OpenClaw reply contract.
