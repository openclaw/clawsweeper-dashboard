## Summary

Keep open. Current main and v2026.6.6 still deliver Feishu streamed finals as `interactive` CardKit messages with no searchable fallback, while the focused fallback PR closed unmerged because live tenant search proof and product acceptance of duplicate delivery were missing.

## What This Changes

The issue requests that Feishu streamed interactive-card replies remain searchable in Feishu conversation search or have the limitation handled or documented.

## Best Possible Solution

Keep this issue open as the canonical Feishu streaming-card searchability limitation until owners choose a live-proven default-off fallback, a card/text default change for the overlapping plain-reply case, or explicit documentation of the limitation.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available from this review. Source inspection does reproduce the OpenClaw half: current main and v2026.6.6 send streamed finals as Feishu `interactive` CardKit messages, and no fallback text/post send exists.

## Solution Assessment

Unclear. A default-off searchable fallback is a plausible narrow mitigation, but it is only the best fix if live tenant proof confirms the exact `post` fallback is searchable and Feishu owners accept the duplicate-message/quota tradeoff.
