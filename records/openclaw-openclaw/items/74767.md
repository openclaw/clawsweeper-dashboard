## Summary

Keep open. Current main still delivers Feishu streaming-card replies through CardKit interactive cards without a searchable plain-text fallback, and the linked PR #74808 is open/unmerged with closing syntax for this issue.

## What This Changes

Make Feishu interactive-card streaming replies searchable in Feishu conversation search, or document/avoid the indexing limitation.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned, opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the key precondition: final streaming replies close through interactive CardKit cards while tests assert no normal text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing, not a shared OpenClaw reply contract.
