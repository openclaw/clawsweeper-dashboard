## Summary

Keep open. Current main still delivers completed Feishu streaming replies as interactive CardKit cards without a searchable plain-text fallback, and the linked closing PR remains open rather than merged, so the issue should stay open for that PR's review or replacement.

## What This Changes

The issue requests that Feishu streaming-card replies remain searchable in Feishu conversation search, or that the limitation be documented if Feishu cannot index updated interactive-card content.

## Best Possible Solution

Land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback and docs that explain the visible duplicate-message tradeoff; keep the behavior in the Feishu plugin unless another channel proves it needs a shared contract.

## Reproduction Assessment

Yes. The issue gives a concrete Feishu UI path, and current main matches the reported precondition: streaming final replies are delivered as interactive CardKit cards while tests assert no normal text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback plus documentation is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing, not to a shared OpenClaw reply contract.
