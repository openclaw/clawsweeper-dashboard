## Summary

Keep open. Current main still sends completed Feishu streaming replies as CardKit interactive messages without a searchable plain-text fallback, and https://github.com/openclaw/openclaw/pull/74808 remains the open implementation candidate tied to this issue.

## What This Changes

The issue asks for Feishu streaming interactive-card replies to become searchable in Feishu conversation search like non-streaming plain-text replies.

## Best Possible Solution

Review and land or deliberately replace https://github.com/openclaw/openclaw/pull/74808 with a Feishu-owned opt-in searchable fallback and documentation of the duplicate-message tradeoff.

## Reproduction Assessment

Yes, there is a high-confidence source-reproducible path: enable Feishu streaming, deliver a card-eligible final answer, and current main keeps the answer in CardKit interactive updates without a plain-text send. A live Feishu search check was not run in this read-only review.

## Solution Assessment

Yes as a direction, but it is not solved on main. A Feishu-specific opt-in fallback is the narrowest maintainable path because the limitation is tied to Feishu CardKit indexing and has a visible duplicate-message tradeoff.
