## Summary

Keep open. Current main still sends completed Feishu streaming replies as CardKit interactive-card messages without a searchable plain-text fallback, and open PR #74808 has closing syntax for this issue, so the issue should remain open until that PR lands or is deliberately replaced.

## What This Changes

Request Feishu streaming card replies to remain searchable in Feishu conversation search, likely through a final plain-text fallback or documented limitation.

## Best Possible Solution

Review and land or replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs describing the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The reporter gives concrete Feishu UI steps, and current main matches the critical precondition: final card replies stay interactive CardKit messages while the dispatcher test asserts no normal text fallback is sent; I did not run a live Feishu search.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing, and enabling a duplicate plain-text message by default would be a visible product change.
