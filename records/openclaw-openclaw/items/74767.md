## Summary

Keep open. Current main still sends completed Feishu streaming replies as CardKit interactive-card references/updates without a searchable plain-text fallback, and open PR #74808 explicitly targets this issue with closing syntax.

## What This Changes

Request searchable Feishu streaming-card replies, or documentation of the Feishu search limitation, because interactive-card updates are not indexed by Feishu conversation search.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced: current main creates a CardKit card_id interactive message, updates the visible content later, and tests assert no plain-text fallback is sent. The reporter provides concrete Feishu UI search steps for the external indexing symptom.

## Solution Assessment

Likely yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing and the fallback has a visible duplicate-message tradeoff that maintainers should approve.
