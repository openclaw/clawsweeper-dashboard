## Summary

Keep open. Current main still delivers completed Feishu streaming replies as CardKit interactive-card messages without a searchable plain-text fallback, and PR #74808 remains open, unmerged, and explicitly targets this issue.

## What This Changes

The issue asks for Feishu streaming-card replies to remain searchable in Feishu conversation search, or for the limitation to be documented.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes for the OpenClaw delivery side: the reporter gives concrete Feishu UI steps, and current main matches the key precondition by keeping completed streaming replies in CardKit without a normal text fallback. The actual Feishu search result was not independently live-tested here.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing and the fallback creates a visible duplicate-message tradeoff.
