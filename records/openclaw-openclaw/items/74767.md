## Summary

Keep open. Current main still delivers completed Feishu streaming replies as CardKit interactive-card messages without a searchable plain-text fallback, and #74808 remains an open closing PR for this issue.

## What This Changes

Request that Feishu streaming-card replies be searchable in Feishu conversation search, or be handled/documented with a searchable fallback.

## Best Possible Solution

Land or replace #74808 with a Feishu-owned opt-in searchable final-text fallback plus docs explaining the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the key precondition: completed streaming replies remain interactive CardKit messages while tests assert no normal text fallback is sent.

## Solution Assessment

Yes as a direction. A Feishu-specific opt-in fallback is the narrowest maintainable approach because the limitation is tied to Feishu interactive-card indexing and has a visible duplicate-message tradeoff.
