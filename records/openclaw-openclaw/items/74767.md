## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit interactive-card updates without a searchable plain-text fallback, and open PR #74808 already targets this issue with closing syntax. The report is recent, concrete, and still matches the current Feishu plugin behavior.

## What This Changes

Request that Feishu streaming-card replies be searchable in Feishu conversation search, either by adding an indexable fallback or by documenting the limitation.

## Best Possible Solution

Review, land, or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback and docs that explain the duplicate-message tradeoff, keeping the behavior inside the Feishu plugin unless another channel proves a shared delivery contract is needed.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI reproduction steps, and current main satisfies the key precondition: final streaming replies are delivered as interactive CardKit cards while tests assert no normal text fallback is sent.

## Solution Assessment

Yes for the proposed direction. A Feishu-specific opt-in final text fallback plus documentation is the narrowest maintainable approach because the limitation is tied to Feishu interactive-card indexing, not a shared OpenClaw reply contract.
