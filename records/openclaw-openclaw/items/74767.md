## Summary

Keep open. Current main still routes Feishu streaming replies through interactive CardKit cards without a plain-text searchable fallback, and the linked fix PR remains open and unmerged, so the issue should stay open for review, landing, or intentional replacement of that PR.

## What This Changes

Request Feishu streaming-card replies become searchable in Feishu conversation search, either through a fallback delivery mode or clear documentation of the limitation.

## Best Possible Solution

Land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback and docs explaining the duplicate-message tradeoff; keep the behavior inside the Feishu plugin unless another channel proves a shared delivery contract is needed.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the precondition: final streaming replies are delivered as interactive CardKit cards while tests assert no normal text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback plus documentation is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing, not a shared OpenClaw reply contract.
