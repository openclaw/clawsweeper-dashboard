## Summary

Keep open. Current main still routes Feishu streaming replies through interactive CardKit cards without a searchable plain-text fallback, and PR #74808 is open/unmerged with closing syntax, so this issue should remain open for review, landing, or deliberate replacement of that PR.

## What This Changes

The issue requests making Feishu streaming-card replies searchable in Feishu conversation search, likely by adding a final plain-text fallback or documenting the Feishu indexing limitation.

## Best Possible Solution

Review and land or intentionally replace PR #74808 with a Feishu-owned, opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff. Keep the behavior inside the Feishu plugin unless another channel proves a shared delivery contract is needed.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the precondition: final streaming replies are delivered through interactive CardKit cards while tests assert no normal text fallback is sent.

## Solution Assessment
