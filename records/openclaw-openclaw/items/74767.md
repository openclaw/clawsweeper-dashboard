## Summary

Keep open. Current main still delivers completed Feishu streaming replies as CardKit interactive messages without a searchable plain-text fallback, and the linked closing pull request remains open rather than merged.

## What This Changes

The issue asks for Feishu streaming card replies to be discoverable in Feishu conversation search or otherwise mitigated/documented.

## Best Possible Solution

Review and land or deliberately replace https://github.com/openclaw/openclaw/pull/74808 with a Feishu-owned opt-in searchable fallback and docs for the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes for the OpenClaw side: the reporter provided concrete Feishu search steps, and current main is source-reproducible for the prerequisite behavior that completed streaming text remains in CardKit interactive updates without a plain-text fallback. I did not rerun live Feishu search in this read-only pass.

## Solution Assessment

Yes as a direction, but it is not solved on main. A Feishu-specific default-off fallback is the narrowest maintainable approach because the limitation is tied to Feishu CardKit indexing and has a visible duplicate-message tradeoff.
