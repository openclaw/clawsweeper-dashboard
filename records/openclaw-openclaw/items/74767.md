## Summary

Keep open. Current main still sends Feishu streaming-card replies through CardKit interactive-card updates without a searchable plain-text fallback, and PR #74808 is open/unmerged with closing syntax for this issue.

## What This Changes

Feishu streaming-card replies should remain searchable in Feishu conversation search, or the limitation should be documented and mitigated.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned, opt-in searchable fallback plus docs explaining the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The report gives concrete Feishu UI steps, and current main matches the key precondition: final streaming-card replies are delivered as CardKit interactive messages while tests assert no normal text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing, not a shared OpenClaw reply contract.
