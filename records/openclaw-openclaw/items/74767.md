## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit interactive messages without a searchable plain-text fallback, and open PR #74808 is the linked implementation candidate with closing syntax for this issue.

## What This Changes

Request to make Feishu streaming-card replies searchable in Feishu conversation search, either by adding a final text fallback or documenting the Feishu limitation.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main matches the key precondition: final streaming-card replies remain CardKit interactive messages with no plain-text fallback while tests assert sendMessageFeishu is not called.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing rather than a shared OpenClaw reply contract.
