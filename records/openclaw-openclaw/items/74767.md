## Summary

Keep open. Current main still sends completed Feishu streaming replies as CardKit interactive cards without a plain-text searchable fallback, and the linked closing PR remains open and unmerged, so this issue should wait for that PR's review or an intentional replacement.

## What This Changes

The issue requests that Feishu streaming-card replies be searchable in Feishu conversation search like non-streaming plain-text replies.

## Best Possible Solution

Review and land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff; keep the behavior local to the Feishu plugin unless another channel proves a shared contract is needed.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main matches the reported precondition: final streaming replies are delivered as interactive CardKit cards while tests assert no normal text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in plain-text fallback with documentation is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing, not to the shared OpenClaw reply contract.
