## Summary

Keep open. Current main and release v2026.6.10 still route Feishu streamed final replies through interactive CardKit messages without a searchable text/post fallback, while the direct fallback PR is closed unmerged and the related default-rendering issue shows the remaining fix needs Feishu product direction.

## What This Changes

The issue asks OpenClaw to make Feishu streaming-card reply content searchable in Feishu conversation search, or choose/document a fallback for that limitation.

## Best Possible Solution

Choose the Feishu delivery policy first, then either restore searchable native text delivery for default plain replies or add a documented opt-in searchable fallback with live tenant proof and explicit duplicate-message tradeoffs.

## Reproduction Assessment

Yes for source-level reproduction: current main emits streamed Feishu final replies as `interactive` CardKit messages and does not emit an added text/post fallback. No for high-confidence live reproduction, because this read-only review did not exercise Feishu tenant conversation search.

## Solution Assessment

Unclear. The closed fallback PR is a plausible narrow repair, but the best fix depends on whether maintainers want an opt-in duplicate/searchable fallback or a default-rendering change that keeps plain replies on the native searchable path.
