## Summary

Keep open. Current main still sends Feishu streaming replies as interactive CardKit cards without a searchable plain-text fallback, and the linked closing PR #74808 is open and unmerged. Because that PR uses closing syntax for this issue, the issue should remain open until the PR lands or is deliberately replaced.

## What This Changes

The issue requests that Feishu streaming interactive-card replies become searchable in Feishu conversation search, or that the limitation be documented clearly.

## Best Possible Solution

Review and land PR #74808, or replace it with an equivalent Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff. Keep the behavior in the Feishu plugin unless another channel needs the same searchable-final-reply contract.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main matches the source-level reproduction: streaming sends an interactive CardKit card and tests assert no plain-text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback plus docs is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing rather than a shared core reply contract.
