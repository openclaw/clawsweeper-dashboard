## Summary

Keep open. Current main still delivers completed Feishu streaming replies as CardKit interactive messages without a searchable plain-text fallback, and the open PR at https://github.com/openclaw/openclaw/pull/74808 explicitly closes this issue but has not merged.

## What This Changes

The issue requests Feishu streaming card replies to remain discoverable in Feishu conversation search, likely through a searchable final text fallback or documented limitation.

## Best Possible Solution

Review and land or deliberately replace https://github.com/openclaw/openclaw/pull/74808 with a Feishu-owned opt-in searchable fallback and docs for the duplicate-message tradeoff.

## Reproduction Assessment

Yes. The reporter provided concrete live Feishu search steps, and current main is source-reproducible for the prerequisite behavior: final streaming content remains in CardKit interactive updates with no plain-text fallback.

## Solution Assessment

Yes as a direction, but it is not solved on main. A Feishu-specific opt-in fallback is the narrowest maintainable path because the limitation is tied to Feishu CardKit indexing and has a visible duplicate-message tradeoff.
