## Summary

Keep open. Current main still routes searchable Feishu streaming content through interactive CardKit card updates without a plain-text/post fallback, and the linked fix PR remains open and unmerged, so this issue should stay paired with that review instead of being closed by cleanup.

## What This Changes

The issue asks Feishu streaming-card replies to remain searchable in Feishu conversation search, or to mitigate/document the CardKit indexing limitation.

## Best Possible Solution

Review, validate, and land PR #74808 if its opt-in fallback is acceptable, or replace it with an equivalent Feishu-owned fallback and docs that clearly explain the visible duplicate-message tradeoff. Keep this behavior inside the Feishu plugin unless more channels prove a shared reply-delivery contract is needed.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main matches the required precondition: final streamed replies are delivered as interactive CardKit cards while tests assert no ordinary message fallback is sent.

## Solution Assessment

Yes for the current direction. A Feishu-specific opt-in searchable fallback plus documentation is narrower and more maintainable than changing shared OpenClaw reply semantics for a Feishu CardKit indexing limitation.
