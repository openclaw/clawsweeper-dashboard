## Summary

Keep open. Current main still delivers completed Feishu streaming replies as CardKit interactive messages without a searchable plain-text fallback, and open PR #74808 is the unmerged implementation candidate for this issue.

## What This Changes

The issue requests that completed Feishu streaming-card replies become searchable in Feishu conversation search, or that the limitation be documented with a supported workaround.

## Best Possible Solution

Feishu should expose an opt-in searchable fallback that sends final streamed text as a normal message and documents the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main matches the key precondition: completed streaming replies remain interactive CardKit messages while tests assert no normal text fallback is sent.

## Solution Assessment

Yes. A Feishu-specific opt-in fallback is the narrowest maintainable direction because the limitation is tied to Feishu interactive-card indexing, not a shared OpenClaw reply contract.
