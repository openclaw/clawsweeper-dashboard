## Summary

Keep open. Current main still completes Feishu streaming replies as CardKit interactive messages without a searchable plain-text fallback, and open PR #74808 already targets this issue with closing syntax, so the issue should stay open until that PR lands or is deliberately replaced.

## What This Changes

Make Feishu streaming-card replies searchable in Feishu conversation search, likely by adding a plain-text fallback or documenting the platform limitation.

## Best Possible Solution

Review and land or deliberately replace #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff.

## Reproduction Assessment

## Solution Assessment

Yes as a direction. A Feishu-specific opt-in fallback is the narrowest maintainable solution because the limitation is tied to Feishu CardKit indexing and has a visible duplicate-message tradeoff.
