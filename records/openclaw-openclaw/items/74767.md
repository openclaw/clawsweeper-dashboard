## Summary

Keep open. Current main and release v2026.6.10 still send Feishu streamed final replies through interactive CardKit messages with no searchable post/text fallback, the direct fallback PR is closed unmerged, and the remaining fix is a Feishu product decision that needs live tenant proof.

## What This Changes

The issue asks OpenClaw to make Feishu streaming-card reply content searchable in Feishu conversation search, or otherwise mitigate/document the interactive-card indexing limitation.

## Best Possible Solution

Keep this open until Feishu owners choose and live-prove a product path: restore stricter renderMode:auto text-vs-card behavior, add a default-off searchable fallback, or document a confirmed tenant-specific Feishu search limitation.

## Reproduction Assessment

Partially. The OpenClaw side is source-reproducible because current main emits streamed final replies as Feishu interactive cards with no post/text fallback; the end-to-end Feishu search-index behavior was not live-reproduced in this review.

## Solution Assessment

Unclear. The closed post fallback PR is a plausible mitigation, but it is only the best solution if live tenant proof confirms the search result and maintainers accept the duplicate-delivery tradeoff; the related renderMode:auto issue covers only a narrower default-behavior subset.
