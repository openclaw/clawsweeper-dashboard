## Summary

Keep open. Current main still sends completed Feishu streaming replies as CardKit interactive messages without a searchable plain-text fallback, and an open PR at https://github.com/openclaw/openclaw/pull/74808 explicitly targets this issue.

## What This Changes

Request that Feishu streaming card replies be searchable in Feishu conversation search, either by adding a searchable fallback or documenting the limitation.

## Best Possible Solution

Review and land or deliberately replace https://github.com/openclaw/openclaw/pull/74808 with a Feishu-owned opt-in searchable fallback and docs for the duplicate-message tradeoff.

## Reproduction Assessment

Yes, source-reproducible with concrete live UI steps from the report. Current main still sends the final answer through CardKit interactive updates and tests assert no plain-text fallback is sent.

## Solution Assessment

Yes as a direction, but it is not solved on main. A Feishu-specific opt-in fallback is the narrowest maintainable path because the limitation is tied to Feishu CardKit indexing and has a visible duplicate-message tradeoff.
