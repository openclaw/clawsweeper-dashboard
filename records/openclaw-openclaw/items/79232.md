## What This Changes

Request that the macOS Gateway LaunchAgent service PATH include `/opt/homebrew/bin` and `/opt/homebrew/sbin` so Homebrew-installed Node can start after `openclaw gateway restart`.

## Best Possible Solution

Keep the PATH policy discussion and any eventual implementation in https://github.com/openclaw/openclaw/issues/59665, preserving this Homebrew-specific evidence there; handle the external APFS LaunchAgent path problem in https://github.com/openclaw/openclaw/issues/60398.

No live macOS LaunchAgent restart was run, so the exit-78 symptom remains report-supported rather than independently reproduced.
