## Summary

Close as superseded by prior outbound-send security triage: current main still has no Slack outbound DM recipient allowlist, but that is not a documented `dmPolicy` bug; it is the same optional outbound-target policy hardening class previously triaged as product work rather than a security-boundary bypass.

## What This Changes

The issue asks for `openclaw message send` to block outbound Slack DMs when the recipient is not included in the configured Slack DM allowlist.

## Best Possible Solution

Keep `allowFrom` as the documented inbound sender policy; any future recipient restriction should be designed as an explicit outbound message target policy with config, docs, and shared enforcement semantics.

## Reproduction Assessment

No, not as a bug. Source inspection confirms current main does not apply `dmPolicy` to outbound Slack recipients, but the documented contract defines that policy as inbound sender access rather than outbound recipient authorization.

## Solution Assessment

No. Reusing Slack `allowFrom` as an outbound recipient ACL would change the documented policy semantics; the safer solution is explicit outbound target policy design, as already covered by the prior outbound-send discussions.
