## Summary

Keep open: current main still has a source-visible Slack outbound DM path that does not consult `dmPolicy`, but the documented `allowFrom` contract is for sender/inbound access, so enforcing it for outbound recipients needs maintainer security/product judgment rather than an automatic bug-fix PR.

## What This Changes

The issue requests blocking Slack outbound DM sends to users outside `channels.slack.allowFrom` when `dmPolicy` is `allowlist`.

## Best Possible Solution

Define a maintainer-approved outbound message target policy, preferably generic across message actions/providers, and only reuse Slack `allowFrom` for outbound recipients if that semantic change is explicitly chosen and documented.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows Slack CLI/message-tool sends can reach the Slack send path without a `dmPolicy` target check, while inbound DMs do use the allowlist helpers.

## Solution Assessment

Unclear. A narrow Slack-only check against `allowFrom` is mechanically possible, but the safer maintainable direction is an explicit outbound recipient policy or a clear maintainer decision to make `dmPolicy` bidirectional.
