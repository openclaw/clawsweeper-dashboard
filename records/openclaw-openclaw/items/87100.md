## Summary

Keep this PR open for now: current main and the latest release still contain the direct `.trim()` crash path, but this branch is not merge-ready because its empty-string fallback can silently keep or skip required credential/text values and its proof is still mock-only. ## What This Changes

The PR changes channel setup token and generic text-input prompts to normalize non-string `prompter.text()` results to empty strings and adds unit coverage for undefined token prompt results.

## Best Possible Solution

Preserve the narrow setup-wizard guard while keeping required credential/text inputs strict, then land this PR with real CLI setup proof or close it only after a stricter proven replacement such as https://github.com/openclaw/openclaw/pull/90284 merges.
