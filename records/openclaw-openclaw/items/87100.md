## Summary

Keep this PR open, but not merge-ready: the linked onboarding crash is still present on current main and v2026.6.9, yet this branch changes malformed required credential/text prompt results into empty strings that can keep or skip values instead of preserving required validation, and it still lacks real CLI setup proof.

## What This Changes

The PR replaces direct `.trim()` calls on channel setup wizard text prompt results with `normalizeOptionalString(raw) ?? ""` in token and generic text-input paths, plus unit tests for undefined token prompt results.

## Best Possible Solution

Preserve the setup-wizard guard while rejecting or re-prompting malformed required credential/text input, then merge after redacted real CLI proof or let the stricter replacement PR land.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.9 still call `.trim()` directly on awaited setup prompt text in the token replacement and generic text-input paths. I did not run the full interactive macOS onboarding sequence in this read-only review.

## Solution Assessment

No, not as submitted. Guarding the setup prompt boundary is the right direction, but required token/text prompt failures should reject or re-prompt instead of becoming keep or skip outcomes.
