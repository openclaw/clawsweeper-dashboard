## Summary

Keep open: current main and v2026.6.8 still default normal Gateway Windows Scheduled Tasks to `gateway.cmd` unless an internal hidden-launcher env var is set, and the focused closing PR is closed unmerged.

## What This Changes

The issue asks the Windows daemon installer to launch the Gateway Scheduled Task through a windowless launcher instead of directly running `gateway.cmd`.

## Best Possible Solution

Make the existing VBS hidden launcher the default for Gateway Windows Scheduled Tasks and updates, while preserving explicit opt-out behavior, cleanup, docs, XML battery settings, and focused daemon coverage.

## Reproduction Assessment

Yes, source-level: current main still selects `gateway.cmd` by default when the hidden-launcher env is unset, and the issue includes concrete Windows 11 steps. I did not run a live Windows desktop reproduction in this read-only review.

## Solution Assessment

Yes: reusing the existing VBS launcher is the narrow maintainable fix for this Scheduled Task console-window bug. The implementation should also update docs and cleanup semantics so upgrade and opt-out behavior stay coherent.
