## What This Changes

Make the Windows installer-created OpenClaw Gateway Scheduled Task launch through a windowless launcher instead of directly running `gateway.cmd`.

## Best Possible Solution

Review and land or replace the linked hidden-launcher PR with native Windows proof, docs alignment, and cleanup coverage, then close this issue as implemented.

Yes, source-level: the normal gateway service env does not set the hidden-launcher flag, and the predicate only selects `gateway.vbs` for explicit truthy values. I did not run a live Windows desktop reproduction in this read-only review.

## Solution Assessment

Yes, reusing the existing VBS hidden launcher is the narrowest maintainable fix for the reported Scheduled Task console window. The current open PR is the right implementation path, but it still needs proof and compatibility cleanup before merge.
