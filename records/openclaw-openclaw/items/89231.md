## Summary

Keep open. Current main and the latest release still leave the gateway Scheduled Task hidden launcher as opt-in only, and the direct fixing PR was closed unmerged, so the reported Windows daemon bug remains.

## What This Changes

The issue requests that Windows installer-created gateway Scheduled Tasks use a windowless launcher instead of launching `gateway.cmd` directly with a visible console.

## Best Possible Solution

Land a narrow Windows daemon fix that makes gateway Scheduled Task launches windowless by default, preserves a deliberate opt-out, removes both launcher variants during cleanup, and updates the Windows service docs.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.8` leave the gateway Scheduled Task on the direct `gateway.cmd` action unless hidden-launcher env is supplied. I did not run a live Windows desktop reproduction in this read-only review.

## Solution Assessment

Yes, reusing the existing VBS hidden-launcher mechanism is the narrowest maintainable fix for the reported launcher bug. The closed unmerged PR is useful source context but still needs native Windows proof plus cleanup and docs coverage before a replacement lands.
