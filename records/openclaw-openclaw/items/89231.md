## Summary

Keep open: current main and the latest release still leave normal Windows Gateway Scheduled Tasks on `gateway.cmd` unless the hidden-launcher env is explicitly set, and the prior closing PR is closed unmerged.

## What This Changes

Windows installer-created Scheduled Tasks should launch the Gateway through a windowless launcher instead of running `gateway.cmd` directly.

## Best Possible Solution

Make the existing VBS hidden launcher the default for Gateway Windows Scheduled Tasks and upgrade existing tasks to that action while preserving `gateway.cmd`, explicit opt-out behavior, XML/battery settings, Startup-folder fallback cleanup, and focused daemon coverage.

## Reproduction Assessment

Yes, source-level: the default launcher predicate still leaves `gateway.cmd` as the normal Scheduled Task action when the hidden-launcher env is unset, and the issue includes Windows 11 reproduction steps. I did not run live Windows desktop proof in this read-only review.

## Solution Assessment

Yes for the repair direction: reusing the existing VBS launcher is the narrowest maintainable fix for the Scheduled Task console window, but the closed unmerged PR means the issue still needs a replacement implementation or renewed PR review.
