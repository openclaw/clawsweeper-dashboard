## Summary

Keep open: current `main` and the latest release still default the Windows gateway Scheduled Task to `gateway.cmd`; the prior exact fix PR was closed unmerged, and the open VBS fallback PR explicitly excludes the main Task Scheduler path.

## What This Changes

The issue requests making the Windows installer-created `OpenClaw Gateway` Scheduled Task launch the gateway through a windowless launcher instead of running `gateway.cmd` directly.

## Best Possible Solution

Switch the gateway Scheduled Task install/update path to the existing hidden VBS launcher by default, preserve an explicit opt-out, update cleanup/docs, and prove the no-console behavior on native Windows.

## Reproduction Assessment

Yes at source level: current `main` and `v2026.6.8` still leave gateway Scheduled Task installs on `gateway.cmd` unless the hidden-launcher env flag is set. I did not perform a live Windows desktop visual reproduction in this read-only review.

## Solution Assessment

Yes: reusing the existing VBS hidden launcher is the narrowest maintainable fix for the reported launcher bug; broader PowerShell ScheduledTask rewrites are a separate Windows service-policy discussion.
