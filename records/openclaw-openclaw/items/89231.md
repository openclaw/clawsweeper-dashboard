## Summary

Keep open: current main and v2026.6.8 still leave the gateway Scheduled Task hidden launcher opt-in for gateway services, while https://github.com/openclaw/openclaw/pull/94923 is an open unmerged candidate using closing syntax for this issue.

## What This Changes

Windows gateway Scheduled Task installs and restarts should use a windowless launcher instead of launching `gateway.cmd` with a visible console.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94923 or an equivalent narrow fix that defaults gateway Scheduled Tasks to the existing hidden VBS launcher and validates install/update behavior.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 still leave gateway Scheduled Task installs on `gateway.cmd` unless the hidden-launcher env flag is set. I did not perform a live Windows desktop visual reproduction in this read-only review.

## Solution Assessment

Yes: reusing the existing VBS hidden launcher as the default for gateway Scheduled Tasks is the narrowest maintainable fix for the reported launcher bug. The current open PR is the active candidate, but it is not merged yet.
