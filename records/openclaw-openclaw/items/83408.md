## Summary

Keep open: current main has Xiaomi-native Anthropic `reasoning_content` replay only when thinking is enabled, while the reported `mimo-v2-flash` config is marked non-reasoning/off by default. The exact off/default path still omits both replay and an explicit Xiaomi-native disabled-thinking payload, so a focused provider-transport fix is still useful.

## What This Changes

The issue asks OpenClaw to preserve and resend Xiaomi MiMo Anthropic-format `reasoning_content` so follow-up turns do not fail with a 400 error.

## Best Possible Solution

Add focused Xiaomi-native Anthropic handling for default-off/non-reasoning MiMo routes: either send an explicit disabled-thinking payload when that is the intended mode, or capture and replay returned `reasoning_content` safely, with a `mimo-v2-flash` regression test.

## Reproduction Assessment

No high-confidence live repro is available because no Xiaomi credentials were exercised. Source inspection does show the reported config's non-reasoning/default-off path will not preserve `reasoning_content` and will not send a Xiaomi-native disabled-thinking toggle.

## Solution Assessment

Unclear whether the reporter's requested preserve-and-replay behavior is the narrowest fix for `reasoning: false`. The maintainable fix should first decide whether Xiaomi-native flash should be explicitly disabled or whether any returned sidecar must be stored and replayed, then cover that exact path in transport tests.
