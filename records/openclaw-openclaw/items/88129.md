## Summary

Keep open: the cleanup split is directionally right for preserving active transcript writers, but the PR introduces a stale-but-report-only lock state without updating doctor output, so users can be told to run a fix command that intentionally will not remove the lock.

## What This Changes

## Best Possible Solution

Keep the live-lock preservation policy, but carry or derive removability in doctor so report-only stale locks are described as protected active writers instead of fixable cleanup targets.

## Solution Assessment

No, not merge-ready as submitted. The cleanup policy is the right shape, but doctor needs a removable/report-only distinction and regression coverage before the operator-facing behavior is safe.
