## Summary

Keep open: the proposed surface fits the plugin hook architecture, but the patch leaves native CLI-backed `before_prompt_build` runs without `scopeEnvelope` and needs the required public docs/changelog follow-through.

## What This Changes

The PR adds a typed `scopeEnvelope` session field, a gateway `sessions.patch` parameter, Swift protocol fields, and passes the value into Pi `before_prompt_build` hook events.

## Best Possible Solution

Land the feature after `scopeEnvelope` is carried through every `before_prompt_build` runtime path, including native CLI providers, with protocol/types/docs/changelog/tests aligned.

## Reproduction Assessment

Yes. A CLI-backed session with `sessionEntry.scopeEnvelope` and a `before_prompt_build` hook would reproduce the gap because `runCliAgent`/`prepareCliRunContext` currently have no session entry or envelope parameter to pass into the shared hook helper.

## Solution Assessment

No. The data model and gateway patch direction are reasonable, but the implementation should use the shared hook helper consistently for both Pi and CLI providers and update the public plugin documentation and release notes.
