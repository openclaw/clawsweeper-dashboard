## Summary

Keep open. The updated branch appears to address the earlier cron propagation, changelog, and docs findings, but the new hook payload still depends on callers passing `sessionEntry`, and the realtime voice consult path resolves a session entry without forwarding it, so session-backed `before_prompt_build` hooks can still miss `scopeEnvelope`.

## What This Changes

This PR adds typed `scopeEnvelope` session metadata to session patching/storage, Swift protocol models, plugin hook event types/docs, and embedded-runner propagation for `before_prompt_build` hooks.

## Best Possible Solution

Thread the resolved session entry through every production session-backed embedded-run caller, including realtime voice consult, then add focused coverage and provide real after-fix hook output.

## Reproduction Assessment

Yes, from source inspection: patch a realtime voice consult session with `scopeEnvelope` and run a `before_prompt_build` hook; the proposed hook payload reads only `params.sessionEntry`, but that caller does not pass the resolved entry. I did not run a live voice scenario.

## Solution Assessment

No. The additive hook/session metadata direction is reasonable, but the implementation is not complete while a session-backed embedded-run path still drops the session entry before hook payload construction.
