## Summary

Keep this PR open: the feature is useful, but the proposed plumbing is incomplete for at least the isolated cron run path and the public API change is missing required release/docs coverage.

## What This Changes

The PR adds a typed `scopeEnvelope` session metadata object, gateway session patch/schema support, Swift protocol model fields, and `before_prompt_build` hook payload plumbing.

## Best Possible Solution

Complete end-to-end propagation for every production embedded-run path that owns a session entry, then add the required changelog/docs/generated API coverage before merge.

## Reproduction Assessment

Yes. Patch a cron session with `scopeEnvelope`, run an isolated cron turn with a `before_prompt_build` hook, and the proposed code still omits `scopeEnvelope` because the cron executor does not forward `params.cronSession.sessionEntry`.

## Solution Assessment

No. Passing an optional `sessionEntry` through only selected callers is not the narrowest reliable solution unless all real session-backed callers are updated or the embedded runner resolves the active entry centrally.
