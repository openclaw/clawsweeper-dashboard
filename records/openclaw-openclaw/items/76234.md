## Summary

Keep this PR open: the additive hook metadata direction is useful, but the current patch still drops the resolved session entry in realtime voice consults, so `before_prompt_build` can receive an undefined `scopeEnvelope` on that session-backed path.

## What This Changes

The PR adds typed `scopeEnvelope` session metadata to session patch/storage surfaces and forwards it into `before_prompt_build` hook events for embedded session-backed runs, with docs, changelog, and generated API metadata updates.

## Best Possible Solution

Thread the resolved realtime voice consult `sessionEntry` into `runEmbeddedPiAgent`, add focused `agent-consult-runtime.test.ts` coverage, and include redacted after-fix hook output showing `scopeEnvelope` delivered.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: patch a realtime voice consult session with `scopeEnvelope` and run a `before_prompt_build` hook. The proposed hook payload reads `params.sessionEntry`, but that caller does not pass the resolved entry.

## Solution Assessment

No. The additive metadata plumbing is a reasonable direction, but the implementation is incomplete while a production session-backed embedded-run caller still drops the session entry before hook payload construction.
