## Summary

Keep open; the direction is useful, but the patch is not merge-ready because cron runs still omit the session entry that the new hook payload depends on, and the public plugin/protocol surfaces need docs, changelog, and generated API proof before merge.

## What This Changes

The PR adds typed `scopeEnvelope` session metadata, gateway patch/schema/client support, and `before_prompt_build` hook payload plumbing.

## Best Possible Solution

Complete end-to-end propagation for every session-backed embedded runner path, then update public docs, changelog, and generated API proof before merge.

## Reproduction Assessment

Yes. Patch a cron session with `scopeEnvelope`, run an isolated cron turn with a `before_prompt_build` hook, and the hook still receives no envelope because the cron executor never forwards `params.cronSession.sessionEntry` into `runEmbeddedPiAgent`.

## Solution Assessment

No. Adding a typed optional payload is a reasonable API shape, but the current implementation only forwards the source session entry through selected callers; the safer fix is to pass the active entry at every session-backed call site or resolve it centrally before hook execution.
