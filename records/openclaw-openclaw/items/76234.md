## Summary

This PR should stay open because the proposed hook payload is directionally useful but incomplete: `scopeEnvelope` still would not reach CLI-backed prompt builds or isolated cron embedded runs, and the public docs/changelog coverage required for a user-facing plugin hook and gateway patch feature is missing.

## What This Changes

The PR adds typed `scopeEnvelope` session metadata, gateway session patch support, Swift protocol fields, session-state preservation, and `before_prompt_build` hook payload plumbing.

## Best Possible Solution

Complete end-to-end `scopeEnvelope` propagation for every production `before_prompt_build` runner, then land the matching docs, changelog, and API/protocol validation.

## Reproduction Assessment

Yes. Patch a session with `scopeEnvelope`, then run an isolated cron turn or CLI-backed turn with a `before_prompt_build` hook; the proposed code still omits the field because those paths do not pass session metadata into the shared hook preparation.

## Solution Assessment

No. The schema and event-shape direction is reasonable, but the implementation is not yet the narrowest reliable fix because session-backed hook paths are updated selectively instead of consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Thread scopeEnvelope through CLI prompt builds:** `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:152`
  - body: `resolvePromptBuildHookResult` is shared by embedded and CLI runners, but the PR only adds `scopeEnvelope` to embedded-run params. CLI-backed turns still call `prepareCliRunContext` without any session entry or scope envelope, so `before_prompt_build` hooks on supported CLI providers will continue receiving `undefined`. Please add the field to the CLI params/callers or resolve the active session metadata before this shared hook call.
  - confidence: 0.87
- **[P2] Pass the cron session entry into embedded runs:** `src/agents/pi-embedded-runner/run/attempt.ts:2568`
  - body: This hook payload now reads from `params.sessionEntry`, but the isolated cron executor already has `params.cronSession.sessionEntry` and still calls `runEmbeddedPiAgent` without forwarding it. Cron sessions patched with `scopeEnvelope` would therefore lose the field before `before_prompt_build`; pass the cron session entry or centralize session-entry resolution.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/gateway/protocol/schema/sessions.ts:212`
  - body: This is a user-facing feature that changes the public plugin hook payload and gateway session patch surface, but the patch does not update `CHANGELOG.md`. Repo policy requires user-facing `feat` changes to be recorded before merge.
  - confidence: 0.88
- **[P3] Document the new hook field:** `src/plugins/hook-before-agent-start.types.ts:25-27`
  - body: The public plugin hooks docs still describe `before_prompt_build` as receiving only the prompt and session messages. After adding `scopeEnvelope` to the hook event, plugin authors need that field documented on the hooks page.
  - confidence: 0.84
