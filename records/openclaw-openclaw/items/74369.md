## Summary

Keep open for maintainer review. Current main and the latest release still lack the composer attributes, and this PR remains the focused open candidate for the linked numeric-input corruption bug; the remaining decision is whether maintainers accept the composer-wide typing-assistance tradeoff.

## What This Changes

The PR adds plain-text input attributes to the Control UI chat composer textarea and adds a regression assertion for those rendered attributes.

## Best Possible Solution

Land the composer-boundary mitigation only if maintainers accept the global typing-assistance tradeoff; otherwise choose a narrower input policy before merge.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection shows the composer forwards textarea.value into the send path, so browser/OS rewriting at the textarea boundary would reach chat.send.

## Solution Assessment

Yes, the composer textarea is the right boundary to prevent value rewriting before draft capture. Downstream deformatting would be broader and could corrupt intentionally formatted text, but maintainers still need to accept the typing-assistance tradeoff.
