Keep open for maintainer review. The patch is a focused, plausible fix for the linked numeric Control UI composer corruption, and current main still lacks it, but it intentionally changes browser typing-assistance defaults for every composer user.

## What This Changes

The PR adds five plain-text attributes to the Control UI chat composer textarea and adds a regression assertion for those attributes.

## Best Possible Solution

Land the composer-textarea boundary fix only after maintainers explicitly accept the global typing-assistance tradeoff, and avoid downstream numeric deformatting that could corrupt intentional separators.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection supports the report because current main captures textarea.value and sends that draft without numeric repair, but the browser/OS formatter trigger itself was not recreated here.

## Solution Assessment

Yes, the composer textarea is the best fix boundary for the reported numeric corruption because it prevents rewriting before draft capture. Downstream deformatting would be broader and could corrupt intentionally formatted text, but maintainers still need to accept the typing-assistance tradeoff.
