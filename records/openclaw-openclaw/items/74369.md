## What This Changes

Adds five plain-text input attributes to the Control UI chat composer textarea and a regression test asserting those rendered attributes.

## Best Possible Solution

Land this focused textarea-boundary mitigation if maintainers accept the composer-wide typing-assistance tradeoff; otherwise choose a narrower composer input policy before merging.

## Reproduction Assessment

No, no high-confidence live reproduction of the original browser/OS formatter trigger was established in this read-only pass. Source inspection shows textarea.value is what reaches chat.send, so any boundary rewrite would be preserved into the outgoing payload.

## Solution Assessment

Yes, the composer textarea is the right boundary to prevent browser-side value rewriting before draft capture. Downstream deformatting would be broader and could corrupt intentionally formatted text, but maintainers still need to accept the typing-assistance tradeoff.
