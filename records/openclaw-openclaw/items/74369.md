## What This Changes

The PR adds autocomplete, autocorrect, autocapitalize, spellcheck, and inputmode attributes to the Control UI chat composer textarea, plus a rendered-attribute regression test.

## Best Possible Solution

Land this focused textarea-boundary fix if maintainers accept the composer-wide typing-assistance tradeoff; otherwise define a narrower composer input policy before merging.

## Reproduction Assessment

No high-confidence live reproduction of the original browser or OS formatter trigger was established in this read-only pass. Source inspection shows textarea-side rewriting would flow into the outgoing `chat.send` payload because the app forwards the textarea value unchanged.

## Solution Assessment

Yes, conditionally: the composer textarea is the narrowest visible boundary to prevent browser-side value rewriting before draft capture. The maintainer choice is whether disabling autocomplete, autocorrect, autocapitalize, spellcheck, and text input hints for all composer drafts is acceptable.
