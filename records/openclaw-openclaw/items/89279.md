## What This Changes

The branch changes task registry terminal delivery and regression tests so successful ACP parent-review completions can direct-send to an explicit Discord channel thread when the requester origin has channel, target, and thread metadata.

## Best Possible Solution

Land this branch or an equivalent narrow fix after maintainers record the proof decision and owner-boundary acceptance; if the pattern expands, move the predicate behind a generic or channel-owned delivery capability.

## Reproduction Assessment

Yes. Current main source shows successful ACP child-session parent-review completions queue through the requester session for group/channel owner sessions instead of direct-sending to a bound Discord thread; I did not run live Discord in this read-only review.

## Solution Assessment

Unclear as the final product shape, but acceptable as a narrow mitigation if maintainers accept the core exception. The safer long-term shape is a generic or channel-owned delivery capability if more channel-specific cases appear.
