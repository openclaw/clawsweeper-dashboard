## Summary

Keep this PR open. The feature direction is narrow and useful for local-model token pressure, and current main does not already implement it, but the proposed patch is incomplete: it reads a new config key without adding it to the manual config type, leaves the generated public schema/metadata stale, and lacks the docs/changelog coverage required for a user-facing config option.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` setting so primary embedded-agent sessions can choose `full` or `minimal` prompts while subagent and cron sessions remain forced to `minimal`.

## Best Possible Solution

Land this as a complete config contract: add the `promptMode` union to manual config types, keep Zod/generated schema/UI labels/help aligned, document the operator-facing token-pressure knob in config and token-use docs, add a changelog entry, preserve `full` as the default, and explicitly decide the compaction behavior.

## Reproduction Assessment

Yes. Current main has a focused helper and tests showing regular or undefined sessions resolve to `full`, while subagent and cron sessions resolve to `minimal`; the PR's type-contract issue is reproducible from the new config read against `OpenClawConfig` while `AgentDefaultsConfig` lacks `promptMode`.

## Solution Assessment

No, not as currently proposed. The config knob is a narrow maintainable direction, but the patch needs the missing type, generated-schema, metadata, docs, changelog, and compaction-semantics follow-through before it is safe to merge.
