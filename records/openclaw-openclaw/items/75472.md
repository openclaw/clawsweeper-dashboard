## Summary

Keep open. The feature is plausible, but the current patch is incomplete for a public config knob: it misses the manual config type, does not apply to compaction prompts, leaves config help/docs/baseline drift, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` setting intended to let primary embedded-agent sessions use `full` or `minimal` system prompts while keeping subagent and cron sessions minimal.

## Best Possible Solution

Land this only as a complete config contract: preserve `full` as the default, force subagent and cron runs to `minimal`, apply the configured primary-session mode in normal and compaction prompt builders, align type/schema/help/label/docs/baseline/tests, and require real behavior proof.

## Reproduction Assessment

Yes for the current behavior from source: primary sessions resolve to `full`, subagent and cron sessions resolve to `minimal`, and current main has no `agents.defaults.promptMode` config path. This is a feature request, not a broken existing contract.

## Solution Assessment

No, not as currently proposed. The config knob is a narrow maintainable direction, but this patch must complete the config contract, compaction path, docs/baseline, and real behavior proof before merge.

- **[P1] Declare promptMode on agent defaults config:** `src/agents/pi-embedded-runner/run/attempt.ts:1138`
  - body: The new `params.config?.agents?.defaults?.promptMode` read goes through `OpenClawConfig`, but `AgentDefaultsConfig` does not declare this field. Add `promptMode?: "full" | "minimal"` to the manual config type so strict type lanes and typed config consumers accept the new path.
  - confidence: 0.93
- **[P2] Apply promptMode to compaction prompts:** `src/agents/pi-embedded-runner/run/attempt.ts:1139-1141`
  - body: This threads the configured mode only through normal embedded attempts. `compact.ts` still hard-codes primary compaction prompts to `full` while building the same embedded system prompt, so operators using `agents.defaults.promptMode: "minimal"` would still pay the full prompt cost during compaction calls.
  - confidence: 0.88

  - body: The patch adds `FIELD_HELP["agents.defaults.promptMode"]` without a matching `FIELD_LABELS` entry. `schema.help.quality.test.ts` requires every help key to have a label, so the config help guard will fail until the label and UI hint metadata are added.
  - confidence: 0.9
- **[P2] Refresh the config docs baseline:** `src/config/zod-schema.agent-defaults.ts:88-94`
  - body: Adding a new Zod config key changes the generated config documentation baseline. Regenerate and commit `docs/.generated/config-baseline.sha256` so `pnpm config:docs:check` does not report schema/help drift.
  - confidence: 0.86
- **[P3] Document the user-facing prompt mode:** `src/config/schema.help.ts:205-206`
  - body: This creates a user-facing config feature, but the current system-prompt docs still say `promptMode` is not user-facing config and the agent config docs do not explain the new setting. Update the docs so operators can discover the knob and understand what `minimal` removes.
  - confidence: 0.84
