## Summary

Keep open: this is a plausible active feature PR, but the patch is not merge-ready because it leaves the public config contract incomplete, misses the compaction prompt path, and still lacks contributor real behavior proof for the new setting.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` config path, threads it into normal embedded-agent prompt-mode selection, and adds focused helper tests plus schema/help text.

## Best Possible Solution

Land this only as a complete config contract: keep `full` as the default, keep subagent and cron runs minimal, apply the configured primary-session mode in normal and compaction prompt builders, align type/schema/help/labels/docs/baseline/tests, and require real behavior proof.

## Reproduction Assessment

Yes for current behavior from source: current main resolves ordinary sessions to `full`, subagent/cron sessions to `minimal`, and has no `agents.defaults.promptMode` config path. This is a feature request, not a broken existing contract.

## Solution Assessment

No, not as currently proposed. The config knob is a plausible direction, but this patch must complete the config contract, compaction path, docs/baseline, and real behavior proof before merge.

Full review comments:

- **[P1] Declare promptMode on agent defaults config:** `src/agents/pi-embedded-runner/run/attempt.ts:1138`
  - body: The new `params.config?.agents?.defaults?.promptMode` read goes through `OpenClawConfig`, but `AgentDefaultsConfig` does not declare this field. Add `promptMode?: "full" | "minimal"` to the manual config type so strict type lanes and typed config consumers accept the new path.
  - confidence: 0.93
- **[P2] Thread promptMode through compaction prompts:** `src/agents/pi-embedded-runner/run/attempt.ts:1139-1141`
  - body: This threads the configured mode only through normal embedded attempts. `compact.ts` still hard-codes primary compaction prompts to `full` while building the same embedded system prompt, so operators using `agents.defaults.promptMode: "minimal"` would still pay the full prompt cost during compaction calls.
  - confidence: 0.88
- **[P2] Add promptMode label metadata:** `src/config/schema.help.ts:205-206`
  - body: The patch adds `FIELD_HELP["agents.defaults.promptMode"]` without a matching `FIELD_LABELS` entry. `schema.help.quality.test.ts` requires every help key to have a label, so the config help guard will fail until the label metadata is added.
  - confidence: 0.9
- **[P2] Refresh the config docs baseline:** `src/config/zod-schema.agent-defaults.ts:88-94`
  - body: Adding a new Zod config key changes the generated config documentation baseline. Regenerate and commit `docs/.generated/config-baseline.sha256` so `pnpm config:docs:check` does not report schema/help drift.
  - confidence: 0.86
- **[P3] Document the user-facing prompt mode:** `src/config/schema.help.ts:205-206`
  - body: This creates a user-facing config feature, but the current system-prompt docs still say `promptMode` is not user-facing config and the agent config docs do not explain the new setting. Update the docs so operators can discover the knob and understand what `minimal` removes.
  - confidence: 0.84
