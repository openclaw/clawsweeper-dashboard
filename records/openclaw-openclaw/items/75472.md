## Summary

Keep this PR open. The feature is useful and not implemented on current main, but the branch is not merge-ready because it adds the prompt-mode read without completing the typed config contract, generated schema/UI metadata, compaction path, docs, and changelog.

## What This Changes

This PR adds optional `agents.defaults.promptMode` support so primary embedded-agent sessions can choose `full` or `minimal` prompts while subagent and cron sessions stay minimal.

## Best Possible Solution

Land this as a complete public config contract: default to `full`, keep subagent/cron forced to `minimal`, apply the setting in both normal and compaction prompt builders, and align types, schemas, generated metadata, docs, tests, and changelog.

## Reproduction Assessment

Yes. Current main can be verified from source: primary sessions resolve to `full`, subagent/cron sessions resolve to `minimal`, and the PR diff shows an incomplete config read plus schema/help additions.

## Solution Assessment

No, not as currently proposed. The config knob is the right narrow boundary for local-model token pressure, but the patch needs the missing contract, generated-schema, compaction, and documentation pieces before merge.
