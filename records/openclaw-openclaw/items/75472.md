## Summary

Keep this PR open, but it is not ready to merge. Current main still lacks a configurable primary-session prompt mode, and the PR is a reasonable narrow direction, but it leaves the config type, generated/public schema metadata, docs/changelog, and compaction prompt path incomplete.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` setting so primary embedded-agent sessions can choose `full` or `minimal` prompts while subagent and cron sessions stay forced to `minimal`.

## Best Possible Solution

Land this as a complete config contract: keep `full` as the default, force subagent/cron to `minimal`, apply the configured primary-session mode consistently in normal and compaction prompt builders, and align types, schemas, generated metadata, docs, tests, and changelog.

## Reproduction Assessment

Yes. Current main can be verified from source: primary sessions resolve to `full`, subagent/cron resolve to `minimal`, and the PR's branch diff shows a config read without the matching `AgentDefaultsConfig` and generated-schema surfaces.

## Solution Assessment

No, not as currently proposed. The config knob is a narrow maintainable solution for local-model token pressure, but the patch must complete the public config contract and compaction path before merge.
