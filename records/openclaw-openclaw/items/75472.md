## Summary

Keep open. Current main does not already implement this feature, and the requested opt-in prompt-size reduction is plausible, but the branch is not merge-ready: it adds a typed config read without updating the manual config type and leaves the generated public config schema, docs, and changelog stale.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` setting to choose full or minimal system prompts for primary embedded-agent sessions, with helper tests and schema/help additions.

## Best Possible Solution

Land this as a complete config contract: add `promptMode` to the manual config type, regenerate public schema/hints, document the key in config and token-pressure docs, add a user-facing changelog entry, and preserve the existing default-full plus subagent/cron-minimal behavior.

## Reproduction Assessment

Yes. Current main has a focused helper path showing primary sessions resolve to `full`, and the proposed runtime access would be checked by the TypeScript lane because `OpenClawConfig` currently lacks `agents.defaults.promptMode`.

## Solution Assessment

No, not as currently proposed. The field is a reasonable narrow solution, but the patch needs config type, generated schema, docs/changelog, and compaction-path consistency work before it is the maintainable implementation.
