## Summary

Keep this PR open, but it still needs changes before merge. The proposed config knob is a plausible feature for local-model token pressure, but the patch does not complete the config contract, misses the compaction prompt path, and lacks after-fix real behavior proof from a running setup.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` setting so primary embedded-agent sessions can choose `full` or `minimal` prompts while subagent and cron sessions stay forced to `minimal`.

## Best Possible Solution

Land this only as a complete config contract: keep `full` as the default, force subagent and cron runs to `minimal`, apply the configured primary-session mode in normal and compaction prompt builders, align type/schema/help/label/docs/baseline/tests, and require real behavior proof.

## Reproduction Assessment

Yes for the current behavior, from source: primary sessions resolve to `full`, subagent and cron sessions resolve to `minimal`, and current main has no `agents.defaults.promptMode` config path. This is a feature request rather than a broken existing contract.

## Solution Assessment

No, not as currently proposed. The config knob is a narrow maintainable direction for local-model token pressure, but the patch must complete the public config contract, compaction path, docs, and real behavior proof before merge.

  - body: The new `params.config?.agents?.defaults?.promptMode` read is typed through `OpenClawConfig`, but `AgentDefaultsConfig` does not declare `promptMode`. Add the `"full" | "minimal"` field to the manual config type so the production type lane accepts this config path.
