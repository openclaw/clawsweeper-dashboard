## Summary

Keep this PR open. Current main does not implement configurable `agents.defaults.promptMode`, so the contribution is not obsolete, but the patch is not merge-ready because it reads an undeclared config type, leaves generated public config metadata stale, and still needs user-facing docs/changelog plus a decision on compaction-path consistency.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` setting so primary embedded-agent sessions can use `full` or `minimal` system prompts while subagent and cron sessions remain forced to `minimal`.

## Best Possible Solution

Land this as a complete config contract: add the `promptMode` union to the manual config types, keep the Zod schema and generated public schema/UI metadata aligned, document the operator-facing token-pressure knob, add a changelog entry, preserve `full` as the default, and explicitly handle or document compaction behavior.

## Reproduction Assessment

Yes. Current main has a focused helper and tests showing regular/undefined sessions resolve to `full`, while subagent and cron sessions resolve to `minimal`; the PR's type-contract issue is also reproducible from the missing `AgentDefaultsConfig.promptMode` declaration.

## Solution Assessment

No, not as currently proposed. The config knob is a narrow maintainable direction, but the implementation needs the missing type/generated-schema/docs/changelog surfaces before it is safe to merge.
