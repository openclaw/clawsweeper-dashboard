## Summary

Keep this PR open. Current main still hard-codes primary embedded sessions to full prompt mode, so the feature is not obsolete, but the proposed patch is not ready to merge because it leaves the manual config type, generated public schema/UI metadata, docs/changelog, and compaction prompt path incomplete.

## What This Changes

The branch adds an optional `agents.defaults.promptMode` setting so primary embedded-agent sessions can choose `full` or `minimal` prompt mode while subagent and cron sessions remain forced to `minimal`, with helper tests and schema help updates.

## Best Possible Solution

Land the feature only as a complete config contract: add a typed `full`/`minimal` agent-defaults field, keep Zod, generated schema, labels/help, docs, and changelog aligned, preserve `full` as the default, keep subagent and cron sessions forced to `minimal`, and either apply the setting consistently to compaction prompt builds or document and test a deliberate exclusion.

## Reproduction Assessment

Yes. Current main can be verified from source: regular or undefined sessions resolve to `full`, `runEmbeddedAttempt` does not read `agents.defaults.promptMode`, and the PR diff reads a config property that the manual `AgentDefaultsConfig` type does not declare.

## Solution Assessment

No, not as currently proposed. The config knob is a narrow maintainable direction, but the implementation needs the missing type/generated-schema/docs/changelog surfaces and compaction handling before it is safe to merge.
