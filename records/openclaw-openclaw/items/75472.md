## Summary

Keep this PR open. The feature direction is narrow, but the current patch is not mergeable because it reads an undeclared config field, leaves compaction prompts on the old full-mode path, and does not complete the generated schema, label/help, docs, and changelog surfaces.

## What This Changes

The PR adds an optional `agents.defaults.promptMode` knob and helper tests so primary embedded sessions can use `full` or `minimal` prompts while subagent and cron sessions remain minimal.

## Best Possible Solution

Land this as a complete config contract: keep `full` as the default, force subagent and cron sessions to `minimal`, apply configured primary-session mode to normal and compaction prompt builders, and align types, Zod schema, generated schema, labels/help, docs, tests, and changelog.

## Reproduction Assessment

Yes. Current main source shows primary sessions resolve to `full` while subagent and cron sessions resolve to `minimal`, and the PR diff is enough to reproduce the review blockers by checking the missing type/schema/docs surfaces.

## Solution Assessment

No, not as currently patched. A default-level prompt-mode knob is a narrow maintainable solution for local-model token pressure, but it must complete the public config contract and compaction path before merge.
