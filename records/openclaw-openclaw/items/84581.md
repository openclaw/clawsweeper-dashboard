## Summary

This PR should stay open: latest head fixes the earlier custom-provider marker gap, but the new migration still runs only after skip/noop returns, so a catalog-only existing `models.json` can remain prompt-facing with a plaintext key. The supplied real behavior proof is also stale relative to the latest marker-preservation commit.

## What This Changes

The PR strips plaintext provider `apiKey` values from generated agent `models.json`, migrates eligible legacy keys into auth profiles, preserves marker compatibility, and adds regression tests.

## Best Possible Solution

Keep the prompt-facing catalog secret-free across generated, merge-mode, and catalog-only paths, preserve non-secret provider markers, and land only after latest-head proof plus explicit upgrade-safety acceptance.
