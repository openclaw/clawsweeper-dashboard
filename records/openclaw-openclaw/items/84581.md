## Summary

Needs changes before merge: the no-plaintext `models.json` direction is security-positive and the supplied proof is strong, but the branch still risks making built-in providers with only a literal config `apiKey` appear unavailable through PI model discovery after stripping the catalog key.

The PR strips plaintext provider `apiKey` values from generated and legacy `models.json`, migrates eligible legacy catalog keys into `auth-profiles.json`, preserves non-secret markers, and adds agents regression coverage.

## Best Possible Solution

Keep `models.json` prompt-safe, but bridge configured and migrated credentials into PI discovery through a non-prompt-facing auth source so built-in and custom provider availability stays aligned with runtime auth.
