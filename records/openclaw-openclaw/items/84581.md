## Summary

Keep open: this PR addresses a real prompt-facing secret exposure, but it currently introduces a blocking custom-provider auth regression for config-only keys.

## What This Changes

The PR strips plaintext model provider `apiKey` values from generated and legacy `models.json`, migrates eligible legacy keys into `auth-profiles.json`, preserves non-secret markers, and adds agents regression coverage.

## Best Possible Solution

Strip prompt-facing catalog secrets only after config-backed and migrated provider credentials remain available to runtime request auth, with release guidance for the storage change.

## Reproduction Assessment

Yes, at source level. The PR branch writes `secretref-managed` for config-only custom-provider model rows, while current runtime discovery does not load the config key into auth storage and upstream request auth treats provider `apiKey` as env-or-literal.

## Solution Assessment

No, not yet. The stripping direction is right, but it must preserve runtime request auth for config-only custom providers before replacing `models.json` keys with a non-secret marker.
