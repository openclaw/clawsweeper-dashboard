The PR changes agent model-catalog planning/writing to strip plaintext provider `apiKey` values, migrate legacy `models.json`-only keys into auth profiles, preserve custom-provider marker compatibility, and add regression tests.

## Best Possible Solution

Keep generated `models.json` prompt-safe across config-backed and catalog-only providers, preserve source-managed env markers, and land only after fresh latest-head proof plus explicit maintainer acceptance of the fail-closed upgrade behavior.

## Solution Assessment

No. The direction is right, but the maintainable fix needs migration or stripping before early returns and a marker-aware strip step that preserves source-managed env refs.
