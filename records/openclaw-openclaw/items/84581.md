## Risks / Open Questions

- Existing scripts or workflows that read plaintext credentials directly from `agents/*/agent/models.json` will stop finding them after this PR lands.
- If auth-profile persistence fails during legacy migration, catalog refresh now fails closed and leaves the old file unchanged; that protects credentials but can interrupt upgrade until the auth store is writable.
- A legacy `models.json` key is not migrated when another auth profile already exists for that provider, so maintainers should explicitly accept the profile-precedence behavior.
- The supplied real-behavior proof uses temporary local agent directories and redacted dummy keys rather than a paid third-party provider request.
