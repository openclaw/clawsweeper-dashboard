## Summary

Keep open: the PR addresses a real prompt-facing secret leak and the supplied proof is strong, but a blocking auth upgrade regression remains because the migration checks merged auth profiles before stripping a local legacy catalog key.

## What This Changes

The branch strips plaintext provider `apiKey` values from generated and legacy agent `models.json`, migrates eligible legacy keys into `auth-profiles.json`, preserves non-secret markers, and adds model-config regression coverage.

## Best Possible Solution

Land the strip-and-migrate approach only after the migration guard checks target-agent persisted profiles separately from main-store and external overlays, with focused upgrade coverage for that case.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro path: create a non-main agent with a local legacy `models.json` plaintext key and a main/external auth profile for the same provider; the PR's merged-store check skips migration before the stripped catalog is written.

## Solution Assessment

No, not yet. The strip-and-migrate direction is the right narrow fix for the leak, but it needs target-agent-local profile semantics before it is safe to merge.
