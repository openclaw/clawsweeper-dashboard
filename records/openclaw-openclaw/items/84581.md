## Label Justifications

- `P1`: The PR addresses provider API keys being persisted in a prompt-facing model catalog, which is an urgent security/auth boundary issue.
- `merge-risk: 🚨 compatibility`: Merging intentionally removes plaintext provider keys from `models.json`, which can break existing external or local consumers that read that file as a credential source.
- `merge-risk: 🚨 auth-provider`: Merging changes the upgrade auth path by moving legacy catalog keys into `auth-profiles.json` and relying on existing auth-profile/config resolution afterward.
- `merge-risk: 🚨 security-boundary`: The changed files directly control where provider credentials are persisted and whether prompt-facing catalog data can contain secrets.

## Summary

Keep open for maintainer review. The patch appears security-positive and I found no blocking correctness defect, but it intentionally changes credential storage and removes plaintext `models.json` key availability, so maintainers should explicitly accept the compatibility, auth-provider, and security-boundary impact before merge.

## What This Changes

The PR strips plaintext provider `apiKey` values from generated and legacy `models.json`, migrates eligible legacy catalog keys into `auth-profiles.json`, preserves non-secret pi compatibility markers, and adds agent regression coverage.

## Best Possible Solution

Land a prompt-safe catalog boundary that never persists plaintext provider keys in `models.json`, with legacy keys moved to the existing auth-profile store and release notes that call out the intentional compatibility change.

Yes, at source level. Current main can serialize provider `apiKey` values into `models.json`, including merge-preserved legacy keys, while the PR body supplies live-output proof that the branch strips and migrates those keys.

## Solution Assessment

Yes, pending maintainer acceptance of the compatibility break. The PR keeps `models.json` prompt-safe, preserves pi custom-provider discovery with a non-secret marker, and migrates legacy keys before writing the stripped catalog.
