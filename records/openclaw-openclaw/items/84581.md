## Summary

Keep this PR open for maintainer/security-owner review: the patch appears functionally sound and security-positive, but it intentionally changes the credential boundary for existing `models.json` plaintext keys.

## What This Changes

The PR strips plaintext provider `apiKey` values from generated and legacy agent `models.json`, migrates eligible legacy keys into `auth-profiles.json`, preserves non-secret markers, and adds focused agent regression tests.

## Best Possible Solution

Land a prompt-safe catalog boundary that never persists plaintext provider keys in `models.json`, with legacy keys moved to auth profiles and release/docs notes for the intentional compatibility change.

## Reproduction Assessment

## Solution Assessment

Yes, pending maintainer acceptance of the compatibility break. The patch keeps `models.json` prompt-safe, preserves custom-provider discovery with a non-secret marker, and migrates legacy keys before writing the stripped catalog.
