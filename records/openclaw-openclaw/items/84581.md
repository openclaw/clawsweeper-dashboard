## Summary

Keep this PR open: it addresses a real prompt-facing credential leak, but the current patch has a blocking upgrade compatibility bug for existing custom `models.json` entries that use bare environment-variable names as `apiKey` values.

## What This Changes

This PR strips plaintext provider `apiKey` values from generated and legacy agent `models.json`, migrates eligible legacy keys into `auth-profiles.json`, preserves non-secret markers, and adds model-config regression coverage.

## Best Possible Solution

Strip prompt-facing plaintext `models.json` secrets while preserving supported env-var markers as env refs or non-secret markers, then land with maintainer-approved credential-precedence wording.

## Reproduction Assessment

Yes, source-level: the PR classifies bare env-var `models.json` API-key values as plaintext, then writes that value into `auth-profiles.json` as a literal key. The upstream `ModelRegistry` contract currently resolves those strings through `process.env` before treating them as literals.

## Solution Assessment

No, not yet: the strip-and-migrate direction is right, but it must preserve supported env-var marker credentials before merge. The safer fix is to migrate env-shaped legacy catalog values as env refs or keep them as non-secret discovery markers while stripping real plaintext.
