## Summary

Keep this PR open, but it is not merge-ready: the security goal is valid, while the current branch still performs legacy credential migration from the runtime models path, has a duplicate metadata-snapshot resolver in the merge result, and only supplies real behavior proof for an older head.

The branch strips plaintext provider `apiKey` values from generated and legacy `models.json`, migrates old catalog keys into `auth-profiles.json`, adjusts metadata snapshot reuse, and adds focused agent regression tests.

## Best Possible Solution
