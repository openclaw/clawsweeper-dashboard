## Summary

Keep this PR open: the security direction is useful, but the latest patch still has a merge-blocking auth-boundary bug where inherited or external profiles can prevent migration of a local legacy catalog key before the catalog is stripped.

## What This Changes

The branch strips plaintext provider `apiKey` values from generated and legacy agent `models.json`, migrates eligible legacy keys to `auth-profiles.json`, preserves non-secret markers, and adds focused regression tests.

## Best Possible Solution

Keep the prompt-facing catalog stripped, but decide migration using only profiles persisted in the target `agentDir` store and add upgrade coverage for inherited/external same-provider profiles before merge.

## Reproduction Assessment

Yes, by source inspection. The PR calls the merged runtime auth store before deciding whether to migrate, and current auth store code merges main-agent and external profiles into non-main agent stores.

## Solution Assessment

No. The stripping and migration direction is the right fix, but the migration guard must use target-local persisted profiles rather than the runtime merged auth view.
