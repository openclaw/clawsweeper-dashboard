## Risks / Open Questions

- Existing scripts or workflows that read plaintext provider credentials directly from `agents/*/agent/models.json` will stop finding them after this PR lands.
- Legacy `models.json` keys are not migrated when an auth profile already exists for that provider, so maintainers should explicitly accept the profile-precedence behavior, especially for stale or unusable profiles that current auth resolution may skip.
- If auth-profile persistence fails during legacy migration, catalog refresh now fails closed and leaves the old file unchanged; that protects credentials but can interrupt startup or model discovery until the auth store is writable.
- The supplied real-behavior proof uses temporary local agent directories and redacted dummy keys, not a paid third-party provider request.
