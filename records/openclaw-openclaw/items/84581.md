## Summary

Keep open: the PR is directionally right and has strong proof, but the latest head still breaks legacy `models.json` credentials that intentionally use arbitrary environment-variable names by migrating them as literal auth-profile keys.

## What This Changes

The PR strips plaintext provider `apiKey` values from generated and legacy agent `models.json`, migrates eligible legacy keys into `auth-profiles.json`, preserves prompt-safe markers, and adds model-config regression coverage.

## Best Possible Solution

Strip prompt-facing plaintext while preserving pi's env-var value-resolution semantics: env-shaped legacy `models.json` values should become env-backed auth profile refs or another prompt-safe runtime path, with regression coverage for arbitrary names.

## Reproduction Assessment

Yes, source-level: pi resolves arbitrary `models.json` `apiKey` strings through `process.env` before literal fallback, while this PR migrates such strings as literal auth-profile keys. I did not run a local repro because this review was read-only.

## Solution Assessment

No: stripping plaintext from the prompt-facing catalog is the right direction, but the current migration is not the safest implementation until arbitrary env-var-backed legacy credentials are preserved.
