## Summary

## What This Changes

The PR strips plaintext provider `apiKey` values from generated and legacy agent `models.json`, migrates eligible legacy keys into `auth-profiles.json`, preserves non-secret markers, and adds focused agent regression tests.

## Best Possible Solution

Land a prompt-safe catalog boundary that never persists plaintext provider keys in `models.json`, with legacy keys moved to auth profiles and maintainer-approved release or docs notes for the intentional compatibility change.

## Solution Assessment
