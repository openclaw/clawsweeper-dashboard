## What This Changes

The branch changes `src/infra/heartbeat-runner.ts` so heartbeat task timestamps and delivered-heartbeat dedupe metadata are written via `updateSessionStore` instead of load/mutate/save snapshots.

## Best Possible Solution

Add a single-line Unreleased/Fixes changelog entry for the heartbeat/session-store race, then land the existing locked-update runtime change after the normal changed-gate validation for the touched surface.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to let the heartbeat load a session-store snapshot, apply a concurrent inbound metadata update through the locked path before the heartbeat save obtains the lock, then observe the heartbeat save overwriting the newer metadata.

## Solution Assessment

Yes. Reusing `updateSessionStore` is the narrowest maintainable fix because it applies the existing lock-and-re-read contract to the two unsafe heartbeat writes without adding a new concurrency mechanism; the missing changelog is a process repair, not a design problem.
