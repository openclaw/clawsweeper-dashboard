## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` so heartbeat task timestamp writes and delivered-heartbeat dedupe writes use `updateSessionStore` instead of separate `loadSessionStore` and `saveSessionStore` snapshots.

## Best Possible Solution

Land the locked re-read heartbeat store writes with the required single-line `CHANGELOG.md` fix entry and normal changed-lane validation; optional focused regression coverage should preserve concurrent inbound metadata while heartbeat updates task state and dedupe fields.

## Solution Assessment

Yes. Reusing `updateSessionStore` is the narrowest maintainable fix because it applies the existing lock and re-read contract to the two unsafe heartbeat writes without adding a new concurrency mechanism; the PR still needs the mechanical changelog entry before merge.
