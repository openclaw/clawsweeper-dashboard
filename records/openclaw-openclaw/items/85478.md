## Summary

Keep this PR open: the classifier and channel rendering direction is sound, but benign no-result command outputs still flow into reply and follow-up failure-suppression predicates as non-zero exits, which can hide later real tool-error warnings after showing `No matches found`.

## What This Changes

The branch adds narrow `rg`/`xargs rg` no-result classification, forwards optional command-output status metadata through reply/channel progress, renders benign progress as `No matches found` across bundled channels, and increases two image-tool test timeouts.

## Best Possible Solution

Land the narrow classifier and cross-channel rendering only after the direct reply and follow-up failure predicates treat `benign_no_result` as non-failure and the additive progress metadata shape is maintainer-accepted.

## Reproduction Assessment

Yes for the review blocker from source: a `command_output` payload can carry `outcomeClassification: "benign_no_result"` plus a non-zero exit code, and both direct reply and follow-up predicates still classify any non-zero exit as failed. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. The classifier and rendering are a maintainable direction, but the failure-suppression predicates need to honor the new benign classification before this is the best fix.
