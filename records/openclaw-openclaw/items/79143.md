## Summary

Keep open: the reported update failure is source-reproducible and the fix direction is plausible, but the latest draft still has a correctness blocker, missing user-facing docs/changelog updates, and no contributor-supplied real update-run proof.

## What This Changes

The PR adds a post-core plugin convergence pass to `openclaw update` that repairs missing configured plugin payloads, statically validates install records, and escalates convergence failures before gateway restart.

## Best Possible Solution

Land a version that scopes convergence validation to the existing active/configured/trusted plugin set, updates docs and changelog, and includes redacted real update-run proof before readiness.

## Reproduction Assessment

Yes from source inspection: current main defers configured plugin repair during `OPENCLAW_UPDATE_IN_PROGRESS=1`, and only a post-core plugin `status: "error"` blocks restart. I did not run a live package-manager update scenario in this read-only review.

## Solution Assessment

No. Mandatory post-core convergence is a reasonable fix direction, but this implementation still smoke-checks every returned tracked install record rather than the active/configured/trusted scope and lacks required public-facing follow-through.
