## Summary

Keep open: the PR targets a valid update failure with supplied terminal proof, but the new smoke check can still block valid plugin packages by statting an explicit `main` entry as a literal path instead of using package resolution semantics.

## What This Changes

Adds a post-core plugin convergence phase to `openclaw update`, static payload validation, invalid-config error gating, docs/changelog updates, and regression tests.

## Best Possible Solution

Keep the post-core convergence gate, but align static entry validation with Node/OpenClaw package entry semantics and cover extensionless `main` before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible for the invalid-config skip and update-time plugin repair deferral paths, and the PR body supplies before/after terminal output for the changed behavior.

## Solution Assessment

No, not as written. The convergence-before-restart boundary is maintainable, but the smoke check needs to resolve explicit `main` entries like real package loading or avoid rejecting OpenClaw-loadable packages.
