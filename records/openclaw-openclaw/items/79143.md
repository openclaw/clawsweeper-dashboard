## What This Changes

The PR adds post-core plugin convergence to `openclaw update`, with update-phase env helpers, missing configured plugin repair, static payload smoke checks, update-command wiring, and unit tests.

## Best Possible Solution

Land a version that preserves the existing active/configured plugin scope for convergence validation, adds the user-facing changelog entry, and includes redacted proof from an actual update run that repairs or blocks the missing-payload scenario before restart.

## Reproduction Assessment

Yes, from source inspection: current main defers configured plugin repair during the package swap and only blocks restart on `postUpdate.plugins.status === "error"`. I did not run a live package-manager update scenario.

## Solution Assessment

No. Mandatory post-core convergence is a reasonable fix boundary, but this implementation still smoke-checks unrelated tracked records and needs a changelog entry plus real update-run proof before merge.
