## Summary

Keep this PR open: it is a plausible fix for a real update/doctor bug, but the latest draft still introduces an over-broad convergence failure path, lacks required user-facing release/docs updates, and has no contributor real behavior proof.

## What This Changes

The PR adds post-core plugin convergence to `openclaw update`, including update-phase helpers, missing configured plugin repair plumbing, static plugin payload validation, update-command gating, and tests.

## Best Possible Solution

Land a version that scopes convergence validation to the same active/configured/trusted plugin set as post-update sync, updates the user-facing docs and changelog, and includes redacted proof from an actual update run before marking the PR ready.

## Reproduction Assessment

Yes. Source inspection shows current main defers configured plugin repair during `OPENCLAW_UPDATE_IN_PROGRESS=1`, while restart blocking only happens when post-update plugins return `status: "error"`; I did not run a live package-manager update scenario.

## Solution Assessment

No. Mandatory post-core convergence is a good fix direction, but the latest implementation still smoke-checks unrelated tracked records instead of preserving the existing active/configured scope.
