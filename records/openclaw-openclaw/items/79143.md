## Summary

Keep open: this is a plausible fix direction for a real update/plugin repair failure, but the current PR is still draft, lacks sufficient real `openclaw update` proof, and introduces a blocking scope bug in the new convergence smoke check.

## What This Changes

The PR adds a post-core `openclaw update` convergence pass that repairs configured plugin installs, statically validates plugin payload records, and escalates invalid-config or convergence failures before gateway restart.

## Best Possible Solution

Land a narrowed version that validates only the active/configured/trusted plugin set, updates docs and changelog, and includes redacted real update-run proof before readiness.

## Reproduction Assessment

Yes, source-reproducible: current main defers configured plugin repair during the update package-swap phase and only a plugin post-update `status: "error"` blocks restart. I did not run a live package-manager update scenario in this read-only review.

## Solution Assessment

No. Mandatory post-core convergence is a reasonable fix direction, but this implementation still smoke-checks the full tracked install-record map and lacks the required public-facing docs, changelog, and real update-run proof.
