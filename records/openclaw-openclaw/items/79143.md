## Summary

Keep open: the convergence fix direction is sound and real behavior proof is supplied, but the new smoke check can reject valid active plugin packages whose `package.json` uses an extensionless `main` entry.

## What This Changes

Adds a post-core `openclaw update` plugin convergence pass, static plugin payload validation, invalid-config error handling, docs/changelog updates, and regression tests.

## Best Possible Solution

Keep the convergence design, but make static entry validation match Node/OpenClaw load semantics and add a regression test for extensionless `main` before maintainer merge.

## Reproduction Assessment

Yes. Current main is source-reproducible for the reported failure boundary, and the PR body provides before/after terminal output for invalid config and corrupt payload shapes.

## Solution Assessment

No, not as written. The convergence and error-gating approach is maintainable, but the smoke check needs to resolve `package.json` `main` the way Node or OpenClaw actually loads plugins.
