## Summary

The PR should stay open: it targets a real update failure, but the current diff can turn unrelated disabled or stale plugin records into hard update failures, and it still needs docs/changelog plus stronger real updater proof.

## What This Changes

The PR adds a post-core `openclaw update` convergence pass that repairs and statically validates plugin install records, hard-fails invalid config or convergence failures before gateway restart, and adds related tests.

## Best Possible Solution

Land a narrowed fix that runs convergence against the active/configured/trusted-official plugin set, updates update docs and the changelog, and includes redacted real `openclaw update` restart-blocking proof.

## Reproduction Assessment

Yes, source-reproducible: current main defers configured plugin repair under `OPENCLAW_UPDATE_IN_PROGRESS`, and only a post-core plugin `status: "error"` blocks restart. I did not run a live package-manager update scenario in this read-only review.

## Solution Assessment

No. Mandatory post-core convergence is a reasonable direction, but this implementation should filter smoke-check inputs to the active/configured/trusted-official set and update docs/changelog before merge.
