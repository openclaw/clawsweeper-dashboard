## Summary

Keep open: this PR is a focused candidate for the still-open mixed hook/tool plugin closure-state issue, but it is not merge-ready because current main has broader live hook registry composition than the PR checks and the external contributor has not supplied real behavior proof.

## What This Changes

This PR skips cached descriptor wrappers for plugins whose active/channel live registry already owns both hooks and tools, and adds a regression test for shared closure state.

## Best Possible Solution

Rebase or replace this with a narrow resolver fix that aligns cached tool resolution with current live registry composition and proves the mixed hook/tool behavior in a real setup; if closure identity is rejected, define an explicit shared-state plugin API instead.
