## Summary

Keep open: current main still resolves cached descriptor tools before the live registry path, and this focused patch looks source-plausible, but the external PR still needs real behavior proof and a branch refresh before merge.

## What This Changes

The branch changes plugin tool resolution to skip cached descriptor wrappers when an active registry already has both hooks and tools for the same plugin, with a regression test for shared closure state.

## Best Possible Solution

Land or supersede a narrow resolver fix that preserves one live register() closure for hook-and-tool plugins while keeping descriptor caching for ordinary tool-only plugins.

## Reproduction Assessment

Yes at source level: current main can return cached descriptor-backed tools before the live registry path, and the PR models a warmed descriptor cache plus live hook-and-tool registry. I did not establish a live mem0 gateway reproduction in this read-only review.

## Solution Assessment

Likely yes: the patch targets the implicated descriptor-cache bypass point and leaves the ordinary cached descriptor path intact for plugins without live hook state. Disabling descriptor caching globally would be broader and worse for the plugin metadata/laziness boundary.
