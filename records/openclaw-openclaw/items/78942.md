## Summary

Keep open: current main still has a source-level path where a successful block stream removes an unsent text-only final reply before dispatch, matching the Feishu report.

## What This Changes

Deliver text-only Feishu group-chat replies when `blockStreaming` has already streamed earlier block content.

## Best Possible Solution

Preserve unsent text-only final payloads unless `blockReplyPipeline.hasSentPayload` proves the same final text already went out, while keeping media de-duplication intact.

## Reproduction Assessment

Yes, as a source-level reproduction: call `buildReplyPayloads` with block streaming enabled, a pipeline reporting `didStream() === true` and `isAborted() === false`, `hasSentPayload() === false`, and a text-only final payload; current main returns no payloads. I did not run a live Feishu account reproduction.

## Solution Assessment

Yes. The narrowest maintainable fix is in core final-payload suppression, with regression coverage for unsent text-only finals after unrelated block-stream output; Feishu defaults and config shape should stay unchanged.
