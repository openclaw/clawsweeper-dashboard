## Summary

This should stay open: current main still has a source-level path where successful block streaming removes an unsent text-only final reply before dispatch, and the related open repair PR still needs changes before it can safely land.

## What This Changes

Fix Feishu/block-streaming reply dispatch so text-only final replies are not discarded after a successful block stream when that final text was never sent.

## Best Possible Solution

Repair or replace https://github.com/openclaw/openclaw/pull/62157 with a content-coverage-aware core fix that delivers genuinely unsent text-only finals while preserving coalesced-stream suppression, media de-dupe, error fallback, and Feishu defaults.

## Reproduction Assessment

Yes, as a source-level reproduction: call `buildReplyPayloads` with `blockStreamingEnabled: true`, a pipeline reporting `didStream() === true`, `isAborted() === false`, and `hasSentPayload() === false`, plus a text-only final payload; current main returns no payloads. I did not run a live Feishu account reproduction.

## Solution Assessment

Yes on direction, but not via the existing open PR as submitted. The narrow maintainable solution is in core final-payload suppression: keep already-streamed/coalesced content suppressed while allowing text-only final payloads that were never sent.
