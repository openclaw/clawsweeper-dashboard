## Summary

Keep open: current main and the latest release still render Android chat links as styled text without URL metadata or tap handling, and an open draft PR already targets this issue with closing linkage.

## What This Changes

Make bare URLs and markdown links in Android assistant chat messages tappable while keeping broader message copy/text selection separate.

## Best Possible Solution

Review, validate, and land or replace https://github.com/openclaw/openclaw/pull/82392 with the URL tap fix in ChatMarkdown, while leaving copy/select behavior to https://github.com/openclaw/openclaw/issues/57754.

## Reproduction Assessment

Yes from source, though not from a live Android device in this review. Current main parses bare URLs as CommonMark links but renders them through plain Text with style-only link spans and no URL tap handler.

## Solution Assessment

Yes: the narrow maintainable fix is to preserve Compose URL annotations/tap behavior in the Android ChatMarkdown inline renderer. It should not be expanded into the separate copy/text-selection feature request.
