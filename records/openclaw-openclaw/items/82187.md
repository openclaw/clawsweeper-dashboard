## Summary

Keep open: current main and the latest shipped release still parse Android chat URLs as markdown links but render them without any URL annotation, click handler, or URI opener. The linked copy/text-selection issue is related fallback work, not a duplicate of this narrower URL tap bug.

## What This Changes

Make bare URLs in Android assistant chat messages tappable, while leaving broader message copy/text selection to the related Android issue.

## Best Possible Solution

Add URL annotations and tap handling in Android ChatMarkdown for bare autolinks and explicit markdown links, with focused coverage and no message-copy feature creep.

## Reproduction Assessment

Yes from source inspection, though not from a live Android device in this review. Current main parses and styles link nodes, then renders them through plain Text without any URL tap handler.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Android markdown inline renderer; the broader copy/text-selection fallback should remain with https://github.com/openclaw/openclaw/issues/57754.
