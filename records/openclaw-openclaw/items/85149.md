## Label Justifications

- `P1`: The original bug broke documented Teams managed-identity deployments by letting inbound work complete while outbound replies failed.
- `impact:message-loss`: The reported failure dropped Teams replies after the agent completed work.

## Summary

Current main and the latest release now route Teams federated managed-identity auth through the Teams SDK FIC path, so the raw managed-identity Bot Framework token failure described here is no longer present.

## What This Changes

The issue asks OpenClaw to stop sending raw managed-identity app tokens for Teams federated managed-identity replies and to expose better delivery-failure diagnostics.

## Best Possible Solution

Keep the shipped Teams SDK-managed FIC path in v2026.6.6; track any remaining desire for status details on the WARN log line as a narrower logging issue.

## Reproduction Assessment

No for current main: the old source path is gone, and current main delegates managed-identity bot tokens to the Teams SDK FIC branch. The historical failure was source-reproducible from the old direct ManagedIdentityCredential path and the reporter's Azure/Teams logs.

## Solution Assessment

Yes. The merged SDK migration is the best current fix because it uses the Teams SDK token manager's FIC exchange instead of keeping a parallel OpenClaw-managed Azure Identity token provider.
