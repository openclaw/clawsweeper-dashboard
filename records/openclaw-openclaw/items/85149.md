## Label Justifications

- `P1`: The report describes a deterministic production Teams workflow break where inbound messages run the agent but outbound replies fail for managed-identity deployments.
- `impact:message-loss`: The affected path drops Teams replies after agent work completes, leaving the Teams user with no delivered response.
- `impact:auth-provider`: The root cause is token acquisition and bot identity selection for Teams federated managed-identity authentication.

## Summary

Keep open: current main and v2026.5.20 still route Teams federated managed-identity auth through raw `ManagedIdentityCredential`, while the documented setup depends on a bot-app federated credential exchange. The exact narrow fix PR was closed unmerged, and the broader Teams SDK migration at https://github.com/openclaw/openclaw/pull/76262 is still open, so there is no shipped implementation to close against.

## What This Changes

The issue reports that Microsoft Teams federated managed-identity auth requests Bot Framework tokens directly from `ManagedIdentityCredential` instead of performing the bot-app federated credential exchange, causing outbound replies to fail.

## Best Possible Solution

Keep this issue open until either the linked Teams SDK migration lands with equivalent managed-identity behavior or a narrow `ClientAssertionCredential` exchange fix lands with focused tests and, when feasible, live Azure/Teams proof.

## Reproduction Assessment

Yes for source-level reproduction: current main constructs `ManagedIdentityCredential` and calls it with the Teams SDK scope instead of exchanging an MI assertion for the configured bot app. A live repro requires an Azure VM/FIC Teams setup, which I did not run here.

## Solution Assessment

Yes, the narrow maintainable fix is to keep the existing `authType: "federated"` plus `useManagedIdentity` contract and perform the bot-app `ClientAssertionCredential` exchange. The only open sequencing question is whether maintainers prefer that narrow repair or the broader Teams SDK migration already in review.
