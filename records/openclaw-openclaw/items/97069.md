## Summary

Keep open: current main and v2026.6.10 still show policy-only Allow Always unavailable copy when the filtered decision list can also come from one-shot/non-persistable command persistence, and multiple open candidate PRs now target the fix path.

## What This Changes

The issue asks OpenClaw to explain non-persistable or one-shot redirected exec approvals instead of saying the effective policy requires approval every time.

## Best Possible Solution

Keep the one-shot safety behavior and land one canonical fix that carries an explicit unavailable reason, or equivalent ask/persistence fact, through agent, gateway, UI, reaction, and channel approval renderers.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main filters Allow Always for both ask-always policy and one-shot persistence, while affected renderers still print the policy-only explanation when Allow Always is absent. I did not run a fresh live gateway prompt in this read-only pass.

## Solution Assessment

Yes: a narrow reason-plumbing and renderer-copy fix is the best direction because it preserves approval policy and persistence safety. Changing the approval policy or making redirected commands persistable is not needed to fix the misleading prompt.
