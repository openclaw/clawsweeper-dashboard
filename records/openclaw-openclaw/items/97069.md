## Summary

## What This Changes

The issue asks OpenClaw to explain that Allow Always is unavailable because a redirected/non-persistable command cannot be safely saved, rather than blaming the effective approval policy.

## Best Possible Solution

Keep this issue open as the canonical tracker, then land one corrected PR that carries an explicit Allow Always unavailable reason through gateway protocol, approval state, Control UI, chat/Telegram rendering, and resolve-error paths.
