## What This Changes

The issue requests accurate approval prompt copy when Allow Always is unavailable because a redirected/non-persistable exec command is one-shot rather than because the effective policy is ask=always.

## Best Possible Solution

Land one canonical fix that preserves one-shot safety and carries an explicit unavailable reason, or an equivalent policy-safe fact, through gateway protocol, tool-result, Control UI, and channel approval renderers.
