## Summary

Keep open: current `main` and v2026.6.9 still lack the requested opt-in `agent` RPC image-offload behavior, while source and tests confirm `agent` remains strict and `chat.send` is the entry point that stages text-only images for media understanding. This is a valid, well-scoped feature request, but it adds public gateway protocol surface and needs maintainer API/product approval before implementation.

## What This Changes

The issue requests an opt-in `agent` RPC parameter that lets text-only primary models offload image attachments to the configured image model while preserving strict default behavior.

## Best Possible Solution

Add a maintainer-approved additive `agent` RPC option that preserves the strict default, passes the existing parser offload signal only when opted in, and leaves image description in the shared media-understanding path with focused schema, handler, docs, and regression coverage.

## Reproduction Assessment

Yes from source: current `agent` does not pass the parser's offload trigger, and the adjacent test expects text-only image input to be rejected. I did not run a live gateway because this review is read-only and the source/test path is direct.

## Solution Assessment

Unclear until maintainers approve the public API contract. The proposed opt-in flag is a plausible narrow design, but the exact protocol shape and ownership boundary need product/API acceptance.
