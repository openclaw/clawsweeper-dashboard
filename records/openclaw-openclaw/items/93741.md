## Summary

Keep open: current main and v2026.6.9 still lack the requested opt-in `agent` RPC image-offload behavior. Source shows `agent` still rejects image attachments for text-only models while `chat.send` routes the same case through media offload, and the remaining decision is public gateway protocol/API direction.

## What This Changes

Request adds an opt-in `agent` RPC parameter to route image attachments through `agents.defaults.imageModel` when the resolved primary model is text-only, while preserving strict default behavior.

## Best Possible Solution

Add a maintainer-approved additive `agent` RPC option that keeps strict behavior as the default and proves text-only image offloads reach the shared media-understanding/imageModel path with protocol, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: `agent` passes `supportsInlineImages` without the `supportsImages: false` offload trigger, and the parser throws for image input when the resolved model is text-only.

## Solution Assessment

Unclear until maintainer API direction. The opt-in flag is a plausible narrow shape, but the implementation must prove the `agent` RPC path actually invokes media-understanding/imageModel rather than only changing attachment parsing.
