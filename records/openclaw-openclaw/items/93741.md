## Summary

Keep open: current `main` and the latest release still reject text-only `agent` RPC image attachments instead of offering the requested opt-in offload path, and no open canonical item owns this exact `agent` RPC behavior. The request is plausible but adds public gateway protocol surface, so it needs maintainer product/API approval before implementation.

## What This Changes

The issue requests an opt-in `agent` RPC parameter that lets text-only primary models offload image attachments through the configured image model while preserving strict default behavior.

## Best Possible Solution

Add a maintainer-approved, additive gateway protocol option only if the product/API direction is accepted, keep the strict `agent` default unchanged, and avoid reintroducing gateway-parse-time paid image-description calls.
