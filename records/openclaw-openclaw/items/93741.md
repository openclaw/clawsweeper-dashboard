## Summary

Keep open: current main and the latest release still lack the requested default-off `agent` RPC image-offload option, and the current source still rejects text-only image attachments on the strict `agent` path. The request is well-supported by source and user impact, but it adds public gateway protocol behavior and needs maintainer API/product approval before implementation.

## What This Changes

Request to add a default-off `agent` RPC parameter so image attachments sent to text-only primary models can be offloaded through the configured `agents.defaults.imageModel` while preserving `extraSystemPrompt`, `provider`, and `model` overrides.

## Best Possible Solution

Make a maintainer-approved additive `agent` RPC contract that keeps strict behavior by default and, when enabled, stages text-only image offloads into the existing shared media-understanding path with focused schema, handler, docs, and regression coverage.

## Reproduction Assessment

Yes, source-reproducible: the current `agent` handler omits the parser's `supportsImages: false` offload trigger, and the adjacent agent test expects text-only image input to be rejected. I did not run a live gateway turn because this review is read-only and the source/test path is direct.

## Solution Assessment

Unclear until maintainer direction: the default-off flag is a narrow plausible shape, but it adds public `agent` RPC surface and must decide how the direct agent path stages `MediaPaths` for shared media understanding.
