## What This Changes

The issue requests an opt-in `agent` RPC parameter that lets text-only primary models offload image attachments through the configured `imageModel` while preserving strict default behavior.

## Best Possible Solution

Keep this issue open until maintainers decide the public RPC shape, then implement the smallest opt-in path that reuses existing attachment parsing and media-understanding staging without changing strict default behavior.

## Reproduction Assessment

Yes, by source inspection: current main has no agent RPC opt-in field, and the agent path passes only `supportsInlineImages` before `parseMessageWithAttachments` throws for non-ACP text-only image inputs. I did not run a live gateway repro because this review is read-only.

## Solution Assessment

Unclear. The proposed opt-in flag is a plausible narrow solution, but public RPC naming, authorization, and paid offload semantics need maintainer product/API confirmation before implementation.
