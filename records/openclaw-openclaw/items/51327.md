## Summary

Keep open: current main still scopes `sessions.usage` discovery to configured gateway agents, while `agents.delete` can remove config/store visibility and preserve transcript files when `deleteFiles=false`. The PR targets the right bug, but it needs after-fix real behavior proof and a maintainer/contributor update against the current usage-cache and session-family code before merge.

## What This Changes

The PR adds usage-only disk-backed session discovery for preserved agent session directories, lets transcript discovery accept an explicit `sessionsDir`, and expands `sessions.usage` regression coverage.

## Best Possible Solution

Land this PR or a maintainer-owned successor after rebasing it onto current main, preserving the durable usage cache and family grouping behavior while keeping deleted-agent transcript discovery scoped to usage APIs only.

## Reproduction Assessment

Yes. The source path is high-confidence: create usage for an agent, call `agents.delete` with `deleteFiles=false`, then `sessions.usage` on current main still enumerates only configured agents while the preserved transcript directory is no longer represented in config/store visibility.

## Solution Assessment

Yes, directionally. Usage-only disk-backed discovery with the real `sessionsDir` is the narrow maintainable fix, but it should be rebased into the current cache/family implementation before merge.
