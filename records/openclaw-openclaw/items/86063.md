## Summary

Keep open: current main includes two important partial repairs, but active inbound context is still prepended for the current prompt while persisted/replayed user turns remain bare and historical inbound metadata is stripped, so the remaining cache-prefix stability problem is not fully resolved.

## What This Changes

The issue asks OpenClaw to keep Anthropic/full-resend provider prompt-cache prefixes byte-stable across turns, including cache-control marker placement and user-turn metadata serialization.

## Best Possible Solution

Choose one owner-approved model-facing representation that keeps ephemeral inbound metadata out of stale context without mutating bytes inside the reusable cache prefix, then add active-to-historical byte-stability tests and live Anthropic cache-read proof.

## Reproduction Assessment

Yes at source level: current main separately prepends active inbound context, persists bare user turns, and strips historical metadata, though a fresh live current-main Anthropic trace is still missing.

## Solution Assessment

Unclear: the partial fixes are good for their slices, but the remaining inbound-metadata representation requires a maintainer decision because the possible fixes change historical context semantics.
