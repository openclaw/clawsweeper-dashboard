## Summary

Keep open: current main has landed the tool-loop cache-marker and timestamp/content-form fixes, but the remaining current-turn inbound-context versus bare persisted-history split still needs a maintainer-owned context policy decision.

## What This Changes

The issue requests stable Anthropic prompt-cache conversation-prefix serialization across turns, including cache-control marker placement and inbound metadata handling.

## Best Possible Solution

Choose a stable, maintainer-approved inbound-metadata representation that keeps useful current-turn context out of the reusable cache prefix or makes it deterministic across replay, then prove it with cache-stability tests and a current Anthropic cache-read/write run.

Yes for source-level reproduction: current main still builds the active model prompt with currentInboundContext while persisting the user turn as bare text. ## Solution Assessment

Unclear. The merged repairs are good partial fixes, but the remaining inbound-metadata byte-stability fix needs a maintainer choice among deterministic replay, sidecar/runtime context, stable placeholders, or dropping metadata from the prompt.
