## Summary

Keep open: current main has merged partial repairs for tool-loop marker movement and timestamp/content-form stability, but the remaining active-turn inbound-context versus bare-history prompt shape still needs a maintainer-owned cache/context policy decision before this can close.

## What This Changes

The issue reports Anthropic prompt-cache misses caused by conversation-history bytes changing as user/tool-result turns move from the active prompt boundary into history.

## Best Possible Solution

Choose a stable inbound-context representation or cache-anchor policy that keeps useful untrusted current-turn context without placing volatile metadata inside the reusable prompt-cache prefix, then prove it with focused cache-stability tests and a current Anthropic read/write run.
