## Summary

Keep open: current main includes the merged tool-loop marker and timestamp/content-form repairs, but source still keeps current-turn inbound context model-facing while persisting the user turn as bare history. That leaves the remaining prompt-stability behavior as a maintainer product/context-policy decision rather than a conservative cleanup close.

## What This Changes

The issue reports Anthropic prompt-cache prefix instability across turns from cache marker movement and inbound/current-turn metadata serialization asymmetry.

## Best Possible Solution

Pick an owner-approved representation that keeps useful current-turn inbound metadata while preventing volatile metadata bytes from living inside the reusable prompt-cache prefix, then prove it with cache-stability tests and a current Anthropic cache-read/write run.

## Reproduction Assessment

Yes. Source inspection shows current inbound context is model-facing on the active turn while the persisted user turn stays bare, and issue comments add live Slack, Telegram, and Discord Anthropic observations. I did not run a fresh paid current-main Anthropic session in this read-only review.

## Solution Assessment

Unclear. The merged PRs are good partial repairs, but the remaining inbound-metadata byte-stability fix requires maintainer choice among deterministic replay, metadata sidecar/runtime context, stable placeholder, or dropping metadata from prompt history.
