## What This Changes

The issue reports that Anthropic conversation-history prompt caching misses every turn because historical inbound metadata is stripped and the moving latest-block cache_control marker changes previously cached message bytes.

## Best Possible Solution

Maintainers should choose one stable Anthropic conversation-cache design, then add regression coverage and live cache proof showing historical prefix bytes remain stable while inbound metadata remains current-turn-only.

## Reproduction Assessment

No high-confidence executable current-main reproduction was established in this read-only pass. Source inspection confirms both reported mechanisms, and the reporter supplied detailed Anthropic console and payload-hash evidence, but live cache proof is still needed.

## Solution Assessment

Unclear: the report correctly avoids prescribing a one-line fix because skipping metadata stripping or removing the moving breakpoint only solves one mechanism. The maintainable solution needs a maintainer choice between a stable anchor, metadata sidecar/persistence redesign, or an explicit reduced-cache mode.
