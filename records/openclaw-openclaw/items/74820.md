## Summary

Keep open. Current main still lacks WSL2 auto-disable for the Bonjour advertiser, and this PR is a narrow, technically valid change with matching tests and docs. The remaining action is maintainer review and targeted validation, not closing or replacing the PR.

## What This Changes

The PR adds WSL2 detection to the Bonjour advertiser disable path, mocks that runtime helper in advertiser tests, and updates gateway Bonjour/discovery docs for WSL2 auto-disable and explicit opt-in.

## Best Possible Solution

Merge the WSL2 auto-disable behavior after maintainer review and targeted validation, while leaving #74778 to handle probing-loop recovery and #74209 to track broader bundled-plugin startup/default policy.

## Reproduction Assessment

Yes. On current main, a WSL2 environment without OPENCLAW_DISABLE_BONJOUR still falls through isDisabledByEnv() and proceeds toward ciao service creation; the PR’s mocked WSL2 unit test is a focused reproduction for the intended fix.

## Solution Assessment

Yes. The PR changes the existing Bonjour disable boundary, uses an existing public runtime-env helper, preserves OPENCLAW_DISABLE_BONJOUR=0 as an explicit opt-in, and keeps the policy inside the Bonjour plugin rather than core.
