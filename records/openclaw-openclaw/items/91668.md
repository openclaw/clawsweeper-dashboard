## Summary

## What This Changes

The PR imports the shared subagent run-liveness helper into `src/agents/subagent-orphan-recovery.ts` and finalizes stale aborted subagent runs instead of resuming them during orphan recovery.

## Best Possible Solution

Land one canonical fix that uses the shared liveness policy, has a single finalization owner, preserves fresh and long-timeout recovery, updates adjacent docs/tests, and includes redacted restart/orphan-recovery proof.

## Reproduction Assessment

Yes, by source inspection rather than an executed repro: current main excludes `abortedLastRun` rows from stale restored-run pruning and has an adjacent fixture that recovers a 3-hour-old restart-aborted run. The PR proof does not yet exercise the actual restart/orphan-recovery path.

## Solution Assessment

No as written. The right shape is to apply the shared liveness policy in recovery while keeping one terminal finalization owner and adding recovery-path tests/proof for stale, fresh, and timeout-extended runs.
