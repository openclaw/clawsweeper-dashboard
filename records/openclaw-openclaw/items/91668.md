## Summary

Keep open: the PR targets a real subagent session-state bug, but the patch is not correct as submitted because it adds a separate session-timestamp cutoff that conflicts with the existing run-liveness policy and still misses restored stale aborted runs with recent session metadata.

## What This Changes

The PR adds a 60-minute age gate to subagent orphan recovery and passes the session entry timestamp into the recovery gate before automatic resume.

## Best Possible Solution

Use the existing run-record stale-liveness policy for both restored registry reconciliation and direct orphan recovery, update the subagent recovery docs/tests, and require real restart-path proof that stale runs are not resumed while fresh aborted runs still recover.

## Solution Assessment

No. The submitted fix is a plausible guard, but the best fix should reuse the existing run-liveness policy and cover both restored registry reconciliation and direct orphan recovery instead of adding a separate session timestamp threshold.
