The branch replaces selected successful provider `response.json()` reads in Vydra, xAI, Runway, PixVerse, DeepInfra, Together, ElevenLabs, Microsoft Foundry, and Microsoft speech paths with `readResponseWithLimit` plus local JSON parsing, and adds two test mocks.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main still has direct successful `response.json()` reads in several targeted external-provider paths. A streamed oversized `Response` harness would reproduce the unbounded-buffering class, but this read-only review did not execute it.

## Solution Assessment

No. Bounded reads are the right fix class, but this PR should use the shared provider JSON reader, preserve current-main overlap, choose payload-appropriate caps, and add real behavior proof before it is the best solution.
