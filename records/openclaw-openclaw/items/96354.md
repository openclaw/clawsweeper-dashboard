## Summary

Keep this PR open for maintainer review, but it is not merge-ready: the hardening goal is valid, while the branch is conflicting, overlaps merged and narrower active fixes, and uses repeated local 1 MiB JSON readers instead of the shared provider JSON contract.

## What This Changes

The branch replaces selected provider success-path `response.json()` reads across Vydra, xAI, Runway, PixVerse, DeepInfra, Together, ElevenLabs, Microsoft Foundry, and Microsoft speech paths with local 1 MiB `readResponseWithLimit` parsing and adds generic HTTP proof/tests.

## Best Possible Solution

Split or rebase into narrow provider fixes that reuse `readProviderJsonResponse` or payload-specific shared caps, preserve merged/open overlaps, and include provider-path streamed-response proof.

## Reproduction Assessment

Yes, at source level: current main still has direct successful `response.json()` reads in several targeted external-provider paths. I did not execute a failing current-main OOM repro in this read-only review.

## Solution Assessment

No. Bounded reads are the right fix class, but this PR should reuse the shared provider JSON reader, choose payload-appropriate caps, preserve current-main overlap, and provide representative provider-path proof.
