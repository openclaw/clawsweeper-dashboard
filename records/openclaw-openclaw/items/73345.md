## Summary

Keep open: current main still lacks `/hooks/agent` thread delivery support and this PR contains useful targeted work, but it is not merge-ready because numeric thread values are dropped and real threaded-delivery proof is missing.

## What This Changes

The PR adds optional `/hooks/agent` `thread` payload handling, forwards it into cron delivery thread routing, scopes hook replay by thread, and updates docs, changelog, and gateway tests.

## Best Possible Solution

Land a rebased hook-threading change that feeds the existing delivery.threadId contract, accepts finite numeric and string thread IDs, scopes replay by thread, documents the chosen hook field or alias, and includes redacted real topic/thread delivery proof.

## Reproduction Assessment

Yes, at source level for the PR defect: PR head normalizes payload.thread with normalizeOptionalString while delivery.threadId accepts strings and finite numbers. I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is narrow, but the implementation should preserve the existing string-or-number thread contract, settle the public hook field name, and provide real threaded-delivery proof before merge.
