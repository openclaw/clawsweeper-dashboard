Keep open: current main still does not force ordinary MiniMax Anthropic simple completions through OpenClaw's Anthropic transport, and the linked fixing PR was closed unmerged, so the reported CLI bug remains actionable.

## What This Changes

The issue reports that non-streaming `openclaw infer model run` fails for MiniMax Anthropic-compatible responses whose `content` array starts with `thinking` and later contains `text`.

## Best Possible Solution

Add a shared-transport simple-completion path for MiniMax Anthropic models that preserves OAuth authHeader, usage/cost accounting, and thinking-first text extraction with focused regression tests.

## Reproduction Assessment

Yes at source level: the issue gives a concrete failing CLI command and direct MiniMax response shape, and current main still routes normal MiniMax portal infer runs away from the Anthropic transport that already preserves text after thinking. I did not run a live MiniMax OAuth repro.

## Solution Assessment

Yes: routing MiniMax Anthropic simple completions through the existing OpenClaw Anthropic transport is the narrowest maintainable fix. A CLI-only parser tweak would miss the shared auth, transport policy, usage, and parser boundary.
