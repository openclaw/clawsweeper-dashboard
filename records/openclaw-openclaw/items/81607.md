## Summary

Keep open: current main still leaves ordinary MiniMax Anthropic simple completions on the `@earendil-works/pi-ai` simple path instead of the OpenClaw Anthropic transport path that preserves text seeded on `content_block_start`, and the linked fixing PR is closed unmerged.

## What This Changes

The issue requests that `openclaw infer model run` return later MiniMax Anthropic-compatible `text` blocks when a `thinking` block appears first.

## Best Possible Solution

Route MiniMax Anthropic simple completions through the shared OpenClaw Anthropic transport, or an equivalent shared parser path, while preserving OAuth authHeader handling, usage/cost accounting, and focused regression coverage.
