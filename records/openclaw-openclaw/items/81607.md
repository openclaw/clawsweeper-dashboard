## Summary

Keep open: current main still routes local `openclaw infer model run` through simple completion without a confirmed MiniMax Anthropic non-streaming parser fix, and the open linked PR uses closing syntax for this issue but has not merged.

## What This Changes

The issue reports that `minimax-portal` non-streaming `openclaw infer model run` returns `No text output returned` when a valid Anthropic-compatible response contains a `thinking` block before a later `text` block.

## Best Possible Solution

Keep this issue as the canonical bug until the linked PR, or a replacement, fixes thinking-first MiniMax Anthropic responses through the existing simple-completion transport/provider boundary without bypassing auth, proxy/TLS guardrails, or usage accounting.

## Reproduction Assessment

Yes at source level: a fake Anthropic-compatible MiniMax response with `thinking` before `text` can exercise the implicated local simple-completion path, and the reporter supplied live MiniMax evidence. I did not run a live OAuth call in this read-only review.

## Solution Assessment

Unclear until the linked PR is repaired: concatenating later text blocks is the right observable behavior, but the maintainable fix should live in the shared simple-completion transport/provider path rather than a CLI-only parser or direct provider fetch.
