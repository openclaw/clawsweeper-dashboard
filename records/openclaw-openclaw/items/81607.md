## Summary

Keep open: the report is fresh, specific, and current main still leaves the local `openclaw infer model run` path exposed to the MiniMax Anthropic-compatible text extraction gap even though the OpenClaw-owned Anthropic transport already handles seeded text after thinking blocks.

## What This Changes

The issue requests that `minimax-portal` non-streaming model runs return visible text when MiniMax sends a `thinking` content block before a later `text` block.

## Best Possible Solution

Add a narrow regression for MiniMax Anthropic-compatible responses that carry thinking before seeded visible text, then make the local model-run and embedded/gateway paths use the same robust text extraction without adding a config knob.

## Reproduction Assessment

No, not as a live current-main repro in this review because MiniMax OAuth credentials were unavailable. Source and dependency inspection make the failing path clear enough to reproduce with a fake Anthropic-compatible response fixture carrying thinking before seeded text.

## Solution Assessment

No, the suggested CLI-side iteration is not sufficient because current CLI code already joins all text blocks it receives. The better fix is in the transport/simple-completion boundary so the visible text block is populated before the CLI checks for output.
