## What This Changes

The issue reports that `openclaw infer model run` for `minimax-portal` returns `No text output returned` when the Anthropic-compatible response contains a leading `thinking` block followed by a valid `text` block.

## Best Possible Solution

Route MiniMax and MiniMax Portal Anthropic-compatible local simple completions through the shared OpenClaw Anthropic transport/parser path, preserving OAuth `authHeader`, configured base URL handling, usage/cost accounting, and adding regression coverage for thinking-before-text responses.

## Reproduction Assessment

Yes at source level: the issue provides a concrete failing CLI command and direct MiniMax `content=[thinking,text]` response, and current main still leaves ordinary MiniMax local simple completion outside the shared Anthropic transport path. I did not run a live MiniMax OAuth repro, so this is not marked as reproduced.

## Solution Assessment

Yes: using the existing shared Anthropic transport/parser path is the narrowest maintainable fix because it keeps auth handling, payload shaping, transport policy, usage, and text extraction in one boundary. A CLI-only text collector change would miss the layer that currently fails to return the trailing text block.
