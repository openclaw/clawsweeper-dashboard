Keep open: current main still sends the local infer model-run path through simple completion without a MiniMax Anthropic non-streaming parser, and the linked fixing PR is open with review blockers rather than merged.

## What This Changes

The issue asks OpenClaw to return MiniMax text blocks that follow a thinking block in non-streaming `openclaw infer model run` instead of reporting empty output.

## Best Possible Solution

Land a corrected version of the linked PR, or a replacement, that handles thinking-first MiniMax Anthropic responses through the shared simple-completion transport/provider boundary while preserving auth, network policy, and usage accounting.

## Reproduction Assessment

Yes, at source level: the issue provides the failing CLI command and response shape, and current main routes that CLI path through simple completion before throwing when no text block reaches the CLI. I did not run a live MiniMax OAuth repro in this read-only review.

## Solution Assessment

No, not as a CLI-only parser tweak. The expected behavior is right, but the maintainable fix should live in the simple-completion transport/provider path so parsing, auth, transport policy, and usage accounting remain shared.
