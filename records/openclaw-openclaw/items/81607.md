## Summary

Keep this open: current main has MiniMax M3 embedded/gateway thinking-policy fixes, but the original local `openclaw infer model run` simple-completion path still has no MiniMax simple-completion wrapper tying the provider-owned thinking/text policy into direct model runs.

## What This Changes

The issue requests that MiniMax Anthropic-compatible responses with a thinking block before a text block still return the visible text from `openclaw infer model run`.

## Best Possible Solution

Wire the existing MiniMax thinking/text preservation policy into the provider simple-completion hook and add focused regression coverage for M2.x and M3 direct `infer model run` response shapes.

## Reproduction Assessment

No live current-main MiniMax repro was established. Source inspection still shows a high-signal path gap: the local infer simple-completion route has a provider wrapper seam, but MiniMax does not register a simple-completion hook for the thinking/text policy.

## Solution Assessment

No, closing is not the best solution yet. The narrow maintainable fix is to reuse the existing MiniMax provider wrapper policy through `wrapSimpleCompletionStreamFn`, rather than adding a new config option or separate CLI-only parser.
