## Summary

Keep open: current main and latest v2026.6.6 still lack the MiniMax simple-completion hook needed for this direct `infer model run` path, and the linked closing PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to return visible MiniMax text when an Anthropic-compatible response contains a `thinking` block before a later `text` block.

## Best Possible Solution

Land the linked MiniMax simple-completion hook fix after review and validation, then close this issue from the merged PR; track any remaining M3/gateway behavior separately if it still reproduces.

## Reproduction Assessment

Yes by source inspection, but not by live current-main execution: the direct simple-completion path can only apply provider payload wrappers through `wrapSimpleCompletionStreamFn`, and current MiniMax hooks do not register that hook on main.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing provider-owned MiniMax thinking wrapper through the simple-completion hook, while keeping broader M3/gateway parsing scope out of this issue unless maintainers choose to expand it.
