## Repair Work Prompt

Fix the Codex native `post_tool_use` relay gap from https://github.com/openclaw/openclaw/issues/95597. Current main gates `post_tool_use` only on `hasGlobalHooks("after_tool_call")` and `runNativeHookRelayPostToolUse` only calls `runAgentHarnessAfterToolCallHook`, so Tokenjuice-style plugins that register only `agentToolResultMiddleware` for `runtime: "codex"` are skipped. Add a generic native relay path that treats active/declared Codex tool-result middleware as local post-tool work, runs the native `PostToolUse` payload through `createAgentToolResultMiddlewareRunner({ runtime: "codex", ...context })`, preserves existing `after_tool_call` observers, and does not restore idle relay subprocess fan-out fixed by https://github.com/openclaw/openclaw/pull/78004. Do not add Tokenjuice-specific production branches, config flags, or changelog edits. Preserve Codex result semantics for real `@openai/codex` 0.139.0 `PostToolUse` payloads, including failed/status-less exec cases; stop for maintainer review if that requires a product decision or upstream Codex contract change.
