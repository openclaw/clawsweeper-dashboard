## Summary

Keep open. Current main and the latest release still gate Codex native post_tool_use relay on legacy after_tool_call hooks and dispatch only that legacy hook, while the linked candidate PR is open and unmerged.

## What This Changes

The issue reports that Codex native post_tool_use relay skips the existing Codex agentToolResultMiddleware contract, so Tokenjuice stops receiving native tool results and fresh artifacts are not captured.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/95611 with a narrow native relay fix that activates Codex tool-result middleware, uses the same activation source at install and execution time, and preserves idle relay suppression.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows native post_tool_use only activates for after_tool_call and then runs only the legacy hook; I did not rerun the live Tokenjuice artifact timestamp flow in this read-only review.

## Solution Assessment

Yes, the right solution is in the generic Codex native relay because the documented plugin middleware contract already exists. A Tokenjuice-specific workaround or a new config knob would create a parallel behavior path.
