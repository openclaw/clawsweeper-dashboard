## Summary

Keep open. Current main and v2026.6.9 still skip Codex tool-result middleware in native `post_tool_use`, and the open draft PR that references this issue is only a candidate fix until it lands.

## What This Changes

The issue reports that Codex native post-tool hook relay skips Codex `agentToolResultMiddleware`, preventing Tokenjuice from receiving fresh tool result artifacts.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95611 if its native relay change is correct, or replace it with a narrow generic relay fix that honors Codex tool-result middleware without reintroducing idle hook subprocess churn.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.9` gate native `post_tool_use` on legacy `after_tool_call`, while Tokenjuice declares `agentToolResultMiddleware` for `codex` and the native post handler never invokes that runner. I did not run the live Codex/Tokenjuice artifact command during this read-only review.

## Solution Assessment

Yes, the best fix direction is to make the generic native Codex relay honor the existing tool-result middleware contract while preserving the no-work relay optimization. A Tokenjuice-specific workaround would be the wrong layer because the skipped contract is in core relay dispatch.
