## Summary

Keep open. Current `main` and `v2026.6.9` still gate native Codex `post_tool_use` relay work on legacy `after_tool_call` hooks, while the linked open draft PR is the active candidate fix and should be reviewed before this issue closes.

## What This Changes

The issue reports that native Codex `post_tool_use` events do not invoke `agentToolResultMiddleware`, so Tokenjuice stops creating fresh tool-result artifacts.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/95611 so native Codex `PostToolUse` treats Codex tool-result middleware as local work and invokes it through the middleware runner before this issue is closed.

## Reproduction Assessment

Yes by source inspection: current `main` only relays native Codex `post_tool_use` when legacy `after_tool_call` hooks exist, and Tokenjuice registers Codex `agentToolResultMiddleware` instead. I did not run a live Codex plus Tokenjuice turn in this read-only review.

## Solution Assessment

No, current `main` is not the best end state because it keeps middleware-only plugins out of native `PostToolUse`. The best path is the linked PR or an equivalent fix that adds middleware-aware relay gating and native result middleware dispatch with Codex-contract proof.
