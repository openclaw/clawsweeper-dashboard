## Summary

Keep open. Current main still treats `openclaw agent --deliver` as an agent turn and only sends model-produced payloads; #57300 is an open focused PR with closing syntax, so this issue should wait for that PR or a maintainer-approved replacement.

## What This Changes

Request direct channel delivery for `openclaw agent --deliver` and a direct session-injection surface instead of routing notification text through an LLM turn.

## Best Possible Solution

Review and land #57300, or an equivalent fix that directly delivers non-internal-event `--deliver` messages while preserving the internal-events path, then handle session injection as a separate decided CLI/API surface.

## Reproduction Assessment

Yes from source inspection. A current-main `--deliver` request flows through `agentViaGatewayCommand` to `dispatchAgentRunFromGateway`, runs `runAgentAttempt`, and skips outbound delivery when the agent returns no payloads.

## Solution Assessment

Yes for the direct channel-delivery portion: bypassing the LLM turn for ordinary `--deliver` while preserving internal-event synthesis is the narrowest maintainable direction shown by #57300. The session-injection command should be split or explicitly designed because current `chat.inject` is a UI transcript injection, not channel delivery or user-message injection.
