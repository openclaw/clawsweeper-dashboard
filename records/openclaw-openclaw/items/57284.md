## Summary

Keep open. Current main still routes `openclaw agent --deliver` through a normal agent run and only delivers model-produced payloads; open PR #57300 directly tracks the requested fix with closing syntax, so this issue should wait for that PR or a maintainer-approved replacement.

## What This Changes

Request that `openclaw agent --deliver` deliver the provided message directly instead of running an LLM turn, with a follow-up direct session injection surface.

## Best Possible Solution

Review/land #57300 or implement equivalent direct-delivery behavior for non-internal-event `--deliver`, while separately deciding the session-injection CLI/API surface.

## Reproduction Assessment

Yes from source inspection. Current main still sends `--deliver` through `agentCommandFromIngress`/`runAgentAttempt`, and `deliverAgentCommandResult` returns without outbound delivery when the agent produces no payloads.

## Solution Assessment

Yes for the direct channel-delivery portion: bypassing the LLM turn for ordinary `--deliver` while preserving the internal-events path is the narrowest code direction shown by #57300. Unclear for session injection, which should be split or explicitly decided because current `chat.inject` semantics are not the same as the requested CLI command.
