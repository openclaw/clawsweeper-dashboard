## Summary

Keep open: current main and the latest release still return `JSON.stringify(value)` directly in the ACP-core helper, while this PR adds the missing fallback and focused regression coverage with no blocking findings found.

## What This Changes

The PR updates ACP-core non-Error cause stringification to fall back to an object tag when `JSON.stringify` returns `undefined`, and adds a colocated Vitest regression test.

## Best Possible Solution

Land the narrow helper fallback with the colocated regression test after ordinary maintainer and CI gates.

## Reproduction Assessment

Yes for the helper-level bug: current main returns `JSON.stringify(value)` directly, and a Node contract check shows that functions, symbols, and `undefined` serialize to `undefined`. I did not rerun an end-to-end ACP runtime flow in this read-only review.

## Solution Assessment

Yes. The helper owns the `string` return contract, and using the same nullish fallback as the sibling infra helper is narrower than adding defensive handling at each caller.
