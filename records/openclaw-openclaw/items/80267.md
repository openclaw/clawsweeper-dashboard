Keep open: this protected, maintainer-labeled Plugin SDK PR still has a concrete Codex auth-context regression, security-sensitive credential-scope concern, and only test/metadata proof for the latest head.

## What This Changes

This PR consolidates Plugin SDK host workflow seams, Gateway session actions, session attachments, scheduled turns, run-context/event facades, Codex structured extraction, Telegram outbound formatting, docs, tests, and generated Swift protocol models.

## Best Possible Solution

Forward the caller's Codex `agentDir` and config into structured-extraction client startup with focused coverage, add current-head real runtime proof, and then have maintainers decide whether to land this broad SDK consolidation as one PR.

## Reproduction Assessment

Yes for the review finding: source inspection of the latest head shows `extractCodexStructured` drops `req.agentDir` and `req.cfg`, while current-main Codex client startup uses those fields for auth profile context. I did not run the PR branch because this review is read-only.

## Solution Assessment

No. The grouped SDK direction may be reasonable, but this implementation should first preserve the caller's Codex auth context and provide current-head real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Pass the Codex auth context into extraction clients:** `extensions/codex/media-understanding-provider.ts:197-205`
