## Summary

The false-warning path is still present on current main, and the newer MCP comment is a related instance of the same catalog-timing limitation rather than a reason to close.

## What This Changes

The issue asks OpenClaw to stop warning that enabled optional plugin tools and configured MCP tool patterns in tool allowlists are unknown when they are valid but not materialized yet.

## Best Possible Solution

Keep the issue open and teach the warning analysis to recognize enabled declared plugin tools and configured MCP server prefixes before warning, with regression coverage for `llm-task`, MCP `server__*` patterns, and real unknown entries.

## Reproduction Assessment

Yes, source-reproducible: an allowlist entry such as `llm-task` or `paperless__*` is marked unknown when it is not present in the current `pluginGroups`/core set at analysis time. I did not run a live gateway repro in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable direction is to keep the warning but add knowledge of enabled plugin declarations and configured MCP server prefixes. A wording-only change like the open related PR is not enough for the enabled `llm-task` false positive.
