The issue asks for tool-policy diagnostics to stop warning that enabled plugin tools or configured MCP server tool globs in `tools.alsoAllow`/`tools.allow` are unknown before those tools are materialized.

## Best Possible Solution

Review and land the canonical declared-context fix so policy diagnostics recognize enabled plugin and configured MCP tool declarations before materialization while preserving warnings for real unknown or denied entries.

## Reproduction Assessment

Yes, at source level. Current main classifies allowlist entries only against core tools and currently materialized plugin groups, so a declared MCP glob such as `paperless__*` remains unknown before session MCP materialization.

## Solution Assessment

Yes, the right solution is in the tool-policy declared-context/analyzer layer. It should consume canonical plugin/MCP declaration facts rather than starting transports or suppressing all unknown entries.
