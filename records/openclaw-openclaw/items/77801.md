## Summary

The reported warning path is still present on current main and matches the documented `llm-task` optional-plugin setup, so this should stay open for a narrow fix.

## What This Changes

The issue asks OpenClaw to stop warning that an enabled optional plugin tool in `tools.alsoAllow` is unknown before plugin tools are registered.

## Best Possible Solution

Teach the warning analysis to recognize enabled-but-not-yet-materialized plugin tool names from manifest or plugin metadata, then add regression coverage for enabled `llm-task` plus `tools.alsoAllow` without hiding real unknown entries.

## Reproduction Assessment

Yes, source-reproducible: `tools.alsoAllow` becomes an allowlist entry and `analyzeAllowlistByToolType` marks it unknown when plugin groups are not populated. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to keep the existing warning pipeline but let it consult enabled plugin declarations before warning. A blanket suppression of unknown plugin-looking names would be riskier because it would hide real config mistakes.
