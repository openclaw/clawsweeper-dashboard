## Summary

Keep open: current main still has a source-visible false-warning path for declared future tools, especially configured MCP server glob entries, and the linked direct fix PR was closed unmerged. The exact optional plugin path appears partly improved by manifest-based tool loading, but the policy analyzer still only recognizes tools already present in the current tool list.

## What This Changes

The issue asks OpenClaw to stop warning that enabled plugin or configured per-session MCP tool allowlist entries are unknown before those tools are registered.

## Best Possible Solution

Teach the tool policy analyzer about declared-but-not-yet-materialized plugin tools and configured MCP server namespaces, while preserving warnings for true unknown entries.

## Reproduction Assessment

Yes, at source level: `paperless__*` is not a core tool, plugin id, or current plugin tool name, so `analyzeAllowlistByToolType` classifies it as unknown before per-session MCP tools are materialized. I did not run a live gateway repro in this read-only pass.

## Solution Assessment

Unclear for the closed PR shape: adding known plugin tool names is plausible, but the best fix should also cover configured MCP server namespace globs without starting MCP transports or hiding real typos.
