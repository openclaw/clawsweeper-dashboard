## Summary

Keep open: current main still analyzes unknown allowlist entries from the currently registered tool catalog and exact plugin ids/tool names, while the reported optional plugin and configured MCP server cases can be valid before runtime tools are registered. The direct fix PR was closed unmerged, and no canonical replacement appears to own the remaining work.

## What This Changes

Suppress false unknown-allowlist warnings when an allowlist names an enabled optional plugin tool or configured per-session MCP server tool pattern that is not yet registered in the runtime catalog.

## Best Possible Solution

Make warning analysis consult enabled manifest-declared plugin tools and configured/materialized MCP server name patterns using the same glob semantics as enforcement, while preserving typo warnings.

## Reproduction Assessment

Yes, source-reproducible: call the policy pipeline with an allowlist entry such as `llm-task` or `paperless__*` before that tool is present in `pluginGroups.all`, and the analyzer classifies it as unknown. I did not run a live gateway restart or log capture in this read-only review.

## Solution Assessment

Yes: the maintainable fix is narrow warning-analysis alignment, not blanket suppression. The analyzer should receive enabled manifest tool declarations and configured MCP server patterns without changing actual tool filtering or widening plugin loading.
