## Summary

Keep open: current main has improved optional-plugin discovery, but the warning analysis still recognizes only exact plugin ids/tool names and can still warn for valid delayed plugin or MCP allowlist patterns; the direct fix PR was closed unmerged.

## What This Changes

The issue reports misleading unknown-tool warnings for enabled plugin tools and per-session MCP tools listed in `tools.alsoAllow` or `tools.allow`.

## Best Possible Solution

Make warning analysis consult declared enabled plugin tools and configured or materialized MCP server patterns using the same glob semantics as enforcement, while preserving typo warnings.

## Reproduction Assessment

Yes, source-reproducible: valid glob-style plugin/MCP allowlist entries can pass the policy matcher but still be classified unknown by `analyzeAllowlistByToolType` because it only checks exact plugin ids/tool names. I did not run a live gateway restart or log capture.

## Solution Assessment

Yes, the best fix is narrow warning-analysis alignment, not blanket suppression. The analyzer should recognize enabled plugin declarations and configured/materialized MCP server patterns without changing actual tool filtering or widening plugin loading.
