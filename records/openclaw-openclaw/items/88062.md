## Summary

Keep open for maintainer review: the patch is coherent and covered by focused tests and proof, but the remaining question is whether OpenClaw should delay operator-visible `session.stalled` diagnostics for owned silent model calls until the abort threshold.

## What This Changes

The PR reclassifies owned silent `model_call` liveness diagnostics as `session.long_running` before `diagnostics.stuckSessionAbortMs`, preserves stalled handling for abort-threshold and ownerless cases, and updates focused tests and docs.

## Best Possible Solution

Land after maintainer signoff that pre-abort owned silent model calls should be treated as long-running diagnostics and that `session.stalled` should represent the abort-threshold or ownerless-stale cases.

## Reproduction Assessment

Yes. Current main and v2026.5.27 source classify stale active model calls as `session.stalled`, and the PR body supplies a real logged session shape plus patched classifier output; I did not run a live restarted gateway.

## Solution Assessment

Yes, if maintainers accept the diagnostic semantics. The implementation keeps abort-threshold and ownerless-stale behavior intact; if early `session.stalled` is still needed for alerting, a separate signal would be safer than keeping the false stalled classification.
