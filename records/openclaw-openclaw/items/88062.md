## What This Changes

## Best Possible Solution

Restrict the pre-abort long-running classification to owned active embedded model calls, preserve stalled diagnostics for ownerless stale model activity, remove the changelog edit, and add redacted after-fix gateway/log proof before merge.

## Solution Assessment

No, not yet: suppressing early stalled warnings for owned silent model calls is a reasonable fix direction, but the implementation should not reclassify ownerless stale model-call activity as long-running and should drop the release-owned changelog edit.
