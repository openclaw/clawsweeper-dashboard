## Summary

Keep open. Current main still renders collect-mode busy batches with the fixed title and `Queued #N` item labels, and the focused tests still assert that old prompt shape; the requested total-aware markers are not implemented.

## What This Changes

Add total-aware sequence markers to collect-mode queued-message prompts, such as `[Queued 1/3]`, so agents can track multi-message batches.

## Best Possible Solution

Update collect-mode prompt rendering to include total-aware title/item markers while preserving sender suffixes, auth-group splitting, routing metadata, retry behavior, image forwarding, dedupe behavior, and overflow summary placement.
