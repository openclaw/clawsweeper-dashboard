## Summary

Keep open. Current main still renders collect-mode busy-session batches with the fixed title and `Queued #N` item markers, and the focused tests still assert that old shape, so total-aware sequence markers remain unimplemented.

## What This Changes

Add total-aware sequence markers to queued collect-mode user messages, such as `[Queued 1/3]`, so the model can track all messages in a busy-session batch.

## Best Possible Solution

Implement total-aware collect prompt formatting for delivered user-message batches while preserving sender suffixes, auth-group splitting, routing metadata, retry behavior, image forwarding, dedupe behavior, and overflow summary placement.

## Reproduction Assessment

Yes. Static current-main inspection shows same-route collect followups still build prompts containing `Queued #1` and `Queued #2`, and existing tests assert that exact shape; no runtime test was run because this review is read-only.

## Solution Assessment

Yes. The requested counted title and `[Queued i/N]` marker change is the narrow maintainable solution; a broader queue architecture change is not needed for this prompt-format issue.

## Repair Work Prompt

Fix the collect-mode queued message prompt format for https://github.com/openclaw/openclaw/issues/60657. In `src/auto-reply/reply/queue/drain.ts`, change delivered busy-session collect batches from `Queued #N` to total-aware markers like `[Queued i/N]`, and include the delivered batch size in the collect title using the repo-preferred ASCII form, for example `[Queued messages while agent was busy - 3 total]`. Count the messages actually delivered in each auth/routing group, not the original queue before auth splits, cross-channel fallback, drops, or overflow summaries. Preserve sender suffixes such as `(from Guest)`, image forwarding, routing metadata, auth-group splitting, retry semantics, dedupe behavior, and overflow summary placement. Update `src/auto-reply/reply/queue.collect.test.ts` and any adjacent queue tests that assert the old marker shape. If changing `buildCollectPrompt` in `src/utils/queue-helpers.ts` to pass total into the renderer, update `src/agents/subagent-announce-queue.ts` and its tests deliberately, or keep the helper backward compatible and leave announce behavior unchanged. Update `docs/channels/bluebubbles.md` if the exact visible queue title changes there, and add a single-line user-facing `CHANGELOG.md` fix entry with `Thanks @justmaker`. Do not change queue defaults, debounce/cap/drop policy, authorization grouping, routing keys, or Feishu-specific queue behavior.
