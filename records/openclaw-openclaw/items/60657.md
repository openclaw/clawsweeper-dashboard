## Summary

Keep open. Current main still renders collect-mode busy batches with the fixed title and `Queued #N` item labels, and the focused tests still assert that older prompt shape rather than total-aware markers.

## What This Changes

Add total-aware sequence markers to collect-mode queued-message prompts, such as `[Queued 1/3]`, so agents can track multi-message batches.

## Best Possible Solution

Update collect-mode prompt rendering to include total-aware title and item markers while preserving sender suffixes, auth-group splitting, routing metadata, retry behavior, image forwarding, dedupe behavior, and overflow summary placement.

## Reproduction Assessment

Yes. Static inspection of current main shows same-route collect followups still build prompts containing `Queued #1` and `Queued #2`, and existing tests assert that shape.

## Solution Assessment

Yes. The narrowest maintainable fix is a prompt-format change in the collect drain path with focused tests, not a broader queue architecture change.
