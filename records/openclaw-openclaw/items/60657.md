## Summary

Keep open. Current main and the latest shipped release still render collect batches with the fixed `[Queued messages while agent was busy]` title and `Queued #N` item markers, while focused tests continue to assert that older shape. The related open Telegram coalescing PR improves whether messages reach collect mode, but it does not implement total-aware sequence markers for the shared collect prompt.

## What This Changes

Add total-aware sequence markers, such as `[Queued 1/3]`, to collected queued-message prompts so the model can track every queued message in a burst.

## Best Possible Solution

Update the shared collect prompt rendering to include a delivered-count title and total-aware item markers while preserving sender suffixes, routing groups, auth splits, retries, images, and overflow summaries.

## Reproduction Assessment

Yes, at source level. Current main still renders collect prompts with `Queued #N` markers and focused tests assert that shape, so the missing total-aware markers can be verified without a live channel run.

## Solution Assessment

Yes. The narrowest maintainable path is a shared collect prompt-format update with focused regression tests, not a broader queue scheduler or Telegram-specific coalescing change.
