## Summary

Keep open. Current main and the latest shipped release still render collect batches with a fixed title and `Queued #N` item markers, while focused tests assert that older shape, so total-aware sequence markers remain unimplemented.

## What This Changes

Add total-aware sequence markers/counts to collect-mode queued-message prompts so multi-message batches are easier for the LLM to account for.

## Best Possible Solution

Update collect prompt rendering to include a delivered-count title and total-aware item markers in the shared collect path, with regression tests for grouped, retry, and auth-split batches.

## Reproduction Assessment

Yes, at source level. Current main still builds collect prompts with `Queued #1`/`Queued #2` and focused tests assert that shape, so the requested total-aware markers can be verified without a live channel run.

## Solution Assessment

Yes. The narrowest maintainable solution is a prompt-format update in the collect rendering/shared helper path with focused regression coverage, not a broader queue scheduling or Telegram debouncer change.
