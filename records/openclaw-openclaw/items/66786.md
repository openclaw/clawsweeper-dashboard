## Summary

Keep open. Current main still resolves the default agent workspace from `OPENCLAW_HOME`/profile fallback instead of `OPENCLAW_WORKSPACE_DIR`, and the closed unmerged fix PR leaves no implemented or superseding change to close against.

## What This Changes

The issue asks for the system prompt and downstream agent workspace resolution to honor `OPENCLAW_WORKSPACE_DIR` for Docker/headless installs.

## Best Possible Solution

Update the shared default workspace resolver to prefer a non-empty `OPENCLAW_WORKSPACE_DIR` before home/profile fallback while preserving explicit config precedence, then lock the contract with focused resolver and consumer tests.

## Reproduction Assessment

Yes. Set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace` config, then resolve the default agent workspace or a missing run workspace; current source returns the home/profile fallback and the system prompt renders that value.

## Solution Assessment

Yes. Fixing `src/agents/workspace-default.ts` is the narrow maintainable solution because it repairs all downstream consumers; changing prompt rendering alone would hide only one symptom.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/66786. Current main ignores `OPENCLAW_WORKSPACE_DIR` in the default-agent workspace fallback, so Docker/headless installs can prompt agents with the home/profile fallback path instead of the configured mounted workspace. Implement the env-aware default workspace contract in `src/agents/workspace-default.ts`: preserve explicit per-agent `workspace` and `agents.defaults.workspace` precedence in `resolveAgentWorkspaceDir`, but when the shared default resolver is used, prefer a non-empty `OPENCLAW_WORKSPACE_DIR` before `OPENCLAW_PROFILE` / `OPENCLAW_HOME` fallback. Use existing path/user-home helpers rather than ad hoc expansion. Add focused regression coverage for the resolver, default-agent consumer, and missing run-workspace fallback; system-prompt coverage is only needed if prompt plumbing changes. Treat closed PR #66831 as useful source context, not as merged code. Add a user-facing `CHANGELOG.md` fix entry, but do not change Docker docs unless behavior/API wording needs alignment. Do not alter non-default agent fallback semantics or explicit workspace config precedence.
