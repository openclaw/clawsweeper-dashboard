## Summary

Keep open. Current `main` still ignores `OPENCLAW_WORKSPACE_DIR` in the shared default workspace resolver, and that value flows through the default-agent fallback, run-workspace fallback, and system prompt rendering. The related fix PR #66831 proposed the right narrow change but was closed unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue requests that the system-prompt workspace path and related default-agent workspace fallback honor `OPENCLAW_WORKSPACE_DIR` instead of using the home/profile default path.

## Best Possible Solution

Implement the env-aware default workspace contract in `src/agents/workspace-default.ts`, using a non-empty `OPENCLAW_WORKSPACE_DIR` before profile/home fallback while leaving explicit config precedence intact, then add focused regression coverage for the resolver, default-agent consumer, and missing run-workspace fallback paths.

## Reproduction Assessment

Yes. Set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace` config, then resolve the default agent workspace or a missing run workspace; current `main` returns the home/profile fallback, and the system prompt prints that resolved value.

## Solution Assessment

Yes. The narrow maintainable fix is to update the shared default resolver and lock it with targeted agent/workspace tests; changing prompt rendering directly would only mask one downstream symptom.
