## Summary

Keep open. Current main still sends default-agent and run-workspace fallbacks through a resolver that ignores `OPENCLAW_WORKSPACE_DIR`, and an open fix PR already targets this issue with closing linkage.

## What This Changes

Make the system-prompt/default workspace path honor `OPENCLAW_WORKSPACE_DIR` for Docker and headless deployments.

## Best Possible Solution

Review and land, rebase, or replace https://github.com/openclaw/openclaw/pull/81447 so the shared default workspace resolver honors `OPENCLAW_WORKSPACE_DIR` and regression tests cover the fallback path.

## Reproduction Assessment

Yes. By source inspection, set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace`; the default-agent fallback returns the home/profile workspace and the system prompt renders that value.

## Solution Assessment

Yes. Fixing the shared default workspace resolver is the narrow maintainable solution because it repairs prompt rendering and other fallback consumers; changing only system-prompt text would leave the resolver contract broken.
