## Summary

Keep open. Current main still ignores `OPENCLAW_WORKSPACE_DIR` in the shared default workspace resolver, so the default-agent and run-workspace fallbacks can pass the home/profile workspace into the system prompt; the linked fix PR was closed unmerged and the later workspace-path report was closed as a duplicate of this issue.

## What This Changes

The issue reports that system prompt workspace context and default agent workspace fallback should respect `OPENCLAW_WORKSPACE_DIR` instead of showing the home-directory workspace.

## Best Possible Solution

Update the shared default workspace resolver to prefer a non-empty `OPENCLAW_WORKSPACE_DIR` before home/profile fallback, preserve explicit workspace config precedence, and add focused resolver plus consumer regression coverage.

## Reproduction Assessment

Yes by source inspection: set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace` config, then resolve the default agent workspace or a missing run workspace; current main returns the home/profile fallback and the system prompt renders that value.

## Solution Assessment

Yes. Fixing `src/agents/workspace-default.ts` is the narrow maintainable path because it repairs all downstream consumers; changing only prompt rendering would leave other workspace users on the wrong path.
