## Summary

Keep open. Current main still ignores `OPENCLAW_WORKSPACE_DIR` in the shared default workspace resolver, so the default-agent fallback and system prompt can still point at the home/profile workspace instead of the configured mounted workspace; the referenced fix PR is closed and unmerged.

## What This Changes

Default-agent workspace fallback should use `OPENCLAW_WORKSPACE_DIR` for the system prompt and downstream workspace consumers when no explicit workspace config is set.

## Best Possible Solution

Update the shared default workspace resolver to prefer a non-empty `OPENCLAW_WORKSPACE_DIR` before home/profile fallback while preserving explicit per-agent and `agents.defaults.workspace` precedence, then add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace` config, then resolve the default agent workspace or a missing run workspace; current main returns the home/profile fallback and the prompt renders that value.

## Solution Assessment

Yes. Fixing `src/agents/workspace-default.ts` is the narrow maintainable solution because it repairs the shared resolver for prompt, run fallback, and gateway/CLI consumers; changing prompt rendering alone would only mask one symptom.

## Repair Work Prompt

Fix openclaw/openclaw#66786. The bug is that the default agent workspace fallback ignores `OPENCLAW_WORKSPACE_DIR`, so prompts and fallback workspace consumers can point at the home/profile workspace even when Docker/headless operators configured a mounted workspace path. Update the shared resolver in `src/agents/workspace-default.ts` so a non-empty `OPENCLAW_WORKSPACE_DIR` takes precedence before `OPENCLAW_PROFILE`/`OPENCLAW_HOME` fallback, while preserving explicit per-agent `workspace` and `agents.defaults.workspace` precedence. Add focused regression coverage for the resolver and at least one default-agent consumer path (`resolveAgentWorkspaceDir` or `resolveRunWorkspaceDir`), and keep the change out of prompt rendering except through the corrected resolved workspace. Add a user-facing changelog entry because this changes runtime behavior. Do not add a new config option, migrate startup config, or change non-default agent fallback semantics.
