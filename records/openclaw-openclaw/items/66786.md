## Summary

Keep open. Current main still ignores OPENCLAW_WORKSPACE_DIR in the default workspace resolver, and the downstream default-agent, run-workspace, and system-prompt paths still expose that fallback. The related fix PR was closed unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue asks for the system prompt and default agent workspace resolution to honor OPENCLAW_WORKSPACE_DIR in headless/Docker environments.

## Best Possible Solution

Implement the env-aware default workspace contract in the shared resolver, with OPENCLAW_WORKSPACE_DIR used only when no explicit per-agent or agents.defaults.workspace value exists, and add focused regression coverage for the resolver, default-agent consumer, and run-workspace fallback paths.

## Reproduction Assessment

Yes. A source-level reproduction is clear: set OPENCLAW_WORKSPACE_DIR with no explicit workspace config, then resolve the default agent workspace or a missing run workspace; current main returns the home/profile fallback and the prompt prints that value.

## Solution Assessment

Yes. The narrow maintainable solution is to make resolveDefaultAgentWorkspaceDir prefer a non-empty OPENCLAW_WORKSPACE_DIR while leaving explicit config precedence intact, then lock it with the targeted agent/workspace tests.
