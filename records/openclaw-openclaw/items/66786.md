## Summary

Keep open. Current `main` still ignores `OPENCLAW_WORKSPACE_DIR` in the shared default workspace resolver, and that resolver feeds default-agent workspace selection, missing run-workspace fallback, and the workspace line rendered into the system prompt. The linked PR #66831 described the right narrow fix, but it is closed and unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue requests that default agent workspace resolution and the system prompt honor `OPENCLAW_WORKSPACE_DIR` when no explicit workspace config is set.

## Best Possible Solution

Implement the env-aware default workspace contract in `src/agents/workspace-default.ts`, using a non-empty `OPENCLAW_WORKSPACE_DIR` before profile/home fallback while keeping explicit config precedence unchanged, then lock it with focused regression tests.

## Reproduction Assessment

Yes. Set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace` config, then resolve the default agent workspace or a missing run workspace; current source returns the home/profile fallback, which is then rendered by the system prompt.

## Solution Assessment

Yes, for the future repair. Updating the shared default resolver is the narrow maintainable fix; changing only prompt rendering would leave other workspace consumers pointing at the wrong path.
