## Summary

Keep open. Current main still ignores `OPENCLAW_WORKSPACE_DIR` in the shared default workspace resolver, so default-agent and run-workspace fallbacks can pass a home/profile workspace into the system prompt; the focused fix PR was closed unmerged and the later duplicate issue points back here.

## What This Changes

The issue asks for system-prompt/default workspace resolution to use `OPENCLAW_WORKSPACE_DIR` instead of a home/profile fallback in Docker or headless installs.

## Best Possible Solution

Update the shared default workspace resolver to prefer a non-empty `OPENCLAW_WORKSPACE_DIR` before home/profile fallback, then add focused resolver and default-agent consumer regression coverage.

## Reproduction Assessment

Yes by source inspection: set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace` config, then resolve the default agent workspace or a missing run workspace; current main returns the home/profile fallback and the prompt renders that value. I did not run a live Docker/ECS gateway in this read-only pass.

## Solution Assessment

Yes. Fixing `src/agents/workspace-default.ts` is the narrow maintainable path because it repairs all downstream workspace consumers; changing only prompt rendering would leave other resolver users on the wrong path.
