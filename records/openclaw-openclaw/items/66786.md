## Summary

Keep open. Current main still ignores `OPENCLAW_WORKSPACE_DIR` in the shared default workspace resolver, so the configured workspace can be missed before the system prompt renders it; the linked fix PR #66831 was closed unmerged and does not supersede the issue.

## What This Changes

The issue asks for default agent workspace resolution and the system prompt to honor `OPENCLAW_WORKSPACE_DIR` in headless Docker/Linux installs.

## Best Possible Solution

Update the shared default workspace resolver to prefer a non-empty `OPENCLAW_WORKSPACE_DIR` before the home/profile fallback, preserve explicit config precedence, and add focused resolver plus consumer regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_WORKSPACE_DIR` with no explicit per-agent or `agents.defaults.workspace` config, then the default resolver still returns the home/profile fallback that feeds run workspace resolution and the prompt.

## Solution Assessment

Yes. Fixing `src/agents/workspace-default.ts` or an equivalent shared resolver seam is the narrow maintainable solution; changing prompt rendering alone would hide only one symptom.
