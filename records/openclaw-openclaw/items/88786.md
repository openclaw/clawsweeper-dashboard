## What This Changes

This PR reorders Control UI WebChat final/aborted terminal handling to clear active stream/run state before appending the terminal assistant message and adds regression assertions for assignment order.

## Best Possible Solution

Land this ordering fix after contributor or maintainer proof exercises a real Control UI WebChat send and shows the duplicate render is gone; keep broader persisted-transcript dedupe on its separate track.

## Reproduction Assessment

Yes, at source level: call handleChatEvent for an active run with non-null chatStream and a final or aborted payload, and current main writes chatMessages before reconcileTerminalRun clears the stream. I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes, this is the best fix shape I found: snapshot fallback content, clear terminal run/stream state first, then commit the terminal message. Renderer-level dedupe or broad transcript filtering would be a wider workaround and would risk hiding separate backend duplication cases.
