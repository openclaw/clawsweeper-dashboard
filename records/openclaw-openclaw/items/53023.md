## Summary

Keep open: current main and v2026.5.19 still serialize new session lanes at one active task, expose only global/subagent concurrency defaults, keep the queue wait warning fallback at 2 seconds unless a caller passes an override, and support yielding only through explicit sessions_yield; the related open PR and issues cover slices but not the full umbrella request.

## What This Changes

The issue requests configurable per-session lane concurrency, automatic yielding during long tool execution, and a configurable lane-wait warning threshold.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers decide auto-yield semantics, review or replace https://github.com/openclaw/openclaw/pull/63864 for the session-concurrency slice, and leave the warning-threshold slice to https://github.com/openclaw/openclaw/issues/14747 unless it is intentionally folded into one design.

## Reproduction Assessment

Yes, at source level. Current main still creates unconfigured session lanes with maxConcurrent: 1, routes embedded runs through session:<key>, lacks the requested config keys, and only yields when sessions_yield is explicitly called.

## Solution Assessment

Unclear as a single implementation. Session concurrency, lane warning thresholds, and auto-yield are separable changes, and auto-yield needs maintainer decisions around cancellation, transcript ordering, and session mutation safety.
