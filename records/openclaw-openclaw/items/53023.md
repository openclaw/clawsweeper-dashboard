## Summary

Keep open: current main and v2026.5.22 still serialize normal session lanes at one active run, expose no requested session-concurrency/auto-yield/lane-wait config keys, and have only partial open trackers for slices of the umbrella.

## What This Changes

The issue requests configurable same-session lane concurrency, automatic yielding during long tool execution, and a configurable lane-wait diagnostic threshold.

## Best Possible Solution

Keep this as the umbrella tracker until maintainers split or approve the scheduler design, then land bounded changes for session concurrency, warning thresholds, and auto-yield with compatibility tests.

## Reproduction Assessment

Yes, at source level: current main still routes embedded runs through per-session lanes whose default cap is one active task, and the requested config keys are absent. I did not run a live multi-minute queue-delay reproduction because this review is read-only and the source path already proves the reported limitation.

## Solution Assessment

Unclear as one combined change. Configurable session concurrency, diagnostic-threshold configuration, and auto-yield are separable changes, and auto-yield needs a maintainer decision around cancellation, transcript ordering, and session mutation safety.
