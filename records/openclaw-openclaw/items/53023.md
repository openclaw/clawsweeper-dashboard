## Summary

Keep open: current main and the latest release still serialize each session lane, expose only global/subagent concurrency, keep the lane-wait warning fallback at 2 seconds, and implement yield only through explicit `sessions_yield`; the related PR and issues cover slices but not the full umbrella request.

## What This Changes

The issue requests configurable same-session lane concurrency, automatic yielding, and configurable lane-wait warning thresholds for long-running interactive agent sessions.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers decide the auto-yield semantics, review or replace the open session-concurrency PR, and leave the warning-threshold slice to the narrower diagnostic-threshold issue.

## Reproduction Assessment

Yes, at source level. Current main still creates unconfigured lanes with `maxConcurrent: 1`, routes embedded runs through `session:<key>`, lacks the requested config keys, and only yields when `sessions_yield` is explicitly called.

## Solution Assessment

Unclear as a single implementation. Session concurrency, warning thresholds, and auto-yield are separable changes, and auto-yield needs maintainer decisions around cancellation, transcript ordering, and session mutation safety.
