## Summary

Keep open. Current main still has serialized per-session lanes, no requested agent-default config keys, a hardcoded lane-wait warning fallback, and only explicit `sessions_yield`; the related open PR and narrower open issue cover only parts of this umbrella.

## What This Changes

Request configurable per-session lane concurrency, automatic yielding after sustained tool execution, and a configurable lane wait warning threshold.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers review or replace the session-concurrency PR, keep the warning-threshold work tracked separately, and design auto-yield with explicit cancellation and transcript-ordering semantics.

## Reproduction Assessment

Yes, at source level. Two same-session embedded runs still share a `session:<key>` lane with default concurrency 1, omitted lane warning thresholds still fall back to 2,000 ms, and the requested config keys are absent from current schema/resolver surfaces.

## Solution Assessment

Unclear as a single patch. Session concurrency and warning thresholds are separable config work, while auto-yield needs maintainer design around cancellation, transcript ordering, and session mutation safety.
