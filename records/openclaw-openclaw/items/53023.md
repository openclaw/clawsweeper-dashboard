## Summary

Keep open. Current main still has serialized dynamically created session lanes, no agent-defaults config for session lane concurrency or auto-yield, a hardcoded default lane-wait warning threshold, and only explicit `sessions_yield` behavior. The supplied related context also shows one open PR covering only the session-concurrency slice and a separate open issue for warning-threshold configurability, so this umbrella is still valid.

## What This Changes

The issue requests configurable per-session lane concurrency, automatic yielding after long tool execution, and a configurable lane wait warning threshold for interactive agent sessions.

## Best Possible Solution

Keep this as the umbrella tracker until the product shape is settled: land or rework the session-concurrency slice, decide whether the diagnostic-threshold slice remains tracked separately, and define an explicit auto-yield contract in the embedded runner/tool loop before adding config, docs, and tests.
