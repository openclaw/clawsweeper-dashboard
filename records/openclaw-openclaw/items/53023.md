## Summary

Keep open: current main and the latest release still serialize same-session agent runs at one active session lane, expose no requested session-lane or auto-yield agent-default keys, and keep the default lane-wait warning at two seconds; the open related PR only covers the session-concurrency slice and is still proof/risk gated.

## What This Changes

Request configurable same-session lane concurrency, automatic yielding after long tool execution, and a configurable lane-wait warning threshold.

## Best Possible Solution

Keep this as the umbrella tracker until maintainers approve the scheduler design: review or rework https://github.com/openclaw/openclaw/pull/63864 for the concurrency slice, keep https://github.com/openclaw/openclaw/issues/14747 for the warning threshold, and design auto-yield with explicit transcript and cancellation semantics.

## Reproduction Assessment

Yes, at source level. Current main still routes embedded runs through `session:<key>` lanes capped at one active task, lacks the requested config keys, and only yields through the explicit `sessions_yield` tool.

## Solution Assessment

Unclear as one implementation. The concurrency knob, diagnostic threshold, and auto-yield policy are separable, and auto-yield needs maintainer decisions on cancellation, transcript ordering, and session mutation safety.
