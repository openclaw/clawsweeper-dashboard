## Summary

Keep open. Current main still serializes per-session lanes, lacks the requested agent-default config keys, keeps the default lane-wait warning fallback at 2 seconds, and only supports explicit `sessions_yield`; the related open session-concurrency PR and narrower warning-threshold issue cover only parts of this umbrella.

## What This Changes

The issue requests configurable per-session lane concurrency, automatic yielding during long tool execution, and a configurable lane-wait warning threshold.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers review or replace the session-concurrency PR, leave the warning-threshold slice to https://github.com/openclaw/openclaw/issues/14747, and design auto-yield with explicit cancellation and transcript-ordering semantics before implementation.

## Reproduction Assessment

Yes, at source level. New session lanes still default to one active run, embedded runs still enter `session:<key>`, and the requested config keys are absent from current schema/resolver surfaces.

## Solution Assessment

Unclear as a single patch. Session concurrency and lane warning thresholds are separable config work, while auto-yield needs maintainer design around cancellation, transcript ordering, and session mutation safety.
