## Summary

Keep open: current main still serializes each session lane, only exposes global/subagent concurrency, keeps the default lane-wait warning at 2 seconds, and only yields through explicit `sessions_yield`. The related open work covers slices of this request, but the general auto-yield and same-session concurrency semantics still need maintainer product review.

## What This Changes

Request configurable same-session lane concurrency, automatic yield after long tool execution, and a configurable lane-wait warning threshold.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers decide auto-yield semantics, review or replace the open session-concurrency pull request, and leave the warning-threshold slice to the narrower diagnostic-threshold issue.

## Reproduction Assessment

Yes, at source level. Current main still creates new lanes with `maxConcurrent: 1`, routes embedded runs through `session:<key>` before the global lane, lacks the requested config keys, and only yields when `sessions_yield` is explicitly called.

## Solution Assessment

Unclear as a single implementation. Session concurrency, lane warning thresholds, and auto-yield are separable changes, and auto-yield needs maintainer decisions around cancellation, transcript ordering, and session mutation safety.
