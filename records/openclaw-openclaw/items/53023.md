## Summary

Keep open: current main and the latest release still serialize each session lane at one active run, lack the requested agent-defaults keys, and only support explicit `sessions_yield`. The open session-concurrency PR covers one slice, while the warning-threshold slice and auto-yield semantics remain separate maintainer decisions.

## What This Changes

The issue requests configurable per-session lane concurrency, automatic yielding during long tool execution, and configurable lane-wait diagnostic thresholds.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers review or replace https://github.com/openclaw/openclaw/pull/63864, keep https://github.com/openclaw/openclaw/issues/14747 as the warning-threshold slice unless folded into one design, and specify auto-yield semantics before implementation.

## Reproduction Assessment

Yes, at source level. Current main still creates session lanes with `maxConcurrent: 1`, routes embedded runs through `session:<key>`, lacks the requested config keys, and only yields when `sessions_yield` is explicitly called.

## Solution Assessment

Unclear as one implementation. Session concurrency, diagnostic thresholds, and auto-yield are separable changes, and auto-yield needs maintainer decisions around cancellation, transcript ordering, and session mutation safety.
