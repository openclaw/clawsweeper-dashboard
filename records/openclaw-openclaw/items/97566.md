## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Normalize the Pioneer auto catalog id:** `extensions/pioneer/openclaw.plugin.json:59`
  - body: `pioneer/auto` is parsed as provider `pioneer` plus model `auto` because OpenClaw strips a duplicated self-provider prefix when resolving model refs. Declaring the catalog row id itself as `pioneer/auto` makes the advertised default primary point at a different catalog key, so fresh installs can default to a model that is not the actual catalog entry; declare the row as `auto` and let `normalizeModelId` send `pioneer/auto` to the API, or make the literal double-prefixed ref intentional everywhere.
  - confidence: 0.87
- **[P2] Remove the unsupported defaultModelId option:** `extensions/pioneer/onboard.ts:25`
  - body: `applyProviderConfigWithModelCatalog` does not accept a `defaultModelId` property in its parameter type, so this object literal will fail the core TypeScript check. The primary default is already set by `applyAgentDefaultModelPrimary`, so remove this field or switch to a helper that actually owns that option.
  - confidence: 0.93
- **[P2] Remove the unused scopedPluginIdSet:** `src/commands/models/list.provider-catalog.ts:349`
  - body: This new local is never read, and `tsconfig.core.json` enables `noUnusedLocals`, so the PR cannot pass type checking as submitted. Remove the declaration or use it in the provider filter that survived the revert.
  - confidence: 0.96
- **[P3] Align the docs with the Pioneer default:** `docs/providers/pioneer.md:37-38`
  - body: The implementation sets `PIONEER_DEFAULT_MODEL_REF` to `pioneer/auto`, but the provider docs tell users onboarding sets `pioneer/claude-sonnet-4-6`. Update the docs or the implementation so setup instructions match the actual generated config.
  - confidence: 0.91
