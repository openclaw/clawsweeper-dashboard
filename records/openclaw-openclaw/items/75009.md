## Summary

Keep open. The PR is a useful WhatsApp feature, but the latest diff still lets global direct-chat `messages.visibleReplies` override the documented WhatsApp DM `automatic` mode, and the new user-facing config key is missing the required changelog entry.

## What This Changes

The PR adds `channels.whatsapp.dmVisibleReplies` with per-account overrides, WhatsApp DM message-tool-only dispatch, same-chat final-reply fallback handling, prompt copy, docs, schema updates, and targeted tests.

## Best Possible Solution

Keep the feature, but make WhatsApp DM mode precedence explicit before shared dispatch, add the active-release changelog entry, and then run the targeted tests plus the changed gate.

## Reproduction Assessment

Yes. Configure `messages.visibleReplies: "message_tool"` with a WhatsApp direct chat whose resolved `dmVisibleReplies` is unset or `"automatic"`; the PR passes `undefined`, so shared dispatch can resolve that DM to `message_tool_only` instead of documented automatic final delivery.

## Solution Assessment

No. The feature direction and fallback design are narrow, but the implementation should pass explicit `"automatic"` for automatic WhatsApp DMs or intentionally document/test global precedence; the current docs and code disagree.
