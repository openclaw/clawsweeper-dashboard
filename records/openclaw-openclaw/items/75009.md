## Summary

Keep this PR open. Current main does not already have the WhatsApp `dmVisibleReplies` surface, so the feature is not obsolete, but the branch still has an actionable code/docs mismatch around automatic DM mode precedence and is missing the required user-facing changelog entry.

## What This Changes

The branch adds a WhatsApp DM `dmVisibleReplies` config with account overrides, routes opted-in DMs through message-tool-only dispatch with a suppressed-final fallback, updates prompt/docs/schema surfaces, and adds targeted tests.

## Best Possible Solution

Keep the feature direction, but update the branch so WhatsApp DM visible-reply precedence is explicit and tested, add the required changelog entry, then run the targeted WhatsApp/config/dispatch tests plus the normal changed gate before merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `messages.visibleReplies: "message_tool"` with a WhatsApp direct chat whose resolved `dmVisibleReplies` is `"automatic"`; the PR passes `undefined` at `process-message.ts:220`, so shared dispatch can still resolve the DM to `message_tool_only`.

## Solution Assessment

No. The PR’s architecture is mostly narrow and maintainable, but the automatic-mode precedence gap means the proposed config does not reliably match its docs; the safer fix is to encode the intended WhatsApp-vs-global precedence and lock it with a regression test.
