## What This Changes

This PR adds a Telegram forum-topic hint from supergroup `is_topic_message` to forum resolution call sites and regression tests for routing when `chat.is_forum` is omitted.

## Best Possible Solution

Land the centralized helper-based fix with the focused regressions if maintainers accept the intended topic-session split, and use optional live Telegram proof only if they want transport coverage beyond the provided logs.

## Reproduction Assessment

Yes, by source inspection. Current main already uses `is_topic_message` in the sequential lane but later `resolveTelegramForumFlag` ignores it and can return false when `chat.is_forum` is absent and `getChat` is unavailable or fails.

## Solution Assessment

Yes. Centralizing the supergroup topic-message hint in the existing forum resolver and passing it from the affected call sites is the narrow maintainable fix; the remaining question is upgrade/session-risk acceptance, not a better code path.
