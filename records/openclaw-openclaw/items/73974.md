## Summary

Keep open; current main still lacks the macOS app ownership guard, and the proposed patch has a blocking label-only detection path that can suppress CLI repair for `OPENCLAW_PROFILE=mac`.

## What This Changes

The PR adds macOS OpenClaw.app LaunchAgent detection to doctor daemon/service-config repair paths, docs, changelog, and launchd scanner regression tests.

## Best Possible Solution

Keep the app ownership guard, but require app-specific launchd evidence before suppressing CLI repair and preserve repair for profile-named CLI services.

## Reproduction Assessment

Yes, source-backed. Current main can still reach LaunchAgent bootstrap/install/restart without an app-owner guard, and the PR regression is source-reproducible with `OPENCLAW_PROFILE=mac` plus a CLI-owned `ai.openclaw.mac` plist.

## Solution Assessment

No. The direction is narrow, but label-only app ownership detection is not the best implementation; require app-specific `ProgramArguments` or equivalent launchd evidence before skipping CLI repair.
