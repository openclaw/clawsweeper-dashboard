## Summary

Keep open. The PR targets the right macOS doctor/gateway surface and carries useful context from #67631 and #42573, but the proposed app-owner detection can falsely suppress CLI repair for the existing `OPENCLAW_PROFILE=mac` LaunchAgent path.

## What This Changes

The PR adds macOS OpenClaw.app LaunchAgent ownership detection to doctor daemon and service-config repairs, plus docs, changelog, and focused launchd regression tests.

## Best Possible Solution

Keep the product direction, but require app-specific launchd evidence before suppressing CLI repair: validate the installed or loaded job points at `OpenClaw.app/Contents/MacOS/OpenClaw`, preserve `OPENCLAW_PROFILE=mac` CLI behavior, and add regression coverage for that collision.

## Reproduction Assessment

Yes. The original duplicate-runtime path is backed by #67631/#42573, and the PR regression is reproducible by combining `OPENCLAW_PROFILE=mac` with a CLI-managed plist labeled `ai.openclaw.mac`.

## Solution Assessment

No. Skipping CLI repair when OpenClaw.app owns launchd is the right narrow direction, but label-only ownership detection is not safe because the existing profile label contract can produce the same label.
