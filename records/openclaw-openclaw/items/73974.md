## Summary

Keep this PR open, but it needs changes before merge. The branch targets a real current-main gap for macOS app-owned launchd repair, but the patch currently treats the CLI profile label `ai.openclaw.mac` as proof that OpenClaw.app owns the runtime, which can suppress valid CLI repair for users running `OPENCLAW_PROFILE=mac`.

## What This Changes

The PR adds macOS OpenClaw.app LaunchAgent ownership detection to doctor daemon and service-config repair paths, updates doctor docs and the changelog, and adds launchd scanner/regression tests.

## Best Possible Solution

Keep the product direction, but require app-specific launchd evidence before suppressing CLI repair: validate the plist or loaded job points at `OpenClaw.app/Contents/MacOS/OpenClaw`, preserve default-profile behavior, and add regression coverage for `OPENCLAW_PROFILE=mac` plus nearby profile labels.

## Reproduction Assessment

Yes. The linked issue has concrete macOS steps for the original duplicate-runtime problem, and the PR regression is reproducible with a CLI-managed `OPENCLAW_PROFILE=mac` LaunchAgent whose label is `ai.openclaw.mac`.

## Solution Assessment

No. The proposed direction is narrow and appropriate, but label-only ownership detection is not the best fix because the existing profile-label contract can produce the same label for a CLI-managed service.
