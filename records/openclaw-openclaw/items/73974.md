## Summary

Keep open. Current main still lacks the macOS app LaunchAgent ownership guard, but this PR should not merge as-is because its label-only detection can treat a legitimate `OPENCLAW_PROFILE=mac` CLI LaunchAgent as OpenClaw.app ownership and skip needed repair.

## What This Changes

The PR adds macOS OpenClaw.app LaunchAgent detection to doctor daemon/service-config repair paths, docs, changelog, and focused launchd scanner regression tests.

## Best Possible Solution

Keep the app ownership guard, but require app-specific launchd evidence before suppressing CLI repair and preserve repair for profile-named CLI services.

## Reproduction Assessment

Yes, source-backed. Current main can still reach LaunchAgent bootstrap/install/restart/config-rewrite paths without an app-owner guard, and the PR regression is source-reproducible with `OPENCLAW_PROFILE=mac` plus a CLI-owned `ai.openclaw.mac` plist.

## Solution Assessment

No. The direction is narrow and useful, but label-only app ownership detection is not the best implementation; require app-specific `ProgramArguments` or equivalent launchd evidence before skipping CLI repair.
