## Summary

Keep this PR open because it still carries a useful narrow fix, but the current merge ref is not safe to merge: `macosExecArgs` is used before its `const` declaration, so the macOS update lane would fail before reaching the intended Parallels smoke path.

## What This Changes

The PR changes the Parallels npm-update smoke harness so macOS guest script writes and chmod run through the resolved desktop-user exec transport, restores mode `700`, and adds a focused regression assertion.

## Best Possible Solution

Keep the macOS same-identity script writer with owner-only permissions, preserve the Linux root-owned path, make the ordering safe on current `main`, and require focused plus live macOS Parallels validation before merge.

## Reproduction Assessment

Yes for the PR regression by source inspection: the merge ref uses `macosExecArgs` before initialization. The original permission bug is source-reproducible from the root-owned script write plus desktop-user execution path, but still needs live macOS Parallels confirmation.

## Solution Assessment

No as currently mergeable: the same-identity exec transport is the narrow maintainable fix, but the merge ref must resolve `macosExecArgs` before passing it to `writeGuestScript`.
