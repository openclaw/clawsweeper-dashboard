## Summary

Keep this PR open. Current main has a readability workaround for the macOS Parallels update script, but it does not implement the PR's safer same-transport, owner-only script path; the proposed change is narrow and I found no blocking correctness issue in the patch itself.

## What This Changes

The PR changes the Parallels npm-update smoke harness so macOS update scripts are written and chmodded through the resolved macOS exec transport, restores chmod 700, and adds a regression test for that transport shape.

## Best Possible Solution

Rebase this focused PR onto current main, preserve the later fallback behavior only where still needed, and land after the changed-file gate plus macOS Parallels npm-update smoke prove the script remains owner-readable only while Linux keeps its root-owned path.

## Reproduction Assessment

Unclear: current main no longer matches the original permission-denied shape because it chmods the script to 755 and chowns the sudo fallback case. The remaining concern is source-verifiable, with live validation through `timeout --foreground 75m pnpm test:parallels:npm-update -- --platform macos --json` on a Parallels host.

## Solution Assessment

Yes. Using the same resolved exec args for tee, chmod, and bash is narrower and safer than the current 755/chown workaround because it aligns writer and runner identity while avoiding world-readable provider credential material.
