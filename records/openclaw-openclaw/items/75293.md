Current main has a later chmod 755 plus sudo-fallback chown mitigation that likely avoids the original permission-denied failure, but it does not implement the PR's narrower same-transport chmod 700 approach, and GitHub currently reports the branch as dirty/unrebaseable against main. ## What This Changes

The PR changes the Parallels npm-update smoke harness so macOS guest scripts are written and chmodded through the resolved macOS exec transport, restores chmod 700, and adds a regression test for that transport shape.

## Best Possible Solution

Rebase or adapt the PR onto current main so the macOS update script is written, chmodded, and executed through the same resolved user transport with non-world-readable permissions, while keeping Linux on the root-owned path and preserving any current fallback behavior that remains necessary.

## Solution Assessment
