## Summary

Keep open: the PR addresses a real Parallels macOS smoke-harness regression that is still present on current main, but the live merge ref currently introduces a TDZ crash before the macOS lane can write the guest script.

## What This Changes

The PR changes the Parallels npm-update macOS smoke harness to write and chmod the guest script through the same desktop-user exec transport used to run it, and adds a focused regression assertion.

## Best Possible Solution

Keep the same-identity macOS script writer, preserve the Linux default path, move macosExecArgs resolution before the write call in the merge context, and require focused plus live macOS Parallels validation before merge.

## Reproduction Assessment

Yes for the PR regression: the current merge ref has a deterministic source-level TDZ ReferenceError because macosExecArgs is used before its const declaration. The original permission issue is also source-reproducible from the root write plus desktop-user execute path, but still needs live macOS Parallels confirmation.

## Solution Assessment

No as currently mergeable: routing write/chmod through the resolved macOS exec transport is the narrow maintainable fix, but the merge ref must resolve macosExecArgs before passing it to writeGuestScript.
