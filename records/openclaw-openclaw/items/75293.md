## Summary

Keep open: this PR still targets a real Parallels npm-update smoke regression that is not fixed on current main, but the live merge ref introduces a TDZ `ReferenceError` by using `macosExecArgs` before it is declared.

## What This Changes

The PR routes macOS Parallels npm-update guest script writes and chmod through the resolved execution args, restores mode `700`, and adds a regression test.

## Best Possible Solution

Repair the merge-ref ordering bug, keep private same-identity script handling for macOS, preserve the Linux root-owned path, and require focused plus live macOS Parallels proof before merge.

## Reproduction Assessment

Yes for the PR regression: source inspection of the live merge ref shows `macosExecArgs` used before declaration in `guestMacos()`. The original Parallels permission behavior still needs live macOS host validation.

## Solution Assessment

No as currently merged: the same-transport `700` script approach is the narrow maintainable direction, but the resolver must run before `writeGuestScript()` receives `macosExecArgs`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Resolve macOS exec args before writing the script:** `scripts/e2e/parallels/npm-update-smoke.ts:665`
  - body: In the live merge ref, `writeGuestScript(..., macosExecArgs)` runs before `const macosExecArgs = this.resolveMacosUpdateExecArgs(ctx)`. That will throw a TDZ `ReferenceError` before the macOS npm-update smoke can write or run the guest script, so move the resolver above the write call while keeping the Linux default path intact.
  - confidence: 0.96
