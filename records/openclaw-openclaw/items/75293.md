## Summary

Keep open: this PR still carries a useful macOS Parallels smoke hardening, but the current GitHub merge ref is incorrect because `macosExecArgs` is passed before it is declared.

## What This Changes

This PR routes macOS Parallels npm-update guest script writes and chmod through the same exec transport used to run the script, restores mode `700`, and adds a regression test.

## Best Possible Solution

Repair the stale merge ordering, keep same-identity `700` script handling, preserve the Linux root-owned path, and require focused plus live macOS Parallels proof before merge.

## Reproduction Assessment

Yes for the PR regression: source inspection of the GitHub merge ref shows a TDZ `ReferenceError` path before the macOS smoke can write the guest script. The original Parallels permission behavior still needs live host validation.

## Solution Assessment

No as currently merged: the same-identity writer approach is the right narrow direction, but the merge ref must resolve `macosExecArgs` before passing it to `writeGuestScript()`.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Resolve macOS exec args before writing the script:** `scripts/e2e/parallels/npm-update-smoke.ts:665`
