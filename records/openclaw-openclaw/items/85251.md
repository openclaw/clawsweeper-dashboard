## Summary

Keep open: current main and v2026.5.22 still let a current-turn Codex `turn/started` notification clear the short completion-idle guard before real progress or a terminal event, and the direct guard PR closed unmerged while the open plugin realpath PR is only a partial root-cause path.

## What This Changes

Add bounded handling for Codex app-server turns that emit `turn/started` and then stop producing progress, so embedded runs fail retryably instead of waiting for stuck-session recovery.

## Best Possible Solution

Land the narrow Codex completion-idle guard fix for this issue, while keeping broader plugin-discovery and diagnostic recovery work tracked through https://github.com/openclaw/openclaw/pull/85494 and https://github.com/openclaw/openclaw/issues/85532.

## Solution Assessment
