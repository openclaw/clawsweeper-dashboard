## Summary

Keep open: current main still uses the synchronous systemd restart path, so the useful Linux/systemd self-restart fix is not obsolete. This PR is not merge-ready because the proposed transient handoff passes a shell script with raw dollar variables through systemd-run, the branch is dirty against current main, and exact-head real user-systemd proof is still insufficient.

## What This Changes

The PR adds Linux/systemd self-restart detection, a detached systemd-run restart handoff, marker-based SIGTERM restart handling, focused tests, and a changelog entry.

## Best Possible Solution

Rebase or port the self-restart handoff onto current main using an expansion-safe systemd-run invocation or argv-based helper, remove the changelog edit, and merge only after exact-head real user-systemd proof.

## Reproduction Assessment

Yes from source inspection, though I did not run a live systemd repro in this read-only review. Current main generates KillMode=control-group systemd units and still runs systemctl restart synchronously, matching systemd's documented cgroup SIGTERM behavior.

## Solution Assessment

No as currently written. The detached handoff is the right layer, but the transient systemd command must be expansion-safe or use the existing argv-based helper pattern before this is the best fix.

- **[P1] Escape the systemd-run handoff script dollars:** `src/daemon/systemd.ts:260`
  - body: systemd-run creates a transient service, and systemd applies ExecStart-style environment expansion to command arguments before /bin/sh runs.
