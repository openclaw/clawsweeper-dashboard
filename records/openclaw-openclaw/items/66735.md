## Summary

Keep open: current main still lacks the Linux/systemd self-restart handoff, and a maintainer already repaired this branch to keep the contributor path canonical. It is not merge-ready because the branch conflicts with current main, current-head real systemd proof is insufficient, and the release-owned changelog edit should be removed.

## What This Changes

This PR adds a Linux/systemd self-restart handoff that detects in-unit restarts, schedules a detached `systemd-run --user` restart with a short-lived marker, treats the resulting `SIGTERM` as supervised restart semantics, and adds daemon/run-loop coverage.

## Best Possible Solution

Rebase the useful systemd self-restart handoff onto current main, align it with the current run-loop and launchd handoff patterns, remove the release-owned changelog edit, and merge only after exact-head real user-systemd proof.

## Reproduction Assessment

Yes from source inspection, though I did not run a live systemd repro in this read-only review. Current main uses `KillMode=control-group` for generated systemd units and still runs `systemctl --user restart` synchronously, matching systemd's documented cgroup SIGTERM behavior.

## Solution Assessment

Unclear as a merge-ready solution. The detached handoff is directionally consistent with the launchd sibling pattern, but this stale/conflicting branch must be rebased onto the current restart loop and proved on a real user-systemd service before it is the best fix.
