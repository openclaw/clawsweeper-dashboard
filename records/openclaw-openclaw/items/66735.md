## Summary

Keep open: current main still lacks the Linux/systemd self-restart handoff and related user reports point to this PR as the canonical fix path, but the branch is not safe to merge because it introduces a predictable shared-tmp restart marker, lacks final-head real systemd proof, and conflicts with newer restart-intent lifecycle code.

## What This Changes

The PR adds Linux/systemd in-unit restart detection, schedules self-restarts through a detached `systemd-run --user` handoff, treats matching SIGTERM as restart semantics, and adds daemon/run-loop tests plus a changelog entry.

## Best Possible Solution

Land a hardened Linux/systemd self-restart handoff that uses the state-dir restart-intent contract or an equivalent protected marker, preserves current restart-intent payload behavior, and includes final-head user-systemd proof.

## Reproduction Assessment

Yes, at source level: current main synchronously awaits `systemctl --user restart` for a generated `KillMode=control-group` user unit, and systemd documents restart as stop-then-start with control-group processes killed on stop. I did not run a live user-systemd reproduction in this read-only review.

## Solution Assessment

No, not yet. The detached handoff direction is appropriate, but the marker IPC needs state-dir/atomic/nonce-style hardening and the branch must be rebased onto current restart-intent payload handling before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Store restart markers outside shared tmp:** `src/daemon/systemd.ts:84-88`
  - body: The marker path is derived from the unit name under `TMPDIR`/`os.tmpdir()`, and the consumer trusts any readable matching file without ownership, mode, link-count, or nonce checks. A local process can pre-create, block, or spoof the marker so a later unrelated `SIGTERM` is handled as a restart; use the OpenClaw state dir with atomic `0600` writes and link checks, or reuse the restart-intent contract.
  - confidence: 0.86
