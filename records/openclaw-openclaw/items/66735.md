## Summary

Keep open: current main still lacks the Linux/systemd self-restart handoff, and the PR remains useful, but it is not merge-ready because the branch is conflicting and exact-head real user-systemd proof is still insufficient.

## What This Changes

This PR adds a Linux/systemd self-restart path that detects in-unit gateway restarts, schedules a detached `systemd-run --user` restart handoff, and treats the expected SIGTERM as supervised restart semantics.

## Best Possible Solution

Rebase the useful systemd self-restart handoff onto current main, remove the release-owned changelog edit, and merge only after exact-head live user-systemd proof covers both external restart and in-unit self-restart.

## Reproduction Assessment

Yes, as a source-level reproduction: current main generates `KillMode=control-group` units and still runs `systemctl --user restart` synchronously, while systemd documents that control-group stops SIGTERM the unit cgroup. I did not run a live systemd user-service repro in this read-only review.

## Solution Assessment

Unclear as a merge-ready solution. The detached handoff is the right owner-boundary direction and matches the launchd sibling pattern, but this branch must be rebased onto current main and proved on a real user-systemd service before it is the best fix to land.
