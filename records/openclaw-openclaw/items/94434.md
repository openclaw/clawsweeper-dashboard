## Summary

Keep open: current main and the latest release still treat local `exec.workdir` as a literal path and fall back before execution, while two open closing PRs now own competing fix semantics.

## What This Changes

The issue asks local exec workdir handling to stop treating leading-tilde paths as literal values and then running commands from a fallback directory.

## Best Possible Solution

Land one canonical exec workdir fix with focused regression coverage; maintainers should choose expansion-only, fail-fast, or expand-then-fail-fast semantics before closing this issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: local gateway `exec` with `workdir: "~/..."` reaches `resolveWorkdir()`, stats the literal value, warns, and then uses the fallback cwd before execution.

## Solution Assessment

Unclear which final semantics are best without maintainer choice. Keeping this issue open is correct while maintainers choose between the open fail-fast and OS-home expansion candidates, or require a combined rule.
