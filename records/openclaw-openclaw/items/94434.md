## Summary

Keep open: current main and v2026.6.10 still stat the literal local `exec.workdir` and fall back before execution, while https://github.com/openclaw/openclaw/pull/94441 and https://github.com/openclaw/openclaw/pull/94449 are open candidate fixes with different semantics.

## What This Changes

The issue requests that local `exec` stop running commands from a fallback cwd when an explicit `workdir` such as `~/...` is unavailable, either by expanding leading home paths or failing fast.

## Best Possible Solution

Land one canonical exec workdir fix with focused regression coverage, making the chosen local semantics explicit and keeping sandbox and remote-node cwd handling separate.

## Reproduction Assessment

Yes. Source inspection shows local gateway exec passes an explicit `workdir` through `resolveWorkdir()`, which stats the literal `~/...` path and falls back before command execution.

## Solution Assessment

Unclear. The report is valid, but maintainers need to choose fail-fast, leading-home expansion, or a combined expand-then-fail-fast rule before one candidate PR lands.
