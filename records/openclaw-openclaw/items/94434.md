## Summary

Keep open: current main and v2026.6.9 still stat the literal local `exec.workdir` and fall back before command execution, while https://github.com/openclaw/openclaw/pull/94441 and https://github.com/openclaw/openclaw/pull/94449 are open closing PRs with different proposed semantics.

## What This Changes

The issue reports that local exec tool calls with an explicit `workdir` beginning with `~` are treated literally, fall back to another cwd, and still execute the command.

## Best Possible Solution

Review and land one canonical exec workdir fix, keeping sandbox and node routing separate and making the chosen local explicit-workdir semantics covered by focused regression tests.

## Reproduction Assessment

Yes by source inspection: local `exec` with `workdir: "~/..."` reaches `resolveWorkdir()`, which stats the literal value and returns fallback cwd/home before command execution. I did not run a live current-main repro in this read-only review.

## Solution Assessment

Unclear which implementation semantics are best without maintainer choice. Keeping this issue open is the right path until maintainers choose and land one linked PR, or request a combined expand-then-fail-fast fix.
