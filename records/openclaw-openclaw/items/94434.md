## Summary

Keep open: current main and v2026.6.8 still route explicit local exec workdirs through the literal-stat fallback path, and several open closing PRs are candidate fixes rather than merged fixes.

## What This Changes

The issue asks OpenClaw exec to stop running commands from a fallback directory when an explicit local `workdir` such as `~/...` is unavailable or unexpanded.

## Best Possible Solution

Land one canonical local exec workdir fix with focused regression coverage, then let the merged PR close this issue and supersede duplicate fix attempts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: local exec with `workdir: "~/..."` reaches `resolveWorkdir()`, stats the literal string, warns, and returns a fallback; I did not run a live command because this review is read-only.

## Solution Assessment

Unclear which implementation is best until maintainers choose fail-fast versus leading-home expansion. The best current process is to keep the issue open and review or consolidate the linked fix PRs.
