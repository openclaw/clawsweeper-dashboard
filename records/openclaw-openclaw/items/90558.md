## What This Changes

The PR changes Discord read-target authorization and tests so freshly fetched one-to-one DM channels can be read only when the single recipient is authorized by the effective DM policy and allowlist, while disabled, group, malformed, and DM-channel-id cases fail closed.

## Best Possible Solution

Land this PR or a canonical equivalent after redacted live Discord proof shows allowlisted one-to-one DM reads succeed and disabled, group, malformed, nonmatching, and DM-channel-id targets fail closed.

## Reproduction Assessment

Yes, at source level. Current main traces readMessages and fetchMessage into assertReadTargetAllowed, where fetched one-to-one DM metadata without a guild id cannot match the guild-channel fallback by recipient allowlist.

## Solution Assessment

Yes for the source shape: reusing resolveDiscordDmCommandAccess keeps the fix in the Discord plugin and avoids a second allowlist parser. It is not merge-ready until real Discord proof and maintainer acceptance of the deliberate fail-closed boundary are in place.
