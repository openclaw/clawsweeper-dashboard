## What This Changes

The PR changes Discord read-target authorization and tests so fetched one-to-one DM channels can be read only when the single recipient is authorized by the effective DM policy and allowlist.

## Best Possible Solution

## Solution Assessment

Yes for the source shape: reusing resolveDiscordDmCommandAccess keeps the fix in the Discord plugin and avoids a second allowlist parser. It is not merge-ready until live Discord proof and maintainer acceptance of the deliberate fail-closed boundary are in place.
