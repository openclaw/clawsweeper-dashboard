## Summary

Keep open. Current main has the safer main-store read-through behavior for part of the multi-agent symptom, but the token entrypoints and setup-token guidance still do not fully cover the reported flow, and an open closing PR already owns the active implementation discussion.

## What This Changes

The issue requests a reliable Anthropic setup-token onboarding path covering clearer Claude CLI guidance, token persistence validation, and multi-agent auth-profile availability.

## Best Possible Solution

Revise or replace https://github.com/openclaw/openclaw/pull/52912 with a narrow maintainer-approved fix that covers the intended Anthropic setup-token and paste-token surfaces, preserves OAuth isolation policy, and updates user guidance or validation where maintainers want that contract.

## Reproduction Assessment

No high-confidence current-main gateway-startup reproduction is established. Source inspection does show selected-agent-only token writes and sparse Anthropic setup-token prompting, but main-store read-through can make the reported multi-agent failure configuration-dependent.

## Solution Assessment

No, not as currently proposed by the active PR. A narrower token-focused persistence and guidance fix is safer than broad credential fanout, especially for OAuth refresh material.
