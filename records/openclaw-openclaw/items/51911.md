## What This Changes

The issue asks OpenClaw to make Anthropic setup-token onboarding reliably persist credentials, explain the Claude CLI token flow, and propagate manual token auth across configured agents.

## Best Possible Solution

Settle the auth-store contract, then land or reject a focused fix for Anthropic setup-token/manual-token onboarding that covers the chosen propagation model, prompt guidance, validation, and regression proof.

## Reproduction Assessment

Partly: source inspection shows setup-token/plugin auth and paste-token persistence still write one selected agent store. Runtime read-through may prevent the exact gateway failure on current main, and no live Anthropic setup-token/gateway repro was run.

## Solution Assessment

Unclear: copying token profiles to sibling agents is only one plausible fix, and maintainers need to choose between read-through inheritance, static-token-only sync, and stronger validation/prompting before implementation lands.
