## Summary

Keep open: current main and v2026.6.10 still write Anthropic setup-token and paste-token profiles only to the selected agent store, while runtime read-through mitigates the missing-sibling crash but does not settle the setup-token UX, write validation, or static-token propagation contract.

## What This Changes

The issue requests fixes for Anthropic setup-token onboarding prompt clarity, token persistence validation, and multi-agent auth-profile availability.

## Best Possible Solution

Settle the auth-store contract, then land or replace a focused fix that improves setup-token guidance, adds post-write validation, and implements the chosen static-token sharing or read-through behavior while preserving OAuth isolation.

## Reproduction Assessment

Unclear for the exact live gateway crash: current runtime read-through may now prevent it. Source inspection does show that setup-token and paste-token still persist credentials to one selected agent store, and the setup-token guidance remains thin.

## Solution Assessment

Unclear: sibling sync is plausible for portable static tokens, but current main also has main-agent read-through and an explicit OAuth non-copying policy. Maintainers should choose the intended credential-sharing contract before implementation lands.
