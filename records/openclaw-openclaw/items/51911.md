## What This Changes

Issue requests fixes for Anthropic setup-token onboarding documentation, token persistence validation, and multi-agent credential propagation.

## Best Possible Solution

Settle the auth-store contract, then land or replace a focused fix that covers setup-token prompt guidance, post-write validation, and the chosen static-token sharing/read-through model while keeping OAuth copying explicitly opt-in.

Unclear for the exact live gateway crash: current runtime read-through may now prevent it. Source inspection does reproduce the remaining gap that setup-token and paste-token writes still target one selected store while OAuth sibling sync remains a separate explicit path.

## Solution Assessment

Unclear: sibling sync is a plausible repair for portable static tokens, but current main also has main-agent read-through and an OAuth non-copying portability policy. Maintainers should choose the intended credential-sharing contract before implementation lands.
