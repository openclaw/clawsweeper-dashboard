## Summary

Keep open: current main and v2026.6.8 still persist Anthropic setup-token, generic plugin auth, and paste-token credentials to one selected agent store. Runtime read-through mitigates the original multi-agent startup failure, but the setup-token UX, post-write validation, and credential propagation contract still need maintainer security/product review before the linked fork PR or a replacement can resolve this.

## What This Changes

Request is to make Anthropic setup-token/manual token onboarding reliable by clarifying prompts, validating writes, and ensuring configured agents can resolve the credential without manual duplication.

## Best Possible Solution

Settle the auth-store contract, then land or replace a focused fix that covers setup-token prompt guidance, post-write validation, and the chosen static-token sharing/read-through model while preserving OAuth isolation.

## Reproduction Assessment

Unclear for the exact live gateway crash: current runtime read-through may now prevent it. Source inspection does show that setup-token, generic plugin auth, and paste-token still persist credentials to one selected agent store.

## Solution Assessment

Unclear: sibling sync is plausible for portable static tokens, but current main also has main-agent read-through and an explicit OAuth non-copying policy. Maintainers should choose the intended credential-sharing contract before implementation lands.
