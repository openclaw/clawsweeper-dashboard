## Summary

Keep open: current main and v2026.6.10 still write Anthropic setup-token and paste-token profiles to one selected agent store, while runtime read-through only mitigates part of the missing-sibling failure. The setup-token UX, post-write validation, and static-token propagation contract still need maintainer/security review, especially with an open conflicting fork PR and prior same-repo PRs closed unmerged.

## What This Changes

The issue asks for Anthropic setup-token/paste-token onboarding to explain the two-step Claude CLI flow, reliably persist token auth profiles, and make multi-agent credentials available without manual copying.

## Best Possible Solution

Settle the auth-store sharing contract, then land or replace a focused fix that improves setup-token guidance, adds post-write validation, and implements the chosen static-token sharing or inheritance behavior while preserving OAuth isolation.

## Reproduction Assessment

Partly. Source inspection shows setup-token and paste-token still persist to one selected agent store, but I did not establish a live current-main reproduction of the exact gateway startup failure because runtime read-through may now mask it.

## Solution Assessment

Unclear. Sibling sync is plausible for portable static setup tokens, but current main also has main-agent read-through and an explicit OAuth non-copying policy, so maintainers should choose the intended credential-sharing contract before implementation lands.
