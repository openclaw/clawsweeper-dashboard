## Summary

Keep open: current main and v2026.6.6 likely cover part of the multi-agent runtime read-through symptom, but the Anthropic setup-token/paste-token onboarding path still writes a single selected auth store and the UX/validation questions remain unresolved with an open linked fix PR.

## What This Changes

The issue asks for Anthropic setup-token onboarding to guide the Claude token flow, persist the credential reliably, and make it usable across configured agents without manual copying.

## Best Possible Solution

Settle the auth-store contract, then land or reject the linked fix with focused tests and docs that match the chosen design for Anthropic setup-token and multi-agent auth.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection gives a medium-confidence path: token setup writes one selected store while runtime inheritance may now prevent the exact gateway failure, and no live Anthropic setup-token/gateway repro was run.

## Solution Assessment

Is this the best way to solve the issue? Unclear: copying tokens to every sibling agent is only one plausible fix, and maintainers need to decide whether main-store inheritance plus docs/tests, static-token-only sync, or stronger validation is the right long-term contract.
