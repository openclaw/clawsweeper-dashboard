## Summary

Keep this PR open: it addresses a real wide-area DNS path-validation problem and now includes terminal proof, but the shared resolver still collapses invalid configured domains into `null`, leaving background callers to report or behave as if the domain is absent. That remaining compatibility/diagnostic issue should be repaired before merge.

## What This Changes

The PR adds DNS-label validation, DNS zone path containment, CLI invalid-domain handling, and targeted wide-area DNS regression tests.

## Best Possible Solution

Keep the DNS-label/path-containment hardening, but preserve invalid-vs-missing resolver state and update every current caller to emit an explicit invalid-domain diagnostic while still skipping unsafe writes.

## Reproduction Assessment

Yes, source-reproducible. Current main appends a trailing dot and derives zone paths from the domain text without DNS-label validation; I did not run the CLI because this review is read-only.

## Solution Assessment

No, not quite. The path validation and CLI diagnostic are the right direction, but the shared resolver should not erase the difference between missing and invalid configured domains for the remaining callers.
