## Summary

Keep open: the sessions projection fix is directionally useful, but this PR is incomplete for the linked request and still bundles an unrelated Ollama wire-shape change that conflicts with the current OpenClaw and upstream Ollama contract. It also needs contributor-supplied real CLI behavior proof before merge.

## What This Changes

The PR adds optional sessions JSON metadata projection fields and changes native Ollama `think` request parameters from top-level fields to `options.think`.

## Best Possible Solution

Narrow the PR to the sessions JSON projection, map the full existing optional lineage/control field set with focused JSON regression coverage, and leave Ollama thinking behavior to its separate review path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessionsCommand` emits JSON rows built from `toSessionDisplayRow`, and the current display row drops store fields that `SessionEntry` and Gateway rows already carry.

## Solution Assessment

No. Adding optional sessions metadata is the right direction, but this PR is not the best mergeable form because it omits part of the requested field set and mixes in an unrelated Ollama request-shape change.
