## Summary

Keep open: this addresses a real sessions JSON projection gap, but the branch is not mergeable as-is because it omits requested lineage/control fields, carries unrelated Ollama wire-shape changes, and lacks real CLI behavior proof.

## What This Changes

The PR adds optional session metadata fields to `SessionDisplayRow`/`toSessionDisplayRow` and also changes native Ollama `think` request handling and tests to use `options.think`.

## Best Possible Solution

Narrow this PR to the sessions JSON projection, map the full existing lineage/control/lifecycle field set with focused JSON regression coverage, and leave Ollama thinking behavior to its separate review.

## Reproduction Assessment

Yes, via source inspection: current `sessionsCommand` serializes rows produced by `toSessionDisplayRow`, and that projection omits stored subagent metadata already defined on `SessionEntry` and Gateway rows. I did not run the CLI because this review was read-only.

## Solution Assessment

No. The sessions mapping direction is right, but the branch should be sessions-only, include the full requested metadata set, and preserve the current/upstream native Ollama `think` contract.
