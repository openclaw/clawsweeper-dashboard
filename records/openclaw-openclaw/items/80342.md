Keep open: the sessions projection fix is useful, but this branch is not mergeable as-is because it omits part of the requested lineage/control metadata, bundles unrelated Ollama request-shape changes, and still lacks real after-fix CLI proof. A prior Codex review flagged the same blockers, and source/related-thread checks still support that verdict.

## What This Changes

The branch adds optional session metadata fields to the sessions display row JSON projection and also moves native Ollama `think` request values into `options.think` with test updates.

## Best Possible Solution

Narrow the branch to the sessions JSON projection, include the full existing optional lineage/control field set with focused JSON regression coverage, and leave the Ollama API-contract work to the separate Ollama review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row containing subagent metadata is reduced through `toSessionDisplayRow` before JSON serialization, so omitted fields cannot appear in `openclaw sessions --json`.

## Solution Assessment

No. The sessions projection direction is right, but this patch is incomplete and mixed with unrelated Ollama behavior; the safer solution is a sessions-only projection/test change plus separate Ollama contract review.
