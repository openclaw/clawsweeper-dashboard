## Summary

Keep open: the sessions projection fix is valid, but this PR is not ready because it omits part of the requested lineage metadata, carries an unrelated Ollama wire-shape change, and the supplied proof is unit-test output rather than real CLI behavior.

## What This Changes

The branch adds optional session metadata fields to `SessionDisplayRow`/`toSessionDisplayRow` and also moves Ollama `think` request fields into `options.think` with test updates.

## Best Possible Solution

Narrow the PR to the sessions JSON projection, map the full existing lineage/control field set with focused JSON regression coverage, and leave Ollama thinking behavior to its separate validated review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessionsCommand` writes JSON rows built by `toSessionDisplayRow`, which drops stored subagent metadata that `SessionEntry` and Gateway rows already carry.

## Solution Assessment

No. The sessions direction is the right narrow fix, but this branch is incomplete and mixed with unrelated Ollama behavior; the maintainable solution is a sessions-only projection/test change.
