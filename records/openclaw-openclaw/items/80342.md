## Summary

Keep open: the sessions metadata gap is real, but this PR is not mergeable as-is because it omits part of the requested lineage/control set, bundles an unrelated Ollama request-shape change, and still has only test-based behavior proof.

## What This Changes

The branch adds optional subagent/lifecycle fields to the sessions JSON display row and also moves native Ollama `think` values into `options.think` with test updates.

## Best Possible Solution

Narrow the PR to the sessions JSON projection, map the full existing lineage/control field set with focused JSON regression coverage, and leave Ollama thinking behavior to its separate review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessionsCommand` writes JSON rows built by `toSessionDisplayRow`, which drops stored subagent metadata that `SessionEntry` and Gateway rows already carry.

## Solution Assessment

No. The sessions direction is the right narrow fix, but this branch is incomplete and mixed with unrelated Ollama behavior; the maintainable solution is a sessions-only projection/test change.
