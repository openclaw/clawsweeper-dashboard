## Summary

Keep open: the sessions JSON projection gap is real, but this branch is incomplete and still bundles an unrelated Ollama wire-shape change that should not ride along with the sessions fix.

## What This Changes

This PR adds optional session metadata fields to `SessionDisplayRow`/`toSessionDisplayRow` and also changes native Ollama `think` request placement with test updates.

## Best Possible Solution

Narrow the branch to the sessions JSON projection, map the full existing lineage/control/lifecycle field set with focused JSON regression coverage, and leave Ollama thinking behavior to its separate review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessionsCommand` serializes rows built by `toSessionDisplayRow`, which drops metadata already present on `SessionEntry` and gateway rows.

## Solution Assessment

No. The sessions projection direction is right, but this patch is incomplete and mixed with unrelated Ollama request-shaping behavior; the maintainable fix is sessions-only plus the missing fields and regression coverage.
