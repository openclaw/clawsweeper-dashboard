## Best Possible Solution

Land this focused Telegram rotation only after redacted live Telegram proof shows the preamble remains visible and the post-tool answer starts a new message.

## Solution Assessment

Yes, this appears to be the right code boundary: Telegram dispatch owns block/tool draft rotation, while the deliverer and draft stream only handle lane delivery and send/edit mechanics. The missing live Telegram proof prevents a merge-ready verdict, not the code shape itself.
