## Summary

Keep open: the patch is focused and has strong live proof for the post-tool partial-resume case, but source review found the deferred rotation is only consumed by post-tool partial ingestion, so a supported final-only post-tool answer can still overwrite the in-flight preamble.

## What This Changes

The PR adds Telegram partial-streaming logic, regression tests, and an opt-in live Bot API proof harness to preserve an in-flight streamed answer preamble across a tool boundary.

## Best Possible Solution

Keep the Telegram-channel fix, but consume the deferred tool-boundary rotation in every answer-resume path, including final-only delivery, with regression coverage for partial, final-only, below-debounce, and multi-boundary continuations.

## Reproduction Assessment

Yes for source-level review: current main lacks the in-flight tool-boundary rotation, and the PR proof reproduces the partial-resume overwrite with live Bot API output. I did not run live Telegram in this read-only review, and the final-only gap is source-reproducible from the callback and delivery paths.

## Solution Assessment

No: deferring rotation to classify fresh resumes is the right layer, but the current patch only consumes that deferred state on post-tool partials. The narrower complete fix is to run the same classification before final answer delivery as well.
