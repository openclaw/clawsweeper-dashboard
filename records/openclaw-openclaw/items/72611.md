## Summary

Keep open: current main still lacks an operator-facing Dreaming session/cron exclusion config, and the only PR that attempted the full filter contract was closed unmerged. Main has partial cron archive mitigation, so this is no longer a simple reproduction of the original production leak, but the requested public filter path remains real and unresolved.

## What This Changes

The issue requests reliable Dreaming exclusion of cron/isolated session transcripts plus operator-facing session/cron filter configuration.

## Best Possible Solution

Approve a memory-core Dreaming filter contract, then land a focused replacement for the useful parts of https://github.com/openclaw/openclaw/pull/72913 with schema, resolver, docs, and regression coverage aligned to current cron archive filtering.

## Solution Assessment

Yes in direction, but not as an automated cleanup fix yet. A public memory-core session filter contract with focused ingestion tests is the narrow maintainable path; the exact keys and default policy need maintainer approval first.
