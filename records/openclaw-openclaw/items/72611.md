## Summary

Keep open: current main has partial built-in cron/archive skipping, but it still lacks the operator-facing Dreaming session/cron exclusion contract requested here. The prior implementation PR for those filters was closed unmerged after review, and the related open denoise/promotion work does not supersede this config/API decision.

## What This Changes

The issue asks for Dreaming to keep isolated cron transcripts out of the session corpus and to add configurable exclusions by cron job, session key prefix, agent, or source path.

## Best Possible Solution

Approve a narrow Dreaming session-ingestion filter contract, then implement schema, resolver, docs, ingestion predicates, and focused regression coverage while preserving the existing cron/archive hardening.

## Reproduction Assessment
