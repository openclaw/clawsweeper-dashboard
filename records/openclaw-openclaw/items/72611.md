## Summary

Keep open. Current main still lacks operator-facing Dreaming session/cron exclusion config, and the ingestion path still has no operator predicate beyond path-set classification and strict cron-run key matching. The paired implementation PR #72913 is open and unmerged, so this issue should stay open for that review rather than be closed or sent to a competing repair lane.

## What This Changes

Add configurable Dreaming session/cron exclusions and make cron transcript skipping reliable for isolated and rotated cron artifacts.

## Best Possible Solution

Ship one memory-core fix that exposes documented Dreaming session-filter config, applies it during session corpus ingestion, preserves plugin boundaries, and covers strict cron-run keys, base cron keys, and rotated transcript artifacts in focused tests.

## Reproduction Assessment

Yes for the remaining actionable config gap: current main source, manifest, resolver, and docs show no Dreaming session/cron exclusion config. The exact production corpus contamination path was not live-replayed, and current main partially mitigates deleted/checkpoint artifacts, so the transcript-leak side still needs regression coverage around the reported cron shapes.

## Solution Assessment

Yes: keeping this open while #72913 is reviewed is the best current handling. The narrow maintainable solution belongs in memory-core config/manifest/docs and memory-host transcript classification helpers, not in a competing ClawSweeper repair while a paired PR is open.
