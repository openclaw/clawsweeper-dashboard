## Summary

Keep open. Current main still lacks the requested operator-facing Dreaming session/cron exclusion config, and the explicitly paired implementation PR #72913 is open and unmerged, so this issue should remain open for that PR review rather than be closed or routed to a competing repair lane.

## What This Changes

The issue requests configurable Dreaming transcript-ingestion exclusions for cron jobs, agents, session-key prefixes, and source paths, plus more reliable skipping of isolated or rotated cron transcripts before they enter the session corpus.

## Best Possible Solution

Review and land #72913, or replace it with an equivalent narrow fix that adds documented Dreaming session filters, preserves plugin boundaries, keeps cron transcript exclusion reliable for live and archived artifacts, and includes regression coverage for strict cron-run keys, base cron keys, and archive handling.

## Reproduction Assessment

Yes for the remaining actionable gap: current main source, manifest, tests, and docs show no operator-facing Dreaming session/cron exclusion config. The exact production corpus leak was not live-replayed, and main already contains partial archive and direct cron-prompt defenses.

## Solution Assessment
