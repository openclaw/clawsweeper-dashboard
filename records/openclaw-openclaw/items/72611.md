## Summary

Keep open: current main has partial cron transcript hardening, but Dreaming still has no operator-facing session/cron exclusion config and the closed implementation PR never merged.

## What This Changes

Request configurable Dreaming exclusions for cron jobs, session-key prefixes, agents, or source paths so high-volume isolated cron transcripts stay out of the session corpus.

## Best Possible Solution

Approve a memory-core Dreaming ingestion filter contract, then land a focused implementation with schema, resolver, docs, and regression coverage while preserving the existing cron archive skip behavior.

## Reproduction Assessment

Yes for the remaining config gap and source-level leak boundary: current main has no Dreaming session-filter schema, resolver field, docs entry, or ingestion predicate. I did not replay the original 2026.4.24 production deployment end-to-end.

## Solution Assessment

Yes for keeping it open, but no for an automatic fix lane. The durable solution needs a maintainer-approved public filter contract rather than a one-off hard-coded cron rule.
