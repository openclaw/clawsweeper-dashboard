## Summary

Keep open. Current main partially mitigates direct cron archive leakage, but it still has no operator-facing Dreaming session/cron exclusion config, and the related open/closed work only covers narrower archive or subagent-classification pieces rather than the full requested policy.

## What This Changes

The issue requests reliable exclusion of isolated cron transcripts from Dreaming session-corpus ingestion plus configurable operator filters for sessions, cron jobs, agents, or source paths.

## Best Possible Solution

Dreaming should get a maintainer-approved ingestion policy: either fix the remaining cron/archive/subagent classification gaps without a new knob, or add a narrow documented filter config with focused tests and upgrade guidance.

## Reproduction Assessment

Yes at source level for the missing config path: memory-core's manifest has Dreaming phase/storage/execution fields but no session or cron exclusion block. The exact original cron leak is only partly source-reproducible because current buildSessionEntry now detects direct cron prompts in usage-counted archives.

## Solution Assessment

Unclear as an implementation direction. A new filter knob may be useful, but OpenClaw's config-surface policy means maintainers should first decide whether canonical cron/archive classification fixes are enough or whether an operator-facing filter contract should be added.
