## Summary

Keep open: current main partially filters isolated and rotated cron transcripts, but it still lacks the operator-facing Dreaming session/cron exclusion config requested here. The linked implementation at https://github.com/openclaw/openclaw/pull/72913 was closed unmerged, and targeted GitHub search found no newer PR owning the exact `dreaming.sessionFilter` surface.

## What This Changes

Add configurable Dreaming exclusions so operators can keep cron jobs, session-key prefixes, agents, or source paths out of session-corpus ingestion.

## Best Possible Solution

Approve a narrow memory-core Dreaming session-filter contract, then land schema, resolver, docs, ingestion predicate, and focused regression tests on top of the current built-in cron filtering.
