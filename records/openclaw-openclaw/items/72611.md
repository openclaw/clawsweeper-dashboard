## Summary

Keep open: current main has partial cron/archive filtering, but it still lacks the requested operator-facing Dreaming session/cron exclusion config, and the linked closing PR #72913 is open and unmerged.

## What This Changes

Request configurable Dreaming transcript exclusions for cron jobs, session-key prefixes, agents, or source paths, plus more reliable exclusion of isolated cron transcripts from the session corpus.

## Best Possible Solution

Review and land #72913 or an equivalent focused memory-core change that adds documented `dreaming.sessionFilter` controls and robust cron transcript attribution for rotated and non-run-key shapes.

## Reproduction Assessment

Yes for the remaining actionable gap: current main has no `dreaming.sessionFilter` schema or resolved config, and cron classification still depends on a strict run-key shape. The original production corpus leak was not replayed locally.

## Solution Assessment

Yes for triage: keeping this issue open while #72913 or an equivalent fix is reviewed is the narrowest maintainable path. Closing should wait until the linked implementation lands or maintainers choose a different filter contract.
