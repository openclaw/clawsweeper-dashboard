## Summary

## What This Changes

The PR adds an `inputProvenance` envelope to the native `spawnSubagentDirect` child `agent` call and adds a workspace test asserting that payload.

## Best Possible Solution

## Reproduction Assessment

Yes, by source inspection: current main lacks `inputProvenance` on the native child `agent` call, and the ACP sibling path also lacks it. I did not run a live runtime reproduction in this read-only review.

## Solution Assessment

No, this is the right layer for the native path but not the best complete fix because `sessions_spawn({ runtime: "acp" })` still misses the same provenance contract. The safer solution is one shared provenance shape applied to both runtime dispatch paths.
