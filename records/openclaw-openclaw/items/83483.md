## Best Possible Solution

Land this implementation only after maintainers accept the alias/discriminator contract and its compatibility/security-boundary impact, preserving the added docs and regression coverage.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the public hook contract change. The alias plus explicit discriminator is safer than an alias-only path because broad policies can cover code mode while mechanism-specific policies can exclude `toolKind: "code_mode_exec"`.
