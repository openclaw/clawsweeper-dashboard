## Best Possible Solution

Land this shape only after maintainers accept the alias/discriminator contract, keeping the added coverage for marked code-mode exec, plain exec, trusted rewrites, divergent aliases, and after-hook params.

## Solution Assessment

Yes if maintainers want shared exec-shaped policy semantics. The alias plus host-authoritative discriminator is a maintainable answer to the OpenClaw `exec` name collision, but the public security contract needs explicit maintainer signoff.
