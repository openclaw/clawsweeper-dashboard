## Summary

## What This Changes

The PR adds code-mode exec `command` aliasing, hook-visible tool identity metadata, trusted-policy normalization, docs, and regression coverage for before/after hook payloads.

## Best Possible Solution

Land this shape only after maintainers accept the alias/discriminator contract, keeping the added coverage for marked code-mode exec, plain exec, trusted rewrites, divergent aliases, and after-hook params.

## Reproduction Assessment

Yes at source level. Current main's code-mode exec reads only `params.code`, and hook events do not carry `toolKind`/`toolInputKind`, so command-only exec-shaped policies cannot observe code-mode source through the proposed alias path.

## Solution Assessment

Unclear until maintainers accept the public contract. The implementation is bounded and well covered if OpenClaw wants shared exec-shaped policy semantics; the safer alternative is to keep code-mode code-only or use a distinct tool identity without command aliasing.
