## What This Changes

The PR catches transcript-path resolution failures during `sessions cleanup --fix-missing` and adds integration coverage for dry-run and applied pruning of a key-shaped `sessionId`.

## Best Possible Solution

Land the narrow cleanup recovery after normal maintainer and CI gates, while keeping transcript ID validation strict outside the explicit `--fix-missing` repair path.

## Reproduction Assessment

Yes, at source level. Current main passes a stored `sessionId` such as `agent:main:main` into strict transcript path resolution during `--fix-missing`, so validation throws before cleanup can continue.

## Solution Assessment

Yes. The PR keeps the shared resolver and validator strict, and only treats cleanup-time path resolution failure as a missing or corrupt store row under the explicit repair path.
