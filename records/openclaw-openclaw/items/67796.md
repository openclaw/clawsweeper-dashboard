## Summary

Keep open: this is a plausible Mattermost bug-fix PR, but the branch still has a blank-media fallback bug, lacks real after-fix Mattermost proof, and is only a partial piece of the open Mattermost message-tool issue cluster.

## What This Changes

The PR changes Mattermost `handleAction` send handling to derive `mediaUrl` from `params.media`, then `params.filePath`, then `params.path`.

## Best Possible Solution

Land a coordinated Mattermost handler fix that normalizes media aliases independently, preserves reply/thread precedence, forwards scoped media access, and adds focused `handleAction` regression tests.

## Reproduction Assessment

Yes, at source level. Current main still has a direct Mattermost handler path that reads only `params.media`, and the PR head still fails a blank-media plus valid-alias case by inspection.

## Solution Assessment

No. The handler is the right fix location, but this patch should use normalized optional-string fallback for each candidate, add alias/blank-media coverage, and avoid closing the broader Mattermost issue before the remaining pieces land.
