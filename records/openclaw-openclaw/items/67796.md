## What This Changes

The PR changes Mattermost message-tool send handling to derive mediaUrl from params.media, then params.filePath, then params.path.

## Best Possible Solution

Land a corrected Mattermost handler fix that normalizes media, filePath, and path independently, adds focused handleAction regression tests, and keeps the broader Mattermost threadId/mediaLocalRoots cluster coordinated.

## Reproduction Assessment

Yes. Source inspection shows current main reads only params.media in Mattermost handleAction, and the PR head still fails a media blank-plus-filePath/path case because the fallback is guarded by typeof params.media.

## Solution Assessment

No. Resolving aliases in the Mattermost action adapter is the right location, but this patch should normalize each candidate independently and add alias plus blank-fallthrough coverage before merge.
