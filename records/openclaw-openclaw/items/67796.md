## Summary

Keep open: this targets a real Mattermost message-tool alias bug, but the PR is not safe to merge as-is because blank `media` short-circuits valid aliases, real Mattermost proof is missing, and the linked broader Mattermost issue still has companion fixes open.

## What This Changes

The PR changes `extensions/mattermost/src/channel.ts` so Mattermost `handleAction` derives `mediaUrl` from `media`, then `filePath`, then `path`.

## Best Possible Solution

Land a corrected Mattermost alias fix that normalizes each media candidate independently, adds focused `handleAction` tests for `media`, `filePath`, `path`, and blank-media fallthrough, and coordinates issue-closing text with the remaining Mattermost thread/media-root work.

## Reproduction Assessment

Yes, at source level. Current main's Mattermost `handleAction` send path only reads `params.media`, and the PR head still fails `media: " "` plus a valid `filePath` or `path` by inspection.

## Solution Assessment

No. The handler is the right place, but this patch should use normalized optional-string fallback for each candidate, add regression coverage, and avoid closing the broader Mattermost issue before the companion fixes land.

## Review Findings

Overall correctness: patch is incorrect
