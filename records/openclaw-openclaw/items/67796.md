## Best Possible Solution

Land a corrected Mattermost-owned handler fix that normalizes each media candidate independently, adds focused `handleAction` tests, and keeps https://github.com/openclaw/openclaw/issues/67784 coordinated with the remaining Mattermost PRs.

## Reproduction Assessment

Yes, at source level. Current main ignores `filePath`/`path` in Mattermost `handleAction`, and the PR head still fails a payload with `media: " "` plus a valid alias by inspection.

## Solution Assessment

No, not as-is. The handler is the right surface, but the patch should use normalized optional-string fallback for each candidate and add direct regression coverage before merge.
