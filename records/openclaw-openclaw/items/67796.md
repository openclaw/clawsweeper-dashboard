## Summary

Keep open: current main still lacks direct Mattermost handleAction alias handling, and the PR's fallback has a blank-media short-circuit plus no real behavior proof before merge.

## What This Changes

The PR changes Mattermost message-tool send handling to derive mediaUrl from params.media, then params.filePath, then params.path.

## Best Possible Solution

Land a corrected Mattermost action-adapter fix that resolves media/path/filePath with normalized optional strings, adds handleAction tests for both aliases and blank fallthrough, and keeps the broader Mattermost threadId/mediaLocalRoots issue coordinated with its related PRs.

## Reproduction Assessment

Yes. A focused source-level unit path can call Mattermost handleAction with action send, media as whitespace, and a valid filePath; the PR head would still pass mediaUrl as undefined, while current main ignores filePath/path in direct handleAction.

## Solution Assessment

No. The fix belongs in the Mattermost action adapter, but this patch should normalize each candidate independently before falling through and add focused regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:
