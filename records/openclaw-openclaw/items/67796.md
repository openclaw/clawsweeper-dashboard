## Summary

Keep this PR open. Current main still has the Mattermost alias gap, and the PR is aimed at the right handler, but the proposed nested fallback still drops a valid filePath/path when media is present as an empty or whitespace string and the branch has no focused regression coverage for the new aliases.

## What This Changes

The PR changes Mattermost message-tool send handling to derive mediaUrl from params.media, then params.filePath, then params.path.

## Best Possible Solution

Land a corrected Mattermost alias fix: normalize each candidate independently, preserve the intended media -> filePath -> path precedence, add focused handleAction tests for filePath, path, and blank-media fallthrough, and keep the separate #67784 threadId/mediaLocalRoots work coordinated with #52120 and #58439.

## Reproduction Assessment

Yes. A focused unit test can call Mattermost handleAction with action: "send", media: "   ", and a valid filePath; the PR head would forward mediaUrl as undefined, while current main ignores filePath/path entirely.

## Solution Assessment

No, not as currently patched. The direction is narrow and maintainable, but the implementation should use normalized optional-string fallback semantics for each candidate and add regression tests before merge.
