## Summary

Keep open: this is a coherent WhatsApp feature request, not an implemented or duplicate cleanup item. Current main and the latest shipped tag still expose list responses only as already-clicked user selections, while list prompts, row-id structured context, and a `list-reply` action are not present.

## What This Changes

Request first-class WhatsApp Web support for inbound interactive list prompts and outbound row-id list replies.

## Best Possible Solution

Add the capability inside the WhatsApp plugin boundary: parse list prompts into readable body plus untrusted structured context, make the content gate wrapper-aware for prompts, and expose a validated row-id reply action through the established message-action/config contracts.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing-behavior bug. Source inspection gives high confidence that current main lacks list prompt extraction and a list-reply send path.

## Solution Assessment

Yes in direction, but the public API shape needs maintainer judgment. The maintainable path is a WhatsApp plugin implementation plus small shared message-action/config updates, not raw Baileys payload exposure or plain-text guessing.
