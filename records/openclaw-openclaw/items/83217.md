## Summary

Keep open. Current main does not expose inbound WhatsApp `listMessage` rows as body or structured context, and the WhatsApp message action surface only advertises reactions and polls; the related merged content-gate work at https://github.com/openclaw/openclaw/pull/73823 is useful precedent but only covers list selections, not list prompts or outbound list replies.

## What This Changes

Request to add WhatsApp Web support for inbound interactive list prompts with row ids and an outbound list-reply action using `selectedRowId`.

## Best Possible Solution

Implement this inside the WhatsApp plugin with typed list extraction, untrusted structured context, a channel-owned list-reply action, focused tests, docs, and live payload validation.

Implement WhatsApp interactive list support for https://github.com/openclaw/openclaw/issues/83217. Keep the change inside the WhatsApp plugin: parse direct and wrapper-contained `listMessage` prompts into readable inbound body text plus an untrusted structured context entry labeled `WhatsApp list`; preserve existing text/media/contact/location behavior; make list prompts pass the inbound content gate; add a channel-owned `list-reply` message action that validates a non-empty row id and sends the Baileys `listResponseMessage`/`singleSelectReply.selectedRowId` payload with optional quote context for the original list message.
