## Summary

Keep open: the request is coherent and current main still lacks inbound WhatsApp listMessage prompt extraction, untrusted structured list context, and an outbound row-id list-reply action; the related content-gate work only covers interactive responses after a selection, not exposing list prompts or sending list selections.

## What This Changes

Request first-class WhatsApp interactive list support so agents can read list rows/row ids and reply with a selected row id.

## Best Possible Solution

Implement this inside the WhatsApp plugin boundary: parse listMessage prompts into readable body plus untrusted structured context, treat them as inbound content, add a validated row-id list-reply action through existing message-action contracts, and cover direct/wrapped inbound plus outbound send behavior with focused WhatsApp tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing-behavior bug. Source inspection gives high confidence that current main lacks the requested listMessage prompt extraction and outbound list-reply action paths.

## Solution Assessment

Yes in direction, but the exact public action shape needs maintainer judgment. The maintainable path is a WhatsApp plugin implementation plus narrow shared message-action contract updates, not raw Baileys payload exposure or plain-text guessing.
