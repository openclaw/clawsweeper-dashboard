## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Gate automatic `.backupignore` loading:** `src/infra/backup-exclude.ts:190-194`
  - body: This loads `stateDir/.backupignore` on every `backup create`, even when no exclude flag is passed. A pre-existing local file would silently omit restore contents from unchanged backup commands, so this needs to be explicit opt-in or a maintainer-approved default with docs and upgrade coverage.
  - confidence: 0.9
- **[P1] Compose excludes with the current archive writer:** `src/infra/backup-create.ts:469-475`
  - body: The exclude path archives `result.assets` through a separate tar/gzip flow. Current main's writer snapshots SQLite, skips volatile files, retries tar EOF races, chooses a safe temp root, disables hardlink caching, and remaps snapshot sources; keep those protections and add filtering inside that path instead of replacing it.
  - confidence: 0.92
- **[P2] Apply excludes outside the state directory:** `src/infra/backup-exclude.ts:362-364`
  - body: Returning `true` for every path outside `baseDir` means external workspaces and custom config/credential assets are archived in full even when user patterns match. A warning still leaves security-sensitive exclude behavior incomplete; use per-asset bases or narrow the documented contract.
  - confidence: 0.88
- **[P2] Preserve ordered duplicate ignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Gitignore rules are order-sensitive, but this drops later duplicate patterns before handing them to `ignore`. A sequence with negation followed by a duplicate rule can change meaning after dedupe, so preserve the full ordered rule list and track source metadata separately.
  - confidence: 0.84
- **[P2] Avoid full archive scans during create:** `src/infra/backup-create.ts:528-529`
  - body: This reads the completed `.tar.gz` and stores every entry path before publishing every archive. For the multi-GB backups this PR targets, that adds a second full pass plus O(entry count) memory; stream validation or keep the full scan behind `--verify`.
  - confidence: 0.82
