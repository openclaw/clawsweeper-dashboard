## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Run exclude archives through the current backup pipeline:** `src/infra/backup-create.ts:469-476`
  - body: With excludes enabled, this branch writes tar archives through a separate two-step path. Current main's backup path snapshots SQLite databases, filters volatile state, skips extension `node_modules`, retries tar EOF races, and remaps snapshot paths; the exclude path needs to compose with that pipeline or it can regress backup integrity and availability for the same users this feature targets.
  - confidence: 0.91
- **[P1] Apply excludes to every backup asset root:** `src/infra/backup-create.ts:364-368`
  - body: The filter is built with only `canonicalStateDir` as its base, and the PR now warns that external workspaces and custom config locations bypass excludes. A warning is not enough for patterns like `.env*`, `*.pem`, or `logs/`: those assets are still archived in full even though they are part of the backup plan and the feature promises gitignore-like exclusion.
  - confidence: 0.93
- **[P1] Keep hardlink metadata for backup verification:** `src/commands/backup-shared.ts:49-56`
  - body: Moving `listArchiveEntries` to return only strings drops each tar entry's `type` and `linkpath`. Current main uses that metadata to reject unsafe or missing hardlink targets during `backup verify`; this PR's verifier can no longer perform that safety check.
  - confidence: 0.88
- **[P2] Preserve ordered gitignore rule semantics:** `src/infra/backup-exclude.ts:225-233`
  - body: This deduplication removes later duplicate patterns before passing rules to `ignore`, but gitignore rules are order-sensitive. A sequence like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; dropping the final duplicate changes the user's declared result.
  - confidence: 0.86
- **[P2] Do not rescan every archive into memory by default:** `src/infra/backup-create.ts:528-535`
  - body: This unconditional integrity check reads every path from the newly created `.tar.gz` into an array even when the user did not request `--verify`. On the large backup cases motivating the PR, that adds a second full archive pass and O(entry count) memory pressure to the default create path.
  - confidence: 0.82
