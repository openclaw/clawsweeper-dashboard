## Summary

Keep open. Current main still lacks the requested exclude-pattern backup CLI, so the PR is not obsolete, but the PR head is not merge-ready because it lacks real behavior proof and still has blocking backup correctness/security findings.

## What This Changes

The PR adds backup create exclude-pattern flags, smart defaults, protected-path guards, `.backupignore` handling, manifest exclusion stats, and backup create/verify coverage.

## Best Possible Solution

Land a rebased exclusion implementation that composes with current main's retry and volatile-file filtering, applies patterns per archived root, preserves ordered gitignore semantics, avoids unconditional archive readback in normal create, documents the final CLI contract, and includes redacted real-run proof.

## Reproduction Assessment

Yes, at source level for the review findings: PR-head code shows the external-asset bypass, duplicate-rule collapse, unconditional archive readback, and raw tar create paths that need current-main hardening. I did not run a live backup because this was a read-only review.

## Solution Assessment

No. Native backup excludes are the right direction, but this branch needs a rebase plus narrower composition with the current backup pipeline and a resolved external-root filtering design before it is the safest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve current backup hardening on exclude archives:** `src/infra/backup-create.ts:469-475`
  - body: This exclude path invokes `tar.c` directly, but current main now wraps archive writes with tar EOF retries and volatile-state filtering from the merged backup hardening. Rebase this branch and route the exclude path through the same retry/filter behavior, or large/live backups can regress to the EOF/tmp-file failures and archive live queue/log/session files again.
  - confidence: 0.87
- **[P2] Apply excludes to every archived asset root:** `src/infra/backup-create.ts:378-386`
  - body: This warning still leaves external workspaces, config paths, or credential roots outside `stateDir` archived in full. Since `resolveBackupPlanFromDisk()` can include those roots, user rules like `.env*`, `*.pem`, or `logs/` will not protect all content the backup writes.
  - confidence: 0.91
- **[P2] Preserve duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore precedence because negations are order-sensitive. A list such as `*.log`, `!important.log`, `*.log` should end with `important.log` ignored, but this de-duplication removes the final rule and flips the result.
  - confidence: 0.9
- **[P2] Avoid full archive rescans during create:** `src/infra/backup-create.ts:528-535`
  - body: `listArchiveEntries()` materializes every tar entry before publishing the archive even when the user did not request `--verify`. For the multi-GB backups this feature targets, that adds another full archive pass and memory proportional to entry count to normal create.
  - confidence: 0.84
