- **[P1] Run excludes through the current backup pipeline:** `src/infra/backup-create.ts:462-476`
  - body: The exclude-enabled branch creates a separate archive flow over `result.assets`, so it bypasses current main's SQLite snapshot staging, volatile-file filter, temp-root selection, retry wrapper, hardlink cache, and source-path remaps. Any backup using `--exclude` or `--smart-exclude` can therefore produce less safe or less reliable archives than the normal path; rebase this into the current single `tarFilter`/`writeTarArchiveWithRetry` pipeline instead of forking archive creation.
  - confidence: 0.92
- **[P1] Preserve hardlink target validation:** `src/commands/backup-shared.ts:49-58`
  - body: This shared listing helper returns only path strings, but current main's verifier keeps `path`, `linkpath`, and `type` so it can reject hardlinks that point outside the archive root or to missing entries. Replacing verification with string-only entries weakens archive safety for backups that contain hardlinks.
  - confidence: 0.86
- **[P2] Keep ordered duplicate ignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore semantics because negations are order-sensitive: `*.log`, `!important.log`, `*.log` should end with `important.log` ignored, but this dedupe removes the final rule and flips the result. Keep the rule list ordered as supplied and track source metadata without collapsing duplicates.
  - confidence: 0.82
- **[P2] Document the new backup flags and output contract:** `src/cli/program/register.backup.ts:30-52`
  - body: The CLI now exposes six public flags and a new `excludedStats` manifest/JSON shape, while the linked public backup docs still show only the existing backup create options.
