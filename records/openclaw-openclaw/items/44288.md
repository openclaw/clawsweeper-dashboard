## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Apply excludes to every backed-up asset:** `src/infra/backup-create.ts:364-368`
  - body: The filter is built once with `canonicalStateDir`, and the later warning still lets external workspaces or custom config locations bypass the requested exclude rules. For example, `--exclude '.env*'` will not filter a workspace outside the state directory, so sensitive or bulky files are archived in full despite the user's pattern. Build filters per asset/root or fail until the semantics are safe.
  - confidence: 0.88
- **[P2] Avoid rescanning the whole archive on every create:** `src/infra/backup-create.ts:528-529`
  - body: This always calls `listArchiveEntries()` and stores every entry before publishing, even when the user did not ask for `--verify`. Large installations are the path this PR is trying to help, so adding a second full archive pass plus O(entry count) memory can reintroduce stalls or memory pressure; gate it behind verification or stream only the checks you need.
  - confidence: 0.81
