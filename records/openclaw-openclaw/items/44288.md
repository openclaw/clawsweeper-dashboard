## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate automatic `.backupignore` loading:** `src/infra/backup-exclude.ts:187-194`
  - body: Loading `stateDir/.backupignore` by default changes what existing backups contain even when the user did not pass an exclude flag. That can silently make an upgrade produce incomplete archives from a local file, so this needs to be opt-in or explicitly approved and documented as a backup-content contract.
  - confidence: 0.91
- **[P1] Rebase excludes onto the current archive writer:** `src/infra/backup-create.ts:469-476`
  - body: The exclude path uses a separate tar/gzip writer over `result.assets`, so it does not preserve current main's backup writer hardening such as live SQLite snapshots, volatile-file filtering, extension dependency skips, tar retry behavior, and cleanup safety. The feature needs to integrate with the current writer instead of reviving the older path.
  - confidence: 0.88
- **[P2] Apply excludes to external backup assets:** `src/infra/backup-exclude.ts:358-364`
  - body: Returning `true` for entries outside `baseDir` means `--exclude` and `.backupignore` rules do not filter valid backed-up assets that live outside `stateDir`, such as workspace or custom config roots. Users can still archive sensitive or large external files despite explicit patterns.
  - confidence: 0.86
- **[P2] Preserve ordered duplicate ignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns before passing them to `ignore` changes gitignore semantics because later duplicate rules can intentionally override an earlier negation. This can include or exclude files differently from the user's declared order.
  - confidence: 0.78
- **[P2] Avoid scanning the full archive during create:** `src/infra/backup-create.ts:525-541`
  - body: The post-create integrity check reads every tar entry into memory on every exclude backup. For large installations, this adds a second full archive pass and can worsen the same memory/runtime pressure the linked backup issues are trying to reduce.
  - confidence: 0.8
