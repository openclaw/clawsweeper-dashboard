## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Apply excludes before archiving external assets:** `src/infra/backup-create.ts:386`
  - body: This warning still proceeds to archive each external workspace, custom config, or external credentials asset in full. Explicit excludes such as `*.env`, `*.key`, or smart defaults do not apply outside `stateDir`, so large or sensitive files can remain in the archive despite the user's rule.
  - confidence: 0.92
- **[P2] Preserve ordered duplicate exclude rules:** `src/infra/backup-exclude.ts:225-233`
  - body: `resolveExcludePatterns` drops later duplicate patterns before `ignore` evaluates them, but gitignore rules are order-sensitive after negations. A list such as `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; dropping the final rule flips that result.
  - confidence: 0.9
- **[P2] Avoid rescanning every created archive:** `src/infra/backup-create.ts:528-529`
  - body: The post-create cross-check now lists the full tarball and stores every entry after every backup, including the no-exclude path that was supposed to stay unchanged. Large state directories get a second full archive pass plus O(entry count) memory, which can worsen the same large-installation performance problem this PR targets.
  - confidence: 0.86
- **[P3] Document the backup exclude CLI surface:** `src/cli/program/register.backup.ts:30-52`
  - body: This adds user-facing flags and manifest/JSON behavior, but the public backup reference still lists only the old create forms and size guidance. The docs should cover the new exclude options, protected-path behavior, manifest stats, and any accepted external-asset limitation before this CLI surface ships.
  - confidence: 0.82
