## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-exclude.ts:362-363`
  - body: `createBackupArchive()` builds one filter rooted at `stateDir`, and this guard then includes any entry outside that base. External workspaces, custom config paths, or external credentials can therefore be archived in full even when the user's pattern would match them; the warning in the caller does not make the backup obey the exclude policy.
  - confidence: 0.92
- **[P2] Preserve ordered duplicate gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: This deduplication removes later duplicate patterns before `ignore` evaluates them. Gitignore rules are order-sensitive around negations, so a sequence like `*.log`, `!important.log`, `*.log` changes meaning when the final duplicate is dropped, causing backups to include or exclude files differently from the declared rule order.
  - confidence: 0.9
- **[P2] Avoid mandatory full archive rescans on create:** `src/infra/backup-create.ts:528-529`
  - body: Every successful backup now calls `listArchiveEntries()` before publish, and that helper stores every tar entry path in an array. On large installs this adds a second full gzip/tar pass plus O(entry count) memory to the default create path, which can recreate the stalls or memory pressure this feature is trying to mitigate.
  - confidence: 0.84
