Full review comments:

- **[P1] Preserve the current backup writer safeguards:** `src/infra/backup-create.ts:469-475`
  - body: This direct exclude tar path bypasses current main's retrying writer, volatile-state filter, temp-root placement, and installed-plugin dependency pruning. Merging it can reintroduce live-file EOF stalls, duplicate manifest temp-root failures, and oversized plugin dependency archives.
  - confidence: 0.91
- **[P1] Apply excludes to non-state backup assets:** `src/infra/backup-exclude.ts:358-364`
  - body: Returning `true` for every path outside the state-dir base makes `--exclude`, `--exclude-file`, and `.backupignore` no-op for external workspaces and custom config assets. A warning still leaves users with archives containing files they explicitly expected to omit, including large or sensitive workspace files.
  - confidence: 0.9
- **[P2] Preserve duplicate gitignore rule ordering:** `src/infra/backup-exclude.ts:225-233`
  - body: Gitignore rules are ordered, and duplicate patterns can be meaningful around negation, such as `*.log`, `!important.log`, `*.log`. Removing later duplicates before `ignore` sees them can silently flip include/exclude results from the user's declared rule order.
  - confidence: 0.83
- **[P2] Avoid a mandatory full archive rescan:** `src/infra/backup-create.ts:528-529`
  - body: This always reads the completed archive and stores every entry in memory, even when the user did not ask for `--verify`. Large backups are the motivating case for this PR, so an unconditional second full pass can materially worsen runtime and memory pressure.
  - confidence: 0.8
