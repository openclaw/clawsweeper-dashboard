  - body: The exclude path calls `tar.c` directly, while current main wraps archive creation in retry handling and also stages SQLite snapshots and volatile-file filtering.   - confidence: 0.87

  - body: This warning still archives external workspaces and custom config assets in full, even when users pass excludes for secrets such as `.env`, `*.pem`, or key files. A backup exclude feature needs per-asset filter bases, or it will silently fail on normal backup roots outside `stateDir`.
  - confidence: 0.91
- **[P1] Preserve hardlink metadata for backup verification:** `src/commands/backup-shared.ts:49-58`
  - body: The shared archive lister returns only path strings, so `backup verify` loses the `type` and `linkpath` metadata that current main uses to reject hardlinks pointing outside the archive root or at missing entries. Keep structured entries for verify while sharing any normalization helper.
  - confidence: 0.86
- **[P2] Preserve ordered gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns before giving them to `ignore` changes order-sensitive gitignore semantics, for example `*.log`, `!important.log`, `*.log` should end with `important.log` ignored again. Keep the ordered rule list and solve source attribution without collapsing duplicates.
  - confidence: 0.84
- **[P2] Avoid scanning every created archive into memory:** `src/infra/backup-create.ts:528-535`
  - body: This integrity check lists every archive entry into an array and then scans it for each asset on every backup, even without `--verify`. On large backups that adds a second full archive pass plus O(entry count) memory pressure, which can recreate the large-backup stall class this feature is trying to relieve.
  - confidence: 0.85
