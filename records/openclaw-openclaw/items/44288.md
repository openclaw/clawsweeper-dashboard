Full review comments:

- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-create.ts:378-382`
  - body: The PR now warns when excludes are active and an asset lives outside `stateDir`, but it still archives those external workspaces/config paths in full. A user who passes an exclude for logs, caches, or sensitive workspace files can therefore get a backup that ignores their rule for exactly the large or sensitive path they meant to omit; the fix should apply the matcher relative to each backed-up asset root or explicitly reject unsupported combinations.
  - confidence: 0.91
- **[P2] Preserve ordered gitignore rule duplicates:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore semantics because `ignore` evaluates rules in order and later matches can override earlier negations. For example, `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this deduplication removes the final rule and can include or exclude backup contents differently from the user's file.
  - confidence: 0.87
