Full review comments:

- **[P1] Preserve the hardened archive pipeline:** `src/infra/backup-create.ts:469-475`
  - body: Current main wraps tar creation in `writeTarArchiveWithRetry`, places the staged manifest outside every backed-up asset, skips volatile live files, and prunes installed-plugin `node_modules`. The exclude branch here calls raw `tar.c` with only `excludeFilter`, so any user enabling excludes loses those protections and can hit the same EOF stalls or archive bloat current main already fixed.
  - confidence: 0.92
- **[P2] Apply excludes to every archived root:** `src/infra/backup-create.ts:383-387`
  - body: This warning acknowledges that exclude patterns do not run for external workspaces or custom config assets, but the command still archives those assets in full. Users who pass `--exclude` or `.backupignore` can still include sensitive or bulky files they explicitly meant to omit, so this needs per-asset filter bases or a deliberately narrower documented scope before merge.
  - confidence: 0.9
- **[P2] Keep ordered ignore rules intact:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore semantics because negation and later matches are order-sensitive. For example, `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this dedupe removes the final rule and flips the result.
  - confidence: 0.86
