  - body: The exclude branch here tars `result.assets` directly, but current main now snapshots live SQLite state, skips volatile files, retries tar EOF races, and avoids hardlink reuse.   - body: The filter is built with `canonicalStateDir` as its only base, so external workspaces and custom config paths bypass `--exclude` and `.backupignore` and are archived in full.   - confidence: 0.86

  - body: Gitignore rules are order-sensitive, but this de-duplicates patterns before they reach `ignore`. A rule sequence like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but the final rule is dropped and the result flips.
  - confidence: 0.88

  - confidence: 0.82
