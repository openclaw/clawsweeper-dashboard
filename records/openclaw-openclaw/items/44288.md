## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve repeated rules in the declared order:** `src/infra/backup-exclude.ts:225-233`
  - body: Gitignore rules are order-sensitive, and this PR chooses `ignore` specifically for that contract. Dropping later duplicate strings before `ignore.add()` changes declarations such as `*.log`, `!important.log`, `*.log`, so backups can include or omit files contrary to the user's exclude file.
  - confidence: 0.9
- **[P1] Apply exclude rules to every backup asset root:** `src/infra/backup-exclude.ts:358-364`
  - body: This returns `true` for any entry outside the single stateDir base. Backup plans can include external workspace/config roots, so `--exclude` and `.backupignore` silently do not filter those trees; a warning still leaves the archive containing files the user explicitly tried to exclude.
  - confidence: 0.93
- **[P1] Remove the extra full archive passes:** `src/infra/backup-create.ts:467-508`
  - body: When excludes are active, this writes an uncompressed tar, gzips it, and then reads every archive entry into memory for the integrity check. On the multi-GB backups this feature is meant to rescue, that extra disk and scan can recreate stalls or memory pressure; collect presence facts during the write or make verification streaming/opt-in.
  - confidence: 0.86
