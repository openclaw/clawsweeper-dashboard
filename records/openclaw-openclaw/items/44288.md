- **[P1] Preserve the current backup tar filters:** `src/infra/backup-create.ts:469-475`
  - body: The new exclude archive path calls `tar.c` with only `excludeFilter`, while the current main backup writer also snapshots SQLite databases, skips volatile live files, filters extension `node_modules`, and retries EOF-class tar races.   - confidence: 0.92
- **[P1] Apply excludes to every backup asset root:** `src/infra/backup-exclude.ts:358-364`
  - body: Returning `true` for every path outside `baseDir` means explicit user excludes do not run for external workspaces, custom config paths, or external credentials that the backup plan can include.   - confidence: 0.96
- **[P2] Do not rescan every archive during create:** `src/infra/backup-create.ts:528-529`
  - body: This unconditional `listArchiveEntries` call reads the whole `.tar.gz` and stores every entry path even when the user did not request `--verify`. On large backups this adds a second full pass and O(entry count) memory to the same workflow the PR is trying to make reliable.
  - confidence: 0.91
- **[P2] Keep duplicate rules in their declared order:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore precedence because `ignore` evaluates rules in insertion order and later rules can override an earlier negation. Pattern lists such as `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this dedupe removes the final rule and flips the result.
  - confidence: 0.88
