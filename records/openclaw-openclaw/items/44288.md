## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply excludes to external backup assets:** `src/infra/backup-exclude.ts:362-363`
  - body: `resolveBackupPlanFromDisk` can include workspaces, custom config, and credentials paths that are outside `stateDir`, but this branch builds one filter rooted at `stateDir` and returns `true` for every outside path. That means `--exclude '*.key'`, `.backupignore`, and `--smart-exclude` do not filter those backed-up assets at all; a warning still leaves large or sensitive external trees archived contrary to the exclude contract.
  - confidence: 0.92
- **[P2] Preserve duplicate exclude rules in order:** `src/infra/backup-exclude.ts:229-231`
  - body: Gitignore rules are order-sensitive, including repeated patterns around negations. By dropping later duplicates before calling `ignore`, a sequence like `*.log`, `!important.log`, `*.log` is changed so `important.log` stays included even though the user's final rule re-excluded it.
  - confidence: 0.86
