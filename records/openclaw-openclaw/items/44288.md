## Summary

Keep open. Current main still lacks the requested backup exclude CLI surface, and the PR head is not ready because it still changes gitignore rule semantics and intentionally lets excludes bypass external backup assets.

## What This Changes

This PR adds backup exclude-pattern CLI flags, `.backupignore` loading, smart exclude defaults, protected-path guards, pattern-level manifest stats, backup verification changes, tests, and the `ignore` dependency.

## Best Possible Solution

Land a version that preserves exact gitignore rule order, applies excludes consistently to every included backup asset or explicitly narrows the contract, and updates docs, changelog, and focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: resolve `*.log`, `!important.log`, `*.log` and inspect the final rule order, then include an external workspace or custom config asset outside `stateDir` with any exclude pattern active.

## Solution Assessment

No. The PR direction is useful, but warning-only external asset bypasses and deduped gitignore rules are not the narrowest maintainable solution for a backup exclusion contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve duplicate exclude rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: `resolveExcludePatterns` removes later duplicate patterns before handing rules to `ignore`, but gitignore rules are order-sensitive. A list such as `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; dropping the final rule flips the user's declared result.
  - confidence: 0.9
- **[P2] Apply excludes to external backup assets:** `src/infra/backup-exclude.ts:358-363`
  - body: The filter returns `true` for paths outside `baseDir`, while `createBackupArchive` builds it with only `stateDir`. External workspaces or custom config paths are still archived in full under `--exclude`/`.backupignore`; a warning does not honor the filtering contract and can include sensitive files users tried to exclude.
  - confidence: 0.88
- **[P3] Document the new backup CLI surface:** `src/cli/program/register.backup.ts:30-41`
  - body: This adds user-facing backup flags, but the PR file list does not update `docs/cli/backup.md` or `CHANGELOG.md`. OpenClaw policy expects behavior/API docs and a changelog entry for user-facing feature and fix changes.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds backup exclusion controls, but current PR head can still archive sensitive external files users explicitly tried to exclude.

Concerns:

- **[medium] Exclude rules bypass external assets:** `src/infra/backup-exclude.ts:362`
  - body: When a workspace or custom config path lives outside `stateDir`, the filter includes it without applying exclude patterns. That can archive files such as `.env`, keys, logs, or model artifacts despite a user-supplied exclusion rule.
  - confidence: 0.88
