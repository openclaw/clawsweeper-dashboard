## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.98

Full review comments:

- **[P1] Preserve repeated rules in the declared order:** `src/infra/backup-exclude.ts:225-233`
  - body: Do not deduplicate the rule stream before passing it to `ignore`. Repeated rules can intentionally override an intervening negation—for example `*.log`, `!important.log`, `*.log`—so first-occurrence deduplication silently changes which files are backed up.
  - confidence: 0.99
- **[P1] Apply exclusion rules to every backup asset root:** `src/infra/backup-create.ts:370-378`
  - body: The matcher is rooted only at `stateDir`, so external workspaces and custom config paths bypass explicit exclusions. A warning does not satisfy the CLI contract and can leave large or sensitive files archived; construct matching relative to each planned asset root instead.
  - confidence: 0.98
- **[P1] Remove the full completed-archive rescan:** `src/infra/backup-create.ts:528-535`
  - body: This calls `tar.t` on every excluded backup and retains every entry name after already writing and compressing the archive. On the multi-gigabyte installations motivating this PR, that adds another full decompression pass and unbounded entry-list memory; validate required roots during writing or verification without materializing the entire list.
  - confidence: 0.97
