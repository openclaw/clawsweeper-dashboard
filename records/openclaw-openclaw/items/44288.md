## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Apply excludes to every backup asset root:** `src/infra/backup-create.ts:364-368`
  - body: This constructs the matcher with only `canonicalStateDir`, and the filter explicitly includes paths that resolve outside that base. Because backup plans can include external workspaces and custom config assets, `--exclude "*.pem"` or `--smart-exclude` can leave those assets archived in full despite the user-visible exclude contract; the fix should apply patterns per included asset root or narrow the CLI contract before merge.
  - confidence: 0.9
- **[P2] Avoid rescanning large archives during create:** `src/infra/backup-create.ts:528-535`
  - body: The create path now lists every entry in the completed tarball and stores the full list in memory before publishing. On the multi-GB/high-entry-count backups this PR is meant to unblock, that adds a second full gzip read plus O(entry count) memory pressure even when the user did not ask for `--verify`; use a streaming presence check or make this part of explicit verification instead.
  - confidence: 0.82
