## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-exclude.ts:362-363`
  - body: The filter is built with `stateDir` as its only base and returns `true` for every `../` or cross-drive path, so custom config files and external workspaces are archived in full even when the user supplied matching exclude rules. A warning does not make the backup obey the requested filter; the fix needs per-asset bases or a narrower documented contract before these flags are safe.
  - confidence: 0.9
- **[P2] Preserve duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns before calling `ignore().add()` changes gitignore precedence when the same rule appears on both sides of a negation, for example `*.log`, `!important.log`, `*.log`. Preserve the ordered rule list and track sources without dropping later duplicates so user-supplied pattern files behave like gitignore syntax.
  - confidence: 0.86
- **[P2] Avoid rescanning every archive entry on create:** `src/infra/backup-create.ts:528-535`
  - body: This cross-check calls `listArchiveEntries()` after writing every backup and stores every entry path in memory, adding a second full pass over the `.tar.gz`. On the multi-GB backups this PR is meant to help, that extra scan can materially increase runtime and memory pressure; validate roots during write or make the expensive scan opt-in with `--verify`.
  - confidence: 0.79
