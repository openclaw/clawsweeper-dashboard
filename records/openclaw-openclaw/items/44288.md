## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Compose excludes with the current tar filter:** `src/infra/backup-create.ts:473`
  - body: The exclude path passes only `excludeFilter` into `tar.c`, and the no-exclude path has no filter at all, so this branch drops current main's volatile-file skip, plugin `node_modules` pruning, and EOF retry wrapper. That can reintroduce the live-backup stalls and bloated archives that current main already hardened against.
  - confidence: 0.9
- **[P2] Keep the staged manifest outside backup sources:** `src/infra/backup-create.ts:425`
  - body: This recreates the temp directory directly under `os.tmpdir()`, but current main deliberately picks a temp root outside every backed-up asset. If TMPDIR points inside the state tree, the recursive tar walk can include the staged manifest again and produce duplicate root manifest entries that verify rejects.
  - confidence: 0.86
- **[P2] Apply excludes to every archived root:** `src/infra/backup-create.ts:364-367`
  - body: The filter is rooted only at `canonicalStateDir`, and the later warning still archives external workspaces or custom config assets in full. Users who pass `--exclude` for bulky or sensitive files will still get those files from non-state roots, so the explicit rule is not actually enforced for all backed-up content.
  - confidence: 0.84
- **[P2] Do not rescan every archive during create:** `src/infra/backup-create.ts:528`
  - body: `listArchiveEntries()` reads the full compressed archive and stores every entry after each create, even when the user did not request `--verify`. On the large backup cases this PR is meant to improve, that adds a second full archive pass and O(entry count) memory pressure to the default path.
  - confidence: 0.8
- **[P2] Preserve ordered ignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: The resolver removes later duplicate patterns before handing rules to the ignore matcher, but ignore-style negation is order-sensitive. A sequence like exclude, re-include, exclude can be changed by dropping the final duplicate, producing archive contents that differ from the user's declared rule order.
  - confidence: 0.76
