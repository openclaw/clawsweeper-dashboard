Full review comments:

- **[P1] Preserve repeated rules in declared order:** `src/infra/backup-exclude.ts:225-233`
  - body: Do not deduplicate the rule stream before passing it to `ignore`. Repeated rules can intentionally override an intervening negation, so first-occurrence deduplication changes which files are backed up.
  - confidence: 0.99
- **[P1] Apply exclusion rules to every backup asset root:** `src/infra/backup-create.ts:364-368`
  - body: The matcher is rooted only at `stateDir`, so external workspaces and custom config paths bypass explicit exclusions. A warning does not satisfy the CLI contract; construct matching relative to each planned asset root instead.
  - confidence: 0.98
- **[P1] Remove the full completed-archive rescan:** `src/infra/backup-create.ts:528-535`
  - body: This calls `tar.t` on every excluded backup and retains every entry name after writing and compressing the archive. On large installations, that adds another full decompression pass and unbounded entry-list memory.
  - confidence: 0.97
