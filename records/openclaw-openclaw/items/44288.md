- **[P1] Keep the default backup path on the current writer:** `src/infra/backup-create.ts:515-521`
  - body: The no-exclude branch now calls `tar.c()` directly instead of the current `writeTarArchiveWithRetry` path and its shared tar filter.   - confidence: 0.9
- **[P1] Compose exclude archives with current backup filters:** `src/infra/backup-create.ts:469-475`
  - body: The exclude path builds an uncompressed tar using only `filter: excludeFilter`, so `--smart-exclude` bypasses current main's volatile-file skip and installed-plugin dependency pruning while also losing the retry wrapper used for live-write races. This makes the new path less reliable for the large live backups it is intended to fix.
  - confidence: 0.88
- **[P2] Preserve ordered duplicate ignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: `resolveExcludePatterns` removes later duplicate patterns before passing them to `ignore`, but gitignore rules are order-sensitive around negation. A rule set like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; this deduplication drops the final rule and changes the user's declared precedence.
  - confidence: 0.86
- **[P2] Apply excludes to external assets before claiming support:** `src/infra/backup-create.ts:383-387`
  - body: The new warning acknowledges that assets outside `stateDir` are archived in full even when excludes are active. External workspaces and custom config paths are valid backup assets, so explicit patterns such as `node_modules/`, `*.pem`, or `.env*` can silently fail to filter the files users are trying to keep out of the archive.
  - confidence: 0.82
- **[P2] Avoid rescanning the full archive during create:** `src/infra/backup-create.ts:528-535`
  - body: `listArchiveEntries` reads the finished `.tar.gz` and stores every entry path before publishing. On the multi-GB/high-entry-count backups this feature targets, that adds a second full archive pass and O(entry count) memory pressure in the create path.
  - confidence: 0.8
