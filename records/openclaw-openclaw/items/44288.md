## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Apply exclude rules to every backup asset root:** `src/infra/backup-create.ts:364-368`
  - body: The filter is built with the state directory as its only base, and `buildExcludeFilter` returns `true` for paths that resolve outside that base. Any external workspace or custom config asset therefore ignores `--exclude`/`.backupignore`, so large or sensitive files the user explicitly excluded are still archived; the flags need per-asset bases or equivalent path normalization before merge.
  - confidence: 0.91
- **[P2] Preserve ordered duplicate gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: This deduplication removes later duplicate rules before handing the list to `ignore`, but gitignore semantics are order-sensitive. A rule set like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, while this code drops the final rule and flips the result.
  - confidence: 0.88
- **[P2] Avoid scanning each created archive into memory:** `src/infra/backup-create.ts:528-535`
  - body: Every successful create now calls `listArchiveEntries()` and stores all paths to validate asset roots. For the multi-GB backups this change targets, that adds a second full archive pass plus O(entry count) memory before publish, which can materially worsen stalls or OOM pressure.
  - confidence: 0.84
