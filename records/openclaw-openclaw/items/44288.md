Full review comments:

- **[P1] Apply exclude rules to every included asset:** `src/infra/backup-create.ts:383-387`
  - body: This branch knowingly leaves external workspaces and custom config paths outside the exclude filter, so `--exclude`/`.backupignore` can still archive files the user explicitly tried to omit. Because `backup create` includes workspace directories by default, the feature should either filter each asset with the correct base or narrow/document the contract before merge.
  - confidence: 0.86
- **[P2] Preserve ordered duplicate gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: `resolveExcludePatterns` removes later duplicates before calling `ignore`, but gitignore rules are order-sensitive around negations. A sequence such as `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; dropping the final duplicate flips the result and makes backups disagree with the declared rule order.
  - confidence: 0.9
- **[P3] Document the new backup exclude flags:** `src/cli/program/register.backup.ts:30-52`
  - body: The PR adds user-facing backup flags and manifest behavior, but the linked backup CLI reference still lists only the old options. Please update `docs/cli/backup.md` with the exclude flags, protected-path behavior, manifest metadata, and any accepted external-asset limitation so users following the CLI docs see the supported contract.
  - confidence: 0.78
