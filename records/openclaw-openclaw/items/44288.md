## What This Changes

The PR adds `backup create` exclude-pattern flags, `.backupignore` loading, smart defaults, protected-path guards, manifest `excludedStats`, a new `ignore` dependency, and backup create/verify tests.

## Best Possible Solution

Rebase and rework native backup excludes so they compose with current retry/volatile/plugin filters, apply or explicitly scope external assets, preserve gitignore order, avoid unconditional archive rescans, update docs/tests, and include redacted real-run proof.

## Solution Assessment

No. Native backup excludes are the right direction, but this branch needs a rebase and design fixes before it is the narrowest maintainable solution for data-loss/security-sensitive backup behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Compose exclude writes with current backup filters:** `src/infra/backup-create.ts:469-475`
  - body: When excludes are active, this path creates and gzips a raw tar with only `excludeFilter`. That bypasses current main's tar EOF retry, live volatile-file skip, and installed-plugin dependency filter, so merging it would regress backup reliability and archive contents for the exact large/live backups this feature targets.
  - confidence: 0.93
- **[P2] Apply excludes to external assets or fail clearly:** `src/infra/backup-create.ts:383-387`
  - body: This warning still lets the command succeed while external workspaces or custom config assets are archived in full. Patterns such as `*.pem`, `.env*`, or `logs/` therefore do not protect those roots; the implementation should apply filters per asset base or reject/require an explicit opt-out before merge.
  - confidence: 0.9
- **[P2] Preserve ordered duplicate ignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns before handing them to `ignore` changes gitignore semantics because repeated rules can intentionally override intervening negations. For example, `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this code drops the final rule and flips the result.
  - confidence: 0.88
- **[P2] Avoid rescanning every archive during create:** `src/infra/backup-create.ts:528-529`
  - body: The PR always lists the full `.tar.gz` into memory before publishing, even when `--verify` was not requested. On multi-GB backups this adds a second full archive pass plus O(entry count) memory, which can recreate the large-backup stalls the change is trying to avoid.
  - confidence: 0.84

## Security Review
