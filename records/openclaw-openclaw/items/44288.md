## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve ordered exclude rules:** `src/infra/backup-exclude.ts:225-232`
  - body: Deduplicating patterns before passing them to `ignore` changes gitignore's order-sensitive semantics. A rule sequence like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this code drops the final duplicate and flips the result, so backups can include or exclude files differently from the user's declared rules.
  - confidence: 0.9
- **[P1] Apply excludes to every backup asset root:** `src/infra/backup-create.ts:364-368`
  - body: The filter is built only against `stateDir`, and the PR now warns that assets outside that root bypass excludes entirely. That means explicit patterns do not apply to external workspaces or custom config roots, so large or sensitive files can still be archived despite the user requesting exclusion.
  - confidence: 0.91
- **[P2] Avoid rescanning whole archives after creation:** `src/infra/backup-create.ts:528-529`
  - body: `listArchiveEntries(tempArchivePath)` reads the completed `.tar.gz` entry list into memory after writing it. On the multi-GB backups this PR is intended to unblock, that adds another full archive pass and O(entry count) memory pressure, which can reintroduce stalls or OOMs.
  - confidence: 0.86
