## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Protect actual protected-path contents:** `src/infra/backup-exclude.ts:261-262`
  - body: The preflight guard only probes each pattern against `credentials`, `extensions`, `cron`, and one synthetic child. A broad file rule such as `*.json`, `*.pem`, or `*.key` can still match real files under those protected directories in the tar filter without `--allow-exclude-protected`, so enforce the protected-path rule against actual filtered paths or prove every protected descendant is covered before accepting the pattern.
  - confidence: 0.91
- **[P1] Apply excludes to every asset root:** `src/infra/backup-exclude.ts:362-363`
  - body: Returning true for every path outside the single `stateDir` base means external workspaces and custom config/credential assets are archived in full even when they contain files matching `--exclude` or `.backupignore`. A warning still leaves the backup behavior wrong for users trying to remove large or sensitive external trees; use per-asset filter bases or fail/disable excludes for unsupported roots.
  - confidence: 0.9
- **[P1] Preserve ordered gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: This deduplication drops later duplicate rules before passing them to `ignore`, but gitignore semantics are order-sensitive. A rule set like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; keeping only the first `*.log` flips the result and can silently keep or drop different files than the user requested.
  - confidence: 0.89
- **[P2] Avoid rescanning every archive on create:** `src/infra/backup-create.ts:528-529`
  - body: This always reads the completed `.tar.gz` and stores every entry path in memory after archive creation, even when the user did not request `--verify`. For the multi-GB backup path this feature is meant to improve, that adds another full archive pass and O(entry count) memory pressure; keep this check on explicit verify or replace it with bounded streaming validation.
  - confidence: 0.86
