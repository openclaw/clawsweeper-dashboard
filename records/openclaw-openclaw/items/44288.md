## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve ordered duplicate exclude rules:** `src/infra/backup-exclude.ts:225-233`
  - body: `ignore` applies rules in order, but this pre-pass drops later duplicate patterns before the matcher sees them. A pattern list such as `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; after this de-duplication it stays unignored, so backups can include or exclude files differently from the user’s gitignore-style rule order.
  - confidence: 0.85
- **[P2] Avoid rescanning every completed backup:** `src/infra/backup-create.ts:528`
  - body: This unconditional `listArchiveEntries()` call reads the finished `.tar.gz` and stores every entry name in memory for every create path. On the multi-GB, high-entry-count backups this PR targets, that adds a second archive pass and O(entry count) memory pressure even when the user did not request `--verify`, so it can reintroduce stalls or OOM-style failures.
  - confidence: 0.78
