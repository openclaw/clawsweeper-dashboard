- **[P1] Run excludes through the current backup writer:** `src/infra/backup-create.ts:469-476`
  - body: The exclude-enabled path creates a separate tar/gzip pipeline instead of extending the current `tarFilter` path, so it bypasses current-main safeguards such as SQLite snapshots/remaps, volatile-file skips, extension `node_modules` filtering, tar EOF retries, safe temp-root selection, and backup retry logging. That can produce incomplete or stale recovery archives exactly where backup correctness matters most.
  - confidence: 0.94
- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-exclude.ts:358-364`
  - body: The filter treats any path outside the single stateDir base as automatically included, so excludes do not apply to custom config files, external credentials, or workspace trees that `resolveBackupPlanFromDisk()` includes as valid backup assets. A warning still leaves sensitive or oversized files in the archive despite explicit exclude rules, so the filter needs per-asset bases or equivalent semantics before merge.
  - confidence: 0.93
- **[P2] Preserve ordered gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: De-duplicating patterns before constructing the `ignore` matcher changes order-sensitive gitignore behavior when duplicate rules appear around negations, such as `*.log`, `!important.log`, `*.log`. Keep the ordered rule list intact and track source metadata without collapsing duplicate patterns.
  - confidence: 0.86
- **[P2] Avoid rescanning large archives during create:** `src/infra/backup-create.ts:528-541`
  - body: The post-create cross-check reads every entry from the freshly written `.tar.gz` and stores all paths in memory on every create, adding a second full archive pass to the large-backup path this PR is meant to improve. Move this to explicit `--verify` or stream the check without O(entry count) memory before making it unconditional.
  - confidence: 0.84
- **[P2] Document the new backup recovery contract:** `src/cli/program/register.backup.ts:30-52`
  - body: The command help adds six public flags and the implementation adds `.backupignore`, protected-path override, and `excludedStats` behavior, but the linked backup docs still describe only coarse backup controls. User-facing recovery flags and archive/JSON metadata need docs before merge so scripts and operators know the supported contract and limitations.
  - confidence: 0.79
