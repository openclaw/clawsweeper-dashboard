## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Apply excludes to each backup asset root:** `src/infra/backup-exclude.ts:362-363`
  - body: The filter includes every path outside the state directory, while backup plans can include external workspaces and custom config or credential assets. That means `--exclude "*.pem"` or `--smart-exclude` can still archive those assets in full; apply patterns per asset root or narrow the CLI contract before merge.
  - confidence: 0.9
- **[P2] Avoid rescanning large archives during create:** `src/infra/backup-create.ts:528-535`
  - body: This lists every entry in the completed archive and stores the full list in memory before publishing, even when the user did not pass `--verify`. On multi-GB or high-entry backups this adds the same kind of time and memory pressure this feature is meant to reduce, so make it streaming or verify-gated.
  - confidence: 0.84
- **[P2] Preserve duplicate exclude rule order:** `src/infra/backup-exclude.ts:229-232`
  - body: Deduplicating exact patterns before handing them to `ignore` changes gitignore precedence when duplicates are separated by negation, for example `*.log`, `!important.log`, `*.log`. The final rule should win, but this drops it and can include or exclude files contrary to the user's pattern file.
  - confidence: 0.78
