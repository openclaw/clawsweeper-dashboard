## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Preserve repeated rules in the declared order:** `src/infra/backup-exclude.ts:225-233`
  - body: Do not deduplicate the rule stream before passing it to `ignore`. Gitignore rules are order-sensitive, so a repeated rule after a negation can intentionally change the final decision; first-occurrence deduplication can silently back up or drop the wrong files.
  - confidence: 0.99
- **[P1] Apply exclusions to every backup asset root:** `src/infra/backup-create.ts:370-378`
  - body: The matcher is rooted only at `stateDir`, and the new warning confirms external workspaces or custom config paths bypass user rules. A backup exclusion flag should not leave large or sensitive external assets archived in full; construct matching relative to each planned asset root instead.
  - confidence: 0.98
- **[P1] Remove the full completed-archive rescan:** `src/infra/backup-create.ts:528-535`
  - body: This path calls `tar.t` on the completed `.tar.gz` and stores every entry name after already writing and compressing the backup. On the multi-gigabyte installations motivating the PR, that adds another full pass and unbounded entry-list memory; validate required roots during writing or verification without materializing the whole archive list.
  - confidence: 0.97
