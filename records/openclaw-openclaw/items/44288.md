  - body: This exclude path writes a raw tar and gzips it directly instead of composing with current main's `writeTarArchiveWithRetry`, volatile-file skip, temp-root handling, and installed-plugin dependency filter.   - confidence: 0.94

  - body: This warning still archives external workspaces and custom config assets in full, so patterns like `.env*`, `*.pem`, or `logs/` do not protect those roots. The feature should either apply filters per asset base or fail/require an explicit scoped opt-in before merge.
  - confidence: 0.9
- **[P2] Keep ordered ignore rules intact:** `src/infra/backup-exclude.ts:225-233`
  - body: Removing later duplicate patterns changes gitignore semantics because `ignore` evaluates rules in order and negation can be overridden by a later duplicate. Preserve the user's ordered rule list instead of deduplicating by pattern string.
  - confidence: 0.88
- **[P2] Avoid rescanning every archive during create:** `src/infra/backup-create.ts:525-528`
  - body: `listArchiveEntries` reads the whole `.tar.gz` and stores every entry after every create, even without `--verify`. On multi-GB backups this adds a second full pass plus O(entry count) memory and can reintroduce the large-backup stalls this PR is trying to solve.
  - confidence: 0.87
