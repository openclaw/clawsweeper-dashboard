  - body: When exclude patterns are active, this calls `tar.c` directly with only `excludeFilter`, so current main's retry wrapper, volatile-file skip, and installed-plugin dependency filter are not applied.   - confidence: 0.91

  - body: Removing later duplicate patterns changes gitignore precedence when negations sit between repeated rules, for example `*.log`, `!important.log`, `*.log`. Preserve the rule list order and track source metadata separately so user-declared rules do not flip results.
  - confidence: 0.88

  - body: This unconditional `listArchiveEntries` pass reads the finished `.tar.gz` and stores every entry on every backup, even without `--verify`. On the multi-GB states this feature targets, it doubles I/O and adds O(entry count) memory; keep create lightweight or make the check verify-only/streaming.
  - confidence: 0.82
