## Summary

Keep open. The PR addresses a real backup exclusion need, but the current head is not safe to merge: it drops current-main backup reliability safeguards, leaves explicit excludes as a no-op for external backup roots, changes ordered gitignore semantics, adds an unconditional full archive scan, and still lacks real behavior proof.

## What This Changes

This PR adds `openclaw backup create` exclude flags, gitignore-style pattern handling, smart defaults, protected-path guards, `.backupignore` support, per-pattern manifest stats, and backup create/verify tests.

## Best Possible Solution

Land a rebased native backup exclusion design that composes with current backup filters, retry, and temp-root safeguards, applies ordered gitignore rules consistently across every archived root, and includes redacted real-run proof.

## Reproduction Assessment

Yes for the review findings: PR-head source shows the dropped current-main safety paths, external-root bypass, duplicate-rule collapse, and unconditional archive readback. I did not run a live backup because this is a read-only review and the contributor proof gate is still unmet.

## Solution Assessment

No. Native backup excludes are the right direction, but this branch should first be rebased onto current main, preserve existing backup safety behavior, define external-root semantics, preserve gitignore rule order, and avoid normal-create full rescans.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve current tar retry and volatile filtering:** `src/infra/backup-create.ts:469-476`
  - body: This exclude archive path calls `tar.c` directly instead of going through current main's retry wrapper and volatile-state filter. Merging this would reintroduce live-write EOF failures and archive live sessions/logs/queues in the same large-backup path this PR is meant to improve.
  - confidence: 0.9
- **[P2] Keep temp manifests outside backed-up sources:** `src/infra/backup-create.ts:425`
  - body: The branch stages the temporary manifest under `os.tmpdir()` again. Current main chooses a temp root outside every source asset; without that guard, `TMPDIR` inside `~/.openclaw` can make tar include the manifest twice and cause backup verification to fail.
  - confidence: 0.88
- **[P2] Compose excludes with current backup filters:** `src/infra/backup-create.ts:473`
  - body: Passing only `excludeFilter` means the existing installed-plugin `extensions/**/node_modules` filter is not applied when excludes are active, and the no-exclude path on this branch also lacks the current-main filter. That regresses the shipped backup-size behavior for plugin dependencies.
  - confidence: 0.86
- **[P2] Apply excludes per archived asset root:** `src/infra/backup-create.ts:364-368`
  - body: A single filter rooted at `canonicalStateDir` makes user patterns a no-op for external workspaces, config, or credential assets. A warning still leaves files such as `.env*`, `*.pem`, or large external logs archived despite explicit exclude rules.
  - confidence: 0.9
- **[P2] Preserve duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore precedence because negations are order-sensitive. A list like `*.log`, `!important.log`, `*.log` should end with `important.log` ignored, but this de-duplication removes the final rule and flips the result.
  - confidence: 0.88
- **[P2] Avoid full archive rescans during create:** `src/infra/backup-create.ts:528-535`
  - body: `listArchiveEntries()` materializes every tar entry before publishing the archive even when the user did not ask for `--verify`. For the multi-GB backups this feature targets, that adds a second full archive pass and memory proportional to entry count to normal create.
  - confidence: 0.84
