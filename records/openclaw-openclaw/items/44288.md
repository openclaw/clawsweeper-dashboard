## Summary

Keep open: the requested backup exclude feature is still useful and not on current main, but this PR is outdated against the current backup pipeline, has blocking correctness/security issues, and lacks inspectable real behavior proof.

## What This Changes

The PR adds backup exclude CLI flags, smart defaults, protected-path checks, `.backupignore` loading, manifest `excludedStats`, an `ignore` dependency, and backup create/verify tests.

## Best Possible Solution

Rebase onto current main, compose exclude handling with the shipped backup safety filters and retry/temp-root behavior, define per-asset exclude semantics, then provide redacted real backup proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Compose excludes with current backup safety filters:** `src/infra/backup-create.ts:473`
  - body: With excludes active, this path passes only `excludeFilter` to `tar.c`; current main also filters volatile live files and `extensions/*/node_modules`. Rebase and compose the filters so exclude backups do not reintroduce large dependency trees or live-write races.
  - confidence: 0.92
- **[P2] Apply excludes to every backup asset root:** `src/infra/backup-exclude.ts:362-363`
  - body: The filter returns `true` for paths outside the state-dir `baseDir`, so external workspaces or custom config assets are archived in full despite explicit exclude patterns. A warning still violates the promised exclude behavior; use per-asset bases or equivalent mapping.
  - confidence: 0.88
- **[P2] Run archive writes through the retry wrapper:** `src/infra/backup-create.ts:469-476`
  - body: The exclude archive path calls `tar.c` directly instead of current main's EOF retry and error-enrichment wrapper. A single live-file race can abort the backup rather than retrying and cleaning the partial temp archive.
  - confidence: 0.87
- **[P2] Keep the temp manifest outside source paths:** `src/infra/backup-create.ts:425-426`
  - body: This stages the manifest under `os.tmpdir()` directly. Current main chooses a temp root outside every backup source, so this branch can reintroduce duplicate root `manifest.json` entries when `TMPDIR` lives under the state directory.
  - confidence: 0.86
- **[P2] Preserve duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore semantics, where later rules can override earlier negations. A sequence such as `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but first-wins dedupe flips the result.
  - confidence: 0.84
- **[P2] Avoid scanning created archives into memory:** `src/infra/backup-create.ts:528-529`
  - body: `listArchiveEntries` reads the whole archive entry list into an array on every create path. For the large-backup cases this feature targets, that adds a second full archive pass and O(entry count) memory pressure before publish.
  - confidence: 0.8
