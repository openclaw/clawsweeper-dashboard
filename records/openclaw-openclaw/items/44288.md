Full review comments:

- **[P1] Run excludes through the current backup pipeline:** `src/infra/backup-create.ts:462-475`
  - body: When excludes are active this branch switches to a separate uncompressed-tar path that archives only `result.assets` and never invokes the current main logic for SQLite snapshots, volatile-state skips, tar EOF retries, plugin dependency filtering, or safe temp-root selection. A smart-exclude backup can therefore regress recovery safeguards that normal backups now rely on; rebase and compose the exclude filter with the current tar pipeline instead of carrying a second create path.
  - confidence: 0.91
- **[P1] Apply excludes to every backup asset root:** `src/infra/backup-exclude.ts:358-363`
  - body: Returning `true` for `../` or cross-drive paths means any external workspace, custom config, or external credentials asset is archived in full even when the user passed matching `--exclude` rules. The warning added later does not make the command semantics safe: sensitive or oversized external files still bypass explicit patterns.
  - confidence: 0.9
- **[P1] Preserve hardlink validation when sharing entry listing:** `src/commands/backup-shared.ts:49-56`
  - body: This new shared helper returns only path strings and discards `type`/`linkpath`, while current main verifies hardlink targets stay inside the archive root and point at existing entries. Merging this shape would remove a backup-verify security check for crafted archives unless the helper preserves full entry metadata and the verifier keeps the current validation.
  - confidence: 0.86
- **[P2] Preserve ordered gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns before passing them to `ignore` changes ordered gitignore semantics, where later duplicate rules after a negation can intentionally override an earlier rule. Keep the raw ordered rule list for matching and track source/stats separately so `*.log`, `!important.log`, `*.log` behaves as declared.
  - confidence: 0.84
- **[P2] Avoid rescanning large archives during create:** `src/infra/backup-create.ts:528-535`
  - body: The post-create cross-check reads every entry from the finished `.tar.gz` and stores all paths in memory for every backup, even when `--verify` was not requested. On the multi-GB, high-entry-count installations this PR targets, that adds a second full archive pass and can reintroduce the stall/memory-pressure failure mode.
  - confidence: 0.82
- **[P2] Document the new backup flags and manifest contract:** `src/cli/program/register.backup.ts:30-52`
  - body: The PR registers public backup CLI flags and adds `excludedStats` output/archive metadata, but the user-facing backup docs on current main still only describe coarse controls like `--no-include-workspace` and `--only-config`. Public recovery commands need matching docs before merge so users understand protected paths, external-asset limitations, and JSON/manifest behavior.
  - confidence: 0.8
