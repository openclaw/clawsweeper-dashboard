## Summary

This PR still carries useful backup-exclude functionality that is not present on current main, but it should stay open because the branch has unresolved correctness and security-sensitive backup semantics plus missing after-fix real behavior proof.

## What This Changes

The branch adds `backup create` exclude flags, `.backupignore` and exclude-file handling, protected-path guards, per-pattern manifest stats, the `ignore` dependency, and backup exclude tests.

## Best Possible Solution

Keep the native exclude feature work, but rebase it into a smaller implementation that composes with current main's hardened writer, defines per-asset exclude semantics, avoids unconditional full-archive rescans, and includes redacted real-run proof.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR diff shows the external-asset bypass and unconditional archive rescan paths. I did not run a live backup because this is a read-only review.

## Solution Assessment

No. Adding exclude support is a reasonable direction, but this branch is not the best merge shape until it preserves current backup hardening and resolves how exclude rules apply to every backed-up asset base.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply excludes to every backed-up asset base:** `src/infra/backup-exclude.ts:362-363`
  - body: `buildExcludeFilter` returns `true` for paths outside the single `stateDir` base, and the create path constructs only that one filter. As a result, external workspaces or custom config assets are still archived in full when users pass `--exclude`, `.backupignore`, or `--smart-exclude`, so sensitive or large files such as `.env`, `*.pem`, or dependency trees can bypass explicit exclude rules.
  - confidence: 0.9
- **[P2] Avoid rescanning every archive during create:** `src/infra/backup-create.ts:528-535`
  - body: The post-create integrity check reads the entire `.tar.gz` into an in-memory entry array before publishing the archive, even when `--verify` was not requested. On the multi-GB backup trees this feature targets, that adds a second full archive pass and O(entry count) memory pressure in the create path.
  - confidence: 0.84
