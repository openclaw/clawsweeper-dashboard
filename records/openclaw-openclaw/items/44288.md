## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Gate .backupignore loading behind explicit opt-in:** `src/infra/backup-exclude.ts:190-194`
  - body: `resolveExcludePatterns` reads `stateDir/.backupignore` on every backup create, even when the user passes no exclude flags. That violates the PR's no-flag compatibility claim and can silently omit restorable state for any existing installation that already has this file; require an explicit flag/product decision or keep no-flag backups identical.
  - confidence: 0.91
- **[P1] Apply excludes to every backed-up asset:** `src/infra/backup-exclude.ts:362-363`
  - body: Returning `true` for every path outside `baseDir` makes external workspaces, custom config paths, and external credentials immune to the new exclude rules. A warning later in `createBackupArchive` is not enough for `--exclude "*.key"` or `--smart-exclude` because the archive still contains files the user explicitly tried to filter.
  - confidence: 0.9
- **[P2] Avoid rescanning the whole archive on create:** `src/infra/backup-create.ts:528`
  - body: The new integrity cross-check calls `listArchiveEntries` after every backup create, reading the completed `.tar.gz` and retaining every entry path in memory even when `--verify` was not requested. On the multi-GB backups this feature targets, that extra full pass can materially increase runtime and memory pressure.
  - confidence: 0.86
