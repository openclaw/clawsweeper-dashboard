## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Compose excludes with current backup safety filters:** `src/infra/backup-create.ts:473`
  - body: The exclude path passes only `excludeFilter` to `tar.c`. Current main also filters volatile live state and `extensions/*/node_modules`; with any exclude pattern enabled, those paths can return to the archive and reintroduce bloat and live-write races. Compose the new matcher with the current tar filter before writing.
  - confidence: 0.92
- **[P2] Apply excludes to every backup asset root:** `src/infra/backup-exclude.ts:362-363`
  - body: The filter is built with `stateDir` as its only base and returns `true` for paths outside it, but backups can include external workspaces and custom config paths. A warning still archives data that the user explicitly excluded; use per-asset filter bases or an equivalent mapping so patterns apply to all included assets.
  - confidence: 0.88
- **[P2] Run exclude archive writes through the retry wrapper:** `src/infra/backup-create.ts:469-476`
  - body: The exclude path calls `tar.c` directly. Current main wraps tar writes in retry and error-enrichment handling for EOF races from live files, so an exclude backup can fail once and abort instead of following the hardened mainline behavior.
  - confidence: 0.87
- **[P2] Keep the temp manifest outside source paths:** `src/infra/backup-create.ts:425-426`
  - body: This stages the manifest under `os.tmpdir()`. In the no-exclude path the manifest is written before tar walks the assets, so cron or sandbox environments with `TMPDIR` under `~/.openclaw` can include it twice and fail verification; preserve current main’s `chooseBackupTempRoot` behavior.
  - confidence: 0.86
- **[P2] Preserve duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns with the first occurrence winning changes gitignore semantics, where the last matching rule decides. A sequence like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this drops the final rule and flips the result.
  - confidence: 0.84
- **[P2] Avoid scanning every created archive into memory:** `src/infra/backup-create.ts:528-529`
  - body: `listArchiveEntries` reads the full `.tar.gz` entry list into an array on every create path, adding a second full archive pass and O(entry count) memory to the large-backup scenarios this feature is meant to help. Stream the integrity check or reserve the full scan for explicit verification.
  - confidence: 0.8
