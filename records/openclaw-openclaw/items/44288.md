Full review comments:

- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-exclude.ts:358-363`
  - body: `resolveBackupPlanFromDisk` can include external workspaces or custom config paths, but this filter returns `true` for anything outside the single `stateDir` base. That makes `--exclude`, `--exclude-file`, and `.backupignore` a no-op for those assets, so large or sensitive workspace files are still archived in full despite matching user rules; the later warning does not make the new exclude feature correct.
  - confidence: 0.92
- **[P2] Preserve ordered duplicate gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: The `ignore` dependency applies gitignore rules in order, including later duplicate rules that can intentionally override an earlier negation. Dropping later duplicates here changes cases like `*.log`, `!important.log`, `*.log`, so files can be included or excluded differently from the user's declared rule order.
  - confidence: 0.86
- **[P2] Avoid scanning every created archive into memory:** `src/infra/backup-create.ts:525-529`
  - body: This post-create check always reads the full `.tar.gz` and stores every entry path even when the user did not request `--verify`. For the large backup cases this PR is trying to fix, that adds a second full archive pass plus O(entry count) memory before publish, so the integrity check should be streamed, bounded to the needed asset roots, or kept behind verification.
  - confidence: 0.78
