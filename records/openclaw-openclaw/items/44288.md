## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Apply exclude filters to every backup asset root:** `src/infra/backup-create.ts:364-368`
  - body: The backup plan can include workspaces or custom config paths outside `stateDir`, but this builds one filter rooted at `canonicalStateDir` and `buildExcludeFilter` returns `true` for outside paths. That means `--exclude "*.pem"` or `--smart-exclude` does not apply to external workspace trees; warning about the gap still archives the large or sensitive files in full.
  - confidence: 0.9
- **[P2] Preserve ordered duplicate gitignore rules:** `src/infra/backup-exclude.ts:225-232`
  - body: This de-duplicates patterns before passing them to `ignore`, but gitignore rules are order-sensitive and duplicates can be meaningful around negation, such as `*.log`, `!important.log`, `*.log`. Dropping the later duplicate changes the user's declared include/exclude result, so keep the ordered rule list and track stats without collapsing rule order.
  - confidence: 0.84
- **[P2] Avoid scanning every created archive into memory:** `src/infra/backup-create.ts:528-535`
  - body: This integrity check lists every archive entry into an array and then scans it for each asset on every backup, even when the user did not ask for `--verify`. For multi-GB state/workspace backups, that adds a second full archive pass and O(entry count) memory pressure, which can reintroduce the large-backup stall class this feature is meant to relieve.
  - confidence: 0.85
