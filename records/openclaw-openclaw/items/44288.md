  - confidence: 0.94
- **[P2] Avoid a full archive rescan in create:** `src/infra/backup-create.ts:528-529`
  - body: `listArchiveEntries` reads the whole gzip archive and stores every entry path before publish, even when the user did not request `--verify`. Large backups are the target workflow here, so this adds an extra full pass and O(entry count) memory to create and can worsen stalls or memory pressure.
  - confidence: 0.91
- **[P2] Keep duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: Dropping later duplicate patterns changes gitignore precedence because `ignore` evaluates rules in insertion order and later rules can override earlier negations. A list like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this dedupe removes the final rule and flips the result.
  - confidence: 0.88
