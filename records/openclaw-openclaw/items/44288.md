Full review comments:

- **[P1] Apply excludes to every planned backup asset:** `src/infra/backup-create.ts:383-387`
  - body: This warning still lets external workspaces and custom config paths bypass all exclude patterns and get archived in full. Since those assets are included by the backup plan by default, `--exclude "*.pem"` or `--smart-exclude` can still leave large or sensitive external files in the archive; apply filters per asset/base or fail before writing instead of continuing with a warning-only caveat.
  - confidence: 0.88
- **[P1] Port excludes onto the current backup writer:** `src/infra/backup-create.ts:469-476`
  - body: The exclude path creates a tar directly from `result.assets`, but current main's writer now snapshots live SQLite files, skips volatile runtime files, filters extension dependency trees, and retries tar EOF races. Rebase and integrate the exclude filter into that current pipeline, otherwise conflict resolution can regress backup safety and reintroduce large/live backup failures.
  - confidence: 0.82
- **[P2] Preserve duplicate rule order:** `src/infra/backup-exclude.ts:225-233`
  - body: Gitignore rules are order-sensitive, so duplicate patterns can be meaningful when negations appear between them, for example `*.log`, `!important.log`, `*.log`. Deduplicating here drops the final rule and changes what gets backed up; keep the ordered rule list passed to `ignore` intact and track source metadata separately.
  - confidence: 0.86
- **[P2] Pass raw ignore-file lines to the matcher:** `src/infra/backup-exclude.ts:88-92`
  - body: Trimming every non-comment line before `ignore.add()` removes escaped trailing spaces and other whitespace that the `ignore` package is designed to parse according to gitignore rules. This breaks the promised gitignore syntax for `.backupignore` and `--exclude-file`; preserve raw pattern text except for line ending removal and comment/blank filtering that respects escapes.
  - confidence: 0.78
- **[P3] Document the new backup flags:** `src/cli/program/register.backup.ts:30-52`
  - body: These flags are new public CLI behavior, but `docs/cli/backup.md` remains unchanged on the PR. Add user docs for the exclude flags, protected paths, external-asset semantics, manifest output, and proof expectations so the CLI contract matches the published backup reference.
  - confidence: 0.9
