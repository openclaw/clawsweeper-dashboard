Full review comments:

- **[P1] Preserve the hardened archive pipeline:** `src/infra/backup-create.ts:469-475`
  - body: Current main wraps tar writes in retry handling, keeps temp manifests outside source roots, skips volatile live state, and prunes installed plugin dependency trees. This direct `tar.c` exclude path does not compose those filters or the retry wrapper, so using exclude flags can reintroduce EOF/live-file failures and bloated plugin dependency archives.
  - confidence: 0.92
- **[P2] Apply excludes to every archived root:** `src/infra/backup-exclude.ts:362-363`
  - body: Entries outside `baseDir` return true before matching, while the backup plan can include external workspaces, custom config, or external credentials roots. That makes `--exclude` and `.backupignore` a no-op for those assets; a warning still leaves the archive wider than the requested rules.
  - confidence: 0.9
- **[P2] Keep ordered ignore rules intact:** `src/infra/backup-exclude.ts:225-233`
  - body: `ignore` evaluates gitignore rules in order and supports later negations or re-ignores. Removing later duplicates changes lists such as `*.log`, `!important.log`, `*.log`, so the archive can include or omit files differently from the user's exclude file.
  - confidence: 0.83
- **[P2] Avoid unconditional archive rescans:** `src/infra/backup-create.ts:528-529`
  - body: `listArchiveEntries` reads the entire `.tar.gz` and stores every entry path for every create, even when `--verify` was not requested. Large backups now pay a second full gzip read plus O(entry count) memory, which can recreate the stall/OOM class this feature is trying to reduce.
  - confidence: 0.82
