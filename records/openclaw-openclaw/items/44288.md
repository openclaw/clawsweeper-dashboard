  - body: The archive path under review still uses the older direct/two-step tar writer.   - confidence: 0.87
- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-exclude.ts:358-364`
  - body: Paths outside the single `stateDir` base are unconditionally included, so explicit `--exclude` and `.backupignore` rules do nothing for valid external workspace, config, or credential assets. A security-facing exclude feature cannot leave large or sensitive external backup roots archived in full after the user supplied matching rules.
  - confidence: 0.89
- **[P2] Preserve duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: The resolver drops later duplicate patterns before handing rules to `ignore`, but gitignore precedence is order-sensitive when negations are involved. A sequence like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; keep the full pattern list and track source metadata separately.
  - confidence: 0.8
- **[P2] Avoid scanning the whole archive during create:** `src/infra/backup-create.ts:525-542`
  - body: `listArchiveEntries()` reads every tar entry into memory immediately after writing the archive just to cross-check asset roots. On the large backups this PR targets, that adds a second full archive pass and O(entry-count) memory; verify required roots during streaming or use a bounded check instead.
  - confidence: 0.82
