  - body: This deduplication drops later duplicate patterns before they reach `ignore`, but gitignore semantics are order-sensitive around negations.   - confidence: 0.84
- **[P2] Preserve the manifest size cap while verifying:** `src/commands/backup-verify.ts:202-210`
  - body: The PR version of `extractManifest` buffers manifest chunks without the current main 1 MB cap, so verifying a malicious or corrupted archive can allocate unbounded memory before validation. Keep the bounded read behavior from current main when adding tolerant `excludedStats` parsing.
  - confidence: 0.82
- **[P2] Avoid a full archive rescan in create:** `src/infra/backup-create.ts:528-535`
  - body: This unconditional `listArchiveEntries` pass reads every entry into memory after every successful create, even when the user did not ask for `--verify`. For the large installations this PR targets, that adds a second full archive pass and O(entry count) memory pressure on the hot create path.
  - confidence: 0.8
