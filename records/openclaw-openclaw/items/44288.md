## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-create.ts:364-368`
  - body: `openclaw backup create` includes workspace directories by default, but this constructs a single filter rooted at the state directory and then only warns when workspace/config assets live outside that root. In those common setups `--exclude`, `.backupignore`, and `--smart-exclude` do not filter the external asset at all, so large or sensitive files can still be archived despite explicit rules. Build filters relative to each asset root, or otherwise map entries to the owning asset before calling `ignore`.
  - confidence: 0.9
- **[P2] Preserve ordered duplicate gitignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Gitignore rules are order-sensitive, and the `ignore` dependency evaluates the rule list in sequence. Deduplicating before matching changes cases like `*.log`, `!important.log`, `*.log`, where the final duplicate is meant to override the negation; the PR instead drops it and keeps `important.log`. Keep the original ordered rule list for matching, and track stats/source separately if needed.
  - confidence: 0.88
- **[P2] Avoid a full archive rescan on every create:** `src/infra/backup-create.ts:528-529`
  - body: The post-tar cross-check now reads the entire generated archive into an array on every backup, even without `--verify`. For the multi-GB backups this PR is meant to help, that adds another full compressed-archive pass plus O(entry count) memory before publish. Either stream the check without retaining every path, scope it to explicit verify/debug behavior, or prove this extra pass is safe for the large-backup cases.
  - confidence: 0.78
