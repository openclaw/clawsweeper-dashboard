## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Rebase excludes onto the current backup writer:** `src/infra/backup-create.ts:469-475`
  - body: When excludes are active, this branch writes `result.assets` through a separate `tar.c`/append/gzip flow instead of the current writer. That path skips SQLite snapshots, volatile-file filtering, EOF retry, hardlink cache, and source-path remaps, so enabling excludes can produce torn or incomplete backups compared with current main.
  - confidence: 0.94
- **[P1] Gate automatic .backupignore loading:** `src/infra/backup-exclude.ts:187-194`
  - body: `resolveExcludePatterns()` loads `stateDir/.backupignore` for every backup unless `--include-all` is set, so an existing file changes backup contents even when the user passes no exclude option. That is a default backup-scope change and needs opt-in behavior or explicit maintainer-approved upgrade/docs coverage.
  - confidence: 0.91
- **[P2] Apply excludes outside the state directory:** `src/infra/backup-exclude.ts:358-364`
  - body: The filter returns `true` for paths outside the single `stateDir` base, but current backup scope includes external config, credentials, and workspace assets. As a result, `--exclude` can silently fail to remove sensitive or bulky external files from the archive.
  - confidence: 0.9
- **[P2] Preserve ordered duplicate ignore rules:** `src/infra/backup-exclude.ts:225-233`
  - body: Gitignore semantics are order-sensitive, and the `ignore` engine evaluates later matching rules after earlier ones. First-occurrence dedupe changes cases like `*.log`, `!important.log`, `*.log`, causing different inclusion results than the user-declared rule order.
  - confidence: 0.86
- **[P2] Avoid full archive scans during create:** `src/infra/backup-create.ts:528-529`
  - body: The post-create cross-check reads the full `.tar.gz` and stores every entry before publishing the archive. On the multi-GB backup cases this PR targets, that adds another full archive pass and O(entry count) memory pressure in the create path.
  - confidence: 0.82
