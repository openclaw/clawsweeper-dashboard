## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply excludes to every backed-up asset root:** `src/infra/backup-exclude.ts:362-363`
  - body: `createBackupArchive` builds one filter rooted at the state dir, and this branch returns `true` for paths outside that base. Any external workspace or custom config path from the backup plan is therefore archived in full despite `--exclude` or `.backupignore`, which breaks the new CLI contract for large or sensitive external assets. Please apply patterns relative to each backed-up asset root, or explicitly narrow and document the feature as state-dir-only.
  - confidence: 0.91
- **[P2] Preserve ordered duplicate gitignore rules:** `src/infra/backup-exclude.ts:225-232`
  - body: This deduplication drops later duplicate patterns before handing the list to `ignore`, but gitignore rules are order-sensitive. A pattern sequence like `*.log`, `!important.log`, `*.log` should end with `important.log` ignored; removing the later duplicate flips the result and can include or exclude files differently from the user's declared rules.
  - confidence: 0.88
