## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate `.backupignore` behind an exclude opt-in:** `src/infra/backup-exclude.ts:190-194`
  - body: `resolveExcludePatterns` auto-loads `stateDir/.backupignore` on every backup create, even when the user passes no exclude flags. That breaks the stated compatibility guarantee because an existing file at that path can silently switch a normal backup into exclude mode and omit restorable state without an explicit CLI choice.
  - confidence: 0.9
- **[P2] Preserve duplicate rules in pattern order:** `src/infra/backup-exclude.ts:225-233`
  - body: This deduplication drops later repeated patterns before handing the list to `ignore`, but gitignore rules are order-sensitive. A sequence like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded; with the final duplicate removed, this patch changes the user's declared rule order and can include or exclude the wrong files.
  - confidence: 0.86
