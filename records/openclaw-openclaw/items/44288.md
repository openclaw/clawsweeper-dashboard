- **[P1] Preserve the hardened archive pipeline:** `src/infra/backup-create.ts:469-475`
  - body: This exclude path writes the tar directly and then gzips it, bypassing current main's retry wrapper, volatile-file skip filter, source-safe temp root, and installed-plugin dependency pruning.   - confidence: 0.91
- **[P2] Apply excludes to every archived root:** `src/infra/backup-exclude.ts:362-363`
  - body: Returning true for paths outside `baseDir` means exclude rules are ignored for valid external workspaces and custom config assets. The runtime warning still archives those trees in full, so users can leak or bloat backups despite an explicit `--exclude` or `.backupignore` rule.
  - confidence: 0.88
- **[P2] Keep ordered ignore rules intact:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns before calling `ignore` changes gitignore precedence when a duplicate rule intentionally appears after a negation, such as `*.log`, `!important.log`, then `*.log`. The backup should preserve the user's ordered rule list and track stats without collapsing semantics.
  - confidence: 0.86
- **[P2] Avoid rescanning every created archive:** `src/infra/backup-create.ts:528-535`
  - body: `listArchiveEntries` reads the entire finished archive into memory on every create path, adding a second full pass even when the user did not request `--verify`. On the large backups this feature targets, that can materially increase runtime and memory pressure.
  - confidence: 0.78
