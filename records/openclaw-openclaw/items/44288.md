## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Compose excludes with the existing backup filters:** `src/infra/backup-create.ts:473`
  - body: The exclude archive path passes only `excludeFilter` to `tar.c`, so enabling any exclude pattern bypasses current-main filtering for volatile live state and `extensions/*/node_modules`. Compose the new matcher with the existing backup filters before tar sees entries.
  - confidence: 0.91
- **[P2] Run exclude archive writes through the retry wrapper:** `src/infra/backup-create.ts:469-476`
  - body: The exclude path calls `tar.c` directly. Current main wraps backup tar writes in retry/error-enrichment logic for live-file EOF races, so exclude backups can fail once and abort instead of using the hardened mainline behavior.
  - confidence: 0.86
- **[P2] Keep the temp manifest outside source paths:** `src/infra/backup-create.ts:425-426`
  - body: This stages `manifest.json` under `os.tmpdir()`. Current main moved that staging root outside every backed-up asset because cron/sandbox environments can set `TMPDIR` under `~/.openclaw`, causing duplicate manifest entries and verify failures.
  - confidence: 0.88
- **[P2] Preserve duplicate ignore rules in order:** `src/infra/backup-exclude.ts:225-233`
  - body: Deduplicating patterns with first occurrence winning changes gitignore semantics, where later matching rules can intentionally override earlier ones. A set like `*.log`, `!important.log`, `*.log` should end with `important.log` excluded, but this drops the final rule.
  - confidence: 0.84
- **[P2] Apply exclude patterns to external backup assets:** `src/infra/backup-create.ts:383-387`
  - body: The branch now warns when external assets bypass excludes, but it still archives those workspaces or custom config roots in full. That leaves `--exclude` as a no-op for a documented backup source and for common workspace patterns like dependency directories.
  - confidence: 0.78
