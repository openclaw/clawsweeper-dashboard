## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve repeated exclude rules in declared order:** `src/infra/backup-exclude.ts:225-233`
  - body: `ignore` evaluates rules in order, so dropping later duplicates changes gitignore semantics whenever a negation sits between two identical excludes. For example, `*.key`, `!public.key`, `*.key` should end with `public.key` excluded, but this dedupe keeps only the first `*.key` and silently includes it in the backup.
  - confidence: 0.92
- **[P1] Apply excludes to each backup asset root:** `src/infra/backup-create.ts:364-368`
  - body: The filter is rooted at `canonicalStateDir`, and entries outside that root return true, so external workspaces or custom config/credential assets are archived in full even when they match `--exclude` or `.backupignore`. A warning does not make the requested exclude contract true and can still leave large or sensitive files in the archive.
  - confidence: 0.9
- **[P1] Remove the unconditional full archive rescan:** `src/infra/backup-create.ts:525-529`
  - body: After creating every backup, this reads the completed `.tar.gz` and keeps every entry path in memory before publish. That adds a second full decompression/list pass to the multi-gigabyte backup path the feature is meant to improve; keep this behind explicit verify or make the integrity check streaming/bounded.
  - confidence: 0.86
