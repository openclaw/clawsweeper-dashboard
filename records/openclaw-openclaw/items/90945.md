## Summary

Keep open: current main and v2026.6.8 still lack a gateway-wide startup or periodic sweep for stale SQLite `channel_ingress_events` claims, and the direct closing implementation at https://github.com/openclaw/openclaw/pull/90989 remains open and unmerged.

## What This Changes

The issue asks OpenClaw to recover stale `channel_ingress_events` SQLite claims after gateway/session crashes so Telegram DM ingress does not deadlock.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/90989, or an equivalent core channel/gateway fix, so stale SQLite ingress claims are recovered with age, owner, null-lane, restart-lifecycle, and claim-token race safeguards.

## Solution Assessment

Yes for the direction. The right owner is core channel/gateway SQLite ingress recovery, and this issue should stay open until the linked closing PR or an equivalent safe fix lands.
