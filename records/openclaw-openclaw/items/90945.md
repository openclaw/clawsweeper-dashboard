## Summary

Keep open: current main and latest v2026.6.6 still lack the generic startup SQLite ingress-claim sweep, and this issue is paired with an open proof-supplied PR that uses closing syntax, so closing before merge would be premature.

## What This Changes

Request recovery of stale `channel_ingress_events` SQLite claims after a session or gateway crash so Telegram inbound messages do not deadlock behind permanent `claimed` rows.

## Best Possible Solution
