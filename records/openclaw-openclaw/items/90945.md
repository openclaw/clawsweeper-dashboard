## Summary

Keep open. Current main and v2026.6.6 still have only per-queue stale-claim recovery, while the live linked implementation PR is open and marked as closing this issue, so the issue should remain the canonical bug record until that PR merges or maintainers choose a different recovery design.

## What This Changes

The issue requests recovery of stale SQLite channel ingress claims so Telegram DM ingress does not deadlock after a gateway or session crash.

## Best Possible Solution

Keep this issue open and use https://github.com/openclaw/openclaw/pull/90989 as the current implementation path, with maintainer review deciding whether startup-only recovery is enough or whether periodic/stronger owner-liveness handling is required.
