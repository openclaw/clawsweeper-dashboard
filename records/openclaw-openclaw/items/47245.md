## What This Changes

The branch changes only `ui/src/styles/components.css` to split command palette hover and active-row background colors by theme.

## Best Possible Solution

Land a focused CSS update that keeps hover and keyboard-active rows distinct in both themes, gives active selection precedence when states overlap, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source inspection shows current main uses one shared hover/active background, and the PR regression follows directly from CSS specificity when a row has both `:hover` and `.cmd-palette__item--active`.

## Solution Assessment

No, not as written. The CSS-only boundary is the right maintainable approach, but the active selectors need equal-or-higher specificity or explicit active-hover rules, plus the missing changelog entry.
