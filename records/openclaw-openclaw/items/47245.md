## Summary

Keep open: the PR addresses a real source-reproducible command palette styling ambiguity, but the current head still lets hover styling override the active-row styling when the selected row is under the cursor, and the external proof gate is still missing.

## What This Changes

The PR changes command palette CSS so hover and keyboard-active rows use separate theme-aware background colors.

## Best Possible Solution

Land a small CSS update that gives active rows equal or higher specificity than hover in all theme modes, preserves distinct states, and includes redacted real dashboard proof.

## Reproduction Assessment

Yes, at source level. Current main gives hover and active rows the same background while the command palette renders `.cmd-palette__item--active` from `activeIndex`, so the ambiguity is clear without running a browser.

## Solution Assessment

No, not as written. The CSS-only boundary is right, but the PR needs an active-hover or equal-specificity active override plus inspectable real behavior proof before merge.
