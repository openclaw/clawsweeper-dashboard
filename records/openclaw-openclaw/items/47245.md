## Summary

Keep open: current main still has the shared command palette hover/active background, and this PR targets the right surface but still needs real UI proof plus a CSS specificity fix before merge.

## What This Changes

The PR splits command palette hover and active row backgrounds into separate theme-aware `color-mix` rules in `ui/src/styles/components.css`.

## Best Possible Solution

Land a focused CSS update that gives hover and active distinct colors in both themes and ensures active styling wins when selectors overlap, backed by redacted light/dark Control UI proof.

## Reproduction Assessment

Yes, at source level. Current main has one shared hover/active background rule, and the command palette source applies active selection while hover can also match; I did not run the browser UI in this read-only review.

## Solution Assessment

No. Splitting the colors is the right direction, but this branch's hover selectors outrank active selectors when both apply and there is no artifact proving actual light/dark UI contrast.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
