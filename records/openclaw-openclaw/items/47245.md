Splits the dashboard command palette hover and active row background rules in `ui/src/styles/components.css` into theme-aware `color-mix` colors.

## Best Possible Solution

Land a focused CSS update that keeps hover and active colors distinct with active winning on overlap, backed by redacted light/dark Control UI proof.

## Reproduction Assessment

Yes, at source level. Current main uses one shared CSS rule for hover and active rows, and the command palette source applies active selection while pointer hover can also be present; I did not run the browser UI in this read-only review.

## Solution Assessment

No. Splitting the colors is the right direction, but active styling must win when hover and active coincide, and the PR still needs inspectable UI proof.
