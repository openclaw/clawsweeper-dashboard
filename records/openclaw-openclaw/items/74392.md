## Summary

Keep this PR open. Current main still skips manifest-less library extensions in packaged runtime-deps scan and doctor repair, while the PR is a relevant narrow implementation. It needs maintainer review for the intentional plugin-gate bypass and a required changelog entry before merge.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library extensions in runtime-deps selection, documents their gate exception, and adds bundled runtime-deps and doctor regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow change once maintainers accept or adjust the library-extension gate semantics, add the required changelog entry, and preserve the existing on-demand public-artifact runtime preparation path.

## Reproduction Assessment

Yes. A packaged root containing `dist/extensions/media-understanding-core/package.json` with `stageRuntimeDependencies: true` and no `openclaw.plugin.json` is skipped by current main's manifest-based selector, and doctor repair uses that selector.

## Solution Assessment

Unclear. The package.json fallback is the right narrow shape for manifest-less library packages, but the chosen bypass of plugin disable/deny gates is a product and supply-chain policy decision that should be explicitly accepted or narrowed before merge.
