## Summary

Keep open. Current main still lacks package.json discovery for manifest-less bundled library runtime deps, so the PR addresses a real packaged repair gap, but the latest diff is not landable because the new library fallback can bypass the post-config repair scoping added on main and the required changelog entry is missing.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` discovery for manifest-less bundled library extensions in packaged runtime-deps planning, plus bundled runtime-deps and doctor regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch after the package.json library fallback is constrained to the intended packaged doctor/unscoped repair path or made to honor `includeEnabledByDefaultPlugins: false`, with tests covering the main gap, plugin gates, post-config scoping, and the required changelog entry.

## Reproduction Assessment

Yes. The current-main gap is reproducible with a packaged `dist/extensions/media-understanding-core/package.json` that has `stageRuntimeDependencies: true` and no manifest; the PR regression is reproducible at source level by planning post-config repair with `includeEnabledByDefaultPlugins: false` and confirming the library package is not selected.

## Solution Assessment

No. The package.json fallback is the right direction, but the current placement is too broad for scoped post-config repair; a safer fix keeps doctor/unscoped packaged repair coverage without undoing the recent post-config scoping contract.
