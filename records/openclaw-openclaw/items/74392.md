## Summary

Keep this PR open, but do not merge it as-is. The manifest-less library runtime-deps gap is real, but the current patch still intentionally bypasses plugin disable/deny gates and documents a broader exception while maintainer discussion says that contract is unresolved.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library extension directories, adds bundled runtime-deps and doctor regression tests, and documents library-extension gate behavior.

## Best Possible Solution

Land this PR or an equivalent narrow change only after the library-extension contract is settled: package.json-only bundled library packages should be discoverable for the intended packaged scan and doctor repair paths, with consistent gate semantics in code, tests, and docs, and startup claims covered by a real bootstrap regression if startup prestage is in scope.

## Reproduction Assessment

Yes. A packaged-root fixture with `dist/extensions/media-understanding-core/package.json` declaring `stageRuntimeDependencies: true` and `sharp` reproduces the current-main configured scan/doctor gap; the PR’s added tests also reproduce the proposed gate-bypass behavior.

## Solution Assessment

No, not as-is. The package.json detection belongs near the shared runtime-deps selector, but returning true before disable/deny policy and documenting a startup-adjacent exception is not the narrowest maintainable fix without explicit maintainer approval and matching coverage.
