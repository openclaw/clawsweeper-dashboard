## Summary

Keep this PR open, but do not merge it as-is. Current main still has a real packaged runtime-deps coverage gap for manifest-less media library packages, while the PR intentionally bypasses explicit plugin disable/deny gates and still lacks the startup/bootstrap proof requested by the maintainer review.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library packages in runtime-deps scanning, plus doctor/runtime-deps tests and plugin documentation for the gate exception.

## Best Possible Solution

The desired end state is a narrow packaged runtime-deps repair where manifest-less OpenClaw-owned library packages with `stageRuntimeDependencies` are discoverable by the intended scan and doctor paths, with the allow/deny/global-disable policy explicitly chosen by maintainers, documented in the right section, and covered by scanner plus bootstrap tests if startup prestage is claimed.

## Reproduction Assessment

Yes. The review blockers reproduce by inspection of the PR head: the new early return selects a staged manifest-less package before policy checks, and the added tests assert inclusion under `plugins.enabled:false` and `plugins.deny`; I did not run tests because this review was required to stay read-only.

## Solution Assessment

No, not as currently written. The safer path is either to keep existing enable/deny gates and scope the package.json fallback to doctor/unscoped repair, or to make an explicit maintainer-approved library-package contract change with matching docs and a real startup/bootstrap regression.
