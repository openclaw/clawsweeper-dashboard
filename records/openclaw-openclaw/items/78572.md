## Summary

This looks like a real current-main Discord message-tool routing bug, not a cleanup close: the allowFrom-aware DM disambiguation exists in Discord-specific code, but the generic message-tool target resolver can normalize a bare allowed user snowflake to `channel:<id>` before delivery.

## What This Changes

The issue reports that Discord `message(action="send")` fails with `Unknown Channel` for a bare numeric DM target even though Discord DMs and channel resolution work.

## Best Possible Solution

Wire the Discord allowFrom-aware target resolver into the generic message-tool target path, preserving explicit `channel:<id>` behavior and adding regression coverage for a bare numeric target listed in effective `allowFrom`.

## Reproduction Assessment

Yes by source inspection, though not by live Discord execution: a bare numeric `target` is normalized through the generic message-tool path to `channel:<id>` before delivery, despite Discord docs promising an allowFrom DM exception.

## Solution Assessment

No current-main solution is complete for this path: the Discord-specific allowFrom resolver exists, but the registered generic target resolver does not delegate to it, so a narrow wiring/test fix is the maintainable direction.
