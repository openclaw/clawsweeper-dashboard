## Summary

Keep open: current main and the latest release still represent channel reloads as whole-channel restarts, while the direct closing PR remains open and conflicting.

## What This Changes

Request account-scoped gateway hot reload planning for `channels.<channel>.accounts.<accountId>.*` changes instead of restarting the whole channel.

## Best Possible Solution

Land or replace the direct account-scoped reload fix so the planner carries channel/account restart targets and full-channel restarts still win for channel-global or mixed changes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: a strict Telegram account path matches the broad `channels.telegram` prefix, becomes a channel-only restart plan, and the handler performs whole-channel stop/start.

## Solution Assessment

Is this the best way to solve the issue? Yes; carrying account-scoped restart targets through the generic gateway planner and handler is the narrow maintainable fix because the lower channel manager already accepts optional account ids.
