## What This Changes

The issue asks for account-scoped gateway reload planning so changes under channel account config restart only the affected account instead of the entire channel.

## Best Possible Solution

Land or replace the linked account-scoped reload fix so strictly account-scoped channel config edits call the existing account-scoped channel lifecycle, while global or mixed changes continue to use full-channel restarts.

## Reproduction Assessment

Yes at source level: current main maps `channels.<channel>.*` reload prefixes into channel-level restart actions and the handler restarts the whole channel, while the channel manager already accepts account-scoped start/stop calls. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the gateway reload planner and handler because the lower channel lifecycle already supports account-scoped start/stop. The current open PR is the right direction but must be refreshed, proven, or replaced before this issue can close.
