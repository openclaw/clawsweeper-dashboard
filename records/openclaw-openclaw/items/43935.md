## Summary

Keep open. Current main still routes account-scoped channel config changes into whole-channel restart targets, and the linked implementation PR is still open and unmerged, so this issue should remain open until that PR is repaired, landed, replaced, or closed.

## What This Changes

The issue requests account-scoped gateway hot reload for `channels.<channel>.accounts.<accountId>.*` changes instead of restarting the entire channel.

## Best Possible Solution

Land a corrected account-scoped reload implementation that preserves full-channel precedence and updates hot reload, secrets reload, rollback, and tests together.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix, the planner adds `telegram` to `restartChannels`, and the hot reload handler calls channel stop/start without an account id; the discussion also includes matching runtime logs.

## Solution Assessment

Yes. Account-scoped reload planning through the existing account-aware channel lifecycle is the narrow maintainable direction, provided every reload-plan consumer preserves current deferral, failure, and rollback semantics.
