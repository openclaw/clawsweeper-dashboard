## Summary

## What This Changes

The issue requests account-scoped gateway hot reloads for `channels.<channel>.accounts.<accountId>.*` config changes instead of restarting the whole channel.

## Best Possible Solution

Land or replace the linked fix with account targets carried through planner, plugin reload interactions, active-work deferral, `secrets.reload` rollback, focused tests, and redacted runtime proof.

Yes, source-level. `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's `channels.telegram` reload prefix, the plan only has `restartChannels`, and reporter logs show the matching whole-channel restart followed by Telegram send failures.

## Solution Assessment

Yes. Account-scoped restart planning is the right direction, but the linked PR must preserve active-work deferral and `secrets.reload` restart/rollback semantics before merge.
