## Summary

Keep this open. Current main still routes account-scoped channel config changes through whole-channel reload targets, and the linked open fix PR has closing syntax but has not merged.

## What This Changes

Request account-scoped gateway hot reload for `channels.<channel>.accounts.<accountId>.*` changes instead of restarting the whole channel.

## Best Possible Solution

Land a rebased fix that carries account-scoped restart targets through planner, hot reload deferral, plugin reload, and secrets.reload rollback paths, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection shows `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad `channels.telegram` reload prefix and current main plans only a whole-channel restart; the issue comments add matching runtime logs.

## Solution Assessment

Yes. Account-scoped restart planning is the narrow maintainable direction, but it needs to preserve the existing active-work deferral and secrets.reload restart/rollback behavior before merge.
