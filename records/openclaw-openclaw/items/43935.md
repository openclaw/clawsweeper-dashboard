## Summary

Keep this open: current main and v2026.5.20 still collapse account-scoped Telegram config edits into whole-channel hot reloads, and the linked closing PR is still open and conflicting. The issue discussion adds runtime logs showing Telegram send failures and dropped replies during that restart window.

## What This Changes

The issue asks for account-scoped gateway hot reloads when only `channels.<channel>.accounts.<accountId>.*` changes, instead of restarting the entire channel.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/43938 so account-only channel config patches use account-scoped lifecycle calls while global and mixed changes keep full-channel restart safety contracts.

## Reproduction Assessment

## Solution Assessment

Yes, account-scoped restart planning is the narrow maintainable direction. The issue should stay open until the linked PR or a replacement lands with current reload deferral, plugin reload, and secrets rollback contracts preserved.
