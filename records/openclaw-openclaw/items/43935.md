## Summary

Keep open: current main and the latest release still collapse account-scoped channel config edits into whole-channel restarts, and the linked same-repo fix PR is still open rather than merged.

## What This Changes

The issue requests gateway hot reloads to restart only the affected channel account for `channels.<channel>.accounts.<accountId>.*` config changes while preserving full-channel restarts for global or mixed changes.

## Best Possible Solution

Carry account ids through reload planning and handler execution so strictly account-scoped changes call account-scoped lifecycle hooks, while channel-global or mixed changes still use the existing full-channel restart path.

## Reproduction Assessment

Yes at source level: an account-scoped path like `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix, the planner stores only `restartChannels`, and the handler restarts the channel without an account id. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes on direction, but not landed yet. The narrow maintainable fix is account-scoped reload planning plus handler execution with full-channel precedence for global or mixed changes; the linked PR attempts that path but needs rebase/proof/review.
