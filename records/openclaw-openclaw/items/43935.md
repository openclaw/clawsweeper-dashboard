## Summary

Keep open. Current `main` still plans account-scoped Telegram config changes as whole-channel restarts, and an open linked PR explicitly closes this issue, so the issue should remain open until that PR or a corrected replacement lands.

## What This Changes

The issue asks for account-scoped gateway hot reload when only `channels.<channel>.accounts.<accountId>.*` changes, instead of restarting the whole channel.

## Best Possible Solution

Land a rebased or replacement version of https://github.com/openclaw/openclaw/pull/43938 that carries account-scoped restart targets through reload planning, hot reload, plugin reload, secrets reload/rollback, active-work deferral, and focused regression proof.

## Reproduction Assessment

Yes, source-reproducible. On current `main`, `channels.telegram.accounts.<id>.*` matches Telegram's broad `channels.telegram` prefix, becomes `restartChannels: {telegram}`, and is applied through whole-channel `stopChannel(name)` / `startChannel(name)` calls; the issue comment adds matching runtime logs.

## Solution Assessment

Yes, the maintainable direction is account-scoped reload planning using the existing account-aware channel lifecycle. The current open PR is the right target to review or replace, but the issue should not close before a corrected implementation lands.
