## Label Justifications

- `P1`: The issue describes a broken gateway/channel hot-reload path that can interrupt active Telegram traffic and drop in-flight replies.
- `impact:message-loss`: Reporter logs show `sendMessage` and final reply failures after an account-scoped config patch triggered a whole Telegram channel restart.

## Summary

Keep open: current main and the latest release still collapse account-scoped channel config changes into channel-wide restarts, and the direct closing PR remains open and conflicting rather than merged.

## What This Changes

The issue requests account-scoped gateway hot reloads so `channels.<channel>.accounts.<accountId>.*` changes restart only the affected account instead of the whole channel.

## Best Possible Solution

Land or replace the direct account-scoped reload fix so the planner carries channel/account restart targets and full-channel restarts still win for channel-global or mixed changes.

## Reproduction Assessment

Yes at source level: an account-scoped Telegram path matches the broad `channels.telegram` reload prefix, becomes a channel-only `restartChannels` entry, and the handler calls whole-channel stop/start. The reporter also supplied timestamped logs showing failed Telegram delivery after that restart.

## Solution Assessment

Yes, the requested account-scoped reload plan is the narrow maintainable fix because the lower channel manager already accepts an optional account id. The remaining path is PR review or replacement, not a separate issue close.
