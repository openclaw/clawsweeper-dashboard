## Summary

Keep open: current main and v2026.6.10 still collapse account-scoped channel config changes into whole-channel restarts, and the direct closing PR remains open and conflicting rather than merged.

## What This Changes

The issue asks for gateway hot reload to restart only the affected channel account for `channels.<channel>.accounts.<accountId>.*` config changes while preserving full-channel restarts for channel-global or mixed changes.

## Best Possible Solution

Land or replace the direct account-scoped reload fix so the planner carries channel/account restart targets and full-channel restarts still win for channel-global or mixed changes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: an account-scoped Telegram path matches the broad `channels.telegram` reload prefix, becomes a channel-only restart plan, and the handler performs whole-channel stop/start.

## Solution Assessment

Is this the best way to solve the issue? Yes; carrying account-scoped restart targets through the generic gateway planner and handler is the narrow maintainable fix because the lower channel manager already accepts optional account ids.
