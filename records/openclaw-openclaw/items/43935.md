## Summary

Keep open. Current main and the latest shipped release still route account-scoped channel config paths through channel-wide restart targets, and the open implementation PR that closes this issue has not merged.

## What This Changes

The issue asks the gateway hot-reload planner to restart only the affected channel account for `channels.<channel>.accounts.<accountId>.*` changes instead of restarting the whole channel.

## Best Possible Solution

Land a corrected account-scoped reload implementation that carries account targets through planner, hot reload, plugin reload, and secrets reload rollback paths while preserving full-channel precedence and active-work deferral.

## Reproduction Assessment

Yes. Source inspection shows `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix, enters `plan.restartChannels`, and is applied through channel-only stop/start calls; the issue discussion adds matching runtime logs.

## Solution Assessment

Yes. Account-scoped reload planning through the existing account-aware channel lifecycle is the narrow maintainable direction, but the open implementation needs to cover deferral and all reload-plan consumers before it is merge-ready.
