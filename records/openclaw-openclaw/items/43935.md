## What This Changes

The issue requests Gateway hot reload to restart only the affected channel account for `channels.<channel>.accounts.<accountId>.*` config changes instead of restarting the entire channel.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/43938 so account-only channel config patches use account-scoped lifecycle calls while global and mixed changes keep full-channel restart safety contracts.

## Reproduction Assessment

Yes, at source level: `channels.telegram.accounts.<id>.*` matches Telegram's broad reload prefix, the planner stores only `restartChannels`, and the handler restarts Telegram with no account id. The issue discussion adds matching runtime delivery-loss logs, but I did not run a live Telegram reproduction in this read-only pass.

## Solution Assessment

Yes, account-scoped restart planning is the narrow maintainable direction. The issue should stay open until the linked PR or a current-main replacement lands with reload deferral, plugin reload, and secrets rollback contracts preserved.
