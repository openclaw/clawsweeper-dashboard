## Summary

Keep open. Current main and the latest release still use channel-wide reload planning for account-scoped channel config paths, and the linked implementation PR is open but conflicting, so this should remain on the active fix/review path rather than be swept closed.

## What This Changes

The issue reports that account-scoped channel account config edits restart an entire channel instead of only the affected account, widening the disruption window for active traffic.

## Best Possible Solution

Land a current-base fix that carries account-scoped restart targets through the reload plan and handler, while preserving full-channel restart precedence for channel-global or mixed changes.

## Reproduction Assessment

Yes at source level: current main maps account-scoped channel config paths into channel-level restart actions, and the reload handler restarts the whole channel with no account id. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes in direction: account-scoped reload planning is the narrow maintainable fix because the lifecycle already supports account ids. A broader outbound queue/drain redesign is adjacent but not required to solve this issue's reload-granularity bug.
