## Summary

Keep open: current main still collapses account-scoped channel config changes into full-channel restarts, and an open linked PR is the active fix candidate rather than a reason to close the issue before merge.

## What This Changes

The issue asks for account-scoped gateway hot reloads so changes under channel account config restart only the affected channel account instead of the whole channel.

## Best Possible Solution

Land or replace the linked gateway reload fix so account IDs are carried through planning and handler execution, while preserving full-channel restart precedence for channel-global and mixed edits.

## Reproduction Assessment

Yes. Current-main source inspection shows account-scoped config paths still collapse to a channel-only restart plan, and the handler restarts the whole channel; the issue comment supplies concrete Telegram delivery-loss logs on that path.

## Solution Assessment

Yes. The best fix direction is narrow gateway reload granularity: carry affected account IDs through the reload plan and handler, and keep full-channel restart behavior for channel-global or mixed changes.
