## Summary

Keep open: current main and the latest release still route account-scoped channel config changes through channel-wide restart planning, and the direct fix candidate remains open rather than merged.

## What This Changes

The issue requests account-scoped gateway hot reloads for `channels.<channel>.accounts.<accountId>.*` changes so active traffic on other accounts is not disrupted by a full channel restart.

## Best Possible Solution

Land or replace the linked gateway reload fix so affected account ids are carried through planning and handler execution, while preserving full-channel restart precedence for channel-global and mixed edits.

## Reproduction Assessment

Yes, at source level. Current main maps account-scoped channel paths into a channel-only restart plan and the issue comment provides concrete Telegram delivery-loss logs on that path.

## Solution Assessment

Yes, the requested fix direction is the narrow maintainable one. The reload planner should carry account ids to the handler and use existing account-scoped lifecycle calls while keeping channel-wide restarts for global or mixed changes.
