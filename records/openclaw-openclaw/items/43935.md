## Summary

Keep this open. Current main still collapses account-scoped Telegram config changes into whole-channel restart targets, and the open closing PR has not merged.

## What This Changes

The issue asks the gateway hot-reload planner to restart only the affected channel account for `channels.<channel>.accounts.<accountId>.*` config changes instead of restarting the whole channel.

## Best Possible Solution

Land or replace the linked fix with account targets carried through the planner, active-work deferral, plugin reload interactions, `secrets.reload` rollback, focused regression coverage, and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection shows `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix and current main records only a whole-channel restart, while the issue comments provide matching runtime logs.

## Solution Assessment

Yes. Account-scoped restart planning is the narrow maintainable direction, but the open PR needs to preserve active-work deferral and `secrets.reload` semantics before merge.
