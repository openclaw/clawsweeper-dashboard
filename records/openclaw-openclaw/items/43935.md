## Summary

Keep open. Current main still represents channel reload targets as whole-channel `restartChannels`, and the already-open PR #43938 is the paired implementation path with closing syntax for this issue.

## What This Changes

The issue requests account-scoped reload planning for `channels.<channel>.accounts.<accountId>.*` config changes so only impacted channel accounts restart.

## Best Possible Solution

Carry account-scoped restart targets through the reload plan and all channel-restart consumers, with full-channel restart precedence for channel-global, plugin reload, or mixed changes.

## Reproduction Assessment

Yes. A source-level reproduction is clear: `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix, the planner adds `telegram` to `restartChannels`, and the handler calls channel stop/start without an account id; the discussion also includes matching runtime logs.

## Solution Assessment

Yes. Account-scoped reload planning is the narrow maintainable fix for the reported blast radius; Telegram reply retry or drain hardening should remain separate unless failures still reproduce after reload granularity is fixed.
