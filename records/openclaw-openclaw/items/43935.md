## Summary

Keep open: current main still collapses account-only channel config changes into channel-wide restarts, and the open closing PR #43938 is the correct implementation vehicle to review before this issue closes.

## What This Changes

The issue requests account-scoped gateway hot reloads for `channels.<channel>.accounts.<accountId>.*` changes instead of restarting the whole channel.

## Best Possible Solution

Review and land a narrow fix that carries account ids through reload planning and hot reload handling, while preserving channel-wide restarts for global or mixed channel config changes.

## Reproduction Assessment

Yes, from source inspection. On current main, `channels.telegram.accounts.engineer.dmPolicy` matches the Telegram `channels.telegram` reload prefix, records only `restartChannels=telegram`, and the hot reload handler calls channel-wide stop/start without an account id.

## Solution Assessment

Yes. Account-scoped restart planning is the narrow maintainable fix because the lower channel lifecycle already accepts optional account ids, and #43938 is already scoped to that behavior with full-channel precedence for mixed changes.
