## Summary

Keep open. Current main still collapses account-scoped channel config changes into whole-channel restart targets, and open PR #43938 uses closing syntax for this issue, so the issue should remain open until that implementation path lands, is replaced, or is closed.

## What This Changes

Use account-scoped hot reload for `channels.<channel>.accounts.<accountId>.*` changes instead of restarting the entire channel.

## Best Possible Solution

Carry account-scoped restart targets through reload planning and each restart consumer, while preserving full-channel precedence for channel-global, plugin reload, and mixed changes.

## Reproduction Assessment

Yes. A source-level reproduction is clear: `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix, the planner adds only `telegram` to `restartChannels`, and the hot reload handler calls channel stop/start without an account id.

## Solution Assessment

Yes. Account-scoped reload planning is the narrow maintainable fix for the reported blast radius; Telegram reply retry or drain hardening should remain separate unless failures still reproduce after reload granularity is fixed.
