## What This Changes

## Best Possible Solution

Refresh, review, and land a narrow account-scoped reload fix through https://github.com/openclaw/openclaw/pull/43938 or a replacement that preserves full-channel restart behavior for channel-global and mixed changes.

## Reproduction Assessment

Yes, source-reproducible: current main maps account-scoped channel config paths to `restartChannels` and applies them with whole-channel stop/start calls. I did not run a live Telegram reproduction in this read-only pass, but the reporter supplied concrete live log excerpts showing delivery loss on this path.

## Solution Assessment

Yes, the narrow maintainable fix is to extend reload planning/application to carry account ids to the existing account-aware channel lifecycle. The open linked PR is directionally aligned but needs rebase/conflict resolution and real behavior proof before it can be the merge path.
