## Summary

Keep open. Current main still collapses account-scoped channel config paths into whole-channel restart targets, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/43938 is still open and unmerged with prior review blockers.

## What This Changes

Gateway hot reload should restart only the affected channel account for `channels.<channel>.accounts.<accountId>.*` changes instead of restarting the whole channel.

## Best Possible Solution

Land a corrected account-scoped reload implementation that preserves full-channel precedence and updates hot reload, secrets reload, rollback, and regression tests together.

## Reproduction Assessment

Yes, at source level: `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix, the planner adds only `telegram` to `restartChannels`, and hot reload calls channel stop/start without an account id. The issue discussion also includes matching runtime logs from a real Telegram setup.

## Solution Assessment

Yes, account-scoped reload planning through the existing account-aware channel lifecycle is the narrow maintainable direction. The current open PR is not the best final form until it also preserves deferral and updates every reload-plan consumer, especially `secrets.reload`.
