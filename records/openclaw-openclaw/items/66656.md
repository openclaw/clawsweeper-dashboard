## Summary

Keep open: the branch is a plausible narrow hardening, but it only filters diagnostic transcript entries in the sync preview reader while current main also has an async reader feeding `sessions.list` rows. It also lacks after-fix real behavior proof, so it is not merge-ready.

## What This Changes

The branch adds `diagnostic.*` filtering to sync gateway last-message preview scanning, focused session-utils regression tests, and a changelog fix entry.

## Best Possible Solution

Refresh the branch so one shared diagnostic-entry predicate is used by both sync and async transcript-preview readers, add async regression coverage, and keep the separate display-name fallback work in https://github.com/openclaw/openclaw/pull/66544 separate.

## Reproduction Assessment

Yes, at source level: current main accepts an assistant-shaped diagnostic tail row because both preview readers only check `message.role`. I did not run a live gateway/WebChat reproduction in this read-only review.

## Solution Assessment

No. Filtering diagnostic transcript entries is the right narrow direction, but the current patch is incomplete because it does not cover the async reader used by session-list hydration.
