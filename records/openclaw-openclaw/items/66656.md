## Summary

Keep open. Current main still lacks diagnostic-entry filtering in the transcript last-message preview path, while this PR is a focused patch with matching regression coverage and no blocking review findings; the remaining action is maintainer review and landing, not cleanup closure.

## What This Changes

The PR filters `diagnostic.*` transcript entries out of session last-message preview extraction, adds regression coverage for diagnostic tail rows, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent focused patch so all `diagnostic.*` transcript rows are ignored before deriving `lastMessagePreview`, keep the regression tests and contributor changelog credit, and review #66544 separately for the display-name fallback path.

## Reproduction Assessment

Yes. A source-level reproduction is a transcript whose latest JSONL row has `type: "diagnostic.heartbeat"` and `message.role: "assistant"`; current main will return that row's content because it only checks `parsed.message.role`.

## Solution Assessment

Yes. Filtering the diagnostic namespace inside the shared transcript-preview helper is the narrowest maintainable fix because it protects gateway session listings and their UI consumers without changing normal user/assistant preview extraction.
