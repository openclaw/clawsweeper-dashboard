## Summary

Keep open: this PR targets a real source-reproducible shared-session regression, but the epoch shortcut can re-baseline an external same-window mutation once any co-tenant acquisition advances the epoch, and after-fix real behavior proof is still missing.

## What This Changes

The PR adds a process-local session write epoch to session write-lock acquisition, uses it to accept same-process co-tenant writes during the embedded prompt I/O window, documents the tradeoff, and adds focused tests.

## Best Possible Solution

Allow co-tenant progress with a lock-mediated generation or fingerprint design that still detects uncoordinated writes in the same prompt window, backed by focused regression coverage and real shared-session proof.

## Reproduction Assessment

Yes, source-reproducible: current main records a fingerprint at `releaseForPrompt()` and throws when the file stat changes during reacquire, while shipped shared-session writers can use the same session file during slow prompt I/O. I did not run a live heartbeat or channel-ingress reproduction in this read-only review.

## Solution Assessment

No. The epoch-only shortcut fixes the false positive for coordinated writers, but as written it also accepts an external lock-bypassing mutation in the same prompt window; a lock-mediated generation or fingerprint-chain design would be safer.
