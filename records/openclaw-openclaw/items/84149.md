## Summary

Keep open: the PR targets a valid P1 source-reproducible session-lock regression, but the epoch shortcut intentionally accepts same-window external bypass writes and still lacks real behavior proof.

## What This Changes

The branch adds a process-local session write epoch to tolerate in-process co-tenant transcript writes during the prompt lock-release window, plus focused tests and a tradeoff comment.

## Best Possible Solution

Allow lock-coordinated co-tenant writes using ownership-aware fingerprint or generation metadata that cannot be advanced by unowned file changes, with regression coverage for co-tenant-only, external-only, and mixed-window cases plus real runtime proof.

## Reproduction Assessment

Yes, as a source-level path: current main records a fingerprint at releaseForPrompt and rejects any fingerprint mismatch when withSessionWriteLock reacquires; a co-tenant lock-mediated append changes that fingerprint during the model I/O window. I did not run a live heartbeat/channel reproduction in this read-only review.

## Solution Assessment

No. The target behavior is right, but accepting all mismatches when the in-process epoch advanced is not the safest fix because it also accepts same-window external bypass writes; an ownership-aware design should preserve fail-closed external detection.
