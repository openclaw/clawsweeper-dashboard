## Summary

Keep this PR open, but not merge-ready: it targets a valid shared-session regression while the epoch shortcut intentionally weakens same-window takeover detection and still lacks real behavior proof.

## What This Changes

The PR adds a process-local session write epoch, uses it in embedded attempt session-lock fence checks, and adds tests for co-tenant and external write scenarios.

## Best Possible Solution

Allow coordinated co-tenant progress with a lock-mediated generation, fingerprint chain, or equivalent fence design that still fails closed on uncoordinated same-window mutations, backed by mixed-window regression coverage and real shared-session proof.

## Reproduction Assessment

Yes, source-reproducible: current main records a fingerprint at releaseForPrompt and throws when the file stat changes during reacquire, while shared-session writers can use the same session file during slow prompt I/O. I did not run a live heartbeat or channel-ingress reproduction in this read-only review.

## Solution Assessment

No. The epoch-only shortcut fixes the false positive for coordinated writers, but as written it also accepts an external lock-bypassing mutation in the same prompt window; a lock-mediated generation or fingerprint-chain design would be safer.
