## Summary

Keep open. Current main still lacks the requested post-subagent-completion/pre-announcement plugin interception point; the adjacent hooks are either routing-only or observation-only and do not expose completion content, delivery status, or announcement mutation semantics.

## What This Changes

This issue requests a public plugin hook that fires after subagent completion and before the parent completion announcement is delivered, with access to completion metadata and optional announcement enrichment, filtering, or suppression.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract that defines bounded child metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms the requested hook is absent and adjacent hooks cannot modify or observe the final completion announcement as requested.

## Solution Assessment

Unclear as a concrete API. A post-completion hook is directionally aligned with the plugin strategy, but maintainers should decide whether it belongs as a dedicated hook or a broader lifecycle/delivery contract.
