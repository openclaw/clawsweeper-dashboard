## Summary

Keep open. Current main still exposes only adjacent subagent observation and routing hooks, while the requested post-completion/pre-announcement plugin interception point remains absent and needs maintainer API design.

## What This Changes

Add a public plugin hook that fires after a subagent finishes and before its completion announcement is delivered, with access to child metadata/output and optional announcement mutation.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract that defines bounded child metadata/output access, delivery-result visibility, hook ordering, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms the requested hook is absent and the adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook is directionally aligned with the plugin strategy, but maintainers should decide whether it belongs as a dedicated hook or a broader lifecycle/delivery contract.
