## Summary

Keep open. Current main still lacks a public post-subagent-completion/pre-announcement interception hook; the adjacent hooks are observation/routing-only and the issue has recent maintainer and user follow-up saying the API gap remains.

## What This Changes

Request a new extension hook that fires after a spawned subagent completes and before the completion announcement is delivered, with access to completion context and optional announcement mutation/filtering.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract for post-subagent completion that defines bounded metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook matches OpenClaw's plugin direction, but maintainers should decide the exact lifecycle/delivery contract and security boundaries before implementation.
