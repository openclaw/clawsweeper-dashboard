## Summary

Keep open: current main and v2026.6.6 still expose only adjacent subagent observer/routing hooks, not the requested post-completion/pre-announcement plugin hook that can inspect results, observe delivery status, or enrich/suppress the announcement. The open related implementation PR is only a candidate slice and is not merged or clean enough to supersede the issue.

## What This Changes

Request a public extension hook that fires after a spawned subagent completes and before the parent-session completion announcement is delivered.

## Best Possible Solution

Keep this as the canonical design item for a bounded typed plugin hook, or a broader lifecycle/delivery contract, that covers post-subagent completion, safe payload access, delivery-result visibility, and explicit announcement mutation semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent subagent observer/routing hooks but no mutable post-completion/pre-announcement hook with result and delivery semantics. This is a missing feature/API surface rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API and security design. A typed plugin hook at this seam fits the project direction, but payload bounds, delivery-result visibility, mutation semantics, and the relationship to https://github.com/openclaw/openclaw/pull/56904 need maintainer approval before implementation.
