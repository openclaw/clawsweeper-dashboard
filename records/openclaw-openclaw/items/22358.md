## Summary

Keep open. Current main still has only adjacent subagent observation and routing hooks; the requested post-completion/pre-announcement plugin interception point is absent and needs maintainer API and security-boundary design.

## What This Changes

Add a plugin hook that fires after a subagent completes and before requester announcement delivery so plugins can inspect results, persist artifacts, and enrich, suppress, or filter the announcement.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract that defines bounded child metadata/output access, delivery-result visibility, hook ordering, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook is directionally aligned with the plugin strategy, but maintainers should decide whether it belongs as a dedicated hook or a broader lifecycle/delivery contract.
