## Summary

Keep open. Current main and v2026.6.5 still expose only adjacent subagent observation/routing hooks; the requested post-completion/pre-announcement plugin interceptor remains missing and needs maintainer API/security design.

## What This Changes

The issue requests a plugin extension hook that fires after a subagent completes and before the parent completion announcement is delivered, allowing plugins to inspect completion data, persist artifacts, and modify or suppress the announcement.

## Best Possible Solution

Design a narrow typed plugin lifecycle hook at the post-subagent-completion/pre-announcement boundary, or explicitly cover that contract in the broader lifecycle-hooks design.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; yes for source-level verification of the API gap. Current main and v2026.6.5 expose adjacent subagent hooks only, and the announce flow has no public hook before delivery.

## Solution Assessment

Unclear until maintainers choose the API contract. A typed hook at this boundary is plausible, but payload scope, delivery-result visibility, mutation/suppression rights, timeouts, and security boundaries need design first.
