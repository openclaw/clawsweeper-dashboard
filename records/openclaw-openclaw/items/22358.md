## Summary

Keep open. Current main still has adjacent subagent lifecycle and routing hooks, but no public post-completion/pre-announcement plugin hook that can inspect the finished child run, observe delivery outcome, or mutate the completion announcement.

## What This Changes

Request a public plugin hook that runs after a spawned subagent completes and before its completion announcement is delivered, allowing inspection, persistence, enrichment, filtering, or suppression.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract that defines bounded child metadata/output access, delivery-result visibility, hook ordering, timeout/error behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook is directionally aligned with OpenClaw's plugin strategy, but maintainers should decide whether it belongs as a dedicated hook or a broader lifecycle/delivery contract.
