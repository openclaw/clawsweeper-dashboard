## Summary

Keep open. Current main still lacks a public post-subagent-completion/pre-announcement interception hook; existing subagent hooks are observation or routing seams, and recent related discussion keeps this as the canonical API gap.

## What This Changes

Request a public extension hook after spawned subagents finish and before completion announcement delivery so plugins can inspect results, persist artifacts, and enrich or filter announcements.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract covering bounded child metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook matches the plugin direction, but maintainers should decide the lifecycle/delivery contract, payload limits, and security boundaries before implementation.

- The request overlaps broader lifecycle and delivery architecture work in https://github.com/openclaw/openclaw/issues/43454 and https://github.com/openclaw/openclaw/issues/66638, so the final API should align with those threads.
