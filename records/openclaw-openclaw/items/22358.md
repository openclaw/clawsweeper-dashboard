## Summary

Keep open. Current main and the latest checked release still expose only adjacent subagent observation/routing hooks; the requested post-subagent-completion/pre-announcement interception point is missing, and related discussion keeps this issue as the canonical tracker for that API gap.

## What This Changes

The issue requests a public extension hook that fires after a spawned subagent finishes and before the completion announcement is sent, allowing plugins to inspect, persist, enrich, suppress, or react to the completion event.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract covering bounded child metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, tests, and latest-release inspection give high-confidence evidence that the requested public interception hook is absent.

## Solution Assessment

Unclear as a concrete API. A post-completion hook fits the plugin direction, but maintainers need to choose payload limits, return semantics, delivery ordering, and security boundaries before implementation.
