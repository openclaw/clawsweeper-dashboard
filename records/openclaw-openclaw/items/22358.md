## Summary

Keep open. Current main and v2026.6.6 still only provide adjacent subagent observation/routing hooks, while the requested post-completion/pre-announcement interception point remains missing; the open guard-hook PR overlaps but is not merged and still has API/security review blockers.

## What This Changes

Request a plugin extension hook that fires after a spawned subagent finishes and before the parent completion announcement is sent, allowing inspection, enrichment, filtering, suppression, and post-run processing.

## Best Possible Solution

Design a maintainer-approved typed plugin lifecycle hook at the post-subagent-completion/pre-announcement boundary, or fold the same semantics into a canonical lifecycle/delivery API with minimized payloads, delivery-status visibility, bounded return semantics, docs, and targeted ordering/security tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose only adjacent subagent observation/routing hooks, and the announce flow has no public plugin interception before delivery. This is a missing capability rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. A post-completion/pre-announcement plugin hook is plausible, but the best solution needs owner decisions on payload minimization, sync versus async behavior, delivery-result visibility, and suppression semantics.

- The request overlaps broader lifecycle-hook work at https://github.com/openclaw/openclaw/issues/43454, so maintainers should choose a coherent plugin lifecycle model rather than adding an incompatible one-off hook.
