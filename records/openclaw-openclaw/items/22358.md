## Summary

Keep open. Current main still lacks a public post-subagent-completion/pre-announcement interception hook; the existing subagent hooks are observation or routing seams, and recent related discussion confirms this issue remains the canonical tracker for the API gap.

## What This Changes

Request a public plugin hook that fires after a subagent finishes and before its completion announcement so plugins can inspect output/history, persist artifacts, and enrich or filter delivery.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract covering bounded child metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook matches OpenClaw's plugin direction, but maintainers should decide the lifecycle/delivery contract and security boundaries before implementation.

## Risks / Open Questions

- The hook would expose child output or history and potentially delivery results, so the API needs explicit data-minimization, permission, timeout, error, and audit semantics.
- Allowing plugins to suppress or rewrite completion announcements could create silent-delivery regressions unless return semantics and ordering are designed carefully.
- The request overlaps broader lifecycle notification architecture in https://github.com/openclaw/openclaw/issues/66638, so maintainers should decide whether this remains a focused hook or part of a larger event-delivery contract.
