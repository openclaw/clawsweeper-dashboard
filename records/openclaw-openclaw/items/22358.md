## Summary

Keep open: current main and the latest v2026.6.6 release still expose only adjacent subagent observation/routing hooks, while this request asks for a mutable post-completion/pre-announcement plugin API. The related implementation at https://github.com/openclaw/openclaw/pull/56904 is useful context but is open, conflicting, maintainer-labeled, and not a safe replacement yet.

## What This Changes

The issue requests a plugin extension hook after subagent completion and before the parent completion announcement is sent.

## Best Possible Solution

Keep this issue open as the narrow canonical design item, then land either a bounded typed plugin hook at the post-subagent-completion/pre-announcement seam or a broader lifecycle/delivery API that explicitly covers child metadata, safe output access, delivery-result visibility, and announcement mutation semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent subagent_ended and subagent_delivery_target hooks but no mutable post-completion/pre-announcement hook. This is a missing feature/API surface, not a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API and security design. A typed plugin hook at this seam is plausible, but the best solution must settle payload bounds, delivery-result visibility, mutation semantics, and the relationship to broader lifecycle hook work at https://github.com/openclaw/openclaw/issues/43454.
