## Summary

Keep open: current main and v2026.6.6 still expose only adjacent subagent observation/routing hooks, while the requested post-completion/pre-announcement interception API remains missing and needs maintainer API/security design.

## What This Changes

The issue requests a public plugin extension hook that runs after a spawned subagent completes and before the parent completion announcement is delivered or surfaced.

## Best Possible Solution

Keep this issue open as the canonical maintainer API design thread for a narrow, typed post-subagent-completion/pre-announcement plugin hook or equivalent lifecycle/delivery contract with bounded payloads, explicit return semantics, docs, and focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 expose adjacent subagent observation and routing hooks, but not the requested post-completion/pre-announcement hook that can inspect delivery state or mutate the announcement.

## Solution Assessment

Unclear pending maintainer API design. A narrow typed plugin hook is plausible, but payload minimization, ordering, return semantics, failure handling, and security review need owner approval first.
