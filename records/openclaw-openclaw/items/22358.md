## Summary

Keep open. Current main still lacks a public post-subagent-completion/pre-announcement plugin hook, and the closest existing hooks remain observation/routing hooks that cannot inspect final child output, expose delivery outcome as a public plugin contract, or enrich/suppress the announcement before delivery. The discussion also confirms related delivery fixes improved symptoms without covering this API gap, so this needs maintainer API design rather than cleanup closure.

## What This Changes

Request a public plugin extension hook that runs after a spawned subagent finishes and before its completion announcement is delivered to the parent session.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract covering bounded child metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested public interception hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook fits the plugin direction, but maintainers should choose the lifecycle/delivery contract, payload limits, return semantics, and security boundaries before implementation.
