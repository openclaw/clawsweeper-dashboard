## Summary

Keep open. Current main still exposes only adjacent observation/routing subagent hooks; there is no public hook that can inspect completion output, observe delivery result, or modify/suppress the completion announcement, and prior maintainer discussion explicitly left this as API-design follow-up.

## What This Changes

The issue requests a public plugin hook that runs after subagent completion and before or around the parent completion announcement, with completion metadata plus announcement enrichment/filtering semantics.

## Best Possible Solution

Design a documented typed plugin lifecycle/delivery contract at the post-completion/pre-announcement point, with bounded metadata, safe access to completion output, delivery status/error visibility where appropriate, and explicit return semantics for enriching, suppressing, or filtering announcements.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection on current main shows the requested public hook does not exist while the existing hooks cannot exercise the requested announcement-interception behavior.

## Solution Assessment

Unclear whether the exact proposed API name and return shape are the best final design. The narrow maintainable path is a maintainer-designed public plugin contract that covers the same lifecycle point without leaking unsafe child data or creating ambiguous delivery semantics.
