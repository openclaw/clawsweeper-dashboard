## Summary

Keep open. Current main still exposes adjacent subagent lifecycle and routing hooks, but not the requested post-completion/pre-announcement plugin interception point; recent related cleanup also treats this issue as the canonical tracker for that API gap.

## What This Changes

The issue requests a typed plugin hook after subagent completion and before parent announcement delivery, with access to result/history/delivery context and optional announcement mutation.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract that defines bounded child metadata/output access, delivery-result visibility, hook ordering, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main confirms the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook is directionally aligned with the plugin strategy, but maintainers should decide whether it belongs as a dedicated hook or a broader lifecycle/delivery contract.
