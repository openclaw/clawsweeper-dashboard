## Summary

Keep open. Current main still has adjacent subagent lifecycle and routing hooks, but no public post-subagent-completion/pre-announcement hook that can inspect the finished run, observe delivery outcome, or enrich/suppress the completion announcement.

## What This Changes

Requests a public hook after subagent completion and before parent-session announcement delivery so plugins can inspect output, persist artifacts, and modify or filter the announcement.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract covering bounded child metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook fits the plugin direction, but maintainers should choose the lifecycle/delivery contract, payload limits, and security boundaries before implementation.
