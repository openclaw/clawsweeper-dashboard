## Summary

Keep open. Current main still lacks the requested public post-subagent-completion/pre-announcement hook; adjacent lifecycle and routing hooks cannot inspect final output, observe delivery outcome as a public contract, or enrich/suppress the completion announcement.

## What This Changes

Request to add a plugin extension hook that runs after a spawned subagent finishes and before the parent completion announcement is delivered.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract covering bounded child metadata/output access, delivery-result visibility, hook ordering, failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested public interception hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook fits the plugin direction, but maintainers should choose the lifecycle/delivery contract, payload limits, return semantics, and security boundaries before implementation.
