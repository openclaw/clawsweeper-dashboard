## Summary

Keep open. Current main still exposes only adjacent subagent observation and routing hooks; none provide a post-completion/pre-announcement interception point, and the discussion already notes related delivery fixes did not cover this API gap.

## What This Changes

Request a public plugin hook that runs after subagent completion and before parent announcement delivery, with child result metadata and optional announcement enrichment or filtering.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract that defines bounded child metadata/output access, delivery-result visibility, hook ordering, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms the requested hook is absent and adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook fits the plugin direction, but maintainers should decide whether this is a dedicated hook or part of a broader lifecycle/delivery contract.
