## Summary

Keep open: current `main` and the latest `v2026.6.6` release still expose only adjacent subagent observation/routing hooks, while this request asks for a mutable post-completion/pre-announcement plugin API. The live related implementation at https://github.com/openclaw/openclaw/pull/56904 is useful context but is open, conflicting, protected by maintainer review, and not a safe replacement yet.

## What This Changes

The issue requests a public post-subagent-completion plugin hook that can inspect a finished child run and optionally affect the completion announcement before it is surfaced upstream.

## Best Possible Solution

Keep this issue open as the narrow canonical design item, then land either a bounded typed plugin hook at the post-subagent-completion/pre-announcement seam or a broader lifecycle/delivery API that explicitly covers child metadata, safe output access, delivery-result visibility, and announcement mutation semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent `subagent_ended` and `subagent_delivery_target` hooks but no mutable post-completion/pre-announcement hook. This is a missing feature/API surface, not a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API and security design. A typed plugin hook at this seam is plausible, but the best solution must settle payload bounds, delivery-result visibility, mutation semantics, and the relationship to the broader lifecycle API request at https://github.com/openclaw/openclaw/issues/43454.
