## Summary

Keep open: current main and v2026.6.6 still expose only adjacent subagent observation/routing hooks, while the requested post-completion/pre-announcement plugin interception point is still missing; the linked implementation PR is relevant but open, conflicting, protected, and still under API/security review.

## What This Changes

Request to add a programmatic plugin hook after a subagent completes and before completion announcement delivery, allowing inspection, post-processing, and announcement mutation or suppression.

## Best Possible Solution

Approve one typed plugin lifecycle/delivery contract for post-subagent completion that defines bounded metadata, safe completion output or history access, timing/usage fields, delivery status visibility, and return semantics for enrichment or suppression, then update docs and focused hook/announce tests together.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent subagent lifecycle and routing hooks but no post-completion/pre-announcement plugin hook with payload mutation or delivery-result semantics. This is a missing capability rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API/security design. A post-completion/pre-announcement hook is plausible, but the best solution depends on approved payload bounds, sync/async behavior, failure handling, suppression semantics, and alignment with broader lifecycle hook work.
