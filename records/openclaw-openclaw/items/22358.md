## Summary

Keep open. Current main and v2026.6.6 still expose only adjacent subagent lifecycle and routing hooks; the requested post-completion, pre-announcement interception point remains a real public plugin API gap that needs maintainer API and security design.

## What This Changes

Request a typed extension hook that fires after a subagent finishes and before its completion announcement is delivered, with access to completion metadata and possible announcement enrichment or suppression.

## Best Possible Solution

Keep this as the focused design tracker for a public typed plugin hook or canonical lifecycle/delivery contract that covers safe child metadata, approved completion output access, delivery result visibility, and explicit announcement mutation semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent subagent hooks, but no post-completion/pre-announcement hook that can inspect or mutate the completion announcement. This is a missing capability, not a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. A typed plugin hook at this seam is plausible, but the maintainable solution needs agreed payload limits, ordering, delivery-status semantics, and failure handling before implementation.
