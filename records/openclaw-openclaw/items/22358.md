## Summary

Keep open. Current main still lacks a public post-subagent-completion/pre-announcement hook; the existing subagent hooks cover spawn, routing, spawned, and ended observation only, and the issue discussion confirms related delivery fixes did not cover this extension point.

## What This Changes

Request a public plugin hook that fires after a subagent completes and before the parent completion announcement is sent, with access to child result metadata and optional announcement mutation or filtering.

## Best Possible Solution

Design a typed public plugin SDK hook, or broader lifecycle/delivery hook design, at the post-completion/pre-announcement boundary with bounded data exposure, clear delivery-status visibility, return semantics, docs, and ordering regression tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. The high-confidence verification path is source inspection of the current hook catalog, hook runner, and announce flow, which shows no public post-completion/pre-announcement interception point.

## Solution Assessment

Unclear as a final API, but the requested direction belongs in the core plugin SDK rather than a prompt workaround. Maintainer-led design is the safest next step because the hook would expose child-run data and change announcement delivery behavior.
