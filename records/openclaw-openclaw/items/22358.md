## Summary

Keep open. Current main still has only adjacent subagent lifecycle/routing hooks, and the issue discussion explicitly notes that later delivery fixes improved symptoms without adding the requested post-completion/pre-announcement extension point.

## What This Changes

The issue requests a public post-subagent-completion hook that can inspect a finished child run and optionally enrich, suppress, or filter the parent-facing completion announcement.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery hook, or a broader lifecycle API, that covers bounded child metadata/output access, delivery status visibility, and explicit announcement mutation or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms the requested hook is absent and the adjacent hooks cannot modify the completion announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook is directionally aligned with OpenClaw's plugin API strategy, but maintainers should decide whether it should be a dedicated hook or part of a broader lifecycle/delivery contract.
