## Label Justifications

- `P2`: This is a useful plugin API improvement with production workflow impact, but it requires maintainer design rather than urgent regression repair.
- `impact:session-state`: The requested hook would expose and act on subagent completion state, child run metadata, and potentially session-derived output.
- `impact:security`: The requested API could expose conversation-derived child data to plugin code and allow modification or suppression of completion delivery.
- `impact:message-loss`: One stated use case is detecting failed completion announce delivery and triggering fallback notification for long-running subagent work.

## Summary

Keep open. Current main and the latest release still expose adjacent subagent observation and routing hooks only; the requested post-subagent-completion/pre-announcement interception point is missing and needs maintainer API, product, and security design.

## What This Changes

Add a public extension hook after a spawned subagent finishes and before the parent completion announcement so plugins can inspect the child run, persist artifacts, react to delivery state, and enrich, suppress, or filter the announcement.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract covering bounded child metadata or approved output access, delivery-result visibility, hook ordering, timeout and failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main and v2026.5.12 gives high-confidence evidence that the requested public completion-interception hook is absent and adjacent hooks cannot modify the announcement.

## Solution Assessment

Unclear as a concrete API. A post-completion hook fits OpenClaw's plugin direction, but maintainers need to choose payload limits, return semantics, ordering, delivery-failure visibility, and security boundaries before implementation.
