## Label Justifications

- `P2`: This is a normal-priority plugin API feature with real production workflow impact, but it needs maintainer API/security design rather than an urgent bug fix.
- `impact:session-state`: The request is meant to let plugins inspect child run/session state and persist trajectories, summaries, memory, or learning artifacts after subagent completion.
- `impact:security`: The hook could expose sensitive child output or transcript context and let trusted plugins modify or suppress delivery, making the API boundary security-sensitive.
- `impact:message-loss`: One explicit use case is detecting failed completion announcements and sending fallback notifications before users lose visibility into finished subagent work.

## Summary

Keep open: current main and v2026.6.6 still expose only adjacent observation/routing hooks, while the requested post-completion/pre-announcement plugin API remains missing and security-sensitive. The open guard-hook PR overlaps this request but is conflicting, maintainer-labeled, and not a safe canonical replacement yet.

## What This Changes

Request a public extension hook after a spawned subagent completes and before the completion announcement is delivered so plugins can inspect, persist, enrich, filter, or suppress the completion output.

## Best Possible Solution

Approve a narrow typed plugin lifecycle/delivery contract for subagent completion, with bounded payloads, explicit trust permissions, deterministic return semantics, docs, and focused announce/hook regression tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent subagent observation/routing hooks but no mutable post-completion/pre-announcement hook. This is a missing feature/API surface, not a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API/security design. A typed plugin hook at this seam is plausible and vision-aligned, but payload bounds, delivery-result visibility, mutation semantics, and the relationship to https://github.com/openclaw/openclaw/pull/56904 need maintainer approval first.
