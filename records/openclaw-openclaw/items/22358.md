## Summary

Keep open. Current main and v2026.6.5 still expose adjacent subagent observation/routing hooks only; the requested mutable post-completion/pre-announcement plugin extension point is still missing and needs maintainer API/security design.

## What This Changes

Requested a plugin hook that fires after a spawned subagent finishes and before the parent completion announcement is delivered, with access to inspect, enrich, suppress, or persist completion data.

## Best Possible Solution

Keep this open for maintainer API design: either add a typed plugin hook at the post-subagent-completion/pre-announcement point or define a broader lifecycle/delivery contract that covers bounded child/run metadata, safe completion access, delivery status, and announcement mutation semantics.

## Reproduction Assessment

Yes for the source-level API gap: current main exposes adjacent subagent lifecycle and routing hooks, but no hook that can inspect completion output and modify or suppress the parent announcement before delivery.

## Solution Assessment

Unclear as a concrete implementation. The requested hook is plausible, but the best durable solution needs maintainer API design because it defines sensitive plugin data access and message-delivery mutation semantics.

## Risks / Open Questions

- The requested hook would expose child completion output or history to plugins, so the API needs an explicit privacy, permission, and redaction boundary.
- Allowing plugins to enrich, suppress, or filter announcements could create silent-delivery regressions unless return semantics, timeout behavior, and error handling are designed deliberately.
- The design overlaps with broader lifecycle and event-delivery work at https://github.com/openclaw/openclaw/issues/43454 and https://github.com/openclaw/openclaw/issues/66638, so maintainers should avoid locking in a narrow contract that conflicts with the eventual lifecycle model.
