## Summary

Keep open: current main and v2026.6.8 still expose only adjacent subagent observation/routing hooks, while this request asks for a mutable post-completion/pre-announcement plugin API. The closest implementation PR is open and review-blocked, so it is not a merged replacement.

## What This Changes

The issue requests a plugin extension hook that runs after a subagent completes and before its completion announcement is delivered or surfaced upstream.

## Best Possible Solution

Keep this as the narrow canonical design item, then land either a bounded typed plugin hook at the post-subagent-completion/pre-announcement seam or a broader lifecycle/delivery API that explicitly covers safe child metadata, output access, delivery-result visibility, and announcement mutation semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 expose adjacent subagent observation/routing hooks but no mutable post-completion/pre-announcement hook. This is a missing feature/API surface, not broken documented behavior.

## Solution Assessment

Unclear pending maintainer API and security design. A typed plugin hook at this seam is plausible, but the best solution must settle payload bounds, delivery-result visibility, mutation semantics, and overlap with broader lifecycle hook work at https://github.com/openclaw/openclaw/issues/43454.
