## Summary

Keep open: current main and v2026.6.6 still expose only adjacent observation/routing hooks, while this issue asks for a programmable post-subagent-completion/pre-announcement interception point with announcement mutation or suppression semantics.

## What This Changes

Request a public extension hook that fires after a subagent finishes and before its completion announcement is sent, allowing plugins to inspect the result and enrich, filter, or suppress the announcement.

## Best Possible Solution

Design one typed plugin lifecycle/delivery contract for the post-subagent-completion/pre-announcement point, with bounded payloads, explicit trust gates, clear delivery-outcome visibility, return semantics for enrichment or suppression, docs, and focused agent/plugin tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 lack a post-subagent-completion/pre-announcement hook or equivalent mutable delivery interception point. The existing subagent_ended hook is observation-only, and subagent_delivery_target only changes routing.

## Solution Assessment

Unclear pending maintainer API design. A typed plugin hook at this point is plausible, but the maintainable solution must define payload bounds, trust gates, delivery outcome visibility, and suppression/enrichment semantics before implementation.
