## Summary

Keep open. Current main and v2026.6.5 expose adjacent subagent observation and routing hooks, but they still do not provide the requested post-completion/pre-announcement plugin interception point for inspecting child output/history, observing delivery failure, or enriching/suppressing the parent announcement.

## What This Changes

Request a public extension hook that runs after a subagent completes and before the completion announcement is surfaced to the parent session.

## Best Possible Solution

Design a narrow typed plugin lifecycle hook at the post-subagent-completion/pre-announcement boundary, or fold that exact contract into the broader lifecycle-hook design with safe data access, explicit return semantics, docs, and ordering tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence way to verify the gap: current main and v2026.6.5 only provide observation/routing hooks, not a post-completion/pre-announcement interceptor.

## Solution Assessment

Unclear until maintainers choose the API contract. A typed plugin hook at this boundary is plausible and aligned, but the payload, delivery-status visibility, mutation rights, and security model need explicit product/API design.
