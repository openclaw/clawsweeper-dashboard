## Summary

Keep open. Current main and v2026.6.6 still expose only adjacent observation/routing hooks, while the requested post-subagent-completion hook needs maintainer API and security design before implementation.

## What This Changes

The issue requests a typed extension hook that fires after a subagent finishes and before the parent completion announcement is sent, with access to completion metadata and return semantics for enrichment, suppression, or filtering.

## Best Possible Solution

Define one public lifecycle/delivery hook contract with bounded payloads, explicit trust requirements, delivery-status visibility where appropriate, return semantics for enrichment or suppression, docs, and focused ordering/regression tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose only adjacent observation/routing hooks, and the announce flow has no public completion-payload interception point. This is a missing extension point, not a failing documented behavior path.

## Solution Assessment

Unclear until maintainer API/security review. A typed plugin hook near post-completion or announcement preparation appears to be the right boundary, but payload limits, mutability, delivery-result visibility, and failure semantics need a product decision.
