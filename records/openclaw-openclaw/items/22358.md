## Label Justifications

- `P2`: This is a useful plugin API improvement with production workflow impact, but it requires maintainer design rather than urgent regression repair.
- `impact:session-state`: The requested hook would observe and act on subagent completion state, child run metadata, and potentially session-derived output.
- `impact:security`: The requested API could expose conversation-derived child data to plugin code and allow modification or suppression of completion delivery.
- `impact:message-loss`: One stated use case is detecting failed completion announce delivery and triggering fallback notification for long-running subagent work.

## Summary

Keep open. Current main and the latest release still expose adjacent subagent observation and routing hooks only; the requested post-subagent-completion/pre-announcement interception point is missing, and the discussion shows related delivery fixes did not cover this API gap.

## What This Changes

Request a public plugin extension hook that fires after a subagent finishes and before the parent completion announcement is sent, with access to completion context and optional announcement modification.

## Best Possible Solution

Design a typed public plugin lifecycle/delivery contract that covers bounded child metadata or approved output access, delivery-result visibility, hook ordering, timeout and failure behavior, and announcement enrichment or suppression semantics with docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, tests, and v2026.5.18 show the requested public completion-interception hook is absent while adjacent observation and routing hooks exist.

## Solution Assessment

Unclear as a concrete API. A post-completion hook fits OpenClaw's plugin direction, but maintainers need to choose payload limits, return semantics, ordering, delivery-failure visibility, and security boundaries before implementation.
