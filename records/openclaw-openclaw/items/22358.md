## Summary

Keep open: current main and v2026.6.10 still expose only adjacent subagent observation/routing hooks, while this request needs a mutable post-completion/pre-announcement plugin API. The closest implementation candidate remains open and review-blocked, so it is not a merged replacement.

## What This Changes

The issue requests a public hook after spawned subagents finish and before completion announcements are delivered, with access to child run state and optional announcement enrichment, filtering, or suppression.

## Best Possible Solution

Keep this as the narrow canonical design item, then land either a bounded typed plugin hook at the post-subagent-completion/pre-announcement seam or a broader lifecycle/delivery API that explicitly covers safe child metadata, output access, delivery-result visibility, and announcement mutation semantics.
