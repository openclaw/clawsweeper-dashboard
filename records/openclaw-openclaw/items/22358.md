## Summary

Keep open: current main and v2026.6.6 still expose only adjacent observation/routing hooks, while the requested post-subagent-completion/pre-announcement interception API remains a maintainer product and security design item.

## What This Changes

Requests a public extension hook that fires after a spawned subagent completes and before the completion announcement is delivered to the parent session.

## Best Possible Solution

Keep this open for a maintainer-approved plugin lifecycle/delivery API that covers post-completion observation plus bounded announcement mutation or delivery-failure visibility with tests and docs.

## Reproduction Assessment

Yes at source level: current main exposes adjacent subagent hooks but not a post-completion/pre-announcement plugin interception point. This is a missing capability rather than a broken existing behavior.

## Solution Assessment

Unclear pending maintainer API design. A typed plugin lifecycle hook is plausible, but payload bounds, delivery status visibility, ordering, suppression semantics, and security policy need maintainer approval first.
