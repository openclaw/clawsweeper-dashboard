## Summary

Keep open. The PR targets a real Control UI typing-lag path, but it is incomplete against current main because ordinary textarea input still schedules host updates through the slash-menu refresh path, and the current history-key handler also needs explicit refresh handling once the draft is non-reactive.

## What This Changes

The PR removes Lit `@state()` from `OpenClawApp.chatMessage` and adds explicit draft-sync `requestUpdate()` calls around abort, failed-send restore, and history recall paths.

## Best Possible Solution

Land a narrow Control UI fix that keeps ordinary draft typing out of full Lit host renders, gates slash-menu/accessibility refreshes to actual visible state changes, explicitly refreshes programmatic draft changes, and adds focused regression coverage for plain typing, slash-menu updates, history recall, abort, and failed-send restore.

## Reproduction Assessment

Yes. The code-level reproduction path is clear: type in the Control UI textarea and `handleInput` calls `updateSlashMenu`, which unconditionally invokes the host update callback even for plain non-slash text.

## Solution Assessment

No. Removing `@state()` from `chatMessage` is a reasonable direction, but this PR is not the best current-main fix as-is because another per-keystroke host-update path remains and the newer history-key controller path is not updated.

  - body: Removing `@state()` from `chatMessage` will not stop current-main typing from re-rendering the host: every textarea input still calls `updateSlashMenu(...)`, and that helper unconditionally calls `props.onRequestUpdate` even for plain text. The `props.onHistoryKeydown` branch mutates the draft through the history controller and returns without calling `requestUpdate`, so ArrowUp/ArrowDown recall can leave the textarea value stale after this PR is rebased.
  - confidence: 0.84

  - body: The disconnected active-run abort path clears `host.chatMessage` and sets `pendingAbort`, but `pendingAbort` is not a reactive app field. Once the draft is non-reactive, pressing Stop while disconnected can leave the visible composer draft stale unless this branch also schedules an explicit update.
