## Summary

Keep open: the optimization direction is useful, but the branch is conflicting against current main, still misses current WebChat input/render paths, and has no after-fix browser proof.

## What This Changes

The branch removes Lit `@state()` from WebChat `chatMessage` and adds explicit `requestUpdate()` calls for abort, failed-send restore, and older inline history-recall paths.

## Best Possible Solution

Land or replace this with a current-main WebChat fix that keeps ordinary typing outside host state, guards slash-menu updates, refreshes all programmatic draft mutations, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes, at source level: current main routes textarea input through `updateSlashMenu` and `onDraftChange`, and `onDraftChange` mutates Lit `@state()` draft state. I did not run a live browser timing trace in this read-only review.

## Solution Assessment

No, not as-is. Making the draft non-reactive is a plausible direction, but the branch must be rebased and repaired for current slash-menu, history, optional update-hook, and draft-clear paths before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard the plain-text input update path:** `ui/src/ui/app.ts:157-159`
  - body: Removing `@state()` from `chatMessage` does not stop ordinary typing from scheduling renders because `handleInput` still calls `updateSlashMenu`, and current main's plain-text branch unconditionally calls `requestUpdate()`. Guard that call so it fires only when slash-menu state actually changes.
  - confidence: 0.9
- **[P2] Refresh the current history navigation path:** `ui/src/ui/views/chat.ts:1065-1077`
  - body: The added refreshes are on an older inline history branch. Current main routes ArrowUp and ArrowDown through `onHistoryKeydown` and `navigateChatInputHistory`, so that current path also needs a refresh after `chatMessage` becomes non-reactive or recalled drafts can stay invisible.
  - confidence: 0.88
- **[P2] Keep requestUpdate optional on ChatHost:** `ui/src/ui/app-chat.ts:83-86`
  - body: `ChatHost` declares `requestUpdate` as optional, but the patch casts every host to `OpenClawApp` and calls it directly during abort and failed-send restore. Use `host.requestUpdate?.()` or make the contract required with matching tests so non-app hosts do not throw.
  - confidence: 0.86
- **[P2] Refresh detached-send draft clears:** `ui/src/ui/app.ts:157-159`
  - body: Once `chatMessage` is non-reactive, `clearSubmittedComposerState` can clear a detached-send draft without any DOM refresh when no other reactive property changes. Add an explicit refresh for that clear/restore path so submitted text does not remain visible.
  - confidence: 0.78
