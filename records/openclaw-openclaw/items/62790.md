## Summary

Keep open: the PR targets a real WebChat responsiveness bug, but the branch still has blocking correctness gaps against current main and lacks required after-fix browser proof from the contributor.

## What This Changes

The PR removes Lit `@state()` from `OpenClawApp.chatMessage` and adds explicit `requestUpdate()` calls for abort, failed-send restore, and older inline history recall paths.

## Best Possible Solution

Land or replace a current-main WebChat fix that keeps ordinary draft typing out of Lit state, avoids plain-text slash-menu renders, preserves explicit programmatic draft sync, and includes focused tests plus real browser proof.

## Reproduction Assessment

Yes, at source level. Current main wires textarea input to `onDraftChange`, which assigns `chatMessage` while it is Lit `@state()`, and the slash-menu helper also requests updates during ordinary plain-text typing.

## Solution Assessment

No as-is. Making the draft non-reactive is a plausible fix direction, but this patch needs current-main history coverage, safer update-hook usage, the remaining slash-menu and detached-send render paths fixed, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard slash-menu renders for plain text:** `ui/src/ui/app.ts:157-159`
  - body: After this change, ordinary input still calls `updateSlashMenu()`, whose non-command branch calls `requestUpdate()`, so typing plain text can still schedule host renders. Update the slash-menu path to request an update only when visible slash-menu state actually changes.
  - confidence: 0.9
- **[P2] Refresh the current history-key path:** `ui/src/ui/views/chat.ts:1068-1077`
  - body: The lines updated here are the older inline `inputHistory` branch. Current main routes ArrowUp/ArrowDown through `onHistoryKeydown` and `navigateChatInputHistory`, which mutates `chatMessage`; after `chatMessage` is non-reactive that path also needs a refresh so recalled text appears.
  - confidence: 0.88
- **[P2] Call the optional update hook safely:** `ui/src/ui/app-chat.ts:83-86`
  - body: `ChatHost` still declares `requestUpdate` as optional, so casting every host to `OpenClawApp` and calling it can throw in tests or non-app hosts during abort or failed-send restore. Use `host.requestUpdate?.()` or make the contract required with matching tests.
  - confidence: 0.86
- **[P2] Refresh detached send draft clears:** `ui/src/ui/app-chat.ts:132`
  - body: `clearSubmittedComposerState` clears `chatMessage` before detached `/btw` sends, and a successful detached send may not mutate another reactive property. Once `chatMessage` is non-reactive, add an explicit refresh for that clear or the submitted text can remain visible.
  - confidence: 0.78
