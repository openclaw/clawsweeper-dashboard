## Summary

## What This Changes

The PR removes Lit reactivity from the WebChat draft field and adds explicit `requestUpdate()` calls around selected abort, failed-send, and history-recall draft mutations.

## Best Possible Solution

Land or replace a current-main WebChat fix that keeps ordinary typing out of Lit host state, guards slash-menu renders, preserves programmatic draft sync, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes, at source level: current main wires textarea input through `updateSlashMenu` and `onDraftChange`, and `onDraftChange` assigns the Lit `@state()` draft value. I did not run a live browser timing trace in this read-only review.

## Solution Assessment

No, not as-is. Making the draft non-reactive is plausible, but the PR must be rebased and repaired for current slash-menu, history, optional update-hook, and draft-clear paths before merge.
