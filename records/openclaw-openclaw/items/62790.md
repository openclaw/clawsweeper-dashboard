## Summary

Keep open. Current main still has the WebChat per-keystroke render source path, but this PR is merge-conflicting, lacks required after-fix real behavior proof, and misses current-main slash-menu/history behavior that a safe fix must preserve.

## What This Changes

The PR removes Lit `@state()` from the WebChat draft field and adds explicit update requests for some programmatic draft changes.

## Best Possible Solution

Land or replace a current-main WebChat fix that stops ordinary draft typing from triggering host renders, guards slash-menu updates, safely invokes optional update hooks, preserves history sync, and includes real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main still wires textarea input to `onDraftChange`, which assigns a Lit `@state()` `chatMessage`, and `updateSlashMenu` also requests an update on ordinary input; I did not run a live browser performance capture.

## Solution Assessment

No as-is. Making the draft non-reactive is a plausible direction, but this branch is stale/merge-conflicting, calls an optional update hook unsafely, leaves slash-menu renders on ordinary typing, and updates an older history path instead of the current `onHistoryKeydown` path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Call the optional update hook safely:** `ui/src/ui/app-chat.ts:83-86`
  - body: `ChatHost` still declares `requestUpdate` as optional, and test/non-app hosts can omit it. Calling it through an `OpenClawApp` cast can throw when abort or failed-send restore tries to refresh the draft; use the optional host hook safely or make the contract required with matching tests.
  - confidence: 0.88
- **[P2] Stop slash-menu updates from rendering plain text:** `ui/src/ui/app.ts:157-159`
  - body: Removing `@state()` from `chatMessage` does not stop ordinary typing from scheduling host renders because `handleInput` still calls `updateSlashMenu(...)`, and that helper unconditionally calls `requestUpdate()` for normal text. Guard the slash-menu update so regular typing only requests a render when slash-menu state actually changes.
  - confidence: 0.86
- **[P2] Refresh the current history-key path:** `ui/src/ui/views/chat.ts:1068-1080`
  - body: Current main no longer uses the inline `inputHistory.up()/down()` path updated here; Arrow recall now goes through `onHistoryKeydown` and `navigateChatInputHistory`, which mutates `chatMessage`. After the draft becomes non-reactive, that current path also needs an explicit refresh or the textarea can remain stale.
  - confidence: 0.87
