## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Constrain cross-session continuation fanout:** `src/agents/tools/continue-delegate-tool.ts:49-67`
  - body: `continue_delegate` exposes explicit target session keys and `fanoutMode: "all"` directly to the model. The return path can enqueue system events and wakeups into sessions beyond the requester lineage, so continuation-enabled prompt injection could inject context into unrelated conversations. Restrict returns to the requester/ancestor set or require an explicit operator-approved targeting policy before exposing this.
  - confidence: 0.92
- **[P1] Mark delegate task system events untrusted:** `src/auto-reply/continuation/delegate-dispatch.ts:212`
  - body: The delegate `task` comes from a model tool call and may include copied user, channel, or tool text, but these failure/spawn notifications enqueue it through `enqueueSystemEvent` without `trusted: false`. Since system events default to trusted privileged context, this can elevate untrusted text and is already flagged by Opengrep; pass `trusted: false` or sanitize and explicitly document trusted-only cases.
  - confidence: 0.9
- **[P2] Keep readLocalFileFromRoots on the SDK barrel:** `src/plugin-sdk/file-access-runtime.ts:3-8`
  - body: `openclaw/plugin-sdk/file-access-runtime` is a published SDK subpath and current main exports `readLocalFileFromRoots` from it. This patch removes that export while adding `appendFileWithinRoot`, which breaks existing plugins importing the media-source helper. Add the new helper without dropping the existing export.
  - confidence: 0.9
- **[P2] Redact session keys in fanout spans:** `src/infra/continuation-tracer.ts:762`
  - body: `emitContinuationFanoutSpan` records full target session keys in span attributes. Session keys can include channel, account, or conversation identifiers and diagnostics-otel may export spans outside the local process, so fanout telemetry should use counts, hashes, or redacted labels instead of raw keys.
  - confidence: 0.87
- **[P3] Add braces to the signal scan guard:** `src/auto-reply/continuation/signal.ts:90`
  - body: The current head fails `check-lint` because this guard omits braces required by the repo lint rule. Add a block around the `continue` so the lint gate can pass.
  - confidence: 0.96

## Security Review

Status: needs_attention

Summary: The diff introduces concrete cross-session injection and telemetry privacy risks, plus trusted system-event insertion of model-controlled delegate text.

Concerns:

- **[high] Unrestricted host-wide session injection:** `src/agents/tools/continue-delegate-tool.ts:49`
  - body: The model-facing `continue_delegate` schema allows explicit target sessions and `fanoutMode: "all"`, and the delivery path enqueues internal system events/wakeups for those recipients. Without an approved targeting policy, a compromised continuation-enabled session can inject context into unrelated sessions.

  - body: Delegate task text is interpolated into `enqueueSystemEvent` calls without `trusted: false`. Because system events default to trusted model-visible context, untrusted content copied into a delegate task can be elevated across later turns.

  - body: Fanout spans include the full target session key list. These identifiers may contain channel/account/conversation data and can leave the process through OTEL exporters.
  - confidence: 0.87
