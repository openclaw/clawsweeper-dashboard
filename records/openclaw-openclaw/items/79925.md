## Summary

Keep open: current main does not implement this feature, and the active draft PR still has blocking correctness, security-policy, and current-head proof issues that need maintainer review.

## What This Changes

The PR adds opt-in self-elected continuation tools, delegate return modes, compaction requests, continuation queues, telemetry/status/docs, and tests across agents, auto-reply, gateway, infra, config, and TaskFlow.

## Best Possible Solution

Narrow the continuation contract to authorized recipient scopes, preserve existing failover and tool-hook behavior, and require current-head live proof before maintainer review proceeds.

## Reproduction Assessment

Yes for the review findings via source inspection: PR head exposes host-wide return targeting and omits existing allowFormatRetry and hook opt-outs. Reproduction is not applicable to the feature request itself because this is a new capability rather than a broken current-main behavior.

## Solution Assessment

No. The proposed patch needs a narrower authorized target policy, preservation of current failover/tool-hook contracts, and current-head runtime proof before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Constrain cross-session continuation fanout:** `src/agents/tools/continue-delegate-tool.ts:63-67`
  - body: `fanoutMode="all"` and arbitrary target session keys are model-callable, and the return path enqueues internal system events/wakeups for resolved host sessions. Restrict returns to the requester/ancestor set or require an explicit maintainer-approved recipient policy before exposing host-wide targeting.
  - confidence: 0.9
- **[P1] Preserve retryable format-error recovery:** `src/agents/pi-embedded-runner/run.ts:2276-2288`
  - body: The PR still treats format failures as retryable only when `allowFormatRetry` is true, but the assistant failover call sites no longer pass `cloudCodeAssistFormatError`. Cloud Code Assist format errors that currently rotate/retry with sanitized tool calls will surface immediately instead.
  - confidence: 0.87
- **[P2] Keep shared tool hook opt-outs:** `src/agents/pi-tools.ts:752-753`
  - body: These tools are wrapped again after schema normalization in `createOpenClawCodingTools`, so removing the explicit opt-out lets OpenClaw-owned tools run `before_tool_call` twice. Restore the opt-out here and in gateway tool resolution, or remove the later shared wrapper instead.
  - confidence: 0.86
- **[P3] Forward traceparent into delegate spawns:** `src/auto-reply/continuation/delegate-dispatch.ts:261-268`
  - body: The tool accepts and validates `traceparent`, but normal delegate dispatch never passes it into `spawnSubagentDirect`. Completion traces therefore cannot stitch child spans back to the caller despite the advertised schema.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces model-controlled cross-session continuation delivery without an evident authorization policy for host-wide or explicit session targeting.

Concerns:

- **[medium] Unrestricted host-wide session injection:** `src/agents/tools/continue-delegate-tool.ts:63`
  - body: `fanoutMode="all"` can resolve every known session on the host and enqueue internal continuation events and wakeups for them. Because this is exposed through a model-callable tool, prompt-injected or compromised sessions could push context into unrelated conversations when continuation is enabled.
  - confidence: 0.88
