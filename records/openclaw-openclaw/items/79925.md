Adds opt-in continuation tools and runtime plumbing for self-scheduled turns, delegated subagent returns, volitional compaction, TaskFlow/session-delivery persistence, telemetry, status/protocol/docs updates, and broad tests.

## Best Possible Solution

Land only after maintainers choose and implement the targeting policy and trust boundary, with cross-session fanout gated by explicit operator policy and delegate output untrusted unless that policy elevates it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Constrain cross-session continuation targeting:** `src/agents/tools/continue-delegate-tool.ts:49-67`
  - body: `continue_delegate` lets the model specify arbitrary target session keys or `fanoutMode: "all"`. The return path resolves those targets to host sessions and enqueues system events/wakeups, so a compromised continuation-enabled session can inject context into unrelated conversations. Gate this behind an operator policy/allowlist or restrict returns to requester/ancestor lineage before exposing it.
  - confidence: 0.92
- **[P1] Gate trusted delegate return delivery:** `src/auto-reply/continuation/targeting.ts:106-110`
  - body: Targeted delegate completion text is inserted into recipient sessions with `trusted: true`. Even though the runtime wraps the child result, the content is produced by agent/delegate execution and can be routed cross-session, so it becomes privileged `System:` context in another conversation. Default this to untrusted, or only mark it trusted after an explicit targeting policy approves the sender/recipient pair.
  - confidence: 0.9
