## Summary

Keep open: this is an active, recent feature PR, but the current branch still exposes model-controlled cross-session continuation targeting, trusted cross-session System-context delivery, and a new unpinned binary download in a privileged proof workflow. Those need maintainer security/product decisions before merge.

## What This Changes

The PR adds opt-in continuation tools and runtime plumbing for self-scheduled turns, delegate dispatch/return modes, volitional compaction, status/docs/diagnostics, and related CI/proof infrastructure.

## Best Possible Solution

Split or narrow the branch, then land continuation only after maintainers choose and encode an explicit cross-session targeting/trust policy and the added proof workflow pins downloaded executables by immutable version and checksum.

## Reproduction Assessment

Yes for the review blockers: source inspection of the latest PR head shows the model-facing targeting fields, host-wide fanout resolution, and trusted System-context delivery path. The workflow supply-chain issue is also directly visible in the added workflow source.

## Solution Assessment

No. The continuation direction may be useful, but this merge shape is not the safest solution until the cross-session policy boundary is explicit and unrelated privileged workflow/tooling changes are removed or pinned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Constrain cross-session continuation targeting:** `src/agents/tools/continue-delegate-tool.ts:49-67`
  - body: `continue_delegate` lets the model address arbitrary target sessions or request `fanoutMode: "all"`, and the delivery path resolves that to known host sessions and queues/wakes them. Gate this behind an operator policy/allowlist or restrict it to requester/ancestor lineage before exposing it to continuation-enabled sessions.
  - confidence: 0.91
- **[P1] Gate trusted delegate return delivery:** `src/auto-reply/continuation/targeting.ts:106-110`
  - body: Targeted delegate completion text is inserted into recipient sessions with `trusted: true`, which current main renders as privileged `System:` context. Default these returns to untrusted, or only elevate them after an explicit sender/recipient policy authorizes that trust edge.
  - confidence: 0.9
- **[P1] Pin downloaded proof binaries:** `.github/workflows/mantis-telegram-desktop-proof.yml:306-313`
  - body: This workflow downloads FFmpeg from a moving `latest` release URL, installs it with sudo, and keeps the binary available for later proof steps that receive live proof secrets and write-capable tokens. Use a pinned release asset plus checksum verification, or a trusted package source, before installing and executing it in this privileged job.
  - confidence: 0.86
