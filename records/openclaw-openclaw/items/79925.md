Keep this PR open: it is active broad feature work, current main does not already contain the continuation surface, and the current head still has a source-reproducible delivery bug plus unresolved proof and security/product approval gates.

## What This Changes

The PR adds opt-in continuation tools, delegate routing and persistence, post-compaction dispatch, tracing/diagnostics, status/protocol/docs/tests, and continuation-related dependency/config surfaces.

## Best Possible Solution

Keep the feature disabled by default, reuse the shared targeting helper for every delivery-time gate, add regression coverage for normal `fanoutMode: "tree"` returns under the default policy, and require current-head proof plus maintainer approval for trusted cross-session fanout before merge.

## Reproduction Assessment

Yes. Source reproduction is high-confidence: create a normal `continue_delegate` with `fanoutMode: "tree"` while `crossSessionTargeting` is disabled, let the child complete, and the normal return path rejects it even though the tool and shared helper allow lineage routing.

## Solution Assessment

No, not as currently proposed. The narrow fix is to use `hasCrossSessionDelegateTargeting` or equivalent normalized logic in `src/agents/subagent-announce.ts` and cover this default-policy tree return path with a regression test.
