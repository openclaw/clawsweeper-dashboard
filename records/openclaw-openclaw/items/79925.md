## Summary

Keep this PR open. The continuation feature is not on current main, and the current PR head still has a source-reproducible default-policy regression in normal `fanoutMode: "tree"` returns plus a cross-session trusted-fanout decision that needs maintainer security/product approval.

## What This Changes

The PR adds opt-in continuation tools, delegate return routing, compaction-triggered dispatch, trace propagation, diagnostics/status/protocol/docs/tests, and a new root `uuid` dependency.

## Best Possible Solution

Keep the feature disabled by default, make all delivery-time gates use the same targeting helper so lineage `tree` routing works under the default policy, and require maintainer approval for trusted host-wide fanout before merge.

## Reproduction Assessment

Yes, source reproduction is high-confidence: create a normal continuation delegate with `fanoutMode: "tree"` while `crossSessionTargeting` is disabled, let the child complete, and the normal targeted-return branch rejects it at delivery time even though tool/dispatch policy allows tree lineage routing.

## Solution Assessment

No, not as currently proposed. The narrow fix is to reuse `hasCrossSessionDelegateTargeting` or equivalent normalized logic in `src/agents/subagent-announce.ts` and add a regression test for normal `fanoutMode: "tree"` returns under the default policy.
