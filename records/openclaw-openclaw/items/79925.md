## Summary

Keep open. Current main does not contain the requested continuation surface, and this draft PR still carries unique feature work, but the current patch has blocking security/privacy and SDK compatibility issues plus real-behavior proof that is not tied to the current head SHA.

## What This Changes

The PR adds opt-in agent continuation tools, delegate queues and return targeting, compaction lifecycle hooks, telemetry, config/docs/tests, and related gateway, TaskFlow, SDK, status, and diagnostics wiring.

## Best Possible Solution

Keep this as a focused design/review branch until continuation targeting has an explicit security policy, SDK compatibility is preserved, sensitive telemetry is redacted, and current-head live proof covers all three tools end to end.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? For the PR review blockers, yes by source inspection: the current branch exposes model-controlled cross-session targeting, drops a published SDK export, and emits raw session keys in span attributes. For the feature's real behavior, no current-head live reproduction is supplied because the proof is tied to 0831fb5 rather than ae71f064.

## Solution Assessment

Is this the best way to solve the issue? No. The safer path is to constrain continuation returns to requester/ancestor or an explicit operator-approved allowlist, preserve published SDK exports, redact diagnostic identifiers, and then attach current-head real runtime proof before normal review continues.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain cross-session continuation fanout:** `src/agents/tools/continue-delegate-tool.ts:49-67`
  - body: `continue_delegate` exposes `targetSessionKey`, `targetSessionKeys`, and `fanoutMode: "all"` directly to the model. Because the return path can enqueue silent system events and wakeups into other sessions on the host, a prompt-controlled session with continuation enabled can inject context into unrelated conversations. Restrict this to the requester/ancestor set or require an explicit operator-approved targeting policy before exposing host-wide fanout.
  - confidence: 0.91
- **[P2] Keep readLocalFileFromRoots on the SDK barrel:** `src/plugin-sdk/file-access-runtime.ts:3-8`
  - body: `openclaw/plugin-sdk/file-access-runtime` is a published and documented plugin SDK subpath, and current main exports `readLocalFileFromRoots` from it. Dropping that export while adding `appendFileWithinRoot` is a breaking change for existing plugins that import the media-source helper. Add the new helper without removing the existing export.
  - confidence: 0.9
- **[P2] Redact session keys in fanout spans:** `src/infra/continuation-tracer.ts:762`
  - body: `emitContinuationFanoutSpan` records full target session keys in span attributes. Session keys can contain channel/account/conversation identifiers and diagnostics-otel can export spans outside the local process, so fanout telemetry should use counts, stable hashes, or redacted labels instead of raw keys.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces model-controlled cross-session return targeting and exports raw session identifiers into telemetry attributes.

Concerns:

- **[medium] Unrestricted host-wide session injection:** `src/agents/tools/continue-delegate-tool.ts:49`
  - body: `fanoutMode: "all"` and explicit target session keys let a continuation delegate return enqueue internal system events or wakeups for sessions beyond the requester lineage. Because this is exposed through model tool calls and fallback syntax, prompt injection or a compromised session could inject context across unrelated conversations when continuation is enabled.
  - confidence: 0.91
- **[medium] Raw session keys exported to telemetry:** `src/infra/continuation-tracer.ts:762`
  - body: Fanout spans include full target session keys as attributes. These keys may include channel or conversation identifiers and can leave the host through configured OTEL exporters, so they should be redacted or hashed before export.
  - confidence: 0.86
