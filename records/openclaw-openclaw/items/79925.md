## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain cross-session continuation fanout:** `src/agents/tools/continue-delegate-tool.ts:63-67`
  - body: `continue_delegate` exposes explicit target session keys and `fanoutMode: "all"` directly to model output. The return path can enqueue internal system events and wake other sessions, so continuation can cross conversation boundaries without an evident authorization policy. Restrict returns to requester/ancestor scope or require an approved policy before exposing host-wide targeting.
  - confidence: 0.9
- **[P1] Preserve format-error retry failover:** `src/agents/pi-embedded-runner/run/assistant-failover.ts:183-194`
  - body: The PR drops `allowFormatRetry: params.cloudCodeAssistFormatError` from the assistant failover decision. CloudCodeAssist format errors that currently retry under that gate will now be treated as terminal format failures instead of taking the intended retry path.
  - confidence: 0.88
- **[P2] Keep hook wrapping disabled for shared tool paths:** `src/agents/pi-tools.ts:752-753`
  - body: The coding-tool path still flows through `createOpenClawTools`, whose before-tool-call hook wrapping is opt-out. Removing `wrapBeforeToolCallHook: false` means shared OpenClaw-owned tools can run `before_tool_call` twice, duplicating approvals, diagnostics, or loop-accounting side effects.
  - confidence: 0.87
- **[P3] Warn or gate partial continuation registration:** `src/agents/openclaw-tools.ts:474-477`
  - body: `continue_delegate` registers when `drainsContinuationDelegateQueue !== false`, so `undefined` enables it even when the sibling callbacks for `continue_work` and `request_compaction` are absent. Add the promised guard/warning or require explicit runtime options so operators do not get a silent partial continuation surface.
  - confidence: 0.82
- **[P3] Forward traceparent when spawning delegates:** `src/auto-reply/continuation/delegate-dispatch.ts:255-268`
  - body: The tool validates and stores `traceparent`, but the normal delegate spawn params omit `delegate.traceparent`. Regular continuation delegates therefore lose the advertised trace stitching unless the spawn call forwards the field.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff exposes model-controlled cross-session targeting/fanout that can enqueue system events and wake other sessions without an evident authorization boundary.

Concerns:

- **[medium] Unrestricted host-wide session injection:** `src/agents/tools/continue-delegate-tool.ts:63`
  - body: `fanoutMode: "all"` and explicit target session keys let a continuation delegate return enqueue internal system events or wakeups for sessions beyond the requester lineage. Because this is exposed to model/tool output, prompt injection or a compromised session could leak or inject context across unrelated conversations when continuation is enabled.
  - confidence: 0.9
