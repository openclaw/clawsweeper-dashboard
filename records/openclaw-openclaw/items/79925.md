## Summary

Keep open. The current head still has blocking security and SDK compatibility issues, even though CI and real-host proof are now much stronger than the earlier review.

## What This Changes

The PR adds opt-in continuation tools, delegate return targeting and fanout, volitional compaction, TaskFlow/session-delivery persistence, telemetry/status/protocol/docs updates, and broad tests.

## Best Possible Solution

Land only after session targeting is constrained or allowlisted, delegate-return trust boundaries and telemetry redaction are fixed, SDK exports remain backward-compatible, and maintainers accept the new continuation config/product surface.

## Reproduction Assessment

Yes for the review blockers: the unsafe fanout surface, trusted cross-session event insertion, telemetry privacy issue, and SDK export removal are all visible from current-head source and current-main contracts. I did not run the full feature live; the PR body and CI proof gate provide supplemental runtime proof.

## Solution Assessment

No. The feature direction may be useful, but this implementation is not yet the narrowest maintainable solution because it exposes model-controlled cross-session targeting, inserts delegate output as trusted context, leaks raw session keys to telemetry, and removes an existing SDK export.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Constrain cross-session continuation fanout:** `src/agents/tools/continue-delegate-tool.ts:49-67`
  - body: `continue_delegate` exposes explicit target session keys and `fanoutMode: "all"` directly to the model. The return path can enqueue system events and wakeups into sessions beyond the requester lineage, so a compromised continuation-enabled session could inject context into unrelated conversations. Restrict returns to the requester/ancestor set or require an explicit operator-approved targeting policy before exposing this.
  - confidence: 0.92
- **[P1] Mark targeted delegate returns untrusted:** `src/auto-reply/continuation/targeting.ts:106-110`
  - body: Targeted delegate completion text is written into each recipient with `trusted: true`. That text comes from agent/delegate execution and can include copied user, channel, or tool output, so cross-session enrichment can become privileged system context. Pass `trusted: false` unless the content is constructed entirely by trusted runtime code.
  - confidence: 0.9
- **[P2] Keep readLocalFileFromRoots on the SDK barrel:** `src/plugin-sdk/file-access-runtime.ts:3-8`
  - body: `openclaw/plugin-sdk/file-access-runtime` is a published SDK subpath and current main exports `readLocalFileFromRoots` from it. This patch removes that export while adding `appendFileWithinRoot`, which breaks existing plugins importing the media-source helper. Add the new helper without dropping the existing export.
  - confidence: 0.9
- **[P2] Redact session keys in fanout spans:** `src/infra/continuation-tracer.ts:759-763`
  - body: `emitContinuationFanoutSpan` records full target session keys in span attributes. Session keys can include channel, account, or conversation identifiers and diagnostics-otel may export spans outside the local process, so fanout telemetry should use counts, hashes, or redacted labels instead of raw keys.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces concrete cross-session injection and telemetry privacy concerns in the continuation return path.

Concerns:

- **[high] Unrestricted host-wide session injection:** `src/agents/tools/continue-delegate-tool.ts:49`
  - body: The model-facing `continue_delegate` schema allows explicit target sessions and `fanoutMode: "all"`, and the delivery path enqueues internal system events/wakeups for those recipients. Without an approved targeting policy, a compromised continuation-enabled session can inject context into unrelated sessions.
  - confidence: 0.92
- **[high] Delegate output enters trusted system events:** `src/auto-reply/continuation/targeting.ts:106`
  - body: Targeted delegate return text is enqueued with `trusted: true`. Because system events render trusted content as privileged prompt context, untrusted delegate output should not be trusted by default.
  - confidence: 0.9
- **[medium] Raw session keys exported to telemetry:** `src/infra/continuation-tracer.ts:762`
  - body: Fanout spans include full target session keys, which can contain channel/account/conversation identifiers and can be exported by diagnostics-otel.
  - confidence: 0.87
