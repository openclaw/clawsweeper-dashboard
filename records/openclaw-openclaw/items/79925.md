Keep open: current main does not implement the continuation feature, and this PR is the active replacement for the earlier closed continuation branch, but live head still has blocking correctness/security and dependency-ownership issues. ## What This Changes

The PR adds opt-in continuation tools and runtime plumbing for self-scheduled agent turns, delegated subagents, volitional compaction, tracing, status/protocol/docs/tests, and related dependency changes.

## Best Possible Solution

Re-check crossSessionTargeting at all delayed delivery surfaces, keep plugin-owned dependencies in extension packages, and land only after current-head real behavior proof plus maintainer approval for trusted cross-session returns.

## Reproduction Assessment

Yes for the review blockers: source inspection at live PR head shows queued post-compaction delegates can carry cross-session targeting fields to spawnSubagentDirect without a live policy re-check, and CI logs show the root dependency ownership failure. I did not run a live gateway because this review is read-only.

## Solution Assessment

No. The feature direction may be viable, but the current patch is not the best mergeable shape until queued post-compaction delivery enforces the same policy gate, root dependency ownership is restored, and maintainers approve the trusted targeted-return model.
