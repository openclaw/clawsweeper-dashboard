## Summary

Keep open: the current head still contains blocking regressions unrelated to continuation, including Telegram stop debounce bypass removal, Feishu subagent delivery-origin loss, Codex transcript output loss, and public SDK health export removal. The continuation proof is substantial, but these source-proven regressions and the trusted cross-session context policy need fixes and maintainer approval before merge.

The PR adds opt-in continuation runtime/tooling for `continue_work`, `continue_delegate`, and `request_compaction`, plus queue persistence, cross-session targeting, context-pressure/compaction hooks, OTel tracing, docs, tests, workflow updates, and a `uuid` dependency.

## Best Possible Solution

Restore the current-main behavior that the branch accidentally reverts, keep the continuation changes scoped to their intended surfaces, and get explicit maintainer approval for the opt-in trusted cross-session return model.

Full review comments:

- **[P1] Preserve Telegram stop bypass handling:** `extensions/telegram/src/bot-handlers.runtime.ts:1542`
  - body: The PR removes the abort-control branch and now sends every non-command text through the fragment/debounce path. A plain `stop` no longer cancels queued same-chat inbound debounce and will be processed like user text; current main has regression coverage for `stop` and `/stop@openclaw_bot`. Restore the abort detection, authorization, and cancel path.
  - confidence: 0.94
- **[P1] Return Feishu delivery origin for thread-bound subagents:** `extensions/feishu/src/subagent-hooks.ts:347-350`
  - body: After binding the Feishu conversation, this returns only `status` and `threadBindingReady`. The core spawn path reads `result.deliveryOrigin` to seed the child session routing context, so Feishu thread-mode subagent completions lose their routable origin and can miss or misroute replies. Keep returning `resolveFeishuDeliveryOrigin(...)` and its tests.
  - confidence: 0.93
- **[P1] Keep streamed Codex command output in transcripts:** `extensions/codex/src/app-server/event-projector.ts:923`
  - body: This drops the output-delta accumulator and records `itemOutputText(params.item)` only. When Codex final command snapshots have `aggregatedOutput: null`, streamed native command output is lost from mirrored transcripts and trajectory exports, regressing the current-main coverage for that exact case.
  - confidence: 0.92
- **[P1] Keep the public plugin health SDK export:** `package.json:135-137`
  - body: Removing the `./plugin-sdk/health` export and deleting `src/plugin-sdk/health.ts` makes existing `openclaw/plugin-sdk/health` imports fail at module resolution. That subpath was added as part of the doctor health-check contract on current main; preserve it or make an explicit versioned SDK break with caller migration.
