## Summary

Keep open. This is a recent, broad feature PR that is not implemented on current main, and the current branch still has a source-reproducible cross-session policy gap in the normal targeted-return path before it can be merged.

## What This Changes

Adds opt-in continuation tools, TaskFlow-backed delegate returns, context-pressure and compaction controls, trace stitching, status/docs/protocol updates, tests, and a direct `uuid` dependency.

## Best Possible Solution

Mirror the post-compaction delivery-time `crossSessionTargeting` check in normal targeted returns, keep the default-deny policy intact, and land only after maintainer policy approval, dependency review, and green CI.

## Reproduction Assessment

Yes for the review finding by source inspection: enable cross-session targeting at delegate spawn, disable it before child completion, and the normal targeted-return branch still enqueues trusted delivery without a live policy recheck. No current-main bug reproduction applies because the continuation feature is not on main.

## Solution Assessment

No, not as-is. The feature direction may be viable, but normal targeted-return delivery should re-read the live continuation policy before trusted cross-session delivery, matching the post-compaction path.
