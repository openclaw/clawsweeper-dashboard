## Summary

Keep open: the feature is not implemented on current main, the latest head has supplied live-runtime proof, but the patch still has a blocking targeted-return durability bug and a separate trusted cross-session context policy surface that needs maintainer approval.

## What This Changes

The PR adds opt-in agent continuation tools for self-continuation, delegation, and volitional compaction across agent runtime, gateway/session delivery, diagnostics, protocol generation, docs, tests, and dependencies.

## Best Possible Solution

Fix the targeted-return durable delivery contract so already-consumed live returns cannot replay after restart, then have maintainers explicitly approve or adjust the trusted cross-session targeting policy before merge.

## Reproduction Assessment

Yes for the review finding: source inspection shows a targeted return both persists a pending systemEvent delivery and queues the same in-memory event, while the normal in-memory drain has no ack path for the durable file. Reproduction is not applicable to the PR’s central request because it is a new opt-in feature, not a report of broken existing behavior.

## Solution Assessment

No as currently written. The disabled-by-default continuation shape is plausible, but targeted-return durability needs to distinguish live attached delivery from restart recovery, and trusted cross-session fanout needs explicit maintainer policy approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Avoid replaying live targeted returns after restart:** `src/auto-reply/continuation/targeting.ts:100-118`
  - body: `enqueueContinuationReturnDeliveries` writes a durable `systemEvent` file and then immediately queues the same text in the recipient's in-memory system-event queue. When the recipient consumes that live event, nothing acks the durable file; the startup recovery path later drains all pending session-delivery files and will re-enqueue the stale delegate result, waking the session a second time after restart. Ack live-attached deliveries after confirmed consumption, or only persist entries that still need restart recovery.
  - confidence: 0.86

## Security Review

Status: needs_attention

The default-deny gate reduces exposure, but maintainers should explicitly approve the policy shape, allowed targeting modes, and operator controls before this becomes a core runtime capability.
