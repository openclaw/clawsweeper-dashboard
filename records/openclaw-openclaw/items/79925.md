## Summary

This PR should stay open: it is a new opt-in continuation feature that is not implemented on current main, but the current head still has a correctness blocker in normal `fanoutMode: "tree"` return delivery, the live proof is not pinned to the current head, and trusted cross-session fanout needs maintainer security/product approval.

## What This Changes

The PR adds opt-in continuation tools, delegate return routing, compaction-triggered dispatch, trace propagation, diagnostics/status/protocol/docs/tests, and a new root `uuid` dependency.

## Best Possible Solution

Keep the feature disabled by default, align all delivery-time gates on the shared targeting helper so lineage `tree` routing works, and require current-head real behavior proof plus maintainer approval for trusted cross-session fanout before merge.

## Reproduction Assessment

Yes for the review finding: source inspection shows a normal continuation delegate using `fanoutMode: "tree"` is admitted under the default disabled cross-session policy, but the normal targeted-return delivery branch rejects it before enqueueing the lineage return. I did not run tests or live scenarios because this review was read-only.

## Solution Assessment

No, not as currently proposed. The narrow fix is to reuse `hasCrossSessionDelegateTargeting` or equivalent normalized logic in `src/agents/subagent-announce.ts` and add a regression test for normal `fanoutMode: "tree"` returns under the default policy.
