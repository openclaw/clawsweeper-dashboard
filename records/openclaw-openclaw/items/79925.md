## Summary

Keep open: current main does not contain this continuation surface, and the PR still has a concrete P2 policy gap in normal targeted returns that can deliver trusted cross-session context after the operator disables `crossSessionTargeting`. The current-head proof is now SHA-matched, but merge still needs the code fix plus maintainer product/security approval for trusted cross-session fanout.

## What This Changes

The PR adds opt-in continuation tools and config, durable delegate and post-compaction dispatch, cross-session targeted returns, trace propagation/OTel support, status/protocol/docs/tests, and a direct `uuid` dependency.

## Best Possible Solution

Add a delivery-time `crossSessionTargeting` recheck to normal targeted returns, keep the feature disabled by default, and have maintainers explicitly decide the trusted cross-session fanout policy before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: spawn a targeted delegate while `crossSessionTargeting` is enabled, disable it before the child completes, and the normal targeted-return branch still enqueues trusted recipient context. No current-main bug reproduction applies because this continuation feature is not on main.

## Solution Assessment

No, not as currently proposed. The feature direction may be viable, but normal targeted returns should mirror the post-compaction delivery-time gate and the trusted fanout model needs maintainer approval.
