## What This Changes

This PR adds disabled-by-default continuation tools, runtime queues, compaction lifecycle hooks, cross-session delegate targeting controls, trace propagation, status/docs, and tests for `continue_work`, `continue_delegate`, and `request_compaction`.

## Best Possible Solution

Keep the feature disabled by default, add an ack or replay-prevention path for durable targeted returns, then have maintainers approve or narrow the opt-in trusted cross-session fanout semantics before merge.

## Reproduction Assessment

Yes for the review finding: source inspection shows targeted returns are both persisted and injected in-memory, while prompt drain consumes only the in-memory event and cannot ack the durable file. This is a new feature PR rather than a current-main bug report.

## Solution Assessment

No. The feature shape has strong proof and default-deny gates, but it should not merge until consumed targeted returns cannot replay after restart and the trusted cross-session policy is approved or narrowed.

## Repair Work Prompt

Fix the targeted continuation return durability bug in this PR branch. `enqueueContinuationReturnDeliveries` currently writes a durable `systemEvent` and immediately injects the same event into the recipient's in-memory system-event queue; if the recipient drains that in-memory event before a gateway restart, the durable file is never acked and can replay later. Add an ack/replay-prevention path that preserves durability for non-attached recipients, idempotency, traceparent propagation, wake behavior, and the default-deny `crossSessionTargeting` policy. Prefer a small change around `src/auto-reply/continuation/targeting.ts`, `src/auto-reply/reply/session-system-events.ts`, and session-delivery queue metadata rather than changing the public feature shape. Add a regression test proving a targeted return consumed before recovery is not replayed and does not leave a pending durable delivery; do not remove the trusted cross-session policy gates or broaden fanout semantics.
