Keep open: current head has real behavior proof and the earlier mechanical regressions appear resolved, but it still exposes model-controlled cross-session return targeting and trusted System-context delivery that need maintainer product/security policy before merge.

## What This Changes

The PR adds opt-in continuation tools and supporting runtime, queue, telemetry, protocol, docs, and tests for self-scheduled agent turns, delegated continuation work, and volitional compaction.

## Best Possible Solution

Land only after maintainers choose and encode an explicit trust boundary for cross-session targeting, preferably default-deny or lineage-only unless operator configuration authorizes broader fanout and trust elevation.

## Reproduction Assessment

Yes for the review blockers: PR-head source exposes model-selected target sessions and host-wide fanout, then inserts delegate output as trusted System context. A live exploit run is not necessary to prove those deterministic source paths exist.

## Solution Assessment

No. The continuation direction may be valuable, but the current PR is not the safest maintainable shape until cross-session recipient selection and trust elevation are behind a maintainer-approved policy boundary.

## Security Review
