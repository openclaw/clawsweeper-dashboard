## Summary

This PR should stay open: the continuation feature is not on current main, and the latest head still has cross-session targeting policy bypasses plus a dependency ownership failure that blocks CI.

## What This Changes

The PR adds opt-in continuation tools (`continue_work`, `continue_delegate`, `request_compaction`), durable delegate queues, post-compaction delivery, cross-session targeting policy, tracing, status/protocol/docs/tests, and related runtime plumbing.

## Best Possible Solution

Apply the documented `crossSessionTargeting` live gate to every queued and sub-agent bracket delivery path, remove extension-owned root dependency churn, then get maintainer approval for the trusted targeted-return policy before broad validation.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the post-compaction queued release and sub-agent bracket continuation paths can forward cross-session target fields to `spawnSubagentDirect` without the documented live policy check. CI logs also reproduce the root dependency ownership failure.

## Solution Assessment

No. The feature direction may be viable, but the current patch does not yet enforce its own default-deny policy across all delivery paths and keeps an extension-owned OTel dependency in the root package.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Re-check targeting policy before post-compaction spawn:** `src/auto-reply/reply/post-compaction-delegate-dispatch.ts:517-523`
  - body: Post-compaction delegates can be staged while cross-session targeting is enabled, persisted, and then delivered after the operator disables that policy. This release path reads config for chain and cost limits but still forwards `targetSessionKey`, `targetSessionKeys`, or `fanoutMode` to `spawnSubagentDirect`, so queued releases bypass the documented live-read gate.
  - confidence: 0.92
- **[P1] Gate sub-agent bracket continuation targeting:** `src/agents/subagent-announce.ts:974-982`
  - body: A continuation child can emit `[[CONTINUE_DELEGATE: ... | target=...]]` or `fanout=all`, and this announce-side chain path forwards those fields into `spawnSubagentDirect` without checking `crossSessionTargeting`. That lets a delegate spawned under the default-deny policy create a cross-session return from inside the chain.
  - confidence: 0.88
- **[P2] Remove the root OTel dependency:** `package.json:1738`
  - body: `@opentelemetry/api` is already declared by the diagnostics-otel plugin package, and CI now fails both deadcode and root-ownership checks because the root dependency is unused and extension-owned. Keep this dependency in `extensions/diagnostics-otel/package.json` rather than the root package.
  - confidence: 0.97
