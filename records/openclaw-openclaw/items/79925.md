## Summary

Keep open: this active external XL feature has supplied real-host proof and green check-run evidence, but the current head still exposes model-controlled cross-session targeting and trusted delegate-return insertion. The author explicitly frames those as intentional product/security questions for maintainer adjudication, so this PR is not cleanup-closeable or merge-ready.

## What This Changes

The PR adds opt-in continuation tools and runtime plumbing for self-scheduled agent turns, delegated subagent returns, volitional compaction, TaskFlow/session-delivery persistence, telemetry, status/protocol/docs updates, and broad tests.

## Best Possible Solution

Land only after maintainers choose and implement an explicit operator policy/trust boundary for cross-session targeting, with default-deny or lineage-only behavior unless a configured policy authorizes broader fanout and trusted returns.

## Reproduction Assessment

Yes for the review blockers: current PR head source exposes model-controlled targeting fields and inserts targeted delegate returns as trusted system events. I did not need a live exploit repro because the delivery and formatting paths are deterministic in source, while the PR separately supplies real-host proof for the feature path.

## Solution Assessment

No. The continuation direction may be useful, but this PR is not yet the safest maintainable shape because cross-session recipient selection and System-trusted delegate output need an explicit maintainer-approved policy boundary before exposure.
