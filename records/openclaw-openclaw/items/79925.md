## Summary

Keep open. This is an active external XL feature PR with useful current-head proof, but the patch still has a concrete post-compaction cross-session targeting policy bypass and needs maintainer security/product judgment before merge.

## What This Changes

Adds an opt-in continuation subsystem with `continue_work`, `continue_delegate`, and `request_compaction`, plus config/schema, session delivery, subagent, compaction, diagnostics, status, Telegram-adjacent, Swift protocol, docs, tests, and dependency changes.

## Best Possible Solution

Add a live policy re-check before queued post-compaction delegate spawn, cover enabled-then-disabled config with a regression test, and get explicit maintainer sign-off on trusted cross-session return semantics before merge.

## Reproduction Assessment

Yes for the review blocker: source inspection shows queued post-compaction delegates can carry cross-session target fields to `spawnSubagentDirect` without re-checking the live `crossSessionTargeting` policy. I did not run a live reproduction because this review is read-only.

## Solution Assessment

No. The continuation direction may be viable, but this patch misses a documented enforcement point and still needs maintainer adjudication on trusted cross-session delivery.
