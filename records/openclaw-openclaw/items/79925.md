## Summary

Keep open: current main does not implement the continuation feature, and this active draft PR is not mergeable yet because it introduces unresolved cross-session targeting policy, regresses tool hook wrapping, and still lacks current-head real behavior proof.

## What This Changes

The PR adds opt-in continuation tools/signals, delegate queues and targeting, compaction hooks, tracing/status telemetry, config/schema support, docs, and tests across agents, auto-reply, gateway, infra, and TaskFlow.

## Best Possible Solution

Land a narrower continuation contract only after maintainers approve the session-targeting policy, existing tool hook behavior is preserved, and current-head real runtime proof shows the new tools working.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review findings via source inspection: the PR exposes model-selected host-wide return targeting and removes existing hook-wrapping opt-outs; I did not run live behavior in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. The current patch should restrict or explicitly authorize cross-session continuation returns, keep the shared tool-hook boundary intact, and then proceed through maintainer product/security review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Constrain cross-session continuation fanout:** `src/agents/tools/continue-delegate-tool.ts:63-67`
  - body: `continue_delegate` exposes arbitrary `targetSessionKey(s)` plus `fanoutMode="all"` to the model. The return path later enumerates every known session on the host and enqueues internal system events/wakes for those sessions, so one prompt-injected session can push context into unrelated conversations once continuation is enabled. Restrict returns to the requester/ancestor set or require an explicit owner-approved policy before exposing host-wide targeting.
  - confidence: 0.88
- **[P2] Keep hook wrapping disabled for shared tool paths:** `src/agents/pi-tools.ts:752`
  - body: `createOpenClawCodingTools` still wraps every normalized tool at the later shared boundary, and gateway tool resolution also expects raw tools here. Removing the `wrapBeforeToolCallHook: false` opt-out means OpenClaw-owned tools can run `before_tool_call` twice, duplicating approvals, diagnostics, and loop-accounting side effects. Restore the opt-out at these callers or remove the later wrapper instead.
  - confidence: 0.86
- **[P3] Forward traceparent when spawning delegates:** `src/auto-reply/continuation/delegate-dispatch.ts:261-268`
  - body: The tool accepts and validates a `traceparent`, but normal tool-delegate dispatch never passes it into `spawnSubagentDirect`. As a result, the completion path cannot stitch the delegate span into the caller's trace despite the advertised schema and docs. Include `delegate.traceparent` in the spawn params when present.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: the PR adds model-controlled cross-session continuation returns, including host-wide fanout, without an evident authorization or recipient policy.

Concerns:

- **[medium] Unrestricted host-wide session injection:** `src/agents/tools/continue-delegate-tool.ts:63`

Because the option is model-callable and not visibly constrained to the requester or owner-approved recipients, it can leak or inject internal context across unrelated conversations when continuation is enabled.
