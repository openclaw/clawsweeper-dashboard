Full review comments:

- **[P1] Require recipient policy before host-wide fanout:** `src/agents/tools/continue-delegate-tool.ts:204-208`
  - body: `crossSessionTargeting: "enabled"` globally allows any continuation-enabled session to use `targetSessionKey`, `targetSessionKeys`, or `fanoutMode: "all"` against other host sessions. Add an operator policy/allowlist or equivalent recipient gate before exposing non-self targets to model-selected tool input.
  - confidence: 0.9
- **[P1] Avoid trusting delegate output in recipient sessions:** `src/auto-reply/continuation/targeting.ts:113-118`
  - body: Targeted continuation returns enqueue the delegate completion text with `trusted: true`, and the existing event renderer treats trusted entries as privileged `System` context. Cross-session or targeted delegate results should be untrusted by default, or elevated only after an approved sender/recipient policy authorizes that trust edge.
  - confidence: 0.88
