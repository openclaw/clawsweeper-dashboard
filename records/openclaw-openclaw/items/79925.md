Keep open. Current main does not contain this continuation feature, and the live PR head still has a P1 cross-session targeting policy bypass in the queued post-compaction delivery path; the trusted cross-session return design also needs maintainer product/security approval.

## What This Changes

The PR adds opt-in continuation tools and broad agent/gateway plumbing for self-elected turns, delegated subagents, volitional compaction, tracing, status/protocol/docs/tests, and dependency updates.

## Best Possible Solution

Re-check crossSessionTargeting during queued post-compaction delivery, add enabled-then-disabled regression coverage, refresh current-head real behavior proof, and get maintainer approval for trusted cross-session returns before merge.

## Reproduction Assessment

Yes for the review blocker: source inspection at live PR head shows queued post-compaction delegates can carry cross-session target fields to spawnSubagentDirect without a live policy re-check. I did not run a live setup because this review is read-only.

## Solution Assessment

No. The feature direction may be viable, but the current patch documents live-read enforcement at post-compaction release while this queued delivery path misses that gate; the narrower maintainable fix is to re-check policy at delivery and cover enabled-then-disabled behavior.
