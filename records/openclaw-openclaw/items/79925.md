## Summary

Keep open: this is a recent active feature PR with supplied real-host proof, but the current head still intentionally exposes cross-session fanout and trusted delegate-return delivery that need maintainer product/security adjudication before merge.

## What This Changes

The PR adds opt-in continuation tools and runtime plumbing for self-scheduled agent turns, delegated subagent returns, request-driven compaction, persistence/telemetry/status/docs updates, and broad regression coverage.

## Best Possible Solution

Land only after maintainers choose and implement an explicit trust and targeting policy, preferably default-deny or lineage-only unless an operator-configured allowlist authorizes broader fanout and trusted returns.

## Reproduction Assessment

Yes for the review blockers: current PR head source exposes model-controlled target fields and host-wide fanout, then inserts targeted delegate returns as trusted system events. The current-main formatter makes the trusted/untrusted distinction deterministic in source.

## Solution Assessment

No. The continuation direction may be useful, but the current PR is not yet the safest maintainable shape because cross-session recipient selection and System-trusted delegate output need an explicit maintainer-approved policy boundary.
