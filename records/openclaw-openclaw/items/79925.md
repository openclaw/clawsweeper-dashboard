Keep open: the current PR head still has a source-reproducible cross-session targeting policy bypass in queued post-compaction delivery, the after-fix proof is pinned to an older SHA, and trusted targeted returns need maintainer security/product review.

## What This Changes

Adds opt-in continuation tools and runtime plumbing for self-elected turns, delegated subagents, volitional compaction, cross-session returns, OTel tracing, status/protocol/docs/tests, and dependency updates.

## Best Possible Solution

Re-check crossSessionTargeting during queued post-compaction delivery, add enabled-then-disabled regression coverage, refresh current-head real behavior proof, and get maintainer approval or a revised policy for trusted cross-session returns.

## Reproduction Assessment

Yes for the review blocker: source inspection at current PR head shows queued post-compaction delegates can carry cross-session target fields to spawnSubagentDirect without a live policy re-check. I did not run a live reproduction because this review is read-only.

## Solution Assessment

No. The feature direction may be viable, but the current patch documents live-read post-compaction enforcement while the queued release implementation misses that gate; the safer path is a narrow gate re-check plus maintainer-approved trusted-return policy.
