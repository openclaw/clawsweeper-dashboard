## Security Review

Status: needs_attention

Summary: The diff introduces model-controlled cross-session context injection and trusted delegate-return insertion without an approved policy gate.

Concerns:

- **[high] Model-controlled cross-session context injection:** `src/agents/tools/continue-delegate-tool.ts:49`
  - body: The continuation tool schema exposes explicit target sessions and host-wide fanout to the model, and the delivery path writes or wakes the resolved target sessions. Without an operator policy or lineage restriction, a compromised continuation-enabled session can affect unrelated sessions on the same host.
  - confidence: 0.92
- **[high] Delegate output enters trusted System context:** `src/auto-reply/continuation/targeting.ts:106`
  - body: Targeted delegate return text is enqueued with `trusted: true`, so agent/delegate-produced text becomes privileged System context in another recipient session. Cross-session delivery should not elevate that text unless a policy explicitly authorizes the sender/recipient edge.
  - confidence: 0.9
