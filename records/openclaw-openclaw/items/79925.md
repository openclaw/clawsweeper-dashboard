The PR adds opt-in continuation tools, durable delegate and post-compaction dispatch, context-pressure awareness, trace stitching, status/docs/protocol updates, tests, and a direct `uuid` dependency.

## Best Possible Solution

Add a live delivery-time `crossSessionTargeting` recheck before normal targeted returns, then require maintainer security/product approval, dependency review, and green CI before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: enable cross-session targeting at delegate spawn, disable it before child completion, and the normal targeted-return delivery path has no live policy recheck. No current-main bug reproduction applies because the continuation feature is not on main.

## Solution Assessment

No for the current PR as-is. The feature direction may be viable, but the normal targeted-return path should mirror the post-compaction delivery-time gate before maintainers evaluate the broader policy fit.
