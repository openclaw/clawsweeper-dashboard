## Summary

Keep open for maintainer review: current main still lets unresolved Slack thread replies fall through as ordinary messages, and this PR is a narrow follow-up to the merged SDK-backed participation store with no blocking review findings found.

## What This Changes

The PR updates Slack thread resolution, prepare/dispatch logging, shared mention-gating compatibility helpers, tests, and the changelog so unresolved thread replies are marked ambiguous and skipped while thread-continuation gating uses SDK-backed participation state.

## Best Possible Solution
