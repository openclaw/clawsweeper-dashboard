## Summary

Keep open. Current main still lets unresolved Slack thread replies flow as ordinary messages, while this PR appears to address that gap with focused Slack resolver/prepare changes and no blocking review finding found in the diff.

## What This Changes

The PR marks unresolved Slack thread replies as ambiguous, drops them during Slack prepare, threads SDK-backed participation state through mention-gating diagnostics/helpers, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR after maintainer review and exact-head validation, keeping the fix scoped to Slack thread resolution, prepare-time ambiguity dropping, and compatibility mention-gating helpers.

## Reproduction Assessment

Yes. On current main, a Slack message with parent_user_id, no thread_ts, and a conversations.history response without thread_ts or an error returns with thread_ts undefined and no ambiguity marker, and prepare has no guard to drop it.

## Solution Assessment

Yes. The PR reuses the merged SDK-backed participation store, avoids the earlier competing JSON-store/fallback direction, and adds an explicit ambiguous-reply marker plus prepare-time drop with focused tests.
