## Summary

Keep open. Current main already has the SDK-backed Slack thread participation store, but it still returns unresolved `parent_user_id` messages as ordinary messages and the deprecated mention-gating helpers do not expose the new thread-continuation signal. The PR appears directionally correct with no blocking functional or security finding from static review, but it needs the required user-facing changelog entry before merge.

## What This Changes

This PR hardens Slack thread continuation by using the existing persistent participation lookup, marking and dropping unresolved missing-`thread_ts` replies as ambiguous, adding diagnostic logging, wiring `threadContinuation` through deprecated mention-gating helpers, and adding focused Slack/channel/Docker coverage.

Land a reviewed version of this PR after adding the missing changelog entry and passing the targeted Slack/channel tests, keeping the implementation scoped to Slack resolver/prepare handling and the shared deprecated helper compatibility surface.
