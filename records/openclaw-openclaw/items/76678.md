## Summary

Keep open. Current main has prompt-level direct-chat `NO_REPLY` guidance and newer GPT/Gemini action-ack guards, but the reported Telegram/Ollama social-acknowledgment repeat path still lacks deterministic or tested acknowledgment-only handling.

## What This Changes

The issue asks OpenClaw to stop repeating a full previous task summary when a Telegram direct-message user sends a short social acknowledgment such as “Okay cool. Thanks!”.

## Best Possible Solution

Add focused direct-chat acknowledgment handling so social acknowledgments choose the existing direct `NO_REPLY` fallback path while preserving actionable approval turns and queued followup behavior.

## Reproduction Assessment

Partly/source-only: the current Telegram DM path for a social acknowledgment is inspectable as a normal model turn with advisory `NO_REPLY` guidance, but I did not live-reproduce the exact Ollama full-repeat output.

## Solution Assessment

Yes on direction: the narrow maintainable fix is to strengthen/test the existing direct-chat no-response behavior or add a small acknowledgment guard, not to add a broad intent classifier or change queue draining.
