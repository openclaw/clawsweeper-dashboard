## Summary

Keep open. Current main has direct-chat `NO_REPLY` fallback guidance, but there is no deterministic or tested handling for acknowledgement-only direct messages, and the related busy-session replay fix does not prove this completed-turn Telegram/Ollama repeat is fixed.

## What This Changes

Direct Telegram acknowledgements like `Okay cool. Thanks!` should produce a brief or no-op response instead of repeating the previous task summary.

## Best Possible Solution

Add focused direct-chat acknowledgement guidance or a small guarded helper so acknowledgement-only turns select the existing `NO_REPLY` fallback path without changing queued followup behavior.

## Reproduction Assessment

Partly. The current source path for a Telegram DM acknowledgement is a normal model turn with prompt-level `NO_REPLY` guidance, so the missing deterministic guard is inspectable; the exact full-repeat output remains model-dependent and was not live-reproduced.

## Solution Assessment

Yes on direction. The narrowest maintainable fix is to strengthen and test the existing direct-chat silent-reply behavior or add a very small acknowledgement guard, not to change queue/drain behavior or add a broad intent classifier.
