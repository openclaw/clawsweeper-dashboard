Keep open. Current main still lacks outbound Telegram album/media-group sending, and this issue is paired with a live draft external PR that uses closing syntax, so cleanup should wait for maintainers to review, port, land, or explicitly supersede that implementation.

## What This Changes

Request support for sending Telegram media albums from the message tool and related outbound delivery paths.

## Best Possible Solution

Implement Telegram-owned sendMediaGroup support for eligible 2-10 media groups across message actions, reply delivery, and outbound payload delivery, with single-media/non-groupable fallbacks, focused tests, docs, and real Telegram proof.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still sends outbound Telegram media one item at a time or sequentially instead of using sendMediaGroup.

## Solution Assessment

Yes on the requested direction, but the exact public API shape is still a maintainer decision. The maintainable fix belongs in the bundled Telegram plugin plus shared message-action schema, preserving existing single-media and non-groupable fallbacks.
