## Summary

Keep open. Current main and the latest release still do not implement outbound Telegram album delivery through `sendMediaGroup`; current code can collect or fan out multiple media URLs sequentially, but that does not solve the requested grouped-album behavior, and maintainers already have an open fork PR plus a narrower OpenClaw PR to evaluate before cleanup closes this issue.

## What This Changes

Request support for sending Telegram media groups/albums from the message tool with multiple media items and a shared caption.

## Best Possible Solution

Implement album support inside the Telegram plugin with `sendMediaGroup` for eligible 2-10 media groups across message actions, reply delivery, and outbound payload delivery, while preserving existing single/sequential fallbacks and documenting the chosen API.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and v2026.5.22 gives high confidence that outbound Telegram media remains singular or sequential rather than grouped through `sendMediaGroup`.

## Solution Assessment

Yes on the requested direction, but the exact public API and fallback policy need maintainer judgment. The maintainable solution belongs in the Telegram plugin send/delivery paths using Telegram album semantics, not a core-only workaround.
