## Summary

Keep open. Current main still lacks outbound Telegram album/media-group sending in the shared message tool schema, Telegram direct action, reply delivery, and outbound payload paths, and this issue is already paired with an open fork PR using closing syntax that maintainers should review, port, land, or explicitly supersede before cleanup.

## What This Changes

Request support for Telegram media groups/albums in the message tool so 2-10 related media items can be sent via Telegram `sendMediaGroup` with a shared caption.

## Best Possible Solution

Review the open fork implementation and either land or port a current-main version in the bundled Telegram plugin that adds `sendMediaGroup` support across direct message actions, reply delivery, and outbound payload delivery, with focused tests and Telegram docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: pass two media URLs through current Telegram reply or outbound payload delivery and current main sends them one at a time; the direct message action also only accepts a single media value.

## Solution Assessment

Unclear for the open fork PR until maintainers review it against current main. The maintainable direction is to integrate album sending into the existing Telegram send/delivery pipeline rather than adding a separate action that leaves existing `mediaUrls` paths sequential.
