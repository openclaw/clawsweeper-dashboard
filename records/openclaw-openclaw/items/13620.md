Keep open: current main still lacks outbound Telegram album/media-group sending, and the issue is already linked to an open external implementation PR with closing syntax. Maintainers should review, port, land, or explicitly supersede that PR before this issue is closed or a separate repair lane is queued.

## What This Changes

Add Telegram media-group/album sending to the message tool so multiple media paths can be sent through Telegram `sendMediaGroup` with shared caption behavior.

## Best Possible Solution

Review the open external implementation and either land, port, or explicitly supersede Telegram plugin support for `sendMediaGroup` with focused tests and Telegram/message-tool docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that message-tool, direct Telegram send, reply delivery, and outbound payload paths still send one media item at a time or sequentially.

## Solution Assessment

Yes on the requested direction, but the exact public API shape still needs maintainer judgment. The maintainable fix belongs in the bundled Telegram plugin plus shared message-action schema, using `sendMediaGroup` only for eligible groups while preserving existing fallbacks.
