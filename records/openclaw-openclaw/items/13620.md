## Summary

Keep open. Current main still lacks outbound Telegram album/media-group sending, and this issue is paired with an open external PR using closing syntax, so maintainers should review, port, land, or explicitly supersede that implementation before cleanup.

## What This Changes

Add Telegram media-group/album support so message sends can deliver 2-10 related photos or videos with a shared caption instead of separate messages.

## Best Possible Solution

Review the open fork implementation and either land or port a current-main version in the bundled Telegram plugin that integrates `sendMediaGroup` across direct message actions, reply delivery, and outbound payload delivery with focused tests and docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: send two media URLs through current Telegram reply or outbound payload delivery and current main sends them one at a time; the direct message action also lacks an array/album parameter.

## Solution Assessment

Unclear for the open fork PR until maintainers review it against current main. The maintainable direction is to integrate album sending into the existing Telegram send/delivery pipeline rather than adding an isolated action that leaves existing `mediaUrls` paths sequential.
