## Summary

Keep open: current main still sends Telegram outbound media one item at a time, and an open external PR with closing syntax remains the implementation candidate for maintainers to review, port, or supersede.

## What This Changes

Requested Telegram album/media-group support for outbound Telegram message-tool sends so multiple related media items can be sent as one album.

## Best Possible Solution

Review the open external implementation and land or port Telegram plugin support that uses sendMediaGroup for eligible 2-10 media groups while preserving single-media and non-groupable fallbacks with focused tests and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks outbound Telegram album sending because the direct action, send helper, reply delivery, and outbound payload adapter still dispatch one media item at a time.

## Solution Assessment

Yes on the direction, but the exact API shape needs maintainer judgment. The maintainable boundary is the bundled Telegram plugin, likely reusing existing mediaUrls payloads and adding sendMediaGroup only for Telegram-eligible groups with sequential fallbacks.
