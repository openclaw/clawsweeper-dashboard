## Summary

Keep open. Current main still lacks outbound Telegram album/media-group sending, and a live external implementation already links back to this issue, so cleanup should wait for maintainers to review, port, land, or explicitly supersede that implementation.

## What This Changes

The issue requests outbound Telegram album/media-group support in the message tool using Telegram sendMediaGroup for grouped media with a shared caption.

## Best Possible Solution

Review the open external implementation and either land, port, or explicitly supersede Telegram plugin support for sendMediaGroup with focused tests and message-tool/Telegram docs.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives high confidence that message-tool, direct Telegram send, reply delivery, and outbound payload paths still send one media item at a time or sequentially.

## Solution Assessment

Yes on the requested direction, but the exact public API shape is still a maintainer decision. The maintainable fix belongs in the bundled Telegram plugin plus shared message-action schema, using sendMediaGroup only for eligible groups while preserving existing fallbacks.
