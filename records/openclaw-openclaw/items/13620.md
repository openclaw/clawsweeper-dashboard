## Summary

Keep open. Current main and v2026.6.5 still do not implement outbound Telegram album delivery through `sendMediaGroup`; existing code supports inbound media-group buffering and sequential outbound `mediaUrls` fan-out, but that does not satisfy the requested grouped-album behavior, and an open fork PR with closing syntax still needs maintainer review, porting, or explicit supersession.

## What This Changes

Request support for Telegram media-group album sends from the message tool, using 2-10 media items with a shared caption.

## Best Possible Solution

Implement album support in the bundled Telegram plugin using `sendMediaGroup` for eligible 2-10 media groups across message actions, reply delivery, and outbound payload delivery, while preserving single/sequential fallbacks and documenting the chosen API.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and v2026.6.5 gives high confidence that outbound Telegram media remains singular or sequential rather than grouped through `sendMediaGroup`.

## Solution Assessment

Yes on the requested direction, but the exact public API and fallback policy need maintainer judgment. The maintainable solution belongs in the Telegram plugin send/delivery paths using Telegram album semantics, not a core-only workaround.
