## Summary

Keep open: current main still lacks outbound Telegram album/media-group sending, and an open fork PR already references this issue with closing syntax, so maintainers should review, port, land, or supersede that implementation path.

## What This Changes

Add Telegram message-tool support for sending multiple media items as a grouped album via Telegram sendMediaGroup.

## Best Possible Solution

Implement Telegram album sending in the bundled Telegram plugin and shared message-action schema, using sendMediaGroup for eligible grouped media while preserving existing single-media fallbacks, reply/thread behavior, and focused docs/tests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives high confidence that the relevant message-tool, direct Telegram send, reply delivery, and outbound payload paths still send one media item at a time or sequentially.

## Solution Assessment

Yes on the requested direction, but the exact public API is still a maintainer decision. The maintainable path is Telegram-plugin support backed by the shared message-action schema, with sendMediaGroup only for eligible groups and existing fallbacks intact.
