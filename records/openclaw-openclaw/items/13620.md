## Summary

Keep open. Current main and v2026.5.20 still lack outbound Telegram sendMediaGroup album support, and the linked implementation work is either an open fork PR with closing syntax or a narrower OpenClaw PR that explicitly leaves album semantics out of scope.

## What This Changes

The issue requests Telegram album/media-group support so the message tool can send 2-10 related photos or videos as one grouped Telegram album with a shared caption.

## Best Possible Solution

Implement Telegram album support inside the Telegram plugin using sendMediaGroup for eligible 2-10 media groups, preserve existing single/sequential fallbacks for non-groupable media, and document the chosen message-tool API.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and v2026.5.20 gives high confidence that outbound Telegram media is still singular or sequential rather than grouped through sendMediaGroup.

## Solution Assessment

Yes on the requested direction, but the exact public API and fallback policy need maintainer judgment. The maintainable solution belongs in the Telegram plugin send/delivery paths using Telegram album semantics, not a core-only workaround.
