## Summary

Keep open. Current main still lacks outbound Telegram album sending: direct actions and send helpers accept one media item, reply and outbound payload paths fan out media arrays sequentially, and an open external implementation with closing syntax remains unmerged.

## What This Changes

Request support for Telegram `sendMediaGroup` albums from the message tool/media delivery path so 2-10 related photos/videos can send as one album with a shared caption.

## Best Possible Solution

Review the open external implementation and land or port a current-main Telegram plugin change that uses `sendMediaGroup` for eligible 2-10 photo/video groups across message-tool, reply delivery, and outbound payload delivery while preserving existing sequential fallbacks.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks outbound Telegram album sending because every relevant path still accepts or dispatches one media item at a time.

## Solution Assessment

Yes on the broad direction, but the exact API shape needs maintainer judgment. The maintainable boundary is the Telegram plugin, likely integrating with existing `mediaUrls` payloads and preserving sequential fallback behavior rather than adding core-specific Telegram policy.
