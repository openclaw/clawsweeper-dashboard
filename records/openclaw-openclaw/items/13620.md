## Summary

Keep open. Current main and the latest release still lack outbound Telegram `sendMediaGroup` album support, and the issue has an open fork PR with closing syntax plus a narrower OpenClaw PR that only handles sequential media URL fan-out.

## What This Changes

The issue requests Telegram album/media-group support in the message tool so 2-10 related media items can be sent as one Telegram album with shared caption semantics.

## Best Possible Solution

Implement Telegram album support in the bundled Telegram plugin using `sendMediaGroup` for eligible media groups, keep existing single and sequential fallbacks for non-groupable media, and document the chosen message-tool API.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and v2026.5.20 gives high confidence that outbound Telegram media is still singular or sequential rather than grouped through `sendMediaGroup`.

## Solution Assessment

Yes on the requested direction, but the exact public API and fallback policy need maintainer judgment. The maintainable solution belongs in the Telegram plugin send/delivery paths using Telegram album semantics, not a core-only workaround.
