## Summary

Keep open. Current main has improved sequential Telegram multi-media fan-out, but it still does not implement outbound Telegram album/media-group delivery through `sendMediaGroup`; the open fork PR with closing syntax and the narrower OpenClaw PR mean maintainers should choose whether to land, port, or supersede the album API before cleanup closes this issue.

## What This Changes

Request to add Telegram media group/album support to the message tool so eligible multiple media items can be sent as one Telegram album with shared caption behavior.

## Best Possible Solution

Implement album support inside the Telegram plugin with `sendMediaGroup` for eligible 2-10 media groups, preserve existing single/sequential fallbacks, and document the chosen message-tool API.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and v2026.5.22 gives high confidence that outbound Telegram media remains singular or sequential rather than grouped through `sendMediaGroup`.

## Solution Assessment

Yes on the requested direction, but the exact public API and fallback policy need maintainer judgment. The maintainable solution belongs in the Telegram plugin send/delivery paths using Telegram album semantics, not a core-only workaround.
