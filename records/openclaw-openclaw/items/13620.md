## Summary

Keep open. Current main and the latest release still lack outbound Telegram album/media-group sending, and the issue is paired with an open fork PR plus a separate narrower open PR that explicitly does not implement album behavior.

## What This Changes

Request support for Telegram media groups/albums in the message tool so 2-10 related photos or videos can be sent via Telegram `sendMediaGroup` with shared caption behavior.

## Best Possible Solution

Review, port, or supersede a current-main Telegram implementation that uses `sendMediaGroup` for eligible 2-10 media groups while preserving existing single-media and fallback behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and `v2026.5.18` gives high confidence that outbound Telegram media is still sent sequentially rather than through a media-group send path.

## Solution Assessment

Yes on the requested direction, but the exact public API and fallback semantics need maintainer judgment. The maintainable path belongs in the bundled Telegram plugin send/delivery surfaces using grammY `sendMediaGroup`, not a core-only workaround.
