## Summary

Keep open. Current main and the latest shipped release still lack outbound Telegram album sending, and a live draft external PR with closing syntax remains the implementation candidate that needs maintainer review for API shape, fallbacks, and live Telegram proof.

## What This Changes

The issue requests Telegram album/media-group support for the message tool so multiple related media files can be sent via Telegram sendMediaGroup with a shared caption.

## Best Possible Solution

Implement Telegram-owned sendMediaGroup support in the existing message-action, final-reply, and outbound payload delivery paths, using grammY's native API and preserving current single-media and non-groupable fallbacks with focused tests, docs, and live Telegram proof.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.12 still send outbound Telegram media one item at a time or sequentially instead of using sendMediaGroup.

## Solution Assessment

Yes on the requested direction, but the exact public API shape is still a maintainer decision. The maintainable fix belongs in the bundled Telegram plugin plus shared message-action schema, using grammY's sendMediaGroup and preserving existing single-media/non-groupable behavior.
