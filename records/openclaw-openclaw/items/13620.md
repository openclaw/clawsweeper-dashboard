Keep open: current main still lacks outbound Telegram album/media-group sending, and an open external PR with closing syntax is already the implementation candidate. Maintainers should review, port, land, or explicitly supersede that PR before this issue is closed or a separate repair lane is queued.

## What This Changes

Requested support for sending multiple Telegram photos/videos as a grouped media album from the message tool.

## Best Possible Solution

Review the open external implementation and either land, port, or explicitly supersede Telegram plugin support for sendMediaGroup with focused tests and message-tool/Telegram docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that message-tool, direct Telegram send, reply delivery, and outbound payload paths still send one media item at a time or sequentially.

## Solution Assessment

Yes on the requested direction, but the exact public API shape still needs maintainer judgment. The maintainable fix belongs in the bundled Telegram plugin plus shared message-action schema, using sendMediaGroup only for eligible groups while preserving existing fallbacks.
