## Summary

Keep open. Current main at 9aa46843ec894adef74e19f9a4ac418d6c5e630f and v2026.5.12 still lack outbound Telegram album sending, and the open draft fork PR https://github.com/julia-meshcheryakova/openclaw/pull/1 has closing syntax, so maintainers should review, port, land, or explicitly supersede that implementation before cleanup closes this issue.

## What This Changes

Request support for sending multiple Telegram photos or videos as one album through the message tool instead of sending each media file separately.

## Best Possible Solution

Implement Telegram-owned sendMediaGroup support across message actions, final reply delivery, and outbound payload delivery while preserving existing single-media and non-groupable fallbacks, with focused tests, docs, and real Telegram proof.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.12 still send outbound Telegram media one item at a time or sequentially instead of using sendMediaGroup.

## Solution Assessment

Yes on the requested direction, but the exact public API shape and fallback semantics still need maintainer judgment. The narrow maintainable path is in the bundled Telegram plugin and shared message-action schema, using grammY's native sendMediaGroup support rather than a separate core-wide media abstraction first.
