Keep open. Current main still lacks outbound Telegram album sending, and the live issue is paired with an open draft fork PR using closing syntax, so maintainers should review, port, land, or explicitly supersede that implementation before cleanup closes this request.

## What This Changes

The issue requests Telegram media-group album support for outbound message-tool sends using Telegram sendMediaGroup with a shared caption.

## Best Possible Solution

Review the open draft PR and either land or port a current-main Telegram implementation that uses sendMediaGroup for eligible 2-10 albums while preserving single-media and non-groupable fallbacks.

## Reproduction Assessment

## Solution Assessment

Yes on the requested direction, but the exact public API and fallback semantics still need maintainer judgment. The narrow maintainable path belongs in the bundled Telegram plugin and existing message-action delivery surfaces using Telegram/grammY-native sendMediaGroup support.
