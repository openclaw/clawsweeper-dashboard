Keep open: current main still has no outbound Telegram `sendMediaGroup`/album path, and an open external PR with closing syntax is already the implementation candidate. Maintainers should review, port, land, or explicitly supersede that PR before a separate repair lane is queued.

## What This Changes

The request is to let the Telegram message tool send multiple related media files as one Telegram album/media group with a shared caption.

## Best Possible Solution

Review the open external implementation and either land, port, or explicitly supersede Telegram plugin support for `sendMediaGroup` with focused tests and Telegram/message-tool docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that message-tool and Telegram outbound paths still send one media item at a time or sequentially.

## Solution Assessment

Yes on the direction, but the exact API shape still needs maintainer judgment. The maintainable boundary is the bundled Telegram plugin plus the shared message-action schema, using `sendMediaGroup` only for eligible groups while preserving current fallbacks.
