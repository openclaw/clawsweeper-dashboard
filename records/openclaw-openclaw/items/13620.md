## Summary

Current main and v2026.6.6 accept multiple media inputs but still send each Telegram attachment separately; native album delivery remains missing. The linked draft implementation is not yet a safe canonical landing path because it lacks live Telegram proof and introduces a parallel action/schema instead of using the existing `mediaUrls` contract.

## What This Changes

Add outbound Telegram album sending for multiple related photo or video attachments through the message tool and delivery pipeline.

## Best Possible Solution

Implement one Telegram-owned album batching path over canonical `mediaUrls`, use it across all eligible outbound entry points, and retain existing per-item delivery for single or unsupported media while preserving delivery metadata.

## Reproduction Assessment

Yes at the source level: pass multiple `mediaUrls` through a Telegram outbound path and current code sends them individually; neither current main nor v2026.6.6 contains a `sendMediaGroup` call.

## Solution Assessment

No for the originally proposed `mediaGroup` parameter or the draft PR's new `send-media-group` action; current main already has canonical `mediaUrls`, so grouping should be a Telegram transport decision rather than a competing tool contract.
