## Summary

Keep this issue open as the canonical Telegram album request. Current main now preserves multiple media URLs and sends them sequentially, but neither current main nor release v2026.6.6 implements Telegram `sendMediaGroup`; the linked draft fork PR remains an implementation candidate that needs maintainer review of its API shape and live Telegram proof.

## What This Changes

Add native Telegram album delivery for eligible multiple outbound media items sent through the message tool.

## Best Possible Solution

Reuse the existing generic `mediaUrls` and reply media-list surfaces, then group eligible 2–10 photo/video items inside the bundled Telegram delivery layer with deterministic per-item fallback and live Telegram proof.

## Reproduction Assessment

Not applicable: this requests new Telegram album semantics rather than reporting broken documented behavior. Source inspection clearly establishes that current main sends multiple items individually.

## Solution Assessment

No, the proposed standalone global `send-media-group` action is not yet shown to be the best solution. Grouping the existing multi-media payload within Telegram would avoid a competing message-tool API and cover direct actions, replies, and outbound payloads consistently.
