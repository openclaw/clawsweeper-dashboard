## Summary

Keep open: current main still sends Telegram media one item at a time, and an open external implementation with closing syntax remains unmerged.

## What This Changes

Request Telegram album/media-group support in the message tool so multiple media items can be sent with a shared caption.

## Best Possible Solution

Review the open external implementation and land or port Telegram plugin support that uses sendMediaGroup for eligible 2-10 media groups while preserving existing single-media and fallback behavior with focused tests and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks outbound Telegram album sending because the direct action, send helper, reply delivery, and outbound payload adapter all still dispatch one media item at a time.

## Solution Assessment

Yes on the direction, but the exact API shape needs maintainer judgment. The maintainable boundary is the bundled Telegram plugin, likely reusing existing mediaUrls payloads and adding sendMediaGroup only for Telegram-eligible groups with sequential fallbacks.
