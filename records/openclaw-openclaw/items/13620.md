## Summary

Keep open. Current main and v2026.6.5 accept multiple Telegram media inputs but still deliver them through sequential single-media calls rather than Telegram albums, and an open draft fork PR with closing syntax remains the paired implementation candidate. Maintainers should review, port, or explicitly supersede that work before starting a competing implementation.

## What This Changes

Requests Telegram album delivery for multiple outbound media items through the message tool.

## Best Possible Solution

Port the useful album behavior into the current bundled Telegram delivery pipeline so eligible batches from existing portable media arrays use grammY `sendMediaGroup`, while retaining the established sequential path for single, unsupported, or failed groups and preserving all delivery metadata.

## Reproduction Assessment

Not applicable as a bug reproduction: this issue requests a new Telegram album capability. Source inspection does provide a high-confidence demonstration of the missing capability because all current multi-media paths resolve to sequential single-media sends.

## Solution Assessment

No, the proposed separate Telegram action is not clearly the best permanent solution because current main already supports portable multi-media inputs. Album grouping should be a Telegram-native delivery optimization over those existing inputs rather than a competing message-tool API.
