## Summary

Keep open: this PR is a plausible launchd bug-fix attempt, but it is not mergeable or correct as-is because it lacks real macOS proof, changes clean-exit relaunch semantics used by restart/update flows, and bundles sandbox Docker changes that are partly superseded and introduce a default-image provenance regression.

## What This Changes

This PR changes the macOS LaunchAgent plist, launchd tests, and service audit checks to use `KeepAlive` with `SuccessfulExit=false`, while also modifying sandbox Docker daemon and image handling in browser, Docker, and doctor paths.

## Best Possible Solution

Ship a focused launchd failure-policy fix for https://github.com/openclaw/openclaw/issues/73673 and related startup-loop reports that bounds failure retries for debugging while preserving clean-exit restart/update relaunches; keep sandbox Docker handling on the merged https://github.com/openclaw/openclaw/pull/73671 path.

## Reproduction Assessment

Yes, source-backed rather than live-reproduced: current main emits `KeepAlive=true`, launchd docs show `SuccessfulExit=false` restarts only after nonzero exits, and current launchd restart/update paths exit 0 for supervisor relaunch.

## Solution Assessment

No. The global `SuccessfulExit=false` switch changes intentional clean-exit relaunch semantics, and the branch also carries stale sandbox changes already superseded by a safer merged PR.
