## Summary

Keep this PR open, but do not merge the current patch. The crash-loop debugging problem is real, yet the proposed global `SuccessfulExit=false` policy breaks current clean-exit restart/update handoff behavior and the branch also carries stale sandbox Docker provenance changes.

## What This Changes

The PR changes macOS LaunchAgent KeepAlive/audit/test expectations to `SuccessfulExit=false` and adds Docker daemon/image handling in sandbox browser, Docker, and doctor paths.

## Best Possible Solution

Replace or revise this into a focused daemon lifecycle fix for #73673 that prevents unbounded crash-loop noise without breaking clean-exit restart/update relaunches, and keep sandbox Docker provenance work separate or aligned with current #73671/#74520 behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the PR's `SuccessfulExit=false` prevents launchd relaunch after exit 0, while current main's supervised restart/update paths intentionally exit 0 under launchd.

## Solution Assessment

No. The global plist switch is not the narrow maintainable fix because current restart and update handoff depends on clean-exit relaunch, and the unrelated sandbox changes should be split or dropped.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve launchd-supervised relaunches:** `src/daemon/launchd-plist.ts:181`
  - body: Changing the LaunchAgent to `SuccessfulExit=false` means launchd will not relaunch after exit 0. Current restart and update paths return supervised mode and then call `exitProcess(0)`, so intentional restarts or updates can leave the gateway stopped.
  - confidence: 0.92
- **[P1] Do not substitute the default sandbox image:** `src/agents/sandbox/docker.ts:315-327`
  - body: When the default sandbox image is missing, this branch pulls `debian:bookworm-slim` and tags it as the OpenClaw image. Current main explicitly refuses that substitution because the OpenClaw image carries Python and helper tooling that plain Debian lacks.
  - confidence: 0.94
- **[P2] Remove duplicate browser imports:** `src/agents/sandbox/browser.ts:129`
  - body: The browser change adds unused Docker imports and a mid-file type import that reimports `SandboxConfig` while `SandboxBrowserConfig` is unused. This should be cleaned up before normal lint/type hygiene can pass.
  - confidence: 0.84
- **[P3] Remove the unused lowercase copy:** `src/commands/doctor-sandbox.ts:89-90`
  - body: `lower` is assigned but never read before calling `isDockerDaemonUnavailable(stderr)`, introducing an avoidable lint failure in the changed doctor path.
