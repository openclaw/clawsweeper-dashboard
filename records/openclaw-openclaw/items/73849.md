## Summary

Keep this PR open, but it should not merge as-is. Current main relies on clean-exit launchd relaunch for restart/update handoff, and the branch also introduces sandbox image provenance and hygiene issues already called out in review.

## What This Changes

The PR changes the generated macOS LaunchAgent plist to `KeepAlive` with `SuccessfulExit=false`, updates launchd audit/tests, and adds Docker daemon/image handling in sandbox/browser/doctor code.

## Best Possible Solution

Replace this with a focused daemon lifecycle fix for #73673/#60885 that preserves explicit restart/update relaunches, and keep sandbox Docker behavior in the already-merged/current-main path or a separate provenance-preserving PR.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main returns supervised mode and exits 0 for launchd restart/update handoff, while `SuccessfulExit=false` tells launchd not to relaunch after successful exit.

## Solution Assessment

No. The plist primitive may help crash-loop debugging, but applying it globally breaks intentional clean-exit relaunch semantics; a narrower lifecycle policy should distinguish crash debugging from restart/update handoff.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve launchd-supervised relaunches:** `src/daemon/launchd-plist.ts:181`
  - body: Changing `KeepAlive` to `SuccessfulExit=false` means launchd will not relaunch after exit 0. Current restart and update handoffs return supervised mode and then exit cleanly under launchd, so intentional restarts or updates can leave the gateway stopped instead of restarted.
  - confidence: 0.92
- **[P1] Do not tag plain Debian as the default sandbox image:** `src/agents/sandbox/docker.ts:317-326`
  - body: When the OpenClaw sandbox image is missing, this branch pulls `debian:bookworm-slim` and tags it as the default OpenClaw sandbox image. Current docs/tests require failing fast because the OpenClaw image includes Python and helper tooling that plain Debian lacks, so sandbox sessions silently degrade.
  - confidence: 0.94
- **[P2] Remove duplicate browser type imports:** `src/agents/sandbox/browser.ts:129`
  - body: The browser change adds a mid-file `SandboxConfig` type import even though `SandboxConfig` is already imported at the top, and `SandboxBrowserConfig` is unused. This should be consolidated before normal lint/type hygiene can pass.
  - confidence: 0.84
- **[P3] Remove the unused lowercase copy:** `src/commands/doctor-sandbox.ts:89`
  - body: `lower` is assigned but never read before calling `isDockerDaemonUnavailable(stderr)`, introducing an avoidable lint failure in the changed doctor path.
  - confidence: 0.86
