## Summary

Keep this PR open, but it is not mergeable as-is: the launchd plist change conflicts with current clean-exit restart/update relaunch behavior, and the branch also carries unrelated sandbox image provenance and import hygiene regressions.

## What This Changes

This PR changes the macOS LaunchAgent plist to use `KeepAlive` with `SuccessfulExit=false`, updates launchd audit/test expectations, and adds Docker daemon/image handling in sandbox, browser, and doctor code.

## Best Possible Solution

A focused lifecycle fix should preserve intentional restart/update relaunches while adding a debuggable crash-loop path, with sandbox daemon/image handling reviewed separately and without changing image provenance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current launchd restart/update paths exit with status 0 after `mode: "supervised"`, while `SuccessfulExit=false` tells launchd not to relaunch after a successful exit.

## Solution Assessment

No. The proposed plist primitive may help crash-loop debugging, but applying it globally is not the narrow maintainable fix because current restart and update handoff paths rely on clean-exit relaunch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve launchd-supervised relaunches:** `src/daemon/launchd-plist.ts:181`
  - body: Changing the LaunchAgent to `SuccessfulExit=false` means launchd will not relaunch after exit 0. Current restart and update paths return supervised mode and then exit cleanly under launchd, so intentional restarts or updates can leave the gateway stopped.
  - confidence: 0.92
- **[P1] Do not substitute the default sandbox image:** `src/agents/sandbox/docker.ts:315-327`
  - body: When the default sandbox image is missing, this branch pulls `debian:bookworm-slim` and tags it as the OpenClaw image. The current contract requires failing because the OpenClaw image includes Python and helper tooling that plain Debian lacks, so this silently degrades sandbox behavior.
  - confidence: 0.94

  - body: The browser change adds unused Docker imports and a mid-file type import that reimports `SandboxConfig` while `SandboxBrowserConfig` is unused. This should be cleaned up before normal lint/type hygiene can pass.
  - confidence: 0.84
