## Summary

Keep open, but not mergeable as-is. The launchd change conflicts with current supervised restart/update handoff, the branch mixes in unrelated sandbox Docker provenance changes, and the external PR still lacks after-fix real behavior proof from a macOS LaunchAgent setup.

## What This Changes

The PR changes macOS LaunchAgent plist generation from boolean `KeepAlive=true` to `SuccessfulExit=false`, updates launchd tests/audit checks, and adds Docker daemon/image handling in sandbox-related code.

## Best Possible Solution

Replace this branch with a focused launchd failure-policy change for https://github.com/openclaw/openclaw/issues/73673 and https://github.com/openclaw/openclaw/issues/66436 that preserves intentional supervised restarts and leaves sandbox Docker handling to the already-merged https://github.com/openclaw/openclaw/pull/73671 path.

## Reproduction Assessment

Yes. Source inspection shows launchd still uses unconditional `KeepAlive=true` for nonzero startup failures, while current restart/update handoff also intentionally exits 0 and depends on `KeepAlive` relaunching the service.

## Solution Assessment

No. The proposed global `SuccessfulExit=false` switch is not the best fix because it breaks intentional clean-exit relaunches, and the PR also bundles unrelated sandbox Docker changes that have already been superseded in a safer merged PR.
