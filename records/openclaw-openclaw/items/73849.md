## Summary

Keep this PR open, but the current branch should not merge as-is: the LaunchAgent change breaks current supervised restart/update semantics, and the branch also carries unrelated sandbox Docker provenance changes that conflict with current main's no-substitution contract.

## What This Changes

The PR changes macOS LaunchAgent KeepAlive to `SuccessfulExit=false`, updates launchd audit/test expectations, and adds Docker daemon/image handling in sandbox, browser, and doctor code.

## Best Possible Solution

Replace this with a focused daemon lifecycle fix for #73673 that preserves restart/update relaunches and keeps sandbox Docker behavior in a separate, provenance-preserving review.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main exits 0 for launchd-supervised restart/update paths, while `SuccessfulExit=false` tells launchd not to relaunch after a successful exit.

## Solution Assessment

No. The plist primitive may help crash-loop debugging, but applying it globally is not the narrow maintainable fix because current restart/update handoff depends on clean-exit relaunch; the unrelated sandbox changes should be split or dropped.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve launchd-supervised relaunches:** `src/daemon/launchd-plist.ts:181`
  - body: Changing the LaunchAgent to `SuccessfulExit=false` means launchd will not relaunch after exit 0. Current restart and update paths return supervised mode and then exit cleanly under launchd, so intentional restarts or updates can leave the gateway stopped.
  - confidence: 0.92
- **[P1] Do not substitute the default sandbox image:** `src/agents/sandbox/docker.ts:315-327`
  - body: When the default sandbox image is missing, this branch pulls `debian:bookworm-slim` and tags it as the OpenClaw image. The current contract requires failing because the OpenClaw image includes Python and helper tooling that plain Debian lacks, so this silently degrades sandbox behavior.
  - confidence: 0.94
- **[P2] Remove duplicate browser imports:** `src/agents/sandbox/browser.ts:129`
  - body: The browser change adds unused Docker imports and a mid-file type import that reimports `SandboxConfig` while `SandboxBrowserConfig` is unused. This should be cleaned up before normal lint/type hygiene can pass.
  - confidence: 0.84
- **[P3] Remove the unused lowercase copy:** `src/commands/doctor-sandbox.ts:89`
  - body: `lower` is assigned but never read before calling `isDockerDaemonUnavailable(stderr)`, introducing an avoidable lint failure in the changed doctor path.
  - confidence: 0.86
