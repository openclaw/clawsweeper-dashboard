## Summary

Keep open. This PR is not ready to merge because it changes launchd clean-exit restart semantics, bundles already-superseded sandbox Docker changes with a concrete image-provenance regression, and still lacks after-fix macOS LaunchAgent proof.

## What This Changes

This PR changes macOS LaunchAgent plist generation, tests, and service audit checks to use `KeepAlive` with `SuccessfulExit=false`, while also modifying sandbox Docker daemon/image handling in browser, Docker, and doctor paths.

## Best Possible Solution

Revise or replace this with a focused launchd failure-policy fix for https://github.com/openclaw/openclaw/issues/73673 and https://github.com/openclaw/openclaw/issues/66436 that preserves intentional clean-exit restarts and leaves sandbox Docker handling to the merged https://github.com/openclaw/openclaw/pull/73671 path.

## Reproduction Assessment

Yes, source-backed but not live-reproduced on macOS. Current main still emits launchd `KeepAlive=true`, and the PR's proposed `SuccessfulExit=false` conflict with clean-exit restart/update paths is visible in `src/cli/gateway-cli/run-loop.ts`.

## Solution Assessment

No. The global `SuccessfulExit=false` switch is not the best fix because it changes intentional clean-exit relaunch behavior, and the branch also carries unrelated sandbox changes already superseded by a safer merged PR.
