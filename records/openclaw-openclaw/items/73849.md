## Summary

Keep this PR open, but do not merge it as-is. The launchd change conflicts with current restart/update handoff behavior that still depends on clean-exit relaunch, and the branch also introduces unrelated sandbox image-provenance and hygiene regressions already called out in prior review comments.

## What This Changes

The PR changes the macOS LaunchAgent KeepAlive policy and audit/test expectations to `SuccessfulExit=false`, while also modifying sandbox Docker/browser/doctor handling for Docker daemon and image availability.

## Best Possible Solution

Use this branch as context for #73673, but replace it with a focused daemon lifecycle fix that covers crash-loop debuggability, explicit stop, manual restart, update restart, and launchd handoff semantics together. Keep the sandbox Docker/browser/doctor behavior out of this PR unless it is split into a separate review that preserves the no-silent-Debian-substitution contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main returns supervised mode under launchd and exits 0 for restart/update handoff, while `SuccessfulExit=false` tells launchd not to restart after a successful exit. The sandbox regression is also reproducible by missing the default sandbox image while Docker is available.

## Solution Assessment

No. `SuccessfulExit=false` may be one useful launchd primitive for crash debugging, but applying it alone is not the narrow maintainable fix because current restart/update paths depend on clean-exit relaunch; the unrelated sandbox image changes are a separate and security-sensitive concern.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve launchd-supervised relaunches:** `src/daemon/launchd-plist.ts:181`
  - body: Changing the LaunchAgent to `SuccessfulExit=false` means launchd will not relaunch after exit 0. Current restart and update paths return supervised mode and the run loop exits cleanly under launchd, so intentional restarts or updates can leave the gateway stopped.
  - confidence: 0.92

  - confidence: 0.94

  - body: The browser change adds unused Docker imports and a mid-file type import that reimports `SandboxConfig` while `SandboxBrowserConfig` is unused. This should be cleaned up before normal lint/type hygiene can pass.
