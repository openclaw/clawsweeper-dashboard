Keep open: the patch is focused and I found no line-level correctness defect, but it still needs live or equivalent Telegram-topic proof and maintainer acceptance of changing visible reply admission from a bounded skip to an unbounded wait.

## What This Changes

The PR makes visible reply dispatches wait indefinitely behind an active same-session reply operation and adds a Telegram forum-topic regression test for that admission path.

## Best Possible Solution

Land the focused admission fix after live or equivalent Telegram-topic proof shows visible messages wait and deliver after the active reply clears while heartbeat/background turns still skip.

## Reproduction Assessment

Yes at source level: current main passes no waitTimeoutMs from dispatch admission, and the registry default can return false after 15s, producing the reply-operation-active skip path. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Mostly yes, but not proven enough for merge: the patch reuses the existing admission timeout hook and keeps heartbeat skipping intact. The remaining question is whether maintainers accept an unbounded visible wait or want a bounded/diagnostic variant before live Telegram proof lands.
