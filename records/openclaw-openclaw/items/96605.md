## Summary

Keep this PR open: current main and the latest release still read Google Veo REST operation JSON with an unbounded `response.text()`, while the PR applies the existing bounded-reader pattern with focused regression coverage. I found no blocking correctness finding, but the fixed 16 MiB operation-response cap remains a maintainer compatibility choice before merge.

## What This Changes

The PR replaces the Google Veo REST operation `response.text()` read with a 16 MiB `readResponseWithLimit` call and adds streaming regression coverage for the REST fallback path.

## Best Possible Solution

Land the bounded read after a maintainer accepts or tunes the 16 MiB Veo operation-metadata ceiling, keeping the fix local to the Google plugin.
