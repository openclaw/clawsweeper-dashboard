## What This Changes

The issue requests that `openclaw nodes camera snap --device-id <iphone-continuity-id> --facing front` select the iPhone Continuity front camera instead of always capturing from the rear camera.

## Best Possible Solution

Define the macOS Continuity Camera `deviceId` plus `facing` contract, then either select distinct front/back Continuity devices when AVFoundation exposes them or return a clear unsupported-facing diagnostic with docs for single external Continuity devices.
