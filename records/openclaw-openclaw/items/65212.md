## Summary

Keep open: current main still leaves direct QMD file paths as collection roots, while this PR is a narrow resolver/test fix with sufficient terminal proof, green checks, and no blocking review findings.

## What This Changes

The PR changes memory-host-sdk QMD custom path resolution to turn existing direct file paths into parent-directory collections with escaped exact filename masks, plus regression coverage.

## Best Possible Solution

Land this resolver-level normalization after normal maintainer merge approval so QMD receives a directory root and literal filename mask for direct-file entries.

Yes. Source inspection gives a high-confidence current-main path: configure `memory.qmd.paths` with an existing file, and the resolver emits that file as the QMD collection root with `**/*.md` while QMD scans the mask relative to the collection root.

## Solution Assessment

Yes. Normalizing existing file targets in `backend-config` is the narrow maintainable fix because the QMD manager already consumes prepared collection path and mask facts.
