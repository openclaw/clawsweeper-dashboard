## Summary

Keep open for normal maintainer landing: the patch fixes documented QMD file-path behavior in the resolver, has focused regression coverage, supplied terminal proof, green relevant checks, and no blocking review findings.

## What This Changes

This PR updates QMD custom path resolution so existing direct file paths become parent-directory collections with escaped exact-file masks, and adds a regression test for glob metacharacters in the filename.

## Best Possible Solution

Land the resolver-level normalization with the included regression test so QMD receives a directory root and escaped exact-file mask for configured direct files.

## Reproduction Assessment

Yes. I did not run a local test in this read-only pass, but current-main source plus QMD's `fastGlob` contract show why a direct file path is scanned as the root rather than as an exact mask, and the PR body supplies after-fix terminal proof.

## Solution Assessment

Yes. Normalizing in backend-config is the narrow maintainable fix because QmdManager only receives the prepared path and pattern, and QMD interprets the pattern relative to that path.
