## Summary

Keep open: current main still has the QMD direct-file collection gap, and this PR has the right fix shape but remains blocked by literal brace handling and missing real QMD behavior proof.

## What This Changes

The branch changes QMD custom collection resolution to stat existing file paths, convert file targets to parent-directory collections with escaped basename patterns, and add a backend-config regression test.

## Best Possible Solution

Land the parent-directory normalization with a fully literal-safe filename pattern strategy, brace regression coverage, and redacted real QMD setup proof.

## Reproduction Assessment

Yes, at source level: configure QMD with a custom path that points to an existing Markdown file, and current main leaves that file as the collection root with a glob pattern. I did not run a live QMD sidecar in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right maintainable fix shape, but this PR should make file-derived patterns literal-safe for braces and include real QMD behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Escape braces in file-derived QMD patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
