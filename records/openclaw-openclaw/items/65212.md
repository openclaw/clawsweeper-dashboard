## Summary

Keep open for normal maintainer review and merge consideration: current main still has the direct-file QMD path bug, while this focused PR implements the resolver fix, has sufficient terminal real-behavior proof, and I found no blocking correctness or security issue.

## What This Changes

The PR normalizes existing QMD custom collection entries that point directly at files into parent-directory collections with escaped exact-file masks and adds regression coverage.

## Best Possible Solution

Land the resolver-level normalization and regression test so QMD always receives a directory root plus literal filename mask for existing direct-file paths.

## Reproduction Assessment

Yes. Current main is source-reproducible: the resolver passes an existing file path as the collection root with a glob pattern, while QMD scans that pattern relative to the collection path.

## Solution Assessment

Yes. Preparing the parent directory and escaped basename in backend-config is the narrow maintainable fix because QmdManager already forwards prepared path and mask values to QMD.
