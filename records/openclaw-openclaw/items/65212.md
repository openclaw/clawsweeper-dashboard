## Summary

Keep open: current main still lacks the documented direct-file QMD path normalization, and this PR is a plausible narrow bug fix, but the current head still leaves brace expansion active for exact filename patterns and lacks contributor real behavior proof.

## What This Changes

The PR changes QMD memory backend config resolution so existing file paths become parent-directory collections with escaped filename patterns, plus a regression test for bracket metacharacters.

## Best Possible Solution

Land the normalization after file-derived patterns are literal-safe for all QMD glob magic, including braces, with focused regression coverage and redacted real QMD behavior proof from the contributor.

## Reproduction Assessment

Yes, at source level: current main leaves an existing direct file as the QMD collection path despite docs saying memory paths may be files. The PR-specific brace edge is also source-reproducible from minimatch and fast-glob contracts, though no live QMD run was provided.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but the current implementation should make the derived filename fully literal-safe, such as by passing magicalBraces, before merge.
