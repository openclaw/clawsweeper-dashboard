## Summary

Keep this PR open: current main still has the direct-file QMD collection bug, the PR is a focused resolver/test fix, and I did not find a blocking correctness or security issue.

## What This Changes

The PR normalizes existing direct-file QMD custom paths into parent-directory collections with escaped exact-file masks and adds regression coverage for brace/bracket filenames.

## Best Possible Solution

Land the resolver-level normalization so QMD continues receiving prepared collection path and mask values without adding policy to QmdManager.

## Reproduction Assessment

Yes. From source, configuring memory.qmd.paths with an existing file on current main emits that file as the collection root with **/*.md, while QMD scans the pattern relative to the root; the PR body and regression test show the after-fix path.

## Solution Assessment

Yes. Normalizing existing file targets in backend-config is the narrow maintainable fix because QmdManager already forwards prepared path and mask values to QMD, and escaping the basename preserves exact-file matching under fast-glob.
