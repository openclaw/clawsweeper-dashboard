## Summary

Keep open. Current main still lacks direct-file normalization for QMD custom paths, and the PR is useful, but the latest branch is not merge-ready because it passes a raw basename into QMD's glob-pattern field, so direct files with glob metacharacters are not matched literally.

## What This Changes

The PR branch modifies `packages/memory-host-sdk/src/host/backend-config.ts` so custom QMD paths that stat as files become parent-directory collections with the file basename as the collection pattern.

## Best Possible Solution

Keep the package-owned resolver change, but make file-derived patterns literal and glob-safe for QMD, preserve directory and unresolved-path behavior, add regression coverage for ordinary and metacharacter filenames, and include the required user-facing changelog entry before merge.

## Reproduction Assessment

Yes. Configure QMD with an existing direct file such as `notes[1].md`; the PR normalizes it to the parent directory with raw pattern `notes[1].md`, which OpenClaw passes to QMD as `--mask`/`--glob` syntax instead of an exact filename.

## Solution Assessment

No. Parent-directory normalization is the right shape, but the current implementation is not the best fix until the basename is escaped or otherwise represented literally and the metacharacter case is covered by tests.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Escape direct-file basenames before using them as patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
