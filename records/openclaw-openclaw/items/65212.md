## Summary

Keep open: this PR addresses a real QMD direct-file path bug, but the current patch still leaves brace-containing filenames vulnerable to glob expansion and the external PR lacks after-fix real QMD behavior proof.

## What This Changes

The PR changes QMD custom collection resolution to convert existing direct file paths into parent-directory collections with escaped basename patterns and adds a bracket-filename regression test.

## Best Possible Solution

Land the parent-directory normalization after escaping all file-basename glob metacharacters, adding brace regression coverage, and receiving redacted real QMD indexing proof.

## Reproduction Assessment

Yes, at source level: current main documents QMD paths as directories or files, but `resolveCustomPaths` keeps an existing file path as the collection root with a glob pattern. I did not run a live QMD sidecar in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but this implementation should make file-derived patterns fully literal-safe for braces and include real QMD behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Escape braces in file-derived QMD patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
  - body: `escapeGlobPattern(path.basename(resolved))` leaves `{` and `}` unescaped by default in minimatch, while QMD indexes the mask through fast-glob with brace expansion enabled. A direct file named `notes{a,b}.md` can be treated as alternatives instead of the literal filename, so escape braces too and add brace coverage.
  - confidence: 0.91
