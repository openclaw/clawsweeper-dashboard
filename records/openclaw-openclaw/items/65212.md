## Summary

Keep this PR open: the updated head addresses the earlier bracket-glob review concern, but file-derived patterns still leave brace expansion active and the user-facing fix still needs a changelog entry.

## What This Changes

The PR normalizes existing QMD file-path collection entries into parent-directory collections with escaped exact-file patterns and adds regression coverage for bracket metacharacters.

## Best Possible Solution

Land the normalization after file-derived patterns are literal-safe for all fast-glob magic, including brace expansion, and add the required user-facing changelog entry without changing directory collection behavior.

## Reproduction Assessment

Yes. Source inspection shows current main treats an existing direct file path as the QMD collection root, and the remaining PR edge case can be reproduced with a direct file such as notes{a,b}.md because fast-glob brace expansion remains active.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but the latest PR should escape braces with minimatch's magicalBraces option or an equivalent literal pattern strategy before merge.
