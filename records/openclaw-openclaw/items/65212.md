## Summary

Keep open: this is a valid QMD direct-file path bug and the PR has the right normalization shape, but the current patch still leaves brace-containing filenames active as glob patterns and the contributor has not supplied real behavior proof.

## What This Changes

The PR normalizes existing direct-file QMD collection paths into parent-directory collections with an escaped basename pattern and adds a backend-config regression test.

## Best Possible Solution

Land the normalization after deriving a fully literal-safe filename pattern for QMD glob parsing and after the contributor adds redacted real QMD behavior proof.

## Reproduction Assessment

Yes, at source level: current main leaves an existing direct-file path as the QMD collection root even though the documented extra path contract allows files. I did not run a live QMD setup in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but the implementation must make file-derived patterns literal-safe for braces before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Escape braces in file-derived QMD patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
  - body: `minimatch.escape()` does not escape `{` or `}` unless `magicalBraces` is true, while this pattern is later passed to QMD as `--mask`/`--glob` and fast-glob enables brace expansion by default. A direct file named `notes{a,b}.md` can still be interpreted as alternatives instead of one literal filename, so use a fully literal pattern strategy and add brace coverage.
  - confidence: 0.91
