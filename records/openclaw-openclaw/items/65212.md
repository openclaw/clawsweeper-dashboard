## Summary

Keep open. The PR targets the right package-owned QMD config resolver and fixes the simple direct-file shape, but the latest patch still stores the raw filename as a QMD pattern, so filenames with glob metacharacters are not treated as exact file targets.

## What This Changes

The PR changes the memory-host SDK QMD backend config resolver so custom paths that stat as files become parent-directory collections with the basename as the collection pattern.

## Best Possible Solution

Keep the package-owned resolver change, but make direct-file targets produce a literal glob-safe filename pattern or another QMD-supported literal-file representation, while preserving directory-path behavior, collection naming, and dedupe semantics. Add focused regression coverage for a normal file and a filename containing glob metacharacters.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a QMD custom path pointing at an existing file named `notes[1].md`: the PR normalizes it to parent directory plus raw pattern `notes[1].md`, which QMD consumes as glob syntax rather than an exact filename.

## Solution Assessment

No. Parent-directory normalization is the right surface, but the proposed implementation is not the best fix until it escapes or otherwise represents the basename literally and adds package-level regression tests for glob metacharacters.
