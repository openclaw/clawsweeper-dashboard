## Summary

Keep this PR open for normal maintainer landing: current main still has the documented direct-file QMD path bug, and the latest PR head fixes the resolver path/mask shape with focused coverage, sufficient real behavior proof, and green live checks.

## What This Changes

The PR normalizes QMD custom collection entries that point at an existing file into a parent-directory collection with an escaped exact-filename mask, and adds resolver regression coverage.

## Best Possible Solution

Land the resolver-level normalization so QMD receives prepared collection path and literal mask facts while QmdManager remains a consumer of those resolved facts.

## Reproduction Assessment

Yes. Source inspection shows current main forwards a direct file path as the QMD collection root while QMD scans a glob mask under that root; the PR body also supplies after-fix QMD and OpenClaw terminal proof.

## Solution Assessment

Yes. Normalizing in backend-config is the narrowest maintainable fix because the QMD manager already forwards the resolved collection path and mask directly to QMD.
