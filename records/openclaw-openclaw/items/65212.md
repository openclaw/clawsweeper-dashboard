## Summary

Keep open: this is a focused bug fix for an existing QMD config contract. Current main still sends direct file paths to QMD as collection roots, while the PR normalizes them at the resolver boundary with escaped exact-file masks, includes targeted regression coverage, has sufficient terminal proof, and shows no blocking review finding.

## What This Changes

The PR updates QMD memory backend path resolution so existing direct file entries become parent-directory collections with escaped exact filename masks, with a regression test.

## Best Possible Solution

Land the resolver-level normalization so QMD manager continues receiving prepared collection path and mask facts for both directory and direct-file custom entries.

## Reproduction Assessment

Yes. Current-main source shows an existing file path in memory.qmd.paths is emitted as the QMD collection root with **/*.md, and the PR body supplies after-fix OpenClaw and QMD terminal proof for the direct-file path.

## Solution Assessment

Yes. Normalizing existing file targets in backend-config to parent directory plus escaped basename is the narrow maintainable fix because QMD manager already forwards prepared path and mask values to QMD.
