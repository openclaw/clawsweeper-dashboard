## Summary

Keep open: the bug is valid and the PR is close, but the current head still does not make brace-containing direct filenames literal-safe and it lacks contributor-provided real QMD behavior proof.

## What This Changes

The PR changes QMD custom collection resolution so existing file paths are collected from their parent directory with an exact filename pattern, and adds a regression test for bracket escaping.

## Best Possible Solution

Land the parent-directory normalization with fully literal-safe file-derived patterns, brace regression coverage, and redacted real QMD setup proof.

## Reproduction Assessment

Yes, at source level: current main documents QMD paths as directories or files, but `resolveCustomPaths` keeps an existing file path as the collection root with a glob pattern. I did not run a live QMD sidecar in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but this implementation should make file-derived patterns fully literal-safe for braces and include real QMD behavior proof before merge.
