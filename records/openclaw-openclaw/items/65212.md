## Summary

Keep this PR open for maintainer merge review. Current main still leaves direct QMD file paths as collection roots even though the config contract allows files, while this PR is narrow, has sufficient terminal proof, and I found no concrete correctness defect in the patch.

## What This Changes

The PR stats configured QMD paths, rewrites existing file targets to parent-directory collections with escaped exact-file masks, and adds regression coverage for brace/bracket filenames.

## Best Possible Solution

Land the resolver-level normalization and regression test so QMD always receives a directory root and literal mask for existing direct file targets.

## Reproduction Assessment

Yes. Current main is source-reproducible by configuring memory.qmd.paths with an existing file path: the resolver emits that file as the collection root with **/*.md, while QMD scans the mask relative to the collection root.

## Solution Assessment

Yes. Normalizing existing file targets in backend-config is the narrow maintainable fix because QMD manager already forwards prepared collection path and mask facts directly to QMD.
