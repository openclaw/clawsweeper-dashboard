## Summary

Keep open. Current main still lacks the direct-file QMD normalization, and the latest PR branch is useful but still has a correctness blocker because file basenames are stored as QMD glob patterns rather than literal filenames.

## What This Changes

The PR changes the memory host QMD custom path resolver so existing file paths become parent-directory collections with the file basename as the collection pattern.

## Best Possible Solution

Land the resolver change only after file-derived patterns are literal and glob-safe for QMD, with focused regression coverage and the required changelog entry.

## Reproduction Assessment

Yes, from source inspection. Configure QMD with an existing direct file such as `notes[1].md`; current main keeps the file path as the collection root, while the PR normalizes to the parent directory but passes `notes[1].md` as glob syntax.

## Solution Assessment

No. Parent-directory normalization is the right shape, but this PR is not the best final fix until the derived basename is literal/glob-safe and the metacharacter case is covered by tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Escape direct-file basenames before using them as patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:309`
  - body: This field is passed to `qmd collection add` as `--mask`/`--glob`, and QMD indexes with fast-glob. A direct file named `notes[1].md`, `draft?.md`, or `a*b.md` can be missed or overmatched instead of indexed exactly; escape the basename or use a literal-file representation and cover that case in the regression test.
  - confidence: 0.92
