## Summary

Keep open for normal maintainer landing: current main still leaves the documented direct-file QMD path shape unresolved, while this PR is a narrow source-backed fix with sufficient OpenClaw/QMD terminal proof and no blocking review findings.

## What This Changes

The PR updates QMD custom path resolution so existing direct-file entries become parent-directory collections with escaped exact-file masks, and adds a regression test for glob metacharacter filenames.

## Best Possible Solution

Land the resolver-level normalization and keep QMD manager behavior as a simple consumer of prepared collection path and mask facts.

## Reproduction Assessment

Yes. Source inspection shows current main sends a direct file path as the QMD collection root with a markdown glob, while QMD scans `globPattern` under `cwd: collectionPath`; the PR body also supplies a real OpenClaw/QMD terminal proof after the patch.

## Solution Assessment

Yes. Normalizing in backend-config to parent directory plus escaped basename is the narrow maintainable fix because the QMD manager already forwards prepared path and mask facts to QMD.
