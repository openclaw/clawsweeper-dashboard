## Summary

Keep open; this is a focused bug-fix PR that is not implemented on current main and appears ready for normal maintainer review. Current main still passes direct file paths through as QMD collection roots, while the PR normalizes existing files into parent-directory collections with escaped exact-file masks; supplied terminal proof and CI are sufficient, and I found no blocking findings.

## What This Changes

This PR updates QMD custom path resolution so existing direct file paths become parent-directory collections with escaped exact-file masks, and adds a regression test.

## Best Possible Solution

Land the resolver-level normalization with the existing regression test so QMD receives a parent collection root and escaped exact-file mask for configured direct files.

## Reproduction Assessment

Yes. Source inspection shows current main passes an existing direct file as the QMD collection root with a glob pattern, while QMD evaluates the pattern relative to collectionPath; the PR body supplies after-fix OpenClaw and QMD terminal proof.

## Solution Assessment

Yes. Normalizing in backend-config is the narrow maintainable fix because QmdManager already owns only the prepared path and pattern handoff to QMD, and escaping the basename matches the QMD fast-glob mask contract.
