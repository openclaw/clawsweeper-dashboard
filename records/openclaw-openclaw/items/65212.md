## What This Changes

The PR normalizes existing direct-file QMD custom collection paths into parent-directory collections with escaped exact-file masks and adds regression coverage for glob metacharacter filenames.

## Best Possible Solution

Land the resolver-level normalization and keep QMD manager behavior as a consumer of prepared collection path and mask facts.

## Solution Assessment

Yes. Normalizing in backend-config to parent directory plus escaped basename is the narrow maintainable fix because the QMD manager already forwards prepared collection path and mask facts to QMD.
