## What This Changes

The PR rewrites existing direct-file QMD custom memory paths into parent-directory collections with escaped exact-file masks and adds a regression test for glob metacharacter filenames.

## Best Possible Solution

Land the resolver-level normalization with its regression test once maintainers complete ordinary merge approval.

## Reproduction Assessment

Yes. By source inspection, configuring `memory.qmd.paths` with an existing file such as `notes{a,b}[1].md` makes current main emit that file as the collection root while QMD applies the mask under `cwd=collectionPath`.

## Solution Assessment

Yes. Normalizing in backend-config before QmdManager hands prepared path and pattern values to QMD is the narrow maintainable fix.
