## Summary

Keep open: this PR targets a real source-level QMD direct-file bug, but the latest head still is not literal-safe for brace-containing filenames and the contributor has not provided after-fix proof from a real QMD setup.

## What This Changes

The PR normalizes QMD custom collection entries that point at existing files into parent-directory collections with escaped exact-file patterns and adds a regression test.

## Best Possible Solution

Land the parent-directory normalization with fully literal-safe filename patterns, brace regression coverage, and redacted real QMD behavior proof.

## Reproduction Assessment

Yes, at source level: configure the QMD backend with a custom path that points to an existing Markdown file, and current main leaves that file as the collection root with the default or configured glob pattern. I did not run a live QMD sidecar in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but the implementation should make file-derived patterns literal-safe for braces and provide real QMD behavior proof before merge.
