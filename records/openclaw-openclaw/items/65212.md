## Summary

Keep open: this PR addresses a real documented QMD direct-file path bug, but the latest head still leaves brace-containing filenames non-literal and the external contributor has not provided after-fix real QMD behavior proof.

## What This Changes

The PR changes QMD custom path resolution to convert existing direct file paths into parent-directory collections with escaped basename patterns and adds a bracket-filename regression test.

## Best Possible Solution

Land the parent-directory normalization after making file-derived patterns fully literal-safe for braces, adding brace regression coverage, and receiving redacted real QMD indexing proof.

## Reproduction Assessment

Yes, at source level: current main documents file paths as valid memory inputs, but `resolveCustomPaths` keeps an existing file path as the collection root with a glob pattern that QMD receives directly. I did not run a live QMD sidecar in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but this implementation should escape brace metacharacters as literal filename characters and include real QMD behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Escape braces in file-derived QMD patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
