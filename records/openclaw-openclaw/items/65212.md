## Summary

Keep open: the PR targets a real source-level QMD bug, but the latest diff still leaves brace-containing filenames active as glob patterns and the external contributor has only test/CI proof, not real QMD behavior proof.

## What This Changes

The PR normalizes QMD custom collection entries that point to files into parent-directory collections with escaped basename patterns and adds a backend-config regression test.

## Best Possible Solution

Land the direct-file normalization with a fully literal-safe filename pattern, brace regression coverage, and redacted real QMD behavior proof.

## Reproduction Assessment

Yes, at source level: current main leaves an existing direct-file path as the QMD collection root even though the docs allow additional directories or files to index. I did not run a live QMD setup in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but the implementation should make file-derived patterns literal-safe for braces and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Escape braces in file-derived QMD patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
  - body: `minimatch.escape()` without `magicalBraces` leaves `{` and `}` unchanged, while this value is later passed to QMD as `--mask`/`--glob` and glob-style matching expands braces by default. A direct file like `notes{a,b}.md` can still be treated as alternatives rather than one literal filename, so use a fully literal pattern strategy and add brace coverage.
  - confidence: 0.91
