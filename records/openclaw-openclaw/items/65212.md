## Summary

Keep this PR open: it fixes a real current-main QMD path-resolution bug, but the latest patch still leaves brace characters active in file-derived glob patterns and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR normalizes QMD custom path entries that resolve to files into parent-directory collections with escaped filename patterns and adds a regression test.

## Best Possible Solution

Land the direct-file normalization after file-derived patterns are literal-safe for all QMD glob magic, with focused brace coverage and redacted real QMD behavior proof.

## Reproduction Assessment

Yes, at source level: current main leaves an existing direct file as the QMD collection root even though docs allow file paths, and QMD later consumes the pattern as a glob/mask. I did not run a live QMD setup in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but the current patch should make file-derived patterns fully literal-safe, such as by passing magicalBraces, before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Escape braces in file-derived QMD patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
  - body: `minimatch.escape()` leaves `{` and `}` unescaped unless `magicalBraces` is set, while this value is later passed to QMD as `--mask`/`--glob` and QMD scans with fast-glob brace expansion enabled. A direct file named `notes{a,b}.md` can still be interpreted as alternatives instead of one literal filename, so use a fully literal pattern strategy and add brace coverage.
  - confidence: 0.91
