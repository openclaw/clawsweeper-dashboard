## Summary

Keep open: the reported QMD bug is valid on current main, but this PR is not merge-ready because contributor real-behavior proof is still missing and the patch leaves literal-pattern and dependency-declaration issues.

## What This Changes

The PR normalizes existing direct QMD file paths into parent-directory collections with exact basename patterns and adds a regression test for bracket filenames.

## Best Possible Solution

Land the normalization after making file-derived masks fully literal-safe, using a declared runtime dependency or local helper, and receiving redacted real QMD indexing proof.

## Reproduction Assessment

Yes, at source level: current main documents extra paths as directories or files, but `resolveCustomPaths` keeps an existing file path as the collection root and passes a markdown glob to QMD. I did not run a live QMD sidecar in this read-only review.

## Solution Assessment

No. Parent-directory normalization is the right fix shape, but this implementation should escape brace metacharacters, declare or avoid the new dependency, and include real QMD behavior proof before merge.

## Review Findings

- **[P2] Declare the new glob escaping dependency:** `packages/memory-host-sdk/src/host/backend-config.ts:3`
  - body: `backend-config.ts` now imports `minimatch`, but the root runtime dependencies do not declare it. That makes the published package rely on whatever transitive layout happens to exist, so backend config loading can fail in stricter installs; add a direct runtime dependency or use a local escape helper.
  - confidence: 0.86
- **[P2] Escape braces in file-derived QMD patterns:** `packages/memory-host-sdk/src/host/backend-config.ts:310`
  - body: `escapeGlobPattern(path.basename(resolved))` leaves `{` and `}` unescaped unless `magicalBraces` is enabled, while QMD reindexes masks through fast-glob. A direct file like `notes{a,b}.md` can be treated as brace alternatives instead of one literal filename, so escape braces too and add coverage.
  - confidence: 0.91
