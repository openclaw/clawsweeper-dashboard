## What This Changes

The issue asks OpenClaw to make explicitly authorized read-only operational/SSH diagnostics less likely to be blocked by model safety and to distinguish model refusal, authorization denial, and tool failure.

## Best Possible Solution

Define a maintainer-approved trusted-operator diagnostics contract, then add runtime or observability that separates model refusal, authorization denial, policy denial, and tool/SSH failure while preserving exec approvals and sandbox/tool-policy gates.

## Reproduction Assessment

## Solution Assessment

Unclear. A broad prompt relaxation would be risky; the better path is a maintainer-designed trusted-operator diagnostics contract with explicit observability for model, authorization, policy, and tool-failure outcomes.
