## Summary

Keep open: current main has execution-bias prompting, exec approvals, approval event reporting, and some refusal metadata, but the requested trusted operator diagnostics contract and model-level soft-refusal observability still need maintainer product and security direction.

## What This Changes

The issue asks OpenClaw to distinguish model soft refusals from authorization/tool failures and handle explicitly authorized read-only SSH or infrastructure diagnostics more reliably.

## Best Possible Solution

Define a maintainer-approved operator diagnostics contract that preserves hard authorization in tool policy, exec approvals, and elevated controls, then add model-refusal versus authorization/tool-failure observability plus narrow read-only infra routing or prompt changes.

## Reproduction Assessment

No. The report describes observed soft refusals but lacks the provider/model, transcript, configured tool policy/elevated/approval state, and SSH target needed for a high-confidence current-main reproduction.

## Solution Assessment

Unclear. A broad trusted-admin mode may be useful, but the safer maintainable path is to define the hard-policy boundary first, then add narrow observability/routing changes with live provider proof.
