## Summary

Keep open: current main has exec approvals, elevated mode, and structured approval/tool-failure reporting, but this issue is about model-level soft refusal behavior and a possible trusted-operator diagnostics contract that still needs maintainer product and security direction.

## What This Changes

Request to make OpenClaw prefer explicitly authorized operational execution over model-level soft refusals, especially for read-only SSH and infrastructure diagnostics, while distinguishing model refusal from tool, permission, and connection failures.

## Best Possible Solution

Define a maintainer-approved operator diagnostics contract that preserves hard authorization in tool policy, exec approvals, and elevated mode, then add observability for model refusal versus authorization/tool failure and targeted prompt or skill routing for read-only infrastructure diagnostics.

## Reproduction Assessment

No. The report describes observed soft refusals but lacks the provider/model, transcript, configured tool policy/elevated/approval state, and SSH target needed for a high-confidence current-main reproduction.

## Solution Assessment

Unclear. A broad trusted-admin mode may be useful, but the safer maintainable path is to first define the hard-policy boundary and observability contract, then implement narrow prompt/routing changes with live provider proof.
