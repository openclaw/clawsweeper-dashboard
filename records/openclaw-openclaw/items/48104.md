## Summary

Keep open: current main has execution-bias prompting, structured exec approval/tool-failure events, and limited provider stop-reason metadata, but it still does not define the trusted operator diagnostics contract or reliably separate pre-tool model soft refusals from authorization and SSH/tool failures.

## What This Changes

The issue asks OpenClaw to reduce and expose model-level soft refusals during explicitly authorized SSH and operational diagnostics, especially read-only admin tasks.

## Best Possible Solution

Define a maintainer-approved operator diagnostics contract that preserves hard authorization in tool policy and exec approvals, then add structured observability for model refusal versus authorization and tool failures plus narrow read-only diagnostics routing with live provider proof.

## Reproduction Assessment

No. The issue describes observed soft refusals, but lacks the provider/model, transcript, configured tool policy, approval/elevated state, and SSH target needed for a high-confidence current-main reproduction.

## Solution Assessment

Unclear. A broad trusted-admin mode might help, but the safer maintainable path is to define the hard-policy boundary first and then add narrow observability or routing changes with live provider evidence.
