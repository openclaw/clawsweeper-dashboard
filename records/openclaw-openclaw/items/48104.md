## Summary

Keep open: current main has partial execution follow-through mitigations and hard host-exec approval controls, but it still does not define a trusted operator diagnostics/refusal-observability contract for distinguishing model soft refusal from authorization denial or SSH/tool failure.

## What This Changes

The issue requests safer handling and clearer observability when models avoid explicitly authorized read-only operator diagnostics such as SSH log and service checks.

## Best Possible Solution

Define a maintainer-approved operator diagnostics contract that preserves hard exec authorization, then add narrow observability for model-refusal versus auth denial versus SSH/tool failure with redacted live provider proof.

## Reproduction Assessment

No. Source inspection shows plausible observability gaps, but the issue lacks provider/model, transcript, configured tool policy, approval/elevated state, and SSH target details for a high-confidence current-main reproduction.

## Solution Assessment

Unclear. A broad trusted-admin mode is not the safest fix by itself; the maintainable direction is a narrow diagnostics/refusal classification contract that keeps hard execution controls intact.
