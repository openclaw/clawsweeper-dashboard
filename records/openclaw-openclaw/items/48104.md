## Summary

Keep open: current main has partial prompt, approval, and refusal-observability scaffolding, but it does not fully cover the reported soft-refusal versus tool/permission-failure distinction or define a trusted read-only diagnostics mode.

## What This Changes

The issue asks OpenClaw to distinguish model soft refusals from tool or permission failures and better support explicitly authorized read-only infrastructure diagnostics.

## Best Possible Solution

Define a core prompt and telemetry contract that reports model refusal, authorization denial, and tool execution failure separately while keeping exec approvals and sandbox policy as the hard enforcement layer.

## Reproduction Assessment

No. The issue gives representative SSH/admin prompts, but not a deterministic transcript, model/provider, config, or live target needed to reproduce a current-main soft refusal.

## Solution Assessment

Not yet. Current main partially pushes execution and records some provider refusals, but the requested trusted read-only diagnostics and general soft-refusal visibility need maintainer product/security direction.
