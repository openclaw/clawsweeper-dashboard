Keep open: current main has partial strict-agentic follow-through and hard exec approval controls, but it still lacks a maintainer-approved operator diagnostics/refusal contract for distinguishing model soft refusal from authorization denial or SSH/tool failure.

Request clearer trusted-operator handling for authorized read-only SSH and infrastructure diagnostics so model soft refusals do not look like permission or tool failures.

## Best Possible Solution

Define a maintainer-approved operator diagnostics contract that preserves hard exec authorization, then add structured classification for model refusal versus auth denial versus SSH/tool failure with redacted live provider proof.

## Solution Assessment

Unclear. A broad trusted-admin mode is not the safest fix by itself; the maintainable direction is a narrow diagnostics/refusal classification contract that keeps hard execution controls intact.
