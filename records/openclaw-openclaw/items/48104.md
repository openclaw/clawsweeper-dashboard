## Summary

Keep open: current main has partial plan-only follow-through mitigations and hard exec-approval controls, but it still lacks a maintainer-approved trusted diagnostics/refusal-observability contract for distinguishing model soft refusal from authorization denial and SSH/tool failure.

## What This Changes

The issue asks for trusted operator diagnostics behavior and observability so authorized SSH/read-only infrastructure tasks are not blocked by model soft refusals.

## Best Possible Solution

Define a maintainer-approved operator diagnostics contract that preserves hard exec authorization, then add narrow observability for model-refusal versus auth denial versus SSH/tool failure with live provider proof.

## Reproduction Assessment

No. Source inspection shows plausible observability gaps, but the issue lacks provider/model, transcript, configured tool policy, approval/elevated state, and SSH target data for a high-confidence current-main reproduction.

## Solution Assessment

Unclear. A broad trusted-admin mode is not the safest fix by itself; the maintainable direction is a narrow diagnostics/refusal classification contract that keeps hard execution controls intact.
