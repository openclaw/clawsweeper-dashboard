Request to fire the session-memory/memory-finalization path when sessions reset or are pruned, not only on compaction or explicit session commands.

## Best Possible Solution

Define one maintainer-approved session finalization contract for destructive endings, covering explicit reset, idle/daily rollover, explicit delete, shutdown where feasible, and maintenance prune before deletion with bounded async behavior, privacy rules, dedupe, and regression coverage.
