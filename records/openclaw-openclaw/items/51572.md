## Summary

Keep this open. Current main has useful adjacent work: plugin `session_end` hooks now fire for explicit, idle, daily, and compaction rollovers, and memory-core Dreaming can ingest archived transcripts, but the bundled `session-memory` hook still only listens to command resets and session-store pruning still does not fire that hook before deletion.

## What This Changes

The issue asks OpenClaw to run session-memory capture when sessions end through idle/daily reset or maintenance pruning, not only existing reset/compaction-related paths.

## Best Possible Solution

Define one supported lifecycle memory contract, preferably using nonblocking archived-transcript context through session lifecycle hooks or clearly documenting memory-core Dreaming as the replacement, before adding heavy capture work to session-store maintenance.

## Reproduction Assessment

Yes for the current behavior, by source inspection. The bundled hook is command-only, while idle/daily resets use plugin lifecycle hooks and pruning archives/deletes without firing session-memory.

## Solution Assessment

Unclear. Expanding session-memory directly is plausible, but maintainers should first choose whether the supported path is legacy internal hooks, plugin `session_end`/`before_reset`, or memory-core Dreaming to avoid duplicate lifecycle memory systems.
