## Summary

Keep open. Current main and v2026.5.20 still keep bundled session-memory limited to manual command hooks, while automatic daily/idle rollovers only emit session lifecycle hooks and maintenance pruning archives/deletes stale entries without a pre-prune memory contract.

## What This Changes

Request that session-memory run on automatic daily/idle reset and before maintenance prune so prior session context can be preserved.

## Best Possible Solution

Choose one lifecycle memory contract that captures exactly once for automatic daily/idle rollover and before prune, with bounded runtime behavior, privacy/retention rules, docs, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Configure daily or idle reset with session-memory enabled, let an existing session go stale, then send the next message: current main captures previousSessionEntry but only emits lifecycle hooks, while maintenance pruning archives/deletes stale entries without a memory hook.

## Solution Assessment

Unclear as a direct one-line fix. Reusing session-memory is plausible, but maintainers need to select the reset/prune contract and privacy bounds because the current reset PRs do not cover prune-before-delete.
