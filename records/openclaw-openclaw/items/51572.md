## Summary

Keep open: current main and the latest checked release still do not run bundled session-memory finalization for automatic daily/idle rollover or destructive session-store pruning, but the permanent fix needs a maintainer-chosen lifecycle contract rather than an autonomous narrow patch.

## What This Changes

## Best Possible Solution

Define a maintainer-approved lifecycle contract for memory finalization before automatic daily/idle rollover and destructive maintenance prune, with bounded async behavior, privacy/dedupe rules, docs, and regression coverage.

Yes. Source inspection shows current main can rotate daily/idle sessions and prune stale store entries while bundled session-memory handles only internal command:new and command:reset events; I did not run a live gateway scenario because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the hook/API contract. The requested behavior is valid, but the best permanent fix depends on reset/prune event shape, timeout behavior, plugin ownership, and privacy semantics.
