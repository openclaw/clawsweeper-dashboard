## Summary

Keep open: current main still does not run the bundled session-memory path for idle/daily rollover or maintenance pruning, and the adjacent plugin lifecycle and memory-core dreaming paths do not exactly cover the requested pre-reset/pre-prune flush timing.

## What This Changes

Request that session-memory preservation run before idle/daily session rollover and maintenance pruning, not only for the current manual reset/new path.

## Best Possible Solution

Define a lifecycle-backed memory preservation path for session end/delete that covers idle, daily, and prune without doing unbounded hook/model work inside the store writer, then document and test that contract.

## Reproduction Assessment

Yes by source inspection. The bundled handler only accepts command new/reset events, while current idle/daily rollover emits plugin session_end and maintenance pruning archives removed transcripts without firing session-memory.

## Solution Assessment

Unclear. Reusing the existing internal session-memory hook directly may not be the safest design; a generic session-end/delete memory flush contract or plugin lifecycle bridge needs maintainer judgment first.
