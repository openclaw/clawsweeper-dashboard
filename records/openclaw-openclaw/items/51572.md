## Summary

Keep open: current main still does not run the bundled session-memory path for automatic idle/daily rollover or maintenance pruning, while the open repair PRs only cover the rollover half and prune timing needs maintainer design.

## What This Changes

Request that session-memory preservation run on automatic idle/daily session rollovers and before maintenance pruning, not only explicit reset/new command paths.

## Best Possible Solution

Land one canonical daily/idle rollover fix from the open PRs, then design a bounded lifecycle-backed preservation path for maintenance prune that avoids doing unbounded model or filesystem work inside the session-store writer.

## Reproduction Assessment

Yes, by source inspection. Seed a stale daily or idle session and the current initSessionState path rolls over through plugin lifecycle hooks without the internal command hook required by bundled session-memory; maintenance pruning similarly deletes/archive-tracks entries without a pre-prune hook point.

## Solution Assessment

Unclear. Reusing the existing session-memory command hook is plausible for idle/daily rollover, but pre-prune preservation needs maintainer judgment on lifecycle semantics and async side effects before it is safe to implement.
