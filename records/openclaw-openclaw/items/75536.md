## Summary

Keep open. Current main still forwards out-of-range positive read offsets into the pinned upstream read tool, and this PR is a focused, mergeable fix with regression coverage, changelog credit, green GitHub check runs, and no blocking review findings.

## What This Changes

The PR adds EOF-offset recovery to the OpenClaw read-tool wrapper, adds focused regression tests for explicit-limit and adaptive reads, and records the user-facing fix in the changelog.

## Best Possible Solution

Land a narrow wrapper-level recovery that handles the pinned upstream EOF offset error in both explicit-limit and adaptive read flows, while keeping dependency version, sandbox/path validation, image normalization, and unrelated read failures unchanged.

## Reproduction Assessment

Yes. A nonempty text file read through `createOpenClawReadTool` or `createSandboxedReadTool` on current main with a positive `offset` greater than the file's line count reproduces from source because the wrapper forwards the call to upstream code that throws on `startLine >= allLines.length`.

## Solution Assessment

Yes. The proposed wrapper retry is the narrowest maintainable fix because OpenClaw already owns this read wrapper for truncation, sandbox, and image behavior, and the patch limits recovery to the exact upstream EOF error shape for positive finite offsets.
