## What This Changes

The PR adds EOF-offset recovery to the OpenClaw read-tool wrapper, adds regression coverage for explicit/adaptive/empty/trailing-newline cases, and records the user-facing fix in CHANGELOG.md.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main can be reproduced by reading any nonempty text file through `createOpenClawReadTool` or `createSandboxedReadTool` with a positive `offset` greater than the file line count; the wrapper forwards that to the pinned upstream read implementation that throws the EOF error.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it retries only the exact upstream EOF-offset error for positive finite offsets and keeps unrelated errors, dependency behavior, path validation, sandboxing, and image handling unchanged.
