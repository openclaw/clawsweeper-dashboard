## Summary

Keep open: current main avoids QMD semantic work by default and falls back after QMD search failures, but semantic QMD modes still invoke QMD/node-llama-cpp without an Alder Lake-N/native-Windows guard or specific documentation for the reported 0xC0000005 crash.

## What This Changes

Make QMD semantic search on native Windows avoid node-llama-cpp access-violation crashes on Alder Lake-N CPUs while preserving BM25 fallback/workarounds.

## Best Possible Solution

Add owner-approved native-Windows QMD semantic-mode mitigation: detect or diagnose affected CPU/binary crashes, fall back cleanly without taking down OpenClaw, document WSL2/source-build workarounds, and coordinate any binary fix upstream.
