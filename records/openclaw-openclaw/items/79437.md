## Summary

Keep open: current main and the latest release have BM25-oriented defaults and generic QMD fallback, but semantic QMD modes still route through QMD/node-llama-cpp without an Alder Lake-N/native-Windows access-violation guard or specific documentation. ## What This Changes

The issue reports native-Windows QMD semantic memory crashes when node-llama-cpp prebuilt binaries load GGUF models on Intel Alder Lake-N CPUs.

## Best Possible Solution

Add an owner-approved native-Windows QMD semantic-mode mitigation: clear diagnostics or detection for affected CPUs/binaries, graceful BM25 fallback where possible, updated QMD docs, and upstream coordination for binary compatibility.
