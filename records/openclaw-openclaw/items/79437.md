## Summary

Keep open: current main has safer QMD defaults and generic fallback behavior, but semantic QMD modes still run through QMD/node-llama-cpp without an Alder Lake-N-specific guard, native-Windows diagnostic, or documented limitation, and no canonical PR currently fixes that root cause.

## What This Changes

Report that QMD semantic memory paths crash on Windows Alder Lake-N CPUs when node-llama-cpp loads GGUF models, leaving BM25-only recall usable.

## Best Possible Solution

Keep the issue open and choose an owner-approved mitigation that prevents semantic QMD from hard-failing on affected native-Windows CPUs, with clear diagnostics, BM25 fallback, and docs while coordinating any binary fix upstream.

## Reproduction Assessment

No, not as a live repro in this review: reproducing the crash requires native Windows on Alder Lake-N hardware with QMD semantic mode. Source inspection does confirm that current main still reaches QMD search/embed paths without CPU-specific handling.

## Solution Assessment

Unclear. A docs-only warning, BM25 fallback, CPU detection, automatic source build, or separate binary distribution each has different ownership and packaging tradeoffs, so maintainers should choose the policy before implementation.
