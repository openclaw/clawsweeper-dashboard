## Summary

Keep open: current main still avoids QMD semantic work by default and has builtin fallback after QMD search failures, but semantic QMD modes still spawn QMD/node-llama paths with no Alder Lake-N guard, source-build fallback, E-core-safe binary path, or explicit native-Windows limitation for this crash.

## What This Changes

The issue asks OpenClaw to prevent or document QMD/node-llama-cpp native-Windows crashes on Alder Lake-N CPUs when semantic memory paths load GGUF models.

## Best Possible Solution

Keep QMD semantic modes from hard failing on affected native-Windows CPUs by adding owner-approved detection, diagnostics, fallback behavior, and public documentation while coordinating any binary compatibility fix upstream.

## Reproduction Assessment

No high-confidence live reproduction was established; it requires native Windows on Alder Lake-N hardware with QMD semantic mode. Source inspection does confirm that non-BM25 QMD paths still route into QMD embed/query without CPU-specific handling.

## Solution Assessment

Unclear. The requested remedies span upstream binaries, automatic source builds, runtime fallback, and documentation, so the maintainable fix needs maintainer choice before implementation.
