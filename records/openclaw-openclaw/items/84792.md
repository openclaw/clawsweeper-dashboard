## What This Changes

The PR reorders direct `runReplyAgent` pre-run maintenance so `runMemoryFlushIfNeeded` runs before `runPreflightCompactionIfNeeded`, and adds a regression assertion for that order.

## Best Possible Solution

Land the direct-run flush-before-preflight ordering if maintainers accept the session-state timing, while keeping queued-followup expansion and in-turn catch-up in separate canonical work.

## Reproduction Assessment

## Solution Assessment

Yes for the direct pre-turn path. Moving the existing memory-flush call before the existing preflight-compaction call is the narrow owner-boundary fix, while queued followups and in-turn auto-compaction remain separate scope decisions.
