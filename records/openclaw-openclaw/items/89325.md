## Best Possible Solution

Owner review should first live-reproduce the restart/drain path on 2026.5.28 or newer, then choose a security-preserving refresh/re-registration or typed stale-relay outcome that restores native tools without weakening fail-closed policy.

## Reproduction Assessment

## Solution Assessment

Unclear; falling back to normal policy on relay unavailability is not clearly the best fix because current source intentionally treats unavailable PreToolUse as fail-closed unless no local policy existed. A safer fix likely refreshes or re-registers stale session relay state while preserving explicit approval semantics.
