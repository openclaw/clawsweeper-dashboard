## Summary

Current main and the latest release already recover full Slack DM text from Slack rich-text blocks when Slack sends a shortened `message.text` preview, which matches this report from 2026.4.14.

## What This Changes

The issue asks OpenClaw Slack socket-mode inbound handling to deliver complete long message text to the agent instead of a shortened preview.

## Best Possible Solution

Keep the shipped block-text recovery and close this report as covered by the v2026.5.6 Slack fix; reopen only with a new current-version payload shape that still loses text.

## Reproduction Assessment

Yes for the historical failure: the matching report and fix prove the Slack Socket Mode shape where `message.text` is a shortened preview and top-level rich-text blocks contain the full body. On current main, the regression test covers that path and the failure no longer reproduces.

## Solution Assessment

Yes. Recovering the full body inside the Slack plugin's inbound message-content resolver is the narrow owner-local fix, and it keeps core channel contracts unchanged.
