## Summary

Keep open. Current main still treats SQLite as a retrieval index over Markdown/session memory, not as the requested durable typed memory source of truth with canonical records, revisions, statuses, confidence, entities, and validity windows.

## What This Changes

Define a v1 SQLite schema and canonical record model for local-first durable memory records.

## Best Possible Solution

Keep this open for a maintainer-owned Memory MVP design that introduces a versioned SQLite schema, migrations, exported record/revision/entity/status types, and integration with the write, provenance, review, and forget flows.

## Reproduction Assessment

## Solution Assessment

Unclear as an autonomous fix because the request is foundational and crosses storage, public contracts, write-path policy, provenance, and operator flows. The maintainable path is a coordinated Memory MVP schema/API design rather than an opportunistic schema patch.
