## Summary

Keep open. Current main still implements the default SQLite memory path as a Markdown/session search index, not as the requested durable typed memory source of truth with canonical records, revisions, statuses, confidence, entities, and validity windows.

## What This Changes

Define a v1 SQLite schema and canonical typed record model for durable local-first memory records.

## Best Possible Solution

Keep this open for maintainer-owned Memory MVP design: a versioned SQLite schema and migration plus exported record, revision, entity, status, confidence, provenance, and validity types integrated with the write pipeline and review/forget flows.

## Reproduction Assessment

Not applicable as a runtime reproduction; this is a schema/product request. Source and docs inspection provide a high-confidence verification path, and current main lacks the requested durable typed SQLite record model.

## Solution Assessment

Unclear as an autonomous fix because the direction is foundational and crosses storage, SDK, write-path, provenance, and operator flows. The narrow maintainable path is a coordinated Memory MVP schema/API design rather than an opportunistic table patch.
