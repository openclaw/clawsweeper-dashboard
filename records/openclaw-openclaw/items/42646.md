## What This Changes

Define a v1 SQLite-backed durable memory schema with typed records, canonical current state, revisions, statuses, timestamps, confidence, validity windows, entity normalization, and optional links.

## Best Possible Solution

Settle the durable-memory contract at the memory-core and memory-host SDK boundary, then implement schema, migrations, write/provenance/review semantics, docs, and tests as one coordinated Memory MVP slice.
