## Summary

Keep open: this PR is directionally fixing the Control UI reconnect flash, but it is still merge-blocked by a scope-mismatch reset gap and missing real behavior proof.

## What This Changes

The PR adds a sticky `hasEverConnected` Control UI state, resets it for selected auth/pairing/revocation closes, adds gateway tests, and updates the changelog.

## Best Possible Solution

Keep the sticky prior-hello design, add `AUTH_SCOPE_MISMATCH` to the reset path with regression coverage, rebase onto main, and require redacted real Control UI proof before merge.

## Reproduction Assessment

Yes, at source level. Current main flips `connected=false` into the login-gate render path on reconnect, and PR head still keeps cached UI mounted after a structured `AUTH_SCOPE_MISMATCH` close because that detail is missing from the reset allowlist.

## Solution Assessment

No, not as-is. The sticky prior-hello state is a narrow maintainable shape, but it must match the current gateway auth contract and include real after-fix behavior proof before merge.
