## Summary

Keep open. The PR addresses a real Control UI reconnect bug and the author already narrowed the auth detail-code allowlist after review, but the latest diff still misses code/reason-only auth or device revocation closes, so cached admin UI can stay mounted after credentials are invalidated.

## What This Changes

The branch adds sticky Control UI `hasEverConnected` state across the gateway host, Lit/view state, renderer, tests, and changelog so transient WebSocket reconnects keep chat mounted while structured auth or pairing detail codes restore the credentials gate.

## Best Possible Solution

Keep the sticky prior-session behavior, but clear it for explicit auth/session/device revocation close frames or propagate structured close details from the gateway client, with focused tests for post-hello no-error close events.

## Reproduction Assessment

Yes. The linked bug has concrete tab-refocus reproduction steps on current main, and the PR-specific defect is reproducible after a successful hello by emitting `onClose({ code: 4001, reason: "gateway auth changed", error: undefined })`, which leaves the sticky state true as written.

## Solution Assessment

No, not as written. A sticky prior-hello flag is a maintainable fix for transient reconnect flashes, but the implementation must also treat current server revocation close frames as re-auth failures before it is the narrow safe solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clear sticky state on revocation closes:** `ui/src/ui/app-gateway.ts:405-406`
  - body: After a successful hello, server-initiated auth/device revocation closes reach `onClose` with only code and reason, while this branch clears `hasEverConnected` only when a structured detail code produced `host.lastErrorCode`. Current server paths use `4001` with reasons like `gateway auth changed` and `device removed`, so cached Control UI content can remain mounted after revocation. Please treat those close code/reason pairs as auth failures or propagate structured details, and add a no-error close test.
  - confidence: 0.86
