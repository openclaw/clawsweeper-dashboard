## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid timestamp-only duplicate rejection:** `extensions/feishu/src/dedup.ts:224-226`
  - body: The new create_time filter rejects `createTimeMs <= lastTime`. Feishu timestamps may be second-level after normalization, so two legitimate messages from the same sender in the same second can share a timestamp and the later one is dropped before handling; add a message/event-id tie-breaker or avoid treating equal timestamps as duplicates.
  - confidence: 0.9
- **[P1] Scope contact caches by Feishu account:** `extensions/feishu/src/contacts.ts:22`
  - body: The contacts cache keys rows only by `open_id` while the sync/search APIs accept `accountId` and existing tools route by contextual account. In multi-account setups, one tenant's cached contact can satisfy, prune, or drive send/calendar actions for another tenant; add an account discriminator to schema, reads, writes, and prunes.
  - confidence: 0.88
- **[P1] Scope group caches by Feishu account:** `extensions/feishu/src/groups.ts:54`
  - body: The groups cache uses `chat_id` as a global primary key and list/search return local cache hits before account-specific sync. In multi-account setups, a group from one Feishu account can be returned or pruned while operating under another account; include account scope in schema, reads, writes, and cleanup.
  - confidence: 0.86
- **[P1] Preserve the dedupe claim for direct abort dispatch:** `extensions/feishu/src/monitor.account.ts:369-378`
  - body: `dispatchFeishuMessageDirect` calls `handleFeishuMessage` without `processingClaimHeld: true`, but the receive handler already claimed the message ID before debounce. Normal stop messages can be treated as duplicates by finalization and return before aborting the in-flight run.
  - confidence: 0.84
- **[P2] Gate proactive file sends explicitly:** `extensions/feishu/src/feishu-ops-tools.ts:64-67`
  - body: The new `feishu_send` file action accepts a local `file_path` and is registered with the bundled Feishu plugin. Without an explicit opt-in policy or path allowlist, enabling the plugin adds a host-file upload path to Feishu for agent tool calls.
  - confidence: 0.78
- **[P2] Bound create_time dedupe state:** `extensions/feishu/src/dedup.ts:205`
  - body: `lastProcessedCreateTime` is a process-wide map keyed by account/chat/sender and entries are never evicted. Long-lived gateways with many chats or senders will grow this state monotonically; use the existing bounded/TTL dedupe pattern or another capped cache.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR introduces bundled Feishu ops tools, cross-account persistence risks, and a local-file upload path.

Concerns:

- **[high] Cross-tenant Feishu cache exposure:** `extensions/feishu/src/contacts.ts:22`
  - body: The proposed contacts and groups SQLite caches use global identifiers as primary keys while APIs accept accountId, so data from one configured Feishu account can be returned, pruned, or used for sends/calendar actions under another account.
  - confidence: 0.88
- **[medium] New local file upload path:** `extensions/feishu/src/feishu-ops-tools.ts:64`
  - body: The proposed `feishu_send` file action can read a local path from tool arguments and upload it to Feishu; this should require explicit opt-in, path constraints, or equivalent policy before being bundled by default.
  - confidence: 0.78
