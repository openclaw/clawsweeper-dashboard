- **[P1] Add a tie-breaker to create-time dedupe:** `extensions/feishu/src/dedup.ts:226-227`
  - body: `createTimeMs <= lastTime` rejects legitimate messages that share a second-level timestamp or arrive out of order from the same sender. This can silently drop Feishu messages, so the retry filter needs an id/event tie-breaker or a narrower retry fingerprint instead of a monotonic timestamp watermark.
  - confidence: 0.93
- **[P1] Preserve the direct-dispatch processing claim:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: `im.message.receive_v1` claims the message id before debounce, but `dispatchFeishuMessageDirect` calls `handleFeishuMessage` without `processingClaimHeld: true`. `finalizeFeishuMessageProcessing` can then treat a fast `/stop` as a duplicate and skip the abort path.
  - confidence: 0.91
- **[P1] Filter recalled entries instead of dropping the batch:** `extensions/feishu/src/monitor.account.ts:345-348`
  - body: The queued dispatch returns when any source id in a merged debounce batch was recalled, which also drops newer unrecalled messages in that batch. Recall handling should remove only recalled entries and keep dispatching the remaining user text.
  - confidence: 0.88
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:22-34`
  - body: The contacts table has no account discriminator and local search returns before account-specific sync. In multi-account setups, one account's cached `open_id` can satisfy another account's lookup and drive sends or calendar invites to the wrong tenant.
  - confidence: 0.9
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:55-62`
  - body: The groups table is keyed only by `chat_id`, and search/list reads the global cache before syncing the requested account. A group from one Feishu tenant can be returned or pruned while operating on another account.
  - confidence: 0.89
- **[P1] Route file sends through media access policy:** `extensions/feishu/src/proactive-send.ts:87-93`
  - body: `sendFileMessage` accepts a model-supplied `filePath`, checks existence, and opens it directly. That bypasses the existing outbound media local-root, sandbox, content-type, and size checks, so the tool can upload arbitrary host files.
  - confidence: 0.9
