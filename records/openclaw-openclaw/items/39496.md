## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Route file uploads through authorized media loading:** `extensions/feishu/src/proactive-send.ts:93`
  - body: `filePath` comes from the model-facing `feishu_send` tool, but this opens it directly with `fs.createReadStream` before upload. Use the existing outbound media loader/local-roots authorization path so a tool call cannot upload arbitrary files readable by the gateway process.
  - confidence: 0.97
- **[P1] Do not reject distinct messages by timestamp watermark:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The `createTimeMs <= lastTime` watermark can reject a second distinct message with the same normalized timestamp, or any older message delivered after a newer one. Deduplicate with stable event/message identity or a bounded compound retry key instead of chronological suppression.
  - confidence: 0.98
- **[P1] Preserve unrecalled messages in merged batches:** `extensions/feishu/src/monitor.account.ts:346-348`
  - body: Returning when any source message was recalled drops every other message merged into the queued dispatch. Filter recalled entries before building the merged event instead of cancelling the whole batch.
  - confidence: 0.97
- **[P1] Scope contact cache data by Feishu account:** `extensions/feishu/src/contacts.ts:23`
  - body: The contacts table is keyed only by `open_id`, and local reads/pruning are not account-scoped, so syncing or searching one tenant can return or delete another tenant's contact IDs. Include account identity in keys, queries, and pruning.
  - confidence: 0.98
- **[P1] Scope group cache data by Feishu account:** `extensions/feishu/src/groups.ts:56`
  - body: The groups table is keyed only by `chat_id`, and local reads/pruning are not account-scoped, so one account's sync can erase or return another account's group IDs. Include account identity in keys, queries, and pruning.
  - confidence: 0.98
