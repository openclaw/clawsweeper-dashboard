Full review comments:

- **[P1] Preserve the dedupe claim in direct abort dispatch:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: `im.message.receive_v1` claims the message before debounce, and the queued dispatch preserves that claim with `processingClaimHeld: true`, but this direct fast-abort call does not. `handleFeishuMessage` can then treat the `/stop` message as a duplicate and return before aborting the active run.
  - confidence: 0.93
- **[P1] Do not reject equal create_time messages:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The new stale-message guard rejects `createTimeMs <= lastTime`. Feishu create_time values can be second-level timestamps, so two legitimate messages from the same chat and sender in the same second can be dropped as stale message loss.
  - confidence: 0.91
- **[P1] Scope Feishu ops caches by account:** `extensions/feishu/src/contacts.ts:428-430`
  - body: `searchContactsOrSync` accepts `accountId`, but it returns unscoped local cache hits before syncing the requested account, and the table schema has no account discriminator. In multi-account setups this can surface another tenant's `open_id`; the groups cache has the same pattern for `chat_id`.
  - confidence: 0.9
- **[P1] Gate local file uploads before exposing file_path:** `extensions/feishu/src/feishu-ops-tools.ts:64-65`
  - body: The new model-facing `file_path` parameter is passed to `sendFileMessage`, which reads the local path directly and uploads it to Feishu. Without a sandbox, allowlist, or explicit confirmation policy, a tool call can exfiltrate arbitrary readable host files.
  - confidence: 0.92
- **[P2] Store Feishu caches through plugin state:** `extensions/feishu/src/contacts.ts:14-15`
  - body: The PR adds a `~/.openclaw/data/feishu-contacts.db` sqlite3 CLI sidecar for runtime cache state. Current OpenClaw plugin state is backed by shared `state/openclaw.sqlite`, so this adds an unmanaged state location and a host `sqlite3` dependency that can fail on user installs.
  - confidence: 0.86
