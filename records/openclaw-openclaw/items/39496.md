## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Route file uploads through authorized media loading:** `extensions/feishu/src/proactive-send.ts:87-93`
  - body: `file_path` is model-controlled, but this path opens it directly with `fs.createReadStream`. That permits uploading any file readable by the gateway process, bypassing the existing allowed-root and filesystem-policy checks. Resolve the file through the canonical outbound media loader before opening or uploading it.
  - confidence: 0.97
- **[P1] Do not reject distinct messages with equal or older timestamps:** `extensions/feishu/src/dedup.ts:225-229`
  - body: This watermark rejects `createTimeMs <= lastTime`. Distinct Feishu messages can share a second-level timestamp, and webhook delivery can be reordered, so valid messages are silently lost. Deduplicate with stable event/message identifiers or a bounded compound key rather than chronological rejection.
  - confidence: 0.98
- **[P1] Preserve unrecalled messages in merged debounce batches:** `extensions/feishu/src/monitor.account.ts:341-349`
  - body: Returning when any `sourceMessageIds` entry was recalled drops every other message merged into the queued dispatch. Filter recalled entries before constructing the merged event instead of cancelling the entire batch.
  - confidence: 0.97
- **[P1] Scope cached contacts and groups by Feishu account:** `extensions/feishu/src/contacts.ts:193-197`
  - body: The tables are globally keyed and pruning deletes every ID absent from one account's snapshot. In multi-account installations, syncing one tenant can erase or return another tenant's IDs, causing wrong-recipient sends and failed calendar operations. Include account identity in keys, queries, and pruning.
  - confidence: 0.98
