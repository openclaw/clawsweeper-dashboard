Full review comments:

- **[P1] Gate ops tools behind explicit config:** `extensions/feishu/src/feishu-ops-tools.ts:106-110`
  - body: Existing Feishu tools are gated through the account tools config, but this suite registers send, @all, contacts, groups, and calendar mutation tools whenever any Feishu account exists. Deployments that only configured the channel would suddenly expose sensitive agent-callable operations, so add explicit config/defaults and tests before registering these tools.
  - confidence: 0.9
- **[P1] Constrain file sends to trusted media roots:** `extensions/feishu/src/proactive-send.ts:87-93`
  - body: `feishu_send` accepts a raw `file_path`, checks host existence, and opens it directly with `fs.createReadStream`. Current Feishu media sends require `mediaLocalRoots` for local paths, so this new tool would let an agent read and send arbitrary host files outside the existing media sandbox.
  - confidence: 0.93
- **[P1] Add a tie-breaker to create-time dedupe:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The new create-time dedupe drops any message whose timestamp is `<=` the last processed value for the chat/sender. Feishu timestamps can be second-normalized in this patch, so two legitimate messages in the same second or delivered out of order can be silently discarded; include message/event identity or restrict this to a safer retry fingerprint.
  - confidence: 0.91
- **[P1] Preserve the direct-dispatch processing claim:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: The receive path claims the message before debounce, but the direct abort dispatch calls `handleFeishuMessage` without `processingClaimHeld: true`. `finalizeFeishuMessageProcessing` can then treat the stop message as already in flight and skip it, breaking the fast-abort path this PR adds.
  - confidence: 0.88
- **[P1] Scope ops caches by Feishu account:** `extensions/feishu/src/contacts.ts:427-430`
  - body: Contacts and groups are stored and read from account-global SQLite tables, and the lookup returns local matches before syncing the requested account. In multi-account deployments this can surface another tenant's open_id/chat_id and drive later send or calendar actions through the wrong account.
  - confidence: 0.88
- **[P1] Skip recalled entries without dropping the whole batch:** `extensions/feishu/src/monitor.account.ts:345-348`
  - body: The queued pre-dispatch guard returns when any source message in a merged debounce batch was recalled. If a user recalls only the first message while a later merged message is still valid, the unrecalled message is dropped too; filter the recalled entries or rebuild the dispatch instead of returning for the whole batch.
  - confidence: 0.86
