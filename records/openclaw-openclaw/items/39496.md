## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Route file uploads through authorized media loading:** `extensions/feishu/src/proactive-send.ts:93`
  - body: `filePath` is model-controlled, but this opens it directly with `fs.createReadStream`, bypassing the existing `sendMediaFeishu` path that requires `mediaLocalRoots` for local file reads. Resolve file uploads through the canonical outbound media loader before sending.
  - confidence: 0.97
- **[P1] Do not reject distinct messages by timestamp watermark:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The `createTimeMs <= lastTime` check can reject distinct same-second Feishu messages or older messages delivered out of order. Deduplicate with stable event/message identifiers or a bounded compound retry key instead of chronological rejection.
  - confidence: 0.98
- **[P1] Preserve unrecalled messages in merged batches:** `extensions/feishu/src/monitor.account.ts:346-348`
  - body: Returning when any `sourceMessageIds` entry was recalled drops every other message merged into the queued dispatch. Filter recalled entries before building the merged event instead of cancelling the whole batch.
  - confidence: 0.97
- **[P1] Scope contact cache data by Feishu account:** `extensions/feishu/src/contacts.ts:23`
  - body: The contacts table is keyed only by `open_id`, and local reads/pruning are not account-scoped, so syncing or searching one tenant can return or delete another tenant's contacts. Add account scope to the cache schema and all local queries before enabling the tool.
  - confidence: 0.95
- **[P1] Scope group cache data by Feishu account:** `extensions/feishu/src/groups.ts:56`
  - body: The groups table is keyed only by `chat_id`, and list/search return local rows before account-specific sync, so multi-account installs can surface or prune another tenant's group IDs. Include account scope in the schema, reads, writes, and stale cleanup.
  - confidence: 0.95
- **[P2] Avoid a runtime sqlite3 CLI cache dependency:** `extensions/feishu/src/contacts.ts:44`
  - body: This runtime path shells out to a `sqlite3` binary that is not declared as a Feishu plugin dependency and stores cache data outside the established plugin/shared-state helpers. Use the plugin state/shared SQLite path so fresh installs do not fail based on host CLI availability.
  - confidence: 0.9
- **[P1] Preserve account context in ops tool calls:** `extensions/feishu/src/feishu-ops-tools.ts:128-131`
  - body: The tools register after finding enabled accounts, but execution omits `accountId` for contacts, send, and calendar operations, so calls fall back to the configured default account rather than the account that made the tool available. Pass an explicit account selection or make the tool account-scoped.
  - confidence: 0.92
