  - body: This rejects `createTimeMs <= lastTime` for each account/chat/sender. Different legitimate messages can share the same second-level `create_time`, and reordered deliveries can arrive with an older timestamp, so this can silently drop real Feishu messages instead of only deduping retries.

  - body: `im.message.receive_v1` already claims the message before enqueueing, but the fast-abort direct path calls `handleFeishuMessage` without `processingClaimHeld: true`. `finalizeFeishuMessageProcessing` then treats the same message as a duplicate, so normal stop commands can fail to abort.
  - confidence: 0.9
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:22-34`
  - body: The contacts table is keyed only by `open_id` and local search returns cache hits before syncing the requested account. In multi-account setups this can expose another tenant's contact and drive follow-up send/calendar actions to the wrong account.
  - confidence: 0.9
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:55-62`
  - body: The groups table is keyed only by `chat_id`, and list/search reads the shared cache before account-specific sync. A hit from one account can satisfy another account's lookup or be pruned by another account's sync, misrouting group sends.
  - confidence: 0.9
- **[P1] Enforce a local-file upload boundary:** `extensions/feishu/src/proactive-send.ts:87-93`
  - body: `sendFileMessage` accepts any existing host path and uploads it to Feishu. Existing Feishu document uploads route file reads through `loadWebMedia` with localRoots, so this tool needs the same workspace/local-root policy and explicit product approval before exposing file upload to the model.
  - confidence: 0.88
