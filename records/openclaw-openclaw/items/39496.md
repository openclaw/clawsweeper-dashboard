Full review comments:

- **[P1] Avoid timestamp-only stale-message rejection:** `extensions/feishu/src/dedup.ts:226-227`
  - body: `createTimeMs <= lastTime` treats the Feishu create_time watermark as a duplicate signal. Because callers can normalize second-level timestamps to milliseconds, two distinct messages from the same sender in the same second, or an older message delivered after a newer one, can be skipped before handling.
  - confidence: 0.9
- **[P2] Bound create-time dedupe state:** `extensions/feishu/src/dedup.ts:208`
  - body: `lastProcessedCreateTime` is process-wide state with no TTL or size cap. Long-running gateways that see many chat/sender pairs will grow this map monotonically, unlike the bounded dedupe state already used in this module.
  - confidence: 0.86
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:23-34`
  - body: The contacts table is keyed only by `open_id`, but the tool accepts an `accountId` and can run in multi-account configs. A cache hit or stale prune from one account can return or delete another account's contact data and drive send/calendar actions against the wrong tenant.
  - confidence: 0.88
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:56-62`
  - body: The groups table is global and keyed only by `chat_id`, while lookup/sync paths accept `accountId`. In multi-account deployments, one account's sync can remove or return group ids for another account, which can misroute later `feishu_send` calls.
  - confidence: 0.86
- **[P1] Preserve the held claim for direct abort dispatch:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: `dispatchFeishuMessageDirect` calls `handleFeishuMessage` without `processingClaimHeld: true`, but receive handling has already claimed the same message ID before debounce/flush. Normal stop messages can be treated as duplicates and fail to abort the in-flight run.
  - confidence: 0.84
- **[P1] Drop only recalled messages from queued batches:** `extensions/feishu/src/monitor.account.ts:345-349`
  - body: The queued dispatch guard returns if any source message ID was recalled, which drops the whole merged batch. If a user recalls one message after multiple debounced messages, unrecalled later messages are silently lost instead of being delivered.
  - confidence: 0.84
