## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid timestamp-only stale-message rejection:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The new create_time watermark rejects createTimeMs <= lastTime. Since the caller accepts second-level timestamps, two legitimate messages from the same sender in one second, or a delayed earlier event, can be dropped before handling; add a retry-specific key or tie-breaker instead.
  - confidence: 0.9
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:22-23`
  - body: The contacts table uses open_id as a global primary key and search reads cache rows before any account-specific sync. In multi-account setups, one tenant's contact can satisfy, prune, or drive send/calendar actions for another tenant; include account scope in schema, reads, writes, and prunes.
  - confidence: 0.88
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:55-56`
  - body: The groups table uses chat_id as a global primary key, and list/search can return local cache hits before account-specific sync. In multi-account setups this can return or delete another tenant's group IDs; include account scope in schema, reads, writes, and cleanup.
  - confidence: 0.86
- **[P1] Preserve the held claim for direct abort messages:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: dispatchFeishuMessageDirect calls handleFeishuMessage without processingClaimHeld: true, but the receive handler has already claimed the message id before debounce. Direct stop messages can be treated as duplicates during finalization and return before aborting the in-flight run.
  - confidence: 0.84
- **[P2] Gate proactive file sends explicitly:** `extensions/feishu/src/feishu-ops-tools.ts:64-65`
  - body: The new feishu_send file action accepts a host-local file_path and uploads it to Feishu from an agent tool call. Bundling that surface by default needs an explicit opt-in, path allowlist, or equivalent policy before merge.
  - confidence: 0.78
- **[P2] Route ops tools through an explicit enabled account:** `extensions/feishu/src/feishu-ops-tools.ts:128-132`
  - body: The tools register after finding enabled Feishu accounts, but execute paths call contacts/groups/send/calendar helpers without accountId. A multi-account config can fail or operate on the wrong tenant; resolve the intended enabled account for each tool call.
  - confidence: 0.82
- **[P2] Bound the create-time dedupe map:** `extensions/feishu/src/dedup.ts:208`
  - body: lastProcessedCreateTime is a process-wide map keyed by account/chat/sender and entries are never evicted. Long-lived gateways with many chats or senders will grow this state monotonically; use a TTL/LRU cache like the existing dedupe structures.
  - confidence: 0.82
- **[P2] Filter recalled messages without dropping the whole batch:** `extensions/feishu/src/monitor.account.ts:345-348`
  - body: When a merged debounced batch has any recalled source message, this guard returns before dispatching the entire merged event. Recalling the first of two quick messages can drop the unrecalled later message; filter recalled entries before merging or preserve the remaining batch.
  - confidence: 0.76
