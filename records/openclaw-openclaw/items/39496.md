Full review comments:

- **[P1] Route file uploads through authorized media loading:** `extensions/feishu/src/proactive-send.ts:93`
  - body: `filePath` is model-controlled, but this opens it directly with `fs.createReadStream`, bypassing the existing `sendMediaFeishu` path that enforces `mediaLocalRoots` for local file reads. Resolve file uploads through the canonical outbound media loader before sending.
  - confidence: 0.97
- **[P1] Do not reject distinct messages by timestamp watermark:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The `createTimeMs <= lastTime` check can reject distinct same-second Feishu messages or older messages delivered out of order. Deduplicate with stable event/message identifiers or a bounded compound retry key instead of chronological rejection.
  - confidence: 0.98
- **[P1] Filter recalled entries without dropping the merged batch:** `extensions/feishu/src/monitor.account.ts:346-348`
  - body: Returning when any `sourceMessageIds` entry was recalled drops every other message merged into the queued dispatch. Filter recalled entries before building the merged event instead of cancelling the whole batch.
  - confidence: 0.97
- **[P1] Use account-aware tool context for ops tools:** `extensions/feishu/src/feishu-ops-tools.ts:125-132`
  - body: These new tools execute without a contextual or explicit `accountId`, so multi-account agents fall back to the configured default account. Register them with the account-aware tool pattern used by existing Feishu tools.
  - confidence: 0.93
- **[P1] Scope contact cache data by Feishu account:** `extensions/feishu/src/contacts.ts:22-23`
  - body: The contacts table is keyed only by `open_id`, and search returns local hits before syncing the requested account. In multi-account deployments, one account's cached contact can satisfy another account's lookup and drive sends or calendar invites to the wrong tenant.
  - confidence: 0.95
- **[P1] Scope group cache data by Feishu account:** `extensions/feishu/src/groups.ts:55-56`
  - body: The groups table is keyed only by `chat_id`, and list/search return global local hits before account-specific sync. A sync or prune from one account can erase or return group IDs for another account.
  - confidence: 0.95
