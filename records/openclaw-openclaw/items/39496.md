## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Route file uploads through authorized media loading:** `extensions/feishu/src/proactive-send.ts:93`
  - body: `filePath` is model-controlled, but this opens it directly with `fs.createReadStream`, bypassing the existing `sendMediaFeishu` path that requires `mediaLocalRoots` for local file reads. Resolve file uploads through the canonical outbound media loader before sending.
  - confidence: 0.97
- **[P1] Do not reject distinct messages by timestamp watermark:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The `createTimeMs <= lastTime` check can reject distinct same-second Feishu messages or older messages delivered out of order. Deduplicate with stable event/message identifiers or a bounded compound key instead of chronological rejection.
  - confidence: 0.98
- **[P1] Preserve unrecalled messages in merged batches:** `extensions/feishu/src/monitor.account.ts:346-348`
  - body: Returning when any `sourceMessageIds` entry was recalled drops every other message merged into the queued dispatch. Filter recalled entries before building the merged event instead of cancelling the whole batch.
  - confidence: 0.97
- **[P1] Scope contact cache data by Feishu account:** `extensions/feishu/src/contacts.ts:23`
  - body: The contacts table is keyed only by `open_id`, and local reads/pruning are not account-scoped, so syncing or searching one tenant can return or delete another tenant's contact IDs. Include account identity in keys, queries, and pruning.
  - confidence: 0.98
- **[P1] Scope group cache data by Feishu account:** `extensions/feishu/src/groups.ts:56`
  - body: The groups table is keyed only by `chat_id`, and local reads/pruning are not account-scoped, so one account's sync can erase or return group IDs from another tenant. Include account identity in the schema and every lookup/prune path.
  - confidence: 0.98
- **[P1] Preserve account context in new Feishu ops tools:** `extensions/feishu/src/feishu-ops-tools.ts:128-132`
  - body: The tools register after finding enabled accounts but execute without passing an `accountId`, so they fall back to default-account resolution even when a different account should own the tool call. Route each tool call through a known or explicit enabled account.
  - confidence: 0.94
