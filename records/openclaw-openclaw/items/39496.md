Full review comments:

- **[P1] Avoid timestamp-only stale-message rejection:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The create_time watermark rejects `createTimeMs <= lastTime` for each chat/sender. Two legitimate messages with the same normalized timestamp, or a delayed older webhook, can be dropped before handling; this needs a retry-specific tie-breaker or bounded dedupe design that does not discard real messages.
  - confidence: 0.9
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:22-23`
  - body: The contacts table uses global `open_id` rows and local search returns before account-specific sync. Multi-account installs can return, prune, or use another tenant's contact for send/calendar actions.
  - confidence: 0.88
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:55-56`
  - body: The groups table uses global `chat_id` rows and local list/search read before account-specific sync. One tenant's group can satisfy another tenant's lookup or be deleted by another tenant's sync.
  - confidence: 0.86
- **[P1] Preserve the held claim for direct abort messages:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: `dispatchFeishuMessageDirect` calls `handleFeishuMessage` without `processingClaimHeld: true`, but receive handling has already claimed the same message ID. Normal stop messages can be treated as duplicates and fail to abort the in-flight run.
  - confidence: 0.84
- **[P2] Route ops tool calls to an explicit account:** `extensions/feishu/src/feishu-ops-tools.ts:128-130`
  - body: The tools register after finding enabled accounts, but executions do not pass an account ID into contact/group/send/calendar calls. In multi-account or disabled-default setups, a registered tool can still route through the wrong default account or fail as unconfigured.
  - confidence: 0.8
- **[P2] Gate proactive file sends explicitly:** `extensions/feishu/src/feishu-ops-tools.ts:64-65`
  - body: The new file action accepts a local `file_path` and uploads it from the host during agent tool calls. Bundling this needs explicit opt-in, path constraints, or equivalent policy before merge.
  - confidence: 0.78
