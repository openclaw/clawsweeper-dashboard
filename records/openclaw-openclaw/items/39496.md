Keep open: current main has not absorbed the central Feishu ops-tool work, but this broad PR is not mergeable as-is because the PR head still has account-scoping, dedupe, abort, file-send policy, security, and proof blockers.

## What This Changes

The branch modifies the Feishu plugin for streaming cards, event/create-time dedupe, abort/recall handling, contacts/groups/send/calendar tools and skills, plus daemon working-directory resolution.

## Best Possible Solution

Split this into focused Feishu PRs: streaming/card fixes, bounded dedupe/abort fixes, daemon cwd separately, recall via the narrower open recall PR, and ops tools only after account scoping, explicit file-send policy, tests, and live Feishu proof.

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid timestamp-only stale-message rejection:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The new create_time watermark rejects createTimeMs <= lastTime. With same-timestamp messages or reordered webhooks, legitimate messages from the same sender can be dropped before handling; use a retry-specific tie-breaker or bounded dedupe instead.
  - confidence: 0.9
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:22-23`
  - body: The contacts cache uses open_id as a global primary key and local search returns rows before account-specific sync. In multi-account setups, one tenant's contact can satisfy, prune, or drive send/calendar actions for another tenant.
  - confidence: 0.88
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:55-56`
  - body: The groups cache uses chat_id as a global primary key, and list/search can return local cache hits before account-specific sync. This can return or delete another tenant's group IDs in multi-account setups.
  - confidence: 0.86
- **[P1] Preserve the held claim for direct abort messages:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: dispatchFeishuMessageDirect calls handleFeishuMessage without processingClaimHeld: true, but receive handling has already claimed the same message ID before debounce. Normal stop messages can be treated as duplicates and return before aborting the in-flight run.
  - confidence: 0.84
- **[P2] Route ops tools through an explicit account:** `extensions/feishu/src/feishu-ops-tools.ts:128-132`
  - body: The tools register after finding enabled Feishu accounts, but execution calls contacts/groups/send/calendar helpers without an accountId. A multi-account config with a disabled or different default can fail or operate on the wrong tenant.
  - confidence: 0.82
- **[P2] Gate proactive file sends explicitly:** `extensions/feishu/src/feishu-ops-tools.ts:64-65`
  - body: The new feishu_send file action accepts a local file_path and uploads it from the host. Bundling that tool without explicit opt-in, path constraints, or equivalent policy exposes a broad local-file upload surface to agent tool calls.
  - confidence: 0.78
- **[P2] Skip only recalled entries before queued dispatch:** `extensions/feishu/src/monitor.account.ts:345-348`
  - body: The queued dispatch guard returns if any source message in a merged batch was recalled, which drops unrecalled messages in the same batch. Recall handling should filter recalled entries instead of discarding the entire merged dispatch.
  - confidence: 0.78
