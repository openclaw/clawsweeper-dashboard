## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Route file uploads through authorized media loading:** `extensions/feishu/src/proactive-send.ts:93`
  - body: `filePath` is model-controlled, but this opens it directly with `fs.createReadStream`, bypassing the existing `sendMediaFeishu` path that enforces `mediaLocalRoots` for local file reads. Resolve file uploads through the canonical outbound media loader before sending.
  - confidence: 0.97
- **[P1] Do not reject distinct messages by timestamp watermark:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The `createTimeMs <= lastTime` check can reject distinct same-second Feishu messages or older messages delivered out of order. Deduplicate with stable event/message identifiers or a bounded compound key instead of chronological rejection.
  - confidence: 0.98
- **[P1] Filter recalled entries without dropping the merged batch:** `extensions/feishu/src/monitor.account.ts:346-348`
  - body: Returning when any `sourceMessageIds` entry was recalled drops every other unrecalled message merged into that queued dispatch. Filter recalled source entries before dispatch instead of canceling the whole batch.
  - confidence: 0.96
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:22-23`
  - body: The contacts table is keyed only by `open_id`, and local search returns before any account-specific sync. In multi-account setups this can return a contact from another tenant and drive send/calendar actions to the wrong account.
  - confidence: 0.94
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:55-56`
  - body: The groups table is keyed only by `chat_id`, and list/search returns cached rows before syncing the requested account. A sync or prune from one Feishu account can erase or surface groups for another tenant.
  - confidence: 0.94
- **[P1] Pass an explicit account through every ops tool:** `extensions/feishu/src/feishu-ops-tools.ts:128-132`
  - body: The tool handlers call the contacts, groups, send, and calendar helpers without an `accountId`, so registration may succeed because one account is enabled while execution silently falls back to the default account. Select a configured account explicitly or expose a safe account parameter.
  - confidence: 0.92
