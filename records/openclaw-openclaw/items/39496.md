Overall confidence: 0.9

Full review comments:

- **[P1] Avoid timestamp-only stale-message rejection:** `extensions/feishu/src/dedup.ts:226-227`
  - body: The create_time watermark rejects createTimeMs <= lastTime for each chat/sender. Same-second messages or reordered webhooks can be dropped before handling, so this needs a retry-specific tie-breaker or bounded dedupe design.
  - confidence: 0.9
- **[P1] Scope contact cache rows by Feishu account:** `extensions/feishu/src/contacts.ts:22-23`
  - body: The contacts cache uses open_id as a global primary key and local search returns rows before account-specific sync. Multi-account installs can return, prune, or use another tenant's contact for send/calendar actions.
  - confidence: 0.88
- **[P1] Scope group cache rows by Feishu account:** `extensions/feishu/src/groups.ts:55-56`
  - body: The groups cache uses chat_id as a global primary key and local lookups run before account-specific sync. One tenant's group can satisfy or be deleted by another tenant's sync.
  - confidence: 0.86
- **[P1] Preserve the held claim for direct abort messages:** `extensions/feishu/src/monitor.account.ts:373-381`
  - body: dispatchFeishuMessageDirect calls handleFeishuMessage without processingClaimHeld: true, but receive handling has already claimed the same message ID. Normal stop messages can be treated as duplicates and fail to abort the in-flight run.
  - confidence: 0.84
- **[P2] Route ops tools through an explicit account:** `extensions/feishu/src/feishu-ops-tools.ts:128-132`
  - body: The tools register after finding enabled Feishu accounts, but execution calls contacts/groups/send/calendar helpers without an accountId. Multi-account configs can fail or operate on the wrong tenant.
  - confidence: 0.82
- **[P2] Gate proactive file sends explicitly:** `extensions/feishu/src/feishu-ops-tools.ts:64-65`
  - body: The new file action accepts a local file_path and uploads it from the host during agent tool calls. Bundling this needs explicit opt-in, path constraints, or equivalent policy before merge.
  - confidence: 0.78
- **[P2] Skip only recalled entries before queued dispatch:** `extensions/feishu/src/monitor.account.ts:345-348`
  - body: The queued-dispatch guard returns when any source message in a merged batch was recalled, which drops unrecalled messages in the same batch. It should filter recalled entries instead of discarding the whole dispatch.
  - confidence: 0.78
