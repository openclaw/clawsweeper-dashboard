## Summary

This PR should stay open: it addresses the linked Codex ACP auth regression in the right area, but the latest diff still introduces a secret-backup exposure on the copy fallback and leaves the no-auth test dependent on the caller's default Codex home.

## What This Changes

The PR bridges host Codex `auth.json` into ACPX's isolated Codex home, documents that behavior, updates the changelog, and adds ACPX/Codex regression coverage.

## Best Possible Solution

Land an auth bridge that preserves isolated Codex config while ensuring copied credentials are not archived by default, with hermetic auth-present/auth-missing and backup-exclusion coverage.

## Reproduction Assessment

Yes. Source inspection shows current main forces Codex ACP into an isolated `CODEX_HOME` with only generated config, and the linked reports show direct ACPX works once `auth.json` is bridged; I did not run tests because this review was read-only.

## Solution Assessment

No. Bridging only `auth.json` while keeping Codex config hooks isolated is the narrow direction, but the copy fallback must be made backup-safe and the no-auth test must isolate the default home before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep copied Codex auth out of backups:** `extensions/acpx/src/codex-auth-bridge.ts:281`
  - body: When symlink creation fails, this fallback writes the host Codex `auth.json` under `stateDir/acpx/codex-home`. `openclaw backup create` archives the state directory and the current filter only skips extension `node_modules`, so routine backups can include Codex tokens.
  - confidence: 0.93
- **[P2] Isolate the no-auth test from the default Codex home:** `extensions/acpx/src/codex-auth-bridge.test.ts:95`
  - body: Deleting `CODEX_HOME` still lets the bridge fall back to `os.homedir()/.codex/auth.json`. On an authenticated developer or CI home, `generated.authPath` will exist and this ENOENT assertion can fail; point the test at an auth-free temp home instead.
  - confidence: 0.86
