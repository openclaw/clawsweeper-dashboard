## Summary

Keep this PR open. Current main still lacks the Codex ACP auth bridge, but this branch is not ready to merge because its copy fallback can persist Codex auth inside the OpenClaw state tree that default backups archive recursively; the branch is also currently dirty against main.

## What This Changes

The PR bridges host Codex `auth.json` into the ACPX-managed isolated Codex home, updates ACP docs and the changelog, adds ACPX regression tests, and adjusts one Codex plugin test type.

## Best Possible Solution

Keep the host-auth reuse direction, but make the credential bridge backup-safe before merge: prefer symlink-only behavior, move copied credentials to a credential location that backups explicitly handle, or add a precise backup exclusion with regression coverage proving copied Codex auth cannot enter default archives.

## Reproduction Assessment

Yes. Current main can be verified by inspection: the Codex ACP wrapper forces an isolated `CODEX_HOME`, `prepareIsolatedCodexHome` writes only generated config, and the existing test still asserts source Codex auth is not copied; the PR adds focused synthetic `CODEX_HOME/auth.json` tests for the intended bridge.

## Solution Assessment

No. Bridging only `auth.json` while keeping Codex config hooks isolated is the narrow right direction, but the copy fallback into `<stateDir>/acpx/codex-home/auth.json` is not the safest implementation because default backups include that state tree.
