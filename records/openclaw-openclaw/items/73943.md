## Summary

Keep open for maintainer/security review: current main still has the reported isolated-auth problem, but this PR's copy fallback can persist Codex credentials under the OpenClaw state tree that default backups archive.

## What This Changes

The PR bridges host Codex `auth.json` into ACPX's isolated Codex home, updates ACP docs/changelog, adds ACPX regression tests, and adjusts one Codex plugin test type.

## Best Possible Solution

Keep the host-auth reuse direction, but make the bridge backup-safe before merge: prefer symlink-only behavior, place copied credentials in a credential-managed location, or add a precise backup exclusion with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the PR's symlink-failure branch writes `auth.json` under `stateDir/acpx/codex-home`, while backup planning archives the state directory and only filters plugin `node_modules`.

## Solution Assessment

No. Bridging only `auth.json` while keeping Codex config hooks isolated is the narrow direction, but copying credentials into the default state backup scope is not the safest implementation.
