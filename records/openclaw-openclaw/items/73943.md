## Summary

Keep this PR open for author/maintainer follow-up: the auth bridge addresses a real current-main ACPX/Codex gap, but the latest patch still has a merge-blocking secret-handling issue because the copy fallback can put Codex credentials inside default OpenClaw backup archives.

## What This Changes

This PR bridges an existing Codex `auth.json` into ACPX's isolated Codex home, updates ACP docs/changelog, adds ACPX regression coverage, and adjusts one Codex plugin test type.

## Best Possible Solution

Keep the host-auth reuse direction, but avoid materializing copied Codex credentials in default backup scope or add a precise backup exclusion with regression coverage.

## Reproduction Assessment

Yes. The high-confidence reproduction path is to force symlink creation to fail so the PR writes `stateDir/acpx/codex-home/auth.json`, then inspect `openclaw backup create` behavior against the current state-dir backup plan and filter.

## Solution Assessment

No. Bridging only `auth.json` while keeping Codex config hooks isolated is the right narrow direction, but the copied fallback is not the safest implementation while backups include the state tree.
