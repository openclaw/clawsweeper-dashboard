## Summary

Keep this PR open: the auth-bridge direction addresses the linked ACPX/Codex bug, but the final head still leaves copied Codex credentials inside the default backup scope and one regression test remains host-home dependent.

## What This Changes

The PR bridges host Codex `auth.json` into ACPX's isolated Codex home, documents the behavior, updates the changelog, and adds ACPX/Codex regression coverage.

## Best Possible Solution

Keep the narrow `auth.json` bridge, but make copied credentials backup-safe and make the regression tests independent of host Codex auth.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR copy fallback writes `auth.json` under `stateDir/acpx/codex-home`, while backup creation archives the state directory with only plugin `node_modules` filtered; the test issue reproduces when the default home has `.codex/auth.json`.

## Solution Assessment

No. Bridging only `auth.json` while keeping Codex config hooks isolated is the narrow direction, but the current fallback should not place token material in default backups and the no-auth test needs a fully isolated source home.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep copied Codex auth out of backups:** `extensions/acpx/src/codex-auth-bridge.ts:281`
  - body: When symlink creation fails, this copies the host Codex `auth.json` into `stateDir/acpx/codex-home`. `openclaw backup create` includes the state directory and the current filter only skips plugin `node_modules`, so a routine backup can contain Codex credentials.
  - confidence: 0.93
- **[P2] Isolate the no-auth test from the default Codex home:** `extensions/acpx/src/codex-auth-bridge.test.ts:95`
  - body: Deleting `CODEX_HOME` still lets `sourceCodexAuthPath()` fall back to `os.homedir()/.codex/auth.json`. On authenticated developer or CI homes this creates `generated.authPath` and breaks the new absence assertion; point the source home lookup at an auth-free temp home instead.
  - confidence: 0.86
