## Summary

Keep open. The underlying Discord button-context bug is still present on current main, but this PR should not merge as-is because it patches the pre-migration Discord path and puts Discord-specific prompt wording in core.

## What This Changes

The PR adds stored card text to Discord component entries, appends it to synthesized button/select interaction text, and hard-codes Discord Components v2 send guidance in core prompt assembly.

## Best Possible Solution

Capture visible component text in the active Discord plugin entries, append it only to synthesized fallback interaction text, and keep Discord-specific prompt guidance in the plugin `messageToolHints` surface.

## Reproduction Assessment

Yes. Build or register a Discord component message with visible text and a button without explicit `callbackData`, click it, and current source/tests show the fallback agent body is synthesized as `Clicked "Approve".` without the card text.

## Solution Assessment

No. The behavior is directionally correct, but the branch patches stale `src/discord` files and crosses the core/plugin boundary; a plugin-local port is the narrow maintainable fix.
