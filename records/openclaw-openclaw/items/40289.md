## What This Changes

The branch adds a `cardText` field to old Discord component entries, appends it to synthesized interaction text, and inserts Discord Components v2 prompt guidance in core system prompt assembly.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible on current main. The active entry type and builder do not persist visible card text, and the fallback handler formats click/select text from only the label and selected values.
