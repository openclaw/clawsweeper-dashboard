## Summary

The missing-context bug appears valid, but this PR is stale against current main: it changes legacy Discord paths and core prompt text instead of the active Discord plugin implementation.

## What This Changes

The PR stores card text on Discord component entries, appends it to synthesized button/select fallback text, and adds Discord-specific Components v2 guidance to prompt assembly.

## Best Possible Solution

Port the useful behavior into the active Discord plugin: capture visible component text when building entries, persist it on button/select entries, append it only to synthesized fallback event text, and keep Discord prompt wording in plugin messageToolHints.

Yes. Source inspection and current tests give a high-confidence path: build/register a Discord component message with visible text and a button/select that falls through to built-in routing, and current main dispatches only the synthesized click/select text.

## Solution Assessment

No. The proposed behavior is directionally right, but the patch changes stale src/discord files and hard-codes Discord prompt guidance in core instead of updating the active plugin-owned Discord implementation.
