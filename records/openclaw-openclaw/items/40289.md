## Summary

Keep open. The requested Discord context behavior still appears missing on current main, but this PR is not correct as-is because it modifies removed `src/discord` files and core prompt text instead of the active `extensions/discord` plugin, and it lacks required after-fix behavior proof.

## What This Changes

The PR stores collected Discord card text on component entries, appends it to synthesized button/select event text, and adds a Discord Components v2 hint to the message-tool prompt.

## Best Possible Solution

Port the behavior into the active Discord plugin by carrying visible Components v2 card text on component entries, appending it to synthesized fallback button/select event text, and keeping Discord prompt wording in plugin `messageToolHints`.

## Reproduction Assessment

Yes, source-reproducible. Current main's active Discord entry type lacks a card-text field, the builder does not persist visible text to entries, the handler passes only kind/label/values to fallback text, and tests assert context-free click/select bodies.

## Solution Assessment

No. The behavior is useful, but this branch is not the best solution because it updates removed `src/discord` files and core prompt assembly instead of the active plugin-owned implementation.
