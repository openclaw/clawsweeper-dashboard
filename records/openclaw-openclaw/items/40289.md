## Summary

Keep open: the reported Discord behavior is still source-reproducible on current main, but this PR is not mergeable as the fix because it edits the removed `src/discord` implementation and adds Discord-specific prompt guidance in core instead of the active Discord plugin.

## What This Changes

The branch adds Discord card text to component entries and synthesized click/select fallback text under the old `src/discord` implementation, and adds a Discord Components v2 hint in `src/agents/system-prompt.ts`.

## Best Possible Solution

Port the behavior into the active Discord plugin: capture visible Components v2 text on component entries, append it to synthesized fallback button/select event text, and keep Discord prompt wording in plugin `messageToolHints`.

## Reproduction Assessment

Yes, source-reproducible. Current main's active entry type lacks a card-text field, its builder does not persist visible text to entries, and tests still assert context-free fallback click/select bodies.

## Solution Assessment

No. The behavior is useful, but this branch is not the best fix because it updates removed `src/discord` files and core prompt assembly instead of the active plugin-owned Discord implementation.
