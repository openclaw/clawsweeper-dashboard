## Summary

Keep open. The bug is still valid on current main, but this PR is not merge-ready because it modifies the old `src/discord` path while the active Discord component implementation now lives under `extensions/discord`, and it adds Discord-specific prompt wording in core instead of using the existing plugin hint seam.

## What This Changes

The PR stores Discord card text on component entries, appends that text to synthesized button/select interaction text, and adds Discord Components v2 guidance to system prompt assembly.

## Best Possible Solution

Port the useful behavior into the active Discord plugin: capture visible component text when entries are built, persist it on button/select entries, append it only to synthesized fallback event text, preserve explicit `callbackData` and handled plugin dispatch behavior, and keep Discord prompt wording in plugin `messageToolHints`.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence path: build or register a Discord component message with visible text and a button without explicit `callbackData`, click the button, and the fallback agent body is synthesized as `Clicked "Approve".` without the card text.

## Solution Assessment

No. The proposed behavior is directionally right, but this PR applies it to stale files and crosses the core/plugin boundary; a plugin-local port is the narrow maintainable fix.
