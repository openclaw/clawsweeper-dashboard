## Summary

Keep this PR open, but do not merge it as written: the reported missing-context bug is valid, yet the patch edits the stale pre-migration Discord path and puts Discord-specific prompt guidance in core instead of the active plugin-owned surface.

## What This Changes

The PR stores Discord component card text on component entries, appends it to synthesized button/select fallback text, and adds Discord-specific Components v2 prompt guidance in core prompt assembly.

## Best Possible Solution

Port the useful behavior into the active Discord plugin: capture visible component text when entries are built, persist it on button/select entries, append it only to synthesized fallback event text, and keep Discord prompt wording in plugin `messageToolHints`.

Yes. ## Solution Assessment

No. The proposed behavior is directionally right, but the patch is not the best solution because it changes stale `src/discord` files and core prompt assembly instead of the active plugin-owned Discord implementation.
