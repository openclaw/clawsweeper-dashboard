## Summary

Keep open: the reported Discord component context gap is still present on current main, but this branch edits stale `src/discord` paths and core prompt code instead of the active plugin-owned Discord implementation.

## What This Changes

The PR stores Discord card text on component entries, appends it to synthesized button/select event text, and adds Discord-specific Components v2 prompt guidance in core.

## Best Possible Solution

Port the behavior into the active Discord plugin: capture visible component text when entries are built, persist it on button/select entries, append it only to synthesized fallback event text, and keep Discord prompt wording in plugin `messageToolHints`.

## Reproduction Assessment

Yes. Source inspection on current main shows component entries do not retain card text, and existing tests assert context-free fallback bodies such as `Clicked "Approve".` and `Selected Alpha from "Pick".`

## Solution Assessment

No. The behavior is useful, but this patch is not the best solution because it changes removed `src/discord` files and core prompt assembly instead of the active plugin-owned Discord code.
