## Summary

Keep open and do not merge as-is. The missing Discord component context bug is valid, but this PR targets removed `src/discord` files and adds Discord-specific prompt guidance in core while current main uses the active `extensions/discord` plugin path.

## What This Changes

The PR stores Discord card text on component entries, appends it to synthesized button/select fallback interaction text, and adds Discord Components v2 guidance to system prompt assembly.

## Best Possible Solution

Port the useful behavior into the active Discord plugin: capture visible component text when entries are built, persist it on button/select entries, append it only to synthesized fallback event text, and keep Discord prompt wording in plugin `messageToolHints`.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence path: build or register a Discord component message with visible text and a button without explicit `callbackData`, click it, and the fallback agent body is synthesized as `Clicked "Approve".` without the card text.

## Solution Assessment

No. The proposed behavior is directionally right, but this diff patches stale files and crosses the core/plugin boundary; a plugin-local port is the narrow maintainable fix.
