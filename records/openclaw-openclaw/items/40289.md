## Summary

Keep open: the Discord context gap still appears valid on current main, but this branch targets removed `src/discord` code and puts Discord prompt guidance in core instead of the active Discord plugin.

## What This Changes

The PR stores card text on Discord component entries, appends it to synthesized button/select fallback text, and adds a Discord Components v2 hint to core prompt assembly.

## Best Possible Solution

Port the behavior into the active Discord plugin by carrying visible Components v2 text on component entries, including it in fallback click/select event text, and keeping Discord prompt wording in plugin `messageToolHints`.

## Reproduction Assessment

Yes, source-reproducible. Current main's active Discord entry type has no card-text field, the builder only stores routing metadata, and tests still assert context-free click/select fallback text.

## Solution Assessment

No. The useful behavior should be implemented in `extensions/discord/src` and the Discord plugin hints, not in removed `src/discord` files or core prompt assembly.
