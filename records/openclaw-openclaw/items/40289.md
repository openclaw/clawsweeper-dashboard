## Summary

Keep open. The user problem is still real on current main, but this PR branch is not merge-ready because it patches the removed `src/discord` implementation and adds Discord-specific prompt wording in core instead of the active Discord plugin.

## What This Changes

The PR stores Discord card text on component entries, appends it to synthesized button/select interaction text, and adds Discord Components v2 guidance to system prompt assembly.

## Best Possible Solution

Port the useful behavior into `extensions/discord`: collect visible component text while building entries, store it on button/select entries, append it only to synthesized fallback event text, preserve explicit `callbackData` and handled plugin dispatch semantics, keep Discord prompt wording in `messageToolHints`, and add focused regression coverage plus changelog credit.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: create a Discord component message with visible text and a button without explicit `callbackData`, click the button, and the fallback agent body is synthesized as `Clicked "Approve".` without the card text.

## Solution Assessment

No. The intended fix is directionally right, but this branch edits stale files and crosses the core/plugin boundary; a plugin-local port is the narrow maintainable solution.
