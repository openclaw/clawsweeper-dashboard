Full review comments:

- **[P1] Use current fs-safe exports:** `src/gateway/server-methods/agents-workspace.ts:5-8`
  - body: `agents-workspace.ts` imports `mkdirPathWithinRoot` and `removePathWithinRoot`, but current `src/infra/fs-safe.ts` does not export either helper. Rebase to the current `root(...)` API or add an approved wrapper before this file can compile on main.
  - confidence: 0.94
- **[P1] Pin recursive deletes to the workspace root:** `src/gateway/server-methods/agents-workspace.ts:234`
  - body: `workspaceDelete` verifies `realPath`, then calls `fs.rm(fullPath, { recursive: true })` on the original path string. A workspace-writable process can swap a checked ancestor before `fs.rm` resolves it again, so this admin RPC can delete outside the workspace; use a root-pinned recursive helper or narrow the delete contract.
  - confidence: 0.84
- **[P1] Pin moves instead of renaming checked path strings:** `src/gateway/server-methods/agents-workspace.ts:343`
  - body: `workspaceMove` validates source and destination ancestors, but then performs `fs.rename(fromFullPath, toFullPath)` on mutable path strings. A symlink-ancestor swap between validation and rename can redirect the operation outside the workspace root, so move needs a root-pinned implementation or a narrower contract.
  - confidence: 0.86
- **[P2] Keep set defaults aligned with the API contract:** `src/gateway/server-methods/agents-workspace.ts:527`
  - body: Omitted `createDirs` is converted to `false`, while the linked workspace API contract says `agents.workspace.set` creates parent directories by default. Clients sending a nested new path without the optional flag will fail despite that contract; use `params.createDirs !== false` or update the contract and tests together.
  - confidence: 0.88
- **[P2] Keep mkdir defaults aligned with the API contract:** `src/gateway/server-methods/agents-workspace.ts:617`
  - body: Omitted `parents` is converted to `false`, while the linked workspace API contract describes `agents.workspace.mkdir` as creating parents by default. Use `params.parents !== false`, or update the public contract, UI assumptions, and tests to make non-recursive the documented default.
  - confidence: 0.88
- **[P3] Document the workspace RPC contract:** `src/gateway/protocol/schema/agents-workspace.ts:24`
  - body: This adds public `agents.workspace.*` protocol methods, but the user-facing protocol docs still only describe `agents.files.*`. Add the method summary, scopes, encoding, path, size-limit, and symlink behavior so clients have a stable contract to follow.
  - confidence: 0.84
