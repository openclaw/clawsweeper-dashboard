The PR adds `agents.workspace.*` gateway RPCs, protocol/schema/client model updates, scope registrations, server handlers/tests, Swift model generation, and a Control UI Workspace tab for browsing, editing, uploading, downloading, deleting, creating, and moving agent workspace files.

## Best Possible Solution

Keep the PR open and repair the narrow blockers: make `workspace` part of the shared `AgentsPanel` type instead of declaring a competing type in `agents.ts`, add the required changelog attribution, then run targeted UI/type validation and changed-lane checks before maintainer review of the new workspace RPC surface.

## Security Review

Status: needs_attention

Summary: The diff adds a write-capable workspace RPC surface, but this pass did not find a remaining concrete security or supply-chain regression in the latest head.

Concerns:

- **[high] Move destination can escape workspace root:** `src/gateway/server-methods/agents-workspace.ts:307`
  - body: `workspaceMove` detects an outside-root resolved ancestor, but throws inside a broad `try` whose `catch` treats every error as a missing ancestor. A symlinked destination ancestor can therefore be ignored and the subsequent raw `fs.mkdir`/`fs.rename` can write outside the agent workspace.
