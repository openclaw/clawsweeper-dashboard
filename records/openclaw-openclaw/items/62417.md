  - body: `onWorkspaceNavigate` only compares the requested path before applying a list response.   - confidence: 0.91

  - body: The broad `startsWith("..")` check rejects valid relative filenames such as `...` and `..notes`, even though only an exact `..` path segment is unsafe. The PR's own `...` test expects this path to succeed, so rely on absolute-path and segment validation instead.
  - confidence: 0.95
- **[P2] Default workspace writes to create parents:** `src/gateway/server-methods/agents-workspace.ts:527`
  - body: Omitted `createDirs` is converted to `false`, but the PR's API description says `agents.workspace.set` creates parent directories by default. API clients sending a nested new path without the optional flag will fail despite the advertised contract; use `params.createDirs !== false` or update the contract and tests together.
  - confidence: 0.88
- **[P2] Keep mkdir recursive by default:** `src/gateway/server-methods/agents-workspace.ts:617`
  - body: Omitted `parents` is converted to `false`, but the feature request/API text says mkdir parents default to true and the omitted-parents test stubs the recursive helper path. Use `params.parents !== false`, or change the contract and tests together.
  - confidence: 0.94
- **[P2] Mock recursive delete through fs.rm:** `src/gateway/server-methods/agents-workspace.test.ts:608`
  - body: The recursive delete implementation calls `fs.rm`, but this test only stubs `removePathWithinRoot` while the `node:fs/promises` mock does not replace `rm`. In the mocked suite, the test can hit the real filesystem instead of exercising the recursive branch; mock `fs.rm` and assert that path.
  - confidence: 0.9

  - confidence: 0.94

  - confidence: 0.82
