  - body: This file imports/re-exports `AgentsPanel` from `agents.types.ts` and then declares another exported `AgentsPanel` union locally.   - body: The tab array now contains the existing localized tabs and a second hard-coded copy with the new Workspace tab. This will render duplicate Overview/Files/Tools/Skills/Channels/Cron tabs and duplicate active states; keep one localized tab list and insert only the workspace entry.
  - confidence: 0.95
- **[P2] Allow valid dot-prefixed workspace filenames:** `src/gateway/server-methods/agents-workspace.ts:53`
  - body: The `startsWith("..")` check rejects valid filenames like `...` or `..notes`, even though only an exact `..` path segment is unsafe and the PR's own test expects `...` to work. Drop the broad prefix check and rely on absolute-path and segment validation.
  - confidence: 0.93
- **[P2] Keep mkdir's default parent behavior consistent:** `src/gateway/server-methods/agents-workspace.ts:618`
  - body: `params.parents || false` makes omitted `parents` non-recursive, but the linked API design and new tests treat the default as parent-creating. Use an explicit default such as `params.parents !== false`, or update the contract and tests together; as written, omitted-parent mkdir behavior is inconsistent.
  - confidence: 0.9
- **[P2] Mock the recursive delete path:** `src/gateway/server-methods/agents-workspace.test.ts:607`
  - body: The implementation now uses `fs.rm` for recursive directory deletes, but this test still stubs `removePathWithinRoot` and the `fs/promises` mock does not replace `rm`. It will hit the real filesystem for the mocked workspace path instead of exercising the recursive branch.
  - confidence: 0.91
