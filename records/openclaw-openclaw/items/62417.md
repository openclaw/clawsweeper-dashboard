## Summary

Keep open. Current main still does not implement full workspace file management, and this PR is the active implementation path for the linked #61368 request, but the current head is not merge-ready because new workspace filesystem RPCs still use path-string delete/move operations after boundary prechecks and the branch has API/test/docs/changelog mismatches.

## What This Changes

The PR adds `agents.workspace.*` gateway protocol schemas, scope mappings, server handlers, generated Swift models, server tests, a Control UI Workspace tab for browsing/editing/uploading/downloading files, and a changelog entry.

## Best Possible Solution

Keep the feature PR open, repair the filesystem boundary model with root-pinned operations or a narrower supported contract, then align path validation, default semantics, docs, changelog credit, generated clients, and focused tests before running the changed gate.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproductions for the review findings: `fs.rm`/`fs.rename` run after prechecks on mutable path strings, and the current PR test expecting `...` to be accepted conflicts with the validator.

## Solution Assessment

No. The feature direction fits #61368, but the current implementation is not the best mergeable solution until destructive workspace operations are root-pinned and the public API contract is made consistent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pin recursive deletes to the workspace root:** `src/gateway/server-methods/agents-workspace.ts:234`
  - body: The handler verifies `realPath`, then calls `fs.rm(fullPath, { recursive: true })` on the original path string. A workspace-writable process can replace an already checked ancestor with a symlink before `fs.rm` resolves it again, letting this admin RPC delete outside the workspace; use a pinned/root-scoped delete helper or defer recursive delete until one exists.
  - confidence: 0.84
- **[P1] Pin moves instead of renaming checked paths:** `src/gateway/server-methods/agents-workspace.ts:343`
  - body: `workspaceMove` validates the current source and destination ancestors, but then performs `fs.rename(fromFullPath, toFullPath)` on mutable path strings. A symlink-ancestor swap between validation and rename can redirect the destination outside the workspace, so this needs a root-pinned move implementation or a narrower supported move contract.
  - confidence: 0.86
- **[P2] Allow non-traversal double-dot names:** `src/gateway/server-methods/agents-workspace.ts:52`
  - body: The broad `startsWith("..")` check rejects valid relative names such as `...` and `..notes`, and the PR's own `...` test expects that path to succeed. Reject exact `..` path segments and absolute paths instead of every leading double-dot prefix.
  - confidence: 0.96
- **[P2] Default workspace writes to create parents:** `src/gateway/server-methods/agents-workspace.ts:527`
  - body: Omitted `createDirs` is converted to `false`, but the linked workspace API contract says `agents.workspace.set` creates parent directories by default. API clients sending a nested new path without the optional flag will fail despite that contract; use `params.createDirs !== false` or change the contract and tests together.
  - confidence: 0.88
- **[P2] Keep mkdir recursive by default:** `src/gateway/server-methods/agents-workspace.ts:617`
  - body: Omitted `parents` is converted to `false`, while the linked workspace API contract describes `agents.workspace.mkdir` as creating parents by default. Use `params.parents !== false`, or update the public contract, docs, UI assumptions, and tests to make non-recursive the default.
  - confidence: 0.88
- **[P3] Credit the workspace changelog entry:** `CHANGELOG.md:2955`
  - body: This is a user-facing feature entry from a non-bot contributor, and OpenClaw's changelog policy requires contributor attribution. Add the appropriate credit, for example `Thanks @WilShi.`.
  - confidence: 0.94
