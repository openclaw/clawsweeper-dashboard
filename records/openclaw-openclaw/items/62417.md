## Summary

Keep open: current main still lacks the workspace-management API/UI, and this PR remains useful but has blocking filesystem-boundary, contract, docs, and proof issues before merge.

## What This Changes

The PR adds `agents.workspace.*` gateway schemas, scopes, handlers, tests, generated Swift models, and a Control UI Workspace tab for browsing, editing, uploading, downloading, deleting, creating, moving, and stat-ing agent workspace files.

## Best Possible Solution

Keep the feature direction, but land it only after destructive operations use root-pinned semantics or a narrower contract, defaults/docs/generated clients/tests agree, and the contributor adds real behavior proof.

## Reproduction Assessment

Yes for the blocking PR findings at source level: latest head calls `fs.rm` and `fs.rename` on mutable path strings after boundary prechecks and coerces omitted `createDirs`/`parents` to `false`. I did not run live tests in this read-only review.

## Solution Assessment

No. The feature matches the linked workspace-management request, but the current implementation is not the narrowest maintainable solution until root-boundary semantics and the public protocol contract are repaired.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pin recursive deletes to the workspace root:** `src/gateway/server-methods/agents-workspace.ts:234`
  - body: `workspaceDelete` verifies the resolved path, then calls `fs.rm(fullPath, { recursive: true })` on the original path string. A workspace-writable process can swap a checked ancestor before `fs.rm` resolves it again, so this admin RPC can delete outside the workspace; use a root-pinned recursive helper or narrow the contract.
  - confidence: 0.84
- **[P1] Pin moves instead of renaming checked path strings:** `src/gateway/server-methods/agents-workspace.ts:343`
  - body: `workspaceMove` validates source and destination ancestors, but then performs `fs.rename(fromFullPath, toFullPath)` on mutable path strings. A symlink-ancestor swap between validation and rename can redirect the operation outside the workspace root, so move needs a root-pinned implementation or a narrower contract.
  - confidence: 0.86
- **[P2] Keep set defaults aligned with the public contract:** `src/gateway/server-methods/agents-workspace.ts:527`
  - body: Omitted `createDirs` is converted to `false`, while the linked workspace API contract says `agents.workspace.set` creates parent directories by default. Clients sending a nested new path without the optional flag will fail despite that contract; use `params.createDirs !== false` or update the contract and tests together.
  - confidence: 0.88
- **[P2] Keep mkdir defaults aligned with the public contract:** `src/gateway/server-methods/agents-workspace.ts:617`
  - body: Omitted `parents` is converted to `false`, while the linked workspace API contract describes `agents.workspace.mkdir` as creating parents by default. Use `params.parents !== false`, or update the public contract, UI assumptions, and tests to make non-recursive the documented default.
  - confidence: 0.88
- **[P3] Document the workspace RPC contract:** `src/gateway/protocol/schema/agents-workspace.ts:24`
  - body: This adds public `agents.workspace.*` protocol methods, but the user-facing protocol docs still only describe existing agent helpers and `agents.files.*`. Add the method summary, scopes, encoding, path, and size-limit behavior so clients have a stable contract to follow.
  - confidence: 0.86
