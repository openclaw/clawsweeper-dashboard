Full review comments:

- **[P1] Move the workspace panel variant into agents.types.ts:** `ui/src/ui/views/agents.ts:29-36`
  - body: This file imports and re-exports `AgentsPanel` from `agents.types.ts`, then declares another exported `AgentsPanel` union locally. That leaves the shared type without the `workspace` variant and creates a duplicate export in this module. Update `agents.types.ts` and remove the local redeclaration.
  - confidence: 0.96
- **[P2] Guard workspace navigation responses by agent:** `ui/src/ui/app-render.ts:2271-2274`
  - body: The navigation list callback only checks `workspacePath`. If the user switches agents while a root-path list is in flight, the old response can still apply because the reset path is also `""`, showing the previous agent's tree under the new selected agent. Capture the requested agent and check it before mutating workspace entries or error state.
  - confidence: 0.9
- **[P2] Allow valid dot-prefixed workspace filenames:** `src/gateway/server-methods/agents-workspace.ts:53-54`
  - body: The `startsWith("..")` guard rejects valid filenames such as `...` and `..notes`; the unsafe case is an exact `..` path segment, which the segment check below already handles. This also contradicts the new test that expects `...` to be valid.
  - confidence: 0.94
- **[P2] Keep mkdir's default parents behavior recursive:** `src/gateway/server-methods/agents-workspace.ts:618`
  - body: Using `params.parents || false` makes omitted `parents` non-recursive, while the proposed API and tests describe the default as creating parent directories. Use an explicit default such as `params.parents !== false`, or update the schema/tests/API text together.
  - confidence: 0.89
- **[P2] Mock the recursive delete path:** `src/gateway/server-methods/agents-workspace.test.ts:607`
  - body: The implementation now uses `fs.rm` for recursive directory deletes, but this test still stubs `removePathWithinRoot`. Since `fs.rm` is not mocked, the test can hit the real `/tmp/test-workspace/...` path and return the wrong result instead of covering the recursive branch.
  - confidence: 0.91
- **[P3] Credit the workspace feature changelog entry:** `CHANGELOG.md:2955`
  - body: This is a user-facing feature entry, and repo policy requires a non-bot `Thanks @...` attribution. Add the correct contributor credit to the new single-line changelog bullet.
  - confidence: 0.9
