  - body: The branch adds public `agents.workspace.*` schemas under `src/gateway/protocol`, but current main's canonical protocol surface is `packages/gateway-protocol`. Keeping the old tree would reintroduce a parallel protocol source that clients and generated models can drift from.
  - confidence: 0.92
- **[P1] Register workspace methods through core descriptors:** `src/gateway/server-methods.ts:115`
  - body: The branch wires the new handlers through the old eager aggregate, while current main builds core Gateway dispatch from `CORE_GATEWAY_METHOD_SPECS` and throws when a core handler has no descriptor. Add these methods to the descriptor table with the intended scopes instead of relying on the removed scopes/list registry.
  - confidence: 0.9
- **[P1] Bound recursive workspace listings:** `src/gateway/server-methods/agents-workspace.ts:143-145`
  - body: When `recursive` is true, `workspaceList` descends every directory and accumulates all entries without a depth, entry-count, byte, or time budget. A large workspace can make a read-scoped RPC consume Gateway CPU and memory, so this needs pagination/bounds or the recursive mode should be removed before merge.
  - confidence: 0.9
