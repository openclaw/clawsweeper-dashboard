## Best Possible Solution

Land the focused parser, runner, and classifier fix after normal checks if maintainers accept the targeted fallback semantics; keep explicit `NO_REPLY` silent replies out of fallback behavior.

## Solution Assessment

Yes. Parser-level detection plus runner metadata propagation is the best fix boundary because the channel layer cannot distinguish a deliberate silent reply from a CLI-generated placeholder once the runner has collapsed the turn to a generic empty response.
