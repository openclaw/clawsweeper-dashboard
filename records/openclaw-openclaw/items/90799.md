## Summary

Keep open. The PR is a plausible fix for the linked Claude CLI synthetic-placeholder bug, but it is not merge-ready because the new classifier can mark a run as `empty_result` even when a later CLI result contains visible assistant text, and the branch is currently conflicting with `main` while carrying broad unrelated formatting churn.

## What This Changes

The branch detects Claude CLI `<synthetic>` placeholder JSONL records, carries a `syntheticPlaceholder`/`terminalReplyKind` marker through CLI run results, classifies that marker as fallback-worthy empty output, and adds parser/classifier/runner tests plus broad unrelated formatting churn.

## Best Possible Solution

Land a narrowed, mergeable repair that classifies exact Claude CLI synthetic placeholders as fallback-worthy only when the final run has no non-silent visible output and no committed delivery, with regression coverage for both empty and later-non-empty result streams.
