## What This Changes

This PR detects known Claude CLI `model: "<synthetic>"` no-output placeholders, preserves a `syntheticPlaceholder` marker through CLI parsing and `runPreparedCliAgent`, and classifies the terminal result as `empty_result` for model fallback.

## Best Possible Solution

Land the narrow parser, runner, and classifier fix if maintainers accept that exact Claude CLI synthetic placeholders with no visible assistant text should route through existing model fallback as empty results; keep qa-lab Telegram fixture support as a follow-up.

## Reproduction Assessment

Yes. Current main is source-reproducible: the reported assistant text block is not extracted, the empty terminal result remains empty output, and `runPreparedCliAgent` throws or silences before a synthetic-placeholder classification can exist.

## Solution Assessment

Yes, with a maintainer semantics decision. Parser-level marker detection plus runner metadata and classifier handling is the narrow owner-boundary fix; a Telegram-only status workaround would leave the core fallback path blind.
