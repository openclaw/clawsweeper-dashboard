## What This Changes

The branch detects exact Claude CLI `<synthetic>` placeholder assistant events, carries a `syntheticPlaceholder`/`terminalReplyKind` marker through CLI run results, classifies it as `empty_result`, and adds parser/classifier/runner tests alongside broad formatting churn.

## Best Possible Solution

Land a narrowed parser-to-runner-to-classifier fix that only marks synthetic-placeholder turns when the final result has no visible assistant output, with focused coverage for empty, allow-empty, and synthetic-then-visible cases.

## Solution Assessment
