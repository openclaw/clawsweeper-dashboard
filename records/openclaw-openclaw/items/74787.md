## Summary

Keep open. The PR addresses a real acpx/Codex ACP orphan-process leak that is still present on current main, but the latest diff still has source-level blockers around generated wrapper ownership, process-list fallback cleanup, and unsupported startup scan noise before it should merge.

## What This Changes

The PR adds an acpx process reaper, hooks startup and cancel/close cleanup for OpenClaw-owned Codex ACP/acpx process trees, hardens generated ACP wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the fix inside the acpx plugin boundary, pass the generated wrapper root used by prepareAcpxCodexAuthConfig into startup reaping, preserve a recorded-root PID kill fallback when process enumeration is unavailable, and make unsupported startup scans quiet best-effort behavior before landing with focused acpx tests and the changed gate.

## Reproduction Assessment

Yes. Current main source and docs show no PID-level cancel/close cleanup and manual stale-process recovery, while the PR discussion provides installed v2026.4.29 evidence of PPID=1 codex-acp accumulation; the remaining PR blockers are reproducible by source inspection around custom stateDir and process-list failure paths.

## Solution Assessment

No, not as-is. The ownership-limited acpx-local reaper is the right boundary, but this diff should repair the wrapper-root mismatch, list-failure fallback, and unsupported-scan warning behavior before it is the narrow maintainable fix.

- **[P2] Pass the generated wrapper root to startup reaping:** `extensions/acpx/src/service.ts:221-223`
  - body: Startup generates Codex/Claude ACP wrappers under ctx.stateDir/acpx, but this call filters stale-wrapper ownership with pluginConfig.stateDir.   - confidence: 0.94

  - body: When process listing throws, this path records an empty process list and cleanup receives that empty list. The helper can validate rootCommand, but collectProcessTree finds no root and sends no signal to the recorded PID, so cancel/close cleanup is lost on hosts where ps is unavailable or incomplete.
  - confidence: 0.88
- **[P3] Keep unsupported startup scans quiet:** `extensions/acpx/src/service.ts:229-231`
  - body: Startup invokes the Unix ps scanner unconditionally and logs any failure as a warning. On Windows or hosts without that scanner, every acpx startup can warn for a best-effort cleanup path; gate unsupported platforms or make unsupported listing return an empty result.
  - confidence: 0.8
