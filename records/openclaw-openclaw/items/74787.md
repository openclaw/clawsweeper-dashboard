## Summary

Keep open. The acpx-local reaper direction matches the live orphan-process evidence, but the current PR head still misses generated wrapper roots and Claude wrapper ownership, and one fallback path sends no signal when process listing fails.

## What This Changes

The PR adds an acpx-local process reaper, wires it into service startup and runtime cancel/close, hardens generated Codex/Claude ACP wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Land an acpx-local, ownership-limited reaper after aligning the startup wrapper root, covering intended generated wrapper variants, preserving recorded-PID fallback behavior, and keeping unsupported scans quiet.

## Reproduction Assessment

Yes. Current main still delegates cancel/close and documents manual stale-process cleanup, while the PR discussion provides installed v2026.4.29 PPID=1 codex-acp orphan evidence; the remaining PR bugs reproduce by comparing the generated wrapper paths and ownership filters in the diff.

## Solution Assessment

No, not as-is. The plugin-local cleanup boundary is right, but this head needs the focused root, Claude wrapper, process-list fallback, and unsupported-scan repairs before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the generated wrapper root to startup reaping:** `extensions/acpx/src/service.ts:221-223`
  - body: Startup writes generated Codex/Claude ACP wrappers under `ctx.stateDir/acpx`, but this call filters wrapper ownership with `pluginConfig.stateDir`. When those paths differ, stale generated wrapper roots are skipped and can survive the startup cleanup this PR is meant to provide.
  - confidence: 0.94
- **[P2] Recognize generated Claude wrappers in ownership checks:** `extensions/acpx/src/process-reaper.ts:56-64`
  - body: `prepareAcpxCodexAuthConfig` also writes `claude-agent-acp-wrapper.mjs`, and this PR hardens the shared wrapper script, but the reaper only accepts `codex-acp-wrapper.mjs`. Recorded close/cancel cleanup and startup reaping therefore skip OpenClaw-generated Claude ACP wrapper trees.
  - confidence: 0.9
- **[P3] Signal the recorded PID when process listing fails:** `extensions/acpx/src/runtime.ts:527-532`
  - body: When `listProcesses` throws, `resolveCleanupContextFromRecord` passes an empty process list through. `cleanupOpenClawOwnedAcpxProcessTree` can prove ownership from `rootCommand`, but with no root entry it collects an empty tree and sends no signal to the stored PID.
  - confidence: 0.88
- **[P3] Keep unsupported startup scans quiet:** `extensions/acpx/src/process-reaper.ts:117-122`
  - body: Startup always reaches the Unix `ps` scanner through the default reaper path and logs failures as warnings. On Windows or hosts without that scanner, every acpx startup can warn for a best-effort cleanup path; gate unsupported scans or treat them as an empty process list.
  - confidence: 0.8
