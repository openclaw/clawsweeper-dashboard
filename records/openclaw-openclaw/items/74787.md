## Summary

Keep open. The underlying acpx orphan-process bug is valid and this PR is pointed at the right plugin-local boundary, but the current head still misses intended generated wrapper paths and has two cleanup fallback gaps that should be fixed before merge.

## What This Changes

The PR adds an acpx-local process reaper, wires it into startup and runtime cancel/close, hardens generated Codex/Claude ACP wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Land an acpx-owned, ownership-limited reaper after aligning the startup wrapper root, covering generated Claude wrappers, preserving recorded-PID fallback cleanup, and keeping unsupported scans quiet.

## Reproduction Assessment

Yes. Current main lacks PID/tree cleanup on acpx cancel/close and still documents manual stale-process cleanup, while the PR discussion supplies installed v2026.4.29 PPID=1 codex-acp orphan evidence; the remaining PR defects reproduce by inspecting the diff paths and filters.

## Solution Assessment

No, not as-is. The plugin-local ownership-limited approach is the right maintainable direction, but this head needs the focused root, Claude-wrapper, process-list fallback, and unsupported-scan fixes before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the generated wrapper root to startup reaping:** `extensions/acpx/src/service.ts:221-223`
  - body: Startup writes generated Codex/Claude ACP wrappers under `ctx.stateDir/acpx`, but this call filters wrapper ownership with `pluginConfig.stateDir`. When those paths differ, stale OpenClaw-generated wrappers are skipped and can survive the startup cleanup this PR is meant to provide.
  - confidence: 0.94
- **[P2] Recognize generated Claude ACP wrappers:** `extensions/acpx/src/process-reaper.ts:45-50`
  - body: `prepareAcpxCodexAuthConfig` also writes `claude-agent-acp-wrapper.mjs`, and this PR hardens the shared wrapper script, but the reaper only accepts `codex-acp-wrapper.mjs`. Recorded close/cancel cleanup and startup reaping therefore skip OpenClaw-generated Claude ACP wrapper trees.
  - confidence: 0.9
- **[P3] Signal the recorded PID when process listing fails:** `extensions/acpx/src/runtime.ts:531`
  - body: When `listProcesses` throws, cleanup receives an empty process list. Ownership can still be proven from `rootCommand`, but with no root entry `collectProcessTree` returns an empty tree and sends no signal to the stored PID.
  - confidence: 0.88
- **[P3] Keep unsupported startup scans quiet:** `extensions/acpx/src/process-reaper.ts:100-102`
  - body: The default startup reaper shells out to `ps` and the service logs any failure as a warning. On Windows or hosts without that scanner, every acpx startup can warn for a best-effort cleanup path; gate unsupported scans or treat them as an empty process list.
  - confidence: 0.8
