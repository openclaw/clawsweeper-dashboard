## Summary

Keep open. The PR addresses a real acpx orphan-process bug and the live v2026.4.29 evidence supports the direction, but the latest head still has source-backed cleanup gaps that would leave generated Claude wrappers or some recorded process roots unreaped.

## What This Changes

The PR adds an acpx-local process reaper, generated Codex/Claude wrapper signal hardening, cancel/close process-tree cleanup, startup orphan reaping, tests, docs, and a changelog entry.

## Best Possible Solution

Land an ownership-limited acpx reaper after it uses the generated wrapper root, covers both generated wrappers, preserves recorded-PID fallback cleanup, and treats unsupported process scans as quiet best effort.

## Reproduction Assessment

Yes. Current main has no PID-level cleanup on acpx cancel/close and still documents manual stale-process cleanup, and the PR discussion adds live v2026.4.29 PPID=1 codex-acp orphan evidence.

## Solution Assessment

No, not as-is. The acpx-local, ownership-limited cleanup boundary is the right maintainable approach, but this head needs focused fixes for wrapper-root selection, Claude wrapper ownership, recorded-PID fallback, and unsupported startup scans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the generated wrapper root to startup reaping:** `extensions/acpx/src/service.ts:221-223`
  - body: Startup generates the Codex/Claude wrappers under ctx.stateDir/acpx, but this call filters wrapper ownership with pluginConfig.stateDir. When those paths differ, stale generated wrappers are skipped by the startup cleanup this PR is meant to provide.
  - confidence: 0.94
- **[P2] Recognize generated Claude wrappers in ownership checks:** `extensions/acpx/src/process-reaper.ts:56-58`
  - body: prepareAcpxCodexAuthConfig also writes claude-agent-acp-wrapper.mjs, and this PR hardens the shared wrapper script, but the reaper only accepts codex-acp-wrapper.mjs. Recorded close/cancel cleanup and startup reaping therefore skip OpenClaw-generated Claude ACP wrapper trees.
  - confidence: 0.91
- **[P3] Signal the recorded PID when process listing fails:** `extensions/acpx/src/runtime.ts:508-512`
  - body: When listProcesses throws, cleanup receives an empty process list. The cleanup path can prove ownership from rootCommand, but with no root entry it collects an empty tree and sends no signal to the stored PID.
  - confidence: 0.9
- **[P3] Keep unsupported startup scans quiet:** `extensions/acpx/src/service.ts:229-232`
  - body: Startup always reaches the Unix ps scanner through the default reaper path and logs failures as warnings. On Windows or hosts without that scanner, every acpx startup can warn for a best-effort cleanup path; gate unsupported scans or treat them as an empty process list.
  - confidence: 0.82
