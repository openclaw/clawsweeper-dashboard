## Summary

Keep open. The PR targets a real acpx orphan-process problem, but the latest head still leaves file-local cleanup gaps that can skip the generated wrappers it is meant to reap.

## What This Changes

The PR adds an acpx-local process reaper for OpenClaw-owned Codex ACP wrapper/plugin-runtime-deps processes, wires it into startup and cancel/close, hardens generated wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Land an acpx-local, ownership-limited reaper after aligning the startup wrapper root, covering generated wrapper variants intentionally, preserving PID fallback behavior, and keeping unsupported scans quiet.

## Reproduction Assessment

Yes. Current main only delegates acpx cancel/close and still documents manual stale-process cleanup, and the PR discussion provides installed v2026.4.29 evidence of PPID=1 codex-acp orphans cleared by SIGTERM.

## Solution Assessment

No, not as-is. The acpx plugin boundary is the right place, but the current diff needs the focused ownership/root/fallback repairs before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the generated wrapper root to startup reaping:** `extensions/acpx/src/service.ts:221-223`
  - body: Startup creates generated Codex/Claude ACP wrappers under `ctx.stateDir/acpx`, but this call filters wrapper ownership with `pluginConfig.stateDir`. With a custom acpx `stateDir`, stale wrappers under the service state directory are skipped, so the PPID=1 processes this cleanup targets can survive startup.
  - confidence: 0.94
- **[P2] Recognize generated Claude wrappers in ownership checks:** `extensions/acpx/src/process-reaper.ts:56-64`
  - body: `prepareAcpxCodexAuthConfig` also generates `claude-agent-acp-wrapper.mjs`, and the PR hardens the shared wrapper script, but the reaper only accepts `codex-acp-wrapper.mjs` and Codex/acpx runtime-deps commands. Recorded close/cancel cleanup and startup reaping therefore skip OpenClaw-generated Claude ACP wrapper trees.
  - confidence: 0.88
- **[P3] Signal the recorded PID when process listing fails:** `extensions/acpx/src/runtime.ts:527-532`
  - body: When `listProcesses` throws, `resolveCleanupContextFromRecord` records an empty process list and this call passes it through. `cleanupOpenClawOwnedAcpxProcessTree` can prove ownership from `rootCommand`, but with no root entry in `processes` it collects an empty tree and sends no signal to the stored PID.
  - confidence: 0.9
- **[P3] Keep unsupported startup scans quiet:** `extensions/acpx/src/process-reaper.ts:117-122`
  - body: Startup always reaches the Unix `ps` scanner through the default reaper path and logs failures as warnings. On Windows or hosts without that scanner, every acpx startup can warn for a best-effort cleanup path; gate unsupported scans or treat them as an empty process list.
  - confidence: 0.8
