## Summary

Keep open. The PR targets a valid ACP/acpx orphan-process bug and the acpx-local ownership boundary is appropriate, but the current head still has source-level correctness gaps that can leave generated wrapper trees unreaped or produce noisy startup warnings.

## What This Changes

The PR adds an acpx-owned process reaper, cancel/close cleanup for recorded ACP harness trees, generated Codex/Claude wrapper hardening, tests, docs, and a changelog entry.

## Best Possible Solution

Land an acpx-local, ownership-limited reaper after fixing wrapper-root selection, Claude wrapper ownership, recorded-PID fallback cleanup, and unsupported scan handling while leaving native Codex app-server and normal Codex CLI processes untouched.

## Reproduction Assessment

Yes. Current main source lacks PID-level acpx cleanup on cancel/close and still documents manual stale-process cleanup, and the PR discussion adds live v2026.4.29 PPID=1 codex-acp orphan evidence; I did not execute a live reproducer in this read-only pass.

## Solution Assessment

No, not as-is. The acpx-local cleanup boundary is the narrow maintainable solution, but this PR head needs focused fixes for generated wrapper ownership, fallback signaling, and unsupported process scans before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the generated wrapper root to startup reaping:** `extensions/acpx/src/service.ts:222`
  - body: `prepareAcpxCodexAuthConfig` writes generated Codex/Claude wrappers under `ctx.stateDir/acpx`, but startup reaping filters wrapper ownership with `pluginConfig.stateDir`. When a configured acpx state dir differs from the plugin service state dir, stale generated wrapper roots are skipped and can survive startup cleanup.
  - confidence: 0.94
- **[P2] Recognize generated Claude wrappers in ownership checks:** `extensions/acpx/src/process-reaper.ts:57-58`
  - body: This PR also generates and hardens `claude-agent-acp-wrapper.mjs`, but the reaper only accepts `codex-acp-wrapper.mjs`. Recorded close/cancel cleanup and startup reaping therefore skip OpenClaw-generated Claude ACP wrapper trees.
  - confidence: 0.91
- **[P3] Signal the recorded PID when process listing fails:** `extensions/acpx/src/runtime.ts:511-512`
  - body: When `listProcesses` throws, cleanup receives an empty process list. The cleanup path can still prove ownership from `rootCommand`, but with no root entry it collects an empty tree and sends no signal to the stored PID.
  - confidence: 0.9
- **[P3] Keep unsupported startup scans quiet:** `extensions/acpx/src/service.ts:221-222`
  - body: Startup always reaches the Unix `ps` scanner through the default reaper path and logs failures as warnings. On Windows or hosts without that scanner, every acpx startup can warn for a best-effort cleanup path; gate unsupported scans or treat them as an empty process list.
  - confidence: 0.82
