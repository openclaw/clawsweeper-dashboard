## Summary

Keep open. Current main still lacks PID-level acpx/Codex ACP process-tree cleanup, and the PR addresses a real, live-reproduced leak, but the latest head is not merge-ready because it still has source-level correctness and type-safety blockers in the acpx startup/runtime cleanup path.

## What This Changes

The PR adds acpx process reaping for OpenClaw-owned Codex ACP wrappers and plugin-runtime-deps processes, wires cleanup into cancel/close/startup, hardens generated wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Land a scoped acpx plugin fix that keeps ownership limited to OpenClaw-generated wrappers and plugin-runtime-deps ACP commands, uses the generated wrapper root for startup reaping, preserves a direct recorded-PID cleanup fallback, keeps unsupported startup scans quiet, and preserves the upstream acpx runtime test-option contract.

## Reproduction Assessment

Yes. Current main source and docs reproduce the missing PID-level cleanup path, and the PR discussion adds installed v2026.4.29 live evidence of PPID=1 codex-acp accumulation; the PR defects are reproducible by static inspection of custom stateDir wrapper generation, process-list failure handling, non-Unix startup behavior, and acpx 0.6.1 constructor types.

## Solution Assessment

No, not as-is. The acpx-local reaper is the right maintainable boundary, but this branch needs narrow repairs before merge so it actually reaches generated wrappers, remains type-safe, and degrades quietly when best-effort process enumeration is unavailable.

Full review comments:

- **[P2] Keep the acpx runtime test options type-safe:** `extensions/acpx/src/runtime.ts:436`
  - body: The wrapper now narrows `testOptions` to `AcpxRuntimeTestOptions`, then assigns it to the upstream `BaseAcpxRuntime` test-options type. acpx 0.6.1 only accepts `managerFactory`/`probeRunner` there, so this line is a type error and no longer cleanly forwards the upstream test hooks. Split the OpenClaw cleanup deps from the delegate test options or extend the wrapper type to include both shapes.
  - confidence: 0.9
- **[P2] Pass the generated wrapper root to startup reaping:** `extensions/acpx/src/service.ts:221-223`
  - body: Startup generates the Codex/Claude ACP wrappers under `ctx.stateDir/acpx`, but this call filters stale-wrapper ownership with `pluginConfig.stateDir`. With a custom acpx stateDir, stale generated wrappers under the service state directory are skipped, so startup reaping misses the process tree this PR is meant to clear.
  - confidence: 0.94
- **[P3] Keep a recorded PID cleanup fallback when listing fails:** `extensions/acpx/src/runtime.ts:508-512`
  - body: `resolveCleanupContextFromRecord` turns a process-list failure into `[]`, and cleanup then sees no recorded root process to signal. The stored PID and `rootCommand` can still prove an OpenClaw-owned root, so cancel/close cleanup is lost when process enumeration is unavailable or incomplete.
  - confidence: 0.88
- **[P3] Keep unsupported startup scans quiet:** `extensions/acpx/src/service.ts:229-231`
  - body: Startup invokes the Unix `ps` scanner and logs any failure as a warning. On Windows or hosts without that scanner, every acpx startup can warn for a best-effort cleanup path; gate unsupported platforms or make unsupported listing return an empty result.
  - confidence: 0.81
