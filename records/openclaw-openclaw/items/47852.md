## Summary

Current main already provides the central per-agent generation-parameter surface through `agents.list[].params`, merges it over global and model defaults for the active agent, and applies those params during embedded agent runs. Remaining provider-specific direct penalty passthrough is already tracked separately at https://github.com/openclaw/openclaw/issues/32496.

## What This Changes

Request to let individual agents configure generation parameters such as temperature, max token budget, sampling, and penalty settings in `openclaw.json`.

## Best Possible Solution

Keep the shipped flat `agents.list[].params` surface, and leave direct provider-specific penalty aliases or broader adaptive thinking policy to the existing narrower follow-up issues.
