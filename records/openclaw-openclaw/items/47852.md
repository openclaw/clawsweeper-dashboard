## Summary

Keep open. Current main partially covers per-agent tuning through flat `agents.list[].params`, but it still lacks the requested `generation` block and does not generically normalize or forward top-p/top-k and penalty fields across the main stream path.

## What This Changes

The issue requests optional per-agent generation parameters such as temperature, top-p/top-k, penalties, and max tokens in `openclaw.json`.

## Best Possible Solution

Keep this open until maintainers choose the public config contract, then either extend documented `agents.list[].params` or add a documented `generation` alias with provider-safe normalization for supported sampling fields.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main accepts flat per-agent `params` but rejects the requested `generation` block and does not generically forward the requested sampling aliases through the main stream wrapper.

## Solution Assessment

Unclear. The requested `generation` block is reasonable, but current main already has a documented `params` surface, so the best fix depends on maintainer API-contract judgment and provider compatibility policy.
