## Summary

Keep open. Current main still has the session-rotation path that this PR targets, and the PR is the right narrow direction, but the proposed patch would regress current heartbeat/default overflow guidance by replacing dynamic recovery copy with a static preserved-session message.

## What This Changes

The PR removes the compaction-failure session reset hook, returns preserved-session recovery guidance, updates docs/changelog, and adds regression tests for embedded overflow payloads and thrown compaction failures.

## Best Possible Solution

Land a refreshed version that preserves active session mappings on failed overflow/compaction, retains current dynamic overflow guidance, leaves role-ordering reset behavior unchanged, and keeps docs/changelog/tests aligned.

## Reproduction Assessment

Yes. On current main, a context-overflow payload or thrown compaction-failure error reaches `runAgentTurnWithFallback`, calls `resetSessionAfterCompactionFailure`, and `resetReplyRunSession` rewrites the active session mapping.

## Solution Assessment

No, not as-is. Preserving the mapping is the right fix, but the PR should build its preserved-session message on the existing dynamic overflow hint path so heartbeat bleed and `reserveTokensFloor` guidance are retained.
