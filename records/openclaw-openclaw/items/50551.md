## Summary

Keep open. Current main still has the plausible SwiftUI layout coupling from the report: the scrollable message list and the variable-height iOS composer are siblings in one vertical layout, and no current-main change or changelog entry shows composer-height changes being isolated from long message-list layout work.

## What This Changes

The issue reports that the iOS/iPad chat app briefly freezes when the multiline text input bar grows vertically, likely from bottom inset or padding changes forcing a full message-list relayout.

## Best Possible Solution

Stabilize the shared SwiftUI chat layout so composer height changes update only the composer/bottom affordance instead of invalidating expensive transcript layout, then verify on iPhone/iPad with a long transcript and profiling evidence.

## Reproduction Assessment

Unclear. The best reproduction path is to run the iOS app with a long chat transcript, focus the composer, type enough text for multiline growth, and profile SwiftUI layout invalidation or main-thread stalls.

## Solution Assessment

Unclear. The fix belongs in the shared SwiftUI chat UI, but profiling should choose the narrowest maintainable approach, such as isolating the composer from the transcript layout, stabilizing geometry, or reducing scroll/list invalidation.
