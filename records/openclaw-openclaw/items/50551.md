## Summary

Keep open. Current main still has the same plausible SwiftUI layout coupling path between the iOS chat message list and the variable-height composer, and I did not find a merged fix or direct duplicate that isolates composer-height changes from list relayout.

## What This Changes

Request to prevent iOS/iPad chat input composer height changes from briefly freezing the app by avoiding full message-list relayout when the input bar grows.

## Best Possible Solution

Stabilize the shared SwiftUI chat layout so composer growth changes only the bottom composer area, then verify on iPhone/iPad with a long transcript and SwiftUI layout profiling or before/after responsiveness evidence.

## Reproduction Assessment

Unclear. The source path is plausible, but a high-confidence reproduction requires running the iOS app with a long chat transcript, focusing the composer, typing enough multiline text for height changes, and observing or profiling the freeze.

## Solution Assessment

Unclear. The fix belongs in shared SwiftUI chat UI, but profiling should choose the narrowest approach, such as overlaying the composer, using a stable bottom inset, deferring scroll changes, or reducing row invalidation.
