## Summary

Keep open. Current main still has the plausible iOS SwiftUI layout coupling described in the report: the scrollable message list and composer are in one vertical layout, and the iOS composer still uses a focus-compacted, bounded-height TextEditor. I did not find a merged current-main change, changelog entry, or related PR that isolates composer-height changes from chat-list layout work.

## What This Changes

The issue reports that the iOS/iPad chat app briefly freezes when the text input bar grows vertically during multi-line typing.

## Best Possible Solution

Stabilize the shared SwiftUI chat layout so composer height changes do not force expensive message-list layout work, then verify on iPhone/iPad with a long transcript and before/after responsiveness or profiling evidence.

## Reproduction Assessment

Unclear. The best reproduction path is to run the iOS app with a long chat transcript, focus the composer, type enough text for multiline growth, and profile SwiftUI layout invalidation or main-thread stalls.

## Solution Assessment

Unclear. The fix belongs in the shared SwiftUI chat UI, but profiling should choose the narrowest maintainable approach, such as isolating the composer, stabilizing geometry, deferring scroll updates, or reducing row invalidation.
