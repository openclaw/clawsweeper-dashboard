## What This Changes

This PR deletes `replyTo` normalization and `message_reference` spreading from the legacy Discord webhook payload builder at `src/discord/send.outbound.ts`.

## Best Possible Solution

Port the webhook-only removal into `extensions/discord/src/send.webhook.ts`, add focused regression coverage, keep bot-token reply support in `send.message-request.ts`, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `sendWebhookMessageDiscord` serializes `message_reference` whenever `replyTo` is set, and the outbound adapter can pass `replyToId` into that sender.

## Solution Assessment

No, not as submitted. Removing the unsupported webhook field is the narrow maintainable fix, but this PR applies it to a stale `src/discord` path instead of the active Discord plugin sender.
