## Summary

Keep open: the PR targets a real Telegram ingress stall and has strong before/after Telegram proof, but it is not merge-ready because it can fail a claim still owned by another live process using timeout plus local handler absence as the only stale-owner signal.

## What This Changes

The PR adds a Telegram polling-session path that fails timed-out live-owned spooled ingress claims before draining later same-lane updates, plus a regression test.

## Best Possible Solution

Tie live-owned claim recovery to durable owner leases, heartbeat or diagnostic non-progress evidence, or explicit maintainer acceptance of timeout-based dead-lettering, then land the focused Telegram fix with the existing proof.

## Solution Assessment

No. The PR targets the right failure mode and has strong proof, but timeout plus this gateway having no active handler state is not enough proof that another live worker is stale.
