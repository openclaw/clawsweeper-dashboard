## Summary

Keep open: the Mantis Telegram Desktop proof now clears the real-behavior proof gate, but the patch still fails other-live-process claims based on timeout age plus local handler absence, which is not enough proof that the owner stopped processing.

## What This Changes

The PR adds Telegram polling-session logic to fail timed-out live-owned spooled ingress claims before draining later same-lane updates, plus a focused regression test.

## Best Possible Solution

Tie live-owned claim recovery to a maintainer-approved stale-owner signal, such as diagnostic non-progress evidence or a durable owner lease, then keep the token-guarded failure path and Telegram proof.

Yes. Mantis proof shows current main leaving the stuck-lane Telegram message unanswered while the PR candidate replies, and source inspection shows current main keeps other-live-process claims guarded.

## Solution Assessment

No. The PR targets the right failure, but timeout age plus local handler absence is not sufficient stale-owner proof for failing another live worker's claim; diagnostic-backed non-progress proof or a durable owner lease would be safer.
