## Summary

Keep open: the PR targets a real Telegram ingress stall, but the patch changes live-owner claim semantics without enough stale-owner proof and still lacks after-fix real Telegram or gateway behavior proof.

## What This Changes

The PR changes Telegram polling-session draining to fail timed-out live-owned spooled claims with no active local lane, plus a regression test for same-lane drainage.

## Best Possible Solution

Tie live-owned claim recovery to durable owner leases or diagnostic non-progress evidence, keep the live-owner guard as the default safety boundary, and validate with real Telegram or gateway proof.

## Reproduction Assessment

Yes. Source inspection shows current main keeps other-live-process Telegram claims guarded while later same-lane updates remain pending, and the linked issue provides production logs and claimed-row evidence; I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The PR targets the right failure, but timeout age plus local handler absence is not sufficient stale-owner proof for failing another live worker's claim; diagnostic-backed non-progress proof or a durable owner lease would be safer.
