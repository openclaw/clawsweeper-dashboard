## Summary

Keep this PR open, but it is not merge-ready: the patch addresses the right Telegram stuck-ingress symptom and has strong Mantis proof, yet it can fail a claim owned by another live process using only age plus this gateway's lack of active handler state.

## What This Changes

This PR adds a Telegram polling-session recovery path that fails timed-out live-owned spooled ingress claims before draining later same-lane updates, plus a regression test.

## Best Possible Solution

Tie live-owned claim recovery to durable owner leases, diagnostic non-progress evidence, or explicit maintainer acceptance of timeout-based dead-lettering, then land the focused Telegram fix with the existing Mantis proof.
