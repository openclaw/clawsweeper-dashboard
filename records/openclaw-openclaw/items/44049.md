## Summary

Keep open: the production change is plausibly correct, but the added regression test bypasses the changed ACP arg builder, so the patch is not merge-ready.

## What This Changes

This PR changes ACP client server-argument construction to drop one leading `acp` from forwarded args and adds a regression test under `src/acp/client.test.ts`.

## Best Possible Solution

Keep the narrow one-token normalization, but replace the helper-level duplicate test with coverage of `buildServerArgs` or an end-to-end `createAcpClient` spawn assertion before landing.

## Reproduction Assessment

Yes. Current main source shows `buildServerArgs` prepends `acp` to raw `serverArgs`, and `createAcpClient` sends that built argv to the spawn path.

## Solution Assessment

No, not as proposed. The production fix is the narrow maintainable change, but the proposed test bypasses the changed builder and would not catch a regression.
