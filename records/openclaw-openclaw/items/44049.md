## Summary

Keep open: the production change is a narrow fix for a source-reproducible ACP client bug, but the added regression test sits below the changed argument-builder boundary and would still pass if the production fix were removed.

## What This Changes

The branch drops one leading `acp` token from ACP client `serverArgs` before prepending the canonical subcommand and adds a spawn-invocation helper test.

## Best Possible Solution

Land the one-token ACP argument normalization with regression coverage at the argument builder or `createAcpClient` spawn boundary, preserving verbose handling, env stripping, Windows wrapper behavior, and session semantics.

## Reproduction Assessment

Yes. Current main source shows `buildServerArgs` prepending `acp` to unnormalized forwarded args, and `createAcpClient` sends that result to the spawn path; I did not execute the CLI in this read-only review.

## Solution Assessment

No, not fully as proposed. The production normalization is the narrow maintainable fix, but the regression test should cover the builder or `createAcpClient` spawn argv instead of the lower-level spawn materializer.
