## What This Changes

The PR normalizes ACP client server arguments by dropping one leading `acp` before prepending the canonical subcommand, and adds a regression test.

## Best Possible Solution

Land the one-token ACP argument normalization with regression coverage at the builder or end-to-end client spawn boundary, while preserving existing `--server-verbose` behavior and ACP server/session semantics.

## Reproduction Assessment

## Solution Assessment

No, not fully as proposed. The production normalization is the narrow maintainable fix, but the regression test should cover `buildServerArgs` directly or assert the `createAcpClient` spawn argv, and the branch needs refresh.
