## What This Changes

The PR strips one leading `acp` token from ACP client `serverArgs` before prepending the canonical subcommand and adds a helper-level spawn invocation test.

## Best Possible Solution

Land the one-token normalization with regression coverage at `buildServerArgs` or the `createAcpClient` spawn boundary while preserving verbose handling, env stripping, Windows wrapper behavior, and session semantics.
