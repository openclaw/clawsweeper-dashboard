The PR normalizes ACP client `serverArgs` by removing one leading `acp` before prepending the canonical ACP subcommand and adds a regression test.

## Best Possible Solution

Land the one-token normalization with regression coverage that exercises `buildServerArgs` directly or reaches it through `createAcpClient`, while preserving existing `--server-verbose` behavior.
