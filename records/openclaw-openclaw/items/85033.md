## What This Changes

The PR prunes Docker runtime images by removing omitted package-excluded plugin source workspaces and their runtime dependency closures unless those plugins are opted in through OPENCLAW_EXTENSIONS.

## Best Possible Solution

Land this pruning approach after maintainer approval, while relying on release Docker gates or an optional multi-arch smoke to catch architecture-specific native package gaps.

## Solution Assessment

Yes: the PR extends the existing package-exclusion and OPENCLAW_EXTENSIONS contract instead of adding a new Docker option, and the kept-closure check is the right guard against removing packages still needed by opted-in plugins.
