## Summary

Keep open: the fix is useful for the linked cron truncation bug, but the current patch can route preserved auth prompts into noninteractive delivery when the captured tail normalizes to empty, and external real behavior proof is still missing.

## What This Changes

The PR preserves bounded action-critical cron command output lines for interactive run history, propagates an optional deliverySummary through cron events, and sends tail-only summaries to announce/webhook delivery with regression tests.

## Best Possible Solution

Keep the preserved-line approach, but make noninteractive delivery use an explicit tail-only result even when the tail is empty, preserve the existing safe webhook event shape, and add redacted real cron proof before merge.

## Reproduction Assessment

Yes by source inspection, not by a live run in this read-only review. Current main tail-caps command output before summarizing, and the PR defect is source-reproducible with an auth prompt followed by enough whitespace to leave an empty normalized captured tail.

## Solution Assessment

No. Preserving action-critical lines is the right owner-boundary direction, but the current truthy deliverySummary gate lets preserved interactive output fall back into delivery; the safer fix is an explicit delivery projection that handles empty tails.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep delivery summaries explicit when the tail is empty:** `src/cron/command-runner.ts:213`
  - body: `deliverySummary` is only added when the tail summary is truthy. If a command prints a preserved auth prompt and then enough whitespace for the captured tail to normalize away, this omits `deliverySummary`; `server-cron.ts` then falls back to the interactive `summary` and can send the preserved prompt to announce/webhook delivery. Please represent an empty tail-only delivery result explicitly so noninteractive delivery is suppressed instead of using the preserved summary.
  - confidence: 0.87
- **[P2] Preserve webhook diagnostic entries while overriding the summary:** `src/gateway/server-cron-notifications.ts:101`
  - body: When `deliverySummary` exists, the webhook payload replaces `diagnostics` with `{ summary, entries: [] }`. Current webhooks receive the event diagnostics, and command-runner diagnostic entries do not contain the preserved prompt text, so dropping them only for this new path is an avoidable compatibility break. Please keep the existing entries and override only the delivery-facing summary unless maintainers explicitly accept the contract change.
  - confidence: 0.78
