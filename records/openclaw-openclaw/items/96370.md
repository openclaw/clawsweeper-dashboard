## Summary

Keep this PR open: the bug is valid and the preservation approach is plausible, but the patch currently breaks the cron completion webhook payload shape when `deliverySummary` is present and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR adds preserved-output-line capture for cron command jobs, separates interactive summaries from tail-only delivery summaries, propagates `deliverySummary` through cron events, and adds cron/gateway regression tests.

## Best Possible Solution

Keep the preservation fix, but make delivery-summary webhook handling preserve the full existing event shape while overriding only delivery-facing summary/diagnostics, then add real cron run proof before merge.

## Reproduction Assessment

Yes. Current main source shows command output is capped to the captured tail and `runCronCommandJob` summarizes only that tail, matching the linked synthetic cron reproduction.

## Solution Assessment

No. Preserving action-critical lines is a good fix direction, but the current webhook projection is not the best implementation because it drops existing event fields instead of preserving the event shape and overriding only delivery-facing content.
