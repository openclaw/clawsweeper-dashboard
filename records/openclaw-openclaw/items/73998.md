## Summary

Keep open; the PR targets a real Feishu reconnect gap, but it is not mergeable because malformed WebSocket endpoint responses can still stay inside the SDK's infinite retry loop and the branch regresses current-main identity-preserving recovery cleanup.

## What This Changes

The PR replaces Feishu WSClient constructor `wsConfig` handling with a sanitizing HTTP instance and SDK lifecycle hooks, rewrites monitor retry cleanup around SDK reconnect callbacks, adds Feishu tests, and adds a changelog entry.

## Best Possible Solution

Rebase on current main, add a Feishu-owned malformed endpoint policy that reaches bounded recovery, preserve #73739 identity cleanup, and cover both client sanitization and monitor retry behavior in focused tests.

## Solution Assessment

Current main deliberately preserves those maps during supervisor recovery so mentions, reactions, and card actions keep the bot identity while reconnecting; carry that `clearIdentity: false` behavior forward and test it.
  - confidence: 0.8
