## Summary

Keep open. Current main still has the generated workspace template ordered so Memory comes before Red Lines, External vs Internal, and Tools, and the bootstrap trimmer still uses fixed head/tail truncation without section priority. The issue is also paired with open, unmerged PR #75248 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to preserve load-bearing Red Lines, external-action guidance, and tool-use guidance from generated workspace AGENTS.md when users set small agents.defaults.bootstrapMaxChars budgets.

## Best Possible Solution

Review and land PR #75248, or an equivalent narrow fix, with regression coverage proving Red Lines, External vs Internal, and Tools survive a 1500-character AGENTS.md bootstrap budget. Broader content-priority truncation or tiered bootstrap loading can remain follow-up work.

## Reproduction Assessment

Yes. The reporter provided live session evidence, and current main has a source-level reproduction: applying the current trimmer to the current generated AGENTS.md at bootstrapMaxChars=1500 excludes Red Lines, External vs Internal, and Tools.

## Solution Assessment

Yes for the near-term direction. Reordering the generated template is the narrowest maintainable fix for newly seeded workspaces; priority-aware truncation or tiered bootstrap loading would be larger follow-ups rather than blockers for this bug.
