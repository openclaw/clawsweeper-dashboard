## Summary

Keep open. Current main is unchanged in the relevant Discord native plugin-command paths since the prior ClawSweeper review: an unmatched Discord application command still returns without acknowledgement, and plugin native command deployment still depends on runtime-registered command state rather than a startup readiness contract.

## What This Changes

The issue asks for Discord plugin-registered slash commands such as `/vc`, `/voice`, and `/codex` to acknowledge within Discord's 3-second window instead of failing with `Unknown interaction`.

## Best Possible Solution

Define a Discord native plugin-command readiness and reconciliation contract that ensures every deployed slash command has a local defer-first wrapper before interactions can arrive, while removing or updating stale Discord commands without broad eager plugin loading.

## Reproduction Assessment

Unclear. The issue includes concrete live logs from 2026.4.26, and current source shows the no-local-wrapper application-command path that would miss acknowledgement, but I did not run a live Discord smoke against current main.

## Solution Assessment

No. The issue's blanket eager-loading suggestion is not clearly the best fit for the plugin architecture; a readiness and deploy-reconciliation contract that preserves selective plugin activation is safer.
