## Summary

Keep open. Current main still has a source-level Discord application-command path that returns without acknowledging when Discord sends a command that has no local wrapper, and plugin native command wrappers are still derived from runtime `registerCommand()` state rather than manifest aliases.

## What This Changes

Discord plugin slash commands should acknowledge within Discord's 3-second window instead of failing when plugin command wrappers are not ready.

## Best Possible Solution

Define a Discord-native plugin-command readiness contract that guarantees every deployed plugin command has a local defer-first wrapper before Discord can invoke it, while preserving manifest-first/lazy plugin activation where possible.

## Reproduction Assessment

Unclear. The report has concrete logs and current source shows a missing-wrapper no-ack path, but this read-only review did not run a live Discord interaction against current main.

## Solution Assessment

No. The issue's blanket eager-loading suggestion is not clearly the best fit with the plugin architecture; a defer-first wrapper/readiness contract is the safer maintainable direction.
