## Summary

Keep open. Current main still has a Discord native-command path where a remotely deployed command without a local wrapper returns without acknowledging the interaction, and plugin native command specs still depend on runtime-registered command state rather than an explicit readiness contract.

## What This Changes

Ensure Discord plugin-registered native slash commands are locally ready and acknowledged within Discord's 3-second interaction deadline.

## Best Possible Solution

Define a Discord native command readiness and reconciliation contract where every remotely deployed command has a local defer-first wrapper before interactions can arrive, while preserving manifest-first plugin loading and removing stale remote commands predictably.

## Reproduction Assessment

No. The issue has concrete 2026.4.26 live logs and current source shows a source-reproducible missing-wrapper path, but I did not establish a live Discord failure against current main.

## Solution Assessment

No. The suggested blanket eager-loading direction is risky for the plugin architecture; a readiness and stale-command reconciliation contract is the narrower maintainable fix.
