## Summary

Keep open. Current main still has a plausible Discord-native readiness gap: known local command wrappers defer immediately, but a remotely registered plugin slash command with no matching local `client.commands` wrapper returns from interaction dispatch without any acknowledgement.

## What This Changes

Plugin-registered Discord slash commands should acknowledge within Discord's 3-second window instead of timing out when the owning plugin is not ready locally.

## Best Possible Solution

Define a Discord-native plugin-command readiness contract: Discord should either deploy only commands with local immediate-ack handlers, or install manifest-backed placeholders that defer first and then activate or dispatch the owning plugin deliberately.

## Reproduction Assessment

Unclear: the issue has concrete logs and current source shows a no-ack path for remotely registered commands absent from local `client.commands`, but this review did not run a live Discord interaction. A high-confidence reproduction should enable Discord native commands, invoke a plugin slash command whose local wrapper is absent, and capture received-to-defer timing plus plugin registration timing.

## Solution Assessment

No: blanket eager loading of plugin commands is not clearly the best fix because it conflicts with the repo's manifest-first and lazy activation direction. A narrower readiness/deployment or defer-first placeholder contract is safer.
