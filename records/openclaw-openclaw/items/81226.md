## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] Move inbound persistence behind completed processing:** `extensions/discord/src/monitor/listeners.ts:79-84`
  - body: The new listener records the message in recent-inbound after `messageHandler` returns, but the current handler only awaits debounce enqueue; the actual Discord run is processed later by the run queue. If the process aborts or dies after enqueue but before `processDiscordQueuedMessage` commits, reconnect backfill will see this persisted id and skip the only REST recovery path. Record this marker from the successful queued-processing commit path instead.
  - confidence: 0.91
- **[P2] Store reconnect recovery state in the canonical state DB:** `extensions/discord/src/recent-outbound.ts:70`
  - body: This adds steady-state JSON sidecars under the OpenClaw state dir plus env overrides for them. Root policy requires OpenClaw-owned runtime state and caches under state dirs to use SQLite/plugin KV or a named artifact, and the channel ingress/durable receive helpers already provide a DB-backed pattern. Keep this bounded in memory or move the persistent recovery state to the canonical SQLite/plugin-state surface before merge.
  - confidence: 0.84
