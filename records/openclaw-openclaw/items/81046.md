## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Reserve the new session field for plugin slots:** `src/config/sessions/types.ts:222`
  - body: Adding `exhaustedModels` to `SessionEntry` without also classifying it in `SESSION_ENTRY_RESERVED_SLOT_KEY_LIST` leaves the compile-time slot-key assertion failing as `Type '"exhaustedModels"' does not satisfy the constraint 'never'`. This is already breaking plugin SDK DTS/build/check lanes, so the new field needs the matching reserved-slot update.
  - confidence: 0.96
- **[P2] Pass the real session store identity into persistence:** `src/agents/model-fallback.ts:235`
  - body: The resolver derives `agentId` from `path.basename(agentDir)`, but normal resolved agent dirs end in `/agent`, and the production `agent-command` caller still does not pass `sessionId` or `sessionAgentId`. Real runs will either disable this persistence path or read/write the wrong session store; thread the active session id plus real session agent id/key from the caller instead.
  - confidence: 0.94
- **[P2] Clear cooldown state when a session rolls over:** `src/agents/model-fallback.ts:270-280`
  - body: The PR persists `exhaustedModels` on the session entry, but existing reset/rollover code copies the previous entry while replacing `sessionId`. That can carry active cooldowns into a fresh session and skip models after reset, so bind the map to the session generation or explicitly drop it in reset and related model-switch cleanup paths.
  - confidence: 0.86
- **[P3] Document the cooldown config key in schema hints:** `src/config/zod-schema.agent-defaults.ts:58`
  - body: The strict schema now accepts `agents.defaults.modelExhaustionRetryMinutes`, but the config help/label tables have no matching entry. Generated schema hints will expose an undocumented knob unless the new field gets a label and help text with default/scope.
  - confidence: 0.78
