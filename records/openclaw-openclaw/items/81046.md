## Summary

Keep open: the feature direction is plausible, but this PR is not merge-ready because the latest head still misses real production session identity wiring, can carry cooldowns across session resets, leaves the new config key out of generated schema hints, lacks real behavior proof, and overlaps another open implementation.

## What This Changes

The PR adds session-store backed exhausted-model cooldown persistence to model fallback, a new `agents.defaults.modelExhaustionRetryMinutes` config field, `SessionEntry.exhaustedModels`, and focused tests.

## Best Possible Solution

Choose one canonical implementation that threads the real `sessionId` plus session agent/store identity through fallback, clears cooldown state on session reset or rollover, documents the config surface, keeps focused regression coverage, and includes redacted live proof.

## Reproduction Assessment

Yes for the review blockers. Source inspection shows the production agent fallback call does not pass `sessionId`, and the PR derives the session-store agent id from an agent directory basename that is normally `agent`.

## Solution Assessment

No. Session-scoped cooldown persistence may be the right direction, but this branch needs correct production session identity, reset-aware cleanup, schema help/label alignment, reconciliation with the overlapping PR, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Thread real session identity into cooldown persistence:** `src/agents/model-fallback.ts:939-943`
  - body: The new session cooldown resolver only works when `sessionId` is present and it derives the store agent id from `path.basename(agentDir)`. The main production caller does not pass `sessionId`, and normal agent dirs end in `/agent`, so real agent runs will skip persistence or read the wrong store; pass the real session id and session agent/store identity from the caller instead.
  - confidence: 0.94
- **[P2] Clear cooldowns when the session rolls over:** `src/config/sessions/types.ts:222`
  - body: Adding `exhaustedModels` to `SessionEntry` makes it durable, but existing reset/update flow spreads the previous entry while replacing `sessionId`. A reset or idle rollover can therefore carry active cooldown markers into the fresh session; drop this map on rollover or bind it to the session generation.
  - confidence: 0.86
- **[P3] Document the cooldown config key in schema hints:** `src/config/zod-schema.agent-defaults.ts:58`
  - body: The strict schema accepts `agents.defaults.modelExhaustionRetryMinutes`, but the generated config hint metadata is not updated. Add matching `FIELD_HELP` and `FIELD_LABELS` entries so diagnostics and config help expose the new knob consistently.
  - confidence: 0.78
