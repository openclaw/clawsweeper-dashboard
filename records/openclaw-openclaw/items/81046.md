## Summary

Keep open: the branch now includes production code, but it still has blocking config-contract and session-write issues, and the external PR lacks required real behavior proof.

## What This Changes

The branch adds session-store backed exhausted-model cooldowns to `runWithModelFallback`, a new `agents.defaults.modelExhaustionRetryMinutes` type field, a `SessionEntry.exhaustedModels` field, and focused Vitest coverage.

## Best Possible Solution

Consolidate on one implementation that wires session model cooldowns through the validated config schema/help/docs, merges session writes safely, keeps focused regression coverage, and includes redacted real runtime proof.

## Reproduction Assessment

Yes. Source inspection is enough for the PR defects: the strict agent-defaults schema lacks the new config key, and the session-store update callback ignores the fresh locked entry it receives.

## Solution Assessment

No. The direction is plausible, but this branch should not merge until the config contract and locked session-write issues are fixed and the contributor adds real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Wire the cooldown knob into the config schema:** `src/config/types.agent-defaults.ts:213`
  - body: This adds `agents.defaults.modelExhaustionRetryMinutes` as a typed config field and `runWithModelFallback` reads it via `any`, but `AgentDefaultsSchema` is strict and still has no matching property/help/label. A user setting the advertised knob will be rejected by normal config validation, so the configurable window cannot be used through the supported config path.
  - confidence: 0.9
- **[P2] Merge cooldown writes from the locked session entry:** `src/agents/model-fallback.ts:273-280`
  - body: `updateSessionStoreEntry` gives the callback the latest locked session entry, but this callback ignores it and writes from the stale `params.state.exhaustedModels` snapshot captured before the lock. Concurrent lanes that mark or clear different exhausted models can overwrite each other's entries; build the next map from the callback's `entry.exhaustedModels` and then update the in-memory snapshot from the persisted result.
  - confidence: 0.84
