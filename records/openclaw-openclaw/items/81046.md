## Summary

Keep open: current main does not include persisted session-scoped model exhaustion cooldowns, and the PR head still has a production auto-reply session-store routing bug plus no real behavior proof. The branch also overlaps https://github.com/openclaw/openclaw/pull/81054, so maintainers should choose one canonical cooldown implementation rather than merge both independently.

## What This Changes

The PR adds session-store persisted exhausted-model cooldowns, a new `agents.defaults.modelExhaustionRetryMinutes` config field, reset clearing, schema labels/help, and focused tests.

## Best Possible Solution

Adopt one canonical implementation that threads explicit session agent identity through CLI, auto-reply, and memory fallback callers, preserves configured `agentDir` support, coordinates the overlapping PR, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows PR-head auto-reply fallback callers provide `sessionId` and `agentDir` but not `sessionAgentId`, while the new resolver falls back to deriving store identity from `agentDir`. That is enough to prove configured arbitrary agent directories can route cooldown reads/writes to the wrong store.

## Solution Assessment

No. Session-scoped cooldown persistence is plausible, but this patch should thread `run.agentId` or `sessionAgentId` through the shared auto-reply fallback options instead of inferring store identity from a path, and it should be reconciled with the overlapping implementation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Thread session agent ids through auto-reply fallback:** `src/agents/model-fallback.ts:236-238`
  - body: The new resolver falls back to deriving the store agent id from `agentDir` when `sessionAgentId` is absent, but auto-reply and memory flush still use `resolveModelFallbackOptions()`, which passes only `agentDir`. For configured `agentDir` values that are not `<state>/agents/<id>/agent`, those real sessions can read or write the wrong session store and lose the cooldown persistence this PR is adding. Pass the active run agent id through the shared helper instead of relying on path inference.
