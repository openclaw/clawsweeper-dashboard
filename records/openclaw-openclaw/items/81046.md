## Summary

Keep open: the latest head fixed the earlier reset-clearing and schema-help gaps, but the cooldown persistence is still not wired through the main agent path and the external PR still lacks real behavior proof. The same author also has an overlapping open implementation at https://github.com/openclaw/openclaw/pull/81054, so maintainers should choose one canonical path rather than merge both.

## What This Changes

The PR adds session-store persisted model exhaustion cooldowns, a new `agents.defaults.modelExhaustionRetryMinutes` config field, `SessionEntry.exhaustedModels`, reset clearing, schema help/labels, and focused Vitest coverage.

## Best Possible Solution

Adopt one canonical session-cooldown implementation that passes explicit `sessionId` plus `sessionAgentId` or `sessionKey` through fallback callers, preserves arbitrary configured `agentDir`, clears cooldowns on reset, keeps focused regression coverage, and includes redacted live proof.

## Reproduction Assessment

Yes for the review finding: source inspection is enough because current `agent-command` does not pass `sessionId` to `runWithModelFallback`, while the PR resolver only persists when `params.sessionId` is present. The configured-agent-dir path also shows why deriving store identity from `agentDir` is not reliable.

## Solution Assessment

No. Session-scoped cooldown persistence is plausible, but this PR should thread explicit production session identity and reconcile https://github.com/openclaw/openclaw/pull/81054 before merge.
