Keep open: current main does not already implement session-backed exhausted-model cooldowns, but this branch is not mergeable because the config contract is incomplete, locked session writes can clobber concurrent cooldown state, agent-store resolution is not reliable, and real behavior proof is still absent.

## Best Possible Solution

Choose one canonical implementation that persists session-scoped model cooldowns through the validated config/schema/help/docs, resolves the session store from real agent/session identity, performs locked session updates from the fresh entry, and includes focused regression plus redacted real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass a real agent id into the session lookup:** `src/agents/model-fallback.ts:235`
  - body: `resolveStoredSessionKeyForSessionId` expects an agent id, but this passes `path.basename(agentDir)`. Normal `resolveAgentDir` paths end in `/agent`, so non-default or custom agent stores resolve under the wrong id and the cooldown marker will not be found or persisted for the intended session.
  - confidence: 0.88
- **[P2] Wire the cooldown knob into the config schema:** `src/config/types.agent-defaults.ts:213`
  - body: This adds `agents.defaults.modelExhaustionRetryMinutes` to the TypeScript type and reads it via `any`, but `AgentDefaultsSchema` is strict and still lacks the key. Users setting the advertised knob through normal config validation will get an unknown-field rejection instead of a configurable cooldown window.
  - confidence: 0.9
- **[P2] Merge cooldown writes from the locked session entry:** `src/agents/model-fallback.ts:273-280`
  - body: `updateSessionStoreEntry` provides the latest locked `SessionEntry`, but this callback ignores it and writes from the snapshot captured before the lock. Concurrent attempts that mark or clear different exhausted models can overwrite each other; build the next map from the callback entry and then update the in-memory state from the persisted result.
  - confidence: 0.85
