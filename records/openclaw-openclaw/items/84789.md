## Summary

Close as implemented: the reported OpenClaw 2026.5.2 Active Memory Telegram forum-topic dirName failure matches the root cause fixed by https://github.com/openclaw/openclaw/pull/76719, and both current main and v2026.5.19 contain the narrower caller-boundary fix plus regression coverage.

## What This Changes

The issue asks for Active Memory to stop crashing when Telegram forum-topic session IDs containing colons reach plugin dirName validation.

## Best Possible Solution

Keep the shipped Active Memory boundary fix: treat scoped Telegram topic IDs as conversation metadata and run embedded recall through the Telegram provider while leaving shared plugin dirName validation strict.

## Reproduction Assessment

For the originally reported OpenClaw 2026.5.2 bug, yes: the issue logs and older matching report show the scoped Telegram topic ID reaching dirName validation. Against current main, no failing reproduction remains from source inspection because the Active Memory embedded run now uses the Telegram provider instead of the scoped topic ID, with regression coverage.

## Solution Assessment

Yes. The current implementation fixes the Active Memory caller boundary and preserves strict plugin directory validation, which is narrower and safer than sanitizing colon-containing values inside the shared loader.
