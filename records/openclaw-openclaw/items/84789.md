## Label Justifications

- `P2`: The report describes a real but scoped Active Memory failure for Telegram forum-topic sessions rather than a whole-runtime outage.
- `impact:session-state`: The failure is caused by topic-scoped session/channel identity being misused as a runnable plugin directory name.

## Summary

Current main already fixes this exact Active Memory Telegram forum-topic dirName failure, and the fix is included in the shipped v2026.5.19 release; the report is against OpenClaw 2026.5.2.

## What This Changes

The issue reports that active memory crashes for Telegram forum topic sessions because the topic session key segment with colons reaches bundled plugin directory-name validation.

## Best Possible Solution

Keep the shipped Active Memory channel-resolution fix and ask for a fresh report only if the same failure reproduces on v2026.5.19 or newer with current logs.

## Reproduction Assessment

Yes for the original 2026.5.2 failure path: the prior canonical report and PR reproduced the same hook context, and current main now has a focused regression test for it. The failing behavior is not expected to reproduce on current main because the test asserts the fixed `messageChannel: "telegram"` behavior.

## Solution Assessment

Yes. The implemented fix keeps bundled plugin path validation strict and changes Active Memory to pass a runnable provider/channel id to the embedded recall run while preserving the topic id as session context.
