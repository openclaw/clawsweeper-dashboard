## Label Justifications

- `P2`: The bug blocked Active Memory recall in Telegram forum topics but did not make the whole runtime unusable.
- `impact:session-state`: The reported failure prevents Active Memory from injecting expected session memory/context for topic-scoped Telegram sessions.

## Summary

Close as implemented: the reported 2026.5.2 Active Memory Telegram forum-topic dirName crash is the same path fixed by https://github.com/openclaw/openclaw/pull/76719, and current main plus shipped release provenance show topic conversation IDs are no longer used as bundled plugin directory names.

## What This Changes

The issue asks for Active Memory to avoid passing colon-bearing Telegram forum topic session identifiers into plugin directory-name resolution.

## Best Possible Solution

Keep the shipped Active Memory channel-resolution fix and ask affected 2026.5.2 users to upgrade to v2026.5.3 or newer; no further core change is needed unless a fresh current-stable repro appears.

## Reproduction Assessment

Yes for the affected old release: the prior report and merged regression show a before_prompt_build call with a Telegram topic session key plus raw topic channelId reached bundled-plugin dirName validation. Current main no longer has that failing path.

## Solution Assessment

Yes. The shipped fix preserves topic-qualified session identity while passing only a runnable provider/channel id such as 'telegram' into embedded recall plugin loading, which is narrower than globally sanitizing session keys.
