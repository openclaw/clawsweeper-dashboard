## Label Justifications

- `P2`: The bug affects active-memory recall for a specific Telegram forum-topic workflow, not the whole runtime.
- `impact:session-state`: The failure is caused by topic-qualified session/conversation identifiers crossing into a filesystem-backed plugin runtime path.

## Summary

Current main and the latest release already handle this Telegram forum-topic active-memory failure by preventing scoped conversation ids from being used as runnable bundled-plugin directory names.

## What This Changes

Active memory should support Telegram forum topic sessions whose session or conversation identifiers contain `:topic:` segments.

## Best Possible Solution

Keep the shipped resolver guard and have affected users update to v2026.5.19 or newer; reopen with fresh logs only if the same failure persists after upgrade.

## Reproduction Assessment

Yes. The source and regression test show the reported Telegram topic session and scoped `channelId` path, although I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Yes. Keeping bundled-plugin dirName validation strict while making active memory pass the runnable provider/channel name is the narrow, maintainable boundary fix.
