## Summary

Keep open. Current main still runs Sharp-backed image operations in-process, while this draft PR proposes a new default worker-process boundary; the direction is plausible, but it needs maintainer review for the default runtime change and a changelog entry before merge.

## What This Changes

The PR routes non-sips Sharp-backed image metadata, resize, conversion, and alpha checks through a spawned image worker, adds a packaged worker entrypoint, and extends media input-guard tests for worker routing and timeouts.

## Best Possible Solution

Land a maintainer-reviewed version that preserves the sips path, ships the worker entrypoint, documents or release-notes the default worker behavior and env knobs, and validates both packaged and source-mode worker execution.

## Reproduction Assessment

Not applicable; this PR is a runtime-hardening change rather than a reported bug reproduction. Source inspection does confirm current main uses in-process media image ops and the PR routes that path through a worker.

## Solution Assessment

Unclear; process isolation is a plausible way to contain Sharp/native failures, but making it default-on with new env controls is a runtime policy change that should be accepted by maintainers after validation.
