## Summary

Keep open. Current main and the latest release reduce the original 1s launchd respawn throttle to 10s, but missing config still logs the guard error and exits 78 under an unconditional `KeepAlive=true` LaunchAgent, so the remaining config-error retry loop is still source-reproducible.

## What This Changes

Requests bounded macOS launchd retry behavior for the gateway when startup exits on missing config and produces repeated error logs.

## Best Possible Solution

Add macOS launchd-aware handling for config-error exit 78 that stops, backs off, or waits for missing config while preserving intentional clean-exit restart/update relaunches and normal crash recovery.

## Reproduction Assessment

Yes, source-backed. Current main renders `KeepAlive=true` with `ThrottleInterval=10`, and the missing-config startup guard logs the reported message before exiting 78, with no launchd equivalent to systemd's `RestartPreventExitStatus=78`.

## Solution Assessment

No current-main fix exists. A focused launchd config-error policy is the maintainable direction; the related `SuccessfulExit=false` PR is not sufficient because config errors are nonzero exits and clean-exit relaunches still matter.
