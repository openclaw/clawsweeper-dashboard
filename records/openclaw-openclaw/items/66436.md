## Summary

Keep open. Current main reduced the launchd retry interval from the shipped 1s value to 10s, but missing config still logs the reported guard message and exits 78 under an unconditional `KeepAlive=true` LaunchAgent, so macOS can still retry forever; the open related PR does not solve this config-error loop and has separate blockers.

## What This Changes

The issue requests bounding or avoiding launchd respawn loops when missing config makes the gateway exit under `KeepAlive` and `ThrottleInterval`.

## Best Possible Solution

Add a macOS launchd-aware config-error policy that stops, backs off, or waits on exit 78 for missing config while preserving intentional restart/update relaunches and normal crash recovery.

## Reproduction Assessment

Yes, source-backed rather than live: current main still generates `KeepAlive=true` with `ThrottleInterval=10`, and missing config exits 78 after logging the guard message. The latest shipped release still has the original 1s throttle.

## Solution Assessment

No current-main fix exists. The simple `SuccessfulExit=false` direction in the open related PR is not sufficient for this issue because config errors are nonzero exits and clean-exit relaunches still need to work.
