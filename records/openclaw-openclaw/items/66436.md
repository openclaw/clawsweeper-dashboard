## Summary

Keep open. Current main has reduced the LaunchAgent throttle from the reported 1s to 10s, but the missing-config path still logs the guard error and exits under unconditional launchd `KeepAlive`, so macOS can still retry indefinitely; the closed auto-update issue explicitly left this missing-config retry policy as separate work.

## What This Changes

Prevent macOS LaunchAgent missing-config startup failures from repeatedly respawning and growing gateway error logs.

## Best Possible Solution

Add a macOS launchd-aware config-error policy that stops or backs off repeated missing-config retries while preserving normal crash recovery and intentional fast restarts.

## Reproduction Assessment

Yes, source-backed but not live-reproduced on macOS: missing config still produces the guard error and exits 78, while the generated LaunchAgent still has `KeepAlive=true`. Current main changes the retry interval to 10s, so the exact 1s storm is reduced but the repeated retry path remains.

## Solution Assessment

Unclear until maintainers choose the launchd policy. Raising the throttle helped the adjacent auto-update issue, but the missing-config case likely needs stop/backoff/wait-for-config behavior rather than only another interval tweak.
