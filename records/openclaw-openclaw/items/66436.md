## Summary

Keep open. Current main still emits a macOS LaunchAgent with `KeepAlive=true` and `ThrottleInterval=1`, and the missing-config startup guard still exits with sysexits 78 before the gateway loop can apply any in-process restart delay. The related #60885 remains adjacent lifecycle work but does not by itself close this missing-config log-storm case.

## What This Changes

The issue requests bounded macOS LaunchAgent retry behavior, backoff, or wait-for-config handling so missing gateway config does not respawn every second and flood logs.

## Best Possible Solution

Add macOS-specific service/startup behavior that prevents repeated missing-config or config-error respawns while preserving normal crash recovery and intentional restarts. The end state should include regression coverage for the generated LaunchAgent and startup guard path, plus troubleshooting docs that tell users how to repair a stopped or unconfigured gateway.

## Reproduction Assessment

Yes. The source path is high-confidence: no config yields the missing-config guard, `runGatewayCommand()` exits with code 78 before `runGatewayLoop()`, and the generated LaunchAgent still has `KeepAlive=true` with a one-second throttle.

## Solution Assessment

Unclear until a patch exists. The narrowest maintainable direction is likely a macOS launchd-aware config-error suppression/backoff path rather than only raising `ThrottleInterval`, because normal crash recovery and intentional restarts should stay fast.
