## Summary

Keep open. Current main reduced the reported one-second launchd retry to 10 seconds, but missing config still emits the guard error and exits 78 under a generated LaunchAgent with unconditional KeepAlive, so macOS can still retry indefinitely; the open related PR does not directly fix this config-error loop and has separate review blockers.

## What This Changes

Request to prevent macOS launchd from repeatedly restarting the gateway and flooding logs when startup fails because the OpenClaw config is missing or invalid.

## Best Possible Solution

Add a macOS launchd-aware config-error policy that stops, backs off, or waits on exit 78 for missing config while preserving intentional restart/update relaunches and normal crash recovery.

## Reproduction Assessment

Yes, source-backed but not live-reproduced on macOS: current main still generates KeepAlive=true with ThrottleInterval=10, and missing config still logs the guard message then exits 78. The original one-second interval is fixed, but the repeated launchd retry path remains.

## Solution Assessment

No current-main fix exists. A simple global SuccessfulExit=false switch is not the best solution for this specific report because config errors are nonzero exits and intentional clean-exit relaunches also need to keep working.
