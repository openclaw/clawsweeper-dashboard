## Summary

Keep open. Current main still has the reported macOS launchd retry/log-storm path: the LaunchAgent renders `KeepAlive=true` with `ThrottleInterval=1`, and the missing-config startup guard exits with sysexits 78 before any gateway loop can apply backoff or recovery behavior. The adjacent #60885 lifecycle report is related but tracks auto-update downtime and restart mechanics, not this missing-config respawn loop.

## What This Changes

Prevent macOS LaunchAgent missing-config startup failures from respawning every 1-2 seconds and filling gateway error logs.

## Best Possible Solution

Add macOS-specific service/startup handling that prevents repeated configuration-error respawns while preserving normal crash recovery and fast intentional restarts, with regression coverage and troubleshooting docs.

## Reproduction Assessment

Yes. The source path is high-confidence: a missing config produces the guard error and exits 78, while the generated LaunchAgent still has `KeepAlive=true` and a one-second throttle.

## Solution Assessment

Unclear until a patch exists. The narrowest maintainable direction is likely launchd-aware config-error suppression, backoff, or wait-for-config behavior rather than only raising `ThrottleInterval`, because the one-second interval is used for intentional fast restarts.
