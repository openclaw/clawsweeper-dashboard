## Summary

Current main has reduced the original 1 second launchd throttle and added shutdown timeout, but the missing-config path still exits with sysexits config error under an unconditional launchd restart policy, so the repeated log growth problem remains source-reproducible and should stay open for a targeted macOS lifecycle fix.

## What This Changes

The issue requests launchd/gateway behavior that stops, backs off, or waits instead of repeatedly respawning and logging when the gateway starts with missing or invalid config.

## Best Possible Solution

Add a launchd-aware config-error policy that stops, backs off, or waits only for missing/invalid configuration while preserving intentional restart/update relaunches and ordinary crash recovery.

## Reproduction Assessment

Yes, source-backed. Current main still renders boolean launchd KeepAlive with a 10 second throttle, and the missing-config startup guard logs the reported message before exiting 78.

## Solution Assessment

No complete current-main solution exists. The maintainable fix is a focused launchd config-error policy rather than only increasing the throttle or using SuccessfulExit=false, which would not stop nonzero config-error restarts.
