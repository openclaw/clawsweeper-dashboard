## Summary

Keep open. Current main still generates a macOS LaunchAgent with `KeepAlive=true` and `ThrottleInterval=1`, while the missing-config startup guard exits with sysexits 78 before the gateway loop can apply any in-process retry behavior. The related #60885 lifecycle work is adjacent but does not close this missing-config log-storm path.

## What This Changes

Fix the macOS LaunchAgent missing-config crash loop where a one-second throttle repeatedly respawns the gateway and floods error logs.

## Best Possible Solution

Add macOS-specific service/startup behavior that prevents repeated configuration-error respawns while preserving normal crash recovery and fast intentional restarts. The end state should include regression coverage for the generated LaunchAgent and startup guard path, plus troubleshooting docs for repairing an unconfigured or stopped gateway.

## Reproduction Assessment

Yes. The source path is high-confidence: no config yields the missing-config guard, the CLI exits with sysexits 78, and the generated LaunchAgent still has `KeepAlive=true` with a one-second throttle.

## Solution Assessment

Unclear until a patch exists. The narrowest maintainable direction is likely a launchd-aware config-error suppression/backoff or wait-for-config path rather than only raising `ThrottleInterval`, because the one-second interval is currently used for intentional fast restarts.
