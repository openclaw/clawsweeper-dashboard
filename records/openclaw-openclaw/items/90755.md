## Summary

Keep open: current main and v2026.6.6 still expose the in-flight run snapshot through gateway/TUI paths, but the Control UI history loader does not adopt it after session switching clears local stream state. ## What This Changes

Ensure the Control UI restores or at least indicates an in-progress agent response when returning to a running chat session.

## Best Possible Solution

Adopt the existing chat.history inFlightRun contract in the Control UI, restore chatRunId/chatStream or a running indicator on history reload, and prove the switch-back path with focused UI tests plus real browser evidence.

Yes, at source level: chat.history can return inFlightRun and the TUI consumes it, but the Control UI history result and loader do not. I did not run a live browser reproduction in this read-only pass.
