## Summary

Keep open: current main and the latest release still expose `inFlightRun` from Gateway history and consume it in TUI, but Control UI history reload still does not adopt that payload. The current linked fix PR is open but unmerged and already reviewed as an unsafe/superseded branch, so this issue remains the canonical tracker.

## What This Changes

The issue reports that returning to a running Control UI chat session hides the agent's in-progress response, making the session look idle or stalled.

## Best Possible Solution

Keep this issue as the canonical bug until a merged fix safely adopts or refines the Gateway in-flight run contract for Control UI without duplicating cumulative streamed text.

## Reproduction Assessment

Yes. Source inspection shows Gateway history can return `inFlightRun` and TUI consumes it, while Control UI `loadChatHistory` ignores that field and can clear local stream state on history reload.

## Solution Assessment

Yes. The best fix is a narrow Control UI adoption or refinement of the existing Gateway/TUI `inFlightRun` contract with cumulative-text reconciliation; a broad reconnect subsystem is not needed for the central bug.
