## Summary

Keep open: current main and v2026.6.10 still expose memory prompt supplements as global builders with only tool/citation inputs, and fixing this requires a plugin SDK or memory-wiki config scoping decision rather than conservative cleanup.

## What This Changes

Request to scope memory-wiki compiled digest prompt supplements per agent or sessionKey, either through config include/exclude patterns or by passing agent/session identity to MemoryPromptSectionBuilder.

## Best Possible Solution

Choose one scoped prompt-supplement contract, keep existing builders compatible, then update memory-wiki config/docs and add regression coverage for included interactive sessions and excluded cron/session-key paths.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 build memory prompt supplements without agentId or sessionKey, and memory-wiki's digest builder can only see global config plus available tools. I did not run a live multi-session gateway in this read-only pass.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers still need to choose whether the narrowest durable fix is a config predicate, builder context, broader per-agent plugin config, or coordination with vault-isolation work.
