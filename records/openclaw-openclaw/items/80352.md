## Summary

Keep open: current main still exposes the memory-wiki compiled digest and additive memory prompt supplements as global prompt additions, with no agent/session scope. This is a valid feature/API gap rather than a strict bug, and the adjacent per-agent vault issue does not cover session-key prompt injection controls.

## What This Changes

Request scoped controls for memory-wiki compiled digest prompt injection and the `registerMemoryPromptSupplement` builder contract by agent, session, or session-key pattern.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context or generic memory-wiki context scope, with schema/manifest/docs updates and focused regression tests for deterministic include/exclude behavior.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high confidence that current main and `v2026.5.7` lack the requested scope because supplements receive only `availableTools` and `citationsMode`, while memory-wiki config has only the global digest boolean.

## Solution Assessment

Unclear: the issue presents plausible options, but choosing config-level scope versus builder params is a public plugin SDK and product contract decision. The maintainable path should be additive and preserve existing third-party prompt supplement builders.
