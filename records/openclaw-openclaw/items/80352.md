## Summary

Keep open: current main still has a global memory-wiki digest prompt toggle and a memory prompt supplement contract without agent or session context, and the follow-up comment confirms this is still useful to users. This is a feature/API gap rather than a broken existing contract.

## What This Changes

Request per-agent or per-sessionKey scoping for memory-wiki compiled digest injection and additive memory prompt supplements.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or memory-wiki context scope with deterministic matching, public SDK/docs updates, manifest/schema coverage, and focused regression tests.

Not applicable: this is a feature/API request, not a report of broken documented behavior. Source inspection on current main and v2026.5.7 gives high-confidence proof that the requested scoping capability is absent.

## Solution Assessment

Unclear: the issue proposes plausible directions, but choosing config-level scope versus builder-level session context is a public plugin SDK and product contract decision. The maintainable path should be additive and should not force third-party prompt supplements to break.
