## Summary

Keep open: current main and the latest release still expose memory prompt supplements as global prompt additions without agent or session-key context, and memory-wiki still has only a global compiled digest prompt toggle. The related memory-wiki vault isolation work is adjacent but does not supersede this prompt-supplement scoping request.

## What This Changes

Request scoped memory-wiki compiled digest prompt injection and generic memory prompt supplement filtering by agent, session key, or session-key pattern.

## Best Possible Solution

Add an additive, backward-compatible scoped prompt supplement contract, with deterministic matching semantics, memory-wiki config/schema/docs, SDK docs, and regression coverage for prompt-cache-stable session filtering.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/API request. Source inspection gives high confidence that current main lacks the requested scope because supplements receive only availableTools/citationsMode and memory-wiki exposes only a global digest boolean.

## Solution Assessment

Unclear: the requested direction is valid, but choosing config-level scope, builder params, or both changes a public plugin SDK and user-facing configuration contract. The maintainable path should be additive and deterministic rather than a memory-wiki-only workaround.
