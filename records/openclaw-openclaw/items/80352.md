## Summary

Keep open: current main and the latest release still provide only a global memory-wiki digest prompt toggle, and the memory prompt supplement builder has no agent or session context. This is a valid plugin SDK/config feature gap, not a strict bug or a duplicate of the broader additive-memory work.

## What This Changes

Request to scope memory-wiki compiled digest prompt supplements by agent or session key instead of injecting them into every non-minimal prompt build.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or memory-wiki context scope, with schema, manifest/docs updates, and focused tests for deterministic include/exclude behavior.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high confidence that current main and v2026.5.7 lack the requested scope because supplements receive only availableTools/citationsMode and memory-wiki config has only the global digest boolean.

## Solution Assessment

Unclear: the requested direction is valid, but maintainers need to choose between config-level scope, builder session context, or a combined additive SDK contract before implementation.
