## Summary

Keep open: current main and the latest release still expose the memory-wiki digest prompt as a global supplement with no agent/session scope, and a recent user comment confirms this remains useful. This is a feature/API gap that needs maintainer SDK/config direction rather than conservative cleanup.

## What This Changes

Request per-agent or per-sessionKey scoping for memory-wiki compiled digest prompt injection and generic memory prompt supplements.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or memory-wiki scope config, then update schema, manifest metadata, docs, and focused regression tests for deterministic include/exclude behavior.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection on current main and v2026.5.7 gives high-confidence proof that prompt supplements only receive availableTools/citationsMode and memory-wiki only has a global digest toggle.

## Solution Assessment

Unclear: the requested direction is valid, but choosing config-level scope versus builder params is a public SDK/product decision. The maintainable path is an additive contract that preserves third-party plugin compatibility and deterministic prompt output.
