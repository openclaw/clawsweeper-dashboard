## Summary

Keep open: current main and the latest release still expose only a global memory-wiki digest prompt toggle, and the memory prompt supplement contract has no agent or session context. This is a real feature/API gap, not a strict bug, because solving it needs a new public SDK/config scoping contract.

## What This Changes

Request scoped memory-wiki compiled digest prompt supplements by agent, session, or session-key patterns instead of the current global prompt supplement behavior.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or memory-wiki context scope, with schema, manifest, SDK docs, and focused tests for deterministic include/exclude matching.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection is high-confidence that current main lacks the requested scope because supplement builders receive only availableTools/citationsMode and memory-wiki config exposes only the global digest boolean.

## Solution Assessment

Unclear: the issue presents plausible options, but choosing config-level scope versus builder params is a public plugin SDK and product contract decision. The maintainable path should be additive and should not break third-party prompt supplement builders.
