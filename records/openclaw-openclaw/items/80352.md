## Summary

This should stay open: current main still exposes memory prompt supplements as a global prompt-building hook with no session or agent context, and memory-wiki still has only the global compiled digest toggle. This is a real plugin SDK/config feature gap, not a broken existing contract or a duplicate of the vault-path isolation work.

## What This Changes

Request per-agent, per-session, or session-key-pattern scoping for memory-wiki compiled digest prompt injection and the generic memory prompt supplement hook.

## Best Possible Solution

Add an additive session-aware memory prompt context and/or memory-wiki context scope contract, with schema, manifest, SDK docs, and focused tests for deterministic include/exclude behavior.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection is high-confidence that current main lacks the requested scope because prompt supplements still receive only tool/citation context and memory-wiki config still has only the global digest boolean.

## Solution Assessment

Unclear: the requested direction is valid, but maintainers need to choose whether scoping belongs in generic plugin config, SDK builder params, or both. The maintainable path should be backward-compatible and avoid forcing every plugin to reinvent session-key matching.
