## Summary

Keep open: current main and the latest release still make the memory-wiki compiled digest prompt and generic memory prompt supplements global, with no agent or session-key context for filtering. The linked and searched related work covers nearby memory isolation and hook topics, but no merged or open canonical item appears to cover this exact prompt-supplement scoping contract.

## What This Changes

Request per-agent or per-session-key scoping for memory-wiki compiled digest prompt injection and generic memory prompt supplements so selected sessions can opt out.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or memory-wiki context scope, with schema, manifest, SDK docs, deterministic matching tests, and prompt-cache-aware behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested scope: supplements receive only availableTools/citationsMode and memory-wiki has only the global digest boolean.

## Solution Assessment

Unclear: the issue proposes plausible options, but choosing config-level scope versus builder params is a public plugin SDK and product-contract decision. The safest implementation direction is additive, backward-compatible, and deterministic.
