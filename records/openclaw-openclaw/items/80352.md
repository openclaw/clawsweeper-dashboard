## Summary

Keep open: current main and v2026.5.7 still expose only a global memory-wiki digest prompt toggle, and memory prompt supplements receive no agent or session context. This is a real feature/API gap that needs maintainer design rather than a strict bug fix.

## What This Changes

Request adding agent/session-key scoping to the memory-wiki compiled digest prompt and the generic memory prompt supplement contract.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and, if maintainers want operator-level control, a memory-wiki context scope schema with docs and focused regression tests for deterministic include/exclude matching.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high confidence that current main and v2026.5.7 lack the requested scope because supplements receive only availableTools/citationsMode and memory-wiki exposes only the global digest boolean.

## Solution Assessment

Unclear: the issue presents plausible options, but choosing config-level scope versus builder params is a public plugin SDK and product contract decision. The maintainable path should be additive and preserve existing third-party MemoryPromptSectionBuilder implementations.
