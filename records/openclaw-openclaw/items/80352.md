## Summary

Keep open: current main still exposes memory prompt supplements as a global prompt path with no agent or session-key scope, so this remains a real feature/API gap rather than a cleanup-close candidate.

## What This Changes

Request per-agent or per-session-key scoping for memory-wiki compiled digest prompt injection and generic memory prompt supplements.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or generic memory-wiki context scope, then align config schema, manifest metadata, SDK docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection is high-confidence that current main and `v2026.5.7` lack the requested scope because supplements receive only `availableTools` and `citationsMode`, while memory-wiki config has only the global digest boolean.

## Solution Assessment

Unclear: the requested direction is valid, but choosing config-level scope versus builder params is a public plugin SDK and product contract decision. The maintainable path should be additive and should not break third-party prompt supplement builders.
