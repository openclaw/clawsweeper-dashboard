## Summary

Keep open: current main and the latest release still make memory prompt supplements global, with no agent/session-key context and no memory-wiki config scope for the compiled digest prompt. The related memory-wiki vault isolation and generic per-agent plugin config issues overlap, but neither supersedes this specific prompt-supplement scoping contract.

## What This Changes

The issue requests agent or session-key scoping for memory-wiki compiled digest prompt injection and generic memory prompt supplements.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or memory-wiki context scope, with schema, manifest, SDK docs, deterministic matching tests, and prompt-cache-aware behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested scope: supplements receive only availableTools/citationsMode and memory-wiki has only the global digest boolean.

## Solution Assessment

Unclear: the suggested options are plausible, but maintainers need to choose config-level scope versus builder context because this changes a public plugin SDK/config contract. The safest direction is additive, backward-compatible, deterministic, and prompt-cache-aware.
