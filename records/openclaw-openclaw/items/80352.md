## Summary

Keep open: current main still exposes memory prompt supplements as global, and memory-wiki only has a single digest boolean. This is a valid feature/API gap, with recent user confirmation, and needs maintainer direction on the public SDK/config contract before implementation.

## What This Changes

Request per-agent or per-sessionKey scoping for memory-wiki compiled digest prompts and any registered memory prompt supplement.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and, if approved, a memory-wiki scope schema with manifest, SDK docs, and focused regression coverage for deterministic include/exclude matching.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection confirms current main has no session or agent field in the prompt-supplement builder and no memory-wiki scope config.

## Solution Assessment

Unclear: the issue proposes plausible designs, but choosing config-level scope versus builder context is a public plugin SDK/config decision. A backward-compatible additive contract is safer than a plugin-local workaround.
