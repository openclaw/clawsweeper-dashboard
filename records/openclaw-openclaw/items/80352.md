## Summary

Keep open: current main and the latest release still expose the memory-wiki digest prompt as a global supplement with no agent, session, or session-key scope. The request is a valid feature/API gap with a small data-isolation angle, but it needs maintainer direction because the fix changes public plugin SDK/config behavior.

## What This Changes

Request to scope memory-wiki digest prompt supplements per agent, session, or session-key pattern instead of using one global include toggle.

## Best Possible Solution

Add an additive, backward-compatible memory prompt scope contract, update memory-wiki config/schema/manifest/docs, and add focused tests for deterministic agent/session include and exclude behavior while keeping this distinct from the open vault-isolation work.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high confidence that current main and v2026.5.7 lack the requested scope because prompt supplements receive only availableTools/citationsMode and memory-wiki exposes only the global digest boolean.

## Solution Assessment

Unclear: the issue offers plausible directions, but choosing config-level scope versus builder context is a public plugin SDK/product decision. The safest direction is an additive contract that lets existing third-party plugins keep working while enabling session-aware filtering.
