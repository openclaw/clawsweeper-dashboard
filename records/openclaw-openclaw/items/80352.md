## Summary

Keep open: current main and v2026.5.12 still have only a global memory-wiki digest toggle and a memory prompt supplement builder with no agent/session context; adjacent issues and PRs cover vault or generic plugin config work but do not supersede this prompt-scoping API request.

## What This Changes

Request to add per-agent or per-session scoping for memory-wiki compiled digest prompt injection and generic memory prompt supplements.

## Best Possible Solution

Add a backward-compatible, documented memory prompt scope contract that threads optional session/agent context and/or memory-wiki include/exclude config through SDK types, schema, manifest metadata, docs, and focused deterministic tests.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high confidence that current main cannot express the requested behavior because prompt supplements receive no agent/session context and memory-wiki config has only the global digest toggle.

## Solution Assessment

Unclear: both suggested directions are plausible, but maintainers need to choose the public contract before implementation. The narrowest durable path is likely additive session-aware builder context with memory-wiki config sugar for common include/exclude patterns.
