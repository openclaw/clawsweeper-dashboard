## Summary

Keep open: current main and the latest release still expose the memory-wiki digest prompt and memory prompt supplements as global prompt additions, with no agent/session scope. This is a valid feature/API gap, and the later +1 comment confirms the use case is still live.

## What This Changes

Request per-agent or per-sessionKey scoping for memory-wiki compiled digest prompt supplements through config patterns or session-aware prompt builder params.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or memory-wiki context scope, with schema, docs, SDK exports, and deterministic include/exclude matching tests.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection is high-confidence that current main cannot scope memory prompt supplements by agent or session because the builder contract and call path only carry availableTools/citationsMode.

## Solution Assessment

Unclear: the issue proposes plausible solutions, but choosing config-level scope versus builder context is a public plugin SDK and product contract decision. The safest direction is an additive, backwards-compatible contract with tests and docs.
