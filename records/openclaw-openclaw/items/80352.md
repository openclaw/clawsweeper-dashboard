## Summary

Keep open: current main still has only a global memory-wiki digest prompt toggle, and the prompt supplement SDK contract still lacks agent/session context. This is a real feature/API gap with fresh user confirmation, not stale cleanup.

## What This Changes

Request per-agent or per-sessionKey scoping for memory-wiki digest prompt injection and generic memory prompt supplements.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or generic memory-wiki context scope, with schema, manifest, docs, and focused deterministic include/exclude tests.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection on current main and `v2026.5.7` gives high confidence the requested scope is absent because prompt supplements receive only `availableTools`/`citationsMode` and memory-wiki config exposes only the global digest boolean.

## Solution Assessment

Unclear: the issue proposes plausible approaches, but choosing config-level scope versus builder params is a public plugin SDK/product contract decision. The maintainable path should be additive and avoid breaking third-party prompt supplement builders.
