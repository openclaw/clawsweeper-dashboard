## Summary

Keep open: current main and the latest release still expose only a global memory-wiki digest prompt toggle, and the memory prompt supplement contract has no agent or session context. This is a real feature/API gap, not a strict bug, because it asks for a new plugin config or SDK scoping contract.

## What This Changes

Request adding per-agent or per-session-key scoping for memory prompt supplements so memory-wiki compiled digest injection can be limited to selected sessions.

## Best Possible Solution

Add a backward-compatible session-aware memory prompt context and/or generic memory-wiki context scope, with schema, manifest, SDK docs, and focused regression tests for deterministic include/exclude matching.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection is high-confidence that the current main and `v2026.5.7` contracts lack the requested scope because prompt supplements receive only `availableTools` and `citationsMode`, while memory-wiki config has only the global digest boolean.

## Solution Assessment

Unclear: the issue presents plausible options, but choosing config-level scope versus builder params is a public plugin SDK and product contract decision. The maintainable path is an additive design that threads session context without breaking third-party plugins.
