## Summary

Keep open: current main and v2026.5.12 still expose memory prompt supplements as global prompt additions with no agent, sessionKey, or session-pattern scope. The related memory-wiki vault isolation work is useful context, but it does not supersede this exact prompt-supplement scoping contract.

## What This Changes

Request per-agent or per-sessionKey scoping for memory-wiki compiled digest prompts and generic memory prompt supplements.

## Best Possible Solution

Add an additive, backward-compatible session-aware memory prompt context and/or memory-wiki scope config with schema, SDK docs, deterministic matching tests, and prompt-cache-aware behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested scope: supplements receive only availableTools/citationsMode and memory-wiki has only the global digest boolean.

## Solution Assessment

Unclear: the proposed directions are plausible, but choosing config-level scope versus builder params is a public plugin SDK and product-contract decision. The safest implementation direction is additive and backward-compatible.
