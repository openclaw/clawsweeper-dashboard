## What This Changes

The issue requests a Discord slash-command comparator fix so CJK and multi-line descriptions that Discord normalizes do not trigger redundant reconcile PATCH calls and startup 429s.

## Best Possible Solution

Review PR #76588 and land a comparator-only normalization fix with regression coverage, preserving authored descriptions on writes and keeping the deploy hash cache as a separate mitigation.
