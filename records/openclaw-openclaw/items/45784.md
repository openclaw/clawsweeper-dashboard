## Summary

Keep open: the reported path bug is source-reproducible on current main, but this PR’s patch is too broad, incomplete, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds an `OPENCLAW_HOME` branch to `resolveConfigDir()` so any nonblank explicit home is returned directly as the config directory.

## Best Possible Solution

Land a narrow basename/state-dir guard in both `resolveStateDir()` and `resolveConfigDir()` with regression coverage for `.openclaw`, legacy state dirs, tilde expansion, and parent-home negative cases.

## Reproduction Assessment

Yes, at source level. `OPENCLAW_HOME=/home/user/.openclaw` is preserved by the home resolver, then current main’s state and config helpers append another `.openclaw` segment.

## Solution Assessment

No. Returning every explicit `OPENCLAW_HOME` directly changes the documented parent-home contract, and changing only `resolveConfigDir()` leaves `resolveStateDir()` unresolved.
