## Summary

Keep open. The underlying bug is valid, but this patch is not mergeable as-is because it broadens the documented `OPENCLAW_HOME` parent-directory contract, leaves `resolveStateDir()` nesting unresolved, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds a `resolveConfigDir()` branch that returns the resolved `OPENCLAW_HOME` directly whenever it is set.

## Best Possible Solution

Use a narrow state-directory basename guard in both `resolveStateDir()` and `resolveConfigDir()`, with regression coverage for `OPENCLAW_HOME=~/.openclaw` and preservation of `/srv/app -> /srv/app/.openclaw`.

## Reproduction Assessment

Yes, at source level. With `OPENCLAW_HOME=~/.openclaw`, current main resolves that as the effective home and then both state/config helpers append another `.openclaw`.

## Solution Assessment

No. Returning every explicit `OPENCLAW_HOME` directly is broader than the docs and tests allow, and changing only `resolveConfigDir()` leaves the state-dir path that drives canonical config placement unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve .openclaw for parent OPENCLAW_HOME values:** `src/utils.ts:296-307`
  - body: This branch returns `OPENCLAW_HOME` itself for every explicit home override, so a setup like `OPENCLAW_HOME=/srv/openclaw-home` stops using `/srv/openclaw-home/.openclaw` for config-derived roots. That conflicts with current tests, docs, and `resolveStateDir()` semantics; the guard should only skip the suffix when the resolved basename is already a state directory.
  - confidence: 0.94
- **[P2] Fix state-dir resolution with the config-dir change:** `src/utils.ts:296-297`
  - body: The reported onboarding/config path still flows through `resolveStateDir()` and `resolveCanonicalConfigPath()`, but this PR only changes `resolveConfigDir()`. Add the same narrow guard in the state-dir helper with regression coverage, otherwise state-derived config paths can still be nested.
  - confidence: 0.91
