## Summary

Keep open: the underlying nested `OPENCLAW_HOME` bug is still source-reproducible on current main, but this PR is not safe to merge because it broadens the parent-home contract and only changes the config helper.

## What This Changes

The PR changes `src/utils.ts` so `resolveConfigDir()` returns any explicit `OPENCLAW_HOME` directly instead of appending `.openclaw`.

## Best Possible Solution

Land a narrow basename/state-directory guard in both `resolveStateDir()` and `resolveConfigDir()`, with regression coverage for `.openclaw`, legacy state names, tilde expansion, and non-state-dir parent paths.

## Reproduction Assessment

Yes. Source inspection is enough: `OPENCLAW_HOME=/home/user/.openclaw` is preserved by `resolveRequiredHomeDir()`, then both current state and config helpers append another `.openclaw` segment.

## Solution Assessment

No. Returning every explicit `OPENCLAW_HOME` directly changes the documented parent-home contract and changing only `resolveConfigDir()` leaves `resolveStateDir()` unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep parent OPENCLAW_HOME values under .openclaw:** `src/utils.ts:296-307`
  - body: This branch returns `OPENCLAW_HOME` itself for every explicit home override, so `OPENCLAW_HOME=/srv/openclaw-home` changes config-derived directories from `/srv/openclaw-home/.openclaw` to `/srv/openclaw-home`. Existing docs and tests define `OPENCLAW_HOME` as a home replacement; only skip the suffix when the basename already names a state directory.
  - confidence: 0.94
- **[P2] Apply the nesting guard to state-dir resolution too:** `src/utils.ts:296-297`
  - body: The reported onboarding/config placement also flows through `resolveStateDir()` and `resolveCanonicalConfigPath()`, but this PR only changes `resolveConfigDir()`. With `OPENCLAW_HOME` already ending in `.openclaw`, canonical state/config paths can still nest unless the same narrow guard is added in `src/config/paths.ts` with coverage.
  - confidence: 0.9
