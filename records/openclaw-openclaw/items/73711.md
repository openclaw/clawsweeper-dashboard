## Summary

Keep this PR open. The product direction is still useful and current main does not already provide the requested thumbnail/add-more composer UI, but the current PR head still has a blocking duplicate PhotosPicker observer regression and is missing the required user-facing changelog entry.

## What This Changes

The PR changes the shared SwiftUI chat composer attachment row from filename capsules to image thumbnail tiles with remove buttons and a trailing add-more picker tile.

## Best Possible Solution

Keep this PR as the canonical UI candidate, but update it to observe `pickerItems` exactly once from an always-present composer-level owner with an empty-selection guard, remove the stale dual-observer/idempotency comment, add the required changelog entry, and then validate toolbar pick, focused add-more pick, multi-select, remove, and send on a real iPhone.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on PR head `fc916584d936`: in the standard unfocused composer with at least one attachment, both the toolbar picker and add-more picker are mounted and observe the same `pickerItems`, so one selection can start two loader tasks before the shared state is cleared.

## Solution Assessment

No. The requested UI direction is reasonable, but the current implementation is not the best fix because sibling picker observers reintroduce the duplicate-load bug; a single stable observer is narrower and easier to maintain.
