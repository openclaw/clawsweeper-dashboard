## Summary

Keep this PR open as a useful implementation candidate, but do not merge the current head as-is. Current main still lacks the requested focused-composer add-more thumbnail affordance, while the PR head still has duplicate `pickerItems` observers that can process one PhotosPicker selection twice and also lacks the required user-facing changelog entry.

## What This Changes

The PR modifies `ChatComposer.swift` to replace the current capsule attachment row with shrinking thumbnail tiles, per-tile remove buttons, and a trailing add-more PhotosPicker/file-picker tile.

## Best Possible Solution

Keep this PR as the canonical UI candidate, but revise it so `pickerItems` is observed exactly once from a stable composer-level owner with an empty-selection guard, remove the stale duplicate-observer comment, add the required Unreleased changelog entry, and then validate the toolbar picker, focused add-more picker, multi-select, remove, and send flows on a real iPhone.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on the PR head: when the toolbar picker and the add-more tile are both mounted, both observe the same `pickerItems` change while the async loader clears state only after processing, so one pick can append twice; the author also reported the same behavior from real iPhone testing in the PR discussion.

## Solution Assessment

No. The UI direction is reasonable, but the current implementation is not the narrowest maintainable fix because it duplicates observers; a single stable observer plus an empty reset guard solves the focused-state drop without creating duplicate attachment processing.
