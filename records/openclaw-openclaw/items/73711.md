## Summary

Keep open: this is an active external UI PR, but the latest head still contains the duplicate picker-observer bug that earlier review identified, and the posted iPhone verification describes a single-observer fix that is not present in the current head.

## What This Changes

The PR changes the native iOS chat composer attachment strip to thumbnail tiles with remove buttons and a persistent add-more picker tile.

## Best Possible Solution

Keep the thumbnail/add-more direction, but move picker selection handling to one always-mounted observer, guard empty resets, remove stale idempotency claims, and attach current-head iPhone proof.

## Reproduction Assessment

Yes. Source inspection of the current PR head gives a high-confidence reproduction path: start with an existing attachment while the toolbar and add-more tile are both mounted, pick another image, and both picker observers can process the same selection before reset.

## Solution Assessment

No. The UI direction is reasonable, but the current implementation should use one stable observer for pickerItems instead of sibling PhotosPicker observers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Use one stable picker observer:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatComposer.swift:248-255`
  - body: The add-more PhotosPicker adds a second `.onChange` on `pickerItems` while the toolbar picker still observes the same state. With an existing attachment and the toolbar visible, one picker selection can start two loader tasks before `pickerItems` is reset, causing duplicate attachments to be appended; move the observer to one always-mounted parent and ignore empty resets.
  - confidence: 0.93
