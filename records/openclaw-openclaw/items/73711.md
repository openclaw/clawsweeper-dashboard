## Summary

Keep open: current main does not include the thumbnail/add-more attachment strip, but the latest PR head still has the duplicate PhotosPicker observer pattern that the discussion already reproduced as adding duplicate thumbnails. The contributor's real-device proof applies to an earlier head that claimed a single body-level observer, while the current head is fc916584 and still mirrors `.onChange` on the add-more tile.

## What This Changes

This PR changes `ChatComposer.swift` to replace attachment capsules with thumbnail tiles, per-tile removal, and a trailing add-more picker tile for iOS PhotosPicker and macOS file picking.

## Best Possible Solution

Move picker selection handling to one always-mounted observer, ignore empty resets, remove the stale dual-observer comment, then re-run the focused iPhone attachment-count smoke before maintainer review.

## Reproduction Assessment

Yes. Source inspection of current PR head shows two mounted `.onChange` observers on the same `pickerItems` state, and the PR discussion includes an iPhone reproduction where that exact pattern added two thumbnails for one pick.

## Solution Assessment

No. The thumbnail/add-more direction is reasonable, but the current implementation is not the narrowest maintainable fix because picker selections should be observed once from a stable parent instead of from sibling picker views.
