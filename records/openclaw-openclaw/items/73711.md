## Summary

Keep this PR open, but do not merge the current head. Current main still lacks the requested thumbnail/add-more attachment strip, but the latest PR head observes the same `pickerItems` state from both picker views, which can double-process one selection, and the user-facing change is missing the required changelog entry.

## What This Changes

This PR changes OpenClawKit `ChatComposer` to render selected attachments as shrinking thumbnail tiles with remove buttons and a trailing add-more PhotosPicker/file-picker tile.

## Best Possible Solution

Keep this PR as the active UI candidate, but revise it to observe `pickerItems` exactly once from a stable composer-level owner with an empty-selection guard, remove the stale dual-observer/idempotency comment, add the active Unreleased changelog entry, and validate the toolbar picker, focused add-more picker, multi-select, remove, and send flows before landing.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on the PR head: when both the toolbar picker and add-more tile are mounted, both observe the same non-empty `pickerItems` change while the async loader clears only after appending; the author also reported the same duplicate-add behavior during iPhone testing.

## Solution Assessment

No. The thumbnail strip direction is reasonable, but the current implementation is not the narrowest maintainable fix because it duplicates picker observers; a single stable observer with a reset guard solves the focused-state drop without duplicate processing.
