## Summary

Keep this PR open as the active UI candidate, but do not merge the current head: current main still lacks the thumbnail/add-more strip, and the PR head still has duplicate `pickerItems` observers plus no required changelog entry.

## What This Changes

The PR changes the shared SwiftUI chat composer to show selected attachments as thumbnail tiles with per-tile removal and a trailing add-more PhotosPicker/file-picker tile.

## Best Possible Solution

Land this UI direction only after the composer observes picker selections once from a stable parent, the stale duplicate-observer comment is removed, and the user-facing change is documented in the active changelog.

## Reproduction Assessment

Yes. Static inspection of the PR head shows two mounted `.onChange` observers can start concurrent `loadPhotosPickerItems` tasks before `pickerItems` is cleared, and the author independently reproduced the duplicate thumbnail behavior on a real iPhone.

## Solution Assessment

No. The thumbnail/add-more UI is a reasonable direction, but the current implementation is not the narrowest maintainable fix; a single stable observer with an empty-selection guard avoids both focused-state drops and duplicate processing.
