## Summary

Keep open: current main still lacks the focused-composer add-more affordance, and the latest PR head has a blocking duplicate picker-observer regression plus insufficient current-head real behavior proof.

## What This Changes

The PR changes the native chat composer attachment row into thumbnail tiles with per-item removal and a trailing add-more picker tile.

## Best Possible Solution

Keep the thumbnail/add-more direction, but move picker selection handling to one always-mounted observer, ignore empty resets, remove the stale idempotency comment, and attach current-head iPhone proof covering add, multi-select, remove, and send.

## Reproduction Assessment

Yes. Source inspection of the PR head shows two observers on the same pickerItems state, and the contributor already reported that this pattern produced duplicate thumbnails during iPhone testing.

## Solution Assessment

No. The UI direction is reasonable, but the current implementation should not observe the same picker state from sibling PhotosPicker views; a single stable observer is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

- **[P2] Move picker observation to one stable owner:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatComposer.swift:254-255`
