## Summary

Keep this PR open, but do not merge the current head. The UI direction still addresses a real iOS composer gap on main, but the latest PR head reintroduces the duplicate picker-observer pattern that the discussion already confirmed can double-add thumbnails, and it still lacks the required user-facing changelog entry.

## What This Changes

This PR changes the shared SwiftUI chat composer to show selected attachments as thumbnail tiles with per-tile removal and a trailing add-more PhotosPicker/file-picker tile.

## Best Possible Solution

Land this UI direction after the picker selection is observed once from a stable always-mounted owner, the stale idempotence comment/body are corrected, and the user-facing change is documented in the active changelog.

## Reproduction Assessment

Yes. Static inspection of current head shows two mounted `.onChange` observers can call the async loader for the same `pickerItems` snapshot, and the PR discussion includes a real-iPhone duplicate-thumbnail reproduction from that exact pattern.

## Solution Assessment

No. The thumbnail/add-more UI is reasonable, but the current implementation is not the narrowest maintainable fix; a single stable observer with an empty-selection guard avoids both focused-state drops and duplicate processing.
