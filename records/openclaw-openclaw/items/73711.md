## Summary

Keep open. Current main still lacks the thumbnail/add-more composer UI, so the PR is not obsolete, but the latest head should not merge because it reintroduces two picker observers that can duplicate attachments and it still lacks the required changelog entry.

## What This Changes

The PR modifies `ChatComposer.swift` to replace the attachment capsule row with thumbnail tiles, per-tile removal, and a trailing add-more PhotosPicker/file-picker tile.

## Best Possible Solution

Land this UI direction only after picker selections are observed once from a stable parent, empty resets are ignored, stale duplicate-observer comments are removed, and the user-facing change is documented in the active changelog.

## Reproduction Assessment

Yes. Static inspection of the latest PR head shows two mounted `.onChange` observers can start concurrent loader tasks before `pickerItems` is cleared, and the PR discussion includes a real-device reproduction of the duplicate-thumbnail behavior.

## Solution Assessment

No. The thumbnail/add-more UI is a reasonable direction, but the current implementation is not the narrowest maintainable fix; a single stable observer with an empty-selection guard avoids both focused-state drops and duplicate processing.
