## Summary

Keep open: current main still has a plausible frontend freeze path in the scoped settings tabs, and the Telegram event-loop-blocking explanation is not proven enough to close or queue a narrow automated fix.

## What This Changes

The issue reports dashboard-v2 freezes on Communications, AI & Agents, and Automation, attributed to Telegram getMe timeouts plus large config schema rendering.

## Best Possible Solution

Profile this with the broader Control UI sluggishness thread, then reduce synchronous full-schema work in scoped config tabs and keep channel probes bounded/asynchronous; add a Telegram-specific timeout knob only if profiling proves the existing controls are insufficient.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. Source inspection does show a plausible current-main frontend path where scoped config pages synchronously analyze the full root schema, but it does not prove the Telegram getMe event-loop stall.

## Solution Assessment

Unclear. The narrowest maintainable direction is profiling plus a focused fix for the proven bottleneck; adding only Telegram httpTimeoutMs is likely incomplete because the reported menus still fetch and normalize the full config schema.
