## Summary

Keep open for maintainer review. ## What This Changes

The PR strips `-free`, `-paid`, and `-trial` tier suffixes when matching reasoning_content replay model IDs and adds regression coverage for OpenCode Zen DeepSeek, OpenRouter MiMo, and Kimi suffixed routes.

## Best Possible Solution

Land the focused suffix-candidate fix with its regression tests if maintainers accept tier suffixes as contract-preserving; otherwise narrow the helper to only the observed `-free` suffix before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: current main never strips `-free` before checking the replay allowlist, so `deepseek-v4-flash-free` falls through and `reasoning_content` is stripped; the PR body also includes before/after boundary output.

## Solution Assessment

Is this the best way to solve the issue? Yes for the reported preserved-replay failure: adding suffix-normalized allowlist candidates is narrow and keeps the existing base-model allowlist gate; the separate OpenCode wrapper/backfill question can stay out of this PR.
