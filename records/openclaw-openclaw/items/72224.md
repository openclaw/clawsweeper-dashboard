## Summary

Keep open: current main still has the non-systemd gateway stop/restart detection gap, and this PR contains useful repair work, but the branch should not merge as-is because its lsof fallback loses argv verification and current main has newer lifecycle behavior to preserve.

## What This Changes

The PR updates gateway lifecycle/process discovery to recognize bare `openclaw-gateway`, add Linux `/proc` and `ss` listener PID fallbacks, reuse restart probe auth, and add regression tests.

## Best Possible Solution

Port the useful process-verifier and Linux listener-discovery changes onto current main while preserving current lifecycle, safe-restart, restart-intent, start-repair, and health-auth behavior.

## Reproduction Assessment

Yes. The source path is high-confidence: current main rejects a bare `["openclaw-gateway"]` argv before unmanaged signaling, and the lsof-unavailable path still returns no listener PIDs; I did not rerun the container smoke.

## Solution Assessment

Yes for the direction, but no for merging this branch as-is. The safer solution is a narrow current-main port that keeps the existing lsof argv fallback and newer daemon lifecycle behavior.
