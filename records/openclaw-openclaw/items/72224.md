## Summary

Keep this PR open because it addresses a real current-main bug, but it still needs repair before merge: the branch drops the existing lsof argv-verification fallback and is stale against newer gateway restart lifecycle work.

## What This Changes

The PR changes gateway restart and stop-related process detection for non-systemd Linux environments, including bare gateway process-title verification, Linux listener PID discovery fallbacks, probe-auth reuse, and regression tests.

## Best Possible Solution

Port the verifier and Linux listener-discovery changes onto current main while preserving current restart-intent, safe-restart, start-repair, and restart-health auth behavior.

## Reproduction Assessment

Yes. The source path is clear: current main rejects bare `openclaw-gateway` argv before unmanaged signaling, and non-Windows listener PID discovery still returns no PIDs when `lsof` is unavailable.

## Solution Assessment

Yes for the direction, but no for merging this branch as-is. The safer solution is a narrow current-main port that keeps the existing lsof argv fallback and the newer lifecycle behavior.
