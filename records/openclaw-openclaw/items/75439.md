## Summary

Keep open: the shared cooldown gate is the right direction for the exec-event heartbeat runaway, but the latest PR head still breaks existing immediate wake contracts and removes a root dependency that current main explicitly guards for packaged Bonjour startup.

## What This Changes

The PR adds a shared heartbeat cooldown helper and runner bookkeeping/tests/changelog for non-interval wakes, updates agent command test mocks, and removes the root `@homebridge/ciao` dependency.

## Best Possible Solution

Keep the centralized cooldown gate, but limit it to loop-prone feedback reasons while preserving explicit wake-now/task notification contracts and restoring the Bonjour packaged-runtime dependency contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `shouldDeferWake({ reason: "hook:wake" | "cron:job" | "background-task-blocked", lastRunStartedAtMs, now < nextDueMs })` on the PR head, plus the deterministic Bonjour manifest guard against the package removal.

## Solution Assessment

No. The shared helper and exec-event regression coverage are maintainable, but the current policy is too broad and the dependency deletion is unrelated to the heartbeat fix.
