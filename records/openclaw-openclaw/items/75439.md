## Summary

This PR should stay open because the central cooldown shape addresses the exec-event runaway, but the current head still defers documented immediate hook/cron wake-now paths and blocked-task follow-up wakes behind `nextDueMs`.

## What This Changes

The PR adds a centralized heartbeat wake cooldown helper, runner bookkeeping, regression coverage, a changelog entry, and removes an unused `qa-lab` workspace dependency.

## Best Possible Solution

Keep the shared cooldown/flood guard, but preserve explicit immediate intent for hook wake-now, cron wake-now fallback, and blocked task follow-up reasons while continuing to gate loop-prone exec/acp-style wakes.

## Reproduction Assessment

Yes. Current main's bug is reproducible from the runner paths that skip cooldown for non-interval targeted/broadcast wakes, and the PR-head regressions are reproducible by calling `shouldDeferWake` after a prior run with `hook:wake`, `cron:<job>`, or `background-task-blocked` while `now < nextDueMs`.

## Solution Assessment

No. The centralized helper is the right maintainable direction, but the current allowlist is incomplete and should carry documented immediate-wake contracts without reopening the exec-event feedback loop.

## Review Findings

  - confidence: 0.92
