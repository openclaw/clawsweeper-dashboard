## Summary

Keep this PR open: it addresses a real heartbeat runaway path, but the latest head still regresses documented wake-now behavior for hook and cron wake paths by routing them through the same cooldown as feedback-loop events.

## What This Changes

The PR adds a centralized heartbeat cooldown helper, records per-agent run/flood bookkeeping in the scheduler, adds heartbeat cooldown/scheduler tests, and adds a changelog entry for the exec-event runaway fix.

## Best Possible Solution

Keep the centralized cooldown seam, but make the wake request carry explicit immediacy/cooldown policy so feedback-loop reasons such as exec-event, acp:spawn:*, retry, and non-immediate queued wakes are throttled while documented wake-now requests still execute promptly.

## Reproduction Assessment

Yes. The original bug is reproducible from source on current main by sending repeated targeted exec-event wakes within nextDueMs; the remaining PR regression is reproducible by recording a prior heartbeat run, then issuing hook:wake or a cron wakeMode now queued fallback before nextDueMs and observing the PR helper return not-due.

## Solution Assessment

No. The shared helper is the right maintainability direction, but the current reason-only allowlist is too coarse because it cannot distinguish explicit wake-now intent from loop-prone event feedback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve explicit cron and hook wake-now requests:** `src/infra/heartbeat-cooldown.ts:70`
  - body: The latest helper still treats `hook:wake` and `cron:*` as event-driven reasons, and the new tests assert both return `not-due` once a prior run set `lastRunStartedAtMs`. That breaks documented `/hooks/wake` `mode: "now"` and cron `wakeMode: "now"` fallback behavior after any recent heartbeat, making those requests wait for the scheduled interval. Carry explicit wake-now intent through the request, or otherwise distinguish those immediate paths from loop-prone `exec-event`/ACP reasons.
