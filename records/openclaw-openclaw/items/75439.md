## Summary

Keep this PR open: it addresses the real exec-event runaway on current main and the earlier review blockers were mostly handled, but the latest diff still collapses explicit wake-now sources into the same cooldown bucket as feedback-style events. That would make documented hook wake-now and cron wakeMode now fallback paths wait for the next heartbeat interval after a prior run.

## What This Changes

The PR adds a heartbeat cooldown helper, wires targeted and broadcast heartbeat wakes through shared per-agent cooldown/flood bookkeeping, adds scheduler/cooldown regression tests, and adds a changelog entry for the heartbeat runaway fix.

## Best Possible Solution

Land a centralized heartbeat cooldown that throttles self-feeding exec/acp feedback loops while carrying an explicit wake-immediacy signal for cron and hook wake-now paths, with regression tests for both the runaway exec-event case and the preserved now contracts.

## Reproduction Assessment

Yes. The original bug has a source-level reproduction on current main with repeated session-scoped exec-event wakes calling runOnce repeatedly within nextDueMs. The remaining PR regression can be checked by simulating a prior heartbeat run, then issuing hook:wake or a cron wakeMode-now fallback before nextDueMs and observing the PR helper return not-due.

## Solution Assessment

No. The shared cooldown helper is the right seam, but the current reason-only policy is too coarse because it cannot distinguish feedback events from explicit wake-now requests. A safer fix should preserve immediacy via exact reasons or an explicit flag while keeping exec-event/acp-style loops gated.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Preserve cron and hook wake-now requests:** `src/infra/heartbeat-cooldown.ts:70`
  - body: The immediate allowlist only includes `manual`, `wake`, and `background-task`, so `hook:wake` and queued `cron:${job.id}` wakeMode-now fallbacks fall through to the event-driven gate. Carry an explicit immediacy flag or exact wake-now reason so these paths stay immediate while self-feeding exec/acp reasons remain gated.
