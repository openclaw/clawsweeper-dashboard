## Summary

Keep open: the core heartbeat cooldown fix is valuable, but the latest PR head still drops documented immediate wake-now and blocked-task follow-up paths, so it would regress hooks/cron/task delivery while fixing the exec-event runaway.

## What This Changes

The PR adds a centralized heartbeat cooldown helper, wires targeted and broadcast heartbeat dispatch through it, adds regression tests and changelog text, exports the heartbeat reason type, and switches qa-lab to the qa-channel package API.

## Best Possible Solution

Keep the shared cooldown gate, but carry explicit immediate intent or exact immediate reason policy for webhook/cron wake-now and blocked task follow-ups while leaving exec-event, ACP spawn, and other feedback-loop reasons throttled.

## Reproduction Assessment

Yes. After one heartbeat records lastRunStartedAtMs and nextDueMs remains in the future, the PR helper returns not-due for hook:wake, cron:<id>, and background-task-blocked even though current docs/source use those as immediate follow-up paths.

## Solution Assessment

No. The centralized helper is the right shape, but the raw allowlist is still too narrow; the safer fix is to preserve immediate intent explicitly and add regression tests for each preserved path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve hook and cron wake-now reasons:** `src/infra/heartbeat-cooldown.ts:70`
  - body: The immediate allowlist still omits `hook:wake` and `cron:<jobId>`. Both are used by documented `mode: "now"` / `--wake now` paths, so after a recent heartbeat they now hit `not-due` and wait for the scheduled interval instead of delivering the queued event promptly. Carry wake-now intent through the request or explicitly preserve these exact immediate paths while keeping loop-prone exec/acp reasons gated.
  - confidence: 0.93

Since the wake coalescer keeps the later same-priority reason for the same target, a blocked-task follow-up can replace the earlier `background-task` wake and then be deferred by the cooldown, delaying the user-visible blocked-task message until the next heartbeat interval.
  - confidence: 0.94
