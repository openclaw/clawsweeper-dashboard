## Summary

Keep this PR open: the centralized cooldown shape addresses the reported exec-event runaway and the author supplied useful live-gateway verification, but the latest head still has blocking regressions around immediate wake contracts and package dependency metadata.

## What This Changes

The PR adds a shared heartbeat cooldown helper, wires targeted and broadcast heartbeat dispatch through it, adds heartbeat regression tests and changelog text, and includes package/test/SDK metadata adjustments.

## Best Possible Solution

Keep the shared cooldown gate, but throttle only loop-prone feedback reasons while preserving explicit wake-now notifications, blocked-task follow-ups, and the existing Bonjour runtime dependency contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `shouldDeferWake({ reason: "hook:wake" | "cron:job" | "background-task-blocked", lastRunStartedAtMs, now < nextDueMs })`, which returns `not-due` on the PR despite current now-mode source paths; the package removal also directly violates `extensions/bonjour/manifest.test.ts`.

## Solution Assessment

No. The centralized helper is the right maintainable direction, but the current reason policy is too broad and the unrelated dependency removal breaks an existing packaging guard; the safer fix is a narrower immediate-reason policy plus aligned package metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve immediate hook and cron wakes:** `src/infra/heartbeat-cooldown.ts:70`
  - body: The allowlist excludes `hook:*` and `cron:*`, so after any recent heartbeat a `POST /hooks/wake` mode `now` or main-session cron wake-now fallback is skipped as `not-due` until the scheduled interval. Those paths enqueue user-visible events with explicit immediate intent, so they need to remain outside the normal interval cooldown while loop-prone exec/spawn reasons stay gated.
  - confidence: 0.92
- **[P2] Keep blocked-task follow-ups immediate:** `src/infra/heartbeat-cooldown.ts:70`
  - body: `background-task-blocked` is not included in the immediate reasons. That source queues a blocked-task follow-up for the owner session, but this PR defers it after any prior run, delaying the user-visible blocked-task notice by the heartbeat interval.
  - confidence: 0.9
- **[P2] Restore the root ciao dependency:** `package.json:1618`
  - body: Removing `@homebridge/ciao` from the root dependencies breaks the existing Bonjour manifest guard, which still asserts that packaged startup runtimes can resolve ciao from the root package. Restore the dependency and lock entry unless the Bonjour packaging contract and its test are intentionally changed together.
  - confidence: 0.96
