## Summary

Keep this PR open. Current main still has the heartbeat exec-event cooldown bug, and the PR is aimed at the right scheduler seam, but the latest diff still defers hook `mode: now` wake paths and is missing the required user-facing changelog entry.

## What This Changes

The PR adds a shared heartbeat cooldown helper, routes targeted and broadcast heartbeat wakes through it, and adds cooldown plus scheduler regression tests for repeated exec-event wakes.

## Best Possible Solution

Keep the centralized cooldown helper, but preserve explicit wake-now contracts such as `/hooks/wake` while throttling self-feeding exec/spawn feedback. Add focused regression coverage for that boundary and a single-line Unreleased/Fixes changelog entry before merge.

## Reproduction Assessment

Yes. Current main can be reproduced at source/test level by firing repeated scoped `exec-event` wakes within `nextDueMs`; the targeted branch reaches `runOnce` every time because it has no cooldown gate.

## Solution Assessment

No. The shared gate is the narrow maintainable seam, but this version still gates hook wake-now delivery and lacks the required changelog entry; a safer repair should carry or encode explicit immediacy separately from feedback-prone event reasons.
