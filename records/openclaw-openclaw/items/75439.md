## Summary

Keep open: the PR addresses a real current-main heartbeat cooldown gap, but latest head still defers documented wake-now hook, cron, and blocked-task follow-up paths behind `nextDueMs`.

## What This Changes

The PR adds a centralized heartbeat cooldown helper, runner bookkeeping, regression tests, a changelog entry, exports `HeartbeatReasonKind`, and adds a `plugin-sdk/tool-descriptors` package subpath.

## Best Possible Solution

Keep the centralized cooldown and flood guard, but preserve explicit wake-now contracts for hook, cron, and blocked-task follow-up reasons while continuing to gate loop-prone exec/acp/default wakes.

## Reproduction Assessment

Yes. Current main reproduces the original bug by sending repeated targeted `exec-event` wakes within `nextDueMs`; PR-head regressions are reproducible by calling the new cooldown gate after a prior run with `hook:*`, `cron:*`, or `background-task-blocked` while `now < nextDueMs`.

## Solution Assessment

No. The shared helper is the right maintainable direction, but the current immediate-reason policy is incomplete and breaks existing delivery contracts that should stay prompt without reopening the exec-event feedback loop.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve hook and cron wake-now requests:** `src/infra/heartbeat-cooldown.ts:70`
  - body: The immediate allowlist excludes `hook:*` and `cron:*`. After any prior heartbeat run, those wake-now paths fall through to the event-driven `now < nextDueMs` deferral, so webhook mode `now`, hook-agent announcements/errors, and main-session cron `--wake now` can wait for the next scheduled heartbeat instead of delivering promptly.
  - confidence: 0.94
- **[P2] Preserve blocked task follow-up wakes:** `src/infra/heartbeat-cooldown.ts:70`
  - body: Blocked task delivery wakes the owner session with `reason: "background-task-blocked"`, but the allowlist only preserves `background-task`. Once the owner agent has run, that follow-up is deferred behind the heartbeat interval, breaking the documented push-driven task completion behavior.
  - confidence: 0.92
