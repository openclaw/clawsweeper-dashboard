## Summary

Keep this PR open: the exec-event cooldown direction is well supported, but the latest head still defers documented wake-now hook/cron and blocked-task follow-up paths, and it removes the root Bonjour runtime dependency guarded on current main.

## What This Changes

The PR adds a centralized heartbeat cooldown helper, wires heartbeat dispatch through it, adds scheduler/cooldown tests and changelog text, updates agent command tests, and removes the root @homebridge/ciao dependency.

## Best Possible Solution

Keep the shared cooldown gate, but throttle only loop-prone feedback reasons while preserving explicit wake-now/blocking follow-up contracts and restoring the Bonjour dependency package contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction is shouldDeferWake({ reason: "hook:wake" | "cron:job" | "background-task-blocked", lastRunStartedAtMs, now < nextDueMs }), which defers on the PR despite current immediate wake contracts; extensions/bonjour/manifest.test.ts also deterministically conflicts with the package removal.

## Solution Assessment

No. The centralized helper and exec-event regression coverage are the right maintainable direction, but the current policy is too broad and the unrelated dependency removal breaks an existing packaging guard.
