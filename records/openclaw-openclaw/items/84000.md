## Summary

Keep open: the change targets a plausible tool-policy leak, but it is not merge-ready because the new gate treats `alsoAllow`-only inputs as restrictive and no real restricted-session proof is present.

## What This Changes

This PR adds a construction-time `sessions_yield` policy gate in `createOpenClawTools` and a unit test file for default inclusion, deny, allowlist exclusion, and `pluginToolDenylist`.

## Best Possible Solution

Keep the construction-time gate, but make it honor only true restrictive allowlists and denylists while preserving default core tools for `alsoAllow`-only policies, then add proof for the real restricted-session path.

## Reproduction Assessment

No high-confidence live reproduction of the linked runtime bypass is present. Source inspection confirms unconditional construction on main and a source-level `alsoAllow` compatibility regression in this PR.

## Solution Assessment

No, not as written. The construction-time fix is reasonable, but it must preserve additive `alsoAllow` semantics instead of treating every factory allowlist entry as a restrictive core-tool allowlist.
