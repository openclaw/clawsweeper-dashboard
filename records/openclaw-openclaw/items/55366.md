## Summary

Keep open. Current main still hard-resets after context-overflow or compaction failure and only preserves a bounded user/assistant transcript tail, so the requested reset reason, task goal, and progress handoff are still not present.

## What This Changes

The issue asks OpenClaw to inject minimal recovery context after hard context resets so an in-progress task can resume.

## Best Possible Solution

Keep hard reset as the final safety valve, but have the reset path write a bounded agent-visible handoff with reset reason, last user instruction or task goal, and safe recent progress context.

## Reproduction Assessment

Yes, from source inspection. Triggering runAgentTurnWithFallback with an embedded context-overflow payload or compaction-failure exception still reaches resetSessionAfterCompactionFailure and then only bounded user/assistant replay is written to the new transcript.

## Solution Assessment

Yes. A narrow change in the reset/replay helper path is the best maintainable solution because it addresses the data-loss UX without changing provider behavior, bootstrap policy, or the broader retry loop.
