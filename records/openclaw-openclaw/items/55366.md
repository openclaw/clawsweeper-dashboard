## Summary

Keep open. Current main still hard-resets after context-overflow or compaction failure and only replays a bounded user/assistant transcript tail, so the requested reset reason, task goal, and progress handoff are still missing.

## What This Changes

The issue requests that hard context resets preserve a minimal recovery handoff with reset reason, task goal, and progress state.

## Best Possible Solution

Keep hard reset as the final safety valve, but write a bounded agent-visible handoff with reset reason, last user instruction or task goal, and safe recent progress context into the rotated session.

## Reproduction Assessment

Yes, at source level. A context-overflow payload or compaction-failure exception still reaches resetSessionAfterCompactionFailure, which rotates the transcript and only replays bounded chat tail into the new session.

## Solution Assessment

Yes. A narrow reset/replay-path handoff is the best maintainable fix for this issue because it preserves recovery context without changing provider contracts, bootstrap policy, or the broader retry loop.
