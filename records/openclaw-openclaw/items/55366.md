## Summary

Keep open. Current main still has a hard context-overflow reset path that rotates the session and replays only a bounded user/assistant transcript tail, so the requested reset reason, task goal, and progress handoff are not fully implemented.

## What This Changes

The issue requests a bounded recovery context after hard context-limit resets so an agent can resume long-running work instead of losing task state.

## Best Possible Solution

Keep the hard reset safety valve, but write a bounded agent-visible handoff with reset reason, last user instruction or task goal, and safe recent progress context into the rotated session.

## Reproduction Assessment

Yes, at source level. A context-overflow payload or compaction-failure exception still reaches resetSessionAfterCompactionFailure, which rotates the session and replays only bounded chat tail into the new transcript.

## Solution Assessment

Yes. A narrow reset/replay-path handoff is the most maintainable fix for this issue because it improves recovery without changing provider contracts or the broader overflow retry policy.
