## Summary

Keep open. Current main still handles overflow/compaction-failure by rotating the session and returning the hard-reset message, while continuity is only a bounded replay of recent user/assistant records rather than the requested reset reason, task goal, or progress handoff.

## What This Changes

The issue requests a minimal recovery handoff after hard context resets so agents can resume in-progress work instead of starting blank.

## Best Possible Solution

Keep the hard reset as a safety valve, but make the rotated session explicitly resumable with a bounded agent-visible handoff: reset reason, last user instruction, and safe recent assistant-visible progress.

## Reproduction Assessment

Yes, from source inspection. Trigger runAgentTurnWithFallback with an embedded context-overflow meta error or compaction-failure exception and current main calls resetSessionAfterCompactionFailure, then writes only bounded user/assistant replay into the new transcript.

## Solution Assessment

Yes. A narrow fix in the reset/replay helper path is the best maintainable direction because it addresses the data-loss UX without changing provider behavior, channel ownership, or broader compaction retry policy.
