## Summary

Keep open. Current main still rotates the session after compaction/context-overflow failure and only replays a small user/assistant transcript tail, so the requested reset reason, task goal, and progress handoff are still missing.

## What This Changes

The issue requests a minimal recovery context after hard context-overflow resets so long-running tasks can resume instead of starting blank.

## Best Possible Solution

Keep hard reset as a safety valve, but make the rotated session resumable with a bounded agent-visible handoff containing reset reason, last user instruction or task goal, and safe recent progress context.

## Reproduction Assessment

Yes, from source inspection. Trigger runAgentTurnWithFallback with an embedded context-overflow payload or compaction-failure exception and current main calls resetSessionAfterCompactionFailure, then writes only bounded user/assistant replay into the new transcript.

## Solution Assessment

Yes. A narrow change in the reset/replay helper path is the best maintainable direction because it addresses the data-loss UX without changing provider behavior, channel ownership, or broader compaction retry policy.
