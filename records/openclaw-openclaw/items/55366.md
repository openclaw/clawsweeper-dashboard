## Summary

Keep open. Current main still has a reachable context-overflow hard-reset path that rotates the session and replays only a bounded user/assistant transcript tail, while the requested reset reason, task goal, and progress handoff are not implemented; related open work is adjacent but not merged and not a complete replacement.

## What This Changes

The issue asks OpenClaw to preserve enough recovery context after a hard context reset so an in-progress agent task can resume.

## Best Possible Solution

Avoid blank recoveries after unrecoverable overflow: either land a safe preserved-session path or add a bounded agent-visible handoff with reset reason, last user instruction or task goal, and safe recent progress context.

## Reproduction Assessment

Yes, at source level. A context-overflow payload or compaction-failure exception still reaches resetSessionAfterCompactionFailure, which rotates the session and replays only bounded user/assistant chat tail into the new transcript.

## Solution Assessment

Yes in direction, but maintainer scoping is needed before implementation. A narrow reset/replay or preserved-session recovery fix is maintainable only if it stays bounded and does not replay raw tool, system, compaction, or sensitive records.
