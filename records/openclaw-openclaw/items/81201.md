## Summary

The reported DM-vs-group difference matches the current documented visible-reply policy for group rooms, not a source-level WhatsApp delivery bug: normal final replies are private by default in groups unless the model uses the message tool or config opts back into automatic visible replies.

## What This Changes

The issue asks for WhatsApp group session auto-replies to be delivered like direct-message auto-replies.

## Best Possible Solution

Keep the current group default, use `messages.groupChat.visibleReplies: "automatic"` for legacy visible final replies, and reopen with redacted logs only if automatic mode still routes a group turn to the wrong target.

## Reproduction Assessment

No, there is not a high-confidence current-main bug reproduction. Source and docs show that a generated final answer not appearing in a WhatsApp group is expected under the default group `message_tool` visible-reply policy unless the model calls `message(action=send)` or config opts into automatic visible replies.

## Solution Assessment

Yes, the current solution is the maintainable one: group/channel visibility is controlled by the shared visible-reply policy, and WhatsApp uses the same durable delivery path rather than a special-case auto-reply fork. The safer user action is config or model-tooling adjustment, not a code change from this report.
