## Summary

Keeping this PR open: it has a protected `maintainer` label, and the current head still has blocking correctness and security-sensitive review findings in the Control UI/gateway config paths.

## What This Changes

The PR reorganizes Control UI settings/sidebar navigation, adds sidebar recent-session and new-session shortcuts, simplifies Cron Jobs creation, and adds a short-lived gateway config schema response cache with related tests.

## Best Possible Solution

Keep the UI direction only if the gateway schema cache is keyed to real config/schema metadata identity and the sidebar shortcut renderer reuses the drawer-aware collapsed state, with proof for the visible sidebar and Cron flows before maintainer review.

## Reproduction Assessment

Yes for the review findings: source inspection shows a time-only schema cache feeding redaction hints, and the sidebar session helper can diverge from the drawer-aware collapsed state. I did not execute the UI or gateway in this read-only review.

## Solution Assessment

No. The broad UI direction may be acceptable, but the cache needs contract-aware invalidation and the sidebar sessions should use the single computed rendered collapse state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Key schema caching to real schema changes:** `src/gateway/server-methods/config.ts:279-288`
  - body: The new cache returns the previous `ConfigSchemaResponse` for five seconds without checking whether runtime schema inputs changed outside the edited config write handlers. Since config responses redact snapshots with `schema.uiHints`, stale hints can miss newly sensitive plugin-defined fields; key the cache to runtime config/schema metadata identity or remove the time-only reuse.
  - confidence: 0.86
- **[P2] Pass rendered collapse state to sidebar sessions:** `ui/src/ui/app-render.ts:267-269`
  - body: When the saved sidebar preference is collapsed and the drawer is opened, the shell uses a drawer-aware `navCollapsed` value but the new session shortcut renderer still reads the persisted setting. That leaves the New session label and recent-session list hidden while the rest of the drawer is expanded; pass the computed value into the helper and cover this case.
  - confidence: 0.9
