## Summary

Keep this PR open but not merge-ready: the proposed index rewrite may be useful, but the branch removes the forced QMD managed-collection rebuild that current main still uses for duplicate-document and null-byte repair, and it lacks real QMD repair proof.

## What This Changes

The PR writes a QMD `index.yml` during collection ensure, routes managed collection repair through `ensureCollections()`, and updates QMD manager tests to expect no repair-time removals.

## Best Possible Solution

Preserve repair-time forced remove/add semantics, add a YAML-safe `index.yml` rewrite only if real QMD repair proof supports it, and cover both normal ensure and repair paths with focused tests.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: if QMD lists matching managed collections and `qmd update` fails with a duplicate-document constraint, the proposed `ensureCollections()` repair path skips remove/add before retrying. I did not run tests because this review is read-only.

## Solution Assessment

No. Rewriting `index.yml` may be a useful resilience step, but replacing forced repair rebuilds with normal idempotent collection ensure is not the narrowest maintainable fix.
