## Summary

Keep open: the patch still has a blocking repair-semantics regression, unsafe generated YAML, and no real QMD repair proof after the change.

## What This Changes

The PR adds a hand-written QMD `index.yml` writer during collection ensure, delegates repair rebuilds to `ensureCollections()`, and updates tests to expect no repair-time collection removals.

## Best Possible Solution

Keep forced managed-collection rebuild semantics for repair, add a YAML-safe index rewrite only if real repair evidence supports it, and cover both paths with focused tests.

## Reproduction Assessment

Yes for the PR regression by source inspection: if QMD still lists matching managed collections and `qmd update` fails with a duplicate-document constraint, the proposed repair path skips remove/add before retrying. I did not run tests because this review is read-only.

## Solution Assessment

No. Rewriting `index.yml` may be a useful resilience step, but replacing forced repair rebuilds with the normal idempotent ensure path is not the narrowest maintainable fix.
