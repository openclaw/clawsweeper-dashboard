## Summary

Keep this PR open: it contains a potentially useful QMD index rewrite, but the current patch regresses forced managed-collection repair semantics and still lacks real behavior proof from an actual QMD repair run.

## What This Changes

The PR writes QMD `index.yml` before collection ensure and changes managed collection repair to call `ensureCollections()` instead of explicitly removing and re-adding every managed collection.

## Best Possible Solution

Keep any needed index rewrite, but serialize QMD config safely and preserve explicit forced remove/add repair semantics with focused coverage for listed collections and YAML-special values.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: make `qmd collection list --json` return matching managed collections, then have `qmd update` fail with a duplicate-document constraint and the proposed ensure path skips remove/add before retrying.

## Solution Assessment

No. Rewriting `index.yml` may be a reasonable resilience step, but replacing the forced repair rebuild with the normal idempotent ensure path is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve forced repair rebuilds:** `extensions/memory-core/src/memory/qmd-manager.ts:940`
  - body: `rebuildManagedCollectionsForRepair()` now delegates to `ensureCollections()`, which skips remove/add when QMD lists matching path and pattern metadata. Duplicate-document or null-byte recovery can then retry against the same broken database state instead of resetting managed collections as current main does.
  - confidence: 0.91
- **[P2] Quote generated YAML scalars:** `extensions/memory-core/src/memory/qmd-manager.ts:461-462`
  - body: The new `index.yml` writer interpolates collection names and paths directly while only quoting `pattern`. Values containing YAML syntax such as `#` can parse as a different key or path, so use a YAML serializer or quote generated scalars consistently.
  - confidence: 0.86
