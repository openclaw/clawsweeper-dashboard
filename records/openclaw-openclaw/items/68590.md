## Summary

Keep this PR open, but do not merge it as-is. The index rewrite direction is useful, but the patch regresses the managed-collection repair path by replacing the current forced remove/add reset with the normal idempotent ensure path, and it writes YAML scalars without quoting.

## What This Changes

The PR adds a QMD `index.yml` rewrite inside managed collection bootstrap, changes managed update repair to call `ensureCollections()`, and updates qmd-manager tests to expect repair without collection removes.

## Best Possible Solution

Keep the `index.yml` rewrite concept, but implement it with YAML-safe scalar serialization and preserve repair reset semantics through forced remove/add or an explicit force mode. Add focused tests for YAML-special characters and for duplicate-document repair when QMD still lists matching managed collections.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to mock `qmd collection list --json` returning matching `memory-root-main` and `memory-dir-main`, make `qmd update` fail with `UNIQUE constraint failed: documents.collection, documents.path`, and verify the PR skips remove/add before retrying.

## Solution Assessment

No. Rewriting the QMD index before collection reconciliation is a reasonable part of the fix, but replacing the forced repair loop with the normal idempotent ensure path is not the narrowest maintainable solution.
