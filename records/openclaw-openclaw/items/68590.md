## Summary

Keep this PR open, but it is not ready to merge: the index rewrite may be useful, but the patch changes repair from a forced remove/add rebuild to the idempotent bootstrap path, which can leave duplicate-document or null-byte repair running against the same broken QMD state.

## What This Changes

The PR writes a per-agent QMD index.yml before collection ensure, delegates managed-collection repair to ensureCollections(), and updates QMD manager tests to expect no repair-time removals.

## Best Possible Solution

Preserve forced managed-collection rebuild semantics for repair, add any needed index.yml rewrite with safe YAML serialization and focused coverage, then require redacted real QMD repair proof before merge.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: if QMD lists matching managed collections and qmd update fails with a duplicate-document constraint, the proposed ensureCollections() repair path skips remove/add before retrying. I did not run tests because this review is read-only.

## Solution Assessment

No. Rewriting index.yml may be a reasonable resilience step, but replacing the forced repair rebuild with the normal idempotent ensure path is not the narrowest maintainable fix.
