## Summary

Keep this PR open, but the current head should not merge: the two correctness blockers remain valid against current main, and the contributor still has not supplied real QMD repair proof.

## What This Changes

The PR writes a QMD `index.yml` during collection ensure, changes managed collection repair to reuse `ensureCollections()`, and updates QMD manager tests to expect no repair-time removals.

## Best Possible Solution

Keep any needed index rewrite, but serialize QMD config safely and preserve explicit forced remove/add repair semantics with focused coverage for listed collections and YAML-special values.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: make `qmd collection list --json` return matching managed collections, then have `qmd update` fail with a duplicate-document constraint and the proposed ensure path skips remove/add before retrying. I did not run tests because this review is read-only.

## Solution Assessment

No. Rewriting `index.yml` may be a reasonable resilience step, but replacing the forced repair rebuild with the normal idempotent ensure path is not the narrowest maintainable fix.
