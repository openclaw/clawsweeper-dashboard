## Summary

Keep this PR open for contributor or maintainer follow-up: current main still has the forced QMD collection rebuild behavior, while the PR replaces it with the normal idempotent ensure path and lacks real repair-run proof.

## What This Changes

The PR writes a QMD `index.yml` before collection ensure, delegates managed collection repair to `ensureCollections()`, and updates QMD manager tests to expect no repair-time removals.

## Best Possible Solution

Keep forced managed-collection rebuild semantics for repair, add a YAML-safe index rewrite only if real repair evidence supports it, and cover both paths with focused tests.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: if QMD lists matching managed collections and `qmd update` fails with a duplicate-document constraint, the proposed `ensureCollections()` repair path skips remove/add before retrying. I did not run tests because this review is read-only.

## Solution Assessment

No. Rewriting `index.yml` may be a reasonable resilience step, but replacing the forced repair rebuild with normal idempotent collection ensure is not the narrowest maintainable fix.
