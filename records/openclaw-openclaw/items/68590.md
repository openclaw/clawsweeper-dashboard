## Summary

Keep this PR open but not merge-ready. The prior Greptile and ClawSweeper comments still match current evidence: the useful index rewrite is coupled to a repair regression that can skip the forced remove/add rebuild, the hand-written YAML is unsafe for special characters, and the PR still lacks real QMD repair proof.

## What This Changes

The PR writes a QMD index.yml during collection ensure, replaces repair-time remove/add loops with ensureCollections(), and updates QMD manager tests to expect no repair-time removals.

## Best Possible Solution

Preserve any needed index rewrite only with safe YAML serialization and focused coverage, while keeping forced managed-collection rebuild semantics for repair.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: make qmd collection list --json return matching managed collections, then have qmd update fail with a duplicate-document constraint and the proposed ensure path skips remove/add before retrying. I did not run tests because this review is read-only.

## Solution Assessment

No. Rewriting index.yml may be a reasonable resilience step, but replacing the forced repair rebuild with the normal idempotent ensure path is not the narrowest maintainable fix.

Overall correctness: patch is incorrect
