## Summary

Keep open: this external PR still weakens the managed QMD repair path by replacing the forced remove/add rebuild with the normal idempotent ensure path, and it adds an unsafe hand-written YAML writer without real behavior proof.

## What This Changes

The branch writes a QMD `index.yml` during collection ensure, changes managed collection repair to call `ensureCollections()`, and updates QMD manager repair tests to expect no collection removals.

## Best Possible Solution

Keep any needed index rewrite, but serialize QMD config safely and preserve explicit forced remove/add repair semantics with focused coverage for listed collections and YAML-special values.

## Reproduction Assessment

Yes. From source inspection, make `qmd collection list --json` return matching managed collections, then make `qmd update` fail with a duplicate-document constraint; the PR's ensure path skips remove/add before retrying.

## Solution Assessment

No. Rewriting `index.yml` may be useful, but replacing the forced repair rebuild with the normal idempotent ensure path is not the narrowest maintainable fix.
