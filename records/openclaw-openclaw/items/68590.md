## Summary

Keep this PR open because the index rewrite is useful source work, but the proposed patch regresses the existing forced QMD repair path and has an unsafe hand-written YAML writer. Current main still performs remove/add during null-byte and duplicate-document repair, while this PR can skip that reset when QMD lists matching collection metadata.

## What This Changes

The PR adds a QMD `index.yml` rewrite to `ensureCollections()`, changes managed update repair to delegate to `ensureCollections()`, and updates qmd-manager tests to expect no collection removes during repair.

## Best Possible Solution

Keep the safe `index.yml` rewrite idea, but implement it as a YAML-safe writer shared by bootstrap and repair while preserving repair reset semantics through forced remove/add or an explicit force mode. Add focused tests for YAML-special characters and for duplicate-document repair when QMD still lists matching managed collections.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to mock `qmd collection list --json` returning matching `memory-root-main` and `memory-dir-main`, then make `qmd update` fail with `UNIQUE constraint failed: documents.collection, documents.path`; the PR will skip remove/add and retry unchanged.

## Solution Assessment

No. Rewriting `index.yml` before ensuring collections is a reasonable direction, but replacing the forced repair loop with the normal idempotent ensure path is not the narrowest safe fix.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep forced rebuild in repair path:** `extensions/memory-core/src/memory/qmd-manager.ts:939-942`
  - body: `rebuildManagedCollectionsForRepair` now delegates to `ensureCollections()`, which skips collections when `collection list --json` reports matching path/pattern metadata. In that case duplicate-document or null-byte repair only rewrites `index.yml` and retries against the same QMD database state, whereas current main deliberately removes and re-adds each managed collection during repair.
  - confidence: 0.91
- **[P2] Quote generated YAML collection values:** `extensions/memory-core/src/memory/qmd-manager.ts:462-464`
  - body: `collection.name` and `collection.path` are interpolated directly into YAML while only `pattern` is quoted. A path or name containing YAML syntax such as `#` can be parsed differently from the configured value, so quote all generated scalar values or use a YAML-safe serializer.
  - confidence: 0.85
