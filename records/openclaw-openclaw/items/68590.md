## Summary

Keep this PR open for changes. The index rewrite is useful source work and is not already present on current main, but the patch weakens the managed-collection repair path by replacing the forced remove/add reset with the idempotent bootstrap path, and it hand-writes YAML without quoting collection names or paths. Existing Greptile and ClawSweeper review comments identify the same concrete blockers.

## What This Changes

The PR writes a QMD `index.yml` from managed collections during `ensureCollections()`, changes managed collection repair to call `ensureCollections()`, and updates qmd-manager tests to expect no collection removals during repair.

## Best Possible Solution

Revise the PR so bootstrap and repair share a safe `index.yml` writer, while repair still forces a managed collection reset or uses an explicit force mode. Add focused tests for generated YAML quoting and for duplicate-document repair when QMD still lists matching collections.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to mock `qmd collection list --json` returning matching `memory-root-main` and `memory-dir-main`, then trigger the duplicate-document update failure; the PR would skip remove/add and retry against unchanged QMD state.

## Solution Assessment

No. Rewriting `index.yml` before managed collection setup is a reasonable direction, but replacing the forced repair loop with the normal idempotent ensure path is not the safest repair; factor the writer separately and preserve reset semantics.
