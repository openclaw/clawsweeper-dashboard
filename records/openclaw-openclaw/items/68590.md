## Summary

Keep open: the index-writer idea is useful, but this PR should not merge as-is because it weakens the forced QMD repair reset and writes YAML scalars unsafely.

## What This Changes

The PR adds a QMD index.yml writer in ensureCollections(), changes update repair to call ensureCollections(), and updates qmd-manager repair tests to expect no collection removals.

## Best Possible Solution

Keep the index.yml rewrite concept, serialize the generated YAML safely, and preserve repair reset semantics with forced remove/add or an explicit force mode plus focused tests.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to mock `qmd collection list --json` returning matching managed collections, make `qmd update` fail with `UNIQUE constraint failed: documents.collection, documents.path`, and verify the PR skips remove/add before retrying.

## Solution Assessment

No. Rewriting the QMD index before reconciliation is reasonable, but replacing the forced repair loop with the normal idempotent ensure path is not the narrowest maintainable fix.
