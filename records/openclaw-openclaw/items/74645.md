## Summary

Keep this PR open. Current main still has the macOS Talk routing bug for non-local providers, and the PR is a narrow functional fix that matches the existing Gateway `talk.speak` contract, but it needs small mechanical cleanup before merge.

## What This Changes

The PR adds a macOS Talk playback plan that routes non-local Talk providers through Gateway `talk.speak`, preserves the ElevenLabs/MLX/system paths, forwards configured voice IDs for remote providers, and updates the Swift playback-plan test.

## Best Possible Solution

Land this PR or an equivalent narrow macOS runtime change after the mechanical review items are fixed, keeping provider-selection UI as a separate product decision if it remains desired.

## Reproduction Assessment

Yes. On current main, `TalkModeRuntime.playbackPlan(provider: "openai", apiKey: nil, voiceId: "onyx")` reaches the default branch and returns `.systemVoiceOnly`, so macOS never attempts Gateway `talk.speak` for that configured provider.

## Solution Assessment

Yes for the routing bug. The PR reuses the existing macOS `playGatewayTalkSpeak` helper and Gateway RPC instead of adding a new API, while preserving explicit `system`, `mlx`, and direct ElevenLabs behavior; it should also update release notes and the stale provider diagnostic before landing.
