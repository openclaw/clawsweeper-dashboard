## Summary

Keep open. Current main has generic empty-result fallback, `message_sending` delivery cancel/rewrite, and `before_agent_finalize` revision support, but it still lacks the requested post-completion quality gate with finish/provider metadata and reject-to-model-fallback semantics.

## What This Changes

Request a post-completion, pre-delivery hook that can inspect, modify, or reject model output and send rejected content through the configured fallback chain.

## Best Possible Solution

Define a documented, bounded final-result classification seam before terminal acceptance that exposes response content and model metadata, then maps replay-safe rejection into existing model fallback classification.

## Reproduction Assessment

No live Gemini/WhatsApp reproduction is available from the report. Source inspection on current main does give a high-confidence reproduction of the feature gap: the requested hook contract and reject-to-fallback path are absent.

## Solution Assessment

Unclear. A literal delivery-stage `message:before_send` hook is likely too late for safe fallback, while a finalization/result-classification seam before terminal acceptance looks like the narrower maintainable direction.
