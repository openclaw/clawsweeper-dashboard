## Summary

Keep open. Current main covers generic pre-delivery rewrite/cancel and built-in empty-result model fallback, but it still lacks the requested plugin-defined post-completion quality gate that can reject a response and enter the configured fallback chain.

## What This Changes

The issue requests a new post-completion, pre-delivery hook that can inspect response content/model metadata and reject low-quality output into model fallback.

## Best Possible Solution

Define a documented, bounded final-result classification seam before terminal acceptance that exposes response content and model metadata, then maps replay-safe plugin rejection into the existing model fallback classifier.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main has no message:before_send/reject-to-fallback hook, and existing hooks only cancel delivery or request a same-harness revision. No live Gemini/WhatsApp reproduction is attached for the provider symptom.

## Solution Assessment

Unclear as proposed. A literal delivery-stage message:before_send hook is probably too late for safe model fallback; a final-result classification seam before runWithModelFallback accepts success is the narrower maintainable direction.
