## Summary

Keep open. Current main covers pre-delivery rewrite/cancel and built-in empty-result fallback, but it still does not provide the requested plugin-defined post-completion quality gate that can reject a response and enter model fallback.

## What This Changes

The issue requests a post-completion, pre-delivery content-quality hook that can pass, modify, or reject an LLM result and send rejects into model fallback.

## Best Possible Solution

Define a documented, bounded final-result classification seam before terminal acceptance that exposes response content and model metadata, then maps replay-safe rejection into existing model fallback classification.

## Reproduction Assessment

No live Gemini/WhatsApp reproduction is available from the report. Source inspection on current main gives a high-confidence reproduction of the feature gap: the requested hook contract and reject-to-fallback path are absent.

## Solution Assessment

Unclear. A literal delivery-stage `message:before_send` hook is likely too late for safe fallback, while a finalization/result-classification seam before terminal acceptance looks like the narrower maintainable direction.
