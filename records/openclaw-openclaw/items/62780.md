## Summary

Keep open. Current main has adjacent outbound cancellation, final-answer revision, and built-in empty-result fallback classification, but it still lacks the requested plugin-defined post-completion quality gate that can reject a successful response and re-enter the configured model fallback chain.

## What This Changes

The issue requests a pre-send/post-completion hook that can pass, modify, or reject assistant output so empty or low-quality successful model responses can trigger model fallback before channel delivery.

## Best Possible Solution

Define a documented, bounded final-result classification seam before terminal model success is accepted, expose plugin quality checks there, and map replay-safe rejections into the existing fallback classifier.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main has no hook that can reject a completed successful model response into fallback. No live Gemini/WhatsApp reproduction is attached for the provider-specific empty-content symptom.

## Solution Assessment

Unclear as proposed. A literal delivery-stage message:before_send hook may be too late for safe model fallback; a final-result classification seam before run success is accepted looks like the narrower maintainable direction.
