## Summary

Keep open. Current main covers generic empty terminal-result fallback and generic pre-send cancel/rewrite, but it still lacks the requested post-completion content-quality hook with provider/finish metadata and reject-to-fallback semantics.

## What This Changes

Request a post-LLM, pre-delivery plugin hook that can inspect, modify, or reject generated content and route rejections into model fallback.

## Best Possible Solution

Define a documented, bounded final-reply decision seam before terminal acceptance that exposes response content and model metadata, then maps replay-safe rejection into existing fallback classification.

## Reproduction Assessment

No high-confidence live reproduction is available for the Gemini/WhatsApp 200-empty behavior. Source inspection does reproduce the actionable feature gap: the requested hook contract and reject-to-fallback path are absent on current main.

## Solution Assessment

Unclear. A literal delivery-stage `message:before_send` hook may be too late for safe fallback; a finalization/result-classification seam before terminal acceptance looks like the narrower maintainable direction.
