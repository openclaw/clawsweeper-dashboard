## Summary

Keep open. Current main still renders the chat delete confirmation as a static absolutely positioned popover above the trigger with side-only alignment, while the related placement-fix PRs were closed unmerged and the later listener-cleanup work does not address viewport overflow.

## What This Changes

The issue asks for the Control UI chat delete confirmation popover to stay within the visible page instead of overflowing when deleting a chat message.

## Best Possible Solution

Add viewport-aware placement for the grouped chat delete confirmation, preserve the existing dismiss behavior, cover edge geometry in focused tests, and keep the broader localization suggestion separate.

## Reproduction Assessment

Yes. The reporter’s steps map directly to current main: clicking a delete button near the top or edge inserts a popover inside a clipped chat thread and positions it statically above/side without viewport measurement.

## Solution Assessment

Yes. A narrow placement helper in the grouped chat render/CSS path is the maintainable fix; removing absolute positioning would avoid the screenshot overflow but risks changing message layout flow.
