## Summary

Keep open. Current main and the latest release still render the chat delete confirmation with static above-trigger placement inside the chat scroll area, and the related viewport-placement PRs were closed unmerged rather than shipped.

## What This Changes

The issue asks for the chat delete confirmation popover to stay inside the visible chat page so the Delete button remains clickable.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation, preserving current skip/cancel/delete/outside-click/toggle cleanup behavior and keeping the broader localization suggestion separate.

## Reproduction Assessment

Yes. The reporter's click-delete flow maps directly to current main source: the popover is appended in the chat scroll path and positioned above the trigger without viewport measurement; I did not run a live browser smoke in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is a placement helper in the grouped chat render/CSS path that chooses above or below based on available space and clamps horizontally; removing absolute positioning or always moving the popover below would trade one overflow case for another.

The Control UI chat delete confirmation popover can overflow outside the visible chat area because `.chat-delete-confirm` is statically positioned above the delete button inside the scroll container. Add viewport-aware placement in `ui/src/ui/chat/grouped-render.ts` and `ui/src/styles/chat/grouped.css`: measure the trigger/popover, choose above or below based on available space, clamp horizontally within the viewport, and preserve current skip-confirm, Cancel, Delete, outside-click, and same-button toggle cleanup behavior. Keep the broader Chinese localization suggestion out of scope. Add focused regression coverage in `ui/src/ui/chat/grouped-render.test.ts`; use the closed PRs for context but do not copy their known one-sided or flawed placement behavior. No changelog is required until maintainer landing decides release wording.
