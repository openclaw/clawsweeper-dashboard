## Summary

Keep open. Current main and the latest release still render the chat delete confirmation as an absolute above-trigger popover with no viewport-aware flip or clamp logic, and the related fix PRs are closed unmerged.

## What This Changes

The issue asks for the Control UI chat message delete confirmation to stay inside the visible page instead of overflowing out of reach.

## Best Possible Solution

Add viewport-aware placement for the chat delete confirmation: measure the trigger and popover, choose above or below based on available space, clamp horizontally, preserve skip-confirm and dismiss behavior, and cover those edge cases in focused tests.

## Reproduction Assessment

Yes, source-reproducible: the reporter's delete-click flow maps to current main code that appends an above-trigger absolute popover inside a scrollable chat thread without viewport measurement. I did not run a live browser smoke in this read-only review.

## Solution Assessment

Yes. The maintainable fix is viewport-aware placement in the grouped chat delete-confirm path; a static top/bottom CSS swap or removing absolute positioning would only trade one overflow edge case for another.
