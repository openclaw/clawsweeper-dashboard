## Summary

Keep open. Current main still uses a static above-trigger delete confirmation inside the chat scroll area, with no viewport measurement, flip, or clamp logic; the latest release tag has the same CSS and the related viewport-placement PRs were closed unmerged.

## What This Changes

The issue reports that the Control UI chat delete confirmation can overflow the page boundary, making the Delete button unreachable.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation: measure the trigger and popover, choose above or below based on available space, clamp horizontally, and preserve existing skip-confirm, cancel, delete, outside-click, and toggle cleanup behavior.

## Reproduction Assessment

Yes, with source-level confidence: the reporter's click-delete flow maps to current main code that appends an absolutely positioned above-trigger popover inside the chat scroll area without viewport measurement. I did not run a live browser smoke in this read-only review, so the structured status is source-reproducible rather than live reproduced.

## Solution Assessment

Yes. The best narrow fix is viewport-aware placement in the grouped chat delete-confirm path; always moving the popover below or removing absolute positioning would only trade one edge case for another.
