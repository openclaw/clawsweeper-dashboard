## Summary

Keep open. Current main and the latest release still anchor the chat delete confirmation with static absolute/bottom placement and no viewport flip or clamp, while the exact placement-fix PRs were closed unmerged.

## What This Changes

The issue reports that the chat message delete confirmation popover can overflow the visible page boundary and make the Delete button unreachable.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation, preserve existing skip/cancel/delete/outside-click dismissal behavior, and keep the broader localization suggestion separate.

## Reproduction Assessment

Yes. The reporter's click-delete flow maps directly to current main source: the popover is inserted inside the chat scroll container and positioned above the trigger without viewport measurement.

## Solution Assessment

Yes. A narrow placement helper in the grouped chat render/CSS path is the maintainable fix; removing absolute positioning alone would risk changing message layout flow.
