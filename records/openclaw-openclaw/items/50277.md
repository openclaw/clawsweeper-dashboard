## Summary

Keep open. Current main and the latest release still render the chat delete confirmation as an absolutely positioned popover above the trigger with no viewport flip or clamp logic, while the related placement-fix PRs were closed unmerged and the remaining open related PR addresses listener cleanup rather than overflow placement.

## What This Changes

The issue requests fixing the Control UI chat delete confirmation popover so it stays within the visible page bounds, with a separate broader suggestion for Chinese localization.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation: measure the trigger and popover after insertion, choose above or below based on available space, clamp horizontally within the viewport, preserve skip-confirm/cancel/delete/outside-click behavior, and cover top-edge, bottom-edge, and narrow-width cases in focused tests. Keep the broader localization request separate from this bug fix.

## Reproduction Assessment

Yes. The reporter’s screenshot and steps map directly to current main: clicking a chat delete button near the top or edge creates a popover positioned above the trigger inside a clipped chat thread with no viewport measurement path.

## Solution Assessment

Yes. The current popover is appended under `.chat-delete-wrap` and styled as `position: absolute; bottom: calc(100% + 6px)`, which can push it outside or clip it near the top/edges of the chat thread. Validate with the targeted UI test, targeted oxfmt check, manual browser smoke or DOM geometry proof, and `pnpm check:changed` via Testbox before handoff.
