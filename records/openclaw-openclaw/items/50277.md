## Summary

Keep open. Current main still renders the chat delete confirmation as an above-trigger absolute popover inside the scrollable chat area, with no viewport flip or clamp logic; the latest release tag keeps the same placement CSS and the related viewport-fix PRs were closed unmerged.

## What This Changes

Fix the Control UI chat delete confirmation so it stays inside the viewport when deleting messages near visible edges.

## Best Possible Solution

Add viewport-aware placement for the chat delete confirmation: measure the trigger and popover, choose above or below based on available space, clamp horizontally, preserve existing dismiss and skip-confirm behavior, and cover the edge cases in focused tests.

## Reproduction Assessment

Yes, source-reproducible: the reporter's delete-click flow maps to current main code that appends an absolutely positioned above-trigger popover inside a scrollable chat thread without viewport measurement. I did not run a live browser smoke in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is viewport-aware placement in the grouped chat delete-confirm path; removing absolute positioning or always forcing the popover below would only trade one edge case for another.
