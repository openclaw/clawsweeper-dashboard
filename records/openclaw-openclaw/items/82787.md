## Summary

Keep open. Current main still finalizes a parent subagent wait timeout as `timed_out` without first reconciling the child session state, and the linked delivery PR explicitly leaves late-success lifecycle reconciliation for follow-up.

## What This Changes

The issue asks OpenClaw to avoid losing subagent success when the parent wait times out before a delayed child run starts and later succeeds.

## Best Possible Solution

Add timeout reconciliation in the subagent registry wait manager so persisted terminal child state wins over a stale `agent.wait` timeout, and nonterminal children remain supervised until real terminal state or explicit cancellation.

## Reproduction Assessment

Yes, at source level: make `agent.wait` return `timeout` before the child session reaches terminal state, then let the child session become `done`; current main finalizes the parent as `timed_out` and does not keep a watcher for the later success.

## Solution Assessment

Yes for the narrow fix direction. Reconcile timeout wait results against the child session store and retry/defer while the child is still running; keep broader queue-budget policy or durable outbox design in the canonical reliability work.
