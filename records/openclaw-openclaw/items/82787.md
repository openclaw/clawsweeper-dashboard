## Summary

Keep open: current main still has a source-reproducible subagent lifecycle gap where the parent wait can consume the full timeout before the child starts, then a later child success is not reconciled into the already timed-out parent task. The only related open PR I found, https://github.com/openclaw/openclaw/pull/82834, explicitly says it does not solve the late-success/reconcile work for this issue.

## What This Changes

The issue asks OpenClaw to preserve or surface successful subagent child results when the parent wait times out before the delayed child run starts.

## Best Possible Solution

Implement a narrow lifecycle fix that separates queued/not-started wait budget from active child execution or keeps a late-completion watcher that reconciles and notifies successful child completion without weakening real terminal failure semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: have agent.wait time out before a lifecycle start, then emit or record a later successful child result with the same run id; current task-registry logic preserves timed_out over that late success.

## Solution Assessment

Yes. Separating queue/start and active execution budgets or retaining a late-completion reconciler is the narrow maintainable direction; increasing the timeout alone would leave successful child results vulnerable to the same stale parent state.
