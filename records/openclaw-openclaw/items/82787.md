## Summary

Keep open: current main still allows the parent subagent wrapper wait to expire before the child run starts, and the later child success is not reconciled back into the parent subagent task or requester-visible completion state.

## What This Changes

The issue asks OpenClaw to separate queued subagent time from active child runtime and reconcile or notify when a child completes successfully after the parent wait timed out.

## Best Possible Solution

Add a narrow subagent lifecycle reconciliation path that either starts the active execution budget on child lifecycle start or keeps a watcher that propagates late child terminal results to the parent task and requester.

## Reproduction Assessment

Yes. A focused source reproduction can mock agent.wait returning timeout before any lifecycle start, then emit or record a later successful child completion with the same run id and observe the parent subagent task remains timed_out.

## Solution Assessment

Yes. Separating queued/start wait from active runtime, or retaining a late-completion reconciler, is the narrow maintainable fix; increasing the timeout alone would only hide the timing race.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/82787. Current main can mark a parent subagent task timed_out because the wrapper wait budget starts before the child CLI run actually starts; if the child later succeeds under the same run id, the parent subagent task and requester-visible completion state are not reconciled. Keep the fix narrow: separate queued/not-started wait from active child execution budget, or keep a late child terminal watcher/reconciler that updates the parent subagent task and notifies or wakes the requester for this delayed-start late-success case. Preserve monotonic terminal handling for genuine cancellations, failures, lost tasks, and unrelated run-scoped updates. Add regression coverage for the delayed-start/late-success path in the subagent lifecycle, gateway agent.wait, and task-registry surfaces. Do not add a new config option or broad product policy.
