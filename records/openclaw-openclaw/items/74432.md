## Summary

Keep open: the latest update usefully addresses Greptile's PDF whitespace and duplicate-/v1 comments by reusing one resolver, but the normal default Anthropic production path still skips ANTHROPIC_BASE_URL because catalog models already carry https://api.anthropic.com as model.baseUrl before the helper runs.

## What This Changes

The PR exports an Anthropic Messages URL resolver, adds ANTHROPIC_BASE_URL fallback tests for it, and reuses that resolver for native Anthropic PDF requests.

## Best Possible Solution

Land a source-aware Anthropic base URL fix that preserves explicit provider config precedence, lets non-empty ANTHROPIC_BASE_URL override catalog defaults, shares one /v1/messages normalizer for streaming and PDF, and adds focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Resolve a default Anthropic catalog model with ANTHROPIC_BASE_URL set and no models.providers.anthropic.baseUrl; current main and this PR head pass the catalog https://api.anthropic.com value into the helper, so the env var is skipped.

## Solution Assessment

No. The shared URL normalizer and PDF reuse are useful, but the env fallback must happen before explicit provider config and catalog defaults become indistinguishable, or the resolver needs base-url provenance.
