## Summary

Keep this PR open: the underlying Anthropic env-routing bug is valid, and the second commit addresses the earlier PDF URL-normalization concern, but the latest diff still misses the normal production path where Anthropic catalog models already carry `https://api.anthropic.com` as `model.baseUrl`.

## What This Changes

The PR exports an Anthropic Messages URL resolver with `ANTHROPIC_BASE_URL` fallback, adds resolver unit tests, and routes native Anthropic PDF fetches through that helper.

## Best Possible Solution

Land a source-aware Anthropic base URL fix that preserves explicit provider config precedence, treats catalog defaults as env-overridable, shares one `/v1/messages` normalizer for streaming and native PDF, and documents the user-facing fix in the changelog.

## Solution Assessment

No. The shared URL normalizer is useful, but the fix needs to happen before catalog defaults are indistinguishable from explicit user config, or the transport/PDF path needs explicit base-url provenance.
