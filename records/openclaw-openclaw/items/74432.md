## Summary

Keep open: the PR targets a valid current-main Anthropic routing bug, but latest head still treats default catalog `baseUrl` values as explicit, so env-only `ANTHROPIC_BASE_URL` users can still route to Anthropic's public endpoint, and the external PR has no real behavior proof yet.

## What This Changes

The PR exports the Anthropic Messages URL resolver, adds `ANTHROPIC_BASE_URL` fallback unit tests, and uses that resolver for native Anthropic PDF requests.

## Best Possible Solution

Make Anthropic URL resolution source-aware so explicit provider config wins, default catalog public URLs remain env-overridable, streaming and native PDF share normalization, and the branch includes focused regression tests plus redacted real proxy/PDF proof.

## Reproduction Assessment

Yes, source-level. Current main ignores `ANTHROPIC_BASE_URL` in the stream resolver and native PDF path, and normal catalog resolution can pass the public Anthropic `baseUrl` as a non-empty value before the PR helper sees the env var.

## Solution Assessment

No. Sharing URL normalization is the right direction, but the fix needs to distinguish explicit provider config from default catalog metadata before request URL construction.
