## Summary

Keep open: the intended no-config Anthropic env fallback is still incomplete because default resolved Anthropic models already pass the public catalog base URL into the new helper, so `ANTHROPIC_BASE_URL` remains skipped on the main production path.

## What This Changes

The PR exports the Anthropic Messages URL resolver, adds `ANTHROPIC_BASE_URL` fallback tests for it, and routes native Anthropic PDF fetches through the same resolver.

## Best Possible Solution

Land a source-aware Anthropic base URL fix that preserves explicit provider config precedence, treats catalog defaults as env-overridable, shares one `/v1/messages` normalizer for streaming and native PDF, and adds the required changelog entry.

## Reproduction Assessment

Yes, from source inspection. Resolve a default Anthropic catalog model with `ANTHROPIC_BASE_URL` set and no `models.providers.anthropic.baseUrl`; the current and proposed helper-level path still receives `https://api.anthropic.com` as `model.baseUrl`, so the env var is skipped.

## Solution Assessment

No. The shared URL normalizer is useful, but the fix needs source-aware base URL precedence before catalog defaults become indistinguishable from explicit user config, or equivalent provenance passed into the transport/PDF helper.
