## Summary

Keep open: the user-facing Anthropic proxy-routing bug is valid, but the current PR head does not fix the normal default Anthropic path because catalog model baseUrl is already populated before the new helper sees it; the PR also needs the required changelog entry.

## What This Changes

The PR adds ANTHROPIC_BASE_URL fallback to the Anthropic messages URL helper, exports tests for that helper, and reuses it for native Anthropic PDF requests.

## Best Possible Solution

Land a source-aware Anthropic base URL fix that applies the env fallback only when no explicit provider config exists, shares one /v1/messages normalizer for streaming and PDF, and adds focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Resolve a default Anthropic catalog model with ANTHROPIC_BASE_URL set and no models.providers.anthropic.baseUrl; current and PR-head production paths still pass the catalog https://api.anthropic.com baseUrl into the helper, so the env var is skipped.

## Solution Assessment

No. The shared URL normalizer is useful, but the env fallback needs to happen before catalog defaults are indistinguishable from explicit user config, or the resolver needs explicit base-url provenance.
