## Summary

Keep open. The PR targets a real Anthropic routing bug, but the current patch still misses the normal no-provider-config path because default Anthropic catalog models already pass `https://api.anthropic.com` as `model.baseUrl` before the new helper can fall back to `ANTHROPIC_BASE_URL`; the external PR also still lacks real after-fix behavior proof.

## What This Changes

The branch exports an Anthropic Messages URL resolver with `ANTHROPIC_BASE_URL` fallback, adds resolver unit tests, and makes native Anthropic PDF requests use that resolver.

## Best Possible Solution

Move Anthropic base URL selection to a source-aware resolver or default-catalog layer so only explicit provider config wins over `ANTHROPIC_BASE_URL`, then cover streaming and native PDF with regression tests and redacted real proxy/PDF proof before merge.

## Reproduction Assessment

Yes, source-level. Default Anthropic catalog models carry `https://api.anthropic.com` as `model.baseUrl`, so with only `ANTHROPIC_BASE_URL` set the PR helper still receives a non-empty base URL and skips the env fallback.

## Solution Assessment

No. Sharing URL normalization is useful, but it runs too late for catalog defaults; the fix should distinguish explicit provider config from default catalog metadata before constructing streaming and native PDF request URLs.
