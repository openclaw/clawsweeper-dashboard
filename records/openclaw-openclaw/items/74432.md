## Summary

Keep open. The requested provider-routing fix is valid, but the current patch still misses the normal default Anthropic model path because `model.baseUrl` has already been populated from the catalog before the new helper checks `ANTHROPIC_BASE_URL`; it also needs the required user-facing changelog entry.

## What This Changes

The PR exports an Anthropic Messages URL resolver with explicit/env/default precedence, adds resolver unit tests, and reuses that resolver for native Anthropic PDF requests.

## Best Possible Solution

Resolve the env-backed default before base-url provenance is lost, or pass explicit/default provenance into the URL resolver. The final behavior should preserve explicit config precedence, honor non-empty `ANTHROPIC_BASE_URL` for default Anthropic streaming and native PDF calls, share one `/v1/messages` normalizer, and document the release-note-visible fix.

## Reproduction Assessment

Yes. Resolve a default Anthropic catalog model with only `ANTHROPIC_BASE_URL` set, then exercise streaming or native PDF; the current PR helper receives `https://api.anthropic.com` as `baseUrl` and skips the env value.

## Solution Assessment

No. Sharing the URL normalizer is useful, but this PR applies the env fallback too late in the pipeline; the safer fix is source-aware model/provider default resolution or an explicit provenance parameter.
