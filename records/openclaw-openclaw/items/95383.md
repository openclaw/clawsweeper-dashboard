## What This Changes

The PR reorders SDK agent tool/item/command_output event normalization so failed or blocked terminal statuses map to `tool.call.failed`, and adds focused normalize tests.

## Best Possible Solution

Land the narrow classifier fix with regression coverage after maintainers accept the release-note-worthy SDK event behavior correction.

## Reproduction Assessment

Yes. A current-main call to `normalizeGatewayEvent` with an agent item or command_output event containing `phase: "end"` and `status: "failed"` or `"blocked"` will hit the completed branch before the failed branch.

## Solution Assessment

Yes. The SDK normalizer is the public event-type boundary and producers already carry terminal status, so reordering the existing checks is narrower than changing producer payloads or adding another SDK type.
