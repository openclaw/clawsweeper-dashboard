## Summary

Keep open: current main still classifies failed or blocked terminal tool/item/command_output events as `tool.call.completed`, while this PR is a narrow and well-supported SDK classifier fix. The remaining decision is maintainer acceptance of the release-note-worthy public SDK event behavior change before merge.

## What This Changes

The PR reorders SDK agent-event normalization so failed/blocked terminal tool, item, and command_output events map to `tool.call.failed`, and adds regression tests.

## Best Possible Solution

Land the narrow classifier fix with regression coverage after maintainers explicitly accept the public SDK event behavior correction.

## Reproduction Assessment

Yes. A current-main call to `normalizeGatewayEvent` with an `agent` item or command_output payload containing `phase:"end"` and `status:"failed"` or `"blocked"` reaches the completed branch before the failed branch by source inspection.

## Solution Assessment

Yes. The SDK normalizer is the public event-type boundary and producers already carry terminal status, so reordering the existing checks is narrower than changing producers or adding a parallel event type.
