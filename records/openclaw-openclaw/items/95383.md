## Summary

Keep open: current main and the latest release still classify failed/blocked terminal SDK tool-item events as `tool.call.completed`. The PR is a narrow, well-supported bug fix, but the public SDK event-type correction is an observable compatibility change that needs maintainer acceptance before merge.

## What This Changes

This PR reorders SDK agent-event normalization so failed/blocked terminal tool/item events map to `tool.call.failed` and adds focused regression coverage.

## Best Possible Solution

Land the narrow SDK classifier fix with regression coverage and release-note context for the public event behavior correction after maintainer review accepts the compatibility impact.

## Solution Assessment

Yes. Moving the failure-status branch inside the SDK normalizer is the narrowest maintainable fix because producers already preserve the real status and the SDK client centralizes normalized event typing there.
