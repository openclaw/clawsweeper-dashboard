## What This Changes

The PR reorders SDK gateway event normalization so failed or blocked terminal tool/item/command-output events map to `tool.call.failed`, and adds focused regression tests for item events.

## Best Possible Solution

Land the narrow SDK classifier fix with regression coverage after maintainers accept the release-note-worthy public event behavior correction.

## Reproduction Assessment

Yes. Source inspection shows current main classifies `stream: "item"` or `"command_output"` events with `phase: "end"` and `status: "failed"` or `"blocked"` as completed because the completed branch runs first.

## Solution Assessment

Yes. The SDK normalizer is the public event-type boundary and producers already carry terminal status, so reordering the existing checks is narrower than changing producers or adding a parallel event type.
