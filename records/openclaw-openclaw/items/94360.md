## Summary

Keep open: current main and the latest release still have source-backed paths that can synthesize user-visible exec/tool-error warnings, and https://github.com/openclaw/openclaw/pull/94374 now explicitly closes this issue, so the issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue asks OpenClaw to keep Feishu exec stderr/tool-error diagnostics out of user-visible replies when the agent's actual response should be delivered.

## Best Possible Solution

Resolve this through a central core payload policy that suppresses raw exec stderr warnings after assistant replies by default, preserves actionable fatal/verbose warnings, and proves Feishu delivery in review of https://github.com/openclaw/openclaw/pull/94374 or its replacement.

## Reproduction Assessment

Yes at source level, but not as a live Feishu replay: current main still builds visible failed-tool warning payloads and Feishu lacks the sibling command-output progress handling that would mark the failure as already surfaced.

## Solution Assessment

Unclear until the open fix candidate is reviewed: suppressing exec warnings after an assistant reply is a plausible core fix, but it needs maintainer confirmation that this is the right failure-visibility policy and live Feishu proof that the reply is no longer covered.
