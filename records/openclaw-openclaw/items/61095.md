## Summary

Keep open. Current main still has the source-level exec environment leak path, and the linked implementation PR is open, unmerged, and conflicting, so this issue should stay open until that PR or a narrow successor lands.

## What This Changes

The issue reports that agent exec-spawned OpenClaw CLI commands inherit Gateway runtime environment markers and fail local WebSocket RPC authentication/handshake.

## Best Possible Solution

Land or replace the linked PR with a narrow host exec sanitizer fix that strips exact internal Gateway/runtime markers, preserves documented service controls and OPENCLAW_SHELL=exec, and adds regression coverage for auth precedence plus clean exit.

## Reproduction Assessment

Yes. Source inspection shows Gateway services set the reported markers, host exec inherits process.env, the current sanitizer policy does not filter those markers, and credential resolution changes behavior when OPENCLAW_SERVICE_KIND=gateway; I did not run a live systemd/Gateway repro in this read-only sweep.

## Solution Assessment

Yes, the host exec environment boundary is the right place to fix this, but the current open PR needs adjustment because blocking the whole OPENCLAW_SERVICE_ prefix would strip documented operator controls.
