## What This Changes

The issue reports that agent exec-spawned `openclaw` CLI commands inherit Gateway service environment markers, causing local gateway RPC authentication or connection failures and sometimes hanging after output.

## Best Possible Solution

Land a rebased sanitizer fix or narrow successor that strips exact internal Gateway runtime marker keys at the host exec boundary, preserves documented service controls and `OPENCLAW_SHELL=exec`, updates parity artifacts, and proves the CLI exits cleanly.

## Reproduction Assessment

Yes, source-level. Gateway service env sets the markers, host exec inherits `process.env`, the current sanitizer leaves them intact, and credential resolution changes behavior when `OPENCLAW_SERVICE_KIND=gateway`.

## Solution Assessment

Yes on the repair boundary, but not on the open PR's exact broad-prefix shape. The maintainable fix is to strip precise internal runtime marker keys at host exec while preserving documented service controls and proving clean CLI exit.
