## Summary

Keep open. Current main still passes the reported Gateway service markers through host exec, and #62204 is an open unmerged implementation candidate with closing syntax, so this should stay open until that PR or a narrow successor lands.

## What This Changes

Prevent agent exec-spawned OpenClaw CLI commands from inheriting Gateway service runtime environment markers that make them behave like the Gateway process.

## Best Possible Solution

Ship a narrow host exec environment sanitizer update that strips Gateway service runtime markers and supervisor unit markers while preserving OPENCLAW_SHELL=exec, with regression coverage for credential precedence and clean child-process exit.

## Reproduction Assessment

Yes. Current main still sets Gateway service markers, exec inherits process.env into host command env, and credential selection branches on OPENCLAW_SERVICE_KIND=gateway, which matches the reporter's reproducible command path.

## Solution Assessment

Yes. Sanitizing internal Gateway markers at the host exec boundary is the narrowest maintainable direction; changing all CLI credential precedence would be broader and could affect legitimate Gateway-service behavior.
