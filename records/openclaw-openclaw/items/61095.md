## Summary

Keep open. Current main and the latest release still expose the reported source path, and an open conflicting PR explicitly closes this issue, so maintainers should review, repair, or replace that PR rather than close this issue or queue a duplicate fix.

## What This Changes

The issue asks OpenClaw to stop agent exec children from inheriting Gateway internal environment markers that make nested CLI commands use the wrong service identity.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/62204 or a narrow successor that strips exact internal Gateway runtime markers at the host exec boundary, preserves OPENCLAW_SHELL=exec and documented service controls, and proves the CLI exits cleanly.

## Reproduction Assessment

Yes, source-level. Gateway service env exports the markers, host exec inherits process.env, the current sanitizer does not strip those markers, and credential resolution changes when OPENCLAW_SERVICE_KIND=gateway.

## Solution Assessment

Yes on the boundary, but not on the open PR's broad-prefix shape. The maintainable fix is exact internal runtime marker stripping at host exec, with explicit preservation of documented service controls and proof that the nested CLI exits cleanly.
