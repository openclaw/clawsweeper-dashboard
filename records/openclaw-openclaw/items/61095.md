## Summary

Keep open. Current main and the latest release still have the reported path: Gateway service markers are exported, agent host exec inherits `process.env`, the host env sanitizer does not strip those markers, and credential selection still changes when `OPENCLAW_SERVICE_KIND=gateway`; the linked closing PR is still open and conflicting.

## What This Changes

Strip Gateway internal service env markers from agent exec-spawned child processes so OpenClaw CLI commands run as clients and exit cleanly.

## Best Possible Solution

Review, revise, and land https://github.com/openclaw/openclaw/pull/62204 or a narrow successor that strips exact internal Gateway runtime markers at the host exec boundary, preserves documented service controls and `OPENCLAW_SHELL=exec`, and proves the CLI command exits cleanly.

## Reproduction Assessment

Yes, source-level. Gateway service env sets the markers, host exec inherits `process.env`, the current sanitizer leaves them intact, and credential resolution changes behavior when `OPENCLAW_SERVICE_KIND=gateway`.

## Solution Assessment

Yes on the boundary, but not on the open PR's exact broad-prefix shape. The maintainable fix is to strip the precise internal runtime marker keys at host exec while preserving documented service controls and proving clean CLI exit.
