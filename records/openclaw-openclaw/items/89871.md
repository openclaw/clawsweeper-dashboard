## Summary

Keep open: current main and v2026.6.8 still lack the requested cross-channel per-channel `replyInThread` setting. ## What This Changes

Request to add an optional per-channel Slack and Discord config setting that forces bot replies into a thread anchored to the triggering message.

## Best Possible Solution

Maintainers should choose one canonical Slack and Discord threading config model, then implement it through the owning channel config/schema/runtime/docs/tests without adding a parallel long-lived alias unless intentionally approved.
