## Summary

Keep open: current main and v2026.6.8 still do not provide the requested cross-channel per-channel `replyInThread` setting. Slack has account/chat-type `replyToMode`, Discord has per-channel `autoThread`, and the open Slack-only PR is partial context rather than a canonical resolution for the broader Slack and Discord config decision.

## What This Changes

The issue requests a per-channel `replyInThread` boolean for Slack and Discord so bot replies can be forced into threads by channel config.

## Best Possible Solution

Maintainers should choose one canonical channel-threading config model, likely by extending Slack per-channel reply mode and deciding whether Discord `autoThread` should be documented, renamed, or aliased, then cover both channels with focused tests and docs.

## Reproduction Assessment

Yes at source level for absence, not as a bug repro: current main and v2026.6.8 have no Slack/Discord `replyInThread` config key, and current behavior is expressed by Slack reply modes and Discord autoThread.

## Solution Assessment

Unclear. The requested boolean is plausible, but the maintainable fix needs a maintainer-approved config/API shape because existing Slack `replyToMode` and Discord `autoThread` concepts already overlap it.
