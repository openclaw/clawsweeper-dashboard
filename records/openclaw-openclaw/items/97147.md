## Risks / Open Questions

- Existing custom or local OpenAI-compatible completions endpoints now enter OpenClaw guarded fetch behavior and may fail closed under SSRF, proxy, timeout, retry, or stream-normalization policy that the SDK default fetch did not enforce.
- Malformed or unusually large streams that exceed the 64 KiB unterminated-SSE cap or 16 MiB synthesized-JSON cap will now fail instead of continuing to read unbounded data; that is intended hardening but still an availability change.
- The shared helper still returns non-OK responses before SSE sanitation, so maintainers should not treat this PR as complete coverage for SDK error-body reads unless that separate path is accepted or tracked.
