## Summary

Keep open for maintainer handling. The useful URL-button preservation part is already shipped, but this PR still proposes automatic direct-chat URL-to-Mini-App conversion, which is a security/product decision and should not merge as-is.

## What This Changes

The PR adds Telegram URL inline button support, maps HTTPS presentation URL buttons in direct chats to Telegram Mini App web_app buttons, adds tests, and documents Mini App constraints.

## Best Possible Solution

Keep the shipped/current-main model: preserve url buttons as links and use explicit webApp/web_app buttons for Telegram Mini Apps; close or replace this branch if maintainers do not want automatic conversion.

## Reproduction Assessment

Yes for the review finding. PR-head source and tests show direct-chat HTTPS presentation url buttons become web_app buttons, while current main and v2026.6.8 keep url and explicit webApp semantics separate.

## Solution Assessment

No. Current main's explicit webApp/web_app presentation action is the narrower maintainable solution; ordinary url links should remain URL buttons unless the producer asks for a Mini App.
