## Summary

Keep open for maintainer review: this is an active feature PR that reintroduces browser Web Speech dictation in Control UI, but current product evidence points toward Gateway/server-side dictation and the supplied proof does not exercise the real browser microphone flow.

## What This Changes

The PR adds a Web Speech API dictation button to the Control UI chat composer and tests supported and unsupported SpeechRecognition behavior.

## Best Possible Solution

Decide whether Control UI should expose browser Web Speech at all; if yes, document the privacy boundary and localize the new control, otherwise route dictation through the existing Gateway/server-side transcription direction.

Not applicable: this is a feature PR, not a bug report. The diff and the related open feature request provide the intended behavior, but there is no broken existing contract to reproduce.
