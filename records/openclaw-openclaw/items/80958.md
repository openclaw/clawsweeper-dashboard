## Summary

This is a user-facing Control UI feature PR with blocking review issues: it introduces direct browser Web Speech dictation where current main intentionally has Talk controls without the stale browser dictation button, and it still needs maintainer product/privacy direction plus stronger browser proof.

## What This Changes

The PR adds a browser Web Speech API dictation button to the Control UI chat composer, streams transcripts into the draft, cleans up dictation state, and adds chat view tests for supported and unsupported browsers.

## Best Possible Solution

Maintainers should decide the intended dictation UX and microphone privacy boundary, then implement it with localized labels, tests, docs, and either the existing Gateway transcription path or an explicitly approved Web Speech path.
