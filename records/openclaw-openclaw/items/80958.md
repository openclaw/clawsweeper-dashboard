## Summary

Keep open: this is an active feature PR, and the patch still needs maintainer product/privacy direction because it adds direct browser SpeechRecognition where current docs and history favor Gateway-mediated voice paths.

## What This Changes

The PR adds a Web Speech API dictation button to the Control UI chat composer, streams recognition results into the draft, resets dictation state on cleanup, and adds UI tests for supported and unsupported speech recognition.

## Best Possible Solution

Decide the browser dictation privacy and UX direction, then either route dictation through the Gateway/Talk transcription path with localized labels and browser proof, or explicitly approve and document Web Speech behavior.

## Solution Assessment

No. Direct browser SpeechRecognition is not the best maintainable path without a product/privacy decision, and the new visible labels should be added through the locale surface.
