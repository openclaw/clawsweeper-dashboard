## Summary

Keep open. Current main and the latest release already inject fixed workspace bootstrap files and have a bounded bare `/new`/`/reset` daily-memory prelude, but they still do not enforce arbitrary AGENTS.md startup steps or load custom files such as `memory/session-handoff.md` as a general session-start contract.

## What This Changes

The issue asks OpenClaw to auto-inject or enforce custom startup context files before an agent answers the first user message.

## Best Possible Solution

Define a bounded startup-context/bootstrap contract: keep fixed bootstrap injection, route arbitrary in-workspace startup files through the canonical bootstrap-file API discussion, route directory-relative AGENTS.md / CLAUDE.md through the directory-context issue, and expose diagnostics showing exactly what was loaded.

## Reproduction Assessment

Partially. Source inspection gives a high-confidence reproduction for the remaining gap: current main does not load `memory/session-handoff.md`, does not enforce arbitrary AGENTS.md startup steps, and only applies recent daily-memory prelude to the bare reset/new path.

## Solution Assessment

Unclear as a final product shape. Auto-injecting fixed bootstrap files already exists; the safer path is maintainer-approved bounded startup-file configuration and diagnostics rather than treating arbitrary AGENTS.md instructions as executable runtime policy.
