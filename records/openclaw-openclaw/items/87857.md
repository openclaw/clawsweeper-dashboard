## Summary

Keep open. Current main and the latest release inject recognized workspace bootstrap files and a bounded daily-memory prelude for bare new/reset flows, but they still do not provide runtime-enforced ordered startup actions or arbitrary per-agent first-turn file loading.

## What This Changes

The issue requests runtime-enforced startup context loading or ordered startup actions before an agent answers the first user message.

## Best Possible Solution

Define a bounded startup-context contract with explicit file/action semantics, diagnostics, and privacy/session filters, while keeping arbitrary AGENTS.md prose non-executable.

## Reproduction Assessment

Yes, at source level. Current main has no per-agent startupCommands/init hook schema, rejects arbitrary extra bootstrap basenames, and only builds daily-memory startup context for bare new/reset handling.

## Solution Assessment

Unclear as a final product shape. The maintainable direction is a bounded startup-context/config contract with diagnostics and privacy controls, not automatic execution of arbitrary AGENTS.md text.
