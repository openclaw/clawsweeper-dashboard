## Summary

Keep open. Current main and v2026.6.8 still provide fixed workspace bootstrap injection and a bounded daily-memory prelude for bare reset/new model runs, but they do not enforce ordered AGENTS.md startup actions or accept a per-agent startup command/file contract before the first user message.

## What This Changes

Request a runtime-enforced session startup mechanism so agent-specific AGENTS.md/SOP memory files are loaded or actions run before the first user turn.

## Best Possible Solution

Define a bounded startup-context contract in core or plugin hooks, with diagnostics for exactly which files/actions ran before the first user turn.

## Reproduction Assessment

Partially. Source inspection gives a high-confidence path for the remaining gap: current main does not define per-agent startup commands and only injects fixed bootstrap files plus a bounded daily-memory prelude on reset/new model runs.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers need to choose between bounded config, deterministic bootstrap files, plugin hooks, or diagnostics before implementation.
