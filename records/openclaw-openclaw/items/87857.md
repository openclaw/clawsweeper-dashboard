## Summary

Keep open. Current main already injects canonical workspace bootstrap files and has bounded daily-memory startup context for reset/new flows, but it still does not enforce arbitrary AGENTS.md startup steps or load a custom `memory/session-handoff.md` as a general session-start contract.

## What This Changes

The issue requests runtime-enforced loading of AGENTS.md-directed startup context files before an agent answers the first user message.

## Best Possible Solution

Define one bounded startup-context contract that keeps canonical bootstrap files deterministic and routes custom handoff or project files through the existing bootstrap-extra-files/arbitrary-file work instead of parsing arbitrary AGENTS.md prose as executable runtime policy.

## Reproduction Assessment

Yes at source level for the remaining gap: current code has fixed bootstrap and bounded startup daily memory paths, but no general AGENTS.md startup-step enforcement or arbitrary `memory/session-handoff.md` bootstrap loader. I did not reproduce the original WhatsApp/Sunny session live.

## Solution Assessment

Unclear; enforcing arbitrary AGENTS.md startup prose before every first reply is not obviously the best fix. A bounded, typed startup-context or bootstrap-extra-files contract is safer than making runtime parse freeform instructions as policy.
