## Summary

Keep open. Current main already injects the fixed workspace bootstrap files and has a bare `/new`/`/reset` daily-memory prelude, but the requested deterministic AGENTS.md startup-sequence enforcement and custom `memory/session-handoff.md` loading are not implemented as a general session-start contract.

## What This Changes

The issue requests runtime enforcement or automatic injection of workspace startup context files referenced by AGENTS.md before an agent can answer the first user message.

## Best Possible Solution

Define a bounded startup-context contract that keeps fixed bootstrap files in Project Context, optionally supports configured extra startup files such as a handoff note, and adds diagnostics so users can prove what was injected without parsing arbitrary AGENTS.md prose as runtime policy.

## Reproduction Assessment

Partially. Source inspection shows current main does not enforce arbitrary AGENTS.md startup steps or include `memory/session-handoff.md` in the fixed bootstrap set, but I did not reproduce the reported WhatsApp/Sunny failure in a live session.

## Solution Assessment

Unclear. Auto-injecting fixed bootstrap files already exists; a safer next step is a maintainer-approved, bounded startup-context surface rather than trying to parse and enforce arbitrary AGENTS.md instructions before every reply.
