## What This Changes

The PR adds `retain: true` to the Telegram generation-mismatch superseded-preview callback and updates the in-flight `forceNewMessage()` race test to expect that retention contract.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes for the code shape: adding `retain: true` aligns the generation-mismatch path with the existing overflow retained-preview path and the regression test targets the race. The remaining gap is proof quality, not a different implementation.
