## Summary

Keep open: current main still only persists indexed token totals for cron run logs, while task and subagent run-table accounting remains missing and needs a maintainer decision on the permanent token/cost contract.

## What This Changes

Request to persist token and optionally cost totals on task and subagent run records, matching cron run log accounting.

## Best Possible Solution

Maintainers should choose and document the canonical per-run accounting contract, then implement additive SQLite columns, migration guards, generated types, store mappings, finalization-time population, and focused regression coverage for task and subagent runs.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the gap: cron run logs expose `total_tokens`, while task and subagent run tables, generated types, and store mappings lack equivalent token or cost fields.

## Solution Assessment

Unclear. Nullable run-table columns are a plausible additive direction, but the best fix depends on maintainer choice for cost persistence and whether finalization should read session totals or receive usage explicitly.
