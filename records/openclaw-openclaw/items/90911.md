## Summary

Keep open: current main and v2026.6.6 still persist indexed token totals only for cron run logs, while task and subagent run-table accounting remains missing and needs a maintainer schema/source-of-truth decision.

## What This Changes

The issue asks OpenClaw to persist token and optional cost accounting on task and subagent run tables with parity to cron run logs.

## Best Possible Solution

Maintainers should choose the canonical per-run accounting contract, then implement additive SQLite columns, migration guards, generated types, store mappings, finalization-time population, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the gap: cron run logs persist total_tokens, while task_runs and subagent_runs lack token or cost columns and their store mappings do not carry those fields.

## Solution Assessment

Unclear. Nullable run-table columns are a plausible additive direction, but the best fix depends on a maintainer decision about cost persistence and whether finalization should read session totals or receive usage explicitly.
