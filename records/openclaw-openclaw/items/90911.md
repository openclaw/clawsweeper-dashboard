## Summary

Keep open: current main and the latest release still persist indexed token totals only for cron run logs, while task and subagent run tables lack equivalent token/cost fields and need a maintainer choice on the durable accounting contract.

## What This Changes

Request to persist nullable per-run token and optional cost accounting on `task_runs` and `subagent_runs`, matching `cron_run_logs.total_tokens` for hook-driven and subagent runs.

## Best Possible Solution

Maintainers should choose the canonical run-accounting contract, then implement additive SQLite columns, migration guards, generated types, store mappings, finalization-time population, and focused regression coverage for task and subagent runs.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the gap: cron run logs expose `total_tokens`, while task and subagent run tables, generated types, and store mappings lack equivalent token or cost fields.

## Solution Assessment

Unclear. Nullable run-table columns are a plausible additive path, but the best fix depends on maintainer decisions about cost persistence and whether finalization should read session snapshots or persist aggregate run usage explicitly.
