## Summary

Keep open: current main and the latest release still persist indexed token totals only for cron run logs, while task and subagent run records lack equivalent token or cost fields and the durable accounting contract needs maintainer direction.

## What This Changes

Request persisting per-run token and optional cost totals on `task_runs` and `subagent_runs`, matching `cron_run_logs.total_tokens` for hook-driven and subagent runs.

## Best Possible Solution

Define the canonical run-accounting contract first, then implement additive SQLite columns, migration guards, generated types, store mappings, finalization-time population, and focused regression coverage for both task and subagent runs.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.8` shows `cron_run_logs.total_tokens` exists while `task_runs` and `subagent_runs` lack token or cost fields in schema, generated types, migration guards, and store mappings.

## Solution Assessment

Unclear. Nullable run-table columns are a plausible additive path, but maintainers still need to choose cost persistence and whether values come from session snapshots or explicitly plumbed aggregate run usage.
