## Summary

Keep open. Current main still does not expose the requested persisted config opt-out and package-level bundled runtime dependency repair still defaults to the shared install root; the related open PR covers only the opt-out piece and leaves the isolated-root default and channel-scope questions unresolved.

## What This Changes

Request to expose `plugins.installBundledRuntimeDeps: false`, make bundled runtime dependency staging use an isolated execution root by default, and document the resulting operator-managed dependency matrix.

## Best Possible Solution

Land a cohesive config, runtime-deps, gateway, docs, test, and changelog update: use #75511 or its replacement for the opt-out seam, add isolated package-level install execution by default where appropriate, and explicitly define how bundled channel staging follows the same or a sibling policy.

## Reproduction Assessment

Yes. The code-level reproduction is high-confidence: inspect current main and observe that the public schema rejects the requested key, gateway prestage has no config-driven skip, and package-level repair omits an isolated execution root. This review did not run live staging against the reported production deployment.

## Solution Assessment

Yes for the overall direction, but only as a complete change. Exposing the existing loader seam is narrow and maintainable, while the default isolated execution root and channel behavior should be settled before treating the issue as fixed.
