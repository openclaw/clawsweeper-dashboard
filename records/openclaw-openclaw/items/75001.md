## Summary

Keep open. Current main still filters Feishu broadcast-only `@all`/`@_all` before mention-gated dispatch, while docs and regression tests encode that as the shipped #37706 false-positive protection, so this needs Feishu policy judgment rather than cleanup closure.

## What This Changes

Request that Feishu broadcast `@all`/`@_all` mentions remain visible to agents, or become prompt/config controlled, instead of being filtered by channel-level mention detection.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention gate while preserving the shipped false-positive protection and documenting whichever default or explicit policy knob maintainers choose.

## Reproduction Assessment

Yes, as a source-level reproduction. Current source and focused tests show broadcast-only `@_all`, `@all` metadata, and rich-post `user_id: all` produce `mentionedBot=false`, and mention-gated groups then return before dispatch.

## Solution Assessment

No for the requested direct revert as written. A safer fix would preserve broadcast-only false-positive protection while adding explicit context or configuration if maintainers want agents to reason about Feishu `@all`.
