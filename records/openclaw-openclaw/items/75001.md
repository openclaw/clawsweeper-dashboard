## Summary

Keep open. Current main still intentionally filters Feishu broadcast-only `@all`/`@_all` out of the bot-mention gate, so the reported behavior is real, but changing that default conflicts with the shipped false-positive fix and needs Feishu maintainer policy direction.

## What This Changes

The issue asks OpenClaw to stop dropping Feishu broadcast `@all`/`@_all` mentions at the channel gate and instead expose them to agent prompt policy or configuration.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention gate, preserve the #37706 false-positive protection, and document/test the chosen default or explicit Feishu config option.

## Reproduction Assessment

Yes. Current source and focused tests provide a high-confidence static reproduction path: broadcast-only `@_all`, `@all` metadata, and rich-post `user_id: all` produce `mentionedBot=false`, and mention-gated groups return before dispatch.

## Solution Assessment

No for the requested direct revert as written. A safer solution would keep broadcast-only false-positive protection while adding explicit context or configuration if maintainers want agents to reason about Feishu `@all`.
