## Summary

Keep open: current main and latest release still intentionally filter Feishu broadcast-only `@all`/`@_all` from bot mention detection, while this issue asks maintainers to change that Feishu policy so agents can see or respond to broadcast summons through a prompt/context/config path.

## What This Changes

Request to stop dropping Feishu broadcast-only `@all`/`@_all` at the channel mention gate and instead handle suppression through agent prompt policy or an opt-in Feishu config surface.

## Best Possible Solution

Decide the Feishu broadcast-mention contract explicitly, then implement one documented and tested path that preserves the safe default while allowing agent awareness only through a maintainer-approved prompt/context or opt-in policy surface.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows broadcast-only Feishu `@all`/`@_all` yields `mentionedBot=false`, and mention-gated group handling returns before dispatch; no live Feishu tenant replay was attempted in this read-only review.

## Solution Assessment

Unclear. Reverting the filter would satisfy the reporter's use case but is not clearly the best fix because it would undo the safe-default behavior from the merged Feishu @all fix; maintainers should choose prompt context, opt-in config, or the current channel gate deliberately.
