## Summary

Keep open. Current main still intentionally treats Feishu broadcast-only `@all`/`@_all` as not mentioning the bot, and the remaining question is a Feishu policy/config decision rather than a cleanup-close candidate.

## What This Changes

Request to stop Feishu broadcast `@all`/`@_all` mentions from being filtered by mention gating and instead expose them to the agent prompt or a config-controlled policy.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention gate, preserve the #37706 false-positive protection, and document/test the chosen default or config option.

## Reproduction Assessment

Yes. Current source and regression tests provide a high-confidence static reproduction path: broadcast-only `@_all`, `@all` metadata, and post `user_id: all` produce `mentionedBot=false`, and mention-gated groups return before dispatch.

## Solution Assessment

No for the requested direct revert as written. A safer solution would preserve broadcast-only false-positive protection while adding explicit context or a config path if maintainers want agents to reason about `@all`.
