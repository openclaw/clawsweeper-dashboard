## Summary

Keep open. Current main still intentionally treats Feishu broadcast-only @all/@_all as not mentioning the bot, and this issue is challenging that live Feishu channel policy rather than describing obsolete behavior.

## What This Changes

Request to change Feishu/Lark @all/@_all handling so broadcast mentions are delivered as context and controlled by prompt or config instead of being hard-filtered from bot mention detection.

## Best Possible Solution

Keep this open for Feishu channel policy. The maintainable end state would separate broadcast mention context from the boolean bot-mention gate, then document and test whether broadcast-only messages are ignored by default, passed to the agent as context, or controlled by an explicit config or prompt opt-in.

## Reproduction Assessment

Yes. Construct a Feishu group event with broadcast-only @all/@_all mention metadata and requireMention=true; current parsing produces mentionedBot=false and the group handler returns before dispatch. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Unclear for the issue's requested direct revert. A safer solution would preserve the #37706 false-positive protection while adding an explicit broadcast-mention context/config path if maintainers want agents to reason about @all.
