## Summary

Keep open. Current main still intentionally excludes Feishu broadcast-only `@all`/`@_all` from the bot-mention gate and mention-forward targets, and mention-gated groups return before dispatch when that gate is false. The report is a live Feishu channel policy conflict with the shipped #37706 fix, not an obsolete item or a safe close candidate.

## What This Changes

The issue asks OpenClaw to stop dropping Feishu broadcast `@all` mentions at the channel API layer and instead expose them to agent prompt policy or an explicit Feishu config option.

## Best Possible Solution

Keep the issue open for an explicit Feishu channel policy decision. The maintainable end state should preserve the #37706 protection that broadcast-only `@all` does not equal a direct bot mention, while adding a separate prompt-visible broadcast context or documented config option if maintainers want agents to reason about `@all` without treating it as the mention gate.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence static reproduction: broadcast-only `@_all`, `@all` metadata, and post `user_id: all` produce `mentionedBot=false`, and mention-gated groups return before dispatch. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No for the requested direct revert as written. A safer solution would separate broadcast mention awareness from the boolean direct-bot-mention gate, then document and test whether Feishu `@all` is ignored by default, prompt-visible only, or controlled by a config option.
