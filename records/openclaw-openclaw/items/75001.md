## Summary

Keep open. Current main still treats Feishu broadcast-only `@all`/`@_all` as not a bot mention and mention-gated groups return before agent dispatch, but docs/tests/changelog encode that as the shipped #37706 false-positive protection, so changing it needs Feishu policy review rather than cleanup closure.

## What This Changes

Request that Feishu broadcast `@all`/`@_all` mentions reach the agent or prompt layer instead of being hard-filtered by the mention gate, optionally with explicit policy control.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention activation gate, preserve #37706 false-positive protection, and document/test the chosen default or explicit policy control.

## Reproduction Assessment

Yes. Source inspection and focused tests show broadcast-only `@all`/`@_all` and rich-post `user_id: all` yield `mentionedBot=false`, and `bot.ts` returns before dispatch when `requireMention` is true.

## Solution Assessment

No for the requested direct revert. A safer fix would decouple broadcast visibility from bot-mention activation, or add an explicit policy, while keeping direct bot mentions and the #37706 false-positive protection intact.
