## Summary

Keep open. Current main and the latest shipped Feishu docs still intentionally treat broadcast-only `@all`/`@_all` as not being a bot mention, with tests covering that false-positive protection, so changing it needs Feishu channel policy review rather than cleanup closure.

## What This Changes

Request to change Feishu broadcast `@all` handling so agents can see or respond based on prompt or explicit configuration instead of the channel mention gate dropping broadcast-only mentions.

## Best Possible Solution

Define a Feishu-owned `@all` policy that preserves the default false-positive protection while explicitly documenting or configuring when broadcast mentions are visible to agents and allowed to satisfy activation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: broadcast-only Feishu `@all`/`@_all` produces `mentionedBot=false`, and mention-gated group handling returns before agent dispatch.

## Solution Assessment

No for a direct revert. The safer maintainable direction is to separate broadcast mention visibility from the activation gate or add an explicit opt-in policy/config surface while keeping the shipped false-positive protection by default.
