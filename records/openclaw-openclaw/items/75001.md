## Summary

Keep open: current main and the latest release still intentionally treat broadcast-only Feishu @all/@_all as not mentioning the bot, while this request asks maintainers to change that policy or add an escape hatch.

## What This Changes

Request that Feishu broadcast @all/@_all mentions be surfaced to the agent or controlled by an explicit policy/config surface instead of being filtered at mention detection.

## Best Possible Solution

Decide the Feishu broadcast-mention contract explicitly, then implement one documented and tested path that preserves safe defaults while allowing agent awareness only through a maintainer-approved prompt or policy surface.

## Reproduction Assessment

Yes, source-level. Current source and tests show broadcast-only Feishu @all sets mentionedBot=false, and the mention-gated group path returns before dispatch when requireMention is true.

## Solution Assessment

Unclear. Reverting the filter would satisfy this requested behavior but may undo the safe-default behavior from the earlier Feishu @all fix; maintainers should choose prompt context, opt-in config, or the current channel gate deliberately.
