## Summary

Keep open. Current main still intentionally filters Feishu broadcast-only `@all`/`@_all` out of the bot-mention gate and dispatch path, while docs and tests encode that behavior, so the remaining question is a Feishu product policy decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to make Feishu `@all` broadcasts visible to agents instead of suppressing them in the channel mention gate.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention gate, preserve the #37706 false-positive protection, and document/test the chosen Feishu `@all` policy or explicit opt-in behavior.

## Reproduction Assessment

Yes. Current source and focused tests provide a high-confidence static reproduction path: broadcast-only `@_all`, `@all` metadata, and rich-post `user_id: all` produce `mentionedBot=false`, and mention-gated groups return before dispatch.

## Solution Assessment

No for the requested direct revert. A safer solution would preserve broadcast-only false-positive protection while adding explicit context or configuration if maintainers want agents to reason about Feishu `@all`.
