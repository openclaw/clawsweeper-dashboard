## Summary

Keep open: current main and `v2026.6.8` still suppress Feishu broadcast-only `@all` / `@_all` under mention gating, and related closed reports now point back here as the canonical policy question.

## What This Changes

The issue asks OpenClaw to stop Feishu channel-layer suppression of broadcast-only `@all` / `@_all` so agents can see or respond via prompt or an explicit policy.

## Best Possible Solution

Decide the Feishu broadcast-mention contract explicitly, then implement one documented and tested path that preserves safe defaults while allowing agent awareness only through a maintainer-approved prompt or policy surface.

## Reproduction Assessment

Yes. Source inspection shows broadcast-only Feishu `@all` sets `mentionedBot=false`, mention-gated group activation returns before dispatch, and focused tests assert that behavior.

## Solution Assessment

Unclear. Reverting the filter would satisfy the reporter's use case but may undo the safe-default behavior from the merged Feishu @all fix; maintainers should choose prompt context, opt-in config, or the current channel gate deliberately.
