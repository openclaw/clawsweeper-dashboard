## Summary

Keep open. Current main still intentionally filters Feishu broadcast-only `@all`/`@_all` before the message can satisfy mention-gated dispatch, and the source, tests, docs, changelog, and latest release all encode that policy. The related composite issue was closed in favor of this canonical policy thread, so this needs Feishu maintainer judgment rather than cleanup.

## What This Changes

The issue asks OpenClaw to preserve Feishu broadcast `@all`/`@_all` mentions for agent context and handle suppression through prompt or config policy instead of filtering them in the Feishu mention gate.

## Best Possible Solution

Keep this open for Feishu channel maintainer policy. The maintainable end state would model broadcast mentions as a distinct channel fact that can be passed into context, while separately documenting and testing whether broadcast-only messages are ignored by default, context-only, or controlled by explicit config/prompt opt-in.

## Reproduction Assessment

Yes. A fixture-style Feishu group event with broadcast-only `@_all`/`@all` reaches `mentionedBot=false`; with `requireMention=true`, `handleFeishuEvent` returns before dispatch. I did not run tests because this review had to keep the checkout read-only.

## Solution Assessment

No for the requested direct revert as written. A safer solution would preserve the #37706 false-positive protection while adding a distinct broadcast-mention context/config path if maintainers want agents to reason about `@all`.
