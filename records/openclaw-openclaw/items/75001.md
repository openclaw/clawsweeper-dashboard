## Summary

Keep open. Current main still intentionally filters Feishu broadcast-only `@all`/`@_all` out of bot mention detection and mention-forward targets, so the reported behavior is real and needs Feishu policy judgment rather than cleanup closure.

## What This Changes

The issue asks for Feishu `@all`/`@_all` mentions to remain visible to agents instead of being suppressed at the channel mention-detection layer, possibly via prompt guidance or config.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention gate, preserve the #37706 false-positive protection, and document/test either the chosen default or an explicit Feishu config option.

## Reproduction Assessment

Yes. Current source and focused tests provide a high-confidence static reproduction path: broadcast-only `@_all`, `@all` metadata, and rich-post `user_id: all` produce `mentionedBot=false`, and mention-gated groups return before dispatch.

## Solution Assessment

No for the requested direct revert as written. A safer solution would preserve broadcast-only false-positive protection while adding explicit context or a config path if maintainers want agents to reason about Feishu `@all`.
