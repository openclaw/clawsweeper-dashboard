## Summary

Keep open. Current main still intentionally excludes Feishu broadcast-only `@all`/`@_all` from the bot-mention gate, and the source, tests, docs, and changelog all encode that behavior; the remaining question is a live Feishu channel policy decision, not an obsolete or already-fixed report.

## What This Changes

The issue requests changing Feishu `@all` handling so broadcast mentions are preserved for agent or prompt-level handling instead of being filtered by channel-level mention gating.

## Best Possible Solution

Keep the issue open for Feishu channel policy. The maintainable end state would separate broadcast mention context from the boolean bot-mention gate, preserve the #37706 false-positive protection, and then document and test the chosen behavior as either default ignore, prompt-visible context, or an explicit config option.

## Reproduction Assessment

Yes. Current source and regression tests provide a high-confidence reproduction path: broadcast-only `@_all`, `@all` metadata, and post `user_id: all` produce `mentionedBot=false`, and mention-gated groups return before dispatch. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No for the requested direct revert as written. A safer solution would keep broadcast-only messages from satisfying the bot-mention gate while adding an explicit context or config path if maintainers want agents to reason about `@all`.
