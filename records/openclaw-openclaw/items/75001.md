## Summary

Keep open. Current main still intentionally filters Feishu broadcast-only `@all`/`@_all` before mention-gated dispatch, so the reported behavior is real and needs Feishu policy judgment rather than cleanup closure.

## What This Changes

Request that Feishu broadcast `@all` mentions remain visible to agents, or become configurable/prompt-controlled, instead of being dropped by channel-level mention detection.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention gate, preserve the #37706 false-positive protection, and document/test either the chosen default or an explicit Feishu config option.

## Reproduction Assessment

Yes. Current source and focused tests provide a high-confidence static reproduction path: broadcast-only `@_all`, `@all` metadata, and rich-post `user_id: all` produce `mentionedBot=false`, and mention-gated groups return before dispatch.

## Solution Assessment

No for the requested direct revert as written. A safer solution would preserve broadcast-only false-positive protection while adding explicit context or configuration if maintainers want agents to reason about Feishu `@all`.
