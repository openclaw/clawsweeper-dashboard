## Summary

Keep open. Current main still filters Feishu broadcast-only `@all`/`@_all` out of bot-mention detection and skips mention-gated dispatch, but docs and tests encode that as the shipped #37706 false-positive protection, so changing it needs Feishu policy review rather than cleanup closure.

## What This Changes

Requests changing Feishu `@all`/`@_all` handling so broadcast mentions can reach agent context or be controlled by prompt/config instead of being filtered by the channel mention gate.

## Best Possible Solution

Separate broadcast mention visibility from the boolean bot-mention activation gate, preserve the #37706 false-positive protection, and document/test the chosen Feishu default or explicit policy control.

## Reproduction Assessment

Yes. Source inspection shows broadcast-only `@all`/`@_all` yields `mentionedBot=false`, and mention-gated groups return before dispatch when that flag is false; focused tests codify the same cases.

## Solution Assessment

No for the requested direct revert. A safer solution would decouple broadcast visibility from bot-mention activation, or add an explicit Feishu policy, while keeping direct bot mentions and the #37706 false-positive guard intact.
