## Summary

Keep open: current main and the latest stable release still treat Feishu broadcast-only `@all` / `@_all` as not mentioning the bot, so the unresolved work is the intended Feishu broadcast-mention policy rather than cleanup that can be closed.

## What This Changes

The issue asks OpenClaw to stop dropping Feishu `@all` mentions at the channel layer and instead expose them to agents, suppress them via prompt policy, or provide an operator-controlled setting.

## Best Possible Solution

Decide the Feishu broadcast-mention contract explicitly, then implement one documented and tested path that preserves safe defaults while allowing agent awareness only through a maintainer-approved prompt or config policy.

## Reproduction Assessment

Yes, from source inspection: broadcast-only Feishu @all sets `mentionedBot=false`, mention-gated group activation returns before dispatch, and focused tests assert that behavior. I did not run a live Feishu event because this read-only cleanup review can establish the behavior from current source, docs, tests, and release contents.

## Solution Assessment

Unclear: reverting the channel-level filter would satisfy the reporter's use case, but it may undo the safe-default behavior from the merged Feishu @all fix. The best fix needs maintainer approval for whether prompt context, an opt-in config surface, or the current channel gate should own broadcast suppression.
