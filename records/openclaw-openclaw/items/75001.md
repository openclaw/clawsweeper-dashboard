## Summary

Keep open: current main and the latest stable release still treat broadcast-only Feishu `@all` / `@_all` as not mentioning the bot, so the remaining question is the intended Feishu broadcast-mention policy and escape-hatch shape.

## What This Changes

The issue asks for Feishu broadcast-only `@all` / `@_all` mentions to reach agent context, or for suppression to become an explicit prompt/config policy instead of a hard channel gate.

## Best Possible Solution

Maintainers should decide the Feishu broadcast-mention contract, then implement one documented and tested Feishu-plugin path that preserves safe defaults while allowing agent awareness only through the approved prompt or config policy.

## Reproduction Assessment

Yes. Source inspection shows broadcast-only Feishu `@all` sets `mentionedBot=false`, mention-gated group activation returns before dispatch, and focused tests assert that behavior; I did not run a live Feishu event in this read-only review.

## Solution Assessment

Unclear. Reverting the channel-level filter would satisfy the reporter's use case, but it conflicts with the documented safe default from the merged Feishu `@all` fix; the best fix needs maintainer approval for prompt context, opt-in config, or keeping the channel gate.
