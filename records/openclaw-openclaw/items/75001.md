## Summary

Keep open: current main and v2026.6.10 still intentionally treat Feishu broadcast-only @all/@_all as not mentioning the bot, so the requested prompt/config-aware behavior is not implemented. The remaining work is a Feishu product-policy decision because changing the gate could reopen the noisy all-bot reply behavior that the merged Feishu fix addressed.

## What This Changes

The issue requests changing Feishu @all handling so broadcast mentions are not silently filtered by mention gating and can instead be handled by prompt or configuration policy.

## Best Possible Solution

Maintainers should define the Feishu broadcast-mention contract explicitly, keep the safe default unless they choose otherwise, and add a documented opt-in path only if agent awareness of @all is a supported use case.

## Reproduction Assessment

Yes. Source inspection shows broadcast-only Feishu @all sets mentionedBot=false and the mention-gated group path returns before dispatch; focused tests and the latest release tag assert the same behavior.

## Solution Assessment

Unclear. Reverting the filter would satisfy the reporter's use case but may undo the safe-default behavior from the merged Feishu @all fix; a prompt/config-aware path needs maintainer-confirmed product direction first.
