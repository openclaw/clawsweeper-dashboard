## Summary

Keep open. Current main still intentionally filters Feishu broadcast-only `@all`/`@_all` out of the bot-mention activation gate, and the docs/tests encode that behavior as the shipped false-positive protection, so changing it needs Feishu channel policy judgment rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop dropping Feishu broadcast `@all` mentions at the channel layer and instead expose them to the agent prompt or make suppression configurable.

## Best Possible Solution

Define a Feishu policy that separates broadcast mention visibility from the bot activation gate, preserving false-positive protection while documenting or configuring whether agents may see and react to broadcast summons.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: broadcast-only Feishu `@all`/`@_all` yields `mentionedBot=false`, and mention-gated group handling returns before agent dispatch.

## Solution Assessment

No for a direct revert. The safer direction is to decouple broadcast visibility from activation or add an explicit opt-in policy, while preserving the shipped false-positive protection for broadcast-only group notifications.
