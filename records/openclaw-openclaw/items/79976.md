## What This Changes

Adds `Copilot-Integration-Id: vscode-chat` to the GitHub Copilot `/v2/token` request and includes a large `CHANGELOG.md` update.

## Best Possible Solution

Keep the fix in the Copilot auth boundary, update regression coverage for the token exchange, account for old cached tokens if maintainers want immediate post-upgrade behavior, and require redacted live proof before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level reproduction: `origin/main` exchanges the Copilot token without `Copilot-Integration-Id` while runtime and `/models` Copilot requests use `vscode-chat`, and the linked issue has the observed HTTP 400. I did not run a live subscription-backed image request in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Partly: adding the header in `resolveCopilotApiToken` is the narrowest maintainable fix, but the PR should update the exact header assertion, consider old token cache behavior, and supply live proof before merge.
