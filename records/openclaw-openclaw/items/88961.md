## Best Possible Solution

Derive pre-registration known entries from canonical discovered and active plugin facts plus any configured MCP namespace contract, keep true missing or typo IDs warning, and narrow the linked issue reference before merge.

## Reproduction Assessment

Yes, at source level: current main classifies allowlist entries against only current core/plugin tool groups, so deferred plugin or configured MCP entries can be warned as unknown before materialization. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The maintainable fix should consume canonical discovery and activation facts rather than reconstructing partial plugin enablement from raw config in the embedded runner.
