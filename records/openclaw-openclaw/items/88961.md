The PR threads known plugin ids into agent tool-policy allowlist analysis so enabled plugin-only entries are not warned as unknown before plugin tools are registered.

## Best Possible Solution

Derive known plugin ids from the same effective plugin activation semantics OpenClaw already uses, preserve typo and unavailable-core warnings, and add focused coverage for `plugins.allow` plus blocked-plugin cases before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows the bundled final policy pass can analyze an allowlist before plugin tools are present, and the analyzer only knows plugin ids from registered tool metadata.

## Solution Assessment

No. The patch is in the right general layer, but collecting only `plugins.entries.<id>.enabled` is not the best fix because OpenClaw already has broader effective plugin activation semantics for `plugins.allow`, deny, and disabled states.
