## Summary

Keep open. Current main still has a source-reproducible Bedrock AWS SDK credential refresh gap: docs promise shared-credential default-chain auth, but the `aws-sdk` sentinel path has no runtime auth state, and the AWS SDK shared INI loader caches credential-file reads unless explicitly bypassed.

## What This Changes

The issue asks Bedrock `aws-sdk` auth to pick up refreshed shared AWS credentials without restarting the gateway after session-token rotation.

## Best Possible Solution

Add provider-owned Bedrock `aws-sdk` credential refresh or cache invalidation that re-reads shared AWS credentials on auth failures while preserving the default-chain contract and avoiding a new config knob.

## Reproduction Assessment

Yes, source-reproducible with high confidence: the AWS SDK shared INI loader caches credential file reads, and OpenClaw's Bedrock `aws-sdk` sentinel path cannot trigger runtime-auth refresh on auth errors. A live AWS rotation run was not performed, so this is not marked live-reproduced.

## Solution Assessment

No current-main fix is present. The narrowest maintainable direction is a Bedrock-owned refresh/cache-bypass path for rotated shared credentials, with regression coverage for credentials files that omit `expiration`.
