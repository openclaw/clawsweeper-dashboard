## Summary

Keep open. Current main still leaves Bedrock `aws-sdk` auth on the sentinel path with no runtime-auth state, while auth-error refresh is gated on that state; the linked open PR is the right implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks Bedrock `aws-sdk` auth to pick up rotated shared AWS credentials without restarting the gateway.

## Best Possible Solution

Land #77593 or an equivalent provider-owned Bedrock runtime-auth fix with regression coverage, then close this issue from the merged implementation.

## Reproduction Assessment

Yes for the OpenClaw-side refresh gap: source inspection shows Bedrock `aws-sdk` auth can fall back to a sentinel with null runtime-auth state, and auth-error refresh then cannot run. The full AWS `ExpiredTokenException` rotation path was not live-reproduced here.

## Solution Assessment

Yes, the narrow maintainable direction is a Bedrock-owned runtime-auth refresh or client recreation path on auth failures, without adding a user-facing config knob. The linked open PR appears to target that boundary, so review and land or replace that PR rather than closing the issue early.
