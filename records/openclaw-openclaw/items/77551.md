## Summary

Keep open. The report matches a real Bedrock auth surface on current main: docs promise AWS SDK default-chain auth for Bedrock, but the `aws-sdk` sentinel path leaves no refreshable runtime-auth state, so OpenClaw's auth-error refresh hook cannot help when the underlying shared credentials go stale.

## What This Changes

The issue requests that Bedrock `aws-sdk` authentication re-read refreshed shared AWS credentials without requiring a gateway restart after token rotation.

## Best Possible Solution

Keep the AWS SDK default-chain contract, but make the Amazon Bedrock provider/runtime path invalidate or recreate stale shared-credential resolution on auth-class failures without adding a user-facing config knob.

## Reproduction Assessment

Yes for the OpenClaw-side refresh gap, from source inspection: `aws-sdk` sentinel auth clears `runtimeAuthState`, and auth-error refresh is gated on that state. A live AWS rotation repro was not run, so the full ExpiredTokenException path remains source-reproducible rather than live-reproduced.

## Solution Assessment

No current-main fix is present. The narrowest maintainable direction is provider-owned Bedrock credential refresh/recreation on auth failures, with regression coverage for shared credentials that rotate without an expiration field.
