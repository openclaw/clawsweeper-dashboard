## Summary

Keep open. Current main still has a source-reproducible Bedrock `aws-sdk` refresh gap, and the linked open PR #77593 uses closing syntax, so this issue should stay open until that PR or an equivalent fix is merged.

## What This Changes

The issue requests that Bedrock `aws-sdk` auth re-read refreshed shared AWS credentials instead of requiring a gateway restart after session-token rotation.

## Best Possible Solution

Land a Bedrock-owned credential refresh fix that preserves AWS SDK default-chain auth and adds regression coverage, then close once the linked PR or an equivalent fix is merged.

## Reproduction Assessment

Yes for the OpenClaw-side refresh gap from source inspection: Bedrock has no runtime-auth hook, sentinel fallback clears `runtimeAuthState`, and auth-error refresh is gated on that state. A live AWS rotation run was not performed, so the full external credential-expiration path is not live-reproduced here.

## Solution Assessment

No current-main fix is present. The narrowest maintainable solution is a provider-owned Bedrock runtime-auth refresh/recreation path on auth failures, with tests for shared credentials that rotate without adding a user-facing config knob.
