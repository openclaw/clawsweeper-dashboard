## What This Changes

The issue asks OpenClaw to refresh Bedrock AWS SDK credentials after shared credential files or IMDS tokens rotate, without requiring a gateway restart.

## Best Possible Solution

Land a provider-owned Bedrock runtime-auth fix that invalidates or refreshes AWS SDK-backed credentials on auth failures, with regression coverage and redacted real Bedrock rotation proof.

## Reproduction Assessment

Yes for the OpenClaw source path: current main clears runtime auth state on the Bedrock `aws-sdk` sentinel fallback, and auth-error refresh is gated on that state. A live AWS credential rotation or IMDS run was not performed.

## Solution Assessment

Yes, the narrow maintainable direction is to keep the fix in the Bedrock provider/runtime-auth path and reuse the existing auth-error refresh flow. The linked PR appears to target that shape, but it still needs real behavior proof and conflict cleanup.
