## Summary

Keep open: the report matches a real Bedrock aws-sdk credential-refresh gap on current main, and the older Bedrock auth-marker issue is related context but not the same failure mode.

## What This Changes

The issue reports that Bedrock aws-sdk auth stops picking up refreshed shared AWS credentials after v2026.5.3 until the gateway is restarted.

## Best Possible Solution

Fix Bedrock aws-sdk model calls so shared credentials are re-read after rotation, preferably in the amazon-bedrock plugin/runtime boundary with regression coverage and no new config knob.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: use `AWS_PROFILE` shared-file credentials without an `expiration`, make a Bedrock call, rewrite `~/.aws/credentials`, then make another call in the same gateway process. The AWS shared ini loader caches file reads unless `ignoreCache` is set, and current OpenClaw/pi-ai does not set it for Bedrock inference.

## Solution Assessment

Yes, the best fix is a provider-owned credential-refresh path for Bedrock aws-sdk mode rather than a core auth policy change. The amazon-bedrock plugin owns this provider behavior, and the existing SDK/plugin seams are sufficient without adding user-facing configuration.

## Repair Work Prompt

Fix the Bedrock aws-sdk shared-credential refresh bug reported in https://github.com/openclaw/openclaw/issues/77551. Current main delegates Bedrock inference to pi-ai, whose AWS SDK path does not bypass the shared ini file cache; AWS shared-ini reads are cached unless `ignoreCache` is used, and temp STS credentials without `expiration` can remain stale until process restart. Keep the fix provider-owned under the bundled `amazon-bedrock` plugin/runtime boundary; do not add a user-facing config option and do not move Bedrock-specific policy into core. Prefer the smallest maintainable approach: either route Bedrock inference through a provider-owned stream that preserves pi-ai Bedrock request behavior while using a refresh-safe AWS credentials provider, or make the dependency-backed fix if the repo already has an accepted way to patch/wrap pi-ai. Add regression coverage that simulates shared credentials file rotation/no-expiration cache behavior without live AWS secrets. Update `CHANGELOG.md` because this is a user-facing auth fix. Stop and ask for maintainer review if the solution would require vendoring a large copy of pi-ai Bedrock transport or changing dependency versions/patches.
