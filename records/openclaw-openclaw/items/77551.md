## Summary

Keep open. Current main still has the Bedrock `aws-sdk` auth refresh gap described in the report, and the linked fixing PR is open but not merged, so this should stay open until that PR or an equivalent provider-owned fix lands.

## What This Changes

The issue reports that Bedrock `aws-sdk` authentication stops picking up rotated shared AWS credentials after v2026.5.3, causing `ExpiredTokenException` until the gateway restarts.

## Best Possible Solution

Keep the Bedrock AWS SDK default-chain contract, and make the provider/runtime auth path recreate or invalidate stale shared-credential resolution on auth-class failures without adding a user-facing config knob.

## Reproduction Assessment

Yes for the OpenClaw-side refresh gap: current source shows Bedrock `aws-sdk` auth can fall back to a sentinel with no refreshable runtime-auth state, while auth-error refresh requires that state. A live AWS rotation run was not performed, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No current-main fix is present. The narrowest maintainable path is the provider-owned direction already represented by the open linked PR: keep Bedrock `aws-sdk` auth on a refreshable runtime-auth path and add regression coverage for rotated shared credentials.
