## Summary

Keep open. Current main still leaves Bedrock `aws-sdk` fallback auth without refreshable runtime-auth state, and an open PR already references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue reports that Bedrock `aws-sdk` auth stops picking up refreshed shared AWS credentials during a long-running gateway process, causing `ExpiredTokenException` until restart.

## Best Possible Solution

Review and land the paired Bedrock runtime-auth refresh PR, with regression coverage for shared credentials that rotate without an expiration field, then close this issue as implemented.

## Reproduction Assessment

Yes for the OpenClaw-side refresh gap: source inspection shows sentinel-only `aws-sdk` auth clears runtime-auth state while auth-error refresh requires that state. A live AWS rotation run was not performed, so the full `ExpiredTokenException` path remains source-reproducible rather than live-reproduced here.

## Solution Assessment

No current-main fix is present. The narrowest maintainable solution is provider-owned Bedrock runtime-auth refresh or client recreation on auth failures, validated against rotated shared AWS credentials and covered by focused auth-controller and Bedrock tests.
