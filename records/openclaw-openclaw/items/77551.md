## Summary

Keep open. Current main still has a source-level Bedrock aws-sdk refresh gap: OpenClaw promises AWS SDK default-chain auth, but the aws-sdk sentinel path has no refreshable runtime-auth state while the AWS shared-ini loader caches credential-file reads unless explicitly refreshed.

## What This Changes

The issue asks OpenClaw to keep Amazon Bedrock aws-sdk auth working after shared AWS credentials rotate without restarting the gateway.

## Best Possible Solution

Preserve Bedrock AWS SDK default-chain auth, but make the Bedrock-owned runtime path invalidate shared-ini credential caches and retry once on auth-class failures, with regression coverage for rotated temporary profile credentials that lack expiration metadata.

## Reproduction Assessment

Yes, from source inspection. The documented Bedrock aws-sdk path reaches a sentinel state with no refresh hook, and the AWS SDK shared-ini loader caches credential-file contents unless forced to refresh; a live AWS credential-rotation run was not performed.

## Solution Assessment

No current-main fix is present. The narrowest maintainable direction is a provider-owned Bedrock auth refresh/retry path that clears or bypasses AWS shared-ini caching without adding a user-facing config option.
