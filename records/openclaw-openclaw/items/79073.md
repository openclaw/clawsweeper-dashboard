## Summary

Close as implemented: current main and v2026.5.4 include the external channel `secret-contract-api` dist-sidecar fix, which matches the reported 2026.5.3-1 npm Discord SecretRef startup failure.

## What This Changes

Allow `channels.discord.token` SecretRefs in npm-installed Discord plugins to resolve during gateway startup instead of leaving the channel not configured.

## Best Possible Solution

Keep the generic loader fix shipped in v2026.5.4; users on 2026.5.3-1 should upgrade to v2026.5.4 or newer.

## Reproduction Assessment

No current-main failing reproduction remains. The old 2026.5.3-1 path is explained by the missing `<rootDir>/dist/secret-contract-api` lookup fixed by https://github.com/openclaw/openclaw/pull/77421, and current source contains that lookup plus a regression test.

## Solution Assessment

Yes. The generic external channel secret-contract loader is the narrow maintainable fix because it resolves the published npm plugin layout without adding Discord-specific fallback policy.
