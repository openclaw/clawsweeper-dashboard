## Label Justifications

- `P1`: The report and related production evidence describe gateway RSS growth, OOM risk, lock stalls, and broken agent/channel workflows under normal session churn.
- `impact:crash-loop`: The reported failure mode is process-level memory pressure that can make the gateway unresponsive or OOM-killed.
- `impact:session-state`: The affected surface is persisted session metadata, transcript history, session caches, and cleanup behavior.

## Summary

Keep open. Current main has mitigations for session-list lookup, bounded recent transcript tails, cache hits, and retention, but large cache misses still parse and serialize the whole `sessions.json`, and the recent-history stats path can still build and cache a parsed full-transcript index before returning bounded history.

## What This Changes

The issue reports gateway RSS growth from loading large session metadata and transcript files into memory and asks for lazy, streaming, or bounded session storage behavior.

## Best Possible Solution

Keep this issue as the cross-cutting gateway/session memory tracker until session metadata is bounded or indexed and recent-history count/read paths avoid full parsed transcript indexes, while coordinating static-payload work through https://github.com/openclaw/openclaw/issues/55334 and https://github.com/openclaw/openclaw/pull/78259 plus transcript API work through https://github.com/openclaw/openclaw/issues/79904 and https://github.com/openclaw/openclaw/issues/79902.

## Reproduction Assessment

Yes, at source level. Current main still has whole-store `sessions.json` parse/stringify paths and a recent-history count path that can build a full parsed transcript index on cache miss, though I did not run a long-uptime RSS benchmark.

## Solution Assessment

No complete solution is on main yet. Existing bounded tail reads, cache hits, and retention controls are useful mitigations, but the maintainable fix needs a coordinated persisted-store and transcript count/index design rather than closing this as implemented or fully duplicated.
