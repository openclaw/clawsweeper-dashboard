## Summary

Keep open. Current main still treats parameter-validation-shaped mutating tool failures as generic tool errors that can produce final user-facing warning payloads, and the related retry-loop PR only addresses retryability metadata for a different issue rather than chat-surface suppression.

## What This Changes

The issue requests suppressing raw tool parameter validation failures from user-facing chat payloads while keeping them available to the agent/runtime.

## Best Possible Solution

Add a typed parameter-validation classification from required-parameter checks through adapter normalization and ToolErrorSummary into final payload policy, then suppress only that class from normal chat warnings.

## Reproduction Assessment

Yes, by source inspection: call buildEmbeddedRunPayloads with a mutating lastToolError such as { toolName: "message", meta: "reply", error: "text required" } and no assistant reply, and current tests assert it emits a visible warning payload. I did not run tests because this review is read-only.

## Solution Assessment

Yes. A typed validation-only signal is the narrowest maintainable fix; matching error text or globally hiding mutating warnings would risk masking real failed side effects.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/65235. Current main can turn tool parameter validation failures into final user-facing chat warning payloads such as "Write failed" or "Message failed" because validation failures are stored as generic ToolErrorSummary values and the payload policy shows mutating tool errors. Add a typed validation/input-error classification that starts at required-parameter validation and adapter normalization, is carried through lastToolError/ToolErrorSummary, and suppresses only that class from final normal chat warning payloads. Preserve visible warnings for real unresolved mutating side-effect failures, do not rely on matching error text, do not add a config option, and avoid duplicating the retryability-only scope of https://github.com/openclaw/openclaw/pull/68070. Add focused regression tests and a single-line CHANGELOG.md fix entry because this changes user-visible chat behavior.
