## Summary

Keep open. Current main still has no typed validation/internal-only marker on tool errors, and the final embedded payload policy can still render mutating validation-shaped tool failures as user-facing chat warnings instead of suppressing them.

## What This Changes

The issue asks OpenClaw to suppress missing or invalid tool-parameter validation failures from user chat surfaces while keeping the error available to the agent runtime and logs.

## Best Possible Solution

Add a typed validation/internal-only classification at the parameter validation or adapter boundary, propagate it through ToolErrorSummary, and have final chat payload policy suppress only that class while preserving visible warnings for real mutating side-effect failures.

## Reproduction Assessment

Yes. Source inspection on current main shows mutating validation-shaped tool errors still enter the visible warning branch, and the existing payload error test asserts a visible warning for `text required`.

## Solution Assessment

Yes. A typed validation/internal-only signal is the narrow maintainable fix because retryability-only metadata or string matching would not give final chat payload policy a reliable suppression boundary.
