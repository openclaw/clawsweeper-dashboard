## Summary

Keep open. Current main still has the reported UX gap: missing required tool parameters are converted into ordinary tool errors, the stored summary has no validation-only marker, and final payload policy still shows warning summaries for mutating write/message-style failures. Related PR #68070 is adjacent retry-loop work, remains open, and does not resolve final chat-surface suppression.

## What This Changes

The issue asks OpenClaw to keep missing or invalid tool-argument validation failures internal to the agent runtime instead of sending warning payloads to user chat channels.

## Best Possible Solution

Carry a typed parameter-validation-only marker from the required-parameter wrapper through adapter normalization, tool-error summary storage, and final payload policy. Suppress normal chat warning payloads for that marker while preserving logs, model/runtime feedback, verbose diagnostics, cron exec timeouts, and real mutating side-effect failure warnings.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists on current main: pass buildEmbeddedRunPayloads a mutating write or message lastToolError whose error is a required/missing-parameter validation string, and current policy produces a user-facing warning summary. I did not run tests because this review is read-only.

## Solution Assessment

Yes. A typed validation-only classification is the narrowest maintainable fix; it is safer than globally suppressing mutating-tool warnings or relying only on message keyword matching.
