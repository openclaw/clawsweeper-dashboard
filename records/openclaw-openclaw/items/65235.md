## Summary

Keep open. Current main still treats required-parameter failures as generic tool errors and final payload policy can show mutating write/message warning summaries to chat; related PR #68070 is open and only addresses retry-loop retryability, not final chat-surface suppression.

## What This Changes

Suppress tool-parameter validation failures from user-visible chat payloads while keeping runtime feedback and logs.

## Best Possible Solution

Add a typed parameter-validation-only signal through validation, adapter normalization, tool-error summary storage, and final payload policy, then suppress only those normal chat warnings.

## Reproduction Assessment

Yes. A static reproduction path exists on current main: pass buildEmbeddedRunPayloads a mutating write or message lastToolError with a missing/required-parameter validation message and the current policy produces a user-facing warning summary.

## Solution Assessment

Yes. A typed validation-only classification is the narrowest maintainable solution; globally suppressing mutating warnings or matching error text alone would be less safe.
