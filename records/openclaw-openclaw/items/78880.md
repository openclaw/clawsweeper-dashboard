## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No, not as-is: stable lifecycle handlers in `src/acp/translator.ts` are the right direction, but `session/list` must not treat rows with unknown workspace metadata as matches for a requested cwd.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not match unknown sessions to the requested cwd:** `src/acp/translator.ts:1394`
  - body: When `session/list` is called with a `cwd` filter, rows without `spawnedWorkspaceDir` are mapped to that requested cwd and then pass the filter. That returns unrelated Gateway sessions under the workspace-scoped request; preserve those unknown rows for unfiltered lists, but exclude or separately handle them when a cwd filter is present.
  - confidence: 0.86
- **[P3] Reject cursors when the cwd filter changes:** `src/acp/translator.ts:683-684`
  - body: This guard only rejects mismatches when the cursor itself has a cwd. A cursor from an unfiltered page can be reused with a new cwd filter, applying the old offset to a different result set and skipping valid sessions; compare the normalized cursor cwd and requested cwd even when one side is absent.
  - confidence: 0.78
