## Solution Assessment

No, not as-is: stable lifecycle handlers in `src/acp/translator.ts` are the right direction, but `session/list` must not treat unknown-workspace rows as matches for a requested cwd and cursors must be scoped to the same filter.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not match unknown sessions to the requested cwd:** `src/acp/translator.ts:1394`
  - body: When `session/list` is called with a `cwd` filter, rows without `spawnedWorkspaceDir` are assigned that requested cwd before filtering. That can return unrelated Gateway sessions in a workspace-scoped list and expose their session key/title/kind metadata; keep unknown rows for unfiltered lists, but exclude them when a cwd filter is present unless the row actually matches.
  - confidence: 0.87
- **[P3] Reject cursors when the cwd filter changes:** `src/acp/translator.ts:683-684`
  - body: This guard only rejects mismatches when the cursor itself has a cwd. A cursor from an unfiltered page can be reused with a new cwd filter, applying the old offset to a different result set and skipping valid sessions; compare the normalized cursor cwd and requested cwd even when one side is absent.
  - confidence: 0.78
