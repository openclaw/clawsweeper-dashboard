## What This Changes

The branch filters Control UI Recent sessions and session-list refreshes by selected agent, converts pending session refresh tracking to a target map, adds UI regression tests, and normalizes auth-profile blockedUntil timestamps.

## Best Possible Solution

Keep the selected-agent filtering, but make the default-agent list path include legacy unscoped session rows before applying the limit and cover that path with a regression test.

## Reproduction Assessment

Yes. Current main's sidebar Recent filter excludes archived/global/cron/subagent/spawned rows but does not check agent ownership, so mixed agent rows in `sessionsResult` can render together; the PR's render proof exercises that path.

## Solution Assessment

No. The selected-agent filter is the right direction, but the proposed list-scoping helper needs a default-agent legacy-row compatibility fix before merge.
