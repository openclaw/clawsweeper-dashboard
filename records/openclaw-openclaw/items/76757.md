## Summary

This draft maintainer PR should stay open: current head still has concrete P2 regressions in direct plugin tool invocation, configured PDF model startup ownership, and OAuth provider formatting, and the protected `maintainer` label also prevents cleanup closure.

## What This Changes

The PR prepares and reuses model, auth, plugin, tool, and runtime state across reply execution, embedded runs, gateway startup, and runtime postbuild alias generation.

## Best Possible Solution

Revise the branch so prepared-state reuse preserves direct plugin tool discovery, includes all public configured model refs used for startup ownership including `pdfModel`, and keeps provider `formatApiKey` hooks on OAuth runtime paths.

## Reproduction Assessment

Yes. Source inspection of PR head shows the direct requested plugin tool is no longer a standalone discovery allow entry, `pdfModel` is omitted from configured startup refs, and non-bundled OAuth providers can return before the provider formatter hook runs.

## Solution Assessment

No. The performance direction is reasonable, but this is not the best mergeable form while it changes established tool discovery, PDF model startup ownership, and OAuth formatting contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve direct requested plugin tools:** `src/gateway/tool-resolution.ts:87-96`
  - body: `gatewayRequestedTools` now only reaches plugin discovery through policies that already have an `allow` array. With no profile allowlist, `mergeAlsoAllowPolicy` leaves those policies undefined and the requested plugin tool is absent from `pluginToolAllowlist`, so `/tools/invoke` can skip the plugin before it can execute. Keep the standalone requested-tool allow entry or synthesize an equivalent allow policy.
  - confidence: 0.91
- **[P2] Include pdfModel in configured startup refs:** `src/config/model-refs.ts:32-37`
  - body: Gateway startup now derives model-owner plugins from `collectConfiguredModelRefs`, but this collector omits `agents.defaults.pdfModel`. Since `pdfModel` is the documented first source for the PDF tool, a configured PDF provider plugin can be left out of startup/prewarm runtime surfaces.
  - confidence: 0.88
- **[P2] Keep formatApiKey hooks on OAuth credentials:** `src/agents/auth-profiles/oauth.ts:113-115`
  - body: For non-bundled OAuth providers without an already pinned gateway-runtime formatter, this returns `credentials.access` before calling the existing provider plugin formatter. `formatApiKey` is the documented hook for custom runtime token shapes, so custom OAuth providers can silently lose required formatting.
  - confidence: 0.82
