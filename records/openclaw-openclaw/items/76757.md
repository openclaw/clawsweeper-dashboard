## Summary

Keep open: this is a draft protected maintainer PR, and the latest head still has P2 regressions in direct plugin tool invocation, configured PDF model startup ownership, and OAuth `formatApiKey` hook handling.

## What This Changes

The PR threads prepared auth, model, plugin, tool-planning, and startup runtime state through reply execution, embedded runs, and gateway startup, with stage timing and postbuild alias handling changes.

## Best Possible Solution

Revise the branch so prepared-state reuse preserves direct plugin tool discovery, collects all public configured model refs needed for startup ownership, and keeps provider OAuth formatting hooks on the runtime path.

## Reproduction Assessment

Yes. Source inspection of PR head shows the requested plugin tool is no longer a standalone plugin discovery allow entry, `pdfModel` is omitted from configured model ref collection, and non-Google OAuth can return before the provider formatter hook runs.

## Solution Assessment

No. The performance direction is reasonable, but this patch is not the best mergeable form while it regresses established tool discovery, PDF model provider startup, and OAuth provider formatting behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve direct requested plugin tools:** `src/gateway/tool-resolution.ts:87-96`
  - body: `gatewayRequestedTools` now only reaches discovery through profile policies that already have an `allow` array. With no profile allowlist, the requested plugin tool is absent from `pluginToolAllowlist`, so optional plugin discovery can skip the plugin before `/tools/invoke` can execute it. Keep the standalone requested-tool allow entry or synthesize an equivalent allow policy.
  - confidence: 0.91
- **[P2] Include pdfModel in configured startup refs:** `src/config/model-refs.ts:32-37`
  - body: Gateway startup now derives model-owner plugins from `collectConfiguredModelRefs`, but this collector omits `agents.defaults.pdfModel`. Since `pdfModel` is the documented first source for the PDF tool, a configured PDF provider plugin can be left out of startup/prewarm runtime surfaces.
  - confidence: 0.88
- **[P2] Keep formatApiKey hooks on OAuth credentials:** `src/agents/auth-profiles/oauth.ts:113-115`
  - body: For non-Google providers without an already pinned gateway-runtime formatter, this returns `credentials.access` before calling the existing provider plugin formatter. `formatApiKey` is the documented hook for custom runtime token shapes, so custom OAuth providers can silently lose required formatting.
  - confidence: 0.82
