## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve direct requested plugin tools:** `src/gateway/tool-resolution.ts:87-96`
  - body: `gatewayRequestedTools` is now only merged into profile policies that already have an `allow` array. With no profile allowlist, `mergeAlsoAllowPolicy` returns the original undefined policy, `explicitToolAllowlist` stays empty, and optional plugin tools requested through `/tools/invoke` can be skipped before execution. Keep the standalone requested-tool allow policy or synthesize an equivalent allow entry.
  - confidence: 0.92
- **[P2] Collect image and PDF model refs for startup ownership:** `src/config/model-refs.ts:32-37`
  - body: Gateway startup now derives configured model-owner plugins from `collectConfiguredModelRefs`, but the collector omits the public `agents.defaults.imageModel` and `agents.defaults.pdfModel` keys. A plugin that owns a configured image/PDF model can therefore be left out of the startup/runtime surfaces this PR is trying to pin.
  - confidence: 0.9
- **[P2] Keep OAuth formatApiKey hooks reachable:** `src/agents/auth-profiles/oauth.ts:113-115`
  - body: For non-bundled OAuth providers without an already pinned gateway-runtime formatter, this returns `credentials.access` before calling the existing provider plugin formatter lookup. `formatApiKey` is the documented hook for custom runtime token shapes, so external OAuth providers can silently lose required formatting.
  - confidence: 0.86
