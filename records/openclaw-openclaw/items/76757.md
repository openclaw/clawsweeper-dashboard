## Summary

Keep open: the PR is a draft maintainer-labeled performance refactor, and the current head still has P2 regressions in requested plugin-tool materialization, configured model-owner startup coverage, and OAuth formatter lookup.

## What This Changes

This PR threads prepared auth, model, plugin/runtime, tool-planning, and stage-timing state through reply, embedded, and Gateway startup paths to reduce repeated reply-path discovery work.

## Best Possible Solution

Revise the branch so prepared-state reuse preserves the existing tool invocation, model-ref collection, OAuth formatter, and changelog contracts before maintainer re-review.

## Reproduction Assessment

Yes. Source inspection of PR head against current main shows the missing standalone requested-tool allow entry, omitted `imageModel`/`pdfModel` collector keys, and the OAuth formatter short-circuit without needing a live Gateway run.

## Solution Assessment

No. Reusing prepared runtime surfaces is a reasonable performance direction, but this head is not mergeable while it changes established tool discovery, configured model ownership, OAuth formatting, and changelog expectations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve direct requested plugin tools:** `src/gateway/tool-resolution.ts:87-96`
  - body: `gatewayRequestedTools` is now only merged into profile policies that already have an `allow` array. With no profile allowlist, `mergeAlsoAllowPolicy` returns the original undefined policy, `explicitToolAllowlist` lacks the direct request, and optional plugin tools requested through `/tools/invoke` can be skipped before execution. Re-add the standalone requested-tool allow policy or synthesize an equivalent allow entry.
  - confidence: 0.92
- **[P2] Collect image and PDF model refs for startup ownership:** `src/config/model-refs.ts:32-37`
  - body: Gateway startup now derives configured model-owner plugins from `collectConfiguredModelRefs`, but this collector omits the public `agents.defaults.imageModel` and `agents.defaults.pdfModel` keys. A plugin that owns a configured image/PDF model can therefore be left out of the startup/runtime surfaces this PR is trying to pin.
  - confidence: 0.9
- **[P2] Keep OAuth formatApiKey hooks reachable:** `src/agents/auth-profiles/oauth.ts:113-115`
  - body: For non-Google OAuth providers without an already pinned gateway-runtime formatter, this returns `credentials.access` before calling the existing provider plugin formatter lookup. `formatApiKey` is the documented hook for custom runtime token shapes, so external OAuth providers can silently lose required formatting.
  - confidence: 0.87
- **[P3] Add the required changelog entry:** `src/gateway/server-startup-post-attach.ts:332`
  - body: The branch advertises user-visible reply latency and Gateway startup performance changes but does not update `CHANGELOG.md`. Repo policy requires user-facing `perf` changes to have an active-version changelog entry before merge.
  - confidence: 0.8
