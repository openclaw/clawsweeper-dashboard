## Summary

This protected maintainer draft should stay open; the latest head still has P2 regressions in direct plugin tool invocation, PDF model startup ownership, and OAuth `formatApiKey` handling.

## What This Changes

The PR prepares and reuses model, auth, plugin, tool, and runtime state across reply execution, embedded runs, gateway startup, and runtime postbuild alias generation.

## Best Possible Solution

Revise the branch so prepared-state reuse preserves direct plugin tool discovery, includes every public configured model ref including `pdfModel`, and always honors provider OAuth formatter hooks before re-review.

## Reproduction Assessment

Yes. Source inspection of PR head against current main shows the missing direct requested-tool allow entry, omitted `pdfModel` collector key, and skipped OAuth formatter hook paths without needing a live gateway run.

## Solution Assessment

No. The performance direction is reasonable, but this is not the best mergeable form while it changes established tool discovery, PDF model startup ownership, and OAuth formatting contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve direct requested plugin tools:** `src/gateway/tool-resolution.ts:87-96`
  - body: `gatewayRequestedTools` is now only merged into profile policies that already have an `allow` array. With no profile allowlist, `mergeAlsoAllowPolicy` returns the original undefined policy and the requested plugin tool is absent from `pluginToolAllowlist`, so `/tools/invoke` can skip the plugin before it can execute. Keep the standalone requested-tool allow entry or synthesize an equivalent allow policy.
  - confidence: 0.91
- **[P2] Include pdfModel in configured startup refs:** `src/config/model-refs.ts:32-37`
  - body: Gateway startup now derives model-owner plugins from `collectConfiguredModelRefs`, but this collector omits `agents.defaults.pdfModel`. Since `pdfModel` is the documented first source for the PDF tool, a configured PDF provider plugin can be left out of startup/prewarm runtime surfaces.
  - confidence: 0.88
- **[P2] Keep formatApiKey hooks on OAuth credentials:** `src/agents/auth-profiles/oauth.ts:113-115`
  - body: For non-bundled OAuth providers without an already pinned gateway-runtime formatter, this returns `credentials.access` before calling the existing provider plugin formatter. `formatApiKey` is the documented hook for custom runtime token shapes, so custom OAuth providers can silently lose required formatting.
  - confidence: 0.82
