## Summary

Keep this PR open for maintainer follow-up: the prepared-runtime direction is useful, but the current head still has three source-backed P2 regressions in direct plugin tool invocation, configured PDF model-owner startup coverage, and provider OAuth formatting. This review covers https://github.com/openclaw/openclaw/pull/76757.

## What This Changes

The PR threads prepared auth, model, plugin, tool-planning, and startup runtime state through reply execution and gateway startup to reduce repeated discovery.

## Best Possible Solution

Keep the prepared-state refactor only after preserving existing direct plugin-tool invocation semantics, including pdfModel in configured model-owner startup refs, and retaining provider formatApiKey hooks across gateway and non-gateway OAuth paths.

## Reproduction Assessment

Yes. Source inspection of PR head shows the requested plugin tool is no longer a standalone plugin discovery allowlist entry, pdfModel is omitted from the configured model ref collector, and non-Google OAuth formatting can bypass the existing provider hook path.

## Solution Assessment

No. The performance/refactor direction is reasonable, but this patch is not the best mergeable form while it regresses established tool discovery, PDF model provider startup, and OAuth provider formatting behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve requested plugin tools without profile allowlists:** `src/gateway/tool-resolution.ts:87-96`
  - body: `gatewayRequestedTools` now only reaches discovery through merged profile policies. When there is no existing `allow` policy, `mergeAlsoAllowPolicy` returns undefined, `pluginToolAllowlist` omits the direct requested tool, and optional plugin discovery can skip the plugin before `/tools/invoke` can execute it. Keep the standalone requested-tool allow entry or an equivalent discovery path.
  - confidence: 0.9
- **[P2] Include pdfModel in configured model refs:** `src/config/model-refs.ts:31-35`
  - body: Gateway startup now uses `collectConfiguredModelRefs` to include model-owner plugins, but this collector omits `agents.defaults.pdfModel` even though pdfModel is a public model config used by the PDF tool. A configured PDF provider plugin can therefore be left out of startup/prewarm runtime surfaces.
  - confidence: 0.86
- **[P2] Keep provider formatApiKey hooks on OAuth paths:** `src/agents/auth-profiles/oauth.ts:113-116`
  - body: For non-Google providers without a pinned gateway-runtime formatter, this returns `credentials.access` before calling the existing plugin formatter. `ProviderPlugin.formatApiKey` is the documented hook for custom runtime token shapes, so custom or non-gateway OAuth providers can silently lose required formatting.
  - confidence: 0.78
