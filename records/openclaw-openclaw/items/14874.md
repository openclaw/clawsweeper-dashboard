## Summary

Keep open. Current main still lacks loader-time SIE envelope or issuer-signature verification for skills; the direct implementation PR is closed unmerged, and the open SHA-256 integrity PR is adjacent rather than a substitute for signed-instruction policy.

## What This Changes

The issue requests optional skill-loader SIE verification with warn and strict unsigned modes plus fail-closed invalid-signature handling.

## Best Possible Solution

Define a maintainer-approved, default-off TypeScript SIE verification contract in the core skills loader/config/docs, with deterministic reason codes and a clear relationship to ClawHub install/update flows.

## Reproduction Assessment

Not applicable. This is a feature/security-design proposal rather than broken existing behavior; source inspection verifies current main lacks the requested SIE loader/config path.

## Solution Assessment

Unclear until maintainers choose the security policy. The likely implementation boundary is core skill loading plus config/docs, but an automated patch should not choose trust roots, envelope format, revocation, or failure semantics.
