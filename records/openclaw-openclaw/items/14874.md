## Summary

Keep open. Current main still lacks loader-time SIE envelope/signature verification, the direct implementation PR is closed unmerged, and the adjacent SHA-256 integrity proposal does not replace issuer-authenticated skill instructions.

## What This Changes

The issue requests a default-off SIE verification gate in the skill loader with warn and strict handling for unsigned skills and rejection for invalid signatures.

## Best Possible Solution

Define a maintainer-approved, default-off TypeScript SIE contract in skill loading, config/schema, tests, and docs, with explicit trust-root, issuer, revocation, warning/logging, invalid-signature rejection, and ClawHub composition semantics.

## Reproduction Assessment

Not applicable. This is a feature/security-design proposal rather than broken existing behavior, and source inspection on current main confirms the requested SIE loader/config path is absent.

## Solution Assessment

Unclear until maintainers choose the security policy. The loader boundary appears to be the right product surface, but automation should not choose the envelope format, trust roots, issuer policy, revocation behavior, or strict-mode failure semantics.
