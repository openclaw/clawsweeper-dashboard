## Summary

Keep open. Current main still has no skill-loader/config SIE envelope or issuer signature verification path, the directly linked implementation PR was closed unmerged, and the open SHA-256 integrity PR is adjacent but not a substitute for signed-instruction verification.

## What This Changes

Request optional feature-flagged SIE verification in the skill loader with warn/strict handling for unsigned skills and rejection of invalid signatures.

## Best Possible Solution

Define a maintainer-approved, default-off TypeScript SIE verification contract in the core skills loader/config/docs, with deterministic reason codes, unsigned warn/strict behavior, invalid-signature fail-closed behavior, and a clear relationship to ClawHub install/update flows.

## Reproduction Assessment

Not applicable. This is a feature/security-design proposal rather than a broken existing-behavior report; source inspection verifies current main lacks the requested SIE loader/config path.

## Solution Assessment

Unclear until maintainers choose the security policy. The right implementation boundary appears to be core skill loading and config, but an automated patch should not choose trust roots, envelope format, revocation, or fail-open/fail-closed policy.
