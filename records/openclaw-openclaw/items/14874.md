## Summary

Keep open. Current main still has only path/size/frontmatter-based skill loading, while the requested optional SIE warn/strict signature-verification gate remains a security-sensitive feature/design item rather than implemented behavior.

## What This Changes

Add an optional, feature-flagged Signed Instruction Envelope verification gate to the skill loader with warn and strict modes.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, including config/schema/docs, deterministic reason codes, trusted issuer and revocation policy, warn/strict unsigned handling, and invalid-signature fail-closed behavior.

## Reproduction Assessment

Not applicable. This is a feature/security design request, and source inspection of current main establishes that the requested SIE loader gate is absent rather than failing an existing contract.

## Solution Assessment

Unclear until maintainer design is settled. The direction is plausible, but the narrow maintainable solution needs explicit policy choices for trust roots, envelope format, revocation, config shape, and how it relates to ClawHub install/update flows.
