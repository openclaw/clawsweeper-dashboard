## Summary

Keep open. Current main still lacks SIE envelope or issuer-signature verification across skill loading, config, and docs; the direct implementation PR is closed unmerged, and the SHA-256 integrity PR is adjacent rather than a substitute.

## What This Changes

The issue requests an optional, default-compatible SIE verification gate in the skill loader with warn and strict modes for unsigned or invalid skill instructions.

## Best Possible Solution

Define a maintainer-approved, default-off TypeScript SIE verification contract in the core skills loader/config/docs, with deterministic reason codes, unsigned warn/strict behavior, invalid-signature fail-closed behavior, and a clear ClawHub install/update relationship.

## Reproduction Assessment

Not applicable. This is a feature/security-design proposal rather than a broken existing-behavior report; source inspection verifies current main lacks the requested SIE loader/config path.

## Solution Assessment

Unclear until maintainers choose the security policy. The likely implementation boundary is core skill loading plus config/docs, but an automated patch should not choose trust roots, envelope format, revocation, or fail-open/fail-closed policy.
