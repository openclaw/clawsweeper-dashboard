## Summary

Keep open. Current main still has no skill-loader SIE envelope/signature verification path, the direct implementation PR is closed unmerged, and the open SHA-256 integrity PR is adjacent rather than a substitute for issuer-authenticated skill instructions.

## What This Changes

The issue requests an optional, feature-flagged SIE verification gate for skills with warn/strict unsigned handling and invalid-signature rejection.

## Best Possible Solution

Define a maintainer-approved, default-off TypeScript SIE verification contract in core skill loading, config/schema, tests, and docs, with deterministic reason codes and explicit composition with ClawHub install/update flows.

## Solution Assessment

Unclear until maintainers choose the security policy. The likely implementation boundary is core skill loading plus config/docs, but automation should not choose trust roots, envelope format, revocation, or failure semantics.
