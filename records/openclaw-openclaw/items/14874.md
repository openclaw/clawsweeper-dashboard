## Summary

Keep open. Current main still lacks optional SIE envelope/signature verification in the skill loader, workspace merge path, config schema, tests, and public skills docs; the direct implementation PR is closed unmerged, and the adjacent SHA-256 integrity PR does not settle issuer/signature verification.

## What This Changes

Request optional default-off SIE verification for skill loading, with warn/strict unsigned handling and invalid-signature rejection.

## Best Possible Solution

Define a maintainer-approved, default-off TypeScript SIE verification contract across all skill roots, with config/schema/docs support, deterministic reason codes, trust-root and revocation policy, warn/strict unsigned handling, and invalid-signature fail-closed behavior.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path. Targeted search plus inspection of the loader, workspace loading, config schema, docs, tests, and related PR state show the requested SIE behavior is absent on current main.

## Solution Assessment

Unclear until maintainers choose the trust model. The requested direction is plausible, but the best fix should be a maintainer-approved loader/config contract rather than the closed standalone reference PR or SHA-256 hashing alone.
