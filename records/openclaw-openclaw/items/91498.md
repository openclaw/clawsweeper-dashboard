## What This Changes

Request to let model aliases, model refs, or runtimes declare a preferred auth profile while keeping `auth.order.<provider>` as fallback.

## Best Possible Solution

Define one canonical model-scoped auth-profile preference contract, update schema/types/docs/status output and regression tests, and keep provider-wide `auth.order.<provider>` as the fallback.

Yes from source inspection: strict model config schemas lack an auth-profile selector, and auth selection uses provider-level order plus session/request overrides.
