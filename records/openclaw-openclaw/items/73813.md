## Summary

Keep open. Current main still records yielded isolated cron runs as terminal `ok` outcomes from the first parent result, so the reported short `durationMs`/early `finished` observability bug remains even though the embedded runner now exposes yielded/paused state.

## What This Changes

The issue asks for isolated cron `sessions_yield` runs to remain observable as in-progress, or at least report duration/status including the descendant subagent work instead of the pre-yield parent turn only.

## Best Possible Solution

Define the yielded isolated-cron lifecycle contract, then update cron bookkeeping so yielded parent turns are observable as nonterminal until the resumed parent and descendants reach a real terminal outcome, or add explicit backward-compatible yielded/pending metadata while preserving existing consumers. Regression coverage should include `delivery.mode: "none"` and descendant completion/resume behavior.

## Reproduction Assessment

Yes. The issue gives concrete isolated-cron steps with a sleeping subagent, and source inspection on current main maps the same path: yielded embedded results are marked paused, but cron finalization and timer emission still treat the first parent result as terminal `ok`.

## Solution Assessment

Unclear. Deferred finalization best matches the reported status and duration semantics, while additive yielded metadata is a smaller compatibility step; a maintainer should choose the public cron lifecycle contract before implementation.
