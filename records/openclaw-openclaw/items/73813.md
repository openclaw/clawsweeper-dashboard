## Summary

Keep open. Current main still maps a yielded isolated cron run to terminal `ok` bookkeeping and a pre-yield duration, while the embedded runner now exposes enough yielded/paused metadata for cron to make a better lifecycle decision.

## What This Changes

The issue asks cron not to report isolated `sessions_yield` runs as finished `ok` with only pre-yield duration while subagents and parent follow-up work continue.

## Best Possible Solution

Cron run lifecycle should avoid terminal success bookkeeping until yielded parent and descendant work reaches a real terminal outcome, with additive yielded metadata only if compatibility requires an interim state.

## Reproduction Assessment

Yes from source inspection, though I did not execute a live cron run. Current main still produces embedded yielded metadata, then cron maps the first returned embedded result to terminal `ok` without consuming that metadata.

## Solution Assessment

Unclear. Deferring `finished` best matches the reported status and duration semantics, while adding yielded metadata is smaller but only partially fixes metrics; maintainers should choose the event/run-log contract first.
