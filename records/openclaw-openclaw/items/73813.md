## Summary

Keep open. Current main still marks the first yielded isolated cron turn as terminal `ok` and records the pre-yield duration, while the discussion correctly identifies that only the embedded-runner half has landed.

## What This Changes

The issue asks cron run bookkeeping for isolated `sessions_yield` orchestration to reflect the full parent/subagent lifecycle instead of treating the initial yielded turn as final success.

## Best Possible Solution

Define the cron lifecycle contract for yielded isolated runs, then update cron outcome/run-log/hook schemas and regression tests so terminal status and duration represent real completion or explicitly expose an interim yielded state.

## Reproduction Assessment

Yes from source inspection. A yielded embedded run returns `meta.yielded` and `livenessState: "paused"`, but isolated cron maps that first returned result to `status: "ok"` and records duration immediately.

## Solution Assessment

Unclear. Deferring `finished` best matches the reported duration/status semantics, while adding yielded metadata is a smaller compatibility step; maintainers should choose the public cron contract first.
