## Summary

Keep open. Current main still has the reported cron observability gap for isolated runs that yield without delivery: cron can classify the first yielded embedded turn as terminal `ok` and compute duration from that short execution window rather than the resumed parent-plus-descendant workflow.

## What This Changes

Make isolated cron run status and duration reflect the full `sessions_yield` plus subagent completion lifecycle instead of the pre-yield parent turn.

## Best Possible Solution

Cron should represent yielded isolated runs as non-terminal until the resumed parent and descendants reach a real terminal outcome, or expose an explicit partial/yielded state so monitoring cannot interpret the first turn as a completed job.

## Reproduction Assessment

Yes, source-reproducible. The issue’s isolated-cron steps match current main: `sessions_yield` produces yielded/paused embedded metadata, but cron’s no-delivery finalization path still emits terminal `ok` bookkeeping without consuming it.

## Solution Assessment

Unclear. Deferring `finished` best matches the expected status and duration semantics, while an additive yielded field is a smaller compatibility step; maintainers need to choose the cron event/run-log contract.
