## Summary

Keep open. Current main still has the reported cron observability gap: yielded embedded runs are marked paused/yielded, but cron outcome/status/duration bookkeeping still resolves the first isolated-run result as terminal `ok`.

## What This Changes

The issue asks cron run logging to avoid reporting terminal `ok` and a short duration while an isolated `sessions_yield` parent/subagent workflow is still running.

## Best Possible Solution

Cron should treat yielded isolated orchestration as nonterminal until the resumed parent and active descendants settle, or add a clearly documented backward-compatible yielded state if maintainers choose to preserve immediate finished events.

## Reproduction Assessment

Yes. The issue provides concrete steps, and current source inspection maps that path to `meta.yielded` from the embedded runner followed by terminal cron `ok`/`finished` bookkeeping before descendant completion is guaranteed.

## Solution Assessment

Unclear. The current implementation is not sufficient; the maintainer decision is whether to land the smaller yielded-metadata contract first or make cron lifecycle/duration wait for the true terminal parent/subagent result.
