## Summary

Keep open. Current main still has the reported cron observability gap: embedded runs can report `yielded`/`paused`, but isolated cron still records a terminal `ok` result and first-window duration without consuming that metadata.

## What This Changes

The issue asks cron run status and duration to reflect the full isolated `sessions_yield` subagent workflow rather than the first pre-yield parent turn.

## Best Possible Solution

Cron run history should not present a pre-yield pause as a completed run; it should expose a nonterminal/yielded state or record the total resumed parent-plus-descendant duration.

## Reproduction Assessment

Yes, from source inspection rather than live execution. The issue gives concrete steps, docs define `sessions_yield` as waiting for a follow-up turn, the embedded runner marks that state as yielded/paused, and cron finalization ignores it.

## Solution Assessment

Unclear. Deferring `finished` best matches the expected status and duration semantics, while additive yielded metadata is smaller and more compatible but only partial; maintainers need to choose the cron event/run-log contract.
