## Summary

Keep open. Current `main` still exposes the reported cron observability gap: yielded embedded runs now carry paused/yielded metadata, but isolated cron run finalization still maps the first clean parent turn to terminal `status: "ok"` and the timer emits `finished` immediately from that first core result.

## What This Changes

The issue requests accurate cron run status and duration for isolated `agentTurn` jobs that spawn subagents and call `sessions_yield`.

## Best Possible Solution

Cron run bookkeeping should treat yielded isolated orchestration as nonterminal until the resumed parent and descendant subagents reach a true terminal outcome. If maintainers want a staged rollout, add explicit yielded/pending metadata first while keeping deferred finalization tracked as the complete fix.

## Reproduction Assessment

Yes. The issue gives concrete isolated cron steps, and current source inspection traces the same path: `sessions_yield` produces paused/yielded embedded metadata, then cron finalization and the timer still emit a terminal `ok` result from the first parent turn.

## Solution Assessment

No current solution is in place. Deferred finalization best matches the expected status and duration behavior; additive yielded metadata would be a safer partial step but would not by itself make `durationMs` reflect the full orchestration.
