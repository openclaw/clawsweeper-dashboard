## Summary

Keep open. Current main has useful descendant-follow-up handling in the isolated cron delivery path, but the cron run outcome and timer still do not model yielded work as nonterminal; yielded embedded runs can still be converted into terminal `ok` bookkeeping from the first parent result, especially when delivery is disabled or bypassed.

## What This Changes

The issue reports that isolated cron jobs using `sessions_yield` after spawning a subagent are recorded as successful with a short duration before descendant work and the resumed parent finish.

## Best Possible Solution

Cron bookkeeping should make yielded isolated orchestration observable as nonterminal until the resumed parent and descendant subagents reach a real terminal outcome, with any additive yielded metadata kept backward-compatible for consumers that cannot immediately handle deferred `finished` events.

## Reproduction Assessment

Yes. The issue gives concrete isolated-cron steps, and source inspection on current main maps the same path: embedded runs can return yielded/paused metadata while cron finalization and timer emission still produce terminal `ok`/short-duration bookkeeping from the first parent result.

## Solution Assessment

Unclear. Deferred finalization best matches the reported status and duration semantics, while additive yielded metadata is a smaller compatibility step; the issue discussion is correctly waiting for maintainer choice between those contracts.
