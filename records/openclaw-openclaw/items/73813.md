## Summary

Keep open. Current main still treats the first yielded isolated cron runner return as a terminal `ok` cron outcome, while the embedded runner now exposes `yielded`/`paused` metadata that cron does not consume.

## What This Changes

Cron isolated agent jobs that yield to subagents should not report terminal `ok` with a short `durationMs` before resumed parent and descendant work actually completes.

## Best Possible Solution

Define the cron lifecycle for yielded isolated runs, preferably by deferring terminal `finished` bookkeeping and duration calculation until resumed parent and descendant work reaches a real terminal outcome.

## Reproduction Assessment

Yes, source-reproducible with high confidence. A yielded embedded run now returns paused/yielded metadata, but cron still resolves the first returned result to terminal `ok` bookkeeping and computes duration from that first return.

## Solution Assessment

Unclear. Deferring `finished` best matches the reported status and duration semantics, while additive `yielded` metadata is a smaller compatibility step that does not fully fix the duration metric by itself.
