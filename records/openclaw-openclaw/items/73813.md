## Summary

Keep open. Current main still exposes yielded/paused metadata from the embedded runner, but the cron isolated-agent path maps the first returned embedded result to terminal `ok` and emits `finished` with the pre-yield duration.

## What This Changes

The issue asks cron isolated agent runs that call `sessions_yield` after spawning subagents to avoid reporting terminal `ok` and a pre-yield duration before the resumed parent work completes.

## Best Possible Solution

Cron should make yielded isolated-agent runs observable as non-terminal work until resumed parent and descendant work reaches a real terminal outcome; if compatibility blocks that, add explicit yielded metadata instead of reporting plain `ok`.

## Reproduction Assessment

Yes from source inspection, though not from a live run in this read-only review. The embedded runner returns yielded/paused metadata, while cron ignores those fields and emits terminal `finished`/`ok` as soon as the first embedded run returns.

## Solution Assessment

Unclear until maintainers choose the public run-log contract. Deferring `finished` fixes both status and duration semantics, while an additive `yielded` field is smaller but leaves duration partial.
