## Summary

Keep open. Current main still has the same cron bookkeeping gap: yielded embedded runs expose paused/yielded metadata, but isolated cron still resolves the first embedded result to terminal `ok` status and duration based on that first return.

## What This Changes

The issue asks cron run logging for isolated `sessions_yield` jobs to reflect the full parent/subagent lifecycle instead of reporting `ok` and a short pre-yield duration immediately.

## Best Possible Solution

Define yielded isolated cron lifecycle semantics, preferably deferring terminal `finished` bookkeeping and duration calculation until resumed parent and descendant work reaches a real terminal outcome.

## Reproduction Assessment

Yes, source-reproducible with high confidence. A yielded embedded run now carries `meta.yielded` and `livenessState: "paused"`, while cron still records the first returned result as terminal `ok` and computes duration from that first return.

## Solution Assessment

Unclear. Deferring `finished` best matches the reported status and duration semantics, while additive `yielded` metadata is smaller and compatible but would not by itself make `durationMs` reflect the full run.
