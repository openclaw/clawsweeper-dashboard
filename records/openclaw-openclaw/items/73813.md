## Summary

Keep open. Current main still exposes yielded/paused metadata from the embedded runner, but isolated cron run finalization does not consume it and still records a terminal `ok`/short duration at the first returned embedded result.

## What This Changes

The issue asks isolated cron run logs to avoid reporting terminal success and a pre-yield duration while a `sessions_yield` subagent flow is still running.

## Best Possible Solution

Define the yielded isolated-cron lifecycle contract, then update cron bookkeeping so terminal status and duration either reflect real completion or explicitly expose yielded/paused state for observability consumers.

## Reproduction Assessment

Yes, from source inspection. The embedded runner marks yielded attempts as paused/yielded, while cron maps the first returned isolated run result to terminal `ok` and emits `finished` using that pre-yield duration.

## Solution Assessment

Unclear. Adding yielded metadata is the narrowest compatibility step, but deferring `finished` is the only option that fully fixes duration semantics, so maintainers should choose the observable contract first.
