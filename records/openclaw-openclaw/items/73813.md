## Summary

Keep open. Current main still has the reported cron observability gap: yielded embedded runs expose paused/yielded metadata, but isolated cron finalization and run-log typing still collapse the first nonfatal result to terminal `ok` with the first execution window's duration.

## What This Changes

Fix isolated cron run logging so `sessions_yield` plus subagent orchestration does not record terminal `ok` and a short duration before descendant work and parent resume have actually completed.

## Best Possible Solution

Cron run history should model yielded isolated orchestration as nonterminal until the parent and descendants reach a real terminal result, with any additive yielded metadata documented only if maintainers choose it as a compatibility step.

## Reproduction Assessment

Yes. The issue gives concrete isolated-cron steps, and current source still maps a yielded embedded run into terminal cron `ok` bookkeeping without consuming yielded/paused metadata.

## Solution Assessment

Unclear. Deferring `finished` best matches the requested status and duration semantics; adding a `yielded` flag is smaller but only makes the partial duration observable.
