## Summary

Keep open. Current main still has the reported cron observability gap: a yielded isolated cron agent turn can be returned as terminal `ok`, and the cron timer records duration around that returned run instead of the later parent/subagent completion.

## What This Changes

The issue asks cron run status and duration to reflect the full isolated `sessions_yield` plus subagent workflow instead of the first pre-yield execution window.

## Best Possible Solution

Define the cron lifecycle contract for yielded isolated runs, then implement it consistently so monitoring can distinguish in-progress/yielded work from real terminal success.

## Reproduction Assessment

Yes, from source inspection. The issue gives concrete isolated-cron steps, and current main still returns yielded embedded runs through cron `ok` finalization without consuming the yielded or paused metadata.

## Solution Assessment

Unclear. Deferring terminal `finished` events best fixes status and duration, while adding yielded metadata is smaller but preserves early terminal `ok`; maintainers need to choose the cron event/run-log contract.
