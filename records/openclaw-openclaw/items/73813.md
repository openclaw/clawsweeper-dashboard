## Summary

Keep open. Current main still has the reported cron observability gap: the embedded runner exposes yielded/paused metadata, but isolated cron finalization does not consume it and the timer records a terminal finished event from the first cron core result.

## What This Changes

The issue requests accurate cron run status and duration when an isolated cron job yields to a subagent and later resumes.

## Best Possible Solution

Cron run bookkeeping should treat yielded isolated orchestration as nonterminal until the resumed parent and descendant subagents reach a true terminal outcome. If maintainers want a staged rollout, cron should at least surface explicit yielded/pending metadata without pretending the measured duration is complete.

## Reproduction Assessment

Yes. The issue gives concrete isolated cron steps, and current source inspection traces the same path: sessions_yield produces paused/yielded embedded metadata, then cron can convert the first clean parent result into terminal ok status and short duration.

## Solution Assessment

No current fix is in place. Deferred finalization best matches the expected status and duration behavior; additive yielded metadata is a safer partial step but does not by itself make durationMs reflect the full orchestration.
