## Summary

Keep open. Current main has a delivery-path mitigation for descendant subagents, but cron outcome bookkeeping still does not treat a yielded embedded run as nonterminal when delivery is none or bypassed, so the reported short `ok` run duration remains source-reproducible.

## What This Changes

The issue asks cron run status and duration to reflect the full isolated `sessions_yield` plus subagent workflow instead of finishing at the parent turn's yield point.

## Best Possible Solution

Cron run lifecycle should stop treating yielded embedded runs as terminal success and should either defer terminal run-log bookkeeping until parent and descendant work reach a real terminal state, or expose a clearly nonterminal yielded/pending outcome for compatibility.

## Reproduction Assessment

Yes, source-reproducible. The issue gives concrete isolated-cron steps, and current source still maps yielded embedded results into terminal cron `ok` bookkeeping outside the delivery-only descendant wait path.

## Solution Assessment

Unclear. Deferring cron `finished` best matches the reported status and duration semantics, while additive yielded metadata is smaller but changes the observability contract less; maintainers need to choose the cron run-log contract.
