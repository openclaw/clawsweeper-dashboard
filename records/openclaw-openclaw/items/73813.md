## Summary

Keep open. Current main still has the reported cron observability gap: yielded embedded runs expose paused/yielded metadata, but isolated cron finalization and run-log events still collapse the first nonfatal run result to terminal `ok` with the pre-yield duration.

## What This Changes

The issue asks isolated cron agent runs that yield to subagents to avoid logging an immediate terminal `ok` result and partial duration before the yielded parent flow completes.

## Best Possible Solution

Cron run logging should make yielded isolated orchestration visibly nonterminal until the resumed parent and descendants reach a real terminal outcome, or add documented yielded/pending metadata as an intentional compatibility step.

## Reproduction Assessment

Yes. The issue gives concrete isolated-cron steps, and current source confirms that a yielded embedded result still flows into terminal cron `ok` bookkeeping without consuming yielded/paused metadata.

## Solution Assessment

Unclear. The current behavior is not sufficient, but the best fix needs a maintainer choice between a smaller additive observability field and a broader deferred-finished lifecycle change.
