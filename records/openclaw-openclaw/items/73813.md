## Summary

Keep open. Current main still has the reported cron observability gap: isolated cron execution can return a yielded embedded run as terminal `ok`, and the cron finished event records the first execution window duration instead of the full resumed parent-plus-descendant workflow.

## What This Changes

The issue asks cron run status and duration to reflect the full isolated `sessions_yield` subagent workflow rather than the first pre-yield parent turn.

## Best Possible Solution

Cron run lifecycle should reflect real terminal orchestration state: avoid terminal `finished` records until yielded parent and descendant work completes, or explicitly mark partial yielded outcomes if compatibility requires an interim step.

## Reproduction Assessment

Yes. The issue gives concrete isolated-cron steps, and current source still maps yielded embedded results into terminal cron `ok` bookkeeping without consuming the yielded or paused metadata.

## Solution Assessment

Unclear. Deferring `finished` best matches the expected status and duration semantics, while additive yielded metadata is smaller but partial; maintainers need to choose the cron event/run-log contract.
