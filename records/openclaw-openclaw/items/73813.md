## Summary

Keep open. Current main still has the cron observability gap for isolated cron runs that yield to subagents, especially with `delivery.mode: "none"`: the embedded runner exposes yielded metadata, but cron run classification and timer finalization still treat the first yielded parent turn as a terminal successful cron run.

## What This Changes

The issue asks cron run status and duration for isolated jobs that call `sessions_yield` after spawning subagents to stay nonterminal, or otherwise reflect the full parent-plus-subagent orchestration time.

## Best Possible Solution

Cron run bookkeeping should treat yielded isolated agent turns as nonterminal until the resumed parent and descendant subagents reach a real terminal outcome, so `finished` and `durationMs` represent the full orchestration. If maintainers want a staged path, add explicit yielded/pending metadata first while keeping deferred finalization tracked.

## Reproduction Assessment

Yes. The report gives a concrete isolated cron `agentTurn` with `sessions_spawn`, `sessions_yield`, and a long-running child; current source inspection shows the no-delivery cron path still returns a normal `ok` outcome before descendant completion is consumed.

## Solution Assessment

Unclear. Deferred finalization best matches the expected monitoring semantics, while an additive yielded marker is safer and narrower but only partially addresses misleading duration and terminal status.
