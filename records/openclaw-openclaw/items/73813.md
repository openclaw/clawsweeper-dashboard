## Summary

Keep open. Current main still has the reported cron observability gap: yielded embedded runs carry paused metadata, but cron outcome/finalization and finished-event bookkeeping still treat the first nonfatal isolated run result as terminal `ok` with the short first-run duration.

## What This Changes

The issue asks for isolated cron run status and duration to reflect subagent work after `sessions_yield`, not only the parent turn before yield.

## Best Possible Solution

Cron bookkeeping should represent yielded isolated orchestration as nonterminal or explicitly yielded until the resumed parent and descendant subagents reach a real terminal result, with run-log duration semantics documented and tested.

## Reproduction Assessment

Yes. The issue gives concrete isolated-cron steps, and current source maps to the same path: an embedded `meta.yielded` result can still flow into terminal cron `ok` bookkeeping before descendant work drains.

## Solution Assessment

Unclear. Deferring `finished` best matches the reported status and duration semantics, while adding yielded metadata is a smaller backward-compatible step that does not fully fix duration metrics by itself.
