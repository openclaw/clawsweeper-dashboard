## Summary

Keep open. Current `main` still maps Gmail payloads into `messageTemplate`, forwards that text as a hook `agentTurn`, wraps it as untrusted content, and submits it as the next prompt. I found no `inputRole`-style hook mapping field, no Gmail default to tool-scoped ingestion, and no synthetic assistant-tool-call/tool-result append path for hook payloads.
