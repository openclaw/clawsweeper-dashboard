## Summary

Keep open. Current main still routes ordinary non-sandboxed browser tool calls to the shared host browser control path and shared default `openclaw` profile; the sandbox browser can isolate by agent/session, but it remains opt-in and does not satisfy the requested default isolation.

## What This Changes

Request default per-agent browser isolation so simultaneous agents do not share Chrome process, tabs, cookies, storage, or MCP tool state.

## Best Possible Solution

Define the browser isolation contract, then implement default per-agent host profiles or automatic sandbox-browser isolation while preserving an explicit opt-in shared profile path.

## Reproduction Assessment

Yes. In current main, two non-sandboxed agents calling `browser` without `target` or `profile` both use the host target and forward `profile: undefined`, which resolves to the single default `openclaw` profile.

## Solution Assessment

Unclear. The existing sandbox-browser path is a useful building block, but making it the default or deriving host profiles per agent needs a maintainer decision on lifecycle, persistence, and compatibility.
