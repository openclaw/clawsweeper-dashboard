## Summary

Keep open. The PR targets valid Control UI UX feedback from #75947, but the patch is not merge-ready because the search bypass misses top-level self-matches and the new `docLink` field is not carried through the protocol schema.

## What This Changes

The PR marks selected config fields as advanced, renders them in a collapsed Control UI disclosure, and adds a `docLink` hint rendered as a Docs anchor.

## Best Possible Solution

Update this PR so active search always bypasses the advanced disclosure, `docLink` is represented in the public hint contract, and the UX change has focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is enough: search criteria that match the top-level section itself leave `childSearchCriteria` undefined, so the PR line that checks only `childSearchCriteria` fails to bypass the new advanced split.

## Solution Assessment

No. The basic/advanced split is a reasonable direction, but it should key the bypass off the original active `searchCriteria`, complete the `docLink` protocol contract, and add a render/search regression test before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bypass the split for self-matched searches:** `ui/src/ui/views/config-form.node.ts:1064`
  - body: When the search matches a top-level section itself, `childSearchCriteria` is intentionally cleared so children render unfiltered. This new check then decides search is inactive and still wraps advanced children in the collapsed disclosure, so queries such as `tag:advanced` or a section-label search no longer show all matching fields as promised. Use the original active `searchCriteria` to bypass the split.
  - confidence: 0.9
- **[P2] Add docLink to the protocol hint schema:** `src/shared/config-ui-hints-types.ts:11`
  - body: The new `docLink` property is only added to the shared TypeScript type. `ConfigUiHintSchema` still has `additionalProperties: false` and does not allow that key, so any future hint using this affordance will violate the `config.schema` / lookup contract for schema-aware clients. Add the field to the protocol schema and matching public hint surfaces before relying on it.
  - confidence: 0.82
