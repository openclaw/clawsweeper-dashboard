Issue requests DashScope usage API support so the UI does not show usage as zero while tokens are actually consumed.

## Best Possible Solution

Add Qwen-owned usage auth and usage snapshot support, with DashScope alias wiring, docs, and tests after the provider quota endpoint contract and UI window semantics are confirmed.

Yes, source-level: current main's provider usage ID union and resolver omit Qwen/DashScope aliases, so the auth-status usage path cannot select them. I did not run a live DashScope credential scenario in this read-only review.

## Solution Assessment
