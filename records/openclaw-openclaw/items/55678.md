## Summary

Keep this open. Current main still lacks DashScope/Qwen provider-usage quota snapshots: Qwen/DashScope do not resolve to a supported usage-provider id, the Qwen plugin has no usage auth or snapshot hook, and the status/UI usage paths only load quota windows for resolved usage providers.

## What This Changes

The issue requests DashScope/Qwen usage API support so UI/status token usage no longer shows zero while tokens are consumed.

## Best Possible Solution

Add Qwen-owned usage-window support through the existing provider plugin hooks, with the smallest necessary core type/alias additions so qwen, qwencloud, modelstudio, and dashscope resolve to the bundled Qwen usage provider. Keep provider-specific endpoint/auth parsing in extensions/qwen, add focused usage/status tests, and update Qwen plus token-use docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure/use provider qwen, qwencloud, modelstudio, or dashscope and resolveUsageProviderId returns undefined, so the status and gateway auth/status usage paths never request Qwen/DashScope quota windows.

## Solution Assessment

No current solution exists on main. The narrow maintainable fix is not only adding dashscope to usageProviders; it also needs Qwen-owned resolveUsageAuth and fetchUsageSnapshot behavior backed by official DashScope/Qwen usage API contracts.
