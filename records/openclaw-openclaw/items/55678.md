## Summary

Keep this open. Current main still lacks Qwen/DashScope provider quota snapshots: Qwen/DashScope is a bundled provider surface, but the usage-provider union and allowlist exclude it, the Qwen plugin has no usage auth/snapshot hooks, and status/UI enrichment only runs after a provider resolves to a supported usage id.

## What This Changes

Add Qwen/DashScope usage API support so UI/status usage surfaces can report provider usage for DashScope accounts.

## Best Possible Solution

Implement provider-owned Qwen/DashScope usage-window hooks in the bundled Qwen plugin, with minimal core usage-id alias/type support and aligned docs, changelog, and regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: configure Qwen/DashScope and the status/auth usage path calls `resolveUsageProviderId`, which returns `undefined`, so no Qwen usage summary is requested.

## Solution Assessment

No current solution exists on main. The maintainable fix is Qwen-owned `resolveUsageAuth` and `fetchUsageSnapshot` support plus minimal alias/type wiring; adding only `dashscope` to the static allowlist would not fetch provider usage data.
