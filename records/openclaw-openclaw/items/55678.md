## What This Changes

The issue requests DashScope/Qwen provider usage API support so UI usage and cost/quota surfaces do not show zero while Qwen tokens are consumed.

## Best Possible Solution

Add Qwen-owned resolveUsageAuth and fetchUsageSnapshot support, plus narrow usage-provider alias/type/docs/test wiring, after confirming the DashScope quota endpoint contract.

## Reproduction Assessment

Yes, source-level: current main's provider-usage union/resolver cannot return Qwen or DashScope aliases, and models.authStatus maps through that resolver before loading usage summaries. I did not run a live Qwen credential scenario in this read-only review.

## Solution Assessment
