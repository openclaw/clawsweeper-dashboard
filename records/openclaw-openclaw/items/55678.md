## Summary

Keep open: current main and the latest release still omit Qwen/DashScope from provider-usage IDs, the Qwen plugin declares the affected provider aliases but has no usage hooks, and the related closed DashScope issues/PRs cover per-response streaming token capture rather than provider usage-window snapshots.

## What This Changes

Add DashScope/Qwen provider usage API support so UI/status usage surfaces can show provider-reported usage windows for Qwen users.

## Best Possible Solution

Add Qwen-owned usage-window support through `resolveUsageAuth` and `fetchUsageSnapshot`, with minimal core alias/type wiring, docs, and regression coverage after verifying DashScope endpoint semantics.

## Reproduction Assessment

Yes, source-level. `resolveUsageProviderId('dashscope')` and `resolveUsageProviderId('qwen')` are excluded by the closed allowlist/type, and `models.authStatus` calls that resolver before loading usage summaries.

## Solution Assessment

No, not as a static allowlist-only patch. The maintainable solution is provider-owned Qwen usage auth and snapshot hooks plus narrow alias/type/docs wiring once the DashScope usage endpoint contract is verified.
