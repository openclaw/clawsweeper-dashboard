## Summary

Keep open: current main and v2026.6.10 still do not support Qwen/DashScope provider account-usage snapshots, and the remaining work needs provider-owner/product decisions beyond adding an allowlist entry.

## What This Changes

Request to add DashScope/Qwen provider-usage support so UI/account usage no longer appears empty or zero while tokens are being consumed.

## Best Possible Solution

Add Qwen-owned usage auth and usage snapshot hooks, then route Qwen/DashScope aliases through the existing resolver with docs, focused tests, and redacted live endpoint proof.

## Reproduction Assessment

Yes, at source level: resolveUsageProviderId cannot return Qwen/DashScope IDs, and models.authStatus filters through that resolver before usage polling. I did not run a live DashScope credential scenario in this read-only review.

## Solution Assessment

No, the issue's allowlist-only proposal is not the best complete fix because provider polling also needs Qwen-owned auth extraction and quota/balance response normalization. The maintainable path is a provider plugin usage hook plus resolver alias wiring and endpoint-contract proof.
