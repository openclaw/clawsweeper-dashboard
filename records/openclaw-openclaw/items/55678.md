## Summary

Keep this open. Current main still lacks Qwen/DashScope provider quota snapshots: the supported usage-provider set excludes Qwen/DashScope, Qwen does not provide usage auth/snapshot hooks, and the UI/status paths only load quota windows after a provider resolves to a supported usage id.

## What This Changes

Add DashScope/Qwen provider usage API support so UI and status usage surfaces can show provider quota usage for China/APAC Qwen users.

## Best Possible Solution

Add Qwen-owned usage-window support through the existing provider plugin hooks, with the smallest necessary core type/alias additions so `qwen`, `qwencloud`, `modelstudio`, and `dashscope` resolve to the bundled Qwen usage provider.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: configure a Qwen/DashScope provider and `resolveUsageProviderId` returns `undefined`, so status/UI usage paths never request a Qwen/DashScope quota window.

## Solution Assessment

No current solution exists on main. The narrow maintainable fix is provider-owned Qwen/DashScope `resolveUsageAuth` and `fetchUsageSnapshot` support, not only adding `dashscope` to the static allowlist.
