## Summary

Keep open: current main and the latest release still exclude Qwen/DashScope from provider-usage IDs, so the UI/status quota path never asks the Qwen plugin for a usage snapshot; related closed DashScope token-streaming issues cover per-response token capture, not this usage-window provider support.

## What This Changes

Request Qwen/DashScope usage API support so the UI can show provider usage/cost instead of zero for DashScope users.

## Best Possible Solution

Add Qwen-owned usage-window support through `resolveUsageAuth` and `fetchUsageSnapshot`, with minimal core alias/type wiring, docs, and regression coverage after verifying the DashScope endpoint contract.

## Reproduction Assessment

Yes, source-level. On current main, `resolveUsageProviderId('dashscope')` and `resolveUsageProviderId('qwen')` are excluded by the closed allowlist/type, and the status/UI paths call that resolver before loading usage summaries.

## Solution Assessment

No, not as a static-list-only patch. The maintainable solution is provider-owned Qwen usage auth and snapshot hooks plus narrow alias/type/docs wiring once the DashScope usage endpoint contract is verified.
