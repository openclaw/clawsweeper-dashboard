## What This Changes

The branch adds Gateway chat-run state for streamed `NO_REPLY` underscore-fragment provenance, suppresses matching final Control UI chat payloads, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a current-main Gateway fix that narrowly tracks buffered silent-token lead-fragment provenance, suppresses `NO_` finals, preserves intentional `NO`, and keeps broader silent-token policy changes separate.

## Reproduction Assessment

Yes, at source level: on current main, an assistant event with `data.text: "NO_"` is buffered during delta suppression, then lifecycle end finalizes with lead-fragment suppression disabled. I did not run a live Gateway/WebChat runtime in this read-only review.

## Solution Assessment

Yes for the narrow bug: the PR suppresses final payloads only when prior streamed underscore-fragment provenance exists, while preserving uppercase `NO`. The broader prefix/suffix cleanup should remain in https://github.com/openclaw/openclaw/issues/40165.
