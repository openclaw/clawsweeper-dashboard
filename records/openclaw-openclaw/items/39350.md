## Summary

Keep open. Current main still leaves the lifecycle-ending `NO_` Gateway/WebChat final-message leak reachable, and this PR is the narrow fix candidate, but the current head is dirty against main and the latest provenance change suppresses by a process-wide caller-provided run key instead of preserving run/session or fragment equality.

## What This Changes

The PR adds Gateway chat state, final-message suppression logic, regression coverage, and a changelog entry for suppressing buffered `NO_REPLY` lead fragments while preserving intentional uppercase `NO` replies.

## Best Possible Solution

Land a rebased narrow Gateway fix that records lead-fragment provenance on a server-owned run/session identity, or keeps the exact suppressed fragment text check, with regression coverage for lifecycle-ending `NO_` and intentional uppercase `NO`. Keep the broader suffix/prefix hardening discussion in #40165 separate from this final-fragment fix.

## Reproduction Assessment

Yes. A focused reproduction is to register a chat run, emit one assistant event with `data.text: "NO_"`, then emit lifecycle end; current main suppresses the delta but finalization reprojects with lead-fragment suppression disabled and can emit the fragment as the final message.

## Solution Assessment

No. The PR direction is the narrow maintainable fix, but this head is not the best landing state until it is rebased and the suppression provenance is scoped or compared precisely enough to avoid cross-run final suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve provenance when suppressing final fragments:** `src/gateway/server-chat.ts:715`
  - body: The new `Set` turns lead-fragment provenance into a boolean keyed only by `clientRunId`, and finalization suppresses any later buffered text for that key. Since `clientRunId` comes from caller-supplied idempotency keys and the chat state is process-wide, a reused key can make an unrelated final message disappear; keep the stored fragment-text equality check or namespace this by a server-owned run/session identity.
  - confidence: 0.82

## Security Review

Status: needs_attention
