## Summary

Keep open. Current main still has the reported Control UI reconnect bug, and the linked implementation candidate #72522 remains open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Prevent Control UI WebChat from replacing an authenticated chat session with the credentials gate during transient WebSocket reconnects after tab refocus.

## Best Possible Solution

Review and land #72522 or an equivalent narrow Control UI fix that preserves the authenticated chat UI across transient disconnects and only returns to the credentials gate before first connection or after explicit auth/pairing failure.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: after a successful session, a recoverable close sets connected to false and renderApp immediately returns the credentials gate before reconnect restores chat.

## Solution Assessment

Yes. A sticky prior-successful-session or explicit reconnecting state is the narrow maintainable direction; a timing-only debounce would still conflate transient transport loss with first-connect and auth-failure states.
