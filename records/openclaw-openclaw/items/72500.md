## Summary

Keep open. Current main still renders the full Control UI credentials gate whenever the app is disconnected, the gateway reconnect path still clears `connected` immediately, and the linked closing PR is open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that the Control UI webchat preserve an authenticated chat view, or show a reconnecting state, instead of flashing the credentials gate during transient reconnects after tab refocus.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72522, or an equivalent narrow Control UI fix that preserves the authenticated chat UI across transient disconnects and only returns to the credentials gate before first connection or after explicit auth/pairing failure.

## Reproduction Assessment

Yes, by source inspection on current main. A reconnect or close path clears `connected`, and `renderApp` maps `connected === false` directly to the credentials gate.

## Solution Assessment

Yes, directionally. A sticky prior-successful-session or explicit reconnecting state is the narrow maintainable fix; a timing-only debounce would still conflate transient transport loss with first-connect and auth-failure states.
