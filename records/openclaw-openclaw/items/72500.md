## Summary

Keep open. Current main still renders the Control UI credentials gate on any disconnected state, and #72522 remains the open unmerged implementation candidate for the same behavior.

## What This Changes

The issue asks Control UI webchat to keep authenticated chat content mounted during transient reconnects instead of flashing the full credentials gate after tab refocus.

## Best Possible Solution

Review and land #72522, or an equivalent narrow Control UI fix, so first-connect/auth-failure is distinct from transient transport reconnects and covered by focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a recoverable close sets `connected` false, and the renderer immediately returns the credentials login gate before the next successful hello restores chat.

## Solution Assessment

Yes. A sticky prior-successful-connection flag or explicit reconnecting state is the narrow maintainable fix; a timing-only debounce would still conflate transient transport loss with first-connect and auth-failure states.
