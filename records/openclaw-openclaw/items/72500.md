## Summary

Keep open. Current main still has the reported Control UI reconnect path, and the linked closing PR remains open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Control UI webchat avoid showing the credentials gate during transient reconnects after a successful session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72522, or an equivalent narrow Control UI fix that preserves the authenticated chat UI across transient disconnects and returns to the credentials gate only before first connection or after explicit auth/pairing failure.

## Reproduction Assessment

Yes, by source inspection on current main. A recoverable close or reconnect sets `connected` to false, and `renderApp` immediately maps that state to the full credentials gate.

## Solution Assessment

Yes, directionally. A sticky prior-successful-session or explicit reconnecting state is the narrow maintainable fix; a timing-only debounce would still conflate transient transport loss with first-connect and auth-failure states.
