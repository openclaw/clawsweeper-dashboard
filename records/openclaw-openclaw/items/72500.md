## Summary

Keep open. Current main still has the reported Control UI state collapse: transient reconnects set `connected` false and `renderApp` immediately renders the credentials gate, while #72522 is the open unmerged PR candidate that explicitly closes this issue.

## What This Changes

The issue asks Control UI WebChat to preserve the authenticated chat surface during transient reconnects instead of flashing the full credentials gate.

## Best Possible Solution

Review and land #72522, or an equivalent narrow Control UI fix, so first-connect/auth-failure remains distinct from transient transport reconnects and the behavior is covered by focused tests.

## Reproduction Assessment

Yes. The source path is high confidence: after a successful session, an active close sets `connected` false and `renderApp` immediately returns the credentials gate before reconnect can restore chat.

## Solution Assessment

Yes. A sticky prior-successful-session flag or explicit reconnecting state is the maintainable fix direction; a timing-only debounce would still conflate transient transport loss with first-connect and auth-failure states.
