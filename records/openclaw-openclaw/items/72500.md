## What This Changes

The issue asks for Control UI webchat to keep an authenticated chat view mounted, or show a reconnecting state, instead of flashing the credentials gate during transient tab-refocus reconnects.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/72522, or an equivalent replacement, that keeps authenticated chat content mounted during transient reconnects while returning to the credentials gate on first connect and explicit auth, pairing, or revocation failure.

## Solution Assessment

Yes, directionally. A sticky prior-successful-session state or explicit reconnecting state is the narrow maintainable fix shape, but the active PR needs to land or be replaced before this issue can close.
