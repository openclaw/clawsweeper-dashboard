## Summary

Keep open: current main and the latest release still route every disconnected Control UI render to the credentials gate, and the related fix PRs are closed unmerged rather than a safe canonical landing path.

## What This Changes

The issue reports that Control UI replaces an already-authenticated chat/dashboard view with the credentials gate during transient WebSocket reconnects.

## Best Possible Solution

Add an auth-safe prior-success or reconnecting state in Control UI so transient transport reconnects preserve the mounted shell while explicit auth, pairing, scope, device, or revocation failures return to the credentials gate.

## Reproduction Assessment

Yes, at source level: current render code returns the login gate for every disconnected state, and gateway reconnect/close paths set connected=false. I did not run live browser or Pi proof in this read-only pass.

## Solution Assessment

Unclear as a final patch: preserving the shell after first success is the right direction, but previous closed PRs show the auth failure reset boundary needs maintainer judgment.
