## Summary

Keep open. Current main and the latest release still have the reported two-state Control UI reconnect behavior, and https://github.com/openclaw/openclaw/pull/72522 is an open, unmerged, currently conflicting implementation candidate that references this issue with closing syntax.

## What This Changes

The issue reports that Control UI webchat briefly renders the credentials gate after tab refocus because transient WebSocket closes set `connected=false` and the renderer treats that as unauthenticated.

## Best Possible Solution

Control UI should preserve the authenticated dashboard across transient reconnects and return to the credentials gate only before first connection or after explicit auth, pairing, or revocation failure.

## Reproduction Assessment

Yes. By source inspection on current main, `connectGateway` and `onClose` set `connected=false`, and `renderApp` immediately renders the credentials gate for any `!connected` state.

## Solution Assessment

Yes, directionally. A sticky prior-successful-session or explicit reconnecting state is the narrow maintainable fix; the linked PR is the current candidate but should land or be replaced before this issue closes.
