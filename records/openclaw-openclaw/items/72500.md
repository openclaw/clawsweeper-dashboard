## Summary

Keep open. Current main still routes every disconnected Control UI state to the credentials gate, and the linked open PR #72522 remains the implementation candidate rather than a merged fix.

## What This Changes

The issue asks Control UI webchat to keep the authenticated chat surface mounted during transient reconnects instead of flashing the credentials gate on tab refocus.

## Best Possible Solution

Land #72522 or an equivalent narrow Control UI fix that records a successful session, keeps chat mounted during recoverable transport reconnects, and restores the credentials gate only for first connect or explicit auth/pairing/device-identity failure.

## Reproduction Assessment

Yes. The source path is high confidence: after a successful hello, a recoverable WebSocket close calls the app close handler, clears `connected`, and `renderApp` returns the login gate before reconnect recovery completes.

## Solution Assessment

Yes. A sticky prior-successful-session or explicit reconnecting state is the maintainable fix direction; a timing-only debounce would still conflate transient transport loss with first-connect and auth-failure states.
