## Summary

Keep open. Current main still routes every disconnected Control UI state to the credentials gate, and the provided GitHub context shows an open, unmerged PR with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks Control UI WebChat to keep the authenticated chat UI mounted across transient tab/refocus reconnects instead of immediately showing the credentials gate whenever `connected` is false.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72522 or an equivalent narrow Control UI change that records a successful session, keeps chat UI mounted during recoverable transport reconnects, restores the credentials gate only for first-connect or explicit auth/pairing/device-identity failures, and covers those transitions with focused tests.

## Reproduction Assessment

Yes. The source path is high confidence: a recoverable WebSocket close calls `onClose`, `app-gateway` clears `connected`, and `renderApp` immediately selects the login gate before reconnect is scheduled.

## Solution Assessment

Yes. A sticky prior-successful-session or explicit reconnecting state is the maintainable fix; a timing-only debounce would still conflate transient transport loss with first-connect and auth-failure states.
