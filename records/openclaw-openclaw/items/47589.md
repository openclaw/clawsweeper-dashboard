## Summary

The central false-positive bug still exists on current main, but this PR’s dot-all conjunction matcher can create new cross-sentence false positives; the branch also needs current-main refresh and real delivered-reply proof before merge.

## What This Changes

The PR replaces broad bare-`remember` detection with narrower direct, `remember to`, and conjunction patterns and adds focused reminder-guard unit tests.

## Best Possible Solution

Remove bare memory-retention matching while constraining any conjunction recovery to the same local commitment clause, with negative cross-sentence coverage and real before/after delivered-reply proof.

## Reproduction Assessment

Yes, from source: current main matches `I'll remember your preference`, and the reply runner appends the note when session cron suppression is false. No live current-main channel reproduction was performed during this read-only review.

## Solution Assessment

No, not yet: removing bare `remember` is the right direction, but an unbounded dot-all conjunction matcher replaces one false-positive class with another; a clause-bounded form is safer.
