## What This Changes

The PR changes memory-wiki source ingest to generate collision-resistant source page IDs/paths from a readable slug plus canonical-path hash, reuse matching legacy source pages, and add regression tests for collisions, symlinks, fallback titles, and lookup scope.

## Best Possible Solution

Land the focused memory-wiki plugin fix after adding the changelog entry, refreshing against current main, and rerunning the memory-wiki/changed validation; leave broader source identity indexing or source-heavy performance redesign as a separate follow-up.

## Reproduction Assessment

Yes. Current main can be reproduced from source alone: ingesting two local files whose titles normalize to the same slug computes the same `source.<slug>` ID and `sources/<slug>.md` path, then writes the second file to the same page.

## Solution Assessment

Yes for the behavior. A canonical-path hash suffix plus legacy source-page reuse is the narrowest maintainable fix for same-slug overwrites; the remaining changes needed are release-note and refresh work, not a different design.
