## Summary

Keep open. The underlying bug is still valid on current main, but this PR should not merge as-is because it changes the removed `src/plugins/memory-host` embedding path while the active OpenAI memory embedding provider now lives under `extensions/openai`.

## What This Changes

The PR adds OpenAI `outputDimensionality` validation and `dimensions` request-body plumbing for memory-search embeddings, with tests, under the old memory-host embedding files.

## Best Possible Solution

OpenAI memory embeddings should preserve the unset request shape, validate configured dimensions for supported OpenAI embedding models, send `dimensions` in both inline and batch request bodies, and include the value in cache/reindex identity with docs and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `memorySearch.outputDimensionality` is accepted, resolved, and passed to provider adapters, while the active OpenAI inline and batch request bodies omit `dimensions`.

## Solution Assessment

No. The proposed logic is directionally useful, but the branch changes removed memory-host files; the maintainable fix is a port or replacement against `extensions/openai` and the current memory embedding tests.
