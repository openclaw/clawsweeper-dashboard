## Summary

Keep open: the underlying OpenAI embedding dimensionality bug is still valid on current main, but this PR targets obsolete `src/plugins/memory-host` files instead of the active `extensions/openai` provider path, so it needs a port or replacement rather than merge as-is.

## What This Changes

The PR adds OpenAI `outputDimensionality` validation, `dimensions` request-body plumbing, and tests under the old memory-host embedding provider files.

## Best Possible Solution

Port the intended behavior into the current provider-plugin architecture: store the configured OpenAI output dimensionality on the OpenAI embedding client/runtime, send `dimensions` for supported `text-embedding-3-*` inline and batch requests only when configured, include it in cache/reindex identity, and document the OpenAI setting.

## Reproduction Assessment

Yes. Source inspection on current main shows `memorySearch.outputDimensionality` is accepted and resolved, while the active OpenAI inline and batch request bodies omit `dimensions`.

## Solution Assessment

No. The proposed logic is directionally useful, but the branch changes removed memory-host files; the narrow maintainable solution is a replacement or rebase that changes `extensions/openai` and the current memory embedding tests.
