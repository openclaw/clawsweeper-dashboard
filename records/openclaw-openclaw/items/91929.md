## Summary

Keep this open as a provider feature request, not a confirmed regression: OpenClaw intentionally requires a registered image-generation provider, and the bundled MiniMax provider does not currently forward image requests to arbitrary configured proxy hosts.

## What This Changes

Support MiniMax image generation through an operator-configured custom proxy endpoint.

## Best Possible Solution

Add an explicit MiniMax-plugin-owned image endpoint override, or another equally explicit provider contract, that supports the native MiniMax request/response shape and uses the existing `request.allowPrivateNetwork` safeguards; do not implicitly convert arbitrary chat providers into image providers.

## Reproduction Assessment

Yes, at source level with high confidence: `custom/image-01` selects provider ID `custom`, and the image runtime emits the reported error because no image-generation provider is registered under that ID. The supplied proxy request also uses MiniMax’s native protocol, which current MiniMax code only sends to official hosts.

## Solution Assessment

No, implicitly reusing any custom chat-provider definition is not the best solution because image protocols are separately registered and may differ from chat APIs. A narrow MiniMax image-endpoint contract is safer and keeps protocol, credentials, and network policy with the owning plugin.
