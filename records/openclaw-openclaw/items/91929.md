## Label Justifications

- `P3`: The issue is a useful but limited custom-proxy enhancement; official MiniMax image/search paths remain available and core chat is unaffected.
- `impact:auth-provider`: Provider identity, credential lookup, and endpoint routing prevent the proxy-backed specialized capabilities from being selected.

## Summary

This remains necessary as a provider-capability enhancement, not a broken existing custom-model contract: current main deliberately requires registered plugins for image generation and web search, and the MiniMax plugin explicitly ignores private custom text endpoints for image generation while selecting fixed regional search endpoints. The reporter’s follow-up usefully confirms the gap spans both specialized capabilities, so maintainers should decide on a capability-specific proxy configuration rather than closing this as ordinary setup support.

## What This Changes

Add custom-proxy endpoint support for MiniMax image generation and web search, rather than expecting a generic custom chat-provider declaration to register those specialized capabilities.

## Best Possible Solution

Add explicit MiniMax plugin-owned capability endpoint overrides—separate image-generation and web-search URLs if their contracts differ—using the shared guarded provider HTTP helpers, canonical `minimax/*` model IDs, focused fresh/upgrade config coverage, and documentation that custom inference providers do not implicitly register specialized tools.

## Reproduction Assessment

Yes, at source level with high confidence: `custom/image-01` reaches a registry lookup that cannot resolve `custom`, and current tests explicitly prove MiniMax image generation ignores a private configured text endpoint. A live instance of the reporter's proxy was not exercised.

## Solution Assessment

No, implicitly promoting every custom inference provider into image-generation and web-search providers would create an unsafe competing contract. The narrower solution is explicit capability-specific proxy support owned by the MiniMax plugin.
