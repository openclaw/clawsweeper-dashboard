## Summary

Close as ClawHub/plugin-scoped: current main intentionally resolves image generation and web search through registered capability providers, and the existing plugin API can support a custom proxy adapter without adding provider-specific behavior to OpenClaw core.

## What This Changes

The issue requests using a custom `models.providers` proxy entry for image generation and web search, after chat works but `imageGenerationModel` fails with no registered image-generation provider.

## Best Possible Solution

Keep core on registered capability providers, and route custom proxy-specific image-generation or web-search adapters through a ClawHub/provider plugin using the existing SDK seams.

## Reproduction Assessment

Yes for the source path: a custom `imageGenerationModel` provider id without a registered image-generation provider reaches the reported error in current main. I did not run the reporter's Dingtalk/proxy setup because the core-vs-plugin boundary is source and docs proven.

## Solution Assessment

No for a core `models.providers` shortcut. The better solution is a provider plugin that registers image-generation and web-search capability providers, preserving OpenClaw's existing capability ownership boundary.
