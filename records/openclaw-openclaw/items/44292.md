## Summary

Keep open. Current main still lacks an autofix, generator, or copy-paste-ready missing-label stub workflow: the parity test still asserts one key at a time, `FIELD_LABELS` remains a manual metadata table, existing config generators only cover schema/docs baselines, and the directly related implementation PR #46304 is closed unmerged.

## What This Changes

The issue requests a developer workflow that makes missing config help labels obvious and quick to fix, such as generated label stubs, an autofix command, or schema-driven label metadata.

## Best Possible Solution

Make the config help parity failure aggregate all missing root/help label keys and print conservative, editable additions for `src/config/schema.labels.ts`; keep the manual label table as the authoritative review point and leave any write-mode generator as optional follow-up.
