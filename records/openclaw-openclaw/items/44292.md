The issue requests a developer workflow improvement that makes missing config field labels obvious and quick to fix when config help keys are added.

## Best Possible Solution

Update `src/config/schema.help.quality.test.ts` to aggregate missing labels/help and print deterministic editable stubs for `src/config/schema.labels.ts`, without changing production config metadata.

## Reproduction Assessment

Yes, by source inspection. Adding a `FIELD_HELP` key without a matching `FIELD_LABELS` entry still enters the all-help-key parity loop, calls `requireLabel()`, and throws on the first missing key.

## Solution Assessment

Yes. Improving the existing parity-test diagnostic is the narrowest maintainable repair; a write-mode generator would be broader than necessary for the acceptance criteria.
