## Summary

Keep open: the PR addresses a real Discord voice session-state bug, but the current patch bypasses the central session resolver's freshness/reset guard by forwarding a stored sessionId unconditionally.

## What This Changes

The PR changes Discord STT/TTS voice turns to read the route's stored sessionId and pass it into agentCommandFromIngress, with a test mock for the session-store SDK subpath.

## Best Possible Solution

Reuse the stored Discord voice session only when the existing session resolver would consider that entry reusable, and add focused coverage for fresh reuse plus stale/reset rotation.

## Reproduction Assessment

Yes, source inspection plus issue #97688's live logs support the Discord STT/TTS statelessness path; this review did not run a fresh Discord voice repro.

## Solution Assessment

No, this is only a plausible workaround: it restores fresh-session reuse but bypasses the resolver freshness contract for stale sessions.
