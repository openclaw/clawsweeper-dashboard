## Summary

Keep open. The security hardening goal is plausible, but this PR currently introduces Windows command execution regressions already flagged by prior reviews, and it still lacks after-fix real Windows proof.

## What This Changes

This PR broadens Windows cmd.exe argument validation in `src/process/exec.ts`, adds PowerShell/null/control-character checks, and exports a validator.

## Best Possible Solution

Use a focused cmd.exe wrapper policy that preserves Windows paths and ordinary flags, redacts rejected values, and adds regression coverage for `.cmd` paths, slash switches, equals flags, and any separate PowerShell-specific path.

## Reproduction Assessment

Yes, by source inspection of the potential merge result. On `win32`, `buildCmdExeCommandLine()` sends the resolved `.cmd`/`.bat` path and every argument through `escapeForCmdExe()`, so the new validator rejects common paths and flags before spawn.

## Solution Assessment

No. The PR conflates cmd.exe metacharacters, PowerShell policy, Windows path syntax, and ordinary CLI flags; narrower validation with redacted errors and targeted tests is the safer direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Allow Windows path separators in cmd validation:** `src/process/exec.ts:53`
  - body: The expanded blocklist rejects `\\` and `/` for every value passed through `escapeForCmdExe()`, including resolved `.cmd`/`.bat` paths and valid slash-prefixed arguments. Normal Windows wrapper calls can throw before spawn instead of executing.
  - confidence: 0.94
- **[P2] Permit equals signs in wrapper arguments:** `src/process/exec.ts:53`
  - body: Rejecting `=` breaks valid arguments this repo emits, including npm update/install flags like `--loglevel=error` and `--omit=optional` when Windows falls back through command shims. `=` is not a cmd.exe command separator by itself.
  - confidence: 0.91
- **[P2] Redact rejected argument values:** `src/process/exec.ts:82-100`
  - body: The new validation errors echo `JSON.stringify(arg)`, so a rejected argument containing a token or password can be copied into exception messages, stack traces, or logs. Report the rejected class, context, and length without echoing the full value.
  - confidence: 0.9
