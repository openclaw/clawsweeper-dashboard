## Summary

Keep open: current main has not absorbed this PR’s proposed validation helper, and the branch still introduces command-execution regressions plus missing real Windows proof. Current main has separately hardened trusted `cmd.exe` resolution, but that does not make the broad argument blacklist safe or obsolete.

## What This Changes

The PR modifies `src/process/exec.ts` to broaden Windows cmd.exe argument validation, add PowerShell/null/control-character checks, and export a validation helper.

## Best Possible Solution

Keep the current narrow cmd.exe wrapper guard unless a focused hardening change rejects only true wrapper metacharacters, redacts rejected values, and adds Windows regression proof for `.cmd` paths, slash switches, equals flags, and any PowerShell-specific path.

## Reproduction Assessment

Yes, for the PR regression by source inspection. On `win32`, `.cmd` and `.bat` commands use `buildCmdExeCommandLine()`, so the proposed validator would reject resolved command paths and common flags before spawn.

## Solution Assessment

No. The broad blacklist conflates cmd.exe metacharacters, PowerShell policy, Windows path syntax, and ordinary CLI flags; a narrower wrapper policy with redacted errors and targeted Windows tests is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Allow Windows path separators in cmd validation:** `src/process/exec.ts:45`
  - body: The expanded blocklist rejects `\\` and `/` for every value passed through `escapeForCmdExe()`. That includes resolved `.cmd`/`.bat` paths and valid slash-prefixed arguments, so normal Windows wrapper calls can throw before spawn instead of executing.
  - confidence: 0.94
- **[P2] Permit equals signs in wrapper arguments:** `src/process/exec.ts:45`
  - body: Rejecting `=` breaks valid arguments this repo emits, including npm update/install flags such as `--loglevel=error` and `--omit=optional` when Windows falls back through command shims. `=` is not a cmd.exe command separator by itself.
  - confidence: 0.91
- **[P2] Redact argument values in validation errors:** `src/process/exec.ts:72-92`
  - body: The new validation errors include `JSON.stringify(arg)`, so a rejected argument containing a token, password, or API key can be copied into exception messages, stack traces, or logs. Report the rejected class, context, and length without echoing the full value.
  - confidence: 0.9
