## Summary

Keep open. The PR is a focused security-hardening proposal, but the current diff is not mergeable: it rejects normal Windows path and flag syntax in the cmd wrapper, exposes raw rejected arguments in errors, and still has no real Windows after-fix proof.

## What This Changes

The PR broadens Windows cmd.exe argument validation in `src/process/exec.ts`, adds PowerShell/null/control-character checks, and exports a validator helper.

## Best Possible Solution

Rework this as narrow cmd-wrapper hardening: reject actual cmd metacharacters, keep ordinary Windows paths, switches, and equals flags valid, redact validation errors, cover `.cmd`/npm/corepack wrapper cases in tests, and provide redacted Windows runtime proof.

## Reproduction Assessment

Yes, for the PR regression by source inspection: on `win32`, `.cmd` and `.bat` commands flow through `buildCmdExeCommandLine(resolvedCommand, ...)`, so the new blocklist rejects the resolved path and common flags before spawn. I did not run live Windows proof in this read-only review.

## Solution Assessment

No. The proposed blacklist conflates cmd.exe metacharacters, PowerShell policy, Windows path syntax, and ordinary CLI flags; a narrower wrapper-specific policy with redacted errors is the safer solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Allow Windows path separators in cmd validation:** `src/process/exec.ts:45`
  - body: The expanded blocklist rejects `\` and `/` for every value passed through `escapeForCmdExe()`. `buildCmdExeCommandLine()` passes the resolved `.cmd`/`.bat` path itself plus normal CLI arguments, so Windows wrapper calls with absolute paths or slash switches can throw before spawn.
  - confidence: 0.94
- **[P2] Permit equals signs in wrapper arguments:** `src/process/exec.ts:45`
  - body: Rejecting `=` breaks valid arguments this repo emits, including npm update/install flags such as `--loglevel=error` and `--omit=optional` when Windows falls back to command shims. `=` is not a cmd.exe command separator by itself.
  - confidence: 0.91
- **[P2] Redact argument values in validation errors:** `src/process/exec.ts:74-92`
  - body: The new errors include `JSON.stringify(arg)` for dangerous-character, PowerShell-pattern, and null-byte failures. Rejected arguments can contain tokens or passwords, so full values can leak into stack traces or logs; report class, context, and length instead.
  - confidence: 0.9
