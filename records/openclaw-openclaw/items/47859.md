## Summary

Keep open. Current main still lacks TUI image attachment support, so the PR is not obsolete, but the proposed patch is not mergeable because it rewrites ordinary prompts, can leak raw local `@path` tokens, and needs a maintainer decision around local-file upload behavior across Gateway, embedded local, and Crestodian TUI backends.

## What This Changes

The branch adds TUI image path extraction, a `/image` slash command, attachment forwarding from the Gateway TUI client to `chat.send`, and parser/loader tests.

## Best Possible Solution

Land a maintainer-reviewed TUI implementation that preserves user prompt bytes unless image tokens are actually removed, never sends raw local paths to the transcript or model, threads attachments through the shared `TuiBackend` contract, deliberately supports or rejects embedded local and Crestodian modes, reuses Gateway media limits/validation, updates TUI docs, and covers parser plus command-handler behavior in tests.

## Reproduction Assessment

Yes. The feature gap is reproducible by static inspection of current main: there is no TUI image parser, `/image` command, or TUI attachment field. The PR defects are reproducible by tracing `extractImagePaths` line 59 and the `cleanText || text` fallback at line 565 in the PR head.

## Solution Assessment

No. Reusing the existing Gateway `chat.send.attachments` contract is the right direction, but this patch is not the best mergeable implementation because it rewrites prompts, leaks raw local path tokens for image-only messages, and does not handle the current shared TUI backend/local-mode design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve prompts when no image refs are present:** `src/tui/tui-image-extract.ts:59`
  - body: `extractImagePaths` always collapses repeated whitespace and trims the result, and `sendMessage` uses that result for every non-BTW message. A normal prompt with indentation, leading whitespace, or intentional double spaces is silently changed before it reaches the model even when no image reference was extracted.
  - confidence: 0.95
- **[P2] Avoid sending raw @path text for attachment-only messages:** `src/tui/tui-command-handlers.ts:565`
  - body: For a message such as `@/tmp/screenshot.png`, `cleanText` is empty, so `message = cleanText || text` falls back to the original raw path token. That displays and sends the local path instead of sending only the attachment payload or a neutral placeholder.
  - confidence: 0.93
- **[P2] Handle local TUI mode before advertising /image:** `src/tui/commands.ts:116`
  - body: The command is added unconditionally, but current main uses the shared `TuiBackend` for Gateway, embedded local mode, and Crestodian. Embedded and Crestodian sends currently forward text only, so `/image` would appear available without a defined attachment path unless the backend contract is extended or the command is gated.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff adds local-file upload behavior and currently has concrete path-leak and size-check concerns.

Concerns:

- **[medium] Attachment-only prompts leak local file paths:** `src/tui/tui-command-handlers.ts:565`
  - body: The `cleanText || text` fallback sends raw `@/local/path` text when an image reference is the whole prompt, exposing local path details to the Gateway/model transcript instead of stripping the reference.
  - confidence: 0.93
- **[low] File size guard races the read:** `src/tui/tui-image-extract.ts:88`
  - body: The loader checks `fs.stat` before `fs.readFile`, so a file can be replaced or enlarged between the check and read; reading first and checking `buffer.byteLength` would avoid oversized local reads after the guard.
  - confidence: 0.78
