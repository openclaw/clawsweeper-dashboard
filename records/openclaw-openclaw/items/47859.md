## Summary

Keep open: current main still lacks the requested TUI image-entry feature, but this PR is not mergeable as-is because source review confirms prompt mutation, local-path leakage, and a stale Gateway-only backend boundary against the current shared TUI code.

## What This Changes

The PR adds a TUI image parser, inline `@path` extraction, a `/image` command, Gateway `chat.send` attachment forwarding, and parser/loader tests.

## Best Possible Solution

Land a rebased, maintainer-approved TUI implementation that extends the shared backend contract or gates unsupported modes, preserves user text, strips local path tokens, reuses Gateway media validation, and updates tests, docs, and changelog.

## Reproduction Assessment

Yes. Current main source shows no `/image` command, no TUI image parser, and no attachment field on the shared `TuiBackend`; the PR defects are reproducible by tracing the proposed parser and `sendMessage` fallback in the diff.

## Solution Assessment

No. Reusing the existing Gateway `chat.send.attachments` seam is the right direction, but this patch should not land until it preserves prompts, strips attachment-only path tokens, and resolves the shared Gateway/local TUI backend boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve prompts when no image refs are present:** `src/tui/tui-image-extract.ts:59`
  - body: `extractImagePaths` always collapses repeated whitespace and trims `cleanText`, and `sendMessage` uses that value for every non-BTW message. A normal prompt with indentation, leading spaces, or intentional double spaces is changed before it reaches the model even when no image token was extracted.
  - confidence: 0.95
- **[P2] Strip attachment-only paths before sending:** `src/tui/tui-command-handlers.ts:565`
  - body: For a message such as `@/tmp/screenshot.png`, `cleanText` is empty, so `message = cleanText || text` sends and displays the raw local path token instead of sending only the attachment payload or a neutral placeholder.
  - confidence: 0.93
- **[P2] Gate image commands until all TUI backends support them:** `src/tui/commands.ts:116`
  - body: The `/image` command is added to the shared command list, but the diff only forwards attachments through the Gateway client while current main also runs this command surface in local embedded mode. Gate the command for unsupported modes or add attachment handling to the shared `TuiBackend` and embedded backend.
  - confidence: 0.84
- **[P3] Check the bytes that are actually read:** `src/tui/tui-image-extract.ts:88-101`
  - body: The loader checks `fs.stat` before `fs.readFile`, so the file can be replaced or enlarged after passing the 5 MB guard. Read the buffer first and enforce the limit on `buffer.byteLength` before base64 encoding.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/tui/commands.ts:116`
  - body: This is a user-facing TUI feature, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active version before merge.
  - confidence: 0.9
