## Summary

Keep open. Current main already fixed the microphone-deny regression, but it still lacks a `permissionsPolicy` config key; this PR is useful in that direction but currently needs docs, a rebase that preserves the explicit same-origin microphone default, contributor real behavior proof, and maintainer security/API-shape review.

## What This Changes

The PR adds `gateway.http.securityHeaders.permissionsPolicy` config/schema/runtime/header plumbing and tests, and changes the gateway Permissions-Policy fallback.

## Best Possible Solution

Land a scoped, maintainer-approved `permissionsPolicy` option with docs, schema metadata, tests, and the current explicit `microphone=(self)` default, while leaving broader CSP/header policy design to https://github.com/openclaw/openclaw/issues/47565.

## Solution Assessment

No as-is. Extending `gateway.http.securityHeaders` is plausible, but this branch needs public docs, a current-main rebase that preserves `microphone=(self)`, contributor real behavior proof, and maintainer agreement on the raw header config shape.
