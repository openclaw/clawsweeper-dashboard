## What This Changes

The PR adds gateway Permissions-Policy runtime configuration, threads it through schema/runtime/HTTP handling, changes the default header, and adds focused gateway tests.

## Best Possible Solution

Land a scoped implementation of `gateway.http.securityHeaders.permissionsPolicy` through the existing `securityHeaders` contract, with public gateway docs and tests, while preserving the current microphone-safe default unless maintainers deliberately choose a different policy.

## Solution Assessment

No as-is. Extending the existing gateway `securityHeaders` path is the right shape, but the PR needs public docs, a current-main rebase/alignment, and real behavior proof before merge.
