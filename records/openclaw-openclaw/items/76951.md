## What This Changes

The PR changes Telegram sequentializer keying so supergroup messages marked `is_topic_message` use per-topic lanes even when `chat.is_forum` is absent, with regression tests and a changelog entry.

## Best Possible Solution

Land the narrow Telegram sequential-key fix after the draft is ready, with contributor credit in the changelog and the targeted Telegram formatter/Vitest checks green.

## Solution Assessment

Yes, with a small changelog-credit tweak: using `is_topic_message` as a supergroup-only fallback is the narrow maintainable fix because it avoids splitting ordinary supergroup reply threads on `message_thread_id` alone.
