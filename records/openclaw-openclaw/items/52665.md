## Summary

Keep open: current main still builds the sandbox common path through plain `apt-get update` and `apt-get install` with no apt retry handling, and the only directly related implementation PR was closed unmerged after validation and `--fix-missing` concerns.

## What This Changes

The issue requests apt retry and `--fix-missing` handling for sandbox-common Docker image builds to reduce transient network failures.

## Best Possible Solution

Add bounded, fail-closed apt retry handling to the base and common sandbox apt update/install layers, and avoid `--fix-missing` unless required package verification still makes missing tools fail the build.

## Reproduction Assessment

Yes for the source-level issue: current main still has no retry handling in the active sandbox-common and base sandbox apt layers. The actual bad-network failure is intermittent, so a live failing Docker build was not reproduced in this read-only review.

## Solution Assessment

No, not exactly as proposed: bounded `Acquire::Retries` and an update retry loop match existing repo precedent, but `--fix-missing` should be avoided or paired with hard post-install verification.
