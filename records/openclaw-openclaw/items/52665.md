## What This Changes

The issue requests bounded APT retries and `--fix-missing` for sandbox-common Docker image builds to reduce transient network build failures.

## Best Possible Solution

Land a narrow retry-only sandbox Dockerfile fix through https://github.com/openclaw/openclaw/pull/85833 or a replacement, keeping package installs fail-closed and verifying both sandbox images build.

## Reproduction Assessment

## Solution Assessment

No, not exactly as proposed: bounded APT retries are the maintainable fix, but `--fix-missing` should be removed or paired with explicit package verification because it can hold back selected packages.
