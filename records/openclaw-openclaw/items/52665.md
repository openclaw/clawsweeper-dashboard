## Summary

Keep open: current main still builds the sandbox-common and base sandbox apt layers without retry handling, the documented setup path is active, and the related implementation PR #52724 was closed without merge after validation concerns.

## What This Changes

The issue requests bounded apt retries and fix-missing handling for the sandbox-common Docker image build path.

## Best Possible Solution

Add bounded apt retry handling to the sandbox Docker image update/install steps and cover it with source-level tests, while keeping package installation fail-closed unless missing-package tolerance is explicitly verified.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: inspect the current sandbox-common and base sandbox Dockerfiles and observe that their apt update/install layers have no retry contract; the live network failure itself is intermittent.

## Solution Assessment

Unclear as proposed. Bounded `Acquire::Retries` on update and install is the narrow maintainable fix and matches repo precedent, but adding `--fix-missing` is not the best default unless the image also verifies required packages and fails loudly when they are unavailable.
