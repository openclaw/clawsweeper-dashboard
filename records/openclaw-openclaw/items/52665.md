## Summary

Keep open: current main still routes the active sandbox-common setup through Dockerfiles with plain `apt-get update` / `apt-get install`, and the related PR was closed unmerged after reviewers flagged `--fix-missing` and missing update retries. This remains a bounded sandbox build reliability improvement with a clear repair shape.

## What This Changes

The issue requests adding apt retry handling and `--fix-missing` to the sandbox-common Docker image build.

## Best Possible Solution

Add bounded, fail-closed apt retry handling to the base and common sandbox apt layers, and avoid `--fix-missing` unless hard post-install verification keeps missing required tools from passing the build.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source-level condition: current main's active sandbox-common path lacks apt retry handling in both base and common Dockerfiles. I did not reproduce a live bad-network build failure because the reported failure is intermittent.

## Solution Assessment

Is this the best way to solve the issue? Not exactly as proposed: bounded `Acquire::Retries` and an update retry loop fit existing repo precedent, but `--fix-missing` should be avoided or paired with hard package/binary verification.
