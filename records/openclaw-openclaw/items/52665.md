## Summary

Keep open: current main still routes the documented sandbox-common setup through Dockerfiles with plain apt update/install and no retry handling, while the only related implementation PR was closed unmerged after validation concerns.

## What This Changes

Request to add apt retry handling and `--fix-missing` behavior to the sandbox-common Docker image build so transient network failures are less likely to break setup.

## Best Possible Solution

Add bounded, fail-closed apt retry handling to the base and common sandbox apt update/install layers, and avoid `--fix-missing` unless required package verification still makes missing tools fail the build.

## Reproduction Assessment

Yes for the source-level issue: current main still has no retry handling in the active sandbox-common and base sandbox apt layers. The actual bad-network failure is intermittent, so a live failing build was not reproduced here.

## Solution Assessment

No, not exactly as proposed: bounded `Acquire::Retries` and an update retry loop match existing repo precedent, but `--fix-missing` should be avoided or paired with hard post-install verification.
