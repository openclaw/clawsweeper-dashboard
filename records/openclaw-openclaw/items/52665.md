## Summary

Keep open: current main still routes the documented sandbox-common setup through Dockerfiles with plain apt update/install, and the earlier PR for this same idea closed unmerged after review and validation concerns.

## What This Changes

The issue requests bounded retry handling and optional missing-package recovery for apt operations in the sandbox common Docker image build.

## Best Possible Solution

Add bounded, fail-closed apt retry handling to the base and common sandbox apt layers, and avoid --fix-missing unless required package verification still makes missing tools fail the build.

## Reproduction Assessment

Yes for the source-level condition: current main still lacks apt retry handling in the active sandbox-common and base sandbox Dockerfiles. No live bad-network build failure was reproduced because the reported failure is intermittent.

## Solution Assessment

No, not exactly as proposed: bounded Acquire::Retries and an update retry loop match existing repo precedent, but --fix-missing should be avoided or paired with hard post-install verification.
