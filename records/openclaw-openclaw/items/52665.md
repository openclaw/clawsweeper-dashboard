## Summary

Keep open: current main still lacks apt retry handling in the active sandbox-common and base sandbox Docker build paths, and the related implementation PR was closed unmerged after review and validation concerns.

## What This Changes

The issue requests bounded apt retry and missing-package handling for the sandbox-common Docker image build path.

## Best Possible Solution

Add bounded apt retry handling to the sandbox-common and base sandbox apt update/install steps, cover it with focused source-level tests, and keep package installation fail-closed unless missing-package tolerance is explicitly verified.

## Reproduction Assessment

Yes. Source inspection on current main shows the documented sandbox-common path still reaches Dockerfiles whose apt update/install layers have no retry handling; the actual network failure is intermittent.

## Solution Assessment

No as originally proposed. Bounded `Acquire::Retries` and an update retry loop match existing repo precedent, but `--fix-missing` should be avoided or paired with explicit verification so the image cannot build successfully while missing required tools.
