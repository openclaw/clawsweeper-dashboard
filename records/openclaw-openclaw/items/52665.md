## Summary

Keep open: current main still uses plain apt update/install in the active sandbox-common image path, and the only related implementation PR was closed unmerged after validation concerns.

## What This Changes

Request to make the sandbox-common Docker image build more resilient by adding apt retries, originally with `--fix-missing`, for dependency installation.

## Best Possible Solution

Add bounded, fail-closed apt retry handling to the sandbox-common and base sandbox apt update/install steps, with focused source tests and sandbox build validation.

## Reproduction Assessment

Yes, for the source-level issue. Current main still routes the documented sandbox-common setup through Dockerfiles whose apt update/install layers have no retry handling, though the actual bad-network failure is intermittent.

## Solution Assessment

No, not as originally proposed. Bounded `Acquire::Retries` and an update retry loop match existing repo precedent, but `--fix-missing` should be avoided or paired with explicit post-install verification so the image cannot build while missing required tools.
