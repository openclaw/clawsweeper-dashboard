## Summary

Keep open; the latest head still bypasses OpenClaw's provider runtime/auth path and lacks real behavior proof for a session actually generating, persisting, and displaying an AI title.

## What This Changes

Adds default-on agent session-title config, a `SessionEntry.aiTitle` field, Gateway run-end AI title generation, and Control UI/session-row display wiring.

## Best Possible Solution

Implement title generation through the existing simple-completion/provider runtime using the resolved session agent/model/auth context, then add focused tests, docs/changelog, and live Control UI proof.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR paired with #77165. The blocking PR defects are source-reproducible from latest head: raw fetch at `src/gateway/session-auto-title.ts:239` and global-only provider resolution at `src/gateway/session-auto-title.ts:174`.

## Solution Assessment

No. The feature direction may be valid, but the current implementation is not the narrowest maintainable path because OpenClaw already has provider/simple-completion helpers that preserve auth, provider compatibility, and session model context.
