## Summary

Keep open: the feature direction is useful and paired with #77165, but this PR should not merge as-is because title generation bypasses OpenClaw's provider/auth runtime, lacks live proof of generated titles, and is missing the required changelog entry.

## What This Changes

Adds configurable AI session auto-title generation, persists `SessionEntry.aiTitle`, triggers Gateway run-end title generation, and wires config schema plus Control UI session-row display support.

## Best Possible Solution

Implement title generation through the existing simple-completion/provider runtime using the resolved session agent, model, and auth context, then add focused coverage, changelog/docs as needed, and live proof that the title is generated, stored, and displayed.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR linked to #77165. The blocking PR defects are source-reproducible from the diff: raw provider fetch at `src/gateway/session-auto-title.ts:239` and global/env-only provider resolution at `src/gateway/session-auto-title.ts:174`.

## Solution Assessment

No. The requested feature is plausible, but the current implementation is not the narrowest maintainable solution because OpenClaw already has simple-completion/provider runtime helpers that preserve auth, provider compatibility, and session model context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route title generation through provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` posts transcript excerpts directly to an OpenAI-compatible URL with ad hoc auth. That bypasses OpenClaw's provider runtime, auth-profile, SecretRef, runtime-auth, and compatibility layers, so title generation should use the existing simple-completion/provider runtime path.
  - confidence: 0.94
- **[P2] Preserve the session model and auth selection:** `src/gateway/session-auto-title.ts:174-184`
  - body: `resolveTitleProviderConfig` resolves only the global default provider/model and env credential fallback. Sessions using an agent-specific model, runtime-selected model, model alias, or auth profile can fail title generation or send transcript excerpts to the wrong provider.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/gateway/session-auto-title.ts:1`
  - body: This user-facing feature changes session naming, Gateway config, and Control UI display, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active Unreleased section before merge.
  - confidence: 0.88
