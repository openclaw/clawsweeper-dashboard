## Summary

Keep open: this feature PR is the implementation candidate for #77165, but the current patch should not merge because generated-title calls bypass OpenClaw provider runtime guardrails and the PR still lacks live proof of an actual generated, persisted, and displayed title.

## What This Changes

The PR adds default-on session auto-title config, an `aiTitle` session field, Gateway post-run title generation, session-row/UI display support, and generated config schema/help metadata.

## Best Possible Solution

Route title generation through the existing provider runtime/request-policy/auth-profile machinery, align schema/docs/changelog/tests, and require live proof of a multi-turn session gaining an AI title.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking defects are source-reproducible in the PR head, and the missing runtime proof is confirmed by the PR body.

## Solution Assessment

No. The feature direction may be useful, but the current implementation is not the best path because it bypasses provider runtime policy/auth handling and still lacks changelog, focused regression coverage, and live generated-title proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route title calls through the provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` sends conversation excerpts to a raw `/chat/completions` URL with ad hoc auth. That bypasses OpenClaw's provider auth/profile, SecretRef, request policy, proxy/TLS/private-network handling, and non-chat provider adapters, so title generation should use the selected model provider runtime instead.
  - confidence: 0.94
- **[P2] Honor the session's resolved model context:** `src/gateway/session-auto-title.ts:174-187`
  - body: `resolveTitleProviderConfig` resolves only the global default model/provider and env/baseUrl fallback, ignoring the session's provider/model/auth-profile overrides. Sessions run with a different provider or profile can fail title generation or send transcript excerpts to the wrong endpoint.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/gateway/session-auto-title.ts:1`
  - body: This is a user-facing `feat` that changes session naming, config, Gateway rows, and Control UI display, but the PR head does not add a `CHANGELOG.md` entry. Add a single-line entry under the active release section before merge.
  - confidence: 0.88
