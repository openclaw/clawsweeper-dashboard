## Summary

Keep open: the feature is useful and linked to an open request, but this PR is not merge-ready because the implementation bypasses OpenClaw's provider/auth runtime and the supplied proof does not show a title being generated, persisted, and displayed.

## What This Changes

The PR adds `agents.defaults.sessionTitle`, `SessionEntry.aiTitle`, a Gateway post-run title generator, and UI/session-row support for AI-generated session titles.

## Best Possible Solution

Implement session titles through the existing simple-completion/provider runtime using the resolved session agent, model, and auth context, then add focused coverage and redacted live proof that the title is generated, stored, and shown.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR for https://github.com/openclaw/openclaw/issues/77165. The blocking PR defects are source-reproducible from `src/gateway/session-auto-title.ts:174` and `src/gateway/session-auto-title.ts:239`.

## Solution Assessment

No. The feature direction is plausible, but the implementation should use OpenClaw's prepared simple-completion/provider runtime with session-scoped model and auth selection instead of a raw OpenAI-compatible fetch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route title generation through provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` posts transcript excerpts directly to an OpenAI-compatible endpoint with a hand-built bearer header. That bypasses OpenClaw's provider runtime, auth profiles, SecretRefs, runtime auth preparation, local provider handling, and compatibility layers, so title generation should use the existing prepared simple-completion path.
  - confidence: 0.94
- **[P2] Preserve session-scoped model and auth selection:** `src/gateway/session-auto-title.ts:174-184`
  - body: `resolveTitleProviderConfig` resolves only the global default provider/model and environment key. Sessions using an agent-specific model, session override, runtime-selected model, alias, or auth profile can generate titles with the wrong backend or fail even though the session itself works.
  - confidence: 0.9
