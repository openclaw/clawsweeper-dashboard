## Summary

Keep open: current main does not implement AI-generated session titles, and this PR has blocking config, provider-resolution, and security-sensitive transport issues before it can merge.

## What This Changes

The PR adds persisted `aiTitle` session metadata, derives session rows from it, triggers async AI title generation after run completion, and adds config/help metadata for generated session titles.

## Best Possible Solution

Implement the feature behind a documented and validated gateway/session config, persist generated titles with clear precedence over manual names, and route title calls through the selected provider/model runtime transport with focused regression coverage.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a feature PR. The patch defects are source-reproducible from the PR head: the new config read is not backed by the strict schema, and the title request bypasses the existing provider transport path.

## Solution Assessment

No. The feature direction may be useful, but this implementation should use the real config/schema/type surfaces and selected provider transport instead of a separate raw OpenAI-compatible call path.
