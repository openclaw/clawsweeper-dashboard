## Summary

Keep open: this is a useful implementation candidate for the linked auto-title feature, but the current PR head is not merge-ready because it still routes transcript-bearing title generation through a raw provider request, has config-schema/changelog gaps, and lacks required real behavior proof.

## What This Changes

The PR adds AI-generated session titles with new `agents.defaults.sessionTitle` config, persists `aiTitle` on session entries after run completion, and surfaces it in Gateway and Control UI session rows.

## Best Possible Solution

Implement title generation through the selected-model provider runtime/request-policy path, keep the generated config contract and changelog aligned, and merge only after focused tests plus real title-generation proof.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking defects are source-reproducible from the PR diff and current provider/config contracts.

## Solution Assessment

No. The feature direction may be useful, but the current implementation should not use an ad hoc OpenAI-compatible fetch for transcript-bearing model calls and should align generated schemas, changelog, tests, and live proof before merge.
