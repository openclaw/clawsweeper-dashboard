## Summary

Keep open: this feature PR is still not merge-ready because the latest diff bypasses the provider runtime for transcript title calls, ignores session model/auth overrides, duplicates generated schema entries, lacks the required changelog entry, and only proves config availability rather than a generated title in a real session.

## What This Changes

The PR adds default-on `agents.defaults.sessionTitle` config, `SessionEntry.aiTitle`, Gateway post-run title generation, session row/UI display wiring, and generated config metadata.

## Best Possible Solution

Implement title generation through the existing simple-completion provider runtime using the resolved session/agent model and auth context, clean up generated config output, add changelog/docs and focused tests, then attach live proof of the generated title in Control UI.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR paired with #77165. The blocking PR defects are source-reproducible from the diff, and the runtime proof gap is stated in the PR body.

## Solution Assessment

No. The feature direction may be valid, but the current implementation is not the best path because OpenClaw already has provider/simple-completion runtime helpers that preserve auth, provider compatibility, and session model context.
