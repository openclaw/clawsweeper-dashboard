## Summary

Keep open: the feature is not implemented on current main, and the PR still needs provider-runtime, session-context, changelog, and real-behavior-proof fixes before it is mergeable.

## What This Changes

The PR adds configurable AI-generated session titles, persists `SessionEntry.aiTitle`, triggers generation after Gateway run completion, and surfaces `aiTitle` in session rows and the Control UI.

## Best Possible Solution

Implement title generation through the existing simple-completion/provider runtime using the resolved session agent, model, and auth context, then add focused coverage, changelog/docs as needed, and live proof that the title is generated, stored, and displayed.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR linked to #77165. The blocking PR defects are source-reproducible from the diff at `src/gateway/session-auto-title.ts:169` and `src/gateway/session-auto-title.ts:239`.

## Solution Assessment

No. The feature direction is plausible, but the implementation should use OpenClaw's provider-safe simple-completion path and session-scoped model/auth context instead of a raw OpenAI-compatible fetch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route title generation through provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` posts transcript excerpts directly to an OpenAI-compatible URL with an ad hoc Bearer header. That bypasses OpenClaw's provider runtime, auth-profile, SecretRef, runtime-auth, and compatibility layers, so title generation should use the existing simple-completion/provider runtime path.
  - confidence: 0.94
- **[P2] Preserve the session model and auth selection:** `src/gateway/session-auto-title.ts:174-184`
  - body: `resolveTitleProviderConfig` resolves only the global default provider/model plus env credentials. Sessions using an agent-specific model, runtime-selected model, model alias, or auth profile can fail title generation or send transcript excerpts to the wrong provider.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/gateway/session-auto-title.ts:1`
  - body: This user-facing feature changes session naming, Gateway config, and Control UI display, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active Unreleased section before merge.
  - confidence: 0.88
