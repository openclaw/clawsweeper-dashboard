## Summary

Keep open. The feature is not implemented on current main, and this PR still has merge-blocking design/security issues: it performs raw provider fetches, ignores session-scoped model/auth selection, and lacks real proof that a title is generated, persisted, and displayed.

## What This Changes

The PR adds configurable AI-generated Gateway session titles, persists `SessionEntry.aiTitle`, derives UI session labels from it, and triggers title generation after run completion.

## Best Possible Solution

Implement title generation through the existing simple-completion/provider runtime using the resolved session agent/model/auth context, emit a fresh session update after persistence, align config/docs/tests, and settle the default/privacy policy before merge.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR for https://github.com/openclaw/openclaw/issues/77165. The blocking PR defects are source-reproducible from the PR head diff around raw provider fetches, session model/auth resolution, and post-save session updates.

## Solution Assessment

No. The feature direction is plausible, but this implementation should reuse OpenClaw's provider-safe simple-completion path, preserve session-scoped model/auth selection, and publish updated session row state after saving the generated title.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route title generation through provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` posts transcript excerpts directly to an OpenAI-compatible URL with an ad hoc Bearer header. That bypasses OpenClaw's provider runtime, auth-profile, SecretRef, runtime-auth, and compatibility layers, so title generation should use the existing simple-completion/provider runtime path.
  - confidence: 0.94
- **[P2] Use the session's selected model and auth profile:** `src/gateway/session-auto-title.ts:174-184`
  - body: `resolveTitleProviderConfig` resolves only the global default provider/model and environment credentials. Sessions using an agent-specific model, runtime-selected model, model alias, or auth profile can fail title generation or send transcript excerpts to the wrong provider.
  - confidence: 0.9
- **[P2] Publish a session update after saving titles:** `src/gateway/session-auto-title.ts:316-320`
  - body: The new path writes `aiTitle` asynchronously after run completion, but it does not emit a later `sessions.changed` payload with the updated row. The existing run-end event is broadcast without waiting for title generation, so the Control UI can keep showing the fallback title until a manual reload or unrelated session event.
  - confidence: 0.83
