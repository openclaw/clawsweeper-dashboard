## Summary

Keep open. Current main does not implement session AI titles, and this PR still has merge-blocking provider/auth routing, security, changelog, and real-behavior-proof gaps.

## What This Changes

The PR adds an `aiTitle` session field, default-on `agents.defaults.sessionTitle` config, Gateway post-run title generation, and Control UI session display wiring for #77165.

## Best Possible Solution

Implement title generation through the existing simple-completion/provider runtime path, keep session model/auth context intact, add focused regression coverage plus changelog/docs, and require live proof of a generated title in the UI.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking PR defects are source-reproducible from the diff, and the proof gap is stated in the PR body.

## Solution Assessment

No. The feature direction may be valid, but the current implementation is not the narrow maintainable path because OpenClaw already has a simple-completion/provider runtime helper for this exact class of lightweight model call.
