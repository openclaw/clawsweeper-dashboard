## Summary

Keep open: current main does not implement AI-generated session titles, and this PR still has blocking provider-config and provider-transport/security issues. The related feature request #77165 remains open and should stay open until a correct implementation merges.

## What This Changes

The PR adds configurable AI-generated session titles, persists `aiTitle` on sessions after successful runs, and surfaces the title in Gateway and Control UI session rows.

## Best Possible Solution

Implement the feature through the existing selected-model/provider runtime and policy-aware request path, with focused tests for title precedence, disable/threshold behavior, lifecycle persistence, and a changelog entry.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking defects are source-reproducible from the PR head and current config/provider contracts.

## Solution Assessment

No. The feature direction may be useful, but this implementation should use the existing provider runtime/auth/request-policy path instead of a raw OpenAI-compatible fetch, and it should resolve the real `models.providers` config path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Read providers from models.providers:** `src/gateway/session-auto-title.ts:168`
  - body: `OpenClawConfig` does not have a top-level `providers` object; provider settings live under `models.providers`. This line should fail strict type checking and also misses configured provider base URLs at runtime, so auto-title generation cannot reliably resolve the selected provider.
  - confidence: 0.96
- **[P1] Use the provider transport for title calls:** `src/gateway/session-auto-title.ts:218-228`
  - body: This raw fetch posts transcript excerpts and a bearer token to a configured URL outside the existing model-provider runtime. It bypasses auth profiles, SecretRef/request handling, proxy/TLS overrides, attribution, and private-network policy, so title generation should route through the guarded provider path.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `src/gateway/session-auto-title.ts:1`
  - body: This is a user-facing feature that changes session naming, config, Gateway rows, and Control UI display, but the PR does not update `CHANGELOG.md`. Add a single-line user-facing changelog entry before merge.
  - confidence: 0.86
