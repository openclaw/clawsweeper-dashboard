## Summary

Keep open for maintainer review. Current main does not implement AI-generated session titles, and this PR is not mergeable yet because the title generator reads the wrong provider config path, bypasses existing provider request policy with a raw transcript-bearing fetch, and overlaps the still-open implementation PR #77216.

## What This Changes

The PR adds optional AI-generated session titles with `aiTitle` persistence, `agents.defaults.sessionTitle` config, async gateway lifecycle generation, and Control UI/session-row display plumbing.

## Best Possible Solution

Consolidate around one reviewed implementation that routes generation through the existing selected-model/provider runtime, preserves manual title precedence, adds focused tests and docs/config coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking PR defects are source-reproducible from the PR head and current `OpenClawConfig` plus provider request contracts.

## Solution Assessment

No. The feature direction is plausible, but this implementation should use the existing provider runtime/auth/request-policy path and the real `models.providers` config surface instead of a raw OpenAI-compatible fetch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Read providers from `models.providers`:** `src/gateway/session-auto-title.ts:168`
  - body: `OpenClawConfig` does not expose a top-level `providers` object; provider entries live under `models.providers`. This should fail strict typing and also misses configured provider base URLs at runtime, so title generation will not reliably resolve the selected provider.
  - confidence: 0.95
- **[P1] Route title calls through the provider transport:** `src/gateway/session-auto-title.ts:218`
  - body: This raw `fetch` posts transcript excerpts and a bearer token outside OpenClaw's model-provider runtime. It bypasses configured auth profiles, request headers, proxy/TLS overrides, attribution, and private-network policy, so the title call should reuse the guarded provider path.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `src/gateway/session-auto-title.ts:1`
  - body: This user-facing feature changes session naming, config, Gateway rows, and Control UI display, but the PR does not update `CHANGELOG.md`. Add a single-line user-facing changelog entry before merge.
  - confidence: 0.86
