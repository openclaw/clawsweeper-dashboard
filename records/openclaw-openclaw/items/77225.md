## Summary

Keep open for maintainer review: the feature is plausible, but this PR is not mergeable because it bypasses the provider runtime/auth path, does not reliably publish generated titles to the live UI, and its supplied proof does not demonstrate title generation end to end.

## What This Changes

The PR adds configurable AI-generated session titles via `agents.defaults.sessionTitle`, persists them as `SessionEntry.aiTitle`, triggers generation after gateway run completion, and wires the value into session rows and UI display types.

## Best Possible Solution

Implement the feature through the existing simple-completion/provider runtime using the resolved session agent, model, and auth context; persist generated title metadata with clear precedence, publish a post-persist session update, document privacy/config semantics, and require redacted live proof.

## Reproduction Assessment

Not applicable as a current-main bug because this is a feature PR for https://github.com/openclaw/openclaw/issues/77165. The blocking PR defects are source-reproducible from `src/gateway/session-auto-title.ts:174`, `src/gateway/session-auto-title.ts:239`, and `src/gateway/server-chat.ts:382`.

## Solution Assessment

No. The direction is plausible, but this implementation should use OpenClaw's prepared provider runtime with session-scoped model/auth and publish a post-persist session update instead of using a raw fetch and fire-and-forget UI path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route title generation through provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` posts transcript excerpts directly to an OpenAI-compatible endpoint with an ad hoc bearer header. That bypasses OpenClaw provider runtime handling for auth profiles, SecretRefs, runtime auth exchange, local no-auth behavior, and provider compatibility, so title generation should use the existing prepared simple-completion path.
  - confidence: 0.94
- **[P2] Preserve session-scoped model and auth selection:** `src/gateway/session-auto-title.ts:174-184`
  - body: `resolveTitleProviderConfig` only uses the global default model/provider and environment key. Sessions using agent defaults, session model overrides, runtime-selected models, aliases, or auth profiles can generate titles with the wrong backend or fail even though the session itself works.
  - confidence: 0.9
- **[P2] Notify subscribers after persisting generated titles:** `src/gateway/server-chat.ts:382`
  - body: This starts title generation in the background, then immediately sends the normal `sessions.changed` event before `aiTitle` can be persisted. Since there is no later post-persist event and the UI merge path does not copy `aiTitle`, a generated title will not appear in the live sidebar until some unrelated reload happens.
  - confidence: 0.86
