## Summary

Keep open: this is a useful implementation candidate for #77165, but the patch is not merge-ready because source review found blocking type/config issues, a provider/runtime bypass with privacy impact, and missing UI/test/changelog follow-through.

## What This Changes

The PR adds AI-generated session titles via a new `aiTitle` session field, a gateway auto-title hook after completed runs, config schema/help entries, and session-row projection changes.

## Best Possible Solution

Land a rebased, narrow implementation that routes title generation through OpenClaw’s existing model/provider runtime, updates the typed config/docs/UI/changelog surfaces, and adds focused regression coverage for title priority, disabling, persistence, and session-list display.

## Reproduction Assessment

Yes for the PR review findings: the compile-time config/store mismatches and UI projection gap are visible from the PR source against current main contracts. Runtime title generation was not live-tested in this read-only pass.

## Solution Assessment

No. The maintainable direction is to use OpenClaw’s existing model/runtime/provider and credential abstractions, make the privacy/default behavior explicit, and wire the generated title through the actual UI session-list path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Route title generation through OpenClaw model runtime:** `src/gateway/session-auto-title.ts:164-172`
  - body: This raw `/chat/completions` request bypasses the existing provider/runtime/auth/fallback path and only works for OpenAI-compatible env or ad hoc `baseUrl` fields. Normal `agents.defaults.models` entries do not define those fields, so title generation will silently do nothing or use the wrong external endpoint; use the configured OpenClaw model runtime instead.
  - confidence: 0.9
- **[P2] Add the new sessionTitle config type:** `src/gateway/server-chat.ts:186`
  - body: The PR reads `cfg.agents.defaults.sessionTitle`, but the typed `AgentDefaultsConfig` surface is not updated. Even with zod/schema help entries, the gateway code will fail typechecking until the public config type includes this option.
  - confidence: 0.87
- **[P2] Require a store path before updating the session:** `src/gateway/session-auto-title.ts:270`
  - body: `triggerAutoTitleGeneration` accepts `storePath` as possibly undefined and passes it to `updateSessionStoreEntry`, whose current contract requires a concrete string. Make the helper require a string or guard before this write so the patch typechecks and cannot call the store writer without a path.
  - confidence: 0.88
- **[P2] Surface generated titles in the Control UI:** `src/gateway/session-utils.ts:1688`
  - body: Adding `aiTitle` to the gateway row does not make the session list show it: the current UI row type/render path does not read `aiTitle` or `derivedTitle`, and session loading does not request derived titles. Wire the generated title into the actual session-list display path with tests.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff introduces automatic external transmission of session transcript snippets and should not merge without an explicit provider/runtime and privacy design.
