## Summary

Keep open: current main does not implement AI-generated session titles, and this PR is not merge-ready because it still sends transcript excerpts through an ad hoc fetch path, leaves the generated config schema out of sync, lacks the required changelog entry, and proves config visibility rather than live title generation.

## What This Changes

The PR adds a default-on `agents.defaults.sessionTitle` feature that stores `aiTitle` on session entries, triggers title generation after Gateway run completion, and exposes the value in session rows and the Control UI.

## Best Possible Solution

Route title generation through the existing provider runtime/request-policy/auth-profile machinery, align generated config schema and changelog/docs, add focused regression coverage, and require live proof of a multi-turn session gaining an AI title.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking defects are source-reproducible from the PR's raw fetch path, generated schema drift, missing changelog, and missing live generated-title proof.

## Solution Assessment

No. The feature direction may be useful, but the current implementation should not bypass provider request policy/auth/profile handling and should align schema, changelog, tests, and real behavior proof before merge.
