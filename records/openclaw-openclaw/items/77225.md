## Summary

Keep this PR open: current main does not implement AI-generated session titles, and the proposed feature branch is not merge-ready because it adds a new config/product surface while bypassing existing provider transport and config contracts.

## What This Changes

The PR adds AI-generated session titles with an `aiTitle` session field, config/help metadata, lifecycle-triggered summarization, and session row exposure.

## Best Possible Solution

Implement auto-title generation behind a validated, documented session/gateway config and route title calls through the existing selected-model/provider runtime transport after maintainers decide privacy and default behavior.

## Reproduction Assessment

Not applicable as a current-main bug reproduction; this is a feature PR. The merge blockers are source-reproducible from the PR head: the new config read lacks a strict schema field and the title call bypasses provider transport.

## Solution Assessment

No. The direction may be useful, but the implementation should use real config/schema/type surfaces and existing provider runtime/auth/guarded fetch machinery instead of a separate raw OpenAI-compatible call path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add the real sessionTitle config schema:** `src/gateway/server-chat.ts:186`
  - body: This reads `cfg.agents.defaults.sessionTitle`, but `AgentDefaultsSchema` is strict and the PR head does not define `sessionTitle` there. The advertised config cannot be validated or typed as a real runtime field.
  - confidence: 0.93
- **[P2] Resolve the configured provider path:** `src/gateway/session-auto-title.ts:203-219`
  - body: `agents.defaults.models` entries only contain alias/params/streaming metadata, while endpoints and credentials live under `models.providers.*`. Normal configs will return null or fall back to unrelated OpenAI env settings instead of the selected provider.
  - confidence: 0.88
- **[P1] Use the guarded provider transport for title calls:** `src/gateway/session-auto-title.ts:164-173`
  - body: This raw fetch posts conversation-derived text and a bearer token directly to `baseUrl`, bypassing OpenClaw's provider transport, SecretRef/auth-profile handling, SSRF policy, proxy/TLS settings, and private-network opt-in.
  - confidence: 0.9
