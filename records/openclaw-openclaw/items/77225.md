## Summary

Keep open: current main still derives titles from displayName, subject, first user message, then session id, so the requested feature is not implemented; however, this PR should not merge as-is because the current head has a config/type error and a security-sensitive provider transport bypass.

## What This Changes

The PR adds an optional AI-generated session title field/config, derives titles from it, triggers async title generation after successful runs, and exposes the field to Gateway/UI session rows.

## Best Possible Solution

Implement the feature through the existing selected-model/provider runtime and documented config surface, with tests for title precedence, disable/threshold behavior, non-blocking lifecycle updates, and a user-facing changelog entry.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking PR defects are source-reproducible from the diff and current types/provider contracts.

## Solution Assessment

No. The feature direction may be useful, but this implementation should use the existing provider runtime/auth/request-policy path instead of a raw OpenAI-compatible fetch and should resolve the real models.providers config path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Read model providers from the real config path:** `src/gateway/session-auto-title.ts:168`
  - body: `OpenClawConfig` does not have a top-level `providers` object; provider settings live under `models.providers`. This line should fail strict type checking and also misses configured provider base URLs at runtime, so auto-title generation cannot reliably resolve the selected provider.
  - confidence: 0.95
- **[P1] Use the provider transport for title calls:** `src/gateway/session-auto-title.ts:218`
  - body: This raw fetch posts transcript excerpts and a bearer token to a configured URL outside the existing model provider runtime. It bypasses auth profiles/SecretRef handling, request proxy/TLS overrides, attribution, and private-network policy, so route title generation through the guarded provider path instead.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `src/gateway/session-auto-title.ts:1`
  - body: This is a user-facing `feat` that changes session naming, config, Gateway rows, and Control UI display, but the PR does not update `CHANGELOG.md`. Add a single-line user-facing changelog entry before merge.
  - confidence: 0.86
