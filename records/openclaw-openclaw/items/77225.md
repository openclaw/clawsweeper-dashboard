Keep open for maintainer review, but the PR is not mergeable as-is: the new config key is not in the strict runtime schema/types, title calls bypass the existing model-provider transport/auth path, and the branch is currently merge-conflicting.

## What This Changes

The PR adds an `aiTitle` session field, a generated-title module, a lifecycle trigger, session row exposure, and config/help metadata for AI-generated session titles.

## Best Possible Solution

Implement auto-title generation behind a documented, validated session/gateway config using the existing selected-model/provider auth and guarded transport path, with regression coverage for title precedence, disable/threshold behavior, and non-blocking lifecycle updates.

## Reproduction Assessment

Not applicable as a current-main bug reproduction; this is a feature PR. The patch defects are source-reproducible by comparing the PR's new config and fetch code with the strict agent-default schema/types and provider transport contract.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first use the real config/schema/type surfaces and existing provider runtime/auth/guarded fetch machinery instead of a separate raw OpenAI-compatible call path.

Full review comments:

- **[P1] Add the real sessionTitle config schema:** `src/gateway/server-chat.ts:186`
  - body: This reads `cfg.agents.defaults.sessionTitle`, but the PR only adds generated/help metadata; `AgentDefaultsSchema` and `AgentDefaultsConfig` still have no `sessionTitle`, and the schema is strict. That makes the new config unusable and should fail type/schema validation.
  - confidence: 0.93
- **[P2] Resolve the configured provider path:** `src/gateway/session-auto-title.ts:203-219`
  - body: `agents.defaults.models` entries only contain alias/params/streaming, while endpoints and credentials live under `models.providers.*`; for normal configs this returns null and never generates a title, or falls back to unrelated OpenAI env settings. Resolve the selected provider/model through the existing model config/auth code.
  - confidence: 0.88
- **[P1] Use the guarded provider transport for title calls:** `src/gateway/session-auto-title.ts:164-173`
  - body: This raw fetch posts transcript text and a bearer token directly to `baseUrl`, bypassing the existing provider transport, SecretRef/auth-profile handling, SSRF policy, proxy/TLS settings, and private-network opt-in. Route title generation through the same provider request path as model calls.
  - confidence: 0.9
