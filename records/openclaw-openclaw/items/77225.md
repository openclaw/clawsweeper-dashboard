## Summary

Keep open: current main does not implement automatic AI session titles, and this PR is not merge-ready because it bypasses OpenClaw's provider/auth runtime and still lacks real proof of a session title being generated, persisted, and displayed.

## What This Changes

The PR adds default-on session auto-title config, a new `SessionEntry.aiTitle` field, Gateway run-end title generation, and Control UI/session-row display wiring.

## Best Possible Solution

Implement title generation through the existing simple-completion/provider runtime using the resolved session agent/model/auth context, then add focused tests, docs/changelog, and live Control UI proof.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR paired with #77165. The blocking defects are source-reproducible from the PR diff, and the proof gap is stated in the PR body.

## Solution Assessment

No. The feature direction may be valid, but the current implementation is not the best path because OpenClaw already has provider/simple-completion helpers that preserve auth, provider compatibility, and session model context.
