## Summary

Keep open: this is an active feature PR for AI-generated session titles, but the current diff has source-reproducible provider/runtime and live-update defects, and the supplied proof does not show title generation, persistence, or display.

## What This Changes

The PR adds configurable AI-generated session titles, stores them as `SessionEntry.aiTitle`, triggers generation after gateway run completion, and threads `aiTitle` through gateway and UI session rows.

## Best Possible Solution

Land one product-approved implementation that uses the prepared provider/runtime path with session-scoped model/auth, persists generated title metadata without overriding manual names, publishes a post-persist session update, documents config/privacy semantics, and includes redacted live proof.

## Reproduction Assessment

Not applicable as a current-main bug because this is a feature PR for https://github.com/openclaw/openclaw/issues/77165. The blocking PR defects are source-reproducible from `src/gateway/session-auto-title.ts:174`, `src/gateway/session-auto-title.ts:239`, `src/gateway/server-chat.ts:382`, and `ui/src/ui/controllers/sessions.ts:61`.

## Solution Assessment

No. The direction is plausible, but this implementation should use OpenClaw's prepared provider runtime with session-scoped model/auth and publish a post-persist session update instead of using a raw fetch and fire-and-forget UI path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route title generation through provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` posts transcript excerpts directly to an OpenAI-compatible endpoint with an ad hoc bearer header. That bypasses OpenClaw provider runtime handling for auth profiles, SecretRefs, runtime auth exchange, local no-auth behavior, and provider compatibility, so title generation should use the existing prepared simple-completion path.
  - confidence: 0.94
- **[P2] Preserve session-scoped model and auth selection:** `src/gateway/session-auto-title.ts:174-184`
  - body: `resolveTitleProviderConfig` only uses the global default model/provider and environment key. Sessions using agent defaults, session model overrides, runtime-selected models, aliases, or auth profiles can generate titles with the wrong backend or fail even though the session itself works.
  - confidence: 0.9
- **[P2] Notify subscribers after persisting generated titles:** `src/gateway/server-chat.ts:382`
  - body: This starts title generation in the background, then immediately sends the normal `sessions.changed` event before `aiTitle` can be persisted. There is no later post-persist event, and the UI event merge whitelist still lacks `aiTitle`, so the live sidebar will not update until a reload or unrelated event.
  - confidence: 0.88
