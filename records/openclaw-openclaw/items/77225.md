## Summary

Keep open: current main does not implement AI-generated session titles, and this feature PR is still useful but not mergeable because the diff bypasses provider runtime/auth handling, can ignore the session’s model/auth context, lacks the required changelog entry, and still does not show real after-fix behavior proof.

## What This Changes

The PR adds configurable AI session auto-title generation, persists `SessionEntry.aiTitle`, triggers Gateway run-end title generation, and wires config schema plus Control UI session-row display support.

## Best Possible Solution

Implement title generation through the existing simple-completion/provider runtime using the resolved session agent, model, and auth context, then add focused coverage, a changelog entry, docs if needed, and live proof that the title is generated, stored, and displayed.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR linked to open #77165. The blocking PR defects are source-reproducible from the diff at `src/gateway/session-auto-title.ts:174` and `src/gateway/session-auto-title.ts:239`.

## Solution Assessment

No. The requested feature is plausible, but the current implementation is not the narrowest maintainable solution because OpenClaw already has simple-completion/provider runtime helpers that preserve auth, provider compatibility, and session model context.
