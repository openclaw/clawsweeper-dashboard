## Summary

Current main and the v2026.6.6 release already implement the central request by exposing a credential-aware optional `available` field on `models.list` rows, so this can close as implemented.

## What This Changes

The issue asks OpenClaw to expose the per-model `available` signal to plugin and Gateway RPC consumers so external model pickers do not reimplement CLI-only availability heuristics.

## Best Possible Solution

Keep the shipped additive `models.list` availability field as the canonical public surface; open a narrower follow-up only if maintainers want a direct in-process SDK helper beyond the RPC path.

## Reproduction Assessment

Not applicable as a current-main bug reproduction: this was an API feature request, and the old source-level gap is no longer present in current `models.list` source or v2026.6.6.

## Solution Assessment

Yes. The shipped additive RPC field matches the issue's preferred Option 1 and avoids promoting broad CLI/registry internals through the plugin SDK.
