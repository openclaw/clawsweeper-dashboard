## Summary

Current main and the shipped v2026.6.6 release already expose credential-aware `available` on `models.list`, including coverage for the Codex/ChatGPT availability case called out in the report.

## What This Changes

Request to expose OpenClaw's per-model availability flag to plugins or gateway RPC clients so third-party model pickers can avoid duplicating auth heuristics.

## Best Possible Solution

Keep the shipped additive `models.list` availability field as the canonical public surface; open a narrower follow-up only if a direct in-process SDK helper is still needed beyond the RPC path.

## Reproduction Assessment

Not applicable as a current bug reproduction: this is an API feature request, and the old source-level gap is now closed in current main and the v2026.6.6 release.

## Solution Assessment

Yes. The shipped RPC field implements the issue's preferred Option 1 and avoids exposing broad CLI or registry internals through the plugin SDK.
