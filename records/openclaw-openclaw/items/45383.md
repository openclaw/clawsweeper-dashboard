## Best Possible Solution

Land a rebased, root-aware inbound-media guard that uses a constant trusted label or moves path metadata inside guarded data, with redacted real attachment-read proof.

## Reproduction Assessment

Yes, from source inspection. Current main stages sandboxed attachments under `media/inbound/*` and then returns read-tool text after image sanitization without an untrusted-data wrapper; I did not run a live messaging flow.

## Solution Assessment

No. The wrapper direction is right, but the current branch should keep dynamic sender path text out of trusted prompt labels and classify inbound reads against the actual staged root instead of any absolute path segment.
