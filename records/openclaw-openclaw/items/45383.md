## Summary

Keep this PR open: it targets a real security bug, but the current patch still has blocking review findings around trusted prompt metadata and inbound-root classification, and the external contributor has not supplied after-fix real behavior proof.

## What This Changes

This PR wraps read-tool text returned from staged `media/inbound` files as untrusted prompt data, threads `containerWorkdir` into sandboxed read tools, and adds inbound-guard tests.

## Best Possible Solution

Land a root-aware inbound-media wrapper that keeps attacker-controlled metadata out of trusted prompt text, preserves normal read output, and includes redacted real attachment-read proof.

## Reproduction Assessment

Yes. Source inspection shows current main stages sandboxed attachments under `media/inbound/*` and then returns read-tool text without an untrusted-data wrapper; I did not run a live messaging flow in this read-only review.

## Solution Assessment

No. The wrapper direction is right, but the branch should keep sender-controlled path text out of trusted wrapper labels and classify inbound reads against the actual staged root instead of any absolute `media/inbound` segment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep inbound filenames out of trusted wrapper labels:** `src/agents/pi-tools.read.ts:798`
  - body: The staged filename is sender-controlled, and after control-character sanitization it still goes into `params.label`, which the wrapper emits before `<untrusted-text>`. Use a constant trusted label or move path metadata inside the guarded block.
  - confidence: 0.82
- **[P2] Anchor inbound matching to the staged root:** `src/agents/pi-tools.read.ts:140`
  - body: The regex classifies any absolute path containing `/media/inbound/` as inbound, and the added test expects `/workspace/src/media/inbound/parser.ts` to be true. That wraps and HTML-escapes ordinary source reads for projects with that path shape; compare against the resolved staged root instead.
  - confidence: 0.86
