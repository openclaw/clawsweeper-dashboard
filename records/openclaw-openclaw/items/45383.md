## Summary

Keep open. Current main still lacks read-time untrusted wrapping for staged inbound media, so the linked security problem remains real; however, this PR should not merge as-is because its absolute-path classifier can wrap unrelated trusted workspace files that merely contain a `media/inbound` segment.

## What This Changes

The PR adds inbound-media path detection, wraps text read-tool results from `media/inbound/*` with untrusted prompt-data markers, propagates `containerWorkdir`, and adds focused regression tests.

## Best Possible Solution

Land a maintained read-tool guard that classifies sandbox-staged inbound media from the resolved workspace-relative path, wraps only text blocks with sanitized labels, preserves image blocks, covers alias and non-canonical path forms, and keeps the linked security issue open until the fix merges.

## Reproduction Assessment

Yes for the underlying issue: current main stages inbound attachments as `media/inbound/<name>` and the read tool returns text without untrusted wrapping. A focused check is reading a staged text attachment containing instruction-like content through the normal read tool path.

## Solution Assessment

No, not yet. The PR uses the right wrapper and covers important bypass forms, but its broad absolute-path match is not the narrowest maintainable classifier because it can wrap unrelated trusted workspace files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Anchor absolute inbound paths to the staging root:** `src/agents/pi-tools.read.ts:140`
  - body: `isInboundMediaPath` treats any absolute path with `/media/inbound/` as inbound, and the new tests lock in `/workspace/src/media/inbound/parser.ts`. In sandboxed runs, agents can read trusted workspace files through `/workspace/...`, so repositories with a normal `src/media/inbound` directory would have source text wrapped and HTML-escaped. Classify against the resolved workspace-relative path or thread the sandbox root/workdir through instead of matching every absolute segment.
  - confidence: 0.86
