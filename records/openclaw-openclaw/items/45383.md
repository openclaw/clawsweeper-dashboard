## What This Changes

The PR wraps read-tool text returned for staged `media/inbound/*` files as untrusted prompt data, forwards `containerWorkdir`, and adds inbound-guard tests.

## Best Possible Solution

Land a root-aware inbound-media wrapper that keeps sender-controlled metadata out of trusted prompt text, preserves normal read output, and includes redacted real attachment-read proof.

## Reproduction Assessment

Yes. Source inspection shows current main stages sandboxed attachments under `media/inbound/*`, rewrites message context paths to those staged paths, and returns read-tool text without an untrusted-data wrapper.

## Solution Assessment

No. The wrapper direction is right, but the current PR should keep sender-controlled path text out of trusted wrapper labels and classify inbound reads against the staged root contract instead of arbitrary absolute path segments.
