## Summary

Keep open. Current main still stages inbound attachments into `media/inbound/*` and returns read-tool text without untrusted wrapping, so the PR targets a real linked security bug, but the current diff is not merge-ready because its absolute-path classifier still wraps trusted workspace files that merely contain a nested `/media/inbound/` segment.

## What This Changes

The PR adds inbound-media path detection, wraps text read-tool results from `media/inbound/*` with untrusted prompt-data markers, threads `containerWorkdir`, and adds focused regression tests.

## Best Possible Solution

Land a root-aware read-tool guard that classifies staged inbound media from the resolved workspace-relative path, wraps only text blocks with sanitized labels, preserves image blocks, covers supported alias forms, and leaves the linked security issue open until the corrected fix merges.

## Reproduction Assessment

Yes. For the underlying bug, current main stages channel attachments as `media/inbound/<name>` and the read tool returns text without untrusted wrapping; for the PR regression, the PR test itself shows `isInboundMediaPath("/workspace/src/media/inbound/parser.ts")` returning true.

## Solution Assessment

No, not yet. The wrapper choice is appropriate, but the current classifier should be anchored to the actual staged inbound root or to a resolved workspace-relative path instead of matching every absolute `/media/inbound/` segment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Anchor absolute inbound paths to the staging root:** `src/agents/pi-tools.read.ts:140`
  - body: `isInboundMediaPath` treats any absolute path containing `/media/inbound/` as inbound, and the new tests lock in `/workspace/src/media/inbound/parser.ts`. Sandbox reads can target trusted workspace files through `/workspace/...`, so repositories with a normal nested `src/media/inbound` directory would have source text wrapped and HTML-escaped. Classify against the resolved workspace-relative path or pass the sandbox root/workdir through instead of matching every absolute segment.
  - confidence: 0.86
