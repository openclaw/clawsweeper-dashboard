## Summary

Keep open. The security direction is useful and the linked report at https://github.com/openclaw/openclaw/issues/11207 remains open, but this PR is not merge-ready because it still puts sender-controlled path text in trusted wrapper metadata, overmatches absolute `/media/inbound/` paths, and only provides unit-test proof.

## What This Changes

This PR adds an inbound-media read-tool guard that detects `media/inbound` paths, wraps text read results with untrusted prompt-data markers, forwards `containerWorkdir`, and adds focused tests.

## Best Possible Solution

Land a rebased, root-aware inbound-media read guard that uses the shared external-content wrapper or a constant trusted label, plus redacted real attachment-read proof.

## Solution Assessment
