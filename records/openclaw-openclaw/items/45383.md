## Summary

Keep open: the PR addresses a real inbound-media prompt-injection gap, but the current diff still overmatches absolute paths containing `/media/inbound/`, so trusted workspace files such as `/workspace/src/media/inbound/parser.ts` would be wrapped and HTML-escaped.

## What This Changes

The PR adds inbound-media path detection, wraps text read results from `media/inbound/*` with `wrapUntrustedPromptDataBlock`, threads `containerWorkdir`, and adds inbound-guard tests.

## Best Possible Solution

Revise the PR so inbound classification is root-aware, wraps only staged top-level `media/inbound/*` text with sanitized labels, preserves trusted workspace source reads, and includes the required changelog entry.
