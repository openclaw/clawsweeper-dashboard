## Summary

Keep open: this addresses a real current-main prompt-injection gap, but PR head is not merge-ready because it leaves filename/path metadata in trusted prompt text, overmatches absolute paths, and lacks real behavior proof.

## What This Changes

The PR adds an inbound-media read guard for `media/inbound` paths, forwards `containerWorkdir`, and adds tests for path classification and read-result wrapping.

## Best Possible Solution

Land a root-aware inbound-media read guard with a constant trusted label, guarded filename metadata, focused tests, green checks, and redacted real attachment-read proof.

## Reproduction Assessment

Yes, source-reproducible: current main stages inbound attachments under `media/inbound` and the read tool returns sanitized text without an untrusted wrapper. I did not run a live messaging flow in this read-only review.

## Solution Assessment

No. The wrapper direction is right, but the patch should move attacker-controlled path metadata out of trusted label text and anchor inbound classification to the resolved staged root.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep inbound filenames out of trusted labels:** `src/agents/pi-tools.read.ts:798`
  - body: `sanitizeForPromptLiteral` strips control characters, but the sender-controlled filename still appears before `<untrusted-text>` as trusted prompt text. A hostile filename can look like an instruction outside the guarded block, so use a constant trusted label and put any path metadata inside guarded data.
  - confidence: 0.82
- **[P2] Anchor inbound matching to the staged root:** `src/agents/pi-tools.read.ts:140`
  - body: The absolute-path regex treats any path containing `/media/inbound/` as inbound, and the tests lock in `/workspace/src/media/inbound/parser.ts` as true. That wraps and HTML-escapes normal repository files under that path shape; classify against the resolved workspace-relative staged path or actual sandbox root instead.
  - confidence: 0.86
