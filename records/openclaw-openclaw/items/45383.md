## Summary

Keep open. The PR is useful security hardening, but it still duplicates the current shared external-content boundary, can alter trusted reads for absolute paths containing `media/inbound`, and has no after-fix real inbound attachment proof.

## What This Changes

This PR adds inbound media path detection to the agent read tool, wraps text results from `media/inbound` reads, forwards sandbox workdir metadata, and adds focused read-tool tests.

## Best Possible Solution

Land a rebased security fix that wraps inbound read-tool text with the shared external-content/file wrapper, classifies only the actual staged inbound roots, and includes redacted real attachment proof.

## Solution Assessment

No as-is. Wrapping inbound read-tool text is the right direction, but the implementation should reuse the shared external-content wrapper, avoid broad trusted-file false positives, and include real behavior proof.
