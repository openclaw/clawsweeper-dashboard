## Summary

Keep open: the underlying security gap is still present on current main, but this PR has a blocking path-classifier false positive and still lacks after-fix real behavior proof.

## What This Changes

The PR adds inbound-media path detection, wraps read-tool text blocks from staged inbound files with untrusted prompt-data markers, forwards sandbox workdir context, and adds regression tests.

## Best Possible Solution

Land a root-aware inbound-media read guard that wraps only staged inbound text blocks with sanitized labels while preserving normal trusted workspace source reads.

## Reproduction Assessment

Yes. Source inspection shows current main stages attachments as media/inbound/<filename> and returns read-tool text without an untrusted wrapper, while PR-head source/tests expose the absolute-path false positive.

## Solution Assessment

No. The wrapper direction is appropriate, but the current PR classifier is too broad because it matches every absolute /media/inbound/ segment instead of the resolved staged root.
