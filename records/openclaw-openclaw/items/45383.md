## Summary

Keep open: this PR targets a real source-reproducible inbound-media prompt-injection gap, and the author did address several earlier bot-review issues, but the current branch still has blocking security/correctness findings and only mocked unit-test proof.

## What This Changes

The PR adds an inbound-media path classifier, wraps text read from `media/inbound/*` with untrusted-text markers, propagates `containerWorkdir`, and adds focused unit tests.

## Best Possible Solution

Land a root-aware inbound-media read wrapper that uses trusted constant metadata or places path metadata inside guarded content, with focused regression tests and redacted real attachment-read proof.

## Reproduction Assessment

Yes. Current main source shows sandboxed attachments are staged under `media/inbound/*`, while the read tool returns sanitized content without an untrusted wrapper; I did not run a live messaging flow in this read-only review.

## Solution Assessment

No. The wrapper direction is right, but the branch should keep sender-controlled filenames out of trusted prompt labels and classify inbound reads against the resolved staged root instead of any absolute `media/inbound` segment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep inbound filenames out of trusted labels:** `src/agents/pi-tools.read.ts:798`
  - body: `sanitizeForPromptLiteral` strips control characters, but the staged filename still appears before `<untrusted-text>` as trusted prompt text. Since sender-controlled basenames can contain instruction-like words, use a constant trusted label and omit the path or place path metadata inside guarded content.
  - confidence: 0.82
- **[P2] Anchor inbound matching to the staged root:** `src/agents/pi-tools.read.ts:140`
  - body: The absolute-path regex classifies any path containing `/media/inbound/` as inbound, and the added tests expect `/workspace/src/media/inbound/parser.ts` to return true. That wraps and HTML-escapes normal repository files with this path shape, so classify against the resolved workspace-relative staged path or actual sandbox root.
  - confidence: 0.86

## Security Review
