## Summary

Keep open. The PR targets a real current-main inbound-media prompt-injection gap, but the latest branch still has security-sensitive trust-boundary issues and lacks after-fix real behavior proof.

## What This Changes

The PR adds an inbound-media path classifier, wraps text read from `media/inbound/*` in untrusted-text markers, forwards sandbox `containerWorkdir`, and adds inbound-guard unit tests.

## Best Possible Solution

Land a root-aware inbound-media wrapper that keeps attacker-controlled path text out of trusted prompt metadata, preserves normal read accuracy, adds focused regression tests, and includes redacted real attachment-read proof.

## Reproduction Assessment

Yes, from source inspection: current main stages sandboxed media at `media/inbound/*`, while `createOpenClawReadTool` returns sanitized read output without an untrusted wrapper. I did not run a live messaging flow in this read-only review.

## Solution Assessment

No. The wrapper direction is right, but this branch should use trusted/static label metadata and classify inbound reads against the actual staged root instead of any absolute `media/inbound` segment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep inbound filenames out of trusted labels:** `src/agents/pi-tools.read.ts:798`
  - body: `sanitizeForPromptLiteral` removes control characters, but the sender-controlled path is still placed before `<untrusted-text>` as trusted prompt metadata. Use a constant trusted label or put path metadata inside the guarded block so filename words cannot sit outside the data boundary.
  - confidence: 0.82
- **[P2] Anchor inbound matching to the staged root:** `src/agents/pi-tools.read.ts:140`
  - body: The absolute-path regex classifies any path containing `/media/inbound/` as inbound, and the tests lock in `/workspace/src/media/inbound/parser.ts` as true. That wraps and HTML-escapes normal source files with that path shape, so compare against the resolved sandbox/workspace inbound root instead.
  - confidence: 0.86
