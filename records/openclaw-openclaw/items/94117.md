## Summary

This issue should stay open: current main still emits Telegram rich anchors for explicit local/relative Markdown hrefs, the reported failure affects shipped Telegram final replies, and three open closing PRs are already candidate fixes awaiting maintainer selection.

## What This Changes

The issue requests that Telegram rich-message rendering degrade local or relative Markdown link hrefs to visible text instead of emitting anchors that Telegram rejects.

## Best Possible Solution

Land one narrow Telegram plugin fix that filters unsupported rich link hrefs before `sendRichMessage`, preserves supported public links, and adds formatter/send-path regression coverage.

## Reproduction Assessment

Yes, from source inspection and two live reports: current main still emits anchors for explicit local/relative Markdown hrefs on the Telegram rich path, and the shipped logs show Telegram rejecting those payloads with `RICH_MESSAGE_URL_INVALID`. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is in the Telegram formatter/rich-message boundary: suppress unsupported anchors before sending, preserve supported schemes, and add regression coverage. A broader parse-mode config request is adjacent but would not directly prevent one bad href from dropping the final reply.
