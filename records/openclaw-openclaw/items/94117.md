- `P1`: A real Telegram direct-chat final reply can be dropped on the latest release when rich messages contain invalid local href anchors.
- `impact:message-loss`: The reported failure loses the assistant's final response after transient progress messages.

## Summary

Keep open: current main still emits rich Telegram anchors for explicit local or relative Markdown hrefs, and the issue is already paired with open candidate PRs that use closing references. The source path and two live reports support a real message-loss bug, so this should close only after one candidate fix lands.

## What This Changes

Telegram rich final replies should degrade local or relative Markdown link hrefs to visible label text instead of sending rejected rich anchors.

## Best Possible Solution

Land one narrow Telegram formatter fix with regression coverage and real Telegram or Mantis proof, then close this issue through the merged PR.

## Reproduction Assessment

Yes. The live reports show `RICH_MESSAGE_URL_INVALID` on v2026.6.8, and current source still sends explicit local/relative Markdown hrefs as rich `<a href>` anchors before `sendRichMessage`.

## Solution Assessment

Yes, the best fix is local to the Telegram rich formatter: suppress unsupported href anchors while preserving label text and supported links. A broader parse-mode config is not needed for this specific delivery failure.
