## What This Changes

The PR updates models.json planning, provider normalization, and focused tests to avoid persisting newly generated plaintext provider apiKeys and selected sensitive provider headers while preserving non-secret markers and some legacy values.

## Best Possible Solution

Land a sanitizer that covers every prompt-visible provider credential surface in generated models.json, preserves only documented non-secret markers, and keeps the broader security roadmap open or explicitly retargeted.

## Solution Assessment

No. The direction is maintainable, but the sanitizer should follow the full sensitive provider credential surface instead of only apiKey plus a header-name allowlist, and the roadmap-closing syntax needs maintainer intent.
