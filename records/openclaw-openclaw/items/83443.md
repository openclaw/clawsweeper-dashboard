## What This Changes

The PR stops Discord progress-mode draft previews from registering `onPartialReply` while adding regression coverage that default progress mode still creates a draft stream and delivers the final reply.

## Best Possible Solution

Land the narrow progress-mode hook guard with its regression test after the draft is ready and required Discord/changed gates pass on the PR head.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main registers a Discord progress-mode `onPartialReply` hook, the hook records preview text in the shared runner, and matching final text can then be suppressed. I did not run a live Discord bot repro in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to avoid registering a text partial preview hook for progress mode, because progress-mode `updateFromPartial` intentionally no-ops while partial and block modes still need the hook.
