## Summary

Keep open. Current main still forwards bundle MCP tool input without schema-aware coercion, and this PR is the canonical fix path for #70872, but the patch is not merge-ready because it adds security-sensitive parsing without type-array, safe-object, parse-budget, or changelog coverage.

## What This Changes

The branch adds top-level stringified object/array coercion for bundle MCP tool arguments based on `inputSchema` and adds materialization regression tests.

## Best Possible Solution

Land a hardened materialization-boundary fix with schema-declared object/array coercion, JSON Schema type-array support, safe copied-argument objects, a conservative parse budget, focused regression tests, and an Unreleased changelog entry.

## Reproduction Assessment

Yes. A materialized bundle MCP tool with `params: { type: "object" }` and input `{ params: "{}" }` is forwarded unchanged on current main, matching the strict-server validation failure reported in #70872.

## Solution Assessment

No, not yet. The materialization boundary is the right place, but this branch needs schema-type coverage, argument-copy hardening, parse limits, and changelog coverage before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept JSON Schema array type forms:** `src/agents/pi-bundle-mcp-materialize.ts:50-51`
  - body: MCP tool property schemas can use JSON Schema's array-form `type`, for example `["object", "null"]`. This guard only accepts a string `type`, so nullable object or array params still pass through as JSON strings and strict servers can continue rejecting them.
  - confidence: 0.87
- **[P2] Use a safe result object for copied arguments:** `src/agents/pi-bundle-mcp-materialize.ts:43`
  - body: The helper copies model-controlled argument keys into a plain object. If a call includes a special key such as `__proto__` and another property is coerced, the returned object can have its prototype changed before it is passed to the MCP runtime; use a null-prototype result or block prototype keys.
  - confidence: 0.84
- **[P2] Bound JSON parsing for coerced arguments:** `src/agents/pi-bundle-mcp-materialize.ts:54`
  - body: Every string value under an object or array schema is synchronously parsed without a length or budget check. Oversized tool arguments can block the event loop or spike memory during tool execution, so this path should reject or skip values above a conservative limit.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `CHANGELOG.md:32`
  - body: This is a user-facing MCP fix, but the branch does not update `CHANGELOG.md`. Repo policy requires an Unreleased fix entry before merging user-facing fixes.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain changes, but it introduces model-controlled object copying and unbounded synchronous parsing in the MCP tool-call path.

Concerns:

  - body: `coerceStringifiedObjectProperties` copies tool-call keys into a plain object; special keys such as `__proto__` can alter the returned object's prototype when a coerced result is returned downstream.
