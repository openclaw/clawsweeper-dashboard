## Summary

Keep open. The reported MCP argument-shape bug is still present on current main, but this PR has merge-blocking correctness and security gaps in the proposed normalization path and is missing the required user-facing changelog entry.

## What This Changes

The PR adds top-level coercion of stringified object/array MCP tool arguments before bundle MCP tool calls and adds materialization regression tests.

## Best Possible Solution

Land a hardened materialization-boundary fix with schema-aware object/array coercion, JSON Schema type-array support, safe copied-argument objects, conservative parse limits, focused regression tests, and an Unreleased changelog entry.

## Reproduction Assessment

Yes. A bundle MCP tool whose schema declares `params` as an object will currently pass input like `{ params: "{}" }` through `materializeBundleMcpToolsForRun` and `SessionMcpRuntime.callTool` unchanged, matching the strict MCP server validation failure path.

## Solution Assessment

No, not yet. The materialization boundary is the right place, but this branch needs schema type-array handling, argument-copy hardening, parse limits, and changelog coverage before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept JSON Schema array type forms:** `src/agents/pi-bundle-mcp-materialize.ts:50-51`
  - body: MCP property schemas can use JSON Schema's array-form `type`, such as `["object", "null"]`. This guard only accepts a string `type`, so nullable object or array params still pass through as JSON strings and strict servers can continue rejecting them.
  - confidence: 0.87
- **[P2] Use a safe result object for copied arguments:** `src/agents/pi-bundle-mcp-materialize.ts:43`
  - body: The helper copies model-controlled argument keys into a plain object. If a call includes a special key such as `__proto__` alongside any coerced value, assignment can mutate the returned object's prototype before it is passed to the MCP runtime; use a null-prototype result or filter prototype keys.
  - confidence: 0.84
- **[P2] Bound JSON parsing for coerced arguments:** `src/agents/pi-bundle-mcp-materialize.ts:54`
  - body: Every string value under an object or array schema is synchronously parsed without a length or budget check. Oversized tool arguments can block the event loop or spike memory during tool execution, so this path should reject or skip values above a conservative limit.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `CHANGELOG.md:33`
  - body: This is a user-facing MCP fix, but the PR does not update `CHANGELOG.md`. Repo policy requires an Unreleased fix entry before merging user-facing fixes.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain changes, but it introduces model-controlled object copying and unbounded synchronous parsing in the MCP tool-call path.

Concerns:

  - body: `coerceStringifiedObjectProperties` copies tool-call keys into a plain object; special keys such as `__proto__` can alter the returned object's prototype when a coerced result is returned downstream.
