## Summary

Keep open because current main still has the reported MCP argument-shape bug and the proposed patch needs correctness and security hardening before merge.

## What This Changes

The PR adds top-level coercion of stringified object/array bundle MCP tool arguments and tests that those values reach the MCP runtime as native objects/arrays.

## Best Possible Solution

Land a hardened materialization-boundary fix with schema-declared object/array coercion, JSON Schema type-array support, prototype-safe copied arguments, a conservative parse budget, focused regression tests, and a changelog entry.

## Reproduction Assessment

Yes. Current main materializes bundle MCP tools from `inputSchema` but forwards execute input unchanged to `SessionMcpRuntime.callTool`, which then passes record arguments as-is to the MCP SDK client.

## Solution Assessment

No, not yet. The materialization boundary is the right place, but this branch needs schema type-array handling, safe copying, parse limits, rebase cleanup, and changelog coverage before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept JSON Schema array type forms:** `src/agents/pi-bundle-mcp-materialize.ts:50-51`
  - body: MCP property schemas can use JSON Schema's array-form `type`, such as `["object", "null"]`. This guard only accepts a string `type`, so nullable object or array params still pass through as JSON strings and strict servers can continue rejecting them.
  - confidence: 0.87
- **[P2] Use a safe result object for copied arguments:** `src/agents/pi-bundle-mcp-materialize.ts:43`
  - body: The helper copies model-controlled argument keys into a plain object. If a coerced call also includes a special key such as `__proto__`, assignment can mutate the returned object's prototype before it is passed to the MCP runtime; use a null-prototype result or block prototype keys.
  - confidence: 0.84
- **[P2] Bound JSON parsing for coerced arguments:** `src/agents/pi-bundle-mcp-materialize.ts:54`
  - body: Every string value under an object or array schema is synchronously parsed without a length or budget check. Oversized tool arguments can block the event loop or spike memory during tool execution, so this path should reject or skip values above a conservative limit.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `CHANGELOG.md:33`
  - body: This is a user-facing MCP fix, but the branch does not update `CHANGELOG.md`. Repo policy requires an Unreleased fix entry before merging user-facing fixes.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain changes, but it introduces model-controlled object copying and unbounded synchronous parsing in the MCP tool-call path.

Concerns:

  - body: `coerceStringifiedObjectProperties` copies tool-call keys into a plain object; special keys such as `__proto__` can alter the returned object's prototype when a coerced result is returned downstream.
