## Summary

Keep open. The underlying MCP argument bug is still present on current main, and the PR targets the right materialization boundary, but the branch has blocking correctness and security issues before it should merge.

## What This Changes

The PR adds bundle MCP materialization logic that parses top-level stringified object and array arguments according to each tool input schema before calling MCP runtimes, with regression tests for stringified and already-native params.

## Best Possible Solution

Land a hardened version at the bundle MCP materialization boundary: coerce only schema-declared object and array properties, handle JSON Schema type arrays, copy through a safe result object, apply a conservative parse budget, and keep focused regression coverage.

## Reproduction Assessment

Yes. On current main, a materialized bundle MCP tool with schema property `params: { type: "object" }` and input `{ params: "{}" }` is forwarded unchanged through `materializeBundleMcpToolsForRun` and `SessionMcpRuntime.callTool`, matching the #70872 failure path.

## Solution Assessment

No, not yet. The materialization boundary is the right place to solve this, but the current branch needs JSON Schema type-array support and security hardening before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept JSON Schema array type forms:** `src/agents/pi-bundle-mcp-materialize.ts:50-51`
  - body: MCP tool schemas carry JSON Schema property definitions, and JSON Schema allows `type` to be an array such as `["object", "null"]`. This guard only accepts a string `type`, so nullable object or array params still pass through as JSON strings and strict servers can continue rejecting them.
  - confidence: 0.87
- **[P2] Use a null-prototype result object:** `src/agents/pi-bundle-mcp-materialize.ts:43`
  - body: The helper returns a plain object after copying model-controlled argument keys. If a call includes `__proto__` with an object value alongside a coerced field, assignment can mutate the returned object's prototype before it is passed to the MCP runtime; create the result with a null prototype or define own data properties.
  - confidence: 0.84
- **[P2] Bound JSON.parse for coerced arguments:** `src/agents/pi-bundle-mcp-materialize.ts:54`
  - body: Every string value under an object or array schema is synchronously parsed without a size budget. An oversized tool argument can block the event loop or spike memory during tool execution, so this path should reject or skip values above a conservative limit.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain changes, but it introduces model-controlled object copying and unbounded synchronous parsing in the MCP tool-call path.

Concerns:

- **[medium] Prototype mutation through copied argument keys:** `src/agents/pi-bundle-mcp-materialize.ts:43`
  - body: `coerceStringifiedObjectProperties` copies tool-call keys into a plain object; special keys such as `__proto__` can alter the returned object's prototype when a coerced result is returned downstream.
