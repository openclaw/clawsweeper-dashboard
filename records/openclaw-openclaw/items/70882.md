## Summary

Keep open. The PR targets a real current-main bundle MCP bug at the right boundary, but the branch still needs schema coverage and security hardening before it is mergeable.

## What This Changes

The PR parses top-level stringified object and array MCP tool arguments according to each tool input schema and adds regression tests for that materialization path.

## Best Possible Solution

Land a hardened materialization-boundary fix: schema-declared object/array coercion, JSON Schema type-array support, safe copied-argument objects, a conservative parse budget, focused regression tests, and a user-facing changelog entry.

## Reproduction Assessment

Yes. A materialized bundle MCP tool with `params: { type: "object" }` and input `{ params: "{}" }` is forwarded unchanged on current main, matching the linked strict-server validation failure path.

## Solution Assessment

No, not yet. The materialization boundary is the right place, but this branch needs JSON Schema array-type handling, argument-copy hardening, parse limits, and changelog coverage before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept JSON Schema array type forms:** `src/agents/pi-bundle-mcp-materialize.ts:50-51`
  - body: MCP tool property schemas can use JSON Schema's array-form `type`, for example `["object", "null"]`. This guard only accepts a string `type`, so nullable object or array params still pass through as JSON strings and strict servers can continue rejecting them.
  - confidence: 0.87
- **[P2] Use a safe result object for copied arguments:** `src/agents/pi-bundle-mcp-materialize.ts:43`
  - body: The helper copies model-controlled argument keys into a plain object. If a call includes a special key such as `__proto__`, assignment can mutate the returned object's prototype before it is passed to the MCP runtime; create a null-prototype result or block prototype keys.
  - confidence: 0.84
- **[P2] Bound JSON parsing for coerced arguments:** `src/agents/pi-bundle-mcp-materialize.ts:54`
  - body: Every string value under an object or array schema is synchronously parsed without a length or budget check. Oversized tool arguments can block the event loop or spike memory during tool execution, so this path should reject or skip values above a conservative limit.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `CHANGELOG.md:31`
  - body: This is a user-facing MCP fix, but the PR does not update `CHANGELOG.md`. Repo policy requires an Unreleased fix entry before merging user-facing fixes.
  - confidence: 0.86
