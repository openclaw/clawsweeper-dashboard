## Summary

Keep open: current main and the latest release fixed locally resolvable Gateway token SecretRefs and adjacent reachable-Gateway diagnostics, but the runtime-healthy/CLI-unavailable Doctor warning remains a security-sensitive auth-policy question.

## What This Changes

The issue asks Doctor to stop warning that a SecretRef-managed Gateway token is unavailable when the Gateway runtime is healthy and using that SecretRef successfully.

## Best Possible Solution

Define a Gateway-auth-specific runtime proof or active-runtime snapshot that downgrades this warning only when the configured SecretRef is known usable by Gateway, while preserving fail-closed diagnostics and never writing plaintext fallback credentials.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure local token auth with a `gateway.auth.token` SecretRef unavailable to the Doctor command path while Gateway is healthy, and the current gateway-auth contribution can warn before any Gateway health proof is collected.

## Solution Assessment

No. The existing fix is best for locally resolvable refs, but broad suppression based only on Gateway reachability is not the best remaining fix because it could mask real unresolved configured secrets.
