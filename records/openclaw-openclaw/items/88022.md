## What This Changes

The issue asks Doctor to avoid misleading Gateway auth SecretRef warnings when Gateway runtime can use the token but the CLI audit context cannot resolve the SecretRef.

## Best Possible Solution

Define a Gateway-auth-specific runtime proof or active-runtime snapshot that can downgrade this warning only when the configured SecretRef is known usable by Gateway, while preserving fail-closed unresolved diagnostics and never writing plaintext fallback credentials.

## Reproduction Assessment

Yes, source-reproducible: configure local token auth with a `gateway.auth.token` SecretRef unavailable to the CLI command path, then Doctor reaches the warning branch before any Gateway health proof is collected. I did not run the reporter's live macOS file/exec setup.

## Solution Assessment
