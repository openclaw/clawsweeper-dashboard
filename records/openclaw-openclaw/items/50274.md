## Summary

Keep open. Current main and v2026.6.10 still lack a benchmark-backed fallback ordering policy for the named active chain, and the model-input guardrail remains an unresolved compatibility decision because current config accepts audio/video while the session registry still rejects those modalities in generated catalog shards.

## What This Changes

The issue asks OpenClaw to benchmark and define fallback model order, add guardrails for unsupported configured model input modalities, and document the resulting fallback/model-tier policy.

## Best Possible Solution

Define a benchmark-backed fallback policy for the active chain, then settle one capability-precedence contract that aligns config schema, session registry, provider catalogs, docs, tests, and any needed doctor migration or override path.

## Reproduction Assessment

Yes, at source level. Current main preserves configured fallback order and has conflicting model-input contracts: config accepts audio/video while the session registry still only accepts text/image for generated model catalogs.

## Solution Assessment

Unclear until maintainers choose the fallback ranking criteria and model-capability authority. The best path is not blanket schema tightening; it needs benchmark policy plus compatibility-safe semantic validation.
