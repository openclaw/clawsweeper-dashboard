The PR adds guards and handler-level tests for undefined `commandName` and `targetIds` in `secrets.resolve` requests.

## Best Possible Solution

Move the fix and regression to the gateway connect/pairing normalization path that can throw during handshake, preserving fail-closed scope semantics and proving the CLI write path after the change.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: the linked issue maps CLI write commands into the gateway connect path, and current main still has a non-string-unsafe trim there, but this read-only review did not run a live gateway smoke.

## Solution Assessment

Is this the best way to solve the issue? No. The proposed guards are after schema validation in `secrets.resolve`, while the reported crash occurs during gateway connect before that handler is reached.
