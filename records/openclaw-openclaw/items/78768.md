## Summary

Keep open: the latest head still has a concrete P2 regression in the Mantis recording path, and the PR carries the protected maintainer label. The broader fs-safe staging direction looks reasonable, but this PR should not merge until the recoverable recording-failure artifact path is preserved.

## What This Changes

This PR bumps the pinned `@openclaw/fs-safe` commit, exports `writeExternalFileWithinRoot` through OpenClaw security facades, and routes selected browser, media, channel, and QA external file outputs through staged finalization.

## Best Possible Solution

Keep the fs-safe staged-output hardening, but make the Mantis recording path publish any non-empty staged video before surfacing the expected record failure and cover that behavior with a regression test.

## Reproduction Assessment

Yes, source-reproducible. Current main writes `crabbox record` directly to `videoPath` and still checks that artifact after a recoverable nonzero exit, while the PR writes to a staged path that is cleaned up before final publication when the command throws.

## Solution Assessment

No, not yet. `writeExternalFileWithinRoot` is the right primitive for successful external writers, but this specific Mantis call needs an opt-in preservation path for the existing recoverable record failure.
