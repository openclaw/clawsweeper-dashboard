## Summary

Keep open. Current main still lacks Feishu native exec approval cards, so the PR is not obsolete, but the latest PR head still has concrete merge-blocking issues in approver normalization and plugin-approval handling, plus the required changelog entry is missing.

## What This Changes

The PR adds Feishu native exec approval cards, a handler-based delivery/runtime path, card action resolution through the Gateway, Feishu `execApprovals` config, and supporting tests.

## Best Possible Solution

Keep the feature in the Feishu plugin, preserve the generic approval capability seams, validate Feishu approver IDs after parsing, avoid plugin approvals being rendered as exec approval cards unless a plugin-specific Feishu card renderer is added, and document the user-facing feature in the changelog before merge.

## Reproduction Assessment

Yes. The remaining defects reproduce by static inspection of the PR head: `user:foo` is counted as an approver even though card callbacks authorize `event.operator.open_id`, and plugin approval events flow into an exec-card resolved renderer.

## Solution Assessment

No. The Handler pattern is the right direction for a Feishu-owned implementation, but the current patch needs the normalization and plugin-event fixes plus changelog coverage before it is the best mergeable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Validate parsed Feishu approver IDs:** `extensions/feishu/src/exec-approvals.ts:32-34`
  - body: `normalizeFeishuDirectApproverId` returns the parsed `user:`/`dm:`/`open_id:` suffix before checking that it is an `ou_` open_id. A config such as `approvers: ["user:foo"]` therefore enables the native client with an approver that can never match `event.operator.open_id`, leaving card approvals unresolvable. Validate the parsed value before returning it and add a regression test.
  - confidence: 0.92
- **[P2] Keep plugin approvals out of the exec card handler:** `extensions/feishu/src/exec-approvals-handler.ts:59`
  - body: The handler subscribes to plugin approvals, but `finalizeResolved` always builds a Feishu exec approval resolved card. A plugin approval delivered through this path is replaced with a “command execution approval” card and loses its plugin title/description. Either limit this handler to exec events or render plugin-specific pending/resolved cards.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `extensions/feishu/src/config-schema.ts:141-149`
  - body: This adds user-facing Feishu approval buttons and a new `channels.feishu.execApprovals` config surface, but the PR does not update `CHANGELOG.md`. Repo policy requires a user-facing `feat` change to add an active-version changelog entry with appropriate contributor credit.
  - confidence: 0.93
