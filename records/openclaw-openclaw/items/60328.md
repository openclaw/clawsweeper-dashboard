## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate parsed approver IDs before enabling the handler:** `extensions/feishu/src/exec-approvals.ts:33-34`
  - body: `parseFeishuDirectConversationId()` returns the suffix for `user:`, `dm:`, and `open_id:` values before checking prefixes, so `user:foo` is returned here as a configured approver. That can make `execApprovals.enabled` turn on with an approver that will never match `event.operator.open_id` in `card-action.ts`, leaving approval cards unresolvable. Validate the parsed value is a Feishu `ou_` open_id before returning it, and add a regression case for prefixed invalid IDs.
  - confidence: 0.93
- **[P2] Use a focused approval gateway seam:** `extensions/feishu/src/card-action.ts:1-5`
  - body: This adds new bundled-plugin imports from `openclaw/plugin-sdk/infra-runtime`, and the branch exposes raw `callGateway` through that deprecated compatibility barrel to make the callback path compile. Current main explicitly says repo code must not add new `infra-runtime` imports and should use focused SDK subpaths, so this will conflict with the SDK boundary checks and broadens the plugin-facing gateway RPC surface. Add or use a narrow approval/gateway-runtime helper instead, then remove the infra-runtime export.
  - confidence: 0.88
