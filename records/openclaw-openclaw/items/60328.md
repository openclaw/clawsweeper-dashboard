- **[P1] Validate parsed Feishu approver IDs before returning them:** `extensions/feishu/src/exec-approvals.ts:33-34`

Values like `user:foo` or uppercase `user:OU_123` can therefore enable the native approval client while never matching `event.operator.open_id`, leaving card approvals unresolvable.
  - confidence: 0.91

  - body: `getFeishuExecApprovalApprovers` falls back to `allowFrom`, and this branch's enabled helper treats an unset `execApprovals.enabled` as enabled when approvers exist. That means existing Feishu configs with only `allowFrom` can unexpectedly start the native handler and suppress fallback prompts even though the PR describes the feature as opt-in.
  - confidence: 0.82
- **[P1] Use the focused approval gateway SDK seam:** `src/plugin-sdk/infra-runtime.ts:52-53`
  - body: Adding `callGateway` and gateway client mode constants to `openclaw/plugin-sdk/infra-runtime` exposes a broad raw gateway API through a public plugin SDK barrel for one approval callback.
