Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate parsed approver IDs before enabling native approvals:** `extensions/feishu/src/exec-approvals.ts:33-34`
  - body: `parseFeishuDirectConversationId()` returns any `user:`, `dm:`, or `open_id:` suffix, so this early return lets entries like `user:foo` count as configured approvers and enable the native client even though Feishu callbacks compare against `operator.open_id`. Validate the parsed value is an `ou_` open_id before returning it and add coverage for invalid prefixed IDs.
  - confidence: 0.93
- **[P2] Keep plugin approvals out of exec card finalizers:** `extensions/feishu/src/exec-approvals-handler.ts:59`
  - body: Subscribing this handler to `plugin` approval events means plugin requests are sent as fallback text cards, but the shared finalizer always replaces the message with `createExecApprovalResolvedCard()`. That overwrites plugin approval context with a command-exec approval result, so either limit this handler to exec events or add plugin-specific resolved/expired rendering.
  - confidence: 0.89
- **[P1] Use the focused approval gateway resolver:** `src/plugin-sdk/infra-runtime.ts:52-53`
  - body: This adds `callGateway` and gateway client constants to the deprecated broad `infra-runtime` SDK shim, while current main already exposes `openclaw/plugin-sdk/approval-gateway-runtime` for this exact approval resolution path. Keeping generic gateway RPC out of the broad plugin SDK avoids a larger public security surface and preserves the approval-specific exec/plugin fallback behavior.
  - confidence: 0.86
