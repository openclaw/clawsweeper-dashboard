## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate parsed Feishu approver IDs:** `extensions/feishu/src/exec-approvals.ts:32-34`
  - body: `parseFeishuDirectConversationId()` can return arbitrary prefixed suffixes like `user:foo` and bare `on_...`, but this code returns that parsed value before enforcing the `ou_` open_id check used by Feishu card callbacks. Invalid entries can enable the native client with approvers that can never match `event.operator.open_id`; validate the parsed value before returning it and add coverage for prefixed invalid IDs.
  - confidence: 0.94
- **[P2] Keep plugin approvals out of exec card finalizers:** `extensions/feishu/src/exec-approvals-handler.ts:59`
  - body: Subscribing this handler to plugin approvals means plugin prompts can be delivered here, but the resolved and expired finalizers always call `createExecApprovalResolvedCard()`. Resolving or expiring a plugin approval can replace the plugin prompt with an exec approval result and drop the plugin title/description; either subscribe only to exec events or render plugin resolved/expired cards explicitly.
  - confidence: 0.88
