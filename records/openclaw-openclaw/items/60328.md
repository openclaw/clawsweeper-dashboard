## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Validate parsed Feishu approver IDs:** `extensions/feishu/src/exec-approvals.ts:32-34`
  - body: `parseFeishuDirectConversationId()` can return arbitrary `user:`, `dm:`, or `open_id:` suffixes before the later `ou_` check runs, so values like `user:foo` are kept as approvers and can enable the native client with IDs that will not match Feishu card callback `event.operator.open_id`. Validate the parsed value against the same `ou_` rule before returning it and add coverage for prefixed invalid IDs.
  - confidence: 0.94
- **[P2] Keep plugin approvals out of exec card finalizers:** `extensions/feishu/src/exec-approvals-handler.ts:59`
  - body: Subscribing this handler to `plugin` events sends plugin approvals through finalizers that always build exec approval result cards. Resolving or expiring a plugin approval can replace the plugin prompt with command-execution approval UI and drop the plugin title/description; either subscribe only to exec events or render plugin-specific resolved/expired cards.
  - confidence: 0.9
- **[P2] Use focused approval gateway exports:** `src/plugin-sdk/infra-runtime.ts:52-53`
  - body: This adds `callGateway` to the deprecated broad `infra-runtime` SDK barrel and new Feishu code imports from that guarded subpath. The repo scanner blocks bundled plugin use of `openclaw/plugin-sdk/infra-runtime`, and the existing `approval-gateway-runtime` helper already covers approval resolution without exposing generic gateway RPC through this barrel.
  - confidence: 0.86
