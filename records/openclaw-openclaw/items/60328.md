## Summary

Keep open: current main still does not implement Feishu native exec approval cards, and PR head still has source-reproducible approval correctness blockers plus missing real Feishu proof.

## What This Changes

The PR adds a Feishu native exec approval card handler, card-action gateway resolution, `channels.feishu.execApprovals` config, approval capability wiring, SDK exports, and Feishu approval tests.

## Best Possible Solution

Land Feishu-owned native approval cards on the current shared approval runtime with strict `ou_` approver normalization, correct exec/plugin rendering, docs/config alignment, focused regression coverage, and redacted real Feishu proof.

## Reproduction Assessment

Yes. The blocking review findings are source-reproducible by tracing PR-head approver normalization into Feishu `event.operator.open_id` checks and by tracing plugin approval events through the Feishu handler's exec-card finalizer.

## Solution Assessment

No. The Feishu-plugin direction is appropriate, but this branch should validate parsed IDs, scope or render plugin approval events correctly, update to current approval-runtime patterns as needed, and include real Feishu proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate parsed Feishu approver IDs:** `extensions/feishu/src/exec-approvals.ts:32-34`
  - body: `parseFeishuDirectConversationId()` accepts prefixed suffixes like `user:foo` and bare `on_...`, but this normalizer returns that parsed value before enforcing the `ou_` open_id check used by Feishu card callbacks. Invalid entries can enable the native client with approvers that can never match `event.operator.open_id`, leaving approvals unresolvable; validate the parsed value before returning it and add coverage for prefixed invalid IDs.
  - confidence: 0.94
- **[P2] Keep plugin approvals out of exec cards:** `extensions/feishu/src/exec-approvals-handler.ts:59`
  - body: Subscribing this handler to `plugin.approval.*` means plugin approval cards are delivered here, but `finalizeResolved()` always calls `createExecApprovalResolvedCard()`. Resolving a plugin approval can replace the plugin prompt with an exec approval result and drop the plugin title/description; either subscribe only to exec events or render plugin resolved/expired cards explicitly.
  - confidence: 0.87
