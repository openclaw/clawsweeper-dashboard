Full review comments:

- **[P1] Validate parsed Feishu approver IDs:** `extensions/feishu/src/exec-approvals.ts:32-34`
  - body: `parseFeishuDirectConversationId()` can return arbitrary `user:`, `dm:`, or `open_id:` suffixes, and this early return accepts them before the `ou_` check. Values like `user:foo` or `user:OU_123` can count as approvers and enable the native client, but they will never match Feishu card callback `event.operator.open_id`, leaving approvals unresolvable. Validate the parsed value against the same `ou_` rule before returning it and add coverage for prefixed invalid IDs.
  - confidence: 0.94
- **[P2] Avoid exec result cards for plugin approvals:** `extensions/feishu/src/exec-approvals-handler.ts:59`
  - body: This handler subscribes to `plugin` approval events, but `finalizeResolved` and `finalizeExpired` always build exec approval result cards. A plugin approval can therefore be replaced in-place with command-execution approval UI and lose its plugin title/description; either subscribe only to exec events or render plugin-specific resolved/expired cards.
  - confidence: 0.9
- **[P2] Use the focused approval gateway resolver:** `src/plugin-sdk/infra-runtime.ts:52-53`
  - body: This adds generic `callGateway` and gateway constants to the deprecated `infra-runtime` SDK barrel, and the new Feishu code imports from that guarded subpath. The existing `approval-gateway-runtime` resolver already covers approval resolution without expanding generic gateway RPC through a broad public barrel.
  - confidence: 0.87
