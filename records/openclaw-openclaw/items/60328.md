Full review comments:

- **[P1] Validate parsed Feishu approver IDs:** `extensions/feishu/src/exec-approvals.ts:33-34`
  - body: `normalizeFeishuDirectApproverId` returns the parsed prefixed value before checking that it is an `ou_` open_id. Since `parseFeishuDirectConversationId` returns any `user:`, `dm:`, or `open_id:` suffix, a value like `user:not-open-id` can count as an approver and enable the native client while never matching `event.operator.open_id` at runtime. Re-check the parsed value before returning it and add coverage for invalid prefixed IDs.
  - confidence: 0.91
- **[P1] Use the approval gateway runtime instead of exporting callGateway:** `src/plugin-sdk/infra-runtime.ts:52-53`
  - body: This broadens `openclaw/plugin-sdk/infra-runtime` with raw gateway RPC helpers so the Feishu button path can call `exec.approval.resolve` directly. Existing native approval channels use the focused `openclaw/plugin-sdk/approval-gateway-runtime` helper, which keeps the operator-approvals contract narrow and avoids turning general gateway RPC into a public plugin SDK convenience export. Please route Feishu card clicks through `resolveApprovalOverGateway` and drop these exports.
  - confidence: 0.88
- **[P2] Align the new Feishu execApprovals config surface:** `extensions/feishu/src/config-schema.ts:198`
  - body: Adding `channels.feishu.execApprovals` creates a user-facing config block, but this PR does not update Feishu docs or generated channel config metadata/hints. Add the docs and regenerated metadata/contract coverage with the schema change.
  - confidence: 0.82
