Keep open: the docs cover a real Feishu dynamic-agent code path, but the proposed guidance currently conflicts with OpenClaw's session-isolation/security docs and overstates what the feature guarantees.

## What This Changes

The PR adds Feishu/Lark dynamicAgentCreation configuration docs and zh-CN glossary entries under docs/.

## Best Possible Solution

Revise the Feishu docs to describe dynamicAgentCreation accurately, including config-write behavior, workspace/bootstrap timing, dmScope tradeoffs, and realistic security boundaries, then keep the glossary entries that still match the final terms.

## Reproduction Assessment

Not applicable as a runtime bug; the docs defects are source-reproducible by comparing the PR's new Feishu section against current routing, workspace bootstrap, and security docs on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Correct the dmScope recommendation:** `docs/channels/feishu.md:436`
  - body: The new example makes `session.dmScope: "main"` the critical setting for a public multi-user bot, but current routing uses `main` to collapse direct messages to the agent main session while the security/config docs recommend `per-channel-peer` or `per-account-channel-peer` for multi-user DMs. Since this setting is global, the proposed guidance can weaken isolation for other direct-message routes; document the dynamic-agent tradeoff instead of making `main` required.
  - confidence: 0.89
- **[P3] Describe bootstrap creation accurately:** `docs/channels/feishu.md:447`
  - body: The Feishu dynamic-agent helper creates the workspace and agentDir directories, adds an agent entry, and writes a binding; it does not copy `AGENTS.md`, `SOUL.md`, or `USER.md` at first DM time. Bootstrap files are created later by the shared workspace helper when the agent workspace is ensured, so this step should be reworded to avoid documenting behavior the handler does not perform.
  - confidence: 0.86
- **[P2] Qualify the isolation guarantee:** `docs/channels/feishu.md:524`
  - body: `dynamicAgentCreation` gives each sender a separate agent/workspace route, but it is not a hard security boundary, and the security docs explicitly describe DM/session isolation as a messaging-context boundary rather than hostile co-tenant isolation. Replace the `100%` guarantee with precise workspace/session isolation wording plus a security caveat.
  - confidence: 0.9
