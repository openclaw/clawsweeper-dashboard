## Summary

Keep open: the docs direction is useful, but the current examples still conflict with the source contract for `agentDirTemplate` and could steer per-agent state into a relative process directory.

## What This Changes

This PR adds Feishu/Lark `dynamicAgentCreation` documentation and zh-CN glossary entries for per-user dynamic agents.

## Best Possible Solution

Merge a docs-only revision that uses the real dynamic-agent path templates, keeps the session/security caveats, and leaves named-account runtime fixes to the canonical Feishu issue.

## Reproduction Assessment

Yes for the docs defect: the PR diff shows `agentDirTemplate: "{agentId}"`, and current main treats that value as a filesystem path template used directly as `agentDir`. No live Feishu run was needed for this source-contract check.

## Solution Assessment

No, not as written; documenting the feature is the right direction, but the examples should omit `agentDirTemplate` or use `~/.openclaw/agents/{agentId}/agent` and describe it as a path template.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the real agentDirTemplate path:** `docs/channels/feishu.md:429`
  - body: The examples override `agentDirTemplate` with `{agentId}`, but the Feishu helper treats this setting as a filesystem path template and only expands `~`. Following the docs would create each agent directory relative to the gateway process cwd instead of under `~/.openclaw/agents/{agentId}/agent`, where per-agent auth/runtime state is expected to live. Use the default full path or omit the override, and describe the option as a path template.
  - confidence: 0.91
