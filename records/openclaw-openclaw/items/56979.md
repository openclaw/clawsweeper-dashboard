## Summary

Keep open. Current main still writes every normal `model-fallback/decision` warning directly, and the only existing suppression is probe-console-only after file logging, so the expired-token fallback log spam remains source-provable and actionable.

## What This Changes

The issue requests throttling or coalescing repeated model fallback decision logs when an auth provider token expires.

## Best Possible Solution

Add bounded duplicate coalescing in the model fallback observability path, keyed by stable decision/requested/candidate/reason/next/error fields, without changing fallback selection or auth cooldown policy.

## Reproduction Assessment

Yes, at source level. Repeated fallback runs with all profiles for a provider in cooldown reach `observeDecision` and `logModelFallbackDecision`, which currently warns directly without coalescing; I did not run a live expired-token gateway setup.

## Solution Assessment

Yes. The narrow maintainable fix is duplicate coalescing in `src/agents/model-fallback-observation.ts` or a small logging helper, not a change to provider fallback policy or auth cooldown semantics.
