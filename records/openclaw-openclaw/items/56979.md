## Summary

Keep open. Current main still emits each normal `model-fallback/decision` warning directly, and the existing probe suppression only filters console output after file logging, so the expired-token fallback log spam remains actionable.

## What This Changes

The issue asks OpenClaw to reduce repeated model fallback decision logs when an auth provider token expires and auth profiles enter cooldown.

## Best Possible Solution

Add bounded duplicate coalescing in the model fallback observability path, keyed by stable decision/requested/candidate/reason/next/error fields, without changing fallback selection or auth cooldown policy.

## Reproduction Assessment

Yes, at source level. Repeated `runWithModelFallback` calls with all profiles for a provider in cooldown reach `observeDecision` and `logModelFallbackDecision`, which currently warns directly without coalescing; I did not run a live expired-token gateway setup.

## Solution Assessment

Yes. The narrow maintainable fix is duplicate coalescing in `src/agents/model-fallback-observation.ts` or a small logging helper, not a change to provider fallback policy or auth cooldown semantics.
