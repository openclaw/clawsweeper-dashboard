## Summary

Keep open. Current main and the latest release still route auth-cooldown fallback decisions through an uncoalesced warning logger, and the related search did not find a canonical merged or open PR that owns this exact log-spam repair.

## What This Changes

The issue asks OpenClaw to reduce repeated `model-fallback/decision` warning logs when an expired auth provider token triggers fallback/cooldown behavior.

## Best Possible Solution

Add bounded duplicate coalescing in the model fallback observability path, keyed by stable decision/requested/candidate/reason/next/error fields, without changing fallback selection or auth cooldown policy.

## Reproduction Assessment

Yes, at source level. Repeated fallback runs with all profiles for a provider in cooldown reach `observeDecision` and `logModelFallbackDecision`, which currently emits each normal warning without duplicate coalescing.

## Solution Assessment

Yes. The narrow maintainable fix is coalescing in `src/agents/model-fallback-observation.ts` or a small shared logging helper, not a change to provider fallback policy or auth cooldown semantics.
