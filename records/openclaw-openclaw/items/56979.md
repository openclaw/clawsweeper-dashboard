## Summary

Keep open. Current main has related fallback/cooldown work, but normal auth skip/cooldown paths still feed the warning-level `model-fallback/decision` logger without a default coalescing throttle, so the central log-spam problem remains source-reproducible.

## What This Changes

The issue asks OpenClaw to reduce repeated `model-fallback/decision` logging when an auth provider token expires and fallback repeatedly skips or retries the same provider.

## Best Possible Solution

Keep fallback routing semantics unchanged and add default-on coalescing or throttling for repeated auth/cooldown fallback decision warnings, with focused regression coverage for normal runs.

## Reproduction Assessment

Yes from source at medium confidence: normal auth skip/cooldown fallback paths still feed the warning-level decision logger. I did not run a live expired-token gateway scenario in this read-only review.

## Solution Assessment

Yes: a narrow coalescing/throttle around repeated auth/cooldown decision logs is the best fit, because changing fallback routing or adding another config knob would be broader than the reported logging bug.
