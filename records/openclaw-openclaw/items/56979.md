## Summary

Keep open. Current main still writes every normal `model-fallback/decision` warning directly, and the existing probe suppression only affects console output after file logging, so the expired-token/cooldown log spam remains actionable.

## What This Changes

The issue asks OpenClaw to reduce repeated `model-fallback/decision` warnings when an auth provider token expires and fallback/cooldown handling repeats.

## Best Possible Solution

Add bounded duplicate coalescing in the model fallback observability path, keyed by stable decision/requested/candidate/reason/next/error fingerprint, without changing fallback selection or auth cooldown policy.

## Reproduction Assessment

Yes, at source level. Put all auth profiles for a provider into cooldown and repeatedly invoke `runWithModelFallback`; current main reaches the direct decision warning path for each skipped, probed, or failed candidate, though I did not run a live expired-token gateway setup.

## Solution Assessment

Yes. The narrow maintainable fix is coalescing in `src/agents/model-fallback-observation.ts` or a small logging helper, not changing provider fallback policy or auth cooldown semantics.
