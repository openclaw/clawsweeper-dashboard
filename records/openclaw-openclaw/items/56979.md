## Summary

Keep open. Current main and the latest release still emit warn-level `model-fallback/decision` records for each non-probe fallback decision, and the auth skip cache that could reduce repeat auth attempts remains opt-in, so the reported log spam is still source-reproducible.

## What This Changes

The issue requests reducing repeated auth-triggered model fallback decision logs when a provider token expires.

## Best Possible Solution

Add a bounded default coalescer for repeated auth-class fallback decision observations while preserving first-occurrence structured logs, auth cooldown behavior, and the separate provider-quarantine policy thread at https://github.com/openclaw/openclaw/issues/47910.

## Reproduction Assessment

Yes, by source inspection rather than a fresh live expired-token run. Expired-token text classifies as `auth`, the fallback loop sends failures/skips to the warn-level decision logger, and the default skip cache remains disabled.

## Solution Assessment

Yes for the narrow logging bug: coalescing repeated auth-class fallback decision observations is a better fit than changing provider routing semantics. The broader provider-health/quarantine behavior should stay in https://github.com/openclaw/openclaw/issues/47910.
