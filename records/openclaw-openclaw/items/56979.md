## What This Changes

The issue reports excessive repeated `model-fallback/decision` logs when an auth provider token expires and model fallback keeps retrying auth-failing candidates.

## Best Possible Solution

Review and land the open coalescing PR if maintainers accept the diagnostic tradeoff, while keeping broader provider quarantine policy in https://github.com/openclaw/openclaw/issues/47910.

## Reproduction Assessment

Yes, for a source-level reproduction: current main classifies auth/cooldown fallback decisions into `logModelFallbackDecision`, and that function warns unconditionally. I did not run a live expired-token gateway scenario because this read-only review has no real provider credentials.

## Solution Assessment

Yes: a narrow coalescer in fallback decision logging is the best current fix for this issue because it reduces duplicate diagnostics without changing auth cooldown, provider selection, or fallback routing.
