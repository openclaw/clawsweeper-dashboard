## Summary

Keep open. Current main still emits each normal `model_fallback_decision` warning directly, cooldown skip/probe paths feed that logger on repeated fallback runs, and the duplicate-search pass did not find a canonical open fix PR owning the log-spam repair.

## What This Changes

The issue asks OpenClaw to reduce repeated `model-fallback/decision` warnings when expired provider credentials put auth profiles into cooldown and trigger model fallback.

## Best Possible Solution

Add bounded duplicate coalescing in the model fallback observability path, keyed by stable decision/requested/candidate/reason/next/error fields, without changing fallback selection or auth cooldown policy.

## Reproduction Assessment

Yes, at source level. Repeated fallback runs with all profiles for a provider in cooldown can reach `observeDecision` and `logModelFallbackDecision`, which currently emits each normal warning without duplicate coalescing.

## Solution Assessment

Yes. The narrow maintainable fix is coalescing in `src/agents/model-fallback-observation.ts` or a small shared logging helper, not a change to provider fallback policy or auth cooldown semantics.
