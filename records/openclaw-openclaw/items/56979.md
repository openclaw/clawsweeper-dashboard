## Summary

Keep open. Current main still emits each normal `model-fallback/decision` warning directly, and existing probe suppression is console-only, so the reported expired-token/cooldown log spam remains actionable.

## What This Changes

Reduce repeated model fallback decision logs when an auth provider token expires or enters cooldown.

## Best Possible Solution

Add bounded duplicate coalescing in the model fallback observability path, keyed by stable decision/requested/candidate/reason/next/error fingerprint, without changing fallback selection or auth cooldown policy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to put a provider in auth cooldown and repeatedly invoke `runWithModelFallback`; current main reaches the direct decision warning path for each skipped or failed candidate.

## Solution Assessment

Yes. The narrow maintainable fix is coalescing in `src/agents/model-fallback-observation.ts` or a small logging helper, not changing provider fallback policy.
