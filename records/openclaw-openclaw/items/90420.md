The issue asks for `models.list` and WebUI model picker rows to expose agent runtime metadata for canonical OpenAI model choices routed through Codex after the `openai-codex` migration.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90328, or an equivalent single canonical optional `ModelChoice.agentRuntime` contract, while preserving canonical `openai/*` model values.

## Reproduction Assessment

Yes, for the source-level observability gap: current main lacks `agentRuntime` on `ModelChoice`, `models.list` does not decorate rows with runtime metadata, and the WebUI picker label path has no runtime suffix. I did not replay the original Telegram upgrade scenario.

## Solution Assessment
