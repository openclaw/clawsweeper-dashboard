## Summary

## What This Changes

The issue requests that Gateway/UI model surfaces expose when canonical OpenAI model choices are routed through the Codex runtime after the 2026.6.1 migration.

## Best Possible Solution

Land or refine https://github.com/openclaw/openclaw/pull/90328 or an equivalent additive fix that preserves canonical openai/* model values while exposing non-default/diagnostic agentRuntime metadata in models.list and the model picker.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main resolves runtime metadata for sessions/agents, but the current models.list schema/result and UI model option label path do not expose or render model-choice agentRuntime metadata. I did not run a live Gateway because this review is read-only.

## Solution Assessment

Yes, exposing runtime metadata separately while preserving canonical openai/* model values is the right boundary; reintroducing openai-codex as a visible provider route would conflict with the migration policy. The exact protocol/UI shape should be reviewed through the linked PR or an equivalent maintainer-approved patch.
