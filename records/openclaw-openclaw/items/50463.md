## Summary

Keep open: the UX improvement is useful and still missing on current main, but this branch needs rework before merge because it targets stale Pi APIs and does not hook the current missing-tool path safely.

## What This Changes

The PR adds an embedded-runner hook and error helper to rewrite unknown-tool results into a SKILL.md hint when the missing tool name matches a registered skill.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary, using the skills visible to the current run and preserving tool-name collision behavior.

## Reproduction Assessment

Yes, source-based. Current main still returns generic missing-tool text and has no SKILL.md hint path, while the current Pi dependency shows missing tools are immediate prepare results.

## Solution Assessment

No. The proposed UX direction is reasonable, but the branch should be rebased to the current @earendil API and moved to the current missing-tool/run-visible-skill boundary instead of layering on the stale afterToolCall setter path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:3-8`
  - body: Current main depends on @earendil-works/pi-agent-core 0.74.0, where Agent exposes afterToolCall as an option/property rather than the old @mariozechner imports plus setAfterToolCall. As written, this branch will not compile or compose after rebasing onto current main.
  - confidence: 0.96
- **[P2] Route immediate missing-tool results through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1937`
  - body: The current Pi loop returns `Tool X not found` as an immediate prepare result and only calls afterToolCall for prepared tool executions. Installing only an afterToolCall hook leaves the normal skill-called-as-tool path with the generic error.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: The runner removes skills for sandboxed non-read-write runs before building the prompt/runtime state, but this hook checks `params.skillsSnapshot`. A model that guesses a hidden skill name could get confirmation that the skill exists even though it was not visible in the run.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved skills:** `src/agents/pi-embedded-helpers/errors.ts:573-575`
  - body: `resolvedSkills` is optional, and current main reloads skill entries when it is absent. Returning false when the lightweight snapshot still names the skill can suppress the hint for a skill that the run prompt/runtime rebuilt from entries.
  - confidence: 0.84
- **[P3] Parse quoted skill names completely:** `src/agents/pi-embedded-helpers/errors.ts:558-562`
  - body: The parser captures only `[a-z0-9_-]+`, but skill names can contain spaces or punctuation. A valid error such as `Tool "GitHub Actions" not found` would be truncated before matching the registered skill.
  - confidence: 0.8
