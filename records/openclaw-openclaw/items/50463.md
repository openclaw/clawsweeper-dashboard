## Summary

Keep open: the requested behavior is still useful and current main still only has generic unknown-tool handling, but this branch is not correct against current main because it uses stale Pi package APIs and misses the current missing-tool path and active skill-visibility boundary.

## What This Changes

The PR adds a helper and runner hook intended to rewrite unknown-tool errors into a SKILL.md hint when the unknown name matches a registered skill.

## Best Possible Solution

Land the skill hint at the current OpenClaw-owned missing-tool result boundary using @earendil-works contracts, active-run visible skills, robust quoted-name parsing, and focused regression coverage.

## Reproduction Assessment

Yes, source-based. Current main still formats unknown-tool errors generically, and the pinned @earendil-works package shows the missing-tool result path that needs interception; I did not run a live current-main agent.

## Solution Assessment

No. The feature direction is good, but this PR wires it through stale hook APIs and the wrong visibility source; the narrower maintainable fix is to patch the current missing-tool boundary with active-run skills.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent hook surface:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main uses @earendil-works/pi-agent-core@0.74.0, whose Agent exposes afterToolCall as an option/property and has no public setAfterToolCall API. Importing @mariozechner hook types and calling setAfterToolCall/private _afterToolCall will not compose after rebase.
  - confidence: 0.96
- **[P2] Route immediate missing tools through the rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1941`
  - body: In the currently pinned @earendil-works agent loop, a missing tool returns an immediate Tool X not found result and skips finalizeExecutedToolCall/afterToolCall. Installing only an afterToolCall hook leaves the common skill-called-as-tool path generic.
  - confidence: 0.94
- **[P2] Match against the active run's visible skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: The runner can clear or narrow the skill snapshot before building the active prompt, but this hook checks params.skillsSnapshot. A guessed hidden or sandbox-filtered skill name could receive a SKILL.md hint even when that skill was not exposed to the run.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved runtime skills:** `src/agents/pi-embedded-helpers/errors.ts:573-577`
  - body: resolvedSkills is optional, and current main reloads workspace skills when it is absent. Requiring resolvedSkills here suppresses the hint for callers that still advertise skills through the snapshot's required skills array or through reloaded runtime entries.
  - confidence: 0.84
- **[P3] Parse the full quoted skill name:** `src/agents/pi-embedded-helpers/errors.ts:557-560`
  - body: The parser only captures [a-z0-9_-]+, but prompt-facing skill names can include spaces, punctuation, or a leading $. Those valid names are truncated or ignored, so the hint silently fails for common skill spellings.
  - confidence: 0.8
