## Summary

Keep this PR open, but the current patch is not ready to merge. The requested skill hint is still useful and not implemented on current main, but this branch wires it through stale/private Pi hook APIs and can cross the run-visible skill boundary.

## What This Changes

The PR extracts unknown-tool parsing and adds an embedded-runner `afterToolCall` rewrite so missing-tool errors matching registered skills return SKILL.md guidance.

## Best Possible Solution

Implement the hint at the current missing-tool result boundary or an approved OpenClaw-owned middleware seam, using only run-visible skill names and covering punctuation-bearing skill names with focused tests.

## Reproduction Assessment

Yes, source-based. Current main has no `SKILL.md first` rewrite, and the inspected PR screenshot demonstrates the intended after-fix text plus a non-skill control case.

## Solution Assessment

No. The direction is useful, but this patch is not the best implementation because it depends on stale/private Pi APIs and uses the full skill snapshot instead of the run-visible skill set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route the hint through the current missing-tool boundary:** `src/agents/pi-embedded-runner/run/attempt.ts:1940`
  - body: Current `@earendil-works/pi-agent-core@0.74.0` creates missing-tool results in `prepareToolCall` as an immediate outcome, and `afterToolCall` is only applied in `finalizeExecutedToolCall` for prepared tools. Since this patch only rewrites from `afterToolCall`, the main skill-called-as-tool case still returns the generic missing-tool message after rebase.
  - confidence: 0.92
- **[P2] Use the current Pi hook surface:** `src/agents/pi-embedded-runner/run/attempt.ts:1937-1940`
  - body: The branch calls `activeSession.agent.setAfterToolCall` and reads `_afterToolCall`, but the pinned current `Agent` type exposes a public `afterToolCall` property and no `setAfterToolCall` method. This will not compose or typecheck correctly against current main's dependency surface.
  - confidence: 0.9
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: Sandboxed non-rw runs intentionally clear `skillsSnapshotForRun` before exposing skills, but the rewrite checks `params.skillsSnapshot`. A guessed hidden skill name can therefore be confirmed by the hint even though that skill was not visible to the model for this run.
  - confidence: 0.88
- **[P3] Parse complete prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:558-562`
  - body: The helper captures only `[a-z0-9_-]+`, while OpenClaw renders raw skill names in the prompt. Valid names containing `$`, spaces, or other punctuation are truncated or missed, so the hint will fail for real visible skills with those names.
  - confidence: 0.82
