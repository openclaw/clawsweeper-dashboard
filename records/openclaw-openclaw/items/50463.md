## Summary

Keep open: this is a plausible user-facing improvement, but the proposed hook misses the source-level missing-tool path in the currently pinned agent-core dependency, so the stated behavior is not reliably fixed despite sufficient screenshot proof.

## What This Changes

The PR adds an agent error-rewrite helper and `afterToolCall` hook to turn unknown-tool errors into SKILL.md hints when the missing name matches a registered skill.

## Best Possible Solution

Route the rewrite through the actual immediate missing-tool result path, or first land a verified agent-core contract where `afterToolCall` covers those results, and use the same effective skill snapshot that built the run prompt.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main pins `@earendil-works/pi-agent-core@0.74.0`, whose missing-tool immediate result bypasses `afterToolCall`; I did not run a live model session.

## Solution Assessment

No. Installing only an `afterToolCall` rewrite is not the best fix while the pinned dependency bypasses that hook for missing tools; the rewrite needs to happen on the missing-tool path or after a verified dependency contract change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Route missing-tool results through a path that actually runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1937`
  - body: The hook is installed through `afterToolCall`, but the pinned `@earendil-works/pi-agent-core@0.74.0` returns missing tools as immediate `prepareToolCall` results and only invokes `afterToolCall` from `finalizeExecutedToolCall` for prepared calls. A model calling a skill name as a missing tool will still get the generic `Tool X not found` result; move the rewrite onto that immediate path or use a verified dependency contract that invokes the hook for it.
  - confidence: 0.93
- **[P2] Use the effective visible skill snapshot:** `src/agents/pi-embedded-runner/run/attempt.ts:1953`
  - body: The runner builds the prompt from `skillsSnapshotForRun`, which can be `undefined` when sandbox settings hide workspace skills, but the hook passes `params.skillsSnapshot`. That can tell the model a hidden skill is available; pass the same effective run-visible snapshot used for the prompt.
  - confidence: 0.86
