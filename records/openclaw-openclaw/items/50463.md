## Summary

Keep open: the feature is useful and not implemented on current main, but this branch is not mergeable as-is because it targets stale Pi imports/hooks, misses current immediate unknown-tool results, and can match skills outside the run-visible prompt.

## What This Changes

The PR adds helper logic and an embedded-runner afterToolCall hook to rewrite unknown-tool errors as SKILL.md hints when the missing name matches a registered skill.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary using @earendil contracts, run-visible hydrated skills, robust quoted-name parsing, and focused regression tests.

## Reproduction Assessment

Yes, source-based. Current main still returns generic missing-tool handling with no SKILL.md hint, and the inspected PR screenshots show the intended after-fix behavior in a live agent session.

## Solution Assessment

No. The direction is useful, but this branch is not the best implementation because it targets stale Pi APIs and wires the rewrite through a hook that does not cover current main's immediate missing-tool path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-8`
  - body: Current main imports @earendil-works/pi-agent-core, where Agent exposes afterToolCall as an option/property. This branch adds @mariozechner types and calls setAfterToolCall/private _afterToolCall, so it will not compile or compose correctly after rebase.
  - confidence: 0.96
- **[P2] Route missing-tool results through the hint rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1939-1940`
  - body: In @earendil-works/pi-agent-core 0.74.0, missing tools are returned as immediate prepareToolCall results and afterToolCall only runs for executed prepared tools. Installing only this hook leaves the normal Tool X not found path generic.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1955-1957`
  - body: The runner clears or narrows the skill snapshot before constructing the active run prompt, but the patch checks params.skillsSnapshot. A guessed hidden skill name could still receive a SKILL.md hint even when it was removed from the actual run context.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved runtime skills:** `src/agents/pi-embedded-helpers/errors.ts:573-577`
  - body: resolvedSkills is optional and stripped from persisted sessions, while the runner reloads skill entries when it is absent. Requiring resolvedSkills here suppresses the hint for resumed sessions that still advertise the skill after hydration.
  - confidence: 0.84
- **[P3] Parse the full quoted tool name:** `src/agents/pi-embedded-helpers/errors.ts:560-561`
  - body: The parser captures only [a-z0-9_-]+, but prompt-facing skill names can contain spaces, punctuation, or path-like separators. Those names are truncated or ignored, so the hint will not fire for valid skill spellings.
  - confidence: 0.8
