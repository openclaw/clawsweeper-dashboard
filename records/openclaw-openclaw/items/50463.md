## Summary

Keep open: the feature is useful and is not implemented on current main, but this patch is not merge-ready because the current pi-agent-core dependency does not call `afterToolCall` for immediate missing-tool results, so the proposed hook misses the main user path.

## What This Changes

The PR adds a helper and agent hook intended to rewrite unknown-tool errors that match registered skills into a `SKILL.md` hint.

## Best Possible Solution

Keep the feature direction, but wire it through a path that actually sees current pi-agent-core immediate missing-tool results, preserve existing afterToolCall handlers, and add a focused regression test for a visible skill name called as a tool.

## Reproduction Assessment

Yes, source-level. With a visible skill name absent from the actual tool set, `@earendil-works/pi-agent-core@0.75.4` returns an immediate `Tool X not found` result before `afterToolCall`, so the PR hook does not reach the target path.

## Solution Assessment

No. The maintainable fix is to rewrite immediate missing-tool results at the source path, or first update the dependency contract so those results are passed through an official hook.
