## Summary

Keep this PR open: the feature direction is useful and current main still lacks the SKILL.md hint, but the branch is not correct against the current agent/runtime contracts and needs maintainer handling before it can land.

## What This Changes

The PR adds an unknown-tool error rewrite that hints when the missing tool name matches a registered skill, plus a runner hook to apply that rewrite after tool calls.

## Best Possible Solution

Land the hint only at a current @earendil-works-compatible missing-tool boundary, using active-run visible skills, robust quoted-name parsing, and focused regression coverage.

## Reproduction Assessment

Yes, source-based. Current main still returns generic unknown-tool handling, and the current @earendil-works dependency creates generic immediate Tool X not found results; I did not run a live agent in this read-only review.

## Solution Assessment

No. The feature direction is good, but this branch uses stale package APIs and the wrong skill visibility source; the safer solution is to patch the current missing-tool path with active-run skills.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent package API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main pins and imports @earendil-works/pi-agent-core, whose Agent exposes afterToolCall as a property and does not expose setAfterToolCall. Reintroducing @mariozechner imports and calling setAfterToolCall will not compose after rebase.
  - confidence: 0.96
- **[P2] Route immediate missing-tool results through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1941`
  - body: The current @earendil-works loop returns missing tools as immediate Tool X not found results, and afterToolCall only runs for prepared tool calls. Installing only an afterToolCall rewrite leaves the common skill-called-as-tool path generic.
  - confidence: 0.94
- **[P2] Match only active visible skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1956`
  - body: The runner can clear or narrow skillsSnapshotForRun before building the prompt, but this hook checks params.skillsSnapshot. A hidden or sandbox-filtered skill name can therefore receive a SKILL.md hint even though it was not exposed to the run.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved runtime skills:** `src/agents/pi-embedded-helpers/errors.ts:575-578`
  - body: resolvedSkills is optional, and current main reloads workspace skills when it is absent. Requiring resolvedSkills here suppresses the hint for callers that still have prompt-facing skills or reloaded runtime entries.
  - confidence: 0.84
- **[P3] Parse the full quoted skill name:** `src/agents/pi-embedded-helpers/errors.ts:561-563`
  - body: The parser only captures [a-z0-9_-]+, but prompt-facing skill names can include spaces, punctuation, or a leading $. Those valid names are truncated or ignored, so the hint silently fails for common skill spellings.
  - confidence: 0.8
