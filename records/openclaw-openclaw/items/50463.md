  - confidence: 0.93
- **[P1] Route immediate missing-tool results through the rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1941`
  - body: The current @earendil-works agent loop returns missing tools as `kind: "immediate"` from `prepareToolCall` and emits them without running `afterToolCall`. Hooking only `afterToolCall` leaves the main `Tool X not found` path generic, so the feature does not reliably fire for the case it is trying to fix.
  - confidence: 0.9
- **[P2] Constrain the hint to skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1955`
  - body: The hook passes `params.skillsSnapshot` even though current main strips the skills prompt when `toolsAllow` is set. That can confirm a guessed skill name that was not visible in this run, so the rewrite should use the active visible skill set instead.
  - confidence: 0.82
- **[P2] Parse full quoted skill names:** `src/agents/pi-embedded-helpers/errors.ts:555-558`
  - body: `resolveUnknownToolName()` only captures `[a-z0-9_-]+`, but skill names can be rendered with spaces or punctuation. A valid error such as `Tool "GitHub Actions" not found` would capture only `GitHub` and skip the hint.
  - confidence: 0.8
