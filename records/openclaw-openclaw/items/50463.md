## Summary

Keep open: the requested hint is useful and still missing from current main, but this branch is not mergeable as-is because it targets stale Pi APIs, misses the current unknown-tool path, and can confirm skills that are hidden for the run.

## What This Changes

The PR adds an embedded-runner unknown-tool rewrite that turns missing-tool errors matching registered skills into SKILL.md guidance and refactors unknown-tool parsing.

## Best Possible Solution

Implement the hint at the current missing-tool/result boundary using only run-visible skills, while preserving tool collisions and covering nested or punctuation-bearing skill names.

## Reproduction Assessment

Yes, source-based. Current main still has generic unknown-tool handling, and the PR screenshot shows the intended after-fix hint plus a non-skill control, though I did not run a live agent during this read-only review.

## Solution Assessment

No. The direction is good, but this patch uses a stale/private Pi hook surface, misses the current missing-tool path, and checks the full skill snapshot instead of the run-visible skill set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the current Pi package and hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main pins @earendil-works/pi-agent-core@0.74.0, but this branch imports @mariozechner/pi-agent-core types and calls setAfterToolCall. The current Agent exposes an afterToolCall property and no setter, so the branch will not compose or typecheck after rebasing onto main.
  - confidence: 0.94
- **[P2] Route missing-tool results through the skill-hint rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1940`
  - body: In the pinned Pi loop, an unknown tool returns an immediate Tool not found result from prepareToolCall, and afterToolCall only runs for prepared/executed tools. Installing this rewrite only as an afterToolCall hook leaves the main skill-called-as-tool case unchanged.
  - confidence: 0.92
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1956`
  - body: Current main clears the run-visible skill snapshot for sandboxed non-rw runs, but this rewrite checks params.skillsSnapshot. A model that guesses a hidden skill name could get confirmation that it exists even though it was not visible in the run.
  - confidence: 0.88
- **[P3] Parse complete prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:558-561`
  - body: The new helper captures only [a-z0-9_-]+, while skill names rendered to the prompt can include nested names like name/subskill or other punctuation. Those valid visible skills would be truncated or missed, so the hint silently fails for part of the supported skill surface.
  - confidence: 0.8
