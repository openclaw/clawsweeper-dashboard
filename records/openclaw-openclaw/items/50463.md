## Summary

Keep open: current main still lacks the requested skill hint, but this branch is not correct against the current runner and dependency contract and can disclose hidden skills in sandboxed runs.

## What This Changes

The PR extracts unknown-tool parsing and adds an embedded-runner `afterToolCall` hook that rewrites matching missing-tool results with a `SKILL.md` hint.

## Best Possible Solution

Implement the hint at the current missing-tool result boundary, preserve existing hook composition, use only the run-visible skill set, and cover skill match, non-skill, hidden skill, punctuation, and tool-collision cases.

## Reproduction Assessment

Yes, source-based. Current main has no skill-specific `SKILL.md` hint, and the pinned agent-loop source creates a generic missing-tool result for unknown tool calls.

## Solution Assessment

No. The UX direction is reasonable, but this PR wires the rewrite through the wrong current-main boundary, uses hidden persisted skills instead of the run-visible set, and needs parser and hook-composition fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Route skill hints through the missing-tool boundary:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1938`
  - body: The pinned agent loop returns unknown tools as immediate `Tool X not found` results from `prepareToolCall`, and those results do not enter `finalizeExecutedToolCall` where `afterToolCall` runs. Because this PR only installs an `afterToolCall` rewrite, the path it is meant to fix still receives the generic missing-tool result.
  - confidence: 0.9
- **[P2] Compose with the public `afterToolCall` hook:** `src/agents/pi-embedded-runner/run/attempt.ts:1936-1938`
  - body: Current `Agent` exposes `afterToolCall` as a public mutable property, and the loop config reads that property. Reading a private `_afterToolCall` field and calling `setAfterToolCall` does not match the current dependency surface after rebase and can bypass the existing extension handler.
  - confidence: 0.86
- **[P2] Use the run-visible skills snapshot:** `src/agents/pi-embedded-runner/run/attempt.ts:1954`
  - body: Sandboxed non-rw runs intentionally clear `skillsSnapshotForRun` before exposing skills to the model, but this hook checks `params.skillsSnapshot`. A guessed hidden skill name can therefore be confirmed by the new hint even when that skill was not visible in the run.
  - confidence: 0.88
- **[P3] Parse the full prompt-facing skill name:** `src/agents/pi-embedded-helpers/errors.ts:555-559`
  - body: The helper only captures `[a-z0-9_-]+`, while OpenClaw renders raw skill names in the available-skills prompt. Valid skill names containing spaces or punctuation will be truncated or missed before matching, so they will not get the intended hint.
  - confidence: 0.8
