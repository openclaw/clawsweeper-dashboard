## What This Changes

The PR rewrites unknown-tool error results into SKILL.md guidance when the missing tool name matches a registered skill and wires that rewrite from the embedded runner.

## Best Possible Solution

Land the hint at the current OpenClaw-owned missing-tool result boundary using @earendil-works contracts, active-run visible skills, robust quoted-name parsing, and focused regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: current source still has a generic unknown-tool path with no SKILL.md hint, and the PR body includes live after-fix output, but I did not run a current-main live agent session.

## Solution Assessment

Is this the best way to solve the issue? No. The feature direction is useful, but the branch should be moved to the current @earendil-works missing-tool boundary and should filter against the active run's visible skills.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main uses `@earendil-works/pi-agent-core` and exposes `afterToolCall` as an Agent option/property, but this PR imports hook types from `@mariozechner/pi-agent-core` and calls `setAfterToolCall` on the agent. That stale API will not compose with the runner after rebase, so the change needs to target the pinned package contract.
  - confidence: 0.96
- **[P2] Rewrite prepare-time missing tools:** `src/agents/pi-embedded-runner/run/attempt.ts:1937-1941`
  - body: In the pinned `@earendil-works/pi-agent-core@0.75.1` loop, a missing tool returns an immediate `Tool X not found` result from `prepareToolCall`, and immediate results are emitted without `finalizeExecutedToolCall` invoking `afterToolCall`. Installing only an after-tool hook can leave the main skill-called-as-tool path unchanged.
  - confidence: 0.94
- **[P2] Match only active visible skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1956`
  - body: The runner can clear or narrow `skillsSnapshotForRun` before building the active prompt, but this hook checks `params.skillsSnapshot`. That can disclose hidden or sandbox-filtered skills and can also diverge from the actual skills shown to this run.
  - confidence: 0.9
- **[P3] Parse full quoted skill names:** `src/agents/pi-embedded-helpers/errors.ts:556-559`
  - body: `resolveUnknownToolName()` only captures `[a-z0-9_-]+`, while skill names rendered in the prompt are not limited to that character set. Quoted names with spaces, punctuation, or a leading `$` can be truncated or ignored, so the hint silently fails for common skill spellings.
  - confidence: 0.82
