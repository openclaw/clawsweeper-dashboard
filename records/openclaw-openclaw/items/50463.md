## Summary

Keep open: the goal is useful and current main does not provide this SKILL.md hint, but the PR should not merge as-is because the hook is on a dependency path that does not handle immediate missing-tool results and it uses the raw skill snapshot instead of the effective run-visible skill set.

## What This Changes

The PR adds helper logic and an agent `afterToolCall` hook to rewrite unknown-tool tool-result errors into SKILL.md hints when the missing name matches a registered skill.

## Best Possible Solution

Route the rewrite through the actual missing-tool immediate-result path, or first land a verified dependency contract where `afterToolCall` covers those results, then match against the same effective skill set used to build the run prompt.

## Reproduction Assessment

Yes. Source inspection against current main and `@earendil-works/pi-agent-core@0.74.0` shows missing tools are emitted as immediate prepare results with generic `Tool X not found` text, while current main has no SKILL.md hint path.

## Solution Assessment

No. The proposed hook-only solution is not the best fix because the pinned dependency skips `afterToolCall` for immediate missing-tool results; the rewrite should be placed on a supported path that actually sees the target result and should use the run-visible skill source.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route missing-tool results through a path that runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1936`
  - body: The hook is installed via `afterToolCall`, but the pinned `@earendil-works/pi-agent-core@0.74.0` returns missing tools as immediate `prepareToolCall` results and skips `finalizeExecutedToolCall`, where `afterToolCall` is invoked. A model calling a skill name as an unavailable tool still gets the generic `Tool X not found` result unless this rewrite moves to that immediate path or lands with a verified dependency contract change.
  - confidence: 0.93
- **[P2] Use the effective visible skill set for hints:** `src/agents/pi-embedded-runner/run/attempt.ts:1955`
  - body: The run can intentionally set `skillsSnapshotForRun` to `undefined` for sandboxed non-rw workspaces, and that effective value is what builds the skills prompt. Passing `params.skillsSnapshot` into the hook can tell the model that a hidden skill is available once the hook path is fixed; use the same effective run-visible skill data and cover missing `resolvedSkills` cases.
  - confidence: 0.86
