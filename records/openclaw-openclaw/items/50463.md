## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-8`
  - body: Current main uses @earendil-works/pi-agent-core 0.74.0, where Agent exposes `afterToolCall` as an option/property. This branch imports @mariozechner types and calls `setAfterToolCall` plus private `_afterToolCall`, so it will not compile or compose after rebase.
  - confidence: 0.96
- **[P2] Apply the hint where missing tools are produced:** `src/agents/pi-embedded-runner/run/attempt.ts:1925`
  - body: In the current Pi loop, unknown tools are returned as immediate prepare results and do not pass through `finalizeExecutedToolCall`, where `afterToolCall` runs. Installing only this hook leaves the normal `Tool X not found` path generic.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: The runner can clear or filter skills before building the run prompt/runtime state, but the patch checks `params.skillsSnapshot`. A model that guesses a hidden skill name could still get a SKILL.md hint even when that skill was removed from the actual run context.
  - confidence: 0.9
- **[P2] Handle snapshots without runtime-resolved skills:** `src/agents/pi-embedded-helpers/errors.ts:571-574`
  - body: `resolvedSkills` is optional while the lightweight `skills` list is required, and the runner reloads runtime entries when resolved skills are absent. Returning false in that case can suppress the hint for sessions that still advertise the skill.
  - confidence: 0.84
- **[P3] Parse the full quoted tool name:** `src/agents/pi-embedded-helpers/errors.ts:557-558`
  - body: The parser captures only `[a-z0-9_-]+`, but prompt-facing skill names can include `$`, spaces, punctuation, or path-like separators. Those valid names are truncated or ignored, so the new hint will not fire for common skill spellings.
  - confidence: 0.8
