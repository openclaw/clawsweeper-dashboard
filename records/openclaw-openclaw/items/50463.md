## Summary

Keep open: current main still lacks the requested skill hint, but this branch is not mergeable as-is because it hooks the wrong runtime boundary and can reveal hidden skill names in sandboxed runs.

## What This Changes

The PR extracts unknown-tool parsing and adds an embedded-runner hook that rewrites missing-tool results matching registered skills into SKILL.md guidance.

## Best Possible Solution

Implement the hint at the current missing-tool result boundary, compose with existing tool-result middleware, match only run-visible skills using complete prompt-facing names, and add focused helper/runner coverage.

## Reproduction Assessment

Yes, source-based. Current main has no `SKILL.md first` rewrite, and the PR screenshot shows the intended after-fix behavior for skill-match and non-skill cases.

## Solution Assessment

No. The direction is useful, but this patch should move to the current missing-tool boundary, use the current public hook surface where applicable, and match against the run-visible skill snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route skill hints through missing-tool preparation:** `src/agents/pi-embedded-runner/run/attempt.ts:1937`
  - body: In the current `@earendil-works/pi-agent-core@0.74.0`, missing tools are returned as immediate results from preparation and are emitted without `finalizeExecutedToolCall`, where `afterToolCall` runs. Because this patch rewrites only from `afterToolCall`, the main skill-as-unknown-tool case can still return the generic error on current main.
  - confidence: 0.91
- **[P2] Use the current afterToolCall API:** `src/agents/pi-embedded-runner/run/attempt.ts:1937`
  - body: Current `Agent` exposes `afterToolCall` as a public property and has no public `setAfterToolCall` or private `_afterToolCall` contract to compose with. Rework this through the current public surface so the existing extension `tool_result` handler is preserved after rebase.
  - confidence: 0.9
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1955`
  - body: Sandboxed non-rw runs clear `skillsSnapshotForRun` before exposing skills, but this hook passes the full `params.skillsSnapshot`. If the rewrite fires, a guessed hidden skill name can be confirmed even though it was not visible in that run.
  - confidence: 0.88
- **[P3] Parse complete prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:556-559`
  - body: `resolveUnknownToolName()` only captures `[a-z0-9_-]+`, while OpenClaw renders raw skill names and allows names with symbols or spaces. Valid names like `$openclaw-pr-maintainer` or `GitHub Actions` will be truncated or missed.
  - confidence: 0.82
