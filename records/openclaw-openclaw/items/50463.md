## Summary

Keep open: the user-visible hint is useful and current main still lacks it, but this branch is not merge-ready against current main because it targets stale Pi APIs and misses the current missing-tool result boundary.

## What This Changes

The PR adds an agent helper and hook that rewrites unknown-tool errors into SKILL.md hints when the unknown name matches a registered skill, while preserving generic errors for non-skills and tool-name collisions.

## Best Possible Solution

Implement the hint on current main at the OpenClaw-owned missing-tool result boundary using current @earendil contracts, run-visible hydrated skills, robust quoted-name parsing, collision preservation, and focused regression tests.

## Reproduction Assessment

Yes, source-based: current main still returns generic missing-tool handling with no SKILL.md hint, and the PR screenshot shows the intended after-fix behavior in a live agent session.

## Solution Assessment

No. The direction is useful, but this branch is not the best implementation because it targets stale Pi APIs and wires the rewrite through a hook that does not cover current main's immediate missing-tool results.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-8`
  - body: Current main uses @earendil-works/pi-agent-core 0.74.0, where Agent exposes afterToolCall as an option/property. This branch imports @mariozechner types and calls setAfterToolCall plus private _afterToolCall, so it will not compile or compose after rebase.
  - confidence: 0.96
- **[P2] Route missing-tool results through the hint rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1937`
  - body: In the current @earendil loop, unknown tools are returned as immediate prepareToolCall results and afterToolCall only runs for prepared/executed tools. Installing only this hook leaves the normal Tool X not found path generic.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1956-1957`
  - body: The runner can clear or rebuild the skill snapshot before constructing the active run prompt, but the patch checks params.skillsSnapshot. A guessed hidden skill name could still receive a SKILL.md hint even when it was removed from the actual run context.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved runtime skills:** `src/agents/pi-embedded-helpers/errors.ts:573-577`
  - body: resolvedSkills is optional and stripped from persisted sessions, while the runner reloads skill entries when it is absent. Requiring resolvedSkills here suppresses the hint for resumed sessions that still advertise the skill after hydration.
  - confidence: 0.84
- **[P3] Parse the full quoted tool name:** `src/agents/pi-embedded-helpers/errors.ts:560-561`
  - body: The parser captures only [a-z0-9_-]+, but prompt-facing skill names can include prefixes, spaces, punctuation, or path-like separators. Those names are truncated or ignored, so the hint will not fire for valid skill spellings.
  - confidence: 0.8
