## Summary

Keep this PR open, but do not merge the current diff. The feature direction is useful and the screenshot proof is adequate, but the branch is stale against current main's Pi dependency surface, misses the current missing-tool path, and can disclose skills that were intentionally hidden for the run.

## What This Changes

The branch extracts unknown-tool parsing and adds an embedded-runner `afterToolCall` rewrite that turns missing-tool errors matching registered skills into SKILL.md guidance.

## Best Possible Solution

Implement the hint at an approved current missing-tool/result boundary using only run-visible skills, with focused coverage for hidden skills, tool/skill collisions, and punctuation-bearing skill names.

## Reproduction Assessment

Yes, source-based. Current main lacks the `SKILL.md first` hint, the PR screenshot demonstrates the intended after-fix text plus a non-skill control, and the blocking regressions are visible from current main and the pinned dependency source.

## Solution Assessment

No. The direction is useful, but this patch is not the best implementation because it depends on a stale/private Pi hook surface and uses the full skill snapshot instead of the run-visible skill set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route the hint through the missing-tool boundary:** `src/agents/pi-embedded-runner/run/attempt.ts:1940`
  - body: Current `@earendil-works/pi-agent-core@0.74.0` returns missing tools as `prepareToolCall` immediate results, while `afterToolCall` only runs from `finalizeExecutedToolCall` for prepared tools. Since this rewrite is installed only through `afterToolCall`, the main skill-called-as-tool case still returns the generic missing-tool text after rebase.
  - confidence: 0.92
- **[P2] Use the current public Pi hook surface:** `src/agents/pi-embedded-runner/run/attempt.ts:1937-1940`
  - body: The branch calls `activeSession.agent.setAfterToolCall` and reads `_afterToolCall`, but current main's pinned `Agent` exposes a public `afterToolCall` property and no `setAfterToolCall` method. This will not compose or typecheck correctly against the current dependency surface.
  - confidence: 0.9
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: Sandboxed non-rw runs intentionally clear `skillsSnapshotForRun` before exposing skills, but the rewrite checks `params.skillsSnapshot`. A guessed hidden skill name can therefore be confirmed by the hint even though that skill was not visible to the model for this run.
  - confidence: 0.88
- **[P3] Parse complete prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:558-562`
  - body: The helper captures only `[a-z0-9_-]+`, while OpenClaw renders raw skill names in the prompt. Valid visible skill names containing `$`, spaces, or other punctuation are truncated or missed, so the hint fails for real skills with those names.
  - confidence: 0.82
