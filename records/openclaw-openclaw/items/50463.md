## What This Changes

The PR adds a skill-aware unknown-tool error rewrite in the embedded agent runner and refactors unknown-tool name parsing in assistant error helpers.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary using @earendil-works contracts, run-visible skills, collision preservation, robust quoted-name parsing, and focused regression coverage.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-8`
  - body: Current main uses @earendil-works/pi-agent-core 0.74.0, where Agent exposes `afterToolCall` as an option/property; the older @mariozechner import plus `setAfterToolCall`/private `_afterToolCall` access will not compile or compose after rebase.
  - confidence: 0.96
- **[P2] Apply the hint where missing tools are produced:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1949`
  - body: In the current Pi loop, unknown tools are returned as immediate prepare results and do not go through `finalizeExecutedToolCall`, where `afterToolCall` runs. Installing only this hook leaves the normal `Tool X not found` path generic.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: The runner can clear or filter skills before building the run prompt/runtime state, but this patch checks `params.skillsSnapshot`. That can confirm a guessed hidden skill name in runs where the skill was removed from the actual visible context.
  - confidence: 0.9
- **[P2] Handle snapshots without runtime-resolved skills:** `src/agents/pi-embedded-helpers/errors.ts:571-574`
  - body: `resolvedSkills` is optional while `skills` is required, and persisted snapshots may need hydration. Returning false when `resolvedSkills` is absent suppresses the hint even when the prompt/catalog still advertises the skill.
  - confidence: 0.84
- **[P3] Parse the full quoted tool name:** `src/agents/pi-embedded-helpers/errors.ts:557-558`
  - body: The parser captures only `[a-z0-9_-]+`, so names such as `$openclaw-pr-maintainer` or `GitHub Actions` are truncated or ignored. Those valid prompt-facing skill names will not receive the hint.
  - confidence: 0.8
