## Summary

Keep this PR open, but it still needs changes before merge: the feature direction is useful, while the patch is stale against the current Pi dependency surface and can miss the main missing-tool path or disclose hidden skills.

## What This Changes

The PR extracts unknown-tool parsing and adds an embedded-runner `afterToolCall` hook to rewrite missing-tool results that match registered skills into `SKILL.md` guidance.

## Best Possible Solution

Implement the hint at the current missing-tool result boundary, compose with existing extension `tool_result` behavior, match only run-visible skills with complete prompt-facing names, and add focused helper/runner coverage.

## Reproduction Assessment

Yes, source-based. Current main has no `SKILL.md first` rewrite, the pinned dependency emits missing tools as immediate generic results, and the PR screenshot demonstrates the intended after-fix behavior.

## Solution Assessment

No. The direction is useful, but this patch should move to the current missing-tool boundary, use the current public hook surface where applicable, and match against `skillsSnapshotForRun`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route skill hints through missing-tool preparation:** `src/agents/pi-embedded-runner/run/attempt.ts:1941`
  - body: In the current `@earendil-works/pi-agent-core@0.74.0`, missing tools are returned as `kind: "immediate"` from preparation and emitted without `finalizeExecutedToolCall` or `afterToolCall`. Because this patch rewrites only from `afterToolCall`, the main skill-as-unknown-tool case still returns the generic error.
  - confidence: 0.91
- **[P2] Use the public afterToolCall property:** `src/agents/pi-embedded-runner/run/attempt.ts:1941`
  - body: Current `Agent` exposes `afterToolCall` as a public property and has no public `setAfterToolCall`/private `_afterToolCall` contract to compose with. This needs to preserve the existing `AgentSession` `tool_result` handler through the current public surface.
  - confidence: 0.9
- **[P2] Match only run-visible skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1955`
  - body: Sandboxed non-rw runs clear `skillsSnapshotForRun` before exposing skills, but this passes `params.skillsSnapshot`. If the rewrite runs, a guessed hidden skill name can be confirmed even though it was not visible in that run.
  - confidence: 0.88
- **[P3] Parse complete prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:555-559`
  - body: `resolveUnknownToolName()` only captures `[a-z0-9_-]+`, while OpenClaw renders raw skill names in `<available_skills>` and supports names with symbols or spaces. Valid names such as `$openclaw-pr-maintainer` or `GitHub Actions` will be truncated or missed.
  - confidence: 0.82
