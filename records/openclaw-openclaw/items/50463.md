## Summary

Keep open: the requested skill-specific unknown-tool hint is useful and the contributor provided after-fix behavior proof, but the current patch is stale against the current agent-core API and has a concrete hidden-skill disclosure bug.

## What This Changes

The PR extracts unknown-tool parsing and adds an embedded-runner afterToolCall hook that rewrites missing-tool results matching registered skills into SKILL.md guidance.

## Best Possible Solution

Implement the hint at the current missing-tool result boundary, compose with current public hook or middleware surfaces, match only run-visible skill names, and add focused regression coverage.

## Reproduction Assessment

Yes, source-based. Current main still lacks the SKILL.md-first rewrite, and the PR screenshot shows the intended after-fix behavior for skill-match and non-skill cases.

## Solution Assessment

No. The direction is useful, but the current patch should move off the stale private hook path, use the current dependency contract, and match against the run-visible skill snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route the hint through the current missing-tool path:** `src/agents/pi-embedded-runner/run/attempt.ts:1937-1941`
  - body: In @earendil-works/pi-agent-core@0.74.0, missing tools return an immediate result from prepareToolCall and do not run afterToolCall. Because this patch only rewrites from afterToolCall, the main skill-called-as-tool case still returns the generic missing-tool error on current main.
  - confidence: 0.92
- **[P2] Use the current Agent hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:1937-1941`
  - body: Current main uses @earendil-works/pi-agent-core, whose Agent exposes an afterToolCall property and no setAfterToolCall or _afterToolCall contract. This stale hook wiring will not typecheck or preserve the existing hook after rebase.
  - confidence: 0.9
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1955`
  - body: Sandboxed non-rw runs clear skillsSnapshotForRun before exposing skills, but this hook checks the full params.skillsSnapshot. A guessed hidden skill name could therefore be confirmed even though it was not visible in the run.
  - confidence: 0.88
- **[P3] Parse complete prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:556-559`
  - body: resolveUnknownToolName only captures [a-z0-9_-]+, while OpenClaw renders raw skill names and separately sanitizes command names. Valid names with spaces or punctuation will be truncated or missed, so the new hint silently fails for those skills.
  - confidence: 0.82
