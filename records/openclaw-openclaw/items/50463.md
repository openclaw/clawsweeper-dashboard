## Summary

Keep open. The feature direction is useful, but current main does not implement the skill-aware missing-tool hint and this PR is not mergeable as written because it relies on an unsupported Agent hook API and misses the pinned dependency's immediate missing-tool path.

## What This Changes

The PR extracts unknown-tool parsing and adds an embedded-runner hook intended to rewrite missing-tool errors that match visible skills into `SKILL.md` guidance.

## Best Possible Solution

Implement the hint through a supported missing-tool result path that preserves existing tool-result hooks, defines skill/tool collision semantics, handles full skill names, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: a tool call whose name is absent from `currentContext.tools` returns `Tool X not found` from `prepareToolCall` in pinned `pi-agent-core`, and current main has no skill-aware rewrite for that result.

## Solution Assessment

No. This PR is not the best way to solve it because it relies on unsupported Agent internals and an `afterToolCall` path that misses the target missing-tool result; a supported immediate-result rewrite seam or equivalent tool-result transform is safer.
