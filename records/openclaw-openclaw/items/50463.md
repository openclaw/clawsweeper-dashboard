## Summary

Keep open: the feature is useful and current main still lacks the skill-specific hint, but this PR should not merge as-is because it wires the rewrite through a hook that the current missing-tool path does not call and it can expose skills hidden from the run.

## What This Changes

The PR adds a skill-specific unknown-tool error rewrite in the embedded agent runner and extracts a reusable unknown-tool parser in agent error formatting.

## Best Possible Solution

Implement the hint at the active missing-tool result boundary or after an upstream Pi contract update, use the same run-visible skill snapshot that built the prompt, preserve existing hook behavior, and add regression coverage for skill match, non-match, sandbox-hidden skills, and broader skill names.

## Reproduction Assessment

Yes, source-based. Current main has no skill-specific hint, and the pinned Pi dependency returns generic missing-tool immediate results that bypass afterToolCall.

## Solution Assessment

No. The PR's hook placement is not the narrowest maintainable fix for current main; the rewrite needs to run where missing-tool results are actually produced and must use the run-visible skill snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Route missing-tool results through an active rewrite point:** `src/agents/pi-embedded-runner/run/attempt.ts:1941`
  - body: Current main pins @earendil-works/pi-agent-core@0.74.0, where missing tools return a generic immediate result from prepareToolCall and bypass afterToolCall. Because this PR only installs the rewrite through setAfterToolCall, the main `Tool X not found` case can still reach the model unchanged.
  - confidence: 0.91
- **[P2] Use the run-visible skill snapshot for hints:** `src/agents/pi-embedded-runner/run/attempt.ts:1954`
  - body: The runner clears `skillsSnapshotForRun` when a sandboxed run should not expose workspace skills, but this rewrite uses `params.skillsSnapshot`. That can advertise a skill that was intentionally hidden from the model for this run.
  - confidence: 0.85
- **[P3] Match full registered skill names:** `src/agents/pi-embedded-helpers/errors.ts:557-558`
  - body: The parser only captures `[a-z0-9_-]+`, while OpenClaw renders raw skill names and separately sanitizes command names because skills may contain spaces or symbols. A valid skill such as `GitHub Actions` would be truncated before comparison and skip the hint.
  - confidence: 0.78
