## Summary

Keep open: the feature is useful and not implemented on current main, but this branch is stale against the current Pi package/API and misses the current missing-tool and visible-skill boundaries.

## What This Changes

The PR rewrites unknown-tool tool results into a SKILL.md hint when the missing tool name matches a registered skill and wires that rewrite through the embedded Pi runner.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary using `@earendil-works` contracts, run-visible skills, collision preservation, robust quoted-name parsing, and focused regression coverage.

## Reproduction Assessment

Yes, source-based. Current main still has the generic missing-tool behavior, and the pinned Pi loop produces missing tools as immediate `Tool X not found` results that this branch's current hook would not see.

## Solution Assessment

No. The direction is reasonable, but this branch is not the best current implementation because it targets stale Pi APIs and should be reworked around the current missing-tool result and run-visible skill boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi package API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main pins `@earendil-works/pi-agent-core` 0.74.0, but this branch adds imports from the old `@mariozechner` namespace and later calls `setAfterToolCall`, which the current Agent type does not expose. After rebasing, this will not compile or compose against the current dependency contract.
  - confidence: 0.96
- **[P2] Route missing-tool results through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1941`
  - body: The current Pi loop creates missing-tool failures as immediate prepare results and only invokes `afterToolCall` from finalized executed tools. Because this patch only adds an after-tool-call hook, the common `Tool X not found` path this feature targets will still return the generic error.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1956-1957`
  - body: The runner can clear skills for sandboxed non-read-write sessions before building the run state, but the patch checks `params.skillsSnapshot`. A model that guesses a hidden skill name could get a SKILL.md hint, so the rewrite needs to use the run-visible skill set.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved skills:** `src/agents/pi-embedded-helpers/errors.ts:572-578`
  - body: `resolvedSkills` is optional while the lightweight `skills` list is always present, and the runner reloads skill entries when `resolvedSkills` is absent. Returning false in that case can suppress the hint even when the session advertises the skill.
  - confidence: 0.84
- **[P3] Parse quoted skill names completely:** `src/agents/pi-embedded-helpers/errors.ts:557-558`
  - body: The new parser only captures `[a-z0-9_-]+`, but prompt-facing skill names may include a leading `$`, spaces, or punctuation. Valid skill calls such as `Tool "GitHub Actions" not found` would be truncated and miss the hint.
  - confidence: 0.8
