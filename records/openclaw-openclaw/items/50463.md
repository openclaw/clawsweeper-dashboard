## Summary

Keep this PR open. The idea is useful, but the current branch wires the hint through `afterToolCall`, while the currently pinned agent-core package still bypasses that hook for immediate missing-tool results, and the helper also misses some skill names that OpenClaw currently supports.

## What This Changes

The PR extracts unknown-tool parsing, adds a skill-aware rewrite helper, and registers an embedded agent hook to turn matching unknown-tool error results into SKILL.md guidance.

## Best Possible Solution

Route missing-tool immediate results through a supported hook or upstream agent-core change, then match against the actual run-visible skill names, including snapshot-only and non-identifier names, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of the pinned agent-core package shows missing tools are returned as immediate results before `afterToolCall`, and current OpenClaw source shows skill names and snapshots have valid forms the PR matcher does not cover.

## Solution Assessment

No. The current hook is not the best implementation because it misses the dependency's immediate missing-tool path; a supported upstream/core path should perform the rewrite before the tool result is emitted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route missing-tool results through the hint rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1941`
  - body: The hook is installed with `setAfterToolCall`, but the pinned agent-core package returns missing tools as `kind: "immediate"` from `prepareToolCall()` and emits those results without calling `afterToolCall`. That is the exact path for a model calling a skill name as a nonexistent tool, so the new hint is skipped for the primary behavior this PR is trying to fix.
  - confidence: 0.94
- **[P2] Fall back to visible snapshot skill names:** `src/agents/pi-embedded-helpers/errors.ts:575-578`
  - body: `resolvedSkills` is optional runtime-only data, and the runner reloads entries when it is absent. A persisted or lightweight snapshot can still have `prompt`/`skills` visible to the model, but this matcher returns false and suppresses the hint because it ignores `skillsSnapshot.skills`.
  - confidence: 0.9
- **[P2] Parse full quoted skill names:** `src/agents/pi-embedded-helpers/errors.ts:558-561`
  - body: The new parser captures only `[a-z0-9_-]+`, while OpenClaw renders raw skill names and separately sanitizes command aliases. A valid skill name like `GitHub Actions` would be parsed as `GitHub`, fail the skill match, and keep the generic unknown-tool error.
  - confidence: 0.88
