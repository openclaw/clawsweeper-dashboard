## Summary

Keep this PR open because the user-facing hint is useful and the attached proof shows the intended behavior, but the current branch is not mergeable as-is against current main and misses the current missing-tool execution path.

## What This Changes

The PR adds a skill-aware rewrite for unknown-tool errors so a tool call matching a registered skill tells the model to read that skill's SKILL.md.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary, using current @earendil contracts, active-run visible skills, robust quoted-name parsing, and focused regression tests.

## Reproduction Assessment

Yes, source-based. Current main still returns generic missing-tool handling with no SKILL.md hint, and the PR screenshots show the intended after-fix behavior, but I did not run a live agent in this read-only review.

## Solution Assessment

No. The direction is good, but this branch targets stale Pi APIs and hooks a path that current immediate missing-tool results skip; the safer fix is to patch the current missing-tool boundary directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent package and hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main imports @earendil-works/pi-agent-core, whose Agent exposes afterToolCall as an option/property. This branch adds @mariozechner types and calls setAfterToolCall/private _afterToolCall, so it will not typecheck or compose correctly after rebase.
  - confidence: 0.96
- **[P2] Route immediate missing-tool results through the rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1941`
  - body: In @earendil-works/pi-agent-core 0.74.0, missing tools are returned as immediate prepareToolCall results and skip finalizeExecutedToolCall/afterToolCall. Installing only this hook leaves the normal Tool X not found path generic, which is the case this feature is trying to fix.
  - confidence: 0.94
- **[P2] Match only skills visible in the active run:** `src/agents/pi-embedded-runner/run/attempt.ts:1955-1957`
  - body: The runner narrows or clears the skill snapshot before building the active prompt, but the patch checks params.skillsSnapshot. A guessed hidden skill name could receive a SKILL.md hint even when the active run did not expose that skill.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved runtime skills:** `src/agents/pi-embedded-helpers/errors.ts:573-577`
  - body: resolvedSkills is optional and current runner reloads workspace skill entries when it is absent. Requiring resolvedSkills here suppresses the hint for persisted snapshots that still advertise skills after hydration.
  - confidence: 0.84
- **[P3] Parse the full quoted skill name:** `src/agents/pi-embedded-helpers/errors.ts:557-560`
  - body: The parser captures only [a-z0-9_-]+, but prompt-facing skill names can contain spaces, punctuation, or a leading $. Those valid names are truncated or ignored, so the hint will not fire for common skill spellings.
  - confidence: 0.8
