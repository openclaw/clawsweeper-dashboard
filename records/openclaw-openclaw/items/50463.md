## Summary

Keep open: the UX direction is useful and the PR has real behavior proof, but the current patch is not merge-ready because it hooks the wrong missing-tool boundary for the pinned dependency and can reveal skills hidden from the run.

## What This Changes

The PR adds an embedded-runner rewrite that turns unknown-tool errors matching a registered skill into guidance to read that skill's SKILL.md.

## Best Possible Solution

Implement the hint at the missing-tool result boundary, derive matches from the skills visible to this run, preserve existing hooks through the current Agent.afterToolCall API, and cover skill match, non-skill, hidden skill, collision, and punctuation cases.

## Reproduction Assessment

Yes, source-based. Current main still returns generic unknown-tool handling, and the pinned dependency shows the PR's afterToolCall-only hook does not receive immediate missing-tool results.

## Solution Assessment

No, not as written. The maintainable fix is to rewrite immediate missing-tool results using the run-visible skill set and current hook surface, with focused regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Rewrite immediate missing-tool results before afterToolCall:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1937`
  - body: With pinned @earendil-works/pi-agent-core@0.74.0, a missing tool returns an immediate result from prepareToolCall and never reaches finalizeExecutedToolCall, which is where afterToolCall runs. This hook therefore leaves the skill-as-tool path with the generic not-found message on current main.
  - confidence: 0.92
- **[P2] Use the run-visible skill set for hints:** `src/agents/pi-embedded-runner/run/attempt.ts:1954`
  - body: Sandboxed non-rw runs intentionally clear skillsSnapshotForRun before exposing skills in the prompt. Passing params.skillsSnapshot here can reveal a hidden persisted skill name when the model guesses it as a tool, so the matcher needs the same visible skill set used for the run.
  - confidence: 0.9
- **[P2] Compose with Agent.afterToolCall instead of private fields:** `src/agents/pi-embedded-runner/run/attempt.ts:1938-1942`
  - body: The current Agent exposes afterToolCall as a mutable public property, and the 0.74.0 types do not define setAfterToolCall or _afterToolCall. Wrap the public property instead so the code works after rebase and preserves any existing handler.
  - confidence: 0.86
- **[P3] Match the full visible skill name:** `src/agents/pi-embedded-helpers/errors.ts:555-559`
  - body: The regex only captures [a-z0-9_-]+, but OpenClaw renders raw skill names and has separate command sanitization because names may include spaces or symbols. Valid names such as GitHub Actions or $openclaw-pr-maintainer will miss or truncate the hint.
  - confidence: 0.82
