## Summary

This PR should stay open: the user-facing hint is still useful, but the branch is stale against current main and has concrete correctness blockers in the agent runtime path.

## What This Changes

The PR adds helper logic to rewrite unknown-tool error results into a skill-specific SKILL.md hint and wires it through the embedded Pi runner after-tool-call path.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary using current `@earendil-works` contracts, visible run skills, collision preservation, and focused regression coverage.

## Reproduction Assessment

Yes, source-based. Current main still has generic missing-tool behavior around the dependency's immediate missing-tool result, and that path can be covered with a focused runner/helper regression test.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best current implementation because it is stale against the Pi package namespace/API and needs to hook the current missing-tool result boundary with run-visible skill matching.

## Review Findings

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi package API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main imports Pi types from `@earendil-works/pi-agent-core`, but this branch adds imports from the old `@mariozechner` namespace and calls `setAfterToolCall`. After rebasing, this will not compile or compose against the current dependency contract.
  - confidence: 0.96
- **[P2] Route missing-tool results through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1941`
  - body: The missing-tool failure is produced as an immediate `Tool X not found` result by the Pi loop's prepare phase, and the dependency only calls `afterToolCall` from finalized executed tools. A fix needs to route that immediate path through the rewrite.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1956-1957`
  - body: The runner can clear skills for sandboxed non-read-write sessions before building runtime state. Matching against `params.skillsSnapshot` can confirm hidden skill names, so the rewrite should use the run-visible skill set.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved skills:** `src/agents/pi-embedded-helpers/errors.ts:572-578`
  - body: `resolvedSkills` is optional while the lightweight `skills` list is the stable prompt-facing representation. Returning false when `resolvedSkills` is absent can suppress the hint even when the session still advertises the skill name.
  - confidence: 0.84
- **[P3] Parse prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:557-558`
  - body: Skill names are not limited to `[a-z0-9_-]+`, and prompt-visible names may include a leading `$`, spaces, or punctuation. The helper should parse the quoted tool name format rather than truncating valid skill names.
  - confidence: 0.8
