## Summary

Keep open: the requested hint is still useful and not implemented on current main, but this PR needs changes before merge because its hook misses the main missing-tool path and is based on the old Pi Agent setter API.

## What This Changes

The PR adds a helper and embedded-runner hook to rewrite unknown-tool errors into a SKILL.md hint when the missing tool name matches a registered skill.

## Best Possible Solution

Implement the hint at a boundary that affects the model-visible immediate missing-tool result on current @earendil packages, preserve generic errors for non-skills and real tool-name collisions, and cover the behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main lacks the skill hint, and the PR's afterToolCall hook does not run for immediate missing-tool results in the current Pi dependency contract.

## Solution Assessment

No. The intent is sound, but the current implementation hooks the wrong lifecycle boundary and uses the stale @mariozechner/setter API instead of current @earendil Agent APIs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route missing-tool results through the skill hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1938`
  - body: The new hook is installed through afterToolCall, but the targeted `Tool X not found` case is produced as an immediate result by prepareToolCall. Immediate results are finalized without entering finalizeExecutedToolCall, so the common skill-as-tool failure still returns the generic error. Move the rewrite to a boundary that handles immediate missing-tool results, or use a Pi API that explicitly runs for them.
  - confidence: 0.96
- **[P2] Rebase onto the current Pi Agent hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-8`
  - body: This branch imports hook types from @mariozechner/pi-agent-core and calls `setAfterToolCall`, but current main uses @earendil-works/pi-agent-core 0.74.0, where Agent exposes `afterToolCall` as a property and no setter. As written, this will not typecheck after rebasing onto current main.
  - confidence: 0.9
