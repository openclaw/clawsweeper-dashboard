## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Retarget the hook to current missing-tool behavior:** `src/agents/pi-embedded-runner/run/attempt.ts:1940`
  - body: The new rewrite is installed only through afterToolCall, but current @earendil-works/pi-agent-core@0.75.4 builds `Tool X not found` as an immediate `prepareToolCall` result and emits it without running `afterToolCall`. That is the exact skill-called-as-tool case this PR targets, so after rebasing to current main the model would still receive the generic unknown-tool result.
  - confidence: 0.88
- **[P2] Match against the skills actually visible for the run:** `src/agents/pi-embedded-runner/run/attempt.ts:1956`
  - body: This passes only `params.skillsSnapshot` into the matcher, and the helper then requires `resolvedSkills`. Current runner code can build the visible skills prompt from freshly loaded `skillEntries` when a snapshot is absent or lacks `resolvedSkills`, so those visible skills would still miss the new hint.
  - confidence: 0.8
- **[P2] Parse the full registered skill name:** `src/agents/pi-embedded-helpers/errors.ts:560-561`
  - body: `resolveUnknownToolName` captures only `[a-z0-9_-]+`, but OpenClaw skill names come from the raw `skill.name` rendered into the skills prompt and may include spaces or symbols. A valid skill such as `GitHub Actions` would be parsed as only `GitHub`, so the hint silently fails for some registered skills.
  - confidence: 0.73
