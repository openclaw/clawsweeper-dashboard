## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi Agent API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main pins `@earendil-works/pi-agent-core@0.74.0`, whose `Agent` exposes `afterToolCall` as an option/property and has no `setAfterToolCall` method. Importing the old `@mariozechner` types and calling the setter will not compile or compose after rebasing.
  - confidence: 0.96
- **[P2] Route prepare-time missing tools through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1937`
  - body: In the pinned Pi loop, unknown tools return an immediate `Tool X not found` result from `prepareToolCall`, and `afterToolCall` only runs for prepared/executed tools. Installing the rewrite only as an `afterToolCall` hook leaves the main skill-called-as-tool case unchanged.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1953`
  - body: Current main clears `skillsSnapshotForRun` for sandboxed non-rw runs before building prompt/runtime state, but this rewrite checks `params.skillsSnapshot`. A guessed hidden skill name could receive confirmation even though it was not visible to the model in that run.
  - confidence: 0.9
- **[P2] Handle snapshots without resolvedSkills:** `src/agents/pi-embedded-helpers/errors.ts:573-578`
  - body: `hasVisibleSkillMatch` returns false when `resolvedSkills` is absent, but current session/runtime code deliberately reloads skill entries in that case and can still render skills into the prompt. The hint should use the run-visible skill state rather than disappearing for prompt/skills-only snapshots.
  - confidence: 0.86
- **[P3] Parse full prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:558-561`
  - body: `resolveUnknownToolName` only captures `[a-z0-9_-]+`, while the skills prompt renders raw skill names and supported names can contain spaces or punctuation. Valid visible skills such as `GitHub Actions` would be truncated and miss the hint.
  - confidence: 0.82
