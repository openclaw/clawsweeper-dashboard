## Summary

This PR should stay open: the requested hint is a useful agent UX improvement and is not present on current main, but the branch is stale against the current Pi package/API surface and still has correctness and hidden-skill disclosure issues.

## What This Changes

The PR extracts unknown-tool parsing and adds an embedded-runner hook that rewrites missing-tool results matching registered skills to include SKILL.md guidance.

## Best Possible Solution

Implement the hint at a current OpenClaw-owned missing-tool result boundary, using only run-visible skill names and focused coverage for punctuation-bearing skill names.

## Reproduction Assessment

Yes, source-based. Current main has no SKILL.md hint for skill-name-as-tool errors, and the PR screenshot shows the intended after-fix text plus a non-skill control case.

## Solution Assessment

No. The direction is useful, but this patch is not the best current implementation because it depends on stale/private Pi APIs and uses the full skill snapshot instead of the run-visible skill set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Use the current Pi hook surface:** `src/agents/pi-embedded-runner/run/attempt.ts:3-9`
  - body: Current main uses @earendil-works/pi-agent-core 0.74.0, whose Agent exposes a public afterToolCall property and no setAfterToolCall method. This stale import/private-field wiring will not compose or typecheck after rebase.
  - confidence: 0.92
- **[P2] Route immediate missing-tool results through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1937-1940`
  - body: The pinned Pi loop returns missing tools as prepareToolCall immediate results and only runs afterToolCall from finalizeExecutedToolCall for prepared tools. Because this rewrite is only installed as afterToolCall, the main skill-called-as-tool case still returns the generic error.
  - confidence: 0.9
- **[P2] Use only run-visible skills for the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: Sandboxed non-rw runs intentionally clear the run-visible skill snapshot before exposing skills, but this rewrite checks params.skillsSnapshot. A guessed hidden skill name can therefore be confirmed by the hint.
  - confidence: 0.88
- **[P3] Parse complete prompt-facing skill names:** `src/agents/pi-embedded-helpers/errors.ts:558-562`
  - body: The helper only captures [a-z0-9_-]+, while OpenClaw renders raw skill names in the prompt. Valid skill names containing spaces, $, or other punctuation are truncated or missed, so the hint silently fails for them.
  - confidence: 0.82
