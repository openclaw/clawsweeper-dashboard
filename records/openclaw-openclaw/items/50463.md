## Summary

This PR remains a plausible focused UX improvement, but it should stay open because current review evidence still shows the branch may miss the active missing-tool path and may reveal skills outside the run-visible boundary. The contributor has provided real-session screenshot proof for the happy path, so the remaining blocker is code-path correctness rather than proof alone.

## What This Changes

The PR adds a skill-specific rewrite for unknown-tool error results by extracting unknown-tool parsing helpers and registering an embedded-runner `afterToolCall` hook.

## Best Possible Solution

Implement the hint at the actual missing-tool result boundary, use the same run-visible skill snapshot that built the prompt, preserve prior hook behavior, and cover skill match, non-skill unknown tool, hidden-skill, and broad skill-name cases with focused regression tests.

## Reproduction Assessment

Yes, source-based. Current main still lacks a skill-specific unknown-tool hint, and the PR body plus screenshot show the intended live behavior for skill-match and non-match cases.

## Solution Assessment

No. The desired behavior is good, but the branch should be moved to the active missing-tool boundary and use the run-visible skill snapshot before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Route missing-tool results through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1937`
  - body: The rewrite is installed only as an `afterToolCall` hook, but the currently pinned pi-agent-core missing-tool path can produce an immediate `Tool X not found` result. If that result bypasses `afterToolCall`, the model still sees the generic error in the main case this feature targets.
  - confidence: 0.88
- **[P2] Use the run-visible skill snapshot:** `src/agents/pi-embedded-runner/run/attempt.ts:1954`
  - body: The runner can clear `skillsSnapshotForRun` before exposing skills to a sandboxed run, but this hook is described as using the persisted snapshot. That can advertise a skill name that was intentionally hidden from the current run.
  - confidence: 0.82
- **[P3] Match the full skill-name surface:** `src/agents/pi-embedded-helpers/errors.ts:555-559`
  - body: The unknown-tool parser only captures `[a-z0-9_-]+`, while OpenClaw skill names can include spaces, punctuation, or the `$` spelling users and prompts commonly use. Those valid skill names will miss the hint or be truncated before comparison.
  - confidence: 0.76
