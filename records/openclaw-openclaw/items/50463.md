## Summary

Keep open, but do not merge the current patch as-is. The feature remains useful, but the branch targets stale Pi APIs, misses the current prepare-time missing-tool path, and can hint skills that were hidden from the run.

## What This Changes

The PR modifies embedded Pi runner and assistant-error helper code to rewrite unknown-tool errors into SKILL.md hints when the unknown name matches a registered skill.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary, using current @earendil contracts, run-visible skills only, collision preservation, robust quoted-name parsing, and focused regression coverage.

## Reproduction Assessment

Yes, source-based. Current main has generic unknown-tool handling without a skill hint, and the current Pi package source shows the common missing-tool result is a prepare-time immediate result.

## Solution Assessment

No. The proposed hint is useful, but the current patch is not the best implementation because it hooks stale Pi APIs and misses the current prepare-time unknown-tool path that produces the target message.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:3-8`
  - body: Current main uses `@earendil-works/pi-agent-core` 0.74.0, where the Agent exposes `afterToolCall` as an option/property, not the old `@mariozechner` imports plus `setAfterToolCall`. After rebasing, this branch will not compile or compose against the dependency currently pinned by OpenClaw.
  - confidence: 0.96
- **[P2] Route prepare-time missing tools through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1937`
  - body: The current Pi loop returns missing tools as immediate prepare results and only applies `afterToolCall` from `finalizeExecutedToolCall` for prepared/executed tools. Because this patch only installs an after-tool-call hook, the normal `Tool X not found` path this feature targets still returns the generic error.
  - confidence: 0.94
- **[P2] Use the run-visible skills snapshot:** `src/agents/pi-embedded-runner/run/attempt.ts:1956-1957`
  - body: The runner can clear skills for sandboxed non-read-write sessions before building the run state, but the patch checks `params.skillsSnapshot`. A model that guesses a hidden skill name could get a SKILL.md hint even when that skill was removed from this run's prompt/runtime state.
  - confidence: 0.9
- **[P3] Parse quoted skill names completely:** `src/agents/pi-embedded-helpers/errors.ts:558-559`
  - body: The parser only captures `[a-z0-9_-]+`, but prompt-facing skill names come from raw `skill.name` values and can include spaces or punctuation. Valid errors like `Tool "GitHub Actions" not found` would be truncated and fail the skill match.
  - confidence: 0.8
