## Summary

Keep this PR open: the requested skill hint is useful and not implemented on current main, but the branch is stale against the current Pi dependency and misses important current-main runtime boundaries.

## What This Changes

The PR adds unknown-tool parsing and an embedded-runner hook to rewrite missing-tool results into a SKILL.md hint when the missing name matches a registered skill.

## Best Possible Solution

Implement the hint at the current OpenClaw-owned missing-tool result boundary using current @earendil-works contracts, run-visible skills, collision preservation, robust quoted-name parsing, and focused regression coverage.

## Reproduction Assessment

Yes, source-based. Current main still has generic unknown-tool handling with no SKILL.md hint path, and the PR body screenshot shows the intended improved result in a real agent session.

## Solution Assessment

No. The direction is reasonable, but this branch is not the best implementation because it targets stale Pi imports and wires the rewrite through a hook that does not cover current prepare-time missing-tool results.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi package API:** `src/agents/pi-embedded-runner/run/attempt.ts:3-8`
  - body: Current main has moved this runner to the `@earendil-works` Pi packages, but the patch imports Agent types from `@mariozechner` and calls the older `setAfterToolCall`/private-field API. After rebase this will not compile or compose with the dependency OpenClaw currently pins.
  - confidence: 0.96
- **[P2] Route prepare-time missing tools through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1936`
  - body: In `@earendil-works/pi-agent-core@0.74.0`, missing tools are returned as immediate prepare results and bypass `afterToolCall`. Because this patch only installs an after-tool-call hook, the normal `Tool X not found` path this feature targets can still return the generic error.
  - confidence: 0.94
- **[P2] Match only skills visible in this run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: The runner can clear or filter skills before building the run state, but the patch checks `params.skillsSnapshot`. A model that guesses a hidden skill name could receive a SKILL.md hint even when that skill was removed from this run's prompt/runtime state.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved skills:** `src/agents/pi-embedded-helpers/errors.ts:571-574`
  - body: `resolvedSkills` is optional while the lightweight `skills` list is required, and the runner reloads entries when `resolvedSkills` is absent. Returning false in that case can suppress the hint for sessions that still advertise the skill through the snapshot prompt/catalog.
  - confidence: 0.84
- **[P3] Parse quoted skill names completely:** `src/agents/pi-embedded-helpers/errors.ts:557-558`
  - body: The parser only captures `[a-z0-9_-]+`, but prompt-facing skill names can include `$`, spaces, punctuation, or path-like separators. Those valid names would be truncated or ignored, so the hint would not fire for common skill spellings.
  - confidence: 0.8
