## Summary

Keep this PR open: the feature is useful and not implemented on current main, but the branch is stale against the current Pi runtime and the proposed hook misses the main missing-tool path. The attached screenshots are sufficient real behavior proof for the contributor’s tested setup, so the remaining blocker is a focused code repair rather than more proof.

## What This Changes

The PR adds a skill-aware unknown-tool rewrite so tool-result errors matching registered skills tell the model to read the skill’s SKILL.md, with helper extraction in assistant error formatting and runner hook wiring.

## Best Possible Solution

Keep the skill hint, but implement it at the current OpenClaw-owned missing-tool result boundary using the current `@earendil-works` contracts, active-run visible skills, robust quoted-name parsing, and regression coverage.

## Reproduction Assessment

Yes, source-based. Current main still has only generic missing-tool handling, and the PR screenshots show the intended after-fix live output, but I did not run a live agent in this read-only review.

## Solution Assessment

No. The direction is good, but the branch wires the hint through stale Pi APIs and an `afterToolCall` path that current immediate missing-tool results skip; patching the current missing-tool boundary is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current Pi agent package and hook API:** `src/agents/pi-embedded-runner/run/attempt.ts:4-10`
  - body: Current main uses `@earendil-works/pi-agent-core`, whose Agent exposes `afterToolCall` as an option/property. This branch imports old `@mariozechner` hook types and calls `setAfterToolCall`/private `_afterToolCall`, so it will not compose with the current package surface after rebase.
  - confidence: 0.96
- **[P2] Route immediate missing-tool results through the rewrite:** `src/agents/pi-embedded-runner/run/attempt.ts:1925-1941`
  - body: In `@earendil-works/pi-agent-core@0.74.0`, missing tools are returned as immediate prepare-time results and skip `finalizeExecutedToolCall`/`afterToolCall`. Installing only this hook leaves the normal `Tool X not found` path generic, which is the case this feature is trying to fix.
  - confidence: 0.94
- **[P2] Match only skills visible in the active run:** `src/agents/pi-embedded-runner/run/attempt.ts:1954-1957`
  - body: The runner narrows or clears the skill snapshot before building the active prompt, but the patch checks `params.skillsSnapshot`. A guessed hidden or sandbox-filtered skill name could receive a SKILL.md hint even when that skill was not exposed to the active run.
  - confidence: 0.9
- **[P2] Handle snapshots without resolved runtime skills:** `src/agents/pi-embedded-helpers/errors.ts:573-577`
  - body: `resolvedSkills` is optional, and the current runner reloads workspace skill entries when it is absent. Requiring `resolvedSkills` here suppresses the hint for persisted snapshots or callers that still advertise skills after hydration.
  - confidence: 0.84
- **[P3] Parse the full quoted skill name:** `src/agents/pi-embedded-helpers/errors.ts:557-560`
  - body: The parser captures only `[a-z0-9_-]+`, but prompt-facing skill names can contain spaces, punctuation, or a leading `$`. Those valid names are truncated or ignored, so the hint will not fire for common skill spellings.
  - confidence: 0.8
