## Summary

Keep open. Current main still lacks the requested skill-aware unknown-tool hint, but this PR is not merge-ready because it uses an unsupported pi-agent-core hook API and wires the rewrite to a path that misses the pinned dependency's immediate missing-tool result.

## What This Changes

The branch extracts unknown-tool parsing, adds a skill-aware tool-result rewrite helper, and wires an embedded-runner afterToolCall hook to return a SKILL.md hint when a missing tool name matches a visible skill.

## Best Possible Solution

Keep the feature direction, but implement it through a supported immediate missing-tool result seam, compose with AgentSession's existing afterToolCall property, define skill-name matching semantics, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main has no skill-aware rewrite, and the pinned pi-agent-core 0.71.1 missing-tool result is created as an immediate Tool X not found result that bypasses afterToolCall.

## Solution Assessment

No. This PR is not the best way to solve the issue because it calls a nonexistent setter/private hook and relies on afterToolCall for a missing-tool path that does not invoke that hook in the pinned dependency.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the supported afterToolCall property:** `src/agents/pi-embedded-runner/run/attempt.ts:1940`
  - body: @mariozechner/pi-agent-core@0.71.1 exposes `agent.afterToolCall` as a public property; it does not expose `setAfterToolCall` or `_afterToolCall`. This call will fail against the current type/runtime contract and also fails to compose with AgentSession's existing handler.
  - confidence: 0.97
- **[P1] Route missing-tool results through a supported seam:** `src/agents/pi-embedded-runner/run/attempt.ts:1940`
  - body: The target `Tool X not found` result is produced by `prepareToolCall()` as an immediate result, and the sequential/parallel paths emit immediate results without calling `finalizeExecutedToolCall()`, where `afterToolCall` runs. Even with the setter fixed, this hook misses the main case the PR is trying to improve.
  - confidence: 0.94
- **[P3] Match complete skill names before adding the hint:** `src/agents/pi-embedded-helpers/errors.ts:560-561`
  - body: `resolveUnknownToolName()` captures only `[a-z0-9_-]+`, but visible skill names come from raw skill metadata and may contain spaces or punctuation. A valid skill such as `GitHub Actions` would be extracted as `GitHub`, so the new hint silently fails for those skills.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/agents/pi-embedded-helpers/errors.ts:602`
  - body: This adds new user-visible guidance, but the branch does not update `CHANGELOG.md`. OpenClaw requires user-facing fix/feature changes to include an active-version changelog entry before merge.
  - confidence: 0.86
