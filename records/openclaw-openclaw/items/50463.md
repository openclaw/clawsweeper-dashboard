## Summary

Keep open: the improvement is useful, but this patch is not correct for current main because the pinned Pi loop bypasses `afterToolCall` for immediate missing-tool results, and the hook would use a skill snapshot that this run intentionally hid.

## What This Changes

Adds skill-specific unknown-tool error hints by exporting a helper in agent error formatting and installing an embedded-runner `afterToolCall` rewrite hook.

## Best Possible Solution

Implement the hint at the current missing-tool result boundary, use the same run-visible skill snapshot that built the prompt, preserve existing hook composition, and add focused regression coverage for skill match, non-skill unknown tool, sandbox-hidden skills, and broader skill-name parsing.

## Reproduction Assessment

Yes, source-based. Current main pins `@earendil-works/pi-agent-core@0.74.0`, whose missing-tool path returns a generic immediate `Tool X not found` result, and current OpenClaw main has no skill-specific rewrite for that path.

## Solution Assessment

No. The branch wires the hint through a hook that the current immediate missing-tool path does not call, and it should instead rewrite at the active missing-tool result boundary using the run-visible skill snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Route immediate missing-tool results through the hint:** `src/agents/pi-embedded-runner/run/attempt.ts:1936-1937`
  - body: Current main pins `@earendil-works/pi-agent-core@0.74.0`, where `prepareToolCall()` returns missing tools as `kind: "immediate"`; those immediate results are emitted without calling `afterToolCall`, so this hook misses the main case the feature is trying to fix.
  - confidence: 0.91
- **[P2] Use the run-visible skill snapshot for hints:** `src/agents/pi-embedded-runner/run/attempt.ts:1954`
  - body: The runner clears `skillsSnapshotForRun` when a sandboxed run should not expose workspace skills, but the PR passes `params.skillsSnapshot` to the hint rewrite, so a hidden persisted skill can still be advertised in the tool result.
  - confidence: 0.84
- **[P3] Match full registered skill names:** `src/agents/pi-embedded-helpers/errors.ts:556-557`
  - body: `resolveUnknownToolName()` only captures `[a-z0-9_-]+`, while skill names are later sanitized because raw names may contain spaces or symbols; quoted names such as `GitHub Actions` would be truncated before comparison and silently skip the hint.
  - confidence: 0.76
