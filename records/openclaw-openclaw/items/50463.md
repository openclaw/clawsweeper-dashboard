## Summary

Keep open: the feature is useful, and the PR has sufficient screenshot proof, but the patch is not mergeable against current main. Current main still lacks the SKILL.md hint, and the branch uses a stale Pi package API while wiring the rewrite behind a hook that current missing-tool results bypass.

## What This Changes

The PR adds an unknown-tool error rewrite so a tool call whose name matches a registered skill gets a `Read its SKILL.md first` hint.

## Best Possible Solution

Implement the hint on the current Pi immediate missing-tool result path or another supported current API, while preserving existing tool-result and plugin-hook behavior.

## Reproduction Assessment

Yes. Source inspection against current main and `@earendil-works/pi-agent-core@0.74.0` shows missing tools become immediate `Tool X not found` results, while current main has no SKILL.md hint rewrite.

## Solution Assessment

No. The intent is sound, but the current branch uses a stale private setter and hooks after a boundary that current missing-tool results do not cross; the fix should target the generated immediate result path or a supported current hook.
