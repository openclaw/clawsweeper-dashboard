## Summary

Keep this PR open: current main only partially covers the fork-secret failure class, because `docs-sync-publish.yml` now skips token-backed steps when the token is absent, but the release translation workflow still dispatches with `OPENCLAW_DOCS_SYNC_TOKEN` and no repository guard.

## What This Changes

The PR adds job-level `github.repository == 'openclaw/openclaw'` conditions to the docs publish sync and release translation GitHub Actions jobs.

## Best Possible Solution

Refresh against current main, then land both repository guards or narrow the diff to the still-missing release-translation guard if maintainers want to preserve the current docs-publish token-empty fork behavior.

## Reproduction Assessment

Yes. The screenshots prove the docs publish fork failure and after-fix skip, and current source shows the release translation workflow still uses `OPENCLAW_DOCS_SYNC_TOKEN` without a repository guard.

## Solution Assessment

Yes, with a maintainer policy caveat. A job-level repository guard is a narrow workflow-side fix for upstream-only secret-backed jobs, though maintainers may choose a release-translation-only guard if they want to preserve current docs-publish fork behavior.
