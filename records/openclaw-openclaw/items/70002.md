## Summary

Keep open: current main still lacks the fork guards, and this PR is a narrow, correct infra bug fix. Because it touches secret-backed docs workflows and already has the `triage: risky-infra` label, the remaining step is maintainer review rather than cleanup closure.

## What This Changes

This PR adds job-level `github.repository == 'openclaw/openclaw'` guards to `.github/workflows/docs-sync-publish.yml` and `.github/workflows/docs-translate-trigger-release.yml`.

## Best Possible Solution

Land this PR, or an equivalent two-line job-level repository guard, so the docs publish workflows stay upstream-only without changing token scope, scripts, or upstream runtime behavior.

## Reproduction Assessment

Yes. Current main statically shows both secret-backed jobs run without a repository guard, and GitHub's Actions contract explains why non-`GITHUB_TOKEN` secrets are missing in fork-triggered runs; the PR body also includes fork failure screenshots and an after-fix skipped run screenshot.

## Solution Assessment

Yes. A job-level `if: github.repository == 'openclaw/openclaw'` is the narrowest maintainable fix because it skips the whole secret-backed job in forks while leaving upstream behavior unchanged.
