## What This Changes

This PR adds job-level `github.repository == 'openclaw/openclaw'` guards to `.github/workflows/docs-sync-publish.yml` and `.github/workflows/docs-translate-trigger-release.yml`.

## Best Possible Solution

Land this PR, or an equivalent two-line job-level repository guard, so both docs publish workflows remain upstream-only while their token usage, permissions, scripts, and runtime behavior stay unchanged on `openclaw/openclaw`.

## Reproduction Assessment

Yes. Current main statically reproduces the failure path: both target jobs are unguarded and consume `OPENCLAW_DOCS_SYNC_TOKEN`, while GitHub does not pass non-`GITHUB_TOKEN` secrets to fork-triggered workflow runs; the PR discussion also includes fork failure screenshots and patched dispatch verification for the docs sync workflow.

## Solution Assessment

Yes. A job-level `if: github.repository == 'openclaw/openclaw'` is the narrowest maintainable fix because it skips all secret-backed steps outside upstream and matches existing OpenClaw workflow patterns without changing the publish scripts or token scope.
