## Best Possible Solution

Land this PR, or an equivalent two-line job-level repository guard, so the docs publish and release translation jobs are upstream-only while the publish script, tokens, workflow permissions, and runtime/package code stay unchanged.

## Reproduction Assessment

Yes. The PR discussion also includes screenshots and author-reported fork dispatch verification for the patched docs sync job.

## Solution Assessment

Yes. A job-level `if: github.repository == 'openclaw/openclaw'` is the narrowest maintainable fix because it skips all secret-backed steps outside upstream and leaves upstream job behavior unchanged.
