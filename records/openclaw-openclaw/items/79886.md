## Summary

The reported Telegram group behavior matches current documented and tested visible-reply policy: group/channel final replies are private by default, while visible room output is expected to go through the message tool unless `messages.groupChat.visibleReplies` is set to `automatic`.

## What This Changes

Assistant replies from Telegram group chat sessions are reported to stop using the native Telegram delivery path while explicit message tool sends still work.

## Best Possible Solution

Keep the documented default behavior, and use `messages.groupChat.visibleReplies: "automatic"` for deployments that want legacy native final replies in group rooms.

## Reproduction Assessment

Yes for the reported delivery split: current source intentionally classifies Telegram groups as group chats, resolves them to `message_tool_only`, and suppresses native automatic final delivery while the message tool remains usable. No separate failing Telegram transport path is shown beyond that expected policy.

## Solution Assessment

Yes: the best current solution is configuration, not a Telegram code change. Set `messages.groupChat.visibleReplies` to `automatic` to restore legacy native final replies, or keep the default and require visible output through `message(action=send)`.
