## Summary

Keep this PR open, but do not merge it as-is. Current main still has the active non-streaming Pi steering gap the PR targets, while the proposed patch leaves normal non-reply steering paths able to inherit interrupt behavior and is missing the required user-facing changelog entry.

## What This Changes

The PR removes the streaming-only embedded Pi steering gate, adds a forward-compatible `toolCallSteeringBehavior` option, classifies reply steering as inject versus interrupt, and expands focused steering tests.

## Best Possible Solution

Land one coordinated active-run steering fix that steers active non-streaming Pi runs, defaults normal steering to inject, reserves interrupt for stop-like text, includes caller coverage and a changelog entry, and validates the final Pi API contract.

## Reproduction Assessment

Yes. Current main is source-reproducible through the `runReplyAgent` streaming-only gate and `queueEmbeddedPiMessage` rejecting non-streaming handles; the PR defect is source-reproducible by following subagent steering callers that omit the new option into the session-wide interrupt default.

## Solution Assessment

No, not as-is. The direction is maintainable, but the PR should default omitted Pi tool-call steering behavior to `inject` for each steer or make every caller explicit, then leave only stop-like reply text on the interrupt path.
