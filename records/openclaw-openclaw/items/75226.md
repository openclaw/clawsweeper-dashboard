## Summary

Keep this PR open, but do not merge it as-is. Current main still has the reported active-run steering gap, while the proposed patch has a blocking behavior regression: normal steering callers that omit the new Pi option would inherit the PR's session-wide interrupt default instead of inject semantics.

## What This Changes

The PR changes embedded Pi reply steering to accept active non-streaming runs, adds a per-message `toolCallSteeringBehavior` option, initializes a forward-compatible Pi agent property, and expands focused steering tests.

## Best Possible Solution

Land a narrow steering fix after omitted/default Pi steering is treated as normal inject behavior, stop-like text is the only interrupt path, subagent/default queue callers are covered, release notes are added, and the final Pi API contract is validated or the branch is consolidated with #73365.

## Reproduction Assessment

Yes. The current-main gap is source-reproducible through `runReplyAgent` requiring `isStreaming` and `queueEmbeddedPiMessage` rejecting `!isStreaming`; the PR defect is source-reproducible by following subagent/default queue calls that omit `toolCallSteeringBehavior` into the PR's session-wide interrupt default.

## Solution Assessment

No, not as-is. Steering active non-streaming Pi runs is the right direction, but the patch should default omitted behavior to `inject` or make every caller explicit, and keep interrupt behavior limited to abort-like requests.
