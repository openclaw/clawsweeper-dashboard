## Summary

Keep open: this PR should not merge as written because it removes current Vydra bounded JSON response hardening and deletes the regression coverage that proves oversized provider control responses are rejected.

## What This Changes

The PR replaces bounded provider JSON reads in Vydra job polling, speech, and video generation with direct `Response.json()` calls and removes the associated cap-rejection tests.

## Best Possible Solution

Keep the shared bounded reader and oversized-response tests for Vydra control JSON; only relax that cap after explicit maintainer security acceptance plus real proof.

## Reproduction Assessment

Yes at source level: the PR replaces capped reads with direct `Response.json()` calls, and Node's implementation fully reads the body before parsing. I did not run a live OOM reproduction.

## Solution Assessment

No. The existing `readProviderJsonResponse` helper is the narrow owner-boundary solution for provider JSON caps, and removing it here reopens the exact failure mode current tests cover.
