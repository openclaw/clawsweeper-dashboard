## Summary

Keep open. The underlying diagnostic-warning gap still exists on current main, but this PR is not mergeable as-is because it targets an older raw `fs.appendFile`/`drain()` writer shape instead of the current safe `safeAppendFile`/`flush()` contract, and it still lacks after-fix real behavior proof.

The PR adds QueuedFileWriter consecutive failure tracking, periodic warnings, a drain helper, and related tests/mock updates for diagnostic file writers.
