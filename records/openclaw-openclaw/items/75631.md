## Summary

Keep this open. Current main still contains the macOS full-config save path and local `saveDict` fallback that can write a supplied snapshot without preserving `gateway.auth`; the same author has opened #75815 as a focused implementation candidate, so the right cleanup is to review that PR rather than close this issue now.

## What This Changes

The issue requests that macOS app/dashboard config writes preserve `gateway.auth` and unrelated config fields, prefer partial/conflict-aware mutation, and avoid unnecessary Gateway restarts for Talk/UI settings.

## Best Possible Solution

Review and land #75815 or an equivalent macOS-side guard that preserves `gateway.auth`, rejects destructive `gateway.mode`/auth clobbers before rename, and prevents stale/invalid Gateway write failures from falling back to an unsafe full-file write. Longer term, move UI settings edits toward `config.patch` or another conflict-aware partial mutation path with regression coverage for auth preservation and restart behavior.

## Reproduction Assessment

Yes. The issue gives concrete macOS steps, affected RPCs, and log signatures, and current main still contains the full-snapshot save/fallback path needed for a stale snapshot clobber; I did not run a live macOS app repro in this read-only review.

## Solution Assessment

No for current main. The Gateway RPC path has base-hash and merge protections, but the macOS direct writer needs the same auth-preserving/destructive-write rejection behavior; #75815 appears to be the narrow implementation candidate to review first.
