## What This Changes

The issue reports that macOS app/dashboard settings writes can replace persisted config, drop `gateway.auth`, and force Gateway restarts that disconnect Control UI clients.

## Best Possible Solution

Keep one canonical macOS/dashboard config-write fix thread open and land a single PR that makes app/dashboard settings writes conflict-aware and patch/merge-based, preserves unknown/auth fields including SecretRefs/redacted values, and avoids Gateway restarts for Talk/voice settings unless the reload plan truly requires one.
