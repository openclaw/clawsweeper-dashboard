## Summary

Keep this open. Current main still has macOS config paths that can submit or write full config snapshots, including a local file-write fallback, while the protected gateway RPC path is stronger but does not cover every direct macOS writer or explicitly guard `gateway.auth` preservation.

## What This Changes

The issue reports that macOS app/dashboard settings writes can replace persisted config, drop `gateway.auth`, and restart or disconnect Gateway clients.

## Best Possible Solution

Route all macOS app and dashboard settings changes through conflict-aware partial mutation or a fresh read/merge/validate/write path, preserve unknown fields and `gateway.auth` by default, and either remove or harden direct local `saveDict` fallbacks with pre-commit auth clobber rejection and regression tests.

## Reproduction Assessment

Yes. The issue gives concrete macOS symptoms, and current source still exposes a reproducible stale/partial snapshot path: save a root missing `gateway.auth` through `ConfigStore.save` or the local `saveDict` fallback and the persisted config can lose the auth block; auth-path changes are restart-triggering in current tests.

## Solution Assessment

No. Current main improves gateway RPC writes with base hashes and merge-patch behavior, but the macOS direct/local paths still need the same conflict-aware contract and an auth-preservation guard.
