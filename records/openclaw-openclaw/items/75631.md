## Summary

Close as a duplicate of the open canonical macOS Talk/config-writer bug. The remaining behavior described here is already tracked by #64973, with the same stale/full config write, gateway auth loss, token mismatch, and restart symptoms, and current main still has the relevant macOS config writer surface that should be fixed under that canonical report.

## What This Changes

Make macOS app/dashboard settings writes preserve `gateway.auth` and unrelated config while avoiding unnecessary Gateway restarts and WebSocket disconnects.

## Best Possible Solution

Keep #64973 as the canonical tracker and implement a macOS-owned patch/merge-based writer that preserves `gateway.auth` and other unowned fields, with regression coverage for ConfigStore/AppState/Talk settings writes and the expected no-restart behavior for simple Talk/UI settings.

## Reproduction Assessment

Yes. The issue body gives a concrete macOS 2026.4.27 repro with redacted logs, and #64973 provides the same controlled Talk-mode path with stale `config.set`, token mismatch, missing `gateway.auth` in the bad backup, and restart logs.

## Solution Assessment

Yes for cleanup: closing this duplicate and routing the evidence to #64973 is the narrowest maintainer path. The technical fix should still happen in the canonical issue, likely by replacing full macOS config replacement with conflict-aware patch/merge behavior.
