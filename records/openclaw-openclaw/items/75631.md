## Summary

Keep this open. Current main still has macOS config paths that can submit or write a full config snapshot, including a local `saveDict` fallback, while the gateway/Control UI path has stronger base-hash and destructive-write guards; the reported `gateway.auth` loss is therefore not proven fixed by the current code or related open work.

## What This Changes

The issue asks for macOS app/dashboard settings writes to preserve `gateway.auth` and unrelated config, prefer patch/conflict-aware mutation, and avoid unnecessary Gateway restarts for Talk/UI changes.

## Best Possible Solution

Fix the macOS config layer so app/UI settings changes use conflict-aware partial mutation or merge against a fresh gateway snapshot, preserve unknown and auth fields by default, and either remove or harden the local `saveDict` fallback with pre-commit clobber rejection for `gateway.auth`. Add regression coverage for preserving `gateway.auth` and unrelated fields while changing Talk/voicewake/dashboard settings, plus restart/no-restart expectations for non-gateway config changes.

## Reproduction Assessment

Yes. The issue provides concrete macOS steps and log signatures, and current-main source still exposes the full-config macOS save/fallback path needed to reproduce a stale snapshot clobber; this review did not run the macOS app live.

## Solution Assessment

No current implementation is not the best solution. The gateway RPC and browser Control UI paths have base-hash/destructive-write protections, but the macOS app still needs the same patch/conflict-aware contract instead of full snapshot replacement and unguarded local fallback writes.
