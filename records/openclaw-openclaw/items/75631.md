## Summary

Keep this open. Current main still has macOS config paths that can submit or write full config snapshots, including a local `saveDict` fallback, and the same-author open PR #75815 is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to prevent macOS app/dashboard settings writes from dropping persisted `gateway.auth`, causing Gateway restarts and dashboard WebSocket disconnects.

## Best Possible Solution

Review and land #75815 or an equivalent focused macOS fix that blocks destructive local fallback writes, preserves `gateway.auth` and mandatory gateway envelope fields during generic settings saves, and adds Swift regression coverage. A later migration from full snapshot saves to narrower `config.patch` calls can remain follow-up work if the guard is landed first.

## Reproduction Assessment

Yes. The issue gives concrete macOS steps and log signatures, and current-main source still exposes the full-config save/fallback path needed for a stale snapshot clobber; this review did not perform a live macOS reproduction.

## Solution Assessment

Yes for keeping the issue open and routing through the paired macOS config-writer fix. Gateway RPC base-hash protections are useful but do not cover direct macOS file writes, so closing from current main would be premature.
