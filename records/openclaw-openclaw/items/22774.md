## Summary

Keep this open. Current main has inter-session provenance, `spawnedBy` lineage, and `senderLabel` display hints, but not the durable per-message human/main-agent/sub-agent sender identity contract requested across `chat.send`, `sessions.send`/`sessions.steer`, subagent completion delivery, transcript history/live events, and Control UI avatars. The prior linked PRs are closed unmerged or narrower adjacent work, and m13v's comment usefully confirms that persistent sender identity, not only visual avatars, is needed for auditability.

## What This Changes

Request durable multi-role sender attribution in Chat UI so human, main-agent, and sub-agent messages render with distinct identity in multi-agent workflows.

## Best Possible Solution

Define an additive sender metadata contract for messages, persist it in transcripts, expose it through history/live events, carry it through chat/session/subagent paths, and update Control UI grouping and avatar rendering around that contract.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the reported limitation: the relevant schemas and UI types only expose provenance, `spawnedBy`, and `senderLabel`, not durable sender id/kind/avatar metadata. I did not run a live multi-agent UI session because this read-only review was limited to source/docs/tests/history inspection.

## Solution Assessment

Yes, keeping this open is the best current outcome. Partial `senderLabel`, `InputProvenance`, and `spawnedBy` mechanisms are useful but do not solve the requested audit-grade sender identity contract, and the linked related work is not a merged replacement.
