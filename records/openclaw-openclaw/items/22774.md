## Summary

Keep open. Current main has inter-session provenance, a `spawnedBy` chat event field, and limited `senderLabel` rendering, but it still lacks the durable per-message human/main-agent/sub-agent identity contract requested for multi-agent auditability across gateway send paths, transcript history, live events, and Control UI avatars.

## What This Changes

Request durable multi-role sender attribution in the Chat UI so human, main-agent, and sub-agent messages can be distinguished in multi-agent workflows.

## Best Possible Solution

Define an additive per-message sender metadata contract with sender id, kind, display name, source session/agent, and renderable avatar/color metadata; persist it in transcripts, expose it through `chat.history` and live events, carry it through `chat.send`, `sessions.send`, `sessions.steer`, and subagent announce delivery, then render grouping and avatars from that metadata in Control UI.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send between sessions or deliver a subagent announce, then inspect `chat.history` or Control UI: the resulting inter-session turn remains a user-role message with provenance/source hints but no renderable sender identity contract.

## Solution Assessment

Yes for keeping this open. The existing `senderLabel`, `spawnedBy`, and provenance pieces are useful but narrower than the requested audit trail; #72951 is also only a partial sender-label recovery, not the durable contract.
