## Summary

Keep open. Current main has useful source provenance, `spawnedBy`, and limited `senderLabel` display, but it still lacks the requested durable per-message human/main-agent/sub-agent identity contract across protocol, transcript history, live events, and Control UI avatars.

## What This Changes

The issue requests multi-role sender identity in the Chat UI so human, main-agent, and sub-agent messages can be distinguished in multi-agent workflows.

## Best Possible Solution

Define an additive per-message sender metadata contract, persist it in transcripts, expose it through history/live events, carry it through send/steer/subagent announce paths, and render grouping plus avatars from that metadata in Control UI.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: inter-session sends and subagent announces record provenance/source hints, but the resulting user-role history/UI messages have no durable sender id/kind/avatar/color identity contract.

## Solution Assessment

Yes for keeping this open, but no to an avatar-only fix. The narrow maintainable solution is a protocol/storage/UI sender metadata contract rather than more local display heuristics.
