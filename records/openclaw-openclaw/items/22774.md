## Summary

Keep this open. Current main has useful adjacent provenance, browser-local identity, `senderLabel`, and `spawnedBy` plumbing, but it still lacks the requested durable per-message human/main-agent/sub-agent identity contract through Gateway send APIs, transcript history, live events, subagent announce delivery, and Control UI avatar rendering.

## What This Changes

Request support for distinct human, main-agent, and sub-agent sender identity and avatars in the Chat UI for multi-agent workflows.

## Best Possible Solution

Add an additive per-message sender metadata contract that is persisted in transcripts, exposed through history and live events, carried by `chat.send`, `sessions.send`, `sessions.steer`, and subagent announce paths, and rendered by Control UI with stable labels and visual identity.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main capability gap: sender identity metadata is not accepted by the relevant Gateway send schemas or rendered by Control UI beyond user-role `senderLabel`.

## Solution Assessment

No current implementation is the best solution yet. The maintainable direction is an additive durable sender metadata contract rather than a role-only or UI-only special case.
