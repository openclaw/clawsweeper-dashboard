## Summary

Keep open: current main still has trace provenance, browser-local identity, and user-role sender labels, but not the durable human/main-agent/sub-agent sender identity contract requested for multi-agent auditability.

## What This Changes

Request durable multi-role sender attribution in Chat UI so human, main-agent, and sub-agent messages render with distinct identities in multi-agent workflows.

## Best Possible Solution

Define an additive durable per-message sender metadata contract, persist it in transcripts, expose it through history/live events, carry it through send and subagent paths, and render it in Control UI with explicit trust boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives a high-confidence capability gap in the Gateway schemas, transcript writer, subagent announce path, and Control UI renderer.

## Solution Assessment

Yes in direction, but the maintainable solution is stronger than avatars alone. Durable per-message sender metadata is needed; reusing browser-local identity or senderLabel recovery would not preserve audit identity across protocol, transcripts, history, live events, and subagent delivery.
