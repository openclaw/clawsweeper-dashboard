- `impact:session-state`: The request concerns durable per-message attribution in session transcripts, history, live events, and chat UI state.

## Summary

Keep open: current main still has trace provenance, browser-local identity, `senderLabel`, and `spawnedBy`, but it does not provide the durable per-message sender identity contract needed to distinguish human, main-agent, and sub-agent messages across protocol, transcripts, live events, and Control UI rendering.

## What This Changes

Request durable multi-role sender attribution in the Chat UI so human, main-agent, and sub-agent messages are visually and historically distinguishable in multi-agent workflows.

## Best Possible Solution

Define one additive per-message sender metadata contract with explicit trust semantics, persist it in transcripts, expose it through history and live events, carry it through `chat.send`, `sessions.send`/`sessions.steer`, and subagent announce delivery, then render names, avatars, grouping, and colors in Control UI.

## Reproduction Assessment

Not applicable as a strict bug repro because this is a feature request, but the missing capability is source-reproducible. Current main lacks durable sender identity fields across the Gateway schemas, transcript row builder, subagent delivery, and Control UI grouping/rendering paths.

## Solution Assessment

Yes in direction, but avatars alone are not the best solution. The maintainable path is a durable sender metadata contract with protocol, transcript, event, UI, and trust-model follow-through rather than reusing browser-local identity, `senderLabel`, or `spawnedBy` hints as the permanent contract.
