## Summary

Keep this open. Current main has adjacent provenance, browser-local identity, user-role `senderLabel`, and session-level `spawnedBy`, but it still does not provide a durable per-message sender identity contract that can distinguish human, main-agent, and sub-agent messages across Gateway APIs, transcripts, live events, and Control UI rendering.

## What This Changes

The issue requests Chat UI support for distinct human, main-agent, and sub-agent sender identity in multi-agent workflows.

## Best Possible Solution

Add an additive per-message sender metadata contract that is persisted in transcripts, exposed through history and live events, carried through `chat.send`, `sessions.send`, `sessions.steer`, and subagent announce delivery, then render stable labels and visual identity in Control UI.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence of the current-main capability gap: the relevant Gateway send schemas and Control UI rendering path do not carry or render durable sender identity metadata.

## Solution Assessment

No current implementation is the best solution yet. The maintainable path is an additive protocol/transcript/UI sender metadata contract, not another role-only or user-label-only special case.
