## Summary

Keep open: current main and v2026.6.5 still provide provenance, browser-local identity, coarse `senderLabel` display, and the merged #89191 WebChat projection fix, but they do not implement a durable per-message human/main-agent/sub-agent sender identity contract across Gateway protocol, transcripts, live events, subagent delivery, and Control UI rendering.

## What This Changes

The issue requests multi-role Chat UI attribution so multi-agent workflows can distinguish human, main-agent, and sub-agent messages by sender identity, avatar, color, or label.

## Best Possible Solution

Define one additive per-message sender metadata and trust contract, persist it in transcripts, expose it through history and live events, carry it through `chat.send`, `sessions.send`/`sessions.steer`, and subagent announce delivery, then render names, avatars, grouping, and colors in Control UI.

## Reproduction Assessment

Yes, source-reproducible as a feature gap: current schemas, transcript builders, subagent delivery, and UI normalizers lack durable sender identity fields even though exact live multi-agent UI proof was not rerun in this read-only review.

## Solution Assessment

Yes in direction, but the best solution is a durable sender metadata and trust contract, not a UI-only avatar/color patch or reuse of coarse `senderLabel`/`spawnedBy` hints.
