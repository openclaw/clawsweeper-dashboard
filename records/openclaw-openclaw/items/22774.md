## Summary

Keep open: current main still has provenance hints, browser-local identity, `senderLabel`, and `spawnedBy`, but not a durable per-message sender identity contract that can distinguish human, main-agent, and sub-agent messages across protocol, transcripts, live events, and Control UI rendering.

## What This Changes

Request to add durable multi-role sender identity in Chat UI and session history for human, main-agent, and sub-agent messages in multi-agent workflows.

## Best Possible Solution

Define one additive per-message sender metadata contract with explicit trust semantics, persist it in transcripts, expose it through history/live events, carry it through `chat.send`, `sessions.send`/`sessions.steer`, and subagent announce delivery, then render sender names, avatars, grouping, and colors in Control UI.

## Reproduction Assessment

Not applicable as a strict bug repro; this is a feature request. Source inspection gives high confidence that current main lacks durable sender identity fields across Gateway schemas, transcript persistence, subagent delivery, and Control UI grouping/rendering.

## Solution Assessment

Yes in direction, but avatars alone are not enough. The maintainable solution is a durable sender metadata contract with protocol, transcript, event, UI, and trust-model follow-through rather than reusing browser-local identity or `senderLabel` hints.
