## Summary

Keep open: current main still has trace provenance, browser-local identity, `senderLabel` hints, and `spawnedBy` events, but not the durable sender identity contract needed for multi-agent transcripts, live events, and Control UI rendering. Related issues have been closed into this as the canonical tracker rather than replacing it.

## What This Changes

Add durable per-message sender identity so the Chat UI can distinguish human, main-agent, and sub-agent messages in multi-agent workflows.

## Best Possible Solution

Define one additive per-message sender metadata contract with explicit trust semantics, persist it in transcripts, expose it through history/live events, carry it through `chat.send`, `sessions.send`/`sessions.steer`, and subagent completion delivery, then render sender names, avatars, grouping, and colors in Control UI.

## Reproduction Assessment

Not applicable as a strict bug repro; this is a feature request. Source inspection gives high confidence that current main lacks durable sender identity fields across Gateway schemas, transcript persistence, subagent delivery, and Control UI grouping/rendering.

## Solution Assessment

Yes in direction, but avatars alone are not enough. The maintainable solution is a durable sender metadata contract with protocol, transcript, event, UI, and trust-model follow-through rather than reusing browser-local identity or `senderLabel` hints.
