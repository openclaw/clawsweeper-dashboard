## Summary

Keep open: current main now has a useful display-only `sessions_send` forwarded-label projection, but the requested durable human/main-agent/sub-agent identity contract with per-message sender metadata, avatars, and colors is still missing across protocol, transcript, subagent announce, and Control UI rendering.

## What This Changes

Request durable multi-role sender identity in the Chat UI so human, main-agent, and sub-agent messages are visually and auditably distinct in multi-agent workflows.

## Best Possible Solution

Define an additive durable per-message sender metadata contract, persist and expose it through transcript/history/live events, carry it through chat and session handoff paths, and render Control UI names/avatars/colors from that metadata.

## Reproduction Assessment

No live high-confidence UI reproduction was run. Source inspection gives a clear current-main path: messages still carry role plus optional `senderLabel`/provenance rather than durable sender identity metadata for human/main-agent/sub-agent rendering.

## Solution Assessment

Unclear until maintainers choose the durable sender metadata contract. A UI-only avatar/color patch would be too narrow because `chat.send`, `sessions.send`/`sessions.steer`, subagent announce, transcript history, and live events need a consistent sender identity shape.
