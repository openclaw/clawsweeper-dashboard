## Summary

Keep open. Current main and v2026.6.6 have partial forwarded-label display for `sessions_send`, but they still do not provide the requested durable human/main-agent/sub-agent sender identity contract across protocol inputs, transcript persistence, subagent delivery, history/live events, and per-message Control UI avatars.

## What This Changes

Request durable multi-role sender attribution in Chat UI so human, main-agent, and sub-agent messages are distinguishable in multi-agent workflows.

## Best Possible Solution

Define one additive trusted sender metadata shape, persist it with each applicable message, propagate it through session handoffs and subagent announcements, expose it in history and live events, and render stable names/avatars/colors without changing provider-facing message roles.

## Reproduction Assessment

Yes, as a source-level capability check. Current schemas, provenance types, persistence, subagent delivery, and Control UI rendering provide derived labels and role-based avatars, not a durable human/main-agent/sub-agent sender identity contract.

## Solution Assessment

Yes in direction, but not as new provider message roles or a UI-only avatar patch. The maintainable solution is additive trusted sender metadata that preserves canonical transcript roles while giving UI and history consumers durable renderable identity.
