## Summary

Keep open: current main still has trace provenance, browser-local identity, user-role sender labels, and spawnedBy session metadata, but not the durable human/main-agent/sub-agent sender identity contract needed for multi-agent auditability across protocol, transcripts, live/history events, and Control UI rendering.

## What This Changes

Request durable multi-role Chat UI attribution so human, main-agent, and sub-agent messages have distinct identity, avatar/color, and optional labels in multi-agent workflows.

## Best Possible Solution

Define an additive per-message sender metadata contract, persist it, expose it through history/live events, carry it through chat.send, sessions.send/steer, and subagent completion delivery, then render it in Control UI with explicit trust boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks durable sender identity fields across Gateway schemas, transcript provenance, subagent delivery, and Control UI rendering.

## Solution Assessment

Yes in direction, but avatars alone are not enough. The maintainable fix is an additive durable sender metadata contract with protocol, transcript, event, and UI follow-through rather than reusing browser-local identity or top-level senderLabel hints.
