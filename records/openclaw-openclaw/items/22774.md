## Summary

Keep open. Current main has browser-local identity, top-level `senderLabel` handling, inter-session provenance, and newly surfaced `spawnedBy` event/session lineage, but the requested durable message sender identity is still missing from the closed send protocol schemas, persisted inter-session messages, subagent announce delivery, chat history/live events, and Control UI avatar/group rendering. The related maintainer review and user context point to the same durable audit-trail gap, and the open related PR #72951 is narrower sender-label recovery rather than a replacement.

## What This Changes

Request durable multi-role sender attribution in Chat UI so human, main-agent, and sub-agent messages render with distinct identity instead of routed inter-session messages appearing as the local user.

## Best Possible Solution

Keep this as the focused durable sender-identity tracker. The eventual fix should add an additive per-message sender metadata contract, persist it in transcripts, expose it through `chat.history` and live session/chat events, carry it through `chat.send`, `sessions.send`/`sessions.steer`, and subagent announce delivery, and render distinct human/main-agent/sub-agent identity in Control UI without relying on local browser identity or session-level lineage alone.
