## Summary

Keep this open: current main has inter-session provenance, `spawnedBy`, and optional `senderLabel` display hints, but it still lacks a durable human/main-agent/sub-agent sender identity contract that is persisted, exposed through chat/session APIs, and rendered as distinct avatars/colors in Control UI.

## What This Changes

The issue requests multi-role sender identity in Chat UI so human, main-agent, and sub-agent messages can be distinguished in multi-agent workflows.

## Best Possible Solution

Add an additive per-message sender metadata contract, persist it in transcripts, expose it through chat history/live events and session send/steer paths, and render it in Control UI with stable labels and avatars.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main still lacks the requested durable sender identity fields and role-aware UI rendering.

## Solution Assessment

Yes, keeping this open is the right direction. The maintainable fix is a protocol/transcript/UI sender metadata contract rather than another local avatar-only or label-only heuristic.
