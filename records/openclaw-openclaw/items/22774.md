## Label Justifications

- `P2`: Missing attribution materially impairs multi-agent auditability and debugging, but it is not an emergency or core-runtime outage.
- `impact:session-state`: The issue concerns preserving and displaying which human or agent authored messages across related sessions and transcript projections.

## Summary

Current main only partially addresses the symptom by projecting `sessions.send` messages as assistant-side “Forwarded from …” entries. The requested durable sender identity remains absent from protocol inputs, persisted provenance, subagent completion delivery, live/history contracts, and per-message Control UI identity, so this issue is still the canonical cross-surface tracker rather than a duplicate or implemented item.

## What This Changes

Add durable multi-role message attribution so the Chat UI can distinguish humans, primary agents, and subagents with appropriate names and visual identity.

## Best Possible Solution

Keep this issue as the canonical tracker and design an additive authoritative per-message sender identity object—stable sender kind/id, source agent/session, display name, and safe visual metadata—persisted in transcripts, emitted through history and live events, carried by inter-session and subagent delivery paths, and rendered consistently by the Control UI.

## Reproduction Assessment

Yes, at source level. Current main has only a derived `sessions_send` forwarded label, while its closed send schemas, persisted provenance, and subagent delivery cannot represent the requested durable sender identity.

## Solution Assessment

No, adding three UI-only roles is not the best solution. The narrow maintainable path is an additive authoritative identity contract at the protocol/transcript boundary, followed by consistent history, live-event, and UI rendering.
