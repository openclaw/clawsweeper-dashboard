## Summary

Keep open: current main still has inter-session provenance, browser-local identity, user-only sender labels, and spawnedBy event metadata, but it does not provide the requested durable human/main-agent/sub-agent sender identity contract across protocol schemas, transcripts, live/history events, subagent delivery, and Control UI rendering.

## What This Changes

Requests durable multi-role sender identity in Control UI so human, main-agent, and sub-agent messages are visually and auditably distinct in multi-agent workflows.

## Best Possible Solution

Define an additive per-message sender metadata contract, persist it, expose it through history/live events, carry it through chat.send, sessions.send/steer, and subagent completion delivery, then render it in Control UI with explicit trust boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks durable sender identity fields across Gateway schemas, transcript provenance, subagent delivery, and Control UI rendering.

## Solution Assessment

Yes in direction, but avatars alone are not enough. The maintainable solution is an additive durable sender metadata contract with protocol, transcript, event, UI, and trust-model follow-through rather than reusing browser-local identity or senderLabel hints.
