## Summary

Keep open: current main still has trace provenance, browser-local identity, user-only sender labels, and adjacent spawnedBy event metadata, but not the requested durable human/main-agent/sub-agent sender identity contract across protocol schemas, transcript storage, live/history events, subagent delivery, and Control UI rendering.

## What This Changes

Request durable Chat UI sender attribution that distinguishes human, main-agent, and sub-agent messages in multi-agent workflows.

## Best Possible Solution

Define an additive per-message sender metadata contract, persist it, expose it through history/live events, carry it through chat.send, sessions.send/sessions.steer, and subagent completion delivery, then render it in Control UI with explicit trust boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.12 lack durable sender identity fields across Gateway schemas, transcript provenance, subagent delivery, and Control UI rendering.

## Solution Assessment

Yes in direction, but avatars alone are not enough. The maintainable solution is an additive durable sender metadata contract with protocol, transcript, event, UI, and trust-model follow-through rather than reusing browser-local identity or senderLabel hints.
