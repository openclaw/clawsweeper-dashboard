## Summary

Keep open: current main has partial forwarded labels, browser-local identity, and inter-session provenance, but it still lacks the requested durable human/main-agent/sub-agent sender identity contract across protocol input, transcripts, subagent announcements, live/history payloads, and per-message Control UI avatars.

## What This Changes

The issue requests multi-role Chat UI attribution so human, main-agent, and subagent messages remain distinguishable in multi-agent workflows.

## Best Possible Solution

Define one additive trusted per-message sender metadata contract and propagate it through chat.send, sessions.send/steer, subagent announce delivery, transcripts, history/live events, and Control UI rendering without changing provider-facing roles.

## Reproduction Assessment

Yes, at source level. Current schemas, provenance types, sessions_send forwarding, subagent delivery, history projection, and Control UI rendering provide trace fields or derived labels rather than a durable human/main-agent/sub-agent sender identity contract.

## Solution Assessment

Yes in direction, but not as new provider message roles or a UI-only avatar patch. The maintainable solution is additive trusted sender metadata that preserves canonical transcript roles while giving history and UI consumers durable renderable identity.
