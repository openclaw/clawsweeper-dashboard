## Summary

Keep open: current main still has trace provenance, browser-local identity, and role/`senderLabel` rendering, but not the durable per-message human/main-agent/sub-agent sender identity contract needed for multi-agent auditability. The related issues and PRs are narrower, broader, open, or closed unmerged, so none replaces this focused tracker.

## What This Changes

Request durable multi-role sender identity and visual attribution in Chat UI so human, main-agent, and sub-agent messages are distinguishable in multi-agent workflows.

## Best Possible Solution

Add an additive durable per-message sender metadata contract, persist it in transcripts, expose it through history/live events, carry it through send and subagent paths, and render it in Control UI with clear trust boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main capability gap across the Gateway schemas, transcript writer, subagent announce path, and Control UI renderer.

## Solution Assessment

Yes, a durable per-message sender identity contract is the maintainable direction. A UI-only role/color patch or `senderLabel` recovery would be too shallow because it would not preserve audit identity across protocol, transcripts, history, live events, and subagent delivery.
