## Summary

Keep open: current main and the latest release still lack the durable human/main-agent/sub-agent sender identity contract requested here; the related open and closed items cover narrower author, prompt, sender-label, or broad multi-agent UX work rather than replacing this focused protocol/transcript/Control UI attribution tracker.

## What This Changes

The issue requests durable multi-role sender identity in Chat UI so human, main-agent, and sub-agent messages have distinct, auditable attribution.

## Best Possible Solution

Define one additive per-message sender metadata contract with explicit trust semantics, persist it in transcripts, expose it through history/live events, carry it through chat.send, sessions.send/sessions.steer, and subagent completion delivery, then render it in Control UI names, avatars, grouping, and colors.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.12 lack durable sender identity fields across Gateway schemas, transcript persistence, subagent delivery, and Control UI rendering.

## Solution Assessment

Yes in direction, but avatars alone are not enough. The maintainable solution is a durable sender metadata contract with protocol, transcript, event, UI, and trust-model follow-through rather than reusing browser-local identity or senderLabel hints.
