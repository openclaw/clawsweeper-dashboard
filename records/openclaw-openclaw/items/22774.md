## Summary

Keep open: current main and v2026.6.5 still expose provenance hints, browser-local identity, `senderLabel`, and `spawnedBy`, but not the durable per-message sender identity needed to distinguish human, main-agent, and sub-agent messages across protocol, transcripts, live events, and Control UI rendering. The related open work at https://github.com/openclaw/openclaw/issues/77355 covers a narrower LLM replay/history attribution gap and is not a replacement for this Chat UI/session identity tracker.

## What This Changes

Requested durable multi-role Chat UI attribution so human, main-agent, and sub-agent messages have distinct renderable identities in multi-agent sessions.

## Best Possible Solution

Define one additive per-message sender metadata contract with explicit trust semantics, persist it in transcripts, expose it through history and live events, carry it through `chat.send`, `sessions.send`/`sessions.steer`, and subagent announce delivery, then render names, avatars, grouping, and colors in Control UI.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request, but the missing capability is source-reproducible. Current main lacks durable sender identity fields across the Gateway schemas, persisted user-turn builder, subagent delivery, and Control UI grouping/rendering paths.

## Solution Assessment

Yes in direction, but avatars alone are not sufficient. The maintainable solution is a durable sender metadata and trust contract with protocol, transcript, live-event, and UI follow-through rather than treating `senderLabel`, browser-local identity, or `spawnedBy` as the permanent contract.
