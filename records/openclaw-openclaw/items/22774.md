## Summary

Keep this open: current main has useful provenance, lineage, and display-label pieces, but not the durable per-message human/main-agent/sub-agent identity contract requested for multi-agent auditability.

## What This Changes

Request durable multi-role sender attribution in Chat UI so human, main-agent, and sub-agent messages can be distinguished across multi-agent workflows.

## Best Possible Solution

Define an additive message sender metadata contract, persist it in transcripts, expose it through history/live events, carry it through chat/session/subagent paths, and render Control UI groups/avatars from that metadata.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the reported limitation: current schemas and UI types expose provenance, `spawnedBy`, and `senderLabel`, but not durable per-message sender id/kind/avatar metadata.

## Solution Assessment

Yes, keeping this open is the best current outcome. The existing partial mechanisms are useful building blocks but do not solve the requested audit-grade sender identity contract, and the linked related work is not a merged replacement.
