## Summary

Keep open: current main and the latest release still lack a durable sender identity contract across protocol schemas, transcripts, subagent handoffs, live/history projection, and Control UI rendering; related items are narrower satellites or adjacent identity work rather than replacements.

## What This Changes

Request durable multi-role Chat UI attribution so human, main-agent, and sub-agent messages render with distinct identity in multi-agent workflows.

## Best Possible Solution

Define one additive per-message sender metadata contract with explicit trust semantics, persist it in transcripts, expose it through history/live events, carry it through chat.send, sessions.send/sessions.steer, and subagent completion delivery, then render name/avatar/color metadata in Control UI.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.12 lack durable sender identity fields across Gateway schemas, transcript persistence, subagent delivery, and Control UI rendering.

## Solution Assessment

Yes in direction, but avatars alone are not enough. The maintainable solution is a durable sender metadata contract with protocol, transcript, event, UI, and trust-model follow-through rather than reusing browser-local identity or senderLabel hints.
