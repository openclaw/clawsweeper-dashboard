## Label Justifications

- `impact:session-state`: The issue concerns preserving and displaying which human or agent authored messages across related sessions and transcript projections.

## Summary

Keep open. Current main and the latest release only provide a narrow derived `sessions_send` label, while the durable human/main-agent/sub-agent sender identity contract is still absent from protocol inputs, persisted provenance, subagent completion delivery, and per-message Control UI identity rendering.

## What This Changes

Request durable multi-role Chat UI attribution so human, main-agent, and sub-agent messages have distinguishable identity, avatar, and visual treatment in multi-agent workflows.

## Best Possible Solution

Keep this issue as the canonical tracker for an additive authoritative per-message sender identity object that is persisted, emitted through history/live events, carried by inter-session and subagent paths, and rendered consistently by Control UI.

## Reproduction Assessment

Yes, at source level. Current main has only a derived `sessions_send` forwarded label, while its closed send schemas, persisted provenance, and subagent delivery cannot represent the requested durable sender identity.

## Solution Assessment

No, a UI-only three-role/avatar patch is not the best solution. The maintainable path is an additive authoritative identity contract at the protocol/transcript boundary, followed by consistent history, live-event, and UI rendering.
