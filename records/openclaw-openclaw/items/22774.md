## Summary

Keep open. Current main and v2026.6.8 have partial forwarded-label rendering for `sessions_send`, but they still lack the requested durable human/main-agent/sub-agent sender identity contract across protocol input, transcript persistence, live/history events, subagent delivery, and Control UI avatars/colors.

## What This Changes

Request durable multi-role chat attribution so human, main-agent, and sub-agent messages render with distinct identity in Control UI multi-agent workflows.

## Best Possible Solution

Define one additive sender metadata contract for persisted messages and projected chat events, then render that contract in Control UI with stable sender kind/name/avatar/color while coordinating narrower related issues.

## Reproduction Assessment

Yes, at source level. Current main lacks sender identity fields in `chat.send`, `sessions.send`, provenance, transcript/display projection, and Control UI normalization/rendering, while tests only cover a narrow `senderLabel` projection.

## Solution Assessment

Unclear until maintainers choose the product and trust contract. The best direction is not another UI-only role patch, but an additive persisted sender metadata contract consumed by history/live events and Control UI rendering.
