## Summary

Keep this open: current main still has only trace provenance, browser-local identity, and role-based `senderLabel` rendering, not the durable human/main-agent/sub-agent sender identity contract requested for multi-agent auditability. The linked related items are either narrower, broader, still open, or closed unmerged, so none replaces this focused tracker.

## What This Changes

The issue requests durable multi-role sender identity in Chat UI so human, main-agent, and sub-agent messages can be distinguished in multi-agent workflows.

## Best Possible Solution

Add an additive durable per-message sender metadata contract, persist it in transcripts, expose it through history/live events, carry it through send and subagent paths, and render it in Control UI with clear trust boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main capability gap: the relevant Gateway schemas, transcript writer, subagent announce path, and Control UI renderer do not carry durable sender identity metadata.

## Solution Assessment

Yes, a durable per-message sender identity contract is the maintainable direction. A UI-only role/color patch or existing `senderLabel` recovery would be too shallow because it would not preserve audit identity across protocol, transcripts, history, live events, and subagent delivery.
