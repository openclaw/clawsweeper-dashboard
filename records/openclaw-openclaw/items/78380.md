## Summary

Keep open: current main and the latest release still treat a raw SIGTERM without an OpenClaw restart intent as a stop path, so it bypasses the restart drain/safe coordinator. Related open items cover forced restart and skip-deferral reply-drain edges, but they do not settle the raw self-restart-from-chat policy reported here.

## What This Changes

The issue asks OpenClaw to prevent channel-originated chat turns from losing their visible reply when the agent restarts the gateway through inline tool or service-manager commands.

## Best Possible Solution

Keep this open until maintainers choose and implement a restart-safety policy that routes channel-initiated maintenance through the safe coordinator or refuses unsafe inline service-manager restarts.

## Reproduction Assessment

Yes at source level: current main sends SIGTERM without an OpenClaw restart intent down the stop path, so restart drain and safe-restart coordination are skipped. I did not run a live destructive channel restart in this read-only review.

## Solution Assessment

Unclear as an implementation path: the safe restart coordinator is the right foundation, but maintainers need to decide whether raw service-manager restarts from exec/tool turns should be blocked, converted, or documented as unsupported.
