## Summary

Keep open: current main has an explicit safe restart path, but the reported inline systemd/SIGTERM path can still bypass restart intent and reply-drain coordination, so this should be reconciled with the related open restart work rather than closed.

## What This Changes

Prevent channel-originated gateway self-restarts from dropping the current Telegram or Discord reply by blocking, deferring, or safely handing off the restart.

## Best Possible Solution

Route channel-turn Gateway restarts through one drain-aware path that preserves or defers final delivery, and add an explicit guard or documented refusal for raw service-manager restarts that cannot be made safe.

## Reproduction Assessment

Yes, at source level: reply dispatchers are counted for scheduled restart deferral, but a SIGTERM without restart intent still takes the stop path, so a raw systemd restart can bypass the drain and kill the in-flight reply path.

## Solution Assessment

No, current main is not complete for this report. The safe restart coordinator is the right foundation, but prompt guidance plus explicit `--safe` does not protect raw service-manager restarts launched from a chat turn.
