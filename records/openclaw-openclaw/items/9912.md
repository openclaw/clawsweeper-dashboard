## Summary

Keep open. Current main still lacks the requested generic defaults/per-agent hard turn or tool-call budget; the adjacent controls are strict config schemas without those fields, pattern-based loop detection, and an OpenResponses `max_tool_calls` field that is accepted but documented as ignored.

## What This Changes

Request a global/defaults or per-agent config such as `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to cap LLM-tool iteration loops and force a final response.

## Best Possible Solution

Add a typed, documented defaults/per-agent agent-run budget enforced before the next tool execution, with explicit timeout and exhaustion behavior coordinated with existing loop detection and usage-budget work.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a config feature request, and current-main source/docs inspection confirms the generic agent-level hard turn/tool-call budget does not exist.

## Solution Assessment

Partly. A generic defaults/per-agent hard budget is a plausible maintainable direction, but the exact config contract, precedence, and forced-response behavior need maintainer API/product decisions before implementation.
