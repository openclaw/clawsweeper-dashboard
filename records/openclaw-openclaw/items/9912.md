## Summary

Keep open. Current main still lacks the requested global/per-agent hard turn or tool-call budget; existing safeguards are pattern-based loop detection, a narrow post-compaction guard, and tool/provider-specific settings rather than the public agent-run config this issue asks for.

## What This Changes

The issue requests global/per-agent agent-run limits such as `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to prevent runaway tool-call loops.

## Best Possible Solution

Keep this as the canonical global/per-agent hard iteration budget request and design an additive typed config plus runtime enforcement path that composes with existing loop detection and channel-specific budget work.
