## Summary

Keep open. Current main and the latest shipped tag still lack a global or per-agent hard max-turn/max-tool-call budget for agent runs; existing safeguards are pattern-based loop detection, post-compaction guards, run-retry limits, or plugin-local xAI tool settings, and related open issues cover adjacent but separate scopes.

## What This Changes

Request a configurable agent-level `maxTurns`, `maxToolCalls`, and related timeout budget to stop runaway tool-call loops and force a response.

## Best Possible Solution

Design an additive `agents.defaults` plus per-agent hard budget contract, enforce counters before the next tool execution, document precedence with existing guards and timeouts, and add focused exhaustion tests.

Yes for a source-level absence check: current schema, docs, resolver, shipped tag, and before-tool enforcement paths still lack the requested global/per-agent hard budget.

## Solution Assessment

Unclear. A hard budget is plausible, but maintainers need to choose the public config shape, enforcement point, timeout-after-first-tool-call semantics, and final-response behavior before implementation is safe.
