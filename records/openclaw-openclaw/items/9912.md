## Summary

Keep open. Current main and the latest release still do not provide a global or per-agent hard `maxTurns`/`maxToolCalls` budget; adjacent loop detection, run retries, whole-run timeouts, and OpenResponses schema compatibility do not enforce the requested agent iteration limit.

## What This Changes

Request a global/per-agent `maxTurns`, `maxToolCalls`, and `toolCallTimeout` config surface that forces an agent run to stop or respond after too many model-tool iterations.

## Best Possible Solution

Design an additive `agents.defaults` plus per-agent hard-budget contract, enforce counters before the next tool execution, document precedence with existing loop detection/timeouts/retries, and keep per-channel auto-delegation or broader retry-budget work in their separate trackers.

## Solution Assessment

Unclear. A hard budget is plausible, but maintainers need to choose the public config shape, enforcement point, timeout-after-first-tool-call semantics, and final-response behavior before implementation is safe.
