## Summary

Keep open. Current main and v2026.6.5 still do not provide the requested global or per-agent hard agent-iteration budget; existing loop detection, run timeouts, xAI tool-local maxTurns, and ignored OpenResponses max_tool_calls do not satisfy this issue.

## What This Changes

Request a global or per-agent maxTurns, maxToolCalls, and toolCallTimeout config surface to stop runaway agent tool iterations and force a final response.

## Best Possible Solution

Design an additive agents.defaults plus per-agent hard-budget contract, enforce counters before the next tool execution in the trusted runtime path, document precedence with existing loop detection and timeouts, and keep per-channel auto-delegation in its separate tracker.
