## Summary

Keep open. Current main still lacks the requested global/per-agent hard turn or tool-call budget; existing protections are whole-run timeouts, accepted-but-ignored OpenResponses metadata, and optional pattern-based loop detection rather than configurable maxTurns/maxToolCalls enforcement.

## What This Changes

Add agent-level configuration to cap LLM-tool iterations, total tool calls, and optionally elapsed time after the first tool call before forcing a response.

## Best Possible Solution

Add a documented, typed agent-run budget under defaults and per-agent config, enforce it before the next tool execution, and keep channel-specific overflow/delegation behavior tracked separately.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature request, and source inspection confirms current main has no strict config or runtime path for the requested hard agent iteration budget.

## Solution Assessment

Partly. A generic agent-level hard budget is the right maintainable direction, but the exact config shape, precedence, and final-response behavior need maintainer product/API decisions before implementation.
