## Summary

Keep open. Current main still lacks a generic `agents.defaults` or per-agent hard iteration/tool-call budget; adjacent controls are pattern-based loop detection, xAI tool-local `maxTurns`, and OpenResponses `max_tool_calls` parsing that is documented as ignored.

## What This Changes

Request an agent-level config option such as `maxTurns`, `maxToolCalls`, or `toolCallTimeout` to cap agent tool-call iterations and force a response.

## Best Possible Solution

Add a typed, documented defaults/per-agent agent-run budget enforced before the next tool execution, with explicit timeout and exhaustion behavior coordinated with channel and usage-budget policy work.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/config request, and current-main source and docs inspection confirms the generic agent-level hard turn/tool-call budget does not exist.

## Solution Assessment

Partly. A generic defaults/per-agent hard budget is a plausible maintainable direction, but the exact config contract, precedence, and forced-response behavior require maintainer API/product decisions before implementation.
