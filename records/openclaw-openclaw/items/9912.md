## Summary

Keep open. Current main still lacks a generic global/per-agent hard turn or tool-call budget, and the related open items track adjacent but narrower surfaces rather than superseding this canonical request.

## What This Changes

The issue requests agent-level configuration to cap model-tool turns, total tool calls, and timeout-after-first-tool-call behavior before an agent continues looping.

## Best Possible Solution

Define an additive agent-run budget contract under agents.defaults and agents.list[], enforce counters before the next tool executes, document precedence and exhaustion behavior, and add focused config/runtime coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source and docs inspection gives a high-confidence check that the requested generic agent-level hard turn/tool-call budget is absent on current main.

## Solution Assessment

Unclear until maintainers choose the public contract. A defaults/per-agent hard budget is plausible, but field names, timeout semantics, precedence, and forced-response behavior require product/API judgment first.
