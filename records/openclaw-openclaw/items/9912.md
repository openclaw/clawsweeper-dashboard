## Summary

Keep open. Current main still lacks a global/defaults/per-agent hard turn or tool-call budget; the adjacent controls are whole-run timeouts, optional pattern-based loop detection, xAI tool-local `maxTurns`, and OpenResponses `max_tool_calls` parsing that remains documented as ignored.

## What This Changes

Request a defaults/per-agent config option such as `maxTurns`, `maxToolCalls`, or `toolCallTimeout` to force an agent to stop tool iteration and respond.

## Best Possible Solution

Add a typed, documented defaults/per-agent agent-run budget enforced before the next tool execution, coordinated with related channel and token-budget policy work.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/config request, and source/docs inspection confirms current main has no generic hard budget path for the requested agent iteration limits.

## Solution Assessment

Partly: a generic agent-level hard budget is the right maintainable direction, but the exact config scope, precedence, and final-response behavior need maintainer product/API decisions before implementation.
