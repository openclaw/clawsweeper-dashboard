## Summary

Keep open. Current main still lacks a generic defaults/per-agent hard turn or tool-call budget; existing controls are whole-run timeouts, optional pattern-based loop detection, and ignored OpenResponses `max_tool_calls` metadata.

## What This Changes

The issue requests configurable agent-level `maxTurns`, `maxToolCalls`, and `toolCallTimeout` limits to stop runaway tool-call iterations.

## Best Possible Solution

Add a typed, documented defaults/per-agent agent-run budget enforced before the next tool execution, while keeping per-channel auto-delegation tracked separately in #13271.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/config request, and source inspection confirms current main has no generic hard budget path for the requested agent iteration limits.

## Solution Assessment

Partly. A generic agent-level hard budget is the right maintainable direction, but the exact config shape, precedence, and final-response behavior need maintainer product/API decisions before implementation.
