## Summary

Keep open. Current main and the latest shipped tag still have adjacent retry and pattern-loop guards, but no global or per-agent hard budget for model/tool turns, total tool calls, or timeout-after-first-tool-call behavior.

## What This Changes

The issue requests global or per-agent hard limits such as `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to stop runaway agent tool-call loops and force a response.

## Best Possible Solution

Design an additive `agents.defaults` plus per-agent hard budget contract, enforce counters before the next tool execution in the embedded/tool-call runtime, define timeout-after-first-tool-call semantics, document precedence with existing guards and OpenResponses, and add focused exhaustion tests.

## Solution Assessment
