## Summary

Keep open. Current main still has pattern-based loop detection and a post-compaction guard, but no public global/per-agent hard turn or tool-call budget and no timeout-after-first-tool-call enforcement; the related open requests cover Ralph-specific, per-channel, or broader safeguard designs rather than superseding this focused request.

## What This Changes

Add global/per-agent `maxTurns`, `maxToolCalls`, and `toolCallTimeout` config to cap agent tool-loop iterations and force a response when exhausted.

## Best Possible Solution

Keep this as the canonical global/per-agent hard iteration budget request, and design an additive typed config plus runtime enforcement path that composes with existing loop detection and channel-specific budget work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, schema, docs, and related-issue inspection give a high-confidence absence check for the requested hard budget, but the specific production model loop was not replayed.

## Solution Assessment

Unclear. A hard global/per-agent budget is plausible, but maintainers need to choose the public config shape, enforcement point, timeout semantics, and final-response behavior before a narrow implementation is safe.
