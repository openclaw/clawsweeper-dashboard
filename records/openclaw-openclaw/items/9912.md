## Summary

Keep open. Current main and the latest shipped tag still lack a global/per-agent hard max-turn or max-tool-call budget; existing protections are loop-pattern detection, run retry bounds, and timeouts rather than the requested configurable iteration cap.

## What This Changes

Request to add agent-level config such as `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to cap tool-call iterations and force an agent response.

## Best Possible Solution

Keep this as the canonical global/per-agent hard iteration-budget request and design an additive typed config plus runtime enforcement path that composes with existing loop detection, retry limits, and scoped-budget issues.

Unclear. A hard global/per-agent budget is plausible, but maintainers need to choose the public config shape, enforcement point, timeout semantics, and final-response behavior before implementation is safe.
