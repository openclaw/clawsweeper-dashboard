Keep open: current main and the latest release still do not provide a global or per-agent hard budget for model-tool turns or total tool calls, and adding that config/runtime contract needs maintainer product design rather than cleanup closure.

## What This Changes

Request a global/per-agent `maxTurns`, `maxToolCalls`, and related timeout configuration to force an agent to stop tool-call iterations and deliver a response.

## Best Possible Solution

Define one typed opt-in agent-run budget policy with global defaults and per-agent overrides, enforce it before the next tool executes in the canonical runtime loop, and document deterministic terminal-response semantics with focused tests.
