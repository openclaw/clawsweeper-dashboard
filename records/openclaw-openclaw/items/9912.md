## Summary

Keep open. Current main still lacks the requested generic agent-level hard `maxTurns`, `maxToolCalls`, or `toolCallTimeout` budget, and the live discussion identifies this as the canonical tracker rather than a duplicate of the per-channel, Ralph-loop, or broader budget issues.

## What This Changes

Request an agent/defaults config surface to cap model-tool iterations, total tool calls, and post-tool response time so runaway tool loops are forced to stop and respond.

## Best Possible Solution

Define an additive agent-run budget contract under `agents.defaults` and `agents.list[]`, enforce counters in the trusted embedded/before-tool path before executing the next tool, document precedence and exhaustion behavior, and add focused config/runtime tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/config request, and source/docs inspection gives a high-confidence check that the generic agent-level hard turn/tool-call budget does not exist on current main.

## Solution Assessment

Unclear until maintainers choose the public contract. A defaults/per-agent hard budget is plausible, but field names, precedence, timeout behavior, and forced-response semantics need product/API decisions before implementation.
