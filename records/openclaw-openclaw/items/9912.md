## Summary

Keep open. Current main still lacks the requested global/per-agent hard turn or tool-call budget; the adjacent `maxTurns` hits are xAI/web-tool internals, loop detection is pattern-based, and OpenResponses `max_tool_calls` is documented as accepted but ignored.

## What This Changes

Request an agent-level `maxTurns`, `maxToolCalls`, and `toolCallTimeout` configuration surface to stop runaway tool-call iterations and force a final response.

## Best Possible Solution

Add a typed, documented defaults/per-agent agent-run budget enforced before the next tool execution, with explicit timeout and exhaustion behavior coordinated with existing loop detection and broader usage-budget work.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a config feature request, and source/docs inspection on current main verifies the generic agent-level hard turn/tool-call budget does not exist.

## Solution Assessment

Partly. A generic defaults/per-agent hard budget is a plausible maintainable direction, but the exact config contract, precedence, and forced-response behavior need maintainer API/product decisions before implementation.
