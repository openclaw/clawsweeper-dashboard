## Summary

Keep open. Current main still lacks a generic defaults/per-agent hard turn or tool-call budget; adjacent controls are pattern-based loop detection, whole-run timeouts, xAI tool-local `maxTurns`, and an OpenResponses `max_tool_calls` field that is documented as accepted but ignored.

## What This Changes

Request adds global/per-agent `maxTurns`, `maxToolCalls`, and `toolCallTimeout` configuration to cap agent tool-call iterations and force a final response after exhaustion.

## Best Possible Solution

Add a typed, documented defaults/per-agent run budget enforced before the next tool execution, with explicit exhaustion and timeout semantics plus focused config/runtime/docs tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/config request, and current-main source and docs inspection provides high-confidence evidence that the requested generic hard budget does not exist.

## Solution Assessment

Partly. A defaults/per-agent hard budget is a plausible core direction, but maintainers need to decide the public config contract, precedence rules, and exhaustion behavior before implementation.
