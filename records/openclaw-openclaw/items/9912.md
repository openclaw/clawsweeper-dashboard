## Summary

Keep open: current main still lacks a strict global/per-agent maxTurns, maxToolCalls, or toolCallTimeout control; the nearby safeguards are disabled-by-default pattern loop detection and an OpenResponses max_tool_calls field that is documented as ignored.

## What This Changes

Request a global/per-agent config option to cap agent model-tool turns, total tool calls, and timeout after the first tool call.

## Best Possible Solution

Add a typed global/per-agent hard iteration budget enforced before the next tool execution or continuation turn, document precedence with existing loop detection/timeouts, and keep per-channel delegation semantics tracked separately in #13271.

## Reproduction Assessment

Not applicable as a feature request rather than a broken existing contract. Source inspection gives high confidence that the requested config fields and counter-based enforcement are absent on current main.

## Solution Assessment

Unclear until maintainers settle the config/API shape and exhaustion semantics. A hard budget is plausible, but it needs product decisions before a safe implementation PR.
