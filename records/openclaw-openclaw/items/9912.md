## Summary

Current main and v2026.6.5 still lack a global or per-agent hard iteration budget; existing safeguards stop detected repetition or elapsed-time overruns, so the central request remains necessary but needs a maintainer product decision before implementation.

## What This Changes

Requests configurable per-agent or global limits on model/tool iterations, total tool calls, and time after the first tool call.

## Best Possible Solution

Define one typed run-budget contract at agent-default and per-agent scope, enforce it before the next tool execution, and specify counting, precedence, observability, and a graceful terminal outcome.

## Reproduction Assessment

Yes at source level: current main provides pattern-based loop guards and elapsed-time ceilings but no configured total model/tool iteration budget. The exact NVIDIA-hosted KIMI delay was not rerun live, but the missing enforcement path is clear.

## Solution Assessment

No, not in the exact three-knob form proposed. A unified budget model with explicit scope, counting, precedence, and terminal semantics is narrower and more maintainable than independent limits with overlapping behavior.
