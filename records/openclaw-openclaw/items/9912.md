## Summary

Keep open. Current main and the latest release still lack a global or per-agent hard max-turn/max-tool-call budget, and the existing loop detection, run retry limits, idle timeout, and OpenResponses fields do not force the requested exhaustion behavior.

## What This Changes

Request an agent-level config contract for `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to cap model-tool iteration loops and force a final response.

## Best Possible Solution

Design an additive `agents.defaults` plus per-agent hard-budget contract, enforce counters before the next tool execution, document precedence with existing guards and timeouts, and keep per-channel budget/autodelegation work separate.

## Solution Assessment

Unclear. A hard budget is plausible, but maintainers need to choose the public config shape, enforcement point, timeout-after-first-tool-call semantics, and final-response behavior before implementation is safe.
