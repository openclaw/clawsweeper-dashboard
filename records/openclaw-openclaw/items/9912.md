## Summary

Keep open. Current main and the latest shipped tag still lack the requested global/per-agent hard max-turn or max-tool-call budget; existing protection is pattern-based loop detection, retry limits, and whole-run/provider timeouts rather than a configurable iteration cap. Prior maintainer/bot review also kept this as the canonical global/per-agent request, separate from the per-channel budget request at https://github.com/openclaw/openclaw/issues/13271.

## What This Changes

The issue requests a global or per-agent config option to cap agent model-tool turns, total tool calls, and optional post-tool timeout behavior.

## Best Possible Solution

Keep this as the canonical global/per-agent hard iteration-budget request and design an additive typed config plus runtime enforcement path that composes with existing loop detection, retry limits, and related scoped-budget issues.

## Reproduction Assessment

Not applicable as a strict bug reproduction; the specific KIMI/NVIDIA loop was not replayed. Source, docs, release-tag, and related-issue checks give a high-confidence absence check for the requested hard budget on current main.

## Solution Assessment

Unclear. A hard global/per-agent budget is plausible, but maintainers need to choose the public config shape, enforcement point, timeout semantics, and final-response behavior before implementation is safe.
