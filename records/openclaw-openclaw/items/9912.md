## Summary

Keep open: current main still lacks a strict global/per-agent maxTurns, maxToolCalls, or toolCallTimeout control. The only nearby behavior is disabled-by-default pattern loop detection, and OpenResponses max_tool_calls is still accepted but documented as ignored.

## What This Changes

Request adds agent-level maxTurns, maxToolCalls, and toolCallTimeout configuration to cap LLM/tool iteration loops and force a response when exhausted.

## Best Possible Solution

Add a typed global/per-agent hard iteration budget, enforce it before the next tool execution or model-tool turn, document precedence with existing loop detection/timeouts, and keep per-channel delegation semantics in #13271.

## Reproduction Assessment

Not applicable as a feature request, not a broken existing contract. Source inspection gives high confidence the requested config fields and hard budget enforcement are absent on current main.

## Solution Assessment

Unclear until maintainer product decisions are made. A hard budget is plausible, but the exact exhaustion behavior and relationship to #13271 and #58826 should be settled before an automated implementation PR.
