## Summary

Keep open. Current main still lacks the requested global/per-agent hard max-turn or max-tool-call budget; existing safeguards are pattern-based or tool-specific, and related issue search shows this remains the canonical global budget request rather than a cleanup close.

## What This Changes

Request a config surface such as `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to cap agent tool iterations and force a final response when exhausted.

## Best Possible Solution

Design an additive `agents.defaults` plus per-agent hard budget contract, enforce counters before the next tool call, document precedence with existing guards/timeouts, and add focused exhaustion tests.
