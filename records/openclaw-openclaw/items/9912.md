## Summary

Keep open. Current main still lacks a typed global/per-agent hard turn or tool-call budget; the existing protections are whole-run timeouts, retry-loop guards, and optional pattern-based loop detection, which do not cover the requested configurable maxTurns/maxToolCalls behavior.

## What This Changes

The issue requests agent-level max turn, max tool-call, and post-tool timeout controls to stop runaway tool-calling loops.

## Best Possible Solution

Add a generic, documented agent-run budget under agents defaults and per-agent config, enforce it before the next tool execution, and keep per-channel overflow/delegation policy tracked separately.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature request, and source inspection confirms current main has no strict config or runtime path for the requested hard agent iteration budget.

## Solution Assessment

Partly. A generic agent-level hard budget is the right maintainable direction, but the exact config shape, scope, and final-response behavior need maintainer product/API decisions before implementation.
