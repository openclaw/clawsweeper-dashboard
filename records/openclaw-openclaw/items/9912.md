## Summary

Keep open. Current main still lacks a generic defaults/per-agent hard turn or tool-call budget; the only adjacent controls are whole-run timeouts, optional pattern-based loop detection, xAI tool-local `maxTurns`, and OpenResponses `max_tool_calls` parsing that is still documented as ignored.

## What This Changes

The issue requests `maxTurns`, `maxToolCalls`, and `toolCallTimeout` configuration to cap agent tool-iteration loops and force a response when exhausted.

## Best Possible Solution

Add a typed, documented defaults/per-agent agent-run budget enforced before the next tool execution, with explicit exhaustion semantics and coordination with channel and usage-budget policy work.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/config request, and current source/docs inspection confirms no generic agent-level hard turn or tool-call budget exists on main.

## Solution Assessment

Partly. A generic defaults/per-agent hard budget is a plausible maintainable direction, but the exact scope, precedence, and forced-response behavior need maintainer API/product decisions before implementation.
