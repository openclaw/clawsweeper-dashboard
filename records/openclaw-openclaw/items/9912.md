## Summary

Keep open. Current main and the latest shipped release still lack the requested global/per-agent hard max-turn or max-tool-call budget; existing protections are tool-specific `maxTurns`, pattern-based loop detection, run retry bounds, and timeouts, not the requested agent-run contract.

## What This Changes

Request an agent-level `maxTurns`/`maxToolCalls`/`toolCallTimeout` config surface that caps LLM-to-tool iteration loops and forces a bounded response path.

## Best Possible Solution

Keep this as the canonical global/per-agent hard iteration-budget request and design an additive typed config plus runtime enforcement path that composes with existing loop detection, retry limits, and scoped-budget follow-ups.

## Solution Assessment
