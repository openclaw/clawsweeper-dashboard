## Summary

Keep open. Current main and the latest shipped tag still lack a global/per-agent hard max-turn or max-tool-call budget; existing safeguards are pattern-based, tool-specific, or unrelated provider/tool settings, and related open items cover adjacent but separate scopes.

## What This Changes

The issue requests agent-level config options such as `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to cap runaway agent tool-call loops.

## Best Possible Solution

Design an additive `agents.defaults` plus per-agent hard budget contract, enforce counters before the next tool execution, document precedence with existing guards/timeouts, and add focused exhaustion tests.

## Reproduction Assessment

## Solution Assessment

Unclear. A hard budget is plausible, but maintainers need to choose the public config shape, enforcement point, timeout-after-first-tool-call semantics, and final-response behavior before implementation is safe.
