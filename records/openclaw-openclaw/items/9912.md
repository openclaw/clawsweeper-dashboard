## Summary

Keep open. Current main still lacks a global or per-agent hard max-turn/max-tool-call budget, and the existing loop detection, run retry, timeout, and OpenResponses fields are adjacent safeguards rather than the requested forced exhaustion behavior.

## What This Changes

Request to add agent-level `maxTurns`, `maxToolCalls`, and `toolCallTimeout` configuration that limits model-tool iterations and forces a response when exhausted.

## Best Possible Solution

Design an additive `agents.defaults` plus per-agent hard-budget contract, enforce counters before the next tool execution, document precedence with existing guards and timeouts, and keep per-channel budget/autodelegation work separate.
