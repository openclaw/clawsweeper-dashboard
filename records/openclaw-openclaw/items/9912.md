## Summary

Keep open. Current main still lacks the requested generic defaults/per-agent hard turn or tool-call budget; adjacent code only provides strict config schemas, pattern-based loop detection, provider/tool-specific `maxTurns`, and an OpenResponses `max_tool_calls` field that is accepted but documented as ignored.

## What This Changes

Add `maxTurns`, `maxToolCalls`, and `toolCallTimeout` style agent-level configuration to cap model-tool iterations and force a final response when the cap is reached.

## Best Possible Solution

Add a typed, documented defaults/per-agent agent-run budget enforced before the next tool execution, with explicit timeout and exhaustion behavior coordinated with existing loop detection and broader usage-budget work.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a config feature request, and source/docs inspection confirms the generic agent-level hard turn/tool-call budget does not exist on the inspected current-main blobs.

## Solution Assessment

Partly. A generic defaults/per-agent hard budget is a plausible maintainable direction, but the exact config contract, precedence, and forced-response behavior need maintainer API/product decisions before implementation.
