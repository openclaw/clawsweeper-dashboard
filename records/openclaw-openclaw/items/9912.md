Keep open: current main and v2026.6.10 still do not provide the requested global or per-agent hard budget for model-tool turns or total tool calls; existing loop detection, run retries, and OpenResponses compatibility fields cover adjacent but different behavior.

## What This Changes

The issue requests global/per-agent agent-run limits such as `maxTurns`, `maxToolCalls`, and `toolCallTimeout` to force a final response before runaway tool loops suppress channel replies.

## Best Possible Solution

Define one typed opt-in agent-run budget policy with global defaults and per-agent overrides, enforce it before the next tool executes in the canonical runtime loop, and document deterministic terminal-response semantics with focused tests.
