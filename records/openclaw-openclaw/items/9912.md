## Summary

Keep open. Current main still lacks the requested generic defaults/per-agent hard `maxTurns`, `maxToolCalls`, or `toolCallTimeout` budget; the adjacent loop detector, xAI-specific `maxTurns`, and OpenResponses `max_tool_calls` acceptance do not enforce this issue's requested agent-run limit.

## What This Changes

Request a global/defaults or per-agent config to cap model-tool turns, total tool calls, and post-tool waiting time before forcing a response.

## Best Possible Solution

Define an additive agent-run budget contract and enforce it in the trusted embedded/before-tool path with documented precedence, timeout semantics, exhaustion result shape, and focused config/runtime/docs tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a config feature request, and current-main source/docs inspection gives a high-confidence check that the generic agent-level hard turn/tool-call budget does not exist.

## Solution Assessment

Unclear until maintainers choose the public contract. A defaults/per-agent hard budget is a plausible direction, but the exact field names, precedence, timeout behavior, and forced-response semantics need product/API decisions before implementation.
