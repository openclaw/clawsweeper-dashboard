## Summary

Keep open. Current main does not implement the compact inbound metadata option, and this PR is a useful implementation candidate for the linked Telegram metadata reports, but it is not ready to merge because compact-direct mode drops the metadata source used to preserve visible sender labels in chat history and the branch lacks the required changelog entry.

## What This Changes

The PR adds optional `messages.inboundMetadataMode: "compact-direct"` config/schema/docs/tests, passes it into inbound prompt construction, strips compact direct metadata from history text, and also changes `wiki`/`memory` CLI loading to include the configured memory-slot plugin.

## Best Possible Solution

Land a revised compact-direct implementation that reduces direct-chat prompt noise while preserving user-visible sender attribution, keeps defaults and group/rich-context behavior unchanged, adds focused sanitizer coverage plus a changelog entry, and handles the memory CLI loader change as a separate maintainer-approved scope if needed.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to build a compact-direct direct-chat prompt at PR head, then sanitize a user message without an explicit `senderLabel`; the compact line lacks label/name fields and the existing sanitizer can only recover labels from the full JSON blocks.

## Solution Assessment

No, not as-is. The optional compact prompt direction is narrow and appropriate for #72704 and #62077, but the implementation should preserve sender labels out of band or add compact label extraction with gateway sanitizer coverage before merge.
