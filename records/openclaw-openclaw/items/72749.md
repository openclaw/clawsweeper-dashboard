## Summary

Keep open. The PR targets a real gap and current main still lacks compact inbound metadata, but the proposed patch drops visible sender attribution in compact-direct history and is missing the required user-facing changelog entry.

## What This Changes

The PR adds an optional `messages.inboundMetadataMode: "compact-direct"` prompt mode with config/schema/docs/tests, teaches metadata stripping about the compact line, and broadens wiki/memory CLI plugin loading to include the configured memory slot plugin.

## Best Possible Solution

Land a revised compact metadata implementation that reduces direct Telegram prompt noise while preserving sender attribution in displayed history, keeps default/group/rich-context behavior unchanged, documents the new option, and records the user-facing change in the active changelog.

## Reproduction Assessment

Yes. Build a compact-direct direct-chat prompt at PR head, pass a resulting user message without an explicit `senderLabel` through `stripEnvelopeFromMessage`, and the current sanitizer has no full sender/conversation JSON block to recover the label from.

## Solution Assessment

No, not as-is. The compact prompt direction is narrow and appropriate for #72704 and #62077, but the patch should preserve sender labels via an out-of-band label or a compact extraction path with a gateway sanitizer regression test, then add the required changelog entry.
