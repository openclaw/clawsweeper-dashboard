## Summary

Keep this PR open. Current main still emits full direct-chat inbound metadata and lacks the new config knob, so the PR targets a real linked UX problem, but the head patch has a concrete sender-label regression and is missing the required changelog entry for a user-facing config/behavior change.

## What This Changes

The PR adds optional `messages.inboundMetadataMode: "compact-direct"` config/schema/docs/tests to compact direct-chat inbound metadata, updates sanitizer stripping for the compact line, and also changes plugin CLI primary-command loading for `wiki`/`memory` commands.

## Best Possible Solution

Land a revised compact metadata implementation that reduces Telegram DM prompt noise while preserving visible sender attribution, keeps default and group/rich-context behavior unchanged, and separates the memory/wiki CLI loader behavior if maintainers do not want it coupled to this prompt-format change.

## Reproduction Assessment

Yes. Build a compact-direct direct Telegram-style prompt at PR head, pass it through `stripEnvelopeFromMessage` without an explicit `senderLabel`, and the compact line cannot recover a display label because the label source was removed and the extractor only parses the old fenced JSON blocks.

## Solution Assessment

No, not as-is. The config-based compact prompt direction is narrow and maintainable, but the patch must preserve sender attribution through an out-of-band field or a compact-safe extraction path, plus the required changelog entry, before merge.
