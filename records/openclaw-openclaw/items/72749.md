## Summary

Keep open. Current main still lacks compact inbound metadata, and this PR is a useful implementation candidate for #72704 and #62077, but the patch drops parseable sender label data in compact-direct history and needs the required changelog entry before merge.

## What This Changes

The PR adds optional `messages.inboundMetadataMode: "compact-direct"` config/schema/docs/tests, threads it into inbound prompt construction, strips compact direct metadata from history, and also adjusts wiki/memory CLI plugin loading to include the configured memory slot plugin.

## Best Possible Solution

Land a revised compact-direct implementation that reduces direct-chat prompt noise while preserving visible sender attribution, keeps defaults and group/rich-context behavior unchanged, updates schema/docs/changelog, and leaves the memory CLI loader change either explicitly approved or separated.

## Reproduction Assessment

Yes. Build a compact-direct direct Telegram prompt at PR head and then sanitize a user-role message without an explicit `senderLabel`; the compact line has no label/name fields, and the current sanitizer only recovers sender labels from the full JSON metadata blocks.

## Solution Assessment

No, not as-is. The compact-direct configuration is a narrow maintainable direction for the linked reports, but the patch should preserve sender attribution and add release-note coverage before merge.
