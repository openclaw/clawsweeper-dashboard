## Summary

Keep open: current main still lacks the compact inbound metadata mode, and this PR is a useful implementation candidate, but the current patch drops compact-direct sender attribution and is missing the required changelog entry.

## What This Changes

The PR adds an opt-in `messages.inboundMetadataMode: "compact-direct"` config/schema/docs/tests, routes it into inbound prompt construction, strips compact metadata in history views, and also changes `wiki`/`memory` CLI loading to include the configured memory-slot plugin.

## Best Possible Solution

Land a revised compact-direct implementation that reduces direct-chat prompt noise while preserving user-visible sender attribution, keeps defaults and group/rich-context behavior unchanged, adds focused sanitizer coverage plus a changelog entry, and separates the memory CLI loader change if maintainers want that handled independently.

## Reproduction Assessment

Yes. Build a compact-direct direct-chat prompt at PR head, then sanitize a user message without an explicit `senderLabel`; the compact line lacks label/name fields and the sanitizer can only recover labels from full fenced metadata blocks.

## Solution Assessment

No, not as-is. The compact prompt direction is narrow and appropriate for #72704 and #62077, but the implementation should preserve sender labels out of band or add compact label extraction with gateway sanitizer coverage before merge.
