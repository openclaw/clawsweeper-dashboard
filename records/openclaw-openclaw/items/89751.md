- **[P2] Normalize invite reply targets before streaming sends:** `extensions/feishu/src/monitor.vc-meeting-invited-handler.ts:267`
  - body: The handler passes `user:${target}` into `createFeishuReplyDispatcher` as `chatId`, but the streaming-card path sends that value directly as `receive_id` while only the normal send path strips `user:`/`chat:` prefixes. Pass a raw/normalized Feishu id to the dispatcher, or normalize before `FeishuStreamingSession.start`.
