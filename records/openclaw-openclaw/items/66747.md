## Summary

Keep open. Current main still has the default npm CLI entrypoint and dist bootstrap path implicated by the macOS provenance/shebang report, and the task plus Task Flow SQLite restore paths still call `chmodSync` unconditionally before opening the databases. The related wrapper work is a useful opt-in mitigation, and current main has newer empty-response heartbeat hardening, but the combined package/probe, chmod, and main-session heartbeat symptoms are not fully resolved enough to close.

## What This Changes

The issue requests fixes or investigation for macOS CLI gateway probe failures with WebSocket `1006`, task and Task Flow registry restore `EPERM` on `chmod`, and a later heartbeat/main-session `systemEvent` regression after `2026.4.25`.

## Best Possible Solution

Keep the report open until maintainers either split it into focused follow-ups or land a coordinated fix: default-safe macOS packaged CLI/LaunchAgent behavior or a clear provenance diagnostic, task and Task Flow registry restore hardening that tolerates benign chmod EPERM when existing modes are safe, and live/package validation for main-session heartbeat system events under the reported provider path.

## Reproduction Assessment

No. The issue provides high-signal macOS steps, versions, commands, logs, and source-matching symptoms, but this Linux checkout cannot high-confidence live-reproduce macOS 26 provenance behavior or the OpenRouter/Gemini heartbeat failure.

## Solution Assessment

No. The current wrapper support is a useful opt-in workaround and current main has some empty-response hardening, but neither is a complete fix for the default npm shebang path, registry chmod restore behavior, or the reported main-session heartbeat dependency chain.
