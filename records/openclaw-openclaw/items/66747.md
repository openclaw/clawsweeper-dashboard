Keep open: current main still preserves the reported risk surfaces, and the issue combines macOS packaging/probe behavior, registry restore semantics, and live provider heartbeat behavior that were not fully reproduced in this read-only review.

## What This Changes

The issue reports macOS CLI gateway probes returning abnormal WebSocket closure `1006`, task/task-flow registry restore warnings from `EPERM` on `chmod`, and a later heartbeat/main-session cron regression on macOS releases around 2026.4.14-2026.4.25.

## Best Possible Solution

Split this into focused maintainer-owned fixes: packaged macOS CLI/probe diagnostics or launcher changes, chmod `EPERM` tolerance after validating safe registry modes, and live-provider proof for main-session heartbeat/systemEvent execution.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available here. Source inspection does show that current main still logs task and task-flow restore failures if registry chmod throws `EPERM`, but the macOS package/probe path and live OpenRouter heartbeat failure were not reproduced.

## Solution Assessment

No. The current wrapper and heartbeat changes are useful mitigations, but the maintainable path is to split the packaging, registry chmod tolerance, and heartbeat/provider behavior into focused fixes with their own proof.
