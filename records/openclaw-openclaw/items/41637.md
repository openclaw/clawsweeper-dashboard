## Summary

Keep this PR open: it targets a real Feishu multi-account webhook bug, but the branch is stale against current main and would regress current webhook authentication/body-guard hardening and pooled-server lifecycle behavior before it is mergeable.

## What This Changes

The PR adds a Feishu webhook server pool keyed by host:port, routes shared-server requests by token/appId, and updates pooled monitor cleanup.

## Best Possible Solution

Land a refreshed Feishu-owned shared webhook server implementation that preserves current raw-body signature checks, 64 KiB / 5 s pre-auth guards, per-account lifecycle semantics, targeted multi-account tests, and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. Configure two Feishu webhook accounts with the same `webhookHost:webhookPort`; current main creates and listens on one `http.Server` per account, so the second listener has a clear `EADDRINUSE` path.

## Solution Assessment

No. Pooling by host:port is the right direction, but this PR head is not the best mergeable solution until it is rebased onto current main and preserves the existing webhook security and lifecycle contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve raw-body signature verification:** `extensions/feishu/src/monitor.transport.ts:45-69`
  - body: This branch verifies Feishu signatures from `JSON.stringify(params.payload)`, uses a direct string comparison, and returns valid when `encryptKey` is absent.   - confidence: 0.95
- **[P1] Reject joined monitors when the shared server errors:** `extensions/feishu/src/monitor.transport.ts:282-295`
  - body: The shared server `error` handler clears state for every pooled route but only rejects the creator monitor promise. Accounts that joined through `joinExistingServer` remain pending until abort, so they can stay marked running after their route and socket are gone.
  - confidence: 0.88
- **[P2] Keep the tightened pre-auth body limits:** `extensions/feishu/src/monitor.state.ts:31-32`
  - body: The PR head keeps the older 1 MiB / 30 s webhook limits, while current main uses 64 KiB / 5 s with regression coverage for oversized and slow unsigned bodies. The pooled buffering path should preserve those current limits before routing or replaying requests.
  - confidence: 0.93
