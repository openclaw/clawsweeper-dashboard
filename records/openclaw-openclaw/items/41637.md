- **[P1] Propagate shared-server errors to joined accounts:** `extensions/feishu/src/monitor.transport.ts:282-295`
  - body: When the shared server emits `error`, this handler clears state for every route but only rejects the creator account's monitor promise. Joined accounts can remain pending or marked running with no live route, so restart handling is skipped.
  - confidence: 0.9
- **[P1] Rate-limit unroutable pooled requests before buffering:** `extensions/feishu/src/monitor.transport.ts:428-433`
  - body: The pooled dispatcher calls `applyBasicWebhookRequestGuards` without a rate limiter before buffering, and route misses return 404 without ever hitting an account limiter. Invalid-token floods can consume socket and memory budget outside the current per-account throttling path.
  - confidence: 0.88
- **[P2] Disambiguate token-only pooled routing:** `extensions/feishu/src/monitor.transport.ts:519-522`
  - body: After token+appId matching fails, the token-only fallback returns the first matching route. Accounts can inherit a shared verification token, so payloads without app_id can be delivered to the wrong account or path.
  - confidence: 0.82
