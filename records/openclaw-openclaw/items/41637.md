## Summary

Keep open: the reported Feishu multi-account webhook bind problem is source-reproducible on current main, but this PR head is not mergeable as-is because it regresses current webhook security and lifecycle contracts and still lacks real behavior proof.

## What This Changes

The PR adds a Feishu webhook server pool keyed by host:port, per-account route registration, pooled request routing by token/appId, body replay, and pool-aware cleanup.

## Best Possible Solution

Land a refreshed Feishu-owned pooled-server fix that coalesces same host:port accounts while preserving raw-body signature verification, current pre-auth guards, deterministic account routing, per-account lifecycle/error semantics, focused regression coverage, and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main can start multiple webhook-mode Feishu accounts, and each account independently creates and listens an HTTP server on its resolved host:port, so shared defaults can produce `EADDRINUSE` without an external live run.

## Solution Assessment

No, not as currently proposed. Pooling by host:port is the right direction, but this PR head must be refreshed to preserve current webhook authentication, body limits, routing, and lifecycle contracts before it is the best mergeable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve raw-body signature verification:** `extensions/feishu/src/monitor.transport.ts:45-69`
  - body: This branch verifies signatures from parsed JSON with `JSON.stringify(payload)`, returns true when `encryptKey` is missing, and compares with `===`. Current main verifies the raw request body before parsing and uses `safeEqualSecret`, so signed non-canonical bodies and missing-key paths would regress.
  - confidence: 0.95
- **[P1] Propagate pooled server errors to joined accounts:** `extensions/feishu/src/monitor.transport.ts:282-295`
  - body: When the shared server emits `error`, this handler clears state for every route but only rejects the creator account's monitor promise. Joined accounts created by `joinExistingServer` can remain marked running with no live route or socket.
  - confidence: 0.9
- **[P2] Keep current pre-auth body limits:** `extensions/feishu/src/monitor.state.ts:31-32`
  - body: The PR head resets Feishu webhook limits to 1 MiB and 30 seconds, while current main uses 64 KiB and 5 seconds with coverage for oversized and slow unsigned bodies. The pooled buffering path should preserve the current pre-auth limits.
  - confidence: 0.93
- **[P2] Disambiguate token-only pooled routing:** `extensions/feishu/src/monitor.transport.ts:518-522`
  - body: The token-only fallback returns the first matching route before considering path or ambiguity. Accounts can inherit or share a verification token, so URL verification or token-only payloads can be delivered to the wrong account handler.
  - confidence: 0.8
- **[P2] Preserve malformed JSON response semantics:** `extensions/feishu/src/monitor.transport.ts:501-507`
  - body: `resolveRoute` returns `undefined` on JSON parse failure and the caller maps every miss to `404 Not Found`. A signed malformed body should remain a `400 Invalid JSON` response, matching the existing account handler behavior.
  - confidence: 0.76
