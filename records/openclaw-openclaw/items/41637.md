  - body: This new pooled path intentionally omits the rate limiter before body buffering and route lookup. Requests with invalid or unknown tokens can therefore consume the multi-account body buffer and JSON parse path without any per-account rate-limit key ever being charged.
  - confidence: 0.88

  - body: On a shared server error, this handler deletes state for every route but only rejects the creator account's monitor promise. Joiner promises created by joinExistingServer only resolve on abort, so they can remain running with no live route and no restart signal.
  - confidence: 0.87
- **[P2] Disambiguate token-only pooled routing:** `extensions/feishu/src/monitor.transport.ts:518-531`
  - body: The token-only fallback returns the first route with a matching token, and url_verification falls back to any route.
