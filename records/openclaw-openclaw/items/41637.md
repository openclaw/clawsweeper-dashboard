## Summary

Keep open. Current main still starts one Feishu webhook HTTP server per account, so the reported same-host:port collision remains valid, but this PR is not merge-ready because the branch is stale against current webhook security hardening and still has pooled lifecycle error-handling bugs.

## What This Changes

The PR adds a Feishu webhook server pool keyed by host:port so multiple accounts can share one listener and route requests by verification token or app_id.

## Best Possible Solution

Land a refreshed Feishu-owned pooled webhook listener that preserves current raw-body auth guards, has deterministic cleanup/error propagation, and includes multi-account lifecycle/security tests plus a changelog entry.

## Reproduction Assessment

Yes. Configure two Feishu webhook accounts with the same webhookHost:webhookPort, or both using the documented 127.0.0.1:3000 defaults; current main creates and listens a separate http.Server per account, so the second listener has a clear EADDRINUSE path.

## Solution Assessment

No. Pooling by host:port is the right narrow direction, but this branch should be rebuilt or carefully rebased on current main so it keeps raw-body signature verification, tightened pre-auth limits, and correct joined-account lifecycle semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve raw-body signature validation:** `extensions/feishu/src/monitor.transport.ts:45-69`
  - body: This branch verifies Feishu signatures from `JSON.stringify(payload)`, treats a missing `encryptKey` as valid, and compares strings directly. Current main verifies the exact raw body with `safeEqualSecret` before JSON parsing, so rebase the pool change onto that flow to avoid rejecting valid signed JSON variants and weakening fail-closed webhook auth.
  - confidence: 0.95
- **[P1] Reject joined monitor tasks on shared-server errors:** `extensions/feishu/src/monitor.transport.ts:282-295`
  - body: When the pooled server emits `error`, this handler clears state for every route but only rejects the creator account's monitor promise. Accounts that joined through `joinExistingServer()` can remain pending and marked running after their shared listener was torn down, so server errors need to settle or restart all joined routes.
  - confidence: 0.89
- **[P2] Keep the tightened pre-auth body limits:** `extensions/feishu/src/monitor.state.ts:31-32`
  - body: The branch restores the older 1 MiB / 30 s webhook body limits, while current main uses 64 KiB / 5 s and has regression coverage for oversized and slow unsigned bodies. The pooled buffering path must preserve the current limits before merge.
  - confidence: 0.93
