## Summary

Keep open: the PR targets a real Feishu multi-account webhook `EADDRINUSE` bug, but the branch is not mergeable as-is because it regresses current webhook authentication, pre-auth resource limits, routing/rate-limit behavior, and shared-server lifecycle handling while still lacking inspectable real behavior proof.

## What This Changes

The PR adds a Feishu webhook server pool keyed by host:port with per-account routes, shared cleanup, and multi-account body buffering/replay in `extensions/feishu/src/monitor.state.ts` and `extensions/feishu/src/monitor.transport.ts`.

## Best Possible Solution

Land a refreshed pooled-server implementation that preserves current raw-body auth, pre-auth limits, per-account rate-limit normalization, deterministic route/error semantics, focused tests, and real Feishu/Lark proof.

## Reproduction Assessment

Yes, by source inspection: current main creates and listens a separate Feishu webhook `http.Server` for each account, and Feishu startup can run multiple account monitors in one provider run. I did not run a live Feishu/Lark setup in this read-only review.

## Solution Assessment

No. Pooling by host:port is the right direction, but this PR is not the best mergeable form because it regresses current webhook auth, resource limits, rate limiting, routing, and lifecycle contracts.

  - confidence: 0.9

  - body: The pooled dispatcher calls `applyBasicWebhookRequestGuards` without a `rateLimiter` or key before it buffers the body.   - confidence: 0.86

  - body: The PR head resets Feishu webhook limits to 1 MiB and 30 seconds, while current main uses 64 KiB and 5 seconds with coverage for oversized and slow unsigned bodies. The pooled buffering path should preserve the current pre-auth resource limits.
  - confidence: 0.94

  - confidence: 0.86

  - body: The token-only fallback returns the first route with a matching token. In multi-account webhook mode, shared or inherited verification tokens can send a webhook to the wrong account handler when the payload lacks an app id.
