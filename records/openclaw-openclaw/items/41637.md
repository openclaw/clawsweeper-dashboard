## What This Changes

The PR adds a Feishu webhook server pool keyed by host:port and dispatches shared-server requests to account handlers by verification token or appId.

## Best Possible Solution

Land a refreshed pooled-server Feishu webhook fix that preserves raw-body signature auth, current pre-auth limits, per-account lifecycle/error settlement, unambiguous routing, focused tests, and real behavior proof.

Yes. Source inspection shows two enabled Feishu webhook accounts can both reach `monitorWebhook`, and each account currently creates and listens its own HTTP server on the resolved host:port; I did not run a live repro in this read-only review.

## Solution Assessment

No. Pooling by host:port is the right direction, but this PR head is not the best mergeable solution until it is rebased and preserves current webhook security, routing, and lifecycle contracts.

Full review comments:

- **[P1] Preserve raw-body signature verification:** `extensions/feishu/src/monitor.transport.ts:45-69`
  - body: This branch verifies signatures from `JSON.stringify(payload)` after JSON parsing, returns true when `encryptKey` is absent, and compares with `===`. Current main verifies the raw request body before parsing and uses `safeEqualSecret`, so signed non-canonical bodies and missing-key paths would regress.
  - confidence: 0.95
- **[P1] Propagate pooled server errors to joined accounts:** `extensions/feishu/src/monitor.transport.ts:282-295`
  - body: When the shared server emits `error`, this handler clears state for every route but only rejects the creator account's monitor promise. Joined accounts created by `joinExistingServer` can remain marked running with no live route or socket.
  - confidence: 0.88
- **[P2] Keep current pre-auth body limits:** `extensions/feishu/src/monitor.state.ts:31-32`
  - body: The PR head resets Feishu webhook limits to 1 MiB and 30 seconds, while current main uses 64 KiB and 5 seconds with coverage for oversized and slow unsigned bodies. The pooled buffering path should preserve the current pre-auth limits.
  - confidence: 0.93
- **[P2] Disambiguate token-only pooled routing:** `extensions/feishu/src/monitor.transport.ts:519-522`
  - body: The token-only fallback returns the first matching route before considering path or ambiguity. Accounts can inherit or share a verification token, so URL verification or token-only payloads can be delivered to the wrong account handler.
  - confidence: 0.8
- **[P2] Preserve malformed JSON response semantics:** `extensions/feishu/src/monitor.transport.ts:475-477`
  - body: `resolveRoute` returns `undefined` on JSON parse failure and the caller maps every miss to `404 Not Found`. A signed malformed body should remain a `400 Invalid JSON` response, matching the existing account handler behavior.
  - confidence: 0.76
