## Summary

Keep open: current main still lacks proactive owner notifications for DM pairing requests, and this PR is meaningful feature work but not mergeable as-is because it is stale against current hook/startup/outbound seams and lacks real behavior proof.

## What This Changes

Adds a `pairing.notify` config block, a `pairing:request` hook emitted for new pairing requests, owner notification dispatch, gateway registration, and focused tests.

## Best Possible Solution

Land a rebased, documented, maintainer-reviewed implementation that emits the current hook event, registers in the active post-attach gateway startup path, routes through the channel-keyed outbound seam, normalizes channel-specific targets, and keeps sanitized requester data outside agent context.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Source and docs on current main show only manual pairing request listing/approval, with no owner-notification config or hook.

## Solution Assessment

No, not as-is. The direction is plausible, but the branch needs a current-main rebase plus hook-type, startup-registration, outbound-dispatch, Discord-target, documentation, and real-proof fixes before it is the narrowest maintainable solution.
