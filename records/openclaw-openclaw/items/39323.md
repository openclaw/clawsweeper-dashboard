## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks proactive pairing-owner notifications, and the linked #38998 discussion shows the feature is useful, but the branch is stale against current hook/startup structure and the channel-keyed outbound contract.

## What This Changes

The PR adds a `pairing.notify` config block, a `pairing:request` internal hook, owner notification dispatch through configured channels, gateway registration, and pairing-notification tests.

## Best Possible Solution

Land a rebased, documented, maintainer-reviewed implementation that emits a current hook event, registers it in the active gateway sidecar path, uses the generic channel outbound seam, normalizes channel-specific targets, and keeps requester text sanitized outside agent context.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Source inspection confirms current main does not implement owner notifications, and the Discord target blocker is backed by current tests.

## Solution Assessment

No, not as-is. The feature direction is reasonable, but this branch needs a current-main rebase plus hook/startup/outbound/docs fixes before it is the narrowest maintainable solution.
