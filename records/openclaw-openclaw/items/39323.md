The branch is not mergeable as-is because it is stale against current main, has blocking hook/startup/outbound findings, and still lacks real behavior proof from a live pairing notification run.

The branch adds a `pairing.notify` config block, a `pairing:request` internal hook, owner notification dispatch, gateway registration, and pairing-notification tests.

Land a rebased, documented, maintainer-reviewed implementation that emits the current hook event, registers in the active post-attach gateway sidecar path, uses the generic channel-keyed outbound seam, normalizes channel-specific targets, and keeps requester text sanitized outside agent context.
