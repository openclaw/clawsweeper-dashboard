## Summary

Keep open. Current main still leaves the Mattermost message-tool send adapter incomplete: the handler only forwards `replyToId` and `params.media`, while the core action context and Mattermost sender already support scoped media roots/readers and thread routing.

## What This Changes

The issue asks for Mattermost message-tool send actions to honor explicit `threadId`, `filePath`/`path` media aliases, and scoped `mediaLocalRoots`.

## Best Possible Solution

Land one coordinated Mattermost-owned fix in `extensions/mattermost/src/channel.ts` that preserves `replyToId` precedence, accepts explicit `threadId` as the root-post fallback, resolves media aliases consistently, forwards scoped media roots/readers, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to dispatch a Mattermost `send` action with `threadId`, local media, and scoped media access; current main does not pass those fields from `handleAction` to `sendMessageMattermost`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Mattermost action adapter because core already supplies scoped media access and the Mattermost sender already consumes the downstream options.
