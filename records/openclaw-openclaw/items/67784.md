## Summary

Keep open. Current main still drops explicit Mattermost thread and scoped local-media fields in the message-tool send handler; the related open pull requests are partial fix candidates, not grounds to close this issue.

## What This Changes

The issue requests Mattermost message-tool sends to honor `threadId`, `filePath`/`path` media aliases, and scoped `mediaLocalRoots` for local file uploads.

## Best Possible Solution

Land one coordinated Mattermost-owned fix in `extensions/mattermost/src/channel.ts` that preserves `replyToId` precedence, accepts `threadId` as the fallback root post, resolves media aliases consistently, forwards `mediaLocalRoots`/`mediaReadFile`, and adds focused `handleAction` regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call Mattermost `handleAction` for `send` with `threadId`, `filePath` or `path`, and scoped media access; current main drops those fields before calling `sendMessageMattermost`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Mattermost action adapter because core already supplies scoped media access and the Mattermost sender already consumes those options.
