## Summary

Keep open. Current main still leaves the Mattermost `handleAction` send path incomplete for explicit thread replies and agent-scoped local media, and the related Mattermost PRs are open partial fixes rather than merged implementation.

## What This Changes

Mattermost message-tool sends should honor `threadId`, `filePath`/`path` media aliases, and `mediaLocalRoots` when uploading local files into threads.

## Best Possible Solution

Land or replace the partial Mattermost PRs with one coordinated handler fix that preserves reply precedence, resolves media aliases, forwards media roots/readers, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level: dispatch the Mattermost `handleAction` send path with `threadId`, `filePath` or `path`, and scoped media fields, and current main will not forward the complete set to `sendMessageMattermost`. I did not run tests because this review had to keep the checkout read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is in `extensions/mattermost/src/channel.ts` plus focused `channel.test.ts` coverage because core dispatch and the Mattermost sender already expose the needed contract.
