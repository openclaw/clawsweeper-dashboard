## What This Changes

`doctor --fix` should recover when an orphan local plugin directory under `.openclaw/extensions` lacks `openclaw.plugin.json` and blocks config validation or gateway startup.

## Best Possible Solution

Land or replace the open canonical PR with the typed missing-manifest reason approach, preserving hard errors for config/global candidates and unsafe, parse, or invalid manifests while making opportunistic workspace not-found diagnostics non-blocking.

## Reproduction Assessment

Yes. The issue body gives concrete steps, and current-main source shows the same path: workspace discovery can create a candidate, manifest loading reports a missing manifest as an error, and config validation turns that into an invalid config issue.

## Solution Assessment

Yes. The canonical PR’s typed failure reason is the narrow maintainable solution because it avoids string matching and leaves real configured or unsafe plugin failures as hard errors.
