## Summary

Keep open. Current main has prompt/context and gateway-write hardening, but the model-facing owner gateway tool and RPC schemas still allow restart, allowed config writes, and update runs without a recent explicit-consent artifact.

## What This Changes

The issue asks OpenClaw to preserve core prompt context and prevent model-initiated config changes or restarts without explicit user consent during long conversations.

## Best Possible Solution

Add a server-enforced, recent explicit-consent mechanism for model-initiated gateway restart, config.apply, config.patch, and update.run while preserving the existing owner-only exposure, admin scopes, path allowlist, dangerous-flag checks, base-hash validation, rate limiting, and post-compaction context refresh.

## Reproduction Assessment

Unclear for the reporter's exact long-session model behavior because no transcript or logs were provided. Source inspection does give a high-confidence path for the remaining consent gap: current main exposes restart, config.apply/config.patch, and update.run without approval or consent parameters.

## Solution Assessment

No, the current implementation is not the best complete solution because prompt reminders, owner-only tool exposure, admin scopes, allowlists, and rate limits do not prove fresh user consent. The maintainable direction is a hard approval/consent contract enforced at the gateway tool and server protocol layers.
