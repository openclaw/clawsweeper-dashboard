## Summary

Keep open. Current main has context-refresh and fail-closed config-write hardening, but the model-facing gateway tool and RPC schemas still allow restart, allowlisted config writes, and update runs without a server-verifiable fresh consent artifact.

## What This Changes

Reporter asks OpenClaw to preserve core context across long conversations and prevent agents from modifying configuration or restarting without explicit user consent.

## Best Possible Solution

Add a gateway-enforced recent explicit-consent mechanism for model-initiated restart, config.apply, config.patch, and update.run while preserving owner-only exposure, admin scopes, allowlisted config paths, dangerous-flag checks, base-hash validation, rate limits, and post-compaction context refresh.

## Reproduction Assessment

Unclear for the reporter's exact long-session model behavior because no transcript or logs were provided. Yes for the remaining consent gap by source inspection: current main exposes restart, config.apply/config.patch, and update.run without an approval or consent parameter.

## Solution Assessment

No, the current implementation is not the complete fix because prompt reminders, owner-only tool exposure, admin scopes, allowlists, and rate limits do not prove fresh user consent. The narrower maintainable direction is a server-verifiable approval contract at the gateway tool and RPC layers.
