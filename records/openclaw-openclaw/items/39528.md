## Summary

Keep open. Current main includes useful context-bootstrap and fail-closed config guard mitigations, but model-facing gateway restart/update/config actions and related raw exec self-management paths still lack a hard fresh-consent boundary.

## What This Changes

The issue asks OpenClaw to preserve core context in long conversations and prevent model-initiated config edits or gateway restarts without explicit user consent.

## Best Possible Solution

Define and enforce a shared fresh-consent contract for model-initiated gateway restart, update.run, config writes, and OpenClaw self-management shell commands, with protocol fields and tests that keep prompt guidance as advisory only.

## Reproduction Assessment

Unclear for the reporter's exact long multi-turn context loss because no transcript, config, or logs were attached. Source inspection does show a current-main path where model-facing config/restart/update actions and raw host exec full/off behavior lack a hard fresh-consent artifact.

## Solution Assessment

No, current main is not a complete solution. The bootstrap and fail-closed config guards are good mitigations, but the maintainable fix is a protocol/tool-policy consent boundary plus coordinated raw exec self-management rules, not more prompt wording.
