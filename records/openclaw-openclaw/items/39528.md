## Summary

Keep open. Current main has useful bootstrap-context and config-write hardening, but model-facing gateway restart, update, and allowlisted config writes still lack a server-verifiable fresh-consent boundary; the related raw-exec issues remain open and do not fully supersede this gateway-control-plane gap.

## What This Changes

The issue asks OpenClaw to preserve core prompt context during long conversations and prevent model-initiated config edits or restarts without explicit user consent.

## Best Possible Solution

Define and enforce a fresh consent or approval artifact for model-initiated gateway restart, update.run, and config writes, while keeping the current bootstrap-context mitigations and coordinating raw-exec policy through the related open issues.

## Reproduction Assessment

Unclear for the reporter's exact long-session behavior because no transcript or logs were attached. Source inspection does show a current-main path where model-facing gateway restart, update.run, and allowlisted config writes can proceed without a consent artifact.

## Solution Assessment

No, current main is not a complete fix. Prompt guidance, bootstrap refresh, and config allowlists are useful mitigations, but the maintainable solution is a gateway/protocol consent boundary rather than relying on model obedience alone.
