## Summary

Keep open. Current main has stronger bootstrap-context injection and fail-closed config mutation allowlists, but model-facing gateway restart, update, and allowlisted config-write paths still lack a hard fresh-consent boundary.

## What This Changes

The issue asks OpenClaw to preserve soul/user/memory context across long conversations and prevent model-initiated config edits or gateway restarts without explicit user consent.

## Best Possible Solution

Define and enforce a shared fresh-consent contract for model-initiated gateway restart, update.run, config writes, and related raw-exec self-management commands while preserving the shipped bootstrap-context and config-allowlist hardening.

## Reproduction Assessment

Unclear for the reporter's exact long-session behavior because no transcript or logs were provided. Source inspection does show a current-main path where model-facing restart, update.run, allowlisted config writes, and full/off raw exec lack a fresh consent artifact.

## Solution Assessment

No, current main is not a complete fix. The allowlist and bootstrap-context changes are useful mitigation, but the maintainable solution is a gateway/protocol consent boundary plus coordinated raw-exec policy, not prompt guidance alone.
