## Summary

Keep open. Current main has shipped useful bootstrap/context and fail-closed config-write hardening, but model-facing gateway restart/update actions and allowlisted config writes still lack a server-verifiable fresh-consent boundary.

## What This Changes

The issue asks OpenClaw to preserve core context through long conversations and prevent model-initiated config edits or gateway restarts without explicit user consent.

## Best Possible Solution

Define and enforce a fresh user-consent contract for model-initiated gateway restart, update.run, and config writes, while coordinating the remaining context and raw-exec work through the linked open items.

## Reproduction Assessment

Unclear for the reporter's exact long-session behavior because no transcript or logs were provided. Source inspection does show a current-main path where model-facing restart, update.run, and allowlisted config writes have no consent artifact to validate.

## Solution Assessment

No, current main is not a complete fix. The allowlist and bootstrap-context changes are useful mitigation, but the maintainable solution is a gateway/protocol consent boundary rather than prompt guidance alone.
