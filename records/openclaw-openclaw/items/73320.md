## Summary

Keep open: the core problem still exists on current main and this branch targets it, but the PR head is conflicting/stale against current prompt-data hardening and still lacks required real behavior proof.

## What This Changes

The PR adds a fixed 12,000-character head/tail cap for task-completion internal event results, plus focused formatter, subagent announce, session search, and provider catalog test updates.

## Best Possible Solution

Rebase onto current main, apply the result cap at the event construction/delivery boundary while preserving `wrapPromptDataBlock`, keep the focused coverage, and land only after redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main sends a long child completion into `internalEvents[0].result`, then through prompt formatting and gateway serialization without a dedicated cap. I did not run a live reproduction because this was a read-only review.

## Solution Assessment

Mostly yes after rebase. Capping before requester-agent delivery is the right maintainable boundary, but the patch must be adapted to current main’s prompt-data wrapper and backed by real behavior proof.
