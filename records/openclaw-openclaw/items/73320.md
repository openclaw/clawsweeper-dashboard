## Summary

Keep open: current main still lacks a durable task-completion result cap, and this PR targets the right boundary, but it is not merge-ready because it lacks contributor real-behavior proof and its stale head can regress current prompt-data wrapping during rebase.

## What This Changes

The PR adds a fixed head/tail cap for subagent task-completion results before parent-agent delivery, adds formatter/announce/session-search coverage, and updates provider catalog test expectations.

## Best Possible Solution

Rebase onto current main, apply the result cap at event construction/delivery while preserving current prompt-data wrapping, keep the origin-label and exact catalog test fixes, then land after focused tests and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a large child completion flows into `internalEvents[0].result`, through prompt formatting and gateway request serialization, without a dedicated task-completion cap.

## Solution Assessment

Mostly yes, after rebase. Bounding the event result before delivery is the right maintainable boundary, but the PR must be adapted to current main’s prompt-data wrapper and backed by real behavior proof before merge.
