## Summary

Keep open: the payload-limit direction is useful, but this PR still caps too late because direct and queued gateway delivery carry the raw internal event result before prompt formatting. It also contains stale provider catalog test edits that weaken exact duplicate detection, and it needs contributor-supplied real behavior proof before merge.

## What This Changes

The PR adds formatter-side head/tail truncation for task-completion internal event results plus tests, and changes Codex provider catalog test expectations.

## Best Possible Solution

Bound `AgentInternalEvent.result` before it enters queued or direct delivery, keep formatter truncation as defense in depth, preserve full child output in child session history, and drop the stale provider catalog test edits.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a large child completion becomes `internalEvents[0].result`, is forwarded through queued/direct gateway delivery, and is serialized before this PR's formatter-only cap runs.

## Solution Assessment

No. The cap belongs at the event construction or delivery boundary so the transport/runtime payload is bounded; the provider test edits should be removed because current main already tracks the Codex mini row exactly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
