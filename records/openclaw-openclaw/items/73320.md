## Summary

## What This Changes

The PR bounds subagent task-completion result text with a fixed head/tail cap before delivering it to the requester agent, adds prompt/delivery coverage, and includes small session-search/provider-test updates.

## Best Possible Solution

Rebase the PR onto current main, cap the task-completion result before requester-agent delivery while preserving current prompt-data wrapping, and land only after focused tests plus redacted real-run proof of a large subagent completion.

## Reproduction Assessment

Yes. Source inspection on current main shows a long child completion can flow into `internalEvents[0].result`, through requester-agent delivery, prompt rendering, and gateway serialization without a dedicated task-completion cap.

## Solution Assessment

Mostly yes, after rebase. Bounding at event construction/delivery is the right narrow boundary, but the implementation must be adapted to current main’s prompt-data wrapper and backed by real behavior proof before merge.
