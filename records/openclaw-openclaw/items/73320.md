## Summary

Keep open. The PR targets a valid oversized subagent-result problem, but the current diff caps only prompt rendering while the raw `internalEvents` payload still travels through queued/direct gateway delivery; it also carries stale provider test loosening and lacks the required changelog entry.

## What This Changes

The PR adds a 12,000-character head/tail cap for internal task-completion prompt formatting, adds formatter coverage, and changes Codex provider catalog test expectations.

## Best Possible Solution

Land a focused version that bounds `AgentInternalEvent.result` before queued/direct gateway delivery, keeps formatter truncation as defense in depth, preserves full output in child session history, drops the stale provider test changes, and adds the changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: produce a large subagent completion result, watch it become `internalEvents[0].result`, and observe queued/direct gateway `agent` calls serialize that raw event before prompt formatting.

## Solution Assessment

No. The formatter cap is useful but too late for the runtime/transport payload; the narrow maintainable solution is to cap the event result before it is delivered or queued, then render from that bounded value.
