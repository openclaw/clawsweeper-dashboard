## Summary

Keep open. The PR addresses the related oversized subagent completion problem, but the current patch caps only prompt rendering while the raw `internalEvents` payload can still be forwarded through gateway delivery and JSON serialization; it also carries stale/unrelated provider catalog test changes and lacks the required changelog entry. Greptile's earlier `session-utils.ts` concern appears to have been against an older head because the current provided pull-file list no longer includes that file.

## What This Changes

The PR adds a fixed 12,000-character head/tail cap for rendered task-completion internal-event prompts, adds formatter tests, and loosens/updates Codex provider catalog test expectations.

## Best Possible Solution

Land a focused version that bounds task-completion result text before it is placed on or delivered as an internal event, renders prompts from that bounded copy, preserves head/tail context with an explicit truncation marker, restores exact provider catalog assertions, and adds the required user-facing changelog entry.

## Reproduction Assessment

Yes. A code-level reproduction is clear on current main: large child output flows into `internalEvents[0].result`, then both the formatter path and gateway request path can carry it without a size bound.

## Solution Assessment

No. The current PR caps rendered prompt strings, but the safer maintainable fix is to cap the internal event itself before queue/direct gateway delivery and keep the formatter cap as defense in depth.
