## Summary

Keep open. The PR addresses the right subagent-result growth problem, and related issue #57148 remains open for that behavior, but the current patch still bounds only the formatted prompt text while raw `internalEvents` continue through direct and queued gateway delivery. The PR also carries stale provider catalog test loosening and lacks the required user-facing changelog entry.

## What This Changes

The PR adds a fixed 12,000-character head/tail cap for task-completion prompt formatting, adds internal-event formatter tests, and changes Codex provider catalog test expectations/helpers.

## Best Possible Solution

Land a focused version that bounds task-completion result text before it is stored on or delivered as an `AgentInternalEvent`, keeps formatter truncation as defense in depth, preserves full child output in child history, removes the stale provider catalog test loosening, and adds the required changelog entry. After that merges, #57148 can be closed as implemented.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a large child reply or `childCompletionFindings` value flows into `internalEvents[0].result`, then into direct/queued gateway `agent` requests and prompt formatting without a current-main cap.

## Solution Assessment

No, not as currently patched. A formatter cap is useful but too late for the transport/runtime payload; the narrow maintainable fix is to bound the result at event construction or delivery and render from that bounded value.
