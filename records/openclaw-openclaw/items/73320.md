## Summary

Keep open: the PR targets a real oversized subagent-result problem, but the current patch caps only rendered prompt text while queued and direct gateway delivery can still carry the full `internalEvents` payload; it also weakens an unrelated provider catalog assertion and lacks the required changelog entry.

## What This Changes

The PR adds formatter-level head/tail truncation for task-completion internal-event results, adds prompt-formatting tests, and changes Codex provider catalog test expectations.

## Best Possible Solution

Land a focused agents fix that bounds task-completion internal-event results before queue/direct delivery while preserving full child session history, keeps exact provider catalog tests, and documents the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: produce a large child completion, observe it assigned to `internalEvents[0].result`, and follow queued/direct delivery into a serialized gateway `agent` request before the PR's formatter cap would run.

## Solution Assessment

No. Formatter truncation is useful as defense in depth, but the maintainable fix is to cap the event result at construction or delivery and render from that bounded value while leaving unrelated provider catalog tests exact.
