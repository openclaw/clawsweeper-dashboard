## Summary

Keep open. The reported problem is real on current main and the PR is useful source work, but the current patch is still incomplete: it bounds prompt rendering, while the raw `internalEvents` payload can still be queued and JSON-serialized for gateway delivery, and it also carries a stale provider-catalog test loosening plus no required changelog entry.

## What This Changes

The PR adds a 12,000-character head/tail cap for task-completion prompt formatting, adds internal-event formatter tests, and adjusts Codex provider catalog test expectations/helpers.

## Best Possible Solution

Land a focused version that bounds task-completion result text before it is stored on or delivered as an `AgentInternalEvent`, keeps prompt formatting bounded as defense in depth, preserves full output in child session history, removes the stale provider catalog test changes, and adds the required changelog entry. After that merges, #57148 can be closed as implemented.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, a large `task_completion.result` is assigned to `internalEvents`, passed through direct/queued announce delivery, and serialized by the gateway client without a size cap.

## Solution Assessment

No, not as currently patched. The formatter cap is useful but too late for the transport/runtime payload; the narrow maintainable fix is to bound the result before event delivery and then render from the bounded value.
