## Summary

Keep open: current main still lacks a task-completion result cap, and this PR targets the right code path, but the current head is conflicting, lacks contributor real-behavior proof, and must preserve current prompt-data wrapping during rebase.

## What This Changes

The PR adds a 12,000-character head/tail cap for task-completion internal event results, covers formatter and subagent announce delivery, preserves origin-label session search, and updates provider catalog test expectations.

## Best Possible Solution

Rebase the PR onto current main, apply the result cap at the task-completion event/delivery boundary while keeping `wrapPromptDataBlock`, then land after focused tests and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a long child completion can flow into `internalEvents[0].result`, direct announce delivery, gateway frame serialization, and prompt formatting without a dedicated cap.

## Solution Assessment

Mostly yes, after rebase. Bounding the delivered task-completion event is the right maintainable boundary, but the PR must keep current prompt-data block escaping and provide real behavior proof before merge.
