## Summary

Keep this PR open for the protected maintainer/automerge flow, but do not treat it as merge-ready. The patch addresses a real gateway memory problem, yet the current head still routes parent-linked transcripts through whole-file SessionManager loading in the new bounded helper path.

## What This Changes

The PR changes gateway session transcript readers, session/history/artifact handlers, injected transcript append behavior, tests, and the changelog to reduce oversized transcript materialization in session hot paths.

## Best Possible Solution

Finish the PR by keeping the bounded append-only strategy while replacing whole-file SessionManager reads in count, tail, recent-message, artifact, subscription, and first-page history paths with streaming logic that preserves parent-linked branch semantics where each caller actually needs them.

## Reproduction Assessment

Yes. A large v3 JSONL transcript with non-session entries carrying `id` and `parentId` will take the PR's `SessionManager.open(filePath).getBranch()` path through `visitSessionMessages`, and the dependency source shows that opening the file reads the whole transcript.

## Solution Assessment

No. The direction is useful, but the current patch is not the narrowest complete fix because parent-linked transcripts still use an unbounded dependency path in helpers that now back memory-sensitive gateway surfaces.
