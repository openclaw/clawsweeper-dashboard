## Summary

Keep this PR open. It is a draft maintainer-labeled PR from a member, so it should not be auto-closed, and the patch is useful but not merge-ready because the new bounded helpers still route modern parent-linked session transcripts through `SessionManager.open(...).getBranch()`, which materializes the transcript before the advertised hot-path cap can help.

## What This Changes

The PR streams or counts gateway session transcript reads across session detail, send, compaction, artifacts, HTTP/SSE history, and subscription paths, adds an oversized append-only `chat.inject` raw-append fallback, regression tests, and a changelog entry.

## Best Possible Solution

Finish the PR by making the bounded count/tail/read paths handle current parent-linked transcripts without whole-file `SessionManager` materialization where branch context is not required, while preserving `SessionManager` writes and true branch semantics for operations that need them. Add a parentId-based large-transcript stress regression so the fix covers the documented transcript format.

## Reproduction Assessment

Yes. A large v3 parent-linked JSONL transcript passed through `sessions.get`, first-page HTTP history, `sessions.send` sequence calculation, or subscription sequence calculation will enter `readRecentSessionMessagesWithStats`/`readSessionMessageCount` and still hit `SessionManager.open(filePath).getBranch()`.

## Solution Assessment

No. The proposed direction is right for append-only transcripts, but the implementation is not the narrowest complete fix because normal parent-linked transcripts still use the dependency path that reads and parses the whole file.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Avoid SessionManager in bounded transcript hot paths:** `src/gateway/session-utils.fs.ts:301`
  - body: This helper is now used by `sessions.get`, history refreshes, send sequence counting, and subscription broadcasts, but parent-linked transcripts still go through `SessionManager.open(filePath).getBranch()`. `SessionManager.open` loads the whole JSONL file before returning a branch, so current v3 transcripts can still OOM on the same hot paths this PR is meant to bound. Please stream/count parent-linked entries for the callers that only need counts or recent tail messages, or keep this path only for operations that truly need full branch traversal.
