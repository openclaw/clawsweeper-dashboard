## Summary

Keep this PR open. The goal is valid and the diff moves several append-only transcript paths toward bounded reads, but normal current transcripts are parent-linked trees and the new bounded helpers still route those files through whole-file `SessionManager` materialization, so the patch is not merge-ready for the hot paths it claims to fix.

## What This Changes

The PR streams or counts gateway session transcript access across session detail, send, compaction, artifacts, HTTP/SSE history, and subscription paths, adds an oversized append-only `chat.inject` raw-append fallback, regression tests, and a changelog entry.

## Best Possible Solution

Finish this PR by making the bounded count, tail, recent-message, and artifact-visit helpers handle current parent-linked transcripts without whole-file `SessionManager` materialization, while keeping `SessionManager` writes and full branch semantics for operations that truly need them. Add large parent-linked transcript regressions for each claimed hot path before automerge.

## Reproduction Assessment

Yes. A large v3 JSONL transcript with `id`/`parentId` entries passed through `sessions.get`, first-page HTTP/SSE history, `sessions.send` sequence calculation, subscription sequence calculation, or `artifacts.list` will hit `visitSessionMessages` and then `SessionManager.open(filePath).getBranch()`.

## Solution Assessment

No. The PR direction is right for append-only transcript fallbacks, but it is not the narrowest complete fix because the documented current transcript format still uses the whole-file dependency path in the new bounded helpers.
