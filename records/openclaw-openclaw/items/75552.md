## Summary

Keep this PR open but blocked from automerge. The current head still implements an explicit raw-append path for oversized append-only transcripts, and the shared bounded traversal still routes parent-linked transcripts through `SessionManager.open(...).getBranch()`, so the patch does not yet deliver the SessionManager-compatible bounded transcript behavior it is trying to add. The `maintainer` label and MEMBER author association also mean this PR needs explicit maintainer/domain handling rather than cleanup closure.

## What This Changes

The PR adds bounded session transcript helpers and uses them across Gateway session detail, HTTP/SSE history, artifacts, send/subscribe sequencing, compaction-related reads, and `chat.inject`, with targeted tests and a changelog entry.

## Best Possible Solution

Keep the bounded-read direction, but land it only after the transcript traversal and injected-write design preserves SessionManager parent chains while avoiding full-file materialization on current-format large transcripts. Legacy append-only fallback behavior should be narrowly justified, tested, and isolated from parent-linked transcripts.

## Reproduction Assessment

Yes. The patch-level regression is reproducible from the added raw-append test and guardrail change, and the remaining hot path can be reasoned from the added `SessionManager.open(filePath).getBranch()` call on parent-linked transcripts.

## Solution Assessment

No. The PR is aimed at the right Gateway hot paths, but the current approach is not the best fix because it weakens the parentId write guardrail and still relies on full SessionManager loading for tree transcripts; a bounded SessionManager-compatible reader/writer seam is safer.
