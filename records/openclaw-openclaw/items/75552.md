## Summary

Keep this PR open: the change is useful and current main still has the unbounded gateway transcript hot paths, but the current PR head still routes parent-linked transcripts through whole-file SessionManager reads in the new bounded helper path.

## What This Changes

The PR streams or tails gateway session transcript access for session detail, send, compact, HTTP/SSE history, artifact listing, subscription broadcasts, chat.inject append behavior, regression tests, and the changelog.

## Best Possible Solution

Keep the bounded append-only strategy, but split the helper contracts so count, tail, recent-message, artifact, subscription, and first-page history paths use streaming logic for parent-linked transcripts while reserving full SessionManager branch traversal only for callers that truly need complete branch semantics.

## Reproduction Assessment

Yes. A large v3 JSONL transcript with non-session entries carrying `id` and `parentId` will take the PR's `visitSessionMessages` tree branch, and the pinned dependency source shows `SessionManager.open()` reads the entire transcript before returning a branch.

## Solution Assessment

No. The direction is right, but the current patch is not the best complete fix because the shared bounded helper still has an unbounded parent-linked path; the safer alternative is separate streaming count/tail traversal with focused parent-linked stress coverage.
