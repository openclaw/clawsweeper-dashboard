## Summary

Keep this PR open but do not merge the current head. The PR is protected by MEMBER authorship and the `maintainer` label, and the diff still introduces two functional blockers in the gateway transcript hot paths: raw `chat.inject` transcript writes without `parentId`, and parent-linked transcript reads that still materialize the whole session through `SessionManager.open()`.

## What This Changes

The PR replaces several gateway session transcript reads with streaming/count/tail helpers, changes oversized `chat.inject` transcripts to raw append, updates history/artifact/compaction/send/subscription paths and tests, and adds a changelog entry.

## Best Possible Solution

Keep the bounded transcript-read direction, but implement it with a SessionManager-compatible design: injected writes must preserve the parent chain, and count, tail, first-page history, artifact scans, send sequence, subscription broadcasts, and manual compaction tails should use bounded JSONL traversal that does not full-load current-format parent-linked transcripts unless a caller explicitly needs complete branch context.

## Reproduction Assessment

Yes. The raw-append regression is reproducible by the existing guardrail assertions for `chat.inject` parentId writes, and the remaining OOM path is reproducible from code by using a large current-format v3 JSONL transcript with `id`/`parentId` entries and calling the new shared count/visit/history paths.

## Solution Assessment

No. The PR targets the right hot paths, but the current implementation reintroduces a forbidden raw Pi message append and still sends parent-linked transcripts through `SessionManager.open()`, so it is not the narrowest maintainable fix.
