## What This Changes

Fix isolated cron media workflows so `sessions_yield` pauses cleanly and the generated-media callback resumes the same session for remaining actions.

## Best Possible Solution

Consolidate on one canonical PR that uses authoritative cooperative-yield state for both wait and terminal-lifecycle decisions, preserves the isolated session through callback resumption, and proves ordinary waits, follow-up execution, fallback delivery, and delete-after-run cleanup together.

## Reproduction Assessment

Yes, at source level with high confidence. `sessions_yield` aborts the attempt signal with reason `sessions_yield`, and the current cron media wait immediately converts that already-aborted signal into the reported `AbortError: aborted`.

## Solution Assessment

Yes on the core direction, but the two proposed implementations should be consolidated. The narrow maintainable fix must classify authoritative yield state before waiting and prove that callback resumption and session cleanup remain correct, rather than merely suppressing the error.
