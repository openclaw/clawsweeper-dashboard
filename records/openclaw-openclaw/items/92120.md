## Summary

Current main still has the source path that can turn an intentional `sessions_yield` interruption into an aborted isolated cron run, so this issue remains actionable rather than fixed by the earlier cron-media wait change.

## What This Changes

The issue reports an isolated cron `agentTurn` workflow where `image_generate` plus `sessions_yield` delivers the callback but leaves the cron runner in `AbortError` and prevents the resumed follow-up step.

## Best Possible Solution

Teach isolated cron `agentTurn` runs to treat an intentional `sessions_yield` interruption as a resumable handoff instead of a failed terminal run, while preserving real abort/cancel semantics and the existing media-completion wait.

## Reproduction Assessment

Yes from source inspection: `sessions_yield` queues a continuation and aborts the active turn, the embedded runner maps that yield interrupt to an aborted result without terminal output, and isolated cron persists non-ok results as errors. I did not run the live cron/provider scenario in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow lifecycle repair in the embedded-runner/isolated-cron handoff rather than another media-tool workaround; the media completion fix from https://github.com/openclaw/openclaw/pull/88695 covers a different path.

## Repair Work Prompt

Fix the isolated cron `agentTurn` lifecycle regression reported at https://github.com/openclaw/openclaw/issues/92120: when a run calls `image_generate`, then explicitly calls `sessions_yield`, and the callback is delivered later, the cron runner must not record the intentional yield interruption as `AbortError: aborted` or kill the resumable session. Keep the fix inside the shared agent-run/cron lifecycle boundary, likely around `src/agents/session-turn-interrupt.ts`, `src/agents/embedded-agent-runner/run/attempt.ts`, `src/agents/agent-run-terminal-outcome.ts`, and `src/cron/isolated-agent-runner.ts`; do not add a config option, provider-specific workaround, or media-tool special case unless source proof shows the lifecycle layer cannot own it. Preserve real user cancels/timeouts as failed/aborted outcomes, preserve the cron-owned media completion wait from https://github.com/openclaw/openclaw/pull/88695, and add focused regression coverage for explicit `sessions_yield` in an isolated cron agent turn that later receives a callback and continues to a follow-up action. Do not edit CHANGELOG.md.
