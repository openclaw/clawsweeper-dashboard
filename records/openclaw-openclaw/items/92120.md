## Summary

Current main still exposes intentional `sessions_yield` interruption as an aborted run, matching the reported v2026.6.5 failure. The merged fix for the earlier report waits for cron-owned media tasks but deliberately suppresses the interactive yield path, so this new callback-resume regression remains necessary and is not a duplicate.

## What This Changes

Fix isolated cron agent turns that abort after `sessions_yield` instead of resuming when an asynchronous `image_generate` callback arrives.

## Best Possible Solution

Represent intentional yield as a first-class paused lifecycle outcome, propagate it without cron error classification, and resume the same isolated session when the asynchronous callback is delivered.

## Reproduction Assessment

Yes at source level. The supplied `sessions_yield` followed by `AbortError` trace matches the current intentional abort implementation and generic aborted-to-error normalization, although this read-only review did not execute a live current-main cron job.

## Solution Assessment

No to an `image_generate` special case or cron-only error suppression. The narrow maintainable solution is a shared intentional-yield outcome plus regression proof that the callback triggers the continuation turn.
