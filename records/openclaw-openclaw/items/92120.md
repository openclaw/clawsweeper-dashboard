## Summary

Keep open: current main still has a source-reproducible yield/media-wait failure, and https://github.com/openclaw/openclaw/pull/92146 is an open draft closing PR rather than a merged fix.

## What This Changes

The issue asks for isolated cron `image_generate` plus `sessions_yield` runs to remain resumable and continue when the media callback arrives instead of ending as `AbortError: aborted`.

## Best Possible Solution

Review and land the linked PR, or a replacement, that skips only the post-yield synchronous media wait and proves the callback resumes the same isolated cron run before direct fallback.

## Reproduction Assessment

Yes at source level. Current main aborts with reason `sessions_yield`, then the generated-media completion-required wait can still run with that already-aborted signal; this matches the reported `AbortError: aborted` trace, though this read-only review did not run a live provider cron job.

## Solution Assessment

Yes, keeping this issue open while the linked fix PR is reviewed is the right path. The best fix is in the shared runner/callback lifecycle, not an `image_generate`-only workaround or early issue close.
