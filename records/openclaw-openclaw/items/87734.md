## Summary

Keep open for maintainer product/security review: current main and the latest release explicitly document one trusted operator boundary and do not provide tenant claims or tenant-scoped scopes, while the request would intentionally change that trust model.

## What This Changes

Request first-class gateway multi-tenant RBAC primitives: tenantId on sessions, runs, approvals, and events; tenant-scoped operator scopes; tenant-aware audit; and opt-in fail-closed tenancy.

## Best Possible Solution

Keep the current separate-gateway trust model until maintainers accept an FRD, then split the work into protocol/auth, session/run store, approval, audit, docs, and migration sub-issues with an additive tenancy enablement path.
