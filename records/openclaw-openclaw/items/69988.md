## Summary

Keep open. Current main still does not implement per-agent compaction overrides, and the discussion plus related reports show this is a real multi-agent config need, but this PR is not ready to merge against current main because it is stale relative to newer compaction schema fields and it exposes more per-agent compaction keys than the runtime actually resolves per agent.

## What This Changes

The PR adds optional `agents.list[].compaction`, extracts a shared compaction Zod schema, merges per-agent compaction over defaults, threads an agent id through selected Pi compaction settings and safeguard-extension setup paths, and adds focused tests plus generated schema updates.

## Best Possible Solution

Rebase the PR on current main, keep a single shared compaction schema in full parity with the existing defaults surface, add a central effective compaction resolver, use it from every compaction runtime path, and update docs/changelog for the new `agents.list[].compaction` contract.

## Reproduction Assessment

Yes. Static reproduction is high confidence: current main's strict `AgentEntrySchema` rejects `agents.list[].compaction`, and the PR diff shows the branch accepts per-agent compaction but does not preserve or route the full current compaction surface.

## Solution Assessment

No. The direction is right, but the current patch is incomplete; the safer solution is a rebased, central effective-compaction resolution path used by schema, Pi settings, safeguard setup, timeout/model/hooks/precheck/transcript rotation, and tests.
