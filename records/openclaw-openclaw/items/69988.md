## Summary

Keep this PR open: the requested per-agent compaction feature is still useful and is not implemented on current main, but the branch has blocking schema/runtime correctness gaps and still needs real behavior proof before merge.

## What This Changes

The PR adds `agents.list[].compaction` schema/type support, merges per-agent compaction over defaults, threads agent IDs into selected embedded Pi compaction paths, and adds focused tests/generated schema output.

## Best Possible Solution

Rebase on current main, keep the defaults compaction schema contract-equivalent, add one effective per-agent compaction resolver with nested merge semantics, and use it from every compaction consumer with focused tests and docs/changelog follow-through.

## Reproduction Assessment

Yes. Static source review is high-confidence: current main rejects `agents.list[].compaction`, while the PR accepts that shape but drops current defaults-schema keys and leaves accepted per-agent keys routed through defaults-only consumers.

## Solution Assessment

No. The feature direction is good, but the patch is not complete until schema extraction preserves the existing defaults contract and effective compaction resolution is centralized across all runtime consumers.
