## Summary

Keep open. The feature remains valid and current main does not implement it, but this PR is not merge-ready because it is behind the current compaction config/runtime surface: it would drop existing defaults-schema keys and expose per-agent keys that several compaction paths still ignore.

## What This Changes

The PR adds optional `agents.list[].compaction`, merges per-agent compaction over defaults, threads an agent id through selected embedded Pi compaction settings and safeguard setup paths, and updates focused tests plus generated schema output.

## Best Possible Solution

Rebase the PR on current main, extract a complete shared compaction schema that preserves the existing defaults contract, add one effective per-agent compaction resolver with nested-object merge semantics, use it from every compaction runtime consumer, and document/changelog the new `agents.list[].compaction` contract.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main rejects `agents.list[].compaction`, while the PR diff accepts it but omits current defaults-schema keys and leaves defaults-only runtime reads for accepted fields such as timeout, model override, post-index sync, memory flush, mid-turn precheck, and transcript rotation.

## Solution Assessment

No. The product direction is right, but the proposed patch is incomplete; the narrower maintainable solution is a central effective-compaction resolution path reused by schema, settings, safeguard setup, timeout/model/hooks/precheck/transcript rotation, and tests.
