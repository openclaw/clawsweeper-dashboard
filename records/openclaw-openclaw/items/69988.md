## Summary

Keep open: current main still lacks per-agent compaction overrides, but this PR is not merge-ready because it narrows the existing defaults compaction schema, accepts per-agent fields that still route through defaults-only consumers, and lacks inspectable real behavior proof.

## What This Changes

The PR adds `agents.list[].compaction` schema/type support, merges per-agent compaction over defaults, threads agent IDs into selected embedded Pi compaction paths, and adds focused tests/generated schema output.

## Best Possible Solution

Rebase on current main, preserve the full defaults compaction contract, add one effective per-agent compaction resolver with nested merge semantics, and use it from every compaction consumer with focused schema/runtime tests and docs/changelog follow-through.

Yes. Source review gives a high-confidence path: current main rejects `agents.list[].compaction`, while the PR accepts that shape but leaves schema and runtime-routing gaps visible in the diff and current consumers.

## Solution Assessment

No. The feature direction is reasonable, but the current patch is not the narrowest maintainable solution until the schema extraction is contract-equivalent and effective compaction config resolution is centralized across consumers.
