## Summary

Keep open: this is a real config feature request with supportive linked production context, but the current PR should not merge yet because it regresses the newer defaults compaction schema, accepts per-agent keys that several runtime consumers still ignore, and lacks inspectable real behavior proof.

## What This Changes

Adds `agents.list[].compaction` schema/type support, merges per-agent compaction over `agents.defaults.compaction`, threads agent IDs through selected embedded Pi compaction paths, and updates focused tests plus generated schema output.

## Best Possible Solution

Rebase and rework the feature around one current-base effective compaction resolver that preserves the full defaults schema and is used by every compaction consumer, then update generated metadata/docs/tests and add contributor-provided real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main's strict `AgentEntrySchema` still rejects `agents.list[].compaction` while defaults compaction exists; I did not run tests because this review is read-only.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best merge shape until it preserves the current defaults contract and routes every accepted per-agent compaction key through the runtime paths that consume it.
