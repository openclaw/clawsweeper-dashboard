## Summary

Keep this PR open. The feature direction is supported by the linked discussion and production context, but current main has not implemented per-agent compaction and the PR is incomplete against the current compaction contract and runtime consumers.

## What This Changes

The PR adds `agents.list[].compaction` to config/schema/types, merges agent compaction over defaults, threads agent ids into selected embedded Pi compaction settings and safeguard paths, and adds focused tests plus generated schema output.

## Best Possible Solution

Rebase on current main, preserve the full defaults compaction schema, implement one effective per-agent compaction resolver with nested merge semantics, and use it from every compaction consumer with focused tests and docs/changelog follow-through.

## Reproduction Assessment

Yes. Static source review is high-confidence: current main rejects `agents.list[].compaction`, while the PR accepts that shape but does not preserve the full current defaults schema or route every accepted key through effective per-agent resolution.

## Solution Assessment

No. The direction is right, but the proposed patch is not the best complete solution until schema extraction is contract-equivalent and runtime compaction resolution is centralized across all consumers.
