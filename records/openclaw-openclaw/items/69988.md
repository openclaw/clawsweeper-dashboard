## Summary

Keep open. Current main still does not provide usable per-agent compaction overrides, and this PR remains a plausible implementation path, but the current patch has blocking schema/runtime gaps and still lacks inspectable real behavior proof.

## What This Changes

The PR adds `agents.list[].compaction` schema/type support, merges per-agent compaction over defaults, threads agent IDs into selected embedded Pi compaction paths, and adds focused tests plus generated schema output.

## Best Possible Solution

Keep the feature path, but rework it around one effective per-agent compaction resolver that preserves the full defaults contract and is used by every compaction consumer, with generated metadata, docs, focused tests, and real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main still rejects `agents.list[].compaction` in the strict runtime schema while defaults compaction exists; the PR's remaining schema and runtime gaps are visible from the head files. No tests were run because this was a read-only review.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best merge shape until the extracted schema is contract-equivalent with defaults and all accepted per-agent compaction keys flow through a shared effective resolver.
