## Summary

Keep open. Current main and latest release v2026.6.5 still model Discord message policy as one coarse `actions.messages` boolean and still use that same gate for send/read/edit/delete, so the requested least-privilege Discord contract remains missing.

## What This Changes

The issue requests splitting Discord `actions.messages` into granular send/read/edit/delete gates while preserving the existing boolean shorthand.

## Best Possible Solution

Choose a backward-compatible Discord least-privilege message-action contract, then align config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests.

## Reproduction Assessment

Yes, by source inspection. Current main still accepts only a boolean Discord `actions.messages` value and checks that same gate for send/read/edit/delete message actions.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed boolean-or-object shape is plausible, but separate top-level gates may also fit; either path must preserve the shipped boolean shorthand and align with message-tool action allowlists.
