## Summary

Keep open. Current main and the latest shipped release still expose Discord `actions.messages` as one coarse boolean and still apply that one gate to send/read/edit/delete message behavior, while this request needs a maintainer-chosen Discord least-privilege config contract.

## What This Changes

Add granular Discord message action gates so send/read/edit/delete can be controlled independently while preserving the existing `actions.messages` boolean shorthand.

## Best Possible Solution

Choose a backward-compatible Discord least-privilege contract, then align config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests while preserving the boolean shorthand.

## Reproduction Assessment

Yes, by source inspection. Current main still accepts only a boolean Discord `actions.messages` value and checks that same gate for send/read/edit/delete message actions.

## Solution Assessment

Unclear until maintainers choose the public contract. The boolean-or-object shape is plausible, but the design should account for `tools.message.actions.allow` and preserve the current boolean shorthand.
