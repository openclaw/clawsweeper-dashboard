## Summary

Keep open. Current main still exposes Discord message action control as one coarse `actions.messages` boolean in the public config type, strict schema, discovery, runtime checks, docs, and tests; the requested granular send/read/edit/delete split remains a feature/config-contract gap.

## What This Changes

Request to add backward-compatible granular Discord message action gates so send, read, edit, and delete can be enabled independently instead of sharing `actions.messages`.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand and applies chosen per-action defaults consistently across config schema, discovery, runtime checks, docs, and focused tests.

## Reproduction Assessment

Yes for source-level reproduction: current main declares `actions.messages` as boolean-only and the same gate controls Discord send/read/edit/delete paths. This is a missing config capability rather than a failing documented behavior requiring live Discord reproduction.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, account inheritance, and `upload-file` semantics need owner approval.
