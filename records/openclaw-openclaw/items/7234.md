## Summary

Keep open: current main still has only one Discord `actions.messages` channel/account gate, and the existing global/per-agent message action allowlist is a useful workaround but not the requested Discord-specific permission contract.

## What This Changes

Request a granular Discord message-action gate so send, read, edit, and delete can be enabled independently while preserving the existing boolean shorthand.

## Best Possible Solution

Define one canonical Discord/account-level message-action permission model, document how it composes with `tools.message.actions.allow`, preserve or migrate the shipped boolean behavior, and update schema, metadata, discovery, runtime, docs, and focused tests together.

## Reproduction Assessment

Yes for the missing capability: current-main types, schema, docs, discovery, runtime checks, and tests still expose one Discord `messages` gate. A live Discord run is not needed to prove that independent Discord send/read/edit/delete gates are absent.

## Solution Assessment

Unclear. Granular gates are the right direction, but maintainers need to choose the canonical public config shape and its relationship to the existing global/per-agent `tools.message.actions.allow` policy before implementation.
