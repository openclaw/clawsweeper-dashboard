## Summary

Keep open. Current main and the latest shipped release still use one Discord `actions.messages` boolean across config, schema, discovery, and runtime, so the requested least-privilege split is not implemented and remains a product/security decision.

## What This Changes

Request to split Discord `channels.discord.actions.messages` into granular send/read/edit/delete gates while preserving backward-compatible boolean behavior.

## Best Possible Solution

Adopt one backward-compatible Discord gate contract for separate send/read/edit/delete policy, then update config types, strict schema, generated metadata, docs, discovery, runtime checks, and focused tests.

## Reproduction Assessment

Yes, by source inspection. Current main accepts only `messages?: boolean`, rejects object-shaped Discord message gates, and routes send/read/edit/delete through the same `messages` runtime gate.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested boolean-or-object shape is plausible, but separate top-level gates may better fit the current boolean action-gate helper.
