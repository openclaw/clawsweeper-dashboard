## Summary

Keep open. Current main still models Discord message action permissions as one coarse `actions.messages` boolean across config, schema, discovery, runtime, docs, and tests; the requested granular send/read/edit/delete split is still a public config-contract feature that needs maintainer direction.

## What This Changes

Request a backward-compatible Discord config split so message send, read, edit, and delete actions can be enabled independently instead of sharing `channels.discord.actions.messages`.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand, then update config types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests together.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows `actions.messages` is boolean-only and the same key gates Discord message send/read/edit/delete runtime paths. A live Discord run is not required to establish that the requested config surface is missing.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, account inheritance, and `upload-file` semantics need owner approval.
