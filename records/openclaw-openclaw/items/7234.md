## Summary

Keep open. Current main still models Discord `actions.messages` as one coarse boolean and applies that gate to Discord send/read/edit/delete behavior, so the requested granular permission model remains missing and needs maintainer approval of the public config shape.

## What This Changes

The issue requests backward-compatible granular Discord message action gates so send, read, edit, and delete can be enabled independently instead of sharing `channels.discord.actions.messages`.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the legacy boolean shorthand, then update types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests after the public shape is approved.

## Reproduction Assessment

Yes for the reported current behavior. Source inspection on current main shows `actions.messages` is boolean-only and one `messages` gate controls Discord send/read/edit/delete runtime and discovery behavior.

## Solution Assessment

Unclear until maintainers choose the public config shape. A backward-compatible granular split is likely the narrow maintainable direction, but nested versus top-level gates and `upload-file` behavior are public contract decisions.
