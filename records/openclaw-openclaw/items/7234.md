## Summary

Keep open. Current main still exposes Discord `actions.messages` as one coarse boolean and applies that gate across send/read/edit/delete behavior, and I found no open paired PR or canonical duplicate that supersedes this request.

## What This Changes

The issue requests granular Discord message action gates so send, read, edit, and delete can be enabled independently while preserving the existing boolean shorthand.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand while updating types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main only accepts `actions.messages` as a boolean, and discovery/runtime still use that single gate for multiple Discord message operations.

## Solution Assessment

Yes. A backward-compatible granular split is the narrow maintainable direction, but the exact public config shape should be agreed before implementation.
