## Summary

Keep open. Current main still uses one Discord `actions.messages` boolean across config, strict validation, discovery, and runtime checks, so the requested least-privilege split remains unimplemented and needs maintainer/security product review.

## What This Changes

Request split Discord `channels.discord.actions.messages` into backward-compatible granular gates for send, read, edit, and delete.

## Best Possible Solution

Adopt one backward-compatible Discord message gate contract for separate send/read/edit/delete policy, then update config types, strict schema, generated metadata, docs, discovery, runtime checks, and focused tests.

## Reproduction Assessment

Yes, by source inspection. Current main accepts only `messages?: boolean`, rejects object-shaped Discord message gates, and routes send/read/edit/delete through the same `messages` runtime gate.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested boolean-or-object shape is plausible, but separate top-level gates may better fit the current boolean action-gate helper.
