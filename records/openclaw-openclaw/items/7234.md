## Summary

Keep open. Current main and v2026.5.22 still expose Discord message policy as one coarse `actions.messages` boolean, and the runtime still checks that same gate for send/read/edit/delete, so the requested least-privilege Discord contract is still missing.

## What This Changes

Add backward-compatible granular Discord message action gates so deployments can allow send/read/edit while denying destructive actions such as delete.

## Best Possible Solution

Choose a backward-compatible Discord least-privilege message-action contract, then align config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests.

## Reproduction Assessment

Yes, by source inspection. Current main still accepts only a boolean Discord `actions.messages` value and checks that same gate for send/read/edit/delete message actions.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed boolean-or-object shape is plausible, but the design should account for `tools.message.actions.allow` and preserve the current boolean shorthand.
