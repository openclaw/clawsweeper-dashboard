## Summary

Keep open: current main and v2026.6.8 still model Discord message permissions as one coarse `actions.messages` boolean, and the requested split remains a security-sensitive public config decision rather than cleanup-close material.

## What This Changes

The issue requests granular Discord message action gates so send/read/edit/delete can be enabled independently instead of sharing one `channels.discord.actions.messages` boolean.

## Best Possible Solution

Define one canonical Discord/account-level message-action permission model, preserve or migrate the shipped boolean shorthand deliberately, and update schema, metadata, discovery, runtime, docs, and focused tests together.

## Reproduction Assessment

Yes. Current main still has one Discord `messages` action gate in config, schema, discovery, runtime checks, docs, and tests; a live Discord run is not needed to prove independent Discord send/read/edit/delete gates are absent.

## Solution Assessment

Unclear. Granular gates fit the least-privilege goal, but maintainers still need to choose the canonical public config shape, inheritance behavior, and composition with the existing message-tool allowlist.
