## Summary

Keep open. Current main still models Discord `actions.messages` as one coarse boolean across config, schema, generated metadata, discovery, runtime checks, docs, and tests; the request is valid and source-reproducible, but it changes a user-facing Discord permission/config contract and needs maintainer API direction before implementation.

## What This Changes

The issue requests granular Discord message action gates so read, send, edit, and delete can be enabled or disabled independently while preserving the existing boolean shorthand.

## Best Possible Solution

Add a backward-compatible Discord message action gate contract, update schema/metadata/docs/discovery/runtime checks together, and cover boolean shorthand plus granular send/read/edit/delete behavior with focused Discord tests.

## Reproduction Assessment

Yes for source reproduction: current main still accepts only a boolean `channels.discord.actions.messages` and runtime/discovery use that one gate for message actions. I did not run live Discord behavior because the missing config contract is established directly from source, docs, metadata, and tests.

## Solution Assessment

Unclear until maintainers choose the public config shape. A boolean-or-object gate is plausible, but equivalent separate top-level gates may be preferred for consistency with other channel action contracts.
