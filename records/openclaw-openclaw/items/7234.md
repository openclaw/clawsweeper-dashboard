## Summary

## What This Changes

Requests independent Discord gates for sending, reading, editing, and deleting messages while retaining the shipped boolean configuration contract.

## Best Possible Solution

Keep the shipped `messages` umbrella contract and add Discord-owned granular boolean overrides with explicit precedence, schema metadata, docs, doctor handling if needed, and focused discovery/runtime tests.

## Reproduction Assessment

Yes for the reported limitation: current-main source and contract tests clearly show that one boolean controls the relevant Discord message actions, though no live Discord environment was exercised during this read-only review.

## Solution Assessment

No, the nested union proposed in the issue is not yet proven to be the best implementation because the shared account gate accepts boolean fields only; Discord-specific granular boolean overrides with defined umbrella precedence are narrower and preserve the shipped contract.
