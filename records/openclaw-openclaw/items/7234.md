## Summary

Keep open. Current main still uses one coarse Discord `actions.messages` boolean across config validation, discovery, runtime checks, docs, and tests; the request is valid but needs a maintainer API/security decision before adding a user-facing permission contract.

## What This Changes

Request granular Discord message action gates so send, read, edit, and delete can be enabled independently instead of sharing one `actions.messages` boolean.

## Best Possible Solution

Add a backward-compatible Discord message gate contract, update schema/metadata/docs/discovery/runtime together, and cover boolean shorthand plus granular send/read/edit/delete behavior with focused Discord tests.

## Reproduction Assessment

Yes for source reproduction: current main accepts only a boolean `channels.discord.actions.messages`, discovery has no per-action gate, and runtime send/read/edit/delete all check that same boolean. I did not run live Discord because the missing contract is visible from source, docs, and tests.

## Solution Assessment

Unclear until maintainers choose the public config shape. The requested boolean-or-object shape is reasonable, but an equivalent set of separate top-level action gates may be more consistent with current channel action contracts.
