## Summary

Keep open. Current main and latest release still expose Discord message policy as one coarse `actions.messages` boolean; the existing generic message-tool action allowlist can hide actions, but it does not replace a Discord account/channel least-privilege contract for send/read/edit/delete.

## What This Changes

The issue requests splitting Discord `actions.messages` into granular send/read/edit/delete gates while preserving the existing boolean shorthand.

## Best Possible Solution

Choose a backward-compatible Discord least-privilege contract, then align config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests while preserving the current boolean shorthand.

## Reproduction Assessment

Yes, by source inspection. Current main still accepts only a boolean Discord `actions.messages` value and checks that same gate for send/read/edit/delete message actions.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed boolean-or-object shape is plausible, but the design should account for the generic message action allowlist and preserve the existing boolean shorthand.
