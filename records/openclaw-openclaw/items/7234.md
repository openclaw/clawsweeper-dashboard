## Summary

Keep open. Current main still has a boolean-only Discord `actions.messages` gate and uses it for Discord send/read/edit/delete behavior, so the requested granular permission model remains unimplemented and needs a config-contract decision.

## What This Changes

Request to split Discord `channels.discord.actions.messages` from one coarse boolean into backward-compatible per-message-action gates.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the legacy boolean shorthand, then update types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests.

## Reproduction Assessment

Yes for the reported current behavior. Source inspection on current main shows `actions.messages` is boolean-only and one `messages` gate controls Discord send/read/edit/delete runtime behavior and read/edit/delete discovery.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, and upload-file semantics need owner approval.
