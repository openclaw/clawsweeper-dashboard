## Summary

Keep open. Current main and the latest shipped release still use one coarse Discord `actions.messages` boolean for send/read/edit/delete behavior, while the existing generic message action allowlist is agent/tool scoped rather than the requested Discord channel/account least-privilege contract.

## What This Changes

Request splitting Discord `channels.discord.actions.messages` into per-action send/read/edit/delete gates while preserving the existing boolean shorthand.

## Best Possible Solution

Choose a backward-compatible Discord least-privilege contract, then align config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests while preserving the boolean shorthand.

## Reproduction Assessment

Yes, by source inspection. Current main still accepts only a boolean Discord `actions.messages` value and checks that same gate for send/read/edit/delete message actions.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed boolean-or-object shape is plausible, but the design should account for `tools.message.actions.allow` and preserve the current boolean shorthand.
