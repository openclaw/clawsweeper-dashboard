## Summary

Keep open: current main and the latest release still expose one Discord `actions.messages` gate, while the existing global/per-agent message action allowlist is useful but not the requested Discord/account-scoped permission contract.

## What This Changes

The issue requests splitting the Discord `actions.messages` boolean into granular send/read/edit/delete gates while preserving the existing boolean behavior.

## Best Possible Solution

Define one canonical Discord/account-level message-action permission model, document how it composes with `tools.message.actions.allow`, preserve or deliberately migrate the shipped boolean behavior, and update schema, metadata, discovery, runtime, docs, and focused tests together.

## Reproduction Assessment

Yes at source level: current main still has one Discord `messages` action gate in config, discovery, runtime, and tests. A live Discord run is not needed to prove that independent Discord send/read/edit/delete gates are absent.

## Solution Assessment

Unclear. Granular gates fit the least-privilege goal, but maintainers need to choose the canonical public config shape and how it composes with the existing global/per-agent `tools.message.actions.allow` policy.
