## Summary

Keep open. Current main still exposes Discord message actions through one coarse `actions.messages` boolean across config, discovery, runtime, docs, and tests, so the requested granular send/read/edit/delete gates remain missing.

## What This Changes

Request to split Discord `channels.discord.actions.messages` into backward-compatible per-message-action gates for send, read, edit, and delete.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand and applies per-action gates consistently across config schema, generated metadata, discovery, runtime checks, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `actions.messages` is boolean-only and the same gate controls Discord send/read/edit/delete execution paths; this is a missing config capability rather than a live-only Discord failure.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, account inheritance, and `upload-file` semantics need owner approval.
