## Summary

Keep open: current main and v2026.6.8 still expose one Discord `actions.messages` gate for send/read/edit/delete, and this remains a security-sensitive config/product decision rather than a cleanup close.

## What This Changes

The issue requests splitting Discord `actions.messages` into separate read, edit, delete, and send gates while preserving the existing boolean behavior.

## Best Possible Solution

Define one canonical Discord/account-level message-action permission model, document how it composes with `tools.message.actions.allow`, preserve or deliberately migrate the shipped boolean behavior, and update schema, metadata, discovery, runtime, docs, and focused tests together.

Yes. Current main still has one Discord `messages` action gate in config, schema, discovery, runtime checks, docs, and tests; a live Discord run is not needed to prove that independent Discord send/read/edit/delete gates are absent.
