## Summary

Keep open. Current main still has one coarse Discord `actions.messages` boolean across config, schema, discovery, runtime checks, docs, and tests; the requested granular send/read/edit/delete controls remain a public config feature gap.

## What This Changes

The issue requests granular Discord message action gates so send, read, edit, and delete can be controlled independently instead of sharing one `actions.messages` boolean.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the boolean shorthand and applies chosen per-action defaults consistently across schema, discovery, runtime, docs, and tests.

## Reproduction Assessment

Yes for source-level reproduction: current main declares `actions.messages` as boolean-only and the same gate controls Discord send/read/edit/delete paths. This is a missing config capability rather than a failing documented behavior requiring live Discord reproduction.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, account inheritance, and `upload-file` semantics need owner approval.
