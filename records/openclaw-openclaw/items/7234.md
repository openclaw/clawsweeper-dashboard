## Summary

Keep open. Current main still models Discord channel/account `actions.messages` as one boolean across config, schema, discovery, and runtime dispatch, so the requested least-privilege split remains missing and needs maintainer product/security review. The generic message action allowlist can mitigate some deployments, but it is not the requested Discord channel gate contract.

## What This Changes

Request adds backward-compatible granular Discord message action gates so send/read/edit/delete can be controlled independently instead of via one `actions.messages` boolean.

## Best Possible Solution

Adopt one backward-compatible Discord action-gate contract for separate send/read/edit/delete policy, then update config types, strict schema, generated metadata, docs, discovery, runtime checks, and focused tests together while preserving existing boolean behavior.

## Reproduction Assessment

Yes, by source inspection. Current main accepts only `messages?: boolean`, rejects object-shaped Discord message gates, and routes send/read/edit/delete through the same `messages` runtime gate.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested boolean-or-object shape is plausible, but separate top-level gates may fit the existing action-gate helper better, and the current generic message action allowlist is only a workaround.
