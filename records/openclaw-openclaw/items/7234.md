## Summary

Keep open. Current main still models Discord `actions.messages` as one coarse boolean and applies that same gate across send/read/edit/delete behavior; the remaining work is a public config contract change, not stale cleanup.

## What This Changes

Request to split Discord `channels.discord.actions.messages` into backward-compatible granular gates for send, read, edit, and delete message actions.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the legacy boolean shorthand, then update types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests after the public shape is approved.

## Reproduction Assessment

Yes for the reported current behavior. Source inspection on current main shows `actions.messages` is boolean-only and one `messages` gate controls Discord send/read/edit/delete runtime and discovery paths.

## Solution Assessment

Unclear until maintainers choose the public config shape. The backward-compatible granular split is the narrow maintainable direction, but it changes the channel configuration contract.
