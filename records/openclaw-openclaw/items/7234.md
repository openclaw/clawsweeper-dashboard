## Summary

Current main still exposes one coarse Discord message-action gate across send, read, edit, and delete, so the least-privilege capability remains missing; the prior review reached the same conclusion, and no canonical replacement issue or PR was found.

## What This Changes

Add granular Discord message action gates so operators can independently allow send, read, edit, and delete.

## Best Possible Solution

Adopt one canonical least-privilege Discord permission model and align config types, validation, generated metadata, discovery, runtime enforcement, public docs, and focused tests with a deliberate upgrade path for the shipped boolean setting.

## Reproduction Assessment

## Solution Assessment

Unclear on the exact proposed shape. Independent operation gates are the appropriate least-privilege direction, but maintainers must select the canonical schema and decide whether the existing boolean remains shorthand or is migrated by `openclaw doctor --fix`.
