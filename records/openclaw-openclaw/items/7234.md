## Summary

Keep open. Current main still models Discord `actions.messages` as one coarse boolean gate and uses that same gate for Discord send/read/edit/delete behavior, so the requested granular message action gates remain unimplemented and need a config-contract decision.

## What This Changes

The issue requests backward-compatible granular Discord message action gates so send/read/edit/delete can be enabled or disabled independently instead of sharing `channels.discord.actions.messages`.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand, then update types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests together.

## Reproduction Assessment

Yes for the current behavior: source inspection shows `actions.messages` is boolean-only and the same key gates Discord send/read/edit/delete runtime behavior. A live Discord run is not required to establish the missing config surface.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, and `upload-file` semantics need owner approval.
