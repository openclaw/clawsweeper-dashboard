## Summary

Keep open. Current main still uses one Discord `actions.messages` boolean across config, schema, discovery, and runtime, so the requested least-privilege split is not implemented and needs maintainer product/security review.

## What This Changes

Request a backward-compatible Discord action-gate split so send, read, edit, and delete message actions can be enabled independently instead of sharing `actions.messages`.

## Best Possible Solution

Adopt one backward-compatible Discord gate contract for separate send/read/edit/delete policy, then update config types, strict schema, generated metadata, docs, discovery, runtime checks, and focused tests while preserving existing boolean behavior.

## Reproduction Assessment

Yes, by source inspection. Current main accepts only `messages?: boolean`, rejects object-shaped Discord message gates, and routes send/read/edit/delete through the same `messages` runtime gate.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested boolean-or-object shape is plausible, but separate top-level gates may fit the existing action-gate helper better, and the per-agent message action allowlist is only a workaround.
