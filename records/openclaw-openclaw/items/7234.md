## Summary

Keep open. Current main still models Discord messages as one boolean action gate across config schema, discovery, and send/read/edit/delete runtime paths, so the granular permission request remains valid.

## What This Changes

Request a backward-compatible split of Discord `actions.messages` into per-message gates for send, read, edit, and delete behavior.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that supports per-action send/read/edit/delete policy while preserving the existing boolean shorthand, then align types, schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the schema only accepts a boolean `actions.messages`, and the runtime uses that same `messages` gate for send, fetch/read, edit, and delete paths.

## Solution Assessment

Yes, the requested backward-compatible granular split is the narrow maintainable direction. The remaining maintainer decision is the exact public config shape and how it should compose with per-account overrides.
