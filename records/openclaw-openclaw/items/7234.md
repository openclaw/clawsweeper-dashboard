## Summary

Current main still uses one coarse Discord `actions.messages` boolean across config, schema, discovery, runtime checks, docs, and tests. The request is valid and source-reproducible, but the fix changes a user-facing config and permission model, so it needs maintainer product/API direction before implementation.

## What This Changes

The issue requests backward-compatible granular Discord message action gates so deployments can disable read, edit, delete, or send independently instead of using one `channels.discord.actions.messages` boolean.

## Best Possible Solution

Add a backward-compatible Discord message action gate contract, update schema/metadata/docs/discovery/runtime checks together, and cover the boolean shorthand plus granular send/read/edit/delete behavior with focused Discord tests.

## Reproduction Assessment

Yes for source reproduction: current main still accepts only a boolean `channels.discord.actions.messages` and runtime/discovery use that one gate for message actions. I did not run live Discord behavior because the missing config contract is established directly from source, docs, metadata, and tests.

## Solution Assessment

Unclear until maintainers choose the public config shape; a boolean-or-object gate is plausible, but equivalent separate top-level gates may be preferred for consistency with other channel action contracts.
