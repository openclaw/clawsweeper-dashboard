## Summary

Keep open. Current main still exposes Discord message actions through one coarse `actions.messages` boolean and runtime gate, so the requested least-privilege split is missing and needs maintainer product/security review before implementation.

## What This Changes

Request split Discord `actions.messages` into separate send/read/edit/delete gates while preserving the existing boolean behavior.

## Best Possible Solution

Adopt one backward-compatible Discord action-gate contract for separate send/read/edit/delete policy, then update config types, strict schema, generated metadata, docs, discovery, runtime checks, and focused tests together.

## Reproduction Assessment

Yes, by source inspection. Current main only accepts `messages?: boolean`, rejects object-shaped gates in the strict schema, and routes send/read/edit/delete through the same `messages` runtime gate.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested boolean-or-object shape is reasonable, but separate top-level gates may be a cleaner fit for the existing boolean action-gate helper.
