## Summary

Keep open. Current main still exposes Discord `actions.messages` as one coarse boolean and applies that same gate across send, read, edit, and delete behavior, so the requested backward-compatible granular gates are not implemented.

## What This Changes

Request granular Discord message action gates so deployments can allow send/read/edit while disabling destructive actions such as delete.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand while aligning types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the schema rejects object-shaped `actions.messages`, and runtime/discovery still use the same `messages` gate for multiple message operations.

## Solution Assessment

Yes, a backward-compatible granular split is the narrow maintainable direction, but the exact public config shape needs maintainer agreement before implementation.
