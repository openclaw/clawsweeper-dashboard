## Summary

Keep open. Current main still has one coarse Discord `actions.messages` boolean across config validation, discovery, docs, and runtime checks, and there is no duplicate or closing pull request; the remaining work is a public Discord permission/config contract decision with least-privilege security impact.

## What This Changes

Request to split Discord `channels.discord.actions.messages` into granular send/read/edit/delete gates while preserving backward-compatible boolean behavior.

## Best Possible Solution

Adopt one backward-compatible Discord action gate API that supports separate send/read/edit/delete policy while retaining existing boolean behavior and account override semantics.

## Reproduction Assessment

Yes for source reproduction. Current main only accepts `messages?: boolean`, strict schema rejects object-shaped gates, discovery has no per-action gate, and runtime send/read/edit/delete paths all consult `messages`.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested boolean-or-object shape is reasonable, but separate top-level gates may be a better fit for the current boolean action-gate helper; either path needs synchronized type, schema, metadata, docs, discovery, runtime, and test updates.
