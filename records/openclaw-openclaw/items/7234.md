## Summary

Keep open. Current main and v2026.6.5 still expose Discord message policy as one coarse `actions.messages` boolean and still use that same gate for send/read/edit/delete, so the requested least-privilege Discord contract is still missing.

## What This Changes

The issue requests splitting Discord `actions.messages` into separate read, send, edit, and delete gates while preserving the existing boolean shorthand.

## Best Possible Solution

Choose a backward-compatible Discord least-privilege message-action contract, then align config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests.

## Reproduction Assessment

Yes, by source inspection. Current main still accepts only a boolean Discord `actions.messages` value and checks that same gate for send/read/edit/delete message actions.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed boolean-or-object shape is plausible, but the design should account for current message-tool allowlist behavior and preserve the shipped boolean shorthand.
