## Summary

Keep open. Current main and the latest release still expose Discord message actions through one coarse `actions.messages` boolean, so the requested least-privilege delete/edit/read/send split is real but needs a maintainer API/security decision before implementation.

## What This Changes

The issue asks for Discord `channels.discord.actions.messages` to be split into independently configurable send/read/edit/delete gates while preserving the existing boolean shorthand.

## Best Possible Solution

Choose a backward-compatible Discord message gate contract, then update the config type, strict schema, generated metadata, docs, discovery, runtime checks, and focused Discord tests together.

## Reproduction Assessment

Yes for source reproduction. Current main accepts only a boolean `channels.discord.actions.messages`, discovery has no per-action gate, and runtime send/read/edit/delete paths all consult that same gate.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested boolean-or-object shape is reasonable, but equivalent separate action keys may be the narrower long-term API fit.
