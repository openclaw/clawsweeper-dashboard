## Summary

Keep open. Current main and latest release still model Discord message policy as one coarse actions.messages boolean and still use that gate for send/read/edit/delete, with no merged or open canonical replacement found for the requested least-privilege contract.

## What This Changes

The issue requests granular Discord message-action gates so operators can allow send/read/edit while disabling delete instead of using one coarse actions.messages boolean.

## Best Possible Solution

Choose a backward-compatible Discord least-privilege message-action contract, then align config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused Discord tests.

## Reproduction Assessment

Yes, by source inspection. Current main and v2026.6.5 accept only a boolean Discord actions.messages value and check that same gate for send/read/edit/delete message actions.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed boolean-or-object shape is plausible, but separate top-level gates may also fit if they preserve shipped boolean behavior.
