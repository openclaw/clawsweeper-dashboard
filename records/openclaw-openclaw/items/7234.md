## Summary

Keep open. Current main still exposes Discord message permissions as one coarse `actions.messages` boolean across config, schema, discovery, runtime, and docs; no canonical duplicate or merged fix for the granular send/read/edit/delete split was found.

## What This Changes

The issue requests a backward-compatible Discord config change that splits `actions.messages` into per-operation gates for send, read, edit, and delete.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand, then update config types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests together.

## Reproduction Assessment

Yes for the current source behavior: current main shows `actions.messages` is boolean-only and the same key gates Discord send/read/edit/delete runtime paths. This is a feature/config-contract gap rather than a failing existing bug that requires live Discord reproduction.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, account inheritance, and `upload-file` semantics need owner approval.
