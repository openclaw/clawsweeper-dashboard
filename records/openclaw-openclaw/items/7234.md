## Summary

Keep open. Current main still has one coarse Discord `actions.messages` boolean for send/read/edit/delete behavior, and the requested granular gates require a public config-contract decision before a focused implementation should proceed.

## What This Changes

Request split Discord `channels.discord.actions.messages` into backward-compatible per-message-action gates for send, read, edit, and delete.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the existing boolean shorthand, then update types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests together.

## Reproduction Assessment

Yes for the current behavior: source inspection shows `actions.messages` is boolean-only and the same key gates Discord send/read/edit/delete runtime behavior. A live Discord run is not required to establish the missing config surface.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, account inheritance, and `upload-file` semantics need owner approval.
