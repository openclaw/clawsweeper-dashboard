## Summary

Current main still has the same coarse Discord `actions.messages` gate, so this remains valid and should stay open for a maintainer config-contract decision.

## What This Changes

The issue requests splitting Discord `actions.messages` into granular read/edit/delete/send gates while preserving the existing boolean shorthand.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the legacy boolean shorthand, then update types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests.

## Reproduction Assessment

Yes for the reported current behavior. Source inspection on current main shows `actions.messages` is boolean-only and the same `messages` gate controls Discord send/read/edit/delete runtime and discovery behavior.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, and `upload-file` semantics need owner approval.
