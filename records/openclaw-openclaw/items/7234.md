## Summary

Keep open. Current main still exposes Discord `actions.messages` as one boolean gate and applies it across send/read/edit/delete behavior, so the requested backward-compatible granular message gates are not implemented.

## What This Changes

The issue requests a backward-compatible split of Discord `channels.discord.actions.messages` into per-action send/read/edit/delete gates while preserving the existing boolean shorthand.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves the boolean shorthand, supports per-action overrides, and keeps config types, validation, generated metadata, discovery, runtime checks, docs, and tests aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the schema rejects an object-shaped `actions.messages`, and the runtime checks the same `messages` gate for send, fetch/read, edit, and delete paths.

## Solution Assessment

Yes. A backward-compatible granular split is the narrow maintainable direction, provided the implementation centralizes gate resolution so discovery, runtime behavior, and account overrides stay consistent.
