## Summary

Current main still lacks Discord/account-scoped granular message action gates, and this remains a security-sensitive configuration feature rather than a strict bug. The existing global/per-agent message action allowlist is adjacent but does not provide the requested Discord `channels.discord.actions.messages` split, so maintainers still need to choose the public permission model before implementation.

## What This Changes

The issue requests splitting Discord `actions.messages` from one boolean into granular send/read/edit/delete permission gates while preserving the existing boolean shorthand.

## Best Possible Solution

Keep one backward-compatible Discord permission model that preserves `actions.messages: boolean`, adds explicit send/read/edit/delete semantics, defines how it composes with `tools.message.actions.allow`, and updates config types, strict schema, generated metadata, discovery, runtime checks, docs, and focused tests together.

## Reproduction Assessment

Yes, at source level: current main still accepts only a boolean `channels.discord.actions.messages` gate and routes Discord fetch/read/send/edit/delete behavior through that coarse gate. A live Discord run is not needed to prove that independent Discord send/read/edit/delete gates are absent.

## Solution Assessment

Unclear until maintainers choose the public permission model. Granular gates fit least-privilege goals, but the best implementation must define the canonical config shape and composition with the existing global/per-agent message action allowlist.
