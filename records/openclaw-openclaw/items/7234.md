## Summary

Keep open. Current main still models Discord `actions.messages` as one coarse boolean and uses that same gate across message send/read/edit/delete behavior; the requested backward-compatible granular split is missing from config, validation, metadata, discovery, runtime, docs, and focused tests.

## What This Changes

Request split Discord `channels.discord.actions.messages` into backward-compatible per-message gates for send, read, edit, and delete.

## Best Possible Solution

Implement a centralized Discord message-action gate resolver that preserves boolean shorthand, supports per-action overrides, and keeps config types, strict schema, generated metadata, discovery, runtime checks, docs, and tests aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `actions.messages: false` disables runtime sends, reads, edits, and deletes through one gate, while the schema rejects the proposed object-shaped granular gate.

## Solution Assessment

Yes. A backward-compatible granular split is the narrow maintainable direction, provided the implementation centralizes gate resolution so discovery, runtime behavior, and account overrides cannot drift.
