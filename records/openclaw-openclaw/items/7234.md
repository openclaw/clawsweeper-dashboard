## Summary

Keep open. Current main still exposes Discord `actions.messages` as one boolean gate and uses it for Discord send/read/edit/delete behavior, so the requested granular action gates remain unimplemented and need a maintainer config-contract decision.

## What This Changes

Requested a backward-compatible Discord config split so message send, read, edit, and delete actions can be enabled independently instead of sharing `actions.messages`.

## Best Possible Solution

Add a centralized Discord message-action gate resolver that preserves legacy boolean behavior, then update types, strict schema, generated metadata, discovery, runtime checks, docs, changelog, and focused Discord tests.

## Reproduction Assessment

Yes for the current behavior: source inspection shows `actions.messages` is boolean-only and the same key gates Discord send/read/edit/delete runtime behavior. A live Discord run is not required to establish the missing config surface.

## Solution Assessment

Unclear until maintainers choose the public config contract. A backward-compatible granular split is the narrow maintainable direction, but nesting, defaults, and `upload-file` semantics need owner approval.
