## Summary

Keep open: current `main` still has the reported Telegram-specific gap. The shared reasoning/directive paths and docs already define configured reasoning defaults, but Telegram’s dispatch-time resolver only reads the session store and falls back to `off`, so the related closed Web UI issue did not cover this surface.

## What This Changes

The issue requests that Telegram reasoning resolution fall back to `agents.list[].reasoningDefault` when a session has no stored reasoning level.

## Best Possible Solution

Update the Telegram plugin’s dispatch-time reasoning resolver to mirror the documented/shared resolution order for session override, per-agent default, global agent default, and fallback, with Telegram-focused regression tests and a changelog entry.

## Reproduction Assessment

Yes from source inspection: with a valid `SessionKey` whose store entry has no `reasoningLevel`, `resolveTelegramReasoningLevel` reaches `return "off"` before consulting `agents.list[].reasoningDefault`. I did not establish a live Telegram failing path in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to make Telegram’s pre-dispatch preview/block-stream reasoning resolver match the existing shared default semantics. Broader global or per-model default feature work from related issues should stay out of this fix.
