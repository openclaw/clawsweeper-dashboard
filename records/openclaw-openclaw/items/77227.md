## Summary

Keep open: current `main` still has the reported Telegram-specific gap, and open PR #77236 already references this issue with closing syntax. Telegram only reads the session store and then falls back to `off`, while the shared reasoning path and docs define configured per-agent/default fallback semantics.

## What This Changes

The issue requests that Telegram reasoning resolution fall back to configured `agents.list[].reasoningDefault` before returning `off` for fresh sessions without a stored `/reasoning` choice.

## Best Possible Solution

Review and land PR #77236 or an equivalent narrow Telegram fix that matches the documented shared reasoning-default order, including appropriate authorization semantics and regression coverage.

## Reproduction Assessment

Yes from source inspection: with a valid Telegram `sessionKey` whose store entry has no `reasoningLevel`, the current resolver reaches `return "off"` before consulting configured defaults. I did not run a live Telegram message path in this read-only review.

## Solution Assessment

Yes: the maintainable fix belongs in the Telegram dispatch-time resolver so it matches the already documented/shared default order. The open linked PR is the current implementation venue, so the issue should stay open until that PR is merged or replaced.
