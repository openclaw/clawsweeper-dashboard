## Summary

Keep open: current main still has the Telegram-specific fallback gap, and open PR #77236 is already linked with closing syntax as the implementation vehicle.

## What This Changes

The issue requests that Telegram's reasoning resolver fall back to `agents.list[].reasoningDefault` when no session `reasoningLevel` is stored.

## Best Possible Solution

Review and land a focused Telegram fix, likely #77236 after any needed authorization-parity adjustment, so Telegram follows the same session, per-agent, global, and `off` ordering as the shared resolver.

## Reproduction Assessment

Yes. From source inspection, a Telegram session with a valid `SessionKey` and no stored `reasoningLevel` reaches `return "off"` before consulting configured reasoning defaults.

## Solution Assessment

Yes. The narrow maintainable fix is Telegram-local resolver parity with the existing documented/shared default semantics, with focused regression tests rather than broader per-model or global feature work.
