Keep open. Current main still only denies unlisted Telegram groups by dropping/logging normal messages or returning a static native-command denial, while the requested configurable ignore, warn, cooldown, and auto-leave behavior remains in the paired open PR #50682.

## What This Changes

Request support for Telegram unknown-group handling with configurable ignore, warn, cooldown, and auto-leave behavior before assistant execution.

## Best Possible Solution

Review and either land PR #50682 or replace it with a narrow Telegram-plugin preflight change that keeps normal messages and native commands consistent and updates public config, docs, and tests.

## Reproduction Assessment

Yes, source-level. Configure Telegram with a groups allowlist and send from an unlisted group; current main reaches group-chat-not-allowed, drops or statically denies it, and has no configurable warn, cooldown, or leave path.

## Solution Assessment

Yes. Telegram inbound preflight is the right boundary because it runs before assistant execution; the maintainable fix should handle only the unknown-group branch and share that behavior across normal message and native command paths.
