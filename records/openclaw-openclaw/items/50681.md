## Summary

Keep open. Current main still fail-closes unknown Telegram groups by blocking or dropping them, but it does not provide the requested configurable ignore/warn/leave behavior; the same-author implementation path remains open at https://github.com/openclaw/openclaw/pull/50682.

## What This Changes

The issue requests a Telegram config surface to ignore, warn, or leave unapproved groups before assistant execution.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/50682 with a narrow Telegram preflight implementation that updates config types/schema/docs, handles normal messages and native commands consistently, preserves forum-topic delivery, and proves cooldown plus leave behavior before closing this issue.

## Reproduction Assessment

Not applicable as a feature request. Source and docs give a high-confidence verification path that current main lacks the requested config and runtime action: unknown groups become group-chat-not-allowed, then normal messages are dropped or native commands get a static denial.

## Solution Assessment

Unclear until maintainers accept the product/security behavior. Telegram inbound preflight is the right owner boundary, but optional auto-leave needs a deliberate contract across allowlists, topics, native commands, and callbacks.
