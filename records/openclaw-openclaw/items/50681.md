## Summary

Keep open: current main and v2026.6.10 still fail-close unknown Telegram groups by dropping/logging normal messages or sending a fixed native-command denial, but they do not provide the requested configurable ignore/warn/leave behavior; the paired PR remains open and needs maintainer product/security review.

## What This Changes

Add opt-in Telegram handling for non-allowlisted groups so OpenClaw can ignore, warn, or leave with optional message/cooldown before assistant execution.

## Best Possible Solution

Review and land, revise, or replace the paired Telegram PR with a narrow opt-in implementation in the Telegram plugin, including schema/types/docs, shared normal-message and native-command behavior, cooldown handling, and real Telegram proof.

## Reproduction Assessment
