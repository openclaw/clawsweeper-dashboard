## Summary

Keep open. Current main still fail-closes unknown Telegram groups by blocking or dropping them, but it does not implement the requested configurable ignore, warn, or leave behavior, and the same-author implementation PR remains open for review.

## What This Changes

Add a stricter Telegram unknown-group policy that can ignore, warn, or leave unapproved group chats before assistant execution.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/50682 with a narrow Telegram preflight implementation that updates config/schema/docs, shares behavior across normal messages and native commands, preserves topic delivery, and proves cooldown plus leave behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs give a high-confidence verification path that current main lacks the requested config and runtime action: unknown groups become group-chat-not-allowed, then normal messages are dropped or native commands get a static denial.

## Solution Assessment

Unclear until maintainers accept the product and security contract. Telegram inbound preflight is the right owner boundary, but optional auto-leave should be implemented through a shared policy path that covers normal messages, native commands, topics, callbacks, and cooldown state.
