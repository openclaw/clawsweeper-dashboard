## Summary

Keep open. Current main has a partial pre-token `waiting` spinner, but TUI state is still modeled as free-form strings and does not expose the requested Idle/Generating/Working/Waiting/Reconnecting/Disconnected contract; yielded-parent progress also overlaps the still-open broader parent-session issue at https://github.com/openclaw/openclaw/issues/58067.

## What This Changes

Clarify the TUI status line by separating idle, token generation, tool work, yielded subagent waits, reconnecting, and disconnected states instead of reusing generic streaming/connected labels.

## Best Possible Solution

Define a typed TUI activity-state contract derived from run lifecycle, token deltas, tool events, yielded waits, and connection status, aligned with the broader parent-session progress semantics.

## Reproduction Assessment

Yes, at source level. Current main still routes TUI status through free-form activity strings and lacks yielded/liveness metadata in the visible TUI state contract, so the requested distinctions cannot be represented consistently.

## Solution Assessment

Unclear. The direction is sound, but the maintainable fix is a typed state contract plus maintainer-approved labels rather than a narrow renderer string rename.
