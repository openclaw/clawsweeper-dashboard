## Summary

Keep open. Current main partially improves the pre-token gap with `waiting`, `running`, `streaming`, and `finishing context`, but the TUI still uses free-form activity strings rather than the requested stable Idle/Generating/Working/Waiting/Reconnecting/Disconnected state contract; yielded-subagent waiting also overlaps the still-open parent-session progress work at https://github.com/openclaw/openclaw/issues/58067.

## What This Changes

Request to replace ambiguous TUI connection/activity text with distinct user-visible labels for idle, token generation, tool work, yielded subagent waiting, reconnecting, and disconnection.

## Best Possible Solution

Define a typed run/session-state contract and map it to stable TUI labels, keeping yielded-subagent semantics aligned with https://github.com/openclaw/openclaw/issues/58067 instead of only renaming current strings.

## Reproduction Assessment

Yes, at source level: current main still models TUI activity as free-form strings and maps events to `waiting`, `streaming`, `running`, and `finishing context`, not the requested stable labels. I did not run a live TUI session in this read-only review.

## Solution Assessment

No, closing or doing a narrow text swap would not be the best fix. The maintainable path is a typed state contract that both TUI labels and yielded-subagent progress can consume consistently.
