## Summary

Keep open. Current main has a partial pre-token `waiting` spinner, but the TUI still uses free-form status strings and does not expose the requested Idle/Generating/Working/Waiting/Reconnecting/Disconnected state contract; the yielded-parent part also overlaps the still-open broader parent-session progress issue at https://github.com/openclaw/openclaw/issues/58067.

## What This Changes

The issue requests distinct TUI connected-session labels for idle, generating, working, yielded waiting, reconnecting, and disconnected states instead of overloading streaming or idle text.

## Best Possible Solution

Define a typed TUI activity-state contract derived from run lifecycle, token deltas, tool events, yielded waits, and connection status, then map that contract to maintainer-approved labels.

## Reproduction Assessment

Yes, at source level. Current main still routes TUI status through free-form activity strings and lacks yielded/liveness metadata in the visible TUI state contract, so the requested distinctions cannot be represented consistently.

## Solution Assessment

Unclear until maintainers choose the exact state contract and labels. The direction is sound, but a maintainable fix should be a typed contract plus label mapping rather than a narrow string rename.
