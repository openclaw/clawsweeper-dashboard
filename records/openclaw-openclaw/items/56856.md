## Summary

Keep open. Current main and the latest release only partially cover the pre-token gap with a `waiting` spinner; the TUI still uses free-form activity strings and does not expose a typed Idle/Generating/Working/Waiting/Reconnecting/Disconnected state contract. The yielded-parent part also overlaps the still-open broader parent-session progress issue at https://github.com/openclaw/openclaw/issues/58067.

## What This Changes

Request clearer TUI session-state labels that distinguish idle, active generation, non-streaming work, yielded waiting, reconnecting, and disconnected states.

## Best Possible Solution

Define a typed TUI activity-state model fed by run lifecycle, token deltas, tool events, yielded waits, and connection state, then map it to maintainer-approved labels consistently with parent-session progress work.

## Reproduction Assessment

Yes, at source level. Current main still routes visible TUI status through free-form strings and lacks yielded/liveness metadata in a typed visible state contract, so the requested distinctions cannot be represented consistently.

## Solution Assessment

Unclear. The direction is sound, but a renderer string rename would be brittle; the maintainable fix is a typed activity-state contract plus maintainer-approved labels.
