## Summary

Keep open: current main and the latest release still keep failover details in error metadata/logs without a channel-delivered notice, and the linked implementation PRs are open and unmerged. The Discord confirmation adds useful cross-channel evidence that this is not a single-plugin report.

## What This Changes

The issue requests a user-visible channel notification when model rate-limit, billing, or overload failover/suspension happens instead of silent dead air.

## Best Possible Solution

Land or replace the linked implementation with a core/channel delivery hook that reuses shared failover classification, emits a tagged gateway_failure notice for rate-limit, billing, and overload failures, and includes focused tests plus real channel proof.
