## Summary

Keep open. Current main still has a source-reproducible Active Memory bug: Telegram forum-topic hook context exposes the topic conversation id as `channelId`, and Active Memory forwards that value as the embedded recall run's channel/plugin id, which can hit bundled-plugin directory-name validation before recall completes.

## What This Changes

Active Memory should run or skip safely for Telegram supergroup forum-topic sessions when group chat types are enabled, instead of failing recall with a bundled-plugin directory-name validation error.

## Best Possible Solution

Keep the hook contract intact and fix Active Memory to use the provider/plugin channel id for embedded recall runs while preserving the topic conversation id only as session context.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call the Active Memory `before_prompt_build` hook with `sessionKey=agent:main:telegram:group:-100123:topic:77`, `messageProvider=telegram`, `channelId=-100123:topic:77`, and group chat types enabled; current code forwards the topic id as the embedded run channel.

## Solution Assessment

Yes. The narrow maintainable fix is to keep `channelId` as conversation metadata for hooks but make Active Memory resolve its embedded run channel from a real provider/channel id such as `telegram` or a strong stored session channel.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76704. Active Memory currently treats hook `ctx.channelId` as a runnable channel id, but for Telegram forum topics that field is the conversation id (`<group-id>:topic:<topic-id>`), which can be forwarded to `runEmbeddedPiAgent` as `messageChannel`/`messageProvider` and rejected by bundled plugin dirName validation. Keep the hook metadata contract intact; in `extensions/active-memory/index.ts`, adjust `resolveRecallRunChannelContext()` so conversation-shaped channel ids do not override a real provider/plugin id. Prefer a strong stored session channel (`lastChannel`/`channel`), otherwise use `messageProvider` for the embedded recall run; preserve existing wrapper-channel behavior. Add regression coverage in `extensions/active-memory/index.test.ts` for a group-allowed Telegram topic session with `sessionKey: "agent:main:telegram:group:-100123:topic:77"`, `messageProvider: "telegram"`, and `channelId: "-100123:topic:77"`, asserting the embedded run uses `messageChannel: "telegram"` and `messageProvider: "telegram"` and still scopes the Active Memory subagent to the parent topic session. Add a single-line user-facing `CHANGELOG.md` fix entry. Do not change Telegram topic session key grammar or the generic bundled plugin dirName validator.
