## Summary

Keep open: current main still has source-backed risk that synchronous builtin memory SQLite work can stall the gateway event loop, leave interrupted memory temp databases behind, and contribute to Slack Socket Mode outages; no merged fix or narrower canonical item covers the memory-core side of this report.

## What This Changes

Investigate Slack Socket Mode outages tied to synchronous builtin memory SQLite maintenance, bloated `main.sqlite`, and orphaned `main.sqlite.tmp-*` files after interrupted reindex work.

## Best Possible Solution

Move heavy builtin memory SQLite reindex and maintenance off the gateway event loop, sweep stale memory temp sidecars on startup or doctor repair, enforce a real embedding-cache bound, and keep Slack Socket Mode recovery self-healing under transient stalls.
