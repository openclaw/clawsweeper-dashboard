## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate scratchpad filtering away from final answers:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: At this call site, a final reply from `openai-codex` is treated as no text whenever the heuristic matches. The new test asserts a `kind: "final"` message sends nothing, so normal English final answers containing words like `could` or `need` can disappear in Feishu with no fallback. Gate the scratchpad filter to non-final/internal chunks or otherwise guarantee final replies are delivered.
  - confidence: 0.94
- **[P2] Stop heartbeat timers during cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: Starting the heartbeat here creates an interval even if streaming start later fails or the dispatcher is cleaned up. `onCleanup` only runs typing cleanup, so the interval can outlive the dispatcher until `closeStreaming()` happens. Stop the timer from cleanup and failed-start paths.
  - confidence: 0.86
- **[P2] Check completion pings before resetting stream state:** `extensions/feishu/src/reply-dispatcher.ts:561-563`
  - body: `closeStreaming()` clears `streamingStartedAt` before this check, so `shouldSendCompletionPing()` always sees `0` and the long-task completion ping never sends. Capture the elapsed/should-ping state before closing or reset state after the ping.
  - confidence: 0.91
