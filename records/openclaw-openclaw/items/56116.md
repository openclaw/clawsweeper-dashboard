Full review comments:

- **[P1] Gate scratchpad filtering away from final answers:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: `hasText` is cleared before `info.kind` is considered, so the broad openai-codex scratchpad heuristic can match normal final English answers containing words like `need` or `could`. That makes `deliver()` return without sending the user's final reply; restrict the filter to non-final/internal chunks or add a final-delivery fallback.
  - confidence: 0.92
- **[P2] Stop heartbeat timers during cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` creates an interval for streaming cards, but `onCleanup` only clears typing state. If a dispatcher is torn down before `onIdle` or `onError`, the interval can keep state alive and continue scheduling note updates, so cleanup should also stop the heartbeat.
  - confidence: 0.88
- **[P2] Check completion pings before resetting stream state:** `extensions/feishu/src/reply-dispatcher.ts:561-563`
  - body: The final streaming path calls `closeStreaming()` before `shouldSendCompletionPing()`, and `closeStreaming()` resets `streamingStartedAt` to zero. That makes the long-task completion ping condition false for the path this PR advertises; capture the duration or ping decision before closing/resetting.
  - confidence: 0.9
- **[P2] Remove the stale card note local:** `extensions/feishu/src/reply-dispatcher.ts:420`
  - body: This block still computes `cardNote`, but the start payload now uses `buildStreamingNote(...)` instead. With extension typechecking enforcing unused locals, this stale local should be removed before the branch can pass the changed-surface typecheck.
  - confidence: 0.86
