## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate scratchpad filtering away from final answers:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: This applies the broad scratchpad heuristic before final delivery as well as block delivery. An ordinary final answer such as `I could not find the file you requested. Please try again.` can match the 8+ word ASCII/could/try branch and return before sending anything, leaving Feishu users with a silent no-reply.
  - confidence: 0.91
- **[P2] Stop heartbeat timers during cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` creates an interval even if the streaming start later fails, but `onCleanup` only cleans typing state and never stops the heartbeat. A torn-down dispatcher can keep a 15s interval alive after the reply lifecycle ends.
  - confidence: 0.84
- **[P2] Check completion duration before clearing stream state:** `extensions/feishu/src/reply-dispatcher.ts:561-563`
  - body: `closeStreaming()` resets `streamingStartedAt` to 0 before this new `shouldSendCompletionPing()` check runs, so the long-task completion ping can never be sent on the streaming-final path. Capture the elapsed state before closing, or move the ping decision before the reset.
  - confidence: 0.89
