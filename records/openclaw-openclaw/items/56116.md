Full review comments:

- **[P1] Gate scratchpad filtering away from final replies:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: `hasText` is set to false whenever the heuristic matches, even for `kind: "final"`. The broad English regex can match normal final answers like “I could not find the file you requested. Please try again.”, which leaves the user with no visible Feishu reply; restrict this to structured non-final reasoning/progress signals or otherwise preserve final answers.
  - confidence: 0.91
- **[P1] Preserve reasoning-preview opt-in during rebase:** `extensions/feishu/src/reply-dispatcher.ts:659-668`
  - body: This branch exposes `onReasoningStream` whenever streaming is enabled, but current main gates Feishu reasoning preview behind `allowReasoningPreview === true`. Carrying this stale block forward would re-open the exact path that can show private reasoning/scratchpad text in chat, so the rebase needs to keep the current opt-in gate.
  - confidence: 0.86
- **[P2] Check the completion ping before resetting stream state:** `extensions/feishu/src/reply-dispatcher.ts:563`
  - body: `closeStreaming()` resets `streamingStartedAt` to `0`, and the code checks `shouldSendCompletionPing()` only afterward. That makes the long-task completion confirmation unreachable; capture the duration decision before closing/resetting or return it from the close path.
  - confidence: 0.88
- **[P2] Stop the heartbeat on failed startup and cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` starts an interval outside the streaming-start async block, but failed startup and `onCleanup` can leave the timer alive without reaching `closeStreaming()`. Stop the heartbeat in cleanup and in the failed-start path so torn-down dispatchers do not keep scheduling note refreshes.
  - confidence: 0.82
- **[P2] Publish the new statusLanguage contract:** `extensions/feishu/src/config-schema.ts:186`
  - body: The PR accepts `statusLanguage` in the Zod schema, but the Feishu plugin manifest and docs list config keys separately and do not expose this option. Add the key to the public metadata/docs surfaces, or avoid adding a config option until the contract is fully aligned.
  - confidence: 0.76
