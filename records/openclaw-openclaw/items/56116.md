## Summary

Keep open: current main does not yet implement the requested Feishu UX, and the PR still has source-reproducible correctness blockers that can silently suppress final replies, leak heartbeat timers, and prevent completion pings. The prior Greptile and ClawSweeper comments point at the same core risks, and source inspection confirms them.

## What This Changes

This PR adds Feishu streaming status notes with localized status/last-activity text, a long-task completion ping, Codex scratchpad filtering, account-level `statusLanguage`, and focused tests.

## Best Possible Solution

Keep the feature in the Feishu plugin, but land it only after final replies are never silently filtered, heartbeat teardown is idempotent, completion ping timing is computed before state reset, current-main streaming fixes are preserved, and docs/changelog cover the new public behavior.

## Reproduction Assessment

Yes. Source inspection of PR head gives high-confidence reproduction paths: deliver an `openai-codex` final English reply that matches the heuristic, run cleanup before idle/error after heartbeat start, and follow the final streaming path where `closeStreaming()` resets `streamingStartedAt` before the ping check.

## Solution Assessment

No. The feature direction belongs in the Feishu plugin, but this implementation is not the best fix until scratchpad suppression is gated away from final user-visible replies and lifecycle state is handled explicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not filter final replies as scratchpad text:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: `hasText` is cleared whenever the broad scratchpad heuristic matches, before checking `info.kind`. For openai-codex sessions this can match normal final English answers containing words like `could` or `need`, causing the final reply to return without sending anything. Restrict the filter to non-final/internal chunks or require a stronger scratchpad signal.
  - confidence: 0.91
- **[P2] Stop heartbeat timers during cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` creates an interval, but `onCleanup` does not call `stopHeartbeat()` or `closeStreaming()`. If the dispatcher is torn down before `onIdle` or `onError`, the interval survives and keeps the dispatcher state reachable. Stop the heartbeat from cleanup as well as closeout.
  - confidence: 0.87
- **[P2] Decide completion pings before closing streaming:** `extensions/feishu/src/reply-dispatcher.ts:563`
  - body: The final streaming path calls `closeStreaming()` before `shouldSendCompletionPing()`, and `closeStreaming()` resets `streamingStartedAt` to zero. That makes the completion ping condition false even for long tasks, so the advertised completion confirmation never fires. Capture the duration or ping decision before resetting stream state.
  - confidence: 0.9
- **[P3] Add the required changelog and docs entry:** `extensions/feishu/src/config-schema.ts:186`
  - body: This PR adds user-facing Feishu streaming status, completion, filtering, and account config behavior, but it does not update `CHANGELOG.md` or the Feishu docs for the new `statusLanguage` option. Repo policy expects user-facing behavior/API changes to be documented before merge.
  - confidence: 0.78
