## Summary

Keep open. The PR targets a real Feishu plugin gap that current main does not implement, but the proposed patch still has concrete correctness blockers already echoed by Greptile and the prior ClawSweeper review: valid final replies can be filtered, heartbeat cleanup can leak, and completion pings never fire after state reset.

## What This Changes

The PR adds Feishu account-level status-language config, localized streaming-card status and last-activity notes, note heartbeat updates, a long-task completion ping, Codex scratchpad filtering, and focused Feishu tests.

## Best Possible Solution

Keep the feature in the Feishu plugin, but land it only after final replies are never silently filtered, heartbeat teardown is idempotent, completion ping timing is computed before reset, and docs/changelog cover the new public behavior.

## Reproduction Assessment

Yes. The PR bugs have high-confidence source-level reproduction paths: deliver an openai-codex final English reply that matches the heuristic, run cleanup before idle/error after heartbeat start, and follow the final streaming path where closeStreaming resets streamingStartedAt before the ping check.

## Solution Assessment

No. The feature direction is appropriate for the Feishu plugin, but this implementation is not the safest solution until scratchpad suppression is gated away from final user-visible replies and lifecycle state is handled explicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not filter final replies as scratchpad text:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: `hasText` is cleared whenever the broad scratchpad heuristic matches, before checking `info.kind`. For openai-codex sessions this can match normal final English answers containing words like "could" or "need", causing the final reply to return without sending anything. Restrict the filter to non-final/internal chunks or require a stronger scratchpad signal.
  - confidence: 0.91
- **[P2] Stop heartbeat timers during cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` creates an interval, but `onCleanup` does not call `stopHeartbeat()` or `closeStreaming()`. If the dispatcher is torn down before `onIdle` or `onError`, the interval survives and keeps the dispatcher state reachable. Stop the heartbeat from cleanup as well as closeout.
  - confidence: 0.87
- **[P2] Decide completion pings before closing streaming:** `extensions/feishu/src/reply-dispatcher.ts:563`
  - body: The final streaming path calls `closeStreaming()` before `shouldSendCompletionPing()`, and `closeStreaming()` resets `streamingStartedAt` to zero. That makes the completion ping condition false even for long tasks, so the advertised completion confirmation never fires. Capture the duration or ping decision before resetting stream state.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `extensions/feishu/src/config-schema.ts:183-184`
  - body: This PR adds user-facing Feishu streaming status, completion, filtering, and account config behavior but does not update `CHANGELOG.md`. Repo policy requires user-facing feature/fix/perf changes to add an active-version changelog entry with appropriate contributor credit.
  - confidence: 0.78
