## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate scratchpad filtering away from final answers:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: `hasText` is cleared through the broad openai-codex scratchpad heuristic before `info.kind` is considered, so normal final English answers containing words like `need` or `could` can be silently dropped. Restrict this filter to non-final/internal chunks or add a final-delivery fallback.
  - confidence: 0.92
- **[P2] Stop heartbeat timers during cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` creates an interval for streaming cards, but `onCleanup` never stops it unless `closeStreaming()` already ran. A dispatcher teardown before `onIdle` or `onError` can leave note updates scheduled after cleanup.
  - confidence: 0.88
- **[P2] Check completion pings before resetting stream state:** `extensions/feishu/src/reply-dispatcher.ts:561-563`
  - body: The final streaming path calls `closeStreaming()` before `shouldSendCompletionPing()`, and `closeStreaming()` resets `streamingStartedAt` to zero. Capture the duration or ping decision before closing so long-task completion pings can fire.
  - confidence: 0.9
- **[P2] Remove the stale card note local:** `extensions/feishu/src/reply-dispatcher.ts:420`
  - body: Changing the start payload to `buildStreamingNote(...)` leaves the earlier `cardNote` local unused. `tsconfig.extensions.json` enables `noUnusedLocals`, so this branch should remove that local before type validation can pass.
  - confidence: 0.84
