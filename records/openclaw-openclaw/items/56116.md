## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate scratchpad filtering away from final answers:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: `hasText` is cleared before `info.kind` is considered. For `openai-codex`, the broad English heuristic can match normal final answers containing words like `need` or `could`, causing `deliver()` to return without sending the user's final reply. Restrict the filter to non-final/internal chunks or add a final-delivery fallback.
  - confidence: 0.92
- **[P2] Stop heartbeat timers during cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` creates an interval for streaming cards, but `onCleanup` only clears typing state. If a dispatcher is torn down before `onIdle` or `onError`, the interval can keep the dispatcher state alive and continue scheduling note updates. Stop the heartbeat from cleanup as well as closeout.
  - confidence: 0.88
- **[P2] Check completion pings before resetting stream state:** `extensions/feishu/src/reply-dispatcher.ts:561-563`
  - body: The final streaming path calls `closeStreaming()` before `shouldSendCompletionPing()`, and `closeStreaming()` resets `streamingStartedAt` to zero. That makes the long-task completion ping condition false for the path this PR advertises. Capture the duration or ping decision before closing/resetting.
  - confidence: 0.9
- **[P2] Remove the stale card note local:** `extensions/feishu/src/reply-dispatcher.ts:426`
  - body: After this line switches the start payload to `buildStreamingNote(...)`, the earlier `const cardNote = resolveCardNote(...)` in the same block is unused. Extension typechecking has `noUnusedLocals` enabled, so the PR will fail the changed-surface typecheck until the stale local is removed.
  - confidence: 0.86
