## What This Changes

## Best Possible Solution

Use trusted session-store or record-level cron provenance for archive-wide suppression while keeping user-authored `[cron:...]` archive content searchable.

## Reproduction Assessment

Yes, source-reproducible: current `buildSessionEntry` and the PR's first-user path can classify a usage-counted archive as cron-generated from user text, clear collected content, and skip later messages before memory sync indexes it. ## Solution Assessment

No. The first-user-message heuristic is a partial fix because ordinary human sessions can start with `[cron:...]`; the durable fix needs trusted cron provenance or an explicit maintainer decision to narrow the supported case.
