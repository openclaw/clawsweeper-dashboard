## Summary

Keep open. Current main still returns a usable shared GitHub Copilot API token cache entry before consulting the selected GitHub token, and https://github.com/openclaw/openclaw/pull/43659 is still open with closing syntax for this exact bug.

## What This Changes

The issue reports that GitHub Copilot auth-profile rotation reuses one cached Copilot API token across distinct GitHub PAT-backed profiles.

## Best Possible Solution

Land or replace the linked provider-auth fix so cache reuse is bound to the selected GitHub token/profile, legacy unbound cache entries miss, and focused regression coverage locks same-profile hits and cross-profile misses.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a valid shared cached Copilot API token with integrationId and more than five minutes remaining is returned before resolveCopilotApiToken uses a different githubToken from a rotated profile.

## Solution Assessment

Yes. The narrow maintainable fix is to make the shared provider-auth cache owner-aware while preserving integrationId behavior and adding focused regression tests for same-profile reuse, cross-profile miss, and legacy unbound cache miss.
