## Summary

Keep open. Current main and the latest release still use a single GitHub Copilot token cache entry that can be returned before the rotated GitHub PAT is used, and the linked closing PR is open rather than merged.

## What This Changes

The issue requests that GitHub Copilot API token caching respect the selected auth profile so rate-limit rotation can use a distinct account token.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43659 with an owner-aware cache fix in the canonical Copilot token helper, including same-profile hit, cross-profile miss, and legacy unowned-cache coverage.

## Reproduction Assessment

Yes, source-level: current main can return a usable shared Copilot token from src/plugin-sdk/provider-auth.ts before it uses the rotated params.githubToken. I did not run a live multi-account rate-limit scenario.

## Solution Assessment

Yes. The narrow maintainable fix is to bind the centralized Copilot token cache to the selected GitHub token or profile and keep this issue open until the linked implementation path merges or is replaced.
