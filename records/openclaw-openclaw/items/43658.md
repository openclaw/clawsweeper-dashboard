## Summary

Keep open. Current main still returns a usable shared GitHub Copilot API-token cache entry before using the rotated GitHub PAT, and https://github.com/openclaw/openclaw/pull/43659 remains open with closing syntax for this bug.

## What This Changes

The issue reports that multiple GitHub Copilot auth profiles rotate at the profile layer but still share one cached Copilot API token, defeating rate-limit recovery.

## Best Possible Solution

Land or replace the linked provider-auth fix so cached Copilot API tokens are validated against the selected GitHub token/profile, legacy unbound cache entries miss once, and focused regression tests cover same-profile hits and cross-profile misses.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a valid shared cache entry in src/plugin-sdk/provider-auth.ts returns before the rotated params.githubToken is used for token exchange.

## Solution Assessment

Yes. The narrow maintainable fix is to make the centralized provider-auth Copilot cache owner-aware while preserving integrationId behavior and adding regression coverage for cross-profile and legacy unbound cache misses.
