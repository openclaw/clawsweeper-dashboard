## Summary

Keep open: current main still has a source-reproducible Copilot token-cache bug, and the open linked repair PR is unmerged and now needs refresh against the current provider-auth helper location.

## What This Changes

The issue reports that GitHub Copilot auth profile rotation reuses one shared cached Copilot API token across different GitHub PAT-backed profiles, so rate-limit rotation does not actually switch accounts.

## Best Possible Solution

Land a focused fix in the canonical Copilot token helper that binds cached Copilot API tokens to the requesting GitHub credential, with regression coverage for different `githubToken` values and legacy cache misses.

## Reproduction Assessment

Yes. Source inspection shows a valid cached Copilot token is returned before `params.githubToken` is used for exchange, so two different profiles can receive the same cached Copilot API token; I did not run a live two-account Copilot repro in this read-only pass.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the canonical cache entry credential-aware in `src/plugin-sdk/provider-auth.ts`, while refreshing the linked PR away from the older wrapper-file implementation.
