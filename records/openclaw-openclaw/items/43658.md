## Summary

Keep open: current main and v2026.6.6 still use a single account-blind Copilot API token cache, and the open closing PR is unresolved/conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to make GitHub Copilot API token cache reuse respect rotated auth profiles instead of sharing one cached token across accounts.

## Best Possible Solution

Land or replace a narrow fix in `src/plugin-sdk/provider-auth.ts` so cache hits are scoped to the GitHub credential/account, with regression coverage and redacted real two-profile proof.

## Reproduction Assessment

Yes at source level: current main returns a valid cached Copilot API token before consulting the selected GitHub token, while auth-profile code can rotate to another profile. I did not perform a live two-account Copilot quota run.

## Solution Assessment

Yes: credential-bound cache identity in the canonical provider-auth helper is the narrow maintainable fix. The linked PR has the right root-cause direction but needs to be reconciled with the current owner and proved in a real setup before this issue closes.
