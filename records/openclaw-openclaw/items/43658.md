## Summary

Keep open: current main and latest release v2026.6.8 still reuse a single GitHub Copilot token cache without tying it to the requesting GitHub credential, and https://github.com/openclaw/openclaw/pull/43659 is open and unmerged as the linked repair candidate.

## What This Changes

The issue reports that GitHub Copilot auth profile rotation after a rate limit still reuses the first account's cached Copilot API token because the token cache is not credential-aware.

## Best Possible Solution

Land a focused fix in the canonical Copilot token helper that binds cached Copilot API tokens to the requesting GitHub credential, with regression coverage for profile switches and legacy cache misses.

## Reproduction Assessment

Yes. Source inspection shows current main returns a usable shared Copilot token cache entry before using params.githubToken, so two different profiles can receive the same cached Copilot API token.

## Solution Assessment

Yes. The narrowest maintainable fix is credential-aware caching in src/plugin-sdk/provider-auth.ts; the linked PR should be refreshed to the current canonical helper surface before landing.
