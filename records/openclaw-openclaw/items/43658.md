## Summary

Keep open: current main and v2026.6.6 still use an account-blind single Copilot token cache, and the open closing PR is unresolved rather than merged.

## What This Changes

The issue reports that GitHub Copilot profile rotation reuses one cached Copilot API token across different GitHub PAT-backed auth profiles.

## Best Possible Solution

Patch the canonical `src/plugin-sdk/provider-auth.ts` helper so Copilot token cache reuse is scoped to the GitHub credential or account while preserving single-profile cache behavior.

## Reproduction Assessment

Yes at source level: current main returns a valid cached Copilot API token before consulting `params.githubToken`, while session auth-profile code can rotate to another profile. I did not perform a live two-account Copilot rate-limit run.

## Solution Assessment

Yes, credential-bound cache identity in the canonical provider-auth helper is the narrow maintainable fix. The current linked PR’s direction is right, but it still needs to be reconciled with the current canonical file and proved in a real setup before the issue closes.
