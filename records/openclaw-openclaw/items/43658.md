## Summary

Keep open. Current main still accepts an unowned shared GitHub Copilot API token cache hit before using the selected GitHub token, and an open same-author PR with closing syntax already tracks the focused fix.

## What This Changes

Bind GitHub Copilot API token cache reuse to the selected auth profile so profile rotation uses a token from the rotated account.

## Best Possible Solution

Review and land or replace the linked fix with a narrow provider-auth change that ties cache reuse to the selected GitHub token/profile and treats legacy unbound cache entries as misses.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a valid cached Copilot API token, resolveCopilotApiToken returns it before using a different githubToken from a rotated profile.

## Solution Assessment

Yes. The narrow maintainable fix is to make the shared provider-auth cache owner-aware, with regression coverage for same-profile hits, cross-profile misses, and legacy cache entries.
