## Summary

Keep open: current main still uses one ownerless GitHub Copilot API-token cache before the rotated profile token is consulted, and an open closing PR is already the active fix path.

## What This Changes

Make the GitHub Copilot API-token cache profile-bound so auth-profile rotation uses the selected GitHub account after rate limits.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43659 with an owner-bound cache fix in src/plugin-sdk/provider-auth.ts plus regression coverage for same-profile hits, cross-profile misses, and legacy unowned cache entries.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: populate the shared Copilot token cache with profile A, then select profile B while the cached token remains usable; current main can return A's cached Copilot API token before using B's GitHub token.

## Solution Assessment

Yes. Binding the centralized Copilot token cache to the selected GitHub token or profile is the narrow maintainable fix, but this issue should stay open until the linked implementation merges or is replaced.
