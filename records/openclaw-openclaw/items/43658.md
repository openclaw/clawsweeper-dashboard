## Summary

Keep open. Current main still reuses any usable shared GitHub Copilot API-token cache entry before consulting the rotated GitHub PAT, and the open linked fix PR still targets this exact bug.

## What This Changes

The issue asks for GitHub Copilot token caching to be bound to the selected auth profile so rate-limit rotation uses the next account's Copilot API token.

## Best Possible Solution

Land or replace the linked provider-auth fix so cached Copilot API tokens are validated against the selected GitHub token/profile, legacy unbound cache entries miss once, and focused regression tests cover same-profile hits and cross-profile misses.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a usable shared cache entry in src/plugin-sdk/provider-auth.ts returns before the rotated params.githubToken is used for exchange.

## Solution Assessment

Yes. The narrow maintainable fix is to make the shared provider-auth cache owner-aware while preserving integrationId behavior and adding focused tests for same-profile reuse, cross-profile miss, and legacy unbound cache miss.
