## Summary

Keep open. Current main still returns a usable shared Copilot token before consulting the selected GitHub token, and #43659 remains the open same-author PR with closing syntax, so this issue should wait for that PR to be reviewed, landed, or closed.

## What This Changes

Bind GitHub Copilot API token cache entries to the selected GitHub auth credential so profile rotation exchanges a distinct Copilot API token.

## Best Possible Solution

Review and land or close #43659; the durable fix should make the shared Copilot token cache profile-bound and cover cross-profile plus legacy-cache cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: with a valid cached Copilot API token, calling resolveCopilotApiToken with a different githubToken returns the cached token before using the new GitHub token.

## Solution Assessment

Yes. Binding cache reuse to a non-secret fingerprint or equivalent profile-safe key in the shared helper is the narrow maintainable fix; the linked open PR targets that direction.
