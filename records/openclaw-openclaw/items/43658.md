## Summary

Keep open. Current main still reuses any usable shared Copilot API token before checking which GitHub token/profile requested it, and an open same-author PR with closing syntax already tracks the narrow fix.

## What This Changes

The issue requests profile-bound GitHub Copilot API token caching so auth profile rotation exchanges tokens with the newly selected GitHub PAT.

## Best Possible Solution

Review and land or close #43659; the durable fix should bind Copilot cache reuse to the selected GitHub token/profile and cover legacy unbound cache entries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a valid cached Copilot API token, resolveCopilotApiToken returns it before using a different githubToken from a rotated profile.

## Solution Assessment

Yes. A profile-safe cache key or non-secret token fingerprint in the shared helper is the narrow maintainable fix, and the linked open PR targets that direction.
