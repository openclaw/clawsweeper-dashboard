## Summary

Keep open. Current main and the latest release still use one GitHub Copilot token cache entry without tying it to the selected auth profile or GitHub token, and the open linked fix PR is the implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue reports that GitHub Copilot auth profile rotation reuses one cached Copilot API token across different GitHub PAT-backed profiles.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/43659 with an owner-aware Copilot token cache fix in the centralized helper, including same-profile cache hit, cross-profile miss, and legacy unowned-cache miss coverage.

## Reproduction Assessment

Yes, source-level: resolveCopilotApiToken returns a usable shared cached token before using the selected GitHub token that the GitHub Copilot provider passes in. I did not run a live multi-profile rate-limit scenario.

## Solution Assessment

Yes. The narrow maintainable fix is to bind the centralized Copilot token cache to the selected GitHub token or profile, treat legacy unowned cache entries as misses, and preserve the existing Copilot integration identity behavior.
