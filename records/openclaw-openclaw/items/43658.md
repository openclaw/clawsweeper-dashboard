## Summary

Keep open: current main and the latest release still return a valid shared Copilot API-token cache entry before considering the rotated GitHub token, and an open linked fix PR already references this issue with closing syntax.

## What This Changes

The issue reports that GitHub Copilot auth-profile rotation reuses one cached Copilot API token instead of exchanging tokens per profile.

## Best Possible Solution

Land a focused fix that binds the cached Copilot API token to the GitHub token or profile in the shared provider-auth helper, with regression coverage for different-profile and legacy-cache cases.

## Reproduction Assessment

Yes. A valid cached Copilot token with the vscode-chat integration identity causes resolveCopilotApiToken to return the cache before using a different params.githubToken, which is a high-confidence source-level reproduction path.

## Solution Assessment

Yes. Keying or validating the Copilot API-token cache by the GitHub token/profile is the narrow maintainable fix; the open linked PR follows that direction but still needs review, conflict resolution, and real behavior proof.
