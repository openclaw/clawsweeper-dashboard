## Summary

Keep open: current main and the latest release still reuse a usable shared Copilot API-token cache entry before checking the selected GitHub credential, and the linked fix PR is still open rather than merged provenance.

## What This Changes

The issue reports that GitHub Copilot auth profile rotation is ineffective because multiple profiles can reuse one shared cached Copilot API token.

## Best Possible Solution

Patch credential-bound cache reuse once in src/plugin-sdk/provider-auth.ts, keep the wrapper files thin, add cross-profile and legacy-cache regression coverage, and land or replace the open linked fix PR with redacted real-environment proof.

## Reproduction Assessment

Yes at source level: the current resolver can return a shared cached Copilot token before using the rotated profile's GitHub token. I did not run a live two-account Copilot quota scenario.

## Solution Assessment

Yes, the root-cause fix should live in the canonical provider-auth resolver because current Copilot token wrapper files re-export that implementation. The open PR's credential-bound cache direction is right, but its implementation needs to be ported away from obsolete wrapper ownership before merge.
