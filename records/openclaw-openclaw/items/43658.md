- `impact:auth-provider`: The issue directly affects GitHub Copilot token selection and auth-profile rotation after rate limits.

## Summary

Keep open: current `main` and `v2026.5.20` still reuse a single unowned GitHub Copilot API-token cache before the rotated profile's GitHub token is used, and an open closing PR is already the active fix path.

## What This Changes

The issue reports that GitHub Copilot auth-profile rotation after rate limits is ineffective because the cached Copilot API token is not bound to the selected GitHub token or profile.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43659 with an owner-bound Copilot token cache fix and regression coverage for same-profile hits, cross-profile misses, and legacy unowned cache entries.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: configure two GitHub Copilot auth profiles, let profile A populate the shared cache, then select profile B while the cache is still usable; current `main` can return A's cached Copilot API token before using B's GitHub token.

## Solution Assessment

Yes. Binding the centralized Copilot token cache to the selected GitHub token or profile is the narrow maintainable fix, but this issue should stay open until the linked implementation merges or is replaced.
