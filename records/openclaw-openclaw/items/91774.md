## Summary

Keep this open: current main still wires `notifyOnExit` only through OpenClaw-managed background exec sessions, while Codex native `commandExecution` completion is projected as Codex turn progress without enqueueing an OpenClaw wake.

## What This Changes

The issue reports that Codex native exec completions do not trigger OpenClaw `notifyOnExit` / `notifyOnExitEmptySuccess` session wake behavior.

## Best Possible Solution

Maintainers should define the Codex native exec notification contract, then either bridge Codex `commandExecution` completions into OpenClaw wake delivery or explicitly scope the docs/config away from Codex native exec.

## Reproduction Assessment

Yes for source-level reproduction: the reporter supplied a provider A/B runtime comparison, and current main still shows no bridge from Codex `commandExecution` completion into OpenClaw exec wake delivery.

## Solution Assessment

Unclear until maintainers choose the contract; the clean solution depends on whether `tools.exec.notifyOnExit` should apply to Codex-owned native exec or only OpenClaw-owned exec/process tools.
