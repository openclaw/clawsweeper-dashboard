## Summary

Keep open. Current main still has the reported authorization-policy gap in the inline skill tool-dispatch path, and the linked implementation PR #75125 is open and unmerged, so this issue should remain tracked until that PR or an equivalent reviewed fix lands.

## What This Changes

The issue reports that user-invocable skill slash commands using `command-dispatch: tool` can execute from the raw OpenClaw tool set instead of the session-effective tool policy.

## Best Possible Solution

Review and land #75125 or an equivalent narrow patch that resolves inline tool-dispatched skill commands through the same effective tool surface used by normal agent turns, with regression coverage for restrictive profiles and leaf subagent sessions.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: define a user-invocable skill with `command-dispatch: tool` targeting a restricted tool such as `sessions_spawn`, invoke it from a restricted or leaf-subagent session, and current main resolves the tool from raw `createOpenClawTools(...)` after only owner-only filtering.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing `createOpenClawCodingTools(...)` effective policy pipeline for inline tool dispatch rather than duplicating raw tool construction.
