## Summary

Keep open. Current main still routes inline `command-dispatch: tool` skill commands through the raw OpenClaw tool factory with only owner-only filtering, while the normal agent-turn path applies profile, provider, group, sandbox, and subagent policy; the open closing PR #75125 should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks for inline skill slash commands using `command-dispatch: tool` to honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Land #75125 or an equivalent narrow hardening patch that resolves tool-dispatched skills through the canonical effective-policy tool builder and adds regression coverage for restricted sessions.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: create a restricted profile or leaf subagent, add a user-invocable `command-dispatch: tool` skill for a denied tool such as `sessions_spawn`, and current main resolves it from `createOpenClawTools(...)`.

## Solution Assessment

Yes. Reusing `createOpenClawCodingTools(...)` is the narrowest maintainable direction because it avoids duplicating the profile, group, sandbox, and subagent policy pipeline in the inline dispatch branch.
