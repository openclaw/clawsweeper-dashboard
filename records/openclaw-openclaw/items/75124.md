## Summary

Keep open. Current main still builds inline `command-dispatch: tool` skill commands from the raw OpenClaw tool set and then only applies owner-only filtering; #75125 remains open and unmerged with closing syntax, so this issue should track the bug until that or an equivalent fix lands.

## What This Changes

The issue reports that `command-dispatch: tool` skill slash commands bypass the session’s effective tool-policy filters.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence static path: configure a restricted profile or leaf subagent, add a `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the inline branch resolves it from raw tools before only owner filtering.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable fix; duplicating the filters in the inline branch would create another drift point.
