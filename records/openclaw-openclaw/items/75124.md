## Summary

Keep open. Current main still builds inline `command-dispatch: tool` skill commands from the raw OpenClaw tool set and applies only owner-only filtering; #75125 is the open closing implementation candidate, so this should remain tracked until that or an equivalent fix lands.

## What This Changes

Skill slash commands using `command-dispatch: tool` should resolve tools through the same effective session policy as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` using the canonical policy session and adds regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure a restricted profile or leaf subagent, add a user-invocable `command-dispatch: tool` skill for a denied tool such as `sessions_spawn`, and the current inline dispatcher resolves it from raw `createOpenClawTools(...)`.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable direction; duplicating profile, group, sandbox, and subagent filters in the inline path would create another drift point.
