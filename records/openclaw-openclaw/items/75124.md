## Summary

Keep open. Current main still resolves `command-dispatch: tool` skill slash commands from the raw OpenClaw tool factory and only applies owner-only filtering, while the normal agent-turn path applies the layered effective tool-policy pipeline; #75125 remains the open, unmerged implementation candidate.

## What This Changes

The issue asks for tool-dispatched skill slash commands to honor the same effective tool policy used by normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with regression coverage for restricted policies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a restricted session or leaf subagent, add a `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the inline branch resolves it from raw tools before only owner filtering.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable fix; duplicating policy filters inside the inline branch would create another drift point.
