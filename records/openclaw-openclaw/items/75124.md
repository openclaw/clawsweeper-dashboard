## Summary

Keep open. Current main still routes `command-dispatch: tool` skill slash commands through the raw OpenClaw tool factory and only owner-only filtering, while the normal coding-tool path applies the effective policy pipeline; the open closing PR #75125 should be reviewed or replaced before this issue closes.

## What This Changes

The issue reports that `command-dispatch: tool` skill slash commands can bypass the effective tool policy applied to normal agent turns.

## Best Possible Solution

Route inline tool-dispatched skill commands through the existing effective tool-policy builder, with focused regression coverage for restricted profiles and leaf subagent sessions.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create a restricted profile or leaf subagent session, add a `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the inline branch resolves it from raw tools before only owner filtering.

## Solution Assessment

Yes. Reusing `createOpenClawCodingTools(...)` for this path is the narrowest maintainable fix because it keeps profile, group/channel, sandbox, and subagent policy in the shared pipeline instead of duplicating filters.
