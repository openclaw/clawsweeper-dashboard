## Summary

Keep open. Current main still routes `command-dispatch: tool` skill slash commands through the raw tool factory and then only owner-only filtering, while the normal agent tool path applies the effective profile/group/sandbox/subagent policy pipeline; #75125 remains the open closing implementation candidate.

## What This Changes

The issue reports that user-invocable skill slash commands using `command-dispatch: tool` can bypass the current session's effective tool policy.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with regression coverage for restricted policies.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: use a restricted session or leaf subagent, add a `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the inline branch resolves it from raw tools before only owner filtering.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable fix; duplicating policy filters inside the inline branch would create another drift point.
