## Summary

Keep open: current main still dispatches `command-dispatch: tool` skill commands through the raw tool builder plus only owner-only filtering, and the linked open PR #75125 is the right place to land the narrow policy fix.

## What This Changes

The issue asks for inline skill slash commands using `command-dispatch: tool` to honor the same effective tool policy as normal session turns.

## Best Possible Solution

Land a fix equivalent to #75125 that builds inline tool-dispatch commands through `createOpenClawCodingTools` with the canonical runtime policy session key and target-session metadata, plus regression coverage for a restricted session.

## Reproduction Assessment

Yes, source-reproducible. A restricted or leaf-subagent session invoking a user-invocable skill mapped to `sessions_spawn` enters `handleInlineActions`, which currently builds raw tools through `createOpenClawTools` and only owner-filters them.

## Solution Assessment

Yes. Routing this branch through the existing `createOpenClawCodingTools` pipeline is the narrow maintainable fix because that is already the central policy path for profile, group, sandbox, sender, model, and subagent filtering.
