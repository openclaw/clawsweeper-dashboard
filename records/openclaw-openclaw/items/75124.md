## Summary

Keep open: current main still routes `command-dispatch: tool` inline skill commands through the raw OpenClaw tool builder and only owner-only filtering, while the normal command path uses the effective tool-policy pipeline. The active fix path is the open maintainer-labeled PR at https://github.com/openclaw/openclaw/pull/78525; the two earlier linked PR attempts are closed.

## What This Changes

The issue asks for `command-dispatch: tool` skill slash commands to honor the same effective tool policy used by normal agent turns.

## Best Possible Solution

Land or replace the active focused fix so inline tool dispatch resolves the same effective policy surface as normal agent turns, with regression coverage for allow/deny, sandbox, group, and subagent policy layers.

## Reproduction Assessment

Yes from source inspection: current `handleInlineActions` calls raw `createOpenClawTools(...)` and only owner-only filtering, while the effective policy pipeline lives in `createOpenClawCodingTools(...)`. ## Solution Assessment
