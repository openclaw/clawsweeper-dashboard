## Summary

Keep open: current `main` still routes inline `command-dispatch: tool` skill commands through raw tool construction plus owner-only filtering, while the normal tool path uses the effective policy pipeline. The issue is also paired with open closing PR #75125, so it should remain open until that fix lands or is replaced.

## What This Changes

Require skill slash commands using `command-dispatch: tool` to honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Land a focused fix that makes the inline tool-dispatch branch resolve tools through `createOpenClawCodingTools` with the same policy session, group, sandbox, subagent, sender, provider, and model context used by normal command handling.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure a restricted effective tool surface, add a `command-dispatch: tool` skill for a denied tool, and the current inline branch will resolve from raw `createOpenClawTools` plus owner-only filtering. I did not execute it live because this was a read-only review.

## Solution Assessment

Yes, the best fix direction is to reuse the existing effective tool-policy builder rather than adding another partial filter. The already-linked open PR is the right vehicle to review unless it fails to pass the same context and regression coverage.
