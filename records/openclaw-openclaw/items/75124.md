## Summary

Keep open: current main still has the reported policy gap in the inline skill tool-dispatch path, and overlapping open PRs already target the fix. This should not be closed until one canonical implementation lands.

## What This Changes

The issue reports that `command-dispatch: tool` skill slash commands can bypass the effective tool-policy pipeline by constructing raw tools and applying only owner-only filtering.

## Best Possible Solution

Land or replace one canonical policy-aware inline dispatch fix so skill tool commands reuse the effective tool surface, with regression coverage for allow/deny, group, sandbox, and subagent policy layers.

## Reproduction Assessment

Yes. Source inspection on current main shows inline skill tool dispatch constructs raw tools and only applies owner-only filtering; a restrictive `tools.deny` or leaf-subagent command-dispatch skill would exercise the failing path.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve inline skill dispatch through a shared effective-tool helper or the normal policy-aware tool pipeline, not to add another ad hoc filter after raw tool construction.
