## What This Changes

This issue asks for `command-dispatch: tool` skill slash commands to honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Land a focused fix that makes inline tool dispatch reuse the effective `createOpenClawCodingTools(...)` pipeline with the canonical session, group, sandbox, subagent, sender, and model context.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: invoke a user-invocable skill command with `command-dispatch: tool` in a restricted session and the current branch builds from raw `createOpenClawTools(...)` plus owner-only filtering instead of the effective policy builder. I did not run a live runtime scenario in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to route the inline dispatch branch through `createOpenClawCodingTools(...)` and pass the same policy context used by normal command turns. The open linked PR’s stated approach matches that direction, so closing should wait for merge or a replacement.
