## Summary

Keep open. Current main still routes inline `command-dispatch: tool` skill slash commands through raw `createOpenClawTools(...)` plus owner-only filtering, while normal agent turns use the layered effective tool-policy pipeline; #75125 is an open, unmerged closing PR, so this issue should remain open until that or an equivalent fix lands.

## What This Changes

Skill slash commands that use `command-dispatch: tool` should honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool dispatch through `createOpenClawCodingTools(...)` with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in a restricted session or leaf subagent, a user-invocable skill targeting a denied tool such as `sessions_spawn` is resolved from raw tools before only owner filtering; I did not run a live reproduction in this read-only review.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable fix; duplicating the policy filters in the inline branch would create another drift point.
