## Summary

Keep open. Current main still builds inline `command-dispatch: tool` skill commands from the raw OpenClaw tool set and then only applies owner-only filtering; #75125 is the open closing implementation candidate, so this issue should remain tracked until that or an equivalent fix lands.

## What This Changes

The issue reports that skill slash commands using `command-dispatch: tool` can resolve tools outside the session’s effective tool-policy pipeline.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through the existing effective tool-policy builder with regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure a restricted profile or leaf subagent, add a user-invocable `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and current main resolves it from raw `createOpenClawTools(...)` before only owner filtering.

## Solution Assessment

Yes. Reusing `createOpenClawCodingTools(...)` with the canonical target-session context is the narrowest maintainable fix; duplicating the policy filters in the inline branch would create another drift point.
