## Summary

Keep open. Current main still builds inline tool-dispatched skill commands from the raw OpenClaw tool set and applies only owner-only filtering, and #75125 is an open closing implementation candidate that should remain linked until it or an equivalent fix lands.

## What This Changes

The issue asks for inline skill `command-dispatch: tool` slash commands to honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Review and land #75125 after any required polish, or replace it with a narrow patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure a restricted profile or leaf subagent, add a user-invocable `command-dispatch: tool` skill for a denied tool such as `sessions_spawn`, and current main resolves it from raw `createOpenClawTools(...)`.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable fix; duplicating profile, group, sandbox, and subagent filters in the inline path would create another drift point.
