## What This Changes

The issue asks for user-invocable `command-dispatch: tool` skill slash commands to honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Land or replace the linked fix so inline tool dispatch builds through `createOpenClawCodingTools` with canonical runtime-policy session, target-session group/subagent metadata, sandbox context, sender/model context, and regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a user-invocable `command-dispatch: tool` skill reaches the inline branch that builds raw tools and only owner-only filters them. I did not execute the runtime slash-command path in this read-only review.

## Solution Assessment
