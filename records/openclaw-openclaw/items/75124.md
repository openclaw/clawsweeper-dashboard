## Summary

Keep open. Current main still builds inline `command-dispatch: tool` skill commands from the raw OpenClaw tool set and applies only owner-only filtering; #75125 is the open closing implementation candidate, so this should remain tracked until that or an equivalent fix lands.

## What This Changes

Ensure `command-dispatch: tool` skill slash commands use the same effective tool-policy pipeline as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` using the canonical policy session and adds regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure a restricted profile or leaf subagent, add a user-invocable `command-dispatch: tool` skill for a denied tool such as `sessions_spawn`, and current main resolves it from raw `createOpenClawTools(...)`.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable direction; duplicating profile, group, sandbox, and subagent filters in the inline branch would create another drift point.
