## Summary

Keep open. Current main still builds inline `command-dispatch: tool` skill commands from raw OpenClaw tools and only applies owner-only filtering, while #75125 remains open and unmerged as the closing implementation candidate.

## What This Changes

This issue requests that user-invocable skill slash commands with `command-dispatch: tool` honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure a restrictive profile or leaf subagent, add a `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the inline branch resolves it from raw tools before only owner filtering.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable fix; duplicating the policy filters in the inline branch would create another drift point.
