## Summary

Keep open: current main still builds raw tools for inline skill tool dispatch and only applies owner-only filtering, and the open linked PR is the implementation path for this issue.

## What This Changes

Ensure user-invocable skill slash commands with `command-dispatch: tool` honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Review and land, or replace, #75125 so inline tool-dispatch skills use `createOpenClawCodingTools(...)` with canonical runtime-policy context and focused regression coverage for restricted profiles and leaf subagents.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence reproduction path: define a `command-dispatch: tool` skill for a tool filtered out by `createOpenClawCodingTools(...)`, then invoke it through the inline slash-command branch that currently builds raw tools.

## Solution Assessment

Yes. Routing this branch through the same effective coding-tool builder used by normal command turns is the narrowest maintainable fix; the open linked PR is the right implementation path to review rather than closing this issue early.
