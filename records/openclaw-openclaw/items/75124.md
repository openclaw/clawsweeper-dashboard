## Summary

Keep open: current main still routes inline `command-dispatch: tool` skill commands through the raw tool builder plus owner-only filtering, so the reported policy bypass remains valid. An open linked PR already uses closing syntax for this issue, so this should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

Route user-invocable skill slash commands using `command-dispatch: tool` through the same effective tool-policy pipeline as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow patch that makes inline tool dispatch use `createOpenClawCodingTools(...)` with canonical runtime policy context plus regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main's inline skill tool dispatch builds raw tools and only owner-only-filters them, while the normal session path uses `createOpenClawCodingTools(...)` with profile, group, sandbox, and subagent policy. I did not run a live slash-command session in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse the existing effective coding-tool builder for inline tool dispatch instead of duplicating raw tool setup. Because #75125 is still open and explicitly tied to this issue, the issue should remain open until that implementation lands or is replaced.
