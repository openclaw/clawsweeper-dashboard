## Summary

Keep open. Current main still has the reported bypass: inline `command-dispatch: tool` skill commands build raw tools and only apply owner-only filtering, while the documented/runtime effective tool surface uses `createOpenClawCodingTools`; https://github.com/openclaw/openclaw/pull/75125 is an open closing PR and should be reviewed before this issue is closed.

## What This Changes

Route user-invocable skill slash commands with `command-dispatch: tool` through the same effective tool-policy pipeline as normal agent turns.

## Best Possible Solution

Merge an equivalent of #75125 that makes inline tool-dispatch use the effective coding-tool builder and adds regression coverage for restricted profiles, group policy, sandbox policy, and subagent sessions.

## Reproduction Assessment

Yes. Source inspection on current main shows a direct reproduction path: invoke a user-invocable `command-dispatch: tool` skill in a restricted session and the inline branch builds raw tools plus owner-only filtering instead of the effective policy pipeline.

## Solution Assessment

Yes. The narrow maintainable fix is to route this inline branch through `createOpenClawCodingTools` with the same canonical session, group, sender, and model context used by normal commands; the linked open PR appears to target that direction.
