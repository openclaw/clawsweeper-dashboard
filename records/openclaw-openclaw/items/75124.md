## Summary

Keep open. Current main still routes `command-dispatch: tool` skill slash commands through the raw OpenClaw tool factory and only applies owner-only filtering, and the open closing PR #75125 should be reviewed or replaced before this is resolved.

## What This Changes

Report that user-invocable skills with `command-dispatch: tool` can bypass the current session's effective tool policy.

## Best Possible Solution

Merge a narrow patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with the canonical policy session and regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: create a restricted profile or leaf subagent, add a `command-dispatch: tool` skill for a denied tool such as `sessions_spawn`, and current main resolves it from the raw factory.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable solution; duplicating profile, group, sandbox, and subagent filters in the inline branch would create another drift point.
