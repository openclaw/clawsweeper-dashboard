## Summary

Keep open. Current main still builds `command-dispatch: tool` skill slash commands from the raw OpenClaw tool factory and then applies only owner-only filtering, while the normal coding-tool path applies the layered effective policy pipeline; #75125 is still the open closing implementation candidate and should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests that inline skill slash commands using `command-dispatch: tool` honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with regression coverage for restricted policies.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create a restricted session or leaf subagent, add a `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the inline branch resolves it from raw tools before only owner filtering.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable fix; duplicating policy filters inside the inline branch would create another drift point.
