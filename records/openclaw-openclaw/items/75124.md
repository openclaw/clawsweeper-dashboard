## Summary

Keep open. Current main still routes `command-dispatch: tool` skill slash commands through the raw OpenClaw tool factory and only applies owner-only filtering, while #75125 is open and unmerged as the linked fix candidate.

## What This Changes

The issue reports that tool-dispatched skill slash commands bypass the effective runtime tool-policy pipeline.

## Best Possible Solution

Review and land #75125, or replace it with an equivalent narrow patch that builds inline tool-dispatch commands from the same effective tool surface as normal agent turns.

## Reproduction Assessment

Yes. Source inspection on current main shows that a restricted profile or leaf subagent with a user-invocable `command-dispatch: tool` skill targeting `sessions_spawn` resolves the raw tool and only owner-filters it.

## Solution Assessment

Yes. Reusing `createOpenClawCodingTools(...)` with the canonical session, group, sandbox, and subagent context is the narrowest maintainable repair; duplicating filters in the inline branch would create another drift point.
