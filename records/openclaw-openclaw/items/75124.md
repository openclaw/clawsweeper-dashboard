## Summary

Keep open. Current main still resolves `command-dispatch: tool` skill slash commands from the raw OpenClaw tool set and then applies only owner-only filtering; the open #75125 PR is the closing implementation candidate, so this issue should remain tracked until that or an equivalent fix lands.

## What This Changes

Require `command-dispatch: tool` skill slash commands to resolve tools through the same effective tool-policy pipeline as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with the canonical policy session and regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: use a restricted profile or leaf subagent, add a user-invocable `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and current main resolves it from raw `createOpenClawTools(...)`.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable direction; duplicating profile, group, sandbox, and subagent filters in the inline branch would create another drift point.
