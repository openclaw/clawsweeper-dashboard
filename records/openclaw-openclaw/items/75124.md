## Summary

Keep open. Current main still resolves `command-dispatch: tool` skill slash commands from the raw OpenClaw tool factory with only owner-only filtering, while #75125 is the open closing implementation candidate and should be reviewed or replaced before this issue closes.

## What This Changes

Ensure `command-dispatch: tool` skill slash commands use the same effective tool-policy pipeline as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent that routes inline tool-dispatched skills through `createOpenClawCodingTools(...)` with regression coverage for restricted profiles and leaf subagents.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure a restricted profile or leaf subagent, add a user-invocable `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the current inline dispatcher resolves it from the raw tool factory.

## Solution Assessment

Yes. Reusing the existing effective tool-policy builder is the narrowest maintainable direction; duplicating profile, group, sandbox, and subagent filters in the inline branch would create another policy drift point.
