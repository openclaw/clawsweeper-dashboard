## Summary

Keep open. Current main still resolves inline `command-dispatch: tool` skill commands through the raw tool factory and then applies only owner-only filtering, while normal agent turns use the layered effective tool-policy pipeline; #75125 remains the open closing implementation candidate.

## What This Changes

Skill slash commands using `command-dispatch: tool` should honor the same effective tool policies as normal agent turns.

## Best Possible Solution

Review and land #75125, or replace it with a narrow equivalent patch that routes inline tool-dispatched skill commands through `createOpenClawCodingTools(...)` with regression coverage for restricted profile, group, sandbox, and subagent policy.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create a restricted session or leaf subagent, add a `command-dispatch: tool` skill targeting a denied tool such as `sessions_spawn`, and the inline branch resolves it from raw tools before only owner filtering.

## Solution Assessment

Yes for the linked fix direction. Reusing the existing effective tool-policy builder is the narrowest maintainable solution; duplicating policy filters inside the inline branch would create another drift point.
