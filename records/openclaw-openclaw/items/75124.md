## Summary

Keep open: current main still builds raw OpenClaw tools for inline skill tool dispatch, and an open linked PR is already the implementation path for this issue.

## What This Changes

The issue reports that `command-dispatch: tool` skill slash commands should honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Route inline tool-dispatch skill commands through `createOpenClawCodingTools(...)` with the same canonical session, group, sender, sandbox, subagent, provider, and model context as the normal command path, then add focused regression coverage.

## Reproduction Assessment

Yes from source inspection. A user-invocable skill with `command-dispatch: tool` still enters `handleInlineActions`, builds raw OpenClaw tools, and applies only owner-only filtering on current main.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to reuse the existing effective tool-policy builder instead of duplicating filtering in the inline branch, and the open linked PR is the right place to review that implementation.
