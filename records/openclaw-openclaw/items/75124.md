## Summary

Keep open: current main still resolves `command-dispatch: tool` inline skill commands through the raw tool factory plus owner-only filtering, while the normal command path uses the effective coding-tool policy pipeline. An open fix PR already explicitly closes this issue, so this should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue asks for skill slash commands using `command-dispatch: tool` to honor the same effective tool policy as normal agent turns.

## Best Possible Solution

Land or replace the focused fix that routes inline skill tool dispatch through `createOpenClawCodingTools(...)` with regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure a restricted or leaf-subagent session, add a user-invocable `command-dispatch: tool` skill for a denied tool such as `sessions_spawn`, and invoke it. Current main resolves that command through raw `createOpenClawTools(...)` plus owner-only filtering instead of the effective policy builder.

## Solution Assessment

Yes, the narrow maintainable solution is to reuse the existing `createOpenClawCodingTools(...)` policy path with the correct runtime policy session, group, sender, model, and subagent context, plus a focused regression test. Do not queue a duplicate repair while #75125 remains open as the implementation candidate.
