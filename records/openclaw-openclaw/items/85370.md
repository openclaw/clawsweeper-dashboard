Request to add a gateway-level `agent:turn` or `agent:loop` event that broadcasts after each agent-loop turn so consumers can persist progress in real time.

## Best Possible Solution

Define an additive, scope-gated turn-boundary contract only if maintainers agree on cross-runtime semantics; otherwise document the existing lifecycle, assistant, tool, compaction, transcript, and internal steering surfaces as the supported persistence boundaries.
