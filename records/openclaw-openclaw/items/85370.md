## What This Changes

The issue requests a new OpenClaw gateway `agent:turn` or `agent:loop` event that broadcasts after each main-loop iteration so consumers can persist agent progress in near real time.

## Best Possible Solution

Define an additive, scope-gated turn-boundary contract only if maintainers agree on cross-harness semantics; otherwise document the existing lifecycle/assistant/tool streams as the supported persistence surface.
