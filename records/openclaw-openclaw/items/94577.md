## Summary

Keep this issue open because current main and the latest release still use the fixed 512-byte recent-session header scan, while https://github.com/openclaw/openclaw/pull/94578 is the open linked fix candidate.

## What This Changes

The issue reports that recent-session resume can ignore a valid JSONL transcript when the session header line is longer than the fixed scan window.

## Best Possible Solution

Review and land the linked focused PR, or an equivalent bounded first-line scanner fix with regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be loadable by loadEntriesFromFile but rejected by findMostRecentSession before continueRecent decides whether to resume; I did not run a live test because this review is read-only.

## Solution Assessment

Yes in shape. The narrow maintainable fix belongs in the recent-session header validation helper with a bounded first-line reader and regression coverage, rather than loading whole transcripts during directory scans.
