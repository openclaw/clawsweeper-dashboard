## Summary

Keep this issue open because current main and the latest release still use the fixed-size recent-session header scan, while the same-author linked open PR remains the active fix candidate.

## What This Changes

The issue reports that recent-session resume can miss a valid JSONL transcript when the first session header line exceeds the fixed 512-byte scan window.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94578, or an equivalent bounded first-line scanner fix with regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be loadable by `loadEntriesFromFile()` but rejected by `findMostRecentSession()` before `continueRecent()` decides whether to resume.

## Solution Assessment

Yes in shape. The narrow maintainable fix belongs in the recent-session header validation helper with a bounded first-line reader and regression coverage, rather than loading whole transcripts during directory scans.
