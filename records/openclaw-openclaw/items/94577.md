## Summary

Keep this issue open: current main still has the fixed-size 512-byte recent-session header scan, and the linked open PR is the active fix candidate rather than a reason to close before merge.

## What This Changes

The issue reports that recent-session resume can skip a valid JSONL transcript when the session header line is longer than the scanner's fixed first-read size.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94578, or an equivalent bounded first-line scanner fix with regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be loadable by `loadEntriesFromFile()` but rejected by `findMostRecentSession()` before `continueRecent()` decides whether to resume.

## Solution Assessment

Yes in shape. The narrow fix belongs in the recent-session validation helper, with regression coverage proving long-but-valid headers are discovered and oversized headers remain bounded.
