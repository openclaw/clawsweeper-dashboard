## Summary

Keep this issue open: current main and the latest release still use the 512-byte recent-session header scan, and the linked open pull request is the active fix candidate rather than a reason to close before merge.

## What This Changes

The issue reports that `continueRecent()` can start a new session when a valid JSONL transcript has a first header line longer than the recent-session scanner's read window.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94578, or an equivalent bounded first-line scanner fix with regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes at source level. Current main reads only 512 bytes when validating recent-session candidates, while the normal loader reads the full JSONL transcript before `continueRecent()` decides whether to resume.

## Solution Assessment

Yes in shape. The narrow maintainable fix belongs in recent-session header validation with a bounded first-line reader and regression coverage, rather than loading every transcript fully during directory scans.
