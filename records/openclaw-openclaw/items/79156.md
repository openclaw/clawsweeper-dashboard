The PR adds Discord `voice.mode`/`voice.realtime` configuration, realtime PCM bridging for talk-buffer and bidi modes, shared consult-policy helpers, docs/changelog updates, and focused tests.

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reuse the same speaker context for bidi consults:** `extensions/discord/src/voice/realtime.ts:277`
  - body: `handleToolCall` consumes a new pending speaker turn for every `openclaw_agent_consult` call. A realtime model can issue multiple consult calls during one utterance; if another speaker has already queued audio, the later call can run with that speaker's `senderIsOwner` and label, so bind consult calls to the originating turn instead of popping a fresh context each time.
  - confidence: 0.88
- **[P1] Subscribe before async ingress checks:** `extensions/discord/src/voice/manager.ts:562-564`
  - body: `handleSpeakingStart` awaits guild/member authorization before creating the receive stream. In `@discordjs/voice@0.19.2`, packets are dropped when no subscription exists, so normal lookup latency can clip authorized realtime turns and delay barge-in; subscribe first, then destroy or ignore the stream if authorization fails.
  - confidence: 0.91
- **[P2] Reset decrypt recovery after realtime audio:** `extensions/discord/src/voice/manager.ts:595-602`
  - body: The realtime branch returns after `processRealtimeAudioCapture`, so successful realtime ingestion never clears the DAVE decrypt-failure counter like the STT path does. Intermittent decrypt errors can accumulate across successful realtime turns and trigger unnecessary rejoin recovery.
  - confidence: 0.87
- **[P2] Close the realtime bridge on every teardown path:** `extensions/discord/src/voice/manager.ts:367`
  - body: `entry.realtime?.close()` only runs from `stop()`, while disconnect-recovery failure and destroyed handlers clear or destroy the session without calling it. Realtime modes can leave the provider bridge/socket active after Discord voice is gone, so route teardown through shared cleanup.
  - confidence: 0.84
