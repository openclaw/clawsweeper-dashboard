## What This Changes

The branch gates the Control UI chat In progress badge and Stop button on local send/stream state and adds render tests for stale abortability.

## Best Possible Solution

Split stale session-list abortability from an authoritative local active-run signal such as `chatRunId`, then prove both stale-idle and active streamless tool-run UI states in a real Control UI setup.

## Reproduction Assessment

Yes, at source level. Current main can have `chatRunId` active while `chatStream` is null, and this PR's new gate treats that state as inactive for visible Stop controls.

## Solution Assessment

No. The stale-session direction is right, but `sending || stream !== null` is not the best boundary because it drops legitimate local active-run states that should remain abortable.
