## What This Changes

The PR gates Control UI chat's In progress badge and Stop button behind local send/stream busy state and adds regression tests for stale session-list abortability.

## Best Possible Solution

Keep the stale session-list-only fix, but make the visible abort guard distinguish local active runs, including null-stream tool phases, from stale session rows and then add real Control UI proof.

## Reproduction Assessment

Yes for source-level review: current main can render stale session-list abortability, and the PR's new predicate can be checked against existing active null-stream run states. I did not run a live browser/gateway repro in this read-only review.

## Solution Assessment

No. Guarding only on `sending || stream !== null` is too narrow because local active runs can keep `chatRunId` set while `chatStream` is null; the best fix separates stale session-row abortability from local active-run abortability.
