## Summary

Keep open: the PR targets a real Control UI bug, but the current guard can hide the Stop control for active streamless tool runs, and the external PR still lacks real Control UI browser/gateway proof.

## What This Changes

The branch gates the Control UI chat In progress badge and Stop button on local `sending || stream !== null` state and adds render tests for stale abortability.

## Best Possible Solution

Pass an explicit authoritative local active-run signal into `renderChat` so stale session rows do not drive the main composer while current runs remain abortable, then prove that in focused tests and real Control UI browser/gateway proof.

## Reproduction Assessment

Yes at source level: current main can keep `chatRunId` set while `chatStream` is null, and the PR's gate treats that state as not active. I did not run a live browser/gateway reproduction in this read-only review.

## Solution Assessment

No: `sending || stream !== null` is a plausible stale-completion heuristic, but it is not the best fix because it drops legitimate local active-run states. The safer fix distinguishes current `chatRunId` from session-list-only stale abortability.
