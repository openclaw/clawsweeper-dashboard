## Summary

Keep open. Current main still rejects or bypasses steering when the active run is not streaming, and the focused implementation work is still open and unmerged.

## What This Changes

The issue reports that `messages.queue.mode: "steer"` fails to inject same-session messages into an active main-session turn during tool execution.

## Best Possible Solution

Land or replace one focused lifecycle-aware steering fix that lets active non-streaming runs accept steer messages at runtime boundaries, with regression coverage for embedded and reply-run paths.

## Reproduction Assessment

Yes, by source inspection. Configure `messages.queue.mode: "steer"`, have a same-session run active while `isStreaming` is false during tool execution, then send another inbound message; current main still rejects steering or queues it as a followup.

## Solution Assessment

Yes. Keeping this issue open behind the focused PR cluster is the right path because current main has not fixed the documented regression and a separate cleanup close would be premature.
