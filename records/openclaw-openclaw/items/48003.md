## Summary

Keep open. Current main still gates steer injection on `isStreaming` and same-key channel queues still serialize inbound work, while focused fix PRs remain open and unmerged.

## What This Changes

The issue asks OpenClaw to inject steer and steer-backlog messages into active main-session turns during tool execution instead of processing them later as followups.

## Best Possible Solution

Land or replace the focused lifecycle-aware steering fix, with regression coverage for active non-streaming tool execution and any channel queue-bypass behavior needed for messages to reach the active run.

## Reproduction Assessment

Yes, by source inspection. Configure `messages.queue.mode: "steer"`, start a same-session run that is active during tool execution but not streaming, then send another inbound message; current main still rejects steering on the non-streaming guards or waits behind same-key channel serialization.

## Solution Assessment

Yes. Keeping this open behind the focused PR cluster is the best path because current main has not fixed the documented regression and a separate repair would duplicate active implementation review.
