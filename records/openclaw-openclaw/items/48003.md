## What This Changes

The issue requests restoring steer-mode active-run message injection so same-session messages sent during active tool phases reach the running turn at runtime boundaries instead of becoming later follow-ups.

## Best Possible Solution

Refresh or replace the coordinated lifecycle-aware steering and same-session admission fixes so active-run messages reach queue policy and are accepted through the full agent-loop lifecycle while preserving backpressure, compaction, and session-state safeguards.

Yes on fix shape, but not yet in current code: lifecycle-aware active/stopped steering plus a same-session admission path is the right direction, while the open PRs still need rebase/repair and maintainer review.
