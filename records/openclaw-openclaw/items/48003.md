## Summary

Keep open. Current main still has the streaming-only steer gates and same-key inbound serialization described in the report, while focused fix PRs are open but unmerged.

## What This Changes

The issue reports that `messages.queue.mode: "steer"` queues same-session messages until the active run ends instead of injecting them into an active main-session turn at tool boundaries.

## Best Possible Solution

Land or replace the focused lifecycle-aware steering fix, keep deliberate followup fallback and diagnostics, then close this issue after current main contains the corrected behavior and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure `messages.queue.mode` to `steer`, start a same-session tool-running turn, and send another inbound message while the run is active but not streaming; current main still serializes same-key inbound work and rejects non-streaming steering.

## Solution Assessment

Yes. Keeping this issue open is the best tracking outcome because the fix is not on main and open implementation PRs already cover the affected steering files; a separate repair job would duplicate active maintainer review.
