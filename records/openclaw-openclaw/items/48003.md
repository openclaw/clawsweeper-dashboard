## Summary

Keep open. Current main and v2026.6.10 still have source-level steer admission gaps, and the fresh linked work is open and explicitly partial rather than a complete implementation.

## What This Changes

Bug report that `messages.queue.mode: "steer"` and explicit steering can leave same-session messages queued until the active run finishes instead of injecting them at supported runtime boundaries.

## Best Possible Solution

Keep this issue as the canonical active-run steering tracker until maintainers land or replace the coordinated lifecycle-aware steering, same-session admission, and safe post-context ingress fixes.

## Reproduction Assessment

Yes, at source level: with queue mode `steer`, a same-session message can be held by ingress/admission or rejected by active `isStreaming` gates before it reaches a non-streaming active run. I did not run a live Discord, Telegram, or WebChat replay in this read-only review.

## Solution Assessment

Yes on the intended fix shape, but it is not fully on main yet. The maintainable path is lifecycle-aware active/not-stopped admission plus safe post-context same-session ingress, not a blind pre-context bypass that could misroute chats or topics.
