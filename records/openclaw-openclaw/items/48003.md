## Summary

Keep open. Current main still contains the reported streaming-only steering gates and Discord same-session keyed queue path, while open PR #73365 remains the active implementation lane for maintainer review.

## What This Changes

Make `messages.queue.mode: "steer"` inject same-session Discord/main-session messages into an active agent turn at tool boundaries instead of queueing them until the turn ends.

## Best Possible Solution

Land a narrow fix that lets only steer-mode messages targeting an active run reach the steering queue during active non-streaming tool execution while preserving normal per-session serialization, with focused tests for Pi/reply acceptance and Discord delivery.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with `messages.queue.mode: "steer"`, send a Discord message during an active main-session tool execution window; current main either holds it behind the same session queue or rejects steering through streaming-only guards.

## Solution Assessment

Unclear for the open implementation path. PR #73365 is the right lifecycle direction, but maintainers should verify or extend it so Discord steer-mode inbound messages can reach the steering path before waiting behind the active keyed run.
