This is a valid current-main QQBot bug: the logger still uses raw string truthiness for `QQBOT_DEBUG`, so false-like values enable debug output. It should stay open and is a narrow repair candidate because the affected logger and focused test surface are clear.

The issue asks for QQBot debug logging to enable only on explicit truthy `QQBOT_DEBUG` values instead of any non-empty environment value.

Match the QQBot logger to OpenClaw's explicit truthy environment semantics and add regression coverage for true-like and false-like `QQBOT_DEBUG` values.
