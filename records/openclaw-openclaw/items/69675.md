## Summary

Keep open. The PR documents parent-assistant rewriting as the default for bound ACP follow-ups, but current main still documents, implements, and tests direct routing of bound follow-up text to the ACP session; #69669 remains the open runtime-policy thread for that behavior.

## What This Changes

This PR adds an ACP follow-up orchestration doc and changes the ACP spawn session accepted note to say bound follow-ups go through the parent assistant unless exact relay is requested.

## Best Possible Solution

Make the ACP contract consistent across runtime, tests, and docs: either implement parent-orchestrated rewriting for bound follow-ups with focused dispatch coverage, or keep the docs and accepted note aligned with current direct routing while #69669 tracks the policy change.

## Reproduction Assessment

Yes. Current main has a focused unit test where a bound Slack ACP follow-up with BodyForAgent "continue" reaches runtime.runTurn as text "continue", and the dispatch source shows the same direct path.

## Solution Assessment

No, not as a docs-only change. The narrow maintainable fix is to resolve the ACP behavior decision first, then update runtime behavior, tests, docs, and accepted-note wording together, or explicitly document current direct routing.
