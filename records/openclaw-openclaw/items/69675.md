## Summary

Keep this PR open for ACP maintainer review. Current main still routes bound ACP follow-up text directly to the ACP runtime and has tests asserting raw follow-up delivery, while this PR documents parent-assistant rewriting as the default before the linked runtime/product issue is resolved.

## What This Changes

This PR adds an ACP follow-up orchestration doc and changes the ACP session accepted note to say thread-bound follow-ups go through the parent assistant unless exact relay is requested.

## Best Possible Solution

Make the documented contract and runtime contract match: either keep docs aligned with direct bound ACP routing for now, or land a runtime rewrite with tests that makes parent-orchestrated follow-up rewriting the implemented default while #69669 tracks the product decision.

## Reproduction Assessment

Yes. Current main has a focused unit test where a bound Slack ACP follow-up with `BodyForAgent: "continue"` reaches `runtime.runTurn` as `text: "continue"`, and the dispatch source shows the same direct path.

## Solution Assessment

No, not as a docs-only change. The narrow maintainable solution is to resolve the ACP behavior decision first, then update docs, accepted-note wording, runtime behavior, and tests together or explicitly label the new wording as future policy.
