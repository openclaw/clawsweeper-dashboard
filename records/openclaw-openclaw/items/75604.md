## Summary

Keep open for normal PR handling. Current main still lacks the bundled-version guard for recorded ClawHub/marketplace installs, and this PR is a focused implementation for the linked bug; I did not find a blocking functional or security issue in the diff. The protected `maintainer` label also means this should not be auto-closed by cleanup.

## What This Changes

The PR adds bundled plugin version metadata, skips ClawHub/marketplace plugin updates when the same-id bundled plugin is newer than the recorded installed version, adds focused update tests, and records the fix in the changelog.

## Best Possible Solution

Land a generic, source-agnostic bundled freshness guard for same-id external plugin records, preserving explicit manual install/pin paths while preventing unattended `openclaw update` from replacing a newer bundled plugin with an older registry artifact.

## Reproduction Assessment

Yes. A focused unit reproduction can seed `plugins.installs.whatsapp` as a ClawHub record at `2026.2.09`, expose a bundled WhatsApp source at `2026.4.25`, and show current main reaches `installPluginFromClawHub` while the PR skips with a warning.

## Solution Assessment

Yes. For the reported ClawHub/marketplace path, the PR is a narrow maintainable fix that reuses existing bundled discovery and semver comparison helpers; a more permissive future variant could compare remote candidate versions first, but that is not required for the safe default described here.
