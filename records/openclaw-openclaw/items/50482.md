## Repair Work Prompt

Implement a narrow additive fix for https://github.com/openclaw/openclaw/issues/50482. Add a generic inbound message-origin/source-modality field to the shared channel inbound context so agents can tell when a Telegram message originated as a voice note before it was normalized to generic audio/transcript text. Populate it from `msg.voice` in the Telegram context path before `resolveTelegramPrimaryMedia` collapses voice to `<media:audio>`, project the field through `MsgContext` and `buildInboundUserContextPrefix` in the appropriate per-turn metadata block, and add focused tests that distinguish voice notes from regular audio/text without exposing local paths, provider endpoints, transcript hashes, or broad STT telemetry. Keep the change additive, generic for future channels, and do not edit `CHANGELOG.md`. Related refs: https://github.com/openclaw/openclaw/issues/50482, https://github.com/openclaw/openclaw/issues/40838, https://github.com/openclaw/openclaw/issues/92500, https://github.com/openclaw/openclaw/pull/92502.
