## What This Changes

The issue requests an agent-visible inbound metadata marker that distinguishes voice messages from typed text or other media.

## Best Possible Solution

Add a backward-compatible generic inbound message-origin or modality field, populate it from Telegram before `msg.voice` is normalized to audio, render it in dynamic agent metadata, and cover voice-note versus ordinary audio/text behavior in tests and docs.
