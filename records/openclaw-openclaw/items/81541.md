## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve config updates for custom ACP adapters:** `extensions/acpx/src/runtime.ts:980-981`
  - body: This fallback now forwards options only for the OpenClaw bridge, so any configured ACP adapter that is not recognized as Codex or Claude stops receiving non-timeout config updates such as `model` or `mode`. The previous behavior forwarded those controls, and the PR only proves that one unknown adapter rejects `timeout`; keep forwarding non-timeout options or add a capability-specific filter so custom adapters do not lose supported runtime configuration.
  - confidence: 0.87
- **[P2] Avoid making Codex auth depend on symlinks:** `extensions/acpx/src/codex-auth-bridge.ts:373`
  - body: `fs.symlink` is now on the required Codex ACP startup path whenever source `auth.json` exists. On Windows or restricted filesystems this can fail with `EPERM`/`EACCES`, turning the auth bridge into a startup failure instead of preserving auth; add a fallback such as a best-effort copy/hardlink or keep the existing behavior when symlink creation is unavailable.
  - confidence: 0.84
- **[P2] Prefer fresh disk ACP metadata over stale snapshots:** `src/config/sessions/store.ts:448-449`
  - body: This merge only imports disk ACP metadata when the writer's cached snapshot has no metadata for the key. If the stale writer has an older `entry.acp` and disk has a newer runtime/session block, the old cached metadata still wins and can make the next ACP turn resume the wrong runtime state; make the fresh disk snapshot authoritative for normal writes while preserving the explicit drop escape hatch.
  - confidence: 0.81
