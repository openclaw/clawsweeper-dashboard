## Summary

Keep open. The PR targets plausible ACPX runtime failures, but it is not merge-ready because CI shows a session-store cache regression, unsupported config options can become silent no-ops for adapters without advertised keys, and the live ACP smoke proof is not inspectable.

## What This Changes

The PR changes the ACPX plugin to filter and normalize adapter config options, link Codex auth into an isolated CODEX_HOME, and merge ACP metadata from disk during session-store updates.

## Best Possible Solution

Keep the adapter-specific ACPX hardening, but preserve the session-store cache contract, model unsupported config controls through capabilities or explicit errors, and attach real after-fix ACP smoke proof for the affected adapters.

## Reproduction Assessment

No for the live adapter failures: the PR body gives only a smoke summary, not exact commands or output. Yes for an introduced regression: PR CI reproduces the session-store cache failure in `src/config/sessions.test.ts`.

## Solution Assessment

No. The direction is plausible, but the current patch breaks an existing cache-path contract and should avoid silent success for unsupported config controls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve the session-store cache fast path:** `src/config/sessions/store.ts:446`
  - body: This new unconditional `skipCache` load makes every `updateSessionStore` writer re-read and parse the session store even when the writer-owned mutable cache is valid. The PR's CI is already failing the existing cache-contract test because `readFileSync` is now called, so the ACP metadata merge needs to avoid regressing that fast path or deliberately replace the contract with updated coverage.
  - confidence: 0.93
- **[P2] Report unsupported ACP config instead of no-oping:** `extensions/acpx/src/runtime.ts:981`
  - body: For non-Codex, non-Claude, non-OpenClaw bridge commands, `setConfigOption` now falls through without delegating or throwing. The control plane persists runtime options after a successful return, so a Gemini/custom ACP session with no advertised config keys can report an option update as successful while the adapter never receives it; return an unsupported-control error or express the lack of support through capabilities instead.
  - confidence: 0.78
