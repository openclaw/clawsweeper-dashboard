## Summary

Keep open: the generic approval-action API addresses the related external AgentKit need, but latest head still sends command actions into Telegram native callbacks, leaves the public hook docs on the old contract, and only supplies checks/tests rather than real behavior proof.

## What This Changes

The PR adds plugin approval action templates and keepPendingWithoutRoute through before_tool_call, Gateway protocol/schema, SDK reply/view models, native channel adapters, generated Swift models, and focused tests.

## Best Possible Solution

Land after Telegram native callbacks are decision-only, docs/plugins/hooks.md documents the new approval action contract, and real external-plugin/channel proof demonstrates the flow.

## Reproduction Assessment

Yes for the PR findings by source inspection: latest head still builds Telegram native buttons from every view action, and current docs still show the old requireApproval shape. Current-main bug reproduction is not applicable because this PR adds a new API surface.

## Solution Assessment

Not yet; the generic API direction is reasonable, but the patch needs decision-only Telegram native callbacks, public docs for the contract, and real behavior proof before it is merge-ready.
