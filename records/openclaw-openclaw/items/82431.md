## Solution Assessment

No, not quite yet. The generic action/no-route seam is a reasonable direction, but Telegram decision actions need a narrower approval-system callback path before the PR is merge-safe.

- **[P2] Generate Telegram approval callbacks from the decision:** `extensions/telegram/src/approval-handler.runtime.ts:106-108`
  - body: `kind: "decision"` means the button should resolve through OpenClaw's approval system, but this path passes the plugin-provided `command` into Telegram `callback_data`.
