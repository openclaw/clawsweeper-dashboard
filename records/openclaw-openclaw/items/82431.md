## Summary

Keep open: the generic approval-action API direction is plausible for the external HITL plugin gap, but this PR still needs Telegram native-action cleanup, public docs, and real behavior proof before merge.

## What This Changes

The branch adds plugin approval action templates and keepPendingWithoutRoute plumbing across Gateway protocol/schema, before_tool_call hooks, SDK approval rendering, channel adapters, generated Swift models, and focused tests.

## Best Possible Solution

Land a documented additive approval-action API after Telegram renders only routed decision buttons natively, command actions remain visible as commands, and a real external-plugin/channel proof shows the flow.

## Reproduction Assessment

Yes for the PR review findings by source inspection: PR head still feeds all approval actions into Telegram native buttons, and the current docs omit the new public fields. Current-main bug reproduction is not applicable because this PR adds a new API surface.

## Solution Assessment

Not yet; the generic approval-action direction is reasonable, but the PR should keep Telegram native callbacks decision-only and document the new public contract before it is the maintainable fix path.
