## Summary

Keep open: the approval action seam is a real missing plugin SDK/Gateway capability, but the current patch still lets plugin-controlled decision labels/styles drive protected approval controls, so it is not merge-ready.

## What This Changes

The PR adds plugin approval action templates and a no-route pending flag across the Gateway protocol, plugin SDK types/renderers, approval view models, native channel adapters, Control UI, docs, and generated API/protocol baselines.

## Best Possible Solution

Canonicalize decision action labels/styles/callbacks, keep custom plugin text on command actions, document the AgentKit HITL use cases, and then let maintainers approve the public API bundle with the companion approval PRs.

## Reproduction Assessment

Yes. The PR finding is source-reproducible: a plugin decision action can carry arbitrary label/style, and the shared view plus native/UI renderers use those display values while resolving the canonical decision.

## Solution Assessment

No. The proposed seam is useful, but protected approval decision controls should derive visible text/style from the decision or reject mismatches; custom plugin wording belongs on command actions or descriptive metadata.
