## Summary

Keep this PR open: the API extraction is plausible, but latest head still renders command actions as Telegram callback buttons, leaves the new public hook fields undocumented, and has only test/check output rather than real external-plugin or live-channel proof.

## What This Changes

The PR adds plugin approval action templates and keepPendingWithoutRoute across before_tool_call plugin approvals, Gateway protocol/schema, SDK reply/view models, native channel adapters, generated Swift models, and focused tests.

## Best Possible Solution

Land the generic approval-action API only after Telegram renders native callbacks from decision-backed actions, plugin docs describe the new contract, and a real external-plugin/channel proof demonstrates the flow.

## Reproduction Assessment

Yes for the PR review findings by source inspection: PR head passes all view.actions to Telegram inline buttons, and PR head docs still show the old requireApproval shape. Current-main bug reproduction is not applicable because this PR adds a new API surface.

## Solution Assessment

Not yet; the generic approval-action direction is reasonable, but the patch needs decision-only Telegram native callbacks, public docs for the new contract, and real behavior proof before it is the maintainable solution.
