## Summary

Keep open: this is a plausible generic plugin approval API for https://github.com/openclaw/openclaw/issues/82336, but the current PR head still has a Telegram callback bug and only supplies tests/checks as behavior proof.

## What This Changes

The PR adds plugin approval action templates and keep-pending-without-route plumbing across Gateway protocol, SDK approval rendering, approval view models, channel adapters, Swift generated models, and focused tests.

## Best Possible Solution

Land the bounded generic approval-action API after Telegram only renders routed decision actions as native buttons, command-only actions remain visible as commands, and the companion verified-resolution surface is reviewed coherently.

## Reproduction Assessment

Yes for the PR review finding by source inspection: PR head turns command-only plugin approval actions into Telegram inline callback buttons, while current main routes callback data only through structured callback handlers or /approve approval parsing. Current-main bug reproduction is not applicable because this PR adds a new feature.

## Solution Assessment

Not yet; the generic approval action direction is reasonable, but Telegram should not render command-only actions as native callback buttons unless those callbacks are actually routed. The safer narrow fix is to keep command actions as visible text and only render decision-backed actions as Telegram buttons.
