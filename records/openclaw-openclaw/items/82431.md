## Summary

Keep open: the latest head addresses the prior decision-action validation and docs-shape findings, but it still needs the Web UI approval path to preserve the new action and allowed-decision metadata before merge.

## What This Changes

The PR adds plugin approval action templates and keepPendingWithoutRoute to hook/Gateway schemas, expands stored approval commands, updates approval rendering/adapters, docs, Swift protocol output, and the SDK API baseline.

## Best Possible Solution

Land the generic host API after the Web UI approval prompt consumes action and allowed-decision metadata consistently with the channel adapters, while keeping the companion verified-resolution and chat metadata PRs aligned.

## Reproduction Assessment

Yes for the review finding: source inspection shows plugin approval actions are stored on the Gateway request, while the PR head Web UI parser drops actions/allowedDecisions and the prompt renders hard-coded decision buttons. I did not run a browser repro in this read-only review.

## Solution Assessment

No, not yet: the generic API direction fits the external HITL plugin need and the previous contract issues are fixed, but the Web UI approval route still needs to honor the same action contract as the native channel adapters.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Carry approval actions into the Web UI:** `src/gateway/server-methods/plugin-approval.ts:141-142`
  - body: When an approval client is active, `keepPendingWithoutRoute` is not the path used; the plugin approval is broadcast to clients. This PR now stores custom actions on the request, but the Web UI still parses only title/description metadata and renders hard-coded Allow Once/Allow Always/Deny buttons. For the documented external flow with `allowedDecisions: ["deny"]` plus a command action, the Web UI shows no plugin command and two allow buttons that the resolver rejects. Please preserve actions/allowedDecisions in the Web UI queue and render the available actions, or deliberately keep command-only plugin approvals out of the Web UI route.
  - confidence: 0.84
