## Summary

Keep open: this is a useful generic host/API PR with supplied external integration proof, but the latest diff still has a correctness/security-sensitive approval metadata issue where command actions can be interpreted as native approval decisions. The public Gateway/SDK surface and companion sequencing also need maintainer review after that narrow fix.

## What This Changes

The PR adds plugin approval action templates and keepPendingWithoutRoute, carries expanded commands through Gateway/SDK/view models, updates native channel adapters, docs, generated Swift protocol, and focused tests.

## Best Possible Solution

Enforce discriminated approval actions, keep native callbacks decision-only, then land the generic host API set with the companion verified-resolution and chat metadata PRs if maintainers accept the public API.

## Reproduction Assessment

Not applicable for a current-main bug; this PR adds public plugin/Gateway API surface. The review finding is source-reproducible because the schema permits command actions with decisions and native adapters filter on decision presence.

## Solution Assessment

No, not quite: the generic host seam is the right direction, but the action contract should reject or normalize inconsistent kind/decision pairs before this becomes public API.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Reject command actions that carry decisions:** `src/gateway/protocol/schema/plugin-approvals.ts:17`
  - body: The schema accepts `decision` on every action, including `kind: "command"`, but the native adapters decide whether to render approval buttons by checking only for `action.decision`. A plugin-owned verification command sent as `{ kind: "command", decision: "allow-once" }` would therefore become a built-in approval button instead of visible command text, bypassing the plugin command flow the docs say should collect/verify external context. Please make the action schema/normalization discriminated, or make native button filters require `kind === "decision" && decision`.
  - confidence: 0.84

## Security Review

Status: needs_attention

Concerns:

  - body: Because `decision` is allowed on command actions and native adapters key off `decision`, a plugin-owned verification command can be rendered as a direct native approval button. That weakens the intended separation between plugin verification commands and OpenClaw approval decisions.
