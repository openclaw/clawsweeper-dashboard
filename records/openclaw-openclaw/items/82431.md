## Summary

Keep open: this is a live feature PR for a missing public plugin approval API, but the current patch still has a Telegram callback correctness bug and only test/check output as real behavior proof. It should remain in review rather than be closed as implemented or superseded.

## What This Changes

The PR adds plugin approval action templates and keep-pending-without-route plumbing across Gateway protocol, SDK approval rendering, approval view models, channel adapters, Swift generated models, and focused tests.

## Best Possible Solution

Land the generic approval-action API only after Telegram filters command-only or unrouted custom commands out of callback buttons, contributor proof shows a real plugin or Telegram flow, and the companion verified-resolution surface is reviewed coherently.

## Reproduction Assessment

Yes for the PR review finding by source inspection: PR head renders every plugin approval action as Telegram callback buttons, while current callback routing only handles registered plugin interactive namespaces and /approve approval callbacks. Current-main bug reproduction is not applicable because this PR adds a new API surface.

## Solution Assessment

Not yet; the generic approval-action direction is reasonable, but Telegram should only render routed decision callbacks as native buttons and leave command-only actions as visible text. The safer path is a narrow adapter fix plus maintainer review of the additive public contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep command-only Telegram actions out of callback buttons:** `extensions/telegram/src/approval-handler.runtime.ts:102`
  - body: This builds Telegram inline buttons from every `view.actions` entry. A plugin action such as `/agentkit approve ...` becomes `callback_data`, not a normal chat message, and current Telegram callback routing only dispatches registered `namespace:payload` callbacks or parses `/approve ...` approvals, so tapping the button will not run the plugin command. Filter this to routed decision actions or add a real Telegram callback before rendering command actions as buttons.
  - confidence: 0.92
