Overall confidence: 0.88

Full review comments:

- **[P1] Include session identity in prepared-param cache keys:** `src/agents/pi-embedded-runner/extra-params.ts:159-160`
  - body: Once a provider can return values derived from `ctx.sessionKey`, the prepared-param cache must distinguish sessions or skip caching for session-scoped output. Otherwise the first session that prepares `{ user: ctx.sessionKey }` can leak into later sessions with the same provider/model/config cache key.
  - confidence: 0.9
- **[P2] Prepare runtime extra params with session context:** `src/agents/pi-embedded-runner/run/attempt.ts:2299-2300`
  - body: Current main can pass `options.preparedExtraParams` from the runtime-plan resolver before this later apply call runs. Adding `sessionKey`/`sessionId` only here leaves runtime-plan providers without the new context, so the public hook would be inconsistently populated.
  - confidence: 0.84
- **[P2] Populate every hook context that inherits these fields:** `src/plugins/types.ts:465-476`
  - body: `ProviderWrapStreamFnContext` and transport extra-param contexts inherit from `ProviderPrepareExtraParamsContext`. Adding session fields to the base type advertises them on hooks this patch does not populate, so plugin authors can see typed fields that are `undefined` on real call paths.
  - confidence: 0.82
