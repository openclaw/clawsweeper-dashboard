## Summary

Keep open. Current main still routes models auth login through the shared config writer, where canonical two-space JSON output is compared against the previous raw file bytes and can still reject a PowerShell/BOM-formatted config as a destructive size drop.

## What This Changes

The issue asks OpenClaw to persist OAuth auth-login config changes when an existing `openclaw.json` shrinks only because PowerShell/BOM formatting is normalized.

## Best Possible Solution

Fix the shared config writer to allow formatting-only shrink through semantic or canonicalized comparison while preserving fail-closed rejection for real destructive writes.

## Reproduction Assessment

Yes, source-level. A valid but raw-large `openclaw.json` can flow through `models auth login` to `updateConfig`, then `writeConfigFile` compares canonical output size against raw previous bytes and can reject the write as `size-drop`.

## Solution Assessment

Yes. A writer-level semantic or canonicalized size-drop check is the narrow maintainable fix; an auth-only bypass would leave the same false positive in other config mutations.
