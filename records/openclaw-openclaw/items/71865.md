## What This Changes

The issue reports that a PowerShell/BOM-formatted openclaw.json can make models auth login appear successful while the shared size-drop guard rejects the persisted config change.

## Best Possible Solution

Refresh, land, or replace the linked writer-level canonical previous-size comparison fix with regressions for verbose/BOM formatting while preserving real destructive-shrink rejection.

## Solution Assessment
