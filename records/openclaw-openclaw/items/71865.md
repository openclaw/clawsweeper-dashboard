## Summary

Keep this open: current main still uses the raw existing config byte count for the size-drop guard, and the issue has an open draft fix PR linked with closing syntax rather than a merged implementation.

## What This Changes

Auth login should persist OAuth config updates even when an existing openclaw.json has PowerShell-style UTF-8 BOM and verbose formatting that makes the canonical rewrite much smaller.

## Best Possible Solution

Land a focused config-writer fix that compares the size-drop guard against canonical bytes for valid existing configs while preserving raw-byte protection for malformed configs, then close this issue after the PR merges.

Yes for source-level reproduction: current main compares canonical output bytes against raw existing file bytes, and model/auth config updates go through the shared writer. I did not run a live Windows/PowerShell repro in this read-only review.

## Solution Assessment
