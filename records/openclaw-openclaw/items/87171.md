## Summary

Keep this PR open: it fixes the marker and notifyUser paths in the right area, but the new room_event bypass is narrower than the current explicit-command policy and can still silently drop marked command replies for supported command contexts that lack CommandSource metadata.

## What This Changes

The PR changes auto-reply dispatch to deliver marked command replies through room_event source suppression, marks terminal command-handler replies for delivery, and decouples compaction notifyUser notices from internal callbacks.

## Best Possible Solution

The dispatch bypass should share the current source-reply explicit-command detector so every supported explicit command shape can deliver marked terminal replies while ambient room_event notices and sendPolicy deny stay suppressed.

## Reproduction Assessment

Yes, source-reproducible: current main defines authorized control-command bodies without CommandSource as explicit-command contexts, while the PR merge ref's room_event bypass only accepts structured native/text command turns. A marked room_event terminal reply in that supported shape still falls through source suppression.

## Solution Assessment

No, not yet: the patch is close, but the room_event bypass should use the same explicit source-reply command policy as the rest of dispatch instead of a narrower structured-turn check.
