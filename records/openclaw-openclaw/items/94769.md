## Summary

Keep open: current main still has high static Dreaming promotion thresholds and only records live short-term recall signals from `memory_search`; the related threshold PR is closed unmerged, while the merged related PRs only cover store growth and skipped-hit diagnostics.

## What This Changes

The issue asks memory-core Dreaming to promote short-term recall reliably with Gemini-like embedding scores and to account for non-search memory access signals.

## Best Possible Solution

Define the memory-core Dreaming calibration and signal-source contract, then implement an upgrade-safe threshold policy plus approved recall-signal sources with docs, regression coverage, and live Gemini-backed proof.

## Reproduction Assessment

Partly. Source inspection on current main shows the static threshold gates and `memory_get` signal gap, but the exact Gemini score distribution and live after-fix promotion path still need a real workspace reproduction.

## Solution Assessment

Unclear. Lowering minScore alone is a partial mitigation; the maintainable solution needs maintainer direction on provider calibration, upgrade safety, and which access paths may safely count as promotion signals.
