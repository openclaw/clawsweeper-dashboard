## Summary

Keep open: current main and the latest release still have static high Dreaming promotion thresholds and memory_get/read/exec do not feed the same short-term recall signal path; the open linked PR is a partial threshold proposal, not a merged or complete fix.

## What This Changes

The issue requests a fix for memory-core Dreaming promotion when Gemini-like embedding scores and missing non-search recall signals prevent short-term recall entries from reaching durable-memory promotion.

## Best Possible Solution

Define the memory-core Dreaming signal and calibration contract, then align defaults, eligibility, tests, docs, and live Gemini-backed proof before closing this issue.

## Reproduction Assessment

Partly. Source inspection shows the static threshold gates and the memory_get signal gap on current main; the exact Gemini score distribution and after-fix live promotion path still need live workspace proof.

## Solution Assessment

Unclear. Lowering minScore alone is a plausible partial mitigation, but the maintainable fix needs a maintainer decision on provider calibration and which access paths should count as promotion signals.
