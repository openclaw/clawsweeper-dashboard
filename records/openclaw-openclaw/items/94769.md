## Summary

Keep open: current main and v2026.6.10 still have static high Dreaming promotion thresholds and no `memory_get` recall-tracking hook; related fixes only cover retention/diagnostics or are closed unmerged, so this remains the canonical memory-core policy/bug item.

## What This Changes

The issue reports that memory-core Dreaming promotion rarely or never promotes with Gemini-like embedding score ranges and does not collect recall signals from non-search memory access paths.

## Best Possible Solution

Define an upgrade-safe memory-core Dreaming calibration and signal-source contract, then align defaults, eligibility, docs, tests, and live Gemini/common-embedding proof before closing this issue.

## Reproduction Assessment

Partly. Source inspection on current main reproduces the static threshold gates and the `memory_get` signal gap, but the exact Gemini score distribution and after-fix live promotion behavior still need a live workspace proof.

## Solution Assessment

Unclear. A threshold-only default change is a partial mitigation; the best fix needs maintainer direction on provider-aware calibration and which access paths should become promotion signals.
