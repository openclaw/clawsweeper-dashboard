## Summary

Keep open: current main still has static high deep Dreaming promotion thresholds and memory_search-only live recall tracking, while the linked open PR covers only the threshold portion and must not close this issue before merge.

## What This Changes

Adjust memory-core Dreaming promotion so Gemini-like embedding scores and non-search memory evidence can reliably promote qualified short-term recall entries.

## Best Possible Solution

Define the Dreaming signal contract and provider threshold policy first, then land a threshold/config/docs/test change and keep this issue open until the broader signal-source gap is either fixed or deliberately split.

## Reproduction Assessment

Yes at source level for the memory_get signal gap and static threshold gates: memory_search queues recall tracking, memory_get only reads content, and deep ranking still gates on fixed defaults. The exact Gemini distribution and end-to-end live promotion failure were not reproduced here.

## Solution Assessment

Unclear. Lowering minScore is a plausible partial mitigation, but the best fix needs maintainer-approved threshold policy plus a clear contract for which memory access signals should count toward durable promotion.
