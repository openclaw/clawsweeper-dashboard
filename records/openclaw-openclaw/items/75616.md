## Summary

Keep open. The issue is fresh, has concrete version-to-version timing evidence, and current `main` still performs the named embedded-run phases on the per-request path. ## What This Changes

The issue reports a `v2026.4.29` per-request embedded-run startup latency regression before model output, with slow `core-plugin-tools`, `auth`, `model-resolution`, and dispatch/setup stages.

## Best Possible Solution

Profile the same embedded-run scenario on current `main`, then move or cache only the work that is contractually safe to amortize: plugin/runtime registry reuse, plugin tool metadata/factory costs, model resolution, and provider auth preparation. Keep request-specific channel, sender, sandbox, and auth state fresh, and add focused regression coverage or a benchmark around the slow stage breakdown.
