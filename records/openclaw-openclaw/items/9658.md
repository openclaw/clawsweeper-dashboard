## Summary

Keep open: current main still routes exec, write, Gateway, and SDK tool invocation through inline JSON/object argument contracts. Existing stream-buffer caps and malformed-argument repairs help narrower failures, but they do not provide the requested streaming or file-backed argument carrier.

## What This Changes

Support streaming, staged, or file-backed tool arguments so large exec/write and Gateway/SDK tool calls are not limited to inline JSON serialization.

## Best Possible Solution

Define one additive staged-payload or streaming argument contract across tool schemas, Gateway HTTP/RPC, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this asks for a new streaming or file-backed argument contract. Source inspection confirms current main still lacks those carriers in exec, write, Gateway, and SDK surfaces.

## Solution Assessment

Unclear. The direction is valid, but maintainers should choose one explicit staged-payload or streaming contract rather than ad hoc shell rewriting, transparent chunking, or only raising JSON and buffer limits.
