## Summary

Keep open. Current main still exposes `exec`, `write`, and Gateway tool invocation through inline JSON object/string arguments, while the open provider-specific empty-argument reports at https://github.com/openclaw/openclaw/issues/53408 and https://github.com/openclaw/openclaw/issues/70033 cover narrower bug fixes rather than this broader staged/streaming argument design.

## What This Changes

Request support for streaming or file-backed tool arguments so large `exec` and `write` payloads do not have to be embedded as JSON strings.

## Best Possible Solution

Design a generic staged-input or large-argument contract at the agent/tool/protocol boundary, while keeping provider-specific empty-argument failures in their narrower linked issues.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Current-main source confirms the requested streaming/file-backed argument contract is absent, and concrete provider failures remain tracked in https://github.com/openclaw/openclaw/issues/53408 and https://github.com/openclaw/openclaw/issues/70033.

## Solution Assessment

Unclear. A staged/file-backed argument design may be the right durable direction, but it changes tool, protocol, and security contracts; provider-specific empty-argument bugs should be fixed narrowly in their canonical issues.
