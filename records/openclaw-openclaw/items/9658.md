Keep open. Current main and v2026.6.5 still model tool arguments as inline JSON/object fields, while the requested streaming or file-backed large-argument contract remains a cross-boundary product/protocol design item.

## What This Changes

The issue requests streaming or file-backed handling for large tool arguments so exec/write-style calls are not constrained by inline JSON serialization.

## Best Possible Solution

Design one explicit staged or streamed large-argument contract that keeps normal inline JSON args while adding a secure, documented large-payload path for model tool calls, gateway invocation, and native execution boundaries.

## Reproduction Assessment

No; no live current-main reproduction was run, and this item is primarily a feature/design request. Source inspection does confirm current main lacks the requested streaming or file-backed argument capability.

## Solution Assessment

Unclear; the issue lists several plausible implementation options rather than one clearly best path. The best fix needs a maintainer-approved staged-payload design because streaming, file-backed args, automatic chunking, and higher JSON limits have different protocol and security tradeoffs.
