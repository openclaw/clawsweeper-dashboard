## Summary

Keep open. Current main and the latest release still model tool arguments as inline JSON/object fields, while the requested streaming or file-backed large-argument contract remains a product/protocol design item rather than an implemented behavior.

## What This Changes

The issue requests a streaming or file-backed way to pass large tool arguments so write/exec-style tool calls are not limited by inline JSON serialization.

## Best Possible Solution

Design one explicit staged or streamed large-argument contract that keeps normal inline JSON args while adding a secure, documented large-payload path for model tool calls, gateway invocation, and Codex/native execution boundaries.

## Reproduction Assessment

No; no live current-main reproduction was run, and this item is primarily a feature/design request. Source inspection does confirm current main lacks the requested streaming or file-backed argument capability.

## Solution Assessment

Unclear; the issue lists several plausible implementation options rather than one clearly best path. The best fix needs a maintainer-approved staged-payload design because streaming, file-backed args, automatic chunking, and higher JSON limits have different protocol and security tradeoffs.
