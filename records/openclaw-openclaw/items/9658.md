## Summary

Keep open. Current main and v2026.5.22 still expose exec/write/client tools as inline JSON/object arguments; existing repairs only parse malformed streamed JSON/provider deltas, while the requested streamed or file-backed argument contract needs maintainer protocol and security design.

## What This Changes

The issue requests a streaming or file-backed way to pass large tool arguments so write/exec-style tool calls are not limited by inline JSON serialization.

## Best Possible Solution

Define an additive large-tool-input contract owned by the agent/tool protocol, with sandbox-aware staging, redacted diagnostics, and compatibility tests, while leaving narrower provider-specific argument-loss bugs tracked separately.

## Reproduction Assessment

## Solution Assessment
