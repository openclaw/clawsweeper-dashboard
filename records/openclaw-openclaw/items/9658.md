## Summary

Keep open. Current main still routes tool input through inline JSON arguments and validates `exec`/`write` from those objects; no streamed, staged, or file-backed argument contract exists, and the linked large-argument failure reports remain open.

## What This Changes

Request streaming or file-backed tool argument delivery so large `exec` and `write` inputs do not have to fit inside inline JSON tool-call arguments.

## Best Possible Solution

Keep this as the API/product tracker for staged or file-backed tool input, while provider-specific empty-argument bugs remain in narrower reports.

## Reproduction Assessment

No, not for the exact greater-than-10KB JSON error against current main. Source inspection does confirm the requested streamed/file-backed API is absent, and linked reports provide concrete large-argument failure context.

## Solution Assessment

Unclear until maintainers choose the API boundary. A staged content or file-backed stdin contract looks safer than automatic shell chunking, while provider-specific argument-loss bugs should stay in their narrower trackers.
