## Summary

Keep open: current main and the latest release still expose only inline JSON/string contracts for exec, write, and tools.invoke, while the related open bug reports cover provider-specific argument-loss failures and this broader feature needs maintainer product and security design.

## What This Changes

Request support for streamed or file-backed tool arguments so large exec/write payloads do not rely entirely on inline JSON serialization.

## Best Possible Solution

Design a generic staged-input or streamed-input contract across agent tool schemas, Gateway protocol, SDK types, docs, execution policy, and audit/security behavior, while keeping provider-specific empty-argument bugs in their narrower issue tracks.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Unclear. Streaming or staged files may be the durable direction, but the public contract needs maintainer product and security design; a schema-only or transport-only patch would be too narrow for this request.
