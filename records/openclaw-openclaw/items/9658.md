## Summary

Keep open: current main still exposes `exec`, `write`, Gateway `tools.invoke`, and SDK tool invocation as inline JSON/string payloads, and the related searches found narrower open bug trackers but no merged staged, streamed, or file-backed argument contract that supersedes this request.

## What This Changes

Add streamed or file-backed tool argument support so large `exec`, `write`, and other tool payloads do not have to be embedded entirely in JSON.

## Best Possible Solution

Define one additive staged-payload or streaming contract and carry it consistently through tool schemas, Gateway HTTP/RPC protocol, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a new transport/API capability request. Source inspection on current main gives high confidence that streamed or file-backed tool arguments are still absent.

## Solution Assessment

Unclear. The direction is valid, but maintainers should choose a single explicit transport contract rather than adding ad hoc shell rewriting, transparent chunking, or only raising JSON body limits.
