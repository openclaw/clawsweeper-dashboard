## Best Possible Solution

Capture raw provider stream events and the resulting normalized tool call on current main, then harden the existing provider/tool-call repair boundary with explicit malformed or truncated argument diagnostics; consider a new typed large-payload protocol only if intact provider payloads demonstrably fail inside OpenClaw.
