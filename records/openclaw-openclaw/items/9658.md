## Summary

Keep open: current main still exposes JSON-only tool argument contracts, and the existing parser/repair safeguards do not provide the requested staged, streamed, or file-backed transport.

## What This Changes

The issue requests streaming or file-based tool arguments so large exec, write, process, and similar tool inputs can bypass inline JSON serialization.

## Best Possible Solution

Define one explicit large-input contract, such as staged file tokens or streamed stdin, then wire it through Gateway protocol, OpenClaw-owned tools, provider adapters, docs, and regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main still accepts JSON string/object tool arguments only on the relevant exec, write, process, and Gateway invoke surfaces.

## Solution Assessment

Unclear which proposed option is best. A maintainer should choose a single explicit transport contract rather than adding ad hoc automatic chunking or command rewriting.
