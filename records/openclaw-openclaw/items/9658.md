## Summary

Keep open: current main still models exec, process, write, Gateway RPC/HTTP tool invokes, and SDK tool invokes as inline JSON string/object arguments, and no exact merged fix or canonical duplicate implements a streamed or staged payload transport.

## What This Changes

The issue requests streaming or file-backed tool arguments so large exec/write-style payloads do not have to be carried entirely inside JSON-serialized tool parameters.

## Best Possible Solution

Define one explicit staged-payload or streaming contract, then wire it through tool schemas, Gateway protocol and HTTP invoke, the SDK, hooks/approvals, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source, docs, and SDK inspection give high confidence that current main still only supports inline JSON arguments on the affected paths.

## Solution Assessment

Unclear which proposed option is best. Maintainers should choose a single explicit transport contract that preserves permission, audit, and lifecycle boundaries instead of adding ad hoc automatic shell-command rewriting.
