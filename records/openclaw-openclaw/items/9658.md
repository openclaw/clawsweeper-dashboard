## Summary

Keep open. Current main still routes dynamic tool calls and exec/write payloads through inline JSON arguments only, and related open issues cover narrower provider failures rather than replacing this broader staged or file-backed API decision.

## What This Changes

Requests streamed, staged, or file-backed tool arguments so large exec/write payloads do not have to be serialized inline through JSON tool-call parameters.

## Best Possible Solution

Keep this as the product/API tracker for staged or file-backed OpenClaw tool input while provider-specific large-argument loss, timeout, and malformed-stream bugs stay in narrower reports.

## Reproduction Assessment

No high-confidence live reproduction of the exact greater-than-10KB JSON parse failure was run. Source inspection does confirm that current main lacks the requested streamed, staged, or file-backed tool-argument API and still uses inline JSON arguments.

## Solution Assessment

Unclear until maintainers choose the API boundary. A staged-content or file-backed stdin contract at the dynamic-tool/protocol layer looks safer than transparent shell chunking, while provider-specific empty-argument bugs should stay in their narrower trackers.
