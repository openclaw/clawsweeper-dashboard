## Summary

Keep open: current main and the latest release still expose inline JSON/string tool-argument contracts, while the linked provider-specific empty-argument reports cover narrower bugs and this broader API change needs maintainer product and security design.

## What This Changes

Request adding streamed or file-backed tool argument support so large exec/write/tool payloads do not depend on inline JSON serialization.

## Best Possible Solution

Design a generic staged-input or streamed-input contract across agent tool schemas, Gateway protocol, SDK types, docs, execution policy, and audit/security behavior, while keeping provider-specific empty-argument bugs in narrower issue tracks.

Unclear. Streaming or staged files may be the durable direction, but the public contract needs maintainer product and security design; a schema-only or transport-only patch would be too narrow.
