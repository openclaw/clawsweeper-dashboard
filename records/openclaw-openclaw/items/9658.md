## Summary

Keep open. Current main and v2026.6.5 still use inline JSON/object/string arguments for OpenClaw tool calls; recent argument-repair paths and Codex native stdin support do not add the requested staged, file-backed, or streaming large-argument contract.

## What This Changes

Request support for streaming, file-backed, automatic chunking, or larger-limit tool arguments so large exec/write payloads do not have to be serialized inline as JSON.

## Best Possible Solution

Design one maintainer-approved staged-argument contract that keeps normal inline JSON args while adding an explicit large-payload path for exec/write and gateway invocation, with documented limits, cleanup, security review, and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Reproduction is not the main gate for this feature request; source inspection confirms current main lacks the requested streaming/file-backed argument capability.

## Solution Assessment

Unclear: the issue body lists plausible options rather than one clearly best implementation. The best path is a maintainer-approved staged-argument design because streaming, file-backed args, automatic chunking, and higher JSON limits have different protocol and security tradeoffs.
