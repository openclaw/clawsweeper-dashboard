## Summary

Keep open: current main still exposes exec, write, Gateway, and SDK tool invocation through inline JSON/string argument contracts, while the requested staged or streaming payload carrier would be a new public contract requiring maintainer design.

## What This Changes

Add streaming or file-backed tool argument carriers so large exec/write payloads do not rely on inline JSON serialization.

## Best Possible Solution

Define one additive staged-payload or streaming argument contract across tool schemas, Gateway HTTP/RPC, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this asks for a new streaming or file-backed argument contract. Source inspection on current main shows those carriers are still absent.

## Solution Assessment

Unclear. The direction is valid, but maintainers should choose one explicit staged-payload or streaming contract instead of ad hoc shell rewriting, transparent chunking, or only raising JSON limits.
