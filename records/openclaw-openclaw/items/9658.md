## Summary

This broad feature request should close because its actionable failure modes are already tracked by two narrower open issues with stronger environment and reproduction evidence: provider/timeout behavior in https://github.com/openclaw/openclaw/issues/53540 and empty or dropped arguments in https://github.com/openclaw/openclaw/issues/53408. Current main also already implements provider-native tool-input streaming and preserves malformed replay arguments, so a separate generic JSON-bypass tracker would fragment ownership rather than clarify the remaining work.

## What This Changes

The issue requests streaming, file-backed, chunked, or otherwise more reliable handling of large tool-call arguments.

## Best Possible Solution

Keep the existing provider-native streaming and malformed-argument preservation paths, and resolve measurable timeout behavior through https://github.com/openclaw/openclaw/issues/53540 and empty or dropped arguments through https://github.com/openclaw/openclaw/issues/53408.

## Reproduction Assessment

No. This issue does not establish its universal greater-than-10KB JSON serialization claim on current main or identify one exact provider path; the two canonical linked issues contain the stronger actionable reproduction evidence.

## Solution Assessment

No. A generic streaming or file-backed tool API is broader than the observed provider timeout, streaming, and malformed-argument boundaries and would create a parallel source of behavior.
