## Summary

Keep open: current main still carries tool arguments as inline JSON strings/objects across exec, write, Gateway HTTP/RPC, and the SDK, and no exact merged fix or canonical duplicate implements streamed or staged payload transport.

## What This Changes

The issue requests streamed or file-backed tool arguments so large exec/write payloads can avoid inline JSON serialization limits.

## Best Possible Solution

Define one additive staged-payload or streaming contract, then wire it through tool schemas, Gateway protocol and HTTP invoke, the SDK, hooks/approvals, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection on current main gives high confidence that the requested streamed or staged transport is still absent.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers should choose a single explicit transport contract rather than adding ad hoc shell rewriting or only raising JSON limits.
