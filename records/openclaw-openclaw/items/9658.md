## Summary

Keep open. Current main and latest v2026.6.5 still expose exec/write and gateway tool invocation through inline JSON object/string arguments; adjacent repair work improves malformed/fragmented tool calls but does not add the requested streamed or file-backed large-argument contract.

## What This Changes

The issue requests streamed or file-backed tool arguments so large exec/write payloads do not rely entirely on inline JSON serialization.

## Best Possible Solution

Design one core tool-argument staging contract that preserves normal JSON object args, adds an explicit large-payload path for exec/write and gateway invocation, documents limits, and covers fresh plus upgrade behavior with regression tests.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection does show the requested capability is still missing: exec/write and gateway invocation still use inline JSON object/string arguments.

## Solution Assessment

Unclear as written: streaming, file-backed args, automatic chunking, and higher limits are all plausible but have different protocol and security tradeoffs. The best path is a maintainer-approved staged-argument design, not a narrow autonomous patch.
