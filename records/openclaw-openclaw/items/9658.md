## Summary

Keep open: current main still exposes inline JSON/string tool-argument contracts for exec/write and Gateway/SDK `tools.invoke`, with no streaming, staged-payload, or file-backed argument carrier. The related open report covers a narrower provider/runtime argument-loss path and does not supersede this API/protocol design request.

## What This Changes

Request support for streaming or file-backed large tool arguments so exec/write and Gateway/SDK tool calls are not limited to inline JSON strings.

## Best Possible Solution

Define one additive staged-payload or streaming argument contract across tool schemas, Gateway HTTP/RPC, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this asks for a new streaming or file-backed argument contract. Source inspection on current main shows the requested carriers are still absent.

## Solution Assessment

Unclear. The direction is valid, but maintainers should choose a single explicit staged-payload or streaming contract instead of ad hoc shell rewriting, transparent chunking, or only raising JSON limits.
