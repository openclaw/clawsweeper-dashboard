## Summary

Keep open. Current main and v2026.5.12 still expose only inline JSON/string arguments for exec, write, and tools.invoke, while the related open reports track narrower provider/transport argument-loss bugs; this broader API contract needs maintainer product and security design.

## What This Changes

Request support for streamed or file-backed tool arguments so large exec/write payloads do not depend on inline JSON serialization.

## Best Possible Solution

Design a generic staged-input or streamed-input contract across agent tool schemas, Gateway protocol, SDK types, docs, and tool security/audit policy, while keeping provider-specific empty-argument failures in their narrower issue tracks.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and latest-release inspection give high confidence that the streamed or file-backed argument contract is absent, while related issues carry the bug-specific reproduction work.

## Solution Assessment

Unclear. Streaming or staged files may be the durable direction, but the exact public contract needs maintainer product and security design; a narrow schema patch would be risky.
