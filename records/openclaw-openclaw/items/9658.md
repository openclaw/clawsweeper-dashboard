## Summary

Keep open: current main still exposes exec, write, Gateway tool invoke, and SDK tool calls as inline JSON/string payloads, and the related searches found narrower open bug reports but no merged staged or streaming payload contract that supersedes this feature request.

## What This Changes

Request support for streamed or file-backed tool arguments so large exec/write payloads do not have to travel as inline JSON strings.

## Best Possible Solution

Define one additive staged-payload or streaming contract and carry it consistently through tool schemas, Gateway protocol and HTTP/RPC invoke, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a new transport/API capability request. Source inspection on current main gives high confidence that the requested streamed or file-backed argument path is still absent.

## Solution Assessment

Unclear. The direction is valid, but maintainers should choose a single explicit transport contract rather than adding ad hoc shell rewriting or only raising JSON size limits.
