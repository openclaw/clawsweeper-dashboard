## Summary

Keep open: current main still exposes exec/write and Gateway/SDK tool invocation through inline string/JSON argument contracts only. The related open report at https://github.com/openclaw/openclaw/issues/53408 is a narrower provider/runtime argument-loss bug and does not supersede this broader staged-payload or streaming tool-argument design request.

## What This Changes

Request support for streaming or file-backed tool arguments so large exec/write payloads do not have to travel as inline JSON strings.

## Best Possible Solution

Define one additive staged-payload or streaming argument contract across tool schemas, Gateway HTTP/RPC, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a strict bug reproduction because this asks for a new streaming or file-backed argument contract; source inspection shows those carriers are absent on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The direction is valid, but maintainers should choose one explicit staged-payload or streaming contract instead of ad hoc shell rewriting, transparent chunking, or only raising JSON limits.
