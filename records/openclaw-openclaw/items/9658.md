## Summary

## What This Changes

Add streaming, file-backed, automatic chunking, or higher-limit handling for large tool arguments that are reported as malformed or dropped.

## Best Possible Solution

Preserve one canonical tool path: first make malformed or truncated provider arguments fail with precise transport diagnostics under the related reliability investigation, then separately specify stdin/file-backed inputs only if real payload testing proves existing provider contracts cannot reliably carry the required data.

## Solution Assessment

No. Streaming or file-backed arguments may be a useful harness capability, but they are broader than the demonstrated defect; provider-boundary preservation and explicit malformed-argument errors are the narrower first solution.
