## Summary

Keep open: current main still carries exec/write and Gateway/SDK tool arguments inline as JSON object/string fields, and the related open issues cover narrower provider/runtime truncation failures rather than this public streaming or file-backed argument contract.

## What This Changes

The issue requests streaming or file-based tool arguments so large exec/write payloads do not have to be carried inline through JSON serialization.

## Best Possible Solution

Define one additive staged-payload or streaming argument contract across tool schemas, Gateway HTTP/RPC, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this asks for a new streaming or file-backed argument carrier. Source inspection on current main shows that carrier is still absent.

## Solution Assessment

Unclear. The direction is valid, but maintainers should choose one explicit staged-payload or streaming contract instead of ad hoc shell rewriting, transparent chunking, or only raising JSON limits.
