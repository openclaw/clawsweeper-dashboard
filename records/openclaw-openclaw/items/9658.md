## Summary

Keep open. Current main and v2026.6.5 still route tool calls through inline JSON object/string arguments; recent repairs improve malformed or oversized streamed tool-call handling, but they do not add the requested streamed or file-backed large-argument contract.

## What This Changes

The issue requests streamed or file-backed large tool arguments so write/exec-style calls can avoid inline JSON serialization limits.

## Best Possible Solution

Design one maintainer-approved staged-argument contract that preserves normal inline JSON args while adding an explicit large-payload path for exec/write and gateway invocation, with documented limits and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source and tests show the remaining capability gap: current main still uses inline JSON object/string arguments and intentionally fails fast beyond the streamed tool-call argument cap.

## Solution Assessment

Unclear as proposed: streaming, file-backed args, automatic chunking, and higher limits have different protocol and security tradeoffs. The best path is a maintainer-approved staged-argument design rather than a narrow autonomous patch.
