## Summary

Keep open. Current main still models OpenClaw/Codex dynamic tool inputs and write/exec payloads as inline JSON arguments, with targeted malformed-argument repairs but no streamed, staged-content, or file-backed argument contract; linked provider-specific large-argument reports remain open.

## What This Changes

Request support for streamed or file-backed large tool arguments so write/exec and subagent tool calls do not rely on oversized inline JSON payloads.

## Best Possible Solution

Keep this as the product/API tracker for staged or file-backed OpenClaw tool input, while resolving provider-specific empty-argument failures in their narrower linked reports.

## Reproduction Assessment

No, not for the exact current-main greater-than-10KB Bad escaped character failure; this read-only sweep did not run live provider/tool traffic. Source inspection does confirm the requested streamed or file-backed argument API is absent and current paths still use inline JSON arguments.

## Solution Assessment

Unclear until maintainers choose the API boundary. A staged-content or file-backed stdin contract at the dynamic-tool/protocol layer looks safer than automatic shell chunking, while provider-specific empty-argument bugs should stay in their narrower trackers.
