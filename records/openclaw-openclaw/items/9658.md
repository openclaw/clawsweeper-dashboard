## What This Changes

Requested support for streamed or file-backed large tool arguments so exec/write-style tool calls can avoid inline JSON serialization limits.

## Best Possible Solution

Define an additive large-tool-input contract in the agent/tool protocol with sandbox-aware staging, redacted diagnostics, compatibility tests, and separate tracking for provider-specific argument-loss bugs.

No high-confidence live reproduction was run. Source inspection confirms current main lacks the requested streamed or file-backed argument path, while related open reports provide field evidence for large write/exec argument loss.

Unclear. Streaming or file-backed arguments are plausible, but the best fix needs maintainer choice across provider streaming, temp-file staging, protocol compatibility, sandboxing, redaction, and cleanup.
