## Summary

Keep open. Current main and the latest release still route `exec`, `write`, SDK `tools.invoke`, and Gateway tool invocation through inline JSON/string arguments; the open related reports are narrower provider/transport argument-loss bugs, so this broader staged or streaming argument contract still needs maintainer product design.

## What This Changes

Request support for streamed or file-backed tool arguments so large `exec` and `write` payloads do not have to be serialized inline as JSON.

## Best Possible Solution

Design a generic staged-input or streamed-input contract across agent tools, Gateway protocol, SDK types, docs, and tool security policy, while keeping provider-specific empty-argument failures in their narrower bug tracks.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection gives high confidence that the requested streamed or file-backed argument contract is absent on current main and in `v2026.5.12`.

## Solution Assessment

Unclear. Streaming or staged file inputs may be the durable direction, but the best shape needs maintainer product and security design because it changes public tool/protocol contracts and execution boundaries.
