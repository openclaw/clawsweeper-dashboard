## Summary

Keep open. Current main and v2026.5.12 still expose exec, write, and tools.invoke as inline JSON/string argument contracts; the linked open reports cover narrower provider-specific empty-argument bugs, but this broader staged or streamed argument model needs maintainer product and security design.

## What This Changes

Request support for streamed or file-backed tool arguments so large exec/write payloads can avoid inline JSON serialization limits.

## Best Possible Solution

Design a generic staged-input or streamed-input contract across agent tool schemas, Gateway protocol, SDK types, docs, and tool security/audit policy, while keeping provider-specific empty-argument bugs in their narrower issue tracks.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and latest-release inspection give high confidence that the streamed or file-backed argument contract is absent, while linked reports carry the bug-specific reproduction work.

## Solution Assessment

Unclear. Streaming or staged files may be the durable direction, but the exact public contract needs maintainer product and security design; a narrow schema-only patch would be risky.
