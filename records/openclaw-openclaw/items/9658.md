## Summary

Keep open. Current main and v2026.5.22 still pass exec/write/tool-invoke inputs as inline JSON object or string parameters; the landed repairs address malformed provider argument parsing and buffer caps, not the requested streamed or file-backed large-argument contract.

## What This Changes

Request support for streamed or file-backed tool arguments so large exec/write payloads do not have to travel as inline JSON strings.

## Best Possible Solution

Define an additive large-tool-input contract in the agent/tool protocol with sandbox-aware staging, redacted diagnostics, compatibility tests, and separate tracking for provider-specific argument-loss bugs.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does confirm that current main lacks the requested streamed or file-backed argument path, and related open reports provide field evidence for large write/exec argument loss.

## Solution Assessment

Unclear. Streaming or file-backed arguments are plausible, but the best fix needs maintainer choice across provider streaming, temp-file staging, protocol compatibility, sandboxing, and redacted diagnostics.
