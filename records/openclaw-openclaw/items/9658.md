## Summary

Keep open: current main still uses inline JSON/string tool-argument contracts for exec, write, Gateway RPC, and SDK invocation, while related open reports cover narrower provider/parser symptoms rather than the requested generic streamed or file-backed argument contract.

## What This Changes

The issue requests streamed or file-backed tool argument support so large exec/write payloads can avoid inline JSON serialization limits.

## Best Possible Solution

Design a generic staged-input or streamed-input contract across agent tool schemas, Gateway protocol, SDK types, docs, execution policy, and audit/security behavior, while keeping provider-specific argument-dropping bugs in narrower issue tracks.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. ## Solution Assessment

Unclear. Streaming or staged files may be the durable direction, but the public contract needs maintainer product and security design; a schema-only or transport-only patch would be too narrow.
