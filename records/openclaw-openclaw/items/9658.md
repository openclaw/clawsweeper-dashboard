## Summary

Keep open: current main still exposes inline JSON string/object tool arguments for exec, process/write, file write, and Gateway `tools.invoke`; the existing malformed-argument repair is narrower than the requested large-argument transport, so this remains public API/product design work.

## What This Changes

The issue requests streaming or file-backed large tool arguments so exec, write, process, and subagent tool calls can avoid inline JSON serialization failures.

## Best Possible Solution

Define one explicit core large-input contract, such as a staged file/token or streamed stdin primitive, then wire it through Gateway protocol, OpenClaw-owned tools, provider adapters, docs, and focused regression coverage without automatic shell-command rewriting.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main still only supports inline JSON string/object arguments for the affected paths.

## Solution Assessment

Unclear which proposed option is best. Maintainers should choose a single explicit transport contract that preserves tool permission and audit boundaries instead of adding ad hoc automatic chunking.
