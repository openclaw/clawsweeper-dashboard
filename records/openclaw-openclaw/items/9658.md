## Summary

Keep open: current main still carries exec, write, HTTP `/tools/invoke`, Gateway `tools.invoke`, and SDK calls as inline string/JSON payloads, while the related open bugs cover narrower provider/runtime failures rather than a shipped staged-payload or streaming argument contract.

## What This Changes

The issue requests streamed or file-backed tool arguments so large exec/write payloads do not have to travel as inline JSON strings.

## Best Possible Solution

Define one additive staged-payload or streaming contract and carry it consistently through tool schemas, Gateway HTTP/RPC invoke, SDK types, approvals/hooks, audit logging, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a new transport/API capability request. Source inspection on current main gives high confidence that the requested streamed or file-backed argument path is still absent.

## Solution Assessment

Unclear. The direction is valid, but maintainers should choose a single explicit staged-payload or streaming contract rather than adding ad hoc shell rewriting, transparent command chunking, or only raising JSON limits.
