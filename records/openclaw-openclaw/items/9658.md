## Summary

Keep open. Current main still routes `exec`, `write`, SDK `tools.invoke`, and Gateway tool invocation through inline JSON/string arguments, while the linked open bugs cover narrower provider-specific empty-argument failures rather than this broader staged or streaming argument contract.

## What This Changes

Request support for streaming or file-backed tool arguments so large `exec`/`write` payloads do not have to be serialized inline as JSON.

## Best Possible Solution

Design a generic staged-input or large-argument contract at the agent tool, Gateway protocol, SDK, and security-policy boundaries, while keeping provider-specific empty-argument bugs in their narrower linked issues.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that the requested streaming/file-backed argument contract is absent on current main.

## Solution Assessment

Unclear. Streaming or staged file inputs may be the durable direction, but the best shape needs maintainer product and security design before implementation because it changes public tool/protocol contracts.
