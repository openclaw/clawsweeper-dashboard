## What This Changes

The issue reports that Telegram selected quote context reaches normalized runtime metadata but is omitted from the final model prompt.

## Best Possible Solution

Keep the v2026.5.3/current-main prompt suffix path that appends sanitized current-turn reply/quote context to embedded model input while leaving transcript text clean.

## Reproduction Assessment

Yes for the affected release: the issue gives concrete 2026.5.2 Telegram selected-quote steps, and v2026.5.2 source lacked the current-turn prompt suffix. On current main and v2026.5.3, source and tests show the selected quote body is appended to promptForModel/finalPromptText.

## Solution Assessment

Yes. The shipped fix is narrow: it passes explicit reply context separately from transcript text and appends it only at embedded prompt assembly, which preserves model visibility without making quoted media paths part of prompt-local image loading.
