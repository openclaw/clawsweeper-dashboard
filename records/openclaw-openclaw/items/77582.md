## Summary

Close: current main and the shipped v2026.5.3 release include explicit Telegram reply/quote context in the model-visible prompt; the report was against 2026.5.2 and the release changelog names the same Agents/Telegram prompt fix.

## What This Changes

Ensure Telegram selected quote context that reaches normalized runtime context is included in the final model prompt/finalPromptText.

## Best Possible Solution

Keep the v2026.5.3/current-main prompt-suffix path and ask for a fresh report if a 2026.5.3+ Telegram selected quote still loses context.

## Reproduction Assessment

Yes. The report gives a concrete Telegram selected-quote flow, and source inspection matches the old boundary while showing current main now carries `ReplyToBody` into the final model prompt.

## Solution Assessment

Yes. The current implementation appends sanitized current-turn reply/quote context at the embedded model prompt boundary while keeping quoted text out of prompt-local image loading, which is the narrow maintainable fix.
