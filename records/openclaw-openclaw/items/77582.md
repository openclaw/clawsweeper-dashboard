## Summary

Current main and the shipped v2026.5.3 release already preserve explicit Telegram reply/quote context in the model-visible prompt. The same boundary was tracked in #76419 and fixed by #76659, with current tests asserting the reply context is present in `finalPromptText`.

## What This Changes

Include Telegram selected quote/reply context in the final model-visible prompt instead of dropping it after normalization.

## Best Possible Solution

Keep the shipped structured current-turn reply-context path and ask affected 2026.5.2 users to upgrade to v2026.5.3 or newer.

## Reproduction Assessment

No current-main failure path remains: source inspection shows the selected quote body is appended to `promptForModel` and recorded as `finalPromptText`. The older failure is the same boundary described in #76419 and covered by current regression tests.

## Solution Assessment

Yes. The shipped fix is narrow: it threads explicit reply/quote context as sanitized current-turn metadata at the embedded-runner boundary while keeping generic runtime context and transcript text separate.
