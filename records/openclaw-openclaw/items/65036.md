Keep open: the PR targets a real models.json fallback path, but the current diff is not mergeable because it can serialize resolved nested request secrets, adds a broken test, and lacks after-fix real behavior proof. ## What This Changes

The PR changes omitted-config models.json planning to use the loaded active config instead of runtimeSource for serialization and adds a regression test for that branch.

## Best Possible Solution

Keep models.json serialization source-authoritative for SecretRef-managed fields, then address provider bloat through a focused provider-discovery semantics fix with regression coverage and real setup proof.

## Reproduction Assessment

Yes for the blocking PR defects by source inspection: active secrets runtime stores resolved and source configs separately, nested provider request SecretRefs resolve into runtime strings, and marker restoration only covers provider apiKey plus top-level headers. The added test's missing import and wrong helper arity are also source-reproducible.

## Solution Assessment

No. Switching the serialized config to the loaded runtime config is not the safest fix because it can persist resolved request secrets; the broader bloat report should be solved in provider discovery behavior or with a source-safe projection.
