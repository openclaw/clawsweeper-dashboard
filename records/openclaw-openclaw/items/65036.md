## Summary

Keep open: the PR has not landed on current main, but the proposed fix still has blocking correctness, test-harness, and security-sensitive SecretRef persistence problems.

## What This Changes

The PR changes omitted-config `models.json` planning to use `loadConfig()` for the planned config while preserving `runtimeSource` for secret-marker restoration, and adds a regression test for that branch.

## Best Possible Solution

Revise the branch into a source-authoritative fallback that preserves SecretRef markers across provider `request.*`, or close it manually in favor of a focused provider-discovery fix for the explicit-config bloat path.

## Reproduction Assessment

Yes for the blocking PR defects: static inspection traces `loaded` to the resolved runtime snapshot and shows provider `request.*` SecretRefs are not restored before serialization; the added test also has direct harness-call errors.

## Solution Assessment

No. The one-line fallback is not the safest fix because it can serialize resolved request secrets and does not address the explicit-config provider-discovery bloat path described by the linked report.

Full review comments:
