## Summary

Keep this PR open, but the current patch is not mergeable: it can serialize resolved provider request secrets, the added test does not fit the current harness, the linked report points to a different production bloat path, and external real behavior proof is still missing.

## What This Changes

This PR changes omitted-config `models.json` planning to use the loaded active config instead of the runtime source snapshot and adds a regression test for that branch.

## Best Possible Solution

Keep `models.json` serialization source-authoritative for SecretRef-managed fields, then address provider bloat through scoped provider-discovery behavior or a canonical follow-up.

## Reproduction Assessment

Yes for the blocking PR defects by source inspection: active secrets runtime stores resolved and source configs separately, request SecretRefs resolve into runtime strings, and marker restoration only covers provider `apiKey` plus top-level `headers`. I did not run tests in this read-only review.

## Solution Assessment

No. Using the loaded runtime config as the serialized config is not the safest fix because it can persist resolved request secrets, while the reported bloat path appears to require a provider-discovery behavior decision instead.
