## Summary

Keep this PR open. Current main has not implemented the proposed fallback change, but the patch is not mergeable as-is because it can feed a resolved runtime snapshot into `models.json` generation and the added regression test is statically broken.

## What This Changes

The PR changes omitted-config `resolveModelsConfigInput()` calls to plan `models.json` from the loaded config while keeping `runtimeSource` for secret markers, and adds a regression test for that omitted-config path.

## Best Possible Solution

Revise the PR only if it can keep `models.json` planning source-authoritative across the full provider SecretRef surface, including nested `request.*` credentials; otherwise close it manually in favor of a narrower provider-discovery fix for the bloat path tracked from #65020.

## Reproduction Assessment

Yes for the blocking PR defects: static inspection traces `loaded` to the resolved runtime snapshot and shows request SecretRefs are not restored before serialization, and the added test has direct harness call errors. The linked startup bloat itself is not reproduced by the omitted-config branch because current startup call sites pass explicit config.

## Solution Assessment

No. The proposed one-line fallback change is not the safest fix because it can serialize resolved request secrets and does not address the explicit-config provider discovery path described in the linked report.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep omitted config from writing resolved request secrets:** `src/agents/models-config.ts:100`
  - body: When this branch uses `config: loaded`, `loaded` can be the active resolved runtime snapshot rather than a source config. Runtime SecretRef resolution replaces `models.providers.*.request.*` refs with plaintext, while the `models.json` marker pass only restores provider `apiKey` and top-level `headers`, so an omitted `ensureOpenClawModelsJson()` call can serialize request tokens, headers, or TLS material. Use a true source config for planning or restore markers for the full request surface before changing this path.
  - confidence: 0.88
- **[P1] Fix the added regression test harness call:** `src/agents/models-config.runtime-source-snapshot.test.ts:376-389`
  - body: The new test calls `withTempHome` without importing it in this file, then calls `expectGeneratedProviderApiKey("openai", "OPENAI_API_KEY")` even though the helper requires `(agentDir, providerId, expected)`. As written, the added test will fail during typechecking or runtime instead of validating the fallback behavior.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The diff can persist resolved provider request secrets into `models.json` when the omitted-config branch runs with an active runtime snapshot.

Concerns:

- **[high] Plaintext provider request secrets can be written:** `src/agents/models-config.ts:100`
  - body: Switching the planned config to `loaded` can feed the resolved runtime snapshot into `models.json` generation. Current source-managed enforcement restores provider `apiKey` and top-level `headers` only, so nested `request.auth`, `request.headers`, `request.tls`, and proxy TLS secrets can remain plaintext in the serialized provider config.
