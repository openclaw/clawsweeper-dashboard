## Summary

Keep open: current main still drops bundle activation metadata, and an open linked PR with closing syntax is the right fix path, so this issue should remain open until that PR or an equivalent implementation lands.

## What This Changes

Bundle-format plugin manifests should preserve normalized activation metadata through parsing and registry record construction.

## Best Possible Solution

Land or supersede the linked PR by threading normalized activation through bundle parsing and buildBundleRecord, with focused registry coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows bundle parsing and bundle record construction both omit activation, so a bundle manifest with activation.onStartup:false will produce a registry record without activation; no runtime command was run in this read-only review.

## Solution Assessment

Yes. Preserving activation at parse time and copying it into bundle registry records is the narrowest maintainable fix because downstream consumers already read PluginManifestRecord.activation.
