## Summary

Keep open: current main does not implement `clientContext`, but the PR is not merge-ready because it stores run attribution in mutable session-scoped diagnostic state and expands gateway, SDK, Swift, and plugin-visible telemetry contracts without maintainer-approved run ownership and privacy policy.

## What This Changes

The PR adds optional `clientContext` to gateway agent requests, SDK and Swift models, normalizes it, and forwards it through trusted diagnostic private data from session diagnostic state.

## Best Possible Solution

Make attribution run-owned, preferably by carrying the normalized bag with the admitted run or indexing it by `runId`, and land only after the telemetry API and privacy contract are maintainer-approved.

## Reproduction Assessment

Yes, source-level. Submit two accepted gateway agent requests with the same session and different `clientContext` values before the earlier run's lifecycle finishes; the later seed can overwrite or clear the session slot before `logSessionStateChange` or `logMessageQueued` reads it.

## Solution Assessment

No. The trusted private-data delivery boundary is plausible, but the implementation should bind attribution to the admitted run or `runId` rather than reusable session state.
