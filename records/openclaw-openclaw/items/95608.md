## Summary

Keep this PR open: it contains meaningful feature work, but the latest head still stores run attribution in reusable diagnostic session state, so same-session runs can mis-associate trusted telemetry before merge.

## What This Changes

The PR adds optional gateway, SDK, and Swift `clientContext` input and forwards normalized attribution metadata to trusted diagnostic private data on lifecycle events.

## Best Possible Solution

Bind attribution to the admitted run or runId, add same-session overlap coverage, and land only after maintainers approve the telemetry contract.

## Reproduction Assessment

Yes. From source, the latest PR writes `clientContext` into one mutable session entry before async dispatch, while later `session.state` and `message.queued` events read that same slot.

## Solution Assessment

No. The trusted private-data channel is a plausible boundary, but the implementation should bind attribution to the admitted run rather than reusable session state.
