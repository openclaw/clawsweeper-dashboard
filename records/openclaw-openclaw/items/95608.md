## Label Justifications

- `P2`: This is a normal-priority gateway diagnostics feature with correctness and privacy-boundary blockers but no active production outage.
- `merge-risk: 🚨 compatibility`: The diff expands gateway request schema, TypeScript SDK params, Swift generated models, and plugin-visible diagnostic private data.

- `merge-risk: 🚨 security-boundary`: Caller-supplied upstream identifiers can be forwarded to trusted exporters and trace storage if attribution is mis-associated.

## Summary

Keep open: current main does not implement this gateway attribution feature, and the PR is not merge-ready because it stores run attribution in mutable session-scoped diagnostic state while also expanding API and trusted telemetry boundaries.

## What This Changes

The PR adds an optional opaque `clientContext` field to gateway agent requests and forwards it through SDK/Swift/protocol surfaces into trusted diagnostic private data for run attribution.

## Best Possible Solution

Make attribution run-owned, preferably by carrying the normalized bag with the admitted run or indexing it by `runId`, then land only after the telemetry API and privacy contract are maintainer-approved.

## Reproduction Assessment

Yes. From source, the PR writes caller context into a reusable diagnostic session slot and later lifecycle events read that mutable slot, so two differently attributed same-session requests are enough to exercise the mis-association path.

## Solution Assessment

No. The trusted private-data channel is a plausible delivery boundary, but this implementation should bind attribution to the admitted run or `runId` rather than reusable session state.
