## Summary

Keep open. Current main still has a source-level Discord path where block-streaming-off text-only assistant blocks wait for final delivery while visible tool/result progress can be delivered first; newer Discord progress-preview defaults mitigate some fresh configurations but do not eliminate the reported ordering bug.

## What This Changes

Fix Discord automatic replies so assistant text segments remain in production order when interleaved with tool calls.

## Best Possible Solution

Preserve production order for Discord automatic source delivery by sending or coalescing text blocks and visible tool/progress payloads through one ordered path, while preserving current preview-streaming and message-tool-only behavior.

## Reproduction Assessment

Yes, at source level. Current main still buffers text-only block replies when block streaming is off, while visible tool/result callbacks can run before final text delivery.

## Solution Assessment

No, defaulting Discord block streaming on is not the best fix. The safer fix is a narrow ordering repair for Discord automatic delivery, because the related block-stream fragmentation issue shows that block streaming has separate UX tradeoffs.
