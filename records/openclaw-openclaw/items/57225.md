## Summary

Keep open. Current main and the latest release still default block streaming off and keep text-only block replies buffered until final delivery; the recent block-reply wait fix only waits for block replies that were already sent, so the reported Discord ordering path remains source-reproducible.

## What This Changes

Discord should preserve assistant text and tool-output order for multi-segment responses instead of delivering tool output before all accumulated text.

## Best Possible Solution

Preserve production order for Discord automatic source delivery by routing visible text segments and tool/progress payloads through one ordered delivery path, while leaving block-streaming defaults and explicit-send policy separate.

## Reproduction Assessment

Yes, at source level. Current main still buffers text-only block replies when block streaming is disabled, has a regression test asserting no immediate onBlockReply call, and keeps the recent dispatcher wait limited to already-sent block replies.

## Solution Assessment

Yes for a narrow ordered-delivery repair; no for changing Discord defaults. The maintainable fix should preserve production order in the existing delivery path without flipping global or per-channel streaming policy.
