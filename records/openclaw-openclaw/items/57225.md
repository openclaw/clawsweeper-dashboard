## Summary

Keep open. Current main still has a Discord path where visible tool/progress delivery can happen while text-only assistant blocks are held for final delivery unless block streaming is enabled, matching the reported ordering problem rather than proving it fixed.

## What This Changes

The issue asks for Discord responses to preserve model-produced ordering when assistant text segments are interleaved with tool calls.

## Best Possible Solution

Preserve production order for Discord automatic source delivery by routing text blocks and visible tool/progress payloads through one ordered path, without globally enabling noisy Discord preview/block streaming defaults or changing message-tool-only guild behavior.

## Reproduction Assessment

Yes. The issue has concrete Discord steps, and current source provides a high-confidence mechanism: text-only blocks are withheld when block streaming is off, while Discord can still emit visible tool/progress payloads before the final batched text.

## Solution Assessment

Unclear that the reporter's suggested Discord default change is the best fix. A narrower fix should preserve ordering only for the affected Discord automatic-delivery path and avoid turning on broad streaming defaults that have documented UX and rate-limit tradeoffs.

## Repair Work Prompt

Fix Discord automatic source reply ordering for assistant text interleaved with tool calls. Current main with default Discord/block-streaming-off can deliver visible tool/progress payloads while text-only assistant blocks are held until final delivery, producing tool -> tool -> text ordering. Implement a narrow fix that preserves model production order for Discord automatic source delivery without globally enabling `channels.discord.streaming` preview defaults, without changing non-Discord channels broadly, and without breaking message-tool-only guild behavior. Consider flushing pending text blocks through the existing ordered dispatcher before visible tool/progress delivery, or otherwise queueing text/tool/final payloads in original order. Add regression coverage for a Discord turn with assistant text1 -> tool -> text2 -> tool -> final under default Discord config. Include a user-facing changelog entry if behavior changes. Do not change dependencies, Carbon pins, generated metadata, or unrelated streaming defaults.
