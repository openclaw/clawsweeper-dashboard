## Summary

Keep open. Current main and the latest release still preserve the disabled block-streaming path where text-only assistant segments are held for final delivery while tool/progress summaries can be emitted earlier, so the Discord ordering bug remains source-reproducible.

## What This Changes

The issue requests that Discord preserve the production order of assistant text segments and tool-call output instead of delivering tool output first and batched text afterward.

## Best Possible Solution

Preserve production order for Discord automatic source delivery by routing visible text blocks and tool/progress payloads through one ordered path, while keeping block-streaming defaults and the broader explicit-send policy separate.

## Reproduction Assessment

Yes, at source level. Current main still buffers text-only block replies when block streaming is disabled, has a regression test asserting no immediate onBlockReply call, and assembles tool summaries before accumulated answer text.

## Solution Assessment

Yes for a narrow ordered-delivery repair; no for changing Discord defaults. The maintainable fix should preserve production order in the existing delivery path without flipping global or per-channel streaming policy.
