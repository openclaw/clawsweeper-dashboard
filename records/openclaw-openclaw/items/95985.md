## Summary

Keep open. Current main and v2026.6.9 still refresh and replace same-session bootstrap file snapshots when file content changes, and there is no session-freeze or configurable dynamic-bootstrap-file contract on current source.

## What This Changes

The issue requests a prompt-cache-stable bootstrap contract for long sessions, either by freezing bootstrap files per session, moving volatile bootstrap files behind the cache boundary, or correcting the docs and operator guidance.

## Best Possible Solution

Maintainers should choose one cache-stability contract, then align bootstrap cache behavior, config/schema/docs if needed, and focused tests without adding parallel runtime fallback paths.

## Reproduction Assessment

Unclear. The issue identifies a real cache-stability gap, but choosing session-freeze versus dynamic-file config versus documentation guidance is a maintainer product and compatibility decision.
