## Summary

Keep open. Current main still refreshes bootstrap files from disk and replaces the session cache when file content changes, so the reported session-freeze/cacheWrite problem is not implemented or safely superseded.

## What This Changes

The issue reports that bootstrap-injected workspace files are re-read after mid-session writes, invalidating Anthropic prompt-cache prefixes and causing high cacheWrite costs in long sessions.

## Best Possible Solution

Keep this issue open until maintainers choose one cache-stability contract, then implement it with explicit tests and docs covering live-file visibility, prompt-cache stability, and upgrade-safe config behavior.

## Reproduction Assessment

Yes, at source level: current bootstrap-cache code reloads workspace files and the focused tests expect changed contents to replace the cached result. I did not run the live Anthropic billing reproduction.

## Solution Assessment

Unclear as proposed: real session-freeze and dynamic-file configuration are both plausible, but the best maintainable fix requires a maintainer decision on prompt-cache semantics and compatibility.
