## Summary

Keep open. Current main and latest v2026.6.9 still refresh bootstrap files from disk and replace the per-session cache when file contents change, while only HEARTBEAT.md is classified as dynamic below the Anthropic cache boundary; related #65438 only covers one possible dynamic-file-config remedy and does not settle the session-freeze or docs-contract decision.

## What This Changes

The issue asks OpenClaw to make mid-session bootstrap-file changes stop invalidating Anthropic prompt-cache prefixes, or to document that live-refresh behavior and its cost impact accurately.

## Best Possible Solution

Maintainers should choose one explicit contract: session-frozen bootstrap prompt copies, configurable dynamic-file classification/order, or documented live-refresh cost guidance, then update runtime, config schema/help, docs, and prompt-cache tests consistently.

## Reproduction Assessment

Yes at source level: current main reloads bootstrap files each turn and replaces the cached snapshot on content changes, while most bootstrap files remain stable-prefix prompt context above the Anthropic cache boundary. I did not run a live Anthropic billing reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract; session freeze and dynamic-file config are both plausible but affect live workspace visibility, cache behavior, config surface, and upgrade expectations.
