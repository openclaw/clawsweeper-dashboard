## Summary

Keep open. Current main and latest release still refresh bootstrap files from disk and replace the per-session cache when file content changes, so the reported session-freeze/cacheWrite problem is neither implemented nor superseded.

## What This Changes

The issue asks for a session-stable bootstrap snapshot, configurable dynamic bootstrap files after the cache boundary, or corrected cost/cache documentation for mid-session writes to injected workspace files.

## Best Possible Solution

Keep this issue open until maintainers choose and document the cache-stability contract, then implement that choice across bootstrap cache behavior, config/docs if needed, and focused prompt-cache tests.
