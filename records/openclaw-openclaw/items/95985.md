## Summary

Keep open. Current main and v2026.6.9 still refresh bootstrap files from disk on later turns and replace the per-session snapshot when file content changes, so the reported session-freeze/cacheWrite gap remains unresolved; the related dynamic-file issue is only one possible fix path and does not fully own the session-freeze or documentation-position request.

## What This Changes

Requests a real cache-stable bootstrap-file contract for long sessions, either through session-freeze behavior, configurable dynamic bootstrap files, or corrected docs/operator guidance.

## Best Possible Solution

Maintainers should choose and document one cache-stability contract, then implement it across bootstrap cache behavior, cache-boundary config if needed, tests, and prompt-caching/operator docs.

## Reproduction Assessment

Yes at source level: current `getOrLoadBootstrapFiles` refreshes disk contents and focused tests expect changed contents to replace the cached result. I did not live-reproduce the Anthropic billing ratio in this read-only review.

## Solution Assessment

Unclear. Real session-freeze and dynamic-file configuration are both plausible, but the best maintainable fix requires a maintainer decision on prompt-cache semantics, live bootstrap visibility, and whether a new config surface is justified.
