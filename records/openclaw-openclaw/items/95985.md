## Summary

Keep open. Current main and v2026.6.9 still refresh bootstrap files from disk and replace the session cache when file content changes, so the session-freeze/cacheWrite problem is not implemented or superseded.

## What This Changes

The issue reports that mid-session writes to bootstrap-injected workspace files invalidate Anthropic prompt-cache prefixes and cause high cacheWrite cost in long sessions.

## Best Possible Solution

Keep this issue open until maintainers choose and document the cache-stability contract, then implement that choice across bootstrap cache behavior, config/docs if needed, and focused prompt-cache tests.

## Reproduction Assessment

Yes, at source level: current bootstrap-cache code reloads workspace files and the focused tests expect changed contents to replace the cached result. I did not run the live Anthropic billing reproduction.

## Solution Assessment

Unclear as proposed: real session-freeze and dynamic-file configuration are both plausible, but the best maintainable fix requires a maintainer decision on prompt-cache semantics and compatibility.
