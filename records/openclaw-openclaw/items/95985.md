## Summary

Keep open. Current main and v2026.6.10 still refresh bootstrap files from disk and replace the session snapshot when content changes, so the reported session-freeze/cacheWrite problem remains unresolved and needs maintainer product direction.

## What This Changes

The issue asks OpenClaw to prevent mid-session writes to bootstrap-injected workspace files from invalidating prompt-cache prefixes, or to provide/document an operator-safe dynamic-file/cache-boundary path.

## Best Possible Solution

Maintainers should choose one documented cache-stability contract, then implement it through bootstrap loading, prompt assembly, config/docs if needed, and focused source plus live prompt-cache validation.

## Reproduction Assessment

Yes at source level: current bootstrap-cache code reloads workspace files and tests expect changed contents to replace the cached result. I did not run the live Anthropic billing reproduction, so the cost magnitude still needs live proof.

## Solution Assessment

Unclear as proposed: real session-freeze and dynamic-file configuration are both plausible, but the best maintainable fix requires a maintainer decision on prompt-cache semantics, stale prompt visibility, and compatibility.
