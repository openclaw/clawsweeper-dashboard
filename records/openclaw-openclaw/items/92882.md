## Summary

Current main and shipped releases after v2026.4.23 already include the acpx plugin-owned ACP backend registration path, so the reported missing-backend failure is fixed rather than remaining actionable.

## What This Changes

The issue reports that AutoClaw Desktop v2026.4.23 cannot configure ACP backends because bundled acpx/codex paths fail to register an ACP runtime backend.

## Best Possible Solution

Keep the shipped acpx plugin-owned backend registration path and direct users on v2026.4.23 to upgrade to v2026.4.27 or newer.

## Reproduction Assessment

Yes at source level: ACP dispatch throws the reported backend-not-configured error when the registry has no backend, and current source now registers the acpx backend during plugin startup. I did not run a Windows Desktop repro.

## Solution Assessment

Yes; the implemented fix keeps ACP backend ownership in the acpx plugin while exposing only the generic runtime registry through the plugin SDK boundary.
