## Summary

## Reproduction Assessment

Yes, by source and dependency inspection: current main writes final providers into PI `models.json`, and the pinned PI registry drops all custom models when one non-built-in custom provider with models lacks `apiKey`. I did not run a live gateway reproduction in this read-only sweep.

## Solution Assessment

No. The PR validates raw OpenClaw provider shape before generation, but the reported failure happens later in PI semantic validation of generated `models.json`; the maintainable fix belongs at that boundary with sanitized diagnostics.
