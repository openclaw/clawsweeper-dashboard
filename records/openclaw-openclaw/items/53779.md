## Summary

Keep open. The PR targets a real provider-validation failure, and jw8957's comment confirms a matching 2026.4.10 runtime symptom, but the proposed patch does not cover the reported PI semantic failure path and adds an unsanitized config-derived warning.

## What This Changes

The PR adds a `sanitizeInvalidProviders` pre-validation pass in `src/config/validation.ts` that drops provider entries failing `ModelProviderSchema.safeParse`, emits `console.warn`, and adds focused unit tests for that pre-pass.

## Best Possible Solution

Implement provider isolation at the `models.json` generation or runtime discovery boundary with PI-equivalent semantic validation. Valid providers should preserve baseUrl, auth, headers, models, overrides, metadata, and capabilities; invalid providers should be skipped with sanitized diagnostics and regression coverage for mixed, all-invalid, missing-apiKey, invalid API enum, and generated/merged provider cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: put one valid custom provider with a baseUrl and one non-built-in provider with custom `models` but no `apiKey` into `models.json`; OpenClaw's schema allows that shape, then PI rejects the whole custom models file and returns empty custom overrides.

## Solution Assessment

No. The PR validates too early and against a weaker schema than the failing PI semantic contract; the safer fix is to isolate or sanitize providers where OpenClaw writes or loads the PI `models.json`, and to route warnings through the existing sanitized config diagnostics path.

## Review Findings

Full review comments:

- **[P2] Recover at the models.json boundary:** `src/config/validation.ts:251`
  - body: This pre-pass only calls `ModelProviderSchema.safeParse`, where `apiKey` is optional, and it runs before generated or merged providers are loaded from `models.json`. A non-built-in provider with custom `models` but no `apiKey` still reaches PI, which rejects the whole file and drops unrelated valid overrides.
  - confidence: 0.9

  - confidence: 0.88
