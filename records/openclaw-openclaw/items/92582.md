## Summary

Keep this open: current main and the latest release still contain the local-provider skipped-probe false-positive path, and three open linked PRs now claim to fix it. The issue should stay open until one canonical fix PR lands.

## What This Changes

The issue reports that `openclaw doctor` warns local memory embeddings are not confirmed ready even when `openclaw memory status --deep` reports the local memory stack is healthy.

## Best Possible Solution

Land one focused canonical fix with regression coverage that treats an intentionally skipped local embedding probe as neutral while preserving warnings for checked not-ready results and transport timeouts.

## Reproduction Assessment

Yes. Source inspection shows current `main` can pass `{ checked: false, ready: false, skipped: true }` into `noteMemorySearchHealth`, and the `local` branch still warns unless the probe is checked and ready.

## Solution Assessment

Yes for the central false warning: the narrow maintainable fix is to mirror the existing skipped-probe guard from sibling key-optional providers in the local-provider branch. Whether `doctor --deep` should also force a live embedding probe can be decided while selecting the canonical PR.
