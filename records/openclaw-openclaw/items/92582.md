The issue asks `openclaw doctor` to stop warning that local memory embeddings are not ready when `openclaw memory status --deep` shows the local memory stack is healthy.

## Best Possible Solution

Keep this issue open until a PR lands that makes `doctor --deep` run the memory embedding readiness probe and prevents intentionally skipped default probes from appearing as fix-required local-memory failures.
