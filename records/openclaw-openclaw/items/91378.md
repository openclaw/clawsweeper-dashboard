- **[P1] Inherit parent session options before compacting:** `src/cli/program/register.status-health-sessions.ts:410-416`
  - body: Every other `sessions` subcommand that can target an agent merges `command.parent?.opts()`, but this new action forwards only child options. `openclaw sessions --agent work compact global` would send no `agentId`, so the gateway resolves the default global session and can compact the wrong agent; parent `--json` is ignored too. Accept the `command` arg, merge parent `agent`/`json`, and add regression coverage.
  - confidence: 0.93
- **[P2] Report pending Codex compaction accurately:** `src/commands/sessions-compact.ts:42-44`
  - body: Current main tests lock a `sessions.compact` success shape where Codex native compaction starts asynchronously as `ok:true, compacted:false` with pending details. This formatter treats every non-compacted success as `No compaction needed`, so the new CLI can tell users nothing happened even though a native compact start was accepted. Detect that pending/details shape or print a neutral status instead.
  - confidence: 0.82
