## What This Changes

Adds a documented `openclaw sessions compact <key>` CLI wrapper for the existing `sessions.compact` RPC, rejects CLI `/compact` messages with exit 1, and guards maxLines compaction before transcript truncation.

## Best Possible Solution

Land this PR as the canonical fix if maintainers accept the exit-1 CLI redirect and active-run interruption semantics; otherwise choose an alternate CLI policy before merge.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 have the `sessions.compact` RPC but no `openclaw sessions compact` CLI/docs surface, and current CLI `/compact` only blocks timeout fallback rather than failing before dispatch.

## Solution Assessment

Yes. A thin wrapper over the existing Gateway RPC plus early rejection of unsupported CLI slash-command use is the narrow owner-boundary fix; bypassing chat slash-command authorization through `agent --message` would be riskier.
