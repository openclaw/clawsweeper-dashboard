## Summary

Keep open: current main does not implement Feishu recall handling, and this PR is useful but incomplete because recall-before-reply suppression and media reply cleanup are still missing. The branch is also dirty against current main and needs normal repair before maintainer review.

## What This Changes

The PR adds a Feishu recalled-message event handler, an in-memory user-to-bot reply map, reply-path send metadata, and text/card reply registration so recalled user messages can delete related bot replies.

## Best Possible Solution

Finish this as a Feishu plugin feature with account-aware recall mapping, pre-send suppression for recalled reply targets, text/card/media reply cleanup, regression tests, setup docs, and a changelog entry.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live bug reproduction; this is a feature PR. Source inspection does show current main lacks any recalled-message handler or recall mapper, and the proposed patch can be reviewed against the active Feishu reply paths.

## Solution Assessment

Is this the best way to solve the issue? No. Keeping the work inside the Feishu plugin is the right boundary, but the submitted patch must also wire the pre-reply recall check, register media reply IDs, and update docs/changelog before it is the narrow maintainable fix.
