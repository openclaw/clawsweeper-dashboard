## What This Changes

The PR adds Feishu `im.message.recalled_v1` handling, an in-memory user-message to bot-reply map, bot-reply deletion on recall, and a `viaReplyPath` send-result flag.

## Best Possible Solution

Implement Feishu inbound recall support in the plugin with private reply tracking, receipt-preserving sends, pre-send deleted/recalled checks, deleted-state handling, focused tests, and redacted live Feishu proof before merge.

## Reproduction Assessment

Yes for the review blockers by source inspection: the PR removes the current receipt return, sends before any recall preflight, and ignores `data.message.deleted`. No live Feishu after-fix reproduction proof is attached.

## Solution Assessment

No. The feature belongs in the Feishu plugin, but this patch should preserve current send receipts, check deleted/recalled state before fallback delivery, handle the SDK deleted field, and include live Feishu proof.
