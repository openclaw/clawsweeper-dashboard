## Summary

This should stay open: the bug is still present on current main, and the linked fixing PR is open and uses closing syntax, so the issue should remain the canonical tracker until that PR lands or is closed.

## What This Changes

The issue reports that the update CLI plugin payload smoke check can emit both `missing-extension-entry` and `missing-main-entry` for the same malformed plugin because the empty/invalid extension branch falls through to the main check.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84284 after targeted validation, then let the merge close this issue.

## Reproduction Assessment

Yes. Source inspection shows a plugin manifest with empty or invalid `openclaw.extensions` plus a declared missing `main` file takes both failure paths on current main.

## Solution Assessment

Yes. Adding a `continue` after the terminal empty/invalid extension-entry failure, with a focused regression test, is the narrowest maintainable fix while preserving the valid-extension `main` check.
