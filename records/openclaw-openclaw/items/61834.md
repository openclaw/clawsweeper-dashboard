## Summary

Keep open. Current main still lacks a QMD no-rerank config or request surface, and the existing direct CLI and mcporter/MCP query paths do not pass QMD's upstream no-rerank controls.

## What This Changes

Expose a QMD rerank toggle for `memory.qmd.searchMode = "query"`, either as `memory.qmd.noRerank` or as a `memory_search` request parameter.

## Best Possible Solution

Add an opt-in `memory.qmd.noRerank` boolean for QMD `query` mode, pass it as `--no-rerank` for direct CLI and `rerank: false` for the unified MCP query call, and defer per-request control to a separate product decision.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict config schema rejects the proposed key, the memory search tool schema has no rerank field, and both direct QMD and MCP query builders omit the upstream controls.

## Solution Assessment

Yes for the config-level path. It is the narrowest maintainable implementation because it stays within the existing QMD backend contract while avoiding a new model-controlled request parameter.
