## Summary

Keep open. Current main still lacks a QMD no-rerank config or request surface, while upstream QMD v2.1.0 explicitly added `qmd query --no-rerank` and `rerank: false` for the MCP query tool.

## What This Changes

Expose QMD's no-rerank capability through OpenClaw's QMD memory query mode.

## Best Possible Solution

Add an opt-in `memory.qmd.noRerank` boolean for QMD `query` mode, pass it as `--no-rerank` for direct CLI queries and `rerank: false` for the unified MCP query call, and defer request-level control to a separate decision.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict config schema rejects `memory.qmd.noRerank`, the memory search tool schema has no `rerank` field, and both QMD query builders omit upstream QMD's no-rerank controls.

## Solution Assessment

Yes for the config-level path. It is the narrowest maintainable solution because it follows the existing QMD backend config contract while avoiding a new model-controlled request parameter.
