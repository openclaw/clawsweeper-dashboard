## Summary

Keep open. Current main and the latest release still do not expose QMD 2.1.0 no-rerank through `memory.qmd` config or the `memory_search` request schema, and the related remote-reranker request is broader rather than a superseding fix.

## What This Changes

Expose a QMD no-rerank toggle so `memory.qmd.searchMode: "query"` can skip QMD reranking.

## Best Possible Solution

Add an opt-in `memory.qmd.noRerank` boolean for QMD `query` mode, pass it as `--no-rerank` in direct CLI mode and `rerank: false` for the unified MCP query call, and treat request-level control as a separate maintainer decision.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main lacks the proposed QMD config field, the `memory_search` schema has no `rerank` parameter, and both QMD query runtime paths omit upstream no-rerank controls.

## Solution Assessment

Yes for the config-level path, unclear for request-level control. A QMD-only config key is the narrowest maintainable surface, while a model-controlled `memory_search.rerank` argument needs a maintainer product decision.
