## Summary

Keep open. Current main still has no QMD no-rerank config or `memory_search` request surface, and the related remote-reranker request at https://github.com/openclaw/openclaw/issues/64438 is broader rather than a superseding fix for this narrow QMD toggle.

## What This Changes

Expose an opt-in way for QMD `query` search mode to skip reranking, either through QMD backend config or a request-level `memory_search` parameter.

## Best Possible Solution

Add an opt-in `memory.qmd.noRerank` boolean for QMD `query` mode, pass it as `--no-rerank` for direct CLI queries and `rerank: false` for the unified MCP query call, and treat request-level control as a separate product decision.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main rejects the proposed `memory.qmd.noRerank` key, the `memory_search` schema has no `rerank` field, and both QMD runtime query paths omit upstream no-rerank controls.

## Solution Assessment

Yes for the config-level path. A QMD-only `memory.qmd.noRerank` option is the narrowest maintainable implementation because it follows the existing backend config boundary without adding a model-controlled request parameter.
