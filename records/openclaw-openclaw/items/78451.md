## Best Possible Solution

Review, repair if needed, and land https://github.com/openclaw/openclaw/pull/78459 or an equivalent narrow fix that removes the forced highspeed path without adding a new config surface.

## Solution Assessment

Yes for the issue direction: the best fix is to stop commitments extraction from forcing fast mode rather than adding a new commitments config field. The linked PR is the current candidate, but maintainers should ensure it preserves explicit configured fast-mode opt-ins before merge.
