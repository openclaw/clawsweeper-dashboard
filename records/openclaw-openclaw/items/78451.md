## What This Changes

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78459 with a narrow commitments runtime fix that removes the unconditional fast-mode flag while preserving explicit fast-mode opt-ins, then close this issue after merge.

## Reproduction Assessment

Yes, at source level: current main passes `fastMode: true` from hidden commitment extraction, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when fast mode is true. I did not run a live MiniMax OAuth account in this read-only review.

## Solution Assessment

Yes, a narrow commitments runtime fix is the right boundary. The safest fix should replace the unconditional fast-mode flag with resolved/default-off fast-mode behavior and regression coverage rather than adding a new config option for this bug.
