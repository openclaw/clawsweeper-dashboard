## Summary

Keep open. Current `main` and latest release still force hidden commitment extraction through MiniMax fast mode, and the open linked PR is the active fix candidate, so this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue reports commitment extraction forcing `MiniMax-M2.7-highspeed` for MiniMax users whose plan only supports `MiniMax-M2.7`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78459 with a fix that stops hidden commitment extraction from forcing highspeed while preserving explicit fast-mode opt-ins.

## Reproduction Assessment

Yes, at source level. Current main passes `fastMode: true` in the commitment extractor, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when that flag is true; I did not run a live MiniMax OAuth account in this read-only review.

## Solution Assessment

Yes, the best fix belongs in the commitment extraction request path, not in a new commitments-specific config field. The linked PR or an equivalent replacement should remove the unconditional true while preserving the shared fast-mode behavior for explicit opt-ins.
