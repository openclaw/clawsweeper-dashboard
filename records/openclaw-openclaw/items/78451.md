## Summary

Keep open. Current main and the latest release still force `fastMode: true` for hidden commitment extraction, and the open closing PR needs review or repair before this issue can close.

## What This Changes

Commitment extraction should stop forcing MiniMax highspeed mode when the configured plan only supports `MiniMax-M2.7`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78459 with a resolver-based fix that defaults off while preserving explicit fast-mode opt-ins.

## Reproduction Assessment

Yes, at source level: current main sends `fastMode: true` from hidden commitment extraction, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when fast mode is true. I did not run a live MiniMax OAuth account in this read-only review.

## Solution Assessment

Yes, fixing the commitments extraction call site is the right layer. The best implementation should use the shared resolved fast-mode state rather than adding a new commitments-specific config knob or leaving the field hardcoded.
