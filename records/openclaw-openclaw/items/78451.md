## Summary

Keep open. Current main and the latest release still force `fastMode: true` for hidden commitment extraction, which reaches the MiniMax highspeed rewrite path; the open linked PR is the implementation candidate and should be reviewed or replaced before this issue closes.

## What This Changes

Fix commitment extraction so it does not force `MiniMax-M2.7-highspeed` via hardcoded fast mode on MiniMax plans that only support the base M2.7 model.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78459 with a fix that uses the shared resolved fast-mode state and covers default-off plus explicit opt-in behavior.

## Reproduction Assessment

Yes, at source level: current main passes `fastMode: true` in hidden commitment extraction, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when that flag is true. I did not run a live MiniMax OAuth account in this read-only review.

## Solution Assessment

Yes, the best fix is in commitment extraction: use the shared resolved fast-mode state instead of hardcoding true or adding a commitments-specific config knob. The issue should stay open until the linked PR or an equivalent fix lands.
