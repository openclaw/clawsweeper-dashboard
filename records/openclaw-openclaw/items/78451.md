## Summary

Keep open. Current main still forces `fastMode: true` for hidden commitment extraction, and the MiniMax stream wrapper still rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed`; the linked closing PR remains open, so this issue should stay open until that fix or an equivalent one lands.

## What This Changes

Commitment extraction should stop unconditionally requesting MiniMax fast mode on non-highspeed MiniMax plans.

## Best Possible Solution

Land the linked fix PR or an equivalent patch that passes the existing resolved fast-mode state into commitment extraction, with regression coverage for default-false MiniMax extraction.

## Reproduction Assessment

Yes. Source inspection on current main shows hidden commitment extraction always passes `fastMode: true`, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when that flag is true.

## Solution Assessment

Yes for the direction, but the maintainable implementation should use the existing fast-mode resolver rather than add a commitments-specific config knob or silently drop per-agent fast-mode defaults.
