Keep open. Current main still forces `fastMode: true` for hidden commitment extraction, MiniMax fast mode rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed`, and the linked closing PR is still open with review/proof blockers.

## What This Changes

Stop hidden commitment extraction from forcing MiniMax highspeed mode on non-highspeed MiniMax plans.

## Best Possible Solution

Update commitment extraction to pass the resolved fast-mode state for the selected agent/model, add regression coverage for default-false and configured fast-mode cases, then land the corrected linked PR.

Yes, by source inspection: current main hardcodes `fastMode: true` in hidden commitment extraction, and the MiniMax stream wrapper rewrites `MiniMax-M2.7` to highspeed only when fast mode is true. The reporter also provided live failure logs, but I did not run a MiniMax account.

## Solution Assessment

Yes, the best direction is a narrow runtime fix that uses the existing fast-mode resolver instead of adding a commitments-specific config knob. The issue should remain open while the linked closing PR is reviewed and corrected.
