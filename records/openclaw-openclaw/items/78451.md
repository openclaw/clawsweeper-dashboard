Keep open. Current main and the latest release still force hidden commitment extraction through MiniMax fast mode, and the linked closing PR remains open as the active fix candidate.

## What This Changes

The issue reports hidden commitment extraction forcing MiniMax M2.7 into the highspeed variant on MiniMax plans that only support the base model.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78459 with a fix that stops hidden commitment extraction from forcing highspeed while preserving explicit fast-mode opt-ins.

## Reproduction Assessment

Yes, at source level. Current main passes `fastMode: true` in the commitment extractor, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when that flag is true.

## Solution Assessment

Yes, the best fix belongs in the commitment extraction request path rather than a new commitments-specific config field. The fix should stop the unconditional true while preserving shared fast-mode opt-ins.
