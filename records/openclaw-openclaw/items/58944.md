## Summary

Keep open: current `main` still lacks a per-call synchronous exec opt-in, but this PR is not merge-ready because it overloads the documented `background:false` default, is conflicting against current schema/description ownership, and still lacks real behavior proof.

## What This Changes

The PR changes `exec` so explicit `background:false` disables the yield timer, updates tool metadata, and adds a unit test for synchronous completion with a low background delay.

## Best Possible Solution

Land an additive foreground/synchronous opt-in, or explicitly approve and document a `background:false` migration with current schema, docs, prompt tests, upgrade evidence, and real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main treats only `background:true` specially and still creates the normal yield timer for explicit `background:false`. I did not run a live command in this read-only review.

## Solution Assessment

No, not as submitted. The runtime idea is narrow, but overloading a documented boolean default is compatibility-sensitive; an additive opt-in or explicit maintainer migration decision is safer.
