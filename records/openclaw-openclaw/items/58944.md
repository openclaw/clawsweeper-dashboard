The PR adds `background: false` as a per-call exec mode that disables auto-backgrounding, updates exec tool/schema text, and adds a regression test for low `backgroundMs`.

Land a rebased implementation in the current exec runtime, schema, description, tests, and public docs that preserves `background:true` and default timer behavior while adding the explicit synchronous `background:false` contract.
