The PR teaches `exec` to interpret `background: false` as a synchronous override, updates tool/schema text, and adds a regression test for completion with a low `backgroundMs`.

## Best Possible Solution

Land a rebased version that implements the `background: false` null-yield branch in current `main`, updates the split schema/description and exec docs, preserves existing `background: true` and default timer behavior, and adds the required changelog entry.

Yes. The semantic approach is the narrow maintainable fix, but the submitted branch is not the best final form because it targets stale module locations and misses release-note/docs updates.
