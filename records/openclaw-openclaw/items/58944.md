## Summary

Keep open: the feature is useful and not implemented on current main, but this PR is not merge-ready because it changes the meaning of the documented default `background:false`, is stale against the current schema/description split, and still lacks contributor real-behavior proof.

## What This Changes

Adds `exec` support for treating `background:false` as forced synchronous execution, updates schema/tool text, and adds a unit test for completion with a low background timer.

## Best Possible Solution

Land an additive, contract-aligned foreground/synchronous opt-in or an explicit maintainer-approved `background:false` migration that updates current runtime, schema, descriptions, public docs, prompt snapshots, and tests with real behavior proof.

## Reproduction Assessment

Yes, source-reproducible: current main treats only `background:true` specially and still arms the normal auto-background timer for `background:false`. I did not run a live command because this is a read-only review.

## Solution Assessment

No, not as submitted. The runtime idea is narrow, but overloading the documented default `false` is compatibility-sensitive and the branch must be rebased to current schema, description, docs, and test owners before merge.
