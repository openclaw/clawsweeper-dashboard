## Best Possible Solution

Yes, by source inspection: current main treats only `background:true` specially, so explicit `background:false` still uses the normal yield timer. This is a feature/API request rather than a broken documented behavior.

## Solution Assessment

No, not as submitted. The runtime idea is narrow, but overloading a documented boolean default is compatibility-sensitive; an additive opt-in or explicit maintainer migration decision is safer.
