## Summary

Keep open: the feature is useful and source-reproducible, but this PR still changes explicit `background:false` semantics, targets stale schema/description locations on current main, and lacks required real behavior proof.

## What This Changes

The PR changes `exec` so `background:false` disables the yield timer, updates tool metadata, and adds a unit test for forced synchronous completion.

## Best Possible Solution

Land an additive foreground/synchronous opt-in, or explicitly approve and document the `background:false` migration with current schema, description, docs, prompt tests, and real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main treats only `background:true` specially and still arms the normal auto-background timer for explicit `background:false`; I did not run a live command in this read-only review.

## Solution Assessment

No, not as submitted. The runtime idea is narrow, but overloading the existing boolean default is compatibility-sensitive and the branch must be rebased to current contract owners before merge.
