## Summary

Keep open. Current main still lacks an explicit `background:false` synchronous exec mode, and the PR's core runtime/test direction is useful, but the branch needs a rebase into the current schema/description/docs layout plus real behavior proof before merge.

## What This Changes

The PR makes exec `background:false` disable the yield timer, updates exec metadata text, and adds a regression test for synchronous completion.

## Best Possible Solution

Rebase and land a contract-aligned implementation that updates the current runtime, schema, description, public docs, and focused tests while preserving existing `background:true` and default timer behavior.

## Reproduction Assessment

Yes, at source level: on current main, `background:false` is not distinguished from the default path and still produces a non-null `yieldWindow` that can arm the timer. I did not run a live exec command in this read-only review.

## Solution Assessment

No, not in the submitted branch as-is. The runtime idea is narrow, but the branch must move metadata to the current owner modules, update public docs, and provide real behavior proof before it is the best mergeable solution.
