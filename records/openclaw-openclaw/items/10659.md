## Summary

Keep open: current main has partial SecretRef, runtime snapshot, host exec env, and redaction hardening, but it still lacks the requested native masked-secret boundary for agent-executed commands and raw credential isolation.

## What This Changes

Request a native masked-secrets system that lets agents use secrets by reference while preventing raw API-key readback, command-output leaks, and env/file exposure.

## Best Possible Solution

Ship or explicitly supersede a native credential boundary that keeps raw values out of agent-readable files, env, and context, performs dispatch-time credential injection, defines exact redaction semantics, and documents the plugin/API contract for external providers.

## Reproduction Assessment

Yes, for the missing requested surface: current-main source inspection shows no `{{secret:NAME}}` support, no `secrets.mask` or `maskedSecrets` contract, and no general policy preventing agents from reading otherwise-readable env files or variables.

## Solution Assessment

No, not as a narrow automatic fix. The proposal is directionally valid, but placeholder parsing, redaction semantics, command denial, credential brokering, and plugin/provider integration need secops and product agreement on the trust boundary first.
