## Summary

Keep open. Current main has meaningful SecretRef, runtime snapshot, redaction, host exec environment, and plugin-hook hardening, but it still does not provide the native masked-secrets boundary requested here; the most relevant implementation PRs are closed unmerged, while https://github.com/openclaw/openclaw/issues/11829 and https://github.com/openclaw/openclaw/issues/59165 remain broader open design threads.

## What This Changes

Add a native masked-secrets system so agents can use API credentials without reading raw values, with placeholder substitution, output redaction, and command/file-read controls.

## Best Possible Solution

Keep this open until OpenClaw ships or explicitly supersedes a native credential-use boundary for agent-executed commands and process output, with raw values kept out of agent-readable files/env/context and clear injection/redaction semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/security architecture request. Source inspection gives high confidence that the requested native masked-secret boundary is still absent on current main.

## Solution Assessment

No for automatic implementation. The direction is valid, but the maintainable path is a maintainer/secops-approved design that decides which pieces belong in core SecretRef, a Credential Provider API, external auth-proxy docs, or plugin hooks before code lands.
