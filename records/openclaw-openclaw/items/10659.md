## Summary

Keep this open: current main and the latest release provide SecretRefs, redaction guidance, and plugin/env hooks, but they still do not implement the requested native masked-secret boundary for arbitrary agent-executed commands or agent-readable plaintext files.

## What This Changes

The issue requests a native masked-secret system that substitutes configured secrets at execution time, redacts outputs, and prevents agents from reading raw API keys through env or file access.

## Best Possible Solution

Keep this issue open until maintainers either ship a core/security-approved credential-isolation design for agent-executed commands or explicitly supersede it with a documented supported boundary.

## Reproduction Assessment

Yes, at source level: current docs explicitly say plaintext credentials remain supported and SecretRefs are not a process-isolation boundary, while the strict schema lacks the requested mask-list and template substitution contract.

## Solution Assessment

No, the current SecretRef and plugin-hook path is only a partial mitigation; the best fix needs maintainer/security direction on the real credential boundary before implementation.
