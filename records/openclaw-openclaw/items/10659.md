## Summary

Keep open: current main and v2026.6.8 ship substantial SecretRef, audit, and redaction mitigations, but they still do not provide the native masked-secret boundary requested for arbitrary agent-executed commands and agent-readable plaintext files.

## What This Changes

The issue requests a native masked-secrets system so agents can use credentials without seeing raw API keys, including placeholder substitution, output redaction, blocked secret reads, and mask-list config.

## Best Possible Solution

Keep this issue open until maintainers ship or explicitly supersede a native credential-isolation or masked-secret design with clear security ownership, migration guidance, and plugin/API boundaries.

## Solution Assessment

No single proposed mechanism is proven as the best final design yet; the current SecretRef implementation is a strong partial mitigation, but the final fix needs maintainer/security product direction on the credential boundary.
