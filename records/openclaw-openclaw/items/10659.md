## Summary

Keep open: main and v2026.6.8 have strong SecretRef, audit, redaction, and plugin-hook mitigations, but they still do not provide the native masked-secret boundary requested for arbitrary agent-executed commands and agent-readable plaintext files.

## What This Changes

The issue requests native masked secrets so agents can use API keys without reading raw values, including placeholder substitution, output redaction, blocked secret-reading commands, and a mask-list config surface.

## Best Possible Solution

Keep this issue open until maintainers ship or explicitly supersede a native credential-isolation or masked-secret design with security owner approval, migration guidance, and plugin/API boundaries.

## Reproduction Assessment

Yes, at source level: current docs explicitly say readable plaintext credentials remain agent-readable, and source search shows no requested placeholder or mask-list contract.

## Solution Assessment

No single proposed mechanism is proven as the best final design yet; SecretRefs and plugin hooks are strong mitigations, but the remaining boundary needs maintainer/security product direction.
