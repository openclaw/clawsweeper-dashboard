## Summary

Keep open: current main and the latest release include partial SecretRef, runtime snapshot, command-scoped resolution, environment sanitization, and redaction hardening, but the native masked-secret boundary requested here remains unimplemented and recent maintainer-associated review explicitly kept it open.

## What This Changes

Requests a native masked-secrets system so agents can use API keys without seeing raw values, including placeholder substitution, output redaction, command blocking, and a configured mask list.

## Best Possible Solution

Keep this open until maintainers ship or explicitly supersede a core credential boundary that keeps raw values out of agent-readable storage, environment, and context while defining dispatch-time injection, exact redaction semantics, and plugin/API integration.
