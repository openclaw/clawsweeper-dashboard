## Summary

## What This Changes

## Best Possible Solution

Keep this open until maintainers ship or explicitly supersede a native credential boundary for agent-executed commands and process output, with raw values kept out of agent-readable files/env/context and clear plugin/SDK integration points.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it asks OpenClaw to define how credentials are isolated from agents, tools, provider auth, and persisted output.

Concerns:

- **[high] Define the credential trust boundary first:**
  - body: A native masked-secret design must choose whether credentials are isolated by a separate process or proxy, plugin provider API, command-scoped injection, redaction, or a combination; that choice affects auth, providers, tool execution, transcripts, and user expectations.
