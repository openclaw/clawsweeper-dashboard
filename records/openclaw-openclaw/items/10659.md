## Summary

Keep this open. Current main has useful SecretRef, runtime snapshot, host exec environment sanitization, redaction, and plugin-hook mitigation surfaces, but it still does not implement the native masked-secrets boundary requested here.

## What This Changes

The issue requests a native masked-secrets system so agents can use API keys without being able to read or leak raw credential values.

## Best Possible Solution

Keep this open until OpenClaw ships or explicitly supersedes a security-owned credential boundary for agent-executed commands and process output, with raw values kept out of agent-readable context and clear plugin hooks for external vault integrations.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested native boundary is absent: there is no `{{secret:NAME}}` substitution, no mask-list schema, and the exec path still runs commands with an assembled process environment.

## Solution Assessment

Unclear that the exact placeholder-plus-blocked-command proposal is the final best design. The safer maintainable direction is a security-owned credential broker or dispatch-time injection boundary, with redaction at tool, log, transcript, and message sinks.
