## Summary

Keep open: current main has meaningful SecretRef, runtime snapshot, redaction, and plugin-hook mitigations, but it still does not implement the native masked-secrets boundary requested here for arbitrary agent-executed commands and agent-readable files.

## What This Changes

The issue requests a native masked-secrets system that lets agents use API credentials without reading raw values, with substitution, output redaction, blocked reads, and config support.

## Best Possible Solution

Keep this issue open until maintainers either ship or explicitly supersede a native credential-boundary design for agent-executed commands, readable secret files, and secret-bearing output.

## Reproduction Assessment

Yes. Current docs explicitly state that SecretRefs do not make arbitrary readable files safe, and current exec code still inherits `process.env` rather than enforcing a native masked-secret boundary.

## Solution Assessment

Unclear. The requested direction is aligned with the security goal, but the exact implementation needs maintainer security/product design because it spans core exec, config, SecretRef, redaction, and plugin boundaries.
