- `P1`: The issue describes real credential exposure and prompt-injection risk across command and auth-provider workflows, even though the final fix requires security/product design.
- `impact:security`: The request is about preventing raw API keys and tokens from appearing in agent-readable files, environments, command output, transcripts, or prompt context.
- `impact:auth-provider`: The affected surface includes SecretRef resolution, provider credentials, API keys, channel tokens, and future credential-provider routing.

## Summary

Keep open: current main has SecretRef, runtime-snapshot, command-path resolution, redaction, and plugin-hook mitigation surfaces, but it still does not provide the native masked-secret boundary requested for arbitrary agent-executed commands; the linked Linux-isolation guide, credential-vault plugin, closed implementation PRs, open security roadmap, and open credential-provider RFC are related context rather than a shipped core solution.

## What This Changes

Request a native masked-secrets system that lets agents use API keys without seeing raw values, including placeholder substitution, output redaction, command/file-read protections, and a mask-list config contract.

## Best Possible Solution

Keep this open until maintainers either ship or explicitly supersede a native credential-use boundary that keeps raw credential values out of agent-readable files, environments, prompt context, transcripts, and command output while aligning with SecretRef, provider auth, and plugin hooks.

## Reproduction Assessment

Yes, by source inspection: current main has no `{{secret:NAME}}` parser, no `maskedSecrets` or `secrets.mask` schema, and no exec-time substitution boundary for arbitrary commands. This is a new security capability rather than a regression in an already-defined contract.

## Solution Assessment

Unclear for the exact proposed placeholder and command-blocking shape. The security direction is valid, but the maintainable solution needs a secops-approved trust model rather than a command blacklist or a second credential system that drifts from SecretRef/provider auth.
