## Summary

Keep open. Current main and the latest shipped release have meaningful SecretRef, runtime-snapshot, env-sanitization, plugin-hook, and redaction hardening, but they still do not provide the native masked-secret boundary requested for arbitrary agent-executed commands.

## What This Changes

The issue requests a native masked-secrets mechanism so agents can use credentials without seeing raw API keys.

## Best Possible Solution

Keep this open until maintainers either ship or explicitly supersede a native credential-use boundary for agent-executed commands, with clear policy for dispatch-time credential injection, raw-value isolation, exact output redaction, and plugin/provider integration.

## Reproduction Assessment

The requested behavior is a new security capability rather than a broken existing contract.

## Solution Assessment

Unclear for the exact proposed placeholder and blocked-command shape. The security direction is valid, but the maintainable fix needs a maintainer-approved trust model that coordinates SecretRefs, credential-provider lifecycle, command/tool injection, redaction, and upgrade compatibility.
