## Summary

Keep open: current main and the latest release have strong SecretRef, redaction, and plugin-hook mitigations, but they still explicitly leave plaintext credentials readable to agents when those files are readable and do not implement the native masked-secret boundary requested here.

## What This Changes

Request a native masked-secrets system so agents can use API keys without seeing raw values, including substitution, output redaction, command blocking, and a mask-list config contract.

## Best Possible Solution

Keep this issue open until maintainers ship or explicitly supersede a native credential-isolation or masked-secret design with security-owner approval, migration guidance, and clear plugin/API boundaries.
