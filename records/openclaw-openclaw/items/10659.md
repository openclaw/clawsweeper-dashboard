## Summary

Keep open: current main and the latest release have strong SecretRef, audit, redaction, and plugin-hook mitigations, but they still explicitly support agent-readable plaintext credentials and do not implement the native masked-secret boundary requested here.

## What This Changes

Request a native masked-secret system so agents can use credentials without seeing raw API keys, including placeholder substitution, output redaction, command blocking, and a mask-list config.

## Best Possible Solution

Keep this issue open until maintainers ship or explicitly supersede a native credential-isolation or masked-secret design with security-owner approval, migration guidance, and clear plugin/API boundaries.
