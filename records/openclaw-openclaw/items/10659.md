## Summary

Keep open: current main has meaningful SecretRef, runtime snapshot, command-scoped resolution, host exec environment, and redaction hardening, but it still does not provide the requested native masked-secret boundary for arbitrary agent-executed commands or raw credential isolation.

## What This Changes

The issue requests a native masked-secrets system so agents can use API keys without seeing raw values, including placeholder substitution, output redaction, blocked raw-secret reads, and a mask-list config surface.

## Best Possible Solution

Keep this open until maintainers ship or explicitly supersede a core credential boundary that keeps raw values out of agent-readable storage/context while defining dispatch-time injection, exact redaction semantics, and plugin/API integration.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/security architecture request. The missing surface is high-confidence from current-main source inspection: no `{{secret:NAME}}`, `secrets.mask`, or `maskedSecrets` contract exists.

## Solution Assessment

No, not as a narrow automatic fix. The direction is valid, but placeholder syntax, command denial, output redaction, credential brokering, and plugin/provider boundaries need secops and product agreement first.
