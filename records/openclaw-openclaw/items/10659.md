## Summary

Keep open: current main has meaningful SecretRef, runtime snapshot, host exec environment, and redaction hardening, but it still lacks the requested native masked-secret boundary for arbitrary agent-executed commands and raw credential isolation.

## What This Changes

Requests a native masked-secrets system that lets agents use configured API keys without exposing raw values through files, environment variables, command output, or chat.

## Best Possible Solution

Maintainers should choose and ship, or explicitly supersede, a core credential boundary that keeps raw values out of agent-readable storage and context while defining dispatch-time injection, exact redaction semantics, and plugin/API integration.

Not applicable as a bug reproduction because this is a feature/security architecture request. The missing surface is high-confidence from current-main source inspection: no `{{secret:NAME}}`, `secrets.mask`, or `maskedSecrets` contract exists.

## Solution Assessment

No, not as a narrow automatic fix. The proposal is directionally valid, but placeholder syntax, command denial, output redaction, credential brokering, and plugin/provider boundaries need secops and product agreement first.
