## Summary

Keep open: current main and v2026.6.8 include SecretRef, redaction, audit, and plugin-hook mitigations, but they still do not provide the native masked-secret boundary requested for arbitrary agent-executed commands or agent-readable secret files.

## What This Changes

Request native masked secrets so agents can use API keys through references without seeing raw values, with substitution, output redaction, and command/config policy.

## Best Possible Solution

Define a maintainer-approved credential boundary that keeps raw values outside agent-readable files/env/context, supports dispatch-time injection and exact redaction where appropriate, and leaves plugin hooks or ClawHub mitigations as interim paths.

## Reproduction Assessment

Yes at source level: docs state plaintext secret files remain agent-readable and current source lacks the requested `{{secret}}` syntax or mask-list config; I did not run a live command because this review is read-only.

## Solution Assessment

Unclear until maintainer security direction is chosen; SecretRefs and plugin hooks are strong mitigations, but the requested native boundary likely needs a core/security-owned design rather than ad hoc exec command blocking.
