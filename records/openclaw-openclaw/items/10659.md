## Summary

Keep open: current main has useful SecretRef, runtime snapshot, command-scoped resolution, env sanitization, and redaction mitigations, but it still lacks the native masked-secret boundary requested here. The linked Linux user-isolation guide and openclaw-credential-vault plugin are valuable workarounds, while related core/RFC work remains open or closed unmerged.

## What This Changes

Request a native masked-secrets system so agents can use API keys without seeing raw values, including placeholder substitution, output redaction, command blocking, and mask-list config.

## Best Possible Solution

Keep this open until maintainers ship or explicitly supersede a core credential boundary that keeps raw values out of agent-readable storage, environment, and context while defining dispatch-time injection, exact redaction semantics, and plugin/API integration.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/security architecture request. Source inspection on current main shows the requested {{secret:NAME}}, secrets.mask, maskedSecrets, and general raw-read prevention surfaces are absent.

## Solution Assessment

No for a narrow automatic fix. The direction is valid, but placeholder syntax, command-denial policy, exact redaction semantics, credential-provider integration, and process trust boundaries need secops and product agreement first.
