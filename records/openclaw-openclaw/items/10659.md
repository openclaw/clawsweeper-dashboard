## Summary

Keep open. Current main and the latest release have useful SecretRef, runtime snapshot, host env sanitization, redaction, and plugin-hook mitigations, but they still do not provide the native masked-secret boundary requested for arbitrary agent-executed commands and raw credential isolation.

## What This Changes

The issue requests a native masked-secrets system that lets agents use credentials through references without seeing plaintext, including runtime substitution, output redaction, blocked credential reads, and a config contract.

## Best Possible Solution

Keep this open until maintainers ship or explicitly supersede a secops-owned credential boundary that keeps raw values out of agent-readable files, env, context, logs, and transcripts while defining the core/plugin API shape.

## Reproduction Assessment

Not applicable; this is a feature/security-boundary request rather than a broken existing contract. Source and docs inspection gives high confidence current main and v2026.5.12 still lack the requested native masked-secret boundary.

## Solution Assessment

No for automatic implementation. The direction is valid, but the maintainable solution needs secops/product agreement on the credential trust boundary and config/API shape before code lands.
