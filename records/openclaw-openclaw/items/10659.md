## Summary

Keep open: current main has substantial SecretRef, redaction, command-secret, and plugin-hook mitigations, but it still lacks the native masked-secret boundary requested for arbitrary agent-executed commands and readable secret files.

## What This Changes

The issue requests a masked-secrets capability that lets agents use credentials without exposing raw values to command output, prompt context, environment variables, or readable files.

## Best Possible Solution

Keep this open until maintainers ship or explicitly supersede a native credential-use boundary for agent-executed commands that keeps raw values out of agent-readable files, environments, prompt context, transcripts, and command output while aligning with SecretRef, provider auth, and plugin hooks.

## Reproduction Assessment

Unclear for the exact placeholder and blocked-command shape. The security direction is valid, but the maintainable path needs a secops-approved trust model rather than a command blacklist or a second credential system that drifts from SecretRef/provider auth.
