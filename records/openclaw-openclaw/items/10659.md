## Summary

Keep open: current main has partial SecretRef, runtime-snapshot, redaction, host-env sanitization, and plugin-hook mitigations, but the native masked-secret boundary requested here is still missing and recent maintainer/ClawSweeper comments already kept it open for follow-up.

## What This Changes

Requested a native masked-secrets system that lets agents use API keys without seeing raw values, including placeholder substitution, output redaction, command blocking, and a secrets mask config surface.

## Best Possible Solution

Keep this open until secops/product ships or explicitly supersedes a native credential boundary with dispatch-time injection, exact redaction semantics, and clear guarantees about agent-readable env/files.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that the requested surface is absent: no {{secret:NAME}} parser, no secrets.mask config, and no general exec credential broker boundary.

## Solution Assessment

No, not as a narrow automatic fix. The direction is valid, but the maintainable solution needs secops/product decisions on credential providers, process isolation or proxying, plugin hooks, and redaction semantics before implementation.
