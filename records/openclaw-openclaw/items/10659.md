## Summary

Keep open. Current main has meaningful SecretRef, runtime snapshot, host exec env sanitization, and redaction hardening, but it still lacks the native masked-secret boundary requested for arbitrary agent-executed commands and raw credential isolation.

## What This Changes

Request a native masked-secret system so agents can use API keys without reading raw values, including placeholder substitution, output redaction, command restrictions, and config.

## Best Possible Solution

Keep this open until maintainers ship or explicitly supersede a secops-owned credential boundary that keeps raw values out of agent-readable files, env, context, logs, and transcripts while defining clear core/plugin APIs.

## Reproduction Assessment

Not applicable; this is a feature/security-boundary request rather than a broken existing contract. Source and docs inspection gives high confidence current main still lacks the requested native masked-secret boundary.

## Solution Assessment

No for an automatic implementation. The direction is valid, but the maintainable solution needs secops/product agreement on the credential trust boundary and config/API shape before code lands.
