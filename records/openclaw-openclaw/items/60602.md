## Summary

Keep open. Current main still does not inject or expose Bedrock `requestMetadata`, and #60698 remains open and unmerged with closing syntax for the same request, so this issue should wait for that PR or a successor.

## What This Changes

Request automatic per-agent Bedrock `requestMetadata` injection so multi-agent deployments can attribute Bedrock costs by agent.

## Best Possible Solution

Finish a Bedrock-owned implementation through #60698 or a successor: thread verified session agent identity into Bedrock `requestMetadata`, validate against AWS limits, avoid sensitive values, preserve non-Bedrock behavior, coordinate configurable metadata with #57360, and add focused tests plus docs/changelog if user-facing behavior changes.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main has no `requestMetadata` matches, and the active stream option and Bedrock provider wrapper paths omit it; a live AWS run would only confirm downstream CloudTrail display.

## Solution Assessment

Yes on direction, but the implementation still needs review. The narrow maintainable solution is a provider-owned Bedrock metadata injection path using explicit agent identity, not a broad unvalidated core pass-through.
