## Summary

Keep open. Current main and the latest release still do not inject or forward Bedrock `requestMetadata`, and the open closing PR is conflicting and still has unresolved proof, validation, and privacy/default-policy blockers.

## What This Changes

Request automatic per-agent Bedrock `requestMetadata` injection, or a configurable equivalent, so multi-agent Bedrock usage can be attributed by agent in AWS cost/log surfaces.

## Best Possible Solution

Land or replace the linked PR with a narrow Bedrock-owned metadata path that validates AWS metadata, preserves wrapper precedence, documents privacy/precedence, and coordinates with the configurable metadata issue.

## Reproduction Assessment

Yes, at source level: current main and `v2026.5.22` have no `requestMetadata` forwarding or per-agent injection in the extra-param wrapper, Bedrock provider wrapper, plugin schema, tests, or docs. I did not run a live Bedrock/CloudTrail repro in this read-only review.

## Solution Assessment

Unclear for the exact default behavior. The requested direction is valid, but maintainers still need to choose whether metadata is automatic or opt-in, how `agentName` privacy works, and how automatic tags merge with user-configured metadata.
