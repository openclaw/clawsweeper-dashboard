## Summary

Keep this issue open. Current main still does not inject or expose Bedrock `requestMetadata` on the Bedrock Converse stream path, and #60698 remains open and unmerged with closing syntax for this issue, so the correct maintainer path is to review, rebase, land, or replace that implementation candidate.

## What This Changes

Request automatic per-agent Bedrock `requestMetadata` injection so multi-agent deployments can attribute Bedrock costs by agent.

## Best Possible Solution

Keep the issue open and finish the Bedrock-owned implementation through #60698 or a successor. The durable shape should thread verified agent identity into the provider request path, apply AWS metadata constraints, avoid sensitive values, preserve non-Bedrock behavior, coordinate with #57360 for configurable metadata, and add focused tests around the final stream options or payload.
