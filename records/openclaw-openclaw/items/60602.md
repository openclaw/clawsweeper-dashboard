## Summary

Keep open. Current main still lacks automatic per-agent Bedrock `requestMetadata` injection, and https://github.com/openclaw/openclaw/pull/60698 is still an open, unmerged closing PR for the same request, so this issue should remain open until that PR lands or maintainers choose a replacement.

## What This Changes

The issue requests automatic per-agent Bedrock `requestMetadata` injection so shared IAM-role Bedrock usage can be attributed by agent.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/60698 with a narrow Bedrock-owned implementation that validates metadata and coordinates configurable metadata with https://github.com/openclaw/openclaw/issues/57360.

## Reproduction Assessment

Yes. Source inspection on current main shows AWS/pi-ai support the field, but OpenClaw has no `requestMetadata` path in the stream options, Bedrock wrapper, plugin schema, tests, or docs.

## Solution Assessment

Yes on the requested direction, but the implementation still needs review. The maintainable path is a Bedrock-owned metadata injection seam with validation/privacy rules, while broader user-configured metadata remains tracked separately.
