## Summary

Keep open. Current main still has no Bedrock `requestMetadata` injection or config surface, and https://github.com/openclaw/openclaw/pull/60698 is an open, unmerged closing PR for the same per-agent tagging request.

## What This Changes

The issue requests automatic per-agent Bedrock `requestMetadata` injection, or a related config surface, so multi-agent Bedrock costs can be attributed per agent.

## Best Possible Solution

Review, rebase, split, or replace https://github.com/openclaw/openclaw/pull/60698 with a narrow Bedrock-owned implementation that injects validated per-agent metadata and leaves arbitrary configured metadata coordination to https://github.com/openclaw/openclaw/issues/57360.

## Reproduction Assessment

Yes. Source inspection is enough to show current main has no `requestMetadata` symbol, the stream option wrapper omits it, and the Bedrock provider wrapper does not add it.

## Solution Assessment

Yes on the direction, but not yet on the open branch. The maintainable path is a Bedrock-owned metadata injection seam with AWS constraint validation and explicit privacy handling, then PR review of https://github.com/openclaw/openclaw/pull/60698 or a successor.
