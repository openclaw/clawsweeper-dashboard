## Summary

Keep open: current main and v2026.6.10 can forward Bedrock `requestMetadata` when stream options already contain it, but they do not derive per-agent metadata or expose a supported Bedrock metadata policy. The remaining work needs maintainer product/security review because automatic AWS metadata export can expose local agent identifiers or names.

## What This Changes

The issue requests automatic or configurable per-agent Bedrock `requestMetadata` so shared Bedrock accounts can attribute costs by OpenClaw agent.

## Best Possible Solution

Keep this issue open until maintainers choose the Bedrock metadata policy, then land a narrow provider/runtime implementation with metadata minimization, merge semantics, focused tests, and redacted live Bedrock or CloudTrail proof.

## Reproduction Assessment

Yes. Source-level inspection shows current main has active agent context and Bedrock `requestMetadata` pass-through, but no current-main or v2026.6.10 code derives per-agent metadata or exposes a supported Bedrock metadata config path; I did not run live AWS traffic.

## Solution Assessment

Unclear until maintainers choose the metadata policy. Automatic injection is plausible, but the best fix should minimize exposed values and merge with explicit request metadata rather than overwriting it.
