## Summary

Keep open. Current main still has no Bedrock `requestMetadata` injection or config surface, and #60698 is still an open, unmerged closing PR for the same request, so this issue should remain open until that PR lands or maintainers choose a replacement.

## What This Changes

Request automatic per-agent Bedrock `requestMetadata` injection, or an equivalent config surface, so shared Bedrock usage can be attributed by agent in AWS cost/log tooling.

## Best Possible Solution

Finish #60698 or a successor as a Bedrock-owned implementation that threads validated agent identity into stream options, avoids sensitive metadata, adds focused coverage, and leaves configurable metadata coordination to #57360.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the OpenClaw gap: current main has no `requestMetadata` symbol, the active stream option builder omits it, and the Bedrock wrapper does not add it.

## Solution Assessment

Yes on direction, but the implementation still needs review. The narrow maintainable path is provider-owned Bedrock metadata injection through an explicit agent-identity seam with AWS constraint validation, not an unbounded core pass-through.
