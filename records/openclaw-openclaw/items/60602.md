Keep open: current main and v2026.6.10 can forward Bedrock `requestMetadata` when it is already supplied, but they do not derive per-agent metadata or expose a supported config path for it. ## What This Changes

Add automatic or configurable per-agent Bedrock `requestMetadata` so multi-agent Bedrock calls can be attributed by agent in AWS logs and cost tooling.

## Best Possible Solution

Keep the issue open for a maintainer-owned Bedrock metadata policy decision, then either review the linked PR or replace it with a narrow current-runner/provider implementation.

Unclear until maintainers choose the metadata policy. Automatic injection is plausible, but the best fix needs minimized/sanitized values and explicit merge behavior with any caller-supplied metadata.
