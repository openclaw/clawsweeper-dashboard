## Summary

Keep this open: current `main` and the latest release can forward Bedrock `requestMetadata` when options already contain it, but they do not automatically inject per-agent metadata. The linked implementation PR is still open and unmerged, so this issue should stay open until maintainers review, land, or replace that PR.

## What This Changes

Request automatic per-agent Amazon Bedrock `requestMetadata` on model calls so multi-agent Bedrock spend can be attributed by agent.

## Best Possible Solution

Keep the issue open and land a narrow Bedrock-provider-owned implementation, with focused tests for zero-extra-param and configured-param cases plus redacted CloudTrail or equivalent real Bedrock proof before closing.

## Reproduction Assessment

Yes, source-level: current main has an agent-aware stream wrapper path and a Bedrock provider that forwards `requestMetadata`, but no current-main code injects per-agent metadata. I did not run live AWS/CloudTrail traffic, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Unclear until maintainers choose the policy shape. Automatic injection is plausible, but the best fix should live at the Bedrock provider/runtime boundary, preserve future user-supplied metadata, avoid leaking unnecessary agent names by default, and include live Bedrock proof.
