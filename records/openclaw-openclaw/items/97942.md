## What This Changes

The PR updates agent exec script preflight to recursively unwrap command carriers such as `sudo`, `env`, and shell wrappers, then adds regression coverage for carrier-wrapped Python and Node script validation.

## Best Possible Solution

Keep the exec preflight hardening in `src/agents/bash-tools.exec.ts`, but make the no-script-validated fail-closed branch carrier-aware and add regression coverage for carrier-wrapped shell payloads whose scripts cannot be validated before merge.

## Solution Assessment

No: recursive carrier unwrapping is the right owner-boundary fix, but the fail-closed branch also needs to use carrier-aware context when no script was actually validated. The narrow repair is to carry that context into the fallback and add the missing regression case.
