## What This Changes

The PR routes image-generation success JSON reads through bounded provider JSON readers, adds an inline-image response cap helper exported through the Plugin SDK, and updates provider tests plus SDK baseline hashes.

## Best Possible Solution

Land one coherent image-generation response hardening path with image-sized caps for inline base64 payloads, generic caps for metadata/job responses, and separate handling for broader fal music/video work.

## Reproduction Assessment

## Solution Assessment

Yes, this is the right implementation layer. Reusing readProviderJsonResponse avoids a new reader path, and the follow-up image-sized cap addresses the main inline base64 compatibility concern.
