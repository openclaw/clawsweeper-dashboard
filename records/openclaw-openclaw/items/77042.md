## Summary

Keep open: the PR targets a valid Feishu allowlist bug, but current head introduces a broader regression by treating channel-shaped plugin IDs as known only when the registry entry is bundled, which would skip installed/configured channel plugins already known to the manifest registry.

## What This Changes

The PR changes plugin auto-enable to stop adding unclaimed configured channel IDs to `plugins.allow`, adds a Feishu regression test, and adds a changelog entry.

## Best Possible Solution

Keep the no-claim channel guard, but recognize actual registry plugin IDs before suppressing bare channel-ID fallbacks.

## Reproduction Assessment

Yes. The linked bug is source-reproducible on current main through configured Feishu channel detection returning `feishu` as an auto-enable candidate when no manifest claims it, and the PR-head regression is source-reproducible against the existing configured-entry allowlist contract.

## Solution Assessment

No. The PR is pointed at the right helper, but the bundled-only `isKnownPluginId` check is broader than necessary; registry-known plugin IDs should remain valid regardless of origin.
