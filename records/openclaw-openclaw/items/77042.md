## Summary

Keep open: the PR is not currently mergeable because the proposed logic still preserves the `builtInId` fallback, and Feishu is a catalog-known bundled channel id, so a configured `channels.feishu` with an empty manifest registry can still be materialized into `plugins.allow`.

## What This Changes

The PR adds a changelog entry, a Feishu regression test, and explanatory comments around channel auto-enable fallback behavior.

## Best Possible Solution

Update the auto-enable fallback so unclaimed plugin-owned/catalog channels do not become plugin candidates, while keeping explicit manifest claims and existing core-channel fallback coverage intact.

## Reproduction Assessment

Yes. Source inspection shows `channels.feishu` is a known catalog channel id, so the current fallback still creates a `feishu` auto-enable candidate and appends it to a restrictive `plugins.allow`.

## Solution Assessment

No. The proposed branch adds the right regression intent, but the implementation still returns `[builtInId]` for catalog-known Feishu; the safer fix is to restrict that fallback to truly core-managed channels or manifest-claimed plugins.
