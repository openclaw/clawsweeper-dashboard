## Summary

Keep this PR open for normal review and merge consideration. Current main still has the source-level path that can synthesize a configured channel id as a plugin id, and the PR’s small change, regression test, and changelog entry address the linked Feishu allowlist warning without introducing a blocking defect.

## What This Changes

The PR changes plugin auto-enable to return no plugin candidate when no manifest claims a configured channel, adds Feishu restrictive-allowlist regression coverage, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow fix so only manifest-claimed plugin ids are added to plugins.allow, while keeping the linked issue open until the fix merges.

## Reproduction Assessment

Yes, at source level. On current main, applyPluginAutoEnable with channels.feishu enabled, a restrictive plugins.allow that omits feishu, and an empty manifest registry follows the no-claims fallback and then appends feishu through ensurePluginAllowlisted.

## Solution Assessment

Yes. Returning no candidates when the registry has no channel claims is the narrowest maintainable fix because plugins.allow is a plugin-id list, and the existing claim path still handles installed channel plugins.
