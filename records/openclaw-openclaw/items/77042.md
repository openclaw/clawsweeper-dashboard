## Summary

Keep this PR open: the latest head addresses the original no-claim channel fallback, but the new bundled-only known-plugin check would regress restrictive allowlist preservation for configured plugins whose IDs are also channel IDs.

## What This Changes

The PR changes plugin auto-enable so configured channels without claiming plugin manifests do not add channel IDs to `plugins.allow`, and adds a Feishu regression test plus a changelog entry.

## Best Possible Solution

Keep the no-claim channel guard, but make known-plugin detection honor actual registry plugin IDs regardless of origin before suppressing bare channel-ID fallbacks.

## Reproduction Assessment

Yes. Source inspection reproduces the linked bug path on current main with a configured Feishu channel, restrictive allowlist, and no claiming manifest; the PR's test targets that path, and the new regression is also source-reproducible against the existing configured-entry contract.

## Solution Assessment

No. The PR is pointed at the right narrow area, but requiring a bundled plugin inside `isKnownPluginId` is too broad; the safer fix is to treat registry plugin IDs as known and only block unclaimed bare channel IDs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep installed channel plugins known:** `src/config/plugin-auto-enable.shared.ts:889-895`
