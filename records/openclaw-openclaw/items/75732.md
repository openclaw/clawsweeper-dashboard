## Summary

Keep open: this targets a real current-main image optimization gap, but the patch still has merge-blocking correctness and security-policy issues around the new `vipsthumbnail` backend.

## What This Changes

The PR adds a Linux x64 `vipsthumbnail` fallback in `src/media/image-ops.ts` when Sharp-backed JPEG resizing fails.

## Best Possible Solution

Keep the Linux compatibility fix, but make the libvips invocation match the CLI contract, preserve image-size safety limits, add focused tests and a changelog entry, then get maintainer approval for the backend policy.

## Reproduction Assessment

Yes. Current main can be exercised by forcing the Sharp import or resize path to fail on Linux/non-sips and then calling `optimizeImageToJpeg()`; the PR-specific fallback failure is source-verified because `vipsthumbnail` does not accept the standalone `-Q` argument used by the patch.

## Solution Assessment

No. A libvips fallback may be a reasonable direction, but this implementation is not yet the best fix because the CLI call is incompatible and the new decoder path lacks guard parity with Sharp.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass quality through the vips output spec:** `src/media/image-ops.ts:484`
  - body: `vipsthumbnail` does not accept `-Q` as a standalone option; JPEG quality must be part of the output spec, for example `out.jpg[Q=85]`. With the current arguments, the fallback exits with an option error and `resizeToJpeg()` rethrows the original Sharp failure, so the target SSE2-only path remains broken.
  - confidence: 0.94
- **[P2] Preserve the decoded-pixel limit in the vips path:** `src/media/image-ops.ts:484`
  - body: The existing Sharp backend decodes with `limitInputPixels`, but this fallback hands the uploaded buffer to `vipsthumbnail` after only the header preflight. Add an equivalent decoded-size guard or restrict the fallback to formats whose dimensions are bounded by the existing preflight before invoking the external decoder.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `src/media/image-ops.ts:623-624`
  - body: This is a user-facing `fix` for image uploads on affected Linux systems, so repo policy requires a `CHANGELOG.md` entry before merge. Add a single-line entry under the active Fixes section with appropriate contributor credit.
  - confidence: 0.88
