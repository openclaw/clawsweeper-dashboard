## Summary

Keep open. Current main still copies host pnpm overrides into managed plugin npm roots, and the current host override pins `path-to-regexp` to 8.4.0 while Express 4 declares 0.1.12; the older Microsoft Teams fixes were plugin-specific and do not solve this external-plugin install path.

## What This Changes

The issue reports that Express 4.x-based plugins can crash because managed plugin installs can force `path-to-regexp` 8.x unless each plugin declares its own override.

## Best Possible Solution

Define a managed npm-root override policy that does not force host-only dependency pins into third-party plugin dependency trees, then add regression coverage for an Express 4 plugin while preserving OpenClaw's Express 5/router pins.

## Reproduction Assessment

Yes from source and dependency metadata: managed plugin installs inherit host pnpm overrides, and the inherited `path-to-regexp` 8.4.0 pin conflicts with Express 4's declared 0.1.12 dependency. I did not run a live install/runtime repro because this review is read-only and avoids dependency installation.

## Solution Assessment

No for the preferred blanket override: forcing 0.1.x globally would likely break OpenClaw's Express 5/router path. The maintainable fix is to scope or filter managed plugin overrides, or add a clear validator, after maintainers choose the dependency policy.
