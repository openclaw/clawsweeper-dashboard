## Label Justifications

- `P2`: The issue reports a valid plugin startup crash path with limited blast radius to Express 4.x npm plugins installed through managed roots.
- `impact:crash-loop`: Affected plugins can fail their HTTP route startup path when path-to-regexp 8.x is forced into an Express 4 dependency graph.

## Summary

Keep open: current main and the latest release still propagate OpenClaw's path-to-regexp 8.x managed override into npm plugin roots, while the reported plugin path uses Express 4.x. The blanket override suggested in the issue is not safe without maintainer direction because OpenClaw's own Express 5/router stack requires path-to-regexp 8.x.

## What This Changes

The issue requests a core-level fix or validator for Express 4.x npm plugins whose route startup crashes when managed installs inherit OpenClaw's path-to-regexp 8.x override.

## Best Possible Solution

Define a scoped managed-npm dependency ownership policy that preserves plugin-owned Express 4 dependency graphs without downgrading core Express 5/router, then cover install/update behavior with docs and regression tests.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current managed npm installs inherit the host path-to-regexp 8.4.0 override, while the reported plugin depends on Express 4.x and Express 4.x resolves path-to-regexp 0.1.x; no live install was run in this read-only review.

## Solution Assessment

No for the issue's preferred blanket override, because core Express 5/router needs path-to-regexp 8.x. The maintainable fix is a maintainer-approved scoped managed-root policy or a deliberate validator-only boundary.
