## Summary

Keep open: current main still propagates OpenClaw's workspace override set, including path-to-regexp 8.4.0, into managed npm plugin roots, while the reported Express 4.x plugin path needs path-to-regexp 0.1.x. Closed Microsoft Teams fixes and the merged managed-peer EOVERRIDE repair are related but do not solve this external Express 4 plugin dependency-ownership problem.

## What This Changes

The issue asks OpenClaw to prevent Express 4.x npm plugins from crashing when managed plugin installs inherit OpenClaw's path-to-regexp 8.x override.

## Best Possible Solution

Define a scoped managed-npm-root policy that preserves third-party plugin dependency graphs without globally downgrading OpenClaw's Express 5 stack, and cover it with install/update regression tests and docs.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main reads the workspace path-to-regexp 8.4.0 override into managed plugin roots, while npm metadata shows Express 4.x requires path-to-regexp 0.1.12 and the reported example plugin depends on Express 4. I did not run a live plugin install because this cleanup review is read-only.

## Solution Assessment

No, the issue's preferred blanket top-level downgrade is not the best fix because OpenClaw's current Express 5/router stack needs path-to-regexp 8.x. A scoped managed-root policy or a maintainer-approved validator is the safer boundary.
