## Label Justifications

- `P2`: The issue reports a real but bounded plugin startup crash for Express 4.x plugins installed through managed npm roots.
- `impact:crash-loop`: Affected plugins can fail HTTP route startup when path-to-regexp 8.x is forced into an Express 4 dependency graph.

## Summary

Keep open: current main and v2026.6.10 still propagate the host managed override set that includes path-to-regexp 8.4.0 into managed npm plugin roots, while Express 4 plugins need path-to-regexp 0.1.x. The issue is valid, but the preferred blanket downgrade is not the best fix because OpenClaw's Express 5/router stack depends on path-to-regexp 8.x.

## What This Changes

The issue reports that Express 4.x-based plugins can crash in managed OpenClaw npm installs because host path-to-regexp 8.x override policy can override the plugin's expected 0.1.x dependency graph.

## Best Possible Solution

Define a narrow managed-npm-root dependency policy that preserves plugin-owned Express 4 dependency graphs without globally downgrading OpenClaw's Express 5/router path, and add docs plus regression coverage for install/update behavior.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main propagates managed overrides that include path-to-regexp 8.4.0 into managed npm roots, while Express 4.21.2 declares path-to-regexp 0.1.12 and the reported plugin example depends on Express 4.

## Solution Assessment

No. The issue's preferred blanket central downgrade is not the best fix because current OpenClaw uses Express 5.2.1 and router 2.2.0, which depends on path-to-regexp ^8.0.0; the safer direction is a maintainer-approved scoped managed-root policy or validator.
