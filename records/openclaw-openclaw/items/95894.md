## Summary

Keep open: current main and the latest release still propagate the host `pnpm-workspace.yaml` override set into managed npm plugin roots, including `path-to-regexp: 8.4.0`; the related msteams fixes and the merged managed-peer-pin repair do not cover third-party Express 4 plugin graphs.

## What This Changes

The issue requests central handling or validation so Express 4.x-based plugins are not forced onto OpenClaw's host `path-to-regexp` 8.x override.

## Best Possible Solution

Keep this as the canonical maintainer decision point for scoped managed-root override policy: either filter/shape host overrides for third-party plugin roots, add targeted Express 4 validation with an actionable error, or document a tested plugin packaging contract.

## Reproduction Assessment

Yes, source-level: managed npm plugin installs read the host workspace overrides and current main ships `path-to-regexp: 8.4.0`, while Express 4.21.2 declares `path-to-regexp: 0.1.12`. I did not run a full packaged plugin install repro in this read-only pass.

## Solution Assessment

No for the proposed blanket downgrade: OpenClaw's current Express 5/router 2.x stack depends on path-to-regexp 8.x. The best fix needs maintainer choice between scoped override propagation, targeted validation, or an explicit plugin packaging contract.
