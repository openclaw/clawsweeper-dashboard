## Label Justifications

- `P2`: The issue reports a real plugin startup/runtime crash path with limited blast radius to Express 4.x plugins installed through managed npm roots.
- `impact:crash-loop`: Affected plugins can fail their HTTP route startup path when `path-to-regexp` 8.x is forced into an Express 4 dependency graph.

## Summary

Keep open: current main still propagates the host `path-to-regexp` 8.4.0 override into managed npm plugin roots, which is incompatible with Express 4.x plugin dependency graphs, while a blanket downgrade would conflict with OpenClaw's own Express 5/router stack.

## What This Changes

The issue asks OpenClaw core to prevent Express 4.x plugin route crashes caused by managed plugin installs inheriting `path-to-regexp` 8.x.

## Best Possible Solution

Define a scoped managed-npm-root dependency policy that preserves plugin-owned Express 4 dependency graphs without downgrading OpenClaw's Express 5/router path, with focused install/update regression coverage and docs.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current managed npm install code reads host workspace overrides and writes them into plugin project manifests, while Express 4.21.2 declares `path-to-regexp` 0.1.12 and OpenClaw pins 8.4.0.

## Solution Assessment

No. The issue's preferred blanket override/downgrade is not the best fix because current Express 5/router needs `path-to-regexp` 8.x; the safer direction is a scoped managed-root policy or validator chosen by maintainers.
