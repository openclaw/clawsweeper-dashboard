## Summary

Keep open: current main and the latest release still propagate OpenClaw’s host npm overrides into managed plugin npm roots, which can force path-to-regexp 8.x into Express 4 plugin dependency graphs. The issue remains necessary because no merged or open canonical fix owns the managed-root dependency policy decision.

## What This Changes

The issue asks OpenClaw core to stop Express 4.x-based plugins from crashing when plugin installs inherit a path-to-regexp 8.x instead of Express 4's 0.1.x dependency.

## Best Possible Solution

Define a narrow managed-npm-root dependency policy that preserves plugin-owned Express 4 dependency graphs without globally downgrading OpenClaw's Express 5/router path, with docs and regression coverage for install/update behavior.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence because current managed npm installs inherit the host path-to-regexp 8.4.0 override, while Express 4.x package metadata requires path-to-regexp 0.1.12.

## Solution Assessment

No. The issue's preferred blanket downgrade is not the best fix because OpenClaw's current Express 5/router stack depends on path-to-regexp 8.x; the safer solution is a maintainer-approved scoped managed-root policy or diagnostic that does not break core.
