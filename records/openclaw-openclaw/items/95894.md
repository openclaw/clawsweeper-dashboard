## Summary

Keep open: current main still has a credible managed-plugin dependency-resolution bug where OpenClaw-managed npm plugin roots inherit the host `path-to-regexp` override, while Express 4 plugins require the 0.1.x dependency line. The requested blanket downgrade is not the right fix, but the underlying plugin install failure is actionable.

## What This Changes

Ensure OpenClaw plugin installs do not force `path-to-regexp` 8.x onto Express 4.x plugins, avoiding per-plugin overrides and runtime router crashes.

## Best Possible Solution

Keep the issue open and fix the managed npm override policy so plugin roots respect Express 4's dependency contract while any Express 5/router patch remains scoped and tested.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: managed npm plugin roots inherit host overrides, and upstream metadata shows Express 4 requires `path-to-regexp` 0.1.12. A live install/startup run was not performed under the read-only review constraints.

## Solution Assessment

No, the issue's blanket downgrade proposal is not the best fix because OpenClaw currently uses Express 5/router paths that need the patched 8.x line. The maintainable fix is a scoped managed-root override policy plus regression coverage for an Express 4 plugin fixture.
