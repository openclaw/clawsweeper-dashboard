## Summary

Keep open: current main still propagates host pnpm overrides into managed plugin npm roots, and the latest release tag still contains the same override inheritance plus the path-to-regexp 8.4.0 pin. The report is valid, but the requested blanket downgrade is not the right automatic fix because OpenClaw's current Express 5/router stack depends on path-to-regexp 8.x.

## What This Changes

The issue requests a core fix for managed npm plugin installs that can force path-to-regexp 8.x into Express 4.x plugin dependency graphs.

## Best Possible Solution

Define a narrow managed-npm-root dependency policy that preserves plugin-owned Express 4 graphs without globally downgrading OpenClaw's Express 5/router path, then add focused install/update docs and regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current managed npm install code applies root pnpm overrides to plugin project roots, while Express 4.21.2 requires path-to-regexp 0.1.12 and the host override pins 8.4.0; I did not run a live plugin install in this read-only cleanup review.

## Solution Assessment

No. The issue's preferred blanket downgrade is not the best fix because OpenClaw's Express 5/router dependency path needs path-to-regexp ^8.0.0; a scoped managed-root policy or validator needs maintainer approval first.
