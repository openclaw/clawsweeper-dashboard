## Summary

Keep open. The leak concern is plausible from source, but this patch removes the only post-dispose Discord gateway `error` listener and would reintroduce the unhandled EventEmitter error path that current main explicitly suppresses and tests.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` to Discord gateway supervisor disposal.

## Best Possible Solution

Release disposed-supervisor state with a bounded late-error sink or equivalent design that frees large closures while preserving late gateway `error` suppression, backed by focused regression coverage and runtime proof.

## Reproduction Assessment

Yes. Source inspection plus current main tests give a high-confidence path: with the PR diff applied, a post-dispose `emitter.emit("error", ...)` can have no listener, and Node throws unhandled `error` events.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because current lifecycle code depends on a listener after disposal; a bounded sink or equivalent cleanup is safer.
