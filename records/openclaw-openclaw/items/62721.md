## Summary

Keep open: the one-line cleanup still removes the Discord gateway emitter's late-error handler, and existing tests plus PR CI show that turns post-dispose gateway errors into uncaught EventEmitter errors. The PR also still lacks after-fix real behavior proof.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` at the start of Discord gateway supervisor disposal.

## Best Possible Solution

Release disposed-supervisor state through a bounded late-error sink or state split that keeps post-dispose gateway `error` events handled, with focused tests and redacted runtime proof.

## Reproduction Assessment

Yes for the PR regression: PR CI failed the focused late-error tests, and the source path is clear by disposing the supervisor and then emitting a gateway `error`. This is not a current-main bug; current main deliberately suppresses that late error.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because shutdown and startup-cleanup paths depend on post-dispose error suppression; a bounded sink or state-splitting cleanup is safer.
