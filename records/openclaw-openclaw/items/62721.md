## Summary

Keep open: the proposed cleanup is a real concern area, but the current one-line detach breaks the existing Discord gateway shutdown contract by allowing late EventEmitter `error` events to throw after dispose.

## What This Changes

Adds an `emitter.off("error", onGatewayError)` call at the start of Discord gateway supervisor disposal.

## Best Possible Solution

Release disposed-supervisor state through a bounded late-error sink or equivalent state split that keeps post-dispose gateway `error` events handled, with focused tests and real runtime proof.

## Reproduction Assessment

Yes for the PR regression: PR CI on the head commit failed the focused late-error tests, and the source path is clear by disposing the supervisor and then emitting a gateway `error`. This is not a current-main bug; current main deliberately suppresses that late error.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because shutdown and startup-cleanup paths depend on post-dispose error suppression; a bounded sink or state-splitting cleanup is safer.
