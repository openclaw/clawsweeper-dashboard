## Summary

Keep open: the leak concern may be worth solving, but this PR breaks the current Discord shutdown contract by removing the post-dispose `error` sink, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` at the start of Discord gateway supervisor disposal.

## Best Possible Solution

Free disposed-supervisor state with a bounded late-error sink or equivalent design that preserves late gateway `error` suppression, then add focused regression coverage and runtime proof.

## Reproduction Assessment

Yes for the proposed patch regression: source inspection plus a Node EventEmitter probe show that detaching the only `error` listener leaves late post-dispose errors uncaught, while current and shipped tests expect suppression.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because shutdown and startup-cleanup paths rely on a post-dispose error sink; a bounded sink or narrower cleanup is safer.
