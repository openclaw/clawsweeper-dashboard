## Label Justifications

- `merge-risk: 🚨 compatibility`: The patch changes the existing post-dispose late-error suppression behavior that current tests rely on during gateway cleanup.

## Summary

Keep this PR open but blocked: the one-line cleanup conflicts with the current Discord gateway supervisor contract because late gateway `error` events after disposal are intentionally still handled. The leak concern may be real, but this patch removes the only listener without a replacement sink, and the PR CI already shows the resulting post-dispose errors throw.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` to Discord gateway supervisor disposal.

## Best Possible Solution

Release disposed-supervisor state through a bounded late-error sink or equivalent state split that keeps post-dispose gateway `error` events handled.

## Reproduction Assessment

Yes for the PR regression: current tests and PR CI show that emitting a gateway `error` after dispose throws when this patch removes the listener. The source path is also clear from Node EventEmitter's no-listener `error` behavior.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because shutdown and startup-cleanup paths depend on post-dispose error suppression; the safer direction is a bounded sink or state-splitting cleanup.
