## Summary

Keep open. The PR targets a plausible cleanup, but the one-line detach removes the only post-dispose Discord gateway error listener, conflicting with current shutdown tests and Node EventEmitter behavior; it needs a safer rewrite and real behavior proof before merge.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` to Discord gateway supervisor disposal.

## Best Possible Solution

Detach heavyweight disposed supervisor state only with a replacement bounded late-error sink that preserves post-dispose error suppression and regression coverage.

## Reproduction Assessment

Yes. Source inspection plus the Node EventEmitter contract show that after this PR's `off()` call, a post-dispose `emitter.emit("error", ...)` with no other listener would throw; current main has tests for that path.

## Solution Assessment

No. Directly removing the listener is not the best fix because it trades a retained-closure concern for a process-crash path; a minimal bounded sink or retaining the current behavior is safer.
