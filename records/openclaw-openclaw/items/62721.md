## Summary

Keep open but do not merge as-is: the PR addresses a plausible retained-listener concern, but the one-line cleanup removes the only Discord gateway `error` listener and conflicts with current late-error suppression behavior. The prior Greptile approval was reviewed, but current source, tests, Node EventEmitter behavior, and the existing Codex P1 review support blocking this patch.

## What This Changes

Adds `emitter.off("error", onGatewayError)` at the start of Discord gateway supervisor disposal.

## Best Possible Solution

Free disposed-supervisor state with a bounded late-error sink or equivalent design that preserves late gateway `error` suppression, then back it with focused regression coverage and runtime proof.

## Reproduction Assessment

Yes for the proposed patch regression. Source inspection plus a local Node EventEmitter smoke show that applying the diff leaves post-dispose `error` events without a listener, while current main tests expect those events to be suppressed.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because current lifecycle code depends on post-dispose error handling; a bounded sink or narrower cleanup is safer.
