## Summary

Keep open for changes: the listener-leak concern is plausible, but this patch removes the Discord gateway's only `error` listener and breaks the existing late-error suppression contract that current tests and PR CI cover.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` to Discord gateway supervisor disposal.

## Best Possible Solution

Release disposed-supervisor state through a bounded late-error sink or equivalent state split that preserves post-dispose gateway `error` handling.

## Reproduction Assessment

Yes for the PR regression: PR CI reproduced thrown late gateway `error` events after the listener was removed, and a Node EventEmitter probe confirms the no-listener `error` behavior. The claimed retained-listener leak is source-visible but still lacks real runtime proof.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because shutdown and startup-cleanup paths depend on post-dispose error suppression.
