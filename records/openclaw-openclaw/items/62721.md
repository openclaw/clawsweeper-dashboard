## Summary

Keep open: the patch removes the Discord gateway supervisor's only `error` listener after dispose, which conflicts with current shutdown behavior and tests that suppress late gateway errors instead of letting Node treat them as uncaught.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` during Discord gateway supervisor disposal.

## Best Possible Solution

Release retained disposed-supervisor state only with a bounded late-error sink or equivalent design that preserves a listener for late gateway errors, plus focused regression coverage and real runtime proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the PR regression: after the added `off()` call, a post-dispose `emitter.emit("error", ...)` with no other listener follows Node's unhandled `error` throw path. Current main has focused tests for that late-error suppression contract.

## Solution Assessment

No. Directly detaching the listener is not the best fix because it trades a retained-closure concern for a process-crash path; a small bounded sink or equivalent cleanup that keeps post-dispose error suppression is safer.
