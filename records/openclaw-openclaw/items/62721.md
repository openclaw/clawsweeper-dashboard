## Summary

Keep open: the PR is a focused bug-fix attempt, but the one-line detach removes the Discord gateway supervisor's post-dispose error sink and can turn late gateway errors into uncaught EventEmitter errors.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` at the start of Discord gateway supervisor disposal.

## Best Possible Solution

Detach heavyweight disposed supervisor state only with a replacement bounded late-error sink that preserves post-dispose error suppression and has regression coverage.

## Reproduction Assessment

Yes. Source inspection plus a Node EventEmitter smoke show that after the PR's `off()` call, a post-dispose `emitter.emit("error", ...)` with no other listener would throw; current main has tests for that late-error path.

## Solution Assessment

No. Directly removing the listener is not the best fix because it trades a possible retained-closure issue for a process-crash path; a bounded replacement sink or a more careful state split is safer.
