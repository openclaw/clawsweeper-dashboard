## Summary

Keep open: the PR is a focused bug-fix attempt, but the one-line change breaks the current Discord gateway late-error contract and still lacks after-fix real behavior proof.

## What This Changes

Adds `emitter.off("error", onGatewayError)` at the start of Discord gateway supervisor disposal.

## Best Possible Solution

Free disposed-supervisor state with a bounded late-error sink or equivalent design that preserves late gateway `error` suppression, then add focused regression coverage and runtime proof.

## Reproduction Assessment

Yes for the proposed patch regression: source inspection plus the Node EventEmitter contract show that detaching the only `error` listener leaves late post-dispose errors uncaught, while current and shipped tests expect them to be suppressed.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because existing shutdown and startup-cleanup paths rely on a post-dispose error sink; a bounded sink or narrower cleanup is safer.
