## Summary

Keep open: the one-line detach removes the Discord gateway supervisor's post-dispose error sink, conflicting with current main tests and shutdown behavior that keep late gateway errors handled. The external PR also lacks after-fix real behavior proof.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` to Discord gateway supervisor disposal.

## Best Possible Solution

Release disposed-supervisor state through a bounded late-error sink or equivalent design that keeps post-dispose gateway `error` events handled, with focused regression coverage and runtime proof.

## Reproduction Assessment

Yes for the proposed patch regression: current tests show late gateway errors must remain handled after dispose, and a Node EventEmitter with no `error` listener throws. The PR removes that listener without a replacement sink.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because shutdown and startup-cleanup paths depend on post-dispose error suppression; a bounded sink or state-splitting cleanup is safer.
