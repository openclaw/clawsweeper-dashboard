## Summary

This PR should stay open but is not merge-ready: the one-line listener removal conflicts with the current Discord shutdown contract and can turn late gateway errors into uncaught EventEmitter errors. It also still lacks after-fix real behavior proof from a runtime setup.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` to Discord gateway supervisor disposal.

## Best Possible Solution

Release disposed-supervisor state through a bounded late-error sink or equivalent state split that keeps post-dispose gateway `error` events handled.

## Reproduction Assessment

Yes for the PR regression: current tests and Node EventEmitter behavior show that a late gateway `error` after dispose must remain handled. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. Directly detaching `onGatewayError` is not the best fix because shutdown and startup-cleanup paths depend on post-dispose error suppression; a bounded sink or state-splitting cleanup is safer.
