## Summary

Keep open. The PR should not merge as-is because removing the supervisor's only `error` listener after disposal regresses the intentional late-error suppression path and can turn a post-shutdown gateway error into an uncaught EventEmitter exception.

## What This Changes

The PR adds `emitter.off("error", onGatewayError)` to Discord gateway supervisor disposal.

## Best Possible Solution

Preserve late Discord gateway error suppression after shutdown; if retention is actionable, replace the full handler with a minimal bounded late-error sink and update the supervisor/provider cleanup tests.

## Reproduction Assessment

Yes. The current `gateway-supervisor.test.ts` path disposes a supervisor and emits `error`, and applying this PR's detachment would make that EventEmitter path throw when no other `error` listener is present.

## Solution Assessment

No. Directly removing the listener is not the best fix because it trades retained state for a process-crash path; a replacement sink is the safer maintainable direction if the leak needs fixing.
