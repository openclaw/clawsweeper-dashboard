## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Export the broker runtime setter:** `extensions/channel-broker/runtime-api.ts:1-18`
  - body: `index.ts` tells the bundled channel loader to import `setChannelBrokerRuntime` from `./runtime-api.js`, and the loader calls that export during runtime wiring. This new `runtime-api.ts` never exports the setter from `./src/runtime.js`, so registering the channel-broker runtime will fail with a missing export before the plugin can send or receive through the injected runtime. Export the setter here or point `index.ts` at a narrow runtime-setter sidecar like the existing Telegram pattern.
  - confidence: 0.95
- **[P2] Accept the SDK capability shape in config:** `extensions/channel-broker/src/config-schema.ts:5-23`
  - body: The public SDK type for platform capabilities uses nested `delivery`, `live`, `receive`, and `native` groups, and the conformance tests exercise that shape. This strict schema instead accepts only flat keys such as `threads`, `draftPreview`, and `businessApi`, so a provider config following the SDK/docs contract will be rejected during config validation. Align the schema with the SDK shape and add a focused config-schema test.
  - confidence: 0.9
