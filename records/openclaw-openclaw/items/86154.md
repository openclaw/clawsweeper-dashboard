Full review comments:

- **[P1] Export the advertised broker runtime setter:** `extensions/channel-broker/index.ts:16-18`
  - body: This entry declares `runtime.exportName: "setChannelBrokerRuntime"` from `./runtime-api.js`, and `defineBundledChannelEntry` loads that export during registration. The added `runtime-api.ts` never exports it, so registering `channel-broker` will throw `missing export` before the channel can load. Export the setter from the runtime sidecar or point the entry at a module that does.
  - confidence: 0.95
- **[P2] Accept the SDK capability shape in config:** `extensions/channel-broker/src/config-schema.ts:5-23`
  - body: The public config type is based on `BrokerPlatformCapabilities`, whose SDK shape is `delivery`, `live`, `receive`, and `native`, but this Zod schema only accepts flat fields such as `inbound`, `outbound`, and `draftPreview`. A typed provider config using the SDK capability object will be rejected and status will never see those capabilities. Align the schema and normalization with the SDK shape or change the public type/docs.
  - confidence: 0.88
