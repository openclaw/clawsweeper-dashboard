## Summary

This active PR addresses a real Gemini replay regression in the right Google plugin surface, but current head should stay open because the replay helper can still override a valid stored same-route signature with a later cache hit, and the latest production commit only has test/mock proof rather than fresh real behavior proof.

## What This Changes

The PR updates the native Google transport to recover, sanitize, and route-gate Gemini tool-call thought signatures during parsing and replay, with focused regression tests.

## Best Possible Solution

Prefer a valid sanitized same-route signature already stored on the current block, use the replay cache only when that signature is missing, add a regression for the duplicate-key/different-signature case, then refresh real behavior proof for the current head.

## Reproduction Assessment

Yes. The linked issue has a direct Gemini replay matrix for the 400, and current-main source still shows parser and serializer paths that can lose or substitute required Gemini tool-call signatures.

## Solution Assessment

No, not as-is. The owner-boundary fix direction is right, but the PR should preserve a valid stored same-route signature before consulting recovered replay signatures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Prefer stored signatures before replay-cache hits:** `extensions/google/transport-stream.ts:575`
  - body: This helper prefers `replayedThoughtSignature` over `ownSignature` because `retainThoughtSignature` returns the incoming value when present. If the cache has a later signature for the same replay key, an earlier already-signed same-route tool call is serialized with the wrong signature; Google requires returning the signature in the exact part where it was received. Prefer `ownSignature ?? replayedThoughtSignature` here and cover the duplicate-key/different-signature case. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/thought-signatures?hl=en&utm_source=openai))
