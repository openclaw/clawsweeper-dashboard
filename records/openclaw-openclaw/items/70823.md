## Summary

Keep open. Current main still treats plugin slots as bare string selectors, while this RFC asks for a public config/schema/plugin contract change with owner/provenance metadata and overwrite policy that needs maintainer API judgment.

## What This Changes

The issue proposes changing plugin slot values from bare strings to machine-readable ownership records with owner/provenance metadata.

## Best Possible Solution

Have maintainers decide the minimal slot ownership contract first, then stage any accepted implementation through schema/types, read normalization, docs/tests, canonical writes, and explicit overwrite semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new public config/plugin contract. Source inspection gives high-confidence current-main proof that slots are still typed, validated, normalized, written, tested, and documented as string selectors.

## Solution Assessment

Unclear until maintainer design review. The staged rollout is plausible, but the field set, provenance guarantees, migration behavior, and overwrite policy are public API choices rather than a safe automated cleanup fix.
