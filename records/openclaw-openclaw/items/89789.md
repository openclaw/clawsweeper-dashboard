## What This Changes

The branch adds a media-runtime guard around document extractor metadata reads and a regression test for skipping a bad extractor row while preserving method receiver binding.

## Best Possible Solution

Fold or sequence the loader-level descriptor isolation with this runtime guard so the production plugin artifact path and the media dispatch loop are both covered before merge.

## Reproduction Assessment

Partly. Current main would throw if resolvePluginDocumentExtractors returned an entry with a throwing mimeTypes getter, but the production bundled-plugin path reads descriptor metadata in the public-artifact loader before the media runtime sees the entry.

## Solution Assessment

No. The runtime guard is useful defense-in-depth, but the best fix must also isolate malformed descriptors in the loader path or land this with the sibling loader fix.
