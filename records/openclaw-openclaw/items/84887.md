## Summary

Keep open: current `main` and `v2026.5.19` still have the raw provider/model interpolation that can produce doubled provider prefixes, and the linked closing pull request is still open.

## What This Changes

The issue asks OpenClaw to normalize provider-qualified runtime LLM model references so allowlist checks and denial diagnostics do not show duplicated providers such as `openrouter/openrouter/...`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84934 or an equivalent scoped fix that uses a canonical model-ref formatter in runtime LLM policy checks and denial diagnostics, with regression coverage for provider-qualified OpenRouter refs.

## Reproduction Assessment

Yes for a source-level reproduction: current code normalizes provider/model pairs and then interpolates `${provider}/${model}`, which can duplicate a provider already present in the model id. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow canonical model-ref formatting change in the runtime LLM allowlist and diagnostic path. The Lossless provider-precedence bug is separately tracked upstream and should not be solved in OpenClaw core here.
