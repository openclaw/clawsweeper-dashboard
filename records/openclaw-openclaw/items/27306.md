## Summary

Current `main` still treats configured group targets as a multi-user heuristic, still recommends `agents.defaults.sandbox.mode="all"` for intentional shared-user access, and does not expose a suppression/acknowledgement path for this warning. The underlying runtime capability to sandbox only non-main sessions already exists, but the specific request in #27306 is not implemented on `main`.

## Evidence

- **Non-main-only sandboxing exists:** The docs still document `agents.defaults.sandbox.mode: "non-main"`, which sandboxes only non-main sessions such as groups/channels while leaving main direct. That means the runtime part of the reporter's setup is already supported, but this does not address the warning itself.
