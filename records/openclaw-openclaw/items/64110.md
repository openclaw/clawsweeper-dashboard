## What This Changes

The PR adds a cron job existence helper, pre-execution guards in timed and manual cron runs, and seam tests for jobs removed between reservation and execution.

## Best Possible Solution

Land a rebased cron-service fix that rechecks job existence after reservation but before side effects, handles timed reload failures as per-job outcomes, keeps focused seam coverage, and includes redacted real Feishu or cron runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can reserve a timed or manual cron run, remove the stored job before execution starts, and still execute the captured job snapshot because there is no fresh existence check before executeJobCoreWithTimeout.

## Solution Assessment

No. The pre-execution guard is the right narrow direction, but the timed guard must be handled inside per-job outcome logic and the branch needs rebase plus real runtime proof.

- **[P1] Handle job-existence reload failures inside runDueJob:** `src/cron/service/timer.ts:726`
  - body: The new pre-execution guard calls jobExists before runDueJob enters its try/catch. Wrap this check so failures become a per-job error outcome.
