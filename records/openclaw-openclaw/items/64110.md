## What This Changes

The PR adds cron job existence checks before timed and manual cron execution plus regression tests for jobs removed after scheduling or manual-run preparation but before execution.

## Best Possible Solution

Land a revised cron-service fix that rechecks persisted job existence after reservation but before delivery side effects, handles reload failures as per-job outcomes, and includes focused tests plus redacted real Feishu or cron proof.

## Reproduction Assessment

Yes. A high-confidence source path is to collect or prepare a cron job, delete the persisted job before execution starts, and observe current main executing the captured job snapshot without a fresh store-existence recheck.

## Solution Assessment

No. The pre-execution guard is the right narrow direction, but the timer-path existence check needs to be inside per-job handled outcome logic so reload failures do not abort the whole timer pass.
