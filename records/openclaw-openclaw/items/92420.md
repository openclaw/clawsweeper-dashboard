## What This Changes

The PR forwards `cfg` into Telegram automatic delivery and records each successful `deliverReplies` Telegram message id into the existing sent-message ledger.

## Best Possible Solution

Use this broader branch or a maintainer replacement as the canonical fix, then land it only after live Telegram proof or explicit maintainer acceptance of the proof gap.

## Reproduction Assessment

Yes from source: current main emits sent hooks from `deliverReplies` without writing to the sent-message ledger, and own-mode reactions read that ledger through `wasSentByBot(chatId, messageId, cfg)`. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes. The delivery-progress chokepoint is the best inspected fix shape because every successful send branch already marks delivery there, while duplicating writes inside low-level send helpers would drift from delivery accounting.
