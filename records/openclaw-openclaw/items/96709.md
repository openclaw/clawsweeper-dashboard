Full review comments:

- **[P2] Avoid retrying already-sent overflow batches:** `extensions/discord/src/outbound-payload.ts:116-126`
  - body: This wraps one logical send in `withRetry`, but `sendMessageDiscord` can now perform several Discord POSTs for overflow media. If a later batch gets a retryable 429/5xx after an earlier batch succeeded, `retryAsync` reruns the callback and duplicates the successful attachments; move retry inside each batch or make overflow sends idempotent.
  - confidence: 0.88
- **[P2] Wire mediaUrls through Discord actions:** `extensions/discord/src/send.outbound.ts:41`
  - body: The new option is added only to the low-level send path, while the public Discord send action/runtime still read scalar `mediaUrl`/`path`/`filePath` and pass only `mediaUrl`. User-facing multi-attachment sends still cannot pass ordered arrays through that route unless the action path parses and forwards `mediaUrls` with tests.
  - confidence: 0.86
- **[P2] Apply reply references per overflow platform send:** `extensions/discord/src/send.shared.ts:491-496`
  - body: `sendDiscordMediaBatch` creates one Discord message per overflow batch but passes the same `replyTo` into every batch. That bypasses the existing single-use reply fanout behavior, so `replyToMode: first` can thread every overflow message to the source instead of only the first.
  - confidence: 0.78
