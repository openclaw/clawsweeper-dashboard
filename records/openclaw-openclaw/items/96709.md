Full review comments:

- **[P2] Move overflow retry inside each Discord POST:** `extensions/discord/src/outbound-payload.ts:188-192`
  - body: This wraps a full logical send in delivery retry, but that send can now create multiple Discord messages for overflow batches. If a later batch gets a retryable 429/5xx after an earlier batch succeeded, retrying this callback can duplicate attachments that Discord already accepted.
  - confidence: 0.88
- **[P2] Forward mediaUrls from the public send actions:** `extensions/discord/src/send.outbound.ts:41`
  - body: Adding `mediaUrls` only to the low-level send options leaves the public Discord `send` and `sendMessage` action paths reading scalar `mediaUrl`/`path`/`filePath` and forwarding only `mediaUrl`. Users invoking the documented action path still cannot send ordered attachment arrays through this feature.
  - confidence: 0.86
- **[P2] Apply reply references only to the first overflow send:** `extensions/discord/src/send.shared.ts:496`
  - body: Every overflow batch uses the same `replyTo`, bypassing the existing reply fanout behavior that resolves implicit replies once per platform send. With `replyToMode` values that should target only the first send, overflow media can all thread to the source message instead of only the first batch.
  - confidence: 0.8
