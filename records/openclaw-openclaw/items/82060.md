## Summary

Keep open: current main still hardcodes `NO_REPLY` for generic pre-reply runner failures in group/channel contexts before consulting the documented `silentReply` policy. The issue is already paired with two open closing PRs, so the issue should stay open until one fix is reviewed and merged.

## What This Changes

The issue asks OpenClaw to honor existing `agents.defaults.silentReply` and `surfaces.<id>.silentReply` policy overrides when choosing failure-fallback text for group/channel conversations.

## Best Possible Solution

Review and land one focused fix that passes the existing config/session/surface/conversation type into failure-fallback policy resolution, preserves default group/channel silence, and adds regression coverage for an explicit group disallow override.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: a generic runner failure in a group/channel `TemplateContext` returns `SILENT_REPLY_TOKEN` before any existing `silentReply` policy lookup, while docs and tests establish `silentReply.group: "disallow"` as a valid override.

## Solution Assessment

Yes. Reusing `resolveSilentReplyPolicy` with the explicit group/channel conversation type is the narrow maintainable fix; the issue should remain open while the linked PRs are reviewed rather than being closed before merge.
