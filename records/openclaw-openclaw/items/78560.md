## Summary

Keep open: current main does not implement standalone browser-native SpeechRecognition dictation, and the proposal is a new Control UI voice mode that needs maintainer product/UX review rather than conservative cleanup.

## What This Changes

Requests a built-in Control UI microphone button using the browser SpeechRecognition API, with short-click auto-send dictation and long-hold continuous conversation.

## Best Possible Solution

Decide the intended Control UI voice-input model first, then implement it in `ui/src` as a tested Lit/source-level feature or add a dedicated plugin UI seam before accepting composer injection.

## Reproduction Assessment

Not applicable; this is an enhancement request, not a broken current behavior report. Source inspection shows current main intentionally has Talk controls but no browser dictation control.

## Solution Assessment

No; adding a standalone script under built `dist/control-ui` would bypass the Vite/Lit source and current tests. A maintainable fix should be source-first or use a defined Control UI contribution API, with explicit policy for SpeechRecognition behavior.
