## What This Changes

This PR updates Feishu `feishu_doc` create to accept optional initial markdown content, write it after document creation, document the behavior, add regression tests, and add a changelog entry.

## Best Possible Solution

Land a repaired Feishu plugin change that writes create-time markdown through the existing `writeDoc` path while preserving created metadata on partial failure, or explicitly rejects create-time content if maintainers choose not to support it.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: call `feishu_doc` with `action: "create"`, `title`, and `content`; the dispatcher returns from `createDoc` without writing or rejecting the content.

## Solution Assessment

No, not as submitted. Create-plus-write is the narrow maintainable direction, but the branch should preserve created document metadata or roll back if the follow-up write fails, and it still needs real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect
