## Summary

Keep open: this PR addresses a source-reproducible Feishu document bug that is still present on current main, but the current branch still has a failure-path correctness issue and lacks required real Feishu behavior proof.

## What This Changes

The PR adds Feishu `feishu_doc` create-time `content` support by adding a create schema field, chaining `createDoc` to `writeDoc`, stripping `blocks_deleted`, and updating Feishu skill docs, tests, and changelog.

## Best Possible Solution

Land a repaired Feishu plugin fix that implements create-plus-write while preserving created metadata on partial write failure, backed by regression tests and redacted live Feishu proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: call `feishu_doc` create with `title` and `content`; the dispatcher reaches the create branch and returns after `createDoc` without writing or rejecting the content.

## Solution Assessment

No, not as submitted. Create-plus-write is the narrow maintainable direction, but the PR should preserve the created document id/url when the follow-up write fails and still needs live Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve created document metadata on write failure:** `extensions/feishu/src/docx.ts:1481-1487`
  - body: The new create-with-content path creates the document before awaiting `writeDoc`, but any clear/convert/insert/image failure falls through to the outer catch and returns only `{ error }`.
