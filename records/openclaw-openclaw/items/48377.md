## Summary

Keep open: the PR targets a real Feishu plugin bug that current main still has, but the branch is not merge-ready because it lacks live Feishu proof and still drops the created document metadata if the follow-up write fails.

## What This Changes

The PR changes Feishu `feishu_doc` create to accept optional `content`, write it after document creation, omit `blocks_deleted`, and updates schema, skill docs, tests, and changelog.

## Best Possible Solution

Keep this as the canonical Feishu plugin fix or replace it with an equivalent branch that preserves created-document metadata on write failure, resolves conflicts, attaches redacted live proof, and then lands to close the linked issue.

## Reproduction Assessment

Yes. Source inspection on current main shows a `feishu_doc` create call with `content` reaches the create branch and returns after `createDoc`, while the provider schema normalization can still expose `content` from other variants.

## Solution Assessment

No, not as submitted. Create-plus-write is the right narrow direction, but the branch should preserve the created document id/url when the follow-up write fails and needs real Feishu proof before merge.
