## Summary

Keep open: this PR addresses a source-reproducible Feishu bug on current main, but the head still loses created-document recovery details if the post-create write fails and lacks redacted live Feishu proof.

## What This Changes

The PR updates `feishu_doc` create to accept optional initial markdown content, write it after document creation, strip deleted-block counts, and adds matching schema, skill docs, tests, and changelog text.

## Best Possible Solution

Keep this as the canonical Feishu plugin fix, add recoverable failure behavior around the post-create write, attach redacted live Feishu proof, then land or replace it to close the linked issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `feishu_doc` with `action: "create"`, `title`, and `content`; current main reaches the create branch and ignores the supplied content instead of writing or rejecting it.

## Solution Assessment

No, not as submitted. Create-plus-write is the right narrow direction, but the branch should preserve the created document id/url when the follow-up write fails so callers can recover without creating duplicates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the created document when the write step fails:** `extensions/feishu/src/docx.ts:1481-1487`
  - body: In the new create-with-content path, `createDoc` succeeds before `writeDoc` runs, but any write failure falls through to the outer catch and returns only `{ error }`.
