## Summary

Keep open: current main still has the Feishu create path that ignores create-time content, and this PR is the paired implementation candidate, but the branch still needs real Feishu proof and a repair for the post-create write-failure path before merge.

## What This Changes

The PR updates Feishu `feishu_doc` create to accept optional initial markdown content, write it via `writeDoc`, document the behavior, add regression tests, and add a changelog entry.

## Best Possible Solution

Land a repaired Feishu plugin change that either writes create-time markdown through the existing `writeDoc` path while preserving created metadata on partial failure, or explicitly rejects unsupported create-time content; include focused tests and redacted live Feishu proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: call `feishu_doc` with `action: "create"`, `title`, and `content`; the dispatcher returns from `createDoc` without writing or rejecting the content.

## Solution Assessment

No, not as submitted. Create-plus-write is the narrow maintainable direction, but the branch should preserve created document metadata or roll back if the follow-up write fails, and it still needs real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve created document metadata on write failure:** `extensions/feishu/src/docx.ts:1481-1487`
  - body: The new create-with-content path creates the Feishu document before awaiting `writeDoc`. Catch the write failure inside this branch and include the created metadata with the write error, or roll back if Feishu supports it.
  - confidence: 0.87
