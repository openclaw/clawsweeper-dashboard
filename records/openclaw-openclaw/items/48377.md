## Summary

Keep open: current main and the latest release still ignore create-time Feishu document content, so the PR remains relevant, but the branch is not merge-ready because it lacks real Feishu proof and still loses created document metadata if the follow-up write fails.

## What This Changes

The branch adds create-time `content` support to the Feishu document tool and updates the schema, Feishu skill docs, regression tests, and changelog.

## Best Possible Solution

Keep this as the canonical Feishu plugin fix or replace it with an equivalent branch that preserves created-document metadata on write failure, resolves conflicts, attaches redacted live proof, and then lands to close the paired issue.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: call `feishu_doc` create with `title` and `content`; the dispatcher reaches the create branch and returns after `createDoc` without writing or rejecting the supplied content.

## Solution Assessment

No, not as submitted. Create-plus-write is the right narrow direction, but the branch should preserve the created document id/url when the follow-up write fails and needs real Feishu proof before merge.
