## Summary

Keep this PR open. The latest head appears to address the earlier account-inheritance blocker and the patch is focused, but the 24-hour opt-in Discord callback lifetime is a security-boundary/product contract that should get maintainer acceptance before merge.

## What This Changes

Adds Discord `agentComponents.ttlMs` config/schema/docs/generated metadata and forwards the resolved value into component registry registration for sends and edits.

## Best Possible Solution

Land this implementation if maintainers accept the bounded 24-hour callback lifetime; otherwise lower the maximum and update the schema, docs, metadata, tests, and proof together.

## Solution Assessment

Yes, conditionally: reusing the existing registry TTL plumbing and preserving defaults is the narrow implementation shape. The remaining decision is whether the 24-hour maximum is the right public bound.
