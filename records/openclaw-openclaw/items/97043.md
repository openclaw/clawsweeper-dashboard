## Summary

Keep open: current main and v2026.6.10 have the lower Gateway `plugin.approval.request` RPC and hook-owned `before_tool_call.requireApproval`, but not the requested standalone `OpenClawPluginApi` facade for plugin workflow checkpoints.

## What This Changes

The issue requests a new public Plugin SDK `api.approval.request()` method for approval prompts outside `before_tool_call` tool gating.

## Best Possible Solution

Define a narrow additive SDK facade over the existing Gateway plugin approval flow, with explicit context binding, cancellation/no-route behavior, docs, API baseline updates, and focused Gateway/SDK/hook tests.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.10 have Gateway `plugin.approval.request` plus hook `requireApproval`, but `OpenClawPluginApi`, API construction, and public docs do not expose a standalone `api.approval.request()` method.

## Solution Assessment

Unclear pending maintainer design. The existing Gateway RPC is the right lower layer, but the public SDK facade needs an approved API shape for context binding, cancellation, timeout, no-route, and security semantics.
