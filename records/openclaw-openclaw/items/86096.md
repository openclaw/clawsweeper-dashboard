Full review comments:

- **[P2] Parse broker DM aliases before sending:** `extensions/channel-broker/src/target.ts:107-116`
  - body: The new inference path treats `broker:slack:user:U123` and `broker:discord:dm:...` as direct chats, but `parseChannelBrokerTarget` only recognizes `direct|group|channel|thread`. The actual outbound request and session route therefore keep `user:`/`dm:` in the provider conversation id and use the default channel type, so the added DM syntax routes differently when delivered. Normalize the same aliases here before building routes or provider requests.
  - confidence: 0.91
- **[P2] Package the bundled broker runtime:** `package.json:46`
  - body: This PR presents `channel-broker` as a bundled/private channel, but adding `!dist/extensions/channel-broker/**` puts it on the exclusion path that `listBundledPluginPackArtifacts` filters out. Since the new package is `private` and has no `openclaw.install` npm metadata, published OpenClaw installs would not contain a discoverable broker runtime; either remove the exclusion or make it an external official plugin with catalog/install metadata.
  - confidence: 0.82
