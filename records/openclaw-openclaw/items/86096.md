Full review comments:

- **[P2] Declare only implemented durable broker hooks:** `extensions/channel-broker/src/channel.ts:85-87`
  - body: Core treats `reconcileUnknownSend` as special: if an adapter declares that capability, `resolveOutboundDurableFinalDeliverySupport` requires `durableFinal.reconcileUnknownSend` to be a function. This adapter sets `reconcileUnknownSend: true` and also advertises `afterCommit`, but only provides `send.text`/`send.media`, so durable broker sends that require recovery still fail capability resolution instead of using the new channel. Either implement the hook with focused tests or stop declaring it in Phase 1.
