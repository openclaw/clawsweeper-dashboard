## Summary

Keep open. Current main still keeps gateway agent delivery explicit-only, and this issue is paired with a live reporter PR using closing syntax, so cleanup closure would be premature.

## What This Changes

The issue asks for Telegram group-bound agent sessions to default to visible delivery when `deliver` is omitted while preserving explicit session-only overrides.

## Best Possible Solution

Decide the Telegram group default-delivery policy, then land a corrected narrow helper with focused gateway/outbound tests or close the linked PR if maintainers reject that default.

## Reproduction Assessment

Yes. Source inspection is high-confidence: a gateway `agent` request with a Telegram group/topic `sessionKey` and omitted `deliver` flows through `wantsDelivery = false` and dispatches with `deliver = false`; I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers accept the visible-delivery default. If accepted, the helper approach is narrow, but the linked PR should be reviewed or ported rather than landed blindly.
