## Summary

## What This Changes

Request to change Telegram group/topic-bound Gateway agent-session behavior so omitted delivery does not leave successful runs invisible, now framed as a privacy-safe suppressed-final diagnostic or policy decision.

## Best Possible Solution

Maintainers should choose the privacy contract first, then land one narrow Gateway/source-reply diagnostic or explicit opt-in policy that preserves `deliver: false`, internal-channel suppression, and strict private shared-room defaults.

## Reproduction Assessment

## Solution Assessment

No for the original blind auto-delivery direction. The safer path is a maintainer-approved diagnostic or explicit opt-in policy that lets the agent or operator recover without automatically publishing private finals into shared rooms.
