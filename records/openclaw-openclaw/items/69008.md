## Summary

Keep open: current main still requires explicit `deliver === true`, and the requested Telegram group/topic default is a visible-delivery policy change with an open fork implementation rather than cleanup already implemented on main.

## What This Changes

Request default external delivery for Telegram group-bound agent sessions when `deliver` is omitted, while preserving explicit `deliver: false` and unrelated session behavior.

## Best Possible Solution

Maintainers should decide whether a Telegram group-bound session key is enough visible-delivery intent; if yes, land one shared delivery-flag helper across gateway/CLI paths with focused tests and live Telegram proof.

## Reproduction Assessment

Yes, by source inspection: current gateway and CLI delivery paths both require explicit `deliver === true`, so omitted `deliver` remains session-only. I did not run live Telegram because this was a read-only review.

## Solution Assessment

Unclear. The proposed helper is narrow, but changing the visible-delivery default for Telegram group-bound sessions should first be reconciled with the documented group visible-reply policy.
