## Summary

Keep open: current main still requires explicit `deliver === true`, and the requested Telegram group default would change documented visible-reply policy. The issue is also paired with an open linked implementation PR, so it should stay open until that path is reviewed, merged, or rejected.

## What This Changes

Default Telegram group-bound agent sessions to visible external delivery when `deliver` is omitted, while preserving explicit `deliver: false` and non-Telegram/non-group session behavior.

## Best Possible Solution

Maintainers should decide whether a Telegram group-bound session key is sufficient visible-delivery intent; if yes, land one shared delivery-flag helper with focused gateway/CLI tests and live Telegram proof.

## Reproduction Assessment

Yes, by source inspection: current gateway and CLI paths both require an explicit `deliver === true`, so omitted `deliver` remains session-only. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed helper is narrow, but it changes the visible-delivery default for Telegram group-bound sessions and should be reconciled with the existing group visible-reply policy before implementation.
