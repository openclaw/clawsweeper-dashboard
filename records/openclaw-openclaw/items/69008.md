## Summary

Keep open. Current main still makes gateway agent delivery explicit-only, so omitted `deliver` stays session-only even for Telegram group/topic session keys, and the linked reporter implementation candidate should be reviewed, landed, closed, or ported before this issue closes.

## What This Changes

The issue requests default external delivery for Telegram group/topic-bound agent sessions when `deliver` is omitted, while preserving explicit `deliver:false`, internal-channel suppression, and unrelated session behavior.

## Best Possible Solution

Add or land a small gateway-local delivery flag helper that treats only Telegram group/topic session keys as default-visible when `deliver` is omitted, keeps `deliver:false` authoritative, and covers target plus non-target session cases in focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: send a gateway `agent` request with `sessionKey` shaped like `agent:main:telegram:group:-100123` or `agent:main:telegram:group:-100123:topic:99` and omit `deliver`; current main computes both delivery intent and final dispatch delivery as false.

## Solution Assessment

Yes. The proposed gateway helper is the narrowest maintainable direction because the decision is made before outbound dispatch and can be tested without changing Telegram DM, webchat, main-session, Discord group, or internal-channel semantics.
