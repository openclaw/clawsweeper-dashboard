## Summary

Keep open. Current main still makes gateway agent delivery opt-in with `deliver: true`, while the requested Telegram group-bound default would change externally visible group-message policy and already has a linked fork implementation candidate that needs maintainer judgment.

## What This Changes

Request that Telegram group-bound agent session keys default to visible external delivery when `deliver` is omitted while preserving explicit `deliver: false`.

## Best Possible Solution

Decide the Telegram group visible-delivery policy first, then either land a narrow helper with focused tests and real Telegram proof or close the linked fork PR if the default is rejected.

## Reproduction Assessment

Yes for the reported current behavior from source inspection: with `deliver` omitted, `request.deliver === true` is false and the dispatched run receives `deliver: false`. I did not perform a live Telegram delivery check.

## Solution Assessment

Unclear. The proposed helper is mechanically narrow, but making Telegram group-bound session runs visible by default is a policy exception to the current group visible-reply model.
