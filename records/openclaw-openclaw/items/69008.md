## Summary

Keep open: current main still treats agent delivery as enabled only when `deliver` is explicitly true, and defaulting Telegram group-bound sessions to visible delivery is a Telegram delivery-policy change rather than cleanup already implemented on main.

## What This Changes

Telegram group-bound agent sessions are requested to default to external delivery when `deliver` is omitted while preserving `deliver: false` as session-only.

## Best Possible Solution

Maintainers should decide whether a Telegram group-bound session key is sufficient visible-delivery intent; if yes, implement one shared delivery-flag helper with focused gateway/CLI regression tests and live Telegram proof.

## Reproduction Assessment

Yes, by source inspection: current gateway and CLI paths both require an explicit `deliver === true`, so omitted `deliver` remains session-only. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed helper is narrow, but changing the visible-delivery default for Telegram group-bound sessions should first be reconciled with the existing group visible-reply policy.
