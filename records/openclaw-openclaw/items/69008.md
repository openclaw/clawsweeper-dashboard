## Summary

Keep open. Current main still treats gateway agent delivery as explicit-only, the requested Telegram group default is an externally visible policy change, and a live reporter fork PR with closing syntax already targets this behavior.

## What This Changes

Request default external delivery for Telegram group/topic-bound agent session keys when `deliver` is omitted while preserving explicit `deliver:false` and other session types.

## Best Possible Solution

Resolve the Telegram visible-delivery policy, then either land a reviewed narrow helper with focused tests or close the linked fork PR if maintainers reject the default.

## Reproduction Assessment

Yes for the reported current behavior. Source inspection on current main shows omitted `deliver` becomes `wantsDelivery = false` and the dispatched run receives `deliver = false`; I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear. The helper direction is narrow if maintainers want this default, but docs and Telegram maintainer notes make the default-visible group behavior a policy decision before code should land.
