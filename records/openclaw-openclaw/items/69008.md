## Summary

Keep open. Current main and the latest release still use explicit-only agent delivery, but the requested Telegram group default would change the documented and maintainer-noted group visible-reply policy, so this needs maintainer policy review rather than cleanup closure.

## What This Changes

Request that Telegram group-bound agent session keys default to visible external delivery when `deliver` is omitted, while preserving `deliver: false` and excluding other session shapes.

## Best Possible Solution

Maintainers should decide whether a Telegram group-bound session key should imply visible delivery; if accepted, land a narrow helper with focused tests and real Telegram group/topic proof, otherwise keep the documented explicit delivery or `message.send` path.

## Reproduction Assessment

Yes by source inspection: omitted `deliver` becomes `wantsDelivery = false`, and final delivery also requires `request.deliver === true`. I did not run a live Telegram send.

## Solution Assessment

Unclear. The proposed helper is technically narrow, but it changes the group visible-reply contract, so the best solution depends on maintainer policy approval and live Telegram proof.
