## Summary

Keep open. Current main still makes gateway agent delivery explicit-only, while Telegram group and topic session keys remain real routed session shapes; the linked reporter implementation candidate should be reviewed, ported, landed, or closed before this issue is swept.

## What This Changes

Request default-visible delivery for Telegram group-bound agent sessions when `deliver` is omitted, while preserving explicit session-only overrides and non-target session behavior.

## Best Possible Solution

Decide the default-delivery policy, then land or cleanly port a narrow gateway helper with focused tests for Telegram group/topic keys and explicit non-target exclusions.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: on current main, a gateway `agent` request with a Telegram group/topic `sessionKey` and omitted `deliver` flows through `wantsDelivery = false` and dispatches with `deliver = false`. I did not run a live Telegram send because this review is read-only.

## Solution Assessment

Unclear until maintainers accept the policy change. If accepted, the helper direction is the narrowest maintainable fix because it keeps the decision in the gateway delivery path and can preserve explicit `deliver:false`, internal-channel suppression, and non-Telegram session behavior.
