## Summary

Keep open: current main still treats agent delivery as explicit opt-in, while the current visible-reply policy intentionally keeps message-tool-only finals private and log-only. The latest discussion narrows this to a maintainer product/privacy decision for warning or self-repair rather than a safe autonomous cleanup.

## What This Changes

The issue asks OpenClaw to stop leaving Telegram group/topic-bound agent session replies only in session history when `deliver` is omitted, now narrowed toward an agent-visible suppressed-final diagnostic or explicit delivery policy.

## Best Possible Solution

Keep the issue open for a maintainer policy decision: preserve strict-private behavior by default, add an agent-visible suppressed-final diagnostic if approved, and only add auto-repair or policy modes behind an explicit product choice.

## Reproduction Assessment

Yes, at source level: current gateway and CLI delivery both require `deliver === true`, and message-tool-only source replies suppress normal final text. I did not run a live Telegram group/topic reproduction against current main.

## Solution Assessment

Unclear as originally proposed: auto-delivering private finals is not the safest default under the current policy. A maintainer-approved agent-visible diagnostic or explicit policy mode is the safer path before changing delivery behavior.
