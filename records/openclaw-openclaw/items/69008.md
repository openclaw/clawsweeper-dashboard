## Summary

Keep open: current main still makes gateway and CLI agent delivery depend on explicit `deliver === true`, and changing Telegram group-bound sessions to post by default is a visible Telegram delivery-policy decision rather than already-implemented cleanup.

## What This Changes

Default Telegram group-bound agent session runs to visible delivery when `deliver` is omitted, while keeping `deliver: false` session-only and avoiding unrelated session types.

## Best Possible Solution

Maintainers should decide whether a Telegram group-bound session key is explicit enough delivery intent; if yes, implement a small shared delivery-flag helper with focused gateway/CLI regression coverage and live Telegram proof.

## Reproduction Assessment

Yes for the current behavior by source inspection: both gateway and CLI paths treat delivery as enabled only when `deliver` is explicitly true, so omitted `deliver` remains session-only. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed helper is narrow, but changing default visibility for Telegram group sessions should first be reconciled with the existing group/channel visible-reply policy and explicit-delivery contract.
