## Summary

Keep open: current main still requires explicit delivery for gateway and CLI agent runs, while the shipped group/channel fixes address normal source replies rather than this Telegram group-bound session default. The linked fork PR is still open, and the latest live observation shifts the remaining question toward privacy-sensitive delivery policy or agent-visible diagnostics, so this needs maintainer review rather than cleanup close.

## What This Changes

The issue requests visible delivery by default for Telegram group or topic-bound agent sessions when `deliver` is omitted, while preserving explicit session-only behavior.

## Best Possible Solution

Settle the group-bound session visibility policy first, then either land a shared gateway/CLI helper that preserves explicit `deliver:false` and internal-channel suppression, or add an agent-visible suppressed-final warning coordinated with https://github.com/openclaw/openclaw/issues/80520 and https://github.com/openclaw/openclaw/issues/84327.

## Reproduction Assessment

Yes, at source level: current gateway and CLI paths both require `deliver === true`, so omitted delivery remains session-only. I did not run a live Telegram group/topic reproduction in this read-only review.

## Solution Assessment

Unclear. The proposed helper is narrow for the initial agent RPC/CLI default, but current Telegram policy makes visible group delivery privacy-sensitive; an agent-visible suppression diagnostic or explicit policy may be safer than unconditional auto-send.
