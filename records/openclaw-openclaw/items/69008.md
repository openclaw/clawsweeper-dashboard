## Summary

Keep open. Current main still requires explicit `deliver: true`, and existing docs plus Telegram maintainer guidance frame group visible replies as policy-controlled; the linked open fork PR is the right implementation candidate if maintainers accept the behavior.

## What This Changes

Request that Telegram group/topic-bound agent session keys default omitted `deliver` to external delivery while preserving explicit `deliver: false` and unrelated session behavior.

## Best Possible Solution

Decide the Telegram group visible-reply policy, then either land a narrow helper with focused tests and real Telegram proof or document that explicit delivery/message-tool sends remain required.

## Reproduction Assessment

Yes for the current behavior. Source inspection on current main shows omitted `deliver` becomes `wantsDelivery = false` and dispatches `deliver = false`; no live Telegram send was run.

## Solution Assessment

Unclear. The proposed helper is technically narrow, but changing omitted `deliver` into visible Telegram group sends crosses documented group visible-reply policy and should be approved before implementation.
