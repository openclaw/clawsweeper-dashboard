## Summary

Keep open: current main and v2026.6.8 still gate Gateway agent delivery on explicit `deliver === true`, while the later discussion correctly reframes this as a privacy-sensitive delivery-policy decision rather than a blind Telegram group auto-send bug.

## What This Changes

Requests changing Telegram group-bound agent-session delivery semantics so successful runs do not remain invisible when `deliver` is omitted, with later comments favoring a diagnostic or opt-in recovery path over default auto-delivery.

## Best Possible Solution

Keep the issue open until maintainers choose a privacy-preserving path, likely warning or opt-in recovery that does not default-publish private group/topic finals, then land that path with focused source and Telegram-visible proof.

## Reproduction Assessment

Yes at source level: current main still requires explicit `deliver === true`, and current docs/diagnostics show substantive message-tool-only finals can remain private with only an operator warning. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No for the original auto-delivery default: it would cross a documented privacy boundary for shared rooms. A maintainer-approved diagnostic or opt-in recovery policy is the safer solution shape.
