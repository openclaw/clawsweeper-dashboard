## Summary

Keep open. Current main still treats omitted `deliver` as session-only, while public group-chat docs and Telegram review guidance make visible group replies a policy-controlled behavior; the linked contributor fork PR is the implementation candidate if maintainers accept that policy change.

## What This Changes

Default Telegram group-bound agent sessions to external delivery when `deliver` is omitted, while preserving `deliver: false` as a session-only override.

## Best Possible Solution

Maintainers should decide the Telegram group visible-reply policy, then either land a narrow helper with focused tests and real Telegram proof from the linked fork PR or document that explicit delivery/message-tool sends remain required.

## Reproduction Assessment

Yes for the current default behavior: current main computes `wantsDelivery` and final `deliver` from `request.deliver === true`, so an omitted flag stays session-only. No live Telegram send was run.

## Solution Assessment

Unclear. The proposed helper is technically narrow, but changing omitted `deliver` into visible group sends crosses the documented visible-reply policy and should be approved before implementation.
