## Summary

Keep open: current main and v2026.6.8 still require explicit `deliver` for Gateway agent delivery and only log private `message_tool_only` finals; the issue also has an open linked same-author PR and now needs a privacy-sensitive product decision.

## What This Changes

The issue asks OpenClaw to prevent Telegram group/topic agent runs from completing invisibly when final text is persisted but no explicit delivery or message-tool send occurs.

## Best Possible Solution

Keep the issue open until maintainers choose a privacy-safe policy, then land or replace the linked PR with an agent-visible diagnostic, retry, or explicit opt-in mode that preserves the private default.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 still gate Gateway agent delivery on explicit `request.deliver === true`, and current `message_tool_only` final text is logged but kept private. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No for blind default auto-delivery: that conflicts with the documented privacy boundary for tool-gated shared rooms. A maintainer-approved diagnostic, retry, or opt-in policy is the safer solution shape.
