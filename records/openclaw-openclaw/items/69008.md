## Summary

Keep open. Current main still treats omitted `deliver` as session-only, while the documented Telegram/group visible-reply policy keeps normal group replies private unless configured automatic or sent through the message tool; the linked fork PR needs maintainer policy review before this behavior changes.

## What This Changes

Request default visible delivery for Telegram group-bound agent session keys when `deliver` is omitted while keeping `deliver: false` session-only.

## Best Possible Solution

Make an explicit maintainer policy decision, then either keep the documented message-tool/automatic configuration behavior or land a narrow reviewed Telegram group defaulting change with focused tests and real Telegram group/topic proof.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows omitted `deliver` becomes `wantsDelivery = false` and final `deliver = false`. I did not run a live Telegram send.

## Solution Assessment

Unclear. The proposed helper is narrow, but it changes documented group visible-reply policy, so the safest path is maintainer review of the policy and linked implementation PR.
