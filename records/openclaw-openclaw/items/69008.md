## Summary

Keep open: current main still requires explicit delivery for gateway agent runs, while the related message-tool-only work settled on strict private finals plus operator logging rather than the requested default or an agent-visible recovery path.

## What This Changes

Request to make Telegram group/topic-bound agent sessions deliver visible replies by default when `deliver` is omitted, while preserving explicit `deliver:false` and unrelated session types.

## Best Possible Solution

Keep the strict private default unless maintainers approve a narrow Telegram group-bound delivery default or an agent-visible suppressed-final warning, then land it with focused tests and live Telegram group/topic proof.

Yes, source-level: current main gates gateway delivery on explicit `deliver === true`, and current tests/docs confirm message-tool-only group finals stay private. I did not run live Telegram in this read-only review, but the latest comment adds a 2026.5.18 live Telegram topic observation.
