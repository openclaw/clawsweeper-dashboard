## Summary

Keep open. Current main still uses explicit-only gateway delivery, and making omitted `deliver` visible for Telegram group-bound sessions would change the documented group visible-reply policy, so this needs maintainer policy review rather than cleanup closure.

## What This Changes

Default agent runs in Telegram group-bound sessions to external delivery when `deliver` is omitted, while preserving `deliver: false` and leaving unrelated session types unchanged.

## Best Possible Solution

Maintainers should decide whether Telegram group-bound session keys should override the group visible-reply contract, then either land the linked narrow helper with tests and live proof or keep explicit `deliver`/`message.send` as the documented behavior.

## Reproduction Assessment

Yes for the reported current behavior by source inspection: omitted `deliver` becomes `wantsDelivery = false` and dispatch receives `deliver = false`. I did not run a live Telegram send.

## Solution Assessment

Unclear. The proposed helper is narrow technically, but it changes the current group visible-reply policy, so the maintainable solution depends on maintainer approval and live Telegram group/topic proof.
