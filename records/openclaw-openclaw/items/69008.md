## Summary

Keep open. Current main still treats Gateway agent delivery as explicit-only, the documented/maintainer policy keeps visible provider delivery opt-in, and a live reporter PR with closing syntax already targets the requested behavior.

## What This Changes

Request that Telegram group/topic-bound agent sessions deliver externally by default when `deliver` is omitted while preserving explicit `deliver: false` and unrelated session behavior.

## Best Possible Solution

Resolve the Telegram visible-delivery policy, then either land a reviewed narrow helper with focused tests or close the linked fork PR if maintainers reject the default.

## Reproduction Assessment

Yes for the reported current behavior. Source inspection on current main shows omitted `deliver` becomes `wantsDelivery = false` and the dispatched run receives `deliver = false`; I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear. The helper direction is narrow if maintainers want this default, but maintainer notes currently frame group/channel visible replies as policy-controlled, so the policy should be decided before landing.
