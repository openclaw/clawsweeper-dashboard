## Summary

Keep open: current main still treats agent RPC and CLI delivery as explicit opt-in, while the shipped group/channel visible-reply fix only restored automatic source replies for normal inbound group/channel turns. The requested Telegram group/topic-bound session default remains a privacy-sensitive delivery policy decision, with an open fork implementation and overlap with the broader Telegram no-sendMessage cluster at https://github.com/openclaw/openclaw/issues/80520.

## What This Changes

Request that Telegram group/topic-bound agent sessions deliver externally by default when `deliver` is omitted while preserving explicit `deliver: false` as session-only.

## Best Possible Solution

Settle the group-bound session visibility policy first, then either land a shared gateway/CLI helper that preserves explicit `deliver:false` and internal-channel suppression, or add an agent-visible suppressed-final warning/policy coordinated with https://github.com/openclaw/openclaw/issues/80520.

## Reproduction Assessment

Yes, at source level: current gateway and CLI paths both require `deliver === true`, so omitted delivery remains session-only. The v2026.5.18 Telegram observation was not live-reproduced in this read-only review and may overlap broader no-send paths.

## Solution Assessment

Unclear. The proposed helper is narrow for the agent RPC/CLI default, but current docs and maintainer notes make this a visible-delivery policy decision; an agent-visible suppression diagnostic may be safer than auto-sending private final text.
