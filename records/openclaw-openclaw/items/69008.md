## Summary

Keep open. Current main still keeps gateway agent delivery explicit-only, and the requested Telegram group default would change the documented group visible-reply policy, so this needs maintainer policy review rather than conservative cleanup closure.

## What This Changes

Default Telegram group-bound agent and topic sessions to external delivery when `deliver` is omitted, while preserving `deliver: false` and avoiding unrelated session types.

## Best Possible Solution

Maintainers should decide whether Telegram group-bound session keys are an intentional exception to group visible-reply policy; if accepted, land a current-base target-repo PR with focused tests and real Telegram group/topic proof, otherwise keep the explicit `deliver` or `message.send` contract.

## Reproduction Assessment

Yes for the reported current behavior by source inspection: omitted `deliver` keeps `wantsDelivery` false and final `deliver` false. I did not run a live Telegram send because the code path already establishes the session-only default on current main.

## Solution Assessment

Unclear. The proposed helper is technically narrow, but it changes the current group visible-reply policy, so the maintainable solution depends on maintainer approval plus live Telegram group and topic proof.
