## Summary

Keep open. Current main still treats gateway agent delivery as explicit-only, so omitted `deliver` remains session-only for Telegram group/topic session keys, and the issue is already paired with an open reporter PR that explicitly closes it.

## What This Changes

Default Telegram group-bound agent sessions to visible external delivery when `deliver` is omitted while keeping `deliver:false` and unrelated session types session-only.

## Best Possible Solution

Land or port the small gateway-local delivery flag helper with focused tests for Telegram group/topic defaults, explicit `deliver:false`, unrelated session keys, and internal-channel suppression.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: send a gateway `agent` request with `sessionKey` like `agent:main:telegram:group:-100123` or `agent:main:telegram:group:-100123:topic:99` and omit `deliver`; current main computes delivery intent and final dispatch delivery as false.

## Solution Assessment

Yes. The proposed gateway helper is the narrowest maintainable fix because this decision is made before outbound dispatch and can preserve existing non-Telegram, DM, webchat, main-session, and internal-channel behavior.
