## Summary

Keep open. Current main still treats omitted `deliver` as session-only, and changing Telegram group-bound sessions to visible delivery conflicts with the documented group visible-reply policy, so this needs maintainer policy review rather than cleanup closure.

## What This Changes

Requests that Telegram group/topic-bound agent sessions default to external delivery when `deliver` is omitted, while preserving `deliver: false` and other session types.

## Best Possible Solution

Decide the Telegram group visible-reply policy, then either land a narrow reviewed helper with focused tests and real Telegram proof or keep explicit `deliver`/message-tool delivery as the documented behavior.

## Reproduction Assessment

Yes for the reported current behavior: source inspection on current main shows omitted `deliver` becomes `wantsDelivery = false` and final dispatch uses `deliver = false`. I did not run a live Telegram send.

## Solution Assessment

Unclear. The proposed helper is technically narrow, but it changes current group visible-reply policy by making omitted `deliver` visible for Telegram group-bound sessions.
