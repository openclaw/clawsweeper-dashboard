## Summary

Keep open. Current main and the latest release still use explicit-only gateway delivery, but making Telegram group-bound sessions visibly send by default is a policy-sensitive behavior change and the reporter already has an open fork PR with closing syntax targeting it.

## What This Changes

Request that Telegram group-bound agent sessions default to visible Telegram delivery when `deliver` is omitted while preserving explicit `deliver: false` and unrelated session behavior.

## Best Possible Solution

Resolve the Telegram visible-reply policy, then land a narrow reviewed helper with focused gateway/outbound tests or close the fork PR if maintainers keep explicit delivery.

## Reproduction Assessment

Yes for the reported current behavior. Source inspection on current main shows omitted `deliver` becomes `wantsDelivery = false` and the dispatched run receives `deliver = false`; no live Telegram send was run in this read-only review.

## Solution Assessment

Unclear. The proposed helper is technically narrow, but default-visible group delivery changes Telegram privacy/policy semantics and should be decided before code lands.
