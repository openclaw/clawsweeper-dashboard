## Summary

Keep open. The PR targets a real current-main cron/runtime mismatch, but the proposed provider rewrite is too broad and would route mismatched model providers through an unrelated configured CLI backend. It needs a narrow repair before merge.

## What This Changes

The PR changes cron model selection to stamp `agentRuntime.id` as the session provider when it is a configured CLI backend, and adds regression tests for CLI runtime provider stamping.

## Best Possible Solution

Keep the PR open and narrow the fix so cron uses the same provider/runtime compatibility rule as interactive runs: stamp `claude-cli` only for compatible Anthropic selections, preserve unrelated provider overrides, cover both paths in tests, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: configure `agentRuntime.id: "claude-cli"` with an Anthropic cron model and current main returns `provider: "anthropic"`, so the executor does not enter the CLI path; the proposed patch should also be tested against a mismatched `openai/gpt-5.5` override.

## Solution Assessment

No. The direction is right for `anthropic` + `claude-cli`, but the implementation should be provider/runtime-pair aware rather than replacing every resolved provider with any configured CLI backend id.
