## Summary

Keep open. The PR is the active candidate for the linked web-tool refresh bug, but the current head still marks normal config-write refresh prepares as stripped, so deleting web-tool config can keep stale active web_search/web_fetch metadata.

## What This Changes

The PR adds web-tool metadata provenance to prepared secrets runtime snapshots and preserves prior active web-tool metadata for snapshots marked as stripped refreshes.

## Best Possible Solution

Preserve active web-tool metadata only for proven partial or stripped refreshes; canonical persisted config writes that delete web config must clear, with finalize-path regression coverage for both behaviors.

## Reproduction Assessment

Yes. Source inspection shows a persisted config write can pass canonical source config into the registered refresh handler, which marks the fast-path snapshot as stripped and preserves active web metadata.

## Solution Assessment

No. Explicit provenance is the right direction, but this implementation still assigns stripped provenance too broadly; the safer fix is to carry a real partial-refresh authority signal and test the finalize path.
