## Summary

Keep open. Current main still makes Telegram partial preview streaming show tool-progress lines by default, and the latest release tag preserves the same documented/tested behavior with `streaming.preview.toolProgress: false` or `streaming.mode: "off"` as opt-outs.

## What This Changes

Telegram-bound sessions should keep tool calls, working indicators, and raw tool output internal so users receive only the final assistant reply.

## Best Possible Solution

Maintainers should choose the Telegram privacy default for partial preview streaming. If final-only or no-tool-progress-by-default is desired, update the Telegram preview-progress default/resolver, keep an explicit opt-in for progress previews, and align dispatch tests, config hints, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and invoking `onToolStart` plus `onItemEvent`; the focused test currently expects the visible `Working…` progress preview.

## Solution Assessment

Unclear. Hiding tool progress by default is a narrow technical change, but it reverses documented released behavior; the safer path is maintainer product/privacy approval before choosing final-only, opt-in, group-sensitive, or sanitized previews.
