## Summary

Keep open. Current main and the latest release snapshot still intentionally render Telegram tool/progress lines into the visible partial-streaming draft preview by default; only the explicit final-only/off path and explicit `streaming.preview.toolProgress: false` configuration suppress them.

## What This Changes

The issue requests that Telegram-bound sessions hide intermediate tool execution, working indicators, and raw tool output so users receive only the final assistant reply.

## Best Possible Solution

Make an explicit maintainer decision for Telegram partial-streaming privacy: either default Telegram to final-only delivery, keep answer previews but make tool progress opt-in, or define a sanitized structured-progress policy. Then update the Telegram dispatch path, focused tests, docs/config hints, and changelog together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: dispatch Telegram with `streamMode: "partial"`, leave `streaming.preview.toolProgress` unset, invoke `onToolStart` and `onItemEvent`, and the existing test expects a visible `Working…` preview containing tool and command text.

## Solution Assessment

Unclear. `streaming.mode: "off"` and `streaming.preview.toolProgress: false` are documented workarounds, but the default partial-streaming behavior still exposes progress by design; the best fix depends on the maintainer-approved privacy default.
