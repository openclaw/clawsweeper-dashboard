## Summary

Keep open. Current main still renders Telegram tool-progress preview edits by default in partial streaming mode, and the remaining question is a privacy/product policy decision rather than a conservative cleanup close.

## What This Changes

Telegram-bound sessions should hide intermediate tool call progress/output and deliver only the final assistant reply.

## Best Possible Solution

Maintainers should choose the Telegram disclosure policy, then align the runtime default, focused Telegram tests, docs, config hints, and changelog around hidden-by-default or explicit opt-in behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: run Telegram dispatch with `streamMode: "partial"`, leave `streaming.preview.toolProgress` unset, trigger `onToolStart`/`onItemEvent`, and the checked-in test expects a visible `Working…` preview with tool and command text.

## Solution Assessment

Unclear. The reporter’s final-only expectation is privacy-preserving, but current main intentionally documents visible tool-progress previews by default with opt-out controls, so the best fix depends on maintainer policy.
