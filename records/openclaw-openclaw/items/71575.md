## Summary

Keep open. Current main still intentionally renders Telegram tool-progress lines in partial preview mode by default, and the remaining question is whether Telegram should change that privacy/default policy instead of relying on documented opt-outs.

## What This Changes

Hide Telegram intermediate tool-progress, working-status, and raw-output previews so only the final assistant reply is user-visible.

## Best Possible Solution

Maintainers should choose the Telegram privacy policy explicitly, then align runtime defaults, tests, docs, config UI hints, and changelog so tool progress is either hidden/final-only by default or clearly opt-in.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main: dispatch Telegram with `streamMode: "partial"`, leave `streaming.preview.toolProgress` unset, trigger tool progress callbacks, and the checked-in test expects a visible `Working…` preview containing tool and command text.

## Solution Assessment

Unclear. The current implementation is internally consistent with the documented opt-out model, but the reporter’s desired final-only default is a product/privacy decision rather than a narrow mechanical bug fix.
