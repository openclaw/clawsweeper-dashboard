## Summary

Keep open. Current main still renders Telegram tool-progress preview edits by default when partial preview streaming is active, and the latest release contains the same documented/tested behavior with opt-outs rather than the requested final-only default.

## What This Changes

Telegram-bound sessions should keep tool-call progress, working indicators, and raw tool output internal so users see only the final assistant reply.

## Best Possible Solution

Maintainers should choose the Telegram privacy default for partial preview streaming. If final-only or no-tool-progress by default is desired, update the Telegram preview-progress default/resolver, preserve explicit opt-in for progress previews, and align dispatch tests, config hints, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and invoking `onToolStart` plus `onItemEvent`; the focused test currently expects the visible `Working…` progress preview.

## Solution Assessment

Unclear. Hiding tool progress by default is technically narrow, but it reverses a documented released behavior; the safer path is maintainer product/privacy approval before changing the default or choosing a group-sensitive or sanitized alternative.
