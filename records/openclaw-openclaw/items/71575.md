## Summary

Keep open. Current main still intentionally renders Telegram tool/progress updates into the edited preview message by default when `streamMode` is `partial`; the non-streaming final-only path has been fixed, but the reported partial-streaming disclosure remains a product/privacy default decision rather than an obsolete issue.

## What This Changes

Request that Telegram-bound sessions hide intermediate tool-call progress, working indicators, and raw tool output so users receive only the final assistant reply.

## Best Possible Solution

Make an explicit maintainer privacy/default decision for Telegram preview streaming: either keep answer previews but make tool progress opt-in, default Telegram to final-only delivery, or document a sanitized structured-progress policy. Then align `extensions/telegram/src/bot-message-dispatch.ts`, focused Telegram tests, docs, config hints, and changelog in one narrow change.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: dispatch a Telegram turn with `streamMode: "partial"`, leave `streaming.preview.toolProgress` unset, invoke `onToolStart` and `onItemEvent`, and current tests expect a visible `Working…` preview containing tool and command text.

## Solution Assessment

Unclear. `streaming.mode: "off"` already provides final-only delivery, but the reported default/partial preview behavior is still intentionally enabled; the maintainable fix needs a product/privacy decision before a narrow code change.
