## Summary

Keep open. Current main still intentionally shows Telegram tool-progress preview lines by default during partial preview streaming, so the reported privacy behavior remains valid even though non-streaming `streaming.mode: "off"` now suppresses generic progress chatter.

## What This Changes

The issue asks Telegram delivery to hide intermediate tool calls, working indicators, and raw tool output so users receive only the final assistant reply.

## Best Possible Solution

Choose a maintainer-approved Telegram privacy default, then align the runtime default, focused Telegram tests, config hints, docs, and changelog so tool progress is hidden by default or explicitly opt-in.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: dispatch Telegram with `streamMode: "partial"`, leave `streaming.preview.toolProgress` unset, invoke `onToolStart` or `onItemEvent`, and the checked-in test expects a visible `Working…` preview with tool and command text.

## Solution Assessment

Unclear. The current implementation and docs intentionally preserve visible tool-progress previews by default, while the report expects final-only or hidden-by-default delivery; the best fix depends on maintainer policy for Telegram privacy versus live progress UX.
