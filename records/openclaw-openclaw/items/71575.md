## Summary

Keep open. Current main still renders Telegram tool-progress preview lines by default during partial streaming, while the documented final-only workaround is `streaming.mode: "off"`; changing that default is a privacy-sensitive product decision, not a conservative cleanup close.

## What This Changes

Request to prevent Telegram-bound sessions from showing intermediate tool/progress/raw-output messages and deliver only the final assistant reply.

## Best Possible Solution

Choose a maintainer-approved Telegram streaming privacy policy, then align the runtime default, focused Telegram tests, config UI hint, docs, and changelog so tool details are hidden by default or clearly opt-in.

## Reproduction Assessment

Yes. Dispatch Telegram with `streamMode: "partial"` and empty Telegram config, invoke `onToolStart`/`onItemEvent`, and the checked-in test expects a visible `Working…` draft update with tool and command text.

## Solution Assessment

Unclear. The technical fix is narrow, but current docs and tests intentionally preserve visible partial-stream tool progress by default, so the best solution depends on maintainer policy for privacy versus live progress UX.
