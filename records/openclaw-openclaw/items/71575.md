## Summary

Keep open. Current main and the latest release still show Telegram tool-progress lines by default during partial preview streaming, so the reported privacy behavior remains valid even though final-only `streaming.mode: "off"` now suppresses generic progress chatter.

## What This Changes

Telegram partial-streaming sessions should avoid showing intermediate tool/progress output in user-visible chat and deliver only the final assistant response.

## Best Possible Solution

Choose a maintainer-approved Telegram streaming policy, then align the runtime default, focused Telegram tests, config hints, docs, and changelog so tool details are hidden by default or explicitly opt-in.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: dispatch Telegram with `streamMode: "partial"`, leave `streaming.preview.toolProgress` unset, invoke the tool/progress callbacks, and the checked-in test expects a visible `Working…` preview with tool and command text.

## Solution Assessment

Unclear. The current implementation documents and preserves visible tool-progress previews by default, while the reporter expects final-only or default-hidden delivery; the best fix depends on maintainer policy for Telegram privacy versus live progress UX.
