## Summary

Keep open. Current main still renders Telegram tool/progress lines into the visible partial-streaming preview by default; `streaming.mode: "off"` and explicit `streaming.preview.toolProgress: false` are documented workarounds, but the reported default behavior remains unresolved.

## What This Changes

The issue asks Telegram-bound sessions to keep tool execution, working indicators, and raw tool output internal so users receive only the final assistant reply.

## Best Possible Solution

Pick a Telegram-scoped privacy policy, then align runtime, tests, docs, config hints, and changelog. If maintainers want previews without tool disclosure, make Telegram tool-progress preview opt-in while keeping `streaming.mode: "off"` as the final-only mode and avoiding a shared helper default change that would alter other channels.

## Reproduction Assessment

Yes. On current main, dispatch Telegram with `streamMode: "partial"`, leave `streaming.preview.toolProgress` unset, then invoke `onToolStart` and `onItemEvent`; the existing focused test asserts the visible `Working…` draft update.

## Solution Assessment

Unclear. The narrow implementation lever is known, but the best fix depends on whether maintainers want Telegram to default to final-only delivery, answer previews without tool progress, or the existing released progress-preview behavior.
