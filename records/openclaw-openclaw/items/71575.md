## Summary

Keep open. Current main and the latest release still make Telegram partial streaming show visible tool-progress preview edits by default, including `Working…`, tool names, command progress text, and potentially raw progress snippets. The related non-streaming PR was closed unmerged and did not resolve the partial-streaming default reported here.

## What This Changes

The issue requests that Telegram-bound sessions hide intermediate tool call status, working indicators, and raw tool output so users see only the final assistant reply.

## Best Possible Solution

Settle the Telegram privacy default explicitly: either make partial streaming hide tool progress unless users opt in, or choose a documented sanitized/group-sensitive alternative. Then update the Telegram dispatch resolver, focused tests, docs, config hints, and changelog together so preview behavior and user-facing configuration stay aligned.

## Reproduction Assessment

Yes. A source-level reproduction is clear: dispatch a Telegram turn with `streamMode: "partial"`, invoke `onToolStart` plus `onItemEvent`, and the current focused test expects the visible `Working…` preview update with tool and command text.

## Solution Assessment

Unclear. The issue's final-only request is technically available with `streaming.mode: "off"`, but making that or `toolProgress: false` the default changes released behavior; the safest solution is a maintainer decision followed by a narrow Telegram dispatch/config/docs update.
