## Summary

Keep open. Current main and the latest release still intentionally render Telegram partial-streaming tool progress into visible preview edits by default, and the focused Telegram test asserts the reported `Working…` plus tool/command text behavior. The report is valid, privacy-sensitive, and needs an explicit maintainer product decision about whether Telegram tool-progress previews should become opt-in, sanitized, or remain the documented default.

## What This Changes

The issue asks Telegram-bound sessions to hide intermediate tool-call progress, working indicators, and raw tool output so users receive only the final assistant reply.

## Best Possible Solution

Decide the Telegram privacy default explicitly, then update the Telegram dispatch path, shared/default config handling if needed, focused tests, docs, and changelog together. A conservative end state would hide tool-progress preview lines by default for Telegram partial streaming while preserving an opt-in setting for users who want live tool status.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: dispatch a Telegram turn with `streamMode: "partial"`, invoke `onToolStart` and `onItemEvent` with exec progress text, and the focused test expects the visible `Working…` draft update.

## Solution Assessment

Unclear. The issue’s final-only behavior is currently available with `streaming.mode: "off"`, and preview-without-tools is available with `streaming.preview.toolProgress: false`; changing the default is the narrow technical fix but needs a maintainer product/privacy decision first.
