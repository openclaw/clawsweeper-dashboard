## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process is launched with full `coerceEnv(process.env)` while current host exec strips dangerous inherited keys before running commands. This can expose provider tokens, package tokens, SSH agent sockets, proxy settings, and scanner-control env to an optional external binary; build a minimal sanitized scanner env and apply it to both gateway and node hooks.
  - confidence: 0.92
- **[P2] Require a side-effect-free tirith invocation:** `src/security/command-security.ts:44`
  - body: The wrapper calls default `tirith check` without an offline/no-daemon/no-update/no-audit contract. Upstream check can trigger background updates, daemon delegation, remote policy fetches, runtime threat queries, audit/session writes, and webhooks, which is too much side effect for a local pre-approval gate.
  - confidence: 0.9
- **[P2] Run the scan from the effective command cwd:** `src/security/command-security.ts:45-53`
  - body: `execFile` is called without `cwd`, and callers do not pass the resolved command workdir. Tirith policy discovery can therefore use OpenClaw's process cwd instead of the command's effective directory, missing project `.tirith` policies or loading the wrong policy before deciding allow, warn, or block.
  - confidence: 0.87
