## Summary

Keep open: the PR is a useful security-hardening proposal, but the patch introduces blocking security and config-contract issues and needs maintainer security/product review before merge.

## What This Changes

The PR adds optional tirith-backed pre-exec command scanning to gateway, sandbox, and node exec paths, threads `tools.exec.commandSecurity` through config, and adds unit/integration coverage.

## Best Possible Solution

Land a maintainer-approved pre-exec scanner seam that uses sanitized env, explicit cwd/offline behavior, documented config, generated baselines, changelog, and regression tests.

## Reproduction Assessment

Not applicable. This is a security-hardening feature PR rather than a current-main bug report, and source inspection confirms current main does not expose `tools.exec.commandSecurity`.

## Solution Assessment

No. The direction may be useful, but the patch should first use a least-privilege, cwd-aware, explicitly local/offline scanner invocation and then update the public config contract through the current exec architecture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The scanner runs before approval with a full copy of `process.env`, which can expose provider tokens and tirith policy/server credentials to an optional external binary. Build a minimal scanner env or reuse the existing host exec sanitizer, preserving only required values such as `PATH`, `HOME`, and explicit tirith settings; the node path has the same issue.
  - confidence: 0.9
- **[P2] Force tirith to run without daemon or background network paths:** `src/security/command-security.ts:44`
  - body: The wrapper invokes `tirith check` without `--no-daemon`, while upstream `check` can delegate to a running daemon and triggers background threat DB update logic. That contradicts the PR’s no-network/no-background-process claim, so OpenClaw should either force local/offline behavior or document and configure these paths explicitly.
  - confidence: 0.86
- **[P2] Run tirith from the effective command cwd:** `src/security/command-security.ts:45-49`
  - body: `execFile` is called without a `cwd`, but tirith discovers repo policy from its current directory. Commands executed with an explicit OpenClaw workdir can therefore ignore the matching project policy and produce incorrect blocks or warnings; thread the effective command cwd into the scanner options.
  - confidence: 0.8
- **[P2] Document and regenerate the new config surface:** `src/config/zod-schema.agent-runtime.ts:417-425`
  - body: This adds the user-facing `tools.exec.commandSecurity` schema, but the PR does not add config help, generated config docs baseline updates, public exec/security docs, or a changelog entry. Users and config tooling would see an undocumented and drifted config surface.
  - confidence: 0.86
