A maintainer-approved implementation should hook into the current exec gateway/node phase boundaries, run tirith with a least-privilege sanitized environment and correct cwd/network behavior, then update config help, generated schema artifacts, docs, tests, and changelog together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Sanitize the tirith child-process environment:** `src/agents/bash-tools.exec.ts:414`
  - body: The scanner is launched before approval with a copy of `process.env`, which can include provider tokens and other OpenClaw secrets that normal host exec sanitization strips before spawning commands. Pass a minimal or `sanitizeHostExecEnvWithDiagnostics`-derived env instead, preserving only the PATH/HOME/TIRITH settings tirith actually needs; the same issue appears in the node-host path.
  - confidence: 0.9
- **[P2] Make tirith network and daemon behavior explicit:** `src/security/command-security.ts:39`
  - body: The wrapper invokes `tirith check` without `--no-daemon` or another opt-out, while upstream `tirith check` can delegate to a daemon and run background threat-db/update or enrichment paths. That contradicts the PR’s no-network/no-cloud security claim, so either disable those paths for OpenClaw scanning or document/configure the behavior explicitly.
  - confidence: 0.82
- **[P2] Update the public config contract:** `src/config/zod-schema.agent-runtime.ts:416-424`
  - body: This adds a user-facing `tools.exec.commandSecurity` config object, but only wires types/zod/labels. Schema help, generated schema artifacts, docs, and the changelog still omit the new keys, so users and UI/config tooling will see a drifted contract.
  - confidence: 0.86
- **[P2] Run tirith from the command workdir:** `src/security/command-security.ts:42`
  - body: The wrapper does not pass a `cwd` to `execFile`, while tirith discovers policy from its current directory. Commands run from an explicit exec workdir can therefore ignore the matching project policy and produce false blocks/warnings; thread the effective cwd into the scanner where the host has one.
  - confidence: 0.78
