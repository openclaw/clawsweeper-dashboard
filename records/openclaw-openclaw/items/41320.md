## Summary

Keep open: current main still has no tirith-backed exec scanning, and this PR is a meaningful security-hardening feature, but it needs contributor real behavior proof plus fixes for scanner environment, side-effect, and cwd handling before merge.

## What This Changes

The PR adds an optional tirith-backed pre-exec scanner for exec commands, a new `tools.exec.commandSecurity` config object, gateway/sandbox/node hooks, and unit plus approval-flow tests.

## Best Possible Solution

A maintainer-approved design should add a cwd-aware, least-privilege, explicitly offline scanner seam on the current exec architecture, document the config contract, and include real OpenClaw plus tirith proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The scanner runs before approval with `{ ...coerceEnv(process.env) }`, so an optional `tirith` binary receives inherited credentials and host influence variables that the normal exec sanitizer strips. Build a minimal scanner env, or reuse the existing sanitizer while preserving only required values such as `PATH`, `HOME`, and explicit tirith settings; apply the same fix to the node hook.
  - confidence: 0.9
- **[P2] Disable tirith daemon and update side effects:** `src/security/command-security.ts:44`
  - body: The wrapper invokes `tirith check` without `--no-daemon` or another explicit offline/no-update boundary. Upstream `check` can spawn background threat DB updates and delegate to a daemon, which contradicts this PR's local-only/no-background-process security claim for a pre-approval scanner.
  - confidence: 0.87
- **[P2] Run the scanner from the effective command cwd:** `src/security/command-security.ts:45-49`
  - body: `execFile` is called without a `cwd`, while upstream tirith uses the current directory for policy discovery. Commands with an explicit workdir can therefore miss project policy and produce incorrect allow, warn, or block decisions.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR introduces concrete pre-approval scanner risks around inherited secrets, upstream daemon/update side effects, and incorrect policy cwd.

Concerns:

- **[high] Scanner receives unsanitized OpenClaw environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The proposed tirith child process inherits `process.env` before approval, which can expose provider tokens, package tokens, SSH agent sockets, and other secrets to an optional external binary.
  - confidence: 0.9
- **[medium] Scanner may spawn update or daemon side effects:** `src/security/command-security.ts:44`
  - body: The tirith invocation omits `--no-daemon` and an explicit offline/no-update mode even though upstream `check` can trigger background threat DB updates and daemon delegation.
  - confidence: 0.87
- **[medium] Scanner policy discovery can use the wrong directory:** `src/security/command-security.ts:45`
  - body: The wrapper does not pass `cwd` to `execFile`, so tirith policy discovery can run from OpenClaw's process cwd instead of the command's effective workdir.
  - confidence: 0.84
