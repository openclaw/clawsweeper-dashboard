## Summary

Keep open: current main does not implement this security feature, and the PR still has blocking scanner-boundary concerns plus missing real behavior proof, so it needs contributor proof and maintainer security/product review rather than cleanup closure.

## What This Changes

The PR adds a tirith-backed pre-exec command scanner for gateway, sandbox, and node exec paths, exposed through a new tools.exec.commandSecurity config object.

## Best Possible Solution

Land a maintainer-approved core scanner seam only after the opt-in/default policy is settled, the scanner runs with least-privilege env from the command's effective cwd, tirith is invoked through an explicit side-effect-free/offline contract, docs/tests/config are aligned, and redacted real behavior proof is attached.

## Reproduction Assessment

For the feature itself, reproduction is not applicable because current main has no commandSecurity surface. For the review blockers, yes: PR-head and upstream tirith source inspection show the scanner env, side-effect, and cwd paths with high confidence.

## Solution Assessment

No, not as written. The direction may be useful, but a mergeable core solution needs a least-privilege and side-effect-free scanner contract, effective-cwd policy discovery, real behavior proof, and maintainer security/product approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process is launched with full coerceEnv(process.env), while current host exec strips sensitive inherited keys before running commands. This can expose provider tokens, package tokens, SSH agent sockets, and tirith control env to an optional external binary; build a minimal/sanitized scanner env and apply it to both gateway and node hooks.
  - confidence: 0.91
- **[P2] Disable scanner daemon and network side effects:** `src/security/command-security.ts:44`
  - body: The wrapper calls default tirith check without --no-daemon or an offline/no-update contract. Upstream check can spawn background threat DB updates, delegate to a daemon, fetch remote policy, and perform network threat enrichment, which is too much side effect for a local pre-approval gate.
  - confidence: 0.88
- **[P2] Scan from the command's effective cwd:** `src/security/command-security.ts:45-50`
  - body: execFile is called without cwd, and callers do not pass the resolved command workdir. Tirith policy discovery can therefore use OpenClaw's process cwd instead of the command's effective directory, missing project .tirith policies or loading the wrong policy before deciding allow, warn, or block.
  - confidence: 0.86
