## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The scanner runs before approval with a full copy of `process.env`, which can expose provider tokens and other OpenClaw secrets to an optional external binary. Build a minimal env or derive it from the existing host exec sanitizer, preserving only what tirith needs such as `PATH`, `HOME`, and explicit tirith settings; the node-host hook has the same issue.
  - confidence: 0.9
- **[P2] Disable implicit tirith daemon and update paths:** `src/security/command-security.ts:44`
  - body: The wrapper invokes `tirith check` without `--no-daemon` or another explicit offline mode, while upstream `check` may delegate to a running daemon and triggers background threat DB update logic. That contradicts the PR’s no-network/no-background-process claim, so OpenClaw should either disable those paths or document/configure them explicitly.
  - confidence: 0.84
- **[P2] Run tirith from the effective command cwd:** `src/security/command-security.ts:45`
  - body: `execFile` is called without a `cwd`, but tirith discovers policy from its current directory. Commands executed from an explicit workdir can therefore ignore the matching project policy and produce incorrect blocks or warnings; thread the effective command cwd into the scanner options.
  - confidence: 0.78
- **[P2] Update the public config contract:** `src/config/zod-schema.agent-runtime.ts:417-425`
  - body: This adds a user-facing `tools.exec.commandSecurity` object to the runtime schema, but the PR does not add schema help, generated config docs baseline updates, public docs, or a changelog entry. Users and config tooling would see an undocumented and drifted config surface.
  - confidence: 0.86
