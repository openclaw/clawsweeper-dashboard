## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process receives `{ ...coerceEnv(process.env) }`, even though current host exec hardening filters inherited provider tokens, package tokens, database URLs, and SSH agent sockets before host execution. Build a minimal scanner env and apply the same boundary to the node hook.
  - confidence: 0.9
- **[P2] Disable daemon and update side effects:** `src/security/command-security.ts:44`
  - body: The wrapper invokes `tirith check` without `--no-daemon` or an explicit no-update/offline boundary. Upstream `check` can spawn a background threat DB update and delegate to a daemon, which breaks the PR's local-only/no-background-process claim for a pre-approval scanner.
  - confidence: 0.88
- **[P2] Run the scanner from the effective command cwd:** `src/security/command-security.ts:45-49`
  - body: `execFile` is called with only `timeout` and `env`, but upstream tirith policy discovery falls back to its process cwd. Commands with an explicit workdir can therefore miss project policy or load the wrong policy before deciding allow, warn, or block.
  - confidence: 0.85

## Security Review

Status: needs_attention

Summary: The diff introduces pre-approval scanner risks around inherited secrets, daemon/update side effects, and cwd-based policy discovery.

Concerns:

  - body: The proposed tirith child process inherits `process.env` before approval, which can expose provider tokens, package tokens, database URLs, SSH agent sockets, and other host-sensitive variables to an optional external binary.

  - confidence: 0.88
