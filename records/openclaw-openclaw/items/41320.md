## Summary

Keep open: this is a plausible security-hardening feature, but it is not implemented on current main and the PR still has missing real behavior proof plus concrete security/product blockers around scanner environment, side effects, and cwd policy.

## What This Changes

Adds optional tirith-backed pre-exec scanning for gateway, sandbox, and node exec paths with `tools.exec.commandSecurity` config, approval integration, and tests.

## Best Possible Solution

Define a maintainer-approved pre-exec scanner seam that invokes external scanners with a minimal env, explicit effective cwd, side-effect-free/offline contract, documented defaults, and real behavior proof.

## Reproduction Assessment

For the feature itself, reproduction is not applicable because current main has no `commandSecurity` surface. For the review blockers, yes: PR-head and upstream tirith source inspection show the env, side-effect, and cwd paths with high confidence.

## Solution Assessment

No, not as written. A mergeable version needs least-privilege scanner env, effective-cwd policy discovery, a side-effect-free tirith invocation contract, contributor real behavior proof, and maintainer security/product approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process is launched with full `coerceEnv(process.env)` while current host exec strips dangerous inherited keys before running commands. This can expose provider tokens, package tokens, SSH agent sockets, proxy settings, and scanner-control env to an optional external binary; build a minimal sanitized scanner env and apply it to both gateway and node hooks.
  - confidence: 0.92
- **[P2] Require a side-effect-free tirith invocation:** `src/security/command-security.ts:44`
  - body: The wrapper calls default `tirith check` without an offline/no-daemon/no-update/no-audit contract. Upstream check can trigger background updates, daemon delegation, remote policy fetches, runtime threat queries, audit writes, and webhooks, which is too much side effect for a local pre-approval gate.
  - confidence: 0.9
- **[P2] Run the scan from the effective command cwd:** `src/security/command-security.ts:45-53`
  - body: `execFile` is called without `cwd`, and callers do not pass the resolved command workdir. Tirith policy discovery can therefore use OpenClaw's process cwd instead of the command's effective directory, missing project `.tirith` policies or loading the wrong policy before deciding allow, warn, or block.
  - confidence: 0.87
