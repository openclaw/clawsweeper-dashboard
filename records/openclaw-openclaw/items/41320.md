## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413-424`
  - body: The scanner runs before approval with a full copy of `process.env`, which can expose provider tokens and other OpenClaw secrets to an optional external binary. Build a minimal env or reuse the existing host exec sanitizer, preserving only required values such as `PATH`, `HOME`, and explicit tirith settings; the node-host hook has the same issue.
  - confidence: 0.9
- **[P2] Disable implicit daemon and update paths:** `src/security/command-security.ts:44`
  - body: The wrapper invokes `tirith check` without `--no-daemon` or another explicit offline/no-update mode. Upstream `check` can delegate to a daemon and spawns background threat DB update work, which contradicts the PR's local-only/no-background-process security claim unless OpenClaw disables or documents that behavior explicitly.
  - confidence: 0.86
- **[P2] Run tirith from the effective command cwd:** `src/security/command-security.ts:45-49`
  - body: `execFile` is called without a `cwd`, while tirith discovers policy from its current directory. Commands executed with an explicit workdir can therefore miss the matching project policy and produce incorrect blocks or warnings; thread the effective command cwd into the scanner options.
  - confidence: 0.8
- **[P2] Update the public config contract:** `src/config/zod-schema.agent-runtime.ts:417-425`
  - body: This adds the user-facing `tools.exec.commandSecurity` object to the runtime schema, but the PR does not add public exec/config docs, generated config baseline updates, or a changelog entry. Users and config tooling would get an undocumented, drifted config surface.
  - confidence: 0.84
