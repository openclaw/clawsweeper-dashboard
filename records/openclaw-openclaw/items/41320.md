## Summary

Keep open: current main does not implement this feature, but the PR is not merge-ready because it adds a security-sensitive pre-approval external scanner without real behavior proof and with concrete environment, side-effect, and cwd issues.

## What This Changes

The PR adds optional tirith-based pre-exec command scanning for gateway, sandbox, and node exec paths, plus `tools.exec.commandSecurity` config/schema labels and scanner integration tests.

## Best Possible Solution

Define a maintainer-approved core scanner seam that invokes external scanners with a minimal env, explicit effective cwd, side-effect-free contract, documented config defaults, and real behavior proof.

## Reproduction Assessment

For the feature itself, reproduction is not applicable because current main has no `commandSecurity` surface. For the review blockers, yes: PR-head and upstream tirith source inspection show the scanner env, side-effect, and cwd paths with high confidence.

## Solution Assessment

No, not as written. A mergeable version needs least-privilege scanner env, effective-cwd policy discovery, a side-effect-free tirith invocation contract, contributor real behavior proof, and maintainer security/product approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process is launched with full `coerceEnv(process.env)` while current host exec strips dangerous inherited keys before running commands. This can expose provider tokens, package tokens, SSH agent sockets, proxy settings, and `TIRITH=0` scanner-control env to an optional external binary; build a minimal sanitized scanner env and apply it to both gateway and node hooks.
  - confidence: 0.92
- **[P2] Disable scanner daemon and network side effects:** `src/security/command-security.ts:44`
  - body: The wrapper calls default `tirith check` without a no-daemon/offline/no-update contract. Upstream `check` can trigger background threat DB updates, delegate to a daemon, fetch remote policy, perform runtime threat queries, write audit state, and dispatch webhooks, which is too much side effect for a local pre-approval gate.
  - confidence: 0.9
- **[P2] Scan from the command's effective cwd:** `src/security/command-security.ts:45-50`
  - body: `execFile` is called without `cwd`, and callers do not pass the resolved command workdir. Tirith policy discovery can therefore use OpenClaw's process cwd instead of the command's effective directory, missing project `.tirith` policies or loading the wrong policy before deciding allow, warn, or block.
  - confidence: 0.87
