## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413-425`
  - body: The pre-approval tirith process is launched with `{ ...coerceEnv(process.env) }`, even though current host exec hardening strips inherited credentials and sensitive sockets before non-sandboxed exec. Build a minimal scanner env, or a sanitizer-approved subset, and apply the same boundary to the node hook.
  - confidence: 0.91
- **[P2] Disable scanner network and daemon side effects:** `src/security/command-security.ts:44`
  - body: The wrapper invokes default `tirith check` without a side-effect-free contract. Upstream `check` can delegate to a daemon and spawn threat DB updates, and local enrichment can perform network lookups, which contradicts using this as a local pre-approval gate.
  - confidence: 0.88
- **[P2] Scan from the command's effective cwd:** `src/security/command-security.ts:45-50`
  - body: `execFile` is called without `cwd`, so tirith can discover policy from OpenClaw's process cwd instead of the command's effective workdir. Commands with an explicit workdir can miss project `.tirith/policy.yaml` or load the wrong policy before deciding allow, warn, or block.
  - confidence: 0.86
