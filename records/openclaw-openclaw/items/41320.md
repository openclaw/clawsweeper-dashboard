## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process is launched with full `coerceEnv(process.env)`, while current host exec strips dangerous inherited keys before running commands. This can expose provider tokens, package tokens, SSH agent sockets, and tirith control env to an optional external binary; build a minimal/sanitized scanner env and apply it to both gateway and node hooks.
  - confidence: 0.91
- **[P2] Disable scanner daemon and network side effects:** `src/security/command-security.ts:44`
  - body: The wrapper calls default `tirith check` without `--no-daemon` or an offline/no-update contract. Upstream check can run background threat DB updates, delegate to a daemon, fetch or enrich remote threat data, write audits, and dispatch webhooks, which is too much side effect for a local pre-approval gate.
  - confidence: 0.88
- **[P2] Scan from the command's effective cwd:** `src/security/command-security.ts:45-50`
  - body: `execFile` is called without `cwd`, and callers do not pass the resolved command workdir. Tirith policy discovery can therefore use OpenClaw's process cwd instead of the command's effective directory, missing project `.tirith` policies or loading the wrong policy before deciding allow, warn, or block.
  - confidence: 0.86
