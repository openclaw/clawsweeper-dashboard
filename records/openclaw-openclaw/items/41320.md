## Summary

Keep open: current main still has no tirith/commandSecurity implementation, and this PR is not obsolete. The proposed core security hook still needs contributor real-behavior proof plus security/product review because the scanner process inherits the full OpenClaw environment, uses default tirith paths with daemon/update/network-capable behavior, and does not run from the command's effective cwd.

## What This Changes

The PR adds an optional tirith-based pre-exec scanner to gateway, sandbox, and node exec paths, exposed through tools.exec.commandSecurity config and covered by new tests.

## Best Possible Solution

Land a maintainer-approved core scanner seam only after the default policy is settled, the scanner runs with least-privilege env from the effective cwd, tirith is invoked through a side-effect-free/offline contract, and docs/tests/config align with redacted real behavior proof.

## Reproduction Assessment

For the feature itself, reproduction is not applicable because current main has no commandSecurity surface. For the review blockers, yes: PR-head and upstream tirith source inspection show the scanner env, side-effect, and cwd paths with high confidence.

## Solution Assessment

No, not as written. The direction may be useful, but a mergeable core solution needs least-privilege scanner env, effective-cwd policy discovery, a side-effect-free tirith invocation contract, contributor real behavior proof, and maintainer security/product approval.

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process is launched with full coerceEnv(process.env), while current host exec strips dangerous inherited keys before running commands.   - confidence: 0.91

  - body: The wrapper calls default tirith check without --no-daemon or an offline/no-update contract. Upstream check can run background threat DB updates, delegate to a daemon, fetch remote policy, perform runtime network threat enrichment, and dispatch webhooks, which is too much side effect for a local pre-approval gate.
  - confidence: 0.88
- **[P2] Scan from the command's effective cwd:** `src/security/command-security.ts:45-50`
  - body: execFile is called without cwd, and callers do not pass the resolved command workdir. Tirith policy discovery can therefore use OpenClaw's process cwd instead of the command's effective directory, missing project .tirith policies or loading the wrong policy before deciding allow, warn, or block.
  - confidence: 0.86
