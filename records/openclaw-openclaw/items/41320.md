## Summary

## What This Changes

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The scanner runs before approval with `{ ...coerceEnv(process.env) }`, so the optional tirith binary receives inherited credentials and host influence variables that current exec env hardening strips. Build a minimal scanner env, preserving only required values such as `PATH`, `HOME`, and explicit tirith settings, and apply the same boundary to the node hook.
  - confidence: 0.9
- **[P2] Disable tirith daemon and update side effects:** `src/security/command-security.ts:44`
  - body: The wrapper invokes `tirith check` without `--no-daemon` or another explicit offline/no-update boundary. Upstream `check` can spawn background threat DB updates and delegate to a daemon, which contradicts the PR's local-only/no-background-process security claim for a pre-approval scanner.
  - confidence: 0.87
- **[P2] Run the scanner from the effective command cwd:** `src/security/command-security.ts:45-49`
  - body: `execFile` is called without a `cwd`, while upstream tirith policy discovery walks from the current directory when no cwd is supplied. Commands with an explicit workdir can therefore miss project policy and produce incorrect allow, warn, or block decisions.
  - confidence: 0.84
