## Summary

Keep open, but this PR is not safe to merge as-is: current main does not implement tirith scanning, and the proposed default-on integration adds pre-approval execution of a PATH-resolved third-party binary with upstream daemon/webhook/network-capable side effects that need maintainer security review.

## What This Changes

The PR adds a default-on `tools.exec.commandSecurity` config path that invokes tirith before gateway, sandbox, and node exec commands, threads warn/block results into approvals, and adds config types, labels, and tests.

## Best Possible Solution

Keep the PR open for maintainer security review; if accepted, rebase onto current exec refactors and make the scanner opt-in, trusted-path/offline-bounded, documented, and covered by targeted exec approval tests plus a changelog entry.

## Reproduction Assessment

Yes for the review findings by source inspection: the diff's default-on condition and `execFile` call show tirith can run before approvals, and upstream `tirith check` source shows daemon, update, enrichment, and webhook-capable paths. I did not run the PR tests in the read-only checkout.

## Solution Assessment

No. The hardening direction may be useful, but default-on pre-approval execution of a PATH-resolved third-party CLI without an explicit offline/trust boundary is not the narrowest maintainable core integration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Require explicit trust before running tirith:** `src/agents/bash-tools.exec.ts:402`
  - body: This hook is default-on because `commandSecurityConfig` can be undefined, and it runs `tirith` by name before the existing approval/allowlist flow. A malicious or unintended binary on the login PATH would execute with OpenClaw's process environment for any attempted exec, even if the requested command is later denied. Make this opt-in and require a trusted absolute scanner path or equivalent trust check before pre-approval execution.
  - confidence: 0.86
- **[P1] Disable tirith daemon and network-capable side effects:** `src/security/command-security.ts:38`
  - body: The integration passes raw exec commands to `tirith check` without disabling daemon/live-enrichment/webhook paths. Upstream `check` delegates to the daemon by default, triggers background update/enrichment paths, and dispatches configured webhooks, so command text can leave OpenClaw despite this PR's no-network/no-telemetry claim. Use a documented offline mode boundary, such as `--no-daemon` plus policy constraints, before enabling this path.
  - confidence: 0.82
- **[P3] Add docs and changelog for commandSecurity:** `src/config/types.tools.ts:260`
  - body: This adds a user-facing `tools.exec.commandSecurity` config key and default exec behavior but does not update the exec/config docs or `CHANGELOG.md`. The config contract needs schema, help/docs, and changelog aligned before merge.
  - confidence: 0.9
