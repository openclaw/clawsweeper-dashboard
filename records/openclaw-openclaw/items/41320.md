## Summary

Keep open: current main does not have this pre-exec scanner surface, and this security-sensitive PR still has missing real behavior proof plus blocking review findings around default-on behavior, scanner environment exposure, side effects, and effective cwd handling.

## What This Changes

Adds tirith-backed pre-exec command scanning across gateway, sandbox, and node exec paths, a `tools.exec.commandSecurity` config surface, approval warning integration, schema labels, and tests.

## Best Possible Solution

Land only a maintainer-approved scanner seam that is opt-in or explicitly migrated, runs the scanner with a minimal env, passes the effective cwd, constrains tirith side effects, and includes redacted after-fix real behavior proof.

## Reproduction Assessment

For the feature itself, reproduction is not applicable because current main has no `commandSecurity` surface. For the review blockers, yes: PR-head source and upstream tirith source show the default-on, inherited-env, cwd, and side-effect paths with high confidence.

## Solution Assessment

No, not as written. A mergeable version needs maintainer-approved default behavior, least-privilege scanner env, effective-cwd policy discovery, a side-effect-constrained tirith invocation contract, and contributor real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not default-enable tirith for existing installs:** `src/agents/bash-tools.exec.ts:405`
  - body: The scanner runs whenever `commandSecurity` is omitted because this condition treats missing config as enabled; the node hook does the same. Any existing user who already has `tirith` on PATH can start seeing blocked or warning-gated exec commands after upgrade without opting in, so gate this on explicit config or include a maintainer-approved migration/default with fresh-install and upgrade proof.
  - confidence: 0.9
- **[P1] Sanitize the scanner process environment:** `src/agents/bash-tools.exec.ts:413`
  - body: The pre-approval tirith process inherits full `process.env` before the command runs. Current host exec strips dangerous inherited keys, so this can expose provider tokens, package tokens, SSH agent sockets, proxy settings, and scanner-control variables to an optional external binary; build a minimal sanitized scanner env and apply it to both gateway and node hooks.
  - confidence: 0.92
- **[P2] Require a side-effect-free tirith invocation:** `src/security/command-security.ts:44`
  - body: The wrapper calls `tirith check` with only `--non-interactive`; upstream `check` can delegate to the daemon, trigger background threat DB update checks, write audit/session state, fetch configured remote policy, and dispatch configured webhooks. Add an explicit no-daemon/offline/no-audit style contract or equivalent env policy before using this as a pre-approval gate.
  - confidence: 0.88
- **[P2] Run the scan from the effective command cwd:** `src/security/command-security.ts:45-50`
  - body: `execFile` is called without `cwd`, and callers do not pass the resolved command workdir. Tirith policy discovery can therefore use OpenClaw's process cwd instead of the command's effective directory, missing project `.tirith` policies or loading the wrong policy before deciding allow, warn, or block.
  - confidence: 0.87
