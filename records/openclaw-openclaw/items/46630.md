## Summary

Keep open. Current main has related pieces, but not the requested user-defined shortcut/keyword sidebar: the Control UI chat sidebar is currently message/content driven, slash commands are a command-completion path, and the plugin UI descriptor foundation is not yet a rendered panel/action system. The related Control UI contribution-slot RFC and pinned-context issue are active but do not supersede this specific shortcut-browser request.

## What This Changes

Add a collapsible, scrollable Control UI/WebChat sidebar listing user-defined shortcut keywords with labels, prompt insertion, descriptions, optional icons, and optional categories.

## Best Possible Solution

Keep this open until maintainers choose the product boundary: either add a host-owned shortcut browser backed by a documented config/command registry, or let the Control UI contribution-slot work mature enough that a shortcut panel can be delivered as a plugin without missing APIs.

## Reproduction Assessment

Not applicable as a feature request. The source check is enough to verify current main lacks a webchat.shortcuts-style configuration surface and persistent shortcut sidebar.

## Solution Assessment

Unclear. A sidebar is a plausible UX, but the narrowest maintainable solution should be decided alongside the open Control UI contribution-slot RFC so shortcuts, command discovery, and plugin-provided panels do not grow separate incompatible APIs.
