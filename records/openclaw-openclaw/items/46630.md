## Summary

Keep open. Current main still has slash-command discovery and a markdown/canvas chat sidebar, but it does not define user-owned shortcut storage, a `webchat.shortcuts` config contract, or a persistent click-to-insert shortcut panel; the related Control UI panel discussion is adjacent but not a full replacement.

## What This Changes

The issue requests a WebChat/Control UI sidebar or drawer for user-defined shortcut keywords with labels, prompts, descriptions, optional icons/categories, hover help, and click-to-insert behavior.

## Best Possible Solution

Define a versioned shortcut discovery/storage contract, then render a responsive host-owned shortcut panel or drawer that inserts text into the composer without auto-sending or bypassing command and plugin boundaries.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source and docs inspection give high confidence that current main lacks shortcut storage/config, a persistent shortcut sidebar or drawer, and arbitrary click-to-insert prompt snippets.

## Solution Assessment

Unclear. The sidebar direction is plausible, but maintainers should first choose the ownership boundary between host config, skill/slash-command metadata, plugin descriptors, and persistent Control UI panel seams.
