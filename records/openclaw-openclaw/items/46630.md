## Summary

Keep open. Current main and v2026.6.10 still have runtime slash-command discovery, a command palette, and a reader sidebar, but they do not provide a user-defined WebChat shortcut/prompt-snippet browser with click-to-insert prompt text or an approved storage/API owner.

## What This Changes

The issue requests a collapsible WebChat sidebar for user-defined shortcut keywords with click-to-insert prompts, tooltips, icons, categories, and a new shortcut configuration/storage surface.

## Best Possible Solution

Define a maintainer-approved shortcut ownership and storage/API contract first, then add a responsive Control UI shortcut panel or drawer that inserts prompt text without auto-sending and reuses skill/plugin command metadata where that is the chosen boundary.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source, docs, latest-release inspection, and related-item review give high confidence that the requested shortcut browser and prompt-snippet insertion surface are absent.

## Solution Assessment
