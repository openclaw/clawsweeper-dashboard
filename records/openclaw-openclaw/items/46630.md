## Summary

Keep open. Current main has related slash-command discovery, a command palette, a markdown/tool sidebar, and a narrow plugin Control UI descriptor foundation, but it still does not provide the requested user-defined shortcut/keyword sidebar, shortcut storage/config contract, or click-to-insert shortcut panel API.

## What This Changes

Add a collapsible, scrollable WebChat sidebar for user-defined shortcut keywords that insert configured prompt text into the chat composer.

## Best Possible Solution

Keep this open until maintainers choose one versioned shortcut discovery contract that can render in Control UI and insert prompt text into the composer without bypassing the broader command/plugin contribution design.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source inspection gives high confidence that the requested shortcut config/storage contract, sidebar renderer, and click-to-insert shortcut panel are absent on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear; a sidebar is plausible, but maintainers should first decide the ownership boundary between host config, slash-command metadata, and plugin Control UI contribution slots.
