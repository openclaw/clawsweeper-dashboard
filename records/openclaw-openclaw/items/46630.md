## Summary

Keep open. Current main has related slash-command discovery, command-palette entries, and a generic plugin Control UI descriptor contract, but it still lacks the requested user-defined shortcut/keyword sidebar and a settled composer-insertion/config API boundary.

## What This Changes

Request adds a collapsible Control UI/webchat sidebar listing user-defined shortcut keywords with labels, prompts, hover descriptions, optional icons/categories, and click-to-insert behavior.

## Best Possible Solution

Keep this open until maintainers define one supported product boundary for shortcut discovery and insertion, then implement it with matching schema/docs and focused Control UI tests.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior. Source inspection on current main verifies no shortcut-sidebar renderer, no `webchat.shortcuts`/`shortcuts.json` contract, and only narrower slash-command/command-palette discovery.

## Solution Assessment

Unclear. The sidebar direction is plausible, but the maintainable fix should be decided alongside the existing plugin descriptor and command discovery surfaces so OpenClaw does not grow competing shortcut registries.
