## Summary

Keep open. Current main has slash-command discovery and a chat markdown sidebar, but it does not implement a persistent user-defined shortcut/keyword panel or a `webchat.shortcuts` config surface; the related Control UI panel/API discussion in #46812 is also still open.

## What This Changes

Add a collapsible scrollable Control UI/WebChat sidebar listing user-defined shortcut prompts with labels, descriptions/tooltips, optional icons/categories, and click-to-insert behavior.

## Best Possible Solution

Define the shortcut model through the maintained Control UI/plugin-command architecture first, then render it in chat as a discoverable, accessible insertion surface without hard-coding one workflow source.
