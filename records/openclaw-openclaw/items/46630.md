## Summary

Keep open. Current main has runtime slash-command discovery, command-palette insertion, and a markdown/canvas chat side panel, but it still lacks user-owned shortcut storage, a `gateway.webchat.shortcuts` config contract, and a persistent shortcut browser that inserts arbitrary prompt snippets without sending them.

## What This Changes

Add a configurable WebChat/Control UI shortcut sidebar or drawer listing user-defined keyword prompts with labels, descriptions, optional icons/categories, and click-to-insert behavior.

## Best Possible Solution

Define a versioned shortcut storage/discovery contract, then render a responsive host-owned shortcut panel or drawer that inserts text into the composer without auto-sending.
