## Summary

Keep open. Current main has adjacent slash-command discovery, command palette, markdown/canvas sidebars, and generic plugin Control UI descriptors, but it still lacks a user-defined shortcut/keyword storage contract, a persistent shortcut sidebar, and click-to-insert composer behavior.

## What This Changes

Add a webchat sidebar that lists user-defined shortcut keywords with labels, prompts, descriptions, optional icons/categories, and click-to-insert behavior.

## Best Possible Solution

Define a versioned shortcut discovery/storage contract, then add a host-rendered Control UI shortcut surface that can insert prompt text into the composer without bypassing command and plugin ownership rules.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source and docs inspection give high confidence that current main lacks the requested shortcut config/storage, sidebar renderer, and click-to-insert composer behavior.

## Solution Assessment

Unclear; the sidebar direction is plausible, but maintainers should first choose the ownership boundary between host config, slash-command metadata, plugin descriptors, and a persistent Control UI panel seam.
