## Summary

Keep open. Current main and v2026.6.8 still lack a user-defined WebChat shortcut browser or prompt-snippet sidebar; existing slash-command, command-palette, and sidebar-reader surfaces are adjacent but do not provide arbitrary shortcut labels, prompt insertion, tooltip metadata, or an approved storage/API owner.

## What This Changes

The issue requests a collapsible WebChat/Control UI sidebar listing user-defined shortcut keywords that insert configured prompt text into the chat composer without auto-sending.

## Best Possible Solution

Define a maintainer-approved shortcut ownership and storage/API contract first, then add a responsive Control UI shortcut panel or drawer that inserts prompt text without auto-sending and reuses plugin/skill metadata where that is the chosen boundary.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source, docs, release, and related-item inspection give high confidence that the requested shortcut browser and prompt-snippet insertion surface are absent on current main and v2026.6.8.

## Solution Assessment

Unclear: the sidebar UX is plausible, but the proposed config-backed implementation is not clearly the best path until maintainers choose the shortcut storage/API owner and its relationship to command, skill, plugin, and Control UI panel surfaces.
