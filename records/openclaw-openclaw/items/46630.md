## Summary

Keep open. Current main and the latest release have runtime slash-command discovery, a command palette, and existing sidebar/reader surfaces, but they still do not provide a user-defined WebChat shortcut or prompt-snippet browser with click-to-insert behavior; the remaining work needs a maintainer product decision about storage and ownership.

## What This Changes

Request to add a collapsible WebChat sidebar or drawer that lists user-defined shortcut keywords with labels, descriptions, optional icons/categories, and click-to-insert prompt text.

## Best Possible Solution

Define the maintainer-approved shortcut ownership and storage/API contract first, then add a responsive Control UI panel or drawer that inserts prompt text without auto-sending and reuses command, skill, or plugin metadata where that becomes the chosen boundary.
