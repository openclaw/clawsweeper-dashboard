## Summary

Keep open. ## Best Possible Solution

Preserve automation-friendly lint behavior by suppressing optional missing-completion findings in lint/noninteractive contexts, keep findings for stale installed completion state, and add real CLI proof for the changed doctor path.

The lint context does not pass `nonInteractive`, while the detector emits an info finding for that optional missing profile.

## Solution Assessment

No, not as-is. The conversion direction is maintainable, but it should preserve lint/noninteractive quietness for optional missing completion before the PR is merge-ready.
