## Summary

Keep open. The draft/protected PR has useful real CLI proof and a plausible Doctor repair-contract direction, but the latest patch introduces a blocking lint regression: startup channel maintenance is registered as a core health check and always emits an info finding, so default `openclaw doctor --lint` can fail even when no repair is needed.

## What This Changes

The PR adds a `run(ctx)`/`ctx.repair` Doctor health-check contract, a split-check adapter, SDK exports, and converts shell completion, startup maintenance, systemd linger, Policy checks, and related tests to the new repair shape.

## Best Possible Solution

Restore quiet healthy lint behavior and add a regression test first, then land the `run(ctx)` contract only after Doctor/Policy owners accept the plugin SDK compatibility and systemd repair semantics.

## Reproduction Assessment

Yes for the review finding: source inspection shows lint calls every check with `repair:false`, while the new startup maintenance check unconditionally returns an info finding. The broader item is a contract migration rather than a reported user bug.

## Solution Assessment

No as currently written. The `run(ctx)` direction is plausible, but startup maintenance needs a lint-quiet detection path or a separate repair-preview path before this is the right implementation.
