## Summary

Keep this PR open: the contract migration is useful and has real behavior proof, but the diff currently lets the preview-only `repairable` status leak into the split repair API, which can make a real `doctor --fix` path look planned rather than applied. ## What This Changes

This PR adds a `run(ctx)`-first Doctor health-check contract, a split-check adapter, structured repair runner updates, and conversions for shell completion, startup maintenance, systemd linger, browser residue, and Policy checks.

## Best Possible Solution

Keep the `run(ctx)` preview contract, but scope `repairable` to run-result preview output, keep split `repair()` applied-only, update the public docs/types accordingly, and land only after Doctor/Policy owner sign-off plus broad stack CI.
