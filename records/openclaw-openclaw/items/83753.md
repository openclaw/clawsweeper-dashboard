## Summary

## What This Changes

This PR converts selected Doctor and Policy health checks from split detect/repair methods to a normalized `run(ctx)` contract, adds a split-check adapter, updates lint/repair runners, and exports the new health-check types through the plugin SDK.

## Best Possible Solution

Keep the `run(ctx)` preview contract only if Doctor/Policy owners approve it, but scope `repairable` to `HealthCheckRunResult`, preserve applied-repair statuses for split repairs, and land with broad Doctor stack validation.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows `repairable` added to `HealthRepairResult`, and the new repair runner exits early on that status before config application or validation. I did not run tests because this review was read-only.

## Solution Assessment

No as currently patched: the normalized `run(ctx)` direction may be acceptable, but the applied repair result type should not include the preview-only `repairable` state. The safer fix is to keep `repairable` only on `HealthCheckRunResult` and add focused coverage for actual split repairs.
