## Summary

Keep open: current main and the latest reported release have generic `media://inbound` resolver support, but this Discord report is newer than the generic fix and still describes a remaining media-store ID, root, or lifecycle miss where the absolute file works and the URI does not.

## What This Changes

Report that Discord inbound images exposed as `media://inbound` refs can fail in the image tool even though the underlying inbound media file exists and works by absolute path.

## Best Possible Solution

Prove the Discord inbound image path on current main, then fix the boundary that loses the canonical media-store ID, root, or file lifetime while keeping `media://` resolution constrained to OpenClaw-owned inbound media.

## Reproduction Assessment

No high-confidence live current-main reproduction is available. Source inspection shows the Discord save path, prompt `media://` projection, and exact resolver failure path, but the existing generic resolver tests do not exercise Discord DM lifecycle reuse.

## Solution Assessment

Unclear. The maintainable fix should target the specific Discord media-store root, ID, or lifecycle mismatch once reproduced; adding a broad absolute-path fallback would be the wrong boundary.
