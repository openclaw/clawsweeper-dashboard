## Summary

Keep open: current main has generic `media://inbound` resolver support and prior fixes for path rewriting and media pruning, but the Discord report is newer than those fixes and still points to a remaining media-store root, ID, or lifecycle mismatch where the absolute file works and the URI does not.

## What This Changes

The issue reports that Discord inbound image attachments stored under `media/inbound` can become unusable when the image tool receives their `media://inbound/...` reference.

## Best Possible Solution

Prove the Discord inbound image path on current main, then fix the boundary that loses the canonical media-store ID or file lifetime while keeping `media://` resolution constrained to OpenClaw-owned inbound media.

## Reproduction Assessment

No high-confidence live current-main reproduction is available. Source inspection shows the Discord save path, prompt `media://` projection, and exact resolver failure path, but generic current-main resolver tests pass and do not exercise Discord lifecycle reuse.

## Solution Assessment

Unclear. The maintainable fix should target the specific Discord media-store root, ID, or lifecycle mismatch once reproduced; adding a broad absolute-path fallback would be the wrong boundary.
