## Summary

Keep open: current main has generic `media://inbound/*` support and a merged cleanup fix that may address one disappearance mode, but there is no evidence that the reported Discord absolute-path-works/media-URI-fails mismatch is fixed on current main or in v2026.6.8.

## What This Changes

The issue reports that Discord inbound image attachments can be saved on disk while the image tool fails to resolve the corresponding `media://inbound/<filename>` reference.

## Best Possible Solution

Reproduce the Discord DM media path on current main, then fix the boundary that loses the canonical media-store root or saved ID across Discord media capture, prompt/session context, and image-tool execution.

## Reproduction Assessment

No high-confidence full reproduction path yet. Source inspection shows the exact resolver failure remains possible when the ID is absent from the active media store, but I did not reproduce the live Discord DM flow on current main.

## Solution Assessment

Unclear until the Discord path is reproduced. A broad fallback from `media://` to arbitrary absolute paths would be the wrong security boundary; the fix should target the specific root, ID, or lifecycle mismatch.
