## Summary

Keep open: current main and v2026.6.9 have generic `media://inbound` resolver support, but no current-main Discord DM proof shows the reported resolver miss is fixed; the remaining issue is likely a Discord media-store ID, active state-dir, or lifetime mismatch.

## What This Changes

The issue requests that inbound Discord image attachments exposed as `media://inbound/<id>` remain resolvable by the `image` tool instead of failing with a missing media ID.

## Best Possible Solution

Reproduce the Discord DM inbound image path on current main, then fix the precise media-store ID/root/lifetime boundary while keeping `media://inbound` constrained to OpenClaw-owned inbound media.

## Reproduction Assessment

No high-confidence live current-main reproduction is available. Source inspection shows the Discord save path, prompt `media://` projection, and exact resolver failure when the active media store cannot find the ID, but current tests only prove generic resolver behavior.

## Solution Assessment

Unclear. The maintainable fix should target the Discord-specific media-store ID, root, or lifetime mismatch once reproduced; adding a broad absolute-path fallback would be the wrong boundary.
