## Summary

Keep open: current main still has generic `media://inbound` resolver support, but there is no source, test, release, or related-item proof that the Discord-specific missing-file failure is fixed. The remaining problem still looks like a Discord handoff, active state-dir, media ID, or media lifetime mismatch rather than an obsolete generic resolver bug.

## What This Changes

The issue reports that Discord-provided `media://inbound/...` attachment references fail in the image tool even though the corresponding inbound media files exist on disk.

## Best Possible Solution

Reproduce the Discord DM inbound image path on current main, then fix the specific media-store ID, root, or lifetime boundary while keeping `media://inbound` constrained to OpenClaw-owned inbound media.

## Reproduction Assessment

No high-confidence live current-main Discord reproduction was established. Source inspection shows the Discord save path, media URI projection, and exact resolver miss path, while current tests only prove generic media-store URI handling.

## Solution Assessment

Unclear until the live Discord path is reproduced. The maintainable fix should target the specific Discord media-store root, ID, or lifecycle mismatch; adding a broad absolute-path fallback would be the wrong boundary.
