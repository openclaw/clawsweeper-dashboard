## Summary

Keep open: current main has generic `media://inbound` resolver support, but this Discord report is newer than that generic fix and still points to a Discord media handoff, state-root, or lifetime mismatch that is not proven fixed.

## What This Changes

The issue reports that Discord inbound image attachments exposed as `media://inbound/...` paths fail in the `image` tool even though direct filesystem paths work.

## Best Possible Solution

Reproduce the Discord DM attachment path on current main, then fix the exact media-store root, ID, or lifetime mismatch while keeping `media://inbound` limited to OpenClaw-owned inbound media.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows the exact resolver error path and the Discord save/projection/read chain, but current tests only prove generic media-store URI handling with synthetic files.

## Solution Assessment

Unclear. The maintainable fix should target the specific Discord media-store root, ID, or cleanup/session lifetime mismatch once reproduced; adding a broad absolute-path fallback would be the wrong boundary.
