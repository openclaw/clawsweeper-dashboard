## Summary

Keep open: current main has generic `media://inbound` resolver support, but the Discord-specific report is not proven fixed and still points to a media-store ID, state-dir, or lifetime mismatch where a direct filesystem path works but the URI does not.

## What This Changes

The issue reports that Discord inbound image attachments stored on disk cannot be resolved by the image tool when passed as `media://inbound/...` URIs.

## Best Possible Solution

Reproduce the Discord DM inbound image path on current main, then fix the media-store ID/root/lifetime boundary while keeping `media://inbound` constrained to OpenClaw-owned inbound media.

## Reproduction Assessment

No high-confidence current-main live reproduction is available. Source inspection shows the Discord save path, prompt `media://` projection, and exact resolver failure path, but current tests only prove generic media-store URI handling.

## Solution Assessment

Unclear. The maintainable fix should target the specific Discord media-store root, ID, or lifecycle mismatch once reproduced; adding a broad absolute-path fallback would be the wrong boundary.
