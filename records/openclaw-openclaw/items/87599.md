## Label Justifications

- `P2`: This is a real but edge-case ACP restore/session-continuity problem with a fresh-session workaround and limited blast radius.
- `impact:session-state`: The failure is caused by ACP session identity metadata being absent after cleanup, preventing restoration of prior session context.
- `impact:auth-provider`: The requested recovery path depends on ownership/authorization for a Claude/backend ACP session id and must not weaken provider/session access controls.

## Summary

Current main still blocks `sessions_spawn` resume unless the backing ACP/ACPX session id is present in a local `acp.identity` entry owned by the requester, so the reported post-cleanup restore path is not implemented. The remaining work is valid but security/session-state sensitive because it needs an explicit ownership model for backend-session restore rather than a simple bug patch.

## What This Changes

The issue asks for an authorized way to create a fresh OpenClaw ACP wrapper around a known valid Claude/backend ACP session id after local ACP metadata has been cleaned up.

## Best Possible Solution

Define a separate, explicit backend-session restore contract that proves requester/admin authorization, creates fresh ACP metadata and bindings, and keeps stale ACP wrappers from silently respawning.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: remove or lack a local `acp.identity` match for the requested backend id, then call `sessions_spawn` with `resumeSessionId`; current main returns `resume_forbidden` before runtime initialization. I did not run a live Claude ACP restore because the report depends on real backend/session state and this review is read-only.

## Solution Assessment

Unclear as a one-step code fix: the requested behavior is plausible, but the best solution needs maintainer choice about whether to extend `resumeSessionId`, add a distinct `restoreBackendSessionId`, or persist tombstone ownership state. Any implementation should preserve the stale-session safety fix and add regression coverage for both allowed restore and forbidden cross-requester restore.
