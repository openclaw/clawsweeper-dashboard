## Summary

This PR should stay open but is not merge-ready: the feature direction is plausible, but the branch targets the removed Slack runtime path, lacks real Slack proof, and puts fetched channel history into prompt text where it can affect command/directive handling.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty Slack slash-command invocations to fetch recent channel history for the agent response.

## Best Possible Solution

Rework this as an active Slack plugin feature that keeps slash command text separate from bounded recent history, passes history as structured untrusted context, updates config/docs/tests, and includes redacted real Slack proof.

## Reproduction Assessment

Yes for the blocking PR findings by source inspection: current main uses `extensions/slack`, while the branch edits `src/slack` and routes fetched history into prompt fields. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature should be implemented in the active Slack plugin and should carry recent history as structured untrusted context rather than prompt text that can be parsed as command/directive input.
