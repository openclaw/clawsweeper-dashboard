## Summary

Keep open: the requested Slack feature is not implemented on current main, and this PR is a plausible but currently incorrect implementation. It targets the removed `src/slack` layout, lacks real Slack proof, and makes fetched channel history prompt/directive-adjacent in a way that needs security/product review.

## What This Changes

The PR adds a `contextAware` Slack slash-command config flag and changes empty slash-command invocations to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Implement this in the active Slack plugin, keep slash command text separate from fetched history, pass bounded recent history as structured untrusted context, update config/docs/tests, and include redacted real Slack proof.

## Reproduction Assessment

Yes for the blocking PR findings via source inspection: current main routes Slack slash commands through `extensions/slack`, while the branch edits `src/slack` and puts fetched history into prompt fields. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but the patch should be reworked against `extensions/slack` and keep fetched history out of command/directive parsing by using structured untrusted context.
