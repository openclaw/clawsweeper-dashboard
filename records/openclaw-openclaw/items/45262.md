## Summary

Keep open: current main does not implement `contextAware`, and this PR adds a new Slack config mode with a blocking prompt/directive injection concern that needs maintainer product/security direction before merge.

## What This Changes

The PR adds a `slashCommand.contextAware` Slack option that lets an empty single slash command fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

If maintainers want this feature, implement it in the Slack plugin with a documented config option, bounded history fetches, fetched messages passed only as structured `UntrustedContext`, directive parsing isolated from history, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes, from source for the review finding: current main rejects empty single Slack slash commands, while the PR diff would allow them and place fetched channel history into the active prompt path. I did not run live Slack validation in this read-only review.

## Solution Assessment

No. The feature may be useful, but this patch is not the best implementation because untrusted recent channel history can participate in prompt/directive handling instead of staying isolated as untrusted context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:580`
  - body: When `contextAware` builds `effectivePrompt`, recent Slack channel messages become the active prompt while the slash command text is empty. Directive resolution falls back to prompt text when command text is empty, so a prior channel message containing `/think`, `/reasoning`, `/elevated`, or similar directives can affect the authorized slash invocation. Keep fetched history in structured untrusted context and exclude it from command parsing.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR diff introduces a concrete prompt/directive injection concern by feeding raw Slack channel history into the active slash-command prompt path.

Concerns:

- **[medium] Raw Slack history can steer an authorized slash run:** `src/slack/monitor/slash.ts:580`
  - body: Any channel participant can place instruction-like or directive-like text in recent history; the patch interpolates that history into the active slash-command prompt for an empty command, and current directive resolution can fall back to prompt text. This can change model/runtime behavior under the slash invoker's authorization unless history is isolated as untrusted context and excluded from command parsing.
  - confidence: 0.9
