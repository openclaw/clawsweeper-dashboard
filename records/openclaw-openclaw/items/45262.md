## Summary

Keep open: current main still does not implement the requested empty Slack slash-command history mode, but this PR is not mergeable as-is because the proposed implementation puts raw channel history into the prompt/directive path and lacks inspectable real Slack proof. Prior automated review comments already called out the same incomplete wiring and prompt/directive-injection concerns.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty slash commands to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Implement this in the current `extensions/slack` plugin path with bounded history carried as structured untrusted context, keep command fields tied to the actual slash text, document the config, and validate it in a real Slack workspace.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows the PR puts fetched history into `Body`/`BodyForAgent` while command fields can remain empty, and directive parsing falls back to prompt text. I did not establish a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch should keep fetched history out of command/directive parsing, target the current `extensions/slack` layout, remove or wire the unused context value, and include real Slack proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` is used with empty slash text, this line moves fetched channel history into `Body`/`BodyForAgent` while the slash command text remains empty. Since directive resolution falls back to prompt text when command text is empty, a prior channel message containing directive-like text can steer this slash request; keep history in untrusted, non-command context instead.
  - confidence: 0.89
- **[P3] Remove or wire the context history value:** `src/slack/monitor/slash.ts:582`
  - body: `contextHistory` is assigned from fetched Slack messages but is never read afterward. Either pass structured history through the intended context path or remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Concerns:
