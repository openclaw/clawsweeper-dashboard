## Summary

Keep open, but this PR is not merge-ready: the requested Slack feature is not on current main, while the branch is now conflicting, lacks real Slack proof, and crosses the command/session trust boundary by turning recent channel history into model prompt text.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` option intended to let empty Slack slash commands fetch recent channel history and use that history as context.

## Best Possible Solution

Rebase into `extensions/slack`, make ambient-history use explicitly opt-in policy, keep command parsing tied to the actual slash-command text, and pass fetched history as bounded `UntrustedContext` with tests and redacted Slack proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection and CI logs: current main lacks `contextAware`, the branch writes fetched history into prompt text, and the branch failed type checks. I did not run a live Slack workspace reproduction.

## Solution Assessment

No, not as written. The maintainable path is to rebase into `extensions/slack` and pass recent history as bounded untrusted model context while leaving command/directive parsing on the actual slash-command text.
