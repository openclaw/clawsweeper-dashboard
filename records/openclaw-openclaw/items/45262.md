## Summary

Keep open: the requested Slack feature is not implemented on current main, and the proposed patch is stale against the Slack plugin migration and has a prompt/directive safety blocker.

## What This Changes

The branch adds a `contextAware` Slack slash-command option that fetches up to 10 recent channel messages when the single slash command is invoked without text.

## Best Possible Solution

If maintainers want this feature, implement it in the Slack plugin with a documented config option, bounded history fetches, fetched messages passed only as structured `UntrustedContext`, directive parsing isolated from history, focused slash-command tests, docs, and a changelog entry.

## Reproduction Assessment

Yes from source for the review finding: current main rejects empty single Slack slash commands, and the PR diff would put fetched history into the prompt path while command text remains empty. Directive parsing falls back to prompt text when command source is empty.

## Solution Assessment

No. The feature may be useful, but this patch is not the best implementation because it targets stale paths and lets untrusted channel history participate in active slash-command prompt/directive handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:580`
  - body: When `contextAware` builds `effectivePrompt`, recent Slack channel messages become `Body`/`BodyForAgent` while the command fields remain the empty slash text. Directive resolution falls back to prompt text when command text is empty, so a prior channel message containing `/think`, `/reasoning`, `/elevated`, or similar directives can affect the authorized slash invocation. Keep fetched history in structured untrusted context and exclude it from command parsing.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces an indirect prompt/directive injection risk by treating recent Slack history as active slash-command prompt text.

Concerns:

- **[medium] Raw Slack history can steer an authorized slash run:** `src/slack/monitor/slash.ts:580`
  - body: Any channel participant can place instruction-like or directive-like text in recent history; the patch interpolates that history into `BodyForAgent` for an empty slash command, and current directive resolution can fall back to that prompt text. This can change model/runtime behavior under the slash invoker's authorization unless the history is isolated as untrusted context and excluded from command parsing.
  - confidence: 0.9
