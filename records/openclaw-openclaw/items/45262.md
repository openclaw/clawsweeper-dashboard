## Summary

Keep open: the Slack feature idea is plausible, but this PR is not merge-ready because it conflicts with current main, edits the removed `src/slack` runtime path, lacks real Slack proof, and introduces a prompt/session boundary risk around ambient channel history.

## What This Changes

The PR adds a `slashCommand.contextAware` Slack option and changes empty slash-command invocations to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Rework the feature in the active Slack plugin as an explicit opt-in that passes bounded recent history as structured untrusted context, keeps command parsing tied to the actual slash-command text, and includes docs, focused tests, and redacted Slack proof.

## Reproduction Assessment

Yes for the PR blockers via source inspection: current main lacks `contextAware` and rejects empty Slack slash prompts, while the PR head embeds fetched history into `Body`/`BodyForAgent`. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The requested direction may be useful, but this patch should be reworked against the active Slack plugin and use structured untrusted context instead of prompt/command-adjacent text.
