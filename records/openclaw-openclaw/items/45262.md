## Summary

Keep open. Current main still rejects empty Slack slash commands and does not expose a `contextAware` option, while this PR has a blocking security issue because fetched channel history is placed into the active prompt path where directive parsing can fall back to it.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty slash-command handling to fetch recent Slack channel history for the agent prompt.

## Best Possible Solution

If maintainers want this feature, implement it in the Slack plugin with documented config, bounded history fetches, fetched messages passed only as structured `UntrustedContext`, directive parsing isolated to the slash input, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes from source inspection for the review finding: current main rejects empty single Slack slash commands, and the PR diff would put fetched history into `BodyForAgent` while slash command text stays empty. Directive parsing can fall back from empty command text to prompt text.

## Solution Assessment

No. The feature may be useful, but this patch is not the best implementation because untrusted Slack history becomes active prompt/directive input instead of isolated context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:580`
  - body: When `contextAware` builds `effectivePrompt`, recent Slack messages become `Body`/`BodyForAgent` while `RawBody` and `CommandBody` remain the empty slash input. Directive resolution uses `commandSource || promptSource`, so prior channel messages containing directives such as `/think` or `/elevated` can steer an authorized slash run; keep fetched history in `UntrustedContext` and leave command parsing on the slash input only.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces an indirect prompt/directive injection risk by treating recent Slack channel history as active slash-command prompt text.

Concerns:

- **[medium] Raw Slack history can steer an authorized slash run:** `src/slack/monitor/slash.ts:580`
  - body: Any channel participant can place instruction-like or directive-like text in recent history; the patch interpolates that history into `BodyForAgent` for an empty slash command, and current directive resolution can fall back to prompt text. This can change model or runtime behavior under the slash invoker's authorization unless history is isolated as untrusted context and excluded from command parsing.
  - confidence: 0.9
