## Summary

Keep open, but this PR is not merge-ready: the feature is not implemented on current main, and the branch still needs a current Slack-plugin rework, real Slack proof, and safer handling of ambient channel history.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty slash-command invocations to fetch recent Slack channel history into the agent prompt.

## Best Possible Solution

Implement this in the active Slack plugin as an explicit opt-in config, pass bounded recent history as structured untrusted context, keep `CommandBody`/`RawBody` clean, and add docs, tests, and redacted Slack proof.

## Reproduction Assessment

Yes for the PR blockers via source inspection: current main routes Slack slash commands through `extensions/slack` and lacks `contextAware`, while the PR head writes fetched history into `Body`/`BodyForAgent` for empty command text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch should be reworked against the active Slack plugin and should pass history as structured untrusted context rather than embedding it in prompt/command-adjacent text.
