## Summary

Keep open: the feature is plausible but the current patch is not mergeable as-is because it adds a new Slack config/behavior and routes untrusted recent channel history through the active slash-command prompt path, where directive parsing can fall back to prompt text.

## What This Changes

The PR adds a Slack slash-command `contextAware` option that allows empty slash commands to fetch recent channel history and use it as agent context.

## Best Possible Solution

If maintainers want this feature, implement it in the Slack plugin on current paths with documented config, bounded history reads, required Slack scope guidance, recent messages passed only as structured untrusted context, directive parsing isolated from fetched history, focused tests, docs, and changelog coverage.

## Reproduction Assessment

Yes, for the review finding from source inspection. The PR makes fetched channel history become `BodyForAgent` while the command body remains empty, and current directive resolution falls back from command text to prompt text.

## Solution Assessment

No. The feature may be useful, but this patch is not the safest maintainable solution because untrusted Slack history can participate in active slash-command/directive handling and the branch targets stale Slack paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fetched Slack history out of command parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` builds `effectivePrompt`, recent channel messages become `Body`/`BodyForAgent` while the slash command text remains empty. Directive parsing can fall back to prompt text, so earlier channel messages containing `/think`, `/reasoning`, `/elevated`, or similar directives can affect the authorized slash run. Keep fetched history in structured untrusted context and exclude it from command parsing.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete indirect prompt/directive injection risk by treating recent Slack history as active slash-command prompt text.

Concerns:

- **[medium] Raw Slack history can steer an authorized slash run:** `src/slack/monitor/slash.ts:583`
  - body: Any channel participant can place instruction-like or directive-like text in recent history; the patch interpolates that history into `BodyForAgent` for an empty slash command, and directive resolution can fall back to that prompt text. This can change model/runtime behavior under the slash invoker's authorization unless the history is isolated as untrusted context and excluded from command parsing.
  - confidence: 0.9
