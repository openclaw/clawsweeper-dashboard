## Summary

Keep open: current main does not implement the requested Slack context-aware slash command behavior, but the submitted branch is not mergeable as-is because it targets the removed Slack runtime path, lacks real Slack proof, and routes fetched channel history into prompt/directive-adjacent text.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty Slack slash-command invocations to fetch recent channel history as context.

## Best Possible Solution

Rework the feature in the active Slack plugin, keep slash command text in command fields, pass bounded recent history as structured untrusted context, update config/docs/tests, and include redacted real Slack proof.

## Reproduction Assessment

Yes for the blocking PR findings via source inspection: the branch changes removed `src/slack` files and puts fetched history in prompt text while command text remains empty. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch should target `extensions/slack` and keep fetched history out of command/directive parsing as untrusted context.

  - confidence: 0.82
