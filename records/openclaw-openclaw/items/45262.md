## Best Possible Solution

Implement this in the active Slack plugin as an explicit opt-in config, pass bounded recent history as structured untrusted context, keep `CommandBody`/`RawBody` clean, and add docs, tests, and redacted real Slack proof.

## Reproduction Assessment

Yes for the blocking PR findings via source inspection: current main routes Slack slash commands through `extensions/slack`, while the branch edits `src/slack` and puts fetched history into prompt fields. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but the patch should be reworked against the active Slack plugin and use structured untrusted context instead of making history part of command/prompt text.
