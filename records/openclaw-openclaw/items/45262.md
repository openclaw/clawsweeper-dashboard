## Summary

Keep open: current main does not implement the requested Slack empty slash-command behavior, but this PR is not mergeable as-is because it targets the old Slack path, lacks real Slack proof, and routes fetched history through prompt/directive fields instead of structured untrusted context.

## What This Changes

Adds a Slack `slashCommand.contextAware` config option and old `src/slack` slash-handler logic to answer empty slash invocations using recent Slack channel history.

## Best Possible Solution

Implement this in `extensions/slack` with explicit config/docs/generated metadata, bounded history carried as structured untrusted context, directive fields tied to the actual slash text, focused tests, and real Slack proof.

## Reproduction Assessment

Yes for the blocking review finding via source inspection: PR head puts fetched history in `Body`/`BodyForAgent` while `CommandBody` remains empty, and directive parsing falls back to prompt text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but the patch should target `extensions/slack`, keep fetched history out of directive parsing, treat history as untrusted context, and include real Slack proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` is used with empty slash text, this assigns fetched channel history to `Body`/`BodyForAgent` while the command text stays empty. Directive resolution falls back to prompt text when command source is empty, so prior channel messages containing directive-like text can steer this request; keep history in untrusted, non-command context instead.
  - confidence: 0.9
- **[P3] Remove or wire the context history value:** `src/slack/monitor/slash.ts:582`
  - body: `contextHistory` is assigned from fetched Slack messages but is never read afterward. Either pass structured history through the intended context path or remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff copies raw recent Slack messages into the prompt/directive path for an empty slash command.

Concerns:
