## Summary

Keep this PR open for maintainer review: current main does not implement the requested Slack behavior, but the branch is stale against the current Slack plugin path and introduces an untrusted-history prompt/directive safety problem.

## What This Changes

The branch adds a `contextAware` boolean to Slack slash-command config/schema and changes the slash handler to fetch up to 10 recent Slack channel messages when the single slash command is invoked without text.

## Best Possible Solution

If maintainers want this feature, port it to `extensions/slack`, model fetched messages as explicitly untrusted context instead of active command text, and add config-schema, docs, changelog, and focused Slack slash-command tests.

## Reproduction Assessment

Yes from source: current main rejects empty single Slack slash commands, and the PR's blocking prompt/directive issue follows directly from the patch plus directive fallback code. No live Slack run was needed for this review finding.

## Solution Assessment

No. The requested direction may be useful, but this patch is not the best implementation because it injects raw channel history into `BodyForAgent`, can fall back into directive parsing, and needs to be ported to the current Slack plugin path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:580`
  - body: When `contextAware` builds `effectivePrompt`, it puts raw recent channel messages into `BodyForAgent` while `CommandBody` and `RawBody` remain the empty slash-command text. Directive parsing falls back to the prompt text when command text is empty, so a prior channel message containing `/think`, `/reasoning`, `/elevated`, or similar directives can affect the authorized slash invocation. Keep fetched history in a structured untrusted context and ensure the command text cannot fall back to that history.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR introduces an indirect prompt/directive injection risk by treating recent Slack history as current slash-command input.

Concerns:

- **[medium] Raw Slack history can steer an authorized slash run:** `src/slack/monitor/slash.ts:580`
  - body: Any channel participant can place instruction-like or directive-like text in recent history; the patch then interpolates that history into `BodyForAgent` for an empty slash command, and current directive resolution can fall back to that prompt text. This can change model/runtime behavior under the slash invoker's authorization unless the history is isolated as untrusted context and excluded from command parsing.
  - confidence: 0.9
