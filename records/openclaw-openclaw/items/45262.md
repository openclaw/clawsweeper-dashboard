## Summary

Keep this PR open, but it is not merge-ready: current main still does not implement `contextAware`, while the branch now conflicts with the active Slack plugin path and puts ambient Slack history into prompt text across a session/security boundary.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and uses recent channel history when a configured slash command is invoked without text.

## Best Possible Solution

Rebase into `extensions/slack`, make ambient-history use an explicit opt-in policy, keep command parsing tied to the actual slash-command text, and pass fetched history as bounded `UntrustedContext` with tests and redacted Slack proof.

## Reproduction Assessment

## Solution Assessment

No, not as written. The maintainable path is to rebase into `extensions/slack` and pass recent history as bounded untrusted model context while leaving command/directive parsing on the actual slash-command text.
