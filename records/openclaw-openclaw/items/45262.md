## Summary

Keep open. Current main still does not implement the requested Slack `slashCommand.contextAware` option, but this PR is not merge-ready because it targets the pre-plugin Slack path and introduces security-sensitive history-to-agent behavior that bypasses current directive and context-visibility safeguards.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty Slack slash-command invocations to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

If maintainers want this capability, implement it in `extensions/slack` as an opt-in setting that authorizes the invoker before reading history, applies the existing Slack supplemental-context visibility rules, keeps directive parsing tied to the actual slash-command text, passes fetched messages as untrusted/supplemental context, and adds focused tests plus docs/changelog updates.

## Reproduction Assessment

Yes. Current-main behavior is reproducible from source by invoking the configured Slack slash command with no text and hitting the `Message required.` branch; the PR-specific regressions are reproducible from the diff plus directive/context-filter source paths.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the best solution because it is stale against the current plugin layout and routes fetched channel history through prompt fields that existing directive and visibility logic do not treat as safe supplemental context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:597-598`
  - body: When `contextAware` is enabled with an empty slash command, recent Slack history is assigned to `Body` and `BodyForAgent` while the real command text remains empty. Directive resolution falls back to prompt text when command text is empty, so prior channel messages can alter this turn. Keep fetched history in supplemental or untrusted context and keep command fields tied to the slash text.
  - confidence: 0.9
- **[P2] Apply Slack context visibility filters:** `src/slack/monitor/slash.ts:565-582`
  - body: This branch maps every returned `conversations.history()` message into the prompt. Current Slack thread-history handling removes current-bot messages and applies the supplemental context visibility policy before exposing fetched history, so this path can reveal messages normal Slack context would omit.
  - confidence: 0.82
- **[P3] Remove or wire contextHistory:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from formatted Slack message history but is never read afterward. Remove it if prompt-only behavior is intended, or pass it through the selected structured context path once the context model is fixed.
  - confidence: 0.84
