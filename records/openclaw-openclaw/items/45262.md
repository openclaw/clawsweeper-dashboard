## Summary

Keep open: the requested Slack slash-command feature is not implemented on current main, and the proposed patch has a blocking security/correctness issue because fetched channel history becomes active prompt text for an authorized empty slash command.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` option and changes empty single slash-command invocations to fetch recent channel history for the agent prompt.

## Best Possible Solution

If maintainers want this feature, implement it in the Slack plugin with documented opt-in config, bounded history reads, fetched messages passed as structured `UntrustedContext`, directive parsing isolated from fetched history, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes, for the PR finding: source inspection shows the diff puts fetched Slack history into `Body`/`BodyForAgent` while the command fields remain empty, and current directive parsing can fall back to prompt text. The feature request itself is not a bug reproduction; current main intentionally returns `Message required.` for empty single slash commands.

## Solution Assessment

No. The requested UX may be useful, but this patch is not the best implementation because it treats recent untrusted channel history as active prompt text instead of structured context excluded from command parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fetched Slack history out of command text:** `src/slack/monitor/slash.ts:580-581`
  - body: When `contextAware` builds `effectivePrompt`, recent channel messages become `Body`/`BodyForAgent` while the command fields stay empty. Directive resolution falls back to prompt text when command text is empty, so a prior Slack message containing `/think`, `/reasoning`, `/elevated`, or similar tokens can alter the authorized slash invocation. Keep fetched history in structured untrusted context and exclude it from command parsing.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR introduces an indirect prompt/directive injection risk by feeding raw Slack channel history into the active slash-command prompt path.

Concerns:

- **[medium] Raw Slack history can steer authorized slash runs:** `src/slack/monitor/slash.ts:580`
  - body: Any channel participant can place instruction-like or directive-like text in recent history; the patch interpolates that history into `BodyForAgent` for an empty slash command, and directive resolution can fall back to prompt text when command fields are empty. This can change model/runtime behavior under the slash invoker's authorization unless history is isolated as untrusted context.
  - confidence: 0.9
