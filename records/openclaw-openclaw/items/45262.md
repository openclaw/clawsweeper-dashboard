## Summary

Keep this PR open, but it is not merge-ready: the feature is still missing on current main, the branch now conflicts with the active Slack plugin path, and the patch introduces session/security-boundary concerns around ambient Slack history.

## What This Changes

The PR adds an opt-in Slack `slashCommand.contextAware` config field so an empty slash command fetches recent channel history and uses it as the agent prompt.

## Best Possible Solution

Rebase into `extensions/slack`, keep command/directive parsing tied to the actual slash-command text, and pass fetched history only as bounded `UntrustedContext` after maintainers approve the opt-in ambient-history policy.

## Reproduction Assessment

Yes for the PR blockers by source inspection and CI logs: current main lacks `contextAware`, the branch writes fetched history into prompt text, and the branch failed type checks. I did not run a live Slack workspace reproduction.

## Solution Assessment

No, not as written. The maintainable path is to rebase into `extensions/slack` and pass recent history as bounded untrusted model context while leaving command/directive parsing on the actual slash-command text.
