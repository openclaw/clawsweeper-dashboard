## Summary

Keep open: the feature idea is plausible, but this PR is not merge-ready because it targets the removed `src/slack` runtime path, lacks redacted real Slack proof, and puts ambient channel history into prompt text where it can cross the command/directive boundary.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config flag and changes empty Slack slash-command invocations to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Rebase onto `extensions/slack`, make ambient history an explicit opt-in policy, pass fetched messages as bounded untrusted model context while leaving command parsing on the actual slash text, and add tests/docs plus redacted Slack proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current `main` uses `extensions/slack` and lacks `contextAware`, while the PR diff writes fetched Slack history into `Body`/`BodyForAgent` for empty command text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No, not as written. The maintainable path is to rebase into the active Slack plugin and treat fetched history as bounded untrusted context, with command/directive parsing based only on the actual slash-command text.
