## Summary

Keep open: the feature is not implemented on current main, but this PR should not merge as-is because it targets the removed `src/slack` layout, routes fetched Slack history through prompt/directive paths, leaves a half-wired context value, and still lacks real Slack proof.

## What This Changes

Adds a Slack `slashCommand.contextAware` config option and changes slash command handling so empty slash invocations answer using recent channel history.

## Best Possible Solution

Rework the feature in `extensions/slack`, keep the slash text in command fields, carry bounded recent history as structured untrusted context, update config metadata/docs/tests, and include redacted real Slack proof.

## Reproduction Assessment

Yes for the blocking review findings via source inspection: PR head puts fetched history in `Body`/`BodyForAgent` while command text remains empty, and current directive parsing can fall back to prompt text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch should target the active Slack plugin path and keep fetched history out of command/directive parsing as untrusted context.
