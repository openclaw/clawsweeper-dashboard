## Summary

Keep open: current main still does not implement empty Slack slash commands with recent-channel context, but this PR is not mergeable as-is because fetched history is placed on the prompt path where directive parsing can treat it as command input, and the PR still lacks real Slack proof.

## What This Changes

Adds a Slack `slashCommand.contextAware` config option and changes empty slash-command invocations to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Implement this in the current Slack plugin path with bounded history carried as untrusted context or structured inbound history, keep command fields tied to the actual slash text, document the config, and validate it in a real Slack workspace.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows the PR puts fetched history into `Body`/`BodyForAgent` while command fields remain empty, and current directive parsing falls back to prompt text. I did not establish a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch should keep fetched history out of command/directive parsing, target the current `extensions/slack` layout, and include real Slack proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` is used with empty slash text, this line moves fetched channel history into `Body`/`BodyForAgent` while `CommandBody` remains empty. Current directive resolution uses `commandSource || promptSource`, so a prior channel message containing directive-like text can steer this slash request; keep history in untrusted, non-command context instead.
  - confidence: 0.89
- **[P3] Remove or wire the context history value:** `src/slack/monitor/slash.ts:582`
  - body: `contextHistory` is assigned from fetched Slack messages but is never read afterward. Either pass structured history through the intended context path or remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention
