## Summary

Keep this PR open, but do not merge it as-is: the feature is not implemented on current main, and the branch still has a source-provable directive/security issue plus missing real Slack proof.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and lets empty slash-command invocations fetch recent Slack channel history for the agent response.

## Best Possible Solution

Rework the feature in the current Slack plugin path so fetched history is bounded and explicitly treated as untrusted non-command context while slash command fields remain tied to the actual slash text.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows the PR puts fetched history in `BodyForAgent` while command fields are empty, and directive parsing can fall back to that prompt text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch uses the prompt/directive path for fetched channel history, needs to target the current Slack extension layout, and still needs real Slack proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep history out of directive parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` is used with an empty slash command, recent Slack messages become `Body`/`BodyForAgent` while `CommandBody` stays empty. Directive resolution falls back to prompt text when command text is empty, so a prior channel message containing directive-like text can steer this request; keep fetched history in untrusted, non-command context instead.
  - confidence: 0.89
- **[P3] Remove or wire the context history value:** `src/slack/monitor/slash.ts:582`
  - body: `contextHistory` is assigned from fetched Slack messages but is never read afterward. Either pass structured history through the intended untrusted context path or remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff copies raw Slack channel history into a slash-command prompt path that can influence directive parsing.

Concerns:

- **[medium] Raw history can alter slash-command directives:** `src/slack/monitor/slash.ts:583`
  - body: Any channel participant can place directive-looking text in recent history; in empty context-aware mode, that history becomes prompt text and may be parsed as this request's directive input. The fix should keep fetched history out of command/directive parsing and mark it as untrusted context.
  - confidence: 0.88
