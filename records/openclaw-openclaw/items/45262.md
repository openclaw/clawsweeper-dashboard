## Summary

Keep open: the requested Slack feature is not implemented on current main, but this PR is not mergeable as-is because it introduces a prompt/directive injection risk, lacks real Slack behavior proof, and needs to be redesigned for the current plugin layout.

## What This Changes

The PR adds `slashCommand.contextAware` to Slack config and updates old `src/slack` slash-command handling to fetch recent channel history for empty slash invocations.

## Best Possible Solution

Implement this in `extensions/slack` with bounded history carried as structured untrusted context, keep directive fields tied to the actual slash text, update config/docs/tests, and validate in a real Slack workspace.

## Reproduction Assessment

Yes for the blocking review finding via source inspection: the PR writes recent history into `Body`/`BodyForAgent` while `CommandBody` remains the empty slash text, and directive code falls back to prompt text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch should keep fetched history out of command/directive parsing, target the current `extensions/slack` layout, remove or wire the unused context value, and include real Slack proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` is used with empty slash text, this assigns fetched channel history to `Body`/`BodyForAgent` while the command text stays empty. Directive resolution falls back to prompt text when command source is empty, so prior channel messages containing directive-like text can steer this request; keep history in untrusted, non-command context instead.
  - confidence: 0.9
- **[P3] Remove or wire the context history value:** `src/slack/monitor/slash.ts:582`
  - body: `contextHistory` is assigned from fetched Slack messages but is never read afterward. Either pass structured history through the intended context path or remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention because the diff copies raw recent Slack messages into the prompt/directive path for an empty slash command.

Concerns:

- **[medium] Raw Slack history can steer slash directives:** `src/slack/monitor/slash.ts:583`
  - body: Any channel participant can place directive-like text in recent history; in empty context-aware mode, that history becomes prompt text and may be parsed as this request's directive input. The fix should keep fetched history out of command parsing and mark it as untrusted context.
  - confidence: 0.88
