## Summary

This PR still contains useful feature intent, but it should stay open because current `main` does not implement `contextAware` and the proposed patch has a blocking history/directive security issue plus missing real Slack proof.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` option so empty slash-command invocations fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Rework the feature in the Slack plugin path so history is explicit, bounded, and passed as untrusted non-command context while `CommandBody` stays tied to the actual slash text, with docs, focused tests, and redacted Slack proof.

## Reproduction Assessment

Yes for the blocking review issue: source inspection shows empty slash text leaves `CommandBody` empty while recent history is placed in `BodyForAgent`, and directive parsing falls back to prompt text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch puts fetched history on the command/prompt path instead of an untrusted non-command context path and needs to be rebased to the current Slack extension layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep history out of directive parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` is used with an empty slash command, recent Slack messages are copied into `Body`/`BodyForAgent` while `CommandBody` remains empty. Directive resolution falls back to prompt text when command text is empty, so a prior channel message containing `/think`, `/reasoning`, or similar directive text can steer this request; keep fetched history in untrusted, non-command context instead.
  - confidence: 0.89
- **[P3] Remove or wire the context history value:** `src/slack/monitor/slash.ts:582`
  - body: `contextHistory` is assigned from fetched Slack messages but is never read afterward. Either pass structured history through the intended untrusted context path or remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete prompt/directive injection risk by copying Slack channel history into the slash-command prompt path.

Concerns:

- **[medium] Raw history can alter slash-command directives:** `src/slack/monitor/slash.ts:583`
  - body: Any channel participant can place directive-looking text in recent history; in empty-prompt context-aware mode, that history becomes prompt text and may be parsed as this request's directive input. The fix should keep fetched history out of command/directive parsing and mark it as untrusted context.
  - confidence: 0.88
