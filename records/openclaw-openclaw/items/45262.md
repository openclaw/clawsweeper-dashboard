## Summary

Keep open: current main does not implement the requested Slack context-aware slash behavior, and the proposed patch still has a blocking prompt/directive injection issue plus missing real Slack proof.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option so empty slash-command invocations can fetch recent channel history and use it as conversation context.

## Best Possible Solution

Rework the feature in the current Slack plugin path so fetched history is bounded, explicit, and passed as untrusted non-command context while `CommandBody` stays tied to the actual slash text, with focused tests and redacted Slack proof.

## Reproduction Assessment

Yes for the blocking review issue: source inspection of the PR diff and current directive resolver shows empty `CommandBody` falls back to `BodyForAgent`, so raw recent messages can become directive input. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch puts fetched history on the prompt/directive path instead of an untrusted non-command context path and also needs to target the current Slack extension layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep history out of directive parsing:** `src/slack/monitor/slash.ts:583`
  - body: When `contextAware` is used with an empty slash command, recent Slack messages are copied into `Body`/`BodyForAgent` while `CommandBody` stays empty. Directive resolution falls back to prompt text when command text is empty, so a prior channel message containing `/think`, `/reasoning`, or similar directive text can steer this request; keep fetched history in untrusted, non-command context instead.
  - confidence: 0.89
- **[P3] Remove or wire the context history value:** `src/slack/monitor/slash.ts:582`
  - body: `contextHistory` is assigned from fetched Slack messages but is never read afterward. Either pass structured history through the intended untrusted context path or remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: the patch copies raw Slack channel history into the slash-command prompt path, which can influence inline directive parsing.

Concerns:

- **[medium] Raw history can alter slash-command directives:** `src/slack/monitor/slash.ts:583`
  - body: Any channel participant can place directive-looking text in recent history; in empty context-aware mode, that history becomes prompt text and may be parsed as this request's directive input. The fix should keep fetched history out of command/directive parsing and mark it as untrusted context.
  - confidence: 0.88
