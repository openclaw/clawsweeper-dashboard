## Summary

Keep open for maintainer handling: the feature remains unimplemented on main, but the current patch is conflicted and has a blocking directive-injection issue plus missing real Slack proof.

## What This Changes

The branch adds a Slack slash-command `contextAware` config option and, for empty prompts, fetches recent Slack channel messages and uses them as the slash request prompt.

## Best Possible Solution

Implement this in the Slack extension path with history isolated as untrusted, non-command context, keep directive parsing tied to the actual slash text, add focused tests/docs, and require redacted Slack proof before merge.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR diff and directive parser shows empty `CommandBody` falls back to history-backed `BodyForAgent`. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but this implementation puts channel history on the command/prompt path instead of untrusted non-command context and needs to be rebased to the Slack extension path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep history out of directive parsing:** `src/slack/monitor/slash.ts:581`
  - body: When `contextAware` is used with an empty slash command, `CommandBody` stays empty while `BodyForAgent` is filled with recent channel messages. The directive parser falls back to prompt text when command text is empty, so a prior Slack message containing `/think`, `/reasoning`, or similar directives can steer this request; keep history in untrusted/non-command context instead.
  - confidence: 0.88
- **[P3] Remove or wire the unused context history:** `src/slack/monitor/slash.ts:580`
  - body: `contextHistory` is assigned from the fetched Slack messages but never read afterward. If structured history is intended, pass it through an untrusted context path; otherwise remove the local so the implementation is not half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: the diff copies Slack channel history into the slash prompt path, which can become directive input when the slash text is empty.

Concerns:

- **[medium] Raw history can alter slash-command directives:** `src/slack/monitor/slash.ts:581`
  - body: Any channel participant can place directive-looking text in recent history; in empty-prompt context-aware mode, that history becomes `BodyForAgent` and may be parsed as this request's directive text. The fix should keep fetched history out of command/directive parsing and mark it as untrusted context.
  - confidence: 0.86
