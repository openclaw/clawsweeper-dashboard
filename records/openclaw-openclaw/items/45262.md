## Summary

Keep open. The proposal is useful, but the current PR is not merge-ready because it adds a new Slack slash-command mode and routes fetched, untrusted channel history into active prompt/directive handling.

## What This Changes

The branch adds a Slack `slashCommand.contextAware` option and lets empty single slash commands fetch up to 10 recent channel messages for the agent prompt.

## Best Possible Solution

If maintainers want this feature, implement it in the Slack plugin with documented config, bounded history fetches, recent messages passed only as structured `UntrustedContext`, directive parsing isolated from fetched history, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes, from source for the review finding. Current main rejects empty single Slack slash commands, while the PR would put fetched channel history into the prompt path and directive resolution falls back to prompt text when command text is empty.

## Solution Assessment

No. The requested interaction may be useful, but this patch is not the best implementation because untrusted history enters active prompt/directive handling instead of a structured untrusted-context channel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:580`
  - body: When `contextAware` builds `effectivePrompt`, recent Slack channel messages become prompt text while the slash command text remains empty. Directive resolution can fall back from the empty command source to `BodyForAgent`, so a prior channel message containing `/think`, `/reasoning`, `/elevated`, or similar directive-like text can alter the authorized slash run. Keep fetched history in structured untrusted context and exclude it from command parsing.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces an indirect prompt/directive injection path by treating recent Slack channel history as active slash-command prompt text.

Concerns:

- **[medium] Raw Slack history can steer an authorized slash run:** `src/slack/monitor/slash.ts:580`
  - body: Any channel participant can place instruction-like or directive-like text in recent history; the patch interpolates that history into the slash run's prompt text, and current directive resolution can fall back to prompt text when command text is empty. Isolate history as untrusted context and keep slash-command parsing tied to the invoker's command text.
  - confidence: 0.9
