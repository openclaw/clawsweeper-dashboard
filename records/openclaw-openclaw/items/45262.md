## Summary

Keep open: current main still rejects empty Slack slash-command prompts and does not implement the requested context-aware mode, but this PR is not merge-ready because raw channel history can be parsed as current-command directives and the external PR lacks real Slack behavior proof.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty slash-command invocations to fetch recent channel history and send it as the agent prompt.

## Best Possible Solution

Implement the feature against the Slack extension path with history isolated as untrusted, non-command context, keep directive parsing tied to the actual slash text, update config docs/tests, and require redacted Slack proof before merge.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows an empty `CommandBody` lets directive parsing fall back to the history-backed `BodyForAgent`. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The direction is plausible, but embedding raw channel history into the prompt/command path is not the safest maintainable design; use untrusted context or another non-command channel-history field instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep history out of directive parsing:** `src/slack/monitor/slash.ts:581`
  - body: When `contextAware` builds `Body`/`BodyForAgent` from recent channel messages while `CommandBody` is empty, directive parsing falls back to that prompt text. A previous Slack message containing `/think`, `/reasoning`, or `/elevated` can silently steer this slash request, so history needs to stay in untrusted/non-command context and not become directive input.
  - confidence: 0.88
- **[P3] Remove or wire the unused context history:** `src/slack/monitor/slash.ts:580`
  - body: `contextHistory` is assigned from the fetched Slack messages but never read afterward. If structured history is intended, pass it through the untrusted context path; otherwise remove the local so the implementation does not look half-wired.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete prompt/directive injection risk by copying Slack channel history into the current slash-command prompt path.

Concerns:

- **[medium] Raw history can alter slash-command directives:** `src/slack/monitor/slash.ts:581`
  - body: Any channel participant can put directive-looking text in recent history; in empty-prompt context-aware mode, that history becomes `BodyForAgent` and can be parsed as this request's directive text. The fix should keep history out of command/directive parsing and mark it as untrusted context.
  - confidence: 0.86
