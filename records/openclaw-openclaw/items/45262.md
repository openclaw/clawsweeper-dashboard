## Summary

Keep open, but this patch should not merge as-is: the feature is still absent from current main, the branch edits the pre-extension Slack path, and the implementation puts recent Slack history into the prompt/directive path without real Slack proof.

## What This Changes

Adds a Slack `slashCommand.contextAware` config option and changes empty Slack slash invocations to fetch recent channel history and use it as the prompt.

## Best Possible Solution

Rework the feature in `extensions/slack`, keep slash text in command fields, pass bounded recent history as structured untrusted context, update config metadata/docs/tests, and include redacted real Slack proof.

## Reproduction Assessment

Yes for the blocking PR findings via source inspection: the PR puts fetched history in `Body`/`BodyForAgent` while command text remains empty, and current directive parsing can fall back to prompt text. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The feature direction is plausible, but the patch should target the active Slack plugin and keep fetched history out of command/directive parsing as untrusted context.
