## Summary

Keep open: the feature is not implemented on current main, but this PR is stale against the Slack extension migration, lacks real Slack proof, and introduces command/session trust-boundary issues that need rework before merge.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option so empty slash-command invocations can fetch recent channel history and use it as conversation context.

## Best Possible Solution

Rebase into the active Slack plugin, make ambient history an explicit opt-in policy, pass fetched history as bounded `UntrustedContext` while keeping command parsing tied to the actual slash-command text, and add tests, docs, and redacted Slack proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection and CI logs: current main lacks `contextAware`, the PR writes fetched history into prompt text, and the branch failed type checks. I did not run a live Slack workspace reproduction.

## Solution Assessment

No, not as written. The maintainable path is to rebase into `extensions/slack` and pass recent history as bounded untrusted model context while leaving command/directive parsing on the actual slash-command text.
