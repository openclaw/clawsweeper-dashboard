## Summary

Keep open. The PR addresses a real current-main gap, but it should not merge as-is because the new OAuth secret write-back path is profile-only, logs raw error strings in a token-adjacent catch, and lacks the required changelog coverage for this user-facing auth fix. Reviewed PR: https://github.com/openclaw/openclaw/pull/74004

## What This Changes

Adds an OAuth-manager post-refresh hook that writes refreshed Claude CLI OAuth credentials back to Claude Code CLI storage via `writeClaudeCliCredentials`.

## Best Possible Solution

Keep the fix narrow: write back only refreshed `claude-cli` credentials for `anthropic:claude-cli`, use redacted token-adjacent logging, add focused unit coverage, and include a single-line changelog entry.

## Reproduction Assessment

Yes for the source-level behavior. Current main can refresh and persist an expired managed OAuth credential into OpenClaw stores, but the refresh path has no post-refresh call to `writeClaudeCliCredentials`; the live Claude CLI/Keychain race was not run.

## Solution Assessment

No, not as written. A post-refresh hook is a plausible direction, but the implementation needs provider/profile scoping, redacted logging, tests, and changelog coverage before it is the safest maintainable fix.
