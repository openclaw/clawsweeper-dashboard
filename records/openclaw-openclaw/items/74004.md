## Summary

Keep open. Current main still lacks the proposed write-back hook, so the PR addresses a real remaining Claude CLI OAuth refresh gap, but the patch introduces token-adjacent correctness and logging issues that should block merge until maintainer/security follow-up.

## What This Changes

The PR adds an OAuth manager post-refresh hook and wires it to write refreshed `anthropic:claude-cli` OAuth credentials back to Claude CLI file or Keychain storage via the existing Claude credential writer.

## Best Possible Solution

If bidirectional Claude CLI sync is intended, keep the hook narrow: write back only validated `anthropic:claude-cli` / `claude-cli` OAuth credentials after a successful refresh, use the existing redacting error formatter or omit error details, add focused success/no-op/failure tests, and align the comments/docs/changelog with the Claude-only boundary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an expired managed Claude CLI OAuth profile can refresh on current main and save only OpenClaw auth stores, with no call path back to `writeClaudeCliCredentials`.

## Solution Assessment

No, not as written. The direction may be the narrow product fix, but the patch should first guard on both profile id and provider, sanitize write-back logging, and correct the Codex write-back comments before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard Claude CLI write-back by provider:** `src/agents/auth-profiles/external-cli-sync.ts:170-171`
  - body: This writes refreshed OAuth material to Claude CLI storage whenever the profile id is `anthropic:claude-cli`, but it does not verify that the refreshed credential is actually for provider `claude-cli`. A misconfigured or legacy profile using that id with another OAuth provider would copy unrelated tokens into Claude CLI; require the expected provider/profile pair before calling the writer.
  - confidence: 0.84
- **[P2] Redact write-back error messages:** `src/agents/auth-profiles/external-cli-sync.ts:184-185`
  - body: The new catch block logs `String(error)` while handling OAuth credential writes. Unexpected adapter or writer errors can include serialized credential or request payloads, so this should use the repo's redacting `formatErrorMessage()` path or omit the error detail before adding this token-adjacent log sink.
  - confidence: 0.88
- **[P3] Limit the write-back comment to Claude CLI:** `src/agents/auth-profiles/external-cli-sync.ts:151-152`
  - body: The new JSDoc says the helper writes to Claude CLI and Codex CLI stores, but the implementation only handles `CLAUDE_CLI_PROFILE_ID`. Since Codex is intentionally bootstrap-only on current main, the comment should name only Claude CLI unless Codex write-back is actually implemented.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds OAuth secret write-back to an external credential store and needs provider scoping plus redacted logging before merge.

Concerns:

  - body: The proposed helper keys the external write solely on `CLAUDE_CLI_PROFILE_ID`; it should also require `credential.provider === "claude-cli"` before placing refreshed OAuth material into Claude CLI storage.
