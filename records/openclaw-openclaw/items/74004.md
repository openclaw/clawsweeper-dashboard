## Summary

Keep open. The PR addresses a real remaining gap on current main, but the proposed patch is not merge-ready because it adds an external OAuth secret write-back path without provider scoping, redacted error handling, focused coverage, or the required user-facing changelog entry.

## What This Changes

The PR adds an OAuth-manager post-refresh hook and wires it to write refreshed `anthropic:claude-cli` OAuth credentials back to Claude Code CLI credential storage through `writeClaudeCliCredentials`.

## Best Possible Solution

If this direction is accepted, keep the implementation narrow: write back only validated `anthropic:claude-cli` / `claude-cli` OAuth credentials after a successful refresh, use redacted or omitted error details, preserve Codex bootstrap-only behavior, add focused tests, and include the required changelog entry.

## Reproduction Assessment

Yes for the source-level behavior: current main can refresh and save an expired OAuth profile into OpenClaw stores, but there is no post-refresh hook that writes the rotated token back to Claude CLI storage. A live macOS Keychain race was not run because this was a read-only review.

## Solution Assessment

No, not as written. The proposed approach is plausibly the narrow fix, but it needs provider scoping, redacted write-back logging, corrected scope documentation, tests, and changelog coverage before it is the safest maintainable solution.
