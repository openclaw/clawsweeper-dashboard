## Summary

Keep open. Current main still lacks Claude CLI write-back after OpenClaw refreshes a managed OAuth credential, but this PR writes OAuth material to external CLI storage with profile-only scoping and raw error logging that should be fixed before merge.

## What This Changes

The PR adds a post-refresh OAuth-manager adapter hook that writes refreshed `anthropic:claude-cli` OAuth credentials back to Claude Code CLI storage through `writeClaudeCliCredentials`.

## Best Possible Solution

Keep the fix narrow: write back only validated `anthropic:claude-cli` / `claude-cli` OAuth credentials after successful refresh, keep Codex bootstrap-only, redact or omit write-back error details, add focused tests, and add the required changelog entry.

## Reproduction Assessment

Yes for the source-level behavior. Current main can refresh and persist an expired managed OAuth credential into OpenClaw stores, but the refresh path has no post-refresh call to `writeClaudeCliCredentials`; the live Claude CLI/Keychain race was not run.

## Solution Assessment

No, not as written. The direction is plausible, but the implementation needs provider/profile scoping, redacted token-adjacent logging, corrected scope comments, tests, and changelog coverage before it is the safest maintainable fix.
