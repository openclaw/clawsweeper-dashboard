## Summary

Keep open. Current main still lacks the post-refresh Claude CLI write-back path, so the PR addresses a real remaining auth race, but the patch should not merge until provider scoping and token-adjacent logging are fixed.

## What This Changes

The PR adds an OAuth-manager post-refresh hook that writes refreshed `anthropic:claude-cli` OAuth credentials back to Claude Code CLI storage via `writeClaudeCliCredentials`.

## Best Possible Solution

If maintainers accept bidirectional sync, keep it narrow: write back only validated `anthropic:claude-cli`/`claude-cli` refreshed OAuth credentials, preserve Codex bootstrap-only behavior, redact or omit token-adjacent error details, and cover the path with focused tests plus a changelog entry.

## Reproduction Assessment

Yes for the source-level behavior: current main can refresh and save an expired OAuth profile into OpenClaw stores, but there is no post-refresh hook that writes rotated tokens back to Claude CLI storage. A live Keychain race was not run because this was a read-only review.

## Solution Assessment

No, not as written. The proposed hook is a plausible narrow direction, but it needs provider scoping, redacted logging, corrected scope comments, tests, and changelog coverage before it is the safest maintainable solution.

## Security Review
