## Summary

Keep open. Current main still lacks the proposed Claude CLI OAuth write-back path, so the PR is not obsolete, but the proposed patch writes OAuth secret material to an external credential store with concrete provider-guard, redaction, comment, test/changelog issues to resolve before merge.

## What This Changes

The PR adds an optional OAuth-manager post-refresh hook and wires it to write refreshed `anthropic:claude-cli` OAuth credentials back to Claude CLI file or Keychain storage via the existing Claude CLI credential writer.

## Best Possible Solution

If bidirectional Claude CLI sync is accepted, keep the hook narrow: only write back validated `anthropic:claude-cli` / `claude-cli` OAuth credentials after a successful refresh, use redacted or omitted error details, add focused tests for success, no-op, provider mismatch, and failure paths, and add the required user-facing changelog entry.

## Reproduction Assessment

Yes for the source-level behavior: current main can refresh an expired OAuth profile into OpenClaw's store, but there is no hook capable of writing the rotated token back to Claude CLI storage. I did not run a live OAuth/Keychain race because this review was read-only.

## Solution Assessment

No, not as written. The direction may be the right narrow product fix, but the patch should guard on both profile id and provider, sanitize write-back logging, correct the documented scope, and cover the new external secret write boundary before merge.
