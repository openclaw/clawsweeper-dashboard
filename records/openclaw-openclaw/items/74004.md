## Summary

Keep open: current main still lacks the requested post-refresh Claude CLI write-back, and merged https://github.com/openclaw/openclaw/pull/80274 only persists fresher external CLI credentials into OpenClaw's auth store. The PR direction is plausible, but this patch needs provider/profile scoping, redacted token-adjacent logging, and contributor real behavior proof before merge.

## What This Changes

The PR adds an OAuth manager hook that writes refreshed OAuth credentials back to Claude CLI storage after OpenClaw refreshes them.

## Best Possible Solution

Implement a narrow Claude CLI-only post-refresh sync that requires the expected profile and provider, uses existing redacted error formatting, keeps Codex bootstrap-only, and includes focused regression tests plus contributor real behavior proof.

## Reproduction Assessment

Yes for source-level behavior: current main refreshes and stores OAuth credentials without any call to the Claude CLI writer. The live macOS Keychain/file race was not run in this read-only review.

## Solution Assessment

No, not as written. The direction is maintainable, but the patch needs provider/profile scoping, redacted error logging, corrected scope comments, tests, and real behavior proof before it is the safest fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard write-back by provider:** `src/agents/auth-profiles/external-cli-sync.ts:170-171`
  - body: This writes to Claude CLI storage whenever the profile id is `anthropic:claude-cli`, but it does not verify that the refreshed credential is actually for provider `claude-cli`. A misconfigured or legacy profile with that id could copy unrelated OAuth tokens into Claude CLI storage; require the expected provider/profile pair before calling the writer.
  - confidence: 0.86
- **[P2] Redact write-back error messages:** `src/agents/auth-profiles/external-cli-sync.ts:184-185`
  - body: The new token-adjacent catch logs `String(error)`. Unexpected writer or adapter errors can embed serialized credential payloads, so this should use the repo's redacting `formatErrorMessage()` path or omit the message before adding the write-back sink.
  - confidence: 0.88
- **[P3] Limit the write-back comment to Claude CLI:** `src/agents/auth-profiles/external-cli-sync.ts:151-152`
  - body: The new JSDoc says this helper writes to Claude CLI and Codex CLI stores, but the implementation only handles `CLAUDE_CLI_PROFILE_ID`. Since Codex remains bootstrap-only on current main, the comment should name only Claude CLI unless Codex write-back is intentionally added.
  - confidence: 0.82
