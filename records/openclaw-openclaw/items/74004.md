Keep open. Current main still lacks post-refresh Claude CLI write-back, but this PR remains blocked by missing real behavior proof plus provider scoping and redacted logging issues around OAuth credential storage.

## What This Changes

The PR adds a post-refresh OAuth manager hook that writes refreshed Claude CLI OAuth credentials back to Claude CLI Keychain/file storage.

## Best Possible Solution

If maintainers accept bidirectional Claude CLI credential mutation, implement it as a Claude CLI-only sync gated by profile and provider, with redacted logging, focused regression tests, and redacted real setup proof.

## Reproduction Assessment

Yes for source-level behavior: current main refreshes and saves OAuth profiles but has no hook that writes refreshed Claude CLI credentials back to Claude CLI storage. I did not run the live two-process OpenClaw plus Claude CLI race.

## Solution Assessment

No, not as written. The direction is plausible, but the patch needs provider/profile scoping, redacted error handling, Claude-only comments, tests, and real behavior proof before it is the safest fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard write-back by provider:** `src/agents/auth-profiles/external-cli-sync.ts:170-171`
  - body: The hook runs after every OAuth refresh, but this branch writes any credential whose profile id is `anthropic:claude-cli` into Claude CLI storage. Current main already treats external CLI sync as a profile+provider match; require `credential.provider === "claude-cli"` before calling the writer so a legacy or misconfigured profile cannot copy unrelated OAuth tokens into Claude CLI credentials.
  - confidence: 0.86
- **[P2] Redact write-back error messages:** `src/agents/auth-profiles/external-cli-sync.ts:184-185`
  - body: This token-adjacent catch logs `String(error)` instead of the repo's redacting `formatErrorMessage()` path. Unexpected writer errors can include serialized credential payloads, so use the existing redacted formatter or omit the error message before adding this sink.
  - confidence: 0.88
- **[P3] Limit write-back comments to Claude CLI:** `src/agents/auth-profiles/external-cli-sync.ts:151-152`
  - body: The new comments say this helper writes to Claude CLI and Codex CLI stores, but the implementation only handles Claude CLI while Codex remains bootstrap-only on current main. Narrow the comments unless Codex write-back is intentionally implemented and tested.
  - confidence: 0.82
