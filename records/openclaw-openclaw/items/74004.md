## Summary

Keep open. Current main still lacks the Claude CLI write-back hook, so this PR is not obsolete, but the proposed patch needs provider scoping, redacted logging, tests, and changelog coverage before it is mergeable.

## What This Changes

The PR adds an OAuth-manager post-refresh hook that writes refreshed `anthropic:claude-cli` OAuth credentials back to Claude Code CLI storage via `writeClaudeCliCredentials`.

## Best Possible Solution

Land a narrow Claude-only write-back path only after it verifies the expected profile/provider pair, keeps Codex bootstrap-only behavior unchanged, uses redacted or omitted error details, and adds focused tests plus the required changelog entry.

## Reproduction Assessment

Yes for the source-level behavior: current main can refresh and save an expired OAuth profile into OpenClaw stores, but there is no post-refresh call to `writeClaudeCliCredentials`. The exact Claude Code Keychain race was not live-tested during this read-only review.

## Solution Assessment

No, not as written. The hook placement is plausible, but the implementation should be tightened around provider identity, token-adjacent logging, and regression coverage before it is the safest fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard write-back by provider:** `src/agents/auth-profiles/external-cli-sync.ts:170-171`
  - body: This writes refreshed OAuth material to Claude CLI storage whenever the profile id is `anthropic:claude-cli`, even if the stored credential's provider is not `claude-cli`. A misconfigured or legacy profile with the same id could copy unrelated tokens into Claude Code's credential store; require the expected provider/profile pair before calling the writer.
  - confidence: 0.84

  - body: The new token-adjacent catch logs `String(error)`. Unexpected writer or adapter errors can include serialized credential payloads, so this should use the repo's redacting `formatErrorMessage()` path or omit the message before logging around OAuth secrets.
  - confidence: 0.88

  - body: The JSDoc says this helper writes to Claude CLI and Codex CLI stores, but the implementation only handles `CLAUDE_CLI_PROFILE_ID`. Since Codex is bootstrap-only on current main, the comment should name only Claude CLI unless Codex write-back is intentionally added.
  - confidence: 0.82
