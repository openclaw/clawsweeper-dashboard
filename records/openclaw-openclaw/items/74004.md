## Summary

Keep open. The PR targets a real remaining Claude CLI OAuth refresh gap on current main, but the proposed patch should not merge as written because it adds an external OAuth secret write-back path with insufficient provider scoping and raw error string logging.

## What This Changes

The PR adds an OAuth-manager post-refresh hook and wires it to write refreshed Claude CLI OAuth credentials back through the existing Claude CLI credential writer.

## Best Possible Solution

If bidirectional Claude CLI sync is accepted, keep it narrow: only write back validated `anthropic:claude-cli` plus `claude-cli` OAuth credentials after successful refresh, sanitize or omit failure details, preserve best-effort behavior, and add focused coverage plus a changelog entry.

## Reproduction Assessment

Yes for the source-level behavior: current main refreshes into OpenClaw auth stores and has no write-back hook, while Claude CLI read-side sync is present. I did not run a live OAuth/Keychain race because this review was read-only.

## Solution Assessment

No, not as written. The post-refresh hook is a reasonable direction for the reported race, but the patch needs provider scoping, redacted logging, corrected scope comments, tests, changelog, and security-owner review before it is the narrow maintainable fix.
