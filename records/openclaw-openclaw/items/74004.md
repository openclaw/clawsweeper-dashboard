## Summary

Keep open: current main still does not have a Claude CLI write-back hook, but this PR is not merge-ready because it is now conflicting, lacks real behavior proof, and adds token-adjacent write-back without enough provider scoping or redacted logging.

## What This Changes

The PR adds an OAuth manager hook that writes refreshed `anthropic:claude-cli` OAuth credentials back to Claude CLI Keychain/file storage after OpenClaw refreshes them.

## Best Possible Solution

Implement a narrow Claude CLI-only post-refresh sync gated by both profile and provider, using the existing redacted error formatter and current external-CLI persistence model, with focused regression tests and redacted real setup proof.

## Reproduction Assessment

Yes for source-level behavior: current main refreshes and stores OAuth credentials without any Claude CLI write-back hook, while a Claude CLI writer already exists. The live macOS/Windows two-process race was not run in this read-only review.

## Solution Assessment

No, not as written. The direction is plausible, but the patch needs provider/profile scoping, redacted error logging, current-main conflict resolution, tests, and real behavior proof before it is the safest fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard write-back by provider:** `src/agents/auth-profiles/external-cli-sync.ts:170-171`
  - body: This writes to Claude CLI storage whenever the profile id is `anthropic:claude-cli`, but it does not verify that the refreshed credential is actually for provider `claude-cli`. A misconfigured or legacy profile with that id could copy unrelated OAuth tokens into Claude CLI storage; require the expected provider/profile pair before calling the writer.
  - confidence: 0.86
- **[P2] Redact write-back error messages:** `src/agents/auth-profiles/external-cli-sync.ts:184-185`
  - body: The new token-adjacent catch logs `String(error)`. Unexpected writer or adapter errors can embed serialized credential payloads, so this should use the repo's redacting `formatErrorMessage()` path or omit the message before adding the write-back sink.
  - confidence: 0.88
- **[P3] Limit the write-back comments to Claude CLI:** `src/agents/auth-profiles/external-cli-sync.ts:151-152`
  - body: The new comments say this helper writes to Claude CLI and Codex CLI stores, but the implementation only handles `CLAUDE_CLI_PROFILE_ID`. Since Codex remains bootstrap-only on current main, the comments should name only Claude CLI unless Codex write-back is intentionally added.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: the patch writes OAuth material into Claude CLI storage and logs raw error strings, so provider scoping and redacted error handling are required.

Concerns:
