## Summary

Keep open. Current main still lacks the requested post-refresh Claude CLI write-back, and the #1042/#70902 context supports the user problem, but this PR should not merge as written because it can write wrong-provider OAuth tokens and logs unredacted token-adjacent errors.

## What This Changes

The PR adds an OAuth-manager post-refresh hook that writes refreshed Claude CLI OAuth credentials back to Claude Code CLI storage through `writeClaudeCliCredentials`.

## Best Possible Solution

Keep the fix narrow: write back only validated `anthropic:claude-cli` / `claude-cli` OAuth credentials after successful refresh, keep Codex bootstrap-only, redact or omit write-back error details, add focused tests, and add the required changelog entry.

## Reproduction Assessment

Yes for the source-level behavior. Current main can refresh and persist an expired managed OAuth credential into OpenClaw stores, but the refresh path has no post-refresh call to `writeClaudeCliCredentials`; the live Claude CLI/Keychain race was not run in this read-only review.

## Solution Assessment

No, not as written. The direction is plausible, but the implementation needs provider/profile scoping, redacted token-adjacent logging, corrected scope comments, tests, and changelog coverage before it is the safest maintainable fix.
