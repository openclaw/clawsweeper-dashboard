## Summary

Keep open: current main still lacks the requested Claude CLI OAuth write-back path, but this PR should not merge as written because the diff writes OAuth material using profile-only scoping and adds a raw token-adjacent error log sink.

## What This Changes

The PR adds an OAuth-manager post-refresh hook that writes refreshed Claude CLI OAuth credentials back to Claude Code CLI storage.

## Best Possible Solution

Keep the fix narrow: write back only validated `claude-cli` credentials for `anthropic:claude-cli`, keep Codex bootstrap-only, redact or omit write-back error messages, and add focused tests plus the required changelog entry.

## Reproduction Assessment

Yes for the source-level behavior. Current main can refresh and persist an expired OAuth credential into OpenClaw stores, but the refresh path has no post-refresh external CLI writer; the live Claude CLI/Keychain race was not exercised.

## Solution Assessment

No, not as written. The direction is plausible, but profile/provider scoping and redacted logging need to be fixed before this is the safest maintainable implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard write-back by provider as well as profile:** `src/agents/auth-profiles/external-cli-sync.ts:170-171`
  - body: This writes to Claude CLI storage whenever the profile id is `anthropic:claude-cli`, but it does not verify that the refreshed credential is actually for provider `claude-cli`. A misconfigured or legacy profile using that id with another OAuth provider would copy unrelated tokens into Claude CLI storage, so require the expected provider/profile pair before calling the writer.
  - confidence: 0.86
- **[P2] Redact write-back error messages:** `src/agents/auth-profiles/external-cli-sync.ts:184-185`
  - body: The new token-adjacent catch block logs `String(error)`. Unexpected writer or adapter errors can include serialized credential payloads, so this should use the repo's redacting `formatErrorMessage()` path or omit the message before logging.
  - confidence: 0.9
- **[P3] Correct the write-back scope comment:** `src/agents/auth-profiles/external-cli-sync.ts:151-152`
  - body: The new JSDoc says this helper writes to Claude CLI and Codex CLI stores, but the implementation only handles `CLAUDE_CLI_PROFILE_ID`. Since Codex is bootstrap-only on current main, the comment should name only Claude CLI unless Codex write-back is actually implemented.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/auth-profiles/oauth.ts:165-168`
  - body: This is a user-facing auth fix, but the PR changes only auth TypeScript files and does not add a `CHANGELOG.md` entry. Add a single-line Fixes entry before merge.

## Security Review

Status: needs_attention

Summary: The diff introduces external OAuth secret write-back and needs provider scoping plus redacted error logging before merge.

Concerns:

- **[medium] Profile-only guard can write wrong-provider tokens:** `src/agents/auth-profiles/external-cli-sync.ts:170`
  - body: The proposed helper keys the external write solely on `CLAUDE_CLI_PROFILE_ID`; it should also require `credential.provider === "claude-cli"` before placing refreshed OAuth material into Claude CLI storage.
  - confidence: 0.86

  - body: `String(error)` is a new log sink around OAuth credential writes. If a thrown error embeds credential JSON or request payloads, access or refresh tokens could be written to logs.
