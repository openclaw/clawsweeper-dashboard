## Summary

Keep open: current main has upstream edit-result diff details and an optional diffs plugin, but it does not automatically display a unified edit diff in operator-visible OpenClaw tool output. That is a new UX behavior with truncation and privacy tradeoffs, so it needs maintainer product direction before implementation.

## What This Changes

Request to automatically show a unified diff after successful `edit` tool file changes.

## Best Possible Solution

Define a bounded edit-result presentation contract that surfaces successful edit diffs in the chosen operator-visible surfaces, reusing upstream `details.diff` with clear size and privacy limits while keeping the optional `diffs` plugin for richer artifacts.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows current OpenClaw display paths use text content and do not automatically promote edit `details.diff` into visible tool output.

## Solution Assessment

Unclear until maintainers choose the UX surface and limits. A maintainable implementation should likely reuse upstream `details.diff` with truncation and redaction rules rather than generating a separate diff path.
