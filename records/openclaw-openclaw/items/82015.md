## Summary

## What This Changes

The issue requests unified diff output after successful edit-tool file changes so operators can verify what changed.

## Best Possible Solution

Land or replace the linked PR with a current sessions edit-tool fix that populates diff and patch metadata on recovered successes using existing edit-diff helpers and regression coverage.

## Reproduction Assessment

Yes, at source level: a post-write error path that did apply the edit returns a success result with empty details.diff, and the HTML exporter only renders edit diffs when that field is truthy. I did not run a live recovered export scenario in this read-only review.

## Solution Assessment

Yes for the remaining fix boundary: the narrow maintainable solution is to populate diff metadata in src/agents/sessions/tools/edit.ts recovery success. The issue should stay open while the linked PR is reviewed, rebased, or replaced.
