## Summary

Keep open. Current main and the latest release still only parse and log the Feishu bot-added and bot-deleted subscription events, and neither the hook type contract nor public hook docs expose a Feishu channel hook event family.

## What This Changes

Expose Feishu bot added/removed group membership subscription events as internal hook events with chat/account/timestamp context.

## Best Possible Solution

Add Feishu-owned additive hook dispatch for bot-added and bot-removed events through the existing hook runtime, with focused tests and docs that only guarantee contract-confirmed payload fields.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence in the current gap: the Feishu handlers still log the two subscription events without any hook dispatch on current main and v2026.5.18.

## Solution Assessment

Yes. Additive Feishu plugin hook dispatch via the existing hook runtime is narrower than polling the chat list API or reviving a broad plugin hook-runner change, provided the payload fields are confirmed against Feishu/Lark docs.
