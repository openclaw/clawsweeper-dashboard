## Summary

Keep open. Current main and the latest release still parse the Feishu bot membership subscription events and only log them, while the internal hook type contract and public hook docs do not expose a Feishu/channel membership event surface.

## What This Changes

The issue requests `im.chat.member.bot.added_v1` and `im.chat.member.bot.deleted_v1` Feishu events to fire user-visible hook events with chat/account/timestamp context.

## Best Possible Solution

Add Feishu-owned internal hook dispatch for bot-added and bot-removed events, with typed context, focused tests, and public docs after confirming payload fields against the Feishu event envelope.

## Reproduction Assessment

Yes for source-level confirmation: current main registers both Feishu subscription event handlers and only logs them, while hook types and docs omit channel events. No live Feishu tenant reproduction was run.

## Solution Assessment

Yes in direction: Feishu-owned dispatch through the existing internal hook runtime is narrower than polling, but maintainers should settle event names and payload fields before implementation.
