## Summary

Keep open. Current main and the latest release still only parse and log the Feishu bot-added and bot-deleted subscription events, while the internal hook event type/docs do not expose a Feishu channel hook family. No linked or searched PR supersedes this request.

## What This Changes

Expose Feishu bot added/removed subscription events as user-space hook events with chat, account, and timestamp context.

## Best Possible Solution

Add Feishu-owned internal hook dispatch for `channel:feishu:bot-added` and `channel:feishu:bot-removed`, document the payload, and cover both handlers with focused lifecycle tests.

## Reproduction Assessment

Not applicable as a feature request. The current gap is high-confidence from source inspection: both Feishu subscription handlers still parse and log the events without hook dispatch on current main and v2026.5.12.

## Solution Assessment

Yes. Emitting additive Feishu-owned hook events through the existing internal hook runtime is narrower than polling the chat list API or moving Feishu-specific policy into core.
