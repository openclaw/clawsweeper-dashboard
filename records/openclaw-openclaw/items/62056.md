## Summary

Keep open. Current main still handles the Feishu bot added/deleted subscription events by parsing them and writing logs only; the requested hook events are not implemented or documented, and the existing plugin hook runtime makes this a narrow Feishu plugin plus additive hook-contract change.

## What This Changes

Expose Feishu bot-added and bot-removed membership subscription events to internal hooks with chat/account context instead of only logging them.

## Best Possible Solution

Add Feishu-owned hook dispatch for bot added/removed events, preserve the existing logs, document the event keys/context, and cover the dispatcher with focused Feishu monitor tests.

## Reproduction Assessment

Not applicable as a feature request. The current gap is source-verifiable: the registered Feishu bot added/deleted handlers on current main only reach the log branches in `extensions/feishu/src/monitor.account.ts`.

## Solution Assessment

Yes. Emitting already-received Feishu subscription events through the existing hook pipeline is the narrowest maintainable direction, with an additive documented hook contract rather than polling or core Feishu policy.
