## Summary

Current main already solves the central memory-search problem in the moved memory-core plugin path, and it does so more completely than this branch by preserving the full keyword-backed fallback window when strict hybrid scoring returns no results.

## What This Changes

The PR changes hybrid memory search fallback to derive the relaxed keyword threshold from post-merge keyword-backed entries and adds regression coverage for BM25 sub-1.0 and temporal-decay cases.

## Best Possible Solution

Keep the current memory-core fallback on main and avoid merging the obsolete src/memory variant.

## Reproduction Assessment

Source-level yes. The PR branch computes the relaxed fallback threshold as the maximum keyword-backed post-merge score, so multiple below-minScore lexical hits with different scores would collapse to the top tie group instead of filling maxResults.

## Solution Assessment

No. Current main's memory-core implementation is the better settled solution because it preserves the keyword-backed subset after merge-time scoring with a relaxed floor of 0, while this branch targets the removed src/memory path and narrows recall.
