## Summary

Current main already preserves keyword-backed hybrid memory results when strict merged scoring is empty, and it does so in the moved `extensions/memory-core` implementation with a broader fallback than this PR branch.

## What This Changes

The PR changes memory hybrid search fallback so strict-empty hybrid results return keyword-backed entries based on post-merge scores, with regression coverage for keyword-only and temporal-decay cases.

## Best Possible Solution

Keep the current `extensions/memory-core` fallback that returns keyword-backed FTS matches when strict hybrid scoring is empty; the stale `src/memory` branch no longer needs to merge.

## Reproduction Assessment

Yes. The original failure is source-reproducible by using hybrid search with `minScore=0.35`, `vectorWeight=0.7`, `textWeight=0.3`, and a keyword-backed result whose merged score falls below the strict floor; current main has regression coverage for that path.

## Solution Assessment

Yes. Current main's moved memory-core implementation is the better endpoint because it preserves the keyword-backed subset after merge-time scoring instead of keeping only the top post-merge score group from the stale PR branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Return all keyword fallback candidates:** `src/memory/manager.ts:361-367`
  - body: When `strict` is empty, every fallback candidate is below `minScore`; setting `relaxedMinScore` to the maximum keyword-backed score keeps only the top-scoring tie group and drops other valid lexical fallback hits. Use the current main approach of relaxing the keyword-backed subset to `0` so `maxResults` can return multiple matches.
