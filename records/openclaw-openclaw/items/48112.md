## Summary

Current main already implements the useful memory hybrid-search fallback in `extensions/memory-core`, with regression coverage and stronger recall than this PR branch. The fix is current-main-only for now; the latest release `v2026.5.7` still has the older fallback.

## What This Changes

This PR changes the old `src/memory` hybrid fallback and tests so keyword-backed `memory_search` results survive post-merge scoring effects when strict results are empty.

## Best Possible Solution

Close this obsolete branch and keep the current `extensions/memory-core` implementation from https://github.com/openclaw/openclaw/pull/78834; future memory-search tuning should happen in that plugin path.

## Reproduction Assessment

Yes. The linked report and `v2026.5.7` release source show the old hybrid fallback could still filter BM25-backed hits below the relaxed score floor, while current main has source and regression-test coverage for the fixed path.

## Solution Assessment

Yes for closing, not for merging this branch. Current main preserves the keyword-backed fallback set directly in the active memory-core manager, while this PR targets an obsolete path and keeps only the top post-merge score group.
