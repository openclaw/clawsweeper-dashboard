## Summary

Keep open: the reported memory-search bug still appears present in the active `memory-core` path, while this PR edits the old `src/memory` layout and its relaxed threshold would keep only the top keyword score group.

## What This Changes

This PR changes hybrid memory-search fallback thresholds to use post-merge keyword-backed scores and adds two regression tests under the old `src/memory` layout.

## Best Possible Solution

Port the fallback to `extensions/memory-core`, derive the relaxed keyword threshold from post-merge scores, and add current-layout regressions for BM25 `< 1` and enabled temporal decay.

## Reproduction Assessment

Yes. Source inspection on current main shows keyword-only merged scores can remain below the current `hybrid.textWeight` relaxed floor when BM25 is below 1.0 or temporal decay lowers the post-merge score.

## Solution Assessment

No. The approach is directionally right, but this PR is not the best current patch because it edits obsolete files and its max-score relaxed floor keeps only the top keyword score group.
