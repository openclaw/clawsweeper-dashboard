## Summary

Current main already implements the central hybrid memory fallback in `extensions/memory-core`, and the stale branch still targets the removed `src/memory` path while retaining a narrower top-score fallback behavior that can drop useful lexical matches.

## What This Changes

The PR changes the old `src/memory` hybrid-search fallback to derive a relaxed keyword-backed threshold from post-merge scores and adds old-path memory-index regression tests.

## Best Possible Solution

Close this obsolete branch and keep the current-main memory-core implementation from https://github.com/openclaw/openclaw/pull/78834; future tuning should happen in `extensions/memory-core`.

## Reproduction Assessment

Yes. The linked report and release source show BM25-only scores could fall below the default `minScore`, and current main now has source-level implementation plus regression coverage for the keyword-only fallback path.

## Solution Assessment

Yes for closing, not for merging this branch. Current main preserves keyword-backed fallback entries in the moved memory-core plugin path, while this branch targets removed files and can still drop lower-scored lexical fallback candidates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Return all keyword fallback candidates up to maxResults:** `src/memory/manager.ts:362-369`
  - body: When `strict` is empty, every keyword-backed candidate is already below `minScore`, so setting `relaxedMinScore` to the maximum keyword-backed score keeps only the top-scoring tie group. Queries with multiple lexical matches of different scores still lose valid fallback hits instead of returning up to `maxResults`; current main avoids this by relaxing the fallback floor to 0 for the keyword-backed subset.
  - confidence: 0.88
