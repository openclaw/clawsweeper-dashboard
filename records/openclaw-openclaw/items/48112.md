## Summary

Keep open: the reported memory-search bug still appears valid on current main, but this PR targets the old `src/memory` layout and should be ported into the active `memory-core` plugin before merge.

## What This Changes

The PR changes the old `src/memory` hybrid fallback and tests so keyword-backed fallback thresholds are derived from post-merge scores after decay/MMR.

## Best Possible Solution

Port the behavior into `extensions/memory-core`, add current-layout regressions for BM25 `textScore < 1` and temporal-decay fallback, and preserve keyword-backed candidates up to `maxResults` within a bounded relaxed score floor.

## Reproduction Assessment

Yes. Source inspection shows keyword-only merged scores can fall below the current `hybrid.textWeight` relaxed floor when BM25 text score is below 1.0 or temporal decay lowers the post-merge score.

## Solution Assessment

No. The direction is right, but this PR is not the best current patch because it edits the old memory layout and its max-score relaxed floor keeps only the top keyword score group instead of honoring recall up to `maxResults`.
