## What This Changes

The PR changes memory hybrid fallback filtering and regression tests so keyword-backed merged results survive when strict hybrid scoring returns no hits.

## Best Possible Solution

Keep the current main implementation from https://github.com/openclaw/openclaw/pull/78834 and close this older branch instead of merging a stale threshold variant over the moved memory-core path.

## Reproduction Assessment

Yes. Source inspection and the linked report show the old hybrid fallback could apply a second score threshold after merged scoring, so keyword hits below that threshold were dropped even when FTS found them; I did not run tests because this was a read-only review.

## Solution Assessment

No for this branch as-is. Current main is the better solution because it preserves the keyword-backed merged fallback set through `selectScoredResults(..., 0)` after strict results are empty instead of keeping only the top-scoring fallback candidates.
