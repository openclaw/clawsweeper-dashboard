## Summary

Keep open. Current main still couples signed macOS Sparkle automatic checks with background downloads under a default-on setting, and the macOS config reader still audits an update-channel-only clobber without restoring the active config.

## What This Changes

The issue asks for macOS companion updates/reinstalls to preserve `~/.openclaw` configuration and avoid automatically installing beta app updates after restart.

## Best Possible Solution

Keep this open until the macOS companion decouples automatic checks from background download/install behavior, avoids beta app updates without explicit opt-in, and restores or safely merges last-known-good config when an update-channel-only clobber is detected.

## Reproduction Assessment

Yes by source inspection. Current macOS test coverage writes a full config, replaces it with update-channel-only JSON, and asserts the loaded config has no gateway while the updater code still defaults signed builds into coupled automatic checks/downloads.

## Solution Assessment

No. The current implementation is not the best fix because it logs and snapshots suspicious macOS config loss without restoring active config, while the updater still drives checks and downloads from one default-on setting.
