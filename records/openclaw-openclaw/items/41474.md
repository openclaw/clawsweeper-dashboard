## Summary

Keep open. Current main still couples signed macOS Sparkle checks and downloads behind a default-on setting, and the update-channel-only config clobber path records audit evidence but leaves the active config clobbered; the reporter also confirmed the suggested workaround did not resolve it.

## What This Changes

The issue asks OpenClaw to prevent the macOS companion from automatically moving users to incompatible beta updates and to preserve or recover existing `~/.openclaw` configuration after reinstall.

## Best Possible Solution

Keep this open until the macOS companion has an explicit beta/update-download policy and the update-channel-only config clobber path restores, merges, or clearly routes recovery through a safe doctor-backed flow.

## Reproduction Assessment

Yes, by source inspection. The focused macOS config test writes a full config, replaces it with update-channel-only JSON, and current main expects the loaded config to have no gateway, matching the reported config-loss path without running a mutating test.

## Solution Assessment

No. The current implementation is not the best fix because it only audits and snapshots suspicious config loss while the updater still uses one default-on setting for both automatic checks and downloads.
