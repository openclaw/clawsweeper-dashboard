## Summary

Keep open. Current main still has the two core failure modes from the report: signed macOS builds default automatic update checks on and tie that same setting to background downloads, while suspicious update-channel-only config replacement is audited and snapshotted but not restored.

## What This Changes

The issue asks OpenClaw to prevent unexpected beta companion-app auto-updates and preserve or recover macOS `openclaw.json` configuration after companion reinstall/update flows.

## Best Possible Solution

Keep this open until the macOS companion separates checking from background download/install behavior, prevents beta app updates without explicit opt-in, and restores or safely merges last-known-good config when update-channel-only clobbering is detected.

## Reproduction Assessment

Yes, by source inspection. Current main still defaults signed macOS updater automation on, couples checks to background downloads, and has a regression test showing an update-channel-only config clobber loads without the previous gateway configuration.

## Solution Assessment

No. The current implementation is not the best solution because it audits suspicious config loss without recovery and uses one default-on setting for both checking and background downloads.
