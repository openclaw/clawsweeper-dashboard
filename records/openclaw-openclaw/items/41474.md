## Summary

Keep open. Current main still has the reported core failure modes: signed macOS builds default updater automation on, use the same setting for checks and background downloads, and the macOS config path audits an update-channel-only clobber without restoring the active configuration.

## What This Changes

The issue asks OpenClaw to preserve macOS configuration and avoid automatic beta companion app updates after reinstall/update.

## Best Possible Solution

Keep this open until the macOS companion separates update checks from background downloads, prevents beta app updates without explicit opt-in, and restores or safely merges last-known-good config when update-channel-only clobbering is detected.

## Reproduction Assessment

Yes, by source inspection. Current main still defaults signed macOS updater automation on, couples checks to downloads, and has a macOS test showing an update-channel-only config clobber loads without the previous gateway config.

## Solution Assessment

No. The current implementation is not the best solution because it audits suspicious config loss without macOS-side recovery and uses one default-on setting for both checking and background downloads.
