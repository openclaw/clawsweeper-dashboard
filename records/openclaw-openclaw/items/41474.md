## Summary

Keep this open: current main has partial config-clobber diagnostics and update-channel hardening, but the macOS companion still defaults Sparkle automatic checks/downloads on and the native config read path does not restore a clobbered update-only config.

## What This Changes

The issue requests preventing unintended macOS companion beta auto-updates and preserving or recovering the user state-dir config after companion reinstall/update.

## Best Possible Solution

Decide the companion updater policy, then align Sparkle defaults/channel eligibility with the documented update contract and add native config recovery or prevention for update-only clobbers.

## Reproduction Assessment

No end-to-end macOS/Sparkle reproduction was run in this read-only Linux review. Source inspection gives a clear partial path: signed companion builds default automatic Sparkle checks/downloads on, and the macOS config clobber test shows update-only config roots losing gateway data on load.

## Solution Assessment

No. Current main has useful forensics and some core recovery helpers, but it does not fully solve the reported companion updater policy or the native macOS config recovery behavior.
