## Summary

Current main addresses the reported macOS Companion App failure family: the stable appcast no longer advertises beta app builds, and the macOS config writer now blocks or preserves dangerous full-config writes instead of silently clobbering gateway state.

## What This Changes

The issue reports that macOS Companion App auto-updated from stable to beta and that reinstalling the stable app removed the user’s OpenClaw configuration.

## Best Possible Solution

Keep the current macOS config-writer guard and stable appcast handoff, then ship them in the next macOS release so users get the protection outside source builds.

## Solution Assessment

Yes. The current fix is the narrow maintainable solution for this bug family because it blocks stale/destructive fallback writes before replacement, preserves auth/mode state, and the appcast handoff addresses the accidental beta-update vector without changing broader updater policy.
