## Summary

Keep open. Current main still has signed-app Sparkle automatic checks/downloads tied to a default-on setting, and the macOS config-loss path still treats update-channel-only clobbers as audit-only instead of restoring active config; the reporter confirmed the suggested workaround did not help.

## What This Changes

The issue reports that the macOS Companion App automatically installed a beta update and that reinstalling the app caused the user’s OpenClaw configuration to disappear.

## Best Possible Solution

Keep this open until the macOS companion separates Sparkle check/download/install policy, requires explicit beta-channel opt-in for beta app updates, and restores or safely merges active config when an update-channel-only clobber is detected.

## Reproduction Assessment

Yes. Source inspection gives a focused current-main reproduction: `OpenClawConfigFileTests` writes a full config, replaces it with update-channel-only JSON, and currently expects the loaded config to have no gateway instead of recovering it.

## Solution Assessment

No. The current implementation is not the best fix because it logs and snapshots suspicious config loss without restoring the active config, while the updater still drives checks and downloads from one default-on setting.
