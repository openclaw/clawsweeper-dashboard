## Summary

Keep open: current main still has the OpenResponses HTTP commentary leak, and this PR is useful but still incorrect because commentary can remain in the final visible SSE text after delta suppression. The PR also lacks after-fix real behavior proof.

## What This Changes

The PR changes the OpenResponses HTTP SSE handler and Gateway tests to buffer assistant text by phase, suppress commentary deltas, and assert phase metadata on completed output items.

## Best Possible Solution

Keep the PR open and change the streaming handler so hidden commentary never feeds visible SSE delta/done/completed text while phase-tagged structured output remains intact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: emit a commentary assistant chunk, then a final_answer chunk, then lifecycle end; PR head can keep both in `accumulatedText` and finalize visible SSE events from that buffer.

## Solution Assessment

No. Buffering by phase is the right direction, but the branch needs a final-answer-only visible buffer plus mixed commentary/final_answer coverage before merge.

## Review Findings
