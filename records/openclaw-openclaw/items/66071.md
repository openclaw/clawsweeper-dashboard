## Summary

Keep open: current main still has the OpenResponses HTTP commentary leak, but this PR is not merge-ready because it still sends commentary in terminal SSE text events, lacks real behavior proof, and is currently dirty against main.

## What This Changes

The PR buffers OpenResponses HTTP assistant stream text by phase so commentary deltas are not emitted as visible SSE deltas while adding regression tests for commentary and final-answer streaming.

## Best Possible Solution

Rework the streaming handler so only user-visible final-answer text is used for unphased SSE text events, commentary remains distinguishable only in phase-tagged output items, and the refreshed branch includes real HTTP/SSE proof.

## Reproduction Assessment

Yes from source inspection: on the PR head, a phase: commentary assistant event is appended to accumulatedText and then emitted through terminal SSE text events. Current main also lacks phase gating for ordinary response.output_text.delta events, so the central bug is still plausible on current main.

## Solution Assessment

No as-is. The PR is in the right gateway layer, but the best fix must maintain separate commentary and visible-output state instead of hiding only deltas while reusing accumulatedText for final SSE events.
