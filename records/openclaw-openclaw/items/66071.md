## Summary

Keep this PR open: the current `main` path still has the OpenResponses commentary leak, but the PR head still lets commentary enter finalized visible SSE text, so it is not safe to merge as-is.

## What This Changes

The PR changes OpenResponses HTTP SSE streaming to buffer assistant text by phase, suppress commentary deltas, preserve output phase metadata, and add Gateway regression tests.

## Best Possible Solution

Land a corrected Gateway HTTP streaming fix that keeps commentary out of all visible SSE text events, preserves structured phase metadata, covers mixed commentary/final-answer plus tool-call streaming, and includes redacted real SSE proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: emit a commentary assistant chunk followed by a final-answer chunk for one streaming `/v1/responses` run, then inspect `response.output_text.done` or `response.completed` for the commentary text.

## Solution Assessment

No. Buffering by phase is the right direction, but the PR still uses the all-assistant `accumulatedText` buffer for finalized visible SSE events instead of a final-answer-only visible-text buffer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep commentary out of finalized visible text:** `src/gateway/openresponses-http.ts:963-964`
  - body: The new guard suppresses commentary deltas, but commentary chunks are still appended to `accumulatedText` before this return. Later finalization uses that same buffer for `response.output_text.done` and `response.completed`, so a run that emits commentary and then `final_answer` can leak hidden text at completion and produce final text that does not match the streamed deltas.
  - confidence: 0.88
