## Summary

Keep open: this is a valid Gateway bug and current main still streams assistant text without phase filtering, but the PR head remains incorrect because commentary can still reach final SSE events through the shared accumulated text buffer; required real behavior proof is also missing.

## What This Changes

The PR changes OpenResponses HTTP SSE streaming to buffer assistant text by phase, suppress commentary deltas, preserve phase metadata, and add Gateway regression tests.

## Best Possible Solution

Land a corrected Gateway streaming fix that separates hidden commentary from user-visible final text, preserves phase metadata for structured output, covers mixed commentary/final_answer streaming, and includes redacted runtime SSE proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: emit a commentary assistant chunk, then a final_answer chunk, then lifecycle end for one streaming `/v1/responses` run; final SSE events still read from `accumulatedText`, which includes commentary.

## Solution Assessment

No. Buffering by phase is the right direction, but the PR still drives finalized visible SSE text from the all-assistant buffer instead of a final-answer-only visible text buffer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep commentary out of finalized visible text:** `src/gateway/openresponses-http.ts:963-964`
  - body: The new guard suppresses commentary deltas, but commentary chunks are still appended to `accumulatedText` before this return. Later finalization uses that same buffer for `response.output_text.done`, `response.content_part.done`, and `response.completed`, so a run that emits commentary and then `final_answer` can still leak hidden text at completion and produce final text that does not match the streamed deltas.
  - confidence: 0.88
