## Summary

Keep open. Current main still does not suppress phase-tagged commentary in the OpenResponses HTTP SSE handler, and the proposed patch is not merge-ready because hidden commentary can still be reused as finalized visible text.

## What This Changes

This PR changes the OpenResponses HTTP streaming path to buffer phase-tagged assistant text, suppress commentary `response.output_text.delta` events, and add regression coverage for commentary suppression plus explicit final-answer streaming.

## Best Possible Solution

Land a corrected Gateway HTTP streaming fix that keeps user-visible SSE text based only on final-answer or otherwise visible chunks, preserves phase-tagged structured output for tool-call turns, and adds mixed commentary/final-answer regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A focused OpenResponses HTTP SSE test can emit a `phase: "commentary"` assistant chunk followed by `phase: "final_answer"`; the current patch path still stores the commentary in `accumulatedText`, which is later used for final events.

## Solution Assessment

No. Buffering by assistant phase is the right direction, but this implementation needs a separate visible-text buffer and finalization path instead of reusing the all-assistant `accumulatedText` buffer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep commentary out of finalized visible text:** `src/gateway/openresponses-http.ts:963-964`
  - body: `phase === "commentary"` chunks are still appended to `accumulatedText` before the new suppression guard. When a stream emits commentary and later final-answer text, finalization uses that buffer for `response.output_text.done` and `response.completed`, so the hidden commentary can reappear at completion even though it was not streamed as a delta.
  - confidence: 0.88
