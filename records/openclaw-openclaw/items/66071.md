## Summary

Keep open: the PR targets a real OpenResponses HTTP SSE commentary leak, but the latest head still allows commentary to reach final SSE payloads and the contributor has not provided required real behavior proof.

## What This Changes

The PR buffers OpenResponses `/v1/responses` SSE assistant text by phase, suppresses commentary deltas, preserves phase metadata on completed output items, and adds Gateway regression tests.

## Best Possible Solution

Separate hidden commentary from user-visible final text in the OpenResponses streaming handler, keep phase metadata for structured output, add mixed commentary/final_answer regression coverage, and attach redacted live SSE proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: emit a commentary assistant chunk, then a final_answer chunk, then lifecycle end; PR head appends the commentary to `accumulatedText` and finalizes visible SSE events from that buffer.

## Solution Assessment

No. Buffering by phase is the right direction, but this patch still uses the all-assistant buffer for final visible events; the safer fix is to maintain a final-answer-only visible buffer while preserving phase-tagged structured output.

## Review Findings
