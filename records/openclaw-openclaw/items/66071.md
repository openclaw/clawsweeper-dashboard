## Summary

Keep open: the underlying OpenResponses commentary leak is still real on current main, but this PR is not merge-ready because it suppresses commentary deltas while still letting commentary reach final visible SSE payloads and it lacks real HTTP SSE proof.

## What This Changes

The PR changes OpenResponses HTTP SSE streaming to buffer assistant deltas until phase is known and adds gateway regression tests for commentary suppression and final-answer streaming.

## Best Possible Solution

Track user-visible SSE text separately from raw assistant/commentary text, preserve current file-content and abort behavior, and land only with mixed commentary/final-answer regression coverage plus real HTTP SSE proof.

## Reproduction Assessment

Yes from source: current main still streams assistant deltas without a phase check, and the PR head still appends commentary to the buffer used for final visible SSE events. I did not run a live HTTP repro in this read-only review.

## Solution Assessment

No as written. The narrow maintainable fix should separate raw assistant structure from user-visible final text instead of finalizing visible events from an all-phase buffer.
