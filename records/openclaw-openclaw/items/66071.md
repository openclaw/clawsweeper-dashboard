## Summary

Keep this PR open: it addresses a real OpenResponses Gateway commentary leak, but the current patch still allows hidden commentary to appear in finalized visible SSE payloads and lacks after-fix real behavior proof.

## What This Changes

The PR changes `src/gateway/openresponses-http.ts` and focused Gateway tests to buffer assistant text by phase, suppress commentary SSE deltas, preserve phase metadata, and cover commentary/final-answer streaming cases.

## Best Possible Solution

Revise the handler to keep a final-answer-only visible buffer for all visible SSE payloads while preserving phase-tagged structured output for tool-call turns, then add mixed-phase coverage and real SSE proof.

## Reproduction Assessment

Yes, by source inspection: emit a commentary assistant chunk, then a final_answer chunk, then lifecycle end; PR head keeps both in `accumulatedText` and finalizes visible SSE payloads from that buffer.

## Solution Assessment

No. Buffering by phase is the right direction, but the implementation needs a separate visible final-answer buffer and mixed-phase regression coverage before it is the maintainable fix.
