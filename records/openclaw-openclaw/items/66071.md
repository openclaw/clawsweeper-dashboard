## Summary

Keep open: current main still has the OpenResponses SSE commentary leak, and this PR is a useful targeted attempt, but the current head is not merge-ready because hidden commentary can still reach final SSE payloads and the PR has only mock/test proof.

## What This Changes

The PR buffers OpenResponses streaming assistant deltas by phase, suppresses commentary deltas, preserves output item phase metadata, and adds two HTTP SSE regression tests.

## Best Possible Solution

Rework the current-main streaming handler so raw/commentary and client-visible final-answer text stay separate across delta, done, completed, fallback, and tool-call paths, then land it with regression tests plus real redacted HTTP SSE proof.

## Reproduction Assessment

Yes from source: current main streams assistant deltas without phase filtering, and the PR head appends commentary to accumulatedText before finalization sends that text as output_text.done/completed. I did not run a live HTTP repro in this read-only review.

## Solution Assessment

No as-is. The right fix is a current-main change that keeps commentary/raw text separate from client-visible final-answer text through every SSE event path, rather than only suppressing the delta event.

- **[P1] Keep hidden commentary out of final SSE text:** `src/gateway/openresponses-http.ts:963-964`
  - body: This suppresses `phase: "commentary"` from `response.output_text.delta`, but it still appends that same chunk to `accumulatedText` before returning. Track visible/final-answer text separately from raw/commentary text and finalize from the visible buffer.
