## Summary

Keep open. Current main still streams every assistant delta without phase filtering, while this PR head still appends commentary chunks to the final text buffer before suppressing deltas, so mixed commentary plus final-answer runs can leak hidden text at completion.

## What This Changes

The PR changes OpenResponses HTTP SSE streaming to buffer assistant text by phase, suppress commentary deltas, preserve phase metadata, and add focused gateway regression tests.

## Best Possible Solution

Land a corrected Gateway HTTP streaming fix that separates visible final-answer text from structured commentary output, preserves phase metadata, and covers mixed commentary plus final-answer streaming with real redacted SSE proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: emit a commentary assistant chunk followed by a final_answer chunk in one streamed run, and PR head keeps the commentary in `accumulatedText` that is later emitted as final visible text.

## Solution Assessment

No. Buffering by phase is the right direction, but finalization must use a visible-text-only buffer and the regression coverage must include mixed commentary plus final-answer streaming.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep commentary out of finalized visible text:** `src/gateway/openresponses-http.ts:963-964`
  - body: The new guard suppresses commentary deltas, but commentary chunks are still appended to `accumulatedText` before this return. Later finalization uses that same buffer for `response.output_text.done` and `response.completed`, so a run that emits commentary and then final_answer can leak hidden text at completion and produce final text that does not match streamed deltas.
