## Summary

Keep open. The underlying Gateway SSE bug is still valid on current main, but this PR head remains incorrect because hidden commentary can be reused for final SSE events, and the external PR only provides mocked test/build validation rather than real behavior proof.

## What This Changes

This PR changes `/v1/responses` SSE streaming to buffer assistant text by phase, suppress commentary deltas, preserve phase metadata, and add OpenResponses HTTP regression tests.

## Best Possible Solution

Land a corrected Gateway HTTP streaming fix that separates visible final-answer text from structured commentary output, preserves assistant phase metadata, and covers mixed commentary/final-answer plus tool-call streaming with real redacted SSE proof.

## Reproduction Assessment

Yes. A focused SSE test can emit a commentary assistant chunk followed by a `final_answer` chunk for the same run; source inspection of the PR head shows both chunks enter `accumulatedText`, which is later used for final SSE events.

## Solution Assessment

No. Buffering by phase is the right direction, but finalization must use a visible-text buffer rather than the all-assistant `accumulatedText` buffer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep commentary out of finalized visible text:** `src/gateway/openresponses-http.ts:964`
  - body: The new guard suppresses commentary deltas, but commentary chunks are still appended to `accumulatedText` before this return. Later finalization uses that same buffer for `response.output_text.done` and `response.completed`, so a run that emits commentary and then `final_answer` can leak hidden text at completion and produce final text that does not match the streamed deltas.
  - confidence: 0.88
