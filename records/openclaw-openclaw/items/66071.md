## Summary

Keep open: current main still streams every OpenResponses assistant delta, and this PR is useful but not merge-ready because its latest head can still expose commentary through final SSE events.

## What This Changes

This PR changes OpenResponses HTTP SSE streaming to buffer assistant deltas by phase, suppress commentary deltas, preserve phase metadata, and add regression tests.

## Best Possible Solution

Land a corrected Gateway HTTP streaming fix that separates visible text from structured commentary output, preserves assistant phase metadata, and adds mixed commentary/final-answer coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A focused SSE test can emit a commentary assistant chunk followed by a final_answer chunk for the same run; the PR head appends the commentary to accumulatedText and later uses that buffer for final SSE events.

## Solution Assessment

No. Buffering by phase is the right direction, but finalization must use a visible-text buffer instead of the all-assistant accumulatedText buffer.
