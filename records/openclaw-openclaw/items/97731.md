## Summary

Keep open: this PR is aimed at a real P1 process-leak report, but the proposed SIGCHLD reaper is not merge-ready because libuv only waits tracked child handles rather than doing the global orphan sweep the patch relies on.

## What This Changes

The PR adds a SIGCHLD-based zombie reaper, wires it into gateway startup as a periodic timer, and schedules reaping after exec/supervisor process kills.

## Best Possible Solution

Replace the synthetic SIGCHLD approach with a fix at the actual leaking process owner or runtime init/subreaper boundary, backed by Linux/Docker process-table proof.

## Reproduction Assessment

## Solution Assessment

No. A synthetic SIGCHLD nudge is not the best fix because the actual process owner must wait/reap the child, or the runtime/container must provide a real subreaper/init boundary.
