## Summary

Keep open: the report is recent and concrete, and current main still lacks focused proof that Claude stream-json assistant-frame API/socket errors are classified as terminal recoverable failures or retried safely rather than waiting for timeout/reconciliation.

## What This Changes

Claude Code SDK/Claude CLI subagent SSE socket-close failures should be retried or surfaced as terminal recoverable failures instead of being written as normal transcript text and leaving parent Agent calls pending.

## Best Possible Solution

Classify Claude stream-json assistant-frame API/socket errors as terminal recoverable failures at the CLI runtime boundary, notify through existing subagent/task completion paths, and add direct retry only where duplicate tool side effects are ruled out.

## Reproduction Assessment

No high-confidence standalone repro is available yet. Source inspection shows a plausible current-main path where a Claude assistant-frame API Error is parsed as non-terminal until a result, exit, or watchdog timeout, but a redacted failing JSONL/log sample is still needed.

## Solution Assessment

Unclear as written: upstream SDK retry may be useful, but the safer OpenClaw boundary is to fail fast and notify through existing runtime/failover paths first. Any automatic retry should be added only where duplicate tool side effects are ruled out.
