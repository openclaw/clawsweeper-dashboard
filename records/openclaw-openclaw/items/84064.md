- `impact:session-state`: The issue is about parent/subagent run state drifting when the child stream stops or completes as misleading text.

## Summary

Keep open. Current main has related recovery for stale subagent completion announcements, but it does not fully cover this narrower Claude CLI/SDK stream-error report: the live Claude CLI path still treats non-`is_error` stream-json results as successful text, and OpenClaw does not implement safe retry semantics for mid-turn SSE/socket failures.

## What This Changes

The issue requests that Claude Code SDK subagent SSE/socket-close failures retry or surface as abnormal completion instead of being written into transcript text while the parent Agent tool waits indefinitely.

## Best Possible Solution

Add an OpenClaw-owned fail-visible path for Claude CLI `API Error:` stream-json output, document the failure mode, and only add automatic retry/resume after maintainers choose safe semantics for side-effecting subagent work.

## Reproduction Assessment

Yes at source level for the silent text-write part: current main can classify explicit failed Claude CLI output, but the live result path only rejects `is_error` results and otherwise parses result text as success. No live SSE-drop reproduction was run against current main.

## Solution Assessment

Unclear. Failing visibly on Claude CLI API-error text is a narrow OpenClaw fix, but automatic retry/backoff needs a maintainer decision because it may rerun tool calls after a partial subagent turn.
