## Summary

Keep open: current main handles some Claude CLI live-session failures, but the reported assistant-frame `API Error:`/socket-close shape still maps to a plausible gap where non-`result` stream-json records are treated as normal streaming output instead of terminal failure. The issue is recent, concrete, and not fully superseded by the broader subagent reliability umbrella.

## What This Changes

The issue asks OpenClaw to treat Claude Code subagent SSE/socket-close failures as retryable or terminal errors instead of silently writing error text and leaving parent Agent calls pending.

## Best Possible Solution

Classify Claude CLI stream-json assistant API/socket failure frames as terminal recoverable failures at the OpenClaw runtime boundary, route them through existing failover and subagent completion paths, and add direct retries only where the turn is proven idempotent.

## Reproduction Assessment

No high-confidence live reproduction is available yet. The issue provides concrete v2026.5.18 session examples, and current-main source inspection shows a plausible path where assistant-frame `API Error:` records avoid immediate terminal failure, but a redacted JSONL/log fixture is still needed.

## Solution Assessment

Unclear as written: retrying inside the Claude SDK may be ideal upstream, but OpenClaw's safer first step is to fail fast and notify through its own runtime/failover/subagent completion paths. Automatic retry should be added only after idempotency and duplicate tool side effects are addressed.
