Keep open: the report is recent, concrete, and current main still has a plausible Claude live-session path where assistant-frame API/socket error text is parsed as a non-result record rather than immediately failing the turn or notifying the parent subagent path.

## What This Changes

Claude CLI subagent stream/socket/API-error failures should fail or retry through the Agent/subagent completion path instead of being recorded as normal assistant text and leaving parent calls pending.

## Best Possible Solution

Fail fast on Claude CLI assistant-frame API/socket error records at the live-session boundary and route them through existing failover, subagent completion, and notification paths; add retries only where idempotency is proven or explicitly approved.

## Reproduction Assessment

No high-confidence standalone live reproduction is available yet. The issue gives concrete session examples, and current-main source shows a plausible path for assistant-frame API errors to avoid immediate terminal failure, but a redacted failing JSONL/log fixture is still needed.

## Solution Assessment

Unclear as written: upstream Claude SDK retry may be ideal for socket drops, but OpenClaw's safer first fix is to fail fast and notify through its runtime/failover paths. Any automatic retry should be added only where duplicate tool side effects are ruled out.
