## What This Changes

The issue requests that Claude CLI subagent SSE/socket failures retry or fail visibly instead of being written as normal transcript text while the parent Agent call remains open.

## Best Possible Solution

Classify Claude CLI stream-json API/socket failure frames as terminal recoverable failures at the OpenClaw runtime boundary, then route them through existing failover, subagent completion, and task notification paths; only add direct retries where the turn can be proven idempotent.
