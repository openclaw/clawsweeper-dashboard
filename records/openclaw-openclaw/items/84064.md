- `P1`: The report describes current-release Claude subagent runs that can leave parent Agent work pending during long-running user workflows.
- `impact:session-state`: The reported failure can leave parent and child run state diverged when the child stream dies without a terminal completion event.

## Summary

Keep open: the report is recent, concrete, and current main still does not clearly treat a Claude live-session assistant-frame `API Error:` record as an immediate terminal failure. The adjacent stale-completion recovery work helps related symptoms, but it does not supersede this narrower Claude stream-error boundary question.

## What This Changes

The issue asks OpenClaw to retry or fail fast when Claude Code subagent stream-json/SSE socket failures are written as transcript text instead of leaving the parent Agent call pending.

## Best Possible Solution

Classify Claude CLI assistant-frame `API Error:` stream-json records as terminal recoverable failures at the OpenClaw CLI boundary, route them through existing failover and subagent notification paths, and add direct retries only where the whole turn is proven idempotent.

## Reproduction Assessment

No high-confidence standalone reproduction is available yet. Source inspection shows the reported assistant-frame `API Error:` shape would not immediately fail a Claude live turn, but a redacted JSONL/log or live repro is still needed to prove the full parent-stall behavior on current main.

## Solution Assessment

Unclear as written: fail-fast classification at the OpenClaw CLI boundary is the safest first fix, while unconditional retry needs maintainer approval because duplicated native Agent/tool side effects are possible.
