## Summary

Keep open: current main and the latest stable release still leave the reported Claude CLI stream/API-error subagent path plausible, and automatic retry is not yet a safe maintainer-approved behavior for side-effectful subagent turns.

## What This Changes

The issue asks OpenClaw to treat Claude Code subagent SSE/socket-close stream failures as retryable or terminal errors instead of allowing a parent Agent call to wait indefinitely.

## Best Possible Solution

Classify explicit Claude CLI stream API-error/socket-close states as terminal recoverable failures at the OpenClaw CLI boundary, then add automatic retry only where idempotency is proven.

## Reproduction Assessment

No live current-main reproduction was run; source inspection shows a plausible path where assistant `API Error:` stream-json text is non-terminal unless a later `result.is_error` or process failure arrives.

## Solution Assessment

Unclear as stated: retrying the SDK stream is not automatically safe for side-effectful subagent turns, so fail-fast terminal classification is the narrower first fix.
