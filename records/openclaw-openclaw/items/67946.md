## What This Changes

The PR gates persisted spawn lineage to subagent/ACP sessions, avoids stale `sessionEntry.spawnedBy` fallback for ordinary agent runs, adds regression coverage, and moves extension tests to SDK `test-env` platform helpers.

## Best Possible Solution

Land this focused stale-lineage fix after updating the branch onto current main and preserving the existing fork-marker behavior, then let the linked issue close through the merge.

## Reproduction Assessment

Yes. At source level, seed an ordinary session like `agent:main:main` with stale `spawnDepth` or `spawnedBy`; current main reads that stored lineage before `sessions_spawn` computes depth and can reject with the configured depth limit.

## Solution Assessment

Yes. Gating persisted lineage to real subagent/ACP keys in both session initialization and agent-command fallback is the narrow maintainable fix, while preserving live `normalizedSpawned` metadata and `forkedFromParent`.
