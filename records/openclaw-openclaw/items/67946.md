## Summary

## What This Changes

The PR clears persisted subagent lineage fields for ordinary top-level sessions, preserves them for subagent/ACP session keys, stops normal agent attempts from using stored `spawnedBy`, and adds a regression test.

## Best Possible Solution

Land this focused patch after contributor-provided real behavior proof and fresh maintainer validation confirm the rebased diff remains equivalent.

Yes, at source level: seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, run another ordinary top-level turn, and then `sessions_spawn` can compute depth from that stored lineage. I did not run a mutating live reproduction because this review was read-only.

## Solution Assessment

Yes. The PR uses the existing session-key classifiers at the session-state and attempt boundaries, preserving real subagent/ACP lineage while clearing stale top-level lineage.
