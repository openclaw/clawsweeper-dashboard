## Summary

Keep open: current main still accepts longer slash command names through built-in prefix checks, so a prompt-style skill command such as `/config-check` can be parsed as `/config` before the skill turn reaches the agent path.

## What This Changes

The issue requests a slash-command boundary fix so built-in commands like `/config` do not intercept longer skill command names such as `/config-check`.

## Best Possible Solution

Add a reusable slash command boundary helper, apply it to shared and direct built-in slash parsing, and add regression coverage for longer skill command names falling through correctly.

## Reproduction Assessment

Yes, from source inspection: `/config-check` satisfies the shared `/config` prefix parser and the prompt-style skill path later runs built-in command handlers with the original command object. I did not execute a live or Vitest reproduction in this read-only review.

## Solution Assessment

Yes: the maintainable fix is a narrow command-token boundary helper reused by the shared parser and direct built-in handlers, with regression tests for valid `/config` forms and longer command names.
