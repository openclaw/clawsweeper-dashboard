## Summary

Keep open: current main and v2026.5.22 still lack the requested named `/session save` and `/session load` snapshot workflow; existing compaction checkpoint restore, native subagent transcript fork, ACP resume, and the open `/resume` PR cover narrower slices.

## What This Changes

Requested explicit session snapshot save/load/fork/restore workflows that preserve or rehydrate conversation context and related session state.

## Best Possible Solution

Define one native session snapshot lifecycle over session storage, compaction checkpoints, reset/deleted archives, and subagent/ACP fork primitives, then expose agreed save/list/load/branch/restore behavior through slash commands, Gateway protocol, SDK/UI/CLI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, protocol, latest-release, and related-item checks give high confidence that OpenClaw lacks the requested named save/load snapshot workflow while adjacent primitives exist.

## Solution Assessment

Unclear pending maintainer API direction. The requested commands are plausible, but the maintainable path is a unified session snapshot lifecycle contract rather than treating compaction restore, archive restore, ACP resume, or subagent transcript fork as the whole solution.
