## Summary

Keep open: current main has adjacent compaction checkpoint branch/restore and `sessions_spawn` transcript-fork primitives, but it still lacks the requested named `/session save` and `/session load` workflow, reusable snapshot templates, and restore/import from `.reset` or `.deleted` transcript archives.

## What This Changes

Request explicit session snapshot commands and APIs for saving, loading, restoring, and forking conversation context checkpoints.

## Best Possible Solution

Define a native snapshot design that reuses session-store metadata, compaction checkpoint artifacts, reset/deleted archives, and transcript-fork support, then add save/list/load/branch semantics across Gateway protocol, slash commands, UI/CLI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but named save/load and archive restore do not.

## Solution Assessment

No; adding only two slash commands would not be the best fix because the request also covers archive import, branching, reusable templates, retention, permissions, and cache-sensitive fork behavior. The maintainable path is a protocol/UX design first, then implementation over the existing primitives.
