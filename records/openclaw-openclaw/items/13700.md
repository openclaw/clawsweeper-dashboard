## Summary

Keep open: current main has adjacent compaction-checkpoint branch/restore, TUI/internal resume/fork primitives, and subagent transcript forks, but it still lacks the requested named native `/session save` and `/session load` snapshot workflow, reusable templates, and archive restore/import contract.

## What This Changes

Request named session snapshot commands and APIs to save, load, restore, and fork conversation context checkpoints.

## Best Possible Solution

Define a canonical session snapshot contract that uses DAG-safe SessionManager/runtime storage, separates branch, restore, template, and archive-import semantics, then add slash/Gateway/UI/docs/tests around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction; source, protocol, UI, tests, and docs give high-confidence evidence that named native save/load snapshots are absent on current main.

## Solution Assessment

No: the user-facing direction is useful, but the proposed raw JSONL checkpoint store and single load operation are not the best implementation under current storage, transcript-DAG, lifecycle-hook, and branch-versus-restore constraints.
