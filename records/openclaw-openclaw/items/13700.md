Keep open: current main and the latest release still lack the requested named session snapshot save/load workflow, and the related checkpoint, fork, and resume work only covers narrower or broader overlapping slices that need maintainer product/API direction.

## Best Possible Solution

Define one canonical session snapshot contract that composes SessionManager transcript DAGs, session-store metadata, compaction checkpoints, reset/deleted archives, Gateway protocol, slash commands, and docs before implementing save/list/load/branch behavior.
