## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and native subagent transcript-fork primitives, but it still does not implement the requested named `/session save` and `/session load` workflow, reusable session templates, or restore/import from `.reset` and `.deleted` transcript archives; the recent maintainer comment also explicitly kept this design open.

## What This Changes

Request adding session snapshot save/load checkpoints, restore/branching, and reusable session-state workflows for long-running OpenClaw sessions.

## Best Possible Solution

Define one native session snapshot model that builds on SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then expose coherent save/list/load/branch/import semantics through chat commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs: `/session` only accepts `idle`/`max-age`, no exact save/load command surface exists, and the available checkpoint/fork primitives are narrower than named snapshots.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs maintainer decisions on restore semantics, retention, archive import, permissions, and cache-preserving fork behavior before a narrow maintainable fix can be chosen.
