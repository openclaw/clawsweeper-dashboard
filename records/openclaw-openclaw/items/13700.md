## Summary

Keep open: current main and v2026.5.20 still lack the requested named `/session save` and `/session load` snapshot workflow; the existing compaction checkpoint, ACP resume, and archive-restore PRs are adjacent but do not settle the broader session snapshot contract.

## What This Changes

The issue requests named session snapshots that can save, load, restore, and fork conversation context for rollback, branching, reproducibility, and archive recovery.

## Best Possible Solution

Define one native session snapshot lifecycle over session storage, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore behavior through commands, Gateway protocol, SDK/UI/CLI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, protocol, release-tag, and live related-item inspection give high confidence that current OpenClaw lacks the requested named save/load snapshot workflow while adjacent primitives exist.

## Solution Assessment

Unclear pending maintainer API direction. The requested slash commands are plausible, but the maintainable path is a unified session snapshot lifecycle contract rather than treating compaction restore, archive restore, or subagent transcript fork as the whole solution.
