## Summary

Keep open: current main has useful adjacent compaction checkpoint branch/restore primitives and an open partial restore PR, but it still does not implement named session save/load snapshots, reusable templates, or reset/deleted archive import/fork semantics.

## What This Changes

Request native session snapshot commands and APIs to save, load, restore, and fork long-running conversation context checkpoints.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore semantics through protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap: adjacent compaction and subagent primitives exist, but named save/load snapshots, reusable templates, and archive import are not implemented.

## Solution Assessment

No complete implementation exists yet. A restore-only patch may cover one recovery case, but the maintainable solution is a product-approved session snapshot design before standardizing save/load/fork semantics.
