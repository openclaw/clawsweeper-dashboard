## Summary

Keep open: current main and v2026.6.8 still lack the requested named `/session save` and `/session load` snapshot workflow, reusable session templates, and archive import/restore semantics. Existing compaction checkpoint branch/restore, internal fork helpers, and narrower related issues/PRs are useful adjacent work but do not implement the broad session snapshot feature.

## What This Changes

Request adds named session snapshots with `/session save <name>` and `/session load <name>`, plus restore/fork workflows for preserving and branching session state.

## Best Possible Solution

Define one canonical native session snapshot contract that composes current session persistence, transcript DAGs, compaction checkpoints, reset archives, model/provider state, Gateway protocol, slash commands, SDK/UI affordances, docs, tests, and security semantics before implementing save/list/load/branch behavior.
