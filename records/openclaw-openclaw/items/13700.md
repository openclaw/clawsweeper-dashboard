## Summary

Keep open: current main and the latest release still lack the requested named `/session save` and `/session load` snapshot workflow, while adjacent compaction checkpoint, archive, ACP resume, and subagent fork primitives cover only narrower pieces that need product/API scoping.

## Best Possible Solution

Define one native session snapshot lifecycle over session storage, compaction checkpoints, reset/deleted archives, and subagent/ACP fork primitives, then expose agreed save/list/load/branch/restore behavior through slash commands, Gateway protocol, UI/CLI/SDK where appropriate, docs, and focused tests.

## Solution Assessment
