## Summary

Keep open: current main has adjacent compaction-checkpoint branch/restore, reset/deleted transcript archives, and subagent transcript fork primitives, but it still lacks the requested named save/load snapshot workflow, reusable templates, and archive import/restore semantics.

## What This Changes

The issue requests native session snapshots with named `/session save` and `/session load` checkpoints, branching/forking, and recovery of preserved context.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch behavior through the appropriate protocol, command, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but named save/load and archive restore do not.

## Solution Assessment

No current implementation is the complete solution. Adding only two slash commands would be too narrow because the request also covers archive import, branching, reusable templates, retention policy, and cache-sensitive fork behavior.
