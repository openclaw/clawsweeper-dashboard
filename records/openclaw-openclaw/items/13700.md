## Summary

Keep open: current main still has only adjacent compaction-checkpoint restore/branch and native subagent fork primitives, while the requested named save/load snapshots, reusable templates, and `.reset`/`.deleted` archive restore/import semantics remain unresolved. The live discussion already treats this issue as the canonical tracker, and the active restore-command PR at https://github.com/openclaw/openclaw/pull/51889 is a partial implementation candidate with unresolved review and proof concerns.

## What This Changes

Add a native session snapshot workflow for saving, loading, restoring, and branching conversation/session state from named checkpoints or archived transcripts.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch/restore behavior through the appropriate protocol, command, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap: adjacent compaction and ACP/subagent primitives exist, but named save/load snapshots, reusable templates, and archived transcript restore/import do not.

## Solution Assessment

No current implementation is the complete solution. A `/restore`-only patch may address one archive recovery case, but the canonical issue still needs a broader product/API design for named snapshots, branching, archive ownership, and cache-sensitive fork behavior.
