## Summary

Keep open: current main has adjacent compaction checkpoint branch/restore and subagent transcript-fork primitives, but it still lacks the requested named session save/load workflow, reusable templates, and archive import/restore semantics; a recent maintainer comment also explicitly kept this open for design follow-up.

## What This Changes

The request is to add explicit session snapshot save/load, restore, and fork/template behavior for long-running OpenClaw sessions.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch behavior through the appropriate protocol, command, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but named save/load, reusable templates, and archive restore/import do not.

## Solution Assessment

No current implementation is the complete solution. Adding only `/session save` and `/session load` would be too narrow because the request also covers archive import, branching, reusable templates, retention policy, and cache-sensitive fork behavior.
