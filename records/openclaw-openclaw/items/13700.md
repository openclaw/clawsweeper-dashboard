## Summary

Keep open. Current main still has adjacent compaction checkpoint branch/restore and subagent transcript-fork primitives, but it does not provide the requested named session save/load workflow, reusable templates, or reset/deleted transcript archive import; a maintainer comment also explicitly kept this design open for follow-up.

## What This Changes

Request adding explicit session snapshot commands and APIs to save, load, restore, and fork conversation checkpoints.

## Best Possible Solution

Define a native session snapshot lifecycle that builds on SessionManager, session store metadata, compaction checkpoints, reset/deleted archives, and subagent fork support, then expose coherent save/list/load/branch/import semantics through commands, Gateway protocol, UI/CLI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request. The current-main gap is high-confidence from docs and source: /session only accepts idle/max-age, and available checkpoint/fork primitives are narrower than named session snapshots.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs maintainer choices on restore semantics, retention, archive import, permissions, and cache-preserving fork behavior before a narrow maintainable fix can be selected.
