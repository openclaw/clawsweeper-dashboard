## Summary

Keep open: current main has adjacent compaction-checkpoint branch/restore and native subagent transcript forking, but it still lacks the requested named `/session save` and `/session load` snapshot workflow, reusable templates, and `.reset`/`.deleted` archive import semantics. The active restore-command PR is only a partial implementation candidate and is still open.

## What This Changes

Request explicit session snapshot commands and restore/fork semantics for saving, loading, branching, and reusing long-running conversation context.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch/restore behavior through the appropriate protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap: adjacent compaction and subagent fork primitives exist, but named save/load snapshots, reusable templates, and archive restore/import are not implemented.

## Solution Assessment

No current implementation is the complete solution. A `/restore`-only patch may address one archive recovery case, but this needs a broader session snapshot design before the repository can safely standardize save/load/fork semantics.
