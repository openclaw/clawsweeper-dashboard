## Summary

Keep open: current main and v2026.6.10 still have adjacent compaction checkpoint branch/restore and session export primitives, but not the requested named `/session save` and `/session load` workflow, reusable session templates, or reset/deleted archive import.

## What This Changes

The issue requests named session save/load checkpoints plus restore, branch, and fork semantics for OpenClaw sessions.

## Best Possible Solution

Define one native session snapshot design that composes SessionManager transcript lineage, compaction checkpoints, reset/deleted archive discovery, Gateway protocol, slash commands, Control UI or CLI UX, docs, and focused tests.

Not applicable as a bug reproduction; this is a feature request. Source, docs, protocol, tests, latest-release inspection, and live related-item inspection give high-confidence evidence that the requested named save/load snapshot workflow is absent.

## Solution Assessment
