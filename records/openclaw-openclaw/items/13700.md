## Summary

Keep open: current main has adjacent compaction-checkpoint branch/restore and subagent transcript-fork primitives, but still lacks the requested named save/load snapshot workflow, reusable session templates, and reset/deleted archive import. The recent maintainer and ClawSweeper comments also explicitly kept the item open for product/API design follow-up.

## What This Changes

Add native session snapshots so users can save, load, restore, and branch long-running session context checkpoints.

## Best Possible Solution

Define a native session snapshot design over the existing session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch behavior through the appropriate protocol, command, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but named save/load and archive restore do not.

## Solution Assessment

No; adding only two slash commands is not the best complete solution because the request also covers archive import, branching, reusable templates, permissions/retention, and cache-sensitive fork behavior. The maintainable path is protocol and UX design first, then implementation over the existing primitives.
