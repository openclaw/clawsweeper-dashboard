## Summary

Keep open: current main and v2026.6.5 still lack the requested named `/session save` and `/session load` snapshot workflow, while the existing compaction checkpoint, ACP resume, archive history, and open `/resume` work cover narrower pieces that need maintainer product/API scoping.

## What This Changes

The issue requests explicit session snapshot commands to save, load, restore, and branch conversation context across long-running OpenClaw sessions.

## Best Possible Solution

Define one native session snapshot lifecycle over session storage, compaction checkpoints, reset/deleted archives, and subagent/ACP fork primitives, then expose agreed save/list/load/branch/restore behavior through slash commands, Gateway protocol, UI/CLI/SDK where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, protocol, release, and related-item checks give high confidence that current OpenClaw lacks the requested named save/load snapshot workflow while adjacent primitives exist.

## Solution Assessment

Unclear pending maintainer API direction. The requested slash commands are plausible, but the maintainable path is a unified session snapshot lifecycle contract rather than treating compaction restore, archive restore, ACP resume, or subagent transcript fork as the whole solution.
