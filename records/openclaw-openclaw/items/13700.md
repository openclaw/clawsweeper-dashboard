## Summary

Keep open: current main has adjacent compaction checkpoint branch/restore and native subagent fork support, but it still lacks named save/load snapshots, reusable templates, and reset/deleted archive import.

## What This Changes

The issue requests explicit session snapshot save/load, restore, and branching semantics for messages, context, model choice, and reusable session state.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore semantics through protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap: adjacent compaction and subagent fork primitives exist, but named save/load snapshots, reusable templates, and archive import are not implemented.

## Solution Assessment

No current implementation is the complete solution. A restore-only patch may cover one archive recovery case, but the maintainable path is a broader session snapshot design before standardizing save/load/fork semantics.
