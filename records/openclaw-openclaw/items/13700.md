## Summary

Keep open: current main still lacks the requested named save/load snapshot workflow, reusable session templates, and reset/deleted archive import semantics; existing compaction checkpoint and subagent fork primitives are adjacent but narrower.

## What This Changes

The issue requests explicit session snapshot commands for saving, loading, restoring, and forking conversation state checkpoints.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose the agreed save/list/load/branch/restore semantics through protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap: adjacent compaction and subagent fork primitives exist, but named save/load snapshots, reusable templates, and archive restore/import are not implemented.

## Solution Assessment

No current implementation is the complete solution. A /restore-only patch may address one archive recovery case, but this needs a broader session snapshot design before OpenClaw standardizes save/load/fork semantics.
