## Summary

Keep open. Current main still has adjacent compaction checkpoint branch/restore and subagent fork primitives, but it does not provide the requested named `/session save` and `/session load` snapshot workflow, reusable templates, or restore/import from `.reset` and `.deleted` transcript archives; the maintainer discussion also kept the design open.

## What This Changes

Request native session snapshot commands and APIs for saving, loading, restoring, and branching exact session context checkpoints.

## Best Possible Solution

Define one native session snapshot model that builds on SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then expose coherent save/list/load/branch/import semantics through chat commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from docs and source: `/session` only accepts `idle`/`max-age`, and the available checkpoint/fork primitives are narrower than named session snapshots.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs maintainer choices on restore semantics, retention, archive import, permissions, and cache-preserving fork behavior before a narrow maintainable fix can be selected.
