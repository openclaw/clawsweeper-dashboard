## Summary

Keep open: current main and v2026.6.5 only have compaction checkpoint branch/restore plus subagent fork primitives, not the requested named `/session save` and `/session load` snapshot workflow, reusable templates, or archive import from reset/deleted transcripts. ## What This Changes

The issue requests first-class session snapshot commands to save, load, restore, and branch exact conversation context checkpoints.

## Best Possible Solution

Define a native session snapshot design that composes with compaction checkpoints, reset/deleted archives, SessionManager parent chains, Gateway protocol, slash commands, Control UI, and subagent fork semantics before landing implementation pieces.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give high confidence that current main lacks the requested named snapshot commands and only has narrower compaction checkpoint recovery.

## Solution Assessment
