## Summary

Keep open: current main and the latest release still lack named `/session save` and `/session load` snapshots, reusable session templates, and reset/deleted archive import semantics; adjacent compaction checkpoint restore and subagent transcript forking do not cover the full requested workflow.

## What This Changes

The issue requests explicit named session snapshots with save/load, restore, and branching semantics for long-running OpenClaw sessions.

## Best Possible Solution

Define a native session snapshot contract over session storage, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore semantics through protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release-tag inspection give high confidence that named save/load commands and arbitrary snapshot APIs are absent while adjacent fork and compaction-checkpoint primitives exist.

## Solution Assessment

No current implementation solves the full issue. The maintainable path is a maintainer-approved snapshot lifecycle and API contract rather than treating compaction checkpoint restore or the active partial PRs as the full save/load/fork design.
