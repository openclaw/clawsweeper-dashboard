## Summary

Keep open: current main still lacks named session snapshot save/load, reusable snapshot templates, and archive restore/import semantics; existing code only provides adjacent compaction checkpoint branch/restore and subagent transcript-fork primitives, and the discussion points to unresolved product/API design.

## What This Changes

Request explicit session snapshots that can save, load, restore, and fork conversation state across long-running sessions.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch behavior through the appropriate protocol, command, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but named save/load, reusable templates, and archive restore/import do not.

## Solution Assessment

No current implementation is the complete solution. Adding only /session save and /session load would be too narrow because the discussion also covers archive import, branching, reusable templates, retention policy, and cache-sensitive fork behavior.
