## Summary

Keep open: current main still lacks the requested named session snapshot save/load workflow, reusable snapshot templates, and reset/deleted archive restore semantics. The existing compaction checkpoint branch/restore and subagent fork support are useful adjacent primitives, and the active restore-command PR is only a partial implementation with unresolved review concerns.

## What This Changes

Add native session snapshot commands and APIs for saving, loading, restoring, and branching conversation context checkpoints.

## Best Possible Solution

Define a native session snapshot contract over session store metadata, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose coherent save/list/load/branch/restore behavior through protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence gap evidence because current main has only settings-oriented `/session` actions and compaction-checkpoint restore, not named snapshots or archive import.

## Solution Assessment

No current implementation is the complete solution. A `/restore`-only patch can cover one recovery case, but the maintainable fix needs a product/API design for named checkpoints, branching, archive ownership, and cache-sensitive forking.
