## Summary

Keep open: current main and the latest release still lack the requested named `/session save` and `/session load` snapshot workflow, while existing compaction checkpoint restore, ACP resume, and the open archive-restore PR cover only narrower slices.

## What This Changes

Request to add explicit session checkpoint save/load, restore, and branching semantics for long-running OpenClaw sessions.

## Best Possible Solution

Define one native session snapshot lifecycle over session storage, compaction checkpoints, reset/deleted archives, and subagent/ACP fork primitives, then expose agreed save/list/load/branch/restore behavior through slash commands, Gateway protocol, SDK/UI/CLI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, protocol, latest-release, and related-item checks give high confidence that current OpenClaw lacks the requested named save/load snapshot workflow while adjacent primitives exist.

## Solution Assessment

Unclear pending maintainer API direction. The requested slash commands are plausible, but the maintainable path is a unified session snapshot lifecycle contract rather than treating compaction restore, archive restore, ACP resume, or subagent transcript fork as the whole solution.
