The issue requests named session save/load checkpoints plus branch/fork-style reuse and restore of saved or archived session context.

## Best Possible Solution

Define one native session snapshot contract across session storage, transcript DAGs, Gateway protocol, slash commands, archive handling, fork behavior, UI/docs, and tests before implementing save/list/load/branch/import semantics.
