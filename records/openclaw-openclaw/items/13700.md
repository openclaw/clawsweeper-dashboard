## Summary

Keep open: current main has adjacent compaction checkpoint branch/restore and `sessions_spawn` transcript-fork support, but it still lacks the requested named `/session save` and `/session load` workflow, reusable snapshot templates, and restore/import from `.reset` or `.deleted` transcript archives.

## What This Changes

Request adds explicit session snapshot save/load/branching so users can checkpoint, restore, fork, and reuse long-running conversation context.

## Best Possible Solution

Define a native session snapshot design that builds on session store metadata, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch semantics across Gateway protocol, slash commands, UI/CLI/docs, and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but named save/load and archive restore do not.

## Solution Assessment

No; adding only two slash commands would not be the best fix because the request also covers archive import, branching, reusable templates, retention, permissions, and cache-sensitive fork behavior. The maintainable path is protocol/UX design first, then implementation over the existing primitives.
