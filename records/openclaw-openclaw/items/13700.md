## Summary

Keep open: current main still has only adjacent compaction-checkpoint branch/restore and subagent transcript-fork primitives, not the requested named save/load snapshot workflow, reusable templates, or reset/deleted archive import; recent maintainer-facing comments also kept this open for design follow-up.

## What This Changes

Add first-class session snapshot save, load/restore, and fork behavior for preserving and reusing conversation context.

## Best Possible Solution

Define a native session snapshot design over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch behavior through the appropriate protocol, command, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but named save/load and archive restore do not.

## Solution Assessment

No current implementation is the complete solution. Adding only two slash commands would be too narrow because the request also covers archive import, branching, reusable templates, permissions/retention, and cache-sensitive fork behavior.
