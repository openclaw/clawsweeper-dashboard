## Summary

Keep open: current main has adjacent compaction checkpoint controls and native subagent transcript forking, but it still lacks the requested named `/session save` and `/session load` workflow, reusable session templates, and restore/import from `.reset` or `.deleted` archives.

## What This Changes

Request adding named session snapshot save/load commands plus restore and branching semantics for long-running sessions.

## Best Possible Solution

Define a native session snapshot design built on the session store, compaction checkpoint artifacts, reset/deleted archives, and subagent fork primitives, then add save/list/load/branch APIs, user surfaces, docs, and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence current-main gap: adjacent compaction and fork primitives exist, but the requested named save/load and archive restore workflow does not.

## Solution Assessment

No; the current implementation is not yet the requested solution. The narrowest maintainable path is a maintainer-approved snapshot API/UX design that reuses existing checkpoint, archive, and fork primitives instead of trying to bolt `/session save/load` directly onto one surface.
