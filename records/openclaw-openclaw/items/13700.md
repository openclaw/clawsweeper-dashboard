## Summary

Keep open: current main still lacks named `/session save` and `/session load` snapshots, reusable session templates, and reset/deleted archive import semantics. The code now has adjacent compaction checkpoint restore and subagent transcript forking, plus active partial restore/resume PRs, but the remaining feature needs maintainer product/API direction.

## What This Changes

Request to add named session checkpoint save/load/restore/fork commands and APIs for long-running conversation context snapshots.

## Best Possible Solution

Define a native session snapshot contract over session storage, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore semantics through protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that named save/load commands and arbitrary snapshot APIs are absent while adjacent fork and compaction-checkpoint primitives exist.

## Solution Assessment

No complete implementation exists yet. The maintainable path is a maintainer-approved snapshot lifecycle and API contract rather than treating compaction checkpoint restore or the active partial PRs as the full save/load/fork design.
