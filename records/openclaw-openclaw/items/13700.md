## Summary

Keep open: current main and the latest shipped release still lack the requested named `/session save` and `/session load` snapshot workflow, reusable templates, and archive-import contract. The repo has useful adjacent compaction checkpoint branch/restore and native subagent transcript-fork primitives, plus active partial restore/resume work, so this remains the canonical product/API tracker rather than a cleanup close.

## What This Changes

The issue requests explicit OpenClaw session snapshots that can save, load, restore, and fork conversation context checkpoints.

## Best Possible Solution

Define one native session snapshot contract over session storage, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore semantics through commands, Gateway protocol, UI/CLI, docs, and focused tests.

## Solution Assessment

Unclear pending maintainer API direction. The requested slash commands are plausible, but the maintainable path is a unified session snapshot lifecycle contract rather than treating compaction checkpoint restore, archive restore, or subagent transcript fork as the whole solution.
