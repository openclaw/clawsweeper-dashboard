## Summary

Keep open: current main has adjacent compaction checkpoint restore and native subagent transcript forking, plus open partial restore/resume PRs, but it still does not implement named `/session save` and `/session load` snapshots, reusable session templates, or an approved reset/deleted archive import contract.

## What This Changes

Request explicit session snapshot save/load commands to checkpoint, restore, and branch/fork long-running session context.

## Best Possible Solution

Define a native session snapshot contract over the session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore semantics through protocol, commands, UI/CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that named save/load commands and an arbitrary snapshot API are absent, while adjacent fork and compaction-checkpoint primitives exist.

## Solution Assessment

No complete implementation exists yet. The current primitives and open restore/resume PRs are useful partials, but the maintainable solution needs a product-approved snapshot lifecycle and API contract before standardizing save/load/fork semantics.
