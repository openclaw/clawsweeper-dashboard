## Summary

Keep open: current main and v2026.6.5 have adjacent compaction checkpoint branch/restore and session-fork primitives, but they do not provide the requested named `/session save` / `/session load` snapshot workflow, reusable templates, or archive import. The open `/resume` PR is only a partial implementation path and is not merged.

## What This Changes

The issue requests explicit session snapshot commands to save named checkpoints, load/restore them, and branch conversations from saved context.

## Best Possible Solution

Define one native session snapshot contract that builds on SessionManager trees, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch through chosen chat, Gateway, UI, and docs surfaces with focused tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection against current main and v2026.6.5 confirms the named snapshot commands are absent while only narrower compaction checkpoint branch/restore exists.

## Solution Assessment

No, the current adjacent checkpoint restore is not enough to solve the issue. The best path is a product/API design that unifies named snapshots, archive restore, and branching instead of adding one isolated command path.
