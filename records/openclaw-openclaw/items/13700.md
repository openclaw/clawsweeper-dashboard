Keep open: current main and the latest release still lack the requested named session save/load snapshot workflow; compaction checkpoint restore, subagent transcript forking, plugin session-store helpers, and active restore/resume PRs are adjacent but partial.

## What This Changes

The issue requests first-class session checkpoints with `/session save` and `/session load`, plus restore, fork, and reusable snapshot semantics for long-running sessions.

## Best Possible Solution

Define one native session snapshot lifecycle over session storage, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose agreed save/list/load/branch/restore behavior through commands, Gateway protocol, UI/CLI where appropriate, docs, and focused tests.
