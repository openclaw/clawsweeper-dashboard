Keep open: current main has adjacent compaction checkpoint branch/restore and native subagent transcript-fork support, but it still lacks the requested named save/load snapshot workflow, reusable templates, and reset/deleted archive import.

## What This Changes

## Best Possible Solution

Define a native session snapshot design over the existing session store, compaction checkpoints, reset/deleted archives, and subagent fork primitives, then expose save/list/load/branch behavior through the appropriate protocol, command, UI/CLI, docs, and tests.
