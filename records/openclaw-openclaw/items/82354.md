## Summary

Keep open: the PR fixes the queued MS Teams reply path, but it still misses the native personal-DM stream finalization path, so some visible Teams replies would continue bypassing `message:sent` hooks.

## What This Changes

Adds MS Teams outbound sent-hook plumbing, calls it from queued reply flushing, and tests queued personal, group, and failure deliveries.

## Best Possible Solution

Keep the scoped hook module, but emit the same sent-hook event for every visible MS Teams reply delivery path, including native stream finalization, with focused regression coverage.

## Reproduction Assessment

Yes. Current main source has a documented sent-hook contract and Telegram emits it, while MS Teams reply delivery lacks sent-hook calls; PR head still has a source-reproducible path where native stream finalization returns before the new queued-flush hook can run.

## Solution Assessment

No, not as submitted. The hook module and queued-delivery wiring are the right direction, but the maintainable fix must emit from native stream finalization or return enough finalized receipt/content data for the dispatcher to emit once.
