## Summary

Keep open for the PR path: the branch addresses a valid P1 cron fallback bug, but the fallback-success warning should stop reusing skip-only wording before merge.

## What This Changes

This PR updates isolated cron model preflight to try resolved fallback candidates before skipping, with fallback-policy tests, cron docs, and a changelog entry.

## Best Possible Solution

Land the fallback preflight behavior after adjusting the success-path warning to distinguish continuing on a fallback from exhausting all candidates.

## Reproduction Assessment

Yes. Current main source shows unavailable local-provider preflight returns skipped before runWithModelFallback, and the linked issue plus PR proof give an unreachable-loopback path for the failure mode.

## Solution Assessment

Mostly yes. The fallback selection belongs at this preflight boundary and reuses existing policy, but the continuation warning should not emit skip-only text when a fallback is selected.
