## Summary

Keep open: this PR targets a real P1 cron fallback bug and has sufficient terminal proof, but the fallback-success warning still reuses skip-only text from the primary preflight failure.

## What This Changes

This PR changes isolated cron preflight to walk effective model fallback candidates before returning skipped, with focused tests, cron docs, and a changelog entry.

## Best Possible Solution

Land the fallback preflight behavior after replacing the fallback-success warning with continuation-specific text that does not say the run was skipped.

## Reproduction Assessment

Yes. Current main source returns skipped during primary local-provider preflight before runWithModelFallback, and https://github.com/openclaw/openclaw/issues/79329 plus the PR proof give an unreachable-loopback path for the failure mode.

## Solution Assessment

Mostly yes. Applying the existing cron fallback policy at the preflight boundary is the narrow maintainable fix, but the success-path warning should stop reusing the skip-only preflight reason.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.88

Full review comments:

- **[P3] Avoid logging skip text when fallback continues:** `src/cron/isolated-agent/run.ts:667-669`
