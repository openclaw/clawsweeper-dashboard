## Summary

Keep open for the active fix PR. The branch addresses a real current-main cron fallback bug and has sufficient terminal proof, but it still introduces one misleading success-path warning that should be cleaned up before maintainer landing.

## What This Changes

The PR changes isolated cron model preflight to try configured model fallback candidates before returning a skipped run when the primary local provider is unavailable.

## Best Possible Solution

Land the fallback-preflight behavior after adjusting the success-path warning to distinguish continuing on a fallback from exhausting all candidates.

## Reproduction Assessment

Yes. Current main source shows an unavailable local primary returns skipped from prepareCronRunContext before runCronAgentExecution can build the normal model fallback runner.

## Solution Assessment

Mostly yes. The fallback selection belongs at the preflight boundary and reuses existing cron fallback policy, but the continuation warning should not emit skip-only text when a fallback is selected.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.88

Full review comments:

- **[P3] Avoid logging skip text when fallback continues:** `src/cron/isolated-agent/run.ts:667-669`
  - body: In the fallback-success path this logs `preflight.reason`, but that reason is built for the skip path and can say the cron run is being skipped. Operators would see a skip warning for a run that actually continues on a fallback, so build a continuation-specific warning from the preflight fields instead of reusing the skip-only reason.
  - confidence: 0.88
