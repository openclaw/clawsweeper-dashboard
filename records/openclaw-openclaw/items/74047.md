## Summary

Keep open. Current main has useful split-install diagnostics and safer doctor/update behavior, but it does not settle the supported user/system install migration or prove the reported newer-runtime `/v1/chat/completions` timeouts are fixed; #70306 remains open for a related legacy unit resolver gap.

## What This Changes

The report asks OpenClaw to make user/system Gateway upgrades understandable and reliable when the live systemd service is pinned to an older install path.

Keep this as the canonical tracker until status/doctor/update define the split-install policy, legacy unit resolution is handled, and a managed Linux Gateway upgrade is live-validated with `openai-codex/gpt-5.5` through `/v1/chat/completions`.

## Solution Assessment
