## Summary

Keep open. Current main has useful split-install diagnostics, doctor warnings, update restart/version checks, and docs, but it still does not fully answer the supported user/system install migration or explain the reported newer-runtime `/v1/chat/completions` timeouts.

## What This Changes

The issue asks OpenClaw to clarify or fix split user/system installs where the Gateway service remains pinned to an older install path and newer Gateway runtime promotion causes `/v1/chat/completions` timeouts.

## Best Possible Solution

Keep this as the canonical tracker until status/doctor/update reconcile the active service root with the intended install root, #70306 is reviewed or superseded, and a managed Linux Gateway upgrade is live-validated with `openai-codex/gpt-5.5` through `/v1/chat/completions`.

## Reproduction Assessment

No. The report has credible host-state and rollback evidence, but current main has no high-confidence fixture or live run that stages an older system service install, repoints to a newer user install, and reproduces the chat-completions timeout.

## Solution Assessment

Unclear. The current diagnostics, docs, and update restart/version checks are useful partial mitigations, but the maintainable fix still needs an explicit split-install policy plus live Gateway/provider proof.
