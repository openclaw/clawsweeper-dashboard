## Summary

Keep open. Current main has useful service-layout diagnostics, update restart hardening, and split-brain troubleshooting, but it still does not prove a supported split user/system install migration or explain the reported newer-install `/v1/chat/completions` timeout; related PR #70306 remains only a partial legacy-unit fallback proposal.

## What This Changes

Clarify and harden Linux Gateway upgrades where user and system installs diverge, including service path detection/migration and reliable `/v1/chat/completions` after promoting newer `openai-codex/gpt-5.5`.

## Best Possible Solution

Keep this as the canonical tracker until OpenClaw documents and validates split user/system install handling, reconciles CLI/status/doctor/update behavior for the live Gateway service root, and proves a managed Linux Gateway can migrate to the intended newer install while serving `openai-codex/gpt-5.5` through `/v1/chat/completions`.

No. The report has credible rollback evidence, but current main does not include a fixture or live package/service run that repoints a legacy Linux Gateway service from an older system install to a newer user install and reproduces the timeout.

## Solution Assessment
