Keep open. A fresh stable-release reproduction confirms the split user/system install can still leave the Gateway running from an old path, while current main only has partial diagnostics and restart/version verification, not a settled migration or pre-update failure policy.

The issue asks OpenClaw to make split user/system install upgrades safe and understandable when the Gateway service remains pinned to an older install path.
