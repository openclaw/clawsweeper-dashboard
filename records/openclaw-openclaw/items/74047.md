## Summary

Keep open. Current main has better split-install diagnostics, entrypoint drift detection, and active-systemd rewrite protection, but it still does not prove a safe migration from the reported older system install to the newer user install or explain the `/v1/chat/completions` timeout after repointing.

## What This Changes

The issue asks OpenClaw to clarify and harden split user/system Gateway upgrades where the CLI and live systemd service resolve different installs and newer Gateway paths can make `/v1/chat/completions` time out.

## Best Possible Solution

Keep this as the canonical tracker until OpenClaw defines the supported split user/system install policy and validates a managed Linux Gateway upgrade/repoint path that serves `openai-codex/gpt-5.5` reliably through `/v1/chat/completions`.

## Reproduction Assessment

No. The issue has credible rollback evidence, but the repo currently lacks a high-confidence fixture or live run that installs an older system service, adds a newer user install, repoints the service, and reproduces the chat-completions timeout.

## Solution Assessment

Unclear. The current diagnostics and docs are useful, but the maintainable fix likely needs maintainer policy plus Package Acceptance/Testbox proof for the actual systemd service migration and model path.
