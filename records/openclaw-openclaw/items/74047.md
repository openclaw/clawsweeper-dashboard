## Summary

Keep open. Current main has split-install diagnostics, future-version mutation guards, and doctor warnings, but it still does not prove a supported user/system Gateway migration or explain the reported newer-runtime `/v1/chat/completions` timeouts; the same reporter also has open PR #70306 paired with one remaining legacy-unit resolver gap.

## What This Changes

The issue asks OpenClaw to clarify or fix split user/system installs so Gateway service upgrades use the intended install path and remain reliable with newer model/runtime paths.

## Best Possible Solution

Keep this as the canonical tracker until Gateway status/doctor/update define the split-install policy, the legacy-unit resolver gap is reviewed or superseded, and a managed Linux Gateway upgrade is live-validated through `/v1/chat/completions` with the intended model path.

## Reproduction Assessment

No. The issue has credible host-state and rollback evidence, but current main has no high-confidence fixture or live run that stages an older system service install, upgrades a newer user install, repoints the service, and reproduces the chat-completions timeout.

## Solution Assessment

Unclear. The current diagnostics and docs are useful partial mitigations, while the best maintainable fix needs explicit split-install policy plus live Gateway/provider proof; #70306 addresses only one resolver gap.
