## Summary

Keep open. Current main has useful split-install diagnostics, docs, and active-systemd rewrite protection, but it still does not prove a supported user/system install migration or explain the reported newer-runtime `/v1/chat/completions` timeouts; open PR #70306 remains paired with one legacy unit resolver gap.

## What This Changes

Clarify and fix split user/system OpenClaw installs where the live Gateway service can remain pinned to an older runtime while newer local CLI/model paths appear to work.

## Best Possible Solution

Keep this as the canonical tracker until status/doctor/update reconcile the active service root with the intended install root, #70306 is reviewed or superseded, and a managed Linux Gateway upgrade is live-validated with `openai-codex/gpt-5.5` through `/v1/chat/completions`.

## Reproduction Assessment

No. The report has credible host-state and rollback evidence, but current main has no high-confidence fixture or live run that stages an older service install, repoints it to a newer user install, and reproduces the chat-completions timeout.

## Solution Assessment

Unclear. The current diagnostics and docs are useful partial mitigations, and #70306 addresses one resolver gap, but the maintainable fix needs an explicit split-install policy plus live Gateway/provider proof.
