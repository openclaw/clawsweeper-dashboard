## Summary

Keep open. Current main added useful adjacent diagnostics and restart hardening, but it still lacks a proven split user/system install migration path for a legacy Linux Gateway service and does not explain or reproduce the newer-install `/v1/chat/completions` timeout.

## What This Changes

Clarify and fix Gateway upgrades where the CLI and live systemd service can use different OpenClaw installs, especially when repointing the service makes OpenAI-compatible chat completions unreliable.

## Best Possible Solution

Keep this as the canonical tracker until OpenClaw documents and validates split user/system install handling, reconciles CLI and live service install roots in status/doctor/update, and proves a managed Linux Gateway can migrate to the intended newer install while serving `openai-codex/gpt-5.5` through `/v1/chat/completions`.

## Reproduction Assessment

No. The report has credible rollback evidence, but current main lacks a fixture or live package/service run that repoints a legacy Linux Gateway service from an older system install to a newer user install and reproduces the timeout.

## Solution Assessment

Unclear. The current diagnostics are useful, and #70306 may fix one legacy-unit resolver gap, but the best product fix depends on the chosen split-install policy and live Gateway/provider proof.
