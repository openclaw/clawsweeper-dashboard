## Summary

Keep open. Current main has better Gateway service diagnostics, update restart hardening, and split-brain docs, but it still does not prove the reported split user/system install migration or explain the newer-install `/v1/chat/completions` timeout.

## What This Changes

The issue asks OpenClaw to clarify and harden upgrades where the CLI install path and live Gateway service install path diverge, especially when promoting `openai-codex/gpt-5.5` through `/v1/chat/completions`.

## Best Possible Solution

Keep this as the canonical tracker until OpenClaw documents and validates split user/system install handling, reconciles CLI and live service install roots in status/doctor/update, and proves a managed Linux Gateway can migrate to the intended newer install while serving GPT-5.5 through `/v1/chat/completions`.

## Reproduction Assessment

No. The issue has credible rollback evidence, but current main lacks a package/service fixture or live run that repoints a legacy system install to a newer user install and reproduces the `/v1/chat/completions` timeout.

## Solution Assessment

Unclear. The current diagnostics and docs are useful, and #70306 may fix a legacy unit resolver gap, but the maintainable fix depends on the chosen split-install policy plus live Gateway/provider proof.
