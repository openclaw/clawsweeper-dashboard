## Summary

Keep open. Current main has useful split-install diagnostics, duplicate-install installer warnings, docs recovery guidance, and a partial `/v1/chat/completions` streaming timeout mitigation, but it still does not define or prove the full user/system install migration path, and the legacy systemd unit resolver gap remains represented by https://github.com/openclaw/openclaw/pull/70306.

## What This Changes

The request asks for a clear and safe upgrade path when a user install and system Gateway service install diverge, including reliable Gateway promotion to newer GPT-5.5-era runtimes.

## Best Possible Solution

Keep this as the canonical tracker until maintainers define the split-install upgrade policy, land or reject legacy unit fallback handling, and live-validate a managed Linux Gateway upgrade through `/v1/chat/completions` on the intended GPT-5.5 route.
