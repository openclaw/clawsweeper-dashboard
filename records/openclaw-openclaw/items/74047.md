## Summary

Keep open. Current main has useful adjacent gateway service-layout diagnostics, doctor drift warnings, update restart guidance, and OpenAI-compatible endpoint hardening, but it still does not define or prove a safe split user/system Linux install migration or explain the reported newer-install `/v1/chat/completions` timeout.

## What This Changes

The issue asks OpenClaw to make split user/system Linux Gateway upgrades understandable and safe, including warnings or migration guidance when the live service points at an older install and newer runtimes time out on `/v1/chat/completions`.

## Best Possible Solution

Keep this as the canonical tracker until OpenClaw has a documented split user/system install policy, status or doctor output that reconciles CLI and live service install roots and versions, operator-safe handling for system-scoped and legacy units, and package-acceptance or live proof that a managed Linux Gateway can migrate to the intended newer install and serve `/v1/chat/completions` with `openai-codex/gpt-5.5`.

## Reproduction Assessment

No. The report has credible host-state and rollback evidence, but current main does not include a high-confidence fixture or live run that reproduces the split user/system service repoint plus newer-install `/v1/chat/completions` timeout.

## Solution Assessment

Unclear. The current diagnostics and update restart improvements are the right nearby surfaces, but the best fix depends on the timeout root cause and maintainer policy for system-scoped service migration.
