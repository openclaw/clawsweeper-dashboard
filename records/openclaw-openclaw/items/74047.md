## Summary

Keep open. Current main improves Gateway service-layout visibility and safer repair behavior, but it still lacks a proven split user/system install migration or an explanation/fix for the newer-install `/v1/chat/completions` timeouts; related #70306 is only a partial legacy unit resolver fix.

## What This Changes

This issue asks OpenClaw to clarify or fix split user/system install upgrades where the live Gateway service stays pinned to an older path or becomes unreliable after being repointed to a newer runtime.

## Best Possible Solution

Keep this as the canonical tracker until status/doctor/update/docs define the split user/system install policy and Package Acceptance or Testbox proves a managed Linux Gateway can migrate to the intended install while serving `openai-codex/gpt-5.5` through `/v1/chat/completions`.
