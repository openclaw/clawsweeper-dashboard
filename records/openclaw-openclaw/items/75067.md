## Summary

Keep this PR open. The pricing-resilience goal is still useful and current main has not implemented the timeout/cache/breaker/readiness changes, but the PR head is not merge-ready because the disk-cache and circuit-breaker paths can preserve stale pricing longer than intended and bypass current configured pricing.

## What This Changes

The PR lowers Gateway model-pricing fetch timeout, adds disk-backed pricing cache warming and persistence, adds a pricing fetch circuit breaker, adds a readiness startupBlockedReason field, and updates related Gateway tests.

## Best Possible Solution

Keep the Gateway pricing resilience design, but merge only after disk warming overlays current configured and manifest pricing, retry timers use the actual cache age or breaker cooldown, the changelog is updated, and integration with #73486 keeps pricing fetches off the startup-critical path.

## Reproduction Assessment

Yes. The review findings are reproducible from PR-head code inspection: a fresh disk cache returns before configured pricing is seeded, and both warm-cache and open-circuit branches call the fixed 24h scheduler. Focused fake-timer tests around fresh cache age, changed local costs, and breaker cooldown would exercise the failures.

## Solution Assessment

No. The proposed direction is reasonable, but this implementation is not the best fix until it preserves current pricing precedence and schedules refreshes from remaining TTL or circuit cooldown instead of the generic daily refresh helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Overlay configured pricing before returning from disk warm:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: This branch returns as soon as any fresh disk cache exists, before current `models.providers.*.models[].cost` values are collected. If an operator changes local pricing while the disk cache is fresh, stale persisted catalog data can win for the cache window; merge or overlay configured and manifest pricing before returning.
  - confidence: 0.91
- **[P2] Schedule breaker retry at cooldown expiry:** `src/gateway/model-pricing-cache.ts:1232`
  - body: The open-circuit path logs the remaining cooldown but calls `scheduleRefresh`, which waits the 24h cache TTL. After three failures, a recovered pricing service can be ignored until the daily timer instead of being retried after the 10-minute breaker cooldown.
  - confidence: 0.9
- **[P3] Schedule warm-cache refresh for remaining TTL:** `src/gateway/model-pricing-cache.ts:1184`
  - body: A fresh disk cache always schedules a full 24h delay from process start. A 23-hour-old cache can therefore remain in use for almost another day, and repeated restarts can keep postponing live refreshes; schedule from the loaded cache's remaining TTL instead.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/gateway/model-pricing-cache.ts:82`
  - body: This is a user-facing Gateway resilience feature, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active version with appropriate non-bot credit so release notes capture the timeout, disk cache, circuit breaker, and readiness behavior.
  - confidence: 0.88
