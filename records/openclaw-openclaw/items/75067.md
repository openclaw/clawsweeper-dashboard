## Summary

Keep open. The PR still addresses missing Gateway pricing resilience on current main, but the branch is not correct as submitted because fresh disk-cache warmup can bypass configured pricing, breaker retries can be delayed for the daily TTL, and the stale branch must preserve current-main startup semantics.

## What This Changes

The PR reduces Gateway model-pricing fetch timeout to 10s, adds disk-backed pricing cache warm/persist behavior and a failure circuit breaker, adds a readiness `startupBlockedReason`, and updates Gateway tests.

## Best Possible Solution

Keep the pricing resilience direction, but rework it so disk data is only a bootstrap source under current configured/manifest pricing, refresh timers honor remaining TTL and breaker cooldown, current-main readiness/abort behavior is preserved, and the user-facing change is in the changelog.

## Reproduction Assessment

Yes. The original slow-start path is reproducible by stalling OpenRouter/LiteLLM pricing fetches, and the merge blockers are reproducible from code inspection plus focused fake-timer tests for fresh disk cache with changed local costs and breaker cooldown retry behavior.

## Solution Assessment

No. The design direction is maintainable, but the submitted patch is not the best fix until it preserves local pricing precedence, schedules from actual TTL/cooldown, and rebases over current Gateway startup behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Overlay configured pricing before returning from disk warm:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: A fresh disk cache returns before current `models.providers.*.models[].cost` and manifest pricing are collected. That lets stale disk pricing override changed local costs for the cache window; merge current seeded pricing over disk data before returning.
  - confidence: 0.91
- **[P2] Retry when the circuit cooldown expires:** `src/gateway/model-pricing-cache.ts:1232`
  - body: When the breaker is open, this branch calls `scheduleRefresh`, but that helper waits the 24h cache TTL. A recovered upstream can be skipped for roughly a day instead of the documented 10-minute cooldown.
  - confidence: 0.9
- **[P3] Schedule disk refresh for the remaining TTL:** `src/gateway/model-pricing-cache.ts:1184`
  - body: A fresh but nearly expired disk cache always schedules the daily timer from process start. A 23-hour-old file can stay in use for almost another day, and restarts can keep postponing refresh; schedule from `CACHE_TTL_MS - age`.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/gateway/model-pricing-cache.ts:82`
  - body: This changes user-visible Gateway startup and pricing behavior, but the PR file list does not include `CHANGELOG.md`. Add a single-line active changelog item covering the timeout, disk cache, breaker, and readiness behavior.
  - confidence: 0.88
