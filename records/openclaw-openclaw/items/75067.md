## Summary

Keep open. Current main has the prerequisite pricing-startup deferral, but it does not implement this PR's timeout reduction, disk cache, circuit breaker, or additive readiness field; the PR direction is useful, but the current patch has repairable P2 correctness blockers around pricing precedence and retry scheduling.

## What This Changes

The PR lowers Gateway pricing fetch timeout, adds disk-backed pricing cache warm/persist behavior, adds a pricing fetch circuit breaker, exposes a startupBlockedReason field on readiness failures, and updates Gateway tests.

## Best Possible Solution

Keep the pricing resilience work, but revise it so disk cache data is only a bootstrap source under current configured and manifest pricing, refresh timers honor remaining cache TTL and breaker cooldown, readiness changes preserve current main behavior, and the user-facing change is documented in the changelog.

## Reproduction Assessment

Yes. The blockers are reproducible from PR-head code inspection, and focused fake-timer/unit tests can cover fresh disk cache with changed configured costs plus breaker-cooldown retry behavior without live network access.

## Solution Assessment

No. The design direction is maintainable, but this implementation is not the best fix until it preserves local pricing precedence and schedules retries from the actual remaining TTL or circuit-breaker cooldown instead of the generic daily refresh helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Apply configured pricing before disk warm return:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: A fresh disk cache returns before collectSeededPricing reads current models.providers.*.models[].cost. If a user changes local pricing while the disk cache is fresh, stale disk data can win for the cache window; merge or overlay current configured and manifest pricing before returning.
  - confidence: 0.91
- **[P2] Retry when the circuit cooldown expires:** `src/gateway/model-pricing-cache.ts:1232`
  - body: The open-circuit path logs the remaining cooldown but schedules through scheduleRefresh, which waits the 24h cache TTL. After three failures, a recovered pricing service may be ignored until the daily timer instead of being retried after the 10-minute cooldown.
  - confidence: 0.9
- **[P3] Schedule disk refresh for remaining TTL:** `src/gateway/model-pricing-cache.ts:1184`
  - body: A fresh disk cache always schedules a full 24h delay from process start. A 23-hour-old cache can remain in use for almost another day, and repeated restarts can keep postponing the live refresh; schedule from the loaded cache's remaining TTL instead.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/gateway/model-pricing-cache.ts:87`
  - body: This is a user-facing Gateway resilience change, but the PR head does not update CHANGELOG.md. Add a single-line active changelog entry covering the timeout, disk cache, breaker, and readiness behavior.
  - confidence: 0.88
