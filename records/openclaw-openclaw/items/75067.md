## Summary

Keep open. Current main has the prerequisite post-ready pricing deferral, but it still lacks this PR's timeout/cache/breaker work, and the PR head has correctness issues around pricing precedence, retry timing, and current readiness behavior before it can merge.

## What This Changes

The PR lowers Gateway pricing fetch timeout to 10s, adds disk-backed pricing cache and circuit breaker behavior, adds a readiness startupBlockedReason field, and updates Gateway tests.

## Best Possible Solution

Keep the pricing resilience direction, but revise it so disk cache data is a bootstrap input under current configured/manifest pricing precedence, refresh timers use the real remaining cache TTL and breaker cooldown, readiness preserves current reason reporting, and the user-facing behavior is covered by tests and changelog.

## Reproduction Assessment

Yes. The blockers are reproducible from PR-head diff plus current-main code inspection, and fake-timer/unit tests can cover stale disk cache overriding configured costs and circuit cooldown retry behavior without live network access.

## Solution Assessment

No. The design direction is maintainable, but this implementation is not the best fix until disk warmup preserves local pricing precedence and retry scheduling follows the actual cache age and circuit-breaker cooldown.

Full review comments:

- **[P2] Apply configured pricing before disk warm return:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: A fresh disk cache returns before current models.providers.*.models[].cost and manifest pricing are collected. If a user changes local pricing while the disk cache is fresh, stale disk data can win for the cache window; overlay current configured and manifest pricing before returning.
  - confidence: 0.91
- **[P2] Retry when the circuit cooldown expires:** `src/gateway/model-pricing-cache.ts:1232`
  - body: The open-circuit path logs a 10-minute cooldown but schedules through scheduleRefresh, which waits the 24h cache TTL. After three failures, a recovered pricing service may be ignored until the daily timer instead of being retried after cooldown.
  - confidence: 0.9
- **[P2] Preserve startup pending reason on readiness 503s:** `src/gateway/server/readiness.ts:54-56`
  - body: Current main already reports deps.getStartupPendingReason() in failing. When adding startupBlockedReason, keep that dynamic reason instead of hard-coding only sidecars-pending, or current startup diagnostics lose the more specific blocker.
  - confidence: 0.82
- **[P3] Schedule disk refresh for remaining TTL:** `src/gateway/model-pricing-cache.ts:1184`
  - body: A fresh disk cache always schedules a full 24h delay from process start. A 23-hour-old cache can remain in use for almost another day, and repeated restarts can keep postponing live refresh; schedule from the loaded cache's remaining TTL instead.
  - confidence: 0.84
