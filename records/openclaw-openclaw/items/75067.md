## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Overlay current pricing before returning from disk warm:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: This returns as soon as any fresh disk cache exists, before resolving current config or manifest pricing. After a user changes `models.providers.*.models[].cost` or a plugin updates manifest pricing, the old on-disk entries can shadow the current source of truth for up to the cache TTL; collect seeded pricing first and overlay it before skipping the live fetch.
  - confidence: 0.9
- **[P2] Retry when the circuit cooldown ends:** `src/gateway/model-pricing-cache.ts:1232-1233`
  - body: The circuit-open path reports a 10-minute cooldown but schedules the next automatic refresh with the generic 24-hour TTL. That means a transient upstream outage can keep pricing stale until the day-long timer fires or the gateway restarts; schedule the retry for the remaining cooldown instead.
  - confidence: 0.88
- **[P3] Schedule disk-cache refresh for the remaining TTL:** `src/gateway/model-pricing-cache.ts:1184-1185`
  - body: When a nearly expired disk cache warms memory, this starts a fresh 24-hour timer from now, so live refresh can be delayed almost 48 hours from the cache timestamp. Use the remaining cache TTL, bounded to a sensible minimum, when the warm path returns.
  - confidence: 0.8
