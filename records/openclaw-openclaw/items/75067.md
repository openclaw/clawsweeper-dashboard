## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Overlay current pricing before disk warm return:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: When any fresh disk cache exists, this returns before the normal seeded-pricing path runs. That lets stale disk data outrank current `models.providers.*.models[].cost` or manifest-owned catalog prices for up to the cache TTL, so load disk entries as an input and still overlay current config/manifest pricing before returning.
  - confidence: 0.9
- **[P2] Retry when the circuit cooldown ends:** `src/gateway/model-pricing-cache.ts:1232`
  - body: The circuit-open path schedules the next refresh with the normal 24h cache TTL. After the third failure, the background service will not retry when the 10-minute cooldown expires unless some caller manually refreshes, so schedule the next attempt for the remaining cooldown.
  - confidence: 0.88
- **[P3] Schedule disk refresh for the remaining TTL:** `src/gateway/model-pricing-cache.ts:1184`
  - body: A fresh disk cache always schedules the next refresh 24h from process start, even when the file is already near expiry. A 23h-old cache can therefore stay in memory for about 47h, so schedule by `CACHE_TTL_MS - age` instead.
  - confidence: 0.82
