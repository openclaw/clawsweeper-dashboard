## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P2] Apply configured pricing before disk warm return:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: A fresh disk cache returns before current `models.providers.*.models[].cost` and manifest pricing are collected. If a user changes local pricing while the disk cache is fresh, stale disk data can win for the cache window; overlay current configured and manifest pricing before returning.
  - confidence: 0.91
- **[P2] Retry when the circuit cooldown expires:** `src/gateway/model-pricing-cache.ts:1232`
  - body: The open-circuit path logs the remaining 10-minute cooldown but schedules through `scheduleRefresh`, which waits the 24h cache TTL. After three failures, a recovered pricing service can be ignored until the daily timer instead of retrying after cooldown.
  - confidence: 0.9
- **[P3] Schedule disk refresh for remaining TTL:** `src/gateway/model-pricing-cache.ts:1184`
  - body: A fresh disk cache always schedules a full 24h delay from process start. A 23-hour-old cache can remain in use for almost another day, and repeated restarts can keep postponing the live refresh; schedule from the loaded cache's remaining TTL.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/gateway/model-pricing-cache.ts:82`
  - body: This changes user-visible Gateway startup and pricing behavior, but the PR does not update `CHANGELOG.md`. Add a single-line active changelog entry covering the timeout, disk cache, breaker, and readiness behavior.
  - confidence: 0.88
