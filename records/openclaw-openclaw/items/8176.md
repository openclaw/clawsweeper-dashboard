## Summary

Keep open. Current main still has additive user-context and system-prompt hook mutations, but no public field, merge rule, resolver output, or embedded/CLI/Codex application path that can replace the submitted user prompt; https://github.com/openclaw/openclaw/pull/57630 remains the active implementation candidate with review feedback.

## What This Changes

The issue asks for plugin prompt-build hooks, especially legacy `before_agent_start`, to be able to replace the submitted user prompt before model execution.

## Best Possible Solution

Define a modern `before_prompt_build` user-prompt replacement field, mirror it through legacy `before_agent_start` only if approved, and apply it consistently across embedded, CLI, and Codex prompt assembly with docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Current-main source inspection gives high confidence that replacement is unavailable because the public contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable direction is a deliberate `before_prompt_build` replacement contract with compatibility mapping and consistent embedded, CLI, and Codex behavior.
