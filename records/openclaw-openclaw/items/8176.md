## Summary

Keep open: current main and v2026.5.22 still lack a public hook result field or prompt path that lets a plugin replace the submitted user prompt; the remaining work is a Plugin SDK API and prompt-redaction semantics decision already represented by the open implementation PR.

## What This Changes

The issue asks for plugin prompt-build hooks, especially legacy before_agent_start, to replace the submitted user prompt so plugins can truncate or redact oversized current prompts before model submission.

## Best Possible Solution

Define one first-class before_prompt_build prompt replacement contract with legacy before_agent_start compatibility, policy filtering, deterministic precedence, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current hook types, mutation allowlists, and embedded/CLI/Codex prompt paths all build from the original prompt plus context additions. I did not run a live provider call because the missing public contract is clear from source inspection.

## Solution Assessment

No for a legacy-only promptOverride field. The maintainable solution is a first-class before_prompt_build replacement contract with legacy compatibility, runner parity, policy filtering, and explicit security semantics.
