## Summary

Keep open: current main and latest release v2026.6.8 still expose only additive prompt/context and system-prompt hook fields, while the requested user-prompt replacement API remains missing and security-sensitive.

## What This Changes

The issue requests a plugin hook result that can replace the current user prompt before model submission.

## Best Possible Solution

Define a maintainer-approved prompt replacement contract for `before_prompt_build` plus legacy `before_agent_start`, then implement it consistently across embedded, CLI, and Codex harness prompt paths with docs and regression tests.

## Reproduction Assessment

Yes at source level: public hook result types lack a replacement field and embedded, CLI, and Codex paths only wrap the original prompt with prepend/append context.

## Solution Assessment

No for a minimal legacy-only field; the maintainable solution needs a modern `before_prompt_build` contract, legacy compatibility, merge precedence, policy filtering, and runner parity.
