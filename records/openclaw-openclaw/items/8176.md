## Summary

Keep open: current main and v2026.5.12 still expose additive prompt/context mutation only, while user-prompt replacement remains an unresolved Plugin SDK API and security-semantics decision.

## What This Changes

The issue requests allowing plugin prompt hooks to replace the submitted user prompt before model execution.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a consistent prompt-build replacement contract across embedded, CLI, and Codex paths, including policy filtering, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a Plugin SDK capability request. Source inspection of current main and v2026.5.12 gives high confidence that prompt hooks support additive context but not user-prompt replacement.

## Solution Assessment

No for a narrow legacy-only promptOverride field. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
