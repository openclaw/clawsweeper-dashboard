## Summary

Keep open. Current main still has no public hook result field, merge rule, resolver output, runner path, or docs for replacing the submitted user prompt; the old implementation PR is closed unmerged and the newer raw-content mutation issue is broader rather than a direct resolution.

## What This Changes

Allow plugin prompt hooks to replace the effective user prompt before model submission, with context prepended to the replacement when both are present.

## Best Possible Solution

Define a deliberate before_prompt_build user-prompt replacement contract, decide how legacy before_agent_start maps to it, and implement it consistently across embedded, CLI, and Codex harness prompt assembly with docs and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a Plugin SDK capability request. Source inspection on current main gives high confidence that user-prompt replacement is unavailable because contracts, merger, resolver, runners, tests, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a before_agent_start-only change because that hook is documented as legacy. The maintainable path is to define the modern before_prompt_build replacement API first, then add any approved legacy compatibility mapping and align it with the broader raw-content mutation discussion.
