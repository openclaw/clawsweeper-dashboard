## Summary

Keep open. Current main still exposes system-prompt replacement and additive user-prompt context, but there is no public hook result field, merge rule, or runner path that can replace the submitted user prompt. The related implementation PR #8178 is closed unmerged, and the merged Codex parity work in #70313 did not add a general prompt replacement contract.

## What This Changes

The request is to let plugin prompt-build/start hooks replace the current user prompt so oversized or sensitive prompts can be truncated or redacted before model submission.

## Best Possible Solution

Define a documented user-prompt replacement contract on modern before_prompt_build first, specify ordering with prepend/append context, and mirror legacy before_agent_start only as an approved compatibility path.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested plugin API capability. Source inspection of current main shows no result field or execution path for replacing the user prompt.

## Solution Assessment

No, not as a before_agent_start-only change because that hook is legacy. The maintainable fix should be an approved before_prompt_build contract with legacy mapping only if maintainers want compatibility.
