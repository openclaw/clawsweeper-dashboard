## Summary

Keep open: current `main` and the latest release still support additive prompt context and system-prompt mutation, but they do not expose or apply a plugin-owned replacement for the submitted user prompt. The related implementation attempts are closed unmerged, and the remaining work is a plugin API/security product decision rather than conservative cleanup.

## What This Changes

The issue requests a plugin hook result that can replace the current user prompt before model submission, especially for truncation or redaction of oversized prompts.

## Best Possible Solution

Keep this issue open until maintainers either approve a shared prompt replacement contract for `before_prompt_build` plus legacy `before_agent_start`, or explicitly reject that plugin API; an accepted fix should wire the contract consistently through the hook types, mutation filtering, merge precedence, embedded runner, CLI runner, Codex harness, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current `main` and `v2026.6.6` shows the hook result type has no user-prompt replacement field, and embedded, CLI, and Codex paths only wrap the original prompt with additive context.

## Solution Assessment

No, not as the legacy-only minimal change in the issue body. The maintainable solution is a shared, documented prompt-build replacement contract with explicit precedence and security filtering across all prompt submission paths.
