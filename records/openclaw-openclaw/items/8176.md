The issue asks for plugin prompt hooks to support replacing the submitted user prompt before model execution.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a consistent prompt-build replacement contract across embedded, CLI, and Codex paths, including docs, regression coverage, and real redaction/truncation proof.

## Solution Assessment

No for a narrow legacy-only promptOverride field. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
