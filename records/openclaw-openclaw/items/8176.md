## Summary

Current main still lacks any public hook result or runner path for replacing the submitted user prompt. The request remains aligned with OpenClaw’s plugin-extension direction, but it needs an explicit maintainer decision because prompt replacement expands a security-sensitive plugin contract and must behave consistently across embedded, CLI, and Codex runners.

## What This Changes

Add a trusted prompt-build hook result that can replace the outgoing user prompt before prepend and append context are applied.

## Best Possible Solution

Define one reviewed replacement field on modern `before_prompt_build`, retain it through legacy `before_agent_start` compatibility, specify deterministic multi-plugin precedence, and apply the resulting effective prompt identically before prepend/append context in embedded, CLI, and Codex submission paths with focused contract and runner tests.

## Reproduction Assessment

Yes, at source level. The exhaustive public result type, shared merge helper, documented contract, and inspected runner paths provide a high-confidence way to verify that current main cannot replace the submitted user prompt.

## Solution Assessment

No, the original minimal embedded-runner-only proposal is not the best complete fix. The maintainable solution is one public contract with explicit precedence, security semantics, and parity across embedded, CLI, and Codex runners.
