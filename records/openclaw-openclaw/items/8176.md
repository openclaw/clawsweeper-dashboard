## Summary

Keep open: current main and the latest release still expose additive prompt context and system-prompt mutation, but there is no public hook result or runner path that can replace the submitted user prompt. The related open implementation candidate needs product and security review rather than a separate cleanup close.

## What This Changes

Request adding a public plugin hook field that can replace the effective user prompt before provider submission so plugins can redact or truncate oversized prompts.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a consistent prompt-build replacement contract across embedded, CLI, and Codex, including docs, tests, and real redaction proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection of current main and `v2026.5.12` gives high confidence replacement-prompt support is absent while additive context exists.

## Solution Assessment

No for a narrow legacy-only `promptOverride` field. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
