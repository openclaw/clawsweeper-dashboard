## Summary

Keep open. Current main still supports additive prompt/context hook mutations, but it does not expose or apply a hook result that can replace the submitted user prompt; the direct implementation PR was closed unmerged, and adjacent system-prompt/model override work does not solve oversized current-prompt failures.

## What This Changes

Request a plugin hook result field that lets before_prompt_build or legacy before_agent_start replace the current user prompt before model submission.

## Best Possible Solution

Define and document a modern before_prompt_build user-prompt replacement contract, mirror it through legacy before_agent_start if approved, and apply it consistently across embedded, CLI, and Codex harness prompt assembly with regression coverage and changelog.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a plugin API capability request. Source inspection on current main gives high confidence that replacement is unavailable because the contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a before_agent_start-only change because that hook is documented as legacy. The maintainable path is an explicit before_prompt_build replacement field first, with deliberate compatibility mapping for before_agent_start if maintainers approve the API.
