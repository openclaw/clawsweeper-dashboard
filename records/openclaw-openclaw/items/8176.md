## Summary

Keep open. Current main still lacks a public user-prompt replacement field, merge rule, resolver output, and embedded/CLI/Codex application path; https://github.com/openclaw/openclaw/pull/57630 remains the active implementation candidate and still has review feedback.

## What This Changes

Request to let plugin prompt-build and legacy `before_agent_start` hooks replace the current user prompt before model submission.

## Best Possible Solution

Define a modern `before_prompt_build` user-prompt replacement contract, mirror it through legacy `before_agent_start` only if approved, and land it consistently across embedded, CLI, and Codex prompt assembly with docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Current-main source inspection gives high confidence that replacement is unavailable because the contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable solution is a deliberate `before_prompt_build` replacement field with compatibility mapping and consistent embedded, CLI, and Codex behavior.
