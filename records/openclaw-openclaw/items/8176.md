## Summary

Keep open: current main and v2026.5.22 still support additive prompt context and system-prompt mutation, but not hook-owned replacement of the submitted user prompt. The open implementation candidate at https://github.com/openclaw/openclaw/pull/57630 is the right place to continue, but it is still dirty/incomplete and this remains a public Plugin SDK plus prompt-redaction semantics decision.

## What This Changes

The issue asks for plugin prompt hooks to replace the submitted user prompt so trusted plugins can truncate or redact oversized or sensitive current input before model submission.

## Best Possible Solution

Define a first-class before_prompt_build prompt replacement contract with legacy before_agent_start compatibility, deterministic precedence, allowPromptInjection filtering, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current hook types, mutation allowlists, and embedded/CLI/Codex prompt paths all build from the original prompt plus additive context. I did not run a live provider call because the missing public contract is clear from source inspection.

## Solution Assessment

No for a legacy-only promptOverride field. The maintainable solution is a first-class before_prompt_build replacement contract with legacy compatibility, runner parity, policy filtering, and explicit security semantics.
