## Summary

Keep open: current main and v2026.6.5 still lack a hook-owned replacement of the submitted user prompt. The remaining work is a public Plugin SDK and security-semantics decision, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/57630 is not merged.

## What This Changes

The issue requests a plugin hook result field that lets `before_agent_start` or prompt-build hooks replace the submitted user prompt before additive context is applied.

## Best Possible Solution

Define one first-class `before_prompt_build` user-prompt replacement contract with legacy `before_agent_start` compatibility, deterministic precedence, policy filtering, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current hook result types and prompt assembly paths accept additive context and system prompt mutations but have no field or flow that replaces the submitted user prompt. I did not run a live provider call because the missing public contract is clear from source inspection.

## Solution Assessment

No for a legacy-only `promptOverride` field. The maintainable path is a first-class `before_prompt_build` replacement contract with legacy compatibility, runner parity, policy filtering, and explicit security semantics.
