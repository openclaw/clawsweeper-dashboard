## Summary

Keep open: current main and v2026.5.22 still support additive prompt context and system-prompt mutation, but they do not expose or apply a hook-owned replacement for the submitted user prompt. The open implementation at https://github.com/openclaw/openclaw/pull/57630 is the active review path, so this issue should not be closed before that API/security decision lands or is explicitly rejected.

## What This Changes

The issue requests a public plugin hook result field that lets `before_agent_start` and prompt-build hooks replace the submitted user prompt before context is wrapped around it.

## Best Possible Solution

Define and review one first-class prompt replacement contract for `before_prompt_build` with legacy `before_agent_start` compatibility, deterministic precedence, policy filtering, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current hook types, mutation allowlists, and embedded/CLI/Codex prompt paths build from the original prompt plus additive context. I did not run a live provider call because the missing public contract is clear from current source and docs.

## Solution Assessment

No for a legacy-only `promptOverride` field. The maintainable solution is a first-class `before_prompt_build` replacement contract with legacy compatibility, runner parity, prompt-injection policy filtering, and explicit security semantics.
