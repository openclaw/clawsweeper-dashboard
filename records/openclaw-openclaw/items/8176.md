## Summary

Keep open. Current main still supports system-prompt replacement and additive prompt context, but it has no public hook result field or runner path that lets a plugin replace the submitted user prompt; the related implementation PR is closed and unmerged.

## What This Changes

The issue requests a plugin hook result field that lets `before_prompt_build` and legacy `before_agent_start` replace the submitted user prompt before prepend/append context is applied.

## Best Possible Solution

Add and document an explicit user-prompt replacement result field for `before_prompt_build`, keep legacy `before_agent_start` compatibility, merge it through the shared hook helper, and apply it consistently in embedded, CLI, and Codex runner paths before additive context fields.

## Reproduction Assessment

Yes, at source level. A plugin cannot return a typed user-prompt replacement because current main has no hook result field, allowlist entry, merge helper output, or runner consumption path for it.

## Solution Assessment

Yes. An explicit prompt replacement field governed by the existing prompt-mutation policy and applied before prepend/append context is the narrowest maintainable solution; the existing `systemPrompt` and additive context hooks do not address oversized current user prompts.
