## Summary

Keep open. Current main still exposes only system-prompt replacement and additive user-prompt context for `before_prompt_build` and legacy `before_agent_start`; there is no hook result field or runner path that can replace the submitted user prompt. The related implementation PR #8178 is closed unmerged, and the merged Codex parity work in #70313 did not add a general prompt replacement contract.

## What This Changes

The issue requests a plugin hook result field that lets `before_agent_start` replace the user prompt before model submission, with `prependContext` applied afterward.

## Best Possible Solution

Define a documented user-prompt replacement contract on modern `before_prompt_build` first, mirror legacy `before_agent_start` compatibility if accepted, and apply it consistently across embedded, CLI, Codex harness, hook policy, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested plugin API capability. Source inspection of current main shows no result field or execution path for replacing the user prompt.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable fix should be an approved `before_prompt_build` contract with legacy mapping if maintainers want compatibility.
