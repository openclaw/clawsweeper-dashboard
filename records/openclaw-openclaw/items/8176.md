## Summary

Keep open. Current main still has no public hook result field or runner path that lets a plugin replace the submitted user prompt; it supports system-prompt replacement and additive user-prompt context only. The directly related implementation PR was closed unmerged, and the later Codex hook parity work did not add a general replacement contract.

## What This Changes

The issue requests an opt-in plugin hook result field that can replace the submitted user prompt before model submission.

## Best Possible Solution

Define a documented user-prompt replacement contract on `before_prompt_build`, decide whether legacy `before_agent_start` mirrors it, then implement consistent precedence and policy behavior across embedded, CLI, and Codex harness paths with regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested plugin API capability. Source inspection gives high confidence that current main does not define or apply a replacement prompt field.

## Solution Assessment

No, not as a `before_agent_start`-only change. The maintainable path should target the documented `before_prompt_build` hook first, then preserve legacy compatibility only after the API semantics are settled.
