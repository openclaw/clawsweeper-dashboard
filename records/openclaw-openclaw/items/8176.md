## Summary

Keep open. Current main still lacks a public Plugin SDK hook result field or runner path that can replace the submitted user prompt; the original implementation PR is closed unmerged, and the newer implementation candidate is still open and conflicting with review blockers.

## What This Changes

The issue asks for a Plugin SDK hook result that can replace the effective user prompt before model submission.

## Best Possible Solution

Define a modern `before_prompt_build` user-prompt replacement contract, mirror it through legacy `before_agent_start` only if approved, and apply it consistently across embedded, CLI, and Codex prompt assembly with docs and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection gives high confidence that current main supports additive prompt context but not user-prompt replacement.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable direction is a deliberate `before_prompt_build` replacement contract with compatibility mapping and consistent embedded, CLI, and Codex behavior.
