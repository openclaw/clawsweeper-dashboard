## Summary

Keep open. Current main supports system-prompt replacement and additive prompt context from prompt hooks, but it still has no public hook result field or runner path that lets a plugin replace the submitted user prompt; the related implementation PR is closed and unmerged.

## What This Changes

The issue requests a plugin hook result field that can replace the effective user prompt before prepend/append context is applied, mainly so plugins can truncate or redact oversized current prompts.

## Best Possible Solution

Add and document an explicit user prompt replacement result field for `before_prompt_build`, with legacy `before_agent_start` compatibility, merged through the shared hook helper and applied consistently in embedded, CLI, and Codex runner paths before additive context fields.

## Reproduction Assessment

Yes. A source-level reproduction is clear: register a prompt hook that returns a replacement-shaped field and current main has no type, allowlist, merge helper, or runner submission path that can consume it, so the original prompt remains the submitted prompt.

## Solution Assessment

Yes. An explicit prompt replacement field, gated by the existing prompt-mutation policy and applied before prepend/append context, is the narrowest maintainable solution; system-prompt fixes and additive context hooks do not solve oversized current user prompts.
