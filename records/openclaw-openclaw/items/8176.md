## Summary

Current main still lacks a public plugin result field for replacing the submitted user prompt, even though the Codex harness now carries a fully prepared prompt into `turn/start`. The issue remains necessary: closed pull requests proposed variants, but none merged, and the remaining decision is the security-sensitive public API contract and deterministic multi-plugin precedence.

## What This Changes

Allow `before_prompt_build` and legacy `before_agent_start` hooks to replace the model-bound user prompt, with prepend and append context applied around the replacement.

## Best Possible Solution

Add one canonical replacement field to `PluginHookBeforePromptBuildResult`, inherit it through legacy `before_agent_start`, define deterministic higher-priority precedence, and apply replacement before shared prepend/append processing in PI, CLI, and Codex with focused docs and regression tests.

## Reproduction Assessment

Yes, at source level: current main has no public replacement field, no merge precedence for one, and no shared propagation path; PI/CLI remain additive-only and Codex consumes the already-resolved prompt.

## Solution Assessment

No, the original minimal embedded-runner-only proposal is not the best complete fix. The maintainable solution is a canonical `before_prompt_build` contract implemented consistently across all runners, with legacy inheritance and explicit security and precedence semantics.
