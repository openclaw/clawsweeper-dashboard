## Summary

Keep open: current main and v2026.5.19 still expose only additive prompt-context hooks, and the active implementation path is the unmerged https://github.com/openclaw/openclaw/pull/57630, which still needs API and security review.

## What This Changes

The issue requests a plugin hook result that can replace the current user prompt before an agent model call.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with one explicit `before_prompt_build` replacement contract covering legacy compatibility, policy filtering, precedence, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level, but not as a strict bug reproduction. Current main and v2026.5.19 expose additive prompt context fields and runner paths, but no user-prompt replacement field or merge/submission path.

## Solution Assessment

No for the narrow legacy-only `promptOverride` shape. The maintainable solution is an explicit `before_prompt_build` replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
