## Summary

Keep open: current main and v2026.6.6 still expose additive prompt context and system-prompt mutation fields, but there is no public hook result field or runner path that lets a plugin replace the submitted user prompt. The direct implementation attempts are closed unmerged, and the remaining work is a plugin API plus security-boundary decision.

## What This Changes

The issue requests a plugin hook result field that can replace the current user prompt before model submission, especially for truncation or redaction of oversized prompts.

## Best Possible Solution

Approve or reject a shared, documented prompt replacement contract for `before_prompt_build` plus legacy `before_agent_start`, then wire it consistently through hook filtering, merge precedence, embedded, CLI, Codex, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.6 shows hook result types, merge logic, embedded, CLI, and Codex paths only support additive context and system-prompt fields, not replacing the submitted user prompt.

## Solution Assessment

No, not as a legacy-only `before_agent_start` patch. The maintainable solution is a shared prompt-build replacement contract with explicit precedence and security filtering across all model submission paths.
