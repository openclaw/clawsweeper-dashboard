## Summary

Keep open: current main and v2026.5.12 still expose additive prompt context and system-prompt mutation hooks, but no public hook result or runner path can replace the submitted user prompt. The linked implementation PR is still open and needs product/security review rather than being treated as a completed fix.

## What This Changes

Allow `before_agent_start` or prompt-build hooks to replace the current user prompt before provider submission.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a focused public prompt replacement contract applied consistently across embedded, CLI, and Codex prompt assembly, with docs, tests, and security-reviewed precedence semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection of current main and v2026.5.12 gives high confidence that replacement-prompt support is still absent while additive prompt context exists.

## Solution Assessment

No for the narrow legacy-only `promptOverride` shape. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
