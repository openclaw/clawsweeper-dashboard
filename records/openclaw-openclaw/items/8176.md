## Summary

Keep open: current main and v2026.6.8 still lack a public hook result field, merge rule, runner path, or docs entry that lets a plugin replace the submitted user prompt. The direct implementation attempts are closed unmerged, and the open llm_input/llm_output PR is a broader partial overlap rather than a safe canonical fix for this before_prompt_build/before_agent_start API decision.

## What This Changes

The issue requests a plugin hook result field that can replace the user prompt before an agent run, with context additions applied around the effective prompt.

## Best Possible Solution

Define a maintainer-approved prompt replacement contract for before_prompt_build plus legacy before_agent_start, then implement it consistently across embedded, CLI, and Codex harness prompt paths with docs, policy filtering, and regression tests.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence reproduction of the feature gap: the public hook result types lack a prompt replacement field, and embedded, CLI, and Codex paths only prepend or append context around the original prompt.

## Solution Assessment

No for the minimal legacy-only shape. The maintainable solution should cover before_prompt_build, legacy before_agent_start, merge precedence, allowPromptInjection filtering, transcript/model split semantics, docs, and all runner paths rather than only adding promptOverride to the deprecated hook.
