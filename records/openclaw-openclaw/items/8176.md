## Summary

Keep open. Current `origin/main` and the latest release still only support additive prompt context and system-prompt mutation from prompt-build hooks; there is no hook result field or runner path that lets a plugin replace the current user prompt, and the direct implementation attempts are closed unmerged.

## What This Changes

Request to let plugin prompt-build hooks replace the current user prompt before model submission, with context prepended or appended around the effective prompt.

## Best Possible Solution

Define a maintainer-approved prompt replacement API for `before_prompt_build` plus legacy `before_agent_start`, including precedence, prompt-injection filtering, docs, and runner parity across embedded, CLI, and Codex paths.

## Reproduction Assessment

Yes. At source level, current `origin/main` shows the public hook result types and embedded, CLI, and Codex assembly paths carry only additive context and system-prompt mutations, not a replacement prompt.

## Solution Assessment

Unclear until maintainers choose the API semantics. A narrow implementation is plausible, but the best fix should first settle replacement precedence, whether downstream hooks see the original or effective prompt, and how prompt-mutation policy filters this field.
