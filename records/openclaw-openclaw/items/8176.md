## Summary

Keep open. Current main still has no public prompt replacement field, merge rule, resolver output, or embedded/CLI runner handling for replacing the submitted user prompt; the open implementation candidate at https://github.com/openclaw/openclaw/pull/57630 is the right place to review or replace the patch.

## What This Changes

Allow plugins to replace the submitted user prompt from prompt-build or legacy before-agent-start hook results so large current prompts can be truncated or redacted before model submission.

## Best Possible Solution

Land a rebased implementation that adds a modern before_prompt_build prompt replacement contract, maps legacy before_agent_start only as compatibility, applies it consistently in embedded, CLI, and Codex prompt assembly, and updates docs, tests, and changelog.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a plugin API capability request. Source inspection on current main gives high confidence that replacement is unavailable because the contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a before_agent_start-only change because that hook is now documented as legacy. The maintainable path is a before_prompt_build replacement field first, with explicit compatibility behavior for before_agent_start if maintainers approve it.
