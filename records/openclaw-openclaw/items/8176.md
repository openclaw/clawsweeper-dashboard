## Summary

Keep open: current main and the latest release still lack a public hook result field or runner path that can replace the submitted user prompt, and the related implementation PR remains open, dirty, and under API/security review.

## What This Changes

Request a Plugin SDK hook result that lets `before_prompt_build` and legacy `before_agent_start` replace the submitted user prompt before model submission.

## Best Possible Solution

Define or explicitly reject a first-class `before_prompt_build` prompt replacement contract with legacy compatibility, deterministic precedence, policy filtering, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current hook types and mutation allowlists expose only additive/system fields, and embedded, CLI, and Codex paths start from the original prompt without reading a replacement field. A live provider run is not needed to prove the missing public contract.

## Solution Assessment

No for a legacy-only `promptOverride` patch. The maintainable solution is a first-class `before_prompt_build` replacement contract with legacy compatibility, runner parity, policy filtering, and explicit security semantics.
