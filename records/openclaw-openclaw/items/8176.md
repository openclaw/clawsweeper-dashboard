## Summary

Keep open. Current main still exposes only additive user-context fields and system-prompt fields for `before_prompt_build` / legacy `before_agent_start`; no public hook result can replace the submitted user prompt, and the linked implementation attempts are closed unmerged.

## What This Changes

The issue requests a plugin hook result field that can replace the submitted user prompt before model calls while preserving prepend/append ordering.

## Best Possible Solution

Keep this issue open as the canonical design thread until OpenClaw either adds and documents an explicit user-prompt replacement field across hook types, merge logic, embedded/CLI/Codex consumers, policy filtering, and regression tests, or maintainers explicitly reject that API.

## Reproduction Assessment

Yes. Source inspection on current main shows the public hook result type, merger, shared helper, embedded runner, CLI runner, Codex harness bridge, and docs all lack a user-prompt replacement path; this is source-reproducible without a live model call.

## Solution Assessment

Unclear. A replacement field is a plausible fix, but the best solution depends on maintainer decisions about the public API name, merge precedence, security policy filtering, and parity across embedded, CLI, and Codex harness paths.
