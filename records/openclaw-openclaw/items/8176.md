## Summary

Keep open: current main and v2026.6.8 still expose only additive prompt/context and system-prompt hook fields, so the requested user-prompt replacement API remains missing and security-sensitive.

## What This Changes

The issue requests an opt-in plugin hook result that can replace the model-bound user prompt before context additions are applied.

## Best Possible Solution

Define a maintainer-approved user-prompt replacement contract for before_prompt_build plus legacy before_agent_start, then implement it consistently across prompt-building paths with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection reproduces the feature gap: the public hook result types lack a prompt replacement field, and the main runners only prepend or append context around the original prompt.

## Solution Assessment

No for the minimal legacy-only shape. The maintainable solution should cover before_prompt_build, legacy before_agent_start, merge precedence, policy filtering, docs, and runner parity rather than only adding promptOverride to the deprecated hook.
