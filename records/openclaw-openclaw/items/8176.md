## Summary

Keep open: current main and v2026.5.12 still only support additive prompt context/system-prompt mutation, not replacement of the submitted user prompt; the linked implementation candidate remains open and needs maintainer API/security review before this tracker can close.

## What This Changes

The issue requests a Plugin SDK hook result that lets before_prompt_build or legacy before_agent_start replace the submitted user prompt before model calls.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a consistent prompt-build replacement contract across embedded, CLI, and Codex paths, including legacy compatibility, allowPromptInjection filtering, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a Plugin SDK capability request. Source inspection of current main and v2026.5.12 gives high confidence that additive context exists while user-prompt replacement is absent.

## Solution Assessment

No for a narrow legacy-only promptOverride field. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
