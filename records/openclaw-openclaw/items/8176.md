## Summary

Keep open: current main and v2026.5.22 still only support additive user prompt context and system-prompt mutations, not a hook-owned replacement for the submitted user prompt. The open implementation at https://github.com/openclaw/openclaw/pull/57630 is the right candidate to review, but this remains an unresolved public Plugin SDK and prompt-redaction semantics decision.

## What This Changes

The issue requests a Plugin SDK hook result field that lets before_prompt_build and legacy before_agent_start replace the submitted user prompt before model submission.

## Best Possible Solution

Define a first-class before_prompt_build prompt replacement contract with legacy before_agent_start compatibility, deterministic precedence, allowPromptInjection filtering, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current hook types, mutation allowlists, and embedded/CLI/Codex prompt paths all build from the original prompt plus context additions. I did not run a live provider call because the missing public contract is clear from source inspection.

## Solution Assessment

No for a legacy-only promptOverride field. The maintainable solution is a first-class before_prompt_build replacement contract with legacy compatibility, runner parity, policy filtering, and explicit security semantics.
