## Summary

Keep open: current main and the latest release still support additive prompt/context mutation only, while user-prompt replacement remains an unresolved Plugin SDK API and security-semantics decision with an open implementation candidate under review.

## What This Changes

The issue requests an opt-in hook result field so plugins can replace the effective user prompt before model submission, especially for truncation or redaction of oversized current prompts.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with one explicit prompt-build replacement contract that covers legacy compatibility, policy filtering, embedded/CLI/Codex runner parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level, but not as a strict bug reproduction. Current main and v2026.5.12 expose additive prompt context fields and runner paths, but no user-prompt replacement field or merge/submission path.

## Solution Assessment

No for a narrow legacy-only promptOverride field. The maintainable solution is an explicit before_prompt_build replacement API with legacy before_agent_start compatibility and consistent embedded, CLI, and Codex behavior.
