## Summary

Keep open. Current main supports system-prompt replacement plus additive prompt context, but it still has no typed prompt replacement field, mutation allowlist entry, merge rule, shared resolver output, or embedded/CLI/Codex runner path that can replace the submitted user prompt; the related implementation PR #8178 is closed and unmerged.

## What This Changes

Add a hook result field that lets plugins replace the current user prompt from prompt-build hooks so context managers can shrink oversized requests before model submission.

## Best Possible Solution

Add an explicit prompt replacement field to the modern and legacy hook contracts, document precedence and ordering, and apply it before prepend/append context across embedded, CLI, and Codex runner paths after maintainer API approval.

## Reproduction Assessment

Yes, at source level. Current main lacks the typed hook result field, allowlist entry, merge output, shared resolver output, and runner application path needed for a plugin to replace the current user prompt.

## Solution Assessment

Likely yes, but it needs maintainer API approval. An explicit replacement field applied before additive context is the narrowest maintainable direction; `systemPrompt` and prepend/append context cannot reduce an oversized current user prompt.
