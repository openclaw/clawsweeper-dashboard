## Summary

Keep open: current main and v2026.5.20 still lack a public hook result field or runner path that can replace the submitted user prompt. https://github.com/openclaw/openclaw/pull/57630 is the active implementation candidate, but the remaining work needs maintainer API and security review rather than a fresh automated fix.

## What This Changes

The issue requests a plugin hook result that can replace the submitted user prompt before model submission, with context additions applied around the effective prompt.

## Best Possible Solution

Define or explicitly reject a first-class `before_prompt_build` prompt replacement contract with legacy compatibility, prompt-injection policy filtering, deterministic precedence, embedded/CLI/Codex parity, public docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current main defines additive prompt mutation fields only, and the embedded, CLI, and Codex paths all build from the original prompt plus context additions. I did not run a live provider call because the missing contract is clear from the public types and prompt assembly code.

## Solution Assessment

No for the narrow legacy-only `promptOverride` shape. The maintainable solution is an explicit `before_prompt_build` replacement API with legacy compatibility and consistent policy, precedence, and runner behavior.
