## Summary

Keep open. Current main still exposes only additive prompt/context hook fields and system-prompt replacement; there is no typed or applied path for a plugin to replace the effective user prompt, so this is not cleanup-closeable.

## What This Changes

The issue requests an opt-in hook result field that lets plugins replace the effective user prompt before model submission.

## Best Possible Solution

If maintainers accept the API, add and document a prompt replacement field for modern `before_prompt_build` and legacy `before_agent_start`, then wire it through shared helpers and all runner paths with ordering tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the missing capability: source inspection shows hook result types, merging, helper output, and runner consumers cannot type or apply a user-prompt replacement field.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the public API, but the narrow maintainable direction is a single explicit replacement field with documented precedence and coverage across embedded, CLI, and Codex paths.
