## Summary

Keep open. Current main still has no public hook result field or runner path that lets a plugin replace the submitted user prompt; the implemented hook surfaces only support system-prompt replacement and additive prompt context. The earlier implementation PR for this exact request was closed unmerged, and the later Codex hook parity work did not add a general prompt replacement contract.

## What This Changes

The issue requests an optional `promptOverride`-style hook result that replaces the effective user prompt before model submission, with prepend context applied afterward.

## Best Possible Solution

Define a documented user-prompt replacement contract on the modern prompt hook, preserve legacy compatibility if accepted, and implement the same precedence and policy behavior across embedded, CLI, and Codex harness paths with regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested plugin API capability. Source inspection gives high confidence that current main does not define or apply a replacement prompt field.

## Solution Assessment

No, not as a `before_agent_start`-only change. The narrow maintainable solution should target the documented `before_prompt_build` hook first and keep legacy `before_agent_start` behavior compatible only after maintainers settle the API semantics.
