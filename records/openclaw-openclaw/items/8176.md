## Summary

Keep open: current main and the latest release still support additive prompt context and system-prompt mutation, but there is no public hook result or runner path that can replace the submitted user prompt. A related implementation PR is open, so this should stay routed to maintainer product/security review rather than being closed or queued as a separate repair.

## What This Changes

The issue requests a hook result field that lets plugins replace the effective user prompt before provider submission.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a focused public prompt replacement contract applied consistently across embedded, CLI, and Codex prompt assembly, with docs, tests, and security-reviewed precedence semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection of current main and `v2026.5.12` gives high confidence that replacement-prompt support is still absent while additive prompt context exists.

## Solution Assessment

No for a narrow legacy-only `promptOverride` shape. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
