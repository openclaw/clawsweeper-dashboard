## Label Justifications

- `P2`: This is a bounded plugin API enhancement with real context-limit and redaction use cases, but it does not break default agent operation today.
- `impact:security`: The requested hook would control what user-authored prompt content reaches the model, which is directly relevant to redaction and prompt integrity.

## Summary

Keep open: current main and the latest release still support additive prompt context and system-prompt mutation only; no public hook result field can replace the submitted user prompt, and the closed related PRs did not land that remaining behavior.

## What This Changes

The issue asks OpenClaw to let plugin hooks replace the effective user prompt before the model call.

## Best Possible Solution

Either implement and document an explicit user-prompt replacement contract across `before_prompt_build` and legacy `before_agent_start`, with runner parity and security semantics, or record a maintainer decision rejecting that API.

## Reproduction Assessment

Yes, at source level: current main lacks a public replacement field, merge rule, shared hook propagation path, and runner consumption path. I did not run a live oversized Anthropic prompt because the read-only review can prove the missing API from source.

## Solution Assessment

No, the original `before_agent_start`-only shape is too narrow now that `before_prompt_build` is the preferred prompt hook. The maintainable solution should define the public prompt-build contract first, then carry legacy compatibility through the same helper and all runner surfaces.
