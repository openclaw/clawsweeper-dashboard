## Summary

Keep open: current main and v2026.5.12 still support additive prompt/context mutation only, while user-prompt replacement remains an unresolved Plugin SDK API and security-semantics decision.

## What This Changes

Request an opt-in Plugin SDK hook result that can replace the effective user prompt before agent runs, so plugins can truncate or redact oversized current prompts.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a consistent prompt-build replacement contract across embedded, CLI, and Codex paths, including policy filtering, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes at source level, but not as a strict bug reproduction. Current main and v2026.5.12 expose additive prompt context fields and runner paths, but no user-prompt replacement field or merge/submission path.

## Solution Assessment

No for a narrow legacy-only promptOverride field. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
