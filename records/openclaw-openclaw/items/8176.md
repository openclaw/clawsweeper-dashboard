## Summary

Keep open. Current main still has additive prompt context and system-prompt mutation hooks, but no public hook result field or runner path that can replace the submitted user prompt; the active implementation candidate remains open, conflicting, and under review.

## What This Changes

The issue requests a Plugin SDK hook result that can replace the submitted user prompt before model calls.

## Best Possible Solution

Define a deliberate `before_prompt_build` user-prompt replacement contract, mirror it through legacy `before_agent_start` only if approved, and apply it consistently across embedded, CLI, and Codex prompt assembly with docs and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection gives high confidence that current main cannot replace the submitted user prompt because the contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable solution is a modern `before_prompt_build` replacement contract with compatibility mapping and consistent embedded, CLI, and Codex behavior.
