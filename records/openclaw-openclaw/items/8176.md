## Summary

Keep open: current main and the latest release still lack a public plugin hook result that can replace the submitted user prompt, and the remaining work is a security-sensitive plugin API decision rather than stale cleanup.

## What This Changes

Request to let plugin prompt hooks replace the current user prompt before model submission so plugins can truncate or redact oversized input.

## Best Possible Solution

Define a maintainer-approved prompt replacement contract for before_prompt_build with legacy before_agent_start compatibility, explicit precedence, prompt-injection filtering, runner coverage, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows hook result types, merge logic, embedded, CLI, and Codex paths only support additive context and system-prompt fields, not replacing the submitted user prompt.

## Solution Assessment

No, not as a legacy-only before_agent_start patch. The maintainable solution is a shared before_prompt_build replacement contract with legacy compatibility plus explicit precedence, filtering, and sibling-runner coverage.
