## Summary

Keep open: current main still has additive prompt context and system-prompt hook support, but no public hook result field or runner path that can replace the submitted user prompt. The open implementation candidate at https://github.com/openclaw/openclaw/pull/57630 covers the same capability but is still open, conflicting, and needs product/security follow-up rather than a new cleanup close.

## What This Changes

Allow plugin prompt hooks to replace the current user prompt before model calls so oversized or sensitive prompts can be truncated or redacted.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a focused API change that documents a replacement field, applies it consistently across embedded, CLI, and Codex prompt assembly, and locks the security and precedence semantics in tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection gives high confidence that current main lacks user-prompt replacement while supporting additive prompt context.

## Solution Assessment

No current-main solution exists yet. The proposed direction is plausible, but the best maintainable fix needs a deliberate public API and consistent embedded, CLI, and Codex behavior rather than a partial legacy-only hook change.

## Security Review

Status: needs_attention

Summary: A prompt replacement hook needs explicit security review before merge because plugins may use it for redaction before provider calls.

Concerns:
