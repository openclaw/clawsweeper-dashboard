## Summary

Keep open. Current main still has additive prompt/context hook support but no public hook result field, merge rule, resolver output, or runner path that can replace the submitted user prompt; the remaining work is a public hook API decision with an open implementation candidate needing maintainer review.

## What This Changes

Add an opt-in plugin hook result that can replace the current user prompt before model submission.

## Best Possible Solution

Define a modern `before_prompt_build` user-prompt replacement contract, mirror it deliberately through legacy `before_agent_start`, and wire consistent embedded, CLI, and Codex harness behavior with docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested plugin API capability. Source inspection on current main gives high confidence that replacement is unavailable because the contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable path is a deliberate `before_prompt_build` contract first, with compatibility mapping for `before_agent_start` if maintainers approve the API.
