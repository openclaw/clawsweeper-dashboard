## Summary

Keep open: current main and the latest shipped release still expose additive prompt context and system-prompt mutation hooks, but no public hook result or runner path can replace the submitted user prompt. The related open PR targets the same capability, but it is still an implementation candidate needing product/security review rather than a cleanup close.

## What This Changes

The issue requests a Plugin SDK hook result field that can replace the effective user prompt before model submission.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a focused public prompt replacement contract applied consistently across embedded, CLI, and Codex prompt assembly, with docs, tests, and security-reviewed precedence semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection gives high confidence that current main and v2026.5.12 lack a replacement-prompt field while supporting additive prompt context.

## Solution Assessment

No for the narrow legacy-only promptOverride shape. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
