## Summary

Keep open: current main and the latest release still support additive prompt context and system-prompt mutation, but they do not expose or apply a hook result that can replace the submitted user prompt. The related implementation candidate remains open and needs maintainer product/security review rather than a cleanup close.

## What This Changes

The issue requests a Plugin SDK hook result that lets `before_prompt_build` or legacy `before_agent_start` replace the user prompt before model submission.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a consistent prompt-build replacement contract across embedded, CLI, and Codex paths, including docs, regression coverage, and real redaction/truncation proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection of current main and `v2026.5.12` gives high confidence that replacement-prompt support is absent while additive context exists.

## Solution Assessment

No for a narrow legacy-only `promptOverride` field. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
