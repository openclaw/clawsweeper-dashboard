## Summary

Keep open: current main and v2026.5.12 support additive prompt context and system-prompt mutation hooks, but they still do not expose or apply a user-prompt replacement field. The open related PR covers the same capability, but it is still open/conflicting and needs product/security review rather than cleanup closure.

## What This Changes

The issue requests a Plugin SDK hook result field that lets `before_agent_start` or the newer prompt-build hook replace the submitted user prompt before model submission.

## Best Possible Solution

Review or replace the open related PR with a focused public prompt replacement contract that applies consistently across embedded, CLI, and Codex prompt assembly, with docs, tests, and security-reviewed precedence semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection gives high confidence that current main and v2026.5.12 lack a replacement-prompt field while supporting additive prompt context.

## Solution Assessment

No. A legacy-only `promptOverride` would be too narrow; the maintainable path is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
