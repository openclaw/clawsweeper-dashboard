## Summary

Keep open. Current main still lacks a public Plugin SDK hook result field or runner path that can replace the submitted user prompt, and the active implementation candidate remains open and conflicting with review blockers.

## What This Changes

Request a Plugin SDK hook result that can replace the current user prompt before the model call, with context prepended to the effective prompt.

## Best Possible Solution

Define a modern `before_prompt_build` user-prompt replacement contract, mirror it through legacy `before_agent_start` only if approved, and apply it consistently across embedded, CLI, and Codex prompt assembly with docs and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Current-main source inspection gives high confidence that replacement is unavailable because the public contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable direction is a deliberate `before_prompt_build` replacement contract with compatibility mapping and consistent embedded, CLI, and Codex behavior.
