## Summary

Keep open: current main and the latest release still expose additive prompt hook fields only, while user-prompt replacement remains an open Plugin SDK API/security decision with an active implementation candidate under review.

## What This Changes

This issue requests a plugin hook result that can replace the submitted user prompt before model submission, with context applied around the effective replacement prompt.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with an explicit `before_prompt_build` replacement contract covering legacy `before_agent_start`, policy filtering, precedence, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current main defines additive prompt hook fields only, and the embedded, CLI, and Codex paths all build from the original prompt plus context additions rather than a hook-owned replacement.

## Solution Assessment

No for a narrow legacy-only `promptOverride` field. The maintainable solution is an explicit `before_prompt_build` replacement API with legacy compatibility and consistent policy, precedence, runner behavior, docs, and tests.
