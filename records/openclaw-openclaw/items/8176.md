## Summary

Keep open: current main and v2026.5.19 still lack a user-prompt replacement hook, while https://github.com/openclaw/openclaw/pull/57630 remains the active but unmerged implementation path that needs API and security review.

## What This Changes

The issue requests a Plugin SDK hook result that lets prompt hooks replace the current user prompt before model submission for truncation or redaction.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with one explicit `before_prompt_build` prompt replacement contract covering legacy compatibility, policy filtering, precedence, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level, but not as a strict bug reproduction. Current main and v2026.5.19 expose additive prompt context fields and runner paths, but no user-prompt replacement field or merge/submission path.

## Solution Assessment

No for the narrow legacy-only `promptOverride` shape. The maintainable solution is an explicit `before_prompt_build` replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
