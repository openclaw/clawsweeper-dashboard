## Summary

Keep open: current main and v2026.5.12 still lack a hook result that can replace the submitted user prompt, and the open implementation candidate needs maintainer product/security review rather than cleanup closure.

## What This Changes

Allow plugin prompt hooks to replace the effective user prompt before model calls so oversized or sensitive current prompts can be truncated or redacted.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with a consistent prompt-build replacement contract across embedded, CLI, and Codex paths, including allowPromptInjection filtering, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a Plugin SDK capability request. Source inspection of current main and v2026.5.12 gives high confidence that additive prompt context exists while user-prompt replacement is absent.

## Solution Assessment

No for a narrow legacy-only promptOverride field. The maintainable solution is an explicit prompt-build replacement API with legacy compatibility and consistent embedded, CLI, and Codex behavior.
