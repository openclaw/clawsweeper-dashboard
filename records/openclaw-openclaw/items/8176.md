## Summary

Keep open. Current main still supports additive prompt context and system-prompt mutation, but it does not expose or consume a hook result field that replaces the submitted user prompt.

## What This Changes

The issue requests an optional plugin hook field that can replace the effective user prompt before model submission.

## Best Possible Solution

Define and document a modern `before_prompt_build` user-prompt replacement contract first, including merge precedence and prepend/append ordering, then mirror it through legacy `before_agent_start` only if maintainers approve that compatibility path.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested plugin API capability. Source inspection of current main shows no result field, merge rule, or runner path for replacing the user prompt.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The narrow maintainable solution should start with the current `before_prompt_build` contract and then define any legacy mapping deliberately.
