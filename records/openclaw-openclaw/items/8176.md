## Summary

Keep open. Current main still lacks a user-prompt replacement field, merge rule, resolver output, and embedded/CLI/Codex runner application path; the active implementation candidate is still open and has unresolved review feedback.

## What This Changes

Request an opt-in plugin hook result that can replace the current user prompt so plugins can truncate or redact oversized prompts before the model call.

## Best Possible Solution

Define a modern `before_prompt_build` user-prompt replacement contract, mirror it through legacy `before_agent_start` only if approved, and land it consistently across embedded, CLI, and Codex harness prompt assembly with docs and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK capability request. Source inspection on current main gives high confidence that replacement is unavailable because the contract, merger, resolver, runners, and docs expose only additive/system prompt fields.

## Solution Assessment

No, not as a `before_agent_start`-only change because that hook is documented as legacy. The maintainable solution is a deliberate `before_prompt_build` replacement field with compatibility mapping and consistent embedded, CLI, and Codex behavior.
