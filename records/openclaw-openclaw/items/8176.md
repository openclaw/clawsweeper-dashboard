## Summary

Keep open: current main and v2026.5.20 still expose additive prompt/context fields and system-prompt mutation, but no public hook result field or runner path for replacing the submitted user prompt. The active implementation candidate is https://github.com/openclaw/openclaw/pull/57630, so this issue should remain open for maintainer API and security review rather than conservative cleanup.

## What This Changes

Requests a Plugin SDK hook result field that lets `before_prompt_build` and legacy `before_agent_start` replace the submitted user prompt before model submission.

## Best Possible Solution

Adopt or explicitly reject a first-class `before_prompt_build` prompt replacement contract with legacy `before_agent_start` compatibility, deterministic precedence, runner parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current main defines additive prompt mutation fields only, and the embedded, CLI, and Codex paths all build from the original prompt plus context additions. I did not run a live provider call because the missing contract is clear from public types and prompt assembly code.

## Solution Assessment

No for a legacy-only `promptOverride` shape. The maintainable solution is an explicit `before_prompt_build` replacement API with legacy compatibility and consistent policy, precedence, and runner behavior.
