## Summary

Keep open: current main and v2026.5.20 still only support additive prompt/context mutations and system-prompt replacement, not replacement of the submitted user prompt. The active implementation candidate is https://github.com/openclaw/openclaw/pull/57630, but the remaining work is a public Plugin SDK API and security decision rather than conservative cleanup.

## What This Changes

The issue requests a plugin hook result field that can replace the submitted user prompt before model calls.

## Best Possible Solution

Define or explicitly reject a first-class `before_prompt_build` prompt replacement contract with legacy compatibility, policy filtering, deterministic precedence, embedded/CLI/Codex parity, public docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current main defines additive prompt mutation fields only, and the embedded, CLI, and Codex paths all build from the original prompt plus context additions. I did not run a live provider call because the missing contract is clear from the public types and prompt assembly code.

## Solution Assessment

No for the narrow legacy-only `promptOverride` shape. The maintainable solution is an explicit `before_prompt_build` replacement API with legacy compatibility and consistent policy, precedence, and runner behavior.
