## Summary

Keep open: current main and v2026.6.8 still expose only additive prompt/context and system-prompt hook fields, while the requested user-prompt replacement API remains missing and security-sensitive.

## What This Changes

The issue requests a plugin hook result field that can replace the submitted user prompt before model submission.

## Best Possible Solution

Define a maintainer-approved prompt replacement contract for before_prompt_build plus legacy before_agent_start, then implement it consistently across embedded, CLI, and Codex harness prompt paths with docs and regression tests.

## Reproduction Assessment

Yes, source inspection provides a high-confidence reproduction of the feature gap: the public hook result types lack a prompt replacement field, and the runners only prepend or append context around the original prompt.

## Solution Assessment

No for the minimal legacy-only shape: the maintainable solution should cover before_prompt_build, legacy before_agent_start, merge precedence, policy filtering, and all runner paths rather than only adding promptOverride to the deprecated hook.
