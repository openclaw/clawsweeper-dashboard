## Summary

Keep open: current main and the latest release still do not expose a hook result field or runner path that can replace the submitted user prompt. The active implementation candidate needs maintainer API and security review before this can be treated as implemented.

## What This Changes

Request to let plugin prompt hooks replace the submitted user prompt so plugins can truncate or redact oversized current prompts before provider submission.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/57630 with one explicit `before_prompt_build` replacement contract covering legacy compatibility, prompt-injection policy filtering, precedence, embedded/CLI/Codex parity, docs, regression tests, and redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level rather than as a strict runtime bug: current main defines additive prompt fields only, and the embedded, CLI, and Codex paths all build from the original prompt plus context additions.

## Solution Assessment

No for the narrow legacy-only `promptOverride` shape. The maintainable solution is an explicit `before_prompt_build` replacement API with legacy compatibility and consistent policy, precedence, and runner behavior.
