## Summary

## What This Changes

This PR lets task maintenance recover terminal `lost` cron rows with a `backing session missing` signal from durable cron run history and clears stale terminal errors when recovery has no error.

## Best Possible Solution

Land the narrowed recovery after task/session-state owner acceptance, or require live gateway cron proof first if the seeded CLI proof is not enough for this persisted-state change.

## Reproduction Assessment

## Solution Assessment

Yes, subject to owner acceptance. The patch keeps recovery scoped to cron rows marked lost by the backing-session-missing path; pruning would lose durable completion truth, and the adjacent startup-liveness PR covers a different pre-terminal race.
