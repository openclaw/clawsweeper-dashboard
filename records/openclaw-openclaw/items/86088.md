## What This Changes

The PR lets task maintenance recover terminal `lost` cron rows with a `backing session missing` error from durable cron run logs and clears stale terminal errors when the durable recovery has no error.

## Best Possible Solution

Land the narrowed recovery after task/session-state owner acceptance, or require live gateway cron proof first if seeded CLI proof is not enough for this persisted-state rewrite.

## Reproduction Assessment

Yes. Current main's durable cron recovery guard only accepts active cron tasks, so a terminal `lost` cron row with matching durable history is not recovered; the PR body also supplies a seeded real CLI maintenance proof after the patch.

## Solution Assessment

Yes, subject to owner acceptance. The patch keeps recovery scoped to cron rows marked lost by the backing-session-missing path; pruning would lose durable completion truth, and the adjacent startup-liveness and interrupted-restart work covers different cron failure modes.
