## Summary

Keep open. Current main still preserves explicit model selection and thinking/reasoning overrides across `/new`, `/reset`, and Gateway `sessions.reset`, so the requested recovery behavior is not implemented and would change intentional reset semantics.

## What This Changes

The issue asks for a recovery path where `/new` resets an Opus/Kimi-style reasoning loop without requiring a model swap or manually clearing thinking state.

## Best Possible Solution

Add an explicit recovery reset that aborts active work and can clear selected model plus thinking/reasoning overrides, or document the official `/think off`/model-switch workflow if reset should keep preserving preferences.

## Reproduction Assessment

Unclear for the live Opus/Kimi provider loop because I did not execute a real provider/channel run. Source and tests do reproduce the OpenClaw reset semantics: `/new`, `/reset`, and `sessions.reset` preserve thinking/reasoning state and explicit model overrides.

## Solution Assessment

Unclear. Reusing bare `/new` to clear model and thinking state would contradict current tests and docs, so a separate recovery reset mode or documented `/think off` plus model-switch workflow is safer unless maintainers choose a compatibility change.
