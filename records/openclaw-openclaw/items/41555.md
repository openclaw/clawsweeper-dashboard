## Summary

Keep open. Current main still preserves the selected model plus thinking/reasoning session overrides across `/new`, `/reset`, and gateway `sessions.reset`, so the requested bare-reset recovery path for Opus/Kimi thinking loops is not implemented. The issue discussion has enough concrete environment, workaround, and cross-provider confirmation that missing reproduction detail is not the blocker.

## What This Changes

Request that `/new` reset a stuck model/thinking state enough to recover Opus or Kimi reasoning loops without requiring a model swap or manual thinking-mode change.

## Best Possible Solution

Add an explicit recovery/reset path that aborts active work and can clear model plus thinking/reasoning overrides back to defaults, or document the intended recovery sequence if maintainers want `/new` to keep preserving user preferences.

## Reproduction Assessment

Yes for the OpenClaw reset semantics: seed a session with a model and non-off thinking/reasoning override, then issue `/new`, `/reset`, or gateway `sessions.reset`; current code and tests show those values persist. The live provider reasoning loop itself remains unverified here.

## Solution Assessment

Unclear. A bare `/new` semantic change is not obviously the best fix because current preservation is intentional and tested; a separate recovery reset mode or explicit documented `/think off` plus model-switch path is safer unless maintainers decide to change compatibility semantics.
