## Summary

Keep open. Current main still intentionally preserves selected model plus thinking/reasoning session overrides across `/new`, `/reset`, and gateway `sessions.reset`, so the requested recovery path for Opus/Kimi thinking loops is not implemented and this is not stale enough or underspecified enough to close.

## What This Changes

The issue asks for `/new` or an equivalent reset path to recover from Opus/Kimi thinking loops without requiring a model swap.

## Best Possible Solution

Add an explicit recovery reset that aborts active work and can clear selected model plus thinking/reasoning overrides, or document the official recovery sequence if maintainers want `/new` to preserve preferences.

## Reproduction Assessment

Yes for the OpenClaw reset semantics: source and tests show `/new`, `/reset`, and `sessions.reset` preserve model/thinking/reasoning state. The live provider reasoning loop itself was not reproduced in this read-only review.

## Solution Assessment

Unclear. Changing bare `/new` would contradict current tests and docs, so a separate recovery reset mode or documented model-switch plus `/think:off` path is safer unless maintainers choose a compatibility change.
