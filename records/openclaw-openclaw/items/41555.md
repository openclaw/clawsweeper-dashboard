## Summary

Keep open. Current main still preserves explicit model selection plus thinking/reasoning overrides across `/new`, `/reset`, and Gateway `sessions.reset`, so the requested recovery behavior is not implemented and would change intentional reset semantics.

## What This Changes

The issue requests a recovery path where starting a new session resets a stuck reasoning/thinking loop without requiring a model switch.

## Best Possible Solution

Provide an explicit recovery reset that aborts active work and can clear model, thinking, and reasoning selections, or document the supported `/think off` plus model-switch recovery workflow if preference preservation remains policy.

## Reproduction Assessment

Unclear for the live Opus/Kimi loop because I did not run real provider/channel traffic. Current source and tests do show that `/new`, `/reset`, and Gateway `sessions.reset` preserve model, thinking, and reasoning state, which explains why the workaround remains necessary.

## Solution Assessment

Unclear. Reusing bare `/new` to clear model and thinking state is not clearly the best fix because it contradicts current tests and docs; a separate recovery reset or documented workflow is safer unless maintainers choose a compatibility change.
