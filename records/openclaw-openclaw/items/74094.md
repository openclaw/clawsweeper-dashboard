## Summary

Keep open. Current main still exposes raw task status/progress/terminal fields and several independent renderers, but it does not provide the requested shared compressed operational-summary contract with current state, last successful checkpoint, blocker, and recommended next action.

## What This Changes

Add a shared compact operational summary for task status and reuse it across task CLI output, task views, auto-reply status, and session-status surfaces.

## Best Possible Solution

Add an additive TaskOperationalSummary type and deterministic builder in the task subsystem, expose it through task-domain views, and make CLI, /tasks, /status, session_status, and task notification/status rendering consume that shared contract while retaining existing raw fields.
