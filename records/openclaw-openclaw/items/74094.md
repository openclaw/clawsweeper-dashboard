Current main still exposes raw task status, progress, terminal summary, and count fields through several independent renderers; it does not provide the requested shared compressed operational-summary contract with current state, last successful checkpoint, current blocker, and recommended next action.

## Best Possible Solution

Add an additive TaskOperationalSummary type and deterministic builder in the task subsystem, expose it through task-domain views, and make CLI list/show, /tasks, /status, session_status, and task notification/status rendering consume that shared contract while retaining existing raw fields.
