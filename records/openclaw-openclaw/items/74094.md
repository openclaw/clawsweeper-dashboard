## Summary

Keep open. Current main still exposes raw task status/progress fields and several surface-specific renderers, but it does not provide the requested shared compressed operational-summary contract with current state, last successful checkpoint, blocker, and next action.

Add an additive TaskOperationalSummary contract in the task subsystem, derive it deterministically from structured task state/events, expose it through task-domain views, and make CLI, chat, status, session_status, and notification renderers consume it while retaining existing raw fields.
