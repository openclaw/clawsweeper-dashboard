## What This Changes

The issue asks whether malformed tool-call argument repair should also cover `openai-codex-responses`, `openai-responses`, and `azure-openai-responses` instead of only Kimi Anthropic Messages and `openai-completions`.

## Best Possible Solution

Review PR #75281 and settle a narrow transport policy: enable the repair wrapper only for verified Responses-family paths that emit compatible `toolcall_delta` partial-JSON events, keep unverified WebSocket-capable paths excluded, and land focused positive/negative tests plus a changelog entry if behavior changes.
