The PR changes the Control UI Agents model option builder to use catalog display helpers, adds render tests for catalog/custom/duplicate labels, and adds a changelog entry.

## Best Possible Solution

Update the Agents option builder to derive a non-alias catalog base name for configured entries, append only truly custom aliases, and cover the real `models.list` configured catalog shape with alias fields.

## Reproduction Assessment

Yes. Source inspection shows current main renders configured aliases/raw ids first, and the gateway configured catalog path can carry those aliases into catalog rows, which matches the reported missing display-name behavior.

## Solution Assessment

No. Reusing the catalog display path is the right direction, but this implementation feeds alias-preferring catalog labels back into the configured option label and needs a name-first path for this case.
