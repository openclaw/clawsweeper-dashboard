Full review comments:

- **[P1] Require card-form mentions on card paths:** `extensions/feishu/src/outbound-mention.ts:240-241`
  - body: On card and streaming-card paths, `ensureMention(..., { card: true })` returns early when the text already contains the text-message form. The new normalizer also rewrites `<at id=...>` to `user_id`, and the prompt asks the model for `user_id`, so a card reply can still be sent without the lark_md `<at id=...></at>` tag that the peer bot needs to be notified. Convert existing text-form tags to card-form tags, or only treat card-form tags as satisfying a card-path mention.
  - confidence: 0.92
- **[P2] Normalize the real message-tool outbound path:** `extensions/feishu/src/channel.ts:782-785`
  - body: This normalization only runs in the channel action handler, but `messages.groupChat.visibleReplies: "message_tool"` sends through the plugin outbound adapter (`sendPayload`/`sendText`) instead. That leaves explicit `@Name`, `@ou_...`, or card-form mentions unnormalized on the message_tool-only path the PR says remains supported. Move the normalization to the shared Feishu outbound send path or cover both call sites.
  - confidence: 0.8
