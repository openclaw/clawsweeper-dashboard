## Summary

Keep this PR open for review: the dispatcher/automatic Feishu bot-to-bot path is useful and has redacted live proof, but the PR also documents that the supported `message_tool` visible-reply path still silently drops bot-to-bot replies when the model omits the required mention.

## What This Changes

The PR adds Feishu bot-sender detection, bot-name lookup, per-chat mention registry state, outbound mention normalization, dispatcher reply-routing changes, and Feishu tests for bot-to-bot `@mention` delivery.

## Best Possible Solution

Cover the bot-sender `@mention` guarantee in every Feishu visible-reply mode, or explicitly defer and track the `message_tool` mode as a separate maintainer-approved follow-up before calling this feature complete.

## Reproduction Assessment

Yes for the reported gap at source/proof level: the PR body gives live Feishu evidence that dispatcher replies now work and separately reproduces the message_tool silent-drop path. I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No: the dispatcher-layer fix is plausible, but it is too narrow while a documented visible-reply mode still misses the required bot mention. The best fix should either cover message_tool too or make that scope an explicit maintainer-approved follow-up.
