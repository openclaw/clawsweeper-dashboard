## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Mention every bot-to-bot chunk:** `extensions/feishu/src/reply-dispatcher.ts:657-659`
  - body: The new backstop adds `ensureMention` to the whole reply before `sendChunkedTextReply` splits it. When a Feishu bot-to-bot answer exceeds the text/card chunk limit, only the first emitted message contains the peer bot's `<at>` tag, and later Feishu messages can still be invisible to that bot. Apply the required mention per outbound chunk or reserve and reapply it after chunking.
  - confidence: 0.9
- **[P1] Add loop protection before admitting Feishu bots:** `extensions/feishu/src/bot.ts:665-668`
  - body: This path lets non-self Feishu bot messages continue through normal dispatch and the PR also backfills peer bot mentions, making sustained bot pairs possible. Sibling bot-enabled channels prepare shared `botLoopProtection` facts before dispatch, and the docs describe that guard as active when bot-authored messages can reach dispatch. Wire Feishu into that guard or add the maintainer-approved Feishu policy before enabling this by default.
  - confidence: 0.88
- **[P2] Keep mention-forward targets through debounce:** `extensions/feishu/src/monitor.message-handler.ts:135-146`
  - body: After the deletion here, a debounced burst merges mentions and then filters the batch down to bot mentions only. If one entry was an explicit `@bot @Alice` request, the dispatched event no longer carries Alice's mention, so `shouldExposeMentionTargets` cannot give the agent the stable open_id it needs. Preserve the full mention set from the relevant mention-forward entry before filtering ordinary bursts.
  - confidence: 0.86
- **[P2] Escape display names when building at-tags:** `extensions/feishu/src/outbound-mention.ts:30-32`
  - body: `buildTag` interpolates display names from Feishu APIs, the registry, and model-authored `@Name` text directly into the `<at>` body, unlike the existing inbound normalizer that escapes tag delimiters. A name containing `<` or `>` can produce malformed Feishu mention markup or visible extra text. Escape the display text before returning the tag.
  - confidence: 0.75
