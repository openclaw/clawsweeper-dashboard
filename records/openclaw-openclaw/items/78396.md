## Summary

Keep open: current `origin/main` and `v2026.5.5` still contain the forced Matrix bootstrap path that can recreate secret storage and retry `setupNewCrossSigning` after a repairable SSSS error; no merged PR currently closes this issue.

## What This Changes

The issue requests preventing `openclaw matrix verify bootstrap --force-reset-cross-signing` from performing a second forced cross-signing reset after secret-storage repair.

## Best Possible Solution

Update the Matrix plugin bootstrapper so a forced reset performs at most one destructive `setupNewCrossSigning` reset, repairs or re-exports secret storage coherently, and preserves the no-partial-SSSS-mutation UIA guard.

## Reproduction Assessment

Yes for source-level reproduction: current main has a deterministic path where a repairable secret-storage error during forced reset recreates SSSS and calls `setupNewCrossSigning` again. I did not establish a live homeserver reproduction in this read-only review.

## Solution Assessment

Unclear whether the exact patch in the issue is the safest fix because current code deliberately avoids secret-storage mutation before UIA succeeds. The maintainable fix should prevent repeated destructive resets while preserving that partial-mutation guard.
