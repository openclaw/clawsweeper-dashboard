## What This Changes

Fix Matrix `verify bootstrap --force-reset-cross-signing` so one invocation cannot perform repeated destructive cross-signing resets or leave recovery material inconsistent after stale secret-storage repair.

## Best Possible Solution

Patch the Matrix bootstrapper so one forced bootstrap performs at most one destructive cross-signing reset while keeping recovery-key staging, server secret storage, backup creation, and UIA no-partial-mutation safeguards coherent.
