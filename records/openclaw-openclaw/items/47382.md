## Summary

Keep open. Current main still lacks the requested configurable gateway/channel silence watchdog keyed to operator-visible delivery; adjacent run-state telemetry, health-monitor watchdogs, streaming progress, and outbound diagnostics do not implement the threshold/reset/repeat/max-beacon contract, and the broader same-author #47591 remains open for coordination.

## What This Changes

Request a configurable gateway/channel watchdog that sends periodic operator-visible status beacons during active work after visible-output silence.

## Best Possible Solution

Define a generic, documented runtime/channel watchdog seam tied to confirmed operator-visible delivery timestamps, with per-channel config/schema/docs and focused coverage for streaming, final delivery, cancellation, repeat, and max-beacon behavior.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The high-confidence check is source and docs inspection on current main, which shows adjacent liveness mechanisms exist but the requested watchdog contract is absent.

## Solution Assessment

Unclear until maintainer approval. The maintainable direction is a shared channel/runtime seam keyed to actual visible delivery, not prompt rules or per-channel ad hoc timers.
