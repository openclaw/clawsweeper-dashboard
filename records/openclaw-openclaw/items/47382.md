## Summary

Keep open. Current main still lacks the requested configurable gateway/channel silence watchdog keyed to operator-visible delivery, and the broader same-author #47591 remains open for coordinated runtime visibility and delivery-confirmation design.

## What This Changes

The issue requests a configurable gateway/channel silence watchdog that sends runtime beacons during active work after a channel has had no operator-visible output for a configured interval.

## Best Possible Solution

Define a shared, documented runtime/channel watchdog contract tied to confirmed operator-visible delivery timestamps, with per-channel config/schema/docs and focused coverage for streaming, final delivery, cancellation, repeat, and max-beacon behavior coordinated with #47591.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The high-confidence check is source and docs inspection on current main, which shows adjacent liveness and progress mechanisms exist but the requested watchdog contract is absent.

## Solution Assessment

Unclear until maintainer approval. The maintainable direction is a shared channel/runtime seam keyed to actual visible delivery, not model prompt rules or per-channel ad hoc timers.
