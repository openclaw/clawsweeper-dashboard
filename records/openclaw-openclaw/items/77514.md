## Summary

Close as a config/policy support case: current `main` intentionally suppresses normal final replies in group/channel rooms unless automatic visible replies are configured, and the reported message-tool workaround is the documented visible-output path.

## What This Changes

The issue requests that automatic Telegram forum-topic agent replies be posted back to the originating topic without requiring explicit `message` tool use.

## Best Possible Solution

Keep the current documented group/channel visible-reply policy; operators who want legacy topic auto-posting should set `messages.groupChat.visibleReplies: "automatic"`, and a new bug should include that config if delivery still fails.

## Reproduction Assessment

No. The visible no-send symptom is source-explained by current `message_tool_only` group policy; a bug reproduction would need config showing `messages.groupChat.visibleReplies: "automatic"` plus logs showing delivery still skipped.

## Solution Assessment

Not applicable as a code fix. The narrow maintainable path is to use the existing visible-replies config or file a new, config-backed bug if automatic mode fails.
