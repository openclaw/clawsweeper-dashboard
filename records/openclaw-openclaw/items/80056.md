## Best Possible Solution

Keep this in maintainer-reviewed draft state, scope tool metadata lint to explicit tools settings or plugin config opt-in, and land it only after the base doctor/channels policy shape is accepted.

Yes for the review finding by source inspection: a channels-only policy makes policyAreaExists true, which enables requireRisk/requireSensitivity defaults and can produce TOOLS.md findings without tools.settings opt-in. I did not run tests because this review is read-only.

## Solution Assessment
