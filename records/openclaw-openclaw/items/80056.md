## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Gate tool checks on tool settings only:** `extensions/policy/src/doctor/register.ts:571-572`
  - body: `policyRuleEnabledFromSnapshot` defaults every setting to enabled whenever the policy has either a `channels` or `tools` section. That means a channel-only policy with a normal `TOOLS.md` can suddenly fail on missing risk/sensitivity metadata even though the docs make `requireRisk` and `requireSensitivity` separate tool settings. Scope the default to the matching policy area, or require the tool booleans explicitly.
  - confidence: 0.88
- **[P2] Report invalid policy files as findings:** `extensions/policy/src/doctor/register.ts:195-196`
  - body: When `parsePolicyFile` returns error diagnostics, the evaluator returns with no finding and no policy hash. As a result, `policy check --json` can emit `ok: true` and a clean attestation for a malformed `policy.jsonc`, which defeats the conformance gate. Add a policy parse/error finding and make the check fail closed before returning.
  - confidence: 0.84
