## Summary

Keep open: this protected maintainer-labeled draft is useful policy-stack work, but the current branch has merge-blocking approval metadata and config-contract defects that need contributor and maintainer follow-up.

## What This Changes

The PR adds a bundled policy plugin with doctor lint integration, oc-path document helpers, tool metadata evidence, and opt-in runtime trusted-tool policy enforcement.

## Best Possible Solution

Keep the policy-stack direction under maintainer review, but either fully thread approval metadata through the gateway protocol, storage, list/resolve, and display contracts in this slice or defer it, and align the manifest config surface with tools.requireMetadata.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR. The blocking issues are source-reproducible by comparing the PR's new approval metadata field to current main's gateway approval schema/handler and the manifest config keys to the evaluator contract.

## Solution Assessment

No for the current patch. The policy-as-health approach may be viable, but approval metadata and the policy config contract need to be enforceable and documented consistently before merge.
