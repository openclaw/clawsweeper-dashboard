## What This Changes

The PR adds a bundled policy plugin with `openclaw policy` generate/check commands, doctor lint/fix checks, oc-path API usage, docs/tests, and a trusted-tool runtime policy guard.

## Best Possible Solution

Keep this as maintainer-reviewed security/product work, add a bounded policy-health registration path for doctor lint/fix, harden runtime policy schema validation, and require real repair/runtime proof before merge.

## Reproduction Assessment

Yes for the review findings by source inspection: doctor disables plugin loading while lint/fix register only core checks, and malformed deny-rule entries can reach `matchDenyRule`. I did not run tests because this is a read-only review.

## Solution Assessment

No. The direction may be viable after maintainer approval, but this patch is not mergeable until doctor reaches policy health checks through a narrow registration path and runtime validation rejects invalid deny-rule schema before matching.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Load policy checks in doctor lint and fix:** `src/cli/command-catalog.ts:265`
  - body: The docs and plugin manifest say policy contributes to `openclaw doctor --lint` and `doctor --fix`, but `doctor` now disables plugin loading and the lint/repair paths register only core checks. A normal doctor invocation therefore never reaches `registerPolicyDoctorChecks()`, so the advertised policy drift/risk checks and repairs are absent; add a narrow policy-health registration route without broad arbitrary plugin preload.
  - confidence: 0.91
- **[P2] Validate deny-rule entries before matching:** `extensions/policy/src/runtime.ts:342-343`
  - body: `validatePolicySnapshot` verifies only that deny-rule containers are arrays, so values like `denyRules: [null]` or `{ "when": null }` can reach `matchDenyRule` and throw instead of returning the documented fail-closed invalid-policy decision. Validate each rule object and `when` shape before evaluating policy rules.
  - confidence: 0.86
