## What This Changes

Adds a bundled policy plugin with policy generate/check commands, doctor lint/fix health checks, policy.jsonc runtime tool-call guardrails, an oc-path document API seam, docs, tests, label routing, and a new plugin SDK health subpath.

## Best Possible Solution

Keep the PR open, fix the enabled=false policy-check contract, add redacted real proof for doctor --fix and a real runtime guard decision, then let maintainers decide the SDK health and oc-path seam direction.

## Reproduction Assessment

Yes for the review finding by source inspection: policyRuleEnabled can read policy.jsonc per-rule settings and fall back to file existence before respecting a global enabled=false setting, while the new docs describe that setting as a disable switch. This is not a current-main bug because the policy plugin is introduced by this PR.

## Solution Assessment

No. The PR should either make plugins.entries.policy.config.enabled=false a terminal disable for policy doctor checks or change the docs/schema wording so the setting is not advertised as a disable switch.
