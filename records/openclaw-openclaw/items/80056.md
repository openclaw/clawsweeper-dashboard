## Summary

Keep open: this is an active draft with the protected `maintainer` label, and the latest PR head still has a blocking conformance bug where an invalid `policy.jsonc` can produce a clean policy check instead of a finding.

## What This Changes

The PR adds a bundled policy plugin with tool metadata conformance checks, doctor lint integration, oc-path document APIs, and an optional trusted-tool runtime gate.

## Best Possible Solution

Land a plugin-owned policy layer only after invalid policy files fail closed in lint/check output and maintainers accept the doctor-health and runtime-policy boundaries.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows invalid JSONC diagnostics return from `evaluatePolicyUncached` without adding a finding, and `policy check` computes `ok` from that empty finding set.

## Solution Assessment

No. The proposed implementation needs to report invalid policy syntax as an error finding before its conformance attestation can be trusted.
