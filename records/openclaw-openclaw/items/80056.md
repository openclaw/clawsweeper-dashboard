## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Load policy checks after reading the lint config snapshot:** `src/cli/command-catalog.ts:265`
  - body: `doctor --lint` is adding structured invalid-config findings in `runDoctorLintCli`, but this startup policy loads plugins before the command action runs. Plugin registry loading resolves `getRuntimeConfig()`, which fails closed on invalid configs, so `openclaw doctor --lint --json` can throw before it emits the config finding it is supposed to report. Move plugin health-check loading into the lint command after the snapshot is known valid, or otherwise avoid preloading plugins for invalid configs.
  - confidence: 0.9
- **[P2] Classify and document the oc-path SDK subpath:** `scripts/lib/plugin-sdk-entrypoints.json:12`
  - body: Adding `oc-path` here creates a plugin-owned SDK subpath because `extensions/oc-path` exists, but the PR does not classify it in `src/plugin-sdk/entrypoints.ts` or document it in the SDK subpath docs. The package contract explicitly checks plugin-owned SDK subpaths for classification and documentation, so this will fail that guard and leaves the new public API boundary unaudited.
  - confidence: 0.88
- **[P3] Add label routing for the new policy plugin:** `extensions/policy/openclaw.plugin.json:2`
  - body: This PR adds a new bundled `policy` plugin, but `.github/labeler.yml` has no `extensions: policy` entry. Repo policy requires new plugin surfaces to update labeler/GitHub labels, otherwise future policy changes will not route to the right review surface.
  - confidence: 0.82
