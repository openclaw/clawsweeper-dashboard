## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Report bare tool headings as missing policy metadata:** `extensions/policy/src/doctor/register.ts:68`
  - body: When `TOOLS.md` contains a bare heading like `### deploy`, `scanToolHeaders` still treats it as a tool and `policy generate` emits an entry, but this `tool.meta !== ""` guard skips the missing-risk finding; the missing-sensitivity check has the same pattern. With `requireRisk` and `requireSensitivity` enabled, `policy check` can pass a tool that then reaches runtime with no risk or sensitivity and falls through to allow. Drop the metadata-nonempty guard and cover a bare heading so required policy metadata fails closed.
  - confidence: 0.94
