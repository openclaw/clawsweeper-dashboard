## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread metadata through plugin approvals:** `src/agents/pi-tools.before-tool-call.ts:193`
  - body: This adds `metadata` to `plugin.approval.request`, but the gateway request schema still has `additionalProperties: false` with no `metadata` field, and the server payload/list/resolve path does not store it. Policy approvals with metadata will fail validation or lose the audit data; add metadata to the protocol schema, payload persistence/replay/display tests, or stop sending it here.
  - confidence: 0.94
- **[P2] Remove stale metadata config toggles:** `extensions/policy/openclaw.plugin.json:23-29`
  - body: The manifest still accepts `requireRisk` and `requireSensitivity` as policy plugin config, while the checks and docs read authored requirements from `policy.jsonc` via `tools.requireMetadata`. Operators can set valid-looking keys and get no metadata enforcement, so remove these keys or intentionally implement them in the evaluator and docs.
  - confidence: 0.9
