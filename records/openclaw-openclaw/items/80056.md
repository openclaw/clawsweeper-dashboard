## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Load policy checks in doctor lint and fix:** `src/cli/command-catalog.ts:265`
  - body: The docs and plugin manifest say policy contributes to `openclaw doctor --lint` and `doctor --fix`, but the doctor command now sets `loadPlugins: "never"` and the lint/repair paths only register core checks. A normal doctor invocation therefore never runs `registerPolicyDoctorChecks()`, so the advertised policy drift/risk checks and repairs are absent; add a narrow health-contributor path or another bounded registration route without broad arbitrary plugin preload.
  - confidence: 0.91
- **[P2] Validate deny-rule entries before matching:** `extensions/policy/src/runtime.ts:342-343`
  - body: `validatePolicySnapshot` only verifies that deny-rule containers are arrays, so values like `denyRules: [null]` or `{ "when": null }` can reach `matchDenyRule` and throw instead of returning the documented fail-closed invalid-policy decision. Validate each rule object and `when` shape before evaluating policy rules.
  - confidence: 0.86
