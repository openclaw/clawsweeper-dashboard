## Summary

This PR should stay open. It is draft, protected by the maintainer label, and the latest head still has a runtime policy enablement bug that can silently skip the trusted-tool gate for the documented plugin-enabled setup.

## What This Changes

The PR adds a bundled policy plugin with tool metadata conformance checks, doctor lint integration, oc-path document APIs, policy attestations, and an optional trusted-tool runtime gate.

## Best Possible Solution

Align runtime activation with the same plugin-enabled contract used by health registration, add a regression test for plugin-enabled plus runtimeToolPolicy, then continue maintainer architecture and security review.

## Reproduction Assessment

Yes, source-reproducible. With plugins.entries.policy.enabled=true and plugins.entries.policy.config.runtimeToolPolicy=true but no config.enabled, the runtime evaluator returns before loading policy even though docs and health registration treat the plugin as enabled.

## Solution Assessment

No. The trusted-tool hook is a reasonable narrow runtime seam, but this implementation must honor the established plugin enablement contract before it can be treated as an opt-in enforcement gate.

## Review Findings

Overall correctness: patch is incorrect
