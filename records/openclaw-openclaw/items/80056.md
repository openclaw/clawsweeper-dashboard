## What This Changes

The PR adds a bundled policy plugin, doctor lint/health integration, oc-path document API exports, tool metadata checks, docs, tests, and an opt-in trusted-tool runtime gate.

## Best Possible Solution

Keep this in the maintainer-reviewed policy stack, wire approval metadata through the actual gateway protocol/storage/display contract or stop sending it, remove the stale manifest config keys, and refresh real-behavior proof.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR. The blocking review issues are source-reproducible from PR head: metadata is sent to a schema/payload contract that does not accept it, and manifest config keys are not read by the evaluator.

## Solution Assessment

No for the current patch. The policy-as-health direction may be viable, but the runtime approval metadata and config contract must be made enforceable before this is a maintainable implementation.
