## Summary

Keep open: this is a protected draft PR for a new security-sensitive policy surface, and source inspection found a blocking drift-repair bug that can mark stale generated tool policy as current. The submitted proof covers CLI generation/check/lint behavior, but not the runtime guard in a real agent/tool flow.

## What This Changes

Adds a bundled policy plugin with `openclaw policy` generation/check commands, doctor lint/fix health checks, a `plugin-sdk/health` subpath, oc-path document API exports, runtime tool-policy guardrails, docs, tests, and label routing.

## Best Possible Solution

Keep the PR open, make drift repair regenerate `tools.entries` with the hash or refuse repair until `openclaw policy generate --write` runs, add regression coverage, provide runtime guard proof, and then have maintainers decide the new contract seams.

## Reproduction Assessment

Yes for the review finding: source inspection shows `doctor --fix` updates only the generated hash while generated entries and runtime decisions are separate. This is not a current-main bug reproduction because the policy plugin is new in this PR.

## Solution Assessment

No. The direction may be useful, but the current patch should not bless drift without regenerating or validating generated entries, and the runtime behavior still needs real proof plus maintainer approval for the new seams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Regenerate policy entries before accepting drift repair:** `extensions/policy/src/doctor/register.ts:208-211`
  - body: `doctor --fix` currently marks a drifted policy as current by updating only `tools.generatedFrom.hash`. If `TOOLS.md` changes an existing tool from low/public to critical/restricted, the stale `tools.entries` remains but future drift checks pass, so runtime can keep allowing the old lower-risk entry. Either regenerate `tools.entries` with the hash or refuse this repair and require `openclaw policy generate --write`.
  - confidence: 0.91
