## Summary

Keep this open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config; the backend already exposes the path to admin-scoped sessions, but the UI only carries it into the Config view as an Open button tooltip.

## What This Changes

The issue requests showing the active OpenClaw config file path prominently in Control UI Overview or Settings to help identify which running instance is in use.

## Best Possible Solution

Add a visible, copyable active config file path to the Control UI Overview snapshot card and/or Settings/Config header, sourced from the existing admin-scoped path data, with focused UI tests and gateway boundary tests kept green.
