## Summary

Keep open: current main and v2026.5.22 still do not visibly show the active config file path in Control UI Overview or Settings. The path already reaches the authenticated config snapshot flow, and the related implementation PR was closed unmerged after adding a broken new gateway helper.

## What This Changes

Show the active OpenClaw config file path prominently in Control UI Overview or Settings so users can tell which instance is running.

## Best Possible Solution

Add a visible, copyable active config path row or compact card in authenticated Control UI Overview and/or Settings using the existing configSnapshot.path/config.get snapshot flow.

## Reproduction Assessment

Yes, by source inspection. Current main carries the active path in configSnapshot.path and passes it to Settings, but Overview has no configPath prop/rendering and Settings only exposes it as an Open button tooltip.

## Solution Assessment

Yes. The narrow maintainable path is a UI-only visible/copy affordance backed by the existing authenticated config snapshot; the closed related PR's new gateway overview resolver is unnecessary.
