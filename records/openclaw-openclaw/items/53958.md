The issue requests a visible active config file path in the Control Overview or Settings UI to make setup troubleshooting easier.

## Best Possible Solution

Add a visible, copyable active config path row in the existing Control UI using state.configSnapshot?.path or equivalent admin-scoped snapshot data, without adding a new gateway overview API.

## Reproduction Assessment

Yes, source-level. Current main exposes configSnapshot.path to the Settings view and admin-scoped gateway snapshots, but Overview has no config-path prop/rendering and Settings only puts the path in an Open button tooltip.

## Solution Assessment

Yes. The narrowest maintainable solution is UI-only visible/copy rendering from existing admin-scoped config data; the closed related PR’s new gateway overview API path was unnecessary and had correctness problems.
