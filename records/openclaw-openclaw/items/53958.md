## Summary

Keep open: current main and the latest release still do not visibly render the active config file path in Overview or Settings. The data is already available through the config snapshot path, but the UI only uses it as an Open button tooltip, and the directly related implementation PR was closed unmerged.

## What This Changes

The issue requests a visible Control UI affordance showing the active OpenClaw config file path to help users identify which instance is running.

## Best Possible Solution

Add a visible, copyable active config path row or compact status card in Overview and/or Settings using the existing config snapshot path, with focused render coverage and no new backend API.

## Reproduction Assessment

Yes, source-level. Current main exposes the active config path through ConfigSnapshot.path and passes it into Settings, but Overview has no configPath prop/rendering and Settings only places the path in an Open button tooltip.

## Solution Assessment

Yes. The narrow maintainable path is a UI-only visible/copy affordance backed by state.configSnapshot?.path; the closed related PR’s new gateway overview helper is unnecessary.
