## Summary

Keep this open: current main still has the active config path available behind the admin-scoped gateway boundary, but the Control UI Overview does not receive or render it, and Settings/Config only exposes it indirectly through the Open button title.

## What This Changes

Show the active OpenClaw config file path prominently in the Control UI Overview or Settings area so users can tell which instance is running.

## Best Possible Solution

Add a visible, copyable active config path to the existing admin-scoped Control UI surface, preferably in the Overview snapshot card and/or Config header, without adding a backend endpoint or widening path exposure.

## Reproduction Assessment

Yes. Source inspection on current main shows configPath is available from the admin-scoped hello snapshot and Config snapshot, but Overview does not render it and Config only exposes it as a button tooltip.

## Solution Assessment

Yes. The narrowest maintainable solution is a UI-only display/copy affordance using the existing admin-scoped fields while keeping the gateway sensitivity tests intact.
