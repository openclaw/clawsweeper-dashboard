## Summary

Keep this issue open. Current `main` still exposes the active config path from the gateway and passes it to the Config view, but the Overview view does not receive or render it, and Settings/Config only hides the path in the Open button tooltip rather than showing it prominently.

## What This Changes

Show the active OpenClaw config file path prominently in Control UI Overview or Settings/Config so operators can tell which instance is running.

## Best Possible Solution

Add a visible, copyable active config path to the Control UI Overview snapshot card and/or Settings/Config header, sourced from the existing admin-scoped hello snapshot or `config.get` snapshot path, with focused render tests and no backend exposure widening.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: `hello.snapshot.configPath` and `configSnapshot.path` exist, but `renderOverview` has no prop/render path for them and Config only uses the value in a button tooltip.

## Solution Assessment

Yes. The narrowest maintainable fix is a UI-only display/copy affordance using the existing gateway/config fields while preserving the admin-scoped path boundary; no new core API is needed.
