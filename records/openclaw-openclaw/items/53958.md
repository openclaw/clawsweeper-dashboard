## Summary

Keep this open: current main still has the active config path available behind existing admin-scoped gateway/config surfaces, but the Overview view does not receive or render it, and Settings/Config only exposes it as the Open button tooltip. The related source PR is closed and unmerged, so no landed implementation covers the request.

## What This Changes

The issue requests that the Control UI visibly show the active config file path so users can tell which OpenClaw instance/config is running.

## Best Possible Solution

Render a visible, copyable active config path in the existing admin-scoped Control UI, using current snapshot/config.get data and preserving the existing lower-scope redaction boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows configPath is available from admin-scoped gateway/config data, but Overview does not render it and Settings/Config only puts it in an Open button title.

## Solution Assessment

Yes. The narrowest maintainable solution is UI-only wiring/display/copy behavior using existing admin-scoped fields, rather than adding a new gateway endpoint like the closed source PR attempted.
