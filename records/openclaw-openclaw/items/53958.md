## Summary

Keep this open: current `main` still carries the active config path in admin-scoped gateway/config snapshots, but the Overview view does not receive or render it, and Settings/Config only exposes it indirectly through the Open button tooltip.

## What This Changes

The issue requests that the Control UI show the active OpenClaw config file path in Overview or Settings for instance troubleshooting.

## Best Possible Solution

Add a visible, copyable active config path to the existing admin-scoped Control UI surface, preferably the Overview snapshot card and/or Config header, without adding a new backend endpoint or widening path exposure.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the path is available from `hello.snapshot.configPath` and `configSnapshot.path`, but Overview does not render it and Config only hides it in a tooltip.

## Solution Assessment

Yes. The narrowest maintainable fix is UI-only display/copy behavior using the existing admin-scoped fields while keeping the current gateway sensitivity tests intact.
