## Summary

Keep open: current main still only has partial fallback observability, while non-verbose fallback notices, a fallback-transition plugin hook, and model-change-aware workspace context re-injection remain unimplemented and need maintainer API/product design.

## What This Changes

Requests a core fallback transition contract that can notify users outside verbose mode, re-inject workspace bootstrap context on model changes under continuation-skip, and expose a plugin hook.

## Best Possible Solution

Define one core fallback-transition contract: sanitized lifecycle/user notification policy, a documented plugin hook payload, and continuation-skip invalidation on model change, with docs and focused regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested non-verbose notification, fallback-transition hook, and model-change-aware bootstrap re-injection.

## Solution Assessment

Unclear until maintainer API design is done. The requested direction belongs in the core fallback/context/plugin contract rather than only in ClawHub, but the exact config name, default, hook payload, redaction, and re-injection trigger need product approval.
