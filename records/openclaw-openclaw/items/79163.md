## Summary

Keep open: current main still matches the three requested gaps, and the canon-guardian workaround in https://github.com/openclaw/openclaw/issues/75750 shows this is only partly achievable through today’s plugin hooks. The native notification policy, targeted bootstrap refresh, and fallback hook need a maintained API/UX contract before implementation.

## What This Changes

Request an opt-in non-verbose fallback notice, workspace bootstrap re-injection when `continuation-skip` sees a model change, and a plugin-observable fallback-transition hook.

## Best Possible Solution

Define one maintained fallback-transition contract that can drive optional user notification, targeted bootstrap refresh, and plugin observation without treating verbose diagnostics as public API.

## Reproduction Assessment

Yes. Current source and existing regression tests show non-verbose fallback notices are suppressed, `continuation-skip` has no model-change input, and the plugin hook surface lacks a fallback-transition event.

## Solution Assessment

Unclear. The requested three-part direction is plausible, but the user-visible notification policy, config surface, typed hook payload, and context-injection timing need maintainer design before a narrow implementation is safe.
