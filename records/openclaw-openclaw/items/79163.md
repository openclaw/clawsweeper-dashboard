## Summary

Keep open: current main still matches the reported gaps, and the ClawHub canon-guardian workaround in https://github.com/openclaw/openclaw/issues/75750 covers only part of the need through polling on `before_prompt_build`. Native fallback notification, model-change-aware bootstrap refresh, and a fallback-transition hook need maintainer API/UX decisions before cleanup closure.

## What This Changes

Request to add opt-in model fallback notification, targeted workspace context re-injection on fallback under `continuation-skip`, and a plugin hook for fallback transitions.

## Best Possible Solution

Define one maintained fallback-transition contract that can drive optional user notification, targeted bootstrap refresh, and plugin observation without turning verbose diagnostics into the public API.

## Reproduction Assessment

Yes. Source and test inspection on current main show the three reported gaps: non-verbose fallback notices are suppressed, `continuation-skip` has no model-change invalidation input, and the plugin hook surface lacks a fallback-transition event.

## Solution Assessment

Unclear. The requested direction is plausible, but the notification default, config naming, hook event/payload, and context re-injection ordering need maintainer product/API judgment before a narrow PR is safe.
