## Summary

Keep open: current main still has the three native gaps described here, while the ClawHub canon-guardian workaround and the open hook PR only cover parts of the problem. This needs maintainer product/API review rather than cleanup closure.

## What This Changes

The issue requests opt-in user notification, workspace context re-injection, and a plugin hook when model fallback changes the active model mid-session.

## Best Possible Solution

Define one native fallback-transition contract that preserves current defaults until product policy is set, exposes a documented plugin hook, and invalidates continuation-skip bootstrap context only when the active model actually changes.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main still lacks non-verbose fallback notices, fallback-aware context re-injection, and a public fallback hook.

## Solution Assessment

Unclear until maintainers settle the API and config contract. The existing fallback transition state and `before_prompt_build` system-context path are good building blocks, but a narrow automated fix would be premature without product approval.
