## Summary

Keep open: current main still matches the reported fallback-notification, context-refresh, and hook gaps. Related notification-only work does not cover the targeted workspace re-injection or a maintained fallback-transition hook contract.

## What This Changes

Request native user-visible model-fallback notification, model-change-triggered workspace context re-injection for `continuation-skip`, and a plugin-observable fallback transition hook.

## Best Possible Solution

Define one maintained fallback-transition contract that can drive safe notification, targeted bootstrap refresh, and plugin observation while coordinating with the narrower notification and fallback-context follow-ups.

## Reproduction Assessment

Yes. Current source and existing tests give a high-confidence source reproduction: fallback notices are suppressed when verbose is off, `continuation-skip` has no model-change input, and the plugin hook list lacks a fallback-transition event.

## Solution Assessment

Unclear. The three-part direction is plausible, but the config key/default, hook payload, refresh timing, and prompt ordering need maintainer API/UX judgment; notification PR https://github.com/openclaw/openclaw/pull/69399 solves only one slice.
