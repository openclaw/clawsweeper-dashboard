## Summary

Keep open: current main still matches the reported gaps, and the canon-guardian workaround plus the open hook PR only cover part of the requested native behavior. This needs maintainer API and UX decisions before a focused implementation is safe.

## What This Changes

Request user-visible model fallback notifications, model-change-aware workspace context re-injection, and a plugin fallback hook for mid-session model fallback.

## Best Possible Solution

Define one maintained fallback-transition contract that drives optional user notification, targeted bootstrap refresh, and plugin observation without treating verbose diagnostics as the public API.

## Reproduction Assessment

Yes. Source inspection and existing tests show non-verbose fallback notices are suppressed, continuation-skip ignores model-change state, and the current plugin hook list has no fallback-transition event.

## Solution Assessment

Unclear. The three-part direction is plausible, but notification defaults, config naming, hook payload/timing, and bootstrap re-injection ordering need maintainer product/API judgment first.
