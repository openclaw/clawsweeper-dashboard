## Summary

Keep open: current main still matches the reported fallback visibility, context re-injection, and hook-surface gaps. The canon-guardian plugin shared at https://github.com/openclaw/openclaw/issues/75750 is useful workaround evidence, but it does not provide a native same-transition notice, targeted bootstrap refresh, or typed fallback hook.

## What This Changes

The request asks for opt-in/non-verbose fallback notification, fallback-aware workspace context re-injection, and a plugin fallback-transition event.

## Best Possible Solution

Define one native fallback-transition contract that can drive optional user notice, targeted bootstrap refresh on model switch, and plugin observation, then document and test the chosen API.

## Reproduction Assessment

Yes. Source and existing regression tests show non-verbose fallback notices are suppressed, continuation-skip ignores model-change state, and the plugin hook list has no fallback-transition event.

## Solution Assessment

Unclear. The three-part direction is plausible, but config naming/defaults, hook payload, notification delivery, and re-injection ordering need maintainer product/API judgment before implementation is safe.
