## Summary

Keep open: current main still matches the three reported gaps, while the published canon-guardian plugin in https://github.com/openclaw/openclaw/issues/75750 is a useful workaround rather than a native fallback-transition contract. The notification subset is separately represented by https://github.com/openclaw/openclaw/issues/67197 and open PR https://github.com/openclaw/openclaw/pull/69399, but no related item fully supersedes the requested notification, context-refresh, and plugin-hook API work.

## What This Changes

The issue requests a maintained fallback-transition contract that can notify users, refresh workspace bootstrap context on fallback, and expose a plugin hook event.

## Best Possible Solution

Define one maintained fallback-transition contract that preserves default quiet behavior unless explicitly chosen, refreshes bootstrap context only when the selected runtime model changes under continuation-skip, and exposes a typed plugin event with documented ordering and payload semantics.

## Reproduction Assessment

Yes. Static current-main source plus existing mocked e2e/helper tests show verbose-off fallback notices are suppressed, continuation-skip has no model-change input, and the hook catalog lacks a fallback-transition event.

## Solution Assessment

Unclear. The three-part direction is plausible, but maintainers should choose the config key/default, hook name and payload, and context-refresh ordering before implementation.
