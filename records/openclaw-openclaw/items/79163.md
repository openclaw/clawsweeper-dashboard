## Summary

Keep open: current main still has the reported user-facing gaps, and the active hook PR only covers one subset. The canon-guardian workaround in https://github.com/openclaw/openclaw/issues/75750 is useful evidence that `before_prompt_build` can mitigate context drift, but it does not provide native fallback notification or a maintained fallback-transition contract.

## What This Changes

Request opt-in user-visible model fallback notification, model-change-aware workspace bootstrap re-injection for `continuation-skip`, and a plugin hook event for fallback transitions.

## Best Possible Solution

Define one maintained core fallback-transition contract that can drive optional user notification, targeted bootstrap refresh, and plugin observation without treating verbose diagnostics as the public API.

## Reproduction Assessment

Yes. Source inspection and existing regression tests show the current behavior: non-verbose fallback notices are suppressed, `continuation-skip` has no model-change exception, and the current hook catalog has no fallback-transition event.

## Solution Assessment

Unclear. The three requested directions are plausible, but config naming/defaults, hook payload/permissions, and bootstrap re-injection ordering need maintainer API and UX decisions; https://github.com/openclaw/openclaw/pull/70990 is only a partial hook-path candidate.
