## Summary

This should stay open: current main still has the reported source-level gaps, but the request is a feature/API design item rather than a narrow bug fix. The canon-guardian plugin context shows Gap B is partly workable through existing hooks, while Gap A and Gap C still need a native notification and/or hook contract decision.

## What This Changes

Request an opt-in non-verbose model-fallback notice, workspace bootstrap re-injection after fallback model changes, and a plugin hook event for fallback transitions.

## Best Possible Solution

Define one fallback-transition contract that can drive optional user notification, targeted bootstrap refresh, and plugin observation without turning verbose diagnostics into a public API.

## Reproduction Assessment

Yes. Source and regression tests on current main show default non-verbose replies suppress fallback notices, and continuation-skip skips bootstrap after a completed assistant turn without considering model changes.

## Solution Assessment

Unclear. The requested direction is plausible, but a new config key plus a fallback hook/event needs maintainer API and UX judgment before implementation.
