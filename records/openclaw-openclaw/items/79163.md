## Summary

Keep open: current main still matches the reported gaps, and the useful canon-guardian workaround does not replace a native fallback-transition contract for notification, targeted bootstrap refresh, and plugin observation.

## What This Changes

Requests native fallback-transition handling that can notify users, re-inject workspace bootstrap context on model changes under `continuation-skip`, and expose a plugin hook.

## Best Possible Solution

Define one native fallback-transition contract that drives optional user notice, targeted bootstrap refresh on model switch, and plugin observation, then document and test the chosen API.

## Reproduction Assessment

Yes for the reported current behavior, though this is a feature request rather than a strict bug repro. Source and existing tests show non-verbose fallback notices are suppressed, `continuation-skip` ignores model-change state, and the plugin hook list has no fallback-transition event.

## Solution Assessment

Unclear until maintainers choose the API and UX contract. The three-part direction is plausible, but config naming/defaults, hook payload, notification delivery, and re-injection ordering need product/API judgment before a narrow code change is safe.
