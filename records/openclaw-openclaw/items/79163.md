## Summary

Keep open. Current main and v2026.6.6 solved the non-verbose fallback-notice part via the shipped fallback-transition notice work, but the native continuation-skip re-injection and plugin fallback-transition hook remain missing and need maintainer API/product decisions.

## What This Changes

Request native model-fallback transparency, workspace bootstrap re-injection on fallback, and a plugin/config surface for fallback transitions.

## Best Possible Solution

Define one fallback-transition contract: keep the shipped notice behavior or adjust it through the visibility-policy issue, add a narrow fallback lifecycle hook if plugins need the transition event, and decide whether continuation-skip re-injects bootstrap on fallback automatically or through a scoped context profile.
