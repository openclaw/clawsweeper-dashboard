## Summary

Keep open: current main has useful fallback lifecycle/log observability, but the requested native user notification, fallback-aware workspace context re-injection, and plugin hook/API contract are still missing or deliberately policy-gated.

## What This Changes

Request native fallback transition awareness: optional user-visible notification, workspace bootstrap re-injection when the active model changes under continuation-skip, and a plugin hook for fallback events.

## Best Possible Solution

Define a single fallback-transition contract that drives opt-in or policy-approved user notices, invalidates continuation-skip only when the active model changes, and exposes a documented plugin hook without forcing users to poll prompt-build context.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested non-verbose notice, model-change bootstrap invalidation, and plugin fallback hook.

## Solution Assessment

Unclear until maintainers settle the product/API contract. The proposed direction is plausible, but the maintainable fix should coordinate notification policy, context-injection semantics, and hook payload shape rather than patching each gap independently.
