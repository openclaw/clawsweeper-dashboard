## Summary

Keep open. Current main has solved the non-verbose fallback notice gap, but source still skips workspace bootstrap after completed continuation turns without model-change invalidation and exposes no public fallback-transition plugin hook, so the central remaining work needs maintainer API/product decisions.

## What This Changes

The issue requests user-visible model fallback notices, workspace context re-injection on fallback when continuation-skip is active, and a plugin fallback-transition hook.

## Best Possible Solution

Keep this open for one fallback-transition contract: retain or adjust the shipped notice behavior, add a narrow public fallback hook if maintainers want plugins to react, and decide whether continuation-skip re-injects bootstrap on fallback automatically or through an explicit option.

## Reproduction Assessment

Yes, at source level. Current continuation-skip skips bootstrap after a completed turn without model-change input, and the plugin hook list has no fallback-transition event; I did not force a live provider outage.

## Solution Assessment

No final implementation path is selected yet. The reported outcomes are valid, but maintainers still need to choose the hook payload, notification policy, and automatic versus opt-in re-injection semantics before a focused PR is safe.
