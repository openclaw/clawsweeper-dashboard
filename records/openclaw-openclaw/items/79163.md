## Summary

Keep open: current main still only has verbose-gated fallback notices plus lifecycle/log observability, while the requested non-verbose notice/config, fallback-transition plugin hook, and model-change-aware workspace context re-injection remain unimplemented. The published canon-guardian plugin is useful workaround evidence, but the reporter confirms it is a polling `before_prompt_build` workaround and cannot react to the fallback transition in the same turn.

## What This Changes

Request user-visible fallback notification, workspace context re-injection when a fallback changes the active model, and a plugin hook for fallback transitions.

## Best Possible Solution

Define one core fallback-transition contract covering sanitized notification policy, a documented plugin hook payload, and `contextInjection: "continuation-skip"` invalidation on model change, coordinated with the open narrower fallback-notice PR and notification issues.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested non-verbose fallback notice option, fallback-transition hook, and model-change-aware bootstrap re-injection.

## Solution Assessment

Unclear until maintainer API design is done. The requested direction belongs in the core fallback/context/plugin contract, but the exact config name, default, hook payload, and re-injection semantics need product approval.
