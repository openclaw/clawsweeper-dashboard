## Summary

Keep open: current main only has partial fallback visibility, while the requested non-verbose fallback notice, fallback-transition hook, and model-change-aware context re-injection remain unimplemented and need maintainer API/product design.

## What This Changes

Request fallback-transition notifications, workspace bootstrap re-injection when the active model changes, and a plugin hook for model fallback transitions.

## Best Possible Solution

Define one fallback-transition contract in core: sanitized lifecycle/user notification policy, a documented plugin hook payload, and contextInjection="continuation-skip" invalidation on model change, with docs and focused regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection gives high confidence that current main lacks the requested non-verbose notification, fallback-transition hook, and model-change-aware bootstrap re-injection.

## Solution Assessment

Unclear until maintainer API design is done. The requested direction belongs in the core fallback/context/plugin contract rather than only in ClawHub, but the exact config name, default, payload, and re-injection trigger need product approval.
