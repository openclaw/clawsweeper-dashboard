## Summary

Keep open: current main still matches the reported fallback visibility, context re-injection, and hook-surface gaps. The linked canon-guardian plugin in https://github.com/openclaw/openclaw/issues/75750 is useful evidence and a partial workaround, but it does not provide a native fallback-transition event or same-transition user notification/context refresh, and the open notification-only follow-ups do not cover the full request.

## What This Changes

The issue requests opt-in user notification, targeted workspace context re-injection, and a plugin hook when model fallback changes the active model mid-session.

## Best Possible Solution

Define one native fallback-transition contract that can drive optional user notice, targeted bootstrap refresh on model switch, and plugin observation, then document and test the chosen API rather than making verbose diagnostics the public contract.

## Reproduction Assessment

Yes. Source and existing regression tests show the current behavior: non-verbose fallback notices are suppressed, continuation-skip ignores model-change state, and the plugin hook list has no fallback-transition event.

## Solution Assessment

Unclear. The requested three-part direction is plausible, but config naming/defaults, hook payload, notification delivery, and re-injection ordering need maintainer product/API judgment before a focused implementation is safe.
