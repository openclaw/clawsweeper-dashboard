## Summary

Keep open: current main still has the reported user-facing gaps, and the published canon-guardian workaround covers only part of the behavior through polling on an existing prompt hook. The remaining work is a core/API UX contract decision, not conservative cleanup closure.

## What This Changes

The issue requests opt-in user notification, targeted workspace-context re-injection, and a plugin hook when model fallback transitions occur.

## Best Possible Solution

Define one maintained fallback-transition contract that can drive optional user notification, targeted bootstrap refresh, and plugin observation without turning verbose diagnostics into the public API.

## Reproduction Assessment

Yes. Current source and focused regression tests show non-verbose fallback notices are suppressed, `continuation-skip` has no model-change state, and the plugin hook list lacks a fallback-transition event.

## Solution Assessment

Unclear. The requested direction is plausible, but hook naming/payload, notification config, default behavior, and prompt injection ordering need maintainer API and UX judgment before implementation.
