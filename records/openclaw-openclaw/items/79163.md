## Summary

Keep open: current main still has verbose-gated fallback notices plus lifecycle/log observability, but lacks the requested non-verbose/configured notification, fallback-transition plugin hook, and model-change-aware workspace context re-injection. The published canon-guardian ClawHub workaround is useful evidence, and related open issues/PRs cover parts of the problem, but no merged implementation or single canonical thread supersedes this broader hook/context request.

## What This Changes

The issue requests opt-in model fallback notification, workspace context re-injection when a fallback changes the active model, and a plugin hook for fallback transitions.

## Best Possible Solution

Define one maintained fallback-transition contract covering sanitized opt-in notices, documented plugin hook payloads, and `contextInjection: "continuation-skip"` invalidation when the active fallback model changes.

## Reproduction Assessment

No as a strict bug reproduction; this is a feature/API request. Yes for source-level current behavior: current main still verbose-gates fallback notices, lacks model-change input in continuation-skip context injection, and has no fallback-transition hook in the plugin catalog.

## Solution Assessment

Unclear until maintainer API design is done. The requested boundary is plausible, but the exact config key, default, hook name/payload, redaction policy, and re-injection semantics need product review before implementation.
