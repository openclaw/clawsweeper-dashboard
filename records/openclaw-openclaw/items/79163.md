## What This Changes

Request fallback-transition awareness through optional non-verbose user notification, model-change bootstrap context re-injection for `continuation-skip`, and a plugin hook event.

## Best Possible Solution

Yes for the reported current behavior. Source and existing regression tests show verbose-off fallback notices are suppressed, `continuation-skip` ignores model-change state, and current hook names lack a fallback-transition event.

Unclear. The requested direction is plausible, but config naming/defaults, hook payloads, privacy, routing, and prompt-injection ordering need maintainer API/UX judgment before implementation.
