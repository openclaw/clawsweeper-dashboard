Keep open: current main still matches the three reported gaps, and the canon-guardian context in the discussion is a useful partial workaround rather than a native fallback-transition contract.

## What This Changes

The issue requests opt-in user notification, workspace-context re-injection, and a plugin hook when a configured model fallback changes the active model mid-session.

## Best Possible Solution

Define one maintained fallback-transition contract that can drive optional user notification, targeted workspace bootstrap refresh, and plugin observation with documented config and hook payload semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the verbose-only notice path, the continuation-skip helper without model-change state, and the missing hook name; existing tests also assert the default silent fallback and continuation-skip behavior.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact notification policy, hook API, and prompt re-injection ordering need maintainer product/API judgment before a narrow implementation is safe.
