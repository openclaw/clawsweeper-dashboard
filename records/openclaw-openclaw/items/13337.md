## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call provider, and the recent issue discussion kept this as a maintainer product-boundary decision; the documented Clawdia bridge is useful related context but is not `provider: "vapi"` support in `@openclaw/voice-call`.

## What This Changes

Request to add Vapi as a Voice Call telephony provider with config/env support, runtime provider selection, tests, and docs.

## Best Possible Solution

Maintainers should make an explicit ownership decision, then either implement first-class bundled Vapi support end to end or document the external bridge path with any missing public API surfaced deliberately.

## Reproduction Assessment

Not applicable as a runtime bug report. The missing feature is high-confidence by source inspection: `provider: "vapi"` is not accepted, no Vapi provider module exists, and runtime selection has no Vapi branch.

## Solution Assessment

Unclear until maintainers choose the product boundary. If bundled support is approved, the maintainable fix is provider/config/runtime/manifest/docs/tests together; otherwise the safer path is an external bridge/plugin plus any needed public seam.
