## Summary

Keep open. Current main still lacks Vapi as a bundled Voice Call provider, and the maintainer discussion already kept this as a product-boundary decision rather than treating the external Clawdia bridge as a replacement.

## What This Changes

Add a Vapi telephony provider to the official Voice Call plugin, including provider implementation, config/env/schema support, runtime selection, tests, docs, and manifest metadata.

## Best Possible Solution

Maintainers should choose the ownership path, then either implement first-class bundled Vapi support end to end or document/enable the external bridge/plugin path with any missing public telephony seam surfaced deliberately.

## Reproduction Assessment

Not applicable as a runtime bug report. Source inspection gives a high-confidence absence check: provider "vapi" is not accepted, no Vapi provider module exists, runtime selection has no Vapi branch, and docs/manifest omit it.

## Solution Assessment

Unclear until maintainers choose the product boundary. If bundled support is approved, the maintainable fix is provider, config/env/schema, runtime, manifest, tests, docs, and changelog together; otherwise the safer path is an external bridge/plugin plus any needed public seam.
