## Summary

Current main and v2026.6.5 still lack Vapi support, and no viable canonical PR owns the work. The external Clawdia Vapi bridge is useful today, but it does not provide `provider: "vapi"` in Voice Call; because external plugins also lack a public telephony-provider registration seam, this issue still needs a maintainer product/API decision.

## What This Changes

Add Vapi as a telephony provider in the bundled Voice Call plugin, including configuration, runtime integration, tests, and documentation.

## Best Possible Solution

Define a narrow, typed Voice Call telephony-provider extension contract with configuration, lifecycle, webhook-security, and capability hooks, then maintain Vapi as an external ClawHub plugin unless maintainers explicitly accept bundled ownership.

## Reproduction Assessment

Not applicable: this issue requests a new telephony provider rather than reporting broken existing behavior. Source and release inspection do provide a high-confidence verification that Vapi is currently unsupported.

## Solution Assessment

No, not as currently scoped. Directly adding another bundled vendor bypasses the preferred ecosystem boundary; the maintainable path is a public telephony-provider seam followed by an external Vapi plugin, unless maintainers deliberately choose bundled ownership.
