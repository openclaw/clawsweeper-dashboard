The issue requests Vapi support in the Voice Call plugin, including provider code, config/env schema, runtime registration, tests, and docs.

## Best Possible Solution

Maintainers should choose the ownership path first: either implement Vapi end-to-end in the bundled Voice Call plugin, document and rely on the external Clawdia/ClawHub path, or add a public telephony-provider contract before tracking Vapi as external plugin work.
