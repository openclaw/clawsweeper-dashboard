## Summary

Keep open. Current main still builds the webchat session dropdown from persisted `sessions.list` rows plus the active key, so configured agents without stored session rows are still not covered there; the PR’s latest diff is a focused implementation of that missing behavior with tests and changelog credit. It is not closeable as implemented-on-main, and the current PR API reports the branch as dirty with failing broad checks, so it needs maintainer rebase/review rather than cleanup closure.

## What This Changes

The PR updates Control UI session option construction to include configured agents without stored session rows, normalize duplicate detection, respect configured main keys and global scope, add focused regression tests, and add a changelog entry.

## Best Possible Solution

Land a mergeable version of this focused Control UI fix so session options are the union of stored session rows and configured non-global agents, with canonical agent main-session keys, normalized dedupe, regression coverage, and changelog credit preserved.

## Reproduction Assessment

Yes. A unit-level setup with `agentsList` containing `alpha` and `beta` but `sessions.sessions` containing only `agent:alpha:main` reproduces current main because `resolveSessionOptionGroups` never iterates configured agents.

## Solution Assessment

Yes. The PR’s current design is the narrow maintainable fix: use the existing `agents.list` state, synthesize per-agent main session keys through the shared builder, avoid global-scope synthetic targets, and normalize duplicate checks against mixed-case aliases.
