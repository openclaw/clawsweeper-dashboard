## Summary

Keep this PR open: the new terminal proof addresses the prior proof gap for a real CLI plus GitHub codeload path, but the patch still has blocking compatibility and security issues because public ClawHub does not yet serve the new install resolver and GitHub refs are treated as immutable without validation.

## What This Changes

The PR routes unversioned ClawHub skill installs and updates through a new install resolver, adds GitHub codeload-backed skill archive installation, reports install telemetry, and adds unit/E2E coverage around that path.

## Best Possible Solution

Land only after the default/public/custom registry upgrade path is safe and GitHub-backed resolutions are validated as immutable before download and install.

## Reproduction Assessment

Yes for the blocking PR defects: live public ClawHub returns 404 for the new resolver while current detail/download still works, and GitHub codeload accepts mutable refs like main. The PR source then routes unversioned installs through that resolver and does not validate the commit field.

## Solution Assessment

No. GitHub-backed ClawHub installs are the right product direction, but the best fix must preserve or stage the existing install contract and validate immutable GitHub pins before treating the source as safe.
