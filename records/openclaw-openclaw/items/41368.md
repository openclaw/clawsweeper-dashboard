## What This Changes

The PR adds Feishu Bitable helpers, schemas, and tool registrations for deleting one record, deleting records in batches, and deleting a field.

## Best Possible Solution

Land a corrected Feishu plugin change that adds the three delete tools, declares them in the plugin manifest with metadata, adds focused regression coverage and a changelog entry, and preserves existing account routing and SDK error handling.

## Reproduction Assessment

Not applicable for a feature PR. Source inspection on current main confirms the observable gap: Bitable delete APIs are used internally in cleanup but no user-facing delete tools are registered.

## Solution Assessment

No as-is. The helper and SDK-call direction matches existing Bitable patterns, but the best mergeable solution must also update openclaw.plugin.json contracts/toolMetadata, add user-facing changelog coverage, and preferably return the SDK's batch deletion results.

- **[P2] Declare the new tools in the plugin manifest:** `extensions/feishu/src/bitable.ts:809`
  - body: This PR registers three new tool names but does not add them to `extensions/feishu/openclaw.plugin.json`. OpenClaw rejects undeclared plugin tool registrations, so these tools will not be registered/discovered under normal plugin loading until `contracts.tools` and matching `toolMetadata` entries are added.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `extensions/feishu/src/bitable.ts:809`
  - body: This is a user-facing `feat` that exposes new Feishu tools, but the PR does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing feature changes before merge.
  - confidence: 0.86
- **[P3] Return the batch delete response records:** `extensions/feishu/src/bitable.ts:476-477`
  - body: `batchDeleteRecords` reports the input length instead of the API's `data.records` result. The Feishu API and SDK return per-record `{ deleted, record_id }` entries, so callers should receive the actual deletion result rather than only an echoed count.
  - confidence: 0.78
- **[P3] Keep token and table hints consistent:** `extensions/feishu/src/bitable.ts:588-600`
  - body: The new delete schemas shorten the `app_token` and `table_id` descriptions compared with the existing Bitable schemas. These descriptions guide tool calls, so the delete tools should keep the same `feishu_bitable_get_meta` and URL hints used by the other Bitable tools.
  - confidence: 0.73
