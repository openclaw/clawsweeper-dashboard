## Summary

Keep open: the feature is useful and correctly belongs in the Feishu plugin, but the PR only adds runtime registrations. Current plugin policy requires every registered tool to be declared in `openclaw.plugin.json`, so the new tools would be rejected and the bundled tool-contract test would fail.

## What This Changes

The PR adds Feishu Bitable single-record, batch-record, and field delete helpers, schemas, and tool registrations in `extensions/feishu/src/bitable.ts`.

## Best Possible Solution

Land the broader Feishu plugin feature after declaring the new tools in the manifest, adding matching metadata and focused coverage, and preserving the existing tool-contract guard.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: apply the PR as-is and the plugin registry will reject the three new `registerTool` names because they are absent from `contracts.tools`; the bundled contract test checks the same invariant.

## Solution Assessment

No. Exposing these delete APIs in the Feishu plugin is the right boundary, but this patch is not merge-ready until the manifest/tool metadata and required changelog/test coverage are updated.
