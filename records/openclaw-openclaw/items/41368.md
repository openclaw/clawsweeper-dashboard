## Summary

Keep open: current main still lacks user-facing Feishu Bitable delete tools, and this PR targets the right plugin module, but it is not merge-ready because the new runtime tool names are not declared in the Feishu manifest and the destructive delete surface needs maintainer acceptance.

## What This Changes

This PR adds Feishu Bitable helpers, schemas, and tool registrations for deleting one record, deleting records in batch, and deleting a field in `extensions/feishu/src/bitable.ts`.

## Best Possible Solution

If maintainers accept destructive Bitable operations, add the Feishu manifest and `toolMetadata` declarations, keep schema guidance aligned with existing Bitable tools, return SDK response data, add focused tests and a changelog entry, then use this broader PR or a replacement to supersede narrower single-record attempts.

## Reproduction Assessment

Yes for the feature gap: current main has no user-facing `feishu_bitable_delete_*` tool names, while the same module already uses the underlying Lark delete APIs during create-app cleanup. Live Feishu deletion was not run because this was a read-only review.

## Solution Assessment

No as submitted. The Feishu plugin is the right owner, but runtime registration must be paired with manifest/toolMetadata declarations and an explicit maintainer decision on destructive external-data operations.
