## Summary

Keep open. Current main still lacks user-facing Feishu Bitable delete tools and this PR is the broader implementation candidate, but it is not merge-ready because the new tool registrations are not declared in the Feishu plugin manifest and destructive delete exposure needs maintainer acceptance.

## What This Changes

This PR adds Feishu Bitable helpers, schemas, and tool registrations for deleting one record, deleting records in batch, and deleting a field in `extensions/feishu/src/bitable.ts`.

## Best Possible Solution

If maintainers accept exposing destructive Bitable operations, update the Feishu plugin manifest/tool metadata, polish schemas and returned API data, add focused tests and a changelog entry, rebase the branch, and use this broader PR to supersede narrower single-record attempts.

## Reproduction Assessment

Yes for the feature gap: current main has no user-facing Bitable delete tool names, while the same module already uses the underlying Lark delete APIs during cleanup. Live Feishu deletion was not run because this was a read-only review.

## Solution Assessment

No as submitted. The Feishu plugin is the right owner module, but runtime tool registration must be paired with manifest/tool metadata declarations and maintainer acceptance of destructive external-data operations.
