## Summary

## What This Changes

The PR adds Feishu Bitable helpers, schemas, and tool registrations for deleting one record, deleting records in batches, and deleting a field.

## Best Possible Solution

Land a corrected Feishu plugin change that adds the three delete tools, declares them in the plugin manifest with metadata, adds focused regression coverage and a changelog entry, and preserves existing account routing and SDK error handling.

## Reproduction Assessment

Not applicable for a feature PR. Source inspection on current main confirms the observable gap: Bitable delete APIs are used internally in cleanup but no user-facing delete tools are registered.

## Solution Assessment

No as-is. The helper and SDK-call direction matches existing Bitable patterns, but the best mergeable solution must also update openclaw.plugin.json contracts/toolMetadata, add user-facing changelog coverage, and preferably return the SDK's batch deletion results.
