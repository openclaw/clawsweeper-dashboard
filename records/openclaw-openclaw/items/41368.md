Keep open. Current main still lacks the requested Feishu Bitable delete tools, but this PR is not merge-ready because its new runtime tool registrations are undeclared in the Feishu manifest, it lacks real Feishu/Lark behavior proof, and the destructive tool surface needs maintainer acceptance.

## What This Changes

The PR adds Feishu Bitable delete helpers, schemas, and runtime tool registrations for single-record, batch-record, and field deletion in `extensions/feishu/src/bitable.ts`.

## Best Possible Solution

Land one consolidated Feishu plugin change with runtime helpers, manifest/toolMetadata entries, focused contract and runtime coverage, clear destructive-action wording or guard behavior, and redacted live Feishu proof.

## Reproduction Assessment

Yes for the blocking PR defect: source inspection shows the branch registers three new tool names while the current Feishu manifest omits them, and `registerTool` rejects undeclared names. I did not run a live Feishu deletion call in this read-only review.

## Solution Assessment

No, not as-is. Feishu plugin-local delete tools are the right boundary, but this branch needs manifest/toolMetadata updates, safer destructive-tool guidance, focused coverage, real Feishu proof, and maintainer acceptance before merge.
