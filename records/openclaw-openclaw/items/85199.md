## Best Possible Solution

Review and land one focused linked PR that preserves typed alias routing while budgeting Telegram native menu entries canonical-first, then close this issue and supersede duplicate PR attempts.

Yes, source-level: current main maps `nativeAliases` to separate native specs and slices the Telegram menu after aliases have joined the full list. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Yes: the maintainable fix belongs in the registry or Telegram menu budgeting path, preserving typed alias lookup while preventing aliases from displacing canonical commands. A new config knob should be added only if maintainers explicitly want alias menu visibility as operator policy.
