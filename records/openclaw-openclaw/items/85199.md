## Summary

Keep open: current main and the latest shipped tag still expand `nativeAliases` into Telegram menu specs before the 100-command cap, and open fixing PRs already reference this issue and should be reviewed before closure.

## What This Changes

The issue asks OpenClaw to stop Telegram `nativeAliases` such as `/side` from consuming native `/`-menu slots ahead of canonical commands.

## Best Possible Solution

Review and land one narrow fix that prioritizes canonical commands under Telegram's 100-command cap, preserves typed alias routing, and resolves the duplicate fix PRs after merge.

## Reproduction Assessment

Yes for source-level reproduction: current main emits one native spec per alias, `/btw` declares `/side` as a native alias, and Telegram capping slices the expanded list to 100. I did not run a live bot `getMyCommands` check.

## Solution Assessment

Yes: canonical-first menu budgeting with alias routing preserved is the narrowest maintainable fix. A new config knob is optional and should only be added if maintainers want explicit operator control.
