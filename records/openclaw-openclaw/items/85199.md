## Summary

Keep open: current main still appears to expand Telegram `nativeAliases` into the native command menu before applying Telegram's 100-command cap, while typed alias routing is separate and the command config has no alias-level control.

## What This Changes

The issue asks for Telegram native command menu budgeting to avoid counting aliases as separate `/` menu slots, or to expose an operator control for alias visibility.

## Best Possible Solution

Prioritize canonical command specs ahead of aliases for Telegram native menu registration, include aliases only from leftover budget if maintainers want that behavior, and preserve existing typed alias dispatch through `textAliases`.

## Reproduction Assessment

Yes for a high-confidence source reproduction: current main expands `nativeAliases` into the same Telegram native menu list that is later sliced to the 100-command cap. I did not perform a live Telegram API repro in this read-only sweep.

## Solution Assessment

Yes: the narrow maintainable fix is canonical-first Telegram menu budgeting with alias entries omitted or added only after canonical commands have budget. A new config knob is unnecessary unless maintainers explicitly want operator-controlled alias menu visibility.
