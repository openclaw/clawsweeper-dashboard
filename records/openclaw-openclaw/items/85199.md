## Summary

Keep open: current main and the latest shipped tag still expand `nativeAliases` into Telegram native menu entries before applying Telegram's 100-command menu cap, and the strict Telegram command config has no alias-level escape hatch. The source path is clear and there is no linked implementation PR or duplicate that owns this narrower alias-budget bug.

## What This Changes

The issue requests that Telegram native command menus stop counting alias names such as `/side` as independent menu entries ahead of canonical commands.

## Best Possible Solution

Prioritize canonical command specs ahead of aliases for the Telegram native menu cap, include aliases only from leftover budget if desired, preserve typed alias routing, and add regression coverage for the 100-canonical-plus-alias case.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live current-main Telegram run in this read-only review. Current main and `v2026.5.20` expand `/side` from `nativeAliases` into the same list that is sliced to Telegram's 100-command limit.

## Solution Assessment

Yes: the narrow maintainable fix is to make Telegram menu budgeting canonical-first and alias-after-budget while keeping text alias dispatch intact. A new config knob is unnecessary unless maintainers explicitly want operator control over alias menu visibility.
