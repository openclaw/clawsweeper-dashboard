## Summary

Keep open: current main and the latest shipped tag still expand `nativeAliases` into Telegram menu entries before applying Telegram's 100-command cap, and there is no alias-level config escape hatch.

## What This Changes

The issue asks for Telegram native command menu budgeting to keep `nativeAliases` from consuming slots ahead of canonical commands, or to expose a controlled alias-menu setting.

## Best Possible Solution

Prioritize canonical Telegram menu entries before aliases, include aliases only from leftover budget if maintainers still want alias discoverability, and keep typed alias dispatch unchanged.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live current-main Telegram run. Current main and `v2026.5.20` expand `/side` from `nativeAliases` into the same menu list that is sliced to Telegram's 100-command limit.

## Solution Assessment

Yes: the narrow maintainable fix is Telegram menu budgeting that is canonical-first and alias-after-budget while preserving text alias dispatch. A new config knob is only needed if maintainers explicitly want operator control over alias menu visibility.
