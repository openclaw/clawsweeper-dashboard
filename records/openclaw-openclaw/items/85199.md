## Summary

Keep open: current main and the latest shipped release still expand `nativeAliases` into Telegram native menu entries before the 100-command cap, and there are now open linked fix PRs that should be reviewed rather than closing the issue early.

## What This Changes

The issue reports that Telegram `nativeAliases` consume separate `/` menu slots and can push installs over Telegram's 100-command cap.

## Best Possible Solution

Land one focused fix that preserves typed alias routing while budgeting Telegram native menu entries canonical-first, then close this issue and supersede the duplicate PR attempts.

## Reproduction Assessment

Yes for a source-level reproduction: current main expands `/side` from `nativeAliases` into the native command specs and slices the combined Telegram menu to 100 entries afterward. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Yes: the maintainable fix is in the registry/menu budgeting path, preserving typed alias lookup while preventing aliases from displacing canonical commands. A new config knob is not needed unless maintainers explicitly want alias menu visibility as operator policy.
