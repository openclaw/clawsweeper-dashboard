## Summary

Keep open: current main and the latest release still expand Telegram `nativeAliases` into separate native menu specs before applying the 100-command cap, and open linked fix PRs already target this issue.

## What This Changes

Avoid counting Telegram `nativeAliases` as separate `/` menu entries so canonical commands keep the 100-slot Telegram budget while typed aliases still route.

## Best Possible Solution

Review and land one focused fix that keeps typed alias lookup while budgeting Telegram menu entries canonical-first, then close this issue and supersede the duplicate PR attempts.

## Reproduction Assessment

Yes for a source-level reproduction: current main expands `/side` from `nativeAliases` into native command specs and slices the combined Telegram menu to 100 entries afterward. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Yes, the maintainable fix belongs in the registry/menu budgeting path: preserve typed alias lookup while preventing aliases from displacing canonical commands. Maintainers still need to pick the exact alias-visibility policy among the open PRs.
