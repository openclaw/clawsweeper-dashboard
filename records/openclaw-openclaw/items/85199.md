## Summary

Keep open: current main and the latest release still expand Telegram `nativeAliases` into native menu entries before applying the 100-command cap, and open linked PRs already target the fix.

## What This Changes

The issue asks OpenClaw to stop Telegram `nativeAliases` from consuming separate native command menu slots, or to add an operator control for alias menu visibility.

## Best Possible Solution

Review and land one focused fix that preserves typed alias routing while budgeting Telegram native menu entries canonical-first, then close this issue and supersede duplicate PR attempts.

## Reproduction Assessment

Yes for a source-level reproduction: current main expands `/side` from `nativeAliases` into native command specs and slices the combined Telegram menu to 100 entries afterward. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Yes: the maintainable fix belongs in the registry/menu budgeting path, preserving typed alias lookup while preventing aliases from displacing canonical commands. A new config knob is only needed if maintainers explicitly want alias menu visibility as operator policy.
