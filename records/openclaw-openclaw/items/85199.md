Fix the Telegram native command menu budgeting bug reported in https://github.com/openclaw/openclaw/issues/85199. Current main expands `nativeAliases` into native command specs before Telegram's 100-command cap, so `/side` can consume a separate menu slot even though typed `/side` already routes through `/btw`. Implement a bounded fix that prioritizes canonical command menu entries ahead of aliases, includes aliases only after canonical commands have been budgeted if that is the chosen shape, and preserves typed alias dispatch through `textAliases`. Do not add a new config option unless a maintainer explicitly requests it. Add focused regression coverage for a 100-canonical-plus-alias Telegram menu case and keep non-Telegram command behavior unchanged unless tests prove the shared registry contract needs a safe adjustment. Changelog is expected if the PR changes user-visible Telegram menu behavior.
