Keep open: current main and v2026.5.20 still expand `nativeAliases` into Telegram native menu entries before the 100-command cap, and open linked fix PRs should be reviewed rather than closing this issue early.

## What This Changes

Issue reports that Telegram chat-command `nativeAliases` are expanded into separate native menu entries and consume scarce Telegram command slots.

## Best Possible Solution

Review and land one focused fix that preserves typed alias routing while budgeting Telegram native menu entries canonical-first, then close this issue and supersede the duplicate PR attempts.

## Reproduction Assessment

Yes for a source-level reproduction: current main expands `/side` from `nativeAliases` into native command specs and slices the combined Telegram menu to 100 entries afterward. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Yes: the maintainable fix belongs in the registry/menu budgeting path, preserving typed alias lookup while preventing aliases from displacing canonical commands. A new config knob is only needed if maintainers explicitly want alias menu visibility as operator policy.
