## Summary

Keep open: current main and the latest shipped tag still expand `nativeAliases` into Telegram native menu entries before the 100-command cap is applied, and the Telegram command config has no alias-level escape hatch.

## What This Changes

The issue requests changing Telegram native command registration so aliases such as `/side` do not consume separate `/` menu slots ahead of canonical commands.

## Best Possible Solution

Prioritize canonical command specs ahead of aliases for the Telegram native menu cap, include aliases only from leftover budget, preserve typed alias routing, and add regression coverage for the 100-canonical-plus-alias case.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live current-main Telegram run in this read-only review. Current main and `v2026.5.20` expand `/side` from `nativeAliases` into the same menu list that is sliced to Telegram's 100-command limit.

## Solution Assessment

Yes: the narrow maintainable fix is to make Telegram menu budgeting canonical-first and alias-after-budget while keeping text alias dispatch intact. A new config knob is not necessary unless maintainers explicitly want operators to control alias menu visibility.
