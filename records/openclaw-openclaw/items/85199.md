## Summary

Keep open: current main and the latest shipped tag still expand Telegram `nativeAliases` into native menu entries before applying the 100-command cap. There are already open fix PRs for this issue, so this should stay open until one is reviewed and merged.

## What This Changes

Telegram native command aliases should stop consuming separate bot menu slots before the 100-command cap while remaining callable as typed aliases.

## Best Possible Solution

Land one focused Telegram fix that budgets canonical commands before aliases, preserves typed `/side` routing, and adds regression coverage for the 100-canonical-plus-alias case.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: current main expands `/side` from `nativeAliases` into the same native menu list that is sliced to Telegram's 100-command cap. I did not perform a live Telegram Bot API run in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to make Telegram menu budgeting canonical-first while keeping typed alias dispatch intact. A new config knob is only needed if maintainers want an explicit operator policy surface.
