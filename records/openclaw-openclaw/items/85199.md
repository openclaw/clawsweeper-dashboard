## Summary

Keep open: current main and latest release still expand Telegram `nativeAliases` into native menu entries before applying the 100-command cap, and multiple open linked PRs already target the same bug.

## What This Changes

The issue asks Telegram native command menu generation to stop counting `nativeAliases` as separate `/`-menu entries while preserving typed alias routing.

## Best Possible Solution

Review and land one focused fix, preferably the smallest canonical-first Telegram menu budgeting path that preserves typed alias routing, then close this issue and supersede duplicate PR attempts.

## Reproduction Assessment

Yes for a source-level reproduction: current main expands `/side` from `nativeAliases` into native command specs and caps the combined Telegram menu afterward. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Yes: the maintainable fix belongs in the registry/menu budgeting path, preserving typed alias lookup while preventing aliases from displacing canonical commands. A new config knob is only needed if maintainers explicitly want alias menu visibility as operator policy.
