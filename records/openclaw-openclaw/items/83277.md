## What This Changes

The issue reports that WhatsApp Show QR on Windows returns `web login provider is not available` even after `wacli` is installed and authenticated because the OpenClaw WhatsApp channel plugin is not loaded.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90517, or an equivalent narrow gateway fix that reuses the official external plugin repair helper without treating `wacli` as the OpenClaw WhatsApp channel runtime.

## Reproduction Assessment

Yes. Source inspection shows current main returns the exact generic provider-unavailable error whenever no loaded channel plugin advertises `web.login.start` or `web.login.wait`; I did not rerun the Windows Scheduled Task gateway flow live.

## Solution Assessment

Yes. Reusing the existing official-external-plugin repair helper in the gateway no-provider branch is the narrowest maintainable solution, and it avoids incorrectly making `wacli` the WhatsApp channel runtime.
