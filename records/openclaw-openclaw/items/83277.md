## What This Changes

The issue reports WhatsApp Show QR returning a generic provider-unavailable error when WhatsApp is configured but the external WhatsApp plugin is not loaded, despite wacli being installed.

## Best Possible Solution

Keep WhatsApp external, but make `web.login.start` and `web.login.wait` surface the existing official-plugin install or `openclaw doctor --fix` hint when a configured official external channel has no loaded owner plugin.

## Reproduction Assessment

Yes for the source-level path: with no loaded channel plugin advertising `web.login.start` or `web.login.wait`, current main returns the exact generic error. I did not reproduce the full Windows Scheduled Task dashboard flow.

## Solution Assessment

Yes, the best fix is the narrow helper-backed diagnostic path, not making wacli a channel runtime dependency. The closed linked PR demonstrates a plausible patch shape, but the issue still needs a landed replacement or revived fix.
