## Summary

Keep open: current main and the latest release still return the generic web-login provider-unavailable error for the configured-but-missing WhatsApp plugin path, and the matching fix is an open closing PR rather than merged code.

## What This Changes

The issue asks OpenClaw to give useful WhatsApp setup guidance instead of `web login provider is not available` when the WhatsApp channel is configured but its external plugin is not loaded, even if `wacli` is installed.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90517, or replace it with the same narrow gateway fix that reuses the official external plugin repair helper without treating `wacli` as the OpenClaw channel runtime.

## Reproduction Assessment

Yes. Source inspection shows current main returns the exact generic provider-unavailable error whenever no loaded channel plugin advertises `web.login.start` or `web.login.wait`; I did not rerun the Windows Scheduled Task gateway flow live.

## Solution Assessment

Yes. Reusing the existing official-external-plugin repair helper in the gateway no-provider branch is the narrowest maintainable solution, and it avoids incorrectly making `wacli` the WhatsApp channel runtime.
