## Summary

Keep open: current main and the latest release still return the generic web-login provider-unavailable error for the configured-but-missing WhatsApp plugin path, and https://github.com/openclaw/openclaw/pull/90517 is an open closing PR rather than a merged fix.

## What This Changes

Configured WhatsApp on Windows returns the generic web-login unavailable error even though `wacli` is present, because the WhatsApp channel plugin never registers.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90517, or replace it with the same narrow gateway fix that reuses the official external plugin repair helper without making `wacli` a channel dependency.

## Reproduction Assessment

Yes, source-reproducible: current main returns the exact generic `web login provider is not available` error whenever no loaded channel plugin advertises `web.login.start` or `web.login.wait`. I did not run the Windows Scheduled Task gateway flow live.

## Solution Assessment

Yes, the linked PR's direction is the best current fix: keep WhatsApp as an external plugin and reuse the existing official-external-plugin repair helper in the gateway no-provider branch. Treating authenticated `wacli` as the OpenClaw channel runtime would be the wrong layer.
