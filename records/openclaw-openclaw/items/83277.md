## Summary

Keep open: current main still has the exact generic web-login failure path when no QR-capable channel plugin is loaded, and the dashboard exposes that raw error for configured-but-missing WhatsApp installs. The useful fix is not to treat `wacli` as the channel runtime, but to reuse the existing official external plugin repair hint so Windows/package users get actionable install or doctor guidance.

## What This Changes

The issue reports that the WhatsApp channel does not register on Windows, causing the dashboard QR flow to fail with `web login provider is not available` despite `wacli` being installed and authenticated.

## Best Possible Solution

Keep WhatsApp as an external plugin, but make the dashboard/web-login path detect a configured missing official WhatsApp plugin and surface the existing install or `doctor --fix` guidance.

## Reproduction Assessment

Yes, at source level. Configured WhatsApp without a loaded QR-capable channel plugin makes `web.login.start` return the exact generic provider-unavailable error; I did not run the Windows scheduled-task gateway flow.

## Solution Assessment

No, using `wacli` as the channel provider would be the wrong solution. The narrow maintainable fix is to reuse the existing official-external-plugin repair hint in the web-login/dashboard path.
