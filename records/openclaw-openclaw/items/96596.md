## What This Changes

This PR changes the session model resolver to use numeric-aware `localeCompare` for alias and dated-version candidate ordering, and extends resolver tests for double-digit aliases.

## Best Possible Solution

Land this focused comparator-and-test fix or an equivalent sibling after maintainers accept the model-choice ordering change and exact-head checks pass, then close the remaining same-root-cause PRs and the linked issue after merge.

## Reproduction Assessment

Yes. Source inspection shows `parseModelPattern("opus", [claude-opus-4-9, claude-opus-4-10])` reaches the current plain alias comparator, where lexicographic ordering can put the older `4-9` suffix first.

## Solution Assessment

Yes. Numeric-aware full-string collation at the existing `tryMatchModel` sort points is the narrowest maintainable fix; a custom comparator or semver parser would add unnecessary provider-id policy for this resolver path.
