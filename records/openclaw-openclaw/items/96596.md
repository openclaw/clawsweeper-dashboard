This PR changes the session model resolver to use numeric-aware locale collation for alias and dated-version candidate ordering and adds colocated tests for numeric alias sorting while preserving fallback coverage.

## Best Possible Solution

Land one narrow resolver comparator fix with regression tests that preserve existing fallback coverage, then close the canonical issue and sibling duplicate PRs.

## Reproduction Assessment

Yes, at source level. Current main and the latest release route alias/partial model patterns through `parseModelPattern` to plain lexicographic `localeCompare`, and a comparator check shows `4-9` sorts above `4-10` without numeric collation.

## Solution Assessment

Yes. Numeric-aware collation at `tryMatchModel` is the narrowest maintainable fix; a semver parser would be provider-specific overreach, and changing catalog/provider config would miss the shared resolver ordering point.
