## Summary

Keep open: current main has related hardening knobs, but I did not find a single supported proposal-only lockdown map or native read-only/no-tools mode that answers this issue end to end.

## What This Changes

The issue asks for authoritative documentation and validation guidance for disabling plugins, tools, channels, automation, MCP, hooks, sidecars, and owner-elevated commands in a proposal-only/dashboard-only setup.

## Best Possible Solution

Add a maintainer-reviewed security/operator doc or guided preset that maps each requested surface to exact config keys, states which surfaces are mandatory for dashboard chat, and gives a validation command for the effective tool/channel/plugin inventory.

## Reproduction Assessment

Not applicable as a docs/product hardening request. Source and docs review confirm that existing knobs are scattered and do not currently form a documented proposal-only lockdown mode.

## Solution Assessment

Unclear until maintainers choose scope. A docs-only map can answer the immediate request, but a native proposal-only/read-only mode would be a new product/config contract and should align with the related open design issues.
