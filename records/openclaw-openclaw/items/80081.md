## Summary

Keep open: current main has distributed hardening controls, but it still lacks a maintainer-approved proposal-only/no-authority lockdown map or native mode, and the broader Security Profile request does not replace this narrower documentation request.

## What This Changes

The issue requests authoritative config key paths and disabled values for running dashboard/proposal-only OpenClaw without plugin, tool, channel, automation, hook, MCP, app, exec, or owner-elevated authority.

## Best Possible Solution

Add a maintainer-reviewed proposal-only lockdown guide or preset that maps every requested surface to canonical keys, states dashboard-only minimum requirements, and gives verification commands.

Not applicable: this is a documentation/security-posture request, not a broken existing behavior report. Source and docs inspection is sufficient to show the requested end-to-end authoritative map is not present on current main.

## Solution Assessment

Unclear until maintainers choose the supported posture. The narrow maintainable path is a documented lockdown recipe if existing keys are sufficient, or a native preset only if OpenClaw wants proposal-only/read-only behavior as a product contract.
