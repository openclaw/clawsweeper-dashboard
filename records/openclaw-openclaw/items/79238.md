## Summary

Keep open: this maintainer-labeled PR addresses a real mixed-provider Codex routing bug and has sufficient CLI proof, but it is not merge-ready because the doctor cleanup still preserves stale session runtime pins while the PR docs and changelog say those pins are removed.

## What This Changes

The PR changes doctor repair, runtime selection, session/status surfaces, schemas, docs, changelog, and tests to move agent runtime policy from whole-agent/session pins to provider/model-scoped routing.

## Best Possible Solution

Land a provider/model-scoped runtime fix only after doctor cleanup, docs, tests, and generated config baselines agree on how stale whole-agent and session runtime state is removed or explicitly ignored.

## Reproduction Assessment

Yes. Current-main source shows doctor can create a whole-agent Codex runtime pin and session execution can reuse stored harness pins, while the linked bug report gives live 2026.5.7 MiniMax and Anthropic provider-not-found failures.

## Solution Assessment

Not yet as submitted. Provider/model-scoped runtime policy is a reasonable direction, but the patch still preserves stale session runtime pins during doctor repair while documenting that doctor removes them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Clear stale session runtime pins during doctor repair:** `src/commands/doctor/shared/codex-route-warnings.ts:648-653`
  - body: The PR docs/changelog say `doctor --fix` removes stale session runtime pins, but this repair path only rewrites legacy route fields and fallback notices; sessions with `agentHarnessId`/`agentRuntimeOverride` keep those values, and the updated test now expects that preservation. The generic plugin session repair also treats the Codex route as allowed and never inspects `agentRuntimeOverride`, so affected sessions remain stale after doctor cleanup. Delete both persisted runtime keys when repairing legacy Codex session routes, or route them through a cleanup path that clears both fields.
