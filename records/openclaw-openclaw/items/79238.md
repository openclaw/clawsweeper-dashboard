## Best Possible Solution

Land a narrow runtime-policy fix that preserves automatic OpenAI Codex routing, keeps explicit provider/model PI or CLI policies working, cleans stale legacy pins through doctor, and passes the touched runtime/status/doctor gates.

## Reproduction Assessment

Yes. Current-main source shows doctor writes a Codex runtime pin during legacy route repair and session execution can reuse stored harness pins; the PR body also provides a real after-fix doctor run, while latest-head CI reproduces new implementation blockers.

## Solution Assessment

No, not merge-ready yet. Provider/model scoped runtime policy looks like the right direction, but the implementation needs to restore CLI/status behavior and clear lint, typecheck, and regression failures before maintainer approval.
