## Summary

Keep open: current main and the latest release still only provide opt-in exponential recency decay, not durable write-time importance metadata, access-boost persistence, Weibull decay, or importance-weighted retrieval across builtin and QMD memory paths. The KinthAI production comment adds useful design context, while https://github.com/openclaw/openclaw/pull/43306 and https://github.com/openclaw/openclaw/pull/71590 are open, unmerged partial implementation candidates rather than shipped replacements.

## What This Changes

Add write-time memory importance scoring, Weibull time decay, access boosts, and importance-weighted retrieval for long-term memory.

## Best Possible Solution

Add a focused memory-core design that stores backward-compatible importance/access metadata, supports configurable decay, applies weighting consistently to builtin and QMD search, and updates config, docs, changelog, and targeted tests; review the existing open PRs as inputs rather than treating either as already shipped.

## Solution Assessment
