## Summary

Keep open: current main and v2026.6.10 only cover adjacent pieces, mainly opt-in exponential recency decay for memory-core hybrid search, manual importance on the separate LanceDB plugin, and weighted dreaming promotion. The requested write-time importance scoring, Weibull decay, access reinforcement, contradiction handling, and QMD parity remain product/data-model work; the KinthAI and Dakera comments add useful production examples and the contradiction-replacement edge case.

## What This Changes

The issue requests durable memory importance scoring, time decay, access reinforcement, and importance-weighted retrieval for long-term memory.

## Best Possible Solution

Define a maintainer-approved memory-plugin design for durable importance metadata, scoring policy, decay/access reinforcement, contradiction handling, and builtin/QMD behavior, then implement it with schema, docs, migration or doctor coverage, and focused ranking tests.

## Reproduction Assessment

Yes, at source level. Current main exposes exponential temporal recency decay and weighted dreaming promotion, while write-time importance scoring, Weibull decay, access boost, contradiction handling, and QMD parity are absent from the inspected runtime/config paths.

## Solution Assessment

Unclear as a final product direction. Importance-weighted retrieval is plausible, but maintainers need to choose the durable metadata schema, scoring policy, config shape, QMD contract, and contradiction behavior before this is a safe implementation task.
