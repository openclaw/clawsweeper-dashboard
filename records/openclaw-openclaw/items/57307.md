## Summary

Keep open: current main still only provides opt-in exponential recency decay for builtin memory-core hybrid search plus adjacent dreaming promotion ranking; write-time importance scoring, Weibull decay, access reinforcement, contradiction handling, and QMD parity remain unimplemented product/data-model work.

## What This Changes

The issue requests durable memory importance scoring, time decay, access-based reinforcement, and importance-weighted retrieval for long-running memory backends.

## Best Possible Solution

Define a maintainer-approved memory-core/plugin design for durable importance metadata, scoring policy, decay/access reinforcement, contradiction handling, and builtin/QMD behavior, then implement it with schema, docs, migration or doctor coverage, and focused ranking tests.

## Reproduction Assessment

Yes, at source level. Current main exposes only exponential temporal recency decay and weighted dreaming promotion, while the requested write-time importance, Weibull model, access boost, contradiction handling, and QMD parity are absent from the inspected runtime/config paths.

## Solution Assessment

Unclear as a final product direction. Importance-weighted retrieval is plausible, but maintainers need to choose the durable metadata schema, scoring policy, config shape, QMD contract, and contradiction behavior before this is a safe implementation task.
