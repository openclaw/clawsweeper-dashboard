## Summary

Keep open: current main and the latest release still provide only opt-in exponential recency decay and adjacent dreaming recall scoring, not durable write-time importance metadata, access-boost persistence, Weibull decay, contradiction replacement, or importance-weighted retrieval across builtin and QMD memory paths. The KinthAI production comment is useful design input, while https://github.com/openclaw/openclaw/pull/71590 is open and partial and https://github.com/openclaw/openclaw/pull/43306 closed unmerged.

## What This Changes

Add long-term memory importance scoring, configurable decay, access boosts, and importance-weighted retrieval for builtin and QMD memory search.

## Best Possible Solution

Add a focused memory-core design that stores backward-compatible importance/access metadata, supports configurable decay, applies weighting consistently to builtin and QMD search, handles contradiction replacement policy, and updates config, docs, changelog, and targeted tests.
