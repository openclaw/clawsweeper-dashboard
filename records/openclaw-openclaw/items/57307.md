## Summary

Keep open: current main still lacks durable write-time importance metadata, Weibull decay, access-boost persistence, and importance-weighted retrieval across builtin and QMD memory paths. The KinthAI comment adds useful production design context, while #71590 and #43306 are open, unmerged implementation candidates rather than shipped replacements.

## What This Changes

The issue requests durable memory importance scoring, configurable Weibull time decay, access-based boosting, and importance-weighted retrieval for builtin and QMD long-term memory paths.

## Best Possible Solution

Add a memory-core design and implementation that stores backward-compatible importance/access metadata, supports configurable Weibull decay, applies retrieval weighting consistently for builtin search, defines QMD behavior explicitly, and updates config, docs, changelog, and targeted tests.
