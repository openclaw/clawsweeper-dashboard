## Summary

Keep this issue open. Current main has opt-in exponential temporal decay, hybrid BM25/vector retrieval, and short-term promotion signals, but it still lacks durable write-time importance scoring, Weibull decay, access-boost persistence, and importance-weighted ranking across builtin and QMD memory search. The related implementation PRs are still open, so they are candidates for review rather than proof that the request shipped.

## What This Changes

The issue requests durable memory importance scoring, configurable Weibull time decay, access-based boosts, and importance-weighted retrieval for long-running memory backends.

## Best Possible Solution

Implement a cohesive memory-core design that persists backward-compatible importance and access metadata, adds configurable Weibull decay and retrieval weighting for builtin search, defines the QMD contract or limitation explicitly, updates config schema/docs, and adds focused tests. Treat #71590 and #43306 as candidate inputs to that design rather than closing this tracker before merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source and docs inspection on current main, which shows the requested importance and Weibull/access behavior is absent.

## Solution Assessment

No, closing this as implemented would not be the best way to solve it because the central requested behavior is still missing. The narrowest maintainable path is a reviewed memory-core design/PR, likely incorporating or superseding the open related PRs.
