## Summary

Keep open: the second commit fixed the broad exec-helper rewrite, but the PR still corrects paths in the outer workspace guard, so workspace-only write/edit wrappers can silently receive `report.docx` before the inner mutation rejection sees the original hallucinated suffix.

## What This Changes

## Best Possible Solution

Keep the workspace guard XML-only, let read wrappers perform the narrow correction, let write/edit/memory-flush paths reject hallucinated extensions before mutation, and add focused tests for guarded workspace-only write/edit plus updated real behavior proof.

## Solution Assessment

No. The read-correct and mutation-reject split is the right direction, but putting correction in the outer workspace guard makes the implementation incomplete and bypasses the intended mutation rejection path.
