## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep protected skill mounts authoritative:** `src/agents/sandbox/docker.ts:603-606`
  - body: This filter removes the protected read-only skill mount whenever a user bind targets the same container path. Current main explicitly tests that protected skill overlays win over custom binds, and the FS bridge prioritizes `protectedSkill` over `bind` so those paths are never treated as writable. Please keep the protected mount and skip or reject the conflicting custom bind before `docker create`.
  - confidence: 0.92
- **[P2] Parse bind specs with the existing helper:** `src/agents/sandbox/docker.ts:598-599`
  - body: The new filter splits bind strings with `b.split(":")`, which misparses supported Windows drive-letter sources like `C:\Users\me:/workspace/skills:rw`; the conflict is then missed and Docker still sees duplicate targets. Reuse `splitSandboxBindSpec` or a shared helper for both shell and browser paths.
  - confidence: 0.86
