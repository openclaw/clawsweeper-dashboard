## Summary

Keep this PR open. It addresses a real current-main outbound hook regression, but the patch still overstates the public hook contract by documenting outbound `message_sent` and `runId` correlation that delivery does not provide, and it lacks the required changelog entry for user-facing plugin behavior.

## What This Changes

The PR updates routed outbound reply session-key selection, passes the computed outbound session key into `message_sending`, adds outbound delivery tests, and expands hook/session context JSDoc.

## Best Possible Solution

Land the runtime fix after the public contract is made exact: either implement and test the documented outbound `sessionKey`/`runId` propagation for the affected hooks, or narrow the JSDoc to the `message_sending.sessionKey` behavior this patch actually provides, then add the changelog entry and native redirect regression coverage.

## Reproduction Assessment

Yes. Current main can be reproduced by invoking outbound delivery with a `message_sending` hook and `session.key`; the hook context is built without `sessionKey`. The redirect mismatch is visible by comparing `get-reply.ts` agent session resolution with `dispatch-from-config.ts` routed delivery parameters.

## Solution Assessment

No, not as written. The runtime plumbing is the right narrow direction, but the docs advertise fields the patch does not deliver and the PR lacks required changelog coverage; the safer path is to narrow the docs or wire the missing fields with tests.
