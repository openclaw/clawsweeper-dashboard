## Summary

Keep open for the PR path: current main still has the reported outbound hook identity gap, and the PR fixes the right seams with sufficient terminal proof. I found one low-priority context-shape issue where the no-session `message_sending` context should omit the optional property instead of setting it to `undefined`.

## What This Changes

The branch threads outbound `sessionKey` into `message_sending` and `message_sent`, aligns native redirect routed replies with the agent runtime session key, documents the hook/session contract, and adds tests, a proof script, API hash refresh, and changelog text.

## Best Possible Solution

Land the session identity plumbing after the optional-property cleanup and normal maintainer CI/merge checks, while leaving broader outbound `runId` work to https://github.com/openclaw/openclaw/issues/18571.

## Reproduction Assessment

Yes. Current main is source-reproducible: outbound delivery computes a canonical hook session key but does not pass it to `message_sending`, and native redirected replies pass the inbound key even though the agent runtime uses the redirect target key.

## Solution Assessment

Mostly yes. The PR fixes the contract at the existing outbound delivery and routed-reply seams, but the no-session `message_sending` context should omit the optional `sessionKey` property to match the new docs and `message_sent` behavior.
