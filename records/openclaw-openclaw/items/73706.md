## Summary

Keep open: the runtime fix is useful and source-reproducible, but this PR still over-documents outbound `message_sent`/`runId` fields it does not deliver and is missing required changelog and native-redirect regression coverage.

## What This Changes

The PR threads outbound `sessionKey` into `message_sending`, changes routed reply `session.key` selection for native redirects, adds outbound delivery tests, regenerates the plugin SDK hash baseline, and expands hook/session JSDoc.

## Best Possible Solution

Land the narrow session-key and native-redirect fix after the public hook contract is made exact, a changelog entry is added, and focused redirect coverage protects the changed routing behavior; leave broader outbound `runId`/tool-history work to #18571 unless this PR implements it fully.

## Reproduction Assessment

Yes. Source inspection on current main shows `message_sending` receives no `ctx.sessionKey` even though delivery computes `sessionKeyForInternalHooks`, and native redirect delivery uses `ctx.SessionKey` where the agent runtime uses `targetSessionKey || ctx.SessionKey`.

## Solution Assessment

No, not as written. The runtime plumbing is the right narrow direction, but the docs currently promise outbound `message_sent`/`runId` fields that are not delivered, and the semantic routing change still needs changelog and focused regression coverage.
