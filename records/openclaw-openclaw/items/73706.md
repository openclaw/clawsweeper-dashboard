## Summary

Keep this PR open. It fixes a real current-main outbound hook plumbing bug, but the proposed contract docs still promise outbound `message_sent` and `runId` fields that this delivery path does not provide, and the user-facing hook fix still needs a changelog entry.

## What This Changes

The PR changes routed outbound reply session-key selection, threads `sessionKey` into `message_sending`, adds outbound delivery tests, and expands hook/session context JSDoc.

## Best Possible Solution

Land the narrow runtime fix after the public contract is made exact, the changelog records the plugin hook fix, and native redirect coverage proves `session.key` matches the agent runtime key.

## Solution Assessment

No, not as written. The plumbing change is the right narrow fix, but the JSDoc over-promises fields that are not delivered; the safer repair is to narrow the docs to implemented behavior or plumb and test the extra fields end to end.
