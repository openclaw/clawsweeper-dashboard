## Summary

Keep this PR open. Current main still drops the outbound session key before `message_sending`, and the PR's runtime direction is useful, but the patch currently documents `message_sent` and `runId` guarantees that it does not actually deliver and it lacks the required changelog entry.

## What This Changes

The PR changes routed outbound replies to use the agent runtime session key for native redirects, passes the computed session key into `message_sending`, adds outbound delivery tests, and expands JSDoc for outbound session and message hook contracts.

## Best Possible Solution

Land the narrow runtime fix after the public contract is made exact: either implement and test `message_sent` session-key and outbound `runId` propagation, or narrow the new JSDoc to only promise the `message_sending` `sessionKey` behavior this PR actually provides, then add a changelog entry and redirect regression coverage.

## Solution Assessment
