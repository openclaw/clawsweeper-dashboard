## Summary

Keep this PR open. Current main still drops the canonical outbound session key before `message_sending` and still diverges from the agent runtime session key for native redirect replies, so the runtime fix is useful; however the patch is not merge-ready because its new hook JSDoc promises outbound `message_sent`/`runId` fields the diff does not supply, and it lacks the required changelog entry.

## What This Changes

This PR threads a canonical session key into outbound `message_sending`, aligns routed native replies' outbound session key with the agent runtime, and adds tests plus hook/session context documentation.

## Best Possible Solution

Land the narrow session-key runtime fix with an exact public contract: either thread the same fields into every outbound hook the new docs promise, or narrow the JSDoc to the implemented `message_sending` session-key behavior and leave broader `message_sent`/`runId` work to the related hook-contract issue.

## Solution Assessment
