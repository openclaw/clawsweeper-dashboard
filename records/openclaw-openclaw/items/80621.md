## Summary

Keep open. Current main still treats `contracts.tools` as an explicit string-array ownership contract, the boolean manifest shape is source-reproducible as the reported diagnostic, and the linked open PR with closing syntax should be reviewed or closed before this issue is resolved.

## What This Changes

The issue requests support for native plugins declaring `contracts.tools: true` when registering agent tools, plus clearer plugin SDK guidance for the required tool contract.

## Best Possible Solution

Keep explicit string-array tool ownership unless maintainers approve a wildcard mode; otherwise fix this with clearer invalid-manifest diagnostics and docs on the plugin-author pages.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: boolean `contracts.tools` is not an array, normalizes to `[]`, and `api.registerTool` then emits the exact reported diagnostic.

## Solution Assessment

No. Accepting `contracts.tools: true` is not the narrowest fix under the current contract; the safer path is clearer docs and diagnostics unless maintainers intentionally approve and document wildcard tool ownership.
