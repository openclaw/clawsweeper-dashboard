## What This Changes

The issue asks OpenClaw to treat `contracts.tools: true` as sufficient manifest ownership for runtime `api.registerTool` registrations.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/80623 is reviewed, landed, or replaced; if boolean wildcard support is rejected, fix the user problem with clearer invalid-manifest diagnostics and docs that point to explicit `contracts.tools` arrays.
