## Summary

Keep open. Current main still source-reproduces the boolean manifest path, and the linked closing PR is still open, conflicting, and needs maintainer API/proof review before this issue can close.

## What This Changes

The issue requests that native plugins declaring `contracts.tools: true` be able to register agent tools instead of receiving the missing `contracts.tools` diagnostic.

## Best Possible Solution

Resolve this through https://github.com/openclaw/openclaw/pull/80623 or a successor only if maintainers approve the manifest shorthand; otherwise keep explicit tool-name arrays and improve diagnostics/docs for invalid manifests.

## Reproduction Assessment

Yes. Source inspection shows `contracts.tools: true` normalizes to no declared tool names, after which `api.registerTool` emits the exact reported diagnostic and skips registration.

## Solution Assessment

Unclear. Treating boolean `true` as wildcard is not part of the current documented contract, so maintainers need to choose between a new shorthand mode, bounded wildcard support, or clearer invalid-manifest diagnostics.
