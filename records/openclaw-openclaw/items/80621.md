## What This Changes

The issue asks OpenClaw to accept a native plugin manifest with `contracts.tools: true` so `api.registerTool` succeeds and runtime inspection lists the registered tool.

## Best Possible Solution

Keep `contracts.tools` explicit unless maintainers approve and document a wildcard or boolean shorthand mode; otherwise fix this report with clearer invalid-manifest diagnostics and plugin-author docs.

## Reproduction Assessment

Yes. Source inspection shows `contracts.tools: true` normalizes to no declared names, after which `api.registerTool` emits the exact reported diagnostic and skips registration.

## Solution Assessment

Unclear. Treating boolean `true` as wildcard is a new manifest mode, so the maintainable path is either clearer invalid-manifest diagnostics/docs or a maintainer-approved wildcard contract with tests and real behavior proof.
