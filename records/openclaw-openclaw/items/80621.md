## What This Changes

The issue requests that native plugins declaring `contracts.tools: true` be able to register agent tools without the missing-contract diagnostic.

## Best Possible Solution

Choose one explicit contract direction: either approve and document a wildcard/boolean tool-ownership mode with tests and real runtime proof, or keep `contracts.tools` as explicit names and improve docs/diagnostics for invalid manifests.

## Reproduction Assessment

Yes. Source inspection shows boolean `contracts.tools` normalizes to no declared names, after which `api.registerTool` emits the exact reported diagnostic and skips registration.

## Solution Assessment

No, not as a narrow bug fix. Treating boolean `true` as wildcard ownership changes the public manifest semantics; the safer path is maintainer-approved API/docs work or clearer invalid-manifest diagnostics.
