## Summary

Keep open. Current main still source-reproduces the reported boolean-manifest path, and the linked closing PR is still open, so this issue should wait for that PR to land, be replaced, or be explicitly rejected by maintainers.

## What This Changes

The issue asks OpenClaw to accept a plugin manifest with `contracts.tools: true` so `api.registerTool(...)` can register tools successfully.

## Best Possible Solution

Keep `contracts.tools` explicit unless maintainers approve and document a wildcard or shorthand mode; otherwise resolve this with clearer invalid-manifest diagnostics and plugin-author docs.

## Reproduction Assessment

Yes. Source inspection shows `contracts.tools: true` normalizes to no declared names, after which `api.registerTool` emits the exact reported diagnostic and skips registration.

## Solution Assessment

No. Treating boolean `true` as wildcard ownership is not the narrowest current-contract fix unless maintainers intentionally approve that public API change; clearer diagnostics/docs are the safer fallback.
