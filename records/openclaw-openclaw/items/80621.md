## Summary

## What This Changes

The issue asks for native plugins declaring `contracts.tools: true` to be allowed to register agent tools with `api.registerTool`.

## Best Possible Solution

Keep `contracts.tools` explicit unless maintainers approve a documented wildcard or shorthand mode, and fix this report with clearer invalid-manifest diagnostics/docs or the linked PR if that API change is approved.

## Reproduction Assessment

Yes. Source inspection shows `contracts.tools: true` normalizes to no declared names, after which `api.registerTool` emits the exact reported diagnostic and skips registration.

## Solution Assessment

No. Treating boolean `true` as wildcard is not the narrowest current-contract fix; the safer path is clearer diagnostics/docs, or a maintainer-approved wildcard contract with tests and real behavior proof.
