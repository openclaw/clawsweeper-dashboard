## What This Changes

The issue requests that native plugin manifests using `"contracts": { "tools": true }` allow `api.registerTool()` to register agent tools instead of producing the missing-`contracts.tools` diagnostic.

## Best Possible Solution

Keep this issue open until maintainers either approve a documented wildcard/shorthand contract through https://github.com/openclaw/openclaw/pull/80623 or choose a narrower diagnostics/docs fix for invalid manifest shapes.

## Reproduction Assessment

Yes. Source inspection shows `contracts.tools: true` normalizes to no declared names, after which `api.registerTool` records the exact reported diagnostic and skips registration.

## Solution Assessment

No, not yet. Treating boolean `true` as wildcard ownership is a new public manifest mode; the maintainable solution needs maintainer approval, docs/tests, and real behavior proof, or else a narrower invalid-manifest diagnostic/docs update.
