## Summary

Keep open: current main still exposes the queue failure as a boolean plus low-context debug log, and the paired closing pull request is open and unmerged. The existing discussion and https://github.com/openclaw/openclaw/pull/80160 are useful implementation context, but the issue should not close until that PR lands or the remaining tool/health notification criteria are split.

## What This Changes

Gateway/operator failure notifications should carry actionable queue/tool/agent context such as reason, session/run identifiers, fatality, and gateway health instead of only a generic agents-failed alert.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/80160 for the queue-diagnostic portion, then keep the remaining tool-failure and gateway-health notification criteria tracked here or split into narrower follow-ups.
