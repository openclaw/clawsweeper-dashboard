## Summary

Keep this PR open, but the patch should not merge as-is because the remaining behavior change makes unauthorized text reset commands emit visible chat replies where current command authorization is fail-silent unless the command source is native.

## What This Changes

The branch changes auto-reply reset handling to return a visible unauthorized reset denial, adds group/direct/reset-hook regression coverage, and updates the changelog.

## Best Possible Solution

Keep the reset acknowledgement coverage, but preserve silent unauthorized text reset failures unless maintainers approve and document a reset-specific visible-denial policy.

## Reproduction Assessment

Yes, for the review finding: current main returns no reply for unauthorized text reset commands, while the PR head returns a visible denial in both reset branches by source inspection. I did not run tests in this read-only review.

## Solution Assessment

No, not as-is. The acknowledgement coverage is reasonable, but the visible unauthorized-denial branch changes authorization policy and should either be removed or explicitly approved and documented by maintainers.
