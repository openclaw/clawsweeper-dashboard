## Summary

Keep open: the PR has useful reset acknowledgement coverage and supplied live proof, but it still changes unauthorized text reset commands from fail-silent to visible replies, which needs maintainer policy/security approval before merge.

## What This Changes

The PR updates auto-reply reset handling to return visible unauthorized reset denials, adds group/direct reset acknowledgement tests, keeps routed hook replies from duplicating acknowledgements, and adds a changelog entry.

## Best Possible Solution

Keep the reset acknowledgement tests, but preserve silent unauthorized text reset failures unless maintainers approve and document a reset-specific denial policy.

## Reproduction Assessment

Yes, for the review finding by source inspection: current main returns no reply for unauthorized text reset commands, while the PR returns a visible denial in both reset branches. This read-only review did not run the targeted Vitest command.

## Solution Assessment

No, not as-is. The acknowledgement coverage is reasonable, but the unauthorized denial is a user-visible authorization-policy change and should either be removed or explicitly approved and documented by maintainers.
