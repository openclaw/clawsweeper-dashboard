## Summary

Keep open: the PR has useful Feishu group-chat proof and a plausible unauthorized-feedback change, but it also makes direct-chat bare reset commands fall through to model execution despite current documented behavior and current-main tests, and it lacks the required changelog entry. ## What This Changes

The PR changes reset command handling to return visible unauthorized-reset errors, limit built-in bare `/reset` and `/new` success acknowledgments to group chats, and add reset-hook tests.

## Best Possible Solution

Preserve the current no-model bare reset acknowledgement contract for non-ACP chats, add explicit group and unauthorized reset coverage if desired, and align the changelog and any docs with the chosen direct-chat behavior.

## Solution Assessment

No. The unauthorized-feedback part is narrow, but limiting built-in acknowledgments to group chats reopens direct empty reset model runs despite current docs and tests; the safer fix is to preserve direct no-model acknowledgments unless maintainers intentionally change that contract.
