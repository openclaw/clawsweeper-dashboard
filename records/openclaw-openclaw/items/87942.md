## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass the npm prefix into active-host repair:** `src/plugins/install.ts:842-845`
  - body: Here the PR snapshots the active host with `npmRoot`, but this call site's `npmRoot` is the per-plugin project produced by `resolvePluginNpmProjectDir`. The helper then looks for `<npmRoot>/lib/node_modules/openclaw`, so a host installed under the configured prefix and a plugin installed under `projects/<hash>` are missed and the active host dependencies can remain pruned after a successful plugin install.
  - confidence: 0.93
- **[P1] Keep JSON control escapes valid:** `src/llm/utils/json-parse.ts:3`
  - body: Removing `b`, `f`, `n`, `r`, and `t` from `VALID_JSON_ESCAPES` makes `repairJson` treat valid JSON escapes like `\n` and `\t` as invalid unless they look like Windows paths. That changes parsed values for ordinary valid JSON and contradicts the existing regression test that preserves line and tab escapes.
  - confidence: 0.94
- **[P1] Do not reinstall unchanged npm updates without approval:** `src/plugins/update.ts:1245-1267`
  - body: The added maintenance block runs the npm installer even when the registry probe says the installed artifact is unchanged. Current docs promise this path skips downloading, reinstalling, and config rewrites, so existing update commands can now fail on network/install/integrity behavior where they previously returned unchanged unless maintainers explicitly accept and document the compatibility change.
  - confidence: 0.88
