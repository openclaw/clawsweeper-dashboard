## Summary

Keep open: current `main` and the latest release still leave SecretRef-managed dashboard tokens non-tokenized with no dedicated recovery command, and the only linked fix PR was closed unmerged after proof and security-boundary concerns.

## What This Changes

The issue asks for an operable way to recover or avoid pasting a SecretRef-managed gateway token when Control UI sessionStorage is gone.

## Best Possible Solution

Land one maintainer-approved recovery path: session/exchange-token dashboard auth, or an explicit tested copy-token mode that never logs the bearer and updates the dashboard docs.

## Reproduction Assessment

Yes, source-reproducible. Current `dashboardCommand` keeps SecretRef-managed URLs non-tokenized, `runConfigGet` reads the redacted config object instead of resolving the secret, and tests assert clean URL delivery for SecretRef cases.

## Solution Assessment

Unclear. The issue is valid, but maintainers need to choose the security boundary before implementation: session/exchange-token auth is cleaner, while a copy-token command is a narrower interim recovery path with explicit exposure tradeoffs.
