## Summary

Keep open: current main and v2026.6.9 still lack an operable SecretRef token recovery path for dashboard login, and the direct fix PR closed unmerged after functional proof and security-boundary concerns.

## What This Changes

The issue requests a documented, usable way to authenticate the Control UI when `gateway.auth.token` is SecretRef-managed and browser session storage is gone.

## Best Possible Solution

Land one maintainer-approved recovery path: preferably session/exchange-token auth, or an explicit copy-token command that never logs the bearer and ships with focused tests, docs, and redacted Control UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a SecretRef-managed `gateway.auth.token`, `openclaw dashboard` opens/copies a non-tokenized URL and `config get` returns config data rather than plaintext.

## Solution Assessment

Unclear until maintainers choose the security boundary. A copy-token flag is plausible, but session/exchange-token auth may be the cleaner long-term fix.
