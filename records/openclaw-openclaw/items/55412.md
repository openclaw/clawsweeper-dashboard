## Summary

Keep open: current main still resolves GatewayDrainingError to a terminal restart reply instead of entering the retry loop, and the earlier retry PR is closed unmerged.

## What This Changes

GatewayDrainingError should be retried during gateway restart drain windows instead of surfacing as a user-visible agent failure.

## Best Possible Solution

Add a bounded, typed GatewayDrainingError retry at the agent-runner boundary, preserve command-queue drain rejection semantics, and update focused regression coverage for retry-success and retry-exhaustion behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main rejects enqueues during gateway drain, and current runner tests assert that a fallback-wrapped GatewayDrainingError returns restart text instead of retrying.

## Solution Assessment

Yes. The narrow maintainable fix is a one-shot or otherwise bounded retry around the existing run loop for the concrete GatewayDrainingError path, not a queue-level migration or broad startup replay feature.
