## Summary

Keep open for maintainer review and merge. Current main and v2026.5.18 still have the source-reproducible fallback-classification gap, while this PR targets the right boundary with no blocking review findings and supplied terminal proof.

## What This Changes

The PR classifies embedded Pi provider business-denial `isError` payloads through the shared failover matcher before non-GPT results can be treated as success, and adds focused matcher/classifier regression tests.

## Best Possible Solution

Land the focused classifier and matcher repair after normal maintainer validation, then let it close https://github.com/openclaw/openclaw/issues/48680.

## Reproduction Assessment

Yes. Current main is source-reproducible: a non-GPT embedded result whose only signal is the reported unclassified `isError` payload returns `null` from the classifier, and fallback treats a `null` classification as success.

## Solution Assessment

Yes. The PR uses the existing failover reason matcher at the embedded-result boundary before the non-GPT success path, which is narrower and more maintainable than adding a provider-specific fallback coordinator path.
