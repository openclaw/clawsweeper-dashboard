## Summary

Keep open. Current main still lacks the useful ACP final emit, harness diagnostic visibility, and legacy systemd-unit fallback, but the PR is not merge-ready because the discussion contains an explicit maintainer hold plus two concrete branch-specific correctness defects.

## What This Changes

The PR adds ACP child final-output buffering and normalized final system events, process-local harness selection diagnostics in `/acp doctor`, and legacy `CLAWDBOT_SYSTEMD_UNIT` fallback in gateway systemd/restart helpers with tests.

## Best Possible Solution

Land a revised version that keeps the newest ACP child output under the final-buffer cap, records strict harness failures as none/failed or skips those diagnostics, preserves the legacy systemd-unit fallback with escaping/precedence coverage, and leaves the broader split-install reliability work to #74047.

## Reproduction Assessment

Yes. The branch-specific defects have source-level reproduction paths: stream more than 12k non-commentary ACP output with the final marker at the end, and force a missing harness with fallback disabled while inspecting the new diagnostic ring.

## Solution Assessment

No. The PR is aimed at the right surfaces, but the current patch is not the best mergeable solution until the final buffer is tail-preserving and strict no-fallback diagnostics stop reporting a PI run that never happened.
