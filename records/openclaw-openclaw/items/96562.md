## What This Changes

The PR changes the session tool path renderer to shorten only exact or separator-bounded home paths and adds focused regression tests.

## Best Possible Solution

Land the narrow session renderer helper and regression-test fix after ordinary maintainer review, while keeping broader path-display policy work separate.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 rewrite any string starting with os.homedir(), so a sibling path such as `${home}extra/app.ts` would be displayed as `~extra/app.ts`; I did not run a live TUI scenario in this read-only review.

## Solution Assessment

Yes. The PR fixes the implicated local session renderer and matches the separator-bounded behavior already used by the shared path helper without changing broader OPENCLAW_HOME or full-absolute-path policy.
