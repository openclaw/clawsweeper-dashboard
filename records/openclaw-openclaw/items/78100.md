## What This Changes

The issue reports slow-filesystem synchronous plugin discovery in gateway status/restart paths causing event-loop stalls and WhatsApp 408 reconnect loops.

## Best Possible Solution

Review and either land or explicitly reject the linked implementation PRs, then close this issue only after the accepted fix is on main or maintainers decide the current memoization is sufficient.

Yes, source-backed rather than live-run: current main still routes gateway status/start/send paths through synchronous auto-enable and registry metadata loading, and the issue plus linked PRs provide concrete slow-filesystem WhatsApp logs.

## Solution Assessment

Unclear whether current main alone is sufficient. The maintainable end state is to remove or isolate synchronous discovery from gateway hot paths, so the linked async and coalescing PRs should be reviewed rather than starting a separate repair lane.
