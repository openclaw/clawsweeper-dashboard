This PR should stay open because the central Discord teardown race is not implemented on current main, but the current branch is not merge-ready: prior review and this pass both found a startup fail-open regression, mixed scope, and missing real after-fix Discord proof.

## Best Possible Solution

Rebase and narrow the work to a Discord-owned teardown fix that preserves fatal bot identity startup failures, adapts idle draining to the current `createChannelRunQueue` and `provider.cleanup` architecture, and splits unrelated Telegram/plugin type edits if they are still needed.
