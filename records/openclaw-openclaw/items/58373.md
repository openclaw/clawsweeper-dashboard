## Summary

Keep open: current main and the latest release still lack the non-main skip-branch bootstrap, and this PR is the active fix path, but it should not merge until the credential-boundary behavior is sanitized or explicitly approved.

## What This Changes

The PR adds a non-main agent `models.json` bootstrap path from the main agent when model planning skips, plus focused regression coverage for missing, existing, empty-source, retry, and concurrent-write cases.

## Best Possible Solution

Keep the narrow skip-branch bootstrap, but resolve the credential boundary by writing lookup-safe catalog data, reusing existing auth fallback, or recording explicit maintainer/security approval for raw same-user copying before landing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.create` can leave a non-main agent without `models.json`, an empty provider plan returns `skip`, current main writes nothing, and the embedded runner can then throw `Unknown model`.

## Solution Assessment

No, not as-is. The skip-branch bootstrap is the right narrow direction, but verbatim copying of the main `models.json` needs sanitization or explicit maintainer/security approval before merge.
