## Summary

Keep open. The PR addresses a real missing-registry failure and includes sufficient after-fix logs, but the submitted implementation still copies raw main-agent `models.json` bytes into a secondary agent; current schemas/docs treat provider `apiKey` and headers as sensitive, and the existing auth-profile inheritance model is read-through rather than cloning.

## What This Changes

The PR adds a skip-branch bootstrap in `ensureOpenClawModelsJson` to create a missing non-main agent `models.json` from the main agent and adds focused regression tests.

## Best Possible Solution

Land a narrow non-main missing-catalog bootstrap that preserves model catalog availability without broadening persisted credential copies, or document and explicitly approve the same-user raw-copy boundary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a non-main agent, enter provider planning with zero providers, current main returns from the skip branch without creating the registry, and embedded model resolution can then throw `Unknown model`.

## Solution Assessment

No, not as submitted. The one-time bootstrap direction is narrow and the added tests address cache/concurrency cases, but raw credential propagation needs sanitization or explicit maintainer/security acceptance before merge.

## Review Findings

Overall correctness: patch is incorrect
