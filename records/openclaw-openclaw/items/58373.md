## Summary

Keep open for maintainer/security review. The PR targets a real source-proven non-main agent startup failure and now has sufficient real behavior proof plus green CI, but the submitted fix still copies the main agent models.json bytes into a secondary agent, which may duplicate usable provider credentials or sensitive headers.

## What This Changes

The PR adds a skip-branch non-main agent models.json bootstrap from the main/default agent catalog with create-if-missing write logic and regression tests for inheritance, retry, and concurrent creation.

## Best Possible Solution

Land the narrow one-time bootstrap after credential portability is handled deliberately, preferably by copying only catalog metadata or by recording explicit maintainer acceptance for raw same-user models.json copies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: agents.create can create a non-main agent without models.json, empty provider planning returns skip, current main exits without producing the registry, and model resolution can then report Unknown model.

## Solution Assessment

No, not ready as submitted. The one-time bootstrap is the right scope and the cache/concurrency issues appear addressed, but copying raw models.json content needs sanitization or explicit maintainer security acceptance before merge.

## Review Findings

Overall correctness: patch is incorrect
