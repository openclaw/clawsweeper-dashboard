## Summary

Keep open. Current main still has the reported non-main `models.json` gap, and this PR is a plausible implementation candidate linked to the open bug in #58289, but the submitted head is not merge-ready: it is dirty against the current base, is built on older `models-config` cache/provider-discovery plumbing, lacks the required changelog entry for a user-facing fix, and needs an explicit maintainer decision on copying raw `models.json` contents between agents.

## What This Changes

The PR adds a one-time non-main agent `models.json` bootstrap from the main agent when model planning returns `skip`, updates the readiness fingerprint for main `models.json` changes, and adds regression tests for bootstrap, retry, and concurrent-create behavior.

## Best Possible Solution

Port the one-time non-main bootstrap onto current main's `ensureOpenClawModelsJson` implementation, preserving workspace/plugin/provider-discovery fingerprint inputs and cache-key behavior; then make an explicit credential-boundary choice for raw `models.json` copying versus a secret-safe catalog bootstrap, add focused tests and a changelog entry, and land it as the implementation for #58289.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: `agents.create` leaves the non-main agent without `models.json`, and current main's `skip` branch returns without creating one when provider planning is empty.

## Solution Assessment

No, not as currently submitted. The narrow bootstrap direction is reasonable, but the branch must be rebased onto current models-config internals and the raw credential-copy behavior must be accepted or made secret-safe before merge.
