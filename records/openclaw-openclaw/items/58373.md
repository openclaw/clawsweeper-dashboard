## Summary

Keep open. The PR targets a valid source-reproducible non-main agent startup bug and includes after-fix real behavior proof, but the submitted approach still needs maintainer/security judgment because it copies the main agent's raw `models.json` into another agent directory, which can propagate usable provider credentials.

## What This Changes

The PR adds a skip-branch helper in `src/agents/models-config.ts` to bootstrap a missing non-main `models.json` from the main agent and adds focused regression tests for inheritance, retry, and concurrent creation.

## Best Possible Solution

Land a narrow missing-catalog bootstrap for non-main agents only after the credential-copy boundary is made explicit, preferably by preserving catalog metadata without duplicating non-portable secret material unless maintainers accept the same-user raw-copy policy.

## Reproduction Assessment

Yes. The source path is high confidence: `agents.create` can create a non-main agent without `models.json`, empty provider planning returns `skip`, current main returns without creating the registry, and model resolution can then emit `Unknown model`.

## Solution Assessment

No, not ready as submitted. The one-time bootstrap direction is narrow and the cache/atomic/concurrency concerns appear addressed, but raw credential propagation needs sanitization or explicit maintainer/security acceptance before merge.
