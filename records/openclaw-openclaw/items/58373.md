## Summary

Keep open. The reported bug is valid and the PR is narrowly aimed, with sufficient real behavior proof and green CI, but the current diff copies the main agent `models.json` bytes verbatim into a secondary agent; that file can contain usable provider credentials, so this needs sanitization or explicit maintainer/security acceptance before merge.

## What This Changes

The PR adds a skip-branch bootstrap that creates a missing non-main agent `models.json` from the main agent catalog and adds focused regression tests for inheritance, retry, and concurrent creation.

## Best Possible Solution

Keep the one-time missing-catalog bootstrap, but make the credential-copy boundary explicit by sanitizing credential-bearing fields or getting maintainer/security approval for the raw same-user copy policy.

## Reproduction Assessment

Yes. Source inspection shows `agents.create` can leave a non-main agent without `models.json`, empty provider planning returns `skip`, current main returns without creating the catalog, and model resolution can then produce `Unknown model`.

## Solution Assessment

No, not ready as submitted. The bootstrap direction is narrow and the cache/atomic/concurrency concerns appear addressed, but writing `sourceRaw` verbatim is not the safest maintainable solution until the credential boundary is resolved.
