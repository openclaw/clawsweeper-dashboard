## Summary

Keep this PR open: it addresses a real non-main agent startup bug and the latest branch fixes the earlier atomic-write, retry, and concurrency concerns, but the current implementation still copies the main agent's raw `models.json` bytes into another agent and needs maintainer/security judgment before merge.

## What This Changes

The PR changes `ensureOpenClawModelsJson` so the empty-provider `skip` branch can bootstrap a missing non-main agent `models.json` from the main agent, with regression coverage for copy, retry, and concurrent-create cases.

## Best Possible Solution

Land a one-time non-main missing-catalog bootstrap that preserves existing per-agent files and cache behavior while deliberately handling credentials, preferably by copying catalog metadata without duplicating raw secret/header values unless maintainers explicitly accept that boundary.

## Reproduction Assessment

Yes. The source-level path is high confidence: `agents.create` does not create `models.json`, empty provider planning returns `skip`, current main returns without creating the file, and model resolution can later report `Unknown model`.

## Solution Assessment

No, not ready as submitted. The bootstrap direction is narrow and the earlier cache/concurrency problems are addressed, but raw `models.json` credential propagation needs sanitization or explicit maintainer/security acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

  - body: `inheritModelsJsonFromMainAgent` copies the main agent's `models.json` bytes into a secondary agent. Please sanitize credential/header fields during bootstrap or get explicit maintainer security acceptance for this same-user copy boundary.
