## Summary

Keep open. The bug is valid and the branch is close, but it is not ready because the current implementation can duplicate raw model-provider credentials from the main agent catalog into secondary agent files.

## What This Changes

The branch adds a non-main skip-branch models.json bootstrap from the main agent catalog and regression coverage for bootstrap, retry, and concurrent creation.

## Best Possible Solution

Land a targeted non-main missing-catalog bootstrap that copies only portable catalog metadata or SecretRef markers, preserves existing per-agent files, and keeps secrets on the existing auth read-through path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: agents.create can create a non-main agent without models.json, empty provider planning returns skip, current main returns without creating the registry, and runtime model resolution can then report Unknown model.

## Solution Assessment

No, not ready as submitted. The one-time bootstrap direction is narrow and the latest branch addresses cache/concurrency concerns, but raw byte-copying the main models.json needs sanitization or explicit maintainer security acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid cloning raw model credentials:** `src/agents/models-config.ts:209`
  - body: `inheritModelsJsonFromMainAgent` writes the main agent's `models.json` bytes into a secondary agent. That file can contain usable custom provider `apiKey` values or sensitive provider headers, while the existing auth-profile inheritance path deliberately reads inherited credentials without cloning them into the secondary store. Please copy only portable catalog metadata/SecretRef markers, or get explicit maintainer security acceptance for raw same-user credential duplication.
  - confidence: 0.78
