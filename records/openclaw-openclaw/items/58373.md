## Summary

Keep open: the bug remains on current main and the latest release, and this PR has a plausible narrow fix, but it still needs maintainer/security resolution on whether verbatim main-agent model catalog copying is an approved credential boundary.

## What This Changes

The PR adds skip-branch inheritance from the main agent `models.json` for missing non-main agent catalogs and adds focused regression tests for inheritance, retry, and concurrent creation.

## Best Possible Solution

Keep the narrow skip-branch bootstrap, but either write lookup-safe catalog data that relies on existing auth fallback or record explicit maintainer/security approval for verbatim main-to-secondary `models.json` propagation before landing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.create` can leave a non-main agent without `models.json`, an empty provider plan returns `skip`, current main writes nothing, and the embedded runner can then throw `Unknown model`.

## Solution Assessment

No, not as-is. The skip-branch bootstrap is the right narrow direction, but verbatim copying of the main `models.json` needs sanitization or explicit maintainer/security approval before merge.
