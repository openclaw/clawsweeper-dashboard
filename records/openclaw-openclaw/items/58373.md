## Summary

Keep open. The bug and the narrow bootstrap direction are well supported, but this PR still needs maintainer/security judgment because the new path copies raw main-agent `models.json` bytes into a secondary agent, including sensitive provider auth fields.

## What This Changes

The PR adds a skip-branch helper that bootstraps a missing non-main agent `models.json` from the main agent catalog and adds regression tests for bootstrap, retry, and concurrent creation.

## Best Possible Solution

Land a one-time missing-catalog bootstrap that preserves per-agent isolation and cache behavior while handling credential portability deliberately, either by sanitizing to catalog metadata or by explicit maintainer/security acceptance of same-user raw catalog copies.

## Reproduction Assessment

Yes. The source-level path is high confidence: `agents.create` can create a non-main agent without `models.json`, zero-provider planning returns `skip`, and current main returns without creating the registry before model lookup can surface `Unknown model`.

## Solution Assessment

No, not ready as submitted. The bootstrap placement is narrow and the cache/concurrency guardrails are improved, but copying raw `models.json` credential material needs sanitization or explicit maintainer/security approval before merge.

## Review Findings

Overall correctness: patch is incorrect
