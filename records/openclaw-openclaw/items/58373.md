## Summary

Keep open: the reported bug path and the PR’s narrow bootstrap direction are credible, and the PR body supplies adequate after-fix log proof. The blocker is that the current patch copies the main agent’s raw `models.json` bytes into a secondary agent, which can duplicate credential-bearing provider fields without an explicit maintainer/security decision.

## What This Changes

This PR adds a skip-branch bootstrap that copies the main agent `models.json` into a missing non-main agent `models.json` and adds regression tests for inheritance, retry, and concurrent creation.

## Best Possible Solution

Keep the one-time non-main missing-catalog bootstrap, but make credential propagation explicit by sanitizing credential-bearing fields or getting maintainer/security acceptance for the raw-copy boundary.

## Reproduction Assessment

Yes. The source path is high confidence: `agents.create` can create a non-main agent without `models.json`, empty provider planning returns `skip`, current main leaves the file missing, and model resolution can then produce `Unknown model`.

## Solution Assessment

No, not ready as submitted. The bootstrap direction is narrow, but the raw credential propagation needs sanitization or explicit maintainer/security acceptance before merge.
