## Summary

Keep open. The bug and the narrow bootstrap direction are valid, but this PR should not merge as-is because the bootstrap copies raw main-agent `models.json` bytes into a secondary agent directory, including possible provider credentials or sensitive headers.

## What This Changes

The branch updates `src/agents/models-config.ts` to bootstrap a missing non-main agent `models.json` from the main agent when planning returns `skip`, with tests for inheritance, retry, and concurrent creation.

## Best Possible Solution

Keep the one-time missing-catalog bootstrap, but copy only catalog-safe fields or get explicit maintainer/security approval for raw credential propagation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a non-main agent, have provider planning return no providers, then start the embedded runner path; current main returns `skip` without creating the registry and can still surface `Unknown model`.

## Solution Assessment

No, not as submitted. The cache, atomic-write, retry, and concurrency guardrails are present, but copying raw main-agent `models.json` bytes needs sanitization or explicit maintainer/security acceptance before merge.
