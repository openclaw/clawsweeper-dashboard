## Summary

Keep open. The reported bug is real and the branch now covers the cache, retry, and concurrent-create cases, but the current fix still copies the main/default agent `models.json` bytes into a secondary agent and needs maintainer/security judgment on credential propagation before merge.

## What This Changes

The PR adds a skip-branch bootstrap in `ensureOpenClawModelsJson` that creates a missing non-main agent `models.json` from the main/default agent catalog, with regression coverage for inheritance, retry, and concurrent creation.

## Best Possible Solution

Land a narrow missing-catalog bootstrap for non-main agents, but handle credential material deliberately: either write catalog metadata with auth resolved through existing inheritance/markers or document explicit maintainer acceptance of same-user raw catalog copies.

## Reproduction Assessment

Yes. The source-level path is high confidence: `agents.create` can create a non-main agent without `models.json`, empty provider planning returns `skip`, current main returns without creating the catalog, and model resolution can then surface `Unknown model`.

## Solution Assessment

No, not ready as submitted. The bootstrap direction is narrow and the added tests cover prior cache/concurrency concerns, but copying raw `models.json` credential material needs sanitization or explicit maintainer/security acceptance before merge.

## Review Findings
