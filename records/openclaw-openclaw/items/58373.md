## Summary

Keep open. The bug remains on current main and this PR addresses the right narrow skip path, but the current diff copies raw main-agent `models.json` bytes into secondary agent directories; current source and docs treat model `apiKey`/header fields as sensitive and SecretRef-managed writes as marker-based, so the PR needs sanitization or explicit maintainer/security acceptance before merge.

## What This Changes

The PR adds a skip-branch bootstrap that copies a missing non-main agent `models.json` from the main agent and adds regression coverage for inheritance, retry, and concurrent creation.

## Best Possible Solution

Keep the narrow one-time missing-catalog bootstrap, but copy only catalog-safe fields or get explicit maintainer/security approval for raw credential propagation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.create` can leave a non-main agent without `models.json`, empty provider planning returns `skip`, and runtime model resolution can then surface `Unknown model`.

## Solution Assessment

No, not as submitted. The atomic, retry, and concurrency guardrails are present, but copying raw main-agent `models.json` bytes needs sanitization or explicit maintainer/security acceptance before merge.
