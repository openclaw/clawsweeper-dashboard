## Summary

Keep open: the PR is a focused fix for a real non-main agent startup bug and now includes real behavior proof, but it still needs maintainer/security approval or a safer implementation for raw models.json copy semantics before merge.

## What This Changes

The PR adds a skip-branch bootstrap in models-config that creates a missing non-main agent models.json from the main/default agent catalog and adds regression coverage for bootstrap, retry, and concurrent creation.

## Best Possible Solution

Land a one-time non-main missing-catalog bootstrap that preserves current cache/provider-discovery behavior and handles credential portability deliberately, preferably by copying catalog metadata without duplicating non-portable secret material unless maintainers explicitly accept same-user raw copies.

## Reproduction Assessment

Yes. The source-level path is high confidence: agents.create creates the non-main agent without models.json, zero-provider planning returns skip, current main returns without creating the catalog, and model resolution can then emit Unknown model.

## Solution Assessment

No, not ready as submitted. The one-time bootstrap direction is narrow and the previous cache/concurrency concerns are addressed, but copying raw models.json credential material needs maintainer/security approval or sanitization before merge.

## Review Findings

Overall correctness: patch is incorrect
