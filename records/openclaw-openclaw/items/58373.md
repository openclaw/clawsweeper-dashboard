## Summary

Keep open: the PR addresses a real current-main bug, but it still needs maintainer/security approval or a sanitizing change because the bootstrap copies raw main-agent `models.json` bytes into a secondary agent directory.

## What This Changes

The PR adds a skip-branch bootstrap that copies the main agent `models.json` into a missing non-main agent registry and adds targeted regression tests for copy, retry, and concurrent creation.

## Best Possible Solution

Keep the one-time missing-catalog bootstrap, but copy only catalog-safe fields or get explicit maintainer/security approval for same-user raw catalog propagation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a non-main agent, have provider planning return no providers, then start the embedded runner path; current main returns `skip` without creating the registry and can still surface `Unknown model`.

## Solution Assessment

No, not as submitted. The retry, atomic missing-write, and concurrency guardrails are present, but copying raw main-agent catalog bytes needs sanitization or explicit maintainer/security approval before merge.
