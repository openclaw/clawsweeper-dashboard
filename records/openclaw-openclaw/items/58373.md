## Summary

Keep open: the PR addresses a valid source-reproducible bug and the latest revision appears to have handled the earlier cache, atomicity, and concurrency concerns, but it still needs maintainer/security judgment before merge because the bootstrap copies raw main-agent `models.json` bytes into another agent.

## What This Changes

The PR adds a skip-branch `models.json` bootstrap from the main agent for non-main agents, plus regression tests for inheritance, retry after main catalog creation, and concurrent target creation.

## Best Possible Solution

Land the narrow skip-branch non-main bootstrap, but make the credential boundary explicit by sanitizing secrets/headers or documenting maintainer acceptance of same-user raw catalog copies.

## Reproduction Assessment

Yes. The current-main source path is high confidence: `agents.create` can leave a non-main agent without `models.json`, empty provider planning returns `skip`, `ensureOpenClawModelsJson` returns without creating the catalog, and later model resolution can emit `Unknown model`.

## Solution Assessment

No, not ready as submitted. The one-time bootstrap direction is narrow and the cache/atomic/concurrency fixes look addressed, but copying raw main-agent model catalog credential material needs sanitization or explicit maintainer/security acceptance.

## Review Findings

Overall correctness: patch is incorrect
