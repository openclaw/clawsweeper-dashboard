## Summary

Keep this PR open: it targets a valid current and shipped preflight compaction bug, but the submitted fix is not merge-ready because it feeds the configured timeout back through the external-abort path, lacks real runtime proof, and commits issue-specific proof artifacts.

## What This Changes

The PR changes budget/preflight compaction to pass a composed reply-operation signal plus configured timeout signal, and adds one repro script plus evidence markdown under `scripts/repro`.

## Best Possible Solution

Land a narrow repair that keeps compaction timeout ownership in the safety wrapper, defines the allowed preflight cancellation signal, removes PR-only proof artifacts, adds focused regression coverage, and includes redacted runtime proof.

## Solution Assessment

No as submitted. The call site is the right area, but the safer fix should keep the compaction timeout in the existing safety wrapper or pass a dedicated cancellation signal rather than injecting a second timeout through the external-abort parameter.
