Keep open: current main and the latest release still have the Control UI picker path where hidden subagent rows can consume raw pagination while the picker shows only navigable chat rows. ## What This Changes

The issue requests that the Control UI chat session picker remain navigable and show meaningful counts when many spawn-child sessions dominate sessions.list pagination.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/89323 for the UI pagination/count fix, while keeping optional subagent visibility controls in https://github.com/openclaw/openclaw/issues/89774 separate.

## Reproduction Assessment

Yes at source level: current main filters subagent/spawned rows after raw sessions.list pagination and Load More fetches only one raw page per click, so a hidden-only page can leave the visible picker unchanged. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes for the existing UI PR path: advancing through hidden-only pages and counting visible rows is the narrow maintainable fix without adding config or protocol surface. The CLI cursor PR is useful parity work but not a standalone solution for the Control UI issue.
