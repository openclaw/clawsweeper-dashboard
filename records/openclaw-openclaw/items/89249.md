## Summary

Keep open: current main still lets hidden spawned/subagent rows consume raw Control UI picker pages while the picker shows only navigable rows and raw total counts. The direct UI fix is already represented by an open closing PR, so this issue should remain open until that fix lands, is replaced, or is explicitly rejected.

## What This Changes

Fix the Control UI chat session picker so agents with many spawned subagent sessions can still navigate real chat sessions and get sane pagination/count behavior.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/89323 for the UI pagination/count fix, while keeping CLI cursor parity and optional subagent reveal controls separate.

## Reproduction Assessment

Yes at source level: current main filters inactive spawned/subagent rows after raw sessions.list pagination and loads only one raw page per Load More click, so a hidden-only page can leave the visible picker unchanged. I did not run a live browser reproduction because this cleanup review is read-only.

## Solution Assessment

Yes for the open UI PR path: advancing through hidden-only pages and displaying visible-row counts is the narrow maintainable fix without adding Gateway, config, or plugin API surface. The CLI cursor PR is useful parity work but not a standalone solution for the Control UI bug.
