## Summary

Keep open: current main and the latest release still count/page raw session rows while the chat picker hides subagent rows afterward, and an open proof-positive UI PR explicitly targets the remaining dead-end behavior.

## What This Changes

The report asks for Control UI chat session picker pagination/counting to stay navigable when many spawn-child subagent sessions fill raw sessions.list pages.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/89323 for the UI pagination/count fix, while leaving optional subagent visibility controls to https://github.com/openclaw/openclaw/issues/89774.

## Reproduction Assessment

Yes at source level: current main hides subagent/spawnedBy rows after raw sessions.list pagination and displays the raw total count, so a hidden-only page can leave the visible picker unchanged. I did not run a live Control UI reproduction because this review is read-only.

## Solution Assessment

Yes for the open UI PR path: fixing the picker to advance through hidden-only pages and count visible rows addresses the remaining bug without changing sessions.list protocol. The CLI-only nextOffset PR is useful parity work but not the best standalone fix for this issue.
