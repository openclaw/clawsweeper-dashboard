## Summary

Keep open: current main and the latest release still have the Control UI picker dead-end where hidden subagent rows can consume raw pagination pages while the picker shows only navigable chat rows. The right fix path is already represented by the open UI PR with sufficient proof, so this issue should stay open until that PR lands or maintainers choose a different fix.

## What This Changes

The issue reports that the Control UI chat session picker becomes un-navigable when many spawned subagent sessions dominate `sessions.list` pagination.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/89323 for the UI pagination/count fix, while keeping optional subagent visibility controls in https://github.com/openclaw/openclaw/issues/89774 separate.

## Reproduction Assessment

Yes. Source inspection shows current main filters subagent/spawned rows after raw `sessions.list` pagination and still displays raw total counts, so a hidden-only page can leave the visible picker unchanged; I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes for the open UI PR path. Fixing the picker to advance through hidden-only pages and count visible rows addresses the remaining bug without adding config, protocol, or product surface; the CLI cursor PR is useful parity work but not a standalone fix for this issue.
